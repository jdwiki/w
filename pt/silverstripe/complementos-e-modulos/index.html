A instalação dos módulos é recomendada através do uso do Composer

Módulo de Extensões de Campo de Grade SilverStripe #

O SilverStripe Grid Field Extensions Module tem alguns recursos muito bons para aprimorar o GridField básico…

  • GridFieldAddExistingSearchButton - um formulário de pesquisa mais avançado para adicionar itens
  • GridFieldAddNewInlineButton - se baseia em GridFieldEditableColumns para permitir a criação inline de registros.
  • GridFieldAddNewMultiClass - permite que o usuário selecione de uma lista de classes para criar um novo registro de
  • GridFieldEditableColumns - permite a edição em linha de registros
  • GridFieldOrderableRows - ​​arraste e solte a reordenação de linhas
  • GridFieldRequestHandler - uma classe de utilitário básica que pode ser usada para criar visualizações de detalhes de campos de grade personalizados, incluindo guias, trilhas de pão e outros recursos do CMS
  • GridFieldTitleHeader - um cabeçalho simples que exibe os títulos das colunas

Mais documentação pode ser encontrada em o módulo aqui.

Melhores botões para GridField #

O módulo Better Buttons for GridField adiciona novas ações e botões de formulário ao formulário de detalhes do GridField.

  • Salvar e adicionar outro: Crie um registro, e vá direto para adicionar outro, sem precisar clicar no botão voltar, e depois adicionar novamente
  • Salvar e fechar: Salve o registro e volte para a visualização de lista
  • Exclusão amigável: Extraído da bandeja de ações construtivas e afastado, é menos provável que seja clicado acidentalmente. Inclui confirmação em linha da ação em vez da caixa de alerta do navegador
  • Cancelar: Igual ao botão Voltar, mas em um local mais conveniente
  • Registro anterior/próximo: Navegue para o registro anterior ou seguinte na lista sem retornar à visualização de lista
  • e muitos mais…

Mais documentação (e imagens) na documentação do módulo

Formulários de usuário #

O módulo UserForms permite que os usuários do CMS criem formulários dinâmicos por meio de uma interface de arrastar e soltar e sem se envolver em nenhum código PHP.

Principais características

  • Construa um formulário usando todos os principais campos do formulário (texto, e-mail, menu suspenso, rádio, caixa de seleção…)
  • Capacidade de estender userforms de outros módulos para fornecer campos extras.
  • Capacidade de enviar e-mail para várias pessoas o envio do formulário
  • Visualize os envios enviados e exporte-os para CSV
  • Definir mensagens de erro personalizadas e configurações de validação
  • Opcionalmente, exiba e oculte campos usando javascript com base na entrada do usuário
  • Exibe uma mensagem de confirmação ao sair de um formulário parcialmente preenchido

Mais links de documentação podem ser encontrados aqui no repositório do github

Lógica de exibição #

O [módulo de lógica de exibição] 1 permite adicionar condições para exibir ou ocultar determinados campos de formulário com base no comportamento do lado do cliente. Este módulo é incrivelmente útil para tornar os formulários muito mais profissionais, mostrando apenas os campos apropriados e sem adicionar muito JavaScript personalizado.

Exemplo de uso…

$products->displayIf("HasProducts")->isChecked();

$sizes->hideUnless("ProductType")->isEqualTo("t-shirt")
      ->andIf("Price")->isGreaterThan(10);

$payment->hideIf("Price")->isEqualTo(0);

$shipping->displayIf("ProductType")->isEqualTo("furniture")
           ->andIf()
              ->group()
                ->orIf("RushShipping")->isChecked()
                ->orIf("ShippingAddress")->isNotEmpty()
              ->end();

Existem muitos outros exemplos no module readme.md

Menu CMS Agrupado #

O Módulo de Menu CMS Agrupado permite agrupar itens de menu CMS em listas aninhadas que se expandem quando passa o mouse sobre ele. Isso é útil quando há tantos itens de menu do CMS que o espaço na tela se torna um problema.

Painel #

O módulo Dashboard fornece uma página inicial para o CMS no SilverStripe 3 com widgets configuráveis ​​que exibem informações relevantes. Os painéis podem ser criados e estendidos facilmente. O objetivo do módulo Dashboard é fornecer aos usuários uma barra inicial para ações comuns do CMS, como criar tipos de página específicos ou navegar em novo conteúdo.

Existem imagens e vídeos sobre este módulo que podem ser encontrados em este post do blog.

Existem alguns painéis incluídos por padrão …

  • Páginas editadas recentemente
  • Arquivos enviados recentemente
  • Feed RSS
  • Links Rápidos
  • Editor de seção -Análise do Google
  • Tempo

Quando você tem este módulo instalado, ele cria um painel por membro, então se você tiver uma grande quantidade de membros que nunca usarão o admin e o desempenho se tornar um problema, recomendo criar os membros com essas configurações extras antes de escrevê-lo…

Member::create(array(
   'HasConfiguredDashboard' => 1
));
