Qual é o sistema de configuração #

SilverStripe usa um sistema de configuração global para armazenar as configurações das classes e do aplicativo. Essas variáveis ​​de configuração podem ser usadas para definir a estrutura de modelos, configurações de segurança em controladores ou chaves de API para serviços de terceiros.

Como funciona #

Os valores Config são preenchidos pelo SS_ConfigStaticManifest durante um dev/build e cache flush (anexando ?flush a qualquer URL`) ou na primeira execução do código do aplicativo.

O SS_ConfigStaticManifest verificará todas as classes PHP e arquivos de configuração YAML para quaisquer valores de configuração e criará um cache desses valores.

Ao fazer alterações nas configurações de Config via YAML ou variáveis ​​private static, você precisará liberar o cache para que essas alterações tenham efeito.

Configurando valores de configuração #

Os valores Config podem ser definidos de três maneiras:

  1. Via variáveis ​​private static em qualquer classe dentro de um projeto SilverStripe
  2. Por meio de arquivos de configuração yaml (armazenados em module-folder/_config/[arquivo].yml)
  3. Via PHP em tempo de execução (Config::inst()->update('Director', 'environment_type', 'dev')

Geralmente, é melhor definir os valores de configuração por meio dos 2 primeiros métodos, pois eles são armazenados em cache estaticamente ao liberar o cache.

Configurando com estática privada #


class MyDataObject extends DataObject {

    private static $db = array(
        'Title' => 'Varchar',
    );

}

Todas as variáveis ​​de classe private static no código de um projeto SilverStripe (incluindo módulos, mas não pacotes no diretório vendor/) serão carregadas no Config.

Configurando com YAML #

Você pode adicionar isso a mysite/_config/config.yml (ou qualquer outro arquivo YAML nesse caminho).

Director:
  environment_type: dev

Usar arquivos YAML é uma ótima maneira de substituir os valores padrão de Config para classes ou módulos principais

Configurando em tempo de execução #

Isso normalmente seria feito em mysite/_config.php

Config::inst()->update('Director', 'environment_type', 'dev');

Atualizar o Config no PHP deve ser evitado sempre que possível, pois é mais lento do que usar os valores em cache
