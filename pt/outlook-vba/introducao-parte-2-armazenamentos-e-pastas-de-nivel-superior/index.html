Primeira parte de uma introdução às lojas e às pastas que elas contêm. Contém macros para exibir (1) os nomes dos repositórios acessíveis e (2) os nomes dos repositórios acessíveis e as pastas de nível superior dentro deles.

2.1 Conhecimento prévio esperado #

  • Você é um usuário do Outlook e entende termos como “e-mail”, “tempo de recebimento”, “assunto” e “Painel de pastas”.
  • Você sabe como acessar o Editor Visual Basic do Outlook e criar um módulo. Consulte a Introdução Parte 1, se necessário.
  • Você tem pelo menos um conhecimento básico de VBA. Declaro sub-rotinas e variáveis ​​sem explicação. Eu uso Withs, Ifs e Loops sem explicação. Eu lhe digo que algo é uma coleção. Eu digo para você copiar o código para um módulo e executá-lo. Existem muitos tutoriais online, embora a maioria seja para Excel VBA e se concentre mais no uso da linguagem com o Excel do que na linguagem. A busca por “tutorial VBA” traz alguns que se concentram mais na linguagem do que no aplicativo que parecem satisfatórios.
  • Você não é obrigado a conhecer o Outlook Object Model; este tutorial apresenta uma pequena parte dele.

2.2 Lojas #

O Outlook armazena e-mails, itens de calendário, notas, tarefas e assim por diante em arquivos conhecidos como Lojas. Se você olhar para o painel de pastas, verá algo como:

Aaaaaaaaaa
  Inbox
  Drafts
  Deleted Items
  :  :

Bbbbbbbbbb
  Inbox
  Drafts
  Deleted Items
  :  :

Cccccccccc
:  :

“Aaaaaaaaaa”, “Bbbbbbbbbb” e “Cccccccccc” são os nomes de usuário ou de exibição das Lojas. Sempre aceitei os padrões do Outlook para esses nomes que mudaram ao longo dos anos. Uma vez que o padrão era meu nome agora é meu endereço de e-mail. O nome do arquivo para esses armazenamentos pode ser o mesmo, mas com uma extensão como PST ou OST ou pode ser algo completamente diferente. Uma macro VBA precisa do nome de usuário para acessar um repositório e não se preocupa com os nomes dos arquivos ou a extensão.

Você pode ter quantas lojas desejar. Eu tenho o “arquivo de dados do Outlook” que foi criado para mim quando instalei o Outlook. Quando adicionei contas para meus endereços de email, o Outlook criou novas lojas nomeadas para o endereço de email, como “JohnDoe@hotmail.com” e “DoeJohn@gmail.com”. Para reduzir o tamanho da minha loja principal, salvo e-mails antigos em lojas com nomes como “Arquivo 2015”.

Se você for um usuário corporativo, poderá ter acesso a lojas compartilhadas ou às lojas de colegas.

As macros abaixo mostram três maneiras diferentes de listar as lojas que você pode acessar. Sugiro que você crie um novo módulo para conter o código abaixo e use F4 para acessar as propriedades do módulo para que você possa nomeá-lo como “ModIntro” ou algum outro nome de sua escolha. Se você concluiu a Parte 1 desta série, já terá esse módulo.

Copie essas macros para um módulo e teste se cada uma fornece a mesma saída.

Sub ListStores1()

  Dim InxStoreCrnt As Integer
  Dim NS As NameSpace
  Dim StoresColl As Folders

  Set NS = CreateObject("Outlook.Application").GetNamespace("MAPI")
  Set StoresColl = NS.Folders

  For InxStoreCrnt = 1 To StoresColl.Count
    Debug.Print StoresColl(InxStoreCrnt).Name
  Next

End Sub
Sub ListStores2()
 
 Dim StoresColl As Stores
 Dim StoreCrnt As Store
 
 Set StoresColl = Session.Stores
 
 For Each StoreCrnt In StoresColl
   Debug.Print StoreCrnt.DisplayName
 Next
 
End Sub
Sub ListStores3()

  Dim InxStoreCrnt As Long

  With Application.Session
    For InxStoreCrnt = 1 To .Folders.Count
      Debug.Print .Folders(InxStoreCrnt).Name
    Next
  End With

End Sub

Você descobrirá com o VBA que muitas vezes existem vários métodos para alcançar o mesmo efeito. Acima mostrei três métodos de acesso às lojas. Você não precisa se lembrar de todos eles – escolha o seu favorito – mas você precisa estar ciente de que existem vários métodos porque outras pessoas, cujo código você pode precisar estudar, terão diferentes favoritos.

As variáveis ​​StoresColl nas macros ListStores1() e ListStores2() são ambas coleções, mas contêm diferentes tipos de objeto: Store e Folder. Um objeto Store só pode referenciar um arquivo em seu disco. Uma Pasta pode fazer referência a um arquivo no disco, mas também pode fazer referência a pastas dentro de uma loja, como “Caixa de entrada” e “Itens enviados”. Stores, Folders, Store e Folder fazem parte do Outlook Object Model. Esta série de tutoriais apresenta o modelo, mas não é uma definição formal. Se você quiser uma definição formal, digite “outlook vba object model” em seu mecanismo de pesquisa favorito. Certifique-se de olhar para a versão VBA do modelo.

2.3 Pastas de nível superior #

No meu exemplo de painel de pastas acima, listo apenas três pastas padrão: “Caixa de entrada”, “Rascunhos” e “Itens excluídos”. Existem outras pastas padrão e você pode criar quantas pastas desejar. Algumas pessoas criam pastas na Caixa de entrada, mas eu prefiro criar novas pastas no mesmo nível da Caixa de entrada. Suas pastas podem ter subpastas que podem ter suas próprias subpastas em qualquer profundidade.

A macro a seguir produzirá uma listagem do formulário:

A
   A1
   A2
   A3
B
   B1
   B2
C
   C1
   C2
   C3
   C4

onde A, B e C são armazenamentos e A1, B1, C1 e assim por diante são pastas dentro de A, B e C. Se A1, B1, C1 e assim por diante tiverem subpastas, elas não serão listadas por esta macro. O acesso a pastas aninhadas mais profundamente será abordado na próxima parte deste tutorial.

Sub ListStoresAndTopLevelFolders()

  Dim FldrCrnt As Folder
  Dim InxFldrCrnt As Long
  Dim InxStoreCrnt As Long
  Dim StoreCrnt As Folder

  With Application.Session
    For InxStoreCrnt = 1 To .Folders.Count
      Set StoreCrnt = .Folders(InxStoreCrnt)
      With StoreCrnt
        Debug.Print .Name
        For InxFldrCrnt = .Folders.Count To 1 Step -1
          Set FldrCrnt = .Folders(InxFldrCrnt)
          With FldrCrnt
            Debug.Print "   " & .Name
          End With
        Next
      End With
    Next
  End With

End Sub

2.4 O que você deve lembrar deste tutorial #

  • Uma loja é um arquivo no qual o Outlook armazena e-mails, itens de calendário, notas, tarefas e assim por diante.
  • Uma loja pode conter pastas padrão do Outlook, como “Caixa de entrada” e “Itens enviados”.
  • Uma loja também pode conter pastas criadas pelo usuário.
  • Tanto as pastas padrão do Outlook quanto as pastas criadas pelo usuário podem conter subpastas, subsubpastas e assim por diante em qualquer profundidade.
  • Como listar lojas.
  • Como listar lojas e as pastas de nível superior dentro dessas lojas.

Confissão: Não me lembro de nenhum dos “Comos”. Tenho sub-rotinas e funções que me lembram.
