3.1 Função GetFldrNames() que é necessária para várias das macros de demonstração #

Várias macros de demonstração nesta parte requerem uma função que explicarei mais tarde. Por enquanto, por favor, apenas copie GetFldrNames() para um módulo adequado. Uso esta função com frequência e a mantenho, e outras semelhantes que utilizo em várias macros diferentes, em um módulo chamado “ModGlobalOutlook”. Você pode gostar de fazer o mesmo. Como alternativa, você pode preferir manter a macro com todas as outras macros desta série de tutoriais; você pode movê-lo mais tarde se mudar de ideia.

Public Function GetFldrNames(ByRef Fldr As Folder) As String()

  ' * Fldr is a folder. It could be a store, the child of a store,
  '   the grandchild of a store or more deeply nested.
  ' * Return the name of that folder as a string array in the sequence:
  '    (0)=StoreName (1)=Level1FolderName (2)=Level2FolderName  ...
  
  ' 12Oct16  Coded
  ' 20Oct16  Renamed from GetFldrNameStr and amended to return a string array
  '          rather than a string
  
  Dim FldrCrnt As Folder
  Dim FldrNameCrnt As String
  Dim FldrNames() As String
  Dim FldrNamesRev() As String
  Dim FldrPrnt As Folder
  Dim InxFN As Long
  Dim InxFnR As Long

  Set FldrCrnt = Fldr
  FldrNameCrnt = FldrCrnt.Name
  ReDim FldrNamesRev(0 To 0)
  FldrNamesRev(0) = Fldr.Name
  ' Loop getting parents until FldrCrnt has no parent.
  ' Add names of Fldr and all its parents to FldrName as they are found
  Do While True
    Set FldrPrnt = Nothing
    On Error Resume Next
    Set FldrPrnt = Nothing   ' Ensure value is Nothing if following statement fails
    Set FldrPrnt = FldrCrnt.Parent
    On Error GoTo 0
    If FldrPrnt Is Nothing Then
      ' FldrCrnt has no parent
      Exit Do
    End If
    ReDim Preserve FldrNamesRev(0 To UBound(FldrNamesRev) + 1)
    FldrNamesRev(UBound(FldrNamesRev)) = FldrPrnt.Name
    Set FldrCrnt = FldrPrnt
  Loop

  ' Copy names to FldrNames in reverse sequence so they end up in the correct sequence
  ReDim FldrNames(0 To UBound(FldrNamesRev))
  InxFN = 0
  For InxFnR = UBound(FldrNamesRev) To 0 Step -1
    FldrNames(InxFN) = FldrNamesRev(InxFnR)
    InxFN = InxFN + 1
  Next

  GetFldrNames = FldrNames

End Function

3.2 Referenciando uma pasta padrão #

Em TestDefaultFldr() defino Fldr para a caixa de entrada padrão. A constante olFolderInbox pode ser substituída por outros valores dando acesso a qualquer uma das pastas padrão. Se você digitar Set Fldr = Session.GetDefaultFolder(, o editor VB exibirá uma lista suspensa de todos os valores possíveis.

Sub TestDefaultFldr()

  Dim Fldr As Folder

  Set Fldr = Session.GetDefaultFolder(olFolderInbox)

  Debug.Print Join(GetFldrNames(Fldr), "|")

End Sub

No meu laptop, TestDefaultFldr() exibe Outlook data file|Inbox, o que foi uma surpresa. Eu escrevi GetFldrNames(Fldr) para ter certeza de que a pasta que eu tinha referenciado era a que eu queria. Eu tinha acessado a caixa de entrada padrão e descobri que estava vazia! A loja “Arquivo de dados de saída” veio com a instalação padrão e eu a ignorei desde que o Outlook criou uma loja para cada uma das minhas contas de e-mail. Foi somente depois de descobrir minha caixa de entrada padrão vazia que pensei em como o Outlook saberia qual das minhas contas de email era a conta que eu desejaria como padrão. Das pastas padrão do Outlook, não há padrão ou o padrão está em “Arquivo de dados de saída”. Pode ser possível alterar qual caixa de entrada é a caixa de entrada padrão, mas não investiguei porque não tenho certeza de qual das minhas contas de e-mail eu usaria como padrão se alterasse. Apenas lembre-se de que todos os seus itens de calendário, tarefas e assim por diante estão dentro do “arquivo de dados do Outlook” e certifique-se de incluir “Outlook.pst” em sua lista de arquivos.

A maioria dos objetos do Outlook tem a propriedade Parent. GetFldrNames(Fldr) grava o nome da pasta em um array antes de tentar acessar seu pai. Ele faz um loop adicionando nomes ao final da matriz até chegar à loja. A loja não tem um pai, portanto, a tentativa de acessá-la falha. A sequência de nomes na matriz é invertida e, em seguida, retornada ao chamador. Eu usei ‘Join’ para transformar a matriz de nomes em uma string exibível.

3.3 Referenciando qualquer pasta em qualquer armazenamento acessível #

TestFldrChain() demonstra como referenciar qualquer pasta dentro de qualquer armazenamento acessível:

Sub TestFldrChain()

  Dim Fldr As Folder

  Set Fldr = Session.Folders("A").Folders("A2"). _
                           Folders("A21").Folders("A213")

  Debug.Print Join(GetFldrNames(Fldr), "|")

End Sub

Em TestFldrChain(): A é o nome de uma loja; A2 é o nome de uma pasta dentro de A; A21 é o nome de uma pasta dentro de A2 e A213 é o nome de uma pasta dentro de A21.

O que esta acontecendo aqui?

Session tem uma propriedade Folders que é uma lista de todas as lojas acessíveis.

Session.Folders(integer), que usei na Parte 2 deste tutorial, me permite percorrer as lojas em sequência quando não sei seus nomes. Session.Folders("A") me permite acessar uma pasta quando sei seu nome.

Session.Folders("A") é uma pasta e também tem a propriedade Folders.

Session.Folders("A").Folders("A2") me dá acesso à pasta “A2” dentro da loja “A”.

Eu posso encadear quantas Folders("x")s forem necessárias para alcançar qualquer pasta. Se a cadeia for muito longa para uma linha, você poderá dividir a instrução em várias linhas, como eu fiz.

Procure a pasta aninhada mais profundamente em sua instalação e substitua A, A2, A21 e A213 pelos nomes de sua loja e pastas. Aumente ou diminua o número de pastas na cadeia conforme necessário.

Se você atualizar e executar TestFldrChain(), ele produzirá o seguinte, exceto que A, A2 e assim por diante serão substituídos pelos nomes das suas pastas:

A|A2|A21|A213

3.4 Listando os nomes de cada pasta dentro de cada armazenamento acessível #

Na Parte 2, você aprendeu como listar todos os repositórios acessíveis e as pastas de nível superior em cada repositório. Isso envolveu um loop pelos repositórios e, em seguida, um loop para cada repositório por meio de suas pastas. Acima, você viu como fazer referência a uma pasta conhecida em qualquer profundidade dentro da hierarquia de pastas. Isso envolvia encadear quantas Folders("x") fossem necessárias para alcançar a pasta.

Agora quero listar cada pasta, em qualquer profundidade, dentro de cada loja. A técnica de codificação mais fácil para resolver esse tipo de problema em que você deve descer cadeias de comprimentos variados é a recursão. Se você é um programador sério em outra linguagem ou ferramenta, já deve saber sobre recursão. Se você tem ambições de ser um programador sério, precisará entender a recursão eventualmente, mas não necessariamente hoje. “Recursão” é um daqueles conceitos que muitos acham difícil de entender no início. Você pode digitar “Recursão” em seu mecanismo de busca favorito e ler as várias tentativas de explicar esse conceito. Alternativamente, você pode aceitar esses trabalhos de macro, mas não se preocupe como eles funcionam.

Observe o comentário em ListStoresAndAllFolders(): essas macros precisam de uma referência ao “Microsoft Scripting Runtime”. Clique em Ferramentas na barra de guias na parte superior da janela do Editor VB e clique em Referências. Você obterá uma lista de todas as referências disponíveis (bibliotecas). Alguns no topo já estarão marcados. Os demais estão em ordem alfabética. Role a lista e clique na caixa à esquerda de “Microsoft Scripting Runtime” para marcar. Em seguida, clique em OK

 Sub ListStoresAndAllFolders()

  ' Displays the name of every accessible store
  ' Under each store, displays an indented list of all its folders

  ' Technique for locating desktop from answer by Kyle:
  ' http://stackoverflow.com/a/17551579/973283

  ' Needs reference to “Microsoft Scripting Runtime” if "TextStream"
  ' and "FileSystemObject" are to be recognised

  Dim FileOut As TextStream
  Dim FldrCrnt As Folder
  Dim Fso As FileSystemObject
  Dim InxFldrCrnt As Long
  Dim InxStoreCrnt As Long
  Dim Path As String
  Dim StoreCrnt As Folder

  Path = CreateObject("WScript.Shell").SpecialFolders("Desktop")
  
  Set Fso = CreateObject("Scripting.FileSystemObject")
  Set FileOut = Fso.CreateTextFile(Path & "\ListStoresAndAllFolders.txt", True)

  With Application.Session
    For InxStoreCrnt = 1 To .Folders.Count
      Set StoreCrnt = .Folders(InxStoreCrnt)
      With StoreCrnt
        FileOut.WriteLine .Name
        For InxFldrCrnt = .Folders.Count To 1 Step -1
          Set FldrCrnt = .Folders(InxFldrCrnt)
          Call ListAllFolders(FldrCrnt, 1, FileOut)
        Next
      End With
    Next
  End With

  FileOut.Close

End Sub
Sub ListAllFolders(ByRef Fldr As Folder, ByVal Level As Long, ByRef FileOut As TextStream)

  ' This routine:
  '  1. Output name of Fldr
  '  2. Calls itself for each child of Fldr
  ' It is designed to be called by ListStoresAndAllFolders()

  Dim InxFldrCrnt As Long

  With Fldr
    FileOut.WriteLine Space(Level * 2) & .Name
    For InxFldrCrnt = .Folders.Count To 1 Step -1
      Call ListAllFolders(.Folders(InxFldrCrnt), Level + 1, FileOut)
    Next
  End With

End Sub

Depois de executar ListStoresAndAllFolders, haverá um novo arquivo em seu Desktop chamado “ListStoresAndAllFolders.txt” que conterá a lista prometida de armazenamentos e pastas.

3.5 Movendo uma pasta de uma pasta pai para outra #

Por que eu quero fazer referência a uma pasta? Na próxima parte, mostrarei como acessar e-mails dentro de uma pasta referenciada. Aqui vou mostrar como mover uma pasta. Eu criei uma pasta chamada “Test” na minha caixa de entrada. Em TestMoveFolder(), substituí “A” pelo nome da loja que contém minha caixa de entrada. A execução de TestMoveFolder() moveu “Test” para “Itens Excluídos”.

Sub TestMoveFolder()

  Dim FldrDest As Folder
  Dim FldrToMove As Folder

  Set FldrToMove = Session.Folders("A").Folders("Inbox").Folders("Test")
  Set FldrDest = Session.Folders("A").Folders("Deleted Items")
  
  FldrToMove.MoveTo FldrDest

End Sub
