Sintaxe #

  • usando Compat
  • Compat.String
  • Compat.UTF8String
  • @compat f.(x, y)

Às vezes é muito difícil fazer com que uma nova sintaxe funcione bem com várias versões. Como Julia ainda está em desenvolvimento ativo, muitas vezes é útil simplesmente descartar o suporte para versões mais antigas e, em vez disso, direcionar apenas as mais recentes.

Números de versão #

Julia tem uma implementação embutida de versão semântica exposta através do tipo VersionNumber.

Para construir um VersionNumber como um literal, o @v_str string macro pode ser usado:

julia> vers = v"1.2.0"
v"1.2.0"

Alternativamente, pode-se chamar o construtor VersionNumber; observe que o construtor aceita até cinco argumentos, mas todos, exceto o primeiro, são opcionais.

julia> vers2 = VersionNumber(1, 1)
v"1.1.0"

Os números de versão podem ser comparados usando operadores de comparação e, portanto, podem ser classificados:

julia> vers2 < vers
true

julia> v"1" < v"0"
false

julia> sort([v"1.0.0", v"1.0.0-dev.100", v"1.0.1"])
3-element Array{VersionNumber,1}:
 v"1.0.0-dev.100"
 v"1.0.0"
 v"1.0.1"

Os números de versão são usados ​​em vários lugares em Julia. Por exemplo, a constante VERSION é um VersionNumber:

julia> VERSION
v"0.5.0"

Isso é comumente usado para avaliação de código condicional, dependendo da versão de Julia. Por exemplo, para executar código diferente em v0.4 e v0.5, pode-se fazer

if VERSION < v"0.5"
    println("v0.5 prerelease, v0.4 or older")
else
    println("v0.5 or newer")
end

Cada pacote instalado também está associado a um número de versão atual:

julia> Pkg.installed("StatsBase")
v"0.9.0"

Usando Compat.jl #

O pacote Compat.jl permite o uso de alguns novos recursos e sintaxe do Julia com versões mais antigas do Julia. Seus recursos estão documentados em seu README, mas um resumo de aplicativos úteis é fornecido abaixo.

Tipo de string unificada #

Em Julia v0.4, havia muitos tipos diferentes de strings. Este sistema foi considerado muito complexo e confuso, então no Julia v0.5, resta apenas o tipo String. Compat permite usar o tipo e construtor String na versão 0.4, sob o nome Compat.String. Por exemplo, este código v0.5

buf = IOBuffer()
println(buf, "Hello World!")
String(buf)  # "Hello World!\n"

pode ser traduzido diretamente para este código, que funciona tanto na v0.5 quanto na v0.4:

using Compat
buf = IOBuffer()
println(buf, "Hello World!")
Compat.String(buf)  # "Hello World!\n"

Observe que há algumas ressalvas.

  • Na v0.4, Compat.String é tipificado para ByteString, que é Union{ASCIIString, UTF8String}. Assim, os tipos com campos String não serão de tipo estável. Nestas situações, Compat.UTF8String é recomendado, pois significará String na v0.5 e UTF8String na v0.4, ambos tipos concretos.

  • Deve-se ter cuidado ao usar Compat.String ou import Compat: String, porque String em si tem um significado na v0.4: é um alias obsoleto para AbstractString. Um sinal de que String foi usado acidentalmente em vez de Compat.String é se, a qualquer momento, os seguintes avisos aparecerem:

    WARNING: Base.String is deprecated, use AbstractString instead. likely near no file:0 WARNING: Base.String is deprecated, use AbstractString instead. likely near no file:0

Sintaxe de transmissão compacta #

Julia v0.5 introduz açúcar sintático para broadcast. A sintaxe

f.(x, y)

é reduzido para broadcast(f, x, y). Exemplos de uso dessa sintaxe incluem sin.([1, 2, 3]) para obter o seno de vários números de uma só vez.

Na v0.5, a sintaxe pode ser usada diretamente:

julia> sin.([1.0, 2.0, 3.0])
3-element Array{Float64,1}:
 0.841471
 0.909297
 0.14112

No entanto, se tentarmos o mesmo na v0.4, recebemos um erro:

julia> sin.([1.0, 2.0, 3.0])
ERROR: TypeError: getfield: expected Symbol, got Array{Float64,1}

Felizmente, Compat torna esta nova sintaxe utilizável a partir da v0.4 também. Mais uma vez, adicionamos using Compat. Desta vez, cercamos a expressão com a macro @compat:

julia> using Compat

julia> @compat sin.([1.0, 2.0, 3.0])
3-element Array{Float64,1}:
 0.841471
 0.909297
 0.14112
