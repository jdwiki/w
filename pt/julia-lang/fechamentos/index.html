Sintaxe #

  • x -> [corpo]
  • (x, y) -> [corpo]
  • (xs…) -> [corpo]

Em versões mais antigas do Julia, encerramentos e funções anônimas tinham uma penalidade de desempenho de tempo de execução. Esta penalidade foi eliminada em 0,5.

Introdução aos Fechamentos #

Funções são uma parte importante da programação de Julia. Eles podem ser definidos diretamente dentro dos módulos, caso em que as funções são chamadas de top-level. Mas as funções também podem ser definidas dentro de outras funções. Essas funções são chamadas de “encerramentos”.

Os fechamentos capturam as variáveis ​​em sua função externa. Uma função de nível superior só pode usar variáveis ​​globais de seu módulo, parâmetros de função ou variáveis ​​locais:

x = 0  # global
function toplevel(y)
    println("x = ", x, " is a global variable")
    println("y = ", y, " is a parameter")
    z = 2
    println("z = ", z, " is a local variable")
end

Um encerramento, por outro lado, pode usar todos esses, além de variáveis ​​de funções externas que ele captura:

x = 0  # global
function toplevel(y)
    println("x = ", x, " is a global variable")
    println("y = ", y, " is a parameter")
    z = 2
    println("z = ", z, " is a local variable")

    function closure(v)
        println("v = ", v, " is a parameter")
        w = 3
        println("w = ", w, " is a local variable")
        println("x = ", x, " is a global variable")
        println("y = ", y, " is a closed variable (a parameter of the outer function)")
        println("z = ", z, " is a closed variable (a local of the outer function)")
    end
end

Se executarmos c = toplevel(10), veremos que o resultado é

julia> c = toplevel(10)
x = 0 is a global variable
y = 10 is a parameter
z = 2 is a local variable
(::closure) (generic function with 1 method)

Observe que a expressão de cauda dessa função é uma função em si; ou seja, um fechamento. Podemos chamar a closure c como se fosse qualquer outra função:

julia> c(11)
v = 11 is a parameter
w = 3 is a local variable
x = 0 is a global variable
y = 10 is a closed variable (a parameter of the outer function)
z = 2 is a closed variable (a local of the outer function)

Note que c ainda tem acesso às variáveis ​​y e z da chamada toplevel — mesmo que toplevel já tenha retornado! Cada fechamento, mesmo aqueles retornados pela mesma função, fecha sobre diferentes variáveis. Podemos chamar toplevel novamente

julia> d = toplevel(20)
x = 0 is a global variable
y = 20 is a parameter
z = 2 is a local variable
(::closure) (generic function with 1 method)

julia> d(22)
v = 22 is a parameter
w = 3 is a local variable
x = 0 is a global variable
y = 20 is a closed variable (a parameter of the outer function)
z = 2 is a closed variable (a local of the outer function)

julia> c(22)
v = 22 is a parameter
w = 3 is a local variable
x = 0 is a global variable
y = 10 is a closed variable (a parameter of the outer function)
z = 2 is a closed variable (a local of the outer function)

Observe que apesar de d e c terem o mesmo código e serem passados ​​os mesmos argumentos, sua saída é diferente. São fechamentos distintos.

Implementando Curry #

Uma aplicação de closures é aplicar parcialmente uma função; ou seja, forneça alguns argumentos agora e crie uma função que receba os argumentos restantes. Currying é uma forma específica de aplicação parcial.

Vamos começar com a função simples curry(f, x) que fornecerá o primeiro argumento para uma função e esperará argumentos adicionais posteriormente. A definição é bastante simples:

curry(f, x) = (xs...) -> f(x, xs...)

Mais uma vez, usamos [sintaxe de função anônima](https://www.wikiod.com/pt/julia-lang/funcoes#Funções anônimas), desta vez em combinação com a sintaxe de argumento variável.

Podemos implementar algumas funções básicas no estilo tácito (ou sem pontos) usando esta função curry.

julia> const double = curry(*, 2)
(::#19) (generic function with 1 method)

julia> double(10)
20

julia> const simon_says = curry(println, "Simon: ")
(::#19) (generic function with 1 method)

julia> simon_says("How are you?")
Simon: How are you?

As funções mantêm o generalismo esperado:

julia> simon_says("I have ", 3, " arguments.")
Simon: I have 3 arguments.

julia> double([1, 2, 3])
3-element Array{Int64,1}:
 2
 4
 6

Composição de funções #

Podemos definir uma função para executar composição de funções usando [sintaxe de função anônima](https://www.wikiod.com/pt/julia-lang/funcoes /10470/anonymous-functions#t=201608292058450143587):

f ∘ g = x -> f(g(x))

Observe que esta definição é equivalente a cada uma das seguintes definições:

∘(f, g) = x -> f(g(x))

ou

function ∘(f, g)
    x -> f(g(x))
end

lembrando que em Julia, f ∘ g é apenas açúcar de sintaxe para ∘(f, g).

Podemos ver que esta função compõe corretamente:

julia> double(x) = 2x
double (generic function with 1 method)

julia> triple(x) = 3x
triple (generic function with 1 method)

julia> const sextuple = double ∘ triple
(::#17) (generic function with 1 method)

julia> sextuple(1.5)
9.0

Na versão v0.5, essa definição é muito eficiente. Podemos olhar para o código LLVM gerado:

julia> @code_llvm sextuple(1)

define i64 @"julia_#17_71238"(i64) #0 {
top:
  %1 = mul i64 %0, 6
  ret i64 %1
}

É claro que as duas multiplicações foram dobradas em uma única multiplicação, e que esta função é tão eficiente quanto possível.

Como essa função de ordem superior funciona? Ele cria um chamado fechamento, que consiste não apenas em seu código, mas também acompanha certas variáveis ​​de seu escopo. Todas as funções em Julia que não são criadas no escopo de nível superior são closures.

Pode-se inspecionar as variáveis ​​fechadas através dos campos do fechamento. Por exemplo, vemos que:

julia> (sin ∘ cos).f
sin (generic function with 10 methods)

julia> (sin ∘ cos).g
cos (generic function with 10 methods)
