Sintaxe #

  • start(it)
  • próximo(itr, s)
  • feito (itr, s)
  • tomar(itr, n)
  • drop(itr, n)
  • ciclo (itr)
  • Base.produto(xs, ys)

Parâmetros #

Parâmetro Detalhes
Para Todas as funções
itr O iterável para operar.
Para próximo e concluído
s Um estado iterador que descreve a posição atual da iteração.
Para pegar e soltar
n O número de elementos a serem retirados ou descartados.
Para Base.produto
xs O iterável para obter os primeiros elementos dos pares.
s O iterável para obter os segundos elementos dos pares.
… (Observe que product aceita qualquer número de argumentos; se mais de dois forem fornecidos, ele construirá tuplas de comprimento maior que dois.)

Novo tipo iterável #

Em Julia, ao percorrer um objeto iterável I é feito com a sintaxe for:

for i = I   # or  "for i in I"
    # body
end

Nos bastidores, isso é traduzido para:

state = start(I)
while !done(I, state)
    (i, state) = next(I, state)
    # body
end

Portanto, se você quer que I seja um iterável, você precisa definir os métodos start, next e done para seu tipo. Suponha que você defina um type Foo contendo um array como um dos campos:

type Foo
    bar::Array{Int,1}
end

Instanciamos um objeto Foo fazendo:

julia> I = Foo([1,2,3])
Foo([1,2,3])

julia> I.bar
3-element Array{Int64,1}:
 1
 2
 3

Se quisermos iterar através de Foo, com cada elemento bar sendo retornado por cada iteração, definimos os métodos:

import Base: start, next, done

start(I::Foo) = 1

next(I::Foo, state) = (I.bar[state], state+1)

function done(I::Foo, state)
    if state == length(I.bar)
        return true
    end
    return false
end

Observe que como essas funções pertencem ao módulo Base, devemos primeiro importar seus nomes antes de adicionar novos métodos a eles.

Depois que os métodos são definidos, o Foo é compatível com a interface do iterador:

julia> for i in I
           println(i)
       end

1
2
3

Combinando iteráveis ​​preguiçosos #

A biblioteca padrão vem com uma rica coleção de iteráveis ​​preguiçosos (e bibliotecas como Iterators.jl fornecem ainda mais). Iterables preguiçosos podem ser compostos para criar iteráveis ​​mais poderosos em tempo constante. Os iteráveis ​​preguiçosos mais importantes são take and drop, a partir dos quais muitas outras funções podem ser criadas.

Fatia preguiçosamente um iterável #

Arrays podem ser fatiados com notação de fatia. Por exemplo, o seguinte retorna o 10º ao 15º elementos de uma matriz, inclusive:

A[10:15]

No entanto, a notação de fatia não funciona com todos os iteráveis. Por exemplo, não podemos fatiar uma expressão geradora:

julia> (i^2 for i in 1:10)[3:5]
ERROR: MethodError: no method matching getindex(::Base.Generator{UnitRange{Int64},##1#2}, ::UnitRange{Int64})

Fatiar strings pode não ter o comportamento Unicode esperado:

julia> "αααα"[2:3]
ERROR: UnicodeError: invalid character index
 in getindex(::String, ::UnitRange{Int64}) at ./strings/string.jl:130

julia> "αααα"[3:4]
"α"

Podemos definir uma função lazysub(itr, range::UnitRange) para fazer esse tipo de fatiamento em iteráveis ​​arbitrários. Isso é definido em termos de take e drop:

lazysub(itr, r::UnitRange) = take(drop(itr, first(r) - 1), last(r) - first(r) + 1)

A implementação aqui funciona porque para o valor a:b de UnitRange, as seguintes etapas são executadas:

  • descarta os primeiros elementos a-1
  • pega o elemento ath, a+1th elemento, e assim por diante, até o elemento a+(b-a)=bth

No total, os elementos b-a são tomados. Podemos confirmar que nossa implementação está correta em cada caso acima:

julia> collect(lazysub("αααα", 2:3))
2-element Array{Char,1}:
 'α'
 'α'

julia> collect(lazysub((i^2 for i in 1:10), 3:5))
3-element Array{Int64,1}:
  9
 16
 25

Deslocar lentamente um iterável circularmente #

A operação circshift em arrays irá deslocar o array como se fosse um círculo, então o relineará. Por exemplo,

julia> circshift(1:10, 3)
10-element Array{Int64,1}:
  8
  9
 10
  1
  2
  3
  4
  5
  6
  7

Podemos fazer isso preguiçosamente para todos os iteráveis? Podemos usar os iteráveis ​​cycle, drop e take para implementar esta funcionalidade.

lazycircshift(itr, n) = take(drop(cycle(itr), length(itr) - n), length(itr))

Junto com os tipos preguiçosos sendo mais performáticos em muitas situações, isso nos permite fazer uma funcionalidade do tipo circshift em tipos que de outra forma não a suportariam:

julia> circshift("Hello, World!", 3)
ERROR: MethodError: no method matching circshift(::String, ::Int64)
Closest candidates are:
  circshift(::AbstractArray{T,N}, ::Real) at abstractarraymath.jl:162
  circshift(::AbstractArray{T,N}, ::Any) at abstractarraymath.jl:195

julia> String(collect(lazycircshift("Hello, World!", 3)))
"ld!Hello, Wor"

Fazendo uma tabuada #

Vamos fazer uma tabela de multiplicação usando funções iteráveis ​​preguiçosas para criar uma matriz.

As principais funções a serem usadas aqui são:

  • Base.product, que calcula um produto cartesiano.
  • prod, que calcula um produto regular (como na multiplicação)
  • :, que cria um intervalo
  • map, que é uma função de ordem superior que aplica uma função a cada elemento de uma coleção

A solução é:

julia> map(prod, Base.product(1:10, 1:10))

Listas avaliadas com preguiça #

É possível fazer uma lista simples avaliada lentamente usando tipos mutáveis ​​e fechamentos. Uma lista avaliada com preguiça é uma lista cujos elementos não são avaliados quando são construídos, mas sim quando são acessados. Os benefícios de listas avaliadas preguiçosamente incluem a possibilidade de serem infinitas.

import Base: getindex
type Lazy
    thunk
    value
    Lazy(thunk) = new(thunk)
end

evaluate!(lazy::Lazy) = (lazy.value = lazy.thunk(); lazy.value)
getindex(lazy::Lazy) = isdefined(lazy, :value) ? lazy.value : evaluate!(lazy)

import Base: first, tail, start, next, done, iteratorsize, HasLength, SizeUnknown
abstract List
immutable Cons <: List
    head
    tail::Lazy
end
immutable Nil <: List end

macro cons(x, y)
    quote
        Cons($(esc(x)), Lazy(() -> $(esc(y))))
    end
end

first(xs::Cons) = xs.head
tail(xs::Cons) = xs.tail[]
start(xs::Cons) = xs
next(::Cons, xs) = first(xs), tail(xs)
done(::List, ::Cons) = false
done(::List, ::Nil) = true
iteratorsize(::Nil) = HasLength()
iteratorsize(::Cons) = SizeUnknown()

O que de fato funciona como em uma linguagem como Haskell, onde todas as listas são avaliadas com preguiça:

julia> xs = @cons(1, ys)
Cons(1,Lazy(false,#3,#undef))

julia> ys = @cons(2, xs)
Cons(2,Lazy(false,#5,#undef))

julia> [take(xs, 5)...]
5-element Array{Int64,1}:
 1
 2
 1
 2
 1

Na prática, é melhor usar o pacote Lazy.jl. No entanto, a implementação da lista preguiçosa acima esclarece detalhes importantes sobre como construir seu próprio tipo iterável.
