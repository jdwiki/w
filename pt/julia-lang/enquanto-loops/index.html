Sintaxe #

  • enquanto cond; corpo; fim
  • parar
  • Prosseguir

O loop while não tem valor; embora possa ser usado em posição de expressão, seu tipo é Void e o valor obtido será nada.

Sequência de Collatz #

O loop while executa seu corpo enquanto a condição for válida. Por exemplo, o código a seguir calcula e imprime a sequência Collatz de um determinado número:

function collatz(n)
    while n ≠ 1
        println(n)
        n = iseven(n) ? n ÷ 2 : 3n + 1
    end
    println("1... and 4, 2, 1, 4, 2, 1 and so on")
end

Uso:

julia> collatz(10)
10
5
16
8
4
2
1... and 4, 2, 1, 4, 2, 1 and so on

É possível escrever qualquer loop recursivamente, e para loops while complexos, algumas vezes a variante recursiva é mais clara. No entanto, em Julia, os loops têm algumas vantagens distintas sobre a recursão:

  • Julia não garante a eliminação de chamadas finais, portanto, a recursão usa memória adicional e pode causar erros de estouro de pilha.
  • E além disso, pelo mesmo motivo, um loop pode ter uma sobrecarga reduzida e rodar mais rápido.

Pesquisa em largura #

(Embora este exemplo seja escrito usando a sintaxe introduzida na versão v0.5, ele também pode funcionar com poucas modificações em versões mais antigas.)

Esta implementação de breadth-first search (BFS) em um gráfico representado com listas de adjacência usa loops while e a instrução return. A tarefa que vamos resolver é a seguinte: temos uma sequência de pessoas e uma sequência de amizades (amizades são mútuas). Queremos determinar o grau de conexão entre duas pessoas. Ou seja, se duas pessoas forem amigas, retornaremos 1; se um for amigo de um amigo do outro, retornaremos 2, e assim sucessivamente.

Primeiro, vamos supor que já temos uma lista de adjacências: um Dict mapeando T para Array{T, 1}, onde as chaves são pessoas e os valores são todos os amigos dessa pessoa. Aqui podemos representar pessoas com qualquer tipo ‘T’ que escolhermos; neste exemplo, usaremos Symbol. No algoritmo BFS, mantemos uma fila de pessoas para “visitar” e marcamos sua distância do nó de origem.

function degree(adjlist, source, dest)
    distances = Dict(source => 0)
    queue = [source]

    # until the queue is empty, get elements and inspect their neighbours
    while !isempty(queue)
        # shift the first element off the queue
        current = shift!(queue)

        # base case: if this is the destination, just return the distance
        if current == dest
            return distances[dest]
        end

        # go through all the neighbours
        for neighbour in adjlist[current]
            # if their distance is not already known...
            if !haskey(distances, neighbour)
                # then set the distance
                distances[neighbour] = distances[current] + 1

                # and put into queue for later inspection
                push!(queue, neighbour)
            end
        end
    end

    # we could not find a valid path
    error("$source and $dest are not connected.")
end

Agora, vamos escrever uma função para construir uma lista de adjacências dada uma sequência de pessoas e uma sequência de tuplas (pessoa, pessoa):

function makeadjlist(people, friendships)
    # dictionary comprehension (with generator expression)
    result = Dict(p => eltype(people)[] for p in people)

    # deconstructing for; friendship is mutual
    for (a, b) in friendships
        push!(result[a], b)
        push!(result[b], a)
    end

    result
end

Agora podemos definir a função original:

degree(people, friendships, source, dest) =
    degree(makeadjlist(people, friendships), source, dest)

Agora vamos testar nossa função em alguns dados.

const people = [:jean, :javert, :cosette, :gavroche, :éponine, :marius]
const friendships = [
    (:jean, :cosette),
    (:jean, :marius),
    (:cosette, :éponine),
    (:cosette, :marius),
    (:gavroche, :éponine)
]

Jean está conectado a si mesmo em passos ‘0’:

julia> degree(people, friendships, :jean, :jean)
0

Jean e Cosette são amigos, e por isso têm grau ‘1’:

julia> degree(people, friendships, :jean, :cosette)
1

Jean e Gavroche estão conectados indiretamente através de Cosette e depois Marius, então seu grau é ‘3’:

julia> degree(people, friendships, :jean, :gavroche)
3

Javert e Marius não estão conectados por nenhuma cadeia, então um erro é gerado:

julia> degree(people, friendships, :javert, :marius)
ERROR: javert and marius are not connected.
 in degree(::Dict{Symbol,Array{Symbol,1}}, ::Symbol, ::Symbol) at ./REPL[28]:27
 in degree(::Array{Symbol,1}, ::Array{Tuple{Symbol,Symbol},1}, ::Symbol, ::Symbol) at ./REPL[30]:1

Execute uma vez antes de testar a condição #

Às vezes, deseja-se executar algum código de inicialização uma vez antes de testar uma condição. Em algumas outras linguagens, esse tipo de loop tem uma sintaxe especial do-while. No entanto, esta sintaxe pode ser substituída por um loop while regular e uma instrução break, então Julia não tem uma sintaxe especializada do-while. Em vez disso, escreve-se:

local name

# continue asking for input until satisfied
while true
    # read user input
    println("Type your name, without lowercase letters:")
    name = readline()

    # if there are no lowercase letters, we have our result!
    !any(islower, name) && break
end

Observe que, em algumas situações, esses loops podem ser mais claros com recursão:

function getname()
    println("Type your name, without lowercase letters:")
    name = readline()
    if any(islower, name)
        getname()  # this name is unacceptable; try again
    else
        name       # this name is good, return it
    end
end
