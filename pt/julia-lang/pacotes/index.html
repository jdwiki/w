Sintaxe #

  • Pkg.add(pacote)
  • Pkg.checkout(pacote, branch=“master”)
  • Pkg.clone(url)
  • Pkg.dir(pacote)
  • Pkg.pin(pacote, versão)
  • Pkg.rm(pacote)

Parâmetros #

Parâmetro Detalhes
Pkg.add(pacote) Baixe e instale o pacote registrado fornecido.
Pkg.checkout(package, branch) Confira a ramificação fornecida para o pacote registrado fornecido. branch é opcional e o padrão é "master".
Pkg.clone(url) Clone o repositório Git no URL fornecido como um pacote.
Pkg.dir(pacote) Obtenha o local no disco para o pacote fornecido.
Pkg.pin(pacote, versão) Força o pacote a permanecer na versão especificada. version é opcional e o padrão é a versão atual do pacote.
Pkg.rm(pacote) Remova o pacote fornecido da lista de pacotes necessários.

Instalar, usar e remover um pacote registrado #

Depois de encontrar um pacote oficial do Julia, é simples baixar e instalar o pacote. Em primeiro lugar, é recomendável atualizar a cópia local de METADATA:

julia> Pkg.update()

Isso garantirá que você obtenha as versões mais recentes de todos os pacotes.

Suponha que o pacote que queremos instalar tenha o nome Currencies.jl. O comando a ser executado para instalar este pacote seria:

julia> Pkg.add("Currencies")

Este comando instalará não apenas o pacote em si, mas também todas as suas dependências.

Se a instalação for bem-sucedida, você pode testar se o pacote funciona corretamente:

julia> Pkg.test("Currencies")

Então, para usar o pacote, use

julia> using Currencies

e prossiga conforme descrito na documentação do pacote, normalmente vinculado ou incluído no arquivo README.md.

Para desinstalar um pacote que não é mais necessário, use a função Pkg.rm:

julia> Pkg.rm("Currencies")

Observe que isso pode não remover o diretório do pacote; em vez disso, ele apenas marcará o pacote como não mais necessário. Muitas vezes, isso está perfeitamente bem - economizará tempo caso você precise do pacote novamente no futuro. Mas se necessário, para remover o pacote fisicamente, chame a função rm, então chame Pkg.resolve:

julia> rm(Pkg.dir("Currencies"); recursive=true)

julia> Pkg.resolve()

Confira um branch ou versão diferente #

Às vezes, a última versão marcada de um pacote tem bugs ou está faltando alguns recursos necessários. Usuários avançados podem querer atualizar para a versão de desenvolvimento mais recente de um pacote (às vezes chamado de “mestre”, com o nome usual de um branch de desenvolvimento no Git). Os benefícios disso incluem:

  • Os desenvolvedores que contribuem para um pacote devem contribuir para a versão de desenvolvimento mais recente.
  • A versão de desenvolvimento mais recente pode ter recursos úteis, correções de bugs ou aprimoramentos de desempenho.
  • Os usuários que relatam um bug podem querer verificar se um bug ocorre na versão de desenvolvimento mais recente.

No entanto, há muitas desvantagens em executar a versão de desenvolvimento mais recente:

  • A versão de desenvolvimento mais recente pode ser mal testada e ter bugs sérios.
  • A versão de desenvolvimento mais recente pode mudar com frequência, quebrando seu código.

Para verificar a última ramificação de desenvolvimento de um pacote chamado JSON.jl, por exemplo, use

Pkg.checkout("JSON")

Para fazer check-out de um branch ou tag diferente (sem o nome “master”), use

Pkg.checkout("JSON", "v0.6.0")

No entanto, se a tag representar uma versão, geralmente é melhor usar

Pkg.pin("JSON", v"0.6.0")

Observe que um literal de versão é usado aqui, não uma string simples. A versão Pkg.pin informa o gerenciador de pacotes sobre a restrição da versão, permitindo que o gerenciador de pacotes ofereça feedback sobre quais problemas podem causar.

Para retornar à última versão marcada,

Pkg.free("JSON")

Instale um pacote não registrado #

Alguns pacotes experimentais não estão incluídos no repositório de pacotes METADATA. Esses pacotes podem ser instalados clonando diretamente seus repositórios Git. Observe que pode haver dependências de pacotes não registrados que não são registrados; essas dependências não podem ser resolvidas pelo gerenciador de pacotes e devem ser resolvidas manualmente. Por exemplo, para instalar o pacote não registrado OhMyREPL.jl:

Pkg.clone("https://github.com/KristofferC/Tokenize.jl")
Pkg.clone("https://github.com/KristofferC/OhMyREPL.jl")

Então, como de costume, use using para usar o pacote:

using OhMyREPL
