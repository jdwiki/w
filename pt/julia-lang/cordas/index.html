Parâmetros #

Parâmetro Detalhes
Para sprint(f, xs...)
f Uma função que recebe um objeto IO como seu primeiro argumento.
xs Zero ou mais argumentos restantes para passar para f.

Olá Mundo! #

Strings em Julia são delimitadas usando o símbolo ":

julia> mystring = "Hello, World!"
"Hello, World!"

Note que diferente de algumas outras linguagens, o símbolo ' _não pode ser usado. ' define um literal de caractere; este é um tipo de dados Char e armazenará apenas um único valor escalar Unicode:

julia> 'c'
'c'

julia> 'character'
ERROR: syntax: invalid character literal

Pode-se extrair os valores escalares unicode de uma string iterando sobre ela com um for loop:

julia> for c in "Hello, World!"
           println(c)
       end
H
e
l
l
o
,
 
W
o
r
l
d
!

Grafemas #

O tipo Char de Julia representa um valor escalar Unicode, que apenas em alguns casos corresponde ao que os humanos percebem como um “caractere”. Por exemplo, uma representação do caractere é, como no currículo, é na verdade uma combinação de dois valores escalares Unicode:

julia> collect("é")
2-element Array{Char,1}:
 'e'
 '́'

As descrições Unicode para esses codepoints são “LATIN SMALL LETTER E” e “COMBINING ACUTE ACCENT”. Juntos, eles definem um único caractere “humano”, que em termos Unicode é chamado de grafema. Mais especificamente, o Anexo 29 do Unicode motiva a definição de um grupo de grafemas porque:

É importante reconhecer que o que o usuário considera um “caractere” – uma unidade básica de um sistema de escrita para um idioma – pode não ser apenas um único ponto de código Unicode. Em vez disso, essa unidade básica pode ser composta de vários pontos de código Unicode. Para evitar ambiguidade com o uso do computador do termo caractere, isso é chamado de caractere percebido pelo usuário. Por exemplo, “G” + acento agudo é um caractere percebido pelo usuário: os usuários pensam nele como um único caractere, mas na verdade é representado por dois pontos de código Unicode. Esses caracteres percebidos pelo usuário são aproximados pelo que é chamado de cluster de grafemas, que pode ser determinado programaticamente.

Julia fornece a função graphemes para iterar sobre os clusters de grafemas em uma string:

julia> for c in graphemes("résumé")
           println(c)
       end
r
é
s
u
m
é

Observe como o resultado, imprimindo cada caractere em sua própria linha, é melhor do que se tivéssemos iterado sobre os valores escalares Unicode:

julia> for c in "résumé"
           println(c)
       end
r
e

s
u
m
e

Normalmente, ao trabalhar com caracteres em um sentido percebido pelo usuário, é mais útil lidar com clusters de grafemas do que com valores escalares Unicode. Por exemplo, suponha que queremos escrever uma função para calcular o comprimento de uma única palavra. Uma solução ingênua seria usar

julia> wordlength(word) = length(word)
wordlength (generic function with 1 method)

Notamos que o resultado é contra-intuitivo quando a palavra inclui agrupamentos de grafemas que consistem em mais de um codepoint:

julia> wordlength("résumé")
8

Quando usamos a definição mais correta, usando a função grafemas, obtemos o resultado esperado:

julia> wordlength(word) = length(graphemes(word))
wordlength (generic function with 1 method)

julia> wordlength("résumé")
6

Converte tipos numéricos em strings #

Existem várias maneiras de converter tipos numéricos em strings em Julia:

julia> a = 123
123

julia> string(a)
"123"

julia> println(a)
123

A função string() também pode receber mais argumentos:

julia> string(a, "b")
"123b"

Você também pode inserir (também conhecido como interpolar) inteiros (e alguns outros tipos) em strings usando $:

julia> MyString = "my integer is $a"
"my integer is 123"

Dica de desempenho: os métodos acima podem ser bastante convenientes às vezes. Mas, se você estiver executando muitas, muitas dessas operações e estiver preocupado com a velocidade de execução do seu código, o Julia [guia de desempenho](http://docs.julialang.org/en/release-0.4/manual/performance- dicas/#avoid-string-interpolation-for-i-o) recomenda contra isso e, em vez disso, a favor dos métodos abaixo:

Você pode fornecer vários argumentos para print() e println() que irão operar neles exatamente como string() opera em vários argumentos:

julia> println(a, "b")
123b

Ou, ao gravar em arquivo, você pode usar da mesma forma, por exemplo.

open("/path/to/MyFile.txt", "w") do file
    println(file, a, "b", 13)
end

ou

file = open("/path/to/MyFile.txt", "a")
println(file, a, "b", 13)
close(file)

Eles são mais rápidos porque evitam a necessidade de formar primeiro uma string a partir de determinadas peças e depois produzi-la (para a tela do console ou para um arquivo) e, em vez disso, apenas exibem sequencialmente as várias peças.

Créditos: Resposta baseada em SO Pergunta [Qual é a melhor maneira de converter um Int em uma String em Julia?](http://stackoverflow.com/questions/38676573/whats-the-best-way-to-convert-an- int-to-a-string-in-julia) com resposta de Michael Ohlrogge e entrada de Fengyang Wang

Interpolação de string (inserir valor definido pela variável na string) #

Em Julia, como em muitas outras linguagens, é possível interpolar inserindo valores definidos por variáveis ​​em strings. Para um exemplo simples:

n = 2
julia> MyString = "there are $n ducks"
"there are 2 ducks"

Podemos usar outros tipos que não numéricos, por exemplo.

Result = false
julia> println("test results is $Result")
test results is false

Você pode ter várias interpolações em uma determinada string:

MySubStr = "a32"
MyNum = 123.31
println("$MySubStr  ,   $MyNum")

Dica de desempenho A interpolação é bastante conveniente. Mas, se você vai fazer isso muitas vezes muito rapidamente, não é o mais eficiente. Em vez disso, consulte [Converter tipos numéricos em strings](https://www.wikiod.com/pt/julia-lang/cordas#Converter tipos numéricos em strings) para obter sugestões quando o desempenho é um problema.

Usando sprint para criar strings com funções de IO #

Strings podem ser feitas a partir de funções que funcionam com objetos IO usando a função sprint. Por exemplo, a função code_llvm aceita um objeto IO como o primeiro argumento. Normalmente, é usado como

julia> code_llvm(STDOUT, *, (Int, Int))

define i64 @"jlsys_*_46115"(i64, i64) #0 {
top:
  %2 = mul i64 %1, %0
  ret i64 %2
}

Suponha que queremos essa saída como uma string. Então podemos simplesmente fazer

julia> sprint(code_llvm, *, (Int, Int))
"\ndefine i64 @\"jlsys_*_46115\"(i64, i64) #0 {\ntop:\n  %2 = mul i64 %1, %0\n  ret i64 %2\n}\n"

julia> println(ans)

define i64 @"jlsys_*_46115"(i64, i64) #0 {
top:
  %2 = mul i64 %1, %0
  ret i64 %2
}

Converter os resultados de funções “interativas” como code_llvm em strings pode ser útil para análise automatizada, como testing se o código gerado pode ter regredido.

A função sprint é uma função de ordem superior que usa a função operando em objetos IO como seu primeiro argumento. Nos bastidores, ele cria um IOBuffer na RAM, chama a função fornecida e leva os dados do buffer para um objeto String.
