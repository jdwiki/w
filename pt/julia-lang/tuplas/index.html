Sintaxe #

  • uma,
  • a, b
  • a, b = x
  • ()
  • (uma,)
  • (a, b)
  • (a, b…)
  • Tupla {T, U, V}
  • NTuplo{N, T}
  • Tupla{T, U, Vararg{V}}

As tuplas têm um desempenho de tempo de execução muito melhor do que arrays por dois motivos: seus tipos são mais precisos e sua imutabilidade permite que sejam alocados na pilha em vez do heap. No entanto, essa tipagem mais precisa vem com mais sobrecarga de tempo de compilação e mais dificuldade em alcançar [estabilidade de tipo] 2.

Introdução às Tuplas #

Tuplass são coleções ordenadas imutáveis ​​de objetos distintos arbitrários, sejam do mesmo tipo ou de tipos diferentes. Normalmente, as tuplas são construídas usando a sintaxe (x, y).

julia> tup = (1, 1.0, "Hello, World!")
(1,1.0,"Hello, World!")

Os objetos individuais de uma tupla podem ser recuperados usando a sintaxe de indexação:

julia> tup[1]
1

julia> tup[2]
1.0

julia> tup[3]
"Hello, World!"

Eles implementam a interface iterável e, portanto, podem ser iterados usando loops for:

julia> for item in tup
           println(item)
       end
1
1.0
Hello, World!

As tuplas também suportam uma variedade de funções de coleções genéricas, como reverse ou length:

julia> reverse(tup)
("Hello, World!",1.0,1)

julia> length(tup)
3

Além disso, as tuplas suportam uma variedade de operações de coleção de ordem superior, incluindo any, all, [map][5] ou broadcast:

julia> map(typeof, tup)
(Int64,Float64,String)

julia> all(x -> x < 2, (1, 2, 3))
false

julia> all(x -> x < 4, (1, 2, 3))
true

julia> any(x -> x < 2, (1, 2, 3))
true

A tupla vazia pode ser construída usando ():

julia> ()
()

julia> isempty(ans)
true

No entanto, para construir uma tupla de um elemento, é necessária uma vírgula à direita. Isso ocorre porque os parênteses (( e )) seriam tratados como operações de agrupamento em vez de construir uma tupla.

julia> (1)
1

julia> (1,)
(1,)

Para consistência, uma vírgula à direita também é permitida para tuplas com mais de um elemento.

julia> (1, 2, 3,)
(1,2,3)

[5]: https://www.wikiod.com/pt/julia-lang/funcoes-de-ordem-superior#Mapear, filtrar e reduzir

Tipos de tupla #

O typeof de uma tupla é um subtipo de Tuple:

julia> typeof((1, 2, 3))
Tuple{Int64,Int64,Int64}

julia> typeof((1.0, :x, (1, 2)))
Tuple{Float64,Symbol,Tuple{Int64,Int64}}

Ao contrário de outros tipos de dados, os tipos Tuple são covariantes. Outros tipos de dados em Julia são geralmente invariáveis. Desta forma,

julia> Tuple{Int, Int} <: Tuple{Number, Number}
true

julia> Vector{Int} <: Vector{Number}
false

Este é o caso porque em todos os lugares uma Tuple{Number, Number} é aceita, assim também uma Tuple{Int, Int}, já que ela também tem dois elementos, ambos números. Esse não é o caso de um Vector{Int} versus um Vector{Number}, pois uma função que aceita um Vector{Number} pode desejar armazenar um ponto flutuante (por exemplo, 1.0) ou um número complexo (por exemplo, 1+3im) em tal vetor.

A covariância dos tipos de tupla significa que Tuple{Number} (novamente ao contrário de Vector{Number}) é na verdade um tipo abstrato:

julia> isleaftype(Tuple{Number})
false

julia> isleaftype(Vector{Number})
true

Subtipos concretos de Tuple{Number} incluem Tuple{Int}, Tuple{Float64}, Tuple{Rational{BigInt}}, e assim por diante.

Tipos Tuple podem conter um Vararg finalizador como seu último parâmetro para indicar um número indefinido de objetos. Por exemplo, Tuple{Vararg{Int}} é o tipo de todas as tuplas contendo qualquer número de Ints, possivelmente zero:

julia> isa((), Tuple{Vararg{Int}})
true

julia> isa((1,), Tuple{Vararg{Int}})
true

julia> isa((1,2,3,4,5), Tuple{Vararg{Int}})
true

julia> isa((1.0,), Tuple{Vararg{Int}})
false

enquanto Tuple{String, Vararg{Int}} aceita tuplas consistindo de uma string, seguida por qualquer número (possivelmente zero) de Ints.

julia> isa(("x", 1, 2), Tuple{String, Vararg{Int}})
true

julia> isa((1, 2), Tuple{String, Vararg{Int}})
false

Combinado com covariância, isso significa que Tuple{Vararg{Any}} descreve qualquer tupla. Na verdade, Tuple{Vararg{Any}} é apenas outra maneira de dizer Tuple:

julia> Tuple{Vararg{Any}} == Tuple
true

Vararg aceita um segundo parâmetro de tipo numérico indicando quantas vezes exatamente seu primeiro parâmetro de tipo deve ocorrer. (Por padrão, se não especificado, este segundo parâmetro de tipo é um typevar que pode assumir qualquer valor, razão pela qual qualquer número de Ints é aceito nos Varargs acima.) Tipos Tuple terminando em um especificado Vararg será automaticamente expandido para o número de elementos solicitados:

julia> Tuple{String,Vararg{Int, 3}}
Tuple{String,Int64,Int64,Int64}

Existe notação para tuplas homogêneas com um Vararg especificado: NTuple{N, T}. Nesta notação, N denota o número de elementos na tupla e T denota o tipo aceito. Por exemplo,

julia> NTuple{3, Int}
Tuple{Int64,Int64,Int64}

julia> NTuple{10, Int}
NTuple{10,Int64}

julia> ans.types
svec(Int64,Int64,Int64,Int64,Int64,Int64,Int64,Int64,Int64,Int64)

Observe que NTuples além de um determinado tamanho são mostrados simplesmente como NTuple{N, T}, em vez da forma expandida Tuple, mas ainda são do mesmo tipo:

julia> Tuple{Int,Int,Int,Int,Int,Int,Int,Int,Int,Int}
NTuple{10,Int64}

Despachando em tipos de tupla #

Como as listas de parâmetros da função Julia são elas mesmas tuplas, dispatching em vários tipos de tuplas geralmente é mais fácil através dos próprios parâmetros do método, geralmente com uso liberal para o operador “splatting” .... Por exemplo, considere a implementação de reverse para tuplas, de Base:

revargs() = ()
revargs(x, r...) = (revargs(r...)..., x)

reverse(t::Tuple) = revargs(t...)

A implementação de métodos em tuplas dessa maneira preserva a estabilidade de tipo, que é crucial para o desempenho. Podemos ver que não há sobrecarga nessa abordagem usando a macro @code_warntype:

julia> @code_warntype reverse((1, 2, 3))
Variables:
  #self#::Base.#reverse
  t::Tuple{Int64,Int64,Int64}

Body:
  begin 
      SSAValue(1) = (Core.getfield)(t::Tuple{Int64,Int64,Int64},2)::Int64
      SSAValue(2) = (Core.getfield)(t::Tuple{Int64,Int64,Int64},3)::Int64
      return (Core.tuple)(SSAValue(2),SSAValue(1),(Core.getfield)(t::Tuple{Int64,Int64,Int64},1)::Int64)::Tuple{Int64,Int64,Int64}
  end::Tuple{Int64,Int64,Int64}

Embora um pouco difícil de ler, o código aqui está simplesmente criando uma nova tupla com valores 3º, 2º e 1º elementos da tupla original, respectivamente. Em muitas máquinas, isso é compilado em código LLVM extremamente eficiente, que consiste em cargas e armazenamentos.

julia> @code_llvm reverse((1, 2, 3))

Vários valores de retorno #

As tuplas são frequentemente usadas para vários valores de retorno. Grande parte da biblioteca padrão, incluindo duas das funções da interface iterável (next e done), retorna tuplas contendo dois valores relacionados, mas distintos.

Os parênteses em torno das tuplas podem ser omitidos em determinadas situações, facilitando a implementação de vários valores de retorno. Por exemplo, podemos criar uma função para retornar raízes quadradas positivas e negativas de um número real:

julia> pmsqrt(x::Real) = sqrt(x), -sqrt(x)
pmsqrt (generic function with 1 method)

julia> pmsqrt(4)
(2.0,-2.0)

A atribuição de desestruturação pode ser usada para descompactar os vários valores de retorno. Para armazenar as raízes quadradas nas variáveis ​​a e b, basta escrever:

julia> a, b = pmsqrt(9.0)
(3.0,-3.0)

julia> a
3.0

julia> b
-3.0

Outro exemplo disso são as funções divrem e fldmod, que fazem uma divisão inteira (truncada ou com piso, respectivamente) e operação de resto ao mesmo tempo:

julia> q, r = divrem(10, 3)
(3,1)

julia> q
3

julia> r
1
