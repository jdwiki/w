Parâmetros #

Parâmetro Detalhes
chomp(str) Remova até uma nova linha à direita de uma string.
str A string da qual remover uma nova linha à direita. Observe que strings são imutáveis ​​por convenção. Esta função retorna uma nova string.
open(f, arquivo) Abra um arquivo, chame a função e feche o arquivo depois.
f A função para chamar o fluxo de E/S que abre o arquivo é gerada.
arquivo O caminho do arquivo a ser aberto.

Lendo uma string da entrada padrão #

O fluxo STDIN em Julia se refere a entrada padrão. Isso pode representar a entrada do usuário, para programas de linha de comando interativos, ou a entrada de um arquivo ou pipeline que foi redirecionado para o programa.

A função readline, quando não for fornecido nenhum argumento, lerá dados de STDIN até que uma nova linha seja encontrada, ou o fluxo STDIN entre no estado de fim de arquivo. Esses dois casos podem ser distinguidos se o caractere \n foi lido como o caractere final:

julia> readline()
some stuff
"some stuff\n"

julia> readline()  # Ctrl-D pressed to send EOF signal here
""

Muitas vezes, para programas interativos, não nos importamos com o estado EOF e queremos apenas uma string. Por exemplo, podemos solicitar a entrada do usuário:

function askname()
    print("Enter your name: ")
    readline()
end

Isso não é muito satisfatório, no entanto, por causa da nova linha adicional:

julia> askname()
Enter your name: Julia
"Julia\n"

A função chomp está disponível para remover até uma nova linha final de uma string. Por exemplo:

julia> chomp("Hello, World!")
"Hello, World!"

julia> chomp("Hello, World!\n")
"Hello, World!"

Podemos, portanto, aumentar nossa função com chomp para que o resultado seja o esperado:

function askname()
    print("Enter your name: ")
    chomp(readline())
end

que tem um resultado mais desejável:

julia> askname()
Enter your name: Julia
"Julia"

Às vezes, podemos querer ler tantas linhas quanto possível (até que o fluxo de entrada entre no estado de fim de arquivo). A função readlines fornece essa capacidade.

julia> readlines()  # note Ctrl-D is pressed after the last line
A, B, C, D, E, F, G
H, I, J, K, LMNO, P
Q, R, S
T, U, V
W, X  
Y, Z
6-element Array{String,1}:
 "A, B, C, D, E, F, G\n"
 "H, I, J, K, LMNO, P\n"
 "Q, R, S\n"
 "T, U, V\n"
 "W, X\n"
 "Y, Z\n"

Mais uma vez, se não gostarmos das novas linhas no final das linhas lidas por readlines, podemos usar a função chomp para removê-las. Desta vez, nós transmitimos a função chomp em todo o array:

julia> chomp.(readlines())
A, B, C, D, E, F, G
H, I, J, K, LMNO, P
Q, R, S
T, U, V
W, X  
Y, Z
6-element Array{String,1}:
 "A, B, C, D, E, F, G"
 "H, I, J, K, LMNO, P"
 "Q, R, S"            
 "T, U, V"            
 "W, X  "             
 "Y, Z"

Outras vezes, podemos não nos importar com as linhas e simplesmente queremos ler o máximo possível como uma única string. A função readstring faz isso:

julia> readstring(STDIN)
If music be the food of love, play on,
Give me excess of it; that surfeiting,
The appetite may sicken, and so die.  # [END OF INPUT]
"If music be the food of love, play on,\nGive me excess of it; that surfeiting,\nThe appetite may sicken, and so die.\n"

(o # [END OF INPUT] não faz parte da entrada original; foi adicionado para maior clareza.)

Observe que readstring deve receber o argumento STDIN.

Leitura de números da entrada padrão #

A leitura de números da entrada padrão é uma combinação de leitura de strings e análise de strings como números.

A função parse é usada para analisar uma string no tipo de número desejado:

julia> parse(Int, "17")
17

julia> parse(Float32, "-3e6")
-3.0f6

O formato esperado por parse(T, x) é semelhante, mas não exatamente o mesmo, como o formato que Julia espera de literais de número:

julia> -00000023
-23

julia> parse(Int, "-00000023")
-23

julia> 0x23 |> Int
35

julia> parse(Int, "0x23")
35

julia> 1_000_000
1000000

julia> parse(Int, "1_000_000")
ERROR: ArgumentError: invalid base 10 digit '_' in "1_000_000"
 in tryparse_internal(::Type{Int64}, ::String, ::Int64, ::Int64, ::Int64, ::Bool) at ./parse.jl:88
 in parse(::Type{Int64}, ::String) at ./parse.jl:152

A combinação das funções parse e ​​readline nos permite ler um único número de uma linha:

function asknumber()
    print("Enter a number: ")
    parse(Float64, readline())
end

que funciona como esperado:

julia> asknumber()
Enter a number: 78.3
78.3

As advertências usuais sobre precisão de ponto flutuante se aplicam. Observe que parse pode ser usado com BigInt e BigFloat para remover ou minimizar a perda de precisão.

Às vezes, é útil ler mais de um número da mesma linha. Normalmente, a linha pode ser dividida com espaços em branco:

function askints()
    print("Enter some integers, separated by spaces: ")
    [parse(Int, x) for x in split(readline())]
end

que pode ser usado da seguinte forma:

julia> askints()
Enter some integers, separated by spaces: 1 2 3 4
4-element Array{Int64,1}:
 1
 2
 3
 4

Lendo dados de um arquivo #

Lendo strings ou bytes #

Arquivos podem ser abertos para leitura usando a função open, que é frequentemente usada junto com do bloco sintaxe:

open("myfile") do f
    for (i, line) in enumerate(eachline(f))
        print("Line $i: $line")
    end
end

Suponha que myfile exista e seu conteúdo seja

What's in a name? That which we call a rose
By any other name would smell as sweet.

Então, esse código produziria o seguinte resultado:

Line 1: What's in a name? That which we call a rose
Line 2: By any other name would smell as sweet.

Observe que eachline é um iterável preguiçoso sobre as linhas do arquivo. É preferível readlines por motivos de desempenho.

Como a sintaxe do bloco do é apenas açúcar sintático para funções anônimas, podemos passar funções nomeadas para open também:

julia> open(readstring, "myfile")
"What's in a name? That which we call a rose\nBy any other name would smell as sweet.\n"

julia> open(read, "myfile")

As funções read e readstring fornecem métodos convenientes que abrem um arquivo automaticamente:

julia> readstring("myfile")
"What's in a name? That which we call a rose\nBy any other name would smell as sweet.\n"

Lendo dados estruturados #

Suponha que tivéssemos um [arquivo CSV][3] com o seguinte conteúdo, em um arquivo chamado file.csv:

Make,Model,Price
Foo,2015A,8000
Foo,2015B,14000
Foo,2016A,10000
Foo,2016B,16000
Bar,2016Q,20000

Então podemos usar a função readcsv para ler esses dados em uma Matrix:

julia> readcsv("file.csv")
6×3 Array{Any,2}:
 "Make"  "Model"       "Price"
 "Foo"   "2015A"   8000       
 "Foo"   "2015B"  14000       
 "Foo"   "2016A"  10000       
 "Foo"   "2016B"  16000       
 "Bar"   "2016Q"  20000

Se o arquivo foi delimitado por tabulações, em um arquivo chamado file.tsv, então a função readdlm pode ser usada, com o argumento delim definido como '\t'. Cargas de trabalho mais avançadas devem usar o CSV.jl [pacote][4].
