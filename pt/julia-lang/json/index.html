Sintaxe #

  • usando JSON
  • JSON.parse(str)
  • JSON.json(obj)
  • JSON.print(io, obj, recuo)

Como nem os objetos Julia Dict nem JSON são inerentemente ordenados, é melhor não confiar na ordem dos pares de valores-chave em um objeto JSON.

Instalando JSON.jl #

JSON é um formato popular de intercâmbio de dados. A biblioteca JSON mais popular para Julia é JSON.jl. Para instalar este pacote, use o gerenciador de pacotes:

julia> Pkg.add("JSON")

O próximo passo é testar se o pacote está funcionando em sua máquina:

julia> Pkg.test("JSON")

Se todos os testes forem aprovados, a biblioteca estará pronta para uso.

Analisando JSON #

O JSON que foi codificado como uma string pode ser facilmente analisado em um tipo Julia padrão:

julia> using JSON

julia> JSON.parse("""{
           "this": ["is", "json"],
           "numbers": [85, 16, 12.0],
           "and": [true, false, null]
       }""")
Dict{String,Any} with 3 entries:
  "this"    => Any["is","json"]
  "numbers" => Any[85,16,12.0]
  "and"     => Any[true,false,nothing]

Há algumas propriedades imediatas de JSON.jl dignas de nota:

  • Tipos JSON mapeiam para tipos sensíveis em Julia: Object torna-se Dict, array torna-se Vector, number torna-se Int64 ou Float64, boolean torna-se Bool e null torna-se nothing::Void.
  • JSON é um formato de contêiner não tipado: Assim, os vetores Julia retornados são do tipo Vector{Any}, e os dicionários retornados são do tipo Dict{String, Any}.
  • O padrão JSON não distingue entre números inteiros e decimais, mas JSON.jl sim. Um número sem ponto decimal ou notação científica é analisado em Int64, enquanto um número com um ponto decimal é analisado em Float64. Isso combina de perto com o comportamento dos analisadores JSON em muitas outras linguagens.

Serializando JSON #

A função JSON.json serializa um objeto Julia em uma String Julia contendo JSON:

julia> using JSON

julia> JSON.json(Dict(:a => :b, :c => [1, 2, 3.0], :d => nothing))
"{\"c\":[1.0,2.0,3.0],\"a\":\"b\",\"d\":null}"

julia> println(ans)
{"c":[1.0,2.0,3.0],"a":"b","d":null}

Se uma string não for desejada, o JSON pode ser impresso diretamente em um fluxo de E/S:

julia> JSON.print(STDOUT, [1, 2, true, false, "x"])
[1,2,true,false,"x"]

Observe que STDOUT é o padrão e pode ser omitido na chamada acima.

Uma impressão mais bonita pode ser obtida passando o parâmetro opcional indent:

julia> JSON.print(STDOUT, Dict(:a => :b, :c => :d), 4)
{
    "c": "d",
    "a": "b"
}

Existe uma serialização padrão sã para tipos complexos de Julia:

julia> immutable Point3D
           x::Float64
           y::Float64
           z::Float64
       end

julia> JSON.print(Point3D(1.0, 2.0, 3.0), 4)
{
    "y": 2.0,
    "z": 3.0,
    "x": 1.0
}
