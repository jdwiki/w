Sintaxe #

  • rótulo @goto
  • rótulo @label

O uso excessivo ou inadequado do fluxo de controle avançado dificulta a leitura do código. @goto ou seus equivalentes em outros idiomas, quando usados ​​incorretamente, levam a um código de espaguete ilegível.

Semelhante a linguagens como C, não se pode pular entre funções em Julia. Isso também significa que @goto não é possível no nível superior; ele só funcionará dentro de uma função. Além disso, não se pode saltar de uma função interna para sua função externa, ou de uma função externa para uma função interna.

Validação de entrada #

Embora não sejam tradicionalmente considerados loops, as macros @goto e @label podem ser usadas para um fluxo de controle mais avançado. Um caso de uso é quando a falha de uma parte deve levar à repetição de uma função inteira, geralmente útil na validação de entrada:

function getsequence()
    local a, b

@label start
    print("Input an integer: ")
    try
        a = parse(Int, readline())
    catch
        println("Sorry, that's not an integer.")
        @goto start
    end

    print("Input a decimal: ")
    try
        b = parse(Float64, readline())
    catch
        println("Sorry, that doesn't look numeric.")
        @goto start
    end

    a, b
end

No entanto, esse caso de uso geralmente é mais claro usando recursão:

function getsequence()
    local a, b

    print("Input an integer: ")
    try
        a = parse(Int, readline())
    catch
        println("Sorry, that's not an integer.")
        return getsequence()
    end

    print("Input a decimal: ")
    try
        b = parse(Float64, readline())
    catch
        println("Sorry, that doesn't look numeric.")
        return getsequence()
    end

    a, b
end

Embora ambos os exemplos façam a mesma coisa, o segundo é mais fácil de entender. No entanto, o primeiro é mais performático (porque evita a chamada recursiva). Na maioria dos casos, o custo da chamada não importa; mas em situações limitadas, a primeira forma é aceitável.

Limpeza de erros #

Em linguagens como C, a instrução @goto é frequentemente usada para garantir que uma função limpe os recursos necessários, mesmo no caso de um erro. Isso é menos importante em Julia, porque exceções e blocos try-finally são frequentemente usados.

No entanto, é possível que o código Julia faça interface com o código C e as APIs C e, portanto, às vezes, as funções ainda precisam ser escritas como o código C. O exemplo abaixo é artificial, mas demonstra um caso de uso comum. O código Julia chamará Libc.malloc para alocar alguma memória (isso simula uma chamada de API C). Se nem todas as alocações forem bem-sucedidas, a função deverá liberar os recursos obtidos até o momento; caso contrário, a memória alocada é retornada.

using Base.Libc
function allocate_some_memory()
    mem1 = malloc(100)
    mem1 == C_NULL && @goto fail
    mem2 = malloc(200)
    mem2 == C_NULL && @goto fail
    mem3 = malloc(300)
    mem3 == C_NULL && @goto fail
    return mem1, mem2, mem3

@label fail
    free(mem1)
    free(mem2)
    free(mem3)
end
