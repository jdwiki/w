Sintaxe #

  • MyType imutável; campo; campo; fim
  • digite MeuTipo; campo; campo; fim

Os tipos são fundamentais para o desempenho de Julia. Uma ideia importante para o desempenho é a estabilidade de tipo, que ocorre quando o tipo que uma função retorna depende apenas dos tipos, não dos valores, de seus argumentos.

Despachando em Tipos #

Em Julia, você pode definir mais de um método para cada função. Suponha que definamos três métodos da mesma função:

foo(x) = 1
foo(x::Number) = 2
foo(x::Int) = 3

Ao decidir qual método usar (chamado dispatch), Julia escolhe o método mais específico que corresponde aos tipos de argumentos:

julia> foo('one')
1

julia> foo(1.0)
2

julia> foo(1)
3

Isso facilita o polimorfismo. Por exemplo, podemos criar facilmente uma lista vinculada definindo dois tipos imutáveis, chamados Nil e Cons. Esses nomes são tradicionalmente usados ​​para descrever uma lista vazia e uma lista não vazia, respectivamente.

abstract LinkedList
immutable Nil <: LinkedList end
immutable Cons <: LinkedList
    first
    rest::LinkedList
end

Representaremos a lista vazia por Nil() e quaisquer outras listas por Cons(first, rest), onde first é o primeiro elemento da lista encadeada e rest é a lista encadeada que consiste em todos os restantes elementos. Por exemplo, a lista [1, 2, 3] será representada como

julia> Cons(1, Cons(2, Cons(3, Nil())))
Cons(1,Cons(2,Cons(3,Nil())))

A lista está vazia? #

Suponha que queremos estender a função isempty da biblioteca padrão, que funciona em várias coleções diferentes:

julia> methods(isempty)
# 29 methods for generic function "isempty":
isempty(v::SimpleVector) at essentials.jl:180
isempty(m::Base.MethodList) at reflection.jl:394
...

Podemos simplesmente usar a sintaxe de despacho da função e definir dois métodos adicionais de isempty. Como esta função é do módulo Base, temos que qualificá-la como Base.isempty para estendê-la.

Base.isempty(::Nil) = true
Base.isempty(::Cons) = false

Aqui, não precisamos dos valores dos argumentos para determinar se a lista está vazia. Apenas o tipo por si só é suficiente para computar essa informação. Julia nos permite omitir os nomes dos argumentos, mantendo apenas sua anotação de tipo, caso não precisemos usar seus valores.

Podemos testar que nossos métodos isempty funcionam:

julia> using Base.Test

julia> @test isempty(Nil())
Test Passed
  Expression: isempty(Nil())

julia> @test !isempty(Cons(1, Cons(2, Cons(3, Nil()))))
Test Passed
  Expression: !(isempty(Cons(1,Cons(2,Cons(3,Nil())))))

e, de fato, o número de métodos para isempty aumentou em 2:

julia> methods(isempty)
# 31 methods for generic function "isempty":
isempty(v::SimpleVector) at essentials.jl:180
isempty(m::Base.MethodList) at reflection.jl:394

Claramente, determinar se uma lista encadeada está vazia ou não é um exemplo trivial. Mas isso leva a algo mais interessante:

Qual é o tamanho da lista? #

A função length da biblioteca padrão nos dá o comprimento de uma coleção ou certos iteráveis. Existem muitas maneiras de implementar length para uma lista encadeada. Em particular, usar um loop while é provavelmente mais rápido e mais eficiente em memória em Julia. Mas otimização prematura deve ser evitada, então vamos supor por um segundo que nossa lista vinculada não precisa ser eficiente. Qual é a maneira mais simples de escrever uma função length?

Base.length(::Nil) = 0
Base.length(xs::Cons) = 1 + length(xs.rest)

A primeira definição é direta: uma lista vazia tem comprimento ‘0’. A segunda definição também é fácil de ler: para contar o comprimento de uma lista, contamos o primeiro elemento e depois contamos o comprimento do resto da lista. Podemos testar esse método da mesma forma que testamos isempty:

julia> @test length(Nil()) == 0
Test Passed
  Expression: length(Nil()) == 0
   Evaluated: 0 == 0

julia> @test length(Cons(1, Cons(2, Cons(3, Nil())))) == 3
Test Passed
  Expression: length(Cons(1,Cons(2,Cons(3,Nil())))) == 3
   Evaluated: 3 == 3

Próximos passos #

Este exemplo de brinquedo está muito longe de implementar todas as funcionalidades que seriam desejadas em uma lista vinculada. Está faltando, por exemplo, a interface de iteração. No entanto, ilustra como o dispatch pode ser usado para escrever código curto e claro.

Tipos imutáveis #

O tipo composto mais simples é um tipo imutável. Instâncias de tipos imutáveis, como tuplas, são valores. Seus campos não podem ser alterados depois de criados. De muitas maneiras, um tipo imutável é como uma Tupla com nomes para o próprio tipo e para cada campo.

Tipos de singleton #

Os tipos compostos, por definição, contêm vários tipos mais simples. Em Julia, esse número pode ser zero; ou seja, um tipo imutável pode conter no campos. Isso é comparável à tupla vazia ().

Por que isso pode ser útil? Esses tipos imutáveis ​​são conhecidos como “tipos singleton”, pois apenas uma instância deles poderia existir. Os valores de tais tipos são conhecidos como “valores únicos”. A biblioteca padrão Base contém muitos desses tipos singleton. Aqui está uma breve lista:

  • Void, o tipo de nada. Podemos verificar que Void.instance (que é uma sintaxe especial para recuperar o valor singleton de um tipo singleton) é realmente nada.
  • Qualquer tipo de mídia, como MIME"text/plain", é um tipo singleton com uma única instância, MIME("text/plain").
  • Os tipos Irrational{:π}, Irrational{:e}, Irrational{:φ} e similares são tipos singleton, e suas instâncias singleton são os valores irracionais π = 3,1415926535897..., etc.
  • As características de tamanho do iterador Base.HasLength, Base.HasShape, Base.IsInfinite e Base.SizeUnknown são todos tipos singleton.
  • Na versão 0.5 e posterior, cada função é uma instância singleton de um tipo singleton! Como qualquer outro valor singleton, podemos recuperar a função sin, por exemplo, de typeof(sin).instance.

Como não contêm nada, os tipos singleton são incrivelmente leves e frequentemente podem ser otimizados pelo compilador para não ter sobrecarga de tempo de execução. Assim, eles são perfeitos para características, valores de tags especiais e para coisas como funções nas quais alguém gostaria de se especializar.

Para definir um tipo singleton,

julia> immutable MySingleton end

Para definir a impressão personalizada para o tipo singleton,

julia> Base.show(io::IO, ::MySingleton) = print(io, "sing")

Para acessar a instância singleton,

julia> MySingleton.instance
MySingleton()

Muitas vezes, atribui-se isso a uma constante:

julia> const sing = MySingleton.instance
MySingleton()

Tipos de invólucro #

Se os tipos imutáveis ​​de campo zero são interessantes e úteis, então talvez os tipos imutáveis ​​de um campo sejam ainda mais úteis. Esses tipos são comumente chamados de “tipos wrapper” porque envolvem alguns dados subjacentes, fornecendo uma interface alternativa para esses dados. Um exemplo de um tipo de wrapper em Base é String. Vamos definir um tipo semelhante a String, chamado MyString. Este tipo será apoiado por um vetor (unidimensional array) de bytes (UInt8).

Primeiro, a própria definição de tipo e algumas exibições personalizadas:

immutable MyString <: AbstractString
    data::Vector{UInt8}
end

function Base.show(io::IO, s::MyString)
    print(io, "MyString: ")
    write(io, s.data)
    return
end

Agora nosso tipo MyString está pronto para uso! Podemos alimentá-lo com alguns dados UTF-8 brutos e ele é exibido como gostamos:

julia> MyString([0x48,0x65,0x6c,0x6c,0x6f,0x2c,0x20,0x57,0x6f,0x72,0x6c,0x64,0x21])
MyString: Hello, World!

Obviamente, este tipo de string precisa de muito trabalho antes de se tornar tão utilizável quanto o tipo Base.String.

Tipos compostos verdadeiros #

Talvez mais comumente, muitos tipos imutáveis ​​contenham mais de um campo. Um exemplo é a biblioteca padrão do tipo Rational{T}, que contém dois campos: um campo num para o numerador e um campo den para o denominador. É bastante simples emular este design de tipo:

immutable MyRational{T}
    num::T
    den::T
    MyRational(n, d) = (g = gcd(n, d); new(n÷g, d÷g))
end
MyRational{T}(n::T, d::T) = MyRational{T}(n, d)

Implementamos com sucesso um construtor que simplifica nossos números racionais:

julia> MyRational(10, 6)
MyRational{Int64}(5,3)
