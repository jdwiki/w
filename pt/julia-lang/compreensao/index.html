Compreensão da matriz #

Sintaxe Básica #

As compreensões de array de Julia usam a seguinte sintaxe:

[expression for element = iterable]

Observe que, como com for loops, todos =, in e ∈ são aceitos para a compreensão.

Isso é aproximadamente equivalente a criar um array vazio e usar um loop for para push! itens nele.

result = []
for element in iterable
    push!(result, expression)
end

no entanto, o tipo de compreensão de uma matriz é o mais estreito possível, o que é melhor para o desempenho.

Por exemplo, para obter um array dos quadrados dos inteiros de 1 a 10, o código a seguir pode ser usado.

squares = [x^2 for x=1:10]

Este é um substituto limpo e conciso para a versão mais longa do for-loop.

squares = []
for x in 1:10
    push!(squares, x^2)
end

Compreensão de matriz condicional #

Antes do Julia 0.5, não havia como usar condições dentro das compreensões do array. Mas, não é mais verdade. Em Julia 0.5 podemos usar as condições dentro de condições como as seguintes:

julia> [x^2 for x in 0:9 if x > 5] 
4-element Array{Int64,1}:
 36
 49
 64
 81

A fonte do exemplo acima pode ser encontrada aqui.

Se quisermos usar a compreensão de lista aninhada:

julia>[(x,y) for x=1:5 , y=3:6 if y>4 && x>3 ]
4-element Array{Tuple{Int64,Int64},1}:
 (4,5)
 (5,5)
 (4,6)
 (5,6)

Compreensão de arrays multidimensionais #

Loops for aninhados podem ser usados ​​para iterar sobre vários iteráveis ​​únicos.

result = []
for a = iterable_a
    for b = iterable_b
        push!(result, expression)
    end
end

Da mesma forma, várias especificações de iteração podem ser fornecidas para uma compreensão de matriz.

[expression for a = iterable_a, b = iterable_b]

Por exemplo, o seguinte pode ser usado para gerar o produto cartesiano de 1:3 e 1:2.

julia> [(x, y) for x = 1:3, y = 1:2]
3×2 Array{Tuple{Int64,Int64},2}:
 (1,1)  (1,2)
 (2,1)  (2,2)
 (3,1)  (3,2)

As compreensões de matrizes multidimensionais achatadas são semelhantes, exceto que elas perdem a forma. Por exemplo,

julia> [(x, y) for x = 1:3 for y = 1:2]
6-element Array{Tuple{Int64,Int64},1}:
 (1, 1)
 (1, 2)
 (2, 1)
 (2, 2)
 (3, 1)
 (3, 2)

é uma variante achatada do anterior. A diferença sintática é que um for adicional é usado em vez de uma vírgula.
