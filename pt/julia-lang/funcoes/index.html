Sintaxe #

  • f(n) = …
  • função f(n) … fim
  • n::Tipo
  • x -> …
  • f(n) do… fim

Além das funções genéricas (que são mais comuns), também existem funções internas. Tais funções incluem is, isa, typeof, throw e funções semelhantes. As funções incorporadas são normalmente implementadas em C em vez de Julia, portanto, não podem ser especializadas em tipos de argumento para despacho.

Introdução ao Despacho #

Podemos usar a sintaxe :: para despachar no type de um argumento.

describe(n::Integer) = "integer $n"
describe(n::AbstractFloat) = "floating point $n"

Uso:

julia> describe(10)
"integer 10"

julia> describe(1.0)
"floating point 1.0"

Ao contrário de muitas linguagens, que normalmente fornecem despacho múltiplo estático ou despacho único dinâmico, Julia tem despacho múltiplo dinâmico completo. Ou seja, as funções podem ser especializadas para mais de um argumento. Isso é útil ao definir métodos especializados para operações em determinados tipos e métodos de fallback para outros tipos.

describe(n::Integer, m::Integer) = "integers n=$n and m=$m"
describe(n, m::Integer) = "only m=$m is an integer"
describe(n::Integer, m) = "only n=$n is an integer"

Uso:

julia> describe(10, 'x')
"only n=10 is an integer"

julia> describe('x', 10)
"only m=10 is an integer"

julia> describe(10, 10)
"integers n=10 and m=10"

Argumentos Opcionais #

Julia permite que as funções recebam argumentos opcionais. Nos bastidores, isso é implementado como outro caso especial de despacho múltiplo. Por exemplo, vamos resolver o popular problema do Fizz Buzz. Por padrão, faremos isso para números no intervalo 1:10, mas permitiremos um valor diferente, se necessário. Também permitiremos que diferentes frases sejam usadas para Fizz ou Buzz.

function fizzbuzz(xs=1:10, fizz="Fizz", buzz="Buzz")
    for i in xs
        if i % 15 == 0
            println(fizz, buzz)
        elseif i % 3 == 0
            println(fizz)
        elseif i % 5 == 0
            println(buzz)
        else
            println(i)
        end
    end
end

Se inspecionarmos o fizzbuzz no REPL, ele diz que existem quatro métodos. Um método foi criado para cada combinação de argumentos permitida.

julia> fizzbuzz
fizzbuzz (generic function with 4 methods)

julia> methods(fizzbuzz)
# 4 methods for generic function "fizzbuzz":
fizzbuzz() at REPL[96]:2
fizzbuzz(xs) at REPL[96]:2
fizzbuzz(xs, fizz) at REPL[96]:2
fizzbuzz(xs, fizz, buzz) at REPL[96]:2

Podemos verificar se nossos valores padrão são usados ​​quando nenhum parâmetro é fornecido:

julia> fizzbuzz()
1
2
Fizz
4
Buzz
Fizz
7
8
Fizz
Buzz

mas que os parâmetros opcionais são aceitos e respeitados se os fornecermos:

julia> fizzbuzz(5:8, "fuzz", "bizz")
bizz
fuzz
7
8

Despacho Paramétrico #

É comum que uma função despache em tipos paramétricos, como Vector{T} ou Dict{K,V}, mas os parâmetros de tipo não são fixos. Este caso pode ser tratado usando despacho paramétrico:

julia> foo{T<:Number}(xs::Vector{T}) = @show xs .+ 1
foo (generic function with 1 method)

julia> foo(xs::Vector) = @show xs
foo (generic function with 2 methods)

julia> foo([1, 2, 3])
xs .+ 1 = [2,3,4]
3-element Array{Int64,1}:
 2
 3
 4

julia> foo([1.0, 2.0, 3.0])
xs .+ 1 = [2.0,3.0,4.0]
3-element Array{Float64,1}:
 2.0
 3.0
 4.0

julia> foo(["x", "y", "z"])
xs = String["x","y","z"]
3-element Array{String,1}:
 "x"
 "y"
 "z"

Alguém pode ficar tentado a simplesmente escrever xs::Vector{Number}. Mas isso só funciona para objetos cujo tipo é explicitamente Vector{Number}:

julia> isa(Number[1, 2], Vector{Number})
true

julia> isa(Int[1, 2], Vector{Number})
false

Isso se deve a [invariância paramétrica][4]: o objeto Int[1, 2] é não um Vector{Number}, porque só pode conter Ints, enquanto um Vector{Number} seria esperado para ser capaz de conter qualquer tipo de números.

Escrevendo código genérico #

Dispatch é um recurso incrivelmente poderoso, mas frequentemente é melhor escrever código genérico que funcione para todos os tipos, em vez de código especializado para cada tipo. Escrever código genérico evita a duplicação de código.

Por exemplo, aqui está o código para calcular a soma dos quadrados de um vetor de inteiros:

function sumsq(v::Vector{Int})
    s = 0
    for x in v
        s += x ^ 2
    end
    s
end

Mas este código somente funciona para um vetor de Ints. Não funcionará em um UnitRange:

julia> sumsq(1:10)
ERROR: MethodError: no method matching sumsq(::UnitRange{Int64})
Closest candidates are:
  sumsq(::Array{Int64,1}) at REPL[8]:2

Não funcionará em um Vector{Float64}:

julia> sumsq([1.0, 2.0])
ERROR: MethodError: no method matching sumsq(::Array{Float64,1})
Closest candidates are:
  sumsq(::Array{Int64,1}) at REPL[8]:2

Uma maneira melhor de escrever esta função sumsq deve ser

function sumsq(v::AbstractVector)
    s = zero(eltype(v))
    for x in v
        s += x ^ 2
    end
    s
end

Isso funcionará nos dois casos listados acima. Mas existem algumas coleções que podemos querer somar os quadrados que não são vetores, em nenhum sentido. Por exemplo,

julia> sumsq(take(countfrom(1), 100))
ERROR: MethodError: no method matching sumsq(::Base.Take{Base.Count{Int64}})
Closest candidates are:
  sumsq(::Array{Int64,1}) at REPL[8]:2
  sumsq(::AbstractArray{T,1}) at REPL[11]:2

mostra que não podemos somar os quadrados de um [lazy iterable][5].

Uma implementação ainda mais genérica é simplesmente

function sumsq(v)
    s = zero(eltype(v))
    for x in v
        s += x ^ 2
    end
    s
end

Que funciona em todos os casos:

julia> sumsq(take(countfrom(1), 100))
338350

Este é o código Julia mais idiomático e pode lidar com todos os tipos de situações. Em algumas outras linguagens, a remoção de anotações de tipo pode afetar o desempenho, mas esse não é o caso de Julia; apenas [estabilidade de tipo][6] é importante para o desempenho.

Quadrado de um número #

Esta é a sintaxe mais fácil para definir uma função:

square(n) = n * n

Para chamar uma função, use colchetes (sem espaços entre):

julia> square(10)
100

Funções são objetos em Julia, e podemos mostrá-las no REPL como em qualquer outro objeto:

julia> square
square (generic function with 1 method)

Todas as funções Julia são genéricas (também conhecidas como polimórficas) por padrão. Nossa função square funciona tão bem com valores de ponto flutuante:

julia> square(2.5)
6.25

…ou mesmo matrizes:

julia> square([2 4
               2 1])
2×2 Array{Int64,2}:
 12  12
  6   9

Funções recursivas #

Recursão simples #

Usando recursão e o operador condicional ternário, podemos criar uma implementação alternativa da função factorial integrada:

myfactorial(n) = n == 0 ? 1 : n * myfactorial(n - 1)

Uso:

julia> myfactorial(10)
3628800

Trabalhando com árvores #

As funções recursivas geralmente são mais úteis em estruturas de dados, especialmente estruturas de dados em árvore. Como expressions em Julia são estruturas de árvore, a recursão pode ser bastante útil para metaprogramming. Por exemplo, a função abaixo reúne um conjunto de todas as cabeças usadas em uma expressão.

heads(ex::Expr) = reduce(∪, Set((ex.head,)), (heads(a) for a in ex.args))
heads(::Any) = Set{Symbol}()

Podemos verificar se nossa função está funcionando como pretendido:

julia> heads(:(7 + 4x > 1 > A[0]))
Set(Symbol[:comparison,:ref,:call])

Esta função é compacta e usa uma variedade de técnicas mais avançadas, como a reduce [função de ordem superior][4], o tipo de dados Set e expressões geradoras.

Fatorial imperativo #

Uma sintaxe de formato longo está disponível para definir funções de várias linhas. Isso pode ser útil quando usamos estruturas imperativas, como loops. A expressão na posição de cauda é retornada. Por exemplo, a função abaixo usa um loop [for] 1 para calcular o fatorial de algum inteiro n:

function myfactorial(n)
    fact = one(n)
    for m in 1:n
        fact *= m
    end
    fact
end

Uso:

julia> myfactorial(10)
3628800

Em funções mais longas, é comum ver a instrução return usada. A instrução return não é necessária na posição da cauda, ​​mas ainda é usada algumas vezes para maior clareza. Por exemplo, outra maneira de escrever a função acima seria

function myfactorial(n)
    fact = one(n)
    for m in 1:n
        fact *= m
    end
    return fact
end

que é idêntico em comportamento à função acima.

Funções anônimas #

Sintaxe da seta #

Funções anônimas podem ser criadas usando a sintaxe ->. Isso é útil para passar funções para funções de ordem superior, como a função map. A função abaixo calcula o quadrado de cada número em um array A.

squareall(A) = map(x -> x ^ 2, A)

Um exemplo de uso desta função:

julia> squareall(1:10)
10-element Array{Int64,1}:
   1
   4
   9
  16
  25
  36
  49
  64
  81
 100

Sintaxe de várias linhas #

Funções anônimas de várias linhas podem ser criadas usando a sintaxe function. Por exemplo, o exemplo a seguir calcula o [factorials][4] dos primeiros n números, mas usando uma função anônima em vez do factorial embutido.

julia> map(function (n)
               product = one(n)
               for i in 1:n
                   product *= i
               end
               product
           end, 1:10)
10-element Array{Int64,1}:
       1
       2
       6
      24
     120
     720
    5040
   40320
  362880
 3628800

Sintaxe do bloco #

Por ser tão comum passar uma função anônima como o primeiro argumento para uma função, existe uma sintaxe de bloco do. A sintaxe

map(A) do x
    x ^ 2
end

é equivalente a

map(x -> x ^ 2, A)

mas o primeiro pode ser mais claro em muitas situações, especialmente se muita computação estiver sendo feita na função anônima. A sintaxe do bloco do é especialmente útil para [entrada e saída de arquivo][5] por motivos de gerenciamento de recursos.
