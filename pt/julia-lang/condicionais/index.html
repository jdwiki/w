Sintaxe #

  • se cond; corpo; fim
  • se cond; corpo; senão; corpo; fim
  • se cond; corpo; outra condição; corpo; senão; fim
  • se cond; corpo; outra condição; corpo; fim -cond? iftrue : iffalse
  • cond && se verdadeiro
  • condição || se for falso
  • ifelse(cond, iftrue, iffalse)

Todos os operadores e funções condicionais envolvem o uso de condições booleanas (true ou false). Em Julia, o tipo de booleanos é Bool. Ao contrário de algumas outras linguagens, outros tipos de números (como 1 ou 0), strings, arrays e assim por diante _não podem ser usados ​​diretamente em condicionais.

Normalmente, usa-se funções de predicado (funções que retornam um ‘Bool’) ou operadores de comparação na condição de um operador ou função condicional.

Operadores de curto-circuito: && e || #

Para ramificação #

Os operadores condicionais de curto-circuito && e || podem ser usados ​​como substitutos leves para as seguintes construções:

  • x && y é equivalente a x ? y: x
  • x || y é equivalente a x ? x: y

Um uso para operadores de curto-circuito é como uma maneira mais concisa de testar uma condição e executar uma determinada ação dependendo dessa condição. Por exemplo, o código a seguir usa o operador && para gerar um erro se o argumento x for negativo:

function mysqrt(x)
    x < 0 && throw(DomainError("x is negative"))
    x ^ 0.5
end

O operador || também pode ser usado para verificação de erros, exceto que ele aciona o erro _a menos que uma condição seja válida, em vez de se a condição for válida:

function halve(x::Integer)
    iseven(x) || throw(DomainError("cannot halve an odd number"))
    x ÷ 2
end

Outra aplicação útil disso é fornecer um valor padrão para um objeto, somente se não estiver definido anteriormente:

isdefined(:x) || (x = NEW_VALUE)

Aqui, isso verifica se o símbolo x está definido (ou seja, se há um valor atribuído ao objeto x). Se sim, então nada acontece. Mas, se não, então x será atribuído NEW_VALUE. Observe que este exemplo funcionará apenas no escopo de nível superior.

Em condições #

Os operadores também são úteis porque podem ser usados ​​para testar duas condições, sendo que a segunda só é avaliada dependendo do resultado da primeira condição. Da [documentação] de Julia (http://docs.julialang.org/en/release-0.4/manual/control-flow/#man-short-circuit-evaluation):

Na expressão a && b, a subexpressão b só é avaliada se a for avaliado como true

Na expressão a || b, a subexpressão b só é avaliada se a for avaliada como false

Assim, enquanto a & b e a && b produzirão true se a e b forem true, seu comportamento se a for false será diferente.

Por exemplo, suponha que desejamos verificar se um objeto é um número positivo, onde é possível que ele nem seja um número. Considere as diferenças entre essas duas tentativas de implementação:

CheckPositive1(x) = (typeof(x)<:Number) & (x > 0) ? true : false
CheckPositive2(x) = (typeof(x)<:Number) && (x > 0) ? true : false

CheckPositive1("a")
CheckPositive2("a")

CheckPositive1() produzirá um erro se um tipo não numérico for fornecido a ele como argumento. Isso ocorre porque ele avalia ambas expressões, independentemente do resultado da primeira, e a segunda expressão produzirá um erro quando alguém tentar avaliá-la para um tipo não numérico.

CheckPositive2(), no entanto, produzirá false (em vez de um erro) se um tipo não numérico for fornecido a ele, já que a segunda expressão só é avaliada se a primeira for true.

Mais de um operador de curto-circuito pode ser amarrado. Por exemplo.:

1 > 0 && 2 > 0 && 3 > 5

A função ifelse #

shift(x) = ifelse(x > 10, x + 1, x - 1)

Uso:

julia> shift(10)
9

julia> shift(11)
12

julia> shift(-1)
-2

A função ifelse avaliará ambas as ramificações, mesmo a que não estiver selecionada. Isso pode ser útil quando as ramificações têm efeitos colaterais que devem ser avaliados, ou porque pode ser mais rápido se ambas as ramificações forem baratas.

expressão if…else #

A condicional mais comum em Julia é a expressão if…else. Por exemplo, abaixo implementamos o algoritmo euclidiano para calcular o máximo divisor comum, usando uma condicional para lidar com o caso base:

mygcd(a, b) = if a == 0
    abs(b)
else
    mygcd(b % a, a)
end

A forma if…else em Julia é na verdade uma expressão e tem um valor; o valor é a expressão na posição da cauda (ou seja, a última expressão) no ramo que é obtido. Considere a seguinte entrada de exemplo:

julia> mygcd(0, -10)
10

Aqui, a é 0 e b é -10. A condição a == 0 é true, então a primeira ramificação é feita. O valor retornado é abs(b) que é 10.

julia> mygcd(2, 3)
1

Aqui, a é 2 e b é 3. A condição a == 0 é falsa, então a segunda ramificação é feita, e computamos mygcd(b % a, a), que é mygcd(3 % 2, 2). O operador % retorna o resto quando 3 é dividido por 2, neste caso 1. Assim calculamos mygcd(1, 2), e desta vez a é 1 e b é 2. Mais uma vez, a == 0 é falso, então a segunda ramificação é feita, e calculamos mygcd(b % a, a), que é mygcd(0, 1). Desta vez, a == 0 finalmente e então abs(b) é retornado, o que dá o resultado 1.

if… else declaração #

name = readline()
if startswith(name, "A")
    println("Your name begins with A.")
else
    println("Your name does not begin with A.")
end

Qualquer expressão, como a expressão if…else, pode ser colocada na posição do comando. Isso ignora seu valor, mas ainda executa a expressão para seus efeitos colaterais.

declaração if #

Como qualquer outra expressão, o valor de retorno de uma expressão if…else pode ser ignorado (e, portanto, descartado). Isso geralmente é útil apenas quando o corpo da expressão tem efeitos colaterais, como gravar em um arquivo, alterar variáveis ​​ou imprimir na tela.

Além disso, o ramo else de uma expressão if…else é opcional. Por exemplo, podemos escrever o seguinte código para exibir na tela somente se uma condição específica for atendida:

second = Dates.second(now())
if iseven(second)
    println("The current second, $second, is even.")
end

No exemplo acima, usamos as funções time and date para obter o segundo atual; por exemplo, se for atualmente 10:55:27, a variável second conterá 27. Se este número for par, uma linha será impressa na tela. Caso contrário, nada será feito.

Operador condicional ternário #

pushunique!(A, x) = x in A ? A : push!(A, x)

O operador condicional ternário é uma expressão if…else menos prolixa.

A sintaxe especificamente é:

[condition] ? [execute if true] : [execute if false]

Neste exemplo, adicionamos x à coleção A somente se x ainda não estiver em A. Caso contrário, apenas deixamos A inalterado.

instrução if com várias ramificações #

d = Dates.dayofweek(now())
if d == 7
    println("It is Sunday!")
elseif d == 6
    println("It is Saturday!")
elseif d == 5
    println("Almost the weekend!")
else
    println("Not the weekend yet...")
end

Qualquer número de ramos elseif pode ser usado com uma instrução if, possivelmente com ou sem um ramo else final. As condições subsequentes só serão avaliadas se todas as condições anteriores forem consideradas ‘falsas’.
