Adicionando um endpoint de metadados ao seu serviço #

Os serviços SOAP podem publicar metadados que descrevem os métodos que podem ser invocados pelos clientes. Os clientes podem usar ferramentas como o Visual Studio para gerar código automaticamente (conhecido como client proxies). Os proxies escondem a complexidade de invocar um serviço. Para invocar um serviço, basta invocar um método em um proxy de cliente.

Primeiro você deve adicionar um endpoint de metadados ao seu serviço. Supondo que seu serviço seja igual ao definido no exemplo “Primeiro serviço e host”, você pode fazer as seguintes alterações no arquivo de configuração.

<system.serviceModel>
  <behaviors>
    <serviceBehaviors>
      <behavior name="serviceBehaviour">
        <serviceMetadata httpGetEnabled="true" />
      </behavior>
    </serviceBehaviors>
  </behaviors>
  <services>
    <service name="Service.Example" behaviorConfiguration="serviceBehaviour">
      <endpoint address="mex" binding="mexHttpBinding" name="mexExampleService" contract="IMetadataExchange" />
      <endpoint name="netTcpExample" contract="Service.IExample" binding="netTcpBinding" />
      <host>
        <baseAddresses>
          <add baseAddress="net.tcp://localhost:9000/Example" />
          <add baseAddress="http://localhost:8000/Example" />
        </baseAddresses>
      </host>
    </service>
  </services>      
</system.serviceModel>

O mexHttpbinding expõe a interface sobre http, então agora você pode ir com um navegador da web para

http://localhost:8000/Example http://localhost:8000/Example?wsdl

e exibirá o serviço e seus metadados.

1. Seu primeiro serviço e host #

Crie uma interface decorada com um atributo ServiceContract e funções de membro decoradas com o atributo OperationContract.

namespace Service
{
    [ServiceContract]
    interface IExample
    {
        [OperationContract]
        string Echo(string s);
    }
 }

Crie uma classe concreta implementando essa interface e você terá o serviço.

namespace Service
{
    public class Example : IExample
    {
        public string Echo(string s)
        {
            return s;
        }
    }
}

Em seguida, crie o host de onde o serviço será executado. Pode ser qualquer tipo de aplicação. Console, serviço, aplicativo GUI ou servidor web.

namespace Console
{
    using Service;
  
    class Console
    {
        Servicehost mHost;

        public Console()
        {
            mHost = new ServiceHost(typeof(Example));               
        } 

        public void Open()
        {
            mHost.Open();
        }
    
        public void Close()
        {
            mHost.Close();
        }

        public static void Main(string[] args)
        {
            Console host = new Console();
            
            host.Open();    

            Console.Readline();

            host.Close();
        }
    }
}

A classe ServiceHost lerá o arquivo de configuração para inicializar o serviço.

<system.serviceModel>
  <services>
    <service name="Service.Example">
      <endpoint name="netTcpExample" contract="Service.IExample" binding="netTcpBinding" />
      <host>
        <baseAddresses>
          <add baseAddress="net.tcp://localhost:9000/Example" />
        </baseAddresses>
      </host>
    </service>
  </services>      
</system.serviceModel>

Primeiro cliente de serviço #

Digamos que você tenha um serviço igual ao definido no exemplo “Primeiro serviço e host”.

Para criar um cliente, defina a seção de configuração do cliente na seção system.serviceModel do arquivo de configuração do cliente.

<system.serviceModel>
  <services>
    <client name="Service.Example">
      <endpoint name="netTcpExample" contract="Service.IExample" binding="netTcpBinding" address="net.tcp://localhost:9000/Example" />
    </client>
  </services>      
</system.serviceModel>

Em seguida, copie a definição do contrato de serviço do serviço:

namespace Service
{
    [ServiceContract]
    interface IExample
    {
        [OperationContract]
        string Echo(string s);
    }
}

(OBSERVAÇÃO: você também pode consumir isso adicionando uma referência binária ao assembly que contém o contrato de serviço.)

Em seguida, você pode criar o cliente real usando ChannelFactory<T> e chamar a operação no serviço:

namespace Console
{
    using Service;
  
    class Console
    {
        public static void Main(string[] args)
        {
            var client = new System.ServiceModel.ChannelFactory<IExample>("Service.Example").CreateChannel();
            var s = client.Echo("Hello World");
            Console.Write(s);
            Console.Read();
        }
    }
}

Crie um ServiceHost programaticamente #

Criando um ServiceHost programaticamente (sem arquivo de configuração) em sua forma mais básica:

namespace ConsoleHost
{
  class ConsoleHost
  {
    ServiceHost mHost;

    public Console()
    {
      mHost = new ServiceHost(typeof(Example), new Uri("net.tcp://localhost:9000/Example"));

      NetTcpBinding tcp = new NetTcpBinding();

      mHost.AddServiceEndpoint(typeof(IExample),tcp,"net.tcp://localhost:9000/Example");                      
    }

    public void Open()
    {
        mHost.Open();
    }
    
    public void Close()
    {
        mHost.Close();
    }

    public static void Main(string[] args)
    {
        ConsoleHost host = new ConsoleHost();
        
        host.Open();
        
        Console.ReadLine();
        
        host.Close();
    }
  }
}
  1. Crie uma instância ServiceHost passando o tipo de classe concreta e zero ou mais Uri de endereço base.
  2. Construa a ligação desejada, NetTcpBinding neste caso.
  3. chame AddServiceEndpoint passando o Address, Binding e Contract. (mnemônico ABC para endpoints WCF).
  4. Abra o host.
  5. Mantenha o host aberto até que o usuário pressione a tecla no console.
  6. Feche o host.

Adicionando programaticamente um endpoint de metadados a um serviço #

Quando você também deseja expor metadados sem um arquivo de configuração, pode criar no exemplo programaticamente criando um ServiceHost:

public ConsoleHost()
{
    mHost = new ServiceHost(typeof(Example), new Uri("http://localhost:8000/Example"), new Uri("net.tcp://9000/Example"));

    NetTcpBinding tcp = new NetTcpBinding();

    mHost.AddServiceEndpoint(typeof(IExample), tcp, "net.tcp://localhost:9000/Example");            

    ServiceMetadataBehavior metaBehavior = mHost.Description.Behaviors.Find<ServiceMetadataBehavior>();

    if (metaBehavior == null)
    {
        metaBehavior = new ServiceMetadataBehavior();
        metaBehavior.MetadataExporter.PolicyVersion = PolicyVersion.Policy15;
        metaBehavior.HttpGetEnabled = true;

        mHost.Description.Behaviors.Add(metaBehavior);
        mHost.AddServiceEndpoint(ServiceMetadataBehavior.MexContractName, MetadataExchangeBindings.CreateMexHttpBinding(), "mex");
    }

    mHost.Open();
}
  1. Crie uma instância ServiceHost passando o tipo de classe concreta e zero ou mais Uri de endereço base.
  2. Quando você usa mexHttpBinding, você precisa adicionar http://localhost:8000/Example baseaddress
  3. Construa a ligação desejada, NetTcpBinding neste caso.
  4. chame AddServiceEndpoint passando o Address, Binding e Contract. (ABC).
  5. Construa um ServiceMetadataBehavior
  6. Defina HttpGetEnabled como verdadeiro
  7. Adicione o comportamento de metadados à coleção de comportamentos.
  8. chame AddServiceEndpoint passando as constantes para troca de metadados
  9. Abra o host.
