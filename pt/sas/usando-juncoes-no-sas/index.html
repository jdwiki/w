Usando junções no SAS

Cada banco de dados é uma coleção de tabelas diferentes e cada tabela contém dados diferentes de forma organizada. Ao trabalhar com dados, na maioria das vezes as informações de que precisamos estão espalhadas em mais de uma tabela. Precisamos de joins/merge para obter a saída desejada.

No SAS, usamos joins enquanto trabalhamos com Proc SQL e usamos merge enquanto trabalhamos com Data step. Agora falaremos apenas sobre joins dentro de Proc SQL.

Parâmetros #

Tipo de junção Saída
Proc Sql Procedimento SQL dentro do SAS
Criar Tabela Cria um conjunto de dados SAS
Selecione Seleciona as variáveis ​​necessárias dos respectivos conjuntos de dados
Onde Especifica uma condição específica
Sair Finalizar o procedimento

Como mencionado na introdução, também podemos usar Mergedentro de um data step que será discutido em um tópico separado. As junções desempenham um papel muito importante para combinar e unificar dados de acordo com o requisito.

União Vertical #

A junção vertical anexa o conjunto de dados B ao conjunto de dados A, desde que ambos tenham variáveis ​​semelhantes. Por exemplo, temos vendas para o mês de janeiro de 17 no conjunto de dados A e vendas para fevereiro de 17 no conjunto de dados B. Para criar um conjunto de dados C que tenha vendas de janeiro e fevereiro, usamos a junção vertical.

PROC SQL;
CREATE TABLE C AS
SELECT *
FROM A
UNION 
SELECT *
FROM B;
QUIT;

Agora, o conjunto de dados C tem observações de A e B e é anexado verticalmente.

Junção interna #

A junção interna cria um conjunto de dados que contém registros com valores correspondentes de ambas as tabelas. Por exemplo, temos um conjunto de dados A que contém informações do cliente e um conjunto de dados B que contém detalhes do cartão de crédito. Para obter os detalhes do cartão de crédito dos clientes no conjunto de dados A, vamos criar o conjunto de dados C

PROC SQL;
CREATE TABLE C AS
SELECT A.*, B.CC_NUM
FROM CUSTOMER A, CC_DETAILS B
WHERE A.CUSTOMERID=B.CUSTOMERID
QUIT;

O conjunto de dados C terá apenas observações correspondentes de ambos os conjuntos de dados.

Associação à esquerda #

A junção esquerda retorna todas as observações no conjunto de dados esquerdo, independentemente de seus valores de chave, mas apenas as observações com valores de chave correspondentes do conjunto de dados direito. Considerando o mesmo exemplo acima,

PROC SQL;
CREATE TABLE C AS
SELECT A.*, B.CC_NUMBER, B.START_DATE
FROM CUSTOMER A LEFT JOIN CC_DETAILS B
ON A.CUSTOMERID=B.CUSTOMERID
QUIT;

O conjunto de dados C contém todos os valores da tabela à esquerda, além dos valores correspondentes da tabela da direita ou valores ausentes no caso de não haver correspondência.

Junção direita #

Assim como a junção à esquerda, a junção à direita seleciona todas as observações do conjunto de dados à direita e os registros correspondentes da tabela à esquerda.

PROC SQL;
CREATE TABLE C AS
SELECT A.*, B.CC_NUMBER, B.START_DATE
FROM CUSTOMER A RIGHT JOIN CC_DETAILS B
ON A.CUSTOMERID=B.CUSTOMERID
QUIT;

O conjunto de dados C contém todos os valores da tabela à direita, além dos valores correspondentes da tabela da esquerda ou valores ausentes no caso de não haver correspondência.

Junção completa #

A junção completa seleciona todas as observações de ambos os conjuntos de dados, mas há valores ausentes em que o valor da chave em cada observação é encontrado em apenas uma tabela.

PROC SQL;
CREATE TABLE C AS
SELECT A.*, B.CC_NUMBER, B.START_DATE
FROM CUSTOMER A FULL JOIN CC_DETAILS B 
ON A.CUSTOMERID=B.CUSTOMERID
QUIT;

O conjunto de dados C conterá todos os registros de ambas as tabelas e preencherá . para correspondências ausentes em ambos os lados.
