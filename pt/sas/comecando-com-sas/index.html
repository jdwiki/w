Instalação ou Configuração #

O SAS pode ser executado no modelo cliente-servidor, usando o cliente thin do Enterprise Guide ou o cliente thin do SAS Studio (habilitado para web), ou no modo “servidor local”, onde um sistema SAS totalmente funcional está presente em uma máquina local (Windows ou desktop ou servidor Unix/Linux executando no modo interativo) e executado no modo Display Manager (o cliente local) ou por meio de um dos clientes cliente-servidor acima (conectado ao servidor instalado localmente).

A instalação do SAS normalmente é realizada por um administrador do SAS, que instalará o software a partir de um depósito de software personalizado para o site (e geralmente fornecido diretamente pelo SAS Institute).

Para o propósito de aprender SAS, há também o SAS University Edition gratuito, que pode ser instalado gratuitamente para fins educacionais por qualquer pessoa em um computador Windows, Mac ou Unix/Linux. Ele está disponível diretamente do SAS, atualmente na página SAS University Edition, executando uma instância da AWS (no nível gratuito) ou baixando uma máquina virtual localmente. Consulte o guia de instalação em SAS.com para obter instruções atualizadas ou abaixo para o atual (julho de 2016) instruções.

Para instalá-lo localmente, primeiro baixe e instale o Oracle Virtualbox 5.0 (Windows/Mac/Linux). Em seguida, baixe a mais nova [imagem de disco do SAS University Edition] (http://www.sas.com/store/expresscheckout.ep?item=DPUNVE001_VirtualBox), que tem cerca de 2 GB e requer a configuração de um perfil SAS.com.

Depois de fazer isso, você precisa configurar a máquina virtual no VirtualBox. Importe a VM SAS como um dispositivo (“Import Appliance” no VirtualBox). Crie uma pasta para o SAS usar como armazenamento local (para que você possa colocar os arquivos em um local que o SAS possa vê-los) e configure-a como uma pasta compartilhada na caixa de diálogo de configurações do computador. Configure-o para montagem automática.

Em seguida, inicie a máquina virtual SAS e, uma vez iniciada, você poderá se conectar por meio do navegador da Web, conectando-se a http://localhost:10080/ se tiver usado as configurações padrão.

Se você tiver problemas, os SAS Community Forums - Analytics U são os fóruns do fornecedor para obter suporte ou peça um pergunta em Stack Overflow.

Visão geral do SAS básico #

O SAS é um sistema integrado de soluções de software que permite realizar as seguintes tarefas:

  • entrada, recuperação e gerenciamento de dados
  • Elaboração de relatórios e design gráfico
  • análise estatística e matemática
  • Previsão de negócios e suporte à decisão
  • Pesquisa operacional e gerenciamento de projetos
  • desenvolvimento de aplicações

Como você usa o SAS depende do que você deseja realizar. Algumas pessoas usam muitos dos recursos do Sistema SAS e outros usam apenas alguns.

Olá Mundo! #

Devido à estrutura do SAS, existem três maneiras principais de criar “Hello World!” exemplos:

  1. Dentro de uma etapa de dados para colocar uma mensagem no log do SAS (_null_ indica que nenhum conjunto de dados de saída deve ser criado):

    data null; put “Hell” “o World!”; run;

  2. Dentro de uma etapa de dados para armazenar “Hello World!” dentro de uma variável (foo denota que um conjunto de dados de saída chamado foo deve ser criado que a) contém apenas um registro e b) contém apenas uma variável: bar, que tem um valor de Hello World!):

    data foo ; bar=“Hello” ; put bar= “World!”; run ;

  3. Através da linguagem SAS Macro (em ‘código aberto’ fora de quaisquer etapas de dados). & identifica uma chamada para uma variável macro e . identifica o final da variável (se um caractere de espaço em branco não for desejado):

    %let foo=Hello; %put &foo.o World!;

  4. Híbrido: usando uma variável de macro em uma etapa de dados:

    %let foo=Hello;

    data null ; put “&foo World!”; run ;

Arquitetura de servidor SAS #

Visão geral: normalmente há dois tipos de implantações de SAS:

  1. Instalação somente do SAS Foundation (BASE SAS). Isso normalmente é instalado em um PC. Ele não executa nenhum software de servidor.

  2. Implantação planejada do SAS para sua arquitetura de servidor, que instalará o ambiente do servidor SAS juntamente com possivelmente qualquer software cliente SAS.

Qual destes você possui será indicado em seu e-mail de pedido de software SAS indicando planejamento ou não planejamento. Se você estiver fazendo uma instalação planejada, precisará de um arquivo de plano para seu pedido que primeiro sua topologia.

Nota de instalação 44320: usando planos de implantação durante uma instalação do SAS®

Arquitetura de servidor SAS

O ambiente do servidor SAS é dividido em 3 camadas diferentes:

  1. Servidor(es) de Metadados SAS - O servidor de Metadados SAS é responsável por gerenciar o ambiente do servidor SAS, incluindo bibliotecas, usuários e configuração do servidor.

2. Servidor(es) de aplicativos SAS - O servidor de aplicativos SAS é principalmente um servidor de computação de onde seus clientes normalmente iniciariam trabalhos.

3. SAS Middle Tier(s) = O SAS Middle Tier é principalmente sua camada da Web que executa seus aplicativos da Web.

4. Camada de cliente - A camada de cliente são os aplicativos clientes dos usuários que eles usam para se conectar ao ambiente, como o SAS Enterprise Guide.

Documento 363-2011| Entendendo a anatomia de uma implantação SAS®: o que há na minha sopa de servidor? Mark Schneider, Donna Bennett e Connie Robison, SAS Institute Inc., Cary, NC

Topologia:

A camada de metadados SAS, a camada SAS Application Server e a camada intermediária SAS podem ser instaladas em um único servidor de máquina ou distribuídas em vários servidores. Isso é determinado pelo arquivo de plano que você possui, ele deve atender à topologia desejada para sua implementação.

Normalmente, a maioria, se não toda a camada de cliente, são aplicativos baseados em Windows, portanto, a camada de cliente estaria nas estações de trabalho dos usuários do SAS. Opcionalmente, eles provavelmente também podem ser instalados no(s) servidor(es) se forem baseados em Windows.
