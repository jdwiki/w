Markdown com sabor do GitHub #

O GitHub expande a sintaxe Markdown para fornecer novos recursos úteis.

Cabeçalho #

# Header1
## Header2
### Header3
#### Header4
##### Header5
###### Header6
H1
===
H2
---

Ênfase #

*Italic1* _Italic2_
**Bold1** __Bold2__
***Bold_Italic***
~~Strikethrough~~

Linha horizontal #

---
***
___

Lista #

unordered list:

* item-1
  * sub-item-1
  * sub-item-2
- item-2
  - sub-item-3
  - sub-item-4
+ item-3
  + sub-item-5
  + sub-item-6


ordered list:

1. item-1
  1. sub-item-1
  2. sub-item-2
2. item-2
  1. sub-item-3
  2. sub-item-4
3. item-3

Mesa #

Table Header-1 | Table Header-2 | Table Header-3
:--- | :---: | ---:
Table Data-1 | Table Data-2 | Table Data-3
TD-4 | Td-5 | TD-6
Table Data-7 | Table Data-8 | Table Data-9

referências SHA #

Pull Request e Referências de Emissão #

Qualquer referência a um pull request ou a um problema será automaticamente vinculada a esse pull request ou problema.

Isso pode ser feito colocando um # na frente do número de emissão/Pull Request.

Instalação ou Configuração #

O GitHub é uma enorme coleção de repositórios Git. Em outras palavras, você pode pensar no GitHub como uma coleção de muitos projetos!

Criando uma conta #

  • Visite a página principal do GitHub Aqui
  • Escolha um nome de usuário, digite seu endereço de e-mail e escolha uma senha segura e você está pronto para começar!

Ferramentas úteis #

Para iniciantes do Git/GitHub, entender como o controle de versão funciona pode ser confuso no início. Existe uma versão GUI do GitHub que você pode baixar e usar. GitHub Desktop é exatamente essa ferramenta.

Criando seu primeiro repositório #

Você pode pensar em um repositório como um projeto. Você pode criar um repositório online ou offline. Siga os passos abaixo:

Conectados #

  1. Primeiro faça o login e vá para o seu perfil.
  2. Navegue até a guia “Repositórios” na parte superior da página
  3. Aperte o botão verde “Novo” e você está pronto para rumble!

Desligada #

  1. Baixe e instale git (escolha o sistema operacional que você está executando)
  2. Após o download e a instalação, você pode usar a ferramenta de linha de comando ou fazer download de um cliente GUI.
  3. Após a instalação, crie uma conta no github
  4. No canto superior direito, clique no + e escolha criar um novo repositório ou importar um existente.
  5. Se você escolher um novo, digite o nome do repositório e escolha entre público ou privado.
  6. Clique em: Criar Repositório

N.B. Repositórios privados não estão disponíveis para usuários gratuitos.

arquivo LEIA-ME #

Se seu projeto não tiver README.md, o GitHub pode analisar README.rdoc para exibir detalhes. Se tiver ambos, usará o README.md, ignorando silenciosamente o rdoc.

Um arquivo README pode incluir- #


Título do projeto #

Descreva brevemente sobre seu projeto. Você também pode fornecer o link do site do projeto, crachás, comunidade e informações de contato (ou seja, e-mail, site social).

Download #

Link do arquivo executável (executável ou minificado ou arquivo de instalação). Pode haver links para versões anteriores também.

Instalação #

Como seu trabalho pode ser usado. Pode incluir os pré-requisitos, configurações, bibliotecas de terceiros, uso, cuidados, etc.

Demonstração #

Pode incluir amostra de código, arquivo gif, link de vídeo ou até capturas de tela.

Autores #

Nomes dos autores, informações de contato, etc.

Agradecimentos #

Lista de pessoas ou comunidade ajudadas e inspiradas ao longo do projeto

Contribuindo #

Instruções para contribuir (ou seja, adicionar recurso, relatar bug, enviar patch) para o projeto. Pode incluir link de documentação também.

Licença #

Dê uma breve introdução sobre sua licença. Você também pode fornecer um link para o site de licença.

arquivo de LICENÇA #

O GitHub ajuda você a adicionar rapidamente uma licença ao seu repositório, como uma alternativa para adicionar seu próprio arquivo de texto/markdown.

  1. Em seu repositório, clique em ‘Criar novo arquivo’

  1. Na próxima página:

    1. Type LICENSE.md or LICENSE.txt as the new file’s file name.
    2. The Want to use a new template? dialog will appear.

  2. Escolha sua licença preferida.
