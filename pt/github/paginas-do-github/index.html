Usando o Git para criar páginas do zero #

  1. Crie um novo repositório ou clone um existente.

  2. Crie um novo branch chamado gh-pages sem histórico

    $ git checkout –orphan gh-pages

    ensure you are in the correct directory then, #

    remove all files from the old working tree #

    $ git rm -rf

  3. Adicione um arquivo index.html à raiz do repositório.

    $ echo “Hello World” > index.html $ git add index.html $ git commit -a -m “First pages commit”

  4. Empurre para o Github.

    $ git push origin gh-pages

Agora você pode carregar seu novo site do Github Pages em http(s)://<username>.github.io/<projectname>

Usando o gerador automático de páginas para um repositório #

  1. Acesse o site do GitHub
  2. Abra seu repositório
  3. Clique em Configurações
  4. Em GitHub Pages, clique em “Iniciar o gerador automático de páginas”
  5. Siga as instruções

Criando um URL personalizado para sua página do GitHub #

Você precisará de um nome de domínio de um registrador.

No branch gh-pages do seu repositório de projeto, ou no branch principal do seu repositório username.github.io, crie um arquivo CNAME com o conteúdo www.yourdomain.com - o [canonical domain] (https://www.mattcutts.com/blog/seo-advice-url-canonicalization/).

Na página de configuração de domínio do seu registrador, aponte seu domínio para o site do GitHub. Configure dois registros CNAME (um para o apex raiz (@) e outro para www). Ambos devem apontar para username.github.io ou username.github.io/repository. Se o seu provedor de DNS NÃO oferecer suporte a registros ALIAS no apex raiz (@), basta criar registros A que apontem para 192.30.252.153 e 192.30.252.154.
