Python se oferece não apenas como uma linguagem de script popular, mas também suporta o paradigma de programação orientada a objetos. As classes descrevem dados e fornecem métodos para manipular esses dados, todos englobados em um único objeto. Além disso, as classes permitem a abstração separando detalhes de implementação concretos de representações abstratas de dados.

O código que utiliza classes geralmente é mais fácil de ler, entender e manter.

Introdução às aulas #

Uma classe funciona como um modelo que define as características básicas de um determinado objeto. Aqui está um exemplo:

class Person(object):
     """A simple class."""                            # docstring
     species = "Homo Sapiens"                         # class attribute

     def __init__(self, name):                        # special method
         """This is the initializer. It's a special
         method (see below).
         """
         self.name = name                             # instance attribute

     def __str__(self):                               # special method
         """This method is run when Python tries 
         to cast the object to a string. Return 
         this string when using print(), etc.
         """
         return self.name

     def rename(self, renamed):                       # regular method
         """Reassign and print the name attribute."""
         self.name = renamed
         print("Now my name is {}".format(self.name))

Há algumas coisas a serem observadas ao analisar o exemplo acima.

  1. A classe é composta por atributos (dados) e métodos (funções).
  2. Atributos e métodos são simplesmente definidos como variáveis ​​e funções normais.
  3. Conforme observado na docstring correspondente, o método __init__() é chamado de inicializador. É equivalente ao construtor em outras linguagens orientadas a objetos e é o método que é executado pela primeira vez quando você cria um novo objeto ou uma nova instância da classe.
  4. Os atributos que se aplicam a toda a classe são definidos primeiro e são chamados de atributos de classe.
  5. Os atributos que se aplicam a uma instância específica de uma classe (um objeto) são chamados de atributos de instância. Eles são geralmente definidos dentro de __init__(); isso não é necessário, mas é recomendado (já que atributos definidos fora de __init__() correm o risco de serem acessados ​​antes de serem definidos).
  6. Todo método incluído na definição da classe passa o objeto em questão como seu primeiro parâmetro. A palavra self é usada para este parâmetro (o uso de self é na verdade por convenção, já que a palavra self não tem significado inerente em Python, mas esta é uma das convenções mais respeitadas do Python, e você deve sempre segui-la ).
  7. Aqueles acostumados a programação orientada a objetos em outras linguagens podem se surpreender com algumas coisas. Uma é que o Python não tem um conceito real de elementos private, então tudo, por padrão, imita o comportamento da palavra-chave public do C++/Java. Para obter mais informações, consulte o exemplo “Membros de classe privada” nesta página.
  8. Alguns dos métodos da classe têm a seguinte forma: __functionname__(self, other_stuff). Todos esses métodos são chamados de “métodos mágicos” e são uma parte importante das classes em Python. Por exemplo, a sobrecarga de operadores em Python é implementada com métodos mágicos. Para obter mais informações, consulte a documentação relevante.

Métodos vinculados, não vinculados e estáticos #

A ideia de métodos vinculados e não vinculados foi removida no Python 3. No Python 3, quando você declara um método dentro de uma classe, você está usando uma palavra-chave def, criando assim um objeto de função. Esta é uma função regular e a classe circundante funciona como seu namespace. No exemplo a seguir declaramos o método f dentro da classe A, e ele se torna uma função A.f:

class A(object):
    def f(self, x):
        return 2 * x
A.f
# <function A.f at ...>  (in Python 3.x)

No Python 2 o comportamento era diferente: objetos de função dentro da classe foram substituídos implicitamente por objetos do tipo instancemethod, que eram chamados de métodos não vinculados porque não estavam vinculados a nenhuma instância de classe específica. Foi possível acessar a função subjacente usando a propriedade .__func__.

A.f
# <unbound method A.f>   (in Python 2.x)
A.f.__class__
# <type 'instancemethod'>
A.f.__func__
# <function f at ...>

Os últimos comportamentos são confirmados por inspeção - os métodos são reconhecidos como funções no Python 3, enquanto a distinção é mantida no Python 2.

import inspect

inspect.isfunction(A.f)
# True
inspect.ismethod(A.f)
# False

import inspect

inspect.isfunction(A.f)
# False
inspect.ismethod(A.f)
# True

Em ambas as versões do Python, a função/método A.f pode ser chamado diretamente, desde que você passe uma instância da classe A como o primeiro argumento.

A.f(1, 7)
# Python 2: TypeError: unbound method f() must be called with
#                      A instance as first argument (got int instance instead) 
# Python 3: 14   
a = A()
A.f(a, 20)
# Python 2 & 3: 40

Agora suponha que a seja uma instância da classe A, o que é a.f então? Bem, intuitivamente este deve ser o mesmo método f da classe A, só que deve de alguma forma “saber” que foi aplicado ao objeto a – em Python isso é chamado de método limitado a a.

Os detalhes minuciosos são os seguintes: escrever a.f invoca o método mágico __getattribute__ de a, que primeiro verifica se a tem um atributo chamado f (não tem), então verifica a classe A se ele contém um método com tal nome (ele contém), e cria um novo objeto m do tipo method que tem a referência ao A.f original em m.__func__, e uma referência ao objeto a em m.__self__. Quando este objeto é chamado como uma função, ele simplesmente faz o seguinte: m(...) => m.__func__(m.__self__, ...). Assim, esse objeto é chamado de método vinculado porque, quando invocado, sabe fornecer o objeto ao qual foi vinculado como o primeiro argumento. (Essas coisas funcionam da mesma maneira no Python 2 e 3).

a = A()
a.f
# <bound method A.f of <__main__.A object at ...>>
a.f(2)
# 4

# Note: the bound method object a.f is recreated *every time* you call it:
a.f is a.f  # False
# As a performance optimization you can store the bound method in the object's
# __dict__, in which case the method object will remain fixed:
a.f = a.f
a.f is a.f  # True

Finalmente, Python tem métodos de classe e métodos estáticos – tipos especiais de métodos. Os métodos de classe funcionam da mesma maneira que os métodos regulares, exceto que, quando invocados em um objeto, eles se vinculam à class do objeto em vez de ao objeto. Assim m.__self__ = type(a). Quando você chama esse método vinculado, ele passa a classe de a como o primeiro argumento. Os métodos estáticos são ainda mais simples: eles não vinculam nada e simplesmente retornam a função subjacente sem nenhuma transformação.

class D(object):
    multiplier = 2

    @classmethod
    def f(cls, x):
        return cls.multiplier * x

    @staticmethod
    def g(name):
        print("Hello, %s" % name)

D.f
# <bound method type.f of <class '__main__.D'>>
D.f(12)
# 24
D.g
# <function D.g at ...>
D.g("world")
# Hello, world

Observe que os métodos de classe são vinculados à classe mesmo quando acessados ​​na instância:

d = D()
d.multiplier = 1337
(D.multiplier, d.multiplier)
# (2, 1337)
d.f
# <bound method D.f of <class '__main__.D'>>
d.f(10)
# 20

Vale a pena notar que, no nível mais baixo, funções, métodos, métodos estáticos etc. são na verdade [descritores](https://www.wikiod.com/pt/python/descritor#Descritor simples) que invocam métodos especiais __get__, __set__ e opcionalmente __del__. Herança básica #

A herança em Python é baseada em ideias semelhantes usadas em outras linguagens orientadas a objetos como Java, C++ etc. Uma nova classe pode ser derivada de uma classe existente da seguinte maneira.

class BaseClass(object):
    pass

class DerivedClass(BaseClass):
    pass

A BaseClass é a classe já existente (pai), e a DerivedClass é a nova classe (filho) que herda (ou subclasses) atributos de BaseClass. Observação: a partir do Python 2.2, todas as classes herdam implicitamente da classe object, que é a classe base para todos os tipos internos.

Definimos uma classe pai Rectangle no exemplo abaixo, que herda implicitamente de object:

class Rectangle():
    def __init__(self, w, h):
        self.w = w
        self.h = h
    
    def area(self):
        return self.w * self.h
        
    def perimeter(self):
        return 2 * (self.w + self.h)

A classe Rectangle pode ser usada como uma classe base para definir uma classe Square, pois um quadrado é um caso especial de retângulo.

class Square(Rectangle):
    def __init__(self, s):
        # call parent constructor, w and h are both s
        super(Square, self).__init__(s, s)
        self.s = s

A classe Square herdará automaticamente todos os atributos da classe Rectangle, bem como a classe do objeto. super() é usado para chamar o método __init__() da classe Rectangle, essencialmente chamando qualquer método sobrescrito da classe base. Nota: em Python 3, super() não requer argumentos.

Objetos de classe derivados podem acessar e modificar os atributos de suas classes base:

r.area()
# Output: 12
r.perimeter()  
# Output: 14

s.area()
# Output: 4
s.perimeter()
# Output: 8

Funções internas que funcionam com herança #

issubclass(DerivedClass, BaseClass): retorna True se DerivedClass for uma subclasse da BaseClass

isinstance(s, Class): retorna True se s for uma instância de Class ou qualquer uma das classes derivadas de Class

# subclass check        
issubclass(Square, Rectangle)
# Output: True

# instantiate
r = Rectangle(3, 4)
s = Square(2)

isinstance(r, Rectangle)  
# Output: True
isinstance(r, Square)
# Output: False
# A rectangle is not a square

isinstance(s, Rectangle)
# Output: True
# A square is a rectangle
isinstance(s, Square)
# Output: True

Remendo de macaco #

Neste caso, “macaco patching” significa adicionar uma nova variável ou método a uma classe depois que ela foi definida. Por exemplo, digamos que definimos a classe A como

class A(object):
    def __init__(self, num):
        self.num = num

    def __add__(self, other):
        return A(self.num + other.num)

Mas agora queremos adicionar outra função posteriormente no código. Suponha que esta função seja a seguinte.

def get_num(self):
    return self.num

Mas como adicionamos isso como um método em A? Isso é simples, nós essencialmente colocamos essa função em ‘A’ com uma instrução de atribuição.

A.get_num = get_num

Por que isso funciona? Porque funções são objetos como qualquer outro objeto, e métodos são funções que pertencem à classe.

A função get_num deve estar disponível para todos os existentes (já criados) bem como para as novas instâncias de A

Essas adições estão disponíveis em todas as instâncias dessa classe (ou suas subclasses) automaticamente. Por exemplo:

foo = A(42)

A.get_num = get_num

bar = A(6);

foo.get_num() # 42

bar.get_num() # 6

Observe que, ao contrário de algumas outras linguagens, essa técnica não funciona para certos tipos internos e não é considerada um bom estilo.

Aulas de estilo novo vs. de estilo antigo #

As classes New-style foram introduzidas no Python 2.2 para unificar classes e types. Eles herdam do tipo object de nível superior. Uma classe de novo estilo é um tipo definido pelo usuário e é muito semelhante aos tipos internos.

# new-style class
class New(object):
    pass

# new-style instance
new = New()

new.__class__
# <class '__main__.New'>
type(new)
# <class '__main__.New'>
issubclass(New, object)
# True

Classes antigas não herdam de object. As instâncias de estilo antigo são sempre implementadas com um tipo instance integrado.

# old-style class
class Old:
    pass

# old-style instance
old = Old()

old.__class__
# <class __main__.Old at ...>
type(old)
# <type 'instance'>
issubclass(Old, object)
# False

No Python 3, as classes de estilo antigo foram removidas.

As classes de novo estilo em Python 3 herdam implicitamente de object, então não há mais necessidade de especificar MyClass(object).

class MyClass:
    pass

my_inst = MyClass()

type(my_inst)
# <class '__main__.MyClass'>
my_inst.__class__
# <class '__main__.MyClass'>
issubclass(MyClass, object)
# True

Métodos de classe: inicializadores alternativos #

Os métodos de classe apresentam maneiras alternativas de construir instâncias de classes. Para ilustrar, vejamos um exemplo.

Vamos supor que temos uma classe Person relativamente simples:

class Person(object):

    def __init__(self, first_name, last_name, age):
        self.first_name = first_name
        self.last_name = last_name
        self.age = age
        self.full_name = first_name + " " + last_name
    
    def greet(self):
        print("Hello, my name is " + self.full_name + ".")

Pode ser útil ter uma maneira de construir instâncias dessa classe especificando um nome completo em vez de nome e sobrenome separadamente. Uma maneira de fazer isso seria ter last_name um parâmetro opcional e assumindo que, se não for fornecido, passamos o nome completo em:

class Person(object):

    def __init__(self, first_name, age, last_name=None):
        if last_name is None:
            self.first_name, self.last_name = first_name.split(" ", 2)
        else:
            self.first_name = first_name
            self.last_name = last_name
        
        self.full_name = self.first_name + " " + self.last_name
        self.age = age

    def greet(self):
        print("Hello, my name is " + self.full_name + ".")

No entanto, existem dois problemas principais com este pedaço de código:

  1. Os parâmetros first_name e last_name agora são enganosos, pois você pode inserir um nome completo para first_name. Além disso, se houver mais casos e/ou mais parâmetros que tenham esse tipo de flexibilidade, a ramificação if/elif/else pode se tornar irritante rapidamente.

  2. Não é tão importante, mas ainda vale a pena ressaltar: e se last_name for None, mas first_name não se dividir em duas ou mais coisas por meio de espaços? Temos ainda outra camada de validação de entrada e/ou tratamento de exceção…

Insira métodos de classe. Em vez de ter um único inicializador, vamos criar um inicializador separado, chamado from_full_name, e decorá-lo com o decorador classmethod (integrado).

class Person(object):

    def __init__(self, first_name, last_name, age):
        self.first_name = first_name
        self.last_name = last_name
        self.age = age
        self.full_name = first_name + " " + last_name
    
    @classmethod
    def from_full_name(cls, name, age):
        if " " not in name:
            raise ValueError
        first_name, last_name = name.split(" ", 2)
        return cls(first_name, last_name, age)
    
    def greet(self):
        print("Hello, my name is " + self.full_name + ".")

Observe cls em vez de self como o primeiro argumento para from_full_name. Os métodos de classe são aplicados à classe geral, não a uma instância de uma determinada classe (que é o que self geralmente denota). Então, se cls é nossa classe Person, então o valor retornado do método de classe from_full_name é Person(first_name, last_name, age), que usa __init__ de Person para criar uma instância da classe Pessoa. Em particular, se fôssemos fazer uma subclasse Employee de Person, então from_full_name funcionaria na classe Employee também.

Para mostrar que isso funciona como esperado, vamos criar instâncias de Person de mais de uma maneira sem a ramificação em __init__:

In [2]: bob = Person("Bob", "Bobberson", 42)

In [3]: alice = Person.from_full_name("Alice Henderson", 31)

In [4]: bob.greet()
Hello, my name is Bob Bobberson.

In [5]: alice.greet()
Hello, my name is Alice Henderson.

Herança múltipla #

Python usa o algoritmo C3 linearization para determinar a ordem na qual resolver atributos de classe, incluindo métodos. Isso é conhecido como Ordem de Resolução de Método (MRO).

Aqui está um exemplo simples:

class Foo(object):
    foo = 'attr foo of Foo'
    

class Bar(object):
    foo = 'attr foo of Bar' # we won't see this.
    bar = 'attr bar of Bar'

class FooBar(Foo, Bar):
    foobar = 'attr foobar of FooBar'

Agora, se instanciarmos FooBar, se procurarmos o atributo foo, veremos que o atributo de Foo é encontrado primeiro

fb = FooBar()

e

>>> fb.foo
'attr foo of Foo'

Aqui está o MRO do FooBar:

>>> FooBar.mro()
[<class '__main__.FooBar'>, <class '__main__.Foo'>, <class '__main__.Bar'>, <type 'object'>]

Pode-se simplesmente afirmar que o algoritmo MRO do Python é

  1. Profundidade primeiro (por exemplo, FooBar e depois Foo), a menos que
  2. um pai compartilhado (object) é bloqueado por um filho (Bar) e
  3. não são permitidas relações circulares.

Ou seja, por exemplo, Bar não pode herdar de FooBar enquanto FooBar herda de Bar.

Para obter um exemplo abrangente em Python, consulte a entrada da wikipedia.

Outro recurso poderoso na herança é o super. super pode buscar recursos de classes pai.

class Foo(object):
    def foo_method(self):
        print "foo Method"

class Bar(object):
    def bar_method(self):
        print "bar Method"

class FooBar(Foo, Bar):
    def foo_method(self):
        super(FooBar, self).foo_method()

Herança múltipla com o método init da classe, quando cada classe tem seu próprio método init, então tentamos ineritância múltipla, então apenas o método init é chamado da classe que é herdada primeiro.

para o exemplo abaixo apenas o método init da classe Foo sendo chamado Bar class init não sendo chamado

    class Foo(object):
        def __init__(self):
            print "foo init"

    class Bar(object):
        def __init__(self):
            print "bar init"

    class FooBar(Foo, Bar):
        def __init__(self):
            print "foobar init"
            super(FooBar, self).__init__()

    a = FooBar()

Resultado:

    foobar init
    foo init

Mas isso não significa que a classe Bar não seja herdada. A instância da classe final FooBar também é uma instância da classe Bar e da classe Foo.

print isinstance(a,FooBar)
print isinstance(a,Foo)
print isinstance(a,Bar)

Resultado:

True
True
True

Propriedades #

As classes Python suportam propriedades, que se parecem com variáveis ​​de objetos comuns, mas com a possibilidade de anexar comportamento e documentação personalizados.

class MyClass(object):

    def __init__(self):
       self._my_string = ""
    
    @property
    def string(self):
        """A profoundly important string."""
        return self._my_string

    @string.setter
    def string(self, new_value):
        assert isinstance(new_value, str), \
               "Give me a string, not a %r!" % type(new_value)
        self._my_string = new_value

    @string.deleter
    def x(self):
        self._my_string = None

Os objetos da classe MyClass parecerão ter uma propriedade .string, porém seu comportamento agora é rigidamente controlado:

mc = MyClass()
mc.string = "String!"
print(mc.string)
del mc.string

Assim como a sintaxe útil acima, a sintaxe de propriedade permite a validação ou outras melhorias a serem adicionadas a esses atributos. Isso pode ser especialmente útil com APIs públicas - onde um nível de ajuda deve ser dado ao usuário.

Outro uso comum de propriedades é permitir que a classe apresente ‘atributos virtuais’ - atributos que não são realmente armazenados, mas são computados apenas quando solicitados.

class Character(object):
    def __init__(name, max_hp):
        self._name = name
        self._hp = max_hp
        self._max_hp = max_hp

    # Make hp read only by not providing a set method
    @property
    def hp(self):
        return self._hp

    # Make name read only by not providing a set method
    @property
    def name(self):
        return self.name

    def take_damage(self, damage):
        self.hp -= damage
        self.hp = 0 if self.hp <0 else self.hp

    @property
    def is_alive(self):
        return self.hp != 0

    @property
    def is_wounded(self):
        return self.hp < self.max_hp if self.hp > 0 else False

    @property
    def is_dead(self):
        return not self.is_alive

bilbo = Character('Bilbo Baggins', 100)
bilbo.hp
# out : 100
bilbo.hp = 200        
# out : AttributeError: can't set attribute
# hp attribute is read only.

bilbo.is_alive
# out : True
bilbo.is_wounded
# out : False
bilbo.is_dead
# out : False

bilbo.take_damage( 50 )

bilbo.hp
# out : 50

bilbo.is_alive
# out : True
bilbo.is_wounded
# out : True
bilbo.is_dead
# out : False

bilbo.take_damage( 50 )
bilbo.hp
# out : 0

bilbo.is_alive
# out : False
bilbo.is_wounded
# out : False
bilbo.is_dead
# out : True

Valores padrão para variáveis ​​de instância #

Se a variável contiver um valor de um tipo imutável (por exemplo, uma string), não há problema em atribuir um valor padrão como este

class Rectangle(object):
    def __init__(self, width, height, color='blue'):
        self.width = width
        self.height = height
        self.color = color
    
    def area(self):
        return self.width  * self.height 

# Create some instances of the class
default_rectangle = Rectangle(2, 3)
print(default_rectangle.color) # blue

red_rectangle = Rectangle(2, 3, 'red')
print(red_rectangle.color) # red

É preciso ter cuidado ao inicializar objetos mutáveis, como listas no construtor. Considere o seguinte exemplo:

class Rectangle2D(object):
    def __init__(self, width, height, pos=[0,0], color='blue'):  
        self.width = width
        self.height = height
        self.pos = pos
        self.color = color

r1 = Rectangle2D(5,3)
r2 = Rectangle2D(7,8)
r1.pos[0] = 4
r1.pos # [4, 0]
r2.pos # [4, 0] r2's pos has changed as well

Esse comportamento é causado pelo fato de que em Python os parâmetros padrão são vinculados na execução da função e não na declaração da função. Para obter uma variável de instância padrão que não é compartilhada entre instâncias, deve-se usar uma construção como esta:

class Rectangle2D(object):
    def __init__(self, width, height, pos=None, color='blue'):  
        self.width = width
        self.height = height
        self.pos = pos or [0, 0] # default value is [0, 0]
        self.color = color

r1 = Rectangle2D(5,3)
r2 = Rectangle2D(7,8)
r1.pos[0] = 4
r1.pos # [4, 0]
r2.pos # [0, 0] r2's pos hasn't changed

Composição da classe #

A composição de classes permite relações explícitas entre objetos. Neste exemplo, as pessoas vivem em cidades que pertencem a países. A composição permite que as pessoas acessem o número de todas as pessoas que vivem em seu país:

Listando todos os membros da classe #

A função dir() pode ser usada para obter uma lista dos membros de uma classe:

dir(Class)

Por exemplo:

>>> dir(list)
['__add__', '__class__', '__contains__', '__delattr__', '__delitem__', '__dir__', '__doc__', '__eq__', '__format__', '__ge__', '__getattribute__', '__getitem__', '__gt__', '__hash__', '__iadd__', '__imul__', '__init__', '__iter__', '__le__', '__len__', '__lt__', '__mul__', '__ne__', '__new__', '__reduce__', '__reduce_ex__', '__repr__', '__reversed__', '__rmul__', '__setattr__', '__setitem__', '__sizeof__', '__str__', '__subclasshook__', 'append', 'clear', 'copy', 'count', 'extend', 'index', 'insert', 'pop', 'remove', 'reverse', 'sort']

É comum procurar apenas por membros “não-mágicos”. Isso pode ser feito usando uma compreensão simples que lista os membros com nomes que não começam com __:

>>> [m for m in dir(list) if not m.startswith('__')]
['append', 'clear', 'copy', 'count', 'extend', 'index', 'insert', 'pop', 'remove', 'reverse', 'sort']

Ressalvas:

As classes podem definir um método __dir__(). Se esse método existir, chamar dir() chamará __dir__(), caso contrário, o Python tentará criar uma lista de membros da classe. Isso significa que a função dir pode ter resultados inesperados. Duas citações importantes da documentação oficial do python:

Se o objeto não fornecer dir(), a função tenta ao máximo coletar informações do atributo dict do objeto, se definido, e de seu objeto de tipo. A lista resultante não é necessariamente completa e pode ser imprecisa quando o objeto tem um getattr() personalizado.

Observação: Como dir() é fornecido principalmente como uma conveniência para uso em um prompt interativo, ele tenta fornecer um conjunto interessante de nomes mais do que tenta fornecer um conjunto de nomes rigorosa ou consistentemente definido, e seu o comportamento detalhado pode mudar entre as versões. Por exemplo, atributos de metaclasse não estão na lista de resultados quando o argumento é uma classe.

class Singleton:
    """
    A non-thread-safe helper class to ease implementing singletons.
    This should be used as a decorator -- not a metaclass -- to the
    class that should be a singleton.

    The decorated class can define one `__init__` function that
    takes only the `self` argument. Other than that, there are
    no restrictions that apply to the decorated class.

    To get the singleton instance, use the `Instance` method. Trying
    to use `__call__` will result in a `TypeError` being raised.

    Limitations: The decorated class cannot be inherited from.

    """

    def __init__(self, decorated):
        self._decorated = decorated

    def Instance(self):
        """
        Returns the singleton instance. Upon its first call, it creates a
        new instance of the decorated class and calls its `__init__` method.
        On all subsequent calls, the already created instance is returned.

        """
        try:
            return self._instance
        except AttributeError:
            self._instance = self._decorated()
            return self._instance

    def __call__(self):
        raise TypeError('Singletons must be accessed through `Instance()`.')

    def __instancecheck__(self, inst):
        return isinstance(inst, self._decorated)

class Single:

Descritores e pesquisas pontilhadas #

Descritores são objetos que são (geralmente) atributos de classes e que possuem qualquer um dos métodos especiais __get__, __set__ ou __delete__.

Descritores de dados têm qualquer um de __set__ ou __delete__

Eles podem controlar a pesquisa pontilhada em uma instância e são usados ​​para implementar funções, staticmethod, classmethod e property. Uma pesquisa pontilhada (por exemplo, instância foo da classe Foo procurando o atributo bar - ou seja, foo.bar) usa o seguinte algoritmo:

  1. bar é procurado na classe Foo. Se estiver lá e for um Descritor de dados, o descritor de dados será usado. É assim que property é capaz de controlar o acesso aos dados em uma instância, e as instâncias não podem substituir isso. Se um Descritor de dados não estiver lá, então

  2. bar é procurado na instância __dict__. É por isso que podemos substituir ou bloquear métodos sendo chamados de uma instância com uma pesquisa pontilhada. Se bar existir na instância, ele será usado. Se não, nós então

  3. procure na classe Foo por bar. Se for um Descritor, o protocolo do descritor será usado. É assim que as funções (neste contexto, métodos não vinculados), classmethod e staticmethod são implementadas. Caso contrário, ele simplesmente retorna o objeto lá, ou há um AttributeError
