O Python fornece métodos integrados e bibliotecas externas para criar, modificar, analisar e manipular datas e horas.

Analisando uma string em um objeto datetime com reconhecimento de fuso horário #

Python 3.2+ tem suporte para o formato %z ao analisar uma string em um objeto datetime .

Deslocamento UTC no formato +HHMM ou -HHMM (string vazia se o objeto for ingênuo).

import datetime
dt = datetime.datetime.strptime("2016-04-15T08:27:18-0500", "%Y-%m-%dT%H:%M:%S%z")

Para outras versões do Python, você pode usar uma biblioteca externa como dateutil, que faz com que a análise de uma string com fuso horário em um objeto datetime seja rápida.

import dateutil.parser
dt = dateutil.parser.parse("2016-04-15T08:27:18-0500")

A variável dt agora é um objeto datetime com o seguinte valor:

datetime.datetime(2016, 4, 15, 8, 27, 18, tzinfo=tzoffset(None, -18000))

Construindo datas com reconhecimento de fuso horário #

Por padrão, todos os objetos datetime são ingênuos. Para torná-los cientes do fuso horário, você deve anexar um objeto tzinfo, que fornece o deslocamento UTC e a abreviação do fuso horário como uma função de data e hora.

Fusos horários de deslocamento fixo

Para fusos horários que são um deslocamento fixo do UTC, no Python 3.2+, o módulo datetime fornece a classe timezone, uma implementação concreta de tzinfo, que recebe um timedelta e um parâmetro de nome (opcional):

from datetime import datetime, timedelta, timezone
JST = timezone(timedelta(hours=+9))

dt = datetime(2015, 1, 1, 12, 0, 0, tzinfo=JST)
print(dt)
# 2015-01-01 12:00:00+09:00

print(dt.tzname())
# UTC+09:00

dt = datetime(2015, 1, 1, 12, 0, 0, tzinfo=timezone(timedelta(hours=9), 'JST'))
print(dt.tzname)
# 'JST'

Para versões do Python anteriores a 3.2, é necessário usar uma biblioteca de terceiros, como dateutil. dateutil fornece uma classe equivalente, tzoffset, que (a partir da versão 2.5.3) recebe argumentos no formato dateutil.tz.tzoffset(tzname, offset), onde offset é especificado em segundos:

from datetime import datetime, timedelta
from dateutil import tz

JST = tz.tzoffset('JST', 9 * 3600) # 3600 seconds per hour
dt = datetime(2015, 1, 1, 12, 0, tzinfo=JST)
print(dt)
# 2015-01-01 12:00:00+09:00
print(dt.tzname)
# 'JST'

Zonas com horário de verão

Para zonas com horário de verão, as bibliotecas padrão python não fornecem uma classe padrão, portanto, é necessário usar uma biblioteca de terceiros. pytz e dateutil são bibliotecas populares que fornecem classes de fuso horário.

Além dos fusos horários estáticos, o dateutil fornece classes de fuso horário que usam o horário de verão (consulte a documentação do módulo tz ). Você pode usar o método tz.gettz() para obter um objeto de fuso horário, que pode ser passado diretamente para o construtor datetime:

from datetime import datetime
from dateutil import tz
local = tz.gettz() # Local time
PT = tz.gettz('US/Pacific') # Pacific time

dt_l = datetime(2015, 1, 1, 12, tzinfo=local) # I am in EST
dt_pst = datetime(2015, 1, 1, 12, tzinfo=PT)
dt_pdt = datetime(2015, 7, 1, 12, tzinfo=PT) # DST is handled automatically
print(dt_l)
# 2015-01-01 12:00:00-05:00
print(dt_pst)
# 2015-01-01 12:00:00-08:00
print(dt_pdt)
# 2015-07-01 12:00:00-07:00

CUIDADO: A partir da versão 2.5.3, o dateutil não trata datas ambíguas corretamente, e sempre usará como padrão a data posterior. Não há como construir um objeto com um fuso horário dateutil representando, por exemplo, 2015-11-01 1:30 EDT-4, pois isso é durante uma transição de horário de verão.

Todos os casos extremos são tratados corretamente ao usar pytz, mas os fusos horários pytz não devem ser anexados diretamente aos fusos horários através do construtor. Em vez disso, um fuso horário pytz deve ser anexado usando o método localize do fuso horário:

from datetime import datetime, timedelta
import pytz

PT = pytz.timezone('US/Pacific')
dt_pst = PT.localize(datetime(2015, 1, 1, 12))
dt_pdt = PT.localize(datetime(2015, 11, 1, 0, 30))
print(dt_pst)
# 2015-01-01 12:00:00-08:00
print(dt_pdt)
# 2015-11-01 00:30:00-07:00

Esteja ciente de que, se você executar aritmética de data e hora em um fuso horário compatível com pytz, deverá executar os cálculos em UTC (se desejar tempo decorrido absoluto) ou deverá chamar normalize() no resultado:

dt_new = dt_pdt + timedelta(hours=3) # This should be 2:30 AM PST
print(dt_new)
# 2015-11-01 03:30:00-07:00
dt_corrected = PT.normalize(dt_new)
print(dt_corrected)
# 2015-11-01 02:30:00-08:00

Uso básico de objetos de data e hora #

O módulo datetime contém três tipos principais de objetos - date, time e datetime.

import datetime

# Date object
today = datetime.date.today()
new_year = datetime.date(2017, 01, 01) #datetime.date(2017, 1, 1)

# Time object
noon = datetime.time(12, 0, 0) #datetime.time(12, 0)

# Current datetime
now = datetime.datetime.now()

# Datetime object
millenium_turn = datetime.datetime(2000, 1, 1, 0, 0, 0) #datetime.datetime(2000, 1, 1, 0, 0)

As operações aritméticas para esses objetos são suportadas apenas no mesmo tipo de dados e a execução de aritmética simples com instâncias de tipos diferentes resultará em um TypeError.

# subtraction of noon from today
noon-today
Traceback (most recent call last):
  File "<stdin>", line 1, in <module>
TypeError: unsupported operand type(s) for -: 'datetime.time' and 'datetime.date'
However, it is straightforward to convert between types.

# Do this instead
print('Time since the millenium at midnight: ',
      datetime.datetime(today.year, today.month, today.day) - millenium_turn)

# Or this
print('Time since the millenium at noon: ',
      datetime.datetime.combine(today, noon) - millenium_turn)

Calculando diferenças de tempo #

o módulo timedelta é útil para calcular as diferenças entre os tempos:

from datetime import datetime, timedelta
now = datetime.now()
then = datetime(2016, 5, 23)     # datetime.datetime(2016, 05, 23, 0, 0, 0)

Especificar a hora é opcional ao criar um novo objeto datetime

delta = now-then

delta é do tipo timedelta

print(delta.days)
# 60
print(delta.seconds)
# 40826

Para obter n dias depois e n dias antes da data, poderíamos usar:

n dias após a data:

def get_n_days_after_date(date_format="%d %B %Y", add_days=120):

    date_n_days_after = datetime.datetime.now() + timedelta(days=add_days)
    return date_n_days_after.strftime(date_format)

n dias antes da data:

 def get_n_days_before_date(self, date_format="%d %B %Y", days_before=120):

        date_n_days_ago = datetime.datetime.now() - timedelta(days=days_before)
        return date_n_days_ago.strftime(date_format)

Aritmética de data simples #

As datas não existem isoladamente. É comum que você precise encontrar a quantidade de tempo entre as datas ou determinar qual será a data de amanhã. Isso pode ser feito usando objetos timedelta

import datetime

today = datetime.date.today()
print('Today:', today)

yesterday = today - datetime.timedelta(days=1)
print('Yesterday:', yesterday)

tomorrow = today + datetime.timedelta(days=1)
print('Tomorrow:', tomorrow)

print('Time between tomorrow and yesterday:', tomorrow - yesterday)

Isso produzirá resultados semelhantes a:

Today: 2016-04-15
Yesterday: 2016-04-14
Tomorrow: 2016-04-16
Difference between tomorrow and yesterday: 2 days, 0:00:00

Alternando entre fusos horários #

Para alternar entre fusos horários, você precisa de objetos de data e hora que reconheçam o fuso horário.

from datetime import datetime
from dateutil import tz

utc = tz.tzutc()
local = tz.tzlocal()

utc_now = datetime.utcnow()
utc_now # Not timezone-aware.

utc_now = utc_now.replace(tzinfo=utc)
utc_now # Timezone-aware.

local_now = utc_now.astimezone(local)
local_now # Converted to local time.

Subtraindo meses de uma data com precisão #

Usando o módulo calendário

import calendar
from datetime import date

def monthdelta(date, delta):
    m, y = (date.month+delta) % 12, date.year + ((date.month)+delta-1) // 12
    if not m: m = 12
    d = min(date.day, calendar.monthrange(y, m)[1])
    return date.replace(day=d,month=m, year=y)

next_month = monthdelta(date.today(), 1) #datetime.date(2016, 10, 23)

Usando o módulo dateutils

import datetime
import dateutil.relativedelta

d = datetime.datetime.strptime("2013-03-31", "%Y-%m-%d")
d2 = d - dateutil.relativedelta.relativedelta(months=1)  #datetime.datetime(2013, 2, 28, 0, 0)

Iterar ao longo de datas #

Às vezes, você deseja iterar em um intervalo de datas de uma data de início a uma data de término. Você pode fazer isso usando a biblioteca datetime e o objeto timedelta:

import datetime

# The size of each step in days
day_delta = datetime.timedelta(days=1)

start_date = datetime.date.today()
end_date = start_date + 7*day_delta

for i in range((end_date - start_date).days):
    print(start_date + i*day_delta)

Que produz:

2016-07-23
2016-07-24
2016-07-25
2016-07-26

Analisando uma string com um nome de fuso horário curto em um objeto datetime com reconhecimento de fuso horário #

Usando a biblioteca dateutil como no [exemplo anterior de análise de carimbos de data/hora com reconhecimento de fuso horário](https://www.wikiod.com/pt/python/data-e-hora -time/1592/parsing-a-string-into-a-timezone-aware-datetime-object), também é possível analisar carimbos de data/hora com um nome de fuso horário “curto” especificado.

Para datas formatadas com nomes ou abreviações de fuso horário curtos, que geralmente são ambíguos (por exemplo, CST, que pode ser Horário Padrão Central, Horário Padrão da China, Horário Padrão de Cuba etc., mais informações podem ser encontradas [aqui](https://www. timeanddate.com/time/zones/)) ou não necessariamente disponível em um banco de dados padrão, é necessário especificar um mapeamento entre a abreviação de fuso horário e o objeto tzinfo.

from dateutil import tz
from dateutil.parser import parse

ET = tz.gettz('US/Eastern')
CT = tz.gettz('US/Central')
MT = tz.gettz('US/Mountain')
PT = tz.gettz('US/Pacific')

us_tzinfos = {'CST': CT, 'CDT': CT,
              'EST': ET, 'EDT': ET,
              'MST': MT, 'MDT': MT,
              'PST': PT, 'PDT': PT}

dt_est = parse('2014-01-02 04:00:00 EST', tzinfos=us_tzinfos)
dt_pst = parse('2016-03-11 16:00:00 PST', tzinfos=us_tzinfos)

Depois de executar isso:

dt_est
# datetime.datetime(2014, 1, 2, 4, 0, tzinfo=tzfile('/usr/share/zoneinfo/US/Eastern'))
dt_pst
# datetime.datetime(2016, 3, 11, 16, 0, tzinfo=tzfile('/usr/share/zoneinfo/US/Pacific'))

Vale a pena notar que, se estiver usando um fuso horário pytz com este método, ele não será localizado corretamente:

from dateutil.parser import parse
import pytz

EST = pytz.timezone('America/New_York')
dt = parse('2014-02-03 09:17:00 EST', tzinfos={'EST': EST})

Isso simplesmente anexa o fuso horário pytz ao datetime:

dt.tzinfo # Will be in Local Mean Time!
# <DstTzInfo 'America/New_York' LMT-1 day, 19:04:00 STD>

Se estiver usando este método, você provavelmente deve re-localizar a parte ingênua do datetime após a análise:

dt_fixed = dt.tzinfo.localize(dt.replace(tzinfo=None))
dt_fixed.tzinfo # Now it's EST.
# <DstTzInfo 'America/New_York' EST-1 day, 19:00:00 STD>)

Análise de data e hora difusa (extrair data e hora de um texto) #

É possível extrair uma data de um texto usando o dateutil parser em um modo “fuzzy”, onde os componentes da string não reconhecidos como parte de uma data são ignorados.

from dateutil.parser import parse

dt = parse("Today is January 1, 2047 at 8:21:00AM", fuzzy=True)
print(dt)

dt agora é um objeto datetime e você veria datetime.datetime(2047, 1, 1, 8, 21) impresso.

Analisando um timestamp ISO 8601 arbitrário com bibliotecas mínimas #

O Python tem suporte limitado apenas para analisar carimbos de data e hora ISO 8601. Para strptime você precisa saber exatamente em que formato está. Como complicação, a stringificação de um datetime é um carimbo de data e hora ISO 8601, com espaço como separador e fração de 6 dígitos:

str(datetime.datetime(2016, 7, 22, 9, 25, 59, 555555))
# '2016-07-22 09:25:59.555555'

mas se a fração for 0, nenhuma parte fracionária será emitida

str(datetime.datetime(2016, 7, 22, 9, 25, 59, 0))
# '2016-07-22 09:25:59'

Mas essas 2 formas precisam de um formato diferente para strptime. Além disso, strptime' não suporta a análise de todos os fusos horários de minuto que possuem um :, portanto 2016-07-22 09:25:59+0300pode ser analisado, mas o formato padrão2016-07- 22 09:25:59+03:00` não pode.

Existe uma biblioteca de arquivo único chamada [iso8601](https ://pypi.python.org/pypi/iso8601) que analisa corretamente os carimbos de data e hora ISO 8601 e apenas eles.

Ele suporta frações e fusos horários, e o separador T, todos com uma única função:

import iso8601
iso8601.parse_date('2016-07-22 09:25:59')
# datetime.datetime(2016, 7, 22, 9, 25, 59, tzinfo=<iso8601.Utc>)
iso8601.parse_date('2016-07-22 09:25:59+03:00')
# datetime.datetime(2016, 7, 22, 9, 25, 59, tzinfo=<FixedOffset '+03:00' ...>)
iso8601.parse_date('2016-07-22 09:25:59Z')
# datetime.datetime(2016, 7, 22, 9, 25, 59, tzinfo=<iso8601.Utc>)
iso8601.parse_date('2016-07-22T09:25:59.000111+03:00')
# datetime.datetime(2016, 7, 22, 9, 25, 59, 111, tzinfo=<FixedOffset '+03:00' ...>)

Se nenhum fuso horário for definido, o padrão iso8601.parse_date será UTC. A zona padrão pode ser alterada com o argumento de palavra-chave default_zone. Notavelmente, se for “Nenhum” em vez do padrão, os carimbos de data e hora que não tiverem um fuso horário explícito serão retornados como datas ingênuas:

iso8601.parse_date('2016-07-22T09:25:59', default_timezone=None)
# datetime.datetime(2016, 7, 22, 9, 25, 59)
iso8601.parse_date('2016-07-22T09:25:59Z', default_timezone=None)
# datetime.datetime(2016, 7, 22, 9, 25, 59, tzinfo=<iso8601.Utc>)

Obtenha um carimbo de data/hora ISO 8601 #

Sem fuso horário, com microssegundos #

from datetime import datetime

datetime.now().isoformat()
# Out: '2016-07-31T23:08:20.886783'

Com fuso horário, com microssegundos #

from datetime import datetime
from dateutil.tz import tzlocal

datetime.now(tzlocal()).isoformat()
# Out: '2016-07-31T23:09:43.535074-07:00'

Com fuso horário, sem microssegundos #

from datetime import datetime
from dateutil.tz import tzlocal

datetime.now(tzlocal()).replace(microsecond=0).isoformat()
# Out: '2016-07-31T23:10:30-07:00'

Consulte [ISO 8601] (https://en.wikipedia.org/wiki/ISO_8601) para obter mais informações sobre o formato ISO 8601.
