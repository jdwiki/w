Python faz operadores matemáticos comuns por conta própria, incluindo divisão inteira e float, multiplicação, exponenciação, adição e subtração. O módulo matemático (incluído em todas as versões padrão do Python) oferece funcionalidades expandidas, como funções trigonométricas, operações de raiz, logaritmos e muito mais.

Divisão #

Python faz divisão inteira quando ambos os operandos são inteiros. O comportamento dos operadores de divisão do Python mudou do Python 2.xe 3.x (consulte também https://www.wikiod.com/pt/python/incompatibilidades-migrando-do-python-2-para-o-python-3 2797/divisão inteira).

a, b, c, d, e = 3, 2, 2.0, -3, 10

Em Python 2 o resultado do operador ‘/’ depende do tipo do numerador e do denominador.

a / b                  # = 1 

a / c                  # = 1.5

d / b                  # = -2

b / a                  # = 0

d / e                  # = -1

Observe que, como a e b são ints, o resultado é um int.

O resultado é sempre arredondado para baixo (piso).

Como c é um float, o resultado de a/c é um float.

Você também pode usar o módulo do operador:

import operator        # the operator module provides 2-argument arithmetic functions
operator.div(a, b)     # = 1
operator.__div__(a, b) # = 1

E se você quiser divisão float:

Recomendado:

from __future__ import division # applies Python 3 style division to the entire module
a / b                  # = 1.5 
a // b                 # = 1

Ok (se você não quiser aplicar em todo o módulo):

a / (b * 1.0)          # = 1.5
1.0 * a / b            # = 1.5
a / b * 1.0            # = 1.0    (careful with order of operations)

from operator import truediv
truediv(a, b)          # = 1.5

Não recomendado (pode gerar TypeError, por exemplo, se o argumento for complexo):

float(a) / b           # = 1.5
a / float(b)           # = 1.5

O operador ’ // ’ no Python 2 força a divisão por piso, independentemente do tipo.

a // b                # = 1
a // c                # = 1.0

No Python 3, o operador / executa a divisão ’true’ independentemente dos tipos. O operador // executa a divisão do piso e mantém o tipo.

a / b                  # = 1.5 
e / b                  # = 5.0
a // b                 # = 1
a // c                 # = 1.0

import operator            # the operator module provides 2-argument arithmetic functions
operator.truediv(a, b)     # = 1.5
operator.floordiv(a, b)    # = 1
operator.floordiv(a, c)    # = 1.0

Combinações possíveis (tipos internos):

  • int e int (fornece um int em Python 2 e um float em Python 3)

Adição #

a, b = 1, 2

# Using the "+" operator:
a + b                  # = 3

# Using the "in-place" "+=" operator to add and assign:
a += b                 # a = 3 (equivalent to a = a + b)

import operator        # contains 2 argument arithmetic functions for the examples

operator.add(a, b)     # = 5  since a is set to 3 right before this line

# The "+=" operator is equivalent to: 
a = operator.iadd(a, b)    # a = 5 since a is set to 3 right before this line

"first string " + "second string"    # = 'first string second string'

[1, 2, 3] + [4, 5, 6]                # = [1, 2, 3, 4, 5, 6]

Exponencial #

a, b = 2, 3

(a ** b)               # = 8
pow(a, b)              # = 8

import math
math.pow(a, b)         # = 8.0 (always float; does not allow complex results)

import operator
operator.pow(a, b)     # = 8

Outra diferença entre o pow interno e o math.pow é que o pow interno pode aceitar três argumentos:

a, b, c = 2, 3, 2

pow(2, 3, 2)           # 0, calculates (2 ** 3) % 2, but as per Python docs,
                       #    does so more efficiently

Funções especiais #

A função math.sqrt(x) calcula a raiz quadrada de x.

import math
import cmath
c = 4
math.sqrt(c)           # = 2.0 (always float; does not allow complex results)
cmath.sqrt(c)          # = (2+0j) (always complex)

Para calcular outras raízes, como uma raiz cúbica, eleve o número ao inverso do grau da raiz. Isso pode ser feito com qualquer uma das funções ou operador exponencial.

 import math
 x = 8
 math.pow(x, 1/3) # evaluates to 2.0
 x**(1/3) # evaluates to 2.0

A função math.exp(x) calcula e ** x.

math.exp(0)  # 1.0
math.exp(1)  # 2.718281828459045 (e)

A função math.expm1(x) calcula e ** x - 1. Quando x é pequeno, isso dá uma precisão significativamente melhor do que math.exp(x) - 1.

math.expm1(0)       # 0.0

math.exp(1e-6) - 1  # 1.0000004999621837e-06
math.expm1(1e-6)    # 1.0000005000001665e-06
# exact result      # 1.000000500000166666708333341666...

Funções trigonométricas #

a, b = 1, 2

import math

math.sin(a)  # returns the sine of 'a' in radians
# Out: 0.8414709848078965

math.cosh(b)  # returns the inverse hyperbolic cosine of 'b' in radians
# Out: 3.7621956910836314

math.atan(math.pi)  # returns the arc tangent of 'pi' in radians
# Out: 1.2626272556789115

math.hypot(a, b) # returns the Euclidean norm, same as math.sqrt(a*a + b*b)
# Out: 2.23606797749979

Observe que math.hypot(x, y) também é o comprimento do vetor (ou distância euclidiana) da origem (0, 0) até o ponto (x, y).

Para calcular a distância euclidiana entre dois pontos (x1, y1) & (x2, y2) você pode usar math.hypot da seguinte forma

math.hypot(x2-x1, y2-y1)

Para converter de radianos -> graus e graus -> radianos respectivamente use math.degrees e math.radians

math.degrees(a)
# Out: 57.29577951308232

math.radians(57.29577951308232)
# Out: 1.0

Operações Inplace #

É comum dentro de aplicativos precisar ter um código como este:

a = a + 1

ou

a = a * 2

Existe um atalho eficaz para essas operações no local:

a += 1
# and
a *= 2

Qualquer operador matemático pode ser usado antes do caractere ‘=’ para fazer uma operação inplace:

  • -= decrementa a variável no lugar
  • += incrementa a variável no lugar
  • *= multiplique a variável no lugar
  • /= divide a variável no lugar
  • //= floor divide a variável no lugar # Python 3
  • %= retorna o módulo da variável no lugar
  • **= aumentar para uma potência no lugar

Existem outros operadores in place para os operadores bit a bit (^, | etc)

Logaritmos #

Por padrão, a função math.log calcula o logaritmo de um número, base e. Você pode opcionalmente especificar uma base como o segundo argumento.

import math
import cmath

math.log(5)         # = 1.6094379124341003
# optional base argument. Default is math.e
math.log(5, math.e) # = 1.6094379124341003
cmath.log(5)        # = (1.6094379124341003+0j)
math.log(1000, 10)   # 3.0 (always returns float)
cmath.log(1000, 10)  # (3+0j)

Existem variações especiais da função math.log para diferentes bases.

# Logarithm base e - 1 (higher precision for low values)
math.log1p(5)       # = 1.791759469228055

# Logarithm base 2
math.log2(8)        # = 3.0

# Logarithm base 10
math.log10(100)     # = 2.0
cmath.log10(100)    # = (2+0j)

Módulo #

Como em muitas outras linguagens, o Python usa o operador % para calcular o módulo.

3 % 4     # 3
10 % 2    # 0
6 % 4     # 2

Ou usando o módulo operator:

import operator

operator.mod(3 , 4)     # 3
operator.mod(10 , 2)    # 0
operator.mod(6 , 4)     # 2

Você também pode usar números negativos.

-9 % 7     # 5
9 % -7     # -5
-9 % -7    # -2

Se você precisar encontrar o resultado da divisão inteira e do módulo, você pode usar a função divmod como um atalho:

quotient, remainder = divmod(9, 4)
# quotient = 2, remainder = 1 as 4 * 2 + 1 == 9
