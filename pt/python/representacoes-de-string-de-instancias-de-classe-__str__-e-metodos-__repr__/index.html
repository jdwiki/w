Representações de string de instâncias de classe __str__ e métodos __repr__

  • Motivação

Nesta página

  • Motivação
  • Ambos os métodos implementados, estilo eval-ida e volta repr()

Uma nota sobre a implementação de ambos os métodos #

Quando ambos os métodos são implementados, é um pouco comum ter um método __str__ que retorna uma representação amigável (por exemplo, “Ace of Spaces”) e __repr__ retorna um [eval](https://docs.python .org/3/library/functions.html#eval) representação amigável.

Na verdade, os documentos do Python para repr() observam exatamente isso:

Para muitos tipos, esta função tenta retornar uma string que produziria um objeto com o mesmo valor quando passado para eval(), caso contrário a representação é uma string entre colchetes que contém o nome do tipo do objeto juntamente com informações adicionais, muitas vezes incluindo o nome e endereço do objeto.

O que isso significa é que __str__ pode ser implementado para retornar algo como “Ace of Spaces” como mostrado anteriormente, __repr__ pode ser implementado para retornar Card('Spades', 1)

Esta string pode ser passada diretamente de volta para eval em uma espécie de “ida e volta”:

object -> string -> object

Um exemplo de implementação de tal método pode ser:

def __repr__(self):
    return "Card(%s, %d)" % (self.suit, self.pips)

Notas #

[1] Esta saída é específica da implementação. A string exibida é de cpython.

[2] Você já deve ter visto o resultado desta divisão str()/repr() e não sabia disso. Quando strings contendo caracteres especiais como barras invertidas são convertidas em strings via str() as barras invertidas aparecem como estão (elas aparecem uma vez). Quando eles são convertidos em strings via repr() (por exemplo, como elementos de uma lista sendo exibida), as barras invertidas são escapadas e, portanto, aparecem duas vezes.

Motivação #

Então você acabou de criar sua primeira classe em Python, uma pequena classe que encapsula uma carta de baralho:

class Card:
    def __init__(self, suit, pips):
        self.suit = suit
        self.pips = pips

Em outra parte do seu código, você cria algumas instâncias dessa classe:

ace_of_spades = Card('Spades', 1)
four_of_clubs = Card('Clubs',  4)
six_of_hearts = Card('Hearts', 6)

Você até criou uma lista de cartas, para representar uma “mão”:

my_hand = [ace_of_spades, four_of_clubs, six_of_hearts]

Agora, durante a depuração, você quer ver como está sua mão, então você faz o que vem naturalmente e escreve:

print(my_hand)

Mas o que você recebe de volta é um monte de rabiscos:

[<__main__.Card instance at 0x0000000002533788>, 
 <__main__.Card instance at 0x00000000025B95C8>, 
 <__main__.Card instance at 0x00000000025FF508>]

Confuso, você tenta imprimir apenas um único cartão:

print(ace_of_spades)

E novamente, você obtém esta saída estranha:

<__main__.Card instance at 0x0000000002533788>

Não tenha medo. Estamos prestes a consertar isso.

Antes, porém, é importante entender o que está acontecendo aqui. Quando você escreveu print(ace_of_spades), você disse ao Python que queria que ele imprimisse informações sobre a instância Card que seu código está chamando ace_of_spades. E para ser justo, ele fez.

Essa saída é composta por dois bits importantes: o type do objeto e o [id](https: //docs.python.org/3/library/functions.html#id). A segunda parte sozinha (o número hexadecimal) é suficiente para identificar exclusivamente o objeto no momento da chamada print.[1]

O que realmente aconteceu foi que você pediu ao Python para “colocar em palavras” a essência daquele objeto e então exibi-lo para você. Uma versão mais explícita do mesmo maquinário pode ser:

string_of_card = str(ace_of_spades)
print(string_of_card)

Na primeira linha, você tenta transformar sua instância Card em uma string, e na segunda você a exibe.

O problema #

O problema que você está encontrando surge devido ao fato de que, enquanto você disse ao Python tudo o que precisava saber sobre a classe Card para você criar cartões, você não disse como queria Card instâncias a serem convertidas em strings.

E como ele não sabia, quando você (implicitamente) escreveu str(ace_of_spades), ele lhe deu o que você viu, uma representação genérica da instância Card.

A Solução (Parte 1) #

Mas podemos dizer ao Python como queremos que as instâncias de nossas classes personalizadas sejam convertidas em strings. E a maneira como fazemos isso é com o método __str__ “dunder” (para sublinhado duplo) ou “magic”.

Sempre que você disser ao Python para criar uma string a partir de uma instância de classe, ele procurará um método __str__ na classe e o chamará.

Considere a seguinte versão atualizada da nossa classe Card:

class Card:
    def __init__(self, suit, pips):
        self.suit = suit
        self.pips = pips

    def __str__(self):
        special_names = {1:'Ace', 11:'Jack', 12:'Queen', 13:'King'}

        card_name = special_names.get(self.pips, str(self.pips))

        return "%s of %s" % (card_name, self.suit)

Aqui, agora definimos o método __str__ em nossa classe Card que, após uma simples busca no dicionário por cartões de rosto, retorna uma string formatada da maneira que decidirmos.

(Observe que “returns” está em negrito aqui, para enfatizar a importância de retornar uma string, e não simplesmente imprimi-la. A impressão pode parecer funcionar, mas você teria o cartão impresso quando fizesse algo como str( ace_of_spades), mesmo sem ter uma chamada de função de impressão em seu programa principal. Então, para ser claro, certifique-se de que __str__ retorne uma string.).

O método __str__ é um método, então o primeiro argumento será self e não deve aceitar nem passar argumentos adicionais.

Voltando ao nosso problema de exibir o cartão de uma maneira mais amigável, se executarmos novamente:

ace_of_spades = Card('Spades', 1)
print(ace_of_spades)

Veremos que nossa saída é muito melhor:

Ás de espadas

Muito bom, terminamos, certo?

Bem, apenas para cobrir nossas bases, vamos verificar novamente se resolvemos o primeiro problema que encontramos, imprimindo a lista de instâncias de Card, a mão.

Assim, verificamos novamente o seguinte código:

my_hand = [ace_of_spades, four_of_clubs, six_of_hearts]
print(my_hand)

E, para nossa surpresa, recebemos esses códigos hexadecimais engraçados novamente:

[<__main__.Card instance at 0x00000000026F95C8>, 
 <__main__.Card instance at 0x000000000273F4C8>, 
 <__main__.Card instance at 0x0000000002732E08>]

O que está acontecendo? Dissemos ao Python como queríamos que nossas instâncias Card fossem exibidas, por que aparentemente parece esquecer?

A Solução (Parte 2) #

Bem, o mecanismo dos bastidores é um pouco diferente quando o Python deseja obter a representação de strings de itens em uma lista. Acontece que Python não se importa com __str__ para este propósito.

Em vez disso, ele procura um método diferente, __repr__, e se isso não for encontrado, ele recorre à “coisa hexadecimal”.[2]

Então você está dizendo que eu tenho que fazer dois métodos para fazer a mesma coisa? Um para quando eu quiser ‘imprimir’ meu cartão sozinho e outro quando ele estiver em algum tipo de container?

Não, mas primeiro vamos ver como nossa classe seria se implementássemos os métodos __str__ e __repr__:

class Card:
    special_names = {1:'Ace', 11:'Jack', 12:'Queen', 13:'King'}

    def __init__(self, suit, pips):
        self.suit = suit
        self.pips = pips

    def __str__(self):
        card_name = Card.special_names.get(self.pips, str(self.pips))
        return "%s of %s (S)" % (card_name, self.suit)

    def __repr__(self):
        card_name = Card.special_names.get(self.pips, str(self.pips))
        return "%s of %s (R)" % (card_name, self.suit)

Aqui, a implementação dos dois métodos __str__ e __repr__ são exatamente iguais, exceto que, para diferenciar entre os dois métodos, (S) é adicionado às strings retornadas por __str__ e (R) é adicionado às strings retornadas por __repr__.

Note que assim como nosso método __str__, __repr__ não aceita argumentos e retorna uma string.

Podemos ver agora qual método é responsável por cada caso:

ace_of_spades = Card('Spades', 1)
four_of_clubs = Card('Clubs',  4)
six_of_hearts = Card('Hearts', 6)

my_hand = [ace_of_spades, four_of_clubs, six_of_hearts]

print(my_hand)           # [Ace of Spades (R), 4 of Clubs (R), 6 of Hearts (R)]

print(ace_of_spades)     # Ace of Spades (S)

Como foi abordado, o método __str__ foi chamado quando passamos nossa instância Card para print e o método __repr__ foi chamado quando passamos uma lista de nossas instâncias para print.

Neste ponto, vale ressaltar que, assim como podemos criar explicitamente uma string de uma instância de classe personalizada usando [str()](https://docs.python.org/3/library/functions.html#func- str) como fizemos anteriormente, também podemos criar explicitamente uma representação de string de nossa classe com uma função interna chamada [repr()](https://docs.python.org/3/library /functions.html#repr).

Por exemplo:

str_card = str(four_of_clubs)
print(str_card)                     # 4 of Clubs (S)

repr_card = repr(four_of_clubs)
print(repr_card)                    # 4 of Clubs (R)

Além disso, se definido, podemos poder chamar os métodos diretamente (embora pareça um pouco obscuro e desnecessário):

print(four_of_clubs.__str__())     # 4 of Clubs (S)

print(four_of_clubs.__repr__())    # 4 of Clubs (R)

Sobre essas funções duplicadas… #

Os desenvolvedores de Python perceberam que, no caso de você querer que strings idênticas fossem retornadas de str() e repr(), você poderia ter que duplicar os métodos funcionalmente - algo que ninguém gosta.

Então, em vez disso, existe um mecanismo para eliminar a necessidade disso. Um que eu passei por você até este ponto. Acontece que se uma classe implementa o método __repr__ mas não o método __str__, e você passa uma instância dessa classe para str() (seja implícita ou explicitamente), o Python retornará ao seu __repr__ e use isso.

Então, para ficar claro, considere a seguinte versão da classe Card:

class Card:
    special_names = {1:'Ace', 11:'Jack', 12:'Queen', 13:'King'}

    def __init__(self, suit, pips):
        self.suit = suit
        self.pips = pips

    def __repr__(self):
        card_name = Card.special_names.get(self.pips, str(self.pips))
        return "%s of %s" % (card_name, self.suit)

Observe que esta versão somente implementa o método __repr__. No entanto, chamadas para str() resultam na versão amigável:

print(six_of_hearts)            # 6 of Hearts  (implicit conversion)
print(str(six_of_hearts))       # 6 of Hearts  (explicit conversion)

assim como as chamadas para repr():

print([six_of_hearts])          #[6 of Hearts] (implicit conversion)
print(repr(six_of_hearts))      # 6 of Hearts  (explicit conversion)

Resumo #

Para você capacitar suas instâncias de classe para “se mostrarem” de maneira amigável, você deve considerar implementar pelo menos o método __repr__ de sua classe. Se a memória serve, durante uma palestra, Raymond Hettinger disse que garantir que as classes implementem __repr__ é uma das primeiras coisas que ele procura ao fazer revisões de código Python, e agora deve estar claro o porquê. A quantidade de informações que você poderia ter adicionado a instruções de depuração, relatórios de falhas ou arquivos de log com um método simples é esmagadora quando comparada com as informações insignificantes e muitas vezes menos úteis (tipo, id) fornecidas por padrão .

Se você quiser representações diferentes para quando, por exemplo, dentro de um contêiner, você desejará implementar os métodos __repr__ e __str__. (Mais sobre como você pode usar esses dois métodos de maneira diferente abaixo).

Ambos os métodos implementados, estilo eval-ida e volta repr() #

class Card:
    special_names = {1:'Ace', 11:'Jack', 12:'Queen', 13:'King'}

    def __init__(self, suit, pips):
        self.suit = suit
        self.pips = pips

    # Called when instance is converted to a string via str()
    # Examples:
    #   print(card1)
    #   print(str(card1)
    def __str__(self):
        card_name = Card.special_names.get(self.pips, str(self.pips))
        return "%s of %s" % (card_name, self.suit)

    # Called when instance is converted to a string via repr()
    # Examples:
    #   print([card1, card2, card3])
    #   print(repr(card1))
    def __repr__(self):
        return "Card(%s, %d)" % (self.suit, self.pips)
