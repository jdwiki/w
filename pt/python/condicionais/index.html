Expressões condicionais, envolvendo palavras-chave como if, elif e else, fornecem aos programas Python a capacidade de executar ações diferentes dependendo de uma condição booleana: True ou False. Esta seção cobre o uso de condicionais Python, lógica booleana e instruções ternárias.

Sintaxe #

  • <expression> if <conditional> else <expression> # Operador Ternário

Expressão Condicional (ou “O Operador Ternário”) #

O operador ternário é usado para expressões condicionais embutidas. É melhor usado em operações simples e concisas que são de fácil leitura.

  • A ordem dos argumentos é diferente de muitas outras linguagens (como C, Ruby, Java, etc.), o que pode levar a bugs quando pessoas não familiarizadas com o comportamento “surpreendente” do Python o usam (podem inverter a ordem).

  • Alguns acham isso “desajeitado”, pois vai contra o fluxo normal do pensamento (pensando primeiro na condição e depois nos efeitos).

    n = 5

    “Greater than 2” if n > 2 else “Smaller than or equal to 2”

    Out: ‘Greater than 2’ #

O resultado dessa expressão será como é lido em inglês - se a expressão condicional for True, ela será avaliada como a expressão do lado esquerdo, caso contrário, do lado direito.

As operações tenárias também podem ser aninhadas, como aqui:

n = 5
"Hello" if n > 10 else "Goodbye" if n > 5 else "Good day"

Eles também fornecem um método para incluir condicionais em [funções lambda][1].

[1]: https://www.wikiod.com/pt/python/funcoes#Funções Lambda (em linha/anônimas)

if, elif e else #

Em Python você pode definir uma série de condicionais usando if para o primeiro, elif para o resto, até o final (opcional) else para qualquer coisa que não seja capturada pelas outras condicionais.

number = 5

if number > 2:
    print("Number is bigger than 2.")
elif number < 2:  # Optional clause (you can have multiple elifs)
    print("Number is smaller than 2.")
else:  # Optional clause (you can only have one else)
    print("Number is 2.")

Saídas Número é maior que 2

Usar else if em vez de elif acionará um erro de sintaxe e não é permitido.

Valores de verdade #

Os valores a seguir são considerados falsey, pois são avaliados como “False” quando aplicados a um operador booleano.

  • Nenhum
  • Falso
  • 0, ou qualquer valor numérico equivalente a zero, por exemplo 0L, 0.0, 0j
  • Sequências vazias: '', "", (), []
  • Mapeamentos vazios: {}
  • Tipos definidos pelo usuário onde os métodos __bool__ ou __len__ retornam 0 ou False

Todos os outros valores em Python são avaliados como True.


Nota: Um erro comum é simplesmente verificar a falsidade de uma operação que retorna valores falsos diferentes onde a diferença é importante. Por exemplo, usando if foo() em vez do mais explícito if foo() is None

Expressões lógicas booleanas #

Expressões lógicas booleanas, além de avaliar como True ou False, retornam o valor que foi interpretado como True ou False. É uma maneira Pythonic de representar a lógica que poderia exigir um teste if-else.

E operador #

O operador and avalia todas as expressões e retorna a última expressão se todas as expressões forem avaliadas como True. Caso contrário, ele retorna o primeiro valor avaliado como False:

>>> 1 and 2
2

>>> 1 and 0
0

>>> 1 and "Hello World"
"Hello World"

>>> "" and "Pancakes"
""

Ou operador #

O operador or avalia as expressões da esquerda para a direita e retorna o primeiro valor avaliado como True ou o último valor (se nenhum for True).

>>> 1 or 2
1

>>> None or 1
1

>>> 0 or []
[]

Avaliação preguiçosa #

Ao usar essa abordagem, lembre-se de que a avaliação é preguiçosa. Expressões que não precisam ser avaliadas para determinar o resultado não são avaliadas. Por exemplo:

>>> def print_me():
        print('I am here!')
>>> 0 and print_me()
0

No exemplo acima, print_me nunca é executado porque o Python pode determinar que a expressão inteira é False quando encontra o 0 (False). Tenha isso em mente se print_me precisar ser executado para servir a lógica do seu programa.

Teste para várias condições #

Um erro comum ao verificar várias condições é aplicar a lógica incorretamente.

Este exemplo está tentando verificar se duas variáveis ​​são cada uma maior que 2. A instrução é avaliada como - if (a) e (b > 2). Isso produz um resultado inesperado porque bool(a) é avaliado como True quando a não é zero.

>>> a = 1
>>> b = 6
>>> if a and b > 2:
...     print('yes')
... else:
...     print('no')
    
yes

Cada variável precisa ser comparada separadamente.

>>> if a > 2 and b > 2:
...     print('yes')
... else:
...     print('no')
        
no

Outro erro semelhante é cometido ao verificar se uma variável é um de vários valores. A instrução neste exemplo é avaliada como - if (a == 3) ou (4) ou (6). Isso produz um resultado inesperado porque bool(4) e bool(6) são avaliados como True

>>> a = 1
>>> if a == 3 or 4 or 6:
...     print('yes')
... else:
...     print('no')
       
yes

Novamente cada comparação deve ser feita separadamente

>>> if a == 3 or a == 4 or a == 6:
...     print('yes')
... else:
...     print('no')
        
no

Usar o operador in é a maneira canônica de escrever isso.

>>> if a in (3, 4, 6):
...     print('yes')
... else:
...     print('no')
    
no

Outra declaração #

if condition:
    body
else:
    body

A instrução else executará seu corpo somente se todas as instruções condicionais anteriores forem avaliadas como False.

if True:
    print "It is true!"
else:
    print "This won't get printed.."

# Output: It is true!

if False:
    print "This won't get printed.."
else:
    print "It is false!"

# Output: It is false!

Usando a função cmp para obter o resultado da comparação de dois objetos #

Python 2 inclui uma função cmp que permite determinar se um objeto é menor, igual ou maior que outro objeto. Esta função pode ser usada para escolher uma opção de uma lista com base em uma dessas três opções.

Suponha que você precise imprimir 'maior que' se x > y, 'menor que' se x < y e 'igual' se x == y.

['equal', 'greater than', 'less than', ][cmp(x,y)]

# x,y = 1,1 output: 'equal'
# x,y = 1,2 output: 'less than'
# x,y = 2,1 output: 'greater than'

cmp(x,y) retorna os seguintes valores

Comparação Resultado
x < y -1
x == y 0
x > y 1

Esta função foi removida no Python 3. Você pode usar a função auxiliar cmp_to_key(func) localizada em functools em Python 3 para converter funções de comparação antigas em funções-chave.

Se declaração #

if condition:
    body

As instruções if verificam a condição. Se for avaliado como True, ele executa o corpo da instrução if. Se for avaliado como ‘False’, ele pula o corpo.

if True:
    print "It is true!"
>> It is true!

if False:
    print "This won't get printed.."

A condição pode ser qualquer expressão válida:

if 2 + 2 == 4:
    print "I know math!"
>> I know math!

Testando se um objeto é None e atribuindo-o #

Muitas vezes você desejará atribuir algo a um objeto se for ‘Nenhum’, indicando que não foi atribuído. Usaremos aDate.

A maneira mais simples de fazer isso é usar o teste is None.

if aDate is None:
    aDate=datetime.date.today()

(Observe que é mais Pythonic dizer is None em vez de == None.)

Mas isso pode ser otimizado um pouco explorando a noção de que not None será avaliado como True em uma expressão booleana. O código a seguir é equivalente:

if not aDate:
    aDate=datetime.date.today()

Mas há uma maneira mais Pythonic. O código a seguir também é equivalente:

aDate=aDate or datetime.date.today()

Isso faz uma [avaliação de curto-circuito][1]. Se aDate for inicializado e for not None, ele será atribuído a si mesmo sem efeito líquido. Se is None, então datetime.date.today() é atribuído a aDate.

[1]: https://www.wikiod.com/pt/python/operadores-booleanos#Avaliação de curto-circuito

Avaliação de expressão condicional usando compreensões de lista #

Python permite hackear compreensões de lista para avaliar expressões condicionais.

Por exemplo,

[value_false, value_true][<conditional-test>]

Exemplo:

>> n = 16
>> print [10, 20][n <= 15]
10

Aqui n<=15 retorna False (que equivale a 0 em Python). Então, o que o Python está avaliando é:

[10, 20][n <= 15]
==> [10, 20][False] 
==> [10, 20][0]     #False==0, True==1 (Check Boolean Equivalencies in Python)
==> 10

O método embutido __cmp__ retornou 3 valores possíveis: 0, 1, -1, onde cmp(x,y) retornou 0: se ambos os objetos fossem iguais 1: x > y -1: x < y

Isso pode ser usado com compreensões de lista para retornar o primeiro (ou seja, índice 0), segundo (ou seja, índice 1) e último (ou seja, índice -1) elemento da lista. Dando-nos uma condicional deste tipo:

[value_equals, value_greater, value_less][<conditional-test>]

Finalmente, em todos os exemplos acima, o Python avalia os dois ramos antes de escolher um. Para avaliar apenas o branch escolhido:

[lambda: value_false, lambda: value_true][<test>]()

onde adicionar o () no final garante que as funções lambda sejam chamadas/avaliadas apenas no final. Assim, avaliamos apenas o ramo escolhido.

Exemplo:

count = [lambda:0, lambda:N+1][count==N]()
