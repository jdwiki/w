pip é o gerenciador de pacotes mais usado para o Python Package Index, instalado por padrão com versões recentes do Python.

Sintaxe #

  • pip <comando> [opções] onde <comando> é um dos seguintes:
  • instalar
    • Install packages
  • Desinstalar
    • Uninstall packages
  • congelar
    • Output installed packages in requirements format
  • Lista
    • List installed packages
  • mostrar
    • Show information about installed packages
  • procurar
    • Search PyPI for packages
  • roda
    • Build wheels from your requirements
  • fecho eclair
    • Zip individual packages (deprecated)
  • descompactar
    • Unzip individual packages (deprecated)
  • pacote
    • Create pybundles (deprecated)
  • ajuda
    • Show help for commands

Algumas vezes, o pip irá realizar uma compilação manual do código nativo. No Linux, o python escolherá automaticamente um compilador C disponível em seu sistema. Consulte a tabela abaixo para obter a versão necessária do Visual Studio/Visual C++ no Windows (as versões mais recentes não funcionarão).

Versão do Python Versão do Visual Studio Versão Visual C++
2.6 - 3.2 Visual Studio 2008 Visual C++ 9.0
3.3 - 3.4 Visual Studio 2010 Visual C++ 10.0
3.5 Visual Studio 2015 Visual C++ 14.0

Fonte: wiki.python.org

Instalar pacotes #

Para instalar a versão mais recente de um pacote chamado SomePackage:

$ pip install SomePackage

Para instalar uma versão específica de um pacote:

$ pip install SomePackage==1.0.4

Para especificar uma versão mínima a ser instalada para um pacote:

$ pip install SomePackage>=1.0.4

Se os comandos mostrarem erro de permissão negada no Linux/Unix, use sudo com os comandos

Instale a partir de arquivos de requisitos #

$ pip install -r requirements.txt

Cada linha do arquivo de requisitos indica algo a ser instalado e, como argumentos para instalação do pip, os detalhes sobre o formato dos arquivos estão aqui: Requirements File Format.

Depois de instalar o pacote, você pode verificá-lo usando o comando freeze:

$ pip freeze

Para listar todos os pacotes instalados usando pip #

Para listar os pacotes instalados:

$ pip list
# example output
docutils (0.9.1)
Jinja2 (2.6)
Pygments (1.5)
Sphinx (1.1.2)

Para listar pacotes desatualizados e mostrar a versão mais recente disponível:

$ pip list --outdated
# example output
docutils (Current: 0.9.1 Latest: 0.10)
Sphinx (Current: 1.1.2 Latest: 1.1.3)

Pacotes de atualização #

Corrida

$ pip install --upgrade SomePackage

atualizará o pacote SomePackage e todas as suas dependências. Além disso, o pip remove automaticamente a versão mais antiga do pacote antes da atualização.

Para atualizar o próprio pip, faça

$ pip install --upgrade pip

no Unix ou

$ python -m pip install --upgrade pip

em máquinas Windows.

Desinstalar Pacotes #

Para desinstalar um pacote:

$ pip uninstall SomePackage

Atualizando todos os pacotes desatualizados no Linux #

pip não contém um sinalizador para permitir que um usuário atualize todos os pacotes desatualizados de uma só vez. No entanto, isso pode ser feito conectando comandos em um ambiente Linux:

pip list --outdated --local | grep -v '^\-e' | cut -d = -f 1  | xargs -n1 pip install -U

Este comando pega todos os pacotes no virtualenv local e verifica se eles estão desatualizados. A partir dessa lista, ele obtém o nome do pacote e o canaliza para um comando pip install -U. Ao final deste processo, todos os pacotes locais devem ser atualizados.

Atualizando todos os pacotes desatualizados no Windows #

pip não contém um sinalizador para permitir que um usuário atualize todos os pacotes desatualizados de uma só vez. No entanto, isso pode ser feito canalizando comandos juntos em um ambiente Windows:

for /F "delims= " %i in ('pip list --outdated --local') do pip install -U %i

Este comando pega todos os pacotes no virtualenv local e verifica se eles estão desatualizados. A partir dessa lista, ele obtém o nome do pacote e o canaliza para um comando pip install -U. Ao final deste processo, todos os pacotes locais devem ser atualizados.

Crie um arquivo requirements.txt de todos os pacotes no sistema #

pip auxilia na criação de arquivos requirements.txt fornecendo a opção freeze.

pip freeze > requirements.txt

Isso salvará uma lista de todos os pacotes e suas versões instaladas no sistema em um arquivo chamado requirements.txt na pasta atual.

Crie um arquivo requirements.txt de pacotes apenas no virtualenv atual #

pip auxilia na criação de arquivos requirements.txt fornecendo a opção freeze.

pip freeze --local > requirements.txt

O parâmetro --local produzirá apenas uma lista de pacotes e versões que estão instalados localmente em um virtualenv. Os pacotes globais não serão listados.

Usando uma certa versão do Python com pip #

Se você tiver o Python 3 e o Python 2 instalados, poderá especificar qual versão do Python você gostaria que o pip usasse. Isso é útil quando os pacotes suportam apenas Python 2 ou 3 ou quando você deseja testar com ambos.

Se você deseja instalar pacotes para Python 2, execute:

pip install [package]

ou:

pip2 install [package]

Se você quiser instalar pacotes para Python 3, faça:

pip3 install [package]

Você também pode invocar a instalação de um pacote para uma instalação específica do python com:

\path\to\that\python.exe -m pip install some_package # on Windows OR
/usr/bin/python25 -m pip install some_package # on OS-X/Linux

Em plataformas OS-X/Linux/Unix, é importante estar ciente da distinção entre a versão do sistema do python (que a atualização torna seu sistema inoperante) e a(s) versão(ões) do usuário do python. Você pode, dependendo de qual você está tentando atualizar, precisar prefixar esses comandos com sudo e inserir uma senha.

Da mesma forma, no Windows, algumas instalações python, especialmente aquelas que fazem parte de outro pacote, podem acabar instaladas em diretórios do sistema - aqueles que você terá que atualizar a partir de uma janela de comando em execução no modo Admin - se você achar que precisa fazer isso, é uma muito boa ideia verificar qual instalação python você está tentando atualizar com um comando como python -c"import sys;print(sys.path);" ou py -3.5 - c"import sys;print(sys.path);" você também pode verificar qual pip você está tentando executar com pip --version

No Windows, se você tiver o python 2 e o python 3 instalados, e no seu caminho e seu python 3 for maior que 3.4, você provavelmente também terá o iniciador do python py no caminho do sistema. Você pode então fazer truques como:

py -3 -m pip install -U some_package # Install/Upgrade some_package to the latest python 3
py -3.3 -m pip install -U some_package # Install/Upgrade some_package to python 3.3 if present
py -2 -m pip install -U some_package # Install/Upgrade some_package to the latest python 2 - 64 bit if present
py -2.7-32 -m pip install -U some_package # Install/Upgrade some_package to python 2.7 - 32 bit if present

Se você estiver executando e mantendo várias versões do python, recomendo fortemente ler sobre o python virtualenv ou venv ambientes virtuais que permitem isolar a versão do python e quais pacotes estão presentes.

Instalando pacotes ainda não em pip como rodas #

Muitos pacotes python puros ainda não estão disponíveis no Python Package Index como rodas, mas ainda são instalados bem. No entanto, alguns pacotes no Windows dão o temido erro vcvarsall.bat não encontrado.

O problema é que o pacote que você está tentando instalar contém uma extensão C ou C++ e não está atualmente disponível como uma roda pré-construída do índice de pacotes python, pypi, e no Windows você não tem a ferramenta cadeia necessária para construir tais itens.

A resposta mais simples é ir ao excelente site de Christoph Gohlke e localizar a versão apropriada das bibliotecas que você precisa. Por apropriado no nome do pacote, um -cpNN**-** deve corresponder à sua versão do python, ou seja, se você estiver usando o Windows 32 bits python mesmo no win64 o nome deve inclua -win32- e se estiver usando o python de 64 bits, ele deve incluir -win_amd64- e, em seguida, a versão do python deve corresponder, ou seja, para o Python 34, o nome do arquivo deve incluir *-cp *34-, etc. esta é basicamente a mágica que o pip faz por você no site pypi.

Alternativamente, você precisa obter o kit de desenvolvimento do Windows apropriado para a versão do python que você está usando, os cabeçalhos para qualquer biblioteca para a qual o pacote que você está tentando construir interfaces, possivelmente os cabeçalhos do python para a versão do python, etc.

O Python 2.7 usava o Visual Studio 2008, o Python 3.3 e 3.4 usava o Visual Studio 2010 e o Python 3.5+ usa o Visual Studio 2015.

  • Instale o “Visual C++ Compiler Package for Python 2.7”, que está disponível no site da Microsoft ou
  • Instale o “Windows SDK para Windows 7 e .NET Framework 4” (v7.1), que está disponível no site da Microsoft ou
  • Instale o Visual Studio 2015 Community Edition, (ou qualquer versão posterior, quando estes forem lançados), garantindo que você selecione as opções para instalar o suporte C & C++ não é mais o padrão -* Disseram-me que isso pode levar até* 8 horas para baixar e instalar, portanto, certifique-se de que essas opções estejam definidas na primeira tentativa.

Então você pode precisar localizar os arquivos de cabeçalho, na revisão correspondente para quaisquer bibliotecas às quais o pacote desejado esteja vinculado e fazer o download para os locais apropriados.

Finalmente você pode deixar o pip fazer sua compilação - é claro que se o pacote tiver dependências que você ainda não tem, você também pode precisar encontrar os arquivos de cabeçalho para eles.

Alternativas: Também vale a pena procurar, tanto no pypi quanto no site do Christop, para qualquer versão ligeiramente anterior do pacote que você está procurando que seja python puro ou pré-construído para sua plataforma e versão python e possivelmente usando-os, se encontrados, até que seu pacote esteja disponível. Da mesma forma, se você estiver usando a versão mais recente do python, poderá descobrir que os mantenedores do pacote demoram um pouco para recuperar o atraso, portanto, para projetos que realmente precisam de um pacote específico, talvez seja necessário usar um python um pouco mais antigo para o momento. Você também pode verificar o site de origem dos pacotes para ver se há uma versão bifurcada disponível pré-compilada ou como python puro e procurar pacotes alternativos que forneçam a funcionalidade que você precisa, mas estão disponíveis - um exemplo que vem à mente é o Pillow, mantido ativamente, substituição para PIL atualmente não atualizado em 6 anos e não disponível para python 3.

Posfácio, eu encorajo qualquer pessoa que esteja tendo esse problema a ir ao rastreador de bugs do pacote e adicionar, ou aumentar, se já não houver um, um ticket educadamente solicitando que os mantenedores do pacote forneça uma roda no pypi para sua combinação específica de plataforma e python, se isso for feito, normalmente as coisas melhorarão com o tempo, alguns mantenedores de pacotes não percebem que perderam uma determinada combinação que as pessoas podem estar usando.

Nota sobre a instalação de pré-lançamentos #

Pip segue as regras de Versão Semântica e, por padrão, prefere pacotes lançados a pré-lançamentos. Portanto, se um determinado pacote foi lançado como V0.98 e também há um candidato a lançamento V1.0-rc1, o comportamento padrão de pip install será instalar V0.98 - se você desejar instale o release candidate, é aconselhável testar primeiro em um ambiente virtual, você pode habilitar isso com --pip install --pre package-name ou --pip install --pre --upgrade nome do pacote. Em muitos casos, pré-lançamentos ou candidatos a lançamento podem não ter rodas criadas para todas as combinações de plataforma e versão, portanto, é mais provável que você encontre os problemas acima.

Nota sobre a instalação de versões de desenvolvimento #

Você também pode usar o pip para instalar versões de desenvolvimento de pacotes do github e outros locais, já que esse código está em fluxo, é muito improvável que tenha rodas construídas para ele, portanto, quaisquer pacotes impuros exigirão a presença das ferramentas de compilação, e eles podem ser quebrado a qualquer momento para que o usuário seja fortemente incentivado a instalar esses pacotes apenas em um ambiente virtual.

Existem três opções para tais instalações:

  1. Baixe o instantâneo compactado, a maioria dos sistemas de controle de versão online tem a opção de baixar um instantâneo compactado do código. Isto pode ser baixado manualmente e então instalado com pip install path/to/downloaded/file note que para a maioria dos formatos de compressão o pip irá lidar com a descompactação para uma área de cache, etc.
  2. Deixe o pip lidar com o download e a instalação para você com: pip install URL/of/package/repository - você também pode precisar usar o --trusted-host, --client-cert e /ou --proxy sinalizadores para que isso funcione corretamente, especialmente em um ambiente corporativo. por exemplo:
