Aqui no Stack Overflow muitas vezes vemos duplicatas falando sobre os mesmos erros: "ImportError: No module named '??????', SyntaxError: invalid syntax ou `NameError: name ‘???’ não está definido». Este é um esforço para reduzi-los e ter alguma documentação para vincular.

##TypeErrors Essas exceções são causadas quando o tipo de algum objeto deve ser diferente

TypeError: [definição/método] leva ? argumentos posicionais, mas ? foi dado #

Uma função ou método foi chamado com mais (ou menos) argumentos do que os que ele pode aceitar.

Exemplo #

Se mais argumentos forem fornecidos:

def foo(a): return a
foo(a,b,c,d) #And a,b,c,d are defined

Se forem dados menos argumentos:

def foo(a,b,c,d): return a += b + c + d
foo(a) #And a is defined

Nota: se você quiser usar um número desconhecido de argumentos, você pode usar *args ou **kwargs. Consulte *args e **kwargs

TypeError: tipo(s) de operando não suportado(s) para [operando]: ‘???’ e ‘???’ #

Alguns tipos não podem ser operados juntos, dependendo do operando.

Exemplo #

Por exemplo: + é usado para concatenar e adicionar, mas você não pode usar nenhum deles para ambos os tipos. Por exemplo, tentar fazer um set concatenando (+ing) 'set1' e 'tuple1' dá o erro. Código:

set1, tuple1 = {1,2}, (3,4)
a = set1 + tuple1

Alguns tipos (por exemplo: int e string) usam ambos + mas para coisas diferentes:

b = 400 + 'foo'

Ou eles podem nem ser usados ​​para nada:

c = ["a","b"] - [1,2]

Mas você pode, por exemplo, adicionar um float a um int:

d = 1 + 1.0

TypeError: ‘???’ objeto não é iterável/subscritível: #

Escopos Python e Regra LEGB: #

A chamada Regra LEGB fala sobre os escopos do Python. Seu nome é baseado nos diferentes escopos, ordenados pelas prioridades correspondentes:

Local → Enclosed → Global → Built-in.
  • Local: Variáveis ​​não declaradas globais ou atribuídas em uma função.
  • Enclosing: Variáveis ​​definidas em uma função que é encapsulada dentro de outra função.
  • Global: Variáveis ​​declaradas globais ou atribuídas no nível superior de um arquivo.
  • Built-in: Variáveis ​​pré-atribuídas no módulo de nomes integrados.

Como um exemplo:

for i in range(4):
    d = i * 2
print(d)

d é acessível porque o loop for não marca um novo escopo, mas se o fizesse, teríamos um erro e seu comportamento seria semelhante a:

def noaccess():
    for i in range(4):
        d = i * 2
noaccess()
print(d)

Python diz NameError: name 'd' não está definido

IndentationErrors (ou SyntaxErrors de indentação) #

Na maioria das outras linguagens a indentação não é obrigatória, mas em Python (e outras linguagens: versões anteriores do FORTRAN, Makefiles, Whitespace (linguagem esotérica), etc.) não é o caso, o que pode ser confuso se você vier de outra linguagem, se você estava copiando o código de um exemplo para o seu próprio, ou simplesmente se você é novo.

IndentationError/SyntaxError: recuo inesperado #

Essa exceção é levantada quando o nível de recuo aumenta sem motivo.

Exemplo #

Não há razão para aumentar o nível aqui:

 print "This line is ok"
     print "This line isn't ok"

 print("This line is ok")
     print("This line isn't ok")

Aqui há dois erros: o último e que o recuo não corresponde a nenhum nível de recuo. No entanto, apenas um é mostrado:

 print "This line is ok"
  print "This line isn't ok"

 print("This line is ok")
  print("This line isn't ok")

Outros erros #

AssertError #

A declaração assert existe em quase todas as linguagens de programação. Quando você faz:

assert condition

ou:

assert condition, message

É equivalente a isso:

if __debug__:
    if not condition: raise AssertionError(message)

As asserções podem incluir uma mensagem opcional e você pode desativá-las quando terminar a depuração.

Observação: a variável interna debug é True em circunstâncias normais, False quando a otimização é solicitada (opção de linha de comando -O). As atribuições para debug são ilegais. O valor da variável interna é determinado quando o interpretador é iniciado.

#Interrupção do teclado

Erro gerado quando o usuário pressiona a tecla de interrupção, normalmente Ctrl + C ou del.

ZeroDivisionError #

Você tentou calcular 1/0 que é indefinido. Veja este exemplo para encontrar os divisores de um número:

div = float(raw_input("Divisors of: "))
for x in xrange(div+1): #includes the number itself and zero
    if div/x == div//x:
        print x, "is a divisor of", div

div = int(input("Divisors of: "))
for x in range(div+1): #includes the number itself and zero
    if div/x == div//x:
        print(x, "is a divisor of", div)

Ele gera ZeroDivisionError porque o loop for atribui esse valor a x. Em vez disso, deve ser:

div = float(raw_input("Divisors of: "))
for x in xrange(1,div+1): #includes the number itself but not zero
    if div/x == div//x:
        print x, "is a divisor of", div

div = int(input("Divisors of: "))
for x in range(1,div+1): #includes the number itself but not zero
    if div/x == div//x:
        print(x, "is a divisor of", div)

O motivo mais comum para esse problema são parênteses/colchetes incompatíveis, como mostra o exemplo.
