Como uma das funções mais básicas na programação, os loops são uma peça importante para quase todas as linguagens de programação. Os loops permitem que os desenvolvedores definam certas partes de seu código para serem repetidas por meio de vários loops que são chamados de iterações. Este tópico abrange o uso de vários tipos de loops e aplicações de loops em Python.

Sintaxe #

  • while <expressão booleana>:
  • para <variável> em <iterável>:
  • para <variável> no intervalo(<número>):
  • para <variável> no intervalo(<start_number>, <end_number>):
  • para <variable> no intervalo(<start_number>, <end_number>, <step_size>):
  • for i, <variable> in enumerate(<iterable>): # com índice i
  • para <variable1>, <variable2> em zip(<iterable1>, <iterable2>):

Parâmetros #

Parâmetro Detalhes
expressão booleana expressão que pode ser avaliada em um contexto booleano, por exemplo. x < 10
variável nome da variável para o elemento atual do iterable
iterável qualquer coisa que implemente iterações

Quebra e continua em loops #

instrução break #

Quando uma instrução break é executada dentro de um loop, o fluxo de controle “quebra” do loop imediatamente:

i = 0
while i < 7:
    print(i)
    if i == 4:
        print("Breaking from loop")
        break
    i += 1

A condicional do loop não será avaliada após a execução da instrução break. Note que instruções break são permitidas apenas dentro de loops, sintaticamente. Uma instrução break dentro de uma função não pode ser usada para encerrar loops que chamaram essa função.

A execução do seguinte imprime cada dígito até o número 4 quando a instrução break é atendida e o loop para:

0
1
2
3
4
Breaking from loop

As instruções break também podem ser usadas dentro de loops for, a outra construção de loop fornecida pelo Python:

for i in (0, 1, 2, 3, 4):
    print(i)
    if i == 2:
        break

Executando este loop agora imprime:

0
1
2

Observe que 3 e 4 não são impressos porque o loop terminou.

Se um loop tiver uma cláusula else, ele não será executado quando o loop for finalizado por meio de uma instrução break.

instrução continuar #

Uma instrução continue pulará para a próxima iteração do loop ignorando o restante do bloco atual, mas continuando o loop. Assim como break, continue só pode aparecer dentro de loops:

for i in (0, 1, 2, 3, 4, 5):
    if i == 2 or i == 4:
        continue
    print(i)

0
1
3
5

Observe que 2 e 4 não são impressos, isso ocorre porque continue vai para a próxima iteração em vez de continuar para print(i) quando i == 2 ou i == 4 .

Loops aninhados #

break e continue operam apenas em um único nível de loop. O exemplo a seguir só sairá do loop interno for, não do loop externo while:

while True:
    for i in range(1,5):
        if i == 2:
            break    # Will only break out of the inner loop!

O Python não tem a capacidade de quebrar vários níveis de loop de uma só vez - se esse comportamento for desejado, refatorar um ou mais loops em uma função e substituir break por return pode ser o caminho a seguir.

Use return de dentro de uma função como break #

A [instrução return](https://www.wikiod.com/pt/python/funcoes#Retornando valores de funções) sai de uma função, sem executar o código que vem depois dela.

Se você tem um loop dentro de uma função, usar return de dentro desse loop é equivalente a ter um break já que o resto do código do loop não é executado (note que qualquer código após o loop também não é executado) :

def break_loop():
    for i in range(1, 5):
        if (i == 2):
            return(i)
        print(i)
    return(5)

Se você tiver loops aninhados, a instrução return interromperá todos os loops:

def break_all():
    for j in range(1, 5):
        for i in range(1,4):
            if i*j == 6:
                return(i)
            print(i*j)

emitirá:

1 # 1*1
2 # 1*2
3 # 1*3
4 # 1*4
2 # 2*1
4 # 2*2
# return because 2*3 = 6, the remaining iterations of both loops are not executed

1: https://www.wikiod.com/pt/python/rotacoes#Loops com uma cláusula “else”

Para laços #

Os loops for iteram sobre uma coleção de itens, como list ou dict, e executam um bloco de código com cada elemento da coleção.

for i in [0, 1, 2, 3, 4]:
    print(i)

O loop for acima itera sobre uma lista de números.

Cada iteração define o valor de i para o próximo elemento da lista. Então, primeiro será 0, depois 1, depois 2, etc. A saída será a seguinte:

0  
1
2
3
4

range é uma função que retorna uma série de números em uma forma iterável, portanto, pode ser usada em loops for:

for i in range(5):
    print(i)

dá exatamente o mesmo resultado que o primeiro loop for. Observe que 5 não é impresso, pois o intervalo aqui são os primeiros cinco números contados a partir de 0.

Objetos e iteradores iteráveis #

O loop for pode iterar em qualquer objeto iterável que é um objeto que define uma função __getitem__ ou __iter__. A função __iter__ retorna um iterador, que é um objeto com uma função next que é usada para acessar o próximo elemento do iterável.

Iterando sobre listas #

Para iterar através de uma lista você pode usar for:

for x in ['one', 'two', 'three', 'four']:
    print(x)

Isto irá imprimir os elementos da lista:

one
two
three
four

A função range gera números que também são frequentemente usados ​​em um loop for.

for x in range(1, 6):
    print(x)

O resultado será um tipo de sequência de intervalo especial em python >=3 e uma lista em python <=2. Ambos podem ser repetidos usando o loop for.

1
2
3
4
5

Se você quiser percorrer ambos os elementos de uma lista e ter um índice para os elementos também, você pode usar a função enumerate do Python:

for index, item in enumerate(['one', 'two', 'three', 'four']):
    print(index, '::', item)

enumerate irá gerar tuplas, que são descompactadas em index (um inteiro) e item (o valor real da lista). O loop acima irá imprimir

(0, '::', 'one')
(1, '::', 'two')
(2, '::', 'three')
(3, '::', 'four')

Itere sobre uma lista com manipulação de valor usando map e lambda, ou seja, aplique a função lambda em cada elemento da lista:

x = map(lambda e :  e.upper(), ['one', 'two', 'three', 'four'])
print(x)

Resultado:

['ONE', 'TWO', 'THREE', 'FOUR'] # Python 2.x

NB: no Python 3.x, map retorna um iterador em vez de uma lista, portanto, caso precise de uma lista, você deve lançar o resultado print(list(x)) (consulte http://stackoverflow.com/ documentation/python/809/incompatibilities-between-python-2-and-python-3/8186/map) em https://www.wikiod.com/pt/python/incompatibilidades-migrando-do-python-2-para-o-python-3 -3).

Loops com uma cláusula “else” #

As instruções compostas for e while (loops) podem opcionalmente ter uma cláusula else (na prática, esse uso é bastante raro).

A cláusula else só é executada depois que um loop for termina por iteração até a conclusão, ou depois que um loop while termina quando sua expressão condicional se torna falsa.

for i in range(3):
    print(i)
else:
    print('done')

i = 0
while i < 3:
    print(i)
    i += 1
else:
    print('done')

resultado:

 0
 1
 2
 done

A cláusula else não executa se o loop terminar de alguma outra forma (através de uma instrução break ou levantando uma exceção):

for i in range(2):
    print(i)
    if i == 1:
        break
else:
    print('done')

resultado:

0
1

A maioria das outras linguagens de programação não possui essa cláusula opcional de loops else. O uso da palavra-chave else em particular é muitas vezes considerado confuso.

O conceito original para tal cláusula remonta a Donald Knuth e o significado da palavra-chave else fica claro se reescrevermos um loop em termos de instruções if e goto instruções de dias anteriores antes da programação estruturada ou de uma linguagem assembly de nível inferior.

Por exemplo:

while loop_condition():
    ...
    if break_condition():
        break
    ...

é equivalente a:

# pseudocode

<<start>>:
if loop_condition():
    ...
    if break_condition():
        goto <<end>>
    ...
    goto <<start>>

<<end>>:

Elas permanecem equivalentes se anexarmos uma cláusula else a cada uma delas.

Por exemplo:

while loop_condition():
    ...
    if break_condition():
        break
    ...
else:
    print('done')

é equivalente a:

# pseudocode

<<start>>:
if loop_condition():
    ...
    if break_condition():
        goto <<end>>
    ...
    goto <<start>>
else:
    print('done')

<<end>>:

Um loop for com uma cláusula else pode ser entendido da mesma maneira. Conceitualmente, há uma condição de loop que permanece True enquanto o objeto ou sequência iterável ainda tiver alguns elementos restantes.

Por que alguém usaria essa construção estranha? #

O principal caso de uso para a construção for...else é uma implementação concisa de pesquisa como por exemplo:

a = [1, 2, 3, 4]
for i in a:
    if type(i) is not int:
        print(i)
        break
else:
    print("no exception")

Para tornar o else nesta construção menos confuso, pode-se pensar nele como “if not break” ou “if not found”.

Algumas discussões sobre isso podem ser encontradas em [Python-ideas] Summary of for…else threads, Por que o python usa ’else’ após os loops for e while? e [Else Clauses on Loop Statements](http://python- notes.curiousefficiency.org/en/latest/python_concepts/break_else.html)

Iterando sobre dicionários #

Considerando o seguinte dicionário:

 d = {"a": 1, "b": 2, "c": 3}

Para iterar através de suas chaves, você pode usar:

for key in d:
    print(key)

Resultado:

"a"
"b"
"c"

Isso é equivalente a:

for key in d.keys():
    print(key)

ou em Python 2:

for key in d.iterkeys():
    print(key)

Para iterar através de seus valores, use:

for value in d.values():
    print(value)

Resultado:

1
2
3

Para iterar por meio de suas chaves e valores, use:

for key, value in d.items():
    print(key, "::", value)

Resultado:

a :: 1
b :: 2
c :: 3

Observe que em Python 2, .keys(), .values() e .items() retornam um objeto list. Se você simplesmente precisar iterar pelo resultado, você pode usar os equivalentes .iterkeys(), .itervalues() e .iteritems().

A diferença entre .keys() e .iterkeys(), .values() e .itervalues(), .items() e .iteritems() é que o iter* métodos são geradores. Assim, os elementos dentro do dicionário são gerados um a um à medida que são avaliados. Quando um objeto list é retornado, todos os elementos são empacotados em uma lista e então retornados para avaliação posterior.

Observe também que no Python 3, a ordem dos itens impressos da maneira acima não segue nenhuma ordem.

A Declaração de Passe #

pass é uma instrução nula para quando uma instrução é exigida pela sintaxe do Python (como dentro do corpo de um loop for ou while), mas nenhuma ação é exigida ou desejada pelo programador. Isso pode ser útil como um espaço reservado para código que ainda não foi escrito.

for x in range(10):
    pass #we don't want to do anything, or are not ready to do anything here, so we'll pass

Neste exemplo, nada acontecerá. O loop for será concluído sem erros, mas nenhum comando ou código será acionado. pass nos permite executar nosso código com sucesso sem ter todos os comandos e ações totalmente implementados.

Da mesma forma, pass pode ser usado em loops while, bem como em seleções e definições de funções, etc.

while x == y:
    pass

Enquanto Loop #

Um loop while fará com que as instruções do loop sejam executadas até que a condição do loop seja falsey. O código a seguir executará as instruções de loop um total de 4 vezes.

i = 0 
while i < 4:
    #loop statements
    i = i + 1

Enquanto o loop acima pode ser facilmente traduzido em um loop for mais elegante, os loops while são úteis para verificar se alguma condição foi atendida. O loop a seguir continuará a ser executado até que myObject esteja pronto.

myObject = anObject()
while myObject.isNotReady():
    myObject.tryToGetReady()

Os loops while também podem ser executados sem uma condição usando números (complexos ou reais) ou True:

import cmath

complex_num = cmath.sqrt(-1)
while complex_num:      # You can also replace complex_num with any number, True or a value of any type
    print(complex_num)   # Prints 1j forever

Se a condição for sempre verdadeira, o loop while será executado para sempre (loop infinito) se não for finalizado por uma instrução break ou return ou uma exceção.

while True:
    print "Infinite loop"
# Infinite loop
# Infinite loop
# Infinite loop
# ...

Iterando partes diferentes de uma lista com tamanho de etapa diferente #

Suponha que você tenha uma longa lista de elementos e esteja interessado apenas em todos os outros elementos da lista. Talvez você queira examinar apenas o primeiro ou o último elemento, ou um intervalo específico de entradas em sua lista. O Python possui fortes recursos internos de indexação. Aqui estão alguns exemplos de como alcançar esses cenários.

Aqui está uma lista simples que será usada ao longo dos exemplos:

lst = ['alpha', 'bravo', 'charlie', 'delta', 'echo']

Iteração em toda a lista #

Para iterar sobre cada elemento na lista, um loop for como abaixo pode ser usado:

for s in lst:
    print s[:1] # print the first letter

O loop for atribui s para cada elemento de lst. Isso imprimirá:

a
b
c
d
e

Muitas vezes você precisa tanto do elemento quanto do índice desse elemento. A palavra-chave enumerate executa essa tarefa.

for idx, s in enumerate(lst):
    print("%s has an index of %d" % (s, idx))

O índice idx começará com zero e será incrementado a cada iteração, enquanto o s conterá o elemento que está sendo processado. O snippet anterior produzirá:

alpha has an index of 0
bravo has an index of 1
charlie has an index of 2
delta has an index of 3
echo has an index of 4

Iterar sobre a sublista #

Se quisermos iterar em um intervalo (lembrando que Python usa indexação baseada em zero), use a palavra-chave range.

for i in range(2,4):
    print("lst at %d contains %s" % (i, lst[i]))

Isso geraria:

lst at 2 contains charlie
lst at 3 contains delta

A lista também pode ser dividida. A seguinte notação de fatia vai do elemento no índice 1 até o final com um passo de 2. Os dois loops for dão o mesmo resultado.

for s in lst[1::2]:
    print(s)

for i in range(1, len(lst), 2):
    print(lst[i])

O snippet acima gera:

bravo
delta

Indexação e fatiamento é um tópico próprio.

O “meio loop” do-while #

Ao contrário de outras linguagens, o Python não possui uma construção do-until ou do-while (isso permitirá que o código seja executado uma vez antes que a condição seja testada). No entanto, você pode combinar um while True com um [break](https://www.wikiod.com/pt/python/rotacoes#Quebre e continue em loops) para obter o mesmo propósito.

a = 10
while True:
    a = a-1
    print(a)
    if a<7:
        break
print('Done.')

Isso imprimirá:

9
8
7
6
Done.

Looping e descompactação #

Se você quiser fazer um loop em uma lista de tuplas, por exemplo:

collection = [('a', 'b', 'c'), ('x', 'y', 'z'), ('1', '2', '3')]

ao invés de fazer algo assim:

for item in collection:
    i1 = item[0]
    i2 = item[1]
    i3 = item[2]
    # logic

ou algo parecido com isto:

for item in collection:
    i1, i2, i3 = item
    # logic

Você pode simplesmente fazer isso:

for i1, i2, i3 in collection:
    # logic

Isso também funcionará para a maioria dos tipos de iteráveis, não apenas para tuplas.
