O pacote interno collections fornece vários tipos de coleção especializados e flexíveis que são de alto desempenho e fornecem alternativas para os tipos gerais de coleção de dict, list, tuple e set. O módulo também define classes base abstratas que descrevem diferentes tipos de funcionalidade de coleção (como MutableSet e ItemsView).

Existem três outros tipos disponíveis no módulo collections, a saber:

  1. UserDict
  2. Lista de usuários
  3. UserString

Cada um deles atua como um wrapper em torno do objeto vinculado, por exemplo, UserDict atua como um wrapper em torno de um objeto dict. Em cada caso, a classe simula seu tipo nomeado. O conteúdo da instância é mantido em um objeto de tipo regular, que pode ser acessado por meio do atributo data da instância do wrapper. Em cada um desses três casos, a necessidade desses tipos foi parcialmente suplantada pela capacidade de subclassificar diretamente do tipo básico; no entanto, a classe wrapper pode ser mais fácil de trabalhar porque o tipo subjacente é acessível como um atributo.

coleções.Contador #

Counter é uma subclasse dict que permite contar objetos facilmente. Possui métodos utilitários para trabalhar com as frequências dos objetos que você está contando.

import collections
counts = collections.Counter([1,2,3])

o código acima cria um objeto, counts, que possui as frequências de todos os elementos passados ​​para o construtor. Este exemplo tem o valor Counter({1: 1, 2: 1, 3: 1})

Exemplos de construtor

Contador de cartas

>>> collections.Counter('Happy Birthday')
Counter({'a': 2, 'p': 2, 'y': 2, 'i': 1, 'r': 1, 'B': 1, ' ': 1, 'H': 1, 'd': 1, 'h': 1, 't': 1})

Contador de palavras

>>> collections.Counter('I am Sam Sam I am That Sam-I-am That Sam-I-am! I do not like that Sam-I-am'.split())
Counter({'I': 3, 'Sam': 2, 'Sam-I-am': 2, 'That': 2, 'am': 2, 'do': 1, 'Sam-I-am!': 1, 'that': 1, 'not': 1, 'like': 1})

Receitas

>>> c = collections.Counter({'a': 4, 'b': 2, 'c': -2, 'd': 0})

Obter contagem de elemento individual

>>> c['a']
4

Definir contagem de elemento individual

>>> c['c'] = -3
>>> c
Counter({'a': 4, 'b': 2, 'd': 0, 'c': -3})

Obtenha o número total de elementos no contador (4 + 2 + 0 - 3)

>>> sum(c.itervalues())  # negative numbers are counted!
3

Obter elementos (somente aqueles com contador positivo são mantidos)

>>> list(c.elements())
['a', 'a', 'a', 'a', 'b', 'b']

Remova as chaves com 0 ou valor negativo

>>> c - collections.Counter()
Counter({'a': 4, 'b': 2})

Remover tudo

>>> c.clear()
>>> c
Counter()

Adicionar remover elementos individuais

>>> c.update({'a': 3, 'b':3})
>>> c.update({'a': 2, 'c':2})  # adds to existing, sets if they don't exist
>>> c
Counter({'a': 5, 'b': 3, 'c': 2})
>>> c.subtract({'a': 3, 'b': 3, 'c': 3})  # subtracts (negative values are allowed)
>>> c
Counter({'a': 2, 'b': 0, 'c': -1})

coleções.defaultdict #

collections.defaultdict(default_factory) retorna uma subclasse de dict que tem um valor padrão para chaves ausentes. O argumento deve ser uma função que retorna o valor padrão quando chamada sem argumentos. Se não houver nada passado, o padrão é Nenhum.

>>> state_capitals = collections.defaultdict(str)
>>> state_capitals
defaultdict(<class 'str'>, {})

retorna uma referência a um defaultdict que criará um objeto string com seu método default_factory.

Um uso típico de defaultdict é usar um dos tipos internos como str, int, list ou dict como default_factory, já que estes retornam tipos vazios quando chamados sem argumentos:

>>> str()
''
>>> int()
0
>>> list
[]

Chamar o defaultdict com uma chave que não existe não produz um erro como em um dicionário normal.

>>> state_capitals['Alaska']
''
>>> state_capitals
defaultdict(<class 'str'>, {'Alaska': ''})

Outro exemplo com int:

>>> fruit_counts = defaultdict(int)
>>> fruit_counts['apple'] += 2  # No errors should occur
>>> fruit_counts
default_dict(int, {'apple': 2})
>>> fruit_counts['banana']  # No errors should occur
0
>>> fruit_counts  # A new key is created
default_dict(int, {'apple': 2, 'banana': 0})

Os métodos de dicionário normal funcionam com o dicionário padrão

>>> state_capitals['Alabama'] = 'Montgomery'
>>> state_capitals
defaultdict(<class 'str'>, {'Alabama': 'Montgomery', 'Alaska': ''})

Usar list como default_factory criará uma lista para cada nova chave.

>>> s = [('NC', 'Raleigh'), ('VA', 'Richmond'), ('WA', 'Seattle'), ('NC', 'Asheville')]
>>> dd = collections.defaultdict(list)
>>> for k, v in s:
...     dd[k].append(v)
>>> dd
defaultdict(<class 'list'>, 
    {'VA': ['Richmond'], 
     'NC': ['Raleigh', 'Asheville'], 
     'WA': ['Seattle']})

coleções.OrdenadoDict #

A ordem das chaves nos dicionários Python é arbitrária: elas não são regidas pela ordem em que você as adiciona.

Por exemplo:

>>> d = {'foo': 5, 'bar': 6}
>>> print(d)
{'foo': 5, 'bar': 6}
>>> d['baz'] = 7
>>> print(a)
{'baz': 7, 'foo': 5, 'bar': 6}
>>> d['foobar'] = 8
>>> print(a)
{'baz': 7, 'foo': 5, 'bar': 6, 'foobar': 8}
```

(A ordenação arbitrária implícita acima significa que você pode obter resultados diferentes com o código acima daquele mostrado aqui.)

A ordem em que as chaves aparecem é a ordem em que elas seriam iteradas, por exemplo usando um loop for.

A classe collections.OrderedDict fornece objetos de dicionário que mantêm a ordem das chaves. OrderedDicts podem ser criados como mostrado abaixo com uma série de itens ordenados (aqui, uma lista de pares de valores-chave de tupla):

>>> from collections import OrderedDict
>>> d = OrderedDict([('foo', 5), ('bar', 6)])
>>> print(d)
OrderedDict([('foo', 5), ('bar', 6)])
>>> d['baz'] = 7
>>> print(d)
OrderedDict([('foo', 5), ('bar', 6), ('baz', 7)])
>>> d['foobar'] = 8
>>> print(d)
OrderedDict([('foo', 5), ('bar', 6), ('baz', 7), ('foobar', 8)])

Ou podemos criar um OrderedDict vazio e adicionar itens:

>>> o = OrderedDict()
>>> o['key1'] = "value1"
>>> o['key2'] = "value2"
>>> print(o)
OrderedDict([('key1', 'value1'), ('key2', 'value2')])

A iteração por meio de um OrderedDict permite o acesso às chaves na ordem em que foram adicionadas.

O que acontece se atribuirmos um novo valor a uma chave existente?

>>> d['foo'] = 4
>>> print(d)
OrderedDict([('foo', 4), ('bar', 6), ('baz', 7), ('foobar', 8)])

A chave mantém seu lugar original no OrderedDict.

collections.namedtuple #

Defina um novo tipo Person usando namedtuple assim:

Person = namedtuple('Person', ['age', 'height', 'name'])

O segundo argumento é a lista de atributos que a tupla terá. Você pode listar esses atributos também como string separada por espaço ou vírgula:

Person = namedtuple('Person', 'age, height, name')

ou

Person = namedtuple('Person', 'age height name')

Uma vez definida, uma tupla nomeada pode ser instanciada chamando o objeto com os parâmetros necessários, por exemplo:

dave = Person(30, 178, 'Dave')

Os argumentos nomeados também podem ser usados:

jack = Person(age=30, height=178, name='Jack S.')

Agora você pode acessar os atributos da namedtuple:

print(jack.age)  # 30
print(jack.name)  # 'Jack S.'

O primeiro argumento para o construtor namedtuple (em nosso exemplo 'Person') é o typename. É típico usar a mesma palavra para o construtor e o nome do tipo, mas eles podem ser diferentes:

Human = namedtuple('Person',  'age, height, name')
dave = Human(30, 178, 'Dave')
print(dave)  # yields: Person(age=30, height=178, name='Dave')

coleções.deque #

Retorna um novo objeto deque inicializado da esquerda para a direita (usando append()) com dados de iterable. Se iterable não for especificado, o novo deque estará vazio.

Deques são uma generalização de pilhas e filas (o nome é pronunciado “deck” e é a abreviação de “fila dupla”). Deques suportam anexos e pops seguros para threads e eficientes em memória de ambos os lados do deque com aproximadamente o mesmo desempenho O(1) em qualquer direção.

Embora os objetos de lista suportem operações semelhantes, eles são otimizados para operações rápidas de comprimento fixo e incorrem em custos de movimentação de memória O(n) para operações pop(0) e insert(0, v) que alteram o tamanho e a posição da representação de dados subjacente .

Novo na versão 2.4.

Se maxlen não for especificado ou for None, deques pode crescer até um tamanho arbitrário. Caso contrário, o deque é limitado ao comprimento máximo especificado. Uma vez que um deque de comprimento limitado está cheio, quando novos itens são adicionados, um número correspondente de itens é descartado da extremidade oposta. Deques de comprimento limitado fornecem funcionalidade semelhante ao filtro de cauda no Unix. Eles também são úteis para rastrear transações e outros conjuntos de dados em que apenas a atividade mais recente é de interesse.

Alterado na versão 2.6: Adicionado parâmetro maxlen.

>>> from collections import deque
>>> d = deque('ghi')                 # make a new deque with three items
>>> for elem in d:                   # iterate over the deque's elements
...     print elem.upper()
G
H
I

>>> d.append('j')                    # add a new entry to the right side
>>> d.appendleft('f')                # add a new entry to the left side
>>> d                                # show the representation of the deque
deque(['f', 'g', 'h', 'i', 'j'])

>>> d.pop()                          # return and remove the rightmost item
'j'
>>> d.popleft()                      # return and remove the leftmost item
'f'
>>> list(d)                          # list the contents of the deque
['g', 'h', 'i']
>>> d[0]                             # peek at leftmost item
'g'
>>> d[-1]                            # peek at rightmost item
'i'

>>> list(reversed(d))                # list the contents of a deque in reverse
['i', 'h', 'g']
>>> 'h' in d                         # search the deque
True
>>> d.extend('jkl')                  # add multiple elements at once
>>> d
deque(['g', 'h', 'i', 'j', 'k', 'l'])
>>> d.rotate(1)                      # right rotation
>>> d
deque(['l', 'g', 'h', 'i', 'j', 'k'])
>>> d.rotate(-1)                     # left rotation
>>> d
deque(['g', 'h', 'i', 'j', 'k', 'l'])

>>> deque(reversed(d))               # make a new deque in reverse order
deque(['l', 'k', 'j', 'i', 'h', 'g'])
>>> d.clear()                        # empty the deque
>>> d.pop()                          # cannot pop from an empty deque
Traceback (most recent call last):
  File "<pyshell#6>", line 1, in -toplevel-
    d.pop()
IndexError: pop from an empty deque

>>> d.extendleft('abc')              # extendleft() reverses the input order
>>> d
deque(['c', 'b', 'a'])

coleções.Mapa da Cadeia #

ChainMap é novo na versão 3.3

Retorna um novo objeto ChainMap dado um número de maps. Este objeto agrupa vários dicts ou outros mapeamentos para criar uma visão única e atualizável.

ChainMaps são úteis para gerenciar contextos e sobreposições aninhados. Um exemplo no mundo python é encontrado na implementação da classe Context no template engine do Django. É útil para vincular rapidamente vários mapeamentos para que o resultado possa ser tratado como uma única unidade. Geralmente é muito mais rápido do que criar um novo dicionário e executar várias chamadas update().

Sempre que alguém tem uma cadeia de valores de pesquisa, pode haver um caso para ChainMap. Um exemplo inclui ter valores especificados pelo usuário e um dicionário de valores padrão. Outro exemplo são os mapas de parâmetros POST e GET encontrados no uso da web, por exemplo. Django ou Flask. Através do uso de ChainMap retorna-se uma visão combinada de dois dicionários distintos.

A lista de parâmetros maps é ordenada da primeira à última pesquisada. As pesquisas pesquisam os mapeamentos subjacentes sucessivamente até que uma chave seja encontrada. Por outro lado, gravações, atualizações e exclusões operam apenas no primeiro mapeamento.

import collections

# define two dictionaries with at least some keys overlapping.
dict1 = {'apple': 1, 'banana': 2}
dict2 = {'coconut': 1, 'date': 1, 'apple': 3}

# create two ChainMaps with different ordering of those dicts.
combined_dict = collections.ChainMap(dict1, dict2)
reverse_ordered_dict = collections.ChainMap(dict2, dict1)

Observe o impacto da ordem em que o valor é encontrado primeiro na pesquisa subsequente

for k, v in combined_dict.items():
    print(k, v)
    
date 1
apple 1
banana 2
coconut 1

for k, v in reverse_ordered_dict.items():
    print(k, v)

date 1
apple 3
banana 2
coconut 1
