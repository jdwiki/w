Sobrecarga do operador #

Abaixo estão os operadores que podem ser sobrecarregados em classes, juntamente com as definições de método necessárias e um exemplo do operador em uso em uma expressão.

N.B. O uso de other como nome de variável não é obrigatório, mas é considerado a norma.

Operador Método Expressão
Adição + __add__(self, other) a1 + a2
- Subtração __sub__(self, other) a1 - a2
* Multiplicação __mul__(self, other) a1 * a2
@ Multiplicação de Matrizes __matmul__(self, other) a1 @ a2 (Python 3.5)
/ Divisão __div__(self, other) a1 / a2 (Somente Python 2)
/ Divisão __truediv__(self, other) a1 / a2 (Python 3)
// Divisão de Piso __floordiv__(self, other) a1 // a2
% Módulo/Restante __mod__(self, other) a1 % a2
** Potência __pow__(self, other[, módulo]) a1 ** a2
<< Deslocamento à esquerda bit a bit __lshift__(self, other) a1 << a2
>> Deslocamento à direita bit a bit __rshift__(self, other) a1 >> a2
& E bit a bit __and__(self, other) a1 & a2
^ XOR bit a bit __xor__(auto, outro) a1 ^ a2
| (OR bit a bit) __or__(self, other) a1 | a2
- Negação (Aritmética) __neg__(self) -a1
+ Positivo __pos__(self) +a1
~ bit a bit NÃO __inverter__(auto) ~a1
< Menor que __lt__(self, other) a1 < a2
<= Menor ou igual a __le__(self, other) a1 <= a2
== Igual a __eq__(self, other) a1 == a2
!= Diferente de __ne__(self, other) a1 != a2
> Maior que __gt__(self, other) a1 > a2
>= Maior ou igual a __ge__(self, other) a1 >= a2
[index] Operador de índice __getitem__(self, index) a1[índice]
in Em operador __contains__(self, other) a2 em a1
(*args, ...) Chamando __call__(self, *args, **kwargs) a1(*args, **kwargs)

O parâmetro opcional modulo para __pow__ é usado apenas pela função interna pow.

Cada um dos métodos correspondentes a um operador binário tem um método “certo” correspondente que começa com __r, por exemplo __radd__:

class A:
    def __init__(self, a):
        self.a = a
    def __add__(self, other):
        return self.a + other
    def __radd__(self, other):
        print("radd")
        return other + self.a

A(1) + 2  # Out:  3
2 + A(1)  # prints radd. Out: 3

bem como uma versão inplace correspondente, começando com __i:

class B:
    def __init__(self, b):
        self.b = b
    def __iadd__(self, other):
        self.b += other
        print("iadd")
        return self

b = B(2)
b.b       # Out: 2
b += 1    # prints iadd
b.b       # Out: 3

Como não há nada de especial nesses métodos, muitas outras partes da linguagem, partes da biblioteca padrão e até módulos de terceiros adicionam métodos mágicos por conta própria, como métodos para converter um objeto em um tipo ou verificar as propriedades do objeto. Por exemplo, a função interna str() chama o método __str__ do objeto, se existir. Alguns desses usos estão listados abaixo.

Função Método Expressão

Existem também os métodos especiais __enter__ e __exit__ para gerenciadores de contexto e muitos mais.

Métodos Mágicos/Dunder #

Os métodos mágicos (também chamados de dunder como abreviação de double-underscore) em Python servem a um propósito semelhante à sobrecarga de operadores em outras linguagens. Eles permitem que uma classe defina seu comportamento quando usada como operando em expressões de operadores unários ou binários. Eles também servem como implementações chamadas por algumas funções internas.

Considere esta implementação de vetores bidimensionais.

import math

class Vector(object):
    # instantiation
    def __init__(self, x, y):
        self.x = x
        self.y = y

    # unary negation (-v)
    def __neg__(self):
        return Vector(-self.x, -self.y)

    # addition (v + u)
    def __add__(self, other):
        return Vector(self.x + other.x, self.y + other.y)

    # subtraction (v - u)
    def __sub__(self, other):
        return self + (-other)

    # equality (v == u)
    def __eq__(self, other):
        return self.x == other.x and self.y == other.y

    # abs(v)
    def __abs__(self):
        return math.hypot(self.x, self.y)

    # str(v)
    def __str__(self):
        return '<{0.x}, {0.y}>'.format(self)

    # repr(v)
    def __repr__(self):
        return 'Vector({0.x}, {0.y})'.format(self)

Agora é possível usar naturalmente instâncias da classe Vector em várias expressões.

v = Vector(1, 4)
u = Vector(2, 0)

u + v           # Vector(3, 4)
print(u + v)    # "<3, 4>" (implicit string conversion)
u - v           # Vector(1, -4)
u == v          # False
u + v == v + u  # True
abs(u + v)      # 5.0

Lidando com comportamento não implementado #

Se sua classe não implementa um operador sobrecarregado específico para os tipos de argumento fornecidos, ela deve retornar NotImplemented (observe que esta é uma constante especial, não o mesmo que NotImplementedError). Isso permitirá que o Python volte a tentar outros métodos para fazer a operação funcionar:

Quando NotImplemented for retornado, o interpretador tentará então operação refletida no outro tipo, ou algum outro fallback, dependendo do operador. Se todas as tentativas de operações retornarem NotImplemented, o interpretador levantará uma exceção apropriada.

Por exemplo, dado x + y, se x.__add__(y) retornar não implementado, será tentado y.__radd__(x).

class NotAddable(object):

    def __init__(self, value):
        self.value = value

    def __add__(self, other):
        return NotImplemented


class Addable(NotAddable):

    def __add__(self, other):
        return Addable(self.value + other.value)

    __radd__ = __add__

Como este é o método refletido temos que implementar __add__ e __radd__ para obter o comportamento esperado em todos os casos; felizmente, como ambos estão fazendo a mesma coisa neste exemplo simples, podemos pegar um atalho.

Em uso:

>>> x = NotAddable(1)
>>> y = Addable(2)
>>> x + x
Traceback (most recent call last):
  File "<stdin>", line 1, in <module>
TypeError: unsupported operand type(s) for +: 'NotAddable' and 'NotAddable'
>>> y + y
<so.Addable object at 0x1095974d0>
>>> z = x + y
>>> z
<so.Addable object at 0x109597510>
>>> z.value
3
