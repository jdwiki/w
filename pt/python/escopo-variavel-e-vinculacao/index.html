Sintaxe #

  • global a, b, c
  • não local a, b
  • x = algo # liga x
  • (x, y) = algo # liga x e y
  • x += algo # liga x. Da mesma forma para todos os outros “op=”
  • del x # liga x
  • para x em algo: # liga x
  • com algo como x: # liga x
  • exceto Exception as ex: # vincula ex dentro do bloco

Variáveis ​​não locais #

O Python 3 adicionou uma nova palavra-chave chamada nonlocal. A palavra-chave não local adiciona uma substituição de escopo ao escopo interno. Você pode ler tudo sobre isso em PEP 3104. Isso é melhor ilustrado com alguns exemplos de código. Um dos exemplos mais comuns é criar uma função que possa incrementar:

def counter():
    num = 0
    def incrementer():
        num += 1
        return num
    return incrementer

Se você tentar executar esse código, receberá um UnboundLocalError porque a variável num é referenciada antes de ser atribuída na função mais interna. Vamos adicionar não-local à mistura:

def counter():
    num = 0
    def incrementer():
        nonlocal num
        num += 1
        return num
    return incrementer

c = counter()
c() # = 1
c() # = 2
c() # = 3

Basicamente nonlocal permitirá que você atribua variáveis ​​em um escopo externo, mas não em um escopo global. Então você não pode usar nonlocal em nossa função counter porque então ela tentaria atribuir a um escopo global. Experimente e você obterá rapidamente um SyntaxError. Em vez disso, você deve usar nonlocal em uma função aninhada.

(Observe que a funcionalidade apresentada aqui é melhor implementada usando geradores.)

Variáveis ​​globais #

Em Python, variáveis ​​dentro de funções são consideradas locais se e somente se elas aparecem no lado esquerdo de uma instrução de atribuição, ou alguma outra ocorrência de ligação; caso contrário, tal vinculação é procurada em funções delimitadoras, até o escopo global. Isso é verdade mesmo se a instrução de atribuição nunca for executada.

x = 'Hi'

def read_x():
    print(x)   # x is just referenced, therefore assumed global

read_x()       # prints Hi

def read_y():
    print(y)   # here y is just referenced, therefore assumed global

read_y()       # NameError: global name 'y' is not defined

def read_y():
    y = 'Hey'  # y appears in an assignment, therefore it's local
    print(y)   # will find the local y

read_y()       # prints Hey

def read_x_local_fail():
    if False:
        x = 'Hey'  # x appears in an assignment, therefore it's local
    print(x)   # will look for the _local_ z, which is not assigned, and will not be found

read_x_local_fail()   # UnboundLocalError: local variable 'x' referenced before assignment

Normalmente, uma atribuição dentro de um escopo ocultará quaisquer variáveis ​​externas do mesmo nome:

x = 'Hi'

def change_local_x():
    x = 'Bye'
    print(x)
change_local_x()  # prints Bye
print(x)  # prints Hi

Declarar um nome global significa que, para o resto do escopo, quaisquer atribuições ao nome acontecerão no nível superior do módulo:

x = 'Hi'

def change_global_x():
    global x
    x = 'Bye'
    print(x)

change_global_x()  # prints Bye
print(x)  # prints Bye

A palavra-chave global significa que as atribuições acontecerão no nível superior do módulo, não no nível superior do programa. Outros módulos ainda precisarão do acesso pontilhado usual às variáveis ​​dentro do módulo.

Resumindo: para saber se uma variável x é local para uma função, você deve ler a função entire:

  1. se você encontrou global x, então x é uma variável global
  2. Se você encontrou nonlocal x, então x pertence a uma função delimitadora e não é local nem global
  3. Se você encontrou x = 5 ou for x in range(3) ou alguma outra ligação, então x é uma variável local
  4. Caso contrário, x pertence a algum escopo delimitador (escopo de função, escopo global ou builtins)

Variáveis ​​locais #

Se um nome for bound dentro de uma função, por padrão ele estará acessível apenas dentro da função:

def foo():
    a = 5
    print(a) # ok

print(a) #  NameError: name 'a' is not defined

As construções de fluxo de controle não têm impacto no escopo (com exceção de except), mas acessar a variável que ainda não foi atribuída é um erro:

def foo():
    if True: 
        a = 5
    print(a) # ok

b = 3
def bar():
    if False:
        b = 5
    print(b) # UnboundLocalError: local variable 'b' referenced before assignment

As operações comuns de vinculação são atribuições, loops for e atribuições aumentadas, como a += 5

Ocorrência de vinculação #

x = 5
x += 7
for x in iterable: pass

Cada uma das instruções acima é uma ocorrência de vinculação - x torna-se vinculado ao objeto indicado por 5. Se esta declaração aparecer dentro de uma função, então x será local da função por padrão. Consulte a seção “Sintaxe” para obter uma lista de instruções de associação.

Funções pulam o escopo da classe ao procurar nomes #

As classes têm um escopo local durante a definição, mas as funções dentro da classe não usam esse escopo ao procurar nomes. Como lambdas são funções e as compreensões são implementadas usando o escopo da função, isso pode levar a um comportamento surpreendente.

a = 'global'

class Fred:
    a = 'class'  # class scope
    b = (a for i in range(10))  # function scope
    c = [a for i in range(10)]  # function scope
    d = a  # class scope
    e = lambda: a  # function scope
    f = lambda a=a: a  # default argument uses class scope
    
    @staticmethod  # or @classmethod, or regular instance method
    def g():  # function scope
        return a

print(Fred.a)  # class
print(next(Fred.b))  # global
print(Fred.c[0])  # class in Python 2, global in Python 3
print(Fred.d)  # class
print(Fred.e())  # global
print(Fred.f())  # class
print(Fred.g()) # global

Usuários não familiarizados com o funcionamento desse escopo podem esperar que b, c e e imprimam class.

De PEP 227:

Os nomes no escopo da classe não são acessíveis. Os nomes são resolvidos no escopo da função delimitadora mais interna. Se ocorrer uma definição de classe em uma cadeia de escopos aninhados, o processo de resolução ignorará as definições de classe.

Da documentação do Python sobre nomeação e ligação:

O escopo dos nomes definidos em um bloco de classe é limitado ao bloco de classe; ele não se estende aos blocos de código de métodos – isso inclui compreensões e expressões geradoras, pois são implementadas usando um escopo de função. Isso significa que o seguinte falhará:

classe A: a = 42 b = lista(a + i para i no intervalo(10))

Este exemplo usa referências de esta resposta de Martijn Pieters, que contém uma análise mais aprofundada desse comportamento.

O comando del #

Este comando tem várias formas relacionadas, porém distintas.

del v #

Se v for uma variável, o comando del v remove a variável de seu escopo. Por exemplo:

x = 5
print(x) # out: 5
del x
print(x) # NameError: name 'f' is not defined

Note que del é uma ocorrência de ligação, o que significa que a menos que explicitamente indicado de outra forma (usando nonlocal ou global), del v tornará v local para o escopo atual. Se você pretende deletar v em um escopo externo, use nonlocal v ou global v no mesmo escopo da instrução del v.

Em todos os itens a seguir, a intenção de um comando é um comportamento padrão, mas não é imposta pela linguagem. Uma classe pode ser escrita de uma maneira que invalide essa intenção.

del v.name #

Este comando aciona uma chamada para v.__delattr__(name).

A intenção é tornar o atributo name indisponível. Por exemplo:

class A:
    pass

a = A()
a.x = 7
print(a.x) # out: 7
del a.x
print(a.x) # error: AttributeError: 'A' object has no attribute 'x'

del v[item] #

Este comando aciona uma chamada para v.__delitem__(item).

A intenção é que item não pertença ao mapeamento implementado pelo objeto v. Por exemplo:

x = {'a': 1, 'b': 2}
del x['a']
print(x) #  out: {'b': 2}
print(x['a']) # error: KeyError: 'a'

del v[a:b] #

Isso na verdade chama v.__delslice__(a, b).

A intenção é semelhante à descrita acima, mas com slices - intervalos de itens em vez de um único item. Por exemplo:

x = [0, 1, 2, 3, 4]
del x[1:3]
print(x) #  out: [0, 3, 4]

Veja também [Garbage Collection#The del command](https://www.wikiod.com/pt/python/coleta-de-lixo#Efeitos do comando del).

Escopo local vs global #

O que são escopo local e global? #

Todas as variáveis ​​Python acessíveis em algum ponto do código estão no escopo local ou no escopo global.

A explicação é que o escopo local inclui todas as variáveis ​​definidas na função atual e o escopo global inclui as variáveis ​​definidas fora da função atual.

foo = 1  # global

def func():
    bar = 2  # local
    print(foo)  # prints variable foo from global scope
    print(bar)  # prints variable bar from local scope

Pode-se inspecionar quais variáveis ​​estão em qual escopo. As funções internas locals() e globals() retornam os escopos inteiros como dicionários.

foo = 1

def func():
    bar = 2
    print(globals().keys())  # prints all variable names in global scope
    print(locals().keys())  # prints all variable names in local scope

O que acontece com confrontos de nomes? #

foo = 1

def func():
    foo = 2  # creates a new variable foo in local scope, global foo is not affected

    print(foo)  # prints 2

    # global variable foo still exists, unchanged:
    print(globals()['foo'])  # prints 1
    print(locals()['foo'])  # prints 2

Para modificar uma variável global, use a palavra-chave global:

foo = 1

def func():
    global foo
    foo = 2  # this modifies the global foo, rather than creating a local variable

O escopo é definido para todo o corpo da função!

O que significa é que uma variável nunca será global para metade da função e local depois, ou vice-versa.

foo = 1

def func():
    # This function has a local variable foo, because it is defined down below.
    # So, foo is local from this point. Global foo is hidden.

    print(foo) # raises UnboundLocalError, because local foo is not yet initialized
    foo = 7
    print(foo)

Da mesma forma, o contrário:

foo = 1

def func():
    # In this function, foo is a global variable from the begining

    foo = 7  # global foo is modified

    print(foo)  # 7
    print(globals()['foo'])  # 7

    global foo  # this could be anywhere within the function
    print(foo)  # 7

Funções dentro de funções #

Pode haver muitos níveis de funções aninhadas em funções, mas dentro de qualquer função há apenas um escopo local para essa função e o escopo global. Não há escopos intermediários.

foo = 1

def f1():
    bar = 1

    def f2():
        baz = 2
        # here, foo is a global variable, baz is a local variable
        # bar is not in either scope
        print(locals().keys())  # ['baz']
        print('bar' in locals())  # False
        print('bar' in globals())  # False

    def f3():
        baz = 3
        print(bar)  # bar from f1 is referenced so it enters local scope of f3 (closure)
        print(locals().keys())  # ['bar', 'baz']
        print('bar' in locals())  # True
        print('bar' in globals())  # False

    def f4():
        bar = 4  # a new local bar which hides bar from local scope of f1
        baz = 4
        print(bar)
        print(locals().keys())  # ['bar', 'baz']
        print('bar' in locals())  # True
        print('bar' in globals())  # False

global vs nonlocal (somente Python 3) #

Ambas as palavras-chave são usadas para obter acesso de gravação a variáveis ​​que não são locais para as funções atuais.

A palavra-chave global declara que um nome deve ser tratado como uma variável global.

foo = 0  # global foo

def f1():
    foo = 1  # a new foo local in f1
    
    def f2():
        foo = 2  # a new foo local in f2
        
        def f3():
            foo = 3  # a new foo local in f3
            print(foo)  # 3
            foo = 30  # modifies local foo in f3 only
        
        def f4():
            global foo
            print(foo)  # 0
            foo = 100  # modifies global foo

Por outro lado, nonlocal (consulte https://www.wikiod.com/pt/python/escopo-variavel-e-vinculacao#Variáveis ​​Não Locais ), disponível em Python 3, leva um local variável de um escopo delimitador para o escopo local da função atual.

Da documentação do Python em nonlocal:

A instrução não local faz com que os identificadores listados se refiram a variáveis ​​vinculadas anteriormente no escopo mais próximo, excluindo globais.

def f1():
    
    def f2():
        foo = 2  # a new foo local in f2

        def f3():
            nonlocal foo  # foo from f2, which is the nearest enclosing scope
            print(foo)  # 2
            foo = 20  # modifies foo from f2!
