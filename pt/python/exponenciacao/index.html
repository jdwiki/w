Raiz quadrada: math.sqrt() e cmath.sqrt #

O módulo math contém a função math.sqrt() que pode calcular a raiz quadrada de qualquer número (que pode ser convertido para um float) e o resultado será sempre um float:

import math

math.sqrt(9)                # 3.0
math.sqrt(11.11)            # 3.3331666624997918
math.sqrt(Decimal('6.25'))  # 2.5

A função math.sqrt() gera um ValueError se o resultado for complex:

math.sqrt(-10)

ValueError: erro de domínio matemático

math.sqrt(x) é mais rápido que math.pow(x, 0.5) ou x ** 0.5 mas a precisão dos resultados é a mesma. O módulo cmath é extremamente semelhante ao módulo math, exceto pelo fato de poder calcular números complexos e todos os seus resultados estão na forma de a + bi. Ele também pode usar .sqrt():

import cmath

cmath.sqrt(4)  # 2+0j
cmath.sqrt(-4) # 2j

O que há com o j? j é o equivalente à raiz quadrada de -1. Todos os números podem ser colocados na forma a + bi, ou neste caso, a + bj. a é a parte real do número como o 2 em 2+0j. Como não tem parte imaginária, b é 0. b representa parte da parte imaginária do número como o 2 em 2j. Como não há parte real nisso, 2j também pode ser escrito como 0 + 2j.

Exponenciação usando builtins: ** e pow() #

[Exponenciation] (https://en.wikipedia.org/wiki/Exponentiation) pode ser usado usando a função interna pow ou o operador **:

2 ** 3    # 8
pow(2, 3) # 8

Para a maioria das operações aritméticas (todas em Python 2.x), o tipo do resultado será o do operando mais amplo. Isso não é verdade para **; os seguintes casos são exceções a esta regra:

  • Base: int, expoente: int < 0:

      2 ** -3
  # Out: 0.125 (result is a float)

  • Isso também é válido para Python 3.x.

  • Antes do Python 2.2.0, isso gerava um ValueError.

  • Base: int < 0 ou float < 0, expoente: float != int

      (-2) ** (0.5)  # also (-2.) ** (0.5)    
  # Out: (8.659560562354934e-17+1.4142135623730951j) (result is complex)

  • Antes do python 3.0.0, isso gerava um ValueError.

O módulo operator contém duas funções que são equivalentes ao **-operator:

import operator
operator.pow(4, 2)      # 16
operator.__pow__(4, 3)  # 64

ou pode-se chamar diretamente o método __pow__:

val1, val2 = 4, 2
val1.__pow__(val2)      # 16
val2.__rpow__(val1)     # 16
# in-place power operation isn't supported by immutable classes like int, float, complex:
# val1.__ipow__(val2)

Exponenciação modular: pow() com 3 argumentos #

Fornecer pow() com 3 argumentos pow(a, b, c) avalia a exponenciação modular ab mod c:

pow(3, 4, 17)   # 13

# equivalent unoptimized expression:
3 ** 4 % 17     # 13

# steps:
3 ** 4          # 81
81 % 17         # 13

Para tipos embutidos, o uso de exponenciação modular só é possível se:

  • O primeiro argumento é um int
  • O segundo argumento é um int >= 0
  • O terceiro argumento é um int != 0

Essas restrições também estão presentes no python 3.x

Por exemplo, pode-se usar a forma de 3 argumentos de pow para definir uma função modular inverse:

def modular_inverse(x, p):
    """Find a such as  a·x ≡ 1 (mod p), assuming p is prime."""
    return pow(x, p-2, p)

[modular_inverse(x, 13) for x in range(1,13)]
# Out: [1, 7, 9, 10, 8, 11, 2, 5, 3, 4, 6, 12]

Exponenciação usando o módulo math: math.pow() #

O módulo math contém outra função math.pow(). A diferença para a função interna pow() ou operador ** é que o resultado é sempre um float:

import math
math.pow(2, 2)    # 4.0
math.pow(-2., 2)  # 4.0

O que exclui cálculos com entradas complexas:

math.pow(2, 2+0j)

TypeError: não pode converter complexo para float

e cálculos que levariam a resultados complexos:

math.pow(-2, 0.5)

ValueError: erro de domínio matemático

Função exponencial: math.exp() e cmath.exp() #

Tanto o módulo math quanto o cmath contêm o número Euler: e e usando-o com o pow()- embutido function ou **-operator funciona principalmente como math.exp():

import math

math.e ** 2  # 7.3890560989306495
math.exp(2)  # 7.38905609893065

import cmath
cmath.e ** 2 # 7.3890560989306495
cmath.exp(2) # (7.38905609893065+0j)

No entanto, o resultado é diferente e usar a função exponencial diretamente é mais confiável do que a exponenciação interna com base math.e:

print(math.e ** 10)       # 22026.465794806703
print(math.exp(10))       # 22026.465794806718
print(cmath.exp(10).real) # 22026.465794806718
#     difference starts here ---------------^

Função exponencial menos 1: math.expm1() #

O módulo math contém a função expm1() que pode calcular a expressão math.e ** x - 1 para x muito pequeno com maior precisão que math.exp(x) ou cmath .exp(x) permitiria:

import math

print(math.e ** 1e-3 - 1)  # 0.0010005001667083846
print(math.exp(1e-3) - 1)  # 0.0010005001667083846
print(math.expm1(1e-3))    # 0.0010005001667083417
#                            ------------------^

Para x muito pequeno a diferença fica maior:

print(math.e ** 1e-15 - 1) # 1.1102230246251565e-15
print(math.exp(1e-15) - 1) # 1.1102230246251565e-15
print(math.expm1(1e-15))   # 1.0000000000000007e-15
#                              ^-------------------

A melhoria é significativa na computação científica. Por exemplo, a lei de Planck contém uma função exponencial menos 1:

def planks_law(lambda_, T):
    from scipy.constants import h, k, c  # If no scipy installed hardcode these!
    return 2 * h * c ** 2 / (lambda_ ** 5 * math.expm1(h * c / (lambda_ * k * T)))

def planks_law_naive(lambda_, T):
    from scipy.constants import h, k, c  # If no scipy installed hardcode these!
    return 2 * h * c ** 2 / (lambda_ ** 5 * (math.e ** (h * c / (lambda_ * k * T)) - 1))

planks_law(100, 5000)        # 4.139080074896474e-19
planks_law_naive(100, 5000)  # 4.139080073488451e-19
#                                        ^---------- 

planks_law(1000, 5000)       # 4.139080128493406e-23
planks_law_naive(1000, 5000) # 4.139080233183142e-23
#                                      ^------------

Métodos mágicos e exponenciação: builtin, math e cmath #

Supondo que você tenha uma classe que armazena valores puramente inteiros:

class Integer(object):
    def __init__(self, value):
        self.value = int(value) # Cast to an integer
        
    def __repr__(self):
        return '{cls}({val})'.format(cls=self.__class__.__name__,
                                     val=self.value)
    
    def __pow__(self, other, modulo=None):
        if modulo is None:
            print('Using __pow__')
            return self.__class__(self.value ** other)
        else:
            print('Using __pow__ with modulo')
            return self.__class__(pow(self.value, other, modulo))
    
    def __float__(self):
        print('Using __float__')
        return float(self.value)
    
    def __complex__(self):
        print('Using __complex__')
        return complex(self.value, 0)

Usando a função interna pow ou o operador ** sempre chama __pow__:

Integer(2) ** 2                 # Integer(4)
# Prints: Using __pow__
Integer(2) ** 2.5               # Integer(5)
# Prints: Using __pow__
pow(Integer(2), 0.5)            # Integer(1)
# Prints: Using __pow__  
operator.pow(Integer(2), 3)     # Integer(8)
# Prints: Using __pow__
operator.__pow__(Integer(3), 3) # Integer(27)
# Prints: Using __pow__

O segundo argumento do método __pow__() só pode ser fornecido usando o builtin-pow() ou chamando diretamente o método:

pow(Integer(2), 3, 4)           # Integer(0)
# Prints: Using __pow__ with modulo
Integer(2).__pow__(3, 4)        # Integer(0) 
# Prints: Using __pow__ with modulo

Enquanto as funções math sempre o convertem para um float e usam a computação float:

import math

math.pow(Integer(2), 0.5) # 1.4142135623730951
# Prints: Using __float__

As funções cmath tentam convertê-lo para complex, mas também podem retornar para float se não houver conversão explícita para complex:

import cmath

cmath.exp(Integer(2))     # (7.38905609893065+0j)
# Prints: Using __complex__

del Integer.__complex__   # Deleting __complex__ method - instances cannot be cast to complex

cmath.exp(Integer(2))     # (7.38905609893065+0j)
# Prints: Using __float__

Nem math nem cmath funcionarão se também estiver faltando o método __float__():

del Integer.__float__  # Deleting __complex__ method

math.sqrt(Integer(2))  # also cmath.exp(Integer(2))

TypeError: um float é necessário

Raízes: raiz n com expoentes fracionários #

Embora a função math.sqrt seja fornecida para o caso específico de raízes quadradas, geralmente é conveniente usar o operador de exponenciação (**) com expoentes fracionários para realizar operações de raiz n, como raízes cúbicas.

O inverso de uma exponenciação é a exponenciação pelo recíproco do expoente. Então, se você pode cubo um número colocando-o no expoente de 3, você pode encontrar a raiz cúbica de um número colocando-o no expoente de 1/3.

>>> x = 3
>>> y = x ** 3
>>> y
27
>>> z = y ** (1.0 / 3)
>>> z
3.0
>>> z == x
True

Calculando raízes inteiras grandes #

Mesmo que o Python suporte nativamente grandes números inteiros, tirar a n-ésima raiz de números muito grandes pode falhar no Python.

x = 2 ** 100
cube = x ** 3
root = cube ** (1.0 / 3)

OverflowError: long int muito grande para converter em float

Ao lidar com números inteiros tão grandes, você precisará usar uma função personalizada para calcular a enésima raiz de um número.

def nth_root(x, n):
    # Start with some reasonable bounds around the nth root.
    upper_bound = 1
    while upper_bound ** n <= x:
        upper_bound *= 2
    lower_bound = upper_bound // 2
    # Keep searching for a better result as long as the bounds make sense.
    while lower_bound < upper_bound:
        mid = (lower_bound + upper_bound) // 2
        mid_nth = mid ** n
        if lower_bound < mid and mid_nth < x:
            lower_bound = mid
        elif upper_bound > mid and mid_nth > x:
            upper_bound = mid
        else:
            # Found perfect nth root.
            return mid
    return mid + 1

x = 2 ** 100
cube = x ** 3
root = nth_root(cube, 3)
x == root
# True
