  • Iterar duas ou mais listas simultaneamente dentro da compreensão da lista

Uma compreensão de lista é uma ferramenta sintática para criar listas de forma natural e concisa, conforme ilustrado no código a seguir para fazer uma lista de quadrados dos números de 1 a 10:

[i ** 2 para i no intervalo(1,11)]

O i fictício de uma lista existente range é usado para criar um novo padrão de elemento. É usado onde um loop for seria necessário em linguagens menos expressivas.

Sintaxe #

  • [i for i in range(10)] # compreensão básica da lista
  • [i for i in xrange(10)] # compreensão de lista básica com objeto gerador em python 2.x
  • [i for i in range(20) if i % 2 == 0] # com filtro
  • [x + y para x em [1, 2, 3] para y em [3, 4, 5]] # loops aninhados
  • [i if i > 6 else 0 for i in range(10)] # expressão ternária
  • [i if i > 4 else 0 for i in range(20) if i % 2 == 0] # com filtro e expressão ternária
  • [[x + y para x em [1, 2, 3]] para y em [3, 4, 5]] # compreensão de lista aninhada

As compreensões de lista foram descritas em PEP 202 e introduzidas no Python 2.0.

Compreensão de lista condicional #

Dada uma compreensão de lista, você pode anexar uma ou mais condições if para filtrar valores.

[<expression> for <element> in <iterable> if <condition>]

Para cada <elemento> em <iterável>; se <condition> for avaliado como True, adicione <expression> (geralmente uma função de <element>) à lista retornada.

Por exemplo, isso pode ser usado para extrair apenas números pares de uma sequência de inteiros:

[x for x in range(10) if x % 2 == 0]
# Out: [0, 2, 4, 6, 8]

Demonstração ao vivo

O código acima é equivalente a:

even_numbers = [] 
for x in range(10):
    if x % 2 == 0:
        even_numbers.append(x)

print(even_numbers)
# Out: [0, 2, 4, 6, 8]

Além disso, uma compreensão de lista condicional da forma [e for x in y if c] (onde e e c são expressões em termos de x) é equivalente a list(filter(lambda x: c , map(lambda x: e, y))).

Apesar de fornecer o mesmo resultado, preste atenção ao fato de que o primeiro exemplo é quase 2x mais rápido que o segundo. Para aqueles que estão curiosos, este é uma boa explicação do motivo .

Observe que isso é bem diferente da expressão condicional ... if ... else ... (às vezes conhecida como expressão ternária) que você pode usar para a parte <expressão> do compreensão da lista. Considere o seguinte exemplo:

[x if x % 2 == 0 else None for x in range(10)]
# Out: [0, None, 2, None, 4, None, 6, None, 8, None]

Demonstração ao vivo

Aqui a expressão condicional não é um filtro, mas sim um operador que determina o valor a ser usado para os itens da lista:

<value-if-condition-is-true> if <condition> else <value-if-condition-is-false>

Isso se torna mais óbvio se você combiná-lo com outros operadores:

[2 * (x if x % 2 == 0 else -1) + 1 for x in range(10)]
# Out: [1, -1, 5, -1, 9, -1, 13, -1, 17, -1]

Demonstração ao vivo

Se você estiver usando Python 2.7, xrange pode ser melhor que range por vários motivos, conforme descrito na documentação do xrange.

[2 * (x if x % 2 == 0 else -1) + 1 for x in xrange(10)]
# Out: [1, -1, 5, -1, 9, -1, 13, -1, 17, -1]

O código acima é equivalente a:

numbers = []
for x in range(10):
    if x % 2 == 0:
        temp = x
    else:
        temp = -1
    numbers.append(2 * temp + 1)
print(numbers)
# Out: [1, -1, 5, -1, 9, -1, 13, -1, 17, -1]

Pode-se combinar expressões ternárias e condições if. O operador ternário funciona no resultado filtrado:

[x if x > 2 else '*' for x in range(10) if x % 2 == 0]
# Out: ['*', '*', 4, 6, 8]

O mesmo não poderia ter sido alcançado apenas pelo operador ternário:

[x if (x > 2 and x % 2 == 0) else '*' for x in range(10)]
# Out:['*', '*', '*', '*', 4, '*', 6, '*', 8, '*']

Consulte também: Filters, que geralmente fornecem uma alternativa suficiente para compreensões de lista condicional.

Listar Compreensão com Loops Aninhados #

List Comprehensions pode usar loops for aninhados. Você pode codificar qualquer número de loops for aninhados dentro de uma compreensão de lista, e cada loop for pode ter um teste if associado opcional. Ao fazer isso, a ordem das construções for é a mesma ordem que ao escrever uma série de instruções for aninhadas. A estrutura geral das compreensões de lista se parece com isso:

[ expression for target1 in iterable1 [if condition1]
             for target2 in iterable2 [if condition2]...
             for targetN in iterableN [if conditionN] ]

Por exemplo, o código a seguir nivelando uma lista de listas usando várias instruções for:

data = [[1, 2], [3, 4], [5, 6]]
output = []
for each_list in data:
    for element in each_list:
        output.append(element)
print(output)
# Out: [1, 2, 3, 4, 5, 6]

pode ser escrito de forma equivalente como uma compreensão de lista com várias construções for:

data = [[1, 2], [3, 4], [5, 6]]
output = [element for each_list in data for element in each_list]
print(output)
# Out: [1, 2, 3, 4, 5, 6]

Demonstração ao vivo

Tanto na forma expandida quanto na compreensão de lista, o loop externo (first for statement) vem primeiro.

Além de ser mais compacta, a compreensão aninhada também é significativamente mais rápida.

In [1]: data = [[1,2],[3,4],[5,6]]
In [2]: def f():
   ...:     output=[]
   ...:     for each_list in data:
   ...:         for element in each_list:
   ...:             output.append(element)
   ...:     return output
In [3]: timeit f()
1000000 loops, best of 3: 1.37 µs per loop
In [4]: timeit [inner for outer in data for inner in outer]
1000000 loops, best of 3: 632 ns per loop

A sobrecarga para a chamada de função acima é de cerca de 140ns.

Os ifs embutidos são aninhados de forma semelhante e podem ocorrer em qualquer posição após o primeiro for:

data = [[1], [2, 3], [4, 5]]
output = [element for each_list in data
                if len(each_list) == 2
                for element in each_list
                if element != 5]
print(output)
# Out: [2, 3, 4]

Demonstração ao vivo

Por uma questão de legibilidade, no entanto, você deve considerar o uso de for-loops tradicionais. Isso é especialmente verdadeiro quando o aninhamento tem mais de 2 níveis de profundidade e/ou a lógica da compreensão é muito complexa. a compreensão da lista de loops aninhados múltiplos pode ser propensa a erros ou fornece resultados inesperados.

Filtro de refatoração e mapeamento para compreensões de lista #

As funções filter ou map devem ser frequentemente substituídas por list compreensãos. Guido Van Rossum descreve isso bem em uma carta aberta em 2005:

filter(P, S) é quase sempre escrito de forma mais clara como [x for x in S if P(x)], e isso tem a enorme vantagem de que os usos mais comuns envolvem predicados que são comparações, por exemplo. x==42, e definir um lambda para isso requer muito mais esforço para o leitor (além disso, o lambda é mais lento que a compreensão da lista). Ainda mais para map(F, S) que se torna [F(x) for x in S]. É claro que, em muitos casos, você poderá usar expressões geradoras.

As seguintes linhas de código são consideradas “not pythonic” e gerarão erros em muitos linters python.

filter(lambda x: x % 2 == 0, range(10)) # even numbers < 10
map(lambda x: 2*x, range(10)) # multiply each number by two
reduce(lambda x,y: x+y, range(10)) # sum of all elements in list

Tomando o que aprendemos com a citação anterior, podemos decompor essas expressões filter e map em suas compreensões de lista equivalentes; também removendo as funções lambda de cada um - tornando o código mais legível no processo.

# Filter:
# P(x) = x % 2 == 0
# S = range(10)
[x for x in range(10) if x % 2 == 0]

# Map
# F(x) = 2*x
# S = range(10)
[2*x for x in range(10)]

A legibilidade torna-se ainda mais aparente ao lidar com funções de encadeamento. Onde devido à legibilidade, os resultados de um mapa ou função de filtro devem ser passados ​​como resultado para o próximo; com casos simples, eles podem ser substituídos por uma única compreensão de lista. Além disso, podemos dizer facilmente a partir da compreensão da lista qual é o resultado do nosso processo, onde há mais carga cognitiva ao raciocinar sobre o processo Map & Filter encadeado.

# Map & Filter
filtered = filter(lambda x: x % 2 == 0, range(10))
results = map(lambda x: 2*x, filtered)

# List comprehension
results = [2*x for x in range(10) if x % 2 == 0]

Refatoração - Referência Rápida #

  • Mapa

    map(F, S) == [F(x) for x in S]

  • Filtro

    filter(P, S) == [x for x in S if P(x)]

onde F e P são funções que transformam respectivamente os valores de entrada e retornam um bool

Compreensão da lista aninhada #

As compreensões de lista aninhada, ao contrário das compreensões de lista com loops aninhados, são compreensões de lista dentro de uma compreensão de lista. A expressão inicial pode ser qualquer expressão arbitrária, incluindo outra compreensão de lista.

#List Comprehension with nested loop
[x + y for x in [1, 2, 3] for y in [3, 4, 5]]
#Out: [4, 5, 6, 5, 6, 7, 6, 7, 8]

#Nested List Comprehension
[[x + y for x in [1, 2, 3]] for y in [3, 4, 5]]
#Out: [[4, 5, 6], [5, 6, 7], [6, 7, 8]]

O exemplo aninhado é equivalente a

l = []
for y in [3, 4, 5]:
    temp = []
    for x in [1, 2, 3]:
        temp.append(x + y)
    l.append(temp)

Um exemplo onde uma compreensão aninhada pode ser usada para transpor uma matriz.

matrix = [[1,2,3],
          [4,5,6],
          [7,8,9]] 

[[row[i] for row in matrix] for i in range(len(matrix))]
# [[1, 4, 7], [2, 5, 8], [3, 6, 9]]

Assim como os loops for aninhados, não há limite para o quanto as compreensões profundas podem ser aninhadas.

[[[i + j + k for k in 'cd'] for j in 'ab'] for i in '12']
# Out: [[['1ac', '1ad'], ['1bc', '1bd']], [['2ac', '2ad'], ['2bc', '2bd']]]

Iterar duas ou mais listas simultaneamente dentro da compreensão da lista #

Para iterar mais de duas listas simultaneamente dentro da compreensão da lista, pode-se usar zip() como:

>>> list_1 = [1, 2, 3 , 4]
>>> list_2 = ['a', 'b', 'c', 'd']
>>> list_3 = ['6', '7', '8', '9']

# Two lists
>>> [(i, j) for i, j in zip(list_1, list_2)]
[(1, 'a'), (2, 'b'), (3, 'c'), (4, 'd')]

# Three lists
>>> [(i, j, k) for i, j, k in zip(list_1, list_2, list_3)]
[(1, 'a', '6'), (2, 'b', '7'), (3, 'c', '8'), (4, 'd', '9')]

# so on ...
