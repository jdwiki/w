Alterando a capitalização de uma string #

O tipo string do Python fornece muitas funções que atuam na capitalização de uma string. Esses incluem :

  • str.casefold
  • str.superior
  • str.lower
  • str.capitalize
  • str.title
  • str.swapcase

Com strings unicode (o padrão no Python 3), essas operações não são mapeamentos 1:1 ou reversíveis. A maioria dessas operações destina-se a fins de exibição, em vez de normalização.

str.casefold() #

str.casefold cria uma string em minúsculas que é adequada para comparações que não diferenciam maiúsculas de minúsculas. Isso é mais agressivo do que str.lower e pode modificar strings que já estão em minúsculas ou fazer com que as strings aumentem de tamanho, e não se destina a fins de exibição.

"XßΣ".casefold()
# 'xssσ'

"XßΣ".lower()
# 'xßς'

As transformações que ocorrem em casefolding são definidas pelo Unicode Consortium no arquivo CaseFolding.txt em seu site.

str.upper() #

str.upper pega todos os caracteres em uma string e os converte em seu equivalente em maiúsculas, por exemplo:

"This is a 'string'.".upper()
# "THIS IS A 'STRING'."

str.lower() #

str.lower faz o oposto; ele pega todos os caracteres em uma string e os converte em seu equivalente em minúsculas:

"This IS a 'string'.".lower()
# "this is a 'string'."

str.capitalize() #

str.capitalize retorna uma versão em maiúscula da string, ou seja, faz com que o primeiro caractere seja maiúsculo e o restante seja minúsculo:

"this Is A 'String'.".capitalize() # Capitalizes the first character and lowercases all others
# "This is a 'string'."

str.title() #

str.title retorna a versão em maiúsculas do título da string, ou seja, todas as letras no início de uma palavra são maiúsculas e todas as outras são minúsculas:

"this Is a 'String'".title()
# "This Is A 'String'"

str.swapcase() #

str.swapcase retorna um novo objeto string no qual todos os caracteres minúsculos são trocados para maiúsculas e todos os caracteres maiúsculos para minúsculos:

"this iS A STRiNG".swapcase() #Swaps case of each character
# "THIS Is a strIng"

Uso como métodos de classe str #

Vale a pena notar que esses métodos podem ser chamados em objetos string (como mostrado acima) ou como um método de classe da classe str (com uma chamada explícita para str.upper, etc.)

str.upper("This is a 'string'")
# "THIS IS A 'STRING'"

Isso é mais útil ao aplicar um desses métodos a muitas strings de uma só vez, digamos, uma função map.

map(str.upper,["These","are","some","'strings'"])
# ['THESE', 'ARE', 'SOME', "'STRINGS'"]

str.translate: Traduzindo caracteres em uma string #

Python suporta um método translate no tipo str que permite especificar a tabela de tradução (usada para substituições), bem como quaisquer caracteres que devem ser excluídos no processo.

str.translate(table[, deletechars])
Parâmetro Descrição
tabela É uma tabela de pesquisa que define o mapeamento de um caractere para outro.
deletechars Uma lista de caracteres que devem ser removidos da string.

O método maketrans (str.maketrans em Python 3 e string.maketrans em Python 2) permite gerar uma tabela de tradução.

>>> translation_table = str.maketrans("aeiou", "12345")
>>> my_string = "This is a string!"
>>> translated = my_string.translate(translation_table)
'Th3s 3s 1 str3ng!'

O método translate retorna uma string que é uma cópia traduzida da string original.

Você pode definir o argumento table para None se precisar apenas excluir caracteres.

>>> 'this syntax is very useful'.translate(None, 'aeiou')
'ths syntx s vry sfl'

str.format e f-strings: formata valores em uma string #

Python fornece interpolação de strings e funcionalidade de formatação através da função str.format, introduzida na versão 2.6 e f-strings introduzidas na versão 3.6.

Dadas as seguintes variáveis:

i = 10
f = 1.5
s = "foo"
l = ['a', 1, 2]
d = {'a': 1, 2: 'foo'}

As seguintes afirmações são todas equivalentes

"10 1.5 foo ['a', 1, 2] {'a': 1, 2: 'foo'}"

>>> "{} {} {} {} {}".format(i, f, s, l, d)

>>> str.format("{} {} {} {} {}", i, f, s, l, d)

>>> "{0} {1} {2} {3} {4}".format(i, f, s, l, d)

>>> "{0:d} {1:0.1f} {2} {3!r} {4!r}".format(i, f, s, l, d)

>>> "{i:d} {f:0.1f} {s} {l!r} {d!r}".format(i=i, f=f, s=s, l=l, d=d)

>>> f"{i} {f} {s} {l} {d}"

>>> f"{i:d} {f:0.1f} {s} {l!r} {d!r}"

Para referência, o Python também oferece suporte a qualificadores de estilo C para formatação de strings. Os exemplos abaixo são equivalentes aos acima, mas as versões str.format são preferidas devido aos benefícios em flexibilidade, consistência de notação e extensibilidade:

"%d %0.1f %s %r %r" % (i, f, s, l, d)

"%(i)d %(f)0.1f %(s)s %(l)r %(d)r" % dict(i=i, f=f, s=s, l=l, d=d)

As chaves usadas para interpolação em str.format também podem ser numeradas para reduzir a duplicação ao formatar strings. Por exemplo, os seguintes são equivalentes:

"I am from Australia. I love cupcakes from Australia!"

>>> "I am from {}. I love cupcakes from {}!".format("Australia", "Australia")

>>> "I am from {0}. I love cupcakes from {0}!".format("Australia")

Enquanto a documentação oficial do python é, como sempre, completa o suficiente, pyformat.info tem um grande conjunto de exemplos com explicações detalhadas.

Além disso, os caracteres { e } podem ser escapados usando colchetes duplos:

"{'a': 5, 'b': 6}"

>>> "{{'{}': {}, '{}': {}}}".format("a", 5, "b", 6)

>>> f"{{'{'a'}': {5}, '{'b'}': {6}}"

Consulte Formatação de strings para obter informações adicionais. str.format() foi proposto em PEP 3101 e f-strings em PEP 498.

Divida uma string com base em um delimitador em uma lista de strings #

str.split(sep=Nenhum, maxsplit=-1) #

str.split pega uma string e retorna uma lista de substrings da string original. O comportamento difere dependendo se o argumento sep é fornecido ou omitido.

Se sep não for fornecido ou for None, a divisão ocorrerá onde houver espaço em branco. No entanto, os espaços em branco iniciais e finais são ignorados e vários caracteres de espaço em branco consecutivos são tratados da mesma forma que um único caractere de espaço em branco:

>>> "This is a sentence.".split()
['This', 'is', 'a', 'sentence.']

>>> " This is    a sentence.  ".split()
['This', 'is', 'a', 'sentence.']

>>> "            ".split()
[]

O parâmetro sep pode ser usado para definir uma string delimitadora. A string original é dividida onde a string do delimitador ocorre e o próprio delimitador é descartado. Vários delimitadores consecutivos não são tratados da mesma forma que uma única ocorrência, mas causam a criação de strings vazias.

>>> "This is a sentence.".split(' ')
['This', 'is', 'a', 'sentence.']

>>> "Earth,Stars,Sun,Moon".split(',')
['Earth', 'Stars', 'Sun', 'Moon']

>>> " This is    a sentence.  ".split(' ')
['', 'This', 'is', '', '', '', 'a', 'sentence.', '', '']

>>> "This is a sentence.".split('e')
['This is a s', 'nt', 'nc', '.']

>>> "This is a sentence.".split('en')
['This is a s', 't', 'ce.']

O padrão é dividir em toda ocorrência do delimitador, porém o parâmetro maxsplit limita o número de divisões que ocorrem. O valor padrão de -1 significa sem limite:

>>> "This is a sentence.".split('e', maxsplit=0)
['This is a sentence.']

>>> "This is a sentence.".split('e', maxsplit=1)
['This is a s', 'ntence.']

>>> "This is a sentence.".split('e', maxsplit=2)
['This is a s', 'nt', 'nce.']

>>> "This is a sentence.".split('e', maxsplit=-1)
['This is a s', 'nt', 'nc', '.']

str.rsplit(sep=Nenhum, maxsplit=-1) #

str.rsplit (“divisão à direita”) difere de str.split (“divisão à esquerda”) quando maxsplit é especificado. A divisão começa no final da string e não no início:

>>> "This is a sentence.".rsplit('e', maxsplit=1)
['This is a sentenc', '.']

>>> "This is a sentence.".rsplit('e', maxsplit=2)
['This is a sent', 'nc', '.']

Observação: Python especifica o número máximo de splits executados, enquanto a maioria das outras linguagens de programação especifica o número máximo de substrings criados. Isso pode criar confusão ao portar ou comparar o código.

Substitui todas as ocorrências de uma substring por outra substring #

O tipo str do Python também tem um método para substituir ocorrências de uma substring por outra substring em uma determinada string. Para casos mais exigentes, pode-se usar re.sub.

str.replace(old, new[, count]): #

str.replace recebe dois argumentos old e new contendo a substring old que deve ser substituída pela substring new. O argumento opcional count especifica o número de substituições a serem feitas:

Por exemplo, para substituir 'foo' por 'spam' na seguinte string, podemos chamar str.replace com old = 'foo' e new = 'spam':

>>> "Make sure to foo your sentence.".replace('foo', 'spam')
"Make sure to spam your sentence."

Se a string fornecida contiver vários exemplos que correspondem ao argumento old, todas ocorrências serão substituídas pelo valor fornecido em new:

>>> "It can foo multiple examples of foo if you want.".replace('foo', 'spam')
"It can spam multiple examples of spam if you want."

a menos, é claro, que forneçamos um valor para count. Neste caso, as ocorrências count serão substituídas:

>>> """It can foo multiple examples of foo if you want, \
... or you can limit the foo with the third argument.""".replace('foo', 'spam', 1)
'It can spam multiple examples of foo if you want, or you can limit the foo with the third argument.'

Testando do que uma string é composta #

O tipo str do Python também apresenta vários métodos que podem ser usados ​​para avaliar o conteúdo de uma string. Estes são str.isalpha, str.isdigit, str.isalnum, str.isspace. A capitalização pode ser testada com str.isupper, str.islower e str.istitle.

str.isalpha #

str.isalpha não recebe argumentos e retorna True se todos os caracteres em uma determinada string forem alfabéticos, por exemplo:

>>> "Hello World".isalpha()  # contains a space
False
>>> "Hello2World".isalpha()  # contains a number
False
>>> "HelloWorld!".isalpha()  # contains punctuation
False
>>> "HelloWorld".isalpha()
True

Como um caso extremo, a string vazia é avaliada como False quando usada com "".isalpha().

str.isupper, str.islower, str.istitle #

Esses métodos testam a capitalização em uma determinada string.

str.isupper é um método que retorna True se todos os caracteres em uma determinada string forem maiúsculas e False caso contrário.

>>> "HeLLO WORLD".isupper()
False
>>> "HELLO WORLD".isupper()
True
>>> "".isupper()
False

Por outro lado, str.islower é um método que retorna True se todos os caracteres em uma determinada string forem minúsculos e False caso contrário.

>>> "Hello world".islower()
False
>>> "hello world".islower()
True
>>> "".islower()
False

str.istitle retorna True se a string dada estiver em maiúsculas; ou seja, cada palavra começa com um caractere maiúsculo seguido de caracteres minúsculos.

>>> "hello world".istitle()
False
>>> "Hello world".istitle()
False
>>> "Hello World".istitle()
True
>>> "".istitle()
False

str.isdecimal, str.isdigit, str.isnumeric #

str.isdecimal retorna se a string é uma sequência de dígitos decimais, adequada para representar um número decimal.

str.isdigit inclui dígitos que não estão em um formato adequado para representar um decimal número, como dígitos sobrescritos.

str.isnumeric inclui quaisquer valores numéricos, mesmo que não sejam dígitos, como valores fora do intervalo de 0 a 9.

            isdecimal    isdigit   isnumeric

12345        True        True       True
១2߃໔5        True        True       True
①²³🄅₅       False       True       True
⑩⒓          False       False      True
Five         False       False      False

Bytestrings (bytes em Python 3, str em Python 2), suportam apenas isdigit, que verifica apenas dígitos ASCII básicos.

Assim como em str.isalpha, a string vazia é avaliada como False.

str.isalnum #

Esta é uma combinação de str.isalpha e str.isnumeric, especificamente ela é avaliada como True se todos os caracteres na string fornecida forem alfanuméricos, ou seja, eles consistem em caracteres alfabéticos ou numéricos :

>>> "Hello2World".isalnum()
True
>>> "HelloWorld".isalnum()
True
>>> "2016".isalnum()
True
>>> "Hello World".isalnum()  # contains whitespace
False

str.isspace #

Avalia como True se a string contiver apenas caracteres de espaço em branco.

>>> "\t\r\n".isspace()
True
>>> " ".isspace()
True

Às vezes, uma string parece “vazia”, mas não sabemos se é porque contém apenas espaços em branco ou nenhum caractere

>>> "".isspace()
False

Para cobrir este caso, precisamos de um teste adicional

>>> my_str = ''
>>> my_str.isspace()
False
>>> my_str.isspace() or not my_str
True

Mas a maneira mais curta de testar se uma string está vazia ou apenas contém caracteres de espaço em branco é usar strip(sem argumentos, ele remove todos os caracteres de espaço em branco à esquerda e à direita)

>>> not my_str.strip()
True

Removendo caracteres à esquerda/à direita indesejados de uma string #

Três métodos são fornecidos que oferecem a capacidade de retirar caracteres iniciais e finais de uma string: str.strip, str.rstrip e str.lstrip. Todos os três métodos têm a mesma assinatura e todos os três retornam um novo objeto string com os caracteres indesejados removidos.

str.strip([chars]) #

str.strip atua em uma determinada string e remove (strips) quaisquer caracteres iniciais ou finais contidos no argumento chars; se chars não for fornecido ou for None, todos os caracteres de espaço em branco serão removidos por padrão. Por exemplo:

>>> "    a line with leading and trailing space     ".strip() 
'a line with leading and trailing space'

Se chars for fornecido, todos os caracteres contidos nele serão removidos da string, que será retornada. Por exemplo:

>>> ">>> a Python prompt".strip('> ')  # strips '>' character and space character 
'a Python prompt'

str.rstrip([chars]) e str.lstrip([chars]) #

Esses métodos têm semântica e argumentos semelhantes com str.strip(), sua diferença está na direção de onde eles começam. str.rstrip() começa no final da string enquanto str.lstrip() divide a partir do início da string.

Por exemplo, usando str.rstrip:

>>> "     spacious string      ".rstrip()
'     spacious string'

Enquanto, usando str.lstrip:

>>> "     spacious string      ".rstrip()
'spacious string      '

Invertendo uma string #

Uma string pode ser revertida usando a função interna reversed(), que recebe uma string e retorna um iterador na ordem inversa.

>>> reversed('hello')
<reversed object at 0x0000000000000000>
>>> [char for char in reversed('hello')]
['o', 'l', 'l', 'e', 'h']

reversed() pode ser encapsulado em uma chamada para ''.join() para fazer uma string do iterador.

>>> ''.join(reversed('hello'))
'olleh'

Embora o uso de reversed() possa ser mais legível para usuários não iniciados do Python, usar slicing estendido com uma etapa de -1 é mais rápido e conciso. Aqui, tente implementá-lo como função:

>>> def reversed_string(main_string):
...     return main_string[::-1]
...
>>> reversed_string('hello')
'olleh'

Junte uma lista de strings em uma string #

Uma string pode ser usada como separador para unir uma lista de strings em um string única usando o método join(). Por exemplo, você pode criar uma string onde cada elemento em uma lista é separado por um espaço.

>>> " ".join(["once","upon","a","time"])
"once upon a time"

O exemplo a seguir separa os elementos de string com três hífens.

>>> "---".join(["once", "upon", "a", "time"])
"once---upon---a---time"

String contém #

Python torna extremamente intuitivo verificar se uma string contém uma determinada substring. Basta usar o operador in:

>>> "foo" in "foo.baz.bar"
True

Nota: testar uma string vazia sempre resultará em True:

>>> "" in "test"
True

Contando o número de vezes que uma substring aparece em uma string #

Um método está disponível para contar o número de ocorrências de uma substring em outra string, str.count.

str.count(sub[, start[, end]]) #

str.count retorna um int indicando o número de ocorrências não sobrepostas da substring sub em outra string. Os argumentos opcionais start e end indicam o início e o fim em que a pesquisa ocorrerá. Por padrão, start = 0 e end = len(str) significa que toda a string será pesquisada:

>>> s = "She sells seashells by the seashore."
>>> s.count("sh")
2
>>> s.count("se")
3
>>> s.count("sea")
2
>>> s.count("seashells")
1

Ao especificar um valor diferente para start, end podemos obter uma pesquisa e contagem mais localizadas, por exemplo, se start for igual a 13 a chamada para:

>>> s.count("sea", start)
1

é equivalente a:

>>> t = s[start:]
>>> t.count("sea")
1

Comparações de strings sem distinção entre maiúsculas e minúsculas #

Comparar strings de uma maneira que não diferencia maiúsculas de minúsculas parece algo trivial, mas não é. Esta seção considera apenas strings unicode (o padrão no Python 3). Observe que o Python 2 pode ter fraquezas sutis em relação ao Python 3 - o manuseio de unicode do último é muito mais completo.

A primeira coisa a notar é que as conversões de remoção de maiúsculas e minúsculas em unicode não são triviais. Existe um texto para o qual text.lower() != text.upper().lower(), como "ß":

>>> "ß".lower()
'ß'

>>> "ß".upper().lower()
'ss'

Mas digamos que você queira comparar sem maiúsculas e minúsculas "BUSSE" e "Buße". Caramba, você provavelmente também quer comparar "BUSSE" e "BUẞE" iguais - essa é a forma mais recente de capital. A maneira recomendada é usar casefold:

>>> help(str.casefold)
"""
Help on method_descriptor:

casefold(...)
      S.casefold() -> str
    
     Return a version of S suitable for caseless comparisons.
"""

Não use apenas inferior. Se casefold não estiver disponível, fazer .upper().lower() ajuda (mas apenas um pouco).

Então você deve considerar os acentos. Se o seu renderizador de fonte for bom, você provavelmente pensa "ê" == "ê" - mas não:

>>> "ê" == "ê"
False

Isso porque eles são realmente

>>> import unicodedata

>>> [unicodedata.name(char) for char in "ê"]
['LATIN SMALL LETTER E WITH CIRCUMFLEX']

>>> [unicodedata.name(char) for char in "ê"]
['LATIN SMALL LETTER E', 'COMBINING CIRCUMFLEX ACCENT']

A maneira mais simples de lidar com isso é unicodedata.normalize. Você provavelmente deseja usar a normalização do NFKD, mas fique à vontade para verificar a documentação. Então um faz

>>> unicodedata.normalize("NFKD", "ê") == unicodedata.normalize("NFKD", "ê")
True

Para finalizar, aqui isso é expresso em funções:

import unicodedata

def normalize_caseless(text):
    return unicodedata.normalize("NFKD", text.casefold())

def caseless_equal(left, right):
    return normalize_caseless(left) == normalize_caseless(right)

Testa os caracteres inicial e final de uma string #

Para testar o início e o fim de uma determinada string em Python, pode-se usar os métodos str.startswith() e str.endswith().

str.startswith(prefix[, start[, end]]) #

Como o nome indica, str.startswith é usado para testar se uma determinada string começa com os caracteres fornecidos em prefix.

>>> s = "This is a test string"
>>> s.startswith("T")
True
>>> s.startswith("Thi")
True
>>> s.startswith("thi")  
False

Os argumentos opcionais start e end especificam os pontos inicial e final a partir dos quais o teste começará e terminará. No exemplo a seguir, especificando um valor inicial de 2 nossa string será pesquisada a partir da posição 2 e depois:

>>> s.startswith("is", 2)
True

Isso produz True já que s[2] == 'i' e s[3] == 's'.

Você também pode usar uma tupla para verificar se ela começa com qualquer um de um conjunto de strings

>>> s.startswith(('This', 'That'))
True
>>> s.startswith(('ab', 'bc'))
False

str.endswith(prefix[, start[, end]]) #

str.endswith é exatamente similar a str.startswith com a única diferença de que ele procura por caracteres finais e não por caracteres iniciais. Por exemplo, para testar se uma string termina em ponto final, pode-se escrever:

>>> s = "this ends in a full stop."
>>> s.endswith('.')
True
>>> s.endswith('!')
False

como com startswith mais de um caractere pode ser usado como sequência final:

>>> s.endswith('stop.')
True
>>> s.endswith('Stop.')
False

Você também pode usar uma tupla para verificar se ela termina com qualquer um de um conjunto de strings

>>> s.endswith(('.', 'something'))
True
>>> s.endswith(('ab', 'bc'))
False

Justifique as strings #

O Python fornece funções para justificar strings, permitindo o preenchimento de texto para facilitar o alinhamento de várias strings.

Abaixo está um exemplo de str.ljust e str.rjust:

interstates_lengths = {
    5: (1381, 2222),
    19: (63, 102),
    40: (2555, 4112),
    93: (189,305),
}
for road, length in interstates_lengths.items():
    miles,kms = length
    print('{} -> {} mi. ({} km.)'.format(str(road).rjust(4), str(miles).ljust(4), str(kms).ljust(4)))

  40 -> 2555 mi. (4112 km.)
  19 -> 63   mi. (102  km.)
   5 -> 1381 mi. (2222 km.)
  93 -> 189  mi. (305  km.)

ljust e rjust são muito semelhantes. Ambos têm um parâmetro width e um parâmetro opcional fillchar. Qualquer string criada por essas funções é pelo menos tão longa quanto o parâmetro width que foi passado para a função. Se a string já for maior que width, ela não será truncada. O argumento fillchar, cujo padrão é o caractere de espaço ' ' deve ser um único caractere, não uma string com vários caracteres.

A função ljust preenche o final da string que é chamada com o fillchar até que tenha caracteres width. A função rjust preenche o início da string de maneira semelhante. Portanto, o l e o r nos nomes dessas funções se referem ao lado em que a string original, não o fillchar, está posicionada na string de saída.

Conversão entre dados str ou bytes e caracteres unicode #

O conteúdo de arquivos e mensagens de rede pode representar caracteres codificados. Eles geralmente precisam ser convertidos em unicode para exibição adequada.

No Python 2, pode ser necessário converter dados str em caracteres Unicode. O padrão ('', "", etc.) é uma string ASCII, com quaisquer valores fora do intervalo ASCII exibidos como valores de escape. As strings Unicode são u'' (ou u"", etc.).

# You get "© abc" encoded in UTF-8 from a file, network, or other data source

s = '\xc2\xa9 abc'  # s is a byte array, not a string of characters
                    # Doesn't know the original was UTF-8
                    # Default form of string literals in Python 2
s[0]                # '\xc2' - meaningless byte (without context such as an encoding)
type(s)             # str - even though it's not a useful one w/o having a known encoding

u = s.decode('utf-8')  # u'\xa9 abc'
                       # Now we have a Unicode string, which can be read as UTF-8 and printed properly
                       # In Python 2, Unicode string literals need a leading u
                       # str.decode converts a string which may contain escaped bytes to a Unicode string
u[0]                # u'\xa9' - Unicode Character 'COPYRIGHT SIGN' (U+00A9) '©'
type(u)             # unicode

u.encode('utf-8')   # '\xc2\xa9 abc'
                    # unicode.encode produces a string with escaped bytes for non-ASCII characters

No Python 3, você pode precisar converter matrizes de bytes (referidas como ’literais de bytes’) em strings de caracteres Unicode. O padrão agora é uma string Unicode, e os literais de string de bytes agora devem ser inseridos como b'', b"", etc. Um literal de byte retornará True para isinstance(some_val, byte), assumindo some_val para ser uma string que pode ser codificada como bytes.

# You get from file or network "© abc" encoded in UTF-8

s = b'\xc2\xa9 abc' # s is a byte array, not characters
                    # In Python 3, the default string literal is Unicode; byte array literals need a leading b
s[0]                # b'\xc2' - meaningless byte (without context such as an encoding)
type(s)             # bytes - now that byte arrays are explicit, Python can show that.

u = s.decode('utf-8')  # '© abc' on a Unicode terminal
                       # bytes.decode converts a byte array to a string (which will, in Python 3, be Unicode)
u[0]                # '\u00a9' - Unicode Character 'COPYRIGHT SIGN' (U+00A9) '©'
type(u)             # str
                    # The default string literal in Python 3 is UTF-8 Unicode

u.encode('utf-8')   # b'\xc2\xa9 abc'
                    # str.encode produces a byte array, showing ASCII-range bytes as unescaped characters.
