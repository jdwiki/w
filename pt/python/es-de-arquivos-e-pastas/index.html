Quando se trata de armazenar, ler ou comunicar dados, trabalhar com os arquivos de um sistema operacional é necessário e fácil com o Python. Ao contrário de outras linguagens onde a entrada e saída de arquivos requerem leitura e escrita de objetos complexos, o Python simplifica o processo precisando apenas de comandos para abrir, ler/escrever e fechar o arquivo. Este tópico explica como o Python pode interagir com arquivos no sistema operacional.

Sintaxe #

  • file_object = open(filename [, access_mode][, buffering])

Parâmetros #

Parâmetro Detalhes
nome do arquivo o caminho para o seu arquivo ou, se o arquivo estiver no diretório de trabalho, o nome do arquivo do seu arquivo
access_mode um valor de string que determina como o arquivo é aberto
buffer um valor inteiro usado para buffer de linha opcional

Evitando o inferno de codificação multiplataforma #

Ao usar o open() interno do Python, é uma prática recomendada sempre passar o argumento encoding, se você pretende que seu código seja executado em várias plataformas. A razão para isso é que a codificação padrão de um sistema difere de plataforma para plataforma.

Embora os sistemas linux realmente usem utf-8 como padrão, isso não é necessariamente verdade para MAC e Windows.

Para verificar a codificação padrão de um sistema, tente isto:

import sys
sys.getdefaultencoding()

de qualquer interpretador python.

Modos de arquivo #

Existem diferentes modos com os quais você pode abrir um arquivo, especificados pelo parâmetro mode. Esses incluem:

  • 'r' - modo de leitura. O padrão. Permite apenas a leitura do arquivo, não para modificá-lo. Ao usar este modo, o arquivo deve existir.

  • 'w' - modo de escrita. Ele criará um novo arquivo se ele não existir, caso contrário, apagará o arquivo e permitirá que você grave nele.

  • 'a' - modo de acréscimo. Ele gravará dados no final do arquivo. Ele não apaga o arquivo e o arquivo deve existir para este modo.

  • 'rb' - modo de leitura em binário. Isso é semelhante ao r, exceto que a leitura é forçada no modo binário. Esta também é uma escolha padrão.

  • 'r+' - modo de leitura e modo de escrita ao mesmo tempo. Isso permite que você leia e escreva em arquivos ao mesmo tempo sem ter que usar r e w.

  • 'rb+' - modo de leitura e escrita em binário. O mesmo que r+ exceto que os dados estão em binário

  • 'wb' - modo de escrita em binário. O mesmo que w, exceto que os dados estão em binário.

  • 'w+' - modo de escrita e leitura. Exatamente o mesmo que r+, mas se o arquivo não existir, um novo será criado. Caso contrário, o arquivo será sobrescrito.

  • 'wb+' - modo de escrita e leitura em modo binário. O mesmo que w+ mas os dados estão em binário.

  • 'ab' - anexando em modo binário. Semelhante a a, exceto que os dados estão em binário.

  • 'a+' - modo de adição e leitura. Semelhante ao w+, pois criará um novo arquivo se o arquivo não existir. Caso contrário, o ponteiro do arquivo estará no final do arquivo, se ele existir.

  • 'ab+' - modo de adição e leitura em binário. O mesmo que a+ exceto que os dados estão em binário.

Lendo um arquivo linha por linha #

A maneira mais simples de iterar um arquivo linha por linha:

with open('myfile.txt', 'r') as fp:
    for line in fp:
        print(line)

readline() permite um controle mais granular sobre a iteração linha por linha. O exemplo abaixo é equivalente ao acima:

with open('myfile.txt', 'r') as fp:
    while True:
        cur_line = fp.readline()
        # If the result is an empty string
        if cur_line == '':
            # We have reached the end of the file
            break
        print(cur_line)

Usar o iterador de loop for e readline() juntos é considerado uma prática ruim.

Mais comumente, o método readlines() é usado para armazenar uma coleção iterável das linhas do arquivo:

with open("myfile.txt", "r") as fp:
    lines = fp.readlines()
for i in range(len(lines)):
    print("Line " + str(i) + ": " + line)

Isso imprimiria o seguinte:

Linha 0: Olá

Linha 1: mundo

Iterar arquivos (recursivamente) #

Para iterar todos os arquivos, inclusive em subdiretórios, use os.walk:

import os
for root, folders, files in os.walk(root_dir):
    for filename in files:
        print root, filename

root_dir pode ser “.” para iniciar a partir do diretório atual ou qualquer outro caminho para iniciar.

Se você também deseja obter informações sobre o arquivo, pode usar o método mais eficiente os.scandir assim:

for entry in os.scandir(path):
   if not entry.name.startswith('.') and entry.is_file():
       print(entry.name)

Obtendo o conteúdo completo de um arquivo #

O método preferido de arquivo i/o é usar a palavra-chave with. Isso garantirá que o identificador de arquivo seja fechado assim que a leitura ou gravação for concluída.

with open('myfile.txt') as in_file:
    content = in_file.read()

print(content)

ou, para lidar com o fechamento do arquivo manualmente, você pode abrir mão de with e simplesmente chamar close você mesmo:

in_file = open('myfile.txt', 'r')
content = in_file.read()
print(content)
in_file.close()

Tenha em mente que sem usar uma instrução with, você pode acidentalmente manter o arquivo aberto caso surja uma exceção inesperada como:

in_file = open('myfile.txt', 'r')
raise Exception("oops")
in_file.close()  # This will never be called

Gravando em um arquivo #

with open('myfile.txt', 'w') as f:
    f.write("Line 1")
    f.write("Line 2")
    f.write("Line 3")
    f.write("Line 4")

Se você abrir myfile.txt, verá que seu conteúdo é:

Linha 1Linha 2Linha 3Linha 4

O Python não adiciona quebras de linha automaticamente, você precisa fazer isso manualmente:

with open('myfile.txt', 'w') as f:
    f.write("Line 1\n")
    f.write("Line 2\n")
    f.write("Line 3\n")
    f.write("Line 4\n")

Linha 1 Linha 2 Linha 3 Linha 4

Não use os.linesep como um terminador de linha ao escrever arquivos abertos em modo texto (o padrão); use \n em vez disso.

Se você quiser especificar uma codificação, basta adicionar o parâmetro encoding à função open:

with open('my_file.txt', 'w', encoding='utf-8') as f:
    f.write('utf-8 text')

Também é possível usar a instrução print para gravar em um arquivo. A mecânica é diferente no Python 2 versus Python 3, mas o conceito é o mesmo, pois você pode pegar a saída que teria ido para a tela e enviá-la para um arquivo.

with open('fred.txt', 'w') as outfile:
    s = "I'm Not Dead Yet!"
    print(s) # writes to stdout
    print(s, file = outfile) # writes to outfile

    #Note: it is possible to specify the file parameter AND write to the screen
    #by making sure file ends up with a None value either directly or via a variable
    myfile = None
    print(s, file = myfile) # writes to stdout
    print(s, file = None)   # writes to stdout

Em Python 2 você teria feito algo como

outfile = open('fred.txt', 'w')
s = "I'm Not Dead Yet!"
print s   # writes to stdout
print >> outfile, s   # writes to outfile

Ao contrário de usar a função de gravação, a função de impressão adiciona automaticamente quebras de linha.

Verifique se existe um arquivo ou caminho #

Empregue o estilo de codificação EAFP e tente abri-lo.

import errno

try:
    with open(path) as f:
        # File exists
except IOError as e:
    # Raise the exception if it is not ENOENT (No such file or directory)
    if e.errno != errno.ENOENT:
        raise
    # No such file or directory

Isso também evitará condições de corrida se outro processo excluir o arquivo entre a verificação e quando ele for usado. Essa condição de corrida pode acontecer nos seguintes casos:

  • Usando o módulo os:

    import os
os.path.isfile('/path/to/some/file.txt')

  • Usando pathlib:

    import pathlib
path = pathlib.Path('/path/to/some/file.txt')
if path.is_file():
    ...

Para verificar se um determinado caminho existe ou não, você pode seguir o procedimento EAFP acima ou verificar explicitamente o caminho:

import os
path = "/home/myFiles/directory1"

if os.path.exists(path):
    ## Do stuff

Acesso aleatório a arquivos usando mmap #

O uso do módulo mmap permite que o usuário acesse aleatoriamente locais em um arquivo mapeando o arquivo na memória. Esta é uma alternativa ao uso de operações normais de arquivo.

import mmap

with open('filename.ext', 'r') as fd:
    # 0: map the whole file
    mm = mmap.mmap(fd.fileno(), 0)

    # print characters at indices 5 through 10
    print mm[5:10]

    # print the line starting from mm's current position
    print mm.readline()

    # write a character to the 5th index
    mm[5] = 'a'

    # return mm's position to the beginning of the file
    mm.seek(0)

    # close the mmap object
    mm.close()

Ler um arquivo entre um intervalo de linhas #

Então vamos supor que você queira iterar apenas entre algumas linhas específicas de um arquivo

Você pode usar itertools para isso

import itertools

with open('myfile.txt', 'r') as f:
    for line in itertools.islice(f, 12, 30):
        # do something here

Isso lerá as linhas de 13 a 20, pois em python a indexação começa em 0. Portanto, a linha número 1 é indexada como 0

Como também pode ler algumas linhas extras usando a palavra-chave next() aqui.

E quando você estiver usando o objeto de arquivo como um iterável, por favor, não use a instrução readline() aqui, pois as duas técnicas de percorrer um arquivo não devem ser misturadas

import os
def is_empty_file(fpath):
