Sintaxe #

  • != - Não é igual a

  • == - É igual a

  • > - maior que

  • < - menor que

  • >= - maior ou igual a

  • <= - menor ou igual a

  • is - testa se os objetos são exatamente o mesmo objeto

  • is not = teste se os objetos não são exatamente o mesmo objeto

Parâmetros #

Parâmetro Detalhes
x Primeiro item a ser comparado
e Segundo item a ser comparado

Comparações em cadeia #

Você pode comparar vários itens com vários operadores de comparação com comparação em cadeia. Por exemplo

x > y > z

é apenas uma forma abreviada de:

x > y and y > z

Isso será avaliado como True somente se ambas as comparações forem True.

A forma geral é

a OP b OP c OP d ...

Onde OP representa uma das múltiplas operações de comparação que você pode usar, e as letras representam expressões válidas arbitrárias.

Observe que 0 != 1 != 0 é avaliado como True, mesmo que 0 != 0 seja False. Ao contrário da notação matemática comum em que x != y != z significa que x, y e z têm valores diferentes. O encadeamento de operações == tem o significado natural na maioria dos casos, já que a igualdade é geralmente transitiva.

Estilo #

Não há limite teórico de quantos itens e operações de comparação você usa, desde que tenha a sintaxe adequada:

1 > -1 < 2 > 0.5 < 100 != 24

O acima retorna True se cada comparação retornar True. No entanto, usar encadeamento complicado não é um bom estilo. Um bom encadeamento será “direcional”, não mais complicado do que

1 > x > -4 > y != 8

Efeitos colaterais #

Assim que uma comparação retorna False, a expressão é avaliada imediatamente como False, ignorando todas as comparações restantes.

Observe que a expressão exp em a > exp > b será avaliada apenas uma vez, enquanto no caso de

a > exp and exp > b

exp será calculado duas vezes se a > exp for verdadeiro.

Comparação por is vs == #

Uma armadilha comum é confundir os operadores de comparação de igualdade is e ==.

a == b compara o valor de a e b.

a é b irá comparar as identidades de a e b.

Ilustrar:

a = 'Python is fun!'
b = 'Python is fun!'
a == b # returns True
a is b # returns False

a = [1, 2, 3, 4, 5]
b = a      # b references a
a == b     # True
a is b     # True
b = a[:]   # b now references a copy of a
a == b     # True
a is b     # False [!!]

Basicamente, is pode ser pensado como uma abreviação de id(a) == id(b).

Além disso, há peculiaridades do ambiente de tempo de execução que complicam ainda mais as coisas. Strings curtas e inteiros pequenos retornarão True quando comparados com is, devido à máquina Python tentar usar menos memória para objetos idênticos.

a = 'short'
b = 'short'
c = 5
d = 5
a is b # True
c is d # True

Mas strings mais longas e números inteiros maiores serão armazenados separadamente.

a = 'not so short'
b = 'not so short'
c = 1000
d = 1000
a is b # False
c is d # False

Você deve usar is para testar None:

if myvar is not None:
    # not None
    pass
if myvar is None:
    # None
    pass

Um uso de is é testar um “sentinela” (ou seja, um objeto único).

sentinel = object()
def myfunc(var=sentinel):
    if var is sentinel:
        # value wasn’t provided
        pass
    else:
        # value was provided
        pass

Maior ou menor que #

x > y
x < y

Esses operadores comparam dois tipos de valores, eles são os operadores menor que e maior que. Para números, isso simplesmente compara os valores numéricos para ver qual é maior:

12 > 4
# True
12 < 4
# False
1 < 4
# True

Para strings, eles serão comparados lexicograficamente, o que é semelhante à ordem alfabética, mas não exatamente o mesmo.

"alpha" < "beta"
# True
"gamma" > "beta"
# True
"gamma" < "OMEGA"
# False

Nessas comparações, letras minúsculas são consideradas ‘maiores que’ maiúsculas, razão pela qual "gamma" < "OMEGA" é falso. Se todos fossem maiúsculos, retornaria o resultado esperado da ordenação alfabética:

"GAMMA" < "OMEGA"
# True

Cada tipo define seu cálculo com os operadores < e > de forma diferente, então você deve investigar o que os operadores significam com um determinado tipo antes de usá-lo.

Não é igual a #

x != y

Isso retornará True se x e y não forem iguais e, caso contrário, retornará False.

12 != 1
# True
12 != '12'
# True
'12' != '12'
# False

Igual a #

x == y

Essa expressão avalia se x e y são o mesmo valor e retorna o resultado como um valor booleano. Geralmente, tanto o tipo quanto o valor precisam corresponder, então o int 12 não é o mesmo que a string '12'.

12 == 12
# True
12 == 1
# False
'12' == '12'
# True
'spam' == 'spam'
# True
'spam' == 'spam '
# False
'12' == 12
# False

Observe que cada tipo deve definir uma função que será usada para avaliar se dois valores são iguais. Para tipos internos, essas funções se comportam como você esperaria e apenas avaliam as coisas com base no mesmo valor. No entanto, os tipos personalizados podem definir o teste de igualdade como quiserem, incluindo sempre retornar True ou sempre retornar False.

Comparando Objetos #

Para comparar a igualdade de classes personalizadas, você pode substituir == e != definindo os métodos __eq__ e __ne__. Você também pode substituir __lt__ (<), __le__ (<=), __gt__ (>) e __ge__ (>). Observe que você só precisa substituir dois métodos de comparação, e o Python pode lidar com o resto (== é o mesmo que not < e not >, etc.)

class Foo(object):
    def __init__(self, item):
        self.my_item = item
    def __eq__(self, other):
        return self.my_item == other.my_item
    
a = Foo(5)
b = Foo(5)
a == b     # True
a != b     # False
a is b     # False

Observe que esta comparação simples assume que other (o objeto que está sendo comparado) é o mesmo tipo de objeto. Comparar com outro tipo gerará um erro:

class Bar(object):
    def __init__(self, item):
        self.other_item = item
    def __eq__(self, other):
        return self.other_item == other.other_item
    def __ne__(self, other):
        return self.other_item != other.other_item
    
c = Bar(5)
a == c    # throws AttributeError: 'Foo' object has no attribute 'other_item'

Marcar isinstance() ou similar ajudará a evitar isso (se desejado).

Pegadinha comum: Python não impõe digitação #

Em muitos outros idiomas, se você executar o seguinte (exemplo Java)

if("asgdsrf" == 0) {
    //do stuff
}

… você receberá um erro. Você não pode simplesmente comparar strings com inteiros assim. Em Python, esta é uma declaração perfeitamente legal - ela apenas resolverá como ‘False’.

Uma pegadinha comum é a seguinte

myVariable = "1"
if 1 == myVariable:
    #do stuff

Esta comparação será avaliada como ‘False’ sem erros, todas as vezes, potencialmente ocultando um bug ou quebrando uma condicional.
