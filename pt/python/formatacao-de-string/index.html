Ao armazenar e transformar dados para humanos verem, a formatação de strings pode se tornar muito importante. O Python oferece uma ampla variedade de métodos de formatação de strings descritos neste tópico.

Sintaxe #

  • “{}".format(42) ==> “42”

  • “{0}".format(42) ==> “42”

  • “{0:.2f}".format(42) ==> “42,00”

  • “{0:.0f}".format(42.1234) ==> “42”

  • “{answer}".format(no_answer=41, answer=42) ==> “42”

  • “{answer:.2f}".format(no_answer=41, answer=42) ==> “42,00”

  • “{[chave]}".format({‘chave’: ‘valor’}) ==> “valor”

  • “{1}".format([‘zero’, ‘um’, ‘dois’]) ==> “um”

  • “{answer} = {answer}".format(answer=42) ==> “42 = 42”

  • ’ ‘.join([‘stack’, ‘overflow’]) ==> “stack overflow”

  • Deve verificar PyFormat.info para uma introdução/explicação muito completa e gentil de como funciona.

Noções básicas de formatação de string #

foo = 1
bar = 'bar'
baz = 3.14

Você pode usar str.format para formatar a saída. Os pares de colchetes são substituídos por argumentos na ordem em que os argumentos são passados:

print('{}, {} and {}'.format(foo, bar, baz))
# Out: "1, bar and 3.14"

Os índices também podem ser especificados dentro dos colchetes. Os números correspondem aos índices dos argumentos passados ​​para a função str.format (baseada em 0).

print('{0}, {1}, {2}, and {1}'.format(foo, bar, baz))
# Out: "1, bar, 3.14, and bar"
print('{0}, {1}, {2}, and {3}'.format(foo, bar, baz))
# Out: index out of range error

Os argumentos nomeados também podem ser usados:

print("X value is: {x_val}. Y value is: {y_val}.".format(x_val=2, y_val=3))
# Out: "X value is: 2. Y value is: 3."

Atributos de objetos podem ser referenciados quando passados ​​para str.format:

class AssignValue(object):
    def __init__(self, value):
        self.value = value
my_value = AssignValue(6)
print('My value is: {0.value}'.format(my_value))  # "0" is optional
# Out: "My value is: 6"

As chaves do dicionário também podem ser usadas:

my_dict = {'key': 6, 'other_key': 7}
print("My other key is: {0[other_key]}".format(my_dict))  # "0" is optional
# Out: "My other key is: 7"

O mesmo se aplica aos índices de lista e tupla:

my_list = ['zero', 'one', 'two']
print("2nd element is: {0[2]}".format(my_list))  # "0" is optional
# Out: "2nd element is: two"

Nota: Além de str.format, o Python também fornece o operador de módulo %–também conhecido como string formatação ou operador de interpolação (consulte PEP 3101)–para formatação de strings. str.format é um sucessor de % e oferece maior flexibilidade, por exemplo, tornando mais fácil realizar várias substituições.

Além dos índices de argumento, você também pode incluir uma especificação de formato dentro dos colchetes. Esta é uma expressão que segue regras especiais e deve ser precedida por dois pontos (:). Consulte os docs para obter uma descrição completa da especificação do formato. Um exemplo de especificação de formato é a diretiva de alinhamento :~^20 (^ significa alinhamento central, largura total 20, preenchimento com caractere ~):

'{:~^20}'.format('centered')
# Out: '~~~~~~centered~~~~~~'

format permite comportamento que não é possível com %, por exemplo repetição de argumentos:

t = (12, 45, 22222, 103, 6)
print '{0} {2} {1} {2} {3} {2} {4} {2}'.format(*t)    
# Out: 12 22222 45 22222 103 22222 6 22222

Como format é uma função, pode ser usado como argumento em outras funções:

number_list = [12,45,78]
print map('the number is {}'.format, number_list)
# Out: ['the number is 12', 'the number is 45', 'the number is 78']   
    

from datetime import datetime,timedelta
    
once_upon_a_time = datetime(2010, 7, 1, 12, 0, 0)
delta = timedelta(days=13, hours=8,  minutes=20)
    
gen = (once_upon_a_time + x * delta for x in xrange(5))
    
print '\n'.join(map('{:%Y-%m-%d %H:%M:%S}'.format, gen))
#Out: 2010-07-01 12:00:00
#     2010-07-14 20:20:00
#     2010-07-28 04:40:00
#     2010-08-10 13:00:00
#     2010-08-23 21:20:00

Alinhamento e preenchimento #

O método format() pode ser usado para alterar o alinhamento da string. Você tem que fazer isso com uma expressão de formato da forma :[fill_char][align_operator][width] onde align_operator é um dos seguintes:

  • < força o campo a ser alinhado à esquerda dentro de width.
  • > força o campo a ser alinhado à direita dentro de width.
  • ^ força o campo a ser centralizado dentro de width.
  • = força o preenchimento a ser colocado após o sinal (somente tipos numéricos).

fill_char (se omitido, o padrão é espaço em branco) é o caractere usado para o preenchimento.

'{:~<9s}, World'.format('Hello')
# 'Hello~~~~, World'

'{:~>9s}, World'.format('Hello')
# '~~~~Hello, World'

'{:~^9s}'.format('Hello')
# '~~Hello~~'

'{:0=6d}'.format(-123)
# '-00123'

Nota: você pode obter os mesmos resultados usando as funções de string ljust(), rjust(), center(), zfill(), porém essas funções estão obsoletas desde a versão 2.5.

Formatar literais (f-string) #

Strings de formato literal foram introduzidas em PEP 498 (Python3.6 e superior), permitindo que você anexe f ao início de um string literal para aplicar .format efetivamente a ele com todas as variáveis ​​no escopo atual.

>>> foo = 'bar'
>>> f'Foo is {foo}'
'Foo is bar'

Isso também funciona com strings de formato mais avançado, incluindo alinhamento e notação de ponto.

>>> f'{foo:^7s}'
'  bar  '

Nota: O f'' não denota um tipo específico como b'' para bytes ou u'' para unicode em python2. A formatação é aplicada imediatamente, resultando em uma agitação normal.

As strings de formato também podem ser aninhadas:

>>> price = 478.23
>>> f"{f'${price:0.2f}':*>20s}"
'*************$478.23'

As expressões em uma string f são avaliadas na ordem da esquerda para a direita. Isso é detectável apenas se as expressões tiverem efeitos colaterais:

>>> def fn(l, incr):
...    result = l[0]
...    l[0] += incr
...    return result
...
>>> lst = [0]
>>> f'{fn(lst,2)} {fn(lst,3)}'
'0 2'
>>> f'{fn(lst,2)} {fn(lst,3)}'
'5 7'
>>> lst
[10]

Formatação flutuante #

>>> '{0:.0f}'.format(42.12345)
'42'

>>> '{0:.1f}'.format(42.12345)
'42.1'

>>> '{0:.3f}'.format(42.12345)
'42.123'

>>> '{0:.5f}'.format(42.12345)
'42.12345'

>>> '{0:.7f}'.format(42.12345)
'42.1234500'

Mesma espera para outra forma de referência:

>>> '{:.3f}'.format(42.12345)
'42.123'

>>> '{answer:.3f}'.format(answer=42.12345)
'42.123'

Os números de ponto flutuante também podem ser formatados em notação científica ou como porcentagens:

>>> '{0:.3e}'.format(42.12345)
'4.212e+01'

>>> '{0:.0%}'.format(42.12345)
'4212%'

Você também pode combinar as notações {0} e {name}. Isso é especialmente útil quando você deseja arredondar todas as variáveis ​​para um número pré-especificado de decimais com 1 declaração:

>>> s = 'Hello'
>>> a, b, c = 1.12345, 2.34567, 34.5678
>>> digits = 2

>>> '{0}! {1:.{n}f}, {2:.{n}f}, {3:.{n}f}'.format(s, a, b, c, n=digits)
'Hello! 1.12, 2.35, 34.57'

Formatação de string com data e hora #

Qualquer classe pode configurar sua própria sintaxe de formatação de string através do método __format__. Um tipo na biblioteca padrão do Python que faz uso prático disso é o tipo datetime, onde se pode usar códigos de formatação do tipo strftime diretamente dentro de str.format:

>>> from datetime import datetime
>>> 'North America: {dt:%m/%d/%Y}.  ISO: {dt:%Y-%m-%d}.'.format(dt=datetime.now())
'North America: 07/21/2016.  ISO: 2016-07-21.'

Uma lista completa de formatadores de data e hora pode ser encontrada na documentação oficial.

Espaços reservados nomeados #

As strings de formato podem conter espaços reservados nomeados que são interpolados usando argumentos de palavras-chave para format.

Usando um dicionário (Python 2.x) #

>>> data = {'first': 'Hodor', 'last': 'Hodor!'}
>>> '{first} {last}'.format(**data)
'Hodor Hodor!'

Usando um dicionário (Python 3.2+) #

>>> '{first} {last}'.format_map(data)
'Hodor Hodor!'

str.format_map permite usar dicionários sem ter que descompactá-los primeiro. Além disso, a classe de data (que pode ser um tipo personalizado) é usada em vez de um dict recém-preenchido.

Sem dicionário: #

>>> '{first} {last}'.format(first='Hodor', last='Hodor!')
'Hodor Hodor!'

Formate usando Getitem e Getattr #

Qualquer estrutura de dados que suporte __getitem__ pode ter sua estrutura aninhada formatada:

person = {'first': 'Arthur', 'last': 'Dent'} 
'{p[first]} {p[last]}'.format(p=person) 
# 'Arthur Dent'

Atributos de objetos podem ser acessados ​​usando getattr():

class Person(object):
    first = 'Zaphod'
    last = 'Beeblebrox'

'{p.first} {p.last}'.format(p=Person())
# 'Zaphod Beeblebrox'

Formatando Valores Numéricos #

O método .format() pode interpretar um número em diferentes formatos, como:

>>> '{:c}'.format(65)    # Unicode character
'A'

>>> '{:d}'.format(0x0a)  # base 10
'10'

>>> '{:n}'.format(0x0a)  # base 10 using current locale for separators
'10'

Formatar inteiros para bases diferentes (hex, oct, binário)

>>> '{0:x}'.format(10) # base 16, lowercase - Hexadecimal
'a'

>>> '{0:X}'.format(10) # base 16, uppercase - Hexadecimal
'A'

>>> '{:o}'.format(10) # base 8 - Octal
'12'

>>> '{:b}'.format(10) # base 2 - Binary
'1010'

>>> '{0:#b}, {0:#o}, {0:#x}'.format(42) # With prefix
'0b101010, 0o52, 0x2a'

>>> '8 bit: {0:08b}; Three bytes: {0:06x}'.format(42) # Add zero padding
'8 bit: 00101010; Three bytes: 00002a'

Use a formatação para converter uma tupla flutuante RGB em uma string hexadecimal colorida:

>>> r, g, b = (1.0, 0.4, 0.0)
>>> '#{:02X}{:02X}{:02X}'.format(int(255 * r), int(255 * g), int(255 * b))
'#FF6600'

Apenas números inteiros podem ser convertidos:

>>> '{:x}'.format(42.0)
Traceback (most recent call last):
  File "<stdin>", line 1, in <module>
ValueError: Unknown format code 'x' for object of type 'float'

Formatação aninhada #

Alguns formatos podem receber parâmetros adicionais, como a largura da string formatada ou o alinhamento:

>>> '{:.>10}'.format('foo')
'.......foo'

Eles também podem ser fornecidos como parâmetros para format aninhando mais {} dentro do {}:

>>> '{:.>{}}'.format('foo', 10)
'.......foo'
'{:{}{}{}}'.format('foo', '*', '^', 15)
'******foo******'

No último exemplo, a string de formato '{:{}{}{}}' é modificada para '{:*^15}' (ou seja, “center and pad with * to total length of 15”) antes aplicando-o à string real 'foo' a ser formatada dessa maneira.

Isso pode ser útil nos casos em que os parâmetros não são conhecidos de antemão, por exemplo, ao alinhar dados tabulares:

>>> data = ["a", "bbbbbbb", "ccc"]
>>> m = max(map(len, data))
>>> for d in data:
...     print('{:>{}}'.format(d, m))
      a
bbbbbbb
    ccc

Preenchimento e truncamento de strings, combinados #

Digamos que você queira imprimir variáveis ​​em uma coluna de 3 caracteres.

Nota: duplicar { e } escapa deles.

s = """

{{:3}}             :{a:3}:

truncate
{{:.3}}            :{e:.3}:

combined
{{:>3.3}}          :{a:>3.3}:
{{:3.3}}           :{a:3.3}:
{{:3.3}}           :{c:3.3}:
{{:3.3}}           :{e:3.3}:

Formatação personalizada para uma classe #

Observação:

Tudo abaixo se aplica ao método str.format, bem como ao método função formatar. No texto abaixo, os dois são intercambiáveis.

Para cada valor que é passado para a função format, o Python procura um método __format__ para esse argumento. Sua própria classe personalizada pode, portanto, ter seu próprio método __format__ para determinar como a função format irá exibir e formatar sua classe e seus atributos.

Isso é diferente do método __str__, pois no método __format__ você pode levar em consideração a linguagem de formatação, incluindo alinhamento, largura do campo etc., e até mesmo (se desejar) implementar seus próprios especificadores de formato e sua própria formatação extensões de idioma.1

object.__format__(self, format_spec)

Por exemplo :

# Example in Python 2 - but can be easily applied to Python 3

class Example(object):
    def __init__(self,a,b,c):
        self.a, self.b, self.c = a,b,c

    def __format__(self, format_spec):
        """ Implement special semantics for the 's' format specifier """
        # Reject anything that isn't an s
        if format_spec[-1] != 's':
            raise ValueError('{} format specifier not understood for this object', format_spec[:-1])

        # Output in this example will be (<a>,<b>,<c>)
        raw = "(" + ",".join([str(self.a), str(self.b), str(self.c)]) + ")"
        # Honor the format language by using the inbuilt string format
        # Since we know the original format_spec ends in an 's' 
        # we can take advantage of the str.format method with a 
        # string argument we constructed above
        return "{r:{f}}".format( r=raw, f=format_spec )

inst = Example(1,2,3)
print "{0:>20s}".format( inst )
# out :              (1,2,3)
# Note how the right align and field width of 20 has been honored.

Observação:

Se sua classe personalizada não tiver um método __format__ personalizado e um instância da classe é passada para a função format, Python2 sempre usará o valor de retorno do método __str__ ou __repr__ método para determinar o que imprimir (e se nenhum existir, então o padrão repr será usado), e você precisará usar o formato s especificador para formatar isso. Com Python3, para passar sua classe personalizada para a função format, você precisará definir o método __format__ sua classe personalizada.
