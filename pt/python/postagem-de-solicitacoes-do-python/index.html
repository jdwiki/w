Documentação para o módulo Python Requests no contexto do método HTTP POST e sua função Requests correspondente

Postagem simples #

from requests import post

foo = post('http://httpbin.org/post', data = {'key':'value'})

Irá realizar uma operação HTTP POST simples. Os dados postados podem estar na maioria dos formatos, no entanto, os pares de valores-chave são mais prevalentes.

Cabeçalhos

Os cabeçalhos podem ser visualizados:

print(foo.headers)

Um exemplo de resposta:

{'Content-Length': '439', 'X-Processed-Time': '0.000802993774414', 'X-Powered-By': 'Flask', 'Server': 'meinheld/0.6.1', 'Connection': 'keep-alive', 'Via': '1.1 vegur', 'Access-Control-Allow-Credentials': 'true', 'Date': 'Sun, 21 May 2017 20:56:05 GMT', 'Access-Control-Allow-Origin': '*', 'Content-Type': 'application/json'}

Os cabeçalhos também podem ser preparados antes da postagem:

headers = {'Cache-Control':'max-age=0',
        'Upgrade-Insecure-Requests':'1',
        'User-Agent':'Mozilla/5.0 (Windows NT 10.0; Win64; x64) AppleWebKit/537.36 (KHTML, like Gecko) Chrome/54.0.2840.99 Safari/537.36',
        'Content-Type':'application/x-www-form-urlencoded',
        'Accept':'text/html,application/xhtml+xml,application/xml;q=0.9,image/webp,*/*;q=0.8',
        'Referer':'https://www.groupon.com/signup',
        'Accept-Encoding':'gzip, deflate, br',
        'Accept-Language':'es-ES,es;q=0.8'
        }

 foo = post('http://httpbin.org/post', headers=headers, data = {'key':'value'})

Codificação

A codificação pode ser definida e visualizada da mesma maneira:

 print(foo.encoding)

'utf-8'

foo.encoding = 'ISO-8859-1'

Verificação SSL

Solicitações por padrão validam certificados SSL de domínios. Isso pode ser substituído:

foo = post('http://httpbin.org/post', data = {'key':'value'}, verify=False)

Redirecionamento

Qualquer redirecionamento será seguido (por exemplo, http para https), isso também pode ser alterado:

foo = post('http://httpbin.org/post', data = {'key':'value'}, allow_redirects=False)

Se a operação post foi redirecionada, este valor pode ser acessado:

print(foo.url)

Um histórico completo de redirecionamentos pode ser visualizado:

print(foo.history)

Dados codificados de formulário #

from requests import post

payload = {'key1' : 'value1',
           'key2' : 'value2'
           }

foo = post('http://httpbin.org/post', data=payload)

Para passar dados codificados de formulário com a operação post, os dados devem ser estruturados como dicionário e fornecidos como parâmetro de dados.

Se os dados não quiserem ser codificados por formulário, basta passar uma string ou um inteiro para o parâmetro de dados.

Forneça o dicionário para o parâmetro json para Requests formatar os dados automaticamente:

from requests import post

payload = {'key1' : 'value1', 'key2' : 'value2'}

foo = post('http://httpbin.org/post', json=payload)

Upload de arquivo #

Com o módulo Requests, só é necessário fornecer um handle de arquivo ao invés do conteúdo recuperado com .read():

from requests import post

files = {'file' : open('data.txt', 'rb')}

foo = post('http://http.org/post', files=files)

Nome de arquivo, tipo_de_conteúdo e cabeçalhos também podem ser definidos:

files = {'file': ('report.xls', open('report.xls', 'rb'), 'application/vnd.ms-excel', {'Expires': '0'})}

foo = requests.post('http://httpbin.org/post', files=files)

Strings também podem ser enviadas como um arquivo, desde que sejam fornecidas como parâmetro files.

Vários arquivos

Vários arquivos podem ser fornecidos da mesma maneira que um arquivo:

multiple_files = [
    ('images', ('foo.png', open('foo.png', 'rb'), 'image/png')),
    ('images', ('bar.png', open('bar.png', 'rb'), 'image/png'))]

foo = post('http://httpbin.org/post', files=multiple_files)

Respostas #

Os códigos de resposta podem ser visualizados em uma operação de postagem:

from requests import post

foo = post('http://httpbin.org/post', data={'data' : 'value'})
print(foo.status_code)

Dados retornados

Acessando dados que são retornados:

foo = post('http://httpbin.org/post', data={'data' : 'value'})
print(foo.text)

Respostas brutas

Nos casos em que você precisa acessar o objeto urllib3 response.HTTPResponse subjacente, isso pode ser feito da seguinte maneira:

foo = post('http://httpbin.org/post', data={'data' : 'value'})
res = foo.raw

print(res.read())

Autenticação #

Autenticação HTTP simples

A autenticação HTTP simples pode ser obtida com o seguinte:

from requests import post

foo = post('http://natas0.natas.labs.overthewire.org', auth=('natas0', 'natas0'))

Esta é tecnicamente uma mão curta para o seguinte:

from requests import post
from requests.auth import HTTPBasicAuth

foo = post('http://natas0.natas.labs.overthewire.org', auth=HTTPBasicAuth('natas0', 'natas0'))

Autenticação de resumo HTTP

A autenticação HTTP Digest é feita de maneira muito semelhante, Requests fornece um objeto diferente para isso:

from requests import post
from requests.auth import HTTPDigestAuth

foo = post('http://natas0.natas.labs.overthewire.org', auth=HTTPDigestAuth('natas0', 'natas0'))

Autenticação personalizada

Em alguns casos, os mecanismos de autenticação integrados podem não ser suficientes, imagine este exemplo:

Um servidor é configurado para aceitar a autenticação se o remetente tiver a string correta do agente do usuário, um determinado valor de cabeçalho e fornecer as credenciais corretas por meio da autenticação básica HTTP. Para conseguir isso, uma classe de autenticação personalizada deve ser preparada, subclassificando AuthBase, que é a base para implementações de autenticação de Requests:

from requests.auth import AuthBase
from requests.auth import _basic_auth_str
from requests._internal_utils import to_native_string

class CustomAuth(AuthBase):

    def __init__(self, secret_header, user_agent , username, password):
        # setup any auth-related data here
        self.secret_header =  secret_header
        self.user_agent = user_agent
        self.username = username
        self.password = password

    def __call__(self, r):
        # modify and return the request
        r.headers['X-Secret'] = self.secret_header
        r.headers['User-Agent'] = self.user_agent
        r.headers['Authorization'] = _basic_auth_str(self.username, self.password)
        
        return r

Isso pode ser usado com o seguinte código:

foo = get('http://test.com/admin', auth=CustomAuth('SecretHeader', 'CustomUserAgent', 'user', 'password' ))

Proxies #

Cada operação POST de solicitação pode ser configurada para usar proxies de rede

Proxies HTTP/S

from requests import post

proxies = {
  'http': 'http://192.168.0.128:3128',
  'https': 'http://192.168.0.127:1080',
   }

foo = requests.post('http://httpbin.org/post', proxies=proxies)

A autenticação básica HTTP pode ser fornecida desta maneira:

proxies = {'http': 'http://user:pass@192.168.0.128:312'}
foo = requests.post('http://httpbin.org/post', proxies=proxies)

Proxies SOCKS

O uso de proxies de meias requer dependências de terceiros requests[socks], uma vez que proxies de meias instalados são usados ​​de maneira muito semelhante ao HTTPBasicAuth:

proxies = {
'http': 'socks5://user:pass@host:port',
'https': 'socks5://user:pass@host:port'
}

foo = requests.post('http://httpbin.org/post', proxies=proxies)
