Embora os gerenciadores de contexto do Python sejam amplamente usados, poucos entendem o propósito por trás de seu uso. Essas instruções, comumente usadas com leitura e gravação de arquivos, auxiliam o aplicativo na conservação da memória do sistema e melhoram o gerenciamento de recursos, garantindo que recursos específicos sejam usados ​​apenas para determinados processos. Este tópico explica e demonstra o uso dos gerenciadores de contexto do Python.

Sintaxe #

  • com “context_manager”(como “alias”)(, “context_manager”(como “alias”)?)*:

Os gerenciadores de contexto são definidos em PEP 343. Eles se destinam a ser usados ​​como um mecanismo mais sucinto para gerenciamento de recursos do que as construções try...finally. A definição formal é a seguinte.

Neste PEP, gerenciadores de contexto fornecem métodos __enter__() e __exit__() que são invocados na entrada e saída do corpo da instrução with.

Em seguida, ele passa a definir a instrução with da seguinte forma.

com EXPR como VAR: BLOQUEAR

A tradução da afirmação acima é:

gerente = (EXPR) exit = type(mgr).exit # Não está chamando ainda valor = tipo(mgr).enter(mgr) ex = Verdadeiro tente: tente: VAR = valor # Somente se “como VAR” estiver presente BLOQUEAR exceto:

O caso excepcional é tratado aqui #

ex = Falso se não exit(mgr, *sys.exc_info()): levantar

A exceção é engolida se exit() retornar true #

finalmente:

Introdução aos gerenciadores de contexto e à instrução with #

Um gerenciador de contexto é um objeto que é notificado quando um contexto (um bloco de código) começa e termina. Você geralmente usa um com a instrução with. Cuida da notificação.

Por exemplo, objetos de arquivo são gerenciadores de contexto. Quando um contexto termina, o objeto de arquivo é fechado automaticamente:

open_file = open(filename)
with open_file:
    file_contents = open_file.read()

# the open_file object has automatically been closed.

O exemplo acima geralmente é simplificado usando a palavra-chave as:

with open(filename) as open_file:
    file_contents = open_file.read()

# the open_file object has automatically been closed.

Qualquer coisa que termine a execução do bloco faz com que o método exit do gerenciador de contexto seja chamado. Isso inclui exceções e pode ser útil quando um erro faz com que você saia prematuramente de um arquivo ou conexão aberta. Sair de um script sem fechar corretamente os arquivos/conexões é uma má ideia, que pode causar perda de dados ou outros problemas. Ao usar um gerenciador de contexto, você pode garantir que as precauções sejam sempre tomadas para evitar danos ou perdas dessa maneira. Esse recurso foi adicionado no Python 2.5.

Escrevendo seu próprio gerenciador de contexto #

Um gerenciador de contexto é qualquer objeto que implemente dois métodos mágicos __enter__() e __exit__() (embora possa implementar outros métodos também):

class AContextManager():

    def __enter__(self):
        print("Entered")
        # optionally return an object
        return "A-instance"

    def __exit__(self, exc_type, exc_value, traceback):
        print("Exited" + (" (with an exception)" if exc_type else ""))
        # return True if you want to suppress the exception

Se o contexto sair com uma exceção, as informações sobre essa exceção serão passadas como um triplo exc_type, exc_value, traceback (estas são as mesmas variáveis ​​retornadas pela função sys.exc_info()). Se o contexto sair normalmente, todos os três argumentos serão Nenhum.

Se ocorrer uma exceção e for passada para o método __exit__, o método pode retornar True para suprimir a exceção, ou a exceção será reativada no final da função __exit__.

with AContextManager() as a:
    print("a is %r" % a)
# Entered
# a is 'A-instance'
# Exited

with AContextManager() as a:
    print("a is %d" % a)
# Entered
# Exited (with an exception)
# Traceback (most recent call last):
#   File "<stdin>", line 2, in <module>
# TypeError: %d format: a number is required, not str

Observe que no segundo exemplo, embora ocorra uma exceção no meio do corpo da instrução with, o manipulador __exit__ ainda é executado, antes que a exceção se propague para o escopo externo.

Se você precisa apenas de um método __exit__, você pode retornar a instância do gerenciador de contexto:

class MyContextManager:
    def __enter__(self):
        return self

    def __exit__(self):
        print('something')

Escrevendo seu próprio gerenciador de contexto usando a sintaxe do gerador #

Também é possível escrever um gerenciador de contexto usando a sintaxe do gerador graças ao decorador contextlib.contextmanager:

import contextlib

@contextlib.contextmanager
def context_manager(num):
    print('Enter')
    yield num + 1
    print('Exit')

with context_manager(2) as cm:
    # the following instructions are run when the 'yield' point of the context
    # manager is reached.
    # 'cm' will have the value that was yielded
    print('Right in the middle with cm = {}'.format(cm))

produz:

Enter
Right in the middle with cm = 3
Exit

O decorador simplifica a tarefa de escrever um gerenciador de contexto convertendo um gerador em um. Tudo antes da expressão yield se torna o método __enter__, o valor gerado se torna o valor retornado pelo gerador (que pode ser vinculado a uma variável na instrução with) e tudo após a expressão yield se torna o método __exit__.

Se uma exceção precisa ser tratada pelo gerenciador de contexto, um bloco try..except..finally pode ser escrito no gerador e qualquer exceção levantada no bloco with será tratada por este bloco de exceção.

@contextlib.contextmanager
def error_handling_context_manager(num):
    print("Enter")
    try:
        yield num + 1
    except ZeroDivisionError:
        print("Caught error")
    finally:
        print("Cleaning up")
    print("Exit")

with error_handling_context_manager(-1) as cm:
    print("Dividing by cm = {}".format(cm))
    print(2 / cm)

Isso produz:

Enter
Dividing by cm = 0
Caught error
Cleaning up
Exit

Vários gerenciadores de contexto #

Você pode abrir vários gerenciadores de conteúdo ao mesmo tempo:

with open(input_path) as input_file, open(output_path, 'w') as output_file:

    # do something with both files. 
    
    # e.g. copy the contents of input_file into output_file
    for line in input_file:
        output_file.write(line + '\n')

Ele tem o mesmo efeito que os gerenciadores de contexto de aninhamento:

with open(input_path) as input_file:
    with open(output_path, 'w') as output_file:
        for line in input_file:
            output_file.write(line + '\n')

Atribuindo a um destino #

Muitos gerenciadores de contexto retornam um objeto quando inseridos. Você pode atribuir esse objeto a um novo nome na instrução with.

Por exemplo, usar uma conexão de banco de dados em uma instrução with pode fornecer um objeto cursor:

with database_connection as cursor:
    cursor.execute(sql_query)

Objetos de arquivo retornam a si mesmos, isso possibilita tanto abrir o objeto de arquivo quanto usá-lo como gerenciador de contexto em uma expressão:

with open(filename) as open_file:
    file_contents = open_file.read()

Gerenciar recursos #

class File():
    def __init__(self, filename, mode):
        self.filename = filename
        self.mode = mode

    def __enter__(self):
        self.open_file = open(self.filename, self.mode)
        return self.open_file

    def __exit__(self, *args):
        self.open_file.close()

O método __init__() configura o objeto, neste caso configurando o nome do arquivo e o modo para abrir o arquivo. __enter__() abre e retorna o arquivo e __exit__() apenas o fecha.

Usar esses métodos mágicos (__enter__, __exit__) permite que você implemente objetos que podem ser usados ​​facilmente com a instrução with.

Use a classe Arquivo:

for _ in range(10000):
    with File('foo.txt', 'w') as f:
        f.write('foo')
