Começando #

Python é uma linguagem de programação de alto nível amplamente utilizada para programação de uso geral, criada por Guido van Rossum e lançada pela primeira vez em 1991. Python apresenta um sistema de tipo dinâmico e gerenciamento automático de memória e suporta vários paradigmas de programação, incluindo orientado a objetos, imperativo, programação funcional e estilos procedurais. Possui uma grande e abrangente biblioteca padrão.

Duas versões principais do Python estão atualmente em uso ativo:

  • Python 3.x é a versão atual e está em desenvolvimento ativo.
  • Python 2.x é a versão legada e receberá apenas atualizações de segurança até 2020. Nenhum novo recurso será implementado. Observe que muitos projetos ainda usam o Python 2, embora a migração para o Python 3 esteja ficando mais fácil.

Você pode baixar e instalar qualquer versão do Python aqui. Veja [Python 3 vs. Python 2][comparação de python] para uma comparação entre eles. Além disso, alguns terceiros oferecem versões reempacotadas do Python que adicionam bibliotecas comumente usadas e outros recursos para facilitar a configuração para casos de uso comuns, como matemática, análise de dados ou uso científico. Criando variáveis ​​e atribuindo valores #

Para criar uma variável em Python, tudo o que você precisa fazer é especificar o nome da variável e atribuir um valor a ela.

<variable name> = <value>

Python usa = para atribuir valores a variáveis. Não há necessidade de declarar uma variável antecipadamente (ou atribuir um tipo de dado a ela), atribuir um valor a uma variável em si declara e inicializa a variável com esse valor. Não há como declarar uma variável sem atribuir a ela um valor inicial.

# Integer
a = 2
print(a)
# Output: 2

# Integer    
b = 9223372036854775807
print(b)
# Output: 9223372036854775807

# Floating point
pi = 3.14
print(pi)
# Output: 3.14

# String
c = 'A'
print(c)
# Output: A

# String    
name = 'John Doe'
print(name)
# Output: John Doe

# Boolean    
q = True
print(q)
# Output: True

# Empty value or null data type
x = None
print(x)
# Output: None

A atribuição de variáveis ​​funciona da esquerda para a direita. Portanto, o seguinte fornecerá um erro de sintaxe.

0 = x
=> Output: SyntaxError: can't assign to literal

Você não pode usar as palavras-chave do python como um nome de variável válido. Você pode ver a lista de palavras-chave por:

Regras para nomenclatura de variáveis:

  1. Os nomes das variáveis ​​devem começar com uma letra ou sublinhado.

    x = True # valid _y = True # valid

    9x = False # starts with numeral => SyntaxError: invalid syntax

    $y = False # starts with symbol => SyntaxError: invalid syntax

  2. O restante do nome da variável pode consistir em letras, números e sublinhados.

    has_0_in_it = “Still Valid”

  3. Os nomes diferenciam maiúsculas de minúsculas.

    x = 9
    y = X*5
    =>NameError: name ‘X’ is not defined

Mesmo que não seja necessário especificar um tipo de dados ao declarar uma variável em Python, enquanto aloca a área necessária na memória para a variável, o interpretador Python escolhe automaticamente o tipo interno mais adequado para ela:

Agora que você conhece o básico da atribuição, vamos tirar essa sutileza sobre a atribuição em python do caminho.

Quando você usa = para fazer uma operação de atribuição, o que está à esquerda de = é um nome para o objeto à direita. Finalmente, o que = faz é atribuir a referência do objeto à direita ao nome à esquerda.

Aquilo é:

a_name = an_object  # "a_name" is now a name for the reference to the object "an_object"

Então, de muitos exemplos de atribuição acima, se escolhermos pi = 3.14, então pi é um nome (não o nome, já que um objeto pode ter vários nomes) para o objeto 3.14 . Se você não entender algo abaixo, volte a este ponto e leia novamente! Além disso, você pode dar uma olhada em this para uma melhor compreensão.

Você pode atribuir vários valores a várias variáveis ​​em uma linha. Observe que deve haver o mesmo número de argumentos nos lados direito e esquerdo do operador =:

a, b, c = 1, 2, 3
print(a, b, c)
# Output: 1 2 3

a, b, c = 1, 2
=> Traceback (most recent call last):

=> ValueError: need more than 2 values to unpack

=> Traceback (most recent call last):

=>     a, b = 1, 2, 3
=> ValueError: too many values to unpack

O erro no último exemplo pode ser evitado atribuindo valores restantes a um número igual de variáveis ​​arbitrárias. Essa variável fictícia pode ter qualquer nome, mas é convencional usar o sublinhado (_) para atribuir valores indesejados:

a, b, _ = 1, 2, 3
print(a, b)
# Output: 1, 2

Observe que o número de _ e o número de valores restantes devem ser iguais. Caso contrário, ‘muitos valores para descompactar o erro’ é lançado como acima:

Você também pode atribuir um único valor a várias variáveis ​​simultaneamente.

a = b = c = 1
print(a, b, c)
# Output: 1 1 1

Ao usar tal atribuição em cascata, é importante notar que todas as três variáveis ​​a, b e c referem-se ao mesmo objeto na memória, um objeto int com o valor 1. Em outras palavras , a, b e c são três nomes diferentes dados ao mesmo objeto int. Atribuir um objeto diferente a um deles posteriormente não altera os outros, exatamente como esperado:

a = b = c = 1    # all three names a, b and c refer to same int object with value 1
print(a, b, c)
# Output: 1 1 1
b = 2            # b now refers to another int object, one with a value of 2
print(a, b, c)
# Output: 1 2 1  # so output is as expected.

O acima também é verdade para tipos mutáveis ​​(como list, dict, etc.) assim como é verdade para tipos imutáveis ​​(como int, string, tuple, etc.):

x = y = [7, 8, 9]   # x and y refer to the same list object just created, [7, 8, 9]
x = [13, 8, 9]      # x now refers to a different list object just created, [13, 8, 9]
print(y)            # y still refers to the list it was first assigned
# Output: [7, 8, 9]

Até agora tudo bem. As coisas são um pouco diferentes quando se trata de modificar o objeto (em contraste com atribuir o nome a um objeto diferente, o que fizemos acima) quando a atribuição em cascata é usada para tipos mutáveis. Dê uma olhada abaixo, e você verá em primeira mão:

x = y = [7, 8, 9]     # x and y are two different names for the same list object just created, [7, 8, 9]
x[0] = 13             # we are updating the value of the list [7, 8, 9] through one of its names, x in this case
print(y)              # printing the value of the list using its other name
# Output: [13, 8, 9]  # hence, naturally the change is reflected

Listas aninhadas também são válidas em python. Isso significa que uma lista pode conter outra lista como elemento.

x = [1, 2, [3, 4, 5], 6, 7] # this is nested list
print x[2]
# Output: [3, 4, 5]
print x[2][1]
# Output: 4

Por fim, as variáveis ​​em Python não precisam permanecer do mesmo tipo que foram definidas pela primeira vez - você pode simplesmente usar = para atribuir um novo valor a uma variável, mesmo que esse valor seja de um tipo diferente.

a = 2 
print(a)
# Output: 2

a = "New value"
print(a)
# Output: New value

Se isso te incomoda, pense no fato de que o que está à esquerda de = é apenas um nome para um objeto. Primeiro você chama o objeto int com valor 2 a, então você muda de ideia e decide dar o nome a a um objeto string, tendo o valor ‘Novo valor’. Simples, certo?

Recuo de bloco #

Python usa recuo para definir construções de controle e loop. Isso contribui para a legibilidade do Python, no entanto, exige que o programador preste muita atenção ao uso de espaços em branco. Assim, a calibração incorreta do editor pode resultar em código que se comporta de maneiras inesperadas.

Python usa o símbolo de dois pontos (:) e recuo para mostrar onde os blocos de código começam e terminam (se você vem de outra linguagem, não confunda isso com alguma relação com o operador ternário). Ou seja, blocos em Python, como funções, loops, cláusulas if e outras construções, não possuem identificadores finais. Todos os blocos começam com dois pontos e contêm as linhas recuadas abaixo dele.

Por exemplo:

def my_function():    # This is a function definition. Note the colon (:)
    a = 2             # This line belongs to the function because it's indented
    return a          # This line also belongs to the same function
print(my_function())  # This line is OUTSIDE the function block

ou

if a > b:             # If block starts here
    print(a)          # This is part of the if block
else:                 # else must be at the same level as if
    print(b)          # This line is part of the else block

Blocos que contêm exatamente uma instrução de uma única linha podem ser colocados na mesma linha, embora este formulário geralmente não seja considerado um bom estilo:

if a > b: print(a)
else: print(b)

Tentar fazer isso com mais de uma única instrução não funcionará:

if x > y: y = x
    print(y) # IndentationError: unexpected indent

if x > y: while y != z: y -= 1  # SyntaxError: invalid syntax

Um bloco vazio causa um IndentationError. Use pass (um comando que não faz nada) quando você tem um bloco sem conteúdo:

def will_be_implemented_later():
    pass

Espaços vs. Tabs #

Resumindo: sempre use 4 espaços para recuo.

Usar abas exclusivamente é possível, mas PEP 8, o guia de estilo para código Python, afirma que os espaços são preferidos.

O Python 3 não permite misturar o uso de tabulações e espaços para recuo. Nesse caso, é gerado um erro em tempo de compilação: Uso inconsistente de tabulações e espaços na indentação e o programa não será executado.

Python 2 permite misturar tabulações e espaços na indentação; isso é fortemente desencorajado. O caractere de tabulação completa o recuo anterior para ser um múltiplo de 8 espaços. Como é comum que os editores sejam configurados para mostrar as guias como múltiplos de 4 espaços, isso pode causar bugs sutis.

Citando PEP 8:

Ao invocar o interpretador de linha de comando do Python 2 com a opção -t, ele emite avisos sobre código que mistura tabulações e espaços ilegalmente. Ao usar -tt esses avisos se tornam erros. Essas opções são altamente recomendadas!

Muitos editores têm configuração “tabs para espaços”. Ao configurar o editor, deve-se diferenciar entre a tab character (’\t’) e a tecla Tab.

  • A guia caractere deve ser configurada para mostrar 8 espaços, para corresponder à semântica do idioma - pelo menos nos casos em que a indentação mista (acidental) é possível. Os editores também podem converter automaticamente o caractere de tabulação em espaços.
  • No entanto, pode ser útil configurar o editor de forma que pressionar a tecla Tab insira 4 espaços, em vez de inserir um caractere de tabulação.

O código-fonte Python escrito com uma mistura de tabulações e espaços, ou com um número não padrão de espaços de recuo, pode ser feito em conformidade com o pep8 usando autopep8. (Uma alternativa menos poderosa vem com a maioria das instalações do Python: reindent.py)

Sequências e coleções #

Python diferencia entre sequências ordenadas e coleções não ordenadas (como set e dict).

  • strings (str, bytes, unicode) são sequências

  • reversed: Uma ordem invertida de str com a função reversed

      a = reversed('hello')

  • tuple: Uma coleção ordenada de valores n de qualquer tipo (n >= 0).

      a = (1, 2, 3)
  b = ('a', 1, 'python', (1, 2))
  b[2] = 'something else' # returns a TypeError

    Supports indexing; immutable; hashable if all its members are hashable

  • list: Uma coleção ordenada de valores n (n >= 0)

      a = [1, 2, 3]
  b = ['a', 1, 'python', (1, 2), [1, 2]]
  b[2] = 'something else' # allowed

    Not hashable; mutable.

  • set: Uma coleção não ordenada de valores únicos. Os itens devem ser hashable.

      a = {1, 2, 'a'}

  • dict: Uma coleção não ordenada de pares de valores-chave exclusivos; as chaves devem ser hashable.

      a = {1: 'one',
       2: 'two'}

  b = {'a': [1, 2, 3],
       'b': 'a string'}

Um objeto é hashável se tiver um valor de hash que nunca muda durante seu tempo de vida (precisa de um método __hash__()), e pode ser comparado a outros objetos (precisa de um método __eq__()). Objetos hashable que comparam igualdade devem ter o mesmo valor de hash.

Constantes incorporadas #

Em conjunto com os tipos de dados internos, há um pequeno número de constantes internas no namespace interno:

  • True: O valor verdadeiro do tipo interno bool

  • False: O valor falso do tipo interno bool

  • Nenhum: Um objeto singleton usado para sinalizar que um valor está ausente.

  • Ellipsis ou ...: usado no núcleo do Python3+ em qualquer lugar e uso limitado no Python2.7+ como parte da notação de array. numpy e pacotes relacionados usam isso como uma referência ‘incluir tudo’ em arrays.

  • NotImplemented: um singleton usado para indicar ao Python que um método especial não suporta os argumentos específicos, e o Python tentará alternativas se disponíveis.

    a = None # No value will be assigned. Any valid datatype can be assigned later

Nenhum não tem nenhuma ordenação natural. O uso de operadores de comparação de ordenação (<, <=, >=, >) não é mais suportado e gerará um TypeError.

None é sempre menor que qualquer número (None < -32 é avaliado como True).

Um objeto é chamado mutável se puder ser alterado. Por exemplo, quando você passa uma lista para alguma função, a lista pode ser alterada:

def f(m):
    m.append(3)  # adds a number to the list. This is a mutation.

x = [1, 2]
f(x)
x == [1, 2]  # False now, since an item was added to the list

Um objeto é chamado imutável se não puder ser alterado de forma alguma. Por exemplo, os inteiros são imutáveis, pois não há como alterá-los:

def bar():
    x = (1, 2)
    g(x)
    x == (1, 2)  # Will always be True, since no function can change the object (1, 2)

Observe que as próprias variáveis são mutáveis, então podemos reatribuir a variável x, mas isso não altera o objeto que x havia apontado anteriormente. Ele apenas fez x apontar para um novo objeto.

Os tipos de dados cujas instâncias são mutáveis ​​são chamados de tipos de dados mutáveis e, da mesma forma, para objetos e tipos de dados imutáveis.

Exemplos de tipos de dados imutáveis:

  • int, long, float, complex
  • str
  • bytes
  • tupla
  • congelado

Exemplos de tipos de dados mutáveis:

  • bytearray

Tipos de coleção #

Existem vários tipos de coleção em Python. Enquanto tipos como int e str mantêm um único valor, os tipos de coleção contêm vários valores.

Listas

O tipo list é provavelmente o tipo de coleção mais comumente usado em Python. Apesar do nome, uma lista é mais como um array em outras linguagens, principalmente JavaScript. Em Python, uma lista é meramente uma coleção ordenada de valores Python válidos. Uma lista pode ser criada colocando valores, separados por vírgulas, entre colchetes:

int_list = [1, 2, 3]
string_list = ['abc', 'defghi']

Uma lista pode estar vazia:

empty_list = []

Os elementos de uma lista não estão restritos a um único tipo de dado, o que faz sentido já que Python é uma linguagem dinâmica:

mixed_list = [1, 'abc', True, 2.34, None]

Uma lista pode conter outra lista como seu elemento:

nested_list = [['a', 'b', 'c'], [1, 2, 3]]

Os elementos de uma lista podem ser acessados ​​por meio de um índice, ou representação numérica de sua posição. Listas em Python são zero-indexadas, o que significa que o primeiro elemento da lista está no índice 0, o segundo elemento está no índice 1 e assim por diante:

names = ['Alice', 'Bob', 'Craig', 'Diana', 'Eric']
print(names[0]) # Alice
print(names[2]) # Craig

Os índices também podem ser negativos, o que significa contar a partir do final da lista (-1 sendo o índice do último elemento). Então, usando a lista do exemplo acima:

print(names[-1]) # Eric
print(names[-4]) # Bob

As listas são mutáveis, portanto, você pode alterar os valores em uma lista:

names[0] = 'Ann'
print(names)
# Outputs ['Ann', 'Bob', 'Craig', 'Diana', 'Eric']

Além disso, é possível adicionar e/ou remover elementos de uma lista:

Anexar objeto ao final da lista com L.append(object), retorna Nenhum.

names = ['Alice', 'Bob', 'Craig', 'Diana', 'Eric']
names.append("Sia")
print(names) 
# Outputs ['Alice', 'Bob', 'Craig', 'Diana', 'Eric', 'Sia']

Adicione um novo elemento à lista em um índice específico. L.insert(índice, objeto)

names.insert(1, "Nikki")
print(names)
# Outputs ['Alice', 'Nikki', 'Bob', 'Craig', 'Diana', 'Eric', 'Sia']

Remova a primeira ocorrência de um valor com L.remove(value), retornando Nenhum

names.remove("Bob")
print(names) # Outputs ['Alice', 'Nikki', 'Craig', 'Diana', 'Eric', 'Sia']

Obtenha o índice na lista do primeiro item cujo valor é x. Ele mostrará um erro se não houver tal item.

name.index("Alice")
0

Contar o comprimento da lista

len(names)
6

contar a ocorrência de qualquer item na lista

a = [1, 1, 1, 2, 3, 4]
a.count(1)
3

Inverta a lista

a.reverse()
[4, 3, 2, 1, 1, 1]
# or
a[::-1]
[4, 3, 2, 1, 1, 1]

Remova e retorne o item no índice (o padrão é o último item) com L.pop([index]), retorna o item

names.pop() # Outputs 'Sia'

Você pode iterar sobre os elementos da lista como abaixo:

for element in my_list:
    print (element)

Tuplas

Uma tupla é semelhante a uma lista, exceto que é de tamanho fixo e imutável. Portanto, os valores na tupla não podem ser alterados nem adicionados ou removidos da tupla. As tuplas são comumente usadas para pequenas coleções de valores que não precisarão ser alteradas, como um endereço IP e uma porta. Tuplas são representadas com parênteses em vez de colchetes:

ip_address = ('10.20.30.40', 8080)

As mesmas regras de indexação para listas também se aplicam a tuplas. As tuplas também podem ser aninhadas e os valores podem ser qualquer Python válido.

Uma tupla com apenas um membro deve ser definida (observe a vírgula) desta forma:

one_member_tuple = ('Only member',)

ou

one_member_tuple = 'Only member',   # No brackets

ou apenas usando a sintaxe tuple

one_member_tuple = tuple(['Only member'])

Dicionários

Um dicionário em Python é uma coleção de pares chave-valor. O dicionário é cercado por chaves. Cada par é separado por uma vírgula e a chave e o valor são separados por dois pontos. Aqui está um exemplo:

state_capitals = {
    'Arkansas': 'Little Rock',
    'Colorado': 'Denver',
    'California': 'Sacramento', 
    'Georgia': 'Atlanta'
}

Para obter um valor, consulte-o por sua chave:

ca_capital = state_capitals['California']

Você também pode obter todas as chaves em um dicionário e depois iterar sobre elas:

for k in state_capitals.keys():
    print('{} is the capital of {}'.format(state_capitals[k], k))

Os dicionários se assemelham fortemente à sintaxe JSON. O módulo nativo json na biblioteca padrão do Python pode ser usado para converter entre JSON e dicionários.

definir

Um set é uma coleção de elementos sem repetições e sem ordem de inserção, mas ordem de classificação. Eles são usados ​​em situações em que é importante apenas que algumas coisas sejam agrupadas, e não a ordem em que foram incluídas. Para grandes grupos de dados, é muito mais rápido verificar se um elemento está ou não em um set do que fazer o mesmo para uma list.

Definir um set é muito semelhante a definir um dicionário:

first_names = {'Adam', 'Beth', 'Charlie'}

Ou você pode construir um set usando uma list existente:

my_list = [1,2,3]
my_set = set(my_list)

Verifique a associação do set usando in:

if name in first_names:
    print(name)

Você pode iterar sobre um set exatamente como uma lista, mas lembre-se: os valores estarão em uma ordem arbitrária definida pela implementação.

defaultdict

Um defaultdict é um dicionário com um valor padrão para chaves, de modo que as chaves para as quais nenhum valor foi definido explicitamente podem ser acessadas sem erros. defaultdict é especialmente útil quando os valores no dicionário são coleções (listas, dicts, etc) no sentido de que não precisa ser inicializado toda vez que uma nova chave é usada.

Um defaultdict nunca irá gerar um KeyError. Qualquer chave que não existe obtém o valor padrão retornado.

Por exemplo, considere o seguinte dicionário

>>> state_capitals = {
    'Arkansas': 'Little Rock',
    'Colorado': 'Denver',
    'California': 'Sacramento', 
    'Georgia': 'Atlanta'
}

Se tentarmos acessar uma chave inexistente, python nos retornará um erro da seguinte forma

>>> state_capitals['Alabama']
Traceback (most recent call last):

  File "<ipython-input-61-236329695e6f>", line 1, in <module>
    state_capitals['Alabama']

KeyError: 'Alabama'

Vamos tentar com um defaultdict. Ele pode ser encontrado no módulo de coleções.

>>> from collections import defaultdict
>>> state_capitals = defaultdict(lambda: 'Boston')

O que fizemos aqui é definir um valor padrão (Boston) caso a chave give não exista. Agora preencha o dict como antes:

>>> state_capitals['Arkansas'] = 'Little Rock'
>>> state_capitals['California'] = 'Sacramento'
>>> state_capitals['Colorado'] = 'Denver'
>>> state_capitals['Georgia'] = 'Atlanta'

Se tentarmos acessar o dict com uma chave inexistente, o python nos retornará o valor padrão, ou seja, Boston

>>> state_capitals['Alabama']
'Boston'

e retorna os valores criados para a chave existente como um dicionário normal

>>> state_capitals['Arkansas']
'Little Rock'

Criando um módulo #

Um módulo é um arquivo importável que contém definições e instruções.

Um módulo pode ser criado criando um arquivo .py.

# hello.py
def say_hello():
    print("Hello!")

As funções em um módulo podem ser usadas importando o módulo.

Para os módulos que você criou, eles precisarão estar no mesmo diretório do arquivo para o qual você os está importando. (No entanto, você também pode colocá-los no diretório lib do Python com os módulos pré-incluídos, mas devem ser evitados se possível.)

$ python
>>> import hello
>>> hello.say_hello()
=> "Hello!"

Os módulos podem ser importados por outros módulos.

# greet.py
import hello
hello.say_hello()

Funções específicas de um módulo podem ser importadas.

# greet.py
from hello import say_hello
say_hello()

Os módulos podem ser alias.

# greet.py
import hello as ai
ai.say_hello()

Um módulo pode ser um script executável autônomo.

# run_hello.py
if __name__ == '__main__':
    from hello import say_hello
    say_hello()

Executá-lo!

$ python run_hello.py
=> "Hello!"

Se o módulo estiver dentro de um diretório e precisar ser detectado pelo python, o diretório deverá conter um arquivo chamado __init__.py.

Função String - str() e repr() #

Existem duas funções que podem ser usadas para obter uma representação legível de um objeto.

repr(x) chama x.__repr__(): uma representação de x. eval normalmente irá converter o resultado desta função de volta para o objeto original.

str(x) chama x.__str__(): uma string legível por humanos que descreve o objeto. Isso pode eliminar algum detalhe técnico.

repr() #

Para muitos tipos, esta função tenta retornar uma string que produziria um objeto com o mesmo valor quando passado para eval(). Caso contrário, a representação é uma string entre colchetes angulares que contém o nome do tipo do objeto junto com informações adicionais. Isso geralmente inclui o nome e o endereço do objeto.

str() #

Para strings, isso retorna a própria string. A diferença entre isso e repr(object) é que str(object) nem sempre tenta retornar uma string que seja aceitável para eval(). Em vez disso, seu objetivo é retornar uma string imprimível ou ’legível por humanos’. Se nenhum argumento for fornecido, isso retornará a string vazia, ''.

Exemplo 1:

s = """w'o"w"""
repr(s) # Output: '\'w\\\'o"w\''  
str(s)  # Output: 'w\'o"w'
eval(str(s)) == s  # Gives a SyntaxError 
eval(repr(s)) == s # Output: True

Exemplo 2:

import datetime
today = datetime.datetime.now()
str(today)  # Output: '2016-09-15 06:58:46.915000'
repr(today) # Output: 'datetime.datetime(2016, 9, 15, 6, 58, 46, 915000)'

Ao escrever uma classe, você pode substituir esses métodos para fazer o que quiser:

class Represent(object):

    def __init__(self, x, y):
        self.x, self.y = x, y

    def __repr__(self):
        return "Represent(x={},y=\"{}\")".format(self.x, self.y)

    def __str__(self):
        return "Representing x as {} and y as {}".format(self.x, self.y)

Usando a classe acima, podemos ver os resultados:

r = Represent(1, "Hopper")
print(r)  # prints __str__
print(r.__repr__)  # prints __repr__: '<bound method Represent.__repr__ of Represent(x=1,y="Hopper")>'
rep = r.__repr__()  # sets the execution of __repr__ to a new variable
print(rep)  # prints 'Represent(x=1,y="Hopper")'
r2 = eval(rep) # evaluates rep
print(r2)  # prints __str__ from new object
print(r2 == r)  # prints 'False' because they are different objects

Instalando módulos externos usando pip #

pip é seu amigo quando você precisa instalar qualquer pacote da infinidade de opções disponíveis no índice de pacotes python (PyPI). pip já está instalado se você estiver usando Python 2 >= 2.7.9 ou Python 3 >= 3.4 baixado de python.org. Para computadores que executam Linux ou outro *nix com um gerenciador de pacotes nativo, o pip geralmente deve ser instalado manualmente.

Em instâncias com Python 2 e Python 3 instalados, pip geralmente se refere a Python 2 e pip3 a Python 3. Usar pip só instalará pacotes para Python 2 e pip3 só instalará pacotes para Python 3.

Encontrando/instalando um pacote #

Procurar um pacote é tão simples quanto digitar

$ pip search <query>
# Searches for packages whose name or summary contains <query>

Instalar um pacote é tão simples quanto digitar (em um terminal/prompt de comando, não no interpretador Python)

$ pip install [package_name]           # latest version of the package

$ pip install [package_name]==x.x.x    # specific version of the package

$ pip install '[package_name]>=x.x.x'  # minimum version of the package

onde x.x.x é o número da versão do pacote que você deseja instalar.

Quando seu servidor está atrás do proxy, você pode instalar o pacote usando o comando abaixo:

$ pip --proxy http://<server address>:<port> install

Atualizando pacotes instalados #

Quando novas versões de pacotes instalados aparecem, elas não são instaladas automaticamente em seu sistema. Para obter uma visão geral de quais pacotes instalados ficaram desatualizados, execute:

$ pip list --outdated

Para atualizar um pacote específico, use

$ pip install [package_name] --upgrade

Atualizar todos os pacotes desatualizados não é uma funcionalidade padrão do pip.
