Arredondamento: redondo, piso, teto, tronco #

Além da função interna round, o módulo math fornece as funções floor, ceil e trunc.

x = 1.55
y = -1.55

# round to the nearest integer
round(x)       #  2
round(y)       # -2

# the second argument gives how many decimal places to round to (defaults to 0)
round(x, 1)    #  1.6
round(y, 1)    # -1.6

# math is a module so import it first, then use it.
import math

# get the largest integer less than x
math.floor(x)  #  1
math.floor(y)  # -2

# get the smallest integer greater than x
math.ceil(x)   #  2
math.ceil(y)   # -1

# drop fractional part of x
math.trunc(x)  #  1, equivalent to math.floor for positive numbers
math.trunc(y)  # -1, equivalent to math.ceil for negative numbers

floor, ceil, trunc e round sempre retornam um float.

round(1.3)  # 1.0

round sempre quebra os laços a partir de zero.

round(0.5)  # 1.0
round(1.5)  # 2.0

floor, ceil e trunc sempre retornam um valor Integral, enquanto round retorna um valor Integral se chamado com um argumento.

round(1.3)      # 1
round(1.33, 1)  # 1.3

round desempate em direção ao número par mais próximo. Isso corrige o viés para números maiores ao realizar um grande número de cálculos.

round(0.5)  # 0
round(1.5)  # 2

Aviso! #

Como em qualquer representação de ponto flutuante, algumas frações não podem ser representadas exatamente. Isso pode levar a algum comportamento de arredondamento inesperado.

round(2.675, 2)  # 2.67, not 2.68!

Aviso sobre o piso, trunc e divisão inteira de números negativos #

Python (e C++ e Java) arredondam de zero para números negativos. Considerar:

>>> math.floor(-1.7)
-2.0
>>> -5 // 2
-3

Trigonometria #

Calculando o comprimento da hipotenusa #

math.hypot(2, 4) # Just a shorthand for SquareRoot(2**2 + 4**2)
# Out: 4.47213595499958

Convertendo graus para/de radianos #

Todas as funções math esperam radianos, então você precisa converter graus em radianos:

math.radians(45)              # Convert 45 degrees to radians
# Out: 0.7853981633974483

Todos os resultados das funções trigonométricas inversas retornam o resultado em radianos, então você pode precisar convertê-lo de volta para graus:

math.degrees(math.asin(1))    # Convert the result of asin to degrees
# Out: 90.0

Funções seno, cosseno, tangente e inversa #

# Sine and arc sine
math.sin(math.pi / 2)
# Out: 1.0
math.sin(math.radians(90))   # Sine of 90 degrees
# Out: 1.0

math.asin(1)
# Out: 1.5707963267948966    # "= pi / 2"
math.asin(1) / math.pi
# Out: 0.5

# Cosine and arc cosine:
math.cos(math.pi / 2)
# Out: 6.123233995736766e-17 
# Almost zero but not exactly because "pi" is a float with limited precision!

math.acos(1)
# Out: 0.0

# Tangent and arc tangent:
math.tan(math.pi/2)
# Out: 1.633123935319537e+16 
# Very large but not exactly "Inf" because "pi" is a float with limited precision

math.atan(math.inf)
# Out: 1.5707963267948966 # This is just "pi / 2"

math.atan(float('inf'))
# Out: 1.5707963267948966 # This is just "pi / 2"

Além do math.atan, há também uma função math.atan2 de dois argumentos, que calcula o quadrante correto e evita as armadilhas da divisão por zero:

math.atan2(1, 2)   # Equivalent to "math.atan(1/2)"
# Out: 0.4636476090008061 # ≈ 26.57 degrees, 1st quadrant

math.atan2(-1, -2) # Not equal to "math.atan(-1/-2)" == "math.atan(1/2)"
# Out: -2.677945044588987 # ≈ -153.43 degrees (or 206.57 degrees), 3rd quadrant

math.atan2(1, 0)   # math.atan(1/0) would raise ZeroDivisionError
# Out: 1.5707963267948966 # This is just "pi / 2"

Seno, cosseno e tangente hiperbólicos #

# Hyperbolic sine function
math.sinh(math.pi) # = 11.548739357257746
math.asinh(1)      # = 0.8813735870195429

# Hyperbolic cosine function
math.cosh(math.pi) # = 11.591953275521519
math.acosh(1)      # = 0.0

# Hyperbolic tangent function
math.tanh(math.pi) # = 0.99627207622075
math.atanh(0.5)    # = 0.5493061443340549

Logaritmos #

math.log(x) fornece o logaritmo natural (base e) de x.

math.log(math.e)  # 1.0
math.log(1)       # 0.0
math.log(100)     # 4.605170185988092

math.log pode perder precisão com números próximos a 1, devido às limitações dos números de ponto flutuante. Para calcular logs próximos de 1 com precisão, use math.log1p, que avalia o logaritmo natural de 1 mais o argumento:

math.log(1 + 1e-20)  # 0.0
math.log1p(1e-20)    # 1e-20

math.log10 pode ser usado para logs de base 10:

math.log10(10)  # 1.0

Quando usado com dois argumentos, math.log(x, base) fornece o logaritmo de x na base fornecida (ou seja, log(x) / log(base).

math.log(100, 10) # 2.0
math.log(27, 3)   # 3.0
math.log(1, 10)   # 0.0

Constantes #

Os módulos math incluem duas constantes matemáticas comumente usadas.

  • math.pi - A constante matemática pi

  • math.e - A constante matemática e (base do logaritmo natural)

    from math import pi, e pi 3.141592653589793 e 2.718281828459045

Python 3.5 e superior têm constantes para infinito e NaN (“não é um número”). A sintaxe mais antiga de passar uma string para float() ainda funciona.

math.inf == float('inf')
# Out: True

-math.inf == float('-inf')
# Out: True

# NaN never compares equal to anything, even itself
math.nan == float('nan')
# Out: False

Infinito e NaN (“não é um número”) #

Em todas as versões do Python, podemos representar infinito e NaN (“não é um número”) da seguinte forma:

pos_inf = float('inf')     # positive infinity
neg_inf = float('-inf')    # negative infinity
not_a_num = float('nan')   # NaN ("not a number")

No Python 3.5 e superior, também podemos usar as constantes definidas math.inf e math.nan:

pos_inf = math.inf
neg_inf = -math.inf
not_a_num = math.nan

As representações de string são exibidas como inf e -inf e nan:

pos_inf, neg_inf, not_a_num
# Out: (inf, -inf, nan)

Podemos testar o infinito positivo ou negativo com o método isinf:

math.isinf(pos_inf)
# Out: True

math.isinf(neg_inf)
# Out: True

Podemos testar especificamente para infinito positivo ou infinito negativo por comparação direta:

pos_inf == float('inf')    # or  == math.inf in Python 3.5+
# Out: True

neg_inf == float('-inf')   # or  == -math.inf in Python 3.5+
# Out: True

neg_inf == pos_inf
# Out: False

O Python 3.2 e superior também permite verificar a finitude:

math.isfinite(pos_inf)
# Out: False

math.isfinite(0.0)
# Out: True

Os operadores de comparação funcionam como esperado para infinito positivo e negativo:

import sys

sys.float_info.max
# Out: 1.7976931348623157e+308  (this is system-dependent)

pos_inf > sys.float_info.max
# Out: True

neg_inf < -sys.float_info.max
# Out: True

Mas se uma expressão aritmética produzir um valor maior que o máximo que pode ser representado como um float, ele se tornará infinito:

pos_inf == sys.float_info.max * 1.0000001
# Out: True

neg_inf == -sys.float_info.max * 1.0000001
# Out: True

No entanto, a divisão por zero não dá um resultado de infinito (ou infinito negativo quando apropriado), mas gera uma exceção ZeroDivisionError.

try:
    x = 1.0 / 0.0
    print(x)
except ZeroDivisionError:
    print("Division by zero")

# Out: Division by zero

Operações aritméticas no infinito apenas dão resultados infinitos, ou às vezes NaN:

-5.0 * pos_inf == neg_inf
# Out: True

-5.0 * neg_inf == pos_inf
# Out: True

pos_inf * neg_inf == neg_inf
# Out: True

0.0 * pos_inf
# Out: nan

0.0 * neg_inf
# Out: nan

pos_inf / pos_inf
# Out: nan

NaN nunca é igual a nada, nem mesmo a si mesmo. Podemos testar isso com o método isnan:

not_a_num == not_a_num
# Out: False

math.isnan(not_a_num)
Out: True

NaN sempre compara como “não igual”, mas nunca menor ou maior que:

not_a_num != 5.0   # or any random value
# Out: True

not_a_num > 5.0   or   not_a_num < 5.0   or   not_a_num == 5.0
# Out: False

As operações aritméticas em NaN sempre dão NaN. Isso inclui a multiplicação por -1: não há “NaN negativo”.

5.0 * not_a_num
# Out: nan

float('-nan')
# Out: nan

-math.nan
# Out: nan

Há uma diferença sutil entre as antigas versões float do NaN e do infinito e as constantes da biblioteca math do Python 3.5+:

math.inf is math.inf, math.nan is math.nan
# Out: (True, True)

float('inf') is float('inf'), float('nan') is float('nan')
# Out: (False, False)

Pow para exponenciação mais rápida #

Usando o módulo timeit na linha de comando:

> python -m timeit 'for x in xrange(50000): b = x**3'
10 loops, best of 3: 51.2 msec per loop
> python -m timeit 'from math import pow' 'for x in xrange(50000): b = pow(x,3)' 
100 loops, best of 3: 9.15 msec per loop

O operador ** integrado geralmente é útil, mas se o desempenho for essencial, use math.pow. Certifique-se de observar, no entanto, que pow retorna floats, mesmo que os argumentos sejam inteiros:

> from math import pow
> pow(5,5)
3125.0

Copiando sinais #

No Python 2.6 e superior, math.copysign(x, y) retorna x com o sinal de y. O valor retornado é sempre um float.

math.copysign(-2, 3)    # 2.0
math.copysign(3, -3)    # -3.0
math.copysign(4, 14.2)  # 4.0
math.copysign(1, -0.0)  # -1.0, on a platform which supports signed zero

Números imaginários #

Números imaginários em Python são representados por um “j” ou “J” à direita do número de destino.

1j         # Equivalent to the square root of -1.
1j * 1j    # = (-1+0j)

Números complexos e o módulo cmath #

O módulo cmath é semelhante ao módulo math, mas define funções apropriadamente para o plano complexo.

Em primeiro lugar, os números complexos são um tipo numérico que faz parte da própria linguagem Python, em vez de ser fornecido por uma classe de biblioteca. Assim, não precisamos importar cmath para expressões aritméticas comuns.

Observe que usamos j (ou J) e não i.

z = 1 + 3j

Devemos usar 1j já que j seria o nome de uma variável em vez de um literal numérico.

1j * 1j
Out: (-1+0j)

1j ** 1j
# Out: (0.20787957635076193+0j)     # "i to the i"  ==  math.e ** -(math.pi/2)

Temos a parte real e a parte imag (imaginária), bem como o complexo conjugado:

# real part and imaginary part are both float type
z.real, z.imag
# Out: (1.0, 3.0)

z.conjugate()
# Out: (1-3j)    # z.conjugate() == z.real - z.imag * 1j

As funções internas abs e complex também fazem parte da própria linguagem e não requerem importação:

abs(1 + 1j)
# Out: 1.4142135623730951     # square root of 2

complex(1)
# Out: (1+0j)

complex(imag=1)
# Out: (1j)

complex(1, 1)
# Out: (1+1j)

A função complex pode receber uma string, mas não pode ter espaços:

complex('1+1j')
# Out: (1+1j)

complex('1 + 1j')
# Exception: ValueError: complex() arg is a malformed string

Mas para a maioria das funções precisamos do módulo, por exemplo sqrt:

import cmath

cmath.sqrt(-1)
# Out: 1j

Naturalmente o comportamento de sqrt é diferente para números complexos e números reais. Em matemática não complexa, a raiz quadrada de um número negativo gera uma exceção:

import math

math.sqrt(-1)
# Exception: ValueError: math domain error

As funções são fornecidas para converter de e para coordenadas polares:

cmath.polar(1 + 1j)
# Out: (1.4142135623730951, 0.7853981633974483)    # == (sqrt(1 + 1), atan2(1, 1))

abs(1 + 1j), cmath.phase(1 + 1j)
# Out: (1.4142135623730951, 0.7853981633974483)    # same as previous calculation

cmath.rect(math.sqrt(2), math.atan(1))
# Out: (1.0000000000000002+1.0000000000000002j)

O campo matemático da análise complexa está além do escopo deste exemplo, mas muitas funções no plano complexo têm um “corte de ramificação”, geralmente ao longo do eixo real ou do eixo imaginário. A maioria das plataformas modernas suporta “zero assinado” conforme especificado no IEEE 754, que fornece continuidade dessas funções em ambos os lados do corte de ramificação. O exemplo a seguir é da documentação do Python:

cmath.phase(complex(-1.0, 0.0))
# Out: 3.141592653589793

cmath.phase(complex(-1.0, -0.0))
# Out: -3.141592653589793

O módulo cmath também fornece muitas funções com contrapartidas diretas do módulo math.

Além de sqrt, existem versões complexas de exp, log, log10, as funções trigonométricas e suas inversas (sin, cos, tan, asin, acos , atan), e as funções hiperbólicas e suas inversas (sinh, cosh, tanh, asinh, acosh, atanh). Observe, no entanto, que não há uma contrapartida complexa de math.atan2, a forma de dois argumentos de arctangent.

cmath.log(1+1j)
# Out: (0.34657359027997264+0.7853981633974483j)

cmath.exp(1j * cmath.pi)
# Out: (-1+1.2246467991473532e-16j)   # e to the i pi == -1, within rounding error

As constantes pi e e são fornecidas. Observe que eles são float e não complex.

type(cmath.pi)
# Out: <class 'float'>

O módulo cmath também fornece versões complexas de isinf e (para Python 3.2+) isfinite. Consulte “[Infinito e NaN][1]”. Um número complexo é considerado infinito se sua parte real ou sua parte imaginária for infinita.

cmath.isinf(complex(float('inf'), 0.0))
# Out: True

Da mesma forma, o módulo cmath fornece uma versão complexa do isnan. Consulte “[Infinito e NaN][1]”. Um número complexo é considerado “não um número” se sua parte real ou sua parte imaginária for “não um número”.

cmath.isnan(0.0, float('nan'))
# Out: True

Observe que não há contrapartida cmath das constantes math.inf e math.nan (do Python 3.5 e superior)

cmath.isinf(complex(0.0, math.inf))
# Out: True

cmath.isnan(complex(math.nan, 0.0))
# Out: True

cmath.inf
# Exception: AttributeError: module 'cmath' has no attribute 'inf'

No Python 3.5 e superior, existe um método isclose nos módulos cmath e math.

z = cmath.rect(*cmath.polar(1+1j))

z
# Out: (1.0000000000000002+1.0000000000000002j)

cmath.isclose(z, 1+1j)
# True
