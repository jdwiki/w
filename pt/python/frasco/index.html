Flask é um micro framework web Python usado para executar os principais sites, incluindo Pintrest, Twilio e Linkedin. Este tópico explica e demonstra a variedade de recursos que o Flask oferece para desenvolvimento web de front e back end.

Sintaxe #

  • @app.route("/urlpath", métodos=[“GET”, “POST”, “DELETE”, “PUTS”, “HEAD”, “OPTIONS”])
  • @app.route("/urlpath/<param>", métodos=[“GET”, “POST”, “DELETE”, “PUTS”, “HEAD”, “OPTIONS”])

Arquivos e modelos #

Em vez de digitar nossa marcação HTML nas instruções de retorno, podemos usar a função render_template():

from flask import Flask
from flask import render_template
app = Flask(__name__)

@app.route("/about")
def about():
    return render_template("about-us.html")

if __name__ == "__main__":
    app.run(host="0.0.0.0", port=80, debug=True)

Isso usará nosso arquivo de modelo about-us.html. Para garantir que nosso aplicativo encontre esse arquivo, devemos organizar nosso diretório no seguinte formato:

- application.py
/templates
    - about-us.html
    - login-form.html
/static
    /styles
        - about-style.css
        - login-style.css
    /scripts
        - about-script.js
        - login-script.js

Mais importante ainda, as referências a esses arquivos no HTML devem ser assim:

<link rel="stylesheet" type="text/css", href="{{url_for('static', filename='styles/about-style.css')}}">

que direcionará o aplicativo para procurar about-style.css na pasta de estilos sob a pasta estática. O mesmo formato de caminho se aplica a todas as referências a imagens, estilos, scripts ou arquivos.

O básico #

O exemplo a seguir é um exemplo de um servidor básico:

# Imports the Flask class
from flask import Flask
# Creates an app and checks if its the main or imported
app = Flask(__name__)

# Specifies what URL triggers hello_world()
@app.route('/')
# The function run on the index route
def hello_world():
    # Returns the text to be displayed
    return "Hello World!"

# If this script isn't an import
if __name__ == "__main__":
    # Run the app until stopped
    app.run()

A execução deste script (com todas as dependências corretas instaladas) deve iniciar um servidor local. O host é 127.0.0.1 comumente conhecido como localhost. Este servidor por padrão é executado na porta 5000. Para acessar seu servidor web, abra um navegador web e digite a URL localhost:5000 ou 127.0.0.1:5000 (sem diferença). Atualmente, apenas o seu computador pode acessar o servidor web.

app.run() tem três parâmetros, host, port e debug. O host é, por padrão, 127.0.0.1, mas configurá-lo para 0.0.0.0 tornará seu servidor web acessível de qualquer dispositivo em sua rede usando seu endereço IP privado na URL. a porta é, por padrão, 5000, mas se o parâmetro for definido como porta 80, os usuários não precisarão especificar um número de porta, pois os navegadores usam a porta 80 por padrão. Quanto à opção debug, durante o processo de desenvolvimento (nunca em produção) ajuda definir este parâmetro como True, pois seu servidor será reiniciado quando forem feitas alterações em seu projeto Flask.

if __name__ == "__main__":
    app.run(host="0.0.0.0", port=80, debug=True)

URLs de roteamento #

Com o Flask, o roteamento de URL é tradicionalmente feito usando decoradores. Esses decoradores podem ser usados ​​para roteamento estático, bem como URLs de roteamento com parâmetros. Para o exemplo a seguir, imagine que este script do Flask está executando o site www.example.com.

@app.route("/")
def index():
    return "You went to www.example.com"

@app.route("/about")
def about():
    return "You went to www.example.com/about"

@app.route("/users/guido-van-rossum")
    return "You went to www.example.com/guido-van-rossum"

Com essa última rota, você pode ver que dado um URL com /users/ e o nome do perfil, poderíamos retornar um perfil. Como seria terrivelmente ineficiente e confuso incluir um @app.route() para cada usuário, o Flask oferece para obter parâmetros da URL:

@app.route("/users/<username>")
def profile(username):
    return "Welcome to the profile of " + username

cities = ["OMAHA", "MELBOURNE", "NEPAL", "STUTTGART", "LIMA", "CAIRO", "SHANGHAI"]

@app.route("/stores/locations/<city>")
def storefronts(city):
    if city in cities:
        return "Yes! We are located in " + city
    else:
        return "No. We are not located in " + city

Métodos HTTP #

Os dois métodos HTTP mais comuns são GET e POST. O Flask pode executar códigos diferentes da mesma URL, dependendo do método HTTP usado. Por exemplo, em um serviço da Web com contas, é mais conveniente rotear a página de entrada e o processo de entrada pela mesma URL. Uma requisição GET, a mesma que é feita quando você abre uma URL no seu navegador deve mostrar o formulário de login, enquanto uma requisição POST (portando dados de login) deve ser processada separadamente. Uma rota também é criada para lidar com os métodos DELETE e PUT HTTP.

@app.route("/login", methods=["GET"])
def login_form():
    return "This is the login form"
@app.route("/login", methods=["POST"])
def login_auth():
    return "Processing your data"
@app.route("/login", methods=["DELETE", "PUT"])
def deny():
    return "This method is not allowed"

Para simplificar um pouco o código, podemos importar o pacote request do flask.

from flask import request

@app.route("/login", methods=["GET", "POST", "DELETE", "PUT"])
def login():
    if request.method == "DELETE" or request.method == "PUT":
        return "This method is not allowed"
    elif request.method == "GET":
        return "This is the login forum"
    elif request.method == "POST":
        return "Processing your data"

Para recuperar dados da requisição POST, devemos usar o pacote request:

from flask import request
@app.route("/login", methods=["GET", "POST", "DELETE", "PUT"])
def login():
    if request.method == "DELETE" or request.method == "PUT":
        return "This method is not allowed"
    elif request.method == "GET":
        return "This is the login forum"
    elif request.method == "POST":
        return "Username was " + request.form["username"] + " and password was " + request.form["password"]

Modelo Jinja #

Semelhante ao Meteor.js, o Flask se integra bem aos serviços de modelagem de front-end. Flask usa por padrão Jinja Templating. Os modelos permitem que pequenos trechos de código sejam usados ​​no arquivo HTML, como condicionais ou loops.

Quando renderizamos um modelo, quaisquer parâmetros além do nome do arquivo de modelo são passados ​​para o serviço de modelagem HTML. A rota a seguir passará o nome de usuário e a data de ingresso (de uma função em outro lugar) no HTML.

@app.route("/users/<username>)
def profile(username):
    joinedDate = get_joined_date(username) # This function's code is irrelevant
    awards = get_awards(username) # This function's code is irrelevant
    # The joinDate is a string and awards is an array of strings
    return render_template("profile.html", username=username, joinDate=joinDate, awards=awards)

Quando este template é renderizado, ele pode usar as variáveis ​​passadas para ele da função render_template(). Aqui está o conteúdo de profile.html:

<!DOCTYPE html>
<html>
    <head>
        # if username
            <title>Profile of {{ username }}</title>
        # else
            <title>No User Found</title>
        # endif
    <head>
    <body>
        {% if username %}
            <h1>{{ username }} joined on the date {{ date }}</h1>
            {% if len(awards) > 0 %}
                <h3>{{ username }} has the following awards:</h3>
                <ul>
                {% for award in awards %}
                    <li>{{award}}</li>
                {% endfor %}
                </ul>
            {% else %}
                <h3>{{ username }} has no awards</h3>
            {% endif %}
        {% else %}
            <h1>No user was found under that username</h1>
        {% endif %}
        {# This is a comment and doesn't affect the output #}
    </body>
</html>

Os seguintes delimitadores são usados ​​para diferentes interpretações:

  • {% ... %} denota uma declaração
  • {{ ... }} denota uma expressão onde um modelo é gerado
  • {# ... #} denota um comentário (não incluído na saída do modelo)
  • {# ... ## implica que o resto da linha deve ser interpretado como uma declaração

O objeto de solicitação #

O objeto request fornece informações sobre a solicitação que foi feita à rota. Para utilizar este objeto, ele deve ser importado do módulo do frasco:

from flask import request

uploads de arquivos #

Se um upload de arquivo fez parte do formulário enviado em uma solicitação POST, os arquivos podem ser manipulados usando o objeto request:

@app.route("/upload", methods=["POST"])
def upload_file():
    f = request.files["wordlist-upload"]
    f.save("/var/www/uploads/" + f.filename) # Store with the original filename

Biscoitos #

A solicitação também pode incluir cookies em um dicionário semelhante aos parâmetros de URL.

@app.route("/home")
def home():
    try:
        username = request.cookies.get("username")
        return "Your stored username is " + username
    except KeyError:
        return "No username cookies was found")
