As metaclasses permitem que você modifique profundamente o comportamento das classes Python (em termos de como elas são definidas, instanciadas, acessadas e muito mais) substituindo a metaclasse type que as novas classes usam por padrão.

Ao projetar sua arquitetura, considere que muitas coisas que podem ser realizadas com metaclasses também podem ser realizadas usando uma semântica mais simples:

  • A herança tradicional é muitas vezes mais do que suficiente.
  • Decoradores de classe podem misturar funcionalidades em classes em uma abordagem ad-hoc.
  • Python 3.6 apresenta __init_subclass__() que permite que uma classe participe da criação de sua subclasse.

Metaclasses básicas #

Quando type é chamado com três argumentos ele se comporta como a (meta)classe que é, e cria uma nova instância, ie. ele produz uma nova classe/tipo.

Dummy = type('OtherDummy', (), dict(x=1))
Dummy.__class__              # <type 'type'> 
Dummy().__class__.__class__  # <type 'type'>

É possível criar uma subclasse type para criar uma metaclasse personalizada.

class mytype(type):
    def __init__(cls, name, bases, dict):
        # call the base initializer
        type.__init__(cls, name, bases, dict)

        # perform custom initialization...
        cls.__custom_attribute__ = 2

Agora, temos uma nova metaclasse mytype personalizada que pode ser usada para criar classes da mesma maneira que type.

MyDummy = mytype('MyDummy', (), dict(x=2))
MyDummy.__class__              # <class '__main__.mytype'>
MyDummy().__class__.__class__  # <class '__main__.mytype'>
MyDummy.__custom_attribute__   # 2

Quando criamos uma nova classe usando a palavra-chave class, a metaclasse é por padrão escolhida com base nas classes base.

>>> class Foo(object):
...     pass

>>> type(Foo)
type

No exemplo acima a única classe base é object então nossa metaclass será do tipo object, que é type. É possível substituir o padrão, mas depende se usamos Python 2 ou Python 3:

Um atributo especial de nível de classe __metaclass__ pode ser usado para especificar a metaclasse.

class MyDummy(object):
    __metaclass__ = mytype
type(MyDummy)  # <class '__main__.mytype'>

Um argumento de palavra-chave especial metaclass especifica a metaclasse.

class MyDummy(metaclass=mytype):
    pass
type(MyDummy)  # <class '__main__.mytype'>

Quaisquer argumentos de palavra-chave (exceto metaclass) na declaração de classe serão passados ​​para a metaclasse. Assim, class MyDummy(metaclass=mytype, x=2) passará x=2 como um argumento de palavra-chave para o construtor mytype.

Leia esta descrição detalhada das metaclasses python para obter mais detalhes.

Singletons usando metaclasses #

Um singleton é um padrão que restringe a instanciação de uma classe a uma instância/objeto. Para obter mais informações sobre padrões de design singleton python, consulte aqui.

class SingletonType(type):
    def __call__(cls, *args, **kwargs):
        try:
            return cls.__instance
        except AttributeError:
            cls.__instance = super(SingletonType, cls).__call__(*args, **kwargs)
            return cls.__instance

class MySingleton(object):
    __metaclass__ = SingletonType

class MySingleton(metaclass=SingletonType):
    pass

MySingleton() is MySingleton()  # True, only one instantiation occurs

Usando uma metaclasse #

Sintaxe da metaclasse #

class MyClass(object):
    __metaclass__ = SomeMetaclass

class MyClass(metaclass=SomeMetaclass):
    pass

Compatibilidade Python 2 e 3 com seis #

import six

class MyClass(six.with_metaclass(SomeMetaclass)):
    pass

Introdução às Metaclasses #

O que é uma metaclasse? #

Em Python, tudo é um objeto: inteiros, strings, listas, até mesmo funções e classes são objetos. E cada objeto é uma instância de uma classe.

Para verificar a classe de um objeto x, pode-se chamar type(x), então:

>>> type(5)
<type 'int'>
>>> type(str)
<type 'type'>
>>> type([1, 2, 3])
<type 'list'>

>>> class C(object):
...     pass
...
>>> type(C)
<type 'type'>

A maioria das classes em python são instâncias de type. type em si também é uma classe. Tais classes cujas instâncias também são classes são chamadas de metaclasses.

A Metaclasse Mais Simples #

OK, então já existe uma metaclasse em Python: type. Podemos criar outro?

class SimplestMetaclass(type):
    pass

class MyClass(object):
    __metaclass__ = SimplestMetaclass

Isso não adiciona nenhuma funcionalidade, mas é uma nova metaclasse, veja que MyClass agora é uma instância de SimplestMetaclass:

>>> type(MyClass)
<class '__main__.SimplestMetaclass'>

Uma metaclasse que faz algo #

Uma metaclasse que faz algo normalmente sobrescreve o __new__ do type, para modificar algumas propriedades da classe a ser criada, antes de chamar o __new__ original que cria a classe:

class AnotherMetaclass(type):
    def __new__(cls, name, parents, dct):
        # cls is this class
        # name is the name of the class to be created
        # parents is the list of the class's parent classes
        # dct is the list of class's attributes (methods, static variables)

        # here all of the attributes can be modified before creating the class, e.g.

        dct['x'] = 8  # now the class will have a static variable x = 8

        # return value is the new class. super will take care of that
        return super(AnotherMetaclass, cls).__new__(cls, name, parents, dct)

Funcionalidade personalizada com metaclasses #

Funcionalidade em metaclasses pode ser alterada para que sempre que uma classe for construída, uma string seja impressa na saída padrão ou uma exceção seja lançada. Esta metaclasse imprimirá o nome da classe que está sendo construída.

class VerboseMetaclass(type):

    def __new__(cls, class_name, class_parents, class_dict):
        print("Creating class ", class_name)
        new_class = super().__new__(cls, class_name, class_parents, class_dict)
        return new_class

Você pode usar a metaclasse assim:

class Spam(metaclass=VerboseMetaclass):
    def eggs(self):
        print("[insert example string here]")
s = Spam()
s.eggs()

A saída padrão será:

Creating class Spam
[insert example string here]

A metaclasse padrão #

Você já deve ter ouvido falar que tudo em Python é um objeto. É verdade, e todos os objetos têm uma classe:

>>> type(1)
int

O literal 1 é uma instância de int. Vamos declarar uma classe:

>>> class Foo(object):
...    pass
...

Agora vamos instanciar:

>>> bar = Foo()

Qual é a classe de bar?

>>> type(bar)
Foo

Legal, bar é uma instância de Foo. Mas qual é a classe de Foo em si?

>>> type(Foo)
type

Ok, Foo em si é uma instância de type. Que tal o próprio type?

>>> type(type)
type

Então, o que é uma metaclasse? Por enquanto vamos fingir que é apenas um nome chique para a classe de uma classe. Aprendizado:

  • Tudo é um objeto em Python, então tudo tem uma classe
  • A classe de uma classe é chamada de metaclasse
  • A metaclasse padrão é type, e de longe é a metaclasse mais comum

Mas por que você deveria saber sobre metaclasses? Bem, o próprio Python é bastante “hackeável”, e o conceito de metaclasse é importante se você estiver fazendo coisas avançadas como metaprogramação ou se quiser controlar como suas classes são inicializadas.
