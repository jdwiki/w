Funções em Python fornecem código organizado, reutilizável e modular para realizar um conjunto de ações específicas. As funções simplificam o processo de codificação, evitam lógica redundante e tornam o código mais fácil de seguir. Este tópico descreve a declaração e a utilização de funções em Python.

Python tem muitas funções embutidas como print(), input(), len(). Além dos built-ins, você também pode criar suas próprias funções para fazer trabalhos mais específicos—elas são chamadas de funções definidas pelo usuário.

Sintaxe #

  • def function_name(arg1, … argN, *args, kw1, kw2=default, …, **kwargs): statements
  • lambda arg1, … argN, *args, kw1, kw2=padrão, …, **kwargs: expressão

Parâmetros #

Parâmetro Detalhes
arg1, …, argN Argumentos regulares
*args Argumentos posicionais sem nome
kw1, …, kwN Argumentos somente de palavras-chave
**kwargs O resto dos argumentos de palavras-chave

5 coisas básicas que você pode fazer com funções:

  • Atribuir funções a variáveis

    def f():
  print(20)
y = f
y()
# Output: 20

  • Defina funções dentro de outras funções (https://www.wikiod.com/pt/python/funcoes#Funções aninhadas )

    def f(a, b, y):
    def inner_add(a, b):      # inner_add is hidden from outer code
        return a + b
    return inner_add(a, b)**y

  • As funções podem retornar outras funções

    def f(y):
    def nth_power(x):
        return x ** y
    return nth_power    # returns a function

squareOf = f(2)         # function that returns the square of a number           
cubeOf = f(3)           # function that returns the cube of a number
squareOf(3)             # Output: 9
cubeOf(2)               # Output: 8

  • As funções podem ser passadas como parâmetros para outras funções

    def a(x, y):
    print(x, y)
def b(fun, str):        # b has two arguments: a function and a string 
    fun('Hello', str)
b(a, 'Sophia')           # Output: Hello Sophia

  • As funções internas têm acesso ao escopo anexo (https://www.wikiod.com/pt/python/funcoes#Fecho )

    def outer_fun(name):
    def inner_fun():     # the variable name is available to the inner function
        return "Hello "+ name + "!"
    return inner_fun
greet = outer_fun("Sophia")
print(greet())            # Output: Hello Sophia!

Definindo uma função com um número arbitrário de argumentos #

Número arbitrário de argumentos posicionais: #

Definir uma função capaz de receber um número arbitrário de argumentos pode ser feito prefixando um dos argumentos com um *

def func(*args):
    # args will be a tuple containing all values that are passed in
    for i in args:
        print(i)

func(1, 2, 3)  # Calling it with 3 arguments
# Out: 1
#      2
#      3

list_of_arg_values = [1, 2, 3]
func(*list_of_arg_values)  # Calling it with list of values, * expands the list
# Out: 1
#      2
#      3 

func()  # Calling it without arguments
# No Output

Você não pode fornecer um padrão para args, por exemplo func(*args=[1, 2, 3]) gerará um erro de sintaxe (nem mesmo compilará).

Você não pode fornecer isso pelo nome ao chamar a função, por exemplo func(*args=[1, 2, 3]) irá gerar um TypeError.

Mas se você já tem seus argumentos em um array (ou qualquer outro Iterable), você pode invocar sua função assim: func(*my_stuff).

Esses argumentos (*args) podem ser acessados ​​pelo índice, por exemplo args[0] retornará o primeiro argumento

Número arbitrário de argumentos de palavras-chave #

Você pode pegar um número arbitrário de argumentos com um nome definindo um argumento na definição com dois * na frente dele:

def func(**kwargs):
    # kwargs will be a dictionary containing the names as keys and the values as values
    for name, value in kwargs.items():
        print(name, value)

func(value1=1, value2=2, value3=3)   # Calling it with 3 arguments
# Out: value1 1
#      value2 2
#      value3 3

func()                               # Calling it without arguments
# No Out put

my_dict = {'foo': 1, 'bar': 2}
func(**my_dict)                      # Calling it with a dictionary
# Out: foo 1
#      bar 2

Você não pode fornecer esses nomes sem, por exemplo func(1, 2, 3) irá gerar um TypeError.

kwargs é um dicionário python nativo simples. Por exemplo, args['value1'] dará o valor para o argumento value1. Certifique-se de verificar de antemão que existe tal argumento ou um KeyError será gerado.

Aviso #

Você pode misturá-los com outros argumentos opcionais e obrigatórios, mas a ordem dentro da definição é importante.

Os argumentos posicional/palavra-chave vêm primeiro. (Argumentos necessários). Em seguida, vêm os argumentos arbitrários *arg. (Opcional). Em seguida, os argumentos somente palavras-chave vêm a seguir. (Requeridos). Finalmente vem a palavra-chave arbitrária **kwargs. (Opcional).

#       |-positional-|-optional-|---keyword-only--|-optional-|
def func(arg1, arg2=10 , *args, kwarg1, kwarg2=2, **kwargs):
     pass
  • arg1 deve ser dado, caso contrário um TypeError é gerado. Pode ser dado como argumento posicional (func(10)) ou palavra-chave (func(arg1=10)).
  • kwarg1 também deve ser fornecido, mas só pode ser fornecido como argumento-chave: func(kwarg1=10).
  • arg2 e kwarg2 são opcionais. Se o valor for alterado, aplicam-se as mesmas regras para arg1 (seja posicional ou palavra-chave) e kwarg1 (apenas palavra-chave).
  • *args captura parâmetros posicionais adicionais. Mas note que arg1 e arg2 devem ser fornecidos como argumentos posicionais para passar argumentos para *args: func(1, 1, 1, 1).
  • **kwargs pega todos os parâmetros de palavras-chave adicionais. Neste caso, qualquer parâmetro que não seja arg1, arg2, kwarg1 ou kwarg2. Por exemplo: func(kwarg3=10).
  • No Python 3, você pode usar * sozinho para indicar que todos os argumentos subsequentes devem ser especificados como palavras-chave. Por exemplo, a função math.isclose no Python 3.5 e superior é definida usando def math.isclose (a, b, *, rel_tol=1e-09, abs_tol=0.0), o que significa que os dois primeiros argumentos podem ser fornecidos posicionalmente, mas o terceiro e quarto parâmetros opcionais só podem ser fornecidos como argumentos de palavra-chave.

O Python 2.x não oferece suporte a parâmetros somente de palavras-chave. Este comportamento pode ser emulado com kwargs:

def func(arg1, arg2=10, **kwargs):
    try:
        kwarg1 = kwargs.pop("kwarg1")
    except KeyError:
        raise TypeError("missing required keyword-only argument: 'kwarg1'")

    kwarg2 = kwargs.pop("kwarg2", 2)
    # function body ...

Nota sobre a nomenclatura #

A convenção de nomear argumentos posicionais opcionais args e argumentos de palavras-chave opcionais kwargs é apenas uma convenção que você pode usar os nomes que quiser mas é útil seguir a convenção para que outros saibam o que você está fazendo, ou até mesmo mais tarde então, por favor, faça.

Nota sobre exclusividade #

Qualquer função pode ser definida com nenhum ou um *args e nenhum ou um **kwargs mas não com mais de um de cada. Também *args deve ser o último argumento posicional e **kwargs deve ser o último parâmetro. A tentativa de usar mais de um vai resultar em uma exceção de erro de sintaxe.

Nota sobre funções de aninhamento com argumentos opcionais #

É possível aninhar tais funções e a convenção usual é remover os itens que o código já tratou mas se você estiver passando os parâmetros você precisa passar argumentos posicionais opcionais com um prefixo * e uma palavra-chave opcional args com um prefixo **, caso contrário args serão passados ​​como uma lista ou tupla e kwargs como um único dicionário. por exemplo.:

def fn(**kwargs):
    print(kwargs)
    f1(**kwargs)

def f1(**kwargs):
    print(len(kwargs))

fn(a=1, b=2)
# Out:
# {'a': 1, 'b': 2}
# 2

Definindo e chamando funções simples #

Usar a instrução def é a maneira mais comum de definir uma função em python. Esta instrução é a chamada instrução composta de cláusula única com a seguinte sintaxe:

def function_name(parameters):
    statement(s)

function_name é conhecido como o identificador da função. Como uma definição de função é uma instrução executável, sua execução vincula o nome da função ao objeto de função que pode ser chamado mais tarde usando o identificador.

parameters é uma lista opcional de identificadores que são vinculados aos valores fornecidos como argumentos quando a função é chamada. Uma função pode ter um número arbitrário de argumentos que são separados por vírgulas.

instrução(ões) – também conhecido como corpo da função – são uma sequência não vazia de instruções executadas toda vez que a função é chamada. Isso significa que um corpo de função não pode estar vazio, assim como qualquer [bloco recuado](https://www.wikiod.com/pt/python/introducao-a-linguagem-python#Recuo do bloco).

Aqui está um exemplo de uma definição de função simples cujo objetivo é imprimir Hello cada vez que é chamado:

def greet():
    print("Hello")

Agora vamos chamar a função greet() definida:

greet()
# Out: Hello

Esse é outro exemplo de definição de função que recebe um único argumento e exibe o valor passado toda vez que a função é chamada:

def greet_two(greeting):
    print(greeting)

Depois disso, a função greet_two() deve ser chamada com um argumento:

greet_two("Howdy")
# Out: Howdy

Além disso, você pode dar um valor padrão para esse argumento de função:

def greet_two(greeting="Howdy"):
    print(greeting)

Agora você pode chamar a função sem dar um valor:

greet_two()
# Out: Howdy

Você notará que, diferentemente de muitas outras linguagens, você não precisa declarar explicitamente um tipo de retorno da função. As funções Python podem retornar valores de qualquer tipo através da palavra-chave return. Uma função pode retornar qualquer número de tipos diferentes!

def many_types(x):
    if x < 0:
        return "Hello!"
    else:
        return 0

print(many_types(1))
print(many_types(-1))

# Output:
0
Hello!

Contanto que isso seja tratado corretamente pelo chamador, este é um código Python perfeitamente válido.

Uma função que chega ao fim da execução sem uma instrução return sempre retornará None:

def do_nothing():
    pass

print(do_nothing())
# Out: None

Como mencionado anteriormente, uma definição de função deve ter um corpo de função, uma sequência não vazia de instruções. Portanto, a instrução pass é usada como corpo da função, que é uma operação nula – quando é executada, nada acontece. Ele faz o que significa, ele pula. É útil como um espaço reservado quando uma instrução é necessária sintaticamente, mas nenhum código precisa ser executado.

Funções Lambda (Inline/Anonymous) #

A palavra-chave lambda cria uma função inline que contém uma única expressão. O valor dessa expressão é o que a função retorna quando invocada.

Considere a função:

def greeting():
    return "Hello"

que, quando chamado como:

print(greeting())

imprime:

Hello

Isso pode ser escrito como uma função lambda da seguinte forma:

greet_me = lambda: "Hello"

Veja a nota no final desta seção sobre a atribuição de lambdas a variáveis. Geralmente, não faça isso.

Isso cria uma função inline com o nome greet_me que retorna Hello. Observe que você não escreve return ao criar uma função com lambda. O valor após : é retornado automaticamente.

Uma vez atribuída a uma variável, ela pode ser usada como uma função regular:

print(greet_me())

imprime:

Hello

lambdas também pode receber argumentos:

strip_and_upper_case = lambda s: s.strip().upper()

strip_and_upper_case("  Hello   ")

retorna a string:

HELLO

Eles também podem receber um número arbitrário de argumentos / argumentos de palavras-chave, como funções normais.

greeting = lambda x, *args, **kwargs: print(x, args, kwargs)
greeting('hello', 'world', world='world')

imprime:

hello ('world',) {'world': 'world'}

lambdas são comumente usados ​​para funções curtas que são convenientes para definir no ponto em que são chamadas (normalmente com sorted, filter e map).

Por exemplo, esta linha classifica uma lista de strings ignorando maiúsculas e minúsculas e ignorando espaços em branco no início e no final:

sorted( [" foo ", "    bAR", "BaZ    "], key=lambda s: s.strip().upper())
# Out:
# ['    bAR', 'BaZ    ', ' foo ']

Classifique a lista apenas ignorando os espaços em branco:

sorted( [" foo ", "    bAR", "BaZ    "], key=lambda s: s.strip())
# Out:
# ['BaZ    ', '    bAR', ' foo ']

Exemplos com map:

sorted( map( lambda s: s.strip().upper(), [" foo ", "    bAR", "BaZ    "]))
# Out:
# ['BAR', 'BAZ', 'FOO']

sorted( map( lambda s: s.strip(), [" foo ", "    bAR", "BaZ    "]))
# Out:
# ['BaZ', 'bAR', 'foo']

Exemplos com listas numéricas:

my_list = [3, -4, -2, 5, 1, 7]
sorted( my_list, key=lambda x: abs(x))
# Out:
# [1, -2, 3, -4, 5, 7]

list( filter( lambda x: x>0, my_list))
# Out:
# [3, 5, 1, 7]

list( map( lambda x: abs(x), my_list))
# Out:
[3, 4, 2, 5, 1, 7]

Pode-se chamar outras funções (com/sem argumentos) de dentro de uma função lambda.

def foo(msg):
    print(msg)

greet = lambda x = "hello world": foo(x)
greet()

imprime:

hello world

Isso é útil porque lambda pode conter apenas uma expressão e usando uma função subsidiária pode-se executar várias instruções.

NOTA

Tenha em mente que PEP-8 (o guia de estilo oficial do Python) não recomenda atribuir lambdas a variáveis ​​(como fizemos nos dois primeiros exemplos):

Sempre use uma instrução def em vez de uma instrução de atribuição que vincule uma expressão lambda diretamente a um identificador.

Sim:

def f(x): retorna 2*x Não:

f = lambda x: 2*x A primeira forma significa que o nome do objeto de função resultante é especificamente f em vez do genérico <lambda>. Isso é mais útil para tracebacks e representações de string em geral. O uso da instrução de atribuição elimina o único benefício que uma expressão lambda pode oferecer sobre uma instrução def explícita (ou seja, pode ser incorporada dentro de uma expressão maior).

Definindo uma função com argumentos opcionais #

Os argumentos opcionais podem ser definidos atribuindo (usando =) um valor padrão ao argumento-name:

def make(action='nothing'):
    return action

Chamar esta função é possível de 3 maneiras diferentes:

make("fun")
# Out: fun

make(action="sleep")
# Out: sleep

# The argument is optional so the function will use the default value if the argument is 
# not passed in.
make()   
# Out: nothing

Tipos mutáveis ​​(list, dict, set, etc.) devem ser tratados com cuidado quando fornecidos como atributo default. Qualquer mutação do argumento padrão irá alterá-lo permanentemente. Consulte Definindo uma função com argumentos mutáveis ​​opcionais.

Definindo uma função com argumentos mutáveis ​​opcionais #

Há um problema ao usar argumentos opcionais com um tipo padrão mutável (descrito em [Definindo uma função com argumentos opcionais](https://www.wikiod.com/pt/python/funcoes /definindo-uma-função-com-argumentos-opcionais)), o que pode levar a um comportamento inesperado.

Explicação #

Esse problema surge porque os argumentos padrão de uma função são inicializados uma vez, no ponto em que a função é definida, e não (como muitas outras linguagens) quando a função é chamada. Os valores padrão são armazenados dentro da variável membro __defaults__ do objeto de função.

def f(a, b=42, c=[]):
    pass

print(f.__defaults__)
# Out: (42, [])

Para tipos imutáveis (consulte Passagem e mutabilidade de argumentos), isso não é um problema porque não há como alterar a variável; ele só pode ser reatribuído, deixando o valor original inalterado. Assim, os subsequentes têm a garantia de ter o mesmo valor padrão. No entanto, para um tipo mutável, o valor original pode sofrer mutação, fazendo chamadas para suas várias funções de membro. Portanto, chamadas sucessivas para a função não têm garantia de ter o valor padrão inicial.

def append(elem, to=[]):
    to.append(elem)      # This call to append() mutates the default variable "to"
    return to

append(1)
# Out: [1]

append(2)  # Appends it to the internally stored list
# Out: [1, 2]

append(3, [])  # Using a new created list gives the expected result
# Out: [3]

# Calling it again without argument will append to the internally stored list again
append(4)   
# Out: [1, 2, 4]

Observação: alguns IDEs como o PyCharm emitirão um aviso quando um tipo mutável for especificado como atributo padrão.

Solução #

Se você quiser garantir que o argumento padrão seja sempre aquele especificado na definição da função, a solução é sempre usar um tipo imutável como seu argumento padrão.

Um idioma comum para conseguir isso quando um tipo mutável é necessário como padrão, é usar Nenhum (imutável) como o argumento padrão e, em seguida, atribuir o valor padrão real à variável do argumento se for igual a Nenhum.

def append(elem, to=None):
    if to is None:
        to = []

    to.append(elem)
    return to

Passagem e mutabilidade de argumentos #

Primeiro, alguma terminologia:

  • argumento (parâmetro real): a variável real que está sendo passada para uma função;
  • parâmetro (parâmetro formal): a variável receptora que é usada em uma função.

Em Python, os argumentos são passados ​​por assignment (ao contrário de outras linguagens, onde os argumentos podem ser passados ​​por valor/referência/ponteiro).

  • Mudar um parâmetro mudará o argumento (se o tipo do argumento for mutável).

    def foo(x):        # here x is the parameter
    x[0] = 9       # This mutates the list labelled by both x and y
    print(x)

y = [4, 5, 6]
foo(y)             # call foo with y as argument
# Out: [9, 5, 6]   # list labelled by x has been mutated
print(y)           
# Out: [9, 5, 6]   # list labelled by y has been mutated too

  • Reatribuir o parâmetro não reatribuirá o argumento.

    def foo(x):        # here x is the parameter, when we call foo(y) we assign y to x
    x[0] = 9       # This mutates the list labelled by both x and y
    x = [1, 2, 3]  # x is now labeling a different list (y is unaffected)
    x[2] = 8       # This mutates x's list, not y's list

y = [4, 5, 6]      # y is the argument, x is the parameter
foo(y)             # Pretend that we wrote "x = y", then go to line 1
y
# Out: [9, 5, 6]

Em Python, não atribuímos valores a variáveis, em vez disso, bind (ou seja, atribuir, anexar) variáveis ​​(consideradas como names) a objetos.

  • Imutável: inteiros, strings, tuplas e assim por diante. Todas as operações fazem cópias.

  • Mutável: listas, dicionários, conjuntos e assim por diante. As operações podem ou não sofrer mutações.

    x = [3, 1, 9] y = x x.append(5) # Mutates the list labelled by x and y, both x and y are bound to [3, 1, 9] x.sort() # Mutates the list labelled by x and y (in-place sorting) x = x + [4] # Does not mutate the list (makes a copy for x only, not y) z = x # z is x ([1, 3, 9, 4]) x += [6] # Mutates the list labelled by both x and z (uses the extend function). x = sorted(x) # Does not mutate the list (makes a copy for x only). x

    Out: [1, 3, 4, 5, 6, 9] #

    y

    Out: [1, 3, 5, 9] #

    z

    Out: [1, 3, 5, 9, 4, 6] #

Retornando valores de funções #

As funções podem retornar um valor que você pode usar diretamente:

def give_me_five():
    return 5

print(give_me_five())  # Print the returned value
# Out: 5

ou salve o valor para uso posterior:

num = give_me_five()
print(num)             # Print the saved returned value
# Out: 5

ou use o valor para qualquer operação:

print(give_me_five() + 10)
# Out: 15

Se return for encontrado na função, a função será encerrada imediatamente e as operações subsequentes não serão avaliadas:

def give_me_another_five():
    return 5
    print('This statement will not be printed. Ever.')

print(give_me_another_five())
# Out: 5

Você também pode retornar vários valores (na forma de uma tupla):

def give_me_two_fives():
    return 5, 5  # Returns two 5

first, second = give_me_two_fives()
print(first)
# Out: 5
print(second)
# Out: 5

Uma função com a instrução no return retorna implicitamente None. Da mesma forma, uma função com uma instrução return, mas nenhum valor de retorno ou variável retorna Nenhum.

Fecho #

Fechamentos em Python são criados por chamadas de função. Aqui, a chamada para makeInc cria uma ligação para x que é referenciada dentro da função inc. Cada chamada para makeInc cria uma nova instância desta função, mas cada instância tem um link para uma ligação diferente de x.

def makeInc(x):
  def inc(y):
     # x is "attached" in the definition of inc
     return y + x

  return inc

incOne = makeInc(1)
incFive = makeInc(5)

incOne(5) # returns 6
incFive(5) # returns 10

Observe que enquanto em um fechamento regular a função fechada herda totalmente todas as variáveis ​​de seu ambiente delimitador, nesta construção a função fechada tem apenas acesso de leitura às variáveis ​​herdadas, mas não pode fazer atribuições a elas

def makeInc(x):
  def inc(y):
     # incrementing x is not allowed
     x += y  
     return x

  return inc

incOne = makeInc(1)
incOne(5) # UnboundLocalError: local variable 'x' referenced before assignment

O Python 3 oferece a instrução nonlocal (https://www.wikiod.com/pt/python/escopo-variavel-e-vinculacao#Variáveis ​​Não Locais ) para realizar um fechamento completo com funções aninhadas.

def makeInc(x):
  def inc(y):
     nonlocal x
     # now assigning a value to x is allowed
     x += y  
     return x

  return inc

incOne = makeInc(1)
incOne(5) # returns 6

Forçando o uso de parâmetros nomeados #

Todos os parâmetros especificados após o primeiro asterisco na assinatura da função são somente palavras-chave.

def f(*a, b):
    pass

f(1, 2, 3)
# TypeError: f() missing 1 required keyword-only argument: 'b'

No Python 3 é possível colocar um único asterisco na assinatura da função para garantir que os argumentos restantes só possam ser passados ​​usando argumentos de palavras-chave.

def f(a, b, *, c):
    pass

f(1, 2, 3)
# TypeError: f() takes 2 positional arguments but 3 were given
f(1, 2, c=3)
# No error

Definindo uma função com argumentos #

Os argumentos são definidos entre parênteses após o nome da função:

def divide(dividend, divisor):  # The names of the function and its arguments
    # The arguments are available by name in the body of the function
    print(dividend / divisor)

O nome da função e sua lista de argumentos são chamados de assinatura da função. Cada argumento nomeado é efetivamente uma variável local da função.

Ao chamar a função, forneça valores para os argumentos listando-os em ordem

divide(10, 2)
# output: 5

ou especifique-os em qualquer ordem usando os nomes da definição da função:

divide(divisor=2, dividend=10)
# output: 5

Funções recursivas #

Uma função recursiva é uma função que chama a si mesma em sua definição. Por exemplo, a função matemática, fatorial, definida por factorial(n) = n*(n-1)*(n-2)*...*3*2*1. pode ser programado como

def factorial(n):
    #n here should be an integer
    if n == 0:
        return 1
    else:
        return n*factorial(n-1)

as saídas aqui são:

factorial(0)
#out 1
factorial(1)
#out 1
factorial(2)
#out 2
factorial(3)
#out 6

como esperado. Observe que esta função é recursiva porque o segundo retorna fatorial(n-1), onde a função chama a si mesma em sua definição.

Algumas funções recursivas podem ser implementadas usando lambda, a função fatorial usando lambda seria algo assim:

factorial = lambda n: 1 if n == 0 else n*factorial(n-1)

A função produz o mesmo que acima.

Limite de recursão #

Há um limite para a profundidade de recursão possível, que depende da implementação do Python. Quando o limite é atingido, uma exceção RuntimeError é gerada:

def cursing(depth):
  try:
    cursing(depth + 1) # actually, re-cursing
  except RuntimeError as RE:
    print('I recursed {} times!'.format(depth))

cursing(0)
# Out: I recursed 1083 times!

É possível alterar o limite de profundidade de recursão usando sys.setrecursionlimit(limit) e verificar este limite por sys.getrecursionlimit().

sys.setrecursionlimit(2000)
cursing(0)
# Out: I recursed 1997 times!

A partir do Python 3.5, a exceção é um RecursionError, que é derivado de RuntimeError.

Funções aninhadas #

Funções em python são objetos de primeira classe. Eles podem ser definidos em qualquer escopo

def fibonacci(n):
    def step(a,b):
        return b, a+b
    a, b = 0, 1
    for i in range(n):
        a, b = step(a, b)
    return a

As funções capturam seu escopo delimitador e podem ser passadas como qualquer outro tipo de objeto

def make_adder(n):
    def adder(x):
        return n + x
    return adder
add5 = make_adder(5)
add6 = make_adder(6)
add5(10)
#Out: 15
add6(10)
#Out: 16

def repeatedly_apply(func, n, x):
    for i in range(n):
        x = func(x)
    return x

repeatedly_apply(add5, 5, 1)
#Out: 26

Definindo uma função com vários argumentos #

Pode-se dar a uma função quantos argumentos quiser, as únicas regras fixas são que cada nome de argumento deve ser único e que os argumentos opcionais devem estar após os não opcionais:

def func(value1, value2, optionalvalue=10):
    return '{0} {1} {2}'.format(value1, value2, optionalvalue1)

Ao chamar a função, você pode fornecer cada palavra-chave sem o nome, mas a ordem importa:

print(func(1, 'a', 100))
# Out: 1 a 100

print(func('abc', 14))
# abc 14 10

Ou combine dando os argumentos com nome e sem. Então os com nome devem seguir os sem, mas a ordem dos com nome não importa:

print(func('This', optionalvalue='StackOverflow Documentation', value2='is'))
# Out: This is StackOverflow Documentation
