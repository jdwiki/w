Data Aritmética - Diferença entre Datas em Dias, Horas, Minutos e/ou Segundos #

No oracle, a diferença (em dias e/ou frações) entre dois DATEs pode ser encontrada usando a subtração:

SELECT DATE '2016-03-23' - DATE '2015-12-25' AS difference FROM DUAL;

Emite o número de dias entre as duas datas:

DIFFERENCE
----------
        89

E:

SELECT TO_DATE( '2016-01-02 01:01:12', 'YYYY-MM-DD HH24:MI:SS' )
         - TO_DATE( '2016-01-01 00:00:00', 'YYYY-MM-DD HH24:MI:SS' )
         AS difference
FROM   DUAL

Gera a fração de dias entre duas datas:

DIFFERENCE
----------
    1.0425

A diferença em horas, minutos ou segundos pode ser encontrada multiplicando este número por 24, 24*60 ou 24*60*60 respectivamente.

O exemplo anterior pode ser alterado para obter os dias, horas, minutos e segundos entre duas datas usando:

SELECT TRUNC( difference                       ) AS days,
       TRUNC( MOD( difference * 24,       24 ) ) AS hours,
       TRUNC( MOD( difference * 24*60,    60 ) ) AS minutes,
       TRUNC( MOD( difference * 24*60*60, 60 ) ) AS seconds
FROM   (
  SELECT TO_DATE( '2016-01-02 01:01:12', 'YYYY-MM-DD HH24:MI:SS' )
         - TO_DATE( '2016-01-01 00:00:00', 'YYYY-MM-DD HH24:MI:SS' )
         AS difference
  FROM   DUAL

);

(Observação: TRUNC() é usado em vez de [FLOOR()](https: //docs.oracle.com/cd/B19306_01/server.102/b14200/functions058.htm) para lidar corretamente com diferenças negativas.)

Saídas:

DAYS HOURS MINUTES SECONDS
---- ----- ------- -------
   1     1       1      12

O exemplo anterior também pode ser resolvido convertendo a diferença numérica em um intervalo usando NUMTODSINTERVAL() :

SELECT EXTRACT( DAY    FROM difference ) AS days,
       EXTRACT( HOUR   FROM difference ) AS hours,
       EXTRACT( MINUTE FROM difference ) AS minutes,
       EXTRACT( SECOND FROM difference ) AS seconds
FROM   (
  SELECT NUMTODSINTERVAL(
           TO_DATE( '2016-01-02 01:01:12', 'YYYY-MM-DD HH24:MI:SS' )
             - TO_DATE( '2016-01-01 00:00:00', 'YYYY-MM-DD HH24:MI:SS' ),
           'DAY'
         ) AS difference
  FROM   DUAL
);

Configurando o modelo de formato de data padrão #

Quando o Oracle converte implicitamente de um DATE para uma string ou vice-versa (ou quando TO_CHAR() ou TO_DATE() são chamados explicitamente sem um modelo de formato) o parâmetro de sessão NLS_DATE_FORMAT será usado como modelo de formato na conversão. Se o literal não corresponder ao modelo de formato, uma exceção será gerada.

Você pode revisar este parâmetro usando:

SELECT VALUE FROM NLS_SESSION_PARAMETERS WHERE PARAMETER = 'NLS_DATE_FORMAT';

Você pode definir esse valor em sua sessão atual usando:

ALTER SESSION SET NLS_DATE_FORMAT = 'YYYY-MM-DD HH24:MI:SS';

(Observação: isso não altera o valor para nenhum outro usuário.)

Se você confia no NLS_DATE_FORMAT para fornecer a máscara de formato em TO_DATE() ou TO_CHAR(), então você não deve se surpreender quando suas consultas quebrarem se esse valor for alterado.

Aritmética de datas - Diferença entre datas em meses ou anos #

A diferença em meses entre duas datas pode ser encontrada usando o MONTHS_BETWEEN( date1, date2 ):

SELECT MONTHS_BETWEEN( DATE '2016-03-10', DATE '2015-03-10' ) AS difference FROM DUAL;

Saídas:

DIFFERENCE
----------
        12

Se a diferença incluir meses parciais, ela retornará a fração do mês com base em 31 dias em cada mês:

SELECT MONTHS_BETWEEN( DATE '2015-02-15', DATE '2015-01-01' ) AS difference FROM DUAL;

Saídas:

DIFFERENCE
----------
 1.4516129

Devido a MONTHS_BETWEEN assumir 31 dias por mês, quando pode haver menos dias por mês, isso pode resultar em valores diferentes para diferenças que abrangem os limites entre os meses.

Exemplo:

SELECT MONTHS_BETWEEN( DATE'2016-02-01', DATE'2016-02-01' - INTERVAL '1' DAY ) AS "JAN-FEB",
       MONTHS_BETWEEN( DATE'2016-03-01', DATE'2016-03-01' - INTERVAL '1' DAY ) AS "FEB-MAR",
       MONTHS_BETWEEN( DATE'2016-04-01', DATE'2016-04-01' - INTERVAL '1' DAY ) AS "MAR-APR",
       MONTHS_BETWEEN( DATE'2016-05-01', DATE'2016-05-01' - INTERVAL '1' DAY ) AS "APR-MAY"
FROM   DUAL;

Resultado:

JAN-FEB FEB-MAR MAR-APR APR-MAY
------- ------- ------- -------
0.03226 0.09677 0.03226 0.06452

A diferença em anos pode ser encontrada dividindo a diferença do mês por 12.

Gerando datas sem componente de tempo #

Todos os DATEs possuem um componente de hora; no entanto, é comum armazenar datas que não precisam incluir informações de hora com as horas/minutos/segundos definidos como zero (ou seja, meia-noite).

Use um literal ANSI DATE (usando [formato de data ISO 8601](https://en .wikipedia.org/wiki/ISO_8601#Calendar_dates)):

SELECT DATE '2000-01-01' FROM DUAL;

Converta-o de um literal de string usando TO_DATE():

SELECT TO_DATE( '2001-01-01', 'YYYY-MM-DD' ) FROM DUAL;

(Mais informações sobre os modelos de formato de data podem ser encontradas na documentação do Oracle.)

ou:

SELECT TO_DATE(
         'January 1, 2000, 00:00 A.M.',
         'Month dd, YYYY, HH12:MI A.M.',
         'NLS_DATE_LANGUAGE = American'
       )
FROM   DUAL;

(Se você estiver convertendo termos específicos do idioma, como nomes de meses, é uma boa prática incluir o terceiro parâmetro nlsparam na função TO_DATE() e especificar o idioma esperado.)

Gerando datas com um componente de tempo #

Converta-o de um literal de string usando TO_DATE():

SELECT TO_DATE( '2000-01-01 12:00:00', 'YYYY-MM-DD HH24:MI:SS' ) FROM DUAL;

Ou use um TIMESTAMP literal:

CREATE TABLE date_table(
  date_value DATE
);

INSERT INTO date_table ( date_value ) VALUES ( TIMESTAMP '2000-01-01 12:00:00' );

O Oracle converterá implicitamente um TIMESTAMP em um DATE ao armazená-lo em uma coluna DATE de uma tabela; no entanto, você pode explicitamente CAST() o valor para um DATE:

SELECT CAST( TIMESTAMP '2000-01-01 12:00:00' AS DATE ) FROM DUAL;

O formato de uma data #

No Oracle um tipo de dado DATE não tem formato; quando o Oracle envia um DATE para o programa cliente (SQL/Plus, SQL/Developer, Toad, Java, Python, etc) ele enviará 7 ou 8 bytes que representam a data.

Um DATE que não é armazenado em uma tabela (ou seja, gerado por SYSDATE e tendo “tipo 13” ao usar o comando DUMP()) tem 8 bytes e tem a estrutura (os números à direita são os representação interna de 2012-11-26 16:41:09):

BYTE VALUE                           EXAMPLE
---- ------------------------------- --------------------------------------
1    Year modulo 256                 220
2    Year multiples of 256           7   (7 * 256 + 220 = 2012)
3    Month                           11
4    Day                             26
5    Hours                           16
6    Minutes                         41
7    Seconds                         9
8    Unused                          0

Um DATE que é armazenado em uma tabela (“tipo 12” ao usar o comando DUMP()) tem 7 bytes e tem a estrutura (os números à direita são a representação interna de 2012-11-26 16:41:09):

BYTE VALUE                           EXAMPLE
---- ------------------------------- --------------------------------------
1    ( Year multiples of 100 ) + 100 120
2    ( Year modulo 100 ) + 100       112 ((120-100)*100 + (112-100) = 2012)
3    Month                           11
4    Day                             26
5    Hours + 1                       17
6    Minutes + 1                     42
7    Seconds + 1                     10

Se você quiser que a data tenha um formato específico, precisará convertê-la em algo que tenha um formato (ou seja, uma string). O cliente SQL pode fazer isso implicitamente ou você pode explicitamente [converter o valor em uma string](https://www.wikiod.com/pt/oracle/datas#Convertendo datas em uma string) usando TO_CHAR( date, format_model, nls_params ).

Convertendo datas em uma string #

Use TO_CHAR( date [, format_model [, nls_params]] ):

(Observação: se um modelo de formato não for fornecido, o parâmetro de sessão NLS_DATE_FORMAT será usado como o [modelo de formato padrão](https://www.wikiod.com/pt/oracle/datas#Configurando o modelo de formato de data padrão); isso pode ser diferente para cada sessão, portanto, não deve ser é uma boa prática sempre especificar o modelo de formato.)

CREATE TABLE table_name (
  date_value DATE
);

INSERT INTO table_name ( date_value ) VALUES ( DATE '2000-01-01' );
INSERT INTO table_name ( date_value ) VALUES ( TIMESTAMP '2016-07-21 08:00:00' );
INSERT INTO table_name ( date_value ) VALUES ( SYSDATE );

Então:

SELECT TO_CHAR( date_value, 'YYYY-MM-DD' ) AS formatted_date FROM table_name;

Saídas:

FORMATTED_DATE
--------------
2000-01-01
2016-07-21
2016-07-21

E:

SELECT TO_CHAR(
         date_value,
         'FMMonth d yyyy, hh12:mi:ss AM',
         'NLS_DATE_LANGUAGE = French'
       ) AS formatted_date
FROM   table_name;

Saídas:

FORMATTED_DATE
-----------------------------
Janvier   01 2000, 12:00:00 AM
Juillet   21 2016, 08:00:00 AM
Juillet   21 2016, 19:08:31 PM

Alterando como o SQL/Plus ou SQL Developer exibe as datas #

Quando o SQL/Plus ou o SQL Developer exibem datas, eles realizam uma conversão implícita em uma string usando o modelo de formato de data padrão (consulte [Definindo o modelo de formato de data padrão](https://www.wikiod.com/pt/oracle/datas date/6850/setting-the-default-date-format-model)).

Você pode alterar como uma data é exibida alterando o parâmetro NLS_DATE_FORMAT.

Extraia os componentes do ano, mês, dia, hora, minuto ou segundo de uma data #

Os componentes de ano, mês ou dia de um tipo de dados DATE podem ser encontrados usando o [EXTRACT( [ YEAR | MONTH | DAY ] FROM datevalue )](https://docs.oracle.com/cd/B19306_01/ server.102/b14200/functions050.htm)

SELECT EXTRACT (YEAR  FROM DATE '2016-07-25') AS YEAR,
       EXTRACT (MONTH FROM DATE '2016-07-25') AS MONTH,
       EXTRACT (DAY   FROM DATE '2016-07-25') AS DAY
FROM DUAL;

Saídas:

YEAR MONTH DAY
---- ----- ---
2016     7  25

Os componentes de tempo (hora, minuto ou segundo) podem ser encontrados por:

  • Usando CAST( datevalue AS TIMESTAMP ) para converter o DATE para um TIMESTAMP e depois usando EXTRACT( [ HORA | MINUTO | SEGUNDO ] FROM timestampvalue ); ou
  • Usando TO_CHAR( datevalue, format_model ) para obter o valor como uma string.

Por exemplo:

SELECT EXTRACT( HOUR   FROM CAST( datetime AS TIMESTAMP ) ) AS Hours,
       EXTRACT( MINUTE FROM CAST( datetime AS TIMESTAMP ) ) AS Minutes,
       EXTRACT( SECOND FROM CAST( datetime AS TIMESTAMP ) ) AS Seconds
FROM   (
  SELECT TO_DATE( '2016-01-01 09:42:01', 'YYYY-MM-DD HH24:MI:SS' ) AS datetime FROM DUAL
);

Saídas:

HOURS MINUTES SECONDS
----- ------- -------
    9      42       1

Fusos horários e horário de verão #

O tipo de dados DATE não lida com fusos horários ou alterações no horário de verão.

Qualquer:

  • use o tipo de dados TIMESTAMP WITH TIME ZONE; ou
  • lidar com as mudanças na lógica do seu aplicativo.

Um DATE pode ser armazenado como Coordinated Universal Time (UTC) e convertido para o fuso horário da sessão atual assim:

SELECT FROM_TZ(
         CAST(
           TO_DATE( '2016-01-01 12:00:00', 'YYYY-MM-DD HH24:MI:SS' )
           AS TIMESTAMP
         ),
         'UTC'
       )
       AT LOCAL AS time
FROM   DUAL;

Se você executar ALTER SESSION SET TIME_ZONE = '+01:00';, a saída será:

TIME
------------------------------------
2016-01-01 13:00:00.000000000 +01:00

e ALTER SESSION SET TIME_ZONE = 'PST'; então a saída é:

TIME
------------------------------------
2016-01-01 04:00:00.000000000 PST

Segundos bissextos #

Oracle não lida com segundos bissextos. Consulte a nota do My Oracle Support 2019397.2 e 730795.1 para obter mais detalhes.

Obtendo o dia da semana #

Você pode usar TO_CHAR( date_value, 'D' ) para obter o dia da semana.

No entanto, isso depende do parâmetro de sessão NLS_TERRITORY:

ALTER SESSION SET NLS_TERRITORY = 'AMERICA';        -- First day of week is Sunday
SELECT TO_CHAR( DATE '1970-01-01', 'D' ) FROM DUAL;

Saídas 5

ALTER SESSION SET NLS_TERRITORY = 'UNITED KINGDOM'; -- First day of week is Monday
SELECT TO_CHAR( DATE '1970-01-01', 'D' ) FROM DUAL;

Saídas 4

Para fazer isso independentemente das configurações de NLS, você pode truncar a data para meia-noite do dia atual (para remover quaisquer frações de dias) e subtrair a data truncada para o início da iso-semana atual (que sempre começa na segunda-feira ):

SELECT TRUNC( date_value ) - TRUNC( date_value, 'IW' ) + 1 FROM DUAL
