Manipulando valores NULL

  • Operações contendo NULL são NULL, exceto concatenação
  • NVL2 para obter um resultado diferente se um valor for nulo ou não

Nesta página

  • Operações contendo NULL são NULL, exceto concatenação
  • NVL2 para obter um resultado diferente se um valor for nulo ou não
  • COALESCE para retornar o primeiro valor não NULL
  • Colunas de qualquer tipo de dados podem conter NULLs
  • Strings vazias são NULL
  • NVL para substituir o valor nulo

Uma coluna é NULL quando não tem valor, independentemente do tipo de dados dessa coluna.

Uma coluna nunca deve ser comparada a NULL usando esta sintaxe a = NULL, pois o resultado seria UNKNOWN. Em vez disso, use as condições a IS NULL ou a IS NOT NULL. NULL não é igual a NULL. Para comparar duas expressões em que pode ocorrer null, use uma das funções descritas abaixo.

Todos os operadores, exceto a concatenação, retornam NULL se um de seus operandos for NULL. Por exemplo, o resultado de 3 * NULL + 5 é nulo.

NULL não pode aparecer em colunas restritas por uma PRIMARY KEY ou uma restrição NOT NULL. (Exceção é uma nova restrição com cláusula NOVALIDATE)

Operações contendo NULL são NULL, exceto concatenação #

SELECT 3 * NULL + 5, 'Hello ' || NULL || 'world'   from DUAL;
3*NULO+5 ‘OLÁ’||NULO||‘MUNDO’
(nulo) Olá mundo

NVL2 para obter um resultado diferente se um valor for nulo ou não #

Se o primeiro parâmetro for NOT NULL, NVL2 retornará o segundo parâmetro. Caso contrário, ele retornará o terceiro.

SELECT NVL2(null, 'Foo', 'Bar'), NVL2(5, 'Foo', 'Bar') FROM DUAL;
NVL2(NULL,‘FOO’,‘BAR’) NVL2(5,‘FOO’,‘BAR’)
Barra Foo

COALESCE para retornar o primeiro valor não NULL #

SELECT COALESCE(a, b, c, d, 5) FROM 
    (SELECT NULL A, NULL b, NULL c, 4 d FROM DUAL);
COALESCE(A,B,C,D,5)
4

Em alguns casos, usar COALESCE com dois parâmetros pode ser mais rápido do que usar NVL quando o segundo parâmetro não é uma constante. O NVL sempre avaliará ambos os parâmetros. COALESCE irá parar no primeiro valor não NULL que encontrar. Isso significa que se o primeiro valor não for NULL, COALESCE será mais rápido.

Colunas de qualquer tipo de dados podem conter NULLs #

SELECT 1 NUM_COLUMN, 'foo' VARCHAR2_COLUMN from DUAL
UNION ALL
SELECT NULL, NULL from DUAL;
NUM_COLUMN VARCHAR2_COLUMN
1 foo
(nulo) (nulo)

Strings vazias são NULL #

SELECT 1 a, '' b from DUAL;
A B
1 (nulo)

NVL para substituir o valor nulo #

SELECT a column_with_null, NVL(a, 'N/A') column_without_null FROM
  (SELECT NULL a FROM DUAL);
COLUMN_WITH_NULL COLUMN_WITHOUT_NULL
(nulo) N/A

NVL é útil para comparar dois valores que podem conter NULLs:

SELECT
    CASE WHEN a = b THEN 1 WHEN a <> b THEN 0 else -1 END comparison_without_nvl,
    CASE WHEN NVL(a, -1) = NVL(b, -1) THEN 1 WHEN NVL(a, -1) <> NVL(b, -1) THEN 0 else -1 END comparison_with_nvl
  FROM
    (select null a, 3 b FROM DUAL
     UNION ALL
     SELECT NULL, NULL FROM DUAL);
COMPARISON_WITHOUT_NVL COMPARISON_WITH_NVL
-1 0
-1 1
