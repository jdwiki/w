A seguir estão as etapas para criar uma importação/exportação de bomba de dados:

Monitorar trabalhos do Datapump #

Os trabalhos do Datapump podem ser monitorados usando

1. visualizações do dicionário de dados:

   select * from dba_datapump_jobs; 
   SELECT * FROM DBA_DATAPUMP_SESSIONS; 
   select username,opname,target_desc,sofar,totalwork,message from V$SESSION_LONGOPS where username = 'bkpadmin';

2. Status da bomba de dados:

  • Anote o nome do trabalho dos logs de importação/exportação ou nome do dicionário de dados e

  • Execute o comando anexar:

  • digite status no prompt Importar/Exportar

    impdp / attach=<SYS_IMPORT_SCHEMA_01> Import> status Pressione CTRL+C para sair do prompt Importar/Exportar

Etapa 3/6: Criar diretório #

create or replace directory DATAPUMP_REMOTE_DIR as '/oracle/scripts/expimp';

Etapa 7: Comandos de exportação #

Comandos:

expdp <bkpadmin>/<bkp123>  parfile=<exp.par>

*Por favor, substitua os dados em <> pelos valores apropriados de acordo com seu ambiente. Você pode adicionar/modificar parâmetros conforme suas necessidades. No exemplo acima, todos os parâmetros restantes são adicionados em arquivos de parâmetros conforme indicado abaixo: *

  • Tipo de exportação: Exportação do usuário

  • Exportar esquema inteiro

  • Detalhes do arquivo de parâmetro [digamos exp.par]:

    schemas= directory= DATAPUMP_REMOTE_DIR dumpfile=.dmp logfile=exp_.log

  • Tipo de exportação: Exportação do usuário para esquema grande

  • Exporte todo o esquema para grandes conjuntos de dados: aqui os arquivos de despejo de exportação serão divididos e compactados. Paralelismo é usado aqui (Nota: Adicionar paralelismo aumentará a carga da CPU no servidor)

  • Detalhes do arquivo de parâmetro [digamos exp.par]:

    schemas= directory= DATAPUMP_REMOTE_DIR dumpfile=%U.dmp logfile=exp__.log compression = all parallel=5

  • Tipo de exportação: Exportação de tabela [Exportar conjunto de tabelas]

  • Detalhes do arquivo de parâmetro [digamos exp.par]:

    tables= tname1, tname2, tname3 directory= DATAPUMP_REMOTE_DIR dumpfile=.dmp logfile=exp_.log

Etapa 9: Comandos de importação #

Pré-requisito:

  • Antes da importação do usuário, é uma boa prática descartar o esquema ou a tabela importada.

Comandos:

impdp <bkpadmin>/<bkp123>  parfile=<imp.par>

*Por favor, substitua os dados em <> pelos valores apropriados de acordo com seu ambiente. Você pode adicionar/modificar parâmetros conforme suas necessidades. No exemplo acima, todos os parâmetros restantes são adicionados em arquivos de parâmetros conforme indicado abaixo: *

  • Tipo de importação: Importação do usuário

  • Importar esquema inteiro

  • Detalhes do arquivo de parâmetro [digamos imp.par] :

    schemas= directory= DATAPUMP_REMOTE_DIR dumpfile=.dmp logfile=imp_.log

  • Tipo de importação: Importação do usuário para esquema grande

  • Importar esquema inteiro para grandes conjuntos de dados: o paralelismo é usado aqui (Nota: Adicionar paralelismo aumentará a carga da CPU no servidor)

  • Detalhes do arquivo de parâmetro [digamos imp.par] :

    schemas= directory= DATAPUMP_REMOTE_DIR dumpfile=%U.dmp logfile=imp__.log parallel=5

  • Tipo de Importação: Importação de Tabela [Importar conjunto de tabelas]

  • Detalhes do arquivo de parâmetro [digamos imp.par] :

    tables= tname1, tname2, tname3 directory= DATAPUMP_REMOTE_DIR dumpfile=.dmp logfile=exp_.log TABLE_EXISTS_ACTION= <APPEND /SKIP /TRUNCATE /REPLACE>

1. Etapas da bomba de dados #

Servidor de origem [Exportar dados] Servidor de destino [Importar dados]
1. Crie uma pasta datapump que conterá os arquivos dump de exportação 4. Crie uma pasta datapump que conterá os arquivos de dump de importação
2. Faça login no esquema do banco de dados que realizará a exportação. 5. Faça login no esquema do banco de dados que realizará a importação.
3. Crie um diretório apontando para a etapa 1. 6. Crie um diretório apontando para a etapa 4.
7. Execute as instruções de exportação.
8. Copie/SCP os arquivos de despejo para o servidor de destino.
9. Executar instruções de importação
10. verificar dados, compilar objetos inválidos e fornecer concessões relacionadas

Copiar tabelas entre diferentes esquemas e tablespaces #

 expdp <bkpadmin>/<bkp123> directory=DATAPUMP_REMOTE_DIR dumpfile=<customer.dmp>


 impdp <bkpadmin>/<bkp123> directory=DATAPUMP_REMOTE_DIR dumpfile=<customer.dmp> remap_schema=<source schema>:<target schema> remap_tablespace=<source tablespace>:<target tablespace>
