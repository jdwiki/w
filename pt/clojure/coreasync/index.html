operações básicas de canal: criação, colocação, retirada, fechamento e buffers. #

core.async é sobre fazer processos que pegam valores e colocam valores em canais.

(require [clojure.core.async :as a])

Criando canais com chan #

Você cria um canal usando a função chan:

(def chan-0 (a/chan)) ;; unbuffered channel: acts as a rendez-vous point.
(def chan-1 (a/chan 3)) ;; channel with a buffer of size 3. 
(def chan-2 (a/chan (a/dropping-buffer 3)) ;; channel with a *dropping* buffer of size 3
(def chan-3 (a/chan (a/sliding-buffer 3)) ;; channel with a *sliding* buffer of size 3

Colocando valores em canais com >!! e >! #

Você coloca valores em um canal com >!!:

(a/>!! my-channel :an-item)

Você pode colocar qualquer valor (Strings, números, mapas, coleções, objetos, até mesmo outros canais, etc.) em um canal, exceto nil:

;; WON'T WORK
(a/>!! my-channel nil)
=> IllegalArgumentException Can't put nil on channel

Dependendo do buffer do canal, >!! pode bloquear a thread atual.

(let [ch (a/chan)] ;; unbuffered channel
  (a/>!! ch :item) 
  ;; the above call blocks, until another process 
  ;; takes the item from the channel.
  )
(let [ch (a/chan 3)] ;; channel with 3-size buffer
  (a/>!! ch :item-1) ;; => true
  (a/>!! ch :item-2) ;; => true
  (a/>!! ch :item-3) ;; => true
  (a/>!! ch :item-4) 
  ;; now the buffer is full; blocks until :item-1 is taken from ch.
  )

De dentro de um bloco (go ...), você pode - e deve - usar a/>! em vez de a/>!!:

 (a/go (a/>! ch :item))

O comportamento lógico será o mesmo que a/>!!, mas apenas o processo lógico da goroutine bloqueará em vez do thread real do sistema operacional.

Usar a/>!! dentro de um bloco (go ...) é um anti-padrão:

;; NEVER DO THIS
(a/go 
  (a/>!! ch :item))

Obtendo valores de canais com <!! #

Você pega um valor de um canal usando <!!:

;; creating a channel
(def ch (a/chan 3))
;; putting some items in it
(do 
  (a/>!! ch :item-1)
  (a/>!! ch :item-2)
  (a/>!! ch :item-3))
;; taking a value
(a/<!! ch) ;; => :item-1
(a/<!! ch) ;; => :item-2

Se nenhum item estiver disponível no canal, a/<!! bloqueará o Thread atual até que um valor seja colocado no canal (ou o canal seja fechado, veja mais adiante):

(def ch (a/chan))
(a/<!! ch) ;; blocks until another process puts something into ch or closes it

De dentro de um bloco (go ...), você pode - e deve - usar a/<! em vez de a/<!!:

 (a/go (let [x (a/<! ch)] ...))

O comportamento lógico será o mesmo que a/<!!, mas apenas o processo lógico da goroutine bloqueará em vez do thread real do SO.

Usar a/<!! dentro de um bloco (go ...) é um anti-padrão:

;; NEVER DO THIS
(a/go 
  (a/<!! ch))

Fechando canais #

Você fecha um canal com a/close!:

(a/close! ch)

Uma vez que um canal é fechado e todos os dados no canal se esgotam, takes sempre retornarão nil:

(def ch (a/chan 5))

;; putting 2 values in the channel, then closing it
(a/>!! ch :item-1)
(a/>!! ch :item-2)
(a/close! ch)

;; taking from ch will return the items that were put in it, then nil
(a/<!! ch) ;; => :item-1
(a/<!! ch) ;; => :item-2
(a/<!! ch) ;; => nil
(a/<!! ch) ;; => nil
(a/<!! ch) ;; => nil

;; once the channel is closed, >!! will have no effect on the channel:
(a/>!! ch :item-3)
=> false ;; false means the put did not succeed
(a/<!! ch) ;; => nil

Puts assíncronas com put! #

Como alternativa a a/>!! (que pode bloquear), você pode chamar a/put! para colocar um valor em um canal em outra thread, com um callback opcional.

(a/put! ch :item)
(a/put! ch :item (fn once-put [closed?] ...)) ;; callback function, will receive

No ClojureScript, uma vez que bloquear o Thread atual não é possível, a/>!! não é suportado, e put! é a única maneira de colocar dados em um canal de fora de um bloco (go).

Takes assíncronos com take! #

Como alternativa para a/<!! (que pode bloquear a thread atual), você pode usar a/take! para pegar um valor de um canal de forma assíncrona, passando-o para um retorno de chamada.

(a/take! ch (fn [x] (do something with x)))

Usando buffers de soltar e deslizar #

Com buffers soltos e deslizantes, coloca never block, porém, quando o buffer está cheio, você perde dados. O buffer descartado perde os últimos dados adicionados, enquanto os buffers deslizantes perdem os primeiros dados adicionados.

Exemplo de buffer descartando:

(def ch (a/chan (a/dropping-buffer 2)))

;; putting more items than buffer size
(a/>!! ch :item-1)
=> true ;; put succeeded
(a/>!! ch :item-2)
=> true
(a/>!! ch :item-3)
=> false ;; put failed

;; no we take from the channel
(a/<!! ch)
=> :item-1
(a/<!! ch)
=> :item-2
(a/<!! ch)
;; blocks! :item-3 is lost

Exemplo de buffer deslizante:

(def ch (a/chan (a/sliding-buffer 2)))

;; putting more items than buffer size
(a/>!! ch :item-1)
=> true
(a/>!! ch :item-2)
=> true
(a/>!! ch :item-3)
=> true

;; no when we take from the channel:
(a/<!! ch)
=> :item-2
(a/<!! ch)
=> :item-3
;; :item-1 was lost
