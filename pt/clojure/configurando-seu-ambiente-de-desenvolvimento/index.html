Emacs #

Para configurar o Emacs para trabalhar com Clojure, instale o pacote clojure-mode e cider do melpa:

M-x package-install [RET] clojure-mode [RET]
M-x package-install [RET] cider [RET]

Agora, quando você abrir um arquivo .clj, execute M-x cider-jack-in para conectar a um REPL. Alternativamente, você pode usar C-u M-x (cider-jack-in) para especificar o nome de um projeto lein ou boot, sem ter que visitar nenhum arquivo nele. Agora você deve ser capaz de avaliar expressões em seu arquivo usando C-x C-e.

Editar código em linguagens semelhantes a lisp é muito mais confortável com um plug-in de edição com reconhecimento de pais. O Emacs tem várias boas opções.

  • paredit Um modo de edição Lisp clássico que tem uma curva de aprendizado mais acentuada, mas fornece muito poder uma vez dominado.

    M-x package-install [RET] paredit [RET]

  • smartparens Um projeto mais recente com objetivos e uso semelhantes ao paredit, mas também fornece recursos reduzidos com linguagens não-Lisp.

    M-x package-install [RET] smartparens [RET]

  • parinfer Um modo de edição Lisp muito mais simples que opera principalmente através da inferência de aninhamento de parênteses adequado a partir da indentação.

    Installation is more involved, see the Github page for parinfer-mode for setup instructions.

Para habilitar o paredit no clojure-mode:

(add-hook 'clojure-mode-hook #'paredit-mode)

Para habilitar smartparens no clojure-mode:

(add-hook 'clojure-mode-hook #'smartparens-strict-mode)

IntelliJ IDEA + Cursiva #

Baixe e instale a versão mais recente do IDEA.

Baixe e instale a versão mais recente do plugin Cursive.

Depois de reiniciar o IDEA, o Cursive deve estar funcionando imediatamente. Siga o guia do usuário para ajustar a aparência, combinações de teclas, estilo de código etc.

Observação: Assim como IntelliJ, Cursive é um produto comercial, com período de avaliação de 30 dias. Ao contrário do IntelliJ, ele não possui uma edição da comunidade. Licenças gratuitas não comerciais estão disponíveis para indivíduos para uso não comercial, incluindo hacking pessoal, código aberto e trabalho de estudantes. A licença é válida por 6 meses e pode ser renovada.

Átomo #

Instale o Atom para sua distribuição aqui.

Depois disso, execute os seguintes comandos de um terminal:

apm install parinfer
apm install language-clojure
apm install proto-repl

Spacemacs + CIDRA #

Spacemacs é uma distribuição do emacs que vem com muitos pacotes pré-configurados e facilmente instalados. Além disso, é muito amigável para aqueles que estão familiarizados com o estilo de edição do vim. A Spacemacs fornece uma camada Clojure baseada em CIDER.

Para instalá-lo e configurá-lo para uso com Clojure, primeiro instale o emacs. Em seguida, faça um backup de suas configurações anteriores:

$ mv ~/.emacs.d ~/.emacs.d.backup

Em seguida, clone o repositório dos spacemacs:

$ git clone https://github.com/syl20bnr/spacemacs ~/.emacs.d

Agora, abra o emacs. Ele fará algumas perguntas sobre suas preferências. Em seguida, ele baixa mais pacotes e configura seu emacs. Depois que o spacemacs estiver instalado, você estará pronto para adicionar suporte ao Clojure. Pressione SPC f e d para abrir seu arquivo .spacemacs para configuração. Encontre dotspacemacs-configuration-layers no arquivo, abaixo dele há um parêntese aberto. Em qualquer lugar entre os parênteses em uma nova linha digite clojure.

(defun dotspacemacs/layers ()
  (setq-default
   ;; ...
   dotspacemacs-configuration-layers
   '(clojure
     ;; ...
    )
   ;; ...
))

Pressione SPC f e R para salvar e instalar a camada de clojure. Agora, em qualquer arquivo .clj, se você pressionar , s i, o spacemacs tentará gerar uma nova conexão clojure REPL para seu projeto, e se for bem-sucedido, aparecerá na barra de status, que depois você poderá pressionar , s s para abrir um novo buffer REPL para avaliar seus códigos.

Para mais informações sobre spacemacs e cidra entre em contato com suas documentações. Documentos Spacemacs, Documentos Cider

##Vim Instale os seguintes plugins usando seu gerenciador de plugins favorito:

  1. fireplace.vim: Suporte Clojure REPL
  2. vim-sexp: Para domar aqueles abraços em torno de suas chamadas de função
  3. vim-sexp-mappings-for-regular-people: Mapeamentos sexp modificados que são um pouco mais fáceis de suportar
  4. vim-surround: Exclua, altere, adicione facilmente “surroundings” em par
  5. salve.vim: Suporte Static Vim para Leiningen e Boot.
  6. rainbow_parentheses.vim: Better Rainbow Parentheses

e também se relacionam com realce de sintaxe, conclusão omni, realce avançado e assim por diante:

  1. vim-clojure-static (se você tiver um vim anterior a 7.3.803, as versões mais recentes são fornecidas com isso)
  2. vim-clojure-highlight

Outras opções no lugar do vim-sexp incluem paredit.vim e [vim-parinfer](https://github.com/bhurlow/vim- parir).

Mesa de luz #

O Light Table é um bom editor para aprender, experimentar e executar projetos Clojure.
Você também pode executar projetos lein/boot abrindo o arquivo project.clj. Ele carregará todas as dependências do projeto.

Ele suporta avaliação em linha, plugins e muito mais, então não há necessidade de adicionar instruções de impressão e verificar a saída no console. Você pode executar linhas individuais ou blocos de código pressionando ctrl + enter. Para executar o código parcial, selecione o código e pressione ctrl + enter. verifique a captura de tela a seguir para saber como você pode usar o Light Table para aprender e experimentar o código Clojure.

Binários pré-construídos do Light Table podem ser encontrados aqui. Nenhuma configuração adicional é necessária.

Light Table é capaz de localizar automaticamente seu projeto Leiningen e avaliar seu código. Se você não tiver o Leiningen instalado, instale-o usando as instruções aqui.
