Crie um novo aplicativo Ring com http-kit #

Ring é uma API padrão de fato para aplicativos HTTP clojure, semelhante ao Rack do Ruby e WSGI do Python.

Vamos usá-lo com o servidor web http-kit.

Criar novo projeto Leiningen:

lein new app myapp

Adicione a dependência do http-kit ao project.clj:

  :dependencies [[org.clojure/clojure "1.8.0"]
                 [http-kit "2.1.18"]]

Adicione :require para http-kit em core.clj:

(ns test.core
  (:gen-class)
  (:require [org.httpkit.server :refer [run-server]]))

Defina o manipulador de solicitação de toque. Os manipuladores de solicitação são apenas funções da solicitação à resposta e a resposta é apenas um mapa:

(defn app [req]
  {:status  200
   :headers {"Content-Type" "text/html"}
   :body    "hello HTTP!"})

Aqui apenas retornamos 200 OK com o mesmo conteúdo para qualquer solicitação.

Inicie o servidor na função -main:

(defn -main
  [& args]
  (run-server app {:port 8080}))

Execute com lein run e abra http://localhost:8080/ no navegador.

Nova aplicação web com Luminus #

Luminus é um micro-framework Clojure baseado em um conjunto de bibliotecas leves. O objetivo é fornecer uma plataforma robusta, escalável e fácil de usar. Com Luminus, você pode se concentrar em desenvolver seu aplicativo da maneira que quiser, sem distrações. Ele também tem uma documentação muito boa que cobre alguns dos principais tópicos

É muito fácil começar com o luminus. Basta criar um novo projeto com os seguintes comandos:

lein new luminus my-app
cd my-app
lein run

Seu servidor iniciará na porta 3000

A execução de lein new luminus myapp criará um aplicativo usando o modelo de perfil padrão. No entanto, se desejar anexar mais funcionalidades ao seu modelo, você pode anexar dicas de perfil para a funcionalidade estendida.

Servidores Web #

  • +aleph - adiciona suporte ao servidor Aleph ao projeto
  • +jetty - adiciona suporte Jetty ao projeto
  • +http-kit - adiciona o servidor web HTTP Kit ao projeto

bancos de dados #

  • +h2 - adiciona namespace db.core e dependências de banco de dados H2
  • +sqlite - adiciona namespace db.core e dependências SQLite db
  • +postgres - adiciona namespace db.core e adiciona dependências PostreSQL
  • +mysql - adiciona namespace db.core e adiciona dependências do MySQL
  • +mongodb - adiciona o namespace db.core e as dependências do MongoDB
  • +datomic - adiciona namespace db.core e dependências Datomic

Diversos #

  • +auth - adiciona dependência do Buddy e middleware de autenticação
  • +auth-jwe - adiciona a dependência Buddy com o backend JWE
  • +cider - adiciona suporte para CIDER usando o plugin CIDER nREPL
  • +cljs - adiciona suporte a [ClojureScript][cljs] com Reagent
  • +re-frame - adiciona suporte [ClojureScript][cljs] com re-frame
  • +pepino - um perfil para pepino com clj-webdriver
  • +swagger - adiciona suporte para Swagger-UI usando a biblioteca compojure-api
  • +sassc - adiciona suporte para arquivos SASS/SCSS usando o compilador de linha de comando SassC
  • +service - cria um aplicativo de serviço sem o clichê de front-end, como modelos HTML
  • +war - adiciona suporte de construção de arquivos WAR para implantação em servidores como Apache Tomcat (NÃO deve ser usado para aplicativos Immutant executados no WildFly)
  • +site - cria modelo para site usando o banco de dados especificado (H2 por padrão) e ClojureScript

Para adicionar um perfil basta passá-lo como argumento após o nome do seu aplicativo, por exemplo:

lein new luminus myapp +cljs

Você também pode misturar vários perfis ao criar o aplicativo, por exemplo:

lein new luminus myapp +cljs +swagger +postgres
