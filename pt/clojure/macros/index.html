Sintaxe #

  • O símbolo ' usado no exemplo da macroexpansão é apenas um açúcar sintático para o operador quote. Você poderia ter escrito (macroexpand (quote (infix 1 + 2))) em vez disso.

Macros são apenas funções que são executadas em tempo de compilação, ou seja, durante a etapa eval em um read-eval-print-loop.

As macros de leitura são outra forma de macro que é expandida no tempo de leitura, em vez de no tempo de compilação.

Melhor prática ao definir macro.

  • alfa-renomeação, uma vez que a macro é expandir o conflito de nomes de ligação pode surgir. O conflito de ligação não é muito intuitivo de resolver ao usar a macro. É por isso que, sempre que uma macro adiciona uma ligação ao escopo, é obrigatório usar o # no final de cada símbolo.

Macro Infixo Simples #

Clojure usa notação de prefixo, ou seja: O operador vem antes de seus operandos.

Por exemplo, uma simples soma de dois números seria:

(+ 1 2)
;; => 3

Macros permitem que você manipule a linguagem Clojure até certo ponto. Por exemplo, você pode implementar uma macro que permita escrever código em notação infixa (por exemplo, 1 + 2):

(defmacro infix [first-operand operator second-operand]
    "Converts an infix expression into a prefix expression"
    (list operator first-operand second-operand))

Vamos detalhar o que o código acima faz:

  • defmacro é um formulário especial que você usa para definir uma macro.
  • infix é o nome da macro que estamos definindo.
  • [primeiro operando operador segundo operando] são os parâmetros que esta macro espera receber quando for chamada.
  • (list operator first-operand second-operand) é o corpo da nossa macro. Ele simplesmente cria uma lista com os valores dos parâmetros fornecidos à macro infix e retorna isso.

defmacro é um formulário especial porque se comporta um pouco diferente em comparação com outras construções Clojure: Seus parâmetros não são avaliados imediatamente (quando chamamos a macro). Isto é o que nos permite escrever algo como:

(infix 1 + 2)
;; => 3

A macro infix expandirá os argumentos 1 + 2 para (+ 1 2), que é uma forma válida do Clojure que pode ser avaliada.

Se você quiser ver o que a macro infix gera, você pode usar o operador macroexpand:

(macroexpand '(infix 1 + 2))
;; => (+ 1 2)

macroexpand, como está implícito em seu nome, expandirá a macro (neste caso, usará a macro infix para transformar 1 + 2 em (+ 1 2)), mas não permitirá o resultado da macroexpansão a ser avaliada pelo interpretador de Clojure.

Sintaxe citando e descitando #

Exemplo da biblioteca padrão (core.clj:807):

(defmacro and
  "Evaluates exprs one at a time, from left to right. If a form
  returns logical false (nil or false), and returns that value and
  doesn't evaluate any of the other expressions, otherwise it returns
  the value of the last expr. (and) returns true."
  {:added "1.0"}
  ([] true)
  ([x] x)
  ([x & next]
   `(let [and# ~x]
      (if and# (and ~@next) and#))))
  • ` chamado syntax-quote é como (quote), mas recursiva: causa (let …), (if …) , etc para não avaliar durante a expansão da macro, mas para produzir como está
  • ~ aka unquote cancela a aspas de sintaxe para um único formulário dentro do formulário com aspas de sintaxe. Portanto, o valor de x é gerado ao expandir a macro (em vez de gerar o símbolo x)
  • ~@ aka unquote-splicing é como unquote, mas pega o argumento da lista e o expande, cada item da lista para separar o formulário
  • # anexa id exclusivo aos símbolos para evitar conflitos de nome. Ele anexa o mesmo id para o mesmo símbolo dentro da expressão entre aspas de sintaxe, então and# dentro de let e and# dentro de if terá o mesmo nome
