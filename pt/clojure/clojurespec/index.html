Sintaxe #

  • :: é uma abreviação de uma palavra-chave qualificada por namespace. Por exemplo. se estivermos no namespace user: ::foo é uma abreviação de :user/foo
  • #: ou # - sintaxe map-literal para qualificar chaves em um mapa por um namespace

Clojure spec é uma nova biblioteca de especificações/contratos para clojure disponível a partir da versão 1.9.

As especificações são aproveitadas de várias maneiras, incluindo a inclusão na documentação, validação de dados, geração de dados para teste e muito mais.

Usando um predicado como especificação #

Qualquer função de predicado pode ser usada como uma especificação. Aqui está um exemplo simples:

(clojure.spec/valid? odd? 1)
;;=> true

(clojure.spec/valid? odd? 2)
;;=> false

a função valid? pegará uma especificação e um valor e retornará true se o valor estiver de acordo com a especificação e false caso contrário.

Um outro predicado interessante é a associação definida:

(s/valid? #{:red :green :blue} :red) 
;;=> true

fdef: escrevendo uma especificação para uma função #

Digamos que temos a seguinte função:

(defn nat-num-count [nums] (count (remove neg? nums)))

Podemos escrever uma especificação para esta função definindo uma especificação de função com o mesmo nome:

(clojure.spec/fdef nat-num-count
        :args (s/cat :nums (s/coll-of number?))
        :ret integer?
        :fn #(<= (:ret %) (-> % :args :nums count)))

:args recebe uma especificação regex que descreve a sequência de argumentos por um rótulo de palavra-chave correspondente ao nome do argumento e uma especificação correspondente. A razão pela qual a especificação exigida por :args é uma especificação regex é para suportar múltiplas aridades para uma função. :ret especifica uma especificação para o valor de retorno da função.

:fn é uma especificação que restringe o relacionamento entre o :args e o :ret. Ele é usado como uma propriedade quando executado por meio de test.check. Ele é chamado com um único argumento: um mapa com duas chaves: :args (os argumentos conformados para a função) e :ret (o valor de retorno conformado da função).

Registrando uma especificação #

Além dos predicados funcionando como especificações, você pode registrar uma especificação globalmente usando clojure.spec/def. def requer que uma especificação sendo registrada seja nomeada por uma palavra-chave qualificada por namespace:

(clojure.spec/def ::odd-nums odd?)
;;=> :user/odd-nums

(clojure.spec/valid? ::odd-nums 1)
;;=> true
(clojure.spec/valid? ::odd-nums 2)
;;=> false

Uma vez registrada, uma especificação pode ser referenciada globalmente em qualquer lugar em um programa Clojure.

A sintaxe ::odd-nums é um atalho para :user/odd-nums, assumindo que estamos no namespace user. :: qualificará o símbolo que precede com o namespace atual.

Em vez de passar o predicado, podemos passar o nome da especificação para valid?, e funcionará da mesma maneira.

clojure.spec/e & clojure.spec/ou #

clojure.spec/and & clojure.spec/or podem ser usados ​​para criar especificações mais complexas, usando várias especificações ou predicados:

(clojure.spec/def ::pos-odd (clojure.spec/and odd? pos?))

(clojure.spec/valid? ::pos-odd 1)
;;=> true

(clojure.spec/valid? ::pos-odd -3)
;;=> false

or funciona de forma semelhante, com uma diferença significativa. Ao definir uma especificação or, você deve marcar cada ramo possível com uma palavra-chave. Isso é usado para fornecer ramificações específicas que falham nas mensagens de erro:

(clojure.spec/def ::big-or-small (clojure.spec/or :small #(< % 10) :big #(> % 100)))

(clojure.spec/valid? ::big-or-small 1)
;;=> true

(clojure.spec/valid? ::big-or-small 150)
;;=> true

(clojure.spec/valid? ::big-or-small 20)
;;=> false

Ao conformar uma especificação usando or, a especificação aplicável será retornada, tornando o valor conforme:

(clojure.spec/conform ::big-or-small 5)
;; => [:small 5]

Especificações do registro #

Você pode especificar um registro da seguinte maneira:

(clojure.spec/def ::name string?)
(clojure.spec/def ::age pos-int?)
(clojure.spec/def ::occupation string?)

(defrecord Person [name age occupation])

(clojure.spec/def ::person (clojure.spec/keys :req-un [::name ::age ::occupation]))

(clojure.spec/valid? ::person (->Person "john doe" 25 "programmer"))
;;=> true

(clojure.spec/valid? ::person (->Person "john doe" "25" "programmer"))
;;=> false

Em algum momento no futuro, uma sintaxe de leitor ou suporte interno para qualificar chaves de registro pelo namespace dos registros pode ser introduzido. Este suporte já existe para mapas.

Especificações do mapa #

Você pode especificar um mapa especificando quais chaves devem estar presentes no mapa:

(clojure.spec/def ::name string?)
(clojure.spec/def ::age pos-int?)
(clojure.spec/def ::occupation string?)

(clojure.spec/def ::person (clojure.spec/keys :req [::name ::age ::occupation]))

(clojure.spec/valid? ::person {::name "john" ::age 25 ::occupation "programmer"})
;; => true

:req é um vetor de chaves que devem estar presentes no mapa. Você pode especificar opções adicionais como :opt, um vetor de chaves que são opcionais.

Os exemplos até agora exigem que as chaves no nome sejam qualificadas por namespace. Mas é comum que as chaves do mapa não sejam qualificadas. Para este caso, clojure.spec fornece :req e :opt equivalentes para chaves não qualificadas: :req-un e :opt-un. Aqui está o mesmo exemplo, com chaves não qualificadas:

(clojure.spec/def ::name string?)
(clojure.spec/def ::age pos-int?)
(clojure.spec/def ::occupation string?)

(clojure.spec/def ::person (clojure.spec/keys :req-un [::name ::age ::occupation]))

(clojure.spec/valid? ::person {:name "john" :age 25 :occupation "programmer"})
;; => true

Observe como as especificações fornecidas no vetor :req-un ainda são qualificadas. clojure.spec, confirmará automaticamente as versões não qualificadas no mapa ao conformar os valores.

A sintaxe literal do mapa de namespace permite que você qualifique todas as chaves de um mapa por um único namespace de forma sucinta. Por exemplo:

(clojure.spec/def ::name string?)
(clojure.spec/def ::age pos-int?)
(clojure.spec/def ::occupation string?)

(clojure.spec/def ::person (clojure.spec/keys :req [::name ::age ::occupation]))

(clojure.spec/valid? ::person #:user{:name "john" :age 25 :occupation "programmer"})
;;=> true

Observe a sintaxe especial do leitor #:. Seguimos com o namespace pelo qual desejamos qualificar todas as chaves do mapa. Eles serão verificados em relação às especificações correspondentes ao namespace fornecido.

Coleções #

Você pode especificar coleções de várias maneiras. coll-of permite especificar coleções e fornecer algumas restrições adicionais. Aqui está um exemplo simples:

(clojure.spec/valid? (clojure.spec/coll-of int?) [1 2 3])
;; => true

(clojure.spec/valid? (clojure.spec/coll-of int?) '(1 2 3))
;; => true

As opções de restrição seguem a especificação/predicado principal da coleção. Você pode restringir o tipo de coleção com :kind assim:

(clojure.spec/valid? (clojure.spec/coll-of int? :kind vector?) [1 2 3])
;; => true

(clojure.spec/valid? (clojure.spec/coll-of int? :kind vector?) '(1 2 3))
;; => false

O acima é falso porque a coleção passada não é um vetor.

(clojure.spec/valid? (clojure.spec/coll-of int? :kind list?) '(1 2 3))
;; => true

(clojure.spec/valid? (clojure.spec/coll-of int? :kind set?) #{1 2 3})
;; => true

(clojure.spec/valid? (clojure.spec/coll-of int? :kind set?) #{1 "2" 3})
;; => false

O acima é falso porque nem todos os elementos do conjunto são ints.

Você também pode restringir o tamanho da coleção de algumas maneiras:

(clojure.spec/valid? (clojure.spec/coll-of int? :kind vector? :count 3) [1 2 3])
;; => true

    (clojure.spec/valid? (clojure.spec/coll-of int? :kind vector? :count 3) [1 2])
;; => false

(clojure.spec/valid? (clojure.spec/coll-of int? :min-count 3 :max-count 5) [1 2 3])
;; => true

    (clojure.spec/valid? (clojure.spec/coll-of int? :min-count 3 :max-count 5) [1 2])
;; => false

Você também pode impor a exclusividade dos elementos na coleção com :distinct:

(clojure.spec/valid? (clojure.spec/coll-of int? :distinct true) [1 2])
;; => true

(clojure.spec/valid? (clojure.spec/coll-of int? :distinct true) [2 2])
;; => false

coll-of garante que todos os elementos em uma sequência sejam verificados. Para grandes coleções, isso pode ser muito ineficiente. every se comporta exatamente como coll-of, exceto que apenas amostra um número relativamente pequeno de elementos das sequências para conformidade. Isso funciona bem para grandes coleções. Aqui está um exemplo:

(clojure.spec/valid? (clojure.spec/every int? :distinct true) [1 2 3 4 5])
;; => true

map-of é semelhante a coll-of, mas para mapas. Como os mapas têm chaves e valores, você deve fornecer uma especificação para a chave e uma especificação para o valor:

(clojure.spec/valid? (clojure.spec/map-of keyword? string?) {:red "red" :green "green"})
;; => true

Como coll-of, map-of verifica a conformidade de todas as chaves/valores do mapa. Para mapas grandes, isso será ineficiente. Como coll-of, map-of fornece every-kv para amostragem eficiente de um número relativamente pequeno de valores de um mapa grande:

(clojure.spec/valid? (clojure.spec/every-kv keyword? string?) {:red "red" :green "green"})
;; => true

Sequências #

spec pode descrever e ser usado com sequências arbitrárias. Ele suporta isso por meio de várias operações de especificação de regex.

(clojure.spec/valid? (clojure.spec/cat :text string? :int int?) ["test" 1])
;;=> true

cat requer rótulos para cada especificação usada para descrever a sequência. cat descreve uma sequência de elementos e uma especificação para cada um.

alt é usado para escolher entre várias especificações possíveis para um determinado elemento em uma sequência. Por exemplo:

(clojure.spec/valid? (clojure.spec/cat :text-or-int (clojure.spec/alt :text string? :int int?)) ["test"])
;;=> true

alt também requer que cada especificação seja rotulada por uma palavra-chave.

As sequências Regex podem ser compostas de maneiras muito interessantes e poderosas para criar especificações de descrição de sequências arbitrariamente complexas. Aqui está um exemplo um pouco mais complexo:

(clojure.spec/def ::complex-seq (clojure.spec/+ (clojure.spec/cat :num int? :foo-map (clojure.spec/map-of keyword? int?))))
(clojure.spec/valid? ::complex-seq [0 {:foo 3 :baz 1} 4 {:foo 4}])
;;=> true

Aqui ::complex-seq validará uma sequência de um ou mais pares de elementos, sendo o primeiro um int e o segundo um mapa de palavras-chave para int.
