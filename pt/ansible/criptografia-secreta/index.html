Criptografia secreta

  • Criptografando dados estruturados confidenciais
  • Usando pipes de pesquisa para descriptografar dados criptografados em cofre não estruturados
  • Usando local_action para descriptografar modelos criptografados em cofre

Nesta página

  • Criptografando dados estruturados confidenciais
  • Usando pipes de pesquisa para descriptografar dados criptografados em cofre não estruturados
  • Usando local_action para descriptografar modelos criptografados em cofre

O Ansible oferece o Vault (não deve ser confundido com o HashiCorp Vault!) para lidar com dados confidenciais criptografia. O Vault visa principalmente criptografar quaisquer dados estruturados, como variáveis, tarefas, manipuladores.

Criptografando dados estruturados confidenciais #

Primeiro, crie um arquivo de chave, por exemplo, vault_pass_file, que idealmente contém uma longa sequência de caracteres aleatórios. Em sistemas linux você pode usar pwgen para criar um arquivo de senha aleatório:

pwgen 256 1 > vault_pass_file

Em seguida, use este arquivo para criptografar dados confidenciais, por exemplo, groups_vars/group.yml:

ANSIBLE_VAULT_PASSWORD_FILE=vault_pass_file ansible-vault encrypt group_vars/group.yml

De agora em diante, para rodar um playbook você precisa do vault_pass_file:

ANSIBLE_VAULT_PASSWORD_FILE=vault_pass_file ansible-playbook -i inventories/nodes my-playbook.yml

Observe que você também pode usar o sinalizador --vault-password-file vault_pass_file em vez de definir a variável de ambiente ANSIBLE_VAULT_PASSWORD_FILE.

Para editar ou descriptografar o segredo no disco, você pode usar ansible-vault edit e ansible-vault decrypt respectivamente.

Usando pipes de pesquisa para descriptografar dados criptografados em cofre não estruturados #

Com o Vault você também pode criptografar dados não estruturados, como arquivos de chave privada e ainda ser capaz de descriptografá-los em seu jogo com o módulo lookup.

---

- name: Copy private key to destination
  copy:
    dest=/home/user/.ssh/id_rsa
    mode=0600
    content=lookup('pipe', 'ANSIBLE_VAULT_PASSWORD_FILE=vault_pass_file ansible-vault view keys/private_key.enc')

Usando local_action para descriptografar modelos criptografados em cofre #

Você pode executar um jogo que depende de modelos criptografados em cofre usando o módulo local_action.

---

- name: Decrypt template
  local_action: "shell {{ view_encrypted_file_cmd }} {{ role_path }}/templates/template.enc > {{ role_path }}/templates/template"
  changed_when: False

- name: Deploy template
  template:
    src=templates/template
    dest=/home/user/file

- name: Remove decrypted template
  local_action: "file path={{ role_path }}/templates/template state=absent"
  changed_when: False

Observe o changed_when: False. Isso é importante caso você execute testes de idempotência com seus papéis ansible - caso contrário, cada vez que você executar o playbook, uma mudança será sinalizada. Em group_vars/all.yml você pode definir um comando global decrypt para reutilização, por exemplo, como view_encrypted_file_cmd.

group_vars/all.yml

---

view_encrypted_file_cmd: "ansible-vault --vault-password-file {{ lookup('env', 'ANSIBLE_VAULT_PASSWORD_FILE') }} view"

Agora, ao executar uma jogada, você precisa definir a variável de ambiente ANSIBLE_VAULT_PASSWORD_FILE para apontar para o arquivo de senha do cofre (idealmente com um caminho absoluto).
