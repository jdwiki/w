Instalando o Ansible no Ubuntu #

O Ansible mantém um repositório PPA que pode ser usado para instalar os binários do Ansible:

sudo apt-add-repository ppa:ansible/ansible -y
sudo apt-get update && sudo apt-get install ansible -y

Para instalar uma versão específica, use pip. O PPA pode estar desatualizado.

Instalando a partir da fonte #

O Ansible é melhor usado em um checkout.

Ele é executado como você (não root) e possui dependências mínimas de python.

Instalação da dependência do pip do Python com pip:

sudo pip install paramiko PyYAML Jinja2 httplib2 six

Em seguida, clone o Ansible repo do GitHub:

cd ~/Documents
git clone git://github.com/ansible/ansible.git --recursive 
cd ansible

Por fim, adicione a linha de script de inicialização ansible ao seu ~/.bashrc ou ~/.zshrc :

source ~/Documents/ansible/hacking/env-setup

Reinicie sua sessão de terminal e teste com

ansible --version

Instalando o Ansible no MacOS #

Existem duas maneiras principais de instalar o Ansible no OS X, usando o gerenciador de pacotes Homebrew ou Pip.

Se você tiver homebrew, o Ansible mais recente pode ser instalado usando o seguinte comando:

brew install ansible

Para instalar o branch Ansible 1.9.X, use o seguinte comando:

brew install homebrew/versions/ansible19

Para instalar o branch Ansible 2.0.X, use o seguinte comando:

brew install homebrew/versions/ansible20

Para instalar usando pip, use o seguinte comando: pip install ansible.

Para instalar uma versão específica, use pip install ansible=<required version>.

Instalação em sistemas baseados em Red Hat #

O Ansible pode ser instalado no CentOS ou em outros sistemas baseados em Red Hat. Primeiramente você deve instalar os pré-requisitos:

sudo yum -y update
sudo yum -y install gcc libffi-devel openssl-devel python-pip python-devel

em seguida, instale o Ansible com pip:

sudo pip install ansible

Eu posso recomendar para você atualizar as ferramentas de configuração após a instalação:

sudo pip install --upgrade setuptools

Você também pode usar o Gerenciador de Pacotes local:

yum install ansible

Instalação no Amazon Linux a partir do repositório git #

O Amazon Linux é uma variante do RHEL, portanto, as instruções do Red Hat devem funcionar na maior parte. Há, no entanto, pelo menos uma discrepância.

Houve uma instância em que o pacote python27-devel, em vez de python-devel, era explicitamente necessário.

Aqui, vamos instalar a partir da fonte.

sudo yum -y update
sudo yum -y install python27 python27-devel openssl-devel libffi-devel gcc git

git clone https://github.com/ansible/ansible/<search the github for a preferable branch>

cd ansible
sudo python setup.py build
sudo python setup.py install

Instalando o Ansible em qualquer máquina com sistema operacional (windows) usando o Virtual Box + Vagrant #

Meu laptop está com o Windows 10. Aqui estou dando as etapas que você pode seguir para testar e aprender o Ansible.

ALGUMAS TEORIA

Para o Ansible, você precisa de uma máquina de controle e um host (ou hosts) para executar o Playbook.

  • Control Machine deve ser baseado em Linux ou MacOS (Windows não permitido) e precisa de Python (versão 2.6 ou superior). Aqui o Ansible será instalado.
  • Máquina de destino (host/nó) pode ser Linux/MacOS/windows. Isso precisa apenas do Python para ser instalado. Nenhum software de agente é necessário.

CONFIGURAR

Etapa 1: Instalar Virtual Box

Caixa virtual é um software para criar computadores virtuais de diferentes sistemas operacionais. É como ter vários computadores cada ou diferentes sistemas operacionais e versões diferentes.

Baixe o Virtual Box de acordo com o SO em seu sistema e instale-o.

Etapa 2: Instalar Vagrant

Vagrant é uma interface de linha de comando para criar máquinas virtuais em uma caixa virtual. Isso facilita as coisas. Você precisa aprender comandos básicos do Vagrant.

Etapa 3: Crie uma pasta onde você deseja sua máquina virtual

Etapa 4: Crie uma máquina virtual usando o Vagrant

Abra o terminal e vá para o caminho onde você criou a pasta e execute os dois comandos a seguir.

Você precisa selecionar Virtual Box. Estou instalando o Ubuntu, por exemplo. Você pode escolher qualquer coisa da lista. Você precisa executar esses dois comandos na categoria “virtual box”: vagrant init ubuntu/trusty64 e vagrant up --provider virtualbox. Outras categorias podem ser: hyperv, vmware_desktop etc. (isso levará algum tempo, pois fará o download dos arquivos necessários)

Etapa 4: Instale o Ansible

Para UbuntuOS: sudo apt-get install ansible

Solução alternativa: #

Você pode usar Katacoda para praticar ansible. Não há necessidade de instalar ou configurar nada. Execute dois comandos fornecidos na etapa 2 e depois disso, você está pronto para ir.
