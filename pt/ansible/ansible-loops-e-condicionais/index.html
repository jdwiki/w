Que tipos de condicionais usar? #

Use condicionais via (a sintaxe está em [colchetes]):

  • quando quando:**]

    Task:
   - name: run if operating system is debian
     command: echo "I am a Debian Computer"
     when: ansible_os_family == "Debian"

  • loops [with_items:]

  • loops [with_dicts:]

  • Fatos personalizados [ quando: my_custom_facts == ‘1234’]

  • Importações condicionais

Quando Condição #

Uso Básico #

Use a condição when para controlar se uma tarefa ou função é executada ou ignorada. Isso normalmente é usado para alterar o comportamento do jogo com base em fatos do sistema de destino. Considere este manual:

- hosts: all
  tasks:
    - include: Ubuntu.yml
      when: ansible_os_family == "Ubuntu"
    
    - include: RHEL.yml
      when: ansible_os_family == "RedHat"

Onde Ubuntu.yml e RHEL.yml incluem alguma lógica específica de distribuição.

Outro uso comum é limitar os resultados àqueles em determinados grupos de inventário do Ansible. Considere este arquivo de inventário:

[dbs]
mydb01

[webservers]
myweb01

E esta cartilha:

- hosts: all
  tasks:
    - name: Restart Apache on webservers
      become: yes
      service:
        name: apache2
        state: restarted
      when: webservers in group_names

Isso está usando o group_names variável mágica.

Sintaxe Condicional e Lógica #

Condição única #

Sintaxe

quando: (condição)

Exemplo

  • quando: ansible_os_family == "Debian"
  • quando: ansible_pkg_mgr == "apt"
  • quando: myvariablename está definido

Filtro booleano #

Exemplo

quando: resultado|falha

Várias condições #

Sintaxe

Quando: condição1 e/ou condição2

Exemplo (simples)

quando: ansible_os_family == "Debian" e ansible_pkg_mgr == "apt"

Exemplo (complexo)

Use parênteses para maior clareza ou para controlar a precedência. “AND” tem precedência maior que “OR”.

As cláusulas podem abranger linhas:

when:
  ansible_distribution in ['RedHat', 'CentOS', 'ScientificLinux'] and
  (ansible_distribution_version|version_compare('7', '<') or
  ansible_distribution_version|version_compare('8', '>='))
  or
  ansible_distribution == 'Fedora'
  or
  ansible_distribution == 'Ubuntu' and
  ansible_distribution_version|version_compare('15.04', '>=')

Observe o uso de parênteses para agrupar o “ou” na primeira verificação de distribuição.

Obtenha ansible_os_family e ansible_pkg_mgr com configuração #

Podemos obter fatos (ansible_os_family, ansible_pkg_mgr) com o comando Ad-Hoc do módulo de configuração e filtro.

  • ansible_os_family:

      $ ansible all -m setup -a 'filter=ansible_os_family'
  ra.local | SUCCESS => {
      "ansible_facts": {
          "ansible_os_family": "Debian"
      },
      "changed": false
  }

  • ansible_pkg_mgr:

      $ ansible all -m setup -a 'filter=ansible_pkg_mgr'
  debian.local | SUCCESS => {
      "ansible_facts": {
          "ansible_pkg_mgr": "apt"
      },
      "changed": false
  }

Exemplo(s) simples de “quando” #

Dado:

---
variable_name: True

Então, essas tarefas sempre são executadas.

- name: This is a conditional task
  module: src=/example/ dest=/example
  when: variable_name 

- name: This is a conditional task
  module: src=/example/ dest=/example
  when: True

Esta tarefa nunca será executada.

- name: This is a conditional task
  module: src=/example/ dest=/example
  when: False

Usando until para uma nova tentativa de loop ao vivo #

Este é um exemplo de uso de until/retries/delay para implementar uma verificação ativa para um aplicativo da web que está sendo inicializado. Ele assume que haverá algum período de tempo (até 3 minutos) em que o aplicativo da web está recusando conexões de soquete. Depois disso, ele verifica na página /alive a palavra “OK”. Ele também delega a recuperação da URL para o localhost executando o ansible. Isso faz sentido como a tarefa final em um manual de implantação.

---
- hosts: my-hosts
  tasks:
  - action: uri url=http://{{ ansible_all_ipv4_addresses }}:8080/alive return_content=yes
    delegate_to: localhost
    register: result
    until: "'failed' not in result and result.content.find('OK') != -1"
    retries: 18
    delay: 10

O padrão de repetição até pode ser usado com qualquer ação; A documentação do Ansible fornece um exemplo de como esperar até que um determinado comando shell retorne um resultado desejado: http://docs.ansible.com/ansible/playbooks_loops.html#do-until-loops.
