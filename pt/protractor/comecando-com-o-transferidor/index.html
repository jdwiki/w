Instalando e configurando o transferidor (no Windows) #

Requisitos: O Transferidor requer que as seguintes dependências sejam instaladas antes da instalação:

  • Java JDK 1.7 ou superior
  • Node.js v4 ou superior

Instalação:
Baixe e instale o Node.js a partir deste URL: https://nodejs.org/en/

Para ver se a instalação do Node.js foi bem sucedida, você pode ir e verificar as variáveis ​​de ambiente. O ‘Caminho’ em Variáveis ​​do Sistema será atualizado automaticamente.


Você também pode verificar o mesmo digitando o comando npm -version no prompt de comando que lhe dará a versão instalada.


Agora o Protractor pode ser instalado de duas maneiras: Localmente ou Globalmente.

Podemos instalar o transferidor em uma pasta específica ou local do diretório do projeto. Se instalarmos em um diretório de projeto, toda vez que executarmos, devemos executar apenas a partir desse local.

Para instalar localmente no diretório do projeto, navegue até a pasta do projeto e digite o comando

npm install transferidor


Para instalar o Protractor globalmente, execute o comando:

$ npm install -g protractor

Isso instalará duas ferramentas de linha de comando, transferidor e webdriver-manager. Execute transferidor --version para garantir que o transferidor foi instalado com sucesso.

webdriver-manager é usado para baixar os binários do driver do navegador e iniciar o servidor selenium. Download the browser driver binaries with:

$ webdriver-manager update

Inicie o servidor selenium com:

$ webdriver-manager start

Para baixar o driver do Internet Explorer, execute o comando webdriver-manager update --ie no prompt de comando. Isso fará o download do IEDriverServer.exe na sua pasta de selênio

Primeiro teste usando o Transferidor #

O transferidor precisa de apenas dois arquivos para executar o primeiro teste, arquivo de especificação (código de teste) e arquivo de configuração. O arquivo de especificação contém o código de teste e o outro contém detalhes de configuração como caminho do arquivo de especificação, detalhes do navegador, URL de teste, parâmetros do framework etc. Para escrever o primeiro teste, forneceremos apenas o endereço do servidor selenium e o caminho do arquivo de especificação. , timeout, framework será escolhido para os valores padrão.

O navegador padrão do Protractor é o Chrome.

conf.js - Arquivo de configuração

exports.config = {
  seleniumAddress: 'http://localhost:4444/wd/hub',
  specs: ['spec.js']
};

spec.js - arquivo Spec (código de teste)

describe('first test in protractor', function() {
  it('should verify title', function() {
    browser.get('https://angularjs.org');

    expect(browser.getTitle()).toEqual('AngularJS — Superheroic JavaScript MVW Framework');
  });
});

seleniumAddress - Caminho para o servidor onde o servidor webdriver está sendo executado.

specs - Um elemento de matriz que contém o caminho dos arquivos de teste. Os vários caminhos podem ser especificados por valores separados por vírgula.

describe - Sintaxe da estrutura Jasmine. sintaxe describe sta

Testes de execução seletivos #

O transferidor pode executar seletivamente grupos de testes usando fdescribe() em vez de describe().

fdescribe('first group',()=>{
    it('only this test will run',()=>{
        //code that will run
    });
});
describe('second group',()=>{
    it('this code will not run',()=>{
        //code that won't run
    });
});

O transferidor pode executar seletivamente testes dentro de grupos usando fit() em vez de it().

describe('first group',()=>{
    fit('only this test will run',()=>{
        //code that will run
    });
    it('this code will not run',()=>{
        //code that won't run
    });
});

Se não houver fit() dentro de um fdescribe(), então cada it() será executado. No entanto, um fit() bloqueará chamadas it() dentro do mesmo describe() ou fdescribe().

fdescribe('first group',()=>{
    fit('only this test will run',()=>{
        //code that will run
    });
    it('this code will not run',()=>{
        //code that won't run
    });
});

Mesmo que um fit() esteja em um describe() em vez de um fdescribe(), ele será executado. Além disso, qualquer it() dentro de um fdescribe() que não contenha um fit() será executado.

fdescribe('first group',()=>{
    it('this test will run',()=>{
        //code that will run
    });
    it('this test will also run',()=>{
        //code that will also run
    });
});
describe('second group',()=>{
    it('this code will not run',()=>{
        //code that won't run
    });
    fit('this code will run',(){
        //code that will run
    });
});

Testes pendentes #

O transferidor permite que os testes sejam definidos como pendentes. Isso significa que o transferidor não executará o teste, mas produzirá:

Pending:
1) Test Name
Temporarily disabled with xit

Ou, se desabilitado com xdescribe():

Pending:
1) Test Name
No reason given

Combinações #

  • Um xit() dentro de um xdescribe() produzirá a resposta xit().
  • Um xit() dentro de um fdescribe() ainda será tratado como pendente.
  • Um fit() dentro de um xdescribe() ainda será executado e nenhum teste pendente produzirá nada.

Transferidor: teste E2E para aplicativos angulares corporativos #

Instalação e configuração do transferidor

Etapa 1: Baixe e instale o NodeJS aqui. Certifique-se de ter a versão mais recente do node. Aqui, estou usando o nó v7.8.0. Você precisará ter o Java Development Kit (JDK) instalado para executar o selenium.

Passo 2: Abra seu terminal e digite o seguinte comando para instalar o transferidor globalmente.

npm install -g protractor

Isso instalará duas ferramentas, como o transferidor e o gerenciador de webdriver. Você pode verificar sua instalação do transferidor seguindo o comando:protractor –version. Se o Protractor for instalado com sucesso, o sistema exibirá a versão instalada (ou seja, a versão 5.1.1). Caso contrário, você terá que verificar novamente a instalação. Passo 3: Atualize o gerenciador do webdriver para baixar os binários necessários.

webdriver-manager update

Etapa 4: O comando a seguir iniciará um servidor Selenium. Esta etapa executará o gerenciador de driver da Web em segundo plano e ouvirá todos os testes executados via transferidor.

início do gerenciador de webdriver Você pode ver informações sobre o status do servidor em http://localhost:4444/wd/hub/static/resource/hub.html.

Escrevendo o primeiro caso de teste usando o transferidor:

Antes de começar a escrever o caso de teste, temos que preparar dois arquivos que são o arquivo de configuração e o arquivo de especificação.

No arquivo de configuração:

//In conf.js
exports.config = {
    baseUrl: ‘http://localhost:8800/adminapp’,
    seleniumAddress: ‘http://localhost:4444/wd/hub',
    specs: [‘product/product_test.js’],
    directConnect : true,
    capabilities :{
        browserName: ‘chrome’
    }
}

Compreensão básica das terminologias usadas no arquivo de configuração:

baseUrl – Um URL base para seu aplicativo em teste.

seleniumAddress – Para se conectar a um Selenium Server que já está em execução.

specs – Localização do seu arquivo de especificações

directConnect : true – Para se conectar diretamente aos drivers do navegador.

recursos – Se você estiver testando em um único navegador, use a opção de recursos. Se você estiver testando em vários navegadores, use a matriz multiCapabilities.

Você pode encontrar mais opções de configuração em aqui. Eles descreveram toda a terminologia possível com sua definição.

No arquivo Spec:

//In product_test.js

    describe(‘Angular Enterprise Boilerplate’, function() {
      it('should have a title', function() {
        browser.get('http://localhost:8800/adminapp’);
        expect(browser.getTitle()).toEqual(‘Angular Enterprise Boilerplate’);
      });
    });

Compreensão básica das terminologias usadas no arquivo de especificações:

Por padrão, o Protractor usa a estrutura jasmine para sua interface de teste. A sintaxe ‘describe’ e ‘it’ é do framework jasmine. Você pode aprender mais a partir daqui. Executando o primeiro caso de teste:

Antes de executar o caso de teste, certifique-se de que seu gerenciador de webdriver e seu aplicativo sejam executados em abas diferentes do seu terminal.

Agora, execute o teste com:

Protractor app/conf.js

Você deve ver o navegador Chrome abrir com o URL do seu aplicativo e se fechar. A saída do teste deve ser 1 teste, 1 afirmação, 0 falhas.

Bravo! Você executou com sucesso seu primeiro caso de teste.
