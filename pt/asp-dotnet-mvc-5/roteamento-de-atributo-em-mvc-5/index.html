Sintaxe #

  1. {productId:int}/{productTitle} Mapeado para ProductsController.Show(int id)

  2. {username} mapeado para ProfilesController.Show(string username)

  3. {username}/catalogs/{catalogId:int}/{catalogTitle} Mapeado para CatalogsController.Show(string username, int catalogId)

Roteamento é como o ASP.NET MVC combina um URI com uma ação. O MVC 5 suporta um novo tipo de roteamento, chamado roteamento de atributo. Como o nome indica, o roteamento de atributo usa atributos para definir rotas. O roteamento de atributo oferece mais controle sobre os URIs em seu aplicativo da web.

O estilo anterior de roteamento, chamado roteamento baseado em convenção, ainda é totalmente suportado. Na verdade, você pode combinar as duas técnicas no mesmo projeto.

Restrições de rota #

As restrições de rota permitem restringir como os parâmetros no modelo de rota são correspondidos. A sintaxe geral é {parameter:constraint}. Por exemplo:

// eg: /users/5
[Route(“users/{id:int}”]
public ActionResult GetUserById(int id) { … }
 
// eg: users/ken
[Route(“users/{name}”]
public ActionResult GetUserByName(string name) { … }

Aqui, a primeira rota só será selecionada se o segmento “id” do URI for um inteiro. Caso contrário, a segunda rota será escolhida.

Const Descrição (Corresponde:) Exemplo
alpha Caracteres maiúsculos ou minúsculos do alfabeto latino (a-z, A-Z) {x:alpha}
bool Valor booleano. {x:bool}
datetime Valor DateTime. {x:datetime}
decimal Valor decimal. {x:decimal}
double valor de ponto flutuante de 64 bits. {x:double}
float valor de ponto flutuante de 32 bits. {x:float}
guid valor GUID. {x:guid}
int valor inteiro de 32 bits. {x:int}
length String com o comprimento especificado ou dentro de um intervalo especificado de comprimentos. {x:length(6)} {x:length(1,20)}
long valor inteiro de 64 bits. {x:long}
max Inteiro com valor máximo. {x:max(10)}
maxlength String com comprimento máximo. {x:maxlength(10)}
min Inteiro com valor mínimo. {x:min(10)}
minlength String com um comprimento mínimo. {x:minlength(10)}
range Inteiro dentro de um intervalo de valores. {x:range(10,50)}
regex Expressão regular. {x:regex(^\d{3}-\d{3}-\d{4}$)}

Como implementar a rota de atributo #

Ativando o roteamento de atributo Para habilitar o roteamento de atributo, chame MapMvcAttributeRoutes durante a configuração.

public static void RegisterRoutes(RouteCollection routes)
{
routes.IgnoreRoute(“{resource}.axd/{*pathInfo}”);

routes.MapMvcAttributeRoutes();

routes.MapRoute(
    name: “Default”,
    url: “{controller}/{action}/{id}”,
    defaults: new { controller = “Home”, action = “Index”, id = UrlParameter.Optional }
);
}

Parâmetros de URI opcionais e valores padrão #

Você pode tornar um parâmetro URI opcional adicionando um ponto de interrogação ao parâmetro de rota. Você também pode especificar um valor padrão usando o formulário parâmetro=valor.

public class BooksController : Controller
{
    // eg: /books
    // eg: /books/1430210079
    [Route(“books/{isbn?}”)]
    public ActionResult View(string isbn)
    {
        if (!String.IsNullOrEmpty(isbn))
        {
            return View(“OneBook”, GetBook(isbn));
        }
        return View(“AllBooks”, GetBooks());
    }

// eg: /books/lang
// eg: /books/lang/en
// eg: /books/lang/he
[Route(“books/lang/{lang=en}”)]
public ActionResult ViewByLanguage(string lang)
{
    return View(“OneBook”, GetBooksByLanguage(lang));
}

Neste exemplo, tanto /books quanto /books/1430210079 serão roteados para a ação “Visualizar”, o primeiro resultará na listagem de todos os livros e o último listará o livro específico. Ambos /books/lang e /books/lang/en serão tratados da mesma forma.

Prefixos de rota #

Muitas vezes, todas as rotas em um controlador começam com o mesmo prefixo. Por exemplo:

public class ReviewsController : Controller
{
    // eg: /reviews
    [Route(“reviews”)]
    public ActionResult Index() { … }
    // eg: /reviews/5
    [Route(“reviews/{reviewId}”)]
    public ActionResult Show(int reviewId) { … }
    // eg: /reviews/5/edit
    [Route(“reviews/{reviewId}/edit”)]
    public ActionResult Edit(int reviewId) { … }
}

Você pode definir um prefixo comum para um controlador inteiro usando o atributo [RoutePrefix]:

[RoutePrefix(“reviews”)]
public class ReviewsController : Controller
{
    // eg.: /reviews
    [Route]
    public ActionResult Index() { … }
    // eg.: /reviews/5
    [Route(“{reviewId}”)]
    public ActionResult Show(int reviewId) { … }
    // eg.: /reviews/5/edit
    [Route(“{reviewId}/edit”)]
    public ActionResult Edit(int reviewId) { … }
}

Use um til (~) no atributo method para substituir o prefixo da rota, se necessário:

[RoutePrefix(“reviews”)]
public class ReviewsController : Controller
{
    // eg.: /spotlight-review
    [Route(“~/spotlight-review”)]
    public ActionResult ShowSpotlight() { … }
 
    …
}

Rota Padrão #

Você também pode aplicar o atributo [Route] no nível do controlador, capturando a ação como parâmetro. Essa rota seria aplicada em todas as ações no controlador, a menos que um [Route] específico tenha sido definido em uma ação específica, substituindo o padrão definido no controlador.

[RoutePrefix(“promotions”)]
[Route(“{action=index}”)]
public class ReviewsController : Controller
{
    // eg.: /promotions
    public ActionResult Index() { … }
 
    // eg.: /promotions/archive
    public ActionResult Archive() { … }
 
    // eg.: /promotions/new
    public ActionResult New() { … }
 
    // eg.: /promotions/edit/5
    [Route(“edit/{promoId:int}”)]
    public ActionResult Edit(int promoId) { … }
}
