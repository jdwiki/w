Instalação ou configuração #

Instalação #

CocoaPods #

CocoaPods é um gerenciador de dependências para projetos Cocoa. Você pode instalá-lo com o seguinte comando:

$ gem install cocoapods

O CocoaPods 1.1.0+ é necessário para construir o Alamofire 4.0.0+.

Para integrar o Alamofire em seu projeto Xcode usando CocoaPods, especifique-o em seu Podfile:

source 'https://github.com/CocoaPods/Specs.git'
platform :ios, '10.0'
use_frameworks!

target '<Your Target Name>' do
    pod 'Alamofire', '~> 4.0'
end

Em seguida, execute o seguinte comando:

$ pod install

Cartago #

Carthage é um gerenciador de dependências descentralizado que cria suas dependências e fornece estruturas binárias.

Você pode instalar o Carthage com o Homebrew usando o seguinte comando:

$ brew update
$ brew install carthage

Para integrar Alamofire em seu projeto Xcode usando Carthage, especifique-o em seu Cartfile:

github "Alamofire/Alamofire" ~> 4.0

Execute carthage update para construir o framework e arraste o Alamofire.framework construído para o seu projeto Xcode.

Manualmente #

Se você preferir não usar nenhum dos gerenciadores de dependência mencionados acima, você pode integrar o Alamofire em seu projeto manualmente.

Estrutura incorporada #

  • Abra o Terminal, cd em seu diretório de projeto de nível superior e execute o seguinte comando “se” seu projeto não for inicializado como um repositório git:
$ git init
  • Adicione Alamofire como um git submodule executando o seguinte comando:
$ git submodule add https://github.com/Alamofire/Alamofire.git

  • Abra a nova pasta Alamofire e arraste o Alamofire.xcodeproj para o Project Navigator do projeto Xcode da sua aplicação.

    It should appear nested underneath your application’s blue project icon. Whether it is above or below all the other Xcode groups does not matter.

  • Selecione o Alamofire.xcodeproj no Project Navigator e verifique se o destino de implantação corresponde ao destino de seu aplicativo.

  • Em seguida, selecione seu projeto de aplicativo no Project Navigator (ícone azul do projeto) para navegar até a janela de configuração de destino e selecione o destino do aplicativo sob o título “Destinos” na barra lateral.

  • Na barra de guias na parte superior dessa janela, abra o painel “Geral”.

  • Clique no botão + na seção “Binários incorporados”.

  • Você verá duas pastas Alamofire.xcodeproj diferentes, cada uma com duas versões diferentes do Alamofire.framework aninhadas dentro de uma pasta Products.

    It does not matter which Products folder you choose from, but it does matter whether you choose the top or bottom Alamofire.framework.

  • Selecione o topo Alamofire.framework para iOS e o inferior para OS X.

    You can verify which one you selected by inspecting the build log for your project. The build target for Alamofire will be listed as either Alamofire iOS, Alamofire macOS, Alamofire tvOS or Alamofire watchOS.

  • E é isso!

O Alamofire.framework é adicionado automaticamente como uma dependência de destino, estrutura vinculada e estrutura incorporada em uma fase de criação de arquivos de cópia, que é tudo o que você precisa para construir no simulador e em um dispositivo.
