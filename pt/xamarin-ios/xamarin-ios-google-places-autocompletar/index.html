  • Adicione um controle de UI de preenchimento automático com o controlador de resultados.

Desde que comecei a trabalhar com o Xamarin, houve muitas coisas que eu gostaria que alguém já tivesse documentado. Aqui eu explico como usar 1 aspecto do preenchimento automático de lugares do google, o controle da interface do usuário usando um controlador de resultados. Embora este código seja baseado nos próprios exemplos do Google convertidos para C# do Swift, tentei o meu melhor para garantir que seja um exemplo totalmente funcional. Espero muito que esta documentação ajude os outros.

Adicione um controle de UI de preenchimento automático com o controlador de resultados. #

O controle de IU de preenchimento automático é uma caixa de diálogo de pesquisa com funcionalidade de preenchimento automático integrada. À medida que um usuário insere termos de pesquisa, o controle apresenta uma lista de locais previstos para escolha. Quando o usuário faz uma seleção, uma instância GMSPlace (Place in Xamarin) é retornada, que seu aplicativo pode usar para obter detalhes sobre o local selecionado.

Como mencionado acima, este exemplo usa um controlador de resultados que permite mais controle sobre a interface do usuário de entrada de texto. O controlador de resultados alternará dinamicamente a visibilidade da lista de resultados com base no foco da interface do usuário de entrada.

Instruções:

  1. Primeiro precisamos adicionar a API do google maps ao nosso Visual Studio, ela está disponível através do Nuget, basta procurar por Xamarin.Google.iOS.Maps, adicioná-la ao seu projeto iOS, alternativamente você pode baixá-la do Xamarin Xamarin Google Maps SDK do iOS

  2. Precisamos de algo como um botão para acionar o controlador de visualização de preenchimento automático do Google. Neste exemplo, eu faço isso com um storyboard e adicionei um botão chamado GoogleButton, você pode acioná-lo com um código que realmente não importa.

  3. Em ViewDidLoad em sua classe de controladores de exibição, adicione o seguinte código. No meu exemplo abaixo, eu não estou usando a localização real dos dispositivos móveis, minha solução final obviamente faria isso, mas isso era um teste e eu não queria implementar código extra até provar que isso funcionava ou diluir o que estou tentando mostrar para você:

// Código para abrir o controlador de visualização completa automática do Google Places.

GoogleButton.TouchUpInside += (sender, ea) =>
{

    var FakeCoordinates = new CLLocationCoordinate2D()
    {
        Latitude = 54.135364,
        Longitude = -0.797888
    };

    var north = LocationWithBearing(45, 3000, FakeCoordinates);
    var east = LocationWithBearing(225, 3000, FakeCoordinates);

    var autocompleteController = new AutocompleteViewController();
    autocompleteController.Delegate = new AutoCompleteDelegate();
    autocompleteController.AutocompleteBounds = new CoordinateBounds(north, east);
    PresentViewController(autocompleteController, true, null);
                
};

  1. Isso é opcional, mas eu adicionei uma função para calcular um limite local, estou passando em 3000 esse número está em metros, então se você quiser um limite inicial maior fique à vontade para ajustar, observe que a pesquisa do google ainda encontrará qualquer endereço no mundo, ele apenas pondera os resultados iniciais para esses limites de áreas locais primeiro. Esta função foi emprestada de uma postagem de estouro de pilha, eu a converti de Swift para C# para nossos propósitos:

    public CLLocationCoordinate2D LocationWithBearing(Double bearing, Double distanceMeters, CLLocationCoordinate2D origin) { var distRadians = distanceMeters/(6372797.6);

     var rbearing = bearing*Math.PI/180.0;

 var lat1 = origin.Latitude*Math.PI/180;
 var lon1 = origin.Longitude*Math.PI/180;

 var lat2 = Math.Asin(Math.Sin(lat1)*Math.Cos(distRadians) + Math.Cos(lat1)*Math.Sin(distRadians)*Math.Cos(rbearing));
 var lon2 = lon1 + Math.Atan2(Math.Sin(rbearing)*Math.Sin(distRadians)*Math.Cos(lat1),
                 Math.Cos(distRadians) - Math.Sin(lat1)*Math.Sin(lat2));

 return new CLLocationCoordinate2D(latitude: lat2*180/ Math.PI, longitude: lon2*180/Math.PI);

    }

  2. Este trecho de código final é o delegado para o preenchimento automático, precisamos desse delegado para lidar com tudo o que o google retornará para nós:

    public class AutoCompleteDelegate : AutocompleteViewControllerDelegate {

     public override void DidFailAutocomplete(AutocompleteViewController viewController, NSError error)
 {
     // TODO: handle the error.
     Debug.Print("Error: " + error.Description);
 }


 public override void DidAutocomplete(AutocompleteViewController viewController, Place place)
 {
     Debug.Print(place.Name);
     Debug.Print(place.FormattedAddress);

     viewController.DismissViewController(true, null);
 }

 public override void DidRequestAutocompletePredictions(AutocompleteViewController viewController)
 {
     UIApplication.SharedApplication.NetworkActivityIndicatorVisible = true;
 }

 public override void DidUpdateAutocompletePredictions(AutocompleteViewController viewController)
 {
     UIApplication.SharedApplication.NetworkActivityIndicatorVisible = true;
 }

 public override void WasCancelled(AutocompleteViewController viewController)
 {
     viewController.DismissViewController(true, null);
 }

    }

Execute seu projeto e ele deve funcionar perfeitamente, este exemplo é bastante focado, mas espero que dê um exemplo básico de como qualquer um dos controles de interface do usuário de preenchimento automático do Google precisaria funcionar. Obrigado!
