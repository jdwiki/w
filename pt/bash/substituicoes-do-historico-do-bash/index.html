Referência rápida #

Interação com a história #

# List all previous commands
history

# Clear the history, useful if you entered a password by accident
history -c

Designadores de eventos #

# Expands to line n of bash history
!n

# Expands to last command
!!

# Expands to last command starting with "text"
!text

# Expands to last command containing "text"
!?text

# Expands to command n lines ago
!-n

# Expands to last command with first occurrence of "foo" replaced by "bar"
^foo^bar^

# Expands to the current command
!#

Designadores de palavras #

Eles são separados por : do designador de evento ao qual se referem. Os dois pontos podem ser omitidos se a palavra designada não começar com um número: !^ é o mesmo que !:^.

# Expands to the first argument of the most recent command
!^

# Expands to the last argument of the most recent command (short for !!:$)
!$

# Expands to the third argument of the most recent command
!:3

# Expands to arguments x through y (inclusive) of the last command
# x and y can be numbers or the anchor characters ^ $
!:x-y

# Expands to all words of the last command except the 0th
# Equivalent to :^-$
!*

Modificadores #

Eles modificam o evento anterior ou o designador de palavra.

# Replacement in the expansion using sed syntax
# Allows flags before the s and alternate separators
:s/foo/bar/ #substitutes bar for first occurrence of foo
:gs|foo|bar| #substitutes bar for all foo

# Remove leading path from last argument ("tail")
:t

# Remove trailing path from last argument ("head")
:h

# Remove file extension from last argument
:r

Se a variável do Bash HISTCONTROL contém ignorespace ou ignoreboth (ou, alternativamente, HISTIGNORE contém o padrão [ ]*), você pode evitar que seus comandos sejam armazenados no histórico do Bash acrescentando-os com um espaço:

# This command won't be saved in the history
 foo

# This command will be saved
bar

Repita o comando anterior com sudo #

$ apt-get install r-base
E: Could not open lock file /var/lib/dpkg/lock - open (13: Permission denied)
E: Unable to lock the administration directory (/var/lib/dpkg/), are you root?
$ sudo !!
sudo apt-get install r-base
[sudo] password for <user>:

Pesquisa no histórico de comandos por padrão #

Pressione controlr e digite um padrão.

Por exemplo, se você executou recentemente man 5 crontab, você pode encontrá-lo rapidamente começando a digitar “crontab”. O prompt mudará assim:

(reverse-i-search)`cr': man 5 crontab

O `cr' é a string que digitei até agora. Esta é uma pesquisa incremental, portanto, à medida que você continua digitando, o resultado da pesquisa é atualizado para corresponder ao comando mais recente que continha o padrão.

Pressione as teclas de seta para a esquerda ou para a direita para editar o comando correspondente antes de executá-lo, ou a tecla enter para executar o comando.

Por padrão, a pesquisa encontra o comando executado mais recentemente que corresponde ao padrão. Para ir mais longe na história, pressione controlr novamente. Você pode pressioná-lo repetidamente até encontrar o comando desejado.

Mude para o diretório recém-criado com !#:N #

$ mkdir backup_download_directory && cd !#:1
mkdir backup_download_directory && cd backup_download_directory

Isso substituirá o enésimo argumento do comando atual. No exemplo, !#:1 é substituído pelo primeiro argumento, ou seja, backup_download_directory.

Usando !$ #

Você pode usar o !$ para reduzir a repetição ao usar a linha de comando:

$ echo ping
ping
$ echo !$
ping

Você também pode construir sobre a repetição

$ echo !$ pong
ping pong
$ echo !$, a great game
pong, a great game

Observe que no último exemplo não obtivemos ping pong, a great game porque o último argumento passado para o comando anterior foi pong, podemos evitar problemas como este adicionando aspas. Continuando com o exemplo, nosso último argumento foi game:

$ echo "it is !$ time"
it is game time
$ echo "hooray, !$!"
hooray, it is game time!

Repita o comando anterior com uma substituição #

$ mplayer Lecture_video_part1.mkv
$ ^1^2^
mplayer Lecture_video_part2.mkv

Este comando substituirá 1 por 2 no comando executado anteriormente. Ele substituirá apenas a primeira ocorrência da string e é equivalente a !!:s/1/2/.

Se você quiser substituir todas as ocorrências, você deve usar !!:gs/1/2/ ou !!:as/1/2/.
