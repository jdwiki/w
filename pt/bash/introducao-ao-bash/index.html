Olá Mundo #

Shell interativo #

O shell Bash é comumente usado interativamente: Ele permite que você insira e edite comandos, então os executa quando você pressiona a tecla Return. Muitos sistemas operacionais baseados em Unix e semelhantes a Unix usam o Bash como seu shell padrão (principalmente Linux e macOS). O terminal entra automaticamente em um processo de shell Bash interativo na inicialização.

Emita Hello World digitando o seguinte:

echo "Hello World"
#> Hello World  # Output Example

Notas #

  • Você pode alterar o shell apenas digitando o nome do shell no terminal. Por exemplo: sh, bash, etc.

  • echo é um comando interno do Bash que grava os argumentos que recebe na saída padrão. Ele anexa uma nova linha à saída, por padrão.

Shell não interativo #

O shell Bash também pode ser executado não interativamente a partir de um script, fazendo com que o shell não exija interação humana. O comportamento interativo e o comportamento de script devem ser idênticos – uma consideração importante de design do shell Unix V7 Bourne e do Bash transitivamente. Portanto, qualquer coisa que possa ser feita na linha de comando pode ser colocada em um arquivo de script para reutilização.

Siga estas etapas para criar um script Hello World:

  1. Crie um novo arquivo chamado hello-world.sh

    touch hello-world.sh

  2. Torne o script executável executando chmod +x hello-world.sh

  3. Adicione este código:

    #!/bin/bash
echo "Hello World"

Linha 1: a primeira linha do script deve começar com a sequência de caracteres #!, conhecida como shebang1. O shebang instrui o sistema operacional a executar /bin/bash, o shell Bash, passando o caminho do script como um argumento.

E.g. `/bin/bash hello-world.sh`

Linha 2: Usa o comando echo para escrever Hello World na saída padrão.

  1. Execute o script hello-world.sh na linha de comando usando um dos seguintes:
  • ./hello-world.sh – mais utilizado e recomendado
  • /bin/bash hello-world.sh
  • bash hello-world.sh – assumindo que /bin está em seu $PATH
  • sh hello-world.sh

Para uso em produção real, você omitiria a extensão .sh (o que é enganoso de qualquer maneira, já que este é um script Bash, não um script sh) e talvez mova o arquivo para um diretório dentro de seu PATH para que esteja disponível para você independentemente do seu diretório de trabalho atual, assim como um comando do sistema como cat ou ls.

Erros comuns incluem:

  1. Esquecer de aplicar a permissão de execução no arquivo, ou seja, chmod +x hello-world.sh, resultando na saída de ./hello-world.sh: Permissão negada.

  2. Editando o script no Windows, que produz caracteres de final de linha incorretos que o Bash não pode manipular.

    A common symptom is : command not found where the carriage return has forced the cursor to the beginning of line, overwriting the text before the colon in the error message.

    The script can be fixed using the dos2unix program.

    An example use: dos2unix hello-world.sh

    dos2unix edits the file inline.

  3. Usando sh ./hello-world.sh, sem perceber que bash e sh são shells distintos com recursos distintos (embora como o Bash seja compatível com versões anteriores, o erro oposto é inofensivo).

    Anyway, simply relying on the script’s shebang line is vastly preferable to explicitly writing bash or sh (or python or perl or awk or ruby or…) before each script’s file name.

    A common shebang line to use in order to make your script more portable is to use #!/usr/bin/env bash instead of hard-coding a path to Bash. That way, /usr/bin/env has to exist, but beyond that point, bash just needs to be on your PATH. On many systems, /bin/bash doesn’t exist, and you should use /usr/local/bin/bash or some other absolute path; this change avoids having to figure out the details of that.

1 Também conhecido como sha-bang, hashbang, pound-bang, hash-pling.

Olá Mundo Usando Variáveis #

Crie um novo arquivo chamado hello.sh com o seguinte conteúdo e dê a ele permissões executáveis ​​com chmod +x hello.sh.

Executar/Executar via: ./hello.sh

#!/usr/bin/env bash

# Note that spaces cannot be used around the `=` assignment operator
whom_variable="World"

# Use printf to safely output the data
printf "Hello, %s\n" "$whom_variable"
#> Hello, World

Isso imprimirá Hello, World na saída padrão quando executado.

Para dizer ao bash onde o script está você precisa ser muito específico, apontando-o para o diretório que o contém, normalmente com ./ se for seu diretório de trabalho, onde . é um alias para o diretório atual. Se você não especificar o diretório, o bash tenta localizar o script em um dos diretórios contidos na variável de ambiente $PATH.

O código a seguir aceita um argumento $1, que é o primeiro argumento da linha de comando, e o gera em uma string formatada, seguindo Hello, .

Executar/Executar via: ./hello.sh World

#!/usr/bin/env bash
printf "Hello, %s\n" "$1"
#> Hello, World

É importante notar que $1 deve ser citado entre aspas duplas, não aspas simples. "$1" expande para o primeiro argumento da linha de comando, conforme desejado, enquanto '$1' é avaliado como string literal $1.

Nota de segurança: Leia ** Implicações de segurança de esquecer de citar uma variável em shells bash** para entender a importância de colocar o texto da variável entre aspas duplas.

Hello World com entrada do usuário #

O seguinte solicitará a entrada de um usuário e, em seguida, armazenará essa entrada como uma string (texto) em uma variável. A variável é então usada para dar uma mensagem ao usuário.

#!/usr/bin/env bash
echo  "Who are you?"
read name
echo "Hello, $name."

O comando read aqui lê uma linha de dados da entrada padrão na variável name. Isso é então referenciado usando $name e impresso no padrão usando echo.

Saída de exemplo:

$ ./hello_world.sh
Who are you?
Matt
Hello, Matt.

Aqui o usuário digitou o nome “Matt”, e este código foi usado para dizer Hello, Matt..

E se você quiser anexar algo ao valor da variável ao imprimi-lo, use colchetes ao redor do nome da variável, conforme mostrado no exemplo a seguir:

#!/usr/bin/env bash
echo  "What are you doing?"
read action
echo "You are ${action}ing."

Saída de exemplo:

$ ./hello_world.sh
What are you doing?
Sleep
You are Sleeping.

Aqui, quando o usuário insere uma ação, “ing” é anexado a essa ação durante a impressão.

Importância da citação em Strings #

A citação é importante para a expansão de strings no bash. Com eles, você pode controlar como o bash analisa e expande suas strings.

Existem dois tipos de cotação: #

  • Fraco: usa aspas duplas: “
  • Forte: usa aspas simples: ‘

Se você quiser bash para expandir seu argumento, você pode usar Weak Quoting:

#!/usr/bin/env bash
world="World"
echo "Hello $world"
#> Hello World

Se você não quiser usar o bash para expandir seu argumento, você pode usar Strong Quoting:

#!/usr/bin/env bash
world="World"
echo 'Hello $world'
#> Hello $world

Você também pode usar escape para evitar a expansão:

#!/usr/bin/env bash
world="World"
echo "Hello \$world"
#> Hello $world

Para obter informações mais detalhadas além dos detalhes para iniciantes, você pode continuar lendo aqui.

Visualizando informações para Bash built-ins #

help <command>

Isso exibirá a página de ajuda do Bash (manual) para o built-in especificado.

Por exemplo, help unset mostrará:

desarmar: desarmar [-f] [-v] [-n] [nome …] Desativar valores e atributos de variáveis ​​e funções do shell.

Para cada NOME, remova a variável ou função correspondente.

Opções: -f trata cada NOME como uma função shell -v trata cada NOME como uma variável de shell -n trata cada NOME como uma referência de nome e desmarca a própria variável em vez da variável referenciada

Sem opções, unset primeiro tenta desarmar uma variável e, se isso falhar, tenta desarmar uma função.

Algumas variáveis ​​não podem ser desarmadas; veja também `readonly’.

Situação de Saída: Retorna com sucesso, a menos que uma opção inválida seja fornecida ou um NOME seja somente leitura.

Para ver uma lista de todos os built-ins com uma breve descrição, use

help -d

Hello World no modo “Debug” #

$ cat hello.sh 
#!/bin/bash 
echo "Hello World"
$ bash -x hello.sh 
+ echo Hello World
Hello World

O argumento -x permite que você percorra cada linha do script. Um bom exemplo está aqui:

$ cat hello.sh
#!/bin/bash 
echo "Hello World\n" 
adding_string_to_number="s"
v=$(expr 5 + $adding_string_to_number) 

$ ./hello.sh 
Hello World

expr: non-integer argument

O erro solicitado acima não é suficiente para rastrear o script; no entanto, usar a maneira a seguir fornece uma noção melhor de onde procurar o erro no script.

$ bash -x hello.sh 
+ echo Hello World\n
Hello World

+ adding_string_to_number=s
+ expr 5 + s
expr: non-integer argument
+ v=

Como lidar com argumentos nomeados #

#!/bin/bash

deploy=false
uglify=false

while (( $# > 1 )); do case $1 in
   --deploy) deploy="$2";;
   --uglify) uglify="$2";;
   *) break;
 esac; shift 2
done

$deploy && echo "will deploy... deploy = $deploy"
$uglify && echo "will uglify... uglify = $uglify"

# how to run
# chmod +x script.sh
# ./script.sh --deploy true --uglify false
