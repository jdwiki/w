Sintaxe #

  • [ “$1” = “$2” ] #Um colchete “[” é na verdade um comando. Por isso requer um espaço antes e depois.
  • test “$1” = “$2” #Test é sinônimo do comando “[”

Parâmetros #

Parâmetro para [ ou teste Detalhes
Operadores de arquivo Detalhes
-e "$arquivo" Retorna verdadeiro se o arquivo existir.
-d "$arquivo" Retorna verdadeiro se o arquivo existir e for um diretório
-f "$arquivo" Retorna verdadeiro se o arquivo existir e for um arquivo normal
-h "$arquivo" Retorna verdadeiro se o arquivo existir e for um link simbólico
Comparadores de strings Detalhes
-z "$str" Verdadeiro se o comprimento da string for zero
-n "$str Verdadeiro se o comprimento da string for diferente de zero
"$str" = "$str2" Verdadeiro se a string $str for igual à string $str2. Não é melhor para números inteiros. Pode funcionar, mas será inconsistente
"$str" != "$str2" Verdadeiro se as strings não forem iguais
Inteiro Comparadores       Detalhes
"$int1" -eq "$int2" Verdadeiro se os inteiros forem iguais
"$int1" -ne "$int2" Verdadeiro se os inteiros não forem iguais
"$int1" -gt "$int2" Verdadeiro se int1 for maior que int 2
"$int1" -ge "$int2" Verdadeiro se int1 for maior ou igual a int2
"$int1" -lt "$int2" Verdadeiro se int1 for menor que int 2
"$int1" -le "$int2" Verdadeiro se int1 for menor ou igual a int2

Existem muitos parâmetros comparadores disponíveis no bash. Nem todos ainda estão listados aqui.

Se declaração #

if [[ $1 -eq 1 ]]; then
    echo "1 was passed in the first parameter"
elif [[ $1 -gt 2 ]]; then
    echo "2 was not passed in the first parameter"
else
    echo "The first parameter was not 1 and is not more than 2."
fi

O fechamento fi é necessário, mas as cláusulas elif e/ou else podem ser omitidas.

Os pontos e vírgulas antes de then são sintaxe padrão para combinar dois comandos em uma única linha; eles podem ser omitidos somente se then for movido para a próxima linha.

É importante entender que os colchetes [[ não fazem parte da sintaxe, mas são tratados como um comando; é o código de saída deste comando que está sendo testado. Portanto, você deve sempre incluir espaços ao redor dos colchetes.

Isso também significa que o resultado de qualquer comando pode ser testado. Se o código de saída do comando for zero, a instrução será considerada verdadeira.

if grep "foo" bar.txt; then
    echo "foo was found"
else
    echo "foo was not found"
fi

Expressões matemáticas, quando colocadas entre parênteses duplos, também retornam 0 ou 1 da mesma forma, e também podem ser testadas:

if (( $1 + 5 > 91 )); then
    echo "$1 is greater than 86"
fi

Você também pode encontrar instruções if com colchetes simples. Eles são definidos no padrão POSIX e são garantidos para funcionar em todos os shells compatíveis com POSIX, incluindo Bash. A sintaxe é muito semelhante à do Bash:

if [ "$1" -eq 1 ]; then
    echo "1 was passed in the first parameter"
elif [ "$1" -gt 2 ]; then
    echo "2 was not passed in the first parameter"
else
    echo "The first parameter was not 1 and is not more than 2."
fi

Fazendo um loop em um array #

laço for:

arr=(a b c d e f)
for i in "${arr[@]}";do
    echo "$i"
done

Ou

for ((i=0;i<${#arr[@]};i++));do
    echo "${arr[$i]}" 
done

laço enquanto:

i=0
while [ $i -lt ${#arr[@]} ];do
    echo "${arr[$i]}"
    i=$(expr $i + 1)
done

Ou

i=0
while (( $i < ${#arr[@]} ));do
    echo "${arr[$i]}"
    ((i++))
done

Execução condicional de listas de comandos #

Como usar a execução condicional de listas de comandos #

Qualquer comando, expressão ou função incorporada, bem como qualquer comando ou script externo, pode ser executado condicionalmente usando os operadores &&(and) e ||(or).

Por exemplo, isso só imprimirá o diretório atual se o comando cd for bem-sucedido.

cd my_directory && pwd

Da mesma forma, isso será encerrado se o comando cd falhar, evitando uma catástrofe:

cd my_directory || exit
rm -rf *

Ao combinar várias instruções dessa maneira, é importante lembrar que (ao contrário de muitas linguagens de estilo C) [esses operadores não têm precedência e são associativos à esquerda](https://www.gnu.org/software/bash/manual/ html_node/Lists.html#Lists).

Assim, esta declaração funcionará como esperado…

cd my_directory && pwd || echo "No such directory"
  • Se o cd for bem sucedido, o && pwd será executado e o nome do diretório de trabalho atual será impresso. A menos que pwd falhe (uma raridade) o || echo ... não será executado.
  • Se o cd falhar, o && pwd será ignorado e o || echo ... será executado.

Mas isso não vai (se você está pensando “se…então…senão”)…

cd my_directory && ls || echo "No such directory"
  • Se o cd falhar, o && ls é ignorado e o || echo ... é executado.
  • Se o cd for bem sucedido, o && ls será executado.
  • Se o ls for bem sucedido, o || echo ... é ignorado. (até agora tudo bem)
  • MAS… se o ls falhar, o || echo ... também será executado.

    It is the ls, not the cd, that is the previous command.

Por que usar execução condicional de listas de comandos #

A execução condicional é um fio de cabelo mais rápida do que if...then, mas sua principal vantagem é permitir que funções e scripts saiam mais cedo, ou “curto-circuito”.

Ao contrário de muitas linguagens como C onde a memória é explicitamente alocada para structs e variáveis ​​e tal (e, portanto, deve ser desalocada), o bash lida com isso nos bastidores. Na maioria dos casos, não precisamos limpar nada antes de sair da função. Uma instrução return irá desalocar tudo o que for local para a função e a execução da coleta no endereço de retorno na pilha.

Retornar de funções ou sair de scripts o mais rápido possível pode melhorar significativamente o desempenho e reduzir a carga do sistema, evitando a execução desnecessária de código. Por exemplo…

my_function () {

    ### ALWAYS CHECK THE RETURN CODE

    # one argument required. "" evaluates to false(1)
    [[ "$1" ]]             || return 1

    # work with the argument. exit on failure
    do_something_with "$1" || return 1
    do_something_else      || return 1

    # Success! no failures detected, or we wouldn't be here
    return 0
}

Enquanto Loop #

#! /bin/bash

i=0

while [ $i -lt 5 ] #While i is less than 5
do
    echo "i is currently $i"
    i=$[$i+1] #Not the lack of spaces around the brackets. This makes it a not a test expression
done #ends the loop

Observe que há espaços ao redor dos colchetes durante o teste (após a instrução while). Esses espaços são necessários.

Este loop produz:

i is currently 0
i is currently 1
i is currently 2
i is currently 3
i is currently 4

For Loop com sintaxe estilo C #

O formato básico do loop for no estilo C é:

for (( variable assignment; condition; iteration process ))

Notas:

  • A atribuição da variável dentro do loop for no estilo C pode conter espaços ao contrário da atribuição usual
  • Variáveis ​​dentro do loop for no estilo C não são precedidas por $.

Exemplo:

for (( i = 0; i < 10; i++ ))
do
    echo "The iteration number is $i"
done

Também podemos processar várias variáveis ​​dentro do loop for no estilo C:

for (( i = 0, j = 0; i < 10; i++, j = i * i ))
do
    echo "The square of $i is equal to $j"
done

Até Loop #

Até que o loop seja executado até que a condição seja verdadeira

i=5
until [[ i -eq 10 ]]; do #Checks if i=10
    echo "i=$i" #Print the value of i
    i=$((i+1)) #Increment i by 1
done

Resultado:

i=5
i=6
i=7
i=8
i=9

Quando i atinge 10 a condição em até que o laço se torne verdadeiro e o laço termina.

continuar e quebrar #

Exemplo para continuar

for i in [series]
do
    command 1
    command 2
    if (condition) # Condition to jump over command 3
            continue # skip to the next value in "series"
    fi
    command 3
done

Exemplo para pausa

for i in [series]
do
    command 4
    if (condition) # Condition to break the loop
    then
            command 5 # Command if the loop needs to be broken
            break
    fi
    command 6 # Command to run if the "condition" is never true 
done

Quebra de loop #

Quebre vários loops:

arr=(a b c d e f)
for i in "${arr[@]}";do
    echo "$i"
    for j in "${arr[@]}";do
        echo "$j"
        break 2
    done
done

Resultado:

a
a

Quebrar loop único:

arr=(a b c d e f)
for i in "${arr[@]}";do
    echo "$i"
    for j in "${arr[@]}";do
        echo "$j"
        break
    done
done

Resultado:

a
a
b
a
c
a
d
a
e
a
f
a

Instrução de troca com maiúsculas e minúsculas #

Com a instrução case você pode combinar valores com uma variável.

O argumento passado para case é expandido e tenta corresponder a cada padrão.

Se uma correspondência for encontrada, os comandos até ;; serão executados.

case "$BASH_VERSION" in
 [34]*)
    echo {1..4}
    ;;  
  *)
    seq -s" " 1 4
esac

Padrão não são expressões regulares, mas correspondência de padrões de shell (também conhecidos como globs).

For Loop sem um parâmetro de lista de palavras #

for arg; do
    echo arg=$arg
done

Um loop for sem um parâmetro de lista de palavras irá iterar sobre os parâmetros posicionais. Em outras palavras, o exemplo acima é equivalente a este código:

for arg in "$@"; do
    echo arg=$arg
done

Em outras palavras, se você se pegar escrevendo for i em "$@"; Faz ...; done, apenas remova a parte in e escreva simplesmente `for i; Faz …; feito'.

Para Loop #

#! /bin/bash

for i in 1 "test" 3; do #Each space separated statement is assigned to i
    echo $i
done

Outros comandos podem gerar instruções para fazer um loop. Consulte o exemplo “Usando o loop For para iterar sobre números”.

Isso produz:

1
test
3
