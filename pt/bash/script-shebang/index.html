Sintaxe #

  • Use /bin/bash como o interpretador bash:

    #!/bin/bash

  • Pesquise o interpretador bash na variável de ambiente PATH com envexecutável:

    #!/usr/bin/env bash

Um erro comum é tentar executar arquivos de script \r\n formatados no final do Windows em sistemas UNIX/Linux, neste caso o interpretador de script usado no shebang é:

/bin/bash\r

E obviamente não é encontrado, mas pode ser difícil de descobrir.

Env shebang #

Para executar um arquivo de script com o executável bash encontrado na variável de ambiente PATH usando o executável env, a primeira linha de um arquivo de script deve indicar o caminho absoluto para o executável env com o argumento bash:

#!/usr/bin/env bash

O caminho env no shebang é resolvido e usado somente se um script for executado diretamente assim:

script.sh

O script deve ter permissão de execução.

O shebang é ignorado quando um interpretador bash é explicitamente indicado para executar um script:

bash script.sh

Shebang direto #

Para executar um arquivo de script com o interpretador bash, a primeira linha de um arquivo de script deve indicar o caminho absoluto para o executável bash a ser usado:

#!/bin/bash

O caminho bash no shebang é resolvido e usado somente se um script for executado diretamente assim:

./script.sh

O script deve ter permissão de execução.

O shebang é ignorado quando um interpretador bash é explicitamente indicado para executar um script:

bash script.sh

Outras travessuras #

Existem dois tipos de programas que o kernel conhece. Um programa binário é identificado por seu cabeçalho ELF (ExtenableLoadableFormat), que geralmente é produzido por um compilador. O segundo são scripts de qualquer tipo.

Se um arquivo começa na primeira linha com a seqüência #! então a próxima string tem que ser um nome de caminho de um interpretador. Se o kernel lê esta linha, ele chama o interpretador nomeado por este nome de caminho e fornece todas as palavras a seguir nesta linha como argumentos para o interpretador. Se não houver nenhum arquivo chamado “algo” ou “errado”:

#!/bin/bash something wrong
echo "This line never gets printed"

bash tenta executar seu argumento “algo errado” que não existe. O nome do arquivo de script também é adicionado. Para ver isso claramente, use um echo shebang:

#"/bin/echo something wrong 
# and now call this script named "thisscript" like so:
# thisscript one two
# the output will be:
something wrong ./thisscript one two

Alguns programas como awk usam essa técnica para executar scripts mais longos que residem em um arquivo de disco.
