Manipulação de trabalho #

Criando trabalhos #

Para criar um trabalho, basta anexar um único & após o comando:

$ sleep 10 &
[1] 20024

Você também pode tornar um processo em execução um trabalho pressionando Ctrl + z:

$ sleep 10
^Z
[1]+  Stopped                 sleep 10

Background e foreground de um processo #

Para trazer o processo para o primeiro plano, o comando fg é usado junto com %

$ sleep 10 &
[1] 20024

$ fg %1
sleep 10

Agora você pode interagir com o processo. Para trazê-lo de volta ao plano de fundo, você pode usar o comando bg. Devido à sessão de terminal ocupada, você precisa parar o processo primeiro pressionando Ctrl + z.

$ sleep 10
^Z
[1]+  Stopped              sleep 10

$ bg %1
[1]+ sleep 10 &

Devido à preguiça de alguns programadores, todos esses comandos também funcionam com um único % se houver apenas um processo, ou para o primeiro processo da lista. Por exemplo:

$ sleep 10 &
[1] 20024

$ fg %        # to bring a process to foreground 'fg %' is also working.
sleep 10

ou apenas

$ %           # laziness knows no boundaries, '%' is also working.
sleep 10

Além disso, apenas digitar fg ou bg sem nenhum argumento lida com o último trabalho:

$ sleep 20 &
$ sleep 10 &
$ fg
sleep 10
^C
$ fg
sleep 20

Matando jobs em execução #

$ sleep 10 &
[1] 20024

$ kill %1
[1]+  Terminated              sleep 10

O processo de suspensão é executado em segundo plano com o ID do processo (pid) 20024 e o número do trabalho 1. Para fazer referência ao processo, você pode usar o pid ou o número do trabalho. Se você usar o número do trabalho, deverá prefixá-lo com %. O sinal de kill padrão enviado por kill é SIGTERM, que permite que o processo alvo saia normalmente.

Alguns sinais de morte comuns são mostrados abaixo. Para ver uma lista completa, execute kill -l.

Nome do sinal Valor do sinal Efeito
``AGUARDE'' 1 Desligar
SIGINT 2 Interromper do teclado
SIGKILL 9 Sinal de matar
‘Prazo-alvo’ 15 Sinal de terminação

Iniciar e matar processos específicos #

Provavelmente a maneira mais fácil de matar um processo em execução é selecionando-o através do nome do processo como no exemplo a seguir usando o comando pkill como

pkill -f test.py

(ou) uma maneira mais infalível usando pgrep para procurar o ID do processo real

kill $(pgrep -f 'python test.py')

O mesmo resultado pode ser obtido usando grep sobre ps -ef | grep name_of_process, em seguida, matando o processo associado ao pid resultante (id do processo). Selecionar um processo usando seu nome é conveniente em um ambiente de teste, mas pode ser muito perigoso quando o script é usado em produção: é praticamente impossível ter certeza de que o nome corresponderá ao processo que você realmente deseja eliminar. Nesses casos, a abordagem a seguir é realmente muito segura.

Inicie o script que eventualmente será morto com a seguinte abordagem. Vamos supor que o comando que você deseja executar e eventualmente matar seja python test.py.

#!/bin/bash

if [[ ! -e /tmp/test.py.pid ]]; then   # Check if the file already exists
    python test.py &                   #+and if so do not run another process.
    echo $! > /tmp/test.py.pid
else
    echo -n "ERROR: The process is already running with pid "
    cat /tmp/test.py.pid
    echo
fi

Isso criará um arquivo no diretório /tmp contendo o pid do processo python test.py. Se o arquivo já existir, assumimos que o comando já está em execução e o script retorna um erro.

Então, quando você quiser matá-lo, use o seguinte script:

#!/bin/bash

if [[ -e /tmp/test.py.pid ]]; then   # If the file do not exists, then the
    kill `cat /tmp/test.py.pid`      #+the process is not running. Useless
    rm /tmp/test.py.pid              #+trying to kill it.
else
    echo "test.py is not running"
fi

que matará exatamente o processo associado ao seu comando, sem depender de nenhuma informação volátil (como a string usada para executar o comando). Mesmo neste caso, se o arquivo não existir, o script assume que você deseja eliminar um processo que não está em execução.

Este último exemplo pode ser facilmente aprimorado para executar o mesmo comando várias vezes (anexando ao arquivo pid em vez de sobrescrevê-lo, por exemplo) e para gerenciar casos em que o processo é interrompido antes de ser encerrado.

Verifique qual processo está sendo executado em uma porta específica #

Para verificar qual processo está sendo executado na porta 8080

lsof -i :8080

Recusando trabalho em segundo plano #

$ gzip extremelylargefile.txt &
$ bg
$ disown %1

Isso permite que um processo de longa execução continue assim que seu shell (terminal, ssh, etc) for fechado.

Listar trabalhos atuais #

$ tail -f /var/log/syslog > log.txt
[1]+  Stopped                 tail -f /var/log/syslog > log.txt

$ sleep 10 &

$ jobs
[1]+  Stopped                 tail -f /var/log/syslog > log.txt
[2]-  Running                 sleep 10 &

Lista todos os processos #

Existem duas maneiras comuns de listar todos os processos em um sistema. Ambos listam todos os processos executados por todos os usuários, embora difiram no formato de saída (o motivo das diferenças é histórico).

ps -ef   # lists all processes 
ps aux   # lists all processes in alternative format (BSD)

Isso pode ser usado para verificar se um determinado aplicativo está em execução. Por exemplo, para verificar se o servidor SSH (sshd) está em execução:

ps -ef | grep sshd

Encontrando informações sobre um processo em execução #

ps aux | grep <search-term> mostra os processos correspondentes a search-term

Exemplo:

root@server7:~# ps aux | grep nginx
root       315  0.0  0.3 144392  1020 ?        Ss   May28   0:00 nginx: master process /usr/sbin/nginx
www-data  5647  0.0  1.1 145124  3048 ?        S    Jul18   2:53 nginx: worker process
www-data  5648  0.0  0.1 144392   376 ?        S    Jul18   0:00 nginx: cache manager process
root     13134  0.0  0.3   4960   920 pts/0    S+   14:33   0:00 grep --color=auto nginx
root@server7:~#

Aqui, a segunda coluna é o id do processo. Por exemplo, se você quiser matar o processo nginx, você pode usar o comando kill 5647. É sempre aconselhável usar o comando kill com SIGTERM em vez de SIGKILL.
