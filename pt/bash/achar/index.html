find é um comando para pesquisar recursivamente um diretório por arquivos (ou diretórios) que correspondam a um critério e, em seguida, execute alguma ação nos arquivos selecionados.

encontre a ação search_path selection_criteria

Sintaxe #

  • encontre [-H] [-L] [-P] [-D depura] [-Olevel] [caminho…] [expressão]

Procurando um arquivo por nome ou extensão #

Para encontrar arquivos/diretórios com um nome específico, relativo a pwd:

$ find . -name "myFile.txt"
./myFile.txt

Para encontrar arquivos/diretórios com uma extensão específica, use um curinga:

$ find . -name "*.txt"
./myFile.txt
./myFile2.txt

Para encontrar arquivos/diretórios que correspondam a uma das muitas extensões, use o sinalizador or:

$ find . -name "*.txt" -o -name "*.sh"

Para encontrar arquivos/diretórios cujos nomes começam com abc e terminam com um caractere alfa após um dígito:

$ find . -name "abc[a-z][0-9]"

Para encontrar todos os arquivos/diretórios localizados em um diretório específico

$ find /opt

Para pesquisar apenas arquivos (não diretórios), use -type f:

find /opt -type f

Para procurar apenas diretórios (não arquivos normais), use -type d:

find /opt -type d

Executando comandos em um arquivo encontrado #

Às vezes, precisaremos executar comandos em muitos arquivos. Isso pode ser feito usando xargs.

find . -type d -print | xargs -r chmod 770

O comando acima irá encontrar recursivamente todos os diretórios (-type d) relativos a . (que é seu diretório de trabalho atual), e executará chmod 770 neles. A opção -r especifica ao xargs para não executar o chmod se o find não encontrar nenhum arquivo.

Se os nomes dos seus arquivos ou diretórios tiverem um caractere de espaço neles, este comando pode engasgar; uma solução é usar o seguinte

find . -type d -print0 | xargs -r -0 chmod 770

No exemplo acima, os sinalizadores -print0 e -0 especificam que os nomes dos arquivos serão separados usando um byte null e permitem o uso de caracteres especiais, como espaços, nos nomes dos arquivos. Esta é uma extensão GNU e pode não funcionar em outras versões de find e xargs.

A maneira preferida de fazer isso é pular o comando xargs e deixar find chamar o próprio subprocesso:

find . -type d -exec chmod 770 {} \;

Aqui, o {} é um espaço reservado que indica que você deseja usar o nome do arquivo nesse ponto. find executará chmod em cada arquivo individualmente.

Você pode, alternativamente, passar todos os nomes de arquivos para uma chamada single de chmod, usando

find . -type d -exec chmod 770 {} +

Este também é o comportamento dos snippets xargs acima. (Para chamar cada arquivo individualmente, você pode usar xargs -n1).

Uma terceira opção é deixar o bash fazer um loop sobre a lista de saídas de nomes de arquivos find:

find . -type d | while read -r d; do chmod 770 "$d"; done

Este é sintaticamente o mais desajeitado, mas conveniente quando você deseja executar vários comandos em cada arquivo encontrado. No entanto, isso é inseguro em face de nomes de arquivos com nomes estranhos.

find . -type f | while read -r d; do mv "$d" "${d// /_}"; done

que substituirá todos os espaços nos nomes dos arquivos por sublinhados.(Este exemplo também não funcionará se houver espaços nos nomes dos diretórios iniciais.)

O problema com o acima é que while read -r espera uma entrada por linha, mas os nomes dos arquivos podem conter novas linhas (e também, read -r perderá qualquer espaço em branco à direita). Você pode corrigir isso invertendo as coisas:

find . -type d -exec bash -c 'for f; do mv "$f" "${f// /_}"; done' _ {} +

Desta forma, o -exec recebe os nomes dos arquivos em uma forma completamente correta e portável; o bash -c os recebe como um número de argumentos, que serão encontrados em $@, corretamente citado etc. ( O script precisará lidar com esses nomes corretamente, é claro; toda variável que contém um nome de arquivo precisa estar entre aspas duplas.)

O misterioso _ é necessário porque o primeiro argumento para bash -c 'script' é usado para preencher $0.

Encontrando arquivo por hora de acesso/modificação #

Em um sistema de arquivos ext, cada arquivo tem um acesso armazenado, Modificação e (Status) Tempo de mudança associado a ele - para visualizar esta informação você pode usar stat myFile.txt; usando sinalizadores dentro de find, podemos procurar arquivos que foram modificados dentro de um determinado intervalo de tempo.

Para localizar arquivos que foram modificados nas últimas 2 horas:

$ find . -mmin -120

Para localizar arquivos que não foram modificados nas últimas 2 horas:

$ find . -mmin +120

O exemplo acima está pesquisando apenas na hora modificada - para pesquisar em ahoras de acesso, ou choras alteradas, use a ou c de acordo.

$ find . -amin -120
$ find . -cmin +120

Formato geral:

-mmin n: O arquivo foi modificado n minutos atrás
-mmin -n: O arquivo foi modificado há menos de n minutos
-mmin +n: O arquivo foi modificado há mais de n minutos atrás

Encontre arquivos que foram modificados nos últimos 2 dias:

find . -mtime -2

Encontre arquivos que não foram modificados nos últimos 2 dias

find . -mtime +2

Use -atime e -ctime para tempo de acesso e tempo de mudança de status, respectivamente.

Formato geral:

-mtime n: O arquivo foi modificado nx24 horas atrás
-mtime -n: O arquivo foi modificado há menos de nx24 horas
-mtime +n: O arquivo foi modificado há mais de nx24 horas atrás

Encontre arquivos modificados em um intervalo de datas, de 07/06/2007 a 08/06/2007:

find . -type f -newermt 2007-06-07 ! -newermt 2007-06-08

Encontre arquivos acessados ​​em um intervalo de carimbos de data/hora (usando arquivos como carimbo de data/hora), de 1 hora atrás a 10 minutos atrás:

touch -t $(date -d '1 HOUR AGO' +%Y%m%d%H%M.%S) start_date
touch -t $(date -d '10 MINUTE AGO' +%Y%m%d%H%M.%S) end_date
timeout 10 find "$LOCAL_FOLDER" -newerat "start_date" ! -newerat "end_date" -print

Formato geral:

-newerXY reference : Compara o timestamp do arquivo atual com a referência. XY pode ter um dos seguintes valores: at (tempo de acesso), mt (tempo de modificação), ct (tempo de mudança) e mais. reference é o nome de um arquivo que deseja comparar o timestamp especificado (acesso, modificação, alteração) ou uma string descrevendo um tempo absoluto.

Encontrando arquivos de acordo com o tamanho #

Encontre arquivos maiores que 15MB:

find -type f -size +15M

Encontre arquivos com menos de 12 KB:

find -type f -size -12k

Encontre arquivos com tamanho exato de 12KB:

find -type f -size 12k

Ou

find -type f -size 12288c

Ou

find -type f -size 24b

Ou

find -type f -size 24

Formato geral:

find [options] -size n[cwbkMG]

Encontre arquivos de tamanho n-bloco, onde +n significa mais que n-block, -n significa menos que n-block e n (sem nenhum sinal) significa exatamente n-block

Tamanho do bloco:

  1. c: bytes
  2. w: 2 bytes
  3. b: 512 bytes (padrão)
  4. k: 1 KB
  5. ``Eu’’: 1 MB
  6. G: 1 GB

Filtre o caminho #

O parâmetro -path permite especificar um padrão para corresponder ao caminho do resultado. O padrão também pode corresponder ao próprio nome.

Para encontrar apenas arquivos contendo log em qualquer lugar em seu caminho (pasta ou nome):

find . -type f -path '*log*'

Para encontrar apenas arquivos dentro de uma pasta chamada log (em qualquer nível):

find . -type f -path '*/log/*'

Para encontrar apenas arquivos dentro de uma pasta chamada log ou data:

find . -type f -path '*/log/*' -o -path '*/data/*'

Para encontrar todos os arquivos exceto os contidos em uma pasta chamada bin:

find . -type f -not -path '*/bin/*'

Para encontrar todos os arquivos, todos os arquivos exceto os contidos em uma pasta chamada bin ou arquivos de log:

find . -type f -not -path '*log' -not -path '*/bin/*'

Encontrando arquivos por tipo #

Para encontrar arquivos, use o sinalizador -type f

$ find . -type f

Para encontrar diretórios, use o sinalizador -type d

$ find . -type d

Para encontrar dispositivos de bloco, use o sinalizador -type b

$ find /dev -type b

Para encontrar links simbólicos, use o sinalizador -type l

$ find . -type l

Encontrando arquivos por extensão específica #

Para encontrar todos os arquivos de uma certa extensão dentro do caminho atual, você pode usar a seguinte sintaxe find. Ele funciona usando a construção glob interna do bash para corresponder a todos os nomes com a extensão .extension.

find /directory/to/search -maxdepth 1 -type f -name "*.extension"

Para encontrar todos os arquivos do tipo .txt somente do diretório atual, faça

find . -maxdepth 1 -type f -name "*.txt"
