Sintaxe #

  • Adicionar caminho: PATH=$PATH:/new/path
  • Adicionar caminho: PATH=/new/path:$PATH

Parâmetros #

Parâmetro Detalhes
CAMINHO Variável de ambiente de caminho

Arquivo de configuração Bash:

Este arquivo é originado sempre que um novo shell Bash interativo é iniciado.

Em sistemas GNU/Linux geralmente é o arquivo ~/.bashrc; no Mac é ~/.bash_profile ou ~/.profile

Exportar:

A variável PATH deve ser exportada uma vez (é feito por padrão). Uma vez exportado, ele permanecerá exportado e quaisquer alterações feitas nele serão aplicadas imediatamente.

Aplicar mudanças:

Para aplicar alterações em um arquivo de configuração Bash, você deve recarregar esse arquivo em um terminal (source /path/to/bash_config_file)

Adicione um caminho para a variável de ambiente PATH #

A variável de ambiente PATH é geralmente definida em ~/.bashrc ou ~/.bash_profile ou /etc/profile ou ~/.profile ou /etc/bash.bashrc (arquivo de configuração Bash específico da distro)

$ echo $PATH
/usr/local/sbin:/usr/local/bin:/usr/sbin:/usr/bin:/sbin:/bin:/usr/games:/usr/local/games:/snap/bin:/usr/lib/jvm/jdk1.8.0_92/bin:/usr/lib/jvm/jdk1.8.0_92/db/bin:/usr/lib/jvm/jdk1.8.0_92/jre/bin

Agora, se quisermos adicionar um caminho (por exemplo, ~/bin) à variável PATH:

PATH=~/bin:$PATH
# or
PATH=$PATH:~/bin

Mas isso modificará o PATH apenas no shell atual (e seu subshell). Depois de sair do shell, essa modificação desaparecerá.

Para torná-lo permanente, precisamos adicionar esse pedaço de código ao arquivo ~/.bashrc (ou qualquer outro) e recarregar o arquivo.

Se você executar o seguinte código (no terminal), ele adicionará ~/bin ao PATH permanentemente:

echo 'PATH=~/bin:$PATH' >> ~/.bashrc && source ~/.bashrc

Explicação:

  • echo 'PATH=~/bin:$PATH' >> ~/.bashrc adiciona a linha PATH=~/bin:$PATH no final do arquivo ~/.bashrc (você pode fazer isso com um texto editor)
  • source ~/.bashrc recarrega o arquivo ~/.bashrc
Este é um pouco de código (executado no terminal) que verificará se um caminho já está incluído e adicionará o caminho apenas se não estiver: 
path=~/bin            # path to be included
bashrc=~/.bashrc      # bash file to be written and reloaded
# run the following code unmodified
echo $PATH | grep -q "\(^\|:\)$path\(:\|/\{0,1\}$\)" || echo "PATH=\$PATH:$path" >> "$bashrc"; source "$bashrc"

Remova um caminho da variável de ambiente PATH #

Para remover um PATH de uma variável de ambiente PATH, você precisa editar o arquivo ~/.bashrc ou ~/.bash_profile ou /etc/profile ou ~/.profile ou /etc/bash.bashrc (específico da distro) e remover a atribuição para aquele caminho específico.

Em vez de encontrar a atribuição exata, você poderia apenas fazer uma substituição no $PATH em seu estágio final.

O seguinte removerá com segurança $path de $PATH:

path=~/bin
PATH="$(echo "$PATH" |sed -e "s#\(^\|:\)$(echo "$path" |sed -e 's/[^^]/[&]/g' -e 's/\^/\\^/g')\(:\|/\{0,1\}$\)#\1\2#" -e 's#:\+#:#g' -e 's#^:\|:$##g')"

Para torná-lo permanente, você precisará adicioná-lo no final do arquivo de configuração do bash.

Você pode fazer isso de maneira funcional: 
rpath(){
    for path in "$@";do
        PATH="$(echo "$PATH" |sed -e "s#\(^\|:\)$(echo "$path" |sed -e 's/[^^]/[&]/g' -e 's/\^/\\^/g')\(:\|/\{0,1\}$\)#\1\2#" -e 's#:\+#:#g' -e 's#^:\|:$##g')"
    done
    echo "$PATH"
}

PATH="$(rpath ~/bin /usr/local/sbin /usr/local/bin)"
PATH="$(rpath /usr/games)"
# etc ...

Isso tornará mais fácil lidar com vários caminhos.

Notas:

  • Você precisará adicionar esses códigos no arquivo de configuração do Bash (~/.bashrc ou qualquer outro).
  • Execute source ~/.bashrc para recarregar o arquivo de configuração do Bash (~/.bashrc).
