Uma visão geral das variáveis ​​internas do Bash, onde, como e quando usá-las.

Variáveis ​​internas do Bash em resumo #

Variável Detalhes
$* / $@ Parâmetros posicionais de função/script (argumentos). Expanda da seguinte forma:

$* e $@ são iguais a $1 $2 ... (observe que geralmente não faz sentido deixar sem aspas)
"$* " é o mesmo que "$1 $2 ..." 1
"$@" é o mesmo que "$1" "$2" ...
1. Os argumentos são separados pelo primeiro caractere de $IFS, que não precisa ser um espaço.
$# Número de parâmetros posicionais passados ​​para o script ou função
$! ID do processo do último comando (mais à direita para pipelines) no trabalho mais recente colocado em segundo plano (observe que não é necessariamente o mesmo que o ID do grupo de processos do trabalho quando o controle do trabalho está ativado)
$$ ID do processo que executou o bash
$? Status de saída do último comando
$n Parâmetros posicionais, onde n=1, 2, 3, …, 9
${n} Parâmetros posicionais (os mesmos acima), mas n pode ser > 9
$0 Em scripts, caminho com o qual o script foi chamado; com bash -c 'printf "%s\n" "$0"' name args': name (o primeiro argumento após o script embutido), caso contrário, o argv[0] que o bash recebeu.
$_ Último campo do último comando
$IFS Separador de campo interno
$PATH Variável de ambiente PATH usada para pesquisar executáveis
$OLDPWD Diretório de trabalho anterior
$PWD Diretório de trabalho atual
$FUNCNAME Matriz de nomes de funções na pilha de chamadas de execução
$BASH_SOURCE Array contendo caminhos de origem para elementos no array FUNCNAME. Pode ser usado para obter o caminho do script.
$BASH_ALIASES Array associativo contendo todos os aliases atualmente definidos
$BASH_REMATCH Matriz de partidas da última partida de regex
$BASH_VERSION Cadeia de versão do Bash
$BASH_VERSINFO Uma matriz de 6 elementos com informações de versão do Bash
$BASH Caminho absoluto para o próprio shell Bash atualmente em execução (determinado heuristicamente por bash com base em argv[0] e o valor de $PATH; pode estar errado em casos de canto)
$BASH_SUBSHELL Nível de subshell Bash
$UID Real (não efetivo se diferente) ID do usuário do processo executando bash
$PS1 Prompt de linha de comando principal; consulte Usando as variáveis ​​PS*
$PS2 Prompt de linha de comando secundário (usado para entrada adicional)
$PS3 Prompt de linha de comando terciário (usado no loop de seleção)
$PS4 Prompt de linha de comando quaternário (usado para anexar informações com saída detalhada)
$RANDOM Um inteiro pseudo aleatório entre 0 e 32767
$RESPOSTA Variável usada por read por padrão quando nenhuma variável é especificada. Também usado por select para retornar o valor fornecido pelo usuário
$PIPESTATUS Variável de matriz que contém os valores de status de saída de cada comando no pipeline de primeiro plano executado mais recentemente.

A atribuição de variável não deve ter espaço antes e depois. a=123 não a = 123. O último (um sinal de igual cercado por espaços) isoladamente significa executar o comando a com os argumentos = e 123, embora também seja visto no operador de comparação de strings (que sintaticamente é um argumento para [ ou [[ ou qualquer teste que você esteja usando).

##$@ "$@" expande para todos os argumentos da linha de comando como palavras separadas. É diferente de "$*", que se expande para todos os argumentos como uma única palavra.

"$@" é especialmente útil para [looping][2] por meio de argumentos e manipulação de argumentos com espaços.

Considere que estamos em um script que invocamos com dois argumentos, assim:

$ ./script.sh "␣1␣2␣" "␣3␣␣4␣"

As variáveis ​​$* ou $@ se expandirão para $1␣$2, que por sua vez se expandirá para 1␣2␣3␣4, então o loop abaixo:

for var in $*; do # same for var in $@; do
    echo \<"$var"\>
done

vai imprimir para ambos

<1>
<2>
<3>
<4>

Enquanto "$*" será expandido para "$1␣$2" que por sua vez se expandirá para "␣1␣2␣␣␣3␣␣4␣" e assim o loop:

for var in "$*"; do
    echo \<"$var"\>   
done

só invocará echo uma vez e imprimirá

<␣1␣2␣␣␣3␣␣4␣>

E finalmente "$@" se expandirá em "$1" "$2", que se expandirá em "␣1␣2␣" "␣3␣␣4␣" e assim o loop

for var in "$@"; do
    echo \<"$var"\>
done

vai imprimir

<␣1␣2␣>
<␣3␣␣4␣>

preservando assim tanto o espaçamento interno nos argumentos quanto a separação dos argumentos. Observe que a construção for var em "$@"; do ... é tão comum e idiomático que é o padrão para um loop for e pode ser encurtado para ` for var; fazer…'.

[2]: https://www.wikiod.com/pt/bash/estruturas-de-controle#Loop sobre uma matriz [3]: https://www.wikiod.com/pt/bash/matrizes

##$# Para obter o número de argumentos de linha de comando ou parâmetros posicionais - digite:

#!/bin/bash
echo "$#"

Quando executado com três argumentos, o exemplo acima resultará na saída:

~> $ ./testscript.sh firstarg secondarg thirdarg
3

$FUNCNAME #

Para obter o nome da função atual - digite:

my_function()
{
    echo "This function is $FUNCNAME"    # This will output "This function is my_function"
}

Esta instrução não retornará nada se você digitar fora da função:

my_function

echo "This function is $FUNCNAME"    # This will output "This function is"

$HOME #

O diretório inicial do usuário

~> $ echo $HOME
/home/user

$IFS #

Contém a string do Separador de Campo Interno que o bash usa para dividir strings ao fazer loop etc. O padrão são os caracteres de espaço em branco: \n (nova linha), \t (tab) e espaço. Alterar isso para outra coisa permite que você divida strings usando caracteres diferentes:

IFS=","
INPUTSTR="a,b,c,d"
for field in ${INPUTSTR}; do
    echo $field
done

A saída do acima é:

a
b
c
d

Notas:

  • Isso é responsável pelo fenômeno conhecido como [divisão de palavras](https://www.wikiod.com/pt/bash/divisao-de-palavras#O que, quando e por quê?).

$OLDPWD #

OLDPWD (OLDPrintWorkingDirectory) contém o diretório antes do último comando cd:

~> $ cd directory
directory> $ echo $OLDPWD
/home/user

$PWD #

PWD (PrintWtrabalhoDdiretório) O diretório de trabalho atual em que você está no momento:

~> $ echo $PWD
/home/user
~> $ cd directory
directory> $ echo $PWD
/home/user/directory

$ 1 $ 2 $ 3 etc… #

Parâmetros posicionais passados ​​para o script a partir da linha de comando ou de uma função:

#!/bin/bash
# $n is the n'th positional parameter
echo "$1"
echo "$2"
echo "$3"

A saída do acima é:

~> $ ./testscript.sh firstarg secondarg thirdarg
firstarg
secondarg
thirdarg

Se o número de argumentos posicionais for maior que nove, chaves devem ser usadas.

#  "set -- " sets positional parameters
set -- 1 2 3 4 5 6 7 8 nine ten eleven twelve
# the following line will output 10 not 1 as the value of $1 the digit 1
# will be concatenated with the following 0
echo $10   # outputs 1
echo ${10} # outputs ten
# to show this clearly:
set -- arg{1..12}
echo $10 
echo ${10}

##$* Retornará todos os parâmetros posicionais em uma única string.

testscript.sh:

#!/bin/bash
echo "$*"

Execute o script com vários argumentos:

./testscript.sh firstarg secondarg thirdarg

Resultado:

firstarg secondarg thirdarg

##$! O ID do processo (pid) do último trabalho executado em segundo plano:

~> $ ls &
testfile1 testfile2
[1]+  Done                    ls
~> $ echo $!
21715

##$? O status de saída da última função ou comando executado. Normalmente 0 significa OK, qualquer outra coisa indicará uma falha:

~> $ ls *.blah;echo $?
ls: cannot access *.blah: No such file or directory
2
~> $ ls;echo $?
testfile1 testfile2
0

$$ #

O ID do processo (pid) do processo atual:

~> $ echo $$
13246

$HISTSIZE #

O número máximo de comandos lembrados:

~> $ echo $HISTSIZE
1000

$BASHPID #

ID do processo (pid) da instância atual do Bash. Isso não é o mesmo que a variável $$, mas geralmente dá o mesmo resultado. Isso é novo no Bash 4 e não funciona no Bash 3.

~> $ echo "\$\$ pid = $$  BASHPID = $BASHPID"
$$ pid = 9265  BASHPID = 9265

$BASH_ENV #

Uma variável de ambiente apontando para o arquivo de inicialização do Bash que é lido quando um script é invocado.

$BASH_VERSINFO #

Um array contendo as informações completas da versão dividida em elementos, muito mais conveniente do que $BASH_VERSION se você estiver apenas procurando pela versão principal:

~> $ for ((i=0; i<=5; i++)); do echo "BASH_VERSINFO[$i] = ${BASH_VERSINFO[$i]}"; done
BASH_VERSINFO[0] = 3
BASH_VERSINFO[1] = 2
BASH_VERSINFO[2] = 25
BASH_VERSINFO[3] = 1
BASH_VERSINFO[4] = release
BASH_VERSINFO[5] = x86_64-redhat-linux-gnu

$BASH_VERSION #

Mostra a versão do bash que está sendo executada, isso permite que você decida se pode usar algum recurso avançado:

~> $ echo $BASH_VERSION
4.1.2(1)-release

$EDITOR #

O editor padrão que será invocado por quaisquer scripts ou programas, geralmente vi ou emacs.

~> $ echo $EDITOR
vi

$HOSTNAME #

O nome do host atribuído ao sistema durante a inicialização.

~> $ echo $HOSTNAME
mybox.mydomain.com

$HOSTTYPE #

Esta variável identifica o hardware, pode ser útil para determinar quais binários executar:

~> $ echo $HOSTTYPE
x86_64

$MACHTYPE #

Semelhante ao $HOSTTYPE acima, isso também inclui informações sobre o sistema operacional, bem como o hardware

~> $ echo $MACHTYPE
x86_64-redhat-linux-gnu

$OSTYPE #

Retorna informações sobre o tipo de sistema operacional em execução na máquina, por exemplo.

~> $ echo $OSTYPE
linux-gnu

$PATH #

O caminho de pesquisa para localizar binários para comandos. Exemplos comuns incluem /usr/bin e /usr/local/bin.

Quando um usuário ou script tenta executar um comando, os caminhos em $PATH são pesquisados ​​para encontrar um arquivo correspondente com permissão de execução.

Os diretórios em $PATH são separados por um caractere :.

~> $ echo "$PATH"
/usr/kerberos/bin:/usr/local/bin:/bin:/usr/bin

Então, por exemplo, dado o $PATH acima, se você digitar lss no prompt, o shell procurará por /usr/kerberos/bin/lss, então /usr/local/bin/lss , então /bin/lss, então /usr/bin/lss, nesta ordem, antes de concluir que não existe tal comando.

$PPID #

O ID do processo (pid) do script ou pai do shell, significando que o processo invocou o script ou shell atual.

~> $ echo $$
13016
~> $ echo $PPID
13015

$SECONDS #

O número de segundos que um script está em execução. Isso pode ficar muito grande se mostrado no shell:

~> $ echo $SECONDS
98834

$SHELLOPTS #

Uma lista somente leitura das opções bash é fornecida na inicialização para controlar seu comportamento:

~> $ echo $SHELLOPTS
braceexpand:emacs:hashall:histexpand:history:interactive-comments:monitor

$_ #

Gera o último campo do último comando executado, útil para obter algo para passar para outro comando:

 ~> $ ls *.sh;echo $_
testscript1.sh  testscript2.sh
testscript2.sh
Ele fornece o caminho do script se usado antes de qualquer outro comando:

test.sh:

#!/bin/bash
echo "$_"

Resultado:

~> $ ./test.sh # running test.sh
./test.sh

Observação: Esta não é uma maneira infalível de obter o caminho do script

$RANDOM #

Cada vez que este parâmetro é referenciado, um número inteiro aleatório entre 0 e 32767 é gerado. Atribuir um valor a esta variável propaga o gerador de números aleatórios (source).

~> $ echo $RANDOM 
27119
~> $ echo $RANDOM 
1349

$GROUPS #

Uma matriz contendo os números de grupos em que o usuário está:

#!/usr/bin/env bash
echo You are assigned to the following groups:
for group in ${GROUPS[@]}; do
   IFS=: read -r name dummy number members < <(getent group $group )
   printf "name: %-10s number: %-15s members: %s\n" "$name" "$number" "$members" 
done

$Linho #

Gera o número da linha no script atual. Principalmente útil ao depurar scripts.

#!/bin/bash
# this is line 2
echo something  # this is line 3
echo $LINENO # Will output 4

$SHLVL #

Quando o comando bash é executado, um novo shell é aberto. A variável de ambiente $SHLVL contém o número de níveis de shell sobre os quais o shell current está sendo executado.

Em uma janela de terminal new, a execução do comando a seguir produzirá resultados diferentes com base na distribuição Linux em uso.

echo $SHLVL

Usando o Fedora 25, a saída é “3”. Isso indica que, ao abrir um novo shell, um comando bash inicial executa e executa uma tarefa. O comando bash inicial executa um processo filho (outro comando bash) que, por sua vez, executa um comando bash final para abrir o novo shell. Quando o novo shell é aberto, ele está sendo executado como um processo filho de 2 outros processos do shell, daí a saída de “3”.

No exemplo a seguir (dado que o usuário está executando o Fedora 25), a saída de $SHLVL em um novo shell será definida como “3”. À medida que cada comando bash é executado, $SHLVL é incrementado em um.

~> $ echo $SHLVL
3
~> $ bash
~> $ echo $SHLVL
4
~> $ bash
~> $ echo $SHLVL
5

Pode-se ver que executar o comando ‘bash’ (ou executar um script bash) abre um novo shell. Em comparação, a origem de um script executa o código no shell atual.

teste1.sh

#!/usr/bin/env bash
echo "Hello from test1.sh. My shell level is $SHLVL"
source "test2.sh"

teste2.sh

#!/usr/bin/env bash
echo "Hello from test2.sh. My shell level is $SHLVL"

run.sh

#!/usr/bin/env bash
echo "Hello from run.sh. My shell level is $SHLVL"
./test1.sh

Executar:

chmod +x test1.sh && chmod +x run.sh
./run.sh
