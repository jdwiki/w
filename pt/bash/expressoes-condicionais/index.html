Sintaxe #

  • [[ -OP $nomedoarquivo]]
  • [[ $arquivo1 -OP $arquivo2]]
  • [[ -z $string]]
  • [[ -n $string]]
  • [[ “$string1” == “$string2” ]]
  • [[ “$string1” == $padrão]]

A sintaxe [[ … ]] envolve as expressões condicionais internas do bash. Observe que são necessários espaços em ambos os lados dos colchetes.

Expressões condicionais podem usar operadores unários e binários para testar propriedades de strings, inteiros e arquivos. Eles também podem usar os operadores lógicos &&, || e !.

Testes de tipo de arquivo #

O operador condicional -e testa se um arquivo existe (incluindo todos os tipos de arquivo: diretórios, etc.).

if [[ -e $filename ]]; then
  echo "$filename exists"
fi

Existem testes para tipos de arquivos específicos também.

if [[ -f $filename ]]; then
  echo "$filename is a regular file"
elif [[ -d $filename ]]; then
  echo "$filename is a directory"
elif [[ -p $filename ]]; then
  echo "$filename is a named pipe"
elif [[ -S $filename ]]; then
  echo "$filename is a named socket"
elif [[ -b $filename ]]; then
  echo "$filename is a block device"
elif [[ -c $filename ]]; then
  echo "$filename is a character device"
fi
if [[ -L $filename ]]; then
  echo "$filename is a symbolic link (to any file type)"
fi

Para um link simbólico, além de -L, esses testes se aplicam ao destino e retornam false para um link quebrado.

if [[ -L $filename || -e $filename ]]; then
  echo "$filename exists (but may be a broken symbolic link)"
fi

if [[ -L $filename && ! -e $filename ]]; then
  echo "$filename is a broken symbolic link"
fi

Comparação e correspondência de strings #

A comparação de strings usa o operador == entre strings entre aspas. O operador != nega a comparação.

if [[ "$string1" == "$string2" ]]; then
  echo "\$string1 and \$string2 are identical"
fi
if [[ "$string1" != "$string2" ]]; then
  echo "\$string1 and \$string2 are not identical"
fi

Se o lado direito não estiver entre aspas, então é um padrão curinga com o qual $string1 corresponde.

string='abc'
pattern1='a*'
pattern2='x*'
if [[ "$string" == $pattern1 ]]; then
  # the test is true
  echo "The string $string matches the pattern $pattern"
fi
if [[ "$string" != $pattern2 ]]; then
  # the test is false
  echo "The string $string does not match the pattern $pattern"
fi

Os operadores < e > comparam as strings em ordem lexicográfica (não há operadores menor ou igual ou maior ou igual para strings).

Existem testes unários para a string vazia.

if [[ -n "$string" ]]; then
  echo "$string is non-empty"
fi
if [[ -z "${string// }" ]]; then
  echo "$string is empty or contains only spaces"
fi
if [[ -z "$string" ]]; then
  echo "$string is empty"
fi

Acima, a verificação -z pode significar que $string não está definido ou está definido como uma string vazia. Para distinguir entre vazio e não definido, use:

if [[ -n "${string+x}" ]]; then
    echo "$string is set, possibly to the empty string"
fi
if [[ -n "${string-x}" ]]; then
    echo "$string is either unset or set to a non-empty string"
fi
if [[ -z "${string+x}" ]]; then
    echo "$string is unset"
fi
if [[ -z "${string-x}" ]]; then
    echo "$string is set to an empty string"
fi

onde x é arbitrário. Ou em formato de tabela:

                        +-------+-------+-----------+
            $string is: | unset | empty | non-empty |
+-----------------------+-------+-------+-----------+
| [[ -z ${string} ]]    | true  | true  | false     |
| [[ -z ${string+x} ]]  | true  | false | false     |
| [[ -z ${string-x} ]]  | false | true  | false     |
| [[ -n ${string} ]]    | false | false | true      |
| [[ -n ${string+x} ]]  | false | true  | true      |
| [[ -n ${string-x} ]]  | true  | false | true      |
+-----------------------+-------+-------+-----------+

Alternativamente, o estado pode ser verificado em uma instrução case:

case ${var+x$var} in
  (x) echo empty;;
  ("") echo unset;;
  (x*[![:blank:]]*) echo non-blank;;
  (*) echo blank
esac

Onde [:blank:] são caracteres de espaçamento horizontal específicos da localidade (tab, espaço, etc).

Teste o status de saída de um comando #

Status de saída 0: sucesso
Status de saída diferente de 0: falha

Para testar o status de saída de um comando:

if command;then
    echo 'success'
else
    echo 'failure'
fi

Comparação de arquivos #

if [[ $file1 -ef $file2 ]]; then
  echo "$file1 and $file2 are the same file"
fi

“Mesmo arquivo” significa que a modificação de um dos arquivos no local afeta o outro. Dois arquivos podem ser iguais mesmo que tenham nomes diferentes, por exemplo, se forem links físicos, ou se forem links simbólicos com o mesmo destino, ou se um for um link simbólico apontando para o outro.

Se dois arquivos têm o mesmo conteúdo, mas são arquivos distintos (para que a modificação de um não afete o outro), então -ef os informa como diferentes. Se você quiser comparar dois arquivos byte a byte, use o utilitário cmp.

if cmp -s -- "$file1" "$file2"; then
  echo "$file1 and $file2 have identical contents"
else
  echo "$file1 and $file2 differ"
fi

Para produzir uma lista legível de diferenças entre arquivos de texto, use o utilitário diff.

if diff -u "$file1" "$file2"; then
  echo "$file1 and $file2 have identical contents"
else
  : # the differences between the files have been listed
fi

Testes de acesso a arquivos #

if [[ -r $filename ]]; then
  echo "$filename is a readable file"
fi
if [[ -w $filename ]]; then
  echo "$filename is a writable file"
fi
if [[ -x $filename ]]; then
  echo "$filename is an executable file"
fi

Esses testes levam em consideração as permissões e a propriedade para determinar se o script (ou programas iniciados a partir do script) podem acessar o arquivo.

Cuidado com as condições de corrida (TOCTOU): só porque o teste é bem-sucedido agora não significa que ainda é válido na próxima linha. Geralmente, é melhor tentar acessar um arquivo e lidar com o erro, em vez de testar primeiro e depois lidar com o erro de qualquer maneira, caso o arquivo tenha sido alterado nesse meio tempo.

Comparações numéricas #

Comparações numéricas usam os operadores -eq e amigos

if [[ $num1 -eq $num2 ]]; then
  echo "$num1 == $num2"
fi
if [[ $num1 -le $num2 ]]; then
  echo "$num1 <= $num2"
fi

Existem seis operadores numéricos:

  • -eq igual
  • -ne não é igual
  • -le menor ou igual
  • -lt menor que
  • -ge maior ou igual
  • -gt maior que

Observe que os operadores < e > dentro de [[ … ]] comparam strings, não números.

if [[ 9 -lt 10 ]]; then
  echo "9 is before 10 in numeric order"
fi
if [[ 9 > 10 ]]; then
  echo "9 is after 10 in lexicographic order"
fi

Os dois lados devem ser números escritos em decimal (ou em octal com um zero à esquerda). Como alternativa, use a sintaxe de expressão aritmética ((…)), que executa cálculos inteiros em uma sintaxe semelhante a C/Java/….

x=2
if ((2*x == 4)); then
  echo "2 times 2 is 4"
fi
((x += 1))
echo "2 plus 1 is $x"

Teste de um forro #

Você pode fazer coisas assim:

[[ $s = 'something' ]] && echo 'matched' || echo "didn't match"
[[ $s == 'something' ]] && echo 'matched' || echo "didn't match"
[[ $s != 'something' ]] && echo "didn't match" || echo "matched"
[[ $s -eq 10 ]] && echo 'equal' || echo "not equal"
(( $s == 10 )) && echo 'equal' || echo 'not equal'

Teste de um liner para status de saída:

command && echo 'exited with 0' || echo 'non 0 exit'
cmd && cmd1 && echo 'previous cmds were successful' || echo 'one of them failed'
cmd || cmd1 #If cmd fails try cmd1
