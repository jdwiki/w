Sintaxe #

  • $ shopt -u option # Desativa a ‘opção’ embutida do Bash
  • $ shopt -s option # Ativa a ‘opção’ embutida do Bash

Classes de Personagens

As classes de caracteres válidas para o glob [] são definidas pelo padrão POSIX:

alnum alpha ascii em branco ctrl digit graph espaço de punct de impressão inferior palavra superior xdígito

Dentro de [] mais de uma classe de caractere ou intervalo pode ser usado, por exemplo,

$ echo a[a-z[:blank:]0-9]*

corresponderá a qualquer arquivo que comece com um a e seja seguido por uma letra minúscula ou um espaço em branco ou um dígito.

Deve-se ter em mente, porém, que um glob [] só pode ser totalmente negado e não apenas partes dele. O caractere de negação deve ser o primeiro caractere após a abertura [, por exemplo, esta expressão corresponde a todos os arquivos que não começam com um a

$ echo [^a]*

O seguinte corresponde a todos os arquivos que começam com um dígito ou um ^

$ echo [[:alpha:]^a]*

Ele não corresponde a nenhum arquivo ou pasta que comece com uma letra, exceto um a porque o ^ é interpretado como um ^ literal.

** Caracteres glob de escape**

É possível que um arquivo ou pasta contenha um caractere glob como parte de seu nome. Nesse caso, um glob pode ser escapado com um \ precedente para uma correspondência literal. Outra abordagem é usar aspas duplas "" ou simples '' para endereçar o arquivo. Bash não processa globs que estão dentro de "" ou ''.

Diferença para expressões regulares

A diferença mais significativa entre globs e Expressões Regulares é que uma Expressão Regular válida requer um qualificador, bem como um quantificador. Um qualificador identifica o que combinar e um quantificador informa com que frequência para corresponder ao qualificador. O RegEx equivalente ao glob * é .* onde . significa qualquer caractere e * significa zero ou mais correspondências do personagem anterior. O RegEx equivalente para o glob ? é .{1}. Como antes, o qualificador . corresponde a qualquer caractere e o {1} indica para corresponder ao qualificador anterior exatamente uma vez. Isso não deve ser confundido com o quantificador ?, que corresponde a zero ou uma vez em um RegEx. O glob [] pode ser usado da mesma forma em um RegEx, desde que seja seguido de um quantificador obrigatório.

Expressões regulares equivalentes

Globo RegEx
* .*
? .
[] []

Verifica se uma string corresponde a uma expressão regular #

Verifique se uma string consiste em exatamente 8 dígitos:

$ date=20150624
$ [[ $date =~ ^[0-9]{8}$ ]] && echo "yes" || echo "no"
yes
$ date=hello
$ [[ $date =~ ^[0-9]{8}$ ]] && echo "yes" || echo "no"
no

Comportamento quando um glob não corresponde a nada #

Preparação

$ mkdir globbing
$ cd globbing
$ mkdir -p folder/{sub,another}folder/content/deepfolder/
touch macy stacy tracy "file with space" folder/{sub,another}folder/content/deepfolder/file .hiddenfile
$ shopt -u nullglob
$ shopt -u failglob
$ shopt -u dotglob
$ shopt -u nocaseglob
$ shopt -u extglob
$ shopt -u globstar

Caso o glob não corresponda a nada, o resultado é determinado pelo opções nullglob e failglob. Se nenhum deles estiver definido, o Bash retornará o próprio glob se nada for correspondido

$ echo no*match
no*match

Se nullglob for ativado, nada (null) será retornado:

$ shopt -s nullglob
$ echo no*match

$

Se failglob for ativado, uma mensagem de erro será retornada:

$ shopt -s failglob
$ echo no*match
bash: no match: no*match
$

Observe que a opção failglob substitui a opção nullglob, ou seja, se nullglob e failglob estiverem definidos, então - em caso de não correspondência - um erro é retornado.

Obtenha grupos capturados de uma correspondência de regex com uma string #

O *globo #

Preparação

$ mkdir globbing
$ cd globbing
$ mkdir -p folder/{sub,another}folder/content/deepfolder/
touch macy stacy tracy "file with space" folder/{sub,another}folder/content/deepfolder/file .hiddenfile
$ shopt -u nullglob
$ shopt -u failglob
$ shopt -u dotglob
$ shopt -u nocaseglob
$ shopt -u extglob
$ shopt -u globstar

O asterisco * é provavelmente o glob mais comumente usado. Ele simplesmente corresponde a qualquer Corda

$ echo *acy
macy stacy tracy

Um único * não corresponderá a arquivos e pastas que residem em subpastas

$ echo *
emptyfolder folder macy stacy tracy
$ echo folder/*
folder/anotherfolder folder/subfolder

O ** globo #

Preparação

$ mkdir globbing
$ cd globbing
$ mkdir -p folder/{sub,another}folder/content/deepfolder/
touch macy stacy tracy "file with space" folder/{sub,another}folder/content/deepfolder/file .hiddenfile
$ shopt -u nullglob
$ shopt -u failglob
$ shopt -u dotglob
$ shopt -u nocaseglob
$ shopt -u extglob
$ shopt -s globstar

O Bash é capaz de interpretar dois asteriscos adjacentes como um único glob. Com o globstar opção activada pode ser usada para corresponder às pastas que residem mais profundamente na estrutura de directórios

echo **
emptyfolder folder folder/anotherfolder folder/anotherfolder/content folder/anotherfolder/content/deepfolder folder/anotherfolder/content/deepfolder/file folder/subfolder folder/subfolder/content folder/subfolder/content/deepfolder folder/subfolder/content/deepfolder/file macy stacy tracy

O ** pode ser pensado como uma expansão de caminho, não importa quão profundo seja o caminho. Este exemplo corresponde a qualquer arquivo ou pasta que comece com deep, independentemente de como deep está aninhado:

$ echo **/deep*
folder/anotherfolder/content/deepfolder folder/subfolder/content/deepfolder

O ? globo #

Preparação

$ mkdir globbing
$ cd globbing
$ mkdir -p folder/{sub,another}folder/content/deepfolder/
touch macy stacy tracy "file with space" folder/{sub,another}folder/content/deepfolder/file .hiddenfile
$ shopt -u nullglob
$ shopt -u failglob
$ shopt -u dotglob
$ shopt -u nocaseglob
$ shopt -u extglob
$ shopt -u globstar

O ? simplesmente corresponde exatamente a um caractere

$ echo ?acy
macy
$ echo ??acy
stacy tracy

O globo [ ] #

Preparação

$ mkdir globbing
$ cd globbing
$ mkdir -p folder/{sub,another}folder/content/deepfolder/
touch macy stacy tracy "file with space" folder/{sub,another}folder/content/deepfolder/file .hiddenfile
$ shopt -u nullglob
$ shopt -u failglob
$ shopt -u dotglob
$ shopt -u nocaseglob
$ shopt -u extglob
$ shopt -u globstar

Se houver necessidade de combinar caracteres específicos, ‘[]’ pode ser usado. Algum caractere dentro de ‘[]’ será correspondido exatamente uma vez.

$ echo [m]acy
macy
$ echo [st][tr]acy
stacy tracy

O glob [], no entanto, é mais versátil do que apenas isso. Também permite para uma correspondência negativa e até mesmo intervalos correspondentes de caracteres e classes de personagens. Uma correspondência negativa é obtida usando ! ou ^ como o primeiro caractere seguindo [. Podemos combinar stacy por

$ echo [!t][^r]acy
stacy

Aqui estamos dizendo ao bash que queremos combinar apenas os arquivos que não começa com um t e a segunda letra não é um r e o arquivo termina em acy.

Os intervalos podem ser combinados separando um par de caracteres com um hífen (-). Algum caractere que fica entre esses dois caracteres delimitadores - inclusive - será ser correspondido. Por exemplo, [r-t] é equivalente a [rst]

$ echo [r-t][r-t]acy
stacy tracy

As classes de caracteres podem ser combinadas por [:class:], por exemplo, para corresponder arquivos que contêm um espaço em branco

$ echo *[[:blank:]]*
file with space

Correspondência de arquivos ocultos #

Preparação

$ mkdir globbing
$ cd globbing
$ mkdir -p folder/{sub,another}folder/content/deepfolder/
touch macy stacy tracy "file with space" folder/{sub,another}folder/content/deepfolder/file .hiddenfile
$ shopt -u nullglob
$ shopt -u failglob
$ shopt -u dotglob
$ shopt -u nocaseglob
$ shopt -u extglob
$ shopt -u globstar

A opção interna do Bash dotglob permite combinar arquivos ocultos e pastas, ou seja, arquivos e pastas que começam com um .

$ shopt -s dotglob
$ echo *
file with space folder .hiddenfile macy stacy tracy

Correspondência sem distinção entre maiúsculas e minúsculas #

Preparação

$ mkdir globbing
$ cd globbing
$ mkdir -p folder/{sub,another}folder/content/deepfolder/
touch macy stacy tracy "file with space" folder/{sub,another}folder/content/deepfolder/file .hiddenfile
$ shopt -u nullglob
$ shopt -u failglob
$ shopt -u dotglob
$ shopt -u nocaseglob
$ shopt -u extglob
$ shopt -u globstar

Definir a opção nocaseglob corresponderá ao glob em um caso que não diferencia maiúsculas de minúsculas maneiras

$ echo M*
M*
$ shopt -s nocaseglob 
$ echo M*
macy

Globbing estendido #

Preparação

$ mkdir globbing
$ cd globbing
$ mkdir -p folder/{sub,another}folder/content/deepfolder/
touch macy stacy tracy "file with space" folder/{sub,another}folder/content/deepfolder/file .hiddenfile
$ shopt -u nullglob
$ shopt -u failglob
$ shopt -u dotglob
$ shopt -u nocaseglob
$ shopt -u extglob
$ shopt -u globstar

A opção interna extglob do Bash pode estender os recursos de correspondência de um glob

shopt -s extglob

Os seguintes subpadrões compreendem globs estendidos válidos:

  • ?(lista de padrões) – Corresponde a zero ou uma ocorrência dos padrões fornecidos
  • *(lista de padrões) – Corresponde a zero ou mais ocorrências dos padrões fornecidos
  • +(lista de padrões) – Corresponde a uma ou mais ocorrências dos padrões fornecidos
  • @(lista de padrões) – Corresponde a um dos padrões fornecidos
  • !(lista de padrões) – Corresponde a qualquer coisa, exceto um dos padrões fornecidos

A pattern-list é uma lista de globs separados por |.

$ echo *([r-t])acy
stacy tracy

$ echo *([r-t]|m)acy
macy stacy tracy

$ echo ?([a-z])acy
macy

A própria pattern-list pode ser outro glob estendido aninhado. No acima Por exemplo, vimos que podemos combinar tracy e stacy com *(r-t). Este glob estendido em si pode ser usado dentro do glob estendido negado !(pattern-list) para combinar com macy

$ echo !(*([r-t]))acy
macy

Ele corresponde a qualquer coisa que não comece com zero ou mais ocorrências do letras r, s e t, o que deixa apenas macy como correspondência possível.

Correspondência de regex #

pat='[^0-9]+([0-9]+)'
s='I am a string with some digits 1024'
[[ $s =~ $pat ]] # $pat must be unquoted
echo "${BASH_REMATCH[0]}"
echo "${BASH_REMATCH[1]}"

Resultado:

I am a string with some digits 1024
1024

Em vez de atribuir o regex a uma variável ($pat), também poderíamos fazer:

[[ $s =~ [^0-9]+([0-9]+) ]]

Explicação

  • A construção [[ $s =~ $pat ]] executa a correspondência regex
  • Os grupos capturados, ou seja, os resultados da correspondência estão disponíveis em uma matriz chamada BASH_REMATCH
  • O índice 0 no array BASH_REMATCH é a correspondência total
  • O i’th índice no array BASH_REMATCH é o i’th grupo capturado, onde i = 1, 2, 3 …
