Sintaxe #

  • <!-- ko if:myObservable --><!-- /ko -->

O que é uma ligação #

Essencialmente, uma vinculação ou uma vinculação de dados é uma maneira de vincular seus ViewModels às suas exibições (modelos) e vice-versa. O KnockoutJS usa vinculação de dados bidirecional, o que significa que as alterações no seu ViewModel influenciam o View e as alterações no seu View podem influenciar o ViewModel.

Sob o capô (breve visão geral) #

As ligações são apenas plugins (scripts) que permitem resolver uma tarefa específica. Esta tarefa é mais frequentemente do que não está alterando a marcação (html) de acordo com o seu ViewModel.

Por exemplo, uma ligação text permite exibir texto e alterá-lo dinamicamente sempre que seu ViewModel for alterado.

KnockoutJS vem com muitas ligações poderosas e permite estendê-lo com suas próprias ligações personalizadas.

E o mais importante, as ligações não são mágicas, elas funcionam de acordo com um conjunto de regras e sempre que você não tiver certeza do que uma ligação faz, quais parâmetros ela usa ou quando ela atualizará a visualização, você pode consultar o código-fonte da ligação.

Considere o seguinte exemplo de uma vinculação personalizada:

ko.bindingHandlers.debug = {
    init: function (element, valueAccessor, allBindingsAccessor, viewModel, bindingContext) {
        ko.computed(function () {
            var value = ko.unwrap(valueAccessor());
    
            console.log({
                value: value,
                viewModel: viewModel,
                bindingContext: bindingContext
            });
        }, null, { disposeWhenNodeIsRemoved: element });
    }
};
  1. Uma ligação tem um nome - debug para que você possa usar da seguinte forma:

data-bind="depurar: 'foo'"

  1. O método init é chamado uma vez quando a ligação é iniciada. O resto das atualizações são tratadas por um computador anônimo que é descartado quando o elemento é removido.
  2. O binding imprime para consolar várias coisas: o valor passado em nosso exemplo este valor é foo (este valor também pode ser observável desde que o método ko.unwrap seja usado para lê-lo), o viewModel atual e bindingContext.
  3. Sempre que o valor passado for alterado, a ligação imprimirá informações atualizadas no console.
  4. Esta ligação não pode ser usada com elementos virtuais (em comentários html), somente em elementos reais, pois o sinalizador ko.virtualElements.allowedBindings.debug não está definido como true.

Quando usar parênteses #

Sem quaisquer plugins adicionais, o KnockoutJS só terá atualizações de visualização ao vivo para propriedades no ViewModel que são observáveis (regular observable, mas também computed, pureComputed, observableArray, etc). Um observável seria criado assim:

var vm = { name: ko.observable("John") };

Neste caso, vm.name é uma função com dois “modos” separados:

  1. Getter: vm.name(), sem argumentos, obterá o valor atual;
  2. Setter: vm.name("Johnnyboy"), com um argumento, definirá um novo valor.

Nos data-bindings embutidos você pode sempre usar o formulário getter, e você pode às vezes na verdade omitir os parênteses, e a vinculação irá efetivamente “adicioná-los” para você. Então são equivalentes:

<p data-bind="text: name"></p> ... will work
<p data-bind="text: name()"></p> ... works too

Mas isso vai falhar:

<p data-bind="text: 'Hello, ' + name + '!'"></p> ... FAILS!

Porque assim que você quiser “fazer” algo antes de passar um valor para uma vinculação de dados, incluindo comparações de valor, você precisa “obter” corretamente os valores para todos os observáveis, por exemplo:

<p data-bind="text: 'Hello, ' + name() + '!'"></p> ... works

Veja também este Q&A para mais detalhes.

Para cada #

Semelhante aos repetidores usados ​​em outros idiomas. Essa ligação permitirá que você replique um bloco de html para cada item em uma matriz.

<div data-bind="foreach:contacts">
    <div class="contact">
        <h2 data-bind="text:name">
        <p data-bind="text:info">
    </div>
</div>
 
<script type="text/javascript">
    var contactViewModel = function (data) {
        this.name = ko.observable(data.name);
        this.info = ko.observable(data.info);
    };

    ko.applyBindings({
        contacts: [
            new contactViewModel({name:'Erik', info:'Erik@gmail.com'}),
            new contactViewModel({name:'Audrey', info:'Audrey@gmail.com'})
        ]
    });
</script>

Observe que quando estamos fazendo um loop pelo nosso contexto se torna o item dentro do array, neste caso uma instância do contactViewModel. Dentro de um foreach também temos acesso a

  • $parent - o modelo de visualização que criou esta ligação
  • $root - o modelo de visualização raiz (também pode ser pai)
  • $data - os dados neste índice do array
  • $index - o índice baseado em zero (observável) do item renderizado

Com #

A vinculação with vincula o HTML dentro do nó vinculado a um contexto separado:

<div data-bind="with: subViewModel">
  <p data-bind="text: title"></p>
</div>

A ligação with também pode ser usada sem um elemento container onde um elemento container pode não ser apropriado.

<!-- ko with: subViewModel -->
  <p data-bind="text: title"></p>
<!-- /ko -->

var vm = {
  subViewModel: ko.observable()
};

// Doesn't throw an error on the `text: title`; the `<p>` element 
// isn't bound to any context (and even removed from the DOM)
ko.applyBindings(vm);

// Includes the `<p>` element and binds it to our new object
vm.subViewModel({ title: "SubViewModel" });

A ligação with tem muitas semelhanças com as ligações template ou foreach.

Se / se não #

Você pode usar a ligação if para determinar se os elementos filhos do nó devem ou não ser criados.

<div class="product-info">
    <h2>  Product1  </h2>
    <img src="../products/product1.jpg"/>
    <span data-bind="if:featured">
        <span class="featured"></span>
    </span>
    <span data-bind="ifnot:inStock">
        <span class="out-of-stock"></span>
    </span>
</div>
     
<script>
    ko.applyBindings({
        product: {
            featured: ko.observable(true),
            inStock: ko.observable(false)
        }
    });
</script>

O inverso da ligação if é ifnot

<div data-bind="ifnot: someProperty">...</div>

é equivalente a

<div data-bind="if: !someProperty()">...</div>

Às vezes, você não vai querer controlar a presença de elementos sem ter que criar um container (normalmente para elementos <li> em elementos <ul> ou <option> dentro de um <select> )

Knockout permite isso com sintaxe de fluxo de controle sem contêiner com base em tags de comentário como:

<select>
  <option value="0">fixed option</option>
<!-- ko if: featured-->
  <option value="1">featured option</option>
<!-- /ko -->
</select>

Visível #

A vinculação visible ocultará um elemento aplicando style="display: none;" a ele quando a vinculação for avaliada como falsey.

<input type="text" data-bind="textInput: name"> <span class="error" data-bind="visible: isInvalid">Required!</span>

ko.applyBindings(new ViewModel());

function ViewModel(){
    var vm = this;
  vm.name = ko.observable("test");
  vm.isInvalid = ko.computed(function() {
      return vm.name().length == 0;
  });
}
