Declarando scripts de conteúdo no manifesto #

Scripts de conteúdo podem ser declarados em manifest.json para serem sempre injetados em páginas que correspondam a um conjunto de padrões de URL.

Exemplo mínimo #

"content_scripts" : [
  {
    "js": ["content.js"],
    "css": ["content.css"]
    "matches": ["http://example.com/*"]
  }
]

Esta entrada de manifesto instrui o Chrome a injetar um script de conteúdo content.js, juntamente com o arquivo CSS content.css, após qualquer navegação para uma página que corresponda ao padrão de correspondência http://example.com /*

Ambas as chaves js e css são opcionais: você pode ter apenas uma delas ou ambas dependendo do que você precisa.

A chave content_scripts é uma matriz e você pode declarar várias definições de script de conteúdo:

"content_scripts" : [
  {
    "js": ["content.js"],
    "matches": ["http://*.example.com/*"]
  },
  {
    "js": ["something_else.js"],
    "matches": ["http://*.example.org/*"]
  }
]

Observe que js e matches são arrays, mesmo se você tiver apenas uma entrada.

Mais opções estão disponíveis na documentação oficial e outros exemplos.

Nota importante #

Scripts de conteúdo declarados no manifesto só serão injetados em novas navegações após o carregamento da extensão. Eles não serão injetados em abas existentes. Isso também se aplica a recargas de extensão durante o desenvolvimento e atualizações de extensão após o lançamento.

Se você precisar garantir que as guias abertas no momento sejam cobertas, considere também fazer a injeção programática na inicialização.

Injetando scripts de conteúdo de uma página de extensão #

Se, em vez de sempre ter um script de conteúdo injetado com base na URL, você deseja controlar diretamente quando um script de conteúdo é injetado, você pode usar Programmatic Injection.

Exemplo mínimo #

-Javascript

  chrome.tabs.executeScript({file: "content.js"});
  • CSS
  chrome.tabs.insertCSS({file: "content.css"});

Chamado de uma página de extensão (por exemplo, plano de fundo ou pop-up), e supondo que você tenha permissão para injetar, isso executará content.js ou inserirá content.css como um script de conteúdo no quadro superior da guia atual.

Código embutido #

Você pode executar código embutido em vez de um arquivo como um script de conteúdo:

var code = "console.log('This code will execute as a content script');";
chrome.tabs.executeScript({code: code});

Escolhendo a aba #

Você pode fornecer um ID de guia (geralmente de outros métodos ou mensagens chrome.tabs) para executar em uma guia diferente da atualmente ativa.

chrome.tabs.executeScript({
  tabId: tabId,
  file: "content.js"
});

Mais opções estão disponíveis na chrome.tabs.executeScript() documentação e em outros exemplos.

Permissões #

Usar chrome.tabs.executeScript() não requer permissão "tabs", mas requer permissões de host para o URL da página.

Verificando erros #

Se a injeção de script falhar, pode-se pegá-la no retorno de chamada opcional:

chrome.tabs.executeScript({file: "content.js"}, function() {
  if(chrome.runtime.lastError) {
    console.error("Script injection failed: " + chrome.runtime.lastError.message);
  }
});

Vários scripts de conteúdo no manifesto #

Mesmas condições, vários scripts #

Se você precisar injetar vários arquivos com todas as outras condições iguais, por exemplo, para incluir uma biblioteca, você pode listar todos eles no array "js":

"content_scripts" : [
  {
    "js": ["library.js", "content.js"],
    "matches": ["http://*.example.com/*"]
  }
]

A ordem importa: library.js será executado antes de content.js.

Mesmos scripts, vários sites #

Se você precisar injetar os mesmos arquivos em vários sites, poderá fornecer vários padrões de correspondência:

"matches": ["http://example.com/*", "http://example.org/*"]

Se você precisar injetar em basicamente todas as páginas, poderá usar padrões de correspondência ampla, como "*://*/*" (corresponde a todas as páginas HTTP(S)) ou "<all_urls>" (corresponde a todos os página suportada).

Scripts diferentes ou sites diferentes #

A seção "content_scripts" também é uma matriz, portanto, pode-se definir mais de um bloco de script de conteúdo:

"content_scripts" : [
  {
    "js": ["content.js"],
    "matches": ["http://*.example.com/*"]
  },
  {
    "js": ["something_else.js"],
    "matches": ["http://*.example.org/*"]
  }
]
