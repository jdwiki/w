Com o lançamento do Firefox 48, a assinatura de extensão não pode mais ser desabilitado na versão e na versão beta do canal usando uma preferência. Conforme descrito quando a assinatura da extensão foi anunciada, estamos publicando compilações especializadas que suportam essa preferência para que os desenvolvedores possam continue testando em relação ao código que as compilações beta e de lançamento são gerado de.

WebExtensions estão atualmente em um estado alfa experimental. A partir de Firefox 46, você pode publicar WebExtensions para usuários do Firefox, assim como qualquer outro complemento. Estamos buscando uma primeira versão estável no Firefox 48.

Portabilidade através de WebExtensions #

Antes de falar sobre portar extensões do Firefox de/para, deve-se saber o que é WebExtensions.

WebExtensions - é uma plataforma que representa uma API para criar extensões do Firefox.

Ele usa a mesma arquitetura de extensão do Chromium, como resultado, essa API é compatível de várias maneiras com a API no Google Chrome e no Opera (Opera que é baseado no Chromium). Em muitos casos, as extensões desenvolvidas para esses navegadores funcionarão no Firefox com algumas alterações ou até mesmo sem elas.

MDN recomenda para usar WebExtension para novas extensões:

No futuro, WebExtensions será a forma recomendada para desenvolver Complementos do Firefox e outros sistemas serão descontinuados.

Em vista do exposto, se você deseja portar extensões para o Firefox, você precisa saber como a extensão foi escrita.

Extensões para Firefox podem ser baseadas em WebExtension, Add-on SDK ou XUL.

Extensões compatíveis baseadas em WebExtension #

Ao usar WebExtension, deve-se verificar a lista de incompatibilidades, pois algumas funções são suportadas total ou parcialmente, ou seja, deve-se verificar o manifest.json.

Também permite usar o mesmo namespace:

Neste momento, todas as APIs podem ser acessadas por meio do namespace chrome.. Quando começamos a adicionar nossas próprias APIs, esperamos adicioná-las ao browser. namespace. Os desenvolvedores poderão usar a detecção de recursos para determinar se uma API está disponível no browser.*.

Uma extensão simples que pode funcionar no Firefox e no Google Chrome

manifest.json:

{
  "manifest_version": 2,
  
  "name": "StackMirror",
  
  "version": "1.0",
  
  "description": "Mirror reflection of StackOverflow sites",
  
  "icons": {
    "48": "icon/myIcon-48.png"
  },
  
  "page_action": {
    "default_icon": "icon/myIcon-48.png"
  },
  
  "background": {
    "scripts"   : ["js/background/script.js"],
    "persistent": false
  },
  
  "permissions": ["tabs", "*://*.stackoverflow.com/*"]
}

script fundo:

function startScript(tabId, changeInfo, tab) {

    if (tab.url.indexOf("stackoverflow.com") > -1) {

        chrome.tabs.executeScript(tabId, 
            
            {code: 'document.body.style.transform = "scaleX(-1)";'}, function () {

            if (!chrome.runtime.lastError) {

                chrome.pageAction.show(tabId);
            }
        });
    }
}

chrome.tabs.onUpdated.addListener(startScript);

Pacote de projeto como arquivo zip padrão, mas com extensões .xpi.

Então, você deve carregar a extensão no Firefox.

Abra a página about:addons, acessível através de Menu > Add-ons.

Clique no botão Ferramentas para todos os complementos.

As instruções sobre como carregar a extensão no Google Chrome estão em outro tópico - [Introdução às extensões do Chrome][7].
