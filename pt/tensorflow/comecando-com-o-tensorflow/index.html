Exemplo básico #

O Tensorflow é mais do que apenas uma estrutura de aprendizado profundo. É uma estrutura de computação geral para realizar operações matemáticas gerais de maneira paralela e distribuída. Um exemplo de tal é descrito abaixo.

Regressão linear #

Um exemplo estatístico básico que é comumente utilizado e bastante simples de calcular é ajustar uma linha a um conjunto de dados. O método para fazer isso no tensorflow é descrito abaixo no código e nos comentários.

As principais etapas do script (TensorFlow) são:

  1. Declare espaços reservados (x_ph, y_ph) e variáveis (W, b)
  2. Defina o operador de inicialização (init)
  3. Declare operações nos marcadores e variáveis ​​(y_pred, loss, train_op)
  4. Crie uma sessão (sess)
  5. Execute o operador de inicialização (sess.run(init))
  6. Execute algumas operações gráficas (por exemplo, sess.run([train_op, loss], feed_dict={x_ph: x, y_ph: y}))

A construção do gráfico é feita usando a API Python TensorFlow (também pode ser feita usando a API C++ TensorFlow). A execução do gráfico chamará rotinas C++ de baixo nível.

'''
function: create a linear model which try to fit the line 
          y = x + 2 using SGD optimizer to minimize 
          root-mean-square(RMS) loss function

'''
import tensorflow as tf
import numpy as np

# number of epoch
num_epoch = 100

# training data x and label y
x = np.array([0., 1., 2., 3.], dtype=np.float32)
y = np.array([2., 3., 4., 5.], dtype=np.float32)

# convert x and y to 4x1 matrix
x = np.reshape(x, [4, 1])
y = np.reshape(y, [4, 1])

# test set(using a little trick)
x_test = x + 0.5
y_test = y + 0.5

# This part of the script builds the TensorFlow graph using the Python API

# First declare placeholders for input x and label y
# Placeholders are TensorFlow variables requiring to be explicitly fed by some 
# input data
x_ph = tf.placeholder(tf.float32, shape=[None, 1])
y_ph = tf.placeholder(tf.float32, shape=[None, 1])

# Variables (if not specified) will be learnt as the GradientDescentOptimizer
# is run
# Declare weight variable initialized using a truncated_normal law
W = tf.Variable(tf.truncated_normal([1, 1], stddev=0.1))
# Declare bias variable initialized to a constant 0.1
b = tf.Variable(tf.constant(0.1, shape=[1]))

# Initialize variables just declared 
init = tf.initialize_all_variables()

# In this part of the script, we build operators storing operations
# on the previous variables and placeholders.
# model: y = w * x + b
y_pred = x_ph * W + b

# loss function
loss = tf.mul(tf.reduce_mean(tf.square(tf.sub(y_pred, y_ph))), 1. / 2)
# create training graph
train_op = tf.train.GradientDescentOptimizer(0.1).minimize(loss)

# This part of the script runs the TensorFlow graph (variables and operations
# operators) just built.
with tf.Session() as sess:
    # initialize all the variables by running the initializer operator
    sess.run(init)
    for epoch in xrange(num_epoch):
        # Run sequentially the train_op and loss operators with
        # x_ph and y_ph placeholders fed by variables x and y
        _, loss_val = sess.run([train_op, loss], feed_dict={x_ph: x, y_ph: y})
        print('epoch %d: loss is %.4f' % (epoch, loss_val))

    # see what model do in the test set
    # by evaluating the y_pred operator using the x_test data
    test_val = sess.run(y_pred, feed_dict={x_ph: x_test})
    print('ground truth y is: %s' % y_test.flatten())
    print('predict y is     : %s' % test_val.flatten())

Contando até 10 #

Neste exemplo, usamos o Tensorflow para contar até 10. Sim isso é um exagero total, mas é um bom exemplo para mostrar uma configuração mínima absoluta necessária para usar o Tensorflow

import tensorflow as tf

# create a variable, refer to it as 'state' and set it to 0
state = tf.Variable(0)

# set one to a constant set to 1
one = tf.constant(1)

# update phase adds state and one and then assigns to state
addition = tf.add(state, one)
update = tf.assign(state, addition )

# create a session
with tf.Session() as sess:
  # initialize session variables
  sess.run( tf.global_variables_initializer() )

  print "The starting state is",sess.run(state)

  print "Run the update 10 times..."
  for count in range(10):
    # execute the update
    sess.run(update)

  print "The end state is",sess.run(state)

O importante a perceber aqui é que estado, um, adição e atualização na verdade não contêm valores. Em vez disso, são referências a objetos do Tensorflow. O resultado final não é state, mas é recuperado usando um Tensorflow para avaliá-lo usando sess.run(state)

Este exemplo é de https://github.com/panchishin/learn-to-tensorflow . Existem vários outros exemplos e um bom plano de aprendizado graduado para se familiarizar com a manipulação do gráfico do Tensorflow em python.

Instalação ou configuração #

A partir da instalação do Tensorflow versão 1.0 ficou muito mais fácil de executar. No mínimo, para instalar o TensorFlow, é necessário o pip instalado em sua máquina com uma versão python de pelo menos 2.7 ou 3.3+.

pip install --upgrade tensorflow      # for Python 2.7
pip3 install --upgrade tensorflow     # for Python 3.n

Para tensorflow em uma máquina GPU (a partir de 1.0 requer CUDA 8.0 e cudnn 5.1, GPU AMD não suportada)

pip install --upgrade tensorflow-gpu  # for Python 2.7 and GPU
pip3 install --upgrade tensorflow-gpu # for Python 3.n and GPU

Para testar se funcionou abra a versão correta do python 2 ou 3 e execute

import tensorflow

Se isso for bem-sucedido sem erros, você terá o tensorflow instalado em sua máquina.

Fundamentos do Tensorflow #

O Tensorflow funciona com base no princípio dos gráficos de fluxo de dados. Para realizar algum cálculo, existem duas etapas:

  1. Represente o cálculo como um gráfico.
  2. Execute o gráfico.

Representação: Como qualquer grafo direcionado, um grafo Tensorflow consiste em nós e arestas direcionais.

Node: Um Node também é chamado de Op (significa operação). Um nó pode ter várias arestas de entrada, mas uma única aresta de saída.

Edge: Indica dados de entrada ou saída de um nó. Neste caso entrada(s) e saída de algum Node(Op).

Sempre que dizemos dados, queremos dizer um vetor n-dimensional conhecido como Tensor. Um tensor tem três propriedades: Classificação, Forma e Tipo

  • Classificação significa número de dimensões do Tensor (um cubo ou caixa tem classificação 3).
  • Forma significa valores dessas dimensões (a caixa pode ter a forma 1x1x1 ou 2x5x7).
  • Tipo significa tipo de dados em cada coordenada do Tensor.

Execução: Mesmo que um grafo seja construído, ele ainda é uma entidade abstrata. Nenhuma computação realmente ocorre até que a executemos. Para executar um gráfico, precisamos alocar recursos de CPU para Ops dentro do gráfico. Isso é feito usando sessões do Tensorflow. As etapas são:

  1. Crie uma nova sessão.
  2. Execute qualquer Op dentro do Gráfico. Normalmente executamos o Op final onde esperamos a saída do nosso cálculo.

Uma borda de entrada em um Op é como uma dependência de dados em outro Op. Assim, quando executamos qualquer Op, todas as arestas de entrada nele são rastreadas e as operações do outro lado também são executadas.

Observação: também são possíveis nós especiais chamados de papel de fonte de dados ou coletor. Por exemplo, você pode ter um Op que fornece um valor constante, portanto, sem arestas de entrada (consulte o valor ‘matrix1’ no exemplo abaixo) e similarmente Op sem arestas de saída onde os resultados são coletados (consulte o valor ‘product’ no exemplo abaixo).

Exemplo:

import tensorflow as tf

# Create a Constant op that produces a 1x2 matrix.  The op is
# added as a node to the default graph.
#
# The value returned by the constructor represents the output
# of the Constant op.
matrix1 = tf.constant([[3., 3.]])

# Create another Constant that produces a 2x1 matrix.
matrix2 = tf.constant([[2.],[2.]])

# Create a Matmul op that takes 'matrix1' and 'matrix2' as inputs.
# The returned value, 'product', represents the result of the matrix
# multiplication.
product = tf.matmul(matrix1, matrix2)

# Launch the default graph.
sess = tf.Session()

# To run the matmul op we call the session 'run()' method, passing 'product'
# which represents the output of the matmul op.  This indicates to the call
# that we want to get the output of the matmul op back.
#
# All inputs needed by the op are run automatically by the session.  They
# typically are run in parallel.
#
# The call 'run(product)' thus causes the execution of three ops in the
# graph: the two constants and matmul.
#
# The output of the op is returned in 'result' as a numpy `ndarray` object.
result = sess.run(product)
print(result)
# ==> [[ 12.]]

# Close the Session when we're done.
sess.close()
