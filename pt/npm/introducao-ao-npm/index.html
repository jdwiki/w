Instalação ou Configuração #

Instalar #

O npm vem junto com o Node.js, então se você instalar o Node.js você também terá o npm instalado automaticamente. Você pode escolher entre uma versão Atual e uma LTS

Janelas #

Para o Microsoft Windows, você pode baixar um instalador MSI em https://nodejs.org/en/download/.

REMOÇÃO DO SO #

Para o Apple OS X, você pode baixar um instalador PKG do mesmo local https://nodejs.org/en/download/.

###Linux Para Linux, você pode usar seu gerenciador de pacotes para instalar Node.js e npm.

Você também pode compilar o Node.js da fonte e ainda obterá npm. Há também um script que você pode executar que instalará o npm:

curl -L https://www.npmjs.com/install.sh | sh

Atualize o npm para a versão mais recente #

O método recomendado de atualizar sua instalação do npm é simplesmente fazer com que o próprio npm instale:

npm install -g npm@latest

Como alternativa, você pode atualizar para a versão LTS atual em vez da versão mais recente:

npm install -g npm@lts

Você também pode instalar qualquer versão do Node (e npm) com [nvm](https://www.wikiod.com/pt/node-js/instalando-o-nodejs#Usando o gerenciador de versões do nó (nvm) -nvm#t=201611281916227729431). Ao instalar globalmente com npm com uma instalação nvm, você não precisa usar sudo (ou Executar como Administrador no Windows).

Instalando Pacotes Globais #

Instalar um pacote global

Pacotes instalados globalmente descartam módulos em {prefix}/lib/node_modules, e colocam arquivos executáveis ​​em {prefix}/bin, onde {prefix} geralmente é algo como /usr/local. Instalar um módulo global significa que seus binários terminam em sua variável de ambiente PATH. Normalmente, você deseja instalar um módulo global se for uma ferramenta de linha de comando ou algo que deseja usar em seu shell.

npm install --global package-name

Remover um pacote global

npm uninstall --global package-name

Nota: o argumento --global pode ser simplificado para -g. Assim, por exemplo, o primeiro comando poderia ter sido npm install -g package-name, com exatamente o mesmo resultado.

Nota: em sistemas *nix, a instalação de pacotes globais pode exigir permissões de superusuário. Não fazer isso irá falhar com: EACCES. Nesse caso, execute:

sudo npm install --global package-name

Instalar pacotes #

Observe que os pacotes podem ser instalados Este comando instala a versão mais recente disponível dos pacotes nomeados:

tanto local como globalmente.

Instalação local significa que npm instala seu pacote no diretório de trabalho atual. Módulos de nós vão em ./node_modules, executáveis ​​vão em ./node_modules/.bin/. Normalmente você vai querer instalar módulos locais para uso dentro do seu programa, como uma dependência, e eles funcionarão apenas onde estão instalados.

npm install <package names>

Forma abreviada:

npm i <package names>

npm pode interagir com um arquivo package.json no diretório atual de várias maneiras úteis, através dos objetos dependencies e devDependencies armazenados em package.json (instalando vários módulos):

O comando npm install sem parâmetros

npm install

instala todos os pacotes nomeados como chaves de objeto nos objetos dependencies e devDependencies em package.json, usando restrições de versão semântica conforme indicado pelos valores do objeto.

Ao desenvolver um novo software:

Use a opção -S para anexar os <nomes dos pacotes> e as versões dos módulos npm que você está instalando que devem sempre ser incluídos no seu módulo. Anexa à lista de dependencies rastreadas no arquivo package.json, após a instalação.

npm i <package names> -S

Use a opção -D para anexar os <nomes dos pacotes> e as versões dos módulos npm que você está instalando que são necessários por outros desenvolvedores para desenvolver ou testar seu módulo. Acrescenta à lista de devDependencies rastreados no arquivo package.json, após a instalação.

npm i <package names> -D

Onde lodash e mocha são nomes de pacotes.

Atualizando pacotes #

Em cada ciclo de vida de aplicativos chega um dia em que seus componentes precisam ser atualizados. Todo mundo conhece a dor de atualizar todas as dependências uma a uma. Bem, aqui você só precisa emitir o comando:

npm update (-g)

Se o “-g” estiver lá, o npm atualizará os pacotes globais.

Usando npm para gerenciar dependências #

Então você quer implantar seu aplicativo em vários sites? e seu projeto tem muitas dependências para instalá-los um por um? O Npm tem uma solução, basta emitir o seguinte comando:

npm init

Na pasta raiz do projeto, siga as instruções na tela (digite o valor desejado e pressione enter) e, se quiser salvar uma dependência, adicione:

--save

depois do seu

npm install

comandos por exemplo:

npm install mypackagename --save

E então essa dependência será salva, então você não precisa mover a pasta “node_modules”. Para instalar todos os problemas de dependência salvos:

npm install

e todas as dependências salvas serão instaladas.
