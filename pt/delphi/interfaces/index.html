As interfaces são usadas para descrever as informações necessárias e a saída esperada de métodos e classes, sem fornecer informações da implementação explícita.

As classes podem implementar interfaces, e as interfaces podem herdar umas das outras. Se uma classe estiver implementando uma interface, isso significa que todas as funções e procedimentos expostos pela interface existem na classe.

Um aspecto especial das interfaces no delphi é que as instâncias de interfaces têm um gerenciamento de tempo de vida baseado na contagem de referências. O tempo de vida das instâncias de classe deve ser gerenciado manualmente.

Considerando todos esses aspectos, as interfaces podem ser utilizadas para atingir diferentes objetivos:

  • Fornecer várias implementações diferentes para operações (por exemplo, salvar em um arquivo, banco de dados ou enviar como E-Mail, tudo como Interface “SaveData”)
  • Reduzir dependências, melhorando o desacoplamento e assim tornando o código melhor manutenível e testável
  • Trabalhe com instâncias em várias unidades sem se incomodar com o gerenciamento da vida útil (embora mesmo aqui existam armadilhas, cuidado!)

Definindo e implementando uma interface #

Uma interface é declarada como uma classe, mas sem modificadores de acesso (public, private, …). Além disso, nenhuma definição é permitida, portanto, variáveis ​​e constantes não podem ser usadas.

As interfaces devem sempre ter um Identificador Único, que pode ser gerado pressionando Ctrl + Shift + G.

IRepository = interface
    ['{AFCFCE96-2EC2-4AE4-8E23-D4C4FF6BBD01}']
    function  SaveKeyValuePair(aKey: Integer; aValue: string): Boolean;
end;

Para implementar uma interface, o nome da interface deve ser adicionado atrás da classe base. Além disso, a classe deve ser descendente de TInterfacedObject (isso é importante para o gerenciamento da vida útil).

TDatabaseRepository = class(TInterfacedObject, IRepository)
    function  SaveKeyValuePair(aKey: Integer; aValue: string): Boolean;
end;

Quando uma classe implementa uma interface, ela deve incluir todos os métodos e funções declarados na interface, caso contrário ela não será compilada.

Uma coisa que vale a pena notar é que os modificadores de acesso não têm nenhuma influência, se o chamador trabalha com a interface. Por exemplo, todas as funções da interface podem ser implementadas como membros strict private, mas ainda podem ser chamadas de outra classe se uma instância da interface for usada.

Propriedades em interfaces #

Como a declaração de variáveis ​​em interfaces não é possível, a forma “rápida” de definir as propriedades (valor da propriedade: TObject read FValue write FValue;) não pode ser utilizada. Em vez disso, o Getter e o setter (cada um somente se necessário) também precisam ser declarados na interface.

IInterface = interface(IInterface)
    ['{6C47FF48-3943-4B53-8D5D-537F4A0DEC0D}']
    procedure SetValue(const aValue: TObject);
    function  GetValue(): TObject;

    property Value: TObject read GetValue write SetValue;
end;

Uma coisa que vale a pena notar é que a classe de implementação não precisa declarar a propriedade. O compilador aceitaria este código:

TImplementer = class(TInterfacedObject, IInterface)
    procedure SetValue(const aValue: TObject);
    function  GetValue(): TObject
end;

Uma ressalva, no entanto, é que dessa forma a propriedade só pode ser acessada por meio de uma instância da interface, e não pela própria classe. Além disso, adicionar a propriedade à classe aumenta a legibilidade.

Implementando várias interfaces #

As classes podem implementar mais de uma interface, ao contrário de herdar de mais de uma classe (Multiple Inheritance), o que não é possível para classes Delphi. Para conseguir isso, o nome de todas as interfaces deve ser adicionado separado por vírgulas atrás da classe base.

É claro que a classe implementadora também deve definir as funções declaradas por cada uma das interfaces.

IInterface1 = interface
    ['{A2437023-7606-4551-8D5A-1709212254AF}']
    procedure Method1();
    function Method2(): Boolean;
end;

IInterface2 = interface
    ['{6C47FF48-3943-4B53-8D5D-537F4A0DEC0D}']
    procedure SetValue(const aValue: TObject);
    function  GetValue(): TObject;

    property Value: TObject read GetValue write SetValue;
end;

TImplementer = class(TInterfacedObject, IInterface1, IInterface2)
    // IInterface1
    procedure Method1();
    function Method2(): Boolean;

    // IInterface2
    procedure SetValue(const aValue: TObject);
    function  GetValue(): TObject

    property Value: TObject read GetValue write SetValue;
end;

Herança para interfaces #

As interfaces podem herdar umas das outras, exatamente como as classes também. Uma classe de implementação, portanto, deve implementar funções da interface e de todas as interfaces básicas. Desta forma, porém, o compilador não sabe que a classe implementadora também implementa a interface base, ele apenas conhece as interfaces que estão explicitamente listadas. É por isso que usar as ISuperInterface no TImplementer não funcionaria. Isso também resulta na prática comum, para implementar explicitamente todas as interfaces base também (neste caso TImplementer = class(TInterfacedObject, IDescendantInterface, ISuperInterface)).

ISuperInterface = interface
    ['{A2437023-7606-4551-8D5A-1709212254AF}']
    procedure Method1();
    function Method2(): Boolean;
end;

IDescendantInterface = interface(ISuperInterface)
    ['{6C47FF48-3943-4B53-8D5D-537F4A0DEC0D}']
    procedure SetValue(const aValue: TObject);
    function  GetValue(): TObject;

    property Value: TObject read GetValue write SetValue;
end;

TImplementer = class(TInterfacedObject, IDescendantInterface)
    // ISuperInterface
    procedure Method1();
    function Method2(): Boolean;

    // IDescendantInterface
    procedure SetValue(const aValue: TObject);
    function  GetValue(): TObject

    property Value: TObject read GetValue write SetValue;
end;
