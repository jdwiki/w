Instalação do Drupal 7 em etapas #

Etapa 1: baixar e extrair o Drupal

Drupal está disponível em duas versões suportadas: a mais recente e a anterior. Atualmente, é Drupal 8 (lançado em novembro de 2015) e Drupal 7. As versões recomendadas são as versões estáveis ​​mais recentes de qualquer versão. Para saber mais sobre as versões, consulte a página Informações sobre a versão do Drupal.

Para instalar um site Drupal em um idioma diferente do inglês, consulte http://localize.drupal.org/translate. Você também pode adicionar idiomas adicionais após a instalação.

Observação: Se você instalar o Drupal 8, poderá escolher o idioma de instalação como a primeira opção na interface do próprio script de instalação.

Você pode baixar e extrair o Drupal de seis maneiras diferentes:

  • usando Drush
  • usando o console do Drupal
  • a partir da linha de comando
  • usando FTP
  • usando Git
  • usando o compositor

Então, siga um dos métodos abaixo para fazer o download e depois…

Antes de continuar para a próxima página…

A URL base para sua instalação do Drupal é definida no arquivo de configuração do seu servidor web. Você precisa conhecer esse URL antes de prosseguir para as próximas etapas da instalação. Se você estiver instalando o Drupal em sua máquina local, a URL base pode ser http://localhost. Se você estiver instalando o Drupal em um servidor web, sua URL base pode ser um nome de domínio específico, como http://example.com.

Uvas

Drush é uma ferramenta de linha de comando para manter e administrar sites Drupal. Ele oferece a maneira mais conveniente de baixar o Drupal usando um único comando: pm-download (ou seu alias dl):

drush dl drupal

Este comando baixará a versão recomendada na pasta atual. Verifique o drush help dl para ver opções adicionais, como baixar uma versão específica.

Console Drupal

Drupal Console é a nova interface de linha de comando (CLI) para Drupal. O Drupal Console é uma ferramenta para gerar código clichê, interagir e depurar o Drupal 8. Ele oferece uma maneira conveniente de baixar o Drupal usando um único comando:

site:novo:

# specifying to download version 8.0.2 into folder "mydrupalsite"
drupal site:new mydrupalsite 8.0.2 

# select version from interactive mode
drupal site:new mydrupalsite

Este comando baixará a versão especificada na pasta atual. Verifique o drupal site:new --help ou visite a documentação.

A partir da linha de comando

Baixando

Faça login em seu servidor usando ssh e navegue até o diretório do qual você servirá seu site Drupal. Em muitos computadores *nix o caminho da raiz do servidor será /var/www/html, então cd /var/www/html. Em um servidor compartilhado ou em um servidor que hospeda vários domínios, o caminho será diferente (tente cd ~/www ou cd ~/public_html). Se você não tiver certeza do diretório, peça ajuda ao seu provedor de hospedagem.

Baixe o Drupal na linha de comando, por exemplo, usando wget ou curl. Os comandos são, respectivamente:

wget http://ftp.drupal.org/files/projects/drupal-x.x.tar.gz

ou

curl -O http://ftp.drupal.org/files/projects/drupal-x.x.tar.gz

Observação: A opção de comando curl é a letra maiúscula “O” e não o número que representa zero.

Substitua http://ftp.drupal.org/files/projects/drupal-x.x.tar.gz pelo link da versão que deseja instalar.

Os links para as versões recomendadas estão disponíveis na página do projeto Drupal Core, onde você pode copiá-los da coluna Download. Todas as outras versões estão disponíveis na página Releases for Drupal core.

Extraindo arquivos

Digite o seguinte comando e substitua “x.x” pelo número da versão baixada:

tar -xvzf drupal-x.x.tar.gz

Em seguida, remova a versão compactada do arquivo usando o seguinte comando:

rm drupal-x.x.tar.gz

Movendo-se para o local pretendido

Agora você precisa mover o conteúdo do diretório drupal-x.x um nível “para cima” para a raiz de documentos do servidor web ou seu diretório HTML público:

mv drupal-x.x/* drupal-x.x/.htaccess ./

Para Drupal 7, adicione também:

mv drupal-x.x/.gitignore ./

O Drupal 8 vem com vários arquivos ocultos adicionais que também precisam ser movidos.

Alternativamente, você pode extrair o arquivo tar diretamente no diretório correto digitando:

tar --strip-components=1 -xvzf drupal-x.x.tar.gz

Os arquivos do diretório que você baixou e descompactou agora foram movidos um nível para o seu diretório da web e você pode excluir o diretório (agora vazio) drupal-x.x:

rmdir drupal-x.x

Usando FTP

Você pode baixar o Drupal usando sua ferramenta FTP favorita.

  1. Baixe um arquivo tar.gz ou zip do Drupal para o seu computador local de https://www.drupal.org/project/drupal by clicking on the link for the version you want to install.
  2. Ao baixar o arquivo, seu navegador perguntará o que fazer with it. Choose “Extract” and extract it to your local computer. Or, save the file and extract it using your computer’s software (7-zip for example) that deals with archive files. The exact steps to do this differ by software, but you should end up with a folder/directory called something like “drupal-7.32” on your local computer.
  3. Use o FTP para transferir todo o conteúdo desta pasta, incluindo hidden files like .htaccess, to your hosting account’s HTML document root. Details of how to do this depend on your FTP software.

Observação: Drupal 8 inclui cerca de 10 vezes mais arquivos do que as versões anteriores (devido ao seu novo framework), que pode levar muito mais tempo para carregar em um servidor por FTP do que as versões anteriores. Uma maneira mais rápida de fazer upload do Drupal 8 como um único arquivo ZIP, adequado para alguns hosts, é descrita aqui.

Usando Git

A página do projeto Drupal tem ótimas instruções de controle de versão, que começa com a configuração do repositório pela primeira vez clonando o Drupal.

Ao clonar o Drupal 8.1.x ou superior, você precisará usar o composer para instalar as dependências mais recentes. A partir do diretório raiz do repositório Drupal que você clonou, execute composer install, isso fará o download de todas as dependências para o local esperado pelo Drupal. Veja também este método para construir sites com composer.

Se você não tiver o compositor instalado, siga a [documentação] 12.

Etapa 2: crie o banco de dados

Observação: Desde o 8.x, não é necessário criar um banco de dados antes de instalar o Drupal. Agora, se você inserir as credenciais de um usuário capaz de criar bancos de dados (por exemplo, o privilégio ‘CREATE’ no MySQL/MariaDB ou o privilégio ‘CREATEDB’ no postgresql), o nome do banco de dados especificado será criado no momento da instalação do Drupal se já não existe. Consulte o aviso de alteração relevante para obter mais informações e capturas de tela. Se você tiver um usuário com esses privilégios, poderá passar para a próxima etapa.

Antes de executar o script de instalação, você deve criar um banco de dados vazio e um usuário de banco de dados (um nome de usuário atribuído aos direitos para usar o banco de dados Drupal).

Drupal 7

Se você usar letras maiúsculas no nome do banco de dados, elas serão convertidas em minúsculas. Um “-” (hífen) no nome do banco de dados será codificado como “@002d” no nome da pasta do banco de dados. Por exemplo, o nome da pasta para o banco de dados a-b-c será a@002db@002dc. Outros caracteres além de a..z, 0..9 e “_” (sublinhado) são codificados de forma semelhante. [referência, referência]

Drupal 8

Se você usar letras maiúsculas no nome do banco de dados, elas serão convertidas em minúsculas. Os caracteres permitidos são a..z, 0..9 e “_” (sublinhado).

Etapa 3: Crie settings.php e o diretório de arquivos

Visão geral do arquivo de configuração do Drupal “settings.php” e “services.yml”

Para que o Drupal funcione, você precisa configurar onde está o banco de dados, como o banco de dados é chamado e as credenciais do banco de dados para acessar o banco de dados. Essas informações são armazenadas no arquivo settings.php que está localizado em:

sites/default

O arquivo settings.php é comum ao Drupal 6, 7 e 8

Quando você extrai o Drupal pela primeira vez, ele não vem com um arquivo settings.php, mas sim com default.settings.php. Quando você instalar o Drupal 7 pela primeira vez, ele tentará copiar e renomear default.settings.php -> settings.php para você. Existem alguns casos raros em que você precisará fazer isso manualmente, que são abordados em detalhes mais abaixo nesta página.

Novo no Drupal 8 na pasta sites/default, é um arquivo chamado default.services.yml. Assim como default.settings.php, default.services.yml precisa ser renomeado para funcionar. No entanto, esse arquivo foi projetado para substituir o arquivo principal services.yml se você precisar substituí-lo e 99% dos sites existentes não precisarão substituir o arquivo principal services.yml. Ele é disponibilizado se você precisar substituir essas configurações. No início do desenvolvimento, esse arquivo foi copiado e renomeado automaticamente durante a instalação, no entanto, parar de criar services.yml por padrão substitui o método inicial. Em outras palavras, nunca se preocupe com default.services.yml / services.yml, a menos que algo diga o contrário.

Finalmente, o propósito de ter default.[config-file].php é para que você possa atualizar facilmente o Drupal, sem sobrescrever toda a configuração que roda seu site. Sim, houve um tempo em que isso aconteceu…

Visão geral das configurações automáticas.php

Por padrão, Drupal 7 e 8 tentarão criar e preencher o arquivo settings.php automaticamente quando você usar install.php para configurar o site. O script também altera a permissão no arquivo para protegê-lo assim que for concluído e, em seguida, cria um diretório sites/default/files para hospedar todos os seus arquivos não essenciais. Infelizmente, alguns tipos de hospedagem compartilhada/local são configurados para que PHP e Apache sejam executados como o mesmo usuário. Isso pode resultar na falha do script de instalação ao executar a criação e o preenchimento do arquivo settings.php, juntamente com a configuração de permissões e a criação do diretório de arquivos. Se você receber erros referentes ao arquivo de configurações durante a instalação, você terá que criar manualmente o arquivo settings.php e fazer mais algumas tarefas antes de executar install.php. Depois de criado com permissões de gravação, o script de instalação preencherá automaticamente as informações apropriadas para a configuração do seu site. Depois, você terá que proteger novamente o arquivo settings.php.

Neste ponto, vá para a próxima página etapa: Etapa 4: Execute o script de instalação. Se você tiver problemas com a instalação devido às Configurações, volte aqui e siga as etapas do Manual descritas abaixo.

Manual settings.php Visão geral

Drupal 6, 7 e 8 vêm com um arquivo de configuração settings.php de amostra localizado em:

sites/default/default.settings.php

Antes de executar o script de instalação (install.php), você precisa copiar o arquivo default.settings.php como um novo arquivo chamado settings.php e alterar suas permissões para serem graváveis. Após a instalação, você precisará restringir as permissões novamente.

Manual settings.php Instruções detalhadas

  • Etapa 1 - Navegação e Criação

    Navegue até sites/default da sua instalação raiz do Drupal.
    Copie o arquivo default.settings.php e salve o novo arquivo como settings.php no mesmo diretório (veja a nota abaixo sobre renomeação). Se você tiver acesso ao shell (linha de comando), execute o seguinte comando no diretório que contém seus arquivos de instalação do Drupal:

    cp sites/default/default.settings.php sites/default/settings.php

    Observação: Não basta renomear o arquivo. O instalador do Drupal precisa dos dois arquivos.

    Se você tiver apenas acesso FTP, terá que baixar o arquivo para o seu computador, renomeá-lo e enviá-lo. Alguns provedores de hospedagem têm um gerenciador de arquivos no painel onde o arquivo pode ser copiado e renomeado.

  • Etapa 2 - Verifique se as permissões são graváveis
    Por padrão, o diretório sites/default e o arquivo settings.php devem ser graváveis. Você pode verificar se as permissões de sites/default e settings.php são graváveis ​​emitindo os seguintes comandos:

    ls -l sites/

    A permissão em sites/padrão deve ser 755 [drwxr-xr-x]:

    ls -l sites/default/settings.php

    A permissão em settings.php deve ser 644 [-rw-r–r–]:
    Se eles não forem graváveis, você pode emitir os seguintes comandos:

    chmod 644 sites/default/settings.php

    Observação: Se você estiver no mesmo grupo que o usuário da web, alterar as permissões para 664 será suficiente.
    Várias ferramentas de FTP como Filezilla, Transmit e Fetch etc. permitem que você altere as permissões do arquivo, usando um ‘file attribute’ ou ‘get info’ comando. Nesse caso, a permissão do arquivo deve ser definida como 644. Se o seu cliente FTP tiver caixas de seleção para definir permissões, marque as caixas Ler e Gravar para “Proprietário”, “Grupo” e “Outros” (mas deixe as caixas Executar desmarcadas) . Para algumas situações, você pode precisar de uma permissão de 664. Alguns provedores de hospedagem permitem uma operação semelhante por meio do gerenciador de arquivos do painel.

  • Etapa 3 - Experimente a instalação

    Neste ponto, experimente a instalação. Veja se você consegue passar pela instalação executando http://[seusite]/install.php. Se você for bem sucedido, a primeira página que você vai querer visitar é
    Relatórios -> Relatório de status (admin/relatórios/status)

    Na página de relatórios, procure uma linha que diz: Sistema de arquivos. Se disser algo diferente de “Writeable”, você precisará seguir a Etapa 4 abaixo.

    Em seguida, procure uma linha que diz: Arquivo de configuração. Se ele disser algo diferente de “Protegido”, você precisará proteger novamente os arquivos de configuração conforme descrito na Etapa 5 abaixo.

  • Etapa 4 - Crie o diretório de arquivos

    A instalação deveria ter criado o diretório sites/default/files para você, mas caso isso não aconteça, você precisará criá-lo manualmente e definir as permissões corretas nele.

    mkdir sites/default/files

    Nota: Na maioria dos sistemas Linux, um diretório recém-criado já está configurado com a permissão 755. Caso não seja, você pode emitir o comando:

    chmod 755 sites/default/files

    Isso define o diretório de arquivos para 755 [drwx-rw-rw].

    Dependendo de como sua configuração do apache está configurada, talvez seja necessário executar:

    chmod 777 sites/default/files

    Isso define o diretório de arquivos para 777 [drwxrwxrwx]. É menos seguro que 755, mas não há nada que você possa fazer se for assim que seu servidor estiver configurado.

  • Etapa 5 - Verificação de permissão pós-instalação

    Após a execução do script de instalação, o Drupal tenta definir as permissões automaticamente para:

    555 (ler-executar) [dr-xr-xr-x] para a pasta sites/default.
    e
    444 (somente leitura) [-r–r–r–] para o settings.php

    Se não, você precisará configurá-los manualmente:

    chmod 555 sites/default

    chmod 444 sites/default/settings.php

    Essas permissões estão corretas e não devem ser alteradas, pois alterá-las abre um risco de segurança.

Etapa 4: execute o script de instalação

Script de instalação do Drupal 7

Para executar o script de instalação do Drupal, aponte seu navegador para a URL base do seu site.

A URL base significa a raiz do documento (diretório) onde você colocou seus arquivos Drupal (e é definido no arquivo de configuração do servidor web). Se você instalou o Drupal em um host da web, provavelmente será um nome de domínio como http://www.example.com. Se você instalou o Drupal em uma subpasta, você deve apontar seu navegador para a subpasta (por exemplo, http://example.com/subfolder). Se você instalou o Drupal em sua máquina desktop, a URL pode ser http://localhost/drupal.

Se o processo de instalação não aparecer simplesmente digitando a URL base do seu site, adicione o nome do arquivo install.php ao final da URL do seu site (por exemplo, http://www.example.com/install.php ).

Processo de instalação

Depois de executar install.php, você será guiado por várias páginas:

  1. Escolha qual perfil usar para a instalação (padrão ou minimal or your chosen distribution). Most people should select the “standard” option. The standard option comes with default content types already enabled, such as Article and Page, and with appropriate publishing options already set. (Of course you can later edit these default content types and their settings, or add additional ones.) The standard profile also has a useful collection of modules pre-enabled for you.

    The “minimal” option is targeted toward more experienced Drupal site creators who wish to set up their own content types with associated publishing options. The minimal profile has only three modules enabled: Block, Database logging, and Update status.

    More info about built-in installation profiles.

    Choose the ’name of your distribution’, if you have downloaded a distribution and not a normal drupal at step one.



  2. Se você deseja instalar usando um idioma diferente do padrão English, click the Learn how to install Drupal in other languages link.



  3. Se seu diretório de instalação ainda não estiver configurado corretamente, you’ll be informed on this page. You can correct the settings individually and either refresh the browser screen or click ‘Try again’ to see whether there are any errors left.



    Os erros relatados podem incluir:

    Diretórios ausentes e/ou permissões incorretas
    O instalador tentará configurar automaticamente vários diretórios, mas isso pode falhar devido às configurações de permissão. Nesse caso, você encontrará os diretórios ausentes listados.
    sites/padrão/arquivos
    sites/padrão/privado
    sites/default/private/files

    Esses diretórios devem ser definidos com as seguintes permissões:

    chmod o+w sites/default/files

    OU

    chmod 777 sites/padrão/arquivos
    Settings.php ausentes ou permissões incorretas
    Se settings.php estiver ausente ou não estiver acessível, siga as instruções em Passo 3: Criar o arquivo settings.php.
    Observe que você precisará dos arquivos default.settings.php e settings.php.

  4. Digite o nome do banco de dados, o nome de usuário e a senha do database that you created in Step 2: Create the database. This username and password information allows Drupal to access your database so the install script can create tables. Note that this is not the username and password for administering Drupal; these will be created in the next step.



    As opções Avançadas permitem que você altere o host do banco de dados (’localhost’ geralmente é usado nesta entrada: wamp/bin/apache/Apache2.2.11/bin/php.ini como um exemplo do local em um computador Windows executando WAMP). Você também pode alterar a porta e o prefixo da tabela. Você só precisa alterar a porta se estiver usando um número de porta não padrão. O prefixo da tabela é útil se você estiver instalando várias instâncias de tabelas Drupal que compartilham o mesmo banco de dados.

    Clique em Salvar e continue na parte inferior da página.

  5. Uma barra de progresso aparecerá e exibirá notas do instalador regarding the progress of the installation. If no errors are encountered, the next page will automatically load in your browser.

  6. Após a conclusão do instalador, insira as informações para o primeiro user account (which will be automatically assigned full administration permissions) and provide basic website settings.

    In the Site name field enter the name you wish to use for the site. You can also edit it later through the administration interface.

    No Site e-mail address field, enter the e-mail address that will be used by Drupal when it sends out notifications such as registration information.

    In the Site maintenance account field, enter the Username, E-mail address, and password for the main administration account. Note that as of Drupal 7 there is a distinction between the main administration account that you set up on this page, and the “Administrator” site administrator user role that you will see when you visit the “Roles” and “Permissions” pages in the administration interface. The account you set up in the Site maintenance account section during installation is a super-user who has overall control over every aspect of the management and configuration of the site. (For those of you familiar with the account from earlier versions of Drupal, this will be http://www.example.com/user/1.)

    No campo Configurações do servidor, selecione o país padrão e o fuso horário padrão.

    No campo Notificações de atualização, deixe as duas caixas de seleção marcadas se desejar que o servidor Drupal o alerte quando forem necessárias atualizações. Frequentemente, as atualizações estão relacionadas a problemas de segurança e são importantes de serem executadas. No entanto, se você restringiu a conectividade com a Internet (por exemplo, se estiver atrás de um firewall corporativo), poderá deixar essas configurações desmarcadas e testá-las mais tarde.

    Clique em “Salvar e continuar”. Em caso de sucesso, você verá a tela completa de instalação do Drupal. Se houver alguma mensagem de erro, revise-a e corrija-a agora.

Proteja seu site

Depois que a instalação estiver completa, você vai querer mudar as permissões no arquivo settings.php de volta para que ele fique protegido:

chmod u=rw,o=r,a=r sites/default/settings.php

OU

chmod 644 sites/default/settings.php

Se você fizer alterações manuais no arquivo settings.php posteriormente, certifique-se de protegê-lo novamente após fazer suas modificações.

Importante: Deixar de remover as permissões de gravação para o arquivo settings.php é um risco de segurança.

Embora o local padrão para o arquivo settings.php seja sites/default/settings.php, ele pode estar em outro local se você usar a configuração multisite.
