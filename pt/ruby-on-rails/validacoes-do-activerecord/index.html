Validando o comprimento de um atributo #

class Person < ApplicationRecord
  validates :name, length: { minimum: 2 }
  validates :bio, length: { maximum: 500 }
  validates :password, length: { in: 6..20 }
  validates :registration_number, length: { is: 6 }
end

As opções de restrição de comprimento possíveis são:

  • :minimum - O atributo não pode ter menos que o comprimento especificado.
  • :maximum - O atributo não pode ter mais do que o comprimento especificado.
  • :in (ou :within) - O comprimento do atributo deve ser incluído em um determinado intervalo. O valor para esta opção deve ser um intervalo.
  • :is - O comprimento do atributo deve ser igual ao valor fornecido.

Validando a presença de um atributo #

Este auxiliar valida se os atributos especificados não estão vazios.

class Person < ApplicationRecord
  validates :name, presence: true
end

Person.create(name: "John").valid? # => true
Person.create(name: nil).valid? # => false

Você pode usar o auxiliar absence para validar que os atributos especificados estão ausentes. Ele usa o método present? para verificar valores nulos ou vazios.

class Person < ApplicationRecord
  validates :name, :login, :email, absence: true
end

Observação: Caso o atributo seja booleano, você não pode fazer uso da validação de presença usual (o atributo não seria válido para um valor falso). Você pode fazer isso usando uma validação de inclusão:

validates :attribute, inclusion: [true, false]

Validações personalizadas #

Você pode adicionar suas próprias validações adicionando novas classes herdadas de ActiveModel::Validator ou de ActiveModel::EachValidator. Ambos os métodos são semelhantes, mas funcionam de maneiras ligeiramente diferentes:

ActiveModel::Validator e validates_with #

Implemente o método validate que recebe um registro como argumento e realiza a validação nele. Em seguida, use validates_with com a classe no modelo.

# app/validators/starts_with_a_validator.rb
class StartsWithAValidator < ActiveModel::Validator
  def validate(record)
    unless record.name.starts_with? 'A'
      record.errors[:name] << 'Need a name starting with A please!'
    end
  end
end
 
class Person < ApplicationRecord
  validates_with StartsWithAValidator
end

ActiveModel::EachValidator e validate #

Se você preferir usar seu novo validador usando o método validate comum em um único parâmetro, crie uma classe herdando de ActiveModel::EachValidator e implemente o método validate_each que recebe três argumentos: record, attribute , e valor:

class EmailValidator < ActiveModel::EachValidator
  def validate_each(record, attribute, value)
    unless value =~ /\A([^@\s]+)@((?:[-a-z0-9]+\.)+[a-z]{2,})\z/i
      record.errors[attribute] << (options[:message] || 'is not an email')
    end
  end
end
 
class Person < ApplicationRecord
  validates :email, presence: true, email: true
end

Valida o formato de um atributo #

Valide se o valor de um atributo corresponde a uma expressão regular usando format e a opção with.

class User < ApplicationRecord
  validates :name, format: { with: /\A\w{6,10}\z/ }
end

Você também pode definir uma constante e definir seu valor para uma expressão regular e passá-la para a opção with:. Isso pode ser mais conveniente para expressões regulares realmente complexas

PHONE_REGEX = /\A\(\d{3}\)\d{3}-\d{4}\z/
validates :phone, format: { with: PHONE_REGEX }

A mensagem de erro padrão é é inválido. Isso pode ser alterado com a opção :message.

validates :bio, format: { with: /\A\D+\z/, message: "Numbers are not allowed" }

Validando a numericidade de um atributo #

Essa validação restringe a inserção apenas de valores numéricos.

class Player < ApplicationRecord
  validates :points, numericality: true
  validates :games_played, numericality: { only_integer: true }
end

Além de :only_integer, este auxiliar também aceita as seguintes opções para adicionar restrições a valores aceitáveis:

  • :greater_than - Especifica que o valor deve ser maior que o valor fornecido. A mensagem de erro padrão para esta opção é “deve ser maior que %{count}”.
  • :greater_than_or_equal_to - Especifica que o valor deve ser maior ou igual ao valor fornecido. A mensagem de erro padrão para esta opção é “deve ser maior ou igual a %{count}”.
  • :equal_to - Especifica que o valor deve ser igual ao valor fornecido. A mensagem de erro padrão para esta opção é “deve ser igual a %{count}”.
  • :less_than - Especifica que o valor deve ser menor que o valor fornecido. A mensagem de erro padrão para esta opção é “deve ser menor que %{count}”.
  • :less_than_or_equal_to - Especifica que o valor deve ser menor ou igual ao valor fornecido. A mensagem de erro padrão para esta opção é “deve ser menor ou igual a %{count}”.
  • :other_than - Especifica que o valor deve ser diferente do valor fornecido. A mensagem de erro padrão para esta opção é “deve ser diferente de %{count}”.
  • :odd - Especifica que o valor deve ser um número ímpar se definido como verdadeiro. A mensagem de erro padrão para esta opção é “deve ser ímpar”.
  • :even - Especifica que o valor deve ser um número par se definido como verdadeiro. A mensagem de erro padrão para esta opção é “deve ser par”.

Por padrão, a numericidade não permite valores nulos. Você pode usar a opção allow_nil: true para permitir isso.

Validação de agrupamento #

Valida a inclusão de um atributo #

Você pode verificar se um valor está incluído em um array usando o auxiliar inclusion:. A opção :in e seu alias, :within mostram o conjunto de valores aceitáveis.

class Country < ApplicationRecord
  validates :continent, inclusion: { in: %w(Africa Antartica Asia Australia
                                            Europe North America South America) }
end

Para verificar se um valor não está incluído em uma matriz, use o auxiliar exclusion:

class User < ApplicationRecord
  validates :name, exclusion: { in: %w(admin administrator owner) }
end

Validar a exclusividade de um atributo #

Esse auxiliar valida se o valor do atributo é exclusivo logo antes de o objeto ser salvo.

class Account < ApplicationRecord
  validates :email, uniqueness: true
end

Existe uma opção :scope que você pode usar para especificar um ou mais atributos que são usados ​​para limitar a verificação de exclusividade:

class Holiday < ApplicationRecord
  validates :name, uniqueness: { scope: :year,
    message: "should happen once per year" }
end

Há também uma opção :case_sensitive que você pode usar para definir se a restrição de exclusividade fará distinção entre maiúsculas e minúsculas ou não. Esta opção tem como padrão true.

class Person < ApplicationRecord
  validates :name, uniqueness: { case_sensitive: false }
end

Ignorando validações #

Use os métodos a seguir se desejar ignorar as validações. Esses métodos salvarão o objeto no banco de dados mesmo que seja inválido.

  • diminui!
  • decrement_counter
  • incrementar!
  • increment_counter
  • alternar!
  • toque
  • atualize tudo
  • update_attribute
  • atualização_coluna
  • update_columns
  • update_counters

Você também pode pular a validação enquanto salva passando validate como um argumento para save

User.save(validate: false)

Validação condicional #

Às vezes, você pode precisar validar o registro apenas sob certas condições.

class User < ApplicationRecord
  validates :name, presence: true, if: :admin? 

  def admin?
    conditional here that returns boolean value
  end
end

Se sua condicional for muito pequena, você pode usar um Proc:

class User < ApplicationRecord
  validates :first_name, presence: true, if: Proc.new { |user| user.last_name.blank? }
end

Para condicional negativo, você pode usar unless:

class User < ApplicationRecord
  validates :first_name, presence: true, unless: Proc.new { |user| user.last_name.present? }
end

Você também pode passar uma string, que será executada via instance_eval:

class User < ApplicationRecord
  validates :first_name, presence: true, if: 'last_name.blank?'
end

Confirmação do atributo #

Você deve usar isso quando tiver dois campos de texto que devem receber exatamente o mesmo conteúdo. Por exemplo, você pode querer confirmar um endereço de e-mail ou uma senha. Esta validação cria um atributo virtual cujo nome é o nome do campo que deve ser confirmado com _confirmation anexado.

class Person < ApplicationRecord
  validates :email, confirmation: true
end

Observação Esta verificação é realizada somente se email_confirmation não for nulo.

Para exigir confirmação, certifique-se de adicionar uma verificação de presença para o atributo de confirmação.

class Person < ApplicationRecord
  validates :email,      confirmation: true
  validates :email_confirmation, presence: true
end

Usando a opção :on #

A opção :on permite especificar quando a validação deve acontecer. O comportamento padrão para todos os auxiliares de validação integrados é ser executado ao salvar (tanto ao criar um novo registro quanto ao atualizá-lo).

class Person < ApplicationRecord
  # it will be possible to update email with a duplicated value
  validates :email, uniqueness: true, on: :create
 
  # it will be possible to create the record with a non-numerical age
  validates :age, numericality: true, on: :update
 
  # the default (validates on both create and update)
  validates :name, presence: true
end
