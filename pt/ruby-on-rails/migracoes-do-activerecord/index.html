Parâmetros #

Tipo de coluna Descrição
:chave_primária Chave primária
:string Tipo de dados de string mais curto. Permite a opção limit para o número máximo de caracteres.
:texto Maior quantidade de texto. Permite a opção limit para o número máximo de bytes.
:inteiro Inteiro. Permite a opção limit para o número máximo de bytes.
:grande Inteiro maior
:float Flutuar
:decimal Número decimal com precisão variável. Permite opções de precisão e escala.
:numérico Permite opções de precisão e escala.
:datahora Objeto DateTime para datas/horas.
:hora Objeto de tempo para tempos.
:data Objeto de data para datas.
:binário Dados binários. Permite a opção limit para o número máximo de bytes.
:boolean Booleano

  • A maioria dos arquivos de migração reside no diretório db/migrate/. Eles são identificados por um timestamp UTC no início do nome do arquivo: YYYYMMDDHHMMSS_create_products.rb.

  • O comando rails generate pode ser encurtado para rails g.

  • Se um :type não for passado para um campo, o padrão será uma string.

Adicionando várias colunas a uma tabela #

Para adicionar várias colunas a uma tabela, separe os pares field:type com espaços ao usar o comando rails generate migration.

A sintaxe geral é:

rails generate migration NAME [field[:type][:index] field[:type][:index]] [options]

Por exemplo, o seguinte adicionará os campos name, salary e email à tabela users:

rails generate migration AddDetailsToUsers name:string salary:decimal email:string

O que gera a seguinte migração:

class AddDetailsToUsers < ActiveRecord::Migration[5.0]
  def change
    add_column :users, :name, :string
    add_column :users, :salary, :decimal
    add_column :users, :email, :string
  end
end

Adicione uma coluna de referência a uma tabela #

Para adicionar uma referência a um team à tabela users, execute este comando:

$ rails generate migration AddTeamRefToUsers team:references

Isso gera a seguinte migração:

class AddTeamRefToUsers < ActiveRecord::Migration[5.0]
  def change
    add_reference :users, :team, foreign_key: true
  end
end

Essa migração criará uma coluna team_id na tabela users.

Se você deseja adicionar um index e uma foreign_key apropriados na coluna adicionada, altere o comando para rails generate migration AddTeamRefToUsers team:references:index. Isso gerará a seguinte migração:

class AddTeamRefToUsers < ActiveRecord::Migration
  def change
    add_reference :users, :team, index: true
    add_foreign_key :users, :teams
  end
end

Se você quiser nomear sua coluna de referência diferente do que o Rails gera automaticamente, adicione o seguinte à sua migração: (Ex.: Você pode querer chamar o User que criou o Post como Author na tabela Post )

class AddAuthorRefToPosts < ActiveRecord::Migration
  def change
    add_reference :posts, :author, references: :users, index: true
  end
end

Reverter migrações #

Para reverter a migração mais recente, revertendo o método change ou executando o método down. Comando de execução:

rake db:rollback

rails db:rollback

Reverter as últimas 3 migrações #

rake db:rollback STEP=3

rails db:rollback STEP=3

STEP fornece o número de migrações a serem revertidas.

Reverter todas as migrações #

rake db:rollback VERSION=0

rails db:rollback VERSION=0

Adicione uma nova coluna com um índice #

Para adicionar uma nova coluna indexada email à tabela users, execute o comando:

rails generate migration AddEmailToUsers email:string:index

Isso gerará a seguinte migração:

class AddEmailToUsers < ActiveRecord::Migration[5.0]
  def change
    add_column :users, :email, :string
    add_index :users, :email
  end
end

Executar migração específica #

Para executar uma migração específica para cima ou para baixo, use db:migrate:up ou db:migrate:down.

Até uma migração específica:

rake db:migrate:up VERSION=20090408054555

rails db:migrate:up VERSION=20090408054555

Abaixo uma migração específica:

rake db:migrate:down VERSION=20090408054555

rails db:migrate:down VERSION=20090408054555

O número da versão nos comandos acima é o prefixo numérico no nome do arquivo da migração. Por exemplo, para migrar para a migração 20160515085959_add_name_to_users.rb, você usaria 20160515085959 como o número da versão.

Refazer migrações #

Você pode reverter e migrar novamente usando o comando redo. Este é basicamente um atalho que combina tarefas rollback e migrate.

Comando de execução:

rake db:migrate:redo

rails db:migrate:redo

Você pode usar o parâmetro STEP para voltar mais de uma versão.

Por exemplo, para voltar 3 migrações:

rake db:migrate:redo STEP=3

rails db:migrate:redo STEP=3

Adicionar uma nova coluna a uma tabela #

Para adicionar uma nova coluna name à tabela users, execute o comando:

rails generate migration AddNameToUsers name

Isso gera a seguinte migração:

class AddNameToUsers < ActiveRecord::Migration[5.0]
  def change
    add_column :users, :name, :string
  end
end

Quando o nome da migração estiver no formato AddXXXToTABLE_NAME seguido por uma lista de colunas com tipos de dados, a migração gerada conterá as instruções add_column apropriadas.

Remover uma coluna existente de uma tabela #

Para remover a coluna name existente da tabela users, execute o comando:

rails generate migration RemoveNameFromUsers name:string

Isso gerará a seguinte migração:

class RemoveNameFromUsers < ActiveRecord::Migration[5.0]
  def change
    remove_column :users, :name, :string
  end
end

Quando o nome da migração estiver no formato RemoveXXXFromYYY seguido por uma lista de colunas com tipos de dados, a migração gerada conterá as instruções remove_column apropriadas.

Embora não seja necessário especificar o tipo de dados (por exemplo, :string) como parâmetro para remove_column, é altamente recomendado. Se o tipo de dados não for especificado, a migração não será reversível.

Executando migrações em diferentes ambientes #

Para executar migrações no ambiente test, execute este comando shell:

rake db:migrate RAILS_ENV=test

A partir do Rails 5.0, você pode usar rails em vez de rake:

rails db:migrate RAILS_ENV=test

Crie uma nova tabela #

Para criar uma nova tabela users com as colunas name e salary, execute o comando:

rails generate migration CreateUsers name:string salary:decimal

Isso gerará a seguinte migração:

class CreateUsers < ActiveRecord::Migration[5.0]
  def change
    create_table :users do |t|
      t.string :name
      t.decimal :salary  
    end
  end
end

Quando o nome da migração estiver no formato CreateXXX seguido de lista de colunas com tipos de dados, será gerada uma migração que cria a tabela XXX com as colunas listadas.

Executando migrações #

Comando de execução:

rake db:migrate

rails db:migrate

A especificação da versão de destino executará as migrações necessárias (para cima, para baixo, alteração) até atingir a versão especificada. Aqui, número da versão é o prefixo numérico no nome do arquivo da migração.

rake db:migrate VERSION=20080906120000

rails db:migrate VERSION=20080906120000

Alterar o tipo de uma coluna existente #

Para modificar uma coluna existente no Rails com uma migração, execute o seguinte comando:

rails g migration change_column_in_table

Isso criará um novo arquivo de migração no diretório db/migration (se ainda não existir), que conterá o arquivo prefixado com carimbo de data/hora e nome do arquivo de migração que contém o conteúdo abaixo:

def change
  change_column(:table_name, :column_name, :new_type)
end

Guia do Rails – Alterar colunas

Um método mais longo, mas mais seguro #

O código acima impede que o usuário reverta a migração. Você pode evitar esse problema dividindo o método change em métodos up e down separados:

def up
  change_column :my_table, :my_column, :new_type
end

def down
  change_column :my_table, :my_column, :old_type
end

Adicionar coluna com valor padrão #

O exemplo a seguir adiciona uma coluna admin à tabela users e dá a essa coluna o valor padrão false.

class AddDetailsToUsers < ActiveRecord::Migration[5.0]
  def change
    add_column :users, :admin, :boolean, default: false
  end
end

Migrações com padrões podem levar muito tempo em tabelas grandes com, por exemplo, PostgreSQL. Isso ocorre porque cada linha terá que ser atualizada com o valor padrão para a coluna recém-adicionada. Para contornar isso (e reduzir o tempo de inatividade durante as implantações), você pode dividir sua migração em três etapas:

  1. Adicione uma migração add_column semelhante à acima, mas não defina nenhum padrão
  2. Implante e atualize a coluna em uma tarefa de rake ou no console enquanto seu aplicativo estiver em execução. Certifique-se de que seu aplicativo já grave dados nessa coluna para linhas novas/atualizadas.
  3. Adicione outra migração change_column, que altera o padrão dessa coluna para o valor padrão desejado

Crie uma tabela de junção #

Para criar uma tabela de junção entre students e courses, execute o comando:

$ rails g migration CreateJoinTableStudentCourse student course

Isso gerará a seguinte migração:

class CreateJoinTableStudentCourse < ActiveRecord::Migration[5.0]
  def change
    create_join_table :students, :courses do |t|
      # t.index [:student_id, :course_id]
      # t.index [:course_id, :student_id]
    end
  end
end

Crie uma coluna hstore #

As colunas Hstore podem ser úteis para armazenar configurações. Eles estão disponíveis nos bancos de dados PostgreSQL após você habilitar a extensão.

class CreatePages < ActiveRecord::Migration[5.0]
  def change
    create_table :pages do |t|
      enable_extension 'hstore' unless extension_enabled?('hstore')
      t.hstore :settings
      t.timestamps
    end
  end
end

Adicione uma auto-referência #

Uma auto-referência pode ser útil para construir uma árvore hierárquica. Isso pode ser feito com add_reference em uma migração.

class AddParentPages < ActiveRecord::Migration[5.0]
  def change
    add_reference :pages, :pages
  end
end

A coluna de chave estrangeira será pages_id. Se você quiser decidir sobre o nome da coluna da chave estrangeira, você deve criar a coluna primeiro e adicionar a referência depois.

class AddParentPages < ActiveRecord::Migration[5.0]
  def change
    add_column :pages, :parent_id, :integer, null: true, index: true
    add_foreign_key :pages, :pages, column: :parent_id
  end
end

Crie uma coluna de matriz #

Uma coluna array é suportada pelo PostgreSQL. Rails irá converter automaticamente um array PostgreSQL para um array Ruby e vice-versa.

Crie uma tabela com uma coluna array:

create_table :products do |t|
  t.string :name
  t.text :colors, array: true, default: []
end

Adicione uma coluna array a uma tabela existente:

add_column :products, :colors, array: true, default: []

Adicione um índice para uma coluna array:

add_index :products, :colors, using: 'gin'

Tabelas de mudança #

Se você criou uma tabela com algum esquema errado, então a maneira mais fácil de alterar as colunas e suas propriedades é change_table. Revise o exemplo a seguir:

change_table :orders do |t|
  t.remove :ordered_at # removes column ordered_at
  t.string :skew_number # adds a new column 
  t.index  :skew_number #creates an index
  t.rename :location, :state #renames location column to state
end

A migração acima altera uma tabela orders. Aqui está uma descrição linha por linha das alterações:

  1. t.remove :ordered_at remove a coluna ordered_at da tabela orders.
  2. t.string :skew_number adiciona uma nova coluna do tipo string chamada skew_number na tabela orders.
  3. t.index :skew_number adiciona um índice na coluna skew_number na tabela orders.
  4. t.rename :location, :state renomeia a coluna location na tabela orders para state.

Adicione uma coluna exclusiva a uma tabela #

Para adicionar uma nova coluna única email a users, execute o seguinte comando:

rails generate migration AddEmailToUsers email:string:uniq

Isso criará a seguinte migração:

class AddEmailToUsers < ActiveRecord::Migration[5.0]
  def change
    add_column :users, :email, :string
    add_index :users, :email, unique: true
  end
end

Verificando o status da migração #

Podemos verificar o status das migrações executando

rake db:migrate:status

rails db:migrate:status

A saída ficará assim:

Status   Migration ID    Migration Name
--------------------------------------------------
up     20140711185212  Create documentation pages
up     20140724111844  Create nifty attachments table
up     20140724114255  Create documentation screenshots

No campo de status, up significa que a migração foi executada e down significa que precisamos executar a migração.

Proibir valores nulos #

Para proibir valores null nas colunas da sua tabela, adicione o parâmetro :null à sua migração, assim:

class AddPriceToProducts < ActiveRecord::Migration
  def change
    add_column :products, :float, null: false
  end
end

Adicionando uma restrição NOT NULL aos dados existentes #

Digamos que você queira adicionar uma chave estrangeira company_id à tabela users e queira ter uma restrição NOT NULL nela. Se você já possui dados em users, terá que fazer isso em várias etapas.

class AddCompanyIdToUsers < ActiveRecord::Migration
  def up
    # add the column with NULL allowed
    add_column :users, :company_id, :integer

    # make sure every row has a value
    User.find_each do |user|
      # find the appropriate company record for the user
      # according to your business logic
      company = Company.first
      user.update!(company_id: company.id)
    end

    # add NOT NULL constraint
    change_column_null :users, :company_id, false
  end

  # Migrations that manipulate data must use up/down instead of change
  def down
    remove_column :users, :company_id
  end
end
