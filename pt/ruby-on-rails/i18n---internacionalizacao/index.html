Sintaxe #

  • I18n.t(“chave”)
  • I18n.translate(“chave”) # equivalente a I18n.t("chave")
  • I18n.t(“chave”, contagem: 4)
  • I18n.t(“key”, param1: “Algo”, param2: “Else”)
  • I18n.t(“doesnt_exist”, default: “key”) # especifica um default se a chave estiver faltando
  • I18n.locale #=> :pt
  • I18n.locale = :pt
  • I18n.default_local #=> :pt
  • I18n.default_locale = :en
  • t(".key") # igual a I18n.t("key"), mas com escopo para a ação/modelo de onde é chamado

I18n com argumentos #

Você pode passar parâmetros para o método I18n t:

# Example config/locales/en.yml
en:
  page:
    users: "%{users_count} users currently online"

# In models, controller, etc...
I18n.t('page.users', users_count: 12)

# In views

# ERB
<%= t('page.users', users_count: 12) %>

#SLIM
= t('page.users', users_count: 12)

# Shortcut in views - DRY!
# Use only the dot notation
# Important: Consider you have the following controller and view page#users

# ERB Example app/views/page/users.html.erb
<%= t('.users', users_count: 12) %>

E obtenha a seguinte saída:

"12 users currently online"

Traduzindo atributos do modelo ActiveRecord #

globalize gem é uma ótima solução para adicionar traduções aos seus modelos ActiveRecord. Você pode instalá-lo adicionando isso ao seu Gemfile:

gem 'globalize', '~> 5.0.0'

Se você estiver usando Rails 5, você também precisará adicionar activemodel-serializers-xml

gem 'activemodel-serializers-xml'

As traduções de modelos permitem que você traduza os valores de atributo de seus modelos, por exemplo:

class Post < ActiveRecord::Base
  translates :title, :text
end

I18n.locale = :en
post.title # => Globalize rocks!

I18n.locale = :he
post.title # => גלובאלייז2 שולט!

Depois de definir os atributos do seu modelo que precisam ser traduzidos, você deve criar uma tabela de conversão, por meio de uma migração. globalize fornece create_translation_table! e drop_translation_table!.

Para esta migração você precisa usar up e down, e não change. Além disso, para executar essa migração com sucesso, você deve primeiro definir os atributos traduzidos em seu modelo, como mostrado acima. Uma migração adequada para o modelo Post anterior é esta:

class CreatePostsTranslationTable < ActiveRecord::Migration
  def up
    Post.create_translation_table! title: :string, text: :text
  end

  def down
    Post.drop_translation_table!
  end
end

Você também pode passar opções para opções específicas, como:

class CreatePostsTranslationTable < ActiveRecord::Migration
  def up
    Post.create_translation_table! title: :string,
      text: { type: :text, null: false, default: "Default text" }
  end

  def down
    Post.drop_translation_table!
  end
end

Caso você já tenha algum dado existente nas colunas de tradução necessárias, você pode migrá-lo facilmente para a tabela de traduções, ajustando sua migração:

class CreatePostsTranslationTable < ActiveRecord::Migration
  def up
    Post.create_translation_table!({
      title: :string,
      text: :text
    }, {
      migrate_data: true
    })
      
  end

  def down
    Post.drop_translation_table! migrate_data: true
  end
end

Certifique-se de remover as colunas traduzidas da tabela pai depois que todos os seus dados forem migrados com segurança. Para remover automaticamente as colunas traduzidas da tabela pai após a migração de dados, adicione a opção remove_source_columns à migração:

class CreatePostsTranslationTable < ActiveRecord::Migration
  def up
    Post.create_translation_table!({
      title: :string,
      text: :text
    }, {
      migrate_data: true,
      remove_source_columns: true
    })
      
  end

  def down
    Post.drop_translation_table! migrate_data: true
  end
end

Você também pode adicionar novos campos a uma tabela de traduções criada anteriormente:

class Post < ActiveRecord::Base
  # Remember to add your attribute here too.
  translates :title, :text, :author
end

class AddAuthorToPost < ActiveRecord::Migration
  def up
    Post.add_translation_fields! author: :text
  end

  def down
    remove_column :post_translations, :author
  end
end

Pluralização #

Você pode deixar I18n lidar com a pluralização para você, apenas use o argumento count.

Você precisa configurar seu arquivo de localidade assim:

# config/locales/en.yml
en:
  online_users:
    one: "1 user is online"
    other: "%{count} users are online"

E então use a chave que você acabou de criar passando o argumento count para o auxiliar I18n.t:

I18n.t("online_users", count: 1)
#=> "1 user is online"

I18n.t("online_users", count: 4)
#=> "4 users are online"

Definir localidade por meio de solicitações #

Na maioria dos casos, você pode querer definir I18n locale. Pode-se querer definir a localidade para a sessão atual, o usuário atual ou com base em um parâmetro de URL. controladores.

class ApplicationController < ActionController::Base
  before_action :set_locale

  protected

  def set_locale
    # Remove inappropriate/unnecessary ones
    I18n.locale = params[:locale] ||    # Request parameter
      session[:locale] ||               # Current session
      (current_user.preferred_locale if user_signed_in?) ||  # Model saved configuration
      extract_locale_from_accept_language_header ||          # Language header - browser config
      I18n.default_locale               # Set in your config files, english by super-default
  end

  # Extract language from request header
  def extract_locale_from_accept_language_header
    if request.env['HTTP_ACCEPT_LANGUAGE']
      lg = request.env['HTTP_ACCEPT_LANGUAGE'].scan(/^[a-z]{2}/).first.to_sym
      lg.in?([:en, YOUR_AVAILABLE_LANGUAGES]) ? lg : nil
    end
  end

baseado em URL #

O parâmetro locale pode vir de um URL como este

http://yourapplication.com/products?locale=en

Ou

http://yourapplication.com/en/products

Para conseguir isso, você precisa editar suas routes, adicionando um scope:

# config/routes.rb
scope "(:locale)", locale: /en|fr/ do
  resources :products
end

Ao fazer isso, visitar http://yourapplication.com/en/products definirá sua localidade para :en. Em vez disso, visitar http://yourapplication.com/fr/products o definirá como :fr. Além disso, você não receberá um erro de roteamento ao perder o parâmetro :locale, pois visitar http://yourapplication.com/products carregará a localidade padrão I18n.

Baseado em sessão ou baseado em persistência #

Isso pressupõe que o usuário pode clicar em um botão/sinalizador de idioma para alterar o idioma. A ação pode rotear para um controlador que define a sessão para o idioma atual (e eventualmente persistir as alterações em um banco de dados se o usuário estiver conectado)

class SetLanguageController < ApplicationController
  skip_before_filter :authenticate_user!
  after_action :set_preferred_locale

  # Generic version to handle a large list of languages
  def change_locale
    I18n.locale = sanitize_language_param
    set_session_and_redirect
  end

Você tem que definir repair_language_param com sua lista de idiomas disponíveis, e eventualmente lidar com erros caso o idioma não exista.

Se você tiver muito poucos idiomas, pode valer a pena defini-los assim:

def fr
  I18n.locale = :fr
  set_session_and_redirect
end

def en
  I18n.locale = :en
  set_session_and_redirect
end

private

  def set_session_and_redirect
    session[:locale] = I18n.locale
    redirect_to :back
  end

  def set_preferred_locale
    if user_signed_in?
      current_user.preferred_locale = I18n.locale.to_s
      current_user.save if current_user.changed?
    end
  end
end

Nota: não se esqueça de adicionar algumas rotas às suas ações change_language

Localidade padrão #

Lembre-se de que você precisa definir a localidade padrão do aplicativo. Você pode fazer isso configurando-o em config/application.rb:

config.i18n.default_locale = :de

ou criando um inicializador na pasta config/initializers:

# config/initializers/locale.rb
I18n.default_locale = :it

Obter localidade da solicitação HTTP #

Às vezes, pode ser útil definir a localidade do aplicativo com base no IP da solicitação. Você pode facilmente conseguir isso usando o Geocoder. Entre as muitas coisas que o Geocoder faz, ele também pode informar a localização de uma requisição.

Primeiro, adicione Geocoder ao seu Gemfile

# Gemfile
gem 'geocoder'

Geocoder adiciona os métodos location e safe_location ao objeto padrão Rack::Request para que você possa facilmente procurar a localização de qualquer solicitação HTTP por endereço IP. Você pode usar esses métodos em um before_action em seu ApplicationController:

class ApplicationController < ActionController::Base
  before_action :set_locale_from_request

  def set_locale_from_request
    country_code = request.location.data["country_code"] #=> "US"
    country_sym = country_code.underscore.to_sym #=> :us

    # If request locale is available use it, otherwise use I18n default locale
    if I18n.available_locales.include? country_sym
      I18n.locale = country_sym
    end
end

Esteja ciente de que isso não funcionará em ambientes de desenvolvimento e teste, pois coisas como 0.0.0.0 e localhost são endereços IP válidos da Internet.

Limitações e alternativas #

Geocoder é muito poderoso e flexível, mas precisa ser configurado para funcionar com um serviço de geocodificação (veja mais detalhes); muitos dos quais colocam limites no uso. Também vale a pena ter em mente que chamar um serviço externo em cada solicitação pode afetar o desempenho.

Para resolver isso, também pode valer a pena considerar:

1. Uma solução offline #

Usar uma gem como GeoIP (veja aqui) permite que pesquisas aconteçam em um arquivo de dados local. Pode haver uma compensação em termos de precisão, pois esses arquivos de dados precisam ser mantidos atualizados.

2. Use CloudFlare #

As páginas veiculadas pelo CloudFlare têm a opção de serem geocodificadas de forma transparente, com o código do país sendo adicionado ao cabeçalho (HTTP_CF_IPCOUNTRY). Mais detalhes podem ser encontrados aqui.

Use I18n com tags e símbolos HTML #

# config/locales/en.yml
en:
  stackoverflow:
    header:
      title_html: "Use <strong>I18n</strong> with Tags &amp; Symbols"

Observe a adição de _html extra após o nome title.

E em Visualizações,

# ERB
<h2><%= t(:title_html, scope: [:stackoverflow, :header]) %></h2>

Use I18n nas visualizações #

Supondo que você tenha este arquivo de localidade YAML:

# config/locales/en.yml
en:
  header:
    title: "My header title"

e você deseja exibir sua string de título, você pode fazer isso

# in ERB files
<%= t('header.title') %>

# in SLIM files
= t('header.title')
