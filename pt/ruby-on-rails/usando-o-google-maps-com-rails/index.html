Adicione a tag javascript do Google Maps ao cabeçalho do layout #

Para que o Google Maps funcione corretamente com turbolinks, adicione a tag javascript diretamente ao cabeçalho do layout em vez de incluí-la em uma visualização.

# app/views/layouts/my_layout.html.haml
!!!
%html{:lang => 'en'}
  %head
    - # ...
    = google_maps_api_script_tag

O google_maps_api_script_tag é melhor definido em um auxiliar.

# app/helpers/google_maps_helper.rb
module GoogleMapsHelper
  def google_maps_api_script_tag
    javascript_include_tag google_maps_api_source
  end

  def google_maps_api_source
    "https://maps.googleapis.com/maps/api/js?key=#{google_maps_api_key}"
  end

  def google_maps_api_key
    Rails.application.secrets.google_maps_api_key
  end
end

Você pode registrar seu aplicativo no Google e obter sua chave de API no console de API do Google. O Google tem um breve guia como solicitar uma chave de API para a API de javascript do Google Maps.

A chave api é armazenada no arquivo secrets.yml:

# config/secrets.yml
development:
  google_maps_api_key: '...'
  # ...
production:
  google_maps_api_key: '...'
  # ...

Não se esqueça de adicionar config/secrets.yml ao seu arquivo .gitignore e certifique-se de não submeter a chave api ao repositório.

Geocodifique o modelo #

Suponha que seus usuários e/ou grupos tenham perfis e você queira exibir campos de perfil de endereço em um mapa do Google.

# app/models/profile_fields/address.rb
class ProfileFields::Address < ProfileFields::Base
  # Attributes: 
  # label, e.g. "Work address"
  # value, e.g. "Willy-Brandt-Straße 1\n10557 Berlin"
end

Uma ótima maneira de geocodificar os endereços, ou seja, fornecer longitude e latitude é o geocoder gem.

Adicione geocoder ao seu Gemfile e execute bundle para instalá-lo.

# Gemfile
gem 'geocoder', '~> 1.3'

Adicione colunas de banco de dados para latitude e longitude para salvar o local no banco de dados. Isso é mais eficiente do que consultar o serviço de geocodificação toda vez que você precisar da localização. É mais rápido e você não está atingindo o limite de consultas tão rapidamente.

➜ bin/rails generate migration add_latitude_and_longitude_to_profile_fields \
    latitude:float longitude:float
➜ bin/rails db:migrate  # Rails 5, or:
➜ rake db:migrate       # Rails 3, 4

Adicione o mecanismo de geocodificação ao seu modelo. Neste exemplo, a string de endereço é armazenada no atributo value. Configure a geocodificação para ser executada quando o registro for alterado e somente quando um valor estiver presente:

# app/models/profile_fields/address.rb
class ProfileFields::Address < ProfileFields::Base
  geocoded_by :value
  after_validation :geocode, if: ->(address_field){ 
    address_field.value.present? and address_field.value_changed? 
  }

end

Por padrão, o geocoder usa o google como serviço de pesquisa. Possui muitos recursos interessantes, como cálculos de distância ou pesquisa de proximidade. Para mais informações, dê uma olhada no geocoder README.

Mostrar endereços em um mapa do Google na visualização de perfil #

Na visualização de perfil, mostre os campos de perfil de um usuário ou grupo em uma lista, bem como os campos de endereço em um mapa do Google.

- # app/views/profiles/show.html.haml
%h1 Contact Information
.profile_fields
  = render @profile_fields
.google_map{data: address_fields: @address_fields.to_json }

Os @profile_fields e @address_fields apropriados são definidos no controlador:

# app/controllers/profiles_controller.rb
class ProfilesController < ApplicationController
  def show
    # ...
    @profile_fields = @user_or_group.profile_fields
    @address_fields = @profile_fields.where(type: 'ProfileFields::Address')
  end
end

Inicialize o mapa, coloque os marcadores, defina o zoom e outras configurações do mapa com javascript.

Defina os marcadores no mapa com javascript #

Suponha que haja uma div .google_map, que se tornará o mapa, e que possui os campos de endereço para mostrar como marcadores como atributo data.

Por exemplo:

<!-- http://localhost:3000/profiles/123 -->
<div class="google_map" data-address-fields="[
  {label: 'Work address', value: 'Willy-Brandt-Straße 1\n10557 Berlin', 
  position: {lng: ..., lat: ...}},
  ...
]"></div>

Para usar o evento $(document).ready com turbolinks sem gerenciar os eventos turbolinks manualmente, use a [jquery.turbolinks gem](https: //github.com/kossnocorp/jquery.turbolinks).

Se você quiser realizar outras operações com o mapa, mais tarde, por exemplo filtragem ou janelas de informações, é conveniente ter o mapa gerenciado por uma classe de script de café.

# app/assets/javascripts/google_maps.js.coffee
window.App = {} unless App?
class App.GoogleMap
  constructor: (map_div)->
    # TODO: initialize the map
    # TODO: set the markers

Ao usar vários arquivos de script de café, que são namespaced por padrão, é conveniente definir um namespace de App global, que é compartilhado por todos os arquivos de script de café.

Em seguida, percorra (possivelmente vários) divs .google_map e crie uma instância da classe App.GoogleMap para cada um deles.

# app/assets/javascripts/google_maps.js.coffee
# ...
$(document).ready ->
  App.google_maps = []
  $('.google_map').each ->
    map_div = $(this)
    map = new App.GoogleMap map_div
    App.google_maps.push map

Inicialize o mapa usando uma classe de script de café. #

Com um App.GoogleMap classe de script de café, o mapa do Google pode ser inicializado assim:

# app/assets/javascripts/google_maps.js.coffee
# ...
class App.GoogleMap
  map_div: {}
  map: {}
  
  constructor: (map_div)->
    @map_div = map_div
    @init_map()
    @reference_the_map_as_data_attribute

  # To access the GoogleMap object or the map object itself
  # later via the DOM, for example 
  # 
  #     $('.google_map').data('GoogleMap')
  #
  # store references as data attribute of the map_div.
  #
  reference_the_map_as_data_attribute: ->
    @map_div.data 'GoogleMap', this
    @map_div.data 'map', @map

  init_map: ->
    @map = new google.maps.Map(@dom_element, @map_configuration) if google?

  # `@map_div` is the jquery object. But google maps needs
  # the real DOM element.
  #
  dom_element: ->
    @map_div.get(0)

  map_configuration: -> {
    scrollWheel: true
  }

Para saber mais sobre as possíveis opções de map_configuration, dê uma olhada na documentação do MapOptions do Google e seu guia para adicionar elementos de controle.

Para referência, a classe google.maps.Mapestá amplamente documentada aqui.

Inicialize os marcadores do mapa usando uma classe de script de café #

Forneceu um App.GoogleMap classe de script de café e as informações do marcador que estão sendo armazenadas no atributo data-address-fields da div .google_map, o mapa marcadores podem ser inicializados no mapa assim:

# app/assets/javascripts/google_maps.js.coffee
# ...
class App.GoogleMap
  # ...
  markers: []

  constructor: (map_div)->
    # ...
    @init_markers()

  address_fields: ->
    @map_div.data('address-fields')

  init_markers: ->
    self = this  # to reference the instance as `self` when `this` is redefined.
    self.markers = []
    for address_field in self.address_fields()
      marker = new google.maps.Marker {
        map: self.map,
        position: {
          lng: address_field.longitude,
          lat: address_field.latitude
        },
        # # or, if `position` is defined in `ProfileFields::Address#as_json`:
        # position: address_field.position,
        title: address_field.value
      }
      self.markers.push marker

Para saber mais sobre as opções de marcadores, consulte a documentação de MarkerOptions do Google e seu [guia de marcadores](https ://developers.google.com/maps/documentation/javascript/markers).

Zoom automático de um mapa usando uma classe de script de café #

Forneceu um App.GoogleMap classe de script de café com o google.maps.Map armazenado como @map e o google.maps.Markers armazenado como @markers, o mapa pode ser ampliado automaticamente, ou seja, ajustado para que todos os marcadores fiquem visíveis, assim: no mapa assim:

# app/assets/javascripts/google_maps.js.coffee
# ...
class App.GoogleMap
  # ...
  bounds: {}

  constructor: (map_div)->
    # ...
    @auto_zoom()

  auto_zoom: ->
    @init_bounds()
    # TODO: Maybe, adjust the zoom to have a maximum or 
    # minimum zoom level, here.

  init_bounds: ->
    @bounds = new google.maps.LatLngBounds()
    for marker in @markers
      @bounds.extend marker.position
    @map.fitBounds @bounds

Para saber mais sobre limites, dê uma olhada na [documentação de LatLngBounds] do Google(https://developers.google.com/maps/documentation/javascript/3.exp/reference#LatLngBounds).

Expondo as propriedades do modelo como json #

Para exibir campos de perfil de endereço como marcadores em um mapa do google, os objetos de campo de endereço precisam ser passados ​​como objetos json para javascript.

Atributos regulares do banco de dados #

Ao chamar to_json em um objeto ApplicationRecord, os atributos do banco de dados são expostos automaticamente.

Dado um modelo ProfileFields::Address com atributos label, value, longitude e latitude, address_field.as_json resulta em um Hash, por exemplo. representação,

address_field.as_json  # =>
  {label: "Work address", value: "Willy-Brandt-Straße 1\n10557 Berlin",
    longitude: ..., latitude: ...}

que é convertido em uma string json por to_json:

address_field.to_json  # =>
  "{\"label\":\"Work address\",\"value\":\"Willy-Brandt-Straße 1\\n
    10557 Berlin\",\"longitude\":...,\"latitude\":...}"

Isso é útil porque permite usar label e value posteriormente em javascript, por exemplo, para mostrar dicas de ferramentas para os marcadores de mapa.

Outros atributos #

Outros atributos virtuais podem ser expostos substituindo o método as_json.

Por exemplo, para expor um atributo title, inclua-o no hash as_json mesclado:

# app/models/profile_fields/address.rb
class ProfileFields::Address < ProfileFields::Base
  # ...

  # For example: "John Doe, Work address"
  def title
    "#{self.parent.name}, #{self.label}"
  end

  def as_json
    super.merge {
      title: self.title
    }
  end
end

O exemplo acima usa super para chamar o método as_json original, que retorna o hash de atributo original do objeto e o mescla com a posição necessária cerquilha.

Para entender a diferença entre as_json e to_json, dê uma olhada em esta postagem do blog de jjulian.

Posição #

Para renderizar marcadores, a API do Google Maps, por padrão, requer um hash position que tem longitude e latitude armazenados como lng e lat respectivamente.

Este hash de posição pode ser criado em javascript, posteriormente, ou aqui ao definir a representação json do campo de endereço:

Para fornecer este position como atributo json do campo de endereço, apenas substitua o método as_json no modelo.

# app/models/profile_fields/address.rb
class ProfileFields::Address < ProfileFields::Base
  # ...

  def as_json
    super.merge {
      # ...
      position: {
        lng: self.longitude,
        lat: self.latitude
      }
    }
  end
end
