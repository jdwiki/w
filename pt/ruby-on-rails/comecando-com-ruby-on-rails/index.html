Criando um aplicativo Ruby on Rails #

Este exemplo assume que Ruby e Ruby on Rails já foram instalados corretamente. Se não, você pode encontrar como fazê-lo aqui.

Abra uma linha de comando ou terminal. Para gerar um novo aplicativo rails, use o comando rails new seguido do nome do seu aplicativo:

$ rails new my_app

Se você deseja criar seu aplicativo Rails com uma versão específica do Rails, você pode especificá-lo no momento da geração do aplicativo. Para fazer isso, use rails _version_ new seguido pelo nome do aplicativo:

$ rails _4.2.0_ new my_app

Isso criará um aplicativo Rails chamado MyApp em um diretório my_app e instalará as dependências de gem que já são mencionadas em Gemfile usando bundle install.

Para mudar para o diretório do seu aplicativo recém-criado, use o comando cd, que significa change directory.

$ cd my_app

O diretório my_app possui vários arquivos e pastas gerados automaticamente que compõem a estrutura de uma aplicação Rails. A seguir está uma lista de arquivos e pastas que são criados por padrão:

Arquivo/Pasta Finalidade
aplicativo/ Contém os controllers, models, views, helpers, mailers e assets para seu aplicativo.
caixa/ Contém o script Rails que inicia seu aplicativo e pode conter outros scripts que você usa para configurar, atualizar, implantar ou executar seu aplicativo.
configuração/ Configure as rotas, banco de dados e muito mais do seu aplicativo.
config.ru Configuração de rack para servidores baseados em rack usados ​​para iniciar o aplicativo.
banco de dados/ Contém seu esquema de banco de dados atual, bem como as migrações de banco de dados.
Gemfile Gemfile.lock Esses arquivos permitem que você especifique quais dependências de gem são necessárias para sua aplicação Rails. Esses arquivos são usados ​​pela gem Bundler.
lib/ Módulos estendidos para sua aplicação.
log/ Arquivos de log do aplicativo.
público/ A única pasta vista pelo mundo como está. Contém arquivos estáticos e ativos compilados.
Rakefile Este arquivo localiza e carrega tarefas que podem ser executadas na linha de comando. As definições de tarefas são definidas em todos os componentes do Rails.
README.md Este é um breve manual de instruções para sua aplicação. Você deve editar este arquivo para dizer aos outros o que seu aplicativo faz, como configurá-lo etc
teste/ Testes unitários, acessórios e outros aparelhos de teste.
temperatura/ Arquivos temporários (como arquivos cache e pid).
fornecedor/ Um lugar para todos os códigos de terceiros. Em um aplicativo Rails típico, isso inclui gems vendidas.

Agora você precisa criar um banco de dados do seu arquivo database.yml:

rake db:create
# OR
rails db:create

rake db:create

Agora que criamos o banco de dados, precisamos executar as migrações para configurar as tabelas:

rake db:migrate
# OR
rails db:migrate

rake db:migrate

Para iniciar o aplicativo, precisamos iniciar o servidor:

$ rails server
# OR
$ rails s

Por padrão, o Rails iniciará o aplicativo na porta 3000. Para iniciar o aplicativo com um número de porta diferente, precisamos iniciar o servidor como,

$ rails s -p 3010

Se você navegar para http://localhost:3000 em seu navegador, verá uma página de boas-vindas do Rails, mostrando que seu aplicativo está em execução.

Se lançar um erro, pode haver vários problemas possíveis:

  • Há um problema com o config/database.yml
  • Você tem dependências em seu Gemfile que não foram instaladas.
  • Você tem migrações pendentes. Execute rails db:migrate
  • Caso você mude para a migração anterior rails db:rollback

Se isso ainda gerar um erro, você deve verificar seu config/database.yml

Crie um novo aplicativo Rails com sua escolha de banco de dados e incluindo a Ferramenta de Teste RSpec #

Rails usa sqlite3 como banco de dados padrão, mas você pode gerar uma nova aplicação Rails com um banco de dados de sua escolha. Basta adicionar a opção -d seguida do nome do banco de dados.

$ rails new MyApp -T -d postgresql

Esta é uma lista (não exaustiva) de opções de banco de dados disponíveis:

  • mysql
  • oráculo
  • postgresql -sqlite3
  • base frontal
  • ibm_db
  • servidor SQL
  • jdbcmysql
  • jdbcsqlite3
  • jdbcpostgresql
  • jdbc

O comando -T indica pular a instalação do minitest. Para instalar um conjunto de testes alternativo como RSpec, edite o Gemfile e adicione

group :development, :test do
  gem 'rspec-rails', 
end

Em seguida, inicie o seguinte comando no console:

rails generate rspec:install

Gerando um controlador #

Para gerar um controlador (por exemplo, Posts), navegue até o diretório do projeto a partir de uma linha de comando ou terminal e execute:

$ rails generate controller Posts

Você pode encurtar este código substituindo generate por g, por exemplo:

$ rails g controller Posts

Se você abrir o app/controllers/posts_controller.rb recém-gerado, verá um controlador sem ações:

class PostsController < ApplicationController
    # empty
end

É possível criar métodos padrão para o controlador passando os argumentos do nome do controlador.

$ rails g controller ControllerName method1 method2

Para criar um controlador dentro de um módulo, especifique o nome do controlador como um caminho como parent_module/controller_name. Por exemplo:

$ rails generate controller CreditCards open debit credit close
# OR
$ rails g controller CreditCards open debit credit close

Isso irá gerar os seguintes arquivos:

Controller: app/controllers/credit_cards_controller.rb
Test:       test/controllers/credit_cards_controller_test.rb
Views:      app/views/credit_cards/debit.html.erb [...etc]
Helper:     app/helpers/credit_cards_helper.rb

Um controlador é simplesmente uma classe que é definida para herdar de ApplicationController.

É dentro dessa classe que você definirá os métodos que se tornarão as ações desse controlador.

Instalando trilhos #

Instalando Rails no Ubuntu

Em um Ubuntu limpo, a instalação do Rails deve ser direta

Atualizando pacotes do Ubuntu

sudo apt-get update
sudo apt-get upgrade

Instalar dependências Ruby e Rails

sudo apt-get install git-core curl zlib1g-dev build-essential libssl-dev libreadline-dev libyaml-dev libsqlite3-dev sqlite3 libxml2-dev libxslt1-dev libcurl4-openssl-dev python-software-properties libffi-dev

Instalando o gerenciador de versões ruby. Neste caso, o mais fácil é usar rbenv

git clone https://github.com/rbenv/rbenv.git ~/.rbenv
echo 'export PATH="$HOME/.rbenv/bin:$PATH"' >> ~/.bashrc
echo 'eval "$(rbenv init -)"' >> ~/.bashrc

Instalando o Ruby Build

git clone https://github.com/rbenv/ruby-build.git ~/.rbenv/plugins/ruby-build
echo 'export PATH="$HOME/.rbenv/plugins/ruby-build/bin:$PATH"' >> ~/.bashrc

Reiniciar Shell

exec $SHELL

Instalar ruby

rbenv install 2.3.1
rbenv global 2.3.1
rbenv rehash

Instalando trilhos

gem install rails

Instalando Rails no Windows

Etapa 1: Instalando Ruby

Precisamos da linguagem de programação Ruby instalada. Podemos usar uma versão pré-compilada do Ruby chamada RubyInstaller.

  • Baixe e execute o Ruby Installer em rubyinstaller.org.
  • Execute o instalador. Marque “Adicionar executáveis ​​Ruby ao seu PATH” e instale.
  • Para acessar o Ruby, vá ao menu do Windows, clique em Todos os Programas, role para baixo até Ruby e clique em “Iniciar Prompt de Comando com Ruby”. Um terminal de prompt de comando será aberto. Se você digitar ruby -v e pressionar Enter, você deverá ver o número da versão do Ruby que você instalou.

Passo 2: Kit de Desenvolvimento Ruby

Após instalar o Ruby, podemos tentar instalar o Rails. Mas algumas das bibliotecas das quais o Rails depende precisam de algumas ferramentas de compilação para serem compiladas, e o Windows não possui essas ferramentas por padrão. Você pode identificar isso se vir um erro ao tentar instalar Rails Gem::InstallError: A gem nativa ‘[nome da gem]’ requer ferramentas de compilação instaladas. Para corrigir isso, precisamos instalar o Ruby Development Kit.

  • Baixe o DevKit
  • Execute o instalador.
  • Precisamos especificar uma pasta onde instalaremos permanentemente o DevKit. Eu recomendo instalá-lo na raiz do seu disco rígido, em C:\RubyDevKit. (Não use espaços no nome do diretório.)

Agora precisamos disponibilizar as ferramentas DevKit para Ruby.

  • No prompt de comando, mude para o diretório DevKit. cd C:\RubyDevKit ou qualquer diretório em que você o instalou.
  • Precisamos executar um script Ruby para inicializar a configuração do DevKit. Modelo ruby dk.rb init. Agora vamos dizer ao mesmo script para adicionar o DevKit à nossa instalação do Ruby. Digite ruby dk.rb install.

O DevKit agora deve estar disponível para suas ferramentas Ruby usarem ao instalar novas bibliotecas.

Etapa 3: Trilhos

Agora podemos instalar o Rails. Rails vem como uma gema Ruby. No seu prompt de comando, digite:

rails de instalação de gem

Assim que você pressionar Enter, o programa gem baixará e instalará essa versão da gem Rails, junto com todas as outras gems das quais Rails depende.

Etapa 4: Node.js

Algumas bibliotecas das quais o Rails depende requerem um runtime JavaScript para ser instalado. Vamos instalar o Node.js para que essas bibliotecas funcionem corretamente.

  • Faça o download do instalador do Node.js em aqui.
  • Quando o download terminar, visite sua pasta de downloads e execute o instalador node-v4.4.7.pkg.
  • Leia o contrato de licença completo, aceite os termos e clique em Avançar no restante do assistente, deixando tudo no padrão.
  • Uma janela pode aparecer perguntando se você deseja permitir que o aplicativo faça alterações em seu computador. Clique em “Sim”.
  • Quando a instalação estiver concluída, você precisará reiniciar seu computador para que o Rails possa acessar o Node.js.

Quando o computador reiniciar, não se esqueça de ir ao menu do Windows, clicar em “Todos os Programas”, rolar para baixo até Ruby e clicar em “Iniciar Prompt de Comando com Ruby”.

Crie um novo aplicativo Rails com um adaptador de banco de dados não padrão #

Rails é fornecido por padrão com ActiveRecord, um ORM (Object Relational Mapping) derivado do padrão com o mesmo nome.

Como um ORM, ele foi desenvolvido para lidar com mapeamento relacional e, mais precisamente, para lidar com solicitações SQL para você, daí a limitação apenas para bancos de dados SQL.

No entanto, você ainda pode criar um aplicativo Rails com outro sistema de gerenciamento de banco de dados:

  1. basta criar seu aplicativo sem registro ativo
$ rails app new MyApp --skip-active-record
  1. adicione seu próprio sistema de gerenciamento de banco de dados em Gemfile
gem 'mongoid', '~> 5.0'
  1. bundle install e siga os passos de instalação do banco de dados desejado.

Neste exemplo, mongoid é um mapeamento de objetos para MongoDB e - como muitas outras gems de banco de dados construídas para trilhos - também herda de ActiveModel da mesma forma que ActiveRecord, que fornece uma interface comum para muitos recursos, como como validações, callbacks, traduções, etc.

Outros adaptadores de banco de dados incluem, mas não estão limitados a:

  • pastas de dados
  • sequela-rails

Criando APIs Rails em JSON #

Este exemplo pressupõe que você tenha experiência na criação de aplicativos Rails.

Para criar um aplicativo somente API no Rails 5, execute

rails new name-of-app --api

Adicione active_model_serializers no Gemfile

gem 'active_model_serializers'

instalar pacote no terminal

bundle install

Defina o adaptador ActiveModelSerializer para usar :json_api

# config/initializers/active_model_serializer.rb
ActiveModelSerializers.config.adapter = :json_api
Mime::Type.register "application/json", :json, %w( text/x-json application/jsonrequest application/vnd.api+json )

Gere um novo andaime para seu recurso

rails generate scaffold Task name:string description:text

Isso irá gerar os seguintes arquivos:

Controlador: app/controllers/tasks_controller.rb

Test:       test/models/task_test.rb
            test/controllers/tasks_controller_test.rb
Routes:     resources :tasks added in routes.rb
Migration:  db/migrate/_create_tasks.rb
Model:      app/models/task.rb
Serializer: app/serializers/task_serializer.rb
Controller: app/controllers/tasks_controller.rb

Gerar um recurso com scaffolds #

De guides.rubyonrails.org:

Em vez de gerar um modelo diretamente. . . vamos montar um andaime. Um scaffold no Rails é um conjunto completo de modelo, migração de banco de dados para esse modelo, controlador para manipulá-lo, visualizações para visualizar e manipular os dados e um conjunto de testes para cada um dos itens acima.

Aqui está um exemplo de scaffolding de um recurso chamado Task com um nome de string e uma descrição de texto:

rails generate scaffold Task name:string description:text

Isso irá gerar os seguintes arquivos:

Controller: app/controllers/tasks_controller.rb
Test:       test/models/task_test.rb
            test/controllers/tasks_controller_test.rb
Routes:     resources :tasks added in routes.rb
Views:      app/views/tasks
            app/views/tasks/index.html.erb
            app/views/tasks/edit.html.erb
            app/views/tasks/show.html.erb
            app/views/tasks/new.html.erb
            app/views/tasks/_form.html.erb
Helper:     app/helpers/tasks_helper.rb
JS:         app/assets/javascripts/tasks.coffee 
CSS:        app/assets/stylesheets/tasks.scss
            app/assets/stylesheets/scaffolds.scss

exemplo para excluir arquivos gerados pelo scaffold para o recurso chamado Task

rails destroy scaffold Task
