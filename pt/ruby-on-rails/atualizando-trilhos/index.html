Atualizando do Rails 4.2 para o Rails 5.0 #

Observação: antes de atualizar seu aplicativo Rails, sempre salve seu código em um sistema de controle de versão, como o Git.

Para atualizar do Rails 4.2 para Rails 5.0, você deve estar usando Ruby 2.2.2 ou mais recente. Após atualizar sua versão do Ruby, se necessário, vá para o seu Gemfile e altere a linha:

gem 'rails', '4.2.X'

para:

gem 'rails', '~> 5.0.0'

e na linha de comando execute:

$ bundle update

Agora execute a tarefa de atualização usando o comando:

$ rake rails:update

Isso ajudará você a atualizar os arquivos de configuração. Você será solicitado a substituir os arquivos e terá várias opções para inserir:

  • S - sim, sobrescrever
  • n – não, não sobrescreva
  • a – todos, sobrescrever este e todos os outros
  • q – sair, abortar
  • d – diff, mostra as diferenças entre o antigo e o novo
  • h - ajuda

Normalmente, você deve verificar as diferenças entre os arquivos antigos e novos para ter certeza de que não está recebendo nenhuma alteração indesejada.

Os modelos ActiveRecord do Rails 5.0 herdam de ApplicationRecord, em vez de ActiveRecord::Base. ApplicationRecord é a superclasse para todos os modelos, semelhante a como ApplicationController é a superclasse para controladores. Para levar em conta essa nova maneira de lidar com os modelos, você deve criar um arquivo em sua pasta app/models/ chamado application_record.rb e editar o conteúdo desse arquivo para ser:

class ApplicationRecord < ActiveRecord::Base
  self.abstract_class = true
end

Rails 5.0 também lida com callbacks um pouco diferente. Callbacks que retornam false não irão parar a cadeia de callbacks, o que significa que callbacks subsequentes ainda serão executados, diferentemente do Rails 4.2. Ao atualizar, o comportamento do Rails 4.2 permanecerá, embora você possa alternar para o comportamento do Rails 5.0 adicionando:

ActiveSupport.halt_callback_chains_on_return_false = false

para o arquivo config/application.rb. Você pode interromper explicitamente a cadeia de retorno de chamada chamando throw(:abort).

No Rails 5.0, ActiveJob herdará de ApplicationJob, em vez de ActiveJob::Base como no Rails 4.2. Para atualizar para Rails 5.0, crie um arquivo chamado application_job.rb na pasta app/jobs/. Edite o conteúdo desse arquivo para ser:

class ApplicationJob < ActiveJob::Base
end

Então, você deve alterar todos os seus trabalhos para herdar de ApplicationJob em vez de ActiveJob::Base.

Uma das outras grandes mudanças do Rails 5.0 não requer nenhuma mudança de código, mas mudará a forma como você usa a linha de comando com seus aplicativos Rails. Você poderá usar bin/rails, ou apenas rails, para executar tarefas e testes. Por exemplo, em vez de usar $ rake db:migrate, agora você pode fazer $ rails db:migrate. Se você executar $ bin/rails, poderá visualizar todos os comandos disponíveis. Note que muitas das tarefas que agora podem ser executadas com bin/rails ainda funcionam usando rake.
