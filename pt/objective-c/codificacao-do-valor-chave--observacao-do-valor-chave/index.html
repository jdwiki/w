Exemplo de codificação de valor-chave da vida real mais comum #

A codificação de valor-chave é integrada ao NSObject usando o protocolo NSKeyValueCoding.

O que isto significa?

Isso significa que qualquer objeto id é capaz de chamar o método valueForKey e suas várias variantes como valueForKeyPath etc. '

Isso também significa que qualquer objeto id pode invocar o método setValue e suas várias variantes também.

Exemplo:

id obj = [[MyClass alloc] init];
id value = [obj valueForKey:@"myNumber"];

int myNumberAsInt = [value intValue];
myNumberAsInt = 53;
[obj setValue:@(myNumberAsInt) forKey:@"myNumber"];

Exceções:

O exemplo acima supõe que MyClass tenha uma propriedade NSNumber chamada myNumber. Se myNumber não aparecer na definição da interface MyClass, uma NSUndefinedKeyException pode ser gerada possivelmente nas linhas 2 e 5 - popularmente conhecidas como:

this class is not key value coding-compliant for the key myNumber.

Por que isso é tão poderoso:

Você pode escrever código que pode acessar propriedades de uma classe dinamicamente, sem precisar de interface para essa classe. Isso significa que uma visualização de tabela pode exibir valores de qualquer propriedade de um objeto derivado de NSObject, desde que seus nomes de propriedade sejam fornecidos dinamicamente em tempo de execução.

No exemplo acima, o código também pode funcionar sem que MyClass esteja disponível e id type obj esteja disponível para chamar o código.

Observação do valor-chave #

Configurando a observação do valor da chave.

Neste caso, queremos observar o contentOffset em um objeto que nosso observador possui

//
// Class to observe
//
@interface XYZScrollView: NSObject
@property (nonatomic, assign) CGPoint contentOffset;
@end

@implementation XYZScrollView
@end


//
// Class that will observe changes
//
@interface XYZObserver: NSObject
@property (nonatomic, strong) XYZScrollView *scrollView;
@end

@implementation XYZObserver

// simple way to create a KVO context
static void *XYZObserverContext = &XYZObserverContext;


// Helper method to add self as an observer to 
// the scrollView's contentOffset property
- (void)addObserver {

    // NSKeyValueObservingOptions
    //
    // - NSKeyValueObservingOptionNew
    // - NSKeyValueObservingOptionOld
    // - NSKeyValueObservingOptionInitial
    // - NSKeyValueObservingOptionPrior
    //
    // can be combined:
    // (NSKeyValueObservingOptionNew | NSKeyValueObservingOptionOld)
    
    NSString *keyPath = NSStringFromSelector(@selector(contentOffset));
    NSKeyValueObservingOptions options = NSKeyValueObservingOptionNew;    

    [self.scrollView addObserver: self 
                      forKeyPath: keyPath 
                         options: options
                         context: XYZObserverContext];
}

- (void)observeValueForKeyPath:(NSString *)keyPath ofObject:(id)object change:(NSDictionary<NSString *,id> *)change context:(void *)context {
    
    if (context == XYZObserverContext) { // check the context

        // check the keyPath to see if it's any of the desired keyPath's.
        // You can observe multiple keyPath's
        if ([keyPath isEqualToString: NSStringFromSelector(@selector(contentOffset))]) {

            // change dictionary keys:
            // - NSKeyValueChangeKindKey
            // - NSKeyValueChangeNewKey
            // - NSKeyValueChangeOldKey
            // - NSKeyValueChangeIndexesKey
            // - NSKeyValueChangeNotificationIsPriorKey
            
            // the change dictionary here for a CGPoint observation will
            // return an NSPoint, so we can take the CGPointValue of it.
            CGPoint point = [change[NSKeyValueChangeNewKey] CGPointValue];
            
            // handle point
        }
        
    } else {

        // if the context doesn't match our current object's context
        // we want to pass the observation parameters to super
        [super observeValueForKeyPath: keyPath
                             ofObject: object
                               change: change
                              context: context];
    }
}

// The program can crash if an object is not removed as observer 
// before it is dealloc'd
//
// Helper method to remove self as an observer of the scrollView's
// contentOffset property
- (void)removeObserver {
    NSString *keyPath = NSStringFromSelector(@selector(contentOffset));
    [self.scrollView removeObserver: self forKeyPath: keyPath];
}

@end

Consultando dados KVC #

if ([[dataObject objectForKey:@"yourVariable"] isEqualToString:"Hello World"]) {
    return YES;
} else {
    return NO;
}

Você pode consultar valores armazenados usando o KVC de maneira rápida e fácil, sem precisar recuperá-los ou convertê-los como variáveis ​​locais.

Operadores de coleção #

Operadores de coleção podem ser usados ​​em um caminho de chave KVC para executar uma operação em uma propriedade de “tipo de coleção” (ou seja, NSArray, NSSet e similares). Por exemplo, uma operação comum a ser executada é contar os objetos em uma coleção. Para conseguir isso, você usa o @count operador de coleção:

self.array = @[@5, @4, @3, @2, @1];
NSNumber *count = [self.array valueForKeyPath:@"@count"];
NSNumber *countAlt = [self valueForKeyPath:@"array.@count"];
// count == countAlt == 5

Embora isso seja completamente redundante aqui (poderíamos ter acessado a propriedade count), pode ser útil em algumas ocasiões, embora raramente seja necessário. Existem, no entanto, alguns operadores de coleção que são muito mais úteis, como @max, @min, @sum, @avg e a família @unionOf. É importante observar que esses operadores também exigem um caminho de chave separado seguindo o operador para funcionar corretamente. Aqui está uma lista deles e o tipo de dados com os quais eles trabalham:

| Operador | Tipo de dados | | ——– | ————————————————– ———– | | @conta | (nenhum) | | @max | NSNumber, NSDate, int (e relacionados), etc. | | @min | NSNumber, NSDate, int (e relacionados), etc. | | @soma | NSNumber, int (e relacionado), double (e relacionado), etc. | | @avg | NSNumber, int (e relacionado), double (e relacionado), etc. | | @unionOfObjects | NSArray, NSSet, etc. | | @distinctUnionOfObjects | NSArray, NSSet, etc. | | @unionOfArrays | NSArray<NSArray*> | | @distinctUnionOfArrays | NSArray<NSArray*> | | @distinctUnionOfSets | NSSet<NSSet*> |

@max e @min retornarão o valor mais alto ou mais baixo, respectivamente, de uma propriedade de objetos na coleção. Por exemplo, observe o código a seguir:

// “Point” class used in our collection
@interface Point : NSObject

@property NSInteger x, y;

+ (instancetype)pointWithX:(NSInteger)x y:(NSInteger)y;

@end

...

self.points = @[[Point pointWithX:0 y:0],
                [Point pointWithX:1 y:-1],
                [Point pointWithX:5 y:-6],
                [Point pointWithX:3 y:0],
                [Point pointWithX:8 y:-4],
];

NSNumber *maxX = [self valueForKeyPath:@"points.@max.x"];
NSNumber *minX = [self valueForKeyPath:@"points.@min.x"];
NSNumber *maxY = [self valueForKeyPath:@"points.@max.y"];
NSNumber *minY = [self valueForKeyPath:@"points.@min.y"];

NSArray<NSNumber*> *boundsOfAllPoints = @[maxX, minX, maxY, minY];

...

Em apenas 4 linhas de código e puro Foundation, com o poder dos operadores de coleção Key-Value Coding, conseguimos extrair um retângulo que encapsula todos os pontos do nosso array.

É importante notar que essas comparações são feitas invocando o método compare: nos objetos, então se você quiser tornar sua própria classe compatível com esses operadores, você deve implementar este método.

@sum irá, como você provavelmente pode adivinhar, somar todos os valores de uma propriedade.

@interface Expense : NSObject

@property NSNumber *price;

+ (instancetype)expenseWithPrice:(NSNumber *)price;

@end

...

self.expenses = @[[Expense expenseWithPrice:@1.50],
                  [Expense expenseWithPrice:@9.99],
                  [Expense expenseWithPrice:@2.78],
                  [Expense expenseWithPrice:@9.99],
                  [Expense expenseWithPrice:@24.95]
];

NSNumber *totalExpenses = [self valueForKeyPath:@"expenses.@sum.price"];

Aqui, usamos @sum para encontrar o preço total de todas as despesas no array. Se, em vez disso, quisermos encontrar o preço médio que estamos pagando por cada despesa, podemos usar @avg:

NSNumber *averagePrice = [self valueForKeyPath:@"expenses.@avg.price"];

Finalmente, há a família @unionOf. Existem cinco operadores diferentes nesta família, mas todos funcionam basicamente da mesma forma, com apenas pequenas diferenças entre cada um. Primeiro, há @unionOfObjects que retornará uma matriz das propriedades dos objetos em uma matriz:

// See "expenses" array above

NSArray<NSNumber*> *allPrices = [self valueForKeyPath:
    @"expenses.@unionOfObjects.price"];

// Equal to @[ @1.50, @9.99, @2.78, @9.99, @24.95 ]

@distinctUnionOfObjects funciona da mesma forma que @unionOfObjects, mas remove duplicatas:

NSArray<NSNumber*> *differentPrices = [self valueForKeyPath:
    @"expenses.@distinctUnionOfObjects.price"];

// Equal to @[ @1.50, @9.99, @2.78, @24.95 ]

E, finalmente, os últimos 3 operadores da família @unionOf irão um passo além e retornarão uma matriz de valores encontrados para uma propriedade contida em matrizes aninhadas duplamente:

NSArray<NSArray<Expense*,Expense*>*> *arrayOfArrays =
    @[
        @[ [Expense expenseWithPrice:@19.99],
           [Expense expenseWithPrice:@14.95],
           [Expense expenseWithPrice:@4.50],
           [Expense expenseWithPrice:@19.99]
         ],

        @[ [Expense expenseWithPrice:@3.75],
           [Expense expenseWithPrice:@14.95]
         ]
     ];

// @unionOfArrays
NSArray<NSNumber*> allPrices = [arrayOfArrays valueForKeyPath:
    @"@unionOfArrays.price"];
// Equal to @[ @19.99, @14.95, @4.50, @19.99, @3.75, @14.95 ];

// @distinctUnionOfArrays
NSArray<NSNumber*> allPrices = [arrayOfArrays valueForKeyPath:
    @"@distinctUnionOfArrays.price"];
// Equal to @[ @19.99, @14.95, @4.50, @3.75 ];

O que está faltando neste exemplo é @distinctUnionOfSets, porém isso funciona exatamente da mesma forma que @distinctUnionOfArrays, mas funciona com e retorna NSSets em vez disso (não existe uma versão não-distinta porque em um conjunto, cada objeto deve ser distinto de qualquer maneira).

E é isso! Os operadores de coleção podem ser realmente poderosos se usados ​​corretamente e podem ajudar a evitar ter que percorrer coisas desnecessariamente.

Uma última observação: você também pode usar os operadores de coleção padrão em arrays de NSNumbers (sem acesso adicional à propriedade). Para isso, você acessa a pseudo-propriedade self que apenas retorna o objeto:

NSArray<NSNumber*> *numbers = @[@0, @1, @5, @27, @1337, @2048];

NSNumber *largest = [numbers valueForKeyPath:@"@max.self"];
NSNumber *smallest = [numbers valueForKeyPath:@"@min.self"];
NSNumber *total = [numbers valueForKeyPath:@"@sum.self"];
NSNumber *average = [numbers valueForKeyPath:@"@avg.self"];
