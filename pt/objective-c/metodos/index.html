Sintaxe #

  • - ou +: O tipo de método. Instância ou classe?

  • (): Para onde vai o tipo de retorno. Use void se não quiser devolver nada!

  • Em seguida é o nome do método. Use camelCase e torne o nome fácil de lembrar e entender.

  • Se o seu método precisa de parâmetros, agora é a hora! O primeiro parâmetro vem logo após o nome da função como este :(type)parameterName. Todos os outros parâmetros são feitos desta forma parameterLabel:(type)parameterName

  • O que o seu método faz? Coloque tudo aqui, entre chaves {}!

Métodos de classe #

Um método de classe é chamado na classe à qual o método pertence, não em uma instância dele. Isso é possível porque as classes Objective-C também são objetos. Para denotar um método como um método de classe, altere o - para um +:

+ (void)hello {
  NSLog(@"Hello World");
}

Parâmetros do método #

Se você quiser passar valores para um método quando ele for chamado, use os parâmetros:

- (int)addInt:(int)intOne toInt:(int)intTwo {
  return intOne + intTwo;
}

Os dois pontos (:) separam o parâmetro do nome do método.

O tipo de parâmetro vai entre parênteses (int).

O nome do parâmetro vem após o tipo de parâmetro.

Crie um método básico #

Veja como criar um método básico que registra ‘Hello World" no console:

- (void)hello {
  NSLog(@"Hello World");
}

O - no início denota este método como um método de instância.

O (void) denota o tipo de retorno. Este método não retorna nada, então você digita void.

O ‘hello’ é o nome do método.

Tudo no {} é o código executado quando o método é chamado.

Valores de retorno #

Quando você deseja retornar um valor de um método, você coloca o tipo que deseja retornar no primeiro conjunto de parênteses.

- (NSString)returnHello {
  return @"Hello World";
}

O valor que você quer retornar vai depois da palavra-chave return;

Métodos de chamada #

Chamando um método de instância:

[classInstance hello];

 @interface Sample
 -(void)hello; // exposing the class Instance method
 @end

 @implementation Sample
     -(void)hello{
        NSLog(@"hello");
      }
 @end

Chamando um método de instância na instância atual:

[self hello];

@implementation Sample

     -(void)otherMethod{
       [self hello];
     }

     -(void)hello{
        NSLog(@"hello");
      }
 @end

Chamando um método que recebe argumentos:

[classInstance addInt:1 toInt:2];

 @implementation Sample
     -(void)add:(NSInteger)add to:(NSInteger)to
        NSLog(@"sum = %d",(add+to));
      }
 @end

Chamando um método de classe:

[Class hello];

 @interface Sample
 +(void)hello; // exposing the class method
 @end

 @implementation Sample
     +(void)hello{
        NSLog(@"hello");
      }
 @end

Métodos de instância #

Um método de instância é um método que está disponível em uma instância específica de uma classe, após a instância ter sido instanciada:

MyClass *instance = [MyClass new];
[instance someInstanceMethod];

Veja como você define um:

@interface MyClass : NSObject

- (void)someInstanceMethod; // "-" denotes an instance method

@end

@implementation MyClass

- (void)someInstanceMethod {
    NSLog(@"Whose idea was it to have a method called \"someInstanceMethod\"?");
}

@end

Passar por passagem de parâmetro de valor #

Na passagem por valor do parâmetro passando para um método, o valor real do parâmetro é copiado para o valor do parâmetro formal. Portanto, o valor real do parâmetro não será alterado após retornar da função chamada.

@interface SwapClass : NSObject

-(void) swap:(NSInteger)num1 andNum2:(NSInteger)num2;

@end

@implementation SwapClass

-(void) num:(NSInteger)num1 andNum2:(NSInteger)num2{
    int temp;
    temp = num1;
    num1 = num2;
    num2 = temp;
}
@end

Chamando os métodos:

NSInteger a = 10, b =20;
SwapClass *swap = [[SwapClass alloc]init];
NSLog(@"Before calling swap: a=%d,b=%d",a,b);
[swap num:a andNum2:b];
NSLog(@"After calling swap: a=%d,b=%d",a,b);

Resultado:

2016-07-30 23:55:41.870 Test[5214:81162] Before calling swap: a=10,b=20
2016-07-30 23:55:41.871 Test[5214:81162] After calling swap: a=10,b=20

Passar por passagem de parâmetro de referência #

Na passagem por referência do parâmetro passando para um método, o endereço do parâmetro real é passado para o parâmetro formal. Portanto, o valor real do parâmetro será alterado após retornar da função chamada.

@interface SwapClass : NSObject

-(void) swap:(int)num1 andNum2:(int)num2;

@end

@implementation SwapClass

-(void) num:(int*)num1 andNum2:(int*)num2{
    int temp;
    temp = *num1;
    *num1 = *num2;
    *num2 = temp;
}
@end

Chamando os métodos:

int a = 10, b =20;
SwapClass *swap = [[SwapClass alloc]init];
NSLog(@"Before calling swap: a=%d,b=%d",a,b);
[swap num:&a andNum2:&b];
NSLog(@"After calling swap: a=%d,b=%d",a,b);

Resultado:

2016-07-31 00:01:47.067 Test[5260:83491] Before calling swap: a=10,b=20
2016-07-31 00:01:47.070 Test[5260:83491] After calling swap: a=20,b=10
