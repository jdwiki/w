Sintaxe #

  • @property (atributos_opcionais, …) tipo identificador;
  • @synthesize identificador = opcional_backing_ivar;
  • @dynamic identificador;

Parâmetros #

Atributo Descrição
atômica Implicit. Habilita a sincronização em métodos de acessor sintetizados.
não atômico Desativa a sincronização nos métodos do acessador sintetizado.
readwrite Implicit. Sintetiza getter, setter e backing ivar.
somente leitura Sintetiza apenas o método getter e o ivar de apoio, que podem ser atribuídos diretamente.
getter=nome Especifica o nome do método getter, implícito é propertyName.
setter=nome Especifica o nome do método setter, implicitamente é setPropertyName:. Dois pontos : devem fazer parte do nome.
forte Implícito para objetos sob ARC. O ivar de apoio é sintetizado usando __strong, que evita a desalocação do objeto referenciado.
reter Sinónimo de ‘forte’.
copiar O mesmo que strong, mas o setter sintetizado também chama -copy no novo valor.
unsafe_unretained Implícito, exceto para objetos sob ARC. O backing ivar é sintetizado usando __unsafe_unretained, que (para objetos) resulta em um ponteiro pendente quando o objeto referenciado é desalocado.
atribuir Sinônimo de unsafe_unretained. Adequado para tipos não-objetos.
fraco Backing ivar é sintetizado usando __weak, então o valor será anulado assim que o objeto referenciado for desalocado.
classe Os acessadores de propriedade são sintetizados como métodos de classe, em vez de métodos de instância. Nenhum armazenamento de apoio é sintetizado.
anulável A propriedade aceita valores nil. Usado principalmente para pontes Swift.
não nulo A propriedade não aceita valores nil. Usado principalmente para pontes Swift.
null_resettable A propriedade aceita valores nil em setter, mas nunca retorna valores nil de getter. Sua implementação personalizada de getter ou setter deve garantir esse comportamento. Usado principalmente para pontes Swift.
null_unspecified Implicit. A propriedade não especifica manipulação de valores nil. Usado principalmente para pontes Swift.

Getters e setters personalizados #

Os getters e setters de propriedade padrão podem ser substituídos:

@interface TestClass

@property NSString *someString;

@end

@implementation TestClass

// override the setter to print a message
- (void)setSomeString:(NSString *)newString {
    NSLog(@"Setting someString to %@", newString);
    // Make sure to access the ivar (default is the property name with a _ 
    // at the beginning) because calling self.someString would call the same
    // method again leading to an infinite recursion
    _someString = newString;
}

- (void)doSomething {
    // The next line will call the setSomeString: method
    self.someString = @"Test";
}

@end

Isso pode ser útil para fornecer, por exemplo, inicialização lenta (substituindo o getter para definir o valor inicial se ainda não tiver sido definido):

- (NSString *)someString {
    if (_someString == nil) {
        _someString = [self getInitialValueForSomeString];
    }
    return _someString;
}

Você também pode fazer uma propriedade que calcula seu valor no getter:

@interface Circle : NSObject

@property CGPoint origin;
@property CGFloat radius;
@property (readonly) CGFloat area;

@end

@implementation Circle

- (CGFloat)area {
    return M_PI * pow(self.radius, 2);
}

@end

O que são propriedades? #

Aqui está uma classe de exemplo que tem algumas variáveis ​​de instância, sem usar propriedades:

@interface TestClass : NSObject {
    NSString *_someString;
    int _someInt;
}

-(NSString *)someString;
-(void)setSomeString:(NSString *)newString;

-(int)someInt;
-(void)setSomeInt:(NSString *)newInt;

@end


@implementation TestClass

-(NSString *)someString {
    return _someString;
}

-(void)setSomeString:(NSString *)newString {
    _someString = newString;
}

-(int)someInt {
    return _someInt;
}

-(void)setSomeInt:(int)newInt {
    _someInt = newInt;
}

@end

Isso é muito código clichê para criar uma variável de instância simples. Você precisa criar a variável de instância e criar métodos de acesso que não fazem nada, exceto definir ou retornar a variável de instância. Assim, com o Objective-C 2.0, a Apple introduziu propriedades, que geram automaticamente parte ou todo o código clichê.

Aqui está a classe acima reescrita com propriedades:

@interface TestClass

@property NSString *someString;
@property int someInt;

@end


@implementation testClass

@end

Uma propriedade é uma variável de instância emparelhada com getters e setters gerados automaticamente. Para uma propriedade chamada someString, o getter e o setter são chamados someString e setSomeString: respectivamente. O nome da variável de instância é, por padrão, o nome da propriedade prefixada com um sublinhado (portanto, a variável de instância para someString é chamada de _someString, mas isso pode ser substituído por uma diretiva @synthesize no arquivo seção @implementation:

@synthesize someString=foo;    //names the instance variable "foo"
@synthesize someString;    //names it "someString"
@synthesize someString=_someString;        //names it "_someString"; the default if 
                                           //there is no @synthesize directive

As propriedades podem ser acessadas chamando os getters e setters:

[testObject setSomeString:@"Foo"];
NSLog(@"someInt is %d", [testObject someInt]);

Eles também podem ser acessados ​​usando notação de ponto:

testObject.someString = @"Foo";
NSLog(@"someInt is %d", testObject.someInt);

Propriedades que causam atualizações #

Este objeto, Shape tem uma propriedade image que depende de numberOfSides e sideWidth. Se qualquer um deles estiver definido, a imagem deve ser recalculada. Mas o recálculo é presumivelmente longo e só precisa ser feito uma vez se ambas as propriedades estiverem definidas, então o Shape fornece uma maneira de definir ambas as propriedades e recalcular apenas uma vez. Isso é feito definindo a propriedade ivars diretamente.

Em Shape.h

@interface Shape {
    NSUInteger numberOfSides;
    CGFloat sideWidth;

    UIImage * image;
}

// Initializer that takes initial values for the properties.
- (instancetype)initWithNumberOfSides:(NSUInteger)numberOfSides withWidth:(CGFloat)width;

// Method that allows to set both properties in once call.
// This is useful if setting these properties has expensive side-effects.
// Using a method to set both values at once allows you to have the side-
// effect executed only once.
- (void)setNumberOfSides:(NSUInteger)numberOfSides andWidth:(CGFloat)width;

// Properties using default attributes.
@property NSUInteger numberOfSides;
@property CGFloat sideWidth;

// Property using explicit attributes.
@property(strong, readonly) UIImage * image;

@end

Em Shape.m

@implementation AnObject

// The variable name of a property that is auto-generated by the compiler
// defaults to being the property name prefixed with an underscore, for
// example "_propertyName". You can change this default variable name using
// the following statement:
// @synthesize propertyName = customVariableName;

- (id)initWithNumberOfSides:(NSUInteger)numberOfSides withWidth:(CGFloat)width {
    if ((self = [self init])) {
       [self setNumberOfSides:numberOfSides andWidth:width];
    }

    return self;
}

- (void)setNumberOfSides:(NSUInteger)numberOfSides {
    _numberOfSides = numberOfSides;

    [self updateImage];
}

- (void)setSideWidth:(CGFloat)sideWidth {
    _sideWidth = sideWidth;

    [self updateImage];
}

- (void)setNumberOfSides:(NSUInteger)numberOfSides andWidth:(CGFloat)sideWidth {
    _numberOfSides = numberOfSides;
    _sideWidth = sideWidth;

    [self updateImage];
}

// Method that does some post-processing once either of the properties has
// been updated.
- (void)updateImage {
    ...
}

@end

Quando as propriedades são atribuídas (usando object.property = value), o método setter setProperty: é chamado. Este setter, mesmo se fornecido por @synthesize, pode ser substituído, como neste caso para numberOfSides e sideWidth. No entanto, se você definir o ivar de uma propriedade diretamente (através de property se o objeto for self, ou object->property), ele não chamará o getter ou setter, permitindo que você faça coisas como vários conjuntos de propriedades que apenas chamar uma atualização ou ignorar os efeitos colaterais causados ​​pelo setter.
