Criar projeto Vaadin com Maven #

Com o Maven você pode criar um projeto Vaadin com o arquétipo vaadin-archetype-application. Você também pode adicionar esse arquétipo no IDE para criar um projeto maven com o IDE.

mvn archetype:generate 
   -DarchetypeGroupId=com.vaadin 
   -DarchetypeArtifactId=vaadin-archetype-application 
   -DarchetypeVersion=7.6.8 
   -DgroupId=myvaadin.project 
   -DartifactId=DemoVaadinProject 
   -Dversion=0.1 
   -Dpackaging=war

Depois de executar o comando acima, você terá a seguinte estrutura de projeto.

DemoVaadinProject 
  |-src
     |-main
         |-java
         |   |-myvaadin
         |         |-project
         |            |-MyUI.java
         |-resource
         |    |-myvaadin
         |         |-project
         |            |-MyAppWidgetset.gwt.xml
         |-webapps
              |- VAADIN
                   |-theme   
                      |- mytheme.scss
                      |- addons.scss
                      |- styles.scss
                      |- favicon.ico

O projeto maven padrão criado pode ser importado diretamente no IDE. Para executar o aplicativo maven, devemos compilar os conjuntos de widgets padrão do vaadin.

Observe que podemos usar diretamente o seguinte comando maven para empacotar o aplicativo vaadin e ele compilará os conjuntos de widgets por padrão. Você pode usar o plugin maven jetty para implantar o aplicativo vaadin no Jetty.

cd path/to/DemoVaadinProject
mvn package jetty:run

Isso implantará o aplicativo padrão e começará a executá-lo na porta padrão 8080. Você pode acessar o aplicativo implementado em http://localhost:8080.

Ele está pronto para ser executado sem nenhuma alteração. Por padrão, o arquétipo Vaadin adiciona o tema padrão, widgetset xml e a classe MyUI, que é um ponto de entrada para o aplicativo vaadin.

No navegador veremos o seguinte formulário.

Criar projeto Vaadin no Eclipse #

O plug-in Vaadin para eclipse fornece uma maneira rápida de construir um projeto vaadin com o gerenciador de dependências Apache Ivy. A documentação do Vaadin explica como criar um projeto vaadin com a ajuda do plugin Eclipse.

Para instalar o plug-in basta ir ao Eclipse Marketplace e pesquisar vaadin. A versão atual do plug-in é 3.0.0.

Depois de instalar o plug-in, você terá os seguintes recursos rápidos,

  • Criar projeto Vaadin6 ou Vaadin 7 (Gerenciador de dependência padrão é Ivy)
  • Compilar Widgetsets (Para compilar widgets do lado do cliente)
  • Compilar Tema (Para compilar Tema para construir CSS final)
  • Criar Widget (Para construir seu widget personalizado)
  • Criar tema Vaadin

Assim, após configurar o plug-in, basta criar um novo projeto Vaadin com configuração mínima. Você também pode especificar a versão do vaadin ao criar o projeto.

  • Arquivo > Novo > Projeto Vaadin7
  • Especifique a versão do vaadin a ser usada no projeto
  • Especifique o Target Runtime que você deseja usar
  • Terminar!

Levará tempo para baixar todos os jars necessários para o vaadin, uma vez que Ivy resolva todas as dependências. Você pode executar o projeto diretamente no servidor e verá um Button com Click Me na tela do navegador. Observe que o Vaadin7 é compatível com Java 6 e mais recente.

Plugin Vaadin para Netbeans #

Criando um projeto com o NetBeans IDE

A seguir, orientamos você na criação de um projeto Vaadin no NetBeans e mostramos como executá-lo.

A instalação do NetBeans e do plug-in Vaadin é abordada em Instalando o NetBeans IDE e o plug-in.

Sem o plugin, você pode criar mais facilmente um projeto Vaadin como um projeto Maven usando um arquétipo Vaadin. Você também pode criar um projeto Vaadin como um projeto de aplicativo da Web comum, mas requer muitas etapas manuais para instalar todas as bibliotecas Vaadin, criar a classe de interface do usuário, configurar o servlet, criar tema e assim por diante.

Criando um projeto #

  1. Selecione Arquivo ▸ Novo Projeto… no menu principal ou pressione Ctrl+Shift+N.

  2. Na janela Novo projeto que se abre, selecione o Vaadin category and one of the Vaadin archetypes from the right.

    The archetypes are described in more detail in Overview of Maven Archetypes.

  3. Na etapa Name and Location, insira os parâmetros do projeto.

    Project Name

    A project name. The name must be a valid identifier that may only contains alphanumerics, minus, and underscore. It is appended to the group ID to obtain the Java package name for the sources.

    Project Location

    Path to the folder where the project is to be created.

    Group Id

    A Maven group ID for your project. It is normally your organization domain name in reverse order, such as com.example. The group ID is also used as a prefix for the Java source package, so it should be Java-compatible package name.

    Version

    Initial version of your application. The number must obey the Maven version numbering format.

    Package

    The Java package name to put sources in.

    Additional Creation Properties

    The properties control various names. They are specific to the archetype you chose.

    Click Finish.

    Creating the project can take a while as Maven loads all the needed dependencies.

Explorando o projeto #

O assistente de projeto fez todo o trabalho para você: um esqueleto de classe de interface do usuário foi gravado no diretório src. A hierarquia do projeto mostrada no Project Explorer é mostrada em Um novo projeto Vaadin no NetBeans. Figura 1. Um novo projeto Vaadin no NetBeans

meu tema

O tema da interface do usuário. Consulte Temas para obter informações sobre temas.

MyUI.java

A classe de interface do usuário, que é o principal ponto de entrada do seu aplicativo. Consulte Aplicativos do lado do servidor para obter informações sobre a estrutura básica dos aplicativos Vaadin.

As bibliotecas Vaadin e outras dependências são gerenciadas pelo Maven. Observe que as bibliotecas não são armazenadas na pasta do projeto, embora estejam listadas na pasta virtual Recursos Java ▸ Bibliotecas ▸ Maven Dependências. Executando o Aplicativo

Uma vez criado, você pode executá-lo em um servidor da seguinte maneira.

  1. Na guia Projetos, selecione o projeto e clique no botão Executar Projeto button in the tool bar (or press F6).

  2. Na janela Selecionar servidor de implementação, selecione um servidor na lista Servidor. Deve mostrar GlassFish ou Apache Tomcat ou both, depending on what you chose in NetBeans installation.

    Also, select Remember Permanently if you want to use the same server also in future while developing applications.

    Click OK.

O conjunto de widgets será compilado neste ponto, o que pode demorar um pouco.

Se tudo correr bem, o NetBeans inicia o servidor na porta 8080 e, dependendo da configuração do sistema, inicia o navegador padrão para exibir o aplicativo da web. Caso contrário, você pode abri-lo manualmente, por exemplo, em http://localhost:8080/myproject. O nome do projeto é usado por padrão como o caminho de contexto do aplicativo.

Agora, quando você editar a classe de interface do usuário no editor de origem e salvá-la, o NetBeans reimplantará automaticamente o aplicativo. Depois de terminar após alguns segundos, você pode recarregar o aplicativo no navegador.

Primeiro programa - “Hello World” #

Copie e cole este código e inicie seu programa:

@Theme(ValoTheme.THEME_NAME) //[optional] adds Vaadin built in theming 
public class SampleUI extends UI {
    
@Override
protected void init(VaadinRequest request) {
         final VerticalLayout rootLayout = new VerticalLayout();
         Label label = new Label("Hello World"!);
         rootLayout.addComponent(label);
         setContent(rootLayout);
     }
 }

Após a inicialização ser bem-sucedida, navegue até localhost:8080/yourApplicationName ou http://localhost:8080/ para ver se seu aplicativo está funcionando.
