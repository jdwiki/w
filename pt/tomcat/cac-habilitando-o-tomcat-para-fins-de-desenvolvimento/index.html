Criando os Keystores e configurando o Tomcat #

Este writeup percorre as etapas para configurar o Tomcat para solicitar certificados CAC do cliente. Ele é focado na configuração de um ambiente de desenvolvimento, portanto, alguns recursos que devem ser considerados para produção não estão aqui. (Por exemplo, ele mostra o uso de um certificado autoassinado para https e não considera a verificação de certificados revogados.)

Crie Keystore para habilitar conexões HTTPS

O primeiro passo é configurar o SSL no tomcat. Isso está documentado no site do tomcat aqui: https://tomcat.apache.org/tomcat-8.5-doc/ssl-howto.html para completar, as etapas para configurá-lo com um certificado autoassinado estão listadas abaixo:

Precisamos criar um arquivo keystore que contenha o certificado SSL para o servidor. O certificado é o que é necessário para criar uma conexão https e não tem nada a ver com fazer com que o servidor solicite certificados CAC do cliente, mas as conexões https são necessárias para a autenticação do certificado do cliente. Para um ambiente de desenvolvimento, criar um certificado autoassinado é aceitável, mas é desencorajado para produção. O Java vem com um utilitário chamado keytool (http://docs.oracle.com/javase/8/docs/technotes/tools/windows/keytool.html) que é usado para gerenciar certificados e keystores. Ele pode ser usado para criar um certificado autoassinado e adicioná-lo a um keystore. Para fazer isso, você pode emitir o seguinte comando em um prompt de comando:

keytool -genkey -alias tomcat -keyalg RSA -keystore \path\to\my\keystore -storepass changeit

Você será solicitado a fornecer várias informações e, em seguida, um arquivo de armazenamento de chaves chamado “\path\to\my\keystore” com uma senha de ‘changeit’ será criado e conterá o certificado autoassinado gerado.

Crie um armazenamento confiável contendo certificados raiz do DoD

A próxima coisa necessária é criar um armazenamento confiável que conterá os certificados raiz do DoD. Os certificados neste armazenamento confiável serão considerados confiáveis ​​pelo tomcat e ele só aceitará certificados de cliente que tenham um dos certificados confiáveis ​​em sua cadeia de certificados.

Para criar o armazenamento confiável, precisamos obter uma cópia dos certificados raiz do DoD. Para fazer isso, baixe “InstallRoot 5.0” de http://militarycac.com/dodcerts.htm. Instale-o e execute-o, em seguida, execute-o. Expanda o painel Instalar certificados DoD e clique na guia Certificado:

Em seguida, selecione os três certificados DoD Root CA na lista de certificados e clique em “PEM” no grupo de ferramentas de exportação:

Depois de clicar no botão de exportação “PEM”, escolha um local para exportar os certificados e clique em OK. Isso deve ter criado três arquivos .cer no diretório selecionado. Abra um prompt de comando e navegue até esse diretório.

Aqui usaremos o comando keytool para importar os certificados para um armazenamento confiável. Execute os seguintes comandos para importar os três certificados:

keytool -importcert -file DoD_Root_CA_2__0x05__DoD_Root_CA_2.cer -alias DODRoot2 -keystore truststore.jks -storepass changeit

keytool -importcert -file DoD_Root_CA_3__0x01__DoD_Root_CA_3.cer -alias DODRoot3 -keystore truststore.jks -storepass changeit

keytool -importcert -file DoD_Root_CA_4__0x01__DoD_Root_CA_4.cer -alias DODRoot4 -keystore truststore.jks -storepass changeit

Isso criará um arquivo truststore.jks com uma senha de ‘changeit’ no diretório de trabalho atual. Ele conterá os três DoD Root Certs, você pode ver isso executando:

keytool -list -keystore truststore.jks

Que deve listar algo como:

Seu keystore contém 3 entradas

Configure o Tomcat para usar o Keystore e o Truststore

Agora temos os arquivos keystore e truststore que precisamos, a seguir é configurar o tomcat para usá-los. Para isso devemos alterar o arquivo /conf/server.xml. Abra o arquivo e adicione uma definição de conector como esta:

<Connector 
    clientAuth="true" 
    keystoreFile="path/to/keystore.jks" 
    keystorepass="changeit" 
    keystoreType="jks" 
    truststoreFile="path/to/truststore.jks" 
    truststoreType="jks" 
    truststorepass="changeit"
    maxThreads="150" 
    port="8443" 
    protocol="org.apache.coyote.http11.Http11NioProtocol" 
    scheme="https" 
    secure="true" 
    sslProtocol="TLS" 
    SSLEnabled="true" 
    />

Você pode ir aqui para mais definições de todos os atributos: http://tomcat.apache.org/tomcat-7.0-doc/config/http.html

Feito isso, inicie o tomcat. A partir de um computador que tenha um leitor CAC com um CAC inserido, navegue até https://:8443/ url e, se tudo estiver configurado corretamente, você deverá ser solicitado a escolher um certificado do cartão CAC.
