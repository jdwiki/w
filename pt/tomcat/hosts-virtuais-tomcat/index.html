Host Manager é um aplicativo da web dentro do Tomcat que cria/remove Hosts Virtuais dentro do Tomcat.

Um Host Virtual permite que você defina vários nomes de host em um único servidor, para que você possa usar o mesmo servidor para lidar com solicitações para, por exemplo, ren.myserver.com e stimpy.myserver.com.

Infelizmente, a documentação do lado GUI do Host Manager não parece existir, mas a documentação sobre como configurar os hosts virtuais manualmente em context.xml está aqui:

http://tomcat.apache.org/tomcat-7.0-doc/virtual-hosting-howto.html.

A explicação completa dos parâmetros Host você pode encontrar aqui:

http://tomcat.apache.org/tomcat-7.0-doc/config/host.html.

Adaptado de minha resposta: http://stackoverflow.com/a/26248511/6340

Aplicativo da Web do Tomcat Host Manager #

O aplicativo Host Manager do Tomcat por padrão está localizado em http://localhost:8080/host-manager, mas não é acessível até que um usuário receba permissão no arquivo conf/tomcat-users.xml. O arquivo precisa ter:

  1. Uma função de manager-gui
  2. Um usuário com esta função

Por exemplo:

<tomcat-users>
    ...
    <role rolename="manager-gui"/>
    ....
    <user username="host-admin" password="secretPassword" roles="manager-gui"/>
</tomcat-users>

Adicionando um host virtual por meio do aplicativo da Web Tomcat Host Manager #

Assim que você tiver acesso ao gerenciador de host, a GUI permitirá que você adicione um host virtual.

Observação: No Tomcat 7 e 8, adicionar um host virtual por meio da GUI não grava o vhost nos arquivos de configuração. Você precisará editar manualmente o arquivo server.xml para ter o vhost disponível após uma reinicialização. Ver http://tomcat.apache.org/tomcat-7.0-doc/virtual-hosting-howto.html para mais informações sobre a tag <Host> em server.xml

No mínimo, você precisa dos campos Name e App Base definidos. O Tomcat irá então criar os seguintes diretórios:

{CATALINA_HOME}\conf\Catalina\{Name}
{CATALINA_HOME}\{App Base}
  • App Base será onde os aplicativos da web serão implantados no host virtual. Pode ser relativo ou absoluto.
  • Name geralmente é o nome de domínio totalmente qualificado (por exemplo, ren.myserver.com)
  • Alias pode ser usado para estender o Name também onde dois endereços devem resolver para o mesmo host (por exemplo, www.ren.myserver.com). Observe que isso precisa ser refletido nos registros DNS.

As caixas de seleção são as seguintes:

  • Auto Deploy: reimplante automaticamente os aplicativos colocados no App Base. Perigoso para ambientes de produção!
  • Deploy On Startup: inicializa automaticamente os aplicativos no App Base quando o Tomcat é iniciado
  • Deploy XML: Determina se deve analisar o /META-INF/context.xml do aplicativo
  • Descompactar WARs: Descompacte os arquivos WAR colocados ou carregados no App Base, em vez de executá-los diretamente do WAR.
  • Tomcat 8 Copy XML: Copie o META-INF/context.xml de um aplicativo para o App Base/XML Base na implantação e use-o exclusivamente, independentemente de o aplicativo ser atualizado. Irrelevante se Deploy XML for falso.
  • Manager App: Adicione o aplicativo gerenciador ao Virtual Host (Útil para controlar os aplicativos que você pode ter sob ren.myserver.com)

Adaptado de minha resposta: http://stackoverflow.com/a/26248511/6340

Adicionando um Host Virtual ao server.xml #

Depois que um host virtual for adicionado por meio do aplicativo da Web, os diretórios existirão em:

{CATALINA_HOME}\conf\Catalina\{Name}
{CATALINA_HOME}\{App Base}

Para persistir o host virtual após uma reinicialização, o arquivo server.xml deve ser atualizado com a configuração. Um elemento Host precisa ser adicionado dentro do elemento Engine, semelhante a este:

<Engine name="Catalina" ...>
  ...
  <Host name="my-virtual-app" appBase="virtualApp" autoDeploy="true" unpackWARs="true" ... />
</Engine>

Os atributos no elemento Host devem refletir as seleções feitas na GUI do gerenciador de host (consulte a documentação do Host para obter detalhes), mas podem ser alterados. Observe que a opção Manager App na GUI não corresponde a nenhum atributo Host.
