Mapeamentos XML #

O mapeamento xml usa um arquivo hbm.xml que é um arquivo de mapeamento de hibernação. É um arquivo xml de sintaxe que contém os metadados necessários para o mapeamento objeto/relacional. Os metadados incluem a declaração de classes persistentes e o mapeamento de propriedades (para colunas e relacionamentos de chave estrangeira com outras entidades) para tabelas de banco de dados.

Adicione um arquivo chamado Entity.hbm.xml no projeto e defina-o como recurso incorporado na guia de propriedades. Por exemplo, Customer.hbm.xml:

<?xml version="1.0" encoding="utf-8" ?>
<hibernate-mapping xmlns="urn:nhibernate-mapping-2.2"
    namespace="Project" assembly="Project">

    <class name="Customer" table="CUSTOMERS">

        <id name="Id">
            <column name="Customer_Id" sql-type="int" not-null="true"/>
            <generator class="native" />
        </id>

        <!-- A cat has to have a name, but it shouldn' be too long. -->
        <property name="Name">
            <column name="Name" length="60" not-null="true" />
        </property>
        <property name="Sex" />
        <property name="Weight" />
        <property name="Active" />
        <property name="Birthday" />
    </class>

</hibernate-mapping>

A tag hibernate-mapping contém as informações do namespace e do projeto de montagem. A tag class contém o nome da entidade no projeto e a tabela que está sendo mapeada. A tag id contém o mapeamento para a chave primária onde a coluna é especificada pela tag column e a tag generator define como o id é gerado. A tag property contém informações para as outras colunas do banco de dados.

Mapeamentos NHibernate Fluentes #

O Fluent NHibernate é uma biblioteca para ajudá-lo a mapear as entidades usando código C# em vez de mapeamentos xml. Fluent NHibernate usa o padrão fluente e é baseado em convenções para criar os mapeamentos e lhe dá o poder das ferramentas de estúdio visual (como o intellisense ) para melhorar a maneira como você mapeia suas entidades.

Adicione a referência do Fluent NHibernate from Nuget em seu projeto e adicione uma classe CustomerMap.cs:

namespace Project.Mappings
{
    public class CustomerMap : ClassMap<Customer>
    {
        public CustomerMap()
        {
            Table("CUSTOMERS");

            Id(x => x.Id).Column("Customer_Id").GeneratedBy.Native();
            
            //map a property while specifying the max-length as well as setting 
            //it as not nullable. Will result in the backing column having
            //these characteristics, but this will not be enforced in the model!
            Map(x => x.Name)
                .Length(16)
                .Not.Nullable();

            Map(x => x.Sex);

            Map(x => x.Weight);

            Map(x => x.Active);

            //Map a property while specifying the name of the column in the database
            Map(x => x.Birthday, "BIRTHDAY");

            //Maps a many-to-one relationship
            References(x => x.Company);

            //Maps a one-to-many relationship, while also defining which 
            //column to use as key in the foreign table.
            HasMany(x => x.Orders).KeyColumn("CustomerPk");
        }
    }
}

A classe CustomerMap é derivada de ClassMap<T> que é a classe base para mapeamento e contém todos os métodos necessários para criar o mapa de sua entidade T. O método Table define o nome da tabela que você está mapeando. O método Id é usado para mapear a coluna primery key. O método Map é usado para mapear outras colunas.

Uma amostra do modelo para mapear #

NHibernate usa classes para mapear em tabelas ou visualizações. Criar um Plain Old CLR Object (POCOs, às vezes chamado de Plain Ordinary CLR Objects) é uma boa prática para classes persistentes. Um POCO tem seus dados acessíveis através dos mecanismos de propriedade padrão .NET, protegendo a representação interna da interface publicamente visível.

namespace Project
{
    public class Customer
    {
        public virtual string Id { get; set; }

        public virtual string Name { get; set; }

        public virtual char Sex { get; set; }

        public virtual float Weight { get; set;}

        public virtual bool Active { get; set;}

        public virtual DateTime Birthday { get; set;}

        public Customer()
        {
        }
    }
}

NHibernate não é restrito em seu uso de tipos de propriedade: todos os tipos .NET e primitivos (como string, char e DateTime) podem ser mapeados, incluindo classes dos namespaces System.Collections e System.Collections.Generics. Você também pode mapear uma relação entre as entidades, tendo propriedades que se referem a outro tipo de entidade. Você pode mapeá-los como valores, coleções de valores ou associações a outras entidades. A propriedade chamada Id aqui é uma propriedade especial que representa o identificador do banco de dados (chave primária) dessa classe, o que é altamente recomendado para entidades como um Cat. O NHibernate pode usar identificadores apenas internamente, sem precisar declará-los na classe, mas perderíamos um pouco da flexibilidade em nossa arquitetura de aplicativo.

Nenhuma interface especial precisa ser implementada para classes persistentes nem temos que subclassificar de uma classe persistente raiz especial. O NHibernate também não usa nenhum processamento de tempo de compilação, como manipulação de IL; ele se baseia apenas na reflexão .NET e no aprimoramento da classe de tempo de execução. Assim, sem nenhuma dependência da classe POCO no NHibernate, podemos mapeá-la para uma tabela ou visualização do banco de dados.

Para que o aprimoramento de classe de tempo de execução mencionado acima funcione, o NHibernate requer que todas as propriedades públicas de uma classe de entidade sejam declaradas como ‘virtuais’. A classe de entidade deve ter um construtor sem argumentos (protected ou public) para que o NHibernate crie os objetos.
