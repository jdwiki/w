A instalação do Web Deploy permite a instalação rápida de aplicativos Web diretamente do ambiente de desenvolvimento usando o Visual Studio.

Tipos de instalação:

online - o servidor Windows de destino tem acesso à Internet - a instalação é simples usando Instalando e configurando a implantação da Web em servidores com conexão à Internet exemplo abaixo

offline - o servidor Windows de destino está offline - a instalação é bastante complexa porque vários componentes e configurações devem ser feitos manualmente (ou com script)

Instalando o WebDeploy no IIS com ASP.NET #

  1. Instale a função do servidor Web
  2. Instale o WebDeploy 3.6 do MSDN
  3. Ative o ASP.NET 4.6 em Web Server (IIS) > Web Server > Application Development

Instalando e configurando a implantação da Web em servidores com conexão com a Internet #

Para poder implantar diretamente as alterações do projeto da Web em um servidor da Web, as etapas a seguir devem ser seguidas. Se o servidor de destino tiver acesso à Internet, o processo é bastante simples, pois a Microsoft possui um pacote Web Platform para fazer quase tudo o que é necessário.

  1. Limpeza

Certifique-se de que C:\Program Files\IIS não contém uma versão mais antiga do Microsoft Web Deploy. Se contiver, desinstale-o (eles) de Adicionar/remover programas e remova todos os arquivos restantes

  1. Instalador da plataforma Web

Instale o instalador da plataforma Web (WPI) de aqui

  1. Instalação de implantação na Web

Execute o WPI, procure o Web Deploy 3.6 for Hosting Servers e instale tudo o que ele contém. Ele terá o cuidado de instalar todas as dependências necessárias (por exemplo, para implantação de SQL) e também deixar o IIS pronto para ser configurado para implantações da Web

  1. Configuração do IIS

O IIS deve ser configurado para permitir implantações da Web em um ou mais de seus sites.

  • Direitos: acessar o site -> Permissões do gerenciador do IIS e configurar quais usuários têm permissão para publicar. OBSERVAÇÃO: em alguma configuração específica, tentar selecionar um usuário pode levar ao travamento do gerenciador do IIS. Nesses casos raros, basta inserir o nome de usuário manualmente - por exemplo, Domínio \ nome de usuário

  • Configuração de publicação de implantação na Web - Clique com o botão direito do mouse no site -> Implantar… -> Configurar publicação de implantação na Web

OBSERVAÇÃO: para que a publicação funcione, a porta selecionada (o padrão é 8172) deve ser aberta (pode estar bloqueada em algumas empresas).

Detalhes completos da instalação (passo a passo com fotos relevantes) podem ser encontrados aqui

Instalando e configurando a implantação da Web em servidores sem conexão com a Internet #

A configuração completa da implantação da Web sem conexão com a Internet (offline) é mais difícil de fazer porque o instalador da plataforma Web (UI) opera consultando a lista de pacotes e o conteúdo de um local da Internet.

  1. Limpeza Certifique-se de que C:\Program Files\IIS não contém uma versão mais antiga do Microsoft Web Deploy. Se contiver, desinstale-o (eles) de Adicionar/remover programas e remova todos os arquivos restantes

  2. Web Platform Installer v5 Command Line (WebPICMD.exe) permite exportar e instalar pacotes sem o requisito “online”. As instruções completas de instalação e uso podem ser encontradas aqui.

  3. Pacotes necessários para implantação na Web

  • instale o Web Deploy em uma máquina “online”

  • exporte pacotes para ele usando a linha de comando WPI

    C:\Program Files\Microsoft\Web Platform Installer>WebPICMD.exe /Offline /Product s:“WDeployPS” /Path:C:\OfflineCache

Excluindo arquivos e pastas da implantação da Web #

Normalmente, alguns arquivos do aplicativo Web não devem ser substituídos ao executar a implantação (por exemplo, web.config). Isso pode ser feito por:

1) Excluindo da saída - o que significa definir Ação de compilação como Nenhuma. Essa é a maneira mais fácil, mas pode não funcionar para alguns arquivos ou pastas específicos, que devem estar na saída para que o aplicativo seja executado localmente

2) Excluindo arquivos e pastas de um pacote da Web criando um arquivo xml especial na pasta raiz do aplicativo da Web. Por exemplo.:

File name = [project name].wpp.targets
File content = 
    
<Project ToolsVersion="4.0" 
         xmlns="http://schemas.microsoft.com/developer/msbuild/2003">
  <ItemGroup>   
    <!-- this will exclude all files from Internal folder -->
    <ExcludeFromPackageFolders Include="Internal">
      <FromTarget>ContactManager.Mvc.wpp.targets</FromTarget>
    </ExcludeFromPackageFolders>

    <!-- this will exclude specified files -->
    <ExcludeFromPackageFiles Include="Scripts\jquery-1.4.4-
vsdoc.js;Scripts\jquery-1.4.4.js;Scripts\jquery-ui.js;Scripts\jquery.unobtrusive-ajax.js;Scripts\jquery.validate-vsdoc.js;Scripts\jquery.validate.js;Scripts\jquery.validate.unobtrusive.js;Scripts\MicrosoftAjax.debug.js;Scripts\MicrosoftMvcValidation.debug.js">
      <FromTarget>ContactManager.Mvc.wpp.targets</FromTarget>
    </ExcludeFromPackageFiles>
  </ItemGroup>
</Project>

Backups automáticos de implantação da Web #

A implantação da Web oferece a opção de fazer backup automático do site de destino (não do aplicativo da Web de destino!) na implantação. Isso é recomendado para permitir a reversão do aplicativo da web.

Para configurar backups automáticos, as seguintes etapas devem ser seguidas:

1) Ativar backups

Abra %programfiles%\IIS\Microsoft Web Deploy V3\scripts\BackupScripts.ps1 no Powershell

Execute os seguintes comandos:

# Turns on all backup functionality
TurnOn-Backups -On $true

# Turns off all backup functionality
TurnOn-Backups -On $false

# Changes default global backup behavior to enabled
Configure-Backups -Enabled $true

# Changes default backup behavior for site "foo" to enabled
Configure-Backups -SiteName "foo" -Enabled $true

# Changes the path of where backups are stored to a sibling directory named "siteName_snapshots".  
# For more information about path variables, see the "backupPath" attribute in the section 
# "Configuring  Backup Settings on the Server for Global usage manually in IIS Config"
Configure-Backups -BackupPath "{SitePathParent}\{siteName}_snapshots"

# Configures default backup limit to 5 backups
Configure-Backups -NumberOfBackups 5

# Configures sync behavior to fail if a sync fails for any reason
Configure-Backups -ContinueSyncOnBackupFailure $false

# Adds providers to skip when performing a backup
Configure-Backups -AddExcludedProviders @("dbmysql","dbfullsql")

# Allows a site administrator to enable backups and set the number of backups at the site level
Configure-BackupSettingsProvider -CanSetEnabled $true -CanSetNumBackups $true

# Allows a site administrator to control which providers they want to skip in a backup, as 
# well as whether they can continue a sync after a backup failure
Configure-BackupSettingsProvider -CanSetContinueSyncOnBackupFailure $true -CanAddExcludedProviders $true

2) Verifique as configurações de backup no nível global ou do site

Get-BackupSettings
Get-BackupSettings -SiteName "Default Web Site"

3) Mais personalização de backup

As configurações de backup podem ser configuradas para cada site. Abra applicationHost.config e adicione configurações de backup para seu local específico:

<location path="siteName">
    <system.webServer>
        <wdeploy>
            <backup enabled="true" numberOfBackups="4">
                <excludedProviders>
                    <clear />
                    <provider name="dbfullsql" />
                </excludedProviders>
            </backup>
        </wdeploy>
    </system.webServer>
</location>
