O AR é perfeito quando você precisa excluir, atualizar ou criar um ou mais registros sequencialmente. Seu suporte de atributos sujos (salvando apenas o que foi realmente alterado) resulta em instruções UPDATE otimizadas que retiram significativamente a carga do banco de dados e reduzem as chances de vários conflitos relacionados à edição do mesmo registro por várias pessoas ao mesmo tempo.

Se você não tem uma lógica realmente complexa em seu aplicativo e, portanto, não requer abstração de entidades, o AR é o mais adequado para exclusões, atualizações e criações.

O AR também é adequado para consultas simples, resultando em menos de 100 registros por página. Não é tão eficiente quanto trabalhar com arrays produzidos pelo construtor de consultas ou asArray(), mas é mais prazeroso trabalhar com ele.

AR não é recomendado para consultas complexas. Geralmente, trata-se de agregar ou transformar dados, de modo que o que é retornado não se encaixa no modelo AR de qualquer maneira. É preferível usar o construtor de consultas neste caso.

O mesmo vale para importação e exportação. É melhor usar o construtor de consultas devido a grandes quantidades de dados e consultas possivelmente complexas.

Cláusula Onde #

OPERADORES

$postsGreaterThan = Post::find()->where(['>', 'created_at', '2016-01-25'])->all();
// SELECT * FROM post WHERE created_at > '2016-01-25'

$postsLessThan = Post::find()->where(['<', 'created_at', '2016-01-25'])->all();
// SELECT * FROM post WHERE created_at < '2016-01-25'

$postsNotEqual = Post::find()->where(['<>', 'created_at', '2016-01-25'])->all();
// SELECT * FROM post WHERE created_at <> '2016-01-25'

DENTRO

$postsInArray = Post::find()->where(['id' => [1,2,3]])->all();
// SELECT * FROM post WHERE id IN (1,2,3)

ENTRE

$postsInBetween = Post::find()
->where(['between', 'date', "2015-06-21", "2015-06-27" ])
->all();

NULO

$postsWithNullTitle = Post::find()->where(['title' => null]);
// SELECT * FROM post WHERE title IS NULL

E

$postsAND = Post::find()->where(['title' => null, 'body' => null]);
// SELECT * FROM post WHERE title IS NULL AND body IS NULL

OU

$postsAND = Post::find()->where(['OR', 'title IS NULL', 'body IS NULL']);
// SELECT * FROM post WHERE title IS NULL OR body IS NULL

NÃO

 $postsNotEqual = Post::find()->where(['NOT', ['created_at'=>'2016-01-25']])->all();
// SELECT * FROM post WHERE created_at IS NOT '2016-01-25'

CLÁUSULAS Aninhadas

$postsNestedWhere = Post::find()->andWhere([
    'or',
    ['title' => null],
    ['body' => null]
])->orWhere([
    'and',
    ['not', ['title' => null]],
    ['body' => null]
]);
// SELECT * FROM post WHERE (title IS NULL OR body IS NULL) OR (title IS NOT NULL AND body IS NULL)

LIKE OPERATOR com métodos filterWhere activerecord

Por exemplo, no filtro de pesquisa, você deseja filtrar a postagem pelo título ou descrição da postagem publicada pelo usuário conectado no momento.

$title = 'test';
$description = 'test';

i) eFilterWhere()

$postLIKE = Post::find()->where(['user_id' => Yii::$app->user->getId()])->andfilterWhere(['or', ['title' => $title, 'description' => $description]])->all();
//SELECT * FROM post WHERE user_id = 2 AND ((`title` LIKE '%test%') OR (`description` LIKE '%test%'))

ii) ouFilterWhere()

$postLIKE = Post::find()->where(['user_id' => Yii::$app->user->getId()])->orFilterWhere(['or', ['title' => $title, 'description' => $description]])->all();
//SELECT * FROM post WHERE user_id = 2 OR ((`title` LIKE '%test%') OR (`description` LIKE '%test%'))

iii) filterWhere()

$postLIKE = Post::find()->filterWhere(['AND', ['title' => $title, 'description' => $description]])->andWhere(['user_id' => Yii::$app->user->getId()])->all();
//SELECT * FROM post WHERE ((`title` LIKE '%test%') AND (`description` LIKE '%test%')) AND user_id = 2

Observação: Ao usar filterWhere(), temos que chamar todos andwhere() ou orWhere() após filterWhere(), caso contrário, todas as condições where serão removidas, exceto filterWhere()

Crie uma classe ActiveRecord com valores de campos baseados em eventos #

    <?php
    namespace models;

    use yii\db\ActiveRecord;
    use yii\behaviors\TimestampBehavior;

    class Post extends ActiveRecord
    {
        public static function tableName()
        {
            return 'post';
        }

        public function rules() {
            return [
                [['created_at', 'updated_at'], 'safe'],
            ];
        }
        public function behaviors() {
            parent::behaviors();
    
            return [
              'timestamp' => [
                'class' => TimestampBehavior::className(),
                'attributes' => [
                  ActiveRecord::EVENT_BEFORE_INSERT => ['created_at', 'updated_at'],
                  ActiveRecord::EVENT_BEFORE_UPDATE => ['updated_at']
                ],
                'value' => date('Y-m-d H:i:s'),
              ]
            ];
          }
    }

Encontre todos os registros #

Post::find()->all();
// SELECT * FROM post

ou a abreviação

(Retorna uma instância de modelo de registro ativo por uma chave primária ou uma matriz de valores de coluna.)

Post::findAll(condition);

retorna uma matriz de instâncias ActiveRecord.

Encontrar tudo com condição where

$model = User::find()
        ->where(['id' => $id])
        ->andWhere('status = :status', [':status' => $status])
        ->all();

Encontre tudo com orderB

$model = User::find()
         ->orderBy(['id'=>SORT_DESC])
        ->all();
Or

$model = User::find()
         ->orderBy(['id'=>SORT_ASC])
        ->all();

Encontre um registro #

$customer = Customer::findOne(10);

ou

$customer = Customer::find()->where(['id' => 10])->one();

ou

$customer = Customer::find()->select('name,age')->where(['id' => 10])->one();

ou

$customer = Customer::findOne(['age' => 30, 'status' => 1]);

ou

$customer = Customer::find()->where(['age' => 30, 'status' => 1])->one();

Encontre uma consulta #

Encontre um único registro com base no id.

$model = User::findOne($id);

Selecione uma única coluna com base no id.

$model = User::findOne($id)->name;

Recupere o registro único do banco de dados com base na condição.

$model = User::find()->one();  // give first record

$model = User::find()->where(['id' => 2])->one(); // give single record based on id

Selecione o registro de colunas únicas do banco de dados com base na condição.

$model = User::find()->select('name,email_id')->where(['id' => 1])->one();

OU

$model = User::find()->select(['id','name','email_id'])->where(['id' => 1])->one();

Ordenar por

$model = User::find()->select(['id','name','email_id'])->orderBy(['id' => SORT_DESC])->one();

OR 

$model = User::find()->select(['id','name','email_id'])->orderBy(['id' => SORT_ASC])->one();

Registros ativos com subconsultas #

Exemplo: Clientes que podem criar uma postagem. Cada cliente pode criar várias postagens. Assim que o cliente criar a postagem, a postagem estará sob análise dos administradores. Agora temos que buscar a lista de clientes que tem todas as postagens ativas usando a subconsulta.

Nota: Se um cliente tiver 5 posts, entre 5 posts se ele tiver pelo menos um inativo, temos que excluir este cliente da lista de clientes.

$subQuery = Post::find()->select(['customer_id'])->where(['status' => 2]); //fetch the customers whos posts are inactive - subquery
$query = Customer::find()->where(['NOT IN', 'id', $subQuery])->all(); //Exclude the customers whos posts are inactive by using subquery
