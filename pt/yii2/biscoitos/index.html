Os cookies fazem parte da solicitação HTTP, portanto, é uma boa ideia fazer as duas coisas no controlador, cuja responsabilidade é exatamente lidar com a solicitação e a resposta.

Autenticação entre subdomínios e cookies de identidade #

No caso de login automático ou cookie “lembrar-me”, aplicam-se as mesmas peculiaridades que no caso de cookies de subdomínio. Mas desta vez você precisa configurar o componente do usuário, configurando o array identityCookie para a configuração de cookie desejada.

Abra o arquivo de configuração do aplicativo e adicione os parâmetros identityCookie à configuração do componente do usuário:

$config = [
    // ...
    'components' => [
        // ...
        'user' => [
            'class' => 'yii\web\User',
            'identityClass' => 'app\models\User',
            'enableAutoLogin' => true,
            'loginUrl' => '/user/login',
            'identityCookie' => [ // <---- here!
                'name' => '_identity',
                'httpOnly' => true,
                'domain' => '.example.com',
            ],
        ],
        'request' => [
            'cookieValidationKey' => 'your_validation_key'
        ],
        'session' => [
            'cookieParams' => [
                'domain' => '.example.com',
                'httpOnly' => true,
            ],
        ],

    ],
];

Observe que cookieValidationKey deve ser o mesmo para todos os subdomínios.

Observe que você deve configurar a propriedade session::cookieParams para ter o mesmo domínio que seu user::identityCookie para garantir que o login e o logout funcionem para todos os subdomínios. Esse comportamento é melhor explicado na próxima seção.

Lendo um cookie #

Para ler um cookie, use o seguinte código:

$value = \Yii::$app->getRequest()->getCookies()->getValue('my_cookie');

Observação: este código permite ler cookies que foram configurados usando o componente cookie (porque ele assina todos os cookies por padrão). Portanto, se você adicionar/atualizar o cookie usando o código JS, não poderá lê-lo usando esse método (por padrão).

Cookies para subdomínios #

Por motivos de segurança, por padrão, os cookies são acessíveis apenas no mesmo domínio a partir do qual foram definidos. Por exemplo, se você configurou um cookie no domínio example.com, você não pode obtê-lo no domínio www.example.com. Portanto, se você planeja usar subdomínios (ou seja, admin.example.com, profile.example.com), você precisa definir domain explicitamente:

$cookie = new Cookie([
    'name' => 'cookie_monster',
    'value' => 'Me want cookie everywhere!',
    'expire' => time() + 86400 * 365,
    'domain' => '.example.com' // <<<=== HERE
]);
\Yii::$app->getResponse()->getCookies()->add($cookie);

Agora o cookie pode ser lido de todos os subdomínios de example.com.

Configurando um cookie #

Para definir um cookie, ou seja, para criá-lo e agendar o envio para o navegador, você precisa criar uma nova instância de classe \yii\web\Cookie e adicioná-la à coleção de cookies de resposta:

$cookie = new Cookie([
    'name' => 'cookie_monster',
    'value' => 'Me want cookie!',
    'expire' => time() + 86400 * 365,
]);
\Yii::$app->getResponse()->getCookies()->add($cookie);

No exemplo acima, estamos passando parâmetros para o construtor da classe cookie. Estes basicamente são os mesmos usados ​​com a função nativa PHP setcookie:

  • name - nome do cookie.
  • valor - valor do cookie. Certifique-se de que é uma string. Os navegadores normalmente não ficam satisfeitos com os dados binários nos cookies.
  • domain - domínio para o qual você está configurando o cookie.
  • expire - timestamp unix indicando a hora em que o cookie deve ser excluído automaticamente.
  • path - o caminho no servidor em que o cookie estará disponível.
  • secure - se true, o cookie será definido somente se HTTPS for usado.
  • httpOnly - se true, o cookie não estará disponível via JavaScript.

Parâmetros de cookie de sessão #

Os parâmetros dos cookies de sessão são importantes tanto se você precisar manter a sessão ao obter de um subdomínio para outro ou quando, ao contrário, você hospeda o aplicativo de back-end na URL /admin e deseja manipular a sessão separadamente.

$config = [
    // ...
    'components' => [
        // ...
        'session' => [
            'name' => 'admin_session',
            'cookieParams' => [
                'httpOnly' => true,
                'path' => '/admin',
            ],
        ],
    ],
];
