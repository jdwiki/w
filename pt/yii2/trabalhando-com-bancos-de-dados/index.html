Usando o construtor de consultas Yii2 #

O Yii2 fornece maneiras eficientes de recuperar dados do banco de dados. Considere um exemplo de uma tabela simples de funcionários com os campos emp_id, emp_name e emp_salary. Para recuperar os nomes dos funcionários e seus salários, usamos a consulta.

select emp_name,emp_salary from employee

Para gerar a consulta acima no Yii2, existem vários métodos. Um dos métodos é usar um objeto yii\db\Query.

//creates a new \yii\db\Query() object
$query=new \yii\db\Query(); 

$rows=$query->select(['emp_name','emp_salary']) //specify required columns in an array
             ->from('employee') //specify table name
             ->all(); //returns an array of rows with each row being an associative array of name-value pairs.

Podemos usar um loop foreach para percorrer cada par nome-valor no array $rows.

foreach ($rows as $row) {
    echo "Employee Name: ".$row['emp_name'].",Employee Salary: ".$row['emp_salary']."<br>";
}

Isso vai sair

Nome do Funcionário: Kiran, Salário do Funcionário: 25.000

Nome do Funcionário: Midhun, Salário do Funcionário: 50.000

Nome do Funcionário: Jishnu, Salário do Funcionário: 20.000

Nome do Funcionário: Ajith, Salário do Funcionário: 25.000

Nome do Funcionário: Akshay, Salário do Funcionário: 750000

Mais exemplos

Suponha que precisamos encontrar o nome dos funcionários cujo salário é igual a 25.000. Podemos escrever a consulta em sql como

select emp_name from employee where salary=25000

No Yii2, o código para gerar a consulta acima

$query=new \yii\db\Query(); 

$rows=$query->select(['emp_name']) 
            ->from('employee')
            ->where(['emp_salary'=>25000]) //specify the condition as an associative array where key is column name
            ->all();

Se precisarmos encontrar nomes de funcionários cujo salário seja maior que 25.000, podemos escrever o código em Yii2 como

  $rows=$query->select(['emp_name']) 
        ->from('employee')
        ->where(['>','emp_salary', 25000]) 
//Here first element in the above array specify relational operator used, second element specify the table name and third the value itself.
        ->all();

Mais verificação de condição usando where() #

Múltiplas condições podem ser escritas usando o método where() conforme indicado abaixo.

// Creates a new \yii\db\Query() object
$query = new \yii\db\Query();
$rows = $query->select(['emp_name','emp_salary']) 
        ->from('employee')
        ->where(['emp_name' => 'Kiran', 'emp_salary' => 25000]) // Specify multiple conditions
        ->one(); // Returns the first row of the result

O código acima buscará um funcionário com o nome kiran e salário 25000. Se vários funcionários atenderem à condição acima, a chamada one() garante que apenas o primeiro resultado seja obtido. Para buscar todos os resultados você deve usar all().

Observe que se você usar all() o resultado sempre será um array; Mesmo que haja apenas um ou zero resultados. Esta matriz contém todos os resultados como matrizes ou está vazia quando nenhum registro corresponde. A chamada one() retornará o array resultante diretamente ou false se a consulta não retornar nada.

O código equivalente em sql é dado abaixo.

select emp_name, emp_salary from employee where emp_name = 'Kiran' and emp_salary = 25000 limit 1;

Uma maneira alternativa de escrever a consulta acima no Yii2 é fornecida abaixo.

$rows = $query->select(['emp_name', 'emp_salary']) 
    ->from('employee')
    ->where(['emp_name' => 'Kiran'])
    ->andWhere(['emp_salary' => 25000])
    ->one();

Um conjunto adicional de condições pode ser especificado usando andWhere. Isso será útil se precisarmos adicionar verificação de condição adicional à consulta posteriormente.

Outra maneira de especificar várias condições é usar o formato do operador do método where(). A consulta acima também pode ser escrita conforme indicado abaixo.

 $rows = $query->select(['emp_name','emp_salary']) 
    ->from('employee')
    ->where(['and', 'emp_name="kiran"', 'emp_salary=25000'])        
    ->one();

Aqui especificamos o operador ‘and’ como o primeiro elemento do array. Da mesma forma, também podemos usar ‘ou’, ‘entre’, ‘not between’, ‘in’, ‘not in’, ‘** gosta**’, ‘ou gosta’, ‘não gosta’, ‘ou não gosta’, ‘existe’, ‘não existe’, ‘>’, ‘<=’ etc como operadores.

Exemplos de uso de ‘in’ e ’like’

Suponha que precisamos encontrar os funcionários com salários 20.000, 25.000 e 50.000. No sql normal, escreveríamos a consulta como

select * from employee where salary in (20000,25000,50000)

No Yii2 podemos escrever isso como indicado abaixo.

$rows = $query->from('employee')
        ->where(['emp_salary' => [20000,25000,50000]]) 
        ->all();

Outra maneira de especificar a mesma condição é

$rows = $query->from('employee')
    ->where(['in', 'emp_salary', [20000,25000,50000]]) // Making use of operator format of where() method
    ->all();

Da mesma forma, ‘not in’ pode ser especificado em vez de ‘in’ se quisermos que todos os funcionários não tenham salários de 20.000, 25.000 e 50.000.

Agora vamos ver alguns exemplos do uso de ‘like’ dentro da condição where(). Suponha que precisamos encontrar todos os funcionários com a string ‘gopal’ em seus nomes. Os nomes podem ser venugopal, rajagopal, gopalakrishnan etc. A consulta sql é fornecida abaixo.

select * from employee where emp_name like '%gopal%'

Em Yii2 vamos escrever isso como

 $rows = $query->from('employee')
        ->where(['like', 'emp_name', 'gopal']) // Making use of operator format of where() method
        ->all();

Se precisarmos encontrar todos os funcionários com a string ‘gopal’ e ‘nair’ em seus nomes. Podemos escrever como

   $rows = $query->from('employee')
        ->where(['like', 'emp_name', ['gopal','nair']]) // Making use of operator format of where() method
        ->all();

Isso avaliaria como

selecione * do funcionário onde emp_name como ‘%gopal%’ e ‘%nair%’

Da mesma forma, podemos usar ‘not like’ para indicar todos os funcionários que não têm a string ‘gopal’ e ‘nair’ em seus nomes.

Usando orderBy() #

O método orderBy() especifica o fragmento ORDER BY de uma consulta SQL. Por exemplo, considere nossa tabela de funcionários com os campos emp_id, emp_first_name, emp_last_name e emp_salary. Suponha que precisamos ordenar o resultado aumentando a ordem dos salários dos funcionários. pode fazê-lo em sql como indicado abaixo.

Select * from employee order by emp_salary

No yii2, podemos construir a consulta conforme abaixo

 //creates a new \yii\db\Query() object
    $query=new \yii\db\Query();

    $rows= $query->from('employee')->orderBy([
    'emp_salary' => SORT_ASC //specify sort order ASC for ascending DESC for descending      
    ])->all();

Se precisarmos ordenar os funcionários com o primeiro nome em ordem crescente e depois seus salários em ordem decrescente, podemos escrevê-lo em sql simples da seguinte maneira.

Select * from employee order by emp_first_name ASC, emp_salary DESC

O sql equivalente pode ser construído usando yii2 da seguinte forma

//creates a new \yii\db\Query() object
    $query=new \yii\db\Query();

    $rows= $query->from('employee')->orderBy([
    'emp_first_name' => SORT_ASC
    'emp_salary' => SORT_DESC      
    ])->all();

Você também pode especificar ORDER BY usando uma string, assim como faz ao escrever instruções SQL brutas. Por exemplo, a consulta acima também pode ser gerada conforme indicado abaixo.

//creates a new \yii\db\Query() object
$query=new \yii\db\Query();
$rows=$query->from('employee')->orderBy('emp_first_name ASC, emp_salary DESC')->all();

Você pode chamar addOrderBy() para adicionar colunas adicionais ao fragmento ORDER BY. Por exemplo

//creates a new \yii\db\Query() object
$query=new \yii\db\Query();
$rows=$query->from('employee')->orderBy('emp_first_name ASC')
    ->addOrderBy('emp_salary DESC')->all();
