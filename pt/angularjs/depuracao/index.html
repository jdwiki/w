Usando a extensão chrome ng-inspect #

ng-inspect é uma extensão leve do Chrome para depurar aplicativos AngularJS.

Quando um nó é selecionado no painel de elementos, as informações relacionadas ao escopo são exibidas no painel ng-inspect.

Expõe poucas variáveis ​​globais para acesso rápido de scope/isolateScope.

$s – escopo do nó selecionado $is –isolaScope do nó selecionado $el – referência do elemento jQuery do nó selecionado (requer jQuery) $events – eventos presentes no nó selecionado (requer jQuery)

Proporciona fácil acesso aos Serviços/Fábricas.

Use $get() para recuperar a instância de um serviço/fábrica pelo nome.

O desempenho do aplicativo pode ser monitorado contando o número de escopos, escopos isolados, observadores e ouvintes no aplicativo.

Use $count() para obter a contagem de escopos, escopos isolados, observadores e ouvintes.

Nota: Esta extensão funcionará somente quando o debugInfo estiver habilitado.

Depuração básica na marcação #

Teste de escopo e saída do modelo

<div ng-app="demoApp" ng-controller="mainController as ctrl">
    {{$id}}
    <ul>
        <li ng-repeat="item in ctrl.items">
            {{$id}}<br/>
            {{item.text}}
        </li>
    </ul>
    {{$id}}
    <pre>
       {{ctrl.items | json : 2}}
    </pre>
</div>


angular.module('demoApp', [])
.controller('mainController', MainController);

function MainController() {
    var vm = this;
    vm.items = [{
        id: 0,
        text: 'first'
    },
    {
        id: 1,
        text: 'second'
    },
    {
        id: 2,
        text: 'third'
    }];
}

Às vezes, pode ajudar ver se há um novo escopo para corrigir problemas de escopo. $scope.$id pode ser usado em uma expressão em qualquer lugar da sua marcação para ver se existe um novo $scope.

No exemplo você pode ver que fora da ul-tag está o mesmo escopo ($id=2) e dentro do ng-repeat existem novos escopos filhos para cada iteração.

Uma saída do modelo em uma pré-tag é útil para ver os dados atuais do seu modelo. O filtro json cria uma saída formatada de boa aparência. A pré-tag é usada porque dentro dessa tag qualquer caractere de nova linha \n será exibido corretamente.

Obtendo o escopo do elemento #

Em um aplicativo angular, tudo gira em torno do escopo, se pudermos obter um escopo de elementos, é fácil depurar o aplicativo angular. Como acessar o escopo do elemento:

angular.element(myDomElement).scope();
e.g.
angular.element(document.getElementById('yourElementId')).scope()  //accessing by ID

Obtendo o escopo do controlador: -

 angular.element('[ng-controller=ctrl]').scope()

Outra maneira fácil de acessar um elemento DOM do console (como jm mencionou) é clicar nele na guia ’elementos’ e ele é armazenado automaticamente como $0.

angular.element($0).scope();
