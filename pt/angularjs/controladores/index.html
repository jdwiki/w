Sintaxe #

  • <htmlElement of-controller=“controllerName”> … </htmlElement>
  • <script> app.controller(‘controllerName’, controllerFunction); </script>

Seu primeiro controlador #

Um controlador é uma estrutura básica usada em Angular para preservar o escopo e lidar com certas ações dentro de uma página. Cada controlador é acoplado a uma visualização HTML.

Abaixo está um clichê básico para um aplicativo Angular:

<!DOCTYPE html>

<html lang="en" ng-app='MyFirstApp'>
    <head>
        <title>My First App</title>

        <!-- angular source -->
        <script src="https://code.angularjs.org/1.5.3/angular.min.js"></script>
        
        <!-- Your custom controller code -->
        <script src="js/controllers.js"></script>
    </head>
    <body>
        <div ng-controller="MyController as mc">
            <h1>{{ mc.title }}</h1>
            <p>{{ mc.description }}</p>
            <button ng-click="mc.clicked()">
                Click Me!
            </button>
        </div>
    </body>
</html>

Há algumas coisas a serem observadas aqui:

<html ng-app='MyFirstApp'>

Definir o nome do aplicativo com ng-app permite acessar o aplicativo em um arquivo Javascript externo, que será abordado abaixo.

<script src="js/controllers.js"></script>

Precisaremos de um arquivo Javascript onde você defina seus controladores e suas ações/dados.

<div ng-controller="MyController as mc">

O atributo ng-controller define o controlador para esse elemento DOM e todos os elementos que são filhos (recursivamente) abaixo dele.

Você pode ter vários do mesmo controlador (neste caso, MyController) dizendo ... as mc, estamos dando a esta instância do controlador um alias.

<h1>{{ mc.title }}</h1>

A notação {{ ... }} é uma expressão Angular. Nesse caso, isso definirá o texto interno desse elemento <h1> para qualquer valor de mc.title.

Observação: Angular emprega vinculação de dados de duas vias, o que significa que, independentemente de como você atualiza o valor mc.title, ele será refletido no controlador e na página.

Observe também que as expressões angulares não precisam fazer referência a um controlador. Uma expressão Angular pode ser tão simples quanto {{ 1 + 2 }} ou {{ "Hello " + "World" }}.

<button ng-click="mc.clicked()">

ng-click é uma diretiva Angular, neste caso ligando o evento click para que o botão acione a função clicked() da instância MyController.

Com essas coisas em mente, vamos escrever uma implementação do controlador MyController. Com o exemplo acima, você escreveria este código em js/controller.js.

Primeiro, você precisará instanciar o aplicativo Angular em seu Javascript.

var app = angular.module("MyFirstApp", []);

Observe que o nome que passamos aqui é o mesmo que você definiu em seu HTML com a diretiva ng-app.

Agora que temos o objeto app, podemos usá-lo para criar controladores.

app.controller('MyController', function(){
    var ctrl = this;

    ctrl.title = "My First Angular App";
    ctrl.description = "This is my first Angular app!";

    ctrl.clicked = function(){
        alert("MyController.clicked()");
    };
});

Observação: para qualquer coisa que queiramos fazer parte da instância do controlador, usamos a palavra-chave this.

Isso é tudo o que é necessário para construir um controlador simples.

Criando Controladores, Minificação segura #

Existem algumas maneiras diferentes de proteger a criação do seu controlador da minificação.

A primeira é chamada de anotação de matriz inline. Parece o seguinte:

var app = angular.module('app');
app.controller('sampleController', ['$scope', '$http', function(a, b){
    //logic here
}]);

O segundo parâmetro do método do controlador pode aceitar uma matriz de dependências. Como você pode ver eu defini $scope e $http que devem corresponder aos parâmetros da função do controlador em que a será o $scope, e b seria $http . Observe que o último item na matriz deve ser sua função de controlador.

A segunda opção é usar a propriedade $inject. Parece o seguinte:

var app = angular.module('app');
app.controller('sampleController', sampleController);
sampleController.$inject = ['$scope', '$http'];
function sampleController(a, b) {
    //logic here
}

Isso faz a mesma coisa que a anotação de matriz inline, mas fornece um estilo diferente para aqueles que preferem uma opção à outra.

A ordem das dependências injetadas é importante #

Ao injetar dependências usando o formulário de matriz, certifique-se de que a lista de dependências corresponda à lista correspondente de argumentos passados ​​para a função do controlador.

Observe que no exemplo a seguir, $scope e $http são invertidos. Isso causará um problema no código.

// Intentional Bug: injected dependencies are reversed which will cause a problem
app.controller('sampleController', ['$scope', '$http',function($http, $scope) {
    $http.get('sample.json');
}]);

Usando ControllerAs em Angular JS #

Em Angular $scope é a cola entre o Controller e a View que ajuda com todas as nossas necessidades de vinculação de dados. Controller As é outra maneira de vincular controlador e exibição e é mais recomendado para uso. Basicamente, essas são as duas construções do controlador em Angular (ou seja, $scope e Controller As).

Diferentes maneiras de usar o Controller As são -

controllerAs View Sintaxe

<div ng-controller="CustomerController as customer">
    {{ customer.name }}
</div>

controllerAs Sintaxe do Controlador

function CustomerController() {
    this.name = {};
    this.sendMessage = function() { };
}

controllerAs com vm

function CustomerController() {
    /*jshint validthis: true */
    var vm = this;
    vm.name = {};
    vm.sendMessage = function() { };
}

controllerAs é açúcar sintático sobre $scope. Você ainda pode vincular a View e ainda acessar os métodos $scope. Usar controllerAs é uma das melhores práticas sugeridas pela equipe do núcleo angular. Há muitas razões para isso, poucas delas são -

  • $scope está expondo os membros do controller para a view através de um objeto intermediário. Ao definir this.*, podemos expor exatamente o que queremos expor do controlador para a visualização. Ele também segue a maneira padrão JavaScript de usar isso.

  • usando a sintaxe controllerAs, temos um código mais legível e a propriedade pai pode ser acessada usando o nome do alias do controlador pai em vez de usar a sintaxe $parent.

  • Promove o uso de vinculação a um objeto “pontilhado” na Visualização (por exemplo, customer.name em vez de nome), que é mais contextual, mais fácil de ler e evita problemas de referência que possam ocorrer sem “pontilhar”.

  • Ajuda a evitar o uso de chamadas $parent em Views com controladores aninhados.

  • Use uma variável de captura para isso ao usar a sintaxe controllerAs. Escolha um nome de variável consistente, como vm, que significa ViewModel. Porque a palavra-chave this é contextual e quando usada dentro de uma função dentro de um controller pode mudar seu contexto. Capturar o contexto disso evita encontrar esse problema.

OBSERVAÇÃO: usando a sintaxe controllerAs, adicione à referência de escopo atual ao controlador atual, para que fique disponível como campo

<div ng-controller="Controller as vm>...</div>

vm está disponível como $scope.vm.

Criando controladores angulares seguros para minificação #

Para criar controladores angulares seguros para minificação, você alterará os parâmetros da função controller.

O segundo argumento na função module.controller deve receber um array, em que o último parâmetro é a função do controlador e cada parâmetro antes disso é o nome de cada valor injetado.

Isso é diferente do paradigma normal; que recebe a função do controlador com os argumentos injetados.

Dado:

var app = angular.module('myApp');

O controlador deve ficar assim:

app.controller('ctrlInject', 
    [
        /* Injected Parameters */
        '$Injectable1', 
        '$Injectable2', 
        /* Controller Function */
        function($injectable1Instance, $injectable2Instance) {
            /* Controller Content */
        }
    ]
);

Observação: Os nomes dos parâmetros injetados não precisam corresponder, mas serão vinculados em ordem.

Isso reduzirá para algo semelhante a isto:

var a=angular.module('myApp');a.controller('ctrlInject',['$Injectable1','$Injectable2',function(b,c){/* Controller Content */}]);

O processo de minificação substituirá todas as instâncias de app por a, todas as instâncias de $Injectable1Instance por b e todas as instâncias de $Injectable2Instance por c.

Criando controladores #

angular
    .module('app')
    .controller('SampleController', SampleController)

SampleController.$inject = ['$log', '$scope'];
function SampleController($log, $scope){
    $log.debug('*****SampleController******');
    
    /* Your code below */
}

Nota: O .$inject garantirá que suas dependências não sejam embaralhadas após a minificação. Além disso, verifique se está em ordem com a função nomeada.

Controladores aninhados #

Os controladores aninhados também encadeiam o $scope. Alterar uma variável $scope no controlador aninhado altera a mesma variável $scope no controlador pai.

.controller('parentController', function ($scope) {
    $scope.parentVariable = "I'm the parent";
});

.controller('childController', function ($scope) {
    $scope.childVariable = "I'm the child";

    $scope.childFunction = function () {
        $scope.parentVariable = "I'm overriding you";
    };
});

Agora vamos tentar lidar com ambos, aninhados.

<body ng-controller="parentController">
    What controller am I? {{parentVariable}}
    <div ng-controller="childController">
        What controller am I? {{childVariable}}
        <button ng-click="childFunction()"> Click me to override! </button>
    </div>
</body>

Controladores de aninhamento podem ter seus benefícios, mas uma coisa deve ser lembrada ao fazê-lo. Chamar a diretiva ngController cria uma nova instância do controlador - o que muitas vezes pode criar confusão e resultados inesperados.
