Parâmetros #

Parâmetro Detalhes
*= Para usar a vinculação de dados bidirecional. Isso significa que, se você atualizar essa variável no escopo do componente, a alteração será refletida no escopo pai.
****** Ligações unidirecionais quando queremos apenas ler um valor de um escopo pai e não atualizá-lo.
@ Parâmetros de string.
*& Para retornos de chamada caso seu componente precise enviar algo para seu escopo pai.
- -
Ganchos LifeCycle Detalhes (requer angular.version >= 1.5.3 )
$onInit() Chamado em cada controlador depois que todos os controladores em um elemento foram construídos e tiveram suas ligações inicializadas. Este é um bom lugar para colocar o código de inicialização do seu controlador.
$onChanges(changesObj) Chamado sempre que as ligações unidirecionais são atualizadas. O changesObj é um hash cujas chaves são os nomes das propriedades vinculadas que foram alteradas, e os valores são um objeto da forma { currentValue, previousValue, isFirstChange() } .
$onDestroy() Chamado em um controlador quando seu escopo contido é destruído. Use este gancho para liberar recursos externos, relógios e manipuladores de eventos.
$postLink() Chamado após o elemento deste controlador e seus filhos terem sido vinculados. Este gancho pode ser considerado análogo aos ganchos ngAfterViewInit e ngAfterContentInit em Angular 2.
$doCheck() Chamado em cada turno do ciclo de digestão. Fornece uma oportunidade para detectar e agir sobre as mudanças. Quaisquer ações que você deseja executar em resposta às alterações detectadas devem ser invocadas a partir deste gancho; implementar isso não tem efeito quando $onChanges é chamado.

Componente é um tipo especial de diretiva que usa uma configuração mais simples que é adequada para uma estrutura de aplicativo baseada em componentes. Os componentes foram introduzidos no Angular 1.5, os exemplos nesta seção não funcionarão com versões mais antigas do AngularJS.

Um guia completo do desenvolvedor sobre componentes está disponível em https://docs.angularjs.org/guide/component

Componentes básicos e ganchos do ciclo de vida #

O que é um componente? #

  • Um componente é basicamente uma diretiva que usa um configuração e que é adequado para um componente baseado arquitetura, que é a essência do Angular 2. Pense em um componente como um widget: Um pedaço de código HTML que você pode reutilizar em vários lugares diferentes em seu aplicativo da web.

Componente

angular.module('myApp', [])
    .component('helloWorld', {
        template: '<span>Hello World!</span>'
    });

** Marcação **

<div ng-app="myApp"> 
  <hello-world> </hello-world>
</div>

Demonstração ao vivo

Usando dados externos no componente: #

Poderíamos adicionar um parâmetro para passar um nome ao nosso componente, que seria usado da seguinte forma:

angular.module("myApp", [])
  .component("helloWorld",{
    template: '<span>Hello {{$ctrl.name}}!</span>',
    bindings: { name: '@' }
  });

Marcação

<div ng-app="myApp"> 
  <hello-world name="'John'" > </hello-world>
</div>

Demonstração ao vivo

Usando controladores em componentes #

Vamos dar uma olhada em como adicionar um controlador a ele.

angular.module("myApp", [])
  .component("helloWorld",{
      template: "Hello {{$ctrl.name}}, I'm {{$ctrl.myName}}!",
      bindings: { name: '@' },
      controller: function(){
        this.myName = 'Alain';
      }
  });

** Marcação **

<div ng-app="myApp">  
  <hello-world name="John"> </hello-world>
</div>

Demonstração do CodePen

Os parâmetros passados ​​para o componente estão disponíveis no escopo do controlador logo antes de sua função $onInit ser chamada pelo Angular. Considere este exemplo:

angular.module("myApp", [])
  .component("helloWorld",{
      template: "Hello {{$ctrl.name}}, I'm {{$ctrl.myName}}!",
      bindings: { name: '@' },
      controller: function(){
        this.$onInit = function() {
          this.myName = "Mac" + this.name;
        }
      }
  });

No modelo acima, isso renderia “Hello John, I’m MacJohn!”.

Observe que $ctrl é o valor padrão Angular para controllerAs se um não for especificado.

Usando “require” como um objeto #

Em alguns casos, você pode precisar acessar dados de um componente pai dentro de seu componente.

Isso pode ser alcançado especificando que nosso componente requer aquele componente pai, o require nos dará referência ao controlador de componente necessário, que pode então ser usado em nosso controlador conforme mostrado no exemplo abaixo:

Observe que é garantido que os controladores necessários estarão prontos somente após o gancho $onInit.

angular.module("myApp", [])
  .component("helloWorld",{
      template: "Hello {{$ctrl.name}}, I'm {{$ctrl.myName}}!",
      bindings: { name: '@' },
      require: {
        parent: '^parentComponent'
      },
      controller: function () {
        // here this.parent might not be initiated yet

        this.$onInit = function() {
           // after $onInit, use this.parent to access required controller
           this.parent.foo();
        }

      }
  });

Tenha em mente, porém, que isso cria um acoplamento forte entre o filho e o pai.

Componentes em JS angular #

Os componentes em angularJS podem ser visualizados como uma diretiva personalizada (< html > this em uma diretiva HTML, e algo assim será uma diretiva personalizada < ANYTHING >). Um componente contém uma visão e um controlador. O controlador contém a lógica de negócios que está vinculada a uma visão, que o usuário vê. O componente difere de uma diretiva angular porque contém menos configuração. Um componente angular pode ser definido assim.

angular.module("myApp",[]).component("customer", {})

Os componentes são definidos nos módulos angulares. Eles contém dois argumentos, um é o nome do componente e o segundo é um objeto que contém o par chave-valor, que define qual view e qual controller ele vai usar assim.

angular.module("myApp",[]).component("customer", {
    templateUrl : "customer.html", // your view here 
    controller: customerController, //your controller here
    controllerAs: "cust"        //alternate name for your controller 
})

“myApp” é o nome do aplicativo que estamos construindo e customer é o nome do nosso componente. Agora, para chamá-lo no arquivo html principal, vamos colocá-lo assim

<customer></customer>

Agora esta diretiva será substituída pela visão que você especificou e pela lógica de negócios que você escreveu em seu controlador.

NOTA: Lembre-se de que o componente recebe um objeto como segundo argumento, enquanto a diretiva recebe uma função de fábrica como argumento.
