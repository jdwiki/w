Sintaxe #

  • constante(nome, valor);
  • valor(nome, valor);
  • factory(nome, $getFn);
  • serviço(nome, construtor);
  • provedor(nome, provedor);

Os provedores são objetos singleton que podem ser injetados, por exemplo, em outros serviços, controladores e diretivas. Todos os provedores são registrados usando “receitas” diferentes, onde Provedor é o mais flexível. Todas as receitas possíveis são:

  • Constante
  • Valor
  • Fábrica
  • Serviço
  • Fornecedor

Serviços, Fábricas e Provedores são todos inicializados preguiçosamente, o componente é inicializado somente se o aplicativo depender dele.

Decoradores estão intimamente relacionados aos Provedores. Decoradores são usados ​​para interceptar serviço ou criação de fábrica para alterar seu comportamento ou sobrescrever (partes dele).

Fornecedor #

Provider está disponível nas fases de configuração e execução.

A receita do provedor é definida sintaticamente como um tipo personalizado que implementa um método $get.

Você deve usar apenas a receita do Provedor quando você deseja expor uma API para configuração em todo o aplicativo que deve ser feito antes do início da aplicação. Isso geralmente interessante apenas para serviços reutilizáveis ​​cujo comportamento pode precisar variam ligeiramente entre as aplicações.

angular.module('app',[])
  .provider('endpointProvider', function() {
    var uri = 'n/a';
    
    this.set = function(value) {
      uri = value;
    };

    this.$get = function() {
      return {
        get: function() {
          return uri;
        }
      };
    };
  })
  .config(function(endpointProviderProvider) {
    endpointProviderProvider.set('http://some.rest.endpoint');
  })   
  .controller('MainCtrl', function(endpointProvider) {
    var vm = this;
    vm.endpoint = endpointProvider.get();
  }); 

<body ng-controller="MainCtrl as vm">
  <div>endpoint = {{::vm.endpoint }}</div>
</body>

ponto final = http://some.rest.endpoint

Sem o resultado da fase config

ponto final = n/a

Fábrica #

Factory está disponível na fase de execução.

A receita de fábrica constrói um novo serviço usando uma função com zero ou mais argumentos (são dependências de outros serviços). o o valor de retorno desta função é a instância de serviço criada por esta receita.

Factory pode criar um serviço de qualquer tipo, seja um objeto primitivo literal, função ou até mesmo uma instância de um tipo personalizado.

angular.module('app',[])
  .factory('endpointFactory', function() {
    return {
      get: function() {
        return 'http://some.rest.endpoint';
      }
    };
  })
  .controller('MainCtrl', function(endpointFactory) {
    var vm = this;
    vm.endpoint = endpointFactory.get();
  });

<body ng-controller="MainCtrl as vm">
  <div>endpoint = {{::vm.endpoint }}</div>
</body>

ponto final = http://some.rest.endpoint

Constante #

Constant está disponível nas fases de configuração e execução.

angular.module('app',[])
  .constant('endpoint', 'http://some.rest.endpoint') // define
  .config(function(endpoint) {
    // do something with endpoint
    // available in both config- and run phases
  }) 
  .controller('MainCtrl', function(endpoint) {       // inject
    var vm = this;
    vm.endpoint = endpoint;                          // usage
  });

<body ng-controller="MainCtrl as vm">
  <div>endpoint = {{ ::vm.endpoint }}</div>
</body>

ponto final = http://some.rest.endpoint

Serviço #

Service está disponível na fase de execução.

A receita de Serviço produz um serviço como o Valor ou Fábrica receitas, mas o faz invocando um construtor com o novo operador. O construtor pode receber zero ou mais argumentos, o que representam dependências necessárias para a instância desse tipo.

angular.module('app',[])
  .service('endpointService', function() {
    this.get = function() {
      return 'http://some.rest.endpoint';
    };
  })
  .controller('MainCtrl', function(endpointService) {
    var vm = this;
    vm.endpoint = endpointService.get();
  });

<body ng-controller="MainCtrl as vm">
  <div>endpoint = {{::vm.endpoint }}</div>
</body>

ponto final = http://some.rest.endpoint

Valor #

Value está disponível nas fases de configuração e execução.

angular.module('app',[])
  .value('endpoint', 'http://some.rest.endpoint') // define
  .run(function(endpoint) {
    // do something with endpoint
    // only available in run phase
  }) 
  .controller('MainCtrl', function(endpoint) {    // inject
    var vm = this;
    vm.endpoint = endpoint;                       // usage
  }); 

<body ng-controller="MainCtrl as vm">
  <div>endpoint = {{ ::vm.endpoint }}</div>
</body>

ponto final = http://some.rest.endpoint
