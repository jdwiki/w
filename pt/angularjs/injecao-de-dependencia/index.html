Sintaxe #

  • myApp.controller(‘MyController’, function($scope) { … }); // código não minificado
  • myApp.controller(‘MyController’, [’$scope’, function($scope) { … }]); //suporta minificação
  • função MeuControlador(){}

MyController.$inject = [’$scope’];

myApp.controller(‘MeuControlador’, MeuControlador); //$injeta anotação

  • $injector.get(‘injetável’); //injeção dinâmica/tempo de execução

Os provedores não podem ser injetados em blocos run.

Serviços ou Valores não podem ser injetados em blocos config.

Certifique-se de anotar suas injeções para que seu código não seja quebrado na minificação.

Injeções dinâmicas #

Há também uma opção para solicitar componentes dinamicamente. Você pode fazer isso usando o serviço $injector:

myModule.controller('myController', ['$injector', function($injector) {
    var myService = $injector.get('myService');
}]);

Observação: embora esse método possa ser usado para evitar o problema de dependência circular que pode interromper seu aplicativo, não é considerado uma prática recomendada contornar o problema usando-o. A dependência circular geralmente indica que há uma falha na arquitetura do seu aplicativo e você deve resolver isso.

Carregar dinamicamente o serviço AngularJS em JavaScript vanilla #

Você pode carregar serviços AngularJS em JavaScript vanilla usando o método AngularJS injector(). Cada elemento jqLite recuperado chamando angular.element() tem um método injector() que pode ser usado para recuperar o injetor.

var service;
var serviceName = 'myService';

var ngAppElement = angular.element(document.querySelector('[ng-app],[data-ng-app]') || document);
var injector = ngAppElement.injector();

if(injector && injector.has(serviceNameToInject)) {
    service = injector.get(serviceNameToInject);
}

No exemplo acima tentamos recuperar o elemento jqLite que contém a raiz da aplicação AngularJS (ngAppElement). Para fazer isso, usamos o método angular.element(), procurando por um elemento DOM contendo o atributo ng-app ou data-ng-app ou, se não existir, voltamos para document elemento. Usamos ngAppElement para recuperar a instância do injetor (com ngAppElement.injector()). A instância injector é usada para verificar se o serviço a ser injetado existe (com injector.has()) e então carregar o serviço (com injector.get()) dentro da variável service.

Injeções #

O exemplo mais simples de uma injeção em um aplicativo Angular - injetando $scope em um Controller Angular:

angular.module('myModule', [])
.controller('myController', ['$scope', function($scope) {
    $scope.members = ['Alice', 'Bob'];
    ...
}])

O exemplo acima ilustra uma injeção de um $scope em um controller, mas é o mesmo se você injetar qualquer módulo em qualquer outro. O processo é o mesmo.

O sistema da Angular é responsável por resolver as dependências para você. Se você criar um serviço, por exemplo, você pode listá-lo como no exemplo acima e ele estará disponível para você.

Você pode usar DI - Dependency Injection - onde quer que esteja definindo um componente.

Observe que no exemplo acima usamos o que é chamado de “Inline Array Annotation”. Ou seja, escrevemos explicitamente como strings os nomes de nossas dependências. Fazemos isso para evitar que o aplicativo seja interrompido quando o código for reduzido para Produção. A minificação de código altera os nomes das variáveis ​​(mas não as strings), o que interrompe a injeção. Ao usar strings, o Angular sabe quais dependências queremos.

Muito importante - a ordem dos nomes das strings deve ser a mesma dos parâmetros da função.

Existem ferramentas que automatizam esse processo e cuidam disso para você.

$injetar anotação de propriedade #

De forma equivalente, podemos usar a anotação da propriedade $inject para obter o mesmo acima:

var MyController = function($scope) {
  // ...
}
MyController.$inject = ['$scope'];
myModule.controller('MyController', MyController);
