Envolva um valor simples em uma promessa usando $q.when() #

Se tudo que você precisa é envolver o valor em uma promessa, você não precisa usar a sintaxe longa como aqui:

//OVERLY VERBOSE
var defer;
defer = $q.defer();
defer.resolve(['one', 'two']);
return defer.promise;

Neste caso, você pode simplesmente escrever:

//BETTER
return $q.when(['one', 'two']);

$q.when e seu alias $q.resolve #

Envolve um objeto que pode ser um valor ou uma promessa (de terceiros) então capaz em uma promessa $q. Isso é útil quando você está lidando com um objeto que pode ou não ser uma promessa, ou se a promessa vier de uma fonte que não pode ser confiável.

— Referência de API de serviço AngularJS $q - $q.when

Com o lançamento do AngularJS v1.4.1

Você também pode usar um alias consistente com ES6 resolve

//ABSOLUTELY THE SAME AS when
return $q.resolve(['one', 'two'])

Usando promessas angulares com o serviço $q #

$q é um serviço embutido que ajuda na execução de funções assíncronas e usando seus valores de retorno (ou exceção) quando terminam de processar.

$q é integrado com o mecanismo de observação de modelo $rootScope.Scope, o que significa propagação mais rápida de resolução ou rejeição em seus modelos e evitando repinturas desnecessárias do navegador, o que resultaria em IU piscando.

Em nosso exemplo, chamamos nossa fábrica de getMyData, que retorna um objeto de promessa. Se o objeto for resolvido, ele retornará um número aleatório. Se for rejected, retorna uma rejeição com uma mensagem de erro após 2 segundos.

Na fábrica angular

function getMyData($timeout, $q) {
  return function() {
    // simulated async function
    var promise = $timeout(function() {
      if(Math.round(Math.random())) {
        return 'data received!'
      } else {
        return $q.reject('oh no an error! try again')
      }
    }, 2000);
    return promise;
  }
}

Usando promessas de plantão #

angular.module('app', [])
.factory('getMyData', getMyData)
.run(function(getData) {
  var promise = getData()
    .then(function(string) {
      console.log(string)
    }, function(error) {
      console.error(error)
    })
    .finally(function() {
      console.log('Finished at:', new Date())
    })
})

Para usar promessas, injete $q como dependência. Aqui nós injetamos $q na fábrica getMyData.

var defer = $q.defer();

Uma nova instância de deferred é construída chamando $q.defer()

Um objeto adiado é simplesmente um objeto que expõe uma promessa, bem como os métodos associados para resolver essa promessa. Ele é construído usando a função $q.deferred() e expõe três métodos principais: resolve(), reject() e notify().

  • resolve(valor) – resolve a promessa derivada com o valor.
  • reject(motivo) – rejeita a promessa derivada com o motivo.
  • notify(value) - fornece atualizações sobre o status da execução da promessa. Isso pode ser chamado várias vezes antes que a promessa seja resolvida ou rejeitada.

Propriedades #

O objeto de promessa associado é acessado por meio da propriedade de promessa. promise – {Promise} – objeto de promessa associado a este adiado.

Uma nova instância de promessa é criada quando uma instância deferida é criada e pode ser recuperada chamando deferred.promise.

O objetivo do objeto promise é permitir que as partes interessadas tenham acesso ao resultado da tarefa adiada quando ela for concluída.

Métodos de promessa -

  • then(successCallback, [errorCallback], [notifyCallback]) – Independentemente de quando a promessa foi ou será resolvida ou rejeitada, chama um dos callbacks de sucesso ou erro de forma assíncrona assim que o resultado estiver disponível. Os retornos de chamada são chamados com um único argumento: o resultado ou motivo da rejeição. Além disso, o callback de notificação pode ser chamado zero ou mais vezes para fornecer uma indicação de andamento, antes que a promessa seja resolvida ou rejeitada.

  • catch(errorCallback) – abreviação de promise.then(null, errorCallback)

  • finally(callback, notifyCallback) – permite observar o cumprimento ou a rejeição de uma promessa, mas sem modificar o valor final.

Um dos recursos mais poderosos das promessas é a capacidade de encadeá-las. Isso permite que os dados fluam pela cadeia e sejam manipulados e modificados em cada etapa. Isso é demonstrado com o seguinte exemplo:

Exemplo 1:

// Creates a promise that when resolved, returns 4.
function getNumbers() {

  var promise = $timeout(function() {
    return 4;
  }, 1000);
  
  return promise;
}

// Resolve getNumbers() and chain subsequent then() calls to decrement
// initial number from 4 to 0 and then output a string.
getNumbers()
  .then(function(num) {
      // 4
      console.log(num);
      return --num;
  })
  .then(function (num) {
      // 3
      console.log(num);
      return --num;
  })
   .then(function (num) {
      // 2
      console.log(num);
      return --num;
  })
  .then(function (num) {
      // 1
      console.log(num);
      return --num;
  })
  .then(function (num) {
      // 0
      console.log(num);
      return 'And we are done!';
  })
  .then(function (text) {
      // "And we are done!"
      console.log(text);        
  });

Usando o construtor $q para criar promessas #

A função construtora $q é usada para criar promessas de APIs assíncronas que usam retornos de chamada para retornar resultados.

$q(função(resolver, rejeitar) {…})

A função construtora recebe uma função que é invocada com dois argumentos, resolve e reject, que são funções usadas para resolver ou rejeitar a promessa.

Exemplo 1:

function $timeout(fn, delay) {
    return = $q(function(resolve, reject) {
         setTimeout(function() {
              try {
                 let r = fn();
                 resolve(r);
              } 
              catch (e) {
                 reject(e);
              }
         }, delay);
     };
}

O exemplo acima cria uma promessa da WindowTimers.setTimeout API. A estrutura AngularJS fornece uma versão mais elaborada dessa função. Para uso, consulte a Referência da API de serviço AngularJS $timeout.

Exemplo 2:

$scope.divide = function(a, b) {
    return $q(function(resolve, reject) {
      if (b===0) {
        return reject("Cannot devide by 0")
      } else {
        return resolve(a/b);
      }
    });
}

O código acima mostrando uma função de divisão prometida, ele retornará uma promessa com o resultado ou rejeitará com um motivo se o cálculo for impossível.

Você pode então chamar e usar .then

$scope.divide(7, 2).then(function(result) {
    // will return 3.5
}, function(err) {
    // will not run
})

$scope.divide(2, 0).then(function(result) {
    // will not run as the calculation will fail on a divide by 0
}, function(err) {
    // will return the error string.
})

Usando $q.all para lidar com várias promessas #

Você pode usar a função $q.all para chamar um método .then após um array de promessas ter sido resolvido com sucesso e buscar os dados com os quais eles resolveram.

Exemplo:

JS:

 $scope.data = []

 $q.all([
    $http.get("data.json"),
    $http.get("more-data.json"),
    ]).then(function(responses) {
      $scope.data = responses.map((resp) => resp.data);
    });

O código acima executa $http.get 2 vezes para dados em arquivos json locais, quando ambos os métodos get são concluídos eles resolvem suas promessas associadas, quando todas as promessas no array são resolvidas, o método .then começa com ambos prometem dados dentro do argumento do array responses.

Os dados são então mapeados para que possam ser mostrados no modelo, podemos então mostrar

HTML:

<ul>
   <li ng-repeat="d in data">
      <ul>
         <li ng-repeat="item in d">{{item.name}}: {{item.occupation}}</li>
      </ul>
   </li>
</ul>

JSON:

[{
  "name": "alice",
  "occupation": "manager"
}, {
  "name": "bob",
  "occupation": "developer"
}]

Evite o antipadrão $q diferido #

Evite este Anti-Padrão #

var meuDeferido = $q.defer();

$http(config).then(function(res) { myDeferred.resolve(res); }, função(erro) { myDeferred.reject(erro); });

return myDeferred.promise;

Não há necessidade de fabricar uma promessa com $q.defer pois o serviço $http já retorna uma promessa.

//INSTEAD
return $http(config);

Simplesmente retorne a promessa criada pelo serviço $http.

Adiando operações usando $q.defer #

Podemos usar $q para adiar operações para o futuro enquanto temos um objeto de promessa pendente no presente, usando $q.defer criamos uma promessa que será resolvida ou rejeitada no futuro.

Este método não é equivalente a usar o construtor $q, pois usamos $q.defer para prometer uma rotina existente que pode ou não retornar (ou já retornou) uma promessa.

Exemplo:

var runAnimation = function(animation, duration) {
    var deferred = $q.defer();
    try {
        ...
        // run some animation for a given duration
        deferred.resolve("done");
    } catch (err) {
        // in case of error we would want to run the error hander of .then
        deferred.reject(err);
    }
    return deferred.promise;
}

// and then
runAnimation.then(function(status) {}, function(error) {})

  1. Certifique-se de sempre retornar um objeto deferred.promise ou corre o risco de um erro ao invocar .then

  2. Certifique-se de sempre resolver ou rejeitar seu objeto adiado ou .then pode não ser executado e você corre o risco de um vazamento de memória
