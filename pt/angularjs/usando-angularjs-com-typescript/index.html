Sintaxe #

  • $scope : ng.IScope - esta é a maneira no typescript para definir o tipo para uma determinada variável.

Usando Agrupamento / Minificação #

A maneira como o $scope é injetado nas funções do construtor do controlador é uma maneira de demonstrar e usar a opção básica de injeção de dependência angular, mas não está pronta para produção, pois não pode ser minificada. Isso porque o sistema de minificação altera os nomes das variáveis ​​e a injeção de dependência do anguar usa os nomes dos parâmetros para saber o que deve ser injetado. Então, por exemplo, a função construtora do ExampleController é reduzida para o código a seguir.

function n(n){this.setUpWatches(n)

e $scope é alterado para n!
para superar isso, podemos adicionar um $inject array(string[]). Para que a DI desse angular saiba o que injetar em qual posição é a função construtora dos controladores.
Assim, o texto datilografado acima muda para

module App.Controllers {
    class Address {
        line1: string;
        line2: string;
        city: string;
        state: string;
    }
    export class SampleController {
        firstName: string;
        lastName: string;
        age: number;
        address: Address;
        setUpWatches($scope: ng.IScope): void {
            $scope.$watch(() => this.firstName, (n, o) => {
                //n is string and so is o
            });
        };
        static $inject : string[] = ['$scope'];
        constructor($scope: ng.IScope) {
            this.setUpWatches($scope);
        }
    }
}

Angular Controllers em Typescript #

Conforme definido no AngularJS Documentação

Quando um Controller é conectado ao DOM através do ng-controller diretiva, o Angular instanciará um novo objeto Controller, usando o função construtora do Controller especificada. Um novo escopo filho será criado e disponibilizado como parâmetro injetável ao Função construtora do controlador como $scope.

Os controladores podem ser feitos muito facilmente usando as classes typescript.

module App.Controllers {
    class Address {
        line1: string;
        line2: string;
        city: string;
        state: string;
    }
    export class SampleController {
        firstName: string;
        lastName: string;
        age: number;
        address: Address;
        setUpWatches($scope: ng.IScope): void {
            $scope.$watch(() => this.firstName, (n, o) => {
                //n is string and so is o
            });
        };
        constructor($scope: ng.IScope) {
            this.setUpWatches($scope);
        }
    }
}

O Javascript resultante é

var App;
(function (App) {
    var Controllers;
    (function (Controllers) {
        var Address = (function () {
            function Address() {
            }
            return Address;
        }());
        var SampleController = (function () {
            function SampleController($scope) {
                this.setUpWatches($scope);
            }
            SampleController.prototype.setUpWatches = function ($scope) {
                var _this = this;
                $scope.$watch(function () { return _this.firstName; }, function (n, o) {
                    //n is string and so is o
                });
            };
            ;
            return SampleController;
        }());
        Controllers.SampleController = SampleController;
    })(Controllers = App.Controllers || (App.Controllers = {}));
})(App || (App = {}));
//# sourceMappingURL=ExampleController.js.map

Depois de fazer a classe do controlador, deixe o módulo js angular sobre o controlador ser simples usando a classe

app
 .module('app')
 .controller('exampleController', App.Controller.SampleController)

Usando o controlador com a sintaxe ControllerAs #

O Controller que criamos pode ser instanciado e usado usando a sintaxe controller as. Isso porque colocamos a variável diretamente na classe do controlador e não no $scope.

Usar controller as someName é separar o controlador do próprio $scope. Portanto, não há necessidade de injetar $scope como a dependência no controlador.

Caminho tradicional :

// we are using $scope object.
app.controller('MyCtrl', function ($scope) {
  $scope.name = 'John';
});

<div ng-controller="MyCtrl">
  {{name}}
</div>

Agora, com controlador como Sintaxe :

// we are using the "this" Object instead of "$scope"
app.controller('MyCtrl', function() {
  this.name = 'John';
});

<div ng-controller="MyCtrl as info">
  {{info.name}}
</div>

Se você instanciar uma “classe” em JavaScript, poderá fazer isso :

var jsClass = function () {
  this.name = 'John';
}
var jsObj = new jsClass();

Então, agora podemos usar a instância jsObj para acessar qualquer método ou propriedade de jsClass.

Em angular, fazemos o mesmo tipo de coisa. Usamos controller como sintaxe para instanciação.

Por que a sintaxe do ControllerAs? #

Função do Controlador #

A função do controlador nada mais é do que apenas uma função construtora do JavaScript. Portanto, quando uma visão carrega o contexto da função(this) é definido para o objeto do controlador.

Caso 1 :

this.constFunction = function() { ... }

Ele é criado no controller object, não no $scope. views não podem acessar as funções definidas no objeto controller.

Exemplo :

<a href="#123" ng-click="constFunction()"></a> // It will not work

Caso 2:

$scope.scopeFunction = function() { ... }

Ele é criado no $scope object, não no controller object. views só podem acessar as funções definidas no objeto $scope.

Exemplo :

<a href="#123" ng-click="scopeFunction()"></a> // It will work

Por que ControllerAs? #

  • A sintaxe ControllerAs torna muito mais claro onde os objetos estão sendo manipulados. Ter oneCtrl.name e anotherCtrl.name torna muito mais fácil identificar que você tem um name atribuído por vários controladores diferentes para propósitos diferentes, mas se ambos usarem o mesmo $scope.name e tiverem dois elementos HTML diferentes em uma página, ambos vinculados a {{name}}, será difícil identificar qual é de qual controlador .

  • Escondendo o $scope e expondo os membros do controller para a view através de um objeto intermediário. Ao definir this.*, podemos expor exatamente o que queremos expor do controlador para a visualização.

      <div ng-controller="FirstCtrl">
      {{ name }}
      <div ng-controller="SecondCtrl">
          {{ name }}
          <div ng-controller="ThirdCtrl">
              {{ name }}
          </div>
      </div>
  </div>

Aqui, no caso acima, {{ name }} será muito confuso de usar e também não sabemos qual deles está relacionado a qual controlador.

<div ng-controller="FirstCtrl as first">
    {{ first.name }}
    <div ng-controller="SecondCtrl as second">
        {{ second.name }}
        <div ng-controller="ThirdCtrl as third">
            {{ third.name }}
        </div>
    </div>
</div>

Por que $scope ? #

  • Use $scope quando precisar acessar um ou mais métodos de $scope como $watch, $digest, $emit, $http etc.
  • limitar quais propriedades e/ou métodos são expostos a $scope, então explicitamente passando-os para $scope conforme necessário.
