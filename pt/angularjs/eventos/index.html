Parâmetros #

Parâmetros Tipos de valores
evento Objeto {nome: “eventName”, targetScope: Escopo, defaultPrevented: false, currentScope: ChildScope}
argumentos dados que foram passados ​​junto com a execução do evento

Usando o sistema de eventos angulares #

$scope.$emit #

Usar $scope.$emit irá disparar um nome de evento para cima através da hierarquia de escopo e notificar o $scope. O ciclo de vida do evento começa no escopo no qual $emit foi chamado.

Wireframe funcionando:

$scope.$transmissão #

Usar $scope.$broadcast irá disparar um evento no $scope. Podemos ouvir esses eventos usando $scope.$on

Wireframe funcionando:

Sintaxe: #

// firing an event upwards
$scope.$emit('myCustomEvent', 'Data to send');

// firing an event downwards
$scope.$broadcast('myCustomEvent', {
  someProp: 'some value'
});

// listen for the event in the relevant $scope
$scope.$on('myCustomEvent', function (event, data) {
  console.log(data); // 'Data from the event'
});

Em vez de $scope você pode usar $rootScope, nesse caso seu evento estará disponível em todos os controladores independentemente do escopo desses controladores

Limpar evento registrado no AngularJS #

O motivo de limpar os eventos registrados porque mesmo o controlador foi destruído o tratamento dos eventos registrados ainda estão vivos. Portanto, o código será executado como inesperado, com certeza.

// firing an event upwards
$rootScope.$emit('myEvent', 'Data to send');

// listening an event
var listenerEventHandler = $rootScope.$on('myEvent', function(){
    //handle code
});

$scope.$on('$destroy', function() {
    listenerEventHandler();
});

Sempre cancele o registro de $rootScope.$on listeners no evento $destory do escopo #

Os ouvintes $rootScope.$on permanecerão na memória se você navegar para outro controlador. Isso criará um vazamento de memória se o controlador ficar fora do escopo.

Não

angular.module('app').controller('badExampleController', badExample);

badExample.$inject = ['$scope', '$rootScope'];
function badExample($scope, $rootScope) {

    $rootScope.$on('post:created', function postCreated(event, data) {});

}

Fazer

angular.module('app').controller('goodExampleController', goodExample);

goodExample.$inject = ['$scope', '$rootScope'];
function goodExample($scope, $rootScope) {

    var deregister = $rootScope.$on('post:created', function postCreated(event, data) {});

    $scope.$on('$destroy', function destroyScope() {
        deregister();
    });

}

Usos e significados #

Esses eventos podem ser usados ​​para comunicação entre 2 ou mais controladores.

$emit despacha um evento para cima através da hierarquia de escopo, enquanto $broadcast despacha um evento para baixo para todos os escopos filhos. Isso foi lindamente explicado aqui.

Pode haver basicamente dois tipos de cenário durante a comunicação entre os controladores:

  1. Quando os controladores têm relacionamento Pai-Filho. (podemos usar principalmente $scope em tais cenários)
  1. Quando os controladores não são independentes entre si e precisam ser informados sobre as atividades uns dos outros. (podemos usar $rootScope em tais cenários)

por exemplo: para qualquer site de comércio eletrônico, suponha que temos ProductListController(que controla a página de listagem de produtos quando qualquer marca de produto é clicada) e CartController (para gerenciar itens do carrinho) . Agora, quando clicamos no botão Add to Cart, ele também deve ser informado ao CartController, para que possa refletir a nova contagem/detalhes de itens do carrinho na barra de navegação do site. Isso pode ser feito usando $rootScope.

Com $scope.$emit

<html>
  <head>
    <script src="https://ajax.googleapis.com/ajax/libs/angularjs/1.4.4/angular.js"></script>
    <script>
        var app = angular.module('app', []);

        app.controller("FirstController", function ($scope) {
            $scope.$on('eventName', function (event, args) {
                $scope.message = args.message;
            });
        });

        app.controller("SecondController", function ($scope) {
            $scope.handleClick = function (msg) {
                $scope.$emit('eventName', {message: msg});
            };
        });

    </script>
  </head>
  <body ng-app="app">
    <div ng-controller="FirstController" style="border:2px ;padding:5px;">
        <h1>Parent Controller</h1>
        <p>Emit Message : {{message}}</p>
        <br />
        <div ng-controller="SecondController" style="border:2px;padding:5px;">
            <h1>Child Controller</h1>
            <input ng-model="msg">
            <button ng-click="handleClick(msg);">Emit</button>
        </div>
    </div>
  </body>
</html>

Com $scope.$broadcast:

<html>
  <head>
    <title>Broadcasting</title>
    <script src="https://ajax.googleapis.com/ajax/libs/angularjs/1.4.4/angular.js"></script>
    <script>
        var app = angular.module('app', []);

        app.controller("FirstController", function ($scope) {
            $scope.handleClick = function (msg) {
                $scope.$broadcast('eventName', {message: msg});
            };

        });

        app.controller("SecondController", function ($scope) {
            $scope.$on('eventName', function (event, args) {
                $scope.message = args.message;
            });
        });

    </script>
  </head>
  <body ng-app="app">
    <div ng-controller="FirstController" style="border:2px solid ; padding:5px;">
        <h1>Parent Controller</h1>
        <input ng-model="msg">
        <button ng-click="handleClick(msg);">Broadcast</button>
        <br /><br />
        <div ng-controller="SecondController" style="border:2px solid ;padding:5px;">
            <h1>Child Controller</h1>
            <p>Broadcast Message : {{message}}</p>
        </div>
    </div>
  </body>
</html>
