Aqui você aprenderá sobre o recurso Diretivas do AngularJS. Abaixo você encontrará informações sobre o que são as Diretivas, bem como exemplos Básicos e Avançados de como usá-las.

Parâmetros #

Parâmetro Detalhes
escopo Propriedade para definir o escopo da diretiva. Pode ser definido como falso, verdadeiro ou como um escopo isolado: { @, =, <, & }.
escopo: falso A diretiva usa o escopo pai. Nenhum escopo criado para diretiva.
escopo: verdadeiro A diretiva herda o escopo pai prototipicamente como um novo escopo filho. Se houver várias diretivas no mesmo elemento solicitando um novo escopo, elas compartilharão um novo escopo.
escopo: { @ } Ligação unidirecional de uma propriedade de escopo de diretiva a um valor de atributo DOM. Como o valor do atributo vinculado ao pai, ele mudará no escopo da diretiva.
escopo: { = } Ligação de atributo bidirecional que altera o atributo no pai se o atributo da diretiva for alterado e vice-versa.
escopo: { < } Ligação unidirecional de uma propriedade de escopo de diretiva e uma expressão de atributo DOM. A expressão é avaliada no pai. Isso observa a identidade do valor pai para que as alterações em uma propriedade de objeto no pai não sejam refletidas na diretiva. Alterações em uma propriedade de objeto em uma diretiva serão refletidas no pai, pois ambas fazem referência ao mesmo objeto
escopo: { & } Permite que a diretiva passe dados para uma expressão a ser avaliada no pai.
compilar: função Essa função é usada para realizar a transformação DOM no modelo de diretiva antes da execução da função de link. Ele aceita tElement (o template da diretiva) e tAttr (lista de atributos declarados na diretiva). Ele não tem acesso ao escopo. Pode retornar uma função que será registrada como uma função post-link ou pode retornar um objeto com as propriedades pre e post que será registrado como as funções pre-link e post-link .
link: função/objeto A propriedade link pode ser configurada como uma função ou objeto. Ele pode receber os seguintes argumentos: scope(directive scope), iElement( elemento DOM onde a diretiva é aplicada ), iAttrs( coleção de atributos do elemento DOM ), controller( array de controladores requeridos pela diretiva ), transcludeFn. É usado principalmente para configurar ouvintes DOM, observar as propriedades do modelo para alterações e atualizar o DOM. Ele é executado depois que o modelo é clonado. Ele é configurado independentemente se não houver função de compilação.
função de pré-link Função de link que é executada antes de qualquer função de link filho. Por padrão, as funções de link da diretiva filho são executadas antes das funções de link da diretiva pai e a função de pré-link permite que o pai vincule primeiro. Um caso de uso é se o filho requer dados do pai.
função pós-link Função de link que os executivos após os elementos filho são vinculados ao pai. É comumente usado para anexar manipuladores de eventos e acessar diretivas filho, mas os dados exigidos pela diretiva filho não devem ser definidos aqui porque a diretiva filho já terá sido vinculada.
restringir: string Define como chamar a diretiva de dentro do DOM. Valores possíveis (assumindo que o nome da nossa diretiva é demoDirective): E - Nome do elemento (<demo-directive></demo-directive>), A - Attribute (<div demo-directive></ div>), C - Classe correspondente (<div class="demo-directive"></div>), M - Por comentário (<!-- diretiva: demo-directive --> ). A propriedade restrict também pode suportar várias opções, por exemplo - restrict: "AC" restringirá a diretiva a Attribute OR Class. Se omitido, o valor padrão será "EA" (Elemento ou Atributo).
requer: ‘demoDirective’ Localize o controlador demoDirective no elemento atual e injete seu controlador como o quarto argumento para a função de vinculação. Lança um erro se não for encontrado.
requer: ‘?demoDirective’ Tente localizar o controlador do demoDirective ou passe null para o link fn se não for encontrado.
requer: ‘^demoDirective’ Localize o controlador do demoDirective pesquisando o elemento e seus pais. Lança um erro se não for encontrado.
exigir: ‘^^demoDirective’ Localize o controlador do demoDirective pesquisando os pais do elemento. Lança um erro se não for encontrado.
exigir: ‘?^demoDirective’ Tente localizar o controlador do demoDirective pesquisando o elemento e seus pais ou passe null para o link fn se não for encontrado.
exigir: ‘?^^demoDirective’ Tente localizar o controlador do demoDirective pesquisando os pais do elemento ou passe null para o link fn se não for encontrado.

Criando e consumindo diretivas personalizadas #

As diretivas são um dos recursos mais poderosos do angularjs. As diretivas angularjs personalizadas são usadas para estender a funcionalidade do html criando novos elementos html ou atributos personalizados para fornecer determinado comportamento a uma tag html.

directiva.js

// Create the App module if you haven't created it yet
var demoApp= angular.module("demoApp", []);

// If you already have the app module created, comment the above line and create a reference of the app module
var demoApp = angular.module("demoApp"); 


// Create a directive using the below syntax
// Directives are used to extend the capabilities of html element 
// You can either create it as an Element/Attribute/class      
// We are creating a directive named demoDirective. Notice it is in CamelCase when we are defining the directive just like ngModel
// This directive will be activated as soon as any this element is encountered in html

demoApp.directive('demoDirective', function () {
    
  // This returns a directive definition object
  // A directive definition object is a simple JavaScript object used for configuring the directive’s behaviour,template..etc
  return {
    // restrict: 'AE', signifies that directive is Element/Attribute directive, 
    // "E" is for element, "A" is for attribute, "C" is for class, and "M" is for comment. 
    // Attributes are going to be the main ones as far as adding behaviors that get used the most.
    // If you don't specify the restrict property it will default to "A"
    restrict :'AE',  

    // The values of scope property decides how the actual scope is created and used inside a directive. These values can be either “false”, “true” or “{}”. This creates an isolate scope for the directive.
    // '@' binding is for passing strings. These strings support {{}} expressions for interpolated values.
    // '=' binding is for two-way model binding. The model in parent scope is linked to the model in the directive's isolated scope.
    // '&' binding is for passing a method into your directive's scope so that it can be called within your directive. 
    // The method is pre-bound to the directive's parent scope, and supports arguments.
    scope: { 
      name: "@",  // Always use small casing here even if it's a mix of 2-3 words
    },                  

    // template replaces the complete element with its text. 
    template: "<div>Hello {{name}}!</div>",
                
    // compile is called during application initialization. AngularJS calls it once when html page is loaded.
    compile: function(element, attributes) {
      element.css("border", "1px solid #cccccc");
                    
      // linkFunction is linked with each element with scope to get the element specific data.
      var linkFunction = function($scope, element, attributes) {
        element.html("Name: <b>"+$scope.name +"</b>");
        element.css("background-color", "#ff00ff");
      };
      return linkFunction;
    }
  };
});

Esta diretiva pode então ser usada em App como:

<html>
   
   <head>
      <title>Angular JS Directives</title>
   </head>
   <body>
   <script src = "http://ajax.googleapis.com/ajax/libs/angularjs/1.3.14/angular.min.js"></script>
   <script src="directive.js"></script>   
    <div ng-app = "demoApp">
       <!-- Notice we are using Spinal Casing here -->  
       <demo-directive name="World"></demo-directive>
     
      </div>
   </body>
</html>

Modelo de objeto de definição de diretiva #

demoApp.directive('demoDirective', function () {
  var directiveDefinitionObject = {
    multiElement:  
    priority:
    terminal:
    scope: {},  
    bindToController: {},
    controller:
    controllerAs:
    require:
    restrict:
    templateNamespace:
    template:
    templateUrl:
    transclude:
    compile: 
    link: function(){}                                  
  };
  return directiveDefinitionObject;
});
  1. multiElement - definido como true e quaisquer nós DOM entre o início e o fim do nome da diretiva serão coletados e agrupados como elementos da diretiva
  2. priority - permite a especificação da ordem de aplicação das diretivas quando várias diretivas são definidas em um único elemento DOM. As diretivas com números mais altos são compiladas primeiro.
  3. terminal - definido como verdadeiro e a prioridade atual será o último conjunto de diretivas a serem executadas
  4. scope - define o escopo da diretiva
  5. bind to controller - vincula as propriedades do escopo diretamente ao controlador de diretiva
  6. controller - função construtora do controlador
  7. require - requer outra diretiva e injeta seu controlador como o quarto argumento para a função de ligação
  8. controllerAs - nome de referência ao controlador no escopo da diretiva para permitir que o controlador seja referenciado a partir do modelo de diretiva.
  9. restrict - restringe a diretiva a Elemento, Atributo, Classe ou Comentário
  10. templateNameSpace - define o tipo de documento usado pelo modelo de diretiva: html, svg ou math. html é o padrão
  11. template - marcação html que substitui o conteúdo do elemento da diretiva por padrão, ou envolve o conteúdo do elemento da diretiva se transclude for true
  12. templateUrl - url fornecido de forma assíncrona para o modelo
  13. transclude - Extraia o conteúdo do elemento onde a diretiva aparece e o disponibiliza para a diretiva. Os conteúdos são compilados e fornecidos à diretiva como uma função de transclusão.
  14. compile - função para transformar o template DOM
  15. link - usado somente se a propriedade de compilação não estiver definida. A função link é responsável por registrar os ouvintes do DOM, bem como atualizar o DOM. Ele é executado após a clonagem do modelo.

Como criar um componente reutilizável usando a diretiva #

As diretivas AngularJS são o que controla a renderização do HTML dentro de um aplicativo AngularJS. Eles podem ser um elemento Html, atributo, classe ou um comentário. As diretivas são usadas para manipular o DOM, anexando novos comportamentos aos elementos HTML, vinculação de dados e muito mais. Alguns exemplos de diretivas que o angular fornece são ng-model, ng-hide, ng-if.

Da mesma forma, pode-se criar sua própria diretiva personalizada e torná-la reutilizável. Para criar diretivas personalizadas Referência. O sentido por trás da criação de diretivas reutilizáveis ​​é fazer um conjunto de diretivas/componentes escritos por você exatamente como o angularjs nos fornece usando angular.js . Essas diretivas reutilizáveis ​​podem ser particularmente muito úteis quando você tem um conjunto de aplicativos/aplicativos que requer um comportamento, aparência e comportamento consistentes. Um exemplo desse componente reutilizável pode ser uma barra de ferramentas simples que você pode usar em seu aplicativo ou em aplicativos diferentes, mas deseja que eles se comportem da mesma forma ou tenham a mesma aparência.

Em primeiro lugar, crie uma pasta chamada resuableComponents na pasta do seu aplicativo e faça reusableModuleApp.js

reusableModuleApp.js:

(function(){

 var reusableModuleApp = angular.module('resuableModuleApp', ['ngSanitize']);

 //Remember whatever dependencies you have in here should be injected in the app module where it is intended to be used or it's scripts should be included in your main app  
                               //We will be injecting ng-sanitize

 resubaleModuleApp.directive('toolbar', toolbar)

 toolbar.$inject=['$sce'];

 function toolbar($sce){

     return{ 
     restrict :'AE',
        //Defining below isolate scope actually provides window for the directive to take data from app that will be using this.
        scope : {
                value1: '=',
                value2: '=',
                },

         }
     template : '<ul>  <li><a ng-click="Add()" href="">{{value1}}</a></li>  <li><a ng-click="Edit()" href="#">{{value2}}</a></li> </ul> ',
         link : function(scope, element, attrs){
           
              //Handle's Add function
              scope.Add = function(){


              };

              //Handle's Edit function
              scope.Edit = function(){

              };
     }
  }

});

mainApp.js:

(function(){
   var mainApp = angular.module('mainApp', ['reusableModuleApp']); //Inject resuableModuleApp in your application where you want to use toolbar component
   
   mainApp.controller('mainAppController', function($scope){
      $scope.value1 = "Add";
      $scope.value2 = "Edit"; 
   
   });
 
 });

index.html:

 <!doctype html>
 <html ng-app="mainApp">
 <head>
  <title> Demo Making a reusable component
 <head>
  <body ng-controller="mainAppController">
   
     <!-- We are providing data to toolbar directive using mainApp'controller -->
     <toolbar value1="value1" value2="value2"></toolbar>
  
    <!-- We need to add the dependent js files on both apps here -->
     <script src="js/angular.js"></script>
     <script src="js/angular-sanitize.js"></script>
    
     <!-- your mainApp.js should be added afterwards --->
      <script src="mainApp.js"></script>

     <!-- Add your reusable component js files here -->
      <script src="resuableComponents/reusableModuleApp.js"></script>

  </body>
 </html>

Diretiva são componentes reutilizáveis ​​por padrão. Quando você cria diretivas em um módulo angular separado, ele realmente o torna exportável e reutilizável em diferentes aplicativos angularJs. Novas diretivas podem simplesmente ser adicionadas dentro de reusableModuleApp.js e reusableModuleApp pode ter seu próprio controlador, serviços, objeto DDO dentro da diretiva para definir o comportamento.

1: https://www.wikiod.com/pt/angularjs/diretivas-personalizadas#Criando e consumindo diretivas personalizadas

Exemplo de diretiva básica #

superman-directive.js

angular.module('myApp', [])
  .directive('superman', function() {
    return {
      // restricts how the directive can be used
      restrict: 'E',
      templateUrl: 'superman-template.html',
      controller: function() {
        this.message = "I'm superman!"
      },
      controllerAs: 'supermanCtrl',
      // Executed after Angular's initialization. Use commonly 
      // for adding event handlers and DOM manipulation
      link: function(scope, element, attributes) {
        element.on('click', function() {
          alert('I am superman!')
        });
      }
    }
  });

superman-template.html

<h2>{{supermanCtrl.message}}</h2>

index.html

<!DOCTYPE html>
<html lang="en">
<head>
  <meta charset="UTF-8">
  <title>Document</title>
  <script src="https://ajax.googleapis.com/ajax/libs/angularjs/1.5.0/angular.js"></script>
  <script src="superman-directive.js"><script/>
</head>
<body>
<div ng-app="myApp">
  <superman></superman>
</div>
</body>
</html>

Você pode conferir mais sobre as funções restrict e link da diretiva na documentação oficial do AngularJS sobre Diretivas

Diretiva básica com template e escopo isolado #

A criação de uma diretiva personalizada com escopo isolado separará o escopo dentro da diretiva do escopo externo, para evitar que nossa diretiva altere acidentalmente os dados no escopo pai e o restrinja de ler dados privados de o escopo pai.

Para criar um escopo isolado e ainda permitir que nossa diretiva personalizada se comunique com o escopo externo, podemos usar a opção scope que descreve como mapear as ligações do escopo interno da diretiva com o escopo externo.

As ligações reais são feitas com atributos extras anexados à diretiva. As configurações de vinculação são definidas com a opção scope e um objeto com pares chave-valor:

  • Uma chave, que corresponde à propriedade de escopo isolado da nossa diretiva
  • Um valor, que informa ao Angular como vincular o escopo interno da diretiva a um atributo correspondente

Exemplo simples de uma diretiva com escopo isolado:

var ProgressBar = function() {
  return {
    scope: { // This is how we define an isolated scope
      current: '=', // Create a REQUIRED bidirectional binding by using the 'current' attribute
      full: '=?maxValue' // Create an OPTIONAL (Note the '?'): bidirectional binding using 'max-value' attribute to the `full` property in our directive isolated scope 
    }
    template: '<div class="progress-back">' +
              '  <div class="progress-bar"' +
              '       ng-style="{width: getProgress()}">' +
              '  </div>' + 
              '</div>',
    link: function(scope, el, attrs) {
      if (scope.full === undefined) {
        scope.full = 100;
      }
      scope.getProgress = function() {
        return (scope.current / scope.size * 100) + '%';
      }
    }
  }
}

ProgressBar.$inject = [];
angular.module('app').directive('progressBar', ProgressBar);

Exemplo de como usar esta diretiva e vincular dados do escopo do controlador ao escopo interno da diretiva:

Controlador:

angular.module('app').controller('myCtrl', function($scope) {
    $scope.currentProgressValue = 39;
    $scope.maxProgressBarValue = 50;
});

Visão:

<div ng-controller="myCtrl">
    <progress-bar current="currentProgressValue"></progress-bar>
    <progress-bar current="currentProgressValue" max-value="maxProgressBarValue"></progress-bar>
</div>

Construindo um componente reutilizável #

As diretivas podem ser usadas para construir componentes reutilizáveis. Aqui está um exemplo de um componente “caixa de usuário”:

userBox.js

angular.module('simpleDirective', []).directive('userBox', function() {
  return {
    scope: {
        username: '=username',
        reputation: '=reputation'
    },
    templateUrl: '/path/to/app/directives/user-box.html'
  };
});

Controller.js

var myApp = angular.module('myApp', ['simpleDirective']);

myApp.controller('Controller', function($scope) {

    $scope.user = "John Doe";
    $scope.rep = 1250;

    $scope.user2 = "Andrew";
    $scope.rep2 = 2850;

});

minhaPágina.js

<html lang="en" ng-app="myApp">
  <head>
    <script src="/path/to/app/angular.min.js"></script>
    <script src="/path/to/app/js/controllers/Controller.js"></script>
    <script src="/path/to/app/js/directives/userBox.js"></script>
  </head>

  <body>
  
    <div ng-controller="Controller">
        <user-box username="user" reputation="rep"></user-box>
        <user-box username="user2" reputation="rep2"></user-box>
    </div>
    
  </body>
</html>

user-box.html

<div>{{username}}</div>
<div>{{reputation}} reputation</div>

O resultado será:

John Doe
1250 reputation
Andrew

Decorador de diretivas #

Às vezes, você pode precisar de recursos adicionais de uma diretiva. Em vez de reescrever (copiar) a diretiva, você pode modificar como a diretiva se comporta.

O decorador será executado durante a fase $inject.

Para fazer isso, forneça um .config para seu módulo. A diretiva é chamada myDirective, então você precisa configurar myDirectiveDirective. (isso em uma convenção angular [leia sobre provedores] ).

Este exemplo irá alterar o templateUrl da diretiva:

angular.module('myApp').config(function($provide){
        $provide.decorator('myDirectiveDirective', function($delegate){
             var directive = $delegate[0]; // this is the actual delegated, your directive
             directive.templateUrl = 'newTemplate.html'; // you change the directive template
        return $delegate;
    })
});

Este exemplo adiciona um evento onClick ao elemento de diretiva quando clicado, isso acontece durante a fase de compilação.

angular.module('myApp').config(function ($provide) {
        $provide.decorator('myDirectiveTwoDirective', function ($delegate) {
            var directive = $delegate[0];
            var link = directive.link; // this is directive link phase
            directive.compile = function () { // change the compile of that directive
                return function (scope, element, attrs) {
                    link.apply(this, arguments); // apply this at the link phase
                    element.on('click', function(){ // when add an onclick that log hello when the directive is clicked.
                            console.log('hello!');
                    }); 
                };
            };
            return $delegate;
        });

    });

Abordagem semelhante pode ser usada para provedores e serviços.

Herança e interoperabilidade de diretivas #

As diretivas angulares js podem ser aninhadas ou tornar-se interoperáveis.

Neste exemplo, a diretiva Adir expõe à diretiva Bdir seu controlador $scope, já que Bdir requer Adir.

angular.module('myApp',[]).directive('Adir', function () {
        return {
            restrict: 'AE',
            controller: ['$scope', function ($scope) {
                    $scope.logFn = function (val) {
                            console.log(val);
                        }
                  }]
            }
    })

Certifique-se de definir require: ‘^Adir’ (veja a documentação angular, algumas versões não requerem o caractere ^).

.directive('Bdir', function () {
        return {
            restrict: 'AE',
            require: '^Adir', // Bdir require Adir
            link: function (scope, elem, attr, Parent) { 
            // Parent is Adir but can be an array of required directives.
                elem.on('click', function ($event) {
                    Parent.logFn("Hello!"); // will log "Hello! at parent dir scope
                    scope.$apply(); // apply to parent scope.
                });
            }
        }
    }]);

Você pode aninhar sua diretiva desta maneira:

<div a-dir><span b-dir></span></div>
<a-dir><b-dir></b-dir> </a-dir>

Não é necessário que as diretivas estejam aninhadas em seu HTML.
