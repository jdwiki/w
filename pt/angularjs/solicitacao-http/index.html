Usando $http dentro de um controller #

O serviço $http é uma função que gera uma solicitação HTTP e retorna uma promessa.

Uso Geral

// Simple GET request example:
$http({
  method: 'GET',
  url: '/someUrl'
}).then(function successCallback(response) {
    // this callback will be called asynchronously
    // when the response is available
  }, function errorCallback(response) {
    // called asynchronously if an error occurs
    // or server returns response with an error status.
  });

Uso dentro do controlador

appName.controller('controllerName',
    ['$http', function($http){

    // Simple GET request example:
    $http({
        method: 'GET',
        url: '/someUrl'
    }).then(function successCallback(response) {
        // this callback will be called asynchronously
        // when the response is available
    }, function errorCallback(response) {
        // called asynchronously if an error occurs
        // or server returns response with an error status.
    });
}])

Métodos de atalho

O serviço $http também possui métodos de atalho. Leia sobre métodos http aqui

Sintaxe

$http.get('/someUrl', config).then(successCallback, errorCallback);
$http.post('/someUrl', data, config).then(successCallback, errorCallback);

Métodos de atalho

  • $http.get
  • $http.head
  • $http.post
  • $http.put
  • $http.delete
  • $http.jsonp
  • $http.patch

Usando a solicitação $http em um serviço #

As solicitações HTTP são amplamente usadas repetidamente em todos os aplicativos da Web, portanto, é aconselhável escrever um método para cada solicitação comum e usá-lo em vários locais em todo o aplicativo.

Crie um httpRequestsService.js

httpRequestsService.js

appName.service('httpRequestsService', function($q, $http){

    return {
        // function that performs a basic get request
        getName: function(){
            // make sure $http is injected
            return $http.get("/someAPI/names")
                .then(function(response) {
                    // return the result as a promise
                    return response;
                }, function(response) {
                    // defer the promise
                    return $q.reject(response.data);
                });
        },

        // add functions for other requests made by your app
        addName: function(){
            // some code...
        }
    }
})

O serviço acima realizará uma solicitação de obtenção dentro do serviço. Isso estará disponível para qualquer controlador em que o serviço foi injetado.

Amostra de uso

appName.controller('controllerName',
    ['httpRequestsService', function(httpRequestsService){

        // we injected httpRequestsService service on this controller
        // that made the getName() function available to use.
        httpRequestsService.getName()
            .then(function(response){
                // success
            }, function(error){
                // do something with the error
            })
    }])

Usando essa abordagem, agora podemos usar httpRequestsService.js a qualquer momento e em qualquer controlador.

Tempo de uma solicitação $http #

As solicitações $http requerem tempo que varia dependendo do servidor, algumas podem levar alguns milissegundos e algumas podem levar alguns segundos. Muitas vezes, o tempo necessário para recuperar os dados de uma solicitação é crítico. Supondo que o valor da resposta seja uma matriz de nomes, considere o seguinte exemplo:

Incorreta

$scope.names = [];

$http({
    method: 'GET',
    url: '/someURL'
}).then(function successCallback(response) {
            $scope.names = response.data;
        },
        function errorCallback(response) {
            alert(response.status);
        });

alert("The first name is: " + $scope.names[0]);

Acessar $scope.names[0] logo abaixo da requisição $http geralmente gera um erro - esta linha de código é executada antes que a resposta seja recebida do servidor.

Correto

$scope.names = [];

$scope.$watch('names', function(newVal, oldVal) {
    if(!(newVal.length == 0)) {
        alert("The first name is: " + $scope.names[0]);
    }
});

$http({
    method: 'GET',
    url: '/someURL'
}).then(function successCallback(response) {
            $scope.names = response.data;
        },
        function errorCallback(response) {
            alert(response.status);
        });

Usando o serviço [$watch][1] acessamos o array $scope.names somente quando a resposta é recebida. Durante a inicialização, a função é chamada mesmo que $scope.names tenha sido inicializado antes, portanto, verificar se o newVal.length é diferente de 0 é necessário. Esteja ciente - quaisquer alterações feitas em $scope.names acionarão a função watch.

[1]: https://www.wikiod.com/pt/angularjs/passo-a-passo-do-loop-de-digestao#$digerir e $assistir
