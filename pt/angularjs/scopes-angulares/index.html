Angular usa uma árvore de escopos para vincular a lógica (de controladores, diretivas, etc) à visualização e é o principal mecanismo por trás da detecção de alterações no AngularJS. Uma referência mais detalhada para escopos pode ser encontrada em docs.angularjs.org

A raiz da árvore é acessível via serviço injetável $rootScope. Todos os $scopes filho herdam os métodos e propriedades de seu $scope pai, permitindo que os filhos acessem os métodos sem o uso de Angular Services.

Uma função disponível em todo o aplicativo #

Tenha cuidado, essa abordagem pode ser considerada um design ruim para aplicativos angulares, pois exige que os programadores se lembrem de onde as funções são colocadas na árvore de escopo e estejam cientes da herança de escopo. Em muitos casos, seria preferível injetar um serviço (http://stackoverflow.com/questions/23659315/angular-practice-using-scope-inheritance-vs-injection).

Este exemplo mostra apenas como a herança de escopo pode ser usada para nossas necessidades e como você pode aproveitá-la, e não as práticas recomendadas de criação de um aplicativo inteiro.

Em alguns casos, poderíamos aproveitar a herança de escopo e definir uma função como uma propriedade do rootScope. Dessa forma - todos os escopos no aplicativo (exceto escopos isolados) herdarão essa função e ela poderá ser chamada de qualquer lugar no aplicativo.

angular.module('app', [])
.run(['$rootScope', function($rootScope){
    var messages = []
    $rootScope.addMessage = function(msg){
        messages.push(msg);
    }
}]);

<div ng-app="app">
    <a ng-click="addMessage('hello world!')">it could be accsessed from here</a>
    <div ng-include="inner.html"></div>
</div>

interior.html:

<div>
    <button ng-click="addMessage('page')">and from here to!</button>
</div>

Evite herdar valores primitivos #

Em javascript, atribuir um valor não primitivo (como Objeto, Matriz, Função e muitos mais), mantém uma referência (um endereço na memória) ao valor atribuído.

Atribuir um valor primitivo (String, Number, Boolean ou Symbol) a duas variáveis ​​diferentes e alterar uma não alterará ambas:

var x = 5;
var y = x;
y = 6;
console.log(y === x, x, y); //false, 5, 6

Mas com um valor não primitivo, já que ambas as variáveis ​​estão simplesmente mantendo referências ao mesmo objeto, mudar uma variável vai mudar a outra:

var x = { name : 'John Doe' };
var y = x;
y.name = 'Jhon';
console.log(x.name === y.name, x.name, y.name); //true, John, John

Em angular, quando um escopo é criado, ele recebe todas as propriedades de seu pai No entanto, alterar as propriedades posteriormente afetará o escopo pai apenas se for um valor não primitivo:

angular.module('app', [])
.controller('myController', ['$scope', function($scope){
    $scope.person = { name: 'John Doe' }; //non-primitive
    $scope.name = 'Jhon Doe'; //primitive
}])
.controller('myController1', ['$scope', function($scope){}]);

<div ng-app="app" ng-controller="myController">
    binding to input works: {{person.name}}<br/>
    binding to input does not work: {{name}}<br/>
    <div ng-controller="myController1">
        <input ng-model="person.name" />
        <input ng-model="name" />
    </div>
</div>

Lembre-se: em Angular os escopos podem ser criados de várias maneiras (como diretivas embutidas ou personalizadas, ou a função $scope.$new()), e manter o controle da árvore de escopo é provavelmente impossível.

Usar apenas valores não primitivos como propriedades de escopo o manterá no lado seguro (a menos que você precise de uma propriedade para não herdar ou outros casos em que você esteja ciente da herança de escopo).

Exemplo básico de herança $scope #

angular.module('app', [])
.controller('myController', ['$scope', function($scope){
    $scope.person = { name: 'John Doe' };
}]);

<div ng-app="app" ng-conroller="myController">
    <input ng-model="person.name" />
    <div ng-repeat="number in [0,1,2,3]">
        {{person.name}} {{number}}
    </div>
</div>

Neste exemplo, a diretiva ng-repeat cria um novo escopo para cada um de seus filhos recém-criados.

Esses escopos criados são filhos de seu escopo pai (neste caso o escopo criado por myController) e, portanto, herdam todas as suas proporções, como person.

Como você pode limitar o escopo de uma diretiva e por que você faria isso? #

O escopo é usado como a “cola” que usamos para comunicar entre o controlador pai, a diretiva e o modelo de diretiva. Sempre que o aplicativo AngularJS é inicializado, um objeto rootScope é criado. Cada escopo criado por controladores, diretivas e serviços é prototipicamente herdado de rootScope.

Sim, podemos limitar o escopo de uma diretiva. Podemos fazê-lo criando um escopo isolado para a diretiva.

Existem 3 tipos de escopos de diretivas:

  1. Escopo: Falso (a diretiva usa seu escopo pai)
  2. Escopo: Verdadeiro (diretiva ganha um novo escopo)
  3. Escopo: { } (Diretiva ganha um novo escopo isolado)

Diretivas com o novo escopo isolado: quando criamos um novo escopo isolado, ele não será herdado do escopo pai. Esse novo escopo é chamado de escopo isolado porque é completamente desanexado de seu escopo pai. Por quê? devemos usar escopo isolado: Devemos usar escopo isolado quando queremos criar uma diretiva personalizada porque isso garantirá que nossa diretiva seja genérica e colocada em qualquer lugar dentro do aplicativo. O escopo pai não vai interferir no escopo da diretiva.

Exemplo de escopo isolado:

var app = angular.module("test",[]);

app.controller("Ctrl1",function($scope){
    $scope.name = "Prateek";
    $scope.reverseName = function(){
        $scope.name = $scope.name.split('').reverse().join('');
    };
});
app.directive("myDirective", function(){
    return {
        restrict: "EA",
        scope: {},
        template: "<div>Your name is : {{name}}</div>"+
        "Change your name : <input type='text' ng-model='name'/>"
    };
});

Existem 3 tipos de prefixos que o AngularJS fornece para escopo isolado, são eles:

  1. “@” (encadernação de texto/encadernação unidirecional)
  2. “=” ( Vinculação direta do modelo / vinculação bidirecional )
  3. “&” (vinculação de comportamento/vinculação de método)

Todos esses prefixos recebem dados dos atributos do elemento diretivo como:

<div my-directive 
  class="directive"
  name="{{name}}" 
  reverse="reverseName()" 
  color="color" >
</div>

Criando eventos $scope personalizados #

Assim como os elementos HTML normais, é possível que $scopes tenha seus próprios eventos. Os eventos $scope podem ser inscritos da seguinte maneira:

 $scope.$on('my-event', function(event, args) {
    console.log(args); // { custom: 'data' }
});

Se você precisar cancelar o registro de um ouvinte de eventos, a função $on retornará uma função de desvinculação. Para continuar com o exemplo acima:

var unregisterMyEvent = $scope.$on('my-event', function(event, args) {
    console.log(args); // { custom: 'data' }
    unregisterMyEvent();
});

Há duas maneiras de acionar seu próprio evento $scope personalizado $broadcast e $emit. Para notificar os pais sobre o escopo de um evento específico, use $emit

$scope.$emit('my-event', { custom: 'data' });

O exemplo acima acionará qualquer ouvinte de evento para my-event no escopo pai e continuará na árvore de escopo até $rootScope, a menos que um ouvinte chame stopPropagation no evento. Apenas eventos acionados com $emit podem chamar stopPropagation

O inverso de $emit é $broadcast, que acionará todos os ouvintes de evento em todos os escopos filho na árvore de escopo que são filhos do escopo que chamou $broadcast.

$scope.$broadcast('my-event', { custom: 'data' });

Eventos acionados com $broadcast não podem ser cancelados.

Usando funções $scope #

Embora declarar uma função no $rootscope tenha suas vantagens, também podemos declarar uma função $scope qualquer parte do código que é injetado pelo serviço $scope. Controlador, por exemplo.

Controlador

myApp.controller('myController', ['$scope', function($scope){
    $scope.myFunction = function () {
        alert("You are in myFunction!");
    };
}]);

Agora você pode chamar sua função do controlador usando:

$scope.myfunction();

Ou via HTML que está sob esse controlador específico:

<div ng-controller="myController">
    <button ng-click="myFunction()"> Click me! </button>
</div>

Diretiva

Uma diretiva angular é outro lugar onde você pode usar seu escopo:

myApp.directive('triggerFunction', function() {
    return {
        scope: {
            triggerFunction: '&'
        },
        link: function(scope, element) {
            element.bind('mouseover', function() {
                scope.triggerFunction();
            });
        }
    };
});

E no seu código HTML no mesmo controlador:

<div ng-controller="myController">
    <button trigger-function="myFunction()"> Hover over me! </button>
</div>

Claro, você pode usar ngMouseover para a mesma coisa, mas o que há de especial nas diretivas é que você pode personalizá-las da maneira que desejar. E agora você sabe como usar suas funções $scope dentro delas, seja criativo!
