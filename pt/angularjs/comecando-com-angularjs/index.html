Começando #

Crie um novo arquivo HTML e cole o seguinte conteúdo:

<!DOCTYPE html>
<html ng-app>
<head>
  <title>Hello, Angular</title>
  <script src="https://code.angularjs.org/1.5.8/angular.min.js"></script>
</head>
<body ng-init="name='World'">
  <label>Name</label>
  <input ng-model="name" />
  <span>Hello, {{ name }}!</span>
  <p ng-bind="name"></p>
</body>
</html>

Demonstração ao vivo

Ao abrir o arquivo com um navegador, você verá um campo de entrada seguido pelo texto Hello, World!. A edição do valor na entrada atualizará o texto em tempo real, sem a necessidade de atualizar a página inteira.

Explicação:

  1. Carregue a estrutura Angular de uma Rede de Entrega de Conteúdo.

    <script src="https://code.angularjs.org/1.5.8/angular.min.js"></script>

  2. Defina o documento HTML como um aplicativo Angular com a diretiva ng-app

    <html ng-app>

  3. Inicialize a variável name usando ng-init

    <body ng-init=" name = 'World' ">

Observe que ng-init deve ser usado apenas para fins de demonstração e teste. Ao construir uma aplicação real, os controladores devem inicializar os dados.

  1. Vincule dados do modelo à exibição em controles HTML. Vincule um <input> à propriedade name com ng-model

    <input ng-model="name" />

  2. Exiba o conteúdo do modelo usando chaves duplas {{ }}

    <span>Hello, {{ name }}</span>

  3. Outra maneira de vincular a propriedade name é usar ng-bind em vez de handlebars"{{ }}"

     <span ng-bind="name"></span>

As últimas três etapas estabelecem a two way data-binding. As alterações feitas na entrada atualizam o model, que é refletido no view.

Há uma diferença entre usar o guidão e o ng-bind. Se você usar handlebars, poderá ver o Hello, {{name}} real quando a página for carregada antes que a expressão seja resolvida (antes dos dados serem carregados), enquanto que se você usar ng-bind, ele mostrará apenas o dados quando o nome é resolvido. Como alternativa, a diretiva ng-cloak pode ser usada para evitar que os guidões sejam exibidos antes de serem compilados.

Apresentando todas as construções comuns do Angular #

O exemplo a seguir mostra construções comuns do AngularJS em um arquivo:

<!DOCTYPE html>
<html ng-app="myDemoApp">
  <head>
    <style>.started { background: gold; }</style>
    <script src="https://code.angularjs.org/1.5.8/angular.min.js"></script>
    <script>
      function MyDataService() {
        return {
          getWorlds: function getWorlds() {
            return ["this world", "another world"];
          }
        };
      }

      function DemoController(worldsService) {
        var vm = this;
        vm.messages = worldsService.getWorlds().map(function(w) {
          return "Hello, " + w + "!";
        });
      }

      function startup($rootScope, $window) {
        $window.alert("Hello, user! Loading worlds...");
        $rootScope.hasStarted = true;
      }
      
      angular.module("myDemoApp", [/* module dependencies go here */])
        .service("worldsService", [MyDataService])
        .controller("demoController", ["worldsService", DemoController])
        .config(function() {
          console.log('configuring application');
        })
        .run(["$rootScope", "$window", startup]);
    </script>
  </head>
  <body ng-class="{ 'started': hasStarted }" ng-cloak>
    <div ng-controller="demoController as vm">
      <ul>
        <li ng-repeat="msg in vm.messages">{{ msg }}</li>
      </ul>
    </div>
  </body>
</html>

Cada linha do arquivo é explicada abaixo:

Demonstração ao vivo

  1. ng-app="myDemoApp", a diretiva ngApp que inicializa o aplicativo e informa ao angular que um elemento DOM é controlado por um angular.module específico chamado "myDemoApp";
  2. <script src="angular.min.js"> é o primeiro passo na inicialização da biblioteca AngularJS;

Três funções (MyDataService, DemoController e startup) são declaradas, que são usadas (e explicadas) abaixo.

  1. angular.module(...) usado com um array como segundo argumento cria um novo módulo. Este array é usado para fornecer uma lista de dependências do módulo. Neste exemplo nós encadeamos chamadas no resultado da função module(...);

  2. .service(...) cria um Angular Service e retorna o módulo para encadeamento;

  3. .controller(...) cria um Angular Controller e retorna o módulo para encadeamento;

  4. .config(...) Use este método para registrar o trabalho que precisa ser executado no carregamento do módulo.

  5. .run(...) garante que o código seja executado na inicialização e recebe um array de itens como parâmetro. Use este método para registrar o trabalho que deve ser executado quando o injetor terminar de carregar todos os módulos.

    • the first item is letting Angular know that the startup function requires the built-in $rootScope service to be injected as an argument;
    • the second item is letting Angular know that the startup function requires the built-in $window service to be injected as an argument;
    • the last item in the array, startup, is the actual function to run on startup;

  6. ng-class é a diretiva ngClass para definir uma class dinâmica, e neste exemplo utiliza hasStarted no $rootScope dinamicamente

  7. ng-cloak é uma diretiva para evitar que o template html não renderizado do Angular (por exemplo, “{{ msg }}”) seja mostrado brevemente antes que o Angular tenha carregado completamente o aplicativo.

  8. ng-controller é a diretiva que pede ao Angular para instanciar um novo controlador de nome específico para orquestrar aquela parte do DOM;

  9. ng-repeat é a diretiva para fazer o Angular iterar sobre uma coleção e clonar um template DOM para cada item;

  10. {{ msg }} mostra interpolation: renderização no local de uma parte do escopo ou controlador;

A importância do escopo #

Como o Angular usa HTML para estender uma página da Web e Javascript simples para adicionar lógica, fica fácil criar uma página da Web usando ng-app, *ng-controller * e algumas diretivas internas, como ng-if, ng-repeat etc. Com a nova sintaxe controllerAs, os recém-chegados ao Angular os usuários podem anexar funções e dados ao seu controlador ao invés de usar $scope.

No entanto, mais cedo ou mais tarde, é importante entender o que exatamente é essa coisa de $scope. Ele continuará aparecendo em exemplos, por isso é importante ter algum entendimento.

A boa notícia é que é um conceito simples, mas poderoso.

Quando você cria o seguinte:

<div ng-app="myApp">
 <h1>Hello {{ name }}</h1>
</div>

Onde nome mora?

A resposta é que o Angular cria um objeto $rootScope. Este é simplesmente um objeto Javascript regular e, portanto, name é uma propriedade no objeto $rootScope:

angular.module("myApp", [])
  .run(function($rootScope) {
    $rootScope.name = "World!";
  });

E assim como no escopo global em Javascript, geralmente não é uma boa ideia adicionar itens ao escopo global ou $rootScope.

É claro que, na maioria das vezes, criamos um controlador e colocamos nossa funcionalidade necessária nesse controlador. Mas quando criamos um controlador, o Angular faz mágica e cria um objeto $scope para esse controlador. Isso às vezes é chamado de escopo local.

Então, criando o seguinte controller:

<div ng-app="myApp">
  <div ng-controller="MyController">
    <h1>Hello {{ name }}</h1>
  </div>
</div>

permitiria que o escopo local fosse acessível através do parâmetro $scope.

angular.module("myApp", [])
  .controller("MyController", function($scope) {
    $scope.name = "Mr Local!";
  });

Um controlador sem um parâmetro $scope pode simplesmente não precisar dele por algum motivo. Mas é importante perceber que, mesmo com a sintaxe controllerAs, o escopo local existe.

Como $scope é um objeto JavaScript, Angular magicamente o configura para herdar prototipicamente de $rootScope. E como você pode imaginar, pode haver uma cadeia de escopos. Por exemplo, você pode criar um modelo em um controlador pai e anexá-lo ao escopo do controlador pai como $scope.model.

Então, através da cadeia de protótipos, um controlador filho pode acessar o mesmo modelo localmente com $scope.model.

Nada disso é inicialmente evidente, pois é apenas o Angular fazendo sua mágica em segundo plano. Mas entender o $scope é um passo importante para entender como o Angular funciona.

Minificação em Angular #

O que é Minificação?

É o processo de remover todos os caracteres desnecessários do código-fonte sem alterar sua funcionalidade.

Sintaxe Normal

Se usarmos a sintaxe angular normal para escrever um controlador, depois de minimizar nossos arquivos, ele quebrará nossa funcionalidade.

Controlador (antes da minificação):

var app = angular.module('mainApp', []);    
app.controller('FirstController', function($scope) {
    $scope.name= 'Hello World !';  
});

Depois de usar a ferramenta de minificação, ela será minificada como abaixo.

var app=angular.module("mainApp",[]);app.controller("FirstController",function(e){e.name= 'Hello World !'})

Aqui, a minificação removeu espaços desnecessários e a variável $scope do código. Então, quando usamos esse código minificado, ele não imprimirá nada à vista. Porque $scope é uma parte crucial entre o controlador e a visualização, que agora é substituída pela pequena variável ’e’. Portanto, quando você executa o aplicativo, ele gera um erro de dependência ’e’ do provedor desconhecido.

Existem duas maneiras de anotar seu código com informações de nome de serviço que são seguras para minificação:

Sintaxe de anotação embutida

var app = angular.module('mainApp', []);    
app.controller('FirstController', ['$scope', function($scope) {
    $scope.message = 'Hello World !'; 
}]);

$injetar sintaxe de anotação de propriedade

FirstController.$inject = ['$scope'];
var FirstController = function($scope) {
    $scope.message = 'Hello World !'; 
}

var app = angular.module('mainApp', []);
app.controller('FirstController', FirstController);

Após minificação, este código será

var app=angular.module("mainApp",[]);app.controller("FirstController",["$scope",function(a){a.message="Hello World !"}]);

Aqui, o angular considerará a variável ‘a’ para ser tratada como $scope e exibirá a saída como ‘Hello World !’.

O Angular Hello World mais simples possível. #

Angular 1 é basicamente um compilador DOM. Podemos passá-lo em HTML, como um modelo ou apenas como uma página da Web normal, e depois compilar um aplicativo.

Podemos dizer ao Angular para tratar uma região da página como uma expressão usando a sintaxe de estilo do guidão {{ }}. Qualquer coisa entre as chaves será compilada, assim:

{{ 'Hello' + 'World' }}

Isso irá produzir:

HelloWorld

do aplicativo #

Dizemos ao Angular qual parte do nosso DOM deve ser tratada como o modelo mestre usando a diretiva do ng-app. Uma diretiva é um atributo ou elemento personalizado que o compilador de template Angular sabe como lidar. Vamos adicionar uma diretiva ng-app agora:

<html>
  <head>
    <script src="/angular.js"></script>
  </head>
  <body ng-app>
    {{ 'Hello' + 'World' }}
  </body>
</html>

Agora eu disse ao elemento body para ser o modelo raiz. Tudo nele será compilado.

Diretrizes #

As diretivas são diretivas do compilador. Eles estendem os recursos do compilador Angular DOM. É por isso que Misko, o criador do Angular, descreve o Angular como:

“O que um navegador da web teria sido se tivesse sido criado para aplicativos da web.

Nós literalmente criamos novos atributos e elementos HTML e fazemos com que o Angular os compile em um aplicativo. ng-app é uma diretiva que simplesmente ativa o compilador. Outras diretivas incluem:

  • ng-click, que adiciona um manipulador de cliques,
  • ng-hide, que oculta condicionalmente um elemento, e
  • <form>, que adiciona comportamento adicional a um elemento de formulário HTML padrão.

O Angular vem com cerca de 100 diretivas integradas que permitem realizar as tarefas mais comuns. Também podemos escrever nossas próprias diretivas, e elas serão tratadas da mesma forma que as diretivas incorporadas.

Construímos um aplicativo Angular a partir de uma série de diretivas, conectadas com HTML.
