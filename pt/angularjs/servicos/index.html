Criando um serviço usando angular.factory #

Primeiro defina o serviço (neste caso ele usa o padrão de fábrica):

.factory('dataService', function() {
    var dataObject = {};
    var service = {
        // define the getter method
        get data() {
            return dataObject;
        },
        // define the setter method
        set data(value) {
            dataObject = value || {};
        }
    };
    // return the "service" object to expose the getter/setter
    return service;
})

Agora você pode usar o serviço para compartilhar dados entre controladores:

.controller('controllerOne', function(dataService) {
    // create a local reference to the dataService
    this.dataService = dataService;
    // create an object to store
    var someObject = {
        name: 'SomeObject',
        value: 1
    };
    // store the object
    this.dataService.data = someObject;
})

.controller('controllerTwo', function(dataService) {
    // create a local reference to the dataService
    this.dataService = dataService;
    // this will automatically update with any changes to the shared data object
    this.objectFromControllerOne = this.dataService.data;
})

Como criar um serviço #

angular.module("app")
  .service("counterService", function(){

    var service = {
        number: 0
    };
    
    return service;
   });

Como usar um serviço #

    angular.module("app")
        
        // Custom services are injected just like Angular's built-in services
        .controller("step1Controller", ['counterService', '$scope', function(counterService, $scope) {
            counterService.number++;
            // bind to object (by reference), not to value, for automatic sync
            $scope.counter = counterService;
        })

No modelo usando este controlador, você escreveria:

// editable
<input ng-model="counter.number" />

ou

// read-only
<span ng-bind="counter.number"></span>

É claro que, em código real, você interage com o serviço usando métodos no controlador, que por sua vez delegam ao serviço. O exemplo acima simplesmente incrementa o valor do contador cada vez que o controlador é usado em um modelo.

Os serviços em Angularjs são singletons:

Serviços são objetos singleton que são instanciados apenas uma vez por aplicativo (pelo $injector) e carregados lentamente (criados somente quando necessário).

Um singleton é uma classe que só permite que uma instância de si mesma seja criado - e dá acesso simples e fácil à referida instância. Como afirmado aqui

$sce - higieniza e renderiza conteúdo e recursos em templates #

$sce (“Strict Contextual Escaping”) é um serviço angular integrado que limpa automaticamente o conteúdo e as fontes internas nos modelos.

injetar fontes externas e HTML bruto no modelo requer encapsulamento manual de $sce.

Neste exemplo, criaremos um filtro simples de saneamento $sce :`.

Demonstração

.filter('sanitizer', ['$sce', [function($sce) {
     return function(content) {
          return $sce.trustAsResourceUrl(content);
      };
}]);

Uso no modelo

<div ng-repeat="item in items">
    
    // Sanitize external sources
    <ifrmae ng-src="{{item.youtube_url | sanitizer}}">
    
    // Sanitaize and render HTML 
    <div ng-bind-html="{{item.raw_html_content| sanitizer}}"></div>

</div>

Diferença entre Serviço e Fábrica #

1) Serviços

Um serviço é uma função construtor que é invocada uma vez em tempo de execução com new, assim como o que faríamos com javascript simples com apenas diferença que o AngularJs está chamando o novo nos bastidores.

Há uma regra geral a ser lembrada em caso de serviços

  1. Serviços são construtores que são chamados com new

Vamos ver um exemplo simples onde registraríamos um serviço que usa o serviço $http para buscar os detalhes do aluno e usá-lo no controlador

function StudentDetailsService($http) {
  this.getStudentDetails = function getStudentDetails() {
    return $http.get('/details');
  };
}

angular.module('myapp').service('StudentDetailsService', StudentDetailsService);

Acabamos de injetar este serviço no controlador

function StudentController(StudentDetailsService) {
  StudentDetailsService.getStudentDetails().then(function (response) {
      // handle response
    });
}
angular.module('app').controller('StudentController', StudentController);

Quando usar?

Use .service() sempre que quiser usar um construtor. Geralmente é usado para criar APIs públicas como getStudentDetails(). Mas se você não quer usar um construtor e deseja usar um padrão de API simples, então não há muita flexibilidade em .service().

2) Fábrica

Mesmo que possamos alcançar todas as coisas usando .factory() o que faríamos, usando .services(), isso não torna .factory() “igual a” .service(). É muito mais poderoso e flexível que .service()

Um .factory() é um padrão de design que é usado para retornar um valor.

Existem duas regras básicas a serem lembradas no caso de fábricas

  1. Valores de retorno das fábricas
  2. Fábricas (podem) criar objetos (Qualquer object)

Vamos ver alguns exemplos do que podemos fazer usando .factory()

Devolvendo Literais de Objetos

Vamos ver um exemplo onde a fábrica é usada para retornar um objeto usando um padrão básico do módulo Revelação

function StudentDetailsService($http) {
  function getStudentDetails() {
    return $http.get('/details');
  }
  return {
    getStudentDetails: getStudentDetails
  };
}

angular.module('myapp').factory('StudentDetailsService', StudentDetailsService);

Uso dentro de um controlador

function StudentController(StudentDetailsService) {
  StudentDetailsService.getStudentDetails().then(function (response) {
      // handle response
    });
}
angular.module('app').controller('StudentController', StudentController);

Fechamento de retorno

O que é um encerramento?

Closures são funções que se referem a variáveis ​​que são usadas localmente, MAS definidas em um escopo delimitador.

Segue um exemplo de fechamento

function closureFunction(name) {
  function innerClosureFunction(age) { // innerClosureFunction() is the inner function, a closure
    // Here you can manipulate 'age' AND 'name' variables both
  };
};

A parte “maravilhosa” é que ele pode acessar o name que está no escopo pai.

Vamos usar o exemplo de encerramento acima dentro de .factory()

function StudentDetailsService($http) {
  function closureFunction(name) {
  function innerClosureFunction(age) {
    // Here you can manipulate 'age' AND 'name' variables
    };
  };
};

angular.module('myapp').factory('StudentDetailsService', StudentDetailsService);

Uso dentro de um controlador

function StudentController(StudentDetailsService) {
  var myClosure = StudentDetailsService('Student Name'); // This now HAS the innerClosureFunction()
  var callMyClosure = myClosure(24); // This calls the innerClosureFunction()
};

angular.module('app').controller('StudentController', StudentController);

Criando construtores/instâncias

.service() cria construtores com uma chamada para new como visto acima. .factory() também pode criar construtores com uma chamada para new

Vamos ver um exemplo de como conseguir isso

function StudentDetailsService($http) {
  function Student() {
    this.age = function () {
        return 'This is my age';
    };
  }
  Student.prototype.address = function () {
        return 'This is my address';
  };
  return Student;
};

angular.module('myapp').factory('StudentDetailsService', StudentDetailsService);

Uso dentro de um controlador

function StudentController(StudentDetailsService) {
  var newStudent = new StudentDetailsService();
  
  //Now the instance has been created. Its properties can be accessed.

 newStudent.age();
 newStudent.address();
  
};

angular.module('app').controller('StudentController', StudentController);

Como criar um serviço com dependências usando ‘sintaxe de matriz’ #

angular.module("app")
  .service("counterService", ["fooService", "barService", function(anotherService, barService){

    var service = {
        number: 0,
        foo: function () {
            return fooService.bazMethod(); // Use of 'fooService'
        },
        bar: function () {
            return barService.bazMethod(); // Use of 'barService'
        }
    };
    
    return service;
   }]);

Registrando um serviço #

A maneira mais comum e flexível de criar um serviço usa a fábrica de API angular.module:

angular.module('myApp.services', []).factory('githubService', function() {       
   var serviceInstance = {};
   // Our first service
   return serviceInstance;
});

A função de fábrica de serviço pode ser uma função ou um array, assim como criamos controladores:

// Creating the factory through using the
// bracket notation
angular.module('myApp.services', [])
.factory('githubService', [function($http) {
}]);

Para expor um método em nosso serviço, podemos colocá-lo como um atributo no objeto de serviço.

angular.module('myApp.services', [])
    .factory('githubService', function($http) {
        var githubUrl = 'https://api.github.com';
        var runUserRequest = function(username, path) {
        // Return the promise from the $http service
        // that calls the Github API using JSONP
        return $http({
            method: 'JSONP',
            url: githubUrl + '/users/' +
            username + '/' +
            path + '?callback=JSON_CALLBACK'
        });
    }
// Return the service object with a single function
// events
return {
    events: function(username) {
    return runUserRequest(username, 'events');
    }
};
