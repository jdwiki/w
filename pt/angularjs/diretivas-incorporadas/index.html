Expressões angulares - Texto vs. Número #

Este exemplo demonstra como as expressões angulares são avaliadas ao usar type="text" e type="number" para o elemento de entrada. Considere o seguinte controlador e visualização:

Controlador

 var app = angular.module('app', []);

 app.controller('ctrl', function($scope) {
     $scope.textInput = {
         value: '5'
     };     
     $scope.numberInput = {
         value: 5
     };
 });

Visão

<div ng-app="app" ng-controller="ctrl">
    <input type="text" ng-model="textInput.value">
    {{ textInput.value + 5 }}
    <input type="number" ng-model="numberInput.value">
    {{ numberInput.value + 5 }}
</div>
  • Ao usar + em uma expressão vinculada à entrada text, o operador concatenará as strings (primeiro exemplo), exibindo 55 na tela*.
  • Ao usar + em uma expressão vinculada à entrada number, o operador retorna a soma dos números (segundo exemplo), exibindo 10 na tela*.

* - Isso é até que o usuário altere o valor no campo de entrada, depois o display mudará de acordo.

E se #

ng-if é uma diretiva similar a ng-show mas insere ou remove o elemento do DOM ao invés de simplesmente ocultá-lo. Angular 1.1.5 introduziu a diretiva ng-If. Você pode usar a diretiva ng-if acima das versões 1.1.5. Isso é útil porque o Angular não processará resumos de elementos dentro de um ng-if removido, reduzindo a carga de trabalho do Angular, especialmente para vinculações de dados complexas.

Ao contrário do ng-show, a diretiva ng-if cria um escopo filho que usa herança prototípica. Isso significa que definir um valor primitivo no escopo filho não se aplicará ao pai. Para definir uma primitiva no escopo pai, a propriedade $parent no escopo filho terá que ser usada.

#Javascript

angular.module('MyApp', []);

angular.module('MyApp').controller('myController', ['$scope', '$window', function myController($scope, $window) {
    $scope.currentUser= $window.localStorage.getItem('userName');
}]);

Visão #

<div ng-controller="myController">
    <div ng-if="currentUser">
        Hello, {{currentUser}}
    </div>
    <div ng-if="!currentUser">
        <a href="/login">Log In</a>
        <a href="/register">Register</a>
    </div>
</div>

DOM Se currentUser não for indefinido #

<div ng-controller="myController">
    <div ng-if="currentUser">
        Hello, {{currentUser}}
    </div>
    <!-- ng-if: !currentUser -->
</div>

DOM Se currentUser for indefinido #

<div ng-controller="myController">
    <!-- ng-if: currentUser -->
    <div ng-if="!currentUser">
        <a href="/login">Log In</a>
        <a href="/register">Register</a>
    </div>
</div>

Promessa de Função #

A diretiva ngIf também aceita funções, que logicamente requerem retornar true ou false.

<div ng-if="myFunction()">
    <span>Span text</span>
</div>

O texto span só aparecerá se a função retornar true.

$scope.myFunction = function() {
    var result = false;
    // Code to determine the boolean value of result 
    return result;
};

Como qualquer expressão Angular, a função aceita qualquer tipo de variável.

ngRepetir #

ng-repeat é uma diretiva embutida no Angular que permite iterar um array ou um objeto e lhe dá a habilidade de repetir um elemento uma vez para cada item na coleção.

repete a matriz

<ul>
    <li ng-repeat="item in itemCollection">
       {{item.Name}}
    </li>
</ul>

Onde:
item = item individual na coleção
itemCollection = O array que você está iterando


ng-repetir um objeto

<ul>
    <li ng-repeat="(key, value) in myObject">
       {{key}} : {{value}}
    </li>
</ul>

Onde:
chave = o nome da propriedade
valor = valor do imóvel
myObject = o objeto que você está iterando


filtre seu ng-repeat por entrada do usuário

<input type="text" ng-model="searchText">
<ul>
    <li ng-repeat="string in stringArray | filter:searchText">
       {{string}}
    </li>
</ul>

Onde:
searchText = o texto pelo qual o usuário deseja filtrar a lista
stringArray = uma matriz de strings, por exemplo ['string', 'array']

Você também pode exibir ou fazer referência aos itens filtrados em outro lugar, atribuindo à saída do filtro um alias com as aliasName, assim:

<input type="text" ng-model="searchText">
<ul>
    <li ng-repeat="string in stringArray | filter:searchText as filteredStrings">
       {{string}}
    </li>
</ul>
<p>There are {{filteredStrings.length}} matching results</p>

ng-repeat-start e ng-repeat-end

Para repetir vários elementos DOM definindo um ponto inicial e final, você pode usar as diretivas ng-repeat-start e ng-repeat-end.

<ul>
    <li ng-repeat-start="item in [{a: 1, b: 2}, {a: 3, b:4}]">
        {{item.a}}
    </li>
    <li ng-repeat-end>
        {{item.b}}
    </li>
</ul>

Resultado:

  • 1
  • 2
  • 3
  • 4

É importante sempre fechar ng-repeat-start com ng-repeat-end.

Variáveis

ng-repeat também expõe essas variáveis ​​dentro da expressão

Variável Tipo Detalhes
$index Número Igual ao índice da iteração atual ($index===0 será avaliado como true no primeiro elemento iterado; veja $first)
$primeiro Booleano Avalia como verdadeiro no primeiro elemento iterado
$último Booleano Avalia como verdadeiro no último elemento iterado
$meio Booleano Avalia como verdadeiro se o elemento estiver entre $first e $last
$par Booleano Avalia como verdadeiro em uma iteração de número par (equivalente a $index%2===0)
$ímpar Booleano Avalia como verdadeiro em uma iteração de número ímpar (equivalente a $index%2===1)

Considerações de desempenho

A renderização de ngRepeat pode se tornar lenta, especialmente ao usar grandes coleções.

Se os objetos da coleção tiverem uma propriedade identificadora, você deve sempre rastrear por o identificador em vez do objeto inteiro, que é a funcionalidade padrão. Se nenhum identificador estiver presente, você sempre poderá usar o $index embutido.

<div ng-repeat="item in itemCollection track by item.id">
<div ng-repeat="item in itemCollection track by $index">

Escopo de ngRepeat

ngRepeat sempre criará um escopo filho isolado, portanto, deve-se tomar cuidado se o escopo pai precisar ser acessado dentro da repetição.

Aqui está um exemplo simples mostrando como você pode definir um valor em seu escopo pai a partir de um evento de clique dentro de ngRepeat.

scope val:  {{val}}<br/>
ctrlAs val: {{ctrl.val}}
<ul>
    <li ng-repeat="item in itemCollection">
        <a href="#" ng-click="$parent.val=item.value; ctrl.val=item.value;">
            {{item.label}} {{item.value}}
        </a>
    </li>
</ul>

$scope.val = 0;
this.val = 0;

$scope.itemCollection = [{
    id: 0,
    value: 4.99,
    label: 'Football'
},
{
    id: 1,
    value: 6.99,
    label: 'Baseball'
},
{
    id: 2,
    value: 9.99,
    label: 'Basketball'
}];

Se houver apenas val = item.value em ng-click ele não atualizará o val no escopo pai por causa do escopo isolado. É por isso que o escopo pai é acessado com a referência $parent ou com a sintaxe controllerAs (por exemplo, ng-controller="mainController as ctrl").

Aninhado por repetição

Você também pode usar ng-repeat aninhado.

<div ng-repeat="values in test">
    <div ng-repeat="i in values">
      [{{$parent.$index}},{{$index}}] {{i}}
    </div>
</div>

var app = angular.module("myApp", []);
app.controller("ctrl", function($scope) {
  $scope.test = [
    ['a', 'b', 'c'],
    ['d', 'e', 'f']
  ];
});

Aqui para acessar o índice do pai ng-repeat dentro do filho ng-repeat, você pode usar $parent.$index.

ngCloak #

A diretiva ngCloak é usada para evitar que o template html Angular de ser exibido brevemente pelo navegador em seu estado bruto (descompilado) enquanto seu aplicativo está carregando. - Ver fonte

HTML

<div ng-cloak>
  <h1>Hello {{ name }}</h1>
</div>

ngCloak pode ser aplicado ao elemento body, mas o uso é aplicar várias diretivas ngCloak a pequenas porções do página para permitir a renderização progressiva da visualização do navegador.

A diretiva ngCloak não tem parâmetros.

opções de modelo #

ng-model-options permite alterar o comportamento padrão do ng-model, esta diretiva permite registrar eventos que serão acionados quando o ng-model for atualizado e anexar um efeito debounce.

Esta diretiva aceita uma expressão que será avaliada como um objeto de definição ou uma referência a um valor de escopo.

Exemplo:

<input type="text" ng-model="myValue" ng-model-options="{'debounce': 500}">

O exemplo acima anexará um efeito de debounce de 500 milissegundos em myValue, que fará com que o modelo atualize 500 ms após o usuário terminar de digitar sobre o input (ou seja, quando o myValue terminar de atualizar).

Propriedades de objetos disponíveis

  1. updateOn: especifica qual evento deve ser vinculado à entrada

    ng-model-options="{ updateOn: 'blur'}" // will update on blur

  2. debounce: especifica um atraso de alguns milissegundos para a atualização do modelo

    ng-model-options="{'debounce': 500}" // will update the model after 1/2 second

  3. allowInvalid: um sinalizador booleano que permite um valor inválido para o modelo, contornando a validação padrão do formulário, por padrão esses valores seriam tratados como undefined.

  4. getterSetter: um sinalizador booleano indicando se o ng-model deve ser tratado como uma função getter/setter em vez de um valor de modelo simples. A função será executada e retornará o valor do modelo.

Exemplo:

   <input type="text" ng-model="myFunc" ng-model-options="{'getterSetter': true}">

   $scope.myFunc = function() {return "value";}
  1. timezone: define o fuso horário para o modelo se a entrada for de date ou time. tipos

ngIncluir #

ng-include permite delegar o controle de uma parte da página a um controlador específico. Você pode querer fazer isso porque a complexidade desse componente está se tornando tal que você deseja encapsular toda a lógica em um controlador dedicado.

Um exemplo é:

  <div ng-include
       src="'/gridview'"
       ng-controller='gridController as gc'>
  </div>

Observe que o /gridview precisará ser servido pelo servidor web como um URL distinto e legítimo.

Além disso, observe que o atributo src aceita uma expressão Angular. Isso pode ser uma variável ou uma chamada de função, por exemplo, ou, como neste exemplo, uma constante de string. Nesse caso, você precisa colocar o URL de origem entre aspas simples, para que ele seja avaliado como uma constante de string. Esta é uma fonte comum de confusão.

Dentro do html /gridview, você pode se referir ao gridController como se estivesse enrolado na página, por exemplo:

<div class="row">
  <button type="button" class="btn btn-default" ng-click="gc.doSomething()"></button>
</div>

ngCopiar #

A diretiva ngCopy especifica o comportamento a ser executado em um evento de cópia.

Impedir que um usuário copie dados #

<p ng-copy="blockCopy($event)">This paragraph cannot be copied</p>

No controlador

$scope.blockCopy = function(event) {
    event.preventDefault();
    console.log("Copying won't work");
}

ngClique #

A diretiva ng-click anexa um evento de clique a um elemento DOM.

A diretiva ng-click permite especificar o comportamento personalizado quando um elemento do DOM é clicado.

É útil quando você deseja anexar eventos de clique em botões e manipulá-los em seu controlador.

Esta diretiva aceita uma expressão com o objeto de eventos disponível como $event

HTML

<input ng-click="onClick($event)">Click me</input>

Controlador

.controller("ctrl", function($scope) {   
    $scope.onClick = function(evt) {
        console.debug("Hello click event: %o ",evt);
    } 
})

HTML

<button ng-click="count = count + 1" ng-init="count=0">
  Increment
</button>
<span>
  count: {{count}}
</span>

HTML

<button ng-click="count()" ng-init="count=0">
  Increment
</button>
<span>
  count: {{count}}
</span>

Controlador

...

$scope.count = function(){
    $scope.count = $scope.count + 1;
}
...

Quando o botão é clicado, uma invocação da função onClick imprimirá “Hello click event” seguido pelo objeto de evento.

ngColar #

A diretiva ngPaste especifica o comportamento personalizado a ser executado quando um usuário cola conteúdo

<input ng-paste="paste=true" ng-init="paste=false" placeholder='paste here'>
pasted: {{paste}}

ngOptions #

ngOptions é uma diretiva que simplifica a criação de uma caixa suspensa html para a seleção de um item de um array que será armazenado em um modelo. O atributo ngOptions é usado para gerar dinamicamente uma lista de elementos <option> para o elemento <select> usando o array ou objeto obtido pela avaliação da expressão de compreensão ngOptions.

Com ng-options a marcação pode ser reduzida a apenas uma tag de seleção e a diretiva criará a mesma seleção:

<select ng-model="selectedFruitNgOptions" 
        ng-options="curFruit as curFruit.label for curFruit in fruit">
</select>

Existe outra maneira de criar opções select usando ng-repeat, mas não é recomendado usar ng-repeat pois é usado principalmente para propósitos gerais como o forEach apenas para fazer loop. Considerando que ng-options é especificamente para criar opções de tag select.

O exemplo acima usando ng-repeat seria

<select ng-model="selectedFruit">
    <option ng-repeat="curFruit in fruit" value="{{curFruit}}">
        {{curFruit.label}}
    </option>
</select>

EXEMPLO COMPLETO

Vamos ver o exemplo acima em detalhes também com algumas variações.

Modelo de dados para o exemplo:

$scope.fruit = [
    { label: "Apples", value: 4, id: 2 },
    { label: "Oranges", value: 2, id: 1 },
    { label: "Limes", value: 4, id: 4 },
    { label: "Lemons", value: 5, id: 3 }
];

<!-- label for value in array -->
<select ng-options="f.label for f in fruit" ng-model="selectedFruit"></select>

Etiqueta de opção gerada na seleção:

 <option value="{ label: "Apples", value: 4, id: 2 }"> Apples </option>

Efeitos:

f.label será o rótulo da <opção> e o valor conterá o objeto inteiro.

EXEMPLO COMPLETO

<!-- select as label for value in array -->
<select ng-options="f.value as f.label for f in fruit" ng-model="selectedFruit"></select>

Etiqueta de opção gerada na seleção:

 <option value="4"> Apples </option>

Efeitos:

f.value (4) será o valor neste caso enquanto o rótulo ainda for o mesmo.

EXEMPLO COMPLETO

<!-- label group by group for value in array -->
<select ng-options="f.label group by f.value for f in fruit" ng-model="selectedFruit"></select>

Etiqueta de opção gerada na seleção:

<option value="{ label: "Apples", value: 4, id: 2 }"> Apples </option>

Efeitos:

As opções serão agrupadas com base nesse valor. Opções com o mesmo “valor” cairão em uma categoria

EXEMPLO COMPLETO

<!-- label disable when disable for value in array -->
<select ng-options="f.label disable when f.value == 4 for f in fruit" ng-model="selectedFruit"></select>

Etiqueta de opção gerada na seleção:

<option disabled="" value="{ label: "Apples", value: 4, id: 2 }"> Apples </option>

Efeitos:

“Maçãs” e “Limes” serão desabilitados (não é possível selecionar) por causa da condição desabilitar quando f.value==4. Todas as opções com value=4 devem ser desabilitadas

EXEMPLO COMPLETO

<!-- label group by group for value in array track by trackexpr -->
<select ng-options="f.value as f.label group by f.value for f in fruit track by f.id" ng-model="selectedFruit"></select>

Etiqueta de opção gerada na seleção:

<option value="4"> Apples </option>

Efeitos:

Não há alteração visual ao usar trackBy, mas o Angular detectará alterações pelo id em vez de por referência, o que é sempre uma solução melhor.

EXEMPLO COMPLETO

<!-- label for value in array | orderBy:orderexpr track by trackexpr -->
<select ng-options="f.label for f in fruit | orderBy:'id' track by f.id" ng-model="selectedFruit"></select>

Etiqueta de opção gerada na seleção:

<option disabled="" value="{ label: "Apples", value: 4, id: 2 }"> Apples </option>

Efeitos:

orderBy é um filtro padrão AngularJS que organiza as opções em ordem crescente (por padrão) para que “Laranjas” apareçam em primeiro lugar, pois seu id = 1.

EXEMPLO COMPLETO

Todos os <select> com ng-options devem ter ng-model anexado.

ngModel #

Com ng-model você pode vincular uma variável a qualquer tipo de campo de entrada. Você pode exibir a variável usando chaves duplas, por exemplo {{myAge}}.

<input type="text" ng-model="myName">
<p>{{myName}}</p>

Conforme você digita no campo de entrada ou o altera de qualquer forma, você verá o valor no parágrafo atualizado instantaneamente.

A variável ng-model, neste caso, estará disponível em seu controller como $scope.myName. Se você estiver usando a sintaxe controllerAs:

<div ng-controller="myCtrl as mc">
    <input type="text" ng-model="mc.myName">
    <p>{{mc.myName}}</p>
</div>

Você precisará fazer referência ao escopo do controlador pré-pendente do alias do controlador definido no atributo ng-controller para a variável ng-model. Desta forma você não precisará injetar $scope em seu controlador para referenciar sua variável ng-model, a variável estará disponível como this.myName dentro da função do seu controlador.

ngClass #

Vamos supor que você precise mostrar o status de um usuário e tenha várias classes CSS possíveis que podem ser usadas. Angular torna muito fácil escolher a partir de uma lista de várias classes possíveis que permitem especificar uma lista de objetos que inclui condicionais. Angular é capaz de usar a classe correta com base na veracidade das condicionais.

Seu objeto deve conter pares de chave/valor. A chave é um nome de classe que será aplicado quando o valor (condicional) for avaliado como verdadeiro.

<style>
    .active { background-color: green; color: white; }
    .inactive { background-color: gray; color: white; }
    .adminUser { font-weight: bold; color: yellow; }
    .regularUser { color: white; }
</style>

<span ng-class="{ 
    active: user.active, 
    inactive: !user.active, 
    adminUser: user.level === 1, 
    regularUser: user.level === 2 
}">John Smith</span>

Angular irá verificar o objeto $scope.user para ver o status active e o número do level. Dependendo dos valores dessas variáveis, o Angular aplicará o estilo de correspondência ao <span>.

ngSrc #

Usar marcação Angular como {{hash}} em um atributo src não funciona direito. O navegador irá buscar da URL com o texto literal {{hash}} até que o Angular substitua a expressão dentro de {{hash}}. A diretiva ng-src substitui o atributo src original para o elemento da tag de imagem e resolve o problema

<div ng-init="pic = 'pic_angular.jpg'">
    <h1>Angular</h1>
    <img ng-src="{{pic}}">
</div>

ngLista #

A diretiva ng-list é usada para converter uma string delimitada de uma entrada de texto para um array de strings ou vice-versa.

A diretiva ng-list usa um delimitador padrão de ", " (espaço de vírgula).

Você pode definir o delimitador manualmente atribuindo ng-list um delimitador como este ng-list="; ".

Nesse caso, o delimitador é definido como um ponto e vírgula seguido por um espaço.

Por padrão, ng-list tem um atributo ng-trim que é definido como true. ng-trim quando falso, respeitará o espaço em branco no seu delimitador. Por padrão, ng-list não leva em consideração o espaço em branco, a menos que você defina ng-trim="false".

Exemplo:

angular.module('test', [])
  .controller('ngListExample', ['$scope', function($scope) {
    $scope.list = ['angular', 'is', 'cool!'];
}]);

Um delimitador de cliente é definido como ; . E o modelo da caixa de entrada é definido para o array que foi criado no escopo.

  <body ng-app="test" ng-controller="ngListExample">
    <input ng-model="list" ng-list="; " ng-trim="false">
  </body>

A caixa de entrada será exibida com o conteúdo: angular; é; legal!

ngShow e ngHide #

A diretiva ng-show mostra ou oculta o elemento HTML com base se a expressão passada para ele é verdadeira ou falsa. Se o valor da expressão for falso, ele será ocultado. Se for verdade, então ele vai mostrar.

A diretiva ng-hide é semelhante. No entanto, se o valor for falso, ele mostrará o elemento HTML. Quando a expressão for verdadeira, ela a esconderá.

Exemplo JSBin de trabalho

Controlador:

var app = angular.module('app', []);
  
angular.module('app')
  .controller('ExampleController', ExampleController);
 
function ExampleController() {
  
  var vm = this;
  
  //Binding the username to HTML element
  vm.username = '';
  
  //A taken username
  vm.taken_username = 'StackOverflow';
  
}

Visão

<section ng-controller="ExampleController as main">
    
    <p>Enter Password</p>
    <input ng-model="main.username" type="text">
    
    <hr>
    
    <!-- Will always show as long as StackOverflow is not typed in -->
    <!-- The expression is always true when it is not StackOverflow -->
    <div style="color:green;" ng-show="main.username != main.taken_username">
      Your username is free to use!
    </div>
    
    <!-- Will only show when StackOverflow is typed in -->
    <!-- The expression value becomes falsy -->
    <div style="color:red;" ng-hide="main.username != main.taken_username">
      Your username is taken!
    </div>
    
    <p>Enter 'StackOverflow' in username field to show ngHide directive.</p>
    
 </section>

ngDblclick #

A diretiva ng-dblclick é útil quando você deseja vincular um evento de clique duplo em seus elementos DOM.

Esta diretiva aceita uma expressão

HTML

<input type="number" ng-model="num = num + 1" ng-init="num=0">

<button ng-dblclick="num++">Double click me</button>

No exemplo acima, o valor mantido na entrada será incrementado quando o botão for clicado duas vezes.

ngPadrão #

A diretiva ng-pattern aceita uma expressão que avalia um padrão de expressão regular e usa esse padrão para validar uma entrada textual.

Exemplo:

Digamos que queremos que um elemento <input> se torne válido quando seu valor (ng-model) for um endereço IP válido.

Modelo:

<input type="text" ng-model="ipAddr" ng-pattern="ipRegex" name="ip" required>

Controlador:

$scope.ipRegex = /\b(?:(?:25[0-5]|2[0-4][0-9]|[01]?[0-9][0-9]?)\.){3}(?:25[0-5]|2[0-4][0-9]|[01]?[0-9][0-9]?)\b/;

ngRef #

ngHref é usado em vez do atributo href, se tivermos expressões angulares dentro do valor href. A diretiva ngHref substitui o atributo href original de uma tag html usando o atributo href, como tag , tag etc.

A diretiva ngHref garante que o link não seja quebrado mesmo se o usuário clicar no link antes que o AngularJS tenha avaliado o código.

Exemplo 1

<div ng-init="linkValue = 'http://stackoverflow.com'">
    <p>Go to <a ng-href="{{linkValue}}">{{linkValue}}</a>!</p>
</div>

Exemplo 2 Este exemplo obtém dinamicamente o valor href da caixa de entrada e o carrega como valor href.

<input ng-model="value" />
<a id="link" ng-href="{{value}}">link</a>

ngMouseenter e ngMouseleave #

As diretivas ng-mouseenter e ng-mouseleave são úteis para executar eventos e aplicar estilo CSS quando você passa o mouse sobre ou fora de seus elementos DOM.

A diretiva ng-mouseenter executa uma expressão em um evento de entrada do mouse (quando o usuário insere o ponteiro do mouse sobre o elemento DOM em que esta diretiva reside)

HTML

<div ng-mouseenter="applyStyle = true" ng-class="{'active': applyStyle}">

No exemplo acima, quando o usuário aponta o mouse sobre o div, applyStyle se transforma em true, que por sua vez aplica a classe CSS .active na ng-class.

A diretiva ng-mouseleave executa uma expressão em um evento de saída do mouse (quando o usuário afasta o cursor do mouse do elemento DOM em que esta diretiva reside)

HTML

<div ng-mouseenter="applyStyle = true" ng-mouseleaver="applyStyle = false" ng-class="{'active': applyStyle}">

Reutilizando o primeiro exemplo, agora quando o usuário tira o ponteiro do mouse da div, a classe .active é removida.

ngDesativado #

Esta diretiva é útil para limitar eventos de entrada com base em certas condições existentes.

A diretiva ng-disabled aceita uma expressão que deve ser avaliada como um valor verdadeiro ou falso.

ng-disabled é usado para aplicar condicionalmente o atributo disabled em um elemento input.

HTML

<input type="text" ng-model="vm.name">

<button ng-disabled="vm.name.length===0" ng-click="vm.submitMe">Submit</button>

vm.name.length===0 é avaliado como verdadeiro se o comprimento do input for 0, que por sua vez desabilita o botão, não permitindo que o usuário dispare o evento click de ng-click

ngRequerido #

O ng-required adiciona ou remove o atributo de validação required em um elemento, que por sua vez habilita e desabilita a chave de validação require para a input.

É usado para definir opcionalmente se um elemento input precisa ter um valor não vazio. A diretiva é útil ao projetar validação em formulários HTML complexos.

HTML

<input type="checkbox" ng-model="someBooleanValue">
<input type="text" ng-model="username" ng-required="someBooleanValue">
