angular.igual #

A função angular.equals compara e determina se 2 objetos ou valores são iguais, angular.equals executa uma comparação profunda e retorna verdadeiro se e somente se pelo menos 1 das seguintes condições for atendida.

angular.equals(valor1, valor2)

  1. Se os objetos ou valores passarem na comparação ===
  2. Se ambos os objetos ou valores forem do mesmo tipo e todas as suas propriedades também forem iguais usando angular.equals
  3. Ambos os valores são iguais a NaN
  4. Ambos os valores representam o resultado da mesma expressão regular.

Essa função é útil quando você precisa comparar objetos ou matrizes por seus valores ou resultados em vez de apenas referências.

Exemplos

angular.equals(1, 1) // true
angular.equals(1, 2) // false
angular.equals({}, {}) // true, note that {}==={} is false
angular.equals({a: 1}, {a: 1}) // true
angular.equals({a: 1}, {a: 2}) // false
angular.equals(NaN, NaN) // true

angular.toJson #

A função angular.toJson pegará um objeto e o serializará em uma string formatada em JSON.

Ao contrário da função nativa JSON.stringify, esta função irá remover todas as propriedades que começam com $$ (como angular geralmente prefixa as propriedades internas com $$)

angular.toJson(object)

Como os dados precisam ser serializados antes de passar por uma rede, essa função é útil para transformar qualquer dado que você deseja transmitir em JSON.

Esta função também é útil para depuração, pois funciona de forma semelhante a um método .toString.

Exemplos:

angular.toJson({name: "barf", occupation: "mog", $$somebizzareproperty: 42})
// "{"name":"barf","occupation":"mog"}"
angular.toJson(42)
// "42"
angular.toJson([1, "2", 3, "4"])
// "[1,"2",3,"4"]"
var fn = function(value) {return value}
angular.toJson(fn)
// undefined, functions have no representation in JSON

angular.copiar #

A função angular.copy pega um objeto, array ou valor e cria uma cópia profunda dele.

angular.copy()

Exemplo:

Objetos:

let obj = {name: "vespa", occupation: "princess"};
let cpy = angular.copy(obj);
cpy.name = "yogurt"
// obj = {name: "vespa", occupation: "princess"}
// cpy = {name: "yogurt", occupation: "princess"}

Matrizes:

var w = [a, [b, [c, [d]]]];
var q = angular.copy(w);
// q = [a, [b, [c, [d]]]]

No exemplo acima, angular.equals(w, q) será avaliado como verdadeiro porque .equals testa a igualdade por valor. entretanto w === q será avaliado como false porque a comparação estrita entre objetos e arrays é feita por referência.

angular.isString #

A função angular.isString retorna true se o objeto ou valor dado a ela for do tipo string

angular.isString(valor1)

Exemplos

angular.isString("hello") // true
angular.isString([1, 2]) // false
angular.isString(42) // false

Isso é o equivalente a realizar

typeof someValue === "string"

angular.isArray #

A função angular.isArray retorna true se e somente se o objeto ou valor passado para ela for do tipo Array.

angular .isArray(valor);

Exemplos

angular.isArray([]) // true
angular.isArray([2, 3]) // true
angular.isArray({}) // false
angular.isArray(17) // false

É o equivalente de

Array.isArray(someValue)

angular.merge #

A função angular.merge pega todas as propriedades enumeráveis ​​do objeto de origem para estender profundamente o objeto de destino.

A função retorna uma referência ao objeto de destino agora estendido

angular.merge(destino, origem)

Exemplos

angular.merge({}, {}) // {} 
angular.merge({name: "king roland"}, {password: "12345"})
// {name: "king roland", password: "12345"}
angular.merge({a: 1}, [4, 5, 6]) // {0: 4, 1: 5, 2: 6, a: 1}
angular.merge({a: 1}, {b: {c: {d: 2}}}) // {"a":1,"b":{"c":{"d":2}}}

angular.isDefined e angular.isUndefined #

A função angular.isDefined testa um valor se estiver definido

angular.isDefined(someValue)

Isso é o equivalente a realizar

value !== undefined; // will evaluate to true is value is defined

Exemplos

angular.isDefined(42) // true
angular.isDefined([1, 2]) // true
angular.isDefined(undefined) // false
angular.isDefined(null) // true

A função angular.isUndefined testa se um valor é indefinido (é efetivamente o oposto de angular.isDefined)

angular.isUndefined(someValue)

Isso é o equivalente a realizar

value === undefined; // will evaluate to true is value is undefined

Ou apenas

!angular.isDefined(value)

Exemplos

angular.isUndefined(42) // false
angular.isUndefined(undefined) // true

angular.isDate #

A função angular.isDate retorna true se e somente se o objeto passado para ela for do tipo Date.

angular.isData(valor)

Exemplos

angular.isDate("lone star") // false
angular.isDate(new Date()) // true

angular.isFunction #

A função angular.isFunction determina e retorna true se e somente se o valor passado for uma referência a uma função.

A função retorna uma referência ao objeto de destino agora estendido

angular.isFunção(fn)

Exemplos

var onClick = function(e) {return e};
angular.isFunction(onClick); // true

var someArray = ["pizza", "the", "hut"];
angular.isFunction(someArray ); // false

angular.noop #

O angular.noop ​​é uma função que não realiza nenhuma operação, você passa angular.noop ​​quando precisar fornecer um argumento de função que não fará nada.

angular.noop()

Um uso comum para angular.noop ​​pode ser fornecer um retorno de chamada vazio para uma função que, de outra forma, lançará um erro quando algo diferente de uma função for passado para ela.

Exemplo:

$scope.onSomeChange = function(model, callback) {
    updateTheModel(model);
    if (angular.isFunction(callback)) {
        callback();
    } else {
        throw new Error("error: callback is not a function!");
    }
};

$scope.onSomeChange(42, function() {console.log("hello callback")});
// will update the model and print 'hello callback'
$scope.onSomeChange(42, angular.noop);
// will update the model

Exemplos adicionais:

angular.noop() // undefined
angular.isFunction(angular.noop) // true

angular.isElement #

O angular.isElement retorna true se o argumento passado para ele for um DOM Element ou um jQuery wrap Element.

angular.isElement(elem)

Esta função é útil para verificar se um argumento passado é um elemento antes de ser processado como tal.

Exemplos:

angular.isElement(document.querySelector("body"))
// true
angular.isElement(document.querySelector("#some_id"))
// false if "some_id" is not using as an id inside the selected DOM
angular.isElement("<div></div>")
// false

angular.isNumber #

A função angular.isNumber retorna true se e somente se o objeto ou valor passado para ela for do tipo Number, isso inclui +Infinity, -Infinity e NaN

angular.isNumber(valor)

Esta função não causará uma coerção de tipo, como

"23" == 23 // true

Exemplos

angular.isNumber("23") // false
angular.isNumber(23) // true
angular.isNumber(NaN) // true
angular.isNumber(Infinity) // true

Esta função não causará uma coerção de tipo, como

"23" == 23 // true

angular.fromJson #

A função angular.fromJson desserializará uma string JSON válida e retornará um Object ou um Array.

angular.fromJson(string|object)

Observe que esta função não está limitada apenas a strings, ela produzirá uma representação de qualquer objeto passado para ela.

Exemplos:

angular.fromJson("{\"yogurt\": \"strawberries\"}")
// Object {yogurt: "strawberries"}
angular.fromJson('{jam: "raspberries"}')
// will throw an exception as the string is not a valid JSON
angular.fromJson(this)
// Window {external: Object, chrome: Object, _gaq: Y, angular: Object, ng339: 3…}
angular.fromJson([1, 2])
// [1, 2]
typeof angular.fromJson(new Date())
// "object"

angular.isObject #

O angular.isObject retornará true se e somente se o argumento passado para ele for um objeto, esta função também retornará true para um Array e retornará false para null mesmo que typeof null seja object .

angular.isObject(valor)

Esta função é útil para verificação de tipo quando você precisa de um objeto definido para processar.

Exemplos:

angular.isObject({name: "skroob", job: "president"})
// true
angular.isObject(null)
// false
angular.isObject([null])
// true
angular.isObject(new Date())
// true
angular.isObject(undefined)
// false

##identidade angular A função angular.identity retorna o primeiro argumento passado para ela.

angular.identity(argumento)

Esta função é útil para programação funcional, você pode fornecer esta função como padrão caso uma função esperada não tenha sido passada.

Exemplos:

angular.identity(42) // 42

var mutate = function(fn, num) {
    return angular.isFunction(fn) ? fn(num) : angular.identity(num)
}

mutate(function(value) {return value-7}, 42) // 35
mutate(null, 42) // 42
mutate("mount. rushmore", 42) // 42

angular.forEach #

O angular.forEach aceita um objeto e uma função iteradora. Em seguida, ele executa a função iteradora sobre cada propriedade/valor enumerável do objeto. Esta função também funciona em arrays.

Como a versão JS de Array.prototype.forEach A função não itera sobre propriedades herdadas (propriedades de protótipo), entretanto a função não tentará processar um valor null ou undefined e apenas o retornará.

angular.forEach(object, function(value, key) { // function});

Exemplos:

angular.forEach({"a": 12, "b": 34}, (value, key) => console.log("key: " + key + ", value: " + value))
// key: a, value: 12
// key: b, value: 34
angular.forEach([2, 4, 6, 8, 10], (value, key) => console.log(key))
// will print the array indices: 1, 2, 3, 4, 5
angular.forEach([2, 4, 6, 8, 10], (value, key) => console.log(value))
// will print the array values: 2, 4, 6, 7, 10
angular.forEach(undefined, (value, key) => console.log("key: " + key + ", value: " + value))
// undefined
