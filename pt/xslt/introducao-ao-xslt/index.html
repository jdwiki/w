Instalação ou Configuração #

XSLT é uma linguagem de programação de propósito especial; é amplamente utilizado para transformar documentos XML em um formato XML diferente, em HTML ou em formatos baseados em texto.

Existem duas versões principais do XSLT em uso: XSLT 1.0 e XSLT 2.0. O XSLT 1.0 é mais amplamente implementado, mas tem muitas restrições e limitações em comparação com o XSLT 2.0; você precisará decidir qual versão usar. Se um processador XSLT 2.0 estiver disponível para o ambiente escolhido, essa é quase sempre a melhor escolha.

(O XSLT 1.0 foi lançado em novembro de 1999, e muitas implementações apareceram em um ou dois anos, tanto de fornecedores tradicionais como Microsoft, IBM, Sun e Oracle, quanto de entusiastas individuais trabalhando em seu tempo livre. O XSLT 2.0 foi lançado em janeiro 2007, e muitos dos processadores 1.0 originais nunca foram atualizados, porque seus desenvolvedores perderam o interesse. O processador XSLT 2.0 mais usado é o Saxon, mas não tem campo próprio: outros produtos independentes incluem RaptorXML ), XmlPrime (da CBCL) e Exselt, e processadores 2.0 também estão disponíveis incorporados no WebLogic da IBM, no servidor de banco de dados XML da MarkLogic e no Acelerador XML da Intel.)

A especificação do XSLT 3.0 está tecnicamente completa (recomendação proposta em abril de 2017), mas é melhor considerada como de ponta por enquanto: considere-a apenas se você tiver uma necessidade urgente de seus novos recursos, como streaming, pacotes, suporte a JSON , ou tente/pegue. Existem três implementações conhecidas: Saxon, Exselt e RaptorXML.

Para começar com o XSLT, você tem várias opções:

  • Use uma ferramenta XSLT online. Existem vários disponíveis (procure por “ferramenta XSLT online”), um IDE online muito confiável é http://xsltransform.net/. Esta é uma boa maneira de ter uma ideia inicial da linguagem, mas você não vai querer continuar assim quando estiver fazendo um desenvolvimento real.

  • Use o mecanismo XSLT embutido em cada navegador. Assim como as ferramentas online, elas têm a vantagem de que você não precisa instalar nada; mas os navegadores suportam apenas XSLT 1.0, eles suportam apenas conversão XML para HTML e têm um suporte de depuração muito fraco. Um desenvolvimento muito recente - mas ainda experimental - é o Saxon-JS, que permite a execução do XSLT 2.0 (e partes do XSLT 3.0) no navegador.

  • Instale um processador XSLT (como Saxon ou xsltproc). A maioria desses produtos pode ser chamada usando a linha de comando do sistema operacional ou usando uma API para linguagens de programação específicas, como Java, C, C# ou Python.

  • Instale um ambiente de desenvolvimento XML como Altova XML Spy, oXygen da SyncroSoft ou Stylus Studio. Embora essa seja uma opção mais cara, ela oferece suporte ao desenvolvimento e capacidade de depuração muito mais ricos.

Qualquer que seja a opção escolhida, você deve primeiro decidir se deseja usar XSLT 1.0 ou XSLT 2.0.

Exemplo simples de XSLT #

Aqui está um exemplo simples que usa XSLT para converter dados em um arquivo XML em uma tabela em um arquivo HTML. Você pode usá-lo para experimentar transformações XSLT simples.

Pré-requisito: Instale um Java Runtime Environment e inclua o local do JRE em sua variável PATH. (No Windows, a maioria dos instaladores adicionará Java ao seu caminho para você.) Se isso funcionar, você poderá abrir uma janela de linha de comando e executar o comando java -version e obter uma impressão de informações sobre seu JRE.

  1. Baixe o processador Saxon-HE XSLT para Java aqui: saxon.sourceforge.net e descompacte-o em qualquer lugar do seu computador.
  2. Em um editor de texto, crie um arquivo chamado pets.xml com o seguinte código:
  <pets>
  <petType name="Dogs">
    <pet id="123" name="Sparky" vaccineStatus="vaccinated" healthStatus="healthy"/>
    <pet id="234" name="Sadie" vaccineStatus="unvaccinated" healthStatus="sick"/>
    <pet id="345" name="Herman" vaccineStatus="unvaccinated" healthStatus="unknown"/>
  </petType>
  <petType name="Cats">
    <pet id="456" name="Cleo" vaccineStatus="vaccinated" healthStatus="healthy"/>
    <pet id="567" name="Janet" vaccineStatus="unvaccinated" healthStatus="healthy"/>
    <pet id="678" name="Furry" vaccineStatus="vaccinated" healthStatus="sick"/>
  </petType>
</pets>
  1. In a text editor, create a file named petTransform.xsl with the following code:
<xsl:stylesheet xmlns:xsl="http://www.w3.org/1999/XSL/Transform"
  version="2.0">

<!-- handle the root XML element -->
<xsl:template match="/">
<html><head>
  <title>Pets that are available for adoption</title>
</head>
<body>
  <xsl:apply-templates/>
</body>
</html>
</xsl:template>

<xsl:template match="pets">
<xsl:apply-templates/>
</xsl:template>

<xsl:template match="petType">
<h2><xsl:value-of select="@name"/></h2>
<table id="{@name}">
   <tr>
     <th colname="id">ID</th>
     <th colname="name">Name</th>
     <th colname="vaccinated">Vaccine status</th>
     <th colname="health">Health status</th>
   </tr>
   <tbody>
     <!-- add a row for each pet in this category -->
     <xsl:for-each select="pet">
       <tr>
         <td colname="id"><xsl:value-of select="@id"/></td>
         <td colname="name"><xsl:value-of select="@name"/></td>
         <td colname="vaccinated"><xsl:value-of select="@vaccineStatus"/></td>
         <td colname="health"><xsl:value-of select="@healthStatus"/></td>
       </tr>
     </xsl:for-each>
   </tbody>
 </table>
</xsl:template>

<!-- ignore the content of other tags because we processed them elsewhere -->
<xsl:template match="*">
<!-- do nothing -->
</xsl:template>

</xsl:stylesheet>
  1. Abra uma janela de linha de comando e vá para a pasta com os arquivos XML e XSLT.
  2. Execute o seguinte comando, onde path_to_saxon.jar é o caminho completo para o arquivo saxon9he.jar:
java -jar "path_to_saxon.jar" -o
petOutput.html -s:pets.xml -xsl:pettransform.xslt

Por exemplo:

java -jar "C:\Program Files\SaxonHE9-7-0-7J\saxon9he.jar" -o
petOutput.html -s:pets.xml -xsl:pettransform.xslt

Certifique-se de executar este comando em uma única linha. 6. Abra o arquivo de saída petOutput.html em um editor de texto. Deve ficar assim:
