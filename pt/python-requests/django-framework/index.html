Instalação e configuração #

Django é um framework full stack para desenvolvimento web. Ele alimenta alguns dos sites mais populares da Internet.

Para instalar a estrutura; use a ferramenta pip:

pip install django

Se você estiver instalando no OSX ou Linux, o comando acima pode resultar em um erro de permissão; para evitar esse erro, instale o pacote para sua conta de usuário ou use um ambiente virtual:

pip install --user django

Uma vez instalado - você terá acesso à ferramenta de bootstrapping django-admin, que criará um diretório com alguns padrões para iniciar o desenvolvimento.

django-admin startproject myproject

Isso criará um diretório myproject com o layout padrão do projeto.

Conceitos principais do Django #

O Django é um framework de desenvolvimento web full stack e rico em recursos. Ele reúne muitas funcionalidades para fornecer uma experiência comum, rápida e produtiva para desenvolvedores da web.

Projetos Django consistem em configurações comuns e um ou mais aplicativos. Cada aplicativo é um conjunto de funcionalidades juntamente com dependências (como modelos e modelos) que são agrupados como módulos Python.

O script de bootstrapping do django cria automaticamente um arquivo de configurações para seu projeto, com os recursos mais comuns ativados.

Esse conceito de aplicativos permite fácil plug-and-play de funcionalidade e há uma grande biblioteca de aplicativos disponíveis para lidar com as tarefas mais comuns. Este conceito de aplicações é fundamental para o django; muitas das funcionalidades embutidas (como autenticação de usuário e o site de administração) são simplesmente aplicativos django.

Para criar seu primeiro aplicativo, a partir do diretório do projeto:

python manage.py startapp yourapp

yourapp é o nome do seu aplicativo personalizado.

Cada aplicativo permite que você desenvolva:

  1. Uma série de views - são pedaços de código que são executados em resposta a uma solicitação.

  2. Um ou mais models; que são uma abstração para bancos de dados. Eles permitem que você defina seus objetos como objetos Python, e o ORM integrado fornece uma API amigável para armazenar, recuperar e filtrar objetos de bancos de dados.

  3. Intimamente relacionados aos modelos estão as migrações que são scripts gerados para fornecer um método consistente e confiável de aplicar alterações em seus modelos ao banco de dados.

  4. Um conjunto de urls aos quais o aplicativo responderá.

  5. Uma ou mais classes de administração; para personalizar como o aplicativo se comporta no aplicativo admin django integrado.

  6. Quaisquer testes que você possa escrever.

Conceitos principais - Visualizações #

Uma view é qualquer pedaço de código que responde a uma solicitação e retorna uma resposta. As visualizações normalmente retornam modelos junto com um dicionário (chamado context) que geralmente contém dados para espaços reservados no modelo. Em projetos django, as visualizações estão localizadas no módulo views.py das aplicações.

A visão mais simples, retorna uma resposta direta:

from django.http import HttpResponse

def simple_view(request):
   return HttpResponse('<strong>Hello World</strong>')

No entanto, a maioria das visualizações utiliza um modelo:

from django.shortcuts import render

def simple_template_view(request):
    return render(request, 'some_template.html')

Um modelo é simplesmente qualquer arquivo e, opcionalmente, pode conter marcação especial para funcionalidade adicional; o que isso significa é que as visualizações do django podem retornar qualquer tipo de resposta, não apenas HTML.

Conceitos Principais - Modelos #

No django, um template é simplesmente um arquivo que contém tags especiais que podem ser substituídas por dados da view.

O exemplo de modelo canônico seria:

<strong>Hello {{ name }}, I am a template!</strong>

Aqui, a string {{ name }} identifica um placeholder que pode ser substituído por um contexto.

Para renderizar este template a partir de uma view, podemos passar o valor como um dicionário:

from django.shortcuts import render

def simple_view(request):
    return render(request, 'template.html', {'name': 'Jim'})

Assim que essa visualização for renderizada, o HTML resultante será Hello Jim, I am a template!.

Conceitos principais - URLs #

No django, existe um mapeador de url que mapeia URLs para funções específicas (views) que retornam respostas. Essa separação estrita entre o layout do sistema de arquivos e o layout da URL permite grande flexibilidade ao escrever aplicativos.

Todos os padrões de url são armazenados em um ou mais arquivos urls.py, e existe um arquivo mestre urls.py que é lido pelo django primeiro.

O Django analisa os padrões na ordem em que são escritos e para quando encontra uma correspondência com a URL solicitada pelo usuário. Se nenhuma correspondência for encontrada, um erro será gerado.

No modo de depuração (ativado configurando DEBUG = True em settings.py), o django imprimirá uma mensagem de erro detalhada quando uma url solicitada não corresponder a nenhum padrão. Em produção, no entanto, o django exibirá uma mensagem 404 normal.

Um padrão de url consiste em uma expressão regular do Python, seguida por um callable (um método ou função) a ser chamado quando esse padrão for correspondido. Esta função deve retornar uma resposta HTTP:

url(r'/hello$', simple_view)
