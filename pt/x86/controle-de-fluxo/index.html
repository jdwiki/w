Condições de teste #

Para usar um salto condicional, uma condição deve ser testada. Testar uma condição aqui se refere apenas ao ato de verificar as bandeiras, o salto real é descrito em [Saltos condicionais](https://www.wikiod.com/pt/x86/controle-de-fluxo#Saltos condicionais -saltos).

x86 testa as condições contando com o registrador EFLAGS, que contém um conjunto de sinalizadores que cada instrução pode potencialmente definir.

Instruções aritméticas, como sub ou add, e instruções lógicas, como xor ou and, obviamente “definir os sinalizadores”. Isso significa que os sinalizadores CF, OF, SF, ZF, AF, PF são modificados por essas instruções. Qualquer instrução pode modificar os sinalizadores, por exemplo, cmpxchg modifica o ZF.

Sempre verifique a referência da instrução para saber quais sinalizadores são modificados por uma instrução específica.

x86 tem um conjunto de saltos condicionais, mencionados anteriormente, que saltam se e somente se alguns sinalizadores estiverem definidos ou alguns estiverem limpos ou ambos.

Bandeiras #

As operações aritméticas e lógicas são muito úteis para definir os sinalizadores. Por exemplo, depois de um sub eax, ebx, por agora mantendo valores unsigned, temos:

Bandeira Quando definido Quando claro
ZF Quando o resultado é zero.
EAX - EBX = 0 ⇒ EAX = EBX		 Quando o resultado é não zero.
EAX - EBX ≠ 0 ⇒ EAX ≠ EBX
CF Quando o resultado precisava ser transportado para o MSb.
EAX - EBX < 0 ⇒ EAX < EBX		 Quando o resultado não precisa ser carregado para o MSb.
EAX - EBX ≮ 0 ⇒ EAX ≮ EBX
SF Quando o resultado MSb é definido. Quando o resultado MSb não está definido.
DE Quando ocorreu um estouro assinado. Quando um estouro assinado não ocorreu.
PF Quando o número de bits definido no byte menos significativo do resultado é par. Quando o número de bits definido no byte menos significativo do resultado é ímpar.
AF Quando o dígito BCD inferior gerou um carry.
É o bit 4 de carry.		 Quando o dígito BCD inferior não gerou um transporte.
É o transporte do bit 4.

Ensaios não destrutivos #

As instruções sub e and modificam seu operando de destino e exigiriam duas cópias extras (salvar e restaurar) para manter o destino inalterado.

Para realizar um teste não destrutivo existem as instruções cmp e test. Eles são idênticos à sua contraparte destrutiva exceto que o resultado da operação é descartado e apenas os sinalizadores são salvos.

Destrutivo Não destrutivo
sub cmp
e teste 
test eax, eax             ;and eax, eax
                          ;ZF = 1 iff EAX is zero

test eax, 03h             ;and eax, 03h
                          ;ZF = 1 if both bit[1:0] are clear
                          ;ZF = 0 if at least one of bit[1:0] is set

cmp eax, 241d             ;sub eax, 241d
                          ;ZF = 1 iff EAX is 241
                          ;CF = 1 iff EAX < 241

Testes assinados e não assinados #

A CPU não dá nenhum significado especial para registrar valores1, sinal é uma construção do programador. Não há diferença ao testar valores assinados e não assinados. O processador calcula sinalizadores suficientes para testar as relações aritméticas usuais (igual, menor que, maior que, etc.) tanto se os operandos forem considerados com e sem sinal.

1 Embora tenha algumas instruções que só fazem sentido com formatos específicos, como complemento de dois. Isso é para tornar o código mais eficiente, pois a implementação do algoritmo em software exigiria muito código.

Saltos condicionais #

Com base no estado dos sinalizadores, a CPU pode executar ou ignorar um salto. Uma instrução que executa um salto com base nos sinalizadores se enquadra no nome genérico de Jcc - Salto no código de condição1.

Sinônimos e terminologia #

Para melhorar a legibilidade do código de montagem, a Intel definiu vários sinônimos para o mesmo código de condição. Por exemplo, jae, jnb e jnc são todos o mesmo código de condição CF = 0.

Embora o nome da instrução possa dar uma dica muito forte sobre quando usá-la ou não, a única abordagem significativa é reconhecer os sinalizadores que precisam ser testados e então escolher as instruções adequadamente. A Intel, no entanto, deu nomes às instruções que fazem todo o sentido quando usadas após uma instrução cmp. Para os propósitos desta discussão, será assumido que cmp definiu os sinalizadores antes de um salto condicional.

Igualdade #

Os operandos são iguais se ZF foi definido, eles diferem caso contrário. Para testar a igualdade precisamos de ZF = 1.

je a_label           ;Jump if operands are equal
jz a_label           ;Jump if zero (Synonym)

jne a_label          ;Jump if operands are NOT equal
jnz a_label          ;Jump if not zero (Synonym)
Instrução Bandeiras
é, jz ZF = 1
jne, jnz ZF = 0

Maior que #

Para operandos sem sinal, o destino é maior que a origem se o transporte não for necessário, ou seja, se CF = 0. Quando CF = 0 é possível que os operandos sejam iguais, testar ZF eliminará a ambiguidade.

jae a_label      ;Jump if above or equal (>=)
jnc a_label      ;Jump if not carry (Synonym)
jnb a_label      ;Jump if not below (Synonym)

ja a_label       ;Jump if above  (>)
jnbe a_label     ;Jump if not below and not equal (Synonym)
Instrução Bandeiras
, jae, jnc, jnb FC = 0
ja, jnbe CF = 0, ZF = 0

Para operandos assinados, precisamos verificar se SF = 0, a menos que tenha havido um estouro de sinal, caso em que o SF resultante é revertido. Como OF = 0 se nenhum estouro de sinal ocorreu e 1 caso contrário, precisamos verificar se SF = OF.

ZF pode ser usado para implementar um teste estrito/não estrito.

jge a_label      ;Jump if greater or equal (>=)
jnl a_label      ;Jump if not less (Synonym)

jg a_label       ;Jump if greater (>)
jnle a_label     ;Jump if not less and not equal (Synonym)
Instrução Bandeiras
jge, jnl SF = DE
jg, jnle SF = OF, ZF = 0

Menor que #

Estes usam as condições invertidas acima.

jbe a_label      ;Jump if below or equal (<=)
jna a_label      ;Jump if not above (Synonym)

jb a_label       ;Jump if below (<)
jc a_label       ;Jump if carry (Synonym)
jnae a_label     ;Jump if not above and not equal (Synonym)

;SIGNED 

jle a_label      ;Jump if less or equal (<=)
jng a_label      ;Jump if not greater (Synonym)

jl a_label       ;Jump if less (<)
jnge a_label     ;Jump if not greater and not equal (Synonym)
Instrução Bandeiras
, jbe, jna CF = 1 ou ZF = 1
jb, jc, jnae FC = 1
jle, jng SF != OF ou ZF = 1
jl, jnge SF != DE

Sinalizadores específicos #

Cada sinalizador pode ser testado individualmente com j<flag_name> onde flag_name não contém o F à direita (por exemplo CF → C, PF → P).

Os códigos restantes não cobertos antes são:

Instrução Bandeira
js SF = 1
jns SF = 0
``sim'' DE = 1
jno OF = 0
jp, jpe (e = par) PF = 1
jnp, jpo (o = ímpar) PF = 0

Mais um salto condicional (um extra) #

Um salto condicional x86 especial não testa o sinalizador. Em vez disso, ele testa o valor do registrador cx ou ecx (com base no modo de endereço da CPU atual sendo 16 ou 32 bits), e o salto é executado quando o registrador contém zero.

Esta instrução foi projetada para validação do registro counter (cx/ecx) antes de instruções do tipo rep, ou antes de loops loop.

jcxz  a_label   ; jump if cx (16b mode) or ecx (32b mode) is zero
jecxz a_label   ; synonym of jcxz (recommended in source code for 32b target)
Instrução Registrar (não sinalizar)
jcxz, jecxz cx = 0 (modo 16b)
jcxz, jecxz ecx = 0 (modo 32b)

1 Ou algo assim.

a_label #

Nos exemplos acima, o a_label é o destino de destino da CPU quando a condição testada é “true”. Quando a condição testada for “falsa”, a CPU continuará na próxima instrução após o salto condicional.

Sinônimos #

Existem sinônimos de instrução que podem ser usados ​​para melhorar a legibilidade do código. Por exemplo ja e jnbe (Saltar não abaixo nem igual) são a mesma instrução.

Saltos incondicionais #

jmp a_label                      ;Jump to a_label
jmp bx                           ;Jump to address in BX
jmp WORD [aPointer]              ;Jump to address in aPointer
jmp 7c0h:0000h                   ;Jump to segment 7c0h and offset 0000h
jmp FAR WORD [aFarPointer]       ;Jump to segment:offset in aFarPointer

Relativo perto de saltos #

jmp a_label é:

  • aproximar Ele especifica apenas a parte de deslocamento do endereço lógico do destino. O segmento é considerado CS.
  • relativo A semântica da instrução é saltar rel bytes para frente1 do próximo endereço de instrução ou IP = IP + rel.

A instrução é codificada como EB <rel8> ou EB <rel16/32>, o montador escolhe a forma mais apropriada, geralmente preferindo uma mais curta. A substituição por montador é possível, por exemplo, com NASM jmp SHORT a_label, jmp WORD a_label e jmp DWORD a_label geram as três formas possíveis.

Absoluto indireto perto de saltos #

jmp bx e jmp WORD [aPointer] são:

  • aproximar Eles especificam apenas a parte de deslocamento do endereço lógico de destino. O segmento é considerado CS.
  • indireto absoluto A semântica das instruções é saltar para o endereço em reg ou mem ou IP = reg, IP = mem.

A instrução é codificada como FF /4, para a memória indireta o tamanho do operando é determinado como para qualquer outro acesso à memória.

Saltos distantes absolutos #

jmp 7c0h:0000h é:

  • distante Ele especifica ambas as partes do endereço lógico: o segmento e o deslocamento.

  • absoluto A semântica da instrução é saltar para o endereço segment:offset ou CS = segmento, IP = offset.

A instrução é codificada como EA <imm32/48> dependendo do tamanho do código. É possível escolher entre as duas formas em algum montador, por exemplo com NASM jmp 7c0h: WORD 0000h e jmp 7c0h: DWORD 0000h geram a primeira e a segunda forma.

Saltos distantes indiretos absolutos #

jmp FAR WORD [aFarPointer] é:

  • distante Ele especifica ambas as partes do endereço lógico: o segmento e o deslocamento.

  • Absoluta indireta A semântica da instrução é saltar para o segment:offset armazenado em mem2 ou CS = mem[23:16/32], IP = [15/31:0] .

A instrução é codificada como FF /5, o tamanho do operando pode ser controlador com os especificadores de tamanho. No NASM, um pouco não intuitivo, eles são jmp FAR WORD [aFarPointer] para um operando 16:16 e jmp FAR DWORD [aFarPointer] para um operando 16:32.

Saltos ausentes #

  • quase absoluto Pode ser emulado com um salto quase indireto.

      mov bx, target            ;BX = absolute address of target
  jmp bx

  • parente distante Não faz sentido ou muito estreito de uso de qualquer maneira.

1 O complemento de dois é usado para especificar um deslocamento com sinal e, assim, retroceder. 2 Que pode ser seg16:off16 ou seg16:off32, de tamanhos 16:16 e 16:32.
