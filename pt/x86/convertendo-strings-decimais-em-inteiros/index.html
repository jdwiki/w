Converter strings em inteiros é uma das tarefas comuns.

Aqui mostraremos como converter strings decimais em inteiros.

O pseudocódigo para fazer isso é:

function string_to_integer(str):
    result = 0
    for (each characters in str, left to right):
        result = result * 10
        add ((code of the character) - (code of character 0)) to result
    return result

Lidar com strings hexadecimais é um pouco mais difícil porque os códigos de caracteres geralmente não são contínuos ao lidar com vários tipos de caracteres, como dígitos (0-9) e alfabetos (a-f e A-F). Os códigos de caracteres são normalmente contínuos quando lidamos com apenas um tipo de caracteres (vamos lidar com dígitos aqui), portanto, lidaremos apenas com ambientes nos quais os códigos de caracteres para dígitos são contínuos.

IA-32 assembly, GAS, convenção de chamada cdecl #

# make this routine available outside this translation unit
.globl string_to_integer

string_to_integer:
    # function prologue
    push %ebp
    mov %esp, %ebp
    push %esi

    # initialize result (%eax) to zero
    xor %eax, %eax
    # fetch pointer to the string
    mov 8(%ebp), %esi

    # clear high bits of %ecx to be used in addition
    xor %ecx, %ecx
    # do the conversion
string_to_integer_loop:
    # fetch a character
    mov (%esi), %cl
    # exit loop when hit to NUL character
    test %cl, %cl
    jz string_to_integer_loop_end
    # multiply the result by 10
    mov $10, %edx
    mul %edx
    # convert the character to number and add it
    sub $'0', %cl
    add %ecx, %eax
    # proceed to next character
    inc %esi
    jmp string_to_integer_loop
string_to_integer_loop_end:

    # function epilogue
    pop %esi
    leave
    ret

Este código estilo GAS converterá a string decimal fornecida como primeiro argumento, que é empurrada na pilha antes de chamar esta função, para inteiro e a retornará via %eax. O valor de %esi é salvo porque é registrador de salvamento de callee e é usado.

Caracteres de overflow/wrapping e inválidos não são verificados para simplificar o código.

Em C, este código pode ser usado assim (assumindo que unsigned int e os ponteiros têm 4 bytes):

#include <stdio.h>

unsigned int string_to_integer(const char* str);

int main(void) {
    const char* testcases[] = {
        "0",
        "1",
        "10",
        "12345",
        "1234567890",
        NULL
    };
    const char** data;
    for (data = testcases; *data != NULL; data++) {
        printf("string_to_integer(%s) = %u\n", *data, string_to_integer(*data));
    }
    return 0;
}

Nota: em alguns ambientes, dois string_to_integer no código assembly devem ser alterados para _string_to_integer (adicionar sublinhado) para permitir que funcione com código C.

MS-DOS, função TASM/MASM para ler um inteiro sem sinal de 16 bits #

Leia um inteiro sem sinal de 16 bits da entrada. #

Esta função usa o serviço de interrupção Int 21/AH=0Ah para ler uma string em buffer. O uso de uma string em buffer permite que o usuário revise o que digitou antes de passá-lo ao programa para processamento. Até seis dígitos são lidos (como 65535 = 216 - 1 tem seis dígitos).

Além de realizar a conversão padrão de numeral para number esta função também detecta entrada inválida e estouro (número muito grande para caber 16 bits).

Valores de retorno #

A função retorna o número lido em AX. Os sinalizadores ZF, CF, OF informam se a operação foi concluída com sucesso ou não e por quê.

Erro EIXO ZF FC DE
Nenhum O inteiro de 16 bits Definir Não definido Não definido
Entrada inválida O número parcialmente convertido, até o último dígito válido encontrado Não definido Definir Não definido
Estouro 7ff Não definido Definir Definir

O ZF pode ser usado para diferenciar rapidamente entradas válidas e inválidas.

Uso #

call read_uint16
jo _handle_overflow            ;Number too big (Optional, the test below will do)
jnz _handle_invalid            ;Number format is invalid

;Here AX is the number read

Código #

;Returns:
  ;
  ;If the number is correctly converted:
  ;   ZF = 1, CF = 0, OF = 0
  ;   AX = number
  ;
  ;If the user input an invalid digit:
  ;   ZF = 0, CF = 1, OF = 0
  ;   AX = Partially converted number
  ;
  ;If the user input a number too big
  ;   ZF = 0, CF = 1, OF = 1
  ;   AX = 07fffh
  ;
  ;ZF/CF can be used to discriminate valid vs invalid inputs
  ;OF can be used to discrimate the invalid inputs (overflow vs invalid digit)
  ;
  read_uint16:
    push bp   
    mov bp, sp

    ;This code is an example in Stack Overflow Documentation project.
    ;x86/Converting Decimal strings to integers


    ;Create the buffer structure on the stack

    sub sp, 06h                ;Reserve 6 byte on the stack (5 + CR)
    push 0006h                 ;Header

    push ds
    push bx
    push cx
    push dx

    ;Set DS = SS

    mov ax, ss
    mov ds, ax                           
                  

    ;Call Int 21/AH=0A

    lea dx, [bp-08h]            ;Address of the buffer structure
    mov ah, 0ah
    int 21h

    ;Start converting

    lea si, [bp-06h]
    xor ax, ax
    mov bx, 10
    xor cx, cx

   _r_ui16_convert:

    ;Get current char

    mov cl, BYTE PTR [si]
    inc si

    ;Check if end of string

    cmp cl, CR_CHAR
    je _r_ui16_end                      ;ZF = 1, CF = 0, OF = 0
   
    ;Convert char into digit and check

    sub cl, '0'
    jb _r_ui16_carry_end                ;ZF = 0, CF = 1, OF = X -> 0
    cmp cl, 9
    ja _r_ui16_carry_end                ;ZF = 0, CF = 0 -> 1, OF = X -> 0


    ;Update the partial result (taking care of overflow)

    ;AX = AX * 10
    mul bx

    ;DX:AX = DX:AX + CX
    add ax, cx
    adc dx, 0
       
    test dx, dx
   jz _r_ui16_convert            ;No overflow
   
    ;set OF and CF
    mov ax, 8000h
    dec ax                           
    stc

   jmp _r_ui16_end                      ;ZF = 0, CF = 1, OF = 1
    
   _r_ui16_carry_end:

    or bl, 1                ;Clear OF and ZF
    stc                     ;Set carry
 
    ;ZF = 0, CF = 1, OF = 0

   _r_ui16_end:
    ;Don't mess with flags hereafter!

    pop dx
    pop cx
    pop bx
    pop ds

    mov sp, bp

    pop bp
    ret

    CR_CHAR EQU 0dh

Portabilidade NASM #

Para portar o código para o NASM, remova a palavra-chave PTR dos acessos à memória (por exemplo, mov cl, BYTE PTR [si] torna-se mov cl, BYTE [si])

MS-DOS, função TASM/MASM para imprimir um número de 16 bits em binário, quaternário, octal, hexadecimal #

Imprima um número em binário, quaternário, octal, hexadecimal e uma potência geral de dois #

Todas as bases que são uma potência de dois, como o binário (21), quaternário (22), octal (23) , bases hexadecimais (24), têm um número inteiro de bits por dígito1. Assim, para recuperar cada dígito2 de um numeral, simplesmente quebramos o grupo de introdução do número de n bits a partir do LSb (à direita). Por exemplo, para a base quaternária, dividimos um número de 16 bits em grupos de dois bits. Existem 8 desses grupos. Nem toda potência de duas bases tem um número inteiro de grupos que cabe em 16 bits; por exemplo, a base octal tem 5 grupos de 3 bits que representam 3,5 = 15 bits de 16, deixando um grupo parcial de 1 bit3.

O algoritmo é simples, isolamos cada grupo com um deslocamento seguido de uma operação AND. Este procedimento funciona para todos os tamanhos de grupos ou, em outras palavras, para qualquer potência de base de dois.

Para mostrar os dígitos na ordem correta a função começa isolando o grupo mais significativo (o mais à esquerda), para isso é importante saber: a) quantos bits D é um grupo e b) a posição do bit *S * onde o grupo mais à esquerda começa. Esses valores são pré-calculados e armazenados em constantes cuidadosamente criadas.

Parâmetros #

Os parâmetros devem ser inseridos na pilha. Cada um tem 16 bits de largura. Eles são mostrados em ordem de empurrão.

Parâmetro Descrição
N O número a ser convertido
Base A base a ser usada expressa usando as constantes BASE2, BASE4, BASE8 e BASE16
Imprimir zeros à esquerda Se zero nenhum zero não significativo for impresso, caso contrário, eles serão. O número 0 é impresso como “0” embora

Uso #

push 241
push BASE16
push 0
call print_pow2              ;Prints f1

push 241
push BASE16
push 1
call print_pow2              ;Prints 00f1

push 241
push BASE2
push 0
call print_pow2              ;Prints 11110001

Observação para usuários do TASM: Se você colocar as constantes definidas com EQU após o código que as utiliza, habilite multi-pass com o sinalizador /m de TASM ou você obterá Forward a referência precisa ser substituída.

Código #

;Parameters (in order of push):
;
;number
;base (Use constants below)
;print leading zeros
print_pow2:
 push bp
 mov bp, sp

 push ax
 push bx
 push cx
 push dx
 push si
 push di

 ;Get parameters into the registers

 ;SI = Number (left) to convert
 ;CH = Amount of bits to shift for each digit (D)
 ;CL = Amount od bits to shift the number (S)
 ;BX = Bit mask for a digit

 mov si, WORD PTR [bp+08h]
 mov cx, WORD PTR [bp+06h]            ;CL = D, CH = S

 ;Computes BX = (1 << D)-1
 
 mov bx, 1
 shl bx, cl
 dec bx

 xchg cl, ch              ;CL = S, CH = D

_pp2_convert:
 mov di, si
 shr di, cl
 and di, bx         ;DI = Current digit

 or WORD PTR [bp+04h], di             ;If digit is non zero, [bp+04h] will become non zero
                      ;If [bp+04h] was non zero, result is non zero
 jnz _pp2_print                       ;Simply put, if the result is non zero, we must print the digit

 
 ;Here we have a non significant zero
 ;We should skip it BUT only if it is not the last digit (0 should be printed as "0" not
 ;an empty string)

 test cl, cl
 jnz _pp_continue


_pp2_print:
 ;Convert digit to digital and print it

 
 mov dl, BYTE PTR [DIGITS + di]
 mov ah, 02h
 int 21h


_pp_continue:
 ;Remove digit from the number

 sub cl, ch
jnc _pp2_convert

 pop di
 pop si
 pop dx
 pop cx
 pop bx
 pop ax

 pop bp
 ret 06h

Dados #

This data must be put in the data segment, the one reached by `DS`.

DIGITS    db    "0123456789abcdef"

;Format for each WORD is S D where S and D are bytes (S the higher one)
;D = Bits per digit  --> log2(BASE)
;S = Initial shift count --> D*[ceil(16/D)-1]

BASE2    EQU    0f01h
BASE4    EQU    0e02h
BASE8    EQU    0f03h
BASE16    EQU    0c04h

Portabilidade NASM #

Para portar o código para o NASM, remova a palavra-chave PTR dos acessos à memória (por exemplo, mov si, WORD PTR [bp+08h] torna-se mov si, WORD PTR [bp+08h])

Extensão da função #

A função pode ser facilmente estendida para qualquer base até 2255, embora cada base acima de 216 imprima o mesmo numeral, pois o número é de apenas 16 bits.

Para adicionar uma base:

  1. Defina uma nova constante BASEx onde x é 2n. O byte inferior, denominado D, é D = n. O byte superior, denominado S, é a posição, em bits, do grupo superior. Pode ser calculado como S = n · (⌈16/n⌉ - 1).
  2. Adicione os dígitos necessários à string DIGITS.

Exemplo: adicionar base 32

Temos D = 5 e S = 15, então definimos BASE32 EQU 0f05h. Em seguida, adicionamos mais dezesseis dígitos: DIGITS db "0123456789abcdefghijklmnopqrstuv".

Como deve ficar claro, os dígitos podem ser alterados editando a string DIGITS.

1 Se B for uma base, então terá B dígitos por definição. O número de bits por dígito é, portanto, log2(B). Para potência de duas bases, isso simplifica para log2(2n) = n que é um número inteiro por definição.

2 Neste contexto, assume-se implicitamente que a base em consideração é uma potência de duas bases 2n.

3 Para uma base B = 2n ter um número inteiro de grupos de bits, deve ser n | 16 (n divide 16). Como o único fator em 16 é 2, deve ser que n seja uma potência de dois. Portanto, B tem a forma 22k ou equivalentemente log2(log2(B)) deve ser um número inteiro.

MS-DOS, TASM/MASM, função para imprimir um número de 16 bits em decimal #

Imprimir um número sem sinal de 16 bits em decimal #

O serviço de interrupção Int 21/AH=02h é usado para imprimir os dígitos. A conversão padrão de number para numeral é realizada com a instrução div, o dividendo é inicialmente a maior potência de dez com 16 bits (104) e é reduzido para potências menores em cada iteração.

Parâmetros #

Os parâmetros são mostrados em ordem de push. Cada um é de 16 bits.

Parâmetro Descrição
número O número sem sinal de 16 bits a ser impresso em decimal
mostrar zeros à esquerda Se 0 nenhum zero não significativo for impresso, senão eles são. O número 0 é sempre impresso como “0”

Uso #

push 241
push 0
call print_dec          ;prints 241

push 56
push 1
call print_dec          ;prints 00056

push 0
push 0
call print_dec          ;prints 0

Código #

;Parameters (in order of push):
;
;number
;Show leading zeros
print_dec:
 push bp
 mov bp, sp

 push ax
 push bx
 push cx
 push dx

 ;Set up registers:
 ;AX = Number left to print
 ;BX = Power of ten to extract the current digit
 ;DX = Scratch/Needed for DIV
 ;CX = Scratch

 mov ax, WORD PTR [bp+06h]
 mov bx, 10000d
 xor dx, dx

_pd_convert:     
 div bx                           ;DX = Number without highmost digit, AX = Highmost digit
 mov cx, dx                       ;Number left to print

 ;If digit is non zero or param for leading zeros is non zero
 ;print the digit
 or WORD PTR [bp+04h], ax
 jnz _pd_print

 ;If both are zeros, make sure to show at least one digit so that 0 prints as "0"
 cmp bx, 1
 jne _pd_continue

_pd_print:

 ;Print digit in AL

 mov dl, al
 add dl, '0'
 mov ah, 02h
 int 21h

_pd_continue:
 ;BX = BX/10
 ;DX = 0

 mov ax, bx
 xor dx, dx
 mov bx, 10d
 div bx
 mov bx, ax

 ;Put what's left of the number in AX again and repeat...
 mov ax, cx

 ;...Until the divisor is zero
 test bx, bx
jnz _pd_convert

 pop dx
 pop cx
 pop bx
 pop ax

 pop bp
 ret 04h

Portabilidade NASM #

Para portar o código para o NASM, remova a palavra-chave PTR dos acessos à memória (por exemplo, mov ax, WORD PTR [bp+06h] torna-se mov ax, WORD [bp+06h])
