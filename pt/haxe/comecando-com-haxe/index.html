Olá Mundo #

Requisitos #

  1. Uma versão do kit de ferramentas Haxe deve ser instalada
  2. Haxe deve estar presente no caminho do sistema
  3. A linha de comando deve ser acessível

Código #

Navegue até um diretório de projeto desejado e crie um arquivo fonte Test.hx com o seguinte conteúdo:

class Test {
    static function main() {
        trace("Hello world");
    }
}

Os arquivos de origem Haxe são chamados de módulos. Um módulo deve definir um tipo (abstract, class, enum, interface ou typedef) com o mesmo identificador do nome do módulo - neste caso a classe Test. Uma vez que esse requisito seja atendido, um módulo pode definir um número arbitrário de tipos diferentes.

Os programas Haxe requerem um ponto de entrada, conforme indicado pela função estática main. A classe que implementa o ponto de entrada é a classe de inicialização ou classe principal. Novamente, neste caso a classe principal é a classe Test.

A função trace() é uma função de log de propósito geral exposta ao namespace global por conveniência. Ele gera o identificador de saída padrão do idioma de destino (por exemplo, console do navegador para JavaScript, linha de comando para C++). Consulte a documentação da API para obter mais informações.

Execução #

Navegue até a pasta do projeto na linha de comando. Teste para ver se o Haxe está configurado em seu ambiente chamando:

haxe --help

O interpretador Haxe pode ser usado para testar código que não depende de nenhuma API de idioma de destino específico. Use o interpretador chamando:

``haxe -main Test –interp''

Lembre-se, o módulo Test contém a classe de inicialização Test, razão pela qual -main Test é passado para o compilador.

Fontes Haxe podem compilar (transpile) para fontes/bytecodes de várias linguagens diferentes. A tabela a seguir exibe o idioma de destino, o sinalizador do compilador, o tipo de argumento e o resultado da compilação. Use-o chamando:

haxe -main Test [flag] [argumento].

Idioma Bandeira Argumento Resultado
ActionScript 3 -as3 Diretório Fonte
C# -cs Diretório Fonte + bytecode opcional (.exe)
C++ -cpp Diretório Fonte + binário opcional (nativo)
Flash -swf Arquivo Bytecode (.swf)
HL -hl Arquivo Fonte
Segundo -dois Arquivo Fonte
Java -java Diretório Fonte + bytecode opcional (.jar)
JavaScript -js Arquivo Fonte
Neko -neko Arquivo Bytecode (.n)
PHP -php Diretório Fonte
Python -python Arquivo Fonte
HashLink -hl Arquivo Bytecode (.hl)

Observe que os argumentos de caminho aqui são relativos ao caminho de onde haxe foi chamado. As saídas de bytecode/binário opcionais podem ser desativadas adicionando os sinalizadores -D no-compilation, a fim de evitar uma etapa de compilação adicional envolvendo a chamada do compilador da linguagem de destino.

Instalação #

Haxe está disponível no Windows, Linux e OS X. Ele é distribuído em duas formas:

  • como um instalador, fornecendo uma dependência opcional Neko VM e configurando as variáveis ​​de ambiente haxe e haxelib;
  • como binários, fornecendo apenas o compilador Haxe e o gerenciador de pacotes.

Janelas #

O instalador e os binários estão disponíveis no site da Haxe.

##Linux

Binários (32 bits e 64 bits) estão disponíveis no site da Haxe.

A Fundação Haxe também participa oficialmente da manutenção dos pacotes Haxe e Neko para distribuições Linux populares. Recomenda-se usar esses pacotes, se disponíveis.

Ubuntu #

Recomenda-se usar o Haxe PPA que fornece as versões mais recentes do Haxe e Neko para todas as versões do Ubuntu atualmente suportadas. O PPA também pode ser usado para distribuições baseadas no Ubuntu.

sudo add-apt-repository ppa:haxe/releases -y
sudo apt-get update
sudo apt-get install haxe -y
mkdir ~/haxelib && haxelib setup ~/haxelib

Observe que o Neko é instalado como uma dependência do Haxe.

Debian #

Para instalar as versões estáveis ​​atualmente disponíveis, execute os seguintes comandos:

sudo apt-get install haxe -y
mkdir ~/haxelib && haxelib setup ~/haxelib

Observe que o Neko será instalado como uma dependência do Haxe.

Para instalar versões mais recentes do canal instável, faça o seguinte:

  1. Em /etc/apt/sources.list, adicione

    deb http://httpredir.debian.org/debian unstable main contrib non-free

  2. Em /etc/apt/preferences.d/, crie um novo arquivo chamado unstable com o seguinte conteúdo:

    Package: *
Pin: release a=unstable
Pin-Priority: 100

Package: haxe neko libneko*
Pin: release a=unstable
Pin-Priority: 999

  3. Extraia os arquivos de índice do pacote da fonte recém-adicionada:

    sudo apt-get update

  4. Instale o Haxe (e Neko):

    sudo apt-get install haxe -y

Fedora #

A Fundação Haxe mantém os pacotes RPM Haxe e Neko no repositório Fedora. Os pacotes são atualizados na maioria das vezes. No entanto, quando uma nova versão do Haxe for lançada, levará alguns dias, até 2 semanas, para enviar um pacote atualizado para as versões estáveis ​​do Fedora. As atividades de atualização podem ser rastreadas no Bodhi Fedora Update System.

Para instalar as versões atualmente disponíveis do Haxe e Neko, execute os seguintes comandos:

sudo dnf install haxe -y
mkdir ~/haxelib && haxelib setup ~/haxelib

Observe que o Neko é instalado como uma dependência do Haxe.

openSuse #

A Fundação Haxe mantém os pacotes RPM Haxe e Neko no repositório openSUSE:Factory. Os pacotes são atualizados na maioria das vezes. No entanto, quando uma nova versão do Haxe for lançada, levará alguns dias, até 2 semanas, para ser aceita pelo openSUSE:Factory.

Para instalar as versões atualmente disponíveis do Haxe e Neko, execute os seguintes comandos:

sudo zypper install haxe
mkdir ~/haxelib && haxelib setup ~/haxelib

Observe que o Neko é instalado como uma dependência do Haxe.

Para obter a versão mais recente do Haxe que pode não estar disponível para o openSUSE:Factory ou uma versão do openSUSE, use o projeto devel:languages:haxe no openSUSE Build Service. Visite a página do pacote Haxe, clique em “Baixar pacote” no canto superior direito e siga as instruções. Novamente, o Neko também será instalado como uma dependência do Haxe.

Arch Linux #

Existem pacotes Haxe e Neko no repositório da comunidade Arch Linux. A Fundação Haxe continuará ajudando a manter os pacotes atualizados. No entanto, quando uma nova versão do Haxe for lançada, levará tempo para atualizar o pacote, dependendo da disponibilidade do mantenedor do pacote.

Para as versões atualmente disponíveis do Haxe e Neko, verifique as seguintes páginas:

  • Haxe no Arch Linux
  • Neko no Arch Linux

Para instalar as versões atualmente disponíveis do Haxe e Neko, execute os seguintes comandos:

sudo pacman -S haxe
mkdir ~/haxelib && haxelib setup ~/haxelib

Observe que o Neko é instalado como uma dependência do Haxe.

OS X #

O instalador e os binários estão disponíveis no site da Haxe.

Também é possível instalar a versão estável atual do Haxe através do gerenciador de pacotes Brew.

brew install haxe
