Instale o Magento 2 no Ubuntu 16.04 #

NOTAS: Vamos instalar o Magento 2 no novo Ubuntu Server 16.04 LTS com PHP 7.0, MySQL 5.6 e Apache 2.4.

1. Requisitos de configuração #

  • Apache 2.2 ou 2.4 com módulo mod_rewrite (ou) Nginx >= 1.8.
  • PHP 5.5 ou versão posterior. PHP 7.0 também é suportado.
  • Módulos PHP necessários – PDO/MySQL, mbstring, mcrypt, mhash, SimpleXML, curl, xsl, gd, ImageMagick 6.3.7 (ou posterior) ou ambos, soap, intl, openssl.
  • Compositor e Git.

Você pode usar o seguinte comando para instalar todos os requisitos acima do repositório padrão (xenial).

sudo apt install apache2 git mysql-server
sudo apt install php libapache2-mod-php php-mysql php-dom php-simplexml php-gd
sudo apt install php-curl php-intl php-xsl php-mbstring php-zip php-xml php-mcrypt

Recomendo instalar a partir da página inicial em vez do repositório do Ubuntu.

curl -sS https://getcomposer.org/installer | php
mv composer.phar /usr/local/bin/composer
chmod +x /usr/local/bin/composer

2. Configure o Magento 2 #

a) Baixe do GitHub #

O código Magento2 está disponível no repositório Github. Use o seguinte comando para clonar o repositório Magento2 em seu sistema.

cd /var/www/
git clone https://github.com/magento/magento2.git

b) Baixar via Composer #

Se você não quiser instalar o Magento 2 clonando do GitHub, tudo bem. Você também pode instalá-lo através do Composer.

cd /var/www
composer create-project --repository-url=https://repo.magento.com/ magento/project-community-edition magento2

Agora instale todos os módulos necessários para Magento2 usando composer. Aguarde a conclusão do processo de instalação. (Você não precisará disso se estiver instalando o Magento 2 via Composer)

cd magento2/
composer install

Se o compositor solicitar autenticação como abaixo:

Agora defina as permissões em arquivos e diretórios.

sudo chmod -R 755 /var/www/magento2/
sudo chmod -R 777 /var/www/magento2/{pub,var}

3. Criar banco de dados #

Agora faça login no seu servidor mysql com privilégios de administrador e crie um banco de dados e um usuário para a nova instalação do magento2.

mysql -u root -p

mysql> CREATE DATABASE magento;
mysql> GRANT ALL ON magento.* TO magento@'localhost' IDENTIFIED BY 'magento';
mysql> FLUSH PRIVILEGES;
mysql> quit

4. Configure Apache VirtualHost e PHP #

Crie o arquivo de configuração do Apache para o seu site Magento como /etc/apache2/sites-available/magento2.example.com.conf e adicione o seguinte conteúdo.

<VirtualHost *:80>
   DocumentRoot /var/www/magento2
   ServerName magento2.example.com
   
   <Directory /var/www/magento2>
       AllowOverride all
   </Directory>
</VirtualHost>

Agora habilite o virtualhost usando o seguinte comando.

sudo a2ensite magento2.example.com

Certifique-se também de habilitar o módulo de reescrita do Apache, que é recomendado pelo Magento.

sudo a2enmod rewrite

Você pode querer configurar o PHP memory_limit para evitar o esgotamento da memória, o que também é recomendado pelo Magento.

vi /etc/php.ini (find string by press / and type memory_limit)
memory_limit = 768M

Depois de fazer todas as alterações acima, certifique-se de reiniciar o servidor Apache.

sudo systemctl restart apache2.service

5. Instalando o aplicativo Magento 2 #

a) Via Instalador Web #

Vamos começar a instalação do Magento2 usando o instalador da web. Acesse seu diretório magento2 no navegador da web como abaixo. Ele irá redirecioná-lo para a página inicial da instalação.

http://magento2.example.com/

b) Via Linha de Comando #

Instalar o Magento 2 usando a linha de comando é um milagre, diminuiu o tempo de instalação de 10min para 1min. Apenas execute o comando de uma linha.

cd /var/www/magento2
php bin/magento setup:install --base-url=http://magento2.example.com/ \
--db-host=localhost --db-name=magento \
--db-user=magento --db-password=magento \
--admin-firstname=Magento --admin-lastname=User --admin-email=user@example.com \
--admin-user=admin --admin-password=admin123 --language=en_US \
--currency=USD --timezone=America/Chicago --cleanup-database --use-rewrites=1

6. Agende Magento2 Cronjobs #

Finalmente agende os cronjobs de fundo para sua instalação do magento2. Esses cronjobs fazem algumas atividades como, reindexação, Newsletters, atualização de taxas de câmbio, envio de e-mails automáticos e geração de sitemaps etc. Para agendar esses trabalhos, edite o arquivo crontab. www-data é um usuário do Apache 2, nós nunca devemos agendar o cronjob do Magento 2 com privilégio de root.

crontab -u www-data -e

Um editor de texto é exibido. (Talvez seja necessário escolher um editor de texto primeiro.)

* * * * * /usr/bin/php /var/www/magento2/bin/magento cron:run | grep -v "Ran jobs by schedule" >> /var/www/magento2/var/log/magento.cron.log
* * * * * /usr/bin/php /var/www/magento2/update/cron.php >> /var/www/magento2/var/log/update.cron.log
* * * * * /usr/bin/php /var/www/magento2/bin/magento setup:cron:run >> /var/www/magento2/var/log/setup.cron.log
