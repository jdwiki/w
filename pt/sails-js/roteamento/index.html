Rotas são regras que dizem ao Sails o que fazer quando se depara com uma solicitação recebida.

As rotas são definidas em config/routes.js. A ordem das rotas é significativa, pois as rotas são combinadas de cima para baixo. Isso significa que se você tiver uma rota específica que também pode ser correspondida por uma rota curinga, a rota específica deve ser definida acima da rota curinga.

Quando uma solicitação entra em seu aplicativo, o sails.js pega todos os parâmetros que a acompanham e os disponibiliza para você como params no objeto da solicitação.

As propriedades no objeto de destino de rota serão passadas para o manipulador de rota no objeto req.options. A seguir estão propriedades reservadas que podem afetar o comportamento do manipulador de rotas:

Propriedade Tipos de alvos aplicáveis ​​ Tipo de dados Detalhes
pular Ativos tudo Booleano Defina como true se você não quiser que a rota corresponda a URLs com pontos (por exemplo, myImage.jpg). Isso impedirá que suas rotas com notação curinga correspondam a URLs de ativos estáticos. Útil ao criar slugs de URL.
skipRegex tudo Regexp Se pular todas as URLs contendo um ponto for muito permissivo, ou se você precisar que o manipulador de uma rota seja ignorado com base em critérios completamente diferentes, você pode usar skipRegex. Essa opção permite que você especifique uma expressão regular ou uma matriz de expressões regulares para corresponder à URL de solicitação; se qualquer uma das correspondências for bem-sucedida, o manipulador será ignorado. Observe que, diferentemente da sintaxe para vincular um manipulador a uma expressão regular, o skipRegex espera objetos _regExp reais, não strings.
locais controlador, visualização, blueprint, resposta Dicionário Define variáveis ​​locais padrão para passar para qualquer exibição renderizada durante o tratamento da solicitação.
cors tudo Dicionário ou Booleano ou String Especifica como lidar com solicitações para esta rota de uma origem diferente.
preencher planta Booleano Indica se os resultados em uma ação de blueprint “localizar” ou “localizar” devem ter campos de modelo associados preenchidos. O padrão é o valor definido em config/blueprints.js.
pular, limitar, classificar, onde planta Dicionário Defina critérios para o blueprint “localizar”.

Definições de rota RESTful personalizadas #

module.exports.routes = {
    'GET /foo': 'FooController.index',
    'GET /foo/new': 'FooController.new',
    'POST /foo/create': 'FooController.create',
    'GET /foo/:id/edit': 'FooController.edit',
    'PUT /foo/:id/update': 'FooController.update',
    'GET /foo/:id': 'FooController.show',
    'DELETE /foo/:id': 'FooController.delete',
};

Redirecionar #

module.exports.routes = {
  '/foo': '/bar',
  'GET /google': 'http://www.google.com'
};

Definir variável personalizada para todas as visualizações #

module.exports.routes = {
  // This function will be executed for all http verbs on all urls
  'all /*', function (req, res, next) {
    // Expose the function `fooBar` to all views (via the locals object)
    res.locals.fooBar = function (arg1) {
      return 'foobar' + arg1;
    };
  },
};

Ignorar ativos (urls com pontos neles) da rota curinga #

module.exports.routes = {
  'GET /foo/*': {
     fn: function(req, res) {
       res.send("FOO!");
     },
     skipAssets: true
  },
};

Rotas com RegEx #

module.exports.routes = {
    // sends matching regex (few patterns) as 'page' param
    'r|/foo/([0-9]+)|page': 'FooController.get',
    'r|/foo/(.*)|page': 'FooController.get',
    'r|/foo/(\\w+)|page': 'FooController.get'
};

slugs de URL #

module.exports.routes = {
  'GET /blog/:year/:month/:day/:posttitle/': 'BlogController.showPost',
  'GET /blog/:year/:month/:day/': 'BlogController.showDayArchive',
  'GET /blog/:year/:month/': 'BlogController.showMonthArchive',
  'GET /blog/:year/': 'BlogController.showYearArchive',
};

Os parâmetros passados ​​na URL podem ser acessados ​​nas ações correspondentes do controlador usando req.param('year'), req.param('month') etc.

Por exemplo, uma solicitação GET para /blog/2016/08/ aciona a ação do controlador BlogController.showMonthArchive, com req.param('year') tendo o valor 2016 e req.param ('mês') com o valor 08.
