Lidando com chaves estrangeiras referenciando tabelas grandes #

Por padrão, o Django renderiza os campos ForeignKey como uma entrada <select>. Isso pode fazer com que as páginas sejam carregadas muito lentamente se você tiver milhares ou dezenas de milhares de entradas na tabela referenciada. E mesmo que você tenha apenas centenas de entradas, é bastante desconfortável procurar uma entrada específica entre todas.

Um módulo externo muito útil para isso é o django-autocomplete-light (DAL). Isso permite usar campos de preenchimento automático em vez de campos <select>.

visualizações.py #

from dal import autocomplete

class CityAutocomp(autocomplete.Select2QuerySetView):
    def get_queryset(self):
        qs = City.objects.all()
        if self.q:
            qs = qs.filter(name__istartswith=self.q)
        return qs

##urls.py

urlpatterns = [
    url(r'^city-autocomp/$', CityAutocomp.as_view(), name='city-autocomp'),
]

##forms.py

from dal import autocomplete

class PlaceForm(forms.ModelForm):
    city = forms.ModelChoiceField(
        queryset=City.objects.all(),
        widget=autocomplete.ModelSelect2(url='city-autocomp')
    )

    class Meta:
        model = Place
        fields = ['__all__']

admin.py #

@admin.register(Place)
class PlaceAdmin(admin.ModelAdmin):
    form = PlaceForm

Estilos CSS adicionais e scripts JS para a página de administração #

Suponha que você tenha um modelo Cliente simples:

class Customer(models.Model):
    first_name = models.CharField(max_length=255)
    last_name = models.CharField(max_length=255)
    is_premium = models.BooleanField(default=False)

Você o registra no admin do Django e adiciona o campo de busca por first_name e last_name:

@admin.register(Customer)
class CustomerAdmin(admin.ModelAdmin):
    list_display = ['first_name', 'last_name', 'is_premium']
    search_fields = ['first_name', 'last_name']

Depois de fazer isso, os campos de pesquisa aparecem na página da lista de administração com o espaço reservado padrão: “keyword”. Mas e se você quiser alterar esse espaço reservado para “Pesquisar por nome”?

Você pode fazer isso passando o arquivo Javascript personalizado para o administrador Media:

@admin.register(Customer)
class CustomerAdmin(admin.ModelAdmin):
    list_display = ['first_name', 'last_name', 'is_premium']
    search_fields = ['first_name', 'last_name']
    
    class Media:
        #this path may be any you want, 
        #just put it in your static folder
        js = ('js/admin/placeholder.js', )

Você pode usar a barra de ferramentas de depuração do navegador para encontrar qual id ou classe o Django definiu para esta barra de pesquisa e então escrever seu código js:

$(function () {
   $('#searchbar').attr('placeholder', 'Search by name')
})

Também a classe Media permite que você adicione arquivos css com o objeto de dicionário:

class Media:
    css = {
        'all': ('css/admin/styles.css',)
         }

Por exemplo, precisamos exibir cada elemento da coluna first_name em uma cor específica. Por padrão o Django cria uma coluna de tabela para cada item em list_display, todas as tags <td> terão uma classe css como field-'list_display_name', no nosso caso será field_first_name

.field_first_name {
     background-color: #e6f2ff;
 }

Se você quiser personalizar outro comportamento adicionando JS ou alguns estilos css, você sempre pode verificar id`s e classes de elementos na ferramenta de depuração do navegador.

Lista de alterações #

Digamos que você tenha um aplicativo myblog simples com o seguinte modelo:

from django.conf import settings
from django.utils import timezone

class Article(models.Model):
    title = models.CharField(max_length=70)
    slug = models.SlugField(max_length=70, unique=True)
    author = models.ForeignKey(settings.AUTH_USER_MODEL, models.PROTECT)
    date_published = models.DateTimeField(default=timezone.now)
    is_draft = models.BooleanField(default=True)
    content = models.TextField()

A “lista de alterações” do Django Admin é a página que lista todos os objetos de um determinado modelo.

from django.contrib import admin
from myblog.models import Article

@admin.register(Article)
class ArticleAdmin(admin.ModelAdmin):
    pass

Por padrão, ele usará o método __str__() (ou __unicode__() se você estiver em python2) do seu modelo para exibir o objeto “name”. Isso significa que, se você não a substituiu, verá uma lista de artigos, todos denominados “Objeto de artigo”. Para alterar esse comportamento, você pode definir o método __str__():

class Article(models.Model):
    def __str__(self):
        return self.title

Agora, todos os seus artigos devem ter um nome diferente e mais explícito que “Objeto de artigo”.

No entanto, você pode querer exibir outros dados nesta lista. Para isso, use list_display:

@admin.register(Article)
class ArticleAdmin(admin.ModelAdmin):
    list_display = ['__str__', 'author', 'date_published', 'is_draft']

list_display não se limita aos campos e propriedades do modelo. também pode ser um método do seu ModelAdmin:

from django.forms.utils import flatatt
from django.urls import reverse
from django.utils.html import format_html

@admin.register(Article)
class ArticleAdmin(admin.ModelAdmin):
    list_display = ['title', 'author_link', 'date_published', 'is_draft']
    
    def author_link(self, obj):
        author = obj.author
        opts = author._meta
        route = '{}_{}_change'.format(opts.app_label, opts.model_name)
        author_edit_url = reverse(route, args=[author.pk])
        return format_html(
            '<a{}>{}</a>', flatatt({'href': author_edit_url}), author.first_name)

    # Set the column name in the change list
    author_link.short_description = "Author"
    # Set the field to use when ordering using this column
    author_link.admin_order_field = 'author__firstname'
