Parâmetros #

Parâmetro Detalhes
nulo Se true, valores vazios podem ser armazenados como null no banco de dados
em branco Se true, o campo não será obrigatório nos formulários. Se os campos forem deixados em branco, o Django usará o valor de campo padrão.
escolhas Um iterável de iteráveis ​​de 2 elementos a serem usados ​​como opções para este campo. Se definido, o campo é renderizado como uma lista suspensa no admin. [('m', 'Masculino'),('f','Feminino'),('z','Prefiro não divulgar')]. Para agrupar as opções, basta aninhar os valores: [('Fonte de vídeo',((1,'YouTube'),(2,'Facebook')),('Fonte de áudio',((3, 'Soundcloud'), (4, 'Spotify'))]
db_column Por padrão, o django usa o nome do campo para a coluna do banco de dados. Use isso para fornecer um nome personalizado
db_index Se True, um índice será criado neste campo no banco de dados
db_tablespace O tablespace a ser usado para o índice deste campo. Este campo só é usado se o mecanismo de banco de dados o suportar, caso contrário será ignorado.
padrão O valor padrão para este campo. Pode ser um valor ou um objeto que pode ser chamado. Para padrões mutáveis ​​(uma lista, um conjunto, um dicionário), você deve usar um callable. Devido à compatibilidade com migrações, você não pode usar lambdas.
editável Se for False, o campo não é mostrado no model admin ou em qualquer ModelForm. O padrão é “Verdadeiro”.
mensagens_de_erro Usado para personalizar as mensagens de erro padrão mostradas para este campo. O valor é um dicionário, com as chaves representando o erro e o valor sendo a mensagem. As chaves padrão (para mensagens de erro) são null, blank, invalid, invalid_choice, unique e unique_for_date; mensagens de erro adicionais podem ser definidas por campos personalizados.
texto_ajuda Texto a ser exibido com o campo, para auxiliar os usuários. HTML é permitido.
on_delete Quando um objeto referenciado por uma ForeignKey é deletado, o Django irá emular o comportamento da restrição SQL especificada pelo argumento on_delete. Este é o segundo argumento posicional para os campos ForeignKey e OneToOneField. Outros campos não possuem este argumento.
chave_primária Se for True, este campo será a chave primária. O Django adiciona automaticamente uma chave primária; portanto, isso só é necessário se você deseja criar uma chave primária personalizada. Você só pode ter uma chave primária por modelo.
único Se for “True”, serão gerados erros se forem inseridos valores duplicados para este campo. Esta é uma restrição de nível de banco de dados e não simplesmente um bloco de interface do usuário.
unique_for_date Defina o valor para um DateField ou DateTimeField, e os erros serão gerados se houver valores duplicados para a mesma data ou data hora.
unique_for_month Semelhante a unique_for_date, exceto que os cheques são limitados para o mês.
unique_for_year Semelhante a unique_for_date, exceto que as verificações são limitadas ao ano.
verbose_name Um nome amigável para o campo, usado pelo django em vários lugares (como na criação de rótulos nos formulários de administração e modelo).
validadores Uma lista de validadores para este campo.
  • Você pode escrever seus próprios campos se achar necessário
  • Você pode substituir funções da classe do modelo base, mais comumente a função save()

Campos numéricos #

Exemplos de campos numéricos são fornecidos:

AutoCampo

Um inteiro com incremento automático geralmente usado para chaves primárias.

from django.db import models

class MyModel(models.Model):
    pk = models.AutoField()

Cada modelo recebe um campo de chave primária (chamado id) por padrão. Portanto, não é necessário duplicar um campo id no modelo para fins de chave primária.

BigIntegerField

Um número inteiro de ajuste de -9223372036854775808 a 9223372036854775807(8 Bytes).

from django.db import models

class MyModel(models.Model):
    number_of_seconds = models.BigIntegerField()

Campo inteiro

O IntegerField é usado para armazenar valores inteiros de -2147483648 a 2147483647 (4 Bytes).

from django.db import models

class Food(models.Model):
    name = models.CharField(max_length=255)
    calorie = models.IntegerField(default=0)

O parâmetro default não é obrigatório. Mas é útil definir um valor padrão.

PositiveIntegerField

Como um IntegerField, mas deve ser positivo ou zero (0). O PositiveIntegerField é usado para armazenar valores inteiros de 0 a 2147483647 (4 Bytes). Isso pode ser útil no campo que deve ser semanticamente positivo. Por exemplo, se você estiver registrando alimentos com suas calorias, não deve ser negativo. Este campo evitará valores negativos por meio de suas validações.

from django.db import models

class Food(models.Model):
    name = models.CharField(max_length=255)
    calorie = models.PositiveIntegerField(default=0)

O parâmetro default não é obrigatório. Mas é útil definir um valor padrão.

SmallIntegerField

O SmallIntegerField é usado para armazenar valores inteiros de -32768 a 32767 (2 Bytes). Este campo é útil para valores que não são extremos.

from django.db import models

class Place(models.Model):
    name = models.CharField(max_length=255)
    temperature = models.SmallIntegerField(null=True)

PositiveSmallIntegerField

O SmallIntegerField é usado para armazenar valores inteiros de 0 a 32767 (2 Bytes). Assim como SmallIntegerField, este campo é útil para valores não tão altos e deve ser semanticamente positivo. Por exemplo, pode armazenar a idade que não pode ser negativa.

from django.db import models

class Staff(models.Model):
    first_name = models.CharField(max_length=255)
    last_name = models.CharField(max_length=255)
    age = models.PositiveSmallIntegerField(null=True)

Além de PositiveSmallIntegerField ser útil para escolhas, esta é a maneira Djangoic de implementar o Enum:

from django.db import models
from django.utils.translation import gettext as _

APPLICATION_NEW = 1
APPLICATION_RECEIVED = 2
APPLICATION_APPROVED = 3
APPLICATION_REJECTED = 4

APLICATION_CHOICES = (
    (APPLICATION_NEW, _('New')),
    (APPLICATION_RECEIVED, _('Received')),
    (APPLICATION_APPROVED, _('Approved')),
    (APPLICATION_REJECTED, _('Rejected')),
)

class JobApplication(models.Model):
    first_name = models.CharField(max_length=255)
    last_name = models.CharField(max_length=255)
    status = models.PositiveSmallIntegerField(
        choices=APLICATION_CHOICES, 
        default=APPLICATION_NEW
    )
    ...

A definição das escolhas como variáveis ​​de classe ou variáveis ​​de módulo de acordo com a situação é uma boa maneira de usá-las. Se as opções forem passadas para o campo sem nomes amigáveis, isso criará confusão.

Campo Decimal

Um número decimal de precisão fixa, representado em Python por uma instância Decimal. Ao contrário de IntegerField e seus derivados, este campo possui 2 argumentos obrigatórios:

  1. DecimalField.max_digits: O número máximo de dígitos permitidos no número. Observe que esse número deve ser maior ou igual a decimal_places.
  2. DecimalField.decimal_places: O número de casas decimais a serem armazenadas com o número.

Se você deseja armazenar números até 99 com 3 casas decimais, você precisa usar max_digits=5 e decimal_places=3:

class Place(models.Model):
    name = models.CharField(max_length=255)
    atmospheric_pressure = models.DecimalField(max_digits=5, decimal_places=3)

Chave Estrangeira #

O campo ForeignKey é usado para criar um relacionamento muitos para um entre os modelos. Não como a maioria dos outros campos requer argumentos posicionais. O exemplo a seguir demonstra a relação carro e proprietário:

from django.db import models

class Person(models.Model):
    GENDER_FEMALE = 'F'
    GENDER_MALE = 'M'

    GENDER_CHOICES = (
        (GENDER_FEMALE, 'Female'),
        (GENDER_MALE, 'Male'),
    )

    first_name = models.CharField(max_length=100)
    last_name = models.CharField(max_length=100)
    gender = models.CharField(max_length=1, choices=GENDER_CHOICES)
    age = models.SmallIntegerField()


class Car(model.Model)
    owner = models.ForeignKey('Person')
    plate = models.CharField(max_length=15)
    brand = models.CharField(max_length=50)
    model = models.CharField(max_length=50)
    color = models.CharField(max_length=50)

O primeiro argumento do campo é a classe à qual o modelo está relacionado. O segundo argumento posicional é o argumento on_delete. Nas versões atuais este argumento não é obrigatório, mas será necessário no Django 2.0. A funcionalidade padrão do argumento é mostrada a seguir:

class Car(model.Model)
    owner = models.ForeignKey('Person', on_delete=models.CASCADE)
    ...

Isso fará com que os objetos Car sejam excluídos do modelo quando seu proprietário for excluído do modelo Person. Esta é a funcionalidade padrão.

class Car(model.Model)
    owner = models.ForeignKey('Person', on_delete=models.PROTECT)
    ...

Isso impedirá que objetos Pessoa sejam excluídos se estiverem relacionados a pelo menos um objeto Carro. Todos os objetos Car que fazem referência a um objeto Person devem ser excluídos primeiro. E então o Objeto Pessoa pode ser excluído.

Campo Char #

O CharField é usado para armazenar comprimentos de texto definidos. No exemplo abaixo, até 128 caracteres de texto podem ser armazenados no campo. A inserção de uma string maior que isso resultará em um erro de validação.

from django.db import models

class MyModel(models.Model):
    name = models.CharField(max_length=128, blank=True)

CampoDataHora #

DateTimeField é usado para armazenar valores de data e hora.

class MyModel(models.Model):
    start_time = models.DateFimeField(null=True, blank=True)
    created_on = models.DateTimeField(auto_now_add=True)
    updated_on = models.DateTimeField(auto_now=True)

Um DateTimeField tem dois parâmetros opcionais:

  • auto_now_add define o valor do campo para a data e hora atual quando o objeto é criado.

  • auto_now define o valor do campo para a data e hora atual toda vez que o campo é salvo.

Essas opções e o parâmetro default são mutuamente exclusivos.

Campo Binário #

Este é um campo especializado, usado para armazenar dados binários. Aceita apenas bytes. Os dados são serializados em base64 no armazenamento.

Como está armazenando dados binários, esse campo não pode ser usado em um filtro.

from django.db import models

class MyModel(models.Model):
    my_binary_data = models.BinaryField()
