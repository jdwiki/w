Um Queryset é fundamentalmente uma lista de objetos derivados de um Model, por uma compilação de consultas de banco de dados.

Reduzir o número de consultas no campo ForeignKey (problema n+1) #

Problema #

Os conjuntos de consultas do Django são avaliados de maneira preguiçosa. Por exemplo:

# models.py:
class Author(models.Model):
    name = models.CharField(max_length=100)

class Book(models.Model):
    author = models.ForeignKey(Author, related_name='books')
    title = models.CharField(max_length=100)

# views.py
def myview(request):
    # Query the database
    books = Book.objects.all()

    for book in books:
        # Query the database on each iteration to get author (len(books) times)
        # if there is 100 books, there will have 100 queries plus the initial query
        book.author
        # ...

    # total : 101 queries

O código acima faz com que o django consulte o banco de dados para o autor de cada livro. Isso é ineficiente e é melhor ter apenas uma única consulta.

Solução #

Use select_related em ForeignKey se você souber que precisará acessar posteriormente um campo ForeignKey.

# views.py
def myview(request):
    # Query the database.
    books = Books.objects.select_related('author').all()
    
    for book in books:
        # Does not query the database again, since `author` is pre-populated
        book.author
        # ...

    # total : 1 query

select_related também pode ser usado em campos de pesquisa:

# models.py:
class AuthorProfile(models.Model):
    city = models.CharField(max_length=100)

class Author(models.Model):
    name = models.CharField(max_length=100)
    profile = models.OneToOneField(AuthorProfile)

class Book(models.Model):
    author = models.ForeignKey(Author, related_name='books')
    title = models.CharField(max_length=100)    

# views.py
def myview(request):
    books = Book.objects.select_related('author')\
                        .select_related('author__profile').all()
   
    for book in books:
        # Does not query database
        book.author.name
        # or
        book.author.profile.city
        # ...

    # total : 1 query

Obtenha o conjunto de consultas SQL para Django #

O atributo query no queryset fornece a sintaxe equivalente do SQL para sua consulta.

>>> queryset = MyModel.objects.all()
>>> print(queryset.query)
SELECT "myapp_mymodel"."id", ... FROM "myapp_mymodel"

Aviso:

Esta saída deve ser usada apenas para fins de depuração. O gerado a consulta não é específica do back-end. Como tal, os parâmetros não são citados corretamente, deixando-o vulnerável à injeção de SQL, e a consulta pode nem mesmo ser executável em seu backend de banco de dados.

Consultas avançadas com objetos Q #

Dado o modelo:

class MyModel(models.Model):
    name = models.CharField(max_length=10)
    model_num = models.IntegerField()
    flag = models.NullBooleanField(default=False)

Podemos usar objetos Q para criar condições AND, OR em sua consulta de pesquisa. Por exemplo, digamos que queremos todos os objetos que tenham flag=True OU model_num>15.

from django.db.models import Q
MyModel.objects.filter(Q(flag=True) | Q(model_num__gt=15))

O acima se traduz em WHERE flag=True OR model_num > 15 da mesma forma para um AND que você faria.

MyModel.objects.filter(Q(flag=True) & Q(model_num__gt=15))

Objetos Q também nos permitem fazer consultas NÃO com o uso de ~. Digamos que queremos obter todos os objetos que têm flag=False AND model_num!=15, faríamos:

MyModel.objects.filter(Q(flag=True) & ~Q(model_num=15))

Se estiver usando objetos Q e parâmetros “normais” em filter(), então os objetos Q devem vir primeiro. A consulta a seguir procura modelos com (flag definido como True ou um número de modelo maior que 15) e um nome que começa com “H”.

from django.db.models import Q
MyModel.objects.filter(Q(flag=True) | Q(model_num__gt=15), name__startswith="H")

Observação: objetos Q podem ser usados ​​com qualquer função de pesquisa que receba argumentos de palavras-chave como filter, exclude, get. Certifique-se de que quando você usar com get você retornará apenas um objeto ou a exceção MultipleObjectsReturned será levantada.

Reduza o número de consultas em ManyToManyField (problema n+1) #

Problema #

# models.py:
class Library(models.Model):
    name = models.CharField(max_length=100)
    books = models.ManyToManyField(Book)

class Book(models.Model):
    title = models.CharField(max_length=100)

# views.py
def myview(request):
    # Query the database.
    libraries = Library.objects.all()

    # Query the database on each iteration (len(author) times)
    # if there is 100 librairies, there will have 100 queries plus the initial query
    for library in libraries:
        books = library.books.all()
        books[0].title
        # ...

    # total : 101 queries

Solução #

Use prefetch_related em ManyToManyField se você sabe que precisará acessar mais tarde um campo que é um campo ManyToManyField.

# views.py
def myview(request):
    # Query the database.
    libraries = Library.objects.prefetch_related('books').all()
    
    # Does not query the database again, since `books` is pre-populated
    for library in libraries:
        books = library.books.all()
        books[0].title
        # ...

    # total : 2 queries - 1 for libraries, 1 for books

prefetch_related também pode ser usado em campos de pesquisa:

# models.py:
class User(models.Model):
    name = models.CharField(max_length=100)

class Library(models.Model):
    name = models.CharField(max_length=100)
    books = models.ManyToManyField(Book)

class Book(models.Model):
    title = models.CharField(max_length=100)
    readers = models.ManyToManyField(User)

 # views.py
def myview(request):
    # Query the database.
    libraries = Library.objects.prefetch_related('books', 'books__readers').all()
    
    # Does not query the database again, since `books` and `readers` is pre-populated
    for library in libraries:
        for book in library.books.all():
            for user in book.readers.all():
                user.name
                # ...

    # total : 3 queries - 1 for libraries, 1 for books, 1 for readers

No entanto, uma vez que o queryset foi executado, os dados obtidos não podem ser alterados sem atingir novamente o banco de dados. O seguinte executaria consultas extras, por exemplo:

 # views.py
def myview(request):
    # Query the database.
    libraries = Library.objects.prefetch_related('books').all()
    for library in libraries:
        for book in library.books.filter(title__contains="Django"):
            print(book.name)

O seguinte pode ser otimizado usando um objeto Prefetch, introduzido no Django 1.7:

from django.db.models import Prefetch
# views.py
def myview(request):
    # Query the database.
    libraries = Library.objects.prefetch_related(
        Prefetch('books', queryset=Book.objects.filter(title__contains="Django")
    ).all()
    for library in libraries:
        for book in library.books.all():
            print(book.name)  # Will print only books containing Django for each library

Consultas simples em um modelo autônomo #

Aqui está um modelo simples que usaremos para executar algumas consultas de teste:

class MyModel(models.Model):
    name = models.CharField(max_length=10)
    model_num = models.IntegerField()
    flag = models.NullBooleanField(default=False)

Obtenha um único objeto de modelo em que o id/pk é 4:
(Se não houver itens com o id 4 ou houver mais de um, isso lançará uma exceção.)

MyModel.objects.get(pk=4)

Todos os objetos de modelo:

MyModel.objects.all()

Objetos de modelo que têm flag definido como True:

MyModel.objects.filter(flag=True)

Objetos de modelo com um model_num maior que 25:

MyModel.objects.filter(model_num__gt=25)

Objetos de modelo com o name de “Cheap Item” e flag definido como False:

MyModel.objects.filter(name="Cheap Item", flag=False)

Modelos de pesquisa simples name para string específica (diferencia maiúsculas de minúsculas):

MyModel.objects.filter(name__contains="ch")

Modelos de pesquisa simples name para string específica (não diferencia maiúsculas de minúsculas):

MyModel.objects.filter(name__icontains="ch")

Obtém o primeiro e o último registro do QuerySet #

Para obter o primeiro objeto:

MyModel.objects.first()

Para obter os últimos objetos:

MyModel.objects.last()

Usando o objeto Filter First:

MyModel.objects.filter(name='simple').first()

Usando Filtro Último objeto:

MyModel.objects.filter(name='simple').last()

Consultas avançadas com objetos F #

Um objeto F() representa o valor de um campo de modelo ou coluna anotada. Ele torna possível fazer referência a valores de campo de modelo e realizar operações de banco de dados usando-os sem realmente precisar extraí-los do banco de dados para a memória Python. - expressões F()

É apropriado usar objetos F() sempre que você precisar referenciar o valor de outro campo em sua consulta. Por si só, objetos F() não significam nada, e eles não podem e não devem ser chamados fora de um queryset. Eles são usados ​​para fazer referência ao valor de um campo no mesmo conjunto de consultas.

Por exemplo, dado um modelo …

SomeModel(models.Model):
    ...
    some_field = models.IntegerField()

… um usuário pode consultar objetos onde o valor de some_field é o dobro de seu id [referenciando o valor do campo id](https://docs.djangoproject.com/en/1.10/topics/db/queries /#filters-can-reference-fields-on-the-model) enquanto filtra usando F() assim:

SomeModel.objects.filter(some_field=F('id') * 2)

F('id') simplesmente referencia o valor id para essa mesma instância. O Django o usa para criar a instrução SQL correspondente. Neste caso algo parecido com isso:

SELECT * FROM some_app_some_model 
WHERE some_field = ((id * 2))

Sem expressões F() isso seria feito com SQL bruto ou filtragem em Python (o que reduz o desempenho especialmente quando há muitos objetos).

Referências:

  • Os filtros podem referenciar campos no modelo
  • expressões F
  • Resposta de TinyInstance
