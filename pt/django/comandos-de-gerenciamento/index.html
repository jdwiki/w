Comandos de gerenciamento são scripts poderosos e flexíveis que podem executar ações em seu projeto Django ou no banco de dados subjacente. Além de vários comandos padrão, é possível escrever o seu próprio!

Comparado aos scripts Python regulares, o uso da estrutura de comando de gerenciamento significa que algum trabalho de configuração tedioso é feito automaticamente para você nos bastidores.

Os comandos de gerenciamento podem ser chamados de:

  • django-admin <comando> [opções]
  • python -m django <comando> [opções]
  • python manage.py <comando> [opções]
  • ./manage.py <command> [options] se manage.py tiver permissões de execução (chmod +x manage.py)

Para usar comandos de gerenciamento com o Cron:

*/10 * * * * pythonuser /var/www/dev/env/bin/python /var/www/dev/manage.py <command> [options] > /dev/null

Criando e executando um comando de gerenciamento #

Para executar ações no Django usando linha de comando ou outros serviços (onde o usuário/requisição não é usado), você pode usar os comandos de gerenciamento.

Os módulos do Django podem ser importados conforme necessário.

Para cada comando, um arquivo separado precisa ser criado: myapp/management/commands/my_command.py
(Os diretórios management e commands devem ter um arquivo __init__.py vazio)

from django.core.management.base import BaseCommand, CommandError

# import additional classes/modules as needed
# from myapp.models import Book

class Command(BaseCommand):
    help = 'My custom django management command'

    def add_arguments(self, parser):
       parser.add_argument('book_id', nargs='+', type=int)
       parser.add_argument('author' , nargs='+', type=str)

    def handle(self, *args, **options):
       bookid = options['book_id'] 
       author = options['author']
       # Your code goes here
        
        # For example:
        # books = Book.objects.filter(author="bob")
        # for book in books:
        #    book.name = "Bob"
        #    book.save()

Aqui o nome da classe Command é obrigatório e estende BaseCommand ou uma de suas subclasses.

O nome do comando de gerenciamento é o nome do arquivo que o contém. Para executar o comando no exemplo acima, use o seguinte no diretório do seu projeto:

python manage.py my_command

Observe que iniciar um comando pode levar alguns segundos (por causa da importação dos módulos). Assim, em alguns casos, é aconselhável criar processos daemon em vez de comandos de gerenciamento.

Comandos de gerenciamento integrados #

O Django vem com vários comandos de gerenciamento embutidos, usando python manage.py [comando] ou, quando manage.py tem direitos +x (executáveis) simplesmente ./manage.py [comando] . A seguir estão alguns dos mais usados:

Obtenha uma lista de todos os comandos disponíveis

./manage.py help

Execute seu servidor Django em localhost:8000; essencial para testes locais

./manage.py runserver

Execute um console python (ou ipython se instalado) com as configurações do Django do seu projeto pré-carregadas (tentar acessar partes do seu projeto em um terminal python sem fazer isso falhará).

./manage.py shell

Crie um novo arquivo de migração de banco de dados com base nas alterações feitas em seus modelos. Veja Migrações

./manage.py makemigrations

Aplique quaisquer migrações não aplicadas ao banco de dados atual.

./manage.py migrate

Execute o conjunto de testes do seu projeto. Consulte Teste Unitário

./manage.py test

Pegue todos os arquivos estáticos do seu projeto e coloque-os na pasta especificada em STATIC_ROOT para que possam ser servidos em produção.

./manage.py collectstatic

Permitir criar superusuário.

./manage.py createsuperuser

Altere a senha de um usuário especificado.

./manage.py changepassword username

Usando django-admin em vez de manage.py #

Você pode se livrar do manage.py e usar o comando django-admin. Para fazer isso, você terá que fazer manualmente o que manage.py faz:

  • Adicione o caminho do seu projeto ao seu PYTHONPATH

  • Defina o DJANGO_SETTINGS_MODULE

    export PYTHONPATH="/home/me/path/to/your_project" export DJANGO_SETTINGS_MODULE=“your_project.settings”

Isso é especialmente útil em um virtualenv onde você pode definir essas variáveis ​​de ambiente no script postactivate.

1: https://www.wikiod.com/pt/django/comecando-com-o-django#Ambiente Virtual

O comando django-admin tem a vantagem de estar disponível onde quer que você esteja em seu sistema de arquivos.

Obter lista de comandos existentes #

Você pode obter a lista de comandos disponíveis da seguinte maneira:

>>> python manage.py help

Se você não entender nenhum comando ou procurar argumentos opcionais, poderá usar o argumento -h como este

>>> python manage.py command_name -h

Aqui command_name será o nome do comando desejado, isso mostrará o texto de ajuda do comando.
