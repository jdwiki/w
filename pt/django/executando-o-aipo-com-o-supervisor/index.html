Aipo + RabbitMQ com Supervisor #

O aipo requer um corretor para lidar com a passagem de mensagens. Usamos RabbitMQ porque é fácil de configurar e é bem suportado.

Instale o rabbitmq usando o seguinte comando

sudo apt-get install rabbitmq-server

Quando a instalação estiver concluída, crie um usuário, adicione um host virtual e defina as permissões.

sudo rabbitmqctl add_user myuser mypassword
sudo rabbitmqctl add_vhost myvhost
sudo rabbitmqctl set_user_tags myuser mytag
sudo rabbitmqctl set_permissions -p myvhost myuser ".*" ".*" ".*"

Para iniciar o servidor:

sudo rabbitmq-server

Podemos instalar o aipo com pip:

pip install celery

Em seu arquivo settings.py do Django, a URL do seu corretor seria algo como

BROKER_URL = 'amqp://myuser:mypassword@localhost:5672/myvhost'

Agora inicie o trabalhador de aipo

celery -A your_app worker -l info

Este comando inicia um trabalhador Celery para executar qualquer tarefa definida em seu aplicativo django.

Supervisor é um programa Python que permite controlar e continuar executando qualquer processo unix. Ele também pode reiniciar processos travados. Nós o usamos para garantir que os funcionários do Aipo estejam sempre funcionando.

Primeiro, instale o supervisor

sudo apt-get install supervisor

Crie o arquivo your_proj.conf em seu supervisor conf.d (/etc/supervisor/conf.d/your_proj.conf):

[program:your_proj_celery]
command=/home/your_user/your_proj/.venv/bin/celery --app=your_proj.celery:app worker -l info
directory=/home/your_user/your_proj
numprocs=1
stdout_logfile=/home/your_user/your_proj/logs/celery-worker.log
stderr_logfile=/home/your_user/your_proj/logs/low-worker.log
autostart=true
autorestart=true
startsecs=10

Uma vez que nosso arquivo de configuração é criado e salvo, podemos informar ao Supervisor sobre nosso novo programa através do comando supervisorctl. Primeiro, dizemos ao Supervisor para procurar qualquer configuração de programa nova ou alterada no diretório /etc/supervisor/conf.d com:

sudo supervisorctl reread

Seguido, informando-o para aprovar quaisquer alterações com:

sudo supervisorctl update

Quando nossos programas estiverem em execução, sem dúvida haverá um momento em que queremos parar, reiniciar ou ver seu status.

sudo supervisorctl status

Para reiniciar sua instância de aipo:

sudo supervisorctl restart your_proj_celery

Configuração de aipo #

SALSÃO #

  1. Instalação - pip install django-celery

  2. Adicionar

  3. Estrutura básica do projeto.

     - src/
   - bin/celery_worker_start # will be explained later on
   - logs/celery_worker.log
   - stack/__init __.py
   - stack/celery.py
   - stack/settings.py
   - stack/urls.py
   - manage.py

  4. Adicione o arquivo celery.py à sua pasta stack/stack/.

     from __future__ import absolute_import
 import os
 from celery import Celery
 os.environ.setdefault('DJANGO_SETTINGS_MODULE', 'stack.settings')
 from django.conf import settings  # noqa
 app = Celery('stack')
 app.config_from_object('django.conf:settings')
 app.autodiscover_tasks(lambda: settings.INSTALLED_APPS)

  5. ao seu stack/stack/__init__.py adicione o seguinte código:

     from __future__ import absolute_import
 from .celery import app as celery_app  # noqa

  6. Crie uma tarefa e marque-a, por exemplo, como @shared_task()

     @shared_task()
 def add(x, y):
     print("x*y={}".format(x*y))

  7. Executando o aipo “à mão”:

    celery -A stack worker -l info if you also want to add

Supervisor de corrida #

  1. Crie um script para iniciar o trabalhador de aipo. Insira seu script em seu aplicativo. Por exemplo: stack/bin/celery_worker_start

     #!/bin/bash

 NAME="StackOverflow Project - celery_worker_start"

 PROJECT_DIR=/home/stackoverflow/apps/proj/proj/
 ENV_DIR=/home/stackoverflow/apps/proj/env/

 echo "Starting $NAME as `whoami`"

 # Activate the virtual environment
 cd "${PROJECT_DIR}"

 if [ -d "${ENV_DIR}" ]
 then
     . "${ENV_DIR}bin/activate"
 fi

 celery -A stack --loglevel='INFO'

  2. Adicione direitos de execução ao seu script recém-criado:

    chmod u+x bin/celery_worker_start

  3. Instale o supervisor (pule este teste se o supervisor já estiver instalado)

    apt-get install supervisor

  4. Adicione o arquivo de configuração do seu supervisor para iniciar o aipo. Coloque-o em /etc/supervisor/conf.d/stack_supervisor.conf

     [program:stack-celery-worker]
 command = /home/stackoverflow/apps/stack/src/bin/celery_worker_start
 user = polsha
 stdout_logfile = /home/stackoverflow/apps/stack/src/logs/celery_worker.log
 redirect_stderr = true
 environment = LANG = en_US.UTF-8,LC_ALL = en_US.UTF-8
 numprocs = 1
 autostart = true
 autorestart = true
 startsecs = 10
 stopwaitsecs = 600
 priority = 998

  5. Reler e atualizar o supervisor

     sudo supervisorctl reread
     stack-celery-worker: available
 sudo supervisorctl update
     stack-celery-worker: added process group

  6. Comandos básicos

     sudo supervisorctl status stack-celery-worker                       
     stack-celery-worker      RUNNING    pid 18020, uptime 0:00:50
 sudo supervisorctl stop stack-celery-worker  
     stack-celery-worker: stopped
 sudo supervisorctl start stack-celery-worker                        
     stack-celery-worker: started
 sudo supervisorctl restart stack-celery-worker 
     stack-celery-worker: stopped
     stack-celery-worker: started
