Introdução #

As funções de janela são usadas para fazer operações (geralmente agregação) em um conjunto de linhas chamadas coletivamente de janela. As funções de janela funcionam no Spark 1.4 ou posterior. As funções de janela fornecem mais operações do que as funções internas ou UDFs, como substr ou round (amplamente usado antes do Spark 1.4). As funções de janela permitem que os usuários do Spark SQL calculem resultados como a classificação de uma determinada linha ou uma média móvel em um intervalo de linhas de entrada. Eles melhoram significativamente a expressividade das APIs SQL e DataFrame do Spark.

Em sua essência, uma função de janela calcula um valor de retorno para cada linha de entrada de uma tabela com base em um grupo de linhas, chamado Frame. Cada linha de entrada pode ter um quadro exclusivo associado a ela. Essa característica das funções de janela as torna mais poderosas do que outras funções. Os tipos de funções de janela são

  • Funções de classificação
  • Funções analíticas
  • Funções agregadas

Para usar funções de janela, os usuários precisam marcar que uma função é usada como função de janela por

  • Adicionando uma cláusula OVER após uma função suportada em SQL, por exemplo avg(receita) OVER (...); ou
  • Chamar o método over em uma função suportada na API DataFrame, por exemplo rank().over(...).

Esta documentação visa demonstrar algumas dessas funções com exemplos. Supõe-se que o leitor tenha algum conhecimento sobre operações básicas no Spark DataFrame como: adicionar uma nova coluna, renomear uma coluna etc.

Lendo um conjunto de dados de amostra:

val sampleData = Seq( ("bob","Developer",125000),("mark","Developer",108000),("carl","Tester",70000),("peter","Developer",185000),("jon","Tester",65000),("roman","Tester",82000),("simon","Developer",98000),("eric","Developer",144000),("carlos","Tester",75000),("henry","Developer",110000)).toDF("Name","Role","Salary")

Lista de declarações de importação obrigatórias:

import org.apache.spark.sql.functions._

Soma cumulativa #

Para calcular a média móvel do salário dos empregadores com base em sua função:

val cumSum = sampleData.withColumn("cumulativeSum", sum(sampleData("Salary"))

Funções da janela - Classificar, Lead, Lag, Rank, Análise de Tendência #

Este tópico demonstra como usar funções como withColumn, lead, lag, Level etc usando o Spark. O dataframe do Spark é uma camada abstrata sql nas funcionalidades do núcleo do Spark. Isso permite que o usuário escreva SQL em dados distribuídos. O Spark SQL suporta formatos de arquivo heterogêneos, incluindo JSON, XML, CSV, TSV etc.

Neste blog, temos uma visão geral rápida de como usar Spark SQL e dataframes para casos de uso comuns no mundo SQL. Por uma questão de simplicidade, vamos lidar com um único arquivo que é o formato CSV. O arquivo tem quatro campos, employeeID, employeeName, salário, salárioDate

Salve este arquivo como emp.dat. Na primeira etapa, criaremos um dataframe spark usando o pacote Spark CSV de databricks.

val sqlCont = new HiveContext(sc)

//Show dataframe

Obter e usar a linha anterior (Lag)

LAG é uma função em SQL que é usada para acessar valores de linha anteriores na linha atual. Isso é útil quando temos casos de uso como comparação com o valor anterior. O LAG nos dataframes do Spark está disponível nas funções do Windows

lag(Column e, int offset)
Window function: returns the value that is offset rows before the current row, and null if there is less than offset rows before the current row.


import org.apache.spark.sql.expressions.Window
//order by Salary Date to get previous salary.
//For first row we will get NULL
val window = Window.orderBy("SalaryDate")
//use lag to get previous row value for salary, 1 is the offset
val lagCol = lag(col("Salary"), 1).over(window)
myDF.withColumn("LagCol", lagCol).show()

Obtenha e use a próxima linha (Lead)

LEAD é uma função em SQL que é usada para acessar os valores da próxima linha na linha atual. Isso é útil quando temos casos de uso como comparação com o próximo valor. LEAD em dataframes Spark está disponível em funções de janela

lead(Column e, int offset)
Window function: returns the value that is offset rows after the current row, and null if there is less than offset rows after the current row.


import org.apache.spark.sql.expressions.Window
//order by Salary Date to get previous salary. F

val window = Window.orderBy("SalaryDate")
//use lag to get previous row value for salary, 1 is the offset
val leadCol = lead(col("Salary"), 1).over(window)

Análise de tendências com funções de janela Agora, vamos colocar a função de janela LAG para usar com uma análise de tendência simples. Se o salário for menor que o mês anterior, marcaremos como “DOWN”, se o salário aumentou, “UP”. O código usa a função Window para ordenar por, atrasar e, em seguida, fazer um simples if else com WHEN.

   val window = Window.orderBy("SalaryDate")
    //Derive lag column for salary
    val laggingCol = lag(col("Salary"), 1).over(trend_window)
    //Use derived column LastSalary to find difference between current and previous row
    val salaryDifference = col("Salary") - col("LastSalary")
    //Calculate trend based on the difference
    //IF ELSE / CASE can be written using when.otherwise in spark
    val trend = when(col("SalaryDiff").isNull || col("SalaryDiff").===(0), "SAME")
    .when(col("SalaryDiff").>(0), "UP")
    .otherwise("DOWN")
    myDF.withColumn("LastSalary", laggingCol)
    .withColumn("SalaryDiff",salaryDifference)
   .withColumn("Trend", trend).show()

Média móvel #

Para calcular a média móvel do salário dos empregadores com base em sua função:

val movAvg = sampleData.withColumn("movingAverage", avg(sampleData("Salary"))
             .over( Window.partitionBy("Role").rowsBetween(-1,1)) )

  • withColumn() cria uma nova coluna chamada movingAverage, executando average na coluna Salary

  • over() é usado para definir a especificação da janela.

  • partitionBy() particiona os dados sobre a coluna Role

  • rowsBetween(start, end) Esta função define as linhas que devem ser incluídas na janela. Os parâmetros (start e end) recebem entradas numéricas, 0 representa a linha atual, -1 é a linha anterior, 1 é a próxima linha e assim por diante. A função inclui todas as linhas entre start e end, portanto, neste exemplo, três linhas (-1,0,1) estão incluídas na janela.

      scala> movAvg.show

    +——+———+——+——————+ | Name| Role|Salary| movingAverage| +——+———+——+——————+ | bob|Developer|125000| 116500.0| | mark|Developer|108000|139333.33333333334| | peter|Developer|185000|130333.33333333333| | simon|Developer| 98000|142333.33333333334| | eric|Developer|144000|117333.33333333333| | henry|Developer|110000| 127000.0| | carl| Tester| 70000| 67500.0| | jon| Tester| 65000| 72333.33333333333| | roman| Tester| 82000| 74000.0| |carlos| Tester| 75000| 78500.0| +——+———+——+——————+

O Spark ignora automaticamente as linhas anteriores e seguintes, se a linha atual for a primeira e a última linha, respectivamente.

No exemplo acima, MovingAverage da primeira linha é apenas a média da linha atual e da próxima linha, pois a linha anterior não existe. Da mesma forma, a última linha da partição (ou seja, 6ª linha) é a média da linha atual e anterior, pois a próxima linha não existe.
