Instalando o Theano e configurando a GPU no Ubuntu 14.04 #

Você pode usar as seguintes instruções para instalar o Theano e configurar a GPU (suponha um Ubuntu 14.04 recém-instalado):

# Install Theano
sudo apt-get install python-numpy python-scipy python-dev python-pip python-nose g++ libopenblas-dev git
sudo pip install Theano

# Install Nvidia drivers, CUDA and CUDA toolkit, following some instructions from http://docs.nvidia.com/cuda/cuda-installation-guide-linux/index.html
wget http://developer.download.nvidia.com/compute/cuda/7.5/Prod/local_installers/cuda-repo-ubuntu1404-7-5-local_7.5-18_amd64.deb # Got the link at https://developer.nvidia.com/cuda-downloads
sudo dpkg -i cuda-repo-ubuntu1404-7-5-local_7.5-18_amd64.deb
sudo apt-get update
sudo apt-get install cuda

sudo reboot

Nesse ponto, executar nvidia-smi deve funcionar, mas executar nvcc não funcionará.

# Execute in console, or (add in ~/.bash_profile then run "source ~/.bash_profile"):
export PATH=/usr/local/cuda-7.5/bin:$PATH
export LD_LIBRARY_PATH=/usr/local/cuda-7.5/lib64:$LD_LIBRARY_PATH

Nesse ponto, tanto nvidia-smi quanto nvcc devem funcionar.

Para testar se Theano é capaz de usar a GPU:

Copie e cole o seguinte em gpu_test.py:

# Start gpu_test.py
# From http://deeplearning.net/software/theano/tutorial/using_gpu.html#using-gpu
from theano import function, config, shared, sandbox
import theano.tensor as T
import numpy
import time

vlen = 10 * 30 * 768  # 10 x #cores x # threads per core
iters = 1000

rng = numpy.random.RandomState(22)
x = shared(numpy.asarray(rng.rand(vlen), config.floatX))
f = function([], T.exp(x))
print(f.maker.fgraph.toposort())
t0 = time.time()
for i in xrange(iters):
    r = f()
t1 = time.time()
print("Looping %d times took %f seconds" % (iters, t1 - t0))
print("Result is %s" % (r,))
if numpy.any([isinstance(x.op, T.Elemwise) for x in f.maker.fgraph.toposort()]):
    print('Used the cpu')
else:
    print('Used the gpu')
# End gpu_test.py

e execute-o:

THEANO_FLAGS='mode=FAST_RUN,device=gpu,floatX=float32' python gpu_test.py

que deve retornar:

f@f-Aurora-R4:~$ THEANO_FLAGS='mode=FAST_RUN,device=gpu,floatX=float32' python gpu_test.py
Using gpu device 0: GeForce GTX 690
[GpuElemwise{exp,no_inplace}(<CudaNdarrayType(float32, vector)>), HostFromGpu(GpuElemwise{exp,no_inplace}.0)]
Looping 1000 times took 0.658292 seconds
Result is [ 1.23178029  1.61879349  1.52278066 ...,  2.20771813  2.29967761
  1.62323296]
Used the gpu

Adicionando cuDNN #

Para adicionar cuDNN (instruções de http://deeplearning.net/software/theano/library/sandbox/cuda/dnn.html):

  1. Faça o download do cuDNN em https://developer.nvidia.com/rdp/cudnn-download (precisa de registro, que é gratuito)
  2. tar -xvf cudnn-7.0-linux-x64-v3.0-prod.tgz
  3. Faça um dos seguintes

Opção 1: Copie os arquivos *.h para CUDA_ROOT/include e os arquivos *.so* para CUDA_ROOT/lib64 (por padrão, CUDA_ROOT é /usr/local/cuda no Linux ).

sudo cp cuda/lib64/* /usr/local/cuda/lib64/
sudo cp cuda/include/cudnn.h /usr/local/cuda/include/

Opção 2:

export LD_LIBRARY_PATH=/home/user/path_to_CUDNN_folder/lib64:$LD_LIBRARY_PATH
export CPATH=/home/user/path_to_CUDNN_folder/include:$CPATH
export LIBRARY_PATH=/home/user/path_to_CUDNN_folder/lib64:$LD_LIBRARY_PATH

Por padrão, o Theano detectará se pode usar cuDNN. Se assim for, ele vai usá-lo. Caso contrário, as otimizações Theano não introduzirão operações cuDNN. Portanto, o Theano ainda funcionará se o usuário não os apresentar manualmente.

Para obter um erro se o Theano não puder usar o cuDNN, use este sinalizador do Theano: optimizer_inclusive=cudnn.

Exemplo:

THEANO_FLAGS='mode=FAST_RUN,device=gpu,floatX=float32,optimizer_including=cudnn' python gpu_test.py

Para saber sua versão cuDNN:

cat /usr/local/cuda/include/cudnn.h | grep CUDNN_MAJOR -A 2

Adicionando CNMeM #

A biblioteca CNMeM é uma “biblioteca simples para ajudar as estruturas de Deep Learning a gerenciar a memória CUDA”.

# Build CNMeM without the unit tests
git clone https://github.com/NVIDIA/cnmem.git cnmem
cd cnmem
mkdir build
cd build
sudo apt-get install -y cmake
cmake ..
make

# Copy files to proper location
sudo cp ../include/cnmem.h /usr/local/cuda/include
sudo cp *.so /usr/local/cuda/lib64/
cd ../..

Para usar com Theano, você precisa adicionar o sinalizador lib.cnmem. Exemplo:

THEANO_FLAGS='mode=FAST_RUN,device=gpu,floatX=float32,lib.cnmem=0.8,optimizer_including=cudnn' python gpu_test.py

A primeira saída do script deve ser:

Using gpu device 0: GeForce GTX TITAN X (CNMeM is enabled with initial size: 80.0% of memory, cuDNN 5005)

lib.cnmem=0.8 significa que pode usar até 80% da GPU.

Foi relatado que o CNMeM fornece algumas melhorias de velocidade interessantes e é suportado por Theano, Torch e Caffee.

Theano - fonte 1:

A aceleração depende de vários fatores, como as formas e o próprio modelo. A aceleração vai de 0 a 2x mais rápido.

Theano - fonte 2:

Se você não alterar o sinalizador Theano allow_gc, você pode esperar 20% de velocidade na GPU. Em alguns casos (modelos pequenos), vimos uma aceleração de 50%.

Problemas comuns:

  • Importando theano: AttributeError: objeto ‘module’ não tem atributo ‘find_graphviz’

Instalação ou configuração #

Instruções detalhadas sobre como configurar ou instalar o theano.

Executando Theano em vários núcleos de CPU #

Você pode executar o Theano em vários núcleos de CPU com o sinalizador OMP_NUM_THREADS=[number_of_cpu_cores].

Exemplo:

OMP_NUM_THREADS=4 python gpu_test.py

O script theano/misc/check_blas.py gera informações sobre qual BLAS é usado:

cd [theano_git_directory]
OMP_NUM_THREADS=4 python theano/misc/check_blas.py

Seu primeiro programa theano #

Neste exemplo, vamos compilar funções que calculam soma e diferença dados dois números reais.

from __future__ import print_function
import theano
import theano.tensor as T

#define two symbolic scalar
s_x = T.fscalar()
s_y = T.fscalar()

#compute something
s_sum = s_x + s_y
s_diff = s_x - s_y

#compile a function that adds two number
#theano will call system compiler at here
fn_add = theano.function(inputs=[s_x, s_y], outputs=s_sum)
fn_diff = theano.function(inputs=[s_x, s_y], outputs=s_diff)

#call the compiled functions
print(fn_add(2., 2.)) #4.
print(fn_diff(2., 2.)) #0.
