O que é Navalha?

Razor é uma sintaxe de marcação que permite incorporar código baseado em servidor (Visual Basic e C#) em páginas da Web.

O código baseado em servidor pode criar conteúdo dinâmico da Web em tempo real, enquanto uma página da Web é gravada no navegador. Quando uma página da Web é chamada, o servidor executa o código baseado no servidor dentro da página antes de retornar a página ao navegador. Ao ser executado no servidor, o código pode realizar tarefas complexas, como acessar bancos de dados.

Sintaxe #

  • @{ … }
  • @nome variável
  • @(nome variável)
  • @por(…){ }
  • @(Expressão explícita)
  • @* comentários *@

O ASP.NET Razor inclui mecanismos de exibição para C# e VB.

O mecanismo de visualização C# processa arquivos com uma extensão .cshtml, enquanto o mecanismo de visualização VB trabalha com arquivos .vbhtml.

Sintaxe Básica #

O código Razor pode ser inserido em qualquer lugar dentro do código HTML. Os blocos de código Razor são incluídos em @{ ... }. Variáveis ​​e funções inline começam com @. O código dentro dos colchetes Razor segue as regras normais de C# ou VB.

Declaração de linha única:

@{ var firstNumber = 1; }

Bloco de código de várias linhas:

@{
    var secondNumber = 2;
    var total = firstNumber + secondNumber;
}

Usando uma variável inline:

<h1>The total count is @total</h1>

Usando uma variável inline explicitamente:

<h2>Item@(item.Id)</h2>

Para este exemplo em particular, não poderemos usar a sintaxe implícita porque Item@item.Id se parece com um email e será renderizado como tal pelo Razor.

Inclua o código dentro das instruções de fluxo de controle:

<h1>Start with some HTML code</h1>

@for (int i = 0; i < total; i++){
    Console.Write(i);
}

<p>Mix in some HTML code for fun!</p>
<p>Add a second paragraph.</p>

@if (total > 3)
{
    Console.Write("The total is greater than 3");
}
else
{
    Console.Write("The total is less than 3");
}

Esta mesma sintaxe seria usada para todas as instruções como for, foreach, while, if, switch, etc.

Adicionando código dentro do código:

@if (total > 3)
{
    if(total == 10)
    {
        Console.Write("The total is 10")
    }
}

Observe que você não precisa digitar o @ no segundo if. Após o código, você pode simplesmente digitar outro código por trás do código existente.

Se você quiser adicionar código após um elemento HTML você precisa digitar um @.

Adicione comentários #

Razor tem sua própria sintaxe de comentário que começa com @* e termina com *@.

Comentário embutido:

<h1>Comments can be @*hi!*@ inline</h1>

Comentário de várias linhas:

@* Comments can spread
   over multiple
   lines *@

Comentário HTML

Você também pode usar a sintaxe de comentário HTML normal começando com <!-- e terminando com --> nas visualizações do Razor. Mas, ao contrário de outros comentários, o código Razor dentro de um comentário HTML ainda é executado normalmente.

@{
    var hello = "Hello World!"; 
} 
<!-- @hello -->

O exemplo acima produz a seguinte saída HTML:

<!-- Hello World! -->

Comentários em um bloco de código:

@{
    // This is a comment
    var Input = "test";
}

Adicionando um atributo personalizado com - (hífen) no nome #

Se você precisar adicionar um atributo através do razor que tenha um - (hífen) no nome, você não pode simplesmente fazer

@Html.DropDownListFor(m => m.Id, Model.Values, new { @data-placeholder = "whatever" })

ele não irá compilar. Os atributos data-* são válidos e comuns em html5 para adicionar valores extras aos elementos.

No entanto, o seguinte funcionará

@Html.DropDownListFor(m => m.Id, Model.Values, new { @data_placeholder = "whatever" })

já que “_” é substituído por “-” quando renderizado.

Isso funciona bem, pois sublinhados não são aceitáveis ​​em nomes de atributos em html.

Crie classes e métodos inline usando @functions #

O uso da palavra-chave Razor @functions oferece a capacidade de introduzir classes e métodos para uso embutido em um arquivo Razor:

@functions
{
    string GetCssClass(Status status)
    {
        switch (status)
        {
            case Status.Success:
                return "alert-success";
            case Status.Info:
                return "alert-info";
            case Status.Warning:
                return "alert-warning";
            case Status.Danger:
            default:
                return "alert-danger";
         }
     }
}


<label class="alert @GetCssClass(status)"></label>

O mesmo pode ser feito para as classes:

@functions
{
    class Helpers
    {
        //implementation
    }
}

Exibir HTML no bloco de código Razor #

Enquanto estiver dentro de um bloco de código Razor, o navegador só reconhecerá o código HTML se o código for escapado.

Use @: para uma única linha:

@foreach(int number in Model.Numbers)
{
    @:<h1>Hello, I am a header!</h1>
}

Use <text> ... </text> para várias linhas:

@{
    var number = 1;

    <text>
        Hello, I am text
        <br / >
        Hello, I am more text!
    </text>
}

Observe que o Razor, quando dentro de um bloco de código, entenderá as tags HTML. Portanto, adicionar a tag text em torno das tags HTML é desnecessário (embora ainda correto), como:

@{
    var number = 1;
    <text>
        <div>
            Hello, I am text
            <br / >
            Hello, I am more text!
        </div>
    </text>
}

Escapando @ personagem #

Em muitos casos, o analisador Razor é inteligente o suficiente para descobrir quando o sinal @ deve ser usado como parte do código, em vez de fazer parte de algo como um endereço de e-mail. No exemplo abaixo, não é necessário escapar do sinal @:

<p>Reach out to us at contact@mail.com</p>

No entanto, em alguns casos, o uso do sinal @ é mais ambíguo e deve ser explicitamente escapado com @@, como no exemplo abaixo:

<p>Join us @@ Stack Overflow!</p>

Alternativamente, podemos usar um caractere @ codificado em HTML

<p>Join us &#64; Stack Overflow!</p>

Compartilhe @helpers nas visualizações #

@Helpers podem ser compartilhados entre visualizações.

Eles devem ser criados na pasta App_Code

@helper CreatePrimaryBootstrapButton(string label)
{
    <button type="button" class="btn btn-primary">@label</button>
}

//call

@MenuHelpers.CreatePrimaryBootstrapButton("my button")

Os globais @Url e @Html não estão disponíveis por padrão no @Helper definido em App_code. Você pode adicioná-los da seguinte forma (para cada .cshtml em sua pasta App_code)

@*  Make @Html and @Url available *@
@functions
{
    private new static HtmlHelper<object> Html
    {
        get { return ((WebViewPage)CurrentPage).Html; }
    }

    private static UrlHelper Url
    {
        get { return ((WebViewPage)CurrentPage).Url; }
    }
 }

Modelos de edição #

Os modelos de editor são uma boa maneira de reutilizar o código Razor. Você pode definir modelos de editor como exibições parciais do Razor e usá-los em outras exibições.

Os modelos de editor geralmente existem na pasta Views/Shared/EditorTemplates/, embora também possam ser salvos na pasta Views/ControllerName/EditorTemplates/. O nome da visualização normalmente é o nome do objeto para o qual você deseja usar o modelo, como <tipo>.cshtml.

Aqui está um modelo de editor simples para DateTime:

@model DateTime
<div>
   <span>
      @Html.TextBox("", Model.ToShortDateString(), new { data_date_picker="true" })
   </span>
</div>

Salve o arquivo como Views/Shared/EditorTemplate/DateTime.cshtml.

Em seguida, use EditorFor para chamar este código de modelo em outra visualização:

@Html.EditorFor(m => m.CreatedDate)

Há também um atributo UIHint para especificar o nome do arquivo:

public class UiHintExampleClass
{
    [UIHint("PhoneNumber")]
    public string Phone { get; set; }
}

Defina este modelo de número de telefone em Views/Shared/EditorTemplates/PhoneNumber.cshtml.

Os modelos do editor também podem ser definidos para tipos personalizados.

Aqui está um tipo personalizado chamado SubModel:

public class SubModel
{
    public Guid Id { get; set;} 
    public string FirstName { get; set; }
    public string LastName { get; set; }
}

public class Model
{
    public Guid Id { get; set; }
    public DateTime Created {get; set; }
    
    public SubModel SubModel{get; set; }
}

Este é o EditorTemplate para SubModel:

@model SubModel
<div class="form-group">
    @Html.LabelFor(m => m.FirstName)
    @Html.TextBoxFor(m => m.FirstName)
</div>
<div class="form-group">
    @Html.LabelFor(m => m.LastName)
    @Html.TextBoxFor(m => m.LastName)
</div>

Agora, a View for Model simplesmente se torna:

@model Model
@Html.EditorFor(m => m.CreatedDate)
@Html.EditorFor(m => m.SubModel, new { @Prefix = "New"}) 
@* the second argument is how you can pass viewdata to your editor template*@

Passe o conteúdo do Razor para um @helper #

Envie uma parte do Razor para um @helper, por exemplo, um div HTML.

@helper WrapInBox(Func<Object, HelperResult> content)
{
    <div class="box">@content(null) </div>
}

//call 
@WrapInBox(@<div>
                I'm a inner div
            </div>)
