Abrindo um prompt de comando #

O prompt de comando vem pré-instalado em todos os sistemas operacionais Windows NT, Windows CE, OS/2 e eComStation, e existe como cmd.exe, normalmente localizado em C:\Windows\system32\cmd.exe

No Windows 7, as maneiras mais rápidas de abrir o prompt de comando são:

  • Pressione , digite “cmd” e pressione Enter.

  • Pressione +R, digite “cmd” e então pressione Enter.

Ele também pode ser aberto navegando até o executável e clicando duas vezes nele.

Em alguns casos, você pode precisar executar cmd com permissões elevadas, neste caso clique com o botão direito e selecione “Executar como administrador”. Isso também pode ser feito pressionando Control+ Shift+Enter em vez de Enter.

Navegando no cmd #

Uma das coisas mais comuns que você precisa fazer no prompt de comando é navegar em seu sistema de arquivos. Para fazer isso, utilizaremos as palavras-chave cd e dir. Comece abrindo um prompt de comando usando um dos métodos mencionados [aqui](https://www.wikiod.com/pt/cmd/introducao-ao-cmd#Abrindo um prompt de comando). Você provavelmente verá algo semelhante ao que está abaixo, onde UserName é seu usuário.

C:\Users\UserName>

Independentemente de onde você esteja na sua estrutura de arquivos, se o seu sistema for como a maioria, podemos começar com este comando:

cd C:\

Isso mudará seu diretório atual para a unidade C:\. Observe como a tela agora se parece com isso

C:\>

Em seguida, execute um dir para que possamos ver qualquer coisa na unidade C:\

dir

Isso mostrará uma lista de arquivos e pastas com algumas informações sobre eles, semelhante a esta:

Há muitas informações boas aqui, mas para navegação básica, nos preocupamos apenas com a coluna mais à direita. Observe como temos uma pasta Users. Isso significa que podemos executar isso

cd Users

Agora se você executar dir novamente, você verá todos os arquivos e pastas em seu diretório C:\Users. Agora, não encontramos o que queríamos aqui, então vamos voltar para a pasta pai. Em vez de digitar o caminho para ele, podemos usar .. para subir uma pasta assim

cd ..

Agora estamos de volta em C:\. Se você quiser subir várias pastas de uma vez, você pode colocar uma barra invertida e outro conjunto de pontos como: cd ..\.., mas só precisamos de uma pasta.

Agora queremos olhar na pasta Program Files. Para evitar confundir o sistema, é uma boa ideia colocar aspas nos diretórios, especialmente quando houver espaços no nome. Desta vez, usaremos este comando

C:\>cd "Program Files"

Agora você está em C:\Program Files> e um comando dir agora lhe dirá qualquer coisa que esteja aqui.

Então, digamos que nos cansamos de vagar por aí para encontrar a pasta e procuramos exatamente onde precisávamos ir. Acontece que é C:\Windows\Logs Em vez de fazer um .. para Windows para Logs, podemos simplesmente colocar o caminho completo assim:

cd "C:\Windows\Logs"

E isso é o básico para navegar no prompt de comando. Agora você pode percorrer todas as suas pastas para poder executar seus outros comandos nos locais apropriados.

Características #

Microsoft Command Prompt é um intérprete de linha de comando (CLI) para os sistemas operacionais Windows.

Uma CLI é um programa destinado principalmente a ler as instruções do sistema operacional digitadas em um teclado pelo usuário. É, portanto, endereçada também como uma interface de linha de comando, para contrastá-la com as interfaces gráficas.

Como essas interfaces (textuais ou gráficas) protegem o usuário de acessar diretamente o kernel do sistema operacional, elas também são chamadas de shells.

Dado o nome do arquivo executável do Prompt de Comando, cmd.exe, o Prompt de Comando é amigável chamado cmd. Dado o seu papel de piloto do sistema operacional, também se diz o console.

Como outros shells, o cmd pode ler um lote de instruções de um arquivo. Neste caso, o shell cmd atua como um interpretador de linguagem e o conteúdo do arquivo pode ser considerado como um programa real. Ao executar esses programas em lote, não há fase de compilação intermediária. Eles são normalmente lidos, interpretados e executados linha por linha. Como não há compilação, não há produção de um arquivo executável separado. Por esta razão os programas são denominados batch scripts ou shell scripts.

Observe que as instruções inseridas interativamente podem ter uma sintaxe ligeiramente diferente daquelas enviadas como script, mas o princípio geral é que o que pode ser inserido na linha de comando também pode ser colocado em um arquivo para reutilização posterior.

Olá Mundo #

Os scripts em lote do Prompt de Comando têm extensão .cmd ou .bat, este último por motivos de compatibilidade.

Para criar um script de palavra-olá, primeiro você precisa de um local para digitá-lo. Para scripts simples, o Bloco de Notas do Windows também serve. Se você leva a sério os scripts de shell, precisa de ferramentas mais eficazes. De qualquer forma, existem várias alternativas gratuitas, como Notepad++.

No tipo de editor designado:

echo Hello World
pause

Salve-o como hello.cmd

Se você estiver usando o “Bloco de Notas” como editor, você deve prestar muita atenção ao nome salvo, pois o Bloco de Notas tende a adicionar sempre uma extensão .txt aos seus arquivos, o que significa que o nome real do seu arquivo pode ser hello .cmd.txt. Para evitar isso, na caixa de diálogo salvar:

  1. No campo “Nome do arquivo”, insira o nome entre aspas duplas, por exemplo, "olá.cmd"
  2. No campo Salvar como tipo selecione Todos os Arquivos, em vez da opção Documento de Texto padrão.

Se o arquivo foi salvo corretamente, seu ícone deve ser semelhante a (Windows Vista):

Você também pode considerar desabilitar a opção “Ocultar extensão para tipos de arquivo conhecidos” nas opções de exibição de pasta do Explorador de Arquivos. Nesse caso, os nomes dos arquivos são sempre exibidos com suas extensões.

Para executar hello.cmd existem duas possibilidades. Se você estiver usando o shell gráfico do Windows, basta clicar duas vezes em seu ícone.

Se você quiser usar o Prompt de Comando, você deve primeiro identificar o diretório onde você salvou o hello.cmd. Nesse sentido, se você abrir o Explorador de Arquivos com +E. Nos arquivos de listagem do Windows, você normalmente lê o nome do caminho do diretório que os contém. Você pode, portanto, identificar o diretório de hello.cmd. Os nomes de diretório do Windows tendem a ser bastante longos e digitá-los é propenso a erros. É melhor se você selecionar e copiar o caminho do diretório na área de transferência para colar posteriormente.

Inicie o prompt de comando. Você lê uma linha semelhante a esta.

Microsoft Windows [Version ...]
(c) ... Microsoft Corporation. All rights reserved.
 
C:\Users\...>

A versão/ano do Windows, claro, depende da sua. Na linha final, antes de >, você lê o caminho do diretório atual. Você deve tornar atual o diretório onde está seu script. Por esta razão, digite o comando change directory cd, usando uma linha semelhante à seguinte:

cd <dirpath>

Em vez de <dirpath>, cole o nome do diretório que você copiou anteriormente. Para colar o caminho do diretório, no Windows 10, basta digitar Ctrl-C, como faria em um editor. Para sistemas mais antigos, você poderá fazer isso clicando com o botão direito do mouse na janela cmd. Depois de digitar o comando, observe que o caminho atual, antes de >, muda de acordo.

Agora você pode executar seu script hello simplesmente digitando:

hello

Comentários #

O script imprime uma saída semelhante a:

C:\Users\...>echo Hello World
Hello World

C:\Users\...>pause
Press any key to continue . . .

As linhas que hospedam o símbolo > reafirmam as instruções do script como se você tivesse digitado interativamente. Isso pode ser desativado escrevendo:

@echo off

como a primeira linha do seu script. Isso pode reduzir a confusão, mas você tem menos dicas sobre o que está acontecendo, com relação aos comandos de script que não fornecem saídas visíveis.

O último comando, pause, solicita que você pressione qualquer tecla. Ao fazer isso, você sai do hello. Se você executar hello a partir do console, você realmente não precisa dele, porque, quando hello termina sua execução, cmd.exe permanece aberto e você pode ler a saída hello. Ao clicar duas vezes no Explorer, você inicia o cmd.exe pelo tempo necessário para executar o hello. Quando hello termina, cmd.exe faz o mesmo e você não tem a possibilidade de ler a saída hello. O comando pause impede que hello saia até que você pressione uma tecla, o que também dá a possibilidade de ler a saída.

Por fim, apesar do nome do script ser hello.cmd, não é necessário digitar o nome inteiro, seu radical hello é suficiente. Este mecanismo também funciona para executáveis, com extensão .exe. E se houver um script hello.cmd e um executável hello.exe no mesmo diretório? O primeiro tem prioridade no Prompt de Comando, então hello.cmd será executado.
