Este tópico explica como o mecanismo WinForms funciona para exibir formulários e como você controla seus tempos de vida.

Fechando um formulário modal #

Quando um formulário é mostrado usando o método ShowDialog, é necessário definir a propriedade DialogResult do formulário para fechar o formulário. Essa propriedade pode ser definida usando o enum que também é chamado de DialogResult.

Para fechar um formulário, você só precisa definir a propriedade DialogResult do formulário (para qualquer valor de DialogResult.None) em algum manipulador de eventos. Quando seu código sair do manipulador de eventos, o mecanismo WinForm ocultará o formulário e o código que segue a chamada inicial do método ShowDialog continuará a execução.

private cmdClose_Click(object sender, EventArgs e)
{
    this.DialogResult = DialogResult.Cancel;
}

O código de chamada pode capturar o valor de retorno de ShowDialog para determinar em qual botão o usuário clicou no formulário. Quando exibido usando ShowDialog(), o formulário não é descartado automaticamente (já que foi simplesmente oculto e não fechado), então é importante usar um bloco using para garantir que o formulário seja descartado.

Abaixo está um exemplo de verificação do resultado do uso do OpenFileDialog integrado, verificando o resultado e acessando uma propriedade da caixa de diálogo antes de descartá-la.

using (var form = new OpenFileDialog())
{
    DialogResult result = form.ShowDialog();
    if (result == DialogResult.OK)
    {
        MessageBox.Show("Selected file is: " + form.FileName);
    }
}

Você também pode definir a propriedade DialogResult em um botão. Clicar nesse botão definirá a propriedade DialogResult no formulário para o valor associado ao botão. Isso permite fechar o formulário sem adicionar um manipulador de eventos para definir o DialogResult no código.

Por exemplo, se você adicionar um botão OK ao seu formulário e definir sua propriedade como DialogResult.OK, o formulário será fechado automaticamente quando você pressionar esse botão e o código de chamada receberá um DialogResult.OK em retorno do ShowDialog( ) chamada de método.

Mostra um formulário sem janela restrita ou modal #

Depois de definir a estrutura do seu formulário com o designer do WinForms, você pode exibir seus formulários em código com dois métodos diferentes.

  • Método - Um formulário sem janela restrita

      Form1 aForm1Instance = new Form1(); 
  aForm1Instance.Show();

  • Método - Um Diálogo Modal

      Form2 aForm2Instance = new Form2(); 
  aForm2Instance.ShowDialog();

Os dois métodos têm uma distinção muito importante. O primeiro método (o sem janela restrita) mostra seu formulário e retorna imediatamente sem esperar o fechamento do formulário recém-aberto. Portanto, seu código continua com o que segue a chamada Show. O segundo método (o modal) abre o formulário e bloqueia qualquer atividade em todo o aplicativo até que você feche o formulário através do botão Fechar ou com alguns botões configurados adequadamente para fechar o formulário

Fechando um formulário sem janela restrita #

Um formulário sem janela restrita é empregado (geralmente) quando você precisa mostrar algo permanentemente ao lado da tela principal do aplicativo (pense em uma legenda ou uma exibição em um fluxo de dados vindo de forma assíncrona de um dispositivo ou uma janela filho MDI). Mas um formulário sem janela restrita representa um desafio único quando você deseja fechá-lo. Como recuperar a instância e chamar o método Close nessa instância?

Você pode manter uma variável global referenciando a instância que deseja fechar.

theGlobalInstance.Close();
theGlobalInstance.Dispose();
theGlobalInstance = null;

Mas também podemos optar por usar a coleção Application.OpenForms onde o mecanismo de formulário armazena todas as instâncias de formulário criadas e ainda abertas.

Você pode recuperar essa instância específica desta coleção e chamar o método Close

Form2 toClose = Application.OpenForms.OfType<Form2>().FirstOrDefault();
if(toClose != null)
{
    toClose.Close();
    toClose.Dispose();
}
