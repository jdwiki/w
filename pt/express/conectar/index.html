Conecte-se e Expresse #

O Express é baseado no Connect, que é o que fornece a funcionalidade de middleware do Express. Para entender o que é conectar, você pode ver que ele fornece a estrutura básica do aplicativo que você usa quando usa o express

const connect = require('connect')

const app = connect()
app.listen(3000)

Isso abrirá um servidor http “vazio” que responderá 404 a todas as solicitações.

Middleware #

O middleware é anexado ao objeto do aplicativo, geralmente antes que o listen seja chamado. Exemplo de um middleware de log simples:

app.use(function (req, res, next) {
    console.log(`${req.method}: ${req.url}`)
    next()
})

Tudo isso fará é logar GET: /example se você onde GET localhost:3000/example. Todas as solicitações ainda retornarão 404, pois você não está respondendo com nenhum dado.

O próximo middleware na cadeia será executado assim que o anterior chamar next(), então podemos ir em frente e responder às solicitações adicionando outro middleware como este:

app.use(function (req, res, next) {
    res.end(`You requested ${req.url}`)
})

Agora, quando você solicitar ´localhost:3000/example, você será saudado com "Você solicitou /example". Não há necessidade de chamar next` desta vez, pois este middleware é o último da cadeia (mas nada de ruim acontecerá se você o fizer),

Programa completo até aqui:

const connect = require('connect')

const app = connect()

app.use(function (req, res, next) {
    console.log(`${req.method}: ${req.url}`)
    next()
})

app.use(function (req, res, next) {
    res.end(`You requested ${req.url}`)
    next()
})

app.listen(3000)

Erros e middleware de erro #

Se quisermos limitar o acesso ao nosso aplicativo, podemos escrever um middleware para isso também! Este exemplo só concede acesso às quintas-feiras, mas um exemplo do mundo real pode ser, por exemplo, autenticação de usuário. Um bom lugar para colocar isso seria após o middleware de log, mas antes de qualquer conteúdo ser enviado.

app.use(function (req, res, next) {
    if (new Date().getDay() !== 4) {
        next('Access is only granted on thursdays')
    } else {
        next()
    }
})

Como você pode ver neste exemplo, enviar um erro é tão fácil quanto fornecer um parâmetro para a função next().

Agora, se visitarmos o site em qualquer dia diferente de uma quinta-feira, seremos recebidos com um erro 500 e a string 'Acesso é concedido apenas às quintas'.

Agora, isso não é bom o suficiente para o nosso site. Preferimos enviar ao usuário uma mensagem HTML em outro middleware:

app.use(function (err, req, res, next) {
    res.end(`<h1>Error</h1><p>${err}</p>`)
})

Isso funciona como um bloco catch: qualquer erro no middleware anterior ao middleware de erro será enviado para o primeiro. Um middleware de erro é identificado por seus 4 parâmetros.

Você também pode usar o middleware de erro para se recuperar do erro chamando o próximo método novamente:

app.use(function (err, req, res, next) {
    // Just joking, everybody is allowed access to the website!
    next()
})
