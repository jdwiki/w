Funções anônimas #

No Elixir, uma prática comum é usar funções anônimas. Criar uma função anônima é simples:

iex(1)> my_func = fn x -> x * 2 end
#Function<6.52032458/1 in :erl_eval.expr/5>

A sintaxe geral é:

fn args -> output end

Para facilitar a leitura, você pode colocar parênteses nos argumentos:

iex(2)> my_func = fn (x, y) -> x*y end
#Function<12.52032458/2 in :erl_eval.expr/5>

Para invocar uma função anônima, chame-a pelo nome atribuído e adicione . entre o nome e os argumentos.

iex(3)>my_func.(7, 5)
35

É possível declarar funções anônimas sem argumentos:

iex(4)> my_func2 = fn -> IO.puts "hello there" end
iex(5)> my_func2.()
hello there
:ok

Usando o operador de captura #

Para tornar as funções anônimas mais concisas, você pode usar o operador de captura &. Por exemplo, em vez de:

iex(5)> my_func = fn (x) -> x*x*x end

Você pode escrever:

iex(6)> my_func = &(&1*&1*&1)

Com vários parâmetros, use o número correspondente a cada argumento, contando a partir de 1:

iex(7)> my_func = fn (x, y) -> x + y end

iex(8)> my_func = &(&1 + &2)   # &1 stands for x and &2 stands for y

iex(9)> my_func.(4, 5)
9

Vários corpos #

Uma função anônima também pode ter vários corpos (como resultado de pattern matching):

my_func = fn
  param1 -> do_this
  param2 -> do_that
end

Quando você chama uma função com vários corpos, o Elixir tenta combinar os parâmetros que você forneceu com o corpo da função apropriado.

Funções de captura #

Use & para capturar funções de outros módulos. Você pode usar as funções capturadas diretamente como parâmetros de função ou dentro de funções anônimas.

Enum.map(list, fn(x) -> String.capitalize(x) end)

Pode ser mais conciso usando &:

Enum.map(list, &String.capitalize(&1))

Capturar funções sem passar nenhum argumento requer que você especifique explicitamente sua aridade, por exemplo &String.capitalize/1:

defmodule Bob do
  def say(message, f \\ &String.capitalize/1) do
    f.(message)
  end
end

Listas de palavras-chave como parâmetros de função #

Use listas de palavras-chave para parâmetros de estilo ‘options’ que contenham vários pares de valores-chave:

def myfunc(arg1, opts \\ []) do
  # Function body
end

Podemos chamar a função acima assim:

iex> myfunc "hello", pizza: true, soda: false

que é equivalente a:

iex> myfunc("hello", [pizza: true, soda: false])

Os valores dos argumentos estão disponíveis como opts.pizza e opts.soda respectivamente. Alternativamente, você pode usar átomos: opts[:pizza] e opts[:soda].

Parâmetros padrão #

Você pode passar parâmetros padrão para qualquer função nomeada usando a sintaxe: param \\ value:

defmodule Example do
    def func(p1, p2 \\ 2) do
        IO.inspect [p1, p2]
    end
end

Example.func("a")    # => ["a", 2]
Example.func("b", 4) # => ["b", 4]

Funções nomeadas e funções privadas #

Funções nomeadas

defmodule Math do
    # one way
    def add(a, b) do
        a + b
    end

    # another way
    def subtract(a, b), do: a - b
end

iex> Math.add(2, 3)
5
:ok
iex> Math.subtract(5, 2)
3
:ok

Funções Privadas

defmodule Math do
    def sum(a, b) do
        add(a, b)
    end

    # Private Function
    defp add(a, b) do
        a + b
    end
end

iex> Math.add(2, 3)
** (UndefinedFunctionError) undefined function Math.add/2
Math.add(3, 4)
iex> Math.sum(2, 3)
5

Correspondência de padrões #

Elixir corresponde uma chamada de função ao seu corpo com base no valor de seus argumentos.

defmodule Math do
    def factorial(0): do: 1
    def factorial(n): do: n * factorial(n - 1)
end

Aqui, fatorial de números positivos corresponde à segunda cláusula, enquanto factorial(0) corresponde à primeira. (ignorando números negativos por uma questão de simplicidade). Elixir tenta combinar as funções de cima para baixo. Se a segunda função for escrita acima da primeira, teremos um resultado inesperado, pois vai para uma recursão sem fim. Porque factorial(0) corresponde a factorial(n)

Cláusulas de guarda #

As cláusulas Guard nos permitem verificar os argumentos antes de executar a função. Cláusulas de guarda são geralmente preferidas a if e cond devido à sua legibilidade, e para tornar uma certa técnica de otimização mais fácil para o compilador. A primeira definição de função onde todos os guardas correspondem é executada.

Aqui está um exemplo de implementação da função fatorial usando guardas e correspondência de padrões.

defmodule Math do
    def factorial(0), do: 1
    def factorial(n) when n > 0: do: n * factorial(n - 1)
end

O primeiro padrão corresponde se (e somente se) o argumento for 0. Se o argumento não for 0, a correspondência de padrão falha e a próxima função abaixo é verificada.

Essa segunda definição de função tem uma cláusula de guarda: when n > 0. Isso significa que esta função só corresponde se o argumento n for maior que 0. Afinal, a função fatorial matemática não é definida para inteiros negativos.

Se nenhuma definição de função (incluindo sua correspondência de padrões e cláusulas de proteção) corresponder, um FunctionClauseError será gerado. Isso acontece para esta função quando passamos um número negativo como argumento, pois não está definido para números negativos.

Observe que este FunctionClauseError em si não é um erro. Retornar -1 ou 0 ou algum outro “valor de erro”, como é comum em algumas outras linguagens, esconderia o fato de que você chamou uma função indefinida, ocultando a fonte do erro, possivelmente criando um enorme bug doloroso para um futuro desenvolvedor.
