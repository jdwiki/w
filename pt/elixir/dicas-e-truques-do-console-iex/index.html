Obtenha o valor do último comando com v #

iex(1)> 1 + 1
2
iex(2)> v
2
iex(3)> 1 + v
3

Veja também: https://www.wikiod.com/pt/elixir/dicas-e-truques-do-console-iex#Obtenha o valor de um comando anterior com v

Histórico persistente #

Por padrão, o histórico de entrada do usuário no IEx não persiste em diferentes sessões.

erlang-history adiciona suporte de histórico ao shell Erlang e IEx:

git clone git@github.com:ferd/erlang-history.git
cd erlang-history
sudo make install

Agora você pode acessar suas entradas anteriores usando as teclas de seta para cima e para baixo, mesmo em diferentes sessões IEx.

Recompile o projeto com recompile #

iex(1)> recompile
Compiling 1 file (.ex)
:ok

Tenha seus aliases prontos quando iniciar o IEx #

Se você colocar seus aliases comumente usados ​​em um arquivo .iex.exs na raiz do seu aplicativo, o IEx irá carregá-los para você na inicialização.

alias App.{User, Repo}

Veja a documentação com h #

iex(1)> h List.last

                                 def last(list)

Returns the last element in list or nil if list is empty.

Examples

┃ iex> List.last([])
┃ nil
┃
┃ iex> List.last([1])
┃ 1
┃
┃ iex> List.last([1, 2, 3])
┃ 3

Obtenha o valor de um comando anterior com v #

iex(1)> a = 10
10
iex(2)> b = 20
20
iex(3)> a + b
30

Você pode obter uma linha específica passando o índice da linha:

iex(4)> v(3)
30

Você também pode especificar um índice relativo à linha atual:

iex(5)> v(-1) # Retrieves value of row (5-1) -> 4
30
iex(6)> v(-5) # Retrieves value of row (5-4) -> 1
10

O valor pode ser reutilizado em outros cálculos:

iex(7)> v(2) * 4
80

Se você especificar uma linha inexistente, IEx gerará um erro:

iex(7)> v(100) 
** (RuntimeError) v(100) is out of bounds
    (iex) lib/iex/history.ex:121: IEx.History.nth/2
    (iex) lib/iex/helpers.ex:357: IEx.Helpers.v/1

sair da expressão incompleta #

Quando você digitou algo no IEx que espera uma conclusão, como uma string de várias linhas, o IEx alterará o prompt para indicar que está aguardando você terminar, alterando o prompt para ter reticências (...) em vez de iex.

Se você achar que o IEx está esperando que você termine uma expressão, mas não tem certeza do que precisa para terminar a expressão, ou simplesmente deseja abortar esta linha de entrada, digite #iex:break como a entrada do console. Isso fará com que o IEx lance um TokenMissingError e cancele a espera por mais entrada, retornando você a uma entrada de console de “nível superior” padrão.

iex:1> "foo"
"foo"
iex:2> "bar
...:2> #iex:break
** (TokenMissingError) iex:2: incomplete expression

Mais informações estão disponíveis na documentação do IEx.

Sair do console IEx #

  1. Use Ctrl + C, Ctrl + C para sair
iex(1)>
BREAK: (a)bort (c)ontinue (p)roc info (i)nfo (l)oaded
       (v)ersion (k)ill (D)b-tables (d)istribution
  1. Use Ctrl+\ para sair imediatamente

Criando PID #

Isso é útil quando você não armazenou o PID de um comando anterior

iex(1)> self()
#PID<0.138.0>
iex(2)> pid("0.138.0")
#PID<0.138.0>
iex(3)> pid(0, 138, 0)
#PID<0.138.0>

Quando o console do Elixir está travado… #

Às vezes, você pode acidentalmente executar algo no shell que acaba esperando para sempre e, assim, bloqueando o shell:

iex(2)> receive do _ -> :stuck end

Nesse caso, pressione Ctrl-g. Você vai ver:

Comando de troca de usuário

Digite estes comandos em ordem:

  • k (para matar o processo do shell)
  • s (para iniciar um novo processo de shell)
  • c (para se conectar ao novo processo de shell)

Você terminará em um novo shell Erlang:

Eshell V8.0.2  (abort with ^G)
1>

Para iniciar um shell Elixir, digite:

'Elixir.IEx.CLI':local_start().

(não se esqueça do ponto final!)

Então você verá um novo processo de shell Elixir chegando:

Interactive Elixir (1.3.2) - press Ctrl+C to exit (type h() ENTER for help)
iex(1)> "I'm back"
"I'm back"
iex(2)>

Para escapar do modo “aguardando por mais entrada” (devido a aspas não fechadas, colchetes etc.), digite #iex:break, seguido de retorno de carro (⏎) :

iex(1)> "Hello, "world"
...(1)>
...(1)> #iex:break
** (TokenMissingError) iex:1: incomplete expression
    
iex(1)>

o acima é especificamente útil quando copiar e colar um trecho relativamente grande transforma o console no modo “aguardando mais entradas”.

Carrega um módulo ou script na sessão IEx #

Se você tiver um arquivo de elixir; um script ou módulo e deseja carregá-lo na sessão IEx atual, você pode usar o método c/1:

iex(1)> c "lib/utils.ex"
iex(2)> Utils.some_method

Isso compilará e carregará o módulo no IEx, e você poderá chamar todos os seus métodos públicos.

Para scripts, ele executará imediatamente o conteúdo do script:

iex(3)> c "/path/to/my/script.exs"
Called from within the script!
