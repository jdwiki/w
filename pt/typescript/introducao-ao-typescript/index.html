Instalação e configuração #

Fundo #

TypeScript é um superconjunto tipado de JavaScript que compila diretamente no código JavaScript. Os arquivos TypeScript geralmente usam a extensão .ts. Muitos IDEs suportam o TypeScript sem qualquer outra configuração necessária, mas o TypeScript também pode ser compilado com o pacote TypeScript Node.JS a partir da linha de comando.

IDE #

Estúdio visual #

  • Visual Studio 2015 inclui TypeScript.
  • O Visual Studio 2013 Update 2 ou posterior inclui o TypeScript, ou você pode baixar o TypeScript para versões anteriores.

Código do Visual Studio #

  • Visual Studio Code (vscode) fornece preenchimento automático contextual, bem como ferramentas de refatoração e depuração para TypeScript. vscode é implementado em TypeScript. Disponível para Mac OS X, Windows e Linux.

Tempestade da Web #

  • WebStorm 2016.2 vem com TypeScript e um compilador embutido. [O Webstorm não é gratuito]

IDEIA do IntelliJ #

  • IntelliJ IDEA 2016.2 tem suporte para Typescript e um compilador por meio de um [plugin](https://www.jetbrains.com/help/idea/2016.2 /typescript-support.html) mantido pela equipe Jetbrains. [IntelliJ não é gratuito]

Atom & atom-typescript #

  • Atom suporta TypeScript com o pacote atom-typescript.

Texto sublime #

  • Sublime Text suporta TypeScript com o pacote typescript.

Instalando a interface de linha de comando #

Instale o Node.js #

Instale o pacote npm globalmente #

Você pode instalar o TypeScript globalmente para ter acesso a ele de qualquer diretório.

npm install -g typescript

ou

Instale o pacote npm localmente #

Você pode instalar o TypeScript localmente e salvar em package.json para restringir a um diretório.

npm install typescript --save-dev

Canais de instalação #

Você pode instalar a partir de:

  • Canal estável: npm install typescript
  • Canal beta: npm install typescript@beta
  • Canal de desenvolvimento: npm install typescript@next

Compilando o código TypeScript #

O comando de compilação tsc vem com typescript, que pode ser usado para compilar código.

tsc my-code.ts

Isso cria um arquivo my-code.js.

Compile usando tsconfig.json #

Você também pode fornecer opções de compilação que acompanham seu código por meio de um arquivo tsconfig.json. Para iniciar um novo projeto TypeScript, cd no diretório raiz do seu projeto em uma janela de terminal e execute tsc --init. Este comando irá gerar um arquivo tsconfig.json com opções mínimas de configuração, semelhante ao abaixo.

{
    "compilerOptions": {
        "module": "commonjs",
        "target": "es5",
        "noImplicitAny": false,
        "sourceMap": false,
        "pretty": true
    },
    "exclude": [
        "node_modules"
    ]
}

Com um arquivo tsconfig.json colocado na raiz do seu projeto TypeScript, você pode usar o comando tsc para executar a compilação.

Sintaxe básica #

TypeScript é um superconjunto tipado de JavaScript, o que significa que todo código JavaScript é um código TypeScript válido. O TypeScript adiciona muitos recursos novos além disso.

O TypeScript torna o JavaScript mais parecido com uma linguagem fortemente tipada e orientada a objetos, semelhante a C# e Java. Isso significa que o código TypeScript tende a ser mais fácil de usar para grandes projetos e esse código tende a ser mais fácil de entender e manter. A tipagem forte também significa que a linguagem pode (e é) pré-compilada e que as variáveis ​​não podem receber valores que estejam fora de seu intervalo declarado. Por exemplo, quando uma variável TypeScript é declarada como um número, você não pode atribuir um valor de texto a ela.

Essa forte tipagem e orientação a objetos tornam o TypeScript mais fácil de depurar e manter, e esses eram dois dos pontos mais fracos do JavaScript padrão.

Declarações de tipo #

Você pode adicionar declarações de tipo a variáveis, parâmetros de função e tipos de retorno de função. O tipo é escrito após dois pontos após o nome da variável, assim: var num: número = 5; O compilador irá então verificar os tipos (quando possível) durante a compilação e relatar erros de tipo.

var num: number = 5;
num = "this is a string";  // error: Type 'string' is not assignable to type 'number'.

Os tipos básicos são:

  • number (tanto números inteiros quanto números de ponto flutuante)

  • string

  • booleano

  • Matriz. Você pode especificar os tipos de elementos de uma matriz. Existem duas maneiras equivalentes de definir tipos de array: Array<T> e T[]. Por exemplo: - number[] - array of numbers - Array<string> - array of strings

  • Tuplas. Tuplas têm um número fixo de elementos com tipos específicos. - [boolean, string] - tuple where the first element is a boolean and the second is a string. - [number, number, number] - tuple of three numbers.

  • {} - objeto, você pode definir suas propriedades ou indexador

    • {name: string, age: number} - object with name and age attributes
    • {[key: string]: number} - a dictionary of numbers indexed by string

  • enum - { Red = 0, Blue, Green } - enumeração mapeada para números

  • Função. Você especifica tipos para os parâmetros e o valor de retorno:

    • (param: number) => string - function taking one number parameter returning string
    • () => number - function with no parameters returning an number.
    • (a: string, b?: boolean) => void - function taking a string and optionally a boolean with no return value.

  • any - Permite qualquer tipo. Expressões envolvendo any não têm verificação de tipo.

  • void - representa “nada”, pode ser usado como um valor de retorno de função. Apenas null e undefined fazem parte do tipo void.

  • nunca

    • let foo: never; -As the type of variables under type guards that are never true.
    • function error(message: string): never { throw new Error(message); } - Como o tipo de retorno de funções que nunca retornam.

  • null - digite para o valor null. null é implicitamente parte de todos os tipos, a menos que verificações estritas de nulos estejam habilitadas.

Fundição #

Você pode executar a conversão explícita por meio de colchetes angulares, por exemplo:

var derived: MyInterface;
(<ImplementingClass>derived).someSpecificMethod();

Este exemplo mostra uma classe derivada que é tratada pelo compilador como uma MinhaInterface. Sem a conversão na segunda linha, o compilador lançaria uma exceção, pois não entende someSpecificMethod(), mas a conversão através de <ImplementingClass>derived sugere ao compilador o que fazer.

Outra maneira de converter no Typescript é usar a palavra-chave as:

var derived: MyInterface;
(derived as ImplementingClass).someSpecificMethod();

Desde o Typescript 1.6, o padrão é usar a palavra-chave as, porque usar <> é ambíguo em arquivos .jsx. Isso é mencionado na documentação oficial do Typescript.

Aulas #

As classes podem ser definidas e usadas no código TypeScript. Para saber mais sobre classes, consulte a página de documentação de classes.

Olá Mundo #

class Greeter {
    greeting: string;

    constructor(message: string) {
        this.greeting = message;
    }
    greet(): string {
        return this.greeting;
    }
};

let greeter = new Greeter("Hello, world!");
console.log(greeter.greet());

Aqui temos uma classe, Greeter, que tem um construtor e um método greet. Podemos construir uma instância da classe usando a palavra-chave new e passar uma string que queremos que o método greet produza no console. A instância de nossa classe Greeter é armazenada na variável greeter que nós então chamamos de método greet.

TypeScript REPL em Node.js #

Para usar o TypeScript REPL no Node.js, você pode usar tsun package

Instale-o globalmente com

npm install -g tsun

e execute em seu terminal ou prompt de comando com o comando tsun

Exemplo de uso:

$ tsun
TSUN : TypeScript Upgraded Node
type in TypeScript expression to evaluate
type :help for commands in repl
$ function multiply(x, y) {
..return x * y;
..}
undefined
$ multiply(3, 4)
12

Executando TypeScript usando ts-node #

ts-node é um pacote npm que permite ao usuário executar arquivos typescript diretamente, sem a necessidade de pré-compilação usando tsc. Ele também fornece REPL.

Instale o ts-node globalmente usando

npm install -g ts-node

ts-node não agrupa o compilador typescript, portanto, talvez seja necessário instalá-lo.

npm install -g typescript

Executando script

Para executar um script chamado main.ts, execute

ts-node main.ts

// main.ts
console.log("Hello world");

Exemplo de uso

$ ts-node main.ts
Hello world

Executando REPL

Para executar o REPL, execute o comando ts-node

Exemplo de uso

$ ts-node
> const sum = (a, b): number => a + b;
undefined
> sum(2, 2)
4
> .exit

Para sair do REPL use o comando .exit ou pressione CTRL+C duas vezes.
