Instalação e uso #

Assim como o shell interativo python integrado, o IPython é um shell REPL (Read-Evaluate-Print Loop), com uma variedade de recursos que o tornam mais agradável de usar para o dia-a-dia. desenvolvimento Python atual do que o shell REPL integrado.

Instalação #

Para instalá-lo:

pip install ipython

Ou, via Anaconda:

# To install into the active environment:
$ conda install ipython 

# Or, to create a new environment with IPython installed:
$ conda create -n <env_name> ipython

Ou, via Enthought Canopy:

$ enpkg ipython

Uso #

Após a instalação, execute-o usando seu Python padrão (2 ou 3) usando:

ipython

Ou para usar o Python 3:

ipython3

Conseguindo ajuda #

?

Isso lhe dá uma introdução e visão geral dos recursos do IPython.

object?

Isso lista todos os métodos e campos do objeto e sua documentação (se existir).

object??

Igual ao anterior, fornece ainda mais detalhes sobre o objeto, em particular, tentará localizar e exibir o código-fonte do objeto.

object.<TAB Key>

TAB-completar que lista e itera através de campos/métodos disponíveis de um objeto. Devido à natureza dinâmica do Python, nem todos os métodos podem ser descobertos dessa maneira. Também os métodos privados (começando com _) estarão ocultos por padrão, insira um _ e pressione TAB novamente para exibi-los.

%quickref

Isso exibe uma referência rápida para o shell do IPython.

IPython vs Jupyter #

O IPython tem duas partes: uma interface de linha de comando que substitui o padrão python REPL e uma maneira de executar o Python através do navegador da web como uma interface gráfica do usuário.

Com os desenvolvimentos mais recentes, a parte do navegador foi dividida no projeto Jupyter que permite que várias linguagens de programação usem a interface gráfica. Ainda é possível usar o IPython como um kernel Python para isso.

Instruções de configuração atualizadas para o Jupyter podem ser encontradas nos [documentos oficiais de instalação] (http://jupyter.readthedocs.io/en/latest/install.html).

ipython, ou jupyter console, quando invocado a partir da linha de comando sem quaisquer outros parâmetros, entrará em uma sessão de terminal interativa como abaixo:

jupyter qtconsole, ou ipython qtconsole antes da versão 5, iniciará um console baseado em QT com várias abas:

jupyter notebook, ou ipython notebook antes da versão 5, iniciará um servidor e por padrão abrirá uma página web, em http://localhost:8888/tree, com a visualização “Home” do diretório atual. Isso permite que você abra notebooks existentes ou novos kernels em vários idiomas, dependendo de qual você instalou; cada um será aberto em uma nova guia do navegador.

Notebooks permitem que você misture markdown, incluindo MathJax, código do kernel de sua escolha, gráficos e gráficos, imagens e até vídeos.

Colando no IPython #

%paste

Este é o principal método mágico para colar. Ele cola diretamente o texto da área de transferência do sistema, lidando de forma inteligente com problemas comuns com novas linhas e recuo.

%cpaste

Se você estiver usando IPython via SSH, use %cpaste, pois ele não precisa acessar a área de transferência do sistema remoto.

Desde o IPython 5.0.0, o kit de ferramentas de prompt aprimorado deve lidar diretamente com a colagem de código de várias linhas sem a necessidade de %paste ou %cpaste.

Armazenar variáveis ​​no IPython #

%storemagic armazena variáveis ​​e macros no banco de dados do IPython. Para restaurar automaticamente as variáveis ​​armazenadas na inicialização, adicione isto a ipython_config.py:

c.StoreMagic.autorestore = True

Exemplo:

In [1]: l = ['hello',10,'world']
In [2]: %store l
In [3]: exit

(IPython session is closed and started again...)

ville@badger:~$ ipython
In [1]: l
Out[1]: ['hello', 10, 'world']

Observação:

Deve-se notar que se você alterar o valor de uma variável, você precisa %armazená-la novamente se quiser manter o novo valor.

Observe também que as variáveis ​​precisarão ser pickleable; a maioria dos tipos básicos de python podem ser %armazenados com segurança.
