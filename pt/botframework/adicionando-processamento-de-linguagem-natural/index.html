O Bot Framework oferece suporte a Reconhecedores. Um reconhecedor é usado para reconhecer o que fazer sempre que um usuário enviar qualquer mensagem ao bot. Portanto, você pode projetar seu bot para reconhecer intents com base na entrada do usuário. O reconhecedor pode ser usado com a API do LUIS para adicionar compreensão de linguagem natural ao bot.

Sintaxe #

  • var reconhecedor = new builder.LUISRecognizer(‘URL do seu modelo’);

  • var intents = new builder.IntentDialog({recognizers: [recognizer]});

Inicializando e Adicionando LUISRecognizer #

Quando estiver pronto para um novo projeto com o modelo básico fornecido na introdução, você poderá adicionar um LUISRecognizer assim -

var model = ''    // Your LUIS Endpoint link comes here
var recognizer = new builder.LuisRecognizer(model);

Agora, recognizer é um LUISRecognizer e pode passar intents com base em seu modelo LUIS definido. Você pode adicionar o “reconhecedor” às suas intenções

var intents = new builder.IntentDialog({recognizers: [recognizer]});

Seu bot agora é capaz de lidar com intents do LUIS. Qualquer intenção nomeada no LUIS pode ser detectada usando a propriedade matches da classe IntentDialog. Digamos que um intent chamado hi seja definido no modelo LUIS, para reconhecer o intent no bot,

intents.matches('hi', function(session) {
    session.send("Hey :-)");
});

Definindo um modelo LUIS com intents #

A criação de um modelo LUIS requer pouca ou nenhuma experiência em programação. No entanto, você precisa estar familiarizado com 2 termos importantes que serão usados ​​extensivamente.

  1. Intenções - São como você identifica as funções que precisam ser executadas quando o usuário digita algo. Por exemplo - Um intent chamado Hi identificará uma função que precisa ser executada sempre que o usuário enviar “Hi”. As intenções são nomeadas exclusivamente em seu programa/modelo.
  2. Entidades - Identificam os substantivos em uma declaração. Ex: “Set an alarm for 1:00 pm”, aqui 1:00 pm é uma entidade que precisa ser reconhecida pelo chat-bot para definir um alarme.

Observação: Imagens do site não são fornecidas como front-end, mas o conceito principal permanece o mesmo.

Para criar um novo modelo, acesse LUIS.ai e entre com sua conta da Microsoft para ser levado à página de criação do aplicativo. Onde um projeto em branco pode ser criado.

Definindo intenções:

As intenções podem ser definidas na guia “Intenções”. Eles identificam qual função você precisa executar quando o usuário insere qualquer coisa.

Todos os aplicativos têm uma intenção padrão Nenhum, que é ativada sempre que a entrada do usuário não corresponde a nenhuma outra intenção.

Para definir uma intenção,

  1. Dê a ele um nome exclusivo relevante para a função que deseja executar.
  2. Quando a nomenclatura estiver completa, você deve adicionar enunciados à intenção. Os enunciados são o que você deseja que o usuário envie para ativar a intenção que você está definindo. Tente alimentar o maior número possível de enunciados para que o LUIS associe intenções e expressões corretamente.
  3. Treine seu modelo LUIS, clicando no botão Train na guia Train and Test. Após o treinamento o aplicativo pode ser testado no painel abaixo.
  4. Por fim, publique seu aplicativo na guia Publish App. Agora você deve obter um URL de ponto de extremidade que deve ser colocado ao definir LUISRecognizer em seu código de bot.

Adicionando Entidades ao Modelo LUIS #

Uma entidade é a informação que seu bot extrai de um enunciado específico em conformidade com uma intenção.

Eg- Deixe My name is John Doe pertencer a uma intent chamada introduction. Para que seu bot entenda e extraia o nome John Doe da frase, você precisa definir uma entidade que faça isso. Você pode nomear a entidade como quiser, mas é melhor nomeá-la como algo pertencente ao que ela extrai. Em nosso exemplo, podemos chamar nossa entidade de nome.

As entidades podem ser reutilizadas entre diferentes intenções, para extrair coisas diferentes. Portanto, o melhor princípio seria criar uma entidade que extraia apenas tipos de dados e usá-los em diferentes intenções. Portanto, em nosso exemplo acima, digamos que Reservar um voo na Emirates pertence à intenção reserva, então a mesma entidade, nome, pode ser usada para extrair o nome do voo emirates.

Você precisa ter em mente duas coisas antes de continuar definindo entidades -

  1. As entidades devem ser únicas por enunciado em uma intenção. Uma entidade não pode ser usada duas vezes no mesmo enunciado.
  2. LUIS não diferencia maiúsculas de minúsculas. Isso implica que tudo o que for extraído e recebido por meio da extração de entidade estará em letras minúsculas. Portanto, extrair dados sensíveis a maiúsculas e minúsculas por meio de entidades provavelmente é uma má ideia.

Adicionando entidades pré-construídas

As entidades pré-construídas são, como o nome sugere, pré-construídas, ou seja, já estão configuradas para extrair um tipo específico de dados na intenção à qual são adicionadas. Um exemplo pode ser a entidade number que extrai números da intenção à qual ela foi atribuída. Os números podem ser numéricos ou alfabéticos como 10 ou dez.

Para obter uma lista completa de todas as entidades pré-construídas, você pode visitar [Lista de entidades pré-construídas][1].

Para adicionar entidades pré-construídas,

  1. Vá para a guia entidades.
  2. Clique em ‘Adicionar entidades pré-construídas’ e selecione a entidade que deseja adicionar ao modelo e clique em salvar.

Adicionando Entidades Personalizadas Entidades personalizadas são de 4 tipos,

  1. Simples: Uma entidade simples extrai um dado específico, name nos exemplos acima é uma entidade simples.
  2. Hierárquica: Uma entidade pai com entidades filhas (subtipos) que são dependentes do pai.
  3. Composto: um grupo de 2 ou mais entidades independentes juntas.
  4. Lista: uma entidade que reconhece palavras apenas de uma determinada lista.

Definindo Entidades Simples

  1. Vá até a aba entidades.
  2. Clique em ‘Adicionar Entidades Personalizadas’
  3. Dê um nome à sua entidade, verifique o tipo de entidade necessário e clique em “Salvar”.

Todos os outros tipos de entidades podem ser adicionados da mesma forma apenas alterando o ‘Tipo de Entidade’ para um dos tipos acima. Nos tipos de entidade hierárquica e composta, você também precisará fornecer nomes aos filhos junto com o nome da entidade pai. Definir entidades de lista é um pouco diferente do resto.

Definindo Entidades da Lista

Depois de seguir os passos acima para criar uma ‘Lista de Entidade’ colocando o ‘Tipo de Entidade’ como Lista, você será direcionado para a página de detalhes da entidade que acabou de definir.

  1. Defina um valor canônico. Este é um valor padrão que o bot receberá quando o usuário digitar qualquer um dos sinônimos.
  2. Defina sinônimos para o valor canônico. Eles serão convertidos para o valor canônico ao serem encontrados pela entidade.

Você também pode importar listas inteiras usando uma matriz de objetos JSON, no formato:

[
    {
        "canonicalForm": "Hey",
        "list": [
            "Howdy",
            "Hi"
         ]
    },
    .
    .
    .
]

Associando uma entidade a uma intenção

Entidades pré-construídas e lista já possuem um conjunto de valores definidos que podem ser extraídos de todos os enunciados, porém, enunciados simples, hierárquicos e composto precisam ser treinados para pegar valores.

Isso pode ser feito por

  1. Vá para a guia “intenções” e escolha a intenção à qual deseja adicionar a entidade.
  2. Adicione um enunciado com um valor fictício que você gostaria de extrair. Digamos, você pode adicionar “Meu nome é John Doe” como um enunciado.
  3. Clique e arraste o mouse sobre as palavras que deseja que a entidade extraia. Você precisará destacar john doe no exemplo acima.
  4. Uma lista suspensa será aberta com uma lista de todas as entidades disponíveis em seu projeto. Selecione o correspondente como achar melhor. Name será a entidade selecionada no exemplo acima.
  5. Adicione mais enunciados com valores fictícios diferentes a cada vez e todas as estruturas possíveis que você possa imaginar.
  6. Treine e publique seu modelo LUIS.
