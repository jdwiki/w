Exportando registros via API baseada em contrato REST

  • Exportação de dados em uma única chamada REST
  • Implementando paginação em várias solicitações REST

Nesta página

  • Exportação de dados em uma única chamada REST
  • Implementando paginação em várias solicitações REST

Este tópico demonstrará como exportar registros do Acumtica ERP por meio da API REST baseada em contrato. Em contraste com a API baseada em tela do Acumtica ERP, a API baseada em contrato fornece interfaces SOAP e REST. Para obter mais informações sobre a API baseada em contrato, consulte Documentação do ERP Acumática

Para se comunicar com a API baseada em contrato REST do Acumtica ERP, seu aplicativo cliente deve sempre executar as 3 etapas a seguir:

  1. faça login na instância do ERP Acumática e obtenha o cookie com as informações da sessão do usuário

  2. interagir com um dos endpoints de API baseados em contrato disponíveis na instância do ERP Acumática

  3. Saia do Acumática ERP para fechar a sessão do usuário

Todos os exemplos fornecidos neste tópico foram criados com o endpoint Default, sempre implantado como parte do processo de instalação padrão do Acumtica ERP. Na tela Web Service Endpoints (SM.20.70.60), você pode visualizar os detalhes dos endpoints existentes ou configurar seus endpoints personalizados dos serviços web baseados em contrato do Acumtica ERP:

Para sua referência, abaixo está a implementação da classe RestService usada em todos os exemplos acima para interagir com o serviço web baseado em contrato do Acumtica ERP:

public class RestService : IDisposable
{
    private readonly HttpClient _httpClient;
    private readonly string _acumaticaBaseUrl;
    private readonly string _acumaticaEndpointUrl;

    public RestService(string acumaticaBaseUrl, string endpoint,
        string userName, string password,
        string company, string branch)
    {
        _acumaticaBaseUrl = acumaticaBaseUrl;
        _acumaticaEndpointUrl = _acumaticaBaseUrl + "/entity/" + endpoint + "/";
        _httpClient = new HttpClient(
            new HttpClientHandler
            {
                UseCookies = true,
                CookieContainer = new CookieContainer()
            })
        {
            BaseAddress = new Uri(_acumaticaEndpointUrl),
            DefaultRequestHeaders =
            {
                Accept = {MediaTypeWithQualityHeaderValue.Parse("text/json")}
            }
        };

        var str = new StringContent(
            new JavaScriptSerializer()
                .Serialize(
                    new
                    {
                        name = userName,
                        password = password,
                        company = company,
                        branch = branch
                    }),
                    Encoding.UTF8, "application/json");

        _httpClient.PostAsync(acumaticaBaseUrl + "/entity/auth/login", str)
            .Result.EnsureSuccessStatusCode();
    }

    void IDisposable.Dispose()
    {
        _httpClient.PostAsync(_acumaticaBaseUrl + "/entity/auth/logout",
            new ByteArrayContent(new byte[0])).Wait();
        _httpClient.Dispose();
    }

    public string GetList(string entityName)
    {
        var res = _httpClient.GetAsync(_acumaticaEndpointUrl + entityName)
            .Result.EnsureSuccessStatusCode();

        return res.Content.ReadAsStringAsync().Result;
    }

    public string GetList(string entityName, string parameters)
    {
        var res = _httpClient.GetAsync(_acumaticaEndpointUrl + entityName + "?" + parameters)
            .Result.EnsureSuccessStatusCode();

        return res.Content.ReadAsStringAsync().Result;
    }
}

Exportação de dados em uma única chamada REST #

Neste exemplo, você explorará como exportar os seguintes dados do Acumtica ERP em uma única chamada por meio da API baseada em contrato REST:

  • todos os itens de estoque existentes no aplicativo
  • todos os pedidos de venda do tipo IN

Se você precisar exportar registros do Acumtica ERP, use a seguinte URL: http://<URL da instância do ERP Acumática>/entity/<Nome do terminal>/<versão do terminal>/<entidade de nível superior>

<Entidade de nível superior> é o nome da entidade que você vai exportar

Para exportar todos os itens de estoque em uma única chamada REST: #

Para exportar registros de itens de estoque de uma instância local do AcumticaERP usando o ponto de extremidade Default da versão 6.00.001, você deve usar o seguinte URL: http://localhost/AcumticaERP /entity/Default/6.00.001/StockItem

Abaixo está o código de exemplo escrito em C# para exportar todos os itens de estoque enviando uma única chamada REST para o ponto de extremidade Default da versão 6.00.001:

using (RestService rs = new RestService(
    @"http://localhost/AcumaticaERP/", "Default/6.00.001",
    username, password, company, branch))
{
    string stockItems = rs.GetList("StockItem");
}

Para exportar todos os pedidos de venda do tipo IN em uma única chamada REST: #

Para exportar pedidos de vendas do tipo IN de uma instância local do AcumticaERP usando o ponto de extremidade Default da versão 6.00.001, você deve usar o seguinte URL : http://localhost/AcumaticaERP/entity/Default/6.00.001/SalesOrder?$filter=OrderType eq 'IN'

Abaixo está o código de exemplo escrito em C# para exportar todos os pedidos de venda do tipo IN enviando uma única chamada REST para o ponto de extremidade Default da versão 6.00.001 :

using (RestService rs = new RestService(
    @"http://localhost/StackOverflow/", "Default/6.00.001",
    username, password, company, branch))
{
    var parameters = "$filter=OrderType eq 'IN'";
    string inSalesOrders = rs.GetList("SalesOrder", parameters);
}

Implementando paginação em várias solicitações REST #

Neste exemplo, você explorará como exportar os seguintes dados do Acumtica ERP em lotes por meio da API baseada em contrato REST:

  • itens de estoque existentes no aplicativo em lotes de 10 registros
  • todos os pedidos de venda em lotes de 100 registros

Para exportar itens de estoque em lotes de 10 registros com várias chamadas REST: #

Para exportar os primeiros 10 itens de estoque de uma instância local do AcumaticaERP usando o ponto de extremidade Default da versão 6.00.001, você deve usar o seguinte URL: http://localhost/ AcumticaERP/entity/Default/6.00.001/StockItem?$top=10

Assim, para solicitar itens de estoque de 10 a 20, basta estender a URL acima com o parâmetro filter: http://localhost/AcumticaERP/entity/Default/6.00.001/StockItem?$top=10& $filter=ID do Inventário gt '<ID do Inventário>'

<InventoryID> é o ID do último item de estoque exportado com uma chamada REST anterior

Abaixo está o código de exemplo escrito em C# para exportar todos os itens de estoque em lotes de 10 registros enviando várias chamadas REST para o ponto de extremidade Padrão da versão 6.00.001:

using (RestService rs = new RestService(
    @"http://localhost/StackOverflow/", "Default/6.00.001",
    username, password, company, branch))
{
    var json = new JavaScriptSerializer();
    string parameters = "$top=10";
    string items = rs.GetList("StockItem", parameters);
    var records = json.Deserialize<List<Dictionary<string, object>>>(items);

    while (records.Count == 10)
    {
        var inventoryID = records[records.Count - 1]["InventoryID"] as Dictionary<string, object>;
        var inventoryIDValue = inventoryID.Values.First();
        string nextParameters = parameters + "&" + 
            "$filter=" + string.Format("InventoryID gt '{0}'", inventoryIDValue);
        items = rs.GetList("StockItem", nextParameters);
        records = json.Deserialize<List<Dictionary<string, object>>>(items);
    }
}

Para exportar todos os pedidos de venda em lotes de 100 registros com várias chamadas REST: #

Para exportar os primeiros 100 pedidos de venda de uma instância local do AcumticaERP usando o ponto de extremidade Default da versão 6.00.001, você deve usar o seguinte URL: http://localhost/ AcumticaERP/entity/Default/6.00.001/SalesOrder?$top=100

Como a chave primária da entidade Pedido de vendas é composta pelo Tipo de pedido e pelo Número do pedido, neste exemplo você usará uma combinação de *filtro * parâmetros para os campos Tipo de pedido e Número do pedido:

  • para solicitar pedidos de vendas de 100 a 200 do tipo SO, deve-se utilizar a seguinte URL: http://localhost/AcumticaERP/entity/Default/6.00.001/SalesOrder?$top=100& $filter=OrderType eq 'SO' e OrderNbr gt '<OrderNbr>'

<OrderNbr> é o número do último pedido de venda exportado com uma chamada REST anterior

  • portanto, para solicitar os primeiros 100 pedidos de venda do tipo SO, você deve usar a seguinte URL: http://localhost/AcumticaERP/entity/Default/6.00.001/SalesOrder?$top =100&$filter=OrderType gt 'SO' e OrderNbr gt ''

Abaixo está o código de exemplo escrito em C# para exportar todos os pedidos de vendas em lotes de 100 registros com várias chamadas REST para o ponto de extremidade Default da versão 6.00.001:

using (RestService rs = new RestService(
    @"http://localhost/StackOverflow/", "Default/6.00.001",
    username, password, company, branch))
{
    var json = new JavaScriptSerializer();
    string parameters = "$top=100";
    string items = rs.GetList("SalesOrder", parameters);
    var records = json.Deserialize<List<Dictionary<string, object>>>(items);

    bool sameOrderType = true;
    while (records.Count > 0 && (records.Count == 100 || !sameOrderType))
    {
        var orderType = records[records.Count - 1]["OrderType"] as Dictionary<string, object>;
        var orderTypeValue = orderType.Values.First();
        var orderNbr = records[records.Count - 1]["OrderNbr"] as Dictionary<string, object>;
        var orderNbrValue = orderNbr.Values.First();

        string nextParameters = parameters + "&" + "$filter=" +
            string.Format("OrderType {0} '{1}'", sameOrderType ? "eq" : "gt", orderTypeValue) + " and " +
            string.Format("OrderNbr gt '{0}'", sameOrderType ? orderNbrValue : "''" );
        items = rs.GetList("SalesOrder", nextParameters);
        records = json.Deserialize<List<Dictionary<string, object>>>(items);
        sameOrderType = records.Count == 100;
    }
}
