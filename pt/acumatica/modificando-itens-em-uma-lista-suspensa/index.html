Neste tópico, você aprenderá como modificar atributos de campo herdados dos atributos PXStringList ou PXIntList. A abordagem demonstrada garantirá a não interrupção da funcionalidade do produto básico Acumtica ERP e exigirá manutenção mínima, se houver, ao atualizar suas personalizações para uma versão mais recente do Acumtica.

Em todos os exemplos acima, você fez alterações nos arrays _AllowedValues e _AllowedLabels. Se sua tarefa for modificar apenas o rótulo (valor externo) de um item suspenso, considere usar os Dicionários de Tradução. Para obter mais informações sobre os dicionários de tradução, consulte Documentação do ERP Acumática

Modificando o estado civil #

Neste exemplo, você modificará a lista suspensa Status civil encontrada no formulário Contatos (CR302000):

Para adicionar novos itens ao sucessor PXStringListAttribute #

A melhor maneira de estender itens suspensos codificados dentro de um atributo herdado do atributo PXStringList ou PXIntList é aumentando o tamanho dos arrays _AllowedValues e _AllowedLabels no construtor do seu atributo de campo personalizado:

[PXLocalizable(Messages.Prefix)]
public static class MaritalStatusesMessages
{
    public const string CommonLaw = "Living common law";
    public const string Separated = "Separated (not living common law)";
    public const string DivorcedNoCommonLaw = "Divorced (not living common law)";
    public const string NeverMarried = "Never Married";
}

public class MaritalStatusesCst1Attribute : MaritalStatusesAttribute
{
    public const string CommonLaw = "L";
    public const string Separated = "P";
    public const string NeverMarried = "N";

    public MaritalStatusesCst1Attribute()
    {
        Array.Resize(ref _AllowedValues, _AllowedValues.Length + 3);
        _AllowedValues[_AllowedValues.Length - 3] = CommonLaw;
        _AllowedValues[_AllowedValues.Length - 2] = Separated;
        _AllowedValues[_AllowedValues.Length - 1] = NeverMarried;
        Array.Resize(ref _AllowedLabels, _AllowedLabels.Length + 3);
        _AllowedLabels[_AllowedLabels.Length - 3] = MaritalStatusesMessages.CommonLaw;
        _AllowedLabels[_AllowedLabels.Length - 2] = MaritalStatusesMessages.Separated;
        _AllowedLabels[_AllowedLabels.Length - 1] = MaritalStatusesMessages.NeverMarried;
    }
}

No exemplo acima, você aumentou o tamanho das matrizes _AllowedValues e _AllowedLabels para adicionar 3 itens adicionais à lista suspensa Estado civil. Os valores internos, armazenados no array _AllowedValues, serão atribuídos aos campos DAC e salvos no banco de dados, e os valores externos do array _AllowedValues representam valores internos na interface do usuário.

Para testar os resultados, na extensão Contact DAC, decore o campo MaritalStatus com o MaritalStatusesCst1Attribute:

public class ContactExt : PXCacheExtension<Contact>
{
    [PXRemoveBaseAttribute(typeof(MaritalStatusesAttribute))]
    [PXMergeAttributes(Method = MergeMethod.Append)]
    [MaritalStatusesCst1]
    public string MaritalStatus { get; set; }
}

Agora há 7 itens na lista suspensa Estado civil:

Para remover itens declarados no sucessor PXStringListAttribute #

Para remover um item suspenso específico, que foi codificado dentro de um atributo herdado do atributo PXStringList ou PXIntList, você precisa diminuir o tamanho dos arrays _AllowedValues e _AllowedLabels no construtor do seu atributo de campo personalizado:

public class MaritalStatusesCst2Attribute : MaritalStatusesCst1Attribute
{
    public MaritalStatusesCst2Attribute()
    {
        string[] allowedValues = new string[_AllowedValues.Length - 1];
        string[] allowedLabels = new string[_AllowedLabels.Length - 1];
        Array.Copy(_AllowedValues, 1, allowedValues, 0, _AllowedValues.Length - 1);
        Array.Copy(_AllowedLabels, 1, allowedLabels, 0, _AllowedValues.Length - 1);
        _AllowedValues = allowedValues;
        _AllowedLabels = allowedLabels;
    }
}

No exemplo acima, você diminuiu o tamanho dos arrays _AllowedValues e _AllowedLabels para remover o item Single da lista suspensa Marital Status.

Para testar os resultados, na extensão Contact DAC, decore o campo MaritalStatus com o MaritalStatusesCst2Attribute:

public class ContactExt : PXCacheExtension<Contact>
{
    [PXRemoveBaseAttribute(typeof(MaritalStatusesAttribute))]
    [PXMergeAttributes(Method = MergeMethod.Append)]
    [MaritalStatusesCst2]
    public string MaritalStatus { get; set; }
}

Agora há apenas 6 itens: 3 originais e 3 personalizados - na lista suspensa Estado civil:

Para substituir itens declarados no sucessor PXStringListAttribute #

Para substituir um item suspenso específico, originalmente codificado dentro de um atributo herdado do atributo PXStringList ou PXIntList, você precisa atualizar o valor apropriado nas matrizes _AllowedValues e _AllowedLabels no construtor do seu atributo de campo personalizado:

public class MaritalStatusesCst3Attribute : MaritalStatusesCst2Attribute
{
    public const string DivorcedNoCommonLaw = "V";

    public MaritalStatusesCst3Attribute()
    {
        _AllowedValues[Array.IndexOf(_AllowedValues, Divorced)] = DivorcedNoCommonLaw;
        _AllowedLabels[Array.IndexOf(_AllowedLabels, Messages.Divorced)] = MaritalStatusesMessages.DivorcedNoCommonLaw;
    }
}

No exemplo acima, você substituiu o item D - Divorciado por V - Divorciado (não vive a lei comum) no _AllowedValues e matrizes _AllowedLabels respectivamente. Ao fazer isso, substituímos os valores internos e externos de um item suspenso.

Para testar os resultados, na extensão Contact DAC, decore o campo MaritalStatus com o MaritalStatusesCst3Attribute:

public class ContactExt : PXCacheExtension<Contact>
{
    [PXRemoveBaseAttribute(typeof(MaritalStatusesAttribute))]
    [PXMergeAttributes(Method = MergeMethod.Append)]
    [MaritalStatusesCst3]
    public string MaritalStatus { get; set; }
}

Agora há apenas 6 itens: 2 originais, 3 personalizados e 1 substituído - na lista suspensa Estado civil:
