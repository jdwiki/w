Atributo PXFormula #

Descrição geral #

Uma fórmula no Acumática é um campo DAC calculado com base nos valores de outros campos de objeto.

Para calcular uma fórmula, o framework Aсumatica fornece um conjunto de várias operações e funções (como operações aritméticas, lógicas e de comparação e funções de processamento de strings; consulte Lista de Fórmulas Comuns Integradas). Além dos valores de campo, uma fórmula pode usar várias constantes fornecidas tanto pelo núcleo do Acumática quanto pelas soluções do aplicativo. Além disso, uma fórmula pode obter valores para o cálculo não apenas do registro atual, mas também de outras fontes (consulte Contexto da fórmula e seus modificadores).

A beleza das fórmulas é que elas recalcularão automaticamente o valor no momento certo:

  • Na inadimplência do campo (inserindo uma nova linha; manipulador de eventos FieldDefaulting do campo de fórmula)
  • Na atualização de campos dependentes (manipulador de eventos FieldUpdated de cada campo dependente)
  • Na seleção do banco de dados (somente para campos não vinculados; manipulador de eventos RowSelecting)
  • No banco de dados persistindo, se necessário (o desenvolvedor deve especificá-lo explicitamente; manipulador de eventos RowPersisted)

O recálculo de um valor de campo de fórmula na atualização de um campo dependente gera um evento FieldUpdated para o campo de fórmula. Isso permite que você faça uma cadeia de fórmulas dependentes (consulte Referências circulares diretas e mediadas em fórmulas).

Os desenvolvedores de aplicativos podem escrever suas próprias fórmulas do lado do aplicativo.

Modos de uso #

Uma fórmula pode ser usada em três modos principais:

  • Basta calcular o valor e atribuí-lo ao campo de fórmula (consulte Uso Básico)
  • Calculando o valor agregado dos valores existentes dos campos de fórmula e atribuindo-o ao campo especificado no objeto pai (consulte Uso Agregado)
  • Modo misto: Calculando o valor da fórmula, atribuindo-o ao campo da fórmula, calculando o valor agregado e atribuindo-o ao campo no objeto pai (consulte Uso combinado)

Existe outro modo auxiliar, fórmulas não vinculadas, que é muito semelhante ao modo misto, mas os valores calculados da fórmula não são atribuídos ao campo de fórmula. O valor agregado é calculado imediatamente e atribuído ao campo do objeto pai. Consulte Uso de fórmulas não vinculadas para obter mais informações.

Propriedades do PXFormulaAttribute e Parâmetros do Construtor #

A funcionalidade da fórmula é implementada por PXFormulaAttribute. O construtor de PXFormulaAttribute tem as seguintes assinaturas:

public PXFormulaAttribute(Type formulaType)
{
    // ...
}

O parâmetro único formulaType é um tipo de expressão de fórmula para calcular o valor do campo de outros campos do mesmo registro de dados. Este parâmetro deve atender a uma das seguintes condições:

  • Deve implementar a interface IBqlField
  • Deve ser uma constante BQL
  • Deve implementar a interface IBqlCreator (veja Lista de Fórmulas Comuns Integradas)
public PXFormulaAttribute(Type formulaType, Type aggregateType)
{
    // ...
}

O primeiro parâmetro, formulaType, é o mesmo do primeiro construtor. O segundo parâmetro, aggregateType, é um tipo de fórmula de agregação para calcular o campo de registro de dados pai dos campos de registro de dados filho. Uma função de agregação pode ser usada, como SumCalc, CountCalc, MinCalc e MaxCalc. Os desenvolvedores de aplicativos podem criar suas próprias fórmulas de agregação.

Um tipo de fórmula agregada deve ser um tipo genérico e deve implementar a interface IBqlAggregateCalculator. O primeiro parâmetro genérico do tipo de fórmula agregada deve implementar a interface IBqlField e deve ter o tipo de campo do objeto pai.

public virtual bool Persistent { get; set; }

A propriedade PXFormulaAttribute.Persistent indica se o atributo recalcula a fórmula após as alterações serem salvas no banco de dados. Você pode precisar de recálculo se os campos dos quais a fórmula depende forem atualizados no evento RowPersisting. Por padrão, a propriedade é igual a false.

Uso #

Na maioria dos casos, as fórmulas são usadas para cálculo direto do valor do campo de fórmula de outros campos do mesmo registro de dados.

O exemplo mais simples de uso de fórmulas:

[PXDBDate]
[PXFormula(typeof(FADetails.receiptDate))]
[PXDefault]
[PXUIField(DisplayName = Messages.PlacedInServiceDate)]
public virtual DateTime? DepreciateFromDate { get; set; }

Neste exemplo, o valor do campo ReceiptDate é atribuído ao campo DepreciateFromDate na inserção de um novo registro e na atualização do campo ReceiptDate.

Um exemplo um pouco mais complexo:

[PXCurrency(typeof(APPayment.curyInfoID), typeof(APPayment.unappliedBal))]
[PXUIField(DisplayName = "Unapplied Balance", Visibility = PXUIVisibility.Visible, Enabled = false)]
[PXFormula(typeof(Sub<APPayment.curyDocBal, APPayment.curyApplAmt>))]
public virtual Decimal? CuryUnappliedBal { get; set; }

Aqui, o saldo não aplicado do documento é calculado como a diferença entre o saldo do documento e o valor aplicado.

Exemplo de múltipla escolha com um valor padrão:

[PXUIField(DisplayName = "Class Icon", IsReadOnly = true)]
[PXImage]
[PXFormula(typeof(Switch<
    Case<Where<EPActivity.classID, Equal<CRActivityClass.task>>, EPActivity.classIcon.task,
    Case<Where<EPActivity.classID, Equal<CRActivityClass.events>>, EPActivity.classIcon.events,
    Case<Where<EPActivity.classID, Equal<CRActivityClass.email>,
        And<EPActivity.isIncome, NotEqual<True>>>, EPActivity.classIcon.email,
    Case<Where<EPActivity.classID, Equal<CRActivityClass.email>,
        And<EPActivity.isIncome, Equal<True>>>, EPActivity.classIcon.emailResponse,
    Case<Where<EPActivity.classID, Equal<CRActivityClass.history>>, EPActivity.classIcon.history>>>>>,
    Selector<Current2<EPActivity.type>, EPActivityType.imageUrl>>))]
public virtual string ClassIcon { get; set; }

Ordem dos Campos #

A ordem dos campos no DAC é importante para corrigir o cálculo da fórmula. Todos os campos de origem (a partir dos quais a fórmula é calculada), incluindo outras fórmulas, devem ser definidos no DAC antes do campo de fórmula. Caso contrário, o campo pode ser calculado incorretamente ou causar um erro de tempo de execução.

Contexto da Fórmula e Seus Modificadores #

Por padrão, o contexto do cálculo da fórmula é restrito pelo objeto atual (registro) da classe que contém a declaração da fórmula. Também é permitido usar constantes (descendentes da classe Constant<>).

Uma fórmula que usa apenas os campos de seu objeto:

public partial class Contract : IBqlTable, IAttributeSupport
{
    //...
    [PXDecimal(4)]
    [PXDefault(TypeCode.Decimal, "0.0", PersistingCheck = PXPersistingCheck.Nothing)]
    [PXFormula(typeof(Add<Contract.pendingRecurring, Add<Contract.pendingRenewal, Contract.pendingSetup>>))]
    [PXUIField(DisplayName = "Total Pending", Enabled=false)]
    public virtual decimal? TotalPending { get; set; }
    //...
}

No entanto, é possível obter valores de entrada para o cálculo da fórmula de outras fontes:

  • Um registro atual de qualquer cache no BLC (se atribuído).
  • Um registro estrangeiro especificado por PXSelectorAttribute.
  • Um registro pai especificado por PXParentAttribute.

A fórmula oferece suporte aos seguintes modificadores de contexto.

Current<TRecord.field> e Current2<TRecord.field> #

Busca o valor do campo do registro armazenado na propriedade Current do cache TRecord.

Se a propriedade Current do cache ou o próprio campo contiver null:

  • Current<> força o padrão do campo e retorna o valor do campo padrão.
  • Current2<> retorna nulo.

Exemplo:

[PXFormula(typeof(Switch<
    Case<Where<
        ARAdjust.adjgDocType, Equal<Current<ARPayment.docType>>,
        And<ARAdjust.adjgRefNbr, Equal<Current<ARPayment.refNbr>>>>,
        ARAdjust.classIcon.outgoing>,
    ARAdjust.classIcon.incoming>))]
protected virtual void ARAdjust_ClassIcon_CacheAttached(PXCache sender)

Parent<TParent.field> #

Busca o valor do campo do registro de dados pai conforme definido por PXParentAttribute que reside no DAC atual.

public class INTran : IBqlTable
{
    [PXParent(typeof(Select<
        INRegister, 
        Where<
            INRegister.docType, Equal<Current<INTran.docType>>,
            And<INRegister.refNbr,Equal<Current<INTran.refNbr>>>>>))]
    public virtual String RefNbr { ... }
  
    [PXFormula(typeof(Parent<INRegister.origModule>))]
    public virtual String OrigModule { ... }
}

IsTableEmpty<TRecord> #

Retorna true se a tabela de banco de dados correspondente ao DAC especificado não contiver registros, false caso contrário.

public class APRegister : IBqlTable
{
    [PXFormula(typeof(Switch<
        Case<Where<
            IsTableEmpty<APSetupApproval>, Equal<True>>,
            True,
        Case<Where<
            APRegister.requestApproval, Equal<True>>,
            False>>,
        True>))]
    public virtual bool? DontApprove { get; set; }
}

Seletor<KeyField, ForeignOperand> #

Busca um PXSelectorAttribute definido no campo de chave estrangeira (KeyField) do DAC atual. #

Busca o registro de dados estrangeiros atualmente referenciado pelo seletor. #

Calcula e retorna uma expressão nesse registro de dados conforme definido por ForeignOperand. #

public class APVendorPrice : IBqlTable
{
    // Note: inventory attribute is an
    // aggregate containing a PXSelectorAttribute
    // inside, which is also valid for Selector<>.
    // -
    [Inventory(DisplayName = "Inventory ID")]
    public virtual int? InventoryID
  
    [PXFormula(typeof(Selector<
        APVendorPrice.inventoryID, 
        InventoryItem.purchaseUnit>))]
    public virtual string UOM { get; set; }
}

Usando fórmulas em campos não vinculados #

Se o campo de fórmula for um campo não vinculado marcado com um dos descendentes PXFieldAttribute (como PXIntAttribute ou PXStringAttribute), seu cálculo será acionado adicionalmente durante o evento RowSelecting.

Atributo PXRestrictor #

Introdução #

O atributo PXSelectorAttribute (também conhecido como seletor), embora vital e usado com frequência, tem duas grandes desvantagens:

  • Dá uma mensagem não informativa "<object_name> não pode ser encontrado no sistema" se nenhum item for encontrado para satisfazer a condição do seletor.
  • O produz a mesma mensagem de erro se você atualizar outros campos do registro, mas o objeto referenciado pelo seletor já foi alterado e não atende mais à sua condição. Este comportamento é claramente errado porque a lei não deve ser retroativa.

O PXRestrictorAttribute (também conhecido como restritor) pode ser usado para resolver esses problemas.

Detalhes #

PXRestrictorAttribute não funciona sozinho; ele deve sempre ser emparelhado com um PXSelectorAttribute. Usar o restritor sem o seletor não terá efeito.

O restritor encontra o seletor no mesmo campo, injetando nele uma condição adicional e a mensagem de erro correspondente. A condição do restritor é anexada à condição do seletor por meio de um booleano AND e uma mensagem de erro apropriada é gerada se o objeto referenciado violar a restrição do restritor. Além disso, se o objeto referenciado foi alterado e não atende mais à condição do restritor, nenhuma mensagem de erro será produzida quando você alterar qualquer outro campo do objeto de referência.

Uso geral:

[PXDBInt]
[PXSelector(typeof(Search<FAClass.assetID, Where<FAClass.recordType, Equal<FARecordType.classType>>>),
    typeof(FAClass.assetCD), typeof(FAClass.assetTypeID), typeof(FAClass.description), typeof(FAClass.usefulLife),
    SubstituteKey = typeof(FAClass.assetCD),
    DescriptionField = typeof(FAClass.description), CacheGlobal = true)]
[PXRestrictor(typeof(Where<FAClass.active, Equal<True>>), Messages.InactiveFAClass, typeof(FAClass.assetCD))]
[PXUIField(DisplayName = "Asset Class", Visibility = PXUIVisibility.Visible)]
public virtual int? ClassID { get; set; }

Vários restritores podem ser usados ​​com um atributo seletor. Nesse caso, todas as condições adicionais do restritor são aplicadas em uma ordem não determinada. Uma vez que qualquer condição é violada, a mensagem de erro apropriada é gerada.

A condição Where<> do próprio seletor é aplicada após todas as condições do restritor.

[PXDefault]
//  An aggregate attribute containing the selector inside.
// -
[ContractTemplate(Required = true)]
[PXRestrictor(typeof(Where<ContractTemplate.status, Equal<Contract.status.active>>), Messages.TemplateIsNotActivated, typeof(ContractTemplate.contractCD))]
[PXRestrictor(typeof(Where<ContractTemplate.effectiveFrom, LessEqual<Current<AccessInfo.businessDate>>, 
    Or<ContractTemplate.effectiveFrom, IsNull>>), Messages.TemplateIsNotStarted)]
[PXRestrictor(typeof(Where<ContractTemplate.discontinueAfter, GreaterEqual<Current<AccessInfo.businessDate>>, 
    Or<ContractTemplate.discontinueAfter, IsNull>>), Messages.TemplateIsExpired)]
public virtual int? TemplateID { get; set; }

Opções #

O construtor de PXRestrictorAttribute recebe três parâmetros:

  1. Condição adicional do restritor. Este tipo BQL deve implementar a interface IBqlWhere.
  2. A mensagem de erro apropriada. A mensagem pode conter elementos de formato (colchetes) para mostrar o contexto. A mensagem deve ser uma constante de string definida em uma classe estática localizável (como PX.Objects.GL.Messages).
  3. Uma matriz de tipos de campo. Esses campos devem pertencer ao objeto atual e devem implementar a interface IBqlField. Os valores dos campos serão usados ​​para formatação da mensagem de erro.

Além disso, existem várias opções que especificam o comportamento do restritor.

Substituindo Restritores Herdados #

A propriedade ReplaceInherited indica se o restritor atual deve substituir os restritores herdados. Se esta propriedade for definida como verdadeira, todos os restritores herdados (colocados em qualquer atributo agregado ou atributo base) serão substituídos.

Substituindo restritores herdados:

 [CustomerActive(Visibility = PXUIVisibility.SelectorVisible, Filterable = true, TabOrder = 2)]
 [PXRestrictor(typeof(Where<Customer.status, Equal<CR.BAccount.status.active>,
    Or<Customer.status, Equal<CR.BAccount.status.oneTime>,
    Or<Customer.status, Equal<CR.BAccount.status.hold>,
    Or<Customer.status, Equal<CR.BAccount.status.creditHold>>>>>), Messages.CustomerIsInStatus, typeof(Customer.status), 
    ReplaceInherited = true)] // Replaced all restrictors from CustomerActiveAttribute
[PXUIField(DisplayName = "Customer")]
[PXDefault()]
public override int? CustomerID { get; set; }

Observe que não recomendamos que você use a propriedade ReplaceInherited no código do aplicativo quando existirem alternativas razoáveis. Esta propriedade destina-se principalmente a ser usada em personalizações.

Cache Global #

CacheGlobal suporta a funcionalidade de dicionário global da mesma forma que em PXSelectorAttribute.

Recomendações para uso #

Use apenas condições restritas #

Quando restritores e um seletor são usados ​​juntos, este último não deve conter a cláusula IBqlWhere. Idealmente, todas as condições devem ser movidas para restritores. Essa abordagem fornece mensagens de erro mais fáceis de usar e elimina erros retroativos desnecessários.

Um exemplo ideal:

[PXDBString(5, IsFixed = true, IsUnicode = false)]
[PXUIField(DisplayName = "Type", Required = true)]
[PXSelector(typeof(EPActivityType.type), DescriptionField = typeof(EPActivityType.description))]
[PXRestrictor(typeof(Where<EPActivityType.active, Equal<True>>), Messages.InactiveActivityType, typeof(EPActivityType.type))]
[PXRestrictor(typeof(Where<EPActivityType.isInternal, Equal<True>>), Messages.ExternalActivityType, typeof(EPActivityType.type))]
public virtual string Type { get; set; }

Possíveis erros retroativos:

[PXDBInt]
[PXUIField(DisplayName = "Contract")]
[PXSelector(typeof(Search2<Contract.contractID,
    LeftJoin<ContractBillingSchedule, On<Contract.contractID, Equal<ContractBillingSchedule.contractID>>>,
    Where<Contract.isTemplate, NotEqual<True>,
        And<Contract.baseType, Equal<Contract.ContractBaseType>,
        And<Where<Current<CRCase.customerID>, IsNull,
            Or2<Where<Contract.customerID, Equal<Current<CRCase.customerID>>,
                And<Current<CRCase.locationID>, IsNull>>,
            Or2<Where<ContractBillingSchedule.accountID, Equal<Current<CRCase.customerID>>,
                And<Current<CRCase.locationID>, IsNull>>,
            Or2<Where<Contract.customerID, Equal<Current<CRCase.customerID>>,
                And<Contract.locationID, Equal<Current<CRCase.locationID>>>>,
            Or<Where<ContractBillingSchedule.accountID, Equal<Current<CRCase.customerID>>,
                And<ContractBillingSchedule.locationID, Equal<Current<CRCase.locationID>>>>>>>>>>>>,
    OrderBy<Desc<Contract.contractCD>>>),
    DescriptionField = typeof(Contract.description),
    SubstituteKey = typeof(Contract.contractCD), Filterable = true)]
[PXRestrictor(typeof(Where<Contract.status, Equal<Contract.status.active>>), Messages.ContractIsNotActive)]
[PXRestrictor(typeof(Where<Current<AccessInfo.businessDate>, LessEqual<Contract.graceDate>, Or<Contract.expireDate, IsNull>>), Messages.ContractExpired)]
[PXRestrictor(typeof(Where<Current<AccessInfo.businessDate>, GreaterEqual<Contract.startDate>>), Messages.ContractActivationDateInFuture, typeof(Contract.startDate))]       
[PXFormula(typeof(Default<CRCase.customerID>))]
[PXDefault(PersistingCheck = PXPersistingCheck.Nothing)]
public virtual int? ContractID { get; set; }
