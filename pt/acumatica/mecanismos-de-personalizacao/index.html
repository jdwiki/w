Usando CacheAttached para substituir atributos DAC no gráfico #

Às vezes, você precisa substituir um ou mais atributos de um determinado campo Data Access Class (DAC) apenas para uma tela específica, sem alterar o comportamento existente para outras telas.

Substituindo todos os atributos #

Suponha que os atributos originais do campo DAC sejam declarados conforme mostrado abaixo:

public class ARInvoice : IBqlTable
{
    [PXDBDecimal(4)]
    [PXDefault(TypeCode.Decimal, "0.0")]
    [PXUIField(DisplayName = "Commission Amount")]
    public virtual Decimal? CommnAmt 
    { 
        get; 
        set; 
    }
}

A maneira básica de substituir os atributos do campo no gráfico é declarar um manipulador de eventos CacheAttached no gráfico que segue a convenção padrão para nomear eventos de gráfico (observe a ausência do argumento EventArgs). O corpo do manipulador de eventos não será executado, mas quaisquer atributos que você colocar no manipulador substituirão os atributos no campo DAC correspondente:

[PXDBDecimal(4)]
[PXDefault(TypeCode.Decimal, "0.0")]
[PXUIField(DisplayName = "Commission Amount")]
[PXAdditionalAttribute(NecessaryProperty = true)]
protected virtual void ARInvoice_CommnAmt_CacheAttached(PXCache sender) { }

Anexando um novo atributo ao campo DAC #

O conjunto de atributos colocados no manipulador CacheAttached redefinirá todo o conjunto de atributos colocados no campo no DAC. Isso é quase sempre um exagero; observe como no exemplo anterior, para adicionar apenas um único atributo ao campo, você precisava copiar todas as outras declarações de atributo do DAC. Isso leva à duplicação de código indesejada, bem como a possibilidade de DAC e o gráfico ficarem fora de sincronia. É muito fácil imaginar uma situação em que alguém altera a lógica padrão de, por exemplo, PXDefaultAttribute no DAC, mas esquece de atualizar todos os atributos correspondentes colocados nos manipuladores CacheAttached em vários gráficos.

Para remediar este problema, o Acumatica Framework fornece um atributo especial chamado PXMergeAttributesAttribute. Quando este atributo é colocado em um manipulador CacheAttached, você pode reutilizar os atributos existentes definidos no DAC.

Anexando um atributo usando PXMergeAttributesAttribute:

[PXMergeAttributes(Method = MergeMethod.Append)]
[PXAdditionalAttribute(NecessaryProperty = true)]
protected virtual void ARInvoice_CommnAmt_CacheAttached(PXCache sender) { }

No exemplo acima, todo o conjunto de atributos do DAC original será reutilizado, anexado por quaisquer atributos que você tenha declarado no manipulador de eventos CacheAttached.

PXMergeAttributesAttribute tem outros comportamentos de mesclagem, de acordo com os seguintes valores possíveis para a propriedade Method:

  • MergeMethod.Replace substitui completamente os atributos do DAC (equivalente à ausência de PXMergeAttributesAttribute).
  • MergeMethod.Append anexa os atributos do manipulador CacheAttached aos atributos originais do DAC.
  • MergeMethod.Merge é semelhante a Append; no entanto, ele também verifica se há algum atributo conflitante entre os atributos do manipulador e os atributos do campo DAC. Se houver um conflito, o atributo CacheAttached terá precedência e o atributo DAC correspondente será descartado.

Substituindo uma única propriedade de um atributo #

Um cenário de desenvolvimento de aplicativos muito comum ocorre quando você precisa redefinir apenas uma única propriedade do atributo de um DAC para uma tela específica; considere a situação em que você precisa definir a propriedade DisplayName do PXUIFieldAttribute.

Para isso, você pode usar ainda outro atributo especial fornecido pelo Acumatica Framework: PXCustomizeBaseAttributeAttribute. Seu construtor aceita três valores:

  • O tipo do atributo DAC cuja propriedade precisa ser substituída
  • O nome da propriedade do atributo para substituir (use o operador nameof no C# 6.0 para manutenção do código)
  • O novo valor para a propriedade especificada.

Suponha que haja um requisito para alterar o nome de exibição da interface do usuário de Commission Amount para Base Currency Commission para apenas uma tela. O exemplo de código a seguir demonstra como implementar o comportamento desejado.

[PXMergeAttributes(Method = MergeMethod.Append)]
[PXCustomizeBaseAttribute(typeof(PXUIFieldAttribute), nameof(PXUIFieldAttribute.DisplayName), "Base Currency Commission")]
protected virtual void ARInvoice_CommnAmt_CacheAttached(PXCache sender) { }

Neste exemplo, PXMergeAttributes garante que os atributos originais do DAC sejam preservados, e PXCustomizeBaseAttribute permite que o engenheiro de software substitua o nome de exibição do campo da interface do usuário para o gráfico em questão.

Substituindo um atributo por outro atributo #

Suponha que haja um requisito para substituir o PXDefaultAttribute de um campo DAC por PXDBDefaultAttribute para apenas uma tela.

Isso pode ser conseguido da seguinte maneira:

[PXMergeAttributes(Method = MergeMethod.Append)]
[PXRemoveBaseAttribute(typeof(PXDefaultAttribute))]
[PXDBDefault(typeof(SOShipment.siteID), PersistingCheck = PXPersistingCheck.Nothing)]
protected void SOOrderShipment_SiteID_CacheAttached(PXCache sender) { }

Ordem de aplicação dos atributos de customização de atributos #

  1. PXCustomizeBaseAttribute
  2. PXRemoveBaseAttribute
  3. PXMergeAttributes
