Spring Securitiy para proteger os endpoints da API REST #

Adicione as entradas abaixo em pom.xml.

<dependency>
    <groupId>org.springframework.security</groupId>
    <artifactId>spring-security-web</artifactId>
    <version>3.1.0.RELEASE</version>
</dependency>
<dependency>
    <groupId>org.springframework.security</groupId>
    <artifactId>spring-security-config</artifactId>
    <version>3.1.0.RELEASE</version>
</dependency>

Importante para a versão do Spring superior a 3.1:

Erro de criação de bean para org.springframework.security.filterChains ocorre quando você está usando Spring versão superior a 3.1 e não adicionou dependências manualmente para spring-aop, spring-jdbc, spring-tx e spring -expressions em seu pom.xml.

Adicione as entradas abaixo no contexto do Spring. Queremos proteger dois endpoints REST (helloworld e adeus). Ajuste a versão XSD de acordo com a versão Spring.

<?xml version="1.0" encoding="UTF-8"?>
<beans xmlns="http://www.springframework.org/schema/beans"
   xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
   xmlns:security="http://www.springframework.org/schema/security"
   xmlns:context="http://www.springframework.org/schema/context"
   xsi:schemaLocation="http://www.springframework.org/schema/beans http://www.springframework.org/schema/beans/spring-beans-3.1.xsd
                       http://www.springframework.org/schema/context http://www.springframework.org/schema/context/spring-context-3.1.xsd
                       http://www.springframework.org/schema/security
                       http://www.springframework.org/schema/security/spring-security-3.1.xsd">
                               
    <security:http auto-config='true' create-session="never">
        <security:intercept-url pattern="/helloworld/**" access="ROLE_USER" />
        <security:intercept-url pattern="/goodbye/**" access="ROLE_ADMIN" />
        <security:intercept-url pattern="/**" access="IS_AUTHENTICATED_ANONYMOUSLY" />
        <security:http-basic />
    </security:http>

    <security:authentication-manager>
        <security:authentication-provider>
            <security:user-service>
                <security:user name="username1" password="password1"
                    authorities="ROLE_USER" />
                <security:user name="username2" password="password2"
                    authorities="ROLE_ADMIN" />
            </security:user-service>
        </security:authentication-provider>
    </security:authentication-manager>
</beans>

Adicione as entradas abaixo em web.xml.

<!-- Spring security-->
<filter>
    <filter-name>springSecurityFilterChain</filter-name>
    <filter-class>org.springframework.web.filter.DelegatingFilterProxy</filter-class>
</filter>
<filter-mapping>
    <filter-name>springSecurityFilterChain</filter-name>
    <url-pattern>/*</url-pattern>
</filter-mapping>

<listener>
    <listener-class>org.springframework.web.context.ContextLoaderListener</listener-class>
</listener>

<context-param>
    <param-name>contextConfigLocation</param-name>
    <param-value>classpath:security-context.xml</param-value>
</context-param>

Spring-Security usando spring-boot e autenticação JDBC #

Suponha que você queira impedir que usuários não autorizados acessem a página, então você deve colocar uma barreira para eles autorizando o acesso. Podemos fazer isso usando spring-security, que fornece autenticação básica, protegendo todos os terminais HTTP. Para isso você precisa adicionar a dependência spring-security ao seu projeto, no maven podemos adicionar a dependência como:

<dependency>
    <groupId>org.springframework.boot</groupId>
    <artifactId>spring-boot-starter-security</artifactId>
</dependency>

Aqui está uma configuração de segurança que garante que apenas usuários autenticados possam acessar.

@Configuration
@Order(SecurityProperties.ACCESS_OVERRIDE_ORDER)
public class SecurityConfig extends WebSecurityConfigurerAdapter {

@Autowired
DataSource datasource;

@Override
protected void configure(HttpSecurity http) throws Exception {
    http
        .authorizeRequests()
            .anyRequest()
            .fullyAuthenticated()
            .and()
        .formLogin()
            .loginPage("/login")
            .failureUrl("/login?error")
            .permitAll()
            .and()
        .logout()
            .logoutUrl("/logout")
            .logoutSuccessUrl("/login?logout")
            .permitAll()
            .and()
        .csrf();
}

@Override
protected void configure(AuthenticationManagerBuilder auth) throws Exception {
    auth.jdbcAuthentication().dataSource(datasource).passwordEncoder(passwordEncoder());
}

@Bean
public PasswordEncoder passwordEncoder() {
    PasswordEncoder encoder = new BCryptPasswordEncoder();
    return encoder;
}
}

Configuração | Descrição ——————|——————————– ———————– @Configuration |Indica que a classe pode ser usada pelo container Spring IoC como fonte de definições de bean. @Order (SecurityProperties.ACCESS_OVERRIDE_ORDER)|Ignora as regras de acesso sem alterar nenhum outro recurso configurado automaticamente. Valores mais baixos têm prioridade mais alta. WebSecurityConfigurerAdapter|A classe SecurityConfig estende e sobrescreve alguns de seus métodos para definir algumas especificidades da configuração de segurança. @Autowired de DataSource|Fornecer fábrica para conexões com a fonte de dados física. configure(HttpSecurity) |O método substituído define quais caminhos de URL devem ser protegidos e quais não devem. .authorizeRequests().anyRequest() .fullyAuthenticated()|Indica que todas as requisições para nossa aplicação precisam ser autenticadas. .formLogin()|Configura um login baseado em formulário .loginPage("/login").failureUrl("/login?error").permitAll()|Especifica a localização da página de login e todos os usuários devem ter permissão para acessar a página. .logout().logoutUrl("/logout") .logoutSuccessUrl("/login?logout").permitAll()|A URL para redirecionar após a ocorrência do logout. O padrão é /login?logout. .csrf()|Usado para evitar a falsificação de solicitação entre sites, a proteção CSRF está habilitada (padrão). configure(AuthenticationManagerBuilder){}|Método substituído para definir como os usuários são autenticados. .jdbcAuthentication().dataSource(datasource)|Indica ao Spring que estamos usando autenticação JDBC .passwordEncoder(passwordEncoder())|Indica que estamos usando um codificador de senha para codificar nossas senhas. (É criado um bean para retornar a escolha da senha do Codificador, estamos utilizando BCrypt neste caso) Observe que não configuramos nenhum nome de tabela a ser usado ou nenhuma consulta, isso ocorre porque o spring security por padrão procura as tabelas abaixo:

create table users (   
  username varchar(50) not null primary key,
  password varchar(255) not null,
  enabled boolean not null) ;

create table authorities (
  username varchar(50) not null,
  authority varchar(50) not null,
  foreign key (username) references users (username),
  unique index authorities_idx_1 (username, authority));

Insira as seguintes linhas nas tabelas acima:

INSERT INTO authorities(username,authority) 
VALUES ('user', 'ROLE_ADMIN');

INSERT INTO users(username,password,enabled)
VALUES('user', '$2a$10$JvqOtJaDys0yoXPX9w47YOqu9wZr/PkN1dJqjG9HHAzMyu9EV1R4m', '1');

O nome de usuário em nosso caso é usuário e a senha também é usuário criptografada com o algoritmo BCrypt

Por fim, configure uma fonte de dados no application.properties para usar a inicialização por mola:

spring.datasource.url = jdbc:mysql://localhost:3306/spring
spring.datasource.username = root
spring.datasource.password = Welcome123

Observação: Crie e configure um controlador de login e mapeie-o para o caminho /login e aponte sua página de login para este controlador

Olá Spring Security #

Nota 1: Você precisa de algum conhecimento prévio sobre servlet java page(JSP) e Apache Maven antes de iniciar este exemplos.

Inicie o servidor web (como Apache tomcat) com um projeto web existente ou crie um.

Visite o index.jsp.

Qualquer um pode acessar essa página, é inseguro!

Protegendo o aplicativo #

  1. Atualize as dependências do Maven

Adicionando dependências ao seu arquivo pom.xml

pom.xml

<dependency>
  <groupId>org.springframework.security</groupId>
  <artifactId>spring-security-web</artifactId>
  <version>4.0.1.RELEASE</version>
</dependency>
<dependency>
  <groupId>org.springframework.security</groupId>
  <artifactId>spring-security-config</artifactId>
  <version>4.0.1.RELEASE</version>
</dependency>

Observação 1: se você não estiver usando “Spring” em seu projeto antes, não há dependência sobre “spring-context”. Este exemplo usará a configuração xml com “spring-context”. Então adicione essa dependência também.

<dependência> org.springframework contexto-primavera 4.2.2.RELEASE </dependência>

Observação 2: Se você não estiver usando JSTL em seu projeto antes, não há dependência sobre isso. Este exemplo usará JSTL na página jsp. Então adicione essa dependência também.

<dependência> org.glassfish.web javax.servlet.jsp.jstl 1.2.1 </dependência>

  1. Crie o arquivo de configuração do Spring Security

Faça o nome da pasta “spring” dentro da pasta “WEB-INF” e crie o arquivo security.xml. Copie e cole dos próximos códigos.

WEB-INF/spring/security.xml

<b:beans xmlns="http://www.springframework.org/schema/security"
         xmlns:b="http://www.springframework.org/schema/beans"
         xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
         xsi:schemaLocation="http://www.springframework.org/schema/beans http://www.springframework.org/schema/beans/spring-beans.xsd
                             http://www.springframework.org/schema/security http://www.springframework.org/schema/security/spring-security.xsd">

  <http />

  <user-service>
    <user name="stackoverflow" password="pwd" authorities="ROLE_USER" />
  </user-service>

</b:beans>
  1. Atualize web.xml

Atualize seu web.xml dentro da pasta “WEB-INF”

WEB-INF/web.xml

<filter>
  <filter-name>springSecurityFilterChain</filter-name>
  <filter-class>org.springframework.web.filter.DelegatingFilterProxy</filter-class>
</filter>
<filter-mapping>
  <filter-name>springSecurityFilterChain</filter-name>
  <url-pattern>/*</url-pattern>
</filter-mapping>

Observação: se você não estiver usando “Spring” em seu projeto antes, não há configurações sobre carregamento de contextos Spring. Então adicione este parâmetro e ouvinte também.

contextConfigLocation /WEB-INF/spring/*.xml org.springframework.web.context.ContextLoaderListener

Executando o aplicativo Web seguro #

Depois de executar seu servidor web e visitar index.jsp, você verá a página de login padrão gerada pelo spring security. Porque você não está autenticado.

Você pode entrar

username : stackoverflow
password : pwd

Observação: configuração de nome de usuário e senha em WEB-INF/spring/security.xml

Exibindo nome de usuário #

Adicionando a tag jstl após o “Hello”, que imprime o nome de usuário

index.jsp

<h1>Hello <c:out value="${pageContext.request.remoteUser}" />!!</h1>

Sair #

index.jsp

Adicionando formulário, tags de entrada após “Hello user name”, que o envio gerou o url de logout /logout do spring security.

<h1>Hello <c:out value="${pageContext.request.remoteUser}" />!!</h1>

<form action="/logout" method="post">
  <input type="submit" value="Log out" />
  <input type="hidden" name="${_csrf.parameterName}" value="${_csrf.token}" />
</form>

Ao sair com sucesso, você verá a página de login gerada automaticamente novamente. Porque você não está autenticado agora.
