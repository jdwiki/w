Use Option<‘T> sobre valores nulos #

Em linguagens de programação funcionais como F# null valores são considerados potencialmente prejudicial e estilo pobre (não idiomático).

Considere este código C#:

string x = SomeFunction ();
int    l = x.Length;

x.Length será lançado se x for null vamos adicionar proteção:

string x = SomeFunction ();
int    l = x != null ? x.Length : 0;

Ou:

string x = SomeFunction () ?? "";
int    l = x.Length;

Ou:

string x = SomeFunction ();
int    l = x?.Length;

Em valores idiomáticos F# null não são usados, então nosso código fica assim:

let x = SomeFunction ()
let l = x.Length

No entanto, às vezes é necessário representar valores vazios ou inválidos. Então podemos usar Option<'T>:

let SomeFunction () : string option = ...

SomeFunction retorna o valor Some string ou None. Extraímos o valor string usando correspondência de padrões

let v =
  match SomeFunction () with
  | Some x  -> x.Length
  | None    -> 0

A razão pela qual este código é menos frágil do que:

string x = SomeFunction ();
int    l = x.Length;

É porque não podemos chamar Length em uma string option. Precisamos extrair o valor string usando correspondência de padrões e, ao fazer isso, garantimos que o O valor string é seguro de usar.

O módulo opcional permite a programação orientada para ferrovias #

O tratamento de erros é importante, mas pode transformar um algoritmo elegante em uma bagunça. Programação Orientada para Ferrovia (ROP) é usado para tornar o tratamento de erros elegante e combinável.

Considere a função simples f:

let tryParse s =
  let b, v = System.Int32.TryParse s
  if b then Some v else None

let f (g : string option) : float option =
  match g with
  | None    -> None
  | Some s  ->
    match tryParse s with           // Parses string to int
    | None              -> None
    | Some v when v < 0 -> None     // Checks that int is greater than 0
    | Some v -> v |> float |> Some  // Maps int to float

O propósito de f é analisar o valor de entrada string (se houver Some) em um int. Se o int for maior que 0, nós o convertemos em um float. Em tudo em outros casos, saímos com None.

Embora, uma função extremamente simples, o match aninhado, diminua a legibilidade significativamente.

ROP observa que temos dois tipos de caminhos de execução em nosso programa

  1. Caminho feliz - eventualmente calculará o valor “Algum”
  2. Caminho de erro - Todos os outros caminhos produzem ‘Nenhum’

Como os caminhos de erro são mais frequentes, eles tendem a assumir o controle do código. Gostaríamos que o código de caminho feliz fosse o caminho de código mais visível.

Uma função equivalente g usando ROP poderia ser assim:

let g (v : string option) : float option =
  v
  |> Option.bind    tryParse  // Parses string to int
  |> Option.filter  ((<) 0)   // Checks that int is greater than 0
  |> Option.map     float     // Maps int to float

Parece muito com a forma como tendemos a processar listas e sequências em F#.

Pode-se ver uma Option<'T> como uma List<'T> que só pode conter o elemento 0 ou 1 onde Option.bind se comporta como List.pick (conceitualmente Option.bind mapeia melhor para List.collect mas List.pick pode ser mais fácil de entender).

bind, filter e map tratam dos caminhos de erro e g contém apenas o feliz código do caminho.

Todas as funções que aceitam diretamente Option<_> e retornam Option<_> são diretamente combinável com |> e >>.

ROP, portanto, aumenta a legibilidade e a composição.

Definição de Opção #

Uma Option é uma união discriminada com dois casos, None ou Some.

type Option<'T> = Some of 'T | None

Usando tipos de opções de C# #

Não é uma boa ideia expor os tipos Option ao código C#, pois o C# não tem uma maneira de lidar com eles. As opções são introduzir FSharp.Core como uma dependência em seu projeto C# (que é o que você teria que fazer se estiver consumindo uma biblioteca F# não projetada para interoperabilidade com C#), ou alterar None valores para null.

Pré-F# 4.0 #

A maneira de fazer isso é criar uma função de conversão própria:

let OptionToObject opt =
    match opt with
    | Some x -> x 
    | None -> null

Para tipos de valor você teria que recorrer a boxing ou usar System.Nullable.

let OptionToNullable x = 
    match x with 
    | Some i -> System.Nullable i
    | None -> System.Nullable ()

#F# 4.0

No F# 4.0, as funções ofObj, toObj, ofNullable e toNullable foram introduzidas no módulo Option. Em F# interativo eles podem ser usados ​​da seguinte forma:

let l1 = [ Some 1 ; None ; Some 2]
let l2 = l1 |> List.map Option.toNullable;;

// val l1 : int option list = [Some 1; null; Some 2]
// val l2 : System.Nullable<int> list = [1; null; 2]

let l3 = l2 |> List.map Option.ofNullable;;
// val l3 : int option list = [Some 1; null; Some 2]

// Equality
l1 = l2 // fails to compile: different types
l1 = l3 // true

Observe que None compila para null internamente. No entanto, no que diz respeito ao F#, é um ‘Nenhum’.

let lA = [Some "a"; None; Some "b"]
let lB = lA |> List.map Option.toObj

// val lA : string option list = [Some "a"; null; Some "b"]
// val lB : string list = ["a"; null; "b"]

let lC = lB |> List.map Option.ofObj
// val lC : string option list = [Some "a"; null; Some "b"]

// Equality
lA = lB // fails to compile: different types
lA = lC // true
