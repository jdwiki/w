Introdução aos tipos #

Tipos podem representar vários tipos de coisas. Pode ser um único dado, um conjunto de dados ou uma função.

Em F#, podemos agrupar os tipos em duas categorias.:

  • Tipos de F#:

      // Functions
  let a = fun c -> c

  // Tuples
  let b = (0, "Foo")

  // Unit type
  let c = ignore

  // Records
  type r = { Name : string; Age : int }
  let d = { Name = "Foo"; Age = 10 }

  // Discriminated Unions
  type du = | Foo | Bar
  let e = Bar

  // List and seq
  let f = [ 0..10 ]
  let g = seq { 0..10 }

  // Aliases
  type MyAlias = string

  • Tipos .NET

    • Built-in type (int, bool, string,…)
    • Classes, Structs & Interfaces
    • Delegates
    • Arrays

Abreviações de tipos #

As abreviações de tipo permitem que você crie aliases em tipos existentes para dar-lhes um sentido mais significativo.

// Name is an alias for a string
type Name = string

// PhoneNumber is an alias for a string
type PhoneNumber = string

Então você pode usar o alias como qualquer outro tipo:

// Create a record type with the alias
type Contact = {
    Name : Name
    Phone : PhoneNumber }

// Create a record instance
// We can assign a string since Name and PhoneNumber are just aliases on string type
let c = {
    Name = "Foo"
    Phone = "00 000 000" }

printfn "%A" c

// Output
// {Name = "Foo";
// Phone = "00 000 000";}

Tenha cuidado, os aliases não verificam a consistência do tipo. Isso significa que dois aliases que visam o mesmo tipo podem ser atribuídos um ao outro:

let c = {
    Name = "Foo"
    Phone = "00 000 000" }
let d = {
    Name = c.Phone
    Phone = c.Name }

printfn "%A" d

// Output
// {Name = "00 000 000";
// Phone = "Foo";}

Os tipos são criados em F# usando a palavra-chave type #

F# usa a palavra-chave type para criar diferentes tipos de tipos.

  1. Digite aliases
  2. Tipos de sindicatos discriminados
  3. Tipos de registro
  4. Tipos de interface
  5. Tipos de aula
  6. Tipos de estrutura

Exemplos com código C# equivalente sempre que possível:

// Equivalent C#:
//  using IntAliasType = System.Int32;
type IntAliasType = int // As in C# this doesn't create a new type, merely an alias

type DiscriminatedUnionType =
  | FirstCase
  | SecondCase  of int*string

  member x.SomeProperty = // We can add members to DU:s
    match x with
    | FirstCase         -> 0
    | SecondCase (i, _) -> i

type RecordType =
  {
    Id    : int
    Name  : string
  }
  static member New id name : RecordType = // We can add members to records
    { Id = id; Name = name } // { ... } syntax used to create records

// Equivalent C#:
//  interface InterfaceType
//  {
//    int     Id    { get; }
//    string  Name  { get; }
//    int Increment (int i);
//  }  
type InterfaceType =
  interface // In order to create an interface type, can also use [<Interface>] attribute
    abstract member Id        : int
    abstract member Name      : string
    abstract member Increment : int -> int
  end

// Equivalent C#:
//  class ClassType : InterfaceType
//  {
//    static int increment (int i)
//    {
//      return i + 1;
//    }
//  
//    public ClassType (int id, string name)
//    {
//      Id    = id    ;
//      Name  = name  ;
//    }
//  
//    public int    Id    { get; private set; }
//    public string Name  { get; private set; }
//    public int   Increment (int i)
//    {
//      return increment (i);
//    }
//  }
type ClassType (id : int, name : string) = // a class type requires a primary constructor
  let increment i = i + 1 // Private helper functions

  interface InterfaceType with // Implements InterfaceType
    member x.Id           = id
    member x.Name         = name
    member x.Increment i  = increment i


// Equivalent C#:
//  class SubClassType : ClassType
//  {
//    public SubClassType (int id, string name) : base(id, name)
//    {
//    }
//  }
type SubClassType (id : int, name : string) =
  inherit ClassType (id, name) // Inherits ClassType

// Equivalent C#:
//  struct StructType
//  {
//    public StructType (int id)
//    {
//      Id = id;
//    }
//  
//    public int Id { get; private set; }
//  }
type StructType (id : int) =
  struct  // In order create a struct type, can also use [<Struct>] attribute
    member x.Id = id
  end

Inferência de tipos #

Reconhecimento

Este exemplo é adaptado deste artigo sobre type inference

O que é inferência de tipo?

Type Inference é o mecanismo que permite ao compilador deduzir quais tipos são usados ​​e onde. Este mecanismo é baseado em um algoritmo frequentemente chamado de “Hindley-Milner” ou “HM”. Veja abaixo algumas das regras para determinar os tipos de valores simples e de função:

  • Olhe para os literais
  • Olhe para as funções e outros valores com os quais algo interage
  • Veja quaisquer restrições de tipo explícitas
  • Se não houver restrições em nenhum lugar, generalize automaticamente para tipos genéricos

Veja os literais

O compilador pode deduzir tipos observando os literais. Se o literal é um int e você está adicionando “x” a ele, então “x” também deve ser um int. Mas se o literal é um float e você está adicionando “x” a ele, então “x” deve ser um float também.

aqui estão alguns exemplos:

let inferInt x = x + 1
let inferFloat x = x + 1.0
let inferDecimal x = x + 1m     // m suffix means decimal
let inferSByte x = x + 1y       // y suffix means signed byte
let inferChar x = x + 'a'       // a char
let inferString x = x + "my string"

Veja as funções e outros valores com os quais interage

Se não houver literais em nenhum lugar, o compilador tenta descobrir os tipos analisando as funções e outros valores com os quais eles interagem.

let inferInt x = x + 1
let inferIndirectInt x = inferInt x       //deduce that x is an int

let inferFloat x = x + 1.0
let inferIndirectFloat x = inferFloat x   //deduce that x is a float

let x = 1
let y = x     //deduce that y is also an int

Observe quaisquer restrições ou anotações de tipo explícitas

Se houver restrições de tipo explícitas ou anotações especificadas, o compilador as usará.

let inferInt2 (x:int) = x                // Take int as parameter
let inferIndirectInt2 x = inferInt2 x    // Deduce from previous that x is int

let inferFloat2 (x:float) = x                // Take float as parameter
let inferIndirectFloat2 x = inferFloat2 x    // Deduce from previous that x is float

Generalização automática

Se depois de tudo isso, não houver restrições encontradas, o compilador apenas torna os tipos genéricos.

let inferGeneric x = x 
let inferIndirectGeneric x = inferGeneric x 
let inferIndirectGenericAgain x = (inferIndirectGeneric x).ToString()

Coisas que podem dar errado com a inferência de tipo

A inferência de tipo não é perfeita, infelizmente. Às vezes, o compilador simplesmente não tem ideia do que fazer. Novamente, entender o que está acontecendo realmente o ajudará a ficar calmo em vez de querer matar o compilador. Aqui estão algumas das principais razões para erros de tipo:

  • Declarações fora de ordem
  • Não é informação suficiente
  • Métodos sobrecarregados

Declarações fora de ordem

Uma regra básica é que você deve declarar funções antes de serem usadas.

Este código falha:

let square2 x = square x   // fails: square not defined
let square x = x * x

Mas tudo bem:

let square x = x * x       
let square2 x = square x   // square already defined earlier

Declarações recursivas ou simultâneas

Uma variante do problema “fora de ordem” ocorre com funções ou definições recursivas que precisam se referir umas às outras. Nenhuma quantidade de reordenação ajudará neste caso – precisamos usar palavras-chave adicionais para ajudar o compilador.

Quando uma função está sendo compilada, o identificador da função não está disponível para o corpo. Portanto, se você definir uma função recursiva simples, receberá um erro do compilador. A correção é adicionar a palavra-chave “rec” como parte da definição da função. Por exemplo:

// the compiler does not know what "fib" means
let fib n =
   if n <= 2 then 1
   else fib (n - 1) + fib (n - 2)
   // error FS0039: The value or constructor 'fib' is not defined

Aqui está a versão corrigida com “rec fib” adicionado para indicar que é recursiva:

let rec fib n =              // LET REC rather than LET
   if n <= 2 then 1
   else fib (n - 1) + fib (n - 2)

Não é informação suficiente

Às vezes, o compilador simplesmente não tem informações suficientes para determinar um tipo. No exemplo a seguir, o compilador não sabe em que tipo o método Length deve funcionar. Mas também não pode torná-lo genérico, então reclama.

let stringLength s = s.Length
  // error FS0072: Lookup on object of indeterminate type
  // based on information prior to this program point.
  // A type annotation may be needed ...

Esses tipos de erro podem ser corrigidos com anotações explícitas.

let stringLength (s:string) = s.Length

Métodos sobrecarregados

Ao chamar uma classe ou método externo em .NET, você geralmente receberá erros devido à sobrecarga.

Em muitos casos, como no exemplo de concat abaixo, você terá que anotar explicitamente os parâmetros da função externa para que o compilador saiba qual método sobrecarregado chamar.

let concat x = System.String.Concat(x)           //fails
let concat (x:string) = System.String.Concat(x)  //works
let concat x = System.String.Concat(x:string)    //works

Às vezes, os métodos sobrecarregados têm nomes de argumentos diferentes; nesse caso, você também pode dar uma pista ao compilador nomeando os argumentos. Aqui está um exemplo para o construtor StreamReader.

let makeStreamReader x = new System.IO.StreamReader(x)        //fails
let makeStreamReader x = new System.IO.StreamReader(path=x)   //works
