Nomeando elementos de tuplas dentro de uniões discriminadas #

Ao definir uniões discriminadas, você pode nomear elementos de tipos de tupla e usar esses nomes durante a correspondência de padrões.

type Shape = 
    | Circle of diameter:int
    | Rectangle of width:int * height:int

let shapeIsTenWide = function
    | Circle(diameter=10) 
    | Rectangle(width=10) -> true
    | _ -> false

Além disso, nomear os elementos de uniões discriminadas melhora a legibilidade do código e a interoperabilidade com C# - os nomes fornecidos serão usados ​​para nomes de propriedades e parâmetros de construtores. Os nomes padrão gerados no código de interoperabilidade são “Item”, “Item1”, “Item2”…

Usando uniões discriminadas de caso único como registros #

Às vezes é útil criar tipos de união com apenas um caso para implementar tipos semelhantes a registros:

type Point = Point of float * float

let point1 = Point(0.0, 3.0)

let point2 = Point(-2.5, -4.0)

Eles se tornam muito úteis porque podem ser decompostos por meio de correspondência de padrões da mesma maneira que os argumentos de tupla podem:

let (Point(x1, y1)) = point1
// val x1 : float = 0.0
// val y1 : float = 3.0

let distance (Point(x1,y1)) (Point(x2,y2)) =
    pown (x2-x1) 2 + pown (y2-y1) 2 |> sqrt
// val distance : Point -> Point -> float

distance point1 point2
// val it : float = 7.433034374

Uso básico discriminado da união #

Uniões discriminadas em F# oferecem uma maneira de definir tipos que podem conter qualquer número de tipos de dados diferentes. Sua funcionalidade é semelhante às uniões C++ ou variantes VB, mas com o benefício adicional de ser seguro para tipos.

// define a discriminated union that can hold either a float or a string
type numOrString = 
    | F of float
    | S of string

let str = S "hi" // use the S constructor to create a string
let fl = F 3.5 // use the F constructor to create a float

// you can use pattern matching to deconstruct each type
let whatType x = 
    match x with
        | F f -> printfn "%f is a float" f
        | S s -> printfn "%s is a string" s

whatType str // hi is a string
whatType fl // 3.500000 is a float

Requer acesso qualificado #

Com o atributo RequireQualifiedAccess, os casos de união devem ser referidos como MyUnion.MyCase em vez de apenas MyCase. Isso evita colisões de nomes no namespace ou módulo envolvente:

type [<RequireQualifiedAccess>] Requirements =
    None | Single | All

// Uses the DU with qualified access
let noRequirements = Requirements.None

// Here, None still refers to the standard F# option case
let getNothing () = None

// Compiler error unless All has been defined elsewhere
let invalid = All

Se, por exemplo, System foi aberto, Single se refere a System.Single. Não há colisão com o caso de união Requirements.Single.

Uniões no estilo Enum #

As informações de tipo não precisam ser incluídas nos casos de união discriminada. Ao omitir as informações de tipo, você pode criar uma união que simplesmente representa um conjunto de opções, semelhante a uma enumeração.

// This union can represent any one day of the week but none of 
// them are tied to a specific underlying F# type
type DayOfWeek = Monday | Tuesday | Wednesday | Thursday | Friday | Saturday | Sunday

Convertendo de e para strings com Reflection #

Às vezes é necessário converter uma União Discriminada de e para uma string:

module UnionConversion 
    open Microsoft.FSharp.Reflection
    
    let toString (x: 'a) = 
        match FSharpValue.GetUnionFields(x, typeof<'a>) with
        | case, _ -> case.Name

    let fromString<'a> (s : string) =
        match FSharpType.GetUnionCases typeof<'a> |> Array.filter (fun case -> case.Name = s) with 
        | [|case|] -> Some(FSharpValue.MakeUnion(case, [||])) :?> 'a)
        | _ -> None

União discriminada de caso único #

Uma união discriminada de caso único é como qualquer outra união discriminada, exceto que tem apenas um caso.

// Define single-case discriminated union type.
type OrderId = OrderId of int
// Construct OrderId type.
let order = OrderId 123
// Deconstruct using pattern matching. 
// Parentheses used so compiler doesn't think it is a function definition.   
let (OrderId id) = order

É útil para impor a segurança de tipo e comumente usado em F#, em oposição a C# e Java, onde a criação de novos tipos vem com mais sobrecarga.

As duas definições de tipo alternativo a seguir resultam na mesma união discriminada de caso único sendo declarada:

type OrderId = | OrderId of int

type OrderId =
      | OrderId of int

Sindicatos discriminados recursivos #

Tipo recursivo #

As uniões discriminadas podem ser recursivas, ou seja, podem se referir a si mesmas em sua definição. O principal exemplo aqui é uma árvore:

type Tree =
    | Branch of int * Tree list
    | Leaf of int

Como exemplo, vamos definir a seguinte árvore:

    1
  2   5
3   4

Podemos definir essa árvore usando nossa união discriminada recursiva da seguinte forma:

let leaf1 = Leaf 3
let leaf2 = Leaf 4
let leaf3 = Leaf 5

let branch1 = Branch (2, [leaf1; leaf2])
let tip = Branch (1, [branch1; leaf3])

A iteração sobre a árvore é então apenas uma questão de correspondência de padrões:

let rec toList tree =
    match tree with
    | Leaf x -> [x]
    | Branch (x, xs) -> x :: (List.collect toList xs)

let treeAsList = toList tip // [1; 2; 3; 4; 5]

Tipos recursivos mutuamente dependentes #

Uma maneira de alcançar a recursão é ter tipos mutuamente dependentes aninhados.

// BAD
type Arithmetic = {left: Expression; op:string; right: Expression}
// illegal because until this point, Expression is undefined
type Expression = 
| LiteralExpr of obj
| ArithmeticExpr of Arithmetic

A definição de um tipo de registro diretamente dentro de uma união discriminada está obsoleta:

// BAD
type Expression = 
| LiteralExpr of obj
| ArithmeticExpr of {left: Expression; op:string; right: Expression}
// illegal in recent F# versions

Você pode usar a palavra-chave and para encadear definições mutuamente dependentes:

// GOOD
type Arithmetic = {left: Expression; op:string; right: Expression}
and Expression = 
| LiteralExpr of obj
| ArithmeticExpr of Arithmetic
