Unidades em tempo de execução #

As unidades de medida são usadas apenas para verificação estática pelo compilador e não estão disponíveis em tempo de execução. Eles não podem ser usados ​​em reflexão ou em métodos como ToString.

Por exemplo, C# fornece um double sem unidades para um campo do tipo float<m> definido e exposto a partir de uma biblioteca F#.

Garantir unidades consistentes nos cálculos #

Unidades de medida são anotações de tipo adicionais que podem ser adicionadas a floats ou integers. Eles podem ser usados ​​para verificar em tempo de compilação se os cálculos estão usando unidades de forma consistente.

Para definir anotações:

[<Measure>] type m // meters
[<Measure>] type s // seconds
[<Measure>] type accel = m/s^2 // acceleration defined as meters per second squared

Uma vez definidas, as anotações podem ser usadas para verificar se uma expressão resulta no tipo esperado.

// Compile-time checking that this function will return meters, since (m/s^2) * (s^2) -> m
// Therefore we know units were used properly in the calculation.
let freeFallDistance (time:float<s>) : float<m> = 
    0.5 * 9.8<accel> * (time*time)    

// It is also made explicit at the call site, so we know that the parameter passed should be in seconds
let dist:float<m> = freeFallDistance 3.0<s>
printfn "%f" dist

Conversões entre unidades #

[<Measure>] type m // meters
[<Measure>] type cm // centimeters

// Conversion factor
let cmInM = 100<cm/m>

let distanceInM = 1<m>
let distanceInCM = distanceInM * cmInM // 100<cm>

// Conversion function
let cmToM (x : int<cm>) = x / 100<cm/m>
let mToCm (x : int<m>) = x * 100<cm/m>

cmToM 100<cm> // 1<m>
mToCm 1<m> // 100<cm>

Observe que o compilador F# não sabe que 1<m> é igual a 100<cm>. Tanto quanto se importa, as unidades são tipos separados. Você pode escrever funções semelhantes para converter de metros para quilogramas, e o compilador não se importaria.

[<Measure>] type kg

// Valid code, invalid physics
let kgToM x = x / 100<kg/m>

Não é possível definir unidades de medida como múltiplos de outras unidades, como

// Invalid code
[<Measure>] type m = 100<cm>

No entanto, definir unidades “por algo”, por exemplo Hertz, medindo frequência, é simplesmente “por segundo”, é bastante simples.

// Valid code
[<Measure>] type s
[<Measure>] type Hz = /s

1 / 1<s> = 1 <Hz> // Evaluates to true

[<Measure>] type N = kg m/s // Newtons, measuring force. Note lack of multiplication sign.

// Usage
let mass = 1<kg>
let distance = 1<m>
let time = 1<s>

let force = mass * distance / time // Evaluates to 1<kg m/s>
force = 1<N> // Evaluates to true

Parâmetros de tipo de unidade de medida #

O atributo [<Measure>] pode ser usado em parâmetros de tipo para declarar tipos que são genéricos em relação às unidades de medida:

type CylinderSize<[<Measure>] 'u> =
    { Radius : float<'u>
      Height : float<'u> }

Uso de teste:

open Microsoft.FSharp.Data.UnitSystems.SI.UnitSymbols

/// This has type CylinderSize<m>.
let testCylinder =
    { Radius = 14.<m>
      Height =  1.<m> }

Usando LanguagePrimitives para preservar ou definir unidades #

Quando uma função não preserva unidades automaticamente devido a operações de nível inferior, o módulo LanguagePrimitives pode ser usado para definir unidades nas primitivas que as suportam:

/// This cast preserves units, while changing the underlying type
let inline castDoubleToSingle (x : float<'u>) : float32<'u> =
    LanguagePrimitives.Float32WithMeasure (float32 x)

Para atribuir unidades de medida a um valor de ponto flutuante de precisão dupla, basta multiplicar por um com as unidades corretas:

[<Measure>]
type USD

let toMoneyImprecise (amount : float) =
   amount * 1.<USD>

Para atribuir unidades de medida a um valor sem unidade que não seja System.Double, por exemplo, vindo de uma biblioteca escrita em outro idioma, use uma conversão:

open LanguagePrimitives

let toMoney amount =
   amount |> DecimalWithMeasure<'u>

Aqui estão os tipos de função relatados pelo F# interativo:

val toMoney : amount:decimal -> decimal<'u>
val toMoneyImprecise : amount:float -> float<USD>

Use tipos de unidade padronizados para manter a compatibilidade #

Por exemplo, os tipos para unidades SI foram padronizados na biblioteca principal do F#, em Microsoft.FSharp.Data.UnitSystems.SI. Abra o sub-namespace apropriado, UnitNames ou UnitSymbols, para usá-los. Ou, se forem necessárias apenas algumas unidades SI, elas podem ser importadas com aliases de tipo:

/// Seconds, the SI unit of time. Type abbreviation for the Microsoft standardized type.
type [<Measure>] s = Microsoft.FSharp.Data.UnitSystems.SI.UnitSymbols.s

Alguns usuários tendem a fazer o seguinte, o que não deve ser feito sempre que uma definição já estiver disponível:

/// Seconds, the SI unit of time
type [<Measure>] s // DO NOT DO THIS! THIS IS AN EXAMPLE TO EXPLAIN A PROBLEM.

A diferença se torna aparente ao fazer a interface com outro código que se refere aos tipos padrão do SI. O código que se refere às unidades padrão é compatível, enquanto o código que define seu próprio tipo é incompatível com qualquer código que não use sua definição específica.

Portanto, sempre use os tipos padrão para unidades SI. Não importa se você se refere a UnitNames ou UnitSymbols, pois nomes equivalentes dentro desses dois referem-se ao mesmo tipo:

open Microsoft.FSharp.Data.UnitSystems.SI

/// This is valid, since both versions refer to the same authoritative type.
let validSubtraction = 1.<UnitSymbols.s> - 0.5<UnitNames.second>
