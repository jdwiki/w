Padrões ativos simples #

Padrões ativos são um tipo especial de correspondência de padrões onde você pode especificar categorias nomeadas nas quais seus dados podem se encaixar e, em seguida, usar essas categorias em instruções match.

Para definir um padrão ativo que classifica os números como positivos, negativos ou zero:

let (|Positive|Negative|Zero|) num = 
    if num > 0 then Positive 
    elif num < 0 then Negative
    else Zero

Isso pode ser usado em uma expressão de correspondência de padrões:

let Sign value = 
    match value with
    | Positive -> printf "%d is positive" value
    | Negative -> printf "%d is negative" value
    | Zero -> printf "The value is zero"

Sign -19 // -19 is negative
Sign 2 // 2 is positive
Sign 0 // The value is zero

Padrões Ativos podem ser usados ​​para validar e transformar argumentos de funções #

Um uso interessante, mas bastante desconhecido, de Active Patterns em F# é que eles podem ser usados ​​para validar e transformar argumentos de função.

Considere a maneira clássica de fazer validação de argumento:

// val f : string option -> string option -> string
let f v u =
  let v = defaultArg v "Hello"
  let u = defaultArg u "There"
  v + " " + u

// val g : 'T -> 'T (requires 'T null)
let g v =
  match v with
  | null  -> raise (System.NullReferenceException ())
  | _     -> v.ToString ()

Normalmente adicionamos código no método para verificar se os argumentos estão corretos. Usando Active Patterns em F# podemos generalizar isso e declarar a intenção na declaração do argumento.

O código a seguir é equivalente ao código acima:

let inline (|DefaultArg|) dv ov = defaultArg ov dv

let inline (|NotNull|) v =
  match v with
  | null  -> raise (System.NullReferenceException ())
  | _     -> v

// val f : string option -> string option -> string
let f (DefaultArg "Hello" v) (DefaultArg "There" u) = v + " " + u

// val g : 'T -> string (requires 'T null)
let g (NotNull v) = v.ToString ()

Para o usuário da função f e g não há diferença entre as duas versões diferentes.

printfn "%A" <| f (Some "Test") None  // Prints "Test There"
printfn "%A" <| g "Test"              // Prints "Test"
printfn "%A" <| g null                // Will throw

Uma preocupação é se os padrões ativos adicionam sobrecarga de desempenho. Vamos usar ILSpy para descompilar f e g para ver se esse é o caso.

public static string f(FSharpOption<string> _arg2, FSharpOption<string> _arg1)
{
  return Operators.DefaultArg<string>(_arg2, "Hello") + " " + Operators.DefaultArg<string>(_arg1, "There");
}

public static string g<a>(a _arg1) where a : class
{
  if (_arg1 != null)
  {
    a a = _arg1;
    return a.ToString();
  }
  throw new NullReferenceException();
}

Graças ao inline, os padrões ativos não adicionam sobrecarga extra em comparação com a maneira clássica de fazer validação de argumentos.

Padrões ativos como wrappers de API .NET #

Os padrões ativos podem ser usados ​​para tornar a chamada de algumas APIs .NET mais natural, principalmente aquelas que usam um parâmetro de saída para retornar mais do que apenas o valor de retorno da função.

Por exemplo, você normalmente chamaria o método System.Int32.TryParse da seguinte forma:

let couldParse, parsedInt = System.Int32.TryParse("1")
if couldParse then printfn "Successfully parsed int: %i" parsedInt
else printfn "Could not parse int"

Você pode melhorar isso um pouco usando a correspondência de padrões:

match System.Int32.TryParse("1") with 
| (true, parsedInt) -> printfn "Successfully parsed int: %i" parsedInt
| (false, _) -> printfn "Could not parse int"

No entanto, também podemos definir o seguinte padrão ativo que envolve a função System.Int32.TryParse:

let (|Int|_|) str =
    match System.Int32.TryParse(str) with
    | (true, parsedInt) -> Some parsedInt
    | _ -> None

Agora podemos fazer o seguinte:

match "1" with
| Int parsedInt -> printfn "Successfully parsed int: %i" parsedInt
| _ -> printfn "Could not parse int"

Outro bom candidato para ser envolvido em um Active Patterns são as APIs de expressões regulares:

let (|MatchRegex|_|) pattern input =
    let m = System.Text.RegularExpressions.Regex.Match(input, pattern) 
    if m.Success then Some m.Groups.[1].Value 
    else None  

match "bad" with
| MatchRegex "(good|great)" mood -> 
    printfn "Wow, it's a %s day!" mood
| MatchRegex "(bad|terrible)" mood -> 
    printfn "Unfortunately, it's a %s day." mood
| _ -> 
    printfn "Just a normal day"

Padrões ativos com parâmetros #

Padrões ativos são apenas funções simples.

Como funções, você pode definir parâmetros adicionais:

let (|HasExtension|_|) expected (uri : string) = 
    let result = uri.EndsWith (expected, StringComparison.CurrentCultureIgnoreCase)
    match result with
    | true -> Some true
    | _ -> None

Isso pode ser usado em um padrão de correspondência desta maneira:

    let isXMLFile uri =
        match uri with
        | HasExtension ".xml" _ -> true
        | _ -> false

Padrões Ativos Completos e Parciais #

Existem dois tipos de padrões ativos que diferem um pouco no uso - Completo e Parcial.

Os Padrões Ativos Completos podem ser usados ​​quando você é capaz de enumerar todos os resultados, como “um número é par ou ímpar?”

let (|Odd|Even|) number = 
  if number % 2 = 0
  then Even
  else Odd

Observe que a definição do Padrão Ativo lista os casos possíveis e nada mais, e o corpo retorna um dos casos listados. Quando você o usa em uma expressão de correspondência, esta é uma correspondência completa:

let n = 13
match n with
| Odd -> printf "%i is odd" n
| Even -> printf "%i is even" n

Isso é útil quando você deseja dividir o espaço de entrada em categorias conhecidas que o cobrem completamente.

Por outro lado, Padrões Ativos Parciais permitem que você ignore explicitamente alguns resultados possíveis retornando uma opção. Sua definição usa um caso especial de _ para o caso sem correspondência.

let (|Integer|_|) str =
  match Int32.TryParse(str) with
  | (true, i) -> Some i
  | _ -> None

Dessa forma, podemos corresponder mesmo quando alguns casos não podem ser tratados por nossa função de análise.

let s = "13"
match s with
| Integer i -> "%i was successfully parsed!" i
| _ -> "%s is not an int" s

Padrões Ativos Parciais podem ser usados ​​como uma forma de teste, se a entrada se enquadra em uma categoria específica no espaço de entrada, ignorando outras opções.
