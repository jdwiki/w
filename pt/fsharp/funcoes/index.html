Tubulação para frente e para trás #

Os operadores de pipe são usados ​​para passar parâmetros para uma função de maneira simples e elegante. Permite eliminar valores intermediários e facilitar a leitura das chamadas de função.

Em F#, existem dois operadores de pipe:

  • Forward (|>): Passando parâmetros da esquerda para a direita

      let print message =
      printf "%s" message

  // "Hello World" will be passed as a parameter to the print function
  "Hello World" |> print

  • Backward (<|): Passando parâmetros da direita para a esquerda

      let print message =
      printf "%s" message

  // "Hello World" will be passed as a parameter to the print function
  print <| "Hello World"

Aqui está um exemplo sem operadores de pipe:

// We want to create a sequence from 0 to 10 then:
// 1 Keep only even numbers
// 2 Multiply them by 2
// 3 Print the number

let mySeq = seq { 0..10 }
let filteredSeq = Seq.filter (fun c -> (c % 2) = 0) mySeq
let mappedSeq = Seq.map ((*) 2) filteredSeq
let finalSeq = Seq.map (sprintf "The value is %i.") mappedSeq

printfn "%A" finalSeq

Podemos encurtar o exemplo anterior e torná-lo mais limpo com o operador forward pipe:

// Using forward pipe, we can eliminate intermediate let binding
let finalSeq = 
    seq { 0..10 }
    |> Seq.filter (fun c -> (c % 2) = 0)
    |> Seq.map ((*) 2)
    |> Seq.map (sprintf "The value is %i.")

printfn "%A" finalSeq

Cada resultado de função é passado como parâmetro para a próxima função.

Se você quiser passar vários parâmetros para o operador de pipe, você deve adicionar um | para cada parâmetro adicional e criar uma Tupla com os parâmetros. O operador de pipe F# nativo suporta até três parâmetros (|||> ou <|||).

let printPerson name age =
    printf "My name is %s, I'm %i years old" name age

("Foo", 20) ||> printPerson

Funções de mais de um parâmetro #

Em F#, todas as funções recebem exatamente um parâmetro. Isso parece uma declaração estranha, já que é trivialmente fácil declarar mais de um parâmetro em uma declaração de função:

let add x y = x + y

Mas se você digitar essa declaração de função no interpretador interativo do F#, verá que sua assinatura de tipo é:

val add : x:int -> y:int -> int

Sem os nomes dos parâmetros, essa assinatura é int -> int -> int. O operador -> é associativo à direita, o que significa que esta assinatura é equivalente a int -> (int -> int). Em outras palavras, add é uma função que recebe um parâmetro int e retorna uma função que recebe um int e retorna int. Tente:

let addTwo = add 2
// val addTwo : (int -> int)
let five = addTwo 3
// val five : int = 5

No entanto, você também pode chamar uma função como add de uma maneira mais “convencional”, passando diretamente dois parâmetros, e ela funcionará como esperado:

let three = add 1 2
// val three : int = 3

Isso se aplica a funções com quantos parâmetros você quiser:

let addFiveNumbers a b c d e = a + b + c + d + e
// val addFiveNumbers : a:int -> b:int -> c:int -> d:int -> e:int -> int
let addFourNumbers = addFiveNumbers 0
// val addFourNumbers : (int -> int -> int -> int -> int)
let addThreeNumbers = addFourNumbers 0
// val addThreeNumbers : (int -> int -> int -> int)
let addTwoNumbers = addThreeNumbers 0
// val addTwoNumbers : (int -> int -> int)
let six = addThreeNumbers 1 2 3  // This will calculate 0 + 0 + 1 + 2 + 3
// val six : int = 6

Esse método de pensar em funções multiparâmetros como sendo funções que recebem um parâmetro e retornam novas funções (que podem, por sua vez, receber um parâmetro e retornar novas funções, até chegar à função final que recebe o parâmetro final e finalmente retorna um resultado) chama-se currying, em homenagem ao matemático Haskell Curry, famoso por desenvolver o conceito. (Foi inventado por outra pessoa, mas Curry merecidamente recebe a maior parte do crédito por isso.)

Esse conceito é usado em todo o F# e você deve se familiarizar com ele.

Noções básicas de funções #

A maioria das funções em F# são criadas com a sintaxe let:

let timesTwo x = x * 2

Isso define uma função chamada timesTwo que recebe um único parâmetro x. Se você executar uma sessão interativa do F# (fsharpi no OS X e Linux, fsi.exe no Windows) e colar essa função (e adicionar o ;; que diz ao fsharpi para avaliar o código que você acabou de digitar ), você verá que ele responde com:

val timesTwo : x:int -> int

Isso significa que timesTwo é uma função que recebe um único parâmetro x do tipo int e retorna um int. As assinaturas de função geralmente são escritas sem os nomes dos parâmetros, então você verá esse tipo de função escrito como int -> int.

Mas espere! Como o F# sabia que x era um parâmetro inteiro, já que você nunca especificou seu tipo? Isso se deve à inferência de tipo. Como no corpo da função, você multiplicou x por 2, os tipos de x e 2 devem ser os mesmos. (Como regra geral, o F# não converterá valores implicitamente para tipos diferentes; você deve especificar explicitamente qualquer tipo de conversão que desejar).

Se você deseja criar uma função que não receba parâmetros, esta é a maneira errada de fazer isso:

let hello =  // This is a value, not a function
    printfn "Hello world"

A maneira certa de fazer isso é:

let hello () =
    printfn "Hello world"

Esta função hello tem o tipo unit -> unit, que é explicado em https://www.wikiod.com/pt/docs/f%23/2513/the-unit-type.

Funções Curry vs Tupled #

Existem duas maneiras de definir funções com vários parâmetros em F#, funções Curried e funções Tupladas.

let curriedAdd x y = x + y // Signature: x:int -> y:int -> int
let tupledAdd (x, y) = x + y // Signature:  x:int * y:int -> int

Todas as funções definidas de fora do F# (como o .NET framework) são usadas no F# com o formulário Tupled. A maioria das funções nos módulos principais do F# são usadas com o formulário Curried.

A forma Curried é considerada F# idiomática, pois permite aplicação parcial. Nenhum dos dois exemplos a seguir é possível com o formulário Tupled.

let addTen = curriedAdd 10 // Signature: int -> int

// Or with the Piping operator
3 |> curriedAdd 7 // Evaluates to 10

A razão por trás disso é que a função Curried, quando chamada com um parâmetro, retorna uma função. Bem-vindo à programação funcional !!

let explicitCurriedAdd x = (fun y -> x + y) // Signature: x:int -> y:int -> int
let veryExplicitCurriedAdd = (fun x -> (fun y -> x + y)) // Same signature

Você pode ver que é exatamente a mesma assinatura.

No entanto, ao fazer interface com outro código .NET, como ao escrever bibliotecas, é importante definir funções usando o formulário Tupled.

Inline #

Inlining permite substituir uma chamada para uma função pelo corpo da função.

Isso às vezes é útil por motivos de desempenho na parte crítica do código. Mas a contrapartida é que seu assembly ocupa muito espaço, pois o corpo da função é duplicado em todos os lugares em que ocorreu uma chamada. Você precisa ter cuidado ao decidir se deve ou não inserir uma função.

Uma função pode ser embutida com a palavra-chave inline:

// inline indicate that we want to replace each call to sayHello with the body 
// of the function
let inline sayHello s1 =
    sprintf "Hello %s" s1

// Call to sayHello will be replaced with the body of the function
let s = sayHello "Foo"
// After inlining -> 
// let s = sprintf "Hello %s" "Foo"

printfn "%s" s
// Output
// "Hello Foo"

Outro exemplo com valor local:

let inline addAndMulti num1 num2 =
    let add = num1 + num2
    add * 2

let i = addAndMulti 2 2
// After inlining ->
// let add = 2 + 2
// let i = add * 2

printfn "%i" i
// Output
// 8
