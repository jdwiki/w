Como compor valores e funções usando operadores comuns #

Na Programação Orientada a Objetos uma tarefa comum é compor objetos (valores). Na Programação Funcional é tarefa comum compor valores assim como funções.

Estamos acostumados a compor valores a partir de nossa experiência em outras linguagens de programação usando operadores como +, -, *, / e assim por diante.

Composição de valor

let x = 1 + 2 + 3 * 2

Como a programação funcional compõe funções e valores, não é surpreendente existem operadores comuns para composição de funções como >>, <<, |> e <|.

Composição de funções

// val f : int -> int
let f v   = v + 1
// val g : int -> int
let g v   = v * 2

// Different ways to compose f and g
// val h : int -> int
let h1 v  = g (f v)
let h2 v  = v |> f |> g   // Forward piping of 'v'
let h3 v  = g <| (f <| v) // Reverse piping of 'v' (because <| has left associcativity we need ())
let h4    = f >> g        // Forward functional composition
let h5    = g << f        // Reverse functional composition (closer to math notation of 'g o f')

Em F#, a tubulação direta é preferida à tubulação reversa porque:

  1. A inferência de tipo (geralmente) flui da esquerda para a direita, então é natural que valores e funções fluam da esquerda para a direita também
  2. Porque <| e << devem ter associatividade à direita, mas em F# eles são associativos à esquerda, o que nos obriga a inserir ()
  3. Misturar tubulações direta e reversa geralmente não funciona porque elas têm a mesma precedência.

Composição da Mônada

Como as Mônadas (como Option<'T> ou List<'T>) são comumente usadas na programação funcional também existem operadores comuns, mas menos conhecidos, para compor funções trabalhando com Mônadas como >>=, >=>, <|> e <*>.

let (>>=) t uf  = Option.bind uf t
let (>=>) tf uf = fun v -> tf v >>= uf
// val oinc   : int -> int option
let oinc   v    = Some (v + 1)    // Increment v
// val ofloat : int -> float option
let ofloat v    = Some (float v)  // Map v to float

// Different ways to compose functions working with Option Monad
// val m : int option -> float option
let m1 v  = Option.bind (fun v -> Some (float (v + 1))) v
let m2 v  = v |> Option.bind oinc |> Option.bind ofloat
let m3 v  = v >>= oinc >>= ofloat
let m4    = oinc >=> ofloat

// Other common operators are <|> (orElse) and <*> (andAlso)

// If 't' has Some value then return t otherwise return u
let (<|>) t u =
  match t with
  | Some _  -> t
  | None    -> u

// If 't' and 'u' has Some values then return Some (tv*uv) otherwise return None
let (<*>) t u =
  match t, u with
  | Some tv, Some tu  -> Some (tv, tu)
  | _                 -> None

// val pickOne : 'a option -> 'a option -> 'a option
let pickOne t u v = t <|> u <|> v

// val combine : 'a option -> 'b option  -> 'c option -> (('a*'b)*'c) option
let combine t u v = t <*> u <*> v

Conclusão

Para novos programadores funcionais, a composição de funções usando operadores pode parecer opaca e obscuro, mas isso ocorre porque o significado desses operadores não é tão comumente conhecidos como operadores trabalhando em valores. No entanto, com algum treinamento usando |>, >> , >>= e >=> torna-se tão natural quanto usar +, -, * e /.

Latebinding em F# usando ? operador #

Em uma linguagem de tipagem estática como F# trabalhamos com tipos bem conhecidos em tempo de compilação. Consumimos fontes de dados externas de maneira segura usando tipo provedores.

No entanto, ocasionalmente há necessidade de usar ligação tardia (como dynamic em C#). Por exemplo, ao trabalhar com documentos JSON que não possuem esquema bem definido.

Para simplificar o trabalho com ligação tardia, o F# oferece suporte aos operadores de pesquisa dinâmica ? e ?<-.

Exemplo:

// (?) allows us to lookup values in a map like this: map?MyKey
let inline (?)   m k   = Map.tryFind k m
// (?<-) allows us to update values in a map like this: map?MyKey <- 123
let inline (?<-) m k v = Map.add k v m

let getAndUpdate (map : Map<string, int>) : int option*Map<string, int> =
  let i = map?Hello       // Equivalent to map |> Map.tryFind "Hello"
  let m = map?Hello <- 3  // Equivalent to map |> Map.add "Hello" 3
  i, m

Acontece que o suporte F# para ligação tardia é simples, mas flexível.
