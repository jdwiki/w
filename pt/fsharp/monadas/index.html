Entender as Mônadas vem da prática #

Este tópico é destinado a desenvolvedores de F# intermediários a avançados

“O que são Mônadas?” é uma pergunta comum. Isso é fácil de responder, mas como em [Guia do Mochileiro da Galáxia](https://en.wikipedia.org/wiki/Phrases_from_The_Hitchhiker%27s_Guide_to_the_Galaxy#Answer_to_the_Ultimate_Question_of_Life.2C_the_Universe. 2C_and_Everything_.2842.29) percebemos que não entendemos a resposta porque não sabíamos o que estávamos perguntando.

Muitos acreditam que a maneira de entender as Mônadas é praticando-as. Como programadores, normalmente não nos importamos com a base matemática do Princípio de Substituição de Liskov, subtipos ou subclasses. Ao usar essas idéias, adquirimos uma intuição para o que elas representam. O mesmo vale para as Mônadas.

Para ajudá-lo a começar com o Monads, este exemplo demonstra como construir uma biblioteca Monadic Parser Combinator. Isso pode ajudá-lo a começar, mas a compreensão virá da escrita de sua própria biblioteca monádica.

Chega de prosa, hora do código

O tipo de analisador:

// A Parser<'T> is a function that takes a string and position
//  and returns an optionally parsed value and a position
//  A parsed value means the position points to the character following the parsed value
//  No parsed value indicates a parse failure at the position
type Parser<'T> = Parser of (string*int -> 'T option*int)

Usando esta definição de um analisador, definimos algumas funções fundamentais do analisador

// Runs a parser 't' on the input string 's'
let run t s =
  let (Parser tps) = t
  tps (s, 0)

// Different ways to create parser result
let succeedWith v p = Some v, p
let failAt        p = None  , p

// The 'satisfy' parser succeeds if the character at the current position 
//  passes the 'sat' function
let satisfy sat : Parser<char> = Parser <| fun (s, p) ->
  if p < s.Length && sat s.[p] then succeedWith s.[p] (p + 1)
  else failAt p

// 'eos' succeeds if the position is beyond the last character.
//  Useful when testing if the parser have consumed the full string
let eos : Parser<unit> = Parser <| fun (s, p) ->
  if p < s.Length then failAt p
  else succeedWith () p

let anyChar       = satisfy (fun _ -> true)
let char ch       = satisfy ((=) ch)
let digit         = satisfy System.Char.IsDigit
let letter        = satisfy System.Char.IsLetter

satisfy é uma função que dada uma função sat produz um analisador que é bem sucedido se não passamos EOS e o caractere na posição atual passa a função sat. Usando satisfy criamos vários analisadores de caracteres úteis.

Executando isso no FSI:

> run digit "";;
val it : char option * int = (null, 0)
> run digit "123";;
val it : char option * int = (Some '1', 1)
> run digit "hello";;
val it : char option * int = (null, 0)

Temos alguns analisadores fundamentais no lugar. Vamos combiná-los em analisadores mais poderosos usando funções de combinação de analisadores

// 'fail' is a parser that always fails
let fail<'T>      = Parser <| fun (s, p) -> failAt p
// 'return_' is a parser that always succeed with value 'v'
let return_ v     = Parser <| fun (s, p) -> succeedWith v p

// 'bind' let us combine two parser into a more complex parser
let bind t uf     = Parser <| fun (s, p) ->
  let (Parser tps) = t
  let tov, tp = tps (s, p)
  match tov with
  | None    -> None, tp
  | Some tv ->
    let u = uf tv
    let (Parser ups) = u
    ups (s, tp)

Os nomes e assinaturas são não escolhidos arbitrariamente, mas não vamos nos aprofundar nisso, vamos ver como usamos bind para combinar o analisador em outros mais complexos:

> run (bind digit (fun v -> digit)) "123";;
val it : char option * int = (Some '2', 2)
> run (bind digit (fun v -> bind digit (fun u -> return_ (v,u)))) "123";;
val it : (char * char) option * int = (Some ('1', '2'), 2)
> run (bind digit (fun v -> bind digit (fun u -> return_ (v,u)))) "1";;
val it : (char * char) option * int = (null, 1)

O que isso nos mostra é que bind nos permite combinar dois analisadores em um analisador mais complexo. Como resultado de bind é um analisador que por sua vez pode ser combinado novamente.

> run (bind digit (fun v -> bind digit (fun w -> bind digit (fun u -> return_ (v,w,u))))) "123";;
val it : (char * char * char) option * int = (Some ('1', '2', '3'), 3)

bind será a maneira fundamental de combinar analisadores, embora definamos funções auxiliares para simplificar a sintaxe.

Uma das coisas que podem simplificar a sintaxe são expressões de computação. São fáceis de definir:

type ParserBuilder() =
  member x.Bind       (t, uf) = bind      t   uf
  member x.Return     v       = return_   v
  member x.ReturnFrom t       = t

// 'parser' enables us to combine parsers using 'parser { ... }' syntax
let parser = ParserBuilder()

FSI

let p = parser {
          let! v = digit
          let! u = digit
          return v,u
        }
run p "123"
val p : Parser<char * char> = Parser <fun:bind@49-1>
val it : (char * char) option * int = (Some ('1', '2'), 2)

Isso é equivalente a:

> let p = bind digit (fun v -> bind digit (fun u -> return_ (v,u)))
run p "123";;
val p : Parser<char * char> = Parser <fun:bind@49-1>
val it : (char * char) option * int = (Some ('1', '2'), 2)

Outro combinador de parser fundamental que vamos usar muito é orElse:

// 'orElse' creates a parser that runs parser 't' first, if that is successful
//  the result is returned otherwise the result of parser 'u' is returned
let orElse t u    = Parser <| fun (s, p) ->
  let (Parser tps) = t
  let tov, tp = tps (s, p)
  match tov with
  | None    -> 
    let (Parser ups) = u
    ups (s, p)
  | Some tv -> succeedWith tv tp

Isso nos permite definir letterOrDigit assim:

> let letterOrDigit = orElse letter digit;;
val letterOrDigit : Parser<char> = Parser <fun:orElse@70-1>
> run letterOrDigit "123";;
val it : char option * int = (Some '1', 1)
> run letterOrDigit "hello";;
val it : char option * int = (Some 'h', 1)
> run letterOrDigit "!!!";;
val it : char option * int = (null, 0)

Uma nota sobre os operadores Infix

Uma preocupação comum sobre FP é o uso de operadores infixos incomuns como >>=, >=>, <- e assim por diante. No entanto, a maioria não está preocupada com o uso de +, -, *, / e %, estes são operadores bem conhecidos usados ​​para compor valores. No entanto, uma grande parte do PF é compor não apenas valores, mas também funções. Para um desenvolvedor de FP intermediário, os operadores infixos >>=, >=>, <- são bem conhecidos e devem ter assinaturas específicas, bem como semântica.

Para as funções que definimos até agora, definiríamos os seguintes operadores infixos usados ​​para combinar analisadores:

let (>>=)   t   uf  = bind t uf
let (<|>)   t   u   = orElse t u

Então >>= significa bind e <|> significa orElse.

Isso nos permite combinar analisadores mais sucintos:

let letterOrDigit = letter <|> digit
let p = digit >>= fun v -> digit >>= fun u -> return_ (v,u)

Para definir alguns combinadores de analisadores avançados que nos permitirão analisar expressões mais complexas, definimos mais alguns combinadores de analisadores simples:

// 'map' runs parser 't' and maps the result using 'm'
let map m t       = t >>= (m >> return_)
let (>>!) t m     = map m t
let (>>%) t v     = t >>! (fun _ -> v)

// 'opt' takes a parser 't' and creates a parser that always succeed but
//  if parser 't' fails the new parser will produce the value 'None'
let opt t         = (t >>! Some) <|> (return_ None)

// 'pair' runs parser 't' and 'u' and returns a pair of 't' and 'u' results
let pair t u      = 
  parser {
    let! tv = t
    let! tu = u
    return tv, tu
  }

Estamos prontos para definir many e sepBy que são mais avançados à medida que aplicam os analisadores de entrada até que falhem. Então many e sepBy retornam o resultado agregado:

// 'many' applies parser 't' until it fails and returns all successful
//  parser results as a list
let many t =
  let ot = opt t
  let rec loop vs = ot >>= function Some v -> loop (v::vs) | None -> return_ (List.rev vs)
  loop []

// 'sepBy' applies parser 't' separated by 'sep'. 
//  The values are reduced with the function 'sep' returns
let sepBy t sep     =
  let ots = opt (pair sep t)
  let rec loop v = ots >>= function Some (s, n) -> loop (s v n) | None -> return_ v
  t >>= loop

Criando um analisador de expressão simples

Com as ferramentas que criamos agora podemos definir um analisador para expressões simples como 1+2*3

Começamos de baixo definindo um analisador para inteiros pint

// 'pint' parses an integer
let pint = 
  let f s v = 10*s + int v - int '0'
  parser {
    let! digits = many digit
    return! 
      match digits with
      | [] -> fail
      | vs -> return_ (List.fold f 0 vs)
  }

Tentamos analisar o maior número possível de dígitos, o resultado é char list. Se a lista estiver vazia, falhamos, caso contrário, dobramos os caracteres em um inteiro.

Testando pint no FSI:

> run pint "123";;
val it : int option * int = (Some 123, 3)

Além disso, precisamos analisar os diferentes tipos de operadores usados ​​para combinar valores inteiros:

// operator parsers, note that the parser result is the operator function 
let padd      = char '+' >>% (+)
let psubtract = char '-' >>% (-)
let pmultiply = char '*' >>% (*)
let pdivide   = char '/' >>% (/)
let pmodulus  = char '%' >>% (%)

FSI:

> run padd "+";;
val it : (int -> int -> int) option * int = (Some <fun:padd@121-1>, 1)

Juntando tudo:

// 'pmullike' parsers integers separated by operators with same precedence as multiply
let pmullike  = sepBy pint (pmultiply <|> pdivide <|> pmodulus)
// 'paddlike' parsers sub expressions separated by operators with same precedence as add
let paddlike  = sepBy pmullike (padd <|> psubtract)
// 'pexpr' is the full expression
let pexpr     =
  parser {
    let! v = paddlike
    let! _ = eos      // To make sure the full string is consumed
    return v
  }

Executando tudo no FSI:

> run pexpr "2+123*2-3";;
val it : int option * int = (Some 245, 9)

Conclusão

Definindo Parser<'T>, return_, bind e certificando-se de que obedecem às leis monádicas, construímos um simples, mas poderoso, Combinador de Analisador Monádico estrutura.

Mônadas e Analisadores andam juntos porque os Analisadores são executados em um estado de analisador. O Monads nos permite combinar analisadores enquanto ocultamos o estado do analisador, reduzindo assim a desordem e melhorando a composição.

O framework que criamos é lento e não produz mensagens de erro, isso para manter o código sucinto. FParsec fornece tanto um desempenho aceitável quanto excelentes mensagens de erro.

No entanto, um exemplo por si só não pode dar compreensão das Mônadas. É preciso praticar Mônadas.

Aqui estão alguns exemplos de Mônadas que você pode tentar implementar para alcançar seu entendimento:

  1. Mônada de Estado - Permite que o estado do ambiente oculto seja carregado implicitamente
  2. Tracer Monad - Permite que o estado de rastreamento seja carregado implicitamente. Uma variante do Estado Monad
  3. Turtle Monad - Uma Mônada para criar programas Turtle (Logos). Uma variante do Estado Monad
  4. Mônada de Continuação - Uma Mônada Corrotina. Um exemplo disso é async em F#

A melhor coisa para aprender seria criar um aplicativo para Monads em um domínio com o qual você se sinta confortável. Para mim, isso era analisadores.

//  and returns an optionally parsed value and a position
//  A parsed value means the position points to the character following the parsed value
//  No parsed value indicates a parse failure at the position
type Parser<'T> = Parser of (string*int -> 'T option*int)

// Runs a parser 't' on the input string 's'
let run t s =

  tps (s, 0)

let failAt        p = None  , p

//  passes the 'sat' function

  if p < s.Length && sat s.[p] then succeedWith s.[p] (p + 1)
  else failAt p

// 'eos' succeeds if the position is beyond the last character.
//  Useful when testing if the parser have consumed the full string
let eos : Parser<unit> = Parser <| fun (s, p) ->
  if p < s.Length then failAt p
  else succeedWith () p

let anyChar       = satisfy (fun _ -> true)
let char ch       = satisfy ((=) ch)
let digit         = satisfy System.Char.IsDigit
let letter        = satisfy System.Char.IsLetter

// 'fail' is a parser that always fails
let fail<'T>      = Parser <| fun (s, p) -> failAt p
// 'return_' is a parser that always succeed with value 'v'
let return_ v     = Parser <| fun (s, p) -> succeedWith v p

// 'bind' let us combine two parser into a more complex parser
let bind t uf     = Parser <| fun (s, p) ->
  let (Parser tps) = t
  let tov, tp = tps (s, p)
  match tov with
  | None    -> None, tp
  | Some tv ->
    let u = uf tv
    let (Parser ups) = u
    ups (s, tp)

type ParserBuilder() =
  member x.Bind       (t, uf) = bind      t   uf
  member x.Return     v       = return_   v
  member x.ReturnFrom t       = t

// 'parser' enables us to combine parsers using 'parser { ... }' syntax
let parser = ParserBuilder()

// 'orElse' creates a parser that runs parser 't' first, if that is successful
//  the result is returned otherwise the result of parser 'u' is returned
let orElse t u    = Parser <| fun (s, p) ->
  let (Parser tps) = t
  let tov, tp = tps (s, p)
  match tov with
  | None    -> 
    let (Parser ups) = u
    ups (s, p)
  | Some tv -> succeedWith tv tp

let (>>=) t uf    = bind t uf
let (<|>) t u     = orElse t u

// 'map' runs parser 't' and maps the result using 'm'
let map m t       = t >>= (m >> return_)
let (>>!) t m     = map m t
let (>>%) t v     = t >>! (fun _ -> v)

// 'opt' takes a parser 't' and creates a parser that always succeed but
//  if parser 't' fails the new parser will produce the value 'None'
let opt t         = (t >>! Some) <|> (return_ None)

// 'pair' runs parser 't' and 'u' and returns a pair of 't' and 'u' results
let pair t u      = 
  parser {
    let! tv = t
    let! tu = u
    return tv, tu
  }

// 'many' applies parser 't' until it fails and returns all successful
//  parser results as a list
let many t =

  let rec loop vs = ot >>= function Some v -> loop (v::vs) | None -> return_ (List.rev vs)
  loop []

// 'sepBy' applies parser 't' separated by 'sep'. 
//  The values are reduced with the function 'sep' returns

  let ots = opt (pair sep t)
  let rec loop v = ots >>= function Some (s, n) -> loop (s v n) | None -> return_ v
  t >>= loop

// A simplistic integer expression parser

// 'pint' parses an integer
let pint = 
  let f s v = 10*s + int v - int '0'
  parser {
    let! digits = many digit

      match digits with
      | [] -> fail

Expressões Computacionais fornecem uma sintaxe alternativa para encadear Mônadas #

Relacionados às Mônadas estão F# expressões de computação («CE»). Um programador normalmente implementa um CE para fornecer uma abordagem alternativa para encadear Mônadas, ou seja, em vez disso:

let v = m >>= fun x -> n >>= fun y -> return_ (x, y)

Você pode escrever isso:

let v = ce {
    let! x = m
    let! y = n
    return x, y
  }

Ambos os estilos são equivalentes e depende da preferência do desenvolvedor qual escolher.

Para demonstrar como implementar um CE imagine que você gosta de todos os rastreamentos para inclua um ID de correlação. Este ID de correlação ajudará a correlacionar rastreamentos que pertencem à mesma chamada. Isso é muito útil quando há arquivos de log que contém rastreamentos de chamadas simultâneas.

O problema é que é complicado incluir o ID de correlação como um argumento para todas as funções. Como Monads permite carregar estado implícito, vamos defina um Log Monad para ocultar o contexto do log (ou seja, o ID de correlação).

Começamos definindo um contexto de log e o tipo de função que rastreia com o contexto de log:

type Context =
  {
    CorrelationId : Guid
  }
  static member New () : Context = { CorrelationId = Guid.NewGuid () }

type Function<'T> = Context -> 'T

// Runs a Function<'T> with a new log context
let run t = t (Context.New ())

Também definimos duas funções de rastreamento que serão registradas com o id de correlação do contexto de log:

let trace v   : Function<_> = fun ctx -> printfn "CorrelationId: %A - %A" ctx.CorrelationId v
let tracef fmt              = kprintf trace fmt

trace é uma Function<unit> o que significa que será passado um contexto de log quando invocado. Do contexto de log, pegamos o id de correlação e o rastreamos junto com v

Além disso, definimos bind e return_ e como eles seguem o Leis da Mônada isso forma nosso Log Monad.

let bind t uf : Function<_> = fun ctx ->
  let tv = t ctx  // Invoke t with the log context
  let u  = uf tv  // Create u function using result of t
  u ctx           // Invoke u with the log context

// >>= is the common infix operator for bind
let inline (>>=) (t, uf) = bind t uf

let return_ v : Function<_> = fun ctx -> v

Finalmente definimos o LogBuilder que nos permitirá usar a sintaxe CE para encadear Mônadas Log.

type LogBuilder() =
  member x.Bind   (t, uf) = bind t uf
  member x.Return v       = return_ v

// This enables us to write function like: let f = log { ... }
let log = Log.LogBuilder ()

Agora podemos definir nossas funções que devem ter o contexto de log implícito:

let f x y =
  log {
    do! Log.tracef "f: called with: x = %d, y = %d" x y
    return x + y
  }

let g =
  log {
    do! Log.trace "g: starting..."
    let! v = f 1 2
    do! Log.tracef "g: f produced %d" v
    return v
  }

Executamos g com:

printfn "g produced %A" (Log.run g)

Que imprime:

CorrelationId: 33342765-2f96-42da-8b57-6fa9cdaf060f - "g: starting..."
CorrelationId: 33342765-2f96-42da-8b57-6fa9cdaf060f - "f: called with: x = 1, y = 2"
CorrelationId: 33342765-2f96-42da-8b57-6fa9cdaf060f - "g: f produced 3"
g produced 3

Observe que o CorrelationId é implicitamente carregado de run para g para f, o que nos permite correlacionar as entradas de log durante a resolução de problemas.

CE tem muito mais recursos mas isso deve ajudá-lo a começar a definir seus próprios CE:s.

Código completo:

module Log =
  open System
  open FSharp.Core.Printf

  type Context =
    {
      CorrelationId : Guid
    }
    static member New () : Context = { CorrelationId = Guid.NewGuid () }

  type Function<'T> = Context -> 'T

  // Runs a Function<'T> with a new log context
  let run t = t (Context.New ())

  let trace v   : Function<_> = fun ctx -> printfn "CorrelationId: %A - %A" ctx.CorrelationId v
  let tracef fmt              = kprintf trace fmt

  let bind t uf : Function<_> = fun ctx ->
    let tv = t ctx  // Invoke t with the log context
    let u  = uf tv  // Create u function using result of t
    u ctx           // Invoke u with the log context

  // >>= is the common infix operator for bind
  let inline (>>=) (t, uf) = bind t uf

  let return_ v : Function<_> = fun ctx -> v

  type LogBuilder() =
    member x.Bind   (t, uf) = bind t uf
    member x.Return v       = return_ v

// This enables us to write function like: let f = log { ... }
let log = Log.LogBuilder ()

let f x y =
  log {
    do! Log.tracef "f: called with: x = %d, y = %d" x y
    return x + y
  }

let g =
  log {
    do! Log.trace "g: starting..."
    let! v = f 1 2
    do! Log.tracef "g: f produced %d" v
    return v
  }

[<EntryPoint>]
let main argv =
  printfn "g produced %A" (Log.run g)
  0
