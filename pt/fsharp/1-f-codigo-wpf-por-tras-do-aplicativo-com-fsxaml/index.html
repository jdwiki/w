Nesta página

  • Crie um novo F# Código WPF por trás do aplicativo.
  • 3 : Adicione um ícone a uma janela
  • 4: Adicionar ícone ao aplicativo
  • 2: Adicione um controle
  • Como adicionar controles de bibliotecas de terceiros

A maioria dos exemplos encontrados para programação em F# WPF parece lidar com o padrão MVVM e alguns com MVC, mas não há quase nenhum que mostre corretamente como começar a funcionar com o código “bom e velho” por trás.

O padrão de código por trás é muito fácil de usar para ensino e experimentação. É usado em vários livros de introdução e material de aprendizagem na web. É por isso.

Esses exemplos demonstrarão como criar um código por trás do aplicativo com janelas, controles, imagens e ícones e muito mais.

Crie um novo F# Código WPF por trás do aplicativo. #

Crie um aplicativo de console F#.

Altere o Tipo de saída do aplicativo para Aplicativo do Windows.

Adicione o pacote NuGet FsXaml.

Adicione esses quatro arquivos de origem, na ordem listada aqui.

MainWindow.xaml

<Window
    xmlns="http://schemas.microsoft.com/winfx/2006/xaml/presentation"
    xmlns:x="http://schemas.microsoft.com/winfx/2006/xaml"
    Title="First Demo" Height="200" Width="300">
    <Canvas>
        <Button Name="btnTest" Content="Test" Canvas.Left="10" Canvas.Top="10" Height="28" Width="72"/>
    </Canvas>
</Window>

MainWindow.xaml.fs

namespace FirstDemo

type MainWindowXaml = FsXaml.XAML<"MainWindow.xaml">

type MainWindow() as this =
    inherit MainWindowXaml()

    let whenLoaded _ =
        ()

    let whenClosing _ =
        ()

    let whenClosed _ =
        ()

    let btnTestClick _ =
        this.Title <- "Yup, it works!"

    do
        this.Loaded.Add whenLoaded
        this.Closing.Add whenClosing
        this.Closed.Add whenClosed
        this.btnTest.Click.Add btnTestClick

App.xaml

<Application xmlns="http://schemas.microsoft.com/winfx/2006/xaml/presentation"
             xmlns:x="http://schemas.microsoft.com/winfx/2006/xaml">
    <Application.Resources>
    </Application.Resources>
</Application>

App.xaml.fs

namespace FirstDemo

open System

type App = FsXaml.XAML<"App.xaml">

module Main =

    [<STAThread; EntryPoint>]
    let main _ =
        let app = App()
        let mainWindow = new MainWindow()
        app.Run(mainWindow) // Returns application's exit code.

Exclua o arquivo Program.fs do projeto.

Altere a Build Action para Resource para os dois arquivos xaml.

Adicione uma referência ao assembly .NET UIAutomationTypes.

Compile e execute.

Você não pode usar o designer para adicionar manipuladores de eventos, mas isso não é um problema. Basta adicioná-los manualmente no código por trás, como você vê com os três manipuladores neste exemplo, incluindo o manipulador para o botão de teste.

ATUALIZAÇÃO: Uma maneira alternativa e provavelmente mais elegante de adicionar manipuladores de eventos foi adicionada ao FsXaml. Você pode adicionar o manipulador de eventos em XAML, da mesma forma que em C#, mas precisa fazer isso manualmente e, em seguida, substituir o membro correspondente que aparece no seu tipo F#. Eu recomendo isso.

3 : Adicione um ícone a uma janela #

É uma boa ideia manter todos os ícones e imagens em uma ou mais pastas.

Clique com o botão direito do mouse no projeto e use F# Power Tools / New Folder para criar uma pasta chamada Images.

No disco, coloque seu ícone na nova pasta Imagens.

De volta ao Visual Studio, clique com o botão direito do mouse em Imagens e use Add / Existing Item, depois mostre All Files (.) para ver o arquivo de ícone para que você possa selecioná-lo e, em seguida, Adicione.

Selecione o arquivo de ícone e defina sua Build Action como Resource.

Em MainWindow.xaml, use o atributo Icon assim. As linhas circundantes são mostradas para o contexto.

    xmlns:x="http://schemas.microsoft.com/winfx/2006/xaml"
    Title="First Demo" Height="200" Width="300"
    Icon="Images/MainWindow.ico">
<Canvas>

Antes de executar, faça um Rebuild, e não apenas um Build. Isso ocorre porque o Visual Studio nem sempre coloca o arquivo de ícone no executável, a menos que você reconstrua.

É a janela, e não o aplicativo, que agora tem um ícone. Você verá o ícone no canto superior esquerdo da janela em tempo de execução e o verá na barra de tarefas. O Gerenciador de Tarefas e o Explorador de Arquivos do Windows não mostrarão esse ícone, pois exibem o ícone do aplicativo em vez do ícone da janela.

4: Adicionar ícone ao aplicativo #

Crie um arquivo de texto chamado AppIcon.rc, com o conteúdo a seguir.

1 ICON "AppIcon.ico"

Você precisará de um arquivo de ícone chamado AppIcon.ico para que isso funcione, mas é claro que você pode ajustar os nomes ao seu gosto.

Execute o seguinte comando.

"C:\Program Files (x86)\Windows Kits\10\bin\x64\rc.exe" /v AppIcon.rc

Se você não encontrar o rc.exe neste local, procure-o abaixo de C:\Program Files (x86)\Windows Kits. Se você ainda não conseguir encontrá-lo, baixe o Windows SDK da Microsoft.

Um arquivo chamado AppIcon.res será gerado.

No Visual Studio, abra as propriedades do projeto. Selecione a página Aplicativo.

Na caixa de texto intitulada Resource File, digite AppIcon.res (ou Images\AppIcon.res se você o colocar lá) e feche as propriedades do projeto para salvar.

Uma mensagem de erro aparecerá, informando “O arquivo de recurso inserido não existe. Ignore isso. A mensagem de erro não reaparecerá.

Reconstruir. O executável terá então um ícone de aplicativo, e isso será exibido no Explorador de Arquivos. Ao executar, este ícone também aparecerá no Gerenciador de Tarefas.

2: Adicione um controle #

Adicione esses dois arquivos nesta ordem acima dos arquivos da janela principal.

MeuControle.xaml

<UserControl
             xmlns="http://schemas.microsoft.com/winfx/2006/xaml/presentation"
             xmlns:x="http://schemas.microsoft.com/winfx/2006/xaml"
             xmlns:mc="http://schemas.openxmlformats.org/markup-compatibility/2006"
             xmlns:d="http://schemas.microsoft.com/expression/blend/2008"
             mc:Ignorable="d" Height="50" Width="150">
    <Canvas Background="LightGreen">
        <Button Name="btnMyTest" Content="My Test" Canvas.Left="10" Canvas.Top="10" Height="28" Width="106"/>
    </Canvas>
</UserControl>

MeuControle.xaml.fs

namespace FirstDemo

type MyControlXaml = FsXaml.XAML<"MyControl.xaml">

type MyControl() =
    inherit MyControlXaml()

A Build Action para MyControl.xaml deve ser definida como Resource.

É claro que mais tarde você precisará adicionar “as this” na declaração de MyControl, assim como feito para a janela principal.

No arquivo MainWindow.xaml.fs, na classe para MainWindow, adicione esta linha

let myControl = MyControl()

e adicione essas duas linhas na seção do da classe da janela principal.

    this.mainCanvas.Children.Add myControl |> ignore
    myControl.btnMyTest.Content <- "We're in business!"

Pode haver mais de uma seção do em uma classe, e você provavelmente precisará dela ao escrever muitos códigos code-behind.

O controle recebeu uma cor de fundo verde claro, para que você possa ver facilmente onde está.

Esteja ciente de que o controle bloqueará a visualização do botão da janela principal. Está além do escopo desses exemplos ensinar o WPF em geral, portanto, não corrigiremos isso aqui.

Como adicionar controles de bibliotecas de terceiros #

Se você adicionar controles de bibliotecas de terceiros em um projeto C# WPF, o arquivo XAML normalmente terá linhas como esta.

xmlns:xctk="http://schemas.xceed.com/wpf/xaml/toolkit"

Isso talvez não funcione com o FsXaml.

O designer e o compilador aceitam essa linha, mas provavelmente haverá uma exceção em tempo de execução reclamando que o tipo de terceiros não foi encontrado ao ler o XAML.

Tente algo como o seguinte.

xmlns:xctk="clr-namespace:Xceed.Wpf.Toolkit;assembly=Xceed.Wpf.Toolkit"

Este então é um exemplo de um controle que depende do acima.

<xctk:IntegerUpDown Name="tbInput" Increment="1" Maximum="10" Minimum="0" Canvas.Left="13" Canvas.Top="27" Width="270"/>

A biblioteca usada neste exemplo é o Extended Wpf Toolkit, disponível gratuitamente por meio do NuGet ou como instalador. Se você baixar bibliotecas por meio do NuGet, os controles não estarão disponíveis na caixa de ferramentas, mas ainda serão exibidos no designer se você adicioná-los manualmente em XAML e as propriedades estarão disponíveis no painel Propriedades.
