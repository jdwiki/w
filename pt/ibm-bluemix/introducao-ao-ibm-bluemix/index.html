Implementação contínua da API RESTful (Node.js) via GitHub usando o IBM Bluemix Toolchain #

Etapa 1: crie sua conta no GitHub #

Se você já possui uma conta do GitHub, prossiga para a Etapa 2. Caso contrário, siga abaixo:

1.a Vá para a página do Github.

1.b Digite o nome de usuário desejado, seu endereço de e-mail e a senha desejada. Depois, clique no botão Sign up for GitHub.

Etapa 2: Crie sua conta do IBM Bluemix #

Consulte Bluemix Get Started sobre como criar uma conta ou siga o exemplo anterior sobre como criar sua conta Bluemix em esta página.

Etapa 3: implante seu pacote de API Rest no GitHub #

3.a Se você não sabe como usar o Git, por favor leia o Tutorial do Git ou você pode usar o SourceTree uma implementação do Git baseada em GUI. Por favor, leia o Tutorial SourceTree para saber mais sobre SourceTree.

3.b Faça upload do código da API Rest para o GitHub usando o comando git push. Por outro lado, você pode bifurcar ou clonar minha API Rest no GitHub, aplicar suas próprias alterações conforme necessário e fazer upload do código para sua conta do GitHub. Observação: minha API Rest de amostra contém uma implementação de amostra do MongoDB e APICache.

Etapa 4: Implemente seu repositório GitHub no IBM Bluemix usando Toolchain (ferramenta de Implementação Contínua) #

4.a Faça logon em sua conta do Bluemix.

4.b Clique em Menu para mostrar outros itens.

Em seguida, selecione “Serviços”.

E, em seguida, clique em “DevOps”.

4.c Selecione Cadeias de Ferramentas. Em seguida, clique no botão Criar uma cadeia de ferramentas.

4.d Selecione Outros Modelos

4.e Insira o Nome da cadeia de ferramentas desejado, observe que isso será exibido em seu URL para APIs.

4.f Aguarde alguns segundos para que o Bluemix crie seu Toolchain. Depois, clique em Adicionar uma ferramenta.

4.g Escolha GitHub

4.h Agora, precisamos vincular nosso repositório existente no GitHub a este Toolchain. Em Tipo de repositório, escolha Existente. Em seguida, escolha a URL de repositório correta para sua origem da API Rest. Depois disso, fica a seu critério acompanhar as alterações do repositório via Toolchain. Para fazer isso, marque a caixa de seleção Acompanhar a implantação de alterações de código.

4.i Agora o que precisamos é algo para construir e implantar nosso repositório da API Rest automaticamente. Então, precisamos adicionar outra ferramenta.

4.j Vamos adicionar o Delivery Pipeline

Em seguida, nomeie-o como Build and Deploy.

Após alguns segundos, o pipeline de entrega recém-adicionado será exibido. Clique.

4.k Precisamos adicionar estágios ao nosso Delivery Pipeline. Clique em Adicionar estágio.

4.l Crie o Estágio de construção no qual construirá automaticamente nosso pacote. Siga os passos da figura abaixo:

Precisamos adicionar um JOB que executará o comando npm install.

Selecione a opção Construir.

Em seguida, selecione npm em Builder Type. E então, no Build Shell Command, configure automaticamente para que ele execute o comando npm install.

Certifique-se de clicar no botão Salvar para salvar as alterações.

4.m Após criar o estágio de construção, agora precisamos criar o estágio Deploy. Vá e adicione outro estágio clicando em Add Stage.

Siga os passos definidos na figura abaixo:

Em seguida, na guia JOBS, clique em Add Job.

Em seguida, selecione a opção Implantar.

Em Tipo de Implementador, selecione Cloud Foundry (Serviço de Nuvem padrão do IBM Bluemix).

Nesse caso, selecionei o espaço dev. Para um projeto real, você pode querer selecionar um espaço melhor que possa lidar com seu uso de produção real. Depois, clique no botão Salvar.

4.n Agora todos os estágios estão configurados. Por padrão, todos os estágios estão no status Parar. Precisamos Executar todos os nossos estágios clicando nos botões Jogar curtir.

4.o Se você vir uma figura semelhante abaixo, significa que você configurou e implementou com sucesso nossa Implementação Contínua de nossas APIs RESTful via IBM Bluemix usando ToolChain.

Tente clicar no link como definir a figura acima para ver se a API está sendo executada conforme o esperado.

Etapa 5: Teste a API Rest usando o PostMan #

Postman é uma poderosa ferramenta de teste de API e pode ser instalada via Google Chrome como uma extensão. Certifique-se em seu teste de seguir a convenção de rota que você definiu em seu código expresso. Para saber mais como usar a ferramenta Postman, consulte o Postman Blog.

Instalação ou Configuração #

Instruções detalhadas sobre como configurar ou instalar o ibm-bluemix.

Obtendo o IBM Bluemix Setup para implementar um aplicativo de nó #

Passo 1: Crie uma conta bluemix #

Crie uma conta em https://console.ng.bluemix.net/registration/

Isto irá configurá-lo com um teste de 30 dias. Você não precisa pagar nada pelos recursos gratuitos e não precisa configurar o faturamento até o final da avaliação (embora nem todos os serviços estejam disponíveis).

Etapa 2: Instale as ferramentas de linha de comando do Bluemix e do Cloud Foundry #

Isso é opcional, você pode executar a maioria das ações por meio do console da web a partir da etapa 1. Os utilitários CLI podem ser baixados de https://new-console.ng.bluemix.net/docs/starters/install_cli.html e você deve instalar ambos o bluemix cli e o cloud foundry cli.

Etapa 2a: Conecte-se ao Bluemix #

$ bluemix api https://api.ng.bluemix.net

Etapa 2b: Faça login no Bluemix #

$ bluemix login -u nome de usuário -o org_name -s space_name

Você precisa especificar seu nome de usuário, org_name e space_name que você pode obter no console da Web na Etapa 1.

Após efetuar login com sucesso, você configurou o bluemix. Você pode aprender mais sobre a CLI e obter plugins adicionais em http://clis.ng.bluemix.net/ui/home.html
