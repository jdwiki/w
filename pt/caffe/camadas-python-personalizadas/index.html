Este tutorial guiará as etapas para criar uma camada personalizada simples para o Caffe usando python. Ao final, há alguns exemplos de camadas personalizadas. Normalmente, você criaria uma camada personalizada para implementar uma funcionalidade que não está disponível no Caffe, ajustando-a aos seus requisitos.

Criar uma camada personalizada python adiciona alguma sobrecarga à sua rede e provavelmente não é tão eficiente quanto uma camada personalizada C++. No entanto, desta forma, você não terá que compilar todo o café com sua nova camada.

Parâmetros #

Parâmetro Detalhes
topo Uma matriz com os blobs superiores da sua camada. Acesse os dados passados ​​para ele usando top[i].data, onde i é o índice de um blob específico
inferior Uma matriz com os blobs inferiores da sua camada. Acesse os dados passados ​​para ele usando bottom[i].data, onde i é o índice de um blob específico

- Construção do Caffe com camada Python #

O Caffe precisa ser compilado com a opção WITH_PYTHON_LAYER:

WITH_PYTHON_LAYER=1 make && make pycaffe

- Onde devo salvar o arquivo de classe? #

Você tem duas opções (pelo menos que eu saiba). Você pode salvar o arquivo de camada personalizada na mesma pasta em que executará o comando caffe (provavelmente onde seus arquivos prototxt estariam). Outra maneira, também a minha favorita, é salvar todas as suas camadas personalizadas em uma pasta e adicionar essa pasta ao seu PYTHONPATH.

Modelo de camada #

import caffe

class My_Custom_Layer(caffe.Layer):
    def setup(self, bottom, top):
        pass
        
    def forward(self, bottom, top):
        pass
        
    def reshape(self, bottom, top):
        pass

    def backward(self, bottom, top):
        pass

Coisas tão importantes para lembrar:

  • Sua camada personalizada deve herdar de caffe.Layer (não esqueça de importar caffe);
  • Você deve definir os quatro métodos a seguir: setup, forward, reshape e backward;
  • Todos os métodos possuem parâmetros top e bottom, que são os blobs que armazenam a entrada e a saída passadas para sua camada. Você pode acessá-lo usando top[i].data ou bottom[i].data, onde i é o índice do blob caso você tenha mais de um blob superior ou inferior.

- Método de configuração #

O método Setup é chamado uma vez durante a vida útil da execução, quando Caffe está instanciando todas as camadas. É aqui que você lerá parâmetros, instanciará buffers de tamanho fixo.

- Remodelar método #

Use o método de remodelação para inicialização/configuração que depende do tamanho do blob inferior (entrada da camada). É chamado uma vez quando a rede é instanciada.

- Método de encaminhamento #

O método Forward é chamado para cada lote de entrada e é onde estará a maior parte de sua lógica.

- Método para trás #

O método Backward é chamado durante a passagem para trás da rede. Por exemplo, em uma camada do tipo convolução, seria aqui que você calcularia os gradientes. Isso é opcional (uma camada pode ser somente de encaminhamento).

Modelo de Prototxt #

Ok, então agora você tem sua camada projetada! É assim que você define em seu arquivo .prototxt:

layer {
  name: "LayerName"
  type: "Python"
  top: "TopBlobName"
  bottom: "BottomBlobName"
  python_param {
    module: "My_Custom_Layer_File"
    layer: "My_Custom_Layer_Class"
    param_str: '{"param1": 1,"param2":True, "param3":"some string"}'
  }
  include{
        phase: TRAIN
  }
}

Observações importantes:

  • tipo deve ser Python;
  • Você deve ter um dicionário python_param com pelo menos os parâmetros module e layer;
  • módulo refere-se ao arquivo onde você implementou sua camada (sem o .py);
  • camada refere-se ao nome da sua classe;
  • Você pode passar parâmetros para a camada usando param_str (mais sobre como acessá-los abaixo);
  • Assim como qualquer outra camada, você pode definir em qual fase deseja que ela esteja ativa (veja os exemplos para ver como você pode verificar a fase atual);

Medir Camada #

Neste exemplo, projetaremos uma camada de “medida”, que gera a precisão e uma matriz de confusão para um problema binário durante o treinamento e a precisão, taxa de falsos positivos e taxa de falsos negativos durante o teste/validação. Embora o Caffe já tenha uma camada de precisão, às vezes você quer algo mais, como uma medida F.

Este é meu measureLayer.py com minha definição de classe:

#Remark: This class is designed for a binary problem, where the first class would be the 'negative'
# and the second class would be 'positive'

import caffe
TRAIN = 0
TEST = 1

class Measure_Layer(caffe.Layer):
    #Setup method
    def setup(self, bottom, top):
        #We want two bottom blobs, the labels and the predictions
        if len(bottom) != 2:
            raise Exception("Wrong number of bottom blobs (prediction and label)") 

        #And some top blobs, depending on the phase
        if self.phase = TEST and len(top) != 3:
            raise Exception("Wrong number of top blobs (acc, FPR, FNR)")
        if self.phase = TRAIN and len(top) != 5:
            raise Exception("Wrong number of top blobs (acc, tp, tn, fp and fn)")
       
        #Initialize some attributes
        self.TPs = 0.0
        self.TNs = 0.0
        self.FPs = 0.0
        self.FNs = 0.0
        self.totalImgs = 0

    #Forward method
    def forward(self, bottom, top):
        #The order of these depends on the prototxt definition
        predictions = bottom[0].data
        labels = bottom[1].data

        self.totalImgs += len(labels)

        for i in range(len(labels)): #len(labels) is equal to the batch size
                pred = predictions[i]   #pred is a tuple with the normalized probability 
                                        #of a sample i.r.t. two classes
                lab = labels[i]
                
                if pred[0] > pred[1]:
                        if lab == 1.0:
                                self.FNs += 1.0
                        else:
                                self.TNs += 1.0
                else:
                        if lab == 1.0:
                                self.TPs += 1.0
                        else:
                                self.FPs += 1.0

        acc = (self.TPs + self.TNs) / self.totalImgs
        
        try: #just assuring we don't divide by 0
                fpr = self.FPs / (self.FPs + self.TNs)
        except:
                fpr = -1.0

        try: #just assuring we don't divide by 0
                fnr = self.FNs / (self.FNs + self.TPs)
        except:
                fnr = -1.0
           
       #output data to top blob
       top[0].data = acc
       if self.phase == TRAIN:
           top[1].data = self.TPs
           top[2].data = self.TNs
           top[3].data = self.FPs
           top[4].data = self.FNs
       elif self.phase == TEST:
           top[1].data = fpr
           top[2].data = fnr
           
    def reshape(self, bottom, top):
        """
        We don't need to reshape or instantiate anything that is input-size sensitive
        """
        pass

    def backward(self, bottom, top):
        """
        This layer does not back propagate
        """
        pass

Camada de dados #

Este exemplo é uma camada de dados personalizada, que recebe um arquivo de texto com caminhos de imagem, carrega um lote de imagens e as pré-processa. Apenas uma dica rápida, o Caffe já possui uma grande variedade de camadas de dados e provavelmente uma camada personalizada não é a maneira mais eficiente se você deseja apenas algo simples.
