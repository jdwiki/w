Atualizando objetos #

Adicionando propriedades #

Se desejar adicionar propriedades a um objeto existente, você pode usar o cmdlet Add-Member. Com PSObjects, os valores são mantidos em um tipo de “Propriedades da Nota”

$object = New-Object -TypeName PSObject -Property @{
        Name = $env:username
        ID = 12
        Address = $null
    }

Add-Member -InputObject $object -Name "SomeNewProp" -Value "A value" -MemberType NoteProperty

# Returns
PS> $Object
Name ID Address SomeNewProp
---- -- ------- -----------
nem  12         A value

Você também pode adicionar propriedades com Select-Object Cmdlet (as chamadas propriedades calculadas):

$newObject = $Object | Select-Object *, @{label='SomeOtherProp'; expression={'Another value'}}

# Returns
PS> $newObject
Name ID Address SomeNewProp SomeOtherProp
---- -- ------- ----------- -------------
nem  12         A value     Another value

O comando acima pode ser encurtado para isso:

$newObject = $Object | Select *,@{l='SomeOtherProp';e={'Another value'}}

Removendo propriedades #

Você pode usar o Cmdlet Select-Object para remover propriedades de um objeto:

$object = $newObject | Select-Object * -ExcludeProperty ID, Address

# Returns
PS> $object
Name SomeNewProp SomeOtherProp
---- ----------- -------------
nem  A value     Another value

Criando um novo objeto #

O PowerShell, ao contrário de outras linguagens de script, envia objetos por meio do pipeline. O que isso significa é que quando você envia dados de um comando para outro, é essencial poder criar, modificar e coletar objetos.

Criar um objeto é simples. A maioria dos objetos que você cria serão objetos personalizados no PowerShell, e o tipo a ser usado para isso é PSObject. O PowerShell também permitirá que você crie qualquer objeto que você possa criar no .NET.

Aqui está um exemplo de criação de novos objetos com algumas propriedades:

Opção 1: Novo-Objeto #

$newObject = New-Object -TypeName PSObject -Property @{
    Name = $env:username
    ID = 12
    Address = $null
}

# Returns
PS> $newObject
Name ID Address
---- -- -------
nem  12

Você pode armazenar o objeto em uma variável precedendo o comando com $newObject =

Você também pode precisar armazenar coleções de objetos. Isso pode ser feito criando uma variável de coleção vazia e adicionando objetos à coleção, assim:

$newCollection = @()
$newCollection += New-Object -TypeName PSObject -Property @{
    Name = $env:username
    ID = 12
    Address = $null
}

Você pode então desejar iterar através desta coleção objeto por objeto. Para fazer isso, localize a seção Loop na documentação.

Opção 2: Select-Object #

Uma maneira menos comum de criar objetos que você ainda encontrará na internet é a seguinte:

$newObject = 'unuseddummy' | Select-Object -Property Name, ID, Address
$newObject.Name = $env:username
$newObject.ID = 12

# Returns
PS> $newObject
Name ID Address
---- -- -------
nem  12

Opção 3: acelerador de tipo pscustomobject (PSv3+ necessário) #

O acelerador de tipo ordenado força o PowerShell a manter nossas propriedades na ordem em que as definimos. Você não precisa do acelerador de tipo ordenado para usar [PSCustomObject]:

$newObject = [PSCustomObject][Ordered]@{
    Name = $env:Username
    ID = 12
    Address = $null
}

# Returns
PS> $newObject
Name ID Address
---- -- -------
nem  12

Examinando um objeto #

Agora que você tem um objeto, pode ser bom descobrir o que é. Você pode usar o cmdlet Get-Member para ver o que é um objeto e o que ele contém:

Get-Item c:\windows | Get-Member

Isso rende:

TypeName: System.IO.DirectoryInfo

Seguido por uma lista de propriedades e métodos que o objeto possui.

Outra maneira de obter o tipo de um objeto é usar o método GetType, assim:

C:\> $Object = Get-Item C:\Windows
C:\> $Object.GetType()

IsPublic IsSerial Name                                     BaseType
-------- -------- ----                                     --------
True     True     DirectoryInfo                            System.IO.FileSystemInfo

Para exibir uma lista de propriedades que o objeto possui, juntamente com seus valores, você pode usar o cmdlet Format-List com seu parâmetro Property definido como : * (significando tudo).

Criando instâncias de classes genéricas #

Nota: exemplos escritos para PowerShell 5.1 Você pode criar instâncias de classes genéricas

#Nullable System.DateTime
[Nullable[datetime]]$nullableDate = Get-Date -Year 2012
$nullableDate
$nullableDate.GetType().FullName
$nullableDate = $null
$nullableDate

#Normal System.DateTime
[datetime]$aDate = Get-Date -Year 2013
$aDate
$aDate.GetType().FullName
$aDate = $null #Throws exception when PowerShell attempts to convert null to

$dict.GetType().FullName

$dict.Add(2, 'b')
$dict.Add(3, 'c')
