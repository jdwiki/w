Sintaxe #

  • ’texto’ -match ‘RegExPattern’
  • ’texto’ -substituir ‘RegExPattern’, ’novo valor’
  • [regex]::Match(“text”,“pattern”) #Correspondência única
  • [regex]::Matches(“text”,“pattern”) #Várias correspondências
  • [regex]::Replace(“texto”,“padrão”,“novovalor”)
  • [regex]::Replace(“texto”,“padrão”, {param($m) }) #MatchEvaluator
  • [regex]::Escape(“input”) #Escape de caracteres especiais

Partida única #

Você pode determinar rapidamente se um texto inclui um padrão específico usando Regex. Existem várias maneiras de trabalhar com Regex no PowerShell.

#Sample text
$text = @"
This is (a) sample
text, this is
a (sample text)
"@

#Sample pattern: Content wrapped in ()
$pattern = '\(.*?\)'

Usando o operador -Match #

Para determinar se uma string corresponde a um padrão usando o operador interno -matches, use a sintaxe 'input' -match 'pattern'. Isso retornará true ou false dependendo do resultado da pesquisa. Se houver correspondência, você pode visualizar a correspondência e os grupos (se definidos no padrão) acessando a variável $Matches.

> $text -match $pattern
True

> $Matches

Name Value
---- -----
0    (a)

Você também pode usar -match para filtrar uma matriz de strings e retornar apenas as strings que contêm uma correspondência.

> $textarray = @"
This is (a) sample
text, this is
a (sample text)
"@ -split "`n"

> $textarray -match $pattern
This is (a) sample
a (sample text)

Usando Select-String #

O PowerShell 2.0 introduziu um novo cmdlet para pesquisar texto usando regex. Ele retorna um objeto MatchInfo por entrada de texto que contém uma correspondência. Você pode acessar suas propriedades para encontrar grupos correspondentes etc.

> $m = Select-String -InputObject $text -Pattern $pattern

> $m

This is (a) sample
text, this is
a (sample text)

> $m | Format-List *

IgnoreCase : True
LineNumber : 1
Line       : This is (a) sample
             text, this is
             a (sample text)
Filename   : InputStream
Path       : InputStream
Pattern    : \(.*?\)
Context    : 
Matches    : {(a)}

Assim como -match, Select-String também pode ser usado para filtrar através de um array de strings canalizando um array para ele. Ele cria um objeto MatchInfo por string que inclui uma correspondência.

> $textarray | Select-String -Pattern $pattern

This is (a) sample
a (sample text)

#You can also access the matches, groups etc.
> $textarray | Select-String -Pattern $pattern | fl *


IgnoreCase : True
LineNumber : 1
Line       : This is (a) sample
Filename   : InputStream
Path       : InputStream
Pattern    : \(.*?\)
Context    : 
Matches    : {(a)}

IgnoreCase : True
LineNumber : 3
Line       : a (sample text)
Filename   : InputStream
Path       : InputStream
Pattern    : \(.*?\)
Context    : 
Matches    : {(sample text)}

Select-String também pode pesquisar usando um padrão de texto normal (sem regex) adicionando a opção -SimpleMatch.

Usando [RegEx]::Match() #

Você também pode usar o método estático Match() disponível na classe .NET [RegEx].

> [regex]::Match($text,$pattern)

Groups   : {(a)}
Success  : True
Captures : {(a)}
Index    : 8
Length   : 3
Value    : (a)

> [regex]::Match($text,$pattern) | Select-Object -ExpandProperty Value
(a)

Substituir #

Uma tarefa comum para regex é substituir o texto que corresponde a um padrão por um novo valor.

#Sample text
$text = @"
This is (a) sample
text, this is
a (sample text)
"@

#Sample pattern: Text wrapped in ()
$pattern = '\(.*?\)'

#Replace matches with:
$newvalue = 'test'

Usando o operador -Replace #

O operador -replace no PowerShell pode ser usado para substituir o texto que corresponde a um padrão por um novo valor usando a sintaxe 'input' -replace 'pattern', 'newvalue'.

> $text -replace $pattern, $newvalue
This is test sample
text, this is
a test

Usando o método [RegEx]::Replace() #

A substituição de correspondências também pode ser feita usando o método Replace() na classe .NET [RegEx].

[regex]::Replace($text, $pattern, 'test')
This is test sample
text, this is
a test

Substitua o texto pelo valor dinâmico usando um MatchEvalutor #

Às vezes, você precisa substituir um valor que corresponde a um padrão por um novo valor baseado nessa correspondência específica, tornando impossível prever o novo valor. Para esses tipos de cenários, um MatchEvaluator pode ser muito útil.

No PowerShell, um MatchEvaluator é tão simples quanto um scriptblock com um único parâmetro que contém um objeto Match para a correspondência atual. A saída da ação será o novo valor para essa correspondência específica. MatchEvalutor pode ser usado com o método estático [Regex]::Replace().

Exemplo: Substituindo o texto dentro de () por seu comprimento

#Sample text
$text = @"
This is (a) sample
text, this is
a (sample text)
"@
    
#Sample pattern: Content wrapped in ()
$pattern = '(?<=\().*?(?=\))'

$MatchEvalutor = {
    param($match)

    #Replace content with length of content
    $match.Value.Length

}

Resultado:

> [regex]::Replace($text, $pattern, $MatchEvalutor)

This is 1 sample
text, this is
a 11

Exemplo: coloque “amostra” em maiúscula

#Sample pattern: "Sample"
$pattern = 'sample'

$MatchEvalutor = {
    param($match)

    #Return match in upper-case
    $match.Value.ToUpper()

}

Resultado:

> [regex]::Replace($text, $pattern, $MatchEvalutor)

This is (a) SAMPLE
text, this is
a (SAMPLE text)

Caracteres especiais de escape #

Um padrão regex usa muitos caracteres especiais para descrever um padrão. Ex., . significa “qualquer caractere”, + é “um ou mais” etc.

Para usar esses caracteres, como .,+ etc., em um padrão, você precisa escapar deles para remover seu significado especial. Isso é feito usando o caractere de escape que é uma barra invertida \ em regex. Exemplo: Para procurar por +, você usaria o padrão \+.

Pode ser difícil lembrar de todos os caracteres especiais em regex, então para escapar de cada caractere especial em uma string que você deseja pesquisar, você pode usar o método [RegEx]::Escape("input").

> [regex]::Escape("(foo)")
\(foo\)

> [regex]::Escape("1+1.2=2.2")
1\+1\.2=2\.2

Várias correspondências #

Existem várias maneiras de encontrar todas as correspondências de um padrão em um texto.

#Sample text
$text = @"
This is (a) sample
text, this is
a (sample text)
"@

#Sample pattern: Content wrapped in ()
$pattern = '\(.*?\)'

Usando Select-String #

Você pode encontrar todas as correspondências (correspondência global) adicionando a opção -AllMatches a Select-String.

> $m = Select-String -InputObject $text -Pattern $pattern -AllMatches

> $m | Format-List *

IgnoreCase : True
LineNumber : 1
Line       : This is (a) sample
             text, this is
             a (sample text)
Filename   : InputStream
Path       : InputStream
Pattern    : \(.*?\)
Context    : 
Matches    : {(a), (sample text)}

#List all matches
> $m.Matches

Groups   : {(a)}
Success  : True
Captures : {(a)}
Index    : 8
Length   : 3
Value    : (a)

Groups   : {(sample text)}
Success  : True
Captures : {(sample text)}
Index    : 37
Length   : 13
Value    : (sample text)

#Get matched text
> $m.Matches | Select-Object -ExpandProperty Value
(a)
(sample text)

Usando [RegEx]::Correspondências() #

O método Matches() na classe .NET `[regex] também pode ser usado para fazer uma busca global por múltiplas correspondências.

> [regex]::Matches($text,$pattern)

Groups   : {(a)}
Success  : True
Captures : {(a)}
Index    : 8
Length   : 3
Value    : (a)

Groups   : {(sample text)}
Success  : True
Captures : {(sample text)}
Index    : 37
Length   : 13
Value    : (sample text)

> [regex]::Matches($text,$pattern) | Select-Object -ExpandProperty Value

(a)
(sample text)
