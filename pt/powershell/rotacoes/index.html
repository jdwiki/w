Um loop é uma sequência de instruções que é repetida continuamente até que uma determinada condição seja alcançada. Ser capaz de fazer seu programa executar repetidamente um bloco de código é uma das tarefas mais básicas, porém úteis, na programação. Um loop permite que você escreva uma instrução muito simples para produzir um resultado significativamente maior simplesmente por repetição. Se a condição foi alcançada, a próxima instrução “cai” para a próxima instrução sequencial ou desvia para fora do loop.

Sintaxe #

  • for ( <Inicialização>; <Condição>; <Repetição> ) { <Bloco_Script> }

  • <Coleção> | Foreach-Object { <Script_Block_with_$__as_current_item> }

  • foreach ( <Item> em <Collection> ) { <Script_Block> }

  • while ( <Condição> ){ <Script_Block> }

  • do { <Script_Block> } while ( <Condição> )

  • faça { <Script_Block> } até ( <Condição> )

  • <Collection>.foreach( { <Script_Block_with_$__as_current_item> } )

Para cada #

Existem várias maneiras de executar um loop foreach no PowerShell e todas elas trazem suas próprias vantagens e desvantagens:

Solução Vantagens Desvantagens
Declaração Foreach O mais rápido. Funciona melhor com coleções estáticas (armazenadas em uma variável). Nenhuma entrada ou saída de pipeline
Método ForEach() Mesma sintaxe de scriptblock como Foreach-Object, mas mais rápido. Funciona melhor com coleções estáticas (armazenadas em uma variável). Suporta saída de pipeline. Não há suporte para entrada de pipeline. Requer PowerShell 4.0 ou superior
Foreach-Object (cmdlet) Suporta entrada e saída de pipeline. Suporta begin e end-scriptblocks para inicialização e fechamento de conexões etc. Solução mais flexível. Mais lento

atuação #

$foreach = Measure-Command { foreach ($i in (1..1000000)) { $i * $i } }
$foreachmethod = Measure-Command { (1..1000000).ForEach{ $_ * $_ } }
$foreachobject = Measure-Command { (1..1000000) | ForEach-Object { $_ * $_ } }

"Foreach: $($foreach.TotalSeconds)"
"Foreach method: $($foreachmethod.TotalSeconds)"
"ForEach-Object: $($foreachobject.TotalSeconds)"

Example output:

Foreach: 1.9039875
Foreach method: 4.7559563
ForEach-Object: 10.7543821

Embora o Foreach-Object seja o mais lento, seu suporte a pipeline pode ser útil, pois permite processar itens à medida que eles chegam (enquanto lê um arquivo, recebe dados etc.). Isso pode ser muito útil ao trabalhar com big data e pouca memória, pois você não precisa carregar todos os dados na memória antes do processamento.

Para cada #

ForEach tem dois significados diferentes no PowerShell. Um é uma palavra-chave e o outro é um alias para o [ForEach-Object] (https://www.wikiod.com/pt/powershell/rotacoes#ForEach-Object) cmdlet. O primeiro é descrito aqui.

Este exemplo demonstra a impressão de todos os itens em uma matriz para o host do console:

$Names = @('Amy', 'Bob', 'Celine', 'David')

ForEach ($Name in $Names)
{
    Write-Host "Hi, my name is $Name!"
}

Este exemplo demonstra a captura da saída de um loop ForEach:

$Numbers = ForEach ($Number in 1..20) {
    $Number # Alternatively, Write-Output $Number
}

Como o último exemplo, este exemplo, em vez disso, demonstra a criação de um array antes de armazenar o loop:

$Numbers = @()
ForEach ($Number in 1..20)
{
    $Numbers += $Number
}

Por #

for($i = 0; $i -le 5; $i++){
    "$i"
}

Um uso típico do loop for é operar em um subconjunto dos valores em uma matriz. Na maioria dos casos, se você quiser iterar todos os valores em uma matriz, considere usar uma instrução foreach.

Método ForEach() #

Em vez do cmdlet ForEach-Object, aqui também há a possibilidade de usar um método ForEach diretamente em arrays de objetos como este

(1..10).ForEach({$_ * $_})

ou - se desejado - os parênteses ao redor do bloco de script podem ser omitidos

(1..10).ForEach{$_ * $_}

Ambos resultarão na saída abaixo

1
4
9
16
25
36
49
64
81
100

ForEach-Object #

O cmdlet ForEach-Object funciona de forma semelhante à instrução [foreach](https://www.wikiod.com/pt/powershell/rotacoes#Para cada), mas recebe sua entrada do pipeline .

Uso básico #

$object | ForEach-Object {
    code_block
}

Exemplo:

$names = @("Any","Bob","Celine","David")
$names | ForEach-Object {
    "Hi, my name is $_!"
}

Foreach-Object tem dois aliases padrão, foreach e % (sintaxe abreviada). O mais comum é % porque foreach pode ser confundido com a [instrução foreach](https://www.wikiod.com/pt/powershell/rotacoes#Para cada). Exemplos:

$names | % {  
    "Hi, my name is $_!"
} 

$names | foreach {  
    "Hi, my name is $_!"
}

Uso avançado #

Foreach-Object se destaca das soluções alternativas foreach porque é um cmdlet, o que significa que foi projetado para usar o pipeline. Por isso, tem suporte para três scriptblocks como um cmdlet ou função avançada:

  • Begin: Executado uma vez antes de percorrer os itens que chegam do pipeline. Normalmente usado para criar funções para uso no loop, criar variáveis, abrir conexões (banco de dados, web+) etc.
  • Processo: Executado uma vez por item recebido do pipeline. Bloco de código foreach “Normal”. Este é o padrão usado nos exemplos acima quando o parâmetro não é especificado.
  • Fim: Executado uma vez após o processamento de todos os itens. Normalmente usado para fechar conexões, gerar um relatório etc.

Exemplo:

"Any","Bob","Celine","David" | ForEach-Object -Begin {
    $results = @()
} -Process {
    #Create and store message
    $results += "Hi, my name is $_!"
} -End {
    #Count messages and output
    Write-Host "Total messages: $($results.Count)"
    $results
}

Continuar #

O operador Continue funciona em loops For, ForEach, While e Do. Ele pula a iteração atual do loop, pulando para o topo do loop mais interno.

$i =0
while ($i -lt 20) {
    $i++ 
    if ($i -eq 7) { continue }
    Write-Host $I
}

O código acima produzirá de 1 a 20 para o console, mas perderá o número 7.

Observação: Ao usar um loop de pipeline, você deve usar return em vez de Continue.

Parar #

O operador break sairá de um loop de programa imediatamente. Ele pode ser usado em loops For, ForEach, While e Do ou em uma instrução Switch.

$i = 0
while ($i -lt 15) {
    $i++ 
    if ($i -eq 7) {break}
    Write-Host $i
}

O acima contará até 15, mas pare assim que 7 for atingido.

Observação: Ao usar um loop de pipeline, break se comportará como continue. Para simular break no loop de pipeline você precisa incorporar alguma lógica adicional, cmdlet, etc. É mais fácil ficar com loops que não sejam de pipeline se você precisar usar break.

Quebrar etiquetas

Break também pode chamar um rótulo que foi colocado na frente da instanciação de um loop:

$i = 0
:mainLoop While ($i -lt 15) {
    Write-Host $i -ForegroundColor 'Cyan'
    $j = 0
    While ($j -lt 15) {
        Write-Host $j -ForegroundColor 'Magenta'
        $k = $i*$j
        Write-Host $k -ForegroundColor 'Green'
        if ($k -gt 100) {
            break mainLoop
        }
        $j++
    }
    $i++
}

Nota: Este código incrementará $i para 8 e $j para 13, o que fará com que $k seja igual a 104. Já que $k excede 100, o código irá então sair de ambos os loops.

Enquanto #

Um loop while avaliará uma condição e, se true, executará uma ação. Enquanto a condição for avaliada como verdadeira, a ação continuará a ser executada.

while(condition){
  code_block
}

O exemplo a seguir cria um loop que contará de 10 a 0

$i = 10
while($i -ge 0){
    $i
    $i--
}

Ao contrário do loop Do-Enquanto a condição é avaliada antes da primeira execução da ação. A ação não será executada se a condição inicial for avaliada como falsa.

Observação: ao avaliar a condição, o PowerShell tratará a existência de um objeto de retorno como verdadeiro. Isso pode ser usado de várias maneiras, mas abaixo está um exemplo para monitorar um processo. Este exemplo irá gerar um processo de bloco de notas e, em seguida, suspender o shell atual enquanto esse processo estiver em execução. Quando você fecha manualmente a instância do bloco de notas, a condição while falhará e o loop será interrompido.

Start-Process notepad.exe
while(Get-Process notepad -ErrorAction SilentlyContinue){
  Start-Sleep -Milliseconds 500
}

Fazer #

Do-loops são úteis quando você sempre deseja executar um codeblock pelo menos uma vez. Um Do-loop avaliará a condição após executar o codeblock, ao contrário de um while-loop que o faz antes de executar o codeblock.

Você pode usar do-loops de duas maneiras:

  • Loop enquanto a condição for verdadeira:

    Do {
    code_block
} while (condition)

  • Faça um loop até que a condição seja verdadeira, ou seja, faça um loop enquanto a condição for falsa:

    Do {
    code_block
} until (condition)

Exemplos reais:

$i = 0

Do {
    $i++
    "Number $i"
} while ($i -ne 3)

Do {
    $i++
    "Number $i"
} until ($i -eq 3)

Do-While e Do-Until são loops antônimos. Se o código estiver dentro do mesmo, a condição será invertida. O exemplo acima ilustra esse comportamento.
