CIM x WMI #

A partir do PowerShell 3.0, há duas maneiras de trabalhar com classes de gerenciamento no PowerShell, WMI e CIM. O PowerShell 1.0 e 2.0 suportavam apenas o módulo WMI, que agora é substituído pelo novo e aprimorado módulo CIM. Em uma versão posterior do PowerShell, os cmdlets WMI serão removidos.

Comparação dos módulos CIM e WMI:

CIM-cmdlet WMI-cmdlet O que faz
Get-CimInstance Get-WmiObject Obtém objetos CIM/WMI para uma classe
Invoke-CimMethod Invoke-WmiMethod Invoca um método de classe CIM/WMI
Register-CimIndicationEvent Register-WmiEvent Registra evento para uma classe CIM/WMI
Remove-CimInstance Remove-WmiObject Remover objeto CIM/WMI
Set-CimInstance Set-WmiInstance Atualiza/Salva o objeto CIM/WMI
Get-CimAssociatedInstance N/A Obter instâncias associadas (objetos/classes vinculados)
Get-CimClass Get-WmiObject -Lista Listar classes CIM/WMI
New-CimInstance N/A Criar novo objeto CIM
Get-CimSession N/A Lista sessões CIM
New-CimSession N/A Criar nova sessão CIM
New-CimSessionOption N/A Cria objeto com opções de sessão; protocolo, codificação, desabilitar criptografia etc. (para uso com New-CimSession)
Remove-CimSession N/A Remove/para a sessão CIM

Consultando objetos #

CIM/WMI é mais comumente usado para consultar informações ou configuração em um dispositivo. É uma classe que representa uma configuração, processo, usuário etc. No PowerShell existem várias maneiras de acessar essas classes e instâncias, mas as formas mais comuns são usando o Get-CimInstance (CIM) ou Get-WmiObject ( WMI) cmdlets.

Listar todos os objetos para classe CIM #

Você pode listar todas as instâncias de uma classe.

CIM:

> Get-CimInstance -ClassName Win32_Process

ProcessId Name                         HandleCount WorkingSetSize VirtualSize  
--------- ----                         ----------- -------------- -----------

4         System                       1459        32768          3563520      
480       Secure System                0           3731456        0            
484       smss.exe                     52          372736         2199029891072
....
....

WMI:

Get-WmiObject -Class Win32_Process

Usando um filtro #

Você pode aplicar um filtro para obter apenas instâncias específicas de uma classe CIM/WMI. Os filtros são escritos usando WQL (padrão) ou CQL (adicionar -QueryDialect CQL). -Filter usa a parte WHERE de uma consulta WQL/CQL completa.

CIM:

Get-CimInstance -ClassName Win32_Process -Filter "Name = 'powershell.exe'"

ProcessId Name           HandleCount WorkingSetSize VirtualSize  
--------- ----           ----------- -------------- -----------  
4800      powershell.exe 676         88305664       2199697199104

WMI:

Get-WmiObject -Class Win32_Process -Filter "Name = 'powershell.exe'"

Usando uma consulta WQL: #

Você também pode usar uma consulta WQL/CQL para consultar e filtrar instâncias.

CIM:

Get-CimInstance -Query "SELECT * FROM Win32_Process WHERE Name = 'powershell.exe'"

ProcessId Name           HandleCount WorkingSetSize VirtualSize  
--------- ----           ----------- -------------- -----------  
4800      powershell.exe 673         88387584       2199696674816

Classes e namespaces #

Existem muitas classes disponíveis em CIM e WMI que são separadas em vários namespaces. O namespace mais comum (e padrão) no Windows é root/cimv2. Para encontrar a classe correta, pode ser útil listar todas ou pesquisar.

Listar turmas disponíveis #

Você pode listar todas as classes disponíveis no namespace padrão (root/cimv2) em um computador.

CIM:

Get-CimClass

WMI:

Get-WmiObject -List

Pesquisar uma aula #

Você pode pesquisar classes específicas usando curingas. Ex: Encontre classes que contenham a palavra process.

CIM:

> Get-CimClass -ClassName "*Process*"

   NameSpace: ROOT/CIMV2

CimClassName                        CimClassMethods      CimClassProperties                                                           
------------                        ---------------      ------------------

Win32_ProcessStopTrace              {}                   {SECURITY_DESCRIPTOR, TIME_CREATED, ParentProcessID, ProcessID...}           
CIM_Process                         {}                   {Caption, Description, InstallDate, Name...}                                 
Win32_Process                       {Create, Terminat... {Caption, Description, InstallDate, Name...}                                 
CIM_Processor                       {SetPowerState, R... {Caption, Description, InstallDate, Name...}                                 
Win32_Processor                     {SetPowerState, R... {Caption, Description, InstallDate, Name...}
...

WMI:

Get-WmiObject -List -Class "*Process*"

Listar classes em um namespace diferente #

O namespace root é simplesmente chamado de root. Você pode listar classes em outro namespace usando o parâmetro -NameSpace.

CIM:

> Get-CimClass -Namespace "root/SecurityCenter2"   

   NameSpace: ROOT/SecurityCenter2

CimClassName                        CimClassMethods      CimClassProperties                                                           
------------                        ---------------      ------------------
....                                                   
AntiSpywareProduct                  {}                   {displayName, instanceGuid, pathToSignedProductExe, pathToSignedReportingE...
AntiVirusProduct                    {}                   {displayName, instanceGuid, pathToSignedProductExe, pathToSignedReportingE...
FirewallProduct                     {}                   {displayName, instanceGuid, pathToSignedProductExe, pathToSignedReportingE...

WMI:

Get-WmiObject -Class "__Namespace" -Namespace "root"
