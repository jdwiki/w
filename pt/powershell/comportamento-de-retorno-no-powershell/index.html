Ele pode ser usado para Sair do escopo atual, que pode ser uma função, script ou bloco de script. No PowerShell, o resultado de cada instrução é retornado como saída, mesmo sem uma palavra-chave Return explícita ou para indicar que o fim do escopo foi atingido.

Você pode ler mais sobre a semântica de retorno na página about_Return no TechNet ou invocando get-help return em um prompt do PowerShell.

Perguntas e respostas notáveis ​​com mais exemplos/explicações:

  • http://stackoverflow.com/questions/10286164/function-return-value-in-powershell
  • http://stackoverflow.com/questions/22663848/powershell-function-doesnt-have-proper-return-value

about_return no MSDN explica sucintamente:

A palavra-chave Return sai de uma função, script ou bloco de script. Pode ser usado para sair de um escopo em um ponto específico, para retornar um valor ou para indicam que o fim do escopo foi alcançado.

Usuários familiarizados com linguagens como C ou C# podem querer usar a palavra-chave Return para tornar explícita a lógica de deixar um escopo explícito.

No Windows PowerShell, os resultados de cada instrução são retornados como saída, mesmo sem uma instrução que contenha a palavra-chave Return. Linguagens como C ou C# retornam apenas o valor ou valores que são especificado pela palavra-chave Return.

Saída antecipada #

function earlyexit {
    "Hello"
    return
    "World"
}

“Hello” será colocado no pipeline de saída, “World” não será

Peguei vocês! Retorno no pipeline #

get-childitem | foreach-object { if ($_.IsReadOnly) { return } }

Os cmdlets de pipeline (ex: ForEach-Object, Where-Object, etc) operam em closures. O retorno aqui só passará para o próximo item no pipeline, não sairá do processamento. Você pode usar break em vez de return se quiser sair do processamento.

get-childitem | foreach-object { if ($_.IsReadOnly) { break } }

Retorna com um valor #

(parafraseado de about_return)

Os métodos a seguir terão os mesmos valores no pipeline

function foo {
    $a = "Hello"
    return $a
}

function bar {
    $a = "Hello"
    $a
    return
} 

function quux {
    $a = "Hello"
    $a
}

Peguei vocês! Ignorando saída indesejada #

Inspirado por

  • http://stackoverflow.com/questions/22663848/powershell-function-doesnt-have-proper-return-value

    function bar { [System.Collections.ArrayList]$MyVariable = @() $MyVariable.Add(“a”) | Out-Null $MyVariable.Add(“b”) | Out-Null $MyVariable }

O Out-Null é necessário porque o método .NET ArrayList.Add retorna o número de itens na coleção após a adição. Se omitido, o pipeline conteria 1, 2, "a", "b"

Existem várias maneiras de omitir saídas indesejadas:

function bar
{
    # New-Item cmdlet returns information about newly created file/folder
    New-Item "test1.txt" | out-null
    New-Item "test2.txt" > $null
    [void](New-Item "test3.txt")
    $tmp = New-Item "test4.txt"
}

Observação: para saber mais sobre por que preferir > $null, consulte [tópico ainda não criado].

Como trabalhar com retornos de funções #

Uma função retorna tudo o que não é capturado por outra coisa. Se você usar a palavra-chave return, todas as instruções após a linha de retorno não serão executadas!

Assim:

Function Test-Function
{
    Param
    (
        [switch]$ExceptionalReturn
    )
    "Start"
    if($ExceptionalReturn){Return "Damn, it didn't work!"}
    New-ItemProperty -Path "HKCU:\" -Name "test" -Value "TestValue" -Type "String"
    Return "Yes, it worked!"
}

Função de teste Retornará:

  • Começar
  • A chave de registro recém-criada (isso ocorre porque existem algumas instruções que criam uma saída que você pode não estar esperando)
  • Sim, funcionou!

Test-Function -ExceptionalReturn Retornará:

  • Começar
  • Droga, não funcionou!

Se você fizer assim:

Function Test-Function
{
    Param
    (
        [switch]$ExceptionalReturn
    )
    . {
       "Start"
        if($ExceptionalReturn)
        {
            $Return = "Damn, it didn't work!"
            Return
        }
        New-ItemProperty -Path "HKCU:\" -Name "test" -Value "TestValue" -Type "String"
        $Return = "Yes, it worked!"
        Return 
    } | Out-Null
    Return $Return
}

Função de teste Retornará:

  • Sim, funcionou!

Test-Function -ExceptionalReturn Retornará:

  • Droga, não funcionou!

Com este truque você pode controlar a saída retornada mesmo se você não tiver certeza do que cada declaração irá cuspir.

Funciona assim

.{<Statements>} | Out-Null

a . faz o seguinte scriptblock incluído no código o {} marca o bloco de script o | Out-Null canaliza qualquer saída inesperada para Out-Null (portanto, desapareceu!) Como o scriptblock está incluído, ele obtém o mesmo escopo que o restante da função. Assim você pode acessar variáveis ​​que foram feitas dentro do scriptblock.
