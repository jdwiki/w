Variáveis ​​são usadas para armazenar valores. Seja o valor de qualquer tipo , precisamos armazená-lo em algum lugar para que possamos usá-lo em todo o console/script. Os nomes de variáveis ​​no PowerShell começam com um $, como em $Variable1, e os valores são atribuídos usando =, como $Variable1 = “Value 1”. tipos de variáveis; como cadeias de texto, números inteiros, decimais, matrizes e até tipos avançados, como números de versão ou endereços IP.

Removendo uma variável #

Para remover uma variável da memória, pode-se usar o cmdlet Remove-Item. Nota: O nome da variável NÃO inclui o $.

Remove-Item Variable:\foo

Variable tem um provedor para permitir que a maioria dos cmdlets *-item funcionem como sistemas de arquivos.

Outro método para remover a variável é usar o cmdlet Remove-Variable e seu alias rv

$var = "Some Variable" #Define variable 'var' containing the string 'Some Variable'
$var #For test and show string 'Some Variable' on the console

Remove-Variable -Name var
$var 

#also can use alias 'rv'
rv var

Variável simples #

Todas as variáveis ​​no powershell começam com um sinal de dólar americano ($). O exemplo mais simples disso é:

$foo = "bar"

Esta instrução aloca uma variável chamada foo com um valor de string de “bar”.

Alcance #

O escopo padrão para uma variável é o contêiner delimitador. Se estiver fora de um script ou outro contêiner, o escopo será Global. Para especificar um scope, ele é prefixado ao nome da variável $scope:varname assim:

$foo = "Global Scope"
function myFunc {
    $foo = "Function (local) scope"
    Write-Host $global:foo
    Write-Host $local:foo
    Write-Host $foo
}
myFunc
Write-Host $local:foo
Write-Host $foo

Resultado: <pré> Global Scope Function (local) scope Function (local) scope Global Scope Global Scope

Matrizes #

A declaração de array no Powershell é quase o mesmo que instanciar qualquer outra variável, ou seja, você usa uma sintaxe $name = . Os itens no array são declarados separando-os por vírgulas(,):

$myArrayOfInts = 1,2,3,4
$myArrayOfStrings = "1","2","3","4"

Adicionando a um array #

Adicionar a um array é tão simples quanto usar o operador +:

$myArrayOfInts = $myArrayOfInts + 5
//now contains 1,2,3,4 & 5!

Combinando matrizes #

Novamente, isso é tão simples quanto usar o operador +

$myArrayOfInts = 1,2,3,4
$myOtherArrayOfInts = 5,6,7
$myArrayOfInts = $myArrayOfInts + $myOtherArrayOfInts
//now 1,2,3,4,5,6,7

Listar Atribuição de Várias Variáveis #

O Powershell permite atribuição múltipla de variáveis ​​e trata quase tudo como uma matriz ou lista. Isso significa que, em vez de fazer algo assim:

$input = "foo.bar.baz"
$parts = $input.Split(".")
$foo = $parts[0]
$bar = $parts[1]
$baz = $parts[2]

Você pode simplesmente fazer isso:

$foo, $bar, $baz = $input.Split(".")

Como o Powershell trata as atribuições dessa maneira como listas, se houver mais valores na lista do que itens em sua lista de variáveis ​​para atribuí-los, a última variável se tornará uma matriz dos valores restantes. Isso significa que você também pode fazer coisas assim:

$foo, $leftover = $input.Split(".") #Sets $foo = "foo", $leftover = ["bar","baz"]
$bar = $leftover[0] # $bar = "bar"
$baz = $leftover[1] # $baz = "baz"

Lendo uma saída de CmdLet #

Por padrão, o powershell retornaria a saída para a Entidade de chamada. Considere abaixo o exemplo,

Get-Process -Name excel

Isso simplesmente retornaria o processo em execução que corresponde ao nome excel, para a entidade chamadora. Nesse caso, o host do PowerShell. Imprime algo como,

Handles  NPM(K)    PM(K)      WS(K) VM(M)   CPU(s)     Id  SI ProcessName                                                                                                                     
-------  ------    -----      ----- -----   ------     --  -- -----------                                                                                                                     
   1037      54    67632      62544   617     5.23   4544   1 EXCEL

Agora, se você atribuir a saída a uma variável, ela simplesmente não imprimirá nada. E, claro, a variável mantém a saída. (Seja uma string, Object - Qualquer tipo para esse assunto)

$allExcel = Get-Process -Name excel

Então, digamos que você tenha um cenário em que deseja atribuir uma variável por um nome dinâmico, você pode usar o parâmetro -OutVariable

Get-Process -Name excel -OutVariable AllRunningExcel

Observe que o ‘$’ está faltando aqui. Uma grande diferença entre essas duas atribuições é que ela também imprime a saída além de atribuí-la à variável AllRunningExcel. Você também pode optar por atribuí-lo a outra variável.

$VarOne = Get-Process -Name excel -OutVariable VarTwo

Embora o cenário acima seja muito raro, ambas as variáveis ​​$VarOne e $VarTwo terão o mesmo valor.

Agora considere isso,

Get-Process -Name EXCEL -OutVariable MSOFFICE
Get-Process -Name WINWORD -OutVariable +MSOFFICE

A primeira instrução simplesmente obteria o processo do Excel e o atribuiria à variável MSOFFICE e, em seguida, obteria os processos do MS Word em execução e “Anexá-lo” ao valor existente de MSOFFICE. Ficaria algo assim,

Handles  NPM(K)    PM(K)      WS(K) VM(M)   CPU(s)     Id  SI ProcessName                                                                                                                     
-------  ------    -----      ----- -----   ------     --  -- -----------                                                                                                                     
   1047      54    67720      64448   618     5.70   4544   1 EXCEL                                                                                                                           
   1172      70    50052      81780   584     1.83  14968   1 WINWORD
