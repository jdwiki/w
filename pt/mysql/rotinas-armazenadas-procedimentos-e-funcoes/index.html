Parâmetros #

Parâmetro Detalhes
DEVOLUÇÕES Especifica o tipo de dados que pode ser retornado de uma função.
RETORNO A variável ou valor real seguindo a sintaxe RETURN é o que é retornado para onde a função foi chamada.

Uma rotina armazenada é um procedimento ou uma função.

Um procedimento é invocado usando uma instrução CALL e só pode retornar valores usando variáveis ​​de saída.

Uma função pode ser chamada de dentro de uma instrução como qualquer outra função e pode retornar um valor escalar.

Procedimento armazenado com parâmetros IN, OUT, INOUT #

DELIMITER $$
 
DROP PROCEDURE IF EXISTS sp_nested_loop$$
CREATE PROCEDURE sp_nested_loop(IN i INT, IN j INT, OUT x INT, OUT y INT, INOUT z INT)
BEGIN
    DECLARE a INTEGER DEFAULT 0;
    DECLARE b INTEGER DEFAULT 0;
    DECLARE c INTEGER DEFAULT 0;
    WHILE a < i DO
        WHILE b < j DO
            SET c = c + 1;
            SET b = b + 1;
        END WHILE;    
        SET a = a + 1;
        SET b = 0;        
    END WHILE;
    SET x = a, y = c;
    SET z = x + y + z;
END $$
DELIMITER ;

Invoca (CALL) o procedimento armazenado:

SET @z = 30;
call sp_nested_loop(10, 20, @x, @y, @z);
SELECT @x, @y, @z;

Resultado:

+------+------+------+
|  @x  |  @y  |  @z  |
+------+------+------+
|  10  |  200 |  240 |
+------+------+------+

Um parâmetro IN passa um valor para um procedimento. O procedimento pode modificar o valor, mas a modificação não é visível para o chamador quando o procedimento retorna.

Um parâmetro OUT passa um valor do procedimento de volta para o chamador. Seu valor inicial é NULL dentro do procedimento e seu valor é visível para o chamador quando o procedimento retorna.

Um parâmetro INOUT é inicializado pelo chamador, pode ser modificado pelo procedimento e qualquer alteração feita pelo procedimento fica visível para o chamador quando o procedimento retornar.

Ref: http://dev.mysql.com/doc/refman/5.7/en/create-procedure.html

Crie uma função #

A seguinte função de exemplo (trivial) simplesmente retorna o valor constante INT 12.

DELIMITER ||
CREATE FUNCTION functionname()
RETURNS INT
BEGIN
    RETURN 12;
END;
||
DELIMITER ;

A primeira linha define para que o caractere delimitador (DELIMITER ||) deve ser alterado, isso é necessário para ser definido antes de uma função ser criada, caso contrário, se for deixado em seu padrão ;, então o primeiro ; que é encontrado no corpo da função será tomado como o final da instrução CREATE, que normalmente não é o desejado.

Após a execução de CREATE FUNCTION, você deve definir o delimitador de volta para o padrão ; como é visto após o código da função no exemplo acima (DELIMITER ;).

A execução desta função é a seguinte:

SELECT functionname();
+----------------+
| functionname() |
+----------------+
|             12 |
+----------------+

Um exemplo um pouco mais complexo (mas ainda trivial) pega um parâmetro e adiciona uma constante a ele:

DELIMITER $$
CREATE FUNCTION add_2 ( my_arg INT )
  RETURNS INT
BEGIN
  RETURN (my_arg + 2);
END;
$$
DELIMITER ;

SELECT add_2(12);
+-----------+
| add_2(12) |
+-----------+
|        14 |
+-----------+

Observe o uso de um argumento diferente para a diretiva DELIMITER. Você pode realmente usar qualquer seqüência de caracteres que não apareça no corpo da instrução CREATE, mas a prática usual é usar um caractere não alfanumérico duplicado como \\, || ou $$.

É uma boa prática sempre alterar o parâmetro antes e depois de uma função, procedimento ou criação ou atualização de gatilho, pois algumas GUIs não exigem que o delimitador seja alterado, enquanto a execução de consultas por meio da linha de comando sempre exige que o delimitador seja definido.

Crie um procedimento com uma preparação construída #

DROP PROCEDURE if exists displayNext100WithName;
DELIMITER $$
CREATE PROCEDURE displayNext100WithName
(    nStart int,
    tblName varchar(100)
)
BEGIN
    DECLARE thesql varchar(500); -- holds the constructed sql string to execute

    -- expands the sizing of the output buffer to accomodate the output (Max value is at least 4GB)
    SET session group_concat_max_len = 4096; -- prevents group_concat from barfing with error 1160 or whatever it is

    SET @thesql=CONCAT("select group_concat(qid order by qid SEPARATOR '%3B') as nums ","from (    select qid from ");
    SET @thesql=CONCAT(@thesql,tblName," where qid>? order by qid limit 100 )xDerived");
    PREPARE stmt1 FROM @thesql; -- create a statement object from the construct sql string to execute
    SET @p1 = nStart; -- transfers parameter passed into a User Variable compatible with the below EXECUTE
    EXECUTE stmt1 USING @p1;
     
    DEALLOCATE PREPARE stmt1; -- deallocate the statement object when finished
END$$
DELIMITER ;

A criação do procedimento armazenado mostra o encapsulamento com um DELIMITER necessário em muitas ferramentas do cliente.

Exemplo de chamada:

call displayNext100WithName(1,"questions_mysql");

Cursores #

Os cursores permitem iterar os resultados da consulta um por linha. DECLARAR O comando é usado para iniciar o cursor e associá-lo a uma consulta SQL específica:

DECLARE student CURSOR FOR SELECT name FROM studend;

Digamos que vendemos produtos de alguns tipos. Queremos contar quantos produtos de cada tipo existem.

Nossos dados:

CREATE TABLE product
(
  id   INT(10) UNSIGNED NOT NULL AUTO_INCREMENT PRIMARY KEY,
  type VARCHAR(50)      NOT NULL,
  name VARCHAR(255)     NOT NULL

);
CREATE TABLE product_type
(
  name VARCHAR(50) NOT NULL PRIMARY KEY
);
CREATE TABLE product_type_count
(
  type  VARCHAR(50)      NOT NULL PRIMARY KEY,
  count INT(10) UNSIGNED NOT NULL DEFAULT 0
);

INSERT INTO product_type (name) VALUES
  ('dress'),
  ('food');

INSERT INTO product (type, name) VALUES
  ('dress', 'T-shirt'),
  ('dress', 'Trousers'),
  ('food', 'Apple'),
  ('food', 'Tomatoes'),
  ('food', 'Meat');

Podemos atingir o objetivo usando o procedimento armazenado com o uso do cursor:

DELIMITER //
DROP PROCEDURE IF EXISTS product_count;
CREATE PROCEDURE product_count()
  BEGIN
    DECLARE p_type VARCHAR(255);
    DECLARE p_count INT(10) UNSIGNED;
    DECLARE done INT DEFAULT 0;
    DECLARE product CURSOR FOR
      SELECT
        type,
        COUNT(*)
      FROM product
      GROUP BY type;
    DECLARE CONTINUE HANDLER FOR SQLSTATE '02000' SET done = 1;

    TRUNCATE product_type;

    OPEN product;
    
    REPEAT
      FETCH product
      INTO p_type, p_count;
      IF NOT done
      THEN
        INSERT INTO product_type_count
        SET
          type  = p_type,
          count = p_count;
      END IF;
    UNTIL done
    END REPEAT;
    
    CLOSE product;
  END //
DELIMITER ;

Quando você pode chamar o procedimento com:

CALL product_count();

O resultado estaria na tabela product_type_count:

type   | count
----------------
dress  |   2
food   |   3

Embora esse seja um bom exemplo de um CURSOR, observe como todo o corpo do procedimento pode ser substituído por apenas

INSERT INTO product_type_count
        (type, count)
    SELECT type, COUNT(*)
        FROM product
        GROUP BY type;

Isso vai correr muito mais rápido.

Vários conjuntos de resultados #

Ao contrário de uma instrução SELECT, um Stored Procedure retorna vários conjuntos de resultados. O requer um código diferente para ser usado para reunir os resultados de um CALL em Perl, PHP, etc.

Crie uma função #

DELIMITER $$
CREATE
    DEFINER=`db_username`@`hostname_or_IP`
    FUNCTION `function_name`(optional_param data_type(length_if_applicable))
    RETURNS data_type
BEGIN
    /*
    SQL Statements goes here
    */
END$$
DELIMITER ;

O RETURNS data_type é qualquer tipo de dados MySQL.
