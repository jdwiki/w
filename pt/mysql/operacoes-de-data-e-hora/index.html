Aritmética de data #

NOW() + INTERVAL 1 DAY  -- This time tomorrow

CURDATE() - INTERVAL 4 DAY -- Midnight 4 mornings ago

Mostre as perguntas do mysql armazenadas que foram perguntado 3 a 10 horas atrás (180 a 600 minutos atrás):

SELECT qId,askDate,minuteDiff  
FROM 
(   SELECT qId,askDate,
    TIMESTAMPDIFF(MINUTE,askDate,now()) as minuteDiff 
    FROM questions_mysql 
) xDerived 
WHERE minuteDiff BETWEEN 180 AND 600 
ORDER BY qId DESC 
LIMIT 50;

| 38546707 | 2016-07-23 21:48:46 |        200 |
| 38546687 | 2016-07-23 21:45:26 |        203 |

Páginas de manual do MySQL para TIMESTAMPDIFF().

Cuidado Não tente usar expressões como CURDATE() + 1 para aritmética de datas no MySQL. Eles não retornam o que você espera, especialmente se você estiver acostumado com o produto de banco de dados Oracle. Use CURDATE() + INTERVAL 1 DAY.

SYSDATE(), NOW(), CURDATE() #

  SELECT SYSDATE();

Esta função retorna a data e hora atuais como um valor no formato 'AAAA-MM-DD HH:MM:SS' ou AAAAMMDDHHMMSS, dependendo se a função é usada em um contexto de string ou numérico. Ele retorna a data e a hora no fuso horário atual.

  SELECT NOW();

Esta função é sinônimo de SYSDATE().

  SELECT CURDATE();

Esta função retorna a data atual, sem qualquer hora, como um valor no formato 'AAAA-MM-DD' ou AAAAMMDD, dependendo se a função é usada em um contexto de string ou numérico. Retorna a data no fuso horário atual.

Agora() #

Select Now();

Mostra a data e hora do servidor atual.

Update `footable` set mydatefield = Now();

Isso atualizará o campo mydatefield com a data e hora atuais do servidor no fuso horário configurado do servidor, por exemplo

'2016-07-21 12:00:00'

Testando em um intervalo de datas #

Embora seja muito tentador usar BETWEEN … AND … para um intervalo de datas, é problemático. Em vez disso, esse padrão evita a maioria dos problemas:

WHERE x >= '2016-02-25'
  AND x  < '2016-02-25' + INTERVAL 5 DAY

Vantagens:

  • BETWEEN é ‘inclusivo’, incluindo a data final ou segundo.
  • 23:59:59 é desajeitado e errado se você tiver resolução de microssegundos em um DATETIME.
  • Este padrão evita lidar com anos bissextos e outros cálculos de dados.
  • Funciona se x for DATE, DATETIME ou TIMESTAMP.

Extrair a data da expressão DateTime ou DateTime fornecida #

SELECT DATE('2003-12-31 01:02:03');

A saída será:

2003-12-31

Usando um índice para uma pesquisa de data e hora #

Muitas tabelas de banco de dados do mundo real têm muitas linhas com valores de coluna DATETIME OU TIMESTAMP abrangendo muito tempo, incluindo anos ou mesmo décadas. Muitas vezes é necessário usar uma cláusula WHERE para recuperar algum subconjunto desse intervalo de tempo. Por exemplo, podemos querer recuperar linhas para a data 1-September-2016 de uma tabela.

Uma maneira ineficiente de fazer isso é esta:

 WHERE DATE(x) = '2016-09-01'   /* slow! */

É ineficiente porque aplica uma função – DATE() – aos valores de uma coluna. Isso significa que o MySQL deve examinar cada valor de x, e um índice não pode ser usado.

A melhor maneira de fazer a operação é esta

 WHERE x >= '2016-09-01'
   AND x <  '2016-09-01' + INTERVAL 1 DAY

Isso seleciona um intervalo de valores de x em qualquer lugar do dia em questão, até não incluindo (daí <) meia-noite do dia seguinte.

Se a tabela tiver um índice na coluna x, o servidor de banco de dados poderá executar uma varredura de intervalo no índice. Isso significa que ele pode encontrar rapidamente o primeiro valor relevante de x e, em seguida, varrer o índice sequencialmente até encontrar o último valor relevante. Uma varredura de intervalo de índice é muito mais eficiente do que a varredura de tabela completa exigida por DATE(x) = '2016-09-01.

Não fique tentado a usar isso, mesmo que pareça mais eficiente.

  WHERE x BETWEEN '2016-09-01' AND '2016-09-01' + INTERVAL 1 DAY  /*  wrong! */

Ele tem a mesma eficiência que a varredura de intervalo, mas selecionará linhas com valores de x caindo exatamente à meia-noite de 2 de setembro de 2016, o que não é o que você deseja.
