Qual CHARACTER SET e COLLATION? #

Existem dezenas de conjuntos de caracteres com centenas de agrupamentos. (Um dado agrupamento pertence a apenas um conjunto de caracteres.) Veja a saída de SHOW COLLATION;.

Normalmente existem apenas 4 CHARACTER SETs que importam:

ascii -- basic 7-bit codes.
latin1 -- ascii, plus most characters needed for Western European languages.
utf8 -- the 1-, 2-, and 3-byte subset of utf8.  This excludes Emoji and some of Chinese.
utf8mb4 -- the full set of UTF8 characters, covering all current languages.

Todos incluem caracteres em inglês, codificados de forma idêntica. utf8 é um subconjunto de utf8mb4.

Melhor prática…

  • Use utf8mb4 para qualquer coluna TEXT ou VARCHAR que possa conter uma variedade de idiomas.
  • Use ascii (latin1 é ok) para strings hexadecimais (UUID, MD5, etc) e códigos simples (country_code, postal_code, etc).

utf8mb4 não existia até a versão 5.5.3, então utf8 era o melhor disponível antes disso.

Fora do MySQL, “UTF8” significa as mesmas coisas que o utf8mb4 do MySQL, não o utf8 do MySQL.

Os agrupamentos começam com o nome do conjunto de caracteres e geralmente terminam com _ci para “não diferencia maiúsculas de minúsculas e acentos” ou _bin para “simplesmente comparar os bits.

O agrupamento ‘mais recente’ utf8mb4 é utf8mb4_unicode_520_ci, baseado no Unicode 5.20. Se você estiver trabalhando com um único idioma, talvez queira, digamos, utf8mb4_polish_ci, que reorganizará as letras ligeiramente, com base nas convenções polonesas.

Declaração #

CREATE TABLE foo ( ...
    name CHARACTER SET utf8mb4
    ... );

Conexão #

Vital para usar conjuntos de caracteres é dizer ao servidor MySQL qual é a codificação dos bytes do cliente. Aqui está uma maneira:

SET NAMES utf8mb4;

Cada linguagem (PHP, Python, Java, …) tem sua própria maneira, geralmente preferível a SET NAMES.

Por exemplo: SET NAMES utf8mb4, junto com uma coluna declarada CHARACTER SET latin1 – isto irá converter de latin1 para utf8mb4 quando INSERTing e converter novamente quando SELECTing.

Configurando conjuntos de caracteres em tabelas e campos #

Você pode definir um conjunto de caracteres tanto por tabela quanto por campo individual usando as instruções CHARACTER SET e CHARSET:

CREATE TABLE Address (
    `AddressID`   INTEGER NOT NULL PRIMARY KEY,
    `Street`      VARCHAR(80) CHARACTER SET ASCII,
    `City`        VARCHAR(80),
    `Country`     VARCHAR(80) DEFAULT "United States",
    `Active`      BOOLEAN DEFAULT 1,
) Engine=InnoDB default charset=UTF8;

City e Country usarão UTF8, pois definimos isso como o conjunto de caracteres padrão para a tabela. Street, por outro lado, usará ASCII, como dissemos especificamente para fazer isso.

Definir o conjunto de caracteres correto depende muito do seu conjunto de dados, mas também pode melhorar muito a portabilidade entre os sistemas que trabalham com seus dados.
