Recupere e redefina a senha de root padrão para MySQL 5.7+

  • O que acontece quando a inicialização inicial do servidor
  • Como alterar a senha do root usando a senha padrão
  • redefina a senha do root quando " /var/run/mysqld’ for UNIX socket file não existir"

Nesta página

  • O que acontece quando a inicialização inicial do servidor
  • Como alterar a senha do root usando a senha padrão
  • redefina a senha do root quando " /var/run/mysqld’ for UNIX socket file não existir"

Após o MySQL 5.7, quando instalamos o MySQL às vezes não precisamos criar uma conta de root ou fornecer uma senha de root. Por padrão, quando iniciamos o servidor, a senha padrão é armazenada no arquivo mysqld.log. Precisamos fazer login no sistema usando essa senha e precisamos alterá-la.

Recuperar e redefinir a senha de root padrão usando este método é aplicável apenas para MySQL 5.7+

O que acontece quando a inicialização inicial do servidor #

Dado que o diretório de dados do servidor está vazio:

  • O servidor é inicializado.
  • O certificado SSL e os arquivos de chave são gerados no diretório de dados.
  • O plugin validate_password está instalado e ativado.
  • A conta de superusuário ‘root’@’localhost’ é criada. A senha para o superusuário é definida e armazenada no arquivo de log de erros.

Como alterar a senha do root usando a senha padrão #

Para revelar a senha “root” padrão:

shell> sudo grep 'temporary password' /var/log/mysqld.log

Altere a senha de root o mais rápido possível fazendo login com a senha temporária gerada e defina uma senha personalizada para a conta de superusuário:

shell> mysql -uroot -p 

mysql> ALTER USER 'root'@'localhost' IDENTIFIED BY 'MyNewPass5!';

Observação: o plugin validate_password do MySQL é instalado por padrão. Isso exigirá que as senhas contenham pelo menos uma letra maiúscula, uma letra minúscula, um dígito e um caractere especial, e que o comprimento total da senha seja de pelo menos 8 caracteres.

redefina a senha do root quando " /var/run/mysqld’ for UNIX socket file não existir" #

se eu esquecer a senha, eu recebo um erro.

$ mysql -u root -p

Digite a senha:

ERRO 1045 (28000): Acesso negado para usuário ‘root’@’localhost’ (usando senha: YES)

Tentei resolver o problema conhecendo primeiro o status:

$ systemctl status mysql.service

mysql.service - MySQL Community Server Carregado: carregado (/lib/systemd/system/mysql.service; ativado; predefinição do fornecedor: en Ativo: ativo (em execução) desde Qui 2017-06-08 14:31:33 IST; 38s atrás

Então eu usei o código mysqld_safe --skip-grant-tables & mas recebo o erro:

mysqld_safe O diretório ‘/var/run/mysqld’ para o arquivo de soquete UNIX não existe.

$ systemctl stop  mysql.service
$ ps -eaf|grep mysql
$ mysqld_safe --skip-grant-tables &

resolvi:

$ mkdir -p /var/run/mysqld
$ chown mysql:mysql /var/run/mysqld

Agora eu uso o mesmo código mysqld_safe --skip-grant-tables & e obtenho

mysqld_safe Iniciando o daemon mysqld com bancos de dados de /var/lib/mysql

Se eu usar $ mysql -u root obterei:

Versão do servidor: 5.7.18-0ubuntu0.16.04.1 (Gratuito)

Copyright (c) 2000, 2017, Oracle e/ou suas afiliadas. Todos os direitos reservados.

Oracle é uma marca registrada da Oracle Corporation e/ou de seus afiliados. Outros nomes podem ser marcas comerciais de seus respectivos os Proprietários.

Digite ‘ajuda;’ ou ‘\h’ para ajuda. Digite ‘\c’ para limpar a instrução de entrada atual.

mysql>

Agora é hora de mudar a senha:

mysql> use mysql
mysql> describe user;

Lendo as informações da tabela para completar os nomes das tabelas e colunas Você pode desativar esse recurso para obter uma inicialização mais rápida com -A

Banco de dados alterado

mysql> FLUSH PRIVILEGES;
mysql> SET PASSWORD FOR root@'localhost' = PASSWORD('newpwd');

ou Se você tem uma conta root do mysql que pode se conectar de qualquer lugar, você também deve fazer:

UPDATE mysql.user SET Password=PASSWORD('newpwd') WHERE User='root';

Método alternativo:

   USE mysql
   UPDATE user SET Password = PASSWORD('newpwd')
   WHERE Host = 'localhost' AND User = 'root';

E se você tiver uma conta root que possa acessar de qualquer lugar:

 USE mysql
 UPDATE user SET Password = PASSWORD('newpwd')
 WHERE Host = '%' AND User = 'root';`enter code here

agora precisa sair do mysql e parar/iniciar

FLUSH PRIVILEGES;
sudo /etc/init.d/mysql stop
sudo /etc/init.d/mysql start

agora novamente ` mysql -u root -p’ e use a nova senha para obter

mysql>
