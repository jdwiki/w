Uma expressão regular é uma maneira poderosa de especificar um padrão para uma pesquisa complexa.

REGEXP / RLIKE #

O operador REGEXP (ou seu sinônimo, RLIKE) permite a correspondência de padrões com base em expressões regulares.

Considere a seguinte tabela employee:

+-------------+-------------+-------------+--------------+----------+
| EMPLOYEE_ID | FIRST_NAME  | LAST_NAME   | PHONE_NUMBER | SALARY   |
+-------------+-------------+-------------+--------------+----------+
|         100 | Steven      | King        | 515.123.4567 | 24000.00 |
|         101 | Neena       | Kochhar     | 515.123.4568 | 17000.00 |
|         102 | Lex         | De Haan     | 515.123.4569 | 17000.00 |
|         103 | Alexander   | Hunold      | 590.423.4567 |  9000.00 |
|         104 | Bruce       | Ernst       | 590.423.4568 |  6000.00 |
|         105 | David       | Austin      | 590.423.4569 |  4800.00 |
|         106 | Valli       | Pataballa   | 590.423.4560 |  4800.00 |
|         107 | Diana       | Lorentz     | 590.423.5567 |  4200.00 |
|         108 | Nancy       | Greenberg   | 515.124.4569 | 12000.00 |
|         109 | Daniel      | Faviet      | 515.124.4169 |  9000.00 |
|         110 | John        | Chen        | 515.124.4269 |  8200.00 |
+-------------+-------------+-------------+--------------+----------+

Padrão ^ #

Selecione todos os funcionários cujo FIRST_NAME comece com N.

Consulta

SELECT * FROM employees WHERE FIRST_NAME REGEXP '^N'
-- Pattern start with----------------------------^

Padrão $** #

Selecione todos os funcionários cujo PHONE_NUMBER termine com 4569.

Consulta

SELECT * FROM employees WHERE PHONE_NUMBER REGEXP '4569$'
-- Pattern end with----------------------------------^

NÃO REGEXP #

Selecione todos os funcionários cujo FIRST_NAME não comece com N.

Consulta

SELECT * FROM employees WHERE FIRST_NAME NOT REGEXP '^N'
-- Pattern does not start with---------------^

Contém Regex #

Selecione todos os funcionários cujo LAST_NAME contém in e cujo FIRST_NAME contém a.

Consulta

SELECT * FROM employees WHERE FIRST_NAME REGEXP 'a' AND LAST_NAME REGEXP 'in'
-- No ^ or $, pattern can be anywhere -------------------------------------^

Qualquer caractere entre [ ] #

Selecione todos os funcionários cujo FIRST_NAME comece com A ou B ou C.

Consulta

SELECT * FROM employees WHERE FIRST_NAME REGEXP '^[ABC]'
-------------------------------------------------^^---^

Padrão ou | #

Selecione todos os funcionários cujo FIRST_NAME comece com A ou B ou C e termine com r, e ou i.

Consulta

SELECT * FROM employees WHERE FIRST_NAME REGEXP '^[ABC]|[rei]$'
-- ----------------------------------------------^^---^^^---^^

Contando correspondências de expressões regulares #

Considere a seguinte consulta:

SELECT FIRST_NAME, FIRST_NAME REGEXP '^N' as matching FROM employees

FIRST_NAME REGEXP '^N' é 1 ou 0 dependendo do fato de que FIRST_NAME corresponde a ^N.

Para visualizar melhor:

SELECT 
FIRST_NAME, 
IF(FIRST_NAME REGEXP '^N', 'matches ^N', 'does not match ^N') as matching 
FROM employees

Por fim, conte o número total de linhas correspondentes e não correspondentes com:

SELECT
IF(FIRST_NAME REGEXP '^N', 'matches ^N', 'does not match ^N') as matching,
COUNT(*) 
FROM employees 
GROUP BY matching
