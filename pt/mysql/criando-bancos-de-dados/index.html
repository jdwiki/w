Sintaxe #

  • CRIAR {BANCO DE DADOS | SCHEMA} [IF NOT EXISTS] db_name [create_specification] /// Para criar banco de dados
  • SOLTAR {BANCO DE DADOS | SCHEMA} [IF EXISTS] db_name /// Para descartar o banco de dados

Parâmetros #

Parâmetro Detalhes
CRIAR BANCO DE DADOS Cria um banco de dados com o nome fornecido
CRIAR ESQUEMA Este é um sinônimo para CREATE DATABASE
SE NÃO EXISTE Usado para evitar erro de execução, se o banco de dados especificado já existir
criar_especificação As opções create_specification especificam as características do banco de dados como CHARACTER SET e COLLATE(colação de banco de dados)

Criar banco de dados, usuários e concessões #

Crie um BANCO DE DADOS. Observe que a palavra abreviada SCHEMA pode ser usada como sinônimo.

CREATE DATABASE Baseball; -- creates a database named Baseball

Se o banco de dados já existir, o erro 1007 será retornado. Para contornar esse erro, tente:

CREATE DATABASE IF NOT EXISTS Baseball;

De forma similar,

DROP DATABASE IF EXISTS Baseball; -- Drops a database if it exists, avoids Error 1008
DROP DATABASE xyz; -- If xyz does not exist, ERROR 1008 will occur

Devido às possibilidades de erro acima, as instruções DDL são frequentemente usadas com IF EXISTS.

Pode-se criar um banco de dados com um CHARACTER SET e agrupamento padrão. Por exemplo:

CREATE DATABASE Baseball CHARACTER SET utf8 COLLATE utf8_general_ci;

SHOW CREATE DATABASE Baseball;
+----------+-------------------------------------------------------------------+
| Database | Create Database                                                   |
+----------+-------------------------------------------------------------------+
| Baseball | CREATE DATABASE `Baseball` /*!40100 DEFAULT CHARACTER SET utf8 */ |
+----------+-------------------------------------------------------------------+

Veja seus bancos de dados atuais:

SHOW DATABASES;
+---------------------+
| Database            |
+---------------------+
| information_schema  |
| ajax_stuff          |
| Baseball            |
+---------------------+

Defina o banco de dados atualmente ativo e veja algumas informações:

USE Baseball; -- set it as the current database
SELECT @@character_set_database as cset,@@collation_database as col;
+------+-----------------+
| cset | col             |
+------+-----------------+
| utf8 | utf8_general_ci |
+------+-----------------+

O acima mostra o CHARACTER SET e Collation padrão para o banco de dados.

Crie um usuário:

CREATE USER 'John123'@'%' IDENTIFIED BY 'OpenSesame';

O acima cria um usuário John123, capaz de se conectar com qualquer nome de host devido ao curinga %. A senha para o usuário é definida como ‘OpenSesame’, que é hash.

E crie outro:

CREATE USER 'John456'@'%' IDENTIFIED BY 'somePassword';

Mostre que os usuários foram criados examinando o banco de dados especial mysql:

SELECT user,host,password from mysql.user where user in ('John123','John456');
+---------+------+-------------------------------------------+
| user    | host | password                                  |
+---------+------+-------------------------------------------+
| John123 | %    | *E6531C342ED87 ....................       |
| John456 | %    | *B04E11FAAAE9A ....................       |
+---------+------+-------------------------------------------+

Observe que, neste ponto, os usuários foram criados, mas sem nenhuma permissão para usar o banco de dados Baseball.

Trabalhe com permissões para usuários e bancos de dados. Conceda direitos ao usuário John123 para ter privilégios totais no banco de dados Baseball e apenas SELECT direitos para o outro usuário:

GRANT ALL ON Baseball.* TO 'John123'@'%';
GRANT SELECT ON Baseball.* TO 'John456'@'%';

Verifique o acima:

SHOW GRANTS FOR 'John123'@'%';
+--------------------------------------------------------------------------------------------------------+
| Grants for John123@%                                                                                   |
+--------------------------------------------------------------------------------------------------------+
| GRANT USAGE ON *.* TO 'John123'@'%' IDENTIFIED BY PASSWORD '*E6531C342ED87 ....................        |
| GRANT ALL PRIVILEGES ON `baseball`.* TO 'John123'@'%'                                                  |
+--------------------------------------------------------------------------------------------------------+

SHOW GRANTS FOR 'John456'@'%';
+--------------------------------------------------------------------------------------------------------+
| Grants for John456@%                                                                                   |
+--------------------------------------------------------------------------------------------------------+
| GRANT USAGE ON *.* TO 'John456'@'%' IDENTIFIED BY PASSWORD '*B04E11FAAAE9A ....................        |
| GRANT SELECT ON `baseball`.* TO 'John456'@'%'                                                          |
+--------------------------------------------------------------------------------------------------------+

Observe que o GRANT USAGE que você sempre verá significa simplesmente que o usuário pode fazer o login. Isso é tudo o que isso significa.

MeuBanco de Dados #

Você deve criar seu próprio banco de dados e não usar gravação em nenhum dos bancos de dados existentes. Esta é provavelmente uma das primeiras coisas a fazer depois de se conectar pela primeira vez.

CREATE DATABASE my_db;
USE my_db;
CREATE TABLE some_table;
INSERT INTO some_table ...;

Você pode referenciar sua tabela qualificando com o nome do banco de dados: my_db.some_table.

Bancos de dados do sistema #

Os seguintes bancos de dados existem para uso do MySQL. Você pode lê-los (SELECT), mas não deve escrever (INSERT/UPDATE/DELETE) as tabelas neles. (Há poucas exceções.)

  • mysql – repositório para informações GRANT e algumas outras coisas.
  • information_schema – As tabelas aqui são ‘virtuais’ no sentido de que elas são realmente manifestadas por estruturas na memória. Seu conteúdo inclui o esquema para todas as tabelas.
  • performance_schema – ?? [aceite e edite]
  • outros?? (para MariaDB, Galera, TokuDB, etc)

Criando e Selecionando um Banco de Dados #

Se o administrador criar seu banco de dados para você ao configurar suas permissões, você poderá começar a usá-lo. Caso contrário, você precisa criá-lo você mesmo:

mysql> CREATE DATABASE menagerie;

No Unix, os nomes dos bancos de dados diferenciam maiúsculas de minúsculas (ao contrário das palavras-chave SQL), portanto, você deve sempre se referir ao seu banco de dados como menagerie, não como Menagerie, MENAGERIE ou alguma outra variante. Isso também é verdade para nomes de tabelas. (No Windows, essa restrição não se aplica, embora você deva fazer referência a bancos de dados e tabelas usando a mesma letra em uma determinada consulta. No entanto, por vários motivos, a prática recomendada é sempre usar a mesma letra usada quando o banco de dados foi criado.)

A criação de um banco de dados não o seleciona para uso; você deve fazer isso explicitamente. Para tornar o menagerie o banco de dados atual, use esta instrução:

mysql> USE menagerie
Database changed

Seu banco de dados precisa ser criado apenas uma vez, mas você deve selecioná-lo para uso sempre que iniciar uma sessão mysql. Você pode fazer isso emitindo uma instrução USE conforme mostrado no exemplo. Alternativamente, você pode selecionar o banco de dados na linha de comando ao invocar o mysql. Basta especificar seu nome após quaisquer parâmetros de conexão que você possa precisar fornecer. Por exemplo:

shell> mysql -h host -u user -p menagerie
Enter password: ********
