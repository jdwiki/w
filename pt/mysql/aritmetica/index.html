O MySQL, na maioria das máquinas, usa 64 bits IEEE 754 aritmética de ponto flutuante para seus cálculos.

Em contextos inteiros usa aritmética inteira.

  • RAND() não é um gerador de números aleatórios perfeito. É usado principalmente para gerar rapidamente números pseudo-aleatórios

Operadores aritméticos #

O MySQL fornece os seguintes operadores aritméticos

Operador Nome Exemplo
+ Adição SELECT 3+5; -> 8
SELECT 3,5+2,5; -> 6,0
SELECT 3,5+2; -> 5,5
- Subtração SELECT 3-5; -> -2
* Multiplicação SELECT 3 * 5; -> 15
/ Divisão SELECT 20 / 4; -> 5
SELECT 355 / 113; -> 3.1416
SELECT 10.0 / 0; -> NULL
DIV Divisão de inteiros SELECT 5 DIV 2; -> 2
% ou MOD Módulo SELECT 7 % 3; -> 1
SELECT 15 MOD 4 -> 3
SELECT 15 MOD -4 -> 3
SELECT -15 MOD 4 -> -3
SELECT -15 MOD -4 -> -3
SELECT 3 MOD 2.5 -> 0,5

GRANDE #

Se os números em sua aritmética são todos inteiros, o MySQL usa o tipo de dados inteiro BIGINT (64 bits assinado) para fazer seu trabalho. Por exemplo:

selecione (1024 * 1024 * 1024 * 1024 * 1024 * 1024) + 1 -> 1.152.921.504.606.846.977

e

select (1024 * 1024 * 1024 * 1024 * 1024 * 1024 * 1024 -> BIGINT erro fora do intervalo

EM DOBRO #

Se algum número em sua aritmética for fracionário, o MySQL usa aritmética de ponto flutuante IEEE 754 de 64 bits. Você deve ter cuidado ao usar aritmética de ponto flutuante, porque muitos números de ponto flutuante são, inerentemente, aproximações em vez de valores exatos.

Constantes matemáticas #

Pi #

O seguinte retorna o valor de PI formatado com 6 casas decimais. O valor real é bom para DOUBLE;

SELECT PI();    -> 3.141593

Trigonometria (SIN, COS) #

Os ângulos estão em radianos, não em graus. Todos os cálculos são feitos em IEEE 754 64-bit float point. Todos os cálculos de ponto flutuante estão sujeitos a pequenos erros, conhecidos como machine ε (epsilon) erros, então evite tentar compará-los para igualdade. Não há como evitar esses erros ao usar o ponto flutuante; eles são incorporados à tecnologia.

Se você usar valores DECIMAL em cálculos trigonométricos, eles serão convertidos implicitamente para ponto flutuante e, em seguida, de volta para decimal.

Seno #

Retorna o seno de um número X expresso em radianos

SELECT SIN(PI()); -> 1.2246063538224e-16

Cosseno #

Retorna o cosseno de X quando X é dado em radianos

SELECT COS(PI()); -> -1

Tangente #

Retorna a tangente de um número X expresso em radianos. Observe que o resultado é muito próximo de zero, mas não exatamente zero. Este é um exemplo de máquina ε.

SELECT TAN(PI());   -> -1.2246063538224e-16

Arco Cosseno (cosseno inverso) #

Retorna o arco cosseno de X se X estiver no intervalo -1 a 1

SELECT ACOS(1);    -> 0
SELECT ACOS(1.01); -> NULL

Arc Sine (seno inverso) #

Retorna o arco seno de X se X estiver no intervalo -1 a 1

SELECT ASIN(0.2); -> 0.20135792079033

Arc Tangent (tangente inversa) #

ATAN(x) retorna o arco tangente de um único número.

SELECT ATAN(2); -> 1.1071487177941

ATAN2(X, Y) retorna o arco tangente das duas variáveis ​​X e Y. É semelhante ao cálculo do arco tangente de Y / X. Mas é numericamente mais robusto: t funciona corretamente quando X está próximo de zero, e os sinais de ambos os argumentos são usados ​​para determinar o quadrante do resultado.

A melhor prática sugere escrever fórmulas para usar ATAN2() em vez de ATAN() sempre que possível.

 ATAN2(1,1);    -> 0.7853981633974483 (45 degrees)
 ATAN2(1,-1);   -> 2.356194490192345  (135 degrees)
 ATAN2(0, -1);  -> PI  (180 degrees)  don't try ATAN(-1 / 0)... it won't work

Cotangente #

Retorna a cotangente de X

SELECT COT(12); -> -1.5726734063977

Conversão #

SELECT RADIANS(90) -> 1.5707963267948966
SELECT SIN(RADIANS(90)) -> 1
SELECT DEGREES(1), DEGREES(PI()) -> 57.29577951308232, 180

Arredondamento (REDONDO, PISO, TETO) #

Arredonda um número decimal para um valor inteiro #

Para valores numéricos exatos (por exemplo, DECIMAL): Se a primeira casa decimal de um número for 5 ou superior, esta função arredondará um número para o próximo inteiro distante de zero. Se essa casa decimal for 4 ou inferior, esta função arredondará para o próximo valor inteiro mais próximo de zero.

SELECT ROUND(4.51) -> 5
SELECT ROUND(4.49) -> 4
SELECT ROUND(-4.51) -> -5

Para valores numéricos aproximados (por exemplo, DOUBLE): O resultado da função ROUND() depende da biblioteca C; em muitos sistemas, isso significa que ROUND() usa a regra arredondar para o par mais próximo:

SELECT ROUND(45e-1) -> 4  -- The nearest even value is 4
SELECT ROUND(55e-1) -> 6  -- The nearest even value is 6

Arredonde um número #

Para arredondar um número, use a função CEIL() ou CEILING()

SELECT CEIL(1.23)    -> 2
SELECT CEILING(4.83) -> 5

Arredonde um número #

Para arredondar um número, use a função FLOOR()

SELECT FLOOR(1.99) -> 1

PISO e TETO vão em direção/afastamento de -infinito:

SELECT FLOOR(-1.01), CEIL(-1.01) -> -2 and -1
SELECT FLOOR(-1.99), CEIL(-1.99) -> -2 and -1

Arredonda um número decimal para um número especificado de casas decimais. #

SELECT ROUND(1234.987, 2) -> 1234.99
SELECT ROUND(1234.987, -2) -> 1200

A discussão de para cima versus para baixo e “5” também se aplica.

Eleva um número a uma potência (POW) #

Para elevar um número x a uma potência y, use as funções POW() ou POWER()

SELECT POW(2,2); => 4
SELECT POW(4,2); => 16

Raiz quadrada (SQRT) #

Use a função SQRT(). Se o número for negativo, NULL será retornado

SELECT SQRT(16); -> 4
SELECT SQRT(-3); -> NULL

Números aleatórios (RAND) #

Gerar um número aleatório #

Para gerar um número de ponto flutuante pseudoaleatório entre 0 e 1, use a função RAND()

Suponha que você tenha a seguinte consulta

SELECT i, RAND() FROM t;

Isso retornará algo assim

eu RAND()
1 0,6191438870682
2 0,93845168309142
3 0,83482678498591

Número aleatório em um intervalo #

Para gerar um número aleatório no intervalo a <= n <= b, você pode usar a seguinte fórmula

FLOOR(a + RAND() * (b - a + 1))

Por exemplo, isso gerará um número aleatório entre 7 e 12

SELECT FLOOR(7 + (RAND() * 6));

Uma maneira simples de retornar aleatoriamente as linhas em uma tabela:

SELECT * FROM tbl ORDER BY RAND();

Estes são números pseudorandom.

O gerador de números pseudoaleatórios no MySQL não é criptograficamente seguro. Ou seja, se você usar o MySQL para gerar números aleatórios para serem usados ​​como segredos, um adversário determinado que saiba que você usou o MySQL poderá adivinhar seus segredos mais facilmente do que você imagina.

Valor Absoluto e Sinal (ABS, SINAL) #

Retorna o valor absoluto de um número

SELECT ABS(2);   -> 2
SELECT ABS(-46); -> 46

O sinal de um número o compara a 0.

Assinar Resultado Exemplo
-1 n < 0 SELECIONAR SINAL(42); -> 1
0 n = 0 SELECT SIGN(0); -> 0
1 n > 0 SELECIONAR SINAL(-3); -> -1 
SELECT SIGN(-423421); -> -1
