Começando #

Criando um banco de dados no MySQL

CREATE DATABASE mydb;

Valor de retorno:

Consulta OK, 1 linha afetada (0,05 seg)

Usando o banco de dados criado mydb

USE mydb;

Valor de retorno:

Banco de dados alterado

Criando uma tabela no MySQL

CREATE TABLE mytable
(
  id              int unsigned NOT NULL auto_increment,
  username        varchar(100) NOT NULL,
  email           varchar(100) NOT NULL,
  PRIMARY KEY     (id)
);

CREATE TABLE mytable criará uma nova tabela chamada mytable.

id int unsigned NOT NULL auto_increment cria a coluna id, este tipo de campo irá atribuir um ID numérico único para cada registro na tabela (o que significa que duas linhas não podem ter o mesmo id neste caso), MySQL atribuirá automaticamente um valor novo e exclusivo ao campo id do registro (começando com 1).

Valor de retorno:

Consulta OK, 0 linhas afetadas (0,10 seg)

Inserindo uma linha em uma tabela MySQL

INSERT INTO mytable ( username, email )
VALUES ( "myuser", "myuser@example.com" );

Valor de retorno de exemplo:

Consulta OK, 1 linha afetada (0,06 seg)

O varchar a.k.a strings também pode ser inserido usando aspas simples:

INSERT INTO mytable ( username, email )
VALUES ( 'username', 'username@example.com' );

Atualizando uma linha em uma tabela MySQL

UPDATE mytable SET username="myuser" WHERE id=8

Valor de retorno de exemplo:

Consulta OK, 1 linha afetada (0,06 seg)

O valor int pode ser inserido em uma consulta sem aspas. Strings e Datas devem estar entre aspas simples ' ou aspas duplas ".

Excluindo uma linha em uma tabela MySQL

DELETE FROM mytable WHERE id=8

Valor de retorno de exemplo:

Consulta OK, 1 linha afetada (0,06 seg)

Isso excluirá a linha com id 8.

Selecionando linhas com base nas condições no MySQL

SELECT * FROM mytable WHERE username = "myuser";

Valor de retorno:

+----+----------+---------------------+
| id | username | email               |
+----+----------+---------------------+
|  1 | myuser   | myuser@example.com  |
+----+----------+---------------------+

1 linha em conjunto (0,00 seg)

Mostrar lista de bancos de dados existentes

SHOW databases;

Valor de retorno:

+-------------------+
| Databases         |
+-------------------+
| information_schema|
| mydb              |
+-------------------+

2 linhas em conjunto (0,00 seg)

Você pode pensar em “information_schema” como um “banco de dados mestre” que fornece acesso aos metadados do banco de dados.

Mostrar tabelas em um banco de dados existente

SHOW tables;

Valor de retorno:

+----------------+
| Tables_in_mydb |
+----------------+
| mytable        |
+----------------+

1 linha em conjunto (0,00 seg)

Mostrar todos os campos de uma tabela

DESCRIBE databaseName.tableName;

ou, se já estiver usando um banco de dados:

DESCRIBE tableName;

Valor de retorno:

+-----------+----------------+--------+---------+-------------------+-------+
| Field     | Type           | Null   | Key     | Default           | Extra |
+-----------+----------------+--------+---------+-------------------+-------+
| fieldname | fieldvaluetype | NO/YES | keytype | defaultfieldvalue |       |
+-----------+----------------+--------+---------+-------------------+-------+

Extra pode conter auto_increment por exemplo.

Key refere-se ao tipo de chave que pode afetar o campo. Primário (PRI), Único (UNI) …

n linhas no conjunto (0,00 seg)

Onde n é o número de campos na tabela.

Criando usuário

Primeiro, você precisa criar um usuário e, em seguida, dar ao usuário permissões em determinados bancos de dados/tabelas. Ao criar o usuário, você também precisa especificar de onde esse usuário pode se conectar.

CREATE USER 'user'@'localhost' IDENTIFIED BY 'some_password';

Irá criar um usuário que só pode se conectar na máquina local onde o banco de dados está hospedado.

CREATE USER 'user'@'%' IDENTIFIED BY 'some_password';

Irá criar um usuário que pode se conectar de qualquer lugar (exceto a máquina local).

Valor de retorno de exemplo:

Consulta OK, 0 linhas afetadas (0,00 seg)

Adicionando privilégios

Conceda privilégios básicos comuns ao usuário para todas as tabelas do banco de dados especificado:

GRANT SELECT, INSERT, UPDATE ON databaseName.* TO 'userName'@'localhost';

Conceda todos os privilégios ao usuário para todas as tabelas em todos os bancos de dados (atenção com isso):

GRANT ALL ON *.* TO 'userName'@'localhost' WITH GRANT OPTION;

Como demonstrado acima, *.* tem como alvo todos os bancos de dados e tabelas, databaseName.* tem como alvo todas as tabelas do banco de dados específico. Também é possível especificar banco de dados e tabela como databaseName.tableName.

WITH GRANT OPTION deve ser omitido se o usuário não precisar conceder privilégios a outros usuários.

Os privilégios podem ser qualquer um

ALL

ou uma combinação dos seguintes, cada um separado por uma vírgula (lista não exaustiva).

SELECT
INSERT
UPDATE
DELETE
CREATE
DROP

Observação

Geralmente, você deve tentar evitar o uso de nomes de colunas ou tabelas que contenham espaços ou palavras reservadas em SQL. Por exemplo, é melhor evitar nomes como table ou first name.

Se você precisar usar esses nomes, coloque-os entre os delimitadores `` com acento grave. Por exemplo:

CREATE TABLE `table`
(   
   `first name` VARCHAR(30)
);

Uma consulta contendo os delimitadores de acento grave nesta tabela pode ser:

 SELECT `first name` FROM `table` WHERE `first name` LIKE 'a%';

Exemplos de esquema de informações #

Lista de processos

Isso mostrará todas as consultas ativas e adormecidas nessa ordem e por quanto tempo. 
SELECT * FROM information_schema.PROCESSLIST ORDER BY INFO DESC, TIME DESC;

Isso é um pouco mais detalhado sobre os prazos, pois é em segundos por padrão

SELECT ID, USER, HOST, DB, COMMAND, 
TIME as time_seconds, 
ROUND(TIME / 60, 2) as time_minutes, 
ROUND(TIME / 60 / 60, 2) as time_hours, 
STATE, INFO
FROM information_schema.PROCESSLIST ORDER BY INFO DESC, TIME DESC;

Pesquisa de procedimento armazenado

Pesquise facilmente em todos os “Procedimentos armazenados” por palavras e curingas.

SELECT * FROM information_schema.ROUTINES WHERE ROUTINE_DEFINITION LIKE '%word%';
