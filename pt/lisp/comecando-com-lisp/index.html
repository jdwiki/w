Dialetos de Lisp e suas implementações #

Inventado por John McCarthy por volta de 1958, o Lisp (List Processor) continuou a crescer em uma família inteira de idiomas.

Como o StackOverflow é mais sobre problemas práticos de programação, normalmente os problemas envolvem dialetos Lisp reais ou linguagens derivadas e suas implementações. Problemas geralmente relacionados ao Lisp podem ser marcados com lisp. Existem muitos dialetos e implementações, mas nem todos são significativos para o StackOverflow.

Dialetos importantes e idiomas relacionados são:

  • Common Lisp (tag), uma linguagem de propósito geral com muitas implementações e um padrão – implementações populares são SBCL, CLISP, Clozure CL e outros
  • Emacs Lisp (tag), um dialeto Lisp e sua implementação usado principalmente no editor GNU Emacs
  • Scheme (tag), uma linguagem Lisp-like com muitas implementações e um padrão em evolução
  • Racket (tag), uma linguagem e sua implementação derivada de Scheme
  • Clojure (tag), uma linguagem e uma implementação principal para a JVM

Instalação ou Configuração #

Provavelmente, as duas implementações gratuitas mais populares do Common Lisp são Clozure Common Lisp (CCL) e Steel Bank Common Lisp (SBCL). Ambos estão disponíveis para uma variedade de plataformas, incluindo Linux em x86-64 e Linux em ARM.

CCL: http://ccl.clozure.com/download.html

SBCL: http://www.sbcl.org/getting.html

Além do compilador e do REPL (Read-Eval-Print Loop) básico, você pode querer algum tipo de ambiente de desenvolvimento. Uma configuração popular é usar o Emacs para editar texto interativamente. O Modo de Interação Superior Lisp para Emacs (SLIME) permite que o Emacs se conecte a uma implementação Lisp e avalie o código interativamente, a partir do arquivo de texto editável e de um REPL dentro do editor Emacs:

https://common-lisp.net/project/slime/
