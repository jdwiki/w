Sintaxe #

  • String s = config.getString("path.to.string");
  • int i = config.getInt("path.to.int");
  • double d = config.getDouble("path.to.double");
  • List<String> sl = config.getStringList("path.to.stringlist");
  • List<Double> dl = config.getDoubleList("path.to.doublelist");
  • List<Integer> il = config.getIntegerList("path.to.integerlist");

Os arquivos de configuração do Bukkit são arquivos Y.A.M.L (Yet Another Markup Language) diretos e são implementados como tal.

Plugin Config.yml #

Você pode ter um arquivo config.yml que carrega diretamente do seu arquivo jar. Ele deve ser adicionado à pasta do seu projeto, da mesma forma que o arquivo plugin.yml.

Neste arquivo você tem os valores padrão para sua configuração.

Configuração de exemplo:

# This is an YML comment
adminName: "Kerooker"
moderators: ["Romario", "Pelé", "Cafú"]

O arquivo de configuração de exemplo deve ser adicionado à pasta do projeto.

Para carregar o arquivo de configuração padrão na pasta do seu plugin, o seguinte código deve ser adicionado ao seu onEnable():

saveDefaultConfig();

Isso fará com que seu arquivo config.yml do projeto seja o arquivo de configuração do seu plugin e o adicionará à pasta do seu plugin.

A partir daí, você pode acessar seu arquivo de configuração de qualquer lugar, usando sua instância de plug-in:

JavaPlugin plugin; // Your plugin instance
FileConfiguration config = plugin.getConfig();  //Accessing the config file

A partir daí, podemos acessar tudo o que foi definido na configuração do plugin.

Nota: O arquivo de configuração padrão pode ter seus valores alterados, caso o usuário queira editar o arquivo config.yml gerado para a pasta.

String adminName = config.getString("adminName");
List<String> moderators =  config.getStringList("moderators");

Seção de vários caminhos #

O que pode acontecer no seu arquivo de configuração é ter um caminho para uma variável que passa por várias seções.

Configuração de exemplo

admins:
 first-tier: "Kerooker"
 second-tier: "Mordekaiser"
 third-tier: "Yesh4"

O nome “Kerooker” é da seção “first-tier”, que é da seção “admins”. Para acessar os caminhos internos do nosso arquivo, usamos um simples ‘.’ como uma forma de dizer que queremos a próxima seção. Então, para acessarmos “Kerooker”, vamos:

config.getString("admins.first-tier");
