Sintaxe #

  • lua_State *L = lua_open(); // Cria um novo estado de VM; Lua 5.0
  • two_State *L = twoL_newstate(); // Cria um novo estado de VM; Dois 5.1+
  • int luaL_dofile(lua_State *L, const char *nome do arquivo); // Executa um script lua com o nome do arquivo fornecido usando o lua_State especificado
  • void luaL_openlibs(lua_State *L); // Carrega todas as bibliotecas padrão no lua_State especificado
  • void lua_close(lua_State *L); // Fecha o estado da VM e libera todos os recursos dentro
  • void lua_call(lua_State *L, int nargs, int nresults); // Chama o valor lua no índice -(nargs + 1)

Lua também fornece uma API C adequada para sua máquina virtual. Ao contrário da própria VM, a interface da API C é baseada em pilha. Portanto, a maioria das funções destinadas a serem usadas com dados é adicionar algumas coisas no topo da pilha virtual ou remover dela. Além disso, todas as chamadas de API devem ser usadas com cuidado dentro da pilha e suas limitações.

Em geral, qualquer coisa disponível na linguagem Lua pode ser feita usando sua API C. Além disso, há algumas funcionalidades de adição, como acesso direto ao registro interno, alteração de comportamento do alocador de memória padrão ou coletor de lixo.

Você pode compilar os exemplos fornecidos da API Lua C executando o seguinte em seu terminal:

$ gcc -Wall ./example.c -llua -ldl -lm

Chamando funções Lua #

#include <stdlib.h>

#include <lauxlib.h>
#include <lua.h>
#include <lualib.h>

int main(void)
{
    lua_State *lvm_hnd = lua_open();
    luaL_openlibs(lvm_hnd);

    /* Load a standard Lua function from global table: */
    lua_getglobal(lvm_hnd, "print");

    /* Push an argument onto Lua C API stack: */
    lua_pushstring(lvm_hnd, "Hello C API!");

    /* Call Lua function with 1 argument and 0 results: */
    lua_call(lvm_hnd, 1, 0);

    lua_close(lvm_hnd);

    return EXIT_SUCCESS;
 }

No exemplo acima, estamos fazendo estas coisas:

  • criando e configurando Lua VM como mostrado no primeiro exemplo
  • obter e enviar uma função Lua da tabela Lua global para a pilha virtual
  • empurrando a string "Hello C API" como um argumento de entrada para a pilha virtual
  • instruindo a VM a chamar uma função com um argumento que já está na pilha
  • fechamento e limpeza

NOTA:

Tenha em mente que lua_call() desempilha a função e seus argumentos da pilha deixando apenas o resultado.

Além disso, seria mais seguro usar uma chamada protegida por Lua - lua_pcall().

Criando Máquina Virtual Lua #

#include <lua.h>
#include <lauxlib.h>
#include <lualib.h>

int main(void)
{

  /* Start by creating a new VM state */
  lua_State *L = luaL_newstate();

  /* Load standard Lua libraries: */
  luaL_openlibs(L);

  /* For older version of Lua use lua_open instead */
  lua_State *L = lua_open();

  /* Load standard libraries*/
  luaopen_base(L);
  luaopen_io(L);
  luaopen_math(L);
  luaopen_string(L);
  luaopen_table(L);

    /* do stuff with Lua VM. In this case just load and execute a file: */
    luaL_dofile(L, "some_input_file.lua");

    /* done? Close it then and exit. */
    lua_close(L);

    return EXIT_SUCCESS;
}

Interpretador de Lua incorporado com API personalizada e personalização de Lua #

Demonstre como embutir um interpretador lua no código C, expor uma função definida em C para um script Lua, avaliar um script Lua, chamar uma função definida em C de Lua e chamar uma função definida em Lua de C (o host).

Neste exemplo, queremos que o clima seja definido por um script Lua. Aqui está mood.lua:

-- Get version information from host
major, minor, build = hostgetversion()
print( "The host version is ", major, minor, build)
print("The Lua interpreter version is ", _VERSION)

-- Define a function for host to call
function mood( b )

    -- return a mood conditional on parameter
    if (b and major > 0) then
        return 'mood-happy'
    elseif (major == 0) then
        return 'mood-confused'
    else
        return 'mood-sad'
    end
end

Observe que mood() não é chamado no script. Ele é definido apenas para o aplicativo host chamar. Observe também que o script chama uma função chamada hostgetversion() que não está definida no script.

Em seguida, definimos um aplicativo host que usa ‘mood.lua’. Aqui está o ‘hostlua.c’:

#include <stdio.h>
#include <lua.h>
#include <lualib.h>
#include <lauxlib.h>

/* 
 * define a function that returns version information to lua scripts
 */
static int hostgetversion(lua_State *l)
{
    /* Push the return values */
    lua_pushnumber(l, 0);
    lua_pushnumber(l, 99);
    lua_pushnumber(l, 32);

    /* Return the count of return values */
    return 3;
}

int main (void)
{
    lua_State *l = luaL_newstate();
    luaL_openlibs(l);

    /* register host API for script */
    lua_register(l, "hostgetversion", hostgetversion);

    /* load script */
    luaL_dofile(l, "mood.lua");

    /* call mood() provided by script */
    lua_getglobal(l, "mood");
    lua_pushboolean(l, 1);
    lua_call(l, 1, 1);

    /* print the mood */
    printf("The mood is %s\n", lua_tostring(l, -1));
    lua_pop(l, 1);

    lua_close(l);
    return 0;
}

E aqui está a saída:

The host version is     0    99    32
Lua interpreter version is     Lua 5.2
The mood is mood-confused

Mesmo depois de compilar ‘hostlua.c’, ainda podemos modificar ‘mood.lua’ para alterar a saída do nosso programa!

Manipulação de tabela #

Para acessar ou alterar um índice em uma tabela, você precisa de alguma forma colocar a tabela na pilha.
Vamos supor, para este exemplo, que sua tabela seja uma variável global chamada tbl.

Obtendo o conteúdo em um índice específico: #

int getkey_index(lua_State *L)
{
  lua_getglobal(L, "tbl");    // this put the table in the stack
  lua_pushstring(L, "index"); // push the key to access
  lua_gettable(L, -2);        // retrieve the corresponding value; eg. tbl["index"]

  return 1;                   // return value to caller
}

Como vimos, tudo o que você precisa fazer é empurrar a tabela para a pilha, empurrar o índice e chamar lua_gettable. o argumento -2 significa que a tabela é o segundo elemento do topo da pilha.
lua_gettable aciona metamétodos. Se você não quiser acionar metamétodos, use lua_rawget. Ele usa os mesmos argumentos.

Configurando o conteúdo em um índice específico: #

int setkey_index(lua_State *L)
{
  // setup the stack
  lua_getglobal(L, "tbl");
  lua_pushstring(L, "index");
  lua_pushstring(L, "value");
  // finally assign the value to table; eg. tbl.index = "value"
  lua_settable(L, -3);

  return 0;
}

O mesmo exercício de obter o conteúdo. Você tem que empurrar a pilha, empurrar o índice e então empurrar o valor para a pilha. depois disso, você chama lua_settable. o argumento -3 é a posição da tabela na pilha. Para evitar o acionamento de metamétodos, use lua_rawset em vez de lua_settable. Ele usa os mesmos argumentos.

Transferindo o conteúdo de uma tabela para outra: #

int copy_tableindex(lua_State *L)
{
    lua_getglobal(L, "tbl1"); // (tbl1)
    lua_getglobal(L, "tbl2");// (tbl1)(tbl2)
    lua_pushstring(L, "index1");// (tbl1)(tbl2)("index1")
    lua_gettable(L, -3);// (tbl1)(tbl2)(tbl1.index1)
    lua_pushstring(L, "index2");// (tbl1)(tbl2)(tbl1.index1)("index2")
    lua_pushvalue(L, -2); // (tbl1)(tbl2)(tbl1.index1)("index2")(tbl1.index1)
    lua_settable(L, -4);// (tbl1)(tbl2)(tbl1.index1)
    lua_pop(L, 1);

    return 0;
}

Agora estamos juntando tudo o que aprendemos aqui. Eu coloco o conteúdo da pilha nos comentários para você não se perder.

Colocamos ambas as tabelas na pilha, colocamos o índice da tabela 1 na pilha e obtemos o valor em tbl1.index1. Observe o argumento -3 em gettable. Estou olhando para a primeira mesa (terceira de cima para baixo) e não para a segunda. Então nós empurramos o índice da segunda tabela, copiamos o tbl1.index1 para o topo da pilha e então chamamos lua_settable, no 4º item do topo.

Por uma questão de limpeza doméstica, limpei o elemento superior, de modo que apenas as duas mesas permaneçam na pilha.
