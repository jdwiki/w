Sintaxe #

  • string.find(str, pattern [, init [, plain]]) – Retorna o índice inicial e final da correspondência em str

  • string.match(str, pattern [, index]) – Corresponde a um padrão uma vez (começando no índice)

  • string.gmatch(str, pattern) – Retorna uma função que itera por todas as correspondências em str

  • string.gsub(str, pattern, repl [, n]) – Substitui substrings (até um máximo de n vezes)

  • . representa todos os caracteres

  • %a representa todas as letras

  • %l representa todas as letras minúsculas

  • %u representa todas as letras maiúsculas

  • %d representa todos os dígitos

  • %x representa todos os dígitos hexadecimais

  • %s representa todos os caracteres de espaço em branco

  • %p representa todos os caracteres de pontuação

  • %g representa todos os caracteres imprimíveis exceto espaço

  • %c representa todos os caracteres de controle

  • [set] representa a classe que é a união de todos os caracteres do conjunto.

  • [^set] representa o complemento de set

  • * correspondência gananciosa 0 ou mais ocorrências da classe de caractere anterior

  • + ganancioso corresponde a 1 ou mais ocorrências da classe de caractere anterior

  • - lazy match 0 ou mais ocorrências da classe de caractere anterior

  • ? corresponde exatamente a 0 ou 1 ocorrência da classe de caractere anterior

Ao longo de alguns exemplos, é usada a notação (<string literal>):function <string literal>, que é equivalente a string.function(<string literal>, <string literal>) porque todas as strings possuem uma metatabela com o campo __index definido para a tabela string.

Correspondência de padrões Lua #

Em vez de usar regex, a biblioteca de strings Lua tem um conjunto especial de caracteres usados ​​em correspondências de sintaxe. Ambos podem ser muito semelhantes, mas a correspondência de padrões Lua é mais limitada e tem uma sintaxe diferente. Por exemplo, a sequência de caracteres %a corresponde a qualquer letra, enquanto sua versão em maiúsculas representa todos os caracteres que não são letras, todas as classes de caracteres (uma sequência de caracteres que, como padrão, pode corresponder a um conjunto de itens) estão listados abaixo.

Classe de personagem Seção correspondente
%a letras (A-Z, a-z)
%c caracteres de controle (\n, \t, \r, …)
%d dígitos (0-9)
%l letra minúscula (a-z)
%p caracteres de pontuação (!, ?, &, …)
%s caracteres de espaço
%u letras maiúsculas
%w caracteres alfanuméricos (A-Z, a-z, 0-9)
%x dígitos hexadecimais (\3, \4, …)
%z o personagem com representação 0
. Corresponde a qualquer caractere

Como mencionado acima, qualquer versão em maiúsculas dessas classes representa o complemento da classe. Por exemplo, %D corresponderá a qualquer sequência de caracteres sem dígitos:

string.match("f123", "%D")          --> f

Além das classes de caracteres, alguns caracteres possuem funções especiais como padrões:

( ) % . + - * [ ? ^ $

O caractere % representa um escape de caractere, fazendo com que %? corresponda a uma interrogação e %% corresponda ao símbolo de porcentagem. Você pode usar o caractere % com qualquer outro caractere não alfanumérico, portanto, se precisar de escape, por exemplo, uma aspa, deve usar \\ antes dele, que escapa qualquer caractere de uma string lua.

Um conjunto de caracteres, representado entre colchetes ([]), permite criar uma classe de caracteres especial, combinando diferentes classes e caracteres únicos:

local foo = "bar123bar2341"
print(foo:match "[arb]")            --> b

Você pode obter o complemento do conjunto de caracteres iniciando-o com ^:

local foo = "bar123bar2341"
print(string.match(foo, "[^bar]"))  --> 1

Neste exemplo, string.match encontrará a primeira ocorrência que não seja b, a ou r.

Padrões podem ser mais úteis com a ajuda de modificadores de repetição/opcionais, padrões em lua oferecem estes quatro caracteres:

Personagem Modificador
+ Uma ou mais repetições
* Zero ou mais repetições
- Também zero ou mais repetições
? Opcional (zero ou uma ocorrência)

O caractere + representa um ou mais caracteres correspondentes na sequência e sempre retornará a sequência correspondente mais longa:

local foo = "12345678bar123"
print(foo:match "%d+")  --> 12345678

Como você pode ver, * é semelhante a +, mas aceita zero ocorrências de caracteres e é comumente usado para combinar espaços opcionais entre diferentes padrões.

O caractere - também é semelhante a *, mas em vez de retornar a sequência correspondente mais longa, ele corresponde à sequência mais curta.

O modificador ? corresponde a um caractere opcional, permitindo que você corresponda, por exemplo, a um dígito negativo:

local foo = "-20"
print(foo:match "[+-]?%d+")

O mecanismo de correspondência de padrões Lua fornece alguns itens adicionais de correspondência de padrões:

Item de personagem Descrição
%n para n entre 1 e 9 corresponde a uma substring igual à n-ésima string capturada
%bxy corresponde a substring entre dois caracteres distintos (par balanceado de x e y)
%f[set] padrão de fronteira: corresponde a uma string vazia em qualquer posição, de modo que o próximo caractere
pertença ao conjunto e o caractere anterior não pertença ao conjunto

string.find (Introdução) #

A função localizar #

Primeiro vamos dar uma olhada na função string.find em geral:

A função string.find (s, substr [, init [, plain]]) retorna o índice inicial e final de uma substring se encontrada, e nil caso contrário, iniciando no índice init se for fornecido (o padrão é 1).

("Hello, I am a string"):find "am" --> returns 10 11
-- equivalent to string.find("Hello, I am a string", "am") -- see remarks

Apresentando padrões #

("hello world"):find ".- " -- will match characters until it finds a space
    --> so it will return 1, 6

Todos exceto os caracteres a seguir representam a si próprios ^$()%.[]*+-?). Qualquer um desses caracteres pode ser representado por um % seguindo o próprio caractere.

("137'5 m47ch s0m3 d1g175"):find "m%d%d" -- will match an m followed by 2 digit
    --> this will match m47 and return 7, 9

("stack overflow"):find "[abc]" -- will match an 'a', a 'b' or a 'c'
    --> this will return 3 (the A in stAck)

("stack overflow"):find "[^stack ]"
    -- will match all EXCEPT the letters s, t, a, c and k and the space character
    --> this will match the o in overflow

("hello"):find "o%d?" --> matches o, returns 5, 5
("hello20"):find "o%d?" --> matches o2, returns 5, 6
    -- the ? means the character is optional

("helllllo"):find "el+" --> will match elllll
("heo"):find "el+" --> won't match anything

("helllllo"):find "el*" --> will match elllll
("heo"):find "el*" --> will match e

("helelo"):find "h.+l" -- + will match as much as it gets
    --> this matches "helel"
("helelo"):find "h.-l" -- - will match as few as it can
    --> this wil only match "hel"

("hello"):match "o%d*"
    --> like ?, this matches the "o", because %d is optional
("hello20"):match "o%d*"
    --> unlike ?, it maches as many %d as it gets, "o20"
("hello"):match "o%d"
    --> wouldn't find anything, because + looks for 1 or more characters

A função gmatch #

Como funciona #

A função string.gmatch receberá uma string de entrada e um padrão. Este padrão descreve o que realmente receber de volta. Esta função retornará uma função que na verdade é um iterador. O resultado desse iterador corresponderá ao padrão.

type(("abc"):gmatch ".") --> returns "function"

for char in ("abc"):gmatch "." do
    print char -- this prints:
    --> a
    --> b
    --> c
end

for match in ("#afdde6"):gmatch "%x%x" do
    print("#" .. match) -- prints:
    --> #af
    --> #dd
    --> #e6
end

Apresentando capturas: #

Isso é muito semelhante à função normal, no entanto, retornará apenas as capturas em vez da correspondência completa.

for key, value in ("foo = bar, bar=foo"):gmatch "(%w+)%s*=%s*(%w+)" do
    print("key: " .. key .. ", value: " .. value)
    --> key: foo, value: bar
    --> key: bar, value: foo
end

A função gsub #

não confunda com a função string.sub, que retorna uma substring!

Como funciona #

argumento de string #

("hello world"):gsub("o", "0")
    --> returns "hell0 w0rld", 2
    -- the 2 means that 2 substrings have been replaced (the 2 Os)

("hello world, how are you?"):gsub("[^%s]+", "word")
    --> returns "word word, word word word?", 5

("hello world"):gsub("([^%s])([^%s]*)", "%2%1")
    --> returns "elloh orldw", 2

argumento de função #

local word = "[^%s]+"

function func(str)
    if str:sub(1,1):lower()=="h" then
        return str
    else
        return "no_h"
    end
end
("hello world"):gsub(word, func)
    --> returns "hello no_h", 2

argumento de tabela #

local word = "[^%s]+"

sub = {}
sub["hello"] = "g'day"
sub["world"] = "m8"

("hello world"):gsub(word, sub)
    --> returns "g'day m8"

("hello world, how are you?"):gsub(word, sub)
    --> returns "g'day m8, how are you?"
    -- words that are not in the table are simply ignored
