Iteradores com estado #

Os iteradores com estado carregam algumas informações adicionais sobre o estado atual do iterador.

Usando tabelas #

O estado de adição pode ser empacotado no estado invariável do loop for genérico.

  local function chars_iter(t, i)
    local i = i + 1
    if i <= t.len then
      return i, t.s:sub(i, i)
    end
  end

  local function chars(s)
    -- the iterators state
    local t = {
      s = s,    -- the subject
      len = #s  -- cached length
    }
    return chars_iter, t, 0
  end

  for i, c in chars 'abcde' do
    print(i, c) --> 1 a, 2 b, 3 c, 4 d, 5 e
  end

Usando Fechamentos #

O estado adicional pode ser encapsulado em um encerramento de função. Como o estado está totalmente contido no escopo do fechamento, o estado invariável e a variável de controle não são necessários.

  local function chars(s)
    local i, len = 0, #s
    return function() -- iterator function
      i = i + 1
      if i <= len then
        return i, s:sub(i, i)
      end
    end
  end

  for i, c in chars 'abcde' do
    print(i, c) --> 1 a, 2 b, 3 c, 4 d, 5 e
  end

Usando Corrotinas #

Estado adicional pode estar contido em uma corrotina, novamente o estado invariável e a variável de controle não são necessários.

  local function chars(s)
    return coroutine.wrap(function()
      for i = 1, #s do
        coroutine.yield(s:sub(i, i))
      end
    end)
  end

  for c in chars 'abcde' do
    print(c) --> a, b, c, d, e
  end

Genérico para Loop #

Os iteradores utilizam uma forma do loop for conhecido como loop for genérico.

A forma genérica do loop for usa três parâmetros:

  1. Uma função iteradora que é chamada quando o próximo valor é necessário. Ele recebe o estado invariável e a variável de controle como parâmetros. Retornando nil terminação dos sinais.
  2. O invariant state é um valor que não muda durante a iteração. Normalmente é o assunto do iterador, como uma tabela, string ou userdata.
  3. A variável control representa um valor inicial para iteração.

Podemos escrever um loop for para iterar todos os pares chave-valor em uma tabela usando a função next.

local t = {a=1, b=2, c=3, d=4, e=5}

-- next is the iterator function
-- t is the invariant state
-- nil is the control variable (calling next with a nil gets the first key)
for key, value in next, t, nil do
  -- key is the new value for the control variable
  print(key, value) 
  -- Lua calls: next(t, key)  
end

Iteradores sem estado #

Ambos pares e [ipairs](http://www.lua.org/manual/5.3/manual.html# pdf-ipairs) representam iteradores sem estado. Um iterador sem estado usa apenas a variável de controle generic for loops e o estado invariável para calcular o valor da iteração.

Iterador de Pares #

Podemos implementar o iterador pairs sem estado usando a função next.

-- generator function which initializes the generic for loop
local function pairs(t)
  -- next is the iterator function
  -- t is the invariant state
  -- control variable is nil
  return next, t, nil
end

Ipairs Iterador #

Podemos implementar o iterador ipairs sem estado em duas funções separadas.

-- function which performs the actual iteration
local function ipairs_iter(t, i)
  local i = i + 1  -- next index in the sequence (i is the control variable)
  local v = t[i]   -- next value (t is the invariant state)
  if v ~= nil then
    return i, v    -- index, value
  end
  return nil       -- no more values (termination)
end

-- generator function which initializes the generic for loop
local function ipairs(t)
  -- ipairs_iter is the iterator function
  -- t is the invariant state (table to be iterated)
  -- 0 is the control variable (first index)
  return ipairs_iter, t, 0
end

Iterador de caracteres #

Podemos criar novos iteradores sem estado cumprindo o contrato do loop genérico for.

-- function which performs the actual iteration
local function chars_iter(s, i)
  if i < #s then
    i = i + 1
    return i, s:sub(i, i)
  end
end

-- generator function which initializes the generic for loop
local function chars(s)
  return chars_iter, s, 0
end

-- used like pairs and ipairs
for i, c in chars 'abcde' do
    print(i, c) --> 1 a, 2 b, 3 c, 4 f, 5 e
end

Iterador de números primos #

Este é mais um exemplo simples de um iterador sem estado.

-- prime numbers iterator
local incr = {4, 1, 2, 0, 2}
function primes(s, p, d)
   s, p, d = s or math.huge, p and p + incr[p % 6] or 2, 1
   while p <= s do
      repeat
         d = d + incr[d % 6]
         if d*d > p then return p end
      until p % d == 0
      p, d = p + incr[p % 6], 1
   end
end

-- print all prime numbers <= 100
for p in primes, 100 do  -- passing in the iterator (do not call the iterator here)
   print(p)  -->  2  3  5  7  11 ... 97
end

-- print all primes in endless loop
for p in primes do  -- please note: "in primes", not "in primes()"
   print(p)
end

Iteradores padrão #

A biblioteca padrão Lua fornece duas funções iteradoras que podem ser usadas com um loop for para percorrer pares chave-valor dentro de tabelas.

Para iterar sobre uma tabela de sequência, podemos usar a função de biblioteca ipairs.

for index, value in ipairs {'a', 'b', 'c', 'd', 'e'} do
  print(index, value)  --> 1 a, 2 b, 3 c, 4 d, 5 e
end

Para iterar sobre todas as chaves e valores em qualquer tabela, podemos usar a função de biblioteca pairs.

for key, value in pairs {a=1, b=2, c=3, d=4, e=5} do
  print(key, value)  --> e 5, c 3, a 1, b 2, d 4  (order not specified)
end
