Argumentos Variados

  • Sintaxe
  • Fundamentos
  • Uso Avançado

Nesta página

  • Sintaxe
  • Fundamentos
  • Uso Avançado

Varargs, como são comumente conhecidos, permitem que funções recebam um número arbitrário de argumentos sem especificação. Todos os argumentos dados a tal função são empacotados em uma única estrutura conhecida como vararg list; que é escrito como ... em Lua. Existem métodos básicos para extrair o número de argumentos fornecidos e o valor desses argumentos usando a função select(), mas padrões de uso mais avançados podem alavancar a estrutura para sua utilidade total.

Sintaxe #

  • … – Torna a função cuja lista de argumentos na qual aparece uma função variádica
  • select(what, …) – Se ‘what’ for um número no intervalo 1 para o número de elementos no vararg, retorna o ‘what’th elemento para o último elemento no vararg. O retorno será nulo se o índice estiver fora dos limites. Se ‘what’ for a string ‘#’, retorna o número de elementos no vararg.

Eficiência

A lista vararg é implementada como uma lista encadeada na implementação da linguagem PUC-Rio, isso significa que os índices são O(n). Isso significa que iterar sobre os elementos em um vararg usando select(), como no exemplo abaixo, é uma operação O(n^2).

for i = 1, select('#', ...) do
    print(select(i, ...))
end

Se você planeja iterar sobre os elementos em uma lista vararg, primeiro coloque a lista em uma tabela. Os acessos à tabela são O(1), então a iteração é O(n) no total. Ou, se você quiser, veja o exemplo foldr() da seção de uso avançado; ele usa recursão para iterar sobre uma lista vararg em O(n).

Definição do comprimento da sequência

O vararg é útil porque o comprimento do vararg respeita quaisquer nils explicitamente passados ​​(ou computados). Por exemplo.

function test(...)
    return select('#', ...)
end

test()             --> 0
test(nil, 1, nil)  --> 3

Este comportamento entra em conflito com o comportamento das tabelas no entanto, onde o operador de comprimento # não funciona com ‘buracos’ (nils embutidos) em sequências. O cálculo do comprimento de uma mesa com furos é indefinido e não é confiável. Portanto, dependendo dos valores em ..., tomar o comprimento de {...} pode não resultar na resposta ‘correta’. Em Lua 5.2+ table.pack() foi introduzido para lidar com essa deficiência (há uma função no exemplo que implementa essa função em Lua pura).

Uso Idiomático

Como os varargs carregam seu comprimento, as pessoas os usam como sequências para evitar o problema de buracos nas tabelas. Este não era o uso pretendido e uma implementação de referência de Lua não otimiza. Embora tal uso seja explorado nos exemplos, geralmente é desaprovado.

Fundamentos #

As funções variadicas são criadas usando a sintaxe de elipses ... na lista de argumentos da definição da função.

function id(...)
    return
end

Se você chamou esta função como id(1, 2, 3, 4, 5) então ... (AKA a lista vararg) conteria os valores 1, 2, 3, 4, 5.

As funções podem receber argumentos necessários, bem como ....

function head(x, ...)
    return x
end

A maneira mais fácil de extrair elementos da lista vararg é simplesmente atribuir variáveis ​​a partir dela.

function head3(...)
    local a, b, c = ...
    return a, b, c
end

select() também pode ser usado para encontrar o número de elementos e extrair elementos de ... indiretamente.

function my_print(...)
    for i = 1, select('#', ...) do
        io.write(tostring(select(i, ...)) .. '\t')
    end
    io.write '\n'
end

... pode ser empacotado em uma tabela para facilidade de uso, usando {...}. Isso coloca todos os argumentos na parte sequencial da tabela.

table.pack(...) também pode ser usado para empacotar a lista vararg em uma tabela. A vantagem do table.pack(...) é que ele define o campo n da tabela retornada para o valor de select('#', ...). Isso é importante se sua lista de argumentos pode conter nils (veja a seção de comentários abaixo).

function my_tablepack(...)
    local t = {...}
    t.n = select('#', ...)
    return t
end

A lista vararg também pode ser retornada de funções. O resultado são vários retornos.

function all_or_none(...)
    local t = table.pack(...)
    for i = 1, t.n do
        if not t[i] then
            return    -- return none
        end
    end
    return ...    -- return all
end

Uso Avançado #

Conforme indicado nos exemplos básicos, você pode ter argumentos vinculados a variáveis ​​e a lista de argumentos variáveis ​​(...). Você pode usar esse fato para separar recursivamente uma lista como faria em outras linguagens (como Haskell). Abaixo está uma implementação de foldr() que tira vantagem disso. Cada chamada recursiva liga o cabeçalho da lista vararg a x e passa o resto da lista para uma chamada recursiva. Isso desestrutura a lista até que haja apenas um argumento (select('#', ...) == 0). Depois disso, cada valor é aplicado ao argumento da função f com o resultado calculado anteriormente.

function foldr(f, ...)
    if select('#', ...) < 2 then return ... end
    local function helper(x, ...)
        if select('#', ...) == 0 then
          return x
        end
        return f(x, helper(...))
    end
    return helper(...)
end

function sum(a, b)
    return a + b
end

foldr(sum, 1, 2, 3, 4)
--> 10

Você pode encontrar outras definições de função que aproveitam esse estilo de programação aqui na edição nº 3 até a edição nº 8.

A única estrutura de dados idiomática de Lua é a tabela. O operador de comprimento da tabela é indefinido se houver nils localizados em qualquer lugar em uma sequência. Ao contrário das tabelas, a lista vararg respeita nils explícitos como indicado nos exemplos básicos e na seção de comentários (por favor, leia essa seção se você ainda não leu). Com pouco trabalho, a lista vararg pode executar todas as operações que uma tabela pode além da mutação. Isso torna a lista vararg uma boa candidata para implementar tuplas imutáveis.

function tuple(...)
    -- packages a vararg list into an easily passable value
    local co = coroutine.wrap(function(...)
        coroutine.yield()
        while true do
            coroutine.yield(...)
        end
    end)
    co(...)
    return co
end

local t = tuple((function() return 1, 2, nil, 4, 5 end)())

print(t())                 --> 1    2    nil    4    5    | easily unpack for multiple args
local a, b, d = t()        --> a = 1, b = 2, c = nil      | destructure the tuple
print((select(4, t())))    --> 4                          | index the tuple
print(select('#', t()))    --> 5                          | find the tuple arity (nil respecting)

local function change_index(tpl, i, v)
    -- sets a value at an index in a tuple (non-mutating)
    local function helper(n, x, ...)
        if select('#', ...) == 0 then
            if n == i then
                return v
            else
                return x
            end
        else
            if n == i then
                return v, helper(n+1, ...)
            else
                return x, helper(n+1, ...)
            end
        end
    end
    return tuple(helper(1, tpl()))
end

local n = change_index(t, 3, 3)
print(t())                 --> 1    2    nil    4    5
print(n())                 --> 1    2    3    4    5

A principal diferença entre o que está acima e as tabelas é que as tabelas são mutáveis ​​e possuem semântica de ponteiro, onde a tupla não possui essas propriedades. Além disso, as tuplas podem conter nils explícitos e ter uma operação de comprimento nunca indefinida.
