Sintaxe #

  • coroutine.create(function) retorna uma coroutine (type(coroutine) == ’thread’) contendo a função.

  • coroutine.resume(co, …) resume ou inicia a coroutine. Quaisquer argumentos adicionais fornecidos para retomar são retornados do coroutine.yield() que anteriormente pausou a coroutine. Se a corrotina não foi iniciada, os argumentos adicionais tornam-se os argumentos da função.

  • coroutine.yield(…) produz a corrotina atualmente em execução. A execução é retomada após a chamada para coroutine.resume() que iniciou essa coroutine. Quaisquer argumentos fornecidos para yield são retornados do coroutine.resume() correspondente que iniciou a coroutine.

  • coroutine.status(co) retorna o status da coroutine, que pode ser:

    - "dead" : the function in the coroutine has reached it's end and the coroutine cannot be resumed anymore
- "running" : the coroutine has been resumed and is running
- "normal" : the coroutine has resumed another coroutine
- "suspended" : the coroutine has yielded, and is waiting to be resumed

  • coroutine.wrap(function) retorna uma função que quando chamada retoma a coroutine que teria sido criada por coroutine.create(function).

O sistema de corrotina foi implementado em lua para emular multithreading existente em outras linguagens. Ele funciona alternando em altíssima velocidade entre diferentes funções para que o usuário humano pense que elas são executadas ao mesmo tempo.

Crie e use uma corrotina #

Todas as funções para interagir com corrotinas estão disponíveis na tabela corrotina. Uma nova corrotina é criada usando a função coroutine.create com um único argumento: uma função com o código a ser executado:

thread1 = coroutine.create(function()
            print("honk")
        end)

print(thread1)
-->> thread: 6b028b8c

Um objeto de corrotina retorna um valor do tipo thread, representando uma nova corrotina. Quando uma nova corrotina é criada, seu estado inicial é suspenso:

print(coroutine.status(thread1))
-->> suspended

Para retomar ou iniciar uma corrotina, a função coroutine.resume é usada, o primeiro argumento fornecido é o objeto thread:

coroutine.resume(thread1)
-->> honk

Agora a corrotina executa o código e termina, mudando seu estado para morto, que não pode ser retomado.

print(coroutine.status(thread1))
-->> dead

Coroutines podem suspender sua execução e retomá-la mais tarde graças à função coroutine.yield:

thread2 = coroutine.create(function()
        for n = 1, 5 do
            print("honk "..n)
            coroutine.yield()
        end
    end)

Como você pode ver, coroutine.yield() está presente dentro do loop for, agora quando retomarmos a coroutine, ele executará o código até chegar a um coroutine.yield:

coroutine.resume(thread2)
-->> honk 1
coroutine.resume(thread2)
-->> honk 2

Depois de terminar o loop, o status do encadeamento se torna morto e não pode ser retomado. Coroutines também permite a troca entre dados:

thread3 = coroutine.create(function(complement)
    print("honk "..complement)
    coroutine.yield()
    print("honk again "..complement)
end)
coroutine.resume(thread3, "stackoverflow")
-->> honk stackoverflow

Se a corrotina for executada novamente sem argumentos extras, o complemento ainda o argumento do primeiro currículo, neste caso “stackoverflow”:

coroutine.resume(thread3)
-->> honk again stackoverflow

Finalmente, quando uma corrotina termina, quaisquer valores retornados por sua função vão para o currículo correspondente:

thread4 = coroutine.create(function(a, b)
    local c = a+b
    coroutine.yield()
    return c
end)
coroutine.resume(thread4, 1, 2)
print(coroutine.resume(thread4))
-->> true, 3

Corrotinas são usadas nesta função para passar valores de volta para um thread de chamada de dentro de uma chamada recursiva.

local function Combinations(l, r)
    local ll = #l
    r = r or ll
    local sel = {}
    local function rhelper(depth, last)
        depth = depth or 1
        last = last or 1
        if depth > r then
            coroutine.yield(sel)
        else
            for i = last, ll - (r - depth) do
                sel[depth] = l[i]
                rhelper(depth+1, i+1)
            end
        end
    end
    return coroutine.wrap(rhelper)
end

for v in Combinations({1, 2, 3}, 2) do
    print("{"..table.concat(v, ", ").."}")
end
--> {1, 2}
--> {1, 3}
--> {2, 3}

Coroutines também podem ser usadas para avaliação preguiçosa.

-- slices a generator 'c' taking every 'step'th output from the generator
-- starting at the 'start'th output to the 'stop'th output
function slice(c, start, step, stop)
    local _
    return coroutine.wrap(function()
        for i = 1, start-1 do
            _ = c()
        end
        for i = start, stop do
            if (i - start) % step == 0 then
                coroutine.yield(c())
            else
                _ = c()
            end
        end
    end)
end


local alphabet = {}
for c = string.byte('a'), string.byte('z') do
    alphabet[#alphabet+1] = string.char(c)
end
-- only yields combinations 100 through 102
-- requires evaluating the first 100 combinations, but not the next 5311633
local s = slice(Combinations(alphabet, 10), 100, 1, 102)
for i in s do
    print(table.concat(i))
end
--> abcdefghpr
--> abcdefghps
--> abcdefghpt

Corrotinas podem ser usadas para construções de tubulação conforme descrito em Programação em Lua. O autor do PiL, Roberto Ierusalimschy, também publicou um [documento] 2 sobre o uso de corrotinas para implementar mecanismos de controle de fluxo mais avançados e gerais, como continuações.
