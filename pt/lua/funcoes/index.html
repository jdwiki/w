Sintaxe #

  • funcname = function(paramA, paramB, …) corpo; return exprlist end – uma função simples
  • função funcname(paramA, paramB, …) corpo; return exprlist end – abreviação para acima
  • local funcname = function(paramA, paramB, …) corpo; return exprlist end – um lambda
  • local funcname; funcname = function(paramA, paramB, …) corpo; return exprlist end – lambda que pode fazer chamadas recursivas
  • função local funcname(paramA, paramB, …) corpo; return exprlist end – abreviação para acima
  • funcname(paramA, paramB, …) – chama uma função
  • local var = var ou “Default” – um parâmetro padrão
  • return nil, “mensagens de erro” - maneira padrão de abortar com um erro

As funções geralmente são definidas com function a(b,c) ... end e raramente com a configuração de uma variável para uma função anônima (a = function(a,b) ... end). O oposto é verdadeiro ao passar funções como parâmetros, funções anônimas são usadas principalmente e funções normais não são usadas com tanta frequência.

Definindo uma função #

function add(a, b)
    return a + b
end
-- creates a function called add, which returns the sum of it's two arguments

Vejamos a sintaxe. Primeiro, vemos uma palavra-chave function. Bem, isso é bastante descritivo. Em seguida, vemos o identificador add; o nome. Vemos então os argumentos (a, b) estes podem ser qualquer coisa, e eles são locais. Somente dentro do corpo da função podemos acessá-los. Vamos pular para o final, vemos… bem, o ‘fim’! E tudo o que está no meio é o corpo da função; o código que é executado quando é chamado. A palavra-chave return é o que faz a função realmente fornecer alguma saída útil. Sem ele, a função não retorna nada, o que equivale a retornar nil. É claro que isso pode ser útil para coisas que interagem com IO, por exemplo:

function printHello(name)
    print("Hello, " .. name .. "!");
end

Nessa função, não usamos a instrução return.

As funções também podem retornar valores condicionalmente, o que significa que uma função tem a opção de não retornar nada (nil) ou um valor. Isso é demonstrado no exemplo a seguir.

function add(a, b)
    if (a + b <= 100) then
        return a + b -- Returns a value
    else
        print("This function doesn't return values over 100!") -- Returns nil
    end
end

Também é possível que uma função retorne vários valores separados por vírgulas, conforme mostrado:

function doOperations(a, b)
    return a+b, a-b, a*b
end

added, subbed, multiplied = doOperations(4,2)

As funções também podem ser declaradas locais

do
    local function add(a, b) return a+b end
    print(add(1,2)) --> prints 3
end
print(add(2, 2)) --> exits with error, because 'add' is not defined here

Eles também podem ser salvos em tabelas:

tab = {function(a,b) return a+b end}
(tab[1])(1, 2) --> returns 3

Chamando uma função. #

As funções só são úteis se pudermos chamá-las. Para chamar uma função, a seguinte sintaxe é usada:

print("Hello, World!")

Estamos chamando a função print. Usando o argumento "Hello, World". Como é óbvio, isso imprimirá Hello, World no fluxo de saída. O valor retornado é acessível, assim como qualquer outra variável seria.

local added = add(10, 50) -- 60

Variáveis ​​também são aceitas nos parâmetros de uma função.

local a = 10
local b = 60

local c = add(a, b)

print(c)

Funções que esperam uma tabela ou uma string podem ser chamadas com um puro açúcar sintático: os parênteses ao redor da chamada podem ser omitidos.

print"Hello, world!"
for k, v in pairs{"Hello, world!"} do print(k, v) end

Funções anônimas #

Criando funções anônimas #

As funções anônimas são como as funções normais de Lua, exceto que não têm um nome.

doThrice(function()
    print("Hello!")
end)

Como você pode ver, a função não é atribuída a nenhum nome como print ou add. Para criar uma função anônima, basta omitir o nome. Essas funções também podem receber argumentos.

Entendendo o açúcar sintático #

É importante entender que o código a seguir

function double(x)
    return x * 2
end

na verdade é apenas um atalho para

double = function(x)
    return x * 2
end

No entanto, a função acima não é anônima, pois a função é atribuída diretamente a uma variável!

Funções são valores de primeira classe #

Isso significa que uma função é um valor com os mesmos direitos que valores convencionais, como números e strings. As funções podem ser armazenadas em variáveis, em tabelas, podem ser passadas como argumentos e podem ser retornadas por outras funções.

Para demonstrar isso, também criaremos uma função “half”:

half = function(x)
    return x / 2
end

Então, agora temos duas variáveis, half e double, ambas contendo uma função como valor. E se quiséssemos criar uma função que alimentasse o número 4 em duas funções dadas e calculasse a soma de ambos os resultados?

Vamos querer chamar esta função como sumOfTwoFunctions(double, half, 4). Isso alimentará a função double, a função half e o inteiro 4 em nossa própria função.

function sumOfTwoFunctions(firstFunction, secondFunction, input)
    return firstFunction(input) + secondFunction(input)
end

A função sumOfTwoFunctions acima mostra como as funções podem ser passadas dentro de argumentos e acessadas por outro nome.

Verificando os tipos de argumentos #

Algumas funções funcionam apenas em um certo tipo de argumento:

function foo(tab)
    return tab.bar
end
--> returns nil if tab has no field bar, which is acceptable
--> returns 'attempt to index a number value' if tab is, for example, 3
--> which is unacceptable

function kungfoo(tab)
    if type(tab) ~= "table" then
        return nil, "take your useless " .. type(tab) .." somewhere else!"
    end

    return tab.bar
end

isso tem várias implicações:

print(kungfoo(20)) --> prints 'nil, take your useless number somewhere else!'

if kungfoo(20) then print "good" else print "bad" end --> prints bad

foo = kungfoo(20) or "bar" --> sets foo to "bar"

agora podemos chamar a função com qualquer parâmetro que quisermos, e não travará o programa.

-- if we actually WANT to abort execution on error, we can still do
result = assert(kungfoo({bar=20})) --> this will return 20
result = assert(kungfoo(20)) --> this will throw an error

Então, e se tivermos uma função que faz algo com uma instância de uma classe específica? Isso é difícil, porque classes e objetos geralmente são tabelas, então a função type retornará 'table'.

local Class = {data="important"}
local meta = {__index=Class}

function Class.new()
    return setmetatable({}, meta)
end
-- this is just a very basic implementation of an object class in lua

object = Class.new()
fake = {}

print(type(object)), print(type(fake)) --> prints 'table' twice

Solução: compare as metatabelas

-- continuation of previous code snippet
Class.is_instance(tab)
    return getmetatable(tab) == meta
end

Class.is_instance(object) --> returns true
Class.is_instance(fake) --> returns false
Class.is_instance(Class) --> returns false
Class.is_instance("a string") --> returns false, doesn't crash the program
Class.is_instance(nil) --> also returns false, doesn't crash either

Fechamentos #

do
    local tab = {1, 2, 3}
    function closure()
        for key, value in ipairs(tab) do
            print(key, "I can still see you")
        end
    end
    closure()
    --> 1 I can still see you
    --> 2 I can still see you
    --> 3 I can still see you
end

print(tab) --> nil
-- tab is out of scope

closure()
--> 1 I can still see you
--> 2 I can still see you
--> 3 I can still see you
-- the function can still see tab

exemplo de uso típico #

function new_adder(number)
    return function(input)
        return input + number
    end
end
add_3 = new_adder(3)
print(add_3(2)) --> prints 5

exemplo de uso mais avançado #

function base64.newDecoder(str) -- Decoder factory
    if #str ~= 64 then return nil, "string must be 64 characters long!" end

    local tab = {}
    local counter = 0
    for c in str:gmatch"." do
        tab[string.byte(c)] = counter
        counter = counter + 1
    end

    return function(str)
        local result = ""

        for abcd in str:gmatch"..?.?.?" do
            local a, b, c, d = string.byte(abcd,1,-1)
            a, b, c, d = tab[a], tab[b] or 0, tab[c] or 0, tab[d] or 0
            result = result .. (
                string.char( ((a<<2)+(b>>4))%256 ) ..
                string.char( ((b<<4)+(c>>2))%256 ) ..
                string.char( ((c<<6)+d)%256 )
            )
        end
        return result
    end
end

Parâmetros padrão #

function sayHello(name)
    print("Hello, " .. name .. "!")
end

Essa função é uma função simples e funciona bem. Mas o que aconteceria se apenas chamássemos sayHello()?

stdin:2: attempt to concatenate local 'name' (a nil value)
stack traceback:
    stdin:2: in function 'sayHello'
    stdin:1: in main chunk
    [C]: in ?

Isso não é exatamente ótimo. Existem duas maneiras de corrigir isso:

  1. Você retorna imediatamente da função:

    function sayHello(name)
  if not (type(name) == "string") then
    return nil, "argument #1: expected string, got " .. type(name)
  end -- Bail out if there's no name.
  -- in lua it is a convention to return nil followed by an error message on error

  print("Hello, " .. name .. "!") -- Normal behavior if name exists.
end

  2. Você define um parâmetro default.

    To do this, simply use this simple expression

    function sayHello(name) name = name or “Jack” – Jack is the default, – but if the parameter name is given, – name will be used instead print(“Hello, " .. name .. “!”) end

A expressão idiomática name = name ou "Jack" funciona porque or em curtos-circuitos Lua. Se o item do lado esquerdo de um or for diferente de nil ou false, o lado direito nunca será avaliado. Por outro lado, se sayHello for chamado sem parâmetro, então name será nil, e assim a string "Jack" será atribuída a name. (Observe que este idioma, portanto, não funcionará se o booleano false for um valor razoável para o parâmetro em questão.)

Vários resultados #

Funções em Lua podem retornar vários resultados.

Por exemplo:

function triple(x)
    return x, x, x
end

Ao chamar uma função, para salvar esses valores, você deve usar a seguinte sintaxe:

local a, b, c = triple(5)

O que resultará em a = b = c = 5 neste caso. Também é possível ignorar valores retornados usando a variável descartável _ no local desejado em uma lista de variáveis:

local a, _, c = triple(5)

Nesse caso, o segundo valor retornado será ignorado. Também é possível ignorar os valores de retorno não os atribuindo a nenhuma variável:

local a = triple(5)

A variável a receberá o primeiro valor de retorno e os dois restantes serão descartados.

Quando uma quantidade variável de resultados é retornada por uma função, pode-se armazená-los todos em uma tabela, executando a função dentro dela:

local results = {triple(5)}

Dessa forma, pode-se iterar sobre a tabela results para ver o que a função retornou.

Observação

Isso pode ser uma surpresa em alguns casos, por exemplo:

local t = {}
table.insert(t, string.gsub("  hi", "^%s*(.*)$", "%1")) --> bad argument #2 to 'insert' (number expected, got string)

Isso acontece porque string.gsub retorna 2 valores: a string fornecida, com as ocorrências do padrão substituídas, e o número total de correspondências que ocorreram.

Para resolver isso, use uma variável intermediária ou coloque () ao redor da chamada, assim:

table.insert(t, (string.gsub("  hi", "^%s*(.*)$", "%1"))) --> works. t = {"hi"}

Isso pega apenas o primeiro resultado da chamada e ignora o resto.

Argumentos nomeados #

local function A(name, age, hobby)
    print(name .. "is " .. age .. " years old and likes " .. hobby)
end
A("john", "eating", 23) --> prints 'john is eating years old and likes 23'
-- oops, seems we got the order of the arguments wrong...
-- this happens a lot, specially with long functions that take a lot of arguments
-- and where the order doesn't follow any particular logic

local function B(tab)
    print(tab.name .. "is " .. tab.age .. " years old and likes " .. tab.hobby)
end
local john = {name="john", hobby="golf", age="over 9000", comment="plays too much golf"}
B(john)
--> will print 'John is over 9000 years old and likes golf'
-- I also added a 'comment' argument just to show that excess arguments are ignored by the function

B({name = "tim"}) -- can also be written as
B{name = "tim"} -- to avoid cluttering the code
--> both will print 'tim is nil years old and likes nil'
-- remember to check for missing arguments and deal with them

function C(tab)
    if not tab.age then return nil, "age not defined" end
    tab.hobby = tab.hobby or "nothing"
    -- print stuff
end

-- note that if we later decide to do a 'person' class
-- we just need to make sure that this class has the three fields
-- age, hobby and name, and it will be compatible with these functions

-- example:
local john = ClassPerson.new("John", 20, "golf") -- some sort of constructor
john.address = "some place" -- modify the object
john:do_something("information") -- call some function of the object
C(john) -- this works because objects are *usually* implemented as tables
