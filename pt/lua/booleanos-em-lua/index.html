Booleanos, verdade e falsidade são diretos em Lua. Rever:

  1. Existe um tipo booleano com exatamente dois valores: true e false.
  2. Em um contexto condicional (if, elseif, while, until), um booleano não é necessário. Qualquer expressão pode ser usada.
  3. Em um contexto condicional, false e ​​nil contam como falsos, e todo o resto conta como verdadeiro.
  4. Embora 3 já implique isso: se você está vindo de outras linguagens, lembre-se que 0 e a string vazia contam como verdadeiros em contextos condicionais em Lua.

O tipo booleano #

Booleanos e outros valores #

Ao lidar com lua é importante diferenciar entre os valores booleanos true e false e ​​os valores que são avaliados como true ou false.

Existem apenas dois valores em lua que são avaliados como false: nil e false, enquanto todo o resto, incluindo o numérico 0, é avaliado como true.

Alguns exemplos do que isso significa:

if 0 then print("0 is true") end --> this will print "true"
if (2 == 3) then print("true") else print("false") end --> this prints "false"
if (2 == 3) == false then print("true") end --> this prints "true"
if (2 == 3) == nil then else print("false") end
--> prints false, because even if nil and false both evaluate to false,
--> they are still different things.

Operações lógicas #

Operadores lógicos em lua não necessariamente retornam valores booleanos:

and retornará o segundo valor se o primeiro valor for avaliado como verdadeiro;

or retorna o segundo valor se o primeiro valor for avaliado como falso;

Isso possibilita simular o operador ternário, assim como em outras linguagens:

local var = false and 20 or 30 --> returns 30
local var = true and 20 or 30 --> returns 20
-- in C: false ? 20 : 30

Isso também pode ser usado para inicializar tabelas se elas não existirem

tab = tab or {} -- if tab already exists, nothing happens

ou para evitar o uso de instruções if, tornando o código mais fácil de ler

print(debug and "there has been an error") -- prints "false" line if debug is false
debug and print("there has been an error") -- does nothing if debug is false
-- as you can see, the second way is preferable, because it does not output
-- anything if the condition is not met, but it is still possible.
-- also, note that the second expression returns false if debug is false,
-- and whatever print() returns if debug is true (in this case, print returns nil)

Verificando se as variáveis ​​estão definidas #

Pode-se também verificar facilmente se uma variável existe (se estiver definida), pois variáveis ​​inexistentes retornam nil, que é avaliada como false.

local tab_1, tab_2 = {}
if tab_1 then print("table 1 exists") end --> prints "table 1 exists"
if tab_2 then print("table 2 exists") end --> prints nothing

O único caso em que isso não se aplica é quando uma variável armazena o valor false, caso em que ela existe tecnicamente, mas ainda é avaliada como false. Por causa disso, é um projeto ruim criar funções que retornam false e ​​nil dependendo do estado ou entrada. Ainda podemos verificar se temos um nil ou um false:

if nil == nil then print("A nil is present") else print("A nil is not present") end
if false == nil then print("A nil is present") else print("A nil is not present") end
-- The output of these calls are:
-- A nil is present!
-- A nil is not present

Contextos condicionais #

Contextos condicionais em Lua (if, elseif, while, until) não requerem um booleano. Como muitas linguagens, qualquer valor Lua pode aparecer em uma condição. As regras de avaliação são simples:

  1. false e ​​nil contam como falsos.

  2. Todo o resto conta como verdade.

    if 1 then
  print("Numbers work.")
end
if 0 then
  print("Even 0 is true")
end

if "strings work" then
  print("Strings work.")
end
if "" then
  print("Even the empty string is true.")
end

Operadores lógicos #

Em Lua, os booleanos podem ser manipulados através de operadores lógicos. Esses operadores incluem not, and e or.

Em expressões simples, os resultados são bastante diretos:

print(not true) --> false
print(not false) --> true
print(true or false) --> true
print(false and true) --> false

Ordem de Precedência #

A ordem de precedência é semelhante aos operadores matemáticos unários -, * e +:

  • não
  • então e
  • então ou

Isso pode levar a expressões complexas:

print(true and false or not false and not true)
print( (true and false) or ((not false) and (not true)) )
    --> these are equivalent, and both evaluate to false

Avaliação de atalho #

Os operadores and e or só podem ser avaliados usando o primeiro operando, desde que o segundo seja desnecessário:

function a()
    print("a() was called")
    return true
end

function b()
    print("b() was called")
    return false
end

print(a() or b())
    --> a() was called
    --> true
    --  nothing else
print(b() and a())
    --> b() was called
    --> false
    --  nothing else
print(a() and b())
    --> a() was called
    --> b() was called
    --> false

Idiomático operador condicional #

Devido à precedência dos operadores lógicos, a capacidade de avaliação de atalhos e a avaliação de valores não-false e ​​não-nilcomotrue`, um operador condicional idiomático está disponível em Lua:

function a()
    print("a() was called")
    return false
end
function b()
    print("b() was called")
    return true
end
function c()
    print("c() was called")
    return 7
end

print(a() and b() or c())
    --> a() was called
    --> c() was called
    --> 7
    
print(b() and c() or a())
    --> b() was called
    --> c() was called
    --> 7

Além disso, devido à natureza da estrutura x e a ou b, a nunca será retornado se for avaliado como false, essa condicional sempre retornará b não importa o que x seja .

print(true and false or 1)  -- outputs 1

Tabelas verdade #

Os operadores lógicos em Lua não “retornam” um booleano, mas um de seus argumentos. Usando nil para false e números para true, veja como eles se comportam.

print(nil and nil)       -- nil
print(nil and 2)         -- nil
print(1 and nil)         -- nil
print(1 and 2)           -- 2

print(nil or nil)        -- nil
print(nil or 2)          -- 2
print(1 or nil)          -- 1
print(1 or 2)            -- 1

Como você pode ver, Lua sempre retornará o primeiro valor que torna a verificação fail ou succeed. Aqui estão as tabelas-verdade mostrando isso.

  x  |  y  || and            x  |  y  || or
------------------         ------------------
false|false||  x           false|false||  y   
false|true ||  x           false|true ||  y   
true |false||  y           true |false||  x   
true |true ||  y           true |true ||  x

Para quem precisar, aqui estão duas funções representando esses operadores lógicos.

function exampleAnd(value1, value2)
  if value1 then
    return value2
  end
  return value1
end

function exampleOr(value1, value2)
  if value1 then
    return value1
  end
  return value2
end

Emulando Operador Ternário com operadores lógicos ’e’ ‘ou’. #

Em lua, os operadores lógicos and e or retornam um dos operandos como resultado em vez de um resultado booleano. Como consequência, este mecanismo pode ser explorado para emular o comportamento do operador ternário apesar de lua não ter um operador ternário ‘real’ na linguagem.

Sintaxe #

condição e truthy_expr ou falsey_expr

Use na atribuição/inicialização de variável #

local drink = (fruit == "apple") and "apple juice" or "water"

Usar no construtor de tabela #

local menu =
{
  meal  = vegan and "carrot" or "steak",
  drink = vegan and "tea"    or "chicken soup"
}

Usar como argumento de função #

print(age > 18 and "beer" or "fruit punch")

Use na declaração de retorno #

function get_gradestring(student)
  return student.grade > 60 and "pass" or "fail"
end

Embargo #

Existem situações em que esse mecanismo não tem o comportamento desejado. Considere este caso

local var = true and false or "should not happen"

Em um operador ternário ‘real’, o valor esperado de ‘var’ é ‘falso’. Em lua, no entanto, a avaliação ‘and’ ‘falha’ porque o segundo operando é falso. Como resultado, var termina com should not acontecer.

Duas possíveis soluções alternativas para esse problema, refatore essa expressão para que o operando do meio não seja falso. por exemplo.

local var = not true and "should not happen" or false

ou alternativamente, use a construção clássica if then else.
