Recursos off-line do Firebase

  • Ativar persistência de disco (somente Android / iOS)
  • Mantendo os dados atualizados (somente Android/iOs)

Nesta página

  • Ativar persistência de disco (somente Android / iOS)
  • Mantendo os dados atualizados (somente Android/iOs)

Neste post, você encontrará as diferentes maneiras de implementar recursos offline ao usar o Firebase, informações sobre quando e por que pode ser uma boa ideia habilitar recursos offline e exemplos disso com a plataforma Android.

O que devo usar? Persistência de disco ou chamadas keepSynced?

Pela minha experiência, posso dizer que sempre depende de como seu aplicativo está funcionando e de como você gerencia as transações e o banco de dados do seu aplicativo. Se por exemplo você tem uma aplicação onde o usuário está apenas escrevendo e lendo dados mas não consegue deletar ou editar, usar DiskPersistence seria a escolha certa.

Além disso, DiskPersistence armazenará dados em cache, o que significa que seu aplicativo usará mais espaço nos dispositivos do usuário, o que talvez não seja a melhor ideia no seu caso.

Por outro lado, se seu aplicativo gerencia muitas transações complexas e seus dados são atualizados com muita frequência, possivelmente você deve evitar DiskPersistence e usar keepSynced nas referências que deseja manter atualizadas.

Por que?

DiskPersistence armazena os dados recuperados em local, o que às vezes pode causar muita dessincronização mostrando seus dados se você não os usar junto com ListenerValueEvents contínuo. Por exemplo:

  1. O usuário A escreve uma mensagem “Hello World” em seu aplicativo, que é recebida pelo usuário B
  2. O usuário B baixa a mensagem do usuário A em seu telefone e vê a mensagem “Hello World”
  3. O usuário A edita sua mensagem para “O Firebase é legal”.
  4. O usuário B continuará assistindo a mensagem “Hello World” mesmo que ele atualize os dados, pois a referência do instantâneo é a mesma quando o Firebase o filtra.

Para evitar isso, a melhor ideia é manter ouvintes contínuos nas referências que você deseja acompanhar o tempo todo.

Posso usar os dois juntos?

Claro que você pode, e na maioria dos aplicativos possivelmente é a melhor ideia evitar baixar muitos dados e dar ao usuário a possibilidade de trabalhar com seu aplicativo mesmo que ele não tenha conexão.

Se você não se importa em usar o espaço de cache no dispositivo do usuário, eu recomendo que você habilite diskPersistence no seu objeto FirebaseDatabase e ​​também adicione um sinalizador keepSync para cada referência que pode ter muitas vezes em um curto espaço-tempo ou você quer se manter atualizado o tempo todo.

Ativar persistência de disco (somente Android / iOS) #

Para habilitar a persistência do disco você deve habilitar o sinalizador persistenceEnabled no objeto FirebaseDatabaseInstance da sua aplicação:

Android

FirebaseDatabase.getInstance().setPersistenceEnabled(true);

iOS

Database.database().isPersistenceEnabled = true //Swift
[FIRDatabase database].persistenceEnabled = YES; //Objetive-C

Se você deseja desabilitar a persistência em algum momento do ciclo de vida do seu aplicativo, lembre-se de desativá-la da mesma maneira:

Android

FirebaseDatabase.getInstance().setPersistenceEnabled(false);

iOS

Database.database().isPersistenceEnabled = false //Swift
[FIRDatabase database].persistenceEnabled = NO; //Objetive-C

Mantendo os dados atualizados (somente Android/iOs) #

O Firebase sincroniza e armazena uma cópia local dos dados para listeners ativos quando usados ​​em dispositivos móveis. Além disso, você pode manter locais específicos sincronizados.

Android:

DatabaseReference workoutsRef = FirebaseDatabase.getInstance().getReference("workouts");
scoresRef.keepSynced(true);

iOs:

//Objetive-c
FIRDatabaseReference *scoresRef = [[FIRDatabase database] referenceWithPath:@"scores"];
[scoresRef keepSynced:YES];
//Swift
let scoresRef = Database.database().reference(withPath: "scores")
scoresRef.keepSynced(true)

O cliente Firebase baixa automaticamente os dados nesses locais e os mantém atualizados mesmo que a referência não tenha listeners ativos. Você desabilita a sincronização com a seguinte linha de código.

Android:

scoresRef.keepSynced(false);

iOS:

[scoresRef keepSynced:NO]; //Objetive-C
scoresRef.keepSynced(false) //Swift
