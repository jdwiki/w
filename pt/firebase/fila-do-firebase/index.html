Como usar a fila do Firebase como back-end para seu aplicativo #

O Firebase fornece back-end como serviço, como desenvolvedor de aplicativos, você não tem a opção de ter código de back-end.

Este exemplo mostra como usar a fila do Firebase, criar um back-end que operará no topo do banco de dados do Firebase e servirá como um back-end para seu aplicativo de front-end.

Antes de entrar no código vamos entender a arquitetura, como vai funcionar. Por brevidade, vamos supor que estamos usando o site como front-end e o servidor NodeJs como back-end

Pré-requisitos #

  1. Crie um aplicativo do Firebase usando sua conta do Google

  2. Adicione o Firebase à sua página da web. Use bower install firebase --save

  3. Crie uma conta de serviço usando sua nova conta do Firebase criada (Configurações-> Permissões -> Contas de serviço -> CRIAR CONTA DE SERVIÇO -> (especifique o nome e marque esta caixa de seleção “Fornecer uma nova chave privada”) -> salve o arquivo json, nós vai precisar disso mais tarde.

  4. Configure o servidor NodeJs que pode ser hospedado em seu ambiente preferido

  5. Crie o seguinte endpoint dentro de queue/specs

    “request_response”:

     {
     "error_state": "request_error_processing",
     "finished_state": "finished_state",
     "in_progress_state": "request_in_progress",
     "start_state": "request_started"
 }

  6. Dentro do servidor NodeJs, instale a versão do lado do servidor do firebase, npm install firebase --save e inicialize sua conta de serviço usando o arquivo json que obtivemos na etapa 3, fica assim

    firebase.initializeApp({ serviceAccount: ‘./your_file.json’, databaseURL: ‘get_from_firebase_account’ });

Arquitetura #

Aqui está todo o ciclo como funciona.

No lado do frontend, você fará essas etapas

  1. Usando o Firebase Web SDK, você está gravando suas solicitações diretamente no banco de dados do Firebase no endpoint ‘queue/tasks’, vamos chamar essa solicitação que você está enviando para o back-end.
  2. depois de inserir sua tarefa, você está registrando o listener no endpoint queue/tasks/{taskKey} que seria chamado quando o backend terminar de processar sua solicitação, escrevendo a resposta dentro da tarefa acima

No lado de back-end, você fará essas etapas

  1. Crie um servidor que escute infinitamente a ‘fila/tarefas’ do ponto de extremidade
  2. Processa suas tarefas e grava os dados de resposta dentro de queue/tasks/response
  3. Remova a tarefa

Primeiro, crie esta função auxiliar, que fornece uma maneira de lidar com retornos de chamada e promessas juntos

function createPromiseCallback() {
    var cb;
    var promise = new Promise(function (resolve, reject) {
        cb = function (err, data) {
            if (err) return reject(err);
            return resolve(data);
        };
    });
    cb.promise = promise;
    return cb;
}

No lado do frontend você terá essa função

function sendRequest(kind, params, cb) {

    cb = cb || createPromiseCallback();
    var requestObject = {
        kind: kind,
        params: params
    };
    var tasksRef = firebase.database().ref('queue/tasks');

    var requestKey = tasksRef.push().key;

    var requestRef = tasksRef.child(requestKey);

    function requestHandshake(snap) {
        if (snap && snap.exists() && (snap.val().response || snap.val()._state ===    config.firebase.task.finishState || snap.val()._error_details)) {
            var snapVal = snap.val();
            if (snapVal._error_details) {
                cb(snapVal._error_details.error);
            } else {
            cb(null, snapVal.response);
        }
        requestRef.off('value', requestHandshake);
    }
   }

   var bulkUpdate = {};
   bulkUpdate['queue/tasks/' + requestKey + '/request'] = requestObject;
   bulkUpdate['queue/tasks/' + requestKey + '/_state'] = config.firebase.task.startState;

   firebase.database().ref().update(bulkUpdate)
    .then(function (snap) {
        requestRef.on('value', requestHandshake);
    }).catch(function (err) {
        cb(err);
    });

   return cb.promise;
  }

você pode usar esta função como sendRequest('CreateHouseFacade', {houseName:'Test'}).

O parâmetro Kind é para backend, para saber qual método chamar para processar a solicitação. Params é para passar informações de parâmetros adicionais.

E aqui está o código de back-end

const database = firebase.database();
const queueRef = database.ref('queue');

const queueOptions = {
    'specId': 'request_response',
    'sanitize': false,
    'suppressStack': false,
    'numWorkers': 3
};

function removeTask(task) {
    var taskRef = queueRef.child(`tasks/${task._id}`);
    return taskRef.remove();
}

function processTask(data, progress, resolve, reject) {
    try {
        requestHandler(data.request).then(response => {
            data.response = response || null;
            return resolve(data);
        }).catch(err => {
            return reject(err);
        }).then(snap => {
            removeTask(data);
        });
    } catch (err) {
        reject(err).then(snap => removeTask(data));
    }
}

function requestHandler(request) {
    if (!request || !request.kind) throw new Error('Absent Request or Kind');
    var deferredResponse = requestHandlerFactory(request.kind, request.params);
    return deferredResponse;
}

function requestHandlerFactory(kind, params) {
    // It includes mapping all your backend services
    switch (kind) {
        case 'CreateHouseFacade': return myService(params)
        default: throw new Error(`Invalid kind ${kind} was specified`);
    }
}

A função myService contém seu código de lógica de negócios que vai realizar a requisição CreateHouseFacade.
