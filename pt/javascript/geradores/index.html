As funções do gerador (definidas pela palavra-chave function*) são executadas como corrotinas, gerando uma série de valores conforme são solicitados por meio de um iterador.

Sintaxe #

  • função* nome(parâmetros) { valor do rendimento; valor de retorno }
  • gerador = nome(argumentos)
  • { valor, feito } = gerador.next(valor)
  • { valor, feito } = gerador.retorno(valor)
  • generator.throw(erro)

As funções do gerador são um recurso introduzido como parte da especificação ES 2015 e não estão disponíveis em todos os navegadores. Eles também são totalmente suportados no Node.js a partir da v6.0. Para obter uma lista detalhada de compatibilidade do navegador, consulte a Documentação do MDN e para Node, consulte o node.green.

Funções do Gerador #

Uma função geradora é criada com uma declaração função*. Quando é chamado, seu corpo não é executado imediatamente. Em vez disso, ele retorna um objeto gerador, que pode ser usado para “percorrer” a execução da função.

Uma expressão yield dentro do corpo da função define um ponto no qual a execução pode ser suspensa e retomada.

function* nums() {
    console.log('starting');  // A
    yield 1;                  // B
    console.log('yielded 1'); // C
    yield 2;                  // D
    console.log('yielded 2'); // E
    yield 3;                  // F
    console.log('yielded 3'); // G
}
var generator = nums(); // Returns the iterator. No code in nums is executed

generator.next();  // Executes lines A,B returning { value: 1, done: false }
// console: "starting"
generator.next();  // Executes lines C,D returning { value: 2, done: false }
// console: "yielded 1"
generator.next();  // Executes lines E,F returning { value: 3, done: false }
// console: "yielded 2"
generator.next();  // Executes line G returning { value: undefined, done: true }
// console: "yielded 3"

Saída da iteração inicial #

generator = nums();
generator.next(); // Executes lines A,B returning { value: 1, done: false }
generator.next(); // Executes lines C,D returning { value: 2, done: false }
generator.return(3); // no code is executed returns { value: 3, done: true }
// any further calls will return done = true 
generator.next(); // no code executed returns { value: undefined, done: true }

Lançando um erro para a função do gerador #

function* nums() {
    try {
        yield 1;                  // A
        yield 2;                  // B
        yield 3;                  // C
    } catch (e) {
        console.log(e.message);    // D
    }
}

var generator = nums();

generator.next();  // Executes line A returning { value: 1, done: false }
generator.next();  // Executes line B returning { value: 2, done: false }
generator.throw(new Error("Error!!"));  // Executes line D returning { value: undefined, done: true}
// console: "Error!!"
generator.next();  // no code executed. returns { value: undefined, done: true }

Enviando valores para o gerador #

É possível enviar um valor para o gerador passando-o para o método next().

function* summer() {
    let sum = 0, value;
    while (true) {
        // receive sent value
        value = yield;
        if (value === null) break;

        // aggregate values
        sum += value;
    }
    return sum;
}
let generator = summer();

// proceed until the first "yield" expression, ignoring the "value" argument
generator.next();

// from this point on, the generator aggregates values until we send "null"
generator.next(1);
generator.next(10);
generator.next(100);

// close the generator and collect the result
let sum = generator.next(null).value;  // 111

Delegando a outro Gerador #

De dentro de uma função geradora, o controle pode ser delegado a outra função geradora usando yield*.

function* g1() {
  yield 2;
  yield 3;
  yield 4;
}

function* g2() {
  yield 1;
  yield* g1();
  yield 5;
}

var it = g2();

console.log(it.next()); // 1
console.log(it.next()); // 2
console.log(it.next()); // 3
console.log(it.next()); // 4
console.log(it.next()); // 5
console.log(it.next()); // undefined

Iteração #

Um gerador é iterável. Ele pode ser repetido com uma instrução for...of e usado em outras construções que dependem do protocolo de iteração.

function* range(n) {
    for (let i = 0; i < n; ++i) {
        yield i;
    }
}

// looping
for (let n of range(10)) {
    // n takes on the values 0, 1, ... 9
}

// spread operator
let nums = [...range(3)];  // [0, 1, 2]
let max = Math.max(...range(100));  // 99

Aqui está outro exemplo de gerador de uso para objeto iterável personalizado no ES6. Aqui é usada a função geradora anônima função *.

let user = {
  name: "sam", totalReplies: 17, isBlocked: false
};

user[Symbol.iterator] = function *(){

  let properties = Object.keys(this);
  let count = 0;
  let isDone = false;

  for(let p of properties){
    yield this[p];
  }
};

for(let p of user){
  console.log( p );
}

Fluxo assíncrono com geradores #

Geradores são funções que podem pausar e depois retomar a execução. Isso permite emular funções assíncronas usando bibliotecas externas, principalmente q ou co. Basicamente, ele permite escrever funções que aguardam resultados assíncronos para continuar:

function someAsyncResult() {
    return Promise.resolve('newValue')
}

q.spawn(function * () {
    var result = yield someAsyncResult()
    console.log(result) // 'newValue'
})

Isso permite escrever código assíncrono como se fosse síncrono. Além disso, tente capturar o trabalho em vários blocos assíncronos. Se a promessa for rejeitada, o erro será detectado pela próxima captura:

function asyncError() {
    return new Promise(function (resolve, reject) {
        setTimeout(function () {
            reject(new Error('Something went wrong'))
        }, 100)
    })
}

q.spawn(function * () {
    try {
        var result = yield asyncError()
    } catch (e) {
        console.error(e) // Something went wrong
    }
})

Usar co funcionaria exatamente da mesma forma, mas com co(function * (){...}) em vez de q.spawn

Interface do Iterador-Observador #

Um gerador é uma combinação de duas coisas - um iterador e um observador.

Iterador #

Um iterador é algo que quando invocado retorna um iterável. Um iterável é algo que você pode iterar. A partir do ES6/ES2015, todas as coleções (Array, Map, Set, WeakMap, WeakSet) estão em conformidade com o contrato Iterable.

Um gerador (iterador) é um produtor. Na iteração o consumidor PULL é o valor do produtor.

Exemplo:

function *gen() { yield 5; yield 6; }
let a = gen();

Sempre que você chama a.next(), você está essencialmente puxando o valor do Iterator e pause a execução em yield. Na próxima vez que você chamar a.next(), a execução será retomada a partir do estado de pausa anterior.

Observador #

Um gerador também é um observador usando o qual você pode enviar alguns valores de volta ao gerador.

function *gen() {
  document.write('<br>observer:', yield 1);
}
var a = gen();
var i = a.next();
while(!i.done) {
  document.write('<br>iterator:', i.value);
  i = a.next(100);
}

Aqui você pode ver que yield 1 é usado como uma expressão que avalia algum valor. O valor avaliado é o valor enviado como argumento para a chamada da função a.next.

Assim, pela primeira vez i.value será o primeiro valor gerado (1), e ao continuar a iteração para o próximo estado, enviamos um valor de volta ao gerador usando a.next(100) .

Fazendo assíncrono com geradores #

Os geradores são amplamente usados ​​com a função spawn (de taskJS ou co), onde a função recebe um gerador e nos permite escrever código assíncrono de maneira síncrona. Isso NÃO significa que o código assíncrono é convertido em código de sincronização/executado de forma síncrona. Isso significa que podemos escrever código que se parece com sync mas internamente ainda é async.

A sincronização está BLOQUEANDO; Assíncrono está AGUARDANDO. Escrever código que bloqueia é fácil. Ao puxar, o valor aparece na posição de atribuição. Ao PUSHing, o valor aparece na posição do argumento do retorno de chamada.

Quando você usa iteradores, você PULL o valor do produtor. Quando você usa callbacks, o produtor PUSH é o valor para a posição do argumento do callback.

var i = a.next() // PULL
dosomething(..., v => {...}) // PUSH

Aqui, você puxa o valor de a.next() e no segundo, v => {...} é o retorno de chamada e um valor é PUSHed na posição do argumento v do retorno de chamada função.

Usando este mecanismo pull-push, podemos escrever programação assíncrona como esta,

let delay = t => new Promise(r => setTimeout(r, t));
spawn(function*() {
  // wait for 100 ms and send 1
  let x = yield delay(100).then(() => 1);
  console.log(x); // 1

   // wait for 100 ms and send 2
  let y = yield delay(100).then(() => 2);
  console.log(y); // 2
});

Então, olhando para o código acima, estamos escrevendo um código assíncrono que parece estar ‘bloqueando’ (as instruções yield esperam 100ms e então continuam a execução), mas na verdade está ’esperando’. A propriedade pause e ​​resume do gerador nos permite fazer esse truque incrível.

Como funciona ? #

A função spawn usa promessa de rendimento para PUXAR o estado da promessa do gerador, espera até que a promessa seja resolvida e envia o valor resolvido de volta ao gerador para que ele possa consumi-lo.

Use agora #

Assim, com os geradores e a função spawn, você pode limpar todo o seu código assíncrono no NodeJS para parecer síncrono. Isso facilitará a depuração. Além disso, o código ficará limpo.

Este recurso está chegando a versões futuras do JavaScript - como async...await. Mas você pode usá-los hoje no ES2015/ES6 usando a função spawn definida nas bibliotecas - taskjs, co ou bluebird
