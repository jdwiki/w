async e await são construídos em cima de promessas e geradores para expressar ações assíncronas inline. Isso torna o código assíncrono ou de retorno de chamada muito mais fácil de manter.

Funções com a palavra-chave async retornam uma Promise e ​​podem ser chamadas com essa sintaxe.

Dentro de uma função assíncrona a palavra-chave await pode ser aplicada a qualquer Promise, e fará com que todo o corpo da função após a await seja executado após a promessa ser resolvida.

Sintaxe #

  • função assíncrona foo() {    …
       await asyncCall()
    }
  • função assíncrona() { … }
  • async() => { … }
  • (assíncrono () => {    const data = await asyncCall()
       console.log(data) })()

As funções assíncronas são um açúcar sintático sobre promessas e geradores. Eles ajudam você a tornar seu código mais legível, fácil de manter, mais fácil de detectar erros e com menos níveis de recuo.

Introdução #

Uma função definida como async é uma função que pode executar ações assíncronas, mas ainda parecer síncrona. A maneira como isso é feito é usando a palavra-chave await para adiar a função enquanto ela aguarda uma Promise ser resolvida ou rejeitada.

Observação: as funções assíncronas são uma proposta do Estágio 4 (“Concluído”) em vias de serem incluídas no Padrão ECMAScript 2017.

Por exemplo, usando a [Fetch API] (https://developer.mozilla.org/en/docs/Web/API/Fetch_API) baseada em promessas:

async function getJSON(url) {
    try {
        const response = await fetch(url);
        return await response.json();
    }
    catch (err) {
        // Rejections in the promise will get thrown here
        console.error(err.message);
    }
}

Uma função assíncrona sempre retorna uma Promise, então você pode usá-la em outras funções assíncronas.

Estilo da função de seta #

const getJSON = async url => {
    const response = await fetch(url);
    return await response.json();
}

Aguardar e precedência do operador #

Você deve manter a precedência do operador em mente ao usar a palavra-chave await.

Imagine que temos uma função assíncrona que chama outra função assíncrona, getUnicorn() que retorna uma Promise que resolve para uma instância da classe Unicorn. Agora queremos obter o tamanho do unicórnio usando o método getSize() dessa classe.

Observe o seguinte código:

async function myAsyncFunction() {
    await getUnicorn().getSize();
}

À primeira vista, parece válido, mas não é. Devido à precedência do operador, é equivalente ao seguinte:

async function myAsyncFunction() {
    await (getUnicorn().getSize());
}

Aqui tentamos chamar o método getSize() do objeto Promise, que não é o que queremos.

Em vez disso, devemos usar colchetes para denotar que primeiro queremos esperar pelo unicórnio e, em seguida, chamar o método getSize() do resultado:

async function asyncFunction() {
    (await getUnicorn()).getSize();
}

É claro. a versão anterior poderia ser válida em alguns casos, por exemplo, se a função getUnicorn() fosse síncrona, mas o método getSize() fosse assíncrono.

Funções assíncronas comparadas com Promises #

As funções async não substituem o tipo Promise; eles adicionam palavras-chave de linguagem que tornam as promessas mais fáceis de serem chamadas. Eles são intercambiáveis:

async function doAsyncThing() { ... }

function doPromiseThing(input) { return new Promise((r, x) => ...); }

// Call with promise syntax
doAsyncThing()
    .then(a => doPromiseThing(a))
    .then(b => ...)
    .catch(ex => ...);

// Call with await syntax
try {
    const a = await doAsyncThing();
    const b = await doPromiseThing(a);
    ...
}
catch(ex) { ... }

Qualquer função que use cadeias de promessas pode ser reescrita usando await:

function newUnicorn() {
  return fetch('unicorn.json')                     // fetch unicorn.json from server
  .then(responseCurrent => responseCurrent.json()) // parse the response as JSON
  .then(unicorn =>
    fetch('new/unicorn', {                         // send a request to 'new/unicorn' 
        method: 'post',                            // using the POST method
        body: JSON.stringify({unicorn})            // pass the unicorn to the request body
    })
  )
  .then(responseNew => responseNew.json())
  .then(json => json.success)                      // return success property of response
  .catch(err => console.log('Error creating unicorn:', err));
 }

A função pode ser reescrita usando async / await da seguinte forma:

async function newUnicorn() {
  try {
    const responseCurrent = await fetch('unicorn.json'); // fetch unicorn.json from server
    const unicorn = await responseCurrent.json();        // parse the response as JSON
    const responseNew = await fetch('new/unicorn', {     // send a request to 'new/unicorn'
      method: 'post',                                    // using the POST method
      body: JSON.stringify({unicorn})                    // pass the unicorn to the request body
    });
    const json = await responseNew.json();
    return json.success                                  // return success property of response
  } catch (err) {
    console.log('Error creating unicorn:', err);
  }
}

Esta variante async de newUnicorn() parece retornar uma Promise, mas na verdade havia várias palavras-chave await. Cada um retornou uma Promise, então realmente tínhamos uma coleção de promessas em vez de uma cadeia.

Na verdade, podemos pensar nele como um gerador de função*, com cada await sendo uma rendimento nova promessa. No entanto, os resultados de cada promessa são necessários para a próxima continuar a função. É por isso que a palavra-chave adicional async é necessária na função (assim como a palavra-chave await ao chamar as promessas), pois ela diz ao Javascript para criar automaticamente um observador para esta iteração. A Promise retornada pela função assíncrona newUnicorn() é resolvida quando esta iteração é concluída.

Praticamente, você não precisa considerar isso; await oculta a promessa e async oculta a iteração do gerador.

Você pode chamar funções async como se fossem promessas, e await qualquer promessa ou qualquer função async. Você não precisa aguardar uma função assíncrona, assim como você pode executar uma promessa sem um .then().

Você também pode usar um async IIFE se quiser executar esse código imediatamente:

(async () => {
  await makeCoffee()
  console.log('coffee is ready!')
})()

Looping com espera assíncrona #

Ao usar o async await em loops, você pode encontrar alguns desses problemas.

Se você apenas tentar usar await dentro de forEach, isso lançará um erro de token inesperado.

(assíncrono() => {
dados = [1, 2, 3, 4, 5];
data.forEach(e => {
   const i = await somePromiseFn(e);
   console.log(i);
 });
})();

Isso vem do fato de que você erroneamente viu a função de seta como um bloco. O await estará no contexto da função de retorno de chamada, que não é async. O interpretador nos protege de cometer o erro acima, mas se você adicionar async ao retorno de chamada forEach nenhum erro será lançado. Você pode pensar que isso resolve o problema, mas não funcionará como esperado.

Exemplo:

(assíncrono() => {
dados = [1, 2, 3, 4, 5];
data.forEach(async(e) => {
    const i = await somePromiseFn(e);
    console.log(i);
  });
console.log('isto será impresso primeiro');
})();

Isso acontece porque a função assíncrona de retorno de chamada só pode pausar a si mesma, não a função assíncrona pai.

Você poderia escrever uma função asyncForEach que retorna uma promessa e então você poderia algo como await asyncForEach(async (e) => await somePromiseFn(e), data ) Basicamente você retorna uma promessa que resolve quando todos os retornos de chamada são aguardados e concluídos. Mas existem maneiras melhores de fazer isso, e isso é usar apenas um loop.

Você pode usar um loop for-of ou um loop for/while, não importa qual você escolha.

(assíncrono() => {
dados = [1, 2, 3, 4, 5];
para (seja e de dados) {
    const i = await somePromiseFn(e);
    console.log(i);
  }
console.log('isto será impresso por último');
})();

Mas há outro problema. Esta solução aguardará a conclusão de cada chamada para somePromiseFn antes de iterar na próxima. Isso é ótimo se você realmente quiser que suas invocações somePromiseFn sejam executadas em ordem, mas se você quiser que elas sejam executadas simultaneamente, você precisará await em Promise.all.

(assíncrono() => {
dados = [1, 2, 3, 4, 5];
const p = await Promise.all(data.map(async(e) => await somePromiseFn(e)));
console.log(...p);
})();

Promise.all recebe um array de promessas como seu único parâmetro e retorna uma promessa. Quando todas as promessas na matriz são resolvidas, a promessa retornada também é resolvida. Nós ’esperamos’ por essa promessa e quando ela for resolvida todos os nossos valores estarão disponíveis.

Os exemplos acima são totalmente executáveis. A função somePromiseFn pode ser feita como uma função de eco assíncrona com um tempo limite. Você pode experimentar os exemplos no babel-repl com pelo menos a predefinição stage-3 e observar a saída.

function somePromiseFn(n) {
return new Promise((res, rej) => {
   setTimeout(() => res(n), 250);
 });
}

Menos recuo #

Com promessas:

function doTheThing() {
    return doOneThing()
        .then(doAnother)
        .then(doSomeMore)
        .catch(handleErrors)
}

Com funções assíncronas:

async function doTheThing() {
    try {
        const one = await doOneThing();
        const another = await doAnother(one);
        return await doSomeMore(another);
    } catch (err) {
        handleErrors(err);
    }
}

Observe como o retorno está na parte inferior, e não na parte superior, e você usa a mecânica nativa de tratamento de erros da linguagem (try/catch).

Operações assíncronas (paralelas) simultâneas #

Muitas vezes você desejará realizar operações assíncronas em paralelo. Existe uma sintaxe direta que suporta isso na proposta async/await, mas como await esperará por uma promessa, você pode agrupar várias promessas em Promise.all para esperar por elas:

// Not in parallel

async function getFriendPosts(user) {
    friendIds = await db.get("friends", {user}, {id: 1});
    friendPosts = [];
    for (let id in friendIds) {
        friendPosts = friendPosts.concat( await db.get("posts", {user: id}) );
    }
    // etc.
}

Isso fará cada consulta para obter as postagens de cada amigo em série, mas elas podem ser feitas simultaneamente:

// In parallel

async function getFriendPosts(user) {
    friendIds = await.db.get("friends", {user}, {id: 1});
    friendPosts = await Promise.all( friendIds.map(id => 
      db.get("posts", {user: id})
    );
    // etc.
}

Isso fará um loop sobre a lista de IDs para criar uma matriz de promessas. await irá esperar que all promessas sejam completadas. O Promise.all os combina em uma única promessa, mas eles são feitos em paralelo.
