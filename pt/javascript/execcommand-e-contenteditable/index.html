Ouvindo alterações de conteúdo editável #

Eventos que funcionam com a maioria dos elementos de formulário (por exemplo, change, keydown, keyup, keypress) não funcionam com contenteditable.

Em vez disso, você pode ouvir as alterações do conteúdo contenteditable com o evento input. Assumindo que contenteditableHtmlElement é um objeto JS DOM que é contenteditable:

contenteditableHtmlElement.addEventListener("input", function() {
    console.log("contenteditable element changed");
});

Começando #

O atributo HTML contenteditable fornece uma maneira simples de transformar um elemento HTML em uma área editável pelo usuário

<div contenteditable>You can <b>edit</b> me!</div>

Edição nativa de Rich-Text

Usando JavaScript e execCommandW3C, você também pode passar mais recursos de edição para o contenteditable atualmente focado elemento (especificamente na posição ou seleção do cursor).

O método da função execCommand aceita 3 argumentos

document.execCommand(commandId, showUI, value)
  • String commandId. da lista de commandIds disponíveis (veja: Parâmetros→commandId)
  • showUI Booleano (não implementado. Use falso)
  • value String Se um comando espera uma String value relacionada ao comando, caso contrário "". (consulte: Parâmetros→valor)

Exemplo usando o comando "bold" e "formatBlock" (onde um valor é esperado):

document.execCommand("bold", false, "");          // Make selected text bold
document.execCommand("formatBlock", false, "H2"); // Make selected text Block-level <h2>

Exemplo de início rápido:

<button data-edit="bold"><b>B</b></button>
<button data-edit="italic"><i>I</i></button>
<button data-edit="formatBlock:p">P</button>
<button data-edit="formatBlock:H1">H1</button>
<button data-edit="insertUnorderedList">UL</button>
<button data-edit="justifyLeft">&#8676;</button>
<button data-edit="justifyRight">&#8677;</button>
<button data-edit="removeFormat">&times;</button>

<div contenteditable><p>Edit me!</p></div>

<script>
[].forEach.call(document.querySelectorAll("[data-edit]"), function(btn) {
  btn.addEventListener("click", edit, false);
});

function edit(event) {
  event.preventDefault();
  var cmd_val = this.dataset.edit.split(":");
  document.execCommand(cmd_val[0], false, cmd_val[1]);
}
<script>

demonstração do jsFiddle Exemplo de editor de Rich-Text básico (navegadores modernos)

Pensamentos finais Mesmo estando presente há muito tempo (IE6), as implementações e comportamentos do execCommand variam de navegador para navegador, tornando a “construção de um editor WYSIWYG compatível com vários navegadores” uma tarefa difícil para qualquer desenvolvedor JavaScript experiente. Mesmo que ainda não esteja totalmente padronizado, você pode esperar resultados bastante decentes nos navegadores mais recentes, como Chrome, Firefox, Edge. Se você precisar de um suporte melhor para outros navegadores e mais recursos como edição de HTMLTable etc. uma regra prática é procurar um editor já existente e robusto de Rich-Text.

Formatação #

Os usuários podem adicionar formatação a documentos ou elementos contenteditable usando os recursos do navegador, como atalhos de teclado comuns para formatação (Ctrl-B para negrito, Ctrl-I para itálico, etc.) ou arrastando e soltando imagens, links ou marcações da área de transferência.

Além disso, os desenvolvedores podem usar JavaScript para aplicar formatação à seleção atual (texto realçado).

document.execCommand('bold', false, null); // toggles bold formatting
document.execCommand('italic', false, null); // toggles italic formatting
document.execCommand('underline', false, null); // toggles underline

Copie para a área de transferência da área de texto usando execCommand(“copy”) #

Exemplo:

<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <title></title>
</head>
<body>
    <textarea id="content"></textarea>
    <input type="button" id="copyID" value="Copy" />
    <script type="text/javascript">
        var button = document.getElementById("copyID"),
            input = document.getElementById("content");

        button.addEventListener("click", function(event) {
            event.preventDefault();
            input.select();
            document.execCommand("copy");
        });
    </script>
</body>
</html>

document.execCommand("copy") copia a seleção atual para a área de transferência
