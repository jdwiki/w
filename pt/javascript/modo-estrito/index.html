Sintaxe #

  • ‘usar estrito’;
  • “usar estrito”;
  • `usar estrito`;

O modo estrito é uma opção adicionada no ECMAScript 5 para habilitar alguns aprimoramentos incompatíveis com versões anteriores. As mudanças de comportamento no código do “modo estrito” incluem:

  • Atribuir a variáveis ​​indefinidas gera um erro ao invés de definir novas variáveis ​​globais;
  • Atribuir ou deletar propriedades não graváveis ​​(como window.undefined) gera um erro ao invés de executar silenciosamente;
  • Sintaxe octal herdada (ex. 0777) não é suportada;
  • A instrução with não é suportada;
  • eval não pode criar variáveis ​​no escopo circundante;
  • As propriedades .caller e .arguments das funções não são suportadas;
  • A lista de parâmetros de uma função não pode ter duplicatas;
  • window não é mais usado automaticamente como o valor de this.

NOTA: - O modo ‘strict’ NÃO está ativado por padrão, pois se uma página usa JavaScript que depende de recursos do modo não - estrito, esse código será interrompido. Assim, ele deve ser ativado pelo próprio programador.

Para scripts inteiros #

O modo estrito pode ser aplicado em scripts inteiros colocando a instrução "use strict"; antes de qualquer outra instrução.

"use strict";
// strict mode now applies for the rest of the script

O modo estrito só é habilitado em scripts onde você define "use strict". Você pode combinar scripts com e sem modo estrito, pois o estado estrito não é compartilhado entre scripts diferentes.

Observação: Todos os códigos escritos dentro do ES2015+ módulos e classes são restritos por padrão.

Para funções #

O modo estrito também pode ser aplicado a funções únicas, anexando a instrução "use strict"; no início da declaração da função.

function strict() {
   "use strict";
   // strict mode now applies to the rest of this function
   var innerFunction = function () {
     // strict mode also applies here
   };
}

function notStrict() {
  // but not here
}

O modo estrito também se aplicará a qualquer função com escopo interno.

Alterações nas propriedades #

O modo estrito também impede que você exclua propriedades que não podem ser excluídas.

"use strict";
delete Object.prototype; // throws a TypeError

A instrução acima seria simplesmente ignorada se você não usar o modo estrito, mas agora você sabe por que ela não é executada conforme o esperado.

Também impede que você estenda uma propriedade não extensível.

var myObject = {name: "My Name"}
Object.preventExtensions(myObject);

function setAge() {
    myObject.age = 25;   // No errors
}

function setAge() {
    "use strict";
    myObject.age = 25;  // TypeError: can't define property "age": Object is not extensible
}

Alterações nas propriedades globais #

Em um escopo de modo não estrito, quando uma variável é atribuída sem ser inicializada com a palavra-chave var, const ou let, ela é declarada automaticamente no escopo global:

a = 12;
console.log(a); // 12

No modo estrito, no entanto, qualquer acesso a uma variável não declarada lançará um erro de referência:

"use strict";
a = 12; // ReferenceError: a is not defined
console.log(a);

Isso é útil porque o JavaScript tem vários eventos possíveis que às vezes são inesperados. No modo não estrito, esses eventos geralmente levam os desenvolvedores a acreditar que são bugs ou comportamentos inesperados, portanto, ao habilitar o modo estrito, quaisquer erros lançados os obrigam a saber exatamente o que está sendo feito.

"use strict";
                       // Assuming a global variable mistypedVariable exists
mistypedVaraible = 17; // this line throws a ReferenceError due to the 
                       // misspelling of variable

Este código em modo estrito exibe um cenário possível: ele lança um erro de referência que aponta para o número da linha da atribuição, permitindo que o desenvolvedor detecte imediatamente o erro de tipo no nome da variável.

No modo não-estrito, além do fato de que nenhum erro é lançado e a atribuição é feita com sucesso, o mistypedVaraible será declarado automaticamente no escopo global como uma variável global. Isso implica que o desenvolvedor precisa procurar manualmente essa atribuição específica no código.

Além disso, ao forçar a declaração de variáveis, o desenvolvedor não pode declarar acidentalmente variáveis ​​globais dentro de funções. No modo não estrito:

function foo() { 
   a = "bar"; // variable is automatically declared in the global scope
}
foo();
console.log(a); // >> bar

No modo estrito, é necessário declarar explicitamente a variável:

function strict_scope() { 
   "use strict";
   var a = "bar"; // variable is local
}
strict_scope();
console.log(a); // >> "ReferenceError: a is not defined"

A variável também pode ser declarada fora e depois de uma função, permitindo que ela seja usada, por exemplo, no escopo global:

function strict_scope() { 
   "use strict";
   a = "bar"; // variable is global
}
var a;
strict_scope();
console.log(a); // >> bar

Parâmetros duplicados #

O modo estrito não permite que você use nomes de parâmetros de função duplicados.

function foo(bar, bar) {}  // No error. bar is set to the final argument when called

"use strict";
function foo(bar, bar) {}; // SyntaxError: duplicate formal argument bar

Escopo da função no modo estrito #

No Strict Mode, as funções declaradas em um bloco local são inacessíveis fora do bloco.

"use strict";
{
  f(); // 'hi'
  function f() {console.log('hi');}
}
f(); // ReferenceError: f is not defined

Em termos de escopo, as declarações de função no Strict Mode têm o mesmo tipo de vinculação que let ou const.

Comportamento da lista de argumentos de uma função #

O objeto arguments se comporta de maneira diferente nos modos strict e nonstrict. No modo não estrito, o objeto argument refletirá as alterações no valor dos parâmetros que estão presentes, no entanto, no modo estrito, quaisquer alterações no valor do parâmetro não serão refletidas no argumento objeto.

function add(a, b){
    console.log(arguments[0], arguments[1]); // Prints : 1,2

    a = 5, b = 10;

    console.log(arguments[0], arguments[1]); // Prints : 5,10
}

add(1, 2);

Para o código acima, o objeto arguments é alterado quando alteramos o valor dos parâmetros. No entanto, para o modo estrito, o mesmo não será refletido.

function add(a, b) {
    'use strict';

    console.log(arguments[0], arguments[1]); // Prints : 1,2

    a = 5, b = 10;

    console.log(arguments[0], arguments[1]); // Prints : 1,2
}

Vale a pena notar que, se qualquer um dos parâmetros for undefined, e tentarmos alterar o valor do parâmetro no modo strict-mode ou non-strict, o objeto arguments permanece inalterado.

Modo estrito

function add(a, b) {
    'use strict';

    console.log(arguments[0], arguments[1]); // undefined,undefined 
                                             // 1,undefined
    a = 5, b = 10;

    console.log(arguments[0], arguments[1]); // undefined,undefined
                                             // 1, undefined
}
add();
// undefined,undefined 
// undefined,undefined

add(1)
// 1, undefined
// 1, undefined

Modo Não Estrito

function add(a,b) {

    console.log(arguments[0],arguments[1]);

    a = 5, b = 10;

    console.log(arguments[0],arguments[1]);
}
add();
// undefined,undefined 
// undefined,undefined

add(1);
// 1, undefined
// 5, undefined

Listas de parâmetros não simples #

function a(x = 5) {
  "use strict";
}

é JavaScript inválido e lançará um SyntaxError porque você não pode usar a diretiva "use strict" em uma função com lista de parâmetros não simples como a acima - atribuição padrão x = 5

Parâmetros não simples incluem -

  • Atribuição padrão
function a(x = 1) {
  "use strict";
}
  • Desestruturando
function a({ x }) {
  "use strict";
}
  • Parâmetros de descanso
function a(...args) {
  "use strict";
}
