O que é Programação Funcional?
Programação Funcional ou FP é um paradigma de programação baseado em dois conceitos principais imutabilidade e ausência de estado. O objetivo por trás do FP é tornar seu código mais legível, reutilizável, e portátil.

O que é JavaScript Funcional
Houve um debate para chamar JavaScript de linguagem funcional ou não. No entanto, podemos absolutamente usar JavaScript como funcional devido à sua natureza:

  • Tem funções puras
  • Tem Funções de primeira classe
  • Tem Função de ordem superior
  • Suporta Imutabilidade
  • Possui Fechamentos
  • Recursion, e List Transforation Methods (Arrays) como map,reduce,filter..etc

Os Exemplos devem cobrir cada conceito em detalhes, e os links fornecidos aqui são apenas para referência e devem ser removidos assim que o conceito for ilustrado.

Funções de ordem superior #

Em geral, funções que operam em outras funções, seja tomando-as como argumentos ou retornando-as (ou ambos), são chamadas de funções de ordem superior.

Uma função de ordem superior é uma função que pode receber outra função como argumento. Você está usando funções de ordem superior ao passar retornos de chamada.

function iAmCallbackFunction() {
    console.log("callback has been invoked");
}

function iAmJustFunction(callbackFn) {
    // do some stuff ...

    // invoke the callback function.
    callbackFn();
}

// invoke your higher-order function with a callback function.
iAmJustFunction(iAmCallbackFunction);

Uma função de ordem superior também é uma função que retorna outra função como resultado.

function iAmJustFunction() {
    // do some stuff ...

    // return a function.
    return function iAmReturnedFunction() {
        console.log("returned function has been invoked");
    }
}

// invoke your higher-order function and its returned function.
iAmJustFunction()();

Mônada de Identidade #

Este é um exemplo de implementação da mônada de identidade em JavaScript, e pode servir como ponto de partida para a criação de outras mônadas.

Baseado na conferência de Douglas Crockford sobre mônadas e gônadas

Usar essa abordagem para reutilizar suas funções será mais fácil devido à flexibilidade que essa mônada oferece e aos pesadelos de composição:

f(g(h(i(j(k(value), j1), i2), h1, h2), g1, g2), f1, f2)

legível, agradável e limpo:

identityMonad(value)
    .bind(k)
    .bind(j, j1, j2)
    .bind(i, i2)
    .bind(h, h1, h2)
    .bind(g, g1, g2)
    .bind(f, f1, f2);

function identityMonad(value) {
    var monad = Object.create(null);
    
    // func should return a monad
    monad.bind = function (func, ...args) {
        return func(value, ...args);
    };

    // whatever func does, we get our monad back
    monad.call = function (func, ...args) {
        func(value, ...args);

        return identityMonad(value);
    };
    
    // func doesn't have to know anything about monads
    monad.apply = function (func, ...args) {
        return identityMonad(func(value, ...args));
    };

    // Get the value wrapped in this monad
    monad.value = function () {
        return value;
    };
    
    return monad;
};

Funciona com valores primitivos

var value = 'foo',
    f = x => x + ' changed',
    g = x => x + ' again';

identityMonad(value)
    .apply(f)
    .apply(g)
    .bind(alert); // Alerts 'foo changed again'

E também com objetos

var value = { foo: 'foo' },
    f = x => identityMonad(Object.assign(x, { foo: 'bar' })),
    g = x => Object.assign(x, { bar: 'foo' }),
    h = x => console.log('foo: ' + x.foo + ', bar: ' + x.bar);

identityMonad(value)
    .bind(f)
    .apply(g)
    .bind(h); // Logs 'foo: bar, bar: foo'

Vamos tentar tudo:

var add = (x, ...args) => x + args.reduce((r, n) => r + n, 0),
    multiply = (x, ...args) => x * args.reduce((r, n) => r * n, 1),
    divideMonad = (x, ...args) => identityMonad(x / multiply(...args)),
    log = x => console.log(x),
    substract = (x, ...args) => x - add(...args);

identityMonad(100)
    .apply(add, 10, 29, 13)
    .apply(multiply, 2)
    .bind(divideMonad, 2)
    .apply(substract, 67, 34)
    .apply(multiply, 1239)
    .bind(divideMonad, 20, 54, 2)
    .apply(Math.round)
    .call(log); // Logs 29

Funções puras #

Um princípio básico da programação funcional é que ela evita alterar o estado da aplicação (ausência de estado) e variáveis ​​fora de seu escopo (imutabilidade).

Funções puras são funções que:

  • com uma determinada entrada, sempre retorna a mesma saída
  • eles não dependem de nenhuma variável fora de seu escopo
  • eles não modificam o estado do aplicativo (sem efeitos colaterais)

Vejamos alguns exemplos:

Funções puras não devem alterar nenhuma variável fora de seu escopo

Função impura

let obj = { a: 0 }

const impure = (input) => {
  // Modifies input.a
  input.a = input.a + 1;
  return input.a;
}

let b = impure(obj)
console.log(obj) // Logs { "a": 1 }
console.log(b) // Logs 1

A função alterou o valor obj.a que está fora de seu escopo.

Função pura

let obj = { a: 0 }

const pure = (input) => {
  // Does not modify obj
  let output = input.a + 1;
  return output;
}

let b = pure(obj)
console.log(obj) // Logs { "a": 0 }
console.log(b) // Logs 1

A função não alterou os valores do objeto obj

Funções puras não devem depender de variáveis ​​fora de seu escopo

Função impura

let a = 1;

let impure = (input) => {
  // Multiply with variable outside function scope
  let output = input * a;
  return output;
}

console.log(impure(2)) // Logs 2
a++; // a becomes equal to 2
console.log(impure(2)) // Logs 4

Esta função impura depende da variável a que é definida fora de seu escopo. Então, se a for modificado, o resultado da função de impure será diferente.

Função pura

let pure = (input) => {
  let a = 1;
  // Multiply with variable inside function scope
  let output = input * a;
  return output;
}

console.log(pure(2)) // Logs 2

O resultado da função pure não depende de nenhuma variável fora de seu escopo.

Aceitando funções como argumentos #

function transform(fn, arr) {
    let result = [];
    for (let el of arr) {
        result.push(fn(el)); // We push the result of the transformed item to result
    }
    return result;
}

console.log(transform(x => x * 2, [1,2,3,4])); // [2, 4, 6, 8]

Como você pode ver, nossa função transform aceita dois parâmetros, uma função e uma coleção. Ele irá então iterar a coleção e enviar valores para o resultado, chamando fn em cada um deles.

Parece familiar? Isso é muito semelhante a como Array.prototype.map() funciona!

console.log([1, 2, 3, 4].map(x => x * 2)); // [2, 4, 6, 8]
