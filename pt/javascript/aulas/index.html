Sintaxe #

  • classe Foo {}
  • a classe Foo estende Bar {}
  • class Foo { construtor() {} }
  • class Foo { meuMétodo() {} }
  • class Foo { get myProperty() {} }
  • class Foo { set myProperty(newValue) {} }
  • class Foo { static myStaticMethod() {} }
  • class Foo { static get myStaticProperty() {} }
  • const Foo = classe Foo {};
  • const Foo = classe {};

O suporte class só foi adicionado ao JavaScript como parte do padrão [tag:es6] de 2015.

As classes Javascript são açúcar sintático sobre a herança baseada em protótipo já existente do JavaScript. Essa nova sintaxe não introduz um novo modelo de herança orientado a objetos para JavaScript, apenas uma maneira mais simples de lidar com objetos e herança. Uma declaração class é essencialmente um atalho para definir manualmente uma função do construtor e adicionar propriedades ao protótipo do construtor. Uma diferença importante é que as funções podem ser chamadas diretamente (sem a palavra-chave new), enquanto uma classe chamada diretamente lançará uma exceção.

class someClass {
    constructor () {}
    someMethod () {}
}
 
console.log(typeof someClass);               
console.log(someClass);
console.log(someClass === someClass.prototype.constructor);                         
console.log(someClass.prototype.someMethod);
 
// Output:
// function
// function someClass() { "use strict"; }
// true
// function () { "use strict"; }

Se você estiver usando uma versão anterior do JavaScript, precisará de um transpilador como [tag:babel] ou [tag:google-closure-compiler] para compilar o código em uma versão que a plataforma de destino possa entender.

Construtor de classe #

A parte fundamental da maioria das classes é seu construtor, que configura o estado inicial de cada instância e trata de quaisquer parâmetros que foram passados ​​ao chamar new.

Ele é definido em um bloco class como se você estivesse definindo um método chamado constructor, embora na verdade seja tratado como um caso especial.

class MyClass {
    constructor(option) {
        console.log(`Creating instance using ${option} option.`);
        this.option = option;
    }
}

Exemplo de uso:

const foo = new MyClass('speedy'); // logs: "Creating instance using speedy option"

Uma pequena coisa a notar é que um construtor de classe não pode se tornar estático por meio da palavra-chave static, conforme descrito abaixo para outros métodos.

Herança de classe #

A herança funciona exatamente como em outras linguagens orientadas a objetos: os métodos definidos na superclasse são acessíveis na subclasse de extensão.

Se a subclasse declara seu próprio construtor, ela deve invocar o construtor dos pais via super() antes de poder acessar this.

class SuperClass {

    constructor() {
        this.logger = console.log;
    }

    log() {
        this.logger(`Hello ${this.name}`);
    }

}

class SubClass extends SuperClass {

    constructor() {
        super();
        this.name = 'subclass';
    }

}

const subClass = new SubClass();

subClass.log(); // logs: "Hello subclass"

Métodos estáticos #

Métodos e propriedades estáticos são definidos na própria classe/construtor, não em objetos de instância. Eles são especificados em uma definição de classe usando a palavra-chave static.

class MyClass {
    static myStaticMethod() {
        return 'Hello';
    }

    static get myStaticProperty() {
        return 'Goodbye';
    }
}

console.log(MyClass.myStaticMethod()); // logs: "Hello"
console.log(MyClass.myStaticProperty); // logs: "Goodbye"

Podemos ver que as propriedades estáticas não são definidas nas instâncias do objeto:

const myClassInstance = new MyClass();

console.log(myClassInstance.myStaticProperty); // logs: undefined

No entanto, eles são definidos em subclasses:

class MySubClass extends MyClass {};

console.log(MySubClass.myStaticMethod()); // logs: "Hello"
console.log(MySubClass.myStaticProperty); // logs: "Goodbye"

Getters e Setters #

Getters e setters permitem que você defina um comportamento personalizado para ler e escrever uma determinada propriedade em sua classe. Para o usuário, eles parecem iguais a qualquer propriedade típica. No entanto, internamente, uma função personalizada que você fornece é usada para determinar o valor quando a propriedade é acessada (o getter) e para realizar as alterações necessárias quando a propriedade é atribuída (o setter).

Em uma definição class, um getter é escrito como um método sem argumento prefixado pela palavra-chave get. Um setter é semelhante, exceto que ele aceita um argumento (o novo valor sendo atribuído) e a palavra-chave set é usada em seu lugar.

Aqui está uma classe de exemplo que fornece um getter e um setter para sua propriedade .name. Cada vez que for atribuído, vamos gravar o novo nome em um array interno .names_. Cada vez que for acessado, retornaremos o nome mais recente.

class MyClass {
    constructor() {
        this.names_ = [];
    }

    set name(value) {
        this.names_.push(value);
    }

    get name() {
        return this.names_[this.names_.length - 1];
    }
}

const myClassInstance = new MyClass();
myClassInstance.name = 'Joe';
myClassInstance.name = 'Bob';

console.log(myClassInstance.name); // logs: "Bob"
console.log(myClassInstance.names_); // logs: ["Joe", "Bob"]

Se você definir apenas um setter, tentar acessar a propriedade sempre retornará undefined.

const classInstance = new class {
    set prop(value) {
        console.log('setting', value);
    }
};

classInstance.prop = 10; // logs: "setting", 10

console.log(classInstance.prop); // logs: undefined

Se você definir apenas um getter, tentar atribuir a propriedade não terá efeito.

const classInstance = new class {
    get prop() {
        return 5;
    }
};

classInstance.prop = 10;

console.log(classInstance.prop); // logs: 5

Membros Privados #

O JavaScript não oferece suporte técnico a membros privados como um recurso de idioma. Privacidade - descrita por Douglas Crockford - é emulada por meio de fechamentos (escopo de função preservado) que será gerado a cada chamada de instanciação de uma função construtora.

O exemplo Queue demonstra como, com funções construtoras, o estado local podem ser preservados e tornados acessíveis também através de métodos privilegiados.

class Queue {

  constructor () {                    // - does generate a closure with each instantiation.

    const list = [];                  // - local state ("private member").

    this.enqueue = function (type) {  // - privileged public method
                                      //   accessing the local state
      list.push(type);                //   "writing" alike.
      return type;
    };
    this.dequeue = function () {      // - privileged public method
                                      //   accessing the local state
      return list.shift();            //   "reading / writing" alike.
    };
  }
}


var q = new Queue;            //
                              //
q.enqueue(9);                 // ... first in ...
q.enqueue(8);                 //
q.enqueue(7);                 //
                              //
console.log(q.dequeue());     // 9 ... first out.
console.log(q.dequeue());     // 8
console.log(q.dequeue());     // 7
console.log(q);               // {}
console.log(Object.keys(q));  // ["enqueue","dequeue"]

Com cada instanciação de um tipo Queue o construtor gera um encerramento.

Assim, ambos os métodos próprios de um tipo Queue enqueue e dequeue (veja Object.keys(q)) ainda tem acesso a list que continua vivendo em seu escopo que, no momento da construção, foi preservado.

Fazendo uso deste padrão - emulando membros privados via público privilegiado métodos - deve-se ter em mente que, a cada instância, memória adicional será consumido para cada método propriedade (pois é um código que não pode ser compartilhado/reutilizado). O mesmo vale para a quantidade/tamanho do estado que será armazenado dentro de tal fechamento.

Nomes de métodos dinâmicos #

Há também a capacidade de avaliar expressões ao nomear métodos semelhantes a como você pode acessar as propriedades de um objeto com []. Isso pode ser útil para ter nomes de propriedades dinâmicos, mas é frequentemente usado em conjunto com Símbolos.

let METADATA = Symbol('metadata');

class Car {
  constructor(make, model) {
    this.make = make;
    this.model = model;
  }
  
  // example using symbols
  [METADATA]() {
    return {
      make: this.make,
      model: this.model
    };
  }

  // you can also use any javascript expression

  // this one is just a string, and could also be defined with simply add()
  ["add"](a, b) {
    return a + b;
  }

  // this one is dynamically evaluated
  [1 + 2]() {
    return "three";
  }
}

let MazdaMPV = new Car("Mazda", "MPV");
MazdaMPV.add(4, 5); // 9
MazdaMPV[3](); // "three"
MazdaMPV[METADATA](); // { make: "Mazda", model: "MPV" }

Métodos #

Os métodos podem ser definidos em classes para executar uma função e, opcionalmente, retornar um resultado. Eles podem receber argumentos do chamador.

class Something {
    constructor(data) {
        this.data = data
    }

    doSomething(text) {
        return {
            data: this.data,
            text
        }
    }
}

var s = new Something({})
s.doSomething("hi") // returns: { data: {}, text: "hi" }

Gerenciando Dados Privados com Classes #

Um dos obstáculos mais comuns ao usar classes é encontrar a abordagem adequada para lidar com estados privados. Existem 4 soluções comuns para lidar com estados privados:

Usando símbolos #

Símbolos são novos tipos primitivos introduzidos no ES2015, conforme definido em MDN

Um símbolo é um tipo de dado único e imutável que pode ser usado como identificador para propriedades de objetos.

Ao usar símbolo como chave de propriedade, ele não é enumerável.

Como tal, eles não serão revelados usando for var in ou Object.keys.

Assim, podemos usar símbolos para armazenar dados privados.

const topSecret = Symbol('topSecret'); // our private key; will only be accessible on the scope of the module file
export class SecretAgent{
    constructor(secret){
        this[topSecret] = secret; // we have access to the symbol key (closure)
        this.coverStory = 'just a simple gardner';
        this.doMission = () => {
            figureWhatToDo(topSecret[topSecret]); // we have access to topSecret
        };
    }
}

Como os símbolos são únicos, devemos ter referência ao símbolo original para acessar a propriedade privada.

import {SecretAgent} from 'SecretAgent.js'
const agent = new SecretAgent('steal all the ice cream');
// ok lets try to get the secret out of him!
Object.keys(agent); // ['coverStory'] only cover story is public, our secret is kept.
agent[Symbol('topSecret')]; // undefined, as we said, symbols are always unique, so only the original symbol will help us to get the data.

Mas não é 100% privado; vamos quebrar esse agente! Podemos usar o método Object.getOwnPropertySymbols para obter os símbolos dos objetos.

const secretKeys = Object.getOwnPropertySymbols(agent);
agent[secretKeys[0]] // 'steal all the ice cream' , we got the secret.

Usando WeakMaps #

WeakMap é um novo tipo de objeto que foi adicionado para es6.

Conforme definido em MDN

O objeto WeakMap é uma coleção de pares chave/valor em que as chaves são fracamente referenciadas. As chaves devem ser objetos e os valores podem ser valores arbitrários.

Outra característica importante do WeakMap é, conforme definido em MDN.

A chave em um WeakMap é mantida fracamente. O que isso significa é que, se não houver outras referências fortes à chave, a entrada inteira será removida do WeakMap pelo coletor de lixo.

A ideia é usar o WeakMap, como um mapa estático para toda a classe, para manter cada instância como chave e manter os dados privados como um valor para essa chave de instância.

Assim somente dentro da classe teremos acesso à coleção WeakMap.

Vamos dar uma chance ao nosso agente, com WeakMap:

const topSecret = new WeakMap(); // will hold all private data of all instances.
export class SecretAgent{
    constructor(secret){
        topSecret.set(this,secret); // we use this, as the key, to set it on our instance private data
        this.coverStory = 'just a simple gardner';
        this.doMission = () => {
            figureWhatToDo(topSecret.get(this)); // we have access to topSecret
        };
    }
}

Como o const topSecret é definido dentro de nosso fechamento de módulo, e como não o vinculamos às propriedades de nossa instância, essa abordagem é totalmente privada e não podemos alcançar o agente topSecret.

Defina todos os métodos dentro do construtor #

A ideia aqui é simplesmente definir todos os nossos métodos e membros dentro do construtor e usar o encerramento para acessar membros privados sem atribuí-los a this.

   export class SecretAgent{
        constructor(secret){
            const topSecret = secret;
            this.coverStory = 'just a simple gardner';
            this.doMission = () => {
                figureWhatToDo(topSecret); // we have access to topSecret
            };
        }
    }

Neste exemplo também os dados são 100% privados e não podem ser acessados ​​fora da classe, então nosso agente está seguro.

Usando convenções de nomenclatura #

Decidiremos que qualquer propriedade privada terá o prefixo _.

Observe que, para essa abordagem, os dados não são realmente privados.

export class SecretAgent{
    constructor(secret){
        this._topSecret = secret; // it private by convention
        this.coverStory = 'just a simple gardner';
        this.doMission = () => {
            figureWhatToDo(this_topSecret); 
        };
    }
}

Vinculação de nome de classe #

O nome de ClassDeclaration é vinculado de diferentes maneiras em diferentes escopos -

  1. O escopo no qual a classe é definida - vinculação let
  2. O escopo da própria classe - dentro de { e } em class {} - ligação const
class Foo {
  // Foo inside this block is a const binding
}
// Foo here is a let binding

Por exemplo,

class A {
  foo() {
    A = null; // will throw at runtime as A inside the class is a `const` binding
  }
}
A = null; // will NOT throw as A here is a `let` binding

Isso não é o mesmo para uma função -

function A() {
  A = null; // works
}
A.prototype.foo = function foo() {
  A = null; // works
}
A = null; // works
