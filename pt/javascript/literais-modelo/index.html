Os literais de modelo são um tipo de literal de string que permite que valores sejam interpolados e, opcionalmente, o comportamento de interpolação e construção seja controlado usando uma função “tag”.

Sintaxe #

  • mensagem = `Bem-vindo, ${user.name}!`
  • padrão = new RegExp(String.raw`Bem-vindo, (\w+)!`);
  • query = SQL`INSERT INTO User (name) VALUES (${name})`

Os literais de modelo foram especificados pela primeira vez por ECMAScript 6 §12.2.9.

Interpolação básica e strings de várias linhas #

Literais de modelo são um tipo especial de literal de string que pode ser usado em vez do padrão '...' ou "...". Eles são declarados citando a string com acentos graves em vez das aspas simples ou duplas padrão: `...`.

Os literais de modelo podem conter quebras de linha e expressões arbitrárias podem ser incorporadas usando a sintaxe de substituição ${ expressão }. Por padrão, os valores dessas expressões de substituição são concatenados diretamente na string em que aparecem.

const name = "John";
const score = 74;

console.log(`Game Over!

${name}'s score was ${score * 10}.`);

Game Over!

John's score was 740.

Sequências marcadas #

Uma função identificada imediatamente antes de um literal de modelo ser usado para interpretá-lo, no que é chamado de literal de modelo marcado. A função tag pode retornar uma string, mas também pode retornar qualquer outro tipo de valor.

O primeiro argumento para a função de tag, strings, é um Array de cada parte constante do literal. Os argumentos restantes, ...substitutions, contêm os valores avaliados de cada expressão de substituição ${}.

function settings(strings, ...substitutions) {
  const result = new Map();
  for (let i = 0; i < substitutions.length; i++) {
    result.set(strings[i].trim(), substitutions[i]);
  }
  return result;
}

const remoteConfiguration = settings`
  label    ${'Content'}
  servers  ${2 * 8 + 1}
  hostname ${location.hostname}
`;

Map {"label" => "Content", "servers" => 17, "hostname" => "stackoverflow.com"}

O Array strings tem uma propriedade especial .raw referenciando um Array paralelo das mesmas partes constantes do literal do modelo, mas exatamente como aparecem no código-fonte, sem que nenhuma barra invertida seja substituída.

function example(strings, ...substitutions) {
  console.log('strings:', strings);
  console.log('...substitutions:', substitutions);
}

example`Hello ${'world'}.\n\nHow are you?`;

strings: ["Hello ", ".\n\nHow are you?", raw: ["Hello ", ".\\n\\nHow are you?"]]
substitutions: ["world"]

Cordas brutas #

A função de tag String.raw pode ser usada com literais de modelo para acessar uma versão de seu conteúdo sem interpretar nenhuma sequência de escape de barra invertida.

String.raw`\n` conterá uma barra invertida e a letra minúscula n, enquanto `\n` ou '\n ' conteria um único caractere de nova linha.

const patternString = String.raw`Welcome, (\w+)!`;
const pattern = new RegExp(patternString);

const message = "Welcome, John!";
pattern.exec(message);

["Welcome, John!", "John"]

Modelando HTML com Strings de Modelo #

Você pode criar uma função de tag de string de modelo HTML`…` para codificar automaticamente valores interpolados. (Isso requer que os valores interpolados sejam usados ​​apenas como texto e pode não ser seguro se os valores interpolados forem usados ​​no código, como scripts ou estilos.)

class HTMLString extends String {
  static escape(text) {
    if (text instanceof HTMLString) {
      return text;
    }
    return new HTMLString(
        String(text)
            .replace(/&/g, '&amp;')
            .replace(/</g, '&lt;')
            .replace(/>/g, '&gt;')
            .replace(/"/g, '&quot;')
            .replace(/\\/g, '&#39;'));
  }
}

function HTML(strings, ...substitutions) { 
  const escapedFlattenedSubstitutions =
      substitutions.map(s => [].concat(s).map(HTMLString.escape).join(''));
  const pieces = [];
  for (const i of strings.keys()) {
    pieces.push(strings[i], escapedFlattenedSubstitutions [i] || '');
  }
  return new HTMLString(pieces.join(''));
}

const title = "Hello World";
const iconSrc = "/images/logo.png";
const names = ["John", "Jane", "Joe", "Jill"];

document.body.innerHTML = HTML`
  <h1><img src="${iconSrc}" /> ${title}</h1>

  <ul> ${names.map(name => HTML`
    <li>${name}</li>
  `)} </ul>
`;

Introdução #

Os literais de modelo agem como strings com recursos especiais. Eles são delimitados pelo back-tick `` e podem ser estendidos por várias linhas.

Os literais de modelo também podem conter expressões incorporadas. Essas expressões são indicadas por um sinal $ e chaves {}

//A single line Template Literal  
var aLiteral = `single line string data`;


//Template Literal that spans across lines       
var anotherLiteral = `string data that spans
         across multiple lines of code`;     

//Template Literal with an embedded expression
var x = 2;
var y = 3; 
var theTotal = `The total is ${x + y}`;     // Contains "The total is 5" 



//Comarison of a string and a template literal
var aString = "single line string data"
console.log(aString === aLiteral)                         //Returns true

Existem muitos outros recursos de String Liteals, como Tagged Template Liteals e propriedade Raw. Estes são demonstrados em outros exemplos.
