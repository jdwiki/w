Ao usar a coerção booleana, os seguintes valores são considerados “falsy”:

  • falso
  • 0
  • "" (string vazia)
  • nulo
  • indefinido
  • NaN (não é um número, por exemplo, 0/0)
  • document.all¹ (contexto do navegador)

Todo o resto é considerado “verdadeiro”.

¹ violação intencional da especificação ECMAScript

Igualdade/desigualdade abstrata e conversão de tipo #

O problema #

Os operadores abstratos de igualdade e desigualdade (== e !=) convertem seus operandos se os tipos de operandos não corresponderem. Esse tipo de coerção é uma fonte comum de confusão sobre os resultados desses operadores, em particular, esses operadores nem sempre são transitivos como seria de esperar.

"" ==  0;     // true A
 0 == "0";    // true A
"" == "0";    // false B
false == 0;   // true
false == "0"; // true

"" !=  0;     // false A
 0 != "0";    // false A
"" != "0";    // true B
false != 0;   // false
false != "0"; // false

Os resultados começam a fazer sentido se você considerar como o JavaScript converte strings vazias em números.

Number("");    // 0
Number("0");   // 0
Number(false); // 0

A solução #

Na instrução false B, ambos os operandos são strings ("" e "0"), portanto não haverá nenhuma conversão de tipo e como "" e "0" são não é o mesmo valor, "" == "0" é false como esperado.

Uma maneira de eliminar o comportamento inesperado aqui é garantir que você sempre compare os operandos do mesmo tipo. Por exemplo, se você quiser os resultados da comparação numérica, use a conversão explícita:

var test = (a,b) => Number(a) == Number(b); 
test("", 0);        // true;
test("0", 0);       // true
test("", "0");      // true;
test("abc", "abc"); // false as operands are not numbers

Ou, se você quiser comparação de strings:

var test = (a,b) => String(a) == String(b);
test("", 0);   // false;
test("0", 0);  // true
test("", "0"); // false;

Nota lateral: Number("0") e new Number("0") não são a mesma coisa! Enquanto o primeiro realiza uma conversão de tipo, o último criará um novo objeto. Os objetos são comparados por referência e não por valor, o que explica os resultados abaixo.

Number("0") == Number("0");         // true;
new Number("0") == new Number("0"); // false

Por fim, você tem a opção de usar operadores de igualdade e desigualdade estritos que não realizarão nenhuma conversão de tipo implícita.

"" ===  0;  // false
 0 === "0"; // false
"" === "0"; // false

Propriedade NaN do Objeto Global #

NaN ("Not a Number") é um valor especial definido pelo IEEE Standard for Floating-Point Arithmetic, que é usado quando um valor não numérico é fornecido, mas um número é esperado (1 * "dois"), ou quando um cálculo não tem um resultado number válido (Math.sqrt(-1)).

Qualquer igualdade ou comparações relacionais com NaN retorna false, mesmo comparando-o consigo mesmo. Porque, NaN deve denotar o resultado de uma computação sem sentido e, como tal, não é igual ao resultado de qualquer outra computação sem sentido.

(1 * "two") === NaN  //false

NaN === 0;          // false
NaN === NaN;        // false
Number.NaN === NaN; // false

NaN < 0;            // false
NaN > 0;            // false
NaN > 0;            // false
NaN >= NaN;         // false
NaN >= 'two';       // false

Comparações não iguais sempre retornarão true:

NaN !== 0;          // true
NaN !== NaN;        // true

Verificando se um valor é NaN #

Você pode testar um valor ou expressão para NaN usando a função Number.isNaN():

Number.isNaN(NaN);         // true
Number.isNaN(0 / 0);       // true
Number.isNaN('str' - 12);  // true

Number.isNaN(24);          // false
Number.isNaN('24');        // false
Number.isNaN(1 / 0);       // false
Number.isNaN(Infinity);    // false

Number.isNaN('str');       // false
Number.isNaN(undefined);   // false
Number.isNaN({});          // false

Você pode verificar se um valor é NaN comparando-o com ele mesmo:

value !== value;    // true for NaN, false for any other value

Você pode usar o seguinte polyfill para Number.isNaN():

Number.isNaN = Number.isNaN || function(value) {     
    return value !== value;
}

Por outro lado, a função global isNaN() retorna true não apenas para NaN, mas também para qualquer valor ou expressão que não pode ser convertido em um número:

isNaN(NaN);         // true
isNaN(0 / 0);       // true
isNaN('str' - 12);  // true

isNaN(24);          // false
isNaN('24');        // false
isNaN(Infinity);    // false

isNaN('str');       // true
isNaN(undefined);   // true
isNaN({});          // true

ECMAScript define um algoritmo de “mesmice” chamado SameValue que, desde ECMAScript 6, pode ser invocado com Object.is. Ao contrário da comparação == e ===, usar Object.is() tratará NaN como idêntico a si mesmo (e -0 como não idêntico a +0):

Object.is(NaN, NaN)      // true
Object.is(+0, 0)         // false

NaN === NaN              // false
+0 === 0                 // true

Você pode usar o seguinte polyfill para Object.is() (de [MDN](https://developer.mozilla.org/en-US/docs/Web/JavaScript/Reference/Global_Objects/Object/is#Polyfill_for_non- ES6_navegadores)):

if (!Object.is) {
  Object.is = function(x, y) {
    // SameValue algorithm
    if (x === y) { // Steps 1-5, 7-10
      // Steps 6.b-6.e: +0 != -0
      return x !== 0 || 1 / x === 1 / y;
    } else {
      // Step 6.a: NaN == NaN
      return x !== x && y !== y;
    }
  };
}

Pontos a observar #

O próprio NaN é um número, o que significa que não é igual à string “NaN”, e o mais importante (embora talvez não seja intuitivo):

typeof(NaN) === "number"; //true

Curto-circuito em operadores booleanos #

O operador and (&&) e o operador or (||) empregam curto-circuito para evitar trabalho desnecessário se o resultado da operação não mudar com o trabalho extra.

Em x && y, y não será avaliado se x for avaliado como false, porque toda a expressão é garantida como false.

Em x || y, y não será avaliado se x for avaliado como true, porque toda a expressão é garantida como true.

Exemplo com funções

Tome as duas funções a seguir:

function T() { // True
  console.log("T");
  return true;
}

function F() { // False
  console.log("F");
  return false;
}

Exemplo 1

T() && F(); // false

Resultado:

‘T’
‘F’

Exemplo 2

F() && T(); // false

Resultado:

‘F’

Exemplo 3

T() || F(); // true

Resultado:

‘T’

Exemplo 4

F() || T(); // true

Resultado:

‘F’
‘T’

Curto-circuito para evitar erros

var obj; // object has value of undefined
if(obj.property){ }// TypeError: Cannot read property 'property' of undefined
if(obj.property && obj !== undefined){}// Line A TypeError: Cannot read property 'property' of undefined

Linha A: se você inverter a ordem, a primeira instrução condicional impedirá o erro na segunda, não a executando se lançar o erro

if(obj !== undefined && obj.property){}; // no error thrown

Mas só deve ser usado se você espera undefined

if(typeof obj === "object" && obj.property){}; // safe option but slower

Curto-circuito para fornecer um valor padrão

O operador || pode ser usado para selecionar um valor “verdadeiro” ou o valor padrão.

Por exemplo, isso pode ser usado para garantir que um valor anulável seja convertido em um valor não anulável:

var nullableObj = null;
var obj = nullableObj || {};  // this selects {}

var nullableObj2 = {x: 5};
var obj2 = nullableObj2 || {} // this selects {x: 5}

Ou para retornar o primeiro valor verdadeiro

var truthyValue = {x: 10};
return truthyValue || {}; // will return {x: 10}

O mesmo pode ser usado para retornar várias vezes:

envVariable || configValue || defaultConstValue // select the first "truthy" of these

Curto-circuito para chamar uma função opcional

O operador && pode ser usado para avaliar um retorno de chamada, somente se for passado:

function myMethod(cb) {
    // This can be simplified
    if (cb) {
       cb();
    }

    // To this
    cb && cb();
}

Claro, o teste acima não valida que cb é de fato uma função e não apenas um Object/Array/String/Number.

Igualdade Abstrata (==) #

Operandos do operador de igualdade abstrato são comparados depois de serem convertidos em um tipo comum. A forma como essa conversão acontece é baseada na especificação do operador:

Especificação para o operador ==:

7.2.13 Comparação de Igualdade Abstrata

A comparação `x == y`, onde `x` e `y` são valores, produz `true` ou `false`. Essa comparação é feita da seguinte forma: 1. Se `Type(x)` for o mesmo que `Type(y)`, então: - **a.** Retorna o resultado da execução da Comparação de Igualdade Estrita `x === y`. 2. Se `x` for `null` e `y` for `undefined`, retorne `true`. 3. Se `x` for `undefined` e `y` for `null`, retorne `true`. 4. Se `Type(x)` for `Number` e `Type(y)` for `String`, retorne o resultado da comparação `x == ToNumber(y)`. 5. Se `Type(x)` for `String` e `Type(y)` for `Number`, retorne o resultado da comparação `ToNumber(x) == y`. 6. Se `Type(x)` for `Boolean`, retorne o resultado da comparação `ToNumber(x) == y`. 7. Se `Type(y)` for `Boolean`, retorne o resultado da `comparação x == ToNumber(y)`. 8. Se `Type(x)` for `String`, `Number` ou `Symbol` e `Type(y)` for `Object`, retorne o resultado da comparação `x == ToPrimitive(y)`. 9. Se `Type(x)` for Object e `Type(y)` for `String`, `Number` ou `Symbol`, retorne o resultado da comparação `ToPrimitive(x) == y`. 10. Retorna `falso`.

Exemplos: #

1 == 1;                     // true
1 == true;                  // true  (operand converted to number: true => 1)
1 == '1';                   // true  (operand converted to number: '1' => 1 )
1 == '1.00';                // true
1 == '1.00000000001';       // false
1 == '1.00000000000000001'; // true  (true due to precision loss)
null == undefined;          // true  (spec #2)
1 == 2;                     // false
0 == false;                 // true
0 == undefined;             // false
0 == "";                    // true

Nulo e Indefinido #

As diferenças entre null e undefined #

null e undefined compartilham igualdade abstrata == mas não igualdade estrita ===,

null == undefined   // true
null === undefined  // false

Eles representam coisas ligeiramente diferentes:

  • undefined representa a ausência de um valor, como antes de uma propriedade identificador/objeto ser criada ou no período entre a criação do parâmetro identificador/função e sua primeira configuração, se houver.
  • null representa a ausência intencional de um valor para um identificador ou propriedade que já foi criada.

São diferentes tipos de sintaxe:

  • undefined é uma propriedade do Objeto global, normalmente imutável no escopo global. Isso significa que qualquer lugar que você possa definir um identificador diferente do namespace global pode ocultar undefined desse escopo (embora as coisas ainda possam ser undefined)
  • null é uma palavra literal, então seu significado nunca pode ser alterado e tentar fazer isso lançará um Error.

As semelhanças entre null e undefined #

null e undefined são ambos falsos.

if (null) console.log("won't be logged");
if (undefined) console.log("won't be logged");

Nem null nem undefined são iguais a false (veja esta pergunta).

false == undefined   // false
false == null        // false
false === undefined  // false
false === null       // false

Usando indefinido #

  • Se o escopo atual não puder ser confiável, use algo que seja avaliado como undefined, por exemplo void 0;.
  • Se undefined for sombreado por outro valor, é tão ruim quanto sombrear Array ou Number.
  • Evite setting algo como undefined. Se você deseja remover uma propriedade bar de um Object foo, em vez disso, delete foo.bar;.
  • Identificador de teste de existência foo contra undefined poderia lançar um erro de referência, use typeof foo contra "undefined" em vez disso.

Operadores lógicos com booleanos #

var x = true,
    y = false;

E #

Este operador retornará true se ambas as expressões forem avaliadas como true. Este operador booleano empregará curto-circuito e não avaliará y se x for avaliado como false.

x && y;

Isso retornará false, porque y é false.

OU #

Este operador retornará true se uma das duas expressões for avaliada como true. Este operador booleano empregará curto-circuito e y não será avaliado se x for avaliado como true.

x || y;

Isso retornará true, porque x é true.

NÃO #

Este operador retornará false se a expressão à direita for avaliada como true e retornará true se a expressão à direita for avaliada como false.

!x;

Isso retornará false, porque x é true.

Operadores lógicos com valores não booleanos (coerção booleana) #

O OR lógico (||), lendo da esquerda para a direita, será avaliado para o primeiro valor truthy. Se nenhum valor truthy for encontrado, o último valor será retornado.

var a = 'hello' || '';             // a = 'hello'
var b = '' || [];                  // b = []
var c = '' || undefined;           // c = undefined
var d = 1 || 5;                    // d = 1
var e = 0 || {};                   // e = {}
var f = 0 || '' || 5;              // f = 5
var g = '' || 'yay' || 'boo';      // g = 'yay'

O AND lógico (&&), lendo da esquerda para a direita, será avaliado para o primeiro valor falsy. Se nenhum valor falsey for encontrado, o último valor será retornado.

var a = 'hello' && '';                  // a = ''
var b = '' && [];                       // b = ''
var c = undefined && 0;                 // c = undefined
var d = 1 && 5;                         // d = 5
var e = 0 && {};                        // e = 0
var f = 'hi' && [] && 'done';           // f = 'done'
var g = 'bye' && undefined && 'adios';  // g = undefined

Este truque pode ser usado, por exemplo, para definir um valor padrão para um argumento de função (antes de ES6).

var foo = function(val) {
    // if val evaluates to falsey, 'default' will be returned instead.
    return val || 'default';
}

console.log( foo('burger') );  // burger
console.log( foo(100) );       // 100
console.log( foo([]) );        // []
console.log( foo(0) );         // default
console.log( foo(undefined) ); // default

Apenas tenha em mente que para argumentos, 0 e (em menor grau) a string vazia também são frequentemente valores válidos que devem poder ser explicitamente passados ​​e substituir um padrão, o que, com esse padrão, eles não serão ( porque são falsos).

Matriz vazia #

/* ToNumber(ToPrimitive([])) == ToNumber(false) */
[] == false; // true

Quando [].toString() é executado ele chama [].join() se existir, ou Object.prototype.toString() caso contrário. Esta comparação está retornando true porque [].join() retorna '' que, forçado em 0, é igual a false [ToNumber](http://www.ecma-international.org/ecma -262/5.1/#sec-9.3).

Cuidado, porém, todos os objetos são verdadeiros e Array é uma instância de Object:

// Internally this is evaluated as ToBoolean([]) === true ? 'truthy' : 'falsy'
[] ? 'truthy' : 'falsy'; // 'truthy'

Conversões automáticas de tipo #

Esteja ciente de que os números podem acidentalmente ser convertidos em strings ou NaN (Not a Number).

JavaScript é livremente tipado. Uma variável pode conter diferentes tipos de dados e uma variável pode alterar seu tipo de dados:

var x = "Hello";     // typeof x is a string
x = 5;               // changes typeof x to a number

Ao fazer operações matemáticas, o JavaScript pode converter números em strings:

var x = 5 + 7;       // x.valueOf() is 12,  typeof x is a number
var x = 5 + "7";     // x.valueOf() is 57,  typeof x is a string
var x = "5" + 7;     // x.valueOf() is 57,  typeof x is a string
var x = 5 - 7;       // x.valueOf() is -2,  typeof x is a number
var x = 5 - "7";     // x.valueOf() is -2,  typeof x is a number
var x = "5" - 7;     // x.valueOf() is -2,  typeof x is a number
var x = 5 - "x";     // x.valueOf() is NaN, typeof x is a number

Subtrair uma string de uma string não gera um erro, mas retorna NaN (Not a Number):

"Hello" - "Dolly"    // returns NaN

Operações de comparação de igualdade #

JavaScript tem quatro operações diferentes de comparação de igualdade.

MesmoValor #

Retorna true se ambos os operandos pertencerem ao mesmo tipo e tiverem o mesmo valor.

Nota: o valor de um objeto é uma referência.

Você pode usar este algoritmo de comparação via Object.is (ECMAScript 6).

Exemplos:

Object.is(1, 1);            // true
Object.is(+0, -0);          // false
Object.is(NaN, NaN);        // true
Object.is(true, "true");    // false
Object.is(false, 0);        // false
Object.is(null, undefined); // false
Object.is(1, "1");          // false
Object.is([], []);          // false

Este algoritmo tem as propriedades de uma relação de equivalência:

  • Reflexividade: Object.is(x, x) é true, para qualquer valor x
  • Simetria: Object.is(x, y) é true se, e somente se, Object.is(y, x) for true, para quaisquer valores x e s.
  • Transitivity: se Object.is(x, y) e Object.is(y, z) forem true, então Object.is(x, z) também será true , para quaisquer valores x, y e z.

SameValueZero #

Ele se comporta como SameValue, mas considera +0 e -0 iguais.

Você pode usar este algoritmo de comparação via Array.prototype.includes (ECMAScript 7).

Exemplos:

[1].includes(1);            // true
[+0].includes(-0);          // true
[NaN].includes(NaN);        // true
[true].includes("true");    // false
[false].includes(0);        // false
[1].includes("1");          // false
[null].includes(undefined); // false
[[]].includes([]);          // false

Este algoritmo ainda tem as propriedades de uma relação de equivalência:

  • Reflexivity: [x].includes(x) é true, para qualquer valor x
  • Simetria: [x].includes(y) for true se, e somente se, [y].includes(x) for true, para quaisquer valores x e s.
  • Transitivity: se [x].includes(y) e [y].includes(z) forem true, então [x].includes(z) também será true , para quaisquer valores x, yez`.

Comparação de Igualdade Estrita #

Ele se comporta como SameValue, mas

  • Considera +0 e -0 iguais.
  • Considera NaN diferente de qualquer valor, incluindo ele mesmo

Você pode usar este algoritmo de comparação através do operador === (ECMAScript 3).

Há também o operador !== (ECMAScript 3), que nega o resultado de ===.

Exemplos:

1 === 1;            // true
+0 === -0;          // true
NaN === NaN;        // false
true === "true";    // false
false === 0;        // false
1 === "1";          // false
null === undefined; // false
[] === [];          // false

Este algoritmo tem as seguintes propriedades:

  • Simetria: x === y for true se, e somente se, y === xfortrue, para quaisquer valores xey`.
  • Transitividade: Se x === y e y === z são true, então x === z também é true, para quaisquer valores x, y e z.

Mas não é uma relação de equivalência porque

  • NaN não é reflexivo: NaN !== NaN

Comparação de Igualdade Abstrata #

Se ambos os operandos pertencem ao mesmo Tipo, ele se comporta como a Comparação de Igualdade Estrita.

Caso contrário, ele os coage da seguinte forma:

  • undefined e null são considerados iguais
  • Ao comparar um número com uma string, a string é forçada a um número
  • Ao comparar um booleano com outra coisa, o booleano é forçado a um número
  • Ao comparar um objeto com um número, string ou símbolo, o objeto é coagido a um primitivo

Se houve coação, os valores coagidos são comparados recursivamente. Caso contrário, o algoritmo retorna false.

Você pode usar este algoritmo de comparação através do operador == (ECMAScript 1).

Há também o operador != (ECMAScript 1), que nega o resultado de ==.

Exemplos:

1 == 1;            // true
+0 == -0;          // true
NaN == NaN;        // false
true == "true";    // false
false == 0;        // true
1 == "1";          // true
null == undefined; // true
[] == [];          // false

Este algoritmo tem a seguinte propriedade:

  • Simetria: x == y é true se, e somente se, y == x for true, para quaisquer valores x e y.

Mas não é uma relação de equivalência porque

  • NaN não é reflexivo: NaN != NaN
  • Transitivity não é válido, ex. 0 == '' e 0 == '0', mas '' != '0'

Operadores relacionais (<, <=, >, >=) #

Quando ambos os operandos são numéricos, eles são comparados normalmente:

1 < 2        // true
2 <= 2       // true
3 >= 5       // false
true < false // false (implicitly converted to numbers, 1 > 0)

Quando ambos os operandos são strings, eles são comparados lexicograficamente (de acordo com a ordem alfabética):

'a' < 'b'    // true
'1' < '2'    // true
'100' > '12' // false ('100' is less than '12' lexicographically!)

Quando um operando é uma string e o outro é um número, a string é convertida em um número antes da comparação:

'1' < 2      // true
'3' > 2      // true
true > '2'   // false (true implicitly converted to number, 1 < 2)

Quando a string não é numérica, a conversão numérica retorna NaN (não é um número). Comparar com NaN sempre retorna false:

1 < 'abc'    // false
1 > 'abc'    // false

Mas tenha cuidado ao comparar um valor numérico com strings null, undefined ou vazias:

1 > ''        // true
1 < ''        // false
1 > null      // true
1 < null      // false
1 > undefined // false
1 < undefined // false

Quando um operando é um objeto e o outro é um número, o objeto é convertido em um número antes da comparação. Portanto, null é um caso particular porque Number(null);//0

new Date(2015)< 1479480185280            // true
null > -1                                //true
({toString:function(){return 123}}) > 122  //true

Desigualdade #

O operador != é o inverso do operador ==. Retornará true se os operandos não forem iguais. O mecanismo javascript tentará converter ambos os operandos em tipos correspondentes se eles não forem do mesmo tipo. Observação: se os dois operandos tiverem referências internas diferentes na memória, será retornado false.

Amostra:

1 != '1'     // false
1 != 2       // true

No exemplo acima, 1 != '1' é false porque um tipo de número primitivo está sendo comparado a um valor char. Portanto, o mecanismo Javascript não se importa com o tipo de dados do valor R.H.S.

Operador: !== é o inverso do operador ===. Retornará true se os operandos não forem iguais ou se seus tipos não corresponderem.

Exemplo:

1 !== '1'    // true
1 !== 2      // true
1 !== 1      // false

Agrupando várias instruções lógicas #

Você pode agrupar várias instruções de lógica booleana entre parênteses para criar uma avaliação de lógica mais complexa, especialmente útil em instruções if.

if ((age >= 18 && height >= 5.11) || (status === 'royalty' && hasInvitation)) {
  console.log('You can enter our club');
}

Também poderíamos mover a lógica agrupada para variáveis ​​para tornar a instrução um pouco mais curta e descritiva:

var isLegal = age >= 18;
var tall = height >= 5.11;
var suitable = isLegal && tall;
var isRoyalty = status === 'royalty';
var specialCase = isRoyalty && hasInvitation;
var canEnterOurBar = suitable || specialCase;

if (canEnterOurBar) console.log('You can enter our club');

Observe que neste exemplo em particular (e em muitos outros), agrupar as instruções entre parênteses funciona da mesma forma que se as removêssemos, basta seguir uma avaliação lógica linear e você encontrará o mesmo resultado. Eu prefiro usar parênteses, pois me permite entender mais claramente o que eu pretendia e pode evitar erros de lógica.

Campos de bits para otimizar a comparação de dados de vários estados #

Um campo de bits é uma variável que contém vários estados booleanos como bits individuais. Um pouco ligado representaria verdadeiro e desligado seria falso. No passado, os campos de bits eram usados ​​rotineiramente, pois economizavam memória e reduziam a carga de processamento. Embora a necessidade de usar o campo de bits não seja mais tão importante, eles oferecem alguns benefícios que podem simplificar muitas tarefas de processamento.

Por exemplo, entrada do usuário. Ao obter entrada das teclas de direção de um teclado para cima, para baixo, para a esquerda, para a direita, você pode codificar as várias teclas em uma única variável com cada direção atribuída a um bit.

Exemplo de leitura de teclado via bitfield

var bitField = 0;  // the value to hold the bits
const KEY_BITS = [4,1,8,2]; // left up right down
const KEY_MASKS = [0b1011,0b1110,0b0111,0b1101]; // left up right down
window.onkeydown = window.onkeyup = function (e) {
    if(e.keyCode >= 37 && e.keyCode <41){
        if(e.type === "keydown"){
            bitField |= KEY_BITS[e.keyCode - 37];
        }else{
            bitField &= KEY_MASKS[e.keyCode - 37];
        }
    }    
}

Exemplo de leitura como um array

var directionState = [false,false,false,false];
window.onkeydown = window.onkeyup = function (e) {
    if(e.keyCode >= 37 && e.keyCode <41){
        directionState[e.keyCode - 37] = e.type === "keydown";
    }    
}

Para ativar um bit use bit a bit ou | e o valor correspondente ao bit. Então, se você deseja definir o 2º bit, bitField |= 0b10 irá ativá-lo. Se você deseja desativar um pouco, use bit a bit e & com um valor que tenha tudo ligado pelo bit necessário. Usando 4 bits e desligando o 2º bit bitfield &= 0b1101;

Você pode dizer que o exemplo acima parece muito mais complexo do que atribuir os vários estados de chave a um array. Sim É um pouco mais complexo de configurar mas a vantagem vem ao interrogar o estado.

Se você quiser testar se todas as teclas estão ativadas.

// as bit field
if(!bitfield) // no keys are on

// as array test each item in array
if(!(directionState[0] && directionState[1] && directionState[2] && directionState[3])){

Você pode definir algumas constantes para facilitar as coisas

// postfix U,D,L,R for Up down left right
const KEY_U = 1;
const KEY_D = 2;
const KEY_L = 4;
const KEY_R = 8;
const KEY_UL = KEY_U + KEY_L; // up left
const KEY_UR = KEY_U + KEY_R; // up Right
const KEY_DL = KEY_D + KEY_L; // down left
const KEY_DR = KEY_D + KEY_R; // down right

Você pode então testar rapidamente vários estados do teclado

if ((bitfield & KEY_UL) === KEY_UL) { // is UP and LEFT only down
if (bitfield  & KEY_UL) {             // is Up left down 
if ((bitfield & KEY_U) === KEY_U) {   // is Up only down
if (bitfield & KEY_U) {               // is Up down (any other key may be down)
if (!(bitfield & KEY_U)) {            // is Up up (any other key may be down)
if (!bitfield ) {                     // no keys are down
if (bitfield ) {                      // any one or more keys are down

A entrada do teclado é apenas um exemplo. Os campos de bits são úteis quando você tem vários estados que devem ser combinados. Javascript pode usar até 32 bits para um campo de bits. Usá-los pode oferecer aumentos significativos de desempenho. Vale a pena conhecer.
