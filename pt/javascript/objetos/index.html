Sintaxe #

  • objeto = { }
  • objeto = novo objeto()
  • objeto = Object.create(prototype[, propertiesObject])
  • object.key = valor
  • objeto[“chave”] = valor
  • objeto[Símbolo()] = valor
  • objeto = { chave1: valor1, “chave2”: valor2, ‘chave3’: valor3 }
  • objeto = { método conciso() { … } }
  • objeto = { [computado() + “chave”]: valor }
  • Object.defineProperty(obj, propertyName, propertyDescriptor)
  • property_desc = Object.getOwnPropertyDescriptor(obj, propertyName)
  • Object.freeze(obj)
  • Object.seal(obj)

Parâmetros #

Propriedade Descrição
valor O valor a ser atribuído à propriedade.
gravável Se o valor da propriedade pode ser alterado ou não.
enumerável Se a propriedade será enumerada em loops for in ou não.
configurável Se será possível redefinir o descritor de propriedade ou não.
obter Uma função a ser chamada que retornará o valor da propriedade.
conjunto Uma função a ser chamada quando a propriedade recebe um valor.

Objetos são coleções de pares chave-valor ou propriedades. As chaves podem ser Strings ou Symbols, e os valores são primitivos (números, strings, símbolos) ou referências a outros objetos.

Em JavaScript, uma quantidade significativa de valores são objetos (por exemplo, funções, arrays) ou primitivos que se comportam como objetos imutáveis ​​(números, strings, booleanos). Suas propriedades ou as propriedades de seu protótipo podem ser acessadas usando a notação de ponto (obj.prop) ou colchete (obj['prop']). Exceções notáveis ​​são os valores especiais undefined e null.

Os objetos são mantidos por referência em JavaScript, não por valor. Isso significa que quando copiados ou passados ​​como argumentos para funções, a “cópia” e o original são referências ao mesmo objeto, e uma alteração nas propriedades de um alterará a mesma propriedade do outro. Isso não se aplica a primitivas, que são imutáveis ​​e passadas por valor.

Clonagem superficial #

A função Object.assign() do ES6 pode ser usada para copiar todas as propriedades enumerable de uma instância Object existente para uma nova.

const existing = { a: 1, b: 2, c: 3 };

const clone = Object.assign({}, existing);

Isso inclui propriedades Symbol além das String.

Object rest/spread destructuring, que atualmente é uma proposta de estágio 3, fornece uma maneira ainda mais simples de criar clones superficiais de instâncias de Object:

const existing = { a: 1, b: 2, c: 3 };

const { ...clone } = existing;

Se você precisar oferecer suporte a versões mais antigas de JavaScript, a maneira mais compatível de clonar um objeto é iterando manualmente suas propriedades e filtrando as herdadas usando .hasOwnProperty().

var existing = { a: 1, b: 2, c: 3 };

var clone = {};
for (var prop in existing) {
  if (existing.hasOwnProperty(prop)) {
    clone[prop] = existing[prop];
  }
}

Objeto.congelar #

Object.freeze torna um objeto imutável impedindo a adição de novas propriedades, a remoção de propriedades existentes e a modificação da enumerabilidade, configurabilidade e capacidade de escrita de propriedades existentes. Também impede que o valor das propriedades existentes seja alterado. No entanto, ele não funciona recursivamente, o que significa que os objetos filho não são congelados automaticamente e estão sujeitos a alterações.

As operações após o congelamento falharão silenciosamente, a menos que o código esteja sendo executado no modo estrito. Se o código estiver em modo estrito, um TypeError será lançado.

var obj = {
  foo: 'foo',
  bar: [1, 2, 3],
  baz: {
    foo: 'nested-foo'
  }
};

Object.freeze(obj);

// Cannot add new properties
obj.newProperty = true;

// Cannot modify existing values or their descriptors
obj.foo = 'not foo';
Object.defineProperty(obj, 'foo', {
    writable: true
});

// Cannot delete existing properties
delete obj.foo;

// Nested objects are not frozen
obj.bar.push(4);
obj.baz.foo = 'new foo';

Clonagem de objetos #

Quando você deseja uma cópia completa de um objeto (ou seja, as propriedades do objeto e os valores dentro dessas propriedades, etc…), isso é chamado de clonagem profunda.

Se um objeto puder ser serializado para JSON, você poderá criar um clone profundo dele com uma combinação de JSON.parse e ​​JSON.stringify:

var existing = { a: 1, b: { c: 2 } };
var copy = JSON.parse(JSON.stringify(existing));
existing.b.c = 3; // copy.b.c will not change

Observe que JSON.stringify converterá objetos Date em representações de string no formato ISO, mas JSON.parse não converterá a string de volta em uma Date.

Não há função interna no JavaScript para criar clones profundos e, em geral, não é possível criar clones profundos para cada objeto por vários motivos. Por exemplo,

  • os objetos podem ter propriedades não enumeráveis ​​e ocultas que não podem ser detectadas.
  • getters e setters de objetos não podem ser copiados.
  • os objetos podem ter uma estrutura cíclica.
  • as propriedades da função podem depender do estado em um escopo oculto.

Supondo que você tenha um objeto “legal” cujas propriedades contenham apenas valores primitivos, datas, matrizes ou outros objetos “legais”, a função a seguir pode ser usada para fazer clones profundos. É uma função recursiva que pode detectar objetos com uma estrutura cíclica e gerará um erro nesses casos.

function deepClone(obj) {
    function clone(obj, traversedObjects) {
        var copy;
        // primitive types
        if(obj === null || typeof obj !== "object") {
            return obj;
        }

        // detect cycles
        for(var i = 0; i < traversedObjects.length; i++) {
            if(traversedObjects[i] === obj) {
                throw new Error("Cannot clone circular object.");
            }
        }

        // dates
        if(obj instanceof Date) {
            copy = new Date();
            copy.setTime(obj.getTime());
            return copy;
        }
        // arrays
        if(obj instanceof Array) {
            copy = [];
            for(var i = 0; i < obj.length; i++) {
                copy.push(clone(obj[i], traversedObjects.concat(obj)));
            }
            return copy;
        }
        // simple objects
        if(obj instanceof Object) {
            copy = {};
            for(var key in obj) {
                if(obj.hasOwnProperty(key)) {
                    copy[key] = clone(obj[key], traversedObjects.concat(obj));
                }
            }
            return copy;
        }
        throw new Error("Not a cloneable object.");
    }

    return clone(obj, []);
}

Iteração de propriedades do objeto #

Você pode acessar cada propriedade que pertence a um objeto com este loop

for (var property in object) {
    // always check if an object has a property
    if (object.hasOwnProperty(property)) {
        // do stuff
    }
}

Você deve incluir a verificação adicional para hasOwnProperty porque um objeto pode ter propriedades que são herdadas da classe base do objeto. Não realizar esta verificação pode gerar erros.

Você também pode usar a função Object.keys que retorna um Array contendo todas as propriedades de um objeto e então você pode percorrer este array com a função Array.map ou Array.forEach.

var obj = { 0: 'a', 1: 'b', 2: 'c' };

Object.keys(obj).map(function(key) {
    console.log(key);
}); 
// outputs: 0, 1, 2

Objeto.atribuir #

O método Object.assign() é usado para copiar os valores de todas as propriedades próprias enumeráveis ​​de um ou mais objetos de origem para um objeto de destino. Ele retornará o objeto de destino.

Use-o para atribuir valores a um objeto existente:

var user = {
    firstName: "John"
};

Object.assign(user, {lastName: "Doe", age:39});
console.log(user); // Logs: {firstName: "John", lastName: "Doe", age: 39}

Ou para criar uma cópia superficial de um objeto:

var obj = Object.assign({}, user);

console.log(obj); // Logs: {firstName: "John", lastName: "Doe", age: 39}

Ou mescle muitas propriedades de vários objetos em um:

var obj1 = {
    a: 1
};
var obj2 = {
    b: 2
};
var obj3 = {
    c: 3
};
var obj = Object.assign(obj1, obj2, obj3);

console.log(obj); // Logs: { a: 1, b: 2, c: 3 }
console.log(obj1); // Logs: { a: 1, b: 2, c: 3 }, target object itself is changed

Primitivos serão encapsulados, null e undefined serão ignorados:

var var_1 = 'abc';
var var_2 = true;
var var_3 = 10;
var var_4 = Symbol('foo');

var obj = Object.assign({}, var_1, null, var_2, undefined, var_3, var_4);
console.log(obj); // Logs: { "0": "a", "1": "b", "2": "c" }

Observe que apenas os wrappers de string podem ter propriedades enumeráveis ​​próprias

Use-o como redutor: (mescla um array para um objeto)

return users.reduce((result, user) => Object.assign({}, {[user.id]: user})

Descanso/disseminação do objeto (…) #

A propagação de objetos é apenas açúcar sintático para Object.assign({}, obj1, ..., objn);

Isso é feito com o operador ...:

let obj = { a: 1 };

let obj2 = { ...obj, b: 2, c: 3 };

console.log(obj2); // { a: 1, b: 2, c: 3 };

Como Object.assign ele faz uma mesclagem superficial, não uma mesclagem profunda.

let obj3 = { ...obj, b: { c: 2 } };

console.log(obj3); // { a: 1, b: { c: 2 } };

OBSERVAÇÃO: Esta especificação está atualmente em estágio 3

Object.defineProperty #

Ele nos permite definir uma propriedade em um objeto existente usando um descritor de propriedade.

var obj = { };

Object.defineProperty(obj, 'foo', { value: 'foo' });

console.log(obj.foo);

Saída do console

foo

Object.defineProperty pode ser chamado com as seguintes opções:

Object.defineProperty(obj, 'nameOfTheProperty', {
  value: valueOfTheProperty, 
  writable: true, // if false, the property is read-only
  configurable : true, // true means the property can be changed later
  enumerable : true // true means property can be enumerated such as in a for..in loop
});

Object.defineProperties permite que você defina várias propriedades ao mesmo tempo.

var obj = {};
Object.defineProperties(obj, {
  property1: {
    value: true,
    writable: true
  },
  property2: {
    value: 'Hello',
    writable: false
  }      
});

Propriedades de acesso (obter e definir) #

Trate uma propriedade como uma combinação de duas funções, uma para obter o valor dela e outra para definir o valor nela.

A propriedade get do descritor de propriedade é uma função que será chamada para recuperar o valor da propriedade.

A propriedade set também é uma função, ela será chamada quando a propriedade tiver um valor atribuído, e o novo valor será passado como um argumento.

Você não pode atribuir um value ou writable a um descritor que tenha get ou set

var person = { name: "John", surname: "Doe"};
Object.defineProperty(person, 'fullName', { 
    get: function () { 
        return this.name + " " + this.surname;
    },
    set: function (value) {
        [this.name, this.surname] = value.split(" ");
    }
});

console.log(person.fullName); // -> "John Doe"

person.surname = "Hill";
console.log(person.fullName); // -> "John Hill"

person.fullName = "Mary Jones";
console.log(person.name) // -> "Mary"

Nomes de propriedades dinâmicas/variáveis #

Às vezes, o nome da propriedade precisa ser armazenado em uma variável. Neste exemplo, perguntamos ao usuário qual palavra precisa ser pesquisada e, em seguida, fornecemos o resultado de um objeto que chamei de dicionário.

var dictionary = {
    lettuce: 'a veggie',
    banana: 'a fruit',
    tomato: 'it depends on who you ask',
    apple: 'a fruit',
    Apple: 'Steve Jobs rocks!' // properties are case-sensitive
}

var word = prompt('What word would you like to look up today?')
var definition = dictionary[word]
alert(word + '\n\n' + definition)

Observe como estamos usando a notação de colchetes [] para observar a variável chamada word; se fôssemos usar a notação tradicional ., então levaria o valor literalmente, portanto:

console.log(dictionary.word)  // doesn't work because word is taken literally and dictionary has no field named `word`
console.log(dictionary.apple) // it works! because apple is taken literally

console.log(dictionary[word]) // it works! because word is a variable, and the user perfectly typed in one of the words from our dictionary when prompted
console.log(dictionary[apple]) // error! apple is not defined (as a variable)

Você também pode escrever valores literais com notação [] substituindo a variável word por uma string 'apple'. Veja o exemplo [Propriedades com caracteres especiais ou palavras reservadas].

Você também pode definir propriedades dinâmicas com a sintaxe de colchetes:

var property="test";
var obj={
 [property]=1;
};

console.log(obj.test);//1

Faz o mesmo que:

var property="test";
var obj={};
obj[property]=1;

Arrays são objetos #

Isenção de responsabilidade: não é recomendado criar objetos semelhantes a matrizes. No entanto, é útil entender como eles funcionam, especialmente ao trabalhar com DOM. Isso explicará por que as operações regulares de array não funcionam em objetos DOM retornados de muitas funções de document do DOM. (ou seja, querySelectorAll, form.elements)

Supondo que criamos o seguinte objeto que tem algumas propriedades que você esperaria ver em um Array.

var anObject = {
    foo: 'bar',
    length: 'interesting',
    '0': 'zero!',
    '1': 'one!'
};

Então vamos criar um array.

var anArray = ['zero.', 'one.'];

Agora, observe como podemos inspecionar o objeto e o array da mesma maneira.

console.log(anArray[0], anObject[0]); // outputs: zero.  zero!
console.log(anArray[1], anObject[1]); // outputs: one.  one!
console.log(anArray.length, anObject.length); // outputs: 2 interesting
console.log(anArray.foo, anObject.foo); // outputs: undefined bar

Como anArray é na verdade um objeto, assim como anObject, podemos até adicionar propriedades wordy personalizadas a anArray

Isenção de responsabilidade: Arrays com propriedades personalizadas geralmente não são recomendados, pois podem ser confusos, mas podem ser úteis em casos avançados em que você precisa das funções otimizadas de um Array. (ou seja, objetos jQuery)

anArray.foo = 'it works!';
console.log(anArray.foo);

Podemos até fazer com que anObject seja um objeto do tipo array adicionando um length.

anObject.length = 2;

Então você pode usar o loop for no estilo C para iterar sobre anObject como se fosse um Array. Consulte Iteração de matriz

Observe que anObject é apenas um objeto tipo array. (também conhecido como List) Não é um Array verdadeiro. Isso é importante, porque funções como push e forEach (ou qualquer função de conveniência encontrada em Array.prototype) não funcionarão por padrão em objetos do tipo array.

Muitas das funções document do DOM retornarão uma lista (ou seja, querySelectorAll, form.elements) que é semelhante ao anObject do tipo array que criamos acima. Consulte https://www.wikiod.com/pt/javascript/matrizes#Convertendo objetos semelhantes a matrizes em matrizes

console.log(typeof anArray == 'object', typeof anObject == 'object'); // outputs: true  true
console.log(anArray instanceof Object, anObject instanceof Object); // outputs: true  true
console.log(anArray instanceof Array, anObject instanceof Array); // outputs: true  false
console.log(Array.isArray(anArray), Array.isArray(anObject)); // outputs: true  false

Object.seal #

Object.seal impede a adição ou remoção de propriedades de um objeto. Depois que um objeto é selado, seus descritores de propriedade não podem ser convertidos em outro tipo. Ao contrário de Object.freeze ele permite que as propriedades sejam editadas.

As tentativas de fazer essas operações em um objeto selado falharão silenciosamente

var obj = { foo: 'foo', bar: function () { return 'bar'; } };

Object.seal(obj)

obj.newFoo = 'newFoo';
obj.bar = function () { return 'foo' };

obj.newFoo; // undefined
obj.bar(); // 'foo'

// Can't make foo an accessor property
Object.defineProperty(obj, 'foo', { 
    get: function () { return 'newFoo'; }
}); // TypeError

// But you can make it read only
Object.defineProperty(obj, 'foo', { 
    writable: false
}); // TypeError

obj.foo = 'newFoo';
obj.foo; // 'foo';

No modo estrito, essas operações lançarão um TypeError

(function () {
    'use strict';

    var obj = { foo: 'foo' };

    Object.seal(obj);

    obj.newFoo = 'newFoo'; // TypeError
}());

Recuperando propriedades de um objeto #

Características dos imóveis: #

Propriedades que podem ser recuperadas de um objeto podem ter as seguintes características,

  • Enumerável
  • Não - Enumerável
  • ter

Ao criar as propriedades usando Object.defineProperty(s), podemos definir suas características, exceto “own”. Propriedades que estão disponíveis no nível direto e não no nível protótipo (__proto__) de um objeto são chamadas de propriedades próprias.

E as propriedades que são adicionadas em um objeto sem usar Object.defindProperty(s) não terão sua característica enumerável. Isso significa que deve ser considerado como verdadeiro.

Objetivo da enumerabilidade: #

O objetivo principal de definir características enumeráveis ​​para uma propriedade é tornar a propriedade específica disponível ao recuperá-la de seu objeto, usando diferentes métodos programáticos. Esses diferentes métodos serão discutidos profundamente abaixo.

Métodos de recuperação de propriedades: #

As propriedades de um objeto podem ser recuperadas pelos seguintes métodos,

  1. for..in loop

    This loop is very useful in retrieving enumerable properties from an object. Additionally this loop will retrieve enumerable own properties as well as it will do the same retrieval by traversing through the prototype chain until it sees the prototype as null.

     //Ex 1 : Simple data
 var x = { a : 10 , b : 3} , props = [];

 for(prop in x){
   props.push(prop);
 }

 console.log(props); //["a","b"]

 //Ex 2 : Data with enumerable properties in prototype chain
 var x = { a : 10 , __proto__ : { b : 10 }} , props = [];

 for(prop in x){
   props.push(prop);
 }

 console.log(props); //["a","b"]

 //Ex 3 : Data with non enumerable properties
 var x = { a : 10 } , props = [];
 Object.defineProperty(x, "b", {value : 5, enumerable : false});

 for(prop in x){
   props.push(prop);
 }

 console.log(props); //["a"]

  2. Função Object.keys()

    This function was unveiled as a part of EcmaScript 5. It is used to retrieve enumerable own properties from an object. Prior to its release people used to retrieve own properties from an object by combining for..in loop and Object.prototype.hasOwnProperty() function.

     //Ex 1 : Simple data
 var x = { a : 10 , b : 3} , props;

 props = Object.keys(x);

 console.log(props); //["a","b"]

 //Ex 2 : Data with enumerable properties in prototype chain
 var x = { a : 10 , __proto__ : { b : 10 }} , props;

 props = Object.keys(x);

 console.log(props); //["a"]

 //Ex 3 : Data with non enumerable properties
 var x = { a : 10 } , props;
 Object.defineProperty(x, "b", {value : 5, enumerable : false});

 props = Object.keys(x);

 console.log(props); //["a"]

  3. Função Object.getOwnProperties()

    This function will retrieve both enumerable and non enumerable, own properties from an object. It was also released as a part of EcmaScript 5.

     //Ex 1 : Simple data
 var x = { a : 10 , b : 3} , props;

 props = Object.getOwnPropertyNames(x);

 console.log(props); //["a","b"]

 //Ex 2 : Data with enumerable properties in prototype chain
 var x = { a : 10 , __proto__ : { b : 10 }} , props;

 props = Object.getOwnPropertyNames(x);

 console.log(props); //["a"]

 //Ex 3 : Data with non enumerable properties
 var x = { a : 10 } , props;
 Object.defineProperty(x, "b", {value : 5, enumerable : false});

 props = Object.getOwnPropertyNames(x);

 console.log(props); //["a", "b"]

Diversos : #

Uma técnica para recuperar todas as propriedades (próprias, enumeráveis, não enumeráveis, todas no nível de protótipo) de um objeto é fornecida abaixo,

function getAllProperties(obj, props = []){
  return obj == null ? props :
           getAllProperties(Object.getPrototypeOf(obj),
               props.concat(Object.getOwnPropertyNames(obj)));
}

var x = {a:10, __proto__ : { b : 5, c : 15 }};

//adding a non enumerable property to first level prototype
Object.defineProperty(x.__proto__, "d", {value : 20, enumerable : false});

console.log(getAllProperties(x)); ["a", "b", "c", "d", "...other default core props..."]

E isso será suportado pelos navegadores que suportam EcmaScript 5.

Dado este objeto:

var obj = {

    c: "javascript!",

Você pode converter seus valores em uma matriz fazendo:

var array = Object.keys(obj)

Propriedade somente leitura #

Usando descritores de propriedade, podemos tornar uma propriedade somente leitura, e qualquer tentativa de alterar seu valor falhará silenciosamente, o valor não será alterado e nenhum erro será lançado.

A propriedade writable em um descritor de propriedade indica se essa propriedade pode ser alterada ou não.

var a  = { };

Object.defineProperty(a, 'foo', { value: 'original', writable: false });

a.foo = 'new';

console.log(a.foo);

Saída do console

original

Propriedade não enumerável #

Podemos evitar que uma propriedade apareça em loops for (... in ...)

A propriedade enumerable do descritor de propriedade informa se essa propriedade será enumerada durante o loop pelas propriedades do objeto.

var obj = { };

Object.defineProperty(obj, "foo", { value: 'show', enumerable: true });
Object.defineProperty(obj, "bar", { value: 'hide', enumerable: false });

for (var prop in obj) {
    console.log(obj[prop]);
}

Saída do console

mostrar

Descrição da propriedade de bloqueio #

O descritor de uma propriedade pode ser bloqueado para que nenhuma alteração possa ser feita. Ainda será possível usar a propriedade normalmente, atribuindo e recuperando o valor dela, mas qualquer tentativa de redefini-la lançará uma exceção.

A propriedade configurable do descritor de propriedade é usada para impedir qualquer alteração adicional no descritor.

var obj = {};

// Define 'foo' as read only and lock it
Object.defineProperty(obj, "foo", { 
    value: "original value", 
    writable: false, 
    configurable: false
});
 
Object.defineProperty(obj, "foo", {writable: true});

Este erro será lançado:

TypeError: Não é possível redefinir a propriedade: foo

E a propriedade ainda será somente leitura.

obj.foo = "new value";
console.log(foo);

Saída do console

valor original

Descritores e propriedades nomeadas #

Propriedades são membros de um objeto. Cada propriedade nomeada é um par de (nome, descritor). O nome é uma string que permite o acesso (usando a notação de ponto object.propertyName ou a notação de colchetes object['propertyName']). O descritor é um registro de campos que define o comportamento da propriedade quando ela é acessada (o que acontece com a propriedade e qual é o valor retornado ao acessá-la). Em geral, uma propriedade associa um nome a um comportamento (podemos pensar no comportamento como uma caixa preta).

Existem dois tipos de propriedades nomeadas:

  1. propriedade de dados: o nome da propriedade está associado a um valor.
  2. propriedade de acesso: o nome da propriedade está associado a uma ou duas funções de acesso.

Demonstração:

obj.propertyName1 = 5; //translates behind the scenes into
                       //either assigning 5 to the value field* if it is a data property
                //or calling the set function with the parameter 5 if accessor property


//*actually whether an assignment would take place in the case of a data property
//also depends on the presence and value of the writable field - on that later on

O tipo da propriedade é determinado pelos campos de seu descritor e uma propriedade não pode ser dos dois tipos.

Descritores de dados -

  • Campos obrigatórios: value ou writable ou ambos
  • Campos opcionais:configurável,enumerable

Amostra:

{
   value: 10,
   writable: true;
}

Descritores de acessor -

  • Campos obrigatórios: get ou set ou ambos
  • Campos opcionais: configurable, enumerable

Amostra:

{
    get: function () {
        return 10;
    },
    enumerable: true
}

significado dos campos e seus padrões

`configurable`,`enumerable` e `writable`:
  • Todas essas chaves assumem como padrão false.
  • configurable é true se e somente se o tipo deste descritor de propriedade puder ser alterado e se a propriedade puder ser excluída do objeto correspondente.
  • enumerable é true se e somente se esta propriedade aparece durante a enumeração das propriedades no objeto correspondente.
  • writable é true se e somente se o valor associado à propriedade puder ser alterado com um operador de atribuição.

get e set:

  • Essas chaves padronizam para undefined.
  • get é uma função que serve como getter para a propriedade, ou undefined se não houver getter. A função return será usada como o valor da propriedade.
  • set é uma função que serve como setter para a propriedade, ou undefined se não houver setter. A função receberá como único argumento o novo valor que está sendo atribuído à propriedade.

valor:

  • Esta chave tem como padrão undefined.
  • O valor associado ao imóvel. Pode ser qualquer valor JavaScript válido (número, objeto, função etc.).

Exemplo:

    var obj = {propertyName1: 1}; //the pair is actually ('propertyName1', {value:1,
                                                                    // writable:true,
                                                                    // enumerable:true,
                                                                    // configurable:true})
    Object.defineProperty(obj, 'propertyName2', {get: function() {
                                                    console.log('this will be logged ' + 
                                 'every time propertyName2 is accessed to get its value');
                                                },
                                            set: function() {
                                                    console.log('and this will be logged ' + 
                                'every time propertyName2\'s value is tried to be set')
                      //will be treated like it has enumerable:false, configurable:false
                                                }});
//propertyName1 is the name of obj's data property 
//and propertyName2 is the name of its accessor property





obj.propertyName1 = 3;
console.log(obj.propertyName1); //3

obj.propertyName2 = 3; //and this will be logged every time propertyName2's value is tried to be set
console.log(obj.propertyName2); //this will be logged every time propertyName2 is accessed to get its value

Propriedades com caracteres especiais ou palavras reservadas #

Embora a notação de propriedade de objeto seja geralmente escrita como myObject.property, isso só permitirá caracteres que são normalmente encontrados em [nomes de variáveis ​​JavaScript](http://stackoverflow.com/questions/1661197/what-characters-are-valid- for-javascript-variable-names), que é principalmente letras, números e sublinhado (_).

Se você precisar de caracteres especiais, como espaço, ☺ ou conteúdo fornecido pelo usuário, isso é possível usando a notação de colchetes [].

myObject['special property ☺'] = 'it works!'
console.log(myObject['special property ☺'])

Propriedades de todos os dígitos: #

Além dos caracteres especiais, os nomes de propriedade que são todos dígitos exigirão a notação de colchetes. No entanto, neste caso, a propriedade não precisa ser escrita como uma string.

myObject[123] = 'hi!' // number 123 is automatically converted to a string
console.log(myObject['123']) // notice how using string 123 produced the same result
console.log(myObject['12' + '3']) // string concatenation
console.log(myObject[120 + 3]) // arithmetic, still resulting in 123 and producing the same result
console.log(myObject[123.0]) // this works too because 123.0 evaluates to 123
console.log(myObject['123.0']) // this does NOT work, because '123' != '123.0'

No entanto, zeros à esquerda não são recomendados, pois são interpretados como notação octal. (TODO, devemos produzir e vincular a um exemplo descrevendo a notação octal, hexadecimal e expoente)

Criando um objeto Iterável #

var myIterableObject = {};
// An Iterable object must define a method located at the Symbol.iterator key:
myIterableObject[Symbol.iterator] = function () {
  // The iterator should return an Iterator object
  return {
    // The Iterator object must implement a method, next()
    next: function () {
      // next must itself return an IteratorResult object 
      if (!this.iterated) {
        this.iterated = true;
        // The IteratorResult object has two properties
        return {
          // whether the iteration is complete, and
          done: false,
          // the value of the current iteration
          value: 'One'
        };
      }
      return {
        // When iteration is complete, just the done property is needed
        done: true
      };
    },
    iterated: false
  };
};

for (var c of myIterableObject) {
  console.log(c); 
}

Saída do console

Um

Iterando sobre entradas de objetos - Object.entries() #

O método proposto Object.entries() retorna uma matriz de pares chave/valor para o objeto fornecido. Ele não retorna um iterador como [Array.prototype.entries()](https://www.wikiod.com/pt/javascript/matrizes#O método entradas()), mas o Array retornado por Object.entries() pode ser iterado independentemente.

const obj = {
    one: 1,
    two: 2,
    three: 3
};

Object.entries(obj);

Resulta em:

[
    ["one", 1],
    ["two", 2],
    ["three", 3]
]

É uma maneira útil de iterar sobre os pares chave/valor de um objeto:

for(const [key, value] of Object.entries(obj)) {
    console.log(key); // "one", "two" and "three"
    console.log(value); // 1, 2 and 3
}

Object.values() #

O método Object.values() retorna uma matriz dos próprios valores de propriedade enumeráveis ​​de um determinado objeto, na mesma ordem fornecida por um loop for…in (a diferença é que um loop for-in enumera propriedades no cadeia de protótipos também).

var obj = { 0: 'a', 1: 'b', 2: 'c' };
console.log(Object.values(obj)); // ['a', 'b', 'c']
