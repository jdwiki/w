Sintaxe #

  • window.onerror = function (eventOrMessage, url, lineNumber, colNumber, erro) { … }

Parâmetros #

Parâmetro Detalhes
eventOrMessage Alguns navegadores chamarão o manipulador de eventos com apenas um argumento, um objeto Event. No entanto, outros navegadores, especialmente os mais antigos e os mais antigos, fornecerão uma mensagem String como primeiro argumento.
URL Se um manipulador for chamado com mais de 1 argumento, o segundo argumento geralmente é uma URL de um arquivo JavaScript que é a origem do problema.
linhaNúmero Se um manipulador for chamado com mais de 1 argumento, o terceiro argumento será um número de linha dentro do arquivo de origem JavaScript.
colNumber Se um manipulador for chamado com mais de 1 argumento, o quarto argumento será o número da coluna dentro do arquivo de origem JavaScript.
erro Se um manipulador for chamado com mais de 1 argumento, o quinto argumento às vezes é um objeto ‘Erro’ descrevendo o problema.

Infelizmente, o window.onerror tem sido implementado historicamente de forma diferente por cada fornecedor. As informações fornecidas na seção Parâmetros são uma aproximação do que esperar em diferentes navegadores e suas versões.

Manipulando window.onerror para relatar todos os erros de volta ao lado do servidor #

O exemplo a seguir escuta o evento window.onerror e usa uma técnica de beacon de imagem para enviar as informações por meio dos parâmetros GET de uma URL.

var hasLoggedOnce = false;

// Some browsers (at least Firefox) don't report line and column numbers
// when event is handled through window.addEventListener('error', fn). That's why
// a more reliable approach is to set an event listener via direct assignment.
window.onerror = function (eventOrMessage, url, lineNumber, colNumber, error) {
    if (hasLoggedOnce || !eventOrMessage) {
        // It does not make sense to report an error if:
        // 1. another one has already been reported -- the page has an invalid state and may produce way too many errors.
        // 2. the provided information does not make sense (!eventOrMessage -- the browser didn't supply information for some reason.)
        return;
    }
    hasLoggedOnce = true;
    if (typeof eventOrMessage !== 'string') {
        error = eventOrMessage.error;
        url = eventOrMessage.filename || eventOrMessage.fileName;
        lineNumber = eventOrMessage.lineno || eventOrMessage.lineNumber;
        colNumber = eventOrMessage.colno || eventOrMessage.columnNumber;
        eventOrMessage = eventOrMessage.message || eventOrMessage.name || error.message || error.name;
    }
    if (error && error.stack) {
        eventOrMessage = [eventOrMessage, '; Stack: ', error.stack, '.'].join('');
    }
    var jsFile = (/[^/]+\.js/i.exec(url || '') || [])[0] || 'inlineScriptOrDynamicEvalCode',
        stack = [eventOrMessage, ' Occurred in ', jsFile, ':', lineNumber || '?', ':', colNumber || '?'].join('');

    // shortening the message a bit so that it is more likely to fit into browser's URL length limit (which is 2,083 in some browsers)
    stack = stack.replace(/https?\:\/\/[^/]+/gi, '');
    // calling the server-side handler which should probably register the error in a database or a log file
    new Image().src = '/exampleErrorReporting?stack=' + encodeURIComponent(stack);
    
    // window.DEBUG_ENVIRONMENT a configurable property that may be set to true somewhere else for debugging and testing purposes.
    if (window.DEBUG_ENVIRONMENT) {
        alert('Client-side script failed: ' + stack);
    }
}
