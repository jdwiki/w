Sintaxe #

  • var foo [= valor [, foo2 [, foo3 … [, fooN]]]]];
  • let bar [= valor [, bar2 [, foo3 … [, barN]]]]];
  • const baz = valor [, baz2 = valor2 [, … [, bazN = valorN]]];

Modificando constantes #

Declarando e inicializando constantes #

Declaração #

Existem quatro maneiras principais de declarar uma variável em JavaScript: usando as palavras-chave var, let ou const, ou sem nenhuma palavra-chave (declaração “bare”). O método usado determina o escopo resultante da variável, ou reatribuição no caso de const.

  • A palavra-chave var cria uma variável de escopo de função.

  • A palavra-chave let cria uma variável de escopo de bloco.

  • A palavra-chave const cria uma variável de escopo de bloco que não pode ser reatribuída.

  • Uma declaração simples cria uma variável global.

    var a = ‘foo’; // Function-scope let b = ‘foo’; // Block-scope const c = ‘foo’; // Block-scope & immutable reference

Tenha em mente que você não pode declarar constantes sem inicializá-las ao mesmo tempo.

const foo; // "Uncaught SyntaxError: Missing initializer in const declaration"

(Um exemplo de declaração de variável sem palavra-chave não está incluído acima por motivos técnicos. Continue lendo para ver um exemplo.)

Tipos de dados #

Variáveis ​​JavaScript podem conter muitos tipos de dados: números, strings, arrays, objetos e mais:

// Number
var length = 16;

// String
var message = "Hello, World!"; 

// Array
var carNames = ['Chevrolet', 'Nissan', 'BMW']; 

// Object
var person = {
    firstName: "John",
    lastName: "Doe"
};

JavaScript tem tipos dinâmicos. Isso significa que a mesma variável pode ser usada como tipos diferentes:

var a;              // a is undefined
var a = 5;          // a is a Number
var a = "John";     // a is a String

Indefinido #

A variável declarada sem valor terá o valor undefined

var a;

console.log(a); // logs: undefined

Tentar recuperar o valor de variáveis ​​não declaradas resulta em um ReferenceError. No entanto, tanto o tipo de variáveis ​​não declaradas quanto as unitializadas são “indefinidas”:

var a;
console.log(typeof a === "undefined"); // logs: true
console.log(typeof variableDoesNotExist === "undefined"); // logs: true

Atribuição #

Para atribuir um valor a uma variável declarada anteriormente, use o operador de atribuição, =:

a = 6;
b = "Foo";

Como alternativa à declaração e atribuição independentes, é possível realizar ambas as etapas em uma instrução:

var a = 6;
let b = "Foo";

É nesta sintaxe que as variáveis ​​globais podem ser declaradas sem palavra-chave; se alguém fosse declarar uma variável nua sem uma atribuição imediatamente após a palavra, o interpretador não seria capaz de diferenciar declarações globais a; de referências a variáveis ​​a;.

c = 5;
c = "Now the value is a String.";
myNewGlobal;    // ReferenceError

Observe, no entanto, que a sintaxe acima geralmente é desencorajada e não é compatível com o modo estrito. Isso é para evitar o cenário em que um programador inadvertidamente descarta uma palavra-chave let ou var de sua instrução, criando acidentalmente uma variável no namespace global sem perceber. Isso pode poluir o namespace global e entrar em conflito com bibliotecas e o funcionamento adequado de um script. Portanto, as variáveis ​​globais devem ser declaradas e inicializadas usando a palavra-chave var no contexto do objeto window, para que a intenção seja declarada explicitamente.

Além disso, as variáveis ​​podem ser declaradas várias de cada vez, separando cada declaração (e atribuição de valor opcional) com uma vírgula. Usando essa sintaxe, as palavras-chave var e let precisam ser usadas apenas uma vez no início de cada instrução.

globalA = "1", globalB = "2";
let x, y = 5;
var person = 'John Doe',
    foo,
    age = 14,
    date = new Date();

Observe no trecho de código anterior que a ordem na qual as expressões de declaração e atribuição ocorrem (var a, b, c = 2, d;) não importa. Você pode misturar livremente os dois.

Declaração de função efetivamente cria variáveis ​​também.

Operações matemáticas e atribuição #

Incrementar por #

var a = 9,  
b = 3;  
b += a;

b agora será 12

Isso é funcionalmente o mesmo que

b = b + a;

Diminuir por #

var a = 9,  
b = 3;  
b -= a;

b agora será 6

Isso é funcionalmente o mesmo que

b = b - a;

Multiplique por #

var a = 5,  
b = 3;  
b *= a;

b agora será 15

Isso é funcionalmente o mesmo que

b = b * a;

Dividido por #

var a = 3,  
b = 15;  
b /= a;

b agora será 5

Isso é funcionalmente o mesmo que

b = b / a;

Elevado ao poder de #

var a = 3,  
b = 15;  
b **= a;

b agora será 3375

Isso é funcionalmente o mesmo que

b = b ** a;
