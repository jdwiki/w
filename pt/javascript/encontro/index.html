Sintaxe #

  • Nova data();
  • nova Data(valor);
  • new Date(dateAsString);
  • new Date(ano, mês[, dia[, hora[, minuto[, segundo[, milissegundo]]]]]);

Parâmetros #

Parâmetro Detalhes
valor O número de milissegundos desde 1 de janeiro de 1970 00:00:00.000 UTC (época Unix)
dateAsString Uma data formatada como uma string (consulte os exemplos para obter mais informações)
ano O valor do ano da data. Observe que month também deve ser fornecido, ou o valor será interpretado como um número de milissegundos. Observe também que valores entre 0 e 99 têm um significado especial. Veja os exemplos.
mês O mês, no intervalo ‘0-11’. Observe que usar valores fora do intervalo especificado para isso e os parâmetros a seguir não resultará em erro, mas fará com que a data resultante “passe” para o próximo valor. Veja os exemplos.
dia Opcional: a data, no intervalo “1-31”.
hora Opcional: a hora, no intervalo “0-23”.
minuto Opcional: o minuto, no intervalo ‘0-59’.
segundo Opcional: O segundo, no intervalo 0-59.
milissegundo Opcional: o milissegundo, no intervalo 0-999.

Converter para um formato de string #

Converte para String #

var date1 = new Date();
date1.toString();

Retorna: “Sexta-feira, 15 de abril de 2016 07:48:48 GMT-0400 (horário de verão)”

Converte para Time String #

var date1 = new Date();
date1.toTimeString();

Retorna: “07:48:48 GMT-0400 (Horário de Verão do Leste)”

Converter para string de data #

var date1 = new Date();
date1.toDateString();

Retorna: “Qui 14 de abril de 2016”

Converter para String UTC #

var date1 = new Date();
date1.toUTCString();

Retorna: “Sex, 15 de abril de 2016 11:48:48 GMT”

Converte para String ISO #

var date1 = new Date();
date1.toISOString();

Retorna: “2016-04-14T23:49:08.596Z”

Converter para String GMT #

var date1 = new Date();
date1.toGMTString();

Retorna: “Qui, 14 de abril de 2016 23:49:08 GMT”

Esta função foi marcada como obsoleta para que alguns navegadores não a suportem no futuro. Sugere-se usar toUTCString() em vez disso.

Converter para string de data de localidade #

var date1 = new Date();
date1.toLocaleDateString();

Retorna: “14/04/2016”

Essa função retorna uma string de data sensível à localidade com base na localização do usuário por padrão.

date1.toLocaleDateString([locales [, options]])

pode ser usado para fornecer localidades específicas, mas é específico da implementação do navegador. Por exemplo,

date1.toLocaleDateString(["zh", "en-US"]);

tentaria imprimir a string na localidade chinesa usando o inglês dos Estados Unidos como substituto. O parâmetro options pode ser usado para fornecer formatação específica. Por exemplo:

var options = { weekday: 'long', year: 'numeric', month: 'long', day: 'numeric' };
date1.toLocaleDateString([], options);

resultaria em

“Quinta-feira, 14 de abril de 2016”.

Veja o MDN para mais detalhes.

Crie um novo objeto Date #

Para criar um novo objeto Date use o construtor Date():

  • sem argumentos

Date() cria uma instância Date contendo a hora atual (até milissegundos) e data.

  • com um argumento inteiro

Date(m) cria uma instância Date contendo a hora e a data correspondentes ao horário Epoch (1 de janeiro de 1970 UTC) mais m milissegundos. Exemplo: new Date(749019369738) fornece a data Dom, 26 de setembro de 1993 04:56:09 GMT.

  • com um argumento de string

Date(dateString) retorna o objeto Date que resulta após analisar dateString com Date.parse.

  • com dois ou mais argumentos inteiros

Date(i1, i2, i3, i4, i5, i6) lê os argumentos como ano, mês, dia, horas, minutos, segundos, milissegundos e instancia o objeto Date correspondente. Observe que o mês é indexado em 0 em JavaScript, portanto, 0 significa janeiro e 11 significa dezembro. Exemplo: new Date(2017, 5, 1) dá 1 de junho de 2017.

Explorando datas #

Observe que esses exemplos foram gerados em um navegador no fuso horário central dos EUA, durante o horário de verão, conforme evidenciado pelo código. Onde a comparação com UTC foi instrutiva, Date.prototype.toISOString() foi usado para mostrar a data e hora em UTC (o Z na string formatada denota UTC).

// Creates a Date object with the current date and time from the 
// user's browser
var now = new Date();
now.toString() === 'Mon Apr 11 2016 16:10:41 GMT-0500 (Central Daylight Time)' 
// true
// well, at the time of this writing, anyway

// Creates a Date object at the Unix Epoch (i.e., '1970-01-01T00:00:00.000Z')
var epoch = new Date(0);
epoch.toISOString() === '1970-01-01T00:00:00.000Z' // true

// Creates a Date object with the date and time 2,012 milliseconds 
// after the Unix Epoch (i.e., '1970-01-01T00:00:02.012Z').
var ms = new Date(2012); 
date2012.toISOString() === '1970-01-01T00:00:02.012Z' // true

// Creates a Date object with the first day of February of the year 2012 
// in the local timezone.
var one = new Date(2012, 1);
one.toString() === 'Wed Feb 01 2012 00:00:00 GMT-0600 (Central Standard Time)' 
// true

// Creates a Date object with the first day of the year 2012 in the local 
// timezone.
// (Months are zero-based) 
var zero = new Date(2012, 0);
zero.toString() === 'Sun Jan 01 2012 00:00:00 GMT-0600 (Central Standard Time)' 
// true

// Creates a Date object with the first day of the year 2012, in UTC.
var utc = new Date(Date.UTC(2012, 0));
utc.toString() === 'Sat Dec 31 2011 18:00:00 GMT-0600 (Central Standard Time)'
// true
utc.toISOString() === '2012-01-01T00:00:00.000Z'
// true

// Parses a string into a Date object (ISO 8601 format added in ECMAScript 5.1)
// Implementations should assumed UTC because of ISO 8601 format and Z designation
var iso = new Date('2012-01-01T00:00:00.000Z');
iso.toISOString() === '2012-01-01T00:00:00.000Z' // true

// Parses a string into a Date object (RFC in JavaScript 1.0)
var local = new Date('Sun, 01 Jan 2012 00:00:00 -0600');
local.toString() === 'Sun Jan 01 2012 00:00:00 GMT-0600 (Central Standard Time)'
// true

// Parses a string in no particular format, most of the time. Note that parsing
// logic in these cases is very implementation-dependent, and therefore can vary
// across browsers and versions.
var anything = new Date('11/12/2012'); 
anything.toString() === 'Mon Nov 12 2012 00:00:00 GMT-0600 (Central Standard Time)'
// true, in Chrome 49 64-bit on Windows 10 in the en-US locale. Other versions in 
// other locales may get a different result.

// Rolls values outside of a specified range to the next value.
var rollover = new Date(2012, 12, 32, 25, 62, 62, 1023);
rollover.toString() === 'Sat Feb 02 2013 02:03:03 GMT-0600 (Central Standard Time)'
// true; note that the month rolled over to Feb; first the month rolled over to 
// Jan based on the month 12 (11 being December), then again because of the day 32
// (January having 31 days).

// Special dates for years in the range 0-99
var special1 = new Date(12, 0);
special1.toString() === 'Mon Jan 01 1912 00:00:00 GMT-0600 (Central Standard Time)`
// true

// If you actually wanted to set the year to the year 12 CE, you'd need to use the
// setFullYear() method:
special1.setFullYear(12);
special1.toString() === 'Sun Jan 01   12 00:00:00 GMT-0600 (Central Standard Time)`
// true

Criando uma data do UTC #

Por padrão, um objeto Date é criado como hora local. Isso nem sempre é desejável, por exemplo, ao comunicar uma data entre um servidor e um cliente que não residem no mesmo fuso horário. Nesse cenário, não se quer se preocupar com fusos horários até que a data precise ser exibida na hora local, se isso for necessário.

O problema #

Neste problema, queremos comunicar uma data específica (dia, mês, ano) com alguém em um fuso horário diferente. A primeira implementação usa ingenuamente os horários locais, o que resulta em resultados errados. A segunda implementação usa datas UTC para evitar fusos horários onde não são necessários.

Abordagem ingênua com resultados ERRADOS #

function formatDate(dayOfWeek, day, month, year) {
  var daysOfWeek = ["Sun","Mon","Tue","Wed","Thu","Fri","Sat"];
  var months = ["Jan","Feb","Mar","Apr","May","Jun","Jul","Aug","Sep","Oct","Nov","Dec"];
  return daysOfWeek[dayOfWeek] + " " + months[month] + " " + day + " " + year;
}

//Foo lives in a country with timezone GMT + 1
var birthday = new Date(2000,0,1);
console.log("Foo was born on: " + formatDate(birthday.getDay(), birthday.getDate(),
      birthday.getMonth(), birthday.getFullYear()));

sendToBar(birthday.getTime());

Exemplo de saída: Foo nasceu em: Sat Jan 1 2000

//Meanwhile somewhere else...

//Bar lives in a country with timezone GMT - 1
var birthday = new Date(receiveFromFoo());
console.log("Foo was born on: " + formatDate(birthday.getDay(), birthday.getDate(),
      birthday.getMonth(), birthday.getFullYear()));

Exemplo de saída: Foo nasceu em: Sex 31 de dezembro de 1999

E assim, Bar sempre acreditaria que Foo nasceu no último dia de 1999.

Abordagem correta #

function formatDate(dayOfWeek, day, month, year) {
  var daysOfWeek = ["Sun","Mon","Tue","Wed","Thu","Fri","Sat"];
  var months = ["Jan","Feb","Mar","Apr","May","Jun","Jul","Aug","Sep","Oct","Nov","Dec"];
  return daysOfWeek[dayOfWeek] + " " + months[month] + " " + day + " " + year;
}

//Foo lives in a country with timezone GMT + 1
var birthday = new Date(Date.UTC(2000,0,1));
console.log("Foo was born on: " + formatDate(birthday.getUTCDay(), birthday.getUTCDate(),
      birthday.getUTCMonth(), birthday.getUTCFullYear()));

sendToBar(birthday.getTime());

Exemplo de saída: Foo nasceu em: Sat Jan 1 2000

//Meanwhile somewhere else...

//Bar lives in a country with timezone GMT - 1
var birthday = new Date(receiveFromFoo());
console.log("Foo was born on: " + formatDate(birthday.getUTCDay(), birthday.getUTCDate(), 
      birthday.getUTCMonth(), birthday.getUTCFullYear()));

Exemplo de saída: Foo nasceu em: Sat Jan 1 2000

Criando uma data do UTC #

Se alguém quiser criar um objeto Date baseado em UTC ou GMT, o método Date.UTC(...) pode ser usado. Ele usa os mesmos argumentos que o construtor Date mais longo. Esse método retornará um número que representa o tempo decorrido desde 1º de janeiro de 1970, 00:00:00 UTC.

console.log(Date.UTC(2000,0,31,12));

Saída de amostra: 949320000000

var utcDate = new Date(Date.UTC(2000,0,31,12));
console.log(utcDate);

Saída de amostra: Seg 31 de janeiro de 2000 13:00:00 GMT+0100 (West-Europa (standaardtijd))

Sem surpresa, a diferença entre a hora UTC e a hora local é, na verdade, o deslocamento do fuso horário convertido em milissegundos.

var utcDate = new Date(Date.UTC(2000,0,31,12));
var localDate = new Date(2000,0,31,12);

console.log(localDate - utcDate === utcDate.getTimezoneOffset() * 60 * 1000);

Exemplo de saída: true

Alterando um objeto Date #

Todos os modificadores de objeto Date, como setDate(...) e setFullYear(...) têm um equivalente que recebe um argumento na hora UTC em vez da hora local.

var date = new Date();
date.setUTCFullYear(2000,0,31);
date.setUTCHours(12,0,0,0);
console.log(date);

Saída de amostra: Seg 31 de janeiro de 2000 13:00:00 GMT+0100 (West-Europa (standaardtijd))

Os outros modificadores específicos do UTC são .setUTCMonth(), .setUTCDate() (para o dia do mês), .setUTCMinutes(), .setUTCSeconds() e .setUTCMilliseconds().

Evitando ambiguidade com getTime() e setTime() #

Onde os métodos acima são necessários para diferenciar entre ambiguidade nas datas, geralmente é mais fácil comunicar uma data como a quantidade de tempo que se passou desde 1º de janeiro de 1970, 00:00:00 UTC. Este único número representa um único ponto no tempo e pode ser convertido para a hora local sempre que necessário.

var date = new Date(Date.UTC(2000,0,31,12));
var timestamp = date.getTime();
//Alternatively
var timestamp2 = Date.UTC(2000,0,31,12);
console.log(timestamp === timestamp2);

Exemplo de saída: true

//And when constructing a date from it elsewhere...
var otherDate = new Date(timestamp);

//Represented as an universal date

Formatando uma data JavaScript #

Formatando uma data JavaScript em navegadores modernos #

Em navegadores modernos (*), Date.prototype.toLocaleDateString() permite definir a formatação de uma Data de maneira conveniente.

Requer o seguinte formato:

dateObj.toLocaleDateString([locales [, options]])

O parâmetro locales deve ser uma string com uma tag de idioma BCP 47 ou uma matriz dessas strings.

O parâmetro options deve ser um objeto com algumas ou todas as seguintes propriedades:

  • localeMatcher : os valores possíveis são "lookup" e "melhor ajuste"; o padrão é "melhor ajuste"
  • timeZone : o único valor que as implementações devem reconhecer é "UTC"; o padrão é o fuso horário padrão do tempo de execução
  • hour12 :os valores possíveis são true e false; o padrão depende da localidade
  • formatMatcher : os valores possíveis são "basic" e "best fit"; o padrão é "melhor ajuste"
  • dia da semana : os valores possíveis são "estreito", "curto" e "longo"
  • era : os valores possíveis são "estreito", "curto" e "longo"
  • ano: os valores possíveis são "numérico" e "2 dígitos"
  • mês: os valores possíveis são "numérico", "2 dígitos", "estreito", "curto" e "longo"
  • dia : os valores possíveis são "numérico" e "2 dígitos"
  • hora: os valores possíveis são "numérico" & "2 dígitos"
  • minuto: os valores possíveis são "numérico" & "2 dígitos"
  • segundo : os valores possíveis são "numérico" & "2 dígitos"
  • timeZoneName: os valores possíveis são "short" e "long"

Como usar #

var today = new Date().toLocaleDateString('en-GB', {  
    day : 'numeric',
    month : 'short',
    year : 'numeric'
});

Saída se executada em 24 de janeiro de 2036:

'24 Jan 2036'

#Indo customizado

Se Date.prototype.toLocaleDateString() não for flexível o suficiente para atender a qualquer necessidade que você possa ter, você pode querer considerar a criação de um objeto Date personalizado com esta aparência:

var DateObject = (function() {
    var monthNames = [
      "January", "February", "March",
      "April", "May", "June", "July",
      "August", "September", "October",
      "November", "December"
    ];
    var date = function(str) {
        this.set(str);
    };
    date.prototype = {
        set : function(str) {
            var dateDef = str ? new Date(str) : new Date();
            this.day = dateDef.getDate();
            this.dayPadded = (this.day < 10) ? ("0" + this.day) : "" + this.day;
            this.month = dateDef.getMonth() + 1;
            this.monthPadded = (this.month < 10) ? ("0" + this.month) : "" + this.month;
            this.monthName = monthNames[this.month - 1];
            this.year = dateDef.getFullYear();
        },
        get : function(properties, separator) {
            var separator = separator ? separator : '-'
                ret = [];
            for(var i in properties) {
                ret.push(this[properties[i]]);
            }
            return ret.join(separator);
        }
    };
    return date;
})();

Se você incluísse esse código e executasse new DateObject() em 20 de janeiro de 2019, ele produziria um objeto com as seguintes propriedades:

day: 20
dayPadded: "20"
month: 1
monthPadded: "01"
monthName: "January"
year: 2019

Para obter uma string formatada, você pode fazer algo assim:

new DateObject().get(['dayPadded', 'monthPadded', 'year']);

Isso produziria a seguinte saída:

20-01-2016

(*) De acordo com o MDN, “navegadores modernos” significa Chrome 24+, Firefox 29+, IE11, Edge12+, Opera 15+ e Safari nightly build

Obtém a hora e a data atuais #

Use new Date() para gerar um novo objeto Date contendo a data e hora atuais.

Observe que Date() chamado sem argumentos é equivalente a new Date(Date.now()).

Depois de ter um objeto de data, você pode aplicar qualquer um dos vários métodos disponíveis para extrair suas propriedades (por exemplo, getFullYear() para obter o ano de 4 dígitos).

Abaixo estão alguns métodos de data comuns.

Obter o ano atual #

var year = (new Date()).getFullYear();
console.log(year);
// Sample output: 2016

Pega o mês atual #

var month = (new Date()).getMonth();
console.log(month);
// Sample output: 0

Observe que 0 = janeiro. Isso ocorre porque os meses variam de 0 a 11, portanto, geralmente é desejável adicionar +1 ao índice.

var day = (new Date()).getDate();
console.log(day);
// Sample output: 31

Obtém a hora atual #

var hours = (new Date()).getHours();
console.log(hours);
// Sample output: 10

Obtém os minutos atuais #

var minutes = (new Date()).getMinutes();

Obtém os segundos atuais #

var seconds = (new Date()).getSeconds();
console.log(second);

Obtém os milissegundos atuais #

Para obter os milissegundos (variando de 0 a 999) de uma instância de um objeto Date, use seu método getMilliseconds.

var milliseconds = (new Date()).getMilliseconds();
console.log(milliseconds);
 // Output: milliseconds right now

Converte a hora e a data atuais em uma string legível #

var now = new Date();
// convert date to a string in UTC timezone format:
console.log(now.toUTCString());
// Output: Wed, 21 Jun 2017 09:13:01 GMT

O método estático Date.now() retorna o número de milissegundos decorridos desde 1º de janeiro de 1970 00:00:00 UTC. Para obter o número de milissegundos decorridos desde aquela hora usando uma instância de um objeto Date, use seu método getTime.

// get milliseconds using static method now of Date
console.log(Date.now());

// get milliseconds using method getTime of Date instance
console.log((new Date()).getTime());

Incrementar um objeto de data #

Para incrementar objetos de data em Javascript, geralmente podemos fazer isso:

var checkoutDate = new Date();    // Thu Jul 21 2016 10:05:13 GMT-0400 (EDT)

checkoutDate.setDate( checkoutDate.getDate() + 1 );

console.log(checkoutDate); // Fri Jul 22 2016 10:05:13 GMT-0400 (EDT)

É possível usar setDate para alterar a data para um dia no mês seguinte usando um valor maior que o número de dias no mês atual -

var checkoutDate = new Date();    // Thu Jul 21 2016 10:05:13 GMT-0400 (EDT)
checkoutDate.setDate( checkoutDate.getDate() + 12 );
console.log(checkoutDate); // Tue Aug 02 2016 10:05:13 GMT-0400 (EDT)

O mesmo se aplica a outros métodos como getHours(), getMonth(), etc.

Adicionando dias úteis

Se você deseja adicionar dias úteis (neste caso, estou assumindo de segunda a sexta), você pode usar a função setDate, embora precise de um pouco de lógica extra para contabilizar os fins de semana (obviamente, isso não levará em conta os feriados nacionais) -

function addWorkDays(startDate, days) {
    // Get the day of the week as a number (0 = Sunday, 1 = Monday, .... 6 = Saturday)
    var dow = startDate.getDay();
    var daysToAdd = days;
    // If the current day is Sunday add one day
    if (dow == 0)
        daysToAdd++;
    // If the start date plus the additional days falls on or after the closest Saturday calculate weekends
    if (dow + daysToAdd >= 6) {
        //Subtract days in current working week from work days
        var remainingWorkDays = daysToAdd - (5 - dow);
        //Add current working week's weekend
        daysToAdd += 2;
        if (remainingWorkDays > 5) {
            //Add two days for each working week by calculating how many weeks are included
            daysToAdd += 2 * Math.floor(remainingWorkDays / 5);
            //Exclude final weekend if remainingWorkDays resolves to an exact number of weeks
            if (remainingWorkDays % 5 == 0)
                daysToAdd -= 2;
        }
    }
    startDate.setDate(startDate.getDate() + daysToAdd);
    return startDate;
}
