Usando console.log() #

Introdução #

Todos os navegadores modernos, NodeJs e quase todos os outros ambientes JavaScript suportam a gravação de mensagens em um console usando um conjunto de métodos de registro. O mais comum desses métodos é console.log().

Em um ambiente de navegador, a função console.log() é predominantemente usada para fins de depuração.

Começando #

Abra o Console JavaScript em seu navegador, digite o seguinte e pressione Enter :

console.log("Hello, World!");

Isso registrará o seguinte no console:

No exemplo acima, a função console.log() imprime Hello, World! no console e retorna undefined (mostrado acima na janela de saída do console). Isso ocorre porque console.log() não tem um valor de retorno explícito.

Variáveis ​​de registro #

console.log() pode ser usado para registrar variáveis ​​de qualquer tipo; não apenas cordas. Basta passar a variável que você deseja que seja exibida no console, por exemplo:

var foo = "bar";
console.log(foo);

Se você quiser registrar dois ou mais valores, basta separá-los com vírgulas. Os espaços serão adicionados automaticamente entre cada argumento durante a concatenação:

var thisVar = 'first value';
var thatVar = 'second value';
console.log("thisVar:", thisVar, "and thatVar:", thatVar);

Você pode usar console.log() em combinação com espaços reservados:

var greet = "Hello", who = "World";
console.log("%s, %s!", greet, who);

Usando a API DOM #

DOM significa Document Object Model. É uma representação orientada a objetos de documentos estruturados como XML e HTML.

Definir a propriedade textContent de um Element é uma maneira de produzir texto em uma página da web.

Por exemplo, considere a seguinte tag HTML:

<p id="paragraph"></p>

Para alterar sua propriedade textContent, podemos executar o seguinte JavaScript:

document.getElementById("paragraph").textContent = "Hello, World";

Isso selecionará o elemento com o id paragraph e definirá seu conteúdo de texto como “Hello, World”:

<p id="paragraph">Hello, World</p>

(Veja também esta demonstração)

Você também pode usar JavaScript para criar um novo elemento HTML programaticamente. Por exemplo, considere um documento HTML com o seguinte corpo:

<body>
   <h1>Adding an element</h1>
</body>

Em nosso JavaScript, criamos uma nova tag <p> com uma propriedade textContent e a adicionamos no final do corpo html:

var element = document.createElement('p');
element.textContent = "Hello, World";
document.body.appendChild(element); //add the newly created element to the DOM

Isso mudará seu corpo HTML para o seguinte:

<body>
   <h1>Adding an element</h1>
   <p>Hello, World</p>
</body>

Observe que para manipular elementos no DOM usando JavaScript, o código JavaScript deve ser executado depois que o elemento relevante foi criado no documento. Isso pode ser feito colocando as tags JavaScript <script> depois de todo o seu outro conteúdo <body>. Alternativamente, você também pode usar um ouvinte de evento para ouvir, por exemplo. evento onload do window, adicionar seu código a esse ouvinte de evento atrasará a execução do seu código até que todo o conteúdo da sua página tenha sido carregado.

Uma terceira maneira de certificar-se de que todo o seu DOM foi carregado é envolver o código de manipulação do DOM com uma função de tempo limite de 0 ms. Dessa forma, esse código JavaScript é enfileirado novamente no final da fila de execução, o que dá ao navegador a chance de terminar de fazer algumas coisas não JavaScript que estavam esperando para terminar antes de atender a essa nova parte do JavaScript.

Usando window.alert() #

O método alert exibe uma caixa de alerta visual na tela. O parâmetro do método de alerta é exibido ao usuário em texto simples:

window.alert(message);

Como window é o objeto global, você também pode chamar usando a seguinte abreviação:

alert(message);

Então, o que window.alert() faz? Bem, vamos ao seguinte exemplo:

alert('hello, world');

No Chrome, isso produziria um pop-up como este:

Notas #

O método alert é tecnicamente uma propriedade do objeto window, mas como todas as propriedades window são automaticamente variáveis ​​globais, podemos usar alert como uma variável global em vez de como uma propriedade de window - significando que você pode usar diretamente alert() em vez de window.alert().

Ao contrário de usar console.log, alert atua como um prompt modal, significando que o código que chama alert pausará até que o prompt seja respondido. Tradicionalmente, isso significa que nenhum outro código JavaScript será executado até que o alerta seja dispensado:

alert('Pause!');
console.log('Alert was dismissed');

No entanto, a especificação realmente permite que outro código acionado por evento continue a ser executado mesmo que uma caixa de diálogo modal ainda esteja sendo mostrada. Em tais implementações, é possível que outro código seja executado enquanto a caixa de diálogo modal está sendo exibida.

Mais informações sobre [uso do método alert](https://www.wikiod.com/pt/javascript/modais—prompts#Uso de alerta()) podem ser encontradas nos [prompts de modais](http ://stackoverflow.com/documentation/javascript/3196/modals-prompts).

O uso de alertas geralmente é desencorajado em favor de outros métodos que não impeçam os usuários de interagir com a página - a fim de criar uma melhor experiência do usuário. No entanto, pode ser útil para depuração.

A partir do Chrome 46.0, window.alert() é bloqueado dentro de um <iframe> a menos que seu atributo sandbox tenha o valor allow-modal.

Usando window.prompt() #

Uma maneira fácil de obter uma entrada de um usuário é usando o método prompt().

Sintaxe #

prompt(text, [default]);
  • texto: o texto exibido na caixa de prompt.
  • default: Um valor padrão para o campo de entrada (opcional).

Exemplos #

var age = prompt("How old are you?");
console.log(age); // Prints the value inserted by the user

Se o usuário clicar no botão OK, o valor de entrada é retornado. Caso contrário, o método retorna null.

O valor de retorno de prompt é sempre uma string, a menos que o usuário clique em Cancelar, neste caso ele retorna null. O Safari é uma exceção, pois quando o usuário clica em Cancelar, a função retorna uma string vazia. A partir daí, você pode converter o valor de retorno em outro tipo, como um integer.

Notas #

  • Enquanto a caixa de prompt é exibida, o usuário é impedido de acessar outras partes da página, pois as caixas de diálogo são janelas modais.
  • A partir do Chrome 46.0, este método é bloqueado dentro de um <iframe>, a menos que seu atributo sandbox tenha o valor allow-modal.

Usando window.confirm() #

O método window.confirm() exibe uma caixa de diálogo modal com uma mensagem opcional e dois botões, OK e Cancelar.

Agora, vamos pegar o seguinte exemplo:

result = window.confirm(message);

Aqui, message é a string opcional a ser exibida na caixa de diálogo e result é um valor booleano que indica se OK ou Cancel foi selecionado (true significa OK).

window.confirm() é normalmente usado para pedir a confirmação do usuário antes de fazer uma operação perigosa como deletar algo em um Painel de Controle:

if(window.confirm("Are you sure you want to delete this?")) {
    deleteItem(itemId);
}

A saída desse código ficaria assim no navegador:

Se você precisar dele para uso posterior, basta armazenar o resultado da interação do usuário em uma variável:

var deleteConfirm = window.confirm("Are you sure you want to delete this?");

Notas #

  • O argumento é opcional e não exigido pela especificação.
  • As caixas de diálogo são janelas modais - impedem que o usuário acesse o restante da interface do programa até que a caixa de diálogo seja fechada. Por esse motivo, você não deve abusar de nenhuma função que crie uma caixa de diálogo (ou janela modal). E, independentemente disso, há boas razões para evitar o uso de caixas de diálogo para confirmação.
  • A partir do Chrome 46.0, este método é bloqueado dentro de um <iframe>, a menos que seu atributo sandbox tenha o valor allow-modal.
  • É comumente aceito chamar o método confirm com a notação da janela removida, pois o objeto da janela está sempre implícito. No entanto, é recomendável definir explicitamente o objeto de janela, pois o comportamento esperado pode mudar devido à implementação em um nível de escopo inferior com métodos com nomes semelhantes.

Usando a API DOM (com texto gráfico: Canvas, SVG ou arquivo de imagem) #

Usando elementos de tela

HTML fornece o elemento canvas para construir imagens baseadas em raster.

Primeiro, construa uma tela para armazenar informações de pixel de imagem.

var canvas = document.createElement('canvas');
canvas.width = 500;
canvas.height = 250;

Em seguida, selecione um contexto para a tela, neste caso bidimensional:

var ctx = canvas.getContext('2d');

Em seguida, defina as propriedades relacionadas ao texto:

ctx.font = '30px Cursive';
ctx.fillText("Hello world!", 50, 50);

Em seguida, insira o elemento canvas na página para ter efeito:

document.body.appendChild(canvas);

Usando SVG

SVG é para construir gráficos baseados em vetores escaláveis ​​e pode ser usado em HTML.

Primeiro, crie um contêiner de elemento SVG com dimensões:

var svg = document.createElementNS('http://www.w3.org/2000/svg', 'svg');
svg.width = 500;
svg.height = 50;

Em seguida, construa um elemento text com o posicionamento desejado e as características da fonte:

var text = document.createElementNS('http://www.w3.org/2000/svg', 'text');
text.setAttribute('x', '0');
text.setAttribute('y', '50');
text.style.fontFamily = 'Times New Roman';
text.style.fontSize = '50';

Em seguida, adicione o texto real a ser exibido ao elemento text:

text.textContent = 'Hello world!';

Finalmente adicione o elemento text ao nosso container svg e adicione o elemento container svg ao documento HTML:

svg.appendChild(text);
document.body.appendChild(svg);

Arquivo de imagem

Se você já possui um arquivo de imagem contendo o texto desejado e o coloca em um servidor, você pode adicionar a URL da imagem e adicionar a imagem ao documento da seguinte forma:

var img = new Image();
img.src = 'https://i.ytimg.com/vi/zecueq-mo4M/maxresdefault.jpg';
document.body.appendChild(img);
