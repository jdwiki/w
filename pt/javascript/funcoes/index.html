As funções em JavaScript fornecem código organizado e reutilizável para executar um conjunto de ações. As funções simplificam o processo de codificação, evitam lógica redundante e tornam o código mais fácil de seguir. Este tópico descreve a declaração e utilização de funções, argumentos, parâmetros, declarações de retorno e escopo em JavaScript.

Sintaxe #

  • exemplo de função (x) { return x }

  • var exemplo = função (x) { return x }

  • (função() { … })(); // Expressão de função invocada imediatamente (IIFE)

  • var instância = new Exemplo(x);

  • Métodos

  • fn.apply(valueForThis[, arrayOfArgs])

  • fn.bind(valueForThis[, arg1[, arg2, …]])

  • fn.call(valueForThis[, arg1[, arg2, …]])

  • ES2015+ (ES6+):

  • exemplo const = x => { return x }; // Retorno explícito da função de seta

  • const exemplo = x => x; // Retorno implícito da função de seta

  • const example = (x, y, z) => { … } // Função de seta vários argumentos

  • (() => { … })(); // IIFE usando uma função de seta

Para obter informações sobre funções de seta, consulte a documentação Funções de seta.

Escopo da Função #

Quando você define uma função, ela cria um escopo.

Tudo definido dentro da função não é acessível por código fora do função. Somente o código dentro deste escopo pode ver as entidades definidas dentro do alcance.

function foo() {
  var a = 'hello';
  console.log(a);  // => 'hello'
}

console.log(a);  // reference error

Funções aninhadas são possíveis em JavaScript e as mesmas regras se aplicam.

function foo() {
  var a = 'hello';
  
  function bar() {
    var b = 'world';
    console.log(a);  // => 'hello'
    console.log(b);  // => 'world'
  }

  console.log(a);  // => 'hello'
  console.log(b);  // reference error
}

console.log(a);  // reference error
console.log(b);  // reference error

Quando JavaScript tenta resolver uma referência ou variável, ele começa a procurar no escopo atual. Se não encontrar essa declaração no escopo atual, ele sobe um escopo para procurá-lo. Esse processo se repete até que a declaração seja encontrada. Se o analisador JavaScript atingir o escopo global e ainda não conseguir encontrar o referência, um erro de referência será lançado.

var a = 'hello';

function foo() {
  var b = 'world';

  function bar() {
    var c = '!!';

    console.log(a);  // => 'hello'
    console.log(b);  // => 'world'
    console.log(c);  // => '!!'
    console.log(d);  // reference error
  }
}

Este comportamento de escalada também pode significar que uma referência pode “sombrar” sobre um referência com nome semelhante no escopo externo, pois é vista primeiro.

var a = 'hello';

function foo() {
  var a = 'world';

  function bar() {
    console.log(a);  // => 'world'
  }
}

A maneira como o JavaScript resolve o escopo também se aplica à palavra-chave const. Declarar uma variável com a palavra-chave const implica que você não tem permissão para reatribuir o valor, mas declará-la em uma função criará um novo escopo e com isso uma nova variável.

function foo() {
  const a = true;

  function bar() {
    const a = false;  // different variable
    console.log(a);   // false
  }

  const a = false;    // SyntaxError
  a = false;          // TypeError
  console.log(a);     // true
}

No entanto, as funções não são os únicos blocos que criam um escopo (se você estiver usando let ou const). Declarações let e const têm um escopo da instrução de bloco mais próxima. Veja aqui para uma descrição mais detalhada.

1: https://www.wikiod.com/pt/javascript/alcance#Diferença entre var e let

Escovando #

Currying é a transformação de uma função de n aridade ou argumentos em uma seqüência de funções n recebendo apenas um argumento.

Casos de uso: Quando os valores de alguns argumentos estão disponíveis antes de outros, você pode usar currying para decompor uma função em uma série de funções que completam o trabalho em etapas, à medida que cada valor chega. Isso pode ser útil:

  • Quando o valor de um argumento quase nunca muda (por exemplo, um fator de conversão), mas você precisa manter a flexibilidade de definir esse valor (em vez de codificá-lo como uma constante).
  • Quando o resultado de uma função curry é útil antes que as outras funções curry tenham sido executadas.
  • Para validar a chegada das funções em uma sequência específica.

Por exemplo, o volume de um prisma retangular pode ser explicado por uma função de três fatores: comprimento (l), largura (w) e altura (h):

var prism = function(l, w, h) {
    return l * w * h;
}

Uma versão curry desta função ficaria assim:

function prism(l) {
    return function(w) {
        return function(h) {
            return l * w * h;
        }
    }
}

// alternatively, with concise ECMAScript 6+ syntax:
var prism = l => w => h => l * w * h;

Você pode chamar essa sequência de funções com prism(2)(3)(5), que deve resultar em 30.

Sem alguma maquinaria extra (como com bibliotecas), curry é de flexibilidade sintática limitada em JavaScript (ES 5/6) devido à falta de espaço reservado valores; assim, enquanto você pode usar var a = prism(2)(3) para criar um parcialmente função aplicada, você não pode usar prism()(3)(5).

Expressões de função invocadas imediatamente #

Às vezes você não quer ter sua função acessível/armazenada como uma variável. Você pode criar uma expressão de função chamada imediatamente (IIFE para abreviar). Estas são essencialmente funções anônimas auto-executáveis. Eles têm acesso ao escopo circundante, mas a função em si e quaisquer variáveis ​​internas serão inacessíveis de fora. Uma coisa importante a ser observada sobre o IIFE é que, mesmo que você nomeie sua função, o IIFE não é içado como as funções padrão e não pode ser chamado pelo nome da função com o qual foi declarado.

(function() {
   alert("I've run - but can't be run again because I'm immediately invoked at runtime,
          leaving behind only the result I generate");
}());

Esta é outra maneira de escrever IIFE. Observe que o parêntese de fechamento antes do ponto e vírgula foi movido e colocado logo após o colchete de fechamento:

(function() {
   alert("This is IIFE too.");
})();

Você pode facilmente passar parâmetros para um IIFE:

(function(message) {
   alert(message);
}("Hello World!"));

Além disso, você pode retornar valores para o escopo circundante:

var example = (function() {
   return 42;
}());
console.log(example); // => 42

Se necessário, é possível nomear um IIFE. Embora visto com menos frequência, esse padrão tem várias vantagens, como fornecer uma referência que pode ser usada para uma recursão e tornar a depuração mais simples, pois o nome é incluído na pilha de chamadas.

(function namedIIFE() { 
    throw error; // We can now see the error thrown in 'namedIIFE()'
}());

Embora envolver uma função entre parênteses seja a maneira mais comum de indicar ao analisador Javascript que espera uma expressão, em locais onde uma expressão já é esperada, a notação pode ser mais concisa:

var a = function() { return 42 }();
console.log(a)  // => 42

Versão de seta da função imediatamente invocada:

(() => console.log("Hello!"))(); // => Hello!

Ligando this e argumentos #

Quando você faz uma referência a um método (uma propriedade que é uma função) em JavaScript, ele geralmente não lembra o objeto ao qual foi originalmente anexado. Se o método precisar se referir a esse objeto como this, ele não será capaz, e chamá-lo provavelmente causará uma falha.

Você pode usar o método .bind() em uma função para criar um wrapper que inclua o valor de this e qualquer número de argumentos iniciais.

var monitor = {
  threshold: 5,
  check: function(value) {
    if (value > this.threshold) {
      this.display("Value is too high!");
    }
  },
  display(message) {
    alert(message);
  }
};

monitor.check(7); // The value of `this` is implied by the method call syntax.


var badCheck = monitor.check;
badCheck(15); // The value of `this` is window object and this.threshold is undefined, so value > this.threshold is false

var check = monitor.check.bind(monitor);
check(15); // This value of `this` was explicitly bound, the function works.

var check8 = monitor.check.bind(monitor, 8);
check8(); // We also bound the argument to `8` here. It can't be re-specified.

Quando não estiver no modo estrito, uma função usa o objeto global (window no navegador) como this, a menos que a função seja chamada como um método, vinculada ou chamada com a sintaxe do método .call.

window.x = 12; 

function example() {
  return this.x;
}

console.log(example()); // 12

No modo estrito this é undefined por padrão

window.x = 12; 
    
function example() {
  "use strict";
  return this.x;
}

console.log(example()); // Uncaught TypeError: Cannot read property 'x' of undefined(…)

Operador de Ligação #

Os dois pontos operador de ligação podem ser usados ​​como uma sintaxe abreviada para o conceito explicado acima:

var log = console.log.bind(console); // long version
const log = ::console.log; // short version

foo.bar.call(foo); // long version
foo::bar(); // short version

foo.bar.call(foo, arg1, arg2, arg3); // long version
foo::bar(arg1, arg2, arg3); // short version

foo.bar.apply(foo, args); // long version
foo::bar(...args); // short version

Esta sintaxe permite que você escreva normalmente, sem se preocupar em vincular this em todos os lugares.

Vinculando funções do console a variáveis #

var log = console.log.bind(console);

Uso:

log('one', '2', 3, [4], {5: 5});

Resultado:

one 2 3 [4] Object {5: 5}

Por que você faria isso?

Um caso de uso pode ser quando você tem um registrador personalizado e deseja decidir em tempo de execução qual deles usar.

var logger = require('appLogger');

var log = logToServer ? logger.log : console.log.bind(console);

Funções nomeadas #

As funções podem ser nomeadas ou não nomeadas (funções anônimas):

var namedSum = function sum (a, b) { // named
    return a + b;
}

var anonSum = function (a, b) { // anonymous
    return a + b;
}

namedSum(1, 3);
anonSum(1, 3);

4 4

Mas seus nomes são privados para seu próprio escopo:

var sumTwoNumbers = function sum (a, b) {
    return a + b;
}

sum(1, 3);

Erro de referência não capturado: a soma não está definida

As funções nomeadas diferem das funções anônimas em vários cenários:

  • Quando você estiver depurando, o nome da função aparecerá no erro/rastreamento de pilha
  • As funções nomeadas são elevadas enquanto as funções anônimas não são
  • Funções nomeadas e funções anônimas se comportam de maneira diferente ao lidar com recursão
  • Dependendo da versão do ECMAScript, funções nomeadas e anônimas podem tratar a propriedade name da função de forma diferente

As funções nomeadas são içadas #

Ao usar uma função anônima, a função só pode ser chamada após a linha de declaração, enquanto uma função nomeada pode ser chamada antes da declaração. Considerar

foo();
var foo = function () { // using an anonymous function
    console.log('bar');
}

Uncaught TypeError: foo não é uma função

foo();
function foo () { // using a named function

Funções com um número desconhecido de argumentos (funções variáveis) #

Para criar uma função que aceita um número indeterminado de argumentos, existem dois métodos dependendo do seu ambiente.

Sempre que uma função é chamada, ela tem um objeto do tipo Array arguments em seu escopo, contendo todos os argumentos passados ​​para a função. Indexar ou iterar sobre isso dará acesso aos argumentos, por exemplo

function logSomeThings() {
    for (var i = 0; i < arguments.length; ++i) {
        console.log(arguments[i]);
    }
}

logSomeThings('hello', 'world');
// logs "hello"
// logs "world"

Note que você pode converter arguments em um Array real se necessário; consulte: https://www.wikiod.com/pt/javascript/matrizes#Convertendo objetos semelhantes a matrizes em matrizes

A partir do ES6, a função pode ser declarada com seu último parâmetro usando o operador rest (...). Isso cria um Array que contém os argumentos daquele ponto em diante

function personLogsSomeThings(person, ...msg) {
   msg.forEach(arg => {
       console.log(person, 'says', arg);
   });
}

personLogsSomeThings('John', 'hello', 'world');
// logs "John says hello"
// logs "John says world"

As funções também podem ser chamadas de maneira semelhante, a sintaxe de propagação

const logArguments = (...args) => console.log(args)
const list = [1, 2, 3]

logArguments('a', 'b', 'c', ...list)
// output: Array [ "a", "b", "c", 1, 2, 3 ]

Esta sintaxe pode ser usada para inserir um número arbitrário de argumentos em qualquer posição e pode ser usada com qualquer iterável (apply aceita apenas objetos do tipo array).

const logArguments = (...args) => console.log(args)
function* generateNumbers() {
  yield 6
  yield 5
  yield 4
}

logArguments('a', ...generateNumbers(), ...'pqr', 'b')
// output: Array [ "a", 6, 5, 4, "p", "q", "r", "b" ]

Função anônima #

Definindo uma função anônima #

Quando uma função é definida, você geralmente dá um nome a ela e então a invoca usando esse nome, assim:

foo();

function foo(){
    // ...
}

Quando você define uma função dessa maneira, o tempo de execução do Javascript armazena sua função na memória e cria uma referência a essa função, usando o nome que você atribuiu a ela. Esse nome é então acessível dentro do escopo atual. Essa pode ser uma maneira muito conveniente de criar uma função, mas o Javascript não exige que você atribua um nome a uma função. O seguinte também é perfeitamente legal:

function() {
    // ...
}

Quando uma função é definida sem um nome, ela é conhecida como uma função anônima. A função é armazenada na memória, mas o tempo de execução não cria automaticamente uma referência a ela para você. À primeira vista, pode parecer que tal coisa não teria utilidade, mas existem vários cenários em que as funções anônimas são muito convenientes.

Atribuindo uma função anônima a uma variável #

Um uso muito comum de funções anônimas é atribuí-las a uma variável:

var foo = function(){ /*...*/ };

foo();

Esse uso de funções anônimas é abordado com mais detalhes em https://www.wikiod.com/pt/javascript/funcoes#Funciona como uma variável

Fornecendo uma função anônima como parâmetro para outra função #

Algumas funções podem aceitar uma referência a uma função como parâmetro. Às vezes, eles são chamados de “injeções de dependência” ou “retornos de chamada”, porque permitem que a função que você está chamando “chame de volta” para seu código, dando a você a oportunidade de alterar a maneira como a função chamada se comporta. Por exemplo, a função map do objeto Array permite iterar sobre cada elemento de uma matriz e, em seguida, criar uma nova matriz aplicando uma função de transformação a cada elemento.

var nums = [0,1,2];
var doubledNums = nums.map( function(element){ return element * 2; } ); // [0,2,4]

Seria tedioso, desleixado e desnecessário criar uma função nomeada, que sobrecarregaria seu escopo com uma função necessária apenas neste local e interromperia o fluxo natural e a leitura do seu código (um colega teria que deixar esse código para encontrar seu função para entender o que está acontecendo).

Retornando uma função anônima de outra função #

Às vezes é útil retornar uma função como resultado de outra função. Por exemplo:

var hash = getHashFunction( 'sha1' );
var hashValue = hash( 'Secret Value' );

function getHashFunction( algorithm ){

    if ( algorithm === 'sha1' ) return function( value ){ /*...*/ };
    else if ( algorithm === 'md5' ) return function( value ){ /*...*/ };

}

Chamando imediatamente uma função anônima #

Ao contrário de muitas outras linguagens, o escopo em Javascript é em nível de função, não em nível de bloco. (Consulte https://www.wikiod.com/pt/javascript/funcoes#Escopo da Função ). Em alguns casos, porém, é necessário criar um novo escopo. Por exemplo, é comum criar um novo escopo ao adicionar código por meio de uma tag <script>, em vez de permitir que nomes de variáveis ​​sejam definidos no escopo global (o que corre o risco de outros scripts colidirem com seus nomes de variáveis). Um método comum para lidar com essa situação é definir uma nova função anônima e invocá-la imediatamente, ocultando com segurança suas variáveis ​​no escopo da função anônima e sem tornar seu código acessível a terceiros por meio de um nome de função vazado. Por exemplo:

<!-- My Script -->
<script>
function initialize(){
    // foo is safely hidden within initialize, but...
    var foo = '';
}

// ...my initialize function is now accessible from global scope.
// There's a risk someone could call it again, probably by accident.
initialize();
</script>

<script>
// Using an anonymous function, and then immediately
// invoking it, hides my foo variable and guarantees
// no one else can call it a second time.
(function(){
    var foo = '';
}()) // <--- the parentheses invokes the function immediately
</script>

Funções anônimas autorreferenciais #

Às vezes, é útil que uma função anônima possa se referir a si mesma. Por exemplo, a função pode precisar chamar a si mesma recursivamente ou adicionar propriedades a ela mesma. Se a função for anônima, porém, isso pode ser muito difícil, pois requer conhecimento da variável à qual a função foi atribuída. Esta é a solução menos que ideal:

var foo = function(callAgain){
    console.log( 'Whassup?' );
    // Less then ideal... we're dependent on a variable reference...
    if (callAgain === true) foo(false);
};

foo(true);

// Console Output:
// Whassup?
// Whassup?

// Assign bar to the original function, and assign foo to another function.
var bar = foo;
foo = function(){
    console.log('Bad.')
};

bar(true);

// Console Output:
// Whassup?
// Bad.

A intenção aqui era que a função anônima se chamasse recursivamente, mas quando o valor de foo muda, você acaba com um bug potencialmente difícil de rastrear.

Em vez disso, podemos dar à função anônima uma referência a ela mesma, dando-lhe um nome privado, assim:

var foo = function myself(callAgain){
    console.log( 'Whassup?' );
    // Less then ideal... we're dependent on a variable reference...
    if (callAgain === true) myself(false);
};

foo(true);

// Console Output:
// Whassup?
// Whassup?

// Assign bar to the original function, and assign foo to another function.
var bar = foo;
foo = function(){
    console.log('Bad.')
};

bar(true);

// Console Output:
// Whassup?
// Whassup?

Observe que o nome da função tem escopo para si mesmo. O nome não vazou para o escopo externo:

myself(false); // ReferenceError: myself is not defined

Essa técnica é especialmente útil ao lidar com funções anônimas recursivas como parâmetros de retorno de chamada:

// Calculate the fibonacci value for each number in an array:
var fib = false,
    result = [1,2,3,4,5,6,7,8].map(
        function fib(n){
            return ( n <= 2 ) ? 1 : fib( n - 1 ) + fib( n - 2 );
        });
// result = [1, 1, 2, 3, 5, 8, 13, 21]
// fib = false (the anonymous function name did not overwrite our fib variable)

Parâmetros padrão #

Antes do ECMAScript 2015 (ES6), o valor padrão de um parâmetro podia ser atribuído da seguinte maneira:

function printMsg(msg) {
  msg = typeof msg !== 'undefined' ? // if a value was provided 
        msg :                        // then, use that value in the reassignemnt
        'Default value for msg.';    // else, assign a default value
  console.log(msg);
}

O ES6 forneceu uma nova sintaxe em que a condição e a reatribuição descritas acima não são mais necessárias:

function printMsg(msg='Default value for msg.') {
    console.log(msg);
}

printMsg(); // -> "Default value for msg."
printMsg(undefined); // -> "Default value for msg."
printMsg('Now my msg in different!'); // -> "Now my msg in different!"

Isso também mostra que, se um parâmetro estiver ausente quando a função for invocada, seu valor será mantido como undefined, como pode ser confirmado fornecendo-o explicitamente no exemplo a seguir (usando uma função de seta):

let param_check = (p = 'str') => console.log(p + ' is of type: ' + typeof p);

param_check(); // -> "str is of type: string"
param_check(undefined); // -> "str is of type: string"

param_check(1); // -> "1 is of type: number"
param_check(this); // -> "[object Window] is of type: object"

Funções/variáveis ​​como valores padrão e reutilização de parâmetros #

Os valores dos parâmetros padrão não se restringem a números, strings ou objetos simples. Uma função também pode ser definida como o valor padrão callback = function(){}:

function foo(callback = function(){ console.log('default'); }) {

// custom

Existem certas características das operações que podem ser realizadas por meio de valores padrão:

  • Um parâmetro declarado anteriormente pode ser reutilizado como valor padrão para os próximos valores dos parâmetros.
  • Operações em linha são permitidas ao atribuir um valor padrão a um parâmetro.
  • Variáveis ​​existentes no mesmo escopo da função que está sendo declarada podem ser utilizadas em seus valores padrão.
  • As funções podem ser invocadas para fornecer seu valor de retorno em um valor padrão.
let zero = 0;

add(2, 7);             // 18, 18
add(1, 2, 5);          // 8, 10
add(1, 2, 5, 10);      // 8, 20

add(6);         // [6], not [5, 6]

Passando argumentos por referência ou valor #

Em JavaScript todos os argumentos são passados ​​por valor. Quando uma função atribui um novo valor a uma variável de argumento, essa alteração não será visível para o chamador:

var obj = {a: 2};
function myfunc(arg){
    arg = {a: 5}; // Note the assignment is to the parameter variable itself
}
myfunc(obj);
console.log(obj.a); // 2

No entanto, as alterações feitas nas propriedades (aninhadas) de tais argumentos serão visíveis para o chamador:

var obj = {a: 2};
function myfunc(arg){
    arg.a = 5; // assignment to a property of the argument
}
myfunc(obj);
console.log(obj.a); // 5

Isso pode ser visto como uma chamada por referência: embora uma função não possa alterar o objeto do chamador atribuindo um novo valor a ele, ela pode mutar o objeto do chamador.

Como argumentos de valor primitivo, como números ou strings, são imutáveis, não há como uma função alterá-los:

var s = 'say';
function myfunc(arg){
    arg += ' hello'; // assignment to the parameter variable itself
}
myfunc(s);
console.log(s); // 'say'

Quando uma função deseja alterar um objeto passado como argumento, mas não deseja realmente alterar o objeto do chamador, a variável de argumento deve ser reatribuída:

var obj = {a: 2, b: 3};
function myfunc(arg){
    arg = Object.assign({}, arg); // assignment to argument variable, shallow copy
    arg.a = 5;
}
myfunc(obj);
console.log(obj.a); // 2

Como alternativa à mutação in-loco de um argumento, as funções podem criar um novo valor, com base no argumento, e retorná-lo. O chamador pode então atribuí-lo, mesmo à variável original que foi passada como argumento:

var a = 2;
function myfunc(arg){
    arg++;
    return arg;
}
a = myfunc(a);
console.log(obj.a); // 3

Ligue e candidate-se #

As funções têm dois métodos embutidos que permitem ao programador fornecer argumentos e a variável this de forma diferente: call e apply.

Isso é útil porque as funções que operam em um objeto (o objeto do qual são propriedade) podem ser reaproveitadas para operar em outro objeto compatível. Além disso, os argumentos podem ser fornecidos de uma só vez como arrays, semelhante ao operador spread (...) no ES6.

let obj = {
    a: 1,
    b: 2,
    set: function (a, b) {
        this.a = a;
        this.b = b;
    }
};

obj.set(3, 7); // normal syntax
obj.set.call(obj, 3, 7); // equivalent to the above
obj.set.apply(obj, [3, 7]); // equivalent to the above; note that an array is used

console.log(obj); // prints { a: 3, b: 5 }

let myObj = {};
myObj.set(5, 4); // fails; myObj has no `set` property
obj.set.call(myObj, 5, 4); // success; `this` in set() is re-routed to myObj instead of obj
obj.set.apply(myObj, [5, 4]); // same as above; note the array

console.log(myObj); // prints { a: 3, b: 5 }

ECMAScript 5 introduziu outro método chamado bind() além de call() e apply() para definir explicitamente o valor this da função para um objeto específico.

Ele se comporta de maneira bem diferente dos outros dois. O primeiro argumento para bind() é o valor this para a nova função. Todos os outros argumentos representam parâmetros nomeados que devem ser definidos permanentemente na nova função.

function showName(label) {
    console.log(label + ":" + this.name);
}
var student1 = {
     name: "Ravi"
};
var student2 = {
     name: "Vinod"
};

// create a function just for student1
var showNameStudent1 = showName.bind(student1);
showNameStudent1("student1"); // outputs "student1:Ravi"

// create a function just for student2
var showNameStudent2 = showName.bind(student2, "student2");
showNameStudent2(); // outputs "student2:Vinod"

// attaching a method to an object doesn't change `this` value of that method.
student2.sayName = showNameStudent1;
student2.sayName("student2"); // outputs "student2:Ravi"

Aplicação Parcial #

Semelhante ao currying, o aplicativo parcial é usado para reduzir o número de argumentos passados ​​para uma função. Ao contrário do curry, o número não precisa diminuir em um.

Exemplo:

Esta função…

function multiplyThenAdd(a, b, c) {
    return a * b + c;
}

… pode ser usado para criar outra função que sempre irá multiplicar por 2 e depois adicionar 10 ao valor passado;

function reversedMultiplyThenAdd(c, b, a) {
    return a * b + c;
}

function factory(b, c) {
    return reversedMultiplyThenAdd.bind(null, c, b);
}

var multiplyTwoThenAddTen = factory(2, 10);
multiplyTwoThenAddTen(10); // 30

A parte “aplicação” da aplicação parcial significa simplesmente fixar os parâmetros de uma função.

Composição de funções #

Compor várias funções em uma é uma prática comum de programação funcional;

composição faz um pipeline através do qual nossos dados transitarão e serão modificados simplesmente trabalhando na composição da função (assim como encaixar pedaços de uma faixa) …

você começa com algumas funções de responsabilidade única:

 const capitalize = x => x.replace(/^\w/, m => m.toUpperCase());
 const sign = x => x + ',\nmade with love';

e crie facilmente uma trilha de transformação:

 const formatText = compose(capitalize, sign);

 formatText('this is an example')
 //This is an example,
 //made with love

N.B. A composição é obtida através de uma função de utilidade normalmente chamada compose como em nosso exemplo.

A implementação de compose está presente em muitas bibliotecas de utilitários JavaScript (lodash, rambda, etc.), mas você também pode começar com uma implementação simples, como:

 const compose = (...funs) =>
   x =>
   funs.reduce((ac, f) => f(ac), x);

Funciona como uma variável #

Uma declaração de função normal se parece com isso:

function foo(){
}

Uma função definida como esta é acessível de qualquer lugar dentro de seu contexto por seu nome. Mas às vezes pode ser útil tratar referências de função como referências de objeto. Por exemplo, você pode atribuir um objeto a uma variável com base em algum conjunto de condições e depois recuperar uma propriedade de um ou outro objeto:

var name = 'Cameron';
var spouse;

if ( name === 'Taylor' ) spouse = { name: 'Jordan' };
else if ( name === 'Cameron' ) spouse = { name: 'Casey' };

var spouseName = spouse.name;

Em JavaScript, você pode fazer a mesma coisa com funções:

// Example 1
var hashAlgorithm = 'sha1';
var hash;

if ( hashAlgorithm === 'sha1' ) hash = function(value){ /*...*/ };
else if ( hashAlgorithm === 'md5' ) hash = function(value){ /*...*/ };

hash('Fred');

No exemplo acima, hash é uma variável normal. É atribuída uma referência a uma função, após a qual a função que ela faz referência pode ser invocada usando parênteses, assim como uma declaração de função normal.

O exemplo acima faz referência a funções anônimas… funções que não possuem nome próprio. Você também pode usar variáveis ​​para se referir a funções nomeadas. O exemplo acima poderia ser reescrito assim:

// Example 2
var hashAlgorithm = 'sha1';
var hash;

if ( hashAlgorithm === 'sha1' ) hash = sha1Hash;
else if ( hashAlgorithm === 'md5' ) hash = md5Hash;

hash('Fred');

function md5Hash(value){
    // ...
}

function sha1Hash(value){
    // ...
}

Ou você pode atribuir referências de função das propriedades do objeto:

// Example 3
var hashAlgorithms = {
    sha1: function(value) { /**/ },
    md5: function(value) { /**/ }
};

var hashAlgorithm = 'sha1';
var hash;

if ( hashAlgorithm === 'sha1' ) hash = hashAlgorithms.sha1;
else if ( hashAlgorithm === 'md5' ) hash = hashAlgorithms.md5;

hash('Fred');

Você pode atribuir a referência a uma função mantida por uma variável a outra omitindo os parênteses. Isso pode resultar em um erro fácil de cometer: tentar atribuir o valor de retorno de uma função a outra variável, mas atribuir acidentalmente a referência à função.

// Example 4
var a = getValue;
var b = a; // b is now a reference to getValue.
var c = b(); // b is invoked, so c now holds the value returned by getValue (41)

function getValue(){
    return 41;
}

Uma referência a uma função é como qualquer outro valor. Como você viu, uma referência pode ser atribuída a uma variável e o valor de referência dessa variável pode ser posteriormente atribuído a outras variáveis. Você pode passar referências a funções como qualquer outro valor, incluindo passar uma referência a uma função como o valor de retorno de outra função. Por exemplo:

// Example 5
// getHashingFunction returns a function, which is assigned
// to hash for later use:
var hash = getHashingFunction( 'sha1' );
// ...
hash('Fred');


// return the function corresponding to the given algorithmName
function getHashingFunction( algorithmName ){
    // return a reference to an anonymous function
    if (algorithmName === 'sha1') return function(value){ /**/ };
    // return a reference to a declared function
    else if (algorithmName === 'md5') return md5;
}

function md5Hash(value){
    // ...
}

Você não precisa atribuir uma referência de função a uma variável para invocá-la. Este exemplo, com base no exemplo 5, chamará getHashingFunction e, em seguida, invocará imediatamente a função retornada e passará seu valor de retorno para hashedValue.

// Example 6
var hashedValue = getHashingFunction( 'sha1' )( 'Fred' );

Uma Nota sobre Içamento #

Tenha em mente que, diferentemente das declarações de funções normais, as variáveis ​​que referenciam funções não são “elevadas”. No exemplo 2, as funções md5Hash e sha1Hash são definidas na parte inferior do script, mas estão disponíveis em todos os lugares imediatamente. Não importa onde você defina uma função, o intérprete a “eleva” para o topo de seu escopo, tornando-a imediatamente disponível. Este não é o caso das definições de variáveis, portanto, códigos como o seguinte serão interrompidos:

var functionVariable;

hoistedFunction(); // works, because the function is "hoisted" to the top of its scope
functionVariable(); // error: undefined is not a function.

function hoistedFunction(){}
functionVariable = function(){};

Função recursiva #

Uma função recursiva é simplesmente uma função, que chamaria a si mesma.

function factorial (n) {
    if (n <= 1) {
        return 1;
    }
    
    return n * factorial(n - 1);
}

A função acima mostra um exemplo básico de como realizar uma função recursiva para retornar um fatorial.

Outro exemplo, seria recuperar a soma de números pares em uma matriz.

function countEvenNumbers (arr) {
    // Sentinel value.  Recursion stops on empty array.
    if (arr.length < 1) {
        return 0;
    }
    // The shift() method removes the first element from an array 
    // and returns that element. This method changes the length of the array.
    var value = arr.shift();

    // `value % 2 === 0` tests if the number is even or odd
    // If it's even we add one to the result of counting the remainder of 
    // the array.  If it's odd, we add zero to it.
    return ((value % 2 === 0) ? 1 : 0) + countEvens(arr);
}

É importante que tais funções façam algum tipo de verificação de valor sentinela para evitar loops infinitos. No primeiro exemplo acima, quando n é menor ou igual a 1, a recursão para, permitindo que o resultado de cada chamada seja retornado na pilha de chamadas.

Usando a declaração de retorno #

A instrução return pode ser uma maneira útil de criar saída para uma função. A instrução return é especialmente útil se você ainda não souber em qual contexto a função será usada.

//An example function that will take a string as input and return 
//the first character of the string.

function firstChar (stringIn){
    return stringIn.charAt(0);
}

Agora, para usar esta função, você precisa colocá-la no lugar de uma variável em outro lugar do seu código:

Usando o resultado da função como argumento para outra função:

console.log(firstChar("Hello world"));

A saída do console será:

> H

A instrução return encerra a função

Se modificarmos a função no início, podemos demonstrar que a instrução return termina a função.

function firstChar (stringIn){
    console.log("The first action of the first char function");
    return stringIn.charAt(0);
    console.log("The last action of the first char function");
}

Executando esta função assim ficará assim:

console.log(firstChar("JS"));

Saída do console:

> The first action of the first char function
> J

Ele não imprimirá a mensagem após a instrução de retorno, pois a função foi encerrada.

Instrução de retorno abrangendo várias linhas:

Em JavaScript, você normalmente pode dividir uma linha de código em várias linhas para fins de legibilidade ou organização. Este é um JavaScript válido:

var
    name = "bob",
    age = 18;

Quando o JavaScript vê uma instrução incompleta como var, ele olha para a próxima linha para se completar. No entanto, se você cometer o mesmo erro com a instrução return, não obterá o que esperava.

return
    "Hi, my name is "+ name + ". " +
    "I'm "+ age + " years old.";

Este código retornará undefined porque return por si só é uma instrução completa em Javascript, então não irá procurar a próxima linha para se completar. Se você precisar dividir uma instrução return em várias linhas, coloque um valor próximo a return antes de dividi-lo, assim.

return "Hi, my name is " + name + ". " +
    "I'm " + age + " years old.";

function functionName( func )
{
    // Match:
    // - ^          the beginning of the string
    // - function   the word 'function'
    // - \s+        at least some white space
    // - ([\w\$]+)  capture one or more valid JavaScript identifier characters
    // - \(         followed by an opening brace
    //
    var result = /^function\s+([\w\$]+)\(/.exec( func.toString() )
    
    return result ? result[1] : ''
}
