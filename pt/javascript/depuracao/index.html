Variáveis ​​do interpretador interativo #

Observe que eles funcionam apenas nas ferramentas de desenvolvedor de determinados navegadores.

$_ fornece o valor de qualquer expressão avaliada por último.

"foo"             // "foo"
$_                // "foo"

$0 refere-se ao elemento DOM atualmente selecionado no Inspetor. Portanto, se <div id="foo"> estiver destacado:

$0                      // <div id="foo">
$0.getAttribute('id')   // "foo"

$1 refere-se ao elemento selecionado anteriormente, $2 ao selecionado antes dele, e assim por diante para $3 e $4.

Para obter uma coleção de elementos correspondentes a um seletor CSS, use $$(selector). Este é essencialmente um atalho para document.querySelectorAll.

var images = $$('img');  // Returns an array or a nodelist of all matching elements

Pontos de interrupção #

Os pontos de interrupção pausam seu programa quando a execução atinge um determinado ponto. Você pode então percorrer o programa linha por linha, observando sua execução e inspecionando o conteúdo de suas variáveis.

Existem três maneiras de criar pontos de interrupção.

  1. Do código, usando a instrução debugger;.
  2. No navegador, usando as Ferramentas do desenvolvedor.
  3. De um Ambiente de Desenvolvimento Integrado (IDE).

Declaração do depurador #

Você pode colocar uma instrução debugger; em qualquer lugar do seu código JavaScript. Quando o interpretador JS chegar a essa linha, ele interromperá a execução do script, permitindo que você inspecione as variáveis ​​e percorra seu código.

Ferramentas de desenvolvimento #

A segunda opção é adicionar um ponto de interrupção diretamente no código das Ferramentas do desenvolvedor do navegador.

Abrindo as Ferramentas do Desenvolvedor #

Chrome ou Firefox #

  1. Pressione F12 para abrir as Ferramentas do desenvolvedor
  2. Alterne para a guia Fontes (Chrome) ou guia Depurador (Firefox)
  3. Pressione Ctrl+P e digite o nome do seu arquivo JavaScript
  4. Pressione Enter para abri-lo.

Internet Explorer ou Edge #

  1. Pressione F12 para abrir as Ferramentas do desenvolvedor
  2. Alterne para a guia Depurador.
  3. Use o ícone de pasta próximo ao canto superior esquerdo da janela para abrir um painel de seleção de arquivos; você pode encontrar seu arquivo JavaScript lá.

Safári #

  1. Pressione Comando+Opção+C para abrir as Ferramentas do desenvolvedor
  2. Mude para a guia Recursos
  3. Abra a pasta “Scripts” no painel do lado esquerdo
  4. Selecione seu arquivo JavaScript.

Adicionando um ponto de interrupção nas Ferramentas do desenvolvedor #

Depois de abrir o arquivo JavaScript nas Ferramentas do desenvolvedor, você pode clicar em um número de linha para colocar um ponto de interrupção. Na próxima vez que seu programa for executado, ele fará uma pausa lá.

Observação sobre fontes minificadas: se sua fonte for minificada, você pode imprimir com Pretty Print (converter para formato legível). No Chrome, isso é feito clicando no botão {} no canto inferior direito do visualizador de código-fonte.

IDE #

Código do Visual Studio (VSC) #

O VSC tem suporte integrado para depuração de JavaScript.

  1. Clique no botão Depurar à esquerda ou Ctrl+Shift+D
  2. Se ainda não tiver feito, crie um arquivo de configuração de inicialização (launch.json) pressionando o ícone de engrenagem.
  3. Execute o código do VSC pressionando o botão verde ou pressione F5.

Adicionando um ponto de interrupção no VSC #

Clique ao lado do número da linha em seu arquivo de origem JavaScript para adicionar um ponto de interrupção (ele será marcado em vermelho). Para excluir o ponto de interrupção, clique no círculo vermelho novamente.

Dica: você também pode utilizar os pontos de interrupção condicionais nas ferramentas de desenvolvimento do navegador. Isso ajuda a pular pausas desnecessárias na execução. Cenário de exemplo: você deseja examinar uma variável em um loop exatamente na 5th iteração.

Usando setters e getters para descobrir o que mudou uma propriedade #

Digamos que você tenha um objeto como este:

var myObject = {
    name: 'Peter'
}

Mais tarde em seu código, você tenta acessar myObject.name e obtém George em vez de Peter. Você começa a se perguntar quem mudou e onde exatamente foi alterado. Existe uma maneira de colocar um debugger (ou outra coisa) em cada conjunto (toda vez que alguém faz myObject.name = 'something'):

var myObject = {
    _name: 'Peter',
    set name(name){debugger;this._name=name},
    get name(){return this._name}
}

Note que renomeamos name para _name e vamos definir um setter e um getter para name.

set name é o setter. Esse é um ponto ideal onde você pode colocar debugger, console.trace(), ou qualquer outra coisa que você precise para depurar. O setter irá definir o valor para name em _name. O getter (a parte get name) lerá o valor de lá. Agora temos um objeto totalmente funcional com funcionalidade de depuração.

Na maioria das vezes, porém, o objeto que é alterado não está sob nosso controle. Felizmente, podemos definir setters e getters em objetos existentes para depurá-los.

// First, save the name to _name, because we are going to use name for setter/getter
otherObject._name = otherObject.name;

// Create setter and getter
Object.defineProperty(otherObject, "name", {
    set: function(name) {debugger;this._name = name},
    get: function() {return this._name}
});

Confira setters e getters no MDN para mais informações.

Usando o console #

Em muitos ambientes, você tem acesso a um objeto console global que contém alguns métodos básicos para comunicação com dispositivos de saída padrão. Geralmente, esse será o console JavaScript do navegador (consulte Chrome, [Firefox](https: //developer.mozilla.org/en-US/docs/Tools/Browser_Console), Safari e [Edge](https://developer. microsoft.com/en-us/microsoft-edge/platform/documentation/f12-devtools-guide/console/) para obter mais informações).

// At its simplest, you can 'log' a string
console.log("Hello, World!");

// You can also log any number of comma-separated values
console.log("Hello", "World!");

// You can also use string substitution
console.log("%s %s", "Hello", "World!");

// You can also log any variable that exist in the same scope
var arr = [1, 2, 3];
console.log(arr.length, this);

Você pode usar diferentes métodos de console para destacar sua saída de diferentes maneiras. Outros métodos também são úteis para depuração mais avançada.

Para obter mais documentação, informações sobre compatibilidade e instruções sobre como abrir o console do navegador, consulte o tópico Console.

Nota: se você precisar dar suporte ao IE9, remova console.log ou envolva suas chamadas da seguinte forma, porque console é indefinido até que as Ferramentas do Desenvolvedor sejam abertas:

if (console) { //IE9 workaround
    console.log("test");
}

Pausando a execução automaticamente #

No Google Chrome, você pode pausar a execução sem precisar colocar pontos de interrupção.

Pause on Exception: Enquanto este botão estiver ativado, se seu programa atingir uma exceção não tratada, o programa fará uma pausa como se tivesse atingido um ponto de interrupção. O botão pode ser encontrado próximo aos Controles de Execução e é útil para localizar erros.

Você também pode pausar a execução quando uma tag HTML (nó DOM) é modificada ou quando seus atributos são alterados. Para fazer isso, clique com o botão direito do mouse no nó DOM na guia Elements e selecione “Break on…”.

Inspetor de elementos #

Clicando no botão Selecione um elemento na página para inspecioná-lo no canto superior esquerdo da guia Elementos no Chrome ou na guia Inspetor no Firefox, disponível nas Ferramentas do desenvolvedor , e clicar em um elemento da página destaca o elemento e [o atribui à variável $0](https://www.wikiod.com/pt/javascript/depuracao#Variáveis ​​do interpretador interativo =201607211329474646179).

O Elements inspector pode ser usado de várias maneiras, por exemplo:

  1. Você pode verificar se o seu JS está manipulando o DOM da maneira que você espera,
  2. Você pode depurar seu CSS com mais facilidade, ao ver quais regras afetam o elemento (guia Estilos no Chrome)
  3. Você pode brincar com CSS e HTML sem recarregar a página.

Além disso, o Chrome lembra as últimas 5 seleções na guia Elementos. $0 é a seleção atual, enquanto $1 é a seleção anterior. Você pode ir até $4. Dessa forma, você pode depurar facilmente vários nós sem alternar constantemente a seleção para eles.

Você pode ler mais em Google Developers.

Percorrendo o código #

Depois de pausar a execução em um ponto de interrupção, convém seguir a execução linha por linha para observar o que acontece. [Abra as Ferramentas de Desenvolvedor do seu navegador](https://www.wikiod.com/pt/javascript/depuracao#Pontos de interrupção) e procure os ícones de Controle de Execução. (Este exemplo usa os ícones no Google Chrome, mas eles serão semelhantes em outros navegadores.)

Retomar: Retomar a execução. Atalho:F8(Chrome, Firefox)

Step Over: Execute a próxima linha de código. Se essa linha contiver uma chamada de função, execute a função inteira e vá para a próxima linha, em vez de pular para onde quer que a função esteja definida. Atalho: F10(Chrome, Firefox, IE/Edge), F6(Safari)

Passar em: Execute a próxima linha de código. Se essa linha contiver uma chamada de função, pule na função e faça uma pausa nela. Atalho: F11(Chrome, Firefox, IE/Edge), F7(Safari)

Step Out: Execute o restante da função atual, volte para onde a função foi chamada e faça uma pausa na próxima instrução. Atalho: Shift + F11(Chrome, Firefox, IE/Edge), F8(Safari)

Use-os em conjunto com o Call Stack, que informará em qual função você está atualmente, qual função chamou essa função e assim por diante.

Consulte o guia do Google sobre “How to Step Through the Code” para obter mais detalhes e conselhos .

Quebra quando uma função é chamada #

Para funções nomeadas (não anônimas), você pode interromper quando a função for executada.

debug(functionName);

Na próxima vez que a função functionName for executada, o depurador irá parar em sua primeira linha.
