Sintaxe #

  • let regex = /pattern/[flags]
  • let regex = new RegExp(’pattern’, [flags])
  • let ismatch = regex.test(’text')
  • deixe resultados = regex.exec(’texto')

Parâmetros #

Bandeiras Detalhes
g **global. Todas as partidas (não retorne na primeira partida).
m mmultilinha. Faz com que ^ & $ correspondam ao início/fim de cada linha (não apenas início/fim da string).
eu **insensível. Correspondência sem distinção entre maiúsculas e minúsculas (ignora maiúsculas e minúsculas de [a-zA-Z]).
você unicode : as strings de padrão são tratadas como UTF-16. Também faz com que as sequências de escape correspondam a caracteres Unicode.
e sticky: corresponde apenas do índice indicado pela propriedade lastIndex desta expressão regular na string de destino (e não tenta corresponder a partir de nenhum índice posterior).

O objeto RegExp é tão útil quanto seu conhecimento de Expressões Regulares é forte. Veja aqui para uma cartilha introdutória ou veja [MDN](https://developer.mozilla.org/en-US/docs/ Web/JavaScript/Reference/Global_Objects/RegExp) para uma explicação mais detalhada.

Criando um objeto RegExp #

Criação de padrões #

Recomenda-se usar este formulário apenas ao criar regex a partir de variáveis ​​dinâmicas.

Use quando a expressão puder mudar ou a expressão for gerada pelo usuário.

var re = new RegExp(".*");

Com bandeiras:

var re = new RegExp(".*", "gmi");

Com uma barra invertida: (isso deve ser escapado porque o regex é especificado com uma string)

var re = new RegExp("\\w*");

Inicialização estática #

Use quando souber que a expressão regular não será alterada e souber qual é a expressão antes do tempo de execução.

var re = /.*/;

Com bandeiras:

var re = /.*/gmi;

Com uma barra invertida: (isso não deve ser escapado porque o regex é especificado em um literal)

var re = /\w*/;

Sinalizadores RegExp #

Existem vários sinalizadores que você pode especificar para alterar o comportamento do RegEx. Os sinalizadores podem ser anexados ao final de um literal regex, como especificar gi em /test/gi, ou podem ser especificados como o segundo argumento para o construtor RegExp, como em new RegExp('test ', 'gi').

g - Global. Encontra todas as correspondências em vez de parar após a primeira.

i - Ignora maiúsculas e minúsculas. /[a-z]/i é equivalente a /[a-zA-Z]/.

m - Multilinha. ^ e $ correspondem ao início e fim de cada linha, respectivamente, tratando \n e \r como delimitadores em vez de simplesmente o início e o fim de toda a string.

u - Unicode. Se este sinalizador não for suportado, você deve combinar caracteres Unicode específicos com \uXXXX onde XXXX é o valor do caractere em hexadecimal.

y - Encontra todas as correspondências consecutivas/adjacentes.

Verifique se a string contém padrão usando .test() #

var re = /[a-z]+/;
if (re.test("foo")) {
    console.log("Match exists.");
}

O método test realiza uma pesquisa para ver se uma expressão regular corresponde a uma string. A expressão regular [a-z]+ buscará uma ou mais letras minúsculas. Como o padrão corresponde à string, “match existe” será registrado no console.

Combinando com .exec() #

Corresponder usando .exec() #

RegExp.prototype.exec(string) retorna um array de capturas, ou null se não houver correspondência.

var re = /([0-9]+)[a-z]+/;
var match = re.exec("foo123bar");

match.index é 3, a localização (baseada em zero) da correspondência.

match[0] é a string de correspondência completa.

match[1] é o texto correspondente ao primeiro grupo capturado. match[n] seria o valor do nth grupo capturado.

Percorrer correspondências usando .exec() #

var re = /a/g;
var result;
while ((result = re.exec('barbatbaz')) !== null) {
    console.log("found '" + result[0] + "', next exec starts at index '" + re.lastIndex + "'");
}

Saída esperada

encontrado ‘a’, o próximo exec começa no índice ‘2’ encontrado ‘a’, o próximo exec começa no índice ‘5’ encontrado ‘a’, o próximo exec começa no índice ‘8’

Usando RegExp com Strings #

O objeto String tem os seguintes métodos que aceitam expressões regulares como argumentos.

  • "string".match(...
  • "string".replace(...
  • "string".split(...
  • "string".search(...

Combinar com RegExp #

console.log("string".match(/[i-n]+/));
console.log("string".match(/(r)[i-n]+/));

Saída esperada

Matriz [“in”] Matriz [“rin”, “r”]

Substitua por RegExp #

console.log("string".replace(/[i-n]+/, "foo"));

Saída esperada

olho de corda

Dividir com RegExp #

console.log("stringstring".split(/[i-n]+/));

Saída esperada

Matriz [“str”, “gstr”, “g”]

Pesquise com RegExp #

.search() retorna o índice no qual uma correspondência é encontrada ou -1.

console.log("string".search(/[i-n]+/));
console.log("string".search(/[o-q]+/));

Saída esperada

3 -1

Grupos RegExp #

JavaScript suporta vários tipos de grupo em suas Expressões Regulares, capture groups, non-capture groups e look-aheads. Atualmente, não há suporte para look-behind.

Capturar #

Às vezes, a correspondência desejada depende de seu contexto. Isto significa que um simples RegExp irá encontrar a parte da String que é de interesse, então a solução é escrever um grupo de captura (padrão). Os dados capturados podem então ser referenciados como…

  • Substituição de string "$n" onde n é o n th grupo de captura (começando de 1)
  • O argumento n th em uma função de retorno de chamada
  • Se o RegExp não estiver sinalizado como g, o n+1 th item em um str.match Array retornado
  • Se o RegExp estiver sinalizado como g, str.match descarta as capturas, use re.exec

Digamos que haja uma String onde todos os sinais + precisam ser substituídos por um espaço, mas somente se eles seguirem um caractere de letra. Isso significa que uma correspondência simples incluiria esse caractere de letra e também seria removida. Capturá-lo é a solução, pois significa que a letra correspondente pode ser preservada.

let str = "aa+b+cc+1+2",
    re = /([a-z])\+/g;

// String replacement
str.replace(re, '$1 '); // "aa b cc 1+2"
// Function replacement
str.replace(re, (m, $1) => $1 + ' '); // "aa b cc 1+2"

Não Captura #

Usando o formulário (?:pattern), eles funcionam de maneira semelhante para capturar grupos, exceto que não armazenam o conteúdo do grupo após a partida.

Eles podem ser particularmente úteis se outros dados estiverem sendo capturados dos quais você não deseja mover os índices, mas precisa fazer alguma correspondência avançada de padrões, como um OR

let str = "aa+b+cc+1+2",
    re = /(?:\b|c)([a-z])\+/g;

str.replace(re, '$1 '); // "aa+b c 1+2"

Olhe para frente #

Se a correspondência desejada depende de algo que a segue, em vez de corresponder e capturá-la, é possível usar uma antecipação para testá-la, mas não incluí-la na correspondência. Uma antecipação positiva tem a forma (?=padrão), uma antecipação negativa (onde a correspondência de expressão só acontece se o padrão de antecipação não corresponder) tem a forma (?!pattern)

let str = "aa+b+cc+1+2",
    re = /\+(?=[a-z])/g;

str.replace(re, ' '); // "aa b cc+1+2"

Substituindo correspondência de string por uma função de retorno de chamada #

String#replace pode ter uma função como seu segundo argumento para que você possa fornecer uma substituição com base em alguma lógica.

"Some string Some".replace(/Some/g, (match, startIndex, wholeString) => {
  if(startIndex == 0){
    return 'Start';
  } else {
    return 'End';
  }
});
// will return Start string End

Biblioteca de modelos de uma linha

let data = {name: 'John', surname: 'Doe'}
"My name is {surname}, {name} {surname}".replace(/(?:{(.+?)})/g, x => data[x.slice(1,-1)]);

// "My name is Doe, John Doe"

Usando Regex.exec() com parênteses regex para extrair correspondências de uma string #

Às vezes você não quer simplesmente substituir ou remover a string. Às vezes, você deseja extrair e processar correspondências. Aqui um exemplo de como você manipula correspondências.

O que é uma partida? Quando uma substring compatível é encontrada para toda a regex na string, o comando exec produz uma correspondência. Uma correspondência é uma matriz composta primeiramente por toda a substring que correspondeu e todos os parênteses na correspondência.

Imagine uma string html:

<html>
<head></head>
<body>
  <h1>Example</h1>
  <p>Look a this great link : <a href="https://stackoverflow.com">Stackoverflow</a> http://anotherlinkoutsideatag</p>
  Copyright <a href="https://stackoverflow.com">Stackoverflow</a>
</body>

Você deseja extrair e obter todos os links dentro de uma tag a. A princípio, aqui o regex que você escreve:

var re = /<a[^>]*href="https?:\/\/.*"[^>]*>[^<]*<\/a>/g;

Mas agora, imagine que você quer o href e a âncora de cada link. E você quer isso junto. Você pode simplesmente adicionar um novo regex para cada partida OU você pode usar parênteses :

var re = /<a[^>]*href="(https?:\/\/.*)"[^>]*>([^<]*)<\/a>/g; 
var str = '<html>\n    <head></head>\n    <body>\n        <h1>Example</h1>\n        <p>Look a this great link : <a href="https://stackoverflow.com">Stackoverflow</a> http://anotherlinkoutsideatag</p>\n\n        Copyright <a href="https://stackoverflow.com">Stackoverflow</a>\n    </body>\';\n';
var m;
var links = [];

while ((m = re.exec(str)) !== null) {
    if (m.index === re.lastIndex) {
        re.lastIndex++;
    }
    console.log(m[0]); // The all substring
    console.log(m[1]); // The href subpart
    console.log(m[2]); // The anchor subpart

    links.push({
      match : m[0],   // the entire match
      href : m[1],    // the first parenthesis => (https?:\/\/.*)
      anchor : m[2],  // the second one => ([^<]*)
    });
}

No final do loop, você tem um array de link com anchor e href e pode usá-lo para escrever markdown, por exemplo:

links.forEach(function(link) {
  console.log('[%s](%s)', link.anchor, link.href);
});
