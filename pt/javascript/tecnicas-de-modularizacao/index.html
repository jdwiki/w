Definição de módulo universal (UMD) #

O padrão UMD (Universal Module Definition) é usado quando nosso módulo precisa ser importado por vários carregadores de módulo diferentes (por exemplo, AMD, CommonJS).

O padrão em si consiste em duas partes:

  1. Um IIFE (Immediately-Invoked Function Expression) que verifica o carregador de módulo que está sendo implementado pelo usuário. Isso levará dois argumentos; root (uma referência this ao escopo global) e factory (a função onde declaramos nosso módulo).

  2. Uma função anônima que cria nosso módulo. Isso é passado como o segundo argumento para a parte IIFE do padrão. Esta função recebe qualquer número de argumentos para especificar as dependências do módulo.

No exemplo abaixo, verificamos se há AMD e, em seguida, CommonJS. Se nenhum desses carregadores estiver em uso, voltaremos a disponibilizar o módulo e suas dependências globalmente.

(function (root, factory) {
    if (typeof define === 'function' && define.amd) {
        // AMD. Register as an anonymous module.
        define(['exports', 'b'], factory);
    } else if (typeof exports === 'object' && typeof exports.nodeName !== 'string') {
        // CommonJS
        factory(exports, require('b'));
    } else {
        // Browser globals
        factory((root.commonJsStrict = {}), root.b);
    }
}(this, function (exports, b) {
    //use b in some fashion.

    // attach properties to the exports object to define
    // the exported module properties.
    exports.action = function () {};
}));

Expressões de função invocadas imediatamente (IIFE) #

Expressões de função invocadas imediatamente podem ser usadas para criar um escopo privado ao produzir uma API pública.

var Module = (function() {
  var privateData = 1;

  return {
    getPrivateData: function() {
      return privateData;
    }
  };
})();
Module.getPrivateData(); // 1
Module.privateData; // undefined

Consulte o [Module Pattern](https://www.wikiod.com/pt/javascript/padroes-de-design-de-criacao#Módulos e Padrões de Módulos Reveladores) para obter mais detalhes.

Definição de módulo assíncrono (AMD) #

AMD é um sistema de definição de módulo que tenta resolver alguns dos problemas comuns com outros sistemas como CommonJS e encerramentos anônimos.

A AMD aborda esses problemas:

  • Registrando a função de fábrica chamando define(), em vez de executá-la imediatamente
  • Passando dependências como uma matriz de nomes de módulos, que são então carregados, em vez de usar globais
  • Apenas executando a função de fábrica depois que todas as dependências foram carregadas e executadas
  • Passando os módulos dependentes como argumentos para a função de fábrica

A chave aqui é que um módulo pode ter uma dependência e não segurar tudo enquanto espera o carregamento, sem que o desenvolvedor tenha que escrever um código complicado.

Aqui está um exemplo de AMD:

// Define a module "myModule" with two dependencies, jQuery and Lodash
define("myModule", ["jquery", "lodash"], function($, _) {
    // This publicly accessible object is our module
    // Here we use an object, but it can be of any type
    var myModule = {};

    var privateVar = "Nothing outside of this module can see me";

    var privateFn = function(param) {
        return "Here's what you said: " + param;
    };

    myModule.version = 1;

    myModule.moduleMethod = function() {
        // We can still access global variables from here, but it's better
        // if we use the passed ones
        return privateFn(windowTitle);
    };

    return myModule;
});

Os módulos também podem pular o nome e ser anônimos. Quando isso é feito, eles geralmente são carregados pelo nome do arquivo.

define(["jquery", "lodash"], function($, _) { /* factory */ });

Eles também podem pular dependências:

define(function() { /* factory */ });

Alguns carregadores AMD suportam a definição de módulos como objetos simples:

define("myModule", { version: 1, value: "sample string" });

CommonJS - Node.js #

CommonJS é um padrão de modularização popular usado no Node.js.

O sistema CommonJS é centrado em torno de uma função require() que carrega outros módulos e uma propriedade exports que permite que os módulos exportem métodos acessíveis publicamente.

Aqui está um exemplo de CommonJS, vamos carregar o módulo fs do Lodash e do Node.js:

// Load fs and lodash, we can use them anywhere inside the module
var fs = require("fs"),
    _ = require("lodash");

var myPrivateFn = function(param) {
    return "Here's what you said: " + param;
};

// Here we export a public `myMethod` that other modules can use
exports.myMethod = function(param) {
    return myPrivateFn(param);
};

Você também pode exportar uma função como o módulo inteiro usando module.exports:

module.exports = function() {
    return "Hello!";
};

Módulos ES6 #

No ECMAScript 6, ao usar a sintaxe do módulo (import/export), cada arquivo se torna seu próprio módulo com um namespace privado. Funções e variáveis ​​de nível superior não poluem o namespace global. Para expor funções, classes e variáveis ​​para outros módulos importarem, você pode usar a palavra-chave export.

Observação: Embora este seja o método oficial para criar módulos JavaScript, ele não é suportado por nenhum dos principais navegadores no momento. No entanto, os Módulos ES6 são suportados por muitos transpiladores.

export function greet(name) {
    console.log("Hello %s!", name);
}

var myMethod = function(param) {
    return "Here's what you said: " + param;
};

export {myMethod}

export class MyClass {
    test() {}
}

Usando Módulos #

Importar módulos é tão simples quanto especificar seu caminho:

import greet from "mymodule.js";

greet("Bob");

Isso importa apenas o método myMethod do nosso arquivo mymodule.js.

Também é possível importar todos os métodos de um módulo:

import * as myModule from "mymodule.js";

myModule.greet("Alice");

Você também pode importar métodos com um novo nome:

import { greet as A, myMethod as B } from "mymodule.js";

Mais informações sobre os Módulos ES6 podem ser encontradas no tópico Módulos.
