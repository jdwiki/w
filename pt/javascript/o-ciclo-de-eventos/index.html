O loop de eventos em um navegador da web #

A grande maioria dos ambientes JavaScript modernos funciona de acordo com um event loop. Este é um conceito comum em programação de computadores que essencialmente significa que seu programa espera continuamente que coisas novas aconteçam e, quando isso acontece, reage a elas. O ambiente host chama seu programa, gerando um “turn” ou “tick” ou “task” no loop de eventos, que então executa até a conclusão. Quando esse turno termina, o ambiente host espera que algo mais aconteça, antes que tudo isso comece.

Um exemplo simples disso está no navegador. Considere o seguinte exemplo:

<!DOCTYPE html>
<title>Event loop example</title>

<script>
console.log("this a script entry point");

document.body.onclick = () => {
  console.log("onclick");
};

setTimeout(() => {
  console.log("setTimeout callback log 1");
  console.log("setTimeout callback log 2");
}, 100);
</script>

Neste exemplo, o ambiente do host é o navegador da web.

  1. O analisador HTML primeiro executará o <script>. Ele será executado até a conclusão.
  2. A chamada para setTimeout informa ao navegador que, após 100 milissegundos, ele deve enfileirar uma tarefa para executar a ação especificada.
  3. Enquanto isso, o event loop é responsável por verificar continuamente se há algo mais a fazer: por exemplo, renderizar a página da web.
  4. Após 100 milissegundos, se o loop de eventos não estiver ocupado por algum outro motivo, ele verá a tarefa que o setTimeout enfileira e executará a função, registrando essas duas instruções.
  5. A qualquer momento, se alguém clicar no corpo, o navegador postará uma tarefa no loop de eventos para executar a função do manipulador de cliques. O loop de eventos, conforme verifica continuamente o que fazer, verá isso e executará essa função.

Você pode ver como neste exemplo existem vários tipos diferentes de pontos de entrada no código JavaScript, que o loop de eventos invoca:

  • O elemento <script> é invocado imediatamente
  • A tarefa setTimeout é postada no loop de eventos e executada uma vez
  • A tarefa do manipulador de cliques pode ser postada muitas vezes e executada sempre

Cada volta do loop de eventos é responsável por muitas coisas; apenas alguns deles invocarão essas tarefas JavaScript. Para obter detalhes completos, consulte a especificação HTML

Uma última coisa: o que queremos dizer ao dizer que cada tarefa de loop de eventos “executa até a conclusão”? Queremos dizer que geralmente não é possível interromper um bloco de código que está na fila para ser executado como uma tarefa e nunca é possível executar código intercalado com outro bloco de código. Por exemplo, mesmo se você clicasse no momento perfeito, você nunca poderia obter o código acima para registrar "onclick" entre os dois setTimeout callback log 1/2"s. postagem funciona; é cooperativo e baseado em fila, em vez de preemptivo.

Operações assíncronas e o loop de eventos #

Muitas operações interessantes em ambientes comuns de programação JavaScript são assíncronas. Por exemplo, no navegador vemos coisas como

window.setTimeout(() => {
  console.log("this happens later");
}, 100);

e no Node.js vemos coisas como

fs.readFile("file.txt", (err, data) => {
  console.log("data");
});

Como isso se encaixa com o loop de eventos?

Como isso funciona é que, quando essas instruções são executadas, elas dizem ao ambiente host (ou seja, o navegador ou o tempo de execução do Node.js, respectivamente) para sair e fazer algo, provavelmente em outro thread. Quando o ambiente do host terminar de fazer isso (respectivamente, esperando 100 milissegundos ou lendo o arquivo file.txt), ele postará uma tarefa no loop de eventos, dizendo “chame o retorno de chamada que recebi anteriormente com esses argumentos”.

O loop de eventos está então ocupado fazendo sua coisa: renderizando a página da Web, ouvindo a entrada do usuário e procurando continuamente por tarefas postadas. Quando ele vir essas tarefas postadas para chamar os retornos de chamada, ele chamará de volta para o JavaScript. É assim que você obtém um comportamento assíncrono!
