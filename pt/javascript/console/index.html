O console de depuração de um navegador ou console da web geralmente é usado pelos desenvolvedores para identificar erros, entender o fluxo de execução, registrar dados e muitos outros propósitos em tempo de execução. Esta informação é acessada através do objeto console.

Sintaxe #

  • void console.log(obj1 [, obj2, …, objN]);
  • void console.log(msg [, sub1, …, subN]);

Parâmetros #

Parâmetro Descrição
obj1 … objN Uma lista de objetos JavaScript cujas representações de string são geradas no console
mensagem Uma string JavaScript contendo zero ou mais strings de substituição.
sub1 … subN Objetos JavaScript com os quais substituir strings de substituição em msg.

A informação exibida por um debugging/web console é disponibilizada através dos múltiplos métodos do objeto Javascript console que podem ser consultados através do console.dir(console). Além da propriedade console.memory, os métodos exibidos geralmente são os seguintes (retirados da saída do Chromium):

Abrindo o console #

Na maioria dos navegadores atuais, o JavaScript Console foi integrado como uma guia nas Ferramentas do desenvolvedor. As teclas de atalho listadas abaixo abrirão as Ferramentas do desenvolvedor, pode ser necessário alternar para a guia direita depois disso.

Cromada #

Abrindo o painel “Console” do DevTools do Chrome:

  • Windows / Linux: qualquer uma das seguintes opções.

    • Ctrl + Shift + J
    • Ctrl + Shift + I, then click on the “Web Console” tab or press ESC to toggle the console on and off
    • F12, then click on the “Console” tab or press ESC to toggle the console on and off

  • Mac OS: Cmd + Opt + J

Raposa de fogo #

Abrindo o painel “Console” nas Ferramentas do desenvolvedor do Firefox:

  • Windows / Linux: qualquer uma das seguintes opções.

    • Ctrl + Shift + K
    • Ctrl + Shift + I, then click on the “Web Console” tab or press ESC to toggle the console on and off
    • F12, then click on the “Web Console” tab or press ESC to toggle the console on and off

  • Mac OS: Cmd + Opt + K

Edge e Internet Explorer #

Abrindo o painel “Console” no F12 Developer Ferramentas:

  • F12 e clique na guia “Console”

Safári #

Abrindo o painel “Console” no Web Inspector do Safari, você deve primeiro habilitar o menu de desenvolvimento nas Preferências do Safari

Então você pode escolher “Desenvolver->Mostrar Console de Erros” nos menus ou pressionar ⌘ + Opção + C

Ópera #

Abrindo o “Console” na ópera:

  • Ctrl + Shift + I, depois clique na aba “Console”

Compatibilidade #

Ao usar ou emular o Internet Explorer 8 ou versões anteriores (por exemplo, através do Modo de Exibição de Compatibilidade / Modo Empresarial), o console somente será definido quando as Ferramentas do Desenvolvedor estiverem ativas, portanto, as instruções console.log() podem causar uma exceção e impedir a execução do código. Para atenuar isso, você pode verificar se o console está disponível antes de registrar:

if (typeof window.console !== 'undefined')
{
   console.log("Hello World");
}

Ou, no início do seu script, você pode identificar se o console está disponível e, se não estiver, definir uma função nula para capturar todas as suas referências e evitar exceções.

if (!window.console)
{ 
    console = {log: function() {}}; 
}

Observe que este segundo exemplo interromperá todos os logs do console, mesmo que a janela do desenvolvedor tenha sido aberta.

Usar este segundo exemplo impedirá o uso de outras funções como console.dir(obj) a menos que isso seja adicionado especificamente.

Medindo o tempo - console.time() #

console.time() pode ser usado para medir quanto tempo uma tarefa em seu código leva para ser executada.

Chamar console.time([label]) inicia um novo cronômetro. Quando console.timeEnd([label]) é chamado, o tempo decorrido, em milissegundos, desde o a chamada .time() original é calculada e registrada. Devido a este comportamento, você pode chamar .timeEnd() várias vezes com o mesmo rótulo para registrar o tempo decorrido desde que a chamada original .time() foi feita.

Exemplo 1:

console.time('response in');

alert('Click to continue');
console.timeEnd('response in');

alert('One more time');
console.timeEnd('response in');

emitirá:

response in: 774.967ms
response in: 1402.199ms

Exemplo 2:

var elms = document.getElementsByTagName('*'); //select all elements on the page

console.time('Loop time');

for (var i = 0; i < 5000; i++) {
    for (var j = 0, length = elms.length; j < length; j++) {
        // nothing to do ...
    }
}

console.timeEnd('Loop time');

emitirá:

Loop time: 40.716ms

Formatando a saída do console #

Muitos dos métodos de impressão do console também podem lidar com formatação de string semelhante a C, usando tokens %:

console.log('%s has %d points', 'Sam', 100);

Exibe Sam tem 100 pontos.

A lista completa de especificadores de formato em Javascript é:

Especificador Saída
%s Formata o valor como uma string
%i ou %d Formata o valor como um inteiro
%f Formata o valor como um valor de ponto flutuante
%o Formata o valor como um elemento DOM expansível
%O Formata o valor como um objeto JavaScript expansível
%c Aplica regras de estilo CSS à string de saída conforme especificado pelo segundo parâmetro

Estilo avançado #

Quando o especificador de formato CSS (%c) é colocado no lado esquerdo da string, o método print aceitará um segundo parâmetro com regras CSS que permitem um controle refinado sobre a formatação dessa string:

console.log('%cHello world!', 'color: blue; font-size: xx-large');

Exibe:

É possível usar vários especificadores de formato %c:

  • qualquer substring à direita de um %c tem um parâmetro correspondente no método de impressão;

  • este parâmetro pode ser uma string vazia, caso não seja necessário aplicar regras CSS a essa mesma substring;

  • se dois especificadores de formato %c forem encontrados, a 1st (envolvida em %c) e a 2nd substring terão suas regras definidas na 2< sup>nd e 3rd parâmetro do método de impressão, respectivamente.

  • se três especificadores de formato %c forem encontrados, então as substrings 1st, 2nd e 3rd terão suas regras definidas em o 2nd , 3rd e 4th parâmetro respectivamente, e assim por diante…

    console.log("%cHello %cWorld%c!!", // string to be printed “color: blue;”, // applies color formatting to the 1st substring “font-size: xx-large;”, // applies font formatting to the 2nd substring “/* no CSS rule*/” // does not apply any rule to the remaing substring );

Usando grupos para recuar a saída #

A saída pode ser identificada e incluída em um grupo recolhível no console de depuração com os seguintes métodos:

  • console.groupCollapsed(): cria um grupo de entradas recolhidas que podem ser expandidas através do botão de divulgação para revelar todas as entradas realizadas após este método ser invocado;
  • console.group(): cria um grupo expandido de entradas que podem ser recolhidas para ocultar as entradas após este método ser invocado.

A identificação pode ser removida para entradas posteriores usando o seguinte método:

  • console.groupEnd(): sai do grupo atual, permitindo que novas entradas sejam impressas no grupo pai depois que este método for invocado.

Imprimindo no console de depuração de um navegador #

O console de depuração de um navegador pode ser usado para imprimir mensagens simples. Esta depuração ou console web pode ser aberta diretamente no navegador (tecla F12 na maioria dos navegadores – veja Observações abaixo para mais informações) e o método log do objeto Javascript console pode ser invocado digitando o seguinte:

console.log('My message');

Então, pressionando Enter, isso exibirá Minha mensagem no console de depuração.

console.log() pode ser chamado com qualquer número de argumentos e variáveis ​​disponíveis no escopo atual. Vários argumentos serão impressos em uma linha com um pequeno espaço entre eles.

var obj = { test: 1 };
console.log(['string'], 1, obj, window);

O método log exibirá o seguinte no console de depuração:

['string']  1  Object { test: 1 }  Window { /* truncated */ }

Além de strings simples, console.log() pode lidar com outros tipos, como arrays, objetos, datas, funções, etc.:

console.log([0, 3, 32, 'a string']);
console.log({ key1: 'value', key2: 'another value'});

Exibe:

Array [0, 3, 32, 'a string']
Object { key1: 'value', key2: 'another value'}

Objetos aninhados podem ser recolhidos:

console.log({ key1: 'val', key2: ['one', 'two'], key3: { a: 1, b: 2 } });

Exibe:

Object { key1: 'val', key2: Array[2], key3: Object }

Certos tipos, como objetos Date e functions, podem ser exibidos de forma diferente:

console.log(new Date(0));
console.log(function test(a, b) { return c; });

Exibe:

Wed Dec 31 1969 19:00:00 GMT-0500 (Eastern Standard Time)
function test(a, b) { return c; }

Outros métodos de impressão #

Além do método log, os navegadores modernos também suportam métodos semelhantes:

  • console.info – pequeno ícone informativo (ⓘ) aparece no lado esquerdo da(s) string(s) ou objeto(s) impresso(s).

  • console.warn – pequeno ícone de aviso (!) aparece no lado esquerdo. Em alguns navegadores, o plano de fundo do log é amarelo.

  • console.error – ícone de pequenos tempos (⊗) aparece no lado esquerdo. Em alguns navegadores, o plano de fundo do log é vermelho.

  • console.timeStamp – emite a hora atual e uma string especificada, mas não é padrão:

    console.timeStamp('msg');

Exibe:

  00:00:00.001 msg

  • console.trace – gera o rastreamento de pilha atual ou exibe a mesma saída que o método log se invocado no escopo global.

    function sec() {
   first();
}
function first() {
   console.trace();
}
sec();

Exibe:

  first
  sec
  (anonymous function)

A imagem acima mostra todas as funções, com exceção de timeStamp, na versão 56 do Chrome.

Esses métodos se comportam de maneira semelhante ao método log e em diferentes consoles de depuração podem ser renderizados em cores ou formatos diferentes.

Em alguns depuradores, as informações de objetos individuais podem ser expandidas clicando no texto impresso ou em um pequeno triângulo (►) que se refere às respectivas propriedades do objeto. Essas propriedades de objeto em colapso podem ser abertas ou fechadas no log. Veja o console.dir para informações adicionais sobre isso

Incluindo um rastreamento de pilha ao registrar - console.trace() #

function foo() {
  console.trace('My log statement');
}

foo();

Irá exibir isso no console:

My log statement       VM696:1
  foo                  @ VM696:1
  (anonymous function) @ (program):1

Observação: quando disponível, também é útil saber que o mesmo rastreamento de pilha pode ser acessado como uma propriedade do objeto Error. Isso pode ser útil para pós-processamento e coleta de feedback automatizado.

var e = new Error('foo');
console.log(e.stack);

Tabulação de valores - console.table() #

Na maioria dos ambientes, console.table() pode ser usado para exibir objetos e arrays em um formato tabular.

Por exemplo:

console.table(['Hello', 'world']);

exibe como:

(índice) valor
0 “Olá”
1 “mundo” 
console.table({foo: 'bar', bar: 'baz'});

exibe como:

(índice) valor

Contando - console.count() #

console.count([obj]) coloca um contador no valor do objeto fornecido como argumento. Cada vez que este método é invocado, o contador é aumentado (com exceção da string vazia ''). Um rótulo junto com um número é exibido no console de depuração de acordo com o seguinte formato:

[label]: X

label representa o valor do objeto passado como argumento e X representa o valor do contador.

O valor de um objeto é sempre considerado, mesmo que as variáveis ​​sejam fornecidas como argumentos:

var o1 = 1, o2 = '2', o3 = "";
console.count(o1);
console.count(o2);
console.count(o3);

console.count(1);
console.count('2');
console.count('');

Exibe:

1: 1
2: 1
: 1
1: 2
2: 2
: 1

Strings com números são convertidas em objetos Number:

console.count(42.3);
console.count(Number('42.3'));
console.count('42.3');

Exibe:

42.3: 1
42.3: 2
42.3: 3

As funções apontam sempre para o objeto global Function:

console.count(console.constructor);
console.count(function(){});
console.count(Object);
var fn1 = function myfn(){};
console.count(fn1);
console.count(Number);

Exibe:

[object Function]: 1
[object Function]: 2
[object Function]: 3
[object Function]: 4
[object Function]: 5

String vazia ou ausência de argumento #

Se nenhum argumento for fornecido ao inserir sequencialmente o método de contagem no console de depuração, uma string vazia será assumida como parâmetro, ou seja:

> console.count();
  : 1
> console.count('');
  : 2
> console.count("");
  : 3

Depuração com asserções - console.assert() #

Grava uma mensagem de erro no console se a asserção for false. Caso contrário, se a afirmação for true, isso não fará nada.

console.assert('one' === 1);

Vários argumentos podem ser fornecidos após a afirmação–podem ser strings ou outros objetos–que só serão impressos se a afirmação for falsa:

console.assert não lança um AssertionError (exceto em Node.js), o que significa que este método é incompatível com a maioria dos frameworks de teste e que a execução do código não será interrompida uma afirmação falhada.

Limpando o console - console.clear() #

Você pode limpar a janela do console usando o método console.clear(). Isso remove todas as mensagens impressas anteriormente no console e pode imprimir uma mensagem como “O console foi limpo” em alguns ambientes.

Exibindo objetos e XML interativamente - console.dir(), console.dirxml() #

console.dir(object) exibe uma lista interativa das propriedades do objeto JavaScript especificado. A saída é apresentada como uma listagem hierárquica com triângulos de divulgação que permitem ver o conteúdo dos objetos filho.

var myObject = {
    "foo":{
        "bar":"data"
    }
};

console.dir(myObject);
