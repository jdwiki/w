Sequências de escape

  • Inserindo caracteres especiais em strings e expressões regulares
  • Tipos de sequência de escape

Nesta página

  • Inserindo caracteres especiais em strings e expressões regulares
  • Tipos de sequência de escape

Nem tudo que começa com uma barra invertida é uma sequência de escape. Muitos caracteres simplesmente não são úteis para sequências de escape e simplesmente farão com que uma barra invertida anterior seja ignorada.

"\H\e\l\l\o" === "Hello" // true

Por outro lado, alguns caracteres como “u” e “x” causarão um erro de sintaxe quando usados ​​incorretamente após uma barra invertida. O seguinte não é um literal de string válido porque contém o prefixo de sequência de escape Unicode \u seguido por um caractere que não é um dígito hexadecimal válido nem uma chave:

"C:\Windows\System32\updatehandlers.dll" // SyntaxError

Uma barra invertida no final de uma linha dentro de uma string não introduz uma sequência de escape, mas indica a continuação da linha, ou seja,

"contin\
uation" === "continuation" // true

Semelhança com outros formatos #

Embora as seqüências de escape em JavaScript tenham semelhança com outras linguagens e formatos, como C++, Java, JSON, etc., muitas vezes haverá diferenças críticas nos detalhes. Em caso de dúvida, certifique-se de testar se seu código se comporta conforme o esperado e considere verificar a especificação da linguagem.

Inserindo caracteres especiais em strings e expressões regulares #

A maioria dos caracteres imprimíveis pode ser incluída em strings ou literais de expressão regular exatamente como são, por exemplo,

var str = "ポケモン"; // a valid string
var regExp = /[Α-Ωα-ω]/; // matches any Greek letter without diacritics

Para adicionar caracteres arbitrários a uma string ou expressão regular, incluindo os não imprimíveis, deve-se usar sequências de escape. As sequências de escape consistem em uma barra invertida ("\") seguida por um ou mais outros caracteres. Para escrever uma sequência de escape para um caractere específico, normalmente (mas nem sempre) é necessário conhecer seu [código de caractere] hexadecimal (https://www.wikiod.com/pt/javascript/cordas#Código do caractere =201608112028595843342).

O JavaScript fornece várias maneiras diferentes de especificar sequências de escape, conforme documentado nos exemplos deste tópico. Por exemplo, as seguintes seqüências de escape denotam o mesmo caractere: o line feed (caractere de nova linha Unix), com código de caractere U+000A.

  • \n
  • \x0a
  • \u000a
  • \u{a} novo no ES6, apenas em strings
  • \012 proibido em literais de string no modo estrito e em strings de modelo
  • \cj apenas em expressões regulares

Tipos de sequência de escape #

Sequências de escape de caractere único #

Algumas sequências de escape consistem em uma barra invertida seguida por um único caractere.

Por exemplo, em alert("Hello\nWorld");, a sequência de escape \n é usada para introduzir uma nova linha no parâmetro string, para que as palavras “Hello” e “World” sejam exibidas em linhas consecutivas .

Sequência de escape Personagem Unicode
\b (somente em strings, não em expressões regulares) retrocesso U+0008
\t guia horizontal U+0009
\n alimentação de linha U+000A
\v guia vertical U+000B
\f alimentação de formulários U+000C
\r retorno de carro U+000D

Além disso, a sequência \0, quando não seguida por um dígito entre 0 e 7, pode ser usada para escapar do caractere nulo (U+0000).

As sequências \\, \' e \" são usadas para escapar do caractere que segue a barra invertida. Embora semelhante a sequências sem escape, onde a barra invertida inicial é simplesmente ignorada (ou seja, \? para ?), eles são tratados explicitamente como seqüências de escape de caractere único dentro de strings conforme a especificação.

Sequências de escape hexadecimais #

Caracteres com códigos entre 0 e 255 podem ser representados com uma seqüência de escape onde \x é seguido pelo código de caractere hexadecimal de 2 dígitos. Por exemplo, o caractere de espaço ininterrupto tem o código 160 ou A0 na base 16 e, portanto, pode ser escrito como \xa0.

var str = "ONE\xa0LINE"; // ONE and LINE with a non-breaking space between them

Para dígitos hexadecimais acima de 9, as letras a a f são usadas, em minúsculas ou maiúsculas sem distinção.

var regExp1 = /[\x00-xff]/; // matches any character between U+0000 and U+00FF
var regExp2 = /[\x00-xFF]/; // same as above

Sequências de escape Unicode de 4 dígitos #

Caracteres com códigos entre 0 e 65535 (216 - 1) podem ser representados com uma sequência de escape em que \u é seguido pelo código de caractere hexadecimal de 4 dígitos.

Por exemplo, o padrão Unicode define o caractere de seta para a direita (“→”) com o número 8594 ou 2192 em formato hexadecimal. Portanto, uma sequência de escape para isso seria \u2192.

Isso produz a string “A → B”:

var str = "A \u2192 B";

Para dígitos hexadecimais acima de 9, as letras a a f são usadas, em minúsculas ou maiúsculas sem distinção. Os códigos hexadecimais com menos de 4 dígitos devem ser preenchidos à esquerda com zeros: \u007A para a letra minúscula “z”.

Sequências de escape Unicode de colchetes #

O ES6 estende o suporte a Unicode para o intervalo de código completo de 0 a 0x10FFFF. Para escapar caracteres com código maior que 216 - 1, uma nova sintaxe para sequências de escape foi introduzida:

\u{???}

Onde o código entre chaves é uma representação hexadecimal do valor do ponto de código, por exemplo

alert("Look! \u{1f440}"); // Look! 👀

No exemplo acima, o código 1f440 é a representação hexadecimal do código do caractere Unicode Eyes.

Observe que o código entre chaves pode conter qualquer número de dígitos hexadecimais, desde que o valor não exceda 0x10FFFF. Para dígitos hexadecimais acima de 9, as letras a a f são usadas, em minúsculas ou maiúsculas sem distinção.

Sequências de escape Unicode com chaves só funcionam dentro de strings, não dentro de expressões regulares!

Sequências de escape octais #

As sequências de escape octais estão obsoletas a partir do ES5, mas ainda são suportadas dentro de expressões regulares e no modo não estrito também dentro de strings não-modelo. Uma sequência de escape octal consiste em um, dois ou três dígitos octais, com valor entre 0 e 3778 = 255.

Por exemplo, a letra maiúscula “E” tem o código de caractere 69, ou 105 na base 8. Portanto, pode ser representada com a sequência de escape \105:

/\105scape/.test("Fun with Escape Sequences"); // true

No modo estrito, sequências de escape octais não são permitidas dentro de strings e produzirão um erro de sintaxe. Vale a pena notar que \0, diferentemente de \00 ou \000, não é considerado uma sequência de escape octal e, portanto, ainda é permitido dentro de strings (mesmo strings de modelo) no modo estrito.

Controlar sequências de escape #

Algumas sequências de escape são reconhecidas apenas dentro de literais de expressão regular (não em strings). Eles podem ser usados ​​para caracteres de escape com códigos entre 1 e 26 (U+0001–U+001A). Eles consistem em uma única letra A–Z (caso não faz diferença) precedida por \c. A posição alfabética da letra após \c determina o código do caractere.

Por exemplo, na expressão regular

`/\cG/`

A letra “G” (a 7ª letra do alfabeto) refere-se ao caractere U+0007 e, portanto,

`/\cG`/.test(String.fromCharCode(7)); // true
