Expressões condicionais, envolvendo palavras-chave como if e else, fornecem aos programas JavaScript a capacidade de executar ações diferentes dependendo de uma condição booleana: true ou false. Esta seção abrange o uso de condicionais JavaScript, lógica booleana e instruções ternárias.

Sintaxe #

  • if (condição) instrução;
  • if (condição) instrução_1, instrução_2, …, instrução_n;
  • if (condição) {   declaração }
  • if (condição) {   instrução_1;   instrução_2;   …   instrução_n; }
  • if (condição) {   declaração } senão {   declaração }
  • if (condição) {   declaração } else if (condição) {   declaração } senão {   declaração }
  • switch (expressão) { caso valor1:   declaração   [interromper;] caso valor2:   declaração   [interromper;] caso valorN:   declaração   [interromper;] predefinição:   declaração   [interromper;] }
  • doença ? value_for_true : value_for_false;

As condições podem interromper o fluxo normal do programa executando o código com base no valor de uma expressão. Em JavaScript, isso significa usar instruções if, else if e else e ​​operadores ternários.

Operadores ternários #

Pode ser usado para encurtar as operações if/else. Isso é útil para retornar um valor rapidamente (ou seja, para atribuí-lo a outra variável).

Por exemplo:

var animal = 'kitty';
var result = (animal === 'kitty') ? 'cute' : 'still nice';

Neste caso, result recebe o valor ‘cute’, porque o valor de animal é ‘kitty’. Se animal tivesse outro valor, o resultado obteria o valor ‘ainda bom’.

Compare isso com o que o código gostaria com as condições if/else.

var animal = 'kitty';
var result = '';
if (animal === 'kitty') {
    result = 'cute';
} else {
    result = 'still nice';
}

As condições if ou else podem ter várias operações. Neste caso, o operador retorna o resultado da última expressão.

var a = 0;
var str = 'not a';
var b = '';
b = a === 0 ? (a = 1, str += ' test') : (a = 2);

Como a era igual a 0, torna-se 1 e str torna-se ’não é um teste’. A operação que envolveu str foi a última, então b recebe o resultado da operação, que é o valor contido em str, ou seja, ’não é um teste’.

Operadores ternários sempre esperam condições else, caso contrário você receberá um erro de sintaxe. Como solução alternativa, você pode retornar um zero algo semelhante na ramificação else - isso não importa se você não estiver usando o valor de retorno, mas apenas encurtando (ou tentando encurtar) a operação.

var a = 1;
a === 1 ? alert('Hey, it is 1!') : 0;

Como você vê, if (a === 1) alert('Ei, é 1!'); faria a mesma coisa. Seria apenas um char a mais, já que não precisa de uma condição else obrigatória. Se uma condição else estivesse envolvida, o método ternário seria muito mais limpo.

a === 1 ? alert('Hey, it is 1!') : alert('Weird, what could it be?');
if (a === 1) alert('Hey, it is 1!') else alert('Weird, what could it be?');

Os ternários podem ser aninhados para encapsular lógica adicional. Por exemplo

foo ? bar ? 1 : 2 : 3

// To be clear, this is evaluated left to right 
// and can be more explicitly expressed as:

foo ? (bar ? 1 : 2) : 3

Isso é o mesmo que o seguinte if/else

if (foo) {
  if (bar) {
    1
  } else {
    2
  }
} else {
  3
}

Estilisticamente, isso deve ser usado apenas com nomes de variáveis ​​curtos, pois ternários de várias linhas podem diminuir drasticamente a legibilidade.

As únicas instruções que não podem ser usadas em ternários são instruções de controle. Por exemplo, você não pode usar return ou break com ternários. A expressão a seguir será inválida.

var animal = 'kitty';
for (var i = 0; i < 5; ++i) {
    (animal === 'kitty') ? break:console.log(i);
}

Para instruções de retorno, o seguinte também seria inválido:

var animal = 'kitty';
(animal === 'kitty') ? return 'meow' : return 'woof';

Para fazer o acima corretamente, você retornaria o ternário da seguinte forma:

var animal = 'kitty';
return (animal === 'kitty') ? 'meow' : 'woof';

Instrução de troca #

As instruções switch comparam o valor de uma expressão com 1 ou mais valores e executam diferentes seções de código com base nessa comparação.

var value = 1;
switch (value) {
  case 1:
    console.log('I will always run');
    break;
  case 2:
    console.log('I will never run');
    break;
}

A instrução break “quebra” da instrução switch e garante que nenhum código dentro da instrução switch seja executado. É assim que as seções são definidas e permite que o usuário faça casos de “caída”.

Aviso: a falta de uma instrução break ou return para cada caso significa que o programa continuará a avaliar o próximo caso, mesmo que os critérios de caso não sejam atendidos!

switch (value) {
  case 1:
    console.log('I will only run if value === 1');
    // Here, the code "falls through" and will run the code under case 2
  case 2:
    console.log('I will run if value === 1 or value === 2');
    break;
  case 3:
    console.log('I will only run if value === 3');
    break;
}

O último caso é o caso default. Este será executado se nenhuma outra correspondência for feita.

var animal = 'Lion';
switch (animal) {
  case 'Dog':
    console.log('I will not run since animal !== "Dog"');
    break;
  case 'Cat':
    console.log('I will not run since animal !== "Cat"');
    break;
  default:
    console.log('I will run since animal does not match any other case');
}

Deve-se notar que uma expressão case pode ser qualquer tipo de expressão. Isso significa que você pode usar comparações, chamadas de função, etc. como valores de maiúsculas e minúsculas.

function john() {
  return 'John';
}

function jacob() {
  return 'Jacob';
}

switch (name) {
  case john(): // Compare name with the return value of john() (name == "John")
    console.log('I will run if name === "John"');
    break;
  case 'Ja' + 'ne': // Concatenate the strings together then compare (name == "Jane")
    console.log('I will run if name === "Jane"');
    break;
  case john() + ' ' + jacob() + ' Jingleheimer Schmidt':
    console.log('His name is equal to name too!');
    break;
}

Critérios Inclusivos Múltiplos para Casos #

Como os casos “passam” sem uma instrução break ou return, você pode usar isso para criar vários critérios inclusivos:

var x = "c"
switch (x) {
   case "a":
   case "b":
   case "c":
      console.log("Either a, b, or c was selected.");
      break;
   case "d":
      console.log("Only d was selected.");
      break;
   default:
      console.log("No case was matched.");
      break;  // precautionary break if case order changes
}

Se / Senão Se / Senão Controle #

Em sua forma mais simples, uma condição if pode ser usada assim:

var i = 0;

if (i < 1) {
    console.log("i is smaller than 1");
}

A condição i < 1 é avaliada, e se for avaliada como true o bloco que segue é executado. Se for avaliado como false, o bloco será ignorado.

Uma condição if pode ser expandida com um bloco else. A condição é verificada uma vez como acima, e se for avaliada como false um bloco secundário será executado (que seria ignorado se a condição fosse true). Um exemplo:

if (i < 1) {
    console.log("i is smaller than 1");
} else {
    console.log("i was not smaller than 1");
}

Supondo que o bloco else contenha nada além de outro bloco if (com opcionalmente um bloco else) como este:

if (i < 1) {
    console.log("i is smaller than 1");
} else {
    if (i < 2) {
        console.log("i is smaller than 2");
    } else {
        console.log("none of the previous conditions was true");
    }
}

Então, há também uma maneira diferente de escrever isso que reduz o aninhamento:

if (i < 1) {
    console.log("i is smaller than 1");
} else if (i < 2) {
    console.log("i is smaller than 2");
} else {
    console.log("none of the previous conditions was true");
}

Algumas notas de rodapé importantes sobre os exemplos acima:

  • Se qualquer condição for avaliada como true, nenhuma outra condição nessa cadeia de blocos será avaliada e todos os blocos correspondentes (incluindo o bloco else) não serão executados.

  • O número de partes else if é praticamente ilimitado. O último exemplo acima contém apenas um, mas você pode ter quantos quiser.

  • A condição dentro de uma instrução if pode ser qualquer coisa que possa ser forçada a um valor booleano, veja o tópico em boolean logic para mais detalhes;

  • A escada if-else-if sai no primeiro sucesso. Ou seja, no exemplo acima, se o valor de i for 0,5 então a primeira ramificação é executada. Se as condições se sobrepuserem, o primeiro critério que ocorre no fluxo de execução é executado. A outra condição, que também pode ser verdadeira, é ignorada.

  • Se você tiver apenas uma instrução, as chaves em torno dessa instrução são tecnicamente opcionais, por exemplo, tudo bem:

    if (i < 1) console.log("i is smaller than 1");

E isso também funcionará:

  if (i < 1) 
      console.log("i is smaller than 1");

Se você deseja executar várias instruções dentro de um bloco if, as chaves em torno delas são obrigatórias. Apenas usar recuo não é suficiente. Por exemplo, o seguinte código:

  if (i < 1) 
      console.log("i is smaller than 1");
      console.log("this will run REGARDLESS of the condition"); // Warning, see text!

é equivalente a:

  if (i < 1) {
      console.log("i is smaller than 1");
  }
  console.log("this will run REGARDLESS of the condition");

Estratégia #

Um padrão de estratégia pode ser usado em Javascript em muitos casos para substituir uma instrução switch. É especialmente útil quando o número de condições é dinâmico ou muito grande. Ele permite que o código para cada condição seja independente e testável separadamente.

O objeto de estratégia é um objeto simples com várias funções, representando cada condição separada. Exemplo:

const AnimalSays = {
    dog () {
        return 'woof';
    },

    cat () {
        return 'meow';
    },

    lion () {
        return 'roar';
    },

    // ... other animals

    default () {
        return 'moo';
    }
};

O objeto acima pode ser usado da seguinte forma:

function makeAnimalSpeak (animal) {
    // Match the animal by type
    const speak = AnimalSays[animal] || AnimalSays.default;
    console.log(animal + ' says ' + speak());
}

Resultados:

makeAnimalSpeak('dog') // => 'dog says woof'
makeAnimalSpeak('cat') // => 'cat says meow'
makeAnimalSpeak('lion') // => 'lion says roar'
makeAnimalSpeak('snake') // => 'snake says moo'

No último caso, nossa função padrão lida com qualquer animal ausente.

Usando || e && curto-circuito #

Os operadores booleanos || e && farão um “curto-circuito” e não avaliarão o segundo parâmetro se o primeiro for verdadeiro ou falso, respectivamente. Isso pode ser usado para escrever condicionais curtas como:

var x = 10

x == 10 && alert("x is 10")
x == 10 || alert("x is not 10")
