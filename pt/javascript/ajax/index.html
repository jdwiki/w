AJAX significa “JavaScript assíncrono e XML”. Embora o nome inclua XML, JSON é usado com mais frequência devido à sua formatação mais simples e redundância menor. AJAX permite que o usuário se comunique com recursos externos sem recarregar a página da web.

AJAX significa Asíncrono JavaScript and XML. No entanto, você pode usar outros tipos de dados e—no caso de [tag:XMLHttpRequest]—mudar para o modo síncrono obsoleto.

O AJAX permite que as páginas da Web enviem solicitações HTTP para o servidor e recebam uma resposta, sem a necessidade de recarregar a página inteira.

Enviando e recebendo dados JSON via POST #

A solicitação de busca promete retornar inicialmente objetos Response. Eles fornecerão informações de cabeçalho de resposta, mas não incluem diretamente o corpo da resposta, que pode nem ter sido carregado ainda. Métodos no objeto Response, como .json(), podem ser usados ​​para aguardar o carregamento do corpo da resposta e depois analisá-lo.

const requestData = {
  method : 'getUsers'
};

const usersPromise = fetch('/api', {
  method : 'POST',
  body : JSON.stringify(requestData)
}).then(response => {
  if (!response.ok) {
    throw new Error("Got non-2XX response from API server.");
  }
  return response.json();
}).then(responseData => {
  return responseData.users;
});

usersPromise.then(users => {
  console.log("Known users: ", users);
}, error => {
  console.error("Failed to fetch users due to error: ", error);
});

Adicione um pré-carregador AJAX #

Aqui está uma maneira de mostrar um pré-carregador de GIF enquanto uma chamada AJAX está sendo executada. Precisamos preparar nossas funções add e remove preloader:

function addPreloader() {
  // if the preloader doesn't already exist, add one to the page
  if(!document.querySelector('#preloader')) {
    var preloaderHTML = '<img id="preloader" src="https://goo.gl/cNhyvX" />';
    document.querySelector('body').innerHTML += preloaderHTML;
  }
}

function removePreloader() {
  // select the preloader element
  var preloader = document.querySelector('#preloader');
  // if it exists, remove it from the page
  if(preloader) {
    preloader.remove();
  }
}

Agora vamos ver onde usar essas funções.

var request = new XMLHttpRequest();

Dentro da função onreadystatechange você deve ter uma instrução if com a condição: request.readyState == 4 && request.status == 200.

Se true: a solicitação foi finalizada e a resposta está pronta, é onde usaremos removePreloader().

Else if false: a requisição ainda está em andamento, neste caso vamos executar a função addPreloader()

xmlhttp.onreadystatechange = function() {

  if(request.readyState == 4 && request.status == 200) {
    // the request has come to an end, remove the preloader
    removePreloader();
  } else {
    // the request isn't finished, add the preloader
    addPreloader()
  }

};

xmlhttp.open('GET', your_file.php, true);
xmlhttp.send();

Exibindo as principais perguntas sobre JavaScript do mês da API do Stack Overflow #

Podemos fazer uma solicitação AJAX para a API do Stack Exchange para recuperar uma lista das principais perguntas sobre JavaScript do mês e apresentá-las como uma lista de links. Se a solicitação falhar ou retornar um erro de API, nosso tratamento de erros promise exibirá o erro.

Ver resultados ao vivo na HyperWeb. #

const url =
    'http://api.stackexchange.com/2.2/questions?site=stackoverflow' +
    '&tagged=javascript&sort=month&filter=unsafe&key=gik4BOCMC7J9doavgYteRw((';

fetch(url).then(response => response.json()).then(data => {
  if (data.error_message) {
    throw new Error(data.error_message);
  }

  const list = document.createElement('ol');
  document.body.appendChild(list);

  for (const {title, link} of data.items) {
    const entry = document.createElement('li');
    const hyperlink = document.createElement('a');
    entry.appendChild(hyperlink);
    list.appendChild(entry);

    hyperlink.textContent = title;
    hyperlink.href = link;
  }
}).then(null, error => {
  const message = document.createElement('pre');
  document.body.appendChild(message);
  message.style.color = 'red';

  message.textContent = String(error);
});

Usando GET com parâmetros #

Essa função executa uma chamada AJAX usando GET, permitindo enviar parâmetros (objeto) para um arquivo (string) e lançar um callback (função) quando a solicitação for encerrada.

function ajax(file, params, callback) {

  var url = file + '?';

  // loop through object and assemble the url
  var notFirst = false;
  for (var key in params) {
    if (params.hasOwnProperty(key)) {
      url += (notFirst ? '&' : '') + key + "=" + params[key];
    }
    notFirst = true;
  }

  // create a AJAX call with url as parameter
  var xmlhttp = new XMLHttpRequest();
  xmlhttp.onreadystatechange = function() {
    if (xmlhttp.readyState == 4 && xmlhttp.status == 200) {
      callback(xmlhttp.responseText);
    }
  };
  xmlhttp.open('GET', url, true);
  xmlhttp.send();
}

Veja como usamos:

ajax('cars.php', {type:"Volvo", model:"300", color:"purple"}, function(response) {
  // add here the code to be executed when data comes back to this page      
  // for example console.log(response) will show the AJAX response in console
});

E o seguinte mostra como recuperar os parâmetros de url em cars.php:

if(isset($_REQUEST['type'], $_REQUEST['model'], $_REQUEST['color'])) {
  // they are set, we can use them !
  $response = 'The color of your car is ' . $_REQUEST['color'] . '. ';
  $response .= 'It is a ' . $_REQUEST['type'] . ' model ' . $_REQUEST['model'] . '!';
  echo $response;
}

Se você tivesse console.log(response) na função de retorno de chamada, o resultado no console teria sido:

A cor do seu carro é roxa. É um Volvo modelo 300!

Verifique se existe um arquivo por meio de uma solicitação HEAD #

Esta função executa uma requisição AJAX usando o método HEAD que nos permite verificar se existe um arquivo no diretório fornecido como argumento. Também nos permite lançar um retorno de chamada para cada caso (sucesso, falha).

function fileExists(dir, successCallback, errorCallback) {
    var xhttp = new XMLHttpRequest;

    /* Check the status code of the request */
    xhttp.onreadystatechange = function() {
        return (xhttp.status !== 404) ? successCallback : errorCallback;
    };

    /* Open and send the request */
    xhttp.open('head', dir, false);
    xhttp.send();
};

A API fetch é uma maneira mais recente baseada em promessa de fazer solicitações HTTP assíncronas.

fetch('/').then(response => response.text()).then(text => {
  console.log("The home page is " + text.length + " characters long.");
});

Ouvindo eventos AJAX em nível global #

// Store a reference to the native method
let open = XMLHttpRequest.prototype.open; 

// Overwrite the native method
XMLHttpRequest.prototype.open = function() {
    // Assign an event listener
    this.addEventListener("load", event => console.log(XHR), false);
    // Call the stored reference to the native method
    open.apply(this, arguments);
};
