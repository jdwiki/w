JSON (JavaScript Object Notation) é um formato leve de intercâmbio de dados. É fácil para humanos ler e escrever e fácil para máquinas analisar e gerar. É importante perceber que, em JavaScript, JSON é uma string e não um objeto.

Uma visão geral básica pode ser encontrada no site json.org que também contém links para implementações do padrão em muitas linguagens de programação diferentes.

Sintaxe #

  • JSON.parse(input[, reviver])
  • JSON.stringify(valor[, substituto[, espaço]])

Parâmetros #

Parâmetro Detalhes
JSON.parse Analisar uma string JSON
input(string) String JSON a ser analisada.
reviver(função) Prescreve uma transformação para a string JSON de entrada.
JSON.stringify Serialize um valor serializável
valor(string) Valor a ser serializado de acordo com a especificação JSON.
replacer(function ou String[] ou Number[]) Inclui seletivamente certas propriedades do objeto value.
space(String ou Number) Se um number for fornecido, o número space de espaços em branco será inserido de legibilidade. Se uma string for fornecida, a string (primeiros 10 caracteres) será usada como espaços em branco.

Os métodos do utilitário JSON foram padronizados pela primeira vez em ECMAScript 5.1 §15.12.

O formato foi formalmente definido em The application/json Media Type for JSON (RFC 4627 July 2006), que foi posteriormente atualizado em The JSON Data Interchange Format (RFC 7158 March 2013, ECMA-404 outubro de 2013 e RFC 7159 de março de 2014).

Para disponibilizar esses métodos em navegadores antigos, como o Internet Explorer 8, use o [json2.js] de Douglas Crockford (https://github.com/douglascrockford/JSON-js).

Analisando com uma função reviver #

Uma função reviver pode ser usada para filtrar ou transformar o valor que está sendo analisado.

var jsonString = '[{"name":"John","score":51},{"name":"Jack","score":17}]';

var data = JSON.parse(jsonString, function reviver(key, value) {
  return key === 'name' ? value.toUpperCase() : value;
});

const jsonString = '[{"name":"John","score":51},{"name":"Jack","score":17}]';

const data = JSON.parse(jsonString, (key, value) =>
  key === 'name' ? value.toUpperCase() : value
);

Isso produz o seguinte resultado:

[
  {
    'name': 'JOHN',
    'score': 51
  },
  {
    'name': 'JACK',
    'score': 17
  }
]

Isso é particularmente útil quando os dados devem ser enviados que precisam ser serializados/codificados ao serem transmitidos com JSON, mas se deseja acessá-los desserializados/decodificados. No exemplo a seguir, uma data foi codificada para sua representação ISO 8601. Usamos a função reviver para analisar isso em um JavaScript Date.

var jsonString = '{"date":"2016-01-04T23:00:00.000Z"}';

var data = JSON.parse(jsonString, function (key, value) {
  return (key === 'date') ? new Date(value) : value;
});

const jsonString = '{"date":"2016-01-04T23:00:00.000Z"}';

const data = JSON.parse(jsonString, (key, value) =>
  key === 'date' ? new Date(value) : value
);

É importante certificar-se de que a função reviver retorne um valor útil ao final de cada iteração. Se a função reviver retornar undefined, nenhum valor ou a execução cair no final da função, a propriedade é excluída do objeto. Caso contrário, a propriedade é redefinida para ser o valor de retorno.

Serializando um valor #

Um valor JavaScript pode ser convertido em uma string JSON usando a função JSON.stringify.

JSON.stringify(value[, replacer[, space]])
  1. value O valor a ser convertido em uma string JSON.
/* Boolean */  JSON.stringify(true)             // 'true'
/* Number  */  JSON.stringify(12)               // '12'
/* String  */  JSON.stringify('foo')            // '"foo"'
/* Object  */  JSON.stringify({})               // '{}'
               JSON.stringify({foo: 'baz'})     // '{"foo": "baz"}'
/* Array   */  JSON.stringify([1, true, 'foo']) // '[1, true, "foo"]'
/* Date    */  JSON.stringify(new Date())       // '"2016-08-06T17:25:23.588Z"'
/* Symbol  */  JSON.stringify({x:Symbol()})     // '{}'
  1. replacer Uma função que altera o comportamento do processo de stringificação ou uma matriz de objetos String e Number que servem como uma lista de permissões para filtrar as propriedades do objeto de valor a ser incluído na string JSON. Se esse valor for nulo ou não for fornecido, todas as propriedades do objeto serão incluídas na string JSON resultante.
// replacer as a function
function replacer (key, value) {
    // Filtering out properties
    if (typeof value === "string") {
        return
    }    
    return value
}

var foo = { foundation: "Mozilla", model: "box", week: 45, transport: "car", month: 7 }
JSON.stringify(foo, replacer)
// -> '{"week": 45, "month": 7}'

// replacer as an array
JSON.stringify(foo, ['foundation', 'week', 'month'])
// -> '{"foundation": "Mozilla", "week": 45, "month": 7}'
// only the `foundation`, `week`, and `month` properties are kept
  1. space Para facilitar a leitura, o número de espaços usados ​​para recuo pode ser especificado como o terceiro parâmetro.
JSON.stringify({x: 1, y: 1}, null, 2)  // 2 space characters will be used for indentation
/* output:
    {
      'x': 1,
      'y': 1
    }
*/

Alternativamente, um valor de string pode ser fornecido para uso para recuo. Por exemplo, passar '\t' fará com que o caractere de tabulação seja usado para recuo.

JSON.stringify({x: 1, y: 1}, null, '\t')
/* output:
    {
        'x': 1,
        'y': 1
    }
*/

Serializando e restaurando instâncias de classe #

Você pode usar um método toJSON personalizado e uma função reviver para transmitir instâncias de sua própria classe em JSON. Se um objeto tiver um método toJSON, seu resultado será serializado em vez do próprio objeto.

function Car(color, speed) {
  this.color = color;
  this.speed = speed;
}

Car.prototype.toJSON = function() {
  return {
    $type: 'com.example.Car',
    color: this.color,
    speed: this.speed
  };
};

Car.fromJSON = function(data) {
  return new Car(data.color, data.speed);
};

class Car {
  constructor(color, speed) {
    this.color = color;
    this.speed = speed;
    this.id_ = Math.random();
  }

  toJSON() {
    return {
      $type: 'com.example.Car',
      color: this.color,
      speed: this.speed
    };
  }

  static fromJSON(data) {
    return new Car(data.color, data.speed);
  }
}

var userJson = JSON.stringify({
  name: "John",
  car: new Car('red', 'fast')
});

Isso produz a string com o seguinte conteúdo:

{"name":"John","car":{"$type":"com.example.Car","color":"red","speed":"fast"}}

var userObject = JSON.parse(userJson, function reviver(key, value) {
  return (value && value.$type === 'com.example.Car') ? Car.fromJSON(value) : value;
});

Isso produz o seguinte objeto:

{
  name: "John",
  car: Car {
    color: "red",
    speed: "fast",
    id_: 0.19349242527065402
  }
}

JSON versus literais JavaScript #

JSON significa “JavaScript Object Notation”, mas não é JavaScript. Pense nisso como apenas um formato de serialização de dados que por acaso pode ser usado diretamente como um literal JavaScript. No entanto, não é aconselhável executar diretamente (ou seja, através de eval()) JSON que é obtido de uma fonte externa. Funcionalmente, JSON não é muito diferente de XML ou YAML – alguma confusão pode ser evitada se JSON for apenas imaginado como algum formato de serialização que se parece muito com JavaScript.

Mesmo que o nome implique apenas objetos, e mesmo que a maioria dos casos de uso por meio de algum tipo de API sempre sejam objetos e matrizes, JSON não é apenas para objetos ou matrizes. Os seguintes tipos primitivos são suportados:

  • String (por exemplo, "Hello World!")
  • Número (por exemplo, 42)
  • Booleano (por exemplo, true)
  • O valor null

undefined não é suportado no sentido de que uma propriedade indefinida será omitida do JSON na serialização. Portanto, não há como desserializar o JSON e acabar com uma propriedade cujo valor é undefined.

A string "42" é um JSON válido. JSON nem sempre precisa ter um envelope externo de "{...}" ou "[...]".

Embora o nome JSON também seja JavaScript válido e algum JavaScript também seja JSON válido, existem algumas diferenças sutis entre as duas linguagens e nenhuma linguagem é um subconjunto da outra.

Tome a seguinte string JSON como exemplo:

{"color": "blue"}

Isso pode ser inserido diretamente no JavaScript. Será sintaticamente válido e produzirá o valor correto:

const skin = {"color": "blue"};

No entanto, sabemos que “color” é um nome de identificador válido e as aspas ao redor do nome da propriedade podem ser omitidas:

const skin = {color: "blue"};

Também sabemos que podemos usar aspas simples em vez de aspas duplas:

const skin = {'color': 'blue'};

Mas, se pegarmos esses dois literais e tratá-los como JSON, nenhum deles será sintaticamente válido JSON:

{color: "blue"}
{'color': 'blue'}

O JSON exige estritamente que todos os nomes de propriedade tenham aspas duplas e que os valores de string também sejam aspas duplas.

É comum que os novatos em JSON tentem usar trechos de código com literais JavaScript como JSON e cocem a cabeça sobre os erros de sintaxe que estão recebendo do analisador JSON.

Mais confusão começa a surgir quando terminologia incorreta é aplicada no código ou na conversa.

Um antipadrão comum é nomear variáveis ​​que contêm valores não JSON como “json”:

fetch(url).then(function (response) {
  const json = JSON.parse(response.data); // Confusion ensues!

  // We're done with the notion of "JSON" at this point,
  // but the concept stuck with the variable name.
});

No exemplo acima, response.data é uma string JSON que é retornada por alguma API. JSON para no domínio de resposta HTTP. A variável com o nome incorreto “json” contém apenas um valor JavaScript (pode ser um objeto, uma matriz ou até mesmo um número simples!)

Uma maneira menos confusa de escrever o acima é:

fetch(url).then(function (response) {
  const value = JSON.parse(response.data);

  // We're done with the notion of "JSON" at this point.
  // You don't talk about JSON after parsing JSON.
});

Os desenvolvedores também tendem a usar muito a frase “objeto JSON”. Isso também leva à confusão. Como mencionado acima, uma string JSON não precisa conter um objeto como valor. “String JSON” é um termo melhor. Assim como “string XML” ou “string YAML”. Você obtém uma string, analisa-a e acaba com um valor.

Serializando com uma função de substituição #

Uma função replacer pode ser usada para filtrar ou transformar valores que estão sendo serializados.

const userRecords = [
  {name: "Joe", points: 14.9, level: 31.5},
  {name: "Jane", points: 35.5, level: 74.4},
  {name: "Jacob", points: 18.5, level: 41.2},
  {name: "Jessie", points: 15.1, level: 28.1},
];

// Remove names and round numbers to integers to anonymize records before sharing
const anonymousReport = JSON.stringify(userRecords, (key, value) =>
  key === 'name'
    ? undefined
    : (typeof value === 'number' ? Math.floor(value) : value)
);

Isso produz a seguinte string:

'[{"points":14,"level":31},{"points":35,"level":74},{"points":18,"level":41},{"points":15,"level":28}]'

Analisando uma string JSON simples #

O método JSON.parse() analisa uma string como JSON e retorna uma primitiva, array ou objeto JavaScript:

const array = JSON.parse('[1, 2, "c", "d", {"e": false}]');
console.log(array); // logs: [1, 2, "c", "d", {e: false}]

Valores de objetos cíclicos #

Nem todos os objetos podem ser convertidos em uma string JSON. Quando um objeto tem auto-referências cíclicas, a conversão falhará.

Este é tipicamente o caso de estruturas de dados hierárquicas em que pai e filho fazem referência um ao outro:

const world = {
  name: 'World',
  regions: []
};

world.regions.push({
  name: 'North America',
  parent: 'America'
});
console.log(JSON.stringify(world));
// {"name":"World","regions":[{"name":"North America","parent":"America"}]}

world.regions.push({
  name: 'Asia',
  parent: world
});

console.log(JSON.stringify(world));
// Uncaught TypeError: Converting circular structure to JSON

Assim que o processo detecta um ciclo, a exceção é gerada. Se não houvesse detecção de ciclo, a string seria infinitamente longa.
