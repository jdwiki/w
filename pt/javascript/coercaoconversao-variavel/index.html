Algumas linguagens exigem que você defina antecipadamente que tipo de variável você está declarando. JavaScript não faz isso; ele tentará descobrir isso por conta própria. Às vezes, isso pode criar um comportamento inesperado.

Se usarmos o seguinte HTML

<span id="freezing-point">0</span>

E recuperar seu conteúdo por meio de JS, ele não o converterá em um número, mesmo que se espere que isso aconteça. Se usarmos o seguinte trecho, pode-se esperar que boilingPoint seja 100. No entanto, JavaScript irá converter moreHeat em uma string e concatenar as duas strings; o resultado será 0100.

var el = document.getElementById('freezing-point');
var freezingPoint = el.textContent || el.innerText;
var moreHeat = 100;
var boilingPoint = freezingPoint + moreHeat;

Podemos corrigir isso convertendo explicitamente freezingPoint para um número.

var el = document.getElementById('freezing-point');
var freezingPoint = Number(el.textContent || el.innerText);
var boilingPoint = freezingPoint + moreHeat;

Na primeira linha, convertemos "0" (a string) em 0 (o número) antes de armazená-lo. Depois de fazer a adição, você obtém o resultado esperado (100).

Dupla Negação (!!x) #

A dupla negação !! não é um operador JavaScript distinto nem uma sintaxe especial, mas apenas uma sequência de duas negações. Ele é usado para converter o valor de qualquer tipo para seu valor booleano true ou false apropriado dependendo se é truthy ou falsy.

!!1            // true
!!0            // false
!!undefined    // false
!!{}           // true
!![]           // true

A primeira negação converte qualquer valor para false se for truthy e para true se for falsy. A segunda negação então opera em um valor booleano normal. Juntos, eles convertem qualquer valor truthy em true e qualquer valor falsy em false.

No entanto, muitos profissionais consideram a prática de usar tal sintaxe inaceitável e recomendam alternativas mais simples de ler, mesmo que sejam mais longas para escrever:

x !== 0        // instead of !!x in case x is a number
x != null      // instead of !!x in case x is an object, a string, or an undefined

O uso de !!x é considerado uma prática ruim devido aos seguintes motivos:

  1. Estilisticamente, pode parecer uma sintaxe especial distinta, enquanto na verdade não está fazendo nada além de duas negações consecutivas com conversão de tipo implícita.
  2. É melhor fornecer informações sobre tipos de valores armazenados em variáveis ​​e propriedades por meio do código. Por exemplo, x !== 0 diz que x é provavelmente um número, enquanto !!x não transmite nenhuma vantagem para os leitores do código.
  3. O uso de Boolean(x) permite funcionalidades semelhantes e é uma conversão de tipo mais explícita.

Conversão implícita #

O JavaScript tentará converter automaticamente as variáveis ​​para tipos mais apropriados durante o uso. Geralmente é aconselhável fazer conversões explicitamente (veja outros exemplos), mas ainda vale a pena saber quais conversões ocorrem implicitamente.

"1" + 5 === "15" // 5 got converted to string.
1 + "5" === "15" // 1 got converted to string.
1 - "5" === -4 // "5" got converted to a number.
alert({}) // alerts "[object Object]", {} got converted to string.
!0 === true // 0 got converted to boolean
if ("hello") {} // runs, "hello" got converted to boolean.
new Array(3) === ",,"; // Return true. The array is converted to string - Array.toString();

Algumas das partes mais complicadas:

!"0" === false // "0" got converted to true, then reversed.
!"false" === false // "false" converted to true, then reversed.

Convertendo para booleano #

Boolean(...) converterá qualquer tipo de dado em true ou false.

Boolean("true") === true
Boolean("false") === true
Boolean(-1) === true
Boolean(1) === true
Boolean(0) === false
Boolean("") === false
Boolean("1") === true
Boolean("0") === true
Boolean({}) === true
Boolean([]) === true

Strings vazias e o número 0 serão convertidos em false, e todos os outros serão convertidos em true.

Uma forma mais curta, mas menos clara:

!!"true" === true
!!"false" === true
!!-1 === true
!!1 === true
!!0 === false
!!"" === false
!!"1" === true
!!"0" === true
!!{} === true
!![] === true

Esta forma mais curta aproveita a conversão de tipo implícita usando o operador lógico NOT duas vezes, conforme descrito em https://www.wikiod.com/pt/javascript/operacoes-de-comparacao#Dupla Negação (!!x)

Aqui está a lista completa de conversões booleanas da especificação ECMAScript

  • se myArg do tipo undefined ou null então Boolean(myArg) === false
  • se myArg do tipo boolean então Boolean(myArg) === myArg
  • se myArg do tipo number então Boolean(myArg) === false se myArg for +0, ‑0 ou NaN; caso contrário verdadeiro
  • se myArg do tipo string então Boolean(myArg) === false se myArg for a String vazia (seu comprimento é zero); caso contrário verdadeiro
  • se myArg do tipo symbol ou object então Boolean(myArg) === true

Os valores que são convertidos para false como booleanos são chamados de falsy (e todos os outros são chamados de truthy). Consulte https://www.wikiod.com/pt/javascript/operacoes-de-comparacao

Convertendo uma string em um número #

Number('0') === 0

Number('0') irá converter a string ('0') em um número (0)

Uma forma mais curta, mas menos clara:

+'0' === 0

O operador unário + não faz nada com números, mas converte qualquer outra coisa em um número. Curiosamente, +(-12) === -12.

parseInt('0', 10) === 0

parseInt('0', 10) irá converter a string ('0') em um número (0), não se esqueça do segundo argumento, que é radix. Se não for fornecido, o parseInt pode converter a string em um número errado.

Convertendo um número em uma string #

String(0) === '0'

String(0) irá converter o número (0) em uma string ('0').

Uma forma mais curta, mas menos clara:

'' + 0 === '0'

Convertendo um número para um booleano #

Boolean(0) === false

Boolean(0) irá converter o número 0 em um booleano false.

Uma forma mais curta, mas menos clara:

!!0 === false

Convertendo uma string para um booleano #

Para converter uma string para uso booleano

Boolean(myString)

ou a forma mais curta, mas menos clara

!!myString

Todas as strings, exceto a string vazia (de comprimento zero), são avaliadas como true como booleanos.

Boolean('') === false   // is true
Boolean("") === false   // is true
Boolean('0') === false  // is false
Boolean('any_nonempty_string') === true // is true

Inteiro para Flutuar #

Flutuar para inteiro #

Para converter um float em um inteiro, o JavaScript fornece vários métodos.

A função floor retorna o primeiro inteiro menor ou igual ao float.

Math.floor(5.7); // 5

A função ceil retorna o primeiro inteiro maior ou igual ao float.

Math.ceil(5.3); // 6

A função round arredonda o float.

Math.round(3.2); // 3
Math.round(3.6); // 4

Truncamento (trunc) remove os decimais do float.

Math.trunc(3.7); // 3

Observe a diferença entre truncamento (trunc) e floor:

Math.floor(-3.1); // -4
Math.trunc(-3.1); // -3

Converte string para float #

parseFloat aceita uma string como argumento que converte em um float/

parseFloat("10.01") // = 10.01

Converte um array em uma string #

Array.join(separator) pode ser usado para gerar um array como uma string, com um separador configurável.

Padrão (separador = “,”):

["a", "b", "c"].join() === "a,b,c"

Com um separador de string:

[1, 2, 3, 4].join(" + ") === "1 + 2 + 3 + 4"

Com um separador em branco:

["B", "o", "b"].join("") === "Bob"

Array para String usando métodos de array #

Esta maneira pode parecer inútil porque você está usando uma função anônima para realizar algo que você pode fazer com join(); Mas se você precisar fazer algo nas strings enquanto estiver convertendo o Array em String, isso pode ser útil.

var  arr = ['a', 'á', 'b', 'c']

function upper_lower (a, b, i) {
  //...do something here
  b = i & 1 ? b.toUpperCase() : b.toLowerCase();
  return a + ',' + b
}
arr = arr.reduce(upper_lower); // "a,Á,b,C"
