Sintaxe #

  • for (inicialização; condição; expressão_final) { }
  • for (chave em objeto) { }
  • for (variável de iterável) { }
  • while (condição) { }
  • do { } while (condição)
  • for each (variable in object) { } // ECMAScript para XML

Loops em JavaScript normalmente ajudam a resolver problemas que envolvem a repetição de código específico x quantidade de vezes. Digamos que você precise registrar uma mensagem 5 vezes. Você poderia fazer isso:

console.log("a message");
console.log("a message");
console.log("a message");
console.log("a message");
console.log("a message");

Mas isso é apenas demorado e meio ridículo. Além disso, e se você precisasse registrar mais de 300 mensagens? Você deve substituir o código por um loop tradicional “for”:

for(var i = 0; i < 5; i++){
    console.log("a message");
}

Loops “for” padrão #

Uso padrão #

for (var i = 0; i < 100; i++) {
    console.log(i);
}

Saída esperada:

0 1 …
99

Declarações múltiplas #

Comumente usado para armazenar em cache o comprimento de uma matriz.

var array = ['a', 'b', 'c'];
for (var i = 0; i < array.length; i++) {
    console.log(array[i]);
}

Saída esperada:

‘um’ ‘b’ ‘c’

Alterando o incremento #

for (var i = 0; i < 100; i += 2 /* Can also be: i = i + 2 */) {
    console.log(i);
}

Saída esperada:

0 2 4 …
98

Loop decrementado #

for (var i = 100; i >=0; i--) {
    console.log(i);
}

Saída esperada:

100 99 98 …
0

“para … de” loop #

const iterable = [0, 1, 2];
for (let i of iterable) {
    console.log(i);
}

Saída esperada:

0 1 2

As vantagens do loop for…of são:

  • Esta é a sintaxe mais concisa e direta até agora para percorrer os elementos da matriz
  • Evita todas as armadilhas do for…in
  • Ao contrário de forEach(), funciona com break, continue e return

Suporte de for…of em outras coleções #

Cordas #

for…of tratará uma string como uma sequência de caracteres Unicode:

const string = "abc";
for (let chr of string) {
  console.log(chr);
}

Saída esperada:

um b c

Conjuntos #

for…of funciona em Set objects.

Observação:

  • Um objeto Set eliminará duplicatas.

  • Por favor, verifique esta referência para suporte ao navegador Set().

    const names = [‘bob’, ‘alejandro’, ‘zandra’, ‘anna’, ‘bob’];

    const uniqueNames = new Set(names);

    for (let name of uniqueNames) { console.log(name); }

Saída esperada:

bob alejandra zandra ana

Mapas #

Você também pode usar for…of loops para iterar sobre Maps. Isso funciona de maneira semelhante a matrizes e conjuntos, exceto que a variável de iteração armazena uma chave e um valor.

const map = new Map()
  .set('abc', 1)
  .set('def', 2)

for (const iteration of map) {
  console.log(iteration) //will log ['abc', 1] and then ['def', 2]
}

Você pode usar [atribuição de desestruturação](https://www.wikiod.com/pt/javascript/atribuicao-de-desestruturacao#Desestruturando Arrays) para capturar a chave e o valor separadamente:

const map = new Map()
  .set('abc', 1)
  .set('def', 2)

for (const [key, value] of map) {
  console.log(key + ' is mapped to ' + value)
}
/*Logs:
  abc is mapped to 1
  def is mapped to 2
*/

Objetos #

for…of loops não funcionam diretamente em objetos simples; mas é possível iterar sobre as propriedades de um objeto alternando para um loop for…in ou usando [Object.keys()](https://www.wikiod.com/pt/javascript/objetos 736/object-keys#t=201608201852433521361):

const someObject = { name: 'Mike' };

for (let key of Object.keys(someObject)) {
  console.log(key + ": " + someObject[key]);
}

Saída esperada:

nome: Mike

“para … em” loop #

Aviso
for…in destina-se a iterar sobre chaves de objeto, não índices de matriz. Usá-lo para percorrer uma matriz geralmente é desencorajado. Ele também inclui propriedades do protótipo, então pode ser necessário verificar se a chave está dentro do objeto usando hasOwnProperty. Se algum atributo no objeto for definido pelo método defineProperty/defineProperties e setar o parâmetro enumerable: false, esses atributos ficarão inacessíveis.

var object = {"a":"foo", "b":"bar", "c":"baz"};
// `a` is inaccessible
Object.defineProperty(object , 'a', {
        enumerable: false,
});
for (var key in object) {
    if (object.hasOwnProperty(key)) {
      console.log('object.' + key + ', ' + object[key]);
    }
}

Saída esperada:

object.b, barra object.c, baz

“enquanto” Loops #

Loop Enquanto Padrão #

Um loop while padrão será executado até que a condição dada seja falsa:

var i = 0;
while (i < 100) {
    console.log(i);
    i++;
}

Saída esperada:

0 1 …
99

Loop decrementado #

var i = 100;
while (i > 0) {
    console.log(i);
    i--; /* equivalent to i=i-1 */
}

Saída esperada:

100 99 98 …
1

Faça… while Loop #

Um loop do…while sempre será executado pelo menos uma vez, independentemente de a condição ser verdadeira ou falsa:

var i = 101;
do {
    console.log(i);
} while (i < 100);

Saída esperada:

101

“continuar” um loop #

Continuando um loop “for” #

Quando você coloca a palavra-chave continue em um loop for, a execução salta para a expressão de atualização (i++ no exemplo):

for (var i = 0; i < 3; i++) {
    if (i === 1) {
        continue;
    }
    console.log(i);
}

Saída esperada:

0 2

Continuando um loop while #

Quando você continua em um loop while, a execução pula para a condição (i < 3 no exemplo):

var i = 0;
while (i < 3) {
    if (i === 1) {
        i = 2;
        continue;
    }
    console.log(i);
    i++;
}

Saída esperada:

0 2

“faça … enquanto” loop #

var availableName;
do {
    availableName = getRandomName();
} while (isNameUsed(name));

É garantido que um loop do while seja executado pelo menos uma vez, pois sua condição só é verificada no final de uma iteração. Um loop tradicional while pode ser executado zero ou mais vezes, pois sua condição é verificada no início de uma iteração.

Interromper loops aninhados específicos #

Podemos nomear nossos loops e quebrar o específico quando necessário.

outerloop:
for (var i = 0;i<3;i++){
    innerloup:
    for (var j = 0;j <3; j++){
        console.log(i);
        console.log(j);
        if (j == 1){
            break outerloop;    
        }
    }
}

Resultado:

0
0
0
1

Quebra e continua rótulos #

As instruções break e continue podem ser seguidas por um rótulo opcional que funciona como algum tipo de instrução goto, retoma a execução da posição referenciada pelo rótulo

for(var i = 0; i < 5; i++){
  nextLoop2Iteration:
  for(var j = 0; j < 5; j++){
    if(i == j) break nextLoop2Iteration;
    console.log(i, j);
  }
}

i=0 j=0 pula o resto dos valores de j 1 0 i=1 j=1 pula o resto dos valores de j 2 0 2 1 i=2 j=2 pula o resto dos valores de j 3 0 3 1 3 2 i=3 j=3 pula o resto dos valores de j 4 0 4 1 4 2 4 3 i=4 j=4 não registra e os loops são feitos

“Quebrar” de um loop #

Saindo de um loop while #

var i = 0;
while(true) {
    i++;
    if(i === 42) {
        break;
    }
}
console.log(i);

Saída esperada:

42

Saindo de um loop for #

var i;
for(i = 0; i < 100; i++) {
    if(i === 42) {
        break;
    }
}
console.log(i);

Saída esperada:

42
