Comentários #

# This comment occupies a whole line
- some item # This comment succeeds content of a line
- http://example.com/#nocomment
- "This # does not introduce a comment."
- |
    This is a block scalar.
    A # inside it does not introduce a comment.
   # unless it is less indented than the first line (this is one)

Observe que para um # introduzir um comentário, ele deve

  • ocorrem no início de uma linha, ou
  • ser precedido por espaço em branco.

# deve sempre ser seguido por espaço em branco. # dentro de escalares citados nunca iniciam comentários. # pode introduzir comentários no final de escalares de bloco, mas, portanto, deve ser menos recuado do que o recuo base do escalar de bloco (que geralmente é determinado pelo recuo de sua primeira linha não vazia).

Sintaxe básica do Yaml #

YAML é um formato baseado em texto que permite armazenar dados estruturados em uma hierarquia. O YAML foi projetado para ser legível por humanos e por máquina com um mínimo de sobrecarga. A especificação YAML pode ser encontrada em yaml.org. Há também um cartão de referência

Os comentários começam com # e vão até a nova linha, os comentários devem ser separados de outros tokens por espaços em branco. O espaço em branco não é livre, o recuo deve ser espaços, não tabulações. O YAML considerará que as linhas prefixadas com mais espaços do que a chave pai estão contidas nele. Além disso, todas as linhas devem ser prefixadas com a mesma quantidade de espaços para pertencer ao mesmo mapa.

YAML possui sequências e mapeamentos como tipos de coleção, ambos podem ser representados em fluxo e estilo de bloco.

Uma sequência de strings escalares em YAML se parece com:

[ one, two, three ]   # flow style

# or block style

- one
- two
- three

Um mapeamento consiste em pares de chave/valor:

index: 4  # block style
name: nali

# or 

{ index: 4, name: nali }   # flow style

# or nested (equivalent of { level: { one: { two: fun } } }):

level:

  one:

    two: fun

Dividindo strings de texto em várias linhas #

- Without quotes:
   You can just
   split a long piece of text like this.
- With quotes:
    "[But be careful:
     if you \"need\" punctuation, put double quotes around it. You can ev\
     en split without spaces by using backslashes."
- Or single quotes:
    'This works
     but isn''t as flexible'
- If you want to keep those new line characters: | 
    Then do
    it this way with 
    a pipe (|) character. (This string has three \n characters)
- Or you can have just the one final new line: >
    This string has
    just one \n character, at the very end.
- Block indicators:
    Look up >-, >+, |- and |+ for fine tuning.

Caracteres de escape #

YAML suporta três estilos de notação de escape:

  1. Fugas de Entidade

uma. espaço: “&#x20;”

b. dois pontos: “&#58;”

c. e comercial: “&amp;”

  1. Escapes Unicode

uma. espaço: “\u0020”

b. aspas simples: “\u0027”

c. aspas duplas: “\u0022”

  1. Escapes Citados

uma. aspas duplas em aspas simples: ‘Eu sempre minto’ é uma afirmação verdadeira?’

b. aspas duplas aninhadas: " Ela disse, "eu desisto" "

c. aspas simples aninhadas: ’ Ele ficou sem palavras: ’’ '

Mapeamentos de estilo de bloco #

Com chaves implícitas:

key: value
another key:
  - some
  - more
  - values
[1, 2, 3]: last value, which has a flow style key

Com chaves implícitas e explícitas:

? key
: value
another key:
  - some
  - more
  - values
? [1, 2, 3]
: last value, which has a flow style key

key, another key e [1, 2, 3] são chaves do mesmo mapeamento, embora usem estilos de chave diferentes.

Mapeamentos aninhados:

first level:
  second level:
    ? third level
    :
      forth level: value of implicit key
    ? third level, second key
    : value of explicit key
  ?
    mapping as: key of
    another: mapping
  : scalar value of mapping key
first level, second key:
  last value
