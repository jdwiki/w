Introdução ao Trie #

Você já se perguntou como funcionam os mecanismos de busca? Como o Google alinha milhões de resultados à sua frente em apenas alguns milissegundos? Como um enorme banco de dados situado a milhares de quilômetros de você descobre as informações que você está procurando e as envia de volta para você? A razão por trás disso não é possível apenas usando internet mais rápida e supercomputadores. Alguns algoritmos de pesquisa e estruturas de dados fascinantes funcionam por trás disso. Um deles é Trie.

Trie, também chamada de árvore digital e às vezes árvore de raiz ou árvore de prefixos (pois podem ser pesquisadas por prefixos), é um tipo de árvore de pesquisa—uma estrutura de dados de árvore ordenada usada para armazenar um conjunto dinâmico ou array associativo onde as chaves geralmente são strings. É uma daquelas estruturas de dados que podem ser facilmente implementadas. Digamos que você tenha um enorme banco de dados de milhões de palavras. Você pode usar trie para armazenar essas informações e a complexidade de pesquisar essas palavras depende apenas do comprimento da palavra que estamos procurando. Isso significa que não depende do tamanho do nosso banco de dados. Não é incrível?

Vamos supor que temos um dicionário com estas palavras:

algo
algea
also
tom
to

Queremos armazenar este dicionário na memória de forma que possamos encontrar facilmente a palavra que estamos procurando. Um dos métodos envolveria classificar as palavras lexicograficamente - como os dicionários da vida real armazenam palavras. Então podemos encontrar a palavra fazendo uma pesquisa binária. Outra maneira é usar Prefix Tree ou Trie, em resumo. A palavra ‘Trie’ vem da palavra Retrieval. Aqui, prefixo denota o prefixo da string que pode ser definida assim: Todas as substrings que podem ser criadas a partir do início de uma string são chamadas de prefixo. Por exemplo: ‘c’, ‘ca’, ‘cat’ todos são o prefixo da string ‘cat’.

Agora vamos voltar para Testar. Como o nome sugere, vamos criar uma árvore. A princípio, temos uma árvore vazia com apenas a raiz:

Vamos inserir a palavra ‘algo’. Vamos criar um novo nó e nomear a aresta entre esses dois nós ‘a’. A partir do novo nó, criaremos outra aresta chamada ’l’ e a conectaremos a outro nó. Dessa forma, criaremos duas novas arestas para ‘g’ e ‘o’. Observe que não estamos armazenando nenhuma informação nos nós. Por enquanto, consideraremos apenas a criação de novas arestas a partir desses nós. De agora em diante, vamos chamar uma aresta chamada ‘x’ - edge-x

Agora queremos adicionar a palavra ‘algea’. Precisamos de um edge-a da raiz, que já temos. Portanto, não precisamos adicionar nova aresta. Da mesma forma, temos uma aresta de ‘a’ a ’l’ e ’l’ a ‘g’. Isso significa que ‘alg’ já está na trie. Adicionaremos apenas edge-e e edge-a a ele.

Adicionaremos a palavra ’também’. Temos o prefixo ‘al’ da raiz. Só adicionaremos ‘so’ com ele.

Vamos adicionar ’tom’. Desta vez, criamos uma nova aresta a partir da raiz, pois não temos nenhum prefixo de tom criado antes.

Agora, como devemos adicionar ‘a’? ’to’ é completamente um prefixo de ’tom’, então não precisamos adicionar nenhuma aresta. O que podemos fazer é colocar marcas finais em alguns nós. Colocaremos marcas finais nos nós onde pelo menos uma palavra estiver completa. O círculo verde indica a marca final. A tentativa ficará assim:

Você pode entender facilmente por que adicionamos as marcas finais. Podemos determinar as palavras armazenadas em trie. Os caracteres estarão nas bordas e os nós conterão as marcas finais.

Agora você pode perguntar, qual é o propósito de armazenar palavras como esta? Digamos que você seja solicitado a encontrar a palavra ‘alice’ no dicionário. Você vai atravessar o trie. Você verificará se há um edge-a da raiz. Em seguida, verifique de ‘a’, se houver um aresta-l. Depois disso, você não encontrará nenhum edge-i, então você pode chegar à conclusão de que a palavra alice não existe no dicionário.

Se for solicitado que você encontre a palavra ‘alg’ no dicionário, você percorrerá raiz->a, a->l, l->g , mas você não encontrará um nó verde no final. Então a palavra não existe no dicionário. Se você pesquisar por ’tom’, terminará em um nó verde, o que significa que a palavra existe no dicionário.

Complexidade:

A quantidade máxima de etapas necessárias para pesquisar uma palavra em uma trie é o comprimento da palavra que estamos procurando. A complexidade é O(comprimento). A complexidade para inserção é a mesma. A quantidade de memória necessária para implementar trie depende da implementação. Veremos uma implementação em outro exemplo em que podemos armazenar 106 caracteres (não palavras, letras) em um trie.

Uso de Trie:

  • Para inserir, excluir e pesquisar uma palavra no dicionário.
  • Para descobrir se uma string é um prefixo de outra string.
  • Para descobrir quantas strings tem um prefixo comum.
  • Sugestão de nomes de contato em nossos telefones dependendo do prefixo que inserimos.
  • Descobrir ‘Mongest Common Substring’ de duas strings.
  • Descobrir o comprimento do ‘Prefixo Comum’ para duas palavras usando ‘Mais Longo Ancestral Comum’

Implementação de Trie #

Antes de ler este exemplo, é altamente recomendável que você leia [Introduction to Trie](https://www.wikiod.com/pt/data-structures/trie-arvore-prefixoarvore-radix#Introdução ao teste -tentar) primeiro.

Uma das maneiras mais fáceis de implementar Trie é usar a lista vinculada.

Nó:

Os nós serão compostos por:

  1. Variável para Marca Final.
  2. Matriz de ponteiros para o próximo nó.

A variável End-Mark simplesmente denota se é um ponto final ou não. A matriz de ponteiros denotará todas as arestas possíveis que podem ser criadas. Para alfabetos ingleses, o tamanho do array será 26, pois no máximo 26 arestas podem ser criadas a partir de um único nó. No início, cada valor do array de ponteiros será NULL e a variável end-mark será false.

Node:
Boolean Endmark
Node *next[26]
Node()
    endmark = false
    for i from 0 to 25
        next[i] := NULL
    endfor
endNode

Cada elemento na matriz next[] aponta para outro nó. next[0] aponta para o compartilhamento de nó edge-a, next1 aponta para o compartilhamento de node edge-b e assim por diante. Definimos o construtor de Node para inicializar Endmark como false e colocar NULL em todos os valores do array de ponteiros next[].

Para criar Trie, primeiro precisamos instanciar root. Em seguida, a partir da raiz, criaremos outros nós para armazenar informações.

Inserção:

Procedure Insert(S, root):    // s is the string that needs to be inserted in our dictionary
curr := root
for i from 1 to S.length
    id := S[i] - 'a'
    if curr -> next[id] == NULL
        curr -> next[id] = Node()
    curr := curr -> next[id]
endfor
curr -> endmark = true

Aqui estamos trabalhando com a-z. Portanto, para converter seus valores ASCII para 0-25, subtraímos deles o valor ASCII de ‘a’. Vamos verificar se o nó atual (curr) tem uma vantagem para o caractere em mãos. Se isso acontecer, passamos para o próximo nó usando essa aresta, caso contrário, criamos um novo nó. No final, alteramos o endmark para true.

Procurando:

Procedure Search(S, root)      // S is the string we are searching
curr := root
for i from 1 to S.length
    id := S[i] - 'a'
    if curr -> next[id] == NULL
        Return false
    curr := curr -> id
Return curr -> endmark

O processo é semelhante à inserção. No final, retornamos o endmark. Portanto, se o endmark for verdadeiro, isso significa que a palavra existe no dicionário. Se for falso, então a palavra não existe no dicionário.

Esta foi a nossa principal implementação. Agora podemos inserir qualquer palavra em trie e procurá-la.

Eliminação:

Às vezes, precisaremos apagar as palavras que não serão mais usadas para economizar memória. Para isso, precisamos deletar as palavras não utilizadas:

Procedure Delete(curr):       //curr represents the current node to be deleted
for i from 0 to 25
    if curr -> next[i] is not equal to NULL
        delete(curr -> next[i])
    delete(curr)               //free the memory the pointer is pointing to

Esta função vai para todos os nós filhos de curr, os exclui e então curr é excluído para liberar a memória. Se chamarmos delete(root), ele excluirá todo o Trie.

Trie também pode ser implementado usando Arrays.
