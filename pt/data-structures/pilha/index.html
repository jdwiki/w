Introdução à pilha #

A pilha é uma estrutura de dados LIFO (last-in, first-out), ou seja, o elemento mais recente (ou “last in”) adicionado à pilha será o primeiro elemento removido (“first out”).

Consideremos o exemplo dos livros em uma caixa. Apenas um livro pode ser adicionado ou removido da caixa por vez, e só pode ser adicionado e removido pela parte superior.

Agora, a caixa com os dois primeiros livros fica assim:

 |--------|
| livro 2 | ◀──── topo da caixa
  |--------|
| livro 1 | ◀──── parte inferior da caixa
|--------|

Se adicionarmos o livro 3, ele estará no topo. O restante dos livros, que foram adicionados antes do livro 3, permanecerão abaixo dele:

 |--------|
| livro 3 | ◀──── topo da caixa
  |--------|
| livro 2 |
  |--------|
| livro 1 | ◀──── parte inferior da caixa
|--------|

A caixa é como uma pilha e os livros são elementos de dados. Adicionar um livro à caixa é a operação push, enquanto remover/obter um livro da parte superior da caixa é a operação pop. Se executarmos a operação pop, obteremos o livro 3, e a caixa voltará ao que era antes de empurrarmos o livro 3. Isso significa que o último (ou mais recente) elemento que colocamos foi o primeiro elemento que saiu (LIFO). Para obter o livro 1, temos que realizar mais dois pops: um para remover o livro 2 e o segundo para obter o livro 1.

Implementação da pilha usando um array. Para isso, precisamos de um índice apontando para a localização superior (tos). Toda vez que um elemento é inserido na pilha, o tos é incrementado em um e sempre que uma operação pop é executada, o índice que aponta para o topo da pilha (tos) é decrementado em um.

EMPURRE:

Antes da operação PUSH <pré> tos |
i1 i2 i3 i4
—– —– —– —– —–
void push(dataElment item){
stack[top]=item; //item = i4
top++; 
}

Após a operação PUSH A pilha tem um ponteiro para o topo da pilha. Sempre que a operação push é chamada, ela coloca o valor no topo e atualiza o valor. <pré> tos – Top of the stack tos |
i1 i2 i3
—– —– —– —–
POP:
A operação pop remove o conteúdo do topo da pilha e atualiza o valor de tos decrementando em 1

Antes da operação pop: <pré> tos |
i1 i2 i3 i4
—– —– —– —– —– 
dataElment pop(){
    dataElment value = stack[tos--];
    return value;
}

Após a operação pop: <pré> tos |
i1 i2 i3 i4
—– —– —– —–
When a push operation is performed it overwrites i4.

Usando pilhas para encontrar palíndromos #

Um palíndromo é uma palavra que pode ser lida nos dois sentidos, como ‘caiaque’.

Este exemplo mostrará como descobrir se uma determinada palavra é um palíndromo usando Python3.

Primeiro, precisamos transformar uma string em uma pilha (usaremos arrays como pilhas).

str = "string"
stk = [c for c in str] #this makes an array: ["s", "t", "r", "i", "n", "g"]
stk.append("s") #adds a letter to the array
stk.pop() #pops the last element

Agora, temos que inverter a palavra.

def invert(str):
    stk = [c for c in str]
    stk2 = []
    while len(stk) > 0:
        stk2.append(stk.pop())
    #now, let's turn stk2 into a string
    str2 = ""
    for c in stk2:
        str2 += c
    return str2

Agora podemos comparar a palavra e a forma invertida.

def palindrome(str):
    return str == invert(str)

Implementação de pilha usando array e lista vinculada #

Implementação de matriz

#include<stdio.h>

#define MAX 100000                    //Global variables
int top = -1;
int a[MAX];

void push(int x){
    
    if(top == MAX-1){                         // Check whether stack is full
        printf("Stack Overflow\n");
        return;
    }
    
    a[++top] = x;                            // Otherwise increment top and insert x
}

void pop(){
    if(top == -1){                                     // if top = -1, empty stack
        printf("Empty stack\n");
        return;
    }
    top--;        
}

void print(){                         //printing stack values
    
    for(int i=0;i <= top;i++){
        printf("%d ",a[i]);        
    }        
    printf("\n");        
}

void topValue(){                        // Method can print the top value of a stack
    printf("%d",a[top]);
}
int main(){
        
    push(5);
    push(20);
    push(15);
    print();
    pop();
    print();
    push(35);
    print();
    topValue();    
    
}

Implementação de lista vinculada

#include<stdio.h>
#include<malloc.h>

 struct node{
    int data;
    node* link;
};
node* top = NULL;

void push(int data){
    
    node* temp = (struct node*)malloc(sizeof(struct node*));
    temp->data = data;
    temp->link = top;
    top = temp;
}

void pop(){
    
    if(top == NULL) return;
    node* temp = top;
    top = top->link;
    free(temp);
}
void print(){
    
    node* t = top;
    while(t != NULL){
        printf("%d ",t->data);
        t=t->link;
    }
    printf("\n");
}

void topValue(){
    printf("%d ",top->data);
}
int main(){
        
    push(5);
    push(20);
    push(15);
    print();
    pop();
    print();
    push(35);
    print();            
    topValue();    
}

Verificando parênteses balanceados #

Um colchete é considerado qualquer um dos seguintes caracteres: (, ), {, }, [ ou ].

Dois colchetes são considerados um par combinado se um colchete de abertura (ou seja, (, [ ou {) ocorrer à esquerda de um colchete de fechamento (ou seja, ), ] ou }) do mesmo tipo. Existem três tipos de pares de colchetes correspondentes: [], {} e ().

Um par de colchetes correspondente não é balanceado se o conjunto de colchetes que ele inclui não corresponder. Por exemplo, {[(])} não é balanceado porque o conteúdo entre { e } não é balanceado. O par de colchetes inclui um único colchete de abertura não balanceado, (, e o par de parênteses inclui um único colchete de fechamento não balanceado, ].

Por essa lógica, dizemos que uma sequência de colchetes é considerada equilibrada se as seguintes condições forem atendidas:

Não contém colchetes sem correspondência.

O subconjunto de colchetes dentro dos limites de um par correspondente de colchetes também é um par de colchetes correspondente.

Algoritmo:

  1. Declare uma pilha (digamos stack).
  2. Agora percorra a entrada da string.
  • a) Se o caractere atual for um colchete inicial (ou seja, ( ou { ou [ ) então empurre-o para empilhar.
  • b) Se o caractere atual for um colchete de fechamento (ou seja, ) ou } ou ]) em seguida, pop da pilha. Se o caractere exibido for o correspondente parênteses iniciais, então os parênteses são ’não balanceados'.
  • c) Se o caractere atual for um colchete de fechamento (ou seja, ) ou } ou ]) e a pilha estiver vazia, os parênteses serão não balanceados.
  1. Após a travessia completa, se houver algum colchete inicial deixado na pilha, então a string não está balanceada, senão temos uma string balanceada.
