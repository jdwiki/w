Lista vinculada XOR #

Uma Lista vinculada XOR também é chamada de Lista vinculada com eficiência de memória. É outra forma de uma lista duplamente ligada. Isso depende muito da porta lógica XOR e de suas propriedades.

Por que isso é chamado de lista vinculada com eficiência de memória?

Isso é chamado assim porque usa menos memória do que uma lista duplamente vinculada tradicional.

*

Isso é diferente de uma lista duplamente vinculada?

**É sim.

Uma Lista duplamente vinculada armazena dois ponteiros, que apontam para o nó seguinte e o anterior. Basicamente, se você quiser voltar, vá para o endereço apontado pelo ponteiro back. Se você quiser avançar, vá para o endereço apontado pelo ponteiro next. É como:

Uma Lista vinculada com eficiência de memória, ou seja, a Lista vinculada XOR tem apenas um ponteiro em vez de dois. Isso armazena o endereço anterior (addr (anterior)) XOR (^) o próximo endereço (addr (próximo)). Quando você deseja mover para o próximo nó, você faz certos cálculos e encontra o endereço do próximo nó. Isso é o mesmo para ir para o nó anterior. É como:

*

Como funciona?

Para entender como funciona a lista encadeada, você precisa conhecer as propriedades de XOR (^):

|-------------|------------|------------|
|    Name     |   Formula  |    Result  |
|-------------|------------|------------|
| Commutative |    A ^ B   |    B ^ A   |
|-------------|------------|------------|
| Associative | A ^ (B ^ C)| (A ^ B) ^ C|
|-------------|------------|------------|
| None (1)    |    A ^ 0   |     A      |
|-------------|------------|------------|
| None (2)    |    A ^ A   |     0      |
|-------------|------------|------------|
| None (3)    | (A ^ B) ^ A|     B      |
|-------------|------------|------------|

Agora vamos deixar isso de lado e ver o que cada nó armazena.

O primeiro nó, ou head, armazena 0 ^ addr (next), pois não há nó ou endereço anterior. Parece isto.

Em seguida, o segundo nó armazena addr (anterior) ^ addr (próximo). Parece isto.

A cauda da lista não possui nenhum próximo nó, então ela armazena addr (prev) ^ 0. Parece isto.

Movendo do Head para o próximo nó

Digamos que você esteja agora no primeiro nó, ou no nó A. Agora você quer se mover no nó B. Esta é a fórmula para fazer isso:

Address of Next Node = Address of Previous Node ^ pointer in the current Node

Então isso seria:

addr (next) = addr (prev) ^ (0 ^ addr (next))

Como esta é a cabeça, o endereço anterior seria simplesmente 0, então:

addr (next) = 0 ^ (0 ^ addr (next))

Podemos remover os parênteses:

addr (next) = 0 ^ 0 addr (next)

Usando a propriedade none (2), podemos dizer que 0 ^ 0 sempre será 0:

addr (next) = 0 ^ addr (next)

Usando a propriedade none (1), podemos simplificá-la para:

addr (next) = addr (next)

Você tem o endereço do próximo nó!

Mudando de um nó para o próximo nó

Agora digamos que estamos em um nó do meio, que tem um nó anterior e um próximo.

Vamos aplicar a fórmula:

Address of Next Node = Address of Previous Node ^ pointer in the current Node

Agora substitua os valores:

addr (next) = addr (prev) ^ (addr (prev) ^ addr (next))

Remover parênteses:

addr (next) = addr (prev) ^ addr (prev) ^ addr (next)

Usando a propriedade none (2), podemos simplificar:

addr (next) = 0 ^ addr (next)

Usando a propriedade none (1), podemos simplificar:

addr (next) = addr (next)

E você entendeu!

Mudando de um node para o node em que você estava anteriormente

Se você não está entendendo o título, basicamente significa que se você estava no nó X e agora mudou para o nó Y, você quer voltar para o nó visitado anteriormente, ou basicamente o nó X.

Esta não é uma tarefa complicada. Lembre-se que eu mencionei acima, que você armazena o endereço em que estava em uma variável temporária. Portanto, o endereço do nó que você deseja visitar está em uma variável:

addr (prev) = temp_addr

Mudando de um nó para o nó anterior

Isso não é o mesmo que mencionado acima. Quero dizer que, você estava no nó Z, agora está no nó Y e quer ir para o nó X.

Isso é quase o mesmo que mover de um nó para o próximo nó. Só que isso é vice-versa. Ao escrever um programa, você usará as mesmas etapas que mencionei na mudança de um nó para o próximo nó, apenas que você está encontrando o elemento anterior na lista do que encontrando o próximo elemento.

Exemplo de código em C

/* C/C++ Implementation of Memory efficient Doubly Linked List */
#include <stdio.h>
#include <stdlib.h>
 
// Node structure of a memory efficient doubly linked list
struct node
{
    int data;
    struct node* npx;  /* XOR of next and previous node */
};
 
/* returns XORed value of the node addresses */
struct node* XOR (struct node *a, struct node *b)
{
    return (struct node*) ((unsigned int) (a) ^ (unsigned int) (b));
}
 
/* Insert a node at the begining of the XORed linked list and makes the
   newly inserted node as head */
void insert(struct node **head_ref, int data)
{
    // Allocate memory for new node
    struct node *new_node  = (struct node *) malloc (sizeof (struct node) );
    new_node->data = data;
 
    /* Since new node is being inserted at the begining, npx of new node
       will always be XOR of current head and NULL */
    new_node->npx = XOR(*head_ref, NULL);
 
    /* If linked list is not empty, then npx of current head node will be XOR 
       of new node and node next to current head */
    if (*head_ref != NULL)
    {
        // *(head_ref)->npx is XOR of NULL and next. So if we do XOR of 
        // it with NULL, we get next
        struct node* next = XOR((*head_ref)->npx,  NULL);
        (*head_ref)->npx = XOR(new_node, next);
    }
 
    // Change head
    *head_ref = new_node;
}
 
// prints contents of doubly linked list in forward direction
void printList (struct node *head)
{
    struct node *curr = head;
    struct node *prev = NULL;
    struct node *next;
 
    printf ("Following are the nodes of Linked List: \n");
 
    while (curr != NULL)
    {
        // print current node
        printf ("%d ", curr->data);
 
        // get address of next node: curr->npx is next^prev, so curr->npx^prev
        // will be next^prev^prev which is next
        next = XOR (prev, curr->npx);
 
        // update prev and curr for next iteration
        prev = curr;
        curr = next;
    }
}
 
// Driver program to test above functions
int main ()
{
    /* Create following Doubly Linked List
       head-->40<-->30<-->20<-->10   */
    struct node *head = NULL;
    insert(&head, 10);
    insert(&head, 20);
    insert(&head, 30);
    insert(&head, 40);
 
    // print the created list
    printList (head);
 
    return (0);
}

O código acima realiza duas funções básicas: inserção e transversal.

  • Inserção: É realizada pela função inserir. Isso insere um novo nó no início. Quando esta função é chamada, ela coloca o novo nó como o nó principal e o nó principal anterior como o segundo nó.

  • Traversal: Isto é executado pela função printList. Ele simplesmente passa por cada nó e imprime o valor.

Observe que XOR de ponteiros não é definido pelo padrão C/C++. Portanto, a implementação acima pode não funcionar em todas as plataformas.

Introdução às listas vinculadas #

Uma lista vinculada é uma coleção linear de elementos de dados, chamados nós, que são vinculados a outro(s) nó(s) por meio de um “ponteiro”. Abaixo está uma lista vinculada individualmente com uma referência principal.

         ┌─────────┬─────────┐   ┌─────────┬─────────┐         
 HEAD ──▶│  data   │"pointer"│──▶│  data   │"pointer"│──▶ null 
         └─────────┴─────────┘   └─────────┴─────────┘

Existem muitos tipos de listas vinculadas, incluindo [individualmente](https://www.wikiod.com/pt/data-structures/lista-vinculada#Lista vinculada individualmente) e [duplamente](http:// stackoverflow.com/documentation/data-structures/2967/linked-list/23473/double-linked-list) listas vinculadas e listas vinculadas circulares.

Vantagens

  • As listas vinculadas são uma estrutura de dados dinâmica, que pode crescer e diminuir, alocando e desalocando memória enquanto o programa está em execução.

  • As operações de inserção e exclusão de nós são facilmente implementadas em uma lista vinculada.

  • Estruturas de dados lineares, como pilhas e filas, são facilmente implementadas com uma lista vinculada.

  • As listas vinculadas podem reduzir o tempo de acesso e expandir em tempo real sem sobrecarga de memória.

Lista vinculada individualmente #

Listas vinculadas individualmente são um tipo de [lista vinculada](https://www.wikiod.com/pt/data-structures/lista-vinculada#Introdução às listas vinculadas). Os nós de uma lista vinculada individualmente têm apenas um “ponteiro” para outro nó, geralmente “próximo”. Ela é chamada de lista vinculada individualmente porque cada nó tem apenas um único “ponteiro” para outro nó. Uma lista encadeada simples pode ter uma referência inicial e/ou final. A vantagem de ter uma referência de cauda são os casos getFromBack, addToBack e removeFromBack, que se tornam O(1).

 HEAD ──▶│  data   │"pointer"│──▶│  data   │"pointer"│──▶ null 
         └─────────┴────△────┘   └─────────┴─────────┘         
          SINGLE        │                                      
          REFERENCE ────┘

Lista de Ignorar #

As listas de pular são listas vinculadas que permitem pular para o nó correto. Este é um método que é muito mais rápido do que uma lista simples encadeada normal. É basicamente uma [lista vinculada única] (https://www.wikiod.com/pt/data-structures/lista-vinculada#Lista vinculada individualmente), mas os ponteiros não vão de um nó para o próximo nó , mas pulando alguns nós. Daí o nome “Skip List”.

*

Isso é diferente de uma lista vinculada individualmente?

**É sim.

Lista duplamente vinculada #

Listas duplamente vinculadas são um tipo de [lista vinculada](https://www.wikiod.com/pt/data-structures/lista-vinculada#Introdução às listas vinculadas). Os nós de uma lista duplamente vinculada têm dois “ponteiros” para outros nós, “próximo” e “anterior”. Ela é chamada de lista encadeada dupla porque cada nó tem apenas dois “ponteiros” para outros nós. Uma lista duplamente ligada pode ter um ponteiro de cabeça e/ou cauda.

         ┌─────────┬─────────┬─────────┐   ┌─────────┬─────────┬─────────┐         
 null ◀──┤previous │  data   │  next   │◀──┤previous │  data   │  next   │         
         │"pointer"│         │"pointer"│──▶│"pointer"│         │"pointer"│──▶ null 
         └─────────┴─────────┴─────────┘   └─────────┴─────────┴─────────┘         
                          ▲                     △                   △              
                     HEAD │                     │     DOUBLE        │              
                                                └──── REFERENCE ────┘

Listas duplamente vinculadas são menos eficientes em termos de espaço do que listas vinculadas simples; no entanto, para algumas operações, eles oferecem melhorias significativas na eficiência do tempo. Um exemplo simples é a função get, que para uma lista duplamente ligada com uma referência head e tail começará de head ou tail dependendo do índice do elemento a ser obtido. Para uma lista de elementos n, para obter o elemento indexado n/2 + i, uma lista vinculada simples com referências de cabeça/cauda deve percorrer os nós n/2 + i, porque não pode “voltar” de rabo. Uma lista duplamente vinculada com referências de cabeça/cauda só precisa percorrer os nós n/2 - i, porque pode “voltar” da cauda, ​​percorrendo a lista na ordem inversa.

Um exemplo básico de SinglyLinkedList em Java #

Uma implementação básica para lista vinculada simples em java - que pode adicionar valores inteiros ao final da lista, remover o primeiro valor encontrado da lista, retornar uma matriz de valores em um determinado instante e determinar se um determinado valor está presente na lista.

/**
 * <p> Basic node implementation </p>
 * @since 20161008

 * @version 0.1
 */
public class Node {
    
    private Node next;
    private int value;
    
    public Node(int value) {
        this.value=value;
    }

    public Node getNext() {
        return next;
    }

    public void setNext(Node next) {
        this.next = next;
    }

    public int getValue() {
        return value;
    }
    
    
}

SinglyLinkedList.java

package com.example.so.ds;


/**
 * <p> Basic single-linked-list implementation </p>
 * @since 20161008
 * @author Ravindra HV
 * @version 0.2
 */
public class SinglyLinkedList {
    
    private Node head;
    private volatile int size;
    
    public int getSize() {
        return size;
    }
    
    public synchronized void append(int value) {
        
        Node entry = new Node(value);
        if(head == null) {
            head=entry;
        }
        else {
            Node temp=head;
            while( temp.getNext() != null) {
                temp=temp.getNext();
            }
            temp.setNext(entry);
        }
        
        size++;
    }
    
    
    public synchronized boolean removeFirst(int value) {
        boolean result = false;
        
        if( head == null ) { // or size is zero..
            // no action
        }
        else if( head.getValue() == value ) {
            head = head.getNext();
            result = true;
        }
        else {

            Node previous = head;
            Node temp = head.getNext();
            while( (temp != null) && (temp.getValue() != value) ) {
                previous = temp;
                temp = temp.getNext();
            }
            
            if((temp != null) && (temp.getValue() == value)) { // if temp is null then not found..
                previous.setNext( temp.getNext() );
                result = true;
            }

        }
        
        if(result) {
            size--;
        }
        
        return result;
        
    }
    
    public synchronized int[] snapshot() {
        Node temp=head;
        int[] result = new int[size];
        for(int i=0;i<size;i++) {
            result[i]=temp.getValue();
            temp = temp.getNext();
        }
        return result;
    }
    
    public synchronized boolean contains(int value) {
        boolean result = false;
        Node temp = head;
        
        while(temp!=null) {
            if(temp.getValue() == value) {
                result=true;
                break;
            }
            temp=temp.getNext();
        }
        return result;
    }
    
}
