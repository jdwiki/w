Uma estrutura de dados union-find (ou disjoint-set) é uma estrutura de dados simples, uma partição de um número de elementos em conjuntos disjuntos. Cada conjunto tem um representante que pode ser usado para distingui-lo dos outros conjuntos.

É usado em muitos algoritmos, por exemplo. calcular árvores geradoras mínimas através do algoritmo de Kruskal, calcular componentes conectados em grafos não direcionados e muito mais.

Teoria #

As estruturas de dados de localização de união fornecem as seguintes operações:

  • make_sets(n) inicializa uma estrutura de dados union-find com n singletons
  • find(i) retorna um representante para o conjunto do elemento i
  • union(i,j) mescla os conjuntos contendo i e j

Representamos nossa partição dos elementos 0 para n - 1 armazenando um elemento pai parent[i] para cada elemento i que eventualmente leva a um representante do conjunto contendo i . Se um elemento em si é um representante, ele é seu próprio pai, ou seja, parent[i] == i.

Se, no entanto, queremos mesclar conjuntos maiores, devemos sempre alterar o ponteiro pai do representante do conjunto que deve ser mesclado em outro conjunto: Após merge(0,3), definimos o pai do representante do conjunto contendo 0 para o representante do conjunto contendo 3

Podemos facilitar isso armazenando o representante de cada conjunto diretamente em seu nó pai. Implementação básica #

A implementação mais básica de uma estrutura de dados union-find consiste em um array parent armazenando um elemento pai para cada elemento da estrutura. Seguir esses ‘ponteiros’ pai para um elemento i nos leva ao representante j = find(i) do conjunto contendo i, onde parent[j] = j vale.

using std::size_t;

class union_find {
private:
    std::vector<size_t> parent;  // Parent for every node

public:
    union_find(size_t n) : parent(n) {
        for (size_t i = 0; i < n; ++i)
            parent[i] = i;      // Every element is its own representative
    }

    size_t find(size_t i) const {
        if (parent[i] == i)     // If we already have a representative
            return i;           // return it
        return find(parent[i]); // otherwise return the parent's representative
    }

    void merge(size_t i, size_t j) {
        size_t pi = find(i);
        size_t pj = find(j);
        if (pi != pj) {        // If the elements are not in the same set: 
            parent[pi] = pj;   // Join the sets by marking pj as pi's parent
        }
    }
};

Melhorias: compactação de caminho #

Se fizermos muitas operações merge em uma estrutura de dados union-find, os caminhos representados pelos ponteiros parent podem ser bastante longos. A compactação de caminho, conforme já descrito na parte teórica, é uma maneira simples de mitigar esse problema.

Podemos tentar fazer a compactação de caminho em toda a estrutura de dados após cada operação de mesclagem k-th ou algo semelhante, mas tal operação pode ter um tempo de execução bastante grande.

Assim, a compressão do caminho é usada principalmente apenas em uma pequena parte da estrutura, especialmente no caminho que percorremos para encontrar o representante de um conjunto. Isso pode ser feito armazenando o resultado da operação find após cada subchamada recursiva:

size_t find(size_t i) const {
    if (parent[i] == i)          // If we already have a representative
        return i;                // return it
    parent[i] = find(parent[i]); // path-compress on the way to the representative
    return parent[i];            // and return it
}

Melhorias: União por tamanho #

Em nossa implementação atual de merge, sempre escolhemos o conjunto esquerdo como filho do conjunto direito, sem levar em consideração o tamanho dos conjuntos. Sem essa restrição, os caminhos (sem compressão de caminho) de um elemento para seu representante podem ser bastante longos, levando a grandes tempos de execução em chamadas find.

Outra melhoria comum, portanto, é a heurística união por tamanho, que faz exatamente o que diz: Ao mesclar dois conjuntos, sempre definimos o conjunto maior para ser o pai do conjunto menor, levando a um comprimento de caminho de no máximo passos:

Armazenamos um membro adicional std::vector<size_t> size em nossa classe que é inicializado em 1 para cada elemento. Ao mesclar dois conjuntos, o conjunto maior se torna o pai do conjunto menor e somamos os dois tamanhos:

private:
    ...
    std::vector<size_t> size;

public:
    union_find(size_t n) : parent(n), size(n, 1) { ... }

    ...

    void merge(size_t i, size_t j) {
        size_t pi = find(i);
        size_t pj = find(j);
        if (pi == pj) {            // If the elements are in the same set: 
            return;                // do nothing
        }
        if (size[pi] > size[pj]) { // Swap representatives such that pj
            std::swap(pi, pj);     // represents the larger set
        }
        parent[pi] = pj;           // attach the smaller set to the larger one
        size[pj] += size[pi];      // update the size of the larger set
    }

Melhorias: União por classificação #

Outra heurística comumente usada em vez de união por tamanho é a heurística união por classificação

Sua ideia básica é que não precisamos armazenar o tamanho exato dos conjuntos, uma aproximação do tamanho (neste caso: aproximadamente o logaritmo do tamanho do conjunto) é suficiente para atingir a mesma velocidade da união por tamanho.

Para isso, introduzimos a noção de rank de um conjunto, que é dada da seguinte forma:

  • Singletons têm classificação 0
  • Se dois conjuntos com classificação desigual forem mesclados, o conjunto com classificação maior se tornará o pai enquanto a classificação permanecerá inalterada.
  • Se dois conjuntos de mesma classificação forem mesclados, um deles se tornará pai do outro (a escolha pode ser arbitrária), sua classificação será incrementada.

Uma vantagem de união por classificação é o uso de espaço: como a classificação máxima aumenta aproximadamente como , para todos os tamanhos de entrada realistas, a classificação pode ser armazenada em um único byte ( desde n < 2^255).

Uma implementação simples de união por classificação pode ser assim:

private:
    ...
    std::vector<unsigned char> rank;

public:
    union_find(size_t n) : parent(n), rank(n, 0) { ... }

    ...

    void merge(size_t i, size_t j) {
        size_t pi = find(i);
        size_t pj = find(j);
        if (pi == pj) {
            return;
        }
        if (rank[pi] < rank[pj]) {
            // link the smaller group to the larger one
            parent[pi] = pj;
        } else if (rank[pi] > rank[pj]) {
            // link the smaller group to the larger one
            parent[pj] = pi;
        } else {
            // equal rank: link arbitrarily and increase rank
            parent[pj] = pi;
            ++rank[pi];
        }
    }

Melhoria final: União por classificação com armazenamento fora dos limites #

Embora em combinação com a compactação de caminho, a união por classificação quase atinja operações de tempo constante em estruturas de dados união-localização, há um truque final que nos permite nos livrar completamente do armazenamento rank armazenando a classificação fora dos limites entradas do array parent. É baseado nas seguintes observações:

  • Na verdade, só precisamos armazenar a classificação para representantes, não para outros elementos. Para esses representantes, não precisamos armazenar um pai.
  • Até agora, parent[i] é no máximo size - 1, ou seja, valores maiores não são usados.
  • Todas as classificações são no máximo .

Isso nos leva à seguinte abordagem:

  • Em vez da condição parent[i] == i, agora identificamos representantes por pai[i] >= tamanho
  • Usamos esses valores fora dos limites para armazenar os ranks do conjunto, ou seja, o conjunto com representante i tem rank parent[i] - size
  • Assim, inicializamos o array pai com parent[i] = size em vez de parent[i] = i, ou seja, cada conjunto é seu próprio representante com rank 0.

Como apenas compensamos os valores de classificação por size, podemos simplesmente substituir o vetor rank pelo vetor parent na implementação de merge e só precisamos trocar a condição que identifica os representantes em find:

Implementação finalizada usando união por classificação e compactação de caminho:

using std::size_t;

class union_find {
private:
    std::vector<size_t> parent;

public:
    union_find(size_t n) : parent(n, n) {} // initialize with parent[i] = n

    size_t find(size_t i) const {
        if (parent[i] >= parent.size()) // If we already have a representative
            return i;                   // return it
        return find(parent[i]);         // otherwise return the parent's repr.
    }

    void merge(size_t i, size_t j) {
        size_t pi = find(i);
        size_t pj = find(j);
        if (pi == pj) {
            return;
        }
        if (parent[pi] < parent[pj]) {
            // link the smaller group to the larger one
            parent[pi] = pj;
        } else if (parent[pi] > parent[pj]) {
            // link the smaller group to the larger one
            parent[pj] = pi;
        } else {
            // equal rank: link arbitrarily and increase rank
            parent[pj] = pi;
            ++parent[pi];
        }
    }
};
