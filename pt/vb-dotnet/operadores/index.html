Os operadores são usados ​​para atribuir ou comparar valores. Eles consistem em um único símbolo ou palavra-chave e geralmente são intercalados entre um valor esquerdo e um valor direito. Por exemplo: direita = esquerda.

Os operadores são intrínsecos à linguagem (como =), e não funções como as fornecidas pelo System.Math.

Matemática #

Se você tiver as seguintes variáveis

Dim leftValue As Integer = 5
Dim rightValue As Integer = 2
Dim value As Integer = 0

Adição Realizado pelo sinal de mais +.

value  = leftValue + rightValue

'Output the following:
'7

Subtração Realizado pelo sinal de menos -.

value = leftValue - rightValue

'Output the following:
'3

Multiplicação Realizado pelo símbolo de estrela *.

value = leftValue * rightValue

'Output the following:
'10

Divisão Realizado pelo símbolo de barra /.

value = leftValue / rightValue

'Output the following:
'2.5

Divisão inteira Executado pelo símbolo de barra invertida \.

value = leftValue \ rightValue

'Output the following:
'2

Módulo Executado pela palavra-chave Mod.

value = leftValue Mod rightValue

'Output the following:
'1

Elevar para uma Potência de Realizado pelo símbolo ^.

value = leftValue ^ rightValue

'Output the following:
'25

Concatenação de Strings #

A concatenação de strings é quando você combina duas ou mais strings em uma única variável de string.

A concatenação de strings é realizada com o símbolo &.

Dim one As String = "Hello "
Dim two As String = "there"
Dim result As String = one & two

Valores não string serão convertidos em string ao usar &.

Dim result as String = "2" & 10 ' result  = "210"

Sempre use & (e comercial) para realizar a concatenação de strings.

NÃO FAÇA ISSO Embora seja possível, no mais simples dos casos, usar o símbolo + para fazer a concatenação de strings, você nunca deve fazer isso. Se um lado do símbolo de mais não for uma string, quando Option strict estiver desativado, o comportamento se tornará não intuitivo, quando Option strict estiver ativado, produzirá um erro do compilador. Considerar:

Dim value = "2" + 10    ' result = 12  (data type Double)
Dim value = "2" + "10"  ' result = "210"  (data type String)
Dim value = "2g" + 10   ' runtime error

O problema aqui é que se o operador + vir qualquer operando que seja do tipo numérico, ele irá presumir que o programador deseja realizar uma operação aritmética e tentar converter o outro operando para o tipo numérico equivalente. Nos casos em que o outro operando é uma string que contém um número (por exemplo, “10”), a string é convertida em um número e então aritmeticamente adicionada ao outro operando. Se o outro operando não puder ser convertido em um número (por exemplo, “2g”), a operação falhará devido a um erro de conversão de dados. O operador + só realizará a concatenação de strings se ambos os operandos forem do tipo String.

O operador &, no entanto, é projetado para concatenação de strings e converterá tipos não string em strings.

Atribuição #

Existe um único operador de atribuição em VB.

  • O sinal de igual = é usado tanto para comparação quanto para atribuição de igualdade. Dim value = 5

Notas Cuidado com a comparação de atribuição versus igualdade.

Dim result = leftValue = rightValue

Neste exemplo, você pode ver o sinal de igual sendo usado tanto como operador de comparação quanto como operador de atribuição, ao contrário de outros idiomas. Neste caso, result será do tipo Boolean e conterá o valor da comparação de igualdade entre leftValue e rightValue.

Comparação #

Os operadores de comparação comparam dois valores e retornam a você um booleano (True ou False) como resultado.

Igualdade

  • O sinal de igual = é usado tanto para comparação quanto para atribuição de igualdade. If leftValue = rightValue Then …

Desigualdade

  • O aninhamento do colchete angular esquerdo com o colchete angular direito <> executa uma comparação desigual. If leftValue <> rightValue Then …

Maior que

  • O colchete angular esquerdo «/kbd> executa uma comparação maior que. If leftValue < rightValue Then …

Maior ou igual

  • O aninhamento do sinal de igual para o colchete angular esquerdo => executa uma comparação maior que ou igual. If leftValue =< rightValue Then …

Menor que

  • O colchete angular direito > executa uma comparação menor que. If leftValue > rightValue Then …

Menor ou igual

  • O aninhamento do sinal de igual para o colchete angular direito => executa uma comparação maior que ou igual. If leftValue => rightValue Then …

Curti

  • O operador Like testa a igualdade de uma string e um padrão de pesquisa.
  • O operador Like depende da Instrução de comparação de opções
  • A tabela a seguir lista os padrões disponíveis. Fonte: https://msdn.microsoft.com/en-us/library/swf8kaxw.aspx (seção Observações)
Caracteres no Padrão Corresponde à String

* Zero ou mais caracteres
# Qualquer dígito único (0 - 9)
[carlist] Qualquer caractere único em charlist

Bit a bit #

Estes são os operadores bit a bit no VB.NET: And, Or, Xor, Not

Exemplo de operação E bit a bit

Dim a as Integer
a = 3 And 5

O valor de a será 1. O resultado é obtido após comparar 3 e 5 em binário para. 3 na forma binária é 011 e 5 na forma binária é 101. O operador And coloca 1 se ambos os bits forem 1. Se algum dos bits for 0, o valor será 0

3 And 5 will be  011
                 101
                 ---
                 001

Portanto, o resultado binário é 001 e, quando convertido em decimal, a resposta será 1.

Ou o operador coloca 1 se ambos ou um bit for 1

   3 Or 5 will be 011
                  101
                  ---
                  111

O operador Xor coloca 1 se apenas um dos bits for 1 (não ambos)

   3 Xor 5 will be  011
                    101
                    ---
                    110

O operador não reverte os bits incluindo o sinal

Not 5 will be - 010
