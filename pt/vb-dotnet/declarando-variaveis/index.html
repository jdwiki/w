Declarando e atribuindo uma variável usando um tipo primitivo #

Variáveis ​​em Visual Basic são declaradas usando a palavra-chave Dim. Por exemplo, isso declara uma nova variável chamada counter com o tipo de dados Integer:

Dim counter As Integer

Uma declaração de variável também pode incluir um modificador de acesso, como Public, Protected, Friend ou Private . Isso funciona em conjunto com o escopo da variável para determinar sua acessibilidade.

Modificador de acesso Significado
Público Todos os tipos que podem acessar o tipo anexo
Protegido Apenas a classe envolvente e os que dela herdam
Amigo Todos os tipos no mesmo assembly que podem acessar o tipo delimitador
Amigo Protegido A classe delimitadora e seus herdeiros, ou os tipos no mesmo assembly que podem acessar a classe delimitadora
Privado Apenas o tipo de fechamento
Estático Apenas em variáveis ​​locais e inicializa apenas uma vez.

Como abreviação, a palavra-chave Dim pode ser substituída pelo modificador de acesso na declaração da variável:

Public TotalItems As Integer
Private counter As Integer

Os sufixos integrais também podem ser usados ​​com prefixos hexadecimais (&H) ou octais (&O): exemplo = &H8000S ou exemplo = &O77&

Objetos Date(Time) também podem ser definidos usando a sintaxe literal: Exemplo de dim como data = #7/26/2016 12:8 PM#

Uma vez que uma variável é declarada, ela existirá dentro do Scope do tipo contido, Sub ou Function declarado, como exemplo:

Public Function IncrementCounter() As Integer
    Dim counter As Integer = 0
    counter += 1

    Return counter
End Function

A variável counter só existirá até a Fim Function e então ficará fora do escopo. Se esta variável de contador for necessária fora da função, você terá que defini-la em nível de classe/estrutura ou módulo.

Public Class ExampleClass

    Private _counter As Integer
   
    Public Function IncrementCounter() As Integer
       _counter += 1
       Return _counter
    End Function

End Class

Alternativamente, você pode usar o modificador Static (não confundir com Shared) para permitir que uma variável local retenha seu valor entre as chamadas de seu método delimitador:

Function IncrementCounter() As Integer
    Static counter As Integer = 0
    counter += 1

    Return counter
End Function

Níveis de declaração – Variáveis ​​locais e de membro #

Variáveis ​​locais - As declaradas dentro de um procedimento (sub-rotina ou função) de uma classe (ou outra estrutura). Neste exemplo, exampleLocalVariable é uma variável local declarada dentro de ExampleFunction():

Public Class ExampleClass1

    Public Function ExampleFunction() As Integer
        Dim exampleLocalVariable As Integer = 3
        Return exampleLocalVariable
    End Function

End Class

A palavra-chave Static permite que uma variável local seja mantida e mantenha seu valor após o término (onde geralmente, as variáveis ​​locais deixam de existir quando o procedimento que a contém termina).

Neste exemplo, o console é 024. Em cada chamada para ExampleSub() de Main() a variável estática retém o valor que tinha no final da chamada anterior:

Module Module1

    Sub Main()
        ExampleSub()
        ExampleSub()
        ExampleSub()
    End Sub

    Public Sub ExampleSub()
        Static exampleStaticLocalVariable As Integer = 0
        Console.Write(exampleStaticLocalVariable.ToString)
        exampleStaticLocalVariable += 2
    End Sub

End Module

Variáveis ​​de membro - Declaradas fora de qualquer procedimento, no nível de classe (ou outra estrutura). Elas podem ser variáveis ​​de instância, nas quais cada instância da classe que as contém tem sua própria cópia distinta dessa variável, ou variáveis compartilhadas, que existem como uma única variável associada à própria classe, independente de qualquer instância.

Aqui, ExampleClass2 contém duas variáveis ​​de membro. Cada instância de ExampleClass2 tem uma ExampleInstanceVariable individual que pode ser acessada através da referência de classe. A variável compartilhada ExampleSharedVariable, no entanto, é acessada usando o nome da classe:

Module Module1

    Sub Main()

        Dim instance1 As ExampleClass4 = New ExampleClass4
        instance1.ExampleInstanceVariable = "Foo"

        Dim instance2 As ExampleClass4 = New ExampleClass4
        instance2.ExampleInstanceVariable = "Bar"

        Console.WriteLine(instance1.ExampleInstanceVariable)
        Console.WriteLine(instance2.ExampleInstanceVariable)
        Console.WriteLine(ExampleClass4.ExampleSharedVariable)

    End Sub

    Public Class ExampleClass4

        Public ExampleInstanceVariable As String
        Public Shared ExampleSharedVariable As String = "FizzBuzz"

    End Class

End Module

Exemplo de modificadores de acesso #

No exemplo a seguir, considere que você tem uma solução que hospeda dois projetos: ConsoleApplication1 e SampleClassLibrary. O primeiro projeto terá as classes SampleClass1 e SampleClass2. O segundo terá SampleClass3 e SampleClass4. Em outras palavras, temos dois assemblies com duas classes cada. ConsoleApplication1 tem uma referência a SampleClassLibrary.

Veja como SampleClass1.MethodA interage com outras classes e métodos.

SampleClass1.vb: <pré> Importa SampleClassLibrary

Classe pública SampleClass1 Public Sub MethodA() ‘MethodA can call any of the following methods because ’they all are in the same scope. MethodB() MethodC() MethodD() MethodE()

    'Sample2 is defined as friend. It is accessible within
    'the type itself and all namespaces and code within the same assembly.
    Dim class2 As New SampleClass2() 
    class2.MethodA()
    'class2.MethodB() 'SampleClass2.MethodB is not accessible because
                      'this method is private. SampleClass2.MethodB
                      'can only be called from SampleClass2.MethodA,
                      'SampleClass2.MethodC, SampleClass2.MethodD
                      'and SampleClass2.MethodE
    class2.MethodC()
    'class2.MethodD() 'SampleClass2.MethodD is not accessible because
                      'this method is protected. SampleClass2.MethodD
                      'can only be called from any class that inherits
                      'SampleClass2, SampleClass2.MethodA, SampleClass2.MethodC,
                      'SampleClass2.MethodD and SampleClass2.MethodE
    class2.MethodE()

    Dim class3 As New SampleClass3() 'SampleClass3 resides in other
                                     'assembly and is defined as public.
                                     'It is accessible anywhere.
    class3.MethodA()
    'class3.MethodB() 'SampleClass3.MethodB is not accessible because
                      'this method is private. SampleClass3.MethodB can
                      'only be called from SampleClass3.MethodA,
                      'SampleClass3.MethodC, SampleClass3.MethodD
                      'and SampleClass3.MethodE

    'class3.MethodC() 'SampleClass3.MethodC is not accessible because
                      'this method is friend and resides in another assembly.
                      'SampleClass3.MethodC can only be called anywhere from the
                      'same assembly, SampleClass3.MethodA, SampleClass3.MethodB,
                      'SampleClass3.MethodD and SampleClass3.MethodE

    'class4.MethodD() 'SampleClass3.MethodE is not accessible because
                      'this method is protected friend. SampleClass3.MethodD
                      'can only be called from any class that resides inside
                      'the same assembly and inherits SampleClass3,
                      'SampleClass3.MethodA, SampleClass3.MethodB,
                      'SampleClass3.MethodC and SampleClass3.MethodD      

    'Dim class4 As New SampleClass4() 'SampleClass4 is not accessible because
                                      'it is defined as friend and resides in
                                      'other assembly.
End Sub
