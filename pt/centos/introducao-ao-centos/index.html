Documentação Como guiar para CentOS #

Versões 2 - 5 do CentOS

CentOS versão 7

O CentOS 7 é totalmente baseado no RedHat, a documentação detalhada, exemplos e guias de administração do sistema estão localizados aqui:Documentação completa do CentOS 7

Instalação ou configuração #

Instruções detalhadas sobre como obter a instalação do CentOS 7 e a configuração básica para iniciar.

  1. Baixe o último CentOS .ISO - https://www.centos.org/download/
  2. Depois de baixar a última versão do CentOS usando os links acima ou usando a página de download oficial do CentOS. Grave-o em um DVD ou crie um pendrive inicializável usando o LiveUSB Creator chamado Unetbootin.
  3. Depois de criar a mídia inicializável do instalador, coloque seu DVD/USB na unidade apropriada do sistema, inicie o computador, selecione sua unidade inicializável e o primeiro prompt do CentOS 7 deve aparecer. No prompt, escolha Instalar CentOS 7 e pressione a tecla [Enter].

  1. O sistema começará a carregar o instalador de mídia e uma tela de boas-vindas deve aparecer. Selecione o Idioma do Processo de Instalação, que o auxiliará em todo o procedimento de instalação e clique em Continuar.

  1. A próxima etapa, o prompt da tela atual, é o Resumo da Instalação. Ele contém muitas opções para personalizar totalmente seu sistema. A primeira coisa que você pode querer configurar são suas configurações de tempo. Clique em Data e Hora e selecione a localização física do servidor no mapa fornecido e pressione o botão Concluído superior para aplicar a configuração.

  1. A próxima etapa é escolher as configurações de suporte a idiomas e teclado. Escolha seu idioma principal e extra para o seu sistema e, quando terminar, clique no botão Concluído.

  1. Da mesma forma, escolha o layout do teclado pressionando o botão mais e teste a configuração do teclado usando o campo de entrada correto. Depois de terminar de configurar seu teclado, você pode usar qualquer combinação de teclas para alternar entre teclados, no meu caso estou usando Alt+Ctrl. Após a seleção da combinação de teclas desejada, pressione Concluído novamente para aplicar as alterações e voltar à tela principal em Resumo da instalação.

  1. Agora podemos adicionar SUPORTE AO IDIOMA se você não quiser usar o inglês. Clique em “SUPORTE A IDIOMA” para abrir a caixa de diálogo.

  1. Por padrão, o CentOS vem com o idioma inglês pré-instalado, mas podemos adicionar mais idiomas facilmente. No meu caso, estou adicionando Alemão Alemão com Alemão (Deutschland) como idioma adicional. Pressione Concluído após a seleção.

  1. Na próxima etapa, você pode personalizar sua instalação usando outras Fontes de Instalação além da mídia DVD/USB local, como locais de rede usando protocolos HTTP, HTTPS, FTP ou NFS e até adicionar alguns repositórios adicionais, mas use apenas esses métodos se você sabe o que está fazendo. Portanto, deixe a mídia de instalação padrão detectada automaticamente e clique em Concluído para continuar.

  1. Na próxima etapa, você pode escolher o software de instalação do sistema. Nesta etapa, o CentOS oferece muitos ambientes de plataforma Server e Desktop que você escolhe, mas, se você deseja um alto grau de personalização, especialmente se for usar o CentOS 7 para executar como plataforma de servidor, sugiro que selecione Instalação mínima com bibliotecas de compatibilidade como complementos, que instalarão um software de sistema básico mínimo e, posteriormente, você poderá adicionar outros pacotes conforme suas necessidades, usando:

    [ yum groupinstall “Name of installed package” ] command.

  1. Agora é hora de particionar seu disco rígido. Clique no menu Installation Destination, selecione seu disco e escolha I will configure partitioning. Leia mais sobre qual partição escolher aqui: https://www.centos.org/docs/5/html/Installation_Guide-en-US/s1-diskpartitioning-x86.html

  1. Na próxima tela, escolha LVM (Logical Volume Manager) como layout de partição e, em seguida, clique em Clique aqui para criá-los automaticamente, opção que criará três partições de sistema usando o sistema de arquivos XFS, redistribuindo automaticamente seu espaço em disco rígido e reunindo todos os LVS em um grande Grupo de Volume chamado “centos”.
  • /boot – Não LVM
  • /(raiz) – LVM
  • Troca – LVM

  1. Se você não estiver satisfeito com o layout de partição padrão feito automaticamente pelo instalador, você pode adicionar, modificar ou redimensionar completamente seu esquema de partição e, quando terminar, pressione o botão Concluído e Aceitar alterações no prompt Resumo de alterações.

NOTA: Para aqueles usuários que possuem discos rígidos com mais de 2 TB de tamanho, o instalador converterá automaticamente a tabela de partição para GPT, mas se você desejar usar a tabela GPT em discos menores que 2 TB, deverá usar o argumento inst.gpt para a linha de comando de inicialização do instalador para alterar o comportamento padrão.

  1. A próxima etapa é definir o nome do host do sistema e habilitar a rede. Clique na etiqueta Network & Hostname e digite o FQDN do seu sistema (Fully Qualified Domain Name) no campo Hostname, então habilite sua interface de rede, alternando o botão Ethernet superior para ON. Se você tiver um servidor DHCP funcional em sua rede, ele configurará automaticamente todas as configurações de rede para NIC habilitada, que deve aparecer em sua interface ativa.

  1. Se o seu sistema for destinado a um servidor, é melhor definir a configuração de rede estática na NIC Ethernet clicando no botão Configurar e adicionar todas as configurações de interface estática como na captura de tela abaixo, e quando terminar, clique no botão Salvar, desative e ative a placa Ethernet alternando o botão para OFF e ON e, em seguida, pressione Concluído para aplicar a configuração e voltar ao menu principal.

  1. Adicione as entradas para Endereço, Máscara de Rede e Gateway de acordo com seu ambiente de IP estático. No meu caso estou usando Address como 192.168.1.100, Netmask 255.255.255.0, Gateway como 192.168.1.1 e servidores DNS como 8.8.8.8  8.8.4.4 Esses valores podem variar de acordo com seu ambiente de rede. Depois disso, pressione Salvar.

IMPORTANTE: se você não tiver uma conexão com a Internet IPv6, defina o IPv6 de automático para ignorar na guia IPv6. Caso contrário, você não poderá acessar a Internet a partir deste servidor no IPv4, pois o CentOS parece ignorar a configuração correta do IPv4. e usa IPv6 em vez disso, que falha.

  1. Em seguida, temos que ativar a conexão conforme mostrado na captura de tela abaixo. Além disso, pressione Concluído.

  1. Agora é hora de iniciar o processo de instalação pressionando o botão Iniciar instalação e configurar uma senha forte para a conta root.

  1. O processo de instalação começará agora e você verá uma pequena barra azul de progresso nas próximas janelas. Agora temos que definir a SENHA ROOT e adicionar um novo usuário não raiz na opção CRIAÇÃO DO USUÁRIO. Vou primeiro para a senha de root.

  1. Digite uma senha segura de sua escolha e pressione Concluído

  1. Em seguida iremos para a CRIAÇÃO DE USUÁRIOS.

  1. Em seguida, criarei o usuário, pois no meu caso usei o Nome completo  “Administrador” e Nome de usuário “administrador”, marque a opção Exigir senha para usar esta conta e pressione Concluído. Fora do curso, você pode usar qualquer valor conforme sua escolha.

  1. Pressione Concluir. Tenha paciência e aguarde a conclusão da configuração.

  1. Após a conclusão da instalação, ele pedirá para reiniciar o servidor, basta pressionar Concluir configuração.

  1. O servidor será reiniciado e solicitará seu nome de usuário e senha posteriormente.

Parabéns! Agora você instalou a última versão do CentOS em sua nova máquina. Remova qualquer mídia de instalação e reinicie seu computador para que você possa fazer login em seu novo ambiente CentOS 7 mínimo e executar outras tarefas do sistema, como atualizar seu sistema e instalar outro software útil necessário para executar tarefas diárias.

  1. Agora estamos prontos para fazer login com o usuário que acabamos de criar acima ou podemos usar as credenciais raiz.

Primeiro login no CentOS

Faça login como usuário root no servidor para que possamos fazer algumas etapas finais de instalação.

A primeira é instalar todas as atualizações disponíveis com o yum.

yum update

confirme com “y” para prosseguir com a instalação das atualizações.

Vou instalar dois editores de linha de comando para poder editar arquivos de configuração no shell:

yum install nano vim

configuração de rede

CentOS 7.2 minimal não vem pré-instalado com o comando ifconfig vamos instalá-lo da seguinte forma:

yum install net-tools

Se você quiser alterar ou ver o arquivo de configuração de rede, basta editar o arquivo

nano /etc/sysconfig/network-scripts/ifcfg-ens33

Será assim quando você configurou um endereço IP estático:

    Or like this when you use DHCP:

Nota: O nome do DEVICE acima pode variar, portanto, verifique o arquivo equivalente no diretório /etc/sysconfig/network-scripts.

Ajuste /etc/hosts

Ajuste o arquivo /etc/hosts da seguinte forma:

nano /etc/hosts

Faça os valores assim:

127.0.0.1   localhost localhost.localdomain localhost4 localhost4.localdomain4
192.168.1.100   server1.example.com     server1

::1         localhost localhost.localdomain localhost6 localhost6.localdomain6

Parabéns! Agora temos a configuração mínima básica do servidor CentOS 7

Agora você pode preferir usar a GUI, aqui está uma variedade de sabores que você pode escolher:

Instalando o GNOME-Desktop:

Instale o GNOME Desktop Environment digitando.

# yum -y groups install "GNOME Desktop"

Para iniciar a GUI, digite após terminar a instalação:

# startx

Como usar o GNOME Shell?

O GNOME Desktop padrão do CentOS 7 começa com o modo clássico, mas se você quiser usar o GNOME Shell, defina como segue:

Opção A: Se você iniciar o GNOME com startx, defina como segue.

# echo "exec gnome-session" >> ~/.xinitrc
# startx

Opção B: defina o login gráfico do sistema systemctl set-default graphical.target e reinicie o sistema. Após o sistema iniciar

  1. Clique no botão localizado ao lado do botão “Entrar”.
  2. Selecione “GNOME” na lista. (O padrão é GNOME Classic)
  3. Clique em “Entrar” e faça login com o GNOME Shell.

O shell do GNOME começa da seguinte forma:

Instalando o KDE-Desktop:

Instale o Ambiente de Trabalho KDE digitando

# yum -y groups install "KDE Plasma Workspaces"

Insira um comando como abaixo após terminar a instalação:

# echo "exec startkde" >> ~/.xinitrc
# startx

O Ambiente de Trabalho KDE inicia da seguinte forma:

Instalando o ambiente de desktop MATE:

Instale o MATE Desktop Environment digitando.

# yum --enablerepo=epel -y groups install "MATE Desktop"

Insira um comando como abaixo após terminar a instalação:

# echo "exec /usr/bin/mate-session" >> ~/.xinitrc 
# startx

O ambiente de área de trabalho MATE é iniciado.

Instalando o ambiente de desktop Xfce:

Instale o Xfce Desktop Environment digitando.

# yum --enablerepo=epel -y groups install "Xfce"

Insira um comando como abaixo após terminar a instalação:

# echo "exec /usr/bin/xfce4-session" >> ~/.xinitrc 
# startx

O ambiente de desktop Xfce é iniciado.

OUTRA FORMA DE FAZER:

Em vez de usar o hacking de um comando startx em um arquivo .xinitrc, provavelmente é melhor dizer ao Systemd que você deseja inicializar em uma GUI gráfica em vez do terminal.

Para isso basta fazer o seguinte:

$ sudo yum groupinstall "GNOME Desktop"
$ ln -sf /lib/systemd/system/runlevel5.target /etc/systemd/system/default.target

Em seguida, basta reiniciar.

O último bit associará o destino do nível de execução 5 como seu padrão em relação ao Systemd.

Fazendo isso com Systemd

Você também pode usar o Systemd para fazer isso. Este é sem dúvida o melhor método, pois você está gerenciando o estado do sistema diretamente por meio do Systemd e suas CLIs.

Você pode ver qual é seu destino padrão atual:

$ sudo systemctl get-default
multi-user.target

E então mude para gráfico:

$ sudo systemctl set-default
graphical.target

Alvos

No Systemd, os destinos runlevel5.target e graphical.target são idênticos. Assim também são runlevel2.target e multi-user.target.

Runlevel    Target Units                          Description
0           runlevel0.target, poweroff.target     Shut down and power off the system.
1           runlevel1.target, rescue.target       Set up a rescue shell.
2           runlevel2.target, multi-user.target   Set up a non-graphical multi-user system.
3           runlevel3.target, multi-user.target   Set up a non-graphical multi-user system.
4           runlevel4.target, multi-user.target   Set up a non-graphical multi-user system.
5           runlevel5.target, graphical.target    Set up a graphical multi-user system.
6           runlevel6.target, reboot.target       Shut down and reboot the system.

P. Como instalo todas as ferramentas de desenvolvimento, como compiladores GNU GCC C/C++, make e outros, após instalar CentOS ou RHEL ou Fedora Linux a partir de um prompt de shell?

Você precisa instalar o grupo ‘Ferramentas de Desenvolvimento’ no RHEL/CentOS/Fedora/Scientific/Red Hat Enterprise Linux. Essas ferramentas incluem ferramentas de desenvolvimento essenciais, como automake, gcc, perl, python e depuradores, que são necessárias para compilar software e criar novos rpms:

  16. binutils e todas as dependências.

Instalação:

Abra o terminal ou faça login pela sessão ssh e digite o seguinte comando como usuário root:

# yum groupinstall 'Development Tools'

Como faço para listar todos os serviços atualmente em execução no servidor Linux Fedora / RHEL / CentOS?

Existem várias maneiras e ferramentas para encontrar e listar todos os serviços em execução nos sistemas Fedora / RHEL / CentOS Linux.

service command – list running services

A sintaxe é a seguinte para CentOS/RHEL 6.xe anterior (pré systemd):

service --status-all
service --status-all | more
service --status-all | grep ntpd
service --status-all | less

Imprima o status de qualquer serviço. Para imprimir o status do serviço apache (httpd):

service httpd status

Listar todos os serviços conhecidos (configurados via SysV)

chkconfig --list

Serviço de lista e suas portas abertas

netstat -tulpn

Ativar/desativar o serviço

ntsysv
chkconfig service off
chkconfig service on
chkconfig httpd off
chkconfig ntpd on

ntsysv é uma interface simples para configurar serviços de nível de execução que também são configuráveis ​​através do chkconfig. Por padrão, ele configura o nível de execução atual. Basta digitar ntsysv e selecionar o serviço que deseja executar.

Uma nota sobre o RHEL/CentOS 7.x com systemd

Se você estiver usando uma distribuição baseada em systemd, como Fedora Linux v22/23/24 ou RHEL/CentOS Linux 7.x+. Tente o seguinte comando para listar os serviços em execução usando o comando systemctl. Ele controla o sistema systemd e o gerenciador de serviços.

Para listar serviços systemd no CentOS/RHEL 7.x+, use

A sintaxe é:

systemctl
systemctl | more
systemctl | grep httpd
systemctl list-units --type service
systemctl list-units --type mount
 

Para listar todos os serviços:

systemctl list-unit-files

systemd-cgtop
