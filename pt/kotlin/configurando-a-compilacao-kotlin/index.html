Configuração do Gradle #

kotlin-gradle-plugin é usado para compilar código Kotlin com Gradle. Basicamente, sua versão deve corresponder à versão Kotlin que você deseja usar. Por exemplo. se você quiser usar Kotlin 1.0.3, então você precisa aplicar kotlin-gradle-plugin versão 1.0.3 também.

É uma boa ideia externalizar esta versão em gradle.properties ou em [ExtraPropertiesExtension](https:/ /docs.gradle.org/current/dsl/org.gradle.api.plugins.ExtraPropertiesExtension.html):

buildscript {
   ext.kotlin_version = '1.0.3'

   repositories {
     mavenCentral()
   }

   dependencies {
     classpath "org.jetbrains.kotlin:kotlin-gradle-plugin:$kotlin_version"
   }
}

Então você precisa aplicar este plugin ao seu projeto. A maneira como você faz isso é diferente ao segmentar plataformas diferentes:

JVM de segmentação #

apply plugin: 'kotlin'

Segmentação para Android #

apply plugin: 'kotlin-android'

Segmentação de JS #

apply plugin: 'kotlin2js'

Estes são os caminhos padrão:

  • fontes kotlin: src/main/kotlin
  • fontes java: src/main/java
  • testes kotlin: src/test/kotlin
  • testes java: src/test/java
  • recursos de tempo de execução: src/main/resources
  • recursos de teste: src/test/resources

Você pode precisar configurar SourceSets se estiver usando um layout de projeto personalizado.

Por fim, você precisará adicionar a dependência da biblioteca padrão Kotlin ao seu projeto:

dependencies {
    compile "org.jetbrains.kotlin:kotlin-stdlib:$kotlin_version"
}

Se você quiser usar o Kotlin Reflection, também precisará adicionar compile "org.jetbrains.kotlin:kotlin-reflect:$kotlin_version"

Usando o Android Studio #

O Android Studio pode configurar o Kotlin automaticamente em um projeto Android.

Instale o plug-in #

Para instalar o plug-in Kotlin, vá para File > Settings > Editor > Plugins > Install JetBrains Plugin… > Kotlin > Install e reinicie o Android Studio quando solicitado.

Configurar um projeto #

Crie um projeto do Android Studio normalmente e pressione Ctrl + Shift + A. Na caixa de pesquisa, digite “Configure Kotlin in Project” e pressione Enter.

O Android Studio alterará seus arquivos Gradle para adicionar todas as dependências necessárias.

Convertendo Java #

Para converter seus arquivos Java para arquivos Kotlin, pressione Ctrl + Shift + A e encontre “Converter Arquivo Java para Arquivo Kotlin”. Isso mudará a extensão do arquivo atual para .kt e converterá o código para Kotlin.

Migrando do Gradle usando o script Groovy para o script Kotlin #

Passos:

  • clonar o projeto gradle-script-kotlin

  • copie/cole do projeto clonado para o seu projeto:

  • build.gradle.kts

  • gradlew

  • gradlew.bat

  • settings.gradle

  • atualize o conteúdo do build.gradle.kts com base em suas necessidades, você pode usar como inspiração os scripts no projeto recém clonado ou em um de seus exemplos

  • agora abra o Intellij e abra seu projeto, na janela do explorer, ele deve ser reconhecido como um projeto Gradle, caso contrário, expanda-o primeiro.

  • depois de abrir, deixe o Intellij funcionar, abra o build.gradle.kts e verifique se há algum erro. Se o realce não estiver funcionando e/ou estiver tudo marcado em vermelho, feche e reabra o Intellij

  • abra a janela Gradle e atualize-a

Se você estiver no Windows, poderá encontrar este bug, baixe a distribuição completa do Gradle 3.3 e use-a em vez da fornecida. Relacionado.

OSX e Ubuntu funcionam fora da caixa.

Pequeno bônus, se você quiser evitar todo o incômodo de publicar no Maven e similares, use Jitpack, as linhas a serem adicionadas são quase idênticas em comparação com o Groovy. Você pode se inspirar neste projeto meu.
