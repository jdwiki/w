Elencos inteligentes #

Se o compilador pode inferir que um objeto não pode ser nulo em um determinado ponto, você não precisa mais usar os operadores especiais:

var string: String? = "Hello!"
print(string.length)  // Compile error
if(string != null) {
    // The compiler now knows that string can't be null
    print(string.length)  // It works now!
}

Observação: o compilador não permitirá a conversão inteligente de variáveis ​​mutáveis ​​que possam ser modificadas entre a verificação nula e o uso pretendido.

Se uma variável for acessível de fora do escopo do bloco atual (porque eles são membros de um objeto não local, por exemplo), você precisa criar uma nova referência local que você pode usar e fazer uma conversão inteligente.

Asserção #

Os sufixos !! ignoram a nulidade e retornam uma versão não nula desse tipo. KotlinNullPointerException será lançado se o objeto for um null.

val message: String? = null
println(message!!) //KotlinNullPointerException thrown, app crashes

Tipos anuláveis ​​e não anuláveis #

Tipos normais, como String, não são anuláveis. Para torná-los capazes de manter valores nulos, você deve denotar isso explicitamente colocando um ? atrás deles: String?

var string        : String = "Hello World!"
var nullableString: String? = null

string = nullableString    // Compiler error: Can't assign nullable to non-nullable type.
nullableString = string    // This will work however!

Elimine nulos de um Iterable e array #

Às vezes precisamos mudar o tipo de Collection<T?> para Collections<T>. Nesse caso, filterNotNull é nossa solução.

val a: List<Int?> = listOf(1, 2, 3, null)
val b: List<Int> = a.filterNotNull()

Coalescência Nula / Operador Elvis #

Às vezes, é desejável avaliar uma expressão anulável de uma forma if-else. O operador elvis, ?:, pode ser usado em Kotlin para tal situação.

Por exemplo:

val value: String = data?.first() ?: "Nothing here."

A expressão acima retorna "Nada aqui" se data?.first() ou data produzir um valor null senão o resultado de data?.first().

Também é possível lançar exceções usando a mesma sintaxe para abortar a execução do código.

val value: String = data?.second() 
    ?: throw IllegalArgumentException("Value can't be null!")

Lembrete: NullPointerExceptions podem ser lançadas usando o operador de declaração (por exemplo, data!!.second()!!)

Operadora de chamadas segura #

Para acessar funções e propriedades de tipos anuláveis, você precisa usar operadores especiais.

O primeiro, ?., fornece a propriedade ou função que você está tentando acessar, ou fornece null se o objeto for null:

val string: String? = "Hello World!"
print(string.length)   // Compile error: Can't directly access property of nullable type.
print(string?.length)  // Will print the string's length, or "null" if the string is null.

Idiom: chamando vários métodos no mesmo objeto com verificação nula #

Uma maneira elegante de chamar vários métodos de um objeto com verificação nula é usar o apply do Kotlin assim:

obj?.apply { 
    foo()
    bar()
}

Isso chamará foo e bar em obj (que é this no bloco apply) somente se obj for não nulo, pulando o bloco inteiro caso contrário.

Para trazer uma variável anulável no escopo como uma referência não anulável sem torná-la o receptor implícito de chamadas de função e propriedade, você pode usar let em vez de apply:

nullable?.let { notnull ->
    notnull.foo()
    notnull.bar()
}

notnull pode ser nomeado qualquer coisa, ou até mesmo deixado de fora e usado através do [parâmetro lambda implícito it.][3]

[3]: https://www.wikiod.com/pt/kotlin/funcoes#Funções Lambda

Operador Elvis (?:) #

Em Kotlin, podemos declarar variáveis ​​que podem conter referência nula. Suponha que tenhamos uma referência anulável a, podemos dizer “se a não for nulo, use-o, caso contrário, use algum valor não nulo x”

var a: String? = "Nullable String Value"

Agora, a pode ser nulo. Então, quando precisamos acessar o valor de a, precisamos realizar a verificação de segurança, se contém valor ou não. Podemos realizar esta verificação de segurança pela declaração convencional if...else.

val b: Int = if (a != null) a.length else -1

Mas aqui vem o operador avançado Elvis(Operador Elvis : ?:). Acima de if...else pode ser expresso com o operador Elvis como abaixo:

val b = a?.length ?: -1

Se a expressão à esquerda de ?: (aqui: a?.length) não for nula, o operador elvis a retornará, caso contrário, retornará a expressão à direita (aqui: -1). A expressão do lado direito é avaliada apenas se o lado esquerdo for nulo.
