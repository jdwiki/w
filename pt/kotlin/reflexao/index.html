A reflexão é a capacidade de uma linguagem de inspecionar o código em tempo de execução, em vez de em tempo de compilação.

A reflexão é um mecanismo para introspecção de construções de linguagem (classes e funções) em tempo de execução.

Ao direcionar a plataforma JVM, os recursos de reflexão de tempo de execução são distribuídos em JAR separado: kotlin-reflect.jar. Isso é feito para reduzir o tamanho do tempo de execução, cortar recursos não utilizados e possibilitar o direcionamento de outras plataformas (como JS).

Referenciando uma classe #

Para obter uma referência a um objeto KClass representando alguma classe, use dois pontos duplos:

val c1 = String::class
val c2 = MyClass::class

Interoperando com reflexão Java #

Para obter um objeto Java Class do Kotlin [KClass](https://kotlinlang .org/api/latest/jvm/stdlib/kotlin.reflect/-k-class/index.html) use a propriedade de extensão .java:

val stringKClass: KClass<String> = String::class
val c1: Class<String> = stringKClass.java

val c2: Class<MyClass> = MyClass::class.java

O último exemplo será otimizado pelo compilador para não alocar uma instância intermediária do KClass.

Referenciando uma função #

As funções são cidadãos de primeira classe em Kotlin. Você pode obter uma referência usando dois pontos duplos e depois passá-la para outra função:

fun isPositive(x: Int) = x > 0

val numbers = listOf(-2, -1, 0, 1, 2)
println(numbers.filter(::isPositive)) // [1, 2]

Obtendo valores de todas as propriedades de uma classe #

Dada a classe Example estendendo a classe BaseExample com algumas propriedades:

open class BaseExample(val baseField: String)

class Example(val field1: String, val field2: Int, baseField: String): 
    BaseExample(baseField) {
    
    val field3: String
        get() = "Property without backing field"

    val field4 by lazy { "Delegated value" }

    private val privateField: String = "Private value"
}

Pode-se obter todas as propriedades de uma classe:

val example = Example(field1 = "abc", field2 = 1, baseField = "someText")

example::class.memberProperties.forEach { member ->
    println("${member.name} -> ${member.get(example)}")
}

A execução desse código fará com que uma exceção seja lançada. A propriedade private val privateField é declarada privada e chamar member.get(example) não terá sucesso. Uma maneira de lidar com isso é filtrar propriedades privadas. Para fazer isso, temos que verificar o modificador de visibilidade do getter Java de uma propriedade. No caso de private val o getter não existe, então podemos assumir o acesso privado.

A função auxiliar e seu uso podem ser assim:

fun isFieldAccessible(property: KProperty1<*, *>): Boolean {
    return property.javaGetter?.modifiers?.let { !Modifier.isPrivate(it) } ?: false
}

val example = Example(field1 = "abc", field2 = 1, baseField = "someText")

example::class.memberProperties.filter { isFieldAccessible(it) }.forEach { member ->
    println("${member.name} -> ${member.get(example)}")
}

Outra abordagem é tornar as propriedades privadas acessíveis usando reflexão:

example::class.memberProperties.forEach { member ->
    member.isAccessible = true
    println("${member.name} -> ${member.get(example)}")
}

Configurando valores de todas as propriedades de uma classe #

Como exemplo, queremos definir todas as propriedades de string de uma classe de amostra

class TestClass {
    val readOnlyProperty: String
        get() = "Read only!"

    var readWriteString = "asd"
    var readWriteInt = 23

    var readWriteBackedStringProperty: String = ""
        get() = field + '5'
        set(value) { field = value + '5' }

    var readWriteBackedIntProperty: Int = 0
        get() = field + 1
        set(value) { field = value - 1 }

    var delegatedProperty: Int by TestDelegate()

    private var privateProperty = "This should be private"

    private class TestDelegate {
        private var backingField = 3

        operator fun getValue(thisRef: Any?, prop: KProperty<*>): Int {
            return backingField
        }

        operator fun setValue(thisRef: Any?, prop: KProperty<*>, value: Int) {
            backingField += value
        }
    }
}

A obtenção de propriedades mutáveis ​​baseia-se na obtenção de todas as propriedades, filtrando as propriedades mutáveis ​​por tipo. Também precisamos verificar a visibilidade, pois a leitura de propriedades privadas resulta em exceção de tempo de execução.

val instance = TestClass()
TestClass::class.memberProperties
        .filter{ prop.visibility == KVisibility.PUBLIC }
        .filterIsInstance<KMutableProperty<*>>()
        .forEach { prop ->
            System.out.println("${prop.name} -> ${prop.get(instance)")
        }

Para definir todas as propriedades String para "Our Value", podemos filtrar adicionalmente pelo tipo de retorno. Como o Kotlin é baseado em Java VM, Type Erasure está em vigor e, portanto, Propriedades que retornam tipos genéricos como List<String> serão iguais a List<Any>. Infelizmente, a reflexão não é uma bala de ouro e não há uma maneira sensata de evitar isso, então você precisa tomar cuidado com seus casos de uso.

val instance = TestClass()
TestClass::class.memberProperties
        .filter{ prop.visibility == KVisibility.PUBLIC }
        // We only want strings
        .filter{ it.returnType.isSubtypeOf(String::class.starProjectedType) }
        .filterIsInstance<KMutableProperty<*>>()
        .forEach { prop ->
            // Instead of printing the property we set it to some value
            prop.setter.call(instance, "Our Value")
        }
