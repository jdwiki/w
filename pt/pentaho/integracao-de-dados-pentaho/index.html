Olá Mundo na Integração de Dados Pentaho #

Vamos criar uma transformação simples para converter um CSV em um arquivo XML.

Nossa Transformação tem que fazer o seguinte:

  • Leia o arquivo CSV
  • Construir a mensagem de saudação
  • Salve as saudações no arquivo XML

Crie uma transformação: #

Veja como iniciar a Transformação:

  1. À esquerda da área de trabalho está a Paleta de Etapas. Selecione a categoria de entrada.
  2. Arraste o arquivo CSV para a área de trabalho à direita.
  3. Selecione a categoria Scripts.
  4. Arraste o ícone Valor JavaScript Modificado para a área de trabalho.
  5. Selecione a categoria Saída.
  6. Arraste o ícone Saída XML para a área de trabalho.

Agora vincule a entrada do arquivo CSV com o valor de script Java modificado criando um salto:

  1. Selecione a primeira etapa.
  2. Segure a tecla Shift e arraste o ícone para o segundo passo.
  3. Vincule o Valor do Java Script Modificado à Saída XML por meio desse mesmo processo.

Configurando a etapa de entrada do arquivo CSV #

  1. Clique duas vezes na etapa de entrada do arquivo CSV. A janela de configuração para a etapa será exibida. Aqui você indicará o local do arquivo, o formato do arquivo (por exemplo, delimitadores, caracteres de inclusão etc.) e os metadados da coluna (por exemplo, nome da coluna, tipo de dados etc.)
  2. Altere o nome da etapa por um que seja mais representativo da função desta etapa. Nesse caso, digite na lista de nomes.
  3. Para o campo Nome do arquivo, clique em Procurar e selecione o arquivo de entrada.
  4. Clique em Obter campos para adicionar a lista de nomes de coluna do arquivo de entrada à grade. Por padrão, a Etapa assume que o arquivo possui cabeçalhos (a caixa de seleção Cabeçalho presente na linha está marcada).
  5. A grade agora tem os nomes das colunas do seu arquivo. 6.Clique em Visualizar para garantir que o arquivo seja lido conforme o esperado. Uma janela mostrando os dados do arquivo aparecerá. Clique em OK para concluir a definição da etapa entrada do arquivo CSV.

Configurando a etapa de valor JavaScript modificado #

  1. Clique duas vezes na Etapa de valor de JavaScript modificado.

  2. A janela de configuração da etapa será exibida. Isso é diferente da janela de configuração da etapa anterior, pois permite escrever código JavaScript. Você o usará para construir a mensagem “Hello”, concatenada com cada um dos nomes.

  3. Nomeie esta etapa como Saudações.

  4. A área principal da janela de configuração é para codificação. À esquerda, há uma árvore com um conjunto de funções disponíveis que você pode usar no código. Em particular, as duas últimas ramificações possuem os campos de entrada e saída, prontos para uso no código. Neste exemplo existem dois campos: last_name e name. Escreva o seguinte código:

    var msg = 'Hello, ' + name + "!";

  5. Na parte inferior você pode digitar qualquer variável criada no código. Nesse caso, você criou uma variável chamada msg. Como você precisa enviar esta mensagem para o arquivo de saída, você deve escrever o nome da variável na grade.

  6. Clique em OK para concluir a configuração da etapa Modified Java Script Value.

  7. Selecione a Etapa que acabou de configurar. Para verificar se o novo campo sairá desta Etapa, você verá agora os Campos de Entrada e Saída. Campos de entrada são as colunas de dados que chegam a uma Etapa. Os Campos de Saída são as colunas de dados que saem de uma Etapa. Existem etapas que simplesmente transformam os dados de entrada. Nesse caso, os campos de entrada e saída geralmente são os mesmos. Existem Etapas, no entanto, que adicionam campos à Saída - Calculadora, por exemplo. Existem outras Etapas que filtram ou combinam dados fazendo com que a Saída tenha menos campos que a Entrada - Agrupar por, por exemplo.

  8. Clique com o botão direito do mouse na Etapa para abrir um menu de contexto.

  9. Selecione Mostrar campos de entrada. Você verá que os campos de entrada são last_name e name, que vêm da etapa de entrada do arquivo CSV.

  10. Selecione Mostrar campos de saída. Você verá que não só tem os campos existentes, mas também o novo campo msg.

Configurando a Etapa de Saída XML #

  1. Clique duas vezes na Etapa de Saída XML. A janela de configuração para este tipo de Etapa aparecerá. Aqui você definirá o nome e o local do arquivo de saída e estabelecerá quais campos deseja incluir. Você pode incluir todos ou alguns dos campos que chegam à Etapa.

  2. Nomeie o arquivo da etapa: Saudações.

  3. Na caixa Arquivo, escreva:

    ${Internal.Transformation.Filename.Directory}/Hello.xml

  4. Clique em Obter campos para preencher a grade com os três campos de entrada.

  5. Salve a Transformação novamente.

CORRE #

Clique no botão RUN na barra de menu e inicie a transformação.

Você também pode criar um Job que pode ser usado para agendar várias transformações e depois executá-lo.

Alternativamente, Pan permite que você execute Transformações de uma janela de terminal. O script é Pan.bat no Windows ou pan.sh em outras plataformas e está localizado na pasta de instalação.

Pan /file <Tutorial_folder_path>/Hello.ktr /norep

Instalação do Pentaho DI #

Pentaho Data Integration vem em duas variedades:

  • Community Edition (CE) - Versão gratuita para desenvolvedores
  • Enterprise Edition (EE) - Versão paga para uso corporativo

Passos de instalação: #

  1. Você pode baixar Pentaho Data Integration Community Edition de Sourceforge.net.

    For the Enterprise Edition : Download a 30-Day Trial

  2. Pré-requisitos - O PDI V7 requer o Oracle Java Runtime Environment (JRE) versão 8.

  3. PDI não requer instalação. Simplesmente descompacte o arquivo zip em uma pasta de sua escolha. Em sistemas operacionais do tipo Unix, você pode precisar tornar os scripts de shell executáveis ​​usando o comando chmod:

    cd data-integration
chmod +x *.sh

  4. Em execução: PDI comes with a graphical user interface called Spoon_,_ command-line scripts (Kitchen, Pan) to execute transformations and jobs, and other utilities.
