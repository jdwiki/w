Aninhamento básico #

Sempre que você declara uma nova regra dentro de outra regra, ela é chamada de aninhamento. Com o aninhamento básico, como mostrado abaixo, o seletor aninhado será compilado como um novo seletor CSS com todos os seus pais prefixados, separados por um espaço.


// SCSS
.parent {
  margin: 1rem;

  .child {
    float: left;
  }
}

// CSS output
.parent {
  margin: 1rem;
}

.parent .child {
  float: left;
}

Profundidade de aninhamento #

O aninhamento é um recurso muito poderoso, mas deve ser usado com cautela. Pode acontecer com bastante facilidade e rapidez que você comece a aninhar e continue incluindo todas as crianças em um ninho, de um ninho, de um ninho. Deixe-me demonstrar:

// SCSS
header {
  // [css-rules]

  .holder {
    // [css-rules]

    .dropdown-list {
      // [css-rules]

      ul {
        // [css-rules]

        li {
          margin: 1rem 0 0 1rem;
        }
      }
    }
  }
}

// CSS output of the last rule
header .holder .dropdown-list ul li {
  margin: 1rem 0 0 1rem;
}

Problemas #

Especificidade #

O li do exemplo acima tem um conjunto margin. Digamos que queremos substituir isso em uma consulta de mídia mais tarde.

@media (max-width: 480) {

  // will not work
  .dropdown-list ul li {
    margin: 0;
  }

  // will work
  header .holder .dropdown-list ul li {
    margin: 0;
  }
}

Portanto, aninhando muito profundamente, consequentemente, você terá que aninhar profundamente novamente sempre que quiser substituir um determinado valor. Pior ainda, muitas vezes é aqui que a regra !important é usada.

@media (max-width: 480) {

  // BIG NO-NO, don't do this
  .dropdown-list ul li {
    margin: 0 !important;
  }

Por que a regra !importante é uma má ideia

Você deve escrever seu SCSS de uma maneira adequada para que essas soluções alternativas nem sejam necessárias em primeiro lugar. Usar !important em uma questão tão pequena já o levará a uma toca de coelho!

Reutilização #

Isso é bastante semelhante ao problema de especificidade, mas vale a pena apontar separadamente. Se você estilizar algo como um botão ou até mesmo um menu suspenso, convém reutilizar esses estilos em outro lugar da sua página.

Ao aninhar muito profundamente, seus estilos são vinculados apenas aos elementos localizados dentro do pai mais externo (o elemento na parte superior do seu SCSS). Isso leva você a copiar estilos e colá-los em outro lugar novamente. Possivelmente em outra regra aninhada.

Suas folhas de estilo ficarão cada vez maiores e mais difíceis de manter.

Quão profundo você deve aninhar? #

A maioria dos guias de estilo define a profundidade máxima como 2. Esse é um bom conselho em geral, pois existem poucas ocasiões em que você deseja aninhar mais profundamente. Na maioria das vezes, 2 é suficiente.

Aninhando com @at-root #

O aninhamento provavelmente é usado com mais frequência para criar seletores mais específicos, mas também pode ser usado simplesmente para organização de código. Usando a diretiva @at-root, você pode ‘saltar’ de onde você a aninha em seu Sass, trazendo você de volta ao nível superior. Isso permite manter os estilos agrupados sem criar mais especificidade do que você precisa.

Por exemplo, você poderia fazer algo assim:

.item {
    color: #333;

    @at-root {
        .item-wrapper {
            color: #666;

            img {
                width: 100%;
            }
        }
    }
    
    .item-child {
        background-color: #555;
    }
}

Isso compilaria para isso:

.item {
  color: #333;
}
.item-wrapper {
  color: #666;
}
.item-wrapper img {
  width: 100%;
}
.item .item-child {
  background-color: #555;
}

Ao fazer isso, todos os nossos estilos relacionados à classe .item estão juntos no SCSS, mas não precisamos necessariamente dessa classe em todos os seletores.

Excluindo contextos

Por padrão, declarações dentro de @at-root aparecerão em qualquer contexto. Isso significa que as regras dentro de um bloco @media, por exemplo, permanecerão lá.

@media print {
  .item-wrapper {
    @at-root {
      .item {
        background: white;
      }
    }
  }
}

// Will compile to
@media print {
  .item {
    background: white;
  }
}

Isso nem sempre é o comportamento desejado, então você pode excluir o contexto de mídia, passando media para a opção without da diretiva @at-root.

@at-root (without: media) {..

Para obter mais informações, consulte a documentação oficial

O seletor pai (&) #

O aninhamento é ótimo para manter os seletores relacionados juntos para facilitar a compreensão do código por futuros desenvolvedores. O seletor pai, representado por um e comercial ("&") pode ajudar a fazer isso em situações mais complexas. Existem várias maneiras de usá-lo.

Crie um novo seletor que exija o seletor pai e outro no mesmo elemento, colocando o novo seletor diretamente após um seletor pai.


// SCSS
.parent {

  &.skin {
    background: pink;
  }
}

// CSS output
.parent.skin {
  background: pink;
}

Faça com que o pai apareça depois de um seletor aninhado no CSS compilado, colocando o seletor pai depois do seletor aninhado.


// SCSS
.parent {

  .wrapper & {
    border: 1px solid black;
  }
}

// CSS output
.wrapper .parent {
  border: 1px solid black;
}

Estados e pseudo-elementos #

Além de usar o aninhamento para classes e filhos, o aninhamento com o seletor pai também é comumente usado para combinar os estados de :active, :hover e :focus para links.

// SCSS
a {
  color: blue;

  &:active,
  &:focus {
    color: red;
  }

  &:visited {
    color: purple;
  }
}

// CSS output
a {
  color: blue;
}

a:active,
a:focus {
  color: red;
}

a:visited {
  color: purple;
}

Da mesma forma, você pode estilizar pseudo-elementos aninhando com o seletor pai.

// SCSS
.parent {

  &::after {
    display: table;
    clear: both;
    content: '';
  }

  &::only-child {
    font-weight: bold;
  }
}

// CSS output
.parent::after {
  display: table;
  clear: both;
  content: '';
}

.parent::only-child {
  font-weight: bold;
}

Propriedades de aninhamento #

Algumas propriedades CSS pertencem a um namespace, por exemplo border-right pertence ao namespace border. Para escrever menos, podemos utilizar o aninhamento de propriedades e pular esses prefixos, mesmo em vários níveis.

Se precisássemos criar uma borda à direita e à esquerda de uma classe chamada .borders, poderíamos escrever isto:

 //SCSS
 .borders {
    border: 2px dashed blue;
    border: {
        left: 1px solid black;
        right: 1px solid red;
    }
  }

 // CSS output 
 .borders {
    border: 2px dashed blue;  
    border-left: 1px solid black;
    border-right: 1px solid red;
  }

Isso nos poupa de escrever border-right e border-left, porém ainda estamos escrevendo código repetitivo com as linhas 1px solid black e 1px solid red. Podemos escrever CSS ainda menos repetitivo com o seguinte:

// SCSS
.borders {
  border: 2px dashed blue {
    left: 1px solid black;
    right: {
      color: red;
    }
  }
}

// CSS output
.borders {
   border: 2px dashed blue;
   border-left: 1px solid black;
   border-right-color: red;
}
