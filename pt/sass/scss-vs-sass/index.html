Principais diferenças #

Embora as pessoas muitas vezes digam Sass como o nome deste pré-processador CSS, eles geralmente significam a sintaxe SCSS. Sass usa a extensão de arquivo .sass, enquanto SCSS-Sass usa a extensão .scss. Ambos são referidos como “Sass”.

Falando em geral, a sintaxe SCSS é mais comumente usada. SCSS parece CSS normal com mais recursos, enquanto Sass parece bem diferente do CSS normal. Ambas as sintaxes têm as mesmas habilidades.

Sintaxe #

As principais diferenças são que o Sass não usa chaves ou ponto e vírgula, enquanto o SCSS usa. Sass também é sensível a espaços em branco, o que significa que você tem que recuar corretamente. Em SCSS, você pode formatar e recuar suas regras como quiser.

#SCSS:

// nesting in SCSS
.parent {
  margin-top: 1rem;

  .child {
    float: left;
    background: blue;
  }
}

#SASS:

// nesting in Sass
.parent
  margin-top: 1rem

  .child
    float: left
    background: blue

Após a compilação, ambos produzirão o mesmo CSS a seguir:

.parent {
  margin-top: 1rem;
}
.parent .child {
  float: left;
  background: blue;
}

Mapas #

Quando se trata de mapas, normalmente SCSS é a sintaxe mais fácil. Como o Sass é baseado em recuo, seus mapas devem ser salvos em uma linha.

// in Sass maps are "unreadable"
$white: ( white-50: rgba(255, 255, 255, .1), white-100: rgba(255, 255, 255, .2), white-200: rgba(255, 255, 255, .3), white-300: rgba(255, 255, 255, .4), white-400: rgba(255, 255, 255, .5), white-500: rgba(255, 255, 255, .6), white-600: rgba(255, 255, 255, .7), white-700: rgba(255, 255, 255, .8), white-800: rgba(255, 255, 255, .9), white-900: rgba(255, 255, 255, 1 )

Como você pode formatar seu código em várias linhas com SCSS, você pode formatar seus mapas para serem mais legíveis.

// in SCSS maps are more readable
$white: (
  white-50: rgba(255, 255, 255, .1),
  white-100: rgba(255, 255, 255, .2),
  white-200: rgba(255, 255, 255, .3),
  white-300: rgba(255, 255, 255, .4),
  white-400: rgba(255, 255, 255, .5),
  white-500: rgba(255, 255, 255, .6),
  white-600: rgba(255, 255, 255, .7),
  white-700: rgba(255, 255, 255, .8),
  white-800: rgba(255, 255, 255, .9),
  white-900: rgba(255, 255, 255, 1)
);

Comentários #

Comentários em Sass vs. Scss são bastante semelhantes, exceto quando se trata de multi-linhas. Multi-linhas SASS são sensíveis à indentação, enquanto SCSS depende de terminadores de comentários.

Comentário de linha única #

estilo.scss

// Just this line will be commented!
h1 { color: red; }

estilo.ass

// Exactly the same as the SCSS Syntax!
h1
  color: red

Comentário de várias linhas #

estilo.scss

Iniciador: /*

Terminador: */

/* This comment takes up
 * two lines.
 */
h1 {
   color: red;
}

Isso estilizará os elementos h1 com a cor red.

estilo.ass

Agora, o SASS tem dois iniciadores, mas não os respectivos terminadores. Comentários de várias linhas em SASS são sensíveis a níveis de recuo.

Iniciadores: // e /*

// This is starts a comment,
   and will persist until you 
   return to the original indentaton level.
h1
  color: red

Isso estilizará os elementos h1 com a cor red.

O mesmo pode ser feito com o iniciador /*:

/* This is starts a comment,
   and will persist until you 
   return to the original indentaton level.
h1
  color: red

Então aí está! As principais diferenças entre comentários em SCSS e SASS!

Misturas #

Sass tende a ser a sintaxe mais “preguiçosa”. Nada ilustra isso melhor do que como você define e inclui mixins.

Definindo um mix #

= é como você define um mixin em Sass, @mixin em SCSS.

// SCSS
@mixin size($x: 10rem, $y: 20rem) {
  width: $x;
  height: $y;
}
  
// Sass
=size($x: 10rem, $y: 20rem)
  width: $x
  height: $y

Incluindo um mixin #

+ é como você inclui em Sass, @include em SCSS.

// SCSS
.element {
  @include size(20rem);
}

// Sass
.element
  +size(20rem)

Comparação entre SCSS e SASS #

  • A sintaxe SCSS se assemelha mais a uma sintaxe CSS, mas a sintaxe SASS é um pouco diferente de SCSS, mas ambas produzem exatamente o mesmo código CSS.
  • Em SASS não estamos terminando as propriedades de estilo com ponto e vírgula(;), mas no SCSS estamos terminando as propriedades de estilo com (;).
  • Em SCSS usamos paranthesis {} para fechar as propriedades do estilo, mas em SASS não usamos paranthesis.
  • Indentation é muito importante em SASS. Ele definirá as propriedades aninhadas na class ou id do elemento.
  • Em scss podemos definir múltiplas variáveis ​​em uma única linha, mas em SASS não podemos.

para sintaxe de loop #

Com o lançamento do sass 3.3 e versão plus, a sintaxe das condições @if e else ficou igual. agora podemos usar expressões não apenas com scss, mas também com sass.

sintaxe atrevida

@for $i from 1 through 3 {
  .item-#{$i} { width: 2em * $i; }
}

Compilado para

.item-1 {
  width: 2em;
}
.item-2 {
  width: 4em;
}
.item-3 {
  width: 6em;
}

sintaxe scss

@for $i from 1 through 3 {
  .item-#{$i} { width: 2em * $i; }
}

compilado para

.item-1 {
  width: 2em;
}
.item-2 {
  width: 4em;
}
.item-3 {
  width: 6em;
}
