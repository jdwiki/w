Sintaxe #

  • @mixin mixin-name ($argument1, $argument, ...){ ... }

Mixin com argumento variável #

Existem alguns casos em mixins em que pode haver argumentos únicos ou múltiplos ao usá-lo. Vamos pegar um caso de border-radius onde pode haver um único argumento como border-radius:4px; ou vários argumentos como border-radius:4px 3px 2px 1px;.

Tradicional com a mistura de argumentos de palavras-chave será como abaixo: -

@mixin radius($rad1, $rad2, $rad3, $rad4){
 -webkit-border-radius: $rad1 $rad2 $rad3 $rad4;
 -moz-border-radius: $rad1 $rad2 $rad3 $rad4;
 -ms-border-radius: $rad1 $rad2 $rad3 $rad4;
 -o-border-radius: $rad1 $rad2 $rad3 $rad4;
 border-radius: $rad1 $rad2 $rad3 $rad4;
}

E usado como

.foo{
    @include radius(2px, 3px, 5px, 6px)
}

O exemplo acima é complexo (para codificar, ler e manter) e se você não puder passar apenas um valor ou dois valores, ele gerará um erro, e para usar um, dois ou lá valores você deve definir três outros mixins.

Ao usar variable Argument, você não precisa se preocupar com quantos argumentos você pode passar. Os argumentos de variável podem ser declarados definindo um nome de variável seguido por três pontos(…). A seguir está um exemplo de um argumento variável.

@mixin radius($radius...)
{  
    -webkit-border-radius: $radius;
    -moz-border-radius: $radius;
    -ms-border-radius: $radius;
    -o-border-radius: $radius;
    border-radius: $radius;
}

E usado como

.foo{
    @include radius(2px 3px 5px 6px)
}
.foo2{
    @include radius(2px 3px)
}
.foo3{
    @include radius(2px)
}

O exemplo acima é muito mais simples (para codificar, manter e ler), você não precisa se preocupar com quantos argumentos estão por vir - é um ou mais de um.

Se houver mais de um argumento e em qualquer caso você quiser acessar o segundo argumento, você pode fazê-lo escrevendo propertyname : nth(variable_name, 2).

Crie e use um mixin #

Para criar um mixin use a diretiva @mixin.

@mixin default-box ($color, $borderColor) {
    color: $color;
    border: 1px solid $borderColor;
    clear: both;
    display: block;
    margin: 5px 0;
    padding: 5px 10px;
}

Você pode especificar uma lista de argumentos dentro de um parêntese após o nome do mixin. Lembre-se de iniciar suas variáveis ​​com $ e separá-las com vírgulas.

Para usar o mixin em outro seletor, use a diretiva @include.

footer, header{ @include default-box (#ddd, #ccc); }

Os estilos do mixin agora serão usados ​​no footer e no header, com o valor #ccc para a variável $color e #ddd para a variável $borderColor.

Argumentos opcionais #

Os argumentos opcionais do SASS permitem que você use um parâmetro somente se você especificar seu valor; caso contrário, será ignorado. Vamos dar um exemplo do seguinte mixin:

@mixin galerie-thumbnail ($img-height:14em, $img-width: null) {
    width: $img-width;
    height: $img-height;
    outline: 1px solid lightgray;
    outline-offset: 5px;
}

Então uma chamada para

.default { 
  @include galerie-thumbnail; 
}
.with-width { 
  @include galerie-thumbnail($img-width: 12em);
}
.without-height { 
  @include galerie-thumbnail($img-height: null);
}

simplesmente produzirá o seguinte no arquivo CSS:

.default {
  height: 14em;
  outline: 1px solid lightgray;
  outline-offset: 5px;
}

.with-width {
  width: 12em;
  height: 14em;
  outline: 1px solid lightgray;
  outline-offset: 5px;
}

.without-height {
  outline: 1px solid lightgray;
  outline-offset: 5px;
}

O SASS não gera propriedades com null como seu valor, o que é muito útil quando precisamos incluir um argumento opcional em nossa chamada ou não.

diretiva @content #

Mixins podem receber um bloco de código compatível com SASS, que então se torna disponível dentro do mixin como a diretiva @content.

@mixin small-screen {
  @media screen and (min-width: 800px;) {
    @content;
  }
}

@include small-screen {
  .container {
    width: 600px;
  }
}

E isso resultaria:

  @media screen and (min-width: 800px;) {
    .container {
      width: 600px;
    }
  }

Mixins podem usar a diretiva @content e ainda aceitar parâmetros.

@mixin small-screen($offset) {...

Padrões sensatos #

O SASS oferece a capacidade de omitir qualquer parâmetro, exceto os que você deseja substituir, é claro. Vamos pegar novamente o exemplo default-box:

@mixin default-box ($color: red, $borderColor: blue) {
    color: $color;
    border: 1px solid $borderColor;
    clear: both;
    display: block;
    margin: 5px 0;
    padding: 5px 10px;
}

Aqui vamos agora chamar o mixin tendo sobrescrito o segundo parâmetro

footer, header{ @include default-box ($borderColor: #ccc); }

o valor de $borderColor é #ccc, enquanto $color permanece red
