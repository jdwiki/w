Área sob a curva da característica operacional do receptor (AUROC) #

O AUROC é uma das métricas mais usadas para avaliar o desempenho de um classificador. Esta seção explica como calculá-lo.

AUC (Área Sob a Curva) é usado na maioria das vezes para significar AUROC, o que é uma má prática, pois AUC é ambígua (pode ser qualquer curva), enquanto AUROC não é.

Visão geral – Abreviaturas #

Abreviatura Significado
AUROC Área Sob a Curva da Característica Operacional do Receptor
AUC Área sob a Cura
ROC Característica Operacional do Receptor
TP Verdadeiros Positivos
TN Verdadeiros Negativos
FP Falsos Positivos
FN Falsos Negativos
TPR Taxa de Verdadeiro Positivo
FPR Taxa de falsos positivos

Interpretando o AUROC #

O AUROC tem várias interpretações equivalentes:

  • A expectativa de que um positivo aleatório uniformemente sorteado seja classificado antes de um negativo aleatório sorteado uniformemente.
  • A proporção esperada de positivos classificados antes de um negativo aleatório uniformemente sorteado.
  • A taxa de verdadeiros positivos esperados se a classificação for dividida imediatamente antes de um negativo aleatório uniformemente sorteado.
  • A proporção esperada de negativos classificados após um positivo aleatório uniformemente sorteado.
  • A taxa de falsos positivos esperada se a classificação for dividida logo após um positivo aleatório uniformemente sorteado.

Calculando o AUROC #

Suponha que temos um classificador binário probabilístico, como regressão logística.

Antes de apresentar a curva ROC (= curva Receiver Operating Characteristic), deve-se entender o conceito de matriz de confusão. Quando fazemos uma previsão binária, pode haver 4 tipos de resultados:

  • Prevemos 0 enquanto a classe é realmente 0: isso é chamado de Verdadeiro Negativo, ou seja, prevemos corretamente que a classe é negativa (0). Por exemplo, um antivírus não detectou um arquivo inofensivo como vírus.
  • Prevemos 0 enquanto a classe é realmente 1: isso é chamado de Falso Negativo, ou seja, prevemos incorretamente que a classe é negativa (0). Por exemplo, um antivírus falhou ao detectar um vírus.
  • Prevemos 1 enquanto a classe é realmente 0: isso é chamado de Falso positivo, ou seja, prevemos incorretamente que a classe é positiva (1). Por exemplo, um antivírus considera um arquivo inofensivo um vírus.
  • Prevemos 1 enquanto a classe é realmente 1: isso é chamado de Verdadeiro Positivo, ou seja, prevemos corretamente que a classe é positiva (1). Por exemplo, um antivírus detectou corretamente um vírus.

Para obter a matriz de confusão, analisamos todas as previsões feitas pelo modelo e contamos quantas vezes cada um desses 4 tipos de resultados ocorre:

Neste exemplo de matriz de confusão, entre os 50 pontos de dados classificados, 45 estão classificados corretamente e os 5 estão classificados incorretamente.

Como para comparar dois modelos diferentes, geralmente é mais conveniente ter uma única métrica do que várias, calculamos duas métricas da matriz de confusão, que depois combinaremos em uma:

  • True positive rate (TPR), também conhecido como. sensibilidade, hit rate e recall, que é definido como . Intuitivamente, essa métrica corresponde à proporção de pontos de dados positivos que são corretamente considerados positivos, em relação a todos os pontos de dados positivos. Em outras palavras, quanto maior o TPR, menos pontos de dados positivos perderemos.
  • Taxa de falsos positivos (FPR), também conhecido como. fall-out, que é definido como . Intuitivamente, essa métrica corresponde à proporção de pontos de dados negativos que são erroneamente considerados positivos, em relação a todos os pontos de dados negativos. Em outras palavras, quanto maior o FPR, mais pontos de dados negativos serão classificados incorretamente.

Para combinar o FPR e o TPR em uma única métrica, primeiro calculamos as duas métricas anteriores com muitos limites diferentes (por exemplo ) para a regressão logística e, em seguida, plotamos em um único gráfico, com os valores de FPR nas abcissas e os valores de TPR nas ordenadas. A curva resultante é chamada de curva ROC, e a métrica que consideramos é a AUC dessa curva, que chamamos de AUROC.

Matriz de confusão #

Uma matriz de confusão pode ser usada para avaliar um classificador, com base em um conjunto de dados de teste para os quais os valores verdadeiros são conhecidos. É uma ferramenta simples, que ajuda a dar uma boa visão geral do desempenho do algoritmo que está sendo usado.

Uma matriz de confusão é representada como uma tabela. Neste exemplo, veremos uma matriz de confusão para um classificador binário.

No lado esquerdo, pode-se ver a classe Actual (sendo rotulada como SIM ou NÃO), enquanto a parte superior indica a classe que está sendo prevista e emitida (novamente SIM ou NÃO).

Isso significa que 50 instâncias de teste - que na verdade são instâncias NÃO, foram rotuladas corretamente pelo classificador como NÃO. Estes são chamados de Verdadeiros Negativos (TN). Em contraste, 100 instâncias YES reais foram rotuladas corretamente pelo classificador como instâncias YES. Estes são chamados de Verdadeiros Positivos (TP).

5 instâncias SIM reais, foram rotuladas incorretamente pelo classificador. Estes são chamados de Falsos Negativos (FN). Além disso, 10 instâncias NÃO, foram consideradas instâncias SIM pelo classificador, portanto são Falsos Positivos (FP).

Com base nesses FP,TP,FN e TN, podemos tirar mais conclusões.

  • Taxa de Verdadeiro Positivo:

    • Tries to answer: When an instance is actually YES, how often does the classifier predict YES?

  • Pode ser calculado da seguinte forma: TP/# real YES instâncias = 100/105 = 0,95

  • Taxa de falsos positivos:

    • Tries to answer: When an instance is actually NO, how often does the classifier predict YES?

  • Pode ser calculado da seguinte forma: FP/# instâncias NÃO reais = 10/60 = 0,17

Curvas ROC #

Uma curva ROC (Receiver Operating Characteristic) traça a taxa de TP versus a A taxa de FP como um limite na confiança de uma instância ser positiva é variado

Algoritmo para criar uma curva ROC

  1. classificar as previsões do conjunto de testes de acordo com a confiança de que cada instância é positiva

  2. Percorra a lista classificada de alta a baixa confiança

    i. locate a threshold between instances with opposite classes (keeping instances with the same confidence value on the same side of threshold)

    ii. compute TPR, FPR for instances above threshold

    iii. output (FPR, TPR) coordinate
