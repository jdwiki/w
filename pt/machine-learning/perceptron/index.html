Implementando um modelo Perceptron em C++ #

Neste exemplo vou passar pela implementação do modelo perceptron em C++ para que você possa ter uma ideia melhor de como ele funciona.

Em primeiro lugar, é uma boa prática escrever um algoritmo simples do que queremos fazer.

Algoritmo:

  1. Faça um vetor para os pesos e inicialize-o com 0 (não se esqueça de adicionar o termo de polarização)
  2. Continue ajustando os pesos até obtermos 0 erros ou uma contagem baixa de erros.
  3. Faça previsões sobre dados não vistos.

Tendo escrito um algoritmo super simples, vamos agora escrever algumas das funções que precisaremos.

  • Vamos precisar de uma função para calcular a entrada da rede (e.i x * wT multiplicando as entradas pelos pesos)
  • Uma função degrau para que possamos obter uma previsão de 1 ou -1
  • E uma função que encontre os valores ideais para os pesos.

Então, sem mais delongas, vamos direto ao assunto.

Vamos começar simples criando uma classe perceptron:

class perceptron

Agora vamos adicionar as funções que vamos precisar.

class perceptron
{
public:
    perceptron(float eta,int epochs);
    float netInput(vector<float> X);
    int predict(vector<float> X);
    void fit(vector< vector<float> > X, vector<float> y);
private:

};

Observe como a função fit recebe como argumento um vetor de vetor< float >. Isso ocorre porque nosso conjunto de dados de treinamento é uma matriz de entradas. Essencialmente, podemos imaginar essa matriz como um par de vetores x empilhados um em cima do outro e cada coluna dessa matriz sendo um recurso.

Finalmente vamos adicionar os valores que nossa classe precisa ter. Como o vetor w para guardar os pesos, o número de épocas que indica o número de passagens que faremos sobre o conjunto de dados de treinamento. E a constante eta que é a taxa de aprendizado da qual multiplicaremos cada atualização de peso para tornar o procedimento de treinamento mais rápido discando esse valor ou se eta for muito alto podemos disque-o para obter o resultado ideal (para a maioria das aplicações do perceptron, sugiro um valor eta de 0,1).

class perceptron
{
public:
    perceptron(float eta,int epochs);
    float netInput(vector<float> X);
    int predict(vector<float> X);
    void fit(vector< vector<float> > X, vector<float> y);
private:
    float m_eta;
    int m_epochs;
    vector < float > m_w;
};

Agora com nossa classe definida. É hora de escrever cada uma das funções.

Começaremos a partir do construtor ( perceptron(float eta,int epochs); )

perceptron::perceptron(float eta, int epochs)
{
    m_epochs = epochs; // We set the private variable m_epochs to the user selected value
    m_eta = eta; // We do the same thing for eta
}

Como você pode ver o que vamos fazer é uma coisa muito simples. Então, vamos passar para outra função simples. A função de previsão( int predict(vector X); ). Lembre-se que o que a função all predict faz é pegar a entrada líquida e retornar um valor de 1 se a netInput for maior que 0 e -1 caso contrário.

int perceptron::predict(vector<float> X)
{
    return netInput(X) > 0 ? 1 : -1; //Step Function
}

Observe que usamos uma instrução if embutida para facilitar nossas vidas. Veja como a instrução if inline funciona:

doença ? if_true: else

Até agora tudo bem. Vamos seguir para a implementação da função netInput( float netInput(vector X); )

O netInput faz o seguinte; multiplica o vetor de entrada pela transposição do vetor de pesos

x * wT

Em outras palavras, ele multiplica cada elemento do vetor de entrada x pelo elemento correspondente do vetor de pesos w e então pega sua soma e adiciona o viés.

(x1 * w1 + x2 * w2 + … + xn * wn) + viés

preconceito = 1 * w0

float perceptron::netInput(vector<float> X)
{
    // Sum(Vector of weights * Input vector) + bias
    float probabilities = m_w[0]; // In this example I am adding the perceptron first
    for (int i = 0; i < X.size(); i++)
    {
        probabilities += X[i] * m_w[i + 1]; // Notice that for the weights I am counting
        // from the 2nd element since w0 is the bias and I already added it first.
    }
    return probabilities;
}

Tudo bem, agora estamos praticamente prontos, a última coisa que precisamos fazer é escrever a função fit que modifica os pesos.

void perceptron::fit(vector< vector<float> > X, vector<float> y)
{
    for (int i = 0; i < X[0].size() + 1; i++) // X[0].size() + 1 -> I am using +1 to add the bias term
    {
        m_w.push_back(0); // Setting each weight to 0 and making the size of the vector
        // The same as the number of features (X[0].size()) + 1 for the bias term
    }
    for (int i = 0; i < m_epochs; i++) // Iterating through each epoch
    {
        for (int j = 0; j < X.size(); j++) // Iterating though each vector in our training Matrix
        {
            float update = m_eta * (y[j] - predict(X[j])); //we calculate the change for the weights
            for (int w = 1; w < m_w.size(); w++){ m_w[w] += update * X[j][w - 1]; } // we update each weight by the update * the training sample
            m_w[0] = update; // We update the Bias term and setting it equal to the update
        }
    }
}

Então foi basicamente isso. Com apenas 3 funções, agora temos uma classe perceptron funcional que podemos usar para fazer previsões!

O arquivo class .cpp com as funções:

perceptron::perceptron(float eta, int epochs)
{
    m_epochs = epochs;
    m_eta = eta;
}

void perceptron::fit(vector< vector<float> > X, vector<float> y)
{
    for (int i = 0; i < X[0].size() + 1; i++) // X[0].size() + 1 -> I am using +1 to add the bias term
    {
        m_w.push_back(0);
    }
    for (int i = 0; i < m_epochs; i++)
    {
        int errors = 0;
        for (int j = 0; j < X.size(); j++)
        {
            float update = m_eta * (y[j] - predict(X[j]));
            for (int w = 1; w < m_w.size(); w++){ m_w[w] += update * X[j][w - 1]; }
            m_w[0] = update;
            errors += update != 0 ? 1 : 0;
        }
        m_errors.push_back(errors);
    }
}

float perceptron::netInput(vector<float> X)
{
    // Sum(Vector of weights * Input vector) + bias
    float probabilities = m_w[0];
    for (int i = 0; i < X.size(); i++)
    {
        probabilities += X[i] * m_w[i + 1];
    }
    return probabilities;
}

int perceptron::predict(vector<float> X)
{
    return netInput(X) > 0 ? 1 : -1; //Step Function
}

void perceptron::printErrors()
{
    printVector(m_errors);
}

void perceptron::exportWeights(string filename)
{
    ofstream outFile;
    outFile.open(filename);

    for (int i = 0; i < m_w.size(); i++)
    {
        outFile << m_w[i] << endl;
    }

    outFile.close();
}

void perceptron::importWeights(string filename)
{
    ifstream inFile;
    inFile.open(filename);

    for (int i = 0; i < m_w.size(); i++)
    {
        inFile >> m_w[i];
    }
}

void perceptron::printWeights()
{
    cout << "weights: ";
    for (int i = 0; i < m_w.size(); i++)
    {
        cout << m_w[i] << " ";
    }
    cout << endl;
}

O que exatamente é um perceptron? #

Em sua essência, um modelo perceptron é um dos algoritmos de aprendizado supervisionado mais simples para classificação binária. É um tipo de classificador linear, ou seja, um algoritmo de classificação que faz suas previsões com base em uma função de previsão linear combinando um conjunto de pesos com o vetor de características. Uma maneira mais intuitiva de pensar é como uma Rede Neural com apenas um neurônio.

A forma como funciona é muito simples. Ele obtém um vetor de valores de entrada x dos quais cada elemento é uma característica do nosso conjunto de dados.

Um exemplo: #

Digamos que queremos classificar se um objeto é uma bicicleta ou um carro. Por causa deste exemplo, digamos que queremos selecionar 2 recursos. A altura e a largura do objeto. Nesse caso x = [x1, x2] onde x1 é a altura e x2 é a largura.

Então, uma vez que tenhamos nosso vetor de entrada x, queremos multiplicar cada elemento desse vetor por um peso. Normalmente, quanto maior o valor do peso, mais importante é o recurso. Se, por exemplo, usarmos color como recurso x3 e houver uma bicicleta vermelha e um carro vermelho, o perceptron definirá um peso muito baixo para que a cor não afete o previsão final.

Tudo bem, então nós multiplicamos os 2 vetores x e w e retornamos um vetor. Agora precisamos somar os elementos desse vetor. Uma maneira inteligente de fazer isso é em vez de simplesmente multiplicar x por w, podemos multiplicar x por wT onde **T ** significa transposição. Podemos imaginar a transposição de um vetor como uma versão girada do vetor. Para mais informações, você pode ler a página da Wikipedia. Essencialmente, tomando a transposição do vetor w, obtemos um vetor Nx1 em vez de um 1xN. Assim, se agora multiplicarmos nosso vetor de entrada com tamanho 1xN com este vetor de peso Nx1, obteremos um vetor 1x1 (ou simplesmente um único valor) que será igual a x1 * w1 + x2 * w2 + … + xn * wn. Feito isso, agora temos nossa previsão. Mas há uma última coisa. Essa previsão provavelmente não será um simples 1 ou -1 para poder classificar uma nova amostra. Então, o que podemos fazer é simplesmente dizer o seguinte: Se nossa previsão for maior que 0, dizemos que a amostra pertence à classe 1, caso contrário, se a previsão for menor que zero, dizemos que a amostra pertence à classe -1. Isso é chamado de função degrau.

Mas como obter os pesos certos para que possamos fazer previsões corretas? Em outras palavras, como treinamos nosso modelo perceptron?

Bem, no caso do perceptron, não precisamos de equações matemáticas sofisticadas para treinar nosso modelo. Nossos pesos podem ser ajustados pela seguinte equação:

Δw = eta * (y - previsão) * x(i)

onde x(i) é nossa característica (x1 por exemplo para peso 1, x2 para w2 e assim por diante…).

Também notei que existe uma variável chamada eta que é a taxa de aprendizado. Você pode imaginar a taxa de aprendizado de quão grande queremos que seja a mudança dos pesos. Uma boa taxa de aprendizado resulta em um algoritmo de aprendizado rápido. Um valor muito alto de eta pode resultar em uma quantidade crescente de erros em cada época e resulta no modelo fazendo previsões muito ruins e nunca convergindo. Uma taxa de aprendizado muito baixa pode fazer com que o modelo leve muito tempo para convergir. (Geralmente, um bom valor para definir eta para o modelo perceptron é 0,1, mas pode diferir de caso para caso).

Finalmente, alguns de vocês devem ter notado que a primeira entrada é uma constante ( 1 ) e é multiplicada por w0. Então, o que exatamente é isso? Para obter uma boa previsão, precisamos adicionar um viés. E é exatamente isso que essa constante é.

Para modificar o peso do termo de polarização, usamos a mesma equação que fizemos para os outros pesos, mas neste caso não o multiplicamos pela entrada (porque a entrada é uma constante 1 e, portanto, não precisamos):

Δw = eta * (y - previsão)

Então é basicamente assim que um modelo perceptron simples funciona! Uma vez que treinamos nossos pesos, podemos fornecer novos dados e ter nossas previsões.

NOTA: #

O modelo Perceptron tem uma desvantagem importante! Ele nunca irá convergir (ou seja, encontrar os pesos perfeitos) se os dados não forem linearmente separáveis, o que significa ser capaz de separar as 2 classes em um espaço de recursos por uma linha reta. Portanto, para evitar isso, é uma boa prática adicionar um número fixo de iterações para que o modelo não fique preso ao ajustar pesos que nunca serão perfeitamente ajustados.
