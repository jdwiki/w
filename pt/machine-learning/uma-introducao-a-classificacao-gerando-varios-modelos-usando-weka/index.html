Este tutorial mostrará como usar Weka em código JAVA, carregar arquivo de dados, treinar classificadores e explicar alguns conceitos importantes por trás do aprendizado de máquina.

Weka é um kit de ferramentas para aprendizado de máquina. Ele inclui uma biblioteca de técnicas de aprendizado de máquina e visualização e apresenta uma GUI amigável.

Este tutorial inclui exemplos escritos em JAVA e inclui visuais gerados com a GUI. Sugiro usar a GUI para examinar dados e código JAVA para experimentos estruturados.

Começando: Carregando um conjunto de dados do arquivo #

O conjunto de dados de flores de íris é um conjunto de dados amplamente utilizado para fins de demonstração. Vamos carregá-lo, inspecioná-lo e modificá-lo ligeiramente para uso posterior.

import java.io.File;
import java.net.URL;
import weka.core.Instances;
import weka.core.converters.ArffSaver;
import weka.core.converters.CSVLoader;
import weka.filters.Filter;
import weka.filters.unsupervised.attribute.RenameAttribute;
import weka.classifiers.evaluation.Evaluation;
import weka.classifiers.rules.ZeroR;
import weka.classifiers.bayes.NaiveBayes;
import weka.classifiers.lazy.IBk;
import weka.classifiers.trees.J48;
import weka.classifiers.meta.AdaBoostM1;

public class IrisExperiments {
    public static void main(String args[]) throws Exception
     {
        //First we open stream to a data set as provided on http://archive.ics.uci.edu
         CSVLoader loader = new CSVLoader();
         loader.setSource(new URL("http://archive.ics.uci.edu/ml/machine-learning-databases/iris/iris.data").openStream());
         Instances data = loader.getDataSet();
         
         //This file has 149 examples with 5 attributes
         //In order:
         //  sepal length in cm
         //  sepal width in cm
         //  petal length in cm 
         //  petal width in cm
         //  class ( Iris Setosa , Iris Versicolour, Iris Virginica)

         //Let's briefly inspect the data
         System.out.println("This file has " + data.numInstances()+" examples.");
         System.out.println("The first example looks like this: ");
         for(int i = 0; i < data.instance(0).numAttributes();i++ ){
             System.out.println(data.instance(0).attribute(i));
         } 
         
         // NOTE that the last attribute is Nominal
         // It is convention to have a nominal variable at the last index as target variable
         
         // Let's tidy up the data a little bit
         // Nothing too serious just to show how we can manipulate the data with filters
         RenameAttribute renamer = new RenameAttribute();
         renamer.setOptions(weka.core.Utils.splitOptions("-R last -replace Iris-type"));
         renamer.setInputFormat(data);                          
         data = Filter.useFilter(data, renamer); 
         
         System.out.println("We changed the name of the target class.");
         System.out.println("And now it looks like this:");
         System.out.println(data.instance(0).attribute(4));
         
         //Now we do this for all the attributes
         renamer.setOptions(weka.core.Utils.splitOptions("-R 1 -replace sepal-length"));
         renamer.setInputFormat(data);                          
         data = Filter.useFilter(data, renamer); 
         
         renamer.setOptions(weka.core.Utils.splitOptions("-R 2 -replace sepal-width"));
         renamer.setInputFormat(data);                          
         data = Filter.useFilter(data, renamer); 
         
         renamer.setOptions(weka.core.Utils.splitOptions("-R 3 -replace petal-length"));
         renamer.setInputFormat(data);                          
         data = Filter.useFilter(data, renamer); 
         
         renamer.setOptions(weka.core.Utils.splitOptions("-R 4 -replace petal-width"));
         renamer.setInputFormat(data);                          
         data = Filter.useFilter(data, renamer); 
         
         //Lastly we save our newly created file to disk
         ArffSaver saver = new ArffSaver();
         saver.setInstances(data);
         saver.setFile(new File("IrisSet.arff"));
         saver.writeBatch();
  }
}

Treine o primeiro classificador: Definindo uma linha de base com ZeroR #

ZeroR é um classificador simples. Ele não opera por instância, mas sim na distribuição geral das classes. Ele seleciona a classe com a maior probabilidade a priori. Não é um bom classificador no sentido de que não usa nenhuma informação no candidato, mas é frequentemente usado como linha de base. Observação: outras linhas de base também podem ser usadas, como: classificadores padrão da indústria ou regras artesanais

 // First we tell our data that it's class is hidden in the last attribute
 data.setClassIndex(data.numAttributes() -1);
 // Then we split the data in to two sets
 // randomize first because we don't want unequal distributions
 data.randomize(new java.util.Random(0));
 Instances testset = new Instances(data, 0, 50);
 Instances trainset = new Instances(data, 50, 99);
 
 // Now we build a classifier
 // Train it with the trainset
 ZeroR classifier1 = new ZeroR();
 classifier1.buildClassifier(trainset);
 // Next we test it against the testset
 Evaluation Test = new Evaluation(trainset);
 Test.evaluateModel(classifier1, testset);
 System.out.println(Test.toSummaryString());

A maior classe do conjunto oferece uma taxa correta de 34%. (50 de 149)

Nota: O ZeroR tem um desempenho em torno de 30%. Isso ocorre porque dividimos aleatoriamente em um conjunto de trem e teste. O maior conjunto do conjunto de trens será, portanto, o menor do conjunto de teste. Criar um bom conjunto de teste/trem pode valer a pena

Conhecendo os dados. Treinamento Naive Bayes e kNN #

Para construir um bom classificador, muitas vezes precisaremos ter uma ideia de como os dados são estruturados no espaço de características. A Weka oferece um módulo de visualização que pode ajudar.

Algumas dimensões já separam muito bem as classes. A largura da pétala ordena o conceito de forma bastante organizada, quando comparada à largura da pétala, por exemplo.

Treinar classificadores simples também pode revelar bastante sobre a estrutura dos dados. Eu geralmente gosto de usar Nearest Neighbor e Naive Bayes para essa finalidade. Naive Bayes assume independência, seu bom desempenho é uma indicação de que as dimensões em si guardam informações. k-Nearest-Neighbor funciona atribuindo a classe das k instâncias mais próximas (conhecidas) no espaço de características. Ele é frequentemente usado para examinar a dependência geográfica local, nós o usaremos para examinar se nosso conceito é definido localmente no espaço de recursos.

 //Now we build a Naive Bayes classifier
 NaiveBayes classifier2 = new NaiveBayes();
 classifier2.buildClassifier(trainset);
 // Next we test it against the testset
 Test = new Evaluation(trainset);
 Test.evaluateModel(classifier2, testset);
 System.out.println(Test.toSummaryString());
 
//Now we build a kNN classifier
IBk classifier3 = new IBk();
// We tell the classifier to use the first nearest neighbor as example 
classifier3.setOptions(weka.core.Utils.splitOptions("-K 1"));
classifier3.buildClassifier(trainset);
// Next we test it against the testset
Test = new Evaluation(trainset);
Test.evaluateModel(classifier3, testset);
System.out.println(Test.toSummaryString());

Naive Bayes tem um desempenho muito melhor do que nossa linha de base recém-estabelecida, indicando que recursos independentes contêm informações (lembra da largura da pétala?).

1NN também tem um bom desempenho (na verdade, um pouco melhor neste caso), indicando que algumas de nossas informações são locais. O melhor desempenho pode indicar que alguns efeitos de segunda ordem também contêm informações (Se x e y que classe z).

Juntando: Treinando uma árvore #

As árvores podem construir modelos que funcionam em recursos independentes e em efeitos de segunda ordem. Portanto, eles podem ser bons candidatos para este domínio. Árvores são regras encadeadas, uma regra divide instâncias que chegam a uma regra em subgrupos, que passam para as regras sob a regra.

Os Aprendizes de Árvores geram regras, encadeiam-nas e param de construir árvores quando sentem que as regras se tornam muito específicas, para evitar overfitting. Overfitting significa construir um modelo muito complexo para o conceito que estamos procurando. Modelos superajustados têm bom desempenho nos dados do trem, mas mal nos novos dados

Usamos J48, uma implementação JAVA do C4.5, um algoritmo popular.

//We train a tree using J48
//J48 is a JAVA implementation of the C4.5 algorithm
J48 classifier4 = new J48();
//We set it's confidence level to 0.1
//The confidence level tell J48 how specific a rule can be before it gets pruned
classifier4.setOptions(weka.core.Utils.splitOptions("-C 0.1"));
classifier4.buildClassifier(trainset);
// Next we test it against the testset
Test = new Evaluation(trainset);
Test.evaluateModel(classifier4, testset);
System.out.println(Test.toSummaryString());

System.out.print(classifier4.toString());

//We set it's confidence level to 0.5
//Allowing the tree to maintain more complex rules
classifier4.setOptions(weka.core.Utils.splitOptions("-C 0.5"));
classifier4.buildClassifier(trainset);
// Next we test it against the testset
Test = new Evaluation(trainset);
Test.evaluateModel(classifier4, testset);
System.out.println(Test.toSummaryString());
     
System.out.print(classifier4.toString());

O aprendiz de árvore treinado com a maior confiança gera as regras mais específicas e tem o melhor desempenho no conjunto de teste, aparentemente a especificidade é garantida.

Nota: Ambos os alunos começam com uma regra sobre a largura das pétalas. Lembra como percebemos essa dimensão na visualização ?
