Sintaxe #

  • node [opções] [opções v8] [script.js | -e “script”] [argumentos]

Opções de linha de comando #

-v, --version

Adicionado em: v0.1.3 Imprimir a versão do nó.

-h, --help

Adicionado em: v0.1.3 Opções de linha de comando do nó de impressão. A saída desta opção é menos detalhada do que este documento.

-e, --eval "script"

Adicionado em: v0.5.2 Avalie o seguinte argumento como JavaScript. Os módulos que são predefinidos no REPL também podem ser usados ​​em script.

-p, --print "script"

Adicionado em: v0.6.4 Idêntico a -e, mas imprime o resultado.

-c, --check

Adicionado em: v5.0.0 Sintaxe verifique o script sem executar.

-i, --interactive

Adicionado em: v0.7.7 Abre o REPL mesmo que stdin não pareça ser um terminal.

-r, --require module

Adicionado em: v1.6.0 Pré-carregue o módulo especificado na inicialização.

Segue as regras de resolução do módulo de require(). module pode ser um caminho para um arquivo ou um nome de módulo de nó.

--no-deprecation

Adicionado em: v0.8.0 Silenciar avisos de descontinuação.

--trace-deprecation

Adicionado em: v0.8.0 Imprima rastreamentos de pilha para reprovações.

--throw-deprecation

Adicionado em: v0.11.14 Lance erros para reprovações.

--no-warnings

Adicionado em: v6.0.0 Silencie todos os avisos de processo (incluindo reprovações).

--trace-warnings

Adicionado em: v6.0.0 Imprima rastreamentos de pilha para avisos de processo (incluindo reprovações).

--trace-sync-io

Adicionado em: v2.1.0 Imprime um rastreamento de pilha sempre que a E/S síncrona é detectada após o primeiro turno do loop de eventos.

--zero-fill-buffers

Adicionado em: v6.0.0 Preenche automaticamente todas as instâncias de Buffer e SlowBuffer recém-alocadas.

--preserve-symlinks

Adicionado em: v6.3.0 Instrui o carregador de módulo a preservar links simbólicos ao resolver e armazenar em cache os módulos.

Por padrão, quando o Node.js carrega um módulo de um caminho que está simbolicamente vinculado a um local diferente no disco, o Node.js desreferenciará o link e usará o “caminho real” no disco real do módulo como um identificador e como um caminho raiz para localizar outros módulos de dependência. Na maioria dos casos, esse comportamento padrão é aceitável. No entanto, ao usar dependências de pares simbolicamente vinculadas, conforme ilustrado no exemplo abaixo, o comportamento padrão faz com que uma exceção seja lançada se o módulo A tentar exigir o móduloB como uma dependência de pares:

{appDir}
 ├── app
 │   ├── index.js
 │   └── node_modules
 │       ├── moduleA -> {appDir}/moduleA
 │       └── moduleB
 │           ├── index.js
 │           └── package.json
 └── moduleA
     ├── index.js
     └── package.json

O sinalizador de linha de comando –preserve-symlinks instrui o Node.js a usar o caminho do link simbólico para módulos em oposição ao caminho real, permitindo que as dependências de pares simbolicamente vinculadas sejam encontradas.

Observe, no entanto, que usar –preserve-symlinks pode ter outros efeitos colaterais. Especificamente, módulos nativos simbolicamente vinculados podem falhar ao carregar se estiverem vinculados a partir de mais de um local na árvore de dependências (o Node.js os veria como dois módulos separados e tentaria carregar o módulo várias vezes, fazendo com que uma exceção fosse lançada ).

--track-heap-objects

Adicionado em: v2.4.0 Acompanhe as alocações de objetos de heap para instantâneos de heap.

--prof-process

Adicionado em: v6.0.0 Processe a saída do profiler v8 gerada usando a opção v8 –prof.

--v8-options

Adicionado em: v0.1.3 Imprimir opções de linha de comando v8.

Observação: as opções da v8 permitem que as palavras sejam separadas por traços (-) ou sublinhados (_).

Por exemplo, –stack-trace-limit é equivalente a –stack_trace_limit.

--tls-cipher-list=list

Adicionado em: v4.0.0 Especifique uma lista de cifras TLS padrão alternativa. (Requer que o Node.js seja criado com suporte a criptografia. (Padrão))

--enable-fips

Adicionado em: v6.0.0 Habilite a criptografia compatível com FIPS na inicialização. (Requer que o Node.js seja compilado com ./configure –openssl-fips)

--force-fips

Adicionado em: v6.0.0 Forçar criptografia compatível com FIPS na inicialização. (Não pode ser desabilitado do código de script.) (Mesmos requisitos que –enable-fips)

--icu-data-dir=file

Adicionado em: v0.11.15 Especifique o caminho de carregamento de dados ICU. (substitui NODE_ICU_DATA)

Environment Variables

NODE_DEBUG=module[,…]

Adicionado em: v0.1.32 Lista separada por ‘,’ de módulos principais que devem imprimir informações de depuração.

NODE_PATH=path[:…]

Adicionado em: v0.1.32 ‘:’-lista separada de diretórios prefixados para o caminho de pesquisa do módulo.

Nota: no Windows, esta é uma lista separada por ‘;’.

NODE_DISABLE_COLORS=1

Adicionado em: v0.3.0 Quando definido para 1 as cores não serão usadas no REPL.

NODE_ICU_DATA=file

Adicionado em: v0.11.15 Caminho de dados para dados ICU (objeto Intl). Estenderá dados vinculados quando compilados com suporte a icu pequeno.

NODE_REPL_HISTORY=file

Adicionado em: v5.0.0 Caminho para o arquivo usado para armazenar o histórico de REPL persistente. O caminho padrão é ~/.node_repl_history, que é substituído por essa variável. Definir o valor como uma string vazia ("" ou " “) desativa o histórico de REPL persistente.
