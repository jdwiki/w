Embora tudo no Node.js seja geralmente feito de forma assíncrona, require() não é uma dessas coisas. Como os módulos na prática só precisam ser carregados uma vez, é uma operação de bloqueio e deve ser usada corretamente.

Os módulos são armazenados em cache após a primeira vez que são carregados. Se você estiver editando um módulo em desenvolvimento, será necessário excluir sua entrada no cache do módulo para usar as novas alterações. Dito isto, mesmo que um módulo seja limpo do cache do módulo, o módulo em si não é coletado como lixo, portanto, deve-se tomar cuidado com seu uso em ambientes de produção.

Criando um módulo hello-world.js #

O Node fornece a interface module.exports para expor funções e variáveis ​​a outros arquivos. A maneira mais simples de fazer isso é exportar apenas um objeto (função ou variável), conforme mostrado no primeiro exemplo.

hello-world.js

module.exports = function(subject) {
    console.log('Hello ' + subject);
};

Se não quisermos que a exportação inteira seja um único objeto, podemos exportar funções e variáveis ​​como propriedades do objeto exports. Os três exemplos a seguir demonstram isso de maneiras ligeiramente diferentes:

  • hello-venus.js : a definição da função é feita separadamente e depois adicionada como uma propriedade de module.exports
  • hello-jupiter.js : as definições das funções são colocadas diretamente como o valor das propriedades de module.exports
  • hello-mars.js : a definição da função é declarada diretamente como uma propriedade de exports que é uma versão curta de module.exports

hello-venus.js

function hello(subject) {
    console.log('Venus says Hello ' + subject);
}

module.exports = {
    hello: hello
};

olá-jupiter.js

module.exports = {
    hello: function(subject) {
      console.log('Jupiter says hello ' + subject);
    },

    bye: function(subject) {
      console.log('Jupiter says goodbye ' + subject);
    }
};

olá-mars.js

exports.hello = function(subject) {
    console.log('Mars says Hello ' + subject);
};

Carregando módulo com nome de diretório

Temos um diretório chamado hello que inclui os seguintes arquivos:

index.js

// hello/index.js
module.exports = function(){
    console.log('Hej');
};

main.js

// hello/main.js
// We can include the other files we've defined by using the `require()` method
var hw = require('./hello-world.js'),
    hm = require('./hello-mars.js'),
    hv = require('./hello-venus.js'),
    hj = require('./hello-jupiter.js'),
    hu = require('./index.js');

// Because we assigned our function to the entire `module.exports` object, we
// can use it directly
hw('World!'); // outputs "Hello World!"

// In this case, we assigned our function to the `hello` property of exports, so we must
// use that here too
hm.hello('Solar System!'); // outputs "Mars says Hello Solar System!"

// The result of assigning module.exports at once is the same as in hello-world.js
hv.hello('Milky Way!'); // outputs "Venus says Hello Milky Way!"

hj.hello('Universe!'); //  outputs "Jupiter says hello Universe!"
hj.bye('Universe!'); // outputs "Jupiter says goodbye Universe!"

hu(); //output 'hej'

Carregando e usando um módulo #

Um módulo pode ser “importado” ou “exigido” pela função require(). Por exemplo, para carregar o módulo http fornecido com o Node.js, o seguinte pode ser usado:

const http = require('http');

Além dos módulos que são enviados com o runtime, você também pode exigir módulos que você instalou do npm, como express. Se você já instalou o express em seu sistema via npm install express, você pode simplesmente escrever:

const express = require('express');

Você também pode incluir módulos que você mesmo escreveu como parte de seu aplicativo. Nesse caso, para incluir um arquivo chamado lib.js no mesmo diretório do arquivo atual:

const mylib = require('./lib');

Observe que você pode omitir a extensão, e .js será assumido. Depois de carregar um módulo, a variável é preenchida com um objeto que contém os métodos e propriedades publicados do arquivo necessário. Um exemplo completo:

const http = require('http');

// The `http` module has the property `STATUS_CODES`
console.log(http.STATUS_CODES[404]); // outputs 'Not Found'

// Also contains `createServer()`
http.createServer(function(req, res) {
  res.writeHead(200, {'Content-Type': 'text/html'});
  res.write('<html><body>Module Test</body></html>');
  res.end();
}).listen(80);

Cada módulo injetado apenas uma vez #

O NodeJS executa o módulo apenas na primeira vez que você precisar dele. Quaisquer outras funções require reutilizarão o mesmo objeto, não executando o código no módulo outra vez. Além disso, o Node armazena em cache os módulos na primeira vez que são carregados usando require. Isso reduz o número de leituras de arquivos e ajuda a acelerar o aplicativo.

myModule.js

console.log(123) ; 
exports.var1 = 4 ;

index.js

var a=require('./myModule') ; // Output 123
var b=require('./myModule') ; // No output
console.log(a.var1) ; // Output 4
console.log(b.var1) ; // Output 4
a.var2 = 5 ; 
console.log(b.var2) ; // Output 5

Pasta como um módulo #

Os módulos podem ser divididos em vários arquivos .js na mesma pasta. Um exemplo em uma pasta my_module:

function_one.js

module.exports = function() {
  return 1;
}

function_two.js

module.exports = function() {
  return 2;
}

index.js

exports.f_one = require('./function_one.js');
exports.f_two = require('./function_two.js');

Um módulo como este é usado referindo-se a ele pelo nome da pasta:

var split_module = require('./my_module');

Por favor, note que se você exigiu omitindo ./ ou qualquer indicação de um caminho para uma pasta do argumento da função require, o Node tentará carregar um módulo da pasta node_modules.

Alternativamente, você pode criar na mesma pasta um arquivo package.json com este conteúdo:

{
    "name": "my_module",
    "main": "./your_main_entry_point.js"
}

Dessa forma, você não precisa nomear o arquivo do módulo principal como “index”.

Invalidando o cache do módulo #

No desenvolvimento, você pode descobrir que usar require() no mesmo módulo várias vezes sempre retorna o mesmo módulo, mesmo que você tenha feito alterações nesse arquivo. Isso ocorre porque os módulos são armazenados em cache na primeira vez que são carregados e qualquer carregamento de módulo subsequente será carregado do cache.

Para contornar este problema, você terá que deletar a entrada no cache. Por exemplo, se você carregou um módulo:

var a = require('./a');

Você pode excluir a entrada de cache:

var rpath = require.resolve('./a.js');
delete require.cache[rpath];

E, em seguida, exija o módulo novamente:

var a = require('./a');

Observe que isso não é recomendado em produção porque o delete excluirá apenas a referência ao módulo carregado, não os dados carregados em si. O módulo não é coletado como lixo, portanto, o uso inadequado desse recurso pode causar vazamento de memória.

Construindo seus próprios módulos #

Você também pode fazer referência a um objeto para exportar publicamente e anexar métodos continuamente a esse objeto:

const auth = module.exports = {}
const config = require('../config')
const request = require('request')

auth.email = function (data, callback) {
  // Authenticate with an email address
}

auth.facebook = function (data, callback) {
  // Authenticate with a Facebook account
}

auth.twitter = function (data, callback) {
  // Authenticate with a Twitter account
}

auth.slack = function (data, callback) {
  // Authenticate with a Slack account
}

auth.stack_overflow = function (data, callback) {
  // Authenticate with a Stack Overflow account
}

Para usar qualquer um desses, basta solicitar o módulo como faria normalmente:

const auth = require('./auth')

module.exports = function (req, res, next) {
  auth.facebook(req.body, function (err, user) {
    if (err) return next(err)

    req.user = user
    next()
  })
}

Carregamento de módulo de node_modules #

Módulos podem ser required sem usar caminhos relativos colocando-os em um diretório especial chamado node_modules.

Por exemplo, para requerer um módulo chamado foo de um arquivo index.js, você pode usar a seguinte estrutura de diretórios:

index.js
 \- node_modules
  \- foo
   |- foo.js
   \- package.json

Os módulos devem ser colocados dentro de um diretório, junto com um arquivo package.json. O campo main do arquivo package.json deve apontar para o ponto de entrada do seu módulo – este é o arquivo que é importado quando os usuários fazem require('your-module'). O padrão main é index.js se não for fornecido. Alternativamente, você pode consultar os arquivos relativos ao seu módulo simplesmente anexando o caminho relativo à chamada require: require('your-module/path/to/file').

Os módulos também podem ser required dos diretórios node_modules na hierarquia do sistema de arquivos. Se tivermos a seguinte estrutura de diretórios:

my-project
\- node_modules
 |- foo   // the foo module
  \- ...
 \- baz   // the baz module
  \- node_modules
   \- bar   // the bar module

poderemos requerer o módulo foo de qualquer arquivo dentro de bar usando require('foo').

Observe que o nó corresponderá apenas ao módulo que estiver mais próximo do arquivo na hierarquia do sistema de arquivos, começando em (diretório/node_modules atual do arquivo). O nó corresponde aos diretórios dessa maneira até a raiz do sistema de arquivos.

Você pode instalar novos módulos do registro npm ou de outros registros npm, ou fazer o seu próprio.
