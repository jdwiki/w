Usando o gerenciador de versões do nó (nvm) #

Node Version Manager, também conhecido como nvm, é um script bash que simplifica o gerenciamento de várias versões do Node.js.

Para instalar o nvm, use o script de instalação fornecido:

$ curl -o- https://raw.githubusercontent.com/creationix/nvm/v0.31.3/install.sh | bash

Para o Windows existe um pacote nvm-windows com um instalador. Esta página GithHub contém os detalhes para instalar e usar o pacote nvm-windows.

Depois de instalar o nvm, execute “nvm on” na linha de comando. Isso permite que o nvm controle as versões do nó.

Nota: Você pode precisar reiniciar seu terminal para que ele reconheça o comando nvm recém-instalado.

Em seguida, instale a versão mais recente do Node:

$ nvm install node

Você também pode instalar uma versão específica do Node, passando as versões principal, secundária e/ou patch:

$ nvm install 6
$ nvm install 4.2

Para listar as versões disponíveis para instalação:

$ nvm ls-remote

Você pode então alternar as versões passando a versão da mesma forma que você faz ao instalar:

$ nvm use 5

Você pode definir uma versão específica do Node que você instalou para ser a versão padrão digitando:

$ nvm alias default 4.2

Para exibir uma lista de versões do Node que estão instaladas em sua máquina, digite:

$ nvm ls

Para usar versões de nós específicas do projeto, você pode salvar a versão no arquivo .nvmrc. Dessa forma, começar a trabalhar com outro projeto será menos propenso a erros depois de buscá-lo em seu repositório.

$ echo "4.2" > .nvmrc
$ nvm use
Found '/path/to/project/.nvmrc' with version <4.2>
Now using node v4.2 (npm v3.7.3)

Quando o Node é instalado via nvm, não precisamos usar o sudo para instalar os pacotes globais, pois eles são instalados na pasta home. Assim, npm i -g http-server funciona sem nenhum erro de permissão.

Instalando o Node.js no Mac usando o gerenciador de pacotes #

Homebrew #

Você pode instalar o Node.js usando o gerenciador de pacotes Homebrew.

Comece atualizando o brew:

brew update

Pode ser necessário alterar permissões ou caminhos. É melhor executar isso antes de continuar:

brew doctor

Em seguida, você pode instalar o Node.js executando:

brew install node

Depois que o Node.js estiver instalado, você poderá validar a versão instalada executando:

node -v

Macports #

Você também pode instalar o node.js por meio do Macports.

Primeiro, atualize-o para garantir que os pacotes mais recentes sejam referenciados:

sudo port selfupdate

Em seguida, instale o nodejs e o npm

sudo port install nodejs npm

Agora você pode executar o node através da CLI diretamente invocando node. Além disso, você pode verificar sua versão atual do nó com

node -v

Instalando o Node.js no Windows #

Instalação padrão

Todos os binários, instaladores e arquivos de origem do Node.js podem ser baixados aqui.

Você pode baixar apenas o runtime do node.exe ou usar o instalador do Windows (.msi), que também instalará o npm, o gerenciador de pacotes recomendado para Node.js, e configurará os caminhos.

Instalação pelo gerenciador de pacotes

Você também pode instalar pelo gerenciador de pacotes Chocolatey (Automação de gerenciamento de software).

# choco install nodejs.install

Mais informações sobre a versão atual, você pode encontrar no repositório choco aqui.

Instale o Node.js no Ubuntu #

Usando o gerenciador de pacotes apt #

sudo apt-get update    
sudo apt-get install nodejs
sudo apt-get install npm
sudo ln -s /usr/bin/nodejs /usr/bin/node

# the node & npm versions in apt are outdated. This is how you can update them:
sudo npm install -g npm
sudo npm install -g n
sudo n stable # (or lts, or a specific version)

Usando a última versão específica (por exemplo, LTS 6.x) diretamente do nodesource #

curl -sL https://deb.nodesource.com/setup_6.x | sudo -E bash -
apt-get install -y nodejs

Além disso, para a maneira correta de instalar módulos npm globais, defina o diretório pessoal para eles (elimina a necessidade de sudo e evita erros EACCES):

mkdir ~/.npm-global
echo "export PATH=~/.npm-global/bin:$PATH" >> ~/.profile
source ~/.profile
npm config set prefix '~/.npm-global'

Instalando Node.js com n #

Primeiro, existe um wrapper muito bom para configurar n em seu sistema. Apenas corra:

curl -L https://git.io/n-install | bash

para instalar n. Em seguida, instale os binários de várias maneiras:

Mais recentes

n mais recente

estábulo

n estável

lts

n lts

Qualquer outra versão

n <versão>

por exemplo. n 4.4.7

Se esta versão já estiver instalada, este comando ativará essa versão.

Mudando de versão

n por si só irá produzir uma lista de seleção de binários instalados. Use para cima e para baixo para encontrar o que deseja e Enter para ativá-lo.

Instale o Node.js da fonte com o gerenciador de pacotes APT #

Pré-requisitos

sudo apt-get install build-essential
sudo apt-get install python

[optional]
sudo apt-get install git

Obtenha o código-fonte e construa

cd ~
git clone https://github.com/nodejs/node.git

OU Para a versão mais recente do LTS Node.js 6.10.2

cd ~
wget https://nodejs.org/dist/v6.3.0/node-v6.10.2.tar.gz
tar -xzvf node-v6.10.2.tar.gz

Mude para o diretório de origem, como em cd ~/node-v6.10.2

./configure
make
sudo make install

Instalando o Node.js no Raspberry PI #

Para instalar a v6.x, atualize os pacotes

curl -sL https://deb.nodesource.com/setup_6.x | sudo -E bash -

Usando o gerenciador de pacotes apt

sudo apt-get install -y nodejs

Instalando com o Node Version Manager em Fish Shell com Oh My Fish! #

Node Version Manager (nvm) simplifica bastante o gerenciamento de versões do Node.js, sua instalação e elimina a necessidade de sudo ao lidar com pacotes (por exemplo, npm install ...). Fish Shell (fish) “é uma linha de comando inteligente e fácil de usar shell para OS X, Linux e o resto da família”, que é uma alternativa popular entre os programadores para shells comuns, como o bash. Por último, Oh My Fish (omf) permite personalizar e instalando pacotes dentro do Fish shell.

Este guia pressupõe que você já esteja usando Peixe como sua concha.

Instalar nvm

curl -o- https://raw.githubusercontent.com/creationix/nvm/v0.31.4/install.sh | bash

Install Oh My Fish

curl -L https://github.com/oh-my-fish/oh-my-fish/raw/master/bin/install | fish

(Observação: você será solicitado a reiniciar seu terminal neste momento. Vá em frente e faça isso agora.)

Instale plugin-nvm para Oh My Fish

Vamos instalar plugin-nvm via Oh My Fish para expor os recursos nvm dentro do Fish shell:

omf install nvm

Install Node.js with Node Version Manager

You are now ready to use nvm. You may install and use the version of Node.js of your liking. Some examples:

  • Install the most recent Node version: nvm install node
  • Install 6.3.1 specifically: nvm install 6.3.1
  • List installed verisons: nvm ls
  • Switch to a previously installed 4.3.1: nvm use 4.3.1

Final Notes

Remember again, that we no longer need sudo when dealing with Node.js using this method! Node versions, packages, and so on are installed in your home directory.

Install Node.js from source on Centos, RHEL and Fedora #

Prerequisites

  • git
  • clang and clang++ 3.4^ or gcc and g++ 4.8^
  • Python 2.6 or 2.7
  • GNU Make 3.81^

Get source

Node.js v6.x LTS

git clone -b v6.x https://github.com/nodejs/node.git

Node.js v7.x

git clone -b v7.x https://github.com/nodejs/node.git

Build

nó de cd
./configure
make -jX
su -c make install

X - the number of processor cores, greatly speeds up the build

Cleanup [Optional]

cd
nó rm -rf

Installing using MacOS X Installer #

You can find the installers on Node.js download page. Normally, Node.js recommends two versions of Node, the LTS version (long term support) and the current version (latest release). If you are new to Node, just go for the LTS and then click the Macintosh Installer button to download the package.

If you want to find other NodeJS releases, go here, choose your release then click download. From the download page, look for a file with extension .pkg.

Once you downloaded the file (with extension .pkg ofcourse), double click it to install. The installer packed with Node.js and npm, by default, the package will install both but you can customize which one to install by clicking the customize button in the Installation Type step. Other than that, just follow the installation instructions, it’s pretty straightforward.

Check if Node is installed #

Open terminal (if you don’t know how to open your terminal, look at this wikihow). In the terminal type node --version then enter. Your terminal will look like this if Node is installed:

$ node --version
v7.2.1

The v7.2.1 is your Node.js version, if you receive the message command not found: node instead of that, then it’s mean there is a problem with your installation.
