Acessando variáveis ​​de ambiente #

A propriedade process.env retorna um objeto contendo o ambiente do usuário.

Ele retorna um objeto como este:

{
  TERM: 'xterm-256color',
  SHELL: '/usr/local/bin/bash',
  USER: 'maciej',
  PATH: '~/.bin/:/usr/bin:/bin:/usr/sbin:/sbin:/usr/local/bin',
  PWD: '/Users/maciej',
  EDITOR: 'vim',
  SHLVL: '1',
  HOME: '/Users/maciej',
  LOGNAME: 'maciej',
  _: '/usr/local/bin/node'
}

process.env.HOME // '/Users/maciej'

Se você definir a variável de ambiente FOO para foobar, ela será acessível com:

process.env.FOO // 'foobar'

argumentos de linha de comando process.argv #

process.argv é uma matriz que contém os argumentos da linha de comando. O primeiro elemento será node, o segundo elemento será o nome do arquivo JavaScript. Os próximos elementos serão quaisquer argumentos de linha de comando adicionais.

Exemplo de código:

Soma de saída de todos os argumentos da linha de comando

index.js

var sum = 0;
for (i = 2; i < process.argv.length; i++) {
    sum += Number(process.argv[i]);
}

console.log(sum);

Exemplo de uso:

node index.js 2 5 6 7

A saída será 20

Uma breve explicação do código:

Aqui no loop for for (i = 2; i < process.argv.length; i++) o loop começa com 2 porque os dois primeiros elementos no array process.argv sempre é ['path/to/node. exe', 'caminho/para/js/arquivo', ...]

Convertendo para o número Number(process.argv[i]) porque os elementos no array process.argv sempre são string

Usando diferentes propriedades/configurações para diferentes ambientes como dev, qa, staging etc. #

Aplicativos de grande escala geralmente precisam de propriedades diferentes ao serem executados em ambientes diferentes. podemos conseguir isso passando argumentos para o aplicativo NodeJs e usando o mesmo argumento no processo do nó para carregar o arquivo de propriedades do ambiente específico.

Suponha que temos dois arquivos de propriedades para ambientes diferentes.


  • dev.json

      {
      PORT : 3000,
      DB : {
          host : "localhost",
          user : "bob",
          password : "12345"
      }
  }

  • qa.json

      {
          PORT : 3001,
          DB : {
              host : "where_db_is_hosted",
              user : "bob",
              password : "54321"
          }
  }

O código a seguir no aplicativo exportará o respectivo arquivo de propriedade que queremos usar.

Suponha que o código esteja em environment.js

process.argv.forEach(function (val, index, array) {
    var arg = val.split("=");
    if (arg.length > 0) {
        if (arg[0] === 'env') {
            var env = require('./' + arg[1] + '.json');
            module.exports = env;
        }
    }
});

Damos argumentos para o aplicativo como a seguir

node app.js env=dev

se estivermos usando o gerenciador de processos como * para sempre * do que tão simples quanto

forever start app.js env=dev

Como usar o arquivo de configuração

 var env= require("environment.js");

Carregando propriedades do ambiente de um “arquivo de propriedades” #

  • Instale o leitor de propriedades:

    npm install properties-reader –save

  • Crie um env de diretório para armazenar seus arquivos de propriedades:

    mkdir env

  • Crie environments.js:

    process.argv.forEach(function (val, index, array) { var arg = val.split("="); if (arg.length > 0) { if (arg[0] === ’env’) { var env = require(’./env/’ + arg[1] + ‘.properties’); module.exports = env; } } });

  • Exemplo de arquivo de propriedades development.properties:

    Dev properties #

    [main]

    Application port to run the node server #

    app.port=8080

    [database]

    Database connection to mysql #

    mysql.host=localhost mysql.port=2500 …

  • Exemplo de uso das propriedades carregadas:

    var enviorment = require(’./environments’); var PropertiesReader = require(‘properties-reader’); var properties = new PropertiesReader(enviorment);

    var someVal = properties.get(‘main.app.port’);

  • Iniciando o servidor expresso

    npm start env=development

ou

npm start env=production
