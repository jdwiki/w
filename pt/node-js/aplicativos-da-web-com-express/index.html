O Express é uma estrutura de aplicativo da Web Node.js mínima e flexível, que fornece um conjunto robusto de recursos para a criação de aplicativos da Web.

O site oficial do Express é expressjs.com. A fonte pode ser encontrada no GitHub.

Sintaxe #

  • app.get(caminho [, middleware], callback[, callback…])
  • app.put(caminho [, middleware], callback[, callback…])
  • app.post(caminho [, middleware], callback[, callback…])
  • app[‘delete’](caminho [, middleware], callback[, callback…])
  • app.use(caminho [, middleware], callback[, callback…])
  • app.use(retorno de chamada)

Parâmetros #

Parâmetro Detalhes
caminho Especifica a parte do caminho ou a URL que o retorno de chamada fornecido manipulará.
middleware Uma ou mais funções que serão chamadas antes do retorno de chamada. Essencialmente, um encadeamento de várias funções callback. Útil para tratamento mais específico, por exemplo, autorização ou tratamento de erros.
retorno de chamada Uma função que será usada para lidar com solicitações para o path especificado. Ele será chamado como callback(request, response, next), onde request, response e ​​next são descritos abaixo.
callback pedido Um objeto encapsulando detalhes sobre a solicitação HTTP para a qual o retorno de chamada está sendo chamado.
resposta Um objeto que é usado para especificar como o servidor deve responder à solicitação.
próximo Um retorno de chamada que passa o controle para a próxima rota correspondente. Ele aceita um objeto de erro opcional.

Roteamento básico #

Primeiro crie um aplicativo expresso:

const express = require('express');
const app = express();

Então você pode definir rotas como esta:

app.get('/someUri', function (req, res, next) {})

Essa estrutura funciona para todos os métodos HTTP e espera um caminho como o primeiro argumento e um manipulador para esse caminho, que recebe os objetos de solicitação e resposta. Então, para os métodos HTTP básicos, essas são as rotas

// GET www.domain.com/myPath
app.get('/myPath', function (req, res, next) {})

// POST www.domain.com/myPath
app.post('/myPath', function (req, res, next) {})

// PUT www.domain.com/myPath
app.put('/myPath', function (req, res, next) {})

// DELETE www.domain.com/myPath
app.delete('/myPath', function (req, res, next) {})

Você pode verificar a lista completa de verbos suportados aqui. Se você deseja definir o mesmo comportamento para uma rota e todos os métodos HTTP, você pode usar:

app.all('/myPath', function (req, res, next) {})

ou

app.use('/myPath', function (req, res, next) {})

ou

app.use('*', function (req, res, next) {})

// * wildcard will route for all paths

Você pode encadear suas definições de rota para um único caminho

app.route('/myPath')
  .get(function (req, res, next) {})
  .post(function (req, res, next) {})
  .put(function (req, res, next) {})

Você também pode adicionar funções a qualquer método HTTP. Eles serão executados antes do retorno de chamada final e usarão os parâmetros (req, res, next) como argumentos.

// GET www.domain.com/myPath
app.get('/myPath', myFunction, function (req, res, next) {})

Seus retornos de chamada finais podem ser armazenados em um arquivo externo para evitar colocar muito código em um arquivo:

// other.js
exports.doSomething = function(req, res, next) {/* do some stuff */};

E então no arquivo que contém suas rotas:

const other = require('./other.js');
app.get('/someUri', myFunction, other.doSomething);

Isso deixará seu código muito mais limpo.

Começando #

Você primeiro precisará criar um diretório, acessá-lo em seu shell e instalar o Express usando npm executando npm install express --save

Crie um arquivo e nomeie-o como app.js e adicione o seguinte código que cria um novo servidor Express e adiciona um endpoint a ele (/ping) com o método app.get:

const express = require('express');

const app = express();

app.get('/ping', (request, response) => {
    response.send('pong');
});

app.listen(8080, 'localhost');

Para executar seu script, use o seguinte comando em seu shell:

> node app.js

Seu aplicativo aceitará conexões na porta 8080 do host local. Se o argumento do nome do host para app.listen for omitido, o servidor aceitará conexões no endereço IP da máquina, bem como no host local. Se o valor da porta for 0, o sistema operacional atribuirá uma porta disponível.

Assim que seu script estiver em execução, você poderá testá-lo em um shell para confirmar que obteve a resposta esperada, “pong”, do servidor:

> curl http://localhost:8080/ping
pong

Você também pode abrir um navegador da Web, navegar até a url http://localhost:8080/ping para visualizar a saída

Aplicativo expresso modular #

Para tornar o aplicativo da Web expresso modular, use fábricas de roteadores:

Módulo:

// greet.js
const express = require('express');

module.exports = function(options = {}) { // Router factory
    const router = express.Router();

    router.get('/greet', (req, res, next) => {
        res.end(options.greeting);
    });

    return router;
};

Inscrição:

// app.js
const express = require('express');
const greetMiddleware = require('./greet.js');

express()
    .use('/api/v1/', greetMiddleware({ greeting:'Hello world' }))
    .listen(8080);

Isso tornará seu aplicativo modular, personalizável e seu código reutilizável.

Ao acessar http://<hostname>:8080/api/v1/greet a saída será Hello world

Exemplo mais complicado #

Exemplo com serviços que mostra vantagens de fábrica de middleware.

Módulo:

// greet.js
const express = require('express');

module.exports = function(options = {}) { // Router factory
    const router = express.Router();
    // Get controller
    const {service} = options;

    router.get('/greet', (req, res, next) => {
        res.end(
            service.createGreeting(req.query.name || 'Stranger')
        );
    });

    return router;
};

Inscrição:

// app.js
const express = require('express');
const greetMiddleware = require('./greet.js');

class GreetingService {
    constructor(greeting = 'Hello') {
        this.greeting = greeting;
    }

    createGreeting(name) {
        return `${this.greeting}, ${name}!`;
    }
}

express()
    .use('/api/v1/service1', greetMiddleware({
        service: new GreetingService('Hello'),
    }))
    .use('/api/v1/service2', greetMiddleware({
        service: new GreetingService('Hi'),
    }))
    .listen(8080);

Ao acessar http://<hostname>:8080/api/v1/service1/greet?name=World a saída será Hello, World e acessar http://<hostname>:8080/api/v1 /service2/greet?name=World a saída será Hi, World.

Usando um mecanismo de modelo #

Usando um mecanismo de modelo #

O código a seguir configurará o Jade como mecanismo de modelo. (Nota: Jade foi renomeado para pug em dezembro de 2015.)

const express = require('express');  //Imports the express module
const app = express();  //Creates an instance of the express module

const PORT = 3000; //Randomly chosen port

app.set('view engine','jade'); //Sets jade as the View Engine / Template Engine
app.set('views','src/views'); //Sets the directory where all the views (.jade files) are stored.

//Creates a Root Route
app.get('/',function(req, res){
    res.render('index');  //renders the index.jade file into html and returns as a response. The render function optionally takes the data to pass to the view.
});

//Starts the Express server with a callback
app.listen(PORT, function(err) {
    if (!err) {
        console.log('Server is running at port', PORT);
    } else {
        console.log(JSON.stringify(err));
    }
});

Da mesma forma, outros Template Engines também podem ser usados, como Handlebars(hbs) ou ejs. Lembre-se de npm install o Template Engine também. Para Handlebars usamos o pacote hbs, para Jade temos um pacote jade e para EJS, temos um pacote ejs.

Exemplo de modelo EJS #

Com o EJS (como outros modelos expressos), você pode executar o código do servidor e acessar as variáveis ​​do servidor a partir do seu HTML.
No EJS é feito usando “<%” como tag inicial e “%>” como tag final, variáveis ​​passadas como parâmetros de renderização podem ser acessadas usando <%=var_name%>
Por exemplo, se você tiver uma matriz de suprimentos em seu código de servidor
você pode fazer um loop sobre ele usando

<h1><%= title %></h1>
   <ul>
<% for(var i=0; i<supplies.length; i++) { %>
    <li>
        <a href='supplies/<%= supplies[i] %>'>
            <%= supplies[i] %>
        </a>
    </li>
<% } %>

Servindo arquivos estáticos #

Ao construir um servidor web com o Express, muitas vezes é necessário servir uma combinação de conteúdo dinâmico e arquivos estáticos.

Por exemplo, você pode ter index.html e script.js que são arquivos estáticos mantidos no sistema de arquivos.

É comum usar a pasta chamada ‘pública’ para ter arquivos estáticos. Nesse caso, a estrutura de pastas pode se parecer com:

project root
├── server.js
├── package.json 
└── public
    ├── index.html
    └── script.js

Veja como configurar o Express para servir arquivos estáticos:

const express = require('express');
const app = express();

app.use(express.static('public'));

Nota: uma vez que a pasta estiver configurada, index.html, script.js e todos os arquivos da pasta “public” estarão disponíveis no caminho raiz (você não deve especificar /public/ na url). Isso ocorre porque o express procura os arquivos relativos à pasta estática configurada. Você pode especificar prefixo de caminho virtual conforme mostrado abaixo:

app.use('/static', express.static('public'));

disponibilizará os recursos sob o prefixo /static/.

Várias pastas #

É possível definir várias pastas ao mesmo tempo:

app.use(express.static('public'));
app.use(express.static('images'));
app.use(express.static('files'));

Ao servir os recursos, o Express examinará a pasta na ordem de definição. No caso de arquivos com o mesmo nome, será servido o que estiver na primeira pasta correspondente.

Obtendo informações da solicitação #

Para obter informações da url solicitante (observe que req é o objeto de solicitação na função manipuladora de rotas). Considere esta definição de rota /settings/:user_id e este exemplo específico /settings/32135?field=name

// get the full path
req.originalUrl // => /settings/32135?field=name

// get the user_id param
req.params.user_id // => 32135     

// get the query value of the field
req.query.field // => 'name'

Você também pode obter os cabeçalhos da solicitação, como este

req.get('Content-Type')
// "text/plain"

Para simplificar a obtenção de outras informações, você pode usar middlewares. Por exemplo, para obter as informações do corpo da solicitação, você pode usar o middleware body-parser, que transformará o corpo bruto da solicitação em um formato utilizável.

var app = require('express')();
var bodyParser = require('body-parser');

app.use(bodyParser.json()); // for parsing application/json
app.use(bodyParser.urlencoded({ extended: true })); // for parsing application/x-www-form-urlencoded

Agora suponha um pedido como este

PUT /settings/32135
{
  "name": "Peter"
}

Você pode acessar o nome postado assim

req.body.name
// "Peter"

De forma semelhante, você pode acessar os cookies a partir da solicitação, também precisa de um middleware como cookie-parser

req.cookies.name

Manipulação de erros #

Tratamento básico de erros

Por padrão, o Express irá procurar por uma visão ’erro’ no diretório /views para renderizar. Simplesmente crie a view ’error’ e coloque-a no diretório views para lidar com erros. Os erros são gravados com a mensagem de erro, status e rastreamento de pilha, por exemplo:

views/error.pug

html
  body
      h1= message
      h2= error.status
      p= error.stack

Tratamento avançado de erros

Defina suas funções de middleware de tratamento de erros no final da pilha de funções de middleware. Estes têm quatro argumentos em vez de três (err, req, res, next) por exemplo:

app.js

// catch 404 and forward to error handler
app.use(function(req, res, next) {
    var err = new Error('Not Found');
    err.status = 404;

    //pass error to the next matching route.
    next(err);
});

// handle error, print stacktrace
app.use(function(err, req, res, next) {
    res.status(err.status || 500);

    res.render('error', {
        message: err.message,
        error: err
    });
});

Você pode definir várias funções de middleware de tratamento de erros, assim como faria com funções de middleware regulares.

Adicionando middleware #

As funções de middleware são funções que têm acesso ao objeto de solicitação (req), ao objeto de resposta (res) e à próxima função de middleware no ciclo de solicitação-resposta do aplicativo.

As funções de middleware podem executar qualquer código, fazer alterações nos objetos res e req, encerrar o ciclo de resposta e chamar o próximo middleware.

Um exemplo muito comum de middleware é o módulo cors. Para adicionar suporte ao CORS, basta instalá-lo, requisitá-lo e colocar esta linha:

app.use(cors());

antes de quaisquer roteadores ou funções de roteamento.

Hook: Como executar código antes de qualquer req e depois de qualquer res #

app.use() e middleware podem ser usados ​​para “antes” e uma combinação dos eventos close e finish pode ser usada para “depois”.

app.use(function (req, res, next) {
    function afterResponse() {
        res.removeListener('finish', afterResponse);
        res.removeListener('close', afterResponse);

        // actions after response
    }
    res.on('finish', afterResponse);
    res.on('close', afterResponse);

    // action before request
    // eventually calling `next()`
    next();
});
...
app.use(app.router);

Um exemplo disso é o middleware logger, que será anexado ao log após a resposta por padrão.

Apenas certifique-se de que este “middleware” seja usado antes de app.router, pois a ordem importa.

Tratamento de erros no Express #

No Express, você pode definir um manipulador de erros unificado para lidar com erros ocorridos no aplicativo. Defina então o manipulador no final de todas as rotas e código lógico.

Exemplo

        return res.send('Valid Name');

        next(new Error('Not valid name'));    //pass to error handler
    }
});

//error handler
app.use(function(err, req, res, next){

    return res.status(500).send('Internal Server Occured');
});

Rotas nomeadas no estilo Django #

Um grande problema é que rotas nomeadas valiosas não são suportadas pelo Express pronto para uso. A solução é instalar o pacote de terceiros suportado, por exemplo express-reverse:

Usando middleware e o próximo retorno de chamada #

O Express passa um retorno de chamada next para cada manipulador de rota e função de middleware que pode ser usado para quebrar a lógica de rotas únicas em vários manipuladores. Chamar next() sem argumentos diz ao express para continuar para o próximo middleware ou manipulador de rota correspondente. Chamar next(err) com um erro acionará qualquer middleware de tratamento de erros. Chamar next('route') ignorará qualquer middleware subsequente na rota atual e pulará para a próxima rota correspondente. Isso permite que a lógica de domínio seja desacoplada em componentes reutilizáveis ​​que são autocontidos, mais simples de testar e mais fáceis de manter e alterar.

Várias rotas correspondentes

As solicitações para /api/foo ou para /api/bar executarão o manipulador inicial para procurar o membro e, em seguida, passar o controle para o manipulador real para cada rota.

app.get('/api', function(req, res, next) {
  // Both /api/foo and /api/bar will run this
  lookupMember(function(err, member) {
    if (err) return next(err);
    req.member = member;
    next();
  });
});

app.get('/api/foo', function(req, res, next) {
  // Only /api/foo will run this
  doSomethingWithMember(req.member);
});

app.get('/api/bar', function(req, res, next) {
  // Only /api/bar will run this
  doSomethingDifferentWithMember(req.member);
});

Gerenciador de erros

Manipuladores de erro são middleware com a assinatura function(err, req, res, next). Eles podem ser configurados por rota (por exemplo, app.get('/foo', function(err, req, res, next)), mas normalmente, um único manipulador de erro que renderiza uma página de erro é suficiente.

app.get('/foo', function(req, res, next) {
  doSomethingAsync(function(err, data) {
    if (err) return next(err);
    renderPage(data);
  });
});

// In the case that doSomethingAsync return an error, this special
// error handler middleware will be called with the error as the 
// first parameter.
app.use(function(err, req, res, next) {
  renderErrorPage(err);
});

Middleware

Cada uma das funções acima é na verdade uma função de middleware que é executada sempre que uma solicitação corresponde à rota definida, mas qualquer número de funções de middleware pode ser definido em uma única rota. Isso permite que o middleware seja definido em arquivos separados e a lógica comum seja reutilizada em várias rotas.

app.get('/bananas', function(req, res, next) {
  getMember(function(err, member) {
    if (err) return next(err);
    // If there's no member, don't try to look
    // up data. Just go render the page now.
    if (!member) return next('route');
    // Otherwise, call the next middleware and fetch
    // the member's data.
    req.member = member;
    next();
  });
}, function(req, res, next) {
  getMemberData(req.member, function(err, data) {
    if (err) return next(err);
    // If this member has no data, don't bother
    // parsing it. Just go render the page now.
    if (!data) return next('route');
    // Otherwise, call the next middleware and parse
    // the member's data. THEN render the page.
    req.member.data = data;
    next();
  });
}, function(req, res, next) {
  req.member.parsedData = parseMemberData(req.member.data);
  next();
});

app.get('/bananas', function(req, res, next) {
  renderBananas(req.member);
});

Neste exemplo, cada função de middleware estaria em seu próprio arquivo ou em uma variável em outro lugar no arquivo para que pudesse ser reutilizada em outras rotas.

Lidando com solicitações POST #

Assim como você lida com solicitações de obtenção no Express com o método app.get, você pode usar o método app.post para lidar com solicitações de postagem.

Mas antes que você possa lidar com solicitações POST, você precisará usar o middleware body-parser. Ele simplesmente analisa o corpo de POST, PUT, DELETE e outros pedidos.

O middleware Body-Parser analisa o corpo da solicitação e o transforma em um objeto disponível em req.body

var bodyParser = require('body-parser');

const express = require('express');

const app = express();

// Parses the body for POST, PUT, DELETE, etc.
app.use(bodyParser.json());

app.use(bodyParser.urlencoded({ extended: true }));

app.post('/post-data-here', function(req, res, next){

    console.log(req.body); // req.body contains the parsed body of the request.

});

app.listen(8080, 'localhost');
