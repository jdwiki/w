Olá mundo servidor HTTP #

Primeiro, instale o Node.js para sua plataforma.

Neste exemplo, criaremos um servidor HTTP escutando na porta 1337, que envia Hello, World! para o navegador. Observe que, em vez de usar a porta 1337, você pode usar qualquer número de porta de sua escolha que não esteja sendo usado por nenhum outro serviço.

O módulo http é um [core module] do Node.js(https://www.wikiod.com/pt/node-js/introducao-ao-nodejs#Módulos principais ) (um módulo incluído na fonte do Node.js, que não requer a instalação de recursos adicionais). O módulo http fornece a funcionalidade para criar um servidor HTTP usando o método http.createServer(). To create the application, create a file containing the seguinte código JavaScript.

const http = require('http'); // Loads the http module

http.createServer((request, response) => {

    // 1. Tell the browser everything is OK (Status code 200), and the data is in plain text
    response.writeHead(200, {
        'Content-Type': 'text/plain'
    });

    // 2. Write the announced text to the body of the page
    response.write('Hello, World!\n');

    // 3. Tell the server that all of the response headers and body have been sent
    response.end();

}).listen(1337); // 4. Tells the server what port to be on

Salve o arquivo com qualquer nome de arquivo. Neste caso, se o nomearmos hello.js, podemos executar a aplicação indo para o diretório em que o arquivo está e usando o seguinte comando:

node hello.js

O servidor criado pode ser acessado com a URL http://localhost:1337 ou http://127.0.0.1:1337 no navegador.

Uma página web simples aparecerá com um “Hello, World!” texto na parte superior, conforme mostrado na captura de tela abaixo.

Linha de comando Hello World #

O Node.js também pode ser usado para criar utilitários de linha de comando. O exemplo abaixo lê o primeiro argumento da linha de comando e imprime uma mensagem Hello.

Para executar este código em um sistema Unix:

  1. Crie um novo arquivo e cole o código abaixo. O nome do arquivo é irrelevante.
  2. Torne este arquivo executável com chmod 700 FILE_NAME
  3. Execute o aplicativo com ./APP_NAME David

No Windows você faz o passo 1 e executa com node APP_NAME David

#!/usr/bin/env node

'use strict';

/*
    The command line arguments are stored in the `process.argv` array, 
    which has the following structure:
    [0] The path of the executable that started the Node.js process
    [1] The path to this application
    [2-n] the command line arguments

    Example: [ '/bin/node', '/path/to/yourscript', 'arg1', 'arg2', ... ]
    src: https://nodejs.org/api/process.html#process_process_argv
 */

// Store the first argument as username.
var username = process.argv[2];

// Check if the username hasn't been provided.
if (!username) {

    // Extract the filename
    var appName = process.argv[1].split(require('path').sep).pop();

    //  Give the user an example on how to use the app.
    console.error('Missing argument! Example: %s YOUR_NAME', appName);

    // Exit the app (success: 0, error: 1). 
    // An error will stop the execution chain. For example:
    //   ./app.js && ls       -> won't execute ls
    //   ./app.js David && ls -> will execute ls
    process.exit(1);
}

// Print the message to the console.
console.log('Hello %s!', username);

Olá Mundo com Expresso #

O exemplo a seguir usa o Express para criar um servidor HTTP escutando na porta 3000, que responde com “Hello, World!”. Express é uma estrutura da Web comumente usada que é útil para criar APIs HTTP.

Primeiro, crie uma nova pasta, por exemplo. meuAplicativo. Vá para myApp e crie um novo arquivo JavaScript contendo o seguinte código (vamos chamá-lo de hello.js por exemplo). Em seguida, instale o módulo express usando npm install --save express na linha de comando. Consulte esta documentação para mais informações sobre como instalar pacotes.

// Import the top-level function of express
const express = require('express');

// Creates an Express application using the top-level function
const app = express();

// Define port number as 3000
const port = 3000;

// Routes HTTP GET requests to the specified path "/" with the specified callback function
app.get('/', function(request, response) {
  response.send('Hello, World!');
});

// Make the app listen on port 3000
app.listen(port, function() {
  console.log('Server listening on http://localhost:' + port);
});

Na linha de comando, execute o seguinte comando:

node hello.js

Abra seu navegador e navegue até http://localhost:3000 ou http://127.0.0.1:3000 para ver a resposta.

Para obter mais informações sobre o framework Express, você pode verificar a seção Web Apps With Express

1: https://www.wikiod.com/pt/node-js/npm#Instalando pacotes 2: https://www.wikiod.com/pt/node-js/aplicativos-da-web-com-express

Instalando e executando o Node.js #

Para começar, instale o Node.js em seu computador de desenvolvimento.

Windows: Navegue até a página de download e baixe/execute o instalador.

Mac: Navegue até a página de download e baixe/execute o instalador. Alternativamente, você pode instalar o Node via Homebrew usando brew install node. Homebrew é um gerenciador de pacotes de linha de comando para Macintosh, e mais informações sobre ele podem ser encontradas no site Homebrew.

Linux: Siga as instruções para sua distribuição na página de instalação da linha de comando.

Executando um programa de nó #

Para executar um programa Node.js, basta executar node app.js ou nodejs app.js, onde app.js é o nome do arquivo do código-fonte do aplicativo do nó. Você não precisa incluir o sufixo .js para que o Node encontre o script que deseja executar.

Alternativamente, em sistemas operacionais baseados em UNIX, um programa Node pode ser executado como um script de terminal. Para fazer isso, ele precisa começar com um shebang apontando para o interpretador Node, como #!/usr/bin/env node. O arquivo também deve ser definido como executável, o que pode ser feito usando chmod. Agora o script pode ser executado diretamente a partir da linha de comando.

Depurando seu aplicativo NodeJS #

Você pode usar o node-inspector. Execute este comando para instalá-lo via npm:

npm install -g node-inspector

Então você pode depurar seu aplicativo usando

node-debug app.js

O repositório do Github pode ser encontrado aqui: https://github.com/node-inspector/node-inspector

Depurando nativamente #

Você também pode depurar o node.js nativamente iniciando-o assim:

node debug your-script.js

Para interromper seu depurador exatamente em uma linha de código desejada, use isto:

debugger;

Para obter mais informações, consulte aqui.

No node.js 8, use o seguinte comando:

node --inspect-brk your-script.js

Em seguida, abra about://inspect em uma versão recente do Google Chrome e selecione seu script Node para obter a experiência de depuração do DevTools do Chrome.

Roteamento básico Hello World #

Depois de entender como criar um HTTP Server com node, é importante entender como fazer com que ele “faça” coisas com base no caminho para o qual um usuário navegou. Esse fenômeno é chamado de “roteamento”.

O exemplo mais básico disso seria verificar if (request.url === 'some/path/here'), e então chamar uma função que responde com um novo arquivo.

Um exemplo disso pode ser visto aqui:

const http = require('http');

function index (request, response) {
    response.writeHead(200);
    response.end('Hello, World!');
}

http.createServer(function (request, response) {
    
    if (request.url === '/') {
        return index(request, response);
    }

    response.writeHead(404);
    response.end(http.STATUS_CODES[404]);

}).listen(1337);

Se você continuar a definir suas “rotas” assim, você acabará com uma enorme função de retorno de chamada, e não queremos uma bagunça gigante como essa, então vamos ver se podemos limpar isso.

Primeiro, vamos armazenar todas as nossas rotas em um objeto:

var routes = {
    '/': function index (request, response) {
        response.writeHead(200);
        response.end('Hello, World!');
    },
    '/foo': function foo (request, response) {
        response.writeHead(200);
        response.end('You are now viewing "foo"');
    }
}

Agora que armazenamos 2 rotas em um objeto, podemos verificá-las em nosso retorno de chamada principal:

http.createServer(function (request, response) {
    
    if (request.url in routes) {
        return routes[request.url](request, response);
    }

    response.writeHead(404);
    response.end(http.STATUS_CODES[404]);

}).listen(1337);

Agora, toda vez que você tentar navegar em seu site, ele verificará a existência desse caminho em suas rotas e chamará a respectiva função. Se nenhuma rota for encontrada, o servidor responderá com um 404 (Not Found).

E aí está – o roteamento com a API do servidor HTTP é muito simples.

Olá Mundo no REPL #

Quando chamado sem argumentos, o Node.js inicia um REPL (Read-Eval-Print-Loop) também conhecido como “Node shell”.

Em um prompt de comando digite node.

$ node
>

No prompt do shell do Node > digite “Hello World!”

$ node
> "Hello World!"
'Hello World!'

Como implantar seu aplicativo online #

Quando você implanta seu aplicativo em um ambiente hospedado (específico do Node.js), esse ambiente geralmente oferece uma variável de ambiente PORT que você pode usar para executar seu servidor. Alterar o número da porta para process.env.PORT permite que você acesse o aplicativo.

Por exemplo,

http.createServer(function(request, response) {
   // your server code
}).listen(process.env.PORT);

Além disso, se você quiser acessar isso offline durante a depuração, poderá usar isto:

http.createServer(function(request, response) {
  // your server code
}).listen(process.env.PORT || 3000);

onde 3000 é o número da porta offline.

Todos os módulos principais em resumo #

assert

O módulo assert fornece um conjunto simples de testes de asserção que podem ser usados ​​para invariantes de teste.

buffer

Antes da introdução do TypedArray no ECMAScript 2015 (ES6), o A linguagem JavaScript não tinha mecanismo para ler ou manipular fluxos de dados binários. A classe Buffer foi introduzida como parte do Node.js API para possibilitar a interação com fluxos de octetos no contexto das coisas como fluxos TCP e operações do sistema de arquivos.

Agora que TypedArray foi adicionado no ES6, a classe Buffer implementa o Uin t8Array API de uma maneira mais otimizada e adequada para Node.js' casos de uso.

c/c++_addons

Os complementos Node.js são objetos compartilhados vinculados dinamicamente, escritos em C ou C++, que pode ser carregado no Node.js usando a função require() , e usado como se fossem um módulo Node.js comum. Eles são usados ​​principalmente para fornecer uma interface entre JavaScript executado em bibliotecas Node.js e C/C++.

child_process

O módulo child_process fornece a capacidade de gerar processos filhos em de maneira semelhante, mas não idêntica, ao popen(3).

cluster

Uma única instância do Node.js é executada em um único encadeamento. Para aproveitar os sistemas multinúcleo, o usuário às vezes desejará iniciar um cluster de Node.js processos para lidar com a carga. O módulo de cluster permite que você crie facilmente processos filhos que todas as portas do servidor de compartilhamento.

console

O módulo console fornece um console de depuração simples semelhante ao Mecanismo de console JavaScript fornecido por navegadores da web.

crypto

O módulo crypto fornece funcionalidade criptográfica que inclui um conjunto de wrappers para as funções de hash, HMAC, cifrar, decifrar, assinar e verificar do OpenSSL.

deprecated_apis

O Node.js pode descontinuar APIs quando: (a) o uso da API for considerado inseguro, (b) uma API alternativa melhorada foi disponibilizada, ou (c) alterações importantes na API são esperadas em uma versão principal futura.

dns

O módulo dns contém funções pertencentes a duas categorias diferentes:

  1. Funções que usam os recursos do sistema operacional subjacente para executar resolução de nomes, e que não necessariamente realizam qualquer comunicação de rede. Esta categoria contém apenas uma função: dns.lookup().
  2. Funções que se conectam a um servidor DNS real para realizar a resolução de nomes, e que sempre use a rede para realizar consultas DNS. Esta categoria contém todas as funções no módulo dns exceto dns.lookup() .

domínio

Este módulo está com suspensão de uso pendente. Uma vez que uma API de substituição tenha sido finalizado, este módulo será totalmente obsoleto. A maioria dos usuários finais deve não tem motivos para usar este módulo. Usuários que absolutamente devem ter a funcionalidade que os domínios fornecem pode contar com isso por enquanto mas deve esperar ter que migrar para uma solução diferente no futuro.

Events

Grande parte da API principal do Node.js é construída em torno de uma linguagem assíncrona arquitetura orientada a eventos na qual certos tipos de objetos (chamados de "emissores") periodicamente emitem eventos nomeados que fazem com que objetos Function ("ouvintes") sejam chamado.

fs

A E/S de arquivo é fornecida por wrappers simples em torno de funções POSIX padrão. Para usar este módulo faça require('fs'). Todos os métodos têm formulários assíncronos e síncronos.

http

As interfaces HTTP no Node.js são projetadas para oferecer suporte a muitos recursos do protocolo que têm sido tradicionalmente difíceis de usar. Em particular, mensagens grandes, possivelmente codificadas por partes. A interface é cuidado para nunca armazenar em buffer solicitações ou respostas inteiras - o o usuário pode transmitir dados.

https

HTTPS é o protocolo HTTP sobre TLS/SSL. No Node.js isso é implementado como um módulo separado.

module

O Node.js possui um sistema simples de carregamento de módulos. No Node.js, arquivos e módulos estão em correspondência um-para-um (cada arquivo é tratado como um módulo separado).

net

O módulo net fornece um wrapper de rede assíncrono. Contém funções para criar servidores e clientes (chamados streams). Você pode incluir este módulo com require('net');.

os

O módulo os fornece vários métodos utilitários relacionados ao sistema operacional.

path

O módulo path fornece utilitários para trabalhar com caminhos de arquivos e diretórios.

punycode

A versão do módulo punycode empacotado no Node.js está sendo obsoleta.

querystring

O módulo querystring fornece utilitários para analisar e formatar strings de consulta de URL.

readline

O módulo readline fornece uma interface para leitura de dados de um Legível fluxo (como process.stdin) uma linha por vez.

repl

O módulo repl fornece uma implementação Read-Eval-Print-Loop (REPL) que está disponível como um programa autônomo ou incluído em outros aplicativos.

stream

Um stream é uma interface abstrata para trabalhar com dados de streaming em Node.js. O módulo stream fornece uma API básica que facilita a criação de objetos que implementam a interface de fluxo.

Existem muitos objetos de fluxo fornecidos pelo Node.js. Por exemplo, um solicitação para um servidor HTTP e process.stdout são ambas instâncias de stream.

string_decoder

O módulo string_decoder fornece uma API para decodificar objetos Buffer em strings de uma maneira que preserva UTF-8 e UTF-16 de vários bytes codificados personagens.

timers

O módulo timer expõe uma API global para funções de agendamento para ser chamado em algum período de tempo futuro. Como as funções do temporizador são globals, não há necessidade de chamar require('timers') para usar a API.

As funções de timer no Node.js implementam uma API semelhante à API de timers fornecidos por navegadores da Web, mas usam uma implementação interna diferente que é construído em torno do node.js Event Loop.

tls_(ssl)

O módulo tls fornece uma implementação do Transport Layer Security (TLS) e protocolos Secure Socket Layer (SSL) que são construídos sobre o OpenSSL.

tracing

Trace Event fornece um mecanismo para centralizar as informações de rastreamento geradas por V8, núcleo do nó e código do espaço do usuário.

O rastreamento pode ser ativado passando o sinalizador --trace-events-enabled ao iniciar um Aplicativo Node.js.

tty

O módulo tty fornece as classes tty.ReadStream e tty.WriteStream. Na maioria dos casos, não será necessário ou possível utilizar este módulo diretamente.

dgram

O módulo dgram fornece uma implementação de soquetes de datagrama UDP.

url

O módulo url fornece utilitários para resolução e análise de URL.

util

O módulo util foi desenvolvido principalmente para atender às necessidades do Node.js' próprias APIs internas. No entanto, muitos dos utilitários também são úteis para desenvolvedores de aplicativos e módulos.

v8

O módulo v8 expõe APIs específicas para a versão da V8 embutido no binário Node.js.

Observação: as APIs e a implementação estão sujeitas a alterações a qualquer momento.

vm

O módulo vm fornece APIs para compilar e executar código em contextos de máquina virtual V8. O código JavaScript pode ser compilado e executado imediatamente ou compilado, salvo e executado mais tarde.

Observação: O módulo vm não é um mecanismo de segurança. Não use para executar código não confiável.

zlib

O módulo zlib fornece funcionalidade de compactação implementada usando Gzip e Desinflar/inflar.

Soquete TLS: servidor e cliente #

As únicas grandes diferenças entre esta e uma conexão TCP regular são a chave privada e o certificado público que você terá que definir em um objeto de opção.

Como criar uma chave e um certificado #

A primeira etapa neste processo de segurança é a criação de uma chave privada. E o que é essa chave privada? Basicamente, é um conjunto de ruído aleatório usado para criptografar informações. Em teoria, você pode criar uma chave e usá-la para criptografar o que quiser. Mas é uma boa prática ter chaves diferentes para coisas específicas. Porque se alguém roubar sua chave privada, é semelhante a alguém roubar as chaves de sua casa. Imagine se você usasse a mesma chave para trancar seu carro, garagem, escritório, etc.

openssl genrsa -out private-key.pem 1024

Uma vez que tenhamos nossa chave privada, podemos criar um CSR (pedido de assinatura de certificado), que é nosso pedido para que a chave privada seja assinada por uma autoridade sofisticada. É por isso que você deve inserir informações relacionadas à sua empresa. Essas informações serão vistas pela autoridade de assinatura e usadas para verificar você. No nosso caso, não importa o que você digita, pois na próxima etapa vamos assinar nosso certificado por conta própria.

openssl req -new -key private-key.pem -out csr.pem

Agora que temos nossa papelada preenchida, é hora de fingir que somos uma autoridade legal de assinatura.

openssl x509 -req -in csr.pem -signkey private-key.pem -out public-cert.pem

Agora que você tem a chave privada e o certificado público, pode estabelecer uma conexão segura entre dois aplicativos NodeJS. E, como você pode ver no código de exemplo, é um processo muito simples.

Importante! #

Como nós mesmos criamos o certificado público, com toda a honestidade, nosso certificado não vale nada, porque não somos ninguém. O servidor NodeJS não confiará em tal certificado por padrão, e é por isso que precisamos dizer a ele para realmente confiar em nosso certificado com a seguinte opção rejectUnauthorized: false. Muito importante: nunca defina essa variável como verdadeira em um ambiente de produção.

Servidor de soquete TLS #

'use strict';

var tls = require('tls');
var fs = require('fs');

const PORT = 1337;
const HOST = '127.0.0.1'

var options = {
    key: fs.readFileSync('private-key.pem'),
    cert: fs.readFileSync('public-cert.pem')
};

var server = tls.createServer(options, function(socket) {

    // Send a friendly message
    socket.write("I am the server sending you a message.");

    // Print the data that we received
    socket.on('data', function(data) {

        console.log('Received: %s [it is %d bytes long]',
            data.toString().replace(/(\n)/gm,""),
            data.length);

    });

    // Let us know when the transmission is over
    socket.on('end', function() {

        console.log('EOT (End Of Transmission)');

    });

});

// Start listening on a specific port and address
server.listen(PORT, HOST, function() {

    console.log("I'm listening at %s, on port %s", HOST, PORT);

});

// When an error occurs, show it.
server.on('error', function(error) {

    console.error(error);

    // Close the connection after the error occurred.
    server.destroy();

});

Cliente de soquete TLS #

'use strict';

var tls = require('tls');
var fs = require('fs');

const PORT = 1337;
const HOST = '127.0.0.1'

// Pass the certs to the server and let it know to process even unauthorized certs.
var options = {
    key: fs.readFileSync('private-key.pem'),
    cert: fs.readFileSync('public-cert.pem'),
    rejectUnauthorized: false
};

var client = tls.connect(PORT, HOST, options, function() {

    // Check if the authorization worked
    if (client.authorized) {
        console.log("Connection authorized by a Certificate Authority.");
    } else {
        console.log("Connection not authorized: " + client.authorizationError)
    }

    // Send a friendly message
    client.write("I am the client sending you a message.");

});

client.on("data", function(data) {

    console.log('Received: %s [it is %d bytes long]',
        data.toString().replace(/(\n)/gm,""),
        data.length);

    // Close the connection after receiving the message
    client.end();

});

client.on('close', function() {

    console.log("Connection closed");

});

// When an error ocoures, show it.
client.on('error', function(error) {

    console.error(error);

    // Close the connection after the error occurred.
    client.destroy();

});

Como colocar um servidor web HTTPS básico em funcionamento! #

Depois de instalar o node.js em seu sistema, basta seguir o procedimento abaixo para obter um servidor web básico em execução com suporte para HTTP e HTTPS!

Etapa 1: Construir uma Autoridade de Certificação #

  1. crie a pasta onde você deseja armazenar sua chave e certificado:

    mkdir conf

  1. vá para esse diretório:

    cd conf

  1. pegue este arquivo ca.cnf para usar como atalho de configuração:

    wget https://raw.githubusercontent.com/anders94/https-authorized-clients/master/keys/ca.cnf

  1. crie uma nova autoridade de certificação usando esta configuração:

    openssl req -new -x509 -days 9999 -config ca.cnf -keyout ca-key.pem -out ca-cert.pem

  1. agora que temos nossa autoridade de certificação em ca-key.pem e ca-cert.pem, vamos gerar uma chave privada para o servidor:

    openssl genrsa -out key.pem 4096

  1. Pegue este arquivo server.cnf para usar como atalho de configuração:

    wget https://raw.githubusercontent.com/anders94/https-authorized-clients/master/keys/server.cnf

  1. gere a solicitação de assinatura de certificado usando esta configuração:

    openssl req -new -config server.cnf -key key.pem -out csr.pem

  1. Assine o pedido:

    openssl x509 -req -extfile server.cnf -days 999 -passin "pass:password" -in csr.pem -CA ca-cert.pem -CAkey ca-key.pem -CAcreateserial -out cert.pem

Etapa 2: Instale seu certificado como um certificado raiz #

  1. copie seu certificado para a pasta de certificados raiz:

    sudo cp ca-crt.pem /usr/local/share/ca-certificates/ca-crt.pem

  1. atualize o armazenamento da CA:

    sudo update-ca-certificates

Etapa 3: Iniciando seu servidor de nó #

Primeiro, você deseja criar um arquivo server.js que contenha seu código de servidor real.

A configuração mínima para um servidor HTTPS no Node.js seria algo assim:

var https = require('https');
var fs = require('fs');

var httpsOptions = {
    key: fs.readFileSync('path/to/server-key.pem'),
    cert: fs.readFileSync('path/to/server-crt.pem')
};

var app = function (req, res) {
  res.writeHead(200);
  res.end("hello world\n");
}

https.createServer(httpsOptions, app).listen(4433);

Se você também quiser oferecer suporte a solicitações http, precisará fazer apenas esta pequena modificação:

var http = require('http');
var https = require('https');
var fs = require('fs');

var httpsOptions = {
    key: fs.readFileSync('path/to/server-key.pem'),
    cert: fs.readFileSync('path/to/server-crt.pem')
};

var app = function (req, res) {
  res.writeHead(200);
  res.end("hello world\n");
}

http.createServer(app).listen(8888);
https.createServer(httpsOptions, app).listen(4433);

  1. vá para o diretório onde seu server.js está localizado:

    cd /path/to

  1. execute server.js:

    node server.js
