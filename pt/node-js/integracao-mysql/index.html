Neste tópico você aprenderá como integrar com Node.js usando a ferramenta de gerenciamento de banco de dados MYSQL. Você aprenderá várias maneiras de conectar e interagir com dados que residem no mysql usando um programa e script nodejs.

Conecte-se ao MySQL #

Uma das maneiras mais fáceis de se conectar ao MySQL é usando o módulo mysql. Este módulo lida com a conexão entre o aplicativo Node.js e o servidor MySQL. Você pode instalá-lo como qualquer outro módulo:

npm install --save mysql

Agora você precisa criar uma conexão mysql, que pode ser consultada posteriormente.

const mysql      = require('mysql');
const connection = mysql.createConnection({
  host     : 'localhost',
  user     : 'me',
  password : 'secret',
  database : 'database_schema'
});

connection.connect();

// Execute some query statements
// I.e. SELECT * FROM FOO

connection.end();

No próximo exemplo você aprenderá como consultar o objeto connection.

Usando um pool de conexões #

uma. Executando várias consultas ao mesmo tempo #

Todas as consultas na conexão MySQL são feitas uma após a outra. Isso significa que, se você quiser fazer 10 consultas e cada consulta levar 2 segundos, levará 20 segundos para concluir toda a execução. A solução é criar 10 conexões e executar cada consulta em uma conexão diferente. Isso pode ser feito automaticamente usando o pool de conexões

var pool  = mysql.createPool({
  connectionLimit : 10,
  host            : 'example.org',
  user            : 'bobby',
  password        : 'pass',
  database        : 'schema'
});

for(var i=0;i<10;i++){
  pool.query('SELECT ` as example', function(err, rows, fields) {
    if (err) throw err;
    console.log(rows[0].example); //Show 1
  });
 }

Ele executará todas as 10 consultas em paralelo.

Quando você usa pool, você não precisa mais da conexão. Você pode consultar diretamente o pool. O módulo MySQL irá procurar a próxima conexão gratuita para executar sua consulta.

##b. Alcançar multi-tenancy no servidor de banco de dados com diferentes bancos de dados hospedados nele.

A multilocação é um requisito comum de aplicativos corporativos hoje em dia e a criação de um pool de conexões para cada banco de dados no servidor de banco de dados não é recomendada. então, o que podemos fazer é criar um pool de conexões com o servidor de banco de dados e alterná-los entre os bancos de dados hospedados no servidor de banco de dados sob demanda.

Suponha que nosso aplicativo tenha bancos de dados diferentes para cada empresa hospedada no servidor de banco de dados. Nós nos conectaremos ao respectivo banco de dados da empresa quando o usuário acessar o aplicativo. Aqui está o exemplo de como fazer isso: -

var pool  = mysql.createPool({
      connectionLimit : 10,
      host            : 'example.org',
      user            : 'bobby',
      password        : 'pass'
    });
    
pool.getConnection(function(err, connection){
    if(err){
        return cb(err);
    }
    connection.changeUser({database : "firm1"});
    connection.query("SELECT * from history", function(err, data){
        connection.release();
        cb(err, data);
    });
});

Deixe-me quebrar o exemplo: -

Ao definir a configuração do pool, não dei o nome do banco de dados, mas apenas o servidor de banco de dados, ou seja

{
  connectionLimit : 10,
  host            : 'example.org',
  user            : 'bobby',
  password        : 'pass'
}

então, quando queremos usar o banco de dados específico no servidor de banco de dados, pedimos à conexão para acessar o banco de dados usando: -

    connection.changeUser({database : "firm1"});

você pode consultar a documentação oficial aqui

Consulta um objeto de conexão com parâmetros #

Quando você deseja usar o conteúdo gerado pelo usuário no SQL, ele termina com os parâmetros. Por exemplo, para pesquisar usuário com o nome aminadav você deve fazer:

var username = 'aminadav';
var querystring = 'SELECT name, email from users where name = ?'; 
connection.query(querystring, [username], function(err, rows, fields) {
  if (err) throw err;
  if (rows.length) {
    rows.forEach(function(row) {
      console.log(row.name, 'email address is', row.email);
    });
  } else {
    console.log('There were no results.');
  }
});

Consulta um objeto de conexão sem parâmetros #

Você envia a consulta como uma string e em resposta o callback com a resposta é recebido. O retorno de chamada fornece error, array de rows e campos. Cada linha contém toda a coluna da tabela retornada. Aqui está um trecho para a seguinte explicação.

connection.query('SELECT name,email from users', function(err, rows, fields) {
  if (err) throw err;

  console.log('There are:', rows.length,' users');
  console.log('First user name is:',rows[0].name)
});

Execute várias consultas com uma única conexão de um pool #

Pode haver situações em que você configurou um pool de conexões MySQL, mas tem várias consultas que gostaria de executar em sequência:

SELECT 1;
SELECT 2;

Você poderia apenas executar então usando pool.query [como visto em outro lugar](https://www.wikiod.com/pt/node-js/integracao-mysql#Usando um pool de conexões), no entanto, se você tiver apenas uma conexão livre no pool, deverá aguardar até que uma conexão fique disponível antes de poder executar a segunda consulta.

Você pode, no entanto, manter uma conexão ativa do pool e executar quantas consultas quiser usando uma única conexão usando pool.getConnection:

pool.getConnection(function (err, conn) {
  if (err) return callback(err);

  conn.query('SELECT 1 AS seq', function (err, rows) {
    if (err) throw err;

    conn.query('SELECT 2 AS seq', function (err, rows) {
      if (err) throw err;

      conn.release();
      callback();
    });
  });
});

Nota: Você deve se lembrar de liberar a conexão, caso contrário, haverá uma conexão MySQL a menos disponível para o resto do pool!

Para obter mais informações sobre como agrupar conexões MySQL confira a documentação do MySQL.

Exportar pool de conexões #

// db.js

const mysql = require('mysql');

const pool = mysql.createPool({
  connectionLimit : 10,
  host            : 'example.org',
  user            : 'bob',
  password        : 'secret',
  database        : 'my_db'
});

module.export = {
  getConnection: (callback) => {
    return pool.getConnection(callback);
  } 
}

// app.js

const db = require('./db');

db.getConnection((err, conn) => {
  conn.query('SELECT something from sometable', (error, results, fields) => {
    // get the results
    conn.release();
  });
});

Retorna a consulta quando ocorre um erro #

Você pode anexar a consulta executada ao seu objeto err quando ocorrer um erro:

var q = mysql.query('SELECT `name` FROM `pokedex` WHERE `id` = ?', [ 25 ], function (err, result) {
  if (err) {
    // Table 'test.pokedex' doesn't exist
    err.query = q.sql; // SELECT `name` FROM `pokedex` WHERE `id` = 25
    callback(err);
  }
  else {
    callback(null, result);
  }
});
