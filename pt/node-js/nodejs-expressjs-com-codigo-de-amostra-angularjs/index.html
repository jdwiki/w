Este exemplo mostra como criar um aplicativo expresso básico e, em seguida, servir AngularJS.

Criando nosso projeto. #

Estamos prontos, então corremos, novamente do console:

mkdir our_project
cd our_project

Agora estamos no lugar onde nosso código ficará. Para criar o arquivo principal do nosso projeto você pode executar

Ok, mas como criamos o projeto de esqueleto expresso?

É simples:

npm install -g express express-generator

As distribuições Linux e Mac devem usar sudo para instalar isso porque estão instaladas no diretório nodejs, que só pode ser acessado pelo usuário root. Se tudo correu bem, podemos, finalmente, criar o esqueleto do aplicativo expresso, basta executar

express

Este comando criará dentro de nossa pasta um aplicativo de exemplo expresso. A estrutura é a seguinte:

bin/
public/
routes/
views/
app.js
package.json

Agora, se executarmos npm start e acessar http://localhost:3000, veremos o aplicativo expresso funcionando, é justo que geramos um aplicativo expresso sem muitos problemas, mas como podemos misturar isso com AngularJS?.

Como funciona o expresso, resumidamente? #

Express é um framework construído em cima do Nodejs, você pode ver a documentação oficial no Express Site. Mas para nosso propósito precisamos saber que Express é o responsável quando digitamos, por exemplo, http://localhost:3000/home de renderizar a página inicial de nossa aplicação. A partir do aplicativo criado recentemente, podemos verificar:

FILE: routes/index.js
var express = require('express');
var router = express.Router();

/* GET home page. */
router.get('/', function(req, res, next) {
  res.render('index', { title: 'Express' });
});

module.exports = router;

O que este código está nos dizendo é que quando o usuário vai para http://localhost:3000 ele deve renderizar a visualização index e passar um JSON com uma propriedade title e valor Express. Mas quando verificamos o diretório views e abrimos index.jade podemos ver isso:

extends layout
block content
    h1= title
    p Welcome to #{title}

Esse é outro poderoso recurso do Express, os mecanismos de modelo, que permitem renderizar o conteúdo da página passando variáveis ​​para ela ou herdar outro modelo para que suas páginas sejam mais compactas e compreensíveis por outras pessoas. A extensão do arquivo é .jade até onde eu sei Jade mudou o nome para Pug, basicamente é o mesmo mecanismo de template, mas com algumas atualizações e modificações principais.

Instalando o Pug e atualizando o motor de template Express.

Ok, para começar a usar o Pug como template engine do nosso projeto precisamos rodar:

npm install --save pug

Isso instalará o Pug como uma dependência do nosso projeto e o salvará em package.json. Para usá-lo, precisamos modificar o arquivo app.js:

var app = express();
// view engine setup
app.set('views', path.join(__dirname, 'views'));
app.set('view engine', 'pug');

E substitua o mecanismo de linha de visão por pug e pronto. Podemos executar novamente nosso projeto com npm start e veremos que tudo está funcionando bem.

Como o AngularJS se encaixa em tudo isso? #

AngularJS é uma estrutura Javascript MVW (Model-View-Whatever) usada principalmente para criar SPA (Simple Page Application) a instalação é bastante simples, você pode ir ao site AngularJS e baixar o versão mais recente que é v1.6.4.

Após baixarmos o AngularJS quando devemos copiar o arquivo para nossa pasta public/javascripts dentro do nosso projeto, uma pequena explicação, esta é a pasta que serve os assets estáticos do nosso site, imagens, css, arquivos javacript e etc. Claro que isso é configurável por meio do arquivo app.js, mas vamos simplificar. Agora criamos um arquivo chamado ng-app.js, o arquivo onde nossa aplicação ficará, dentro de nossa pasta pública javascripts, exatamente onde o AngularJS mora. Para trazer o AngularJS, precisamos modificar o conteúdo de views/layout.pug da seguinte forma:

doctype html
html(ng-app='first-app')
  head
    title= title
    link(rel='stylesheet', href='/stylesheets/style.css')
  body(ng-controller='indexController')
    block content

  script(type='text-javascript', src='javascripts/angular.min.js')
  script(type='text-javascript', src='javascripts/ng-app.js')

O que estamos fazendo aqui?, bem, estamos incluindo o núcleo do AngularJS e nosso arquivo recém-criado ng-app.js, então quando o modelo for renderizado, ele trará o AngularJS, observe o uso do ng- app diretiva, isso está dizendo ao AngularJS que este é o nome do nosso aplicativo e deve ser mantido nele.
Assim, o conteúdo do nosso ng-app.js será:

angular.module('first-app', [])
       .controller('indexController', ['$scope', indexController]);

function indexController($scope) {
    $scope.name = 'sigfried';
}

Estamos usando o recurso mais básico do AngularJS aqui, two-way data binding, isso nos permite atualizar o conteúdo da nossa view e controller instantaneamente, essa é uma explicação bem simples, mas você pode fazer uma pesquisa no Google ou StackOverflow para ver como ele realmente funciona.

Então, temos os blocos básicos da nossa aplicação AngularJS, mas há algo que temos que fazer, precisamos atualizar nossa página index.pug para ver as mudanças da nossa aplicação angular, vamos fazer isso:

extends layout
block content
  div(ng-controller='indexController')
    h1= title
    p Welcome {{name}}
    input(type='text' ng-model='name')

Aqui estamos apenas vinculando a entrada ao nosso nome de propriedade definido no escopo AngularJS dentro de nosso controlador:

$scope.name = 'sigfried';

O objetivo disso é que sempre que alterarmos o texto na entrada o parágrafo acima irá atualizar o conteúdo dentro do {{name}}, isso é chamado de interpolação, novamente outro recurso do AngularJS para renderizar nosso conteúdo no template.

Então, tudo está configurado, agora podemos executar npm start ir para http://localhost:3000 e ver nosso aplicativo expresso atendendo a página e o AngularJS gerenciando o frontend do aplicativo.
