ES6, ECMAScript 6 ou ES2015 é a mais recente especificação para JavaScript que introduz um pouco de açúcar sintático na linguagem. É uma grande atualização para o idioma e apresenta muitos novos recursos

Mais detalhes sobre Node e ES6 podem ser encontrados em seu site https://nodejs.org/en/docs/es6/

Suporte ao Node ES6 e criação de um projeto com Babel #

Toda a especificação do ES6 ainda não está totalmente implementada, então você só poderá usar alguns dos novos recursos. Você pode ver uma lista dos recursos atuais do ES6 suportados em http://node.green/

Desde o NodeJS v6 tem havido um suporte muito bom. Portanto, se você estiver usando o NodeJS v6 ou superior, poderá usar o ES6. No entanto, você também pode querer usar alguns dos recursos não lançados e alguns de outros lugares. Para isso, você precisará usar um transpilador

É possível executar um transpilador em tempo de execução e compilação, usar todos os recursos do ES6 e muito mais. O transpilador mais popular para JavaScript é chamado Babel

Babel permite que você use todos os recursos da especificação ES6 e alguns recursos adicionais não especificados com ‘stage-0’, como import thing from 'thing' em vez de var thing = require(’thing’)`

Se quiséssemos criar um projeto em que usamos recursos ‘stage-0’, como importação, precisaríamos adicionar o Babel como transpilador. Você verá projetos usando react e Vue e outros padrões baseados em commonJS implementando o estágio 0 com bastante frequência.

criar um novo projeto de nó

mkdir my-es6-app
cd my-es6-app
npm init

Instale babel o preset ES6 e stage-0

npm install --save-dev babel-preset-es2015 babel-preset-stage-2 babel-cli babel-register

Create a new file called server.js and add a basic HTTP server.

import http from 'http'

http.createServer((req, res) => {
  res.writeHead(200, {'Content-Type': 'text/plain'})
  res.end('Hello World\n')
}).listen(3000, '127.0.0.1')

console.log('Server running at http://127.0.0.1:3000/')

Note that we use an import http from 'http' this is a stage-0 feature and if it works it means we’ve got the transpiler working correctly.

If you run node server.js it will fail not knowing how to handle the import.

Creating a .babelrc file in the root of your directory and add the following settings

{
  "presets": ["es2015", "stage-2"],
  "plugins": []
}

you can now run the server with node src/index.js --exec babel-node

Finishing off it is not a good idea to run a transpiler at runtime on a production app. We can however implement some scripts in our package.json to make it easier to work with.

"scripts": {
    "start": "node dist/index.js",
    "dev": "babel-node src/index.js",
    "build": "babel src -d dist",
    "postinstall": "npm run build"
  },

The above will on npm install build the transpiled code to the dist directory allow npm start to use the transpiled code for our production app.

npm run dev will boot the server and babel runtime which is fine and preferred when working on a project locally.

Going one further you could then install nodemon npm install nodemon --save-dev to watch for changes and then reboot the node app.

This really speeds up working with babel and NodeJS. In you package.json just update the “dev” script to use nodemon

"dev": "nodemon src/index.js --exec babel-node",

Use JS es6 em seu aplicativo NodeJS #

JS es6 (também conhecido como es2015) é um conjunto de novos recursos para a linguagem JS que visa torná-la mais intuitiva ao usar OOP ou ao enfrentar tarefas de desenvolvimento modernas.

Pré-requisitos: #

  1. Confira os novos recursos do es6 em http://es6-features.org - pode esclarecer se você realmente pretende usá-lo em seu próximo aplicativo NodeJS

  2. Verifique o nível de compatibilidade da versão do seu nó em http://node.green

  3. Se tudo estiver ok - vamos codificar!

Aqui está uma amostra muito curta de um aplicativo hello world simples com JS es6

'use strict'

class Program
{
    constructor()
    {
        this.message = 'hello es6 :)';
    }

    print()
    {
        setTimeout(() =>
        {
            console.log(this.message);
            
            this.print();

        }, Math.random() * 1000);
    }
}

new Program().print();

Você pode executar este programa e observar como ele imprime a mesma mensagem repetidamente.

Agora .. vamos dividir linha por linha:

'use strict'

Esta linha é realmente necessária se você pretende usar js es6. O modo strict, intencionalmente, tem uma semântica diferente do código normal (por favor, leia mais sobre isso no MDN - https://developer.mozilla.org/en-US/docs/Web/JavaScript/Reference/Strict_mode)

class Program

Inacreditável - uma palavra-chave class! Apenas para uma referência rápida - antes do es6 a única maneira de definir uma classe em js era com a palavra-chave… function!

function MyClass() // class definition
{

}

var myClassObject = new MyClass(); // generating a new object with a type of MyClass

Ao usar OOP, uma classe é uma habilidade muito fundamental que ajuda o desenvolvedor a representar uma parte específica de um sistema (decompor o código é crucial quando o código está ficando maior… por exemplo: ao escrever código do lado do servidor)

constructor()
{
    this.message = 'hello es6 :)';
}

Você tem que admitir - isso é bastante intuitivo! Este é o c’tor da minha classe - essa “função” exclusiva ocorrerá toda vez que um objeto for criado a partir dessa classe específica (em nosso programa - apenas uma vez)

print()
{
    setTimeout(() => // this is an 'arrow' function
    {
        console.log(this.message);
        
        this.print(); // here we call the 'print' method from the class template itself (a recursion in this particular case)

    }, Math.random() * 1000);
}

Porque print é definido no escopo da classe - na verdade é um método - que pode ser invocado do objeto da classe ou de dentro da própria classe!

Então .. até agora nós definimos nossa classe .. hora de usá-la:

new Program().print();

O que é realmente igual a:

var prog = new Program(); // define a new object of type 'Program'

prog.print(); // use the program to print itself

Em conclusão: JS es6 pode simplificar seu código - torná-lo mais intuitivo e fácil de entender (comparando com a versão anterior do JS).. você pode tentar reescrever um código existente e ver a diferença para você mesma
