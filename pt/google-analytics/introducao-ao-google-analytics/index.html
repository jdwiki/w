Entrando a bordo com o Google Analytics #

  1. Como obter uma conta do GA: If you don’t have an Analytics account, create one. If you do have an Analytics account, sign in. Both options are available at google.com/analytics

  2. Configurando uma propriedade em sua conta do Google Analytics: A property represents your website or app where the data gets aggregated.

  3. Crie uma visualização: Views let you create filtered perspectives of your data. When you create a property one view is created by default. You can create multiple views based on the requirement and filter the reports based on the reporting structure.

  4. Incorpore o Analytics em seu website: vá para propriedade > informações de acompanhamento e obtenha o código de acompanhamento que se parece abaixo

  5. A string ‘UA-XXXXX-Y’ deve ser substituída pelo ID da propriedade (também chamado de “ID de acompanhamento”) da propriedade do Google Analytics que você deseja acompanhar.

Com essas etapas simples, seu site estará pronto para enviar as visualizações de página ao GA.

Adicionando analytics.js ao seu site #

Adicione o seguinte código (conhecido como “snippet de acompanhamento JavaScript”) aos modelos do seu site.

O código deve ser adicionado antes da tag de fechamento e a string ‘UA-XXXXX-Y’ deve ser substituída pelo ID da propriedade (também chamado de “ID de rastreamento”) de a propriedade do Google Analytics que você deseja acompanhar.

<!-- Google Analytics -->
<script>
(function(i,s,o,g,r,a,m){i['GoogleAnalyticsObject']=r;i[r]=i[r]||function(){
(i[r].q=i[r].q||[]).push(arguments)},i[r].l=1*new Date();a=s.createElement(o),
m=s.getElementsByTagName(o)[0];a.async=1;a.src=g;m.parentNode.insertBefore(a,m)
})(window,document,'script','https://www.google-analytics.com/analytics.js','ga');

ga('create', 'UA-XXXXX-Y', 'auto');
ga('send', 'pageview');
</script>
<!-- End Google Analytics -->

O código acima faz quatro coisas principais:

  1. Cria um elemento <script> que inicia o download assíncrono da biblioteca JavaScript analytics.js em https://www.google-analytics.com/analytics.js
  2. Inicializa uma função ga global (chamada fila de comandos ga()) que permite agendar comandos para serem executados assim que a biblioteca analytics.js estiver carregada e pronta para uso.
  3. Adiciona um comando à fila de comandos ga() para criar um novo objeto rastreador para a propriedade especificada através do parâmetro 'UA-XXXXX-Y'.
  4. Adiciona outro comando à fila de comandos ga() para enviar uma visualização de página ao Google Analytics para a página atual.

Visão geral #

O Google Analytics é usado para rastrear a atividade do usuário em seu site ou aplicativo móvel.

Para configurar o google-analytics em um site, você precisará obter um snippet de código JavaScript do Google que você incorpora no cabeçalho de cada página do site que deseja rastrear a atividade do usuário.

Obtenha o snippet de código em www.google.com/analytics e, na guia Admin, selecione “Criar nova conta” no menu suspenso da coluna da conta à esquerda .

Acompanhar páginas chamadas por conteúdo AJAX e não-html #

Para rastrear as chamadas “visualizações de página virtuais”, use o método ga('send') logo após sua solicitação assíncrona:

Sintaxe: ga('send', 'pageview', 'path to your virtual page');

Exemplo (Link Simples):

<a href="http://example.com/my.pdf"
   onClick="ga('send', 'pageview', '/virtual/my.pdf');">Download PDF</a>

Exemplo (JQuery AJAX):

$.ajax({
    url: '/ajax-url/file.json',
    data: {page: 4},
    success: function(data) {
        ga('send', 'pageview', '/ajax-url/file.json');
        console.log("Got response",data); 
    },
    dataType: 'json',
    method: 'GET'
});

Snippet de rastreamento assíncrono alternativo #

Embora o snippet de rastreamento de JavaScript descrito acima garanta que o script seja carregado e executado de forma assíncrona em todos os navegadores, ele tem a desvantagem de não permitir que navegadores modernos pré-carregem o script.

O snippet de rastreamento assíncrono alternativo abaixo adiciona suporte para pré-carregamento, que fornecerá um pequeno aumento de desempenho em navegadores modernos, mas pode degradar para carregamento e execução síncronos no IE 9 e navegadores móveis mais antigos que não reconhecem o atributo de script assíncrono. Use este snippet de rastreamento apenas se seus visitantes usarem principalmente navegadores modernos para acessar seu site.

    <!-- Google Analytics -->
<script>
window.ga=window.ga||function(){(ga.q=ga.q||[]).push(arguments)};ga.l=+new Date;
ga('create', 'UA-XXXXX-Y', 'auto');
ga('send', 'pageview');
</script>
<script async src='https://www.google-analytics.com/analytics.js'></script>
<!-- End Google Analytics -->

Usando plug-ins #

Os plug-ins são scripts que aprimoram a funcionalidade do analytics.js para auxiliar na medição da interação do usuário. Os plug-ins geralmente são específicos para um conjunto de recursos que podem não ser exigidos por todos os usuários do Google Analytics, como comércio eletrônico ou acompanhamento de vários domínios e, portanto, não são incluídos no analytics.js por padrão.

Este guia explica como exigir e usar plug-ins do analytics.js.

O comando require pega o nome de um plugin e o registra para uso com a fila de comandos ga(). Se o plugin aceitar opções de configuração, essas opções podem ser passadas como o argumento final para o comando require.

A seguir está a assinatura completa do comando require:

ga('[trackerName.]require', pluginName, [pluginOptions]);

Por exemplo, veja como você exigiria o plug-in de comércio eletrônico avançado para uso com o rastreador padrão:

ga('require', 'ec');

E aqui está como você precisaria do plug-in Display Features para um rastreador chamado “myTracker” e passaria uma opção de configuração que substitui o valor do nome do cookie padrão:

ga('myTracker.require', 'displayfeatures', {
  cookieName: 'display_features_cookie'
});

Quais dados o snippet de rastreamento captura? #

Ao adicionar um desses snippets de rastreamento ao seu site, você envia uma visualização de página para cada página que seus usuários visitam. O Google Analytics processa esses dados e pode inferir muitas informações, incluindo:

O tempo total que um usuário passa em seu site. O tempo que um usuário gasta em cada página e em que ordem essas páginas foram visitadas. Quais links internos foram clicados (com base no URL da próxima exibição de página). Além disso, o endereço IP, a string do agente do usuário e a inspeção de página inicial que o analytics.js faz ao criar um novo rastreador é usado para determinar coisas como:

A localização geográfica do usuário. Qual navegador e sistema operacional estão sendo usados. Tamanho da tela e se Flash ou Java está instalado. O site de referência.
