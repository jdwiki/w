Há um total de 3 casos de comunicação entre componentes React:

  • Caso 1: Comunicação de Pai para Filho
  • Caso 2: Comunicação entre pais e filhos
  • Caso 3: Comunicação de componentes não relacionados (qualquer componente para qualquer componente)

Componentes filho para pai #

Enviando dados de volta para o pai, para fazer isso, simplesmente passamos uma função como prop do componente pai para o componente filho, e o componente filho chama essa função.

Neste exemplo, alteraremos o estado Parent passando uma função para o componente Child e invocando essa função dentro do componente Child.

import React from 'react';

class Parent extends React.Component {
    constructor(props) {
        super(props);
        this.state = { count: 0 };

        this.outputEvent = this.outputEvent.bind(this);
    }
    outputEvent(event) {
        // the event context comes from the Child
        this.setState({ count: this.state.count++ });
    }

    render() {
        const variable = 5;
        return (
            <div>
                Count: { this.state.count }
                <Child clickHandler={this.outputEvent} />
            </div>
        );
    }
}

class Child extends React.Component {
    render() {
        return (
            <button onClick={this.props.clickHandler}>
                Add One More
            </button>
        );
    }
}

export default Parent;

Observe que o método outputEvent do Parent (que altera o estado do Parent) é invocado pelo evento onClick do botão do Child.

Componentes não relacionados #

A única maneira se seus componentes não tiverem um relacionamento pai-filho (ou estiverem relacionados, mas muito além, como um bisneto) é ter algum tipo de sinal de que um componente se inscreve e o outro grava.

Essas são as 2 operações básicas de qualquer sistema de eventos: assinar/ouvir um evento para ser notificado e enviar/acionar/publicar/despachar um evento para notificar quem quiser.

Existem pelo menos 3 padrões para fazer isso. Você pode encontrar uma comparação aqui.

Aqui está um breve resumo:

  • Padrão 1: Emissor/destino/despachante de evento: os ouvintes precisam referenciar a fonte para assinar.

  • para assinar: otherObject.addEventListener('click', () => { alert('click!'); });

  • para despachar: this.dispatchEvent('click');

  • Padrão 2: Publicar/Assinar: você não precisa de uma referência específica à fonte que aciona o evento, existe um objeto global acessível em todos os lugares que trata de todos os eventos.

  • para se inscrever: globalBroadcaster.subscribe('click', () => { alert('click!'); });

  • para despachar: globalBroadcaster.publish('click');

  • Padrão 3: Sinais: semelhante ao Event Emitter/Target/Dispatcher, mas você não usa nenhuma string aleatória aqui. Cada objeto que pode emitir eventos precisa ter uma propriedade específica com esse nome. Dessa forma, você sabe exatamente quais eventos um objeto pode emitir.

  • para assinar: otherObject.clicked.add( () => { alert('click'); });

  • para despachar: this.clicked.dispatch();

Componentes pai para filho #

Esse é o caso mais fácil, na verdade, muito natural no mundo React e as chances são - você já está usando.

Você pode passar adereços para componentes filhos. Neste exemplo message é o prop que passamos para o componente filho, o nome message é escolhido arbitrariamente, você pode nomeá-lo como quiser.

import React from 'react';

class Parent extends React.Component {
    render() {
        const variable = 5;
        return (
            <div>
                <Child message="message for child" />
                <Child message={variable} />
            </div>
        );
    }
}

class Child extends React.Component {
    render() {
        return <h1>{this.props.message}</h1>
    }
}

export default Parent;

Aqui, o componente <Parent /> renderiza dois componentes <Child />, passando message for child dentro do primeiro componente e 5 dentro do segundo.

Em resumo, você tem um componente (pai) que renderiza outro (filho) e passa para ele alguns adereços.
