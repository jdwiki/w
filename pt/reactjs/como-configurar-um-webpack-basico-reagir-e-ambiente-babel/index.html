Como configurar um webpack básico, reagir e ambiente babel

  • Como construir um pipeline para um “Hello world” personalizado com imagens.

Nesta página

  • Como construir um pipeline para um “Hello world” personalizado com imagens.

Esse pipeline de construção não é exatamente o que você chamaria de “pronto para produção”, mas fornece um começo sólido para você adicionar a ele as coisas que você precisa para obter a experiência de desenvolvimento que está procurando. A abordagem que algumas pessoas adotam (inclusive eu às vezes) é pegar um pipeline totalmente construído do Yeoman.io ou de outro lugar e, em seguida, retirar as coisas que eles não querem até que se adeque ao estilo. Não há nada de errado com isso, mas talvez com o exemplo acima você possa optar pela abordagem oposta e construir a partir do esqueleto.

Algumas coisas que você pode querer adicionar são coisas como uma estrutura de teste e estatísticas de cobertura como Karma com Mocha ou Jasmine. Linting com ESLint. Substituição de módulo quente no webpack-dev-server para que você possa obter essa experiência de desenvolvimento Ctrl+S, F5. Além disso, o pipeline atual só é compilado no modo dev, portanto, uma tarefa de compilação de produção seria boa.

Pegadinhas!

Observe na propriedade context do webpack.config.js que usamos o módulo node path para definir nosso caminho ao invés de apenas concatenar __dirname para a string /src porque windows odeia barras. Portanto, para tornar a solução mais compatível com várias plataformas, use o nó de alavancagem para nos ajudar.

Explicação das propriedades do webpack.config.js

contexto

Este é o caminho de arquivo para o qual o webpack usará como caminho raiz para resolver os caminhos de arquivo relativos. Então em index.jsx onde usamos require('./index.html') esse ponto resolve para o diretório src/ porque nós o definimos como tal nesta propriedade.

entrada

Onde o webpack procura primeiro para começar a agrupar a solução. É por isso que você verá que no index.jsx estamos unindo a solução com os requisitos e as importações.

resultado

É aqui que definimos onde o webpack deve soltar os arquivos de arquivo que encontrou para agrupar. Também definimos um nome para o arquivo no qual nosso javascript e estilos agrupados serão descartados.

devServer

Estas são configurações específicas para webpack-dev-server. A contentBase define onde o servidor deve fazer seu root, definimos a pasta dist/ como nossa base aqui. A port é a porta na qual o servidor será hospedado. open é o que é usado para instruir o webpack-dev-server a abrir seu navegador padrão para você uma vez que o servidor esteja ligado.

módulo > carregadores

Isso define um mapeamento para o webpack usar para que ele saiba o que fazer quando encontrar arquivos diferentes. A propriedade test fornece regex para o webpack usar para determinar se ele deve aplicar este módulo, na maioria dos casos temos correspondências nas extensões de arquivo. loader ou loaders dão o nome do módulo carregador que gostaríamos de usar para carregar o arquivo no webpack e deixar que o carregador cuide do agrupamento desse tipo de arquivo. Há também uma propriedade query no javascript, isso apenas fornece uma string de consulta para o carregador, então provavelmente poderíamos ter usado uma propriedade de consulta no carregador html também, se quiséssemos. É apenas uma maneira diferente de fazer as coisas.

Como construir um pipeline para um “Hello world” personalizado com imagens. #

Etapa 1: instale o Node.js

O pipeline de compilação que você criará é baseado em Node.js, portanto, você deve garantir, em primeira instância, que ele esteja instalado. Para obter instruções sobre como instalar o Node.js, você pode conferir os documentos SO para isso aqui

Etapa 2: inicialize seu projeto como um módulo de nó

Abra a pasta do seu projeto na linha de comando e use o seguinte comando:

npm init

Para os propósitos deste exemplo, você pode ficar à vontade para usar os padrões ou, se quiser mais informações sobre o que tudo isso significa, confira [este][2] SO doc sobre como configurar a configuração do pacote.

Etapa 3: Instale os pacotes npm necessários

Execute o seguinte comando na linha de comando para instalar os pacotes necessários para este exemplo:

npm install --save react react-dom

Em seguida, para as dependências dev, execute este comando:

npm install --save-dev babel-core babel-preset-react babel-preset-es2015 webpack babel-loader css-loader style-loader file-loader image-webpack-loader

Finalmente, webpack e webpack-dev-server são coisas que valem a pena instalar globalmente, em vez de como uma dependência do seu projeto, se você preferir adicioná-lo como uma dependência, isso funcionará, eu não. Aqui está o comando a ser executado:

npm install --global webpack webpack-dev-server

Etapa 3: adicione um arquivo .babelrc à raiz do seu projeto

Isso configurará o babel para usar as predefinições que você acabou de instalar. Seu arquivo .babelrc deve ficar assim:

{
  "presets": ["react", "es2015"]
}

Etapa 4: configurar a estrutura de diretórios do projeto

Configure uma estrutura de diretório que se pareça com a abaixo na raiz do seu diretório:

|- node_modules
|- src/
  |- components/
  |- images/
  |- styles/
  |- index.html
  |- index.jsx
|- .babelrc
|- package.json

NOTA: Os node_modules, .babelrc e package.json já deveriam estar lá desde os passos anteriores, acabei de incluí-los para que você possa ver onde eles se encaixam.

Etapa 5: Preencha o projeto com os arquivos do projeto Hello World

Isso não é realmente importante para o processo de construção de um pipeline, então vou apenas fornecer o código para eles e você pode copiá-los e colá-los em:

src/components/HelloWorldComponent.jsx

import React, { Component } from 'react';

class HelloWorldComponent extends Component {
  constructor(props) {
    super(props);
    this.state = {name: 'Student'};
    this.handleChange = this.handleChange.bind(this);
  }

  handleChange(e) {
    this.setState({name: e.target.value});
  }

  render() {
    return (
      <div>
        <div className="image-container">
          <img src="./images/myImage.gif" />
        </div>
        <div className="form">
          <input type="text" onChange={this.handleChange} />
          <div>
            My name is {this.state.name} and I'm a clever cloggs because I built a React build pipeline
          </div>
        </div>
      </div>
    );
  }
}

export default HelloWorldComponent;

src/images/myImage.gif

Sinta-se à vontade para substituir isso por qualquer imagem que você gostaria, simplesmente está lá para provar que podemos agrupar imagens também. Se você fornecer sua própria imagem e nomeá-la de forma diferente, terá que atualizar o HelloWorldComponent.jsx para refletir suas alterações. Da mesma forma, se você escolher uma imagem com uma extensão de arquivo diferente, precisará modificar a propriedade test do carregador de imagens no webpack.config.js com o regex apropriado para corresponder à sua nova extensão de arquivo.

src/styles/styles.css

.form {
  margin: 25px;
  padding: 25px;
  border: 1px solid #ddd;
  background-color: #eaeaea;
  border-radius: 10px;
}

.form div {
  padding-top: 25px;
}

.image-container {
  display: flex;
  justify-content: center;
}

index.html

<!DOCTYPE html>
<html lang="en">
<head>
  <meta charset="UTF-8">
  <title>Learning to build a react pipeline</title>
</head>
<body>
  <div id="content"></div>
  <script src="app.js"></script>
</body>
</html>

index.jsx

import React from 'react';
import { render } from 'react-dom';
import HelloWorldComponent from './components/HelloWorldComponent.jsx';

require('./images/myImage.gif');
require('./styles/styles.css');
require('./index.html');

render(<HelloWorldComponent />, document.getElementById('content'));

Etapa 6: crie a configuração do webpack

Crie um arquivo chamado webpack.config.js na raiz do seu projeto e copie este código nele:

webpack.config.js

var path = require('path');

var config = {
  context: path.resolve(__dirname + '/src'),
  entry: './index.jsx',
  output: {
    filename: 'app.js',
    path: path.resolve(__dirname + '/dist'),
  },
  devServer: {
    contentBase: path.join(__dirname + '/dist'),
    port: 3000,
    open: true,
  },
  module: {
    loaders: [
      {
        test: /\.(js|jsx)$/,
        exclude: /node_modules/,
        loader: 'babel-loader'
      },
      {
        test: /\.css$/,
        loader: "style!css"
      },
      {
        test: /\.gif$/,
        loaders: [
          'file?name=[path][name].[ext]',
          'image-webpack',
        ]
      },
      { test: /\.(html)$/,
        loader: "file?name=[path][name].[ext]"
      }
    ],
  },
};

module.exports = config;

Etapa 7: crie tarefas npm para seu pipeline

Para fazer isso, você precisará adicionar duas propriedades à chave de scripts do JSON definido no arquivo package.json na raiz do seu projeto. Faça com que sua chave de scripts fique assim:

  "scripts": {
    "start": "webpack-dev-server",
    "build": "webpack",
    "test": "echo \"Error: no test specified\" && exit 1"
  },

O script test já estará lá e você pode escolher se quer mantê-lo ou não, não é importante para este exemplo.

Etapa 8: use o pipeline

Na linha de comando, se você estiver no diretório raiz do projeto, agora poderá executar o comando:

npm run build

Isso irá agrupar o pequeno aplicativo que você construiu e colocá-lo no diretório dist/ que ele criará na raiz da pasta do seu projeto.

Se você executar o comando:

npm start

Em seguida, o aplicativo que você criou será servido em seu navegador da Web padrão dentro de uma instância do servidor de desenvolvimento do webpack.
