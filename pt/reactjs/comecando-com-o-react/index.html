O que é ReactJS? #

O ReactJS é uma biblioteca front-end de código aberto, baseada em componentes, responsável apenas pela camada de visualização do aplicativo. É mantido pelo Facebook.

ReactJS usa mecanismo baseado em DOM virtual para preencher dados (views) em HTML DOM. O DOM virtual funciona rapidamente devido ao fato de que ele altera apenas elementos individuais do DOM em vez de recarregar o DOM completo toda vez

Um aplicativo React é composto de vários componentes, cada um responsável por gerar um pequeno pedaço de HTML reutilizável. Os componentes podem ser aninhados em outros componentes para permitir que aplicativos complexos sejam construídos a partir de blocos de construção simples. Um componente também pode manter o estado interno - por exemplo, um componente TabList pode armazenar uma variável correspondente à guia aberta no momento.

O React nos permite escrever componentes usando uma linguagem específica de domínio chamada JSX. JSX nos permite escrever nossos componentes usando HTML, enquanto misturamos eventos JavaScript. O React converterá internamente isso em um DOM virtual e, finalmente, produzirá nosso HTML para nós.

Reaja “reacts” para declarar alterações em seus componentes de forma rápida e automática para renderizar novamente os componentes no HTML DOM utilizando o DOM virtual. O DOM virtual é uma representação na memória de um DOM real. Ao fazer a maior parte do processamento dentro do DOM virtual em vez de diretamente no DOM do navegador, o React pode agir rapidamente e apenas adicionar, atualizar e remover componentes que foram alterados desde o último ciclo de renderização.

Instalação ou configuração #

ReactJS é uma biblioteca JavaScript contida em um único arquivo react-<version>.js que pode ser incluído em qualquer página HTML. As pessoas também costumam instalar a biblioteca React DOM react-dom-<version>.js junto com o arquivo React principal:

Inclusão Básica

<!DOCTYPE html>
<html>
    <head></head>
    <body>
    <script type="text/javascript" src="/path/to/react.js"></script>
    <script type="text/javascript" src="/path/to/react-dom.js"></script>
    <script type="text/javascript">
        // Use react JavaScript code here or in a separate file
    </script>
    </body>
</html>

Para obter os arquivos JavaScript, acesse a página de instalação da documentação oficial do React.

React também suporta sintaxe JSX. JSX é uma extensão criada pelo Facebook que adiciona sintaxe XML ao JavaScript. Para usar JSX você precisa incluir a biblioteca Babel e mudar <script type="text/javascript"> para <script type="text/babel"> para traduzir JSX para código Javascript.

<!DOCTYPE html>
<html>
    <head></head>
    <body>
    <script type="text/javascript" src="/path/to/react.js"></script>
    <script type="text/javascript" src="/path/to/react-dom.js"></script>
    <script src="https://npmcdn.com/babel-core@5.8.38/browser.min.js"></script>
    <script type="text/babel">
        // Use react JSX code here or in a separate file
    </script>
    </body>
</html>

Instalando via npm

Você também pode instalar o React usando npm fazendo o seguinte:

npm install --save react react-dom

Para usar o React em seu projeto JavaScript, você pode fazer o seguinte:

var React = require('react');
var ReactDOM = require('react-dom');
ReactDOM.render(<App />, ...);

Instalação via Fio

O Facebook lançou seu próprio gerenciador de pacotes chamado Yarn, que também pode ser usado para instalar o React. Depois de instalar o Yarn, você só precisa executar este comando:

yarn add react react-dom

Você pode usar o React em seu projeto exatamente da mesma maneira como se tivesse instalado o React via npm.

Hello World com funções sem estado #

Componentes sem estado estão obtendo sua filosofia da programação funcional. O que implica que: Uma função retorna sempre a mesma coisa exatamente sobre o que é dado a ela.

Por exemplo: #

const statelessSum = (a, b) => a + b;

let a = 0;
const statefulSum = () => a++;

Como você pode ver no exemplo acima, statelessSum é sempre retornará os mesmos valores dados a e b. No entanto, a função statefulSum não retornará os mesmos valores, mesmo sem parâmetros. O comportamento desse tipo de função também é chamado de efeito colateral. Desde então, o componente afeta algo além.

Portanto, é aconselhável usar componentes sem estado com mais frequência, pois eles são isentos de efeitos colaterais e criarão sempre o mesmo comportamento. Isso é o que você quer em seus aplicativos porque o estado flutuante é o pior cenário para um programa de manutenção.

O tipo mais básico de componente de reação é aquele sem estado. Componentes React que são funções puras de seus props e não requerem nenhum gerenciamento de estado interno podem ser escritos como funções JavaScript simples. Estes são chamados de Componentes Funcionais sem Estado porque eles são uma função apenas de props, sem ter nenhum estado para acompanhar.

Aqui está um exemplo simples para ilustrar o conceito de um Componente Funcional sem Estado:

// In HTML
<div id="element"></div>

// In React
const MyComponent = props => {
    return <h1>Hello, {props.name}!</h1>;
};

ReactDOM.render(<MyComponent name="Arun" />, element);
// Will render <h1>Hello, Arun!</h1>

Observe que tudo o que este componente faz é renderizar um elemento h1 contendo a propriedade name. Este componente não acompanha nenhum estado. Aqui está um exemplo ES6 também:

import React from 'react'

const HelloWorld = props => (
    <h1>Hello, {props.name}!</h1>
)

HelloWorld.propTypes = {
    name: React.PropTypes.string.isRequired
}

export default HelloWorld

Como esses componentes não requerem uma instância de backup para gerenciar o estado, o React tem mais espaço para otimizações. A implementação é limpa, mas até agora nenhuma otimizações para componentes sem estado foram implementadas.

Noções básicas absolutas de criação de componentes reutilizáveis #

Componentes e Adereços #

Como o React se preocupa apenas com a visão de uma aplicação, a maior parte do desenvolvimento no React será a criação de componentes. Um componente representa uma parte da visualização de seu aplicativo. “Props” são simplesmente os atributos usados ​​em um nó JSX (por exemplo, <SomeComponent someProp="some prop's value" />) e são a principal maneira pela qual nosso aplicativo interage com nossos componentes. No trecho acima, dentro de SomeComponent, teríamos acesso a this.props, cujo valor seria o objeto {someProp: "some prop's value"}.

Pode ser útil pensar nos componentes do React como funções simples - eles recebem entrada na forma de “props” e produzem saída como marcação. Muitos componentes simples levam isso um passo adiante, tornando-se “Funções Puras”, o que significa que não emitem efeitos colaterais e são idempotentes (dado um conjunto de entradas, o componente sempre produzirá a mesma saída). Esse objetivo pode ser formalmente imposto pela criação de componentes como funções, em vez de “classes”. Existem três maneiras de criar um componente React:

  • Componentes Funcionais (“Sem Estado”)
const FirstComponent = props => (
    <div>{props.content}</div>
);
  • React.createClass()
const SecondComponent = React.createClass({
    render: function () {
        return (
            <div>{this.props.content}</div>
        );
    }
});
  • Aulas ES2015
class ThirdComponent extends React.Component {
    render() {
        return (
            <div>{this.props.content}</div>
        );
    }
}

Esses componentes são usados ​​exatamente da mesma maneira:

const ParentComponent = function (props) {
    const someText = "FooBar";
    return (
        <FirstComponent content={someText} />
        <SecondComponent content={someText} />
        <ThirdComponent content={someText} />
    );
}

Todos os exemplos acima produzirão marcação idêntica.

Os componentes funcionais não podem ter “estado” dentro deles. Portanto, se seu componente precisar ter um estado, vá para componentes baseados em classe. Consulte Criando Componentes para obter mais informações.

Como nota final, os props react são imutáveis ​​depois de passados, o que significa que não podem ser modificados de dentro de um componente. Se o pai de um componente alterar o valor de uma prop, o React lida com a substituição das props antigas pelas novas, o componente será renderizado novamente usando os novos valores.

Veja [Thinking In React][2] e [Reusable Components][3] para mergulhos mais profundos na relação de adereços com componentes.

1: https://www.wikiod.com/pt/reactjs/componentes#Criando componentes [2]: https://facebook.github.io/react/docs/thinking-in-react.html “Pensando em Reagir” [3]: https://facebook.github.io/react/docs/reusable-components.html “Componentes Reutilizáveis”

Criar aplicativo React #

create-react-app é um gerador padrão de aplicativo React criado pelo Facebook. Ele fornece um ambiente de desenvolvimento configurado para facilidade de uso com configuração mínima, incluindo:

  • Transpilação ES6 e JSX
  • Servidor de desenvolvimento com recarga de módulo quente
  • Linting de código
  • Prefixação automática de CSS
  • Construir script com JS, CSS e agrupamento de imagens e mapas de origem
  • Estrutura de teste Jest

Instalação #

Primeiro, instale o create-react-app globalmente com o node package manager (npm).

npm install -g create-react-app

Em seguida, execute o gerador no diretório escolhido.

create-react-app my-app

Navegue até o diretório recém-criado e execute o script de início.

cd my-app/
npm start

Configuração #

create-react-app é intencionalmente não configurável por padrão. Se o uso não padrão for necessário, por exemplo, para usar uma linguagem CSS compilada, como Sass, o comando ejetar poderá ser usado.

npm run eject

Isso permite a edição de todos os arquivos de configuração. N.B. este é um processo irreversível.

Olá Mundo #

Sem JSX

Aqui está um exemplo básico que usa a API principal do React para criar um elemento React e a API DOM do React para renderizar o elemento React no navegador.

<!DOCTYPE html>
<html>
  <head>
    <meta charset="UTF-8" />
    <title>Hello React!</title>

    <!-- Include the React and ReactDOM libraries -->
    <script src="https://fb.me/react-15.2.1.js"></script>
    <script src="https://fb.me/react-dom-15.2.1.js"></script>

  </head>
  <body>
    <div id="example"></div>

    <script type="text/javascript">

      // create a React element rElement
      var rElement = React.createElement('h1', null, 'Hello, world!');

      // dElement is a DOM container
      var dElement = document.getElementById('example');

      // render the React element in the DOM container
      ReactDOM.render(rElement, dElement);

    </script>

  </body>
</html>

Com JSX

Em vez de criar um elemento React a partir de strings, pode-se usar JSX (uma extensão Javascript criada pelo Facebook para adicionar sintaxe XML ao JavaScript), que permite escrever

var rElement = React.createElement('h1', null, 'Hello, world!');

como o equivalente (e mais fácil de ler para alguém familiarizado com HTML)

var rElement = <h1>Hello, world!</h1>;

O código que contém JSX precisa ser colocado em uma tag <script type="text/babel">. Tudo dentro desta tag será transformado em Javascript simples usando a biblioteca Babel (que precisa ser incluída além das bibliotecas React).

Então, finalmente, o exemplo acima se torna:

<!DOCTYPE html>
<html>
  <head>
    <meta charset="UTF-8" />
    <title>Hello React!</title>

    <!-- Include the React and ReactDOM libraries -->
    <script src="https://fb.me/react-15.2.1.js"></script>
    <script src="https://fb.me/react-dom-15.2.1.js"></script>
    <!-- Include the Babel library -->
    <script src="https://npmcdn.com/babel-core@5.8.38/browser.min.js"></script>

  </head>
  <body>
   <div id="example"></div>

    <script type="text/babel">

      // create a React element rElement using JSX
      var rElement = <h1>Hello, world!</h1>;

      // dElement is a DOM container
      var dElement = document.getElementById('example');

      // render the React element in the DOM container
      ReactDOM.render(rElement, dElement);

    </script>

  </body>
</html>

Componente Olá Mundo #

Um componente React pode ser definido como uma classe ES6 que estende a classe base React.Component. Em sua forma mínima, um componente deve definir um método render que especifica como o componente renderiza para o DOM. O método render retorna nós React, que podem ser definidos usando a sintaxe JSX como tags semelhantes a HTML. O exemplo a seguir mostra como definir um Component mínimo:

import React from 'react'

class HelloWorld extends React.Component {
    render() {
        return <h1>Hello, World!</h1>
    }
}

export default HelloWorld

Um Componente também pode receber props. Estas são propriedades passadas por seu pai para especificar alguns valores que o componente não pode conhecer por si mesmo; uma propriedade também pode conter uma função que pode ser chamada pelo componente após a ocorrência de certos eventos - por exemplo, um botão pode receber uma função para sua propriedade onClick e chamá-la sempre que for clicado. Ao escrever um componente, seus props podem ser acessados ​​através do objeto props no próprio Component:

import React from 'react'

class Hello extends React.Component {
    render() {
        return <h1>Hello, {this.props.name}!</h1>
    }
}

export default Hello

O exemplo acima mostra como o componente pode renderizar uma string arbitrária passada para a prop name por seu pai. Observe que um componente não pode modificar as props que recebe.

Um componente pode ser renderizado dentro de qualquer outro componente, ou diretamente no DOM se for o componente mais alto, usando ReactDOM.render e fornecendo a ele o componente e o DOM Node onde você deseja que a árvore React seja renderizada:

import React from 'react'
import ReactDOM from 'react-dom'
import Hello from './Hello'

ReactDOM.render(<Hello name="Billy James" />, document.getElementById('main'))

Até agora você sabe como fazer um componente básico e aceitar props. Vamos dar um passo adiante e apresentar o state.

Para fins de demonstração, vamos fazer com que nosso aplicativo Hello World exiba apenas o primeiro nome se for fornecido um nome completo.

import React from 'react'

class Hello extends React.Component {

    constructor(props){

        //Since we are extending the default constructor,
        //handle default activities first.
        super(props);

        //Extract the first-name from the prop
        let firstName = this.props.name.split(" ")[0];
        
        //In the constructor, feel free to modify the
        //state property on the current context.
        this.state = {
            name: firstName
        }

    } //Look maa, no comma required in JSX based class defs!

    render() {
        return <h1>Hello, {this.state.name}!</h1>
    }
}

export default Hello

Observação: cada componente pode ter seu próprio estado ou aceitar o estado do pai como um prop.
