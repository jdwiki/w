JSX é uma etapa do pré-processador que adiciona a sintaxe XML ao JavaScript. Você pode definitivamente usar o React sem JSX, mas o JSX torna o React muito mais elegante.

Assim como XML, as tags JSX têm um nome de tag, atributos e filhos. Se um valor de atributo estiver entre aspas, o valor será uma string. Caso contrário, coloque o valor entre chaves e o valor será a expressão JavaScript incluída.

Fundamentalmente, JSX apenas fornece açúcar sintático para a função React.createElement(component, props, ...children).

Então, o seguinte código JSX:

class HelloMessage extends React.Component {
  render() {
    return <div>Hello {this.props.name}</div>;
  }
}

ReactDOM.render(<HelloMessage name="Kalo" />, mountNode);

Compila para o seguinte código JavaScript:

class HelloMessage extends React.Component {
  render() {
    return React.createElement(
      "div",
      null,
      "Hello ",
      this.props.name
    );
  }
}

ReactDOM.render(React.createElement(HelloMessage, { name: "Kalo" }), mountNode);

Concluindo, observe que a linha a seguir em JSX não é uma string nem HTML:

const element = <h1>Hello, world!</h1>;

Chama-se JSX e é uma extensão de sintaxe para JavaScript. O JSX pode lembrá-lo de uma linguagem de modelo, mas vem com todo o poder do JavaScript.

A equipe do React diz em seus documentos que eles recomendam usá-lo para descrever como a interface do usuário deve ser.

Adereços em JSX #

Existem várias maneiras diferentes de especificar props em JSX.

Expressões JavaScript #

Você pode passar qualquer expressão JavaScript como prop, cercando-a com {}. Por exemplo, neste JSX:

<MyComponent count={1 + 2 + 3 + 4} />

Dentro do MyComponent, o valor de props.count será 10, pois a expressão 1 + 2 + 3 + 4 é avaliada.

Se instruções e loops for não são expressões em JavaScript, então eles não podem ser usados ​​diretamente em JSX.

String Literais #

Claro, você pode simplesmente passar qualquer string literal como um prop também. Essas duas expressões JSX são equivalentes:

<MyComponent message="hello world" />

<MyComponent message={'hello world'} />

Quando você passa um literal de string, seu valor não tem escape em HTML. Portanto, essas duas expressões JSX são equivalentes:

<MyComponent message="&lt;3" />

<MyComponent message={'<3'} />

Esse comportamento geralmente não é relevante. Só é mencionado aqui para completar.

Valor padrão dos adereços #

Se você não passar nenhum valor para uma prop, o padrão é true. Essas duas expressões JSX são equivalentes:

<MyTextBox autocomplete />

<MyTextBox autocomplete={true} />

No entanto, a equipe do React diz em seus documentos usar essa abordagem não é recomendado, porque pode ser confundido com a abreviação do objeto ES6 {foo} que é a abreviação de {foo: foo} em vez de { foo: true}. Eles dizem que esse comportamento está lá apenas para que corresponda ao comportamento do HTML.

Espalhar Atributos #

Se você já tem props como um objeto e deseja passá-lo em JSX, você pode usar ... como um operador de propagação para passar todo o objeto props. Esses dois componentes são equivalentes:

function Case1() {
  return <Greeting firstName="Kaloyab" lastName="Kosev" />;
}

function Case2() {
  const person = {firstName: 'Kaloyan', lastName: 'Kosev'};
  return <Greeting {...person} />;
}

Crianças em JSX #

Em expressões JSX que contêm uma tag de abertura e uma tag de fechamento, o conteúdo entre essas tags é passado como uma prop especial: props.children. Existem várias maneiras diferentes de passar crianças:

String Literais #

Você pode colocar uma string entre as tags de abertura e fechamento e props.children será apenas essa string. Isso é útil para muitos dos elementos HTML internos. Por exemplo:

<MyComponent>
    <h1>Hello world!</h1>
</MyComponent>

Este é um JSX válido, e props.children em MyComponent será simplesmente <h1>Hello world!</h1>.

Observe que o HTML não tem escape, então você geralmente pode escrever JSX da mesma forma que escreveria HTML.

Tenha em mente que, neste caso, JSX:

  • remove espaços em branco no início e no final de uma linha;
  • remove linhas em branco;
  • novas linhas adjacentes às tags são removidas;
  • novas linhas que ocorrem no meio de literais de string são condensadas em um único espaço.

Filhos JSX #

Você pode fornecer mais elementos JSX como filhos. Isso é útil para exibir componentes aninhados:

<MyContainer>
  <MyFirstComponent />
  <MySecondComponent />
</MyContainer>

Você pode misturar diferentes tipos de filhos, para poder usar literais de string junto com filhos JSX. Esta é outra maneira em que o JSX é como o HTML, de modo que este é um JSX válido e um HTML válido:

<div>
  <h2>Here is a list</h2>
  <ul>
    <li>Item 1</li>
    <li>Item 2</li>
  </ul>
</div>

Observe que um componente React não pode retornar vários elementos React, mas uma única expressão JSX pode ter vários filhos. Então, se você quiser que um componente renderize várias coisas, você pode envolvê-las em um div como no exemplo acima.

Expressões JavaScript #

Você pode passar qualquer expressão JavaScript como filha, colocando-a dentro de {}. Por exemplo, essas expressões são equivalentes:

<MyComponent>foo</MyComponent>

<MyComponent>{'foo'}</MyComponent>

Isso geralmente é útil para renderizar uma lista de expressões JSX de tamanho arbitrário. Por exemplo, isso renderiza uma lista HTML:

const Item = ({ message }) => (
  <li>{ message }</li>
);

const TodoList = () => {
  const todos = ['finish doc', 'submit review', 'wait stackoverflow review'];
  return (
    <ul>
      { todos.map(message => (<Item key={message} message={message} />)) }
    </ul>
  );
};

Observe que as expressões JavaScript podem ser misturadas com outros tipos de filhos.

Funciona como Filhos #

Normalmente, as expressões JavaScript inseridas no JSX serão avaliadas como uma string, um elemento React ou uma lista dessas coisas. No entanto, props.children funciona como qualquer outro prop, pois pode passar qualquer tipo de dados, não apenas os tipos que o React sabe como renderizar. Por exemplo, se você tiver um componente personalizado, poderá fazer com que ele receba um retorno de chamada como props.children:

const ListOfTenThings = () => (
  <Repeat numTimes={10}>
    {(index) => <div key={index}>This is item {index} in the list</div>}
  </Repeat>
);

// Calls the children callback numTimes to produce a repeated component
const Repeat = ({ numTimes, children }) => {
  let items = [];
  for (let i = 0; i < numTimes; i++) {
    items.push(children(i));
  }
  return <div>{items}</div>;
};

Os filhos passados ​​para um componente personalizado podem ser qualquer coisa, desde que esse componente os transforme em algo que o React possa entender antes da renderização. Esse uso não é comum, mas funciona se você quiser estender o que o JSX é capaz.

Valores ignorados #

Observe que false, null, undefined e true são filhos válidos. Mas eles simplesmente não renderizam. Todas essas expressões JSX renderizarão a mesma coisa:

<MyComponent />

<MyComponent></MyComponent>

<MyComponent>{false}</MyComponent>

<MyComponent>{null}</MyComponent>

<MyComponent>{true}</MyComponent>

Isso é extremamente útil para renderizar condicionalmente elementos React. Este JSX só renderiza um

se showHeader for true:

<div>
  {showHeader && <Header />}
  <Content />
</div>

Uma advertência importante é que alguns valores “falsos”, como o número 0, ainda são renderizados pelo React. Por exemplo, este código não se comportará como você poderia esperar porque 0 será impresso quando props.messages for um array vazio:

<div>
  {props.messages.length &&
    <MessageList messages={props.messages} />
  }
</div>

Uma abordagem para corrigir isso é garantir que a expressão antes do && seja sempre booleana:

<div>
  {props.messages.length > 0 &&
    <MessageList messages={props.messages} />
  }
</div>

Por fim, tenha em mente que, se você quiser que um valor como false, true, null ou undefined apareça na saída, você deve convertê-lo em uma string primeiro:

<div>
  My JavaScript variable is {String(myVariable)}.
</div>
