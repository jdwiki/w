Configurando o projeto #

Você precisa do Node Package Manager para instalar as dependências do projeto. Baixe o nó para seu sistema operacional em Nodejs.org. O Node Package Manager vem com node.

Você também pode usar o Node Version Manager para gerenciar melhor suas versões de node e npm. É ótimo para testar seu projeto em diferentes versões de nós. No entanto, não é recomendado para ambiente de produção.

Depois de instalar o node em seu sistema, vá em frente e instale alguns pacotes essenciais para lançar seu primeiro projeto React usando Babel e Webpack.

Antes de começarmos a pressionar comandos no terminal. Veja para que Babel e Webpack são usados.

Você pode iniciar seu projeto executando npm init em seu terminal. Siga a configuração inicial. Depois disso, execute os seguintes comandos no seu terminal-

Dependências:

npm install react react-dom --save

Dependências do desenvolvedor:

npm install babel-core babel-loader babel-preset-es2015 babel-preset-react babel-preset-stage-0 webpack webpack-dev-server react-hot-loader --save-dev

Dependências de desenvolvedor opcionais:

npm install eslint eslint-plugin-react babel-eslint --save-dev

Você pode consultar este amostra package.json

Crie .babelrc na raiz do seu projeto com o seguinte conteúdo:

{
  "presets": ["es2015", "stage-0", "react"]
}

Opcionalmente, crie .eslintrc na raiz do seu projeto com o seguinte conteúdo:

{
  "ecmaFeatures": {
    "jsx": true,
    "modules": true
  },
  "env": {
    "browser": true,
    "node": true
  },
  "parser": "babel-eslint",
  "rules": {
    "quotes": [2, "single"],
    "strict": [2, "never"],
  },
  "plugins": [
    "react"
  ]
}

Crie um arquivo .gitignore para evitar o upload de arquivos gerados para seu repositório git.

node_modules
npm-debug.log
.DS_Store
dist

Crie o arquivo webpack.config.js com o seguinte conteúdo mínimo.

var path = require('path');
var webpack = require('webpack');

module.exports = {
  devtool: 'eval',
  entry: [
    'webpack-dev-server/client?http://localhost:3000',
    'webpack/hot/only-dev-server',
    './src/index'
  ],
  output: {
    path: path.join(__dirname, 'dist'),
    filename: 'bundle.js',
    publicPath: '/static/'
  },
  plugins: [
    new webpack.HotModuleReplacementPlugin()
  ],
  module: {
    loaders: [{
      test: /\.js$/,
      loaders: ['react-hot', 'babel'],
      include: path.join(__dirname, 'src')
    }]
  }
};

E finalmente, crie um arquivo sever.js para poder executar o npm start, com o seguinte conteúdo:

var webpack = require('webpack');
var WebpackDevServer = require('webpack-dev-server');
var config = require('./webpack.config');

new WebpackDevServer(webpack(config), {
  publicPath: config.output.publicPath,
  hot: true,
  historyApiFallback: true
}).listen(3000, 'localhost', function (err, result) {
  if (err) {
    return console.log(err);
  }

  console.log('Serving your awesome project at http://localhost:3000/');
});

Crie o arquivo src/app.js para ver seu projeto React fazer algo.

import React, { Component } from 'react';

export default class App extends Component {
  render() {
    return (
      <h1>Hello, world.</h1>
    );
  }
}

Execute node server.js ou npm start no terminal, se você definiu o que start significa em seu package.json

projeto react-starter #

Sobre este projeto

Este é um projeto clichê simples. Este post irá guiá-lo para configurar o ambiente para ReactJs + Webpack + Bable.

Vamos começar

precisaremos do gerenciador de pacotes do nó para iniciar o servidor expresso e gerenciar as dependências em todo o projeto. se você é novo no gerenciador de pacotes do nó, pode verificar aqui. Nota: A instalação do gerenciador de pacotes do nó é necessária aqui.

Crie uma pasta com o nome adequado e navegue para ela a partir do terminal ou pela GUI. Em seguida, vá para o terminal e digite npm init isso criará um arquivo package.json, Nada assustador, ele fará algumas perguntas como o nome do seu projeto, versão, descrição, ponto de entrada, repositório git, autor, licença, etc. Aqui o ponto de entrada é importante porque o nó irá inicialmente procurá-lo quando você executar o projeto. No final, ele solicitará que você verifique as informações fornecidas. Você pode digitar sim ou modificá-lo. Bem, é isso, nosso arquivo package.json está pronto.

Configuração do servidor expresso execute npm install express@4 –save. Estas são todas as dependências que precisamos para este projeto. Aqui é importante salvar o sinalizador, sem ele o arquivo package.js não será atualizado. A principal tarefa do package.json é armazenar a lista de dependências. Ele adicionará a versão expressa 4. Seu package.json será parecido com "dependencies": { "express": "^4.13.4", ............. },

Após o download completo, você pode ver que existe a pasta node_modules e a subpasta de nossas dependências. Agora na raiz do projeto crie um novo arquivo server.js. Agora estamos configurando o servidor expresso. Vou passar todo o código e explicar mais tarde.

var express = require('express');
// Create our app
var app = express();

app.use(express.static('public'));

app.listen(3000, function () {
  console.log('Express server is using port:3000');
});

var express = require(’express’); isso lhe dará acesso a toda a API expressa.

var app = express(); chamará a biblioteca expressa como função. app.use(); deixe adicionar a funcionalidade ao seu aplicativo expresso. app.use(express.static(‘public’)); especificará o nome da pasta que será exposta em nosso servidor web. app.listen(port, function(){}) aqui, nossa porta será 3000 e a função que estamos chamando verificará se nosso servidor web está funcionando corretamente. É isso o servidor expresso está configurado.

Agora vá para o nosso projeto e crie uma nova pasta public e crie o arquivo index.html. index.html é o arquivo padrão para seu aplicativo e o servidor Express irá procurar por este arquivo. O index.html é um arquivo html simples que se parece com

<!DOCTYPE html>
<html>

<head>
  <meta charset="UTF-8"/>
</head>

<body>
  <h1>hello World</h1>
</body>

</html>

E vá para o caminho do projeto pelo terminal e digite node server.js. Então você verá * console.log(‘Express server is using port:3000’);*.

Vá para o navegador e digite http://localhost:3000 na barra de navegação, você verá hello World.

Agora entre na pasta pública e crie um novo arquivo app.jsx. JSX é uma etapa do pré-processador que adiciona a sintaxe XML ao seu JavaScript. Você pode definitivamente usar o React sem JSX, mas o JSX torna o React muito mais elegante. Aqui está o código de exemplo para app.jsx

ReactDOM.render(
  <h1>Hello World!!!</h1>,
  document.getElementById('app')
);

Agora vá para index.html e modifique o código, deve ficar assim

<!DOCTYPE html>
<html>

<head>
  <meta charset="UTF-8"/>
  <script src="https://cdnjs.cloudflare.com/ajax/libs/babel-core/5.8.23                                              /browser.min.js"></script>
  <script src="https://cdnjs.cloudflare.com/ajax/libs/react/0.14.7/react.js">                           </script>
  <script src="https://cdnjs.cloudflare.com/ajax/libs/react/0.14.7/react-dom.js">    </script>
</head>

<body>
  <div id="app"></div>

  <script type="text/babel" src="app.jsx"></script>
</body>

</html>

Com isso no lugar, tudo está pronto, espero que você ache simples.
