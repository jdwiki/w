Às vezes, é útil obter as propriedades do maven, como a versão atual, no código. Aqui estão algumas maneiras de fazer isso.

Obtendo o número da versão de dentro de um jar #

Se você empacotar seu aplicativo em um jar usando o maven-jar-plugin ou o maven-assembly-plugin, uma maneira fácil de obter a versão atual do pom é adicionar uma entrada no manifesto, que é então disponível em Java.

O segredo é definir o sinalizador addDefaultImplementationEntries para true (e o addDefaultSpecificationEntries é que você também precisa do id do artefato).

configuração do plugin jar:

<build>
  <plugins>
    <plugin>
      <groupId>org.apache.maven.plugins</groupId>
      <artifactId>maven-jar-plugin</artifactId>
      <configuration>
        <archive>
          <manifest>
            <mainClass>...</mainClass>
            <addDefaultImplementationEntries>
              true
            </addDefaultImplementationEntries>
          </manifest>
        </archive>
      </configuration>
    </plugin>
  </plugins>
</build>

configuração do plugin de montagem:

<plugin>
    <groupId>org.apache.maven.plugins</groupId>
    <artifactId>maven-assembly-plugin</artifactId>
    <configuration>
        <descriptorRefs>
            <descriptorRef>jar-with-dependencies</descriptorRef>
        </descriptorRefs>
        <archive>
            <manifest>  
              <addDefaultImplementationEntries>true</addDefaultImplementationEntries> 
            </manifest>
        </archive>
    </configuration>
    <executions>
        <execution .../>
    </executions>
</plugin>

addDefaultImplementationEntries instrui o Maven a adicionar os seguintes cabeçalhos ao MANIFEST.MF do seu jar:

Implementation-Title: display-version
Implementation-Version: 1.0-SNAPSHOT
Implementation-Vendor-Id: test

Agora você pode usar esta linha de código em qualquer lugar do seu jar para acessar o número da versão:

getClass().getPackage().getImplementationVersion()

Mantendo um arquivo de propriedades em sincronia usando o mecanismo de filtragem de propriedades do maven #

Como esta documentação explica,

Às vezes, um arquivo de recurso precisará conter um valor que só pode ser fornecido no momento da compilação. Para fazer isso no Maven, coloque uma referência à propriedade que conterá o valor em seu arquivo de recurso usando a sintaxe ${<nome da propriedade>}. A propriedade pode ser um dos valores definidos em seu pom.xml, um valor definido no settings.xml do usuário, uma propriedade definida em um arquivo de propriedades externo ou uma propriedade do sistema.

Como exemplo, vamos criar um info.txt simples em src/main/resources contendo a versão do pom e o tempo de compilação.

  1. crie um src/main/resources/info.txt com o seguinte conteúdo:

    version=${pom.version} build.date=${timestamp}

  2. peça ao Maven para expandir as propriedades configurando filtering para true:

     <build>
     <resources>
         <resource>
            <directory>src/main/resources</directory>
            <filtering>true</filtering>
         </resource>
     </resources>
 </build>

  3. com isso, a versão será atualizada, mas infelizmente um bug no Maven impede que a propriedade ${maven.build.timestamp} seja passada para o mecanismo de filtragem de recursos (mais informações [aqui](https://dzone .com/articles/stamping-version-number-and)). Então, vamos criar uma propriedade timestamp como solução alternativa! Adicione o seguinte às propriedades do pom:

     <properties>
     <timestamp>${maven.build.timestamp}</timestamp>
     <maven.build.timestamp.format>yyyy-MM-dd'T'HH:mm</maven.build.timestamp.format>  
 </properties>

  4. execute o maven, você deve encontrar um info.txt em target/classes com um conteúdo como:

     version=0.3.2
 build.date=2017-04-20T13:56

Lendo um pom.xml em tempo de execução usando o plugin maven-model #

Os outros exemplos podem ser a melhor e mais estável maneira de obter um número de versão em um aplicativo estaticamente. Esta resposta propõe uma alternativa mostrando como fazer isso dinamicamente durante o tempo de execução, usando a biblioteca maven maven-model.

Adicione a dependência:

<dependency>
  <groupId>org.apache.maven</groupId>
  <artifactId>maven-model</artifactId>
  <version>3.3.9</version>
</dependency>

Em Java, crie um MavenXpp3Reader para ler seu pom. Por exemplo:

package de.scrum_master.app;

import org.apache.maven.model.Model;
import org.apache.maven.model.io.xpp3.MavenXpp3Reader;
import org.codehaus.plexus.util.xml.pull.XmlPullParserException;

import java.io.FileReader;
import java.io.IOException;

public class MavenModelExample {
    public static void main(String[] args) throws IOException, XmlPullParserException {
        MavenXpp3Reader reader = new MavenXpp3Reader();
        Model model = reader.read(new FileReader("pom.xml"));
        System.out.println(model.getId());
        System.out.println(model.getGroupId());
        System.out.println(model.getArtifactId());
        System.out.println(model.getVersion());
    }
}

O log do console é o seguinte:

de.scrum-master.stackoverflow:my-artifact:jar:1.0-SNAPSHOT
de.scrum-master.stackoverflow
my-artifact
1.0-SNAPSHOT
