Instalação ou configuração #

Versões binárias do Maven podem ser baixadas [do site do Maven] (https://maven.apache.org/download.cgi).

O binário vem como um arquivo zip ou como um arquivo tar.gz. Após baixá-lo, as instruções da página de instalação podem ser seguidas:

  • Certifique-se de que a variável de ambiente JAVA_HOME esteja definida e aponte para sua instalação JDK (não JRE). Por exemplo, em uma máquina Windows, esta pasta de instalação pode corresponder a C:\Program Files\Java\jdk1.8.0_51.
  • Extraia o arquivo de distribuição no diretório de sua escolha.
  • Adicione o diretório bin do diretório criado (chamado apache-maven-3.3.9 para Maven 3.3.9) à variável de ambiente PATH. (Referência a [alterar no Windows](http://superuser.com/questions/284342/what-are-path-and-other-environment-variables-and-how-can-i-set-or-use- eles)).
  • Verifique se a configuração está correta executando mvn -version na linha de comando.

Não há necessidade de definir a variável de ambiente M2_HOME ou MAVEN_HOME.

Definindo as configurações de proxy #

Se sua conexão com a Internet for fornecida por meio de um proxy, o Maven não poderá baixar jars de repositórios remotos - um problema comum enfrentado pelas empresas.

Para resolver isso, o Maven precisa receber os detalhes e credenciais do proxy indo para {Maven install location} → conf → settings.xml. Role para baixo até a tag <proxies> e insira os detalhes aqui, usando o formato mencionado nos comentários.

Para usuários do Eclipse

O Eclipse usa seu próprio arquivo settings.xml para executar o Maven, cuja localização pode ser encontrada no menu Window → Preferences → Maven → User Settings → User Settings:. Se o arquivo não estiver disponível no local mencionado, basta criá-lo você mesmo ou criar uma duplicata do arquivo do local acima {Maven install location} → conf → settings.xml.

Para usuários do IntelliJ

Abra as configurações e navegue até Maven -> Importando. (Isso pode ser aninhado em Build, Execution, Deployment -> Build Tools ->, dependendo da versão do IntelliJ que você está usando.)

Defina o campo chamado “Opções de VM para importador” como:

-DproxySet=true -DproxyHost=<HOST> -DproxyPort=<PORT>    
-DproxySet=true -DproxyHost=myproxy.com -DproxyPort=8080

Aplique e reinicie o IntelliJ.

Instalação no Mac OSX com Brew #

  1. Em um terminal execute brew install maven
  2. Quando a instalação terminar, verifique se o maven funciona corretamente com mvn -v. A saída deve ser algo como:
Apache Maven 3.3.9
Página inicial do Maven: /usr/local/Cellar/maven/3.3.9/libexec
Versão Java: 1.8.0_121, fornecedor: Oracle Corporation
Java home: /Library/Java/JavaVirtualMachines/jdk1.8.0_121.jdk/Contents/Home/jre
Locale padrão: en_US, codificação de plataforma: UTF-8
Nome do SO: "mac os x", versão: "10.12.4", arch: "x86_64", família: "mac"

Se isso não funcionar, certifique-se de ter um JDK instalado em seu ambiente javac -version

Instalação no Ubuntu #

  1. Em um terminal execute sudo apt-get install maven

  2. Quando a instalação terminar, verifique se está funcionando corretamente com mvn -v, a saída deve ser semelhante a:

     Apache Maven 3.3.9
 Maven home: /usr/share/maven
 Java version: 1.8.0_121, vendor: Oracle Corporation
 Java home: /usr/lib/jvm/java-8-openjdk-amd64/jre
 Default locale: en_US, platform encoding: UTF-8
 OS name: "linux", version: "4.8.0-parrot-amd64", arch: "amd64", family: "unix"

Se isso não funcionar, certifique-se de ter um JDK instalado em seu ambientejavac -version
