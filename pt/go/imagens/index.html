O pacote image fornece funcionalidades básicas para trabalhar com imagem 2-D. Este tópico descreve várias operações básicas ao trabalhar com imagem, como ler e gravar um formato de imagem específico, cortar, acessar e modificar pixel, conversão de cores, redimensionamento e filtragem básica de imagem.

Conceitos Básicos #

Uma imagem representa uma grade retangular de elementos de imagem (pixel). No pacote image, o pixel é representado como uma das cores definidas em [image/color](https://golang.org/pkg/image/ cor/) pacote. A geometria 2-D da imagem é representada como image.Rectangle, enquanto [image.Point](https://golang. org/pkg/image/#Point) denota uma posição na grade.

A figura acima ilustra os conceitos básicos de uma imagem no pacote. Uma imagem de tamanho 15x14 pixels tem um limites retangular iniciado no canto superior esquerdo (por exemplo, coordenada (-3, -4) na figura acima) e seus eixos aumentam para a direita e para baixo até o canto inferior direito (por exemplo, coordenada (12, 10) na figura). Observe que os limites não necessariamente começam ou contêm o ponto (0,0).

Relacionado com a imagem tipo #

Em Go, uma imagem sempre implementa a seguinte interface image.Image

type Image interface {
    // ColorModel returns the Image's color model.
    ColorModel() color.Model
    // Bounds returns the domain for which At can return non-zero color.
    // The bounds do not necessarily contain the point (0, 0).
    Bounds() Rectangle
    // At returns the color of the pixel at (x, y).
    // At(Bounds().Min.X, Bounds().Min.Y) returns the upper-left pixel of the grid.
    // At(Bounds().Max.X-1, Bounds().Max.Y-1) returns the lower-right one.
    At(x, y int) color.Color
}

em que a interface color.Color é definida como

type Color interface {
    // RGBA returns the alpha-premultiplied red, green, blue and alpha values
    // for the color. Each value ranges within [0, 0xffff], but is represented
    // by a uint32 so that multiplying by a blend factor up to 0xffff will not
    // overflow.
    //
    // An alpha-premultiplied color component c has been scaled by alpha (a),
    // so has valid values 0 <= c <= a.
    RGBA() (r, g, b, a uint32)
}

e color.Model é uma interface declarada como

type Model interface {
    Convert(c Color) Color
}

Acessando a dimensão e o pixel da imagem #

Suponha que tenhamos uma imagem armazenada como variável img, então podemos obter a dimensão e o pixel da imagem por:

// Image bounds and dimension
b := img.Bounds()
width, height := b.Dx(), b.Dy()
// do something with dimension ...

// Corner co-ordinates
top := b.Min.Y
left := b.Min.X
bottom := b.Max.Y
right := b.Max.X
    
// Accessing pixel. The (x,y) position must be
// started from (left, top) position not (0,0)
for y := top; y < bottom; y++ {
    for x := left; x < right; x++ {
        cl := img.At(x, y)
        r, g, b, a := cl.RGBA()
        // do something with r,g,b,a color component
    }
}

Observe que no pacote, o valor de cada componente R,G,B,A está entre 0-65535 (0x0000 - 0xffff), não 0-255.

Carregando e salvando a imagem #

Na memória, uma imagem pode ser vista como uma matriz de pixel (cor). No entanto, quando uma imagem é armazenada em um armazenamento permanente, ela pode ser armazenada como está (formato RAW), Bitmap ou outros formatos de imagem com algoritmo de compactação específico para economizando espaço de armazenamento, por ex. PNG, JPEG, GIF, etc. Ao carregar uma imagem com um formato específico, a imagem deve ser decodificada para image.Image com o algoritmo correspondente. Uma função image.Decode declarada como

func Decode(r io.Reader) (Image, string, error)

é fornecido para este uso específico. Para poder lidar com vários formatos de imagem, antes de chamar a função image.Decode, o decodificador deve ser registrado através de image.RegisterFormat função definida como

func RegisterFormat(name, magic string, 
    decode func(io.Reader) (Image, error), decodeConfig func(io.Reader) (Config, error))

Atualmente, o pacote de imagens suporta três formatos de arquivo: JPEG, GIF e PNG. Para registrar um decodificador, adicione o seguinte

import _ "image/jpeg"    //register JPEG decoder

para o pacote main da aplicação. Em algum lugar do seu código (não é necessário no pacote main), para carregar uma imagem JPEG, use os seguintes snippets:

f, err := os.Open("inputimage.jpg")
if err != nil {
    // Handle error
}
defer f.Close()

img, fmtName, err := image.Decode(f)
if err != nil {
    // Handle error
}

// `fmtName` contains the name used during format registration
// Work with `img` ...

Salvar em PNG #

Para salvar uma imagem em um formato específico, o codificador correspondente deve ser importado explicitamente, ou seja,

import "image/png"    //needed to use `png` encoder

então uma imagem pode ser salva com os seguintes trechos:

f, err := os.Create("outimage.png")
if err != nil {
    // Handle error
}
defer f.Close()

// Encode to `PNG` with `DefaultCompression` level
// then save to file
err = png.Encode(f, img)
if err != nil {
    // Handle error
}

Se você deseja especificar um nível de compactação diferente do nível DefaultCompression, crie um encoder, por exemplo.

enc := png.Encoder{
    CompressionLevel: png.BestSpeed,
}
err := enc.Encode(f, img)

Salvar em JPEG #

Para salvar no formato jpeg, use o seguinte:

import "image/jpeg"

// Somewhere in the same package
f, err := os.Create("outimage.jpg")
if err != nil {
    // Handle error
}
defer f.Close()

// Specify the quality, between 0-100
// Higher is better
opt := jpeg.Options{
    Quality: 90,
}
err = jpeg.Encode(f, img, &opt)
if err != nil {
    // Handle error
}

Salvar em GIF #

Para salvar a imagem em um arquivo GIF, use os seguintes snippets.

import "image/gif"

// Samewhere in the same package
f, err := os.Create("outimage.gif")
if err != nil {
    // Handle error
}
defer f.Close()

opt := gif.Options {
    NumColors: 256, 
    // Add more parameters as needed
}

err = gif.Encode(f, img, &opt)
if err != nil {
    // Handle error
}

Cortar imagem #

A maior parte do tipo de imagem no pacote image tem o método SubImage(r Rectangle) Image, exceto image.Uniform. Com base nesse fato, podemos implementar uma função para cortar uma imagem arbitrária da seguinte forma

func CropImage(img image.Image, cropRect image.Rectangle) (cropImg image.Image, newImg bool) {
    //Interface for asserting whether `img`
    //implements SubImage or not.
    //This can be defined globally.
    type CropableImage interface {
        image.Image
        SubImage(r image.Rectangle) image.Image
    }

    if p, ok := img.(CropableImage); ok {
        // Call SubImage. This should be fast,
        // since SubImage (usually) shares underlying pixel.
        cropImg = p.SubImage(cropRect)
    } else if cropRect = cropRect.Intersect(img.Bounds()); !cropRect.Empty() {
        // If `img` does not implement `SubImage`,
        // copy (and silently convert) the image portion to RGBA image.
        rgbaImg := image.NewRGBA(cropRect)
        for y := cropRect.Min.Y; y < cropRect.Max.Y; y++ {
            for x := cropRect.Min.X; x < cropRect.Max.X; x++ {
                rgbaImg.Set(x, y, img.At(x, y))
            }
        }
        cropImg = rgbaImg
        newImg = true
    } else {
        // Return an empty RGBA image
        cropImg = &image.RGBA{}
        newImg = true
    }

    return cropImg, newImg
}

Observe que a imagem cortada pode compartilhar seus pixels subjacentes com a imagem original. Se for esse o caso, qualquer modificação na imagem recortada afetará a imagem original.

Converter imagem colorida em escala de cinza #

Algum algoritmo de processamento de imagem digital, como detecção de borda, informações transportadas pela intensidade da imagem (ou seja, valor em escala de cinza) é suficiente. O uso de informações de cor (canal R, G, B) pode fornecer um resultado ligeiramente melhor, mas a complexidade do algoritmo será aumentada. Assim, neste caso, precisamos converter a imagem colorida para imagem em tons de cinza antes de aplicar tal algoritmo.

O código a seguir é um exemplo de conversão de imagem arbitrária em imagem em tons de cinza de 8 bits. A imagem é recuperada de um local remoto usando o pacote net/http, convertida em escala de cinza e finalmente salva como imagem PNG.

package main

import (
    "image"
    "log"
    "net/http"
    "os"

    _ "image/jpeg"
    "image/png"
)

func main() {
    // Load image from remote through http
    // The Go gopher was designed by Renee French. (http://reneefrench.blogspot.com/)
    // Images are available under the Creative Commons 3.0 Attributions license.
    resp, err := http.Get("http://golang.org/doc/gopher/fiveyears.jpg")
    if err != nil {
        // handle error
        log.Fatal(err)
    }
    defer resp.Body.Close()

    // Decode image to JPEG
    img, _, err := image.Decode(resp.Body)
    if err != nil {
        // handle error
        log.Fatal(err)
    }
    log.Printf("Image type: %T", img)

    // Converting image to grayscale
    grayImg := image.NewGray(img.Bounds())
    for y := img.Bounds().Min.Y; y < img.Bounds().Max.Y; y++ {
        for x := img.Bounds().Min.X; x < img.Bounds().Max.X; x++ {
            grayImg.Set(x, y, img.At(x, y))
        }
    }

    // Working with grayscale image, e.g. convert to png
    f, err := os.Create("fiveyears_gray.png")
    if err != nil {
        // handle error
        log.Fatal(err)
    }
    defer f.Close()

    if err := png.Encode(f, grayImg); err != nil {
        log.Fatal(err)
    }
}

A conversão de cores ocorre ao atribuir pixels por meio de Set(x, y int, c color.Color) que é implementado em [image.go](https://golang.org/src/image/image.go?s =19292:19335#L691) como

func (p *Gray) Set(x, y int, c color.Color) {
    if !(Point{x, y}.In(p.Rect)) {
        return
    }

    i := p.PixOffset(x, y)
    p.Pix[i] = color.GrayModel.Convert(c).(color.Gray).Y
}

em que, color.GrayModel é definido em color.go como

func grayModel(c Color) Color {
    if _, ok := c.(Gray); ok {
        return c
    }
    r, g, b, _ := c.RGBA()

    // These coefficients (the fractions 0.299, 0.587 and 0.114) are the same
    // as those given by the JFIF specification and used by func RGBToYCbCr in
    // ycbcr.go.
    //
    // Note that 19595 + 38470 + 7471 equals 65536.
    //
    // The 24 is 16 + 8. The 16 is the same as used in RGBToYCbCr. The 8 is
    // because the return value is 8 bit color, not 16 bit color.
    y := (19595*r + 38470*g + 7471*b + 1<<15) >> 24

    return Gray{uint8(y)}
}

Com base nos fatos acima, a intensidade ‘Y’ é calculada com a seguinte fórmula:

Luminância: Y = 0,299R + 0,587G + 0,114B

Se quisermos aplicar formula/algorithms diferentes para converter uma cor em intensidade, por exemplo,

Média: Y = (R + G + B) / 3 Luma: Y = 0,2126R + 0,7152G + 0,0722B Brilho: Y = (min(R, G, B) + max(R, G, B))/2

então, os seguintes snippets podem ser usados.

// Converting image to grayscale
grayImg := image.NewGray(img.Bounds())
for y := img.Bounds().Min.Y; y < img.Bounds().Max.Y; y++ {
    for x := img.Bounds().Min.X; x < img.Bounds().Max.X; x++ {
        R, G, B, _ := img.At(x, y).RGBA()
        //Luma: Y = 0.2126*R + 0.7152*G + 0.0722*B
        Y := (0.2126*float64(R) + 0.7152*float64(G) + 0.0722*float64(B)) * (255.0 / 65535)
        grayPix := color.Gray{uint8(Y)}
        grayImg.Set(x, y, grayPix)
    }
}

O cálculo acima é feito por multiplicação de ponto flutuante, e certamente não é o mais eficiente, mas é suficiente para demonstrar a ideia. O outro ponto é que, ao chamar Set(x, y int, c color.Color) com color.Gray como terceiro argumento, o modelo de cores não realizará a conversão de cores como pode ser visto na função grayModel anterior .
