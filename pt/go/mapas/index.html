Os mapas são tipos de dados usados ​​para armazenar pares de valores-chave não ordenados, de modo que pesquisar o valor associado a uma determinada chave é muito eficiente. As chaves são únicas. A estrutura de dados subjacente cresce conforme necessário para acomodar novos elementos, de modo que o programador não precisa se preocupar com o gerenciamento de memória. Eles são semelhantes ao que outras linguagens chamam de tabelas de hash, dicionários ou matrizes associativas.

Sintaxe #

  • var mapName map[KeyType]ValueType // declara um mapa
  • var mapName = map[KeyType]ValueType{} // declara e atribui um mapa vazio
  • var mapName = map[KeyType]ValueType{key1: val1, key2: val2} // declara e atribui um Map
  • mapName := make(map[KeyType]ValueType) // declara e inicializa o mapa de tamanho padrão
  • mapName := make(map[KeyType]ValueType, length) // declara e inicializa length size map
  • mapName := map[KeyType]ValueType{} // declara automaticamente e atribui um mapa vazio com :=
  • mapName := map[KeyType]ValueType{key1: value1, key2: value2} // declara automaticamente e atribui um mapa com :=
  • value := mapName[key] // Obtém valor por chave
  • value, hasKey := mapName[key] // Obtém valor por chave, ‘hasKey’ é ’true’ se a chave existir no mapa
  • mapName[key] = value // Definir valor por chave

Go fornece um tipo map embutido que implementa uma tabela de hash. Os mapas são o tipo de dados associativo integrado do Go (também chamado de hashes ou dictionaries em outros idiomas).

Declarando e inicializando um mapa #

Você define um mapa usando a palavra-chave map, seguida pelos tipos de suas chaves e seus valores:

// Keys are ints, values are ints.
var m1 map[int]int // initialized to nil

// Keys are strings, values are ints.
var m2 map[string]int // initialized to nil

Os mapas são tipos de referência e, uma vez definidos, eles têm um [valor zero de nil](https://www.wikiod.com/pt/go/mapas#Valor zero de um mapa 201607220859253128466). Gravações em mapas nil causarão pânico e as leituras sempre retornarão o valor zero.

Para inicializar um mapa, use a função make:

m := make(map[string]int)

Com a forma de dois parâmetros de make, é possível especificar uma capacidade de entrada inicial para o mapa, substituindo a capacidade padrão:

m := make(map[string]int, 30)

Como alternativa, você pode declarar um mapa, inicializando-o com seu valor zero e, em seguida, atribuir um valor literal a ele posteriormente, o que ajuda se você empacotar a estrutura em json, produzindo assim um mapa vazio no retorno.

m := make(map[string]int, 0)

Você também pode fazer um mapa e definir seu valor inicial com chaves ({}).

var m map[string]int = map[string]int{"Foo": 20, "Bar": 30}

fmt.Println(m["Foo"]) // outputs 20

Todas as instruções a seguir fazem com que a variável seja vinculada ao mesmo valor.

// Declare, initializing to zero value, then assign a literal value.
var m map[string]int
m = map[string]int{}

// Declare and initialize via literal value.
var m = map[string]int{}

// Declare via short variable declaration and initialize with a literal value.
m := map[string]int{}

Também podemos usar um map literal para [criar um novo mapa com alguns pares iniciais de chave/valor](https://www.wikiod.com/pt/go/mapas#Criando um mapa) .

O tipo de chave pode ser qualquer tipo comparável; notavelmente, isso exclui funções, mapas e fatias. O tipo de valor pode ser qualquer tipo, incluindo tipos personalizados ou interface{}.

type Person struct {
    FirstName string
    LastName  string
}

// Declare via short variable declaration and initialize with make.
m := make(map[string]Person)

// Declare, initializing to zero value, then assign a literal value.
var m map[string]Person
m = map[string]Person{}

// Declare and initialize via literal value.
var m = map[string]Person{}

// Declare via short variable declaration and initialize with a literal value.
m := map[string]Person{}

Verifica o elemento em um mapa #

Para obter um valor do mapa, basta fazer algo como:00

value := mapName[ key ]

Se o mapa contiver a chave, ele retornará o valor correspondente. Se não, ele retorna o valor zero do tipo de valor do mapa (0 se mapa de valores int, "" se mapa de valores string…)

m  := map[string]string{"foo": "foo_value", "bar": ""}
k  := m["foo"]  // returns "foo_value" since that is the value stored in the map
k2 := m["bar"]  // returns "" since that is the value stored in the map
k3 := m["nop"]  // returns "" since the key does not exist, and "" is the string type's zero value

Para diferenciar entre valores vazios e chaves inexistentes, você pode usar o segundo valor retornado do acesso ao mapa (usando como value, hasKey := map["key"]).

Este segundo valor é do tipo boolean e será:

  • true quando o valor está no mapa,
  • false quando o mapa não contém a chave fornecida.

Observe o seguinte exemplo:

value, hasKey = m[ key ]
if hasKey {
    // the map contains the given key, so we can safely use the value
    // If value is zero-value, it's because the zero-value was pushed to the map
} else {
    // The map does not have the given key
    // the value will be the zero-value of the map's type
}

Contando elementos do mapa #

A função interna len retorna o número de elementos em um map

m := map[string]int{}
len(m) // 0

m["foo"] = 1
len(m) // 1

Se uma variável apontar para um mapa nil, então len retornará 0.

var m map[string]int
len(m) // 0

Valor zero de um mapa #

O valor zero de um map é nil e tem um comprimento de 0.

var m map[string]string
fmt.Println(m == nil) // true
fmt.Println(len(m) ==0) // true

Um mapa nil não tem chaves nem chaves podem ser adicionadas. Um mapa nil se comporta como um mapa vazio se lido, mas causa um pânico em tempo de execução se gravado.

var m map[string]string

// reading
m["foo"] == "" // true. Remember "" is the zero value for a string
_, ok = m["foo"] // ok == false

// writing
m["foo"] = "bar" // panic: assignment to entry in nil map

Você não deve tentar ler ou gravar em um mapa de valor zero. Em vez disso, inicialize o mapa (com make ou atribuição) antes de usá-lo.

var m map[string]string
m = make(map[string]string) // OR m = map[string]string{}
m["foo"] = "bar"

Acesso simultâneo de mapas #

Os mapas em movimento não são seguros para simultaneidade. Você deve usar um cadeado para ler e escrever neles se for acessá-los simultaneamente. Normalmente a melhor opção é usar sync.RWMutex porque você pode ter bloqueios de leitura e escrita. No entanto, um sync.Mutex também pode ser usado.

type RWMap struct {
    sync.RWMutex
    m map[string]int
}

// Get is a wrapper for getting the value from the underlying map
func (r RWMap) Get(key string) int {
    r.RLock()
    defer r.RUnlock()
    return r.m[key]
}

// Set is a wrapper for setting the value of a key in the underlying map
func (r RWMap) Set(key string, val int) {
    r.Lock()
    defer r.Unlock()
    r.m[key] = val
}

// Inc increases the value in the RWMap for a key.
//   This is more pleasant than r.Set(key, r.Get(key)++)
func (r RWMap) Inc(key string) {
    r.Lock()
    defer r.Unlock()
    r.m[key]++
}

func main() {

    // Init
    counter := RWMap{m: make(map[string]int)}

    // Get a Read Lock
    counter.RLock()
    _ = counter.["Key"]
    counter.RUnlock()

    // the above could be replaced with
    _ = counter.Get("Key")

    // Get a write Lock
    counter.Lock()
    counter.m["some_key"]++
    counter.Unlock()

    // above would need to be written as 
    counter.Inc("some_key")
}

A compensação das funções do wrapper é entre o acesso público do mapa subjacente e o uso correto dos bloqueios apropriados.

Copiar um mapa #

Assim como as fatias, os mapas contêm referências a uma estrutura de dados subjacente. Assim, ao atribuir seu valor a outra variável, apenas a referência será passada. Para copiar o mapa, é necessário criar outro mapa e copiar cada valor:

// Create the original map
originalMap := make(map[string]int)
originalMap["one"] = 1
originalMap["two"] = 2

// Create the target map
targetMap := make(map[string]int)

// Copy from the original map to the target map
for key, value := range originalMap {
  targetMap[key] = value
}

Iterando os elementos de um mapa #

import fmt

people := map[string]int{
  "john": 30,
  "jane": 29,
  "mark": 11,
}

for key, value := range people {
  fmt.Println("Name:", key, "Age:", value)
}

Observe que ao iterar em um mapa com um loop de intervalo, a ordem de iteração não é especificada e não é garantido que seja a mesma em uma iteração para o próximo.

Você também pode descartar as chaves ou os valores do mapa, se quiser apenas [pegar chaves](https://www.wikiod.com/pt/go/mapas#Iterando as chaves de um mapa -a-map#t=201607220900510314955) ou apenas pegue valores.

Iterando as chaves de um mapa #

people := map[string]int{
  "john": 30,
  "jane": 29,
  "mark": 11,
}

for key, _ := range people {
  fmt.Println("Name:", key)
}

Se você está apenas procurando as chaves, já que elas são o primeiro valor, você pode simplesmente soltar o sublinhado:

for key := range people {
  fmt.Println("Name:", key)
}

Observe que ao iterar em um mapa com um loop de intervalo, a ordem de iteração não é especificada e não é garantido que seja a mesma em uma iteração para o próximo.

Criando um mapa #

Pode-se declarar e inicializar um mapa em uma única instrução usando um literal composto.

Usando tipo automático Declaração de variável curta:

mapIntInt := map[int]int{10: 100, 20: 100, 30: 1000}
mapIntString := map[int]string{10: "foo", 20: "bar", 30: "baz"}
mapStringInt := map[string]int{"foo": 10, "bar": 20, "baz": 30}
mapStringString := map[string]string{"foo": "one", "bar": "two", "baz": "three"}

O mesmo código, mas com tipos de variáveis:

var mapIntInt = map[int]int{10: 100, 20: 100, 30: 1000}
var mapIntString = map[int]string{10: "foo", 20: "bar", 30: "baz"}
var mapStringInt = map[string]int{"foo": 10, "bar": 20, "baz": 30}
var mapStringString = map[string]string{"foo": "one", "bar": "two", "baz": "three"}

Você também pode incluir suas próprias estruturas em um mapa:

Você pode usar tipos personalizados como valor:

// Custom struct types
type Person struct {
  FirstName, LastName string
}

var mapStringPerson = map[string]Person{
  "john": Person{"John", "Doe"},
  "jane": Person{"Jane", "Doe"}}
mapStringPerson := map[string]Person{
  "john": Person{"John", "Doe"},
  "jane": Person{"Jane", "Doe"}}

Seu struct também pode ser a key para o mapa:

type RouteHit struct {
    Domain string
    Route  string
}

var hitMap = map[RouteHit]int{
  RouteHit{"example.com","/home"}: 1,
  RouteHit{"example.com","/help"}: 2}
hitMap := map[RouteHit]int{
  RouteHit{"example.com","/home"}: 1,
  RouteHit{"example.com","/help"}: 2}

Você pode criar um mapa vazio simplesmente não inserindo nenhum valor entre os colchetes {}.

mapIntInt := map[int]int{}
mapIntString := map[int]string{}
mapStringInt := map[string]int{}
mapStringString := map[string]string{}
mapStringPerson := map[string]Person{}

Você pode criar e usar um mapa diretamente, sem a necessidade de atribuí-lo a uma variável. No entanto, você terá que especificar a declaração e o conteúdo.

// using a map as argument for fmt.Println()
fmt.Println(map[string]string{
  "FirstName": "John",
  "LastName": "Doe",
  "Age": "30"})

// equivalent to
data := map[string]string{
  "FirstName": "John",
  "LastName": "Doe",
  "Age": "30"}
fmt.Println(data)

Excluindo um elemento do mapa #

A função interna delete remove o elemento com a chave especificada de um mapa.

people := map[string]int{"john": 30, "jane": 29}
fmt.Println(people) // map[john:30 jane:29]

delete(people, "john")
fmt.Println(people) // map[jane:29]

Se o map for nil ou não existir tal elemento, delete não tem efeito.

people := map[string]int{"john": 30, "jane": 29}
fmt.Println(people) // map[john:30 jane:29]

delete(people, "notfound")
fmt.Println(people) // map[john:30 jane:29]

var something map[string]int
delete(something, "notfound") // no-op

Usando um mapa como um conjunto #

Algumas linguagens têm uma estrutura nativa para conjuntos. Para fazer um conjunto em Go, é uma prática recomendada usar um mapa do tipo de valor do conjunto para uma estrutura vazia (map[Type]struct{}).

Por exemplo, com strings:

// To initialize a set of strings:
greetings := map[string]struct{}{
    "hi":    {},
    "hello": {},
}

// To delete a value:
delete(greetings, "hi")

// To add a value:
greetings["hey"] = struct{}{}

// To check if a value is in the set:
if _, ok := greetings["hey"]; ok {
    fmt.Println("hey is in greetings")
}

Iterando os valores de um mapa #

people := map[string]int{
  "john": 30,
  "jane": 29,
  "mark": 11,
}

for _, value := range people {
  fmt.Println("Age:", value)
}

Observe que ao iterar em um mapa com um loop de intervalo, a ordem de iteração não é especificada e não é garantido que seja a mesma em uma iteração para o próximo.

Criando mapas com fatias como valores #

m := make(map[string][]int)

Acessar uma chave inexistente retornará uma fatia nula como valor. Como nil slices agem como fatias de comprimento zero quando usadas com append ou outras funções internas, você normalmente não precisa verificar se existe uma chave:

// m["key1"] == nil && len(m["key1"]) == 0
m["key1"] = append(m["key1"], 1)
// len(m["key1"]) == 1

A exclusão de uma chave do mapa define a chave de volta para uma fatia nula:

delete(m, "key1")
// m["key1"] == nil
