Go vem com suas próprias instalações de teste que têm tudo o que é necessário para executar testes e benchmarks. Ao contrário da maioria das outras linguagens de programação, geralmente não há necessidade de uma estrutura de teste separada, embora existam algumas.

Teste Básico #

main.go:

package main

import (
    "fmt"
)

func main() {
    fmt.Println(Sum(4,5))
}

func Sum(a, b int) int {
    return a + b
}

main_test.go:

package main

import (
    "testing"
)

// Test methods start with `Test`
func TestSum(t *testing.T) {
    got := Sum(1, 2)
    want := 3
    if got != want {
        t.Errorf("Sum(1, 2) == %d, want %d", got, want)
    }
}

Para executar o teste basta usar o comando go test:

$ go test
ok      test_app    0.005s

Use o sinalizador -v para ver os resultados de cada teste:

$ go test -v
=== RUN   TestSum
--- PASS: TestSum (0.00s)
PASS
ok      _/tmp    0.000s

Use o caminho ./... para testar subdiretórios recursivamente:

$ go test -v ./...
ok      github.com/me/project/dir1    0.008s
=== RUN   TestSum
--- PASS: TestSum (0.00s)
PASS
ok      github.com/me/project/dir2    0.008s
=== RUN   TestDiff
--- PASS: TestDiff (0.00s)
PASS

Execute um teste específico:
Se houver vários testes e você quiser executar um teste específico, isso pode ser feito assim:

go test -v -run=<TestName> // will execute only test with this name

Exemplo:

go test -v run=TestSum

Testes de unidade baseados em tabelas #

Esse tipo de teste é uma técnica popular para testes com valores de entrada e saída predefinidos.

Crie um arquivo chamado main.go com conteúdo:

package main

import (
    "fmt"
)

func main() {
    fmt.Println(Sum(4, 5))
}

func Sum(a, b int) int {
    return a + b
}

Depois de executá-lo, você verá que a saída é 9. Embora a função Sum pareça bastante simples, é uma boa ideia testar seu código. Para isso, criamos outro arquivo chamado main_test.go na mesma pasta que main.go, contendo o seguinte código:

package main

import (
    "testing"
)

// Test methods start with Test
func TestSum(t *testing.T) {
    // Note that the data variable is of type array of anonymous struct,
    // which is very handy for writing table-driven unit tests.
    data := []struct {
        a, b, res int
    }{
        {1, 2, 3},
        {0, 0, 0},
        {1, -1, 0},
        {2, 3, 5},
        {1000, 234, 1234},
    }

    for _, d := range data {
        if got := Sum(d.a, d.b); got != d.res {
            t.Errorf("Sum(%d, %d) == %d, want %d", d.a, d.b, got, d.res)
        }
    }
}

Como você pode ver, uma fatia de estruturas anônimas é criada, cada uma com um conjunto de entradas e o resultado esperado. Isso permite que um grande número de casos de teste seja criado em um só lugar e executado em um loop, reduzindo a repetição de código e melhorando a clareza.

Testando usando a função setUp e tearDown #

Você pode definir uma função de configuração e desmontagem.

  • Uma função setUp prepara seu ambiente para testes.
  • Uma função tearDown faz uma reversão.

Esta é uma boa opção quando você não pode modificar seu banco de dados e precisa criar um objeto que simule um objeto trazido de banco de dados ou precisa iniciar uma configuração em cada teste.

Um exemplo estúpido seria:

// Standard numbers map
var numbers map[string]int = map[string]int{"zero": 0, "three": 3}

// TestMain will exec each test, one by one
func TestMain(m *testing.M) {
    // exec setUp function
    setUp("one", 1)
    // exec test and this returns an exit code to pass to os
    retCode := m.Run()
    // exec tearDown function
    tearDown("one")
    // If exit code is distinct of zero,
    // the test will be failed (red)
    os.Exit(retCode)
}

// setUp function, add a number to numbers slice
func setUp(key string, value int) {
    numbers[key] = value
}

// tearDown function, delete a number to numbers slice
func tearDown(key string) {
    delete(numbers, key)
}

// First test
func TestOnePlusOne(t *testing.T) {
    numbers["one"] = numbers["one"] + 1

    if numbers["one"] != 2 {
        t.Error("1 plus 1 = 2, not %v", value)
    }
}

// Second test
func TestOnePlusTwo(t *testing.T) {
    numbers["one"] = numbers["one"] + 2

    if numbers["one"] != 3 {
        t.Error("1 plus 2 = 3, not %v", value)
    }
}

Outro exemplo seria preparar banco de dados para testar e fazer rollback

 // ID of Person will be saved in database
personID := 12345
// Name of Person will be saved in database
personName := "Toni"

func TestMain(m *testing.M) {
    // You create an Person and you save in database
    setUp(&Person{
            ID:   personID,
            Name: personName,
            Age:  19,
        })
    retCode := m.Run()
    // When you have executed the test, the Person is deleted from database
    tearDown(personID)
    os.Exit(retCode)
}

func setUp(P *Person) {
    // ...
    db.add(P)
    // ...
}

func tearDown(id int) {
    // ...
    db.delete(id)
    // ...
}

func getPerson(t *testing.T) {
    P := Get(personID)
    
    if P.Name != personName {
        t.Error("P.Name is %s and it must be Toni", P.Name)
    }
}

Testes de referência #

Se você quiser medir benchmarks, adicione um método de teste como este:

soma.go:

package sum

// Sum calculates the sum of two integers
func Sum(a, b int) int {
    return a + b
}

sum_test.go:

package sum

import "testing"

func BenchmarkSum(b *testing.B) {
    for i := 0; i < b.N; i++ {
        _ = Sum(2, 3)
    }
}

Então, para executar um benchmark simples:

$ go test -bench=. 
BenchmarkSum-8    2000000000             0.49 ns/op
ok      so/sum    1.027s

Testes de exemplo (testes de autodocumentação) #

Esse tipo de teste garante que seu código seja compilado corretamente e aparecerá na documentação gerada para seu projeto. Além disso, os testes de exemplo podem afirmar que seu teste produz uma saída adequada.

soma.go:

package sum

// Sum calculates the sum of two integers
func Sum(a, b int) int {
    return a + b
}

sum_test.go:

package sum

import "fmt"

func ExampleSum() {
    x := Sum(1, 2)
    fmt.Println(x)
    fmt.Println(Sum(-1, -1))
    fmt.Println(Sum(0, 0))

    // Output:
    // 3
    // -2
    // 0
}

Para executar seu teste, execute go test na pasta que contém esses arquivos OU coloque esses dois arquivos em uma subpasta chamada sum e então da pasta pai execute go test ./sum. Em ambos os casos, você obterá uma saída semelhante a esta:

ok      so/sum    0.005s

Se você está se perguntando como isso está testando seu código, aqui está outra função de exemplo, que na verdade falha no teste:

func ExampleSum_fail() {
    x := Sum(1, 2)
    fmt.Println(x)

    // Output:
    // 5
}

Ao executar go test, você obtém a seguinte saída:

$ go test
--- FAIL: ExampleSum_fail (0.00s)
got:
3
want:
5
FAIL
exit status 1
FAIL    so/sum    0.006s

Se você quiser ver a documentação do seu pacote sum - basta executar:

go doc -http=:6060

e navegue até http://localhost:6060/pkg/FOLDER/sum/, onde FOLDER é a pasta que contém o pacote sum (neste exemplo so). A documentação do método sum é assim:

Testando solicitações HTTP #

main.go:

package main

import (
    "fmt"
    "io/ioutil"
    "log"
    "net/http"
)

func fetchContent(url string) (string, error) {
    res, err := http.Get(url)
    if err != nil {
        return "", nil
    }
    defer res.Body.Close()

    body, err := ioutil.ReadAll(res.Body)
    if err != nil {
        return "", err
    }
    return string(body), nil
}

func main() {
    url := "https://example.com/"
    content, err := fetchContent(url)
    if err != nil {
        log.Fatal(err)
    }
    fmt.Println("Content:", content)
}

main_test.go:

package main

import (
    "fmt"
    "net/http"
    "net/http/httptest"
    "testing"
)

func Test_fetchContent(t *testing.T) {
    ts := httptest.NewServer(http.HandlerFunc(func(w http.ResponseWriter, r *http.Request) {
        fmt.Fprint(w, "hello world")
    }))
    defer ts.Close()

    content, err := fetchContent(ts.URL)
    if err != nil {
        t.Error(err)
    }

    want := "hello world"
    if content != want {
        t.Errorf("Got %q, want %q", content, want)
    }
}

Definir/redefinir função simulada em testes #

Este exemplo mostra como simular uma chamada de função que é irrelevante para nosso teste de unidade e, em seguida, usar a instrução defer para reatribuir a chamada de função simulada de volta à sua função original.

var validate = validateDTD

// ParseXML parses b for XML elements and values, and returns them as a map of 
// string key/value pairs.
func ParseXML(b []byte) (map[string]string, error) {
    // we don't care about validating against DTD in our unit test
    if err := validate(b); err != nil {
        return err
    }

    // code to parse b etc.
}

func validateDTD(b []byte) error {
    // get the DTD from some external storage, use it to validate b etc.
}

Em nosso teste de unidade,

func TestParseXML(t *testing.T) {
    // assign the original validate function to a variable.
    originalValidate = validate
    // use the mockValidate function in this test.
    validate = mockValidate
    // defer the re-assignment back to the original validate function.
    defer func() {
       validate = originalValidate
    }()

    var input []byte
    actual, err := ParseXML(input)
    // assertion etc.
}

func mockValidate(b []byte) error {
    return nil // always return nil since we don't care
}

Veja a cobertura do código no formato HTML #

Execute go test normalmente, mas com o sinalizador coverprofile. Em seguida, use a ferramenta go para visualizar os resultados como HTML.

    go test -coverprofile=c.out
    go tool cover -html=c.out
