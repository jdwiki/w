Expansão em linha

  • Desativando a expansão em linha

Nesta página

  • Desativando a expansão em linha

A expansão em linha é uma otimização comum em código compilado que prioriza o desempenho em relação ao tamanho binário. Ele permite que o compilador substitua uma chamada de função pelo corpo real da função; efetivamente copiar/colar código de um lugar para outro em tempo de compilação. Como o site de chamada é expandido para conter apenas as instruções de máquina que o compilador gerou para a função, não precisamos executar um CALL ou PUSH (o equivalente x86 de uma instrução GOTO ou um push de quadro de pilha) ou seu equivalente em outros arquiteturas.

O inliner toma decisões sobre se deve ou não inline uma função com base em uma série de heurísticas, mas em geral Go inlines por padrão. Como o inliner se livra das chamadas de função, ele efetivamente decide onde o escalonador tem permissão para antecipar uma goroutine.

As chamadas de função não serão embutidas se qualquer um dos seguintes for verdadeiro (há muitos outros motivos também, esta lista está incompleta):

  • As funções são variáveis ​​(por exemplo, elas têm argumentos ...)
  • As funções têm um “max hairyness” maior que o orçamento (elas recorrem demais ou não podem ser analisadas por algum outro motivo)
  • Eles contêm panic, recover ou defer

Desativando a expansão em linha #

A expansão inline pode ser desabilitada com o pragma go:noinline. Por exemplo, se construirmos o seguinte programa simples:

package main
 
func printhello() {
    println("Hello")
}
 
func main() {
    printhello()
}

obtemos uma saída que se parece com isso (aparada para facilitar a leitura):

note que não há CALL para printhello. No entanto, se construirmos o programa com o pragma em vigor:

package main
 
//go:noinline
func printhello() {
    println("Hello")
}
 
func main() {
    printhello()
}
