Uma fatia é uma estrutura de dados que encapsula uma matriz para que o programador possa adicionar quantos elementos forem necessários sem ter que se preocupar com o gerenciamento de memória. Fatias podem ser cortadas em subfatias de forma muito eficiente, pois todas as fatias resultantes apontam para a mesma matriz interna. Os programadores Go geralmente aproveitam isso para evitar copiar arrays, o que normalmente seria feito em muitas outras linguagens de programação.

Sintaxe #

  • slice := make([]type, len, cap) // cria uma nova fatia
  • slice = append(slice, item) // anexa um item a uma fatia
  • slice = append(slice, items…) // anexa uma fatia de itens a uma fatia
  • len := len(fatia) // obtém o comprimento de uma fatia
  • cap := cap(slice) // obtém a capacidade de uma fatia
  • elNum := copy(dst, slice) // copia o conteúdo de um slice para outro slice

Adicionando duas fatias juntas #

slice1 := []string{"!"}
slice2 := []string{"Hello", "world"}
slice  := append(slice1, slice2...)

Corra no Go Playground

Comprimento e capacidade #

As fatias têm comprimento e capacidade. O comprimento de uma fatia é o número de elementos atualmente na fatia, enquanto a capacidade é o número de elementos que a fatia pode conter antes de precisar ser realocada.

Ao criar uma fatia usando a função interna make(), você pode especificar seu comprimento e, opcionalmente, sua capacidade. Se a capacidade não for especificada explicitamente, será o comprimento especificado.

var s = make([]int, 3, 5) // length 3, capacity 5

Você pode verificar o comprimento de uma fatia com a função interna len():

var n = len(s) // n == 3

Você pode verificar a capacidade com a função interna cap():

var c = cap(s) // c == 5

Os elementos criados por make() são definidos com o valor zero para o tipo de elemento da fatia:

for idx, val := range s {
    fmt.Println(idx, val)
}
// output:
// 0 0
// 1 0
// 2 0

Execute em play.golang.org

Você não pode acessar elementos além do comprimento de uma fatia, mesmo se o índice estiver dentro da capacidade:

var x = s[3] // panic: runtime error: index out of range

No entanto, desde que a capacidade exceda o comprimento, você pode anexar novos elementos sem realocar:

var t = []int{3, 4}
s = append(s, t) // s is now []int{0, 0, 0, 3, 4}
n = len(s) // n == 5
c = cap(s) // c == 5

Se você anexar a uma fatia que não tem capacidade para aceitar os novos elementos, a matriz subjacente será realocada para você com capacidade suficiente:

var u = []int{5, 6}
s = append(s, u) // s is now []int{0, 0, 0, 3, 4, 5, 6}
n = len(s) // n == 7
c = cap(s) // c > 5

Portanto, geralmente é uma boa prática alocar capacidade suficiente ao criar uma fatia pela primeira vez, se você souber de quanto espaço precisará, para evitar realocações desnecessárias.

Removendo elementos / fatias “fatiando” #

Se você precisar remover um ou mais elementos de uma fatia, ou se precisar trabalhar com uma sub-fatia de outra já existente; você pode usar o seguinte método.

Os exemplos a seguir usam slice of int, mas isso funciona com todos os tipos de slice.

Então, para isso, precisamos de uma fatia, da qual removeremos alguns elementos:

slice := []int{1, 2, 3, 4, 5, 6} 
// > [1 2 3 4 5 6]

Precisamos também dos índices dos elementos para remover:

// index of first element to remove (corresponding to the '3' in the slice)
var first = 2

// index of last element to remove (corresponding to the '5' in the slice)
var last = 4

E assim podemos “fatiar” a fatia, removendo elementos indesejados:

// keeping elements from start to 'first element to remove' (not keeping first to remove), 
// removing elements from 'first element to remove' to 'last element to remove'
// and keeping all others elements to the end of the slice
newSlice1 := append(slice[:first], slice[last+1:]...)
// > [1 2 6]

// you can do using directly numbers instead of variables
newSlice2 := append(slice[:2], slice[5:]...)
// > [1 2 6]

// Another way to do the same
newSlice3 := slice[:first + copy(slice[first:], slice[last+1:])]
// > [1 2 6]

// same that newSlice3 with hard coded indexes (without use of variables)
newSlice4 := slice[:2 + copy(slice[2:], slice[5:])]
// > [1 2 6]

Para remover apenas um elemento, basta colocar o índice desse elemento como o primeiro E como o último índice a ser removido, assim:

var indexToRemove = 3
newSlice5 := append(slice[:indexToRemove], slice[indexToRemove+1:]...)
// > [1 2 3 5 6]

// hard-coded version:
newSlice5 := append(slice[:3], slice[4:]...)
// > [1 2 3 5 6]

E você também pode remover elementos do início da fatia:

newSlice6 := append(slice[:0], slice[last+1:]...)
// > [6]

// That can be simplified into
newSlice6 := slice[last+1:]
// > [6]

Você também pode remover alguns elementos do final da fatia:

newSlice7 := append(slice[:first], slice[first+1:len(slice)-1]...)
// > [1 2]

// That can be simplified into
newSlice7 := slice[:first]
// > [1 2]

Se a nova fatia tiver que conter exatamente os mesmos elementos que a primeira, você pode usar a mesma coisa, mas com last := first-1. (Isso pode ser útil caso seus índices sejam calculados anteriormente)

Criando Fatias #

Fatias são a maneira típica de os programadores armazenarem listas de dados.

Para declarar uma variável de fatia, use a sintaxe []Type.

var a []int

Para declarar e inicializar uma variável slice em uma linha, use a sintaxe []Type{values}.

var a []int = []int{3, 1, 4, 1, 5, 9}

Outra maneira de inicializar uma fatia é com a função make. São três argumentos: o Type da fatia (ou map), o length e a capacity.

a := make([]int, 0, 5)

Você pode adicionar elementos à sua nova fatia usando append.

a = append(a, 5)

Verifique o número de elementos em sua fatia usando len.

length := len(a)

Verifique a capacidade da sua fatia usando cap. A capacidade é o número de elementos atualmente alocados para estar na memória para a fatia. Você sempre pode anexar a uma fatia na capacidade, pois o Go criará automaticamente uma fatia maior para você.

capacity := cap(a)

Você pode acessar elementos em uma fatia usando a sintaxe de indexação típica.

a[0]  // Gets the first member of `a`

Você também pode usar um loop for sobre slices com range. A primeira variável é o índice na matriz especificada e a segunda variável é o valor do índice.

for index, value := range a {
    fmt.Println("Index: " + index + " Value: " + value)  // Prints "Index: 0 Value: 5" (and continues until end of slice)
}

Filtrando uma fatia #

Para filtrar uma fatia sem alocar uma nova matriz subjacente:

// Our base slice
slice := []int{ 1, 2, 3, 4 }
// Create a zero-length slice with the same underlying array
tmp := slice[:0]

for _, v := range slice {
  if v % 2 == 0 {
    // Append desired values to slice
    tmp = append(tmp, v)
  }
}

// (Optional) Reassign the slice
slice = tmp // [2, 4]

Valor zero da fatia #

O valor zero de slice é nil, que tem comprimento e capacidade 0. Uma fatia nil não tem array subjacente. Mas também há fatias não nulas de comprimento e capacidade 0, como []int{} ou make([]int, 5)[5:].

Qualquer tipo que tenha valores nulos pode ser convertido em fatia nil:

s = []int(nil)

Para testar se uma fatia está vazia, use:

if len(s) == 0　{
    fmt.Ptintf("s is empty.")
}

Anexando à fatia #

slice = append(slice, "hello", "world")

Copiando o conteúdo de uma fatia para outra fatia #

Se você deseja copiar o conteúdo de uma fatia em uma fatia inicialmente vazia, as seguintes etapas podem ser seguidas para realizá-lo-

  1. Crie a fatia de origem:

    var sourceSlice []interface{} = []interface{}{“Hello”,5.10,“World”,true}

  2. Crie a fatia de destino, com:

  • Comprimento = Comprimento do sourceSlice

    var destinationSlice []interface{} = make([]interface{},len(sourceSlice))

  1. Agora que o array subjacente da fatia de destino é grande o suficiente para acomodar todos os elementos da fatia de origem, podemos prosseguir para copiar os elementos usando a copy incorporada:

    copy(destinationSlice,sourceSlice)
