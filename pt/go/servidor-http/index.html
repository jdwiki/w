Olá Mundo #

A maneira típica de começar a escrever servidores web em golang é usar o módulo net/http da biblioteca padrão.

Há também um tutorial para isso aqui.

O código a seguir também o usa. Aqui está a implementação de servidor HTTP mais simples possível. Ele responde "Hello World" a qualquer solicitação HTTP.

Salve o código a seguir em um arquivo server.go em seus espaços de trabalho.

package main

import (
    "log"
    "net/http"
)

func main() {
    // All URLs will be handled by this function
    // http.HandleFunc uses the DefaultServeMux
    http.HandleFunc("/", func(w http.ResponseWriter, r *http.Request) {
        w.Write([]byte("Hello, world!"))
    })

    // Continue to process new requests until an error occurs
    log.Fatal(http.ListenAndServe(":8080", nil))
}

Você pode executar o servidor usando:

$ go run server.go

Ou você pode compilar e executar.

$ go build server.go
$ ./server

O servidor escutará a porta especificada (:8080). Você pode testá-lo com qualquer cliente HTTP. Aqui está um exemplo com cURL:

curl -i http://localhost:8080/
HTTP/1.1 200 OK
Date: Wed, 20 Jul 2016 18:04:46 GMT
Content-Length: 13
Content-Type: text/plain; charset=utf-8

Hello, world!

Pressione Ctrl+C para parar o processo.

Crie um servidor HTTPS #

Gera um certificado #

Para executar um servidor HTTPS, é necessário um certificado. A geração de um certificado autoassinado com openssl é feita executando este comando:

openssl req -x509 -newkey rsa:4096 -sha256 -nodes -keyout key.pem -out cert.pem -subj "/CN=example.com" -days 3650`

Os parâmetros são:

  • req Use a ferramenta de solicitação de certificado
  • x509 Cria um certificado autoassinado
  • newkey rsa:4096 Cria uma nova chave e certificado usando os algoritmos RSA com comprimento de chave 4096 bit
  • sha256 Força os algoritmos de hash SHA256 que os principais navegadores consideram seguros (no ano de 2017)
  • nodes Desabilita a proteção por senha para a chave privada. Sem esse parâmetro, seu servidor precisava pedir a senha toda vez que iniciava.
  • keyout Nomeia o arquivo onde escrever a chave
  • out Nomeia o arquivo onde escrever o certificado
  • subj Define o nome de domínio para o qual este certificado é válido
  • days Por quantos dias este certificado deve ser válido? 3650 são aprox. 10 anos.

Nota: Um certificado autoassinado pode ser usado, por exemplo. para projetos internos, depuração, teste, etc. Qualquer navegador por aí mencionará que este certificado não é seguro. Para evitar isso, o certificado deve ser assinado por uma autoridade de certificação. Principalmente, isso não está disponível gratuitamente. Uma exceção é o movimento “Let’s Encrypt”: https://letsencrypt.org

O código Go necessário #

Você pode manipular configurar o TLS para o servidor com o código a seguir. cert.pem e key.pem são seu certificado e chave SSL, que foram gerados com o comando acima.

package main

import (
    "log"
    "net/http"
)

func main() {
    http.HandleFunc("/", func(w http.ResponseWriter, r *http.Request) {
        w.Write([]byte("Hello, world!"))
    })

    log.Fatal(http.ListenAndServeTLS(":443","cert.pem","key.pem", nil))
}

HTTP Hello World com servidor personalizado e mux #

package main

import (
    "log"
    "net/http"
)

func main() {

    // Create a mux for routing incoming requests
    m := http.NewServeMux()

    // All URLs will be handled by this function
    m.HandleFunc("/", func(w http.ResponseWriter, r *http.Request) {
        w.Write([]byte("Hello, world!"))
    })

    // Create a server listening on port 8000
    s := &http.Server{
        Addr:    ":8000",
        Handler: m,
    }

    // Continue to process new requests until an error occurs
    log.Fatal(s.ListenAndServe())
}

Pressione Ctrl+C para parar o processo.

Respondendo a uma solicitação HTTP usando modelos #

As respostas podem ser gravadas em um http.ResponseWriter usando modelos em Go. Isso prova ser uma ferramenta útil se você deseja criar páginas dinâmicas.

(Para saber como os modelos funcionam em Go, visite a página Go Templates Documentation.)

Continuando com um exemplo simples para utilizar o html/template para responder a uma solicitação HTTP:

package main

import(
    "html/template"
    "net/http"
    "log"
)

func main(){
    http.HandleFunc("/",WelcomeHandler)
    http.ListenAndServe(":8080",nil)
}

type User struct{
    Name string
    nationality string //unexported field.
}

func check(err error){
    if err != nil{
        log.Fatal(err)
    }
}

func WelcomeHandler(w http.ResponseWriter, r *http.Request){
    if r.Method == "GET"{
        t,err := template.ParseFiles("welcomeform.html")
        check(err)
        t.Execute(w,nil)
    }else{
        r.ParseForm()
        myUser := User{}
        myUser.Name = r.Form.Get("entered_name")
        myUser.nationality = r.Form.Get("entered_nationality")
        t, err := template.ParseFiles("welcomeresponse.html")
        check(err)
        t.Execute(w,myUser)
    }
}

Onde, o conteúdo de

  1. welcomeform.html são:

    Help us greet you
    Enter Name: Enter Nationality:

  2. welcomeresponse.html são:

    Greetings, {{.Name}} Greetings, {{.Name}}.
    We know you are a {{.nationality}}!

Observação:

  1. Certifique-se de que os arquivos .html estejam no diretório correto.

  2. Quando http://localhost:8080/ pode ser visitado após iniciar o servidor.

  3. Como pode ser visto após o envio do formulário, o campo de nacionalidade não exportado do struct não pôde ser analisado pelo pacote template, como esperado.

Servindo conteúdo usando ServeMux #

Um servidor de arquivos estático simples ficaria assim:

package main

import (
    "net/http"
)

func main() {
    muxer := http.NewServeMux()
    fileServerCss := http.FileServer(http.Dir("src/css"))
    fileServerJs := http.FileServer(http.Dir("src/js"))
    fileServerHtml := http.FileServer(http.Dir("content"))
    muxer.Handle("/", fileServerHtml)
    muxer.Handle("/css", fileServerCss)
    muxer.Handle("/js", fileServerJs)
    http.ListenAndServe(":8080", muxer)
}

Usando uma função de manipulador #

HandleFunc registra a função do manipulador para o padrão fornecido no servidor mux (roteador).

Você pode passar para definir uma função anônima, como vimos no exemplo básico de Hello World:

http.HandleFunc("/", func(w http.ResponseWriter, r *http.Request) {
    fmt.Fprintln(w, "Hello, world!")
}

Mas também podemos passar um tipo HandlerFunc. Em outras palavras, podemos passar qualquer função que respeite a seguinte assinatura:

func FunctionName(w http.ResponseWriter, req *http.Request)

Podemos reescrever o exemplo anterior passando a referência para um HandlerFunc definido anteriormente. Aqui está o exemplo completo:

package main

import (
    "fmt"
    "net/http"
)

// A HandlerFunc function
// Notice the signature of the function
func RootHandler(w http.ResponseWriter, req *http.Request) {
    fmt.Fprintln(w, "Hello, world!")
}

func main() {
    // Here we pass the reference to the `RootHandler` handler function
    http.HandleFunc("/", RootHandler)
    panic(http.ListenAndServe(":8080", nil))
}

Claro, você pode definir vários manipuladores de função para diferentes caminhos.

package main

import (
    "fmt"
    "log"
    "net/http"
)

func FooHandler(w http.ResponseWriter, req *http.Request) {
    fmt.Fprintln(w, "Hello from foo!")
}

func BarHandler(w http.ResponseWriter, req *http.Request) {
    fmt.Fprintln(w, "Hello from bar!")
}

func main() {
    http.HandleFunc("/foo", FooHandler)
    http.HandleFunc("/bar", BarHandler)

    log.Fatal(http.ListenAndServe(":8080", nil))
}

Manipulando o método http, acessando as strings de consulta e o corpo da solicitação #

Aqui está um exemplo simples de algumas tarefas comuns relacionadas ao desenvolvimento de uma API, diferenciando entre o Método HTTP da solicitação, acessando valores de string de consulta e acessando o corpo da solicitação.

Recursos

  • interface http.Handler

  • http.ResponseWriter

  • http.Request

  • Método disponível e constantes de status

    package main

    import ( “fmt” “io/ioutil” “log” “net/http” )

    type customHandler struct{}

    // ServeHTTP implements the http.Handler interface in the net/http package func (h customHandler) ServeHTTP(w http.ResponseWriter, r *http.Request) {

      // ParseForm will parse query string values and make r.Form available
  r.ParseForm()

  // r.Form is map of query string parameters
  // its' type is url.Values, which in turn is a map[string][]string
  queryMap := r.Form

  switch r.Method {
  case http.MethodGet:
      // Handle GET requests
      w.WriteHeader(http.StatusOK)
      w.Write([]byte(fmt.Sprintf("Query string values: %s", queryMap)))
      return
  case http.MethodPost:
      // Handle POST requests
      body, err := ioutil.ReadAll(r.Body)
      if err != nil {
          // Error occurred while parsing request body
          w.WriteHeader(http.StatusBadRequest)
          return
      }
      w.WriteHeader(http.StatusOK)
      w.Write([]byte(fmt.Sprintf("Query string values: %s\nBody posted: %s", queryMap, body)))
      return
  }

  // Other HTTP methods (eg PUT, PATCH, etc) are not handled by the above
  // so inform the client with appropriate status code
  w.WriteHeader(http.StatusMethodNotAllowed)

    }

    func main() { // All URLs will be handled by this function // http.Handle, similarly to http.HandleFunc // uses the DefaultServeMux http.Handle("/", customHandler{})

      // Continue to process new requests until an error occurs
  log.Fatal(http.ListenAndServe(":8080", nil))

    }
