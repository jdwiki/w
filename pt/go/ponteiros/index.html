Sintaxe #

  • pointer := &variable // pega o ponteiro da variável
  • variable := *pointer // obtém a variável do ponteiro
  • *pointer = value // define o valor da variável através do ponteiro
  • pointer := new(Struct) // obtém o ponteiro da nova estrutura

Ponteiros básicos #

Go suporta ponteiros, permitindo que você passe referências a valores e registros dentro de seu programa.

package main

import "fmt"

// We'll show how pointers work in contrast to values with
// 2 functions: `zeroval` and `zeroptr`. `zeroval` has an
// `int` parameter, so arguments will be passed to it by
// value. `zeroval` will get a copy of `ival` distinct
// from the one in the calling function.
func zeroval(ival int) {
    ival = 0
}

// `zeroptr` in contrast has an `*int` parameter, meaning
// that it takes an `int` pointer. The `*iptr` code in the
// function body then _dereferences_ the pointer from its
// memory address to the current value at that address.
// Assigning a value to a dereferenced pointer changes the
// value at the referenced address.
func zeroptr(iptr *int) {
    *iptr = 0
}

Uma vez que essas funções são definidas, você pode fazer o seguinte:

func main() {
    i := 1
    fmt.Println("initial:", i) // initial: 1

    zeroval(i)
    fmt.Println("zeroval:", i) // zeroval: 1
    // `i` is still equal to 1 because `zeroval` edited
    // a "copy" of `i`, not the original.

    // The `&i` syntax gives the memory address of `i`,
    // i.e. a pointer to `i`. When calling `zeroptr`,
    // it will edit the "original" `i`.
    zeroptr(&i)
    fmt.Println("zeroptr:", i) // zeroptr: 0

    // Pointers can be printed too.
    fmt.Println("pointer:", &i) // pointer: 0x10434114
}

Ponteiro v. Métodos de valor #

Métodos de ponteiro #

Métodos de ponteiro podem ser chamados mesmo se a variável em si não for um ponteiro.

De acordo com a Go Spec,

. . . uma referência a um método não-interface com um receptor de ponteiro usando um valor endereçável tomará automaticamente o endereço desse valor: t.Mp é equivalente a (&t).Mp.

Você pode ver isso neste exemplo:

package main

import "fmt"

type Foo struct {
    Bar int
}

func (f *Foo) Increment() {
    f.Bar += 1
}

func main() {
    var f Foo

    // Calling `f.Increment` is automatically changed to `(&f).Increment` by the compiler.
    f = Foo{}
    fmt.Printf("f.Bar is %d\n", f.Bar)
    f.Increment()
    fmt.Printf("f.Bar is %d\n", f.Bar)
    
    // As you can see, calling `(&f).Increment` directly does the same thing.
    f = Foo{}
    fmt.Printf("f.Bar is %d\n", f.Bar)
    (&f).Increment()
    fmt.Printf("f.Bar is %d\n", f.Bar)
}

Jogue

Métodos de valor #

Da mesma forma que os métodos de ponteiro, os métodos de valor podem ser chamados mesmo se a variável em si não for um valor.

De acordo com a Go Spec,

. . . uma referência a um método sem interface com um receptor de valor usando um ponteiro irá desreferenciar automaticamente esse ponteiro: pt.Mv é equivalente a (*pt).Mv.

Você pode ver isso neste exemplo:

package main

import "fmt"

type Foo struct {
    Bar int
}

func (f Foo) Increment() {
    f.Bar += 1
}

func main() {
    var p *Foo

    // Calling `p.Increment` is automatically changed to `(*p).Increment` by the compiler.
    // (Note that `*p` is going to remain at 0 because a copy of `*p`, and not the original `*p` are being edited)
    p = &Foo{}
    fmt.Printf("(*p).Bar is %d\n", (*p).Bar)
    p.Increment()
    fmt.Printf("(*p).Bar is %d\n", (*p).Bar)
    
    // As you can see, calling `(*p).Increment` directly does the same thing.
    p = &Foo{}
    fmt.Printf("(*p).Bar is %d\n", (*p).Bar)
    (*p).Increment()
    fmt.Printf("(*p).Bar is %d\n", (*p).Bar)
}

Jogue

Para saber mais sobre métodos de ponteiro e valor, visite a seção Go Spec sobre valores de método ou consulte a [seção Go efetiva sobre ponteiros v. valores](https ://golang.org/doc/effective_go.html#pointers_vs_values).

Nota 1: Os parênteses (()) em torno de *p e &f antes de seletores como .Bar existem para fins de agrupamento e devem ser mantidos.

Nota 2: Embora os ponteiros possam ser convertidos em valores (e vice-versa) quando são os receptores de um método, eles não são automaticamente convertidos entre si quando são argumentos dentro de uma função.

Ponteiros de desreferenciamento #

Os ponteiros podem ser desreferenciados adicionando um asterisco * antes de um ponteiro.

package main

import (
    "fmt"
)

type Person struct {
    Name string
}

func main() {
    c := new(Person) // returns pointer
    c.Name = "Catherine"
    fmt.Println(c.Name) // prints: Catherine
    d := c
    d.Name = "Daniel"
    fmt.Println(c.Name) // prints: Daniel
    // Adding an Asterix before a pointer dereferences the pointer
    i := *d
    i.Name = "Ines"
    fmt.Println(c.Name) // prints: Daniel
    fmt.Println(d.Name) // prints: Daniel
    fmt.Println(i.Name) // prints: Ines
}

Fatias são ponteiros para segmentos de matriz #

Fatias são ponteiros para matrizes, com o comprimento do segmento e sua capacidade. Eles se comportam como ponteiros e, ao atribuir seu valor a outra fatia, atribuirá o endereço de memória. Para copiar um valor de fatia para outro, use a função incorporada copiar: func copy(dst, src []Type) int (retorna a quantidade de itens copiados).

package main

import (
    "fmt"
)

func main() {
    x := []byte{'a', 'b', 'c'}
    fmt.Printf("%s", x)       // prints: abc
    y := x
    y[0], y[1], y[2] = 'x', 'y', 'z'
    fmt.Printf("%s", x)       // prints: xyz
    z := make([]byte, len(x))
    // To copy the value to another slice, but 
    // but not the memory address use copy:
    _ = copy(z, x)            // returns count of items copied
    fmt.Printf("%s", z)       // prints: xyz
    z[0], z[1], z[2] = 'a', 'b', 'c'
    fmt.Printf("%s", x)       // prints: xyz
    fmt.Printf("%s", z)       // prints: abc
}

Ponteiros Simples #

func swap(x, y *int) {
  *x, *y = *y, *x
}

func main() {
  x := int(1)
  y := int(2)
  // variable addresses
  swap(&x, &y)
  fmt.Println(x, y)
}
