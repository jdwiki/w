BorderPane #

O BorderPane é separado em cinco áreas diferentes.

As áreas de borda (Top, Right, Bottom, Left) têm tamanho preferencial com base em seu conteúdo. Por padrão, eles apenas levarão o que precisam, enquanto a área ‘Centro’ ocupará qualquer espaço restante. Quando as áreas de fronteira estão vazias, elas não ocupam espaço.

Cada área pode conter apenas um elemento. Ele pode ser adicionado usando os métodos setTop(Node), setRight(Node), setBottom(Node), setLeft(Node), setCenter(Node). Você pode usar outros layouts para colocar mais de um elemento em uma única área.

//BorderPane example
BorderPane pane = new BorderPane();

Label top = new Label("Top");

Label right = new Label("Right");

HBox bottom = new HBox();
bottom.getChildren().addAll(new Label("First"), new Label("Second"));

VBox left = new VBox();
left.getChildren().addAll(new Label("Upper"), new Label("Lower"));

StackPane center = new StackPane();
center.getChildren().addAll(new Label("Lorem"), new Label("ipsum"));

pane.setTop(top);        //The text "Top"
pane.setRight(right);    //The text "Right"
pane.setBottom(bottom);  //Row of two texts
pane.setLeft(left);      //Column of two texts
pane.setCenter(center);  //Two texts on each other

GridPane #

GridPane apresenta seus filhos dentro de uma grade flexível de linhas e colunas.

Filhos do GridPane

Um filho pode ser colocado em qualquer lugar dentro do GridPane e pode abranger várias linhas/colunas (o intervalo padrão é 1) e seu posicionamento dentro da grade é definido por suas restrições de layout:

Restrição Descrição
columnIndex coluna onde começa a área de layout do filho.
rowÍndice linha onde a área de layout da criança começa.
colunaSpan o número de colunas que a área de layout do filho abrange horizontalmente.
rowSpan o número de linhas que a área de layout do filho abrange verticalmente.

O número total de linhas/colunas não precisa ser especificado antecipadamente, pois o painel de grade expandirá/contrairá automaticamente a grade para acomodar o conteúdo.

Adicionando filhos ao GridPane

Para adicionar novos Nodes a um GridPane as restrições de layout nos filhos devem ser definidas usando o método estático da classe GridPane, então esses filhos podem ser adicionados a uma instância do GridPane.

GridPane gridPane = new GridPane();

// Set the constraints: first row and first column
Label label = new Label("Example");
GridPane.setRowIndex(label, 0);
GridPane.setColumnIndex(label, 0);
// Add the child to the grid
gridpane.getChildren().add(label);

GridPane fornece métodos convenientes para combinar estas etapas:

gridPane.add(new Button("Press me!"), 1, 0); // column=1 row=0

A classe GridPane também fornece métodos setter estáticos para definir o row- e columnspan de elementos filho:

Label labelLong = new Label("Its a long text that should span several rows");
GridPane.setColumnSpan(labelLong, 2);
gridPane.add(labelLong, 0, 1);  // column=0 row=1

Tamanho das colunas e linhas

Por padrão, as linhas e colunas serão dimensionadas para caber em seu conteúdo. Caso seja necessário o controle explícito dos tamanhos das linhas e colunas, instâncias RowConstraints e ColumnConstraints podem ser adicionadas ao GridPane. Adicionar essas duas restrições redimensionará o exemplo acima para que a primeira coluna tenha 100 pixels e a segunda coluna 200 pixels.

gridPane.getColumnConstraints().add(new ColumnConstraints(100));
gridPane.getColumnConstraints().add(new ColumnConstraints(200));

Por padrão, o GridPane redimensionará as linhas/colunas para seus tamanhos preferidos, mesmo que o gridpane seja redimensionado para um tamanho maior que seu tamanho preferido. Para oferecer suporte a tamanhos dinâmicos de coluna/linha, ambas as classes contstains fornecem três propriedades: tamanho mínimo, tamanho máximo e tamanho preferencial.

Além disso, ColumnConstraints fornece setHGrow e RowConstraints fornece métodos setVGrow para afetar a prioridade do crescimento e redução. As três prioridades pré-definidas são:

  • Prioridade.SEMPRE: Sempre tente crescer (ou diminuir), compartilhando o aumento (ou diminuição) no espaço com outras áreas de layout que tenham um aumento (ou redução) de SEMPRE

  • Prioridade.ÀS VEZES: Se não houver outras áreas de layout com aumento (ou redução) definido como SEMPRE ou essas áreas de layout não absorveram todo o espaço aumentado (ou diminuído), compartilhará o aumento (ou diminuir) no espaço com outras áreas de layout de ALGUMAS VEZES.

  • Prioridade. NUNCA: A área do layout nunca aumentará (ou diminuirá) quando houver um aumento (ou diminuição) no espaço disponível na região.

    ColumnConstraints column1 = new ColumnConstraints(100, 100, 300); column1.setHgrow(Priority.ALWAYS);

A coluna definida acima tem um tamanho mínimo de 100 pixels e sempre tentará crescer até atingir sua largura máxima de 300 pixels.

Também é possível definir tamanho percentual para linhas e colunas. O exemplo a seguir define um GridPane onde a primeira coluna preenche 40% da largura do gridpane, a segunda preenche os 60%.

GridPane gridpane = new GridPane();
ColumnConstraints column1 = new ColumnConstraints();
column1.setPercentWidth(40);
ColumnConstraints column2 = new ColumnConstraints();
column2.setPercentWidth(60);
gridpane.getColumnConstraints().addAll(column1, column2);

Alinhamento de elementos dentro das células da grade

O alinhamento de Nodes pode ser definido usando o método setHalignment (horizontal) da classe ColumnConstraints e o método setValignment (vertical) da classe RowConstraints.

ColumnConstraints column1 = new ColumnConstraints();
column1.setHalignment(HPos.RIGHT);

RowConstraints row1 = new RowConstraints();
row1.setValignment(VPos.CENTER);

Âncora #

AnchorPane a é um layout que permite colocar o conteúdo a uma distância específica de seus lados.

Existem 4 métodos para definir e 4 métodos para obter as distâncias em AnchorPane. O primeiro parâmetro desses métodos é o filho Node. O segundo parâmetro dos setters é o valor Double a ser usado. Este valor pode ser null indicando que não há restrição para o lado fornecido.

método setter método getter
setBottomÂncora getBottomAnchor
setLeftAnchor getLeftAnchor
setRightAnchor getRightÂncora
setTopAnchor getTopAnchor

No exemplo a seguir, coloca os nós em distâncias especificadas dos lados.

A região center também é redimensionada para manter as distâncias especificadas dos lados. Observe o comportamento quando a janela é redimensionada.

public static void setBackgroundColor(Region region, Color color) {
    // change to 50% opacity
    color = color.deriveColor(0, 1, 1, 0.5);
    region.setBackground(new Background(new BackgroundFill(color, CornerRadii.EMPTY, Insets.EMPTY)));
}

@Override
public void start(Stage primaryStage) {
    Region right = new Region();
    Region top = new Region();
    Region left = new Region();
    Region bottom = new Region();
    Region center = new Region();
    
    right.setPrefSize(50, 150);
    top.setPrefSize(150, 50);
    left.setPrefSize(50, 150);
    bottom.setPrefSize(150, 50);
    
    // fill with different half-transparent colors
    setBackgroundColor(right, Color.RED);
    setBackgroundColor(left, Color.LIME);
    setBackgroundColor(top, Color.BLUE);
    setBackgroundColor(bottom, Color.YELLOW);
    setBackgroundColor(center, Color.BLACK);
    
    // set distances to sides
    AnchorPane.setBottomAnchor(bottom, 50d);
    AnchorPane.setTopAnchor(top, 50d);
    AnchorPane.setLeftAnchor(left, 50d);
    AnchorPane.setRightAnchor(right, 50d);
    
    AnchorPane.setBottomAnchor(center, 50d);
    AnchorPane.setTopAnchor(center, 50d);
    AnchorPane.setLeftAnchor(center, 50d);
    AnchorPane.setRightAnchor(center, 50d);
    
    // create AnchorPane with specified children
    AnchorPane anchorPane = new AnchorPane(left, top, right, bottom, center);

    Scene scene = new Scene(anchorPane, 200, 200);
    
    primaryStage.setScene(scene);
    primaryStage.show();
}

HBox e VBox #

Os layouts HBox e VBox são muito semelhantes, ambos apresentam seus filhos em uma única linha.

Características comuns

Se um HBox ou um VBox tiver uma borda e/ou preenchimento definido, o conteúdo será disposto dentro dessas inserções.

Eles apresentam cada filho gerenciado independentemente do valor da propriedade visível do filho; filhos não gerenciados são ignorados.

O alinhamento do conteúdo é controlado pela propriedade de alinhamento, cujo padrão é Pos.TOP_LEFT.

HBox

HBox apresenta seus filhos em uma única linha horizontal da esquerda para a direita.

O HBox redimensionará os filhos (se redimensionáveis) para sua largura preferidas e usará sua propriedade fillHeight para determinar se deve redimensionar suas alturas para preencher sua própria altura ou manter suas alturas de acordo com sua preferência (o padrão fillHeight é true).

Criando um HBox

// HBox example
HBox row = new HBox();
Label first = new Label("First");
Label second = new Label("Second");
row.getChildren().addAll(first, second);

VBox

VBox apresenta seus filhos em uma única coluna vertical de cima para baixo.

O VBox redimensionará os filhos (se redimensionáveis) para suas alturas preferidas e usará sua propriedade fillWidth para determinar se deve redimensionar suas larguras para preencher sua própria largura ou manter suas larguras de acordo com sua preferência (o padrão fillWidth é true).

Criando uma VBox

// VBox example
VBox column = new VBox();
Label upper = new Label("Upper");
Label lower = new Label("Lower");
column.getChildren().addAll(upper, lower);

StackPane #

StackPane coloca seus filhos em uma pilha de trás para frente.

A ordem z dos filhos é definida pela ordem da lista de filhos (acessível chamando getChildren): o 0º filho sendo o de baixo e o último filho no topo da pilha.

O painel de pilha tenta redimensionar cada filho para preencher sua própria área de conteúdo. No caso de um filho não poder ser redimensionado para preencher a área do StackPane (seja porque não era redimensionável ou seu tamanho máximo o impedia), ele será alinhado dentro da área usando o alignmentProperty do painel de pilha, que o padrão é Pos.CENTER.

Exemplo

// Create a StackPane
StackPane pane = new StackPane();

// Create three squares
Rectangle rectBottom = new Rectangle(250, 250);
rectBottom.setFill(Color.AQUA);
Rectangle rectMiddle = new Rectangle(200, 200);
rectMiddle.setFill(Color.CADETBLUE);
Rectangle rectUpper = new Rectangle(150, 150);
rectUpper.setFill(Color.CORAL);

// Place them on top of each other
pane.getChildren().addAll(rectBottom, rectMiddle, rectUpper);

FlowPane #

FlowPane apresenta nós em linhas ou colunas com base no espaço horizontal ou vertical disponível. Ele envolve os nós para a próxima linha quando o espaço horizontal é menor que o total de larguras de todos os nós; ele envolve os nós para a próxima coluna quando o espaço vertical é menor que o total de alturas de todos os nós. Este exemplo ilustra o layout horizontal padrão:

import javafx.application.Application;
import javafx.fxml.FXMLLoader;
import javafx.scene.Parent;
import javafx.scene.Scene;
import javafx.scene.control.Button;
import javafx.scene.layout.FlowPane;
import javafx.stage.Stage;

public class Main extends Application {
    @Override
    public void start(Stage primaryStage) throws Exception{
        FlowPane root = new FlowPane();
        for (int i=1; i<=15; i++) {
            Button b1=new Button("Button "+String.valueOf(i));
            root.getChildren().add(b1); //for adding button to root
        }
        Scene scene = new Scene(root, 300, 250);
        primaryStage.setTitle("FlowPane Layout");
        primaryStage.setScene(scene);
        primaryStage.show();
    }
    public static void main(String[] args) {
        launch(args);
    }
}

Construtor padrão do FlowPane:

FlowPane root = new FlowPane();

Construtores FlowPane adicionais:

FlowPane() //Creates a horizontal FlowPane layout with hgap/vgap = 0 by default.
FlowPane(double hgap, double gap) //Creates a horizontal FlowPane layout with the specified hgap/vgap.
FlowPane(double hgap, double vgap, Node... children) //Creates a horizontal FlowPane layout with the specified hgap/vgap.
FlowPane(Node... children) //Creates a horizontal FlowPane layout with hgap/vgap = 0.
FlowPane(Orientation orientation) //Creates a FlowPane layout with the specified orientation and hgap/vgap = 0.
FlowPane(Orientation orientation, double hgap, double gap) //Creates a FlowPane layout with the specified orientation and hgap/vgap.
FlowPane(Orientation orientation, double hgap, double vgap, Node... children) //Creates a FlowPane layout with the specified orientation and hgap/vgap.
FlowPane(Orientation orientation, Node... children) //Creates a FlowPane layout with the specified orientation and hgap/vgap = 0.

Adicionar nós ao layout usa os métodos add() ou addAll() do Pane pai:

Button btn = new Button("Demo Button");    
root.getChildren().add(btn); 
root.getChildren().addAll(…);

Por padrão, um FlowPane apresenta nós filhos da esquerda para a direita. Para alterar o alinhamento do fluxo, chame o método setAlignment() passando um valor enumerado do tipo Pos.

Alguns alinhamentos de fluxo comumente usados:

root.setAlignment(Pos.TOP_RIGHT);     //for top right
root.setAlignment(Pos.TOP_CENTER);    //for top Center
root.setAlignment(Pos.CENTER);        //for Center
root.setAlignment(Pos.BOTTOM_RIGHT);  //for bottom right

Painel #

O layout do painel lado a lado é semelhante ao layout do FlowPane. TilePane coloca todos os nós em uma grade na qual cada célula, ou bloco, tem o mesmo tamanho. Ele organiza os nós em linhas e colunas organizadas, horizontal ou verticalmente.

    import javafx.application.Application;
    import javafx.scene.Scene;
    import javafx.scene.control.Button;
    import javafx.scene.layout.TilePane;
    import javafx.stage.Stage;
    
    public class Main extends Application {
    
        @Override
        public void start(Stage primaryStage) {
            primaryStage.setTitle("TilePane Demo");
            double width = 400;
            double height = 300;
            TilePane root = new TilePane();
            root.setStyle("-fx-background-color:blue");
            // to set horizontal and vertical gap
            root.setHgap(20);
            root.setVgap(50);
            Button bl = new Button("Buttons");
            root.getChildren().add(bl);
            Button btn = new Button("Button");
            root.getChildren().add(btn);
            Button btn1 = new Button("Button 1");
            root.getChildren().add(btn1);
            Button btn2 = new Button("Button 2");
            root.getChildren().add(btn2);
            Button btn3 = new Button("Button 3");
            root.getChildren().add(btn3);
            Button btn4 = new Button("Button 4");
            root.getChildren().add(btn4);
    
            Scene scene = new Scene(root, width, height);
            primaryStage.setScene(scene);
            primaryStage.show();
        }
    
    
        public static void main(String[] args) {
            launch(args);
        }
    }
