  • Projeto JavaFX básico usando FXML

JavaFX Scene Builder é uma ferramenta de layout visual que permite aos usuários projetar rapidamente interfaces de usuário de aplicativos JavaFX, sem codificação. É usado para gerar arquivos FXML.

JavaFX Scene Builder é uma ferramenta de layout visual que permite aos usuários projetar rapidamente interfaces de usuário de aplicativos JavaFX, sem codificação. Os usuários podem arrastar e soltar componentes da interface do usuário em uma área de trabalho, modificar suas propriedades, aplicar folhas de estilo e o código FXML para o layout que eles estão criando é gerado automaticamente em segundo plano. O resultado é um arquivo FXML que pode ser combinado com um projeto Java vinculando a interface do usuário à lógica do aplicativo.

Do ponto de vista do Model View Controller (MVC):

  • O arquivo FXML, contendo a descrição da interface do usuário, é a visualização.
  • O controlador é uma classe Java, implementando opcionalmente a classe Initializable, que é declarada como o controlador para o arquivo FXML.
  • O modelo consiste em objetos de domínio, definidos no lado Java, que podem ser conectados à visualização por meio do controlador.

Instalação do Scene Builder

  1. Baixe a versão mais recente do Scene Builder do site da Gluon, selecionando o instalador para sua plataforma ou o jar executável.

  2. Com o instalador baixado, clique duas vezes para instalar o Scene Builder em seu sistema. Um JRE atualizado está incluído.

  3. Clique duas vezes no ícone do Scene Builder para executá-lo como aplicativo autônomo.

  4. Integração IDE

    While Scene Builder is a standalone application, it produces FXML files that are integrated with a Java SE project. When creating this project on an IDE, it is convenient to include a link to the Scene Builder path, so FXML files can be edited.

    • NetBeans: On Windows go to NetBeans->Tools->Options->Java->JavaFX. On Mac OS X go to NetBeans->Preferences->Java->JavaFX. Provide the path for the Scene Builder Home.
    • IntelliJ: On Windows go to IntelliJ->Settings->Languages & Frameworks->JavaFX. On Mac OS X go to IntelliJ->Preferences->Languages & Frameworks->JavaFX. Provide the path for the Scene Builder Home.
    • Eclipse: On Windows go to Eclipse->Window->Preferences->JavaFX. On Mac OS X go to Eclipse->Preferences->JavaFX. Provide the path for the Scene Builder Home.

Um pouco de história

O projeto Scene Builder foi criado usando JavaFX pela Oracle e é open source dentro do projeto OpenJFX.

Binários Oracle forneciam, até o Scene Builder v 2.0, incluindo apenas recursos JavaFX antes do lançamento do Java SE 8u40, portanto, novos recursos como os controles Spinner não estão incluídos.

Gluon assumiu a distribuição de versões binárias, e um Scene Builder 8+ atualizado pode ser baixado para todas as plataformas aqui.

Inclui as alterações mais recentes no JavaFX e também melhorias recentes e correções de bugs.

O projeto de código aberto pode ser encontrado aqui onde podem ser criados problemas, solicitações de recursos e solicitações pull.

Os binários legados da Oracle ainda podem ser baixados aqui.

Projeto JavaFX básico usando FXML #

Este é um projeto básico que utiliza FXML, criado com NetBeans (New Project -> JavaFX -> JavaFX FXML Application). Ele contém apenas três arquivos:

Classe de aplicativo principal

package org.stackoverflow;

import javafx.application.Application;
import javafx.fxml.FXMLLoader;
import javafx.scene.Parent;
import javafx.scene.Scene;
import javafx.stage.Stage;

public class BasicApplication extends Application {

    @Override
    public void start(Stage stage) throws Exception {
        Parent root = FXMLLoader.load(getClass().getResource("BasicFXML.fxml"));

        Scene scene = new Scene(root);

        stage.setScene(scene);
        stage.show();
    }

    public static void main(String[] args) {
        launch(args);
    }
}

Arquivo FXML

<?xml version="1.0" encoding="UTF-8"?>

<?import java.lang.*?>
<?import java.util.*?>
<?import javafx.scene.*?>
<?import javafx.scene.control.*?>
<?import javafx.scene.layout.*?>

<AnchorPane id="AnchorPane" prefHeight="200" prefWidth="320" xmlns:fx="http://javafx.com/fxml/1" fx:controller="org.stackoverflow.BasicFXMLController">
    <children>
        <Button layoutX="126" layoutY="90" text="Click Me!" onAction="#handleButtonAction" fx:id="button" />
        <Label layoutX="126" layoutY="120" minHeight="16" minWidth="69" fx:id="label" />
    </children>
</AnchorPane>

Controlador

package org.stackoverflow;

import javafx.event.ActionEvent;
import javafx.fxml.FXML;
import javafx.scene.control.Label;

public class BasicFXMLController {

    @FXML
    private Label label;

    public void initialize() {
        // TODO
    }   

    @FXML
    private void handleButtonAction(ActionEvent event) {
        label.setText("Hello World!");
    }

}

Corre

Construir e executar o projeto deve exibir uma pequena janela com um botão clicável:

Como funciona

Resumidamente, na classe Application principal, o FXMLLoader carregará basicFXML.fxml do jar/classpath, conforme especificado por FXMLLoader.load(getClass().getResource("BasicFXML.fxml")).

Ao carregar basicFXML.fxml, o carregador encontrará o nome da classe do controlador, conforme especificado por fx:controller="org.stackoverflow.BasicFXMLController" no FXML.

Em seguida, o carregador criará uma instância dessa classe, na qual tentará injetar todos os objetos que possuem um fx:id no FXML e estão marcados com a anotação @FXML na classe do controlador.

Neste exemplo, o FXMLLoader criará o rótulo com base em <Label ... fx:id="label"/> e injetará a instância do rótulo no rótulo @FXML private Label;.

Finalmente, quando todo o FXML tiver sido carregado, o FXMLLoader chamará o método initialize do controlador, e o código que registrar um manipulador de ação com o botão será executado.

Edição

Embora o arquivo FXML possa ser editado no IDE, isso não é recomendado, pois o IDE fornece apenas verificação de sintaxe básica e preenchimento automático, mas não orientação visual.

A melhor abordagem é abrir o arquivo FXML com o Scene Builder, onde todas as alterações serão salvas no arquivo.

O Scene Builder pode ser iniciado para abrir o arquivo:

Ou o arquivo pode ser aberto com o Scene Builder diretamente do IDE:

  • No NetBeans, na aba do projeto, clique duas vezes no arquivo ou clique com o botão direito e selecione Abrir.
  • No IntelliJ, na guia do projeto, clique com o botão direito do mouse no arquivo e selecione ‘Abrir no Scene Builder’.
  • No Eclipse, na aba do projeto, clique com o botão direito do mouse no arquivo e selecione Abrir com o Scene Builder.

Se o Scene Builder estiver instalado corretamente e seu caminho adicionado ao IDE (consulte Comentários abaixo), ele abrirá o arquivo:

As alterações podem ser feitas arrastando novos contêineres ou novos controles dos painéis esquerdos, e as propriedades e os valores de layout podem ser alterados nos painéis direitos.

Observe que uma das tags id que permite injetar o FXML no código Java é fx:id. Ele pode ser definido no painel Code:

Após aplicar as alterações, salve o arquivo (Scene Builder -> Arquivo -> Salvar). Se forem feitas alterações editando o arquivo a partir do IDE, ao salvar o arquivo, elas serão atualizadas no Scene Builder.
