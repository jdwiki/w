Sintaxe #

  • xmlns:fx=“http://javafx.com/fxml”       // declaração de namespace

Propriedades de configuração #

Existem várias maneiras de adicionar dados a um objeto em fxml:

tag <propriedade> #

Uma tag com o nome de uma propriedade pode ser adicionada como filha de um elemento usado para criar uma instância. O filho desta tag é atribuído à propriedade usando o setter ou adicionado ao conteúdo da propriedade (propriedades de lista/mapa somente leitura).

Propriedade padrão #

Uma classe pode ser anotada com a anotação @DefaultProperty. Neste caso, os elementos podem ser adicionados diretamente como elemento filho sem usar um elemento com o nome da propriedade.

atributo property="value" #

As propriedades podem ser atribuídas usando o nome da propriedade como nome do atributo e o valor como valor do atributo. Isso tem o mesmo efeito que adicionar o seguinte elemento como filho da tag:

<property>
    <String fx:value="value" />
</property>

setters estáticos #

As propriedades também podem ser definidas usando setters static. Estes são métodos static chamados setProperty que usam o elemento como primeiro parâmetro e o valor a ser definido como segundo parâmetro. Esses métodos podem residir em qualquer classe e podem ser usados ​​usando ContainingClass.property em vez do nome de propriedade usual.

Observação: Atualmente, parece ser necessário ter um método getter estático correspondente (ou seja, um método estático chamado getProperty tomando o elemento como parâmetro na mesma classe que o setter estático) para que isso funcione, a menos que o tipo de valor é String.

Coerção de tipo #

O mecanismo a seguir é usado para obter um objeto da classe correta durante as atribuições, por exemplo. para se adequar ao tipo de parâmetro de um método setter.

Se as classes forem atribuíveis, o próprio valor será usado.

Caso contrário, o valor é convertido da seguinte forma

Tipo de destino valor usado (valor de origem s)
Boolean, boolean Boolean.valueOf(s)
char, personagem s.toString.charAt(0)
outro tipo primitivo ou tipo wrapper método apropriado para o tipo de destino, caso o s seja um Number, o valueOf(s.toString()) para o tipo wrapper caso contrário
BigInteger BigInteger.valueOf(s.longValue()) é s é um Number, new BigInteger(s.toString()) caso contrário
BigDecimal BigDecimal.valueOf(s.doubleValue()) é s é um Number, new BigDecimal(s.toString()) caso contrário
Número Double.valueOf(s.toString()) se s.toString() contiver um ., Long.valueOf(s.toString()) caso contrário
Classe Class.forName(s.toString()) invocado usando o contexto ClassLoader da thread atual sem inicializar a classe
enum O resultado do método valueOf, adicionalmente convertido em uma String totalmente maiúscula separada por _ inserida antes de cada letra maiúscula, se s for uma String que começa com uma letra minúscula
outro o valor retornado por um método static valueOf no targetType, que tem um parâmetro correspondente ao tipo de s ou uma superclasse desse tipo

Observação: esse comportamento não está bem documentado e pode estar sujeito a alterações.

public enum Location {
    WASHINGTON_DC,
    LONDON;
}

import java.util.ArrayList;
import java.util.HashMap;
import java.util.List;
import java.util.Map;
import javafx.beans.DefaultProperty;

@DefaultProperty("items")
public class Sample {
    
    private Location loaction;

    public Location getLoaction() {
        return loaction;
    }

    public void setLoaction(Location loaction) {
        this.loaction = loaction;
    }

    public int getNumber() {
        return number;
    }

    public void setNumber(int number) {
        this.number = number;
    }
    
    int number;

    private final List<Object> items = new ArrayList<>();

    public List<Object> getItems() {
        return items;
    }
    
    private final Map<String, Object> map = new HashMap<>();

    public Map<String, Object> getMap() {
        return map;
    }
    
    private BigInteger serialNumber;

    public BigInteger getSerialNumber() {
        return serialNumber;
    }

    public void setSerialNumber(BigInteger serialNumber) {
        this.serialNumber = serialNumber;
    }

    @Override
    public String toString() {
        return "Sample{" + "loaction=" + loaction + ", number=" + number + ", items=" + items + ", map=" + map + ", serialNumber=" + serialNumber + '}';
    }
    
}

package fxml.sample;

public class Container {

    public static int getNumber(Sample sample) {
        return sample.number;
    }

    public static void setNumber(Sample sample, int number) {
        sample.number = number;
    }

    private final String value;

    private Container(String value) {
        this.value = value;
    }

    public static Container valueOf(String s) {
        return new Container(s);
    }

    @Override
    public String toString() {
        return "42" + value;
    }

}

Passando parâmetros para FXML - usando um controllerFactory #

Problema: alguns dados precisam ser passados ​​para uma cena carregada de um fxml.

Solução

Especifique uma fábrica de controladores responsável por criar os controladores. Passe os dados para a instância do controlador criada pela fábrica.

FXML

<?xml version="1.0" encoding="UTF-8"?>

<?import javafx.scene.text.*?>
<?import javafx.scene.layout.*?>

<VBox xmlns:fx="http://javafx.com/fxml/1" fx:controller="valuepassing.TestController">
    <children>
        <Text fx:id="target" />
    </children>
</VBox>

Controlador

package valuepassing;

import javafx.fxml.FXML;
import javafx.scene.text.Text;

public class TestController {

    private final String data;

    public TestController(String data) {
        this.data = data;
    }
    
    @FXML
    private Text target;
    
    public void initialize() {
        // handle data once the fields are injected
        target.setText(data);
    }

}

Código usado para carregar o fxml

String data = “Olá Mundo!”;

Map<Class, Callable<?>> creators = new HashMap<>();
creators.put(TestController.class, new Callable<TestController>() {

    @Override
    public TestController call() throws Exception {
        return new TestController(data);
    }

});

FXMLLoader loader = new FXMLLoader(getClass().getResource("test.fxml"));

loader.setControllerFactory(new Callback<Class<?>, Object>() {

    @Override
    public Object call(Class<?> param) {
        Callable<?> callable = creators.get(param);
        if (callable == null) {
            try {
                // default handling: use no-arg constructor
                return param.newInstance();
            } catch (InstantiationException | IllegalAccessException ex) {
                throw new IllegalStateException(ex);
            }
        } else {
            try {
                return callable.call();
            } catch (Exception ex) {
                throw new IllegalStateException(ex);
            }
        }
    }
});

Parent root = loader.load();

Isso pode parecer complexo, mas pode ser útil, se o fxml puder decidir qual classe de controlador ele precisa.

Criação de instância em FXML #

A classe a seguir é usada para demonstrar como instâncias de classes podem ser criadas:

A anotação em Person(@NamedArg("name") String name) deve ser removida, pois a anotação @NamedArg não está disponível.

package fxml.sample;

import javafx.beans.NamedArg;
import javafx.beans.property.SimpleStringProperty;
import javafx.beans.property.StringProperty;

public class Person {

    public static final Person JOHN = new Person("John");

    public Person() {
        System.out.println("Person()");
    }
    
    public Person(@NamedArg("name") String name) {
        System.out.println("Person(String)");
        this.name.set(name);
    }
    
    public Person(Person person) {
        System.out.println("Person(Person)");
        this.name.set(person.getName());
    }

    private final StringProperty name = new SimpleStringProperty();

    public final String getName() {
        System.out.println("getter");
        return this.name.get();
    }

    public final void setName(String value) {
        System.out.println("setter");
        this.name.set(value);
    }

    public final StringProperty nameProperty() {
        System.out.println("property getter");
        return this.name;
    }
    
    public static Person valueOf(String value) {
        System.out.println("valueOf");
        return new Person(value);
    }
    
    public static Person createPerson() {
        System.out.println("createPerson");
        return new Person();
    }
    
}

Suponha que a classe Person já tenha sido inicializada antes de carregar o fxml.

Uma nota sobre importações #

No exemplo fxml a seguir, a seção de importações será deixada de fora. No entanto, o fxml deve começar com

<?xml version="1.0" encoding="UTF-8"?>

seguido por uma seção de importações importando todas as classes usadas no arquivo fxml. Essas importações são semelhantes às importações não estáticas, mas são adicionadas como instruções de processamento. Mesmo as classes do pacote java.lang precisam ser importadas.

Neste caso, as seguintes importações devem ser adicionadas:

<?import java.lang.*?>
<?import fxml.sample.Person?>

 

construtor anotado @NamedArg #

Se houver um construtor onde cada parâmetro é anotado com @NamedArg e todos os valores das anotações @NamedArg estão presentes no fxml, o construtor será usado com esses parâmetros.

<Person name="John"/>

<Person xmlns:fx="http://javafx.com/fxml">
    <name>
        <String fx:value="John"/>
    </name>
</Person>

Ambos resultam na seguinte saída do console, se carregados:

Person(String)

 

Nenhum construtor de argumentos #

Se não houver um construtor anotado @NamedArg adequado disponível, o construtor que não recebe parâmetros será usado.

Remova a anotação @NamedArg do construtor e tente carregar.

<Person name="John"/>

Isso usará o construtor sem parâmetros.

Resultado:

Person()
setter

 

atributo fx:value #

O atributo fx:value pode ser usado para passar seu valor para um método static valueOf pegando um parâmetro String e retornando a instância a ser usada.

Exemplo

<Person xmlns:fx="http://javafx.com/fxml" fx:value="John"/>

Resultado:

valueOf
Person(String)

 

fx:fábrica #

O atributo fx:factory permite a criação de objetos usando métodos static arbitrários que não recebem parâmetros.

Exemplo

<Person xmlns:fx="http://javafx.com/fxml" fx:factory="createPerson">
    <name>
        <String fx:value="John"/>
    </name>
</Person>

Resultado:

createPerson
Person()
setter

 

<fx:copiar> #

Usando fx:copy um construtor de cópia pode ser invocado. Especificando o fx:id de outro O atributo source da tag invocará o construtor de cópia com esse objeto como parâmetro.

Exemplo:

<ArrayList xmlns:fx="http://javafx.com/fxml">
    <Person fx:id="p1" fx:constant="JOHN"/>
    <fx:copy source="p1"/>
</ArrayList>

Resultado

Person(Person)
getter

 

fx:constante #

fx:constant permite obter um valor de um campo static final.

Exemplo

<Person xmlns:fx="http://javafx.com/fxml" fx:constant="JOHN"/>

não produzirá nenhuma saída, pois isso apenas referencia o JOHN que foi criado ao inicializar a classe.

Exemplo FXML #

Um documento FXML simples descrevendo um AnchorPane contendo um botão e um nó de rótulo:

<?xml version="1.0" encoding="UTF-8"?>

<?import java.lang.*?>
<?import java.util.*?>
<?import javafx.scene.*?>
<?import javafx.scene.control.*?>
<?import javafx.scene.layout.*?>

<AnchorPane id="AnchorPane" prefHeight="200" prefWidth="320" xmlns:fx="http://javafx.com/fxml/1" 
        fx:controller="com.example.FXMLDocumentController">
    <children>
        <Button layoutX="126" layoutY="90" text="Click Me!" onAction="#handleButtonAction" fx:id="button" />
        <Label layoutX="126" layoutY="120" minHeight="16" minWidth="69" fx:id="label" />
    </children>
</AnchorPane>

Este arquivo FXML de exemplo está associado a uma classe de controlador. A associação entre o FXML e a classe do controlador, neste caso, é feita especificando o nome da classe como o valor do atributo fx:controller no elemento raiz do FXML: fx:controller="com.example. FXMLDocumentController". A classe do controlador permite que o código Java seja executado em resposta às ações do usuário nos elementos da interface do usuário definidos no arquivo FXML:

package com.example ;

import java.net.URL;
import java.util.ResourceBundle;
import javafx.event.ActionEvent;
import javafx.fxml.FXML;
import javafx.fxml.Initializable;
import javafx.scene.control.Label;

public class FXMLDocumentController {
    
    @FXML
    private Label label;
    
    @FXML
    private void handleButtonAction(ActionEvent event) {
        System.out.println("You clicked me!");
        label.setText("Hello World!");
    }
    
    @Override
    public void initialize(URL url, ResourceBundle resources) {
        // Initialization code can go here. 
        // The parameters url and resources can be omitted if they are not needed
    }    
    
}

Um FXMLLoader pode ser usado para carregar o arquivo FXML:

public class MyApp extends Application {

    @Override
    public void start(Stage stage) throws Exception {

        FXMLLoader loader = new FXMLLoader();
        loader.setLocation(getClass().getResource("FXMLDocument.fxml"));
        Parent root = loader.load();
        
        Scene scene = new Scene(root);
        
        stage.setScene(scene);
        stage.show();
    }

}

O método load executa várias ações, e é útil entender a ordem em que elas acontecem. Neste exemplo simples:

  1. O FXMLLoader lê e analisa o arquivo FXML. Ele cria objetos correspondentes aos elementos definidos no arquivo e anota quaisquer atributos fx:id definidos neles.

  2. Como o elemento raiz do arquivo FXML definiu um atributo fx:controller, o FXMLLoader cria uma nova instância da classe que especifica. Por padrão, isso acontece invocando o construtor sem argumento na classe especificada.

  3. Quaisquer elementos com atributos fx:id definidos que tenham campos no controlador com nomes de campos correspondentes e que sejam públicos (não recomendado) ou anotados @FXML (recomendado) são “injetados” nos correspondentes Campos. Portanto, neste exemplo, como há um Label no arquivo FXML com fx:id="label" e um campo no controlador definido como

     @FXML
 private Label label ;

    the label field is initialized with the Label instance created by the FXMLLoader.

  4. Os manipuladores de eventos são registrados com quaisquer elementos no arquivo FXML com as propriedades onXXX="#..." definidas. Esses manipuladores de eventos invocam o método especificado na classe do controlador. Neste exemplo, como o Button tem onAction="#handleButtonAction", e o controlador define um método

     @FXML
 private void handleButtonAction(ActionEvent event) { ... }

    when an action is fired on the button (e.g. the user presses it), this method is invoked. The method must have void return type, and can either define a parameter matching the event type (ActionEvent in this example), or can define no parameters.

  5. Finalmente, se a classe do controlador define um método initialize, este método é invocado. Observe que isso acontece após os campos @FXML serem injetados, para que possam ser acessados ​​com segurança neste método e serão inicializados com as instâncias correspondentes aos elementos no arquivo FXML. O método initialize() pode não receber parâmetros ou pode receber um URL e um ResourceBundle. No último caso, esses parâmetros serão preenchidos pelo URL representando a localização do arquivo FXML, e qualquer ResourceBundle definido no FXMLLoader via loader.setResources(...). Qualquer um deles pode ser null se não tiver sido definido.

Controladores aninhados #

Não há necessidade de criar toda a interface do usuário em um único FXML usando um único controlador.

A tag <fx:include> pode ser usada para incluir um arquivo fxml em outro. O controlador do fxml incluído pode ser injetado no controlador do arquivo incluído como qualquer outro objeto criado pelo FXMLLoader.

Isso é feito adicionando o atributo fx:id ao elemento <fx:include>. Desta forma, o controlador do fxml incluído será injetado no campo com o nome <fx:id value>Controller.

Exemplos:

fx:valor id nome do campo para injeção
foo fooController
resposta42 answer42Controlador
xYz xYzController

Exemplos de fxmls

Contador

Este é um fxml contendo um StackPane com um nó Text. O controlador para este arquivo fxml permite obter o valor atual do contador, bem como incrementar o contador:

counter.fxml

<?xml version="1.0" encoding="UTF-8"?>

<?import javafx.scene.text.*?>
<?import javafx.scene.layout.*?>

<StackPane prefHeight="200" prefWidth="200" xmlns:fx="http://javafx.com/fxml/1" fx:controller="counter.CounterController">
    <children>
        <Text fx:id="counter" />
    </children>
</StackPane>

ContadorController

package counter;

import javafx.fxml.FXML;
import javafx.scene.text.Text;

public class CounterController {
    @FXML
    private Text counter;

    private int value = 0;
    
    public void initialize() {
        counter.setText(Integer.toString(value));
    }
    
    public void increment() {
        value++;
        counter.setText(Integer.toString(value));
    }
    
    public int getValue() {
        return value;
    }
    
}

Incluindo fxml

<?xml version="1.0" encoding="UTF-8"?>

<?import javafx.scene.control.*?>
<?import javafx.scene.layout.*?>

<BorderPane prefHeight="500" prefWidth="500" xmlns="http://javafx.com/javafx/8" xmlns:fx="http://javafx.com/fxml/1" fx:controller="counter.OuterController">
    <left>
        <Button BorderPane.alignment="CENTER" text="increment" onAction="#increment" />
    </left>
    <center>
        <!-- content from counter.fxml included here -->
        <fx:include fx:id="count" source="counter.fxml" />
    </center>
</BorderPane>

Controlador Externo

O controlador do fxml incluído é injetado neste controlador. Aqui o manipulador para o evento onAction para o Button é usado para incrementar o contador.

package counter;

import javafx.fxml.FXML;

public class OuterController {

    // controller of counter.fxml injected here
    @FXML
    private CounterController countController;

    public void initialize() {
        // controller available in initialize method
        System.out.println("Current value: " + countController.getValue());
    }

    @FXML
    private void increment() {
        countController.increment();
    }

}

Os fxmls podem ser carregados assim, assumindo que o código é chamado de uma classe no mesmo pacote que outer.fxml:

Parent parent = FXMLLoader.load(getClass().getResource("outer.fxml"));

Defina Blocos e fx:reference #

Às vezes, um elemento precisa ser criado fora da estrutura de objeto usual no fxml.

É aqui que Define Blocks entra em jogo:

O conteúdo dentro de um elemento <fx:define> não é adicionado ao objeto criado para o elemento pai.

Cada elemento filho do <fx:define> precisa de um atributo fx:id.

Objetos criados dessa maneira podem ser referenciados posteriormente usando o elemento <fx:reference> ou usando a associação de expressão.

O elemento <fx:reference> pode ser usado para referenciar qualquer elemento com um atributo fx:id que é tratado antes do elemento <fx:reference> ser tratado usando o mesmo valor que o fx:id atributo do elemento referenciado no atributo source do elemento <fx:reference>.

<?xml version="1.0" encoding="UTF-8"?>

<?import javafx.scene.text.*?>
<?import java.lang.*?>
<?import javafx.scene.*?>
<?import javafx.scene.control.*?>
<?import javafx.scene.layout.*?>

Passando dados para FXML - acessando o controlador existente #

Problema: alguns dados precisam ser passados ​​para uma cena carregada de um fxml.

Solução

Especifique um controlador usando o atributo fx:controller e obtenha a instância do controlador criada durante o processo de carregamento da instância FXMLLoader usada para carregar o fxml.

Adicione métodos para passar os dados para a instância do controlador e manipule os dados nesses métodos.

FXML

<?xml version="1.0" encoding="UTF-8"?>

<?import javafx.scene.text.*?>
<?import javafx.scene.layout.*?>

<VBox xmlns:fx="http://javafx.com/fxml/1" fx:controller="valuepassing.TestController">
    <children>
        <Text fx:id="target" />
    </children>
</VBox>

Controlador

package valuepassing;

import javafx.fxml.FXML;
import javafx.scene.text.Text;

public class TestController {

    @FXML
    private Text target;

    public void setData(String data) {
        target.setText(data);
    }

}

Código usado para carregar o fxml

String data = "Hello World!";

FXMLLoader loader = new FXMLLoader(getClass().getResource("test.fxml"));
Parent root = loader.load();
TestController controller = loader.<TestController>getController();
controller.setData(data);

Passando dados para FXML - Especificando a instância do controlador #

Problema: alguns dados precisam ser passados ​​para uma cena carregada de um fxml.

Solução

Defina o controlador usando a instância FXMLLoader usada posteriormente para carregar o fxml.

Certifique-se de que o controlador contenha os dados relevantes antes de carregar o fxml.

Nota: neste caso o arquivo fxml não deve conter o atributo fx:controller.

FXML

<?xml version="1.0" encoding="UTF-8"?>

<?import javafx.scene.text.*?>
<?import javafx.scene.layout.*?>

<VBox xmlns:fx="http://javafx.com/fxml/1">
    <children>
        <Text fx:id="target" />
    </children>
</VBox>

Controlador

import javafx.fxml.FXML;
import javafx.scene.text.Text;

public class TestController {

    private final String data;

    public TestController(String data) {
        this.data = data;
    }
    
    @FXML
    private Text target;
    
    public void initialize() {
        // handle data once the fields are injected
        target.setText(data);
    }

}

Código usado para carregar o fxml

String data = "Hello World!";

FXMLLoader loader = new FXMLLoader(getClass().getResource("test.fxml"));

TestController controller = new TestController(data);
loader.setController(controller);

Parent root = loader.load();
