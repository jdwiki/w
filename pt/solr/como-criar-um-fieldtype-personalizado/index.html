Observações sobre o campo personalizado baseado em classe Java:

Esta é uma pequena seção de um grande artigo classificação personalizada no Solr usando campo externo escrito para classificar documentos Solr com base no comparador de campo personalizado.

Observações sobre o campo personalizado criado a partir de campos Solr existentes:

O Apache criou uma documentação detalhada sobre este tópico - Noções básicas sobre analisadores, tokenizadores e filtros.

Crie um tipo de campo Solr personalizado a partir da própria classe Java personalizada #

Alterações de esquema:

Você precisará definir um novo tipo de campo em seu arquivo de esquema solr e, em seguida, poderá criar campos desse tipo. Exemplo de trecho de esquema:

<!-- Source: solr/example/.../conf/schema.xml -->
<?xml version="1.0" encoding="UTF-8" ?>
<schema name="adam" version="1.3">
  <types>
    ...
    <fieldType name="rank_t" class="org.apache.solr.schema.ext.RankFieldType"/>
  </types>
 <fields>
   ...
   <field name="rank" type="rank_t" indexed="true" stored="true"/>
 </fields>
 ...
</schema>

Classe Java para tipo de campo personalizado:

// Source: src/java/org/apache/solr/schema/ext/RankFieldType.java
package org.apache.solr.schema.ext;

import java.io.IOException;

import org.apache.lucene.document.Fieldable;
import org.apache.lucene.search.SortField;
import org.apache.solr.response.TextResponseWriter;
import org.apache.solr.schema.FieldType;
import org.apache.solr.schema.SchemaField;
import org.apache.solr.search.ext.RankFieldComparatorSource;

public class RankFieldType extends FieldType {

    @Override
    public SortField getSortField(SchemaField field, boolean top) {
        return new SortField(field.getName(), new RankFieldComparatorSource(), top);
    }

    @Override
    // copied verbatim from GeoHashField method
    public void write(TextResponseWriter writer, String name, Fieldable f) throws IOException {
         writer.writeStr(name, f.stringValue(), false);
    }
}

Crie um tipo de campo personalizado a partir dos tipos de campo disponíveis #

Vamos obter algum conhecimento teórico antes de passar para o exemplo. Há três termos importantes sendo usados ​​aqui Analyzers, Tokenizers e Filters. Para criar esse campo personalizado, você precisará criar um analisador com um tokenizer e um ou mais filtros. Como mencionado aqui, você pode ter apenas um tokenizer por analisador, mas existem maneiras de superar essa limitação.

<fieldType name="alphaOnlySort" class="solr.TextField" sortMissingLast="true" omitNorms="true">
  <analyzer>
    <tokenizer class="solr.KeywordTokenizerFactory"/>
    <filter class="solr.LowerCaseFilterFactory"/>
    <filter class="solr.TrimFilterFactory"/>
    <filter class="solr.PatternReplaceFilterFactory" replace="all" replacement="" pattern="([^a-z])"/>
  </analyzer>
</fieldType>

Outro exemplo:

<fieldType name="lowercase_text" class="solr.TextField" positionIncrementGap="150">
  <analyzer>
     <tokenizer class="solr.KeywordTokenizerFactory" />
     <filter class="solr.LowerCaseFilterFactory" />
  </analyzer>
</fieldType>

Mais um exemplo com descrição:

<fieldType name="text_stem" class="solr.TextField">
  <analyzer>
    <tokenizer class="solr.StandardTokenizerFactory"/>
    <filter class="solr.StandardFilterFactory"/>
    <filter class="solr.LowerCaseFilterFactory"/>
    <filter class="solr.EnglishPorterFilterFactory"/>
  </analyzer>
</fieldType>

Este exemplo começa com o tokenizer padrão do Solr, que divide o texto do campo em tokens. Esses tokens passam pelo filtro padrão do Solr, que remove pontos de siglas e executa algumas outras operações comuns. Todos os tokens são então configurados para minúsculas, o que facilitará a correspondência que não diferencia maiúsculas de minúsculas no momento da consulta. O último filtro no exemplo acima é um filtro de lematização que usa o algoritmo de lematização de Porter. Um lematizador é basicamente um conjunto de regras de mapeamento que mapeia as várias formas de uma palavra de volta à palavra base, ou raiz, da qual derivam. Por exemplo, em inglês as palavras “hugs”, “hugging” e “hugged” são todas formas da palavra-tronco “hug”. O lematizador substituirá todos esses termos por “abraço”, que é o que será indexado. Isso significa que uma consulta para “hug” corresponderá ao termo “hugged”, mas não “huge”.

Exemplo de uso desse campo personalizado:

<field name="keywords" type="text_stem" indexed="true" stored="true" />
