DataTables 1.10+ Processamento do lado do servidor #

Tabela de exemplo

Existem várias maneiras de injetar seus dados em DataTables. Processamento do lado do servidor é apenas um método. Dessa forma, o DataTables tem um endpoint pré-configurado para recuperar dados, e esse endpoint é responsável por aceitar todas as solicitações de paginação/filtragem/classificação que o DataTables aplica. Há uma variedade de prós e contras para isso em relação ao envio de seu conjunto de dados completo de volta do servidor e permitir que o DataTables faça tudo do lado do cliente, dependendo do seu caso de uso.

var tbl = $('#example').DataTable({
  processing: true,
  serverSide: true,
  ajax: {
    url: '/echo/json/',
    method: 'post'
  },
  columns: [{
    data: 'First',
    title: 'First Name'
  }, {
    data: 'Last',
    title: 'Last Name'
  }]
});

Exemplo de conexão do evento preXhr para enviar dados adicionais à solicitação ajax

Este evento é acionado diretamente antes que a chamada ajax seja feita, permitindo que você modifique o corpo da solicitação. Isso é útil se houver um formulário que influencie os dados retornados à tabela (imagine talvez uma barra mín./máx. para filtrar dentro de um intervalo de datas ou similar).

tbl.on('preXhr.dt', function(ev, settings, data) {
    $.extend(data, {
      min: $('form [name=min]').val(), 
      max: $('form [name=max]').val()
    });
});

Explicação dos requisitos do lado do servidor para processar a solicitação

Em uma instância típica sem opção de DataTables, toda filtragem, classificação e paginação são tratadas no navegador do cliente. Com o processamento do lado do servidor ativado, essas tarefas são transferidas para o servidor da web. Para conjuntos de dados muito grandes que podem ser ineficientes para enviar em sua totalidade ao cliente, isso pode ajudar.

Existem vários parâmetros padrão que são enviados pela solicitação Datatables quando você configura um endpoint ajax. Pode ficar muito longo, então, em vez de detalhar cada um, uma visão geral das responsabilidades do servidor pode ser mais útil.

Começando com quaisquer parâmetros opcionais que você possa ter fornecido à solicitação, compile seu conjunto de dados e prepare-o para operações de DataTables. Use os parâmetros search[value] e search[regex] para aplicar qualquer filtragem em todas as colunas/propriedades. Existem também parâmetros columns[i][search][value] e columns[i][search][regex] para filtros de coluna individuais, embora estes não sejam comumente usados ​​em instâncias simples de DataTables.

Classifique seus dados filtrados usando os vários parâmetros order. Tal como acontece com os parâmetros de pesquisa, haverá um conjunto de parâmetros de ordenação por coluna na qual a ordenação está habilitada. pedido[i][coluna] e pedido[i][coluna][dir]

Depois que a filtragem e a classificação estiverem concluídas, é hora de paginar os dados com base na solicitação DataTables usando os parâmetros start e length. Em .NET isso pode ser assim:

int start = Int32.TryParse(Request["start"]);
int length = Int32.TryParse(Request["length"]); 
return MyData.Skip(start).Take(length);

Exemplo de resposta

Essa é a estrutura de resposta aproximada que o DataTables espera. Quaisquer que sejam seus dados (array 2d, array de objetos, etc) está aninhado na propriedade data com as outras propriedades propagando o núcleo do DataTable para desenhar informações sobre paginação e filtragem (por exemplo, “Mostrando registros 11-20 de 500 (filtrados de 1000 )”)

{
  "draw": 1,
  "recordsTotal": 57,
  "recordsFiltered": 57,
  "data": [/*your data goes here*/]
}

Consulte também .rows.add(data) e a data option para métodos alternativos de configuração do conteúdo de sua DataTable usando dados JSON. É claro que DataTables também pode ser [inicializado a partir de atributos estáticos HTML ou HTML5 data-] 3

Carregar dados usando ajax com processamento do lado do servidor. #

 var MY_AJAX_ACTION_URL = "path/to/controller.php";

 var table = $('#user_list_table').DataTable({
        "autoWidth": true,
        "paging": true,
        "searching": true,
        "ordering": true,
        "language": {
          "zeroRecords": "No data Found",
          "processing": 'Loading'
        },
        "info": false,
        "stripeClasses": [ "odd nutzer_tr", "even nutzer_tr"],
        "columns": [
            {'data':'uid',"visible": false},
            {'data':'name','orderable': true},
            {'data':'phone','orderable': true},
            {'data':'email','orderable': true },
            {'data':'address','orderable': true}
        ],
        "order": [[ 1, "desc" ]],
        "processing": true,
        "serverSide": true,
        "ajax":MY_AJAX_ACTION_URL
    });

A resposta da chamada para MY_AJAX_ACTION_URL deve estar estritamente no formato abaixo:

{
  "draw": 1,
  "recordsTotal": 2,
  "recordsFiltered": 2,
  "data": [
      {"name":"XYZ","phone":"678654454","email":"xyz@gmail.com","address":"true"},
      {"name":"ABC","phone":"678654455","email":"abc@gmail.com","address":"true"}
  ]
}

Observe que se a saída da chamada não corresponder ao formato acima, resultará em um erro na inicialização de table.

Obter dados JSON da tabela MySQL #

No site oficial do DataTable está um exemplo de como pode ser um processo do lado do servidor com PHP e MySQL. Este exemplo está obsoleto e não pode mais ser usado com o PHP 7 (a função “mysql_pconnect” e as funções associadas estão obsoletas, veja este post).

Portanto, esta função fornece dados JSON bem formados como resposta:

<?php
//include database connection file
$servername = "localhost";
$username = "root";
$password = "root";
$dbname = "name";

$conn = mysqli_connect($servername, $username, $password, $dbname) or die("Connection failed: " . mysqli_connect_error());

/* check connection */
if (mysqli_connect_errno()) {
  printf("Connect failed: %s\n", mysqli_connect_error());
  exit();
}

// initilize all variable
$params = $columns = $totalRecords = $data = array();
$params = $_REQUEST;
//define index of column name
$columns = array(
    0 =>'id',
    1 =>'name',
    2 =>'salery',
);

$where = $sqlTot = $sqlRec = "";

// check search value exist
if( !empty($params['search']['value']) ) {
    $where .=" WHERE ";
    $where .=" ( id LIKE '".$params['search']['value']."%' ";
    $where .=" OR name LIKE '".$params['search']['value']."%' ";
    $where .=" OR salery LIKE '".$params['search']['value']."%' )";
}

// getting total number records without any search
$sql = "SELECT * FROM `employees` ";
$sqlTot .= $sql;
$sqlRec .= $sql;

//concatenate search sql if value exist
if(isset($where) && $where != '') {
    $sqlTot .= $where;
    $sqlRec .= $where;
}

 $sqlRec .=  " ORDER BY ". $columns[$params['order'][0]['column']]."   ".$params['order'][0]['dir']."  LIMIT ".$params['start']." ,".$params['length']." ";

$queryTot = mysqli_query($conn, $sqlTot) or die("database error:". mysqli_error($conn));

$totalRecords = mysqli_num_rows($queryTot);

$queryRecords = mysqli_query($conn, $sqlRec) or die("error to fetch employees data");

while( $row = mysqli_fetch_row($queryRecords) ) {
    $data[] = $row;
}

$json_data = array(
        "draw"            => intval( $params['draw'] ),
        "recordsTotal"    => intval( $totalRecords ),
        "recordsFiltered" => intval($totalRecords),
        "data"            => $data   // total data array
        );

echo json_encode($json_data);  // send data as json format
?>

A resposta parece e pode ser processada pelo DataTable:

      },
      {
        "id":"2",
        "name":"Claudia",
        "salery":"3000"
      },
      {
        "id":"3",

}
