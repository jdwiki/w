Instalação #

Os aplicativos Laravel são instalados e gerenciados com o Composer, um gerenciador de dependências PHP popular. Existem duas maneiras de criar um novo aplicativo Laravel.

Via Compositor #

$ composer create-project laravel/laravel [foldername]

Ou

$ composer create-project --prefer-dist laravel/laravel [foldername]

Substitua [foldername] pelo nome do diretório no qual você deseja que seu novo aplicativo Laravel seja instalado. Não deve existir antes da instalação. Você também pode precisar adicionar o executável do Composer ao caminho do sistema.

Se quiser criar um projeto Laravel usando uma versão específica do framework, você pode fornecer um padrão de versão, caso contrário, seu projeto usará a versão mais recente disponível.

Se você quisesse criar um projeto no Laravel 5.2, por exemplo, você executaria:

$ composer create-project --prefer-dist laravel/laravel 5.2.*

Por que –prefer-dist

Existem duas maneiras de baixar um pacote: source e dist. Para versões estáveis, o Composer usará o dist por padrão. O source é um repositório de controle de versão. Se --prefer-source estiver habilitado, o Composer irá instalar a partir da fonte, se houver.

--prefer-dist é o oposto de --prefer-source, e diz ao Composer para instalar a partir de dist se possível. Isso pode acelerar substancialmente as instalações em servidores de compilação e em outros casos de uso em que você normalmente não executa atualizações do fornecedor. Também permite evitar problemas com o Git caso você não tenha uma configuração adequada.

Através do instalador do Laravel #

O Laravel fornece um utilitário de linha de comando útil para criar rapidamente aplicativos Laravel. Primeiro, instale o instalador:

$ composer global require laravel/installer

Você tem que ter certeza que a pasta de binários do Composer está dentro da sua variável $PATH para executar o instalador do Laravel.

Primeiro, veja se já está na sua variável $PATH

echo $PATH

Se tudo estiver correto, a saída deve conter algo assim:

Users/yourusername/.composer/vendor/bin

Caso contrário, edite seu .bashrc ou, se estiver usando ZSH, seu .zshrc para que ele contenha o caminho para o diretório do fornecedor do Composer.

Uma vez instalado, este comando criará uma nova instalação do Laravel no diretório que você especificar.

laravel new [foldername]

Você também pode usar . (um ponto) no lugar de [nome da pasta] para criar o projeto no diretório de trabalho atual sem criar um subdiretório.

Executando o aplicativo #

Laravel vem com um servidor web baseado em PHP que pode ser iniciado executando

$ php artisan serve

Por padrão, o servidor HTTP usará a porta 8000, mas se a porta já estiver em uso ou se você quiser executar vários aplicativos Laravel de uma só vez, você pode usar o sinalizador --port para especificar uma porta diferente:

$ php artisan serve --port=8080

O servidor HTTP usará localhost como o domínio padrão para executar o aplicativo, mas você pode usar o sinalizador --host para especificar um endereço diferente:

$ php artisan serve --host=192.168.0.100 --port=8080

Usando um servidor diferente #

Se você preferir usar um software de servidor web diferente, alguns arquivos de configuração são fornecidos para você dentro do diretório public do seu projeto; .htaccess para Apache e web.config para ASP.NET. Para outros softwares como o NGINX, você pode converter as configurações do Apache usando várias ferramentas online.

A estrutura precisa que o usuário do servidor web tenha permissões de gravação nos seguintes diretórios:

  • /armazenamento
  • /bootstrap/cache

Em sistemas operacionais *nix, isso pode ser feito por

chown -R www-data:www-data storage bootstrap/cache
chmod -R ug+rwx storage bootstrap/cache

(onde www-data é o nome e grupo do usuário do servidor web)

O servidor web de sua escolha deve ser configurado para servir conteúdo do diretório /public do seu projeto, o que normalmente é feito configurando-o como a raiz do documento. O resto do seu projeto não deve ser acessível através do seu servidor web.

Se você configurar tudo corretamente, navegar até a URL do seu site deve exibir a página de destino padrão do Laravel.

Requisitos #

O framework Laravel tem os seguintes requisitos:

  • PHP >= 5.6.4
  • Extensão XML PHP
  • Extensão PHP PDO
  • Extensão PHP OpenSSL
  • Extensão PHP Mbstring
  • Extensão PHP Tokenizer
  • PHP >= 5.5.9
  • Extensão PHP PDO
  • Laravel 5.1 é a primeira versão do Laravel a suportar PHP 7.0.
  • PHP >= 5.4, PHP < 7
  • Extensão PHP OpenSSL
  • Extensão PHP Tokenizer
  • Extensão PHP Mbstring
  • Extensão JSON PHP (somente no PHP 5.5)
  • PHP >= 5.4
  • Extensão PHP Mbstring
  • Extensão JSON PHP (somente no PHP 5.5)

Exemplo Hello World (Básico) #

Abra o arquivo de rotas. Cole o seguinte código em:

Route::get('helloworld', function () {
    return '<h1>Hello World</h1>';
});

depois de ir para a rota localhost/helloworld, ele exibe Hello World.

O arquivo de rotas está localizado:

Para Web

routes/web.php

Para APIs

routes/api.php

app/Http/routes.php

app/routes.php

Exemplo Hello World (usando o controlador e a visualização) #

  1. Crie um aplicativo Laravel:

     $ composer create-project laravel/laravel hello-world

  2. Navegue até a pasta do projeto, por exemplo.

     $ cd C:\xampp\htdocs\hello-world

  3. Crie um controlador:

     $ php artisan make:controller HelloController --resource

Isso criará o arquivo app/Http/Controllers/HelloController.php. A opção --resource gerará métodos CRUD para o controlador, por exemplo indexar, criar, mostrar, atualizar.

  1. Registre uma rota para o método index do HelloController. Adicione esta linha a app/Http/routes.php (versão 5.0 a 5.2) ou routes/web.php (versão 5.3):
    Route::get('hello', 'HelloController@index');

Para ver suas rotas recém-adicionadas, você pode executar $ php artisan route:list

  1. Crie um template Blade no diretório views:

    resources/views/hello.blade.php:

     <h1>Hello world!</h1>

  2. Agora dizemos ao método index para exibir o template hello.blade.php:

    app/Http/Controllers/HelloController.php

    <?php
    
    namespace App\Http\Controllers;
    
    use Illuminate\Http\Request;
    
    use App\Http\Requests;
    
    class HelloController extends Controller
    {
        /**
         * Display a listing of the resource.
         *
         * @return \Illuminate\Http\Response
         */
        public function index()
        {
            return view('hello');
        }
    
        // ... other resources are listed below the index one above

Você pode servir seu aplicativo usando o seguinte comando PHP Artisan: php artisan serve; ele mostrará o endereço no qual você pode acessar seu aplicativo (geralmente em http://localhost:8000 por padrão).

Alternativamente, você pode ir diretamente para o local apropriado em seu navegador; caso você esteja usando um servidor como o XAMPP (ou: http://localhost/hello-world/public/hello você deve ter instalado sua instância Laravel, hello-world, diretamente em seu xampp/htdocs diretório como em: tendo executado o passo 1 deste Hello Word a partir de sua interface de linha de comando, apontando para seu diretório xampp/htdocs).

Instalação usando LaraDock (Laravel Homestead para Docker) #

LaraDock é um ambiente de desenvolvimento semelhante ao Laravel Homestead, mas para Docker em vez de Vagrant. https://github.com/LaraDock/laradock

Instalação #

*Requer Git e Docker

Clone o repositório LaraDock:

A. Se você já tem um projeto Laravel, clone este repositório em seu diretório raiz Laravel:

git submodule add https://github.com/LaraDock/laradock.git

B. Se você não tem um projeto Laravel e deseja instalar o Laravel do Docker, clone este repositório em qualquer lugar da sua máquina:

git clone https://github.com/LaraDock/laradock.git

Uso Básico #

  1. Executar Containers: (Certifique-se de estar na pasta laradock antes de executar os comandos docker-compose).

Exemplo: Executando NGINX e MySQL: docker-compose up -d nginx mysql

Há uma lista de contêineres disponíveis que você pode selecionar para criar suas próprias combinações.

nginx, hhvm, php-fpm, mysql, redis, postgres, mariadb, neo4j, mongo, apache2, caddy, memcached, beanstalkd, beanstalkd-console, espaço de trabalho

  1. Entre no container Workspace, para executar comandos como (Artisan, Composer, PHPUnit, Gulp, …).

    docker-compose exec workspace bash

  2. Se você ainda não tem um projeto Laravel instalado, siga o passo para instalar o Laravel a partir de um container Docker.

uma. Insira o contêiner Workspace.

b. Instale o Laravel. composer create-project laravel/laravel my-cool-app "5.3.*"

  1. Edite as configurações do Laravel. Abra o arquivo .env do seu Laravel e defina o DB_HOST para o seu mysql:

    DB_HOST=mysql

  2. Abra seu navegador e visite seu endereço de host local.
