Autenticação múltipla #

O Laravel permite que você use vários tipos de autenticação com guardas específicos.

No laravel 5.3 a autenticação múltipla é um pouco diferente do Laravel 5.2

Vou explicar como implementar o recurso de multiautenticação no 5.3

Primeiro você precisa de dois modelos de usuário diferentes

cp App/User.php App/Admin.php

altere o nome da classe para Admin e defina o namespace se você usar modelos diferentes. deve parecer

Aplicativo\Admin.php

<?php

namespace App;

use Illuminate\Foundation\Auth\User as Authenticatable;
use Illuminate\Notifications\Notifiable;

class Admin extends Authenticatable
{
    use Notifiable;

    protected $fillable = ['name', 'email', 'password'];
    protected $hidden   = ['password', 'remember_token'];

}

Além disso, você precisa criar uma migração para o administrador

php artisan make:migration create_admins_table

em seguida, edite o arquivo de migração com o conteúdo da migração de usuário padrão. Se parece com isso

<?php

use Illuminate\Database\Migrations\Migration;
use Illuminate\Database\Schema\Blueprint;
use Illuminate\Support\Facades\Schema;

class CreateAdminsTable extends Migration
{
    /**
     * Run the migrations.
     *
     * @return void
     */
    public function up()
    {
        Schema::create('admins', function (Blueprint $table) {
            $table->increments('id');
            $table->string('name');
            $table->string('email')->unique();
            $table->string('password');        
            $table->rememberToken();
            $table->timestamps();

            $table->softDeletes();
        });
    }

    /**
     * Reverse the migrations.
     *
     * @return void
     */
    public function down()
    {
        Schema::drop('admins');
    }
}

editar config/auth.php

'guards'    => [
        'web'   => [
            'driver'   => 'session',
            'provider' => 'users',
        ],

        'api'   => [
            'driver'   => 'token',
            'provider' => 'users',
        ],
        //Add Admin Guard
        'admin' => [
            'driver'   => 'session',
            'provider' => 'admins',
        ],
    ],

e

'providers' => [
        'users'  => [
            'driver' => 'eloquent',
            'model'  => App\User::class,
        ],
        //Add Admins Provider
        'admins' => [
            'driver' => 'eloquent',
            'model'  => App\Admin::class,
        ],
    ],

Observe que adicionamos duas entradas. um na variável guards um na variável providers.

E é assim que você usa o outro guarda então “web”

Meu aplicativo\Http\Controllers\Admin\LoginController

<?php

namespace App\Http\Controllers\Admin;

use App\Http\Controllers\Controller;
use Illuminate\Foundation\Auth\AuthenticatesUsers;
use Illuminate\Support\Facades\Auth;

class AuthController extends Controller
{

    use AuthenticatesUsers;

    protected $guard = 'admin';

    protected $redirectTo = '/admin/';

    public function showLoginForm()
    {
        return view('admin.login');
    }

    protected function guard()
    {
        return Auth::guard($this->guard);
    }

}

isso precisa de pouca explicação.

em poucas palavras Auth::guard(‘admin’) permitirá que você use métodos de autenticação (como login, logout, registro etc.) com seu admin guard.

Por exemplo

Auth::guard('admin')->login($user)

irá pesquisar $user na tabela de admins e efetuar login com o usuário enquanto

Auth::login($user)

funcionará normalmente com a tabela de usuários. A proteção padrão é especificada em config/auth.php com matriz defaults. Em laravel fresco é “web” .

No controlador, você precisa implementar métodos de AuthenticatesUsers para mostrar seus caminhos de visualização personalizados. E você precisa implementar outras funções, como guard, para usar seus novos guards de usuário.

Neste exemplo, meu login de administrador é admin/login.blade

E ao implementar a função guard() para retornar Auth::guard(‘admin’) todos os métodos de característica AuthenticatesUsers funcionam com o guarda “admin”.

Nas versões anteriores do laravel, isso é um pouco diferente do 5.3

em 5.2 a função getGuard retorna a variável $guard da classe e da função principal (login) use-a em

Auth::guard($guard)->attempt(...)

em 5.3 guard function retorna Auth::guard() inteiro e a função main use-o como

$this->guard()->attempt(...)
