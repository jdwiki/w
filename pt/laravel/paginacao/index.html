Paginação em Laravel #

Em outros frameworks a paginação é dor de cabeça. O Laravel facilita muito, ele pode gerar paginação adicionando algumas linhas de código no Controller e no View.

Uso Básico

Existem muitas maneiras de paginar itens, mas a mais simples é usar o método paginate no construtor de consultas ou uma consulta Eloquent. O Laravel pronto para uso cuida de definir o limite e o deslocamento com base na página atual que está sendo visualizada pelo usuário. Por padrão, a página atual é detectada pelo valor do argumento ?page query string no pedido HTTP. E com certeza, esse valor é detectado pelo Laravel automaticamente e inserido nos links gerados pelo paginator.

Agora digamos que queremos chamar o método paginate na consulta. Em nosso exemplo, o argumento passado para paginar é o número de itens que você gostaria de exibir “por página”. No nosso caso, digamos que queremos exibir 10 itens por página.

<?php

namespace App\Http\Controllers;

use DB;
use App\Http\Controllers\Controller;

class UserController extends Controller
{
    /**
     * Show all of the users for the application.
     *
     * @return Response
     */
    public function index()
    {
        $users = DB::table('users')->paginate(10);

        return view('user.index', ['users' => $users]);
    }
}

Observação: Atualmente, as operações de paginação que usam uma instrução groupBy não pode ser executado eficientemente pelo Laravel. Se você precisar usar um groupBy com um conjunto de resultados paginado, é recomendado que você consulte o banco de dados e crie um paginador manualmente.

Paginação Simples

Digamos que você queira apenas exibir os links Próximo e Anterior em sua visualização de paginação. O Laravel fornece essa opção usando o método simplePaginate.

$users = DB::table('users')->simplePaginate(10);

Exibindo resultados em uma visualização

Agora vamos exibir a paginação em vista. Na verdade, quando você chama os métodos paginate ou simplePaginate na consulta do Eloquent, você recebe uma instância do paginador. Quando o método paginate é chamado, você recebe uma instância de Illuminate\Pagination\LengthAwarePaginator, enquanto quando você chama o método simplePaginate, você recebe uma instância de Illuminate\Pagination\Paginator. Essas instâncias/objetos vem com vários métodos que explicam o conjunto de resultados. Além disso, além desses métodos auxiliares, as instâncias do paginador são iteradoras e podem ser repetidas como uma matriz.

Depois de receber os resultados, você pode renderizar facilmente os links da página usando o blade

<div class="container">
    @foreach ($users as $user)
        {{ $user->name }}
    @endforeach
</div>

{{ $users->links() }}

O método links renderizará automaticamente os links para outras páginas no conjunto de resultados. Cada um desses links conterá o número de página específico, ou seja, a variável de string de consulta ?page. O HTML gerado pelo método links é perfeitamente compatível com o framework Bootstrap CSS.

Alterando visualizações de paginação #

Ao usar a paginação laravel, você pode usar suas próprias visualizações personalizadas. Portanto, ao chamar o método links em uma instância do paginador, passe o nome da visualização como o primeiro argumento para o método como:

{{ $paginator->links('view.name') }}

ou

Você pode personalizar as visualizações de paginação exportando-as para o diretório resources/views/vendor usando o comando vendor:publish:

php artisan vendor:publish --tag=laravel-pagination

Este comando colocará as visualizações no diretório resources/views/vendor/pagination. O arquivo default.blade.php dentro deste diretório corresponde à visualização de paginação padrão. Edite este arquivo para modificar o HTML da paginação.
