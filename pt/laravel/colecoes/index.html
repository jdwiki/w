Sintaxe #

  • $coleção = coletar([‘Valor1’, ‘Valor2’, ‘Valor3’]); // Chaves padrão para 0, 1, 2, …,

Illuminate\Support\Collection fornece uma interface fluente e conveniente para lidar com arrays de dados. Você pode muito bem ter usado isso sem saber, por exemplo, consultas de modelo que buscam vários registros retornam uma instância de Illuminate\Support\Collection.

Para documentação atualizada sobre Coleções, você pode encontrar a documentação oficial aqui

Criando coleções #

Usando o auxiliar collect(), você pode criar facilmente novas instâncias de coleção passando um array como:

$fruits = collect(['oranges', 'peaches', 'pears']);

Se você não quiser usar funções auxiliares, você pode criar uma nova coleção usando a classe diretamente:

$fruits = new Illuminate\Support\Collection(['oranges', 'peaches', 'pears']);

Como mencionado nas observações, os modelos por padrão retornam uma instância Collection, porém você pode criar suas próprias coleções conforme necessário. Se nenhuma matriz for especificada na criação, uma coleção vazia será criada.

Onde() #

Você pode selecionar certos itens de uma coleção usando o método where().

$data = [
    ['name' => 'Taylor',  'coffee_drinker' => true],
    ['name' => 'Matt', 'coffee_drinker' => true]
];

$matt = collect($data)->where('name', 'Matt');

Este pedaço de código irá selecionar todos os itens da coleção onde o nome é ‘Matt’. Nesse caso, apenas o segundo item é retornado.

Aninhamento #

Assim como a maioria dos métodos de array no Laravel, where() também suporta a busca de elementos aninhados. Vamos estender o exemplo acima adicionando um segundo array:

$data = [
    ['name' => 'Taylor',  'coffee_drinker' => ['at_work' => true, 'at_home' => true]],
    ['name' => 'Matt', 'coffee_drinker' => ['at_work' => true, 'at_home' => false]]
];

$coffeeDrinkerAtHome = collect($data)->where('coffee_drinker.at_home', true);

Isso só vai devolver Taylor, já que ele toma café em casa. Como você pode ver, o aninhamento é suportado usando a notação de ponto.

Aditivos #

Ao criar uma coleção de objetos em vez de arrays, eles também podem ser filtrados usando where(). A Coleção tentará então receber todas as propriedades desejadas.

Por favor, note que desde o Laravel 5.3 o método where() tentará comparar livremente os valores por padrão. Isso significa que ao pesquisar por (int)1, todas as entradas contendo '1' também serão retornadas. Se você não gostar desse comportamento, você pode usar o método whereStrict().

Usando o Pluck para extrair certos valores de uma coleção #

Muitas vezes, você se deparará com uma coleção de dados em que está interessado apenas em partes dos dados.

No exemplo abaixo, temos uma lista de participantes em um evento e queremos fornecer ao guia turístico uma lista simples de nomes.

// First we collect the participants
$participants = collect([
    ['name' => 'John', 'age' => 55],
    ['name' => 'Melissa', 'age' => 18],
    ['name' => 'Bob', 'age' => 43],
    ['name' => 'Sara', 'age' => 18],
]);

// Then we ask the collection to fetch all the names
$namesList = $partcipants->pluck('name')
// ['John', 'Melissa', 'Bob', 'Sara'];

Você também pode usar pluck para coleções de objetos ou arrays/objetos aninhados com notação de ponto.

$users = User::all(); // Returns Eloquent Collection of all users
$usernames = $users->pluck('username'); // Collection contains only user names

$users->load('profile'); // Load a relationship for all models in collection

// Using dot notation, we can traverse nested properties
$names = $users->pluck('profile.first_name'); // Get all first names from all user profiles

Usando macro() para estender coleções #

A função macro() permite adicionar novas funcionalidades aos objetos Illuminate\Support\Collection

Uso:

Collection::macro("macro_name", function ($parameters) {
    // Your macro
});

Por exemplo:

Collection::macro('uppercase', function () {
    return $this->map(function ($item) {
        return strtoupper($item);
    });
});

collect(["hello", "world"])->uppercase();

Resultado: ["HELLO", "WORLD"]

Usando sintaxe de matriz #

O objeto Collection implementa a interface ArrayAccess e IteratorAggregate, permitindo que seja usado como um array.

Elemento de coleção de acesso:

 $collection = collect([1, 2, 3]);
 $result = $collection[1];

Resultado: 2

Atribuir novo elemento:

$collection = collect([1, 2, 3]);
$collection[] = 4;

Resultado: $collection é [1, 2, 3, 4]

Coleção de loops:

$collection = collect(["a" => "one", "b" => "two"]);
$result = "";
foreach($collection as $key => $value){
    $result .= "(".$key.": ".$value.") ";        
}

Resultado: $result é (a: um) (b: dois)

Conversão de matriz para coleção:

Para converter uma coleção em um array PHP nativo, use:

$array = $collection->all();
//or
$array = $collection->toArray()

Para converter um array em uma coleção, use:

$collection = collect($array);

Usando coleções com funções de matriz

Por favor, esteja ciente de que coleções são objetos normais que não serão convertidos corretamente quando usados ​​por funções que requerem explicitamente arrays, como array_map($callback).

Certifique-se de converter a coleção primeiro ou, se disponível, use o método fornecido pela classe Collection: $collection->map($callback)

Usando Contains para verificar se uma coleção atende a determinada condição #

Um problema comum é ter uma coleção de itens que precisam atender a determinados critérios. No exemplo abaixo, coletamos dois itens para um plano de dieta e queremos verificar se a dieta não contém alimentos não saudáveis.

// First we create a collection
$diet = collect([
    ['name' => 'Banana', 'calories' => '89'],
    ['name' => 'Chocolate', 'calories' => '546']
]);

// Then we check the collection for items with more than 100 calories
$isUnhealthy = $diet->contains(function ($i, $snack) {
    return $snack["calories"] >= 100;
});

No caso acima, a variável $isUnhealthy será definida como true já que Chocolate atende a condição, e a dieta não é saudável.

Usando sum, avg, min ou max em uma coleção para cálculos estatísticos #

As coleções também fornecem uma maneira fácil de fazer cálculos estatísticos simples.

$books = [
    ['title' => 'The Pragmatic Programmer', 'price' => 20],
    ['title' => 'Continuous Delivery', 'price' => 30],
    ['title' => 'The Clean Coder', 'price' => 10],
]

$min = collect($books)->min('price'); // 10
$max = collect($books)->max('price'); // 30
$avg = collect($books)->avg('price'); // 20
$sum = collect($books)->sum('price'); // 60

Usando reduce() #

O método reduce reduz a coleção a um único valor, passando o resultado de cada iteração para a iteração subsequente. Consulte método de redução.

O método reduce percorre cada item com uma coleção e produz um novo resultado para a próxima iteração. Cada resultado da última iteração é passado pelo primeiro parâmetro (nos exemplos a seguir, como $carry).

Esse método pode fazer muito processamento em grandes conjuntos de dados. Por exemplo, os exemplos a seguir, usaremos os seguintes dados do aluno de exemplo:

 $student = [
    ['class' => 'Math', 'score' => 60],
    ['class' => 'English', 'score' => 61],
    ['class' => 'Chemistry', 'score' => 50],
    ['class' => 'Physics', 'score' => 49],
];

Soma a pontuação total do aluno

$sum = collect($student)
    ->reduce(function($carry, $item){
        return $carry + $item["score"];
    }, 0);

Resultado: 220

Explicação:

  • $carry é o resultado da última iteração.
  • O segundo parâmetro é o valor padrão para $carry na primeira rodada de iteração. Neste caso, o valor padrão é 0

Passe um aluno se todas as pontuações forem >= 50

$isPass = collect($student)
    ->reduce(function($carry, $item){
        return $carry && $item["score"] >= 50;
    }, true);

Resultado: false

Explicação:

  • O valor padrão de $carry é verdadeiro
  • Se toda pontuação for maior que 50, o resultado retornará verdadeiro; se for menor que 50, retorna false.

Reprovar um aluno se alguma pontuação for < 50

$isFail = collect($student)
    ->reduce(function($carry, $item){
        return $carry || $item["score"] < 50;
    }, false);

Resultado: true

Explique:

  • o valor padrão de $carry é false
  • se alguma pontuação for menor que 50, retornar verdadeiro; se todas as pontuações forem maiores que 50, retorne false.

Retorne o assunto com a maior pontuação

$highestSubject = collect($student)
    ->reduce(function($carry, $item){
        return $carry === null || $item["score"] > $carry["score"] ? $item : $carry;
    });

resultado: [ "assunto" => "inglês", "pontuação" => 61 ]

Explique:

  • O segundo parâmetro não é fornecido neste caso.
  • O valor padrão de $carry é nulo, portanto, verificamos isso em nossa condicional.

Usando Get para pesquisar valor ou retornar o padrão #

Muitas vezes você se encontra em uma situação em que precisa encontrar um valor de variável correspondente, e as coleções o cobrem.

No exemplo abaixo, temos três localidades diferentes em uma matriz com um código de chamada correspondente atribuído. Queremos ser capazes de fornecer uma localidade e, em troca, obter o código de chamada associado. O segundo parâmetro em get é um parâmetro padrão se o primeiro parâmetro não for encontrado.

function lookupCallingCode($locale)
{
    return collect([
        'de_DE' => 49,
        'en_GB' => 44,
        'en_US' => 1,
    ])->get($locale, 44);
}

No exemplo acima podemos fazer o seguinte

lookupCallingCode('de_DE'); // Will return 49
lookupCallingCode('sv_SE'); // Will return 44

Você pode até passar um retorno de chamada como o valor padrão. O resultado do retorno de chamada será retornado se a chave especificada não existir:

    return collect([
        'de_DE' => 49,
        'en_GB' => 44,
        'en_US' => 1,
    ])->get($locale, function() {
        return 44;
    });

Usando Map para manipular cada elemento em uma coleção #

Muitas vezes, você precisa alterar a forma como um conjunto de dados é estruturado e manipular determinados valores.

No exemplo abaixo, obtivemos uma coleção de livros com um valor de desconto anexado. Mas preferimos ter uma lista de livros com um preço já com desconto.

$books = [
    ['title' => 'The Pragmatic Programmer', 'price' => 20, 'discount' => 0.5],
    ['title' => 'Continuous Delivery', 'price' => 25, 'discount' => 0.1],
    ['title' => 'The Clean Coder', 'price' => 10, 'discount' => 0.75],
];

$discountedItems =  collect($books)->map(function ($book) {
   return ['title' => $book["title"], 'price' => $book["price"] * $book["discount"]];
});

//[
//    ['title' => 'The Pragmatic Programmer', 'price' => 10],
//    ['title' => 'Continuous Delivery', 'price' => 12.5],
//    ['title' => 'The Clean Coder', 'price' => 5],
//]

Isso também pode ser usado para alterar as chaves, digamos que queiramos alterar a chave title para name esta seria uma solução adequada.

Classificando uma coleção #

Existem várias maneiras diferentes de classificar uma coleção.

Ordenar() #

O método sort ordena a coleção:

$collection = collect([5, 3, 1, 2, 4]);

$sorted = $collection->sort();

echo $sorted->values()->all();

returns : [1, 2, 3, 4, 5]

O método sort também permite passar um retorno de chamada personalizado com seu próprio algoritmo. Sob o capô, a classificação usa o usort do php.

$collection = $collection->sort(function ($a, $b) {
    if ($a == $b) {
        return 0;
    }
    return ($a < $b) ? -1 : 1;
});

Ordenar por() #

O método sortBy classifica a coleção pela chave fornecida:

$collection = collect([
['name' => 'Desk', 'price' => 200],
['name' => 'Chair', 'price' => 100],
['name' => 'Bookcase', 'price' => 150],
]);

$sorted = $collection->sortBy('price');

echo $sorted->values()->all();

returns:  [
        ['name' => 'Chair', 'price' => 100],
        ['name' => 'Bookcase', 'price' => 150],
        ['name' => 'Desk', 'price' => 200],
    ]

O método sortBy permite usar o formato de notação de ponto para acessar uma chave mais profunda para classificar um array multidimensional.

$collection = collect([
    ["id"=>1,"product"=>['name' => 'Desk', 'price' => 200]],
    ["id"=>2, "product"=>['name' => 'Chair', 'price' => 100]],
    ["id"=>3, "product"=>['name' => 'Bookcase', 'price' => 150]],
    ]);

$sorted = $collection->sortBy("product.price")->toArray();

return: [
  ["id"=>2, "product"=>['name' => 'Chair', 'price' => 100]],
  ["id"=>3, "product"=>['name' => 'Bookcase', 'price' => 150]],
  ["id"=>1,"product"=>['name' => 'Desk', 'price' => 200]],
]

SortByDesc() #

Este método tem a mesma assinatura que o método sortBy, mas ordenará a coleção na ordem oposta.
