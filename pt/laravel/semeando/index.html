A propagação de banco de dados permite inserir dados, dados de teste gerais em seu banco de dados. Por padrão existe uma classe DatabaseSeeder em database/seeds.

Semeadores em execução podem ser feitos com

php artisan db:seed

Ou se você deseja processar apenas uma única classe

php artisan db:seed --class=TestSeederClass

Tal como acontece com todos os comandos do artisan, você tem acesso a uma ampla variedade de métodos que podem ser encontrados na documentação da api

Criando um Semeador #

Para criar seeders, você pode usar o comando make:seeder Artisan. Todos os seeders gerados serão colocados no diretório database/seeds.

$ php artisan make:seeder MoviesTableSeeder

Semeadores gerados conterão um método: run. Você pode inserir dados em seu banco de dados neste método.

<?php

use Illuminate\Database\Seeder;
use Illuminate\Database\Eloquent\Model;

class MoviesTableSeeder extends Seeder
{
    /**
     * Run the database seeds.
     *
     * @return void
     */
    public function run()
    {
        App\Movie::create([
            'name' => 'A New Hope',
            'year' => '1977'
        ]);

        App\Movie::create([
            'name' => 'The Empire Strikes Back',
            'year' => '1980'
        ]);
    }
}

Você geralmente vai querer chamar todos os seus seeders [dentro da classe DatabaseSeeder](https://www.wikiod.com/pt/laravel/semeando#Chamando outros semeadores).

Assim que terminar de escrever os seeders, use o comando db:seed. Isto irá executar a função run do DatabaseSeeder.

$ php artisan db:seed

Você também pode especificar para executar uma classe de seeder específica para ser executada individualmente usando a opção --class.

$ php artisan db:seed --class=UserSeeder

Se você deseja reverter e executar novamente todas as migrações e, em seguida, propagar novamente:

$ php artisan migrate:refresh --seed

O comando migrate:refresh --seed é um atalho para estes 3 comandos:

$ php artisan migrate:reset # rollback de todas as migrações
$ php artisan migrate # executa migrações
$ php artisan db:seed # executa seeders

Inserindo dados #

Existem várias maneiras de inserir dados:

Usando a fachada do banco de dados #

public function run()
{
    DB::table('users')
        ->insert([
            'name' => 'Taylor',
            'age'  => 21
        ]);
}

Por meio da instanciação de um modelo #

public function run()
{
    $user = new User;
    $user->name = 'Taylor';
    $user->save();
}

Usando o método de criação #

public function run()
{
    User::create([
        'name' => 'Taylor',
        'age'  => 21
    ]);
}

Usando a fábrica #

public function run()
{
    factory(App\User::class, 10)->create();
}

Semeando && excluindo dados antigos e redefinindo o incremento automático #

public function run()
{
    DB::table('users')->delete();
    DB::unprepared('ALTER TABLE users AUTO_INCREMENT=1;');
    factory(App\User::class, 200)->create();
}

Consulte o exemplo Persisting para obter mais informações sobre como inserir/atualizar dados.

Chamando outros semeadores #

Dentro de sua classe DatabaseSeeder você pode chamar outros seeders

$this->call(TestSeeder::class)

Isso permite que você mantenha um arquivo onde você pode encontrar facilmente seus semeadores. Tenha em mente que você precisa prestar atenção à ordem de suas chamadas em relação às restrições de chave estrangeira. Você não pode fazer referência a uma tabela que ainda não existe.

Resemeadura segura #

Você pode querer propagar novamente seu banco de dados sem afetar suas sementes criadas anteriormente. Para isso, você pode usar firstOrCreate em seu seeder:

EmployeeType::firstOrCreate([
    'type' => 'manager',
]);

Então você pode propagar o banco de dados:

php artisan db:seed

Mais tarde, se você quiser adicionar outro tipo de funcionário, basta adicionar esse novo no mesmo arquivo:

EmployeeType::firstOrCreate([
    'type' => 'manager',
]);
EmployeeType::firstOrCreate([
    'type' => 'secretary',
]);

E semeie seu banco de dados novamente sem problemas:

php artisan db:seed

Observe que na primeira chamada você está recuperando o registro, mas não fazendo nada com ele.
