O Laravel suporta o mecanismo de modelagem Blade pronto para uso. O motor de templates Blade nos permite criar templates master e templates filhos carregando conteúdo de templates master, podemos ter variáveis, loops e declarações condicionais dentro do arquivo blade.

Visualizações: introdução #

As visualizações, em um padrão MVC, contêm a lógica de como apresentar dados ao usuário. Em um aplicativo da Web, normalmente eles são usados ​​para gerar a saída HTML que é enviada de volta aos usuários com cada resposta. Por padrão, as visualizações no Laravel são armazenadas no diretório resources/views.

Uma visão pode ser chamada usando a função auxiliar view:

view(string $path, array $data = [])

O primeiro parâmetro do auxiliar é o caminho para um arquivo de visualização e o segundo parâmetro é uma matriz opcional de dados a serem passados ​​para a visualização.

Portanto, para chamar o resources/views/example.php, você usaria:

view('example');

Arquivos de visualização em subpastas dentro do diretório resources/views, como resources/views/parts/header/navigation.php, podem ser chamados usando a notação de ponto: view('parts.header.navigation');

Dentro de um arquivo de visualização, como resources/views/example.php, você pode incluir HTML e PHP juntos:

<html>
    <head>
        <title>Hello world!</title>
    </head>
    <body>
        <h1>Welcome!</h1>
        <p>Your name is: <?php echo $name; ?></p>
    </body>
</html>

No exemplo anterior (que não usa nenhuma sintaxe específica do Blade), geramos a variável $name. Para passar esse valor para nossa view, passaríamos um array de valores ao chamar o view helper:

view('example', ['name' => $name]);

ou alternativamente, use o auxiliar compact(). Neste caso, a string passada para compact() corresponde ao nome da variável que queremos passar para a view.

view('example', compact('name'));

CONVENÇÃO DE NOMES PARA VARIÁVEIS DE LÂMINA

Ao enviar dados de volta para visualização. Você pode usar underscore para multi-palavras variablemas com - laravel dá erro.

Como este dará erro (observe hífen ( - ) dentro do endereço do usuário

view('example',['user-address' => 'Some Address']);

A maneira correta de fazer isso será

view('example', ['user_address' => 'Some Address']);

Herança de layout #

Um layout é um arquivo de visualização, que é estendido por outras visualizações que injetam blocos de código em seu pai. Por exemplo:

parent.blade.php:

<html>
    <head>
        <style type='text/css'>
        @yield('styling')
        </style>
    </head>
    <body>
        <div class='main'>
        @yield('main-content')
        </div>
    </body>
</html>

filho.blade.php:

@extends('parent')

@section('styling')
.main {
    color: red;
}
@stop

@section('main-content')
This is child page!
@stop

otherpage.blade.php:

@extends('parent')

@section('styling')
.main {
    color: blue;
}
@stop

@section('main-content')
This is another page!
@stop

Aqui você vê duas páginas filhas de exemplo, cada uma estendendo a pai. As páginas filhas definem um @section, que é inserido no pai na instrução @yield apropriada.

Assim, a visão renderizada por View::make('child') dirá “**Esta é a página filha!” em vermelho, enquanto View::make('otherpage') produzirá o mesmo html, exceto com o texto “**Esta é outra página!” em azul.

É comum separar os arquivos de visualização, ex. ter uma pasta de layouts específica para os arquivos de layout e uma pasta separada para as várias visualizações individuais específicas.

Os layouts destinam-se a aplicar o código que deve aparecer em todas as páginas, por exemplo, adicionando uma barra lateral ou cabeçalho, sem ter que escrever todo o clichê html em cada visualização individual.

As visualizações podem ser estendidas repetidamente - ou seja, page3 pode @extend('page2') e page2 pode @extend('page1').

O comando extend usa a mesma sintaxe usada para View::make e @include, então o arquivo layouts/main/page.blade.php é acessado como layouts.main.page.

Estruturas de controle #

Blade fornece sintaxe conveniente para estruturas de controle PHP comuns.

Cada uma das estruturas de controle começa com @[structure] e termina com @[endstructure]. Observe que dentro das tags, estamos apenas digitando HTML normal e incluindo variáveis ​​com a sintaxe Blade.

Condicionais #

Declarações ‘Se’ #

@if ($i > 10)
    <p>{{ $i }} is large.</p>
@elseif ($i == 10)
    <p>{{ $i }} is ten.</p>
@else
    <p>{{ $i }} is small.</p>
@endif

‘A menos que’ declarações #

(Sintaxe curta para ‘se não’.)

@unless ($user->hasName())
    <p>A user has no name.</p>
@endunless

Rotações #

‘Enquanto’ loop #

@while (true)
    <p>I'm looping forever.</p>
@endwhile

Loop ‘Foreach’ #

@foreach ($users as $id => $name)
    <p>User {{ $name }} has ID {{ $id }}.</p>
@endforeach

Loop ‘Forel’ #

(O mesmo que o loop ‘foreach’, mas adiciona uma diretiva especial @empty, que é executada quando a expressão de array iterada está vazia, como uma forma de mostrar o conteúdo padrão.)

@forelse($posts as $post)
    <p>{{ $post }} is the post content.</p>
@empty
    <p>There are no posts.</p>
@endforelse

Dentro dos loops, uma variável especial $loop estará disponível, contendo informações sobre o estado do loop:

Propriedade Descrição
$loop->index O índice da iteração de loop atual (começa em 0).
$loop->iteration A iteração do loop atual (começa em 1).
$loop->restante As iterações de loop restantes.
$loop->contagem O número total de itens na matriz sendo iterados.
$loop->primeiro Se esta é a primeira iteração através do loop.
$loop->last Se esta é a última iteração através do loop.
$loop->profundidade O nível de aninhamento do loop atual.
$loop->pai Quando em um loop aninhado, a variável de loop do pai.

Exemplo:

@foreach ($users as $user)
  @foreach ($user->posts as $post)
        @if ($loop->parent->first)
            This is first iteration of the parent loop.
        @endif
    @endforeach
@endforeach

Desde o Laravel 5.2.22, também podemos usar as diretivas @continue e @break

Propriedade Descrição
@continuar Pare a iteração atual e inicie a próxima.
@quebrar Pare o loop atual.

Exemplo :

@foreach ($users as $user)
    @continue ($user->id == 2)
        <p>{{ $user->id }} {{ $user->name }}</p>
    @break ($user->id == 4)
@endforeach

Então (assumindo que mais de 5 usuários são classificados por ID e nenhum ID está faltando) a página será renderizada

1 Dave
3 John
4 William

Fazendo eco de expressões PHP #

Qualquer expressão PHP dentro de chaves duplas {{ $variable }} será echo depois de ser executada através da e função auxiliar. (Assim, caracteres especiais html (<, >, ", ', &) são substituídos com segurança pelas entidades html correspondentes.) (A expressão PHP deve ser avaliada como string, caso contrário, uma exceção será lançada.)

Ecoando uma variável #

{{ $variable }}

Ecoando um elemento em uma matriz #

{{ $array["key"] }}

Ecoando uma propriedade de objeto #

{{ $object->property }}

Ecoando o resultado de uma chamada de função #

{{ strtolower($variable) }}

Verificando a existência #

Normalmente, em PHP, para verificar se uma variável está definida e imprimi-la, você faria

  • Antes do PHP 7

  • Após o PHP 7 (usando o “operador de coalescência nulo”)

O operador blade or torna isso mais fácil:

{{ $variable or 'Default' }}

Ecos brutos #

Como mencionado, a sintaxe regular de chaves duplas {{ }}, é filtrada através da função htmlspecialchars do PHP, por segurança (prevenindo injeção maliciosa de HTML na visualização). Se você quiser ignorar esse comportamento, por exemplo, se estiver tentando gerar um bloco de conteúdo HTML resultante de uma expressão PHP, use a seguinte sintaxe:

{!! $myHtmlString !!}

Observe que é considerado uma prática recomendada usar a sintaxe padrão {{ }} para escapar de seus dados, a menos que seja absolutamente necessário. Além disso, ao reproduzir conteúdo não confiável (ou seja, conteúdo fornecido por usuários do seu site), você deve evitar usar o {!! !!} sintaxe.

Incluindo visualizações parciais #

Com o Blade, você também pode incluir visualizações parciais (chamadas de ‘parciais’) diretamente em uma página assim:

@include('includes.info', ['title' => 'Information Station'])

O código acima incluirá a visualização em ‘views/includes/info.blade.php’. Ele também passará uma variável $title com o valor ‘Information Station’.

Em geral, uma página incluída terá acesso a qualquer variável à qual a página de chamada tenha acesso. Por exemplo, se tivermos:

{{$user}} // Outputs 'abc123'
@include('includes.info')

E ‘includes/info.blade.php’ tem o seguinte:

<p>{{$user}} is the current user.</p>

Então a página irá renderizar:

abc123
abc123 is the current user.

Incluir cada

Às vezes, você vai querer combinar uma instrução include com uma instrução foreach e acessar as variáveis ​​de dentro do loop foreach no include. Neste caso, use a diretiva @each do Blade:

@each('includes.job', $jobs, 'job')

O primeiro parâmetro é a página a ser incluída. O segundo parâmetro é o array sobre o qual iterar. O terceiro parâmetro é a variável atribuída aos elementos do array. A afirmação acima é equivalente a:

@foreach($jobs as $job)
    @include('includes.job', ['job' => $job])
@endforeach

Você também pode passar um quarto argumento opcional para a diretiva @each para especificar a visualização a ser exibida quando o array estiver vazio.

@each('includes.job', $jobs, 'job', 'includes.jobsEmpty')

Compartilhamento de dados para todas as visualizações #

Executar código PHP arbitrário #

Embora possa não ser apropriado fazer tal coisa em uma visão se você pretende separar as preocupações estritamente, a diretiva php Blade permite uma maneira de executar o código PHP, por exemplo, para definir uma variável:

@php($varName = 'Enter content ')

(igual a:)

@php
    $varName = 'Enter content ';
@endphp

mais tarde:

{{ $varName }}
