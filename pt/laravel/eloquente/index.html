O Eloquent é um ORM (Object Relational Model) incluído no Laravel. Ele implementa o padrão de registro ativo e é usado para interagir com bancos de dados relacionais.

Nomeação da tabela

A convenção é usar “snake_case” no plural para nomes de tabelas e “StudlyCase” no singular para nomes de modelos. Por exemplo:

  • Uma tabela cats teria um modelo Cat
  • Uma tabela jungle_cats teria um modelo JungleCat
  • Uma tabela users teria um modelo User
  • Uma tabela people teria um modelo Person

O Eloquent tentará automaticamente vincular seu modelo a uma tabela que tenha o plural do nome do modelo, conforme indicado acima.

Você pode, no entanto, especificar um nome de tabela para substituir a convenção padrão.

class User extends Model
{ 
    protected $table = 'customers';
}

Introdução #

Eloquent é o ORM embutido no framework Laravel. Ele permite que você interaja com suas tabelas de banco de dados de maneira orientada a objetos, usando o padrão ActiveRecord.

Uma única classe de modelo geralmente mapeia para uma única tabela de banco de dados e também relacionamentos de diferentes tipos (um para um, [um para muitos][4], [muitos para muitos][5 ], polimórfico) pode ser definido entre diferentes classes de modelo.

A seção [Fazendo um modelo][6] descreve a criação e definição de classes de modelo.

Antes de começar a usar os modelos do Eloquent, certifique-se de que pelo menos uma conexão de banco de dados foi configurada em seu arquivo de configuração config/database.php.

Para entender o uso do construtor de consultas eloquentes durante o desenvolvimento, você pode usar o comando php artisan ide-helper:generate. Aqui está o [link][7].

Excluindo #

Você pode excluir dados depois de gravá-los no banco de dados. Você pode excluir uma instância de modelo se tiver recuperado uma ou especificar condições para quais registros excluir.

Para excluir uma instância de modelo, recupere-a e chame o método delete():

$user = User::find(1);
$user->delete();

Alternativamente, você pode especificar uma chave primária (ou uma matriz de chaves primárias) dos registros que deseja excluir através do método destroy():

User::destroy(1);
User::destroy([1, 2, 3]);

Você também pode combinar a consulta com a exclusão:

User::where('age', '<', 21)->delete();

Isso excluirá todos os usuários que correspondem à condição.

Nota: Ao executar uma instrução de exclusão em massa via Eloquent, os eventos de modelo deleting e deleted não serão acionados para os modelos excluídos. Isso ocorre porque os modelos nunca são realmente recuperados ao executar a instrução delete.

Exclusão suave #

Algumas vezes, você não deseja excluir permanentemente um registro, mas o mantém disponível para fins de auditoria ou relatório. Para isso, o Eloquent oferece a funcionalidade soft delete.

Para adicionar a funcionalidade de soft deletes ao seu modelo, você precisa importar o atributo SoftDeletes e adicioná-lo à sua classe de modelo Eloquent:

namespace Illuminate\Database\Eloquent\Model;
namespace Illuminate\Database\Eloquent\SoftDeletes;

class User extends Model
{
     use SoftDeletes;
}

Ao excluir um modelo, ele definirá um carimbo de data/hora em uma coluna de carimbo de data/hora deleted_at na tabela para seu modelo, portanto, certifique-se de criar a coluna deleted_at em sua tabela primeiro. Ou na migração você deve chamar o método softDeletes() em seu blueprint para adicionar o timestamp deleted_at. Exemplo:

Schema::table('users', function ($table) {
    $table->softDeletes();
});

Quaisquer consultas omitirão registros excluídos de forma reversível. Você pode forçar a exibição deles, se desejar, usando o escopo withTrashed():

User::withTrashed()->get();

Se você deseja permitir que os usuários restaurem um registro após a exclusão reversível (ou seja, em uma área do tipo lixeira), então você pode usar o método restore():

$user = User::find(1);
$user->delete();
$user->restore();

Para excluir um registro à força, use o método forceDelete() que realmente removerá o registro do banco de dados:

$user = User::find(1);
$user->forceDelete();

Alterar chave primária e carimbos de data/hora #

Por padrão, os modelos do Eloquent esperam que a chave primária seja chamada de 'id'. Se esse não for o seu caso, você pode alterar o nome da sua chave primária especificando a propriedade $primaryKey.

class Citizen extends Model
{
    protected $primaryKey = 'socialSecurityNo';

    // ...
}

Agora, qualquer método Eloquent que use sua chave primária (por exemplo, find ou findOrFail) usará esse novo nome.

Além disso, o Eloquent espera que a chave primária seja um inteiro com incremento automático. Se sua chave primária não for um inteiro de incremento automático (por exemplo, um GUID), você precisa informar ao Eloquent atualizando a propriedade $incrementing para false:

class Citizen extends Model
{
    protected $primaryKey = 'socialSecurityNo';

    public $incrementing = false;

    // ...
}

Por padrão, o Eloquent espera que as colunas created_at e updated_at existam em suas tabelas. Se você não deseja que essas colunas sejam gerenciadas automaticamente pelo Eloquent, defina a propriedade $timestamps em seu modelo para false:

class Citizen extends Model
{
    public $timestamps = false;

    // ...
}

Se você precisar personalizar os nomes das colunas usadas para armazenar os timestamps, você pode definir as constantes CREATED_AT e UPDATED_AT em seu modelo:

class Citizen extends Model
{
    const CREATED_AT = 'date_of_creation';
    const UPDATED_AT = 'date_of_last_update';

    // ...
}

Persistindo #

Além de ler dados com o Eloquent, você também pode usá-lo para inserir ou atualizar dados com o método save(). Se você criou uma nova instância de modelo, o registro será inserido; caso contrário, se você tiver recuperado um modelo do banco de dados e definido novos valores, ele será atualizado.

Neste exemplo, criamos um novo registro User:

$user = new User();
$user->first_name = 'John';
$user->last_name = 'Doe';
$user->email = 'john.doe@example.com';
$user->password = bcrypt('my_password');
$user->save();

Você também pode usar o método create para preencher campos usando uma matriz de dados:

User::create([
    'first_name'=> 'John',
    'last_name' => 'Doe',
    'email'     => 'john.doe@example.com', 
    'password'  => bcrypt('changeme'),
]);

Ao usar o método create seus atributos devem ser declarados no array preenchível dentro do seu modelo:

class User extends Model
{
    protected $fillable = [
         'first_name',
         'last_name',
         'email',
         'password',
    ];
}

Alternativamente, se você quiser tornar todos os atributos atribuíveis em massa, você pode definir a propriedade $guarded como um array vazio:

class User extends Model
{
    /**
    * The attributes that aren't mass assignable.
    *
    * @var array
    */
    protected $guarded = [];
}

Mas você também pode criar um registro sem alterar o atributo preenchível em seu modelo usando o método forceCreate em vez do método create

User::forceCreate([
    'first_name'=> 'John',
    'last_name' => 'Doe',
    'email'     => 'john.doe@example.com', 
    'password'  => bcrypt('changeme'),
]);

Veja a seguir um exemplo de atualização de um modelo User existente carregando-o primeiro (usando find), modificando-o e salvando-o:

$user = User::find(1);
$user->password = bcrypt('my_new_password');
$user->save();

Para realizar o mesmo feito com uma única chamada de função, você pode usar o método update:

$user->update([
    'password' => bcrypt('my_new_password'),
]);

Os métodos create e update tornam o trabalho com grandes conjuntos de dados muito mais simples do que ter que definir cada par de chave/valor individualmente, conforme mostrado nos exemplos a seguir:

Observe o uso de only e except ao coletar dados de solicitação. É importante que você especifique as chaves exatas que deseja permitir/não permitir que sejam atualizadas, caso contrário, é possível que um invasor envie campos adicionais com sua solicitação e cause atualizações não intencionais.

// Updating a user from specific request data
$data = Request::only(['first_name', 'email']);
$user->find(1);
$user->update($data);

// Create a user from specific request data
$data = Request::except(['_token', 'profile_picture', 'profile_name']);
$user->create($data);

Lança 404 se a entidade não for encontrada #

Se você deseja lançar automaticamente uma exceção ao procurar um registro que não seja encontrado em um modal, você pode usar

Vehicle::findOrFail(1);

ou

Vehicle::where('make', 'ford')->firstOrFail();

Se um registro com a chave primária de 1 não for encontrado, uma ModelNotFoundException será lançada. O que é essencialmente o mesmo que escrever (view source):

$vehicle = Vehicle::find($id);

if (!$vehicle) {
    abort(404);
}

Modelos de clonagem #

Você pode precisar clonar uma linha, talvez alterar alguns atributos, mas precisa de uma maneira eficiente de manter as coisas DRY. O Laravel fornece uma espécie de método ‘oculto’ para permitir que você faça essa funcionalidade. Embora seja completamente não documentado, você precisa pesquisar na API para encontrá-lo.

Usando $model->replicate() você pode facilmente clonar um registro

$robot = Robot::find(1);
$cloneRobot = $robot->replicate();
// You can add custom attributes here, for example he may want to evolve with an extra arm!
$cloneRobot->arms += 1;
$cloneRobot->save();

O acima encontraria um robô que possui um ID de 1 e o clonaria.
