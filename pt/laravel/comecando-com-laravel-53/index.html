Esta seção fornece uma visão geral do que é laravel-5.3 e por que um desenvolvedor pode querer usá-lo.

Ele também deve mencionar qualquer assunto grande dentro do laravel-5.3, e linkar para os tópicos relacionados. Como a Documentação para laravel-5.3 é nova, pode ser necessário criar versões iniciais desses tópicos relacionados.

Exemplo de Hello World (Basic) e usando uma view #

O exemplo básico Abra o arquivo routes/web.php e cole o seguinte código no arquivo:

Route::get('helloworld', function () {
    return '<h1>Hello World</h1>';
});

aqui ‘helloworld’ funcionará como o nome da página que você deseja acessar,

e se você não deseja criar um arquivo blade e ainda deseja acessar a página diretamente então você pode usar o roteamento laravel dessa maneira

agora digite localhost/helloworld na barra de endereços do navegador e você pode acessar a página que exibe o Hello World.

A próxima etapa.
Então você aprendeu a criar um Hello World muito simples! página retornando uma sentença hello world. Mas podemos torná-lo um pouco mais agradável!

Etapa 1.
Vamos começar novamente em nosso arquivo routes/web.php agora, em vez de usar o código acima, usaremos o seguinte código:

Route::get('helloworld', function() {
    return view('helloworld');
});

O valor de retorno desta vez não é apenas um simples texto helloworld, mas uma visão. Uma view no Laravel é simplesmente um novo arquivo. Este arquivo “helloworld” contém o HTML e talvez mais tarde até algum PHP do texto Helloworld.

Etapa 2.
Agora que ajustamos nossa rota para chamar uma view, vamos fazer a view. Laravel trabalha com arquivos blade.php em views. Então, neste caso, nossa rota é chamada helloworld. Então nossa visão será chamada helloworld.blade.php

Estaremos criando o novo arquivo no diretório resources/views e o chamaremos de helloworld.blade.php

Agora vamos abrir este novo arquivo e editá-lo criando nossa frase Hello World. Podemos adicionar várias maneiras diferentes de obter nossa frase, como no exemplo abaixo.

<html>
    <body>
        <h1> Hello World! </h1>

        <?php
            echo "Hello PHP World!";
        ?>

    </body>
</html>

agora vá para o seu navegador e digite sua rota novamente como no exemplo básico: localhost/helloworld você verá sua nova visão criada com todo o conteúdo!

Servidor de desenvolvimento local #

Se você tem o PHP instalado localmente e gostaria de usar o servidor de desenvolvimento embutido do PHP para servir sua aplicação, você pode usar o comando serve Artisan. Este comando iniciará um servidor de desenvolvimento em http://localhost:8000:

php artisan serve

Obviamente, opções de desenvolvimento local mais robustas estão disponíveis por meio de Homestead e Valet.

Também é possível usar uma porta personalizada, algo como 8080. Você pode fazer isso com a opção --port.

php artisan serve --port=8080

Se você tiver um domínio local em seu arquivo hosts, poderá definir o nome do host. Isso pode ser feito pela opção --host.

php artisan serve --host=example.dev

Você também pode executar em um host e porta personalizados, isso pode ser feito pelo seguinte comando.

php artisan serve --host=example.dev --port=8080

Requisitos do servidor #

O framework Laravel possui alguns requisitos de sistema. Obviamente, todos esses requisitos são atendidos pela máquina virtual Laravel Homestead, portanto, é altamente recomendável que você use o Homestead como seu ambiente de desenvolvimento Laravel local.

No entanto, se você não estiver usando o Homestead, precisará verificar se o servidor atende aos seguintes requisitos:

  • PHP >= 5.6.4
  • Extensão PHP OpenSSL
  • Extensão PHP PDO
  • Extensão PHP Mbstring
  • Extensão PHP Tokenizer
  • Extensão XML PHP

Instalando o Laravel #

Requisitos: Você precisa do PHP >= 5.6.4 e do Composer instalados em sua máquina. Você pode verificar a versão de ambos usando o comando: Para PHP:

php -v

Saída assim:

PHP 7.0.9 (cli) (built: Aug 26 2016 06:17:04) ( NTS )
Copyright (c) 1997-2016 The PHP Group
Zend Engine v3.0.0, Copyright (c) 1998-2016 Zend Technologies

Para Compositor Você pode executar o comando no seu terminal/CMD:

composer --version

Saída assim:

composer version 1.2.1 2016-09-12 11:27:19

O Laravel utiliza o Composer para gerenciar suas dependências. Portanto, antes de usar o Laravel, certifique-se de ter o Composer instalado em sua máquina.

Instalador do Laravel #

Primeiro, baixe o instalador do Laravel usando o Composer:

composer global require "laravel/installer"

Certifique-se de colocar o diretório $HOME/.composer/vendor/bin (ou o diretório equivalente para o seu sistema operacional) em seu $PATH para que o executável laravel possa ser localizado pelo seu sistema.

Uma vez instalado, o comando laravel new criará uma nova instalação do Laravel no diretório que você especificar. Por exemplo, laravel new blog criará um diretório chamado blog contendo uma nova instalação do Laravel com todas as dependências do Laravel já instaladas:

laravel new blog

Via Composer Create-Project #

Alternativamente, você também pode instalar o Laravel emitindo o comando create-project do Composer em seu terminal:

composer create-project --prefer-dist laravel/laravel blog

Configurar #

Depois de concluir a instalação do Laravel, você precisará definir permissões para as pastas de armazenamento e Bootstrap.

Observação: definir permissões é um dos processos mais importantes para completo durante a instalação do Laravel.

Servidor de Desenvolvimento Local

Se você tem o PHP instalado localmente e gostaria de usar o servidor de desenvolvimento embutido do PHP para servir sua aplicação, você pode usar o comando serve Artisan. Este comando iniciará um servidor de desenvolvimento em http://localhost:8000:

php artisan serve

Abra o URL de solicitação do seu navegador http://localhost:8000

Exemplo Hello World (Básico) #

Abra o arquivo de rotas. Cole o seguinte código em:

Route::get('helloworld', function () {
    return '<h1>Hello World</h1>';
});

depois de ir para a rota http://localhost/helloworld ele exibe Hello World.

O arquivo de rotas está localizado em /routes/web.php

Configuração do servidor Web para URLs bonitas #

Se você instalou o Laravel via Composer ou o instalador do Laravel, abaixo da configuração você precisará.

Configuração para Apache O Laravel inclui um arquivo public/.htaccess que é usado para fornecer URLs sem o front controller index.php no caminho. Antes de servir Laravel com Apache, certifique-se de habilitar o módulo mod_rewrite para que o arquivo .htaccess seja aceito pelo servidor.

Se o arquivo .htaccess que acompanha o Laravel não funcionar com sua instalação do Apache, tente esta alternativa:

Options +FollowSymLinks
RewriteEngine On

RewriteCond %{REQUEST_FILENAME} !-d
RewriteCond %{REQUEST_FILENAME} !-f
RewriteRule ^ index.php [L]

Configuração para Nginx Se você estiver usando o Nginx, a seguinte diretiva na configuração do seu site direcionará todas as solicitações para o front controller index.php:

location / {
    try_files $uri $uri/ /index.php?$query_string;
}

Claro, ao usar Homestead ou Valet, URLs bonitas serão configuradas automaticamente.
