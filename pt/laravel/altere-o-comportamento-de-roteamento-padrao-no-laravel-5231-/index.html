Altere o comportamento de roteamento padrão no Laravel 5.2.31 +

  • Sintaxe
  • Parâmetros
  • Adicionando api-routes com outro middleware e mantendo o middleware web padrão

Nesta página

  • Sintaxe
  • Parâmetros
  • Adicionando api-routes com outro middleware e mantendo o middleware web padrão

Sintaxe #

  • public function map(Router $router) // Define as rotas para a aplicação.
  • função protegida mapWebRoutes(Router $router) // Define as rotas “web” para a aplicação.

Parâmetros #

Parâmetro Cabeçalho
Roteador $ roteador \Illuminate\Routing\Router $router

Middleware significa que cada chamada para uma rota passará pelo middleware antes de realmente atingir seu código específico de rota. No Laravel, o middleware web é usado para garantir a manipulação da sessão ou a verificação do token csrf, por exemplo.

Existem outros middlewares como auth ou api por padrão. Você também pode criar facilmente seu próprio middleware.

Adicionando api-routes com outro middleware e mantendo o middleware web padrão #

Desde a versão 5.2.31 do Laravel, o middleware da web é aplicado por padrão no RouteServiceProvider (https://github.com/laravel/laravel/commit/5c30c98db96459b4cc878d085490e4677b0b67ed)

Em app/Providers/RouteServiceProvider.php você encontrará as seguintes funções que aplicam o middleware em cada rota dentro de seu app/Http/routes.php

public function map(Router $router)
{
    $this->mapWebRoutes($router);
}

// ...

protected function mapWebRoutes(Router $router)
{
    $router->group([
        'namespace' => $this->namespace, 'middleware' => 'web',
    ], function ($router) {
        require app_path('Http/routes.php');
    });
}

Como você pode ver, a web de middleware é aplicada. Você pode mudar isso aqui. No entanto, você também pode adicionar facilmente outra entrada para poder colocar suas rotas de API, por exemplo, em outro arquivo (por exemplo, routes-api.php)

public function map(Router $router)
{
    $this->mapWebRoutes($router);
    $this->mapApiRoutes($router);
}

protected function mapWebRoutes(Router $router)
{
    $router->group([
        'namespace' => $this->namespace, 'middleware' => 'web',
    ], function ($router) {
        require app_path('Http/routes.php');
    });
}

protected function mapApiRoutes(Router $router) 
{
    $router->group([
        'namespace' => $this->namespace, 'middleware' => 'api',
    ], function ($router) {
        require app_path('Http/routes-api.php');
    });
}

Com isso, você pode facilmente separar suas rotas de API de suas rotas de aplicativo sem o wrapper de grupo confuso em seu routes.php
