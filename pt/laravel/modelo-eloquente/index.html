Fazendo um modelo #

Criação de modelo #

As classes de modelo devem estender Illuminate\Database\Eloquent\Model. A localização padrão para modelos é o diretório /app.

Uma classe de modelo pode ser facilmente gerada pelo comando Artisan:

php artisan make:model [ModelName]

Isso criará um novo arquivo PHP em app/ por padrão, que é chamado [ModelName].php, e conterá todo o clichê do seu novo modelo, que inclui a classe, o namespace e os usos necessários para um configurar.

Se você deseja criar um arquivo de migração junto com seu modelo, use o seguinte comando, onde -m também gerará o arquivo de migração:

php artisan make:model [ModelName] -m

Além de criar o modelo, isso cria uma migração de banco de dados que é conectada ao modelo. O arquivo PHP de migração de banco de dados está localizado por padrão em database/migrations/. Isso não inclui - por padrão - nada além das colunas id e created_at/updated_at, então você precisará editar o arquivo para fornecer colunas adicionais.

Observe que você terá que executar a migração (depois de configurar o arquivo de migração) para que o modelo comece a funcionar usando php artisan migrate da raiz do projeto

Além disso, se você deseja adicionar uma migração posteriormente, após fazer o modelo, pode fazê-lo executando:

php artisan make:migration [migration name]

Digamos, por exemplo, que você queira criar um modelo para seus gatos, você teria duas opções, criar com ou sem migração. Você escolheria criar sem migração se já tivesse uma tabela de gatos ou não quisesse criar uma no momento.

Para este exemplo, queremos criar uma migração porque ainda não temos uma tabela, portanto, executaríamos o seguinte comando.

php artisan make:model Cat -m

Este comando criará dois arquivos:

  1. Na pasta App: app/Cat.php
  2. Na pasta do banco de dados: database/migrations/timestamp_creat_cats_table.php

O arquivo em que estamos interessados ​​é o último, pois é esse arquivo que podemos decidir como queremos que a tabela se pareça e inclua. Para qualquer migração predefinida, recebemos uma coluna de id de incremento automático e colunas de timestamps.

O exemplo abaixo de um extrato do arquivo de migração inclui as colunas predefinidas acima, bem como a adição de um nome do gato, idade e cor:

public function up()
    {
        Schema::create('cats', function (Blueprint $table) {

            $table->increments('id');    //Predefined ID
            $table->string('name');      //Name
            $table->integer('age');      //Age
            $table->string('colour');    //Colour
            $table->timestamps();        //Predefined Timestamps

        });
    }

Então, como você pode ver, é relativamente fácil criar o modelo e migração para uma tabela. Em seguida, para executar a migração e criá-la em seu banco de dados, você executaria o seguinte comando:

php artisan migrate

Que migrará quaisquer migrações pendentes para seu banco de dados.

Localização do arquivo de modelo #

Os modelos podem ser armazenados em qualquer lugar graças ao PSR4.

Por padrão, os modelos são criados no diretório app com o namespace de App. Para aplicativos mais complexos, geralmente é recomendável armazenar modelos em suas próprias pastas em uma estrutura que faça sentido para a arquitetura de seus aplicativos.

Por exemplo, se você tivesse um aplicativo que usasse uma série de frutas como modelos, você poderia criar uma pasta chamada app/Fruits e dentro dessa pasta você criaria Banana.php (mantendo o [StudlyCase](https:// en.wikipedia.org/wiki/Studly_caps) convenção de nomenclatura), você pode criar a classe Banana no namespace App\Fruits:

namespace App\Fruits;

use Illuminate\Database\Eloquent\Model;

class Banana extends Model {
    // Implementation of "Banana" omitted
}

Configuração do modelo #

O Eloquent segue uma abordagem de “convenção sobre configuração”. Ao estender a classe base Model, todos os modelos herdam as propriedades listadas abaixo. A menos que substituídos, os seguintes valores padrão se aplicam:

Propriedade Descrição Padrão
conexão $protegida Nome da conexão do banco de dados Conexão de banco de dados padrão
tabela $ protegida Nome da tabela Por padrão, o nome da classe é convertido para snake_case e ​​pluralizado. Por exemplo, SpecialPerson se torna special_people
protected $primaryKey Tabela PK id
public $incrementing Indica se os IDs são incrementados automaticamente verdadeiro
$ timestamps públicos Indica se o modelo deve receber carimbo de data/hora verdadeiro
const CREATED_AT Nome da coluna de carimbo de data/hora de criação criado_em
const UPDATED_AT Nome da coluna de carimbo de data/hora de modificação updated_at
protegido $datas Atributos que devem ser alterados para DateTime, além dos atributos timestamps []
protected $dateFormat Formato em que os atributos de data serão persistidos Padrão para o dialeto SQL atual.
protegido $com Relacionamentos com eagerload com modelo []
protegido $oculto Atributos omitidos na serialização do modelo []
protegido $visible Atributos permitidos na serialização do modelo []
protected $appends Acessadores de atributo adicionados à serialização do modelo []
protegido $preenchível Atributos que podem ser atribuídos em massa []
protegido $guardado Atributos que estão na lista negra da atribuição em massa [*] (Todos os atributos)
$toques protegidos As relações que devem ser abordadas salvar []
protegido $perPage O número de modelos a serem retornados para paginação. 15
Propriedade Descrição Padrão
$casts protegidos Atributos que devem ser convertidos em tipos nativos []

Atualizar um modelo existente #

$user = User::find(1);
$user->name = 'abc';
$user->save();

Você também pode atualizar vários atributos de uma vez usando update, que não requer o uso de save depois:

$user = User::find(1);
$user->update(['name' => 'abc', 'location' => 'xyz']);

Você também pode atualizar um(s) modelo(s) sem consultá-lo antes:

User::where('id', '>', 2)->update(['location' => 'xyz']);

Se você não quer acionar uma mudança no timestamp updated_at no modelo, então você pode passar a opção touch:

$user = User::find(1);
$user->update(['name' => 'abc', 'location' => 'xyz'], ['touch' => false]);
