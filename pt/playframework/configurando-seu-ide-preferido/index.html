IDEIA do IntelliJ #

Pré-requisitos #

  1. Intellij IDEA instalado (edição Community ou Ultimate)
  2. Plugin Scala instalado no IntelliJ
  3. Um projeto padrão do Play, criado por exemplo com o Activator (activator new [nameoftheproject] play-scala).

Abrindo o projeto #

  1. Abra o IntelliJ IDEA
  2. Vá para o menu File > Open ... > clique em toda a pasta [nameoftheproject] > OK
  3. Um pop-up é aberto com algumas opções. Os valores padrão são bons o suficiente na maioria dos casos e, se você não gostar deles, poderá alterá-los em outro lugar mais tarde. Clique em OK
  4. O Intellij IDEA pensará um pouco, então proporá outro popup para selecionar quais módulos selecionar no projeto. Deve haver dois módulos root e root-build selecionados por padrão. Não altere nada e clique em ‘OK’.
  5. O IntelliJ abrirá o projeto. Você pode começar a visualizar os arquivos enquanto o IntelliJ continua pensando um pouco como você deve ver na barra de status na parte inferior, então ele deve finalmente estar totalmente pronto.

Executando os aplicativos do Intellij #

A partir daí, algumas pessoas usam o IDE apenas para visualizar/editar o projeto, enquanto usam a linha de comando sbt para compilar/executar/iniciar testes. Outros preferem lançá-los de dentro do Intellij. É necessário se você quiser usar o modo de depuração. Passos :

  1. Menu Executar > Editar configurações...
  2. No pop-up, clique em + no canto superior esquerdo > Escolha Play 2 App na lista
  3. Nomeie a configuração, por exemplo, [nameofyourproject]. Deixe as opções padrão e clique em ‘OK’.
  4. A partir do menu Executar ou dos botões na interface do usuário, agora você pode Executar ou Depurar usando esta configuração. Run apenas iniciará o aplicativo, como se você tivesse executado sbt run a partir da linha de comando. Debug fará a mesma coisa, mas permitirá que você coloque pontos de interrupção no código para interromper a execução e analisar o que está acontecendo.

Opção de importação automática #

Esta é uma opção global para o projeto, que está disponível no momento da criação e posteriormente pode ser alterada no menu Intellij IDEA > Preferences > Build, Execution, Deployment > Build tools > SBT > Configurações no nível do projeto > Usar importação automática.

Esta opção não tem nada a ver com as instruções import no código Scala. Ele determina o que o Intellij IDEA deve fazer quando você edita o arquivo build.sbt. Se a importação automática estiver ativada, o Intellij IDEA analisará o novo arquivo de compilação imediatamente e atualizará a configuração do projeto automaticamente. Isso se torna irritante rapidamente, pois essa operação é cara e tende a desacelerar o Intellij quando você ainda está trabalhando no arquivo de compilação. Quando a importação automática está desativada, você deve indicar manualmente ao Intellij que editou o build.sbt e gostaria que a configuração do projeto fosse atualizada. Na maioria dos casos, um pop-up temporário aparecerá perguntando se você gostaria de fazê-lo. Caso contrário, vá para o painel SBT na interface do usuário e clique no sinal de setas circulares azuis para forçar a atualização.

Eclipse como Play IDE - Java, Play 2.4, 2.5 #

Introdução #

O Play tem vários plugins para diferentes IDE-s. O plugin eclipse permite transformar uma aplicação Play em um projeto eclipse funcional com o comando activator eclipse. O plugin Eclipse pode ser definido por projeto ou globalmente por usuário sbt. Depende do trabalho em equipe, qual abordagem deve ser usada. Se toda a equipe estiver usando o Eclipse IDE, o plug-in pode ser definido em um nível de projeto. Você precisa baixar a versão do Eclipse compatível com Scala e Java 8: luna ou mars - de http://scala-ide.org/download/sdk.html.

Configurando o Eclipse IDE por projeto #

Para importar o aplicativo Play para o Eclipse:

  1. Adicione o plugin Eclipse em project/plugins.sbt:

    //Support Play in Eclipse addSbtPlugin(“com.typesafe.sbteclipse” % “sbteclipse-plugin” % “4.0.0”)

  2. Adicione em build.sbt um sinalizador que force a compilação a acontecer quando o comando eclipse for executado:

    EclipseKeys.preTasks := Seq(compile in Compile)

  3. Certifique-se de que um caminho de repositório do usuário no arquivo {user root}.sbt\repositories tenha o formato adequado. Os valores adequados para as propriedades activator-launcher-local e activator-local devem ter pelo menos três barras como no exemplo:

    activator-local: file:////${activator.local.repository-C:/Play-2.5.3/activator-dist-1.3.10//repository}, [organization]/[module]/(scala_[scalaVersion]/)(sbt_[sbtVersion]/)[revision]/[type]s/artifact.[ext] activator-launcher-local: file:////${activator.local.repository-${activator.home-${user.home}/.activator}/repository}, [organization]/[module]/(scala_[scalaVersion]/)(sbt_[sbtVersion]/)[revision]/[type]s/artifact.[ext]

  4. Compile o aplicativo:

    activator compile

  5. Prepare um projeto Eclipse para o novo aplicativo com:

    activator eclipse Agora o projeto está pronto para ser importado para o Eclipse via Existing Projects into Workspace.

Como anexar a fonte Play ao eclipse #

  1. Adicione ao build.sbt:

    EclipseKeys.withSource := true

  2. Compile o projeto

Configurando o Eclipse IDE globalmente #

Adicione a configuração de usuário sbt:

  1. Crie no diretório raiz do usuário uma pasta .sbt\0.13\plugins e um arquivo plugins.sbt. Por exemplo, para o usuário do Windows asch:

    c:\asch.sbt\0.13\plugins\plugins.sbt

  2. Adicione o plugin Eclipse em plugins.sbt:

    //Support Play in Eclipse addSbtPlugin(“com.typesafe.sbteclipse” % “sbteclipse-plugin” % “4.0.0”)

  3. Crie no diretório do usuário .sbt um arquivo sbteclipse.sbt. Por exemplo, para o usuário do Windows asch:

    c:\asch.sbt\0.13\sbteclipse.sbt

  4. Coloque em sbteclipse.sbt um sinalizador que força a compilação a acontecer quando o comando activator eclipse é executado:

    import com.typesafe.sbteclipse.plugin.EclipsePlugin.EclipseKeys EclipseKeys.preTasks := Seq(compile in Compile)

  5. Adicione opcionalmente outras configurações de EclipseKeys.

Depurando do eclipse #

Para depurar, inicie o aplicativo com a porta padrão 9999:

activator -jvm-debug run

ou com a porta diferente:

activator -jvm-debug [port] run

Em eclipse:

  1. Clique com o botão direito do mouse no projeto e selecione Depurar como, Configurações de depuração.
  2. Na caixa de diálogo Debug Configurations, clique com o botão direito do mouse em Remote Java Application e selecione New.
  3. Altere a Porta para relevante (9999 se a porta de depuração padrão foi usada) e clique em Aplicar.

A partir de agora você pode clicar em Depurar para se conectar ao aplicativo em execução. Interromper a sessão de depuração não interromperá o servidor.

##Eclipse IDE Pré-requisitos #

  1. Java8 (1.8.0_91)
  2. Eclipse neon (JavaScript e Web Developer)
  3. Estrutura do Jogo 2.5.4

Instalando Scala no Eclipse #

  1. Inicie o Eclipse
  2. Abra Ajuda > Eclipse Marketplace
  3. Digite Scala em Localizar
  4. Instale o Scala IDE

Configurar sbtelipse #

  1. Abra o projeto de jogo .\project\ plugins.sbt
  2. Adicione o seguinte comando em plugins.sbt para converter o projeto Eclipse

addSbtPlugin(“com.typesafe.sbteclipse” % “sbteclipse-plugin” % “4.0.0”)

  1. Abra o comando e vá para reproduzir o projeto, por exemplo. cd C:\play\play-scala. Digite a seguir na linha de comando

ativador eclipse

Importando projeto #

  1. Vá para o menu Arquivo > Importar no Eclipse
  2. Selecione “Projetos existentes no espaço de trabalho”
  3. Selecione o diretório raiz

Agora seu projeto está pronto para visualização e edição no Eclipse IDE.
