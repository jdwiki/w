Entendendo os arquivos de mídia autônomos #

O conteúdo de exemplo usado aqui é Tears of Steel, da Blender Foundation. Especificamente, usaremos o download intitulado “HD 720p (~365MB, mov, 2.0)”. Este é um único arquivo que termina com a extensão “mov” e será reproduzido em praticamente qualquer reprodutor de mídia moderno.

Observe que a página de download oferece legendas como downloads de arquivos SRT separados. Neste conteúdo de amostra, não há legendas entregues juntas no mesmo arquivo. Portanto, deixamos a análise de legendas fora do escopo deste exemplo.

Uma maneira fácil de analisar vários arquivos de mídia é usar a ferramenta/biblioteca MediaInfo. Embora a funcionalidade de análise apresentada aqui use a GUI para simplificar, todos os recursos também estão disponíveis por meio da API MediaInfo.

Ao abrir este arquivo na GUI do MediaInfo e alternar para a visualização em árvore, você verá três seções: Geral, Vídeo e Áudio. O primeiro contém informações básicas sobre o arquivo, enquanto os dois restantes descrevem uma faixa de mídia encontrada nesse arquivo. Vamos examinar as informações mais relevantes em cada seção da saída.

Em geral #

Os primeiros parâmetros de interesse são Format e Format profile. O primeiro indica que o formato de embalagem é do conjunto de padrões MPEG-4. O MPEG-4 define o ISO Base Media File Format e o MP4 packet format. Além disso, a Apple criou suas próprias especificações que derivam delas, denominadas no MediaInfo como o perfil “QuickTime”.

Nota: Tenha cuidado para não confundir MP4 e MPEG-4 - o primeiro refere-se a um formato de embalagem específico no pacote MPEG-4 de padrões internacionais, que também inclui codecs de vídeo e áudio. Isso pode causar confusão, portanto evite usar o termo MPEG-4 ao se referir a qualquer coisa que não seja o conjunto completo de padrões.

Todos os formatos de empacotamento baseados no ISO Base Media File Format, definido na família de padrões MPEG-4, são muito semelhantes e muitas vezes podem ser processados ​​pelas mesmas ferramentas, com suas diferenças sendo em grande parte uma questão de extensões personalizadas de fornecedores que muitas vezes podem ser ignorado. Assim, podemos esperar que o vídeo de amostra aqui seja altamente compatível com todos os players de vídeo modernos.

Vídeo #

O detalhe mais crucial sobre a faixa de vídeo é o codec usado para transformar dados de cores brutos em um formato compactado. O nome do codec é fornecido pelo parâmetro Format.

O AVC também é conhecido como H.264 e é o codec de vídeo que hoje é o mais difundido, suportado em praticamente todos os dispositivos e plataformas de software modernos. Uma faixa de vídeo codificada usando AVC certamente será reproduzida em praticamente qualquer player.

Os codecs geralmente têm vários perfis que permitem que a funcionalidade do codec seja dividida em camadas, permitindo a evolução da tecnologia de maneira controlada. O parâmetro Format profile indica que este vídeo usa o perfil principal. Esse perfil é relativamente incomum, pois quase todos os dispositivos modernos suportam o perfil High, que oferece maior eficiência de compactação.

A qualidade da faixa de vídeo é muitas vezes de suma importância. Aqui vemos os fatores críticos expressos pelos parâmetros Bit rate, Width e Height. Os dois últimos sugerem que se trata de uma faixa de vídeo de 720p, que é considerada uma qualidade HD inferior. A imagem é realmente mais curta verticalmente do que o quadro padrão de 720p de 1280x720 pixels.

A taxa de bits mede a quantidade de dados que a forma compactada do fluxo de vídeo ocupa, em média, para cada segundo de reprodução. Este é um parâmetro crucial para a otimização, pois a quantidade de dados entregues é uma importante fonte de custo em soluções de vídeo em grande escala.

Os pontos de dados acima sobre a qualidade do vídeo são simplesmente fatos que obtemos da análise - qualquer julgamento sobre a adequação desses parâmetros é um tópico que precisaria de muito mais análise e é abordado por tópicos separados nesta categoria de documentação, assim como muitos outros pontos delicados de trabalhando com faixas de vídeo.

Áudio #

Mais uma vez, conhecer o codec usado para codificar os dados de áudio é de importância crítica. Isso é expresso pelos parâmetros Format e Format profile. “MPEG Audio Layer 3” é mais comumente conhecido como MP3 e é um formato de áudio universalmente suportado que pode ser reproduzido em qualquer lugar.

Assim como no vídeo, os parâmetros de qualidade de áudio são os segundos pontos de dados mais importantes, expressos principalmente pelo parâmetro Taxa de bits.

Resumo da análise #

O conteúdo é empacotado usando um formato de empacotamento muito popular, construído sobre o conjunto de padrões MPEG-4. Ele é codificado usando codecs de vídeo e áudio universalmente adotados. A partir disso, fica claro que o vídeo deve ser facilmente acessível a todos os espectadores - compatibilidade e disponibilidade foram fundamentais para seus autores.

O uso de MP3 mostra a idade do conteúdo de exemplo, pois não é mais considerado à altura dos concorrentes modernos - em vez disso, o AAC (Advanced Audio Coding) é o ganha-pão no campo dos codecs de áudio.

O mesmo pode ser dito sobre o uso do perfil H.264 Main. É muito raro que qualquer perfil H.264 além do High seja usado, já que quase todos os decodificadores o suportam, permitindo que todos aproveitem a eficiência aprimorada proporcionada pelos recursos High Profile.

As taxas de bits usadas são um pouco mais altas do que o esperado para o ambiente atual. Isso pode ser explicado pelo desejo dos autores de alta qualidade ou simplesmente pelas limitações dos codificadores que estavam disponíveis quando o conteúdo foi criado.

Mais #

Outras ferramentas úteis para análise de arquivos de mídia são FFprobe, que faz parte do pacote de software FFmpeg, e as ferramentas Bento4 para trabalhando com arquivos MP4. Ambos também estão disponíveis em forma de biblioteca. Eles são capazes de análises de nível mais baixo do que MediaInfo, em situações em que você precisa examinar elementos individuais que compõem arquivos de mídia.
