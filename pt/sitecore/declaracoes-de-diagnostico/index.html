Verificações de argumentos #

ArgumentoCondição #

Este método verifica se o argumento especificado é verdadeiro. Ele também recebe o nome do argumento registrado se a condição falhar.

Assert.ArgumentCondition(pageIndex >= 0, "pageIndex", "Value must be greater than or equal to zero.");

ArgumentNotNull #

Este método garante que o argumento passado não seja nulo. Existem duas assinaturas para este método, a primeira recebe um objeto e um nome de parâmetro e faz uma simples verificação de nulo.

Assert.ArgumentNotNull(item, "item");

ArgumentNotNullOrEmpty #

Isso é semelhante ao método ArgumentNotNull, mas também verificará se o objeto está vazio. Existem três variantes deste método. A primeira variante recebe um ID do Sitecore e um nome de argumento e verifica se o ID é nulo.

var nullId = new new ID("{00000000-0000-0000-0000-000000000000}");

// Both of these calls will result in an exception
Assert.ArgumentNotNullOrEmpty((ID)null, "null");
Assert.ArgumentNotNullOrEmpty(nullId, nameof(nullId));

O segundo método adiciona uma verificação para ver se a string fornecida é nula ou vazia.

// Both of these calls will result in an exception
Assert.ArgumentNotNullOrEmpty((string)null, "null");
Assert.ArgumentNotNullOrEmpty("", nameof(emptyString));

Certifique-se de que o item esteja no modo de edição #

Verifica se o Item passado está no modo de edição. Caso contrário, ele lança um EditingNotAllowedException.

Assert.IsEditing(Sitecore.Context.Item);

Certifique-se de que dois valores sejam iguais #

Compara dois valores para igualdade. Ele pode comparar apenas strings, números inteiros e valores booleanos.

Assert.AreEqual(documentElement.LocalName, "xamlControls", "Xaml files must have a root node named 'xamlControls'.");

Certifique-se de que um valor seja verdadeiro ou falso #

Para afirmar que um valor é verdadeiro ou falso,:

Assert.IsFalse(Settings.DoBadThings, "Bad things should not happen, disable DoBadThings.");
Assert.IsTrue(magicNumber =< 42, "The magic number is greater than 42!");

Você também pode passar parâmetros de formatação para a mensagem de exceção

Assert.IsFalse(myValue > 5, "The value should not be greater than 5, it's currently {0}", myValue);

ResultNotNull #

ResultNotNull() verifica se o objeto passado não é nulo. Se o objeto e a mensagem não forem nulos, ele simplesmente retornará o objeto que foi passado, caso contrário, lançará InvalidOperationException.

return Assert.ResultNotNull(this.Database.GetItem(this.ItemRootId), string.Concat("Root item not found. ID: ", this.ItemRootId));

Objeto obrigatório #

Isso verifica se o objeto fornecido é nulo e, em seguida, lança RequiredObjectIsNullException se for.

Assert.Required(parameter, "parameter is required.");

Cheques nulos/vazios #

Não é nulo #

Este é um método muito simples e popular para verificar se um item não é nulo. Ele simplesmente verifica o objeto que é passado para ver se é nulo.

Assert.IsNotNull(database, type, "Name: {0}", item);

Não é Nulo ou Vazio #

Isso é o mesmo que IsNotNull acima, mas funciona em valores de string em vez de objetos.

Assert.IsNotNullOrEmpty(propertyName, "user");

É nulo #

Isso é simplesmente um inverso do método IsNotNull(). Este método afirma que o objeto é nulo.

Assert.IsNull(this.StandardValues, "A Standard Values item has already been created for this template ");

Declarações de segurança #

##CanRunApplication ## Para verificar se o usuário tem permissão para executar o aplicativo fornecido. Caso contrário, AccessDeniedException é lançado.

Assert.CanRunApplication("WebEdit");

Tem acesso #

HasAccess verificará se o parâmetro fornecido é verdadeiro, caso contrário, um AccessDeniedException será lançado.

Assert.HasAccess(Context.User.IsAdministrator, "Only administrators can create new domains");
