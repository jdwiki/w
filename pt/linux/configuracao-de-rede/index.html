Este documento abrange os conceitos básicos de rede TCP/IP, administração de rede e configuração do sistema. O Linux pode suportar vários dispositivos de rede. Os nomes dos dispositivos são numerados e começam em zero e contam para cima. Por exemplo, um computador com duas NICs terá dois dispositivos rotulados eth0 e eth1.

Detalhes da interface #

Ifconfig

Liste todas as interfaces disponíveis na máquina

$ ifconfig -a

ip - mostra/manipula roteamento, dispositivos, roteamento de política e túneis

Sintaxe: $ ip { link | ... | percurso | macsec } (por favor veja man ip para lista completa de objetos)

Exemplos

Listar interfaces de rede

$ ip link show

Renomeie a interface eth0 para wan

$ ip link set dev eth0 name wan

Traga a interface eth0 para cima (ou para baixo)

$ ip link set dev eth0 up

Listar endereços para interfaces

$ ip addr show

Adicione (ou del) ip e mask (255.255.255.0)

$ ip addr add 1.2.3.4/24 brd + dev eth0

Adicionando IP a uma interface #

Um endereço IP para uma interface pode ser obtido via DHCP ou atribuição estática

DHCP Se você estiver conectado a uma rede com um servidor DHCP em execução, o comando dhclient pode obter um endereço IP para sua interface

$ dhclient <interface>

ou alternativamente, você pode fazer uma alteração no arquivo /etc/network/interfaces para que a interface seja aberta na inicialização e obtenha o IP DHCP

auto eth0
iface eth0 inet dhcp

Configuração estática (alteração permanente) usando o arquivo /etc/network/interfaces

Se você deseja configurar estaticamente as configurações da interface (mudança permanente), você pode fazê-lo no arquivo /etc/network/interfaces.

Exemplo:

auto eth0 # Bring up the interface on boot
iface eth0 inet static 
    address 10.10.70.10
    netmask 255.255.0.0
    gateway 10.10.1.1
    dns-nameservers 10.10.1.20
    dns-nameservers 10.10.1.30

Essas alterações persistem mesmo após a reinicialização do sistema.

Configuração estática (alteração temporária) usando o utilitário ifconfig

Um endereço IP estático pode ser adicionado a uma interface usando o utilitário ifconfig da seguinte forma

$ ifconfig <interface> <ip-address>/<mask> up

Exemplo:

$ ifconfig eth0 10.10.50.100/16 up

Resolução de DNS local #

Arquivo: /etc/hosts contém uma lista de hosts que devem ser resolvidos localmente (não pelo DNS)

Exemplo de conteúdo do arquivo:

127.0.0.1         your-node-name.your-domain.com  localhost.localdomain  localhost 
XXX.XXX.XXX.XXX   node-name

O formato de arquivo para o arquivo hosts é especificado por RFC 952

Configurar servidores DNS para resolução de nomes de domínio #

Arquivo: /etc/resolv.conf contém uma lista de servidores DNS para resolução de nomes de domínio

Exemplo de conteúdo do arquivo:

nameserver 8.8.8.8 # IP address of the primary name server
nameserver 8.8.4.4 # IP address of the secondary name server

No caso do servidor DNS interno, você pode validar se este servidor resolve os nomes DNS corretamente usando o comando dig:

$ dig google.com @your.dns.server.com +short

Ver e manipular rotas #

Manipule a tabela de roteamento IP usando route #

Exibir tabela de roteamento

$ route # Displays list or routes and also resolves host names
$ route -n # Displays list of routes without resolving host names for faster results

Adicionar/Excluir rota

Opção Descrição
add ou del Adicionar ou excluir uma rota
-host x.x.x.x Adicionar rota a um único host identificado pelo endereço IP
-net x.x.x.x Adicionar rota a uma rede identificada pelo endereço de rede
gw x.x.x.x Especifique o gateway de rede
máscara de rede x.x.x.x Especifique a máscara de rede da rede
padrão Adicionar uma rota padrão

Exemplos:

  • adicionar rota a um host $ route add -host x.x.x.x eth1
  • adicionar rota a uma rede $ route add -net 2.2.2.0 netmask 255.255.255.0 eth0
  • Alternativamente, você também pode usar o formato cidr para adicionar uma rota à rede route add -net 2.2.2.0/24 eth0
  • adicionar gateway padrão $ route add default gw 2.2.2.1 eth0
  • deletar uma rota $ route del -net 2.2.2.0/24

Manipular a tabela de roteamento IP usando ip #

Exibir tabela de roteamento

$ ip route show # List routing table

Adicionar/Excluir rota

Opção Descrição
add ou del ou change ou append ou replace Alterar uma rota
show ou flush o comando exibe o conteúdo das tabelas de roteamento ou remove-o
restaurar restaurar informações da tabela de roteamento de stdin
obter este comando obtém uma única rota para um destino e imprime seu conteúdo exatamente como o kernel o vê

Exemplos:

  • Defina o gateway padrão para 1.2.3.254 $ ip route add default via 1.2.3.254
  • Adiciona uma rota padrão (para todos os endereços) via gateway local 192.168.1.1 que pode ser acessada no dispositivo eth0 $ ip route add default via 192.168.1.1 dev eth0

Configure um nome de host para algum outro sistema em sua rede #

Você pode configurar seu sistema Linux (ou macOS) para vincular um identificador <hostname> ao endereço IP de algum outro sistema em sua rede. Você pode configurá-lo:

  • Todo o sistema. Você deve modificar o arquivo /etc/hosts. Você só precisa adicionar a esse arquivo uma nova linha contendo:

    1. the remote system’s IP address <ip_rem>,
    2. one or more blank spaces, and
    3. the identifier <hostname>.

  • Para um único usuário. Você deve modificar o arquivo ~/.hosts — você teria que criá-lo. Não é tão simples quanto para todo o sistema. Aqui você pode ver uma explicação.

Por exemplo, você pode adicionar esta linha usando a ferramenta Unix cat. Suponha que você queira fazer um ping para um PC em sua rede local cujo endereço IP seja 192.168.1.44 e você queira se referir a esse endereço IP apenas por remote_pc. Então você deve escrever em seu shell:

$ sudo cat 192.168.1.44 remote_pc

Então você pode fazer esse ping apenas por:

$ ping remote_pc
