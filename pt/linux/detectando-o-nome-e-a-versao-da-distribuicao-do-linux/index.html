Detecte em qual distribuição baseada em Debian você está trabalhando #

Apenas execute lsb_release -a.

No Debian:

$ lsb_release -a
No LSB modules are available.
Distributor ID: Debian
Description:    Debian GNU/Linux testing (stretch)
Release:        testing
Codename:       stretch

No Ubuntu:

$ lsb_release -a
No LSB modules are available.
Distributor ID: Ubuntu
Description:    Ubuntu 14.04.4 LTS
Release:        14.04
Codename:       trusty

Caso você não tenha o lsb_release instalado, você pode tentar adivinhar, por exemplo, existe um arquivo /etc/issue que geralmente contém o nome da distribuição. Por exemplo, no Ubuntu:

$ cat /etc/issue
Ubuntu 12.04.5 LTS \n \l

Não use o arquivo /etc/debian_version porque seu conteúdo não corresponde ao nome da distribuição!

Observe que isso também funcionará em distribuições não pertencentes à família Debian como Fedora, RHEL ou openSUSE — mas o lsb_release pode não estar instalado.

Detecte qual distribuição baseada em systemd você está usando #

Esse método funcionará em versões modernas do Arch, CentOS, CoreOS, Debian, Fedora, Mageia, openSUSE, Red Hat Enterprise Linux, SUSE Linux Enterprise Server, Ubuntu e outros. Essa ampla aplicabilidade o torna ideal como primeira abordagem, com recurso a outros métodos se você também precisar identificar sistemas mais antigos.

Olhe para /etc/os-release. Especificamente, observe as variáveis ​​NAME, VERSION, ID, VERSION_ID e PRETTY_NAME.

No Fedora, este arquivo pode se parecer com:

    NAME=Fedora
    VERSION="24 (Workstation Edition)"
    ID=fedora
    VERSION_ID=24
    PRETTY_NAME="Fedora 24 (Workstation Edition)"
    ANSI_COLOR="0;34"
    CPE_NAME="cpe:/o:fedoraproject:fedora:24"
    HOME_URL="https://fedoraproject.org/"
    BUG_REPORT_URL="https://bugzilla.redhat.com/"
    REDHAT_BUGZILLA_PRODUCT="Fedora"
    REDHAT_BUGZILLA_PRODUCT_VERSION=24
    REDHAT_SUPPORT_PRODUCT="Fedora"
    REDHAT_SUPPORT_PRODUCT_VERSION=24
    PRIVACY_POLICY_URL=https://fedoraproject.org/wiki/Legal:PrivacyPolicy
    VARIANT="Workstation Edition"
    VARIANT_ID=workstation

No CentOS, este arquivo pode ficar assim:

    NAME="CentOS Linux"
    VERSION="7 (Core)"
    ID="centos"
    ID_LIKE="rhel fedora"
    VERSION_ID="7"
    PRETTY_NAME="CentOS Linux 7 (Core)"
    ANSI_COLOR="0;31"
    CPE_NAME="cpe:/o:centos:centos:7"
    HOME_URL="https://www.centos.org/"
    BUG_REPORT_URL="https://bugs.centos.org/"
    
    CENTOS_MANTISBT_PROJECT="CentOS-7"
    CENTOS_MANTISBT_PROJECT_VERSION="7"
    REDHAT_SUPPORT_PRODUCT="centos"
    REDHAT_SUPPORT_PRODUCT_VERSION="7"

Este arquivo está documentado no site do freedesktop; em princípio, não é específico do systemd — mas existirá em todas as distribuições baseadas no systemd.

A partir do shell bash, pode-se obter o arquivo /etc/os-release e ​​usar as várias variáveis ​​diretamente, assim:

$ ( source /etc/os-release && echo "$PRETTY_NAME" )
Fedora 24 (Workstation Edition)

Uname - Imprime informações sobre o sistema atual #

Uname é o nome abreviado de unix name. Basta digitar uname no console para obter informações sobre seu sistema operacional.

uname [OPTION]

Se nenhuma OPTION for especificada, uname assume a opção -s.

-a ou --all - Imprime todas as informações, omitindo -p e -i se a informação for desconhecida.

Exemplo: #

> uname -a

SunOS hope 5.7 Generic_106541-08 sun4m sparc SUNW,SPARCstation-10

Todas as opções:

-s, --kernel-name Print the kernel name. -n, --nodename Print the network node hostname. -r, --kernel-release Print the kernel release. -v, --kernel-version Print the kernel version. -m, --machine Print the machine hardware name. -p, --processor Print the processor type, or "unknown". -i, --hardware-platform Print the hardware platform, or "unknown". -o, --operating-system Print the operating system. --help Display a help message, and exit. --version Display version information, and exit.

encontre o nome e o número da versão do seu sistema operacional linux (ambos debian e rpm) #

A maioria das distribuições linux armazena suas informações de versão no arquivo /etc/lsb-release (debian) ou /etc/redhat-release (baseado em RPM). O uso do comando genérico abaixo deve fazer com que você ultrapasse a maioria dos derivados Debian e RPM como Linux Mint e Cent-Os.

Exemplo na máquina Ubuntu:

cat /etc/*release #

DISTRIB_ID=Gratuito DISTRIB_RELEASE=14.04 DISTRIB_CODENAME=confiável DISTRIB_DESCRIPTION=“Ubuntu 14.04 LTS”

usando GNU coreutils #

Portanto, o GNU coreutils deve estar disponível em todos os sistemas baseados em linux (por favor, corrija-me se estiver errado aqui).
