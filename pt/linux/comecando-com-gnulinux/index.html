Atalhos úteis #

Usando o Terminal #

Os exemplos neste documento pressupõem que você esteja usando um shell compatível com POSIX (como bash, sh, zsh, ksh).

Grandes porções da funcionalidade GNU/Linux são alcançadas usando o terminal. A maioria das distribuições do Linux inclui emuladores de terminal que permitem aos usuários interagir com um shell de seu ambiente de desktop. Um shell é um interpretador de linha de comando que executa comandos inseridos pelo usuário. Bash (Bourne Again SHell) é um shell padrão comum entre muitas distribuições Linux e é o shell padrão para macOS.

Esses atalhos funcionarão se você estiver usando o Bash com as combinações de teclas emacs (definidas por padrão):

Abrir terminal #

  • Ctrl + Alt + T ou Super + T

Movimento do cursor #

  • Ctrl + A Vá para o início da linha que você está digitando.
  • Ctrl + E Vá para o final da linha que você está digitando no momento.
  • Ctrl + XX Move entre o início da linha e a posição atual do cursor.
  • Alt + F Move o cursor uma palavra para frente na linha atual.
  • Alt + B Move o cursor para trás uma palavra na linha atual.
  • Ctrl + F Move o cursor um caractere para frente na linha atual.
  • Ctrl + B Move o cursor um caractere para trás na linha atual.

Manipulação de texto #

  • Ctrl + U Corta a linha da posição atual até o início da linha, adicionando-a à área de transferência. Se você estiver no final da linha, corte a linha inteira.
  • Ctrl + K Corta a linha da posição atual até o final da linha, adicionando-a à área de transferência. Se você estiver no início da linha, corte a linha inteira.
  • Ctrl + W Apaga a palavra antes do cursor, adicionando-a à área de transferência.
  • Ctrl + Y Cole a última coisa da área de transferência que você cortou recentemente (desfaça a última exclusão na posição atual do cursor).
  • Alt + T Troca as duas últimas palavras antes do cursor.
  • Alt + L Faz letras minúsculas do cursor até o final da palavra.
  • Alt + U Letra maiúscula do cursor até o final da palavra.
  • Alt + C Capitalize até o final da palavra começando no cursor (palavra inteira se o cursor estiver no início da palavra).
  • Alt + D Excluir até o final da palavra começando no cursor (palavra inteira se o cursor estiver no início da palavra).
  • Alt + . Imprime a última palavra escrita no comando anterior.
  • Ctrl + T Troca os dois últimos caracteres antes do cursor.

Acesso ao histórico #

  • Ctrl + R Permite pesquisar os comandos usados ​​anteriormente.
  • Ctrl + G Sai do modo de pesquisa de histórico sem executar um comando.
  • Ctrl + J Permite copiar o comando correspondente atual para a linha de comando sem executá-lo, permitindo que você faça modificações antes de executar o comando.
  • Alt + R Reverta quaisquer alterações em um comando que você extraiu do seu histórico, se você o editou.
  • Ctrl + P Mostra o último comando executado, ou seja, volta pelo histórico de comandos (semelhante à seta para cima).
  • Ctrl + N Mostra o próximo comando executado, ou seja, avança pelo histórico de comandos (semelhante à seta para baixo).

Controle de terminais #

  • Ctrl + L Limpa a tela, semelhante ao comando clear.
  • Ctrl + S Para todas as saídas para a tela. Isso é útil ao executar comandos com muitas saídas longas. Mas isso não interrompe o comando em execução.
  • Ctrl + Q Retome a saída para a tela depois de pará-la com Ctrl+S.
  • Ctrl + C Finaliza o processo atualmente em execução e retorna o prompt.
  • Ctrl + D Efetue logout da sessão shell atual, semelhante ao comando exit ou logout. Em alguns comandos, atua como sinal de Fim de Arquivo para indicar que o fim de um arquivo foi alcançado.
  • Ctrl + Z Suspende (pausa) o processo de primeiro plano atualmente em execução, que retorna o prompt do shell. Você pode então usar o comando bg permitindo que esse processo seja executado em segundo plano. Para trazer novamente esse processo para o primeiro plano, use o comando fg. Para visualizar todos os processos em segundo plano, use o comando jobs.
  • Tab Auto-completar arquivos e nomes de diretórios.
  • TabTab Mostra todas as possibilidades, quando os caracteres digitados não correspondem exclusivamente a um nome de arquivo ou diretório.

Caracteres especiais #

  • Ctrl + H O mesmo que Backspace.
  • Ctrl + J O mesmo que Return (historicamente alimentação de linha).
  • Ctrl + M O mesmo que Return (historicamente Carriage Return).
  • Ctrl + I O mesmo que Tab.
  • Ctrl + G Caractere de sino.
  • Ctrl + @ Caractere nulo.
  • Esc [Deadkey] (https://en.wikipedia.org/wiki/Dead_key) é equivalente ao modificador Alt.

Fechar Terminal #

  • Ctrl + Shift + W Para fechar a guia do terminal.
  • Ctrl + Shift + Q Para fechar o terminal inteiro.

Como alternativa, você pode alternar para as combinações de teclas vi no bash usando set -o vi. Use set -o emacs para voltar às combinações de teclas emacs.

Comandos de gerenciamento de arquivos #

O Linux usa algumas convenções para diretórios presentes e pais. Isso pode ser um pouco confuso para iniciantes.

Sempre que estiver em um terminal no Linux, você estará no que é chamado de diretório de trabalho atual. Muitas vezes, seu prompt de comando exibirá o diretório de trabalho completo ou apenas a última parte desse diretório. Seu prompt pode se parecer com um dos seguintes:

user@host ~/somedir $
user@host somedir $
user@host /home/user/somedir $

que diz que seu diretório de trabalho atual é /home/user/somedir.

No Linux .. representa o diretório pai e <código> . representa o diretório atual.

Portanto, se o diretório atual for /home/user/somedir, então cd ../somedir não mudará o diretório de trabalho.

A tabela abaixo lista alguns dos comandos de gerenciamento de arquivos mais usados

Navegação no diretório #

Comando Utilitário
pwd Obtenha o caminho completo do diretório de trabalho atual.
cd - Navegue até o último diretório em que você estava trabalhando.
cd ~ ou apenas cd Navegue até o diretório inicial do usuário atual.
cd.. Vá para o diretório pai do diretório atual (cuidado com o espaço entre cd e ..)

Listando arquivos dentro de um diretório #

Comando Utilitário
ls -l Liste os arquivos e diretórios no diretório atual em formato longo (tabela) (Recomenda-se usar -l com ls para melhor legibilidade).
ls -ld dir-name Liste informações sobre o diretório dir-name em vez de seu conteúdo.
ls -a Liste todos os arquivos, incluindo os ocultos (nomes de arquivos que começam com um . são arquivos ocultos no Linux).
ls -F Acrescenta um símbolo no final de um nome de arquivo para indicar seu tipo (* significa executável, / significa diretório, @ significa link simbólico, = significa socket, | significa pipe nomeado, > significa porta).
ls -lt Liste os arquivos classificados pelo horário da última modificação com os arquivos modificados mais recentemente exibidos no topo (lembre-se que a opção -l fornece o formato longo que tem melhor legibilidade).
ls -lh Liste os tamanhos de arquivo em formato legível por humanos.
ls -lR Mostra todos os subdiretórios recursivamente.
árvore Gerará uma representação em árvore do sistema de arquivos a partir do diretório atual.

Arquivo/diretório criar, copiar e remover #

Comando Utilitário
cp -p destino de origem Copiará o arquivo de origem para destino. -p significa preservação. Ele preserva os atributos originais do arquivo ao copiar como proprietário do arquivo, carimbo de data/hora, grupo, permissões etc.
cp -R source_dir destination_dir Copiará o diretório de origem para o destino especificado recursivamente.
mv arquivo1 arquivo2 No Linux não há comando rename como tal. Portanto, mv move/renomeia o arquivo1 para arquivo2.
rm -i nome do arquivo Pergunta antes de cada remoção de arquivo para confirmação. SE VOCÊ É UM NOVO USUÁRIO DA LINHA DE COMANDO LINUX, VOCÊ DEVE SEMPRE USAR rm -i. Você pode especificar vários arquivos.
rm -R dir-name Irá remover o diretório dir-name recursivamente.
rm -rf dir-name Irá remover o diretório dir recursivamente, ignorando arquivos inexistentes e nunca solicitará nada. CUIDADO AO USAR ESTE COMANDO! Você pode especificar vários diretórios.
rmdir dir-name Irá remover o diretório dir-name, se estiver vazio. Este comando só pode remover diretórios vazios.
mkdir dir-name Cria um diretório dir-name.
mkdir -p dir-name/dir-name Cria uma hierarquia de diretórios. Crie diretórios pai conforme necessário, se eles não existirem. Você pode especificar vários diretórios.
touch filename Crie um arquivo filename, se não existir, caso contrário, altere a data e hora do arquivo para a hora atual.

Permissões e grupos de arquivos/diretórios #

Comando Utilitário
chmod <especificação> nome do arquivo Altere as permissões do arquivo. Especificações = u user, g group, o other, + add permission, - remove, r read, w write,x execute.
chmod -R <especificação> dir-name Altere as permissões de um diretório recursivamente. Para alterar a permissão de um diretório e tudo dentro desse diretório, use este comando.
chmod go=+r meuarquivo Adicione permissão de leitura para o proprietário e o grupo.
chmod a +rwx myfile Permitir que todos os usuários leiam, escrevam ou executem myfile.
chmod go -r myfile Remova a permissão de leitura do grupo e de outros.
chown owner1 filename Alterar a propriedade de um arquivo para o usuário owner1.
chgrp grp_owner filename Mude a propriedade do grupo primário do arquivo filename para o grupo grp_owner.
chgrp -R grp_owner dir-name Muda a propriedade do grupo primário do diretório dir-name para o grupo grp_owner recursivamente. Para alterar a propriedade do grupo de um diretório e tudo dentro desse diretório, use este comando.

Olá Mundo #

Digite o seguinte código em seu terminal e pressione Enter:

echo "Hello World"

Isso produzirá a seguinte saída:

Hello World

Procurando arquivos por padrões em nome/conteúdo #

Uma tarefa comum de alguém que usa a linha de comando do Linux (shell) é procurar arquivos/diretórios com um determinado nome ou contendo determinado texto. Existem 2 comandos com os quais você deve se familiarizar para conseguir isso:

Localizar arquivos por nome #

find /var/www -name '*.css'

Isso imprimirá o caminho/nome do arquivo completo para todos os arquivos em /var/www que terminam em .css. Saída de exemplo:

/var/www/html/text-cursor.css
/var/www/html/style.css

Para mais informações:

man find

Localizar arquivos contendo texto #

grep font /var/www/html/style.css

Isso imprimirá todas as linhas contendo o padrão font no arquivo especificado. Saída de exemplo:

font-weight: bold;
font-family: monospace;

Outro exemplo:

grep font /var/www/html/

Isso não funciona como você esperava. Você obtém:

grep: /var/www/html/: Is a directory

Você precisa grep recursivamente para fazê-lo funcionar, usando a opção -R:

grep -R font /var/www/html/

Oi legal! Confira a saída deste:

/var/www/html/admin/index.php:  echo '<font color=red><b>Error: no dice</b></font><br/>';
/var/www/html/admin/index.php:  echo '<font color=red><b>Error: try again</b></font><br/>';
/var/www/html/style.css:  font-weight: bold;
/var/www/html/style.css:  font-family: monospace;

Observe que quando grep está combinando vários arquivos, ele prefixa as linhas correspondentes com os nomes dos arquivos. Você pode usar a opção -h para se livrar disso, se quiser.

Para mais informações:

man grep

Manipulação de arquivos #

Arquivos e diretórios (outro nome para pastas) estão no coração do Linux, portanto, poder criá-los, visualizá-los, movê-los e excluí-los da linha de comando é muito importante e bastante poderoso. Esses comandos de manipulação de arquivos permitem que você execute as mesmas tarefas que um explorador gráfico de arquivos executaria.

Crie um arquivo de texto vazio chamado myFile:

touch myFile

Renomeie myFile para myFirstFile:

mv myFile myFirstFile

Visualize o conteúdo de um arquivo:

cat myFirstFile

Visualize o conteúdo de um arquivo com pager (uma tela de cada vez):

less myFirstFile

Veja as primeiras linhas de um arquivo:

head myFirstFile

Veja as últimas linhas de um arquivo:

tail myFirstFile

Editar um arquivo:

vi myFirstFile

Veja quais arquivos estão em seu diretório de trabalho atual:

ls

Crie um diretório vazio chamado myFirstDirectory:

mkdir myFirstDirectory

Criar diretório de vários caminhos: (cria dois diretórios, src e myFirstDirectory)

mkdir -p src/myFirstDirectory

Mova o arquivo para o diretório:

mv myFirstFile myFirstDirectory/

Você também pode renomear o arquivo:

user@linux-computer:~$ mv myFirstFile secondFileName

Altere o diretório de trabalho atual para myFirstDirectory:

cd myFirstDirectory

Excluir um arquivo:

rm myFirstFile

Mova para o diretório pai (que é representado como ..):

cd ..

Excluir um diretório vazio:

rmdir myFirstDirectory

Exclua um diretório não vazio (ou seja, contém arquivos e/ou outros diretórios):

rm -rf myFirstDirectory

Observe que, ao excluir diretórios, exclua ./ e não /, que limpará todo o seu sistema de arquivos.

Detalhes do arquivo/diretório #

O comando ls tem várias opções que podem ser usadas juntas para mostrar mais informações.

Detalhes/Direitos

A opção l mostra as permissões do arquivo, tamanho e data da última modificação. Então, se o diretório raiz continha um diretório chamado test e um arquivo someFile o comando:

user@linux-computer:~$ ls -l

Produziria algo como

-rw-r--r-- 1 user users   70 Jul 22 13:36 someFile.txt
drwxrwxrwx 2 user users 4096 Jul 21 07:18 test

As permissões estão no formato drwxrwxrwx. O primeiro caractere representa o tipo de arquivo d se for um diretório - caso contrário. Os próximos três rwx são as permissões que o usuário tem sobre o arquivo, os próximos três são as permissões que o grupo tem sobre o arquivo e os três últimos são as permissões que todos os outros têm sobre o arquivo.

O r de rwx significa se um arquivo pode ser lido, o w representa se o arquivo pode ser modificado e o x significa se o arquivo pode ser executado. Se alguma permissão não for concedida, um - será substituído por r, w ou x.

Portanto, acima, o user pode ler e modificar someFile.txt, mas o grupo tem apenas direitos somente leitura.

Para alterar os direitos, você pode usar o comando chmod ### fileName se tiver direitos sudo. r é representado por um valor de 4, w é representado por 2 e x é representado por um 1. Então, se você quiser modificar o conteúdo para o diretório test

Owner rwx = 4+2+1 = 7
Group r-x = 4+0+1 = 5
Other r-x = 4+0+1 = 5

Então todo o comando é

chmod 755 test

Agora, fazer um ls -l mostraria algo como

drwxr-xr-x 2 user users 4096 Jul 21 07:20 test

Tamanho legível

Usada em conjunto com a opção l, a opção h mostra tamanhos de arquivo que podem ser lidos por humanos. Corrida

user@linux-computer:~$ ls -lh

Produziria:

total 4166
-rw-r--r-- 1 user users   70 Jul 22 13:36 someFile.txt
drwxrwxrwx 2 user users 4.0K Jul 21 07:18 test

Escondido

Para visualizar arquivos ocultos, use a opção a. Por exemplo

user@linux-computer:~$ ls -a

Pode listar

.profile
someFile.txt
test

Tamanho total do diretório

Para ver o tamanho do diretório atual use a opção s (a opção h também pode ser usada para tornar o tamanho mais legível).

user@linux-computer:~$ ls -s

Saídas

total 4166
someFile.txt      test

Visão Recursiva

Vamos dizer que o diretório test tinha um arquivo anotherFile e você queria vê-lo da pasta raiz, você poderia usar a opção R que listaria a árvore recursiva.

user@linux-computer:~$ ls -R

Saídas

.:
someFile.txt    test

./test:
anotherFile
