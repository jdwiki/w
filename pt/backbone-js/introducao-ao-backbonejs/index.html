Configuração básica #

Backbone requer Underscore e (opcionalmente) jQuery - para manipulação DOM (usando Backbone.View) e persistência RESTful.

A maneira mais rápida de começar a usar o Backbone é criar um arquivo index.html com tags de script simples no HTML <head>:

<html>
    <head>
        <script src="https://code.jquery.com/jquery-3.1.0.min.js"></script>
        <script src="https://cdnjs.cloudflare.com/ajax/libs/underscore.js/1.8.3/underscore-min.js"></script>

        <script src="https://cdnjs.cloudflare.com/ajax/libs/backbone.js/1.3.3/backbone-min.js"></script>
    </head>
    <body>
    </body>
</html>

O backbone já está disponível para uso na página.

Exemplo de uso de backbone (principalmente Backbone.Model) #

O Backbone.js é composto por quatro componentes separados: Coleções, Modelos, Roteadores e Visualizações. Cada um deles serve a propósitos diferentes:

  • Model - representa um único objeto de dados, mas adiciona funcionalidades adicionais não fornecidas por objetos JavaScript nativos, como um sistema de eventos e uma maneira mais conveniente de recuperar e enviar dados para um servidor remoto

  • Collection - representa um conjunto ou “coleção” de Modelos e fornece funcionalidades para gerenciar seus modelos.

  • View - representa uma única parte da interface do usuário; cada View envolve um elemento HTML DOM e fornece estrutura para trabalhar com esse elemento, bem como recursos de conveniência, como associação de evento simples.

  • Router - habilita um “aplicativo de página única”, permitindo que um aplicativo acione uma lógica diferente (por exemplo, mostre páginas diferentes) em resposta às alterações de URL.

Crie seus próprios sabores #

Antes de vermos como usar cada um desses componentes, vamos dar uma olhada rápida no sistema de classes do Backbone. Para criar uma nova subclasse de uma classe Backbone, basta chamar o método extend da classe original e passar as propriedades da instância e as propriedades da classe (estática) como objetos:

const MyModelClass = Backbone.Model.extend({
    instanceMethod: function() { console.log('Instance method!'); },
}, {
    staticMethod: function() { console.log('Static method!'); },
});

Assim como em qualquer outro sistema de classes, métodos de instância podem ser chamados em instâncias (objetos) da classe, enquanto métodos estáticos são chamados diretamente na própria classe (o construtor):

var myInstance = new MyModelClass();

// Call an instance method on our instance
myInstance.instanceMethod(); // logs "Instance method!"

// Call a static method on our class
MyModelClass.staticMethod(); // logs "Static method!"

Usando uma classe #

Agora, vamos ver um exemplo rápido de como você pode usar cada classe. Começaremos com um Modelo de um livro.

const Book = Backbone.Model.extend({
    idAttribute: 'isbn',
    urlRoot: '/book'
});

Vamos detalhar o que aconteceu lá. Primeiro, criamos uma subclasse Book de Model e demos a ela duas propriedades de instância.

  • idAttribute diz ao Backbone para usar o atributo “isbn” do modelo como seu ID ao realizar operações AJAX.
  • urlRoot, diz ao Backbone para procurar os dados do livro em www.example.com/book.

Agora vamos criar uma instância de um livro e obter seus dados do servidor:

var huckleberryFinn = new Book({ isbn: '0486403491' });
huckleberryFinn.fetch({
    // the Backbone way
    success: (model, response, options) => {
       console.log(model.get('name')); // logs "Huckleberry Finn"
    }
}).done(() => console.log('the jQuery promise way'));

Quando criamos um novo Book passamos um objeto para ele, e o Backbone usa este objeto como os “atributos” iniciais (os dados) do Model. Como o Backbone sabe que o idAttribute é isbn, ele sabe que a URL do nosso novo livro é /book/0486403491. Quando dissermos para fetch, o Backbone usará jQuery para fazer uma solicitação AJAX para os dados do livro. fetch retorna uma promessa (assim como $.ajax), que você pode usar para acionar ações assim que a busca for concluída.

Os atributos podem ser acessados ​​ou modificados usando os métodos get ou set:

huckleberryFinn.get('numberOfPages'); // returns 64

huckleberryFinn.set('numberOfPages', 1); // changes numberOfPages to 1

Models também tem um sistema de eventos que você pode usar para reagir quando coisas acontecem com um Model. Por exemplo, para registrar uma mensagem sempre que o numberOfPages for alterado, você pode fazer:

huckleberryFinn.on('change:numberOfPages', () => console.log('Page change!'));

Para uma introdução mais detalhada às outras classes de Backbone, veja suas páginas de documentação individuais.

Exemplo mostrando os conceitos básicos #

O exemplo a seguir é uma introdução a:

  • Compilação de modelo usando sublinhado
  • Acessando variáveis ​​em um template
  • Criando uma visualização
  • Renderizando uma vista
  • Mostrando uma vista
<html>
<head>
    <script src="https://code.jquery.com/jquery-3.1.0.min.js"></script>
    <script src="https://cdnjs.cloudflare.com/ajax/libs/underscore.js/1.8.3/underscore-min.js"></script>
    <script src="https://cdnjs.cloudflare.com/ajax/libs/backbone.js/1.3.3/backbone-min.js"></script>
</head>
<body>

    <div id="example_container"></div>

    <script type="text/template" id="example_template">
        <label><%= example_label %></label>
        <input type="text" id="example_input" />
        <input type="button" id="example_button" value="Search" />
    </script>
    <script type="text/javascript">
        var ExampleView = Backbone.View.extend({
            // Compile the template using underscore
            template: _.template($("#example_template").html()),
            events: {
                "click #example_button": "onButtonClick"
            },

            initialize: function(options) {
                this.customOption = options.customOption;
            },

            render: function() {
                // Load the compiled HTML into the Backbone "el"
                this.$el.html(this.template({
                    example_label: "My Search"
                }));

                return this; // for chaining, a Backbone's standard for render
            },

            onButtonClick: function(event) {
                // Button clicked, you can access the button that 
                // was clicked with event.currentTarget
                console.log("Searching for " + $("#example_input").val());
            }
        });
        $(function() {
            //show the view inside the div with id 'example_container'
            var exampleView = new ExampleView({
                el: $("#example_container"),
                customOption: 41,
            });
            exampleView.render();
        });
    </script>
</body>
</html>

Hello Web (configuração básica do tipo “Hello World”) #

<html>
    <head>
        <script src="https://code.jquery.com/jquery-3.1.0.min.js"></script>
        <script src="https://cdnjs.cloudflare.com/ajax/libs/underscore.js/1.8.3/underscore-min.js"></script>

        <script src="https://cdnjs.cloudflare.com/ajax/libs/backbone.js/1.3.3/backbone-min.js"></script>
        
        <script>
            $( function(){
                ( function(){
                    var View = Backbone.View.extend( {
                        "el": "body",
                        "template": _.template( "<p>Hello, Web!</p>" ),
    
                        "initialize": function(){
                            this.render();
                        },
                        "render": function(){
                            this.$el.html( this.template() );
                        }
                    } );
    
                    new View();
                })()
            } );
        </script>
    </head>
    <body>
    </body>
</html>
