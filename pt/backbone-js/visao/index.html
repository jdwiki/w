Sintaxe #

  • Criar: var View = Backbone.View.extend( { /* propriedades */ } );
  • Construir: var myView = new View( /* options */ );
  • initialize: método chamado automaticamente após a construção
  • render: método usado para atualizar this.el com novo conteúdo

Uma visualização vinculada ao HTML existente #

Assumindo este HTML na página:

<body>
    <div id="myPage">
    </div>
</body>

Uma visão pode ser vinculada a ela com:

var MyPageView = Backbone.View.extend( {
    "el": "#myPage",
    "template": _.template( "<p>This is my page.</p>" ),

    "initialize": function(){
        this.render();
    },

    "render": function(){
        this.$el.html( this.template() );
    }
} );

new MyPageView();

O HTML no navegador agora mostrará:

<body>
    <div id="myPage">
        <p>This is my page.</p>
    </div>
</body>

Função de inicialização da visão #

initialize é chamado pelo Backbone logo após a construção de uma View.

Parâmetros opcionais #

A função initialize recebe todos os argumentos passados ​​para o construtor da view. Comumente, o hash de opções que é usado para passar as opções padrão da visualização:

['model', 'collection', 'el', 'id', 'attributes', 'className', 'tagName', 'events']

Você pode adicionar quaisquer atributos personalizados ao hash de opções e/ou parâmetros personalizados.

var MyView = Backbone.View.extend({
    initialize: function(options, customParam){
        // ensure that the 'options' is a hash to avoid errors if undefined.
        options = options || {};
        this.customAttribute = options.customAttribute;
        this.customParam = customParam;
    },
});

E construindo a visão:

var view = new MyView({
    model: new Backbone.Model(),
    template: "<p>a template</p>",
    customAttribute: "our custom attribute"
}, "my custom param");

Observe que todas as opções de visualização padrão são adicionadas automaticamente ao objeto de visualização, portanto, não é necessário fazer isso na função initialize.

Renderize imediatamente o padrão #

Um padrão comum para o método initialize é chamar o método render para que qualquer View recém-construída seja imediatamente renderizada

Esse padrão deve ser usado apenas em instâncias em que a construção do objeto deve renderizá-lo imediatamente no documento HTML, vincular todos os ouvintes de eventos e executar todas as outras ações associadas à colocação de conteúdo no DOM.

var MyView = Backbone.View.extend({
    initialize: function() {
        this.render();
    },

    render: function() {
        this.$el.html("<p>I'm running!</p>");
    }
});

Deve-se notar, entretanto, que algumas Views não devem ser renderizadas imediatamente até que .render seja chamado manualmente (ou por algum outro método).

Outro padrão initialize comum é adicionar coisas ao objeto View que serão necessárias mais tarde:

var MyView = Backbone.View.extend({
    el: "body",
    template: _.template( "<p>This is <%= name %>'s page</p>" ),

    initialize: function(){
        this.name = "Bill";
        
        this.render();
    },

    render: function(){
        var viewTemplateData = {
            name: this.name
        };

        this.$el.html( this.template( viewTemplateData ) );
    }
});

O DOM agora conterá <p>Esta é a página do Bill</p> no corpo.
