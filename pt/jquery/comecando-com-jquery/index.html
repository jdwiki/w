Começando #

Crie um arquivo hello.html com o seguinte conteúdo:

<!DOCTYPE html>
<html>
<head>
    <title>Hello, World!</title>
</head>
<body>
    <div>
        <p id="hello">Some random text</p>
    </div>
    <script src="https://code.jquery.com/jquery-2.2.4.min.js"></script>
    <script>
        $(document).ready(function() {
            $('#hello').text('Hello, World!');
        });
    </script>
</body>
</html>

Demonstração ao vivo no JSBin

Abra este arquivo em um navegador da web. Como resultado, você verá uma página com o texto: Hello, World!

Explicação do código #

  1. Carrega a biblioteca jQuery do jQuery CDN:

    <script src="https://code.jquery.com/jquery-2.2.4.min.js"></script>

Isso introduz a variável global $, um alias para a função e namespace jQuery.

Esteja ciente de que um dos erros mais comuns cometidos ao incluir jQuery está falhando ao carregar a biblioteca ANTES de quaisquer outros scripts ou bibliotecas que podem depender ou fazer uso dele.

  1. Adia uma função a ser executada quando o DOM (Document Object Model) é detectado como “ready” pelo jQuery:

    // When the `document` is `ready`, execute this function `...`
$(document).ready(function() { ... });

// A commonly used shorthand version (behaves the same as the above)
$(function() { ... });

  2. Assim que o DOM estiver pronto, o jQuery executa a função de retorno de chamada mostrada acima. Dentro da nossa função, existe apenas uma chamada que faz 2 coisas principais:

  3. Obtém o elemento com o atributo id igual a hello (nosso seletor #hello). Usar um seletor como o argumento passado é o núcleo da funcionalidade e nomeação do jQuery; a biblioteca inteira evoluiu essencialmente da extensão do document.querySelectorAllMDN.

  4. Defina o [text()][text] dentro do elemento selecionado para Hello, World!.

       #    ↓ - Pass a `selector` to `$` jQuery, returns our element
   $('#hello').text('Hello, World!');
   #             ↑ - Set the Text on the element

Para mais, consulte a página jQuery - Documentação.

Evitando colisões de namespace #

Bibliotecas diferentes de jQuery também podem usar $ como um alias. Isso pode causar interferência entre essas bibliotecas e o jQuery.

Para liberar $ para uso com outras bibliotecas:

jQuery.noConflict();

Depois de chamar esta função, $ não é mais um alias para jQuery. No entanto, você ainda pode usar a própria variável jQuery para acessar as funções do jQuery:

jQuery('#hello').text('Hello, World!');

Opcionalmente, você pode atribuir uma variável diferente como um alias para jQuery:

var jqy = jQuery.noConflict();
jqy('#hello').text('Hello, World!');

Por outro lado, para evitar que outras bibliotecas interfiram no jQuery, você pode envolver seu código jQuery em uma [expressão de função imediatamente invocada (IIFE)](https://www.wikiod.com/pt/javascript/funcoes#Expressões de função invocadas imediatamente -function-expressions#t=201607152249399289216) e passe jQuery como argumento:

(function($) {
    $(document).ready(function() {
        $('#hello').text('Hello, World!');
    });
})(jQuery);

Dentro deste IIFE, $ é um alias apenas para jQuery.

Outra maneira simples de proteger o alias $ do jQuery e garantir que o DOM esteja pronto:

jQuery(function( $ ) { // DOM is ready
   // You're now free to use $ alias
   $('#hello').text('Hello, World!');
});

Para resumir,

  • jQuery.noConflict() : $ não se refere mais ao jQuery, enquanto a variável jQuery o faz.
  • var jQuery2 = jQuery.noConflict() - $ não se refere mais ao jQuery, enquanto a variável jQuery faz e também a variável jQuery2.

Agora, existe um terceiro cenário - E se quisermos que o jQuery esteja disponível somente em jQuery2? Usar,

var jQuery2 = jQuery.noConflict(true)

Isso resulta em nem $ nem jQuery referindo-se ao jQuery.

Isso é útil quando várias versões do jQuery devem ser carregadas na mesma página.

<script src='https://code.jquery.com/jquery-1.12.4.min.js'></script>
<script>
    var jQuery1 = jQuery.noConflict(true);
</script>
<script src='https://code.jquery.com/jquery-3.1.0.min.js'></script>
<script>
    // Here, jQuery1 refers to jQuery 1.12.4 while, $ and jQuery refers to jQuery 3.1.0.
</script>

https://learn.jquery.com/using-jquery-core/avoid-conflicts-other-libraries/

namespace jQuery (“jQuery” e “$”) #

jQuery é o ponto de partida para escrever qualquer código jQuery. Pode ser usado como uma função jQuery(...) ou uma variável jQuery.foo.

$ é um alias para jQuery e os dois geralmente podem ser trocados entre si (exceto onde jQuery.noConflict(); foi usado - veja Evitando colisões de namespace).

Supondo que tenhamos esse trecho de HTML -

<div id="demo_div" class="demo"></div>

Podemos querer usar jQuery para adicionar algum conteúdo de texto a esta div. Para fazer isso, podemos usar a função jQuery text(). Isso pode ser escrito usando jQuery ou $. ou seja -

jQuery("#demo_div").text("Demo Text!");

Ou -

$("#demo_div").text("Demo Text!");

Ambos resultarão no mesmo HTML final -

<div id="demo_div" class="demo">Demo Text!</div>

Como $ é mais conciso que jQuery, geralmente é o método preferido de escrever código jQuery.

jQuery usa seletores CSS e no exemplo acima foi usado um seletor de ID. Para mais informações sobre seletores em jQuery veja tipos de seletores.

1: https://www.wikiod.com/pt/jquery/comecando-com-jquery#Evitando colisões de namespace 2: https://api.jquery.com/text/#text2 3: https://www.wikiod.com/pt/jquery/seletores#Tipos de seletores

Carregando jQuery via console em uma página que não o possui. #

Às vezes é preciso trabalhar com páginas que não estão usando o jQuery enquanto a maioria dos desenvolvedores está acostumada a ter o jQuery à mão.

Em tais situações, pode-se usar o console Chrome Developer Tools (F12) para adicionar manualmente jQuery em uma página carregada executando o seguinte:

var j = document.createElement('script');
j.onload = function(){ jQuery.noConflict(); };
j.src = "https://ajax.googleapis.com/ajax/libs/jquery/1.12.4/jquery.min.js";
document.getElementsByTagName('head')[0].appendChild(j);

A versão que você deseja pode ser diferente da acima (1.12.4), você pode obter o link para um que você precisa aqui.

Incluir tag de script no cabeçalho da página HTML #

Para carregar o jQuery do CDN oficial, vá para o jQuery website. Você verá uma lista de diferentes versões e formatos disponíveis.

Agora, copie a fonte da versão do jQuery que você deseja carregar. Suponha que você queira carregar jQuery 2.X, clique na tag uncompressed ou minified que mostrará algo assim:

Copie o código completo (ou clique no ícone de cópia) e cole-o no <head> ou <body> do seu html.

A prática recomendada é carregar qualquer biblioteca JavaScript externa na tag head com o atributo async. Aqui está uma demonstração:

<!DOCTYPE html>
    <html>
      <head>
         <title>Loading jquery-2.2.4</title>
         <script src="https://code.jquery.com/jquery-2.2.4.min.js" async></script>
      </head>
      <body>
          <p>This page is loaded with jquery.</p>
      </body>
   </html>

Ao usar o atributo async, fique atento, pois as bibliotecas javascript são carregadas de forma assíncrona e executadas assim que disponíveis. Se duas bibliotecas forem incluídas onde a segunda biblioteca for dependente da primeira biblioteca, neste caso, se a segunda biblioteca for carregada e executada antes da primeira biblioteca, ela poderá gerar um erro e o aplicativo poderá quebrar.

O objeto jQuery #

Toda vez que jQuery é chamado, usando $() ou jQuery(), internamente ele está criando uma nova instância de jQuery. Este é o código-fonte que mostra a nova instância:

// Define a local copy of jQuery
jQuery = function( selector, context ) {

    // The jQuery object is actually just the init constructor 'enhanced'
    // Need init if jQuery is called (just allow error to be thrown if not included)
    return new jQuery.fn.init( selector, context );
}

Internamente, jQuery se refere ao seu protótipo como .fn, e o estilo usado aqui de instanciar internamente um objeto jQuery permite que esse protótipo seja exposto sem o uso explícito de new pelo chamador.

Além de configurar uma instância (que é como a API do jQuery, como .each, children,filter, etc. é exposta), internamente o jQuery também criará uma estrutura semelhante a um array para corresponder ao resultado do seletor (desde que algo diferente de nada, undefined, null ou similar tenha sido passado como argumento). No caso de um único item, essa estrutura tipo array conterá apenas esse item.

Uma demonstração simples seria encontrar um elemento com um id e, em seguida, acessar o objeto jQuery para retornar o elemento DOM subjacente (isso também funcionará quando vários elementos corresponderem ou estiverem presentes).

var $div = $("#myDiv");//populate the jQuery object with the result of the id selector
var div = $div[0];//access array-like structure of jQuery object to get the DOM Element

Carregando plugins jQuery com namespace #

Normalmente, ao carregar plugins, certifique-se de sempre incluir o plugin depois do jQuery.

<script src="https://code.jquery.com/jquery-3.1.1.min.js"></script>
<script src="some-plugin.min.js"></script>

Se você deve usar mais de uma versão do jQuery, certifique-se de carregar o(s) plugin(s) depois da versão necessária do jQuery seguido pelo código para definir [jQuery.noConflict(true)](http:/ /api.jquery.com/jQuery.noConflict/); em seguida, carregue a próxima versão do jQuery e seus plugins associados:

<script src="https://code.jquery.com/jquery-1.7.0.min.js"></script>
<script src="plugin-needs-1.7.min.js"></script>
<script>
// save reference to jQuery v1.7.0
var $oldjq = jQuery.noConflict(true);
</script>
<script src="https://code.jquery.com/jquery-3.1.1.min.js"></script>
<script src="newer-plugin.min.js"></script>

Agora, ao inicializar os plugins, você precisará usar a versão jQuery associada

<script>
// newer jQuery document ready
jQuery(function($){
  // "$" refers to the newer version of jQuery
  // inside of this function

  // initialize newer plugin
  $('#new').newerPlugin();
});

// older jQuery document ready
$oldjq(function($){
  // "$" refers to the older version of jQuery
  // inside of this function

  // initialize plugin needing older jQuery
  $('#old').olderPlugin();
});
</script>

É possível usar apenas uma função pronta para documento para inicializar ambos os plugins, mas para evitar confusão e problemas com qualquer código jQuery extra dentro da função pronta para documento, seria melhor manter as referências separadas.
