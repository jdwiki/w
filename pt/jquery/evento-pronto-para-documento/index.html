O que é document-ready e como devo usá-lo? #

O código jQuery geralmente é encapsulado em jQuery(function($) { ... }); para que ele seja executado somente após o DOM terminar de carregar.

<script type="text/javascript"> 
  jQuery(function($) {
    // this will set the div's text to "Hello".
    $("#myDiv").text("Hello");
  });
</script>

<div id="myDiv">Text</div>

Isso é importante porque jQuery (e JavaScript em geral) não pode selecionar um elemento DOM que não foi renderizado na página.

<script type="text/javascript">
  // no element with id="myDiv" exists at this point, so $("#myDiv") is an
  // empty selection, and this will have no effect
  $("#myDiv").text("Hello");
</script>

<div id="myDiv">Text</div>

Observe que você pode criar um alias para o namespace jQuery passando um manipulador personalizado para o método .ready(). Isso é útil para casos em que outra biblioteca JS está usando o mesmo alias $ abreviado como jQuery, o que cria um conflito. Para evitar esse conflito, você deve chamar $.noConflict(); - Isso forçando você a usar apenas o namespace jQuery padrão (em vez do alias $ curto). Ao passar um manipulador personalizado para o manipulador .ready(), você poderá escolher o nome do alias para usar jQuery.

$.noConflict();

jQuery( document ).ready(function( $ ) {
   // Here we can use '$' as jQuery alias without it conflicting with other 
   // libraries that use the same namespace 
   $('body').append('<div>Hello</div>')
});

jQuery( document ).ready(function( jq ) {
   // Here we use a custom jQuery alias 'jq' 
   jq('body').append('<div>Hello</div>')
});

Em vez de simplesmente colocar seu código jQuery na parte inferior da página, usar a função $(document).ready garante que todos os elementos HTML foram renderizados e todo o Document Object Model (DOM) está pronto para o código JavaScript ser executado.

jQuery 2.2.3 e anteriores #

Todos são equivalentes, o código dentro dos blocos será executado quando o documento estiver pronto:

$(function() {
  // code
});

$().ready(function() {
  // code
});

$(document).ready(function() {
  // code
});

Como eles são equivalentes, o primeiro é o formato recomendado, o seguinte é uma versão com a palavra-chave jQuery em vez do $ que produz os mesmos resultados:

jQuery(function() {
  // code
});

jQuery 3.0 #

Notação #

A partir do jQuery 3.0, apenas este formulário é recomendado:

jQuery(function($) {
  // Run when document is ready
  // $ (first argument) will be internal reference to jQuery
  // Never rely on $ being a reference to jQuery in the global namespace
});

Todos os outros manipuladores prontos para documentos estão obsoletos no jQuery 3.0.

Assíncrono #

A partir do jQuery 3.0, o manipulador pronto sempre será chamado de forma assíncrona. Isso significa que no código abaixo, o log ‘outside handler’ sempre será exibido primeiro, independentemente de o documento estar pronto no momento da execução.

$(function() {
  console.log("inside handler");
});
console.log("outside handler");

> manipulador externo > manipulador interno

Diferença entre $(document).ready() e $(window).load() #

$(window).load() foi obsoleto na versão 1.8 do jQuery (e completamente removido do jQuery 3.0) e como tal não deve mais ser usado. Os motivos da descontinuação estão indicados na página jQuery sobre este evento

Advertências do evento load quando usado com imagens

Um desafio comum que os desenvolvedores tentam resolver usando o atalho .load() é executar uma função quando uma imagem (ou coleção de imagens) estiver completamente carregada. Existem várias advertências conhecidas com isso que devem ser observadas. Estes são:

  • Não funciona de forma consistente nem confiável entre navegadores
  • Não dispara corretamente no WebKit se a imagem src estiver configurada para o mesmo src de antes
  • Não borbulha corretamente a árvore DOM
  • Pode parar de disparar para imagens que já estão no cache do navegador

Se você ainda deseja usar load(), está documentado abaixo:

$(document).ready() espera até que o DOM completo esteja disponível – todos os elementos no HTML foram analisados ​​e estão no documento. No entanto, recursos como imagens podem não ter sido totalmente carregados neste momento. Se for importante esperar até que todos os recursos sejam carregados, $(window).load() e você estiver ciente das limitações significativas deste evento, então o seguinte pode ser usado:

$(document).ready(function() {
  console.log($("#my_large_image").height()); // may be 0 because the image isn't available
});

$(window).load(function() {
  console.log($("#my_large_image").height()); // will be correct
});

Anexando eventos e manipulando o DOM dentro de ready() #

Exemplo de uso de $(document).ready():

  1. Anexando manipuladores de eventos Anexar manipuladores de eventos jQuery

    $(document).ready(function() { $(“button”).click(function() { // Code for the click function }); });

  2. Execute o código jQuery após a criação da estrutura da página

    jQuery(function($) { // set the value of an element. $("#myElement").val(“Hello”); });

  3. Manipule a estrutura DOM carregada Por exemplo: ocultar um div quando a página for carregada pela primeira vez e mostrá-lo no evento de clique de um botão

    $(document).ready(function() { $("#toggleDiv").hide(); $(“button”).click(function() { $("#toggleDiv").show(); }); });

Diferença entre jQuery(fn) e executar seu código antes de #

Usar o evento document-ready pode ter pequenas desvantagens de desempenho, com atraso na execução de até ~300ms. Às vezes, o mesmo comportamento pode ser obtido pela execução do código logo antes da tag de fechamento </body>:

<body>
  <span id="greeting"></span> world!
  <script>
    $("#greeting").text("Hello");
  </script>
</body>

produzirá um comportamento semelhante, mas funcionará mais cedo, pois não aguarda o acionador de evento pronto para documento, como acontece em:

<head>
  <script>
    jQuery(function($) {
      $("#greeting").text("Hello");
    });
  </script>
</head>
<body>
  <span id="greeting"></span> world!
</body>

Ênfase no fato de que o primeiro exemplo depende do seu conhecimento de sua página e posicionamento do script logo antes da tag de fechamento </body> e especificamente após a tag span.
