Um seletor jQuery seleciona ou encontra um elemento DOM (document object model) em um documento HTML. Ele é usado para selecionar elementos HTML com base em id, nome, tipos, atributos, classe e etc. É baseado em seletores CSS existentes.

Sintaxe #

  • Tag: Sem marcador, use a tag diretamente
  • Código: #id
  • Classe: .className
  • Atributo: [attributeName]
  • Atributo com valor: [attributeName ='value']
  • O atributo começa com o valor ^=: [attributeName ^= 'value']
  • O atributo termina com o valor $=: [attributeName $='value']
  • O atributo contém o valor *= : [attributeName *= 'value']
  • Pseudo-seletor: :pseudo-seletor
  • Qualquer descendente: Espaço entre seletores
  • Filhos diretos: > entre seletores
  • Irmão adjacente após o primeiro: +
  • Irmão não adjacente após o primeiro: ~
  • Ou: , ​​(vírgula) entre seletor

Ao escrever selectors para class ou id ou attribute que contém alguns caracteres especiais como

! " # $ % & ' ( ) * + , . / : ; < = > ? @ [ \ ] ^ { | } ~

os caracteres precisam ser escapados usando duas barras invertidas \\ .

por exemplo.

<span id="temp.foobar"><span>

os seletores serão enquadrados como,

$('#temp\\.foobar')

Tipos de seletores #

No jQuery, você pode selecionar elementos em uma página usando várias propriedades do elemento, incluindo:

  • Modelo
  • Classe
  • EU IRIA
  • Posse de Atributo
  • Valor do atributo
  • Seletor Indexado
  • Pseudo-estado

Se você conhece seletores CSS, você notará que os seletores no jQuery são os mesmos (com pequenas exceções).

Tome o seguinte HTML por exemplo:

<a href="index.html"></a>                  <!-- 1 -->
<a id="second-link"></a>                   <!-- 2 -->
<a class="example"></a>                    <!-- 3 -->
<a class="example" href="about.html"></a>  <!-- 4 -->
<span class="example"></span>              <!-- 5 -->

Selecionando por Tipo:

O seletor jQuery a seguir selecionará todos os elementos <a>, incluindo 1, 2, 3 e 4.

$("a")

Selecionando por Classe

O seletor jQuery a seguir selecionará todos os elementos da classe example (incluindo elementos não-a), que são 3, 4 e 5.

$(".example")

Selecionando por ID

O seletor jQuery a seguir selecionará o elemento com o ID fornecido, que é 2.

$("#second-link")

Seleção por Posse de Atributo

O seletor jQuery a seguir selecionará todos os elementos com um atributo href definido, incluindo 1 e 4.

$("[href]")

Selecionando por Valor de Atributo

O seletor jQuery a seguir selecionará todos os elementos onde o atributo href existe com um valor de index.html, que é apenas 1.

$("[href='index.html']")

Seleção por Posição Indexada (Seletor Indexado)

O seletor jQuery a seguir selecionará apenas 1, o segundo <a> ou seja. o second-link porque o índice fornecido é 1 como eq(1) (Observe que o índice começa em 0, portanto, o segundo foi selecionado aqui!).

$("a:eq(1)")

Seleção com exclusão indexada

Para excluir um elemento usando seu índice :not(:eq())

O seguinte seleciona os elementos <a>, exceto aquele com a classe example, que é 1

$("a").not(":eq(0)")

Seleção com exclusão

Para excluir um elemento de uma seleção, use :not()

O seguinte seleciona os elementos <a>, exceto aqueles com a classe example, que são 1 e 2.

$("a:not(.example)")

Seleção por pseudoestado

Você também pode selecionar no jQuery usando pseudo-estados, incluindo :first-child, :last-child, :first-of-type, :last-of-type, etc.

O seletor jQuery a seguir selecionará apenas o primeiro elemento <a>: número 1.

$("a:first-of-type")

Combinando seletores jQuery

Você também pode aumentar sua especificidade combinando vários seletores jQuery; você pode combinar qualquer número deles ou combinar todos eles. Você também pode selecionar várias classes, atributos e estados ao mesmo tempo.

$("a.class1.class2.class3#someID[attr1][attr2='something'][attr3='something']:first-of-type:first-child")

Isso selecionaria um elemento <a> que:

  • Possui as seguintes classes: class1, class2 e class3
  • Tem o seguinte ID: someID
  • Possui o seguinte Atributo: attr1
  • Possui os seguintes atributos e valores: attr2 com valor something, attr3 com valor something
  • Tem os seguintes estados: first-child e first-of-type

Você também pode separar diferentes seletores com uma vírgula:

$("a, .class1, #someID")

Isso selecionaria:

  • Todos os elementos <a>
  • Todos os elementos que possuem a classe class1
  • Um elemento com o id #someID

Seleção de filhos e irmãos

Os seletores jQuery geralmente seguem as mesmas convenções do CSS, o que permite selecionar filhos e irmãos da mesma maneira.

  • Para selecionar um filho não direto, use um espaço
  • Para selecionar um filho direto, use um >
  • Para selecionar um irmão adjacente após o primeiro, use um +
  • Para selecionar um irmão não adjacente após o primeiro, use um ~

Seleção de curinga

Pode haver casos em que queremos selecionar todos os elementos, mas não há uma propriedade comum para selecionar (classe, atributo etc). Nesse caso podemos usar o seletor * que simplesmente seleciona todos os elementos:

$('#wrapper *')    // Select all elements inside #wrapper element

Visão geral #

Os elementos podem ser selecionados pelo jQuery usando jQuery Selectors. A função retorna um elemento ou uma lista de elementos.

Seletores básicos #

$("*")                       // All elements
$("div")                     // All <div> elements
$(".blue")                   // All elements with class=blue
$(".blue.red")               // All elements with class=blue AND class=red
$(".blue,.red")              // All elements with class=blue OR class=red
$("#headline")               // The (first) element with id=headline
$("[href]")                  // All elements with an href attribute
$("[href='example.com']")    // All elements with href=example.com

Operadores relacionais #

$("div span")          // All <span>s that are descendants of a <div>
$("div > span")        // All <span>s that are a direct child of a <div>
$("a ~ span")          // All <span>s that are siblings following an <a>
$("a + span")          // All <span>s that are immediately after an <a>

Seletores de cache #

Cada vez que você usa um seletor no jQuery, o DOM é pesquisado por elementos que correspondam à sua consulta. Fazer isso com muita frequência ou repetidamente diminuirá o desempenho. Se você se referir a um seletor específico mais de uma vez, deve adicioná-lo ao cache atribuindo-o a uma variável:

var nav = $('#navigation');
nav.show();

Isso substituiria:

$('#navigation').show();

O armazenamento em cache desse seletor pode ser útil se seu site precisar mostrar/ocultar esse elemento com frequência. Se houver vários elementos com o mesmo seletor, a variável se tornará um array desses elementos:

<div class="parent">
    <div class="child">Child 1</div>
    <div class="child">Child 2</div>
</div>

<script>
    var children = $('.child');
    var firstChildText = children[0].text();
    console.log(firstChildText); 
    
    // output: "Child 1"
</script>

OBSERVAÇÃO: o elemento deve existir no DOM no momento de sua atribuição a uma variável. Se não houver nenhum elemento no DOM com uma classe chamada child, você estará armazenando um array vazio nessa variável.

<div class="parent"></div>

<script>
    var parent   = $('.parent');
    var children = $('.child');
    console.log(children);

    // output: []

    parent.append('<div class="child">Child 1</div>');
    children = $('.child');
    console.log(children[0].text());

    // output: "Child 1"
</script>

Lembre-se de reatribuir o seletor à variável após adicionar/remover elementos no DOM com esse seletor.

Observação: Ao armazenar em cache seletores, muitos desenvolvedores iniciarão o nome da variável com um $ para indicar que a variável é um objeto jQuery assim:

var $nav = $('#navigation');
$nav.show();

Combinando seletores #

Considere seguir a estrutura DOM

<ul class="parentUl">
    <li> Level 1
        <ul class="childUl">
            <li>Level 1-1 <span> Item - 1 </span></li>
            <li>Level 1-1 <span> Item - 2 </span></li>
        </ul>
    </li>
    <li> Level 2
        <ul class="childUl">
            <li>Level 2-1 <span> Item - 1 </span></li>
            <li>Level 2-1 <span> Item - 1 </span></li>
        </ul>
    </li>
</ul>

Seletores de descendentes e filhos #

Dado um pai <ul> - parentUl encontre seus descendentes (<li>),

  1. Simples $('pai filho')

>> $('ul.parentUl li')

Isso obtém todos os descendentes correspondentes do ancestral especificado todos os níveis abaixo.

  1. > - $('pai > filho')

>> $('ul.parentUl > li')

Isso encontra todos os filhos correspondentes (apenas 1º nível abaixo).

  1. Seletor baseado em contexto - $('filho','pai')

>> $('li','ul.parentUl')

Isso funciona da mesma forma que 1. acima.

  1. find() - $('pai').find('filho')

>> $('ul.parentUl').find('li')

Isso funciona da mesma forma que 1. acima.

  1. filhos() - $('pai').find('filho')

>> $('ul.parentUl').children('li')

Isso funciona da mesma forma que 2. acima.

Outros combinadores #

Seletor de Grupo: “,” #

Selecione todos os elementos <ul> E todos os elementos <li> E todos os elementos <span> :

$('ul, li, span')

Seletor de múltiplos: "" (sem caractere) #

Selecione todos os elementos <ul> com a classe parentUl:

$('ul.parentUl')

Seletor de irmãos adjacentes: “+” #

Selecione todos os elementos <li> que são colocados imediatamente após outro elemento <li>:

$('li + li')

Seletor Geral de Irmãos: “~” #

Selecione todos os elementos <li> que são irmãos de outros elementos <li>:

$('li ~ li')

Elementos DOM como seletores #

jQuery aceita uma grande variedade de parâmetros, e um deles é um elemento DOM real. Passar um elemento DOM para jQuery fará com que a estrutura subjacente do tipo array do [objeto jQuery](https://www.wikiod.com/pt/jquery/comecando-com-jquery#O objeto jQuery ) para manter esse elemento.

jQuery detectará que o argumento é um elemento DOM inspecionando seu nodeType.

O uso mais comum de um elemento DOM é em callbacks, onde o elemento atual é passado para o construtor do jQuery para obter acesso à API do jQuery.

Como no retorno de chamada each (nota: cada um é uma função iteradora).

$(".elements").each(function(){
    //the current element is bound to `this` internally by jQuery when using each
    var currentElement = this;

    //at this point, currentElement (or this) has access to the Native API
    
    //construct a jQuery object with the currentElement(this)
    var $currentElement = $(this);

    //now $currentElement has access to the jQuery API
});

Strings HTML como seletores #

jQuery aceita uma grande variedade de parâmetros como “seletores”, e um deles é uma string HTML. Passar uma string HTML para jQuery fará com que a estrutura subjacente do tipo array do [objeto jQuery](https://www.wikiod.com/pt/jquery/comecando-com-jquery#O objeto jQuery ) para conter o HTML construído resultante.

jQuery usa regex para determinar se a string que está sendo passada para o construtor é uma string HTML, e também se deve começar com <. Essa regex é definida como rquickExpr = /^(?:\s*(<[\w\W]+>)[^>]*|#([\w-]*))$/ (explanation at regex101.com).

O uso mais comum de uma string HTML como seletor é quando conjuntos de elementos DOM precisam ser criados apenas em código, geralmente isso é usado por bibliotecas para coisas como popouts Modal.

Por exemplo, uma função que retornou uma tag âncora envolvida em um div como modelo

function template(href,text){
    return $("<div><a href='" + href + "'>" + text + "</a></div>");
}

Retornaria um objeto jQuery segurando

<div>
    <a href="google.com">Google</a>
</div>

se chamado como template("google.com","Google").
