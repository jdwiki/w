Sintaxe #

  • var jqXHR = $.ajax(url [,configurações])
  • var jqXHR = $.ajax([configurações])
  • jqXHR.done(function( data, textStatus, jqXHR ) {});
  • jqXHR.fail(function( jqXHR, textStatus, errorThrown ) {});
  • jqXHR.always(function( jqXHR ) {});

Parâmetros #

Parâmetro Detalhes
URL Especifica a URL para a qual a solicitação será enviada
configurações um objeto contendo vários valores que afetam o comportamento da solicitação
tipo O método HTTP a ser usado para a solicitação
dados Dados a enviar pelo pedido
sucesso Uma função de retorno de chamada a ser chamada se a solicitação for bem-sucedida
erro Um retorno de chamada para lidar com o erro
código de estado Um objeto de códigos e funções HTTP numéricos a ser chamado quando a resposta tiver o código correspondente
dataType O tipo de dados que você espera do servidor
contentType Tipo de conteúdo dos dados a serem enviados ao servidor. O padrão é “application/x-www-form-urlencoded; charset=UTF-8”
contexto Especifica o contexto a ser usado dentro de retornos de chamada, geralmente this que se refere ao destino atual.

AJAX significa Asíncrono JavaScript and XML. O AJAX permite que uma página da Web execute uma solicitação HTTP assíncrona (AJAX) ao servidor e receba uma resposta, sem precisar recarregar a página inteira.

Manipulando Códigos de Resposta HTTP com $.ajax() #

Além de .done, .fail e .always, que prometem retornos de chamada, que são acionados com base se a solicitação foi bem-sucedida ou não, há a opção de acionar uma função quando um [HTTP Status Code] específico 1] é retornado do servidor. Isso pode ser feito usando o parâmetro statusCode.

$.ajax({
    type: {POST or GET or PUT etc.},
    url:  {server.url},
    data: {someData: true},
    statusCode: {
        404: function(responseObject, textStatus, jqXHR) {
            // No content found (404)
            // This code will be executed if the server returns a 404 response
        },
        503: function(responseObject, textStatus, errorThrown) {
            // Service Unavailable (503)
            // This code will be executed if the server returns a 503 response
        }           
    }
})
.done(function(data){
    alert(data);
})
.fail(function(jqXHR, textStatus){
    alert('Something went wrong: ' + textStatus);
})
.always(function(jqXHR, textStatus) {
   alert('Ajax request was finished')
});

Como a documentação oficial do jQuery afirma:

Se a solicitação for bem-sucedida, as funções do código de status assumem o mesmo parâmetros como o retorno de chamada de sucesso; se isso resultar em um erro (incluindo redirecionamento 3xx), eles usam os mesmos parâmetros que o error retorno de chamada.

Usando Ajax para enviar um formulário #

Às vezes, você pode ter um formulário e querer enviá-lo usando ajax.

Suponha que você tenha este formulário simples -

<form id="ajax_form" action="form_action.php">
  <label for="name">Name :</label>
  <input name="name" id="name" type="text" />
  <label for="name">Email :</label>
  <input name="email" id="email" type="text" />
  <input type="submit" value="Submit" />
</form>

O seguinte código jQuery pode ser usado (dentro de uma chamada $(document).ready) -

$('#ajax_form').submit(function(event){
  event.preventDefault();
  var $form = $(this);

  $.ajax({
      type: 'POST',
      url: $form.attr('action'),
      data: $form.serialize(),
      success: function(data) {
        // Do something with the response
      },
      error: function(error) {
        // Do something with the error
      }
  });
});

Explicação

  • var $form = $(this) - o formulário, armazenado em cache para reutilização
  • $('#ajax_form').submit(function(event){ - Quando o formulário com ID “ajax_form” for enviado execute esta função e passe o evento como um parâmetro.
  • event.preventDefault(); - Impede que o formulário seja enviado normalmente (Alternativamente, podemos usar return false após a instrução ajax({});, que terá o mesmo efeito)
  • url: $form.attr('action'), - Obtenha o valor do atributo “action” do formulário e use-o para a propriedade “url”.
  • data: $form.serialize(), - Converte as entradas dentro do formulário em uma string adequada para enviar ao servidor. Nesse caso, ele retornará algo como “name=Bob&email=bob@bobsemailaddress.com”

Tudo em um exemplos #

Ajax Get:

Solução 1:

$.get('url.html', function(data){
    $('#update-box').html(data);
});

Solução 2:

 $.ajax({
     type: 'GET',
     url: 'url.php',  
 }).done(function(data){
     $('#update-box').html(data);
 }).fail(function(jqXHR, textStatus){
     alert('Error occured: ' + textStatus);
 });

Ajax Load: Outro método ajax get criado para simplificar

$('#update-box').load('url.html');

.load também pode ser chamado com dados adicionais. A parte de dados pode ser fornecida como string ou objeto.

$('#update-box').load('url.php', {data: "something"});
$('#update-box').load('url.php', "data=something");

Se .load for chamado com um método de retorno de chamada, a solicitação ao servidor será um post

$('#update-box').load('url.php', {data: "something"}, function(resolve){
    //do something
});

Post Ajax:

Solução 1:

$.post('url.php', 
    {date1Name: data1Value, date2Name: data2Value},  //data to be posted
    function(data){
        $('#update-box').html(data);
    }
);

Solução 2:

$.ajax({
    type: 'Post',
    url: 'url.php',  
    data: {date1Name: data1Value, date2Name: data2Value}  //data to be posted
}).done(function(data){
    $('#update-box').html(data);
}).fail(function(jqXHR, textStatus){
    alert('Error occured: ' + textStatus);
});

Ajax Post JSON:

var postData = {
    Name: name,
    Address: address,
    Phone: phone
};

$.ajax({
     type: "POST",
     url: "url.php",
     dataType: "json",
     data: JSON.stringfy(postData),
     success: function (data) {
         //here variable data is in JSON format
     }
 });

Solução 1:

$.getJSON('url.php', function(data){

Solução 2:

      success: function (data) {
          //here variable data is in JSON format
      },

      }
  });

Uploads de arquivos Ajax #

1. Um Exemplo Completo Simples #

Poderíamos usar este código de exemplo para carregar os arquivos selecionados pelo usuário sempre que uma nova seleção de arquivo for feita.

<input type="file" id="file-input" multiple>

 

var files;
var fdata = new FormData();
$("#file-input").on("change", function (e) {
    files = this.files;

    $.each(files, function (i, file) {
        fdata.append("file" + i, file);
    });

    fdata.append("FullName", "John Doe");
    fdata.append("Gender", "Male");
    fdata.append("Age", "24");

    $.ajax({
        url: "/Test/Url",
        type: "post",
        data: fdata, //add the FormData object to the data parameter
        processData: false, //tell jquery not to process data
        contentType: false, //tell jquery not to set content-type
        success: function (response, status, jqxhr) {
            //handle success
        },
        error: function (jqxhr, status, errorMessage) {
            //handle error
        }
    });
});

Agora vamos quebrar isso e inspecioná-lo parte por parte.

2. Trabalhando com entradas de arquivo #

Este Documento MDN ( Usando arquivos de aplicativos da web ) é uma boa leitura sobre vários métodos sobre como lidar com entradas de arquivos. Alguns desses métodos também serão usados ​​neste exemplo.

Antes de fazer o upload de arquivos, primeiro precisamos fornecer ao usuário uma maneira de selecionar os arquivos que deseja enviar. Para este propósito, usaremos uma entrada de arquivo. A propriedade multiple permite selecionar mais de um arquivo, você pode removê-lo se quiser que o usuário selecione um arquivo por vez.

<input type="file" id="file-input" multiple>

Estaremos usando o change event do input para capturar os arquivos.

var files;
$("#file-input").on("change", function(e){
    files = this.files;
});

Dentro da função handler, acessamos os arquivos através da propriedade files de nossa entrada. Isso nos dá um FileList, que é um array como um objeto.

3. Criando e Preenchendo o FormData #

Para fazer upload de arquivos com Ajax vamos usar FormData.

var fdata = new FormData();

FileList que obtivemos na etapa anterior é um objeto tipo array e pode ser iterado usando vários métodos, incluindo for loop, for…of loop e jQuery.each . Vamos ficar com o jQuery neste exemplo.

$.each(files, function(i, file) {
  //...
});

Estaremos usando o método append de FormData para adicionar os arquivos em nosso objeto formdata.

$.each(files, function(i, file) {
  fdata.append("file" + i, file);
});

Também podemos adicionar outros dados que queremos enviar da mesma maneira. Digamos que queremos enviar algumas informações pessoais que recebemos do usuário junto com os arquivos. Poderíamos adicionar essa informação ao nosso objeto formdata.

fdata.append("FullName", "John Doe");
fdata.append("Gender", "Male");
fdata.append("Age", "24");
//...

4. Enviando os arquivos com Ajax #

$.ajax({
    url: "/Test/Url",
    type: "post",
    data: fdata, //add the FormData object to the data parameter
    processData: false, //tell jquery not to process data
    contentType: false, //tell jquery not to set content-type
    success: function (response, status, jqxhr) {
        //handle success
    },
    error: function (jqxhr, status, errorMessage) {
        //handle error
    }
});

Definimos as propriedades processData e contentType como false. Isso é feito para que os arquivos possam ser enviados ao servidor e processados ​​pelo servidor corretamente.

Ajax Aborta uma Chamada ou Solicitação #

var xhr = $.ajax({
    type: "POST",
    url: "some.php",
    data: "name=John&location=Boston",
    success: function(msg){
       alert( "Data Saved: " + msg );
    }
});

//mata o pedido

xhr.abort()

Enviando dados JSON #

jQuery torna o tratamento de respostas jSON indolor, mas um pouco mais de trabalho é necessário quando uma determinada solicitação deseja que você * envie * dados no formato JSON:

 $.ajax("/json-consuming-route", {
      data: JSON.stringify({author: {name: "Bullwinkle J. Moose", 
                                     email: "bullwinkle@example.com"} }),
      method: "POST",
      contentType: "application/json"
   });

Observe que estamos especificando o contentType correto para os dados que estamos envio; esta é uma boa prática em geral e pode ser exigida pela API para a qual você está postando - mas também tem o efeito colateral de instruir o jQuery a não realizar a conversão padrão de %20 para +, que faria se contentType fosse deixado no valor padrão de application/x-www-form-urlencoded. Se por algum motivo você precisar deixar contentType definido como padrão, certifique-se de definir processData como false para evitar isso.

A chamada para [JSON.stringify](https://www.wikiod.com/pt/javascript/json#Serializando um valor) pode ser evitada aqui, mas usando ele nos permite fornecer os dados na forma de um objeto JavaScript (evitando assim erros de sintaxe JSON embaraçosos, como não citar nomes de propriedades).
