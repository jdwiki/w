Sintaxe #

  1. $(seletor).append(conteúdo)
  2. $(conteúdo).appendTo(seletor)

Parâmetros #

Parâmetros Detalhes
conteúdo Tipos possíveis: Elemento, string HTML, texto, array, objeto ou até mesmo uma função que retorna uma string.

  • .append() & .after() podem potencialmente executar código. Isso pode ocorrer por injeção de tags de script ou uso de atributos HTML que executam código (por exemplo, ). Não use esses métodos para inserir strings obtidas de fontes não confiáveis, como parâmetros de consulta de URL, cookies ou entradas de formulário. Fazer isso pode introduzir scripts entre sites (XSS). Remova ou escape de qualquer entrada do usuário antes de adicionar conteúdo ao documento.

  • jQuery não suporta oficialmente SVG. Usando métodos jQuery em SVG documentos, a menos que explicitamente documentados para esse método, podem causar comportamentos inesperados. Exemplos de métodos que suportam SVG a partir de jQuery 3.0 são addClass e removeClass.

Uso eficiente de .append() consecutivo #

Começando:

HTML #

<table id='my-table' width='960' height='500'></table>

JS #

var data = [
    { type: "Name", content: "John Doe" },
    { type: "Birthdate", content: "01/01/1970" },
    { type: "Salary", content: "$40,000,000" },
    // ...300 more rows...
    { type: "Favorite Flavour", content: "Sour" }
];

Anexando dentro de um loop

Você acabou de receber uma grande variedade de dados. Agora é hora de percorrer e renderizá-lo na página.

Seu primeiro pensamento pode ser fazer algo assim:

var i;                        // <- the current item number
var count = data.length;      // <- the total
var row;                      // <- for holding a reference to our row object

// Loop over the array
for ( i = 0; i < count; ++i ) {
    row = data[ i ];

    // Put the whole row into your table
    $('#my-table').append(
        $('<tr></tr>').append(
            $('<td></td>').html(row.type),
            $('<td></td>').html(row.content)
        )
    );
}

Isso é perfeitamente válido e renderizará exatamente o que você esperaria, mas…

Não faça isso. #

Lembra daquelas 300+ linhas de dados?

Cada um forçará o navegador a recalcular os valores de largura, altura e posicionamento de cada elemento, juntamente com quaisquer outros estilos - a menos que sejam separados por um limite de layout, o que infelizmente para este exemplo (como eles são descendentes de um elemento <table>), eles não podem.

Em pequenas quantidades e poucas colunas, essa penalidade de desempenho certamente será insignificante. Mas queremos que cada milissegundo conte.

Melhores opções

  1. Adicionar a uma matriz separada, anexar após a conclusão do loop
/** 
 * Repeated DOM traversal (following the tree of elements down until you reach
 * what you're looking for - like our <table>) should also be avoided wherever possible.
 */

// Keep the table cached in a variable then use it until you think it's been removed
var $myTable = $('#my-table');

// To hold our new <tr> jQuery objects
var rowElements = [];

var count = data.length;
var i;
var row;

// Loop over the array
for ( i = 0; i < count; ++i ) {
    rowElements.push(
        $('<tr></tr>').append(
            $('<td></td>').html(row.type),
            $('<td></td>').html(row.content)
        )
    );
}

// Finally, insert ALL rows at once
$myTable.append(rowElements);

Dessas opções, esta depende mais do jQuery.

  1. Usando métodos modernos de Array.* #

    var $myTable = $(’#my-table’);

    // Looping with the .map() method // - This will give us a brand new array based on the result of our callback function var rowElements = data.map(function ( row ) {

     // Create a row
 var $row = $('<tr></tr>');

 // Create the columns
 var $type = $('<td></td>').html(row.type);
 var $content = $('<td></td>').html(row.content);

 // Add the columns to the row
 $row.append($type, $content);

 // Add to the newly-generated array
 return $row;

    });

    // Finally, put ALL of the rows into your table $myTable.append(rowElements);

Funcionalmente equivalente ao anterior, apenas mais fácil de ler.

  1. Usando strings de HTML (em vez de métodos internos do jQuery)
// ...
var rowElements = data.map(function ( row ) {
    var rowHTML = '<tr><td>';
    rowHTML += row.type;
    rowHTML += '</td><td>';
    rowHTML += row.content;
    rowHTML += '</td></tr>';
    return rowHTML;
});

// Using .join('') here combines all the separate strings into one
$myTable.append(rowElements.join(''));

Perfeitamente válido mas, novamente, não recomendado. Isso força o jQuery a analisar uma quantidade muito grande de texto de uma só vez e não é necessário. jQuery é muito bom no que faz quando usado corretamente.

  1. Crie elementos manualmente, anexe ao fragmento do documento
var $myTable = $(document.getElementById('my-table'));

/**
 * Create a document fragment to hold our columns
 * - after appending this to each row, it empties itself
 *   so we can re-use it in the next iteration.
 */
var colFragment = document.createDocumentFragment();

/**
 * Loop over the array using .reduce() this time.  
 * We get a nice, tidy output without any side-effects.
 *  - In this example, the result will be a 
 *    document fragment holding all the <tr> elements.
 */
var rowFragment = data.reduce(function ( fragment, row ) {

    // Create a row
    var rowEl = document.createElement('tr');

    // Create the columns and the inner text nodes
    var typeEl = document.createElement('td');
    var typeText = document.createTextNode(row.type);
    typeEl.appendChild(typeText);

    var contentEl = document.createElement('td');
    var contentText = document.createTextNode(row.content);
    contentEl.appendChild(contentText);

    // Add the columns to the column fragment
    // - this would be useful if columns were iterated over separately
    //   but in this example it's just for show and tell.
    colFragment.appendChild(typeEl);
    colFragment.appendChild(contentEl);
    
    rowEl.appendChild(colFragment);

    // Add rowEl to fragment - this acts as a temporary buffer to 
    // accumulate multiple DOM nodes before bulk insertion
    fragment.appendChild(rowEl);

    return fragment;
}, document.createDocumentFragment());

// Now dump the whole fragment into your table
$myTable.append(rowFragment);

Meu favorito pessoal. Isso ilustra uma ideia geral do que o jQuery faz em um nível inferior.

Anexando um elemento a um container #

Solução 1:

$('#parent').append($('#child'));

Solução 2:

$('#child').appendTo($('#parent'));

Ambas as soluções estão anexando o elemento #child (adicionando no final) ao elemento #parent.

Antes da:

<div id="parent">
  <span>other content</span>
</div>
<div id="child">

</div>

Depois:

<div id="parent">
  <span>other content</span>
  <div id="child">

  </div>
</div>

Observação: ao anexar conteúdo que já existe no documento, esse conteúdo será removido do contêiner pai original e anexado ao novo contêiner pai. Então você não pode usar .append() ou .appendTo() para clonar um elemento. Se você precisar de um clone, use .clone() -> http://api.jquery.com/clone/
