jQuery lida internamente com eventos através da função addEventListener. Isso significa que é perfeitamente legal ter mais de uma função vinculada ao mesmo evento para o mesmo elemento DOM.

Eventos delegados #

Vamos começar com exemplo. Aqui está um exemplo muito simples de HTML.

Exemplo de HTML

<html>
    <head>
    </head>
    <body>
        <ul>
            <li>
                <a href="some_url/">Link 1</a>
            </li>
            <li>
                <a href="some_url/">Link 2</a>
            </li>
            <li>
                <a href="some_url/">Link 3</a>
            </li>
        </ul>
    </body>
</html>

O problema

Agora, neste exemplo, queremos adicionar um ouvinte de evento a todos os elementos <a>. O problema é que a lista neste exemplo é dinâmica. Os elementos <li> são adicionados e removidos com o passar do tempo. No entanto, a página não é atualizada entre as alterações, o que nos permitiria usar ouvintes de eventos de clique simples para os objetos de link (ou seja, $('a').click()).

O problema que temos é como adicionar eventos aos elementos <a> que vêm e vão.

Informações básicas - Propagação de eventos

Eventos delegados só são possíveis devido à propagação de eventos (geralmente chamada de borbulhamento de eventos). Toda vez que um evento é acionado, ele irá para cima (para a raiz do documento). Eles delegam a manipulação de um evento para um elemento ancestral inalterável, daí o nome de eventos “delegados”.

Portanto, no exemplo acima, clicar no link do elemento <a> acionará o evento ‘click’ nesses elementos nesta ordem:

  • uma
  • li
  • ul
  • corpo
  • html
  • raiz do documento

Solução

Sabendo o que o borbulhamento de eventos faz, podemos pegar um dos eventos desejados que estão se propagando através do nosso HTML.

Um bom lugar para pegá-lo neste exemplo é o elemento <ul>, pois esse elemento não é dinâmico:

$('ul').on('click', 'a', function () {
  console.log(this.href); // jQuery binds the event function to the targeted DOM element
                          // this way `this` refers to the anchor and not to the list
  // Whatever you want to do when link is clicked
});

Acima:

  • Temos ‘ul’ que é o destinatário deste ouvinte de evento
  • O primeiro parâmetro (‘click’) define quais eventos estamos tentando detectar.
  • O segundo parâmetro (‘a’) é usado para declarar de onde o evento precisa se originar (de todos os elementos filhos sob o destinatário do ouvinte do evento, ul).
  • Por último, o terceiro parâmetro é o código que é executado se os requisitos do primeiro e do segundo parâmetros forem atendidos.

Em detalhes como a solução funciona

  1. O usuário clica no elemento <a>
  2. Isso aciona o evento click no elemento <a>.
  3. O evento começa a borbulhar em direção à raiz do documento.
  4. O evento borbulha primeiro para o elemento <li> e depois para o elemento <ul>.
  5. O ouvinte de eventos é executado porque o elemento <ul> tem o ouvinte de eventos anexado.
  6. O ouvinte de eventos primeiro detecta o evento de acionamento. O evento borbulhante é ‘click’ e o ouvinte tem ‘click’, é um passe.
  7. As verificações do ouvinte tentam corresponder o segundo parâmetro (‘a’) a cada item na cadeia de bolhas. Como o último item da cadeia é um ‘a’, isso corresponde ao filtro e também é uma passagem.
  8. O código no terceiro parâmetro é executado usando o item correspondente, pois é this. Se a função não incluir uma chamada para stopPropagation(), o evento continuará se propagando para a raiz (document).

Nota: Se um ancestral inalterável adequado não estiver disponível/conveniente, você deve usar document. Como hábito, não use 'body' pelos seguintes motivos:

  • O body tem um bug, relacionado ao estilo, que pode significar que os eventos do mouse não borbulham nele. Isso depende do navegador e pode acontecer quando a altura do corpo calculada é 0 (por exemplo, quando todos os elementos filhos têm posições absolutas). Eventos de mouse sempre borbulham para document.
  • document sempre existe em seu script, então você pode anexar manipuladores delegados a document fora de um manipulador pronto para DOM e ter certeza de que eles ainda funcionarão.

Anexar e desanexar manipuladores de eventos #

Anexar um manipulador de eventos #

Desde a versão 1.7 jQuery tem a API de eventos .on(). Dessa forma, qualquer evento javascript padrão ou evento personalizado pode ser vinculado ao elemento jQuery selecionado no momento. Existem atalhos como .click(), mas .on() oferece mais opções.

HTML #

<button id="foo">bar</button>

jQuery #

$( "#foo" ).on( "click", function() {
  console.log( $( this ).text() ); //bar
});

Desanexar um manipulador de eventos #

Naturalmente você também tem a possibilidade de desanexar eventos de seus objetos jQuery. Você faz isso usando .off( events [, selector ] [, handler ] ).

HTML #

<button id="hello">hello</button>

jQuery #

$('#hello').on('click', function(){
    console.log('hello world!');
    $(this).off();
});

Ao clicar no botão $(this) fará referência ao objeto jQuery atual e removerá todos os manipuladores de eventos anexados a ele. Você também pode especificar qual manipulador de eventos deve ser removido.

jQuery #

$('#hello').on('click', function(){
    console.log('hello world!');
    $(this).off('click');
});

$('#hello').on('mouseenter', function(){
    console.log('you are about to click');
});

Neste caso, o evento mouseenter ainda funcionará após clicar.

Evento de carregamento de documento .load() #

Se você quiser que seu script espere até que um determinado recurso seja carregado, como uma imagem ou um PDF, você pode usar .load(), que é um atalho para atalho para .on( "load", handler).

HTML

<img src="image.jpeg" alt="image" id="image">

jQuery

$( "#image" ).load(function() {
  // run script
});

Eventos para repetir elementos sem usar IDs #

Problema

Há uma série de elementos repetidos na página que você precisa saber em qual deles ocorreu um evento para fazer algo com essa instância específica.

Solução

  • Dê a todos os elementos comuns uma classe comum
  • Aplicar ouvinte de eventos a uma classe. this dentro do manipulador de eventos é o elemento seletor correspondente no qual o evento ocorreu
  • Atravesse para o contêiner de repetição mais externo para essa instância começando em this
  • Use find() dentro desse contêiner para isolar outros elementos específicos dessa instância

HTML

<div class="item-wrapper" data-item_id="346">
   <div class="item"><span class="person">Fred</span></div>
   <div class="item-toolbar">
      <button class="delete">Delete</button>
   </div>   
</div>
<div class="item-wrapper" data-item_id="393">
   <div clss="item"><span class="person">Wilma</span></div>
   <div class="item-toolbar">
      <button class="delete">Delete</button>
   </div>   
</div>

jQuery

$(function() {
  $('.delete').on('click', function() {
    // "this" is element event occured on
    var $btn = $(this);
    // traverse to wrapper container
    var $itemWrap = $btn.closest('.item-wrapper');
    // look within wrapper to get person for this button instance
    var person = $itemWrap.find('.person').text();
    // send delete to server and remove from page on success of ajax
    $.post('url/string', { id: $itemWrap.data('item_id')}).done(function(response) {
      $itemWrap.remove()
    }).fail(function() {
      alert('Ooops, not deleted at server');
    });
  });
});

evento original #

Às vezes, haverá propriedades que não estão disponíveis no evento jQuery. Para acessar as propriedades subjacentes, use Event.originalEvent

Obter direção de rolagem #

$(document).on("wheel",function(e){
    console.log(e.originalEvent.deltaY)
    // Returns a value between -100 and 100 depending on the direction you are scrolling
})

Ativando e desativando eventos específicos via jQuery. (Ouvintes nomeados) #

Às vezes, você deseja desativar todos os ouvintes registrados anteriormente.

//Adding a normal click handler
$(document).on("click",function(){
    console.log("Document Clicked 1")
});
//Adding another click handler
$(document).on("click",function(){
    console.log("Document Clicked 2")
});
//Removing all registered handlers.
$(document).off("click")

Um problema com este método é que TODOS os listeners vinculados ao document por outros plugins etc também seriam removidos.

Na maioria das vezes, queremos separar todos os ouvintes anexados apenas por nós.

Para conseguir isso, podemos vincular ouvintes nomeados como,

//Add named event listener.
$(document).on("click.mymodule",function(){
    console.log("Document Clicked 1")
});
$(document).on("click.mymodule",function(){
    console.log("Document Clicked 2")
});

//Remove named event listener.
$(document).off("click.mymodule");

Isso garante que qualquer outro ouvinte de clique não seja modificado inadvertidamente.
