Primeiro Plugin no Eclipse #

Pré-requisito #

Este guia assume que você já usou BuildTools e executou o [Spigot server][2] pelo menos uma vez. Ele também pressupõe que você tenha o arquivo jar do Spigot-API que usaremos.

1) Inicie o Eclipse; você pode alterar o local do espaço de trabalho, se desejar.

2) Crie um novo projeto

  1. Defina o nome do projeto como desejar. Aqui, escolhemos MyFirstPlugin.
  2. Clique em próximo.
  3. Selecione Incluir JARs Externos na guia Bibliotecas. Na caixa de diálogo JAR Selection, selecione o arquivo jar spigot-api-shaded, que pode ser encontrado em Spigot/Spigot-API/target/ dentro de sua pasta BuildTools.
  4. Selecione Concluir

3) Adicionar um novo pacote

Clique com o botão direito do mouse em src e clique em Novo > Pacote. Você pode usar qualquer convenção de namespace que desejar, apenas seja consistente. (por exemplo: com.google.android).

4) Crie uma nova turma

  1. Clique com o botão direito do mouse no pacote recém-criado e selecione New > Class.

  2. Dê-lhe qualquer nome; muitas vezes o mesmo nome do projeto. Dentro do editor, a classe Java recém-criada será aberta. O código deve ficar mais ou menos assim:

     package yourpackage;
 public class MyFirstPlugin {
 }

5) Modificar declaração de classe

  1. Sua classe deve se estender do JavaPlugin. O Eclipse produzirá um erro, pois não sabe o que é JavaPlugin. Se você importou com sucesso o Spigot-API, poderá importar o JavaPlugin adicionando a instrução de importação. Você não precisa digitar manualmente essa linha, basta clicar no erro e selecionar a ação apropriada. Seu código agora deve se parecer com:

     package yourpackage;
 import org.bukkit.plugin.java.JavaPlugin;

 public class MyFirstPlugin extends JavaPlugin {

 }

6) Implemente os métodos necessários

A classe JavaPlugin possui alguns métodos abstratos que devem ser implementados pelo seu plugin. Portanto, adicione as funções onEnable e onDisable que serão acionadas quando o plugin for desabilitado ou habilitado no console. Você pode deixá-los em branco por enquanto. Você também deve escrever @Override acima do método.

Nota: Você não precisa adicionar um getLogger quando seu plugin estiver habilitado ou desabilitado, o Bukkit já faz isso para você.

package com.meeku.tutorialPlugin;
import org.bukkit.plugin.java.JavaPlugin;

public class MyFirstPlugin extends JavaPlugin {
    // Fired when plugin is enabled
    @Override
    public void onEnable() {
    }
    // Fired when plugin is disabled
    @Override
    public void onDisable() {

    }
}

7) Crie o arquivo plugin.yml

Clique com o botão direito do mouse no projeto e crie um arquivo Novo > Arquivo. Nomeie-o como plugin.yml. Cole o seguinte:

name: MyFirstPlugin
main: yourpackage.MyFirstPlugin
version: 1.0
commands:

8) Exportação

Como não há erros, podemos exportar este projeto como JAR. Clique com o botão direito do mouse no nome do projeto, selecione Exportar. Na caixa de diálogo consequente, selecione o arquivo JAR. Clique em Avançar. Você pode desmarcar o classpath e o projeto incluir e alterar o destino de exportação para sua pasta de plugins

9) Correndo

Inicie o servidor e você verá que seu plugin foi ativado.
