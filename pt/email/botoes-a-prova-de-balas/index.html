Os botões podem ser uma das partes mais frustrantes do desenvolvimento de e-mail, mas eles podem ser construídos com código em vez de imagens e ainda são exibidos bem em todos os principais clientes de e-mail. Quando construído em HTML/CSS, um botão será exibido em todos os clientes de e-mail, mesmo com imagens desativadas. Tanto o conteúdo quanto o estilo de cada botão podem ser editados rapidamente no modelo HTML/CSS.

Aprimoramento progressivo com consultas de mídia

Para clientes que oferecem suporte a consultas de mídia, qualquer um dos exemplos acima pode ser aprimorado progressivamente no botão.

Por exemplo. @media screen and (-webkit-min-device-pixel-ratio: 0) { /* Insert -webkit-targeted CSS here */ }

Adicione mais preenchimento horizontal no Outlook

A adição condicional de espaço ininterrupto em cada lado de um link de botão pode aumentar o preenchimento horizontal do Outlook. Especialmente útil para botões baseados em bordas.

`<!--[if mso]>&nbsp;&nbsp;&nbsp;<![endif]-->Link Text<!--[if mso]>&nbsp;&nbsp;&nbsp;<![endif]-->`

Buttons.cm Button #

O Campaign Monitor criou uma ferramenta bacana que gera código para botões em e-mail HTML. Ele usa uma combinação de CSS e VML (para Microsoft Outlook) para exibir imagens de fundo ou padrões na maioria dos principais clientes de e-mail.

    <w:anchorlock/>
    <center style="color:#ffffff;font-family:sans-serif;font-size:13px;font-weight:bold;">Stack Overflow</center>
  </v:roundrect>
<![endif]--><a href="http://www.stackoverflow.com"

Botões baseados em preenchimento #

Essa técnica se baseia no preenchimento no nível da célula da tabela para estruturar o botão e em atributos HTML e CSS para estilizar o botão. Nenhum VML significa código mais simples, mas nenhuma imagem de fundo no Outlook.

<table width="100%" border="0" cellspacing="0" cellpadding="0">
  <tr>
    <td>
      <table border="0" cellspacing="0" cellpadding="0">
        <tr>
          <td bgcolor="#0095FF" style="padding: 12px 18px 12px 18px; -webkit-border-radius:3px; border-radius:3px" align="center"><a href="http://www.stackoverflow.com" target="_blank" style="font-size: 14px; font-family: sans-serif; font-weight: normal; color: #ffffff; text-decoration: none; display: inline-block;">Stack Overflow</a></td>
        </tr>
      </table>
    </td>
  </tr>
</table>

A desvantagem dos botões baseados em preenchimento é que toda a área do botão não é clicável, apenas o link dentro da célula da tabela.

Botões baseados em bordas #

Essa técnica se baseia na adição de bordas grossas ao próprio link para criar o CTA do botão. O uso de bordas é universalmente entendido pelos clientes de e-mail, mas limita a aparência do botão com cores sólidas.

<table width="100%" border="0" cellspacing="0" cellpadding="0">
  <tr>
    <td>
      <table border="0" cellspacing="0" cellpadding="0">
        <tr>
          <td bgcolor="#0095FF" style="padding: 12px 18px 12px 18px; -webkit-border-radius:3px; border-radius:3px" align="center"><a href="http://www.stackoverflow.com" target="_blank" style="font-size: 14px; font-family: sans-serif; font-weight: normal; color: #ffffff; text-decoration: none; display: inline-block;">Stack Overflow</a></td>
        </tr>
      </table>
    </td>
  </tr>
</table>

A tag de link é definida para ser um elemento de nível de bloco e as bordas são usadas para fornecer o preenchimento, de modo que todo o botão pode passar o mouse + clicável, mesmo em clientes de desktop mais antigos.

A desvantagem dos botões baseados em bordas é que o Outlook reduzirá o tamanho das bordas em uma pequena quantidade e, como mencionado acima, não há suporte para imagens de plano de fundo.

Preenchimento + Botões baseados em bordas #

Essa técnica usa uma combinação de botões com base em borda e com base em preenchimento, estilizando o link com preenchimento e pelo menos uma borda sólida de 1 px. A cor de fundo precisa ser aplicada ao em vez do nesta instância porque o Outlook reconhece o preenchimento horizontal na marca (já que não é uma marca de nível de bloco).

<table width="100%" border="0" cellspacing="0" cellpadding="0">
  <tr>
    <td>
      <table border="0" cellspacing="0" cellpadding="0">
        <tr>
          <td align="center" style="-webkit-border-radius: 3px; -moz-border-radius: 3px; border-radius: 3px;" bgcolor="#0095FF"><a href="https://www.stackoverflow.com" target="_blank" style="font-size: 14px; font-family: sans-serif; color: #ffffff; text-decoration: none; text-decoration: none; -webkit-border-radius: 3px; -moz-border-radius: 3px; border-radius: 3px; padding: 12px 18px; border: 1px solid #0095FF; display: inline-block;">Stack Overflow</a></td>
        </tr>
      </table>
    </td>
  </tr>
</table>

A desvantagem dos botões padding+border-based é que ele separa o estilo entre uma tag <td> e <a>, então pode ser difícil de manter. Mas apenas um botão é usado, dimensões específicas não são necessárias e imagens de fundo podem ser suportadas.

Botão à prova de balas com espaçadores #

Este botão deriva de este exemplo por Email on Acid. É totalmente baseado em código, por isso será exibido sem imagens baixadas, e o botão inteiro é passível de passar o mouse + clicável.

Além disso, este exemplo também inclui espaçadores para ajudar a controlar quanto espaço vertical aparece antes e depois do botão.

<table width="100%">
    <tr>
      <td>
        <!-- Top Spacer : BEGIN -->
        <table border="0" cellpadding="0" cellspacing="0"> 
          <tr>
            <td height="20" width="100%" style="font-size: 20px; line-height: 20px;">
            &nbsp;
            </td>
          </tr>
        </table>
        <!-- Top Spacer : END -->
        <!-- Button : BEGIN -->
        <table border="0" align="left" cellpadding="0" cellspacing="0">
          <tbody>
          <tr>
            <td align="left">
              <table border="0" cellpadding="0" cellspacing="0" width="150">
                <tr>
                  <td align="center" bgcolor="#0077CC" width="150" style="-moz-border-radius: 4px; -webkit-border-radius: 4px; border-radius: 4px;">
                    <a href="http://www.stackoverflow.com" style="padding: 10px; width:150px; display: block;text-decoration: none; border:0;text-align: center; font-weight: bold; font-size: 14px; font-family: sans-serif; color: #ffffff; background: #0095FF;border: 1px solid #0077CC; -moz-border-radius: 4px; -webkit-border-radius: 4px; border-radius: 4px; line-height:17px;" class="button_link">Stack Overflow</a>
                  </td>
                </tr>
              </table>
            </td>
          </tr>
          </tbody>
        </table>
        <!-- Button : END -->
        <!-- Bottom Spacer : BEGIN -->
        <table border="0" cellpadding="0" cellspacing="0"> 
          <tr>
            <td height="20" width="100%" style="font-size: 20px; line-height: 20px;">
            &nbsp;
            </td>
          </tr>
        </table>
        <!-- Bottom Spacer : END -->
      </td>
    </tr>
  </table>
