Gerenciadores de Geometria Tkinter

Existem três gerenciadores de geometria para posicionar widgets: pack(), grid() e place().

pacote() #

O gerenciador de geometria pack() organiza os widgets em blocos antes de colocá-los no widget pai. Ele usa as opções fill, expand e side.

Sintaxe

widget.pack(opção)

Preencher
Determina se o widget mantém o espaço mínimo necessário ou ocupa algum espaço extra alocado a ele. Atributos: NONE (padrão), X (preencher horizontalmente), Y (preencher verticalmente) ou BOTH (preencher horizontal e verticalmente).

Expandir
Quando definido como YES, o widget se expande para preencher qualquer espaço não usado no pai do widget. Atributos: SIM, NÃO.

Lado
Determina para qual lado do pai do widget ele é empacotado. Atributos: TOP (padrão), BOTTOM, LEFT ou RIGHT.

Exemplo

from tkinter import *
root = Tk()
btn_fill = Button(root, text="Button")
btn_fill.pack(fill=X)

btn_expand = Button(root, text="Button")
btn_expand.pack(expand=YES)

btn_side = Button(root, text="Button")
btn_side.pack(side=RIGHT)

root.mainloop()

rede() #

O gerenciador de geometria grid() organiza os widgets em uma estrutura semelhante a uma tabela no widget pai. O widget mestre é dividido em linhas e colunas, e cada parte da tabela pode conter um widget. Ele usa column, columnspan, ipadx, ipady, padx, pady, row, rowspan e sticky.

Sintaxe

widget.grid(opções)

Coluna
A coluna para colocar o widget. A coluna padrão é 0, que é a coluna mais à esquerda.

Columnspan
Quantas colunas o widget ocupa. O padrão é 1.

Ipadx
Quantos pixels para preencher o widget horizontalmente dentro das bordas do widget.

Ipad
Quantos pixels para preencher o widget verticalmente dentro das bordas do widget.

Padx
Quantos pixels para preencher o widget horizontalmente fora das bordas do widget.

Pad
Quantos pixels para preencher o widget verticalmente fora das bordas do widget.

Linha
A linha para colocar o widget. A linha padrão é 0, que é a coluna superior.

Linha
Quantas linhas o widget ocupa. O padrão é 1.

Pegajoso
Quando o widget é menor que a célula, sticky é usado para indicar em quais lados e cantos da célula o widget fica. A direção é definida pelas direções da bússola: N, E, S, W, NE, NW, SE e SW e zero. Pode ser uma concatenação de strings, por exemplo, NESW faz com que o widget ocupe toda a área da célula.

Exemplo

from tkinter import *
root = Tk()
btn_column = Button(root, text="I'm in column 3")
btn_column.grid(column=3)

btn_columnspan = Button(root, text="I have a columnspan of 3")
btn_columnspan.grid(columnspan=3)

btn_ipadx = Button(root, text="ipadx of 4")
btn_ipadx.grid(ipadx=4)

btn_ipady = Button(root, text="ipady of 4")
btn_ipady.grid(ipady=4)

btn_padx = Button(root, text="padx of 4")
btn_padx.grid(padx=4)

btn_pady = Button(root, text="pady of 4")
btn_pady.grid(pady=4)

btn_row = Button(root, text="I'm in row 2")
btn_row.grid(row=2)

btn_rowspan = Button(root, text="Rowspan of 2")
btn_rowspan.grid(rowspan=2)

btn_sticky = Button(root, text="I'm stuck to north-east")
btn_sticky.grid(sticky=NE)

root.mainloop()

Lugar, colocar() #

O gerenciador place() organiza os widgets colocando-os em uma posição específica no widget pai. Este gerenciador de geometria usa as opções anchor, bordermode, height, width, relheight, relwidth,relx, rely, x e y.

Âncora
Indica onde o widget está ancorado. As opções são direções da bússola: N, E, S, W, NE, NW, SE ou SW, que se relacionam com os lados e cantos do widget pai. O padrão é NW (o canto superior esquerdo do widget)

Modo fronteira
Bordermode tem duas opções: INSIDE, que indica que outras opções se referem ao interior do pai, (Ignorando as bordas do pai) e OUTSIDE, que é o oposto.

Altura
Especifique a altura de um widget em pixels.

Largura
Especifique a largura de um widget em pixels.

Realtura
Altura como um float entre 0,0 e 1,0, como uma fração da altura do widget pai.

Relwidth
Width como um float entre 0,0 e 1,0, como uma fração da largura do widget pai.

Relx
Deslocamento horizontal como um float entre 0,0 e 1,0, como uma fração da largura do widget pai.

Confie
Deslocamento vertical como um float entre 0,0 e 1,0, como uma fração da altura do widget pai.

X
Deslocamento horizontal em pixels.

S
Deslocamento vertical em pixels.

Exemplo

from tkinter import *
root = Tk()
root.geometry("500x500")

btn_height = Button(root, text="50px high")
btn_height.place(height=50, x=200, y=200)

btn_width = Button(root, text="60px wide")
btn_width.place(width=60, x=300, y=300)

btn_relheight = Button(root, text="Relheight of 0.6")
btn_relheight.place(relheight=0.6)

btn_relwidth= Button(root, text="Relwidth of 0.2")
btn_relwidth.place(relwidth=0.2)

btn_relx=Button(root, text="Relx of 0.3")
btn_relx.place(relx=0.3)

btn_rely=Button(root, text="Rely of 0.7")
btn_rely.place(rely=0.7)

btn_x=Button(root, text="X = 400px")
btn_x.place(x=400)

btn_y=Button(root, text="Y = 321")
btn_y.place(y=321)

root.mainloop()
