Criando sua chave SSH #

Você pode criar sua chave ssh usando ssh-keygen, é um programa que faz parte da instalação do ssh. Para isso, basta executá-lo e seguir as instruções na tela.

Aqui está um exemplo:

$ ssh-keygen
Generating public/private rsa key pair.

O diretório padrão onde seu par de chaves ssh será salvo está dentro da pasta .ssh em seu diretório pessoal (você pode alterar isso especificando um caminho válido) e o nome padrão para o par de chaves é id_rsa para a chave privada e id_rsa.pub para a chave pública:

Enter file in which to save the key (/home/nasreddine/.ssh/id_rsa): /home/my_folder/my_ssh_key

Você pode proteger sua chave SSH contra uso não autorizado digitando uma senha. Isso é opcional, mas é recomendável que você use uma senha. Observe que, como em qualquer outro programa de comando, ao inserir sua senha, ela não mostrará nada na tela, mas está sendo gravada:

Enter passphrase (empty for no passphrase):
Enter same passphrase again:

Depois de inserir sua senha, o ssh-keygen gerará uma chave e a salvará no caminho que você escolheu:

Your identification has been saved in /home/my_folder/my_ssh_key.
Your public key has been saved in /home/my_folder/my_ssh_key.pub.

Foram realizadas. Agora nossa chave ssh está pronta para uso.

Como SSH em uma máquina #

Para fazer login na conta de um usuário na máquina com SSH, você pode usar o comando ssh username@ip_address. Ele vai pedir uma senha. Se você digitar a senha correta, será conectado ao shell desse usuário nessa máquina. Caso contrário, ele solicitará a senha novamente.

Por exemplo

root@dev10:~# ssh root@10.11.50.3
root@10.11.50.3's password: 
Welcome to Ubuntu 16.04 LTS (GNU/Linux 4.4.0-31-generic x86_64)
    
Last login: Fri Jul 22 18:33:27 2016 from 10.11.50.10
root@dev2:~#

Se você quiser usar uma chave ssh específica para se conectar a uma máquina, use ssh -i /path/to/ssh_secret_key username@host

Quando você estiver se conectando a uma máquina pela primeira vez, ele solicitará que você verifique a impressão digital da máquina de destino. Este é um mecanismo de segurança para evitar um ataque man-in-the-middle. Você pode ver a impressão digital da máquina de destino emitindo este comando na máquina de destino.

ssh-keygen -l -E md5 -f /etc/ssh/ssh_host_ecdsa_key.pub

Você pode digitar “sim” se ambos forem iguais. Ele irá prosseguir para o prompt de senha.

Exemplo:

root@dev10:~# ssh root@10.11.50.3
The authenticity of host '10.11.50.3 (10.11.50.3)' can't be established.
ECDSA key fingerprint is dd:a3:de💿5b:01:cd:0b:b6:bc:b3:09:c2:c8:1a:68.
Are you sure you want to continue connecting (yes/no)? yes
Warning: Permanently added '10.11.50.3' (ECDSA) to the list of known hosts.
root@10.11.50.3's password: 

Last login: Fri Jul 22 17:45:09 2016 from 10.11.1.71
root@dev2:~#

Instalação ou Configuração #

Versão gratuita da implementação do protocolo SSH, OpenSSH está disponível em todas as distribuições Linux. Consiste nos pacotes do servidor e do cliente.

Instalação #

No Linux baseado em Debian, você pode instalar o openssh usando #

# apt-get install openssh-server openssh-client

No RHEL/CentOS você precisa usar yum: #

# yum install openssh-server openssh-clients

O Fedora atual está usando dnf em vez de yum.

No Arch Linux, use o pacman: #

# pacman -S openssh

No OSX, o openssh já deve estar instalado. #

Se você quiser usar uma versão mais recente, você precisa instalar o openssh do brew:

# brew install openssh --with-brewed-openssl --with-keychain-support

Instruções TODO para Windows #

Configurar #

O cliente openssh não precisa de nenhuma configuração especial e está pronto para uso logo após a instalação. Você pode tentar executar ssh remote, onde remote é o host remoto que executa o servidor ssh.

O servidor openssh geralmente é iniciado após a instalação e a configuração padrão ser aplicada. Caso contrário, você pode iniciá-lo em sistemas baseados em systemd usando

No Linux baseado em Debian com systemd:

# systemctl start ssh

No RHEL/CentOS/Fedora e Arch Linux:

# systemctl start sshd

Ou em sistemas iniciantes usando

# service sshd start

Configuração #

O openssh tem arquivos de configuração em /etc/ssh/. O cliente também está lendo a configuração do cliente em ~/.ssh/config. O servidor está usando um arquivo sshd_config, que contém a maioria dos valores padrão e contém pares de valores-chave simples. Exemplo:

Protocol 2
PasswordAuthentication yes
ChallengeResponseAuthentication no
UsePAM yes
AcceptEnv LANG LC_CTYPE LC_NUMERIC LC_TIME LC_COLLATE LC_MONETARY
X11Forwarding yes
Subsystem    sftp    /usr/libexec/openssh/sftp-server

Adicionando sua chave pública à lista de chaves autorizadas do usuário do servidor #

Para fazer o ssh em um servidor, a chave pública de sua identidade deve ser adicionada à lista de chaves confiáveis. Mais comumente, isso é feito por usuário:

ssh-copy-id -i ~/.ssh/<identity>.pub <user>@<hostname>

O que também pode ser feito manualmente:

cat ~/.ssh/<identity>.pub | ssh <user>@<hostname> 'cat >> ~/.ssh/authorized_keys'

Depois de fazer isso, você deve ser capaz de fazer login sem precisar fornecer a senha do usuário ao passar o arquivo de identidade para a chamada ssh.

Arquivo de configuração #

Os arquivos de configuração do OpenSSH são usados ​​para configuração que deve ser aplicada toda vez que o cliente ssh for executado. A maioria das opções de linha de comando é possível colocar nos arquivos de configuração.

O OpenSSH usa a configuração das seguintes fontes em ordem:

  1. Opções de linha de comando
  2. Arquivo de configuração do usuário ~/.ssh/config
  3. Arquivo de configuração de todo o sistema /etc/ssh/ssh_config

As opções de configuração são listadas uma a uma nos arquivos de configuração.

# This is a comment.

# Parameter can be specified like this, separated with white space.
StrictHostKeyChecking ask

# Or parameter key and value may be separated with white space and =.  
ForwardX11 = yes

# The parameter value can be quoted if it contains white space.    
IdentityFile "/file system/path with/white space"

A lista completa de parâmetros de configuração possíveis está disponível aqui.

Um dos recursos mais úteis do arquivo de configuração é que ele pode ser seccionado com base no nome ou endereço do host. Desta forma, você pode ter diferentes configurações para diferentes hosts.

# Based on host name.    

Host host1.domain.com
    User user1

Host host2.domain.com
    User user2

# Or wildcard matching name or ip.

Host *elastic-cloud.com 10.201.4.?
    User user3

Conectando do script usando senha #

Quando você realmente precisa fazer um script de conexão ssh, canalizar a senha para o comando ssh não funciona (echo passw0rd | ssh host). É porque a senha não é lida da entrada padrão, mas diretamente do TTY (teleprinter, teletypewriter, Teletype por razões históricas).

Mas existe a ferramenta sshpass que resolve este problema. Ele pode ler a senha do parâmetro, arquivo ou variável de ambiente. Mas observe que nenhuma dessas opções não atende aos requisitos de segurança para senhas!

$ sshpass -p passw0rd ssh host
$ sshpass -f /secret/filename ssh host
$ SSHPASS=passw0rd sshpass -e ssh host

As opções de linha de comando podem ser vistas por outros usuários em ps (durante o tempo de execução é mascarado, mas não durante o tempo de início e você não pode confiar nisso):

... 23624  6216 pts/5    Ss   Aug30   0:00  \_ /bin/bash
... 12812  1988 pts/5    S+   08:50   0:00  |   \_ sshpass -p passw0rd ssh host
... 45008  5796 pts/15   Ss+  08:50   0:00  |       \_ ssh host

Observe que as variáveis ​​environemnet de um processo também são acessíveis por outros processos no sistema usando o arquivo /proc/PID/environ.

Finalmente, armazenar a senha no arquivo pode parecer a melhor ideia possível, mas ainda usar chaves como descrito nos outros exemplos é a maneira preferida de usar ssh.

Agente SSH #

Se você definiu uma senha longa e não deseja continuar inserindo-a toda vez que quiser se conectar ao servidor, você pode usar o SSH-Agent para armazenar sua senha enquanto estiver conectado ao seu computador.

Inicie o agente ssh em segundo plano:

eval "$(ssh-agent -s)"
# Agent pid 59566

E adicione sua chave ao agente ssh, você será solicitado a inserir sua senha:

ssh-add ~/.ssh/matrix.ac
# Enter passphrase for /home/osaka/.ssh/matrix.ac: 
# Identity added: /home/osaka/.ssh/matrix.ac (/home/osaka/.ssh/matrix.ac)

Feito. Agora conecte-se com ssh user@matrix.ac e você não deverá ter que digitar sua senha. Você pode estender isso usando programas como gnome-keyring/seahorse, keychain e outros gerenciadores de chaves.
