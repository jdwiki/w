OpenSSH #

Criar um túnel ssh reverso leva apenas um switch -R para o comando original.

Linha de comando #

Vamos supor que você esteja se conectando ao example.com como um usuário guest usando um comando ssh guest@example.com. A abertura do túnel reverso pode ser assim:

ssh -R 2222:localhost:22 guest@example.com

Ele abrirá uma porta 2222 no servidor remoto (somente interface de loopback) e cada conexão a esta porta será encaminhada para o servidor ssh do seu computador local (porta 22).

Isso também assume que você permitiu as opções AllowTcpForwarding yes e PermitOpen any em seu sshd_config em seu servidor. Caso contrário, ele falhará com erro

open failed: administratively prohibited: open failed

Se você deseja permitir que a porta encaminhada seja acessível em outros endereços de rede (além de localhost), você precisa permitir adicionalmente GatewayPorts yes e usar um endereço IP ou nome de host ou IP):

ssh -R 2222:example.com:22 guest@example.com

Configuração #

Além disso, você pode especificar seu encaminhamento de porta remoto em seu ~/.ssh/config para evitar digitar a mesma linha toda vez que você se conectar. A boa prática pode ser configurar também um alias para o host, que terá esse encaminhamento, se você se conectar ao seu host com frequência e não quiser iniciar o encaminhamento de porta todas as vezes:

Host example.com-R
  Hostname example.com
  User guest
  RemoteForward 2222 localhost:22

e, em seguida, crie o encaminhamento de porta remoto simplesmente usando ssh example.com-R

Executando em segundo plano #

O encaminhamento de porta pode ser simplesmente executado em segundo plano usando os switches -N (não execute o comando remoto, apenas o encaminhamento), -f (ir para segundo plano após a autenticação), -T (desativar alocação remota de TTY ). Juntando tudo:

ssh -NTfR 2222:localhost:22 guest@example.com
