Instalação ou Configuração #

Obtendo zsh #

O zsh está disponível em muitas plataformas do tipo UNIX por meio de seus sistemas integrados de gerenciamento de pacotes. Nas distribuições Debian e Ubuntu Linux, o zsh está disponível nos repositórios de pacotes padrão e pode ser instalado usando:

$ sudo apt-get install zsh
# or, on newer Ubuntu distributions
$ sudo apt install zsh

Em distribuições baseadas em RPM, zsh também está frequentemente disponível nos arquivos de pacotes padrão e pode ser instalado usando:

$ yum install zsh

No Fedora 22 e posterior:

$ dnf install zsh

Em sistemas BSD, zsh pode ser instalado usando pkg:

$ pkg install zsh

No OpenBSD, zsh pode ser instalado usando pkg_add:

$ pkg_add zsh

No Arch Linux, zsh pode ser instalado usando pacman:

$ pacman -S zsh

No openSUSE, o zsh pode ser instalado usando o zypper:

$ zypper install zsh

Em sistemas que executam o macOS (OS X), o zsh já está instalado por padrão, embora não esteja definido como shell padrão. Você também pode instalar versões mais recentes via Homebrew:

$ brew install zsh

Alternativamente, o código fonte do zsh pode ser obtido no site oficial.

A partir daí, o shell pode ser iniciado digitando zsh no prompt.

Tornando zsh seu shell padrão #

Na maioria dos sistemas Linux e BSD, zsh pode ser definido como o shell padrão para um usuário usando o comando chsh:

$ chsh -s shell [username]

Onde

  • username é um nome de usuário real (o padrão é o usuário atual se deixado de fora)
  • shell é o caminho para o binário zsh. O caminho deve ser listado no arquivo /etc/shells, que contém uma lista de shells permitidos para uso com chsh. Caso zsh não esteja listado lá - por exemplo, porque você compilou e instalou a partir da fonte - você precisará adicionar uma linha com o caminho absoluto para zsh primeiro. Você pode obter este caminho com which zsh (desde que esteja instalado em um diretório listado em PATH)

Para ver as alterações, saia uma vez e faça login. Abra o emulador de terminal e use

`echo $SHELL`

Se ele exibir /bin/zsh, você alterou com sucesso o shell padrão para zsh.

Configuração #

Ao iniciar o Zsh, ele irá originar os seguintes arquivos nesta ordem por padrão:

  1. /etc/zsh/zshenv Usado para definir variáveis ​​de ambiente de todo o sistema; ele não deve conter comandos que produzem saída ou assumem que o shell está anexado a um tty. Este arquivo sempre será originado, isso não pode ser substituído.

  2. $ZDOTDIR/.zshenv Usado para definir as variáveis ​​de ambiente do usuário; ele não deve conter comandos que produzem saída ou assumem que o shell está anexado a um tty. Este arquivo será sempre originado.

  3. /etc/zsh/zprofile Usado para executar comandos no início, será originado ao iniciar como um shell de login.

Observe que no Arch Linux, por padrão, ele contém uma linha que origina o arquivo /etc/profile.

/etc/profile Este arquivo deve ser originado por todos os shells compatíveis com Bourne no login: ele configura $PATH e outras variáveis ​​de ambiente e configurações específicas do aplicativo (/etc/profile.d/*.sh) Conecte-se.

  1. $ZDOTDIR/.zprofile Usado para executar os comandos do usuário no início, será originado ao iniciar como um shell de login.

  2. /etc/zsh/zshrc Usado para definir a configuração do shell interativo e executar comandos, será originado ao iniciar como um shell interativo.

  3. $ZDOTDIR/.zshrc Usado para definir a configuração do shell interativo do usuário e executar comandos, será originado ao iniciar como um shell interativo.

  4. /etc/zsh/zlogin Usado para executar comandos no final do progresso inicial, será originado ao iniciar como um shell de login.

  5. $ZDOTDIR/.zlogin Usado para executar os comandos do usuário no final do progresso inicial, será originado ao iniciar como um shell de login.

  6. $ZDOTDIR/.zlogout Será originado quando um shell de login for encerrado.

  7. /etc/zsh/zlogout Será originado quando um shell de login for encerrado.

Se $ZDOTDIR não estiver definido, $HOME será usado.

Para uso pessoal geral, é comum editar o arquivo .zshrc do usuário para preferências pessoais

Apelido #

Para alias de um comando em seu arquivo ~/.zshrc, você pode usar a seguinte sintaxe:

alias [alias-name]="[command-to-execute]"

Por exemplo, é comum executar o comando ls -a. Você pode apelidar este comando como la assim:

alias la="ls -a"

Após recarregar o arquivo ~/.zshrc, você poderá digitar la e ls -a será executado.

Aliases de diretório #

É comum ter certas pastas nas quais você ‘cd’ frequentemente. Se este for o caso, você pode criar apelidos para esses diretórios para facilitar o cd para eles. Por exemplo, o seguinte será um alias para a pasta Dropbox:

alias db="cd ~/Dropbox"

permitindo que você insira db e mude os diretórios para ~/Dropbox.

Recarregue a configuração do ZSH #

zsh carrega a configuração do arquivo ~/.zshrc na inicialização. Se você fizer alterações nesse arquivo, poderá reiniciar o zsh ou executar o comando a seguir para recarregar a configuração.

. ~/.zshrc

Você pode usar um apelido para este comando útil em seu ~/.zshrc assim:

alias reload=". ~/.zshrc"
