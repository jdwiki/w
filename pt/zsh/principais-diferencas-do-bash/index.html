Apelido #

Alias ​​globais #

No bash, os aliases só podem ser colocados no início de um comando, mas o zsh suporta aliases em qualquer lugar. Se você colocar a seguinte linha em seu $ZDOTDIR/.zshrc

alias -g G=' | grep -i'

Você pode então executar

cat haystack.txt G "needle"

Aliases de sufixo (Adicionado no ZSH 4.2.x) #

Os aliases de sufixo permitem que você diga ao zsh para abrir arquivos com especificar um programa para abrir arquivos com certas extensões. Exemplos:

alias -s c="emacs"
alias -s php="vim"
alias -s java="$EDITOR"

Agora em seu shell, se você tiver um arquivo php, file.php, e executar o comando

file.php

Ele irá abrir automaticamente file.php no vim.

Tubos e subconchas #

No bash, cada comando em um pipeline é executado em um subshell. Em zsh, o último comando em um pipeline é executado no shell atual. Por exemplo, o código a seguir

var="before"
echo "after" | read var
echo $var

imprimirá antes no bash, mas depois no zsh.

Diferenças na cotação #

No bash, você precisa citar argumentos para preservar o espaço em branco:

# bash

function print_first_argument {
    echo "$1"
}

argument="has white space"
print_first_argument "$argument"

No Zsh, você não precisa das cotações, devido à ordem de avaliação diferente:

# zsh

function print_first_argument {
    echo $1
}

argument="has white space"
print_first_argument $argument

Manipulação de curingas #

Quando nada corresponde a um curinga como * no bash, ele é passado como um literal * para o comando, como se você tivesse digitado \*. No entanto, zsh gera um erro.

Bash:

duncan@K7DXS-Laptop-Arch:~/test$ echo *.txt
*.txt
duncan@K7DXS-Laptop-Arch:~/test$ touch abc.txt
duncan@K7DXS-Laptop-Arch:~/test$ echo *.txt
abc.txt
duncan@K7DXS-Laptop-Arch:~/test$

Zsh:

K7DXS-Laptop-Arch% echo *.txt   
abc.txt
K7DXS-Laptop-Arch% rm abc.txt 
K7DXS-Laptop-Arch% echo *.txt
zsh: no matches found: *.txt
K7DXS-Laptop-Arch%

Isso é mais perceptível em programas que usam um literal *, como find:

duncan@K7DXS-Laptop-Arch:~/test$ ls -R
.:
abc

./abc:
123.txt

Bash:

duncan@K7DXS-Laptop-Arch:~/test$ find -name *.txt
./abc/123.txt
duncan@K7DXS-Laptop-Arch:~/test$

Zsh:

K7DXS-Laptop-Arch% find -name *.txt
zsh: no matches found: *.txt
K7DXS-Laptop-Arch% find -name \*.txt
./abc/123.txt
K7DXS-Laptop-Arch% find -name '*.txt' # Notice single rather than double quotes
./abc/123.txt
K7DXS-Laptop-Arch%
