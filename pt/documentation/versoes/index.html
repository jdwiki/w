Tabela de versões #

Cada tag tem um tópico de visão geral. Este tópico tem uma seção especial chamada “Versões”. Nessa seção, diferentes versões podem ser definidas em uma tabela que deve

  • ter pelo menos 2 colunas:

    1. the first one should be the name of the version

    2. the last one should be the release date of the version, by which versions will be ordered.

      It should be in the yyyy-MM-dd format and a valid date.

Pode haver colunas adicionais entre os dois para incluir mais informações. Faz sentido listar as versões em ordem cronológica.

Esta marcação:

| Version | Additional Information | Release date |
| --- | ----| ----|
| 1.R | version names do not have to be numbers | 1980-11-25 |
| 1 | we have to start somewhere | 1980-11-24 |

produz este resultado:

Versão Informações Adicionais Data de lançamento
1.R nomes de versão não precisam ser números 1980-11-25
1 temos que começar em algum lugar 1980-11-24

Tabelas de várias versões #

Para algumas tags, faz sentido ter várias tabelas de versão. Pode haver diferentes subconjuntos de uma linguagem de programação ou estrutura disponível, digamos, para diferentes tipos de dispositivos, como neste exemplo.

As diferentes tabelas devem ser precedidas por um título.

Esta marcação:

## desktop development kit ##

| Version | Release date |
| --- | ----|
| 1.0 | 1980-12-24 |
| 1 RC | 1980-11-25 |
| 1 beta | 1980-11-24 |

## mobile development kit ##

| Version  | Release date |
| --- | ----|
| M 2 | 1980-12-24 |
| M 1.2 | 1980-11-25 |
| M 1 | 1980-11-24 |

produz este resultado:

kit de desenvolvimento de desktop #

Versão Data de lançamento
1,0 1980-12-24
1 RC 1980-11-25
1 beta 1980-11-24

kit de desenvolvimento móvel #

Versão Data de lançamento
M 2 1980-12-24
M 1.2 1980-11-25
M 1 1980-11-24

Versões de um tópico #

Um tópico pode se aplicar apenas a determinadas versões. Ao criar um novo tópico, as versões apropriadas podem ser escolhidas na seção de título do tópico.

No exemplo a seguir .NET 2.0, 4.0, 4.5.1; Compact Framework 3.7 e Micro Framework 4.2 são selecionados.

Esta seleção aparecerá ao lado do título do tópico na lista de todos os tópicos

Versões de um exemplo (tabela de versão única) #

Um exemplo não pode ser associado diretamente a um número de versão.

No entanto, partes dele podem ser declaradas como aplicáveis ​​apenas a determinadas versões.

<!– se versão [eq Java SE 1.3] –>

Este conteúdo é para Java SE 1.3

<!– versão final if –>

Versões de um exemplo (tabelas de várias versões) #

Se houver muitas tabelas de versão, o título da desejada deve ser incluído entre colchetes angulares.

<!– if version <Micro Framework> [eq 4.4] –>

Este conteúdo é para Micro Framework 4.4

<!– versão final if –>

Condicionais disponíveis #

A ajuda do markdown afirma

As condicionais disponíveis são gt, gte, lt, lte, eq e neq.

<!– if versão [eq C++03] –>

eq igual a

<!– versão final if –>

<!– if versão [neq C++03] –>

neq não é igual a

<!– versão final if –>

<!– if versão [gt C++03] –>

gt maior que

<!– versão final if –>

<!– if versão [gte C++03] –>

gte maior que igual

<!– versão final if –>

<!– if versão [lt C++03] –>

lt menor que

<!– versão final if –>

<!– if versão [lte C++03] –>

lte menor que igual

<!– versão final if –>

Condicionais Múltiplos #

Várias condições podem ser usadas.

<!– if versão [lte 1.2.0] [gt 1.3.12] [gt 1.4.9] [neq 1.2.0] [eq 1.5.7] –>

conteúdo
