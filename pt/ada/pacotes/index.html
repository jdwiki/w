Mais sobre Pacotes #

No Hello World, você foi apresentado ao pacote Ada.Text_IO, e como usá-lo para realizar operações de E/S em seu programa. Os pacotes podem ser manipulados para fazer muitas coisas diferentes.

Renomeando: Para renomear um pacote, você usa a palavra-chave renames em uma declaração de pacote, assim:

package IO renames Ada.Text_IO;

Agora, com o novo nome, você pode usar a mesma notação pontilhada para funções como Put_Line (ou seja, IO.Put_Line), ou você pode simplesmente usar com use IO. Claro, dizer use IO ou IO.Put_Line usará as funções do pacote Ada.Text_IO.

Visibilidade e Isolamento: No exemplo Hello World incluímos o pacote Ada.Text_IO usando uma cláusula with. Mas também declaramos que queríamos usar Ada.Text_IO na mesma linha. A declaração use Ada.Text_IO poderia ter sido movida para a parte declarativa do procedimento:

with Ada.Text_IO;
 
procedure hello_world is
   use Ada.Text_IO;
begin
   Put_Line ("Hello, world!");
end hello_world;

Nesta versão, os procedimentos, funções e tipos de Ada.Text_IO estão disponíveis diretamente dentro do procedimento. Fora do bloco em que o uso Ada.Text_IO é declarado, teríamos que usar a notação pontilhada para invocar, por exemplo:

with Ada.Text_IO;
 
procedure hello_world is
begin
   Ada.Text_IO.Put ("Hello, ");       --  The Put function is not directly visible here
   declare
      use Ada.Text_IO;
   begin
      Put_Line ("world!");            --  But here Put_Line is, so no Ada.Text_IO. is needed
   end;
end hello_world;

Isso nos permite isolar as declarações de uso … onde elas são necessárias.

Relacionamento Pai-Filho #

Como forma de subdividir os programas Ada, os pacotes podem ter os chamados filhos. Estes podem ser pacotes, também. Um pacote filho tem um privilégio especial: ele pode ver as declarações na parte privada do pacote pai. Um uso típico dessa visibilidade especial é ao formar uma hierarquia de tipos derivados na programação orientada a objetos.

package Orders is
   type Fruit is (Banana, Orange, Pear);
   type Money is delta 0.01 digits 6;
   
   type Bill is tagged private;

   procedure Add
     (Slip   : in out Bill;
      Kind   : in     Fruit;
      Amount : in     Natural);
   
   function How_Much (Slip : Bill) return Money;
   
   procedure Pay
     (Ordered : in out Bill;
      Giving  : in     Money);
   
private
   type Bill is tagged record
      --  ...
      Sum : Money := 0.0;
   end record;
end Orders;

Qualquer unidade Ada encabeçada por with Orders; pode declarar objetos do tipo Bill e então chamar as operações Add, How_Much e Pay. No entanto, ele não vê os componentes de Bill, nem mesmo de Orders.Bill, pois a definição completa do tipo está oculta na parte private de Orders. A definição completa não está oculta em pacotes filhos. Essa visibilidade facilita a extensão do tipo, se necessário. Se um tipo for declarado no pacote filho como derivado de Bill, esse tipo de herança pode manipular os componentes de Bill diretamente.

package Orders.From_Home is
   type Address is new String (1 .. 120);
   
   type Ordered_By_Phone is new Bill with private;
   
   procedure Deliver
     (Ordered : in out Ordered_By_Phone;
      Place   : in     Address);

private
   type Ordered_By_Phone is new Bill with
      record
         Delivered : Boolean := False;
         To        : Address;
      end record;
end Orders.From_Home;

Orders.From_Home é um pacote filho de Orders. O tipo Ordered_By_Phone é derivado de Bill e inclui seu componente de registro Sum.
