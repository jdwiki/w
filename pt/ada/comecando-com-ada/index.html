Versão #

A linguagem de programação padrão Ada é definida no Ada Reference Manual. Alterações de versão provisórias e notas de lançamento são discutidas nos documentos de justificativa correspondentes. As implementações normalmente documentam sua conformidade com o padrão na forma de um guia do usuário e/ou manual de referência, por exemplo.

Instalação ou Configuração #

Ada é uma linguagem de programação para a qual existem vários compiladores.

  • Um desses compiladores, e talvez o mais utilizado, é o GNAT. Faz parte da cadeia de ferramentas GCC. Ele pode ser instalado de várias fontes:

  • O lançamento anual da GPL feito pela AdaCore, disponível gratuitamente no libre site. Esta versão passou por todos os testes internos que a AdaCore faz para seus lançamentos profissionais, está disponível em um grande número de plataformas. O compilador e seu tempo de execução são liberados sob a licença GPL e, a menos que você não esteja usando nenhum tempo de execução, quaisquer executáveis ​​que você distribuir também serão cobertos por esta licença. Para acadêmicos e projetos em seus estágios iniciais, isso não é um problema.

  • O FSF gcc recebe os mesmos patches regularmente. A versão do GNAT pode não estar sempre atualizada, mas é atualizada regularmente.

  • Vários contribuidores estão empacotando essa versão FSF para várias distribuições Linux (sistemas baseados em Debian, entre outros) e binários para Mac OS X. Usar o gerenciador de pacotes de sua distribuição pode ser a maneira mais simples de instalar o GNAT . Essas versões vêm com a licença GCC padrão e permitem que você escreva código-fonte fechado.

  • AdaCore também fornece GNAT Pro, que vem com a licença GCC padrão que permite escrever código fonte fechado. Mais importante, talvez, ele vem com suporte, caso você tenha dúvidas sobre o uso da linguagem, ferramentas, como melhor implementar algo e, claro, relatórios de bugs e solicitações de aprimoramento.

Outros número de compiladores estão listados no Ada WikiBook, junto com as instruções de instalação. Getadanow.com apresenta edições do FSF GNAT, prontas para vários sistemas operacionais em vários tipos de hardware ou máquinas virtuais. O site também coleta recursos para aprender e compartilhar Ada.

Olá Mundo #

with Ada.Text_IO;

procedure Hello_World is
begin
   Ada.Text_IO.Put_Line ("Hello World");
end Hello_World;

Alternativamente, após importar o pacote Ada.Text_IO, você pode dizer use Ada.Text_IO; para poder usar Put_Line sem declarar explicitamente de qual pacote ele deve vir, como tal :

with Ada.Text_IO; use Ada.Text_IO;

procedure Hello_World is
begin
    Put_Line ("Hello World");
end Hello_World;

Se você estiver usando o compilador gnat, este programa simples pode ser compilado com

gnatmake hello_world

Isso irá gerar vários arquivos, incluindo um hello_world (ou hello_world.exe no Windows) que você pode executar para ver a famosa mensagem. O nome do executável é calculado automaticamente a partir do nome do subprograma Ada principal. Em Ada um subprograma principal pode ter qualquer nome. Só tem que ser um procedimento sem parâmetros, que você dá como argumento para gnatmake.

Outros compiladores têm requisitos semelhantes, embora, é claro, o comando de compilação seja diferente.

Bibliotecas #

Como para qualquer linguagem de programação, Ada vem com extensas bibliotecas para realizar várias tarefas. Aqui estão algumas dicas para alguns deles, embora pesquisar no github leve a mais.

  • O próprio runtime do Ada, distribuído por todos os compiladores, inclui um extenso conjunto de pacotes e anexos, que vão desde estruturas de dados e contêineres, até entrada/saída, manipulação de strings, manipulação de tempo, arquivos, cálculos numéricos, multitarefa, opções de linha de comando , Números aleatórios,…

  • O compilador GNAT vem com seu próprio runtime estendido, com novos pacotes na hierarquia GNAT, que fornecem suporte para expressões regulares, ordenação, busca, unicode, CRC, tempo de entrada/saída, …

  • gnatcoll é uma biblioteca que está disponível no libre site da AdaCore e inclui uma extensa estrutura de log, estendendo aplicativos com python, mmap, uma extensa estrutura para interface com sistemas de arquivos, análise de mensagens de e-mail e caixas de correio, uma estrutura extensa para interagir com bancos de dados de maneira segura, interface para várias bibliotecas como ícone, linha de leitura, cores do terminal, suporte para tipos contados de referência para gerenciamento automático de memória, arquivos JSON,…

  • XML/Ada é uma biblioteca para analisar e validar documentos XML

  • GtkAda é uma ligação completa para a biblioteca gtk+, que permite escrever interfaces de usuário portáteis em Unix, Windows e OSX.

  • AWS é um framework para criação de servidores web em Ada, com suporte total para vários protocolos como HTTP, Websockets,… e seu próprio sistema de templates.
