Sintaxe #

  • Sintaxe padrão:

  • INSERT OVERWRITE TABLE tablename1 [PARTITION (partcol1=val1, partcol2=val2 …) [IF NOT EXISTS]] select_statement1 FROM from_statement;

  • INSERT INTO TABLE tablename1 [PARTITION (partcol1=val1, partcol2=val2 …)] select_statement1 FROM from_statement;

  • INSERT INTO TABLE tablename1 [PARTITION (partcol1=val1, partcol2=val2 …)] (z,y) select_statement1 FROM from_statement;

  • Extensão Hive (várias inserções):

  • FROM from_statement INSERT OVERWRITE TABLE tablename1 [PARTITION (partcol1=val1, partcol2=val2 …) [IF NOT EXISTS]] select_statement1 [INSERT OVERWRITE TABLE tablename2 [PARTITION … [IF NOT EXISTS]] select_statement2] [INSERT INTO TABLE tablename2 [PARTITION …] select_statement2] …;

  • FROM from_statement INSERT INTO TABLE tablename1 [PARTITION (partcol1=val1, partcol2=val2 …)] select_statement1 [INSERT INTO TABLE tablename2 [PARTITION …] select_statement2] [INSERT OVERWRITE TABLE tablename2 [PARTITION … [IF NOT EXISTS]] select_statement2] …;

  • Extensão Hive (inserções de partição dinâmica):

  • INSERT OVERWRITE TABLE tablename PARTITION (partcol1[=val1], partcol2[=val2] …) select_statement FROM from_statement;

  • INSERT INTO TABLE tablename PARTITION (partcol1[=val1], partcol2[=val2] …) select_statement FROM from_statement;

inserir substituição Uma instrução insert overwrite exclui todos os arquivos existentes na tabela ou partição de destino antes de adicionar novos arquivos com base na instrução select usada. Observe que quando há alterações na estrutura de uma tabela ou da DML usada para carregar a tabela que às vezes os arquivos antigos não são excluídos. Ao carregar em uma tabela usando particionamento dinâmico, somente as partições definidas pela instrução select serão sobrescritas. Quaisquer partições preexistentes no destino permanecerão e não serão excluídas.

insira dentro de Uma instrução insert into anexa novos dados em uma tabela de destino com base na instrução select usada.

inserir substituição #

insert overwrite table yourTargetTable select * from yourSourceTable;

Inserir na tabela #

INSERT INTO irá anexar à tabela ou partição, mantendo os dados existentes intactos.

    INSERT INTO table yourTargetTable SELECT * FROM yourSourceTable;

Se uma tabela for particionada, podemos inserir nessa partição específica de maneira estática, conforme mostrado abaixo.

    INSERT INTO TABLE yourTargetTable PARTITION (state=CA, city=LIVERMORE) 
    select * FROM yourSourceTable;

Se uma tabela for particionada, podemos inserir nessa partição específica de maneira dinâmica, conforme mostrado abaixo. Para realizar inserções de partições dinâmicas, devemos definir as propriedades abaixo.

    Dynamic Partition inserts are disabled by default. These are the relevant configuration properties for dynamic partition inserts:
    SET hive.exec.dynamic.partition=true;
    SET hive.exec.dynamic.partition.mode=non-strict

    INSERT INTO TABLE yourTargetTable PARTITION (state=CA, city=LIVERMORE) (date,time)
    select * FROM yourSourceTable;

Várias inserções em uma tabela.

Extensão de colmeia (várias inserções):

    FROM table_name

    INSERT OVERWRITE TABLE table_one SELECT table_name.column_one,table_name.column_two

    INSERT OVERWRITE TABLE table_two SELECT table_name.column_two WHERE table_name.column_one == 'something'
