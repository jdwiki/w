Sintaxe #

  • CRIAR [TEMPORARY] [EXTERNAL] TABLE [IF NOT EXISTS] [db_name.]table_name

    [(col_name data_type [COMMENT col_comment], ...)]

[COMMENT table_comment] [PARTICIONADO POR (nome_coluna data_type [COMMENT coluna_comentário], …)] [CLUSTERED BY (nome_coluna, nome_coluna, …) [SORTED BY (nome_coluna [ASC|DESC], …)] INTO num_buckets BUCKETS] [SKEWED BY (nome_coluna, nome_coluna, …) – (Observação: disponível no Hive 0.10.0 e posterior)] ON ((col_value, col_value, …), (col_value, col_value, …), …) [STORED AS DIRECTORIES] [ [FORMATO DE LINHA formato_linha] [ARMAZENADO COMO file_format] | STORED BY ‘storage.handler.class.name’ [WITH SERDEPROPERTIES (…)] ] [LOCATION hdfs_path] [TBLPROPERTIES (nome_propriedade=valor_propriedade, …)] [AS select_statement];

  • CRIAR [TEMPORARY] [EXTERNAL] TABLE [IF NOT EXISTS] [db_name.]table_name LIKE existing_table_or_view_name [LOCALIZAÇÃO hdfs_path];
  • data_type: tipo_primitivo,tipo_array,tipo_mapa,tipo_estrutura,tipo_união
  • tipo_primitivo: TINYINT, SMALLINT, INT , BIGINT, BOOLEAN, FLOAT, DOUBLE, STRING, BINARY, TIMESTAMP, DECIMAL, DECIMAL(precisão, escala), DATE, VARCHAR, CHAR
  • array_type: ARRAY < data_type >
  • map_type: MAP < primitivo_type, data_type >
  • struct_type: STRUCT < col_name : data_type [COMMENT col_comment], …>
  • union_type: UNIONTYPE < data_type, data_type, … >
  • formato_linha: DELIMITADO [CAMPOS TERMINADOS POR char [ESCAPED BY char]] [ITENS DE COLEÇÃO TERMINADOS POR char] [MAP KEYS TERMINATED BY char] [LINES TERMINATED BY char] [NULL DEFINED AS char]
    , SERDE serde_name [COM SERDEPROPERTIES (nome_propriedade=valor_propriedade, nome_propriedade=valor_propriedade, …)]
  • formato de arquivo: : SEQUENCEFILE , TEXTFILE , RCFILE , ORC , PARQUET , AVRO , INPUTFORMAT input_format_classname OUTPUTFORMAT output_format_classname
  • CREATE (DATABASE|SCHEMA) [IF NOT EXISTS] database_name [COMMENT database_comment] [LOCATION hdfs_path] [COM DBPROPERTIES (nome_propriedade=valor_propriedade, …)];

Ao trabalhar com tabelas e bancos de dados em HIVE. Os pontos abaixo podem ser úteis.

  • Podemos alternar banco de dados usando use database; command
  • Para conhecer o banco de dados de trabalho atual, podemos usar SELECT current_database()
  • Para ver o DDL usado para a instrução create table, podemos usar SHOW CREATE TABLE tablename
  • Para ver todas as colunas da tabela use DESCRIBE tablename para mostrar detalhes estendidos como localização serde usada e outros DESCRIBE FORMATTED tablename. DESCRIBE também pode ser abreviado como DESC.

Criar a tabela #

Criando uma tabela gerenciada com partição e armazenada como um arquivo de sequência. Supõe-se que o formato de dados nos arquivos seja delimitado por campo por Ctrl-A (^A) e delimitado por linha por nova linha. A tabela abaixo é criada no diretório de armazenamento do hive especificado no valor para a chave hive.metastore.warehouse.dir no arquivo de configuração do Hive hive-site.xml.

CREATE TABLE view
(time INT, 
id BIGINT,
url STRING, 
referrer_url STRING,
add STRING COMMENT 'IP of the User')
COMMENT 'This is view table'
PARTITIONED BY(date STRING, region STRING)
ROW FORMAT DELIMITED
FIELDS TERMINATED BY '\001'
STORED AS SEQUENCEFILE;

Criando uma tabela externa com partições e armazenada como um arquivo de sequência. Supõe-se que o formato de dados nos arquivos seja delimitado por campo por ctrl-A e delimitado por linha por nova linha. A tabela abaixo é criada no local especificado e é útil quando já temos dados. Uma das vantagens de usar uma tabela externa é que podemos descartar a tabela sem excluir os dados. Por exemplo, se criarmos uma tabela e percebermos que o esquema está errado, podemos descartar a tabela com segurança e recriá-la com o novo esquema sem nos preocupar com os dados. Outra vantagem é que, se estivermos usando outras ferramentas como pig nos mesmos arquivos, podemos continuar usando-os mesmo depois de excluirmos a tabela.

CREATE EXTERNAL TABLE view
(time INT, 
id BIGINT,
url STRING, 
referrer_url STRING,
add STRING COMMENT 'IP of the User')
COMMENT 'This is view table'
PARTITIONED BY(date STRING, region STRING)
ROW FORMAT DELIMITED
FIELDS TERMINATED BY '\001'
STORED AS SEQUENCEFILE
LOCATION '<hdfs_location>';

Criando uma tabela usando a consulta de seleção e preenchendo os resultados da consulta, these statements are known as CTAS(Create Table As Select).

Existem duas partes no CTAS, a parte SELECT pode ser qualquer instrução SELECT suportada pelo HiveQL. A parte CREATE do CTAS pega o esquema resultante da parte SELECT e cria a tabela de destino com outras propriedades da tabela, como SerDe e formato de armazenamento.

O CTAS tem estas restrições:

  • A tabela de destino não pode ser uma tabela particionada.

  • A tabela de destino não pode ser uma tabela externa.

  • A tabela de destino não pode ser uma tabela de agrupamento de lista.

    CREATE TABLE new_key_value_store ROW FORMAT SERDE “org.apache.hadoop.hive.serde2.columnar.ColumnarSerDe” STORED AS RCFile AS SELECT * FROM page_view SORT BY url, add;

Criar tabela como:

A forma LIKE de CREATE TABLE permite copiar exatamente uma definição de tabela existente (sem copiar seus dados). Ao contrário do CTAS, a instrução abaixo cria uma nova tabela cuja definição corresponde exatamente à tabela existente em todos os detalhes, exceto no nome da tabela. A nova tabela não contém linhas.

CREATE TABLE empty_page_views
LIKE page_views;

Criar banco de dados #

Criando um banco de dados em um local específico. Se não especificarmos nenhum local para o banco de dados, ele será criado no diretório do warehouse.

CREATE DATABASE IF NOT EXISTS db_name 
COMMENT 'TEST DATABASE'
LOCATION /PATH/HDFS/DATABASE/;

Integração HIVE_HBASE #

A integração Hive-Hbase é suportada desde as versões abaixo. Colmeia: 0,11,0 HBase: 0,94,2 Hadoop: 0,20,2

CREATE TABLE hbase_hive
(id string,
 col1 string,
 col2 string,
 col3 int) 
STORED BY 'org.apache.hadoop.hive.hbase.HBaseStorageHandler'
WITH SERDEPROPERTIES 
("hbase.columns.mapping" = ":key,cf1:col1,cf1:col2,cf1:col3")
TBLPROPERTIES ("hbase.table.name" = "hive_hbase");

Nota: a 1ª coluna deve ser a coluna chave.

Crie uma tabela usando as propriedades da tabela existente. #

CREATE TABLE new_table_name LIKE existing_table_name;

Criação da tabela ACID do Hive. #

As tabelas ACID são suportadas desde a versão 0.14 do hive. A tabela abaixo suporta UPDATE/DELETE/INSERT

Abaixo as alterações de configuração necessárias no hive-site.xml.

 hive.support.concurrency = true
 hive.enforce.bucketing = true
 hive.exec.dynamic.partition.mode = nonstrict
 hive.txn.manager =org.apache.hadoop.hive.ql.lockmgr.DbTxnManager
 hive.compactor.initiator.on = true
 hive.compactor.worker.threads = 1

Atualmente, apenas o formato orc é suportado.

Instrução de criação de tabela.

 create table Sample_Table(
 col1 Int,
 col2 String,
 col3 String) 
 clustered by (col3) into 3 buckets 
 stored as orc 
 TBLPROPERTIES ('transactional'='true');
