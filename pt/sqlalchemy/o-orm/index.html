O SQLAlchemy ORM é construído em cima do SQLAlchemy Core. Por exemplo, embora as classes de modelo usem objetos Column, eles fazem parte do núcleo e a documentação mais relevante será encontrada lá.

As partes principais do ORM são a sessão, a consulta e as classes mapeadas (normalmente usando a extensão declarativa no SQLAlchemy moderno).

Convertendo um resultado de consulta para dict #

Primeiro a configuração para o exemplo:

import datetime as dt
from sqlalchemy import Column, Date, Integer, Text, create_engine, inspect
from sqlalchemy.orm import sessionmaker
from sqlalchemy.ext.declarative import declarative_base

Base = declarative_base()
Session = sessionmaker()

class User(Base):
    __tablename__ = 'users'

    id = Column(Integer, primary_key=True)
    name = Column(Text, nullable=False)
    birthday = Column(Date)

engine = create_engine('sqlite://')
Base.metadata.create_all(bind=engine)
Session.configure(bind=engine)

session = Session()
session.add(User(name='Alice', birthday=dt.date(1990, 1, 1)))
session.commit()

Se você estiver consultando colunas individualmente, a linha é uma KeyedTuple que possui um método _asdict. O nome do método começa com um único sublinhado, para corresponder à API namedtuple (não é privada!).

query = session.query(User.name, User.birthday)
for row in query:
    print(row._asdict())

Ao usar o ORM para recuperar objetos, isso não está disponível por padrão. O SQLAlchemy sistema de inspeção deve ser usado.

def object_as_dict(obj):
    return {c.key: getattr(obj, c.key)
            for c in inspect(obj).mapper.column_attrs}

query = session.query(User)
for user in query:
    print(object_as_dict(user))

Aqui, criamos uma função para fazer a conversão, mas uma opção seria adicionar um método à classe base.

Em vez de usar declarative_base como acima, você pode criá-lo a partir de sua própria classe:

from sqlalchemy.ext.declarative import as_declarative

@as_declarative()
class Base:
    def _asdict(self):
        return {c.key: getattr(self, c.key)
                for c in inspect(self).mapper.column_attrs}

Filtragem #

Dado o seguinte modelo

class User(Base):
    __tablename__ = 'users'

    id = Column(Integer, primary_key=True)
    name = Column(Text, nullable=False)
    birthday = Column(Date)

Você pode filtrar colunas na consulta:

import datetime as dt
session.query(User).filter(User.name == 'Bob')
session.query(User).filter(User.birthday < dt.date(2000, 1, 1))

Para o primeiro caso, existe um atalho:

session.query(User).filter_by(name='Bob')

Os filtros podem ser compostos usando uma relação AND encadeando o método filter:

(session.query(User).filter(User.name.like('B%'))
                    .filter(User.birthday < dt.date(2000, 1, 1)))

Ou de forma mais flexível, usando os operadores bit a bit sobrecarregados & e |:

session.query(User).filter((User.name == 'Bob') | (User.name == 'George'))

Não se esqueça dos parênteses internos para lidar com a precedência do operador.

Ordenar por #

Dado um modelo básico:

class SpreadsheetCells(Base):
    __tablename__ = 'spreadsheet_cells'

    id = Column(Integer, primary_key=True)
    y_index = Column(Integer)
    x_index = Column(Integer)

Você pode recuperar uma lista ordenada encadeando o método order_by.

query = session.query(SpreadsheetCells).order_by(SpreadsheetCells.y_index)

Isso pode ser encadeado após um filtro,

query = session.query(...).filter(...).order_by(...)

ou para compor ainda mais uma consulta existente.

query = session.query(...).filter(...)
ordered_query = query.order_by(...)

Você também pode determinar a direção de classificação de duas maneiras:

  1. Acessando as propriedades do campo asc e dsc:

    query.order_by(SpreadsheetCells.y_index.desc()) # desc query.order_by(SpreadsheetCells.y_index.asc()) # asc

  2. Usando as funções do módulo asc e desc:

    from sqlalchemy import asc, desc

    query.order_by(desc(SpreadsheetCells.y_index)) # desc query.order_by(asc(SpreadsheetCells.y_index)) # asc

Acessando os resultados da consulta #

Depois de ter uma consulta, você pode fazer mais com ela do que apenas iterar os resultados em um loop for.

Configurar:

from datetime import date

class User(Base):
    __tablename__ = 'users'

    id = Column(Integer, primary_key=True)
    name = Column(Text, nullable=False)
    birthday = Column(Date)

# Find users who are older than a cutoff.
query = session.query(User).filter(User.birthday < date(1995, 3, 3))

Para retornar os resultados como uma lista, use all():

reslist = query.all() # all results loaded in memory
nrows = len(reslist)

Você pode obter uma contagem usando count():

nrows = query.count()

Para obter apenas o primeiro resultado, use first(). Isso é mais útil em combinação com order_by().

oldest_user = query.order_by(User.birthday).first()

Para consultas que devem retornar apenas uma linha, use one():

bob = session.query(User).filter(User.name == 'Bob').one()

Isso gera uma exceção se a consulta retornar várias linhas ou se não retornar nenhuma. Se a linha ainda não existir, use one_or_none():

bob = session.query(User).filter(User.name == 'Bob').one_or_none()
if bob is None:
    create_bob()

Isso ainda gerará uma exceção se várias linhas tiverem o nome ‘Bob’.
