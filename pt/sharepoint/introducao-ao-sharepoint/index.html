Instalação do SharePoint 2016 para farm de servidor único #

Introdução #

O SharePoint 2016 é a versão 16 da família de produtos SharePoint. Foi lançado em 4 de maio de 2016. Este exemplo abrange a instalação do SharePoint 2016 usando a configuração de farm de servidor único. Essa configuração abrange os fundamentos da configuração de um farm do SharePoint sem a necessidade de vários servidores. Observe que os cenários cobertos por um farm de servidor único geralmente são limitados a cenários de desenvolvimento e produção muito pequenos.

Requisitos #

Antes de instalar o SharePoint, o ambiente básico deve ser configurado. O SharePoint armazena documentos, bem como metadados, logs, aplicativos personalizados, personalizações e muito mais. Certifique-se de ter espaço em disco e RAM suficientes disponíveis acima dos requisitos da linha de base.

  • 4 núcleos em processadores compatíveis de 64 bits
  • 12 - 24 GB de RAM (dependendo do teste ou implantação do prod)
  • Disco rígido de 80 GB para o sistema
  • Disco rígido de 100 GB como segundo disco
  • Servidor com Windows Server 2012 R2 de 64 bits ou visualização técnica “Limiar”
  • SQL Server 2014 ou SQL Server 2016
  • .NET Framework 4.5.2 ou .NET Framework 4.6
  • Computador ingressado no domínio e contas de serviço de farm delegadas

Todos os outros pré-requisitos podem ser instalados manualmente ou usando o instalador de pré-requisitos do SharePoint incluído na instalação do SharePoint.

Instalação #

  • Execute o instalador de pré-requisitos; ele pode solicitar a reinicialização do servidor antes de continuar
  • Execute Setup.exe da instalação do SharePoint
  • Digite a chave de licença
  • Aceite o contrato de licença
  • Selecione “Completo” na guia Tipo de servidor
  • A configuração deve ser concluída com sucesso
  • Na página completa, deixe a caixa de seleção marcada ao lado de Run Product Configuration Wizard e clique em Close

Configuração #

Se você continuar da etapa anterior, o Assistente de Configuração do Produto do SharePoint 2016 deve ser aberto automaticamente. se a caixa não aparecer ou você estiver executando a configuração posteriormente, abra o assistente de configuração acessando Iniciar -> Produtos do SharePoint 2016 -> Assistente de Configuração do Produto do SharePoint 2016.

  • Clique em próximo na página de boas-vindas
  • Uma caixa de diálogo modal irá aparecer dizendo que alguns serviços podem ser reiniciados durante a configuração; nada foi instalado ainda, então clique em sim
  • Adicione o servidor de banco de dados para o farm
    • Enter the name of the machine running SQL Server; in this case, it is the local machine
    • Enter the name of the Configuration database or keep the default name SharePoint_Config
    • Enter the username of the domain service user who will be accessing the database (in the form of DOMAIN\user) *Enter the password for the domain user
    • Click next when done
  • Digite a senha da fazenda; isso será usado ao ingressar em servidores adicionais no novo farm
  • Selecione a função Farm de Servidor Único
  • Configure o Central Admin Web App (de onde o SharePoint será gerenciado pelos administradores do farm), selecione o número da porta e selecione o tipo de federação de autenticação (NTLM ou Negotate (Kerberose))
  • Revise as configurações nas páginas finais e faça as alterações necessárias
  • Quando estiver pronto, execute a configuração que pode levar alguns minutos
  • Após a conclusão, você abrirá o assistente que permitirá abrir o site Central Admin
  • Em caso de falha, você pode investigar os logs na pasta %COMMONPROGRAMFILES%\Microsoft Shared\Web Server Extensions\16\LOG

Configuração do Farm #

Depois que o aplicativo Web central, o banco de dados de configuração e o administrador central estiverem configurados, você estará pronto para configurar o farm para uso para usuários ou desenvolvimento. Você pode marcar o local do site Central Admin ou acessá-lo por meio de um atalho no mesmo local do Assistente de configuração do produto.

  • Se você estiver iniciando a configuração posteriormente, clique em Quick Launch -> Configuration Wizards -> Farm Configuration Wizard
  • Se você estiver iniciando o Assistente a partir da etapa de instalação, clique em Iniciar o Assistente
  • Escolha se deseja fazer parte do programa de melhoria do cliente clicando em Sim ou Não
  • Na página de configuração do farm, selecione a conta de domínio que executará os serviços em segundo plano no farm
    • While this account may be the same as the database account, they may also be different for separation of roles and privileges
    • Enter the account as DOMAIN\user
  • Valide os serviços que deseja disponibilizar no farm na página Serviços
  • Crie o primeiro conjunto de sites no farm (esta etapa pode ser ignorada e executada posteriormente)
    • Enter the site collection’s title, description, web address (usually the first site is at the server root), and the template
    • Most things can be changed (title, description) can be changed easily, but others like the web URL may take much more work to change; the template can also not be easily rolled back, but SharePoint allows a large amount of customizations that allows you to take any base template and convert the style and layout of the site
  • Ao concluir a configuração, clique em concluir

O farm e o primeiro conjunto de sites agora estão configurados para uso.

Crie uma Web Part com a Estrutura do SharePoint #

dev.office.com/sharepoint é um ótimo lugar para começar a usar a Estrutura do SharePoint.

A Estrutura do SharePoint é uma abordagem moderna do lado do cliente para o Desenvolvimento do SharePoint inicialmente direcionada ao SharePoint Online no Office 365. As Web Parts criadas com a Estrutura do SharePoint são um novo tipo de Web Part e podem ser disponibilizadas para adição em páginas existentes do SharePoint e novas páginas do SharePoint.

Há um ótimo exemplo de hello world para esse processo hospedado em Crie sua primeira Web Part do lado do cliente do SharePoint (Hello World parte 1). Todos os exemplos em dev.office.com estão disponíveis para contribuições da comunidade através do github.

As etapas básicas do Hello World na Estrutura do SharePoint são:

  1. Gere o esqueleto do projeto com o Yeoman SharePoint Generator.

    yo @microsoft/SharePoint

  2. Edite o código gerado no editor de sua preferência. O suporte para Visual Studio Code é forte em todas as plataformas.

  3. Visualize a Web Part usando gulp e o SharePoint Workbench local

    gulp serve

  4. Visualize em seu ambiente do SharePoint Online

Vá para o seguinte URL: ‘https://your-sharepoint-site/_layouts/workbench.aspx’

Logs e logs do ULS do SharePoint #

O SharePoint Unified Logging Service (ULS) fornece recursos de suporte e depuração para operações e desenvolvedores. Entender como ler os logs é um primeiro passo importante para resolver problemas.

Ferramental #

A Microsoft fornece o ULS Viewer para ajudar a ler logs antigos e logs que estão sendo gravados enquanto o farm está em execução. Ele também pode filtrar e aplicar formatação aos logs para ajudar a reduzir um problema.

Identificador de Correlação #

Para isolar um problema, é útil observar apenas um ID de correlação específico. Cada ID de correlação está associado a uma solicitação ou ação de ponta a ponta do sistema (como um time jobber). Se houver um problema com a renderização de uma página da Web, localizar a solicitação nos logs do ULS e isolá-la para o ID de correlação específico remove todo o ruído dos outros logs, ajudando a identificar o problema.

Adicionando SPMonitoredScope ao meu código #

Uma maneira de adicionar log e algum monitoramento de desempenho é adicionar SPMonitoredScope ao seu código.

using (new SPMonitoredScope("Feature Monitor"))
{
    // My code here 
}

Este código registrará o início e o fim de suas solicitações, bem como alguns dados de desempenho. A criação de seu próprio monitor personalizado que implementa o ISPScopedPerformanceMonitor permite definir o nível de rastreamento ou o tempo máximo de execução para um conjunto de códigos.
