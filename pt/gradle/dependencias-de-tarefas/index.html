doÚltimo

Observe que, em uma definição de tarefa de maneira gradle 3.x mais idiomática: usando a notação explicit doLast{closure} em vez do operador “leftShift”(«) preferível.(leftShift foi preterido em um gradle 3.2 está programado para ser removido no gradle 5.0.)

task oldStyle << {
    println 'Deprecated style task'
 }

é equivalente a:

task newStyle {
    doLast {
    println 'Deprecated style task'
    }
 }

Adicionando dependências usando nomes de tarefas #

Podemos alterar a ordem de execução das tarefas com o método dependsOn.

task A << {
    println 'Hello from A'
}
task B(dependsOn: A) << {
    println "Hello from B"
}

Adicionando `dependsOn: causa:

  • tarefa B depende da tarefa A
  • Gradle para executar a tarefa A toda vez antes da execução da tarefa B.

E a saída é:

> gradle -q B
Hello from A
Hello from B

Adicionando dependências de outro projeto #

project('projectA') {
    task A(dependsOn: ':projectB:B') << {
        println 'Hello from A'
    }
}

project('projectB') {
    task B << {
        println 'Hello from B'
    }
}

Para se referir a uma tarefa em outro projeto, você prefixa o nome da tarefa com o caminho do projeto ao qual ela pertence :projectB:B.

E a saída é:

> gradle -q B
Hello from A
Hello from B

Adicionando dependência usando o objeto de tarefa #

task A << {
    println 'Hello from A'
}

task B << {
    println 'Hello from B'
}

B.dependsOn A

É uma maneira alternativa de definir a dependência em vez de usar o [nome da tarefa][1].

E a saída é a mesma:

> gradle -q B
Hello from A
Hello from B

[1]: https://www.wikiod.com/pt/gradle/dependencias-de-tarefas#Adicionando dependências usando nomes de tarefas

Adicionando várias dependências #

Você pode adicionar várias dependências.

task A << {
    println 'Hello from A'
}

task B << {
    println 'Hello from B'
}

task C << {
    println 'Hello from C'
}

task D << {
    println 'Hello from D'
}

Agora você pode definir um conjunto de dependências:

B.dependsOn A
C.dependsOn B
D.dependsOn C

A saída é:

> gradle -q D
Hello from A
Hello from B
Hello from C
Hello from D

Outro exemplo:

B.dependsOn A 
D.dependsOn B
D.dependsOn C

A saída é:

> gradle -q D
Hello from A
Hello from B
Hello from C
Hello from D

Várias dependências com o método DependsOn #

Você pode adicionar várias dependências.

task A << {
    println 'Hello from A'
}

task B(dependsOn: A) << {
    println 'Hello from B'
}

task C << {
    println 'Hello from C'
}

task D(dependsOn: ['B', 'C'] << {
    println 'Hello from D'
}

A saída é:

> gradle -q D
Hello from A
Hello from B
Hello from C
Hello from D
