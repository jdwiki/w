No padrão entidade-componente-sistema, um componente é um bloco de dados reutilizável e modular que conectamos a uma entidade para adicionar aparência, comportamento e/ou funcionalidade.

No A-Frame, os componentes modificam as entidades que são objetos 3D na cena. Misturamos e compomos componentes juntos para construir objetos complexos. Eles nos permitem encapsular três.js e código JavaScript em módulos que podemos usar declarativamente a partir do HTML. Os componentes são aproximadamente análogos ao CSS.

Métodos de manipulação do ciclo de vida de definição #

Com o esquema sendo a anatomia, os métodos do ciclo de vida são a fisiologia; o esquema define a forma dos dados, os métodos do manipulador do ciclo de vida * usam * os dados para modificar a entidade. Os manipuladores geralmente interagem com o API de entidade.

Visão geral dos métodos #

Método Descrição
iniciar Chamado uma vez quando o componente é inicializado. Usado para configurar o estado inicial e instanciar variáveis.
atualização Chamado quando o componente é inicializado e sempre que qualquer uma das propriedades do componente é atualizada (por exemplo, via setAttribute). Usado para modificar a entidade.
remover Chamado quando o componente é removido da entidade (por exemplo, via removeAttribute) ou quando a entidade é desvinculada da cena. Usado para desfazer todas as modificações anteriores na entidade.
marque Chamado em cada loop de renderização ou tick da cena. Usado para mudanças ou verificações contínuas.
jogar Chamado sempre que a cena ou entidade é reproduzida para adicionar qualquer plano de fundo ou comportamento dinâmico. Também chamado uma vez quando o componente é inicializado. Usado para iniciar ou retomar o comportamento.
pausa Chamado sempre que a cena ou entidade pausa para remover qualquer plano de fundo ou comportamento dinâmico. Também chamado quando o componente é removido da entidade ou quando a entidade é desvinculada da cena. Usado para pausar o comportamento.
updateSchema Chamado sempre que qualquer uma das propriedades do componente é atualizada. Pode ser usado para modificar dinamicamente o esquema.

Propriedades do protótipo do componente #

Dentro dos métodos, temos acesso ao protótipo do componente via this:

| Propriedade | Descrição | |—————–|—————————————— ————————————————– ————————————————– ————| | this.data | Propriedades do componente analisadas calculadas a partir dos valores padrão do esquema, mixins e atributos da entidade. | | this.el | Referência à [entidade][entidade] como um elemento HTML. | | this.el.sceneEl | Referência à [cena][cena] como um elemento HTML. | | this.id | Se o componente puder ter [várias instâncias][várias], o ID da instância individual do componente (por exemplo, foo de sound__foo). |

MÉTODOS #

.init()

.init() é chamado uma vez no início do ciclo de vida do componente. Uma entidade pode chamar o manipulador init do componente:

  • Quando o componente é definido estaticamente na entidade no arquivo HTML e a página é carregada.
  • Quando o componente é definido em uma entidade anexada via setAttribute.
  • Quando o componente é definido em uma entidade não anexada e a entidade é então anexado à cena via appendChild.

O manipulador init é frequentemente usado para:

  • Configure o estado inicial e as variáveis
  • Métodos de ligação
  • Anexar ouvintes de eventos

Por exemplo, o init de um componente de cursor definiria variáveis ​​de estado, vincularia métodos e adicione ouvintes de eventos:

AFRAME.registerComponent('cursor', {
  // ...
  init: function () {
    // Set up initial state and variables.
    this.intersection = null;
    // Bind methods.
    this.onIntersection = AFRAME.utils.bind(this.onIntersection, this);
    // Attach event listener.
    this.el.addEventListener('raycaster-intersection', this.onIntersection);
  }
  // ...
});

.update (oldData)

.update (oldData) é chamado sempre que as propriedades do componente mudam, inclusive no início do ciclo de vida do componente. Uma entidade pode chamar um manipulador update do componente:

  • Depois que init() é chamado, no início do ciclo de vida do componente.
  • Quando as propriedades do componente são atualizadas com .setAttribute.

O manipulador update é frequentemente usado para:

  • Faça a maior parte do trabalho fazendo modificações na entidade, usando this.data.
  • Modifique a entidade sempre que uma ou mais propriedades do componente forem alteradas.

Modificações granulares na entidade podem ser feitas [diffing][diff] a corrente dataset (this.data) com o dataset anterior antes da atualização (oldData).

O A-Frame chama .update() tanto no início do ciclo de vida de um componente quanto a cada tempo em que os dados de um componente são alterados (por exemplo, como resultado de setAttribute). o o manipulador de atualização geralmente usa this.data para modificar a entidade. O manipulador de atualização tem acesso ao estado anterior dos dados de um componente por meio de seu primeiro argumento. Podemos usar os dados anteriores de um componente para dizer exatamente qual propriedades alteradas para fazer atualizações granulares.

Por exemplo, a “atualização” do componente visible define a visibilidade de a entidade.

AFRAME.registerComponent('visible', {
  /**
   * this.el is the entity element.
   * this.el.object3D is the three.js object of the entity.
   * this.data is the component's property or properties.
   */
  update: function (oldData) {
    this.el.object3D.visible = this.data;
  }
  // ...
});

.remover ()

.remove() é chamado sempre que o componente é separado da entidade. Um entidade pode chamar o manipulador remove de um componente:

  • Quando o componente é removido da entidade via removeAttribute.
  • Quando a entidade é destacada da cena (por exemplo, removeChild).

O manipulador remove é frequentemente usado para:

  • Remova, desfaça ou limpe todas as modificações do componente na entidade.
  • Desanexar ouvintes de eventos.

Por exemplo, quando o [componente de luz][luz] é removido, o componente de luz removerá o objeto de luz que havia definido anteriormente na entidade, assim retirando-o da cena.

AFRAME.registerComponent('light', {
  // ...
  remove: function () {
    this.el.removeObject3D('light');
  }
  // ...
});

.tick (time, timeDelta)

.tick() é chamado em cada tick ou frame do loop de renderização da cena. A cena chamará o manipulador tick de um componente:

  • Em cada quadro do loop de renderização.
  • Na ordem de 60 a 120 vezes por segundo.
  • Se a entidade ou cena não estiver pausada (por exemplo, o Inspetor está aberto).
  • Se a entidade ainda estiver anexada à cena.

O manipulador tick é frequentemente usado para:

  • Modificar continuamente a entidade em cada quadro ou em um intervalo.
  • Enquete para condições.

O manipulador tick recebe o tempo de atividade global da cena em milissegundos (time) e a diferença de tempo em milissegundos desde o último quadro (timeDelta). Estes podem ser usados ​​para interpolação ou para executar apenas partes do manipulador tick em um intervalo definido.

Por exemplo, o componente de controles rastreados progredirá as animações do controlador, atualizar a posição e rotação do controlador e verifique se os botões são pressionados.

AFRAME.registerComponent('tracked-controls', {
  // ...
  tick: function (time, timeDelta) {
    this.updateMeshAnimation();
    this.updatePose();
    this.updateButtons();
  }
  // ...
});

.pausa ()

.pause() é chamado quando a entidade ou cena pausa. A entidade pode chamar um manipulador pause do componente:

  • Antes que o componente seja removido, antes que o manipulador remove seja chamado.
  • Quando a entidade é pausada com Entity.pause().
  • Quando a cena é pausada com Scene.pause () (por exemplo, o Inspector é aberto).

O manipulador pause é frequentemente usado para:

  • Remova os ouvintes de eventos.
  • Remova qualquer chance de comportamento dinâmico.

Por exemplo, o componente de som pausará o som e removerá um ouvinte de eventos que teria tocado um som em um evento:

AFRAME.registerComponent('sound', {
  // ...
  pause: function () {
    this.pauseSound();
    this.removeEventListener();
  }
  // ...
});

.reproduzir ()

.play() é chamado quando a entidade ou cena é retomada. A entidade pode chamar manipulador play de um componente:

  • Quando o componente é anexado pela primeira vez, depois que o manipulador update é chamado.
  • Quando a entidade foi pausada, mas depois retomada com Entity.play().
  • Quando a cena foi pausada, mas depois retomada com Scene.play ().

O manipulador play é frequentemente usado para:

  • Adicionar ouvintes de eventos.

Por exemplo, o componente de som reproduzirá o som e atualizará o ouvinte de eventos que tocaria um som em um evento:

AFRAME.registerComponent('sound', {
  // ...
  play: function () {
    if (this.data.autoplay) { this.playSound(); }
    this.updateEventListener();
  }
  // ...
});

.updateSchema (dados)

.updateSchema(), se definido, é chamado em cada atualização para verificar se o esquema precisa ser modificado dinamicamente.

O manipulador updateSchema é frequentemente usado para:

  • Atualizar ou estender dinamicamente o esquema, geralmente dependendo do valor de uma propriedade.

Por exemplo, o componente de geometria verifica se o componente “primitivo” propriedade alterada para determinar se o esquema deve ser atualizado para um tipo de geometria:

AFRAME.registerComponent('geometry', {
  // ...
  updateSchema: (newData) {
    if (newData.primitive !== this.data.primitive) {
      this.extendSchema(GEOMETRIES[newData.primitive].schema);
    }
  }
  // ...
});

MÉTODOS DE PROTÓTIPO DE COMPONENTE #

.flushToDOM()

Para economizar tempo de CPU na stringificação, o A-Frame atualizará apenas no modo de depuração a representação serializada do componente no DOM real. Chamar flushToDOM() irá serializar manualmente os dados do componente e atualizar o DOM:

document.querySelector('[geometry]').components.geometry.flushToDOM();

Registre um componente A-Frame personalizado #

AFRAME.registerComponent (nome, definição) #

Registre um componente A-Frame. Devemos registrar componentes antes de usá-los em qualquer lugar em . Significado de um arquivo HTML, os componentes devem vir em ordem antes de .

  • {string} nome - Nome do componente. A API pública do componente representada por meio de um nome de atributo HTML.
  • Definição de {Objeto} - Definição de componente. Contém métodos de manipulador de esquema e ciclo de vida.

Registrando o componente em foo em seu arquivo js, ​​por exemplo, foo-component.js #

AFRAME.registerComponent('foo', {
  schema: {},
  init: function () {},
  update: function () {},
  tick: function () {},
  remove: function () {},
  pause: function () {},
  play: function () {}
});

Uso do componente foo em sua cena #

<html>
  <head>
    <script src="aframe.min.js"></script>
    <script src="foo-component.js"></script>
  </head>
  <body>
    <a-scene>
      <a-entity foo></a-entity>
    </a-scene>
  </body>
</html>

Formulário HTML do componente #

Um componente contém um bucket de dados na forma de uma ou mais propriedades do componente. Os componentes usam esses dados para modificar entidades. Considere um componente do motor, podemos definir propriedades como potência ou cilindros.

Os atributos HTML representam os nomes dos componentes e o valor desses atributos representam os dados dos componentes.

Componente de propriedade única #

Se um componente é um componente de propriedade única, o que significa que seus dados consistem em um único valor, então em HTML, o valor do componente se parece com um atributo HTML normal:

<!-- `position` is the name of the position component. -->
<!-- `1 2 3` is the data of the position component. -->
<a-entity position="1 2 3"></a-entity>

Componente de várias propriedades #

Se um componente for um componente de várias propriedades, o que significa que os dados consistem em várias propriedades e valores, em HTML, o valor do componente se assemelha aos estilos CSS embutidos:

<!-- `light` is the name of the light component. -->
<!-- The `type` property of the light is set to `point`. -->
<!-- The `color` property of the light is set to `crimson`. -->
<a-entity light="type: point; color: crimson"></a-entity>

Definindo objeto de esquema componente #

O esquema é um objeto que define e descreve a propriedade ou propriedades do componente. As chaves do esquema são os nomes da propriedade e os valores do esquema definem os tipos e valores da propriedade (no caso de um componente de várias propriedades):

Definindo o esquema em seu componente

AFRAME.registerComponent('bar', {
  schema: {
    color: {default: '#FFF'},
    size: {type: 'int', default: 5}
  }
}

Substituir padrões de esquema definidos

<a-scene>
  <a-entity bar="color: red; size: 20"></a-entity>
</a-scene>

Esquema de propriedade única #

Um componente pode ser um componente de propriedade única (consistindo em um valor anônimo) ou um componente de várias propriedades (consistindo em vários valores). O A-Frame inferirá se um componente é de propriedade única vs. multi-propriedade com base na estrutura do esquema.

O esquema de um componente de propriedade única contém chaves type e/ou default e os valores do esquema são valores simples em vez de objetos:

AFRAME.registerComponent('foo', {
  schema: {type: 'int', default: 5}
});

<a-scene>
  <a-entity foo="20"></a-entity>
</a-scene>

Tipos de propriedade do esquema de componentes do A-Frame #

Os tipos de propriedade definem principalmente como o esquema analisa os dados de entrada do DOM para cada propriedade. Os dados analisados ​​estarão então disponíveis através do arquivo data propriedade no protótipo do componente. Abaixo estão as propriedades internas do A-Frame tipos:

Tipo de propriedade Descrição Valor padrão
matriz Analisa valores separados por vírgula para array (ou seja, "1, 2, 3" para ['1', '2', '3']). []
ativo Para URLs que apontam para recursos gerais. Pode analisar URL de uma string na forma de url(<url>). Se o valor for um seletor de ID de elemento (por exemplo, #texture), esse tipo de propriedade chamará getElementById e getAttribute('src') para retornar um URL. O tipo de propriedade asset pode ou não mudar para lidar com XHRs ou retornar MediaElements diretamente (por exemplo, elementos <img>). ''
áudio Mesma análise do tipo de propriedade asset. Possivelmente será usado pelo A-Frame Inspector para apresentar recursos de áudio. ''
booleano Analisa string para booleano (ou seja, "false" para false, todo o resto é verdadeiro). falso
cor Atualmente não faz nenhuma análise. Usado principalmente pelo A-Frame Inspector para apresentar um seletor de cores. Além disso, é necessário usar o tipo de cor para que as animações coloridas funcionem. #FFF
int Chama parseInt (por exemplo, "124.5" para 124). 0
mapa Mesma análise do tipo de propriedade asset. Possivelmente será usado pelo A-Frame Inspector para apresentar recursos de textura. ''
modelo Mesma análise do tipo de propriedade asset. Possivelmente será usado pelo A-Frame Inspector para apresentar os ativos do modelo. ''
número Chama parseFloat (por exemplo, '124.5' para '124.5'). 0
seletor Chama querySelector (por exemplo, "#box" para <a-entity id="box">). nulo
seletorTodos Chama querySelectorAll e converte NodeList para Array (por exemplo, ".boxes" para [<a-entity class=“boxes”, …]), nulo
cadeia Não faz nenhuma análise. ''
vec2 Analisa dois números em um objeto {x, y} (por exemplo, 1 -2 para {x: 1, y: -2}. {x: 0, y: 0}
vec3 Analisa três números em um objeto {x, y, z} (por exemplo, 1 -2 3 para {x: 1, y: -2, z: 3}. {x: 0, y: 0, z: 0}
vec4 Analisa quatro números em um objeto {x, y, z, w} (por exemplo, 1 -2 3 -4.5 para {x: 1, y: -2, z: 3, w: -4.5} . {x: 0, y: 0, z: 0, w: 0}

Inferência de tipo de propriedade #

O esquema tentará inferir um tipo de propriedade dado apenas um valor padrão:

schema: {default: 10}  // type: "number"
schema: {default: "foo"}  // type: "string"
schema: {default: [1, 2, 3]}  // type: "array"

O esquema definirá um valor padrão se não for fornecido, considerando o tipo de propriedade:

schema: {type: 'number'}  // default: 0
schema: {type: 'string'}  // default: ''
schema: {type: 'vec3'}  // default: {x: 0, y: 0, z: 0}

Tipo de propriedade personalizada #

Também podemos definir nosso próprio tipo de propriedade ou analisador fornecendo um parse função no lugar de um type:

schema: {
  // Parse slash-delimited string to an array 
  // (e.g., `foo="myProperty: a/b"` to `['a', 'b']`).
  myProperty: {
    default: [],
    parse: function (value) {
      return value.split('/');
    }
  }
}

Acessando os membros e métodos de um componente #

Os membros e métodos de um componente podem ser acessados ​​por meio da entidade a partir do objeto .components. Procure o componente no mapa de componentes da entidade e teremos acesso aos componentes internos do componente. Considere este componente de exemplo:

AFRAME.registerComponent('foo', {
  init: function () {
    this.bar = 'baz';
  },
  qux: function () {
    // ...
  }
});

Vamos acessar o membro bar e o método qux:

var fooComponent = document.querySelector('[foo]').components.foo;
console.log(fooComponent.bar);
fooComponent.qux();
