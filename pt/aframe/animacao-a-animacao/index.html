Animações e transições em A-Frame são definidas usando o elemento <a-animation> como filho. O sistema é baseado aproximadamente na especificação de Animações da Web. O A-Frame usa tween.js internamente.

Atributos #

Aqui está uma visão geral dos atributos de animação. Entraremos em mais detalhes abaixo.

| Atributo | Descrição | Valor padrão | |———–|————————————- ————————————————– ———————————-|—————- | | atributo | Atributo para animar. Para especificar um atributo de componente, use a sintaxe componentName.property (por exemplo, light.intensity). | rotação | | começar | Nome do evento para aguardar antes de iniciar a animação. | ’’ | | atraso | Atraso (em milissegundos) ou nome do evento para aguardar antes de iniciar a animação. | 0 | | direção | Direção da animação (entre from e to). Um de alternate, alternateReverse, normal, reverse. | normais | | durante | Duração em (milissegundos) da animação. | 1000 | | facilitando | Função de facilitação da animação. Há muitos para escolher. | facilidade | | fim | Nome do evento para aguardar antes de parar a animação. | ’’ | | preencher | Determina o efeito da animação quando não está ativamente em jogo. Um de backwards, both, forwards, none. | para a frente | | de | Valor inicial. | Valor atual. | | repetir | Repetir a contagem ou indefinido. | 0 | | para | Valor final. Deve ser especificado. | Nenhum |

Iniciar

O atributo begin define quando a animação deve começar a ser reproduzida.

Isso pode ser um número, representando milissegundos de espera, ou um evento nome para esperar. Por exemplo, podemos definir uma animação que espera 2 segundos antes de dimensionar uma entidade.

<a-entity>
  <a-animation attribute="scale" begin="2000" to="2 2 2"></a-animation>
</a-entity>

Ou podemos definir uma animação que espera que o elemento pai acione um evento chamado fade antes de desvanecer uma entidade.

<a-entity id="fading-cube" geometry="primitive: box" material="opacity: 1">
  <a-animation attribute="material.opacity" begin="fade" to="0"></a-animation>
</a-entity>

// Trigger an event to begin fading.
document.querySelector('#fading-cube').emit('fade');

Direção

O atributo direction define qual caminho animar entre o início valor e o valor final.

Quando definimos uma direção alternada, a animação vai e volta entre os valores from e to como um ioiô. Apenas direções alternadas têm efeito quando repetimos a animação.

Valor Descrição
alternativo Em ciclos pares, anime de from para to. Em ciclos ímpares, animação de to para from
alternativo-reverso Em ciclos ímpares, anime de from para to. Em ciclos pares, animação de to para from
normal Animar de de para para.
reverso Animar de to para from.

Facilitando

O atributo easing define a função de easing da animação, que o padrão é facilidade. Existem muitas funções de facilitação para listar, mas podemos explicá-los implicitamente.

Um valor possível é linear. E as funções básicas de easing são ease, ease-in, ease-out e ease-in-out.

Depois, há mais grupos de funções de atenuação. A flexibilização básica acima funções prefixam cada grupo de funções de atenuação. Os grupos de funções de facilitação são cubic, quad, quart, quint, sine, expo, circ, elastic, back e bounce.

Por exemplo, o grupo cúbico de funções de atenuação consistiria em facilidade-cúbica, facilidade-em-cúbica, facilidade-fora-cúbica, facilidade-in-fora-cúbica.

Preencher

O atributo fill define o efeito da animação quando não está em jogo. Pense em fill como os valores que a animação define na entidade antes e/ou depois de cada ciclo de animação. Abaixo estão os valores possíveis para fill e seus efeitos.

Valor Descrição
para trás Antes de a animação começar, defina o valor inicial para o valor from.
ambos Combine os efeitos de preenchimento para trás e preenchimento para frente.
avançar Depois que a animação terminar, o valor final permanecerá no valor to. O preenchimento padrão.
nenhum Antes de a animação começar, defina o valor inicial para o valor inicial. Depois que a animação terminar, redefina o valor para o valor inicial.

Repetir

O atributo repeat define com que frequência a animação se repete. Nós chamamos cada um repetir a animação um ciclo. A repetição pode ser um número em contagem regressiva em cada ciclo de animação até atingir ‘0’, ponto em que a animação end, ou podemos definir repeat para indefinite e a animação fará um loop continuamente até que a animação seja removida ou interrompida manualmente.

EVENTOS #

O elemento <a-animation> emite alguns eventos.

Nome do evento Descrição
animação final Emitido quando a animação termina. No caso de repetições, emitido quando a contagem de repetições atinge ‘0’. Não emitido para repetições indefinidas.
início da animação Emitido imediatamente quando a animação começa a ser reproduzida.

Animando diferentes tipos de propriedades #

O sistema de animação do A-Frame pode animar diferentes tipos de propriedades.

Propriedades de vec3 #

O A-Frame tem componentes padrão vec3 (ou seja, position, rotation, e escala). Esses componentes consistem em três fatores: X, Y e Z. Podemos passar três números delimitados por espaço para os atributos from e to assim como nós os definiria em uma entidade. Neste caso, o sistema de animação assumirá estamos animando um valor vec3.

Por exemplo, se quisermos animar uma entidade indo de um ponto a outro, pode animar o componente position.

<a-entity>
  <a-animation attribute="position" from="1 1 1" to="2 4 -8"></a-animation>
</a-entity>

Propriedades Booleanas #

A-Frame tem componentes padrão que aceitam um único valor booleano. boleano valores podem ser “animados” também virando o booleano no final de cada ciclo de animação.

Por exemplo, podemos definir uma animação que desativa a visibilidade de um entidade após 5 segundos.

<a-entity>
  <a-animation attribute="visible" dur="5000" to="false" repeat="indefinite"></a-animation>
</a-entity>

Propriedades Numéricas #

Também podemos animar atributos numéricos. Por exemplo, podemos animar o intensidade da luz primitiva.

<a-light intensity="1">
  <a-animation attribute="intensity" to="3"></a-animation>
</a-light>

Propriedades de cor #

Podemos animar qualquer propriedade de componente que tenha um tipo de cor. Por exemplo, nós pode animar uma caixa de branco para vermelho.

<a-entity id="blushing-cube" geometry="primitive: box">
  <a-animation attribute="material.color" from="white" to="red" dur="1000"></a-animation>
</a-entity>

Propriedades do componente #

Podemos animar uma certa propriedade de um componente de várias propriedades. Para isso, nós selecione a propriedade do componente usando a sintaxe de ponto: componentName.propertyName.

Por exemplo, para animar o raio superior de um cone, podemos selecionar o radiusTop valor com geometry.radiusTop.

<a-entity geometry="primitive: cone; radiusTop: 1">
  <a-animation attribute="geometry.radiusTop" to="0.5"></a-animation>
</a-entity>

Animações de exemplo #

Como exemplo introdutório, para definir uma órbita de 5 metros em uma entidade em torno do eixo Y que leva 10 segundos, podemos deslocar a posição e animar a rotação. Esta animação começa com a rotação inicial em torno do eixo Y de 0 graus e gira em torno de 360 ​​graus. É definido com uma duração de 10.000 milissegundos, mantém o valor final em cada ciclo da animação e faz um loop infinito.

<a-entity position="5 0 0" rotation="0 0 0">
  <a-animation attribute="rotation"
               to="0 360 0"
               dur="10000"
               fill="forwards"
               repeat="indefinite"></a-animation>
</a-entity>
