Os controladores são vitais para a imersão de pessoas em um aplicativo de RV. O potencial da VR não é alcançado sem eles, ou seja, controladores que fornecem seis graus de liberdade (6DoF). Com os controladores, as pessoas podem alcançar e contornar a cena e interagir com objetos com as mãos.

O A-Frame fornece componentes para controladores em todo o espectro, conforme suportado por seus respectivos navegadores WebVR por meio da API da Web do Gamepad. Existem componentes para controladores Vive, Oculus Touch, Daydream e GearVR.

É possível que você deva habilitar as extensões do gamepad. Você pode fazer isso usando estas etapas:

  • No Chrome: navegue até chrome://flags
  • No Firefox: navegue até about:config
  • No IE: Vá para o Editor de Diretiva de Grupo em sua área de trabalho
  • No Opera: navegue até opera:config
  • No Edge: navegue até about:flags

Controles manuais #

O A-Frame fornece uma implementação para suportar vários tipos de controladores 6DoF (Vive, Oculus Touch) por meio do componente de controles manuais. O componente de controles manuais é principalmente para controladores 6DoF, pois é voltado para interações em escala de sala, como pegar objetos. O componente de controles manuais funciona em cima dos controladores Vive e Oculus Touch:

  • Configurando o componente vive-controls e oculus-touch-controls

  • Substituindo os modelos do controlador com um modelo de mão simples

  • Mapeamento de eventos específicos do Vive e do Oculus Touch para eventos e gestos de mão (por exemplo, gripdown e triggerdown to thumbup)

Para adicionar o componente de controles manuais:

<a-entity hand-controls="left"></a-entity>
<a-entity hand-controls="right"></a-entity>

Infelizmente, ainda não existe um componente controlador 3DoF que abstraia bem todos os tipos de controladores 3DoF (ou seja, Daydream, GearVR). Poderíamos criar um controlador personalizado que funcionasse com ambos os controladores. Seria bastante fácil de cobrir, pois os controladores 3DoF não oferecem muito potencial de interação (ou seja, apenas rastreamento rotacional com um touchpad).

Os controles manuais fornecem mãos rastreadas (usando um modelo prescrito) com gestos animados. hand-controls envolve os componentes vive-controls e oculus-touch-controls, que por sua vez envolvem o componente tracked-controls. O componente fornece eventos extras e manipula animações e poses de mão.

Controles rastreados #

O componente de controles rastreados é o componente do controlador básico do A-Frame que fornece a base para todos os componentes do controlador do A-Frame. O componente de controles rastreados:

  • Pega um objeto Gamepad da API Gamepad dado um ID ou prefixo.

  • Aplica pose (posição e orientação) da API do Gamepad para ler o movimento do controlador.

  • Procura alterações nos valores dos botões do objeto Gamepad para fornecer eventos quando os botões são pressionados ou tocados e quando o eixo e os touchpads são alterados (ou seja, axischanged, buttonchanged, buttondown, buttonup, touchstart, touchend ).

Todos os componentes do controlador do A-Frame são construídos sobre o componente tracked-controls por:

  • Definir o componente de controles rastreados na entidade com o ID do Gamepad apropriado (por exemplo, Oculus Touch (Direita)). Por exemplo, o componente vive-controls não

    el.setAttribute('tracked-controls', {idPrefix: 'OpenVR'})

tracked-controls se conectará ao objeto Gamepad apropriado para fornecer pose e eventos para a entidade.

  • Abstrair os eventos fornecidos pelos controles rastreados. eventos de controles rastreados são de baixo nível; seria difícil para nós dizer quais botões foram pressionados apenas com base nesses eventos, porque teríamos que conhecer os mapeamentos dos botões de antemão. Os componentes do controlador podem conhecer os mapeamentos de antemão para seus respectivos controladores e fornecer mais eventos semânticos, como triggerdown ou xbuttonup.

  • Fornecer um modelo. tracked-controls sozinho não fornece nenhuma aparência. Os componentes do controlador podem fornecer um modelo que mostra feedback visual, gestos e animações quando os botões são pressionados ou tocados. Os componentes do controlador a seguir são ativados apenas se detectarem que o controlador foi encontrado e visto como conectado na API do Gamepad.

O componente tracked-controls faz interface com controladores rastreados. tracked-controls usa a API do Gamepad para lidar com controladores rastreados e é abstraída pelo componente hand-controls, bem como pelos componentes vive-controls e oculus-touch-controls. Este componente elege o controlador apropriado, aplica pose à entidade, observa o estado dos botões e emite os eventos apropriados.

Observe que, devido a alterações recentes específicas do navegador, os controladores Vive podem ser retornados pela API do Gamepad com valores de id de “OpenVR Gamepad” ou “OpenVR Controller”, portanto, é recomendável usar idPrefix para controladores Vive / OpenVR .

<a-entity tracked-controls="controller: 0; idPrefix: OpenVR"></a-entity>

Controladores 3Dof e 6Dof #

Adicionando controladores 3DoF #

Controladores com 3 graus de liberdade (3DoF) são limitados ao rastreamento rotacional. Os controladores 3DoF não têm rastreamento posicional, o que significa que não podemos estender a mão nem mover a mão para frente e para trás ou para cima e para baixo. Ter um controle com apenas 3DoF é como ter uma mão e um pulso sem braço. Leia mais sobre graus de liberdade para VR.

Os componentes do controlador 3DoF fornecem rastreamento rotacional, um modelo padrão correspondente ao hardware da vida real e eventos para abstrair os mapeamentos de botões. Os controladores para Google Daydream e Samsung GearVR têm 3DoF e ambos suportam apenas um controlador para uma mão.

Controladores do Daydream #

O componente daydream-controls faz interface com os controladores do Google Daydream. Ele envolve o componente de controles rastreados enquanto adiciona mapeamentos de botões, eventos e um modelo de controlador Daydream que destaca os botões tocados e/ou pressionados (trackpad).

Combine o controlador do Daydream, se estiver presente, independentemente da mão.

<a-entity daydream-controls></a-entity>

Combine o controlador do Daydream, se presente e para a mão especificada.

Adicionando controladores 6DoF #

Controladores com 6 graus de liberdade (6DoF) possuem rastreamento rotacional e posicional. Ao contrário dos controladores com 3DoF que são restritos à orientação, os controladores com 6DoF podem se mover livremente no espaço 3D. 6DoF nos permite chegar à frente, atrás das costas, mover as mãos pelo corpo ou perto do rosto. Ter 6DoF é como a realidade onde temos mãos e braços. 6DoF também se aplica ao fone de ouvido e rastreadores adicionais (por exemplo, pés, adereços). Ter 6DoF é o mínimo para fornecer uma experiência de VR verdadeiramente imersiva.

Os componentes do controlador 6DoF fornecem rastreamento completo, um modelo padrão correspondente ao hardware da vida real e eventos para abstrair os mapeamentos de botões. HTC Vive e Oculus Rift com Touch fornecem 6DoF e controladores para ambas as mãos. O HTC Vive também fornece rastreadores para rastrear objetos adicionais no mundo real em VR.

Controles Wasd #

O componente wasd-controls controla uma entidade com o W, A, S e D ou teclas de seta. O componente wasd-controls é comumente anexado a uma entidade com o componente câmera.

<a-entity camera look-controls wasd-controls></a-entity>

Para teclados azerty, você pode usar as teclas Z, Q, S e D

Controles de aparência #

O componente look-controls:

  • Gira a entidade quando giramos um monitor montado na cabeça VR (HMD).
  • Gira a entidade quando clicamos e arrastamos o mouse.
  • Gira a entidade quando tocamos e arrastamos a tela sensível ao toque.

O componente de controles de aparência geralmente é usado junto com o componente de câmera.

<a-entity camera look-controls></a-entity>

Ressalvas #

Se você quiser criar seu próprio componente para controles de aparência, terá que copiar e colar os bits de rastreamento HMD em seu componente. No futuro, poderemos ter um sistema para que as pessoas criem seus controles com mais facilidade.
