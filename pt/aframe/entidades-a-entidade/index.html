A-Frame representa uma entidade através do elemento <a-entity>. Conforme definido no padrão entidade-componente-sistema, as entidades são objetos de espaço reservado aos quais conectamos componentes para fornecer aparência, comportamento e funcionalidade.

Sintaxe #

  • <a-entity> // Considere a entidade abaixo. Por si só, não tem aparência, comportamento ou funcionalidade. Não faz nada:
  • <a-entity geometry="primitive: box" material="color: red"> // Podemos anexar componentes a ela para torná-la renderizada ou fazer alguma coisa. Para dar forma e aparência, podemos anexar os componentes de geometria e material:
  • <a-entity geometry="primitive: box" material="color: red" light="type: point; intensidade: 2.0"> // Ou para fazê-lo emitir luz, podemos anexar ainda mais o componente light:

Parâmetros #

Parâmetro Detalhes
componentes <a-entity>.components é um objeto de componentes anexados à entidade. Isso nos dá acesso aos componentes da entidade, incluindo os dados, estado e métodos de cada componente.
estáJogando Se a entidade está ativa e jogando. Se pausarmos a entidade, isPlaying se tornará false.
object3D <a-entity>.object3D é uma referência à representação three.js Object3D da entidade. Mais especificamente, object3D será um objeto THREE.Group que pode conter diferentes tipos de THREE.Object3Ds como câmeras, malhas, luzes ou sons:
object3DMap O object3DMap de uma entidade é um objeto que dá acesso aos diferentes tipos de THREE.Object3Ds (por exemplo, câmera, malhas, luzes, sons) que os componentes definiram.
cenaEl Uma entidade tem uma referência ao seu elemento de cena.

MÉTODOS #

addState (stateName)

addState enviará um estado para a entidade. Isso emitirá o evento stateadded, e podemos verificar a existência da lata de estado usando .is:

entity.addEventListener('stateadded', function (evt) {
  if (evt.detail.state === 'selected') {
    console.log('Entity now selected!');
  }
});
entity.addState('selected');
entity.is('selected');  // >> true

emitir (nome, detalhe, bolhas)

emit emite um evento DOM personalizado na entidade. Por exemplo, podemos emitir um evento para acionar uma animação:

<a-entity>
  <a-animation attribute="rotation" begin="rotate" to="0 360 0"></a-animation>
</a-entity>

entity.emit('rotate');

Também podemos passar detalhes ou dados do evento como o segundo argumento:

entity.emit('collide', { target: collidingEntity });

O evento será bolha por padrão. podemos dizer para não fazer bolhas passando false para bolha:

entity.emit('sink', null, false);

flushToDOM (recursivo)

flushToDOM serializará manualmente os dados dos componentes de uma entidade e atualizará o DOM.

getAttribute (componentName)

getAttribute recupera dados de componentes analisados ​​(incluindo mixins e padrões).

// <a-entity geometry="primitive: box; width: 3">
entity.getAttribute('geometry');
// >> {primitive: "box", depth: 2, height: 2, translate: "0 0 0", width: 3, ...}
entity.getAttribute('geometry').primitive;
// >> "box"
entity.getAttribute('geometry').height;
// >> 2
entity.getAttribute('position');
// >> {x: 0, y: 0, z: 0}

Se componentName não for o nome de um componente registrado, getAttribute se comportará normalmente:

// <a-entity data-position="0 1 1">
entity.getAttribute('data-position');
// >> "0 1 1"

getDOMAttribute (componentName)

getDOMAttribute recupera apenas dados de componentes analisados ​​que são definidos explicitamente no DOM ou por meio de setAttribute. Se componentName for o nome de um componente registrado, getDOMAttribute retornará apenas os dados do componente definidos no HTML como um objeto analisado. getDOMAttribute para componentes é a forma parcial de getAttribute, pois os dados de componente retornados não incluem mixins aplicados ou valores padrão:

Compare a saída do exemplo acima de getAttribute:

// <a-entity geometry="primitive: box; width: 3">
entity.getDOMAttribute('geometry');
// >> { primitive: "box", width: 3 }
entity.getDOMAttribute('geometry').primitive;
// >> "box"
entity.getDOMAttribute('geometry').height;
// >> undefined
entity.getDOMAttribute('position');
// >> undefined

getObject3D (tipo)

getObject3D procura um filho THREE.Object3D referenciado por type em object3DMap.

AFRAME.registerComponent('example-mesh', {
  init: function () {
     var el = this.el;
     el.getOrCreateObject3D('mesh', THREE.Mesh);
     el.getObject3D('mesh');  // Returns THREE.Mesh that was just created.
  }
});

getOrCreateObject3D (tipo, Construtor)

Se a entidade não tiver um THREE.Object3D registrado em type, getOrCreateObject3D registrará um THREE.Object3D instanciado usando o Constructor passado. Se a entidade tiver um THREE.Object3D registrado em type, getOrCreateObject3D atuará como getObject3D:

AFRAME.registerComponent('example-geometry', {
  update: function () {
    var mesh = this.el.getOrCreateObject3D('mesh', THREE.Mesh);
    mesh.geometry = new THREE.Geometry();
  }
});

pausa ()

pause() interromperá qualquer comportamento dinâmico conforme definido por animações e componentes. Quando pausamos uma entidade, ela interrompe suas animações e chama Component.pause() em cada um de seus componentes. Os componentes decidem implementar o que acontece na pausa, o que geralmente remove os ouvintes de eventos. Uma entidade chamará pause() em suas entidades filhas quando pausarmos uma entidade.

// <a-entity id="spinning-jumping-ball">
entity.pause();

Por exemplo, o componente look-controls em pausa removerá os manipuladores de eventos que escutam a entrada.

jogar ()

play() iniciará qualquer comportamento dinâmico conforme definido por animações e componentes. Isso é chamado automaticamente quando o DOM anexa uma entidade. Quando uma entidade play(), a entidade chama play() em suas entidades filhas.

entity.pause();
entity.play();

Por exemplo, o componente de som em execução começará a reproduzir o som.

setAttribute (componentName, value, [propertyValue | clobber])

Se componentName não for o nome de um componente registrado ou o componente for um componente de propriedade única, setAttribute se comportará normalmente:

entity.setAttribute('visible', false);

No entanto, se componentName for o nome de um componente registrado, ele poderá manipular a análise especial do valor. Por exemplo, o componente de posição é um componente de propriedade única, mas seu analisador de tipo de propriedade permite que ele pegue um objeto:

entity.setAttribute('position', { x: 1, y: 2, z: 3 });

setObject3D (tipo, obj)

setObject3D registrará o obj passado, um THREE.Object3D, como type no object3DMap da entidade. O A-Frame adiciona obj como filho da raiz object3D da entidade.

AFRAME.registerComponent('example-orthogonal-camera', {
  update: function () {
    this.el.setObject3D('camera', new THREE.OrthogonalCamera());
  }
});

removeAttribute (componentName, propertyName)

Se componentName for o nome de um componente registrado, juntamente com a remoção do atributo do DOM, removeAttribute também desanexará o componente da entidade, invocando o método de ciclo de vida remove do componente.

entity.removeAttribute('goemetry');  // Detach the geometry component.
entity.removeAttribute('sound');  // Detach the sound component.

Se propertyName for fornecido, removeAttribute redefinirá o valor da propriedade especificada por propertyName para o valor padrão da propriedade:

entity.setAttribute('material', 'color', 'blue');  // The color is blue.
entity.removeAttribute('material', 'color');  // Reset the color to the default value, white.

removerObject3D (tipo)

removeObject3D remove o objeto especificado por type do THREE.Group da entidade e, portanto, da cena. Isso atualizará o object3DMap da entidade, definindo o valor da chave type como null. Isso geralmente é chamado de um componente, geralmente dentro do manipulador de remoção:

AFRAME.registerComponent('example-light', {
  update: function () {
    this.el.setObject3D('light', new THREE.Light());
    // Light is now part of the scene.
    // object3DMap.light is now a THREE.Light() object.
  },
  remove: function () {
    this.el.removeObject3D('light');
    // Light is now removed from the scene.
    // object3DMap.light is now null.
  }
});

removeState (stateName)

removeState irá remover um estado da entidade. Isso emitirá o evento stateremoved, e podemos verificar o estado de sua remoção usando .is:

entity.addEventListener('stateremoved', function (evt) {
  if (evt.detail.state === 'selected') {
    console.log('Entity no longer selected.');
  }
});
entity.addState('selected');
entity.is('selected');  // >> true
entity.removeState('selected');
entity.is('selected');  // >> false

Ouvindo alterações de componentes #

Podemos usar o evento componentchanged para detectar alterações na entidade:

entity.addEventListener('componentchanged', function (evt) {
  if (evt.detail.name === 'position') {
    console.log('Entity has moved from', 
      evt.detail.oldData, 'to', evt.detail.newData, '!');
  }
});

Ouvindo os Elementos Filhos Sendo Anexados e Desanexados #

Podemos usar os eventos child-attached e child-attached para detectar quando a cena anexa ou desanexa uma entidade:

entity.addEventListener('child-attached', function (evt) {
  if (evt.detail.el.tagName.toLowerCase() === 'a-box') {
    console.log('a box element has been attached');
  };
});

Dados do componente de várias propriedades da entidade (setAttribute) #

Atualizando dados de componentes de várias propriedades #

Para atualizar os dados de um componente de várias propriedades, podemos passar o nome de um componente registrado como componentName e passar um objeto de propriedades como valor. Uma string também é aceitável (por exemplo, type: spot; distance: 30), mas os objetos salvarão o A-Frame algum trabalho na análise:

// Only the properties passed in the object will be overwritten.
entity.setAttribute('light', {
  type: 'spot',
  distance: 30,
  intensity: 2.0
});

Ou para atualizar propriedades individuais para um componente de várias propriedades, podemos passar o nome do componente registrado como componentName, um nome de propriedade como segundo argumento e o valor da propriedade a ser definido como terceiro argumento:

// All previous properties for the material component (besides the color)  will be unaffected.
entity.setAttribute('material', 'color', 'crimson');

Observe que os tipos de propriedade de matriz se comportam de maneira exclusiva:

  • Matrizes são mutáveis. Eles são atribuídos por referência para que as alterações nos arrays sejam visíveis pelo componente.
  • As atualizações nas propriedades do tipo array não acionarão o método de atualização do componente nem emitirão eventos.

Atualizando dados de componentes de várias propriedades #

Se true for passado como terceiro argumento para .setAttribute, as propriedades não especificadas serão redefinidas e eliminadas:

// All previous properties for the light component will be removed and overwritten.
entity.setAttribute('light', {
  type: 'spot',
  distance: 30,
  intensity: 2.0
}, true);

Recuperando uma Entidade #

Podemos simplesmente recuperar uma entidade usando APIs DOM.

<a-entity id="mario"></a-entity>

var el = document.querySelector('#mario');

Recuperando componentes de uma entidade #

Por exemplo, se quiséssemos pegar o objeto de câmera three.js de uma entidade ou objeto material, poderíamos acessar seus componentes

var camera = document.querySelector('a-entity[camera]').components.camera.camera;
var material = document.querySelector('a-entity[material]').components.material.material;

Ou se um componente expõe uma API, podemos chamar seus métodos:

document.querySelector('a-entity[sound]').components.sound.pause();
