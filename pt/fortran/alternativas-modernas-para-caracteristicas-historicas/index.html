Aritmética if declaração #

A instrução aritmética if permite usar três ramificações dependendo do resultado de uma expressão aritmética

if (arith_expr) label1, label2, label3

Esta instrução if transfere o fluxo de controle para um dos rótulos em um código. Se o resultado de arith_expr for negativo label1 está envolvido, se o resultado for zero label2 é usado, e se o resultado for positivo o último label3 é aplicado. Aritmética if requer todos os três rótulos, mas permite a reutilização de rótulos, portanto, esta declaração pode ser simplificada para um if de dois ramos.

Exemplos:

if (N * N - N / 2) 130, 140, 130

if (X) 100, 110, 120

Agora esse recurso está obsoleto com a mesma funcionalidade oferecida pela instrução if e pela construção if-else. Por exemplo, o fragmento

    if (X) 100, 110, 120
100 print*, "Negative"
    goto 200
110 print*, "Zero"
    goto 200
120 print*, "Positive"
200 continue

pode ser escrito como a construção if-else

if (X<0) then
  print*, "Negative"
else if (X==0) then
  print*, "Zero"
else
  print*, "Positive"
end if

Uma substituição de instrução if para

    if (X) 100, 100, 200
100 print *, "Negative or zero"
200 continue

talvez

if (X<=0) print*, "Negative or zero"

Formulário de fonte fixa #

O Fortran foi originalmente projetado para um formato de formato fixo baseado em um cartão perfurado de 80 colunas:

Sim: Esta é uma linha do próprio código do autor

Estes foram criados em uma máquina perfuradora de cartões, assim:

As imagens são fotografias originais do autor

O formato, conforme mostrado no cartão de amostra ilustrado, tinha as cinco primeiras colunas reservadas para os rótulos das declarações. A primeira coluna foi usada para denotar comentários por uma letra C. A sexta coluna foi usada para denotar uma continuação de instrução (inserindo qualquer caractere diferente de zero ‘0’). As últimas 8 colunas foram usadas para identificação e sequenciamento de cartas, o que era muito valioso se você derrubasse seu baralho de cartas no chão! A codificação de caracteres para cartões perfurados tinha apenas um conjunto limitado de caracteres e era apenas em maiúsculas. Como resultado, os programas Fortran ficaram assim:

       DIMENSION A(10)                                                    00000001
C THIS IS A COMMENT STATEMENT TO EXPLAIN THIS EXAMPLE PROGRAM             00000002
       WRITE (6,100)                                                      00000003
 100   FORMAT(169HTHIS IS A RATHER LONG STRING BEING OUTPUT WHICH GOES OVE00000004
      1R MORE THAN ONE LINE, AND USES THE STATEMENT CONTINUATION MARKER IN00000005
      2COLUMN 6, AND ALSO USES HOLLERITH STRING FORMAT)                   00000006
       STOP                                                               00000007
       END                                                                00000008

O caractere de espaço também foi ignorado em todos os lugares, exceto dentro de uma constante de caractere Hollerith (como mostrado acima). Isso significava que os espaços poderiam ocorrer dentro de palavras e constantes reservadas, ou completamente perdidos. Isso teve o efeito colateral de algumas declarações bastante enganosas, como:

       DO 1 I = 1.0

é uma atribuição para a variável DO1I enquanto:

       DO1I = 1,0

é na verdade um loop DO na variável I.

O Fortran moderno agora não exige essa forma fixa de entrada e permite a forma livre usando qualquer coluna. Os comentários agora são indicados por um ! que também pode ser anexado a uma linha de instrução. Espaços agora não são permitidos em nenhum lugar e devem ser usados ​​como separadores, assim como na maioria dos outros idiomas. O programa acima pode ser escrito em Fortran moderno como:

! This is a comment statement to explain this example program
Print *,"THIS IS A RATHER LONG STRING BEING OUTPUT WHICH no longer GOES OVER MORE THAN ONE LINE, AND does not need to USE THE STATEMENT CONTINUATION MARKER IN COLUMN 6, or the HOLLERITH STRING FORMAT"

Embora a continuação de estilo antigo não seja mais usada, o exemplo acima ilustra que instruções muito longas ainda ocorrerão. O Fortran moderno usa um símbolo & no final e no início da continuação. Por exemplo, poderíamos escrever o acima em uma forma mais legível:

! This is a comment statement to explain this example program
Print *,"THIS IS A RATHER LONG STRING BEING OUTPUT WHICH still &
         &GOES OVER MORE THAN ONE LINE, AND does need to USE THE STATEMENT &
         &CONTINUATION notation"

Blocos Comuns #

Nas primeiras formas de Fortran, o único mecanismo para criar armazenamento de variáveis ​​globais visíveis a partir de sub-rotinas e funções é usar o mecanismo de bloco COMMON. Isso permitiu que sequências de variáveis ​​fossem nomes e compartilhadas em comum.

Além dos blocos comuns nomeados, também pode haver um bloco comum em branco (sem nome).

Um bloco comum em branco pode ser declarado como

common i, j

considerando que o bloco nomeado variables poderia ser declarado como

common /variables/ i, j

Como exemplo completo, poderíamos imaginar um heap store que é usado por rotinas que podem adicionar e remover valores:

       PROGRAM STACKING
       COMMON /HEAP/ ICOUNT, ISTACK(1023)
       ICOUNT = 0
       READ *, IVAL
       CALL PUSH(IVAL)
       CALL POP(IVAL)
       END

       SUBROUTINE PUSH(IVAL)
       COMMON /HEAP/ ICOUNT, ISTACK(1023)
       ICOUNT = ICOUNT + 1
       ISTACK(ICOUNT) = IVAL
       RETURN
       END

       SUBROUTINE POP(IVAL)
       COMMON /HEAP/ ICOUNT, ISTACK(1023)
       IVAL = ISTACK(ICOUNT)
       ICOUNT = ICOUNT - 1
       RETURN
       END

Instruções comuns podem ser usadas para declarar implicitamente o tipo de uma variável e especificar o atributo dimension. Este comportamento por si só é muitas vezes uma fonte suficiente de confusão. Além disso, a associação de armazenamento implícita e os requisitos para definições repetidas em unidades de programa tornam o uso de blocos comuns propenso a erros.

Finalmente, os blocos comuns são muito restritos nos objetos que contêm. Por exemplo, uma matriz em um bloco comum deve ser de tamanho explícito; objetos alocáveis ​​podem não ocorrer; tipos derivados não devem ter inicialização padrão.

No Fortran moderno, esse compartilhamento de variáveis ​​pode ser tratado pelo uso de modules. O exemplo acima pode ser escrito como:

module heap
  implicit none
  ! In Fortran 2008 all module variables are implicitly saved
  integer, save :: count = 0
  integer, save :: stack(1023)
end module heap

program stacking
  implicit none
  integer val
  read *, val
  call push(val)
  call pop(val)

contains
  subroutine push(val)
    use heap, only : count, stack
    integer val
    count = count + 1
    stack(count) = val
  end subroutine push

  subroutine pop(val)
    use heap, only : count, stack
    integer val
    val = stack(count)
    count = count - 1
  end subroutine pop
end program stacking

Os blocos comuns nomeados e em branco têm comportamentos ligeiramente diferentes. De importância:

  • objetos em blocos comuns nomeados podem ser definidos inicialmente; objetos em branco comum não serão
  • objetos em blocos comuns em branco se comportam como se o bloco comum tivesse o atributo save; objetos em blocos comuns nomeados sem o atributo save podem ficar indefinidos quando o bloco não está no escopo de uma unidade de programa ativa

Este último ponto pode ser contrastado com o comportamento das variáveis ​​do módulo no código moderno. Todas as variáveis ​​de módulo no Fortran 2008 são salvas implicitamente e não se tornam indefinidas quando o módulo sai do escopo. Antes do Fortran 2008, variáveis ​​de módulo, como variáveis ​​em blocos comuns nomeados, também se tornavam indefinidas quando o módulo saía do escopo.

Construções DO não-bloco #

A construção não-bloco do se parece com

    integer i
    do 100, i=1, 5
100 print *, i

Isto é, onde a instrução de terminação rotulada não é uma instrução continue. Existem várias restrições sobre a declaração que pode ser usada como a declaração de terminação e a coisa toda é geralmente muito confusa.

Tal construção não-bloco pode ser reescrita em forma de bloco como

    integer i
    do 100 i=1,5
      print *, i
100 continue

ou melhor, usando uma instrução de terminação end do,

integer i
do i=1,5
  print *, i
end do

Retorno alternativo #

O retorno alternativo é uma facilidade para controlar o fluxo de execução no retorno de uma sub-rotina. É frequentemente usado como uma forma de tratamento de erros:

real x

call sub(x, 1, *100, *200)
print*, "Success:", x
stop

100 print*, "Negative input value"
stop

200 print*, "Input value too large"
stop

end

subroutine sub(x, i, *, *)
  real, intent(out) :: x
  integer, intent(in) :: i
  if (i<0) return 1
  if (i>10) return 2
  x = i
end subroutine

O retorno alternativo é marcado pelos argumentos * na lista de argumentos fictícios da sub-rotina.

Na instrução call acima de *100 e *200, consulte as instruções rotuladas como 100 e 200, respectivamente.

Na própria sub-rotina os comandos return correspondentes ao retorno alternativo possuem um número. Este número não é um valor de retorno, mas denota o rótulo fornecido para o qual a execução é passada no retorno. Neste caso, return 1 passa a execução para a instrução rotulada 100 e return 2 passa a execução para a instrução rotulada 200. Uma instrução return sem adornos, ou a conclusão da execução da sub-rotina sem uma instrução return, passa a execução imediatamente após a instrução de chamada.

A sintaxe de retorno alternativo é muito diferente de outras formas de associação de argumentos e a facilidade introduz controle de fluxo contrário aos gostos modernos. Um controle de fluxo mais agradável pode ser gerenciado com o retorno de um código de “status” inteiro.

real x
integer status

call sub(x, 1, status)
select case (status)
case (0)
  print*, "Success:", x
case (1)
  print*, "Negative input value"
case (2)
  print*, "Input value too large"
end select

end

subroutine sub(x, i, status)
  real, intent(out) :: x
  integer, intent(in) :: i
  integer, intent(out) :: status

  status = 0

  if (i<0) then
    status = 1
  else if (i>10)
    status = 2
  else
    x = i
  end if

end subroutine

GOTO atribuído #

O GOTO atribuído usa uma variável inteira à qual um rótulo de instrução é atribuído usando a instrução ASSIGN.

100 CONTINUE

...

ASSIGN 100 TO ILABEL

...


GOTO ILABEL

O GOTO atribuído é obsoleto no Fortran 90 e excluído no Fortran 95 e posterior. Isso pode ser evitado no código moderno usando procedimentos, procedimentos internos, ponteiros de procedimento e outros recursos.

GOTO computado #

O GOTO computado permite a ramificação do programa de acordo com o valor de uma expressão inteira.

GOTO (label_1, label_2,... label_n) scalar-integer-expression

Se scalar-integer-expression for igual a 1 o programa continua no rótulo da instrução label_1, se for igual a 2 vai para label_2 e assim por diante. Se for menor que 1 ou maior que n o programa continua na próxima linha.

Exemplo:

ivar = 2

...

GOTO (10, 20, 30, 40) ivar

pulará para o rótulo de instrução 20.

Esta forma de goto é obsoleta no Fortran 95 e posterior, sendo substituída pela construção select case.

Especificadores de formato atribuído #

Antes do Fortran 95 era possível usar formatos atribuídos para entrada ou saída. Considerar

integer i, fmt
read *, i

assign 100 to fmt
if (i<100000) assign 200 to fmt

print fmt, i

100 format ("This is a big number", I10)
200 format ("This is a small number", I6)

end

A instrução assign atribui um rótulo de instrução a uma variável inteira. Essa variável inteira é usada posteriormente como o especificador de formato na instrução print.

Essa atribuição de especificador de formato foi excluída no Fortran 95. Em vez disso, o código mais moderno pode usar alguma outra forma de controle de fluxo de execução

integer i
read *, i

if (i<100000) then
  print 100, i
else
  print 200, i
end if

100 format ("This is a big number", I10)
200 format ("This is a small number", I6)

end

ou uma variável de caractere pode ser usada como o especificador de formato

character(29), target :: big_fmt='("This is a big number", I10)'
character(30), target :: small_fmt='("This is a small number", I6)'
character(:), pointer :: fmt

integer i
read *, i

fmt=>big_fmt
if (i<100000) fmt=>small_fmt

print fmt, i

end

Funções de instrução #

Considere o programa

implicit none
integer f, i
f(i)=i

print *, f(1)
end

Aqui f é uma função de instrução. Ele tem o tipo de resultado inteiro, recebendo um argumento fictício inteiro.1

Essa função de instrução existe dentro do escopo em que é definida. Em particular, ele tem acesso a variáveis ​​e constantes nomeadas acessíveis nesse escopo.

No entanto, as funções de instrução estão sujeitas a muitas restrições e são potencialmente confusas (parecendo casualmente como uma instrução de atribuição de elemento de matriz). As restrições importantes são:

  • o resultado da função e os argumentos fictícios devem ser escalares
  • os argumentos fictícios estão no mesmo escopo que a função
  • funções de instrução não têm variáveis ​​locais
  • funções de instrução não podem ser passadas como argumentos reais

Os principais benefícios das funções de instrução são repetidos por funções internas

implicit none

print *, f(1)

contains

  integer function f(i)
    integer i
    f = i
  end function

end

As funções internas não estão sujeitas às restrições mencionadas acima, embora talvez valha a pena notar que um subprograma interno pode não conter mais subprogramas internos (mas pode conter uma função de instrução).

As funções internas têm seu próprio escopo, mas também têm associação de host disponível.

1 Em exemplos reais de código antigo, não seria incomum ver os argumentos fictícios de uma função de instrução sendo digitados implicitamente, mesmo que o resultado tenha um tipo explícito.
