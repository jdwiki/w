Precisão de números de ponto flutuante #

Números de ponto flutuante do tipo real não podem ter nenhum valor real. Eles podem representar números reais até certa quantidade de dígitos decimais.

O FORTRAN 77 garantiu dois tipos de ponto flutuante e os padrões mais recentes garantem pelo menos dois tipos reais. Variáveis ​​reais podem ser declaradas como

real x
double precision y

x aqui é um real de tipo padrão e y é um real de tipo com maior precisão decimal que x. No Fortran 2008, a precisão decimal de y é de pelo menos 10 e seu intervalo de expoente decimal de pelo menos 37.

 real, parameter             :: single = 1.12345678901234567890
 double precision, parameter :: double = 1.12345678901234567890d0
 
 print *, single
 print *, double

impressões

   1.12345684    
   1.1234567890123457

em compiladores comuns usando a configuração padrão.

Observe o d0 na constante de precisão dupla. Um literal real contendo d em vez de e para denotar o expoente é usado para indicar precisão dupla.

! Default single precision constant
1.23e45
! Double precision constant
1.23d45

O Fortran 90 introduziu tipos reais parametrizados usando tipos. O tipo de um tipo real é um inteiro chamado constante ou constante literal:

real(kind=real_kind) :: x

ou apenas

real(real_kind) :: x

Esta declaração declara que x é do tipo real com uma certa precisão dependendo do valor de real_kind.

Literais de ponto flutuante podem ser declarados com um tipo específico usando um sufixo

1.23456e78_real_kind

O valor exato de real_kind não é padronizado e difere de compilador para compilador. Para consultar o tipo de qualquer variável ou constante real, a função kind() pode ser usada:

print *, kind(1.0), kind(1.d0)

normalmente imprimirá

4 8

ou

1 2

dependendo do compilador.

Os números de tipo podem ser definidos de várias maneiras:

  1. Precisão simples (padrão) e dupla:

    integer, parameter :: single_kind = kind(1.)
integer, parameter :: double_kind = kind(1.d0)

  2. Usando a função intrínseca selected_real_kind([p, r]) para especificar a precisão decimal necessária. O tipo retornado tem precisão de pelo menos dígitos p e permite expoente de pelo menos r.

    integer, parameter :: single_kind = selected_real_kind( p=6, r=37 )
integer, parameter :: double_kind = selected_real_kind( p=15, r=200 )

  3. A partir do Fortran 2003, constantes pré-definidas estão disponíveis através do módulo intrínseco ISO_C_Binding para garantir que tipos reais sejam interoperáveis ​​com os tipos float, double ou long_double do compilador C que o acompanha:

    use ISO_C_Binding

integer, parameter :: single_kind = c_float
integer, parameter :: double_kind = c_double
integer, parameter :: long_kind = c_long_double

  4. A partir do Fortran 2008, constantes pré-definidas estão disponíveis através do módulo intrínseco ISO_Fortran_env. Essas constantes fornecem tipos reais com certo tamanho de armazenamento em bits

    use ISO_Fortran_env

integer, parameter :: single_kind = real32
integer, parameter :: double_kind = real64
integer, parameter :: quadruple_kind = real128

Se determinado tipo não estiver disponível no compilador, o valor retornado por selected_real_kind() ou o valor da constante inteira é -1.

Tipos intrínsecos #

A seguir estão os tipos de dados intrínsecos para Fortran:

integer
real
character
complex
logical

integer, real e complex são tipos numéricos.

character é um tipo usado para armazenar strings de caracteres.

logical é usado para armazenar valores binários .true. ou .false..

Todos os tipos intrínsecos numéricos e lógicos são parametrizados usando tipos.

integer(kind=specific_kind)

ou apenas

integer(specific_kind)

onde specific_kind é um inteiro chamado constante.

Variáveis ​​de caractere, além de terem um parâmetro de tipo, também possuem um parâmetro de comprimento:

character char

declara char como uma variável de caractere de comprimento 1 do tipo padrão, enquanto

character(len=len) name

declara name como uma variável de caractere de tipo padrão e comprimento len. O tipo também pode ser especificado

character(len=len, kind=specific_kind) name
character(kind=specific_kind) char

declara name como um caractere do tipo kind e comprimento len. char é um caractere de comprimento 1 do tipo kind.

Alternativamente, o formulário obsoleto para declaração de caracteres

character*len  name

pode ser visto em código mais antigo, declarando que name tem o comprimento len e o tipo de caractere padrão.

A declaração de uma variável de tipo intrínseco pode ser da forma acima, mas também pode usar a forma type(...):

integer i
real x
double precision y

é equivalente a (mas muito preferido)

type(integer) i
type(real) x
type(double precision) y

Tipos de dados derivados #

Defina um novo tipo, mytype:

type :: mytype
  integer :: int
  real    :: float
end type mytype

Declare uma variável do tipo mytype:

type(mytype) :: foo

Os componentes de um tipo derivado podem ser acessados ​​com o operador %1:

foo%int = 4
foo%float = 3.142

Um recurso do Fortran 2003 (ainda não implementado por todos os compiladores) permite definir tipos de dados parametrizados:

type, public :: matrix(rows, cols, k)
  integer, len :: rows, cols
  integer, kind :: k = kind(0.0)
  real(kind = k), dimension(rows, cols) :: values
end type matrix

O tipo derivado matrix tem três parâmetros de tipo que são listados entre parênteses após o nome do tipo (eles são rows, cols e k). Na declaração de cada parâmetro de tipo deve ser indicado se são parâmetros de tipo tipo (kind) ou comprimento (len).

Os parâmetros de tipo de tipo, como os dos tipos intrínsecos, devem ser expressões constantes, enquanto os parâmetros de tipo de comprimento, como o comprimento de uma variável de caractere intrínseca, podem variar durante a execução.

Observe que o parâmetro k tem um valor padrão, portanto, pode ser fornecido ou omitido quando uma variável do tipo matrix é declarada, como segue

type (matrix (55, 65, kind=double)) :: b, c ! default parameter provided
type (matrix (rows=40, cols=50)     :: m    ! default parameter omitted

O nome de um tipo derivado não pode ser “precisão dupla” ou igual a qualquer um dos tipos intrínsecos.

  1. Muitas pessoas se perguntam por que o Fortran usa % como operador de acesso ao componente, em vez do mais comum .. Isso ocorre porque . já é usado pela sintaxe do operador, ou seja, .not., .and., .my_own_operator..

Constantes literais #

As unidades de programa geralmente fazem uso de constantes literais. Estes cobrem os casos óbvios como

print *, "Hello", 1, 1.0

Exceto em um caso, cada constante literal é um escalar que possui tipo, parâmetros de tipo e valor fornecido pela sintaxe.

As constantes literais inteiras são da forma

1
-1
-1_1   ! For valid kind parameter 1
1_ik   ! For the named constant ik being a valid kind paramter

As constantes literais reais são da forma

1.0    ! Default real
1e0    ! Default real using exponent format
1._1   ! Real with kind parameter 1 (if valid)
1.0_sp ! Real with kind paramter named constant sp
1d0    ! Double precision real using exponent format
1e0_dp ! Real with kind named constant dp using exponent format

Constantes literais complexas são da forma

(1, 1.)       ! Complex with integer and real components, literal constants
(real, imag)  ! Complex with named constants as components

Se os componentes real e imaginário forem ambos inteiros, a constante literal complexa será complexa padrão e os componentes inteiros serão convertidos em real padrão. Se um componente for real, o parâmetro de tipo da constante literal complexa é o do real (e o componente inteiro é convertido para esse tipo real). Se ambos os componentes são reais, a constante literal complexa é do tipo real com a maior precisão.

As constantes literais lógicas são

.TRUE.     ! Default kind, with true value
.FALSE.    ! Default kind, with false value
.TRUE._1   ! Of kind 1 (if valid), with true value
.TRUE._lk  ! Of kind named constant lk (if valid), with true value

Os valores literais de caractere diferem ligeiramente no conceito, pois o especificador de tipo precede o valor

"Hello"       ! Character value of default kind
'Hello'       ! Character value of default kind
ck_"Hello"    ! Character value of kind ck
"'Bye"        ! Default kind character with a '
'''Bye'       ! Default kind character with a '
""            ! A zero-length character of default kind

Conforme sugerido acima, as constantes literais de caractere devem ser delimitadas por apóstrofos ou aspas, e o marcador inicial e final devem corresponder. Apóstrofos literais podem ser incluídos estando dentro de delimitadores de aspas ou aparecendo duplicados. O mesmo para aspas.

As constantes BOZ são distintas das acima, pois especificam apenas um valor: elas não possuem tipo ou parâmetro de tipo. Uma constante BOZ é um padrão de bits e é especificada como

B'00000'    ! A binary bit pattern
B"01010001" ! A binary bit pattern
O'012517'   ! An octal bit pattern
O"1267671"  ! An octal bit pattern
Z'0A4F'     ! A hexadecimal bit pattern
Z"FFFFFF"   ! A hexadecimal bit pattern

Constantes literais BOZ são limitadas em onde podem aparecer: como constantes em instruções data e uma seleção de procedimentos intrínsecos.

Parâmetros de tipo de comprimento assumidos e adiados #

Variáveis ​​de tipo caractere ou de tipo derivado com parâmetro de comprimento podem ter o parâmetro de comprimento assumed ou deferred. A variável de caractere name

character(len=len) name

é de comprimento len durante toda a execução. Por outro lado, o especificador de comprimento pode ser

character(len=*) ...  ! Assumed length

ou

character(len=:) ...  ! Deferred length

As variáveis ​​de comprimento assumidas assumem seu comprimento de outra entidade.

Na função

function f(dummy_name)
  character(len=*) dummy_name
end function f

o argumento fictício dummy_name tem o comprimento do argumento real.

A constante nomeada const_name em

character(len=*), parameter :: const_name = 'Name from which length is assumed'

tem comprimento dado pela expressão constante do lado direito.

Os parâmetros do tipo de comprimento adiado podem variar durante a execução. Uma variável com comprimento adiado deve ter o atributo allocable ou pointer

character(len=:), allocatable :: alloc_name
character(len=:), pointer :: ptr_name

O comprimento de tal variável pode ser definido de qualquer uma das seguintes maneiras

allocate(character(len=5) :: alloc_name, ptr_name)
alloc_name = 'Name'         ! Using allocation on intrinsic assignment
ptr_name => another_name    ! For given target

Para tipos derivados com parametrização de comprimento, a sintaxe é semelhante

  type t(len)
    integer, len :: len
    integer i(len)
  end type t

  type(t(:)), allocatable :: t1
  type(t(5)) t2

  call sub(t2)
  allocate(type(t(5)) :: t1)

contains

  subroutine sub(t2)
    type(t(*)), intent(out) :: t2
  end subroutine sub

end

Acessando substrings de caracteres #

Para a entidade personagem

character(len=5), parameter :: greeting = "Hello"

uma substring pode ser referenciada com a sintaxe

greeting(2:4)  ! "ell"

Para acessar uma única letra não basta escrever

greeting(1)    ! This isn't the letter "H"

mas

greeting(1:1)  ! This is "H"

Para uma matriz de caracteres

character(len=5), parameter :: greeting(2) = ["Hello", "Yo!  "]

temos acesso a substring como

greeting(1)(2:4)  ! "ell"

mas não podemos referenciar os caracteres não contíguos

greeting(:)(2:4)  ! The parent string here is an array

Podemos até acessar substrings de constantes literais

"Hello"(2:4)

Uma parte de uma variável de caractere também pode ser definida usando uma substring como variável. Por exemplo

integer :: i=1
character :: filename = 'file000.txt'

filename(9:11) = 'dat'
write(filename(5:7), '(I3.3)') i

Acessando componentes complexos #

A entidade complexa

complex, parameter :: x = (1., 4.)

tem parte real 1. e parte complexa 4.. Podemos acessar esses componentes individuais como

real(x)  ! The real component
aimag(x) ! The complex component
x%re     ! The real component
y%im     ! The complex component

O formato x%.. é novo no Fortran 2008 e não é amplamente suportado em compiladores. Este formulário, no entanto, pode ser usado para definir diretamente os componentes individuais de uma variável complexa

complex y
y%re = 0.
y%im = 1.

Declaração e atributos #

Ao longo dos tópicos e exemplos aqui veremos muitas declarações de variáveis, funções e assim por diante.

Além do nome, os objetos de dados podem ter atributos. Cobertas neste tópico são declarações de declaração como

integer, parameter :: single_kind = kind(1.)

que dá ao objeto single_kind o atributo parameter (tornando-o uma constante nomeada).

Existem muitos outros atributos, como

  • alvo
  • ponteiro
  • opcional
  • salvar

Os atributos podem ser especificados com as chamadas declarações de especificação de atributo

integer i    ! i is an integer (of default kind)...
pointer i    ! ... with the POINTER attribute...
optional i   ! ... and the OPTIONAL attribute

No entanto, geralmente é considerado melhor evitar o uso dessas instruções de especificação de atributo. Para maior clareza, os atributos podem ser especificados como parte de uma única declaração

integer, pointer, optional :: i

Isso também reduz a tentação de usar tipagem implícita.

Na maioria dos casos nesta documentação Fortran esta declaração de declaração única é preferida.
