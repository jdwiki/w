Se construir #

A construção if (chamada de bloco IF em FORTRAN 77) é comum em muitas linguagens de programação. Ele executa condicionalmente um bloco de código quando uma expressão lógica é avaliada como verdadeira.

[name:] IF (expr) THEN
    block
[ELSE IF (expr) THEN [name]
    block]
[ELSE [name]
   block]
END IF [name]

Onde,

  • nome - o nome da construção if (opcional)
  • expr - uma expressão lógica escalar entre parênteses
  • bloco - uma sequência de zero ou mais instruções ou construções

Um nome de construção no início de uma instrução if then deve ter o mesmo valor que o nome de construção na instrução end if e deve ser exclusivo para a unidade de escopo atual.

Em instruções if, (des)igualdades e expressões lógicas que avaliam uma instrução podem ser usadas com os seguintes operadores:

.LT.  which is <   ! less than
.LE.           <=  ! less than or equal
.GT.           >   ! greater than
.GE.           >=  ! greater than or equal
.EQ.           =   ! equal
.NE.           /=  ! not equal
.AND.              ! logical and
.OR.               ! logical or
.NOT.              ! negation

Exemplos:

! simplest form of if construct
if (a > b) then
    c =  b / 2
end if
!equivalent example with alternate syntax
if(a.gt.b)then
   c=b/2
endif

! named if construct
circle: if (r >= 0) then
    l = 2 * pi * r
end if circle

! complex example with nested if construct
block: if (a < e) then
    if (abs(c - e) <= d) then
        a = a * c
    else
        a = a * d
    end if
else
    a = a * e
end if block

Um uso histórico da construção if está no que é chamado de instrução “aritmética if”. Como isso pode ser substituído por construções mais modernas, no entanto, não é abordado aqui. Mais detalhes podem ser encontrados [aqui][1].

[1]: https://www.wikiod.com/pt/fortran/alternativas-modernas-para-caracteristicas-historicas#Aritmética if declaração

Construção SELECT CASE #

Uma construção select case executa condicionalmente um bloco de construções ou instruções dependendo do valor de uma expressão escalar em uma instrução select case. Esta construção de controle pode ser considerada como um substituto para o goto calculado.

[name:] SELECT CASE (expr)
[CASE (case-value [, case-value] ...) [name]
   block]...
[CASE DEFAULT [name]
   block]
END SELECT [name]

Onde,

  • name - o nome da construção select case (opcional)
  • expr - uma expressão escalar do tipo inteiro, lógico ou caractere (entre parênteses)
  • case-value - um ou mais inteiros escalares, lógicos ou expressões de inicialização de caracteres entre parênteses
  • bloco - uma sequência de zero ou mais instruções ou construções

Exemplos:

! simplest form of select case construct
select case(i)
case(:-1)
    s = -1
case(0)
    s = 0
case(1:)
    s = 1
case default
    print "Something strange is happened"
end select

Neste exemplo, (:-1) case value é um intervalo de valores que corresponde a todos os valores menores que zero, (0) corresponde a zeros e (1:) corresponde a todos os valores acima de zero, default envolve se outras seções não foram executadas.

Construção do bloco DO #

Uma construção do é uma construção de loop que tem um número de iterações governadas por um controle de loop

integer i
do i=1, 5
  print *, i
end do
print *, i

No formulário acima, a variável de loop i passa pelo loop 5 vezes, tomando os valores de 1 a 5 por sua vez. Após a conclusão da construção, a variável de loop tem o valor 6, ou seja, a variável de loop é incrementada mais uma vez após a conclusão do loop.

Mais geralmente, a construção do loop do pode ser entendida como segue

integer i, first, last, step
do i=first, last, step
end do

O loop começa com i com o valor first, incrementando cada iteração por step até que i seja maior que last (ou menor que last se o tamanho do passo for negativo).

É importante notar que desde o Fortran 95, a variável do laço e as expressões de controle do laço devem ser inteiros.

Uma iteração pode ser encerrada prematuramente com a instrução cycle

do i=1, 5
  if (i==4) cycle
end do

e toda a construção pode interromper a execução com a instrução exit

do i=1, 5
  if (i==4) exit
end do
print *, i

As construções do podem ser nomeadas:

do_name: do i=1, 5
end do do_name

o que é particularmente útil quando existem construções do aninhadas

do1: do i=1, 5
  do j=1,6
    if (j==3) cycle        ! This cycles the j construct
    if (j==4) cycle        ! This cycles the j construct
    if (i+j==7) cycle do1  ! This cycles the i construct
    if (i*j==15) exit do1  ! This exits the i construct
  end do
end do1

As construções do também podem ter controle de loop indeterminado, seja “para sempre” ou até que uma determinada condição seja atendida

integer :: i=0
do
  i=i+1
  if (i==5) exit
end do

ou

integer :: i=0
do while (i<6)
  i=i+1
end do

Isso também permite um loop infinito do através de uma instrução .true.

print *,'forever'
do while(.true.)
  print *,'and ever'
end do

Uma construção do também pode deixar a ordem das iterações indeterminada

do concurrent (i=1:5)
end do

observando que a forma de controle de loop é a mesma de um controle forall.

Existem várias restrições nas instruções que podem ser executadas dentro do intervalo de uma construção do concurrent que são projetadas para garantir que não haja dependências de dados entre as iterações da construção. Esta indicação explícita pelo programador pode permitir uma maior otimização (incluindo paralelização) pelo compilador que pode ser difícil determinar de outra forma.

Variáveis ​​"privadas" dentro de uma interação podem ser realizadas pelo uso de uma construção block dentro do do concurrent:

do concurrent (i=1:5, j=2:7)
  block
    real tempval  ! This is independent across iterations
  end block
end do

Outra forma da construção do bloco do usa uma instrução rotulada continue em vez de um end do:

    do 100, i=1, 5
100 continue

É até possível aninhar tais construções com uma instrução de terminação compartilhada

    do 100, i=1,5
    do 100, j=1,5
100 continue

Ambas as formas, e especialmente a segunda (que é obsoleta), geralmente devem ser evitadas por motivos de clareza.

Finalmente, há também uma construção não-bloco do. Isso também é considerado obsoleto e é [descrito em outro lugar](https://www.wikiod.com/pt/fortran/alternativas-modernas-para-caracteristicas-historicas#Construções DO não-bloco 201607242013447800136), juntamente com métodos para reestruturar como uma construção de bloco do.

construção ONDE #

A construção where, disponível no Fortran90 em diante, representa uma construção do mascarada. A instrução de mascaramento segue as mesmas regras da instrução if, mas é aplicada a todos os elementos do array fornecido. O uso de where permite que as operações sejam realizadas em um array (ou vários arrays do mesmo tamanho), cujos elementos atendem a uma determinada regra. Isso pode ser usado para simplificar operações simultâneas em várias variáveis.

Sintaxe:

[name]: where (mask)
    block
[elsewhere (mask)
    block]
[elsewhere
    block]
end where [name]

Aqui,

  • nome - é o nome dado ao bloco (se nomeado)
  • mask - é uma expressão lógica aplicada a todos os elementos
  • bloco - série de comandos a serem executados

Exemplos:

! Example variables
real:: A(5),B(5),C(5)
A = 0.0
B = 1.0
C = [0.0, 4.0, 5.0, 10.0, 0.0]

! Simple where construct use
where (C/=0)
    A=B/C
elsewhere
    A=0.0
end

! Named where construct
Block: where (C/=0)
    A=B/C
elsewhere
    A=0.0
end where Block
