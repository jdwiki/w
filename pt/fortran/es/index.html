Sintaxe #

  • WRITE(unit num, format num) gera os dados após os colchetes em uma nova linha.
  • READ(unidade num, formato num) entradas para a variável após os colchetes.
  • OPEN(número da unidade, ARQUIVO=arquivo) abre um arquivo. (Existem mais opções para abrir arquivos, mas elas não são importantes para E/S.
  • CLOSE(número da unidade) fecha um arquivo.

E/S simples #

Como exemplo de escrita de entrada e saída, pegaremos um valor real e retornaremos o valor e seu quadrado até que o usuário insira um número negativo.

Conforme especificado abaixo, o comando read recebe dois argumentos: o número da unidade e o especificador de formato. No exemplo abaixo, usamos * para o número da unidade (que indica stdin) e * para o formato (que indica o padrão para reais, neste caso). Também especificamos o formato da instrução print. Pode-se alternativamente usar write(*,"The value....") ou simplesmente ignorar a formatação e tê-la como

print *,"The entered value was ", x," and its square is ",x*x

o que provavelmente resultará em algumas strings e valores estranhamente espaçados.

program SimpleIO
   implicit none
   integer, parameter :: wp = selected_real_kind(15,307)
   real(kind=wp) :: x
   
   ! we'll loop over until user enters a negative number
   print '("Enter a number >= 0 to see its square. Enter a number < 0 to exit.")'
   do
      ! this reads the input as a double-pricision value
      read(*,*) x
      if (x < 0d0) exit
      ! print the entered value and it's square
      print '("The entered value was ",f12.6,", its square is ",f12.6,".")',x,x*x
   end do
   print '("Thank you!")'
   
end program SimpleIO

Leia com alguma verificação de erros #

Um exemplo moderno de Fortran que inclui verificação de erros e uma função para obter um novo número de unidade para o arquivo.

module functions

contains

    function get_new_fileunit() result (f)
        implicit none

        logical    :: op
        integer    :: f

        f = 1
        do 
            inquire(f,opened=op)
            if (op .eqv. .false.) exit
            f = f + 1
        enddo

    end function

end module

program file_read
    use functions, only : get_new_fileunit
    implicit none

    integer            :: unitno, ierr, readerr
    logical            :: exists
    real(kind(0.d0))   :: somevalue
    character(len=128) :: filename

    filename = "somefile.txt"

    inquire(file=trim(filename), exist=exists)
    if (exists) then
        unitno = get_new_fileunit()
        open(unitno, file=trim(filename), action="read", iostat=ierr)
        if (ierr .eq. 0) then
            read(unitno, *, iostat=readerr) somevalue
            if (readerr .eq. 0) then
                print*, "Value in file ", trim(filename), " is ", somevalue
            else
                print*, "Error ", readerr, & 
                        " attempting to read file ", & 
                        trim(filename)
            endif
        else
            print*, "Error ", ierr ," attempting to open file ", trim(filename)
            stop
        endif
    else
        print*, "Error -- cannot find file: ", trim(filename)
        stop
    endif

end program file_read

Passando argumentos de linha de comando #

Onde argumentos de linha de comando são suportados, eles podem ser lidos através do intrínseco get_command_argument (introduzido no padrão Fortran 2003). O intrínseco command_argument_count fornece uma maneira de saber o número de argumentos fornecidos na linha de comando.

Todos os argumentos de linha de comando são lidos como strings, portanto, uma conversão de tipo interno deve ser feita para dados numéricos. Como exemplo, este código simples soma os dois números fornecidos na linha de comando:

PROGRAM cmdlnsum
IMPLICIT NONE
CHARACTER(100) :: num1char
CHARACTER(100) :: num2char
REAL :: num1
REAL :: num2
REAL :: numsum

!First, make sure the right number of inputs have been provided
IF(COMMAND_ARGUMENT_COUNT().NE.2)THEN
  WRITE(*,*)'ERROR, TWO COMMAND-LINE ARGUMENTS REQUIRED, STOPPING'
  STOP
ENDIF

CALL GET_COMMAND_ARGUMENT(1,num1char)   !first, read in the two values
CALL GET_COMMAND_ARGUMENT(2,num2char)

READ(num1char,*)num1                    !then, convert them to REALs
READ(num2char,*)num2

numsum=num1+num2                        !sum numbers
WRITE(*,*)numsum                        !write out value

END PROGRAM

O argumento number em get_command_argument varia de maneira útil entre 0 e o resultado de command_argument_count. Se o valor for 0, o nome do comando será fornecido (se suportado).

Muitos compiladores também oferecem intrínsecos não padrão (como getarg) para acessar argumentos de linha de comando. Como não são padrão, a documentação do compilador correspondente deve ser consultada.

O uso de get_command_argument pode ser estendido além do exemplo acima com os argumentos length e status. Por exemplo, com

character(5) arg
integer stat
call get_command_argument(number=1, value=arg, status=stat)

o valor de stat será -1 se o primeiro argumento existir e tiver comprimento maior que 5. Se houver alguma outra dificuldade para recuperar o argumento, o valor de stat será algum número positivo (e arg consistem inteiramente em espaços em branco). Caso contrário, seu valor será ‘0’.

O argumento length pode ser combinado com uma variável de caractere de comprimento adiado, como no exemplo a seguir.

character(:), allocatable :: arg
integer arglen, stat
call get_command_argument(number=1, length=arglen)  ! Assume for simplicity success
allocate (character(arglen) :: arg)
call get_command_argument(number=1, value=arg, status=stat)
