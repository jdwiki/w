Construtores de matriz #

Um valor de matriz de classificação 1 pode ser criado usando um construtor de matriz, com a sintaxe

(/ ... /)
[ ... ]

A forma [...] foi introduzida no Fortran 2003 e é geralmente considerada mais clara para ler, especialmente em expressões complexas. Este formulário é usado exclusivamente neste exemplo.

Os valores apresentados em um construtor de matriz podem ser valores escalares, valores de matriz ou loops implícitos.

Os parâmetros de tipo e tipo da matriz construída correspondem aos valores no construtor da matriz

[1, 2, 3]      ! A rank-1 length-3 array of default integer type
[1., 2., 3.]   ! A rank-1 length-3 array of default real type
["A", "B"]     ! A rank-1 length-2 array of default character type

integer, parameter :: A = [2, 4]
[1, A, 3]      ! A rank-1 length-4 array of default integer type, with A's elements

integer i
[1, (i, i=2, 5), 6]  ! A rank-1 length-6 array of default integer type with an implied-do

Nos formulários acima, todos os valores fornecidos devem ser do mesmo tipo e parâmetro de tipo. A mistura de tipos ou parâmetros de tipo não é permitida. Os exemplos a seguir não são válidos

[1, 2.]      ! INVALID: Mixing integer and default real
[1e0, 2d0]   ! INVALID: Mixing default real and double precision
[1., 2._dp]  ! INVALID: Allowed only if kind `dp` corresponds to default real
["Hello", "Frederick"]  ! INVALID: Different length parameters

Para construir uma matriz usando tipos diferentes, uma especificação de tipo para a matriz deve ser fornecida

[integer :: 1, 2., 3d0]    ! A default integer array
[real(dp) :: 1, 2, 3._sp]  ! A real(dp) array
[character(len=9) :: "Hello", "Frederick"]  ! A length-2 array of length-9 characters

Esta última forma para matrizes de caracteres é especialmente conveniente para evitar preenchimento de espaço, como a alternativa

["Hello    ", "Frederick"]  ! A length-2 array of length-9 characters

O tamanho de um array chamado constante pode ser implícito pelo construtor do array usado para definir seu valor

integer, parameter :: ids(*) = [1, 2, 3, 4]

e para tipos com parâmetros de comprimento, o parâmetro de comprimento pode ser assumido

character(len=*), parameter :: names(*) = [character(3) :: "Me", "You", "Her"]

A especificação de tipo também é necessária na construção de arrays de comprimento zero. A partir de

[ ] ! Not a valid array constructor

os parâmetros de tipo e tipo não podem ser determinados a partir do conjunto de valores inexistentes. Para criar uma matriz de inteiros padrão de comprimento zero:

[integer :: ]

Os construtores de matrizes constroem apenas matrizes de classificação 1. Às vezes, como ao definir o valor de uma constante nomeada, matrizes de classificação mais alta também são necessárias em uma expressão. Matrizes de classificação mais alta podem ser obtidas do resultado de ‘reformar’ com uma matriz de classificação 1 construída

integer, parameter :: multi_rank_ids(2,2) = RESHAPE([1,2,3,4], shape=[2,2])

Em um construtor de matriz, os valores da matriz na ordem dos elementos com quaisquer matrizes na lista de valores sendo como se os elementos individuais fossem dados na ordem dos elementos da matriz. Assim, o exemplo anterior

integer, parameter :: A = [2, 4]
[1, A, 3]      ! A rank-1 length-4 array of default integer type, with A's elements

é equivalente a

[1, 2, 4, 3]   ! With the array written out in array element order

Geralmente os valores no construtor podem ser expressões arbitrárias, incluindo construtores de matrizes aninhadas. Para que tal construtor de matriz atenda a determinadas condições, como ser uma constante ou expressão de especificação, as restrições se aplicam aos valores constituintes.

Embora não seja um construtor de array, certos valores de array também podem ser convenientemente criados usando a função intrínseca spread. Por exemplo

[(0, i=1,10)]  ! An array with 10 default integers each of value 0

também é o resultado da referência de função

SPREAD(0, 1, 10)

Notação básica #

Qualquer tipo pode ser declarado como um array usando o atributo dimension ou apenas indicando diretamente a(s) dimension(s) do array:

! One dimensional array with 4 elements
integer, dimension(4) :: foo

! Two dimensional array with 4 rows and 2 columns
real, dimension(4, 2) :: bar

! Three dimensional array
type(mytype), dimension(6, 7, 8) :: myarray

! Same as above without using the dimension keyword
integer :: foo2(4)
real :: bar2(4, 2)
type(mytype) :: myarray2(6, 7, 8)

A última maneira de declarar array multidimensional, permite a declaração de arrays de mesmo tipo de classificação/dimensões diferentes em uma linha, como segue

real :: pencil(5), plate(3,-2:4), cuboid(0:3,-10:5,6)

A classificação máxima (número de dimensões) permitida é 15 no padrão Fortran 2008 e antes era 7.

Fortran armazena arrays na ordem coluna-major. Ou seja, os elementos de bar são armazenados na memória da seguinte forma:

bar(1, 1), bar(2, 1), bar(3, 1), bar(4, 1), bar(1, 2), bar(2, 2), ...

Em Fortran, a numeração de array começa em 1 por padrão, em contraste com C que começa em 0. Na verdade, em Fortran, você pode especificar explicitamente os limites superior e inferior de cada dimensão:

integer, dimension(7:12, -3:-1) :: geese

Isso declara um array de forma (6, 3), cujo primeiro elemento é geese(7, -3).

Os limites inferior e superior ao longo das 2 (ou mais) dimensões podem ser acessados ​​pelas funções intrínsecas ubound e lbound. De fato, lbound(geese,2) retornaria -3, enquanto ubound(geese,1) retornaria 12.

O tamanho de um array pode ser acessado pela função intrínseca size. Por exemplo, size(geese, dim = 1) retorna o tamanho da primeira dimensão que é 6.

Arrays inteiros, elementos de array e seções de array #

Considere o array declarado como

real x(10)

Então temos três aspectos de interesse:

  1. Todo o array x;
  2. Elementos de matriz, como x(1);
  3. Seções de matriz, como x(2:6).

Matrizes inteiras #

Na maioria dos casos, todo o array x refere-se a todos os elementos do array como uma única entidade. Pode aparecer em instruções executáveis ​​como print *, SUM(x), print *, SIZE(x) ou x=1.

Um array inteiro pode referenciar arrays que não são explicitamente modelados (como x acima):

function f(y)
  real, intent(out) :: y(:)
  real, allocatable :: z(:)

  y = 1.         ! Intrinsic assignment for the whole array
  z = [1., 2.,]  ! Intrinsic assignment for the whole array, invoking allocation
end function

Uma matriz de tamanho presumido também pode aparecer como uma matriz inteira, mas apenas em circunstâncias limitadas (a ser documentada em outro lugar).

Elementos da matriz #

Um elemento de matriz é referido como dando índices inteiros, um para cada classificação da matriz, denotando a localização em toda a matriz:

real x(5,2)
x(1,1) = 0.2
x(2,4) = 0.3

Um elemento de array é um escalar.

Seções da matriz #

Uma seção de matriz é uma referência a vários elementos (talvez apenas um) de uma matriz inteira, usando uma sintaxe envolvendo dois pontos:

real x(5,2)
x(:,1) = 0.         ! Referring to x(1,1), x(2,1), x(3,1), x(4,1) and x(5,1)
x(2,:) = 0.         ! Referring to x(2,1), x(2,2)
x(2:4,1) = 0.       ! Referring to x(2,1), x(3,1) and x(4,1)
x(2:3,1:2) = 0.     ! Referring to x(2,1), x(3,1), x(2,2) and x(3,2)
x(1:1,1) = 0.       ! Referring to x(1,1)
x([1,3,5],2) = 0.   ! Referring to x(1,2), x(3,2) and x(5,2)

O formulário final acima usa um [vector subscript](https://www.wikiod.com/pt/fortran/matrizes#Arrays inteiros, elementos de array e seções de array). Isso está sujeito a várias restrições além de outras seções da matriz.

Cada seção do array é um array, mesmo quando apenas um elemento é referenciado. Ou seja, x(1:1,1) é um array de rank 1 e x(1:1,1:1) é um array de rank 2.

As seções de matriz geralmente não têm um atributo de toda a matriz. Em particular, onde

real, allocatable :: x(:)
x = [1,2,3]     ! x is allocated as part of the assignment
x = [1,2,3,4]   ! x is dealloacted then allocated to a new shape in the assignment

A atribuição

x(:) = [1,2,3,4,5]   ! This is bad when x isn't the same shape as the right-hand side

não é permitido: x(:), embora uma seção de array com todos os elementos de x, não é um array alocável.

x(:) = [5,6,7,8]

está bem quando x tem a forma do lado direito.

Componentes de matriz de matrizes #

type t
   real y(5)
end type t

type(t) x(2)

Também podemos nos referir a matrizes inteiras, elementos de matriz e seções de matriz em configurações mais complicadas.

Do acima, x é um array inteiro. Nos tambem temos

x(1)%y        ! A whole array
x(1)%y(1)     ! An array element
x%y(1)        ! An array section
x(1)%y(:)     ! An array section
x([1,2]%y(1)  ! An array section
x(1)%y(1:1)   ! An array section

Nesses casos, não podemos ter mais de uma parte da referência consistindo em um array de rank 1. O seguinte, por exemplo, não é permitido

x%y             ! Both the x and y parts are arrays
x(1:1)%y(1:1)   ! Recall that each part is still an array section

Matrizes alocáveis #

Arrays podem ter o atributo allocable:

! One dimensional allocatable array
integer, dimension(:), allocatable :: foo
! Two dimensional allocatable array
real, dimension(:,:), allocatable :: bar

Isso declara a variável, mas não aloca nenhum espaço para ela.

! We can specify the bounds as usual
allocate(foo(3:5))

! It is an error to allocate an array twice
! so check it has not been allocated first
if (.not. allocated(foo)) then
  allocate(bar(10, 2))
end if

Uma vez que uma variável não é mais necessária, ela pode ser desalocada:

deallocate(foo)

Se por algum motivo uma instrução allocate falhar, o programa irá parar. Isso pode ser evitado se o status for verificado por meio da palavra-chave stat:

real, dimension(:), allocatable :: geese
integer :: status

allocate(geese(17), stat=status)
if (stat /= 0) then
  print*, "Something went wrong trying to allocate 'geese'"
  stop 1
end if

A instrução deallocate também tem a palavra-chave stat:

deallocate (geese, stat=status)

status é uma variável inteira cujo valor é 0 se a alocação ou desalocação foi bem sucedida.

Especificação da natureza do array: classificação e forma #

O atributo dimension em um objeto especifica que esse objeto é um array. Existem, no Fortran 2008, cinco naturezas de array:1

  • forma explícita
  • forma assumida
  • tamanho assumido
  • forma diferida
  • forma implícita

Pegue as três matrizes de classificação 12

integer a, b, c
dimension(5) a    ! Explicit shape (default lower bound 1), extent 5
dimension(:) b    ! Assumed or deferred shape
dimension(*) c    ! Assumed size or implied shape array

Com isso, pode-se ver que mais contexto é necessário para determinar completamente a natureza de uma matriz.

Forma explícita #

Uma matriz de forma explícita é sempre a forma de sua declaração. A menos que a matriz seja declarada como local para um subprograma ou construção de ‘bloco’, os limites que definem a forma devem ser expressões constantes. Em outros casos, um array de shape explícito pode ser um objeto automático, usando extensões que podem variar a cada chamada de um subprograma ou bloco.

subroutine sub(n)
  integer, intent(in) :: n
  integer a(5)   ! A local explicit shape array with constant bound
  integer b(n)   ! A local explicit shape array, automatic object
end subroutine

Forma assumida #

Uma matriz de forma assumida é um argumento fictício sem o atributo alocável ou ponteiro. Esse array toma sua forma a partir do argumento real ao qual está associado.

integer a(5), b(10)
call sub(a)   ! In this call the dummy argument is like x(5)
call sub(b)   ! In this call the dummy argument is like x(10)

contains

  subroutine sub(x)
    integer x(:)    ! Assumed shape dummy argument
  end subroutine sub

end

Quando um argumento fictício assume forma, o escopo que faz referência ao procedimento deve ter uma interface explícita disponível para esse procedimento.

Tamanho presumido #

Uma matriz de tamanho assumido é um argumento fictício que tem seu tamanho assumido a partir de seu argumento real.

subroutine sub(x)
  integer x(*)   ! Assumed size array
end subroutine

As matrizes de tamanho assumido se comportam de maneira muito diferente das matrizes de forma assumida e essas diferenças estão documentadas em outro lugar.

Forma diferida #

Um array de forma diferida é um array que tem o atributo alocável ou ponteiro. A forma de tal matriz é determinada por sua [alocação](https://www.wikiod.com/pt/fortran/matrizes#Matrizes alocáveis) ou atribuição de ponteiro.

integer, allocatable :: a(:)
integer, pointer :: b(:)

Forma implícita #

Uma matriz de forma implícita é uma constante nomeada que toma sua forma da expressão usada para estabelecer seu valor

integer, parameter :: a(*) = [1,2,3,4]

As implicações dessas declarações de matriz em argumentos fictícios devem ser documentadas em outro lugar.

1Uma Especificação Técnica estendendo o Fortran 2008 adiciona uma sexta natureza de array: classificação assumida. Isso não é coberto aqui.

2 Estes podem ser escritos equivalentemente como

integer, dimension(5) :: a
integer, dimension(:) :: b
integer, dimension(*) :: c

ou

integer a(5)
integer b(:)
integer c(*)

Operações de matriz #

Devido aos seus objetivos computacionais, as operações matemáticas em arrays são diretas no Fortran.

Adição e subtração #

As operações em matrizes da mesma forma e tamanho são muito semelhantes à álgebra matricial. Em vez de percorrer todos os índices com loops, pode-se escrever adição (e subtração):

real, dimension(2,3) :: A, B, C
real, dimension(5,6,3) :: D
A    = 3.    ! Assigning single value to the whole array
B    = 5.    ! Equivalent writing for assignment
C    = A + B ! All elements of C now have value 8.
D    = A + B ! Compiler will raise an error. The shapes and dimensions are not the same

Arrays de fatiamento também são válidos:

integer :: i, j
real, dimension(3,2) :: Mat = 0.
real, dimension(3)   :: Vec1 = 0., Vec2 = 0., Vec3 = 0.
i = 0
j = 0
do i = 1,3
  do j = 1,2
    Mat(i,j) = i+j
  enddo
enddo
Vec1 = Mat(:,1)
Vec2 = Mat(:,2)
Vec3 = Mat(1:2,1) + Mat(2:3,2)

Função #

Da mesma forma, a maioria das funções intrínsecas podem ser usadas implicitamente assumindo operação em componentes (embora isso não seja sistemático):

real, dimension(2) :: A, B
A(1) = 6
A(2) = 44 ! Random values
B    = sin(A) ! Identical to B(1) = sin(6), B(2) = sin(44).

Multiplicação e divisão #

Deve-se tomar cuidado com o produto e a divisão: operações intrínsecas usando símbolos * e / são elementares:

real, dimension(2) :: A, B, C
A(1) = 2
A(2) = 4
B(1) = 1
B(2) = 3
C = A*B ! Returns C(1) = 2*1 and C(2) = 4*3

Isso não deve ser confundido com operações de matriz (veja abaixo).

Operações de matriz #

As operações matriciais são procedimentos intrínsecos. Por exemplo, o produto matricial dos arrays da seção anterior é escrito da seguinte forma:

real, dimension(2,1) :: A, B
real, dimension(1,1) :: C
A(1) = 2
A(2) = 4
B(1) = 1
B(2) = 3
C = matmul(transpose(A),B) ! Returns the scalar product of vectors A and B

Operações complexas permitem o encapsulamento de funções criando arrays temporários. Embora permitido por alguns compiladores e opções de compilação, isso não é recomendado. Por exemplo, um produto incluindo uma transposição de matriz pode ser escrito:

real, dimension(3,3) :: A, B, C
A(:) = 4
B(:) = 5
C = matmul(transpose(A),matmul(B,matmul(A,transpose(B)))) ! Equivalent to A^t.B.A.B^T

Seções de array avançadas: trigêmeos subscritos e subscritos vetoriais #

Conforme mencionado em [outro exemplo](https://www.wikiod.com/pt/fortran/matrizes#Arrays inteiros, elementos de array e seções de array) um subconjunto dos elementos de uma matriz, chamado uma seção de matriz, pode ser referenciada. A partir desse exemplo podemos ter

real x(10)
x(:)   = 0.
x(2:6) = 1.
x(3:4) = [3., 5.]

As seções de matriz podem ser mais gerais do que isso, no entanto. Eles podem assumir a forma de tripletos subscritos ou subscritos vetoriais.

Trigêmeos subscritos #

Um triplo subscrito tem o formato [bound1]:[bound2][:stride]. Por exemplo

real x(10)
x(1:10) = ...   ! Elements x(1), x(2), ..., x(10)
x(1:) = ...     ! The omitted second bound is equivalent to the upper, same as above
x(:10) = ...    ! The omitted first bound is equivalent to the lower, same as above
x(1:6:2) = ...  ! Elements x(1), x(3), x(5)
x(5:1) = ...    ! No elements: the lower bound is greater than the upper
x(5:1:-1) = ... ! Elements x(5), x(4), x(3), x(2), x(1)
x(::3) = ...    ! Elements x(1), x(4), x(7), x(10), assuming omitted bounds
x(::-3) = ...   ! No elements: the bounds are assumed with the first the lower, negative stride

Quando um passo (que não deve ser zero) é especificado, a sequência de elementos começa com o primeiro limite especificado. Se a passada for positiva (resp. negativa) os elementos selecionados seguindo uma sequência incrementada (resp. decrementada) pela passada até o último elemento não maior (resp. menor) que o segundo limite é tomado. Se o passo for omitido, ele será tratado como sendo um.

Se o primeiro limite for maior que o segundo limite e o passo for positivo, nenhum elemento será especificado. Se o primeiro limite for menor que o segundo limite e o passo for negativo, nenhum elemento será especificado.

Deve-se notar que x(10:1:-1) não é o mesmo que x(1:10:1) mesmo que cada elemento de x apareça em ambos os casos.

Subscritos vetoriais #

Um subscrito de vetor é uma matriz de inteiros de classificação 1. Isso designa uma sequência de elementos correspondentes aos valores da matriz.

real x(10)
integer i
x([1,6,4]) = ...     ! Elements x(1), x(6), x(4)
x([(i,i=2,4)]) = ... ! Elements x(2), x(3) and x(4)
print*, x([2,5,2])   ! Elements x(2), x(5) and x(2)

Uma seção de matriz com um subscrito de vetor é restrita em como pode ser usada:

  • não pode ser argumento associado a um argumento fictício definido no procedimento;
  • pode não ser o destino em uma instrução de atribuição de ponteiro;
  • não pode ser um arquivo interno em uma declaração de transferência de dados.

Além disso, tal seção de matriz pode não aparecer em uma declaração que envolva sua definição quando o mesmo elemento é selecionado duas vezes. De cima:

print*, x([2,5,2])   ! Elements x(2), x(5) and x(2) are printed
x([2,5,2]) = 1.      ! Not permitted: x(2) appears twice in this definition
