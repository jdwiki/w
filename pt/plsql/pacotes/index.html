Sobrecarga #

Funções e procedimentos em pacotes podem ser sobrecarregados. O pacote a seguir TEST tem dois procedimentos chamados print_number, que se comportam de forma diferente dependendo dos parâmetros com os quais são chamados.

create or replace package TEST is
  procedure print_number(p_number in integer);
  procedure print_number(p_number in varchar2);
end TEST;
/
create or replace package body TEST is

  procedure print_number(p_number in integer) is
  begin
    dbms_output.put_line('Digit: ' || p_number);    
  end;
  
  procedure print_number(p_number in varchar2) is
  begin
    dbms_output.put_line('String: ' || p_number);    
  end;

end TEST;
/

Chamamos ambos os procedimentos. O primeiro com parâmetro inteiro, o segundo com varchar2.

set serveroutput on;
-- call the first procedure
exec test.print_number(3);
-- call the second procedure
exec test.print_number('three');

A saída do script acima é:

SQL> 
Digit: 3
PL/SQL procedure successfully completed
String: three
PL/SQL procedure successfully completed

Restrições de Sobrecarga #

Apenas subprogramas locais ou empacotados, ou métodos de tipo, podem ser sobrecarregados. Portanto, você não pode sobrecarregar subprogramas autônomos. Além disso, você não pode sobrecarregar dois subprogramas se seus parâmetros formais diferirem apenas em nome ou modo de parâmetro

Defina um cabeçalho e corpo de pacote com uma função. #

Neste exemplo, definimos um cabeçalho de pacote e um corpo de pacote com uma função.
Depois disso estamos chamando uma função do pacote que retorna um valor de retorno.

Cabeçalho do pacote:

CREATE OR REPLACE PACKAGE SkyPkg AS

       FUNCTION GetSkyColour(vPlanet IN VARCHAR2)
       RETURN VARCHAR2;
       
END;
/

Corpo do pacote:

CREATE OR REPLACE PACKAGE BODY SkyPkg AS

        FUNCTION GetSkyColour(vPlanet IN VARCHAR2)
        RETURN VARCHAR2
        AS
               vColour VARCHAR2(100) := NULL;
        BEGIN
               IF vPlanet = 'Earth' THEN
                       vColour := 'Blue';
               ELSIF vPlanet = 'Mars' THEN
                       vColour := 'Red';
               END IF;

               RETURN vColour;
        END;
        
END;
/

Chamando a função do corpo do pacote:

DECLARE
        vColour VARCHAR2(100);
BEGIN
        vColour := SkyPkg.GetSkyColour(vPlanet => 'Earth');
        DBMS_OUTPUT.PUT_LINE(vColour);
END;
/

Uso do pacote #

Os pacotes em PLSQL são uma coleção de procedimentos, funções, variáveis, exceções, constantes e estruturas de dados. Geralmente os recursos em um pacote estão relacionados entre si e realizam tarefas semelhantes.

Por que usar pacotes

  • Modularidade
  • Melhor desempenho/funcionalidade

Partes de um Pacote

Especificação - Às vezes chamado de cabeçalho de pacote. Contém declarações de variáveis ​​e tipos e as assinaturas das funções e procedimentos que estão no pacote que são públicos para serem chamados de fora do pacote.

Corpo do pacote - Contém o código e as declarações privadas.

A especificação do pacote deve ser compilada antes do corpo do pacote, caso contrário, a compilação do corpo do pacote reportará um erro.
