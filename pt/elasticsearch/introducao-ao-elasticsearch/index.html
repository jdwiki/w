Instalando o Elasticsearch no Windows #

Pré-requisitos #

A versão Windows do Elasticsearch pode ser obtida neste link: https://www.elastic.co/downloads/elasticsearch. A versão estável mais recente está sempre no topo.

Como estamos instalando no Windows, precisamos do arquivo .ZIP. Clique no link na seção Downloads: e salve o arquivo em seu computador.

Esta versão do elastic é “portátil”, o que significa que você não precisa executar um instalador para usar o programa. Descompacte o conteúdo do arquivo em um local que você possa lembrar facilmente. Para demonstração, vamos supor que você descompactou tudo em C:\elasticsearch.

Observe que o arquivo contém uma pasta chamada elasticsearch-<version> por padrão, você pode extrair essa pasta para C:\ e renomeá-la para elasticsearch ou criar C:\elasticsearch você mesmo e descompactar apenas o conteúdo da pasta no arquivo para lá.

Como o Elasticsearch é escrito em Java, ele precisa do Java Runtime Environment para funcionar. Portanto, antes de executar o servidor, verifique se o Java está disponível abrindo um prompt de comando e digitando:

java -version

Você deve obter uma resposta parecida com esta:

java version "1.8.0_91"
Java(TM) SE Runtime Environment (build 1.8.0_91-b14)
Java HotSpot(TM) Client VM (build 25.91-b14, mixed mode)

Se você vir o seguinte em vez disso

‘java’ não é reconhecido como um comando interno ou externo, programa operável ou arquivo de lote.

Java não está instalado em seu sistema ou não está configurado corretamente. Você pode seguir este tutorial para (re)instalar o Java. Além disso, certifique-se de que essas variáveis ​​de ambiente estejam definidas com valores semelhantes:

Variável Valor
JAVA_HOME C:\Arquivos de Programas\Java\jre
CAMINHO …;C:\Arquivos de Programas\Java\jre

Se você ainda não sabe como inspecionar essas variáveis ​​consulte este tutorial.

Executar a partir do arquivo em lote #

Com o Java instalado, abra a pasta bin. Ele pode ser encontrado diretamente na pasta para a qual você descompactou tudo, então deve estar em c:\elasticsearch\bin. Dentro desta pasta há um arquivo chamado elasticsearch.bat que pode ser usado para iniciar o Elasticsearch em uma janela de comando. Isso significa que as informações registradas pelo processo estarão visíveis na janela do prompt de comando. Para parar o servidor, pressione CTRLC ou simplesmente feche a janela.

Executar como um serviço do Windows #

Idealmente, você não deseja ter uma janela extra da qual não possa se livrar durante o desenvolvimento e, por esse motivo, o Elasticsearch pode ser configurado para ser executado como um serviço.

Antes de podermos instalar o Elasticsearch como um serviço, precisamos adicionar uma linha ao arquivo C:\elasticsearch\config\jvm.options:

O instalador do serviço requer que a configuração do tamanho da pilha de encadeamentos seja configurada em jvm.options antes de instalar o serviço. Em Windows de 32 bits, você deve adicionar -Xss320k […] e em Windows de 64 bits você deve adicionar -Xss1m ao arquivo jvm.options. [fonte]

Depois de fazer essa alteração, abra um prompt de comando e navegue até o diretório bin executando o seguinte comando:

C:\Users\user> cd c:\elasticsearch\bin

O gerenciamento de serviços é feito pelo elasticsearch-service.bat. Em versões mais antigas, este arquivo pode ser chamado simplesmente de service.bat. Para ver todos os argumentos disponíveis, execute-o sem nenhum:

C:\elasticsearch\bin> elasticsearch-service.bat

Usage: elasticsearch-service.bat install|remove|start|stop|manager [SERVICE_ID]

A saída também nos diz que há um argumento opcional SERVICE_ID, mas podemos ignorá-lo por enquanto. Para instalar o serviço, basta executar:

C:\elasticsearch\bin> elasticsearch-service.bat install

Depois de instalar o serviço, você pode iniciá-lo e interrompê-lo com os respectivos argumentos. Para iniciar o serviço, execute

C:\elasticsearch\bin> elasticsearch-service.bat start

e para pará-lo, corra

C:\elasticsearch\bin> elasticsearch-service.bat stop

Se preferir uma GUI para gerenciar o serviço, você pode usar o seguinte comando:

C:\elasticsearch\bin> elasticsearch-service.bat manager

Isso abrirá o Elastic Service Manager, que permite personalizar algumas configurações relacionadas ao serviço, bem como parar/iniciar o serviço usando os botões encontrados na parte inferior da primeira guia.

Instalando o Elasticsearch no Ubuntu 14.04 #


Pré-requisitos #

Para executar o Elasticsearch, é necessário um Java Runtime Environment (JRE) na máquina. O Elasticsearch requer Java 7 ou superior e recomenda Oracle JDK versão 1.8.0_73.

Instale o Oracle Java 8

sudo add-apt-repository -y ppa:webupd8team/java
sudo apt-get update
echo "oracle-java8-installer shared/accepted-oracle-license-v1-1 select true" | sudo debconf-set-selections
sudo apt-get install -y oracle-java8-installer

Verifique a versão do Java

java -version

Baixe e instale o pacote #

Usando binários

  1. Baixe a versão estável mais recente do Elasticsearch aqui.
  2. Descompacte o arquivo e execute

Linux:

$ bin/elasticsearch

Usando apt-get

Uma alternativa para baixar o elasticsearch do site é instalá-lo, usando apt-get.

wget -qO - https://packages.elastic.co/GPG-KEY-elasticsearch | sudo apt-key add -
echo "deb https://packages.elastic.co/elasticsearch/2.x/debian stable main" | sudo tee -a /etc/apt/sources.list.d/elasticsearch-2.x.list
sudo apt-get update && sudo apt-get install elasticsearch
sudo /etc/init.d/elasticsearch start

Instalando o elasticsearch versão 5.x

wget -qO - https://artifacts.elastic.co/GPG-KEY-elasticsearch | sudo apt-key add -
sudo apt-get install apt-transport-https
echo "deb https://artifacts.elastic.co/packages/5.x/apt stable main" | sudo tee -a /etc/apt/sources.list.d/elastic-5.x.list
sudo apt-get update && sudo apt-get install elasticsearch

Executando como um serviço no Linux: #

Depois de instalar o acima não inicia sozinho. então precisamos iniciá-lo como um serviço. Como iniciar ou parar o Elasticsearch depende se o seu sistema usa SysV init ou systemd. você pode verificá-lo com o seguinte comando.

ps -p 1

Se sua distribuição estiver usando o init do SysV, você precisará executar:

sudo update-rc.d elasticsearch defaults 95 10
sudo /etc/init.d/elasticsearch start

Caso contrário, se sua distribuição estiver usando o systemd:

sudo /bin/systemctl daemon-reload
sudo /bin/systemctl enable elasticsearch.service

Execute o comando CURL em seu navegador ou cliente REST para verificar se o Elasticsearch foi instalado corretamente.

curl -X GET http://localhost:9200/

Indexando e recuperando um documento #

O Elasticsearch é acessado por meio de uma API REST HTTP, normalmente usando a biblioteca cURL. As mensagens entre o servidor de pesquisa e o cliente (seu ou seu aplicativo) são enviadas na forma de strings JSON. Por padrão, o Elasticsearch é executado na porta 9200.

Nos exemplos abaixo, ?pretty é adicionado para informar ao Elasticsearch para embelezar a resposta JSON. Ao usar esses endpoints em um aplicativo, você não precisa adicionar esse parâmetro de consulta.

Documentos de indexação #

Se pretendemos atualizar as informações em um índice posteriormente, é uma boa ideia atribuir IDs exclusivos aos documentos que indexamos. Para adicionar um documento ao índice chamado megacorp, com tipo employee e ID 1, execute:

curl -XPUT "http://localhost:9200/megacorp/employee/1?pretty" -d'
{
    "first_name" : "John",
    "last_name" :  "Smith",
    "age" :        25,
    "about" :      "I love to go rock climbing",
    "interests": [ "sports", "music" ]
}'

Resposta:

{
  "_index": "megacorp",
  "_type": "employee",
  "_id": "1",
  "_version": 1,
  "_shards": {
    "total": 2,
    "successful": 1,
    "failed": 0
  },
  "created": true
}

O índice é criado se não existir quando enviamos a chamada PUT.

Instalando o Elasticsearch e o Kibana no CentOS 7 #

Para executar o Elasticsearch, é necessário um Java Runtime Environment (JRE) na máquina. O Elasticsearch requer Java 7 ou superior e recomenda Oracle JDK versão 1.8.0_73.

Portanto, certifique-se de ter Java em seu sistema. Se não, siga o procedimento:

# Install wget with yum 
yum -y install wget

# Download the rpm jre-8u60-linux-x64.rpm for 64 bit 
wget --no-cookies --no-check-certificate --header "Cookie: gpw_e24=http%3A%2F%2Fwww.oracle.com%2F; oraclelicense=accept-securebackup-cookie" "http://download.oracle.com/otn-pub/java/jdk/8u60-b27/jre-8u60-linux-x64.rpm"

# Download the rpm jre-8u101-linux-i586.rpm for 32 bit
wget --no-cookies --no-check-certificate --header "Cookie: gpw_e24=http%3A%2F%2Fwww.oracle.com%2F; oraclelicense=accept-securebackup-cookie" "http://download.oracle.com/otn-pub/java/jdk/8u101-b13/jre-8u101-linux-i586.rpm"

# Install jre-.*.rpm
rpm -ivh jre-.*.rpm

Java já deve estar instalado em seu sistema centOS. Você pode verificar com:

java -version

Baixe e instale o elasticsearch

# Download elasticsearch-2.3.5.rpm 
wget https://download.elastic.co/elasticsearch/release/org/elasticsearch/distribution/rpm/elasticsearch/2.3.5/elasticsearch-2.3.5.rpm

# Install elasticsearch-.*.rpm
rpm -ivh elasticsearch-.*.rpm

Executando elasticsearch como um serviço systemd na inicialização

sudo systemctl daemon-reload
sudo systemctl enable elasticsearch
sudo systemctl start elasticsearch

# check the current status to ensure everything is okay.
systemctl status elasticsearch

Instalando o Kibana

Primeira importação de chave GPG em rpm

sudo rpm --import http://packages.elastic.co/GPG-KEY-elasticsearch

Em seguida, crie um repositório local kibana.repo

sudo vi /etc/yum.repos.d/kibana.repo

E adicione o seguinte conteúdo:

[kibana-4.4]
name=Kibana repository for 4.4.x packages
baseurl=http://packages.elastic.co/kibana/4.4/centos
gpgcheck=1
gpgkey=http://packages.elastic.co/GPG-KEY-elasticsearch
enabled=1

Agora instale o kibana seguindo o comando:

yum -y install kibana

Comece com:

systemctl start kibana

Verifique o estado com:

systemctl status kibana

Você pode executá-lo como um serviço de inicialização.

systemctl enable kibana

Parâmetros básicos de pesquisa com exemplos: #

Por padrão, o documento indexado completo é retornado como parte de todas as pesquisas. Isso é chamado de fonte (campo _source nos resultados da pesquisa). Se não quisermos que todo o documento de origem seja retornado, podemos solicitar que apenas alguns campos de dentro da origem sejam retornados ou podemos definir _source como false para omitir o campo completamente.

Este exemplo mostra como retornar dois campos, account_number e balance (dentro de _source), da pesquisa:

curl -XPOST 'localhost:9200/bank/_search?pretty' -d '
{
  "query": { "match_all": {} },
  "_source": ["account_number", "balance"]
}'

Observe que o exemplo acima simplesmente reduz as informações retornadas no campo _source. Ele ainda retornará apenas um campo chamado _source, mas apenas os campos account_number e balance serão incluídos.

Se você tem experiência em SQL, o conceito acima é um pouco semelhante à consulta SQL

SELECT account_number, balance FROM bank;

Agora vamos para a parte da consulta. Anteriormente, vimos como a consulta match_all é usada para corresponder a todos os documentos. Vamos agora introduzir uma nova consulta chamada consulta de correspondência, que pode ser considerada uma consulta de pesquisa de campo básica (ou seja, uma pesquisa feita em um campo ou conjunto de campos específico).

Este exemplo retorna a conta com o account_number definido como 20:

curl -XPOST 'localhost:9200/bank/_search?pretty' -d '
{
  "query": { "match": { "account_number": 20 } }
}'

Este exemplo retorna todas as contas que contêm o termo “mill” no address:

curl -XPOST 'localhost:9200/bank/_search?pretty' -d '
{
  "query": { "match": { "address": "mill" } }
}'

Este exemplo retorna todas as contas que contêm o termo “mill” ou “lane” no address:

curl -XPOST 'localhost:9200/bank/_search?pretty' -d '
{
  "query": { "match": { "address": "mill lane" } }
}'

Este exemplo é uma variante de match (match_phrase) que divide a consulta em termos e retorna apenas documentos que contenham todos os termos do address nas mesmas posições relativas uns aos outros[1].

curl -XPOST 'localhost:9200/bank/_search?pretty' -d '
{
  "query": { "match_phrase": { "address": "mill lane" } }
}'

Vamos agora introduzir a consulta bool(ean). A consulta booleana nos permite compor consultas menores em consultas maiores usando lógica booleana.

Este exemplo compõe duas consultas de correspondência e retorna todas as contas que contêm “mill” e “lane” no endereço:

curl -XPOST 'localhost:9200/bank/_search?pretty' -d '
{
  "query": {
    "bool": {
      "must": [
        { "match": { "address": "mill" } },
        { "match": { "address": "lane" } }
      ]
    }
  }
}'

No exemplo acima, a cláusula bool must especifica todas as consultas que devem ser verdadeiras para que um documento seja considerado uma correspondência.

Por outro lado, este exemplo compõe duas consultas de correspondência e retorna todas as contas contendo “mill” ou “lane” no address:

curl -XPOST 'localhost:9200/bank/_search?pretty' -d '
{
  "query": {
    "bool": {
      "should": [
        { "match": { "address": "mill" } },
        { "match": { "address": "lane" } }
      ]
    }
  }
}'

No exemplo acima, a cláusula bool should especifica uma lista de consultas que devem ser verdadeiras para que um documento seja considerado uma correspondência.

Este exemplo compõe duas consultas de correspondência e retorna todas as contas que não contêm “mill” nem “lane” no address:

curl -XPOST 'localhost:9200/bank/_search?pretty' -d '
{
  "query": {
    "bool": {
      "must_not": [
        { "match": { "address": "mill" } },
        { "match": { "address": "lane" } }
      ]
    }
  }
}'

No exemplo acima, a cláusula bool must_not especifica uma lista de consultas, nenhuma das quais deve ser verdadeira para que um documento seja considerado uma correspondência.

Podemos combinar as cláusulas must, should e must_not simultaneamente dentro de uma consulta bool. Além disso, podemos compor consultas booleanas dentro de qualquer uma dessas cláusulas booleanas para imitar qualquer lógica booleana complexa de vários níveis.

Este exemplo retorna todas as contas que pertencem a pessoas que têm exatamente 40 anos e não moram em Washington (WA para abreviar):

curl -XPOST 'localhost:9200/bank/_search?pretty' -d '
{
  "query": {
    "bool": {
      "must": [
        { "match": { "age": "40" } }
      ],
      "must_not": [
        { "match": { "state": "WA" } }
      ]
    }
  }
}'
