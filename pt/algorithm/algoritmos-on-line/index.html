Teoria #

Definição 1: Um problema de otimização Π consiste em um conjunto de instâncias ΣΠ. Para cada instância σ∈ΣΠ existe um conjunto Ζσ de soluções e uma função objetivo fσ : Ζσ → ℜ≥0 que atribui um valor real positivo a cada solução.
Dizemos que OPT(σ) é o valor de uma solução ótima, A(σ) é a solução de um Algoritmo A para o problema Π e wA(σ)=fσ(A(σ)) seu valor.

Definição 2: Um algoritmo online A para um problema de minimização Π tem uma razão competitiva de r ≥ 1 se houver uma constante τ∈ℜ com

wA(σ) = fσ(A(σ)) ≤ r ⋅ OPT(&sigma) + τ

para todas as instâncias σ∈ΣΠ. A é chamado de algoritmo online r-competitivo. é par

wA(σ) ≤ r ⋅ OPT(&sigma)

para todas as instâncias σ∈ΣΠ então A é chamado de algoritmo online estritamente r-competitivo.

Proposição 1.3: LRU e FWF são algoritmos de marcação.

Prova: No início de cada fase (exceto na primeira) FWF tem um erro de cache e limpou o cache. isso significa que temos k páginas vazias. Em cada fase são solicitadas no máximo k páginas diferentes, então agora haverá despejo durante a fase. Portanto, FWF é um algoritmo de marcação.
Vamos supor que LRU não seja um algoritmo de marcação. Então existe uma instância σ onde LRU uma página marcada x na fase i foi despejada. Deixe σt a requisição na fase i onde x é despejado. Como x está marcado, deve haver uma solicitação anterior σt* para x na mesma fase, então t* < t. Depois de t* x é a página mais recente do cache, então para ser despejada em t a sequência σt*+1,…,σt tem que solicitar em menos k de x páginas diferentes. Isso implica que a fase i solicitou pelo menos k+1 páginas diferentes, o que é contraditório com a definição da fase. Portanto, LRU deve ser um algoritmo de marcação.

Proposição 1.4: Todo algoritmo de marcação é estritamente k-competitivo.

Prova: Vamos σ ser uma instância para o problema de paginação e l o número de fases para σ. Se l = 1, todo algoritmo de marcação é ótimo e o algoritmo offline ótimo não pode ser melhor.
Assumimos que l ≥ 2. o custo de cada algoritmo de marcação por exemplo σ é limitado a partir de cima com l ⋅ k porque em cada fase um algoritmo de marcação não pode remover mais de k páginas sem remover uma página marcada.
Agora tentamos mostrar que o algoritmo offline ótimo despeja pelo menos k+l-2 páginas para σ, k na primeira fase e pelo menos uma para cada fase seguinte, exceto a última. Como prova, vamos definir l-2 subsequências disjuntas de σ. A subsequência i ∈ {1,…,l-2} começa na segunda posição da fase i+1 e termina na primeira posição da fase i+2.
Seja x a primeira página da fase i+1. No início da subsequência i existe a página x e no máximo k-1 páginas diferentes no cache ótimo de algoritmos offline. Na subsequência i são k solicitações de página diferentes de x, de modo que o algoritmo off-line ideal precisa remover pelo menos uma página para cada subsequência. Como no início da fase 1 o cache ainda está vazio, o algoritmo offline ótimo causa k despejos durante a primeira fase. Isso mostra que

wA(σ) ≤ l⋅k ≤ (k+l-2)k ≤ OPT ( & sigma; ) & sdot; k

Corolário 1.5: LRU e FWF são estritamente k-competitivos.

Não existe uma constante r para a qual um algoritmo online A é r-competitivo, chamamos de A não competitivo.

Proposição 1.6: LFU e LIFO são não competitivos.

Prova: Seja l ≥ 2 uma constante, k ≥ 2 o tamanho do cache. As diferentes páginas de cache são numeradas 1,…,k+1. Observamos a seguinte sequência:

A primeira página 1 é solicitada l vezes a página 2 e assim uma. No final há (l-1) pedidos alternados para a página k e k+1.
LFU e LIFO preenchem seu cache com as páginas 1-k. Quando a página k+1 é solicitada, a página k é despejada e vice-versa. Isso significa que cada solicitação de subsequência (k,k+1)l-1 despeja uma página. Além disso, são k-1 faltas de cache pela primeira vez no uso das páginas 1-(k-1). Portanto, LFU e LIFO removem páginas exatas k-1+2(l-1).

Paginação #

O problema de paginação surge da limitação de espaço finito. Vamos supor que nosso cache C tenha páginas k. Agora queremos processar uma sequência de solicitações de páginas m que devem ter sido colocadas no cache antes de serem processadas. Claro que se m<=k então nós apenas colocamos todos os elementos no cache e ele funcionará, mas normalmente é m>>k.

Dizemos que uma solicitação é um cache hit, quando a página já está no cache, caso contrário, é chamado de cache miss. Nesse caso, devemos trazer a página solicitada para o cache e despejar outra, supondo que o cache esteja cheio. A meta é um cronograma de despejo que minimiza o número de despejos.

Existem inúmeras estratégias para esse problema, vejamos algumas:

  1. Primeiro a entrar, primeiro a sair (FIFO): a página mais antiga é despejada
  2. Último a entrar, primeiro a sair (LIFO): a página mais recente é despejada
  3. Menos usado recentemente (LRU): página de despejo cujo acesso mais recente foi o mais antigo
  4. Usado com menos frequência (LFU): página de despejo que foi solicitada com menos frequência
  5. Longest forward distance (LFD): Elimine a página no cache que não é solicitada até o mais distante no futuro.
  6. Limpar quando estiver cheio (FWF): limpe o cache assim que ocorrer uma falha de cache

Existem duas maneiras de abordar este problema:

  1. offline: a sequência de solicitações de página é conhecida antecipadamente
  2. online: a sequência de solicitações de página não é conhecida antecipadamente

Abordagem offline #

Para a primeira abordagem, veja o tópico Applications of Greedy Technique. É o terceiro exemplo Offline Caching considera as primeiras cinco estratégias acima e oferece um bom ponto de entrada para o seguinte.

O programa de exemplo foi estendido com a estratégia FWF:

class FWF : public Strategy {
public:
    FWF() : Strategy("FWF")
    {
    }

    int apply(int requestIndex) override
    {
        for(int i=0; i<cacheSize; ++i)
        {
            if(cache[i] == request[requestIndex])
                return i;

            // after first empty page all others have to be empty
            else if(cache[i] == emptyPage)
                return i;
        }

        // no free pages
        return 0;
    }

    void update(int cachePos, int requestIndex, bool cacheMiss) override
    {

        // no pages free -> miss -> clear cache
        if(cacheMiss && cachePos == 0)
        {
            for(int i = 1; i < cacheSize; ++i)
                cache[i] = emptyPage;
        }
    }
};

Abordagem Online #

Agora queremos abordar o problema online da paginação. Mas primeiro precisamos entender como fazer isso. Obviamente, um algoritmo online não pode ser melhor do que o algoritmo offline ótimo. Mas quão pior é? Precisamos de definições formais para responder a essa pergunta:

Definição 1.1: Um problema de otimização Π consiste em um conjunto de instâncias ΣΠ. Para cada instância σ∈ΣΠ existe um conjunto Ζσ de soluções e uma função objetivo fσ : Ζσ → ℜ≥0 que atribui um valor real positivo a cada solução.
Dizemos que OPT(σ) é o valor de uma solução ótima, A(σ) é a solução de um Algoritmo A para o problema Π e wA(σ)=fσ(A(σ)) seu valor.

wA(σ) = fσ(A(σ)) ≤ r ⋅ OPT(σ) + τ

para todas as instâncias σ∈ΣΠ. A é chamado de algoritmo online r-competitivo. é par

wA(σ) ≤ r ⋅ OPT(σ)

para todas as instâncias σ∈ΣΠ então A é chamado de algoritmo online estritamente r-competitivo.

Portanto, a questão é quão competitivo é nosso algoritmo on-line em comparação com um algoritmo off-line ideal. Em seu famoso livro Allan Borodin e Ran El-Yaniv usaram outro cenário para descrever a situação de paginação online:

Existe um adversário maligno que conhece seu algoritmo e o algoritmo offline ideal. A cada passo, ele tenta solicitar uma página que seja pior para você e simultaneamente melhor para o algoritmo offline. o fator competitivo de seu algoritmo é o fator de quão mal seu algoritmo se saiu em relação ao algoritmo offline ideal do adversário. Se você quer tentar ser o adversário, você pode tentar o Jogo do Adversário (tente vencer as estratégias de paginação).

Algoritmos de marcação #

Em vez de analisar cada algoritmo separadamente, vamos analisar uma família de algoritmos online especial para o problema de paginação chamada algoritmos de marcação.

Seja σ=(σ1,…,σp) uma instância para nosso problema e k nosso tamanho de cache, então σ pode ser dividido em fases:

  • A fase 1 é a subsequência máxima de σ desde o início até o máximo de k páginas diferentes serem solicitadas
  • Fase i ≥ 2 é a subsequência máxima de σ do final do passo i-1 até o máximo de k páginas diferentes são solicitadas

Um algoritmo de marcação (implícita ou explicitamente) mantém se uma página está marcada ou não. No início de cada fase estão todas as páginas desmarcadas. É uma página solicitada durante uma fase que fica marcada. Um algoritmo é um algoritmo de marcação iff que nunca remove uma página marcada do cache. Isso significa que as páginas usadas durante uma fase não serão despejadas.
