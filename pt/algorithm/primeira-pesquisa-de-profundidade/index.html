Introdução à pesquisa em profundidade #

Depth-first search é um algoritmo para percorrer ou pesquisar estruturas de dados em árvore ou gráfico. Começa-se na raiz e explora-se o máximo possível ao longo de cada ramo antes de retroceder. Uma versão de busca em profundidade foi investigada pelo matemático francês do século XIX Charles Pierre Trambux como uma estratégia para resolver labirintos.

A busca em profundidade é uma maneira sistemática de encontrar todos os vértices alcançáveis ​​a partir de um vértice de origem. Assim como a busca em largura, o DFS percorre um componente conectado de um determinado grafo e define uma árvore geradora. A ideia básica da busca em profundidade é explorar metodicamente cada aresta. Começamos a partir de vértices diferentes conforme necessário. Assim que descobrimos um vértice, o DFS começa a explorá-lo (diferentemente do BFS, que coloca um vértice em uma fila para explorar depois).

Vejamos um exemplo. Vamos percorrer este gráfico:

Percorreremos o gráfico seguindo estas regras:

  • Vamos começar pela fonte.
  • Nenhum nó será visitado duas vezes.
  • Os nós que ainda não visitamos, serão coloridos de branco.
  • O nó que visitamos, mas não visitamos todos os nós filhos, será colorido em cinza.
  • Nós completamente atravessados ​​serão coloridos em preto.

Vejamos passo a passo:[] 4

Podemos ver uma palavra-chave importante. Isso é backedge. Você pode ver. 5-1 é chamado de backedge. Isso ocorre porque ainda não terminamos com node-1, portanto, passar de outro nó para node-1 significa que há um ciclo no gráfico. No DFS, se pudermos ir de um nó cinza para outro, podemos ter certeza de que o grafo tem um ciclo. Esta é uma das maneiras de detectar o ciclo em um gráfico. Dependendo do nó origem e da ordem dos nós visitados, podemos descobrir qualquer aresta em um ciclo como backedge. Por exemplo: se formos para 5 de 1 primeiro, teríamos descoberto 2-1 como backedge.

A aresta que tomamos para ir do nó cinza ao nó branco é chamada de aresta de árvore. Se apenas mantivermos as bordas da árvore e removermos outras, obteremos a árvore DFS.

No grafo não direcionado, se pudermos visitar um nó já visitado, esse deve ser um backedge. Mas para grafos direcionados, devemos verificar as cores. Se e somente se pudermos ir de um nó cinza para outro nó cinza, isso é chamado de backedge.

No DFS, também podemos manter timestamps para cada nó, que podem ser usados ​​de várias maneiras (por exemplo: Topological Sort).

  1. Quando um nó v é alterado de branco para cinza, o tempo é registrado em d[v].
  2. Quando um nó v é alterado de cinza para preto, o tempo é registrado em f[v].

Aqui d[] significa tempo de descoberta e f[] significa tempo de término. Nosso pesudo-código ficará assim:

Procedure DFS(G):
for each node u in V[G]
    color[u] := white
    parent[u] := NULL
end for
time := 0
for each node u in V[G]
    if color[u] == white
        DFS-Visit(u)
    end if
end for

Procedure DFS-Visit(u):
color[u] := gray
time := time + 1
d[u] := time
for each node v adjacent to u
    if color[v] == white
        parent[v] := u
        DFS-Visit(v)
    end if
end for
color[u] := black
time := time + 1
f[u] := time

Complexidade:

Cada nó e arestas são visitados uma vez. Portanto, a complexidade do DFS é O(V+E), onde V denota o número de nós e E denota o número de arestas.

Aplicativos da pesquisa em profundidade:

  • Encontrar o caminho mais curto de todos os pares em um grafo não direcionado.
  • Detecção de ciclo em um gráfico.
  • Encontrando o caminho.
  • Sorteio Topológico.
  • Testar se um gráfico é bipartido.
  • Encontrando Componente Fortemente Conectado.
  • Resolvendo quebra-cabeças com uma solução.
