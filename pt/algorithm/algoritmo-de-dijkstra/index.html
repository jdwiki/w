Algoritmo de caminho mais curto de Dijkstra #

Antes de prosseguir, é recomendável ter uma breve ideia sobre Matriz de Adjacência e BFS

Algoritmo de Dijkstra é conhecido como algoritmo de caminho mais curto de fonte única. Ele é usado para encontrar os caminhos mais curtos entre nós em um grafo, que pode representar, por exemplo, redes rodoviárias. Foi concebido por Edsger W. Dijkstra em 1956 e publicado três anos depois.

Podemos encontrar o caminho mais curto usando o algoritmo de busca Breadth First Search (BFS). Esse algoritmo funciona bem, mas o problema é que ele assume que o custo de percorrer cada caminho é o mesmo, o que significa que o custo de cada aresta é o mesmo. O algoritmo de Dijkstra nos ajuda a encontrar o caminho mais curto onde o custo de cada caminho não é o mesmo.

A princípio veremos como modificar o BFS para escrever o algoritmo de Dijkstra, depois adicionaremos uma fila de prioridade para torná-lo um algoritmo de Dijkstra completo.

Digamos que a distância de cada nó da fonte seja mantida no array d[]. Como em, d[3] representa que d[3] o tempo é necessário para alcançar o nó 3 da origem. Se não soubermos a distância, armazenaremos infinito em d[3]. Além disso, deixe custo[u][v] representar o custo de u-v. Isso significa que é necessário custo[u][v] para ir do nó u ao nó v.

Precisamos entender o relaxamento de borda. Digamos que, da sua casa, que é fonte, leva 10 minutos para ir ao lugar A. E leva 25 minutos para chegar ao lugar B. Nós temos,

d[A] = 10
d[B] = 25

Agora, digamos que leva 7 minutos para ir do local A ao local B, o que significa:

cost[A][B] = 7

Em seguida, podemos ir para o lugar B da origem indo para o lugar A da origem e depois do lugar A, indo para o lugar B , que levará 10 + 7 = 17 minutos, em vez de 25 minutos. Então,

d[A] + cost[A][B] < d[B]

Então atualizamos,

d[B] = d[A] + cost[A][B]

Isso se chama relaxamento. Iremos do nó u para o nó v e se d[u] + cost[u][v] < d[v] então atualizaremos d[v] = d[u] + custo[u][v].

No BFS, não precisamos visitar nenhum nó duas vezes. Apenas verificamos se um nó é visitado ou não. Se não foi visitado, empurramos o nó na fila, marcamos como visitado e incrementamos a distância em 1. Em Dijkstra, podemos empurrar um nó na fila e em vez de atualizá-lo com visitado, relaxamos ou atualizamos o novo borda. Vejamos um exemplo:

Vamos supor que o Nó 1 seja a Fonte. Então,

d[1] = 0
d[2] = d[3] = d[4] = infinity (or a large value)

Definimos d[2], d[3] e d[4] como infinito porque ainda não sabemos a distância. E a distância da fonte é, obviamente, 0. Agora, vamos para outros nós de origem e, se pudermos atualizá-los, os colocaremos na fila. Digamos, por exemplo, percorreremos a borda 1-2. Como d[1] + 2 < d[2] que fará d[2] = 2. Da mesma forma, percorreremos aresta 1-3, o que faz d[3] = 5.

Podemos ver claramente que 5 não é a distância mais curta que podemos atravessar para ir ao nó 3. Portanto, percorrer um nó apenas uma vez, como BFS, não funciona aqui. Se passarmos do nó 2 para o nó 3 usando aresta 2-3, podemos atualizar d[3] = d[2] + 1 = 3. Assim, podemos ver que um nó pode ser atualizado muitas vezes. Quantas vezes você pergunta? O número máximo de vezes que um nó pode ser atualizado é o número de graus de entrada de um nó.

Vamos ver o pseudo-código para visitar qualquer nó várias vezes. Vamos simplesmente modificar o BFS:

procedure BFSmodified(G, source):
Q = queue()
distance[] = infinity
Q.enqueue(source)
distance[source]=0
while Q is not empty
    u <- Q.pop()
    for all edges from u to v in G.adjacentEdges(v) do
        if distance[u] + cost[u][v] < distance[v]
            distance[v] = distance[u] + cost[u][v]
        end if
    end for
end while
Return distance

Isso pode ser usado para encontrar o caminho mais curto de todos os nós da origem. A complexidade deste código não é tão boa. Aqui está o porquê,

No BFS, quando vamos do nó 1 para todos os outros nós, seguimos o método first come, first serve. Por exemplo, fomos para o nó 3 da origem antes de processar o nó 2. Se formos para o nó 3 da fonte, atualizaremos o nó 4 como 5 + 3 = 8. Quando atualizarmos novamente o nó 3 do nó 2, precisamos atualizar o nó 4 como 3 + 3 = 6 novamente! Portanto, o nó 4 é atualizado duas vezes.

Dijkstra propôs que, em vez de usar o método Primeiro a chegar, primeiro a servir, se atualizarmos os nós mais próximos primeiro, serão necessárias menos atualizações. Se processássemos o nó 2 antes, o nó 3 teria sido atualizado antes e, após atualizar o nó 4 de acordo, obteríamos facilmente a distância mais curta! A ideia é escolher da fila, o nó, que está mais próximo da origem. Portanto, usaremos a Priority Queue aqui para que, quando abrirmos a fila, ela nos traga o nó mais próximo u de source. Como vai fazer isso? Ele verificará o valor de d[u] com ele.

Vamos ver o pseudo-código:

procedure dijkstra(G, source):
Q = priority_queue()
distance[] = infinity
Q.enqueue(source)
distance[source] = 0
while Q is not empty
    u <- nodes in Q with minimum distance[]
    remove u from the Q
    for all edges from u to v in G.adjacentEdges(v) do
        if distance[u] + cost[u][v] < distance[v]
            distance[v] = distance[u] + cost[u][v]
            Q.enqueue(v)
        end if
    end for
end while
Return distance

O pseudocódigo retorna a distância de todos os outros nós da fonte. Se quisermos saber a distância de um único nó v, podemos simplesmente retornar o valor quando v for retirado da fila.

Agora, o Algoritmo de Dijkstra funciona quando há uma borda negativa? Se houver um ciclo negativo, ocorrerá o loop infinito, pois continuará reduzindo o custo todas as vezes. Mesmo se houver uma borda negativa, Dijkstra não funcionará, a menos que retornemos logo após o alvo ser estourado. Mas então, não será um algoritmo de Dijkstra. Vamos precisar de [Bellman–Ford](https://www.wikiod.com/pt/algorithm/algoritmo-de-bellman-ford#Algoritmo de caminho mais curto de fonte única (dado que há um ciclo negativo em um gráfico) negativo-ciclo-em-um-grap) para processar borda/ciclo negativo.

Complexidade:

A complexidade do BFS é O(log(V+E)) onde V é o número de nós e E é o número de arestas. Para Dijkstra, a complexidade é semelhante, mas a classificação de Priority Queue leva O(logV). Portanto, a complexidade total é: O(Vlog(V)+E)

Abaixo está um exemplo Java para resolver o algoritmo de caminho mais curto de Dijkstra usando a matriz de adjacência

import java.util.*;
import java.lang.*;
import java.io.*;

class ShortestPath
{
    static final int V=9;
    int minDistance(int dist[], Boolean sptSet[])
    {
        int min = Integer.MAX_VALUE, min_index=-1;

        for (int v = 0; v < V; v++)
            if (sptSet[v] == false && dist[v] <= min)
            {
                min = dist[v];
                min_index = v;
            }

        return min_index;
    }

    void printSolution(int dist[], int n)
    {
        System.out.println("Vertex Distance from Source");
        for (int i = 0; i < V; i++)
            System.out.println(i+" \t\t "+dist[i]);
    }

    void dijkstra(int graph[][], int src)
    {

        Boolean sptSet[] = new Boolean[V];

        for (int i = 0; i < V; i++)
        {
            dist[i] = Integer.MAX_VALUE;
            sptSet[i] = false;
        }

        dist[src] = 0;

        for (int count = 0; count < V-1; count++)
        {
            int u = minDistance(dist, sptSet);

            sptSet[u] = true;

            for (int v = 0; v < V; v++)

                if (!sptSet[v] && graph[u][v]!=0 &&
                        dist[u] != Integer.MAX_VALUE &&
                        dist[u]+graph[u][v] < dist[v])
                    dist[v] = dist[u] + graph[u][v];
        }

        printSolution(dist, V);
    }

    public static void main (String[] args)
    {
    int graph[][] = new int[][]{{0, 4, 0, 0, 0, 0, 0, 8, 0},
                                {4, 0, 8, 0, 0, 0, 0, 11, 0},
                                {0, 8, 0, 7, 0, 4, 0, 0, 2},
                                {0, 0, 7, 0, 9, 14, 0, 0, 0},
                                {0, 0, 0, 9, 0, 10, 0, 0, 0},
                                {0, 0, 4, 14, 10, 0, 2, 0, 0},
                                {0, 0, 0, 0, 0, 2, 0, 1, 6},
                                {8, 11, 0, 0, 0, 0, 1, 0, 7},
                                {0, 0, 2, 0, 0, 0, 6, 7, 0}
                                };
        ShortestPath t = new ShortestPath();
        t.dijkstra(graph, 0);
    }
}
