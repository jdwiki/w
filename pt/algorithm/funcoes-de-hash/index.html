Códigos de hash para tipos comuns em C# #

Os códigos hash produzidos pelo método GetHashCode() para incorporado e tipos C# comuns do namespace System são mostrados abaixo de.

Booleano #

1 se o valor for verdadeiro, 0 caso contrário.

Byte, [UInt16](https:// github.com/dotnet/coreclr/blob/release/1.1.0/src/mscorlib/src/System/UInt16.cs#L66), [Int32](https://github.com/dotnet/coreclr/blob/release /1.1.0/src/mscorlib/src/System/Int32.cs#L76), [UInt32](https://github.com/dotnet/coreclr/blob/release/1.1.0/src/mscorlib/src/ System/UInt32.cs#L77), Single #

Valor (se necessário convertido para Int32).

SByte #

((int)m_value ^ (int)m_value << 8);

Char #

(int)m_value ^ ((int)m_value << 16);

Int16 #

((int)((ushort)m_value) ^ (((int)m_value) << 16));

Int64, [Double](https:// github.com/dotnet/coreclr/blob/release/1.1.0/src/mscorlib/src/System/Double.cs#L191) #

Xor entre os 32 bits inferiores e superiores do número de 64 bits

(unchecked((int)((long)m_value)) ^ (int)(m_value >> 32));

UInt64, [DateTime](https:// github.com/dotnet/coreclr/blob/release/1.1.0/src/mscorlib/src/System/DateTime.cs#L824), [TimeSpan](https://github.com/dotnet/coreclr/blob/release /1.1.0/src/mscorlib/src/System/TimeSpan.cs#L215) #

((int)m_value) ^ (int)(m_value >> 32);

Decimal #

((((int *)&dbl)[0]) & 0xFFFFFFF0) ^ ((int *)&dbl)[1];

Objeto #

RuntimeHelpers.GetHashCode(this);

A implementação padrão é usada índice de bloco de sincronização.

String #

O cálculo do código hash depende do tipo de plataforma (Win32 ou Win64), recurso de uso de hash de string aleatório, modo Debug / Release. No caso da plataforma Win64:

int hash1 = 5381;
int hash2 = hash1;
int c;
char *s = src;
while ((c = s[0]) != 0) {
    hash1 = ((hash1 << 5) + hash1) ^ c;
    c = s[1];
    if (c == 0)
        break;
    hash2 = ((hash2 << 5) + hash2) ^ c;
    s += 2;
}
return hash1 + (hash2 * 1566083941);

ValueType #

O primeiro campo não estático é procurar e obter seu código de hash. Se o tipo não tiver campos não estáticos, o código de hash do tipo será retornado. O hashcode de um membro estático não pode ser obtido porque se esse membro for do mesmo tipo que o tipo original, o cálculo terminará em um loop infinito.

Nullable<T> #

return hasValue ? value.GetHashCode() : 0;

Array #

int ret = 0;
for (int i = (Length >= 8 ? Length - 8 : 0); i < Length; i++) 
{
    ret = ((ret << 5) + ret) ^ comparer.GetHashCode(GetValue(i));
}

Introdução às funções de hash #

A função hash h() é uma função arbitrária que mapeou dados x ∈ X de tamanho arbitrário para o valor y ∈ Y de tamanho fixo: y = h(x). Boas funções de hash têm as seguintes restrições:

  • funções hash se comportam como distribuição uniforme

  • funções hash são determinísticas. h(x) deve sempre retornar o mesmo valor para um dado x

  • cálculo rápido (tem tempo de execução O(1))

Em geral, o tamanho da função hash é menor que o tamanho dos dados de entrada: |y| < |x|. As funções hash não são reversíveis ou em outras palavras podem ser colisões: ∃ x1, x2 ∈ X, x1 ≠ x2: h(x1) = h(x2). X pode ser um conjunto finito ou infinito e Y é um conjunto finito.

As funções de hash são usadas em muitas partes da ciência da computação, por exemplo, em engenharia de software, criptografia, bancos de dados, redes, aprendizado de máquina e assim por diante. Existem muitos tipos diferentes de funções de hash, com diferentes propriedades específicas de domínio.

Muitas vezes o hash é um valor inteiro. Existem métodos especiais em linguagens de programação para cálculo de hash. Por exemplo, no método C# GetHashCode() para todos os tipos retorna o valor Int32 (número inteiro de 32 bits). Em Java toda classe fornece o método hashCode() que retorna int. Cada tipo de dados possui implementações próprias ou definidas pelo usuário.

Métodos de hash #

Existem várias abordagens para determinar a função hash. Sem perda de generalidade, vamos x ∈ X = {z ∈ ℤ: z ≥ 0} são números inteiros positivos. Freqüentemente m é primo (não muito perto de uma potência exata de 2).

Método Função de hash
Método de divisão h(x) = x mod m
Método de multiplicação h(x) = ⌊m (xA mod 1)⌋, A ∈ {z ∈ ℝ: 0 < z < 1}

#Tabela de hash#

Funções de hash usadas em tabelas de hash para calcular o índice em uma matriz de slots. A tabela de hash é uma estrutura de dados para implementação de dicionários (estrutura chave-valor). Boas tabelas de hash implementadas têm tempo O(1) para as próximas operações: inserir, pesquisar e excluir dados por chave. Mais de uma chave pode ser hash para o mesmo slot. Existem duas maneiras de resolver a colisão:

  1. Encadeamento: a lista encadeada é usada para armazenar elementos com o mesmo valor de hash no slot

  2. Endereçamento aberto: zero ou um elemento é armazenado em cada slot

Os próximos métodos são usados ​​para calcular as sequências de sonda necessárias para endereçamento aberto

Método Fórmula
Sondagem linear h(x, i) = (h'(x) + i) mod m
Sondagem quadrática h(x, i) = (h'(x) + c1*i + c2*i^2) mod m
Hash duplo h(x, i) = (h1(x) + i*h2(x)) mod m

Onde i ∈ {0, 1, ..., m-1}, h'(x), h1(x), h2(x) são funções de hash auxiliares, c1, c2 são constantes auxiliares positivas .
