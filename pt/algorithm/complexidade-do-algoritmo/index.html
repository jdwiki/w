Todos os algoritmos são uma lista de passos para resolver um problema. Cada etapa tem dependências de algum conjunto de etapas anteriores ou do início do algoritmo. Um pequeno problema pode ter a seguinte aparência:

Essa estrutura é chamada de gráfico acíclico direcionado, ou DAG, abreviado. Os links entre cada nó no gráfico representam dependências na ordem das operações e não há ciclos no gráfico.

Como as dependências acontecem? Tomemos por exemplo o seguinte código:

total = 0
for(i = 1; i < 10; i++)
    total = total + i

Neste pseudocódigo, cada iteração do loop for depende do resultado da iteração anterior porque estamos usando o valor calculado na iteração anterior nesta próxima iteração. O DAG para este código pode ser assim:

Se você entender essa representação de algoritmos, poderá usá-la para entender a complexidade do algoritmo em termos de trabalho e amplitude.

Trabalhar #

Trabalho é o número real de operações que precisam ser executadas para atingir o objetivo do algoritmo para um determinado tamanho de entrada n.

Período #

Span às vezes é chamado de caminho crítico e é o menor número de etapas que um algoritmo deve realizar para atingir o objetivo do algoritmo.

A imagem a seguir destaca o gráfico para mostrar as diferenças entre trabalho e extensão em nosso DAG de amostra.

O trabalho é o número de nós no gráfico como um todo. Isso é representado pelo gráfico à esquerda acima. A extensão é o caminho crítico, ou o caminho mais longo do início ao fim. Quando o trabalho pode ser feito em paralelo, os nós destacados em amarelo à direita representam a extensão, o menor número de etapas necessárias. Quando o trabalho deve ser feito em série, o vão é o mesmo que o trabalho.

Tanto o trabalho quanto a extensão podem ser avaliados independentemente em termos de análise. A velocidade de um algoritmo é determinada pelo span. A quantidade de poder computacional necessária é determinada pelo trabalho.

Notação Big-Theta #

Ao contrário da notação Big-O, que representa apenas o limite superior do tempo de execução de algum algoritmo, Big-Theta é um limite estreito; limite superior e inferior. O limite apertado é mais preciso, mas também mais difícil de calcular.

A notação Big-Theta é simétrica: f(x) = Ө(g(x)) <=> g(x) = Ө(f(x))

Uma maneira intuitiva de entender isso é que f(x) = Ө(g(x)) significa que os gráficos de f(x) e g(x) crescem na mesma taxa, ou que os gráficos ‘se comportam’ de forma semelhante para valores suficientemente grandes de x.

A expressão matemática completa da notação Big-Theta é a seguinte:
Ө(f(x)) = {g: N0 -> R e c1, c2, n0< /sub> > 0, onde c1 < abs(g(n) / f(n)), para todo n > n0 e abs é o valor absoluto }

Um exemplo

Se o algoritmo para a entrada n leva as operações 42n^2 + 25n + 4 para terminar, dizemos que é O(n^2), mas também é O(n^3) e O (n^100). No entanto, é Ө(n^2) e não é Ө(n^3), Ө(n^4) etc. Algoritmo que é Ө(f(n)) também é O(f(n)), mas não vice-versa!

Definição matemática formal

Ө(g(x)) é um conjunto de funções.

Ө(g(x)) = {f(x) tal que existem constantes positivas c1, c2, N tais que 0 <= c1*g(x) <= f(x) <= c2*g(x) para todo x > N}

Como Ө(g(x)) é um conjunto, poderíamos escrever f(x) ∈ Ө(g(x)) para indicar que f(x) é um membro de Ө(g(x)). Em vez disso, geralmente escreveremos f(x) = Ө(g(x)) para expressar a mesma noção - essa é a maneira comum.

Sempre que Ө(g(x)) aparece em uma fórmula, nós a interpretamos como uma função anônima que não queremos nomear. Por exemplo, a equação T(n) = T(n/2) + Ө(n), significa T(n) = T(n/2) + f(n) onde f(n) é uma função no conjunto Ө(n).

Sejam f e g duas funções definidas em algum subconjunto dos números reais. Escrevemos f(x) = Ө(g(x)) como x->infinity se e somente se existem constantes positivas K e L e um número real x0 tal que mantém:

K|g(x)| <= f(x) <= L|g(x)| para todos os x >= x0.

A definição é igual a:

f(x) = O(g(x)) e f(x) = Ω(g(x))

Um método que usa limites

se limit(x->infinity) f(x)/g(x) = c ∈ (0,∞) ou seja, o limite existe e é positivo, então f(x) = Ө(g(x))

Comparação das notações assintóticas #

Sejam f(n) e g(n) duas funções definidas no conjunto dos números reais positivos, c, c1, c2, n0 são constantes reais positivas.

Notação f(n) = O(g(n)) f(n) = Ω(g(n)) f(n) = Θ(g(n)) f(n) = o(g(n)) f(n) = ω(g(n))
Definição formal ∃ c > 0, ∃ n0 > 0 : ∀ n ≥ n0, 0 ≤ f(n) ≤ c g(n) ∃ c > 0, ∃ n0 > 0 : ∀ n ≥ n0, 0 ≤ c g(n) ≤ f(n) ∃ c1, c2 > 0, ∃ n0 > 0 : ∀ n ≥ n0, 0 ≤ c1 g(n) ≤ f(n) ≤ c2 g(n) ∀ c > 0, ∃ n0 > 0 : ∀ n ≥ n0, 0 ≤ f(n) < c g(n) ∀ c > 0, ∃ n0 > 0 : ∀ n ≥ n0, 0 ≤ c g(n) < f(n)
Analogia entre a comparação assintótica de f, g e números reais a, b a ≤ b a ≥ b a = b a < b a > b
Exemplo 7n + 10 = O(n^2 + n - 9) n^3 - 34 = Ω(10n^2 - 7n + 1) 1/2 n^2 - 7n = Θ(n^2) 5n^2 = o(n^3) 7n^2 = ω(n)

Notação Big-Omega #

A notação Ω é usada para limite inferior assintótico.

Definição formal #

Sejam f(n) e g(n) duas funções definidas no conjunto dos números reais positivos. Escrevemos f(n) = Ω(g(n)) se houver constantes positivas c e n0 tais que:

0 ≤ c g(n) ≤ f(n) para todo n ≥ n0.

Notas #

f(n) = Ω(g(n)) significa que f(n) cresce assintoticamente não mais lento que g(n). Também podemos dizer sobre Ω(g(n)) quando a análise do algoritmo não é suficiente para a afirmação sobre Θ(g(n)) ou / e O(g(n)).

Das definições de notações segue o teorema:

Para duas funções quaisquer f(n) e g(n) temos f(n) = Ө(g(n)) se e somente se f(n) = O(g(n)) ef(n) = Ω(g(n)).

Graficamente a notação Ω pode ser representada da seguinte forma:

Por exemplo, temos f(n) = 3n^2 + 5n - 4. Então f(n) = Ω(n^2). Também está correto f(n) = Ω(n), ou mesmo f(n) = Ω(1).

Outro exemplo para resolver o algoritmo de correspondência perfeita: Se o número de vértices for ímpar, então imprima “Sem correspondência perfeita”, caso contrário, tente todas as correspondências possíveis.

Gostaríamos de dizer que o algoritmo requer tempo exponencial, mas na verdade você não pode provar um limite inferior Ω(n^2) usando a definição usual de Ω já que o algoritmo é executado em tempo linear para n ímpar. Em vez disso, devemos definir f(n)=Ω(g(n)) dizendo para alguma constante c>0, f(n)≥ c g(n) para infinitas n. Isso dá uma boa correspondência entre os limites superior e inferior: f(n)=Ω(g(n)) iff f(n) != o(g(n)).
