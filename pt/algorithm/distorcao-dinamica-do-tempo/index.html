Introdução à distorção dinâmica do tempo #

Dynamic Time Warping(DTW) é um algoritmo para medir a semelhança entre duas sequências temporais que podem variar em velocidade. Por exemplo, semelhanças na caminhada podem ser detectadas usando DTW, mesmo se uma pessoa estiver andando mais rápido que a outra, ou se houver acelerações e desacelerações durante o curso de uma observação. Ele pode ser usado para combinar um comando de voz de amostra com outro comando, mesmo se a pessoa falar mais rápido ou mais devagar do que a voz de amostra pré-gravada. O DTW pode ser aplicado a sequências temporais de dados de vídeo, áudio e gráficos - de fato, qualquer dado que possa ser transformado em uma sequência linear pode ser analisado com DTW.

Em geral, o DTW é um método que calcula uma correspondência ótima entre duas sequências dadas com certas restrições. Mas vamos nos ater aos pontos mais simples aqui. Digamos que temos duas sequências de voz Sample e Test e queremos verificar se essas duas sequências correspondem ou não. Aqui, a sequência de voz refere-se ao sinal digital convertido da sua voz. Pode ser a amplitude ou frequência da sua voz que denota as palavras que você diz. Vamos assumir:

Sample = {1, 2, 3, 5, 5, 5, 6}
Test   = {1, 1, 2, 2, 3, 5}

Queremos descobrir a correspondência ideal entre essas duas sequências.

Inicialmente, definimos a distância entre dois pontos, d(x, y) onde x e y representam os dois pontos. Deixar,

d(x, y) = |x - y|     //absolute difference

Vamos criar uma matriz 2D Tabela usando essas duas sequências. Calcularemos as distâncias entre cada ponto de Sample com todos os pontos de Test e encontraremos a correspondência ideal entre eles.

+------+------+------+------+------+------+------+------+
|      |   0  |   1  |   1  |   2  |   2  |   3  |   5  |
+------+------+------+------+------+------+------+------+
|   0  |      |      |      |      |      |      |      |
+------+------+------+------+------+------+------+------+
|   1  |      |      |      |      |      |      |      |
+------+------+------+------+------+------+------+------+
|   2  |      |      |      |      |      |      |      |
+------+------+------+------+------+------+------+------+
|   3  |      |      |      |      |      |      |      |
+------+------+------+------+------+------+------+------+
|   5  |      |      |      |      |      |      |      |
+------+------+------+------+------+------+------+------+
|   5  |      |      |      |      |      |      |      |
+------+------+------+------+------+------+------+------+
|   5  |      |      |      |      |      |      |      |
+------+------+------+------+------+------+------+------+
|   6  |      |      |      |      |      |      |      |
+------+------+------+------+------+------+------+------+

Aqui, Tabela[i][j] representa a distância ideal entre duas sequências se considerarmos a sequência até Amostra[i] e Teste[j], considerando todas as distâncias ideais que observamos antes.

Para a primeira linha, se não obtivermos valores de Sample, a distância entre isso e Test será infinito. Então colocamos infinito na primeira linha. O mesmo vale para a primeira coluna. Se não obtivermos valores de Test, a distância entre este e Sample também será infinita. E a distância entre 0 e 0 será simplesmente 0. Nós temos,

+------+------+------+------+------+------+------+------+
|      |   0  |   1  |   1  |   2  |   2  |   3  |   5  |
+------+------+------+------+------+------+------+------+
|   0  |   0  |  inf |  inf |  inf |  inf |  inf |  inf |
+------+------+------+------+------+------+------+------+
|   1  |  inf |      |      |      |      |      |      |
+------+------+------+------+------+------+------+------+
|   2  |  inf |      |      |      |      |      |      |
+------+------+------+------+------+------+------+------+
|   3  |  inf |      |      |      |      |      |      |
+------+------+------+------+------+------+------+------+
|   5  |  inf |      |      |      |      |      |      |
+------+------+------+------+------+------+------+------+
|   5  |  inf |      |      |      |      |      |      |
+------+------+------+------+------+------+------+------+
|   5  |  inf |      |      |      |      |      |      |
+------+------+------+------+------+------+------+------+
|   6  |  inf |      |      |      |      |      |      |
+------+------+------+------+------+------+------+------+

Agora, para cada etapa, consideraremos a distância entre cada ponto em questão e a somaremos com a distância mínima encontrada até agora. Isso nos dará a distância ideal de duas sequências até essa posição. Nossa fórmula será,

Table[i][j] := d(i, j) + min(Table[i-1][j], Table[i-1][j-1], Table[i][j-1])

Para o primeiro, d(1, 1) = 0, Tabela[0][0] representa o mínimo. Portanto, o valor de Tabela[1][1] será 0 + 0 = 0. Para o segundo, d(1, 2) = 0. Tabela[1][1] representa o mínimo. O valor será: Tabela[1][2] = 0 + 0 = 0. Se continuarmos assim, após terminar, a tabela ficará assim:

+------+------+------+------+------+------+------+------+
|      |   0  |   1  |   1  |   2  |   2  |   3  |   5  |
+------+------+------+------+------+------+------+------+
|   0  |   0  |  inf |  inf |  inf |  inf |  inf |  inf |
+------+------+------+------+------+------+------+------+
|   1  |  inf |   0  |   0  |   1  |   2  |   4  |   8  |
+------+------+------+------+------+------+------+------+
|   2  |  inf |   1  |   1  |   0  |   0  |   1  |   4  |
+------+------+------+------+------+------+------+------+
|   3  |  inf |   3  |   3  |   1  |   1  |   0  |   2  |
+------+------+------+------+------+------+------+------+
|   5  |  inf |   7  |   7  |   4  |   4  |   2  |   0  |
+------+------+------+------+------+------+------+------+
|   5  |  inf |  11  |  11  |   7  |   7  |   4  |   0  |
+------+------+------+------+------+------+------+------+
|   5  |  inf |  15  |  15  |  10  |  10  |   6  |   0  |
+------+------+------+------+------+------+------+------+
|   6  |  inf |  20  |  20  |  14  |  14  |   9  |   1  |
+------+------+------+------+------+------+------+------+

O valor em Table[7][6] representa a distância máxima entre essas duas sequências fornecidas. Aqui 1 representa a distância máxima entre Amostra e Teste é 1.

Agora, se retrocedermos desde o último ponto, até o ponto inicial (0, 0), obtemos uma longa linha que se move horizontalmente, verticalmente e diagonalmente. Nosso procedimento de retrocesso será:

if Table[i-1][j-1] <= Table[i-1][j] and Table[i-1][j-1] <= Table[i][j-1]
    i := i - 1
    j := j - 1
else if Table[i-1][j] <= Table[i-1][j-1] and Table[i-1][j] <= Table[i][j-1]
    i := i - 1
else
    j := j - 1
end if

Continuaremos com isso até chegarmos a (0, 0). Cada movimento tem seu próprio significado:

  • Um movimento horizontal representa a exclusão. Isso significa que nossa sequência Test acelerou durante esse intervalo.
  • Um movimento vertical representa a inserção. Isso significa que a sequência de Teste foi desacelerada durante esse intervalo.
  • Um movimento diagonal representa o jogo. Durante este período, Teste e Amostra foram os mesmos.

Nosso pseudo-código será:

Procedure DTW(Sample, Test):
n := Sample.length
m := Test.length
Create Table[n + 1][m + 1]
for i from 1 to n
    Table[i][0] := infinity
end for
for i from 1 to m
    Table[0][i] := infinity
end for
Table[0][0] := 0
for i from 1 to n
    for j from 1 to m
        Table[i][j] := d(Sample[i], Test[j])
                       + minimum(Table[i-1][j-1],      //match
                                 Table[i][j-1],        //insertion
                                 Table[i-1][j])        //deletion
    end for
end for
Return Table[n + 1][m + 1]

Também podemos adicionar uma restrição de localidade. Ou seja, exigimos que se Sample[i] for correspondido com Test[j], então |i - j| não seja maior que w, um parâmetro de janela.

Complexidade:

A complexidade de calcular DTW é O(m * n) onde m e n representam o comprimento de cada sequência. Técnicas mais rápidas para computar DTW incluem PrunedDTW, SparseDTW e FastDTW.
