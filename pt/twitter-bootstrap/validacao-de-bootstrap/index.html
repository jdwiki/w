  • Usando ASP.NET MVC e anotações de dados
  • Exemplo de entrada que requer validação
  • Opcional

  • Esta técnica de validação só pode ser usada em entradas que estão dentro de um formulário.

  • As propriedades devem ter pelo menos um requisito de validação para mostrar destaque em uma validação onSubmit() com falha. Os tipos de dados (exceto string) têm um requisito de tipo de dados oculto, portanto, não exigem uma anotação de dados explícita. Strings não têm isso, então para forçar uma verificação de validação junto com os outros campos, adicione a anotação de dados [MinLengthAttribute(0)].

Usando ASP.NET MVC e anotações de dados #

Adicione o seguinte ao Web.config (na pasta Views), dentro de <appSettings>:

<add key="ClientValidationEnabled" value="true"/>
<add key="UnobtrusiveJavaScriptEnabled" value="true"/>

Adicione o pacote jqueryval ao BundleConfig.cs:

bundles.Add(new ScriptBundle("~/bundles/jqueryval").Include(
        "~/Scripts/jqueryval/jquery.validate*"));

Adicione o seguinte a todas as páginas que precisam de validação (ou _Layout.cshml):

<!-- Reference to the jqueryval bundle -->
@Scripts.Render("~/bundles/jqueryval")

<!-- jQuery to apply bootstrap validation classes and glyphicons to inputs -->
<script type="text/javascript">
    $.validator.setDefaults({
        highlight: function (element) {
            $(element).closest('.form-group').removeClass('has-success has-feedback').addClass('has-error has-feedback'); // red highlighting
            $(element).closest('.form-group').find('.form-control-feedback').removeClass('glyphicon-ok').addClass('glyphicon-remove'); // red cross glyphicon
        },
        unhighlight: function (element) {
            $(element).closest('.form-group').removeClass('has-error has-feedback').addClass('has-success has-feedback'); // green highlighting
            $(element).closest('.form-group').find('.form-control-feedback').removeClass('glyphicon-remove').addClass('glyphicon-ok'); // green tick glyphicon
        }
    });
</script>

Adicione anotações de dados aos campos relevantes no modelo:

using System.ComponentModel.DataAnnotations;

[Required(ErrorMessage = "This field is required.")

Na visualização, adicione o seguinte a cada entrada que precisa ser validada:

<!-- Validation messages -->
<div class="text-danger">@Html.ValidationMessageFor(m => m.SomeField)</div>

<!-- Bootstrap feedback span: -->
<span class="glyphicon form-control-feedback"></span>

Adicione o seguinte à ação do controlador relevante para adicionar a validação do lado do servidor:

if (!ModelState.IsValid)
{
    return View(model);
}
else
{
    // continue with action
}

Exemplo de entrada que requer validação #

Modelo:

[Required(ErrorMessage = "This field is required.")
[StringLength(maximumLength: 10, ErrorMessage = "This field must be 10 characters or less.")]
public string SomeRequiredField { get; set; }

Visão:

<div class="form-group has-feedback">
    <div class="col-md-4">
        @Html.LabelFor(m => m.SomeRequiredField, new { @class = "control-label" })
    </div>
    <div class="col-md-8">
        @Html.TextBoxFor(m => m.SomeRequiredField, new { @class = "form-control" })
        <div class="text-danger">@Html.ValidationMessageFor(m => m.SomeRequiredField)</div>
        <span class="glyphicon form-control-feedback"></span>
    </div>
</div>

Opcional #

Adicione o seguinte jQuery para validar entradas no desfoque, bem como no envio:

$('input').on('blur', function () {
    $(this).valid();
});
