O Bootstrap 4 é uma grande reescrita e há muitas mudanças a serem observadas ao atualizar do Bootstap 3. Aqui estão as mudanças no nome da classe, dicas e exemplos de migrando seu código Bootstrap 3.x para Bootstrap 4.x.

Este é apenas um pequeno exemplo exemplos mais detalhados a serem seguidos.

Barra de navegação do Bootstrap 4 #

O novo componente Bootstrap 4 Navbar foi melhorado em relação ao seu predecessor Bootstrap 3.x. No Bootstrap 4, a Navbar é responsiva por padrão e utiliza o flexbox para facilitar muito o alinhamento do conteúdo da Navbar. É também uma simples questão de usar as novas classes navbar-toggleable-* para alterar o ponto de interrupção da Navbar. Agora a Navbar tem 6 tamanhos de ponto de interrupção ou “estados” para que você possa ter facilmente uma das seguintes opções da Navbar.

  • A barra de navegação nunca é recolhida na visualização móvel vertical e é sempre horizontal.
  • O Navbar é sempre recolhido na visão vertical e alternado através do hambúrguer.
  • A barra de navegação é recolhida na visualização vertical em um dos 4 pontos de interrupção responsivos.

Barra de navegação básica do Bootstrap 4

<nav class="navbar navbar-toggleable-md">
  <button class="navbar-toggler navbar-toggler-right" type="button" data-toggle="collapse" data-target="#navbar1">
    <span class="navbar-toggler-icon"></span>
  </button>
  <a class="navbar-brand" href="#">Navbar</a>
  <div class="collapse navbar-collapse" id="navbar1">
    <ul class="navbar-nav mr-auto">
      <li class="nav-item active">
        <a class="nav-link" href="#">Home</a>
      </li>
      <li class="nav-item">
        <a class="nav-link" href="#">Link</a>
      </li>
    </ul>
  </div>
</nav>

Como você pode ver no código acima, a classe navbar-header foi removida do Bootstrap 4, e o container-fluid não é mais necessário para uma Navbar de largura total.

Mudando o ponto de interrupção da barra de navegação

A classe navbar-toggleable-md faz a Navbar acima colapsar verticalmente (e mostrar o ícone de alternância) no ponto de interrupção médio (md) de 992px. Para mudar isso para um ponto de interrupção diferente, precisamos apenas trocar navbar-toggleable-md por um desses.

  • navbar-toggleable = recolher em xs larguras <576px
  • navbar-toggleable-sm = recolher em larguras sm <768px
  • navbar-toggleable-lg = recolher em larguras lg <1200px

Mudando o alinhamento da barra de navegação

O Flexbox nos permite alterar facilmente o alinhamento da Navbar e seu conteúdo (marca, links, formulários ou texto). O conteúdo padrão da Navbar é alinhado à esquerda. Claro que existem muitos outros cenários de alinhamento…

  • Marca esquerda (padrão), links no centro e à direita
  • Centro da marca, links à esquerda e à direita
  • Marca à esquerda e links à direita
  • Entrada de formulário de marca, links e largura de preenchimento
  • Sem marca, links no centro e à direita
  • Marca à esquerda, links à direita dentro do contêiner
  • Links justificados (largura de preenchimento) centralizados

Centralização do Bootstrap 4 #

Como centralizar um elemento, coluna ou conteúdo dentro de uma coluna funciona de forma diferente no Bootstrap 4.

Centro Horizontal #

  • text-center ainda é usado para elementos display:inline
  • mx-auto substitui center-block para centralizar os elementos display:block
  • offset-* ou mx-auto pode ser usado para centralizar as colunas da grade

mx-auto (margens automáticas do eixo x) centralizará os elementos display:block ou display:flex que possuem uma largura definida, (%, vw, px, etc. ). Flexbox é usado por padrão em colunas de grade, então também existem vários métodos de centralização de flexbox.

Centralizar texto ou elementos embutidos: centro de texto

<div class="container">
    <h1 class="text-center">i'm centered</h1>
    <div class="row">
        <div class="col text-center">i'm centered!</div>
    </div>
</div>

Centro de display:block ou display:flex: mx-auto

<div class="row">
    <div class="col-12">
        <img class="mx-auto d-block" src="//placehold.it/200x150?text=mx-auto">
    </div>
</div>

Centralizar colunas usando deslocamentos: offset-*

<div class="row">
    <div class="col-4 offset-4">
        <h6>I'm .col-4 centered (offset 4)
    </div>
</div>

As colunas também podem ser centralizadas com: mx-auto

<div class="row">
    <div class="col-4 mx-auto">
        <h6>I'm .col-4 centered</h6>
    </div>
</div>

Mudanças CSS do Bootstrap 3 para o Bootstrap 4 #

Como o Bootstrap 4 é uma reescrita principal, muitos dos nomes das classes do Bootstrap 3.x foram alterados ou removidos. A reestruturação de componentes como a Navbar e a introdução de novas classes CSS e suporte Flexbox significa que atualizar para 4.x não é um processo de conversão simples de 3.x.

No entanto, existem algumas classes CSS do Bootstrap 3.x que possuem uma substituição específica do Bootstrap 4.

O nome/seletor da classe CSS muda de Bootstrap 3.3.7 para 4 (alfa 6)

{t} - representa um tier ou ponto de interrupção (ou seja: sm,md,lg,etc..).
xs tier é o padrão e não precisa ser especificado: col- 3,col-6,etc.

{u} - representa um tamanho de col unit (ou seja: 1-12)

Mudanças no layout da coluna do sistema de grade no Bootstrap 4 #

O primeiro bloco de código é escrito no Bootstrap 3. No Bootstrap 3 existem 4 tipos de especificações de coluna, nomeadamente col-md-* col-lg-* col-sm-* col-xs-* . Um layout totalmente responsivo ficará assim no Bootstrap 3:

<div class="row">
    <div class="col-lg-4 col-md-8 col-sm-8 col-xs-8">
        contents
    </div>
    <div class="col-lg-4 col-md-4 col-sm-4 col-xs-4">
        contents
    </div>
</div>

No Bootstrap 4, eles adicionaram uma nova camada de grade sm abaixo de 768px para um controle mais granular. Portanto, o Bootstrap 4 tem col-* (xs), col-sm-*, col-md-*, col-lg-* e col-xl-*. Então, o que costumava ser .col-md-6 na v3 agora é .col-lg-6 na v4. Observe que o infixo -xs foi removido para que .col-6 represente 6 unidades de coluna no ponto de interrupção extra pequeno (padrão).

Então, se agora quisermos escrever o mesmo exemplo acima no Bootstrap 4, ficaria assim:

<div class="row">
    <div class="col-xl-8 col-lg-8 col-md-8 col-sm-8 col-8">
        contents
    </div>
    <div class="col-xl-8 col-lg-8 col-md-4 col-sm-4 col-4">
        contents
    </div>
</div>

Mudanças no suporte do navegador #

No twitter-bootstrap 4, o suporte para IE8, IE9 e iOS 6 foi descartado. v4 agora é apenas IE10+ e iOS 7+. Para sites que precisam de qualquer um desses, use v3.

No twitter-bootstrap 4, o suporte oficial para o navegador e o WebView do ‘Android v5.0 Lollipop’ foi adicionado. Versões anteriores do navegador Android e WebView permanecem apenas com ‘suporte não oficial’.

Remoção de classe de afixo #

O afixo foi removido do Bootstrap 4.

Recomenda-se usar um polyfill position: sticky em vez disso.

Se você estava usando Affix para aplicar estilos adicionais sem posição, os polyfills podem não suportar seu caso de uso. Uma opção para tais usos é a biblioteca ScrollPos-Styler de terceiros.

De acordo com a documentação do Bootstrap

Descartou o plugin Affix jQuery. Recomendamos usar uma posição: polyfill pegajoso em vez disso. Consulte a entrada HTML5 Please para obter detalhes e recomendações específicas de polyfill.

Se você estivesse usando Affix para aplicar estilos adicionais sem posição, o polyfills podem não suportar seu caso de uso. Uma opção para tais usos é a biblioteca ScrollPos-Styler de terceiros.

JAVASCRIPT

$(document).ready(function() {

  var toggleAffix = function(affixElement, scrollElement, wrapper) {
  
    var height = affixElement.outerHeight(),
        top = wrapper.offset().top;
    
    if (scrollElement.scrollTop() >= top){
        wrapper.height(height);
        affixElement.addClass("affix");
    }
    else {
        affixElement.removeClass("affix");
        wrapper.height('auto');
    }
      
  };
  

  $('[data-toggle="affix"]').each(function() {
    var ele = $(this),
        wrapper = $('<div></div>');
    
    ele.before(wrapper);
    $(window).on('scroll resize', function() {
        toggleAffix(ele, $(this), wrapper);
    });
    
    // init
    toggleAffix(ele, $(window), wrapper);
  });
  
});

Bootstrap 4 Alinhamento Vertical #

Alinhar os elementos no centro ou inferior verticalmente sempre foi um desafio com CSS e Bootstrap. O alinhamento vertical desejado pode estar dentro de um contêiner pai ou em relação a elementos adjacentes.

Agora que o Bootstrap 4 é flexbox por padrão, existem muitas abordagens diferentes para o alinhamento vertical usando: Auto-margins, Utilitários Flexbox ou Utilitários de exibição junto com Utilitários de alinhamento vertical.

A princípio, os Utilitários de alinhamento vertical parecem uma escolha óbvia, mas eles somente funcionam com elementos de exibição em linha e em tabela. Aqui estão algumas opções e cenários de alinhamento vertical do Bootstrap 4…

1 - Centro vertical usando margens automáticas:

Uma maneira de centralizar verticalmente é usar my-auto. Isso centralizará o elemento dentro de seu contêiner. Por exemplo, h-100 torna a linha de altura total, e my-auto centralizará verticalmente a coluna col-sm-12.

<div class="row h-100">
    <div class="col-sm-12 my-auto">
        <div class="card card-block w-25">Card</div>
    </div>
</div>

Centro vertical usando margens automáticas Demo

my-auto representa as margens no eixo y vertical e é equivalente a:

margin-top: auto;
margin-bottom: auto;

2 - Centro Vertical com Flexbox:

Como o Bootstrap 4 .row agora é display:flex você pode simplesmente usar align-self-center em qualquer coluna para centralizá-la verticalmente…

       <div class="row">
           <div class="col-6 align-self-center">
                <div class="card card-block">
                 Center
                </div>
           </div>
           <div class="col-6">
                <div class="card card-inverse card-danger">
                    Taller
                </div>

Ordem de 4 colunas do Bootstrap #

Alterar a ordem (ou posição) era possível no Bootstrap 3 usando as classes push pull. No Bootstrap 4, as classes push pull ainda funcionam e, além disso, flexbox order pode ser usado.

No Bootstrap 4, as classes push pull agora são push-{viewport}-{units} e pull-{viewport}-{units} e o infixo xs- foi removido. Considere este exemplo que altera a ordem das colunas para o layout 1-3-2 em xs e sm:

<div class="row">
    <div class="col-3 col-md-6">
        1
    </div>
    <div class="col-3 col-md-6 push-6 push-md-0">
        2
    </div>
    <div class="col-6 col-md-12 pull-3 pull-md-0">
        3
    </div>
</div>

Demonstração Push Pull do Bootstrap 4

Como a nova versão 4 é flexbox, outra opção é usar as classes flexbox utility para alterar a ordem das colunas. Agora, colunas col-*-12 de largura total e 12 unidades podem ser revertidas usando a ordenação flexbox.

<div class="row">
    <div class="col-md-12">
        Col 1
    </div>
    <div class="col-md-12 flex-first flex-md-unordered">
        Col 2
    </div>
</div>
