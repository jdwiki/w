Usando Clearfix em linhas e colunas

Ao criar layouts avançados, pode haver cenários em que você precise usar mais de 12 unidades de coluna em um único elemento .row. O conceito de column wrap e [redefinições responsivas (A.K.A. clearfixes)](http://getbootstrap.com/css/# grid-responsive-resets) são essenciais para entender o design responsivo com Bootstrap.

Noções básicas do Bootstrap Grid

As grades de bootstraps são notavelmente poderosas e elegantes. No entanto, você deve lembrar que o nome do framework é “Bootstrap”, não “WeDidItForYou”. O Bootstrap permite o design responsivo, mas não o garante.

Ainda depende de você tornar seu design realmente responsivo e oferecer aos usuários a melhor experiência possível para o usuário final.

O problema da altura #

Em nosso “exemplo ingênuo”, todas as nossas células tinham a mesma altura. O navegador voluntariamente quebrou as linhas exatamente onde queríamos, e tudo parecia certo com o mundo. Até a altura entrar em cena.

Vamos pegar o exemplo anterior e dar alguma altura a algumas das células, talvez como você veria em uma página do tipo painel.

<div class="container-fluid">
    <div class="row">
        <div class="col-xs-6 col-md-3">1</div>
        <div class="col-xs-6 col-md-3 cell-tall">2</div>
        <div class="col-xs-6 col-md-3 cell-tall">3</div>
        <div class="col-xs-6 col-md-3">4</div>
        <div class="col-xs-6 col-md-3">5</div>
        <div class="col-xs-6 col-md-3 cell-med">6</div>
        <div class="col-xs-6 col-md-3">7</div>
        <div class="col-xs-6 col-md-3">8</div>
        <div class="col-xs-6 col-md-3 cell-med">9</div>
        <div class="col-xs-6 col-md-3 cell-med">10</div>
        <div class="col-xs-6 col-md-3">11</div>
    </div>
</div>

Aqui nós adicionamos alguns CSS cell-tall e cell-med que definimos acima. Isso terá o efeito de alterar a altura de algumas das células. fico imaginando como vai ficar…

Aqui estão eles novamente em tamanhos de tela médios e pequenos:

Oh meu, que confusão. Acho que não era isso que queríamos. No tamanho médio-grande, 5 e 6 estão fora do lugar e, de alguma forma, 7 acabou iniciando uma nova linha. Em tamanho pequeno, temos duas células na primeira linha e quatro na segunda linha, com 4, 5 e 6 todas empilhadas à direita em ambos os tamanhos de tela!

Então, como resolvemos isso?

Clearfix para o resgate #

Uma maneira de ajudar a situação certamente seria usar mais de um row:

<div class="container-fluid">
    <div class="row">
        <!-- cols -->
    </div>
    <div class="row">
        <!-- cols -->
    </div>
</div>

Esta é geralmente a primeira coisa que os novos Bootstrappers tentam. Parece fazer sentido: “Eu quero quatro células em cada linha, então vou criar uma nova ’linha’ para cada 4 divs ‘col’”.

Mas há um problema com esta linha de raciocínio: O objetivo do Bootstrap 3 e da próxima versão 4 é ser responsivo. Ao colocar “quatro col em uma linha”, você não está realmente “pensando de forma responsiva”.

Um bom entendimento da classe CSS clearfix ajudará você a começar a ver que várias divs row realmente obscureceram sua compreensão da maneira como o design responsivo foi destinado a funcionar. Resumindo, você simplesmente não pode saber quantas col colocar em uma linha de qualquer maneira - o navegador ainda não renderizou seu trabalho!

Lembre-se em First Things First, dissemos que você precisa pensar no “inverso de 12”? Sem mais delongas, vamos corrigir nosso problema aqui, usando comentários diretamente no código para esclarecer qualquer confusão. Sim, parece muito mais código, mas a maioria do extra são comentários.

<div class="container-fluid">
    <div class="row">
        <div class="col-xs-6 col-md-3">1</div>
        <div class="col-xs-6 col-md-3 cell-tall">2</div>
        <!--
            We have rendered TWO cells.
            On small and extra small devices, the viewport will render TWO cells
            (12 / 6 = 2), so we need a clearfix every TWO cells. We also need to
            say "don't show this clearfix when the viewport will render FOUR cells",
            which it will do at medium size and up (12 / 3 = 4). We do that by adding
            hidden-md and hidden-lg to our clearfix div, in effect instructing the
            browser to not show it at all on a wider screen.
        -->
        <div class="clearfix hidden-md hidden-lg"></div>
        <!---->
        <div class="col-xs-6 col-md-3 cell-tall">3</div>
        <div class="col-xs-6 col-md-3">4</div>
        <!--
            We have now rendered FOUR cells.
            We are never going to have more than FOUR cells side by side. So every
            FOURTH cell, we place a clearfix that will ALWAYS show. We do this by
            just leaving off any of the hidden-* classes.
        -->
        <div class="clearfix"></div>
        <!---->
        <div class="col-xs-6 col-md-3">5</div>
        <div class="col-xs-6 col-md-3 cell-med">6</div>
        <!--
            We have now rendered SIX cells.
            After the sixth cell, we are at a multiple of TWO, but not FOUR so we
            repeat the clearfix that we used after cell TWO.
        -->
        <div class="clearfix hidden-md hidden-lg"></div>
        <!---->
        <div class="col-xs-6 col-md-3">7</div>
        <div class="col-xs-6 col-md-3">8</div>
        <!--
            Now we have rendered EIGHT cells, which is a multiple of TWO AND FOUR,
            so we put in a clearfix that's always visible.
        -->
        <div class="clearfix"></div>
        <!---->
        <div class="col-xs-6 col-md-3 cell-med">9</div>
        <div class="col-xs-6 col-md-3 cell-med">10</div>
        <!--
            After the 10th cell, once again a multiple of TWO but not FOUR...
        -->
        <div class="clearfix hidden-md hidden-lg"></div>
        <!---->
        <div class="col-xs-6 col-md-3">11</div>
    </div>
</div>

O clearfix é uma classe CSS que renderiza uma pequena (praticamente invisível) div, e sua finalidade é “limpar” os floats left que foram usados ​​pelas divs col.

A genialidade está realmente nas classes hidden-sm, hidden-md, etc. Essas classes são colocadas na div clearfix, NÃO nas divs col! Isso faz com que a div clearfix apareça ou desapareça magicamente do fluxo de renderização em determinadas larguras de viewport! Gênio!

Bootstrap tem uma variedade desconcertante de classes hidden-* e visible-* na versão 3, e infelizmente elas não são realmente o “inverso” uma da outra. Então, acho mais claro e seguro usar sempre as classes hidden-* nos clearfixes.

Parece que isso pode mudar para melhor no Bootstrap 4, com classes como hidden-*-up e hidden-*-down (eles estão se livrando inteiramente das classes visible-*).

Bem, chega de palavrões, como está agora? [![Clearfixed large][4]][4] [![Clearfixed small][5]][5]

É isso que queremos! Na tela grande, sempre temos QUATRO, na tela menor, sempre DOIS. Não há mais empilhamento em lugares estranhos, e as lacunas estão onde esperávamos que estivessem.

Por que as colunas do Bootstrap excederiam 12 em uma linha? #

Existem muitos cenários responsivos em que é necessário ter unidades de coluna superiores a 12 em um único elemento .row. Isso é conhecido como column wrap.

Se mais de 12 colunas forem colocadas em uma única linha, cada grupo de colunas extras serão, como uma unidade, agrupadas em uma nova linha.

Por exemplo, considere um layout onde queremos…

  • 3 colunas em dispositivos grandes e médios e
  • 2 colunas em dispositivos pequenos e menores

Para obter esse layout no Bootstrap, usaríamos (correto)..

<div class="row"> 
  <div class="col-xs-6 col-md-4"> x </div> 
  <div class="col-xs-6 col-md-4"> x </div> 
  <div class="col-xs-6 col-md-4"> x </div> 
  <div class="col-xs-6 col-md-4"> x </div> 
  <div class="col-xs-6 col-md-4"> x </div> 
  <div class="col-xs-6 col-md-4"> x </div> 
</div>

Demonstração do método correto

Como você vê no exemplo, o total de unidades de coluna no elemento .row excede 12. Essa técnica é conhecida como column wrap e é um dos recursos de design responsivo mais poderosos do Bootstrap. O layout desejado não seria possível (além de duplicar a marcação) se tentássemos manter o conceito errôneo comum de que as unidades de coluna devem somar 12 em uma única linha.

O layout não é possível quando não excedemos 12 (errado).

<div class="row"> 
  <div class="col-xs-6 col-md-4"> x </div> 
  <div class="col-xs-6 col-md-4"> x </div> 
</div>
<div class="row"> 
  <div class="col-xs-6 col-md-4"> x </div> 
  <div class="col-xs-6 col-md-4"> x </div> 
</div>
<div class="row"> 
  <div class="col-xs-6 col-md-4"> x </div> 
  <div class="col-xs-6 col-md-4"> x </div> 
</div>

Demonstração do método errado (falha em 3 colunas em grande)

Lembre-se, um .row não é o mesmo que uma única linha na janela de visualização. Um .row é um agrupamento de colunas. As colunas que excederem 12 unidades em um único elemento .row serão reunidos para uma nova linha (na janela de visualização). É por isso que é essencial entender que as 12 colunas representam unidades horizontais na janela de visualização.

Além disso, redefinições responsivas (clearfix) devem ser usadas para encapsulamento uniforme quando [as colunas variam em altura](https://www.wikiod.com/pt/docs/ /twitter-bootstrap/6124/using-clearfix-in-rows-and-cols/21310/the-height-problem#t=201702251436346575702).

2,4,6 Layout com Clearfixes #

Aqui está um layout que renderiza duas, quatro ou seis células dependendo do tamanho da tela.

<div class="container-fluid">
    <div class="row">
        <div class="col-xs-6 col-md-3 col-lg-2">1</div>
        <div class="col-xs-6 col-md-3 col-lg-2 cell-tall">2</div>
        <!--
            On small and extra small devices, the viewport will render TWO cells
            (12 / 6 = 2), so we need a clearfix every TWO cells. We also need to
            say "don't show this clearfix when the viewport will render FOUR cells",
            which it will do at medium size (12 / 3 = 4). We do that by adding
            hidden-md and hidden-lg to our clearfix div, in effect instructing the
            browser to not show it at all on a wider screen.
        -->
        <div class="clearfix hidden-md hidden-lg"></div>
        <!---->
        <div class="col-xs-6 col-md-3 col-lg-2 cell-tall">3</div>
        <div class="col-xs-6 col-md-3 col-lg-2">4</div>
        <!--
            After the FOURTH cell, we need a clearfix, but it still needs to be
            hidden on LARGE viewports, because remember we will have a maximum of
            SIX cells now.
        -->
        <div class="clearfix hidden-lg"></div>
        <!---->
        <div class="col-xs-6 col-md-3 col-lg-2">5</div>
        <div class="col-xs-6 col-md-3 col-lg-2 cell-med">6</div>
        <!--
            After the SIXTH cell, we need to show on SMALL and LARGE, but not on
            MEDIUM. Remember, our MEDIUM viewport only wants a clearfix when we
            are at a multiple of FOUR.
        -->
        <div class="clearfix hidden-md"></div>
        <!---->
        <div class="col-xs-6 col-md-3 col-lg-2">7</div>
        <div class="col-xs-6 col-md-3 col-lg-2">8</div>
        <!--
            Now we have rendered EIGHT cells, which is a multiple of TWO AND FOUR,
            so we put in a clearfix that's not visible on LARGE, because we are NOT
            at a multiple of SIX.
        -->
        <div class="clearfix hidden-lg"></div>
        <!---->
        <div class="col-xs-6 col-md-3 col-lg-2 cell-med">9</div>
        <div class="col-xs-6 col-md-3 col-lg-2 cell-med">10</div>
        <!--
            After the 10th cell, small only.
        -->
        <div class="clearfix hidden-md hidden-lg"></div>
        <!---->
        <div class="col-xs-6 col-md-3 col-lg-2">11</div>
    </div>
</div>

A primeira tentativa ingênua #

Antes de começarmos, vamos definir um pouco de CSS para os exemplos. Esta é a seção head do nosso exemplo. Eu sempre uso border-radius e background-color quando estou testando, porque simplifica a visualização das divisões de células sem adicionar nenhum tamanho de borda que possa afetar o tamanho das células.

<head>
    <title></title>
    <link rel="stylesheet" href="//maxcdn.bootstrapcdn.com/bootstrap/3.3.7/css/bootstrap.min.css">
    <style>
        /* colorize all col- */
        [class^="col-"] {
            min-height: 30px;
            border-radius: 10px;
            background-color: lightblue;
        }
        /* a tall cell */
        .cell-tall {
            height: 100px;
            background-color: orange;
        }
        /* a medium-height cell */
        .cell-med {
            height: 50px;
            background-color: lightgreen;
        }
        /* padding top-bottom for some row examples */
        .row.padded {
            padding: 1rem 0 1rem 0;
        }
    </style>
</head>

Com isso fora do caminho, vamos definir uma grade e observar os resultados perfeitos em todos os tamanhos de viewport!
