Instalação ou Configuração #

Instruções detalhadas sobre como configurar ou instalar o Kinect.Configurando um Sensor Kinect

Kinect para Windows 1.5, 1.6, 1.7, 1.8

Aqui estão alguns passos simples para colocar seu sensor Kinect em funcionamento.

Etapa 1: Monte o sensor em uma superfície estável

Coloque o sensor em uma superfície estável em um local onde não caia ou seja atingido durante o uso. Aqui estão algumas outras dicas.

Do not place the Kinect on or in front of a speaker or on a surface that vibrates or makes noise.
Keep the Kinect out of direct sunlight.
Use the Kinect within its specified operating temperature range of 41 to 95 degrees Fahrenheit (5 to 35 degrees Celsius). If the sensor is exposed to an environment outside of its prescribed temperature range, turn it off and allow the temperature to stabilize within the specified range before you use the sensor again.
Do not tilt a Kinect on its base. Manually tilting the Kinect can damage the sensor. The Kinect tilt angle is controlled by software; see Kinect Explorer - WPF C# Sample for an example.

Etapa 2: Instale o SDK do Kinect para Windows

If you haven't done so already, download and install the free SDK from the Kinect for Windows download page.
If you haven't done so already, download and install the free Developer Toolkit from the Kinect for Windows download page.

Etapa 3: Conecte seu sensor Kinect

Connect the power supply for your Kinect to an external power source.
Connect the Kinect to a USB port on your PC and wait for Windows to recognize it.
All the drivers, including audio, will load seamlessly.

Ao instalar o SDK do Kinect para Windows, o pacote de instalação inclui os drivers do Kinect. Quando os drivers do Kinect são instalados em um PC com Windows, um Kinect conectado à porta USB do PC aparece como um dispositivo USB multicomponente.

Há um problema conhecido em relação à compatibilidade do controlador host USB.

Depois de carregar os drivers, você precisa configurar seu ambiente de desenvolvimento. Se você estiver usando o Visual Studio, continue com Configurando o ambiente de desenvolvimento para ajudá-lo a criar seu primeiro aplicativo C# ou C++.

Dicas de soluções de problemas

As dicas a seguir ajudarão você a começar a usar o Kinect:

If a non-Microsoft driver for the Kinect is installed on your computer, the Kinect for Windows drivers might not install or function correctly. To fix this, uninstall the non-Microsoft drivers before installing the Kinect for Windows SDK.
Connect the power supply for the Kinect to an external power source; if the Kinect has only power from the USB connection, it will be minimally functional and light the LED, but it must be connected to an external power source to be fully functional.
No tools are required for calibration of audio and video.
Your Kinect should be the only device plugged into a USB hub on your computer. If you have more than one Kinect, connect them to different USB controllers. If 2 hubs are connected to the same controller, only 1 Kinect can work at a time.
The Kinect is protected from overheating by a fan. It is controlled by the sensor's firmware, which turns off the camera at 90 degrees Celsius. There is no software interface for applications to control the fan.
Reasonable lighting, neither extremely dark nor extremely bright, is important for capturing images with the RGB camera. Incandescent, fluorescent, and natural lighting provide no special obstacles, but do not point an intense or constant light source at the camera because this can blind the RGB sensor.
The depth sensor functions adequately in typical and reduced lighting, although in near darkness there is increased noise in the signal.
The depth sensor reads depth information from reflected light. Objects that are highly reflective (mirrors and shiny metal) or highly absorptive (fluffy and/or dark materials) may not be registered by the depth sensor as successfully as other objects.

Gratuito/Debian Pacotes oficiais

A partir do Ubuntu 11.10 (Oneiric) e Debian 7 (Wheezy), Ubuntu e Debian fornecem pacotes oficiais do libfreenect. Você pode instalá-los facilmente em um console:

$ sudo apt-get install freenect

No Ubuntu 12.04, o driver do kernel gspca impede que o libfreenect reivindique o dispositivo Kinect no modo de usuário. Remova e coloque na lista negra o módulo

$ sudo modprobe -r gspca_kinect 
$ sudo modprobe -r gspca_main
$ echo "blacklist gspca_kinect" |sudo tee -a /etc/modprobe.d/blacklist.conf

ou instale uma versão recente do libfreenect que pode desanexar automaticamente o driver do kernel adicionando Florian Echtler libtisch PPA (veja abaixo) com pacotes atualizados do libfreenect para o Ubuntu 12.04.

O dispositivo freenect é acessível a qualquer usuário pertencente ao grupo ‘plugdev’. Por padrão, um usuário de desktop pertence ao grupo plugdev, mas se você precisar adicioná-lo ao grupo:

$ sudo adduser $USER plugdev

depois saia e entre novamente Repositório NeuroDebian

Se você deseja uma versão recente do libfreenect, não importa qual versão do Debian ou Ubuntu você usa, backports da versão mais recente do libfreenect para todas as versões suportadas do Debian e do Ubuntu (ou seja, Ubuntu Lucid (10.04), Maverick (10.10), Natty (11.04) ), Oneiric (11.10) e Debian Squeeze e Wheezy no momento da escrita) estão disponíveis no repositório NeuroDebian . Os pacotes disponíveis lá são criados pelos mantenedores do pacote oficial do Debian e seguem os padrões do Debian/Ubuntu.

Para habilitar o repositório NeuroDebian:

$ wget -O- http://neuro.debian.net/lists/$(lsb_release -cs).us-nh | sudo tee /etc/apt/sources.list.d/neurodebian.sources.list
$ sudo apt-key adv --recv-keys --keyserver pgp.mit.edu 2649A5A9
$ sudo apt-get update

A instalação do libfreenect é a mesma de antes:

$ sudo apt-get install freenect

Certifique-se de que seu usuário pertença ao grupo plugdev (o padrão para um usuário de desktop) para acessar o dispositivo sem os privilégios de root. Se não for o caso, adicione-os por:

$ sudo adduser $USER plugdev

e sair e entrar novamente PPA da Líbia

Ubuntu Launchpad PP para Lucid (10.04), Maverick (10.10), Natty (11.04), Oneiric (11.10) e Pangolin (12.04) está disponível neste link

para usá-lo, abra um console e execute:

$ sudo add-apt-repository ppa:floe/libtisch
$ sudo apt-get update

Depois disso, você pode usar:

$ sudo apt-get install libfreenect libfreenect-dev libfreenect-demos

Isso instalará o libfreenect, os cabeçalhos de desenvolvimento e os aplicativos de demonstração.

Depois disso, você precisa se adicionar ao grupo ‘video’ e logar novamente. O pacote já inclui as regras necessárias para o daemon udev para que o acesso ao dispositivo seja possível para os usuários do vídeo do grupo.

$ sudo adduser $USER video

certifique-se de sair e entrar novamente. Você não precisa reiniciar, basta conectar o dispositivo kinect agora (se já estiver conectado, desconecte e conecte novamente).

Para iniciar os aplicativos de demonstração, use:

$ freenect-glview

Problemas ao acessar o dispositivo

Em caso de problemas, execute

$ lsusb | grep Xbox

que deve listar 3 dispositivos

> lsusb | grep Xbox                                                 Bus
> 001 Device 021: ID 045e:02ae Microsoft Corp. Xbox NUI Camera Bus 001
> Device 019: ID 045e:02b0 Microsoft Corp. Xbox NUI Motor Bus 001 Device
> 020: ID 045e:02ad Microsoft Corp. Xbox NUI Audio

Se eles não estiverem presentes, execute

echo -1 | sudo tee -a /sys/module/usbcore/parameters/autosuspend

e reconectar Kinekt e Kinect Camera devem ser listados

Você encontra todos os aplicativos de demonstração começando com o prefixo freenect-. Instalação manual do Ubuntu

Instruções rápidas de copiar e colar para começar a funcionar instantaneamente:

sudo apt-get install git-core cmake libglut3-dev pkg-config build-essential libxmu-dev libxi-dev libusb-1.0-0-dev
git clone git://github.com/OpenKinect/libfreenect.git
cd libfreenect
mkdir build
cd build
cmake ..
make
sudo make install
sudo ldconfig /usr/local/lib64/
sudo freenect-glview

Nota: Se você receber um erro dizendo que o apt-get não pode encontrar libglut3, você pode estar em uma versão mais recente do Ubuntu que tenha freeglut3-* em vez de libglut3-*, então sua instalação inicial do apt-get ficaria assim:

sudo apt-get install git-core cmake freeglut3-dev pkg-config build-essential libxmu-dev libxi-dev libusb-1.0-0-dev

Para usar o Kinect como usuário não root, faça o seguinte:

sudo adduser $USER video

Crie também um arquivo com regras para o gerenciador de dispositivos Linux:

sudo nano /etc/udev/rules.d/51-kinect.rules

Copiar e colar:

# ATTR{product}=="Xbox NUI Motor"
SUBSYSTEM=="usb", ATTR{idVendor}=="045e", ATTR{idProduct}=="02b0", MODE="0666"
# ATTR{product}=="Xbox NUI Audio"
SUBSYSTEM=="usb", ATTR{idVendor}=="045e", ATTR{idProduct}=="02ad", MODE="0666"
# ATTR{product}=="Xbox NUI Camera"
SUBSYSTEM=="usb", ATTR{idVendor}=="045e", ATTR{idProduct}=="02ae", MODE="0666"
# ATTR{product}=="Xbox NUI Motor"
SUBSYSTEM=="usb", ATTR{idVendor}=="045e", ATTR{idProduct}=="02c2", MODE="0666"
# ATTR{product}=="Xbox NUI Motor"
SUBSYSTEM=="usb", ATTR{idVendor}=="045e", ATTR{idProduct}=="02be", MODE="0666"
# ATTR{product}=="Xbox NUI Motor"
SUBSYSTEM=="usb", ATTR{idVendor}=="045e", ATTR{idProduct}=="02bf", MODE="0666"

Certifique-se de sair e entrar novamente.

Se você não consegue acessar ou ainda precisa de privilégios de root para usar seu dispositivo: em alguns casos pode haver conflitos entre as permissões de dois drivers instalados (libfreenect e primesense). Se este for o seu caso, tente reinstalar o driver do sensor do primesense e mantenha apenas o arquivo de regras do primesense /etc/udev/rules.d/55-primesense-usb.rules, removendo o arquivo /etc/udev/rules.d/51 -kinect.rules se você o criou.
