Carregando dados do ElasticSearch #

Antes de verificar a sintaxe específica, vamos dar uma olhada em como configurar seu ambiente para carregar os plugins necessários.

Configurar

Para carregar dados diretamente do ElasticSearch, você precisa baixar o plugin elasticsearch-hadoop. Você tem diferentes maneiras de fazê-lo funcionar, para uma configuração rápida, você pode fazer o seguinte.

Para fazê-lo funcionar, você precisa colocar o arquivo jar elasticsearch-hadoop-<versão>.jar em uma pasta do nó onde você instalou o servidor pig. No meu caso - bastante comum pesquisando - eu também precisava adicionar commons-httpclient-<version>.jar dentro dessa pasta.

Então você pode executar pig no modo shell (chamado grunt) simplesmente digitando pig no console. Então você tem que carregar esses dois jars da seguinte maneira:

REGISTER /path/to/jars/commons-httpclient-<version>.jar;
REGISTER /path/to/jars/elasticsearch-hadoop-<version>.jar;

Agora você está pronto para escrever algum código.

Exemplo

Vamos verificar a sintaxe para carregar dados de um caso complexo.

DATA = LOAD 'my_index/log' USING org.elasticsearch.hadoop.pig.EsStorage(
'es.nodes=https://server1:port1,https://server2:port2,https://server3:port3',
'es.query=?q=*',
'es.net.ssl=true',
'es.net.http.auth.user=user',
'es.net.http.auth.pass=pass',
'es.net.ssl.keystore.type=JKS',
'es.net.ssl.truststore.location=file:///path/to/truststore.jks',
'es.net.ssl.truststore.pass=pass');

Este é o exemplo completo, agora vamos analisá-lo passo a passo.

  • es.nodes contém a lista dos nós do seu cluster ElasticSearch. Você precisa especificar seus nós como uma lista separada por vírgulas, com a porta associada.
  • es.query contém a consulta que será enviada ao ElasticSearch para buscar dados. Você também pode colocar uma consulta no formato DSL, mas cuidado para que apenas a parte correspondente da consulta seja considerada! Se você tentar limitar o número de campos através da consulta DSL não vai funcionar: para conseguir isso você precisa usar o parâmetro es.read.source.filter. exemplo de uma consulta DSL: 'es.query = { "query":{ "match_all":{} } }'
  • es.net.ssl=true é autoexplicativo, você também precisa fornecer as credenciais de login ao ElasticSearch com es.net.http.auth.user e es.net.http.auth.pass.
  • es.net.ssl.keystore.type se você precisar de um truststore, você pode selecionar aqui o tipo. No parâmetro es.net.ssl.truststore.location você define a localização do arquivo, tenha cuidado para adicionar o prefixo file://, e no parâmetro es.net.ssl.truststore.pass você defina a senha do arquivo de armazenamento confiável.

Algumas configurações úteis

  • es.read.source.filter=field1,field2,field3 permite buscar apenas os campos especificados do ElasticSearch (neste exemplo três).
  • es.output.json=true permite que você busque dados em um formato de valor-chave (JSON). Definir como false retornará dados no formato CSV (padrão).
