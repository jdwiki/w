Instalação ou Configuração #

Linux #

Requisitos (r0.16.0)

Obrigatoriedade

De acordo com a documentação atual do Apache-Pig, ele suporta apenas os sistemas operacionais Unix e Windows.

  • Hadoop 0.23.X, 1.X ou 2.X
  • Java 1.6 ou versões posteriores instaladas e variável de ambiente JAVA_HOME definida para o diretório de instalação do Java

Opcional

  • Python 2.7 ou mais (UDFs Python)
  • Ant 1.8 (para builds)

Baixe a versão mais recente do Pig

Baixe a versão mais recente do porco em http://pig.apache.org/releases.html#Download

Instalação

mkdir Pig
cd Downloads/ 
tar zxvf pig-(latest-version).tar.gz 
tar zxvf pig-(latest-version).tar.gz 
mv pig-(latest-version).tar.gz/* /home/Pig/

Configuração

Depois de instalar o Apache Pig, temos que configurá-lo.

Abra o arquivo .bashrc

vim ~/.bashrc

No arquivo .bashrc, defina as seguintes variáveis ​​−

export PIG_HOME = /home/Pig
export PATH  = PATH:/home/Pig/bin

salve o arquivo e recarregue o bashrc novamente no ambiente usando

. ~/.bashrc

Verificando a versão do Pig

pig –version

Se a instalação for bem-sucedida, o comando acima exibirá o número da versão do Pig instalado.

Teste de instalação do porco

pig -h

Isso deve exibir todos os comandos possíveis associados ao porco

Seu porco agora está instalado localmente e você pode executá-lo usando parâmetros locais como

pig -x local

Conectando ao Hadoop

Se o Hadoop1.x ou 2.x estiver instalado no cluster e a variável de ambiente HADOOP_HOME estiver configurada.

você pode conectar o pig ao Hadoop adicionando a linha no .bashrc como antes

export PIG_CLASSPATH = $HADOOP_HOME/conf

Porco Correndo

Modos de Execução

Você pode executar o Pig usando o comando pig (bin/pig) ou executando o arquivo jar (java -cp pig.jar)

Os scripts PIG podem ser executados em 3 modos diferentes:

  • Modo local

      pig -x local ...

  • Modo Mapreduce (modo padrão)

      pig -x mapreduce ...
       (or)
  pig ...

  • Modo Tez Local

      pig -x tez ...

Modo Interativo

O Pig pode ser executado em modo interativo usando o shell Grunt. As instruções e comandos do Pig Latin podem ser inseridos interativamente neste shell.

Exemplo

$ pig -x <mode> <enter>
grunt>

Mode pode ser um dos modos de execução, conforme explicado na seção anterior.

Modo Lote

Pig também pode ser executado em modo batch. Aqui é fornecido um arquivo .pig contendo uma lista de comandos e instruções pig.

Exemplo

$ pig -x <mode> <script.pig>
grunt>

Da mesma forma, Mode pode ser um dos modos de execução, conforme explicado na seção anterior.

Exemplo de contagem de palavras em Pig #

Arquivo de entrada

Mary had a little lamb
its fleece was white as snow
and everywhere that Mary went
the lamb was sure to go.

Código de contagem de palavras do porco

-- Load input from the file named Mary, and call the single
-- field in the record 'line'.
input = load 'mary' as (line);

-- TOKENIZE splits the line into a field for each word.
-- flatten will take the collection of records returned by
-- TOKENIZE and produce a separate record for each one, calling the single
-- field in the record word.
words = foreach input generate flatten(TOKENIZE(line)) as word;

-- Now group them together by each word.
grpd = group words by word;

-- Count them.
cntd = foreach grpd generate group, COUNT(words);

-- Print out the results.
dump cntd;

O que é porco? #

O Pig fornece um mecanismo para executar fluxos de dados em paralelo no Hadoop. Inclui um linguagem, Pig Latin, para expressar esses fluxos de dados. Pig Latin inclui operadores para muitas das operações de dados tradicionais (juntar, classificar, filtrar, etc.), bem como a capacidade de usuários a desenvolver suas próprias funções para leitura, processamento e gravação de dados. Pig é um projeto de código aberto Apache. Isso significa que os usuários são livres para baixá-lo como fonte ou binário, use-o para si mesmo, contribua com ele e—sob os termos do Apache Licença — use-a em seus produtos e altere-a conforme acharem melhor.
