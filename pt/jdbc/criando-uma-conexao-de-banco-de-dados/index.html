Sintaxe #

  • DB_URL = “jdbc:DBMS://DB_HOST:DB_PORT/DB_NAME”

  • DBMS: Data Base Driver Manager, pode ser qualquer DBMS (mysql, oracle, postgresql, sqlite, …), exemplo de mysql: “com.mysql.jdbc.Driver”

  • DB_HOST: seu host base do banco de dados, o endereço IP do seu banco de dados exemplo: 10.6.0.1, o padrão é localhost ou 127.0.0.1

  • DB_PORT: Porta de banco de dados, todo DBMS tem uma porta de derrota exemplo mysql=3306, postegesql=5432

  • DB_NAME: o nome do seu banco de dados

  • Para conectar você deve obter uma referência ao objeto de classe,

  • Class.forName(DRIVER);

  • E para se conectar ao banco de dados, você precisa criar uma conexão

  • java.sql.Connection con = DriverManager.getConnection(DB_URL, DB_USER_NAME, DB_PASSWORD);

  • DB_USER_NAME: o nome de usuário do seu banco de dados

  • DB_PASSWORD : a senha do seu banco de dados

Criando uma conexão com o MySQL #

Para se conectar ao MySQL, você precisa usar o driver MySQL Connector/J. Você pode baixá-lo em http://dev.mysql.com/downloads/connector/j/ ou pode usar o Maven:

<dependency>
    <groupId>mysql</groupId>
    <artifactId>mysql-connector-java</artifactId>
    <version>5.1.39</version>
</dependency>

A URL JDBC básica para MySQL é:

jdbc:mysql://<hostname>[:<port>]/<database>[?<propertyName>=<propertyValue>[&<propertyName>=<propertyValue>]...]

Onde:

Chave Descrição Exemplo
<nome do host> Nome do host do servidor MySQL localhost
<porta> Porta do servidor MySQL (opcional, padrão: 3306) 3306
<banco de dados> Nome do banco de dados foobar
<nome da propriedade> Nome de uma propriedade da conexão useCompression
<valor da propriedade> Valor de uma propriedade de conexão verdadeiro

O URL suportado é mais complexo do que o mostrado acima, mas isso é suficiente para a maioria das necessidades ’normais'.

Para conectar use:

try (Connection connection = DriverManager.getConnection(
        "jdbc:mysql://localhost/foobardb", "peter", "nicepassword")) {
    // do something with connection
}

Para versões mais antigas de Java/JDBC:

// Load the MySQL Connector/J driver
Class.forName("com.mysql.jdbc.Driver");

Connection connection = DriverManager.getConnection(
        "jdbc:mysql://localhost/foobardb", "peter", "nicepassword");
try {
    // do something with connection
} finally {
  // explicitly close connection
  connection.close();
}

Conexão com um banco de dados Microsoft Access com UCanAccess #

UCanAccess é um driver Java JDBC puro que nos permite ler e gravar em bancos de dados Access sem usar ODBC. Ele usa dois outros pacotes, Jackcess e HSQLDB, para realizar essas tarefas.

Uma vez configurado*, podemos trabalhar com dados em arquivos .accdb e .mdb usando um código como este:

import java.sql.*;

Connection conn=DriverManager.getConnection("jdbc:ucanaccess://C:/__tmp/test/zzz.accdb");
Statement s = conn.createStatement();
ResultSet rs = s.executeQuery("SELECT [LastName] FROM [Clients]");
while (rs.next()) {
    System.out.println(rs.getString(1));
}

*Para mais detalhes, veja a seguinte pergunta:

http://stackoverflow.com/q/21955256/2144390

Criando uma conexão usando java.sql.DriverManager #

Para conectar usando java.sql.DriverManager você precisa de uma url JDBC para se conectar ao seu banco de dados. Os URLs JDBC são específicos do banco de dados, mas todos têm o formato

jdbc:<subprotocol>:<subname>

Onde <subprotocol> identifica o driver ou banco de dados (por exemplo postgresql, mysql, firebirdsql, etc), e <subname> é específico do subprotocolo.

Você precisa verificar a documentação do seu banco de dados e do driver JDBC para o subprotocolo de url específico e o formato do seu driver.

Um exemplo simples para criar uma conexão com um banco de dados com a url jdbc:somedb://localhost/foobar:

try (Connection connection = DriverManager.getConnection(
        "jdbc:somedb://localhost/foobar", "anna", "supersecretpassword")) {
    // do something with connection
}

Usamos um [try-with-resources][3] aqui para que a conexão seja fechada automaticamente quando terminarmos, mesmo que ocorram exceções.

No Java 6 (JDBC 4.0) e anterior, try-with-resources não está disponível. Nessas versões, você precisa usar um bloco finally para fechar explicitamente uma conexão:

Connection connection = DriverManager.getConnection(
        "jdbc:somedb://localhost/foobar", "anna", "supersecretpassword");
try {
    // do something with connection
} finally {
    // explicitly close connection
    connection.close();
}

O JDBC 4.0 (Java 6) introduziu o conceito de carregamento automático de driver. Se você usa Java 5 ou anterior, ou um driver JDBC mais antigo que não implementa suporte a JDBC 4, você precisará carregar explicitamente o(s) driver(s):

Class.forName("org.example.somedb.jdbc.Driver");

Essa linha precisa ocorrer (pelo menos) uma vez em seu programa, antes que qualquer conexão seja feita.

Mesmo em Java 6 e superior com um JDBC 4.0 pode ser necessário carregar explicitamente um driver: por exemplo em aplicações web quando o driver não é carregado no container, mas como parte da aplicação web.

Alternativamente, você também pode fornecer um objeto Properties para conectar:

Properties props = new Properties();
props.setProperty("user", "anna");
props.setProperty("password", "supersecretpassword");
// other, database specific, properties
try (Connection connection = DriverManager.getConnection(
        "jdbc:somedb://localhost/foobar", props)) {
    // do something with connection
}

Ou mesmo sem propriedades, por exemplo, se o banco de dados não precisar de nome de usuário e senha:

try (Connection connection = DriverManager.getConnection(
        "jdbc:somedb://localhost/foobar")) {
    // do something with connection
}

[3]: https://www.wikiod.com/pt/java/excecoes-e-tratamento-de-excecoes#A instrução try-with-resources

Conexão Oracle JDBC #

Condutor: #

  • 12c R1
  • 11g R2

(Nota: o driver não está incluído no Maven Central!)

Inicialização da classe do driver: #

    Class.forName("oracle.jdbc.driver.OracleDriver");

URL de conexão #

Formato antigo, com SID

"jdbc:oracle:thin:@<hostname>:<port>:<SID>"

Formato mais recente, com nome do serviço

"jdbc:oracle:thin:@//<hostname>:<port>/<servicename>"

Tnsnames like entry

"jdbc:oracle:thin:@(DESCRIPTION=(ADDRESS=(PROTOCOL=TCPS)(HOST=<hostname>)(PORT=<port>))"
    +"(CONNECT_DATA=(SERVICE_NAME=<servicename>)))"

Cadeia de conexão do cluster RAC para failover

"jdbc:oracle:thin:@(DESCRIPTION=(ADDRESS_LIST=(LOAD_BALANCE=OFF)(FAILOVER=ON)"
    +"(ADDRESS=(PROTOCOL=TCP)(HOST=<hostname1>)(PORT=<port1>))"
    +"(ADDRESS=(PROTOCOL=TCP)(HOST=<hostname2>)(PORT=<port2>)))"
    +"(CONNECT_DATA=SERVICE_NAME=<servicename>)(SERVER=DEDICATED)))"

Exemplo #

connection = DriverManager.getConnection("jdbc:oracle:thin:@localhost:1521:orcl", "HR", "HRPASS");
