Matrizes #

JSON:

[
  {
    "id": 8484,
    "name": "David",
    "height": 173.2,
    "weight": 75.42
  },
  {
    "id": 8485,
    "name": "Ronald",
    "height": 183.73,
    "weight": 83.1
  }
]

Pessoa.java

public class Person {
    public int id;
    public String name;
    public double height;
    public double weight;

    @Override
    public String toString() {
        return "[ id: " + String.valueOf(id) + ", name: " + name + ", height: " + String.valueOf(height) + ", weight: " + String.valueOf(weight) + " ]";
    }
}

Uso:

Gson gson = new Gson();
Person[] persons = gson.fromJson(json, Person[].class);
for(Person person : persons)
    System.out.println(person.toString());

Resultado:

[ id: 8484, name: David, height: 173.2, weight: 75.42 ]
[ id: 8485, name: Ronald, height: 183.73, weight: 83.1 ]

Exemplo Simples #

A biblioteca Gson fornece Gson.class que trata de todas as conversões entre objetos Java e JSON. Uma instância dessa classe pode ser criada invocando o construtor padrão. Você normalmente gostaria de ter uma instância Gson para a maior parte das operações em seu programa.

Gson gson = new Gson();

Primeiro, precisamos criar a classe do nosso objeto com o qual trabalharemos

class Person {
   public String name;
   public int age;

   public Person(String name, int age){
       this.name = name;
       this.age = age;
   }
}

A classe Gson fornece os métodos toJson e fromJson que são os principais pontos de entrada para objetos JSON e java

Vamos tentar converter o objeto java para JSON e voltar para o objeto java

Person person = new Person("Jason", 29);
//using gson object which we created earlier
String json = gson.toJson(person);
System.out.println(json);
//Outputs: {"name": "Jason", "age": 29}

E agora de volta novamente

String json = "{\"name\": \"Jason\", \"age\": 29}";
Person person = gson.fromJson(json, Person.class);
System.out.println(person.age + "yo " + person.name + " walks into a bar");
//Outputs "29 yo Jason walks into a bar"

Instalação #

Para usar o Gson, você deve incluí-lo em seu projeto. Você pode fazer isso adicionando a seguinte dependência da versão Gson disponível no Maven Central:

Maven

Adicionar ao pom.xml

<dependencies>
    <dependency>
      <groupId>com.google.code.gson</groupId>
      <artifactId>gson</artifactId>
      <version>2.8.0</version>
      <scope>compile</scope>
    </dependency>
</dependencies>

Gradle:

Adicionar a build.gradle

compile 'com.google.code.gson:gson:2.8.0'

Converta String para JsonObject sem POJO #

String jsonStr = "{\"name\" : \"Abcd\", \"greeting\": \"Hello\", }"; //Sample Json String

Gson gson = new Gson(); // Creates new instance of Gson
JsonElement element = gson.fromJson (jsonStr, JsonElement.class); //Converts the json string to JsonElement without POJO 
JsonObject jsonObj = element.getAsJsonObject(); //Converting JsonElement to JsonObject

String name = jsonObj.get("name").getAsString(); //To fetch the values from json object
String greeting = jsonObj.get("greeting").getAsString();

Serialização e desserialização #

Gson gson = new Gson(); //Create a Gson object
MyType target = new MyType(); //This is the object you want to convert to JSON   
String json = gson.toJson(target); // serializes target to Json
MyType target2 = gson.fromJson(json, MyType.class); // deserializes json into target2

Usando GSON com herança #

O GSON não suporta herança fora da caixa. Digamos que temos a seguinte hierarquia de classes:

public class BaseClass {
    int a;
 
    public int getInt() {
        return a;
   }
}
 
public class DerivedClass1 extends BaseClass {
     int b;
 
     @Override
     public int getInt() {
         return b;
     }
 }
 
public class DerivedClass2 extends BaseClass {
    int c;
 
    @Override
    public int getInt() {
        return c;
    }
}

E agora queremos serializar uma instância de DerivedClass1 para uma string json

DerivedClass1 derivedClass1 = new DerivedClass1();
derivedClass1.b = 5;
derivedClass1.a = 10;
 
Gson gson = new Gson();
String derivedClass1Json = gson.toJson(derivedClass1);

Agora, em outro lugar, recebemos esta string json e queremos desserializá-la - mas em tempo de compilação sabemos apenas que deve ser uma instância de BaseClass:

BaseClass maybeDerivedClass1 = gson.fromJson(derivedClass1Json, BaseClass.class);
System.out.println(maybeDerivedClass1.getInt());

Mas o GSON não sabe que derivedClass1Json era originalmente uma instância de DerivedClass1, então isso imprimirá 10.

Como resolver isso?

Você precisa construir seu próprio JsonDeserializer, que lida com esses casos. A solução não é perfeitamente limpa, mas não consegui encontrar uma melhor.

Primeiro, adicione o seguinte campo à sua classe base

@SerializedName("type")
private String typeName;

E inicialize-o no construtor da classe base

public BaseClass() {
    typeName = getClass().getName();
}

Agora adicione a seguinte classe:

public class JsonDeserializerWithInheritance<T> implements JsonDeserializer<T> {
 
 @Override
 public T deserialize(
     JsonElement json, Type typeOfT, JsonDeserializationContext context)
     throws JsonParseException {
     JsonObject jsonObject = json.getAsJsonObject();
     JsonPrimitive classNamePrimitive = (JsonPrimitive) jsonObject.get("type");
 
     String className = classNamePrimitive.getAsString();
 
     Class<?> clazz;
     try {
     clazz = Class.forName(className);
     } catch (ClassNotFoundException e) {
     throw new JsonParseException(e.getMessage());
     }
     return context.deserialize(jsonObject, clazz);
 }
}

Tudo o que resta a fazer é conectar tudo -

GsonBuilder builder = new GsonBuilder();
 builder
 .registerTypeAdapter(BaseClass.class, new JsonDeserializerWithInheritance<BaseClass>());
 Gson gson = builder.create();

E agora, executando o seguinte código-

 DerivedClass1 derivedClass1 = new DerivedClass1();
 derivedClass1.b = 5;
 derivedClass1.a = 10;
 String derivedClass1Json = gson.toJson(derivedClass1);
 
 BaseClass maybeDerivedClass1 = gson.fromJson(derivedClass1Json, BaseClass.class);
 System.out.println(maybeDerivedClass1.getInt());

Irá imprimir 5.
