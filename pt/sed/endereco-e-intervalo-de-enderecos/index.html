  • Linhas que correspondem ao padrão de expressão regular

Os comandos Sed podem ser especificados para atuar apenas em determinadas linhas usando endereços ou intervalos de endereços.

Linhas que correspondem ao padrão de expressão regular #

$ cat ip.txt 
address
range
substitution
pattern
sample
Add Sub Mul Div

  • Linhas que correspondem a um padrão

    $ sed '/add/d' ip.txt 
range
substitution
pattern
sample
Add Sub Mul Div

$ sed -n '/t/p' ip.txt 
substitution
pattern

$ sed -n '/[A-Z]/ s| |/|gp' ip.txt 
Add/Sub/Mul/Div

  • Gama de padrões

    $ sed -n '/add/,/sub/p' ip.txt 
address
range
substitution

$ sed -n '/a/,/e/p' ip.txt 
address
range
pattern
sample

Observação

  • No segundo exemplo, ele correspondeu a dois intervalos - linhas 1,2 e linhas 4,5
  • Consulte [Usando delimitadores diferentes](https://www.wikiod.com/pt/sed/expressoes-regulares#Usando delimitadores diferentes) sobre como usar outros caracteres em vez de / para especificar o padrão

  • Correspondência sem distinção entre maiúsculas e minúsculas

    $ sed -n '/add/Ip' ip.txt 
address
Add Sub Mul Div

$ sed -n '/add/I,/sub/p' ip.txt 
address
range
substitution
Add Sub Mul Div

Linha específica #

$ cat ip.txt
address
range
substitution
pattern
sample

  • N linha

    $ sed -n '2p' ip.txt 
range

$ sed '3d' ip.txt 
address
range
pattern
sample

  • Última linha

    $ sed -n '$p' ip.txt 
sample

Faixa específica de linhas #

$ cat ip.txt 
address
range
substitution
pattern
sample

  • O intervalo especificado inclui esses números de linha

    $ sed -n '2,4p' ip.txt 
range
substitution
pattern

  • $ pode ser usado para especificar a última linha. O espaço pode ser usado entre o endereço e o comando para maior clareza

    $ sed -n '3,$ s/[aeiou]//gp' ip.txt 
sbstttn
pttrn
smpl

  • iésima linha até i+jésima linha

    $ sed '2,+2d' ip.txt 
address
sample

  • iésima linha e i+j, i+2j, i+3j, etc.

    $ sed -n '1~2p' ip.txt 
address
substitution
sample

Especificando o intervalo usando o número e o padrão #

$ cat ip.txt 
address
range
substitution
pattern
sample
Add Sub Mul Div

  • Número de linha para padrão de correspondência de linha

    $ sed -n '2,/pat/p' ip.txt 
range
substitution
pattern

  • Padrão de correspondência de linha com o número da linha

    $ sed '/pat/,$d' ip.txt 
address
range
substitution

  • Padrão de correspondência de linha mais o número de linhas que o seguem

    $ sed -n '/add/I,+1p' ip.txt 
address
range
Add Sub Mul Div

  • 0 pode ser usado como número de linha inicial para sinalizar o fim do intervalo quando o padrão corresponder à primeira linha de entrada

    $ sed -n '0,/r/p' ip.txt 
address

$ sed -n '1,/r/p' ip.txt 
address
range

$ sed -n '0,/u/p' ip.txt 
address
range
substitution

Negando o intervalo de endereços #

$ cat ip.txt 
address
range
substitution
1234
search pattern
sample
Add Sub Mul Div

  • Excluindo linhas diferentes do endereço especificado

    $ sed '/[0-9]/!d' ip.txt 
1234

$ sed -n '/[0-9]/p' ip.txt 
1234

$ sed '$!d' ip.txt 
Add Sub Mul Div

$ sed -n '$p' ip.txt 
Add Sub Mul Div

  • Pesquise e substitua em linhas que não correspondem a um padrão

    $ sed '/ /! s/^/#/' ip.txt 
#address
#range
#substitution
#1234
search pattern
#sample
Add Sub Mul Div

$ sed '/add/,/sub/! s/[aeiou]//gi' ip.txt 
address
range
substitution
1234
srch pttrn
smpl
dd Sb Ml Dv
