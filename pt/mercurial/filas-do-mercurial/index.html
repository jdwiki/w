Sintaxe #

  • hg qnew -m “Minha mensagem de confirmação” myPatch
  • hg qpop
  • hg qpush
  • hg qrefresh -m “Minha nova mensagem de confirmação”
  • hg qaplicado
  • hg qseries
  • hg qfinish
  • hg qdelete myPatch
  • hg qfold myPatch
  • hg qqueue –list
  • hg qqueue –create myNewQueue
  • hg qqueue –delete myNewQueue

Ativar extensão #

Edite Mercurial.ini (Windows) ou .hgrc (Linux/OSX):

[extenstions]
mq =

Criar e atualizar patches #

Crie novos patches com hg qnew patch-name e atualize-os com novas alterações com hg qrefresh:

hg qnew myFirstPatch  // Creates a new patch called "myFirstPatch"
...                   // Edit some files
hg qrefresh           // Updates "myFirstPatch" with changes
hg qnew mySecondPatch // Creates a new patch called "mySecondPatch" on top of "myFirstPatch"
...                   // Edid some files
hg qrefresh           // Updates "mySecondPatch" with changes

Crie um novo patch com a mensagem de commit:

hg qnew -m "My first patch" myFirstPatch

Atualizar mensagem de confirmação do patch atual:

hg qrefresh -m "My new message"

Atualize a mensagem de confirmação (multilinha) do patch atual:

hg qrefresh -e

Patches push e pop #

O envio de um patch aplica o patch ao repositório, enquanto o envio de um patch remove o patch do repositório.

Retire o patch atual da fila com hg qpop e empurre-o de volta para a fila com hg qpush:

hg qpop
hg qpush

Pop todos os patches:

hg qpop -a

Empurre todos os patches:

hg qpush -a

Concluir Patches #

O acabamento de patches os torna conjuntos de alterações permanentes.

Conclua o primeiro patch aplicado na fila:

hg qfinish

Conclua todos os patches aplicados na fila:

hg qfinish -a

Raposa #

Para fazer o rebase com os conjuntos de alterações mais recentes no repositório upstream:

hg qpop -a    // Pop all patches
hg pull -u    // Pull in changesets from upstream repo and update
hg qpush -a   // Push all patches on top of new changesets

Se houver algum conflito, você será forçado a mesclar seus patches.

Dobrar/Combinar Patches #

Para dobrar (combinar) dois patches:

hg qnew firstPatch   // Create first patch
hg qnew secondPatch  // Create second patch
hg qpop              // Pop secondPatch
hg qfold secondPatch // Fold secondPatch with firstPatch

Caudas Múltiplas #

Mais de uma fila pode ser criada. Cada fila pode ser considerada como uma ramificação separada.

hg qqueue --create foo // Create a new queue called "foo"
hg qqueue --list       // List all queues
hg qqueue --active     // Print name of active queue
hg qqueue --rename bar // Rename active queue "foo" to "bar"
hg qqueue --delete bar // delete queue "bar"

Crie duas filas e alterne entre elas:

hg qqueue --create foo // Create a new queue called "foo" and make it active
hg qqueue --create bar // Create a new queue called "bar" and make it active
hg qqueue foo          // Switch back to queue "foo"

Comandos #

  • qnew: crie um novo patch
  • qpop: remove o patch atual da pilha
  • qpush: coloca o próximo patch na pilha
  • qrefresh: atualiza o patch atual
  • qapplied: imprime os patches já aplicados
  • qseries: imprime todo o arquivo da série
  • qfinish: mova os patches aplicados para o histórico do repositório
  • qdelete: remove patches da fila
  • qdiff: diferença do patch atual e modificações subsequentes
  • qclone: clone principal e repositório de patches ao mesmo tempo
  • qfold: dobre os patches nomeados no patch atual
  • qgoto: push ou pop patches até que o patch nomeado esteja no topo da pilha
  • qguard: defina ou imprima guardas para um patch
  • qheader: imprime o cabeçalho do patch mais alto ou especificado
  • qimport: importa um patch ou changeset existente
  • qnext: imprime o nome do próximo patch que pode ser enviado
  • qprev: imprime o nome do patch aplicado anteriormente
  • qqueue: gerencie várias filas de patches
  • qrecord: grava interativamente um novo patch
  • qrename: renomear um patch
  • qselect: defina ou imprima patches em quadra para enviar
  • qtop: imprime o nome do patch atual
  • qunapplied: imprime os patches ainda não aplicados

Descontinuada:

  • qinit: inicia um novo repositório de filas (OBSOLETO)
  • qcommit: confirma alterações no repositório de filas (OBSOLETO)
  • qsave: salva o estado atual da fila (OBSOLETO)
  • qsave: restaura o estado da fila salvo por uma revisão (OBSOLETO)
