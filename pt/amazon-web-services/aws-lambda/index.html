O AWS Lambda é um serviço que permite executar código de back-end sem a necessidade de provisionar ou gerenciar servidores. O AWS Lambda cuida do dimensionamento e da alta disponibilidade. O custo depende diretamente da frequência e do tempo de execução do seu código.

Você encontrará exemplos de como criar e implantar funções do AWS Lambda em diferentes idiomas.

  • O código AWS Lambda deve ser escrito sem estado. Embora a instância de um lambda possa ser mantida e reutilizada, você nunca deve esperar isso.

Projeto Java básico do Gradle #

Para implantar o código Java no AWS Lambda, você precisa criar um arquivo zip de distribuição que contenha todas as dependências necessárias durante o tempo de execução. Exemplo de projeto em Gradle:

apply plugin: 'java'

repositories {
    mavenCentral()
}

dependencies {
    compile 'com.amazonaws:aws-lambda-java-core:1.1.0'
}

task buildZip(type: Zip) {
    from compileJava
    from processResources
    into('lib') {
        from configurations.runtime
    }
}

build.dependsOn buildZip

A execução do gradle build criará um arquivo zip com todas as dependências empacotadas com seu código, pronto para implantação.

Código Lambda básico em Java #

Um lambda precisa de um manipulador, que servirá como ponto de entrada para seu aplicativo. Todo manipulador precisa implementar a interface RequestHandler<I, O> onde I é o tipo de entrada e O é o tipo de saída. O tipo de entrada é então passado para o método handleRequest() e o método retorna o tipo de saída.

Um exemplo simples poderia ser assim:

package com.example.lambda;

import com.amazonaws.services.lambda.runtime.Context;
import com.amazonaws.services.lambda.runtime.RequestHandler;

public class HelloNumber implements RequestHandler<Integer, String> {

    @Override
    public String handleRequest(Integer input, Context context) {
        return "You sent " + input;
    }
}

Código Lambda básico em JavaScript (NodeJs) #

Um lambda precisa de um manipulador, que servirá como ponto de entrada para seu aplicativo. No caso mais simples, você obtém sua entrada do context e passa o resultado usando a função callback():

export.handler = (evento, contexto, retorno de chamada) => { callback(null, ‘You sent ’ + event.number); };

Criando o AWS Lambda usando a GUI da web (Java) #

Para criar um lambda Java usando a GUI, primeiro certifique-se de ter seu zip de distribuição pronto. Você também precisará de uma conta da AWS que possa criar lambdas.

  • Mude para o AWS Lambda, saia da tela de introdução (ou leia a documentação Get Started) e pressione o botão Create a Lambda Function.
  • Selecione um projeto apropriado. Um blueprint é um modelo de exemplo que pode ajudá-lo a criar seu lambda. Por enquanto, comece com uma Função em branco.
  • Por enquanto, não se preocupe com gatilhos e pule essa tela.
  • Dê um nome à sua função e carregue o arquivo zip.
  • Na mesma tela de configuração, digite o nome do seu Handler como classe Java totalmente qualificada com pacote, opcionalmente com o método que irá tratar as requisições. Se você usou a interface Java do exemplo Basic Lambda Code, o nome da classe é suficiente.
  • Na mesma tela, selecione Criar uma nova função a partir do(s) modelo(s) e nomeie sua função. A função é usada para dar ao seu lambda acesso a outros recursos da AWS. Nesse caso, deixe-o vazio, pois não precisamos de outros recursos.
  • Selecione a memória mais baixa disponível (128 MB). A memória influencia o custo também. Salve sua função.

Testando o código Lambda básico #

Se você implantou o Lambda com êxito, também poderá testá-lo diretamente na GUI. Quando você clica no botão azul Test pela primeira vez, ele apresenta a criação de um novo evento de teste.

Se você estiver testando o código Java do Código Lambda básico no exemplo Java, exclua todo o corpo e simplesmente adicione um número 1.

Se você estiver testando o código Javascript do exemplo Basic Lambda code in Javascript (Node), crie um json parecido com este:

{
    "number": "1"
}

Execute o teste. Você deve ver o resultado:

“Você enviou 1”

Adicionando o gatilho do AWS API Gateway #

Provavelmente, a maneira mais comum de invocar um Lambda é usar o API Gateway, que mapeia uma chamada REST para o código lambda. Você pode adicionar vários gatilhos ao Lambda a qualquer momento, inclusive ao criar um novo lambda.

Se você estiver familiarizado com o API Gateway, poderá associar qualquer método a um lambda implantado. Basta criar um novo método com o tipo de integração definido para a função Lambda e apontar para seu lambda. Você pode mapear ambas as entradas REST para entradas lambda e saídas para saídas REST.
