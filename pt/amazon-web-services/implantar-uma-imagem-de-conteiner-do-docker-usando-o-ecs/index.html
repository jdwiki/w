Implantar uma imagem de contêiner do Docker usando o ECS

  • Implante um aplicativo de amostra no serviço AWS ECS como prova de conceito
  • exemplo-tarefa.json

Nesta página

  • Implante um aplicativo de amostra no serviço AWS ECS como prova de conceito
  • exemplo-tarefa.json

Antes de adicionar instâncias do ECS a um cluster, você deve primeiro acessar o EC2 Management Console e criar instâncias ecs-optimized com uma função do IAM que tenha a política AmazonEC2ContainerServiceforEC2Role anexada.

  1. Vá para o seu EC2 Dashboard e clique no botão Launch Instance.

  2. Em Community AMIs, procure por ecs-optimized e selecione aquele que melhor atende às necessidades do seu projeto. Qualquer vai funcionar. Clique em próximo.

  3. Quando chegar a Configure Instance Details, clique no create new IAM role link e crie uma nova função chamada ecsInstanceRole.

  4. Anexe a política AmazonEC2ContainerServiceforEC2Role a essa função.

  5. Por padrão, sua instância de contêiner é iniciada em seu cluster default. Se você deseja iniciar em seu próprio cluster em vez do padrão, escolha a lista Detalhes avançados e cole o script a seguir no campo Dados do usuário, substituindo your_cluster_name pelo nome do seu cluster.

    #!/bin/bash echo ECS_CLUSTER=your_cluster_name » /etc/ecs/ecs.config

  6. Em seguida, termine de configurar sua instância do ECS.

NOTA: Se você estiver criando um servidor web, você desejará criar um securityGroup para permitir o acesso à porta 80.

  1. Crie um repositório: aws ecr create-repository --repository-name example-repository
  2. Autentique seu cliente Docker em seu registro: aws ecr get-login --region us-east-1 | sh
  3. Crie sua imagem do Docker: docker build -t example-image .
  4. Marque sua imagem para que você possa enviar a imagem para este repositório: tag docker example-image:latest example-namespace/example-image:latest
  5. Envie esta imagem para seu repositório AWS recém-criado: docker push example-namespace/example-image:latest
  6. Registre uma definição de tarefa do ECS: aws ecs register-task-definition --cli-input-json example-task.json
  7. Execute a tarefa: aws ecs run-task --task-definition example-task

Implante um aplicativo de amostra no serviço AWS ECS como prova de conceito #

Siga as etapas a seguir para experimentar um aplicativo de amostra no serviço AWS ECS como prova de conceito.

  1. Faça login no console de gerenciamento da AWS e vá para Catálogo de serviços da AWS - > Computação - > Ec2

  2. Crie uma VM (instância EC2) usando o sistema operacional Amazon Linux de 64 bits, que usaremos para configurar o docker, o git, a ferramenta de agente do AWS ECS e outras ferramentas. Também usaremos a mesma VM como um nó no cluster do ECS para implantar aplicativos baseados em contêiner. Siga as etapas abaixo para criar uma VM. a) Siga as etapas usuais para criar uma instância do EC2, dê ênfase especial nas etapas subsequentes durante a criação da instância do EC2. b) Selecione uma função do IAM com menos permissões a seguir – AmazonEC2ContainerServiceforEC2Role c) Certifique-se de que o java esteja instalado na VM

  3. Instalando o docker [execute os comandos abaixo] primeiro atualize o repositório de pacotes yum

    sudo yum update –y agora para instalar o docker execute yum install

    sudo yum install -y docker

  4. Inicie o serviço docker

    sudo service docker start

  5. Adicione o usuário ec2 ao grupo docker para que você possa executar comandos do Docker sem usar sudo.

    sudo usermod -a -G docker ec2-user

  6. Faça logout do EC2 e faça login novamente para obter as novas permissões do grupo docker.

  7. Verifique se o usuário do ec2 pode executar comandos do Docker sem sudo.

    docker info

  8. Instalando o Git

    sudo yum install -y git

  9. Clone o aplicativo PHP de amostra na instância Ec2 do git. Usaremos este aplicativo para nosso POC.

    git clone https://github.com/awslabs/ecs-demo-php-simple-app

    cd ecs-demo-php-simple-app

verifique se o Dockerfile existe listando o conteúdo do diretório

ls
  1. Vá para o catálogo de serviços da AWS -> Compute -> Ec2 Container Service
  2. Clique em Começar
  3. Clique em cancelar
  4. Clique em repositórios no menu Repositórios à esquerda
  5. Clique em Começar

15. Digite o nome do repositório e clique em próximo 16. Configurar ferramentas Ec2

aws configure

forneça o ID da chave de acesso da AWS, a chave de acesso secreta, o nome da região padrão de acordo com sua conta 17. Crie, marque e envie a imagem do Docker a) Recupere o comando docker login que você pode usar para autenticar seu cliente Docker em seu registro:

aws ecr get-login --region us-east-1

b) Execute o comando return como saída do passo anterior 18. Construa a imagem do Docker a partir de seu Dockerfile. (Lembre-se da Etapa 9, onde você baixou um aplicativo docker de amostra) a)

docker build -t amazon-ecs-sample .

(Observe que “.” significa diretório atual) b) Execute as imagens do docker para verificar se a imagem foi criada corretamente e se o nome da imagem contém um repositório no qual você pode enviar suas alterações para a imagem do docker

docker images

c) Execute a imagem recém-criada. A opção -p 80:80 mapeia a porta exposta 80 no contêiner para a porta 80 no sistema host (instância Ec2 neste caso).

docker run -p 80:80 amazon-ecs-sample

Ignore o aviso “apache2: não foi possível determinar com segurança o nome de domínio totalmente qualificado do servidor, usando 172.17.0.2 para ServerName” 19. Tente acessar a página da Web do aplicativo de amostra no navegador, certifique-se de que a porta 80 esteja aberta nos grupos de segurança associados à instância

http://<ec2-instance-dns-address>

  1. Pressione a tecla ctrl + c, isso interromperá a imagem do docker. O aplicativo de amostra não deve ser acessível.

  2. Agora, depois de verificar com sucesso nosso aplicativo docker de amostra, tentaremos configurar um cluster para executar o aplicativo de amostra automaticamente. Além disso, para fins de demonstração, tentaremos usar a instância ec2 existente como um nó no cluster. Isso pode ser feito instalando um programa de agente na instância do ec2.

  3. Instalando o Amazon ECS Container Agent na instância ec2 a)

    sudo yum install -y ecs-init

b) Reinicie o daemon do docker

sudo service docker restart

c) Inicie o trabalho inicial do ecs-init

sudo start ecs

d) (Opcional) Você pode verificar se o agente está em execução e ver algumas informações sobre sua nova instância de contêiner com a API de introspecção do agente. Certifique-se de que a porta 51678 esteja aberta no grupo de segurança.

curl http://localhost:51678/v1/metadata

  1. Vá para AWS service catalog -> Compute -> Ec2 Container Service -> Cluster e verifique se um cluster padrão foi criado

  2. Agora continuamos criando um grupo de tarefas e adicionando nossa imagem docker como tarefa a ser executada no cluster a) Examine o arquivo simple-app-task-def.json na pasta ecs-demo-php-simple-app. b) Edite o simple-app-task-def.json e reduza o momeory, para que ele possa ser executado na instância elegível de nível gratuito (suponho que alguém esteja usando a instância ec2 qualificada de nível gratuito para este POC, caso contrário, não há necessidade de reduzir a memória limite) c) Atualize memory=250 em todas as ocorrências no arquivo simple-app-task-def.json d) Registre uma definição de tarefa com o arquivo simple-app-task-def.json.

    aws ecs register-task-definition –cli-input-json file://simple-app-task-def.json

e) Vá para a definição de tarefa na página de serviço de contêiner ec2, você encontrará a definição de tarefa registrada

f) Use o seguinte comando da AWS CLI para executar uma tarefa com a definição de tarefa console-sample-app.

aws ecs run-task --task-definition console-sample-app

g) Abra o aplicativo da Web de exemplo no navegador, ele deve estar acessível (consulte a etapa 19)

exemplo-tarefa.json #

{
  "family": "example-task",
  "containerDefinitions": [
    {
        "environment": [],
        "name": "example-container",
        "image": "example-namespace/example-image:latest",
        "cpu": 10,
        "memory": 500,
        "portMappings": [
            {
                "containerPort": 8080,
                "hostPort": 80
            }
        ],
        "entryPoint": [],
        "essential": true
    }
  ]
}
