Configurando PHP para trabalhar com LDAP #

Depois de configurar seu servidor LDAP corretamente, agora queremos nos conectar. Por exemplo, usando PHP.

  • DN = nome distinto. Isso significa, em qual parte do banco de dados Voce está trabalhando. Pode ser um usuário ou um grupo (ou até mesmo configurações).
  • Entrada: uma entidade, por exemplo, um usuário.
  • Atributo: algo dentro de um entrada, por exemplo, nome, número de telefone e endereço de e-mail.

Conectando

Primeiro, definimos o seguinte:

$server = "server.example.com";  //this is the LDAP server you're connecting with
$ds = ldap_connect("ldaps://$server", 636); //always connect securely via LDAPS when possible

Agora, estamos conectados. A próxima coisa que queremos é que o servidor saiba que somos uma pessoa confiável. A solução mais fácil é usar o DN raiz OU um usuário existente com permissões adequadas para visualizar o banco de dados (provavelmente todos os usuários no banco de dados podem fazer isso por padrão). Nós autenticamos usando a função bind.

Definir opções

Primeiro, declaramos essas opções. Dependendo da configuração do seu servidor, você pode deixar isso de fora.

ldap_set_option($ds, LDAP_OPT_PROTOCOL_VERSION, 3);
ldap_set_option($ds, LDAP_OPT_REFERRALS, 0);

Vinculativo

Suponha que você esteja usando admin e a senha seja pass123notsafe

$dn = "uid=admin,cn=users,dc=server,dc=example,dc=com";
$pass = "pass123notsafe";
$ldapbind = ldap_bind($ds, $dn, $pass); //this is the point we are authenticating

Isso é muito legal. Estamos dentro. Agora podemos realizar muitas operações diferentes. Por exemplo, podemos pesquisar, ler, escrever e modificar usuários e até grupos.

Procurando

Imagine que queremos exibir todos os membros do grupo “usuários”.

$dn = "cn=users,dc=server,dc=example,dc=com"; //very important: in which part of your database are you looking
$filter = "uid=*"; //don't filter anyone out (every user has a uid)
$sr = ldap_search($ds, $dn, $filter) or die ("bummer"); //define your search scope

$results = ldap_get_entries($ds, $sr); //here we are pulling the actual entries from the search we just defined
var_dump($results); //will give you all results is array form.

Você pode usar loops foreach para exibir os dados de uma maneira agradável.

Ok, entendeu? Agora prossiga com alguma filtragem. Queremos exibir apenas os usuários do grupo “proprietários de bicicletas” que registraram um endereço de e-mail. É importante saber que todos os usuários estão em cn=users. Além disso, eles também podem ser membros de outros grupos.

//did the connecting and binding

$dn = "cn=bikeowners,cn=groups,dc=server,dc=example,dc=com"; //note the extra "cn=groups" for looking in a group that is not "users"
$filter = "email=*"; //email address must be set but can be anything
$sr = ldap_search($ds, $dn, $filter) or die ("bummer"); //define your search scope

E agora continue com ldap_get_entries novamente.

Eficiência na obtenção de inscrições

Um exemplo em que a eficiência é abordada, cada vez mais importante com grandes bancos de dados com muitos atributos por entrada. Especialmente quando você está armazenando imagens no atributo jpegphoto, isso pode reduzir significativamente o tempo de carregamento quando você puxa seletivamente suas entradas.

Imagine que queremos procurar usuários que estão no grupo “bikeowners”. Há muitas informações armazenadas nessas entradas, digamos que elas tenham os seguintes atributos: cn, uid, nome, nome de exibição, correio, iniciais, celular, número de telefone, rua, endereço postal, código postal e jpegphoto. Agora só precisamos de seu uid, nome e iniciais.

Para isso, usamos um 4º parâmetro opcional de ldap_search:

$dn = "cn=bikeowners,cn=groups,dc=server,dc=example,dc=com";
$filter = "uid=*"; //
$justhese = array("uid", "name", "initials");
$sr = ldap_search($ds, $dn, $filter, $justthese) or die ("bummer"); //define your search scope

E pronto, executar ldap_get_entries fornecerá apenas esses dados.

Filtragem avançada

Claro, você pode puxar um banco de dados inteiro com LDAP e processá-lo ainda mais em PHP. No entanto, assim como no MySQL, é muito mais eficiente fazer o processamento do lado do servidor. Para demonstrar isso, podemos usar um filtro avançado.

Tome um dos exemplos acima como ponto de partida, mas altere $filter de acordo com a linha abaixo. Em nosso exemplo, queremos exibir os dados dos usuários ativos. Normalmente, o atributo shadowexpire é usado para armazenar esta informação. No entanto, isso pode diferir entre vários sistemas LDAP. Não apenas queremos exibir os usuários ativos, mas também seus nomes devem começar com “a” e devem morar em Amsterdã.

Basicamente, queremos fazer três coisas:

  1. Mais fácil: deve morar em Amsterdã. Neste exemplo, o local de residência é armazenado no atributo ‘postaladdress’

    $filter= “postaladdress=Amsterdam”;

  2. O nome deve começar com “a”. Neste exemplo, o UID é composto pelo nome, portanto:

    $filter= “uid=a*”;

  3. O usuário deve estar ativo. Este valor é armazenado no atributo padrão shadowexpire, com valor -1. Dependendo da configuração do seu servidor, shadowexpire pode conter uma infinidade de valores, até mesmo datas são possíveis. Se um usuário estiver inativo, shadowexpire será 1. Para ter certeza de que obteremos todos os usuários, exceto aqueles que estão realmente inativos, não escolhemos filtrar shadowexpire = -1. Em vez disso, dizemos que não queremos que eles fiquem inativos.

    $filter= “(!(shadowexpire=1))”; //NOT is represented with “!”

Agora a parte mais interessante: combine os três exemplos. Podemos fazer isso com colchetes, expressões AND, OR e NOT

$filter= "(&(postaladdress=Amsterdam)(uid=a*)(!(shadowexpire=1)))";

Finalmente, poderíamos construir uma instrução OR usando “|”, por exemplo, se quisermos que todos os usuários comecem com “a” ou “b”

$filter= "(|(uid=a*)(uid=n*))";

Você pode combinar infinitamente e criar filtros bastante impressionantes, divirta-se tentando!
