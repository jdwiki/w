Java vem com um mecanismo poderoso e flexível para localizar seus aplicativos, mas também é fácil usar mal e acabar com um programa que desconsidera ou distorce a localidade do usuário e, portanto, como eles esperam que seu programa se comporte.

Seus usuários esperarão ver dados localizados nos formatos aos quais estão acostumados, e tentar oferecer suporte a isso manualmente é uma tarefa tola. Aqui está apenas um pequeno exemplo das diferentes maneiras pelas quais os usuários esperam ver o conteúdo que você pode supor que “sempre” é exibido de uma determinada maneira:

China

Reino Unido 20/3/16 1.234,56 £1.000,50 100km
EUA 20/03/16 1.234,56 US$ 1.000,50 1.000,50 MXN 60 milhas

Datas formatadas automaticamente usando “locale” #

SimpleDateFormatter é ótimo em uma pitada, mas como o nome sugere, não se adapta bem.

Se você codificar "MM/dd/aaaa" em todo o seu aplicativo, seus usuários internacionais não ficarão felizes.

Deixe o Java fazer o trabalho para você #

Use os métodos static em DateFormat para recuperar a formatação correta para seu usuário. Para um aplicativo de desktop (no qual você confiará na localidade padrão), basta chamar:

String localizedDate = DateFormat.getDateInstance(style).format(date);

Onde style é uma das constantes de formatação (FULL, LONG, MEDIUM, SHORT, etc.) especificadas em DateFormat.

Para um aplicativo do lado do servidor em que o usuário especifica sua localidade como parte da solicitação, você deve passá-lo explicitamente para getDateInstance():

String localizedDate =
    DateFormat.getDateInstance(style, request.getLocale()).format(date);

Comparação de Strings #

Compare duas Strings ignorando maiúsculas e minúsculas:

"School".equalsIgnoreCase("school"); // true

Não use

text1.toLowerCase().equals(text2.toLowerCase());

Os idiomas têm regras diferentes para converter maiúsculas e minúsculas. Um ‘I’ seria convertido em ‘i’ em inglês. Mas em turco um ‘I’ torna-se um ‘ı’. Se você tiver que usar toLowerCase() use a sobrecarga que espera um Locale: String.toLowerCase(Locale).

Comparando duas Strings ignorando pequenas diferenças:

Collator collator = Collator.getInstance(Locale.GERMAN);
collator.setStrength(Collator.PRIMARY);
collator.equals("Gärten", "gaerten"); // returns true

Classifique Strings respeitando a ordem da linguagem natural, ignorando maiúsculas e minúsculas (use a chave de agrupamento para:

String[] texts = new String[] {"Birne", "äther", "Apfel"};
Collator collator = Collator.getInstance(Locale.GERMAN);
collator.setStrength(Collator.SECONDARY); // ignore case
Arrays.sort(texts, collator::compare); // will return {"Apfel", "äther", "Birne"}

Localidade #

A classe java.util.Locale é usada para representar uma região “geográfica, política ou cultural” para localizar um determinado texto, número, data ou operação. Um objeto Locale pode, portanto, conter um país, região, idioma e também uma variante de um idioma, por exemplo, um dialeto falado em uma determinada região de um país ou falado em um país diferente do país de origem do idioma.

A instância Locale é entregue aos componentes que precisam localizar suas ações, seja convertendo a entrada, saída ou apenas para operações internas. A classe Locale não pode fazer nenhuma internacionalização ou localização por si só

Linguagem #

O idioma deve ser um código de idioma ISO 639 de 2 ou 3 caracteres ou uma subtag de idioma registrado de até 8 caracteres. Caso um idioma tenha um código de idioma de 2 e 3 caracteres, use o código de 2 caracteres. Uma lista completa de códigos de idioma pode ser encontrada no Registro de subtags de idioma da IANA.

Os códigos de idioma não diferenciam maiúsculas de minúsculas, mas a classe Locale sempre usa versões minúsculas dos códigos de idioma

Criando uma localidade #

A criação de uma instância java.util.Locale pode ser feita de quatro maneiras diferentes:

Locale constants
Locale constructors
Locale.Builder class
Locale.forLanguageTag factory method

Java ResourceBundle #

Você cria uma instância ResourceBundle assim:

Locale locale = new Locale("en", "US");
ResourceBundle labels = ResourceBundle.getBundle("i18n.properties");
System.out.println(labels.getString("message"));

Considere que eu tenho um arquivo de propriedades i18n.properties:

message=This is locale

Resultado:

This is locale

Configurando a localidade #

Se você quiser reproduzir o estado usando outras linguagens, você pode usar o método setDefault(). Seu uso:

 setDefault(Locale.JAPANESE); //Set Japanese
