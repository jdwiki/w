Sintaxe #

  • Objeto de data = new Date();
  • Objeto de data = new Date(data longa);

Parâmetros #

Parâmetro Explicação
Sem parâmetro Cria um novo objeto Date usando o tempo de alocação (até o milissegundo mais próximo)
longa data Cria um novo objeto Date com o tempo definido para o número de milissegundos desde “a época” (1 de janeiro de 1970, 00:00:00 GMT)

Representação

Internamente, um objeto Java Date é representado como um longo; é o número de milissegundos desde um tempo específico (referido como época). A classe Java Date original tinha métodos para lidar com fusos horários, etc., mas estes foram preteridos em favor da então nova classe Calendar.

Então, se tudo o que você quer fazer em seu código é representar uma hora específica, você pode criar uma classe Date e armazená-la, etc. Se você quiser imprimir uma versão legível dessa data, no entanto, você cria uma classe Calendar e use sua formatação para produzir horas, minutos, segundos, dias, fusos horários, etc. Lembre-se de que um milissegundo específico é exibido como horas diferentes em fusos horários diferentes; normalmente você deseja exibir um no fuso horário “local”, mas os métodos de formatação devem levar em conta que você pode querer exibi-lo para outro.

Esteja ciente também de que os relógios usados ​​pelas JVMs geralmente não têm precisão de milissegundos; o relógio pode apenas “marcar” a cada 10 milissegundos e, portanto, se estiver cronometrando as coisas, você não pode confiar em medir as coisas com precisão nesse nível.

Declaração de importação

import java.util.Date;

A classe Date pode ser importada do pacote java.util.

Cuidado

As instâncias Date são mutáveis, portanto, usá-las pode dificultar a gravação código thread-safe ou pode acidentalmente fornecer acesso de gravação ao estado interno. Por exemplo, na classe abaixo, o método getDate() permite que o chamador modifique a data da transação:

public final class Transaction {
  private final Date date;

  public Date getTransactionDate() {
    return date;
  }
}

A solução é retornar uma cópia do campo date ou usar as novas APIs em java.time introduzidas no Java 8.

A maioria dos métodos construtores na classe Date foram descontinuados e não devem ser usados. Em quase todos os casos, é aconselhável usar a classe Calendar para operações de data.

Java 8

O Java 8 introduz uma nova API de data e hora no pacote java.time, incluindo LocalDate e LocalTime. As classes no pacote java.time fornecem uma API reformulada que é mais fácil de usar. Se você estiver escrevendo para Java 8, é altamente recomendável que você use essa nova API. Consulte https://www.wikiod.com/pt/java/datas-e-hora-javatime .

Uma saída de data básica #

Usando o código a seguir com a string de formato yyyy/MM/dd hh:mm.ss, receberemos a seguinte saída

2016/04/19 11:45.36

// define the format to use
String formatString = "yyyy/MM/dd hh:mm.ss";

// get a current date object
Date date = Calendar.getInstance().getTime();

// create the formatter
SimpleDateFormat simpleDateFormat = new SimpleDateFormat(formatString);

// format the date
String formattedDate = simpleDateFormat.format(date);

// print it
System.out.println(formattedDate);


// single-line version of all above code
System.out.println(new SimpleDateFormat("yyyy/MM/dd hh:mm.ss").format(Calendar.getInstance().getTime()));

Converter java.util.Date em java.sql.Date #

A conversão de java.util.Date para java.sql.Date geralmente é necessária quando um objeto Date precisa ser escrito em um banco de dados.

java.sql.Date é um wrapper em torno do valor de milissegundos e é usado por JDBC para identificar um tipo SQL DATE

No exemplo abaixo, usamos o construtor java.util.Date(), que cria um objeto Date e o inicializa para representar o tempo até o milissegundo mais próximo. Esta data é usada no método convert(java.util.Date utilDate) para retornar um objeto java.sql.Date

Exemplo

public class UtilToSqlConversion {
    
    public static void main(String args[])
    {
        java.util.Date utilDate = new java.util.Date();
        System.out.println("java.util.Date is : " + utilDate);
        java.sql.Date sqlDate = convert(utilDate);
        System.out.println("java.sql.Date is : " + sqlDate);
        DateFormat df = new SimpleDateFormat("dd/MM/YYYY - hh:mm:ss");
        System.out.println("dateFormated date is : " + df.format(utilDate));
    }

    private static java.sql.Date convert(java.util.Date uDate) {
        java.sql.Date sDate = new java.sql.Date(uDate.getTime());
        return sDate;
    }

}

Resultado

java.util.Date is : Fri Jul 22 14:40:35 IST 2016
java.sql.Date is : 2016-07-22
dateFormated date is : 22/07/2016 - 02:40:35

java.util.Date tem informações de data e hora, enquanto java.sql.Date tem apenas informações de data

Objetos Java 8 LocalDate e LocalDateTime #

Objetos Date e LocalDate não podem ser exatamente convertidos entre si, pois um objeto Date representa um dia e uma hora específicos, enquanto um objeto LocalDate não contém informações de hora ou fuso horário. No entanto, pode ser útil converter entre os dois se você se importar apenas com as informações de data reais e não com as informações de hora.

Cria um LocalDate

// Create a default date
LocalDate lDate = LocalDate.now();

// Creates a date from values

Convertendo Data para um determinado formato de String #

format() da classe SimpleDateFormat ajuda a converter um objeto Date em um determinado objeto String de formato usando a string padrão fornecida.

Date today = new Date();
    
SimpleDateFormat dateFormat = new SimpleDateFormat("dd-MMM-yy"); //pattern is specified here
System.out.println(dateFormat.format(today)); //25-Feb-16

Os padrões podem ser aplicados novamente usando applyPattern()

dateFormat.applyPattern("dd-MM-yyyy");
System.out.println(dateFormat.format(today)); //25-02-2016

dateFormat.applyPattern("dd-MM-yyyy HH:mm:ss E");
System.out.println(dateFormat.format(today)); //25-02-2016 06:14:33 Thu

Observação: Aqui mm (letra m minúscula) indica minutos e MM (M maiúsculo) indica mês. Preste muita atenção ao formatar anos: “Y” maiúsculo (Y) indica a “semana do ano” enquanto “y” minúsculo (y) indica o ano.

Criando uma data específica #

Embora a classe Java Date tenha vários construtores, você notará que a maioria está obsoleta. A única maneira aceitável de criar uma instância de Date diretamente é usando o construtor vazio ou passando um longo (número de milissegundos desde o tempo base padrão). Nenhum deles é útil, a menos que você esteja procurando a data atual ou já tenha outra instância de Date em mãos.

Para criar uma nova data, você precisará de uma instância do Calendar. A partir daí, você pode definir a instância do Calendar para a data necessária.

Calendar c = Calendar.getInstance();

Isso retorna uma nova instância do Calendar definida para a hora atual. O calendário tem muitos métodos para alterar sua data e hora ou defini-lo diretamente. Neste caso, vamos defini-lo para uma data específica.

c.set(1974, 6, 2, 8, 0, 0);
Date d = c.getTime();

O método getTime retorna a instância Date que precisamos. Lembre-se de que os métodos de conjunto de calendário definem apenas um ou mais campos, eles não definem todos eles. Ou seja, se você definir o ano, os outros campos permanecerão inalterados.

ARMADILHA

Em muitos casos, esse trecho de código cumpre sua finalidade, mas lembre-se de que duas partes importantes da data/hora não estão definidas.

  • os parâmetros (1974, 6, 2, 8, 0, 0) são interpretados dentro do fuso horário padrão, definido em outro lugar,
  • os milissegundos não são definidos como zero, mas preenchidos a partir do relógio do sistema no momento em que a instância do Calendar é criada.

Criando objetos de data #

Date date = new Date();
System.out.println(date); // Thu Feb 25 05:03:59 IST 2016

Aqui este objeto Date contém a data e hora atuais quando este objeto foi criado.

Calendar calendar = Calendar.getInstance();
calendar.set(90, Calendar.DECEMBER, 11);
Date myBirthDate = calendar.getTime();
System.out.println(myBirthDate); // Mon Dec 31 00:00:00 IST 1990

Objetos Date são mais bem criados através de uma instância Calendar já que o uso dos construtores de dados é obsoleto e desencorajado. Para isso, precisamos obter uma instância da classe Calendar do método fábrica. Em seguida, podemos definir ano, mês e dia do mês usando números ou no caso de constantes de meses fornecidas pela classe Calendar para melhorar a legibilidade e reduzir erros.

calendar.set(90, Calendar.DECEMBER, 11, 8, 32, 35);
Date myBirthDatenTime = calendar.getTime();
System.out.println(myBirthDatenTime); // Mon Dec 31 08:32:35 IST 1990

Junto com a data, também podemos passar o tempo na ordem de horas, minutos e segundos.

Comparação de datas antes do Java 8 #

Antes do Java 8, as datas podiam ser comparadas usando as classes java.util.Calendar e java.util.Date. A classe Date oferece 4 métodos para comparar datas:

  • depois (Data quando)
  • antes (Data quando)
  • compareTo(Data outraData)
  • equals(Object obj)

Os métodos after, before, compareTo e equals comparam os valores retornados pelo método getTime() para cada data.

O método compareTo retorna um inteiro positivo.

  • Valor maior que 0 : quando a Data é posterior ao argumento Data
  • Valor maior que 0 : quando a Data é anterior ao argumento Data
  • Valor igual a 0 : quando a Data é igual ao argumento Data

Os resultados equals podem ser surpreendentes, conforme mostrado no exemplo, porque valores, como milissegundos, não são inicializados com o mesmo valor se não forem fornecidos explicitamente.

Desde Java 8 #

Com Java 8 está disponível um novo Objeto para trabalhar com Data java.time.LocalDate. LocalDate implementa ChronoLocalDate, a representação abstrata de uma data onde a Cronologia, ou sistema de calendário, é conectável.

Para ter a precisão de data e hora o Objeto java.time.LocalDateTime deve ser usado. LocalDate e LocalDateTime usam o mesmo nome de método para comparação.

Comparar datas usando um LocalDate é diferente de usar ChronoLocalDate porque a cronologia ou sistema de calendário não leva em consideração o primeiro.

Como a maioria dos aplicativos deve usar LocalDate, ChronoLocalDate não está incluído nos exemplos. Leitura adicional aqui.

A maioria dos aplicativos deve declarar assinaturas de métodos, campos e variáveis ​​como LocalDate, não esta interface [ChronoLocalDate].

LocalDate tem 5 métodos para comparar datas:

  • isAfter(ChronoLocalDate other)
  • isBefore(ChronoLocalDate other)
  • isEqual(ChronoLocalDate outro)
  • compareTo(ChronoLocalDate other)
  • equals(Object obj)

No caso do parâmetro LocalDate, isAfter, isBefore, isEqual, equals e compareTo agora use este método:

int compareTo0(LocalDate otherDate) {
    int cmp = (year - otherDate.year);
    if (cmp == 0) {
        cmp = (month - otherDate.month);
        if (cmp == 0) {
            cmp = (day - otherDate.day);
        }
    }
    return cmp;
}

O método equals verifica se a referência do parâmetro é igual à data primeiro, enquanto isEqual chama diretamente compareTo0.

No caso de uma outra instância de classe de ChronoLocalDate as datas são comparadas usando o Epoch Day. A contagem Epoch Day é uma contagem incremental simples de dias em que o dia 0 é 1970-01-01 (ISO).

Fusos horários e java.util.Date #

Um objeto java.util.Date não tem um conceito de fuso horário.

  • Não há como definir um fuso horário para uma data
  • Não há como alterar o fuso horário de um objeto Date
  • Um objeto Date criado com o construtor padrão new Date() será inicializado com a hora atual no fuso horário padrão do sistema

No entanto, é possível exibir a data representada pelo ponto no tempo descrito pelo objeto Date em um fuso horário diferente usando, por exemplo, java.text.SimpleDateFormat:

Date date = new Date();
//print default time zone
System.out.println(TimeZone.getDefault().getDisplayName());
SimpleDateFormat sdf = new SimpleDateFormat("yyyy-MM-dd HH:mm:ss"); //note: time zone not in format!
//print date in the original time zone
System.out.println(sdf.format(date));
//current time in London
sdf.setTimeZone(TimeZone.getTimeZone("Europe/London"));
System.out.println(sdf.format(date));

Resultado:

Central European Time
2016-07-21 22:50:56
2016-07-21 21:50:56

Horário local #

Para usar apenas a parte de hora de uma data, use LocalTime. Você pode instanciar um objeto LocalTime de algumas maneiras

  1. Hora LocalTime = LocalTime.now();
  2. hora = LocalTime.MIDNIGHT;
  3. hora = LocalTime.NOON;
  4. hora = LocalTime.of(12, 12, 45);

O LocalTime também possui um método toString embutido que exibe o formato muito bem.

System.out.println(time);

você também pode obter, adicionar e subtrair horas, minutos, segundos e nanossegundos do objeto LocalTime, ou seja,

time.plusMinutes(1);
time.getMinutes();
time.minusMinutes(1);

Você pode transformá-lo em um objeto Date com o seguinte código:

LocalTime lTime = LocalTime.now();
Instant instant = lTime.atDate(LocalDate.of(A_YEAR, A_MONTH, A_DAY)).
        atZone(ZoneId.systemDefault()).toInstant();
Date time = Date.from(instant);

esta classe funciona muito bem dentro de uma classe de temporizador para simular um despertador.
