Um Stream representa uma sequência de elementos e suporta diferentes tipos de operações para realizar cálculos sobre esses elementos. Com Java 8, a interface Collection tem dois métodos para gerar um Stream: stream() e parallelStream(). As operações Stream são intermediárias ou terminais. As operações intermediárias retornam um Stream para que várias operações intermediárias possam ser encadeadas antes que o Stream seja fechado. As operações de terminal são nulas ou retornam um resultado que não seja de fluxo.

Sintaxe #

  • coleção.stream()
  • Arrays.stream(array)
  • Stream.iterate(firstValue, currentValue -> nextValue)
  • Stream.generate(() -> valor)
  • Stream.of(elementOfT[, elementOfT, …])
  • Fluxo.vazio()
  • StreamSupport.stream( iterable.spliterator(), false )

Usando fluxos #

Um Stream é uma sequência de elementos sobre os quais operações de agregação sequenciais e paralelas podem ser executadas. Qualquer Stream pode potencialmente ter uma quantidade ilimitada de dados fluindo através dele. Como resultado, os dados recebidos de um Stream são processados ​​individualmente à medida que chegam, em vez de realizar o processamento em lote dos dados por completo. Quando combinados com expressões lambda, eles fornecem uma maneira concisa de realizar operações em sequências de dados usando uma abordagem funcional.

Exemplo: (veja como funciona no Ideone)

Stream<String> fruitStream = Stream.of("apple", "banana", "pear", "kiwi", "orange");

fruitStream.filter(s -> s.contains("a"))
           .map(String::toUpperCase)
           .sorted()
           .forEach(System.out::println);

  1. Crie um Stream<String> contendo um Stream ordenado sequencial de elementos String de fruta usando o método de fábrica estático Stream.of(values).

  2. A operação filter() retém apenas os elementos que correspondem a um determinado predicado (os elementos que, quando testados pelo predicado, retornam true). Nesse caso, ele retém os elementos que contêm um "a". O predicado é dado como uma expressão lambda.

  3. A operação map() transforma cada elemento usando uma determinada função, chamada de mapeador. Nesse caso, cada fruta String é mapeada para sua versão maiúscula String usando o method-reference String::toUppercase.

Observe que a operação map() retornará um fluxo com um tipo genérico diferente se a função de mapeamento retornar um tipo diferente do seu parâmetro de entrada. Por exemplo, em um Stream<String> chamar .map(String::isEmpty) retorna um Stream<Boolean>

  1. A operação sorted() ordena os elementos do Stream de acordo com sua ordenação natural (lexicograficamente, no caso de String).

  2. Finalmente, a operação forEach(action) executa uma ação que atua em cada elemento do Stream, passando-o para um Consumer. No exemplo, cada elemento está simplesmente sendo impresso no console. Esta operação é uma operação terminal, impossibilitando a operação novamente.

Observe que as operações definidas no Stream são executadas por causa da operação do terminal. Sem uma operação de terminal, o fluxo não é processado. Os fluxos não podem ser reutilizados. Uma vez que uma operação de terminal é chamada, o objeto Stream torna-se inutilizável.

As operações (como visto acima) são encadeadas para formar o que pode ser visto como uma consulta nos dados.

Streams de fechamento #

Observe que um Stream geralmente não precisa ser fechado. É necessário apenas fechar streams que operam em canais IO. A maioria dos tipos Stream não opera em recursos e, portanto, não requer fechamento.

A interface Stream estende AutoCloseable. Os fluxos podem ser fechados chamando o método close ou usando instruções try-with-resource.

Um exemplo de caso de uso em que um Stream deve ser fechado é quando você cria um Stream de linhas de um arquivo:

try (Stream<String> lines = Files.lines(Paths.get("somePath"))) {
    lines.forEach(System.out::println);
}

A interface Stream também declara o método Stream.onClose() que permite registrar manipuladores Runnable que serão chamados quando o fluxo for fechado. Um exemplo de caso de uso é onde o código que produz um fluxo precisa saber quando ele é consumido para realizar alguma limpeza.

public Stream<String>streamAndDelete(Path path) throws IOException {
    return Files.lines(path).onClose(() -> someClass.deletePath(path));
}

O manipulador run só será executado se o método close() for chamado, explicitamente ou implicitamente por uma instrução try-with-resources.

Processando o Pedido #

O processamento de um objeto Stream pode ser sequencial ou [paralelo](https://www.wikiod.com/pt/java/fluxos#Fluxo paralelo).

No modo sequencial, os elementos são processados ​​na ordem da origem do Stream. Se o Stream for ordenado (como uma implementação de SortedMap ou uma List), o processamento é garantido para corresponder à ordem da origem. Em outros casos, entretanto, deve-se tomar cuidado para não depender da ordenação (veja: a ordem de iteração Java HashMap keySet() é consistente?).

Exemplo:

List<Integer> integerList = Arrays.asList(0, 1, 2, 3, 42); 

// sequential 
long howManyOddNumbers = integerList.stream()
                                    .filter(e -> (e % 2) == 1)
                                    .count(); 

System.out.println(howManyOddNumbers); // Output: 2

Ao vivo no Ideone

O modo Paralelo permite o uso de vários threads em vários núcleos, mas não há garantia da ordem em que os elementos são processados.

Se vários métodos são chamados em um Stream sequencial, nem todos os métodos precisam ser invocados. Por exemplo, se um Stream for filtrado e o número de elementos for reduzido a um, uma chamada subsequente para um método como sort não ocorrerá. Isso pode aumentar o desempenho de um Stream sequencial — uma otimização que não é possível com um Stream paralelo.

Exemplo:

// parallel
long howManyOddNumbersParallel = integerList.parallelStream()
                                            .filter(e -> (e % 2) == 1)
                                            .count();

System.out.println(howManyOddNumbersParallel); // Output: 2

Ao vivo no Ideone

Diferenças de contêineres (ou Coleções) #

Embora algumas ações possam ser executadas em contêineres e fluxos, elas servem a propósitos diferentes e dão suporte a operações diferentes. Os contêineres são mais focados em como os elementos são armazenados e como esses elementos podem ser acessados ​​com eficiência. Um Stream, por outro lado, não fornece acesso direto e manipulação de seus elementos; é mais dedicado ao grupo de objetos como uma entidade coletiva e realizando operações nessa entidade como um todo. Stream e Collection são abstrações de alto nível separadas para esses propósitos diferentes.

Consumindo Streams #

Criando um mapa de frequência #

O coletor groupingBy(classifier, downstream) permite a coleta de elementos Stream em um Map classificando cada elemento em um grupo e realizando uma operação downstream sobre os elementos classificados no mesmo grupo.

Um exemplo clássico desse princípio é usar um Map para contar as ocorrências de elementos em um Stream. Neste exemplo, o classificador é simplesmente a função de identidade, que retorna o elemento como está. A operação downstream conta o número de elementos iguais, usando counting().

Stream.of("apple", "orange", "banana", "apple")
      .collect(Collectors.groupingBy(Function.identity(), Collectors.counting()))
      .entrySet()
      .forEach(System.out::println);

A operação downstream é um coletor (Collectors.counting()) que opera em elementos do tipo String e produz um resultado do tipo Long. O resultado da chamada do método collect é um Map<String, Long>.

Isso produziria a seguinte saída:

banana=1
laranja=1
maçã=2

Fluxos infinitos #

É possível gerar um Stream que não termina. Chamar um método de terminal em um Stream infinito faz com que o Stream entre em um loop infinito. O método limit de um Stream pode ser usado para limitar o número de termos do Stream que o Java processa.

Este exemplo gera um Stream de todos os números naturais, começando com o número 1. Cada termo sucessivo do Stream é um mais alto que o anterior. Ao chamar o método limit deste Stream, apenas os primeiros cinco termos do Stream são considerados e impressos.

// Generate infinite stream - 1, 2, 3, 4, 5, 6, 7, ...
IntStream naturalNumbers = IntStream.iterate(1, x -> x + 1);

// Print out only the first 5 terms
naturalNumbers.limit(5).forEach(System.out::println);

Resultado:

1
2
3
4
5

Outra maneira de gerar um fluxo infinito é usar o método Stream.generate. Este método recebe um lambda do tipo Fornecedor.

// Generate an infinite stream of random numbers
Stream<Double> infiniteRandomNumbers = Stream.generate(Math::random);

// Print out only the first 10 random numbers
infiniteRandomNumbers.limit(10).forEach(System.out::println);

Colete com toList() e toSet() #

Elementos de um Stream podem ser facilmente coletados em um container usando a operação Stream.collect:

System.out.println(Arrays
    .asList("apple", "banana", "pear", "kiwi", "orange")
    .stream()
    .filter(s -> s.contains("a"))
    .collect(Collectors.toList())
);
// prints: [apple, banana, pear, orange]

Outras instâncias de coleção, como um Set, podem ser feitas usando outros métodos internos Collectors. Por exemplo, Collectors.toSet() coleta os elementos de um Stream em um Set.

Controle explícito sobre a implementação de List ou Set

De acordo com a documentação de Collectors#toList() e Collectors#toSet(), não há garantias sobre o tipo, mutabilidade, serialização ou segurança de thread do List ou Set retornado.

Para controle explícito sobre a implementação a ser retornada, Collectors#toCollection(Supplier) pode ser usado, onde o fornecedor fornecido retorna uma coleção nova e vazia.

// syntax with method reference
System.out.println(strings
        .stream()
        .filter(s -> s != null && s.length() <= 3)
        .collect(Collectors.toCollection(ArrayList::new))
);

// syntax with lambda
System.out.println(strings
        .stream()
        .filter(s -> s != null && s.length() <= 3)
        .collect(Collectors.toCollection(() -> new LinkedHashSet<>()))
);

Fluxo paralelo #

Observação: Antes de decidir qual Stream usar, dê uma olhada em Comportamento ParallelStream vs Sequential Stream.

Quando você deseja executar operações Stream simultaneamente, você pode usar qualquer uma dessas maneiras.

Ou:

(Uma possível) saída do Stream paralelo:

Quatro
Um
Dois
Cinco

A ordem pode mudar à medida que todos os elementos são processados ​​em paralelo (o que pode torná-lo mais rápido). Use parallelStream quando a ordem não importa.

Impacto no desempenho #

Caso a rede esteja envolvida, os Streams paralelos podem degradar o desempenho geral de uma aplicação porque todos os Streams paralelos usam um pool de threads fork-join comum para a rede.

Por outro lado, Streams paralelos podem melhorar significativamente o desempenho em muitos outros casos, dependendo do número de núcleos disponíveis na CPU em execução no momento.

Criando um fluxo #

Todos os Collection<E>s java têm métodos stream() e parallelStream() a partir dos quais um Stream<E> pode ser construído:

Collection<String> stringList = new ArrayList<>();
Stream<String> stringStream = stringList.parallelStream();

Um Stream<E> pode ser criado a partir de um array usando um dos dois métodos:

String[] values = { "aaa", "bbbb", "ddd", "cccc" };
Stream<String> stringStream = Arrays.stream(values);
Stream<String> stringStreamAlternative = Stream.of(values);

A diferença entre Arrays.stream() e Stream.of() é que Stream.of() tem um parâmetro varargs, então pode ser usado como:

Stream<Integer> integerStream = Stream.of(1, 2, 3);

Existem também Streams primitivos que você pode usar. Por exemplo:

IntStream intStream = IntStream.of(1, 2, 3);
DoubleStream doubleStream = DoubleStream.of(1.0, 2.0, 3.0);

Esses fluxos primitivos também podem ser construídos usando o método Arrays.stream():

IntStream intStream = Arrays.stream(new int[]{ 1, 2, 3 });

É possível criar um Stream a partir de um array com um intervalo especificado.

int[] values= new int[]{1, 2, 3, 4, 5};
IntStream intStram = Arrays.stream(values, 1, 3);

Observe que qualquer stream primitivo pode ser convertido em stream do tipo boxed usando o método boxed:

Stream<Integer> integerStream = intStream.boxed();

Isso pode ser útil em alguns casos se você quiser coletar os dados, pois o fluxo primitivo não possui nenhum método collect que receba um Collector como argumento.

Concatenar fluxos #

Declaração de variável para exemplos:

Collection<String> abc = Arrays.asList("a", "b", "c");
Collection<String> digits = Arrays.asList("1", "2", "3");
Collection<String> greekAbc = Arrays.asList("alpha", "beta", "gamma");

Exemplo 1 - Concatenar dois Streams

final Stream<String> concat1 = Stream.concat(abc.stream(), digits.stream());

concat1.forEach(System.out::print);
// prints: abc123

Exemplo 2 - Concatenar mais de dois Streams

final Stream<String> concat2 = Stream.concat(
    Stream.concat(abc.stream(), digits.stream()),
    greekAbc.stream());

System.out.println(concat2.collect(Collectors.joining(", ")));
// prints: a, b, c, 1, 2, 3, alpha, beta, gamma

Alternativamente, para simplificar a sintaxe concat() aninhada, os Streams também podem ser concatenados com flatMap():

final Stream<String> concat3 = Stream.of(
    abc.stream(), digits.stream(), greekAbc.stream())
    .flatMap(s -> s);
    // or `.flatMap(Function.identity());` (java.util.function.Function)

System.out.println(concat3.collect(Collectors.joining(", ")));
// prints: a, b, c, 1, 2, 3, alpha, beta, gamma

Tenha cuidado ao construir Streams a partir de concatenação repetida, porque acessar um elemento de um Stream profundamente concatenado pode resultar em cadeias de chamadas profundas ou até mesmo em uma StackOverflowException.

IntStream para String #

Java não tem um Char Stream, então ao trabalhar com Strings e construir um Stream de Characters, uma opção é obter um IntStream de pontos de código usando String método .codePoints(). Então IntStream pode ser obtido como abaixo:

public IntStream stringToIntStream(String in) {
  return in.codePoints();
}

É um pouco mais complicado fazer a conversão de outra forma, ou seja, IntStreamToString. Isso pode ser feito da seguinte forma:

public String intStreamToString(IntStream intStream) {
  return intStream.collect(StringBuilder::new, StringBuilder::appendCodePoint, StringBuilder::append).toString();
}

Encontrando o primeiro elemento que corresponde a um predicado #

É possível encontrar o primeiro elemento de um Stream que corresponda a uma condição.

Para este exemplo, encontraremos o primeiro Integer cujo quadrado é maior que 50000.

IntStream.iterate(1, i -> i + 1) // Generate an infinite stream 1,2,3,4...
    .filter(i -> (i*i) > 50000) // Filter to find elements where the square is >50000
    .findFirst(); // Find the first filtered element

Esta expressão retornará um OptionalInt com o resultado.

Observe que com um Stream infinito, o Java continuará verificando cada elemento até encontrar um resultado. Com um Stream finito, se o Java ficar sem elementos, mas ainda não conseguir encontrar um resultado, ele retornará um OptionalInt vazio.

Usando referências de fluxos e métodos para gravar processos de autodocumentação #

Referências de método fazem um excelente código auto-documentado, e usar referências de método com Streams torna processos complicados simples de ler e entender. Considere o seguinte código:

public interface Ordered {
    default int getOrder(){
        return 0;
    }
}

public interface Valued<V extends Ordered> {
    boolean hasPropertyTwo();
    V getValue();
}

public interface Thing<V extends Ordered> {
    boolean hasPropertyOne();
    Valued<V> getValuedProperty();
}

public <V extends Ordered> List<V> myMethod(List<Thing<V>> things) {
    List<V> results = new ArrayList<V>();
    for (Thing<V> thing : things) {
        if (thing.hasPropertyOne()) {
            Valued<V> valued = thing.getValuedProperty();
            if (valued != null && valued.hasPropertyTwo()){
                V value = valued.getValue();
                if (value != null){
                    results.add(value);
                }
            }
        }
    }
    results.sort((a, b)->{
        return Integer.compare(a.getOrder(), b.getOrder());
    });
    return results;
}

Este último método reescrito usando Streams e referências de método é muito mais legível e cada etapa do processo é rápida e facilmente compreendida - não é apenas mais curta, também mostra rapidamente quais interfaces e classes são responsáveis ​​pelo código em cada degrau:

public <V extends Ordered> List<V> myMethod(List<Thing<V>> things) {
    return things.stream()
        .filter(Thing::hasPropertyOne)
        .map(Thing::getValuedProperty)
        .filter(Objects::nonNull)
        .filter(Valued::hasPropertyTwo)
        .map(Valued::getValue)
        .filter(Objects::nonNull)
        .sorted(Comparator.comparing(Ordered::getOrder))
        .collect(Collectors.toList());
}

Usando fluxos de Map.Entry para preservar os valores iniciais após o mapeamento #

Quando você tem um Stream, você precisa mapear, mas deseja preservar os valores iniciais também, você pode mapear o Stream para um Map.Entry<K,V> usando um método utilitário como o seguinte:

public static <K, V> Function<K, Map.Entry<K, V>> entryMapper(Function<K, V> mapper){
    return (k)->new AbstractMap.SimpleEntry<>(k, mapper.apply(k));
}

Então você pode usar seu conversor para processar Streams tendo acesso aos valores originais e mapeados:

Set<K> mySet;
Function<K, V> transformer = SomeClass::transformerMethod;
Stream<Map.Entry<K, V>> entryStream = mySet.stream()
    .map(entryMapper(transformer));

Você pode continuar processando esse Stream normalmente. Isso evita a sobrecarga de criar uma coleção intermediária.

Redução com streams #

Redução é o processo de aplicar um operador binário a cada elemento de um fluxo para resultar em um valor.

O método sum() de um IntStream é um exemplo de redução; ele aplica adição a cada termo do Stream, resultando em um valor final:

Isso é equivalente a (((1+2)+3)+4)

O método reduce de um Stream permite criar uma redução personalizada. É possível usar o método reduce para implementar o método sum():

IntStream istr;
    
//Initialize istr
    
OptionalInt istr.reduce((a,b)->a+b);

A versão Optional é retornada para que os Streams vazios possam ser tratados adequadamente.

Outro exemplo de redução é combinar um Stream<LinkedList<T>> em um único LinkedList<T>:

Stream<LinkedList<T>> listStream;
    
//Create a Stream<LinkedList<T>>
    
Optional<LinkedList<T>> bigList = listStream.reduce((LinkedList<T> list1, LinkedList<T> list2)->{
    LinkedList<T> retList = new LinkedList<T>();
    retList.addAll(list1);
    retList.addAll(list2);
    return retList;
});

Você também pode fornecer um elemento de identidade. Por exemplo, o elemento de identidade para adição é 0, como x+0==x. Para multiplicação, o elemento de identidade é 1, como x*1==x. No caso acima, o elemento identity é um LinkedList<T> vazio, porque se você adicionar uma lista vazia a outra lista, a lista que você está “adicionando” não muda:

Stream<LinkedList<T>> listStream;

//Create a Stream<LinkedList<T>>

LinkedList<T> bigList = listStream.reduce(new LinkedList<T>(), (LinkedList<T> list1, LinkedList<T> list2)->{
    LinkedList<T> retList = new LinkedList<T>();
    retList.addAll(list1);
    retList.addAll(list2);
    return retList;
});

Observe que quando um elemento de identidade é fornecido, o valor de retorno não é encapsulado em um Opcional—se chamado em um fluxo vazio, reduce() retornará o elemento de identidade.

O operador binário também deve ser associativo, significando que (a+b)+c==a+(b+c). Isso ocorre porque os elementos podem ser reduzidos em qualquer ordem. Por exemplo, a redução de adição acima pode ser realizada assim:

Convertendo um fluxo de opcional em um fluxo de valores #

Você pode precisar converter um Stream emitindo Optional para um Stream de valores, emitindo apenas valores de Optional existente. (ou seja: sem valor null e não lidando com Optional.empty()).

 Optional<String> op1 = Optional.empty();
 Optional<String> op2 = Optional.of("Hello World");

 List<String> result = Stream.of(op1, op2)
                             .filter(Optional::isPresent)
                             .map(Optional::get)
                             .collect(Collectors.toList());

 System.out.println(result); //[Hello World]

Obtenha uma fatia de um stream #

Exemplo: obtenha um Stream de 30 elementos, contendo o 21º ao 50º elemento (inclusive) de uma coleção.

final long n = 20L; // the number of elements to skip
final long maxSize = 30L; // the number of elements the stream should be limited to
final Stream<T> slice = collection.stream().skip(n).limit(maxSize);

Notas:

  • IllegalArgumentException é lançado se n for negativo ou maxSize é negativo
  • tanto skip(long) quanto limit(long) são operações intermediárias
  • se um stream contém menos de n elementos então skip(n) retorna um stream vazio
  • tanto skip(long) quanto limit(long) são operações baratas em pipelines de fluxo sequencial, mas podem ser bastante caras em pipelines paralelos ordenados

Crie um mapa baseado em um fluxo #

Caixa simples sem chaves duplicadas

Stream<String> characters = Stream.of("A", "B", "C");

Map<Integer, String> map = characters
            .collect(Collectors.toMap(element -> element.hashCode(), element -> element));
// map = {65=A, 66=B, 67=C}

Para tornar as coisas mais declarativas, podemos usar o método estático na interface Function - Function.identity(). Podemos substituir este lambda element -> element por Function.identity().

Caso em que pode haver chaves duplicadas

O javadoc para Collectors.toMap afirma:

Se as chaves mapeadas contiverem duplicatas (de acordo com Object.equals(Object)), uma IllegalStateException será lançada quando a operação de coleta for executada. Se as chaves mapeadas podem ter duplicatas, use toMap(Function, Function, BinaryOperator).

Stream<String> characters = Stream.of("A", "B", "B", "C");

Map<Integer, String> map = characters
            .collect(Collectors.toMap(
                element -> element.hashCode(),
                element -> element,
                (existingVal, newVal) -> (existingVal + newVal)));

// map = {65=A, 66=BB, 67=C}

O BinaryOperator passado para Collectors.toMap(...) gera o valor a ser armazenado em caso de colisão. Pode:

  • retorna o valor antigo, para que o primeiro valor no fluxo tenha precedência,
  • retorna o novo valor, para que o último valor no fluxo tenha precedência, ou
  • combinar os valores antigos e novos

Agrupando por valor

Você pode usar Collectors.groupingBy quando precisar executar o equivalente a uma operação “agrupar por” em cascata de banco de dados. Para ilustrar, o seguinte cria um mapa no qual os nomes das pessoas são mapeados para sobrenomes:

List<Person> people = Arrays.asList(
    new Person("Sam", "Rossi"),
    new Person("Sam", "Verdi"),
    new Person("John", "Bianchi"),
    new Person("John", "Rossi"),
    new Person("John", "Verdi")
);

Map<String, List<String>> map = people.stream()
        .collect(
                // function mapping input elements to keys
                Collectors.groupingBy(Person::getName, 
                // function mapping input elements to values,
                // how to store values
                Collectors.mapping(Person::getSurname, Collectors.toList()))
        );

// map = {John=[Bianchi, Rossi, Verdi], Sam=[Rossi, Verdi]}

List<String> data = new ArrayList<>();
data.add("Sydney");
data.add("London");
data.add("New York");
data.add("Amsterdam");
data.add("Mumbai");
data.add("California");

System.out.println(data);

List<String> sortedData = data.stream().sorted().collect(Collectors.toList());

System.out.println(sortedData);

Resultado:

[Sydney, London, New York, Amsterdam, Mumbai, California]
[Amsterdam, California, London, Mumbai, New York, Sydney]

Também é possível usar um mecanismo de comparação diferente, pois há uma versão sobrecarregada sorted que usa um comparador como argumento.

Além disso, você pode usar uma expressão lambda para classificar:

List<String> sortedData2 = data.stream().sorted((s1,s2) -> s2.compareTo(s1)).collect(Collectors.toList());

List<String> reverseSortedData = data.stream().sorted(Comparator.reverseOrder()).collect(Collectors.toList());

Fluxos de primitivos #

Java fornece Streams especializados para três tipos de primitivos IntStream (para ints), LongStream (para longs) e DoubleStream (para doubles). Além de serem implementações otimizadas para suas respectivas primitivas, eles também fornecem diversos métodos terminais específicos, tipicamente para operações matemáticas. Por exemplo.:

IntStream is = IntStream.of(10, 20, 30);
double average = is.average().getAsDouble(); // average is 20.0

Categorias de operações de stream #

As operações de fluxo se dividem em duas categorias principais, operações intermediárias e terminais e duas subcategorias, sem estado e com estado.

Operações Intermediárias: #

Uma operação intermediária é sempre preguiçosa, como um simples Stream.map. Ele não é invocado até que o fluxo seja realmente consumido. Isso pode ser verificado facilmente:

Arrays.asList(1, 2 ,3).stream().map(i -> {
    throw new RuntimeException("not gonna happen");
    return i;
});

As operações intermediárias são os blocos de construção comuns de um fluxo, encadeados após a origem e geralmente seguidos por uma operação de terminal que aciona a cadeia de fluxo.

Operações do Terminal #

As operações do terminal são o que aciona o consumo de um fluxo. Alguns dos mais comuns são Stream.forEach ou Stream.collect. Eles geralmente são colocados após uma cadeia de operações intermediárias e quase sempre estão ansiosos.

Operações sem estado #

Stateless significa que cada item é processado sem o contexto de outros itens. As operações sem estado permitem o processamento de fluxos com eficiência de memória. Operações como Stream.map e Stream.filter que não requerem informações sobre outros itens do stream são consideradas sem estado.

Operações com estado #

Statefulness significa que a operação em cada item depende de (alguns) outros itens do fluxo. Isso requer que um estado seja preservado. As operações de statefulness podem romper com fluxos longos ou infinitos. Operações como Stream.sorted exigem que todo o fluxo seja processado antes que qualquer item seja emitido, o que quebrará em um fluxo de itens suficientemente longo. Isso pode ser demonstrado por um fluxo longo (execute por sua conta e risco):

// works - stateless stream
long BIG_ENOUGH_NUMBER = 999999999;
IntStream.iterate(0, i -> i + 1).limit(BIG_ENOUGH_NUMBER).forEach(System.out::println);

Isso causará falta de memória devido ao estado de Stream.sorted:

// Out of memory - stateful stream
IntStream.iterate(0, i -> i + 1).limit(BIG_ENOUGH_NUMBER).sorted().forEach(System.out::println);

Unindo um stream a uma única String #

Um caso de uso frequente é criar uma String a partir de um stream, onde os stream-items são separados por um determinado caractere. O método Collectors.joining() pode ser usado para isso, como no exemplo a seguir:

Stream<String> fruitStream = Stream.of("apple", "banana", "pear", "kiwi", "orange");

String result = fruitStream.filter(s -> s.contains("a"))
           .map(String::toUpperCase)
           .sorted()
           .collect(Collectors.joining(", "));
           
System.out.println(result);

Resultado:

MAÇÃ, BANANA, LARANJA, PERA

O método Collectors.joining() também pode servir para pré e pós-fixos:

String result = fruitStream.filter(s -> s.contains("e"))
           .map(String::toUpperCase)
           .sorted()
           .collect(Collectors.joining(", ", "Fruits: ", "."));
           
System.out.println(result);

Resultado:

Frutas: MAÇÃ, LARANJA, PERA.

Gerando Strings aleatórias usando Streams #

Às vezes é útil criar Strings aleatórias, talvez como ID de sessão para um serviço da Web ou uma senha inicial após o registro de um aplicativo. Isso pode ser facilmente alcançado usando Streams.

Primeiro precisamos inicializar um gerador de números aleatórios. Para aumentar a segurança das Strings geradas, é uma boa idéia usar SecureRandom.

Observação: Criar um SecureRandom é bastante caro, então é uma boa prática fazer isso apenas uma vez e chamar um de seus métodos setSeed() de tempos em tempos para resemeá-lo.

private static final SecureRandom rng = new SecureRandom(SecureRandom.generateSeed(20)); 
//20 Bytes as a seed is rather arbitrary, it is the number used in the JavaDoc example

Ao criar Strings aleatórios, geralmente queremos que eles usem apenas determinados caracteres (por exemplo, apenas letras e dígitos). Portanto, podemos criar um método retornando um boolean que pode ser usado posteriormente para filtrar o Stream.

//returns true for all chars in 0-9, a-z and A-Z
boolean useThisCharacter(char c){
    //check for range to avoid using all unicode Letter (e.g. some chinese symbols)
    return c >= '0' && c <= 'z' && Character.isLetterOrDigit(c);
}

Em seguida, podemos utilizar o RNG para gerar uma String aleatória de comprimento específico contendo o charset que passa em nossa verificação useThisCharacter.

public String generateRandomString(long length){
    //Since there is no native CharStream, we use an IntStream instead
    //and convert it to a Stream<Character> using mapToObj.
    //We need to specify the boundaries for the int values to ensure they can safely be cast to char
    Stream<Character> randomCharStream = rng.ints(Character.MIN_CODE_POINT, Character.MAX_CODE_POINT).mapToObj(i -> (char)i).filter(c -> this::useThisCharacter).limit(length);

    //now we can use this Stream to build a String utilizing the collect method.
    String randomString = randomCharStream.collect(StringBuilder::new, StringBuilder::append, StringBuilder::append).toString();
    return randomString;
}
