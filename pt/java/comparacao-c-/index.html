Java e C++ são linguagens semelhantes. Este tópico serve como um guia de referência rápida para engenheiros de Java e C++.

Classes definidas dentro de outras construções# #

Definido dentro de Outra Classe #

##C++## Classe aninhada[ref] (precisa de uma referência à classe anexa)

class Outer {
   class Inner {
      public:
         Inner(Outer* o) :outer(o) {}

      private:
         Outer*  outer;
   };
};

##Java## [não estático] Classe aninhada (também conhecida como Inner Class ou Member Class)

class OuterClass {
    ...
    class InnerClass {
        ...
    }
}

Definido Estaticamente em Outra Classe #

##C++## Classe Aninhada Estática

class Outer {
   class Inner {
      ...
   };
};

##Java## Classe aninhada estática (também conhecida como classe de membro estático)[ref]

class OuterClass {
    ...
    static class StaticNestedClass {
        ...
    }
}

Definido dentro de um Método #

(por exemplo, manipulação de eventos) ##C++## Classe local[ref]

void fun() {
   class Test {
      /* members of Test class */
   };
}

Veja também expressões Lambda

##Java## Classe local[ref]

class Test {
    void f() {
        new Thread(new Runnable() {
            public void run() {
                doSomethingBackgroundish();
            }
        }).start();
    }
}

Sobreposição vs Sobrecarga #

Os seguintes pontos de Overriding vs Overloading se aplicam a C++ e Java:

  • Um método substituído tem o mesmo nome e argumentos de sua base método.
  • Um método sobrecarregado tem o mesmo nome, mas diferente argumentos e não depende de herança.
  • Dois métodos com o mesmo nome e argumentos, mas tipo de retorno diferente, são ilegais. Consulte as perguntas relacionadas do Stackoverflow sobre “sobrecarregar com diferentes tipos de retorno em Java” - Pergunta 1; Pergunta 2

Polimorfismo #

Polimorfismo é a capacidade de objetos de diferentes classes relacionadas por herança responderem de forma diferente à mesma chamada de método. Aqui está um exemplo:

  • classe base Shape com area como método abstrato
  • duas classes derivadas, Square e Circle, implementam métodos de área
  • Pontos de referência de forma para Quadrado e área são invocados

Em C++, o polimorfismo é habilitado por métodos virtuais. Em Java, os métodos são virtuais por padrão.

Ordem de Construção/Destruição #

Limpeza de objetos #

Em C++, é uma boa ideia declarar um destruidor como virtual para garantir que o destruidor da subclasse seja chamado se o ponteiro da classe base for excluído.

Em Java, um método finalize é semelhante a um destruidor em C++; no entanto, os finalizadores são imprevisíveis (eles dependem do GC). Prática recomendada - use um método “fechar” para limpar explicitamente.

protected void close() {
    try {
       // do subclass cleanup
    }
    finally {
       isClosed = true;
       super.close();
    }
}

protected void finalize() {
    try {
       if(!isClosed) close();
    }
    finally {
       super.finalize();
    }
}

O Temido Problema do Diamante #

O problema do diamante é uma ambiguidade que surge quando duas classes B e C herda de A e a classe D herda de B e C. Se houver um método em A que B e C substituíram, e D não substitui isso, então qual versão do método D herda: a de B, ou o de C? (da Wikipedia)

Enquanto o C++ sempre foi suscetível ao problema do diamante, o Java era suscetível até o Java 8. Originalmente, o Java não suportava herança múltipla, mas com o advento dos métodos de interface padrão, as classes Java não podem herdar “implementação” de mais de uma classe .

java.lang.Object Class #

Em Java todas as classes herdam, implícita ou explicitamente, da classe Object. Qualquer referência Java pode ser convertida para o tipo Object.

C++ não tem uma classe “Object” comparável.

Membros de classe estáticos #

Membros estáticos têm escopo de classe em oposição ao escopo de objeto

C++ Exemplo #

// define in header
class Singleton {
   public:
      static Singleton *getInstance();

   private:
      Singleton() {}
      static Singleton *instance;
};

// initialize in .cpp
Singleton* Singleton::instance = 0;

Java Exemplo #

public class Singleton {
    private static Singleton instance;
    
    private Singleton() {}
    
    public static Singleton getInstance() {
        if(instance == null) {
            instance = new Singleton();
        }
        return instance;
    }
}

Classes definidas em outras construções #

Definido dentro de Outra Classe #

##C++## Classe aninhada[ref] (precisa de uma referência à classe anexa)

class Outer {
   class Inner {
      public:
         Inner(Outer* o) :outer(o) {}

      private:
         Outer*  outer;
   };
};

##Java## [não estático] Classe aninhada (também conhecida como Inner Class ou Member Class)

class OuterClass {
    ...
    class InnerClass {
        ...
    }
}

Definido Estaticamente em Outra Classe #

##C++## Classe Aninhada Estática

class Outer {
   class Inner {
      ...
   };
};

##Java## Classe aninhada estática (também conhecida como classe de membro estático)[ref]

class OuterClass {
    ...
    static class StaticNestedClass {
        ...
    }
}

Passagem por valor e passagem por referência #

Muitos argumentam que Java é APENAS passagem por valor, mas é mais sutil do que isso. Compare os exemplos de C++ e Java a seguir para ver os vários tipos de passagem por valor (também conhecido como cópia) e passagem por referência (também conhecido como alias).

Exemplo C++ (código completo) #

  // passes a COPY of the object
  static void passByCopy(PassIt obj) {
     obj.i = 22;  // only a "local" change
  }

  // passes a pointer
  static void passByPointer(PassIt* ptr) {
     ptr->i = 33;
     ptr = 0; // better to use nullptr instead if '0'
  }

  // passes an alias (aka reference)
  static void passByAlias(PassIt& ref) {
     ref.i = 44;
  }

  // This is an old-school way of doing it.
  // Check out std::swap for the best way to do this
  static void swap(PassIt** pptr1, PassIt** pptr2) {
     PassIt* tmp = *pptr1;
     *pptr1 = *pptr2;
     *pptr2 = tmp;
  }

Exemplo Java (código completo) #

   // passes a copy of the variable
   // NOTE: in java only primitives are pass-by-copy
   public static void passByCopy(int copy) {
      copy = 33;  // only a "local" change
   }

   // No such thing as pointers in Java
   /*
   public static void passByPointer(PassIt *ptr) {
      ptr->i = 33;
      ptr = 0; // better to use nullptr instead if '0'
   }
   */

   // passes an alias (aka reference)
   public static void passByAlias(PassIt ref) {
      ref.i = 44;
   }

   // passes aliases (aka references),
   // but need to do "manual", potentially expensive copies
   public static void swap(PassIt ref1, PassIt ref2) {
      PassIt tmp = new PassIt(ref1);
      ref1.copy(ref2);
      ref2.copy(tmp);
   }

Herança vs Composição #

C++ e Java são linguagens orientadas a objetos, portanto, o diagrama a seguir se aplica a ambas.

Rebaixamento do Proscrito #

Cuidado ao usar “downcasting” - Downcasting é reduzir a hierarquia de herança de uma classe base para uma subclasse (ou seja, o oposto do polimorfismo). Em geral, use polimorfismo & substituição em vez de instanceof & downcasting.

Exemplo C++ #

// explicit type case required
Child *pChild =  (Child *) &parent;

Exemplo Java #

if(mySubClass instanceof SubClass) {
   SubClass mySubClass = (SubClass)someBaseClass;
   mySubClass.nonInheritedMethod();
}

Métodos e classes abstratas #

Método Abstrato #

declarado sem uma implementação ##C++## método virtual puro

virtual void eat(void) = 0;

##Java## método abstrato

abstract void draw();

Aula Abstrata #

não pode ser instanciado ##C++## não pode ser instanciado; tem pelo menos 1 método virtual puro

class AB {public: virtual void f() = 0;};

##Java## não pode ser instanciado; pode ter métodos não abstratos

abstract class GraphicObject {}

#Interface# nenhum campo de instância ##C++## nada comparável ao Java

##Java## muito semelhante à classe abstrata, mas 1) suporta herança múltipla; 2) nenhum campo de instância

interface TestInterface {}
