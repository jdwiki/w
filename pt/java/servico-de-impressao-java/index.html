A Java Print Service API fornece funcionalidades para descobrir serviços de impressão e enviar solicitações de impressão para eles .

Inclui atributos de impressão extensíveis com base nos atributos padrão especificados no Internet Printing Protocol (IPP) 1.1 da IETF Specification, [RFC 2911](https:/ /tools.ietf.org/html/rfc2911).

Construindo o Doc que será impresso #

Doc é uma interface e a API Java Print Service fornece uma implementação simples chamada SimpleDoc.

Cada instância Doc é basicamente composta por dois aspectos:

  • o próprio conteúdo dos dados de impressão (um e-mail, uma imagem, um documento etc.)
  • o formato de dados de impressão, chamado DocFlavor (tipo MIME + classe Representation).

Antes de criar o objeto Doc, precisamos carregar nosso documento de algum lugar. No exemplo, carregaremos um arquivo específico do disco:

FileInputStream pdfFileInputStream = new FileInputStream("something.pdf");

Então agora, temos que escolher um DocFlavor que corresponda ao nosso conteúdo. A classe DocFlavor tem um monte de constantes para representar os tipos mais usuais de dados. Vamos escolher o INPUT_STREAM.PDF:

DocFlavor pdfDocFlavor = DocFlavor.INPUT_STREAM.PDF;

Agora, podemos criar uma nova instância de SimpleDoc:

Doc doc = new SimpleDoc(pdfFileInputStream, pdfDocFlavor , null);

O objeto doc agora pode ser enviado para a solicitação do trabalho de impressão (consulte [Criando um trabalho de impressão a partir de um serviço de impressão](https://www.wikiod.com/pt/java/servico-de-impressao-java#Criando um trabalho de impressão de um serviço de impressão -a-impressão-trabalho-de-um-serviço de impressão)).

Descobrindo os serviços de impressão disponíveis #

Para descobrir todos os serviços de impressão disponíveis, podemos usar a classe PrintServiceLookup. Vamos ver como:

import javax.print.PrintService;
import javax.print.PrintServiceLookup;

public class DiscoveringAvailablePrintServices {

    public static void main(String[] args) {
        discoverPrintServices();
    }

    public static void discoverPrintServices() {
        PrintService[] allPrintServices = PrintServiceLookup.lookupPrintServices(null, null);
        
        for (Printservice printService : allPrintServices) {
            System.out.println("Print service name: " + printService.getName());
        }
    }

}

Este programa, quando executado em ambiente Windows, imprimirá algo assim:

Print service name: Fax
Print service name: Microsoft Print to PDF
Print service name: Microsoft XPS Document Viewer

Definindo atributos de solicitação de impressão #

Às vezes, precisamos determinar alguns aspectos da solicitação de impressão. Vamos chamá-los de atributo.

São exemplos de atributos de solicitação de impressão:

  • quantidade de cópias (1, 2 etc),
  • orientação (retrato ou paisagem)
  • cromacidade (monocromático, colorido)
  • qualidade (rascunho, normal, alta)
  • lados (um lado, dois lados, etc.)
  • e assim por diante…

Antes de escolher um deles e qual valor cada um terá, primeiro precisamos construir um conjunto de atributos:

PrintRequestAttributeSet pras = new HashPrintRequestAttributeSet();

Agora podemos adicioná-los. Alguns exemplos são:

pras.add(new Copies(5));
pras.add(MediaSize.ISO_A4);
pras.add(OrientationRequested.PORTRAIT);
pras.add(PrintQuality.NORMAL);

O objeto pras agora pode ser enviado para a solicitação do trabalho de impressão (consulte [Criando um trabalho de impressão a partir de um serviço de impressão](https://www.wikiod.com/pt/java/servico-de-impressao-java#Criando um trabalho de impressão de um serviço de impressão -a-impressão-trabalho-de-um-serviço de impressão)).

Descobrindo o serviço de impressão padrão #

Para descobrir o serviço de impressão padrão, podemos usar a classe PrintServiceLookup. Vejamos como::

import javax.print.PrintService;
import javax.print.PrintServiceLookup;

public class DiscoveringDefaultPrintService {

    public static void main(String[] args) {
        discoverDefaultPrintService();
    }

    public static void discoverDefaultPrintService() {
        PrintService defaultPrintService = PrintServiceLookup.lookupDefaultPrintService();
        System.out.println("Default print service name: " + defaultPrintService.getName());
    }

}

Criando um trabalho de impressão de um serviço de impressão #

Um trabalho de impressão é uma solicitação de impressão de algo em um serviço de impressão específico. Consiste, basicamente, em:

  • os dados que serão impressos (consulte [Construindo o documento que será impresso](https://www.wikiod.com/pt/java/servico-de-impressao-java#Construindo o Doc que será impresso -ser impresso))
  • um conjunto de atributos

Após selecionar a instância correta do serviço de impressão, podemos solicitar a criação de um trabalho de impressão:

DocPrintJob printJob = printService.createPrintJob();

A interface DocPrintJob nos fornece o método print:

printJob.print(doc, pras);

O argumento doc é um Doc: os dados que serão impressos.

E o argumento pras é uma interface PrintRequestAttributeSet: um conjunto de PrintRequestAttribute. São exemplos de atributos de solicitação de impressão:

  • quantidade de cópias (1, 2 etc),
  • orientação (retrato ou paisagem)
  • cromacidade (monocromático, colorido)
  • qualidade (rascunho, normal, alta)
  • lados (um lado, dois lados, etc.)
  • e assim por diante…

O método print pode lançar um PrintException.

Ouvindo alteração de status de solicitação de trabalho de impressão #

Para a maioria dos clientes de impressão, é extremamente útil saber se um trabalho de impressão foi concluído ou falhou.

A API Java Print Service fornece algumas funcionalidades para se informar sobre esses cenários. Tudo o que temos a fazer é:

  • fornecer uma implementação para a interface PrintJobListener e
  • registre esta implementação no trabalho de impressão.

Quando o estado do trabalho de impressão mudar, seremos notificados. Podemos fazer o que for necessário, por exemplo:

  • atualizar uma interface de usuário,
  • iniciar outro processo de negócios,
  • gravar algo no banco de dados,
  • ou simplesmente registre-o.

No exemplo abaixo, registraremos todas as alterações de status do trabalho de impressão:

import javax.print.event.PrintJobEvent;
import javax.print.event.PrintJobListener;

public class LoggerPrintJobListener implements PrintJobListener {

    // Your favorite Logger class goes here!
    private static final Logger LOG = Logger.getLogger(LoggerPrintJobListener.class);


    public void printDataTransferCompleted(PrintJobEvent pje) {
        LOG.info("Print data transfer completed ;) ");
    }

    public void printJobCompleted(PrintJobEvent pje) {
        LOG.info("Print job completed =) ");
    }

    public void printJobFailed(PrintJobEvent pje) {
        LOG.info("Print job failed =( ");
    }

    public void printJobCanceled(PrintJobEvent pje) {
        LOG.info("Print job canceled :| ");
    }

    public void printJobNoMoreEvents(PrintJobEvent pje) {
        LOG.info("No more events to the job ");
    }

    public void printJobRequiresAttention(PrintJobEvent pje) {
        LOG.info("Print job requires attention :O ");
    }
}

Por fim, podemos adicionar nossa implementação de ouvinte de trabalho de impressão no trabalho de impressão antes da própria solicitação de impressão, da seguinte maneira:

DocPrintJob printJob = printService.createPrintJob();

printJob.addPrintJobListener(new LoggerPrintJobListener());

printJob.print(doc, pras);

O argumento PrintJobEvent tje #

Observe que todo método tem um argumento PrintJobEvent pje. Não o usamos neste exemplo para fins de simplicidade, mas você pode usá-lo para explorar o status. Por exemplo:

pje.getPrintJob().getAttributes();

Retornará uma instância do objeto PrintJobAttributeSet e você poderá executá-los de uma maneira for-each.

Outra maneira de atingir o mesmo objetivo #

Outra opção para atingir o mesmo objetivo é estender a classe PrintJobAdapter, como o próprio nome diz, é um adaptador para PrintJobListener. Implementando a interface temos obrigatoriamente que implementar todas elas. A vantagem desta forma é que precisamos substituir apenas os métodos que queremos. Vamos ver como isso funciona:

import javax.print.event.PrintJobEvent;
import javax.print.event.PrintJobAdapter;

public class LoggerPrintJobAdapter extends PrintJobAdapter {

    // Your favorite Logger class goes here!
    private static final Logger LOG = Logger.getLogger(LoggerPrintJobAdapter.class);

    public void printJobCompleted(PrintJobEvent pje) {
        LOG.info("Print job completed =) ");
    }

    public void printJobFailed(PrintJobEvent pje) {
        LOG.info("Print job failed =( ");
    }
}

Observe que sobrescrevemos apenas alguns métodos específicos.

Da mesma forma que no exemplo de implementação da interface PrintJobListener, adicionamos o listener ao trabalho de impressão antes de enviá-lo para impressão:

printJob.addPrintJobListener(new LoggerPrintJobAdapter());

printJob.print(doc, pras);
