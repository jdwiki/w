A recursão ocorre quando um método chama a si mesmo. Esse método é chamado de recursivo. Um método recursivo pode ser mais conciso do que uma abordagem não recursiva equivalente. No entanto, para recursão profunda, às vezes uma solução iterativa pode consumir menos espaço de pilha finito de um thread.

Este tópico inclui exemplos de recursão em Java.

Projetando um método recursivo #

Ao projetar um método recursivo, lembre-se de que você precisa:

  • Base Case. Isso definirá quando sua recursão irá parar e produzir o resultado. O caso base no exemplo fatorial é:

     if (n <= 1) {
     return 1;
 }

  • Chamada recursiva. Nesta instrução você chama novamente o método com um parâmetro alterado. A chamada recursiva no exemplo fatorial acima é:

     else {
     return n * factorial(n - 1);
 }

Resultado #

Neste exemplo, você calcula o n-ésimo número fatorial. Os primeiros fatoriais são:

0! = 1

1! = 1

2! = 1 x 2 = 2

3! = 1 x 2 x 3 = 6

4! = 1 x 2 x 3 x 4 = 24

…

Eliminação de Java e Tail-call #

Os compiladores Java atuais (até e incluindo Java 9) não executam a eliminação de chamadas finais. Isso pode afetar o desempenho de algoritmos recursivos, e se a recursão for profunda o suficiente, pode levar a travamentos do StackOverflowError; consulte https://www.wikiod.com/pt/java/recursao#A recursão profunda é problemática em Java

A recursão profunda é problemática em Java #

Considere o seguinte método ingênuo para adicionar dois números positivos usando recursão:

public static int add(int a, int b) {
    if (a == 0) {
        return b;
    } else {
        return add(a - 1, b + 1);  // TAIL CALL
    }
}

Isso é algoritmicamente correto, mas tem um grande problema. Se você chamar add com um a grande, ele irá travar com um StackOverflowError, em qualquer versão do Java até (pelo menos) Java 9.

Em uma linguagem de programação funcional típica (e muitas outras linguagens), o compilador otimiza [recursão de cauda] [1]. O compilador notaria que a chamada para add (na linha marcada) é uma [chamada final][2], e efetivamente reescreveria a recursão como um loop. Essa transformação é chamada de eliminação de chamada de cauda.

No entanto, os compiladores Java da geração atual não executam a eliminação de chamadas finais. (Este não é um simples descuido. Existem razões técnicas substanciais para isso; veja abaixo.) Em vez disso, cada chamada recursiva de add faz com que um novo quadro seja alocado na pilha da thread. Por exemplo, se você chamar add(1000, 1), serão necessárias 1000 chamadas recursivas para chegar à resposta 1001.

O problema é que o tamanho da pilha de encadeamentos Java é fixo quando o encadeamento é criado. (Isso inclui o thread “principal” em um programa de thread único.) Se muitos quadros de pilha forem alocados, a pilha transbordará. A JVM detectará isso e lançará um StackOverflowError.

Uma abordagem para lidar com isso é simplesmente usar uma pilha maior. Existem opções de JVM que controlam o tamanho padrão de uma pilha e você também pode especificar o tamanho da pilha como um parâmetro do construtor Thread. Infelizmente, isso apenas “adianta” o estouro da pilha. Se você precisar fazer um cálculo que exija uma pilha ainda maior, o StackOverflowError volta.

A solução real é identificar algoritmos recursivos onde a recursão profunda é provável e manualmente realizar a otimização de chamada de cauda no nível do código-fonte. Por exemplo, nosso método add pode ser reescrito da seguinte forma:

public static int add(int a, int b) {
    while (a != 0) {
       a = a - 1;
       b = b + 1;
    }
    return b;
}

(Obviamente, existem maneiras melhores de adicionar dois inteiros. O acima é simplesmente para ilustrar o efeito da eliminação manual da chamada de cauda.)

Por que a eliminação de chamada de cauda não está implementada em Java (ainda) #

Há uma série de razões pelas quais adicionar eliminação de chamada de cauda ao Java não é fácil. Por exemplo:

  • Alguns códigos podem contar com StackOverflowError para (por exemplo) colocar um limite no tamanho de um problema computacional.
  • Os gerentes de segurança de sandbox geralmente confiam na análise da pilha de chamadas ao decidir se permitem que um código não privilegiado execute uma ação privilegiada.

Como John Rose explica em [“Tail calls in the VM”][3]:

“Os efeitos da remoção do quadro de pilha do chamador são visíveis para algumas APIs, notadamente verificações de controle de acesso e rastreamento de pilha. É como se o chamador do chamador tivesse chamado diretamente o receptor. Quaisquer privilégios do chamador são descartados após a transferência do controle No entanto, a ligação e a acessibilidade do método chamado são computadas antes da transferência do controle e levam em consideração o chamador de chamada final.”

Em outras palavras, a eliminação da chamada final pode fazer com que um método de controle de acesso pense erroneamente que uma API sensível à segurança estava sendo chamada por um código confiável.

A ideia básica de recursão #

O que é recursão:

Em geral, a recursão é quando uma função invoca a si mesma, direta ou indiretamente. Por exemplo:

// This method calls itself "infinitely"
public void useless() {
    useless();  // method calls itself (directly)
}

Condições para aplicar a recursão a um problema:

Existem duas pré-condições para usar funções recursivas para resolver um problema específico:

  1. Deve haver uma condição base para o problema, que será o ponto final da recursão. Quando uma função recursiva atinge a condição base, ela não faz mais chamadas recursivas (mais profundas).

  2. Cada nível de recursão deve tentar um problema menor. A função recursiva, portanto, divide o problema em partes cada vez menores. Assumindo que o problema é finito, isso garantirá que a recursão termine.

Em Java há uma terceira pré-condição: não deve ser necessário recorrer muito profundamente para resolver o problema; consulte https://www.wikiod.com/pt/java/recursao#A recursão profunda é problemática em Java

Exemplo

A função a seguir calcula fatoriais usando recursão. Observe como o método factorial chama a si mesmo dentro da função. Cada vez que ela chama a si mesma, ela reduz o parâmetro n em 1. Quando n atinge 1 (a condição base) a função não irá recorrer mais profundamente.

public int factorial(int n) {
    if (n <= 1) { // the base condition 
        return 1;
    } else {
        return n * factorial(n - 1);
    }
}

Esta não é uma maneira prática de calcular fatoriais em Java, pois não leva em consideração o estouro de inteiros ou estouro de pilha de chamadas (ou seja, exceções StackOverflowError) para grandes valores de n.

Tipos de recursão #

A recursão pode ser categorizada como Head Recursion ou Tail Recursion, dependendo de onde a chamada do método recursivo é colocada.

Na recursão de cabeça, a chamada recursiva, quando acontece, vem antes de outro processamento na função (pense nela acontecendo no topo, ou cabeça, da função).

Na recursão de cauda, é o oposto — o processamento ocorre antes da chamada recursiva. Escolher entre os dois estilos recursivos pode parecer arbitrário, mas a escolha pode fazer toda a diferença.

Uma função com um caminho com uma única chamada recursiva no início do caminho usa o que é chamado de recursão de cabeça. A função fatorial de uma exibição anterior usa recursão principal. A primeira coisa que ele faz quando determina que a recursão é necessária é chamar a si mesmo com o parâmetro decrementado. Uma função com uma única chamada recursiva no final de um caminho está usando recursão de cauda.

public void tail(int n)              public void head(int n)
{                                       {
   if(n == 1)                             if(n == 0)
      return;                                return;
   else                                   else
      System.out.println(n);                 head(n-1);

   tail(n-1);                              System.out.println(n);
}                                        }

Se a chamada recursiva ocorre no final de um método, ela é chamada de recursão de cauda. A recursão da cauda é ‘semelhante a um loop’. O método executa todas as instruções antes de saltar para a próxima chamada recursiva.

Se a chamada recursiva ocorre no início de um método, ela é chamada de recursão principal. O método salva o estado antes de pular para a próxima chamada recursiva.

Referência: http://stackoverflow.com/questions/21426688/the-difference-between-head-tail-recursion

Calculando o enésimo número de Fibonacci #

O método a seguir calcula o enésimo número de Fibonacci usando recursão.

public int fib(final int n) {
    if (n > 2) {
        return fib(n - 2) + fib(n - 1);
    }
    return 1;
}

O método implementa um caso base (n <= 2) e um caso recursivo (n>2). Isso ilustra o uso da recursão para calcular uma relação recursiva.

No entanto, embora este exemplo seja ilustrativo, também é ineficiente: cada instância do método chamará a própria função duas vezes, levando a um crescimento exponencial no número de vezes que a função é chamada à medida que N aumenta. A função acima é O(2N), mas uma solução iterativa equivalente tem complexidade O(N). Além disso, há uma expressão de “forma fechada” que pode ser avaliada em multiplicações de ponto flutuante O(N).

Calculando a enésima potência de um número #

O método a seguir calcula o valor de num elevado à potência de exp usando recursão:

public long power(final int num, final int exp) {
    if (exp == 0) {
        return 1;
    }
    if (exp == 1) {
        return num;
    }
    return num * power(num, exp - 1);
}

Isso ilustra os princípios mencionados acima: o método recursivo implementa um caso base (dois casos, n = 0 e n = 1) que encerra a recursão e um caso recursivo que chama o método novamente. Esse método é O(N) e pode ser reduzido a um loop simples usando otimização de chamada de cauda.

Percorrendo uma estrutura de dados de árvore com recursão #

Considere a classe Node com dados de 3 membros, ponteiro filho esquerdo e ponteiro filho direito como abaixo.

public class Node {
    public int data;
    public Node left;
    public Node right;
    
    public Node(int data){
        this.data = data;
    }
}

Podemos percorrer a árvore construída conectando vários objetos da classe Node como abaixo, a travessia é chamada de travessia em ordem da árvore.

public static void inOrderTraversal(Node root) {
        if (root != null) {          
            inOrderTraversal(root.left); // traverse left sub tree
            System.out.print(root.data + " "); // traverse current node
            inOrderTraversal(root.right); // traverse right sub tree
        }
    }

Conforme demonstrado acima, usando recursão, podemos percorrer a estrutura de dados em árvore sem usar nenhuma outra estrutura de dados, o que não é possível com a abordagem iterativa.

StackOverflowError e recursão para fazer um loop #

Se uma chamada recursiva for “muito profunda”, isso resultará em um StackOverflowError. Java aloca um novo quadro para cada chamada de método na pilha de seu encadeamento. No entanto, o espaço da pilha de cada thread é limitado. Muitos quadros na pilha levam ao Stack Overflow (SO).

Exemplo #

public static void recursion(int depth) {
    if (depth > 0) {
        recursion(depth-1);
    }
}

Chamar este método com parâmetros grandes (por exemplo, recursion(50000) provavelmente resultará em um estouro de pilha. O valor exato depende do tamanho da pilha de encadeamento, que por sua vez depende da construção do encadeamento, parâmetros de linha de comando como -Xss , ou o tamanho padrão para a JVM.

Gambiarra #

Uma recursão pode ser convertida em um loop armazenando os dados para cada chamada recursiva em uma estrutura de dados. Essa estrutura de dados pode ser armazenada no heap em vez de na pilha de threads.

Em geral, os dados necessários para restaurar o estado de uma invocação de método podem ser armazenados em uma pilha e um loop while pode ser usado para “simular” as chamadas recursivas. Os dados que podem ser necessários incluem:

  • o objeto para o qual o método foi chamado (somente métodos de instância)
  • os parâmetros do método *variáveis ​​locais
  • a posição atual na execução ou o método

Exemplo #

A classe a seguir permite a impressão recursiva de uma estrutura de árvore até uma profundidade especificada.

public class Node {

    public int data;
    public Node left;
    public Node right;

    public Node(int data) {
        this(data, null, null);
    }

    public Node(int data, Node left, Node right) {
        this.data = data;
        this.left = left;
        this.right = right;
    }

    public void print(final int maxDepth) {
        if (maxDepth <= 0) {
            System.out.print("(...)");
        } else {
            System.out.print("(");
            if (left != null) {
                left.print(maxDepth-1);
            }
            System.out.print(data);
            if (right != null) {
                right.print(maxDepth-1);
            }
            System.out.print(")");
        }
    }

}

por exemplo.

Node n = new Node(10, new Node(20, new Node(50), new Node(1)), new Node(30, new Node(42), null));
n.print(2);
System.out.println();

Impressões

(((...)20(...))10((...)30))

Isso pode ser convertido para o seguinte loop:

public class Frame {

    public final Node node;

    // 0: before printing anything
    // 1: before printing data
    // 2: before printing ")"
    public int state = 0;
    public final int maxDepth;

    public Frame(Node node, int maxDepth) {
        this.node = node;
        this.maxDepth = maxDepth;
    }

}

List<Frame> stack = new ArrayList<>();
stack.add(new Frame(n, 2)); // first frame = initial call

while (!stack.isEmpty()) {
    // get topmost stack element
    int index = stack.size() - 1;
    Frame frame = stack.get(index); // get topmost frame
    if (frame.maxDepth <= 0) {
        // termial case (too deep)
        System.out.print("(...)");
        stack.remove(index); // drop frame
    } else {
        switch (frame.state) {
            case 0:
                frame.state++;

                // do everything done before the first recursive call
                System.out.print("(");
                if (frame.node.left != null) {
                    // add new frame (recursive call to left and stop)
                    stack.add(new Frame(frame.node.left, frame.maxDepth - 1));
                    break;
                }
            case 1:
                frame.state++;

                // do everything done before the second recursive call
                System.out.print(frame.node.data);
                if (frame.node.right != null) {
                    // add new frame (recursive call to right and stop)
                    stack.add(new Frame(frame.node.right, frame.maxDepth - 1));
                    break;
                }
            case 2:
                // do everything after the second recursive call & drop frame
                System.out.print(")");
                stack.remove(index);
        }
    }
}
System.out.println();

Observação: este é apenas um exemplo da abordagem geral. Muitas vezes, você pode encontrar uma maneira muito melhor de representar um quadro e/ou armazenar os dados do quadro.

Calculando a soma de inteiros de 1 a N #

O método a seguir calcula a soma de inteiros de 0 a N usando recursão.

public int sum(final int n) {
    if (n > 0) {
        return n + sum(n - 1);
    } else {
        return n;
    }
}

Esse método é O(N) e pode ser reduzido a um loop simples usando otimização de chamada de cauda. Na verdade, existe uma expressão de forma fechada que calcula a soma em operações O(1).
