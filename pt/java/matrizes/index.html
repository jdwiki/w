Os arrays permitem o armazenamento e a recuperação de uma quantidade arbitrária de valores. Eles são análogos aos vetores em matemática. Matrizes de matrizes são análogas a matrizes e atuam como matrizes multidimensionais. Arrays podem armazenar quaisquer dados de qualquer tipo: primitivos como int ou tipos de referência como Object.

Sintaxe #

  • ArrayType[] myArray; // Declarando arrays
  • ArrayType myArray[]; // Outra sintaxe válida (menos comumente usada e desencorajada)
  • ArrayType[][][] myArray; // Declarando arrays multidimensionais irregulares (repetir []s)
  • ArrayType myVar = myArray[index]; // Acessando (lendo) o elemento no índice
  • myArray[index] = value; // Atribui valor à posição index do array
  • ArrayType[] myArray = new ArrayType[arrayLength]; // Sintaxe de inicialização do array
  • int[] ints = {1, 2, 3}; // Sintaxe de inicialização do array com valores fornecidos, comprimento é inferido a partir do número de valores fornecidos: {[value1[, value2]*]}
  • new int[]{4, -5, 6} // Pode ser usado como argumento, sem uma variável local
  • int[] ints = new int[3]; // igual a {0, 0, 0}
  • int[][] ints = {{1, 2}, {3}, null}; // Inicialização de array multidimensional. int[] estende Object (assim como anyType[]) então null é um valor válido.

Parâmetros #

Parâmetro Detalhes
Tipo de Matriz Tipo da matriz. Isso pode ser primitivo (int, long, byte) ou Objetos (String, MyObject, etc).
índice Índice refere-se à posição de um determinado objeto em uma matriz.
comprimento Cada array, ao ser criado, precisa de um comprimento definido especificado. Isso é feito ao criar um array vazio (new int[3]) ou implícito ao especificar valores ({1, 2, 3}).

Criando e inicializando arrays #

Casos básicos #

int[]   numbers1 = new int[3];                 // Array for 3 int values, default value is 0
int[]   numbers2 = { 1, 2, 3 };                // Array literal of 3 int values
int[]   numbers3 = new int[] { 1, 2, 3 };      // Array of 3 int values initialized
int[][] numbers4 = { { 1, 2 }, { 3, 4, 5 } };  // Jagged array literal
int[][] numbers5 = new int[5][];               // Jagged array, one dimension 5 long

Introdução #

Um array é uma estrutura de dados que contém um número fixo de valores primitivos ou referências a instâncias de objetos.

Cada item em uma matriz é chamado de elemento e cada elemento é acessado por seu índice numérico. O comprimento de um array é estabelecido quando o array é criado:

int size = 42;
int[] array = new int[size];

O tamanho de um array é fixo em tempo de execução quando inicializado. Não pode ser alterado após a inicialização. Se o tamanho deve ser mutável em tempo de execução, uma classe Collection como ArrayList deve ser usada. ArrayList armazena elementos em uma matriz e suporta [redimensionamento alocando uma nova matriz](https://www.wikiod.com/pt/java/matrizes#Como alterar o tamanho de um array? ) e copiando elementos do array antigo.

Se a matriz for de um tipo primitivo, ou seja,

int[] array1 = { 1,2,3 };
int[] array2 = new int[10];

os valores são armazenados na própria matriz. Na ausência de um inicializador (como em array2 acima), o valor padrão atribuído a cada elemento é 0 (zero).

Se o tipo de array for uma referência de objeto, como em

SomeClassOrInterface[] array = new SomeClassOrInterface[10];

então o array contém references para objetos do tipo SomeClassOrInterface. Essas referências podem se referir a uma instância de SomeClassOrInterface ou qualquer subclasse (para classes) ou classe de implementação (para interfaces) de SomeClassOrInterface. Se a declaração da matriz não tiver inicializador, o valor padrão de null será atribuído a cada elemento.

Como todos os arrays são indexados por int, o tamanho de um array deve ser especificado por um int. O tamanho do array não pode ser especificado como um long:

long size = 23L;
int[] array = new int[size]; // Compile-time error:
                             // incompatible types: possible lossy conversion from
                             // long to int

Arrays usam um sistema de índice baseado em zero, o que significa que a indexação começa em ‘0’ e termina em ‘comprimento - 1’.

Por exemplo, a imagem a seguir representa um array com tamanho 10. Aqui, o primeiro elemento está no índice ‘0’ e o último elemento está no índice ‘9’, em vez do primeiro elemento estar no índice ‘1’ e o último elemento no índice ‘10’ (veja a figura abaixo).

Os acessos aos elementos dos arrays são feitos em tempo constante. Isso significa que acessar o primeiro elemento da matriz tem o mesmo custo (em tempo) de acessar o segundo elemento, o terceiro elemento e assim por diante.

Java oferece várias maneiras de definir e inicializar arrays, incluindo notações literais e construtor. Ao declarar arrays usando o construtor new Type[length], cada elemento será inicializado com os seguintes valores padrão:

  • 0 para tipos numéricos primitivos: byte, short, int, long, float e double.
  • '\u0000' (caractere nulo) para o tipo char.
  • false para o tipo boolean.
  • null para [tipos de referência] docs-ref-types.

Criando e inicializando arrays de tipo primitivo #

int[] array1 = new int[] { 1, 2, 3 }; // Create an array with new operator and 
                                      // array initializer.
int[] array2 = { 1, 2, 3 };           // Shortcut syntax with array initializer.
int[] array3 = new int[3];            // Equivalent to { 0, 0, 0 }
int[] array4 = null;                  // The array itself is an object, so it
                                      // can be set as null.

Ao declarar um array, [] aparecerá como parte do tipo no início da declaração (após o nome do tipo), ou como parte do declarador para uma variável específica (após o nome da variável), ou ambos:

int array5[];       /* equivalent to */  int[] array5;
int a, b[], c[][];  /* equivalent to */  int a; int[] b; int[][] c;
int[] a, b[];       /* equivalent to */  int[] a; int[][] b;
int a, []b, c[][];  /* Compilation Error, because [] is not part of the type at beginning
                       of the declaration, rather it is before 'b'. */    
// The same rules apply when declaring a method that returns an array:
int foo()[] { ... } /* equivalent to */  int[] foo() { ... }

No exemplo a seguir, ambas as declarações estão corretas e podem ser compiladas e executadas sem problemas. No entanto, tanto a Java Coding Convention quanto o [Google Java Style Guide][google-docs-styleguide] desencorajam o formulário com colchetes após o nome da variável—os colchetes identificam o tipo de matriz e devem aparecer com a designação do tipo. O mesmo deve ser usado para assinaturas de retorno de método.

float array[]; /* and */ int foo()[] { ... } /* are discouraged */
float[] array; /* and */ int[] foo() { ... } /* are encouraged */

O tipo desencorajado é [destinado a acomodar usuários C em transição][so-qa-arraysyntax], que estão familiarizados com a sintaxe para C que tem os colchetes após o nome da variável.

Em Java, é possível ter arrays de tamanho 0:

int[] array = new int[0]; // Compiles and runs fine.
int[] array2 = {};        // Equivalent syntax.

No entanto, como é um array vazio, nenhum elemento pode ser lido ou atribuído a ele:

array[0] = 1;     // Throws java.lang.ArrayIndexOutOfBoundsException.
int i = array2[0]; // Also throws ArrayIndexOutOfBoundsException.

Esses arrays vazios são normalmente úteis como valores de retorno, de modo que o código de chamada só precisa se preocupar em lidar com um array, em vez de um valor null potencial que pode levar a uma [NullPointerException][so-docs-npe] .

O comprimento de uma matriz deve ser um inteiro não negativo:

int[] array = new int[-1]; // Throws java.lang.NegativeArraySizeException

O tamanho do array pode ser determinado usando um campo final público chamado length:

System.out.println(array.length); // Prints 0 in this case.

Observação: array.length retorna o tamanho real do array e não o número de elementos do array que receberam um valor, diferente de [ArrayList.size()][arraylist-size-javadocs] que retorna o número de elementos da matriz que receberam um valor.

Criando e inicializando arrays multidimensionais #

A maneira mais simples de criar uma matriz multidimensional é a seguinte:

int[][] a = new int[2][3];

Ele criará dois arrays int de três comprimentos — a[0] e a[1]. Isso é muito semelhante à inicialização clássica em estilo C de matrizes multidimensionais retangulares.

Você pode criar e inicializar ao mesmo tempo:

int[][] a = { {1, 2}, {3, 4}, {5, 6} };

Ao contrário de C, onde somente arrays multidimensionais retangulares são suportados, arrays internos não precisam ter o mesmo comprimento, ou mesmo definidos:

int[][] a = { {1}, {2, 3}, null };

Aqui, a[0] é um array int de um comprimento, enquanto a[1] é um array int de dois comprimentos e a[2] é null. Arrays como este são chamados de [arrays irregulares ou arrays irregulares][so-docs-jaggedarrays], ou seja, são arrays de arrays. Arrays multidimensionais em Java são implementados como arrays de arrays, ou seja, array[i][j][k] é equivalente a ((array[i])[j])[k]. [Ao contrário de C#][so-qa-jaggeddifference], a sintaxe array[i,j] não é suportada em Java.

Criando e inicializando arrays tipo genérico #

Em classes genéricas, arrays de tipos genéricos não podem ser inicializados assim devido a [type erasure][docs-type-erasure]:

public class MyGenericClass<T> {
    private T[] a;

    public MyGenericClass() {
        a = new T[5]; // Compile time error: generic array creation
    }
}

Em vez disso, eles podem ser criados usando um dos seguintes métodos: (observe que isso gerará avisos não verificados)

  1. Criando um array Object e convertendo-o no tipo genérico:

    a = (T[]) new Object[5];

    This is the simplest method, but since the underlying array is still of type Object[], this method does not provide type safety. Therefore, this method of creating an array is best used only within the generic class - not exposed publicly.

  2. Usando Array.newInstance com um parâmetro de classe:

    public MyGenericClass(Class<T> clazz) {
    a = (T[]) Array.newInstance(clazz, 5);
}

    Here the class of T has to be explicitly passed to the constructor. The return type of Array.newInstance is always Object. However, this method is safer because the newly created array is always of type T[], and therefore can be safely externalized.

Declaração separada e inicialização de arrays #

O valor de um índice para um elemento de matriz deve ser um número inteiro (0, 1, 2, 3, 4, …) e menor que o comprimento da matriz (os índices são baseados em zero). Caso contrário, uma [ArrayIndexOutOfBoundsException][docs-aioobe] será lançada:

int[] array9;             // Array declaration - uninitialized
array9 = new int[3];      // Initialize array  - { 0, 0, 0 }
array9[0] = 10;           // Set index 0 value - { 10, 0, 0 }
array9[1] = 20;           // Set index 1 value - { 10, 20, 0 }
array9[2] = 30;           // Set index 2 value - { 10, 20, 30 }

Arrays não podem ser reinicializados com a sintaxe de atalho do inicializador de array #

não é possível reinicializar um array por meio de uma sintaxe de atalho com um inicializador de array, pois um inicializador de array só pode ser especificado em uma declaração de campo ou declaração de variável local, ou como parte de um expressão de criação de array.

No entanto, é possível criar um novo array e atribuí-lo à variável que está sendo usada para referenciar o array antigo. Embora isso resulte na reinicialização da matriz referenciada por essa variável, o conteúdo da variável é uma matriz completamente nova. Para fazer isso, o operador new pode ser usado com um inicializador de array e atribuído à variável de array:

// First initialization of array

// Prints "1 2 3 ".
for (int i : array) {
    System.out.print(i + " ");
}

// Re-initializes array to a new int[] array.
array = new int[] { 4, 5, 6 };

// Prints "4 5 6 ".
for (int i : array) {
    System.out.print(i + " ");
}

Criando uma lista a partir de um array #

O método Arrays.asList() pode ser usado para retornar uma List de tamanho fixo contendo os elementos do array fornecido. A Lista resultante será do mesmo tipo de parâmetro que o tipo base do array.

String[] stringArray = {"foo", "bar", "baz"};
List<String> stringList = Arrays.asList(stringArray);

Observação: Esta lista é apoiada por (uma visão) do array original, o que significa que qualquer alteração na lista irá alterar o array e vice-versa. No entanto, alterações na lista que alterariam seu tamanho (e, portanto, o comprimento da matriz) lançarão uma exceção.

Para criar uma cópia da lista, use o construtor de [java.util.ArrayList](https://docs.oracle.com/javase/8/docs/api/java/util/ArrayList.html#ArrayList- java.util.Collection-) tomando uma Collection como argumento:

String[] stringArray = {"foo", "bar", "baz"};
List<String> stringList = new ArrayList<String>(Arrays.asList(stringArray));

No Java SE 7 e posterior, um par de colchetes angulares <> (conjunto vazio de argumentos de tipo) pode ser usado, que é chamado de Diamond. O compilador pode determinar os argumentos de tipo do contexto. Isso significa que as informações de tipo podem ser omitidas ao chamar o construtor de ArrayList e serão inferidas automaticamente durante a compilação. Isso é chamado de Type Inference que faz parte do Java Generics.

// Using Arrays.asList()

String[] stringArray = {"foo", "bar", "baz"};
List<String> stringList = new ArrayList<>(Arrays.asList(stringArray));

// Using ArrayList.addAll()

String[] stringArray = {"foo", "bar", "baz"};
ArrayList<String> list = new ArrayList<>();
list.addAll(Arrays.asList(stringArray));

// Using Collections.addAll()

String[] stringArray = {"foo", "bar", "baz"};
ArrayList<String> list = new ArrayList<>();
Collections.addAll(list, stringArray);

Um ponto que vale a pena notar sobre o Diamond é que ele não pode ser usado com Anonymous Classes.

// Using Streams

int[] ints = {1, 2, 3};
List<Integer> list = Arrays.stream(ints).boxed().collect(Collectors.toList());

String[] stringArray = {"foo", "bar", "baz"};
List<Object> list = Arrays.stream(stringArray).collect(Collectors.toList());

Notas importantes relacionadas ao uso do método Arrays.asList() #

  • Este método retorna List, que é uma instância de Arrays$ArrayList(classe interna estática de Arrays) e não java.util.ArrayList. A Lista resultante é de tamanho fixo. Isso significa que adicionar ou remover elementos não é suportado e lançará um UnsupportedOperationException:

     stringList.add("something"); // throws java.lang.UnsupportedOperationException

  • Uma nova List pode ser criada passando uma List baseada em array para o construtor de uma nova List. Isso cria uma nova cópia dos dados, que tem tamanho variável e que não é apoiada pela matriz original:

     List<String> modifiableList = new ArrayList<>(Arrays.asList("foo", "bar"));

  • Chamando <T> List<T> asList(T... a) em um array primitivo, como um int[], irá produzir uma List<int[]> cujo único elemento é a fonte matriz primitiva em vez dos elementos reais da matriz de origem.

A razão para esse comportamento é que tipos primitivos não podem ser usados ​​no lugar de parâmetros de tipo genérico, portanto, todo o array primitivo substitui o parâmetro de tipo genérico neste caso. Para converter um array primitivo em uma List, primeiro converta o array primitivo em um array do tipo wrapper correspondente (ou seja, chame Arrays.asList em um Integer[] em vez de um int[ ]).

Portanto, isso imprimirá false:

   System.out.println(Arrays.asList(arr).contains(1));

Por outro lado, isso imprimirá true:

Criando um Array de uma Coleção #

Dois métodos em java.util.Collection criam um array a partir de uma coleção:

  • Object[] toArray()

  • <T> T[] paraArray(T[] a)

Object[] toArray() pode ser usado da seguinte forma:

Set<String> set = new HashSet<String>();
set.add("red");
set.add("blue");

// although set is a Set<String>, toArray() returns an Object[] not a String[]
Object[] objectArray = set.toArray();

<T> T[] toArray(T[] a) pode ser usado da seguinte forma:

Set<String> set = new HashSet<String>();
set.add("red");
set.add("blue");

// The array does not need to be created up front with the correct size.
// Only the array type matters. (If the size is wrong, a new array will
// be created with the same type.)
String[] stringArray = set.toArray(new String[0]);  

// If you supply an array of the same size as collection or bigger, it
// will be populated with collection values and returned (new array
// won't be allocated)
String[] stringArray2 = set.toArray(new String[set.size()]);

A diferença entre eles é mais do que apenas ter resultados não digitados versus digitados. O desempenho deles também pode diferir (para obter detalhes, leia esta seção de análise de desempenho):

  • Object[] toArray() usa arraycopy vetorizado, que é muito mais rápido do que o arraycopy verificado por tipo usado em T[] toArray(T[] a).
  • T[] toArray(new T[non-zero-size]) precisa zerar o array em tempo de execução, enquanto T[] toArray(new T[0]) não. Tal evitação faz com que a última chamada mais rápido que o anterior. Análise detalhada aqui: Arrays of Wisdom of the Ancients.

A partir do Java SE 8+, onde o conceito de Stream foi introduzido, é possível usar o Stream produzido pela coleção para criar um novo Array usando o Stream.toArray método.

String[] strings = list.stream().toArray(String[]::new);

Matrizes multidimensionais e irregulares #

É possível definir um array com mais de uma dimensão. Em vez de ser acessado fornecendo um único índice, uma matriz multidimensional é acessada especificando um índice para cada dimensão.

A declaração de array multidimensional pode ser feita adicionando [] para cada dimensão a uma declaração regular de array. Por exemplo, para criar um array int bidimensional, adicione outro conjunto de colchetes à declaração, como int[][]. Isso continua para arrays tridimensionais (int[][][]) e assim por diante.

Para definir uma matriz bidimensional com três linhas e três colunas:

int rows = 3;
int columns = 3;
int[][] table = new int[rows][columns];

A matriz pode ser indexada e atribuir valores a ela com essa construção. Observe que os valores não atribuídos são os valores padrão para o tipo de uma matriz, neste caso 0 para int.

table[0][0] = 0;
table[0][1] = 1;
table[0][2] = 2;

Também é possível instanciar uma dimensão de cada vez, e até mesmo fazer arrays não retangulares. Eles são mais comumente chamados de matrizes irregulares.

int[][] nonRect = new int[4][];

É importante notar que embora seja possível definir qualquer dimensão de um array irregular, seu nível anterior deve ser definido.

// valid
String[][] employeeGraph = new String[30][];

// invalid
int[][] unshapenMatrix = new int[][10];

// also invalid
int[][][] misshapenGrid = new int[100][][10];

Índice de matriz fora dos limites de exceção #

A ArrayIndexOutOfBoundsException é lançada quando um índice inexistente de uma matriz está sendo acessado.

As matrizes são indexadas com base em zero, portanto, o índice do primeiro elemento é 0 e o índice do último elemento é a capacidade da matriz menos 1 (ou seja, array.length - 1).

Portanto, qualquer solicitação para um elemento de array pelo índice i tem que satisfazer a condição 0 <= i < array.length, caso contrário o ArrayIndexOutOfBoundsException será lançado.

O código a seguir é um exemplo simples em que um ArrayIndexOutOfBoundsException é lançado.

String[] people = new String[] { "Carol", "Andy" };

// An array will be created:
// people[0]: "Carol"
// people[1]: "Andy"

// Notice: no item on index 2. Trying to access it triggers the exception:
System.out.println(people[2]);  // throws an ArrayIndexOutOfBoundsException.

Resultado:

Exception in thread "main" java.lang.ArrayIndexOutOfBoundsException: 2
    at your.package.path.method(YourClass.java:15)

Observe que o índice ilegal que está sendo acessado também está incluído na exceção (2 no exemplo); essas informações podem ser úteis para encontrar a causa da exceção.

Para evitar isso, basta verificar se o índice está dentro dos limites do array:

int index = 2;
if (index >= 0 && index < people.length) {
    System.out.println(people[index]);
}

Covariância de matriz #

Arrays de objetos são covariantes, o que significa que assim como Integer é uma subclasse de Number, Integer[] é uma subclasse de Number[]. Isso pode parecer intuitivo, mas pode resultar em um comportamento surpreendente:

Integer[] integerArray = {1, 2, 3};
Number[] numberArray = integerArray;  // valid
Number firstElement = numberArray[0]; // valid
numberArray[0] = 4L;                  // throws ArrayStoreException at runtime

Embora Integer[] seja uma subclasse de Number[], ele só pode conter Integers, e tentar atribuir um elemento Long lança uma exceção de tempo de execução.

Observe que esse comportamento é exclusivo para arrays e pode ser evitado usando um List genérico:

List<Integer> integerList = Arrays.asList(1, 2, 3);
//List<Number> numberList = integerList;  // compile error
List<? extends Number> numberList = integerList;
Number firstElement = numberList.get(0);
//numberList.set(0, 4L);                  // compile error

Não é necessário que todos os elementos do array compartilhem o mesmo tipo, desde que sejam uma subclasse do tipo do array:

interface I {}

class A implements I {}
class B implements I {}
class C implements I {}

I[] array10 = new I[] { new A(), new B(), new C() }; // Create an array with new
                                                     // operator and array initializer.

I[] array11 = { new A(), new B(), new C() };         // Shortcut syntax with array
                                                     // initializer.

I[] array12 = new I[3];                              // { null, null, null }

I[] array13 = new A[] { new A(), new A() };          // Works because A implements I.

Object[] array14 = new Object[] { "Hello, World!", 3.14159, 42 }; // Create an array with
                                                     // new operator and array initializer.

Object[] array15 = { new A(), 64, "My String" };     // Shortcut syntax 
                                                     // with array initializer.

Iterando sobre arrays #

Você pode iterar sobre matrizes usando o loop for aprimorado (também conhecido como foreach) ou usando índices de matriz:

int[] array = new int[10];

// using indices: read and write
for (int i = 0; i < array.length; i++) {
    array[i] = i;
}

// extended for: read only
for (int e : array) {
    System.out.println(e);
}

Vale a pena notar aqui que não há uma maneira direta de usar um Iterator em um Array, mas através da biblioteca Arrays ele pode ser facilmente convertido em uma lista para obter um objeto Iterable.

Para matrizes em caixa, use Arrays.asList:

Integer[] boxed = {1, 2, 3};
Iterable<Integer> boxedIt = Arrays.asList(boxed); // list-backed iterable
Iterator<Integer> fromBoxed1 = boxedIt.iterator();

Para arrays primitivos (usando java 8) use streams (especificamente neste exemplo - Arrays.stream -> IntStream):

int[] primitives = {1, 2, 3};
IntStream primitiveStream = Arrays.stream(primitives); // list-backed iterable
PrimitiveIterator.OfInt fromPrimitive1 = primitiveStream.iterator();

Se você não pode usar streams (sem java 8), pode optar por usar a biblioteca guava do google:

Iterable<Integer> fromPrimitive2 = Ints.asList(primitives);

Em matrizes bidimensionais ou mais, ambas as técnicas podem ser usadas de maneira um pouco mais complexa.

Exemplo:

int[][] array = new int[10][10];

for (int indexOuter = 0; indexOuter < array.length; indexOuter++) {
    for (int indexInner = 0; indexInner < array[indexOuter].length; indexInner++) {
        array[indexOuter][indexInner] = indexOuter + indexInner;
    }
}

for (int[] numbers : array) {
    for (int value : numbers) {
        System.out.println(value);
    }
}

É impossível definir um Array para qualquer valor não uniforme sem usar um loop baseado em índice.

Claro que você também pode usar loops while ou do-while ao iterar usando índices.

Uma observação de cautela: ao usar índices de matriz, certifique-se de que o índice esteja entre 0 e array.length - 1 (ambos inclusive). Não faça suposições codificadas sobre o comprimento do array, caso contrário você pode quebrar seu código se o tamanho do array mudar, mas seus valores codificados não.

Exemplo:

int[] numbers = {1, 2, 3, 4};

    for (int i = 0; i < numbers.length; i++) {
        numbers[i] += 1; //or this: numbers[i] = numbers[i] + 1; or numbers[i]++;      
    }
 
    // DON'T DO THIS :
    for (int i = 0; i < 4; i++) {
        numbers[i] += 1;
    }
}

public void fillArrayWithDoubleIndex(int[] array) {

        array[i] = i * 2;

    // DON'T DO THIS :
    int doubleLength = array.length * 2;

Usando Arrays temporários para reduzir a repetição de código

Iterar sobre uma matriz temporária em vez de repetir o código pode tornar seu código mais limpo. Ele pode ser usado onde a mesma operação é executada em várias variáveis.

// we want to print out all of these
String name = "Margaret";
int eyeCount = 16;
double height = 50.2;
int legs = 9;
int arms = 5;


// copy-paste approach:
System.out.println(name);
System.out.println(eyeCount);
System.out.println(height);
System.out.println(legs);
System.out.println(arms);


// temporary array approach:
for(Object attribute : new Object[]{name, eyeCount, height, legs, arms})
    System.out.println(attribute);

// using only numbers
for(double number : new double[]{eyeCount, legs, arms, height})
    System.out.println(Math.sqrt(number));

Tenha em mente que este código não deve ser usado em seções críticas de desempenho, pois uma matriz é criada toda vez que o loop é inserido e as variáveis ​​primitivas serão copiadas para a matriz e, portanto, não podem ser modificadas.

Arrays para uma String #

Desde o Java 1.5 você pode obter uma representação String do conteúdo do array especificado sem iterar sobre cada elemento. Basta usar Arrays.toString(Object[]) ou Arrays.deepToString(Object[]) para arrays multidimensionais:

int[] arr = {1, 2, 3, 4, 5};
System.out.println(Arrays.toString(arr));      // [1, 2, 3, 4, 5]

int[][] arr = {
    {1, 2, 3},
    {4, 5, 6},
    {7, 8, 9}
};
System.out.println(Arrays.deepToString(arr));  // [[1, 2, 3], [4, 5, 6], [7, 8, 9]]

O método Arrays.toString() usa o método Object.toString() para produzir valores String de cada item no array, além do array de tipo primitivo, pode ser usado para todos os tipos de arrays. Por exemplo:

public class Cat { /* implicitly extends Object */
    @Override
    public String toString() {
      return "CAT!";
    }
}

Cat[] arr = { new Cat(), new Cat() };
System.out.println(Arrays.toString(arr));        // [CAT!, CAT!]

Se nenhum toString() substituído existir para a classe, então o toString() herdado de Object será usado. Normalmente, a saída não é muito útil, por exemplo:

public class Dog {
    /* implicitly extends Object */
}

Dog[] arr = { new Dog() };
System.out.println(Arrays.toString(arr));        // [Dog@17ed40e0]

Arrays são objetos que fornecem espaço para armazenar até seu tamanho de elementos do tipo especificado. O tamanho de um array não pode ser modificado após a criação do array.

int[] arr1 = new int[0];

O campo length em um array armazena o tamanho de um array. É um campo final e não pode ser modificado.

        if (arrElt != null) {
            nonEmptyElementsCount++;
        }
    }

                            + "and it contains "+nonEmptyElementsCount+" non-empty values");
}

and it contains 7 non-empty values

Remove um elemento de um array #

Java não fornece um método direto em java.util.Arrays para remover um elemento de um array. Para realizá-lo, você pode copiar o array original para um novo sem o elemento a ser removido ou converter seu array para outra estrutura permitindo a remoção.

Usando ArrayList #

Você pode converter o array em um java.util.List, remover o elemento e converter a lista de volta em um array da seguinte forma:

String[] array = new String[]{"foo", "bar", "baz"};

List<String> list = new ArrayList<>(Arrays.asList(array));
list.remove("foo");

// Creates a new array with the same size as the list and copies the list
// elements to it.
array = list.toArray(new String[list.size()]);

System.out.println(Arrays.toString(array)); //[bar, baz]

Usando System.arraycopy #

System.arraycopy() pode ser usado para fazer uma cópia do array original e remover o elemento desejado. Abaixo um exemplo:

int[] array = new int[] { 1, 2, 3, 4 }; // Original array.
int[] result = new int[array.length - 1]; // Array which will contain the result.
int index = 1; // Remove the value "2".

// Copy the elements at the left of the index.
System.arraycopy(array, 0, result, 0, index);
// Copy the elements at the right of the index.
System.arraycopy(array, index + 1, result, index, array.length - index - 1);

System.out.println(Arrays.toString(result)); //[1, 3, 4]

Usando Apache Commons Lang #

Para remover facilmente um elemento, você pode usar a biblioteca Apache Commons Lang e especialmente o método estático removeElement() da classe ArrayUtils. Abaixo um exemplo:

int[] array = new int[]{1,2,3,4};
array = ArrayUtils.removeElement(array, 2); //remove first occurrence of 2
System.out.println(Arrays.toString(array)); //[1, 3, 4]

Como você altera o tamanho de um array? #

A resposta simples é que você não pode fazer isso. Uma vez que uma matriz foi criada, seu tamanho não pode ser alterado. Em vez disso, um array só pode ser “redimensionado” criando um novo array com o tamanho apropriado e copiando os elementos do array existente para o novo.

String[] listOfCities = new String[3];   // array created with size 3.
listOfCities[0] = "New York";
listOfCities[1] = "London";
listOfCities[2] = "Berlin";

Suponha (por exemplo) que um novo elemento precise ser adicionado ao array listOfCities definido como acima. Para fazer isso, você precisará:

  1. crie um novo array com tamanho 4,
  2. copie os 3 elementos existentes da matriz antiga para a nova matriz nos deslocamentos 0, 1 e 2 e
  3. adicione o novo elemento ao novo array no deslocamento 3.

Existem várias maneiras de fazer o acima. Antes do Java 6, a maneira mais concisa era:

String[] newArray = new String[listOfCities.length + 1];
System.arraycopy(listOfCities, 0, newArray, 0, listOfCities.length);
newArray[listOfCities.length] = "Sydney";

A partir do Java 6, os métodos Arrays.copyOf e Arrays.copyOfRange podem fazer isso de forma mais simples:

String[] newArray = Arrays.copyOf(listOfCities, listOfCities.length + 1);
newArray[listOfCities.length] = "Sydney";

Para outras maneiras de copiar uma matriz, consulte o exemplo a seguir. Tenha em mente que você precisa de uma cópia de matriz com um comprimento diferente do original ao redimensionar.

Uma alternativa melhor para o redimensionamento de array #

Existem duas grandes desvantagens em redimensionar uma matriz conforme descrito acima:

  • É ineficiente. Tornar um array maior (ou menor) envolve copiar muitos ou todos os elementos de array existentes e alocar um novo objeto de array. Quanto maior o array, mais caro fica.
  • Você precisa ser capaz de atualizar quaisquer variáveis ​​"ao vivo" que contenham referências ao array antigo.

Uma alternativa é criar a matriz com um tamanho grande o suficiente para começar. Isso só é viável se você puder determinar esse tamanho com precisão antes de alocar a matriz. Se você não puder fazer isso, o problema de redimensionar a matriz surge novamente.

A outra alternativa é usar uma classe de estrutura de dados fornecida pela biblioteca de classes Java SE ou uma biblioteca de terceiros. Por exemplo, a estrutura de “coleções” do Java SE fornece várias implementações das APIs List, Set e Map com diferentes propriedades de tempo de execução. A classe ArrayList está mais próxima das características de desempenho de um array simples (por exemplo, pesquisa O(N), O(1) get e set, inserção e exclusão aleatória O(N)) enquanto fornece redimensionamento mais eficiente sem o problema de atualização de referência.

(A eficiência de redimensionamento de ArrayList vem de sua estratégia de dobrar o tamanho do array de apoio em cada redimensionamento. Para um caso de uso típico, isso significa que você só redimensiona ocasionalmente. Quando você amortiza durante o tempo de vida da lista, o o custo de redimensionamento por inserção é O(1). Pode ser possível usar a mesma estratégia ao redimensionar uma matriz simples.)

  int index = -1;

Convertendo arrays entre primitivos e tipos em caixa #

Às vezes, a conversão de tipos primitivos para tipos em caixa é necessária.

int[] primitiveArray = {1, 2, 3, 4};

int[] primitiveArray = {1, 2, 3, 4};

Integer[] boxedArray = {1, 2, 3, 4};

Comparando arrays para igualdade #

Os tipos de matriz herdam suas implementações [equals() (e hashCode()) de java.lang.Object](https://www.wikiod.com/pt/java/metodos-de-classe-de-objeto-e-construtor#método equals() -method#t=201607191720385793182), então equals() só retornará true ao comparar exatamente com o mesmo objeto de matriz. Para comparar matrizes para igualdade com base em seus valores, use [java.util.Arrays.equals](https://docs.oracle.com/javase/8/docs/api/java/util/Arrays.html#equals -java.lang.Object:A-java.lang.Object:A-), que é sobrecarregado para todos os tipos de array.

int[] a = new int[]{1, 2, 3};
int[] b = new int[]{1, 2, 3};
System.out.println(a.equals(b)); //prints "false" because a and b refer to different objects
System.out.println(Arrays.equals(a, b)); //prints "true" because the elements of a and b have the same values

Quando o tipo de elemento é um tipo de referência, Arrays.equals() chama equals() nos elementos do array para determinar a igualdade. Em particular, se o tipo de elemento for um tipo de matriz, a comparação de identidade será usada. Para comparar matrizes multidimensionais para igualdade, use [Arrays.deepEquals()](https://docs.oracle.com/javase/8/docs/api/java/util/Arrays.html#deepEquals-java.lang. Object:A-java.lang.Object:A-) como abaixo:

int a[] = { 1, 2, 3 };
int b[] = { 1, 2, 3 };

Object[] aObject = { a }; // aObject contains one element
Object[] bObject = { b }; // bObject contains one element

System.out.println(Arrays.equals(aObject, bObject)); // false
System.out.println(Arrays.deepEquals(aObject, bObject));// true

Como conjuntos e mapas usam equals() e hashCode(), arrays geralmente não são úteis como elementos de conjunto ou chaves de mapa. Envolva-os em uma classe auxiliar que implemente equals() e hashCode() em termos dos elementos do array, ou converta-os em instâncias List e armazene as listas.

Copiando matrizes #

Java fornece várias maneiras de copiar um array.

para loop #

int[] a = { 4, 1, 3, 2 };
int[] b = new int[a.length]; 
for (int i = 0; i < a.length; i++) {
    b[i] = a[i];
}

Observe que usar esta opção com uma matriz de objetos em vez de uma matriz primitiva preencherá a cópia com referência ao conteúdo original em vez de uma cópia dele.

Objeto.clone() #

Como arrays são Objects em Java, você pode usar Object.clone().

int[] a = { 4, 1, 3, 2 };
int[] b = a.clone(); // [4, 1, 3, 2]

Observe que o método Object.clone para um array executa uma cópia superficial, ou seja, ele retorna uma referência a um novo array que faz referência aos mesmos elementos do array de origem.

Arrays.copyOf() #

java.util.Arrays fornece uma maneira fácil de realizar a cópia de um array para outro. Aqui está o uso básico:

int[] a = {4, 1, 3, 2};
int[] b = Arrays.copyOf(a, a.length); // [4, 1, 3, 2]

Observe que Arrays.copyOf também fornece uma sobrecarga que permite alterar o tipo do array:

Double[] doubles = { 1.0, 2.0, 3.0 };
Number[] numbers = Arrays.copyOf(doubles, doubles.length, Number[].class);

System.arraycopy() #

public static void arraycopy(Object src, int srcPos, Object dest, int destPos, int length) Copies an array from the specified source array, começando na posição especificada, até a posição especificada do matriz de destino.

Abaixo um exemplo de uso

int[] a = { 4, 1, 3, 2 };
int[] b = new int[a.length];
System.arraycopy(a, 0, b, 0, a.length); // [4, 1, 3, 2]

Arrays.copyOfRange() #

Usado principalmente para copiar uma parte de um Array, você também pode usá-lo para copiar todo o array para outro conforme abaixo:

int[] a = { 4, 1, 3, 2 };
int[] b = Arrays.copyOfRange(a, 0, a.length); // [4, 1, 3, 2]

Matrizes de fundição #

Arrays são objetos, mas seu tipo é definido pelo tipo dos objetos contidos. Portanto, não se pode simplesmente converter A[] para T[], mas cada membro A do específico A[] deve ser convertido para um objeto T. Exemplo genérico:

public static <T, A> T[] castArray(T[] target, A[] array) {
    for (int i = 0; i < array.length; i++) {
        target[i] = (T) array[i];
    }
    return target;
}

Assim, dado um array A[]:

T[] target = new T[array.Length];
target = castArray(target, array);

Java SE fornece o método Arrays.copyOf(original, newLength, newType) para esta finalidade:

Double[] doubles = { 1.0, 2.0, 3.0 };
Number[] numbers = Arrays.copyOf(doubles, doubles.length, Number[].class);
