Convertendo String para outros tipos de dados. #

Você pode converter uma string numeric em vários tipos numéricos Java da seguinte forma:

String para int:

String number = "12";
int num = Integer.parseInt(number);

String para flutuar:

String number = "12.0";
float num = Float.parseFloat(number);

String para dobrar:

String double = "1.47";
double num = Double.parseDouble(double);

String para booleano:

String falseString = "False";
boolean falseBool = Boolean.parseBoolean(falseString);   // falseBool = false 
    
String trueString = "True";
boolean trueBool = Boolean.parseBoolean(trueString);     // trueBool = true

String para longo:

String number = "47";
long num = Long.parseLong(number);

String para BigInteger:

String bigNumber = "21";
BigInteger reallyBig = new BigInteger(bigNumber);

String para BigDecimal:

String bigFraction = "17.21455";
BigDecimal reallyBig = new BigDecimal(bigFraction);

Exceções de conversão:

Todas as conversões numéricas acima lançarão um (desmarcado) NumberFormatException se você tentar analisar uma string que não seja um número formatado adequadamente ou esteja fora do intervalo para o tipo de destino. O tópico Exceções discute como lidar com essas exceções.

Se você quiser testar se pode analisar uma string, você pode implementar um método tryParse... como este:

boolean tryParseInt (String value) {  
    try {  
        String somechar = Integer.parseInt(value);
        return true;  
     } catch (NumberFormatException e) { 
        return false;  
     }  
}

No entanto, chamar este método tryParse... imediatamente antes da análise é (sem dúvida) uma prática ruim. Seria melhor apenas chamar o método parse... e lidar com a exceção.

Conversão de/para bytes #

Para codificar uma string em uma matriz de bytes, você pode simplesmente usar o método String#getBytes(), com um dos conjuntos de caracteres padrão disponíveis em qualquer tempo de execução Java:

byte[] bytes = "test".getBytes(StandardCharsets.UTF_8);

e para decodificar:

String testString = new String(bytes, StandardCharsets.UTF_8);

você pode simplificar ainda mais a chamada usando uma importação estática:

import static java.nio.charset.StandardCharsets.UTF_8;
...
byte[] bytes = "test".getBytes(UTF_8);

Para conjuntos de caracteres menos comuns, você pode indicar o conjunto de caracteres com uma string:

byte[] bytes = "test".getBytes("UTF-8");

e o inverso:

String testString = new String (bytes, "UTF-8");

no entanto, isso significa que você tem que lidar com o UnsupportedCharsetException marcado.

A chamada a seguir usará o conjunto de caracteres padrão. O conjunto de caracteres padrão é específico da plataforma e geralmente difere entre as plataformas Windows, Mac e Linux.

byte[] bytes = "test".getBytes();

e o inverso:

String testString = new String(bytes);

Observe que caracteres e bytes inválidos podem ser substituídos ou ignorados por esses métodos. Para mais controle - por exemplo para validação de entrada - você é encorajado a usar as classes CharsetEncoder e CharsetDecoder.

Codificação / Decodificação Base64 #

Ocasionalmente, você encontrará a necessidade de codificar dados binários como uma string codificada em base64.

Para isso podemos usar a classe DatatypeConverter do pacote javax.xml.bind:

import javax.xml.bind.DatatypeConverter;
import java.util.Arrays;

// arbitrary binary data specified as a byte array
byte[] binaryData = "some arbitrary data".getBytes("UTF-8");

// convert the binary data to the base64-encoded string
String encodedData = DatatypeConverter.printBase64Binary(binaryData);
// encodedData is now "c29tZSBhcmJpdHJhcnkgZGF0YQ=="

// convert the base64-encoded string back to a byte array
byte[] decodedData = DatatypeConverter.parseBase64Binary(encodedData);

// assert that the original data and the decoded data are equal
assert Arrays.equals(binaryData, decodedData);

Apache commons-codec

Alternativamente, podemos usar Base64 de Apache commons-codec.

import org.apache.commons.codec.binary.Base64;

// your blob of binary as a byte array
byte[] blob = "someBinaryData".getBytes();

// use the Base64 class to encode
String binaryAsAString = Base64.encodeBase64String(blob);

// use the Base64 class to decode
byte[] blob2 = Base64.decodeBase64(binaryAsAString);

// assert that the two blobs are equal
System.out.println("Equal : " + Boolean.toString(Arrays.equals(blob, blob2)));

Se você inspecionar este programa enquanto estiver em execução, verá que someBinaryData codifica para c29tZUJpbmFyeURhdGE=, um objeto UTF-8 String muito gerenciável.

Detalhes para o mesmo podem ser encontrados em Base64

// encode with padding
String encoded = Base64.getEncoder().encodeToString(someByteArray);

// encode without padding
String encoded = Base64.getEncoder().withoutPadding().encodeToString(someByteArray);

// decode a String
byte [] barr = Base64.getDecoder().decode(encoded);

Convertendo outros tipos de dados para String #

  • Você pode obter o valor de outros tipos de dados primitivos como uma String usando um dos métodos valueOf da classe String.

    For example:

    int i = 42;
String string = String.valueOf(i);
//string now equals "42”.

    This method is also overloaded for other datatypes, such as float, double, boolean, and even Object.

  • Você também pode obter qualquer outro objeto (qualquer instância de qualquer classe) como um String chamando .toString nele. Para que isso dê uma saída útil, a classe deve substituir toString(). A maioria das classes de biblioteca Java padrão, como Date e outras.

    For example:

    Foo foo = new Foo(); //Any class.
String stringifiedFoo = foo.toString().

    Here stringifiedFoo contains a representation of foo as a String.

Você também pode converter qualquer tipo de número para String com notação curta como abaixo.

int i = 10;
String str = i + "";

Ou apenas maneira simples é

String str = 10 + "";

Obtendo uma String de um InputStream #

Um String pode ser lido de um InputStream usando o construtor de array de bytes.

import java.io.*;

public String readString(InputStream input) throws IOException {
    byte[] bytes = new byte[50]; // supply the length of the string in bytes here
    input.read(bytes);
    return new String(bytes);
}

Isso usa o conjunto de caracteres padrão do sistema, embora um conjunto de caracteres alternativo possa ser especificado:

return new String(bytes, Charset.forName("UTF-8"));

Analisando Strings para um Valor Numérico #

String para um tipo numérico primitivo ou um tipo de wrapper numérico:

Cada classe de wrapper numérica fornece um método parseXxx que converte uma String para o tipo primitivo correspondente. O código a seguir converte um String em um int usando o método Integer.parseInt:

String string = "59";
int primitive = Integer.parseInteger(string);

Para converter em uma String para uma instância de uma classe wrapper numérica, você pode usar uma sobrecarga do método valueOf das classes wrapper:

String string = "59";
Integer wrapper = Integer.valueOf(string);

ou confie no boxe automático (Java 5 e posterior):

String string = "59";
Integer wrapper = Integer.parseInteger(string);  // 'int' result is autoboxed

O padrão acima funciona para byte, short, int, long, float e double e as classes wrapper correspondentes (Byte, Short, Integer, Long, Float e Double).

String para inteiro usando base:

String integerAsString = "0101"; // binary representation
int parseInt = Integer.parseInt(integerAsString,2);
Integer valueOfInteger = Integer.valueOf(integerAsString,2);
System.out.println(valueOfInteger); // prints 5 
System.out.println(parseInt); // prints 5

Exceções

A exceção desmarcada NumberFormatException será lançada se um método numérico valueOf(String) ou parseXxx(...) for chamado para uma string que não seja uma representação numérica aceitável ou que represente um valor que está fora do alcance.
