Strings (java.lang.String) são pedaços de texto armazenados em seu programa. Strings não são um tipo de dados primitivo em Java, no entanto, são muito comuns em programas Java.

Em Java, Strings são imutáveis, o que significa que não podem ser alteradas. (Clique aqui para uma explicação mais completa da imutabilidade.)

Como as strings Java são imutáveis, todos os métodos que manipulam uma String retornarão um novo objeto String. Eles não alteram a String original. Isso inclui substring e métodos de substituição que os programadores C e C++ esperariam para alterar o objeto String de destino.

Use um StringBuilder em vez de String se desejar concatenar mais de dois objetos String cujos valores não podem ser determinados em tempo de compilação. Esta técnica é mais eficiente do que criar novos objetos String e concatená-los porque o StringBuilder é mutável.

StringBuffer também pode ser usado para concatenar objetos String. No entanto, essa classe tem menos desempenho porque foi projetada para ser thread-safe e adquire um mutex antes de cada operação. Como você quase nunca precisa de segurança de thread ao concatenar strings, é melhor usar StringBuilder.

Se você pode expressar uma concatenação de strings como uma única expressão, então é melhor usar o operador +. O compilador Java converterá uma expressão contendo concatenações + em uma seqüência eficiente de operações usando String.concat(...) ou StringBuilder. O conselho para usar StringBuilder explicitamente só se aplica quando a concatenação envolve várias expressões.

Não armazene informações confidenciais em strings. Se alguém conseguir obter um despejo de memória do seu aplicativo em execução, ele poderá encontrar todos os objetos String existentes e ler seu conteúdo. Isso inclui objetos String que estão inacessíveis e estão aguardando coleta de lixo. Se isso for uma preocupação, você precisará limpar os dados de string confidenciais assim que terminar. Você não pode fazer isso com objetos String porque eles são imutáveis. Portanto, é aconselhável usar objetos char[] para armazenar dados de caracteres confidenciais e limpá-los (por exemplo, sobrescrevê-los com caracteres '\000') quando terminar.

Todas as instâncias String são criadas no heap, mesmo as instâncias que correspondem a literais de string. A coisa especial sobre literais de string é que a JVM garante que todos os literais que são iguais (ou seja, que consistem nos mesmos caracteres) sejam representados por um único objeto String (esse comportamento é especificado em JLS). Isso é implementado por carregadores de classe JVM. Quando um carregador de classes carrega uma classe, ele procura por literais de string que são usados ​​na definição de classe, cada vez que vê um, ele verifica se já existe um registro no pool de strings para este literal (usando o literal como chave) . Se já houver uma entrada para o literal, é usada a referência a uma instância String armazenada como o par desse literal. Caso contrário, uma nova instância String é criada e uma referência à instância é armazenada para o literal (usado como chave) no conjunto de strings. (Veja também string de internamento).

O conjunto de cadeias é mantido no heap Java e está sujeito à coleta de lixo normal.

Nas versões do Java anteriores ao Java 7, o pool de strings era mantido em uma parte especial do heap conhecido como “PermGen”. Esta parte foi coletada apenas ocasionalmente.

No Java 7, o pool de strings foi removido de “PermGen”.

Observe que os literais de string são implicitamente alcançáveis ​​de qualquer método que os use. Isso significa que os objetos String correspondentes só podem ser coletados como lixo se o próprio código for coletado como lixo.

Até o Java 8, objetos String são implementados como um array de caracteres UTF-16 (2 bytes por caractere). Existe uma proposta no Java 9 para implementar String como um array de bytes com um campo de flag de codificação para observar se a string está codificada como bytes (LATIN-1) ou chars (UTF-16).

Não use o operador == para comparar Strings #

A menos que você possa garantir que todas as strings foram internadas (veja abaixo), você não deve usar os operadores == ou != para comparar Strings. Esses operadores realmente testam referências, e como vários objetos String podem representar a mesma String, isso pode dar a resposta errada.

Em vez disso, use o método String.equals(Object), que irá comparar os objetos String com base em seus valores. Para obter uma explicação detalhada, consulte https://www.wikiod.com/pt/java/armadilhas-comuns-do-java#Armadilha: usando == para comparar strings

Comparando Strings com valores constantes #

Ao comparar uma String com um valor constante, você pode colocar o valor constante no lado esquerdo de equals para garantir que você não obterá uma NullPointerException se a outra String for null.

Comparando com Strings internadas #

A especificação de linguagem Java (JLS 3.10.6) afirma o seguinte:

“Além disso, um literal de string sempre se refere à mesma instância da classe String. Isso ocorre porque literais de string - ou, mais geralmente, strings que são os valores de expressões constantes - são internados para compartilhar instâncias únicas, usando o método String.intern.”

Isso significa que é seguro comparar referências a duas strings literais usando ==. Além disso, o mesmo é verdadeiro para referências a objetos String que foram produzidos usando o método String.intern().

Por exemplo:

String strObj = new String("Hello!");
String str = "Hello!";

// The two string references point two strings that are equal
if (strObj.equals(str)) {
    System.out.println("The strings are equal");
}

// The two string references do not point to the same object
if (strObj != str) {
    System.out.println("The strings are not the same object");
}

// If we intern a string that is equal to a given literal, the result is
// a string that has the same reference as the literal.
String internedStr = strObj.intern();

if (internedStr == str) {
    System.out.println("The interned string and the literal are the same object");
}

Nos bastidores, o mecanismo interno mantém uma tabela de hash que contém todas as strings internas que ainda são alcançáveis. Quando você chama intern() em uma String, o método procura o objeto na tabela de hash:

  • Se a string for encontrada, esse valor será retornado como a string interna.
  • Caso contrário, uma cópia da string é adicionada à tabela de hash e essa string é retornada como a string interna.

É possível usar o internamento para permitir que as strings sejam comparadas usando ==. No entanto, existem problemas significativos em fazer isso; consulte https://www.wikiod.com/pt/java/armadilhas-do-java—problemas-de-desempenho#Pitfall - Internar strings para que você possa usar == é uma má ideia =201610101338294726756 para detalhes. Não é recomendado na maioria dos casos.

Mudando o caso de caracteres dentro de uma String #

O tipo String fornece dois métodos para converter strings entre maiúsculas e minúsculas:

  • toUpperCase para converter todos os caracteres em maiúsculas
  • toLowerCase para converter todos os caracteres em minúsculas

Ambos os métodos retornam as strings convertidas como novas instâncias String: os objetos originais String não são modificados porque String é imutável em Java. Veja isto para saber mais sobre imutabilidade: Imutabilidade de Strings em Java

String string = "This is a Random String";
String upper = string.toUpperCase();
String lower = string.toLowerCase();

System.out.println(string);   // prints "This is a Random String"
System.out.println(lower);    // prints "this is a random string"
System.out.println(upper);    // prints "THIS IS A RANDOM STRING"

Caracteres não alfabéticos, como dígitos e sinais de pontuação, não são afetados por esses métodos. Observe que esses métodos também podem lidar incorretamente com determinados caracteres Unicode sob determinadas condições.

Observação: Esses métodos são sensíveis à localidade e podem produzir resultados inesperados se usados ​​em strings que devem ser interpretadas independentemente da localidade. Exemplos são identificadores de linguagem de programação, chaves de protocolo e tags HTML.

Por exemplo, "TITLE".toLowerCase() em uma localidade turca retorna “tıtle”, onde ı (\u0131) é o caractere LATIN SMALL LETTER DOTLESS I. Para obter resultados corretos para strings sem distinção de localidade, passe Locale.ROOT como um parâmetro para o método de conversão de maiúsculas correspondente (por exemplo, toLowerCase(Locale.ROOT) ou toUpperCase(Locale.ROOT)).

Embora o uso de Locale.ENGLISH também seja correto para a maioria dos casos, a forma invariante de idioma é Locale.ROOT.

Uma lista detalhada de caracteres Unicode que requerem maiúsculas e minúsculas pode ser encontrada no site do Unicode Consortium.

Alterar maiúsculas e minúsculas de um caractere específico em uma string ASCII:

Para alterar o caso de um caractere específico de uma string ASCII, o seguinte algoritmo pode ser usado:

Passos:

  1. Declare uma string.
  2. Insira a string.
  3. Converta a string em uma matriz de caracteres.
  4. Insira o caractere a ser pesquisado.
  5. Procure o caractere na matriz de caracteres.
  6. Se encontrado, verifique se o caractere é minúsculo ou maiúsculo.
    • If Uppercase, add 32 to the ASCII code of the character.
    • If Lowercase, subtract 32 from the ASCII code of the character.
  7. Altere o caractere original da matriz de caracteres.
  8. Converta a matriz de caracteres de volta na string.

Voila, o Caso do personagem é alterado.

Um exemplo de código para o algoritmo é:

Scanner scanner = new Scanner(System.in);
System.out.println("Enter the String");
String s = scanner.next();
char[] a = s.toCharArray();
System.out.println("Enter the character you are looking for");
System.out.println(s);
String c = scanner.next();
char d = c.charAt(0);

for (int i = 0; i <= s.length(); i++) {
    if (a[i] == d) {
        if (d >= 'a' && d <= 'z') {
            d -= 32;
        } else if (d >= 'A' && d <= 'Z') {
            d += 32;
        }
        a[i] = d;
        break;
    }
}
s = String.valueOf(a);
System.out.println(s);

Dividindo Strings #

Você pode dividir uma String em um caractere delimitador específico ou uma Expressão Regular, você pode usar o método String.split() que tem a seguinte assinatura:

public String[] split(String regex)

Observe que o caractere delimitador ou a expressão regular são removidos do String Array resultante.

Exemplo usando caractere delimitador:

String lineFromCsvFile = "Mickey;Bolton;12345;121216";
String[] dataCells = lineFromCsvFile.split(";");
// Result is dataCells = { "Mickey", "Bolton", "12345", "121216"};

Exemplo usando expressão regular:

String lineFromInput = "What    do you need    from me?";
String[] words = lineFromInput.split("\\s+"); // one or more space chars
// Result is words = {"What", "do", "you", "need", "from", "me?"};

Você pode até dividir diretamente um literal String:

String[] firstNames = "Mickey, Frank, Alicia, Tom".split(", ");
// Result is firstNames = {"Mickey", "Frank", "Alicia", "Tom"};

Aviso: Não esqueça que o parâmetro é sempre tratado como uma expressão regular.

"aaa.bbb".split("."); // This returns an empty array

No exemplo anterior, . é tratado como o curinga da expressão regular que corresponde a qualquer caractere e, como cada caractere é um delimitador, o resultado é um array vazio.

Divisão com base em um delimitador que é um metacaractere regex

Os seguintes caracteres são considerados especiais (também conhecidos como meta-caracteres) em regex

  < > - = ! ( ) [ ] { } \ ^ $ | ? * + .

Para dividir uma string com base em um dos delimitadores acima, você precisa escape usando \\ ou use Pattern.quote():

  • Usando Pattern.quote():

     String s = "a|b|c";
 String regex = Pattern.quote("|");
 String[] arr = s.split(regex);

  • Escapando os caracteres especiais:

     String s = "a|b|c";
 String[] arr = s.split("\\|");

Dividir remove valores vazios

split(delimiter) por padrão remove strings vazias à direita do array de resultados. Para desativar esse mecanismo, precisamos usar a versão sobrecarregada de split(delimiter, limit) com limite definido como valor negativo, como

String[] split = data.split("\\|", -1);

split(regex) retorna internamente o resultado de split(regex, 0).

O parâmetro limite controla o número de vezes que o padrão é aplicado e, portanto, afeta o comprimento da matriz resultante. Se o limite n for maior que zero, o padrão será aplicado no máximo n - 1 vezes, o comprimento da matriz não será maior que n e a última entrada da matriz conterá todas as entradas além da última correspondência delimitador. Se n for negativo, o padrão será aplicado tantas vezes quanto possível e o array pode ter qualquer comprimento. Se n for zero, o padrão será aplicado tantas vezes quanto possível, a matriz pode ter qualquer comprimento e as sequências vazias à direita serão descartadas.

Pool de strings e armazenamento de heap #

Como muitos objetos Java, todas instâncias String são criadas no heap, até mesmo literais. Quando a JVM encontra um literal String que não tem referência equivalente no heap, a JVM cria uma instância String correspondente no heap e também armazena uma referência para a instância String recém-criada no Piscina de cordas. Quaisquer outras referências ao mesmo literal String são substituídas pela instância String criada anteriormente no heap.

Vejamos o seguinte exemplo:

class Strings
{
    public static void main (String[] args)
    {
        String a = "alpha";
        String b = "alpha";
        String c = new String("alpha");

        //All three strings are equivalent
        System.out.println(a.equals(b) && b.equals(c));

        //Although only a and b reference the same heap object
        System.out.println(a == b);
        System.out.println(a != c);
        System.out.println(b != c);
    }
}

A saída acima é:

true
true
true
true

Quando usamos aspas duplas para criar uma String, ele primeiro procura por String com o mesmo valor no pool de Strings, se encontrado apenas retorna a referência, senão cria uma nova String no pool e depois retorna a referência.

No entanto, usando o operador new, forçamos a classe String a criar um novo objeto String no espaço de heap. Podemos usar o método intern() para colocá-lo no pool ou fazer referência a outro objeto String do pool de strings com o mesmo valor.

O próprio pool de Strings também é criado no heap.

Antes do Java 7, String literais eram armazenados no pool de constantes de tempo de execução na área de métodos de PermGen, que tinha um tamanho fixo.

O pool de Strings também residia em PermGen.

RFC: 6962931

No JDK 7, as strings internas não são mais alocadas na geração permanente do heap Java, mas sim alocadas na parte principal do heap Java (conhecida como geração jovem e antiga), juntamente com os outros objetos criados pelo inscrição. Essa mudança resultará em mais dados residindo no heap Java principal e menos dados na geração permanente e, portanto, pode exigir que os tamanhos de heap sejam ajustados. A maioria dos aplicativos verá apenas diferenças relativamente pequenas no uso de heap devido a essa mudança, mas aplicativos maiores que carregam muitas classes ou fazem uso pesado do método String.intern() verão diferenças mais significativas.

Unindo Strings com um delimitador #

Uma matriz de strings pode ser unida usando o método estático [String.join()](https://docs.oracle.com/javase/8/docs/api/java/lang/String.html#join-java .lang.CharSequence-java.lang.CharSequence…-):

String[] elements = { "foo", "bar", "foobar" };
String singleString = String.join(" + ", elements);

System.out.println(singleString); // Prints "foo + bar + foobar"

Da mesma forma, há uma sobrecarga [String.join()](https://docs.oracle.com/javase/8/docs/api/java/lang/String.html#join-java.lang.CharSequence-java .lang.Iterable-) para Iterables.

Para ter um controle refinado sobre a adesão, você pode usar a classe StringJoiner:

StringJoiner sj = new StringJoiner(", ", "[", "]"); 
    // The last two arguments are optional,

sj.add("foo");
sj.add("bar");
sj.add("foobar");

System.out.println(sj); // Prints "[foo, bar, foobar]"

Para participar de um fluxo de strings, você pode usar o [joining collector](https://docs.oracle.com/javase/8/docs/api/java/util/stream/Collectors.html#joining-java.lang. CharSequence-):

Stream<String> stringStream = Stream.of("foo", "bar", "foobar");

Stream<String> stringStream = Stream.of("foo", "bar", "foobar");
String joined = stringStream.collect(Collectors.joining(", ", "{", "}"));

Concatenação de strings e StringBuilders #

A concatenação de strings pode ser realizada usando o operador +. Por exemplo:

String s1 = "a";
String s2 = "b";
String s3 = "c";
String s = s1 + s2 + s3; // abc

Normalmente, uma implementação de compilador executará a concatenação acima usando métodos envolvendo um StringBuilder sob o capô. Quando compilado, o código ficaria parecido com o abaixo:

StringBuilder sb = new StringBuilder("a");
String s = sb.append("b").append("c").toString();

O StringBuilder tem vários métodos sobrecarregados para anexar diferentes tipos, por exemplo, para anexar um int em vez de um String. Por exemplo, uma implementação pode converter:

String s1 = "a";
String s2 = "b";    
String s = s1 + s2 + 2; // ab2

ao seguinte:

StringBuilder sb = new StringBuilder("a");
String s = sb.append("b").append(2).toString();

Os exemplos acima ilustram uma operação de concatenação simples que é efetivamente feita em um único local no código. A concatenação envolve uma única instância do StringBuilder. Em alguns casos, uma concatenação é realizada de forma cumulativa, como em um loop:

String result = "";
for(int i = 0; i < array.length; i++) {
    result += extractElement(array[i]);
}
return result;

Nesses casos, a otimização do compilador geralmente não é aplicada, e cada iteração criará um novo objeto StringBuilder. Isso pode ser otimizado transformando explicitamente o código para usar um único StringBuilder:

StringBuilder result = new StringBuilder();
for(int i = 0; i < array.length; i++) {
    result.append(extractElement(array[i]));
}
return result.toString();

Um StringBuilder será inicializado com um espaço vazio de apenas 16 caracteres. Se você souber com antecedência que construirá strings maiores, pode ser benéfico inicializá-lo com tamanho suficiente com antecedência, para que o buffer interno não precise ser redimensionado:

StringBuilder buf = new StringBuilder(30); // Default is 16 characters
buf.append("0123456789");
buf.append("0123456789"); // Would cause a reallocation of the internal buffer otherwise
String result = buf.toString(); // Produces a 20-chars copy of the string

Se você estiver produzindo muitas strings, é aconselhável reutilizar StringBuilders:

StringBuilder buf = new StringBuilder(100);
for (int i = 0; i < 100; i++) {
    buf.setLength(0); // Empty buffer
    buf.append("This is line ").append(i).append('\n');
    outputfile.write(buf.toString());
}

Se (e somente se) vários threads estiverem gravando no mesmo buffer, use StringBuffer, que é uma versão sincronizada do StringBuilder. Mas como geralmente apenas um único thread grava em um buffer, geralmente é mais rápido usar o StringBuilder sem sincronização.

Usando o método concat():

String string1 = "Hello ";
String string2 = "world";
String string3 = string1.concat(string2);  // "Hello world"

Isso retorna uma nova string que é string1 com string2 adicionada a ela no final. Você também pode usar o método concat() com literais de string, como em:

"My name is ".concat("Buyya");

Substrings #

String s = "this is an example";
String a = s.substring(11); // a will hold the string starting at character 11 until the end ("example")
String b = s.substring(5, 10); // b will hold the string starting at character 5 and ending right before character 10 ("is an")
String b = s.substring(5, b.length()-3); // b will hold the string starting at character 5 ending right before b' s lenght is out of 3  ("is an exam")

Substrings também podem ser aplicadas para fatiar e adicionar/substituir caracteres em sua String original. Por exemplo, você enfrentou uma data chinesa contendo caracteres chineses, mas deseja armazená-la como uma string de data bem formatada.

String datestring = "2015年11月17日"
datestring = datestring.substring(0, 4) + "-" + datestring.substring(5,7) + "-" + datestring.substring(8,10);
//Result will be 2015-11-17

O método substring extrai uma parte de uma String. Quando fornecido um parâmetro, o parâmetro é o início e a peça se estende até o final da String. Quando dados dois parâmetros, o primeiro parâmetro é o caractere inicial e o segundo parâmetro é o índice do caractere logo após o final (o caractere no índice não é incluído). Uma maneira fácil de verificar é que a subtração do primeiro parâmetro do segundo deve resultar no comprimento esperado da string.

Nas versões JDK <7u6, o método substring instancia uma String que compartilha o mesmo char[] de apoio que a String original e tem os campos internos offset e count definidos para o início e comprimento do resultado . Tal compartilhamento pode causar vazamentos de memória, que podem ser evitados chamando new String(s.substring(...)) para forçar a criação de uma cópia, após a qual o char[] pode ser coletado como lixo.

A partir do JDK 7u6, o método substring sempre copia todo o array char[] subjacente, tornando a complexidade linear em comparação com a constante anterior, mas garantindo a ausência de vazamentos de memória ao mesmo tempo.

Novo separador de linha independente da plataforma #

Como o novo separador de linha varia de plataforma para plataforma (por exemplo, \n em sistemas do tipo Unix ou \r\n no Windows), muitas vezes é necessário ter uma maneira independente de plataforma para acessá-lo. Em Java, ele pode ser recuperado de uma propriedade do sistema:

System.getProperty("line.separator")

Como o novo separador de linha é tão comumente necessário, a partir do Java 7 em um método de atalho que retorna exatamente o mesmo resultado que o código acima está disponível:

System.lineSeparator()

Observação: Como é muito improvável que o novo separador de linha mude durante a execução do programa, é uma boa ideia armazená-lo em uma variável final estática em vez de recuperá-lo da propriedade do sistema sempre que for necessário.

Ao usar String.format, use %n em vez de \n ou ‘\r\n’ para gerar um novo separador de linha independente da plataforma.

System.out.println(String.format('line 1: %s.%nline 2: %s%n', lines[0],lines[1]));

Adicionando o método toString() para objetos personalizados #

Suponha que você tenha definido a seguinte classe Person:

public class Person {

    String name;
    int age;
    
    public Person (int age, String name) {
        this.age = age;
        this.name = name;
    }
}

Se você instanciar um novo objeto Person:

Person person = new Person(25, "John");

e posteriormente em seu código você usa a seguinte instrução para imprimir o objeto:

System.out.println(person.toString());

Demonstração ao vivo no Ideone

você obterá uma saída semelhante à seguinte:

Person@7ab89d

Este é o resultado da implementação do método toString() definido na classe Object, uma superclasse de Person. A documentação de Object.toString() afirma:

O método toString para a classe Object retorna uma string que consiste no nome da classe da qual o objeto é uma instância, o caractere de arroba `@’ e a representação hexadecimal sem sinal do código hash do objeto. Em outras palavras, esse método retorna uma string igual ao valor de:

getClass().getName() + ‘@’ + Integer.toHexString(hashCode())

Então, para uma saída significativa, você terá que substituir o método toString():

@Override
public String toString() {
    return "My name is " + this.name + " and my age is " + this.age;
}

Agora a saída será:

My name is John and my age is 25

Você também pode escrever

System.out.println(person);

Demonstração ao vivo no Ideone

Na verdade, println() invoca implicitamente o método toString no objeto.

Invertendo Strings #

Existem algumas maneiras de reverter uma string para torná-la inversa.

  1. StringBuilder/StringBuffer:

     String code = "code";
 System.out.println(code);

 StringBuilder sb = new StringBuilder(code); 
 code = sb.reverse().toString();

 System.out.println(code);

  2. Matriz de caracteres:

    String code = "code";
System.out.println(code);

char[] array = code.toCharArray();
for (int index = 0, mirroredIndex = array.length - 1; index < mirroredIndex; index++, mirroredIndex--) {
    char temp = array[index];
    array[index] = array[mirroredIndex];
    array[mirroredIndex] = temp;
}

// print reversed
System.out.println(new String(array));

Remover espaços em branco do início e do fim de uma string #

O método trim() retorna uma nova String com os espaços em branco iniciais e finais removidos.

String s = new String("   Hello World!!  ");
String t = s.trim();  // t = "Hello World!!"

Se você trim uma String que não tem nenhum espaço em branco para remover, você receberá a mesma instância de String.

Observe que o método trim() tem sua própria noção de espaço em branco, que difere da noção usada pelo método Character. isWhitespace():

  • Todos os caracteres de controle ASCII com códigos U+0000 a U+0020 são considerados espaços em branco e são removidos por trim(). Isso inclui os caracteres U+0020 'SPACE', U+0009 'CHARACTER TABULATION', U+000A 'LINE FEED' e U+000D 'CARRIAGE RETURN', mas também os caracteres como `U+ 0007 ‘SINO’’.

  • Espaços em branco Unicode como U+00A0 'NO-BREAK SPACE' ou U+2003 'EM SPACE' não são reconhecidos por trim().

Substituindo partes de Strings #

Duas maneiras de substituir: por regex ou por correspondência exata.

Observação: o objeto String original não será alterado, o valor de retorno contém a String alterada.

Combinação exata #

Substitua um caractere único por outro caractere único: #

String replace(char oldChar, char newChar)

Retorna uma nova string resultante da substituição de todas as ocorrências de oldChar nesta string por newChar.

String s = "popcorn";
System.out.println(s.replace('p','W'));

Resultado:

WoWcorn

Substitua a sequência de caracteres por outra sequência de caracteres: #

String replace(CharSequence target, CharSequence replacement)

Substitui cada substring dessa string que corresponde à sequência de destino literal pela sequência de substituição literal especificada.

String s = "metal petal et al.";
System.out.println(s.replace("etal","etallica"));

Resultado:

metallica petallica et al.

Regex #

Observação: o agrupamento usa o caractere $ para referenciar os grupos, como $1.

Substitua todas as correspondências: #

String replaceAll(String regex, String replacement)

Substitui cada substring dessa string que corresponde à expressão regular fornecida pela substituição fornecida.

String s = "spiral metal petal et al.";
System.out.println(s.replaceAll("(\\w*etal)","$1lica"));

Resultado:

spiral metallica petallica et al.

Substitua apenas a primeira correspondência: #

String replaceFirst(String regex, String replacement)

Substitui a primeira substring desta string que corresponde à expressão regular fornecida pela substituição fornecida

String s = "spiral metal petal et al.";
System.out.println(s.replaceAll("(\\w*etal)","$1lica"));

Resultado:

spiral metallica petal et al.

Chave insensível a maiúsculas e minúsculas #

O switch em si não pode ser parametrizado para não diferenciar maiúsculas de minúsculas, mas se for absolutamente necessário, pode se comportar insensível à string de entrada usando toLowerCase() ou toUpperCase:

switch (myString.toLowerCase()) {
     case "case1" :
        ...            
     break;
     case "case2" :
        ...            
     break;
}

Cuidado

  • “Locale” pode afetar como [os casos de alteração acontecem](https://www.wikiod.com/pt/java/cordas#Alterando o caso de caracteres dentro de uma String)!
  • Deve-se tomar cuidado para não ter caracteres maiúsculos nos rótulos - eles nunca serão executados!

Obtendo o comprimento de uma String #

Para obter o comprimento de um objeto String, chame o método length() nele. O comprimento é igual ao número de unidades de código UTF-16 (caracteres) na string.

String str = "Hello, World!";
System.out.println(str.length()); // Prints out 13

Demonstração ao vivo no Ideone

Um char em uma String é um valor UTF-16. Os codepoints Unicode cujos valores são ≥ 0x1000 (por exemplo, a maioria dos emojis) usam duas posições de caractere. Para contar o número de codepoints Unicode em uma String, independentemente de cada codepoint caber em um valor UTF-16 char, você pode usar o método codePointCount:

int length = str.codePointCount(0, str.length());

Você também pode usar um Stream de codepoints, a partir do Java 8:

int length = str.codePoints().count();

Obtendo o enésimo caractere em uma String #

String str = "My String";

System.out.println(str.charAt(0)); // "M"
System.out.println(str.charAt(1)); // "y"
System.out.println(str.charAt(2)); // " "
System.out.println(str.charAt(str.length-1)); // Last character "g"

Para obter o enésimo caractere em uma string, basta chamar charAt(n) em uma String, onde n é o índice do caractere que você gostaria de recuperar

OBSERVAÇÃO: o índice n começa em 0, então o primeiro elemento está em n=0.

Contando ocorrências de uma substring ou caractere em uma string #

O método countMatches de org.apache.commons.lang3.StringUtils é normalmente usado para contar ocorrências de uma substring ou caractere em uma String:

import org.apache.commons.lang3.StringUtils;

String text = "One fish, two fish, red fish, blue fish";

// count occurrences of a substring
String stringTarget = "fish";
int stringOccurrences = StringUtils.countMatches(text, stringTarget); // 4

// count occurrences of a char
char charTarget = ',';
int charOccurrences = StringUtils.countMatches(text, charTarget); // 3

Caso contrário, para fazer o mesmo com APIs Java padrão, você pode usar Expressões Regulares:

import java.util.regex.Matcher;
import java.util.regex.Pattern;
 
String text = "One fish, two fish, red fish, blue fish";
System.out.println(countStringInString("fish", text)); // prints 4
System.out.println(countStringInString(",", text)); // prints 3
 

public static int countStringInString(String search, String text) {
    Pattern pattern = Pattern.compile(search);
    Matcher matcher = pattern.matcher(text);
    
    int stringOccurrences = 0;
    while (matcher.find()) {
      stringOccurrences++;
    }
    return stringOccurrences;
}
