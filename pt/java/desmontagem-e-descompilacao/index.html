Sintaxe #

  • javap [opções] <classes>

Parâmetros #

Nome Descrição
<classes> Lista de classes para desmontar. Pode estar no formato package1.package2.Classname, ou no formato package1/package2/Classname. Não inclua a extensão .class.
-help, --help, -? Imprima esta mensagem de uso
-versão Informações da versão
-v, -verbose Imprimir informações adicionais
-l Imprimir número de linha e tabelas de variáveis ​​locais
-público Mostrar apenas turmas e membros públicos
-protegido Mostrar classes e membros protegidos/públicos
-pacote Mostrar classes e membros do pacote/protegidos/públicos (padrão)
-p, -privado Mostrar todas as classes e membros
-c Desmonte o código
-s Imprimir assinaturas de tipo interno
-sysinfo Mostrar informações do sistema (caminho, tamanho, data, hash MD5) da classe que está sendo processada
-constantes Mostrar constantes finais
-caminho de classe <caminho> Especifique onde localizar arquivos de classe de usuário
-cp <caminho> Especifique onde localizar arquivos de classe de usuário
-bootclasspath <caminho> Substituir a localização dos arquivos de classe de bootstrap

Visualizando bytecode com javap #

Se você quiser ver o bytecode gerado para um programa Java, você pode usar o comando javap fornecido para visualizá-lo.

Supondo que temos o seguinte arquivo de origem Java:

package com.stackoverflow.documentation;

import org.springframework.stereotype.Service;

import java.io.IOException;
import java.io.InputStream;
import java.util.List;

@Service
public class HelloWorldService {

    public void sayHello() {
        System.out.println("Hello, World!");
    }

    private Object[] pvtMethod(List<String> strings) {
        return new Object[]{strings};
    }

    protected String tryCatchResources(String filename) throws IOException {
        try (InputStream inputStream = getClass().getResourceAsStream(filename)) {
            byte[] bytes = new byte[8192];
            int read = inputStream.read(bytes);
            return new String(bytes, 0, read);
        } catch (IOException | RuntimeException e) {
            e.printStackTrace();
            throw e;
        }
    }

    void stuff() {
        System.out.println("stuff");
    }
}

Após compilar o arquivo fonte, o uso mais simples é:

cd <directory containing classes> (e.g. target/classes)
javap com/stackoverflow/documentation/SpringExample

Que produz a saída

Compiled from "HelloWorldService.java"
public class com.stackoverflow.documentation.HelloWorldService {
  public com.stackoverflow.documentation.HelloWorldService();
  public void sayHello();
  protected java.lang.String tryCatchResources(java.lang.String) throws java.io.IOException;
  void stuff();
}

Isso lista todos os métodos não privados da classe, mas isso não é particularmente útil para a maioria dos propósitos. O comando a seguir é muito mais útil:

javap -p -c -s -constants -l -v com/stackoverflow/documentation/HelloWorldService
