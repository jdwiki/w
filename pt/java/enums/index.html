As enumerações Java (declaradas usando a palavra-chave enum) são sintaxe abreviada para quantidades consideráveis ​​de constantes de uma única classe.

Sintaxe #

  • [public/protected/private] enum Enum_name { // Declara um novo enum.
  • ENUM_CONSTANT_1[, ENUM_CONSTANT_2…]; // Declara as constantes de enumeração. Esta deve ser a primeira linha dentro da enumeração e deve ser separada por vírgulas, com um ponto e vírgula no final.
  • ENUM_CONSTANT_1(param)[, ENUM_CONSTANT_2(param)…]; // Declara constantes enum com parâmetros. Os tipos de parâmetro devem corresponder ao construtor.
  • ENUM_CONSTANT_1 {…}[, ENUM_CONSTANT_2 {…}…]; // Declara constantes enum com métodos sobrescritos. Isso deve ser feito se o enum contiver métodos abstratos; todos esses métodos devem ser implementados.
  • ENUM_CONSTANT.name() // Retorna uma String com o nome da constante enum.
  • ENUM_CONSTANT.ordinal() // Retorna o ordinal desta constante de enumeração, sua posição em sua declaração de enumeração, onde a constante inicial recebe um ordinal de zero.
  • Enum_name.values() // Retorna um novo array (do tipo Enum_name[]) contendo cada constante daquele enum toda vez que é chamado.
  • Enum_name.valueOf(“ENUM_CONSTANT”) // O inverso de ENUM_CONSTANT.name() – retorna a constante enum com o nome fornecido.
  • Enum.valueOf(Enum_name.class, “ENUM_CONSTANT”) // Um ​​sinônimo do anterior: O inverso de ENUM_CONSTANT.name() – retorna a constante enum com o nome fornecido.

Restrições #

Enums sempre estendem java.lang.Enum, então é impossível para um enum estender um classe. No entanto, eles podem implementar muitas interfaces.

dicas e truques #

Devido à sua representação especializada, existem maps e sets mais eficientes .oracle.com/javase/7/docs/api/java/util/EnumSet.html) que podem ser usados ​​com enums como suas chaves. Eles geralmente são executados mais rapidamente do que suas contrapartes não especializadas.

Declarando e usando um enum básico #

Enum pode ser considerado um açúcar de sintaxe para uma classe selada que é instanciada apenas um número de vezes conhecido em tempo de compilação para definir um conjunto de constantes.

Um simples enum para listar as diferentes estações seria declarado da seguinte forma:

public enum Season {
    WINTER,
    SPRING,
    SUMMER,
    FALL
}

Embora as constantes enum não precisem necessariamente estar em maiúsculas, é convenção Java que os nomes das constantes sejam inteiramente em maiúsculas, com palavras separadas por sublinhados.

Você pode declarar um Enum em seu próprio arquivo:

/**
 * This enum is declared in the Season.java file.
*/
public enum Season {
    WINTER,
    SPRING,
    SUMMER,
    FALL
}

Mas você também pode declará-lo dentro de outra classe:

 public class Day {

    private Season season;

    public String getSeason() {
        return season.name();
    }

    public void setSeason(String season) {
        this.season = Season.valueOf(season);
    }

    /**
     * This enum is declared inside the Day.java file and 
     * cannot be accessed outside because it's declared as private.
     */
    private enum Season {
        WINTER,
        SPRING,
        SUMMER,
        FALL
    }

}

Finalmente, você não pode declarar um Enum dentro de um corpo de método ou construtor:

public class Day {

    /**
     * Constructor
    */
    public Day() {
        // Illegal. Compilation error
        enum Season {
            WINTER,
            SPRING,
            SUMMER,
            FALL
        }
    }

    public void aSimpleMethod() {
        // Legal. You can declare a primitive (or an Object) inside a method. Compile!
        int primitiveInt = 42;

        // Illegal. Compilation error.
        enum Season {
            WINTER,
            SPRING,
            SUMMER,
            FALL
        }

        Season season = Season.SPRING;
    }
    
}

Constantes enum duplicadas não são permitidas:

public enum Season {
    WINTER,
    WINTER, //Compile Time Error : Duplicate Constants
    SPRING,
    SUMMER,
    FALL
}

Cada constante de enum é public, static e final por padrão. Como todas as constantes são static, elas podem ser acessadas diretamente usando o nome enum.

As constantes de enumeração podem ser passadas como parâmetros de método:

public static void display(Season s) {
    System.out.println(s.name());  // name() is a built-in method that gets the exact name of the enum constant
}

display(Season.WINTER);  // Prints out "WINTER"

Você pode obter uma matriz das constantes enum usando o método values(). Os valores são garantidos na ordem de declaração na matriz retornada:

Season[] seasons = Season.values();

Observação: este método aloca um novo array de valores cada vez que é chamado.

Para iterar sobre as constantes enum:

public static void enumIterate() {
    for (Season s : Season.values()) {
        System.out.println(s.name());
    }
}

Você pode usar enums em uma instrução switch:

public static void enumSwitchExample(Season s) {
    switch(s) {
        case WINTER:
            System.out.println("It's pretty cold");
            break;
        case SPRING:
            System.out.println("It's warming up");
            break;
        case SUMMER:
            System.out.println("It's pretty hot");
            break;
        case FALL:
            System.out.println("It's cooling down");
            break;
    }
}

Você também pode comparar constantes de enumeração usando ==:

Season.FALL == Season.WINTER    // false
Season.SPRING == Season.SPRING  // true

Outra maneira de comparar constantes de enumeração é usando equals() como abaixo, o que é considerado uma prática ruim, pois você pode facilmente cair em armadilhas da seguinte forma:

Season.FALL.equals(Season.FALL); // true
Season.FALL.equals(Season.WINTER); // false
Season.FALL.equals("FALL"); // false and no compiler error

Além disso, embora o conjunto de instâncias no enum não possa ser alterado em tempo de execução, as próprias instâncias não são inerentemente imutáveis ​​porque, como qualquer outra classe, um enum pode conter campos mutáveis ​​como demonstrado abaixo.

public enum MutableExample {
    A,
    B;

    private int count = 0;

    public void increment() {
        count++;
    }

    public void print() {
        System.out.println("The count of " + name() + " is " + count);
    }
}

// Usage:
MutableExample.A.print();       // Outputs 0
MutableExample.A.increment();
MutableExample.A.print();       // Outputs 1 -- we've changed a field   
MutableExample.B.print();       // Outputs 0 -- another instance remains unchanged

No entanto, uma boa prática é tornar as instâncias enum imutáveis, ou seja, quando elas não têm nenhum campo adicional ou todos esses campos são marcados como final e são imutáveis. Isso garantirá que durante toda a vida útil do aplicativo um enum não vaze nenhuma memória e que seja seguro usar suas instâncias em todos os threads.

Enums implementam implicitamente Serializable e Comparable porque a classe Enum faz:

public abstract class Enum<E extends Enum<E>>
extends Object
implements Comparable<E>, Serializable

Enums com construtores #

Um enum não pode ter um construtor público; no entanto, construtores privados são aceitáveis ​​(construtores para enums são package-private por padrão):

public enum Coin {
    PENNY(1), NICKEL(5), DIME(10), QUARTER(25); // usual names for US coins
    // note that the above parentheses and the constructor arguments match
    private int value;

    Coin(int value) { 
        this.value = value;
    }

    public int getValue() {
        return value;
    }
}

int p = Coin.NICKEL.getValue(); // the int value will be 5

É recomendável que você mantenha todos os campos privados e forneça métodos getter, pois há um número finito de instâncias para uma enumeração.

Se você fosse implementar um Enum como uma class, ficaria assim:

public class Coin<T extends Coin<T>> implements Comparable<T>, Serializable{
    public static final Coin PENNY = new Coin(1);
    public static final Coin NICKEL = new Coin(5);
    public static final Coin DIME = new Coin(10);
    public static final Coin QUARTER = new Coin(25);

    private int value;

    private Coin(int value){
        this.value = value;
    }

    public int getValue() {
        return value;
    }
}

int p = Coin.NICKEL.getValue(); // the int value will be 5

As constantes de enumeração são tecnicamente mutáveis, portanto, um setter pode ser adicionado para alterar a estrutura interna de uma constante de enumeração. No entanto, isso é considerado uma prática muito ruim e deve ser evitada.

A melhor prática é tornar os campos Enum imutáveis, com final:

public enum Coin {
    PENNY(1), NICKEL(5), DIME(10), QUARTER(25);

    private final int value;

    Coin(int value){ 
        this.value = value;
    }

    ...

}

Você pode definir vários construtores no mesmo enum. Quando você faz isso, os argumentos que você passa em sua declaração enum decidem qual construtor é chamado:

public enum Coin {
    PENNY(1, true), NICKEL(5, false), DIME(10), QUARTER(25);

    private final int value;
    private final boolean isCopperColored;

    Coin(int value){
        this(value, false);
    }

    Coin(int value, boolean isCopperColored){ 
        this.value = value;
        this.isCopperColored = isCopperColored;
    }

    ...

}

Nota: Todos os campos enum não primitivos devem implementar [Serializable][2] porque a classe [Enum][3] o faz.

Enums com métodos abstratos #

Enums podem definir métodos abstratos, que cada membro enum deve implementar.

enum Action {
    DODGE {
        public boolean execute(Player player) {
            return player.isAttacking();
        }
    },
    ATTACK {
        public boolean execute(Player player) {
            return player.hasWeapon();
        }
    },
    JUMP {
        public boolean execute(Player player) {
            return player.getCoordinates().equals(new Coordinates(0, 0));
        }
    };

    public abstract boolean execute(Player player);
}

Isso permite que cada membro enum defina seu próprio comportamento para uma determinada operação, sem precisar ativar os tipos em um método na definição de nível superior.

Observe que esse padrão é uma forma abreviada do que normalmente é alcançado usando polimorfismo e/ou implementando interfaces.

Implementa Interface #

Este é um enum que também é uma função que pode ser chamada que testa entradas String contra padrões de expressão regular pré-compilados.

import java.util.function.Predicate;
import java.util.regex.Pattern;

enum RegEx implements Predicate<String> {
    UPPER("[A-Z]+"), LOWER("[a-z]+"), NUMERIC("[+-]?[0-9]+");

    private final Pattern pattern;

    private RegEx(final String pattern) {
        this.pattern = Pattern.compile(pattern);
    }

    @Override 
    public boolean test(final String input) {
        return this.pattern.matcher(input).matches();
    }
}

public class Main {
    public static void main(String[] args) {
        System.out.println(RegEx.UPPER.test("ABC"));
        System.out.println(RegEx.LOWER.test("abc"));
        System.out.println(RegEx.NUMERIC.test("+111"));
    }
}

Cada membro do enum também pode implementar o método:

import java.util.function.Predicate;

enum Acceptor implements Predicate<String> {
    NULL {
        @Override
        public boolean test(String s) { return s == null; }
    },
    EMPTY {
        @Override
        public boolean test(String s) { return s.equals(""); }
    },
    NULL_OR_EMPTY {
        @Override
        public boolean test(String s) { return NULL.test(s) || EMPTY.test(s); }
    };
}

public class Main {
    public static void main(String[] args) {
        System.out.println(Acceptor.NULL.test(null));  // true
        System.out.println(Acceptor.EMPTY.test(""));   // true
        System.out.println(Acceptor.NULL_OR_EMPTY.test(" ")); // false
    }
}

Implemente o padrão Singleton com uma enumeração de elemento único #

As constantes de enumeração são instanciadas quando uma enumeração é referenciada pela primeira vez. Portanto, isso permite implementar o padrão de projeto de software Singleton com uma enumeração de elemento único.

public enum Attendant {

    INSTANCE;

    private Attendant() {
        // perform some initialization routine
    }

    public void sayHello() {
        System.out.println("Hello!");
    }
}


public class Main {

    public static void main(String... args) {
        Attendant.INSTANCE.sayHello();// instantiated at this point
    }
}

De acordo com o livro “Effective Java” de Joshua Bloch, uma enumeração de elemento único é a melhor maneira de implementar um singleton. Esta abordagem tem as seguintes vantagens:

  • segurança da rosca
  • garantia de instanciação única
  • serialização pronta para uso

E como mostrado na seção [implements interface][2] este singleton também pode implementar uma ou mais interfaces.

[2]: https://www.wikiod.com/pt/java/enums#Interface de Implementos “interface de implementos”

Usando métodos e blocos estáticos #

Um enum pode conter um método, assim como qualquer classe. Para ver como isso funciona, vamos declarar um enum como este:

public enum Direction {
    NORTH, SOUTH, EAST, WEST;
}

Vamos ter um método que retorna o enum na direção oposta:

public enum Direction {
    NORTH, SOUTH, EAST, WEST;

    public Direction getOpposite(){
        switch (this){
            case NORTH:
                return SOUTH;               
            case SOUTH:
                return NORTH;                
            case WEST:
                return EAST; 
            case EAST:
                return WEST;  
            default: //This will never happen
                return null;
        }
    }
}

Isso pode ser melhorado ainda mais com o uso de campos e blocos inicializadores estáticos:

public enum Direction {
    NORTH, SOUTH, EAST, WEST;
    
    private Direction opposite;
    
    public Direction getOpposite(){
        return opposite;
    }
    
    static {
        NORTH.opposite = SOUTH;
        SOUTH.opposite = NORTH;
        WEST.opposite = EAST;
        EAST.opposite = WEST;
    }
}

Neste exemplo, a direção oposta é armazenada em um campo de instância privada opposite, que é inicializado estaticamente na primeira vez que uma Direction é usada. Neste caso particular (porque NORTH referencia SOUTH e vice-versa), não podemos usar Enums with constructors aqui (Construtores NORTH(SOUTH), SOUTH(NORTH), EAST(WEST), WEST(EAST ) seria mais elegante e permitiria que opposite fosse declarado final, mas seria auto-referencial e, portanto, não é permitido).

Enum como um parâmetro de tipo limitado #

Ao escrever uma classe com genéricos em java, é possível garantir que o parâmetro de tipo seja um enum. Como todos os enums estendem a classe Enum, a seguinte sintaxe pode ser usada.

public class Holder<T extends Enum<T>> {
    public final T value;

    public Holder(T init) {
        this.value = init;
    }
}

Neste exemplo, o tipo T deve ser um enum.

Corpo específico da constante de enumeração #

Em um enum é possível definir um comportamento específico para uma constante particular do enum que substitui o comportamento padrão do enum, esta técnica é conhecida como corpo específico constante.

Suponha que três alunos de piano - John, Ben e Luke - sejam definidos em um enum chamado PianoClass, como segue:

    enum PianoClass {
    JOHN, BEN, LUKE;
        public String getSex() {
            return "Male";
        }
        public String getLevel() {
            return "Beginner";
        }
    }

E um dia chegam outros dois alunos - Rita e Tom - com um sexo (Feminino) e um nível (Intermediário) que não condizem com os anteriores:

    enum PianoClass2 {
    JOHN, BEN, LUKE, RITA, TOM;
        public String getSex() {
            return "Male"; // issue, Rita is a female
        }
        public String getLevel() {
            return "Beginner"; // issue, Tom is an intermediate student
        }
    }

de modo que simplesmente adicionar os novos alunos à declaração constante, como segue, não está correto:

PianoClass2 tom = PianoClass2.TOM;
PianoClass2 rita = PianoClass2.RITA;
System.out.println(tom.getLevel()); // prints Beginner -> wrong Tom's not a beginner
System.out.println(rita.getSex()); // prints Male -> wrong Rita's not a male

É possível definir um comportamento específico para cada uma das constantes, Rita e Tom, que substitui o comportamento padrão PianoClass2 da seguinte forma:

enum PianoClass3 {
    JOHN, BEN, LUKE,
    RITA {
        @Override
        public String getSex() {
            return "Female";
        }
    },
    TOM {
        @Override
        public String getLevel() {
            return "Intermediate";
        }
    };
    public String getSex() {
        return "Male";
    }
    public String getLevel() {
        return "Beginner";
    }
}

e agora o nível do Tom e o sexo da Rita estão como deveriam ser:

PianoClass3 tom = PianoClass3.TOM;
PianoClass3 rita = PianoClass3.RITA;
System.out.println(tom.getLevel()); // prints Intermediate
System.out.println(rita.getSex()); // prints Female

Outra maneira de definir o corpo específico do conteúdo é usando o construtor, por exemplo:

enum Friend {
    MAT("Male"),
    JOHN("Male"),
    JANE("Female");
    
    private String gender;

    Friend(String gender) {
        this.gender = gender;
    }

    public String getGender() {
        return this.gender;
    }
}

e uso:

Friend mat = Friend.MAT;
Friend john = Friend.JOHN;
Friend jane = Friend.JANE;
System.out.println(mat.getGender());     // Male
System.out.println(john.getGender());    // Male
System.out.println(jane.getGender());    // Female

enum Util {
    /* No instances */;

    public static int clamp(int min, int max, int i) {
        return Math.min(Math.max(i, min), max);
    }

    // other utility methods...
}

Assim como enum [pode ser usado para singletons](https://www.wikiod.com/pt/java/enums#Implemente o padrão Singleton com uma enumeração de elemento único) (1 instância classes), ele pode ser usado para classes utilitárias (classes de instância 0). Apenas certifique-se de encerrar a lista (vazia) de constantes enum com um ;.

Obtendo os valores de um enum #

Cada classe enum contém um método estático implícito chamado values(). Este método retorna uma matriz contendo todos os valores dessa enumeração. Você pode usar esse método para iterar sobre os valores. É importante notar, entretanto, que este método retorna um array new toda vez que é chamado.

public enum Day {
    MONDAY, TUESDAY, WEDNESDAY, THURSDAY, FRIDAY, SATURDAY, SUNDAY;
    
    /**
    * Print out all the values in this enum.
    */
    public static void printAllDays() {
        for(Day day : Day.values()) {
            System.out.println(day.name());
        }
    }
}

Se você precisar de um Set você pode usar EnumSet.allOf(Day.class) também.

Padrão de polimorfismo enum #

Quando um método precisa aceitar um conjunto “extensível” de valores enum, o programador pode aplicar polimorfismo como em uma class normal, criando uma interface que será usada em qualquer lugar onde o enums deva ser usado:

public interface ExtensibleEnum {
    String name();
}

Desta forma, qualquer enum marcado (implementando) a interface pode ser usado como parâmetro, permitindo ao programador criar uma quantidade variável de enums que será aceita pelo método. Isso pode ser útil, por exemplo, em APIs onde há um enum padrão (não modificável) e o usuário dessas APIs deseja “estender” o enum com mais valores.

Um conjunto de valores de enumeração padrão pode ser definido da seguinte forma:

public enum DefaultValues implements ExtensibleEnum {
    VALUE_ONE, VALUE_TWO;
}

Valores adicionais podem ser definidos assim:

public enum ExtendedValues implements ExtensibleEnum {
    VALUE_THREE, VALUE_FOUR;
}

Exemplo que mostra como usar os enums - observe como printEnum() aceita valores de ambos os tipos enum:

private void printEnum(ExtensibleEnum val) {
    System.out.println(val.name());
}  

printEnum(DefaultValues.VALUE_ONE);    // VALUE_ONE
printEnum(DefaultValues.VALUE_TWO);    // VALUE_TWO
printEnum(ExtendedValues.VALUE_THREE); // VALUE_THREE
printEnum(ExtendedValues.VALUE_FOUR);  // VALUE_FOUR

Observação: esse padrão não impede que você redefina valores de enum, que já estão definidos em um enum, em outro enum. Esses valores enum seriam instâncias diferentes. Além disso, não é possível usar o switch-on-enum, pois tudo o que temos é a interface, não o enum real.

Comparar e Contém para valores de Enum #

Enums contém apenas constantes e pode ser comparado diretamente com ==. Portanto, apenas a verificação de referência é necessária, não há necessidade de usar o método .equals. Além disso, se .equals for usado incorretamente, pode gerar a NullPointerException enquanto que não é o caso da verificação ==.

enum Day {
    GOOD, AVERAGE, WORST;
}

public class Test {

    public static void main(String[] args) {
        Day day = null;

        if (day.equals(Day.GOOD)) {//NullPointerException!
            System.out.println("Good Day!");
        }

        if (day == Day.GOOD) {//Always use == to compare enum
            System.out.println("Good Day!");
        }

    }
}

Para agrupar, complementar, ranger os valores enum temos a classe EnumSet que contém diferentes métodos.

  • EnumSet#range: Para obter o subconjunto de enum por intervalo definido por dois pontos de extremidade

  • EnumSet#of : Conjunto de enums específicos sem qualquer intervalo. Vários métodos of sobrecarregados estão lá.

  • EnumSet#complementOf: Conjunto de enum que é complemento dos valores de enum fornecidos no parâmetro do método

    enum Page {
   A1, A2, A3, A4, A5, A6, A7, A8, A9, A10
}

public class Test {

  public static void main(String[] args) {
      EnumSet<Page> range = EnumSet.range(Page.A1, Page.A5);

      if (range.contains(Page.A4)) {
          System.out.println("Range contains A4");
      }

      EnumSet<Page> of = EnumSet.of(Page.A1, Page.A5, Page.A3);

      if (of.contains(Page.A1)) {
          System.out.println("Of contains A1");
      }
  }
}

Obtém a constante enum por nome #

Digamos que tenhamos um enum DayOfWeek:

enum DayOfWeek {
    SUNDAY, MONDAY, TUESDAY, WEDNESDAY, THURSDAY, FRIDAY, SATURDAY;
}

Um enum é compilado com um método estático valueOf() que pode ser usado para pesquisar uma constante por seu nome:

String dayName = DayOfWeek.SUNDAY.name();
assert dayName.equals("SUNDAY");

DayOfWeek day = DayOfWeek.valueOf(dayName);
assert day == DayOfWeek.SUNDAY;

Isso também é possível usando um tipo de enumeração dinâmica:

Class<DayOfWeek> enumType = DayOfWeek.class;
DayOfWeek day = Enum.valueOf(enumType, "SUNDAY");
assert day == DayOfWeek.SUNDAY;

Ambos os métodos valueOf() lançarão um IllegalArgumentException se o enum especificado não tiver uma constante com um nome correspondente.

A biblioteca Guava fornece um método auxiliar Enums.getIfPresent() que retorna um Guava [Optional][2] para eliminar o tratamento de exceção explícito:

DayOfWeek defaultDay = DayOfWeek.SUNDAY;
DayOfWeek day = Enums.valueOf(DayOfWeek.class, "INVALID").or(defaultDay);
assert day == DayOfWeek.SUNDAY;

1: https://google.github.io/guava/releases/18.0/api/docs/com/google/common/base/Enums.html#getIfPresent(java.lang.Class,%20java.lang.String ) [2]: https://google.github.io/guava/releases/18.0/api/docs/com/google/common/base/Optional.html

Enum com propriedades (campos) #

Caso queiramos usar enum com mais informações e não apenas como valores constantes, e queremos poder comparar duas enumerações.

Considere o seguinte exemplo:

public enum Coin {
    PENNY(1), NICKEL(5), DIME(10), QUARTER(25);

    private final int value;

    Coin(int value){
        this.value = value;
    }

    public boolean isGreaterThan(Coin other){
        return this.value > other.value;
    }

}

Aqui definimos um Enum chamado Coin que representa seu valor. Com o método isGreaterThan podemos comparar dois enums:

Coin penny = Coin.PENNY;
Coin dime = Coin.DIME;

System.out.println(penny.isGreaterThan(dime)); // prints: false
System.out.println(dime.isGreaterThan(penny)); // prints: true

Converter enum em String #

Às vezes, você deseja converter seu enum em uma String, existem duas maneiras de fazer isso.

Suponha que temos:

public enum Fruit {
    APPLE, ORANGE, STRAWBERRY, BANANA, LEMON, GRAPE_FRUIT;
}

Então, como podemos converter algo como Fruit.APPLE para "APPLE"?


Converta usando name() #

name() é um método interno em enum que retorna a representação String do enum, o retorno String representa exatamente como o valor do enum foi definido.

Por exemplo:

System.out.println(Fruit.BANANA.name());      // "BANANA"
System.out.println(Fruit.GRAPE_FRUIT.name()); // "GRAPE_FRUIT"

Converta usando toString() #

toString() é, por padrão, substituído para ter o mesmo comportamento que name()

No entanto, toString() é provavelmente substituído por desenvolvedores para torná-lo imprimir uma String mais amigável

Não use toString() se você quiser checar seu código, name() é muito mais estável para isso. Só use toString() quando você for enviar o valor para logs ou stdout ou algo assim

Por padrão: #

System.out.println(Fruit.BANANA.toString());      // "BANANA"
System.out.println(Fruit.GRAPE_FRUIT.toString()); // "GRAPE_FRUIT"

Exemplo de substituição #

System.out.println(Fruit.BANANA.toString());      // "Banana"
System.out.println(Fruit.GRAPE_FRUIT.toString()); // "Grape Fruit"

Enums com campos estáticos #

Se sua classe enum precisar ter campos estáticos, lembre-se de que eles são criados depois dos próprios valores enum. Isso significa que o código a seguir resultará em um NullPointerException:

enum Example {
    ONE(1), TWO(2);

    static Map<String, Integer> integers = new HashMap<>();

    private Example(int value) {
        integers.put(this.name(), value);
    }
}

Uma maneira possível de corrigir isso:

enum Example {
    ONE(1), TWO(2);

    static Map<String, Integer> integers;

    private Example(int value) {
        putValue(this.name(), value);
    }

    private static void putValue(String name, int value) {
        if (integers == null)
            integers = new HashMap<>();
        integers.put(name, value);
    }
}

Não inicialize o campo estático:

enum Example {
    ONE(1), TWO(2);

    // after initialisisation integers is null!!
    static Map<String, Integer> integers = null;

    private Example(int value) {
        putValue(this.name(), value);
    }

    private static void putValue(String name, int value) {
        if (integers == null)
            integers = new HashMap<>();
        integers.put(name, value);
    }
    // !!this may lead to null poiner exception!!
    public int getValue(){
        return (Example.integers.get(this.name()));
    }
}

inicialização:

  • criar os valores enum
  • como efeito colateral putValue() chamado que inicializa inteiros
  • os valores estáticos são definidos
  • inteiros = nulo; // é executado após as enumerações para que o conteúdo de inteiros seja perdido
