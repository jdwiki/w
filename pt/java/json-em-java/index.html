JSON (JavaScript Object Notation) é um formato de troca de dados leve, baseado em texto e independente de linguagem que é fácil para humanos e máquinas lerem e escreverem. JSON pode representar dois tipos estruturados: objetos e matrizes. JSON é frequentemente usado em aplicativos Ajax, configurações, bancos de dados e serviços web RESTful. A API Java para processamento JSON fornece APIs portáteis para analisar, gerar, transformar e consultar JSON.

Este exemplo se concentra em analisar e criar JSON em Java usando várias bibliotecas, como a biblioteca Google Gson, Jackson Object Mapper e outras.

Exemplos usando outras bibliotecas podem ser encontrados aqui: Como analisar JSON em Java

Usando o mapeador de objetos Jackson #

Modelo Pojo

public class Model {
    private String firstName;
    private String lastName;
    private int age;
    /* Getters and setters not shown for brevity */        
}

Exemplo: String para Objeto

Model outputObject = objectMapper.readValue(
     "{\"firstName\":\"John\",\"lastName\":\"Doe\",\"age\":23}",
     Model.class);
System.out.println(outputObject.getFirstName());
//result: John

Exemplo: Objeto para String

String jsonString = objectMapper.writeValueAsString(inputObject));
//result: {"firstName":"John","lastName":"Doe","age":23}

Detalhes #

Declaração de importação necessária:

import com.fasterxml.jackson.databind.ObjectMapper;

Dependência Maven: jackson-databind

instância do ObjectMapper #

//creating one
ObjectMapper objectMapper = new ObjectMapper();
  • ObjectMapper é thread-safe
  • recomendado: ter uma instância estática compartilhada

Desserialização: #

<T> T readValue(String content, Class<T> valueType)
  • valueType precisa ser especificado – o retorno será deste tipo
  • Arremessos
  • IOException - no caso de um problema de E/S de baixo nível
  • JsonParseException - se a entrada subjacente contiver conteúdo inválido
  • JsonMappingException - se a estrutura JSON de entrada não corresponder à estrutura do objeto

Exemplo de uso (jsonString é a string de entrada):

Model fromJson = objectMapper.readValue(jsonString, Model.class);

Método para serialização: #

String writeValueAsString(valor do objeto)

  • Arremessos
  • JsonProcessingException em caso de erro
  • Nota: antes da versão 2.1, a cláusula throws incluía IOException; 2.1 removeu.

JSON para objeto (biblioteca Gson) #

Vamos supor que você tenha uma classe chamada Person com apenas name

private class Person {
    public String name;

    public Person(String name) {
        this.name = name;
    }
}

Código:

Gson gson = new Gson();
String json = "{\"name\": \"John\"}";

Person person = gson.fromJson(json, Person.class);
System.out.println(person.name); //John

Você deve ter gson library em seu classpath.

Objeto para JSON (Biblioteca Gson) #

Vamos supor que você tenha uma classe chamada Person com apenas name

private class Person {
    public String name;

    public Person(String name) {
        this.name = name;
    }
}

Código:

Gson g = new Gson();

Person person = new Person("John");
System.out.println(g.toJson(person)); // {"name":"John"}

Claro que o jar Gson deve estar no classpath.

Codificando dados como JSON #

Se você precisar criar um JSONObject e colocar dados nele, considere o seguinte exemplo:

// Create a new javax.json.JSONObject instance.
JSONObject first = new JSONObject();

first.put("foo", "bar");
first.put("temperature", 21.5);
first.put("year", 2016);

// Add a second object.
JSONObject second = new JSONObject();
second.put("Hello", "world");
first.put("message", second);

// Create a new JSONArray with some values
JSONArray someMonths = new JSONArray(new String[] { "January", "February" });
someMonths.put("March");
// Add another month as the fifth element, leaving the 4th element unset.
someMonths.put(4, "May");

// Add the array to our object
object.put("months", someMonths);

// Encode
String json = object.toString();

// An exercise for the reader: Add pretty-printing!
/* {
       "foo":"bar",
       "temperature":21.5,
       "year":2016,
       "message":{"Hello":"world"},
       "months":["January","February","March",null,"May"]
   }
*/

Decodificando dados JSON #

Se você precisar obter dados de um JSONObject, considere o seguinte exemplo:

String json = "{\"foo\":\"bar\",\"temperature\":21.5,\"year\":2016,\"message\":{\"Hello\":\"world\"},\"months\":[\"January\",\"February\",\"March\",null,\"May\"]}";

// Decode the JSON-encoded string
JSONObject object = new JSONObject(json);

// Retrieve some values
String foo = object.getString("foo");
double temperature = object.getDouble("temperature");
int year = object.getInt("year");

// Retrieve another object
JSONObject secondary = object.getJSONObject("message");
String world = secondary.getString("Hello");

// Retrieve an array
JSONArray someMonths = object.getJSONArray("months");

String json = "{\"name\": \"John\", \"age\":21}";

JsonArray para lista Java (Biblioteca Gson) #

Aqui está um JsonArray simples que você gostaria de converter em um Java ArrayList:

{
    "list": [
                "Test_String_1",
                "Test_String_2"
            ] 
}

Agora passe o JsonArray ’list’ para o seguinte método que retorna um Java ArrayList correspondente:

public ArrayList<String> getListString(String jsonList){
    Type listType = new TypeToken<List<String>>() {}.getType();
    //make sure the name 'list' matches the name of 'JsonArray' in your 'Json'.
    ArrayList<String> list = new Gson().fromJson(jsonList, listType);    
    return list;
}

Você deve adicionar a seguinte dependência do maven ao seu arquivo POM.xml:

<!-- https://mvnrepository.com/artifact/com.google.code.gson/gson -->
<dependency>
    <groupId>com.google.code.gson</groupId>
    <artifactId>gson</artifactId>
    <version>2.7</version>
</dependency>

Ou você deve ter o jar com.google.code.gson:gson:jar:<version> em seu classpath.

Desserialize a coleção JSON para a coleção de objetos usando Jackson #

Suponha que você tenha uma classe pojo Person

public class Person {
    public String name;

    public Person(String name) {
        this.name = name;
    }
}

E você deseja analisá-lo em uma matriz JSON ou um mapa de objetos Person. Devido ao apagamento de tipo, você não pode construir classes de List<Person> e Map<String, Person> em tempo de execução diretamente (e, portanto, usá-los para desserializar JSON). Para superar essa limitação, Jackson fornece duas abordagens - TypeFactory e TypeReference.

TipoFábrica

A abordagem adotada aqui é usar uma fábrica (e sua função de utilidade estática) para construir seu tipo para você. Os parâmetros necessários são a coleção que você deseja usar (lista, conjunto, etc.) e a classe que deseja armazenar nessa coleção.

TipoReferência

A abordagem de referência de tipo parece mais simples porque economiza um pouco de digitação e parece mais limpa. TypeReference aceita um parâmetro de tipo, onde você passa o tipo desejado List<Person>. Você simplesmente instancia esse objeto TypeReference e o usa como seu contêiner de tipo.

Agora vamos ver como realmente desserializar seu JSON em um objeto Java. Se seu JSON estiver formatado como uma matriz, você poderá desserializá-lo como uma lista. Se houver uma estrutura aninhada mais complexa, você desejará desserializar para um Mapa. Veremos exemplos de ambos.

Desserializando array JSON #

String jsonString = "[{\"name\": \"Alice\"}, {\"name\": \"Bob\"}]"

Abordagem TypeFactory #

CollectionType listType = 
    factory.constructCollectionType(List.class, Person.class);
List<Preson> list = mapper.readValue(jsonString, listType);

##TypeAbordagem de referência

TypeReference<Person> listType = new TypeReference<List<Person>>() {};
List<Person> list = mapper.readValue(jsonString, listType);

Desserializando o mapa JSON #

String jsonString = "{\"0\": {\"name\": \"Alice\"}, \"1\": {\"name\": \"Bob\"}}"

Abordagem TypeFactory #

CollectionType mapType = 
    factory.constructMapLikeType(Map.class, String.class, Person.class);
List<Person> list = mapper.readValue(jsonString, mapType);

##TypeAbordagem de referência

TypeReference<Person> mapType = new TypeReference<Map<String, Person>>() {};
Map<String, Person> list = mapper.readValue(jsonString, mapType);

Detalhes #

Instrução de importação usada:

import com.fasterxml.jackson.core.type.TypeReference;
import com.fasterxml.jackson.databind.ObjectMapper;
import com.fasterxml.jackson.databind.type.CollectionType;

Instâncias usadas:

ObjectMapper mapper = new ObjectMapper();
TypeFactory factory = mapper.getTypeFactory();

Observação #

Embora a abordagem TypeReference possa parecer melhor, ela tem várias desvantagens:

  1. TypeReference deve ser instanciado usando classe anônima
  2. Você deve fornecer explicitamente

Não fazer isso pode levar à perda do argumento de tipo genérico que levará à falha de desserialização.
