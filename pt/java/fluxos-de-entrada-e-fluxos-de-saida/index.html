Sintaxe #

  • int read(byte[] b) lança IOException

Observe que na maioria das vezes você NÃO usa InputStreams diretamente, mas usa BufferedStreams, ou similar. Isso ocorre porque o InputStream lê da fonte toda vez que o método read é chamado. Isso pode causar uso significativo da CPU em alternâncias de contexto para dentro e para fora do kernel.

Streams de fechamento #

A maioria dos fluxos deve ser fechada quando você terminar com eles, caso contrário, você pode introduzir um vazamento de memória ou deixar um arquivo aberto. É importante que os fluxos sejam fechados mesmo se uma exceção for lançada.

try(FileWriter fw = new FileWriter("outfilename");
    BufferedWriter bw = new BufferedWriter(fw);
    PrintWriter out = new PrintWriter(bw))
{
    out.println("the text");
    //more code
    out.println("more text");
    //more code
} catch (IOException e) {
    //handle this however you 
}

Lembre-se: try-with-resources garante que os recursos foram fechados quando o bloco é encerrado, se isso acontece com o fluxo de controle usual ou por causa de uma exceção.

Às vezes, try-with-resources não é uma opção, ou talvez você esteja suportando uma versão mais antiga do Java 6 ou anterior. Neste caso, o tratamento adequado é usar um bloco finally:

FileWriter fw = null;
BufferedWriter bw = null;
PrintWriter out = null;
try {
    fw = new FileWriter("myfile.txt");
    bw = new BufferedWriter(fw);
    out = new PrintWriter(bw);
    out.println("the text");
    out.close();
} catch (IOException e) {
    //handle this however you want
}
finally {
    try {
        if(out != null)
            out.close();
    } catch (IOException e) {
        //typically not much you can do here...
    }
}

Observe que fechar um fluxo de wrapper também fechará seu fluxo subjacente. Isso significa que você não pode encapsular um fluxo, fechar o wrapper e continuar usando o fluxo original.

Lendo InputStream em uma String #

Às vezes você pode querer ler a entrada de bytes em uma String. Para fazer isso, você precisará encontrar algo que converta entre byte e os Codepoints UTF-16 “Java nativo” usados ​​como char. Isso é feito com um InputStreamReader.

Para acelerar um pouco o processo, é “normal” alocar um buffer, para que não tenhamos muita sobrecarga ao ler de Input.

public String inputStreamToString(InputStream inputStream) throws Exception {
     StringWriter writer = new StringWriter();

     char[] buffer = new char[1024];
     try (Reader reader = new BufferedReader(new InputStreamReader(inputStream, "UTF-8"))) {
           int n;
           while ((n = reader.read(buffer)) != -1) {
                // all this code does is redirect the output of `reader` to `writer` in
                // 1024 byte chunks
                writer.write(buffer, 0, n);
           }
     }
     return writer.toString();
}

Transformar este exemplo em código compatível com Java SE 6 (e inferior) é deixado de fora como exercício para o leitor.

Quebra de fluxos de entrada/saída #

OutputStream e InputStream têm muitas classes diferentes, cada uma com uma funcionalidade única. Ao envolver um fluxo em outro, você obtém a funcionalidade de ambos os fluxos.

Você pode envolver um fluxo quantas vezes quiser, apenas observe a ordem.

Combinações úteis #

Escrevendo caracteres em um arquivo ao usar um buffer

File myFile = new File("targetFile.txt");
PrintWriter writer = new PrintWriter(new BufferedOutputStream(new FileOutputStream(myFile)));

Compactando e criptografando dados antes de gravar em um arquivo usando um buffer

Cipher cipher = ... // Initialize cipher
File myFile = new File("targetFile.enc");
BufferedOutputStream outputStream = new BufferedOutputStream(new DeflaterOutputStream(new CipherOutputStream(new FileOutputStream(myFile), cipher)));

Lista de wrappers de fluxo de entrada/saída #

Embrulho Descrição
BufferedOutputStream/ BufferedInputStream Enquanto o OutputStream grava os dados um byte por vez, o BufferedOutputStream grava os dados em blocos. Isso reduz o número de chamadas do sistema, melhorando assim o desempenho.
DeflaterOutputStream/ DeflaterInputStream Executa a compactação de dados.
InflaterOutputStream/ InflaterInputStream Executa a descompressão de dados.
CipherOutputStream/ CipherInputStream Criptografa/descriptografa dados.
DigestOutputStream/DigestInputStream Gera Message Digest para verificar a integridade dos dados.
CheckedOutputStream/ CheckedInputStream Gera um CheckSum. CheckSum é uma versão mais trivial do Message Digest.
DataOutputStream/DataInputStream Permite a escrita de tipos de dados primitivos e Strings. Destinado a escrever bytes. Plataforma independente.
PrintStream Permite a escrita de tipos de dados primitivos e Strings. Destinado a escrever bytes. Dependente da plataforma.
OutputStreamWriter Converte um OutputStream em um Writer. Um OutputStream lida com bytes enquanto Writers lida com caracteres
PrintWriter Chama automaticamente OutputStreamWriter. Permite a escrita de tipos de dados primitivos e Strings. Estritamente para escrever personagens e melhor para escrever personagens

Gravando bytes em um OutputStream #

Escrevendo bytes em um OutputStream um byte de cada vez

OutputStream stream = object.getOutputStream();

byte b = 0x00;
stream.write( b );

Escrevendo uma matriz de bytes

byte[] bytes = new byte[] { 0x00, 0x00 };

stream.write( bytes );

Escrevendo uma seção de uma matriz de bytes

int offset = 1;
int length = 2;
byte[] bytes = new byte[] { 0xFF, 0x00, 0x00, 0xFF };

stream.write( bytes, offset, length );

Copiando o fluxo de entrada para o fluxo de saída #

Esta função copia dados entre dois fluxos -

void copy(InputStream in, OutputStream out) throws IOException {
    byte[] buffer = new byte[8192];
    while ((bytesRead = in.read(buffer)) > 0) {
        out.write(buffer, 0, bytesRead);
    }
}

Exemplo -

 // reading from System.in and writing to System.out
copy(System.in, System.out);
