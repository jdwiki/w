  • Operações de lógica binária em BigInteger

A classe BigInteger é usada para operações matemáticas envolvendo números inteiros grandes com magnitudes muito grandes para tipos de dados primitivos . Por exemplo, 100-factorial tem 158 dígitos - muito maior do que um ’longo’ pode representar. BigInteger fornece análogos para todos os operadores de inteiros primitivos de Java, e todos os métodos relevantes de java.lang.Math, bem como algumas outras operações.

Sintaxe #

  • BigInteger nome_variável = new BigInteger(“12345678901234567890”); // um inteiro decimal como uma string
  • BigInteger variable_name = new BigInteger(“1010101101010100101010011000110011101011000111110000101011010010”, 2) // um inteiro binário como uma string
  • BigInteger variable_name = new BigInteger(“ab54a98ceb1f0800”, 16) // um inteiro hexadecimal como uma string
  • BigInteger nome_variável = new BigInteger(64, new Random()); // um gerador de números pseudoaleatórios fornecendo 64 bits para construir um inteiro
  • BigInteger nome_variável = new BigInteger(novo byte[]{0, -85, 84, -87, -116, -21, 31, 10, -46}); // representação em complemento de dois com sinal de um inteiro (big endian)
  • BigInteger nome_variável = new BigInteger(1, new byte[]{-85, 84, -87, -116, -21, 31, 10, -46}); // representação em complemento de dois sem sinal de um inteiro positivo (big endian)

BigInteger é imutável. Portanto, você não pode alterar seu estado. Por exemplo, o seguinte não funcionará, pois sum não será atualizado devido à imutabilidade.

BigInteger sum = BigInteger.ZERO;
for(int i = 1; i < 5000; i++) {
   sum.add(BigInteger.valueOf(i));  
}

Atribua o resultado à variável sum para que funcione.

sum = sum.add(BigInteger.valueOf(i));

A documentação oficial do BigInteger afirma que as implementações do BigInteger devem suportar todos os números inteiros entre -2< sup>2147483647 e 22147483647 (exclusivo). Isso significa que BigIntegers pode ter mais de 2 bilhões de bits!

Inicialização #

A classe java.math.BigInteger fornece operações análogas para todos os operadores de inteiros primitivos de Java e para todos os métodos relevantes de java.lang.Math. Como o pacote java.math não é disponibilizado automaticamente, você pode ter que importar java.math.BigInteger antes de poder usar o nome de classe simples.

Para converter valores long ou int para BigInteger use:

long longValue = Long.MAX_VALUE;
BigInteger valueFromLong = BigInteger.valueOf(longValue);

ou, para números inteiros:

int intValue = Integer.MIN_VALUE; // negative
BigInteger valueFromInt = BigInteger.valueOf(intValue);

que ampliará o inteiro intValue para long, usando extensão de bit de sinal para valores negativos, para que os valores negativos permaneçam negativos.

Para converter um String numérico para BigInteger use:

String decimalString = "-1";
BigInteger valueFromDecimalString = new BigInteger(decimalString);

O construtor a seguir é usado para traduzir a representação String de um BigInteger na base especificada em um BigInteger.

String binaryString = "10";
int binaryRadix = 2;
BigInteger valueFromBinaryString = new BigInteger(binaryString , binaryRadix);

Java também suporta conversão direta de bytes para uma instância de BigInteger. Atualmente, apenas a codificação big endian assinada e não assinada pode ser usada:

byte[] bytes = new byte[] { (byte) 0x80 }; 
BigInteger valueFromBytes = new BigInteger(bytes);

Isso gerará uma instância BigInteger com valor -128, pois o primeiro bit é interpretado como o bit de sinal.

byte[] unsignedBytes = new byte[] { (byte) 0x80 };
int sign = 1; // positive
BigInteger valueFromUnsignedBytes = new BigInteger(sign, unsignedBytes);

Isso gerará uma instância BigInteger com valor 128, pois os bytes são interpretados como um número sem sinal e o sinal é explicitamente definido como 1, um número positivo.

Existem constantes predefinidas para valores comuns:

  • BigInteger.ZERO — valor de “0”.
  • BigInteger.ONE — valor de “1”.
  • BigInteger.TEN — valor de “10”.

Há também BigInteger.TWO (valor de “2”), mas você não pode usá-lo em seu código porque é privado.

Exemplos de operações matemáticas BigInteger #

BigInteger está em um objeto imutável, portanto, você precisa atribuir os resultados de qualquer operação matemática a uma nova instância de BigInteger.

Adição: 10 + 10 = 20

BigInteger value1 = new BigInteger("10");
BigInteger value2 = new BigInteger("10");

BigInteger sum = value1.add(value2);
System.out.println(sum);

saída: 20

Subtração: 10 - 9 = 1

BigInteger value1 = new BigInteger("10");
BigInteger value2 = new BigInteger("9");

BigInteger sub = value1.subtract(value2);
System.out.println(sub);

saída: 1

Divisão: 10/5 = 2

BigInteger value1 = new BigInteger("10");
BigInteger value2 = new BigInteger("5");

BigInteger div = value1.divide(value2);
System.out.println(div);

saída: 2

Divisão: 17/4 = 4

BigInteger value1 = new BigInteger("17");
BigInteger value2 = new BigInteger("4");

BigInteger div = value1.divide(value2);
System.out.println(div);

saída: 4

Multiplicação: 105 = 50*

BigInteger value1 = new BigInteger("10");
BigInteger value2 = new BigInteger("5");

BigInteger mul = value1.multiply(value2);
System.out.println(mul);

saída: 50

Poder: 10 ^ 3 = 1000

BigInteger value1 = new BigInteger("10");
BigInteger power = value1.pow(3);
System.out.println(power);

saída: 1000

Restante: 10% 6 = 4

BigInteger value1 = new BigInteger("10");
BigInteger value2 = new BigInteger("6");

BigInteger power = value1.remainder(value2);
System.out.println(power);

saída: 4

GCD: o maior divisor comum (GCD) para 12 e 18 é 6.

BigInteger value1 = new BigInteger("12");
BigInteger value2 = new BigInteger("18");

System.out.println(value1.gcd(value2));

Saída: 6

Máximo de dois BigIntegers:

BigInteger value1 = new BigInteger("10");
BigInteger value2 = new BigInteger("11");

System.out.println(value1.max(value2));

Saída: 11

Mínimo de dois BigIntegers:

BigInteger value1 = new BigInteger("10");
BigInteger value2 = new BigInteger("11");

System.out.println(value1.min(value2));

Saída: 10

Comparando BigIntegers #

Você pode comparar BigIntegers da mesma forma que compara String ou outros objetos em Java.

Por exemplo:

BigInteger one = BigInteger.valueOf(1);
BigInteger two = BigInteger.valueOf(2);

if(one.equals(two)){
    System.out.println("Equal");
}
else{
    System.out.println("Not Equal");
}

Resultado:

Not Equal

Observação:

Em geral, não use o operador == para comparar BigIntegers

  • Operador ==: compara referências; ou seja, se dois valores se referem ao mesmo objeto
  • Método equals(): compara o conteúdo de dois BigIntegers.

Por exemplo, BigIntegers não devem ser comparados da seguinte maneira:

if (firstBigInteger == secondBigInteger) {
  // Only checks for reference equality, not content equality!
}

Fazer isso pode levar a um comportamento inesperado, pois o operador == apenas verifica a igualdade de referência. Se ambos os BigIntegers contiverem o mesmo conteúdo, mas não fizerem referência ao mesmo objeto, isso falhará. Em vez disso, compare BigIntegers usando os métodos equals, conforme explicado acima.

Você também pode comparar seu BigInteger com valores constantes como 0,1,10.

por exemplo:

BigInteger reallyBig = BigInteger.valueOf(1);
if(BigInteger.ONE.equals(reallyBig)){
    //code when they are equal.
}

Você também pode comparar dois BigIntegers usando o método compareTo(), da seguinte forma: compareTo() retorna 3 valores.

  • 0: Quando ambos são iguais.

  • 1: quando o primeiro é maior que o segundo (entre parênteses).

  • -1: quando o primeiro é menor que o segundo.

    BigInteger reallyBig = BigInteger.valueOf(10); BigInteger reallyBig1 = BigInteger.valueOf(100);

    if(reallyBig.compareTo(reallyBig1) == 0){ //code when both are equal. } else if(reallyBig.compareTo(reallyBig1) == 1){ //code when reallyBig is greater than reallyBig1. } else if(reallyBig.compareTo(reallyBig1) == -1){ //code when reallyBig is less than reallyBig1. }

Operações de lógica binária em BigInteger #

BigInteger suporta as operações de lógica binária que também estão disponíveis para tipos Number. Como em todas as operações, elas são implementadas chamando um método.

Binário ou:

BigInteger val1 = new BigInteger("10");
BigInteger val2 = new BigInteger("9");

val1.or(val2);

Saída: 11 (que é equivalente a 10 | 9)

Binário E:

BigInteger val1 = new BigInteger("10");
BigInteger val2 = new BigInteger("9");

val1.and(val2);

Saída: 8 (que é equivalente a 10 & 9)

Xor binário:

BigInteger val1 = new BigInteger("10");
BigInteger val2 = new BigInteger("9");

val1.xor(val2);

Saída: 3 (que é equivalente a 10 ^ 9)

Deslocamento para a direita:

BigInteger val1 = new BigInteger("10");

val1.shiftRight(1);   // the argument be an Integer

Saída: 5 (equivalente a 10 >> 1)

Desvio à esquerda:

BigInteger val1 = new BigInteger("10");

val1.shiftLeft(1);   // here parameter should be Integer

Saída: 20 (equivalente a 10 << 1)

Inversão Binária (Não) :

BigInteger val1 = new BigInteger("10");

val1.not();

Saída: 5

NAND (e-não):*

BigInteger val1 = new BigInteger("10");
BigInteger val2 = new BigInteger("9");

val1.andNot(val2);

Saída: 7

Gerando BigIntegers aleatórios #

A classe BigInteger tem um construtor dedicado a gerar BigIntegers aleatórios, dado uma instância de java.util.Random e um int que especifica quantos bits o BigInteger terá. Seu uso é bastante simples - quando você chama o construtor [BigInteger(int, Random)](https://docs.oracle.com/javase/8/docs/api/java/math/BigInteger.html#BigInteger- int-java.util.Random-) assim:

BigInteger randomBigInt = new BigInteger(bitCount, sourceOfRandomness);

então você terminará com um BigInteger cujo valor está entre 0 (inclusive) e 2bitCount (exclusivo).

Isso também significa que new BigInteger(2147483647, sourceOfRandomness) pode retornar todos os BigInteger positivos com tempo suficiente.

Qual será o sourceOfRandomness depende de você. Por exemplo, um new Random() é bom o suficiente na maioria dos casos:

new BigInteger(32, new Random());

Se estiver disposto a abrir mão da velocidade por números aleatórios de maior qualidade, você pode usar um novo [SecureRandom](https://docs.oracle.com/javase/8/docs/api/java/security/ SecureRandom.html)() em vez disso:

import java.security.SecureRandom;

// somewhere in the code...
new BigInteger(32, new SecureRandom());

Você pode até implementar um algoritmo dinamicamente com uma classe anônima! Observe que lançar seu próprio algoritmo RNG vai acabar com aleatoriedade de baixa qualidade, portanto, sempre use um algoritmo que seja comprovadamente decente, a menos que você queira que os BigInteger (s) resultantes sejam previsíveis.

new BigInteger(32, new Random() {
    int seed = 0;

    @Override
    protected int next(int bits) {
        seed = ((22695477 * seed) + 1) & 2147483647; // Values shamelessly stolen from Wikipedia
        return seed;
    }
});
