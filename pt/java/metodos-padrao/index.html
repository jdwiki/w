Default Method introduzido no Java 8, permite que os desenvolvedores adicionem novos métodos a uma interface sem quebrar as implementações existentes dessa interface. Ele fornece flexibilidade para permitir que a interface defina uma implementação que será usada como padrão quando uma classe que implementa essa interface falhar em fornecer uma implementação desse método.

Sintaxe #

  • public default void methodName() {/* corpo do método */}

Métodos padrão #

  • Pode ser usado dentro de uma interface, para introduzir um comportamento sem forçar as subclasses existentes a implementá-lo.
  • Pode ser substituído por subclasses ou por uma subinterface.
  • Não é permitido sobrescrever métodos na classe java.lang.Object.
  • Se uma classe que implementa mais de uma interface herda métodos padrão com assinaturas de método idênticas de cada uma das interfaces, ela deve substituir e fornecer sua própria interface como se não fossem métodos padrão (como parte da resolução de herança múltipla).
  • Embora tenham a intenção de introduzir um comportamento sem quebrar implementações existentes, as subclasses existentes com um método estático com a mesma assinatura de método que o método padrão recém-introduzido ainda serão quebradas. No entanto, isso é verdade mesmo no caso de introduzir um método de instância em uma superclasse.



Métodos estáticos #

  • Can be used within an interface, primarily intended to be used as a utility method for default methods.
  • Cannot be overridden by subclasses or by a sub-interface (is hidden to them). However as is the case with static methods even now, each class or interface can have its own.
  • Are not allowed to override instance methods in java.lang.Object class (as is presently the case for subclasses as well).

- SUPER_CLASS-INSTANCE-MÉTODO SUPER_CLASS-STATIC-METHOD
SUB_CLASS-INSTANCE-METHOD substitui gera-erro de tempo de compilação
SUB_CLASS-STATIC-METHOD gera-erro de tempo de compilação esconde

Uso básico de métodos padrão #

/**
 * Interface with default method
 */
public interface Printable {
    default void printString() {
        System.out.println( "default implementation" );
    }
}

/**
 * Class which falls back to default implementation of {@link #printString()}
 */
public class WithDefault
    implements Printable
{
}

/**
 * Custom implementation of {@link #printString()}
 */
public class OverrideDefault
    implements Printable {
    @Override
    public void printString() {
        System.out.println( "overridden implementation" );
    }
}

As seguintes declarações

    new WithDefault().printString();
    new OverrideDefault().printString();

Produzirá esta saída:

implementação padrão implementação substituída

Acessando métodos padrão substituídos da classe de implementação #

Em classes, super.foo() irá procurar apenas em superclasses. Se você quiser chamar uma implementação padrão de uma superinterface, você precisa qualificar super com o nome da interface: Fooable.super.foo().

public interface Fooable {
    default int foo() {return 3;}
}

public class A extends Object implements Fooable {
    @Override
    public int foo() {
        //return super.foo() + 1; //error: no method foo() in java.lang.Object
        return Fooable.super.foo() + 1; //okay, returns 4
    }
}

Por que usar métodos padrão? #

A resposta simples é que ele permite que você evolua uma interface existente sem quebrar as implementações existentes.

Por exemplo, você tem a interface Swim que publicou há 20 anos.

public interface Swim {
    void backStroke();
}

Fizemos um ótimo trabalho, nossa interface é muito popular, existem muitas implementações em todo o mundo e você não tem controle sobre o código-fonte.

public class FooSwimmer implements Swim {
    public void backStroke() {
         System.out.println("Do backstroke");
    }
}

Após 20 anos, você decidiu adicionar novas funcionalidades à interface, mas parece que nossa interface está congelada porque quebrará as implementações existentes.

Felizmente, o Java 8 apresenta um novo recurso chamado método padrão.

Agora podemos adicionar um novo método à interface Swim.

public interface Swim {
    void backStroke();
    default void sideStroke() {
        System.out.println("Default sidestroke implementation. Can be overridden");
    }
}

Agora todas as implementações existentes de nossa interface ainda podem funcionar. Mas o mais importante é que eles podem implementar o método recém-adicionado em seu próprio tempo.

Uma das maiores razões para essa mudança, e um de seus maiores usos, está no framework Java Collections. A Oracle não pôde adicionar um método foreach à interface Iterable existente sem quebrar todo o código existente que implementava Iterable. Ao adicionar métodos padrão, a implementação Iterable existente herdará a implementação padrão.

Acessando outros métodos de interface dentro do método padrão #

Você também pode acessar outros métodos de interface de dentro do seu método padrão.

public interface Summable {
    int getA();

    int getB();

    default int calculateSum() {
        return getA() + getB();
    }
}

public class Sum implements Summable {
    @Override
    public int getA() {
        return 1;
    }

    @Override
    public int getB() {
        return 2;
    }
}

A seguinte declaração imprimirá 3:

System.out.println(new Sum().calculateSum());

Métodos padrão também podem ser usados ​​junto com métodos estáticos de interface:

public interface Summable {
    static int getA() {
        return 1;
    }

    static int getB() {
        return 2;
    }

    default int calculateSum() {
        return getA() + getB();
    }
}

public class Sum implements Summable {}

A instrução a seguir também imprimirá 3:

System.out.println(new Sum().calculateSum());

Colisão de herança múltipla do método padrão #

Considere o próximo exemplo:

public interface A {
    default void foo() { System.out.println("A.foo"); }
}

public interface B {
    default void foo() { System.out.println("B.foo"); }
}

Aqui estão duas interfaces declarando o método default foo com a mesma assinatura.

Se você tentar ’estender’ essas duas interfaces na nova interface, você terá que escolher duas, porque Java força você a resolver essa colisão explicitamente.

Primeiro, você pode declarar o método foo com a mesma assinatura que abstract, que substituirá o comportamento A e B.

public interface ABExtendsAbstract extends A, B {
    @Override
    void foo();
}

E quando você vai implementar ABExtendsAbstract na class você terá que fornecer a implementação foo:

public class ABExtendsAbstractImpl implements ABExtendsAbstract {
    @Override
    public void foo() { System.out.println("ABImpl.foo"); }
}

Ou segundo, você pode fornecer uma implementação padrão completamente nova. Você também pode reutilizar o código dos métodos A e B foo por https://www.wikiod.com/pt/java/metodos-padrao#Acessando métodos padrão substituídos da classe de implementação classe.

public interface ABExtends extends A, B {
    @Override
    default void foo() { System.out.println("ABExtends.foo"); }
}

E quando você vai implementar ABExtends na class você não terá que fornecer a implementação foo:

public class ABExtendsImpl implements ABExtends {}

Precedência de classe, classe abstrata e método de interface #

Implementações em classes, incluindo declarações abstratas, têm precedência sobre todos os padrões de interface.

  • O método da classe abstrata tem precedência sobre o Método Padrão da Interface.

    public interface Swim { default void backStroke() { System.out.println(“Swim.backStroke”); } }

    public abstract class AbstractSwimmer implements Swim { public void backStroke() { System.out.println(“AbstractSwimmer.backStroke”); } }

    public class FooSwimmer extends AbstractSwimmer { }

A seguinte declaração

new FooSwimmer().backStroke();

Vai produzir

AbstractSwimmer.backStroke

  • O método de classe tem precedência sobre Método padrão de interface

    public interface Swim { default void backStroke() { System.out.println(“Swim.backStroke”); } }

    public abstract class AbstractSwimmer implements Swim { }

    public class FooSwimmer extends AbstractSwimmer { public void backStroke() { System.out.println(“FooSwimmer.backStroke”); } }

A seguinte declaração

new FooSwimmer().backStroke();

Vai produzir

FooSwimmer.backStroke
