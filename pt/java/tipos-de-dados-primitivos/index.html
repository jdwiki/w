Os 8 tipos de dados primitivos byte, short, int, long, char, boolean, float e double são os tipos que armazenam a maioria dos dados numéricos brutos em programas Java.

Sintaxe #

  • int aInt = 8; // A parte de definição (número) desta declaração int é chamada de literal.

  • int hexInt = 0x1a; // = 26; Você pode definir literais com valores hexadecimais prefixados com 0x.

  • int binInt = 0b11010; // = 26; Você também pode definir literais binários; prefixado com 0b.

  • longo bomLong = 10000000000L; // Por padrão, os literais inteiros são do tipo int. Ao adicionar o L no final do literal, você está dizendo ao compilador que o literal é longo. Sem isso, o compilador lançaria um erro “Número inteiro muito grande”.

  • duplo aDuplo = 3,14; // Os literais de ponto flutuante são do tipo double por padrão.

  • float aFloat = 3,14F; // Por padrão, esse literal teria sido um double (e causou um erro de “Tipos incompatíveis”), mas adicionando um F, informamos ao compilador que é um float.

Java tem 8 tipos de dados primitivos, a saber, boolean, byte, short, char, int, long, float e double. (Todos os outros tipos são tipos reference. Isso inclui todos os tipos de array e tipos/classes de objetos internos que têm significado especial na linguagem Java; por exemplo, String, Class e Throwable e suas subclasses.)

O resultado de todas as operações (adição, subtração, multiplicação, etc) em um tipo primitivo é pelo menos um int, portanto, adicionar um short a um short produz um int, assim como adicionar um byte para um byte, ou um char para um char. Se você quiser atribuir o resultado disso de volta a um valor do mesmo tipo, você deve convertê-lo. por exemplo.

byte a = 1;
byte b = 2;
byte c = (byte) (a + b);

Não converter a operação resultará em um erro de compilação.

Isso se deve à seguinte parte do [Java Language Spec, §2.11.1](http://docs.oracle.com/javase/specs/jvms/se7/html/jvms-2.html#jvms-2.11. 1):

Um compilador codifica cargas de valores literais dos tipos byte e short usando instruções da Java Virtual Machine que estendem esses valores para valores do tipo int em tempo de compilação ou em tempo de execução. Cargas de valores literais dos tipos boolean e char são codificados usando instruções que estendem zero o literal para um valor do tipo int em tempo de compilação ou em tempo de execução. [..]. Assim, a maioria das operações em valores de tipos reais boolean, byte, char e short são executadas corretamente por instruções operando em valores do tipo computacional int.

A razão por trás disso também é especificada nessa seção:

Dado o tamanho de código de operação de um byte da Java Virtual Machine, os tipos de codificação em códigos de operação pressionam o design de seu conjunto de instruções. Se cada instrução digitada suportasse todos os tipos de dados de tempo de execução da Java Virtual Machine, haveria mais instruções do que poderiam ser representadas em um ‘byte’. Instruções separadas podem ser usadas para converter entre tipos de dados não suportados e suportados conforme necessário.

A primitiva char #

Um char pode armazenar um único caractere Unicode de 16 bits. Um literal de caractere é colocado entre aspas simples

char myChar = 'u';
char myChar2 = '5';
char myChar3 = 65; // myChar3 == 'A'

Tem um valor mínimo de \u0000 (0 na representação decimal, também chamado de caractere nulo) e um valor máximo de \uffff (65.535).

O valor padrão de um char é \u0000.

char defaultChar;    // defaultChar == \u0000

Para definir um char de valor ', uma sequência de escape (caractere precedido por uma barra invertida) deve ser usada:

char singleQuote = '\'';

Existem também outras sequências de escape:

char tab = '\t';
char backspace = '\b';
char newline = '\n';
char carriageReturn = '\r';
char formfeed = '\f';
char singleQuote = '\'';
char doubleQuote = '\"'; // escaping redundant here; '"' would be the same; however still allowed
char backslash = '\\';
char unicodeChar = '\uXXXX' // XXXX represents the Unicode-value of the character you want to display

Você pode declarar um char de qualquer caractere Unicode.

char heart = '\u2764';
System.out.println(Character.toString(heart)); // Prints a line containing "❤".

Também é possível adicionar a um char. por exemplo. para percorrer todas as letras minúsculas, você pode fazer o seguinte:

for (int i = 0; i <= 26; i++) {
    char letter = (char) ('a' + i);
    System.out.println(letter);
}

Notas:

  1. A Java Language Specification exige que os tipos integrais assinados (byte a long) usem representação binária de complemento de dois, e os tipos de ponto flutuante usem representações binárias de ponto flutuante IEE 754 padrão.
  2. Java 8 e posteriores fornecem métodos para realizar operações aritméticas sem sinal em int e long. Embora esses métodos permitam que um programa trate valores dos respectivos tipos como não assinados, os tipos permanecem tipos assinados.

  4. Um char representa convencionalmente uma unidade de código Unicode / UTF-16.
  5. Embora um boolean contenha apenas um bit de informação, seu tamanho na memória varia dependendo da implementação da Java Virtual Machine (consulte [boolean type](http://docs.oracle.com/javase/specs/jvms/se8 /html/jvms-2.html#jvms-2.3.4)).

A primitiva flutuante #

Um float é um número de ponto flutuante IEEE 754 de 32 bits de precisão simples. Por padrão, os decimais são interpretados como duplos. Para criar um float, basta anexar um f ao literal decimal.

double doubleExample = 0.5;      // without 'f' after digits = double
float floatExample = 0.5f;       // with 'f' after digits    = float

float myFloat = 92.7f;           // this is a float...
float positiveFloat = 89.3f;     // it can be positive,
float negativeFloat = -89.3f;    // or negative
float integerFloat = 43.0f;      // it can be a whole number (not an int)
float underZeroFloat = 0.0549f;  // it can be a fractional value less than 0

Os flutuadores lidam com as cinco operações aritméticas comuns: adição, subtração, multiplicação, divisão e módulo.

Observação: O seguinte pode variar um pouco como resultado de erros de ponto flutuante. Alguns resultados foram arredondados para fins de clareza e legibilidade (ou seja, o resultado impresso do exemplo de adição foi, na verdade, 34,600002).

// addition
float result = 37.2f + -2.6f;  // result: 34.6

// subtraction
float result = 45.1f - 10.3f;    // result: 34.8

// multiplication
float result = 26.3f * 1.7f;   // result: 44.71

// division
float result = 37.1f / 4.8f;   // result: 7.729166

// modulus
float result = 37.1f % 4.8f;   // result: 3.4999971

Devido à maneira como os números de ponto flutuante são armazenados (ou seja, em forma binária), muitos números não têm uma representação exata.

float notExact = 3.1415926f;
System.out.println(notExact); // 3.1415925

Embora o uso de float seja adequado para a maioria dos aplicativos, nem float nem double devem ser usados ​​para armazenar representações exatas de números decimais (como valores monetários) ou números onde é necessária maior precisão. Em vez disso, a classe BigDecimal deve ser usada.

O valor padrão de um float é 0.0f.

float defaultFloat;    // defaultFloat == 0.0f

Um float tem uma precisão de aproximadamente um erro de 1 em 10 milhões.

Nota: Float.POSITIVE_INFINITY, Float.NEGATIVE_INFINITY, Float.NaN são valores float. NaN significa resultados de operações que não podem ser determinadas, como dividir 2 valores infinitos. Além disso, 0f e -0f são diferentes, mas == produz true:

float f1 = 0f;
float f2 = -0f;
System.out.println(f1 == f2); // true
System.out.println(1f / f1); // Infinity
System.out.println(1f / f2); // -Infinity
System.out.println(Float.POSITIVE_INFINITY / Float.POSITIVE_INFINITY); // NaN

A primitiva int #

Um tipo de dado primitivo como int mantém valores diretamente na variável que o está usando, enquanto uma variável que foi declarada usando Integer mantém uma referência ao valor.

De acordo com java API: “A classe Integer envolve um valor do tipo primitivo int em um objeto. Um objeto do tipo Integer contém um único campo cujo tipo é int.”

Por padrão, int é um inteiro com sinal de 32 bits. Ele pode armazenar um valor mínimo de -231 e um valor máximo de 231 - 1.

int example = -42;
int myInt = 284;
int anotherInt = 73;

int addedInts = myInt + anotherInt; // 284 + 73 = 357
int subtractedInts = myInt - anotherInt; // 284 - 73 = 211

Se você precisar armazenar um número fora desse intervalo, deve-se usar long. Exceder o intervalo de valores de int leva a um estouro de inteiro, fazendo com que o valor que exceda o intervalo seja adicionado ao local oposto do intervalo (positivo se torna negativo e vice-versa). O valor é ((valor - MIN_VALUE) % RANGE) + MIN_VALUE ou ((valor + 2147483648) % 4294967296) - 2147483648

int demo = 2147483647; //maximum positive integer
System.out.println(demo); //prints 2147483647
demo = demo + 1; //leads to an integer overflow
System.out.println(demo); // prints -2147483648

Os valores máximo e mínimo de int podem ser encontrados em:

int high = Integer.MAX_VALUE;    // high == 2147483647
int low = Integer.MIN_VALUE;     // low == -2147483648

O valor padrão de um int é 0

int defaultInt;    // defaultInt == 0

Consumo de memória de primitivos vs. primitivos em caixa #

Primitivo Tipo em caixa Tamanho da memória do primitivo / encaixotado
booleano Booleano 1 byte / 16 bytes
byte Byte 1 byte / 16 bytes
curto Curto 2 bytes / 16 bytes
caractere Caractere 2 bytes / 16 bytes
int Inteiro 4 bytes / 16 bytes
longo Longo 8 bytes / 16 bytes
flutuar Flutuar 4 bytes / 16 bytes
duplo Duplo 8 bytes / 16 bytes

Objetos encaixotados sempre requerem 8 bytes para gerenciamento de tipo e memória, e como o tamanho dos objetos é sempre um múltiplo de 8, os tipos encaixotados todos requerem 16 bytes no total. Além disso, cada uso de um objeto em caixa envolve o armazenamento de uma referência que representa outros 4 ou 8 bytes, dependendo das opções JVM e JVM.

Em operações com uso intenso de dados, o consumo de memória pode ter um grande impacto no desempenho. O consumo de memória cresce ainda mais ao usar arrays: um array float[5] exigirá apenas 32 bytes; enquanto um Float[5] armazenando 5 valores distintos não nulos exigirá 112 bytes no total (em 64 bits sem ponteiros compactados, isso aumenta para 152 bytes).

Caches de valor em caixa #

As despesas gerais de espaço dos tipos em caixa podem ser atenuadas até certo ponto pelos caches de valor em caixa. Alguns dos tipos em caixa implementam um cache de instâncias. Por exemplo, por padrão, a classe Integer armazenará em cache instâncias para representar números no intervalo -128 a +127. No entanto, isso não reduz o custo adicional decorrente da indireção de memória adicional.

Se você criar uma instância de um tipo boxed por autoboxing ou chamando o método estático valueOf(primitive), o sistema de tempo de execução tentará usar um valor em cache. Se seu aplicativo usa muitos valores no intervalo armazenado em cache, isso pode reduzir substancialmente a penalidade de memória do uso de tipos em caixa. Certamente, se você estiver criando instâncias de valor em caixa “manualmente”, é melhor usar valueOf em vez de new. (A operação new sempre cria uma nova instância.) Se, no entanto, a maioria dos seus valores não estiver no intervalo de cache, pode ser mais rápido chamar new e salvar a pesquisa de cache.

Convertendo Primitivos #

Em Java, podemos converter entre valores inteiros e valores de ponto flutuante. Além disso, como cada caractere corresponde a um número na codificação Unicode, os tipos char podem ser convertidos de e para os tipos inteiro e de ponto flutuante. boolean é o único tipo de dados primitivo que não pode ser convertido de ou para qualquer outro tipo de dados primitivo.

Existem dois tipos de conversão: conversão de ampliação e conversão de restrição.

Uma conversão de ampliação ocorre quando um valor de um tipo de dados é convertido em um valor de outro tipo de dados que ocupa mais bits que o anterior. Não há problema de perda de dados neste caso.

Da mesma forma, uma conversão de restrição ocorre quando um valor de um tipo de dados é convertido em um valor de outro tipo de dados que ocupa menos bits que o anterior. A perda de dados pode ocorrer neste caso.

Java executa conversões de ampliação automaticamente. Mas se você quiser realizar uma conversão restrita (se tiver certeza de que nenhuma perda de dados ocorrerá), então você pode forçar o Java a realizar a conversão usando uma construção de linguagem conhecida como cast.

Ampliação da conversão:

int a = 1;    
double d = a;    // valid conversion to double, no cast needed (widening)

Restringir a conversão:

double d = 18.96
int b = d;       // invalid conversion to int, will throw a compile-time error
int b = (int) d; // valid conversion to int, but result is truncated (gets rounded down)
                 // This is type-casting
                 // Now, b = 18

A primitiva dupla #

Um double é um número de ponto flutuante IEEE 754 de 64 bits de precisão dupla.

double example = -7162.37;
double myDouble = 974.21;
double anotherDouble = 658.7;

double addedDoubles = myDouble + anotherDouble; // 315.51
double subtractedDoubles = myDouble - anotherDouble; // 1632.91

double scientificNotationDouble = 1.2e-3;    // 0.0012

Devido à maneira como os números de ponto flutuante são armazenados, muitos números não têm uma representação exata.

double notExact = 1.32 - 0.42; // result should be 0.9
System.out.println(notExact); // 0.9000000000000001

Embora usar double seja bom para a maioria dos aplicativos, nem float nem double devem ser usados ​​para armazenar números precisos, como moeda. Em vez disso, a classe BigDecimal deve ser usada

O valor padrão de um double é 0.0d

public double defaultDouble;    // defaultDouble == 0.0

Nota: Double.POSITIVE_INFINITY, Double.NEGATIVE_INFINITY, Double.NaN são valores double. NaN significa resultados de operações que não podem ser determinadas, como dividir 2 valores infinitos. Além disso, 0d e -0d são diferentes, mas == produz true:

double d1 = 0d;
double d2 = -0d;
System.out.println(d1 == d2); // true
System.out.println(1d / d1); // Infinity
System.out.println(1d / d2); // -Infinity
System.out.println(Double.POSITIVE_INFINITY / Double.POSITIVE_INFINITY); // NaN

O primitivo longo #

Por padrão, long é um inteiro assinado de 64 bits (no Java 8, pode ser assinado ou não). Com sinal, pode armazenar um valor mínimo de -263 e um valor máximo de 263 - 1, e sem sinal, pode armazenar um valor mínimo de 0 e um máximo valor de 264 - 1

long example = -42;
long myLong = 284;
long anotherLong = 73;

//an "L" must be appended to the end of the number, because by default,
//numbers are assumed to be the int type. Appending an "L" makes it a long
//as 549755813888 (2 ^ 39) is larger than the maximum value of an int (2^31 - 1),
//"L" must be appended 
long bigNumber = 549755813888L;

long addedLongs = myLong + anotherLong; // 284 + 73 = 357
long subtractedLongs = myLong - anotherLong; // 284 - 73 = 211

Os valores máximo e mínimo de long podem ser encontrados em:

long high = Long.MAX_VALUE;    // high == 9223372036854775807L
long low = Long.MIN_VALUE;     // low == -9223372036854775808L

O valor padrão de um long é 0L

long defaultLong;    // defaultLong == 0L

Nota: a letra “L” anexada ao final do literal long não diferencia maiúsculas de minúsculas, no entanto, é uma boa prática usar maiúsculas, pois é mais fácil distinguir do dígito um:

2L == 2l;            // true

Aviso: Java armazena em cache instâncias de objetos Integer do intervalo -128 a 127. O raciocínio é explicado aqui: https://blogs.oracle.com/darcy/entry/boxing_and_caches_integer_valueof

Os seguintes resultados podem ser encontrados:

Long val1 = 127L;
Long val2 = 127L;

System.out.println(val1 == val2); // true

Long val3 = 128L;
Long val4 = 128L;

System.out.println(val3 == val4); // false

Para comparar corretamente os valores de 2 objetos longos, use o seguinte código (do Java 1.7 em diante):

Long val3 = 128L;
Long val4 = 128L;

System.out.println(Objects.equal(val3, val4)); // true

Comparar um primitivo long com um Object long não resultará em um falso negativo como a comparação de 2 objetos com ==.

A primitiva booleana #

Um boolean pode armazenar um de dois valores, true ou false

boolean foo = true;
System.out.println("foo = " + foo);                // foo = true

boolean bar = false;
System.out.println("bar = " + bar);                // bar = false

boolean notFoo = !foo;
System.out.println("notFoo = " + notFoo);          // notFoo = false

boolean fooAndBar = foo && bar;
System.out.println("fooAndBar = " + fooAndBar);    // fooAndBar = false

boolean fooOrBar = foo || bar;
System.out.println("fooOrBar = " + fooOrBar);      // fooOrBar = true

boolean fooXorBar = foo ^ bar;
System.out.println("fooXorBar = " + fooXorBar);    // fooXorBar = true

O valor padrão de um boolean é false

boolean defaultBoolean;    // defaultBoolean == false

A primitiva de byte #

Um byte é um inteiro com sinal de 8 bits. Ele pode armazenar um valor mínimo de -27 (-128) e um valor máximo de 27 - 1 (127)

byte example = -36;
byte myByte = 96;
byte anotherByte = 7;

byte addedBytes = (byte) (myByte + anotherByte); // 103
byte subtractedBytes = (byte) (myBytes - anotherByte); // 89

Os valores máximo e mínimo de byte podem ser encontrados em:

byte high = Byte.MAX_VALUE;        // high == 127
byte low = Byte.MIN_VALUE;         // low == -128

O valor padrão de um byte é 0

byte defaultByte;    // defaultByte == 0

Representação de valor negativo #

Java e a maioria das outras linguagens armazenam números inteiros negativos em uma representação chamada notação de complemento de 2.

Para uma representação binária exclusiva de um tipo de dados usando n bits, os valores são codificados assim:

Os bits n-1 menos significativos armazenam um número integral positivo x na representação integral. O valor mais significativo armazena um bit com o valor s. O valor representado por esses bits é

x - s * 2n-1

ou seja, se o bit mais significativo for 1, então um valor que é apenas 1 maior que o número que você poderia representar com os outros bits (2n-2 + 2n- 3 + … + 21 + 20 = 2n-1 - 1) é subtraído permitindo uma representação binária exclusiva para cada valor de - 2n-1 (s = 1; x = 0) a 2n-1 - 1 (s = 0; x = 2n-1 - 1).

Isso também tem o bom efeito colateral, que você pode adicionar as representações binárias como se fossem números binários positivos:

v1 = x1 - s1 * 2n-1
v2 = x2 - s2 * 2n-1
s1 s2 estouro x1 + x2 resultado da adição
0 0 Não x1 + x2 = v1 + v2
0 0 Sim muito grande para ser representado com tipo de dados (estouro)
0 1 Não 
x1 + x2 - 2n-1 = x1 + x2 - s2 * 2n-1
 = v1 + v2
0 1 Sim 
(x1 + x2) mod 2n-1 = x1 + x2 - 2n-1
 = v1 + v2
1 0 * veja acima (swap summands)
1 1 Não muito pequeno para ser representado com o tipo de dados (x1 + x2 - 2n < -2n-1 ; underflow)
1 1 Sim 
(x1 + x2) mod 2n-1 - 2n-1 = (x1 + x2 - 2n -1) - 2n-1
 = (x1 - s1 * 2n-1) + (x2 - s2 * 2 n-1)
 = v1 + v2

O primitivo curto #

Um short é um inteiro com sinal de 16 bits. Tem um valor mínimo de -215 (-32.768) e um valor máximo de 215 ‑1 (32.767)

short example = -48;
short myShort = 987;
short anotherShort = 17;

short addedShorts = (short) (myShort + anotherShort); // 1,004
short subtractedShorts = (short) (myShort - anotherShort); // 970

Os valores máximo e mínimo de short podem ser encontrados em:

short high = Short.MAX_VALUE;        // high == 32767
short low = Short.MIN_VALUE;         // low == -32768

O valor padrão de um short é 0

short defaultShort;    // defaultShort == 0
