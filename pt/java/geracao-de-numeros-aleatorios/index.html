Nada é realmente aleatório e, portanto, o javadoc chama esses números de pseudo-aleatórios. Esses números são criados com um gerador de números pseudoaleatórios.

Números pseudoaleatórios #

Java fornece, como parte do pacote utils, um gerador básico de números pseudo-aleatórios, apropriadamente chamado Random. Este objeto pode ser usado para gerar um valor pseudo-aleatório como qualquer um dos tipos de dados numéricos embutidos (int, float, etc). Você também pode usá-lo para gerar um valor booleano aleatório ou uma matriz aleatória de bytes. Um exemplo de uso é o seguinte:

import java.util.Random;  

...
  
Random random = new Random();
int randInt = random.nextInt();
long randLong = random.nextLong();

double randDouble = random.nextDouble(); //This returns a value between 0.0 and 1.0
float randFloat = random.nextFloat(); //Same as nextDouble

byte[] randBytes = new byte[16];
random.nextBytes(randBytes); //nextBytes takes a user-supplied byte array, and fills it with random bytes. It returns nothing.

NOTA: Esta classe produz apenas números pseudo-aleatórios de baixa qualidade e nunca deve ser usada para gerar números aleatórios para operações criptográficas ou outras situações em que a aleatoriedade de alta qualidade é crítica (para isso, você deve usar o método SecureRandom classe, como indicado abaixo). Uma explicação para a distinção entre aleatoriedade “segura” e “insegura” está além do escopo deste exemplo.

Números pseudo-aleatórios em intervalo específico #

O método nextInt(int bound) de Random aceita um limite superior exclusivo, ou seja, um número que o valor aleatório retornado deve ser menor que. No entanto, apenas o método nextInt aceita um limite; nextLong, nextDouble etc. não.

Random random = new Random();
random.nextInt(1000); // 0 - 999

int number = 10 + random.nextInt(100); // number is in the range of 10 to 109

A partir do Java 1.7, você também pode usar ThreadLocalRandom (source). Esta classe fornece um PRNG thread-safe (gerador de números pseudo-aleatórios). Observe que o método nextInt desta classe aceita um limite superior e inferior.

import java.util.concurrent.ThreadLocalRandom;

// nextInt is normally exclusive of the top value,
// so add 1 to make it inclusive
ThreadLocalRandom.current().nextInt(min, max + 1);

Observe que a documentação oficial afirma que nextInt(int bound) pode fazer coisas estranhas quando bound está próximo de 230+1 (grifo nosso):

O algoritmo é um pouco complicado. Rejeita valores que resultariam em uma distribuição desigual (devido ao fato de 2^31 não ser divisível por n). A probabilidade de um valor ser rejeitado depende de n. O pior caso é n=2^30+1, para o qual a probabilidade de uma rejeição é 1/2 e o número esperado de iterações antes que o loop termine é 2.

Em outras palavras, especificar um limite diminuirá (levemente) o desempenho do método nextInt, e essa diminuição de desempenho se tornará mais pronunciada à medida que o bound se aproximar da metade do valor máximo de int.

Gerando números pseudoaleatórios criptograficamente seguros #

Random e ThreadLocalRandom são bons o suficiente para o uso diário, mas eles têm um grande problema: eles são baseados em um gerador linear congruential, um algoritmo cujo a saída pode ser prevista com bastante facilidade. Assim, essas duas classes não são adequadas para usos criptográficos (como geração de chaves).

Pode-se usar java.security.SecureRandom em situações onde é necessário um PRNG com uma saída muito difícil de prever. Prever os números aleatórios criados por instâncias dessa classe é difícil o suficiente para rotular a classe como criptograficamente segura.

import java.security.SecureRandom;
import java.util.Arrays;

public class Foo {
    public static void main(String[] args) {
        SecureRandom rng = new SecureRandom();
        byte[] randomBytes = new byte[64];
        rng.nextBytes(randomBytes); // Fills randomBytes with random bytes (duh)
        System.out.println(Arrays.toString(randomBytes));
    }
}

Além de ser criptograficamente seguro, o SecureRandom tem um período gigantesco de 2160, comparado ao período Random de 248. Ele tem uma desvantagem de ser consideravelmente mais lento que Random e outros PRNGs lineares, como Mersenne Twister e [Xorshift](https://en.wikipedia. org/wiki/Xorshift), no entanto.

Observe que a implementação do SecureRandom depende da plataforma e do provedor. O padrão SecureRandom (fornecido pelo provedor SUN em sun.security.provider.SecureRandom):

  • em sistemas do tipo Unix, semeados com dados de /dev/random e/ou /dev/urandom.
  • no Windows, semeado com chamadas para CryptGenRandom() em CryptoAPI.

Gerando números aleatórios com uma semente especificada #

//Creates a Random instance with a seed of 12345.
Random random = new Random(12345L);

//Gets a ThreadLocalRandom instance
ThreadLocalRandom tlr = ThreadLocalRandom.current();

//Set the instance's seed.
tlr.setSeed(12345L);

Usar a mesma semente para gerar números aleatórios retornará os mesmos números todas as vezes, portanto, definir uma semente diferente para cada instância Random é uma boa ideia se você não quiser acabar com números duplicados.

Um bom método para obter um Long diferente para cada chamada é System.currentTimeMillis():

Random random = new Random(System.currentTimeMillis());
ThreadLocalRandom.current().setSeed(System.currentTimeMillis());

Selecione números aleatórios sem duplicatas #

/**
 * returns a array of random numbers with no duplicates
 * @param range the range of possible numbers for ex. if 100 then it can be anywhere from 1-100
 * @param length the length of the array of random numbers
 * @return array of random numbers with no duplicates.
 */
public static int[] getRandomNumbersWithNoDuplicates(int range, int length){
    if (length<range){
        // this is where all the random numbers
        int[] randomNumbers = new int[length];
        
        // loop through all the random numbers to set them
        for (int q = 0; q < randomNumbers.length; q++){
            
            // get the remaining possible numbers
            int remainingNumbers = range-q;
            
            // get a new random number from the remainingNumbers
            int newRandSpot = (int) (Math.random()*remainingNumbers);
            
            newRandSpot++;
            
            // loop through all the possible numbers
            for (int t = 1; t < range+1; t++){
                
                // check to see if this number has already been taken
                boolean taken = false;
                for (int number : randomNumbers){
                    if (t==number){
                        taken = true;
                        break;
                    }
                }
                
                // if it hasnt been taken then remove one from the spots
                if (!taken){
                    newRandSpot--;
                    
                    // if we have gone though all the spots then set the value
                    if (newRandSpot==0){
                        randomNumbers[q] = t;
                    }
                }
            }
        }
        return randomNumbers;
    } else {
        // invalid can't have a length larger then the range of possible numbers
    }
    return null;
}

O método funciona fazendo um loop em uma matriz que tem o tamanho do comprimento solicitado e encontra o comprimento restante dos números possíveis. Ele define um número aleatório desses números possíveis newRandSpot e encontra esse número dentro do número não tomado à esquerda. Ele faz isso percorrendo o intervalo e verificando se esse número já foi obtido.

Por exemplo, se o intervalo for 5 e o comprimento for 3 e já tivermos escolhido o número 2. Então temos 4 números restantes para obter um número aleatório entre 1 e 4 e percorrer o intervalo(5) pulando qualquer número que já usamos(2).

Agora vamos dizer que o próximo número escolhido entre 1 e 4 é 3. No primeiro loop, obtemos 1 que ainda não foi obtido, então podemos remover 1 de 3, tornando-o 2. Agora, no segundo loop, obtemos 2 que foi obtido então não fazemos nada. Seguimos esse padrão até chegarmos a 4, onde uma vez que removemos 1, ele se torna 0, então definimos o novo randomNumber como 4.

Gerando números aleatórios usando apache-common lang3 #

Podemos usar org.apache.commons.lang3.RandomUtils para gerar números aleatórios usando uma única linha.

int x = RandomUtils.nextInt(1, 1000);

O método nextInt(int startInclusive, int endExclusive) usa um intervalo.

Além de int, podemos gerar long, double, float e bytes aleatórios usando esta classe.

A classe RandomUtils contém os seguintes métodos-

static byte[] nextBytes(int count) //Creates an array of random bytes.
static double nextDouble() //Returns a random double within 0 - Double.MAX_VALUE
static double nextDouble(double startInclusive, double endInclusive) //Returns a random double within the specified range.
static float nextFloat() //Returns a random float within 0 - Float.MAX_VALUE
static float nextFloat(float startInclusive, float endInclusive) //Returns a random float within the specified range.
static int nextInt() //Returns a random int within 0 - Integer.MAX_VALUE
static int nextInt(int startInclusive, int endExclusive) //Returns a random integer within the specified range.
static long nextLong() //Returns a random long within 0 - Long.MAX_VALUE
static long nextLong(long startInclusive, long endExclusive) //Returns a random long within the specified range.
