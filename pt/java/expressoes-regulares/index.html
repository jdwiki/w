Uma expressão regular é uma sequência especial de caracteres que ajuda a encontrar ou encontrar outras strings ou conjuntos de strings, usando uma sintaxe especializada mantida em um padrão. Java tem suporte para uso de expressões regulares através do pacote java.util.regex. Este tópico apresenta e ajuda os desenvolvedores a entender mais com exemplos de como as Expressões Regulares devem ser usadas em Java.

Sintaxe #

  • Padrão patternName = Pattern.compile(regex);
  • Matcher matcherName = patternName.matcher(textToSearch);
  • matcherName.matches() //Retorna true se textToSearch corresponder exatamente ao regex
  • matcherName.find() //Pesquisa em textToSearch pela primeira instância de uma substring que corresponda à regex. As chamadas subsequentes pesquisarão o restante da String.
  • matcherName.group(groupNum) //Retorna a substring dentro de um grupo de captura
  • matcherName.group(groupName) //Retorna a substring dentro de um grupo de captura nomeado (Java 7+)

Importações #

Você precisará adicionar as seguintes importações antes de usar o Regex:

import java.util.regex.Matcher
import java.util.regex.Pattern

Armadilhas #

Em java, uma barra invertida é escapada com uma barra invertida dupla, portanto, uma barra invertida na string regex deve ser inserida como uma barra invertida dupla. Se você precisar escapar de uma barra invertida dupla (para corresponder uma única barra invertida com a regex, você precisa inseri-la como uma barra invertida quádrupla.

Símbolos importantes explicados #

Personagem Descrição
* Corresponder ao caractere ou subexpressão anterior 0 ou mais vezes
+ Corresponder ao caractere ou subexpressão anterior 1 ou mais vezes
? Corresponder ao caractere ou subexpressão anterior 0 ou 1 vezes

Leitura adicional #

O tópico regex contém mais informações sobre expressões regulares.

Usando grupos de captura #

Se você precisar extrair uma parte da string da string de entrada, podemos usar capture groups de regex.

Para este exemplo, começaremos com um simples regex de número de telefone:

\d{3}-\d{3}-\d{4}

Se parênteses forem adicionados à regex, cada conjunto de parênteses será considerado um grupo de captura. Nesse caso, estamos usando os chamados grupos de captura numerados:

(\d{3})-(\d{3})-(\d{4})
^-----^ ^-----^ ^-----^
Group 1 Group 2 Group 3

Antes de podermos usá-lo em Java, não devemos esquecer de seguir as regras de Strings, escapando das barras invertidas, resultando no seguinte padrão:

"(\\d{3})-(\\d{3})-(\\d{4})"

Primeiro, precisamos compilar o padrão regex para criar um Pattern e, em seguida, precisamos de um Matcher para combinar nossa string de entrada com o padrão:

Pattern phonePattern = Pattern.compile("(\\d{3})-(\\d{3})-(\\d{4})");
Matcher phoneMatcher = phonePattern.matcher("abcd800-555-1234wxyz");

Em seguida, o Matcher precisa encontrar a primeira subsequência que corresponde à regex:

phoneMatcher.find();

Agora, usando o método group, podemos extrair os dados da string:

String number = phoneMatcher.group(0); //"800-555-1234" (Group 0 is everything the regex matched)
String aCode = phoneMatcher.group(1); //"800"
String threeDigit = phoneMatcher.group(2); //"555"
String fourDigit = phoneMatcher.group(3); //"1234"

Nota: Matcher.group() pode ser usado no lugar de Matcher.group(0).

O Java 7 introduziu grupos de captura nomeados. Os grupos de captura nomeados funcionam da mesma forma que os grupos de captura numerados (mas com um nome em vez de um número), embora haja pequenas alterações de sintaxe. O uso de grupos de captura nomeados melhora a legibilidade.

Podemos alterar o código acima para usar grupos nomeados:

(?<AreaCode>\d{3})-(\d{3})-(\d{4})
^----------------^ ^-----^ ^-----^
AreaCode           Group 2 Group 3

Para obter o conteúdo de “AreaCode”, podemos usar:

String aCode = phoneMatcher.group("AreaCode"); //"800"

Usando regex com comportamento personalizado compilando o padrão com sinalizadores #

Um Pattern pode ser compilado com sinalizadores, se o regex for usado como um String literal, use modificadores embutidos:

Pattern pattern = Pattern.compile("foo.", Pattern.CASE_INSENSITIVE | Pattern.DOTALL);
pattern.matcher("FOO\n").matches(); // Is true.

/* Had the regex not been compiled case insensitively and singlelined,
 * it would fail because FOO does not match /foo/ and \n (newline)
 * does not match /./.
 */

Pattern anotherPattern = Pattern.compile("(?si)foo");
anotherPattern.matcher("FOO\n").matches(); // Is true.

"foOt".replaceAll("(?si)foo", "ca"); // Returns "cat".

Caracteres de escape #

Geralmente

Para usar caracteres específicos de expressão regular (?+| etc.) em seu significado literal, eles precisam ser escapados. Na expressão regular comum, isso é feito por uma barra invertida \. No entanto, como tem um significado especial em Java Strings, você deve usar uma barra invertida dupla \\.

Estes dois exemplos não funcionarão:

"???".replaceAll ("?", "!"); //java.util.regex.PatternSyntaxException
"???".replaceAll ("\?", "!"); //Invalid escape sequence

Este exemplo funciona

"???".replaceAll ("\\?", "!"); //"!!!"

Dividindo uma string delimitada por pipe

Isso não retorna o resultado esperado:

"a|b".split ("|"); // [a, |, b]

Isso retorna o resultado esperado:

"a|b".split ("\\|"); // [a, b]

Escape de barra invertida \

Isso dará um erro:

"\\".matches("\\"); // PatternSyntaxException
"\\".matches("\\\"); // Syntax Error

Isso funciona:

"\\".matches("\\\\"); // true

Não corresponde a uma determinada string #

Para corresponder a algo que não contém uma determinada string, pode-se usar lookahead negativo:

Sintaxe Regex: (?!string-to-not-match)

Exemplo:

//not matching "popcorn"
String regexString = "^(?!popcorn).*$";
System.out.println("[popcorn] " + ("popcorn".matches(regexString) ? "matched!" : "nope!"));
System.out.println("[unicorn] " + ("unicorn".matches(regexString) ? "matched!" : "nope!"));

Resultado:

[popcorn] nope!
[unicorn] matched!

Correspondência com um literal regex. #

Se você precisar corresponder caracteres que fazem parte da sintaxe de expressão regular, poderá marcar todo ou parte do padrão como um literal regex.

\Q marca o início do literal regex. \E marca o fim do literal regex.

// the following throws a PatternSyntaxException because of the un-closed bracket
"[123".matches("[123");

// wrapping the bracket in \Q and \E allows the pattern to match as you would expect.
"[123".matches("\\Q[\\E123"); // returns true

Uma maneira mais fácil de fazer isso sem ter que lembrar das sequências de escape \Q e \E é usar Pattern.quote()

"[123".matches(Pattern.quote("[") + "123"); // returns true

Correspondendo a uma barra invertida #

Se você quiser combinar uma barra invertida em sua expressão regular, você terá que escapar dela.

A barra invertida é um caractere de escape em expressões regulares. Você pode usar ‘\\’ para se referir a uma única barra invertida em uma expressão regular.

No entanto, a barra invertida também é um caractere de escape em strings literais Java. Para fazer uma expressão regular a partir de uma string literal, você precisa escapar de cada uma das suas barras invertidas. Em um literal de string ‘\\\\’ pode ser usado para criar uma expressão regular com ‘\\’, que por sua vez pode corresponder a ‘\’.

Por exemplo, considere strings correspondentes como “C:\dir\myfile.txt”. Uma expressão regular ([A-Za-z]):\\(.*) corresponderá e fornecerá a letra da unidade como um grupo de captura. Observe a barra invertida dupla.

Para expressar esse padrão em um literal de string Java, cada uma das barras invertidas na expressão regular precisa ser escapada.

    String path = "C:\\dir\\myfile.txt";
    System.out.println( "Local path: " + path ); // "C:\dir\myfile.txt"
    
    String regex = "([A-Za-z]):\\\\.*"; // Four to match one
    System.out.println("Regex:      " + regex ); // "([A-Za-z]):\\(.*)"
    
    Pattern pattern = Pattern.compile( regex );
    Matcher matcher = pattern.matcher( path );
    if ( matcher.matches()) {
        System.out.println( "This path is on drive " + matcher.group( 1 ) + ":.");
        // This path is on drive C:.
    }

Se você quiser combinar duas barras invertidas, você se verá usando oito em uma string literal, para representar quatro na expressão regular, para corresponder a dois.

    String path = "\\\\myhost\\share\\myfile.txt";
    System.out.println( "UNC path: " + path ); // \\myhost\share\myfile.txt"
    
    String regex = "\\\\\\\\(.+?)\\\\(.*)"; // Eight to match two
    System.out.println("Regex:    " + regex ); // \\\\(.+?)\\(.*) 
    
    Pattern pattern = Pattern.compile( regex );
    Matcher matcher = pattern.matcher( path );
    
    if ( matcher.matches()) {
        System.out.println( "This path is on host '" + matcher.group( 1 ) + "'.");
        // This path is on host 'myhost'.
    }
