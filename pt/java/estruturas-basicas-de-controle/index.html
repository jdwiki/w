Todas as estruturas de controle, salvo indicação em contrário, usam instruções de bloco. Eles são indicados por chaves {}.

Isso difere de instruções normais, que não exigem chaves, mas também vem com uma ressalva rígida em que apenas a linha imediatamente após a declaração anterior seria considerada.

Assim, é perfeitamente válido escrever qualquer uma dessas estruturas de controle sem chaves, desde que apenas uma instrução siga o início, mas é fortemente desencorajada, pois pode levar a implementações de bugs, ou código quebrado.

Exemplo:

// valid, but discouraged
Scanner scan = new Scanner(System.in);
int val = scan.nextInt();
if(val % 2 == 0)
    System.out.println("Val was even!");


// invalid; will not compile
// note the misleading indentation here
for(int i = 0; i < 10; i++)
    System.out.println(i);
    System.out.println("i is currently: " + i);

faça… while Loop #

O loop do...while difere de outros loops porque é garantido que será executado pelo menos uma vez. Também é chamada de estrutura “loop pós-teste” porque a instrução condicional é executada após o corpo do loop principal.

int i = 0;
do {
    i++;
    System.out.println(i);
} while (i < 100); // Condition gets checked AFTER the content of the loop executes.

Neste exemplo, o loop será executado até que o número 100 seja impresso (mesmo que a condição seja i < 100 e não i <= 100), porque a condição do loop é avaliada após a execução do loop.

Com a garantia de pelo menos uma execução, é possível declarar variáveis ​​fora do loop e inicializá-las dentro.

String theWord;
Scanner scan = new Scanner(System.in);
do {
    theWord = scan.nextLine();
} while (!theWord.equals("Bird"));

System.out.println(theWord);

Neste contexto, theWord é definido fora do loop, mas como é garantido que tem um valor baseado em seu fluxo natural, theWord será inicializado.

Para cada #

Com Java 5 e superior, pode-se usar loops for-each, também conhecidos como loops for aprimorados:

List strings = new ArrayList();
        
strings.add("Isso");
strings.add("é");
strings.add("um loop for-each");
        
for (String string: strings) {
    System.out.println(string);
}

Para cada loop pode ser usado para iterar sobre Arrays e implementações da interface Iterable, a última inclui classes Collections, como List ou Set.

A variável de loop pode ser de qualquer tipo atribuível a partir do tipo de origem.

A variável de loop para um loop for aprimorado para Iterable<T> ou T[] pode ser do tipo S, se

  • T estende S
  • ambos T e S são tipos primitivos e atribuíveis sem uma conversão
  • S é um tipo primitivo e T pode ser convertido em um tipo atribuível a S após a conversão de unboxing.
  • T é um tipo primitivo e pode ser convertido em S por conversão de autoboxing.

Exemplos:

T elements = ...
for (S s : elements) {
}
T S Compila
int[] longo sim
longo[] int não
Iterável<Byte> longo sim
Iterable<String> CharSequence sim
Iterable<CharSequence> Cadeia não
int[] Longo não
int[] Inteiro sim

Instrução de troca #

A instrução switch é a instrução branch multi-way do Java. Ele é usado para substituir as cadeias longas if-else if-else e ​​torná-las mais legíveis. Entretanto, diferentemente das declarações if, não se pode usar inequações; cada valor deve ser definido concretamente.

Existem três componentes críticos para a instrução switch:

  • case: Este é o valor que é avaliado para equivalência com o argumento para a instrução switch.
  • default: Esta é uma expressão opcional e abrangente, caso nenhuma das instruções case seja avaliada como true.
  • Conclusão abrupta da instrução case; normalmente break: Isso é necessário para evitar a avaliação indesejada de outras instruções case.

Com exceção de continue, é possível usar qualquer instrução que cause a [conclusão abrupta de uma instrução](https://docs.oracle.com/javase/specs/jls/se7/html/jls-14 .html#jls-14.1). Isso inclui:

  • quebrar
  • retornar
  • lançar

No exemplo abaixo, uma declaração típica switch é escrita com quatro casos possíveis, incluindo default.

Scanner scan = new Scanner(System.in);
int i = scan.nextInt();
switch (i) {
    case 0:
        System.out.println("i is zero");
        break;
    case 1:
        System.out.println("i is one");
        break;
    case 2:
        System.out.println("i is two");
        break;
    default:
        System.out.println("i is less than zero or greater than two");
}

Ao omitir break ou qualquer declaração que teria uma conclusão abrupta, podemos alavancar o que é conhecido como casos “fall-through”, que são avaliados em relação a vários valores. Isso pode ser usado para criar intervalos para um valor ser bem-sucedido, mas ainda não é tão flexível quanto as desigualdades.

Scanner scan = new Scanner(System.in);
int foo = scan.nextInt();
switch(foo) {
    case 1:
        System.out.println("I'm equal or greater than one");
    case 2:
    case 3:    
        System.out.println("I'm one, two, or three");
        break;
    default:
        System.out.println("I'm not either one, two, or three");
}

No caso de foo == 1 a saída será:

I'm equal or greater than one
I'm one, two, or three

No caso de foo == 3 a saída será:

I'm one, two, or three

A instrução switch também pode ser usada com enums.

enum Option {
    BLUE_PILL,
    RED_PILL
}

public void takeOne(Option option) {
    switch(option) {
        case BLUE_PILL:
            System.out.println("Story ends, wake up, believe whatever you want.");
            break;
        case RED_PILL:
            System.out.println("I show you how deep the rabbit hole goes.");
            break;
            
    }
}

A instrução switch também pode ser usada com Strings.

public void rhymingGame(String phrase) {
    switch (phrase) {
        case "apples and pears":
            System.out.println("Stairs");
            break;
        case "lorry":
            System.out.println("truck");
            break;
        default:
            System.out.println("Don't know any more");
    }
}

Continuar declaração em Java #

A instrução continue é usada para pular as etapas restantes na iteração atual e comece com a próxima iteração de loop. O controle vai da instrução continue para o valor do passo (incremento ou decremento), se houver.

String[] programmers = {"Adrian", "Paul", "John", "Harry"};

    //john is not printed out
    for (String name : programmers) {
        if (name.equals("John"))
            continue;
        System.out.println(name);
    }

A instrução continue também pode fazer o controle do programa mudar para o valor do passo (se houver) de um loop nomeado:

Outer: // The name of the outermost loop is kept here as 'Outer'
for(int i = 0; i < 5; )
{
    for(int j = 0; j < 5; j++)
    {
        continue Outer;
    }
}

Se / Senão Se / Senão Controle #

if (i < 2) {
  System.out.println("i is less than 2");
} else if (i > 2) {
  System.out.println("i is more than 2");
} else {
  System.out.println("i is not less than 2, and not more than 2");
}

O bloco if só será executado quando i for 1 ou menos.

A condição else if é verificada apenas se todas as condições anteriores a ela (nas construções else if anteriores e nas construções pai if) foram testadas para false. Neste exemplo, a condição else if só será verificada se i for maior ou igual a 2.

Se o resultado for true, seu bloco será executado, e qualquer construção else if e else depois dele será ignorada.

Se nenhuma das condições if e else if foram testadas para true, o bloco else no final será executado.

Para Loops #

for (int i = 0; i < 100; i++) {
    System.out.println(i);
}

Os três componentes do loop for (separados por ;) são a declaração/inicialização da variável (aqui int i = 0), a condição (aqui i < 100) e a instrução de incremento (aqui i++ ). A declaração da variável é feita uma vez como se fosse colocada dentro do { na primeira execução. Em seguida, a condição é verificada, se for true o corpo do loop será executado, se for false o loop será interrompido. Assumindo que o loop continua, o corpo será executado e, finalmente, quando o } for alcançado, a instrução de incremento será executada antes que a condição seja verificada novamente.

As chaves são opcionais (você pode uma linha com um ponto e vírgula) se o loop contiver apenas uma instrução. Mas, é sempre recomendável usar chaves para evitar mal-entendidos e bugs.

Os componentes do loop for são opcionais. Se sua lógica de negócios contiver uma dessas partes, você poderá omitir o componente correspondente do seu loop for.

int i = obj.getLastestValue(); // i value is fetched from a method
    
for (; i < 100; i++) { // here initialization is not done
    System.out.println(i);
}

A estrutura for (;;) { function-body } é igual a um loop while (true).

Aninhado para Loops

Qualquer instrução de loop que tenha outra instrução de loop dentro chamada de loop aninhado. A mesma maneira de fazer um loop com mais loop interno é chamada de ’nested for loop'.

    for(;;){
        //Outer Loop Statements
        for(;;){
            //Inner Loop Statements
        }
        //Outer Loop Statements
    }

O loop for aninhado pode ser demonstrado para imprimir números em forma de triângulo.

for(int i=9;i>0;i--){//Outer Loop
    System.out.println();
    for(int k=i;k>0;k--){//Inner Loop -1
        System.out.print(" ");
    }
    for(int j=i;j<=9;j++){//Inner Loop -2
        System.out.print(" "+j);
    }
 }

Operador Ternário #

Às vezes, você precisa verificar uma condição e definir o valor de uma variável.

Por ex.

String name;

if (A > B) {
    name = "Billy";
} else {
    name = "Jimmy";
}

Isso pode ser facilmente escrito em uma linha como

String name = A > B ? "Billy" : "Jimmy";

O valor da variável é definido como o valor imediatamente após a condição, se a condição for verdadeira. Se a condição for falsa, o segundo valor será dado à variável.

Enquanto Loops #

int i = 0;
while (i < 100) { // condition gets checked BEFORE the loop body executes
    System.out.println(i);
    i++;
}

Um loop while é executado enquanto a condição dentro dos parênteses for true. Isso também é chamado de estrutura “loop de pré-teste” porque a instrução condicional deve ser atendida antes que o corpo do loop principal seja executado todas as vezes.

As chaves são opcionais se o loop contiver apenas uma instrução, mas algumas convenções de estilo de codificação preferem ter as chaves independentemente.

Se / Senão #

int i = 2;
if (i < 2) {
  System.out.println("i is less than 2");
} else {
  System.out.println("i is greater than 2");
}

Uma instrução if executa o código condicionalmente dependendo do resultado da condição entre parênteses. Quando a condição entre parênteses for verdadeira, ela entrará no bloco da instrução if que é definida por chaves como { e }. colchete de abertura até que o colchete de fechamento seja o escopo da instrução if.

O bloco else é opcional e pode ser omitido. Ele é executado se a instrução if for false e ​​não será executada se a instrução if for verdadeira Porque nesse caso a instrução if é executada.

Parar #

A instrução break encerra um loop (como for, while) ou a avaliação de uma instrução switch.

Ciclo:

while(true) {
   if(someCondition == 5) {
       break;
   }
}

O loop no exemplo seria executado para sempre. Mas quando someCondition é igual a 5 em algum ponto da execução, o loop termina.

Se vários loops estiverem em cascata, apenas o loop mais interno termina usando break.

Tente… Pegue… Finalmente #

A estrutura de controle try { ... } catch ( ... ) { ... } é usada para lidar com Exceções.

String age_input = "abc";
try {
    int age = Integer.parseInt(age_input);
    if (age >= 18) {
        System.out.println("You can vote!");
    } else {
        System.out.println("Sorry, you can't vote yet.");
    }
} catch (NumberFormatException ex) {
    System.err.println("Invalid input.  '" + age_input + "' is not a valid integer.");
}

Isso imprimiria:

Entrada inválida. ‘abc’ não é um inteiro válido.

Uma cláusula finally pode ser adicionada após o catch. A cláusula finally sempre seria executada, independentemente de uma exceção ter sido lançada.

try { ... } catch ( ... ) { ... } finalmente { ... }

String age_input = "abc";
try {
    int age = Integer.parseInt(age_input);
    if (age >= 18) {
        System.out.println("You can vote!");
    } else {
        System.out.println("Sorry, you can't vote yet.");
    }
} catch (NumberFormatException ex) {
    System.err.println("Invalid input.  '" + age_input + "' is not a valid integer.");
} finally {
    System.out.println("This code will always be run, even if an exception is thrown");
}

Isso imprimiria:

Entrada inválida. ‘abc’ não é um inteiro válido. Este código sempre será executado, mesmo se uma exceção for lançada

Quebra/continua aninhada #

É possível break / continue para um loop externo usando instruções de rótulo:

outerloop:
for(...) {
    innerloop:
    for(...) {
        if(condition1)
            break outerloop;

        if(condition2)
            continue innerloop; // equivalent to: continue;
    }
}

Não há outro uso para rótulos em Java.
