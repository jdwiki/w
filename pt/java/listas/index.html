Uma list é uma coleção ordered de valores. Em Java, as listas fazem parte do Java Collections Framework. As listas implementam a interface java.util.List, que estende java.util.Collection.

Sintaxe #

  • ls.add(E elemento); // Adiciona um elemento
  • ls.remove(E elemento); //Remove um elemento
  • for(E element : ls){} //Itera sobre cada elemento
  • ls.toArray(new String[ls.length]); //Converte uma lista de Strings em um array de Strings
  • ls.get(int índice); //Retorna o elemento no índice especificado.
  • ls.set(int índice, elemento E); //Substitui o elemento em uma posição especificada.
  • ls.isVazio(); //Retorna true se o array não contiver elementos, caso contrário, retorna false.
  • ls.indexOf(Objeto o); //Retorna o índice da primeira localização do elemento especificado o, ou, se não estiver presente, retorna -1.
  • ls.lastIndexOf(Objeto o); //Retorna o índice da última localização do elemento especificado o, ou, se não estiver presente, retorna -1.
  • ls.size(); //Retorna o número de elementos na Lista.

Uma lista é um objeto que armazena uma coleção ordenada de valores. “Ordenado” significa que os valores são armazenados em uma ordem específica - um item vem primeiro, outro vem em segundo lugar e assim por diante. Os valores individuais são comumente chamados de “elementos”. As listas Java normalmente fornecem estes recursos:

  • As listas podem conter zero ou mais elementos.
  • As listas podem conter valores duplicados. Em outras palavras, um elemento pode ser inserido em uma lista mais de uma vez.
  • As listas armazenam seus elementos em uma ordem específica, o que significa que um elemento vem primeiro, outro vem depois e assim por diante.
  • Cada elemento possui um index indicando sua posição na lista. O primeiro elemento tem índice 0, o próximo tem índice 1 e assim por diante.
  • As listas permitem inserir elementos no início, no final ou em qualquer índice da lista.
  • Testar se uma lista contém um valor específico geralmente significa examinar cada elemento na lista. Isso significa que o tempo para realizar essa verificação é O(n), proporcional ao tamanho da lista.

Adicionar um valor a uma lista em algum ponto diferente do final moverá todos os elementos a seguir “para baixo” ou “para a direita”. Em outras palavras, adicionar um elemento no índice n move o elemento que costumava estar no índice n para o índice n+1 e assim por diante. Por exemplo:

List<String> list = new ArrayList<>();
list.add("world");
System.out.println(list.indexOf("world"));      // Prints "0"
// Inserting a new value at index 0 moves "world" to index 1
list.add(0, "Hello");
System.out.println(list.indexOf("world"));      // Prints "1"
System.out.println(list.indexOf("Hello"));      // Prints "0"

Classificando uma lista genérica #

A classe Collections oferece dois métodos estáticos padrão para classificar uma lista:

  • sort(List<T> list) aplicável a listas onde T estende Comparable<? super T>, e
  • sort(List<T> list, Comparator<? super T> c) aplicável a listas de qualquer tipo.

A aplicação do primeiro requer a alteração da classe de elementos da lista que está sendo classificada, o que nem sempre é possível. Também pode ser indesejável, pois embora forneça a classificação padrão, outras ordens de classificação podem ser necessárias em circunstâncias diferentes ou a classificação é apenas uma tarefa única.

Considere que temos uma tarefa de ordenar objetos que são instâncias da seguinte classe:

public class User {
    public final Long id;
    public final String username;

    public User(Long id, String username) {
        this.id = id;
        this.username = username;
    }

    @Override
    public String toString() {
        return String.format("%s:%d", username, id);
    }
}

Para usar Collections.sort(List<User> list) precisamos modificar a classe User para implementar a interface Comparable. Por exemplo

public class User implements Comparable<User> {
    public final Long id;
    public final String username;

    public User(Long id, String username) {
        this.id = id;
        this.username = username;
    }

    @Override
    public String toString() {
        return String.format("%s:%d", username, id);
    }

    @Override
    /** The natural ordering for 'User' objects is by the 'id' field. */
    public int compareTo(User o) {
        return id.compareTo(o.id);
    }
}

(Além disso: muitas classes Java padrão como String, Long, Integer implementam a interface Comparable. Isso torna as listas desses elementos classificáveis ​​por padrão e simplifica a implementação de compare ou compareTo em outros Aulas.)

Com a modificação acima, podemos facilmente ordenar uma lista de objetos User com base nas classes ordenação natural. (Neste caso, definimos que a ordenação é baseada em valores id). Por exemplo:

List<User> users = Lists.newArrayList(
    new User(33L, "A"),
    new User(25L, "B"),
    new User(28L, ""));
Collections.sort(users);

System.out.print(users);
// [B:25, C:28, A:33]

No entanto, suponha que queiramos classificar objetos User por name em vez de por id. Alternativamente, suponha que não fomos capazes de alterar a classe para torná-la implementar Comparable.

É aqui que o método sort com o argumento Comparator é útil:

Collections.sort(users, new Comparator<User>() {
    @Override
    /* Order two 'User' objects based on their names. */
    public int compare(User left, User right) {
        return left.username.compareTo(right.username);
    }
});
System.out.print(users);
// [A:33, B:25, C:28]

No Java 8 você pode usar um lambda em vez de uma classe anônima. Este último se reduz a um one-liner:

Collections.sort(users, (l, r) -> l.username.compareTo(r.username));

Além disso, o Java 8 adiciona um método padrão sort na interface List, o que simplifica ainda mais a classificação.

users.sort((l, r) -> l.username.compareTo(r.username))

Converte uma lista de inteiros em uma lista de strings #

List<Integer> nums = Arrays.asList(1, 2, 3);
List<String> strings = nums.stream()
    .map(Object::toString)
    .collect(Collectors.toList());

Lista de Matriz #

public class ArrayList<E>
extends AbstractList<E>
implements List<E>, RandomAccess, Cloneable, Serializable

ArrayList é uma implementação de matriz redimensionável da interface List. Armazenando a lista em um array, ArrayList fornece métodos (além dos métodos que implementam a interface List) para manipular o tamanho do array.

Inicialize ArrayList de Integer com tamanho 100

List<Integer> myList = new ArrayList<Integer>(100); // Constructs an empty list with the specified initial capacity.

- PRÓS:

As operações size, isEmpty, get, set, iterator e listIterator são executadas em tempo constante. Portanto, obter e definir cada elemento da Lista tem o mesmo custo de tempo:

int e1 = myList.get(0);  //   \
int e2 = myList.get(10); //    | => All the same constant cost => O(1)
myList.set(2,10);        //   /

- CONTRAS:

Ser implementado com um array (estrutura estática) adicionando elementos sobre o tamanho do array tem um custo alto devido ao fato de que uma nova alocação precisa ser feita para todo o array. No entanto, de documentação:

A operação de adição é executada em tempo constante amortizado, ou seja, somando n elementos requerem tempo O(n)

A remoção de um elemento requer tempo O(n).

Encontrando elementos comuns entre 2 listas #

Suponha que você tenha duas listas: A e B, e precise encontrar os elementos que existem em ambas as listas.

Você pode fazer isso invocando o método List.retainAll().

Exemplo:

public static void main(String[] args) {
    List<Integer> numbersA = new ArrayList<>();
    List<Integer> numbersB = new ArrayList<>();
    numbersA.addAll(Arrays.asList(new Integer[] { 1, 3, 4, 7, 5, 2 }));
    numbersB.addAll(Arrays.asList(new Integer[] { 13, 32, 533, 3, 4, 2 }));

    System.out.println("A: " + numbersA);
    System.out.println("B: " + numbersB);
    List<Integer> numbersC = new ArrayList<>();
    numbersC.addAll(numbersA);
    numbersC.retainAll(numbersB);

    System.out.println("List A : " + numbersA);
    System.out.println("List B : " + numbersB);
    System.out.println("Common elements between A and B: " + numbersC);

}

Criando uma lista #

Dando um tipo à sua lista

Para criar uma lista, você precisa de um tipo (qualquer classe, por exemplo, String). Este é o tipo de sua Lista. A List só armazenará objetos do tipo especificado. Por exemplo:

List<String> strings;

Pode armazenar "string1", "hello world!", "goodbye", etc, mas não pode armazenar 9.2, no entanto:

List<Double> doubles;

Pode armazenar 9.2, mas não "hello world!".

Iniciando sua lista

Se você tentar adicionar algo às listas acima, receberá um NullPointerException, porque strings e doubles são iguais a null!

Existem duas maneiras de inicializar uma lista:

Opção 1: use uma classe que implemente List

List é uma interface, o que significa que não tem um construtor, mas métodos que uma classe deve sobrescrever. ArrayList é a List mais comumente usada, embora LinkedList também seja comum. Então inicializamos nossa lista assim:

List<String> strings = new ArrayList<String>();

ou

List<String> strings = new LinkedList<String>();

A partir do Java SE 7, você pode usar um operador de diamante:

List<String> strings = new ArrayList<>();

ou

List<String> strings = new LinkedList<>();

Opção 2: use a classe Coleções

A classe Collections fornece dois métodos úteis para criar Listas sem uma variável List:

  • emptyList(): retorna uma lista vazia.
  • singletonList(T): cria uma lista do tipo T e adiciona o elemento especificado.

E um método que usa uma List existente para preencher os dados:

  • addAll(L, T...): adiciona todos os elementos especificados à lista passada como primeiro parâmetro.

Exemplos:

import java.util.List;
import java.util.Collections;

List<Integer> l = Collections.emptyList();
List<Integer> l1 = Collections.singletonList(42);
Collections.addAll(l1, 1, 2, 3);

Operações de Acesso Posicional #

A API List tem oito métodos para operações de acesso posicional:

  • adicionar (tipo T)
  • add(int index, tipo T)
  • remove(Objeto o)
  • remove(int index)
  • get(int index)
  • set(int index, elemento E)
  • int indexOf(Object o)
  • int lastIndexOf(Object o)

Então, se tivermos uma lista:

List<String> strings = new ArrayList<String>();

E queríamos adicionar as strings “Hello world!” e “Adeus mundo!” para isso, faríamos assim:

strings.add("Hello world!");
strings.add("Goodbye world!");

E nossa lista conteria os dois elementos. Agora vamos dizer que queremos adicionar “Program starting!” na frente da lista. Faríamos assim:

strings.add(0, "Program starting!");

OBSERVAÇÃO: o primeiro elemento é 0.

Agora, se quiséssemos remover o “Adeus mundo!” linha, poderíamos fazer assim:

strings.remove("Goodbye world!");

E se quiséssemos remover a primeira linha (que neste caso seria “Program startup!”, poderíamos fazer assim:

strings.remove(0);

Observação:

  1. Adicionar e remover elementos de lista modifica a lista, e isso pode levar a uma ConcurrentModificationException se a lista estiver sendo iterada simultaneamente.

  2. Adicionar e remover elementos pode ser O(1) ou O(N) dependendo da classe da lista, do método usado e se você está adicionando/removendo um elemento no início, no fim ou no meio da lista.

Para recuperar um elemento da lista em uma posição especificada, você pode usar o método E get(int index); da API List. Por exemplo:

strings.get(0);

retornará o primeiro elemento da lista.

Você pode substituir qualquer elemento em uma posição especificada usando o set(int index, E element);. Por exemplo:

strings.set(0,"This is a replacement");

Isso definirá a String “Esta é uma substituição” como o primeiro elemento da lista.

Observação: The set method will overwrite the element at the position 0. It will not add the new String at the position 0 and push the old one to the position 1.

O int indexOf(Object o); retorna a posição da primeira ocorrência do objeto passado como argumento. Se não houver ocorrências do objeto na lista, o valor -1 será retornado. Na continuação do exemplo anterior, se você chamar:

strings.indexOf("This is a replacement")

espera-se que o 0 seja retornado quando definimos a String “Esta é uma substituição” na posição 0 de nossa lista. Caso haja mais de uma ocorrência na lista quando int indexOf(Object o); for chamado então como mencionado o índice da primeira ocorrência será retornado. Chamando o int lastIndexOf(Object o) você pode recuperar o índice da última ocorrência na lista. Então, se adicionarmos outro “Isto é uma substituição”:

strings.add("This is a replacement");
strings.lastIndexOf("This is a replacement");

Desta vez o 1 será retornado e não o 0;

Iterando sobre elementos em uma lista #

Para o exemplo, digamos que temos uma Lista do tipo String que contém quatro elementos: “hello”, “, “how”, “are”, “you?”

A melhor maneira de iterar sobre cada elemento é usando um loop for-each:

public void printEachElement(List<String> list){
    for(String s : list){
        System.out.println(s);
    }
}

Que imprimiria:

hello,
how
are
you?

Para imprimir todos na mesma linha, você pode usar um StringBuilder:

public void printAsLine(List<String> list){
    StringBuilder builder = new StringBuilder();
    for(String s : list){
        builder.append(s);
    }
    System.out.println(builder.toString());
}

Irá imprimir:

hello, how are you?

Como alternativa, você pode usar a indexação de elemento (conforme descrito em https://www.wikiod.com/pt/java/listas ) para iterar uma lista. Aviso: essa abordagem é ineficiente para listas vinculadas.

Criando, Adicionando e Removendo Elemento de um ArrayList #

ArrayList é uma das estruturas de dados embutidas em Java. É um array dinâmico (onde o tamanho da estrutura de dados não precisa ser declarado primeiro) para armazenar elementos (Objetos).

Ele estende a classe AbstractList e implementa a interface List. Um ArrayList pode conter elementos duplicados onde mantém a ordem de inserção. Deve-se notar que a classe ArrayList não é sincronizada, então cuidado deve ser tomado ao lidar com simultaneidade com ArrayList. ArrayList permite acesso aleatório porque o array funciona na base do índice. A manipulação é lenta em ArrayList devido ao deslocamento que geralmente ocorre quando um elemento é removido da lista de arrays.

Um ArrayList pode ser criado da seguinte forma:

List<T> myArrayList = new ArrayList<>();

Onde T ( https://www.wikiod.com/pt/java/genericos ) é o tipo que será armazenado dentro de ArrayList.

O tipo de ArrayList pode ser qualquer Object. O tipo não pode ser um tipo primitivo (use suas classes wrapper).

Para adicionar um elemento ao ArrayList, use o método add():

myArrayList.add(element);

Ou para adicionar um item a um determinado índice:

myArrayList.add(index, element); //index of the element should be an int (starting from 0)

Para remover um item da ArrayList, use o método remove():

myArrayList.remove(element);

Ou para remover um item de um determinado índice:

myArrayList.remove(index); //index of the element should be an int (starting from 0)

Substituição no local de um elemento List #

Este exemplo trata da substituição de um elemento List, garantindo que o elemento substituto esteja na mesma posição que o elemento substituído.

Isso pode ser feito usando estes métodos:

  • set(índice int, tipo T)
  • int indexOf(T tipo)

Considere um ArrayList contendo os elementos “Program starting!”, “Hello world!” e “Adeus mundo!”

List<String> strings = new ArrayList<String>();
strings.add("Program starting!");
strings.add("Hello world!");
strings.add("Goodbye world!");

Se soubermos o índice do elemento que queremos substituir, podemos simplesmente usar set da seguinte forma:

strings.set(1, "Hi world");

Se não conhecermos o índice, podemos procurá-lo primeiro. Por exemplo:

int pos = strings.indexOf("Goodbye world!");
if (pos >= 0) {
    strings.set(pos, "Goodbye cruel world!");
}

Notas:

  1. A operação set não causará uma ConcurrentModificationException.
  2. A operação set é rápida ( O(1) ) para ArrayList mas lenta ( O(N) ) para uma LinkedList.
  3. Uma pesquisa indexOf em uma ArrayList ou LinkedList é lenta ( O(N)).

Tornando uma lista não modificável #

A classe Collections fornece uma maneira de tornar uma lista não modificável:

List<String> ls = new ArrayList<String>();
List<String> unmodifiableList = Collections.unmodifiableList(ls);

Se você quiser uma lista não modificável com um item, você pode usar:

List<String> unmodifiableList = Collections.singletonList("Only string in the list");

Movendo objetos na lista #

A classe Collections permite mover objetos na lista usando vários métodos (ls é a Lista):

Invertendo uma lista:

Collections.reverse(ls);

Posições rotativas de elementos em uma lista

O método de rotação requer um argumento inteiro. Este é o número de pontos para movê-lo ao longo da linha. Um exemplo disso está abaixo:

List<String> ls = new ArrayList<String>();
ls.add(" how");
ls.add(" are");
ls.add(" you?");
ls.add("hello,");
Collections.rotate(ls, 1);

for(String line : ls) System.out.print(line);
System.out.println();

Isto irá imprimir “Olá, como você está?”

Collections.shuffle(ls);

Random random = new Random(12); 
Collections.shuffle(ls, random);
