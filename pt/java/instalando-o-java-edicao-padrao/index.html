Esta página de documentação dá acesso a instruções para instalação do java standard edition em computadores Windows, Linux e macOS.

Configurando %PATH% e %JAVA_HOME% após a instalação no Windows #

Suposições: #

  • Um Oracle JDK foi instalado.
  • O JDK foi instalado no diretório padrão.

Etapas de configuração #

  1. Abra o Windows Explorer.
  2. No painel de navegação à esquerda, clique com o botão direito em Este PC (ou Computador para versões mais antigas do Windows). Existe uma maneira mais curta sem usar o explorador nas versões reais do Windows: Basta pressionar Win+Pausar
  3. Na janela do Painel de Controle recém-aberta, clique com o botão esquerdo em Configurações Avançadas do Sistema que deve estar no canto superior esquerdo. Isso abrirá a janela Propriedades do Sistema.

Alternativamente, digite SystemPropertiesAdvanced (não diferencia maiúsculas de minúsculas) no Run (Win+R), e pressione Enter.

  1. Na guia Avançado de Propriedades do Sistema selecione o botão Variáveis ​​de Ambiente… no canto inferior direito da janela.
  2. Adicione uma Nova Variável de Sistema clicando no botão Nova… em Variáveis ​​de Sistema com o nome JAVA_HOME e cujo valor é o caminho para o diretório onde o JDK estava instalado. Depois de inserir esses valores, pressione OK.

  1. Role para baixo na lista de Variáveis ​​do Sistema e selecione a variável Caminho.

  2. CUIDADO: O Windows depende do Path para encontrar programas importantes. Se algum ou todo ele for removido, o Windows pode não funcionar corretamente. Ele deve ser modificado para permitir que o Windows execute o JDK. Com isso em mente, clique no botão “Editar…” com a variável Path selecionada. Adicione %JAVA_HOME%\bin; ao início da variável Path.

É melhor anexar no início da linha porque o software da Oracle costumava registrar sua própria versão do Java em Path - Isso fará com que sua versão seja ignorada se ocorrer após a declaração da Oracle.


Verifique seu trabalho #

  1. Abra o prompt de comando clicando em Iniciar, digitando cmd e pressionando Enter.
  2. Digite javac -version no prompt. Se for bem-sucedido, a versão do JDK será impressa na tela.

Observação: se você precisar tentar novamente, feche o prompt antes de verificar seu trabalho. Isso forçará o Windows a obter a nova versão do Path.

Instalando um Java JDK no Linux #

Usando o Gerenciador de Pacotes #

As versões JDK e/ou JRE para OpenJDK ou Oracle podem ser instaladas usando o gerenciador de pacotes na maioria das distribuições Linux convencionais. (As opções disponíveis para você dependerão da distribuição.)

Como regra geral, o procedimento é abrir a janela do terminal e executar os comandos mostrados abaixo. (Supõe-se que você tenha acesso suficiente para executar comandos como o usuário “root” … que é o que o comando sudo faz. Se você não tiver, fale com os administradores do seu sistema.)

O uso do gerenciador de pacotes é recomendado porque (geralmente) torna mais fácil manter sua instalação Java atualizada.

apt-get, distribuições Linux baseadas em Debian (Ubuntu, etc)

As instruções a seguir instalarão o Oracle Java 8:

$ sudo add-apt-repository ppa:webupd8team/java
$ sudo apt-get update
$ sudo apt-get install oracle-java8-installer

Observação: para configurar automaticamente as variáveis ​​de ambiente do Java 8, você pode instalar o seguinte pacote:

$ sudo apt-get install oracle-java8-set-default

Criando um arquivo .deb

Se você preferir criar o arquivo .deb você mesmo a partir do arquivo .tar.gz baixado do Oracle, faça o seguinte (assumindo que você baixou o arquivo .tar.gz para ./<jdk>.tar .gz):

$ sudo apt-get install java-package # might not be available in default repos
$ make-jpkg ./<jdk>.tar.gz          # should not be run as root
$ sudo dpkg -i *j2sdk*.deb

Nota: Isso espera que a entrada seja fornecida como um arquivo “.tar.gz”.

Quando o Java é instalado, a versão instalada é definida automaticamente como padrão. Para alternar entre diferentes, use:

As seguintes funções podem ser adicionadas ao ~/.bash_profile (se você usar o shell Bash padrão) para facilitar o uso:

function java_version {
   echo 'java -version';
}

function java_set {
   if [[ $1 == "6" ]]
   then
       export JAVA_HOME='/usr/libexec/java_home -v 1.6';
       echo "Setting Java to version 6..."
       echo "$JAVA_HOME"
   elif [[ $1 == "7" ]]
   then
       export JAVA_HOME='/usr/libexec/java_home -v 1.7';
       echo "Setting Java to version 7..."
       echo "$JAVA_HOME"
   elif [[ $1 == "8" ]]
   then
       export JAVA_HOME='/usr/libexec/java_home -v 1.8';
       echo "Setting Java to version 8..."
       echo "$JAVA_HOME"
   fi            
}

Instalação e funcionamento com versões portáteis

Há casos em que você pode querer instalar o JDK/JRE em um sistema com privilégios limitados como uma VM ou pode querer instalar e usar várias versões ou arquiteturas (x64/x86) do JDK/JRE. As etapas permanecem as mesmas até o momento em que você baixa o instalador (.EXE). As etapas a seguir são as seguintes (as etapas são aplicáveis ​​para JDK/JRE 7 e superior, para versões mais antigas elas são ligeiramente diferentes nos nomes de pastas e arquivos):

  1. Mova o arquivo para um local apropriado onde você deseja que seus binários Java residam permanentemente.

  2. Instale o 7-Zip ou sua versão portátil se você tiver privilégios limitados.

  3. Com o 7-Zip, extraia os arquivos do EXE do instalador Java para o local.

  4. Abra o prompt de comando segurando ‘Shift’ e ‘Clique com o botão direito’ na pasta no explorer ou navegue para esse local de qualquer lugar.

  5. Navegue até a pasta recém-criada. Digamos que o nome da pasta seja jdk-7u25-windows-x64. Então digite cd jdk-7u25-windows-x64. Em seguida, digite os seguintes comandos em ordem:

    cd .rsrc\JAVA_CAB10

    extrac32 111

  6. Isso criará um arquivo tools.zip nesse local. Extraia o tools.zip com o 7-Zip para que os arquivos dentro dele sejam criados em tools no mesmo diretório.

  7. Agora execute estes comandos no prompt de comando já aberto:

    cd tools

    for /r %x in (*.pack) do .\bin\unpack200 -r "%x" "%~dx%~px%~nx.jar"

  8. Aguarde a conclusão do comando. Copie o conteúdo de tools para o local onde você deseja que seus binários estejam.

Dessa forma, você pode instalar qualquer versão do JDK/JRE que precise ser instalada simultaneamente.

Selecionando uma versão apropriada do Java SE #

Houve muitos lançamentos do Java desde o lançamento original do Java 1.0 em 1995. (Consulte Histórico da versão Java para obter um resumo). Isso significa que o fornecedor (normalmente Oracle agora) cessou o novo desenvolvimento para o lançamento e não fornece mais patches públicos/gratuitos para quaisquer bugs ou problemas de segurança. (Os lançamentos de patches privados geralmente estão disponíveis para pessoas/organizações com um contrato de suporte; entre em contato com o escritório de vendas do seu fornecedor.)

Em geral, a versão Java SE recomendada para uso será a atualização mais recente para a versão pública mais recente. Atualmente, isso significa a versão mais recente do Java 8 disponível. O Java 9 deve ser lançado ao público em 2017. (O Java 7 passou do fim da vida útil e o último lançamento público foi em abril de 2015. O Java 7 e versões anteriores não são recomendados.)

Esta recomendação se aplica a todos os novos desenvolvimentos Java e a qualquer pessoa que esteja aprendendo Java. Também se aplica a pessoas que desejam apenas executar software Java fornecido por terceiros. De um modo geral, um código Java bem escrito funcionará em uma versão mais recente do Java. (Mas verifique as notas de lançamento do software e entre em contato com o autor/fornecedor/fornecedor se tiver dúvidas.)

Se você estiver trabalhando em uma base de código Java mais antiga, é recomendável garantir que seu código seja executado na versão mais recente do Java. Decidir quando começar a usar os recursos das versões Java mais recentes é mais difícil, pois isso afetará sua capacidade de oferecer suporte a clientes que não podem ou não desejam sua instalação Java.

Versão Java e nomenclatura de versão #

A nomenclatura da versão Java é um pouco confusa. Na verdade, existem dois sistemas de nomenclatura e numeração, conforme mostrado nesta tabela:

Versão JDK Nome de marketing
jdk-1.0 JDK 1.0
jdk-1.1 JDK 1.1
jdk-1.2 J2SE 1.2
… …
jdk-1.5 J2SE 1.5 renomeado Java SE 5
jdk-1.6 Java SE 6
jdk-1.7 Java SE 7
jdk-1.8 Java SE 8
jdk-91 Java SE 9 (ainda não lançado)

1 - Parece que a Oracle pretende romper com sua prática anterior de usar um esquema de “número de versão semântica” nas strings de versão Java. Resta saber se eles vão continuar com isso.

O “SE” nos nomes de marketing refere-se à Standard Edition. Esta é a versão base para executar o Java na maioria dos laptops, PCs e servidores (além do Android).

Existem duas outras edições oficiais do Java: “Java ME” é a Micro Edition e “Java EE” é a Enterprise Edition. O sabor Android do Java também é significativamente diferente do Java SE. Java ME, Java EE e Android Java estão fora do escopo deste Tópico.

O número completo da versão de uma versão Java é assim:

1.8.0_101-b13

Isso diz JDK 1.8.0, Atualização 101, Build #13. A Oracle se refere a isso nas notas de lançamento como:

Java™ SE Development Kit 8, Update 101 (JDK 8u101)

O número de atualização é importante – a Oracle regularmente emite atualizações para uma versão principal com patches de segurança, correções de bugs e (em alguns casos) novos recursos. O número de compilação geralmente é irrelevante. Observe que Java 8 e Java 1.8 referem-se à mesma coisa; Java 8 é apenas o nome de “marketing” para Java 1.8.

O que eu preciso para o desenvolvimento Java #

Uma instalação JDK e um editor de texto são o mínimo para o desenvolvimento Java. (É bom ter um editor de texto que possa fazer realce de sintaxe Java, mas você pode ficar sem.)

No entanto, para um trabalho de desenvolvimento sério, é recomendável que você também use o seguinte:

  • Um IDE Java como Eclipse, Intellij IDEA ou NetBeans
  • Uma ferramenta de construção Java, como Ant, Gradle ou Maven
  • Um sistema de controle de versão para gerenciar sua base de código (com backups apropriados e replicação externa)
  • Ferramentas de teste e ferramentas de CI (integração contínua)

Configurando e alternando versões do Java no Linux usando alternativas #

Usando alternativas #

Muitas distribuições Linux usam o comando alternatives para alternar entre diferentes versões de um comando. Você pode usar isso para alternar entre diferentes versões do Java instaladas em uma máquina.

  1. Em um shell de comando, defina $JDK para o nome do caminho de um JDK recém-instalado; por exemplo.

     $ JDK=/Data/jdk1.8.0_67

  2. Use alternatives --install para adicionar os comandos no Java SDK às alternativas:

     $ sudo alternatives --install /usr/bin/java java $JDK/bin/java 2
 $ sudo alternatives --install /usr/bin/javac javac $JDK/bin/javac 2
 $ sudo alternatives --install /usr/bin/jar jar $JDK/bin/jar 2

    And so on.

Agora você pode alternar entre diferentes versões de um comando Java da seguinte maneira:

$ sudo alternatives --config javac

There is 1 program that provides 'javac'.

  Selection    Command
-----------------------------------------------
*+ 1           /usr/lib/jvm/java-1.8.0-openjdk-1.8.0.101-1.b14.fc23.x86_64/bin/javac
   2           /Data/jdk1.8.0_67/bin/javac

Enter to keep the current selection[+], or type selection number: 2
$

Para mais informações sobre o uso de alternatives, consulte a entrada manual alternatives(8).

Verificação e configuração pós-instalação no Linux #

Depois de instalar um Java SDK, é aconselhável verificar se ele está pronto para uso. Você pode fazer isso executando estes dois comandos, usando sua conta de usuário normal:

$ java -version
$ javac -version

Esses comandos imprimem as informações de versão do JRE e do JDK (respectivamente) que estão no caminho de pesquisa de comando do shell. Procure a string de versão JDK/JRE.

  • Se um dos comandos acima falhar, dizendo “comando não encontrado”, o JRE ou JDK não está no caminho de pesquisa; vá para Configurando o PATH diretamente abaixo.
  • Se qualquer um dos comandos acima exibir uma string de versão diferente da que você esperava, seu caminho de pesquisa ou o sistema de “alternativas” precisam ser ajustados; vá para Verificando alternativas
  • Se as strings de versão corretas forem exibidas, você está quase pronto; pule para Verificando JAVA_HOME

Configurando o PATH diretamente

Se não houver java ou javac no caminho de pesquisa no momento, a solução simples é adicioná-lo ao seu caminho de pesquisa.

Primeiro, encontre onde você instalou o Java; consulte Onde o Java foi instalado? abaixo se tiver dúvidas.

Em seguida, supondo que bash seja seu shell de comando, use um editor de texto para adicionar as seguintes linhas ao final de ~/.bash_profile ou ~/.bashrc (se você usar Bash como seu shell).

JAVA_HOME=<installation directory>
PATH=$JAVA_HOME/bin:$PATH

export JAVA_HOME
export PATH

… substituindo <diretório de instalação> pelo nome do caminho para seu diretório de instalação Java. Observe que o acima assume que o diretório de instalação contém um diretório bin, e o diretório bin contém os comandos java e javac que você está tentando usar.

Em seguida, gere o arquivo que você acabou de editar, para que as variáveis ​​de ambiente do seu shell atual sejam atualizadas.

$ source ~/.bash_profile

Em seguida, repita as verificações de versão java e javac. Se ainda houver problemas, use which java e which javac para verificar se você atualizou as variáveis ​​de ambiente corretamente.

Por fim, faça logout e login novamente para que as variáveis ​​de ambiente atualizadas sejam pagas para todos os seus shells. Agora você deve terminar.

Verificando alternativas

Se java -version ou javac -version funcionaram, mas forneceram um número de versão inesperado, você precisa verificar de onde os comandos estão vindo. Use which e ls -l para descobrir isso da seguinte forma:

$ ls -l `which java`

Se a saída for assim, :

lrwxrwxrwx. 1 root root 22 Jul 30 22:18 /usr/bin/java -> /etc/alternatives/java

então a mudança de versão alternatives está sendo usada. Você precisa decidir se continua a usá-lo ou simplesmente substitui-lo definindo o PATH diretamente.

  • Configurando e alternando versões do Java no Linux usando alternativas
  • Consulte “Configurando o PATH diretamente” acima.

Onde o Java foi instalado?

Java pode ser instalado em vários lugares, dependendo do método de instalação.

  • Os RPMs Oracle colocam a instalação do Java em “/usr/java”.
  • No Fedora, o local padrão é “/usr/lib/jvm”.
  • Se o Java foi instalado manualmente a partir de arquivos ZIP ou JAR, a instalação pode ser em qualquer lugar.

Se você está tendo dificuldade em encontrar o diretório de instalação, sugerimos que você use find (ou slocate) para encontrar o comando. Por exemplo:

$ find / -name java -type f 2> /dev/null

Isso fornece os nomes de caminho para todos os arquivos chamados java em seu sistema. (O redirecionamento do erro padrão para “/dev/null” suprime mensagens sobre arquivos e diretórios que você não pode acessar.)
