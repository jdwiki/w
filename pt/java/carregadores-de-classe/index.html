Um carregador de classe é uma classe cujo objetivo principal é mediar a localização e o carregamento de classes usadas por um aplicativo. Um carregador de classes também pode localizar e carregar recursos.

As classes padrão do carregador de classe podem carregar classes e recursos de diretórios no sistema de arquivos e de arquivos JAR e ZIP. Eles também podem baixar e armazenar em cache arquivos JAR e ZIP de um servidor remoto.

Os carregadores de classe normalmente são encadeados, de modo que a JVM tentará carregar classes das bibliotecas de classes padrão em preferência às fontes fornecidas pelo aplicativo. Os carregadores de classe personalizados permitem que o programador altere isso. O também pode fazer coisas como descriptografar arquivos de bytecode e modificação de bytecode.

Implementando um classLoader personalizado #

Cada carregador personalizado deve estender direta ou indiretamente a classe java.lang.ClassLoader. Os principais pontos de extensão são os seguintes métodos:

  • findClass(String) - sobrecarregue este método se seu carregador de classe seguir o modelo de delegação padrão para carregamento de classe.
  • loadClass(String, boolean) - sobrecarrega este método para implementar um modelo de delegação alternativo.
  • findResource e findResources - sobrecarregam esses métodos para personalizar o carregamento de recursos.

Os métodos defineClass que são responsáveis ​​por realmente carregar a classe de um array de bytes são finais para evitar sobrecarga. Qualquer comportamento personalizado precisa ser executado antes de chamar defineClass.

Aqui está um simples que carrega uma classe específica de uma matriz de bytes:

public class ByteArrayClassLoader extends ClassLoader {
    private String classname;
    private byte[] classfile;

    public ByteArrayClassLoader(String classname, byte[] classfile) {
        this.classname = classname;
        this.classfile = classfile.clone();
    }

    @Override
    protected Class findClass(String classname) throws ClassNotFoundException {
        if (classname.equals(this.classname)) {
            return defineClass(classname, classfile, 0, classfile.length);
        } else {
            throw new ClassNotFoundException(classname);
        }
    }
}

Uma vez que apenas sobrescrevemos o método findClass, este carregador de classes personalizado irá se comportar da seguinte forma quando loadClass for chamado.

  1. O método loadClass do carregador de classe chama findLoadedClass para ver se uma classe com este nome já foi carregada por este carregador de classe. Se isso for bem-sucedido, o objeto Class resultante será retornado ao solicitante.
  2. O método loadClass então delega ao carregador de classe pai chamando sua chamada loadClass. Se o pai puder lidar com a solicitação, ele retornará um objeto Class que será então retornado ao solicitante.
  3. Se o carregador de classe pai não puder carregar a classe, findClass chama nosso método de substituição findClass, passando o nome da classe a ser carregada.
  4. Se o nome solicitado corresponder a this.classname, chamamos defineClass para carregar a classe real do array de bytes this.classfile. O objeto Class resultante é então retornado.
  5. Se o nome não corresponder, lançamos ClassNotFoundException.

Carregando um arquivo .class externo #

Para carregar uma classe, primeiro precisamos defini-la. A classe é definida pelo ClassLoader. Há apenas um problema, a Oracle não escreveu o código do ClassLoader com este recurso disponível. Para definir a classe precisamos acessar um método chamado defineClass() que é um método privado do ClassLoader.

Para acessá-lo, o que faremos é criar uma nova classe, ByteClassLoader, e estendê-la para ClassLoader. Agora que estendemos nossa classe para ClassLoader, podemos acessar os métodos privados do ClassLoader. Para disponibilizar o defineClass(), criaremos um novo método que funcionará como um espelho para o método privado defineClass(). Para chamar o método privado, precisaremos do nome da classe, name, os bytes da classe, classBytes, o deslocamento do primeiro byte, que será 0 porque os dados classBytes’ começam em classBytes[0], e o deslocamento do último byte, que será classBytes.lenght porque representa o tamanho dos dados, que será o último deslocamento.

public class ByteClassLoader extends ClassLoader {

    public Class<?> defineClass(String name, byte[] classBytes) {
        return defineClass(name, classBytes, 0, classBytes.length);
    }

}

Agora, temos um método público defineClass(). Ele pode ser chamado passando o nome da classe e os bytes da classe como argumentos.

Digamos que temos uma classe chamada MyClass no pacote stackoverflow…

Para chamar o método, precisamos dos bytes da classe, então criamos um objeto Path representando o caminho da nossa classe’ usando o método Paths.get() e passando o caminho da classe binária como argumento. Agora, podemos obter os bytes da classe com Files.readAllBytes(path). Então criamos uma instância ByteClassLoader e usamos o método que criamos, defineClass(). Já temos os bytes da classe, mas para chamar nosso método também precisamos do nome da classe que é dado pelo nome do pacote (ponto) o nome canônico da classe, neste caso stackoverflow.MyClass.

Path path = Paths.get("MyClass.class");

ByteClassLoader loader = new ByteClassLoader();
loader.defineClass("stackoverflow.MyClass", Files.readAllBytes(path);

Observação: O método defineClass() retorna um objeto Class<?>. Você pode salvá-lo se quiser.

Para carregar a classe, basta chamar loadClass() e passar o nome da classe. Este método pode lançar um ClassNotFoundException, então precisamos usar um bloco try cath

try{
    loader.loadClass("stackoverflow.MyClass");
} catch(ClassNotFoundException e){
    e.printStackTrace();
}

Instanciando e usando um carregador de classe #

Este exemplo básico mostra como um aplicativo pode instanciar um carregador de classe e usá-lo para carregar uma classe dinamicamente.

URL[] urls = new URL[] {new URL("file:/home/me/extras.jar")};
Classloader loader = new URLClassLoader(urls);
Class<?> myObjectClass = loader.findClass("com.example.MyObject");

O carregador de classe criado neste exemplo terá o carregador de classe padrão como seu pai, e primeiro tentará encontrar qualquer classe no carregador de classe pai antes de procurar em “extra.jar”. Se a classe solicitada já foi carregada, a chamada findClass retornará a referência à classe carregada anteriormente.

A chamada findClass pode falhar de várias maneiras. Os mais comuns são:

  • Se a classe nomeada não puder ser encontrada, a chamada com throw ClassNotFoundException.
  • Se a classe nomeada depender de alguma outra classe que não pode ser encontrada, a chamada lançará NoClassDefFoundError.
