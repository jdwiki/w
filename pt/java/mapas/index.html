A interface java.util.Map representa um mapeamento entre as chaves e seus valores. Um mapa não pode conter chaves duplicadas; e cada chave pode mapear no máximo um valor.

Como Map é uma interface, você precisa instanciar uma implementação concreta dessa interface para usá-la; existem várias implementações de Map, e as mais usadas são java.util.HashMap e java.util.TreeMap

Um map é um objeto que armazena keys com um value associado para cada chave. Uma chave e seu valor às vezes são chamados de key/value pair ou entry. Os mapas geralmente fornecem esses recursos:

  • Os dados são armazenados no mapa em pares chave/valor.
  • O mapa pode conter apenas uma entrada para uma chave específica. Se um mapa contiver uma entrada com uma chave específica e você tentar armazenar uma segunda entrada com a mesma chave, a segunda entrada substituirá a primeira. Em outras palavras, isso alterará o valor associado à chave.
  • Os mapas fornecem operações rápidas para testar se existe uma chave no mapa, buscar o valor associado a uma chave e remover um par chave/valor.

A implementação de mapa mais comumente usada é HashMap. Funciona bem com chaves que são strings ou números.

Mapas simples como HashMap não são ordenados. A iteração por meio de pares de chave/valor pode retornar entradas individuais em qualquer ordem. Se você precisar iterar pelas entradas do mapa de maneira controlada, observe o seguinte:

  • Mapas ordenados como Mapa Árvore irão iterar através de chaves em sua ordem natural (ou em uma ordem que você possa especificar, fornecendo um Comparador). Por exemplo, espera-se que um mapa ordenado usando números como chaves itere através de suas entradas em ordem numérica.

  • LinkedHashMap permite iterar as entradas na mesma ordem em que foram inseridas no mapa ou na ordem dos acessos mais recentes.

Iterando entradas de mapa com eficiência #

Esta seção fornece código e benchmarks para dez implementações de exemplo únicas que iteram sobre as entradas de um Map<Integer, Integer> e geram a soma dos valores de Integer. Todos os exemplos têm uma complexidade algorítmica de Θ(n), no entanto, os benchmarks ainda são úteis para fornecer informações sobre quais implementações são mais eficientes em um ambiente do “mundo real”.

  1. Implementação usando Iterator com Map.Entry

     Iterator<Map.Entry<Integer, Integer>> it = map.entrySet().iterator();
 while (it.hasNext()) {
     Map.Entry<Integer, Integer> pair = it.next();
     sum += pair.getKey() + pair.getValue();
 }

  2. Implementação usando for com Map.Entry

     for (Map.Entry<Integer, Integer> pair : map.entrySet()) {
     sum += pair.getKey() + pair.getValue();
 }

  3. Implementação usando Map.forEach (Java 8+)

     map.forEach((k, v) -> sum[0] += k + v);

  4. Implementação usando Map.keySet com for

     for (Integer key : map.keySet()) {
     sum += key + map.get(key);
 }

  5. Implementação usando Map.keySet com Iterator

     Iterator<Integer> it = map.keySet().iterator();
 while (it.hasNext()) {
     Integer key = it.next();
     sum += key + map.get(key);
 }

  6. Implementação usando for com Iterator e Map.Entry

     for (Iterator<Map.Entry<Integer, Integer>> entries = 
          map.entrySet().iterator(); entries.hasNext(); ) {
     Map.Entry<Integer, Integer> entry = entries.next();
     sum += entry.getKey() + entry.getValue();
 }

  7. Implementação usando Stream.forEach (Java 8+)

     map.entrySet().stream().forEach(e -> sum += e.getKey() + e.getValue());

  8. Implementação usando Stream.forEach com Stream.parallel (Java 8+)

     map.entrySet()
    .stream()
    .parallel()
    .forEach(e -> sum += e.getKey() + e.getValue());

  9. Implementação usando IterableMap de Apache Collections

     MapIterator<Integer, Integer> mit = iterableMap.mapIterator();
 while (mit.hasNext()) {
     sum += mit.next() + it.getValue();
 }

  10. Implementação usando MutableMap de Eclipse Collections

     mutableMap.forEachKeyValue((key, value) -> {
     sum += key + value;
 });

Usando métodos padrão de mapa do Java 8 #

Exemplos de uso de métodos padrão introduzidos no Java 8 na interface Map

  1. Usando getOrDefault

Retorna o valor mapeado para a chave ou, se a chave não estiver presente, retorna o valor padrão

Map<Integer, String> map = new HashMap<>();
map.put(1, "First element");
map.get(1);                                 // => First element
map.get(2);                                 // => null
map.getOrDefault(2, "Default element");     // => Default element
  1. Usando forEach

Permite realizar a operação especificada na ‘ação’ em cada entrada do mapa

  Map<Integer, String> map = new HashMap<Integer, String>();
  map.put(1, "one");
  map.put(2, "two");
  map.put(3, "three");
  map.forEach((key, value) -> System.out.println("Key: "+key+ " :: Value: "+value));

   // Key: 1 :: Value: one
   // Key: 2 :: Value: two
   // Key: 3 :: Value: three
  1. Usando replaceAll

Substituirá por novo valor somente se a chave estiver presente

 Map<String, Integer> map = new HashMap<String, Integer>();
 map.put("john", 20);
 map.put("paul", 30);
 map.put("peter", 40);
 map.replaceAll((key,value)->value+10);   //{john=30, paul=40, peter=50}
  1. Usando putIfAbsent

O par chave-valor é adicionado ao mapa, se a chave não estiver presente ou mapeada para nula

    Map<String, Integer> map = new HashMap<String, Integer>();
    map.put("john", 20);
    map.put("paul", 30);
    map.put("peter", 40);
    map.putIfAbsent("kelly", 50);     //{john=20, paul=30, peter=40, kelly=50}
  1. Usando remover

Remove a chave apenas se estiver associada ao valor fornecido

    Map<String, Integer> map = new HashMap<String, Integer>();
    map.put("john", 20);
    map.put("paul", 30);
    map.put("peter", 40);
    map.remove("peter",40); //{john=30, paul=40}

  1. Usando substituir

    If the key is present then the value is replaced by new-value. If the key is not present, does nothing.

    Map<String, Integer> map = new HashMap<String, Integer>(); map.put(“john”, 20); map.put(“paul”, 30); map.put(“peter”, 40); map.replace(“peter”,50); //{john=20, paul=30, peter=50} map.replace(“jack”,60); //{john=20, paul=30, peter=50}

  2. Usando computeIfAbsent

Este método adiciona uma entrada no Map. a chave é especificada na função e o valor é o resultado da aplicação da função de mapeamento

    Map<String, Integer> map = new HashMap<String, Integer>();
    map.put("john", 20);
    map.put("paul", 30);
    map.put("peter", 40);
    map.computeIfAbsent("kelly", k->map.get("john")+10); //{john=20, paul=30, peter=40, kelly=30}
    map.computeIfAbsent("peter", k->map.get("john")+10); //{john=20, paul=30, peter=40, kelly=30} //peter already present
  1. Usando computeIfPresent

Este método adiciona uma entrada ou modifica uma entrada existente no Mapa. Não faz nada se uma entrada com essa chave não estiver presente

    Map<String, Integer> map = new HashMap<String, Integer>();
    map.put("john", 20);
    map.put("paul", 30);
    map.put("peter", 40);
    map.computeIfPresent("kelly", (k,v)->v+10); //{john=20, paul=30, peter=40} //kelly not present
    map.computeIfPresent("peter", (k,v)->v+10); //{john=20, paul=30, peter=50} // peter present, so increase the value
  1. Usando computar

Este método substitui o valor de uma chave pelo valor recém-calculado

    Map<String, Integer> map = new HashMap<String, Integer>();
    map.put("john", 20);
    map.put("paul", 30);
    map.put("peter", 40);
    map.compute("peter", (k,v)->v+50); //{john=20, paul=30, peter=90} //Increase the value
  1. Usando mesclar

Adiciona o par chave-valor ao mapa, se a chave não estiver presente ou o valor da chave for nulo Substitui o valor pelo valor recém-calculado, se a chave estiver presente A chave é removida do mapa se o novo valor calculado for nulo

    Map<String, Integer> map = new HashMap<String, Integer>();
    map.put("john", 20);
    map.put("paul", 30);
    map.put("peter", 40);
    
    //Adds the key-value pair to the map, if key is not present or value for the key is null
    map.merge("kelly", 50 , (k,v)->map.get("john")+10); // {john=20, paul=30, peter=40, kelly=50}
    
    //Replaces the value with the newly computed value, if the key is present
    map.merge("peter", 50 , (k,v)->map.get("john")+10); //{john=20, paul=30, peter=30, kelly=50}
   
    //Key is removed from the map , if new value computed is null
    map.merge("peter", 30 , (k,v)->map.get("nancy")); //{john=20, paul=30, kelly=50}

Uso do HashMap #

HashMap é uma implementação da interface Map que fornece uma estrutura de dados para armazenar dados em pares chave-valor.

1. Declarando HashMap

Map<KeyType, ValueType> myMap = new HashMap<KeyType, ValueType>();

KeyType e ValueType devem ser tipos válidos em Java, como - String, Integer, Float ou qualquer classe personalizada como Employee, Student etc.

Por exemplo: Map<String,Integer> myMap = new HashMap<String,Integer>();

2. Colocando valores em HashMap.

Para colocar um valor no HashMap, temos que chamar o método put no objeto HashMap passando a Chave e o Valor como parâmetros.

myMap.put("key1", 1);
myMap.put("key2", 2);

Se você chamar o método put com a Key que já existe no Map, o método substituirá seu valor e retornará o valor antigo.

3. Obtendo valores do HashMap.

Para obter o valor de um HashMap você deve chamar o método get, passando a Key como parâmetro.

myMap.get("key1");    //return 1 (class Integer)

Se você passar uma chave que não existe no HashMap, este método retornará null

4. Verifique se a Chave está no Mapa ou não.

myMap.containsKey(varKey);

5. Verifique se o Valor está no Mapa ou não.

myMap.containsValue(varValue);

Os métodos acima retornarão um valor booleano true ou false se chave, valor existir no Mapa ou não.

Iterando pelo conteúdo de um Map #

Os mapas fornecem métodos que permitem acessar as chaves, valores ou pares chave-valor do mapa como coleções. Você pode iterar por essas coleções. Dado o seguinte mapa, por exemplo:

Map<String, Integer> repMap = new HashMap<>();
repMap.put("Jon Skeet", 927_654);
repMap.put("BalusC", 708_826);
repMap.put("Darin Dimitrov", 715_567);

Iterando por meio de teclas de mapa:

for (String key : repMap.keySet()) {
    System.out.println(key);
}

Impressões:

Darin Dimitrov
Jon Skeet
BalusC

keySet() fornece as chaves do mapa como um Set. Set é usado porque as chaves não podem conter valores duplicados. A iteração através do conjunto produz cada chave por sua vez. HashMaps não são ordenados, então neste exemplo as chaves podem ser retornadas em qualquer ordem.

Iterando pelos valores do mapa:

for (Integer value : repMap.values()) {
    System.out.println(value);
}

Impressões:

715567
927654
708826

values() retorna os valores do mapa como uma Collection. A iteração pela coleção produz cada valor por vez. Novamente, os valores podem ser retornados em qualquer ordem.

Iterando por meio de chaves e valores juntos

for (Map.Entry<String, Integer> entry : repMap.entrySet()) {
    System.out.printf("%s = %d\n", entry.getKey(), entry.getValue());
}

Impressões:

Darin Dimitrov = 715567
Jon Skeet = 927654
BalusC = 708826

entrySet() retorna uma coleção de objetos Map.Entry. Map.Entry dá acesso à chave e valor para cada entrada.

Adicionar vários itens #

Podemos usar V put(K key,V value):

Associa o valor especificado com a chave especificada neste mapa (operação opcional). Se o mapa continha anteriormente um mapeamento para a chave, o valor antigo é substituído pelo valor especificado.

String currentVal;
Map<Integer, String> map = new TreeMap<>();
currentVal = map.put(1, "First element.");
System.out.println(currentVal);// Will print null
currentVal = map.put(2, "Second element.");
System.out.println(currentVal); // Will print null yet again    
currentVal = map.put(2, "This will replace 'Second element'");
System.out.println(currentVal); // will print Second element.
System.out.println(map.size()); // Will print 2 as key having
// value 2 was replaced.

Map<Integer, String> map2 = new HashMap<>();
map2.put(2, "Element 2");
map2.put(3, "Element 3");

map.putAll(map2);

System.out.println(map.size());

Resultado:

3

Para adicionar muitos itens, você pode usar uma classe interna como esta:

Map<Integer, String> map = new HashMap<>() {{
    // This is now an anonymous inner class with an unnamed instance constructor
    put(5, "high");
    put(4, "low");
    put(1, "too slow");
}};

Lembre-se de que criar uma classe interna anônima nem sempre é eficiente e pode levar a vazamentos de memória; portanto, quando possível, use um bloco inicializador:

static Map<Integer, String> map = new HashMap<>();

static {
    // Now no inner classes are created so we can avoid memory leaks
    put(5, "high");
    put(4, "low");
    put(1, "too slow");
}

O exemplo acima torna o mapa estático. Ele também pode ser usado em um contexto não estático removendo todas as ocorrências de static.

Além disso, a maioria das implementações suporta putAll, que pode adicionar todas as entradas de um mapa a outro assim:

another.putAll(one);

Mesclando, combinando e compondo mapas #

Use putAll para colocar cada membro de um mapa em outro. As chaves já presentes no mapa terão seus valores correspondentes sobrescritos.

Map<String, Integer> numbers = new HashMap<>();
numbers.put("One", 1)
numbers.put("Three", 3)
Map<String, Integer> other_numbers = new HashMap<>();
other_numbers.put("Two", 2)
other_numbers.put("Three", 4)

numbers.putAll(other_numbers)

Isso produz o seguinte mapeamento em numbers:

"One" -> 1
"Two" -> 2
"Three" -> 4 //old value 3 was overwritten by new value 4

Se você quiser combinar valores em vez de sobrescrevê-los, você pode usar [Map.merge](http://docs.oracle.com/javase/8/docs/api/java/util/Map.html#merge- K-V-java.util.function.BiFunction-), adicionado no Java 8, que usa um BiFunction fornecido pelo usuário para mesclar valores para chaves duplicadas. merge opera em chaves e valores individuais, então você precisará usar um loop ou Map.forEach. Aqui nós concatenamos strings para chaves duplicadas:

for (Map.Entry<String, Integer> e : other_numbers.entrySet())
    numbers.merge(e.getKey(), e.getValue(), Integer::sum);
//or instead of the above loop
other_numbers.forEach((k, v) -> numbers.merge(k, v, Integer::sum));

Se você quiser impor a restrição de que não há chaves duplicadas, você pode usar uma função de mesclagem que lança um AssertionError:

mapA.forEach((k, v) ->
    mapB.merge(k, v, (v1, v2) ->
        {throw new AssertionError("duplicate values for key: "+k);}));

Compondo Map<X,Y> e Map<Y,Z> para obter Map<X,Z> #

Se você quiser compor dois mapeamentos, você pode fazer da seguinte forma

    Map<String, Integer> map1 = new HashMap<String, Integer>();
    map1.put("key1", 1);
    map1.put("key2", 2);
    map1.put("key3", 3);
    
    Map<Integer, Double> map2 = new HashMap<Integer, Double>();
    map2.put(1, 1.0);
    map2.put(2, 2.0);
    map2.put(3, 3.0);

    Map<String, Double> map3 = new new HashMap<String, Double>();
    map1.forEach((key,value)->map3.put(key,map2.get(value)));

Isso produz o seguinte mapeamento

    "key1" -> 1.0
    "key2" -> 2.0
    "key3" -> 3.0

Os mapas podem conter valores nulos #

Para mapas, é preciso ter cuidado para não confundir “contendo uma chave” com “ter um valor”. Por exemplo, HashMaps pode conter null, o que significa que o seguinte é um comportamento perfeitamente normal:

Map<String, String> map = new HashMap<>();
map.put("one", null);
if (map.containsKey("one")) {
    System.out.println("This prints !"); // This line is reached 
}
if (map.get("one") != null) {
    System.out.println("This is never reached !"); // This line is never reached 
}

Mais formalmente, não há garantia de que map.contains(key) <=> map.get(key)!=null

Use objeto personalizado como chave #

Antes de usar seu próprio objeto como chave, você deve substituir os métodos hashCode() e equals() do seu objeto.

No caso simples, você teria algo como:

class MyKey {
    private String name;
    MyKey(String name) {
        this.name = name;
    }

    @Override
    public boolean equals(Object obj) {            
        if(obj instanceof MyKey) {
            return this.name.equals(((MyKey)obj).name);
        }
        return false;
    }
    
    @Override
    public int hashCode() {
        return this.name.hashCode();
    }
}

hashCode decidirá a qual hash bucket a chave pertence e equals decidirá qual objeto dentro desse hash bucket.

Sem esses métodos, a referência do seu objeto será usada para a comparação acima, que não funcionará a menos que você use a mesma referência de objeto todas as vezes.

Criando e inicializando mapas #

Introdução #

Maps armazena pares de chave/valor, onde cada chave tem um valor associado. Dada uma chave específica, o mapa pode pesquisar o valor associado muito rapidamente.

Maps, também conhecido como array associado, é um objeto que armazena os dados na forma de chaves e valores. Em Java, os mapas são representados usando a interface Map, que não é uma extensão da interface de coleta.
