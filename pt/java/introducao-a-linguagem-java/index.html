Criando seu primeiro programa Java #

Crie um novo arquivo em seu editor de texto ou IDE chamado HelloWorld.java. Em seguida, cole este bloco de código no arquivo e salve:

public class HelloWorld {
    public static void main(String[] args) {
        System.out.println("Hello, World!");
    }
}

Executar ao vivo no Ideone

Nota: Para Java reconhecer isso como uma classe pública (e não lançar um erro de tempo de compilação), o nome do arquivo deve ser o mesmo que o nome da classe (HelloWorld neste exemplo) com uma extensão .java. Deve haver também um modificador de acesso público antes dele.

[Convenções de nomenclatura][4] recomendam que as classes Java comecem com um caractere maiúsculo e estejam no formato camel case (no qual a primeira letra de cada palavra é maiúscula). As convenções recomendam contra sublinhados (_) e cifrões ($).

Para compilar, abra uma janela de terminal e navegue até o diretório de HelloWorld.java:

cd /path/to/containing/folder/

Observação: cd é o comando do terminal para alterar o diretório.

Digite javac seguido pelo nome do arquivo e extensão da seguinte forma:

$ javac HelloWorld.java

É bastante comum receber o erro 'javac' não é reconhecido como um comando interno ou externo, programa operável ou arquivo de lote. mesmo quando você instalou o JDK e é capaz de executar o programa a partir do IDE ex. eclipse etc. Já que o caminho não é adicionado ao ambiente por padrão.

Caso você tenha isso no Windows, para resolver, primeiro tente navegar para o seu caminho javac.exe, provavelmente está em seu C:\Program Files\Java\jdk(version number)\bin. Em seguida, tente executá-lo com abaixo.

$ C:\Program Files\Java\jdk(version number)\bin\javac HelloWorld.java

Anteriormente, quando estávamos chamando javac, era o mesmo que o comando acima. Somente nesse caso seu OS sabia onde javac residia. Então vamos dizer agora, dessa forma você não precisa digitar o caminho todo toda vez. Precisaríamos adicionar isso ao nosso PATH

Para editar a variável de ambiente PATH no Windows XP/Vista/7/8/10:

  • Painel de controle ⇒ Sistema ⇒ Configurações avançadas do sistema
  • Mude para a aba “Avançado” ⇒ Variáveis ​​de Ambiente
  • Em “Variáveis ​​do Sistema”, role para baixo para selecionar “PATH” ⇒ Editar

Você não pode desfazer isso, então tenha cuidado. Primeiro copie seu caminho existente para o bloco de notas. Então, para obter o PATH exato para o seu javac, navegue manualmente até a pasta onde o javac reside e clique na barra de endereço e copie-o. Deve ser algo como c:\Program Files\Java\jdk1.8.0_xx\bin

No campo “Valor da variável”, cole este IN FRONT de todos os diretórios existentes, seguido de um ponto e vírgula (;). NÃO APAGUE quaisquer entradas existentes.

Variable name  : PATH
Variable value : c:\Program Files\Java\jdk1.8.0_xx\bin;[Existing Entries...]

Agora isso deve resolver.

Para sistemas baseados em Linux [experimente aqui][10].

Nota: O comando [javac][5] invoca o compilador Java.

O compilador irá então gerar um arquivo bytecode chamado HelloWorld.class que pode ser executado na [Java Virtual Machine (JVM)][6]. O compilador da linguagem de programação Java, javac, lê arquivos fonte escritos na linguagem de programação Java e os compila em arquivos de classe bytecode. Opcionalmente, o compilador também pode processar anotações encontradas em arquivos de origem e de classe usando a API de processamento de anotações Pluggable. O compilador é uma ferramenta de linha de comando, mas também pode ser invocado usando a API do compilador Java.

Para executar seu programa, digite java seguido do nome da classe que contém o método main (HelloWorld em nosso exemplo). Observe como o .class é omitido:

$ java HelloWorld

Observação: O comando [java][7] executa um aplicativo Java.

Isso irá gerar no seu console:

Olá, Mundo!

Você codificou e construiu com sucesso seu primeiro programa Java!

Nota: Para que os comandos Java (java, javac, etc) sejam reconhecidos, você precisará ter certeza:

  • Um JDK está instalado (por exemplo, [Oracle][8], [OpenJDK][9] e outras fontes)
  • Suas variáveis ​​de ambiente estão corretamente [configuradas][10]

Você precisará usar um compilador (javac) e um executor (java) fornecido pela sua JVM. Para descobrir quais versões você instalou, digite java -version e javac -version na linha de comando. O número da versão do seu programa será impresso no terminal (por exemplo, 1.8.0_73).

Um olhar mais atento ao programa Hello World #

O programa “Hello World” contém um único arquivo, que consiste em uma definição de classe HelloWorld, um método main e uma instrução dentro do método main.

public class HelloWorld {

A palavra-chave class inicia a definição de classe para uma classe chamada HelloWorld. Cada aplicativo Java contém pelo menos uma definição de classe ([Mais informações sobre classes][11]).

    public static void main(String[] args)  {

Este é um método de ponto de entrada (definido por seu nome e assinatura de public static void main(String[])) a partir do qual a JVM pode executar seu programa. Todo programa Java deve ter um. Isso é:

  • public: significa que o método pode ser chamado de qualquer lugar, ou seja, fora do programa também. Consulte [Visibilidade][12] para obter mais informações sobre isso.
  • static: significa que existe e pode ser executado por si só (no nível de classe sem criar um objeto).
  • void: significa que não retorna nenhum valor. Nota: Isto é diferente de C e C++ onde um código de retorno como int é esperado (o modo Java é [System.exit()][13]).

Este método principal aceita:

  • Um [array][14] (normalmente chamado de args) de Strings passados ​​como argumentos para a função principal (por exemplo, de [argumentos de linha de comando][15]).

Quase tudo isso é necessário para um método de ponto de entrada Java.

Peças não obrigatórias:

  • O nome args é um nome de variável, então ele pode ser chamado como você quiser, embora seja normalmente chamado de args.
  • Se seu tipo de parâmetro é um array (String[] args) ou Varargs (String... args) não importa porque arrays podem ser passados ​​para varargs.

Nota: Um único aplicativo pode ter várias classes contendo um método de ponto de entrada (main). O ponto de entrada da aplicação é determinado pelo nome da classe passado como argumento para o comando java.

Dentro do método main, vemos a seguinte declaração:

        System.out.println("Hello, World!");

Vamos dividir esta declaração elemento por elemento:

Elemento Finalidade
Sistema isto denota que a expressão subseqüente irá chamar a classe System, do pacote java.lang.
. este é um “operador de ponto”. Os operadores de ponto fornecem acesso a membros de uma classe1; ou seja, seus campos (variáveis) e seus métodos. Neste caso, este operador de ponto permite referenciar o campo estático out dentro da classe System.
fora este é o nome do campo estático do tipo PrintStream dentro da classe System que contém a funcionalidade de saída padrão.
. este é outro operador de ponto. Este operador de ponto fornece acesso ao método println dentro da variável out.
println este é o nome de um método dentro da classe PrintStream. Este método em particular imprime o conteúdo dos parâmetros no console e insere uma nova linha depois.

Aqui está outro exemplo demonstrando o paradigma OO. Vamos modelar um time de futebol com um (sim, um!) membro. Pode haver mais, mas discutiremos isso quando chegarmos aos arrays.

Primeiro, vamos definir nossa classe Team:

public class Team {
    Member member;
    public Team(Member member) {  // who is in this Team?
        this.member = member;  //  one 'member' is in this Team!
    }
}

Agora, vamos definir nossa classe Member:

class Member {
    private String name;
    private String type;
    private int level; // note the data type here
    private int rank; // note the data type here as well

    public Member(String name, String type, int level, int rank) {
        this.name = name; 
        this.type = type;
        this.level = level;
        this.rank = rank;
    }
}

Por que usamos privado aqui? Bem, se alguém quisesse saber seu nome, deveria perguntar diretamente, em vez de enfiar a mão no bolso e retirar seu cartão do Seguro Social. Este private faz algo assim: impede que entidades externas acessem suas variáveis. Você só pode retornar membros privados por meio de funções getter (mostradas abaixo).

Depois de juntar tudo e adicionar os getters e o método main conforme discutido anteriormente, temos:

public class Team {
    Member member;
    public Team(Member member) {  
        this.member = member;  
    }

    // here's our main method
    public static void main(String[] args) {
       Member myMember = new Member("Aurieel", "light", 10, 1); 
       Team myTeam = new Team(myMember); 
       System.out.println(myTeam.member.getName());
       System.out.println(myTeam.member.getType());
       System.out.println(myTeam.member.getLevel());
       System.out.println(myTeam.member.getRank());
    }
}
