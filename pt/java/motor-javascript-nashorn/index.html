Nashorn é um motor JavaScript desenvolvido em Java pela Oracle, e foi lançado com Java 8. Nashorn permite incorporar Javascript em aplicações Java via JSR-223 e permite desenvolver aplicações Javascript independentes, e fornece melhor desempenho em tempo de execução e melhor conformidade com a especificação Javascript normalizada do ECMA.

Sintaxe #

  • ScriptEngineManager // Fornece um mecanismo de descoberta e instalação para classes ScriptEngine; usa uma SPI (Service Provider Interface)
  • ScriptEngineManager.ScriptEngineManager() // Construtor recomendado
  • ScriptEngine // Fornece a interface para a linguagem de script
  • ScriptEngine ScriptEngineManager.getEngineByName(String shortName) // Método de fábrica para a implementação fornecida
  • Object ScriptEngine.eval(String script) // Executa o script especificado
  • Object ScriptEngine.eval(Reader reader) // Carrega e depois executa um script da fonte especificada
  • ScriptContext ScriptEngine.getContext() // Retorna o provedor de ligações, leitores e gravadores padrão
  • void ScriptContext.setWriter(Writer writer) // Define o destino para enviar a saída do script para

O Nashorn é um mecanismo JavaScript escrito em Java e incluído no Java 8. Tudo o que você precisa está incluído no pacote javax.script.

Observe que o ScriptEngineManager fornece uma API genérica que permite obter mecanismos de script para várias linguagens de script (ou seja, não apenas Nashorn, não apenas JavaScript).

Executar arquivo JavaScript #

// Required imports
import javax.script.ScriptEngineManager;
import javax.script.ScriptEngine;
import javax.script.ScriptException;
import java.io.FileReader;
import java.io.FileNotFoundException;

// Obtain an instance of the JavaScript engine
ScriptEngineManager manager = new ScriptEngineManager();
ScriptEngine engine = manager.getEngineByName("nashorn");

// Load and execute a script from the file 'demo.js'
try {
    engine.eval(new FileReader("demo.js"));
} catch (FileNotFoundException ex) {
    ex.printStackTrace();
} catch (ScriptException ex) {
    // This is the generic Exception subclass for the Scripting API
    ex.printStackTrace();
}

// Outcome:
// 'Script from file!' printed on standard output

demo.js:

print('Script from file!');

Interceptar a saída do script #

// Obtain an instance of JavaScript engine
ScriptEngineManager manager = new ScriptEngineManager();
ScriptEngine engine = manager.getEngineByName("nashorn");

// Setup a custom writer
StringWriter stringWriter = new StringWriter();
// Modify the engine context so that the custom writer is now the default
// output writer of the engine
engine.getContext().setWriter(stringWriter);

// Execute some script
try {
    engine.eval("print('Redirected text!');");
} catch (ScriptException ex) {
    ex.printStackTrace();
}

// Outcome:
// Nothing printed on standard output, but
// stringWriter.toString() contains 'Redirected text!'

Olá Nashorn #

// Obtain an instance of JavaScript engine
ScriptEngineManager manager = new ScriptEngineManager();
ScriptEngine engine = manager.getEngineByName("nashorn");

// Execute an hardcoded script
try {
    engine.eval("print('Hello Nashorn!');");
} catch (ScriptException ex) {
    // This is the generic Exception subclass for the Scripting API
    ex.printStackTrace();
}

// Outcome:
// 'Hello Nashorn!' printed on standard output

Avaliar Strings Aritméticas #

// Obtain an instance of JavaScript engine
ScriptEngineManager manager = new ScriptEngineManager();
ScriptEngine engine = manager.getEngineByName("JavaScript");

//String to be evaluated
String str = "3+2*4+5";
//Value after doing Arithmetic operation with operator precedence will be 16

//Printing the value
try {
    System.out.println(engine.eval(str));
} catch (ScriptException ex) {
    ex.printStackTrace();
}

//Outcome:
//Value of the string after arithmetic evaluation is printed on standard output.
//In this case '16.0' will be printed on standard output.

Definir variáveis ​​globais #

// Obtain an instance of JavaScript engine
ScriptEngineManager manager = new ScriptEngineManager();
ScriptEngine engine = manager.getEngineByName("nashorn");

// Define a global variable
engine.put("textToPrint", "Data defined in Java.");

// Print the global variable
try {
    engine.eval("print(textToPrint);");
} catch (ScriptException ex) {
    ex.printStackTrace();
}

// Outcome:
// 'Data defined in Java.' printed on standard output

Definir e obter variáveis ​​globais #

// Obtain an instance of JavaScript engine
ScriptEngineManager manager = new ScriptEngineManager();
ScriptEngine engine = manager.getEngineByName("nashorn");

try {
    // Set value in the global name space of the engine
    engine.put("name","Nashorn");
    // Execute an hardcoded script
    engine.eval("var value='Hello '+name+'!';");
    // Get value
    String value=(String)engine.get("value");
    System.out.println(value);
} catch (ScriptException ex) {
    // This is the generic Exception subclass for the Scripting API
    ex.printStackTrace();
}

// Outcome:
// 'Hello Nashorn!' printed on standard output

Uso de objetos Java em JavaScript no Nashorn #

É possível passar objetos Java para o motor Nashorn para serem processados ​​em código Java. Ao mesmo tempo, existem algumas construções específicas de JavaScript (e Nashorn), e nem sempre é claro como elas funcionam com objetos Java.

Abaixo há uma tabela que descreve o comportamento de objetos Java nativos dentro de construções JavaScript.

Construções testadas:

  1. Expressão na cláusula if. Na expressão JS na cláusula if não precisa ser booleana ao contrário de Java. É avaliado como falso para os chamados valores falsos (null, undefined, 0, strings vazias etc.)
  2. para cada declaração O Nashorn tem um tipo especial de loop - para cada - que pode iterar sobre diferentes objetos JS e Java.
  3. Obtendo o tamanho do objeto. Em JS os objetos possuem uma propriedade length, que retorna o tamanho de um array ou string.

Resultados:

Tipo Se para cada .comprimento
Java nulo falso Sem iterações Exceção
Cadeia de caracteres vazia Java falso Sem iterações 0
seqüência Java verdadeiro Itera sobre caracteres de string Comprimento da corda
Java inteiro/longo valor != 0 Sem iterações indefinido
Lista de Matrizes Java verdadeiro Itera sobre elementos Comprimento da lista
Java HashMap verdadeiro Itera sobre valores nulo
Java HashSet verdadeiro Itera sobre itens indefinido

Recomendações:

  • É aconselhável usar if (some_string) para verificar se uma string não é nula e não vazia

Implementando uma interface a partir do script #

import java.io.FileReader;
import java.io.IOException;

import javax.script.ScriptEngine;
import javax.script.ScriptEngineManager;
import javax.script.ScriptException;

public class InterfaceImplementationExample {
    public static interface Pet {
        public void eat();
    }

    public static void main(String[] args) throws IOException {
        // Obtain an instance of JavaScript engine
        ScriptEngineManager manager = new ScriptEngineManager();
        ScriptEngine engine = manager.getEngineByName("nashorn");

        try {
            //evaluate a script
            /* pet.js */
            /*
                var Pet = Java.type("InterfaceImplementationExample.Pet");
                
                new Pet() {
                    eat: function() { print("eat"); }
                }            
            */
            
            Pet pet = (Pet) engine.eval(new FileReader("pet.js"));
            
            pet.eat();
        } catch (ScriptException ex) {
            ex.printStackTrace();
        }

        // Outcome:
        // 'eat' printed on standard output
    }
}
