Análise XML usando as APIs JAXP

Nesta página

  • Analisando um documento usando a API StAX

XML Parsing é a interpretação de documentos XML para manipular seu conteúdo usando construções sensatas, sejam elas “nós”, “atributos”, “documentos”, “espaços de nomes” ou eventos relacionados a essas construções.

Java tem uma API nativa para manipulação de documentos XML, chamada JAXP, ou Java API for XML Processing. O JAXP e uma implementação de referência foram empacotados com todas as versões do Java desde o Java 1.4 (JAXP v1.1) e evoluíram desde então. O Java 8 é fornecido com o JAXP versão 1.6.

A API fornece diferentes maneiras de interagir com documentos XML, que são:

  • A interface DOM (Document Object Model)
  • A interface SAX (API simples para XML)
  • A interface StAX (API de streaming para XML)

Princípios da interface DOM #

A interface DOM visa fornecer uma maneira compatível com W3C DOM de interpretar XML. Várias versões do JAXP suportam vários níveis de especificação do DOM (até o nível 3).

Sob a interface Document Object Model, um documento XML é representado como uma árvore, começando com o “Elemento Documento”. O tipo base da API é o tipo Node, permite navegar de um Node para seu pai, seus filhos ou seus irmãos (embora nem todos os Nodes possam ter filhos, por exemplo, nós Text são finais na árvore e nunca têm filhos). As tags XML são representadas como Elements, que estendem notavelmente o Node com métodos relacionados a atributos.

A interface DOM é muito útil, pois permite uma análise “uma linha” de documentos XML como árvores, e permite a modificação fácil da árvore construída (adição de nós, supressão, cópia, …) e, finalmente, sua serialização (de volta ao disco ) pós-modificações. No entanto, isso tem um preço: a árvore reside na memória, portanto, as árvores DOM nem sempre são práticas para documentos XML enormes. Além disso, a construção da árvore nem sempre é a maneira mais rápida de lidar com o conteúdo XML, especialmente se não houver interesse em todas as partes do documento XML.

Princípios da interface SAX #

A API SAX é uma API orientada a eventos para lidar com documentos XML. Sob este modelo, os componentes de um documento XML são interpretados como eventos (por exemplo, “uma tag foi aberta”, “uma tag foi fechada”, “um nó de texto foi encontrado”, “um comentário foi encontrado”). ..

A API SAX usa uma abordagem “push parsing”, onde um SAX Parser é responsável para interpretar o documento XML e invoca métodos em um delegado (um ContentHandler) para lidar com qualquer evento encontrado no documento XML. Normalmente, nunca se escreve um analisador, mas fornece-se um manipulador para reunir todas as informações necessárias do documento XML.

A interface SAX supera as limitações da interface DOM mantendo apenas os dados mínimos necessários no nível do analisador (por exemplo, contextos de namespaces, estado de validação), portanto, apenas as informações que são mantidas pelo ContentHandler - pelas quais você, o desenvolvedor, é responsável - são mantidos na memória. A desvantagem é que não há como “voltar no tempo/documento XML” com tal abordagem: enquanto o DOM permite que um ‘Node’ volte ao seu pai, não existe tal possibilidade no SAX.

Princípios da interface StAX #

A API StAX adota uma abordagem semelhante ao processamento de XML como a API SAX (ou seja, orientada a eventos), a única diferença significativa é que StAX é um analisador pull (onde SAX era um analisador push). No SAX, o Parser está no controle e usa callbacks no ContentHandler. No Stax, você chama o analisador e controla quando/se deseja obter o próximo “evento” XML.

A API começa com XMLStreamReader (ou [XMLEventReader](https://docs.oracle. com/javase/8/docs/api/javax/xml/stream/XMLStreamReader.html)), que são os gateways através dos quais o desenvolvedor pode perguntar nextEvent(), de forma iteradora.

Analisando um documento usando a API StAX #

Considerando o seguinte documento:

<?xml version='1.0' encoding='UTF-8' ?>
<library>
   <book id='1'>Effective Java</book>
   <book id='2'>Java Concurrency In Practice</book>
   <notABook id='3'>This is not a book element</notABook>
</library>

Pode-se usar o código a seguir para analisá-lo e construir um mapa de títulos de livros por id de livro.

import javax.xml.stream.XMLInputFactory;
import javax.xml.stream.XMLStreamConstants;
import javax.xml.stream.XMLStreamReader;
import java.io.StringReader;
import java.util.HashMap;
import java.util.Map;

public class StaxDemo {

public static void main(String[] args) throws Exception {
    String xmlDocument = "<?xml version='1.0' encoding='UTF-8' ?>"
            + "<library>"
                + "<book id='1'>Effective Java</book>"
                + "<book id='2'>Java Concurrency In Practice</book>"
                + "<notABook id='3'>This is not a book element </notABook>"
            + "</library>";

    XMLInputFactory xmlInputFactory = XMLInputFactory.newFactory();
    // Various flavors are possible, e.g. from an InputStream, a Source, ...
    XMLStreamReader xmlStreamReader = xmlInputFactory.createXMLStreamReader(new StringReader(xmlDocument));

    Map<Integer, String> bookTitlesById = new HashMap<>();

    // We go through each event using a loop
    while (xmlStreamReader.hasNext()) {
        switch (xmlStreamReader.getEventType()) {
            case XMLStreamConstants.START_ELEMENT:
                System.out.println("Found start of element: " + xmlStreamReader.getLocalName());
                // Check if we are at the start of a <book> element
                if ("book".equals(xmlStreamReader.getLocalName())) {
                    int bookId = Integer.parseInt(xmlStreamReader.getAttributeValue("", "id"));
                    String bookTitle = xmlStreamReader.getElementText();
                    bookTitlesById.put(bookId, bookTitle);
                }
                break;
            // A bunch of other things are possible : comments, processing instructions, Whitespace...
            default:
                break;
        }
        xmlStreamReader.next();
    }

    System.out.println(bookTitlesById);
}

Isso produz:

Found start of element: library
Found start of element: book
Found start of element: book
Found start of element: notABook
{1=Effective Java, 2=Java Concurrency In Practice}

Neste exemplo, deve-se ter cuidado com algumas coisas:

  1. O uso de xmlStreamReader.getAttributeValue funciona porque verificamos primeiro se o analisador está no estado START_ELEMENT. Em todos os outros estados (exceto ATTRIBUTES), o analisador é obrigado a lançar IllegalStateException, porque os atributos só podem aparecer no início dos elementos.

  2. O mesmo vale para xmlStreamReader.getTextContent(), funciona porque estamos em um START_ELEMENT e sabemos neste documento que o elemento <book> não tem nós filhos não textuais.

Para análise de documentos mais complexos (elementos mais profundos, aninhados, …), é uma boa prática “delegar” o analisador a submétodos ou outros objetos, por exemplo, tenha uma classe ou método BookParser e faça com que ele lide com todos os elementos do START_ELEMENT ao END_ELEMENT da tag XML do livro.

Pode-se também usar um objeto Stack para manter dados importantes para cima e para baixo na árvore.

Analisando e navegando em um documento usando a API DOM #

Considerando o seguinte documento:

<?xml version='1.0' encoding='UTF-8' ?>
<library>
   <book id='1'>Effective Java</book>
   <book id='2'>Java Concurrency In Practice</book>
</library>

Pode-se usar o seguinte código para construir uma árvore DOM a partir de uma String:

import org.w3c.dom.Document;
import org.w3c.dom.Node;
import org.w3c.dom.NodeList;
import org.xml.sax.InputSource;

import javax.xml.parsers.DocumentBuilder;
import javax.xml.parsers.DocumentBuilderFactory;
import java.io.StringReader;

public class DOMDemo {

public static void main(String[] args) throws Exception {
    String xmlDocument = "<?xml version='1.0' encoding='UTF-8' ?>"
            + "<library>"
            + "<book id='1'>Effective Java</book>"
            + "<book id='2'>Java Concurrency In Practice</book>"
            + "</library>";

    DocumentBuilderFactory documentBuilderFactory = DocumentBuilderFactory.newInstance();
    // This is useless here, because the XML does not have namespaces, but this option is usefull to know in cas
    documentBuilderFactory.setNamespaceAware(true);
    DocumentBuilder documentBuilder = documentBuilderFactory.newDocumentBuilder();
    // There are various options here, to read from an InputStream, from a file, ...
    Document document = documentBuilder.parse(new InputSource(new StringReader(xmlDocument)));

    // Root of the document
    System.out.println("Root of the XML Document: " + document.getDocumentElement().getLocalName());

    // Iterate the contents
    NodeList firstLevelChildren = document.getDocumentElement().getChildNodes();
    for (int i = 0; i < firstLevelChildren.getLength(); i++) {
        Node item = firstLevelChildren.item(i);
        System.out.println("First level child found, XML tag name is: " + item.getLocalName());
        System.out.println("\tid attribute of this tag is : " + item.getAttributes().getNamedItem("id").getTextContent());
    }

    // Another way would have been
    NodeList allBooks = document.getDocumentElement().getElementsByTagName("book");
}
}

O código produz o seguinte:

Root of the XML Document: library
First level child found, XML tag name is: book
id attribute of this tag is : 1
First level child found, XML tag name is: book
id attribute of this tag is : 2
