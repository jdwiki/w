Incorporando fontes personalizadas #

Suporte a fontes personalizadas
Os aplicativos que desejam usar fontes personalizadas agora podem incluir essas fontes em seu pacote de aplicativos e registrá-las no sistema incluindo a chave UIAppFonts em seu arquivo Info.plist. O valor dessa chave é uma matriz de strings que identificam os arquivos de fonte no pacote do aplicativo. Quando o sistema vê a chave, ele carrega as fontes especificadas e as disponibiliza para o aplicativo.

Uma vez que as fontes foram definidas no Info.plist, você pode usar suas fontes personalizadas como qualquer outra fonte no IB ou programaticamente.

  1. Arraste e solte sua fonte para a pasta Xcode Supporting Files. Não se esqueça de marcar seu aplicativo na seção “Adicionar aos destinos”. A partir deste momento você pode usar esta fonte no IB e escolhê-la na paleta de fontes.

  1. Para disponibilizar esta fonte no dispositivo, abra Info.plist e adicione Fontes fornecidas pela chave do aplicativo (UIAppFonts). Adicione o nome da fonte como o valor à chave Item 0. Nota: O nome da fonte pode ser diferente do nome do arquivo de fonte.

  2. Obtenha o nome da fonte adicionada personalizada usando o snippet abaixo

[Rápido 3]

for family in UIFont.familyNames {
            print("\(family)")

            for name in UIFont.fontNames(forFamilyName: family) {
                print("   \(name)")
            }
        }

[Objetivo - C]

for (NSString *familyName in [UIFont familyNames]){
        NSLog(@"Family name: %@", familyName);
        for (NSString *fontName in [UIFont fontNamesForFamilyName:familyName]) {
            NSLog(@"--Font name: %@", fontName);
        }
    }

Fontes personalizadas com storyboard #

Fontes personalizadas para componentes de interface do usuário do storyboard podem ser facilmente alcançadas com Atributos de tempo de execução definidos pelo usuário no storyboard e Categorias.

As vantagens são como,

  • Não há necessidade de definir saídas para o elemento ui
  • Não há necessidade de definir fonte para elementos programaticamente.

Passos a seguir

  1. Arquivo de fonte: Adicione o arquivo de fonte (.ttf) ao pacote de aplicativos e adicione a entrada para a fonte em Info.plist em Fonte fornecida por aplicação como nesta documentação de fontes personalizadas.

  2. Definir categorias: Adicione um arquivo como UIKit+IBExtensions e adicione as categorias para elementos de interface do usuário como UILabel, UIButton etc. que você deseja definir a fonte personalizada. Todas as categorias terão um propriedade personalizada diga fontName. Isso será usado a partir do storyboard mais tarde para definir a fonte personalizada (como na etapa 4).

UIKit+IBExtensions.h #

#import <UIKit/UIKit.h>

//Category extension for UILabel
@interface UILabel (IBExtensions)

@property (nonatomic, copy) NSString *fontName;
@end

// Category extension for UITextField
@interface UITextField (IBExtensions)

@property (nonatomic, copy) NSString *fontName;
@end

// Category extension for UIButton
@interface UIButton (IBExtensions)

@property (nonatomic, copy) NSString *fontName;
@end
  1. Getters e Setters: Defina getters e setters para a propriedade fontName para cada categoria adicionada.

UIKit+IBExtensions.m #

#import "UIKit+IBExtensions.h"

@implementation UILabel (IBExtensions)

- (NSString *)fontName {
    return self.font.fontName;
}

- (void)setFontName:(NSString *)fontName {
    self.font = [UIFont fontWithName:fontName size:self.font.pointSize];
}
@end

@implementation UITextField (IBExtensions)

- (NSString *)fontName {
    return self.font.fontName;
}

- (void)setFontName:(NSString *)fontName {
    self.font = [UIFont fontWithName:fontName size:self.font.pointSize];
}
@end

@implementation UIButton (IBExtensions)

- (NSString *)fontName {
    return self.titleLabel.font.fontName;
}

    self.titleLabel.font = [UIFont fontWithName:fontName size:self.titleLabel.font.pointSize];

  1. Definindo fonte no storyboard: Adicione uma entrada em Atributos de tempo de execução definidos pelo usuário com fontName como keyPath e seu Custom Font’s Name como valor com tipo como String como mostrado.

Isso definirá sua fonte personalizada durante a execução do aplicativo.

Notas:

  • Lato-Regular é a fonte personalizada que usei.
  • O mesmo nome no arquivo .ttf adicionado no pacote deve ser usado sem extensão no storyboard.
  • O tamanho da fonte será o mesmo definido no inspetor de atributos do elemento da interface do usuário.

Pegadinhas (dois) #

Alguns deles são repetidos, o que significa que provavelmente vale a pena notar.

  1. Às vezes, o xml do storyboard fica corrompido. Isso exige que você review the structure in Storyboard as Source Code mode (right click the storyboard file > Open As …)
  2. Em alguns casos, a tag navigationItem associada ao atributo runtime definido pelo usuário foi definida como um xml child of the view tag instead of the view controller tag. If so remove it from between the tags for proper operation.
  3. Alterne o NavBar Tint para garantir que a fonte personalizada seja used.
  4. Verifique o parâmetro de tamanho da fonte, a menos que esteja usando uma fonte dinâmica style
  5. A hierarquia de visualização substituirá as configurações. Parece que uma fonte per stack is possible.

  2. Importe os arquivos de fonte para seu projeto Xcode

Solução alternativa de fonte personalizada #

Portanto, o Xcode naturalmente parece que pode lidar com fontes personalizadas no UINavigationItem, mas esse recurso não está sendo atualizado corretamente (a fonte selecionada é ignorada).

Para contornar isso:

Uma maneira é corrigir usando o storyboard e adicionando uma linha de código: primeiro adicione um UIView (UIButton, UILabel ou algum outro UIView subclasse) para o View Controller (não o item de navegação… o Xcode não está permitindo que faça isso). Depois de adicionar o controle você pode modificar a fonte no storyboard e adicionar uma referência como saída para o seu View Controller. Basta atribuir essa visão ao UINavigationItem.titleView. Você também pode definir o nome do texto no código se necessário. Bug relatado (23600285).

@IBOutlet var customFontTitleView: UIButton!

//Sometime later...    
self.navigationItem.titleView = customFontTitleView
