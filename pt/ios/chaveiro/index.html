Sintaxe #

  • kSecClassGenericPassword // Uma chave de valor que representa uma senha não-internet
  • kSecClassInternetPassword // Uma chave de valor representando uma senha de internet
  • kSecClassCertificate // Uma chave de valor representando um certificado
  • kSecClassCertificate // Uma chave de valor representando uma chave
  • kSecClassIdentity // Uma chave de valor representando uma identidade, que é um certificado mais uma chave

O iOS armazena informações privadas, como senhas, chaves de criptografia, certificados e identidades em uma área de armazenamento segura chamada Keychain. Essa área de armazenamento é totalmente gerenciada por um coprocessador chamado Secure Enclave, que está embutido no processador do aplicativo. Como o Keychain é sandboxed no iOS, os itens do keychain só podem ser recuperados pelo aplicativo que os colocou lá em primeiro lugar.

Em alguns casos, você deve ativar o Compartilhamento de Chaves nos recursos do Xcode para evitar erros.

Para interagir com o chaveiro, usamos um framework c chamado Keychain Services. Para obter mais informações, consulte Guia de programação dos serviços de chaveiro da Apple.

Como o Keychain Services está abaixo do nível Foundation, ele é restrito ao uso de tipos CoreFoundation. Como resultado, a maioria dos objetos são representados internamente como CFDictionary contendo CFStrings como suas chaves e uma variedade de tipos CoreFoundation como seus valores.

Embora o Keychain Services esteja incluído como parte do framework Security, importar Foundation geralmente é uma boa opção, pois inclui algumas funções auxiliares no backend.

Além disso, se você não quiser lidar diretamente com os Serviços de Chaveiro, a Apple fornece o [Chaveiro Genérico](https://developer.apple.com/library/content/samplecode/GenericKeychain/Introduction/Intro.html#//apple_ref /doc/uid/DTS40007797) Projeto de amostra Swift que fornece tipos Swift que usam o Keychain Services nos bastidores.

Adicionando uma senha ao chaveiro #

Cada item de chaveiro é mais frequentemente representado como um CFDictionary. Você pode, no entanto, simplesmente usar NSDictionary em Objective-C e aproveitar a ponte, ou em Swift você pode usar Dictionary e explicitamente converter para CFDictionary.

Você pode construir uma senha com o seguinte dicionário:

Rápido #

var dict = [String : AnyObject]()

Primeiro, você precisa de um par de chave/valor que permita que o Keychain saiba que esta é uma senha. Note que porque nossa chave dict é uma String nós devemos converter qualquer CFString para uma String explicitamente no Swift 3. CFString não pode ser usada como a chave para um Dicionário Swift porque não é Hashable.

Rápido #

dict[kSecClass as String] = kSecClassGenericPassword

Em seguida, nossa senha pode ter uma série de atributos para descrevê-la e nos ajudar a encontrá-la posteriormente. Aqui está uma lista de atributos para senhas genéricas.

Rápido #

// The password will only be accessible when the device is unlocked
dict[kSecAttrAccessible as String] = kSecAttrAccessibleWhenUnlocked
// Label may help you find it later
dict[kSecAttrLabel as String] = "com.me.myapp.myaccountpassword" as CFString
// Username
dict[kSecAttrAccount as String] = "My Name" as CFString
// Service name
dict[kSecAttrService as String] = "MyService" as CFString

Finalmente, precisamos de nossos dados privados reais. Certifique-se de não manter isso na memória por muito tempo. Este deve ser CFData.

Rápido #

dict[kSecValueData as String] = "my_password!!".data(using: .utf8) as! CFData

Finalmente, a função de adição dos Serviços de Chaves deseja saber como ela deve retornar o item de chave recém-construído. Como você não deve ficar segurando os dados por muito tempo na memória, veja como você só pode retornar os atributos:

Rápido #

dict[kSecReturnAttributes as String] = kCFBooleanTrue

Agora nós construímos nosso item. Vamos adicioná-lo:

Rápido #

var result: AnyObject?
let status = withUnsafeMutablePointer(to: &result) {
    SecItemAdd(dict as CFDictionary, UnsafeMutablePointer($0))
}
let newAttributes = result as! Dictionary<String, AnyObject>

Isso coloca os novos atributos dict dentro de result. SecItemAdd pega o dicionário que construímos, bem como um ponteiro para onde gostaríamos de nosso resultado. A função então retorna um OSStatus indicando sucesso ou um código de erro. Os códigos de resultado são descritos aqui.

Controle de acesso às chaves (TouchID com fallback de senha) #

O Keychain permite salvar itens com o atributo especial SecAccessControl, que permitirá obter o item do Keychain somente após o usuário ser autenticado com Touch ID (ou senha, se tal fallback for permitido). O aplicativo só é notificado se a autenticação foi bem-sucedida ou não, toda a interface do usuário é gerenciada pelo iOS.

Primeiro, o objeto SecAccessControl deve ser criado:

Rápido #

let error: Unmanaged<CFError>?

guard let accessControl = SecAccessControlCreateWithFlags(kCFAllocatorDefault, kSecAttrAccessibleWhenPasscodeSetThisDeviceOnly, .userPresence, &error) else {
    fatalError("Something went wrong")
}

Em seguida, adicione-o ao dicionário com a chave kSecAttrAccessControl (que é mutuamente exclusiva com a chave kSecAttrAccessible que você está usando em outros exemplos):

Rápido #

var dictionary = [String : Any]()

dictionary[kSecClass as String] = kSecClassGenericPassword
dictionary[kSecAttrLabel as String] = "com.me.myapp.myaccountpassword" as CFString
dictionary[kSecAttrAccount as String] = "My Name" as CFString
dictionary[kSecValueData as String] = "new_password!!".data(using: .utf8) as! CFData
dictionary[kSecAttrAccessControl as String] = accessControl

E salve-o como você fez antes:

Rápido #

let lastResultCode = SecItemAdd(query as CFDictionary, nil)

Para acessar os dados armazenados, basta consultar o Keychain para obter uma chave. Os Serviços de Chaves apresentarão uma caixa de diálogo de autenticação ao usuário e retornarão dados ou nulos, dependendo se a impressão digital adequada foi fornecida ou a senha correspondente.

Opcionalmente, a string de prompt pode ser especificada:

Rápido #

var query = [String: Any]()

query[kSecClass as String] = kSecClassGenericPassword
query[kSecReturnData as String] = kCFBooleanTrue
query[kSecAttrAccount as String] = "My Name" as CFString
query[kSecAttrLabel as String] = "com.me.myapp.myaccountpassword" as CFString
query[kSecUseOperationPrompt as String] = "Please put your fingers on that button" as CFString

var queryResult: AnyObject?
let status = withUnsafeMutablePointer(to: &queryResult) {
    SecItemCopyMatching(query as CFDictionary, UnsafeMutablePointer($0))
}

Preste atenção que status será err se o usuário recusou, cancelou ou falhou na autorização.

Rápido #

if status == noErr {
    let password = String(data: queryResult as! Data, encoding: .utf8)!
    print("Password: \(password)")
} else {
    print("Authorization not passed")
}

Encontrando uma senha no chaveiro #

Para construir uma consulta, precisamos representá-la como um CFDictionary. Você também pode usar NSDictionary em Objective-C ou Dictionary em Swift e converter para CFDictionary.

Precisamos de uma chave de classe:

Rápido #

var dict = [String : AnyObject]()
dict[kSecClass as String] = kSecClassGenericPassword

Em seguida, podemos especificar atributos para restringir nossa pesquisa:

Rápido #

// Label
dict[kSecAttrLabel as String] = "com.me.myapp.myaccountpassword" as CFString
// Username
dict[kSecAttrAccount as String] = "My Name" as CFString
// Service name
dict[kSecAttrService as String] = "MyService" as CFString

Também podemos especificar chaves modificadoras de pesquisa especiais descritas aqui.

Por fim, precisamos dizer como gostaríamos que nossos dados fossem retornados. Abaixo, solicitaremos que apenas a própria senha privada seja retornada como um objeto CFData:

Rápido #

dict[kSecReturnData as String] = kCFBooleanTrue

Agora vamos pesquisar:

Rápido #

var queryResult: AnyObject?
let status = withUnsafeMutablePointer(to: &queryResult) {
    SecItemCopyMatching(dict as CFDictionary, UnsafeMutablePointer($0))
}
// Don't keep this in memory for long!!
let password = String(data: queryResult as! Data, encoding: .utf8)!

Aqui, SecItemCopyMatching recebe um dicionário de consulta e um ponteiro para onde você gostaria que o resultado fosse. Ele retorna um OSStatus com códigos de resultado. Aqui estão as possibilidades.

Atualizando uma senha no chaveiro #

Como de costume, primeiro precisamos de um CFDictionary para representar o item que queremos atualizar. Deve conter todos os valores antigos do item, incluindo os dados privados antigos. Em seguida, é necessário um CFDictionary de quaisquer atributos ou dos próprios dados que você gostaria de alterar.

Então, primeiro, vamos construir uma chave de classe e uma lista de atributos. Esses atributos podem restringir nossa pesquisa, mas você deve incluir quaisquer atributos e valores antigos se for alterá-los.

Rápido #

var dict = [String : AnyObject]()
dict[kSecClass as String] = kSecClassGenericPassword
// Label
dict[kSecAttrLabel as String] = "com.me.myapp.myaccountpassword" as CFString
// Username
dict[kSecAttrAccount as String] = "My Name" as CFString

Agora devemos adicionar os dados antigos:

Rápido #

dict[kSecValueData as String] = "my_password!!".data(using: .utf8) as! CFData

Agora vamos criar os mesmos atributos, mas uma senha diferente:

Rápido #

var newDict = [String : AnyObject]()
newDict[kSecClass as String] = kSecClassGenericPassword
// Label
newDict[kSecAttrLabel as String] = "com.me.myapp.myaccountpassword" as CFString
// Username
newDict[kSecAttrAccount as String] = "My Name" as CFString
// New password
newDict[kSecValueData as String] = "new_password!!".data(using: .utf8) as! CFData

Agora, apenas passamos para o Keychain Services:

Rápido #

let status = SecItemUpdate(dict as CFDictionary, newDict as CFDictionary)

SecItemUpdate retorna um código de status. Os resultados são descritos aqui.

Removendo uma senha do chaveiro #

Precisamos apenas de uma coisa para excluir um item do Keychain: um CFDictionary com atributos descrevendo os itens a serem excluídos. Quaisquer itens que correspondam ao dicionário de consulta serão excluídos permanentemente, portanto, se você pretende excluir apenas um único item, certifique-se de ser específico em sua consulta. Como sempre, podemos usar um NSDictionary em Objective-C ou em Swift podemos usar um Dictionary e depois converter para CFDictionary.

Um dicionário de consulta, neste contexto, inclui exclusivamente uma chave de classe para descrever o que é o item e atributos para descrever informações sobre o item. A inclusão de restrições de pesquisa como kSecMatchCaseInsensitive não é permitida.

Rápido #

var dict = [String : AnyObject]()
dict[kSecClass as String] = kSecClassGenericPassword
// Label
dict[kSecAttrLabel as String] = "com.me.myapp.myaccountpassword" as CFString
// Username
dict[kSecAttrAccount as String] = "My Name" as CFString

E agora podemos simplesmente removê-lo:

Rápido #

let status = SecItemDelete(dict as CFDictionary)

SecItemDelete retorna um OSStatus. Os códigos de resultado são descritos aqui.

Keychain Adicionar, atualizar, remover e localizar operações usando um arquivo. #

Chaveiro.h

#import <Foundation/Foundation.h>
typedef void (^KeychainOperationBlock)(BOOL successfulOperation, NSData *data, OSStatus status);

@interface Keychain : NSObject

-(id) initWithService:(NSString *) service_ withGroup:(NSString*)group_;

-(void)insertKey:(NSString *)key withData:(NSData *)data withCompletion:(KeychainOperationBlock)completionBlock;
-(void)updateKey:(NSString*)key withData:(NSData*) data withCompletion:(KeychainOperationBlock)completionBlock;
-(void)removeDataForKey:(NSString*)key withCompletionBlock:(KeychainOperationBlock)completionBlock;
-(void)findDataForKey:(NSString*)key withCompletionBlock:(KeychainOperationBlock)completionBlock;

@end

Keychain.m

#import "Keychain.h"
#import <Security/Security.h>

@implementation Keychain

{
    NSString * keychainService;
    NSString * keychainGroup;
}

-(id) initWithService:(NSString *)service withGroup:(NSString*)group
{
    self =[super init];
    if(self) {
        keychainService = [NSString stringWithString:service];
        if(group) {
            keychainGroup = [NSString stringWithString:group];
        }
    }
    
    return  self;
}

-(void)insertKey:(NSString *)key
        withData:(NSData *)data
  withCompletion:(KeychainOperationBlock)completionBlock
{
    NSMutableDictionary * dict =[self prepareDict:key];
    [dict setObject:data forKey:(__bridge id)kSecValueData];
    [dict setObject:keychainService forKey:(id)kSecAttrService];
    
    OSStatus status = SecItemAdd((__bridge CFDictionaryRef)dict, NULL);
    if(errSecSuccess != status) {
        DLog(@"Unable add item with key =%@ error:%d",key,(int)status);
        if (completionBlock) {
            completionBlock(errSecSuccess == status, nil, status);
        }
    }
    if (status == errSecDuplicateItem) {
        [self updateKey:key withData:data withCompletion:^(BOOL successfulOperation, NSData *updateData, OSStatus updateStatus) {
            if (completionBlock) {
                completionBlock(successfulOperation, updateData, updateStatus);
            }
            DLog(@"Found duplication item -- updating key with data");
        }];
    }
}

-(void)findDataForKey:(NSString *)key
  withCompletionBlock:(KeychainOperationBlock)completionBlock
{
    NSMutableDictionary *dict = [self prepareDict:key];
    [dict setObject:(__bridge id)kSecMatchLimitOne forKey:(__bridge id)kSecMatchLimit];
    [dict setObject:keychainService forKey:(id)kSecAttrService];
    [dict setObject:(id)kCFBooleanTrue forKey:(__bridge id)kSecReturnData];
    CFTypeRef result = NULL;
    OSStatus status = SecItemCopyMatching((__bridge CFDictionaryRef)dict,&result);
    
    if( status != errSecSuccess) {
        DLog(@"Unable to fetch item for key %@ with error:%d",key,(int)status);
        if (completionBlock) {
            completionBlock(errSecSuccess == status, nil, status);
        }
    } else {
        if (completionBlock) {
            completionBlock(errSecSuccess == status, (__bridge NSData *)result, status);
        }
    }
}

-(void)updateKey:(NSString *)key
        withData:(NSData *)data
  withCompletion:(KeychainOperationBlock)completionBlock
{
    NSMutableDictionary * dictKey =[self prepareDict:key];
    
    NSMutableDictionary * dictUpdate =[[NSMutableDictionary alloc] init];
    [dictUpdate setObject:data forKey:(__bridge id)kSecValueData];
    [dictUpdate setObject:keychainService forKey:(id)kSecAttrService];
    OSStatus status = SecItemUpdate((__bridge CFDictionaryRef)dictKey, (__bridge CFDictionaryRef)dictUpdate);
    if( status != errSecSuccess) {
        DLog(@"Unable to remove item for key %@ with error:%d",key,(int)status);
    }
    if (completionBlock) {
        completionBlock(errSecSuccess == status, nil, status);
    }
}

-(void)removeDataForKey:(NSString *)key
    withCompletionBlock:(KeychainOperationBlock)completionBlock {
    NSMutableDictionary *dict = [self prepareDict:key];
    OSStatus status = SecItemDelete((__bridge CFDictionaryRef)dict);
    if( status != errSecSuccess) {
        DLog(@"Unable to remove item for key %@ with error:%d",key,(int)status);
    }
    if (completionBlock) {
        completionBlock(errSecSuccess == status, nil, status);
    }
}

#pragma mark Internal methods

-(NSMutableDictionary*) prepareDict:(NSString *) key {
    
    NSMutableDictionary *dict = [[NSMutableDictionary alloc] init];
    [dict setObject:(__bridge id)kSecClassGenericPassword forKey:(__bridge id)kSecClass];
    
    NSData *encodedKey = [key dataUsingEncoding:NSUTF8StringEncoding];
    [dict setObject:encodedKey forKey:(__bridge id)kSecAttrGeneric];
    [dict setObject:encodedKey forKey:(__bridge id)kSecAttrAccount];
    [dict setObject:keychainService forKey:(__bridge id)kSecAttrService];
    [dict setObject:(__bridge id)kSecAttrAccessibleAlwaysThisDeviceOnly forKey:(__bridge id)kSecAttrAccessible];
    
    //This is for sharing data across apps
    if(keychainGroup != nil) {
        [dict setObject:keychainGroup forKey:(__bridge id)kSecAttrAccessGroup];
    }
    
    return  dict;
}
