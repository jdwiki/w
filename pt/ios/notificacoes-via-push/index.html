Sintaxe #

  • UIUserNotificationSettings.types: UIUserNotificationType // Uma máscara de bits dos tipos de notificação que seu aplicativo tem permissão para usar
  • UIUserNotificationSettings.categories: Set // Grupos de ações registrados do aplicativo

Parâmetros #

Parâmetro Descrição
userInfo Um dicionário que contém informações de notificação remota, possivelmente incluindo um número de crachá para o ícone do aplicativo, som de alerta, mensagem de alerta, um identificador de notificação e dados personalizados.

Registrando dispositivo para notificações push #

Para registrar seu dispositivo para notificações push, adicione o seguinte código ao seu arquivo AppDelegate no método didFinishLaunchingWithOptions:

Rápido #

func application(application: UIApplication, didFinishLaunchingWithOptions launchOptions: [NSObject: AnyObject]?) -> Bool {
    // Override point for customization after application launch.
    if UIDevice.currentDevice().systemVersion.compare(v, options: .NumericSearch) == NSOrderedAscending { 
        // Register for Push Notitications, if running iOS < 8
        if application.respondsToSelector("registerUserNotificationSettings:") {
            let types:UIUserNotificationType = (.Alert | .Badge | .Sound)
            let settings:UIUserNotificationSettings = UIUserNotificationSettings(forTypes: types, categories: nil)

            application.registerUserNotificationSettings(settings)
            application.registerForRemoteNotifications()
        } else {      
            // Register for Push Notifications before iOS 8
            application.registerForRemoteNotificationTypes(.Alert | .Badge | .Sound)
        }
    } else {
        var center = UNUserNotificationCenter.currentNotificationCenter()

Registrando o ID do aplicativo para uso com notificações push #

Coisas que você precisa #

  • Uma assinatura paga do Apple Developer Program
  • Um ID e identificador de aplicativo válidos para seu aplicativo (como com.example.MyApp) que não é usado antes em nenhum lugar
  • Acesso a developer.apple.com e Central de Membros
  • Um dispositivo iOS para testar (já que as notificações push não funcionam no simulador)

Habilitando o acesso de APNs para App ID no Apple Developer Center #

1- Faça login no centro de membros do developer.apple.com (o link da conta na página inicial)

2- Vá em “Certificados”

3- Selecione “ID do aplicativo” no painel esquerdo

4- Clique em “+” no canto superior direito

5- Adicionar App ID com a opção Push Notifications marcada

6- Clique no ID do aplicativo criado e selecione Editar

7- Clique em Configurar no painel Push Notifications

8- Abra o aplicativo Keychain Access no seu Mac

9- No menu Acesso às Chaves, clique em Assistente de Certificado -> Solicitar um Certificado de uma Autoridade de Certificação

10- Digite seu e-mail no primeiro campo de texto

11- Digite seu nome no segundo campo de texto

12- Deixe o endereço de e-mail da CA vazio

13- Selecione Salvo no disco em vez de Enviado por e-mail para a CA

14- Clique em Continuar e carregue o arquivo gerado

15- Baixe o arquivo gerado pela Apple e abra-o enquanto o Acesso às Chaves estiver aberto

Habilitando o acesso de APNs no Xcode #

1- Selecione seu projeto

2- Abra a guia Recursos

3- Encontre Push Notifications e ative-o

4-Encontre os modos de fundo e ligue-o e verifique as notificações remotas

Testando notificações push #

É sempre uma boa prática testar como as notificações push funcionam mesmo antes de você ter o lado do servidor pronto para elas, apenas para ter certeza de que tudo está configurado corretamente do seu lado. É muito fácil enviar a si mesmo uma notificação por push usando um script PHP a seguir.

  1. Salve o script como um arquivo (send_push.php por exemplo) na mesma pasta do seu certificado (desenvolvimento ou produção)

  2. Edite-o para colocar o token do seu dispositivo, senha do certificado

  3. Escolha o caminho correto para abrir uma conexão, dev_path ou prod_path (é onde ‘Abrir uma conexão com o servidor APNS’ acontece no script)

  4. cd para a pasta no Terminal e execute o comando ‘php send_push’

  5. Receba a notificação no seu dispositivo

Gerando um certificado .pem do seu arquivo .cer, para passar para o desenvolvedor do servidor #

  1. Salve aps.cer em uma pasta

  2. Abra “Keychain access” e exporte a chave que está sob esse certificado para um arquivo .p12 (chame-o de key.p12). Para fazer isso, clique com o botão direito sobre ele e escolha Exportar. Salve-o na mesma pasta da etapa 1. Na exportação, será solicitada uma senha. Faça algo e memorize.

  3. cd para essa pasta no Terminal e execute os seguintes comandos:

  4. Converter .cer em um certificado .pem

    openssl x509 -in aps.cer -inform der -out aps.pem

  5. Converta sua chave para o formato .pem. Para abrir a chave, digite a senha com a qual você a exportou do chaveiro, na etapa 2. Em seguida, digite outra senha que protegerá o arquivo exportado. Você será solicitado a digitá-lo duas vezes para confirmação.

    openssl pkcs12 -nocerts -out key.pem -in key.p12

  6. Mescle os arquivos em um arquivo final

    cat key.pem aps.pem > final_cert.pem

  7. O final_cert.pem é o resultado final. Passe para os desenvolvedores do servidor com a senha da etapa 5, para que eles possam usar o certificado protegido.

Cancelando o registro de notificações push #

Para cancelar o registro de notificações remotas programaticamente, você pode usar

Objetivo-C #

[[UIApplication sharedApplication] unregisterForRemoteNotifications];

Rápido #

UIApplication.sharedApplication().unregisterForRemoteNotifications()

isso é semelhante a entrar na configuração do seu telefone e desligar manualmente as notificações do aplicativo.

OBSERVAÇÃO: pode haver casos raros em que você precisaria disso (por exemplo: quando seu aplicativo não suporta mais notificações push)

Se você deseja apenas permitir que o usuário desative temporariamente as Notificações. Você deve implementar um método para remover o token do dispositivo no banco de dados em seu servidor. caso contrário, se você desativar apenas a Notificação localmente no seu dispositivo, seu servidor ainda enviará mensagens.

Registrando-se para Notificação Push (Não Interativa) #

Recomenda-se que a lógica de registro para notificação push seja adicionada em AppDelegate.swift, pois as funções de retorno de chamada (sucesso, falha) serão chamadas de seus. Para se inscrever basta fazer o seguinte:

let application = UIApplication.sharedApplication()
let settings = UIUserNotificationSettings(forTypes: [.Alert, .Badge, .Sound], categories: nil)
application.registerUserNotificationSettings(settings)

Então a função de callback didRegisterUserNotificationSettings será chamada e nesse caso você apenas aciona o registro assim:

func application(application: UIApplication, didRegisterUserNotificationSettings notificationSettings: UIUserNotificationSettings) {
    application.registerForRemoteNotifications()
}

E, nesse caso, o alerta do sistema será mostrado solicitando permissão para receber notificação por push. Uma das seguintes funções de retorno de chamada será chamada:

func application(application: UIApplication, didRegisterForRemoteNotificationsWithDeviceToken deviceToken: NSData) {
    let tokenChars = UnsafePointer<CChar>(deviceToken.bytes)
    var tokenString = ""
    
    for i in 0..<deviceToken.length {
        tokenString += String(format: "%02.2hhx", arguments: [tokenChars[i]])
    }
    
    print("Push token: \(tokenString)")
}

func application(application: UIApplication, didFailToRegisterForRemoteNotificationsWithError error: NSError) {
    print("didFailToRegisterForRemoteNotificationsWithError: \(error)")
            
}

Em casos muito raros, nem as funções de retorno de chamada de sucesso ou falha são chamadas. Isso acontece quando você tem problemas de conexão com a Internet ou o APNS Sandbox está inativo. O sistema faz uma chamada de API para o APNS para fazer alguma verificação, caso contrário, nenhuma das duas funções de callback será chamada. Visite status do sistema Apple para verificar se está tudo bem.
