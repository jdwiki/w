Adicionando um esquema de URL ao seu próprio aplicativo #

Digamos que você esteja trabalhando em um aplicativo chamado MyTasks e deseja permitir que URLs de entrada criem uma nova tarefa com um título e um corpo. O URL que você está criando pode ser algo assim:

mytasks://create?title=hello&body=world

(Claro, os parâmetros text e body são usados ​​para preencher nossa tarefa que estamos criando!)

Aqui estão os grandes passos para adicionar este esquema de URL ao seu projeto:

  1. Registre um esquema de URL no arquivo Info.plist do seu aplicativo, para que o sistema saiba quando rotear um URL para seu aplicativo.
  2. Adicione uma função ao seu UIApplicationDelegate que aceite e lide com URLs de entrada.
  3. Execute qualquer tarefa que precise ocorrer quando esse URL for aberto.

Passo Um: Registre um esquema de URL no Info.plist: #

Primeiro, precisamos adicionar uma entrada “Tipos de URL” ao nosso arquivo Info.plist. Clique no botão (+) aqui:

…então insira um identificador exclusivo para seu aplicativo, bem como o esquema de URL que você deseja usar. Seja específico! Você não quer que o esquema de URL entre em conflito com a implementação de outro aplicativo. Melhor ser muito longo aqui do que muito curto!

Etapa dois: manipular a URL no UIApplicationDelegate #

Precisamos implementar application:openURL:options: em nosso UIApplicationDelegate. Vamos inspecionar o URL de entrada e ver se há uma ação que podemos tomar!

Uma implementação seria esta:

func application(app: UIApplication, openURL url: NSURL, options: [String : AnyObject]) -> Bool {
    if url.scheme == "mytasks" && url.host == "create" {
        let title = // get the title out of the URL's query using a method of your choice
        let body = // get the title out of the URL's query using a method of your choice
        self.rootViewController.createTaskWithTitle(title, body: body)
        return true
    }

    return false
}

Etapa Três: Execute uma tarefa dependendo da URL. #

Quando um usuário abre seu aplicativo por meio de um URL, ele provavelmente espera que algo aconteça. Talvez seja navegar para um conteúdo, talvez seja criar um novo item - neste exemplo, vamos criar uma nova tarefa no aplicativo!

No código acima, podemos ver uma chamada para self.rootViewController.createTaskWithTitle(:body:) - supondo que seu AppDelegate tenha um ponteiro para seu controlador de visualização raiz que implementa a função corretamente, está tudo pronto !

Abrindo um aplicativo com base em seu esquema de URL #

Para abrir um aplicativo com esquema de URL definido todolist://:

Objetivo-C

NSURL *myURL = [NSURL URLWithString:@"todolist://there/is/something/to/do"];
[[UIApplication sharedApplication] openURL:myURL];

Rápido

let stringURL = "todolist://there/is/something/to/do"
if let url = NSURL(string: stringURL) {
    UIApplication.shared().openURL(url)
}

HTML

<a href="todolist://there/is/something/to/do">New SMS Message</a>

Observação: É útil verificar se o link pode ser aberto para exibir uma mensagem apropriada ao usuário. Isso pode ser feito usando o método canOpenURL:.

Configurando deeplink para seu aplicativo #

Configurar links diretos para seu aplicativo é fácil. Você só precisa de um pequeno URL com o qual deseja abrir seu aplicativo.

Siga as etapas para configurar links diretos para seu aplicativo.

  1. Vamos criar um projeto e nomeá-lo DeepLinkPOC.

  2. Agora selecione o destino do seu projeto.

  3. Após selecionar o alvo, selecione a aba ‘info’.

  4. Role para baixo até ver uma opção de Tipos de URL

  5. Clique na opção ‘+’.

  6. Você verá esquemas de URL adicionar uma string com a qual deseja abrir seu aplicativo. Vamos adicionar “DeepLinking” em esquemas de URL.

Portanto, para abrir seu aplicativo, você pode iniciá-lo digitando “DeepLinking://” em seu safari. Sua string de link direto tem o seguinte formato.

[scheme]://[host]/[path]  --> DeepLinking://path/Page1

Onde, Esquema: “Deep Linking” Hospedeiro: “caminho” caminho: “Página1”

Nota: Mesmo que não adicione host e caminho, ele iniciará o aplicativo, então não se preocupe. Mas você pode adicionar host e caminho para redirecionar adicionalmente para uma página específica após o lançamento do aplicativo.

  1. Agora adicione o seguinte método ao seu appdelegate.

Rápido:

 func application(application: UIApplication, openURL url: NSURL, sourceApplication: String?, annotation: AnyObject) -> Bool

Objetivo-c:

-(BOOL)application:(UIApplication *)application
          openURL:(NSURL *)url
          sourceApplication:(NSString *)sourceApplication
          annotation:(id)annotation

O método acima é chamado sempre que seu aplicativo é iniciado usando uma string de link direto que você definiu para seu aplicativo.
