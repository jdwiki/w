Criando seu próprio botão personalizado “Entrar com o Facebook” #

Às vezes, queremos projetar nossa própria interface do usuário para o botão “Entrar com o Facebook” em vez do botão original que vem com o FacebookSDK.

  1. Em seu storyboard, arraste seu UIButton e defina-o como quiser.
  2. Ctrl + arraste seu botão para o seu controlador de visualização como IBAction.
  3. Dentro do método IBAction, você simulará um toque no botão real do Facebook da seguinte forma:

Rápido:

    let loginButton = FBSDKLoginButton()
    loginButton.delegate = self
    // Your Custom Permissions Array
    loginButton.readPermissions =
    [
                             "public_profile",
                             "email",
                             "user_about_me",
                             "user_photos"
    ]
    // Hiding the button
    loginButton.hidden = true
    self.view.addSubview(loginButton)
    // Simulating a tap for the actual Facebook SDK button
    loginButton.sendActionsForControlEvents(UIControlEvents.TouchUpInside)

Objetivo-C:

FBSDKLoginButton *FBButton = [FBSDKLoginButton new];

// Your Custom Permissions Array
FBButton.readPermissions = @[@"public_profile",
                             @"email",
                             @"user_about_me",
                             @"user_photos"
                             ];
FBButton.loginBehavior = FBSDKLoginBehaviorNative;
[FBButton setDelegate:self];
[FBButton setHidden:true];
[loginButton addSubview:FBButton];

[FBButton sendActionsForControlEvents:UIControlEventTouchUpInside];

Você Terminou.

Integração com o FacebookSDK #

Etapa 1: instale o SDK

Você pode instalar o SDK manualmente ou via CocoaPods. A última opção é altamente recomendada.

Coloque estas linhas em Podfile:

target 'MyApp' do
  use_frameworks!

  pod 'FBSDKCoreKit'
  pod 'FBSDKLoginKit'
  pod 'FBSDKShareKit'
end

Execute pod install no terminal e abra .xcworkspace em vez de .xcodeproj depois.

FBSDKLoginKit e FBSDKShareKit são opcionais. Você pode ou não precisar deles.

Etapa 2: crie um aplicativo no Facebook

Vá para Inícios rápidos - Facebook para desenvolvedores para criar um aplicativo.

O Facebook solicitará que você baixe o SDK depois de criar o aplicativo. Você pode pular esta parte se já instalou o SDK via CocoaPods.

Etapa 3: Edite .plist

uma. Para tornar seu aplicativo capaz de “comunicar” com o Facebook, você precisa colocar algumas configurações em seu arquivo .plist. O Facebook fornecerá o snippet personalizado na página Quick Starts.

b. Edite seu arquivo .plist como código fonte.

c. Cole seu snippet personalizado no código-fonte. Tenha cuidado! O snippet deve ser exatamente o filho da tag <dict>. Seu código-fonte deve ser algo como:

<plist version="1.0">
<dict>
    // ...
    //some default settings
    // ...
    <key>CFBundleURLTypes</key>
    <array>
        <dict>
            <key>CFBundleURLSchemes</key>
            <array>
                <string>fb{FBAppId}</string>
            </array>
        </dict>
    </array>
    <key>FacebookAppID</key>
    <string>{FBAppId}</string>
    <key>FacebookDisplayName</key>
    <string>{FBAppName}</string>
    <key>LSApplicationQueriesSchemes</key>
    <array>
        <string>fbapi</string>
        <string>fb-messenger-api</string>
        <string>fbauth2</string>
        <string>fbshareextension</string>
    </array>
<key>NSAppTransportSecurity</key>
<dict>
    <key>NSExceptionDomains</key>
    <dict>
        <key>facebook.com</key>
        <dict>
            <key>NSIncludesSubdomains</key>
            <true/>
            <key>NSExceptionRequiresForwardSecrecy</key>
            <false/>
        </dict>
        <key>fbcdn.net</key>
        <dict>
            <key>NSIncludesSubdomains</key>
            <true/>
            <key>NSExceptionRequiresForwardSecrecy</key>
            <false/>
        </dict>
        <key>akamaihd.net</key>
        <dict>
            <key>NSIncludesSubdomains</key>
            <true/>
            <key>NSExceptionRequiresForwardSecrecy</key>
            <false/>
        </dict>
    </dict>
</dict>
</plist>
