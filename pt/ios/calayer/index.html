Adicionando Transformações a um CALayer (traduzir, girar, dimensionar) #

Fundamentos #

Existem várias transformações diferentes que você pode fazer em uma camada, mas as básicas são

  • traduzir (mover)
  • escala
  • girar

Para fazer transformações em um CALayer, você define a propriedade transform da camada para um tipo CATransform3D. Por exemplo, para traduzir uma camada, você faria algo assim:

myLayer.transform = CATransform3DMakeTranslation(20, 30, 0)

A palavra Make é usada no nome para criar a transformação inicial: CATransform3DMakeTranslation. As transformações subsequentes que são aplicadas omitem o Make. Veja, por exemplo, esta rotação seguida de uma tradução:

let rotation = CATransform3DMakeRotation(CGFloat(30.0 * M_PI / 180.0), 20, 20, 0)
myLayer.transform = CATransform3DTranslate(rotation, 20, 30, 0)

Agora que temos a base de como fazer uma transformação, vamos ver alguns exemplos de como fazer cada uma. Primeiro, porém, mostrarei como configuro o projeto caso você queira brincar com ele também.

Configurar #

Para os exemplos a seguir, configurei um aplicativo de exibição única e adicionei um UIView com um fundo azul claro ao storyboard. Eu conectei a visão ao controlador de visão com o seguinte código:

import UIKit

class ViewController: UIViewController {
    
    var myLayer = CATextLayer()
    @IBOutlet weak var myView: UIView!
    
    override func viewDidLoad() {
        super.viewDidLoad()
        
        // setup the sublayer
        addSubLayer()
        
        // do the transform
        transformExample()
    }
    
    func addSubLayer() {
        myLayer.frame = CGRect(x: 0, y: 0, width: 100, height: 40)
        myLayer.backgroundColor = UIColor.blueColor().CGColor
        myLayer.string = "Hello"
        myView.layer.addSublayer(myLayer)
    }
    
    //******** Replace this function with the examples below ********

    func transformExample() {
        
        // add transform code here ...
        
        
    }

}

Existem muitos tipos diferentes de CALayer, mas optei por usar CATextLayer para que as transformações fiquem mais claras visualmente.

Traduzir #

A transformação de tradução move a camada. A sintaxe básica é

CATransform3DMakeTranslation(tx: CGFloat, ty: CGFloat, tz: CGFloat)

onde tx é a mudança nas coordenadas x, ty é a mudança em y e tz é a mudança em z.

Exemplo

No iOS a origem do sistema de coordenadas está no canto superior esquerdo, então se quiséssemos mover a camada 90 pontos para a direita e 50 pontos para baixo, faríamos o seguinte:

myLayer.transform = CATransform3DMakeTranslation(90, 50, 0)

Notas

  • Lembre-se que você pode colar isso no método transformExample() no código do projeto acima.
  • Já que vamos lidar apenas com duas dimensões aqui, tz é definido como 0.
  • A linha vermelha na imagem acima vai do centro do local original até o centro do novo local. Isso porque as transformações são feitas em relação ao ponto de ancoragem e o ponto de ancoragem por padrão está no centro da camada.

Escala #

A transformação de escala estica ou comprime a camada. A sintaxe básica é

CATransform3DMakeScale(sx: CGFloat, sy: CGFloat, sz: CGFloat)

onde sx, sy e sz são os números pelos quais dimensionar (multiplicar) as coordenadas x, y e z, respectivamente.

Exemplo

Se quiséssemos metade da largura e triplicar a altura, faríamos o seguinte

myLayer.transform = CATransform3DMakeScale(0.5, 3.0, 1.0)

Notas

  • Como estamos trabalhando apenas em duas dimensões, apenas multiplicamos as coordenadas z por 1,0 para deixá-las inalteradas.
  • O ponto vermelho na imagem acima representa o ponto de ancoragem. Observe como é feito o dimensionamento em relação ao ponto de ancoragem. Ou seja, tudo é esticado em direção ou para longe do ponto de ancoragem.

Girar #

A transformação de rotação gira a camada ao redor do ponto de ancoragem (o centro da camada por padrão). A sintaxe básica é

CATransform3DMakeRotation(angle: CGFloat, x: CGFloat, y: CGFloat, z: CGFloat)

onde angle é o ângulo em radianos que a camada deve ser girada e x, y e z são os eixos sobre os quais girar. Definir um eixo como 0 cancela uma rotação em torno desse eixo específico.

Exemplo

Se quiséssemos girar uma camada no sentido horário 30 graus, faríamos o seguinte:

let degrees = 30.0
let radians = CGFloat(degrees * M_PI / 180)
myLayer.transform = CATransform3DMakeRotation(radians, 0.0, 0.0, 1.0)

Notas

  • Como estamos trabalhando em duas dimensões, queremos apenas que o plano xy seja girado em torno do eixo z. Assim, configuramos x e y para 0.0 e configuramos z para 1.0.
  • Isso girou a camada no sentido horário. Poderíamos ter girado no sentido anti-horário configurando z para -1.0.
  • O ponto vermelho mostra onde está o ponto de ancoragem. A rotação é feita em torno do ponto de ancoragem.

Várias transformações #

Para combinar várias transformações, poderíamos usar concatinação como esta

CATransform3DConcat(a: CATransform3D, b: CATransform3D)

No entanto, vamos apenas fazer um após o outro. A primeira transformação usará o Make em seu nome. As transformações a seguir não usarão Make, mas usarão a transformação anterior como parâmetro.

Exemplo

Desta vez, combinamos todas as três transformações anteriores.

let degrees = 30.0
let radians = CGFloat(degrees * M_PI / 180)

// translate
var transform = CATransform3DMakeTranslation(90, 50, 0)

// rotate
transform = CATransform3DRotate(transform, radians, 0.0, 0.0, 1.0)

// scale
transform = CATransform3DScale(transform, 0.5, 3.0, 1.0)

// apply the transforms
myLayer.transform = transform

Notas

  • A ordem em que as transformações são feitas nas matérias.
  • Tudo foi feito em relação ao ponto de ancoragem (ponto vermelho).

Uma nota sobre o ponto de ancoragem e a posição #

Fizemos todas as nossas transformações acima sem alterar o ponto de ancoragem. Às vezes é necessário alterá-lo, como se você quiser girar em torno de algum outro ponto além do centro. No entanto, isso pode ser um pouco complicado.

O ponto de ancoragem e a posição estão no mesmo lugar. O ponto de ancoragem é expresso como uma unidade do sistema de coordenadas da camada (o padrão é 0,5, 0,5) e a posição é expressa no sistema de coordenadas da supercamada. Eles podem ser definidos assim

myLayer.anchorPoint = CGPoint(x: 0.0, y: 1.0)
myLayer.position = CGPoint(x: 50, y: 50)

Se você definir apenas o ponto de ancoragem sem alterar a posição, o quadro será alterado para que a posição fique no local correto. Ou mais precisamente, o quadro é recalculado com base no novo ponto de ancoragem e na posição antiga. Isso geralmente dá resultados inesperados. Visualização do emissor com imagem personalizada #

Por exemplo, vamos criar uma visão que contém a camada do emissor e anima as partículas.

import QuartzCore

class ConfettiView: UIView {
    // main emitter layer
    var emitter: CAEmitterLayer!

    // array of color to emit
    var colors: [UIColor]!

    // intensity of appearance
    var intensity: Float!
    
    private var active :Bool!
    
    required init?(coder aDecoder: NSCoder) {
        super.init(coder: aDecoder)
        setup()
    }
    
    override init(frame: CGRect) {
        super.init(frame: frame)
        setup()
    }
    
    func setup() {
        // initialization
        colors = [UIColor.redColor(),
                  UIColor.greenColor(),
                  UIColor.blueColor()
                  ]
        intensity = 0.2
        
        active = false
    }
    
    func startConfetti() {
        emitter = CAEmitterLayer()
        
        emitter.emitterPosition = CGPoint(x: frame.size.width / 2.0, y: -20)
        emitter.emitterShape = kCAEmitterLayerLine
        emitter.emitterSize = CGSize(width: frame.size.width, height: 1)
        
        var cells = [CAEmitterCell]()
        for color in colors {
            cells.append(confettiWithColor(color))
        }
        
        emitter.emitterCells = cells
        layer.addSublayer(emitter)
        active = true
    }
    
    func stopConfetti() {
        emitter?.birthRate = 0
        active = false
    }
    
    func confettiWithColor(color: UIColor) -> CAEmitterCell {
        let confetti = CAEmitterCell()

        confetti.birthRate = 10.0 * intensity
        confetti.lifetime = 180.0 * intensity
        confetti.lifetimeRange = 0
        confetti.color = color.CGColor
        confetti.velocity = CGFloat(350.0 * intensity)
        confetti.velocityRange = CGFloat(40.0 * intensity)
        confetti.emissionLongitude = CGFloat(M_PI)
        confetti.emissionRange = CGFloat(M_PI_4)
        confetti.spin = CGFloat(3.5 * intensity)
        confetti.spinRange = CGFloat(4.0 * intensity)
        
        // WARNING: A layer can set this property to a CGImageRef to display the image as its contents.
        confetti.contents = UIImage(named: "confetti")?.CGImage
        return confetti
    }
    
    internal func isActive() -> Bool {
        return self.active
    }
}

Você precisa adicionar a imagem “confetti” ou definir rect com confetti.contentsRect

Sombras #

Você pode usar 5 propriedades em cada camada para configurar suas sombras:

  • shadowOffset - esta propriedade move sua sombra para a esquerda/direita ou para cima/baixo
self.layer.shadowOffset = CGSizeMake(-1, -1); // 1px left and up

self.layer.shadowOffset = CGSizeMake(1, 1); // 1px down and right
  • shadowColor - isso define a cor da sua sombra
self.layer.shadowColor = [UIColor blackColor].CGColor;
  • shadowOpacity - esta é a opacidade da sombra, de 0 a 1
self.layer.shadowOpacity = 0.2;
  • shadowRadius - este é o raio de desfoque (equivalente à propriedade de desfoque no Sketch ou no Photoshop)
self.layer.shadowRadius = 6;
  • shadowPath - esta é uma propriedade importante para o desempenho, quando o iOS desabilitado baseia a sombra no canal alfa da visualização, que pode ser intensivo em desempenho com um PNG complexo com alfa. Essa propriedade permite que você force uma forma para sua sombra e tenha mais desempenho por causa disso.

Objetivo-C

self.layer.shadowPath = [UIBezierPath bezierPathWithOvalInRect:CGRectMake(0,0,100,100)]; //this does a circular shadow

Rápido 3

self.layer.shadowPath = UIBezierPath(ovalIn: CGRect(x: 0, y: 0, width: 100, height: 100)).cgPath

Como adicionar uma UIImage a um CALayer #

Você pode adicionar uma imagem à camada de uma visualização simplesmente usando sua propriedade contents:

myView.layer.contents = UIImage(named: "star")?.CGImage
  • Observe que o UIImage precisa ser convertido em um CGImage.

Se você deseja adicionar a imagem em sua própria camada, você pode fazer assim:

let myLayer = CALayer()
let myImage = UIImage(named: "star")?.CGImage
myLayer.frame = myView.bounds
myLayer.contents = myImage
myView.layer.addSublayer(myLayer)

Como você pode ver, porém, parece pixelizado. Isso ocorre porque o UIImage é menor que o UIView, então está sendo dimensionado para preencher a visualização, que é o padrão se você não especificar mais nada.

Desativar animações #

As animações da propriedade CALayer são habilitadas por padrão. Quando isso for indesejável, eles podem ser desabilitados da seguinte maneira.

Rápido

CATransaction.begin()
CATransaction.setDisableActions(true)

// change layer properties that you don't want to animate

CATransaction.commit()

Objetivo-C

[CATransaction begin];
[CATransaction setDisableActions:YES];

// change layer properties that you don't want to animate

[CATransaction commit];

Cantos arredondados #

layer.masksToBounds = true;
layer.cornerRadius = 8;

Criando um CALayer #

Você pode criar um CALayer e definir seu quadro assim:

Rápido:

let layer = CALayer()
layer.frame = CGRect(x: 0, y: 0, width: 60, height: 80)

Objetivo-C:

CALayer *layer = [[CALayer alloc] init];
layer.frame = CGRectMake(0, 0, 60, 80);

Você pode então adicioná-lo como uma subcamada a um CALayer existente:

Rápido:

existingLayer.addSublayer(layer)

Objetivo-C:

[existingLayer addSublayer:layer];

Observação:

Para fazer isso, você precisa incluir a estrutura QuartzCore.

Rápido:

 @import QuartzCore

Objetivo-C

#import <QuartzCore/QuartzCore.h>
