Sintaxe #

  1. // Objetivo-C

  2. [UIView new] // Obtém um objeto de visualização alocado e inicializado

  3. [[UIView alloc] initWithFrame:(Pass CGRect)] // Obtém a visualização alocada e inicializada com um quadro

  4. [[UIView alloc] init] // Obtém um objeto de visualização alocado e inicializado

  5. // Rápido

  6. UIView() // Cria uma instância UIView com o quadro CGRect.zero

  7. UIView(frame: CGRect) // Cria uma instância UIView especificando o quadro

  8. UIView.addSubview(UIView) // adiciona outra instância de UIView como subview

  9. UIView.hidden // Obtém ou define a visibilidade da visualização

  10. UIView.alpha // Obtém ou define a opacidade da visualização

  11. UIView.setNeedsLayout() // Força a visualização a atualizar seu layout

A classe UIView define uma área retangular na tela e as interfaces para gerenciar o conteúdo dessa área. Em tempo de execução, um objeto de exibição manipula a renderização de qualquer conteúdo em sua área e também manipula quaisquer interações com esse conteúdo.

Faça a visão arredondada #

Para fazer uma UIView arredondada, especifique um cornerRadius para a camada da visualização.

Isso também se aplica a qualquer classe que herda de UIView, como UIImageView.

Programaticamente #

Código Swift

someImageView.layoutIfNeeded()
someImageView.clipsToBounds = true
someImageView.layer.cornerRadius = 10

Código Objetivo-C

[someImageView layoutIfNeeded];
someImageView.clipsToBounds = YES;
someImageView.layer.cornerRadius = 10;

Exemplo

//Swift code
topImageView.layoutIfNeeded()
bottomImageView.layoutIfNeeded()
topImageView.clipsToBounds = true
topImageView.layer.cornerRadius = 10
bottomImageView.clipsToBounds = true
bottomImageView.layer.cornerRadius = bottomImageView.frame.width / 2

//Objective-C code
[topImageView layoutIfNeeded]
[bottomImageView layoutIfNeeded];
topImageView.clipsToBounds = YES;
topImageView.layer.cornerRadius = 10;
bottomImageView.clipsToBounds = YES;
bottomImageView.cornerRadius = CGRectGetWidth(bottomImageView.frame) / 2;

Aqui está o resultado, mostrando o efeito de vista arredondada usando o raio de canto especificado:

![Exemplo 1]

Observação

Para fazer isso, você precisa incluir a estrutura QuartzCore.

 #import <QuartzCore/QuartzCore.h>

Configuração do storyboard #

Um efeito de visualização arredondada também pode ser obtido ’não programaticamente’ definindo as propriedades correspondentes no Storyboard.

Como as propriedades layer não são expostas no Storyboard, você deve modificar o atributo cornerRadius por meio da seção Atributos de tempo de execução definidos pelo usuário.

Extensão Swift #

Você pode usar esta extensão útil para aplicar a visualização arredondada, desde que tenha a mesma largura e altura.

extension UIView {
    @discardableResult
    public func setAsCircle() -> Self {
        self.clipsToBounds = true
        let frameSize = self.frame.size
        self.layer.cornerRadius = min(frameSize.width, frameSize.height) / 2.0
        return self
    }
}

Para usá-lo:

yourView.setAsCircle()

Usando IBInspectable e IBDesignable #

Um (ou dois) dos novos recursos mais legais nas versões recentes do Xcode são as propriedades IBInspectable e IBDesignable UIViews. Isso não tem nada a ver com a funcionalidade do seu aplicativo, mas afeta a experiência do desenvolvedor no Xcode. O objetivo é poder inspecionar visualmente visualizações personalizadas em seu aplicativo iOS sem executá-lo. Portanto, suponha que você tenha uma visualização personalizada com o nome criativo CustomView que herda de UIView. Nesta exibição personalizada, ele exibirá uma sequência de texto com uma cor designada. Você também pode optar por não exibir nenhum texto. Vamos precisar de três propriedades:

var textColor: UIColor = UIColor.blackColor()
var text: String?
var showText: Bool = true

Podemos então substituir a função drawRect na classe:

if showText {
    if let text = text {
        let s = NSString(string: text)
        s.drawInRect(rect,
            withAttributes: [
                NSForegroundColorAttributeName: textColor,
                NSFontAttributeName: UIFont(name: "Helvetica Neue", size: 18)!
            ])
    }
}

Supondo que a propriedade text esteja definida, isso desenhará uma string no canto superior esquerdo da visualização quando o aplicativo for executado. O problema é que não saberemos como é sem executar o aplicativo. É aqui que entram o IBInspectable e o IBDesignable. O IBInspectable nos permite definir visualmente os valores das propriedades da visão no Xcode, assim como com os controles embutidos. IBDesignable nos mostrará uma prévia visual no storyboard. Veja como a classe deve ficar:

@IBDesignable
class CustomView: UIView {
    @IBInspectable var textColor: UIColor = UIColor.blackColor()
    @IBInspectable var text: String?
    @IBInspectable var showText: Bool = true

    override func drawRect(rect: CGRect) {
        // ...
    }
}

Ou no Objetivo C:

IB_DESIGNABLE
@interface CustomView: UIView

@property (nonatomic, strong) IBInspectable UIColor* textColor;
@property (nonatomic, strong) IBInspectable NSString* text;
@property (nonatomic, assign) IBInspectable BOOL showText;

@end

@implementation CustomView

- (instancetype)init {
    if(self = [super init]) {
        self.textColor = [UIColor blackColor];
        self.showText = YES;
    }
    return self;
}

- (void)drawRect:(CGRect)rect {
    //...
}

@end

As próximas capturas de tela mostram o que acontece no Xcode. A primeira é o que acontece depois de adicionar a classe revisada. Observe que há três novos elementos de interface do usuário para as três propriedades. O Text Color exibirá um seletor de cores, Text é apenas uma caixa de entrada e Show Text nos dará as opções para Off e On que são false e ​​true respectivamente.

O próximo é depois de alterar a Cor do Texto para vermelho usando o seletor de cores. Além disso, algum texto foi fornecido para fazer a função drawRect exibi-lo. Observe que a visualização no Interface Builder também foi atualizada.

Finalmente, definir Show Text como Off no inspetor de propriedades faz com que a exibição de texto no Interface Builder desapareça.

No entanto, todos nós encontramos situações em que precisamos criar UIView arredondado em várias visualizações em seu Storyboard. Em vez de declarar IBDesignable para todas as visualizações de Storyboard, é melhor criar uma Extensão de UIView e obtenha uma interface de usuário construída apenas para cada UIView através do projeto para criar uma visão arredondada definindo o raio de canto. Um raio de borda configurável em qualquer UIView que você criar no storyboard.

extension UIView {
    
    @IBInspectable var cornerRadius:CGFloat {
        set {
            layer.cornerRadius = newValue
            clipsToBounds = newValue > 0
        }
        get {
            return layer.cornerRadius
        }
    }
    
}

Utilizando o tamanho do conteúdo intrínseco #

Ao criar uma subclasse UIView, o tamanho do conteúdo intrínseco ajuda a evitar a configuração de restrições de altura e largura codificadas

um vislumbre básico de como uma classe pode utilizar este

class ImageView: UIView {
    var image: UIImage {
        didSet {
            invalidateIntrinsicContentSize()
        }
    }
    // omitting initializers
    // convenience init(image: UIImage)

    override func intrinsicContentSize() -> CGSize {
        return CGSize(width: image.size.width, height: image.size.height)
    }
}

Se você deseja fornecer apenas um tamanho intrinsecamente, pode fornecer o valor UIViewNoIntrinsicMetric para o valor que deseja ignorar.

 override func intrinsicContentSize() -> CGSize {
     return CGSize(width: UIViewNoIntrinsicMetric, height: image.size.width)   
 }

Benefícios ao usar com o AutoLayout e o Interface Builder

Pode-se pegar este ImageView (ou UIImageView) e definir o alinhamento horizontal para o centro de supervisão X e o alinhamento vertical para o centro de supervisão Y.

O construtor de interface irá reclamar com você neste momento dando o seguinte aviso:

É aqui que entra o Placeholder Intrinsic Size.

Indo para o painel do inspetor de tamanho e até o menu suspenso Tamanho intrínseco, você pode alternar esse valor de Padrão para Espaço reservado.

e agora o construtor de interface removerá os avisos anteriores e você pode usar esse tamanho para ter visualizações dimensionadas dinamicamente no construtor de interface.

Gerencie programaticamente a inserção e exclusão de UIView em e de outro UIView #

Suponha que você tenha um parentView no qual deseja inserir um novo subView programaticamente (por exemplo, quando você deseja inserir um UIImageView na visualização de um UIViewController), então você pode fazer como abaixo.

Objetivo-C

[parentView addSubview:subView];

Rápido

parentView.addSubview(subView)

Você também pode adicionar a subView abaixo de outra subView2, que já é uma subview de parentView usando o seguinte código:

Objetivo-C

[parentView insertSubview:subView belowSubview:subView2];

Rápido

parentView.insertSubview(subView, belowSubview: subView2)

Se você quiser inseri-lo acima de subView2, você pode fazer assim:

Objetivo-C

[parentView insertSubview:subView aboveSubview:subView2];

Rápido

parentView.insertSubview(subView, aboveSubview: subView2)

Se em algum lugar do seu código você precisa trazer um certo subView para frente, então acima de todos os outros subviews do parentView, você pode fazer assim:

Objetivo-C

[parentView bringSubviewToFront:subView];

Rápido

parentView.bringSubviewToFront(subView)

Finalmente, se você deseja remover subView de parentView, você pode fazer como abaixo:

Objetivo-C

[subView removeFromSuperview];

Rápido

subView.removeFromSuperview()

Extensão UIView para atributos de tamanho e frame #

Se quisermos obter a coordenada x de origem da visão, precisamos escrever como:

view.frame.origin.x

Para largura, precisamos escrever:

view.frame.size.width

Mas se adicionarmos uma extensão simples a um UIView, podemos obter todos os atributos de forma muito simples, como:

view.x
view.y
view.width
view.height

Também ajudará a definir esses atributos como:

view.x = 10
view.y = 10
view.width = 100
view.height = 200

E a extensão simples seria:

extension UIView {
    
    var x: CGFloat {
        get {
            return self.frame.origin.x
        }
        set {
            self.frame = CGRect(x: newValue, y: self.frame.origin.y, width: self.frame.size.width, height: self.frame.size.height)
        }
    }
    
    var y: CGFloat {
        get {
            return self.frame.origin.y
        }
        set {
            self.frame = CGRect(x: self.frame.origin.x, y: newValue, width: self.frame.size.width, height: self.frame.size.height)
        }
    }
    
    var width: CGFloat {
        get {
            return self.frame.size.width
        }
        set {
            self.frame = CGRect(x: self.frame.origin.x, y: self.frame.origin.y, width: newValue, height: self.frame.size.height)
        }
    }
    
    var height: CGFloat {
        get {
            return self.frame.height
        }
        set {
            self.frame = CGRect(x: self.frame.origin.x, y: self.frame.origin.y, width: self.frame.size.width, height: newValue)
        }
    }
}

Precisamos adicionar este arquivo de classe em um projeto e ele estará disponível para uso em todo o projeto!

UIView *view = [[UIView alloc] init];

Funções

Função para adicionar visualização com altura fixa usando restrições de layout automático

-(void)addView:(UIView*)subView onParentView:(UIView*)parentView withHeight:(CGFloat)height{

subView.translatesAutoresizingMaskIntoConstraints = NO;
    
NSLayoutConstraint *trailing =[NSLayoutConstraint
                                   constraintWithItem:subView
                                   attribute:NSLayoutAttributeTrailing
                                   relatedBy:NSLayoutRelationEqual
                                   toItem:parent
                                   attribute:NSLayoutAttributeTrailing
                                   multiplier:1.0
                                   constant:10.f];


NSLayoutConstraint *top = [NSLayoutConstraint
                               constraintWithItem:subView
                               attribute:NSLayoutAttributeTop
                               relatedBy:NSLayoutRelationEqual
                               toItem:parent
                               attribute:NSLayoutAttributeTop
                               multiplier:1.0
                               constant:10.f];
    
NSLayoutConstraint *leading = [NSLayoutConstraint
                                   constraintWithItem:subView
                                   attribute:NSLayoutAttributeLeading
                                   relatedBy:NSLayoutRelationEqual
                                   toItem:parent
                                   attribute:NSLayoutAttributeLeading
                                   multiplier:1.0
                                   constant:10.f];
[parent addConstraint:trailing];
[parent addConstraint:top];
[parent addConstraint:leading];

NSLayoutConstraint *heightConstraint =[NSLayoutConstraint
                                           constraintWithItem:subView
                                           attribute:NSLayoutAttributeHeight
                                           relatedBy:NSLayoutRelationEqual
                                           toItem:nil
                                           attribute:0
                                           multiplier:0.0
                                           constant:height];
    
[subView addConstraint:heightConstraint];

}

                                   constraintWithItem:subView
                                   attribute:NSLayoutAttributeLeading
                                   relatedBy:NSLayoutRelationEqual
                                   toItem:parent
                                   attribute:NSLayoutAttributeLeading
                                   multiplier:1.0

                                 toItem:parent
