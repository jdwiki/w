Criando um aplicativo de exibição única padrão #

Para desenvolver um aplicativo para iOS, você deve começar com um aplicativo chamado Xcode. Existem outras ferramentas alternativas que você pode usar, mas o Xcode é a ferramenta oficial da Apple. Observe, no entanto, que ele é executado apenas no macOS. A versão oficial mais recente é o Xcode 8.3.3 com o Xcode 9 (atualmente em beta) que deve ser lançado ainda este ano.

  1. Inicialize seu Mac e instale o Xcode da App Store se ainda não estiver instalado.

(Se preferir não usar a App Store ou tiver problemas, você também pode baixar o Xcode do site do desenvolvedor da Apple, mas certifique-se de selecionar a versão de lançamento mais recente e não uma versão beta. )

  1. Abra o Xcode. A seguinte janela será aberta:

A janela apresenta as seguintes opções:

  • Iniciando com um playground: Isso foi introduzido com a linguagem Swift e o Xcode 6. É uma área interativa que pode ser usada para escrever pequenos trechos de código para verificar mudanças no tempo de execução. É uma ótima maneira de os alunos do Swift serem apresentados aos novos recursos do Swift.
  • Crie um novo projeto Xcode: Escolha esta opção, que cria um novo projeto com configuração padrão.
  • Verificar um projeto existente: Isso é usado para fazer check-out de um projeto de um local de repositório, por exemplo, fazer check-out de um projeto do SVN.
  1. Selecione a segunda opção Criar um novo projeto Xcode e o Xcode solicitará que você faça uma configuração inicial do projeto:

Este assistente é usado para selecionar seu modelo de projeto. Existem 5 opções:

  • iOS: usado para criar aplicativos, bibliotecas e estruturas para iOS
  • watchOS: usado para criar aplicativos, bibliotecas e estruturas do watchOS
  • tvOS: usado para criar aplicativos, bibliotecas e estruturas do tvOS
  • macOS: Usado para criar aplicativos, bibliotecas, estruturas, pacotes, AppleScripts do macOS, etc.
  • Plataforma cruzada: usado para criar aplicativos, modelos e conteúdo de compra no aplicativo multiplataforma

Você pode ver que há muitos modelos diferentes para seu aplicativo. Esses modelos são úteis para impulsionar seu desenvolvimento; eles são pré-construídos com algumas configurações básicas do projeto, como interfaces de interface do usuário e arquivos de classe.

Aqui, usaremos a primeira opção, iOS.

  1. Aplicação Mestre-Detalhe:

    This template contains a combined master and detail interface: the master contains objects which are related to the detail interface. Selecting objects in the master will change the details interface. You can see this kind UI in the Settings, Notes and Contacts applications on the iPad.

  2. Aplicativo baseado em página:

    This template is used to create the page-based application. Pages are different views held by one container.

  3. Aplicativo de Visualização Única:

    This is a normal application development template. This is good for beginners to learn application flow.

  4. Aplicativo com guias:

    This template creates tabs at the bottom part of an application. Each tab has a different UI and a different navigation flow. You can see this template used in apps like Clock, iTunes Store, iBooks and App Store.

  5. Jogo:

    This is a starting point for game development. You can go further with game technologies like SceneKit, SpriteKit, OpenGL ES and Metal.

  6. Neste exemplo, começaremos com Single View Application

O assistente ajuda você a definir as propriedades do projeto:

  • Nome do Produto: O nome do projeto/aplicativo
  • Nome da organização: o nome da organização na qual você está envolvido
  • Identificador da organização: o identificador exclusivo da organização usado no identificador do pacote. Recomenda-se seguir a notação de serviço de nome de domínio reverso.
  • Identificador do pacote: Este campo é muito importante. É baseado no nome do seu projeto e no identificador da organização, escolha sabiamente. O identificador do pacote será usado no futuro para instalar o aplicativo em um dispositivo e fazer upload do aplicativo para o iTunes Connect (que é o local onde enviamos aplicativos para serem publicados na App Store). É uma chave exclusiva para identificar seu aplicativo.
  • Idioma: a linguagem de programação que você gostaria de usar. Aqui você pode alterar Objective-C para Swift se não estiver selecionado.
  • Dispositivos: dispositivos compatíveis com seu aplicativo que podem ser alterados posteriormente. Mostra iPhone, iPad e Universal. Os aplicativos universais são compatíveis com dispositivos iPhone e iPad, e é recomendável selecionar esta opção quando não for necessário executar o aplicativo em apenas um tipo de dispositivo.
  • Use Core Data: Se você quiser usar o Core Data Model em seu projeto, marque-o como selecionado e ele criará um arquivo para o .xcdatamodel. Você também pode adicionar este arquivo mais tarde, se não souber com antecedência.
  • Incluir testes de unidade: isso configura o destino do teste de unidade e cria classes para teste de unidade
  • Incluir teste de interface do usuário: isso configura o destino do teste de interface do usuário e cria classes para teste de interface do usuário

Clique em Next e ele solicitará um local onde você deseja criar o diretório do projeto. Clique em Criar e você verá a interface do usuário do Xcode com uma configuração de projeto já definida. Você pode ver algumas classes e arquivos do Storyboard.

Este é um modelo básico para um aplicativo de exibição única.

Interface Xcode #

No Xcode, você tem três áreas separadas de trabalho - Navigators (em vermelho), Debug area (em verde) e Utilities (em azul).

A janela da área de trabalho sempre inclui a área do editor. Quando você seleciona um arquivo em seu projeto, seu conteúdo aparece na área do editor, onde o Xcode abre o arquivo em um editor apropriado. Por exemplo, na imagem acima, a área do editor MainViewController.swift, um arquivo de código rápido que é selecionado na área Navigator à esquerda da janela da área de trabalho.

Área do Navegador #

A janela do navegador contém as oito opções a seguir:

  • Navegador de projetos. Adicione, exclua, agrupe e gerencie arquivos em seu projeto ou escolha um arquivo para visualizar ou editar seu conteúdo na área do editor.
  • Navegador de símbolos. Navegue pelos símbolos em seu projeto como uma lista ou hierarquia. Os botões à esquerda da barra de filtro permitem limitar os símbolos mostrados a uma combinação de apenas classes e protocolos, apenas símbolos em seu projeto ou apenas contêineres.
  • Localizar navegador Use opções de pesquisa e filtros para encontrar rapidamente qualquer string em seu projeto.
  • Navegador de problemas. Visualize problemas como diagnósticos, avisos e erros encontrados ao abrir, analisar e criar seu projeto.
  • Navegador de testes. Crie, gerencie, execute e revise testes de unidade.
  • Navegador de depuração. Examine os threads em execução e as informações de pilha associadas em um ponto ou tempo especificado durante a execução do programa.
  • Navegador de ponto de interrupção. Ajuste os pontos de interrupção especificando características como condições de disparo.
  • Navegador de relatórios. Visualize o histórico de suas tarefas de compilação, execução, depuração, integração contínua e controle de origem.

Os Editores #

A maior parte do trabalho de desenvolvimento no Xcode ocorre na área do editor, a área principal que está sempre visível na janela do espaço de trabalho. Os editores que você usa com mais frequência são:

  • Editor de código-fonte. Escreva e edite o código-fonte.
  • Interface Builder. Crie e edite graficamente os arquivos da interface do usuário.
  • Editor de projetos. Visualize e edite como seus aplicativos devem ser criados, por exemplo, especificando opções de compilação, arquiteturas de destino e direitos de aplicativos.

Configure a área do editor para uma determinada tarefa com os botões de configuração do editor no lado direito da barra de ferramentas:

  • Editor padrão. Preenche a área do editor com o conteúdo do arquivo selecionado.
  • Editor Assistente. Apresenta um painel de editor separado com conteúdo logicamente relacionado ao conteúdo do painel de editor padrão. Você também pode alterar o conteúdo.
  • Version Editor. Mostra as diferenças entre o arquivo selecionado em um painel e outra versão desse mesmo arquivo em um segundo painel. Este editor funciona apenas quando seu projeto está sob controle de origem.

Recursos e Elementos na Área de Utilidades #

A área de utilitários na extremidade direita da janela do espaço de trabalho oferece acesso rápido a estes recursos: Inspetores, para visualizar e modificar características do arquivo aberto em um editor Bibliotecas de recursos prontos para uso em seu projeto

O painel superior da área de utilitários exibe inspetores. O painel inferior dá acesso a bibliotecas.

O primeiro painel (destacado em vermelho) é a barra do Inspetor, use-a para escolher o inspetor mais adequado à sua tarefa atual. Dois inspetores estão sempre visíveis na barra de inspeção (inspetores adicionais estão disponíveis em alguns editores):

  • Inspetor de arquivos. Visualize e gerencie metadados para o arquivo selecionado. Normalmente, você localizará storyboards e outros arquivos de mídia e alterará as configurações dos arquivos da interface do usuário.
  • Ajuda Rápida. Visualize detalhes sobre um símbolo, um elemento de interface ou uma configuração de compilação no arquivo. Por exemplo, a Ajuda Rápida exibe uma descrição concisa de um método, onde e como o método é declarado, seu escopo, os parâmetros que ele usa e sua disponibilidade de plataforma e arquitetura.

Use a barra da biblioteca (a segunda destacada em vermelho) para acessar bibliotecas de recursos prontas para uso em seu projeto:

  • Modelos de arquivo. Modelos para tipos comuns de arquivos e construções de código.
  • Snippets de código. Pequenos trechos de código-fonte para uso em seu software, como declarações de classe, fluxos de controle, declarações de bloco e modelos para tecnologias Apple comumente usadas.
  • Objetos. Itens para a interface do usuário do seu aplicativo.
  • Mídia. Arquivos contendo gráficos, ícones, arquivos de som e similares.

Para usar uma biblioteca, arraste-a diretamente para a área apropriada. Por exemplo, para usar um trecho de código, arraste-o da biblioteca para o editor de origem; para criar um arquivo de origem a partir de um modelo de arquivo, arraste seu modelo para o navegador do projeto.

Para restringir os itens exibidos em uma biblioteca selecionada, digite o texto relevante no campo de texto na barra de filtros (o painel inferior). Por exemplo, digite “botão” no campo de texto para mostrar todos os botões da biblioteca de objetos.

Gerenciar tarefas com a barra de ferramentas do espaço de trabalho #

A barra de ferramentas na parte superior da janela da área de trabalho fornece acesso rápido aos comandos usados ​​com frequência. O botão Executar cria e executa seus produtos. O botão Parar encerra seu código em execução. O menu Scheme permite configurar os produtos que você deseja criar e executar. O visualizador de atividades mostra o andamento das tarefas em execução exibindo mensagens de status, andamento da compilação e outras informações sobre seu projeto.

Os botões de configuração do editor (o primeiro grupo de três botões) permitem configurar a área do editor e os botões de configuração do espaço de trabalho (o segundo grupo de três botões) ocultam ou mostram o navegador opcional, depuração e áreas de utilidades.

O menu Visualizar inclui comandos para ocultar ou mostrar a barra de ferramentas.
