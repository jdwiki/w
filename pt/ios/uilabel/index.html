A classe UILabel implementa uma exibição de texto somente leitura. Você pode usar essa classe para desenhar uma ou várias linhas de texto estático, como aquelas que você pode usar para identificar outras partes da interface do usuário. A classe base UILabel fornece suporte para estilos simples e complexos do texto do rótulo. Você também pode controlar aspectos da aparência, como se o rótulo usa uma sombra ou desenha com realce. Se necessário, você pode personalizar ainda mais a aparência do seu texto por meio de subclasses.

Sintaxe #

  • UILabel.numberOfLines: Int // obtém ou define o número máximo de linhas que o rótulo pode ter. 0 é ilimitado
  • UILabel.text: String? // obtém ou define o texto que o rótulo exibe
  • UILabel.textColor: UIColor! // obtém ou define a cor do texto no rótulo
  • UILabel.tintColor: UIColor! // obtém ou define a cor da tonalidade do rótulo
  • UILabel.attributedText: NSAttributedString? // obtém ou define o texto atribuído do rótulo
  • UILabel.font: UIFont! // obtém ou define a fonte do texto no rótulo
  • UILabel.textAlignment: NSTextAlignment // obtém ou define o alinhamento do texto

UILabels são visualizações que podem ser usadas para exibir uma ou várias linhas de texto. Ele contém várias maneiras de estilizar o texto, como sombras, cores de texto e fontes.

UILabels também podem exibir Strings Atribuídas, que é texto + marcação embutida para aplicar estilos a partes do texto.

UILabel não está em conformidade com o protocolo UIAppearance, portanto, você não pode usar métodos de proxy UIAppearance para personalizar a aparência de UILabels. Veja esta discussão para mais informações.

Crie um UILabel #

Com um quadro #

Quando você souber as dimensões exatas que deseja definir para sua etiqueta, poderá inicializar um UILabel com um quadro CGRect.

Rápido #

let frame = CGRect(x: 0, y: 0, width: 200, height: 21)
let label = UILabel(frame: frame)
view.addSubview(label)

Objetivo-C #

CGRect frame = CGRectMake(0, 0, 200, 21);
UILabel *label = [[UILabel alloc] initWithFrame:frame];
[view addSubview:label];

Com layout automático #

Você pode adicionar restrições em um UILabel quando quiser que o iOS calcule dinamicamente seu quadro em tempo de execução.

Rápido #

let label = UILabel()
label.backgroundColor = .red
label.translatesAutoresizingMaskIntoConstraints = false
view.addSubview(label)

NSLayoutConstraint.activate([
    //stick the top of the label to the top of its superview:
    label.topAnchor.constraint(equalTo: view.topAnchor)

    //stick the left of the label to the left of its superview
    //if the alphabet is left-to-right, or to the right of its 
    //superview if the alphabet is right-to-left:
    label.leadingAnchor.constraint(equalTo: view.leadingAnchor)

    //stick the label's bottom to the bottom of its superview:
    label.bottomAnchor.constraint(equalTo: view.bottomAnchor)

    //the label's width should be equal to 100 points:
    label.widthAnchor.constraint(equalToConstant: 100)
])

Objetivo-C #

UILabel *label = [[UILabel alloc] init];

Com Objective-c + Visual Format Language (VFL) #

  UILabel *label = [UILabel new];
  label.translatesAutoresizingMaskIntoConstraints = NO;
  [self.view addSubview label];
  // add horizontal constraints with 5 left and right padding from the leading and trailing
 
  [self.view addConstraints:[NSLayoutConstraint constraintsWithVisualFormat:@"V:|-5-[labelName]-5-|"
                                                                    options:0
                                                                    metrics:nil
                                                                      views:@{@"labelName":label}]];
  // vertical constraints that will use the height of the superView with no padding on top and bottom
  [self.view addConstraints:[NSLayoutConstraint constraintsWithVisualFormat:@"H:|[labelName]|"
                                                                    options:0
                                                                    metrics:nil
                                                                      views:@{@"labelName":label}]]

Número de linhas #

Quando você cria um rótulo e define seu texto como mais de uma única linha que ele pode exibir, ele será truncado e você verá apenas uma linha de texto terminando com três pontos (…). Isso ocorre porque uma propriedade chamada numberOfLines está definida como 1 e, portanto, apenas uma linha será exibida. É um erro comum no manuseio de UILabels, e muitas pessoas pensam nisso como um bug, ou podem usar mais de um rótulo para mostrar mais de uma linha de texto, mas apenas editando esta propriedade, podemos dizer a um UILabel para aceitar até o número especificado de linhas. Por exemplo, se esta propriedade for definida como 5, o rótulo pode mostrar 1, 2, 3, 4 ou 5 linhas de dados.

Configurando o valor programaticamente #

Para definir essa propriedade, basta atribuir um novo inteiro a ela:

Rápido #

label.numberOfLines = 2

Objetivo-C #

label.numberOfLines = 2;

Nota #

É possível definir esta propriedade como 0. No entanto, isso não significa que ela não aceitará nenhuma linha, mas sim que o rótulo pode ter quantas linhas forem necessárias (também conhecido como “Infinito”):

Rápido #

label.numberOfLines = 0

Objetivo-C #

label.numberOfLines = 0;

Nota #

Se a etiqueta tiver uma restrição de altura, a restrição será respeitada. Neste caso, label.numberOfLines = 0 pode não funcionar como esperado.

Nota #

Para um texto de várias linhas mais complexo, UITextView pode ser mais adequado.*

Em vez de definir numberOfLines programaticamente, você pode usar um Storyboard ou um .xib e definir a propriedade numberOfLines. Dessa forma, obtemos os mesmos resultados que o código acima.

Calcular limites de conteúdo (por exemplo, alturas de células dinâmicas) #

Um caso de uso comum para calcular o quadro que um rótulo ocupará é dimensionar as células de exibição de tabela apropriadamente. A maneira recomendada de fazer isso é usando o método NSString boundingRectWithSize:options:attributes:context:.

options pega as opções de desenho de String:

  • NSStringDrawingUsesLineFragmentOrigin deve ser usado para rótulos com várias linhas
  • NSStringDrawingTruncatesLastVisibleLine deve ser adicionado usando o operador | se houver um número máximo de linhas

attributes é um NSDictionary de atributos que afetam as strings atribuídas (lista completa: Apple Docs), mas os fatores que afetam a altura incluem:

  • NSFontAttributeName: Muito importante, o tamanho e a família de fontes são uma parte crítica do tamanho exibido do rótulo.

  • NSParagraphStyleAttributeName: Para personalizar como o texto é exibido. Isso inclui espaçamento entre linhas, alinhamento de texto, estilo de truncamento e algumas outras opções. Se você não alterou explicitamente nenhum desses valores, não deve se preocupar muito com isso, mas pode ser importante se você alternou alguns valores em IB.

context deve ser nil já que o principal caso de uso NSStringDrawingContext é permitir que a fonte seja redimensionada para caber em um rect especificado, o que não deve ser o caso se estivermos calculando uma altura dinâmica.

Objetivo C

- (CGFloat)tableView:(UITableView *)tableView heightForRowAtIndexPath:(NSIndexPath *)indexPath {
    
    UITableViewCell *cell = [tableView cellForRowAtIndexPath:indexPath];

    NSString *labelContent = cell.theLabel.text;
    // you may choose to get the content directly from the data source if you have done minimal customizations to the font or are comfortable with hardcoding a few values
//    NSString *labelContent = [self.dataSource objectAtIndexPath:indexPath];
    
    // value may be hardcoded if retrieved from data source
    NSFont *labelFont = [cell.theLabel font];
    
    // The NSParagraphStyle, even if you did not code any changes these values may have been altered in IB
    NSMutableParagraphStyle *paragraphStyle = [NSMutableParagraphStyle new];
    paragraphStyle.lineBreakMode = NSLineBreakByWordWrapping; 
    paragraphStyle.alignment = NSTextAlignmentCenter;

    NSDictionary *attributes = @{NSFontAttributeName: labelFont,
                                 NSParagraphStyleAttributeName: paragraphStyle};

    // The width is also important to the height
    CGFloat labelWidth = CGRectGetWidth(cell.theLabel.frame);
    // If you have been hardcoding up to this point you will be able to get this value by subtracting the padding on left and right from tableView.bounds.size.width
//    CGFloat labelWidth = CGRectGetWidth(tableView.frame) - 20.0f - 20.0f;

    CGRect bodyBounds = [labelContent boundingRectWithSize:CGSizeMake(width, CGFLOAT_MAX) options:NSStringDrawingUsesLineFragmentOrigin attributes:attributes context:nil];

    return CGRectGetHeight(bodyBounds) + heightOfObjectsOnTopOfLabel + heightOfObjectBelowLabel;
}

Swfit 3

override func tableView(_ tableView: UITableView, heightForRowAt indexPath: IndexPath) -> CGFloat {
    var cell = tableView.cellForRow(atIndexPath: indexPath)!
    var labelContent = cell.theLabel.text
    var labelFont = cell.theLabel.font
    var paragraphStyle = NSMutableParagraphStyle()

    paragraphStyle.lineBreakMode = .byWordWrapping
    paragraphStyle.alignment = .center

    var attributes = [NSFontAttributeName: labelFont, NSParagraphStyleAttributeName: paragraphStyle]

    var labelWidth: CGFloat = cell.theLabel.frame.width

    var bodyBounds = labelContent.boundingRect(withSize: CGSize(width: width, height: CGFLOAT_MAX), options: .usesLineFragmentOrigin, attributes: attributes, context: nil)

    return bodyBounds.height + heightOfObjectsOnTopOfLabel + heightOfObjectBelowLabel
}

Por outro lado, se você tiver um número máximo de linhas definido, primeiro precisará calcular a altura de uma única linha para garantir que não obtenhamos um valor mais alto que o tamanho permitido:

    // We calculate the height of a line by omitting the NSStringDrawingUsesLineFragmentOrigin option, which will assume an infinitely wide label
    CGRect singleLineRect = [labelContent boundingRectWithSize:CGSizeMake(CGFLOAT_MAX, CGFLOAT_MAX)
                                                 options:NSStringDrawingTruncatesLastVisibleLine
                                                 context:nil];
    CGFloat lineHeight = CGRectGetHeight(singleLineRect);
    CGFloat maxHeight = lineHeight * cell.theLabel.numberOfLines;

    // Now you can call the method appropriately
    CGRect bodyBounds = [labelContent boundingRectWithSize:CGSizeMake(width, maxHeight) options:(NSStringDrawingUsesLineFragmentOrigin|NSStringDrawingTruncatesLastVisibleLine) attributes:attributes context:nil];

    return CGRectGetHeight(bodyBounds) + heightOfObjectsOnTopOfLabel + heightOfObjectBelowLabel;

Altura variável usando restrições #

Você pode fazer um UILabel com uma altura dinâmica usando o layout automático.

Você precisa definir o numberOfLines para zero (0), e adicionar uma altura mínima configurando uma restrição com uma relação do tipo .GreaterThanOrEqual no atributo .Height

Rápido #

label.numberOfLines = 0

let heightConstraint = NSLayoutConstraint(
    item: label,
    attribute: .Height,
    relatedBy: .GreaterThanOrEqual,
    toItem: nil,
    attribute: .NotAnAttribute,
    multiplier: 0,
    constant: 20
)

label.addConstraint(heightConstraint)

Rápido #

label.numberOfLines = 0
label.translatesAutoresizingMaskIntoConstraints = false
label.heightAnchor.constraintGreaterThanOrEqualToConstant(20).active = true

Constantes #

  • Quebra de palavras - a quebra ocorre nos limites da palavra, a menos que a palavra em si não caiba em uma única linha
  • Char Wrapping - a quebra ocorre antes do primeiro caractere que não cabe
  • Recorte - as linhas simplesmente não são desenhadas além da borda do contêiner de texto
  • Cabeça truncada - a linha é exibida de forma que o final caiba no contêiner e o texto ausente no início da linha é indicado por um glifo de reticências
  • Truncando a cauda - a linha é exibida de forma que o início caiba no contêiner e o texto ausente no final da linha é indicado por um glifo de reticências
  • Truncando no meio - a linha é exibida para que o início e o fim caibam no contêiner e o texto ausente no meio seja indicado por um glifo de reticências

Rótulo clicável #

OBSERVAÇÃO: Na maioria dos casos, é melhor usar um UIButton em vez de fazer um UILabel no qual você pode tocar. Use este exemplo apenas se tiver certeza de que não deseja usar um UIButton por algum motivo.

  1. Criar etiqueta
  2. Habilite a interação do usuário
  3. Adicione UITapGestureRecognizer

A chave para criar um UILabel clicável é permitir a interação do usuário.

Rápido #

let label = UILabel()
label.userInteractionEnabled = true

let gesture = UITapGestureRecognizer(target: self, action: #selector(labelClicked(_:)))
label.addGestureRecognizer(gesture)

Objetivo-C #

UILabel *label = [[UILabel alloc] init];
[label setUserInteractionEnabled:YES];

UITapGestureRecognizer* gesture = [[UITapGestureRecognizer alloc] initWithTarget:self action:@selector(labelClicked:)];
[label addGestureRecognizer:gesture];

Etiqueta de tamanho automático para caber no texto #

Este exemplo mostra como a largura de um rótulo pode ser redimensionada automaticamente quando o conteúdo do texto é alterado.

Fixe as bordas esquerda e superior #

Basta usar o layout automático para adicionar restrições para fixar os lados esquerdo e superior do rótulo.

Depois disso, ele será redimensionado automaticamente.

Notas #

  • Este exemplo vem de esta resposta do Stack Overflow.

  • Não adicione restrições de largura e altura. Os rótulos têm um tamanho intrínseco com base no conteúdo do texto.

  • Não há necessidade de definir sizeToFit ao usar o layout automático. O código completo para o projeto de exemplo está aqui:

      import UIKit
  class ViewController: UIViewController {

      @IBOutlet weak var myLabel: UILabel!

      @IBAction func changeTextButtonTapped(sender: UIButton) {
          myLabel.text = "my name is really long i want it to fit in this box"
      }
  }

  • Este método também pode ser usado para espaçar corretamente vários rótulos horizontalmente como em neste exemplo.

  • Se você quiser que seu rótulo enrole a linha, defina o número de linhas para 0 no IB e adicione myLabel.preferredMaxLayoutWidth = 150 // ou qualquer coisa no código. (O botão também é fixado na parte inferior da etiqueta para que ele se mova para baixo quando a altura da etiqueta aumentar.)

Quadro de rótulo dinâmico de comprimento de texto desconhecido #

Às vezes temos que redimensionar um UILabel baseado em conteúdo dinâmico onde o comprimento do texto é desconhecido. Neste exemplo, a largura do UILabel é fixada em 280 pontos e a altura é infinita, digamos 9999. Estimando o quadro em relação ao estilo do texto e maximumLabelSize.

Objetivo-C #

UILabel * label = [[UILabel alloc] init];

NSString *message = @"Some dynamic text for label";

//set the text and style if any.
label.text = message;

label.numberOfLines = 0;

CGSize maximumLabelSize = CGSizeMake(280, 9999); //280:max width of label and 9999-max height of label.

// use font information from the UILabel to calculate the size
CGSize expectedLabelSize = [label sizeThatFits:maximumLabelSize];

//Deprecated in iOS 7.0
//CGSize expectedLabelSize = [message sizeWithFont:label.font constrainedToSize:maximumLabelSize lineBreakMode:NSLineBreakByWordWrapping];

// create a frame that is filled with the UILabel frame data
CGRect newFrame = label.frame;

// resizing the frame to calculated size
newFrame.size.height = expectedLabelSize.height;

// put calculated frame into UILabel frame
label.frame = newFrame;

Rápido #

var message: String = "Some dynamic text for label"
//set the text and style if any.
label.text = message
label.numberOfLines = 0
var maximumLabelSize: CGSize = CGSize(width: 280, height: 9999)
var expectedLabelSize: CGSize = label.sizeThatFits(maximumLabelSize)
// create a frame that is filled with the UILabel frame data
var newFrame: CGRect = label.frame
// resizing the frame to calculated size
newFrame.size.height = expectedLabelSize.height
// put calculated frame into UILabel frame
label.frame = newFrame

Obtenha o tamanho do UILabel estritamente baseado em seu texto e fonte #

NSString fornece o método boundingRectWithSize que pode ser usado para prever o CGSize resultante de um UILabel com base em seu texto e fonte sem a necessidade de criar um UILabel

Objetivo-C

[[text boundingRectWithSize:maxSize options:(NSStringDrawingTruncatesLastVisibleLine | NSStringDrawingUsesLineFragmentOrigin) attributes:@{NSFontAttributeName: fontName} context:nil] size];

Rápido

let nsText = text as NSString?
nsText?.boundingRectWithSize(maxSize, options: [.TruncatesLastVisibleLine, .UsesLineFragmentOrigin], attributes: [NSFontAttributeName: fontName], context: nil).size

Rápido

Crie rótulo e rotule a saída de restrição de altura. Adicione o código abaixo onde você atribuirá o texto ao rótulo.

@IBOutlet var lblDescriptionHeightConstration: NSLayoutConstraint! 
@IBOutlet weak var lblDescription: UILabel!

let maxWidth = UIScreen.mainScreen().bounds.size.width - 40

Nota: “40” é o espaço do lado esquerdo e direito da tela.
