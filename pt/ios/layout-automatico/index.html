O Layout automático calcula dinamicamente o tamanho e a posição de todas as visualizações em sua hierarquia de visualizações, com base nas restrições colocadas nessas visualizações. Fonte

Sintaxe #

  • NSLayoutConstraint(item: Any, attribute: NSLayoutAttribute, relatedBy: NSLayoutRelation, toItem: Any?, attribute: NSLayoutAttribute, multiplicador: CGFloat, constant: CGFloat) // Cria uma restrição programaticamente

Espaço visualiza uniformemente #

É comum querer que duas visualizações fiquem lado a lado, centralizadas em sua supervisão. A resposta comum dada no Stack Overflow é incorporar essas duas visualizações em um UIView e centralizar o UIView. Isso não é necessário ou recomendado. Nos documentos do UILayoutGuide:

Há vários custos associados à adição de visualizações fictícias à sua hierarquia de visualizações. Primeiro, há o custo de criar e manter a própria visão. Em segundo lugar, a visualização fictícia é um membro completo da hierarquia de visualizações, o que significa que adiciona sobrecarga a todas as tarefas que a hierarquia executa. O pior de tudo é que a visualização fictícia invisível pode interceptar mensagens destinadas a outras visualizações, causando problemas muito difíceis de localizar.

Você pode usar UILayoutGuide para fazer isso, em vez de adicionar os botões em um UIView desnecessário. Um UILayoutGuide é essencialmente um espaço retangular que pode interagir com o Auto Layout. Você coloca um UILayoutGuide nos lados esquerdo e direito dos botões e define suas larguras para serem iguais. Isso centralizará os botões. Veja como fazer isso no código:

Estilo de linguagem de formato visual

view.addSubview(button1)
view.addSubview(button2)

let leftSpace = UILayoutGuide()
view.addLayoutGuide(leftSpace)

let rightSpace = UILayoutGuide()
view.addLayoutGuide(rightSpace)

let views = [
    "leftSpace" : leftSpace,
    "button1" : button1,
    "button2" : button2,
    "rightSpace" : rightSpace
]

// Lay the buttons and layout guides out horizontally in a line. 
// Put the layout guides on each end.
NSLayoutConstraint.activateConstraints(NSLayoutConstraint.constraintsWithVisualFormat("H:|[leftSpace][button1]-[button2][rightSpace]|", options: [], metrics: nil, views: views))

// Now set the layout guides widths equal, so that the space on the 
// left and the right of the buttons will be equal
leftSpace.widthAnchor.constraintEqualToAnchor(rightSpace.widthAnchor).active = true

Estilo Âncora

let leadingSpace = UILayoutGuide()
let trailingSpace = UILayoutGuide()
view.addLayoutGuide(leadingSpace)
view.addLayoutGuide(trailingSpace)

leadingSpace.widthAnchor.constraintEqualToAnchor(trailingSpace.widthAnchor).active = true
    
leadingSpace.leadingAnchor.constraintEqualToAnchor(view.leadingAnchor).active = true
leadingSpace.trailingAnchor.constraintEqualToAnchor(button1.leadingAnchor).active = true
    
trailingSpace.leadingAnchor.constraintEqualToAnchor(button2.trailingAnchor).active = true
trailingSpace.trailingAnchor.constraintEqualToAnchor(view.trailingAnchor).active = true

Você precisará adicionar restrições verticais a isso também, mas isso centralizará os botões na visualização sem adicionar visualizações “fictícias”! Isso evitará que o sistema perca tempo de CPU exibindo essas visualizações “fictícias”. Este exemplo usa botões, mas você pode trocar os botões para qualquer visualização na qual deseja colocar restrições.

Se você oferece suporte ao iOS 8 ou anterior, a maneira mais fácil de criar esse layout é adicionar visualizações fictícias ocultas. Com o iOS 9, você pode substituir visualizações fictícias por guias de layout.

Nota: O Interface Builder ainda não suporta guias de layout (Xcode 7.2.1). Então, se você quiser usá-los, você deve criar suas restrições no código. Fonte.

Objetivo-C

- (void)viewDidLoad
{
    [super viewDidLoad];
    UIButton *yourButton = [[UIButton alloc] initWithFrame:CGRectMake(0, 0, 100, 18)];
    [yourButton setTitle:@"Button" forState:UIControlStateNormal];


    [self.view addConstraint:[NSLayoutConstraint constraintWithItem:yourButton attribute:NSLayoutAttributeCenterY relatedBy:NSLayoutRelationEqual toItem:self.view attribute:NSLayoutAttributeCenterY multiplier:1 constant:0]]; //Align veritcally center to superView

    [self.view addConstraint:[NSLayoutConstraint constraintWithItem:yourButton attribute:NSLayoutAttributeCenterX relatedBy:NSLayoutRelationEqual toItem:self.view attribute:NSLayoutAttributeCenterX multiplier:1 constant:0]]; //Align horizontally center to superView

    [self.view addSubview:yourButton]; //Add button to superView
}

Rápido

override func viewDidLoad() 
{
    super.viewDidLoad()
    let yourButton: UIButton = UIButton(frame: CGRect(x: 0, y: 0, width: 100, height: 18))
    yourButton.setTitle("Button", forState: .Normal)

    let centerVertically = NSLayoutConstraint(item: yourButton,
                                   attribute: .CenterX,
                                   relatedBy: .Equal,
                                      toItem: view,
                                   attribute: .CenterX,
                                  multiplier: 1.0,
                                    constant: 0.0)
    let centerHorizontally = NSLayoutConstraint(item: yourButton,
                                   attribute: .CenterY,
                                   relatedBy: .Equal,
                                      toItem: view,
                                   attribute: .CenterY,
                                  multiplier: 1.0,
                                    constant: 0.0)
    NSLayoutConstraint.activateConstraints([centerVertically, centerHorizontally])
}

Noções básicas da linguagem de formato visual: restrições no código! #

HVFL é uma linguagem projetada para restringir os elementos da interface do usuário de maneira simples e rápida. Geralmente, o VFL tem uma vantagem sobre a customização de UI tradicional no Interface Builder porque é muito mais legível, acessível e compacto.

Aqui está um exemplo de VFL, no qual três UIViews são restritas da esquerda para a direita, preenchendo superView.width, com aGradeView

"H:|[bgView][aGradeView(40)][bGradeView(40)]|"

Existem dois eixos nos quais podemos restringir os objetos da interface do usuário, horizontalmente e verticalmente.

Cada linha de VFL sempre começa com H: ou V:. Se nenhum estiver presente, a opção padrão é H:

Seguindo em frente, temos um pipeline. | Este símbolo, ou o tubo, refere-se à supervisão. Se você der uma olhada no trecho de código VFL acima, você notará dois desses pipelines.

Isso significa as duas extremidades horizontais da supervisão, os limites externo esquerdo e externo direito.

Em seguida, você verá alguns colchetes, dentro do primeiro conjunto de colchetes, temos bgView. Quando temos colchetes, está se referindo a um elemento da interface do usuário, agora você pode se perguntar como estabelecemos um link entre o nome e o elemento real da interface do usuário, talvez uma saída?

Vou cobrir isso no final do post.

Se você der uma olhada no segundo par de colchetes [aGradeView(50)], temos alguns parênteses encapsulados também, quando isso está presente, define a largura/altura dependendo dos eixos, que neste caso tem 50 pixels de largura.

Os primeiros colchetes [bgView] não têm uma largura definida explicitamente, o que significa que ele se estenderá o máximo possível.

Tudo bem, é isso o básico, mais sobre as coisas avançadas em outro exemplo.

por exemplo: 

    // 1. create views
    UIView *blueView = [[UIView alloc] init];
    blueView.backgroundColor = [UIColor blueColor];
    [self.view addSubview:blueView];

    UIView *redView = [[UIView alloc] init];
    redView.backgroundColor = [UIColor redColor];
    [self.view addSubview:redView];

    // 2. forbid Autoresizing
    blueView.translatesAutoresizingMaskIntoConstraints = NO;
    redView.translatesAutoresizingMaskIntoConstraints = NO;

    // 3. make contraints
    // horizontal
    NSArray *blueH = [NSLayoutConstraint constraintsWithVisualFormat:@"H:|-20-[blueView]-20-|" options:NSLayoutFormatAlignAllLeft metrics:nil views:@{@"blueView" : blueView}];
    [self.view addConstraints:blueH];

    // vertical
    NSArray *blueVandRedV = [NSLayoutConstraint constraintsWithVisualFormat:@"V:|-20-[blueView(50)]-20-[redView(==blueView)]" options:NSLayoutFormatAlignAllTrailing metrics:nil views:@{@"blueView" : blueView, @"redView" : redView}];
    [self.view addConstraints:blueVandRedV];

    NSLayoutConstraint *redW = [NSLayoutConstraint constraintWithItem:redView attribute:NSLayoutAttributeWidth relatedBy:NSLayoutRelationEqual toItem:blueView attribute:NSLayoutAttributeWidth multiplier:0.5 constant:0];
    [self.view addConstraint:redW];

Como animar com layout automático #

Sem o layout automático, a animação é realizada alterando o quadro de uma visualização ao longo do tempo. Com o Auto Layout, as restrições ditam o quadro da vista, então você precisa animar as restrições. Essa indireção torna a animação mais difícil de visualizar.

Aqui estão as maneiras de animar com o layout automático:

  1. Altere a constante da restrição após a criação usando chamadas periódicas (CADisplayLink, dispatch_source_t, dispatch_after, NSTimer). Em seguida, chame layoutIfNeeded para atualizar a restrição. Exemplo:

Objetivo-C:

self.someConstraint.constant = 10.0;
[UIView animateWithDuration:0.25 animations:^{
    [self.view layoutIfNeeded];
}];

Rápido:

self.someConstraint.constant = 10.0
UIView.animate(withDuration: 0.25, animations: self.view.layoutIfNeeded)

  1. Altere as restrições e chame [view layoutIfNeeded] dentro de um bloco de animação. Isso interpola entre as duas posições ignorando restrições durante a animação.

    [UIView animateWithDuration:0.5 animations:^{ [view layoutIfNeeded]; }]

  2. Altere a prioridade das restrições. Isso é menos intensivo da CPU do que adicionar e remover restrições.

  3. Remova todas as restrições e use máscaras de dimensionamento automático. Para o posterior, você deve definir view.translatesAutoresizingMaskIntoConstraints = YES.

  4. Use restrições que não interfiram na animação pretendida.

  5. Use uma visualização de contêiner. Posicione a supervisão usando restrições. Em seguida, adicione uma subvisão com restrições que não contrariem a animação, por exemplo: um centro relativo à supervisão. Isso descarrega parte das restrições para a supervisualização, para que elas não lutem contra a animação na subvisualização.

  6. Anime camadas em vez de visualizações. As transformações de camada não acionam o layout automático.

    CABasicAnimation* ba = [CABasicAnimation animationWithKeyPath:@“transform”]; ba.autoreverses = YES; ba.duration = 0.3; ba.toValue = [NSValue valueWithCATransform3D:CATransform3DMakeScale(1.1, 1.1, 1)]; [v.layer addAnimation:ba forKey:nil];

  7. Substituir layoutSubvisualizações. Chame [super layoutSubviews] e ajuste as restrições.

  8. Altere o quadro em viewDidLayoutSubviews. O Layout Automático é aplicado em layoutSubviews, então, uma vez feito, altere-o em viewDidLayoutSubviews.

  9. Desative o layout automático e defina as visualizações manualmente. Você pode fazer isso substituindo layoutSubviews/layout sem chamar a implementação da superclasse.

Dica rápida: se o pai da visualização animada não estiver sendo interpolado (ou seja, a animação pula do estado inicial para o final), chame layoutIfNeeded() na visualização mais profunda que é o pai da visualização animada (no outras palavras, isso não é afetado pela animação). Não sei exatamente por que isso funciona.

Layout Proporcional #

Restrição criada como

NSLayoutConstraint(item: myView, attribute: NSLayoutAttribute.Leading, relatedBy: NSLayoutRelation.Equal, toItem: view, attribute: NSLayoutAttribute.LeadingMargin, multiplier: 1.0, constant: 20.0)

ou, do ponto de vista matemático:

view.attribute * multiplier + constant          (1)

Você pode usar o multiplicador para criar um layout proporcional para diferentes fatores de tamanho.

Exemplo:

Turquoise View (V1) é um quadrado com largura de supervisão proporcional à largura com proporção 1:1.1

Gary square(V2) é uma subvisão da V1. Espaço inferior definido por constante = 60, espaço à direita definido por multiplicador = 1,125 e constante = 0

Espaço à direita definido proporcionalmente, espaço inferior definido como uma constante.

Nota: se view.attribute for igual a 0 (por exemplo, espaço à esquerda), a fórmula de restrição (1) será igual a 0. Você precisa alterar o segundo item da restrição ou definir a restrição relativa à margem, para visualizar.attribute != 0.

NSLayoutConstraint: Restrições no código! #

Quando estamos trabalhando em uma estrutura, se as restrições não forem muito complexas, é melhor usar o Interface Builder ou NSLayoutConstraint no código para torná-lo menor o suficiente, em vez de importar Masonry ou SnapKit.

    NSLayoutConstraint *blueTop = [NSLayoutConstraint constraintWithItem:blueView attribute:NSLayoutAttributeTop relatedBy:NSLayoutRelationEqual toItem:self.view attribute:NSLayoutAttributeTop multiplier:1 constant:20];

    NSLayoutConstraint *blueRight = [NSLayoutConstraint constraintWithItem:blueView attribute:NSLayoutAttributeRight relatedBy:NSLayoutRelationEqual toItem:self.view attribute:NSLayoutAttributeRight multiplier:1 constant:-20];
    [self.view addConstraint:blueRight];

    [self.view addConstraint:redRight];

Como usar o layout automático #

O layout automático é usado para organizar as visualizações para que fiquem bem em qualquer dispositivo e orientação. Restrições são as regras que dizem como tudo deve ser estabelecido. Eles incluem fixação de bordas, centralização e configuração de tamanhos, entre outras coisas.

O layout automático é ativado por padrão, mas você pode verificar isso. Se você clicar em Main.storyboard no Project Navigator e mostrar o inspetor de arquivos. Certifique-se de que Layout automático e Classes de tamanho estejam marcados:

As restrições de layout automático podem ser definidas no Interface Builder ou no código. No Interface Builder você encontra as ferramentas Auto Layout no canto inferior direito. Clicar neles revelará diferentes opções para definir as restrições em uma visualização.

Se você deseja ter restrições diferentes para diferentes tamanhos ou orientações de dispositivos, você pode configurá-las em qualquer opção de classe de tamanho encontrada na parte inferior central.

Uso misto de layout automático com layout não automático #

Às vezes você pode querer realizar algumas ações adicionais para cálculos de Layout Automático feitos pelo próprio UIKit.

Exemplo: quando você tem um UIView que tem um maskLayer, pode ser necessário atualizar maskLayer assim que Auto Layout alterar o frame de UIView

// CustomView.m
- (void)layoutSubviews {
    [super layoutSubviews];
    // now you can assume Auto Layout did its job
    // you can use view's frame in your calculations
    CALayer maskLayer = self.maskLayer;
    maskLayer.bounds = self.bounds;
    ...
}

ou se você quiser realizar alguma ação adicional para Auto Layout em ViewController

- (void)viewDidLayoutSubviews {
    [super viewDidLayoutSubviews];
    // now you can assume all your subviews are positioned/resized correctly
    self.customView.frame = self.containerView.frame;
}
