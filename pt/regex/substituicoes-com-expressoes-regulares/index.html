Parâmetros #

Em linha Descrição
$número Substitui a substring correspondente ao número do grupo.
${nome} Substitui a substring correspondente a um nome de [grupo nomeado](https://www.wikiod.com/pt/regex/grupos-de-captura#Grupos de captura nomeados).
$$ Caractere ‘$’ de escape na string de resultado (substituição).
$& ($0) Substitui por toda a string correspondente.
$+ ($&) Substitui o texto correspondente ao último grupo capturado.
$` Substitui todo o texto correspondente por todo texto não correspondente antes da correspondência.
$’ Substitui todo o texto correspondente por todo texto não correspondente após a correspondência.
$_ Substitui todo o texto correspondente na string inteira.
Observação: Termos Itálico significam que as strings são voláteis (podem variar dependendo do seu sabor de regex).

Noções básicas de substituição #

Uma das maneiras mais comuns e úteis de substituir texto por regex é usando [Capture Groups](https://www.wikiod.com/pt/regex/grupos-de-captura#Grupos básicos de captura ).
Ou até mesmo um [Named Capture Group](https://www.wikiod.com/pt/regex/grupos-de-captura#Grupos de captura nomeados), como referência para armazenar ou substituir os dados .

Existem dois termos muito parecidos nos documentos de regex, então pode ser importante nunca confundir Substituições (ou seja, $1) com [Referências anteriores](https://www.wikiod.com/pt/docs/regex /4072/back-reference#t=201704302255256450862) (ou seja, \1). Os termos de substituição são usados ​​em um texto de substituição; Referências anteriores, na expressão Regex pura. Embora algumas linguagens de programação aceitem ambas as substituições, isso não é encorajador.

Digamos que temos esta regex: /hello(\s+)world/i. Sempre que $number for referenciado (neste caso, $1), os espaços em branco correspondentes a \s+ serão substituídos.
O mesmo resultado será exposto com a regex: /hello(?<spaces>\s+)world/i. E como temos um grupo nomeado aqui, também podemos usar ${spaces}.

Neste mesmo exemplo, também podemos usar $0 ou $& (Observação: $& pode ser usado como $+, significando recuperar o grupo de captura LAST em outros mecanismos de regex), dependendo do sabor de regex com o qual você está trabalhando, para obter todo o texto correspondente. (ou seja, $& deve retornar hEllo world para a string: hEllo world of Regex!)

Substituição avançada #

Algumas linguagens de programação têm suas próprias peculiaridades de Regex, por exemplo, o termo $+ (em C#, Perl, VB etc.) que substitui o texto correspondente ao último grupo capturado.

Exemplo:

using System;
using System.Text.RegularExpressions;

public class Example
{
   public static void Main()
   {
      string pattern = @"\b(\w+)\s\1\b";
      string substitution = "$+";
      string input = "The the dog jumped over the fence fence.";
      Console.WriteLine(Regex.Replace(input, pattern, substitution, 
                        RegexOptions.IgnoreCase));
   }
}
// The example displays the following output:
//      The dog jumped over the fence.

Exemplo do Microsoft Official’s Developer Network [1](https://msdn.microsoft.com/en-us/library/ewy2t5e0(v=vs.110).aspx?cs-save-lang =1&cs-lang=csharp#Anchor_7)

Outros termos de substituição raros são $` e $':

$` = Substitui as correspondências pelo texto antes da string correspondente
$’ = Substitui as correspondências com o texto depois da string correspondente

Devido a esse fato, essas strings de substituição devem fazer seu trabalho assim:

Regex: /part2/
Input: "part1part2part3"
Replacement: "$`"
Output: "part1part1part3" //Note that part2 was replaced with part1, due &` term
---------------------------------------------------------------------------------
Regex: /part2/
Input: "part1part2part3"
Replacement: "$'"
Output: "part1part3part3" //Note that part2 was replaced with part3, due &' term

Aqui está um exemplo dessas substituições trabalhando em um pedaço de javascript:

var rgx = /middle/;
var text = "Your story must have a beginning, middle, and end"
console.log(text.replace(rgx, "$`")); 
//Logs: "Your story must have a beginning, Your story must have a beginning, , and end"
console.log(text.replace(rgx, "$'"))
//Logs: "Your story must have a beginning, , and end, and end"
Há também o termo $_ que recupera todo o texto correspondente: 
Regex: /part2/
Input: "part1part2part3"
Replacement: "$_"
Output: "part1part1part2part3part3" //Note that part2 was replaced with part1part2part3,
                                                                         // due $_ term

Convertendo isso para VB nos daria isso:

Imports System.Text.RegularExpressions

Module Example
   Public Sub Main()
      Dim input As String = "ABC123DEF456"
      Dim pattern As String = "\d+"
      Dim substitution As String = "$_"
      Console.WriteLine("Original string:          {0}", input)
      Console.WriteLine("String with substitution: {0}", _
                        Regex.Replace(input, pattern, substitution))      
   End Sub
End Module
' The example displays the following output:
'       Original string:          ABC123DEF456
'       String with substitution: ABCABC123DEF456DEFABC123DEF456

Exemplo da Rede de Desenvolvedores do Microsoft Official [2](https://msdn.microsoft.com/en-us/library/ewy2t5e0(v=vs.110).aspx?cs-save-lang =1&cs-lang=csharp#Anchor_8)

E o último, mas não menos importante, termo de substituição é $$, que traduzido para uma expressão regex seria o mesmo que \$ (uma versão com escape do literal $).

Se você deseja corresponder uma string como esta: USD: $3.99 por exemplo, e deseja armazenar o 3.99, mas substituí-lo como $3.99 por apenas um regex, você pode usar:

Regex: /USD:\s+\$([\d.]+)/
Input: "USD: $3.99"
Replacement: "$$$1"
To Store: "$1"
Output: "$3.99"
Stored: "3.99"

Se você quiser testar isso com Javascript, você pode usar o código:

var rgx = /USD:\s+\$([\d.]+)/;
var text = "USD: $3.99";
var stored = parseFloat(rgx.exec(text)[1]);
console.log(stored); //Logs 3.99
console.log(text.replace(rgx, "$$$1")); //Logs $3.99
