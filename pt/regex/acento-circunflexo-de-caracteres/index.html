Terminologia

O caractere Caret (^) também é referido pelos seguintes termos:

  • chapéu
  • ao controle
  • seta ascendente
  • divisa
  • acento circunflexo

Uso

Ele tem dois usos em expressões regulares:

  • Para denotar o início da linha
  • Se usado imediatamente após um colchete ([^) ele age para negar o conjunto de caracteres permitidos (ou seja, [123] significa que o caractere 1, 2 ou 3 é permitido, enquanto a instrução [^123] significa que qualquer caractere diferente de 1, 2 ou 3 é permitido.

Escape de Personagem

Para expressar um acento circunflexo sem significado especial, ele deve ser precedido por uma barra invertida; ou seja, \^.

Início da linha #

Quando o modificador (?m) de várias linhas é desativado off, ^ corresponde apenas ao início da string de entrada: #

Para o regex

^He

As seguintes strings de entrada correspondem:

  • Hedgehog\nPrimeira linha\nÚltima linha
  • Me ajude, por favor
  • Ele

E as seguintes strings de entrada não correspondem:

  • Primeira linha\nHedgehog\nÚltima linha
  • ``Ouriço''
  • Ouriço (devido aos espaços em branco )

Quando o modificador (?m) de várias linhas é ativado on, ^ corresponde ao início de cada linha: #

^He

O acima corresponderia a qualquer string de entrada que contenha uma linha começando com He.

Considerando \n como o novo caractere de linha, as seguintes linhas correspondem:

  • ‘Olá’
  • Primeira linha\nHedgehog\nÚltima linha (somente segunda linha)
  • Meu\nTexto\nEstá\nAqui (apenas última linha)

E as seguintes strings de entrada não correspondem:

  • Cervejaria Camden Hells
  • Capacete (devido aos espaços em branco )

Combinando linhas vazias usando ^ #

Outro caso de uso típico para o acento circunflexo é a correspondência de linhas vazias (ou uma string vazia se o modificador de várias linhas estiver desativado).

Para corresponder a uma linha vazia (multi-linha on), um acento circunflexo é usado ao lado de um $ que é outro caractere âncora que representa a posição no final da linha (http://stackoverflow.com/ documentação/regex/1603/anchor-characters-dollar ). Portanto, a seguinte expressão regular corresponderá a uma linha vazia:

 ^$

Escapando o caractere de acento circunflexo #

Se você precisar usar o caractere ^ em uma classe de caracteres (https://www.wikiod.com/pt/regex/classes-de-personagens) ), coloque-o em algum lugar diferente do início da classe:

[12^3]

Ou escape do ^ usando uma barra invertida \:

[\^123]

Se você quiser corresponder o próprio caractere de acento circunflexo fora de uma classe de caractere, você precisa escapar dele:

\^

Isso evita que o ^ seja interpretado como o caractere âncora que representa o início da string/linha.

Comparação de início de âncora de linha e início de âncora de string #

Enquanto muitas pessoas pensam que ^ significa o início de uma string, ele na verdade significa início de uma linha. Para um início real da âncora de string, use \A.

A string hello\nworld (ou mais claramente)

hello
world

Seria correspondido pelas expressões regulares ^h, ^w e \Ah, mas não por \Aw

Modificador de várias linhas #

Por padrão, o metacaractere de acento circunflexo ^ corresponde à posição antes do primeiro caractere na string.

Dada a string “charsequence” aplicada contra os seguintes padrões: /^char/ & /^sequence/, o mecanismo tentará corresponder da seguinte forma:

  • /^char/

    • ^ - charsequence
    • c - charsequence
    • h - charsequence
    • a - charsequence
    • r - charsequence

    Match Found

  • /^sequência/

    • ^ - charsequence
    • s - charsequence

    Match not Found

O mesmo comportamento será aplicado mesmo se a string contiver terminadores de linha, como \r?\n. Apenas a posição no início da string será correspondida.

Por exemplo:

/^/g

┊char\r\n \r\n sequência

No entanto, se você precisar combinar após cada terminador de linha, você terá que definir o modo multilinha (//m, (?m)) dentro do seu padrão. Ao fazer isso, o acento circunflexo ^ corresponderá ao “início de cada linha”, que corresponde à posição no início da string e às posições imediatamente após1 dos terminadores de linha .

1 Em alguns tipos (Java, PCRE, …), ^ não corresponderá após o terminador de linha, se o terminador de linha for o último na string.

Por exemplo:

/^/gm

┊char\r\n ┊\r\n ┊sequência

Alguns dos mecanismos de expressão regular que suportam o modificador Multiline:

  • Java

     Pattern pattern = Pattern.compile("(?m)^abc");
 Pattern pattern = Pattern.compile("^abc", Pattern.MULTILINE);

  • .NET

     var abcRegex = new Regex("(?m)^abc");
 var abdRegex = new Regex("^abc", RegexOptions.Multiline)

  • PCRE

     /(?m)^abc/
 /^abc/m

  • Python 2 & 3 (módulo re embutido)

     abc_regex = re.compile("(?m)^abc");
 abc_regex = re.compile("^abc", re.MULTILINE);
