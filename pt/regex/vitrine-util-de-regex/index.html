Corresponder a um endereço de e-mail #

Fazer a correspondência de um endereço de e-mail em uma string é uma tarefa difícil, porque o especificação que o define, o RFC2822, é complexo, tornando-o difícil de implementar como um regex. Para mais detalhes porque é não é uma boa ideia combinar um e-mail com um regex, consulte o exemplo de antipadrão [quando não usar um regex: para correspondência e-mails](https://www.wikiod.com/pt/regex/quando-voce-nao-deve-usar-expressoes-regulares number-or-credit-card-number-or#t=201607301200162540574). O melhor conselho a ser observado nessa página é usar um relatório revisado por pares e amplamente biblioteca em seu idioma favorito para implementar isso.

Validar um formato de endereço de e-mail #

Quando você precisa validar rapidamente uma entrada para certificar-se de que ela parece uma email, a melhor opção é simplificar:

^\S{1,}@\S{2,}\.\S{2,}$

Essa regex verificará se o endereço de e-mail é uma sequência não separada por espaço de caracteres de comprimento maior que um, seguido por um @, seguido por dois seqüências de caracteres não-espaços de comprimento dois ou mais separados por um .. Não é perfeito e pode validar endereços inválidos (de acordo com o formato), mas o mais importante, não está invalidando endereços válidos.

Verifique se o endereço existe #

A única maneira confiável de verificar se um e-mail é válido é verificar sua existência. Costumava haver o comando SMTP VRFY que foi projetado para esse propósito, mas infelizmente, depois de ser abusado por spammers, agora não está mais disponível.

Portanto, a única maneira de verificar se o e-mail é válido e existe é realmente enviar um e-mail para esse endereço.

Enormes alternativas Regex #

No entanto, não é impossível validar um endereço de e-mail usando um regex. As únicas questões é que quanto mais próximo da especificação esses regexes estiverem, maiores eles serão e como consequência, são impossivelmente difíceis de ler e manter. Abaixo, você encontrará exemplo de tal regex mais preciso que estão sendo usados ​​em algumas bibliotecas.

⚠️ O seguinte regex é fornecido para fins de documentação e aprendizado, copiar e colar eles em seu código é uma má ideia. Em vez disso, use essa biblioteca diretamente, para que você possa confiar em código upstream e desenvolvedores de pares para manter seu código de análise de e-mail atualizado e mantido.

Módulo de correspondência de endereços Perl #

Os melhores exemplos de tal regex estão em algumas bibliotecas padrão de linguagens. Por exemplo, há um do módulo RFC::RFC822::Address na biblioteca Perl que tenta ser o mais preciso possível de acordo com o RFC. Para o seu curiosidade, você pode encontrar uma versão desse regex em [este URL] (http://www.ex-parrot.com/%7Epdw/Mail-RFC822-Address.html), que foi gerado a partir da gramática, e se você estiver tentado a copiar e colar, aqui está a citação do autor da regex:

“Não mantenho a expressão regular [linked]. Pode haver bugs nela que já foram corrigidos no módulo Perl.”

.Módulo de correspondência de endereço de rede #

Outra variante mais curta é a usada pela biblioteca padrão .Net no Módulo EmailAddressAttribute:

^((([a-z]|\d|[!#\$%&'\*\+\-\/=\?\^_`{\|}~]|[\u00A0-\uD7FF\uF900-\uFDCF\uFDF0-\uFFEF])+(\.([a-z]|\d|[!#\$%&'\*\+\-\/=\?\^_`{\|}~]|[\u00A0-\uD7FF\uF900-\uFDCF\uFDF0-\uFFEF])+)*)|((\x22)((((\x20|\x09)*(\x0d\x0a))?(\x20|\x09)+)?(([\x01-\x08\x0b\x0c\x0e-\x1f\x7f]|\x21|[\x23-\x5b]|[\x5d-\x7e]|[\u00A0-\uD7FF\uF900-\uFDCF\uFDF0-\uFFEF])|(\\([\x01-\x09\x0b\x0c\x0d-\x7f]|[\u00A0-\uD7FF\uF900-\uFDCF\uFDF0-\uFFEF]))))*(((\x20|\x09)*(\x0d\x0a))?(\x20|\x09)+)?(\x22)))@((([a-z]|\d|[\u00A0-\uD7FF\uF900-\uFDCF\uFDF0-\uFFEF])|(([a-z]|\d|[\u00A0-\uD7FF\uF900-\uFDCF\uFDF0-\uFFEF])([a-z]|\d|-|\.|_|~|[\u00A0-\uD7FF\uF900-\uFDCF\uFDF0-\uFFEF])*([a-z]|\d|[\u00A0-\uD7FF\uF900-\uFDCF\uFDF0-\uFFEF])))\.)+(([a-z]|[\u00A0-\uD7FF\uF900-\uFDCF\uFDF0-\uFFEF])|(([a-z]|[\u00A0-\uD7FF\uF900-\uFDCF\uFDF0-\uFFEF])([a-z]|\d|-|\.|_|~|[\u00A0-\uD7FF\uF900-\uFDCF\uFDF0-\uFFEF])*([a-z]|[\u00A0-\uD7FF\uF900-\uFDCF\uFDF0-\uFFEF])))\.?$

Mas mesmo que seja mais curto ainda é muito grande para ser legível e de fácil manutenção.

Módulo de correspondência de endereço Ruby #

Em ruby ​​uma composição de regex está sendo usada no módulo rfc822 para corresponder a um endereço. Esta é uma ideia legal, pois caso sejam encontrados bugs, será mais fácil identificar a parte regex a ser alterada e consertá-lo.

Módulo de correspondência de endereço Python #

Como contra-exemplo, o python módulo de análise de e-mail não está usando um regex, mas o implementa usando um analisador.

Corresponder a uma data #

Você deve se lembrar que o regex foi projetado para corresponder a uma data (ou não). Dizer que uma data é válida é uma luta muito mais complicada, pois exigirá muito tratamento de exceção (consulte condições de ano bissexto).

Vamos começar combinando o mês (1 - 12) com um 0 inicial opcional:

0?[1-9]|1[0-2]

Para corresponder ao dia, também com um 0 inicial opcional:

0?[1-9]|[12][0-9]|3[01]

E para combinar com o ano (vamos apenas assumir o intervalo 1900 - 2999):

(?:19|20)[0-9]{2}

O separador pode ser um espaço, um traço, uma barra, vazio, etc. Sinta-se à vontade para adicionar qualquer coisa que achar que pode ser usada como separador:

[-\\/ ]?

Agora você concatena tudo e obtém:

(0?[1-9]|1[0-2])[-\\/ ]?(0?[1-9]|[12][0-9]|3[01])[-/ ]?(?:19|20)[0-9]{2} // MMDDYYYY
(0?[1-9]|[12][0-9]|3[01])[-\\/ ]?(0?[1-9]|1[0-2])[-/ ]?(?:19|20)[0-9]{2} // DDMMYYYY
(?:19|20)[0-9]{2}[-\\/ ]?(0?[1-9]|1[0-2])[-/ ]?(0?[1-9]|[12][0-9]|3[01]) // YYYYMMDD

Se você quiser ser um pouco mais pedante, pode usar uma referência inversa para ter certeza de que os dois separadores serão os mesmos:

(0?[1-9]|1[0-2])([-\\/ ]?)(0?[1-9]|[12][0-9]|3[01])\2(?:19|20)[0-9]{2} // MMDDYYYY
                                                 ^ refer to [-/ ]
(0?[1-9]|[12][0-9]|3[01])([-\\/ ]?)(0?[1-9]|1[0-2])\2(?:19|20)[0-9]{2} // DDMMYYYY
(?:19|20)[0-9]{2}([-\\/ ]?)(0?[1-9]|1[0-2])\2(0?[1-9]|[12][0-9]|3[01]) // YYYYMMDD

Corresponder a um número de telefone #

Veja como combinar um código de prefixo (um + ou (00), depois um número de 1 a 1939, com um espaço opcional): Isso não procura um prefixo válido, mas algo que pode ser um prefixo. Veja a lista completa de prefixos

(?:00|\+)?[0-9]{4}

Então, como todo o comprimento do número de telefone é, no máximo, 15, podemos procurar até 14 dígitos: Pelo menos 1 dígito é gasto para o prefixo

[0-9]{1,14}

Os números podem conter espaços, pontos ou traços e podem ser agrupados por 2 ou 3.

(?:[ .-][0-9]{3}){1,5}

Com o prefixo opcional:

(?:(?:00|\+)?[0-9]{4})?(?:[ .-][0-9]{3}){1,5}

Se você deseja corresponder a um formato de país específico, pode usar esta consulta de pesquisa e adicionar o país, a pergunta certamente já foi Perguntou.

Corresponder a um endereço IP #

IPv4

Para corresponder ao formato de endereço IPv4, você precisa verificar os números [0-9]{1,3} três vezes {3} separados por pontos \. e terminando com outro número.

^(?:[0-9]{1,3}\.){3}[0-9]{1,3}$

Esta expressão regular é muito simples - se você quiser que ela seja precisa, você precisa verificar se os números estão entre 0 e 255, com a regex acima aceitando 444 em qualquer posição. Você deseja verificar 250-255 com 25[0-5], ou qualquer outro valor 200 2[0-4][0-9], ou qualquer valor 100 ou menos com [01]?[ 0-9][0-9]. Você quer verificar se é seguido por um ponto \. três vezes {3} e depois uma vez sem ponto.

^(?:(?:25[0-5]|2[0-4][0-9]|[01]?[0-9][0-9]?)\.){3}(?:25[0-5]|2[0-4][0-9]|[01]?[0-9][0-9]?)$

IPv6

Os endereços IPv6 têm a forma de 8 palavras hexadecimais de 16 bits delimitadas com o caractere dois-pontos (:). Nesse caso, verificamos 7 palavras seguidas de dois-pontos, seguidas de uma que não seja. Se uma palavra tiver zeros à esquerda, eles podem ser truncados, o que significa que cada palavra pode conter entre 1 e 4 dígitos hexadecimais.

^(?:[0-9a-fA-F]{1,4}:){7}[0-9a-fA-F]{1,4}$

Isso, porém, é insuficiente. Como os endereços IPv6 podem se tornar bastante “palavrosos”, o padrão especifica que palavras somente zero podem ser substituídas por ::. Isso só pode ser feito uma vez em um endereço (para qualquer lugar entre 1 e 7 palavras consecutivas), pois de outra forma seria indeterminado. Isso produz uma série de variações (bastante desagradáveis):

^::(?:[0-9a-fA-F]{1,4}:){0,6}[0-9a-fA-F]{1,4}$
^[0-9a-fA-F]{1,4}::(?:[0-9a-fA-F]{1,4}:){0,5}[0-9a-fA-F]{1,4}$
^[0-9a-fA-F]{1,4}:[0-9a-fA-F]{1,4}::(?:[0-9a-fA-F]{1,4}:){0,4}[0-9a-fA-F]{1,4}$
^(?:[0-9a-fA-F]{1,4}:){0,2}[0-9a-fA-F]{1,4}::(?:[0-9a-fA-F]{1,4}:){0,3}[0-9a-fA-F]{1,4}$
^(?:[0-9a-fA-F]{1,4}:){0,3}[0-9a-fA-F]{1,4}::(?:[0-9a-fA-F]{1,4}:){0,2}[0-9a-fA-F]{1,4}$
^(?:[0-9a-fA-F]{1,4}:){0,4}[0-9a-fA-F]{1,4}::(?:[0-9a-fA-F]{1,4}:)?[0-9a-fA-F]{1,4}$
^(?:[0-9a-fA-F]{1,4}:){0,5}[0-9a-fA-F]{1,4}::[0-9a-fA-F]{1,4}$
^(?:[0-9a-fA-F]{1,4}:){0,6}[0-9a-fA-F]{1,4}::$

Agora, juntando tudo (usando alternância) produz:

^(?:[0-9a-fA-F]{1,4}:){7}[0-9a-fA-F]{1,4}$|
^::(?:[0-9a-fA-F]{1,4}:){0,6}[0-9a-fA-F]{1,4}$|
^[0-9a-fA-F]{1,4}::(?:[0-9a-fA-F]{1,4}:){0,5}[0-9a-fA-F]{1,4}$|
^[0-9a-fA-F]{1,4}:[0-9a-fA-F]{1,4}::(?:[0-9a-fA-F]{1,4}:){0,4}[0-9a-fA-F]{1,4}$|
^(?:[0-9a-fA-F]{1,4}:){0,2}[0-9a-fA-F]{1,4}::(?:[0-9a-fA-F]{1,4}:){0,3}[0-9a-fA-F]{1,4}$|
^(?:[0-9a-fA-F]{1,4}:){0,3}[0-9a-fA-F]{1,4}::(?:[0-9a-fA-F]{1,4}:){0,2}[0-9a-fA-F]{1,4}$|
^(?:[0-9a-fA-F]{1,4}:){0,4}[0-9a-fA-F]{1,4}::(?:[0-9a-fA-F]{1,4}:)?[0-9a-fA-F]{1,4}$|
^(?:[0-9a-fA-F]{1,4}:){0,5}[0-9a-fA-F]{1,4}::[0-9a-fA-F]{1,4}$|
^(?:[0-9a-fA-F]{1,4}:){0,6}[0-9a-fA-F]{1,4}::$

Certifique-se de escrevê-lo no modo de várias linhas e com uma pilha de comentários para que quem for inevitavelmente encarregado de descobrir o que isso significa não venha atrás de você com um objeto contundente.

Valide uma string de tempo de 12 horas e 24 horas #

Para um formato de 12 horas, pode-se usar:

^(?:0?[0-9]|1[0-2])[-:][0-5][0-9]\s*[ap]m$

Onde

  • (?:0?[0-9]|1[0-2]) é a hora
  • [-:] é o separador, que pode ser ajustado para atender às suas necessidades
  • [0-5][0-9] é o minuto
  • \s*[ap]m seguiu qualquer número de caracteres de espaço em branco e am ou pm

Se você precisar dos segundos:

^(?:0?[0-9]|1[0-2])[-:][0-5][0-9][-:][0-5][0-9]\s*[ap]m$

Para um formato de hora de 24 horas:

^(?:[01][0-9]|2[0-3])[-:h][0-5][0-9]$

Onde:

  • (?:[01][0-9]|2[0-3]) é a hora
  • [-:h] o separador, que pode ser ajustado para atender às suas necessidades
  • [0-5][0-9] é o minuto

Com os segundos:

^(?:[01][0-9]|2[0-3])[-:h][0-5][0-9][-:m][0-5][0-9]$

Onde [-:m] é um segundo separador, substituindo o h para horas por um m para minutos e [0-5][0-9] é o segundo.
