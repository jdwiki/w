Conectar seu aplicativo WatchOS ao seu aplicativo iOS pode ser uma tarefa a ser concluída quando você nunca fez isso antes. Este tutorial mostrará os fundamentos básicos para realizar esta tarefa muito importante.

Configuração do iOS #

Aplicativo para iPhone

  1. importe WatchConnectivity e esteja em conformidade com WCSessionDelegate.
  2. use o delegado de sessão estática via WCSession.default().
  3. Envie dados para o aplicativo Watch usando:
WCSession.default().sendMessage(message, replyHandler:_ errorHandler:_)
  1. O objeto de mensagem deve ser um dicionário do tipo [String:Any]
  2. Se você estiver procurando por dados a serem retornados do aplicativo Watch, forneça a lógica em um encerramento definido no replyHandler; caso contrário, passe em zero.
  3. Para responder às mensagens enviadas do aplicativo Watch, você usará o método de retorno de chamada WCSessionDelegate
func session(_ session: WCSession, didReceiveMessage message: [String : Any], replyHandler: @escaping ([String : Any]) -> Swift.Void){...}

Você precisa importar a estrutura WatchConnectivity para seu arquivo view controller no aplicativo iOS. Isso lhe dá acesso aos objetos de classe que são projetados para se comunicar com o aplicativo complementar Watch. Você também precisará importar isso no lado do aplicativo Watch. A única diferença real com o aplicativo Watch é que você não terá um arquivo view controller, mas um arquivo interface controller.

import WatchConnectivity

Em seguida, você precisará certificar-se de que seu aplicativo pode oferecer suporte a uma sessão para WatchConnectivity. Se puder, você deve definir o controlador de exibição como seu delegado e ativar a sessão padrão. Você receberá um erro aqui. Você precisa estar em conformidade com WCSessionDelegate e implementar alguns métodos antes que o IDE comece a se acalmar.

// MARK: - View Life Cycle Callbacks
override func viewDidLoad() {
    super.viewDidLoad()

    automaticallyAdjustsScrollViewInsets = false
    if WCSession.isSupported() {
        WCSession.default().delegate = self
        WCSession.default().activate()
    }
    else {
        print("\nViewController: connectionManager is nil\n")
    }
}

Para estar em conformidade com WCSessionDelegate, vamos adicionar uma extensão na parte inferior do controlador de exibição. Algumas pessoas odeiam essa abordagem. Eu tenho meu caminho, mas para fins de tutorial, seguirei a abordagem RayWenderlich. Eu sou um grande fã de apenas obter o código diretamente do COMMAND + CLICAR nos delegados e pegar TODOS OS MÉTODOS da especificação e começar a manipulá-lo e entender como as coisas funcionam. Nesta extensão, estou fornecendo a você todos os métodos. Eles já estão marcados para que você veja quando cada um deles disparar no console. Se você se sentir brincalhão, exclua alguns dos que estão marcados como OPCIONAIS para fazer seu arquivo de código parecer elegante.

extension ViewController : WCSessionDelegate {

    func session(_ session: WCSession, activationDidCompleteWith activationState: WCSessionActivationState, error: Error?) {
        print("0. ViewController: ", "activationDidCompleteWith activationState")
    }

    func sessionDidBecomeInactive(_ session: WCSession) {
        print("1. ViewController: ", "sessionDidBecomeInactive")

        print("2. ViewController: ", "sessionDidDeactivate")
    }


    func sessionWatchStateDidChange(_ session: WCSession) {
        print("3. ViewController: ", "sessionDidDeactivate")
    }


    /** ------------------------- Interactive Messaging ------------------------- */

    func sessionReachabilityDidChange(_ session: WCSession) {
        print("4. ViewController: ", "sessionReachabilityDidChange")
    }


    func session(_ session: WCSession, didReceiveMessage message: [String : Any]) {
        print("5. ViewController: ", "didReceiveMessage")

    func session(_ session: WCSession, didReceiveMessage message: [String : Any], replyHandler: @escaping ([String : Any]) -> Swift.Void) {
        print("6. ViewController: ", "didReceiveMessage")
        // This is where you handle any requests coming from your Watch App
    }

    func session(_ session: WCSession, didReceiveMessageData messageData: Data) {
        print("7. ViewController: ", "didReceiveMessageData")
    }

    func session(_ session: WCSession, didReceiveMessageData messageData: Data, replyHandler: @escaping (Data) -> Swift.Void) {
        print("8. ViewController: ", "didReceiveMessageData")
    }

    /** -------------------------- Background Transfers ------------------------- */

    func session(_ session: WCSession, didReceiveApplicationContext applicationContext: [String : Any]) {

        print("10. ViewController: ", "didFinish userInfoTransfer")

        print("11. ViewController: ", "didReceiveUserInfo")

    func session(_ session: WCSession, didFinish fileTransfer: WCSessionFileTransfer, error: Error?) {

Para enviar dados para seu Watch App, uma vez que o Watch App tenha solicitado, você irá lidar com isso no método…

func session(_ session: WCSession, didReceiveMessage message: [String : Any], replyHandler: @escaping ([String : Any]) -> Swift.Void) {
    print("6. ViewController: ", "didReceiveMessage")
    // build out your response message using a Dictionary
    let returnMessage: [String : Any] = [
       "key1" : value1,
       "key2" : value2,
       "key3" : value3
    ]
    // return your data in this manner
    replyHandler(returnMessage)
    // WARNING
    // You must call the replyHandler before the method ends, otherwise, your app will crash.
}

Esta é apenas uma metade da transação! Você deve configurar seu aplicativo Watch para se conectar ao aplicativo para iPhone e lidar com todas as mensagens retornadas!

Configuração da extensão do relógio #

Aplicativo WatchKit

  1. importe WatchConnectivity e esteja em conformidade com WCSessionDelegate.
  2. use o delegado de sessão estática via WCSession.default().
  3. Envie dados para o aplicativo do iPhone usando:
WCSession.default().sendMessage(message, replyHandler:_ errorHandler:_)
  1. O objeto de mensagem deve ser um dicionário do tipo [String:Any]
  2. Se você estiver procurando por dados a serem retornados do aplicativo Watch, forneça a lógica em um encerramento definido no replyHandler; caso contrário, passe em zero.
  3. Para responder às mensagens enviadas do aplicativo para iPhone, você usará o método de retorno de chamada WCSessionDelegate
func session(_ session: WCSession, didReceiveMessage message: [String : Any], replyHandler: @escaping ([String : Any]) -> Swift.Void){...}
  1. Esses métodos não serão necessários em seu aplicativo Watch para estar em conformidade com o WCSessionDelegate:
func sessionDidBecomeInactive(_ session: WCSession)
func sessionDidDeactivate(_ session: WCSession)
func sessionWatchStateDidChange(_ session: WCSession)

E, finalmente, geralmente é uma prática recomendada armazenar todas as imagens comuns, as imagens e os ativos que serão usados ​​continuamente para que o aplicativo Watch seja colocado na pasta xcassets do Watch. Ok, agora que você está confuso, vamos aos detalhes!

Pode ser uma surpresa para você, mas você precisa importar o WatchConnectivity novamente.

import WatchConnectivity

Em seguida, você precisa verificar se a sessão é possível.

// MARK: - View Life Cycle Callbacks
override func awake(withContext context: Any?) {
    super.awake(withContext: context)

    // Configure interface objects here.
    if WCSession.isSupported() {
        WCSession.default().delegate = self
        WCSession.default().activate()
        print("InterfaceController: Session Activated")

        // Request Data from iPhone App
        let requestMessage = ["message":"get-data"]

        WCSession.default().sendMessage(requestMessage, replyHandler: { (replyMessage) in
            print("Got a reply from the phone: \(replyMessage)")

            // handle reply message here

        }, errorHandler: { (error) in
            print("Got an error sending to the phone: \(error)")
        })
    }
    else {
        print("\nViewController: connectionManager is nil\n")
    }
}

Mas nada disso funcionará, a menos que você implemente os métodos necessários para o WCSessionDelegate.

Enviando dados #

Enviar dados para seu aplicativo Watch do seu iPhone ou do seu iPhone para o seu aplicativo Watch após acionar algum evento é muito simples, embora o código possa parecer um pouco complexo no início.

let message = ["key":"value-to-send"]

WCSession.default().sendMessage(message, replyHandler: { (replyMessage) in
    print("Got a reply from the phone: \(replyMessage)")

    if let returnedValues = replyMessage["returned-value"] as? NSArray {
        for val in returnedValues {
            // do something here with the data
            // Dispatch to Main Thread if affecting UI
        }
    }
}, errorHandler: { (error) in
    print("Got an error sending to the phone: \(error)")
})
