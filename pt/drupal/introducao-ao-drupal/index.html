Tipos de entidade

Nas versões anteriores do Drupal, o sistema de campo era usado apenas em tipos de conteúdo. Agora, graças à Entity API, podemos adicionar campos a outras coisas, como comentários. Entidades de campo tornam o Drupal eminentemente flexível. Um tipo de entidade é uma abstração útil para agrupar campos. Abaixo estão os tipos de entidade no núcleo Drupal:

  • Nós (conteúdo)
  • Comentários
  • Arquivos
  • Termos de taxonomia
  • Vocabulários de taxonomia
  • Usuários

Você também pode criar novos tipos de entidades onde as opções acima não atendem às suas necessidades.

Pacotes

Os pacotes são uma implementação de um tipo de entidade ao qual os campos podem ser anexados. Você pode considerar os pacotes configuráveis ​​como subtipos de um tipo de entidade. Com nós de conteúdo (um tipo de entidade), por exemplo, você pode gerar pacotes (subtipos) como artigos, postagens de blog ou produtos. Nem todos os tipos de entidade têm pacotes, no entanto. Por exemplo, os usuários não têm pacotes separados (subtipos). Para os tipos de entidade que permitem pacotes, você pode criar quantos pacotes (subtipos) desejar. Em seguida, usando o sistema Field, você pode adicionar campos diferentes a cada pacote. Os exemplos incluem um campo de download de arquivo em Páginas básicas e um campo de legenda em Artigos.

Campos

Um campo é uma parte reutilizável de conteúdo. Em termos técnicos, cada campo é um tipo de dado primitivo, com validadores e widgets personalizados para edição e formatadores para exibição. Você pode ler mais para obter um guia do desenvolvedor para usar a API Drupal 7 Fields.

O que é importante saber no que se refere às Entidades é que os Campos podem ser adicionados a qualquer um dos pacotes (ou tipos de entidade) para ajudar a organizar seus dados.

Digamos, por exemplo, que você crie um tipo de conteúdo com um campo de texto não estruturado e use HTML para estruturar partes dele, como uma seção de resumo ou preços. Isso tornaria mais difícil, então, controlar como eles eram exibidos ou fazer conexões entre diferentes tipos de conteúdo relacionado.

É aqui que o uso de campos é essencial. Você pode criar um campo de resumo do tipo Texto Longo, bem como campos de preço do tipo Decimal.

Entidade

Uma entidade seria uma instância de um tipo de entidade específico, como um comentário, termo de taxonomia ou perfil de usuário ou de um pacote, como uma postagem de blog, artigo ou produto.

Você pode usar entity_load para carregar qualquer entidade. Observe, no entanto, que o núcleo não fornece uma função salvar ou excluir, mas graças ao módulo Entity API as peças que faltam são adicionadas (entity_create(), entity_save(), entity_delete(), entity_view() e entity_access ()).

Colocando isso em termos de Programação/Design Orientado a Objetos…

Se você vem de uma formação OOD/P e está tentando entender melhor quais são esses conceitos-chave, o mapeamento sugerido a seguir pode ajudar (embora não seja estritamente verdadeiro da perspectiva de um purista): -

  • Um tipo de entidade é uma classe base
  • Um pacote é uma aula estendida
  • Um campo é um membro da classe, propriedade, variável ou instância de campo (dependendo da sua preferência de nomeação) )
  • Uma entidade é um objeto ou instância de uma base ou classe estendida

Todos esses quatro conceitos OOD/P são especiais por serem serializáveis ​​(armazenados - por exemplo, em um banco de dados ou arquivo). A serialização ocorre por meio da API Entity.

Instalando o Drupal com Drush #

drush dl drupal --drupal-project-rename=example
cd example
drush site-install standard --db-url='mysql://[db_user]:[db_pass]@localhost/[db_name]' --site-name=Example

Instalação do Drupal 8 com Drupal Console #

Console Drupal

A nova CLI para Drupal. Uma ferramenta para gerar código clichê, interagir e depurar o Drupal.

Primeiro, precisamos instalar o Drupal Console.

O Drupal Console é necessário não apenas para este momento, mas para futuras instalações.

# Run this in your terminal to get the latest project version:
curl https://drupalconsole.com/installer -L -o drupal.phar

# Or if you don't have curl:
php -r "readfile('https://drupalconsole.com/installer');" > drupal.phar

# Accessing from anywhere on your system:
mv drupal.phar /usr/local/bin/drupal

# Apply executable permissions on the downloaded file:
chmod +x /usr/local/bin/drupal

# Copy configuration files to user home directory:
drupal init --override

# Check and validate system requirements
drupal check

Você pode chamar drupal list para ver todos os comandos disponíveis.

Na próxima etapa vamos baixar o código fonte do Drupal

drupal site:new

O console solicitará que você escolha uma pasta para baixar o Drupal. E na próxima etapa, você será solicitado a escolher a versão do Drupal para download. Eu recomendo selecionar o último.

Então, quando o Drupal é baixado, você precisa instalá-lo.

drupal site:install

Após alguns passos simples, seu site Drupal estará pronto.

Com essa metodologia, uma nova instalação do Drupal leva de 5 a 7 minutos, tudo a partir da linha de comando.
