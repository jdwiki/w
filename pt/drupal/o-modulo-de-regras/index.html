O módulo Rules é um mecanismo que permite aos administradores do site automatizar ações a serem executadas condicionalmente, programaticamente ou em resposta a eventos predeterminados.

As regras podem reagir a Eventos de regras que ocorrem em um site Drupal, como o login de um usuário. E pode realizar Ações de regras personalizadas de acompanhamento, como redirecionar para uma determinada página, que deve ser executado condicionalmente se algumas Condições de regras forem atendidas.

Recursos #

  • Tutoriais em vídeo: Johan Falk fez um trabalho incrível nos primeiros 7 dias do Drupal ao criar um conjunto impressionante de tutoriais para “Aprender a Estrutura de Regras” com um conjunto de mais de 30 vídeos e relacionados blogposts hospedados em nodeone.se (licença = Atribuição-Uso Não-Comercial-Compartilhamento Igual 3.0).

No entanto, o domínio nodeone.se não os hospeda mais. Aprenda Regras é uma tentativa de recuperar essas valiosas postagens de blog (com links relacionados aos vídeos correspondentes).

  • The Tiny Book of Rules é uma introdução (15 páginas) sobre o módulo Rules.

Uma regra personalizada mostrada usando a interface do usuário de regras #

Uma regra personalizada mostrada no formato Exportação de regras #

Aqui está um exemplo de regras em Formato de exportação de regras:

{ "rules_display_userpoints_after_updating_content" : {
    "LABEL" : "Display userpoints after updating content",
    "PLUGIN" : "reaction rule",
    "OWNER" : "rules",
    "REQUIRES" : [ "userpoints_rules", "rules", "rules_conditional" ],
    "ON" : { "node_update" : [] },
    "DO" : [
      { "userpoints_rules_get_current_points" : {
          "USING" : { "user" : [ "site:current-user" ], "tid" : "all" },
          "PROVIDE" : { "loaded_points" : { "total_points" : "Number of points in all categories together" } }
        }
      },
      { "drupal_message" : { "message" : "You now have [total-points:value] points" } },
      { "CONDITIONAL" : [
          {
            "IF" : { "NOT data_is" : { "data" : [ "total-points" ], "op" : "\u003C", "value" : "20" } },
            "DO" : [
              { "drupal_message" : { "message" : "You have sufficient points (you still have [total-points:value] ...)." } }
            ]
          },
          { "ELSE" : [
              { "drupal_message" : { "message" : "You DO NOT have sufficient points (you only have [total-points:value] ...)." } }
            ]
          }
        ]
      }
    ]
  }
}

Ele recupera, como a primeira ação de regras (não a condição de regras!) a quantidade atual de pontos de usuário de um usuário. Se o valor for de pelo menos 20, aparecerá uma mensagem começando com “Você tem pontos suficientes …”, caso contrário a mensagem começa com “Você NÃO tem pontos suficientes …”.

Processando itens de coleção de campo com regras #

Processar itens de Coleção de campo com Regras é realmente divertido! Dê uma olhada nesta Regra (no formato de exportação de regras):

{ "rules_calculate_sum_of_prices_in_all_field_collection_items" : {
    "LABEL" : "Calculate sum of prices in all field collection items",
    "PLUGIN" : "reaction rule",
    "OWNER" : "rules",
    "REQUIRES" : [ "rules" ],
    "ON" : { "node_view--article" : { "bundle" : "article" } },
    "IF" : [
      { "entity_has_field" : { "entity" : [ "node" ], "field" : "field_article_details" } }
    ],
    "DO" : [
      { "drupal_message" : { "message" : "\u003Cstrong\u003EDrupal calculator\u003C\/strong\u003E started ..." } },
      { "variable_add" : {
          "USING" : { "type" : "decimal", "value" : "0" },
          "PROVIDE" : { "variable_added" : { "total_price" : "Price total" } }
        }
      },
      { "LOOP" : {
          "USING" : { "list" : [ "node:field-article-details" ] },
          "ITEM" : { "article_details_item" : "Article details item" },
          "DO" : [
            { "data_calc" : {
                "USING" : {
                  "input_1" : [ "total-price" ],
                  "op" : "+",
                  "input_2" : [ "article-details-item:field-price" ]
                },
                "PROVIDE" : { "result" : { "calculation_result" : "Calculation result" } }
              }
            },
            { "data_set" : { "data" : [ "total-price" ], "value" : [ "calculation-result" ] } },
            { "drupal_message" : { "message" : "After adding a price of \u003Cstrong\u003E[article-details-item:field-price]\u003C\/strong\u003E for field collection item with id \u003Cstrong\u003E[article-details-item:item-id]\u003C\/strong\u003E, subtotal is \u003Cstrong\u003E[calculation-result:value]\u003C\/strong\u003E." } }
          ]
        }
      },
      { "drupal_message" : { "message" : "The \u003Cstrong\u003ETotal price\u003C\/strong\u003E for all prices included as field collection items is \u003Cstrong\u003E[total-price:value]\u003C\/strong\u003E." } },
      { "drupal_message" : { "message" : "\u003Cstrong\u003EDrupal calculator\u003C\/strong\u003E ended ..." } }
    ]
  }
}

Mais alguns detalhes sobre esta regra estão abaixo…

Evento de regras: #

O conteúdo é visualizado (do tipo Artigo), adapte o nome da máquina do tipo de conteúdo artigo para o que for adequado (ou use qualquer outro Evento de Regras que seja adequado).

Condição das regras: #

A entidade tem um campo, enquanto a entidade é “nó”, e o nome da máquina do meu campo de coleta de campos é field_article_details (adapte este nome de máquina para o que for adequado, mas certifique-se de usar o próprio campo de coleta de campos).

Ações de regras: #

Acorde, aqui é onde a mágica (diversão?) vai acontecer… Estas são as Ações das Regras envolvidas:

  1. Mostrar uma mensagem no site, com uma mensagem como esta:

Calculadora Drupal iniciada…

  1. Adicione uma variável, enquanto é uma variável chamada total_price, decimal (2 dígitos), valor inicial 0.

  2. Adicione um loop, para iterar sobre cada item do meu campo de coleção de campos (com o nome da máquina field_article_details), e execute estas ações de regras para cada iteração:

  • Calcular um valor, que calcula a soma de total_price (definido na Ação 2 das Regras acima) e article-details-item:field-price (este é o nome da máquina do campo em a coleção de campos que contém os preços, decimal com 2 dígitos), e armazena o resultado (soma) na variável calculation_result.

  • Defina um valor de dados, que simplesmente copia o valor armazenado na variável calculation_result no meu total_price (definido na Ação 2 das Regras acima). Observação: não tenho certeza (não testei), mas talvez esta variável calculation_result possa ser substituída diretamente por total_price (na ação anterior), para que você não precise dessa ação.

  • Mostrar uma mensagem no site, com uma mensagem assim:

    After adding a price of 3.40 for field collection item with id 3, subtotal is 15.00.

  1. Mostrar uma mensagem no site, com uma mensagem assim:

O preço total para todos os preços incluídos como itens de coleta de campo é 26,23.

  1. Mostrar uma mensagem no site, com uma mensagem assim:

A calculadora Drupal terminou…

Obviamente, esta regra é mais um protótipo. Depois de estar convencido de que funciona como deveria, remova todas as Ações de Regras com Mostrar uma mensagem no site. Para que apenas os itens 2 e 3 (sem seu último sub-bala) sejam deixados como Ações de Regras.
