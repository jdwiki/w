Arquivo .info do tema #

name = MyCompany Theme
description = A Bootstrap Sub-theme.
core = 7.x
base theme = bootstrap

;;;;;;;;;;;;;;;;;;;;;
;; Regions
;;;;;;;;;;;;;;;;;;;;;

regions[navigation]     = 'Navigation'
regions[header]         = 'Top Bar'
regions[highlighted]    = 'Highlighted'
regions[help]           = 'Help'
regions[content]        = 'Content'
regions[sidebar_first]  = 'Primary'
regions[sidebar_second] = 'Secondary'
regions[footer]         = 'Footer'
regions[page_top]       = 'Page top'
regions[page_bottom]    = 'Page bottom'

;;;;;;;;;;;;;;;;;;;;;
;; MyCompany Custom Regions
;;;;;;;;;;;;;;;;;;;;;
regions[footer_menu_left] = 'Footer menu left'
regions[footer_menu_right] = 'Footer menu right'

;;;;;;;;;;;;;;;;;;;;;
;; CSS
;; these css files will be included on every page
;;;;;;;;;;;;;;;;;;;;;

stylesheets[all][] = css/bootstrap.min.css
stylesheets[all][] = css/MyCompany.css

;;;;;;;;;;;;;;;;;;;;;
;; JS
;; this JS file will be included on every page
;;;;;;;;;;;;;;;;;;;;;

scripts[] = js/MyCompany.min.js

Escrevendo arquivos .info do tema #

O arquivo .info é um arquivo de texto estático para definir e configurar um tema. Cada linha no arquivo .info é um par chave-valor com a chave à esquerda e o valor à direita, com um “sinal de igual” entre eles (por exemplo, nome = meu_tema).

O ponto e vírgula é usado para comentar uma linha. Algumas chaves usam uma sintaxe especial com colchetes para construir uma lista de valores associados, chamada de “array”. Se você não estiver familiarizado com arrays, dê uma olhada nos arquivos .info padrão que vêm com o Drupal e leia as explicações dos exemplos a seguir. Mesmo que a extensão de arquivo .info não seja aberta nativamente por um aplicativo, você pode usar o TextEdit em um Mac ou o Bloco de Notas em um computador Windows para visualizar, editar e salvar suas alterações.

Requisitos do nome do tema

O nome deve começar com um caractere alfabético, pode conter números e sublinhados, mas não hífens, espaços ou pontuação. O nome será usado pelo Drupal na formação de várias funções em PHP e, portanto, possui as mesmas limitações.

Não escolha nomes que já sejam usados ​​pelos módulos instalados, pois todos os componentes instalados devem ter nomes exclusivos.

Uma das melhores práticas é usar prefixos ao nomear o tema personalizado de um site, para garantir nomes exclusivos para os temas. Um site chamado example.com pode usar nomes de temas como ex_themename.

Como o arquivo .info é armazenado em cache, você deve limpar o cache antes que as alterações sejam exibidas em seu site.

O arquivo .info também pode especificar quais configurações de tema devem ser acessadas a partir da interface de administração do Drupal, como você verá em breve.

Codificação

O arquivo deve ser salvo como UTF-8 sem uma Byte Order Mark (BOM).

Conteúdo

Drupal entende as chaves listadas abaixo. O Drupal usará valores padrão para as chaves opcionais não presentes no arquivo .info. Veja os exemplos definidos para temas principais.

  • nome obrigatório
  • descrição recomendado
  • captura de tela
  • versão desanimado
  • núcleo obrigatório
  • motor
  • tema base
  • regiões
  • recursos
