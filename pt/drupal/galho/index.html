Twig é o mecanismo de template que faz parte do Drupal 8. No Drupal 8, os arquivos Twig têm a extensão .html.twig e são usados ​​em todos os aspectos do tema Drupal. Entidades, campos, visualizações podem ser renderizados usando arquivos .html.twig.

Neste tópico o objetivo é ter um livro de receitas sobre como trabalhar com o Twig no contexto do Drupal. Se você quiser saber mais sobre a sintaxe ou funções disponíveis consulte a documentação.

Filtro de galhos #

Ao contrário do Drupal 7, você não pode chamar funções PHP regulares em seus templates. No Drupal 8 o caminho a percorrer é criando filtros e funções.

Você deve usar um filtro quando: quiser transformar os dados que deseja exibir. Imagine que você tem um título que deseja que seja sempre maiúsculo. Por exemplo, o twig tem o filtro capitalize por padrão que permite transformar qualquer texto em seu equivalente em maiúsculas.

Para este exemplo, criaremos um filtro que nos permitirá embaralhar uma string. A maneira de criar filtros e funções é exatamente a mesma do Twig normal.

A principal diferença entre o Twig normal e o Drupal 8 Twig é que no Drupal 8 você deve criar uma definição de serviço da classe que está criando e a classe também deve pertencer a um namespace caso contrário não será registrado como um filtro Twig dentro do ambiente Drupal .

Este exemplo assume que você tem um módulo chamado twig_shuffle_extension.

Esta será a definição básica de serviço da pousada twig_shuffle_extension.services.yml

services:
  twig_shuffle_extension.twig_extension:
    class: Drupal\twig_shuffle_extension\TwigExtension\TwigShuffleExtension
    tags:
      - { name: twig.extension }

A chave tags também é absolutamente necessária e é o que diz ao Drupal o que esta classe deve fazer (ou seja, registrá-la como uma extensão Twig).

E agora o código fonte que deve ser colocado no caminho definido na chave class da definição do serviço.

// Don't forget the namespace!
namespace Drupal\twig_shuffle_extension\TwigExtension;

use Twig_Extension;
use Twig_SimpleFilter;

class TwigShuffleExtension extends Twig_Extension  {
  /**
   * This is the same name we used on the services.yml file
   */
  public function getName() {
    return 'twig_shuffle_extension.twig_extension';
  }

  // Basic definition of the filter. You can have multiple filters of course.
  // Just make sure to create a more generic class name ;)
  public function getFilters() {
    return [
      new Twig_SimpleFilter('shuffle', [$this, 'shuffleFilter']),
    ];
  }

  // The actual implementation of the filter.
  public function shuffleFilter($context) {
    if(is_string($context)) {
      $context = str_shuffle($context);
    }
    return $context;
  }
}

Limpe seus caches e agora, se tudo correr conforme o planejado, você pode usar o filtro em seus templates.

{{ "shuffle me!" | shuffle }}

Injeção de dependência em extensões Twig #

Este exemplo mostrará como usar o Dependency Inject para usar outros serviços registrados no ambiente Drupal.

Imagine que você tenha um arquivo de imagem SVG que muda de cor dependendo de alguma coisa aleatória de CSS/Javascript em seu projeto. Para poder direcionar o SVG com CSS, você precisa realmente ter o arquivo SVG no DOM. Então você cria um arquivo SVG base sem cores e o coloca na sua pasta de temas.

Claro que você poderia simplesmente colar o conteúdo do arquivo no modelo Twig, mas isso não seria legal. Você pode criar uma extensão Twig, mas também não deseja codificar o caminho do tema no código-fonte da extensão.

Isso significa que temos que obter o caminho dinamicamente. Você tem duas opções:

  1. Use o equivalente a uma variável global chamando \Drupal::theme()->getActiveTheme()->getPath();
  2. Injete o ThemeManager (fornecido por \Drupal::theme()) em sua classe de extensão

Neste exemplo vamos pegar o segundo exemplo porque ele pode ser amplamente aplicável a qualquer serviço (você importa o Request ou o Database Connection se quiser).

Isso pressupõe que você tenha um módulo chamado twig_svg_extension e um arquivo twig_svg_extension.services.yml:

services:
  twig_svg_extension.twig_extension:
    class: Drupal\twig_svg_extension\TwigExtension\TwigSvgExtension
    arguments: ['@theme.manager']
    tags:
      - { name: twig.extension }

Por favor, não a chave arguments que diz ao Drupal o serviço a ser injetado.

namespace Drupal\twig_svg_Extension\TwigExtension;

use Drupal\Core\Theme\ThemeManager;
use Twig_Extension;
use Twig_SimpleFilter;

class TwigSvgExtension extends Twig_Extension  {
  private $theme;
  
  // Dependency injection at work!
  public function __construct(ThemeManager $theme) {
    $this->theme = $theme;
  }

  public function getFilters() {
    return [
      'svg' =>new Twig_SimpleFilter('svg', [$this, 'svgFilter']),
    ];
  }

  public function getName() {
    return 'twig_svg_extension.twig_extension';
  }

  public function svgFilter(string $filepath) {
    $realpath = realpath($this->theme->getActiveTheme()->getPath().DIRECTORY_SEPARATOR.$filepath);
    $pathinfo = pathinfo($realpath);

    if($realpath !== false && strtolower($pathinfo['extension']) === 'svg') {
      return file_get_contents($realpath);
    }

    return '"'.$filepath.'" does not exist or is not an SVG';
  }
}

Observe o construtor que contém a dependência que injetamos na configuração do serviço, bem como o svgFilter que obtém o caminho do tema ativo atual.

$filepath deve ser um caminho relativo para sua pasta de temas. A extensão converterá o caminho do arquivo no conteúdo do arquivo para o qual aponta.
