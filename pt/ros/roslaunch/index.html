iniciar nós ros e carregar parâmetros do arquivo Yaml #

roslaunch é uma ferramenta importante que gerencia o início e a parada dos nós ROS. Leva um ou mais arquivos “*.launch” como argumentos.

Para este exemplo, vou me referir ao seguinte (como solicitado nesta pergunta), então como podemos executar esses comandos consecutivamente e automaticamente:

roscd stereo_camera
rosparam load marvin_cameras.yaml
rosrun stereo_camera stereo_camera __name:=bumblebeeLeft
rosrun stereo_camera stereo_camera __name:=bumblebeeCenter

roslaunch openni_launch_marvin kinect_left.launch
roslaunch openni_launch_marvin kinect_center.launch

Primeiro de tudo, vamos dividir esses comandos em pedaços. Como pode ser visto, são necessários 4 comandos ros: roscd, rosparam, rosrun e roslaunch. Agora vamos começar!

Este é um exemplo de porque o roslaunch é poderoso. Na verdade, todos esses comandos podem ser incluídos em um e único arquivo roslaunch no ROS. então tudo o que temos a fazer é lançar este arquivo “solution.launch” para uma chamada consecutiva e automática para esses comandos.

Para começar, os arquivos de inicialização são baseados na formatação XML, aqui está um arquivo de inicialização básico no ROS, vamos chamá-lo de “basic_example.launch” e está incluído em um pacote ROS chamado “roslaunch_example”:

<launch>
    
</launch>

o comando para executar este arquivo de inicialização é

$ roslaunch roslaunch_example basic_example.launch

seguindo a especificação:

$ roslaunch package_name launch_file_name.launch

Como nosso arquivo de inicialização não inclui nenhum comando, nada será executado, mais sobre isso depois…

Incluindo nós nos arquivos de inicialização : qualquer nó ROS em qualquer pacote ROS instalado pode ser chamado em arquivos de inicialização. para isso temos que especificar o pacote que contém o nó e seu nome conforme especificado no pacote. por exemplo :

rosrun stereo_camera stereo_camera __name:=bumblebeeLeft
rosrun stereo_camera stereo_camera __name:=bumblebeeCenter

stereo_camera é um nó do pacote stereo_camera e os argumentos especificados são seu nome __name:=bumblebeeLeft e __name:=bumblebeeCenter.

Para adicionar esses nós, temos que adicionar as seguintes linhas:

<launch>    
    <node name="$(arg name)" pkg="stereo_camera" type="stereo_camera" output="screen">
          <param name="name" value="bumblebeeLeft" />
    </node>

    <node name="$(arg name)" pkg="stereo_camera" type="stereo_camera" output="screen">
          <param name="name" value="bumblebeeCenter" />
    </node>
</launch>

Executando este arquivo de inicialização, teremos os dois nós em execução.

adicionando um parâmetro a um nó ROS:

Como pode ser visto, adicionamos um parâmetro “name” para os nós como:

<param name="name" value="bumblebeeCenter" />

Na verdade, podemos adicionar quantos parâmetros quisermos (como criado acima) e depois nos referirmos a eles chamando "$(arg parameter_name)" em vez de fixar seu valor.

Especificando a saída:

A tag de saída pode ser definida como “screen”, se você precisar ver o log do nó no terminal ou “log” para salvar o log nos arquivos de log em (~/.ros).

Incluindo outros arquivos de inicialização do ROS em um arquivo de inicialização do ROS :

Conforme indicado aqui, a tag permite importar outro arquivo XML roslaunch para o arquivo atual. Ele será importado dentro do escopo atual do seu documento, incluindo as tags e . Todo o conteúdo do arquivo de inclusão será importado, exceto a tag : a tag só é obedecida no arquivo de nível superior.

Então, tudo para executa esses comandos

roslaunch openni_launch_marvin kinect_left.launch
roslaunch openni_launch_marvin kinect_center.launch

de um arquivo de inicialização, tudo o que precisamos fazer é adicionar as seguintes linhas:

<include file="$(find openni_launch_marvin)/launch/kinect_left.launch" />
<include file="$(find openni_launch_marvin)/launch/kinect_center.launch" />

roscd para um pacote no arquivo de inicialização do ROS

Para encontrar o arquivo de inicialização que queremos incluir, não precisamos especificar o caminho completo. Em vez disso, o roslaunch fornece a diretiva "$(find package_name)", desta forma, podemos nos referir ao nosso arquivo de lançamento relativo ao pacote racine. No exemplo acima, assumi que o arquivo “kinect_center.launch” está na pasta “openni_launch_marvin)/launch/”.

Carregando parâmetros do arquivo YAML :

Para carregar parâmetros de um arquivo YAML no ROS, o ROS fornece a tag “rosparam”. Conforme declarado no wiki : “A tag permite o uso de arquivos YAML rosparam para carregar e despejar parâmetros do ROS Parameter Server. Ela também pode ser usada para remover parâmetros. A tag pode ser colocado dentro de uma tag , nesse caso o parâmetro é tratado como um nome privado.”

Usando esta tag, podemos carregar nosso arquivo YAML no arquivo de inicialização adicionando esta linha:

<rosparam command="load" file="$(find marvin_cameras)/config/marvin_cameras.yaml" />

Conforme usado acima, presumi que o arquivo YAML “marvin_cameras.yaml” está na pasta “marvin_cameras/config/”.

Montagem de todas as peças

Agora que criamos separadamente o conteúdo do nosso arquivo de inicialização, vamos montá-los em um grande arquivo de inicialização “solution.launch”.

solução.lançamento

<launch> 

    <rosparam command="load" file="$(find marvin_cameras)/config/marvin_cameras.yaml" />  

    <node name="$(arg name)" pkg="stereo_camera" type="stereo_camera" output="screen">
          <param name="name" value="bumblebeeLeft" />
    </node>

    <node name="$(arg name)" pkg="stereo_camera" type="stereo_camera" output="screen">
          <param name="name" value="bumblebeeCenter" />
    </node>

    <include file="$(find openni_launch_marvin)/launch/kinect_left.launch" />
    <include file="$(find openni_launch_marvin)/launch/kinect_center.launch" />

</launch>

Agora, temos nosso único arquivo roslaunch para executar todos os comandos consecutiva e automaticamente.
