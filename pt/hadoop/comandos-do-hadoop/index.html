Sintaxe #

  • Comandos Hadoop v1: hadoop fs -<comando>

  • Comandos do Hadoop v2: hdfs dfs -<comando>

Comandos do Hadoop v1 #

1. Imprima a versão do Hadoop #

hadoop version

2. Liste o conteúdo do diretório raiz no HDFS #

#

hadoop fs -ls /

3. Relate a quantidade de espaço usado e #

disponível no sistema de arquivos atualmente montado #

#

hadoop fs -df hdfs:/

4. Conte o número de diretórios, arquivos e bytes em #

os caminhos que correspondem ao padrão de arquivo especificado #

#

hadoop fs -count hdfs:/

5. Execute um utilitário de verificação do sistema de arquivos DFS #

#

hadoop fsck – /

6. Execute um utilitário de balanceamento de cluster #

#

hadoop balancer

7. Crie um novo diretório chamado “hadoop” abaixo do #

diretório /user/training no HDFS. Já que você está #

atualmente logado com o ID de usuário “treinamento”, #

/user/training é seu diretório inicial no HDFS. #

#

hadoop fs -mkdir /user/training/hadoop

8. Adicione um arquivo de texto de amostra do diretório local #

chamado “data” para o novo diretório que você criou no HDFS #

durante a etapa anterior. #

#

hadoop fs -put data/sample.txt /user/training/hadoop

9. Liste o conteúdo deste novo diretório no HDFS. #

#

hadoop fs -ls /user/training/hadoop

10. Adicione todo o diretório local chamado “varejo” ao #

diretório /user/training no HDFS. #

#

hadoop fs -put data/retail /user/training/hadoop

11. Como /user/training é seu diretório inicial no HDFS, #

qualquer comando que não tenha um caminho absoluto é #

interpretado como relativo a esse diretório. Nas próximas #

comando listará, portanto, seu diretório pessoal e #

deve mostrar os itens que você acabou de adicionar lá. #

#

hadoop fs -ls

12. Veja quanto espaço este diretório ocupa no HDFS. #

#

hadoop fs -du -s -h hadoop/retail

13. Exclua um arquivo ‘clientes’ do diretório “varejo”. #

#

hadoop fs -rm hadoop/retail/customers

14. Certifique-se de que este arquivo não esteja mais no HDFS. #

#

hadoop fs -ls hadoop/retail/customers

15. Exclua todos os arquivos do diretório “varejo” usando um curinga. #

#

hadoop fs -rm hadoop/retail/*

16. Para esvaziar a lixeira #

#

hadoop fs -expunge

17. Por fim, remova todo o diretório de varejo e todos os #

de seu conteúdo em HDFS. #

#

hadoop fs -rm -r hadoop/retail

18. Liste o diretório do hadoop novamente #

#

hadoop fs -ls hadoop

19. Adicione o arquivo Purchases.txt do diretório local #

chamado “/home/training/” para o diretório hadoop que você criou no HDFS #

#

hadoop fs -copyFromLocal /home/training/purchases.txt hadoop/

20. Para visualizar o conteúdo do seu arquivo de texto Purchases.txt #

que está presente em seu diretório hadoop. #

#

hadoop fs -cat hadoop/purchases.txt

21. Adicione o arquivo Purchases.txt do diretório “hadoop” que está presente no diretório HDFS #

para o diretório “data” que está presente em seu diretório local #

#

hadoop fs -copyToLocal hadoop/purchases.txt /home/training/data

22. cp é usado para copiar arquivos entre diretórios presentes no HDFS #

#

hadoop fs -cp /user/training/*.txt /user/training/hadoop

23. O comando ‘-get’ pode ser usado alternativamente ao comando ‘-copyToLocal’ #

#

hadoop fs -get hadoop/sample.txt /home/training/

24. Exiba o último kilobyte do arquivo “purchases.txt” para stdout. #

#

hadoop fs -tail hadoop/purchases.txt

25. As permissões de arquivo padrão são 666 no HDFS #

Use o comando ‘-chmod’ para alterar as permissões de um arquivo #

#

hadoop fs -ls hadoop/purchases.txt
sudo -u hdfs hadoop fs -chmod 600 hadoop/purchases.txt

26. Os nomes padrão do proprietário e do grupo são training,training #

Use ‘-chown’ para alterar o nome do proprietário e o nome do grupo simultaneamente #

#

hadoop fs -ls hadoop/purchases.txt
sudo -u hdfs hadoop fs -chown root:root hadoop/purchases.txt

27. O nome padrão do grupo é treinamento #

Use o comando ‘-chgrp’ para alterar o nome do grupo #

#

hadoop fs -ls hadoop/purchases.txt
sudo -u hdfs hadoop fs -chgrp training hadoop/purchases.txt

28. Mova um diretório de um local para outro #

#

hadoop fs -mv hadoop apache_hadoop

29. O fator de replicação padrão para um arquivo é 3. #

Use o comando ‘-setrep’ para alterar o fator de replicação de um arquivo #

#

hadoop fs -setrep -w 2 apache_hadoop/sample.txt

30. Copie um diretório de um nó do cluster para outro #

Use o comando ‘-distcp’ para copiar, #

-overwrite opção para sobrescrever em arquivos existentes #

-update comando para sincronizar os dois diretórios #

#

hadoop fs -distcp hdfs://namenodeA/apache_hadoop hdfs://namenodeB/hadoop

31. Comando para fazer o nó do nome sair do modo de segurança #

#

hadoop fs -expunge
sudo -u hdfs hdfs dfsadmin -safemode leave

32. Liste todos os comandos shell do sistema de arquivos do hadoop #

#

hadoop fs

33. Obtenha valores de cota de hdfs e a contagem atual de nomes e bytes em uso. #

#

hadoop fs -count -q [-h] [-v] <directory>...<directory>

34. Por último, mas não menos importante, sempre peça ajuda! #

#

hadoop fs -help

Comandos do Hadoop v2 #

appendToFile: anexa um único src ou vários srcs do sistema de arquivos local ao sistema de arquivos de destino. Também lê a entrada de stdin e anexa ao sistema de arquivos de destino. Mantenha o como -

 hdfs dfs -appendToFile [localfile1 localfile2 ..] [/HDFS/FILE/PATH..]

cat: copia os caminhos de origem para stdout.

 hdfs dfs -cat URI [URI …]

chgrp: altera a associação do grupo de arquivos. Com -R, faz a alteração recursivamente por meio da estrutura de diretórios. O usuário deve ser o proprietário do arquivo ou o superusuário.

 hdfs dfs -chgrp [-R] GROUP URI [URI …]

chmod: altera as permissões dos arquivos. Com -R, faz a alteração recursivamente por meio da estrutura de diretórios. O usuário deve ser o proprietário do arquivo ou o superusuário

 hdfs dfs -chmod [-R] <MODE[,MODE]... | OCTALMODE> URI [URI …]

chown: altera o proprietário dos arquivos. Com -R, faz a alteração recursivamente por meio da estrutura de diretórios. O usuário deve ser o superusuário.

 hdfs dfs -chown [-R] [OWNER][:[GROUP]] URI [URI ]

copyFromLocal: funciona de forma semelhante ao comando put, exceto que a fonte é restrita a uma referência de arquivo local.

 hdfs dfs -copyFromLocal <localsrc> URI

copyToLocal: funciona de forma semelhante ao comando get, exceto que o destino é restrito a uma referência de arquivo local.

 hdfs dfs -copyToLocal [-ignorecrc] [-crc] URI <localdst>

count: conta o número de diretórios, arquivos e bytes nos caminhos que correspondem ao padrão de arquivo especificado.

 hdfs dfs -count [-q] [-h] <paths>

cp: Copia um ou mais arquivos de uma fonte especificada para um destino especificado. Se você especificar várias origens, o destino especificado deverá ser um diretório.

 hdfs dfs -cp URI [URI …] <dest>

du: Exibe o tamanho do arquivo especificado ou os tamanhos dos arquivos e diretórios contidos no diretório especificado. Se você especificar a opção -s, exibirá um resumo agregado de tamanhos de arquivo em vez de tamanhos de arquivo individuais. Se você especificar a opção -h, formata os tamanhos dos arquivos de forma “legível para humanos”.

 hdfs dfs -du [-s] [-h] URI [URI …]

dus: exibe um resumo dos tamanhos dos arquivos; equivalente a hdfs dfs -du –s.

 hdfs dfs -dus <args>

expurgar: esvazia a lixeira. Quando você exclui um arquivo, ele não é removido imediatamente do HDFS, mas é renomeado para um arquivo no diretório /trash. Enquanto o arquivo permanecer lá, você poderá recuperá-lo se mudar de ideia, embora apenas a cópia mais recente do arquivo excluído possa ser restaurada.

 hdfs dfs –expunge

get: Copia arquivos para o sistema de arquivos local. Arquivos que falham em uma verificação de redundância cíclica (CRC) ainda podem ser copiados se você especificar a opção -ignorecrc. O CRC é uma técnica comum para detectar erros de transmissão de dados. Os arquivos de soma de verificação CRC têm a extensão .crc e são usados ​​para verificar a integridade dos dados de outro arquivo. Esses arquivos são copiados se você especificar a opção -crc.

 hdfs dfs -get [-ignorecrc] [-crc] <src> <localdst>

getmerge: concatena os arquivos em src e grava o resultado no arquivo de destino local especificado. Para adicionar um caractere de nova linha no final de cada arquivo, especifique a opção addnl.

 hdfs dfs -getmerge <src> <localdst> [addnl]

ls: Retorna estatísticas para os arquivos ou diretórios especificados.

 hdfs dfs -ls <args>

lsr: funciona como a versão recursiva de ls; semelhante ao comando Unix ls -R.

 hdfs dfs -lsr <args>

mkdir: cria diretórios em um ou mais caminhos especificados. Seu comportamento é semelhante ao comando Unix mkdir -p, que cria todos os diretórios que levam ao diretório especificado, caso ainda não existam.

 hdfs dfs -mkdir <paths>

moveFromLocal: Funciona de forma semelhante ao comando put, exceto que a fonte é excluída após ser copiada.

 hdfs dfs -moveFromLocal <localsrc> <dest>

mv: move um ou mais arquivos de uma origem especificada para um destino especificado. Se você especificar várias origens, o destino especificado deverá ser um diretório. Não é permitido mover arquivos entre sistemas de arquivos.

 hdfs dfs -mv URI [URI …] <dest>

put: Copia arquivos do sistema de arquivos local para o sistema de arquivos de destino. Este comando também pode ler a entrada de stdin e gravar no sistema de arquivos de destino.

 hdfs dfs -put <localsrc> ... <dest>

rm: exclui um ou mais arquivos especificados. Este comando não exclui diretórios ou arquivos vazios. Para ignorar a lixeira (se estiver ativada) e excluir os arquivos especificados imediatamente, especifique a opção -skipTrash.

 hdfs dfs -rm [-skipTrash] URI [URI …]

rm r: Serve como a versão recursiva de –rm.

 hdfs dfs -rm -r [-skipTrash] URI [URI …]

setrep: altera o fator de replicação de um arquivo ou diretório especificado. Com -R, faz a alteração recursivamente por meio da estrutura de diretórios.

 hdfs dfs -setrep <rep> [-R] <path>

stat: Exibe informações sobre o caminho especificado.

 hdfs dfs -stat URI [URI …]

tail: exibe o último kilobyte de um arquivo especificado para stdout. A sintaxe suporta a opção Unix -f, que permite que o arquivo especificado seja monitorado. À medida que novas linhas são adicionadas ao arquivo por outro processo, tail atualiza a exibição.

 hdfs dfs -tail [-f] URI

test: Retorna atributos do arquivo ou diretório especificado. Especifica -e para determinar se o arquivo ou diretório existe; -z para determinar se o arquivo ou diretório está vazio; e -d para determinar se o URI é um diretório.

 hdfs dfs -test -[ezd] URI

texto: Gera um arquivo de origem especificado em formato de texto. Os formatos de arquivo de entrada válidos são zip e TextRecordInputStream.

 hdfs dfs -text <src>

touchz: cria um novo arquivo vazio de tamanho 0 no caminho especificado.

 hdfs dfs -touchz <path>
