Expressões condicionais, envolvendo palavras-chave como if, else if e else, fornecem aos programas Swift a capacidade de realizar ações diferentes dependendo de uma condição booleana: True ou False. Esta seção aborda o uso de condicionais Swift, lógica booleana e instruções ternárias.

Para obter mais informações sobre instruções condicionais, consulte The Swift Programming Language.

Cláusulas de vinculação e “where” opcionais #

Optionals devem ser desempacotados antes de poderem ser usados ​​na maioria das expressões. if let é uma vinculação opcional, que funciona se o valor opcional for não nil:

let num: Int? = 10 // or: let num: Int? = nil

if let unwrappedNum = num {
    // num has type Int?; unwrappedNum has type Int
    print("num was not nil: \(unwrappedNum + 1)")
} else {
    print("num was nil")
}

Você pode reutilizar o mesmo nome para a variável recém-vinculada, sombreando o original:

// num originally has type Int?
if let num = num {
    // num has type Int inside this block
}

Combine várias ligações opcionais com vírgulas (,):

if let unwrappedNum = num, let unwrappedStr = str {
    // Do something with unwrappedNum & unwrappedStr
} else if let unwrappedNum = num {
    // Do something with unwrappedNum
} else {
    // num was nil
}

Aplique outras restrições após a vinculação opcional usando uma cláusula where:

if let unwrappedNum = num onde unwrappedNum % 2 == 0 {
    print("num is non-nil, and it's an even number")
}

Se você estiver se sentindo aventureiro, intercale qualquer número de ligações opcionais e cláusulas where:

if let num = num                           // num must be non-nil
    where num % 2 == 1,                    // num must be odd
    let str = str,                         // str must be non-nil
    let firstChar = str.characters.first   // str must also be non-empty
    where firstChar != "x"                 // the first character must not be "x"
{
    // all bindings & conditions succeeded!
}

No Swift 3, as cláusulas where foram substituídas (SE-0099): simplesmente use outro , para separar ligações opcionais e condições booleanas.

if let unwrappedNum = num, unwrappedNum % 2 == 0 {
    print("num is non-nil, and it's an even number")
}

if let num = num, // num deve ser diferente de zero
    num % 2 == 1,                           // num must be odd
    let str = str,                          // str must be non-nil
    let firstChar = str.characters.first,   // str must also be non-empty
    firstChar != "x"                        // the first character must not be "x"
{
    // all bindings & conditions succeeded!
}

Usando Guarda #

O Guard verifica uma condição e, se for falsa, entra na ramificação. As ramificações de verificação de guarda devem deixar seu bloco delimitador via return, break ou continue (se aplicável); não fazer isso resulta em um erro do compilador. Isso tem a vantagem de que quando um guard é escrito não é possível deixar o fluxo continuar acidentalmente (como seria possível com um if).

O uso de guardas pode ajudar a manter os níveis de aninhamento baixos, o que geralmente melhora a legibilidade do código .

func printNum(num: Int) {
    guard num == 10 else {
        print("num is not 10")
        return
    }
    print("num is 10")
}

O Guard também pode verificar se há um valor em um opcional e, em seguida, desdobrá-lo no escopo externo:

func printOptionalNum(num: Int?) {
    guard let unwrappedNum = num else {
        print("num does not exist")
        return
    }
    print(unwrappedNum)
}

O Guard pode combinar opcional desempacotamento e verificação de condição usando a palavra-chave where:

func printOptionalNum(num: Int?) {
guard let unwrappedNum = num, unwrappedNum == 10 else {
    print("num does not exist or is not 10")
    return
}
print(unwrappedNum)
}

Condicionais básicos: instruções if #

Uma instrução if verifica se uma condição Bool é true:

let num = 10

if num == 10 {
    // Code inside this block only executes if the condition was true.
    print("num is 10")
}

let condition = num == 10   // condition's type is Bool
if condition {
    print("num is 10")
}

As instruções if aceitam os blocos else if e else, que podem testar condições alternativas e fornecer um substituto:

let num = 10
if num < 10 {  // Execute the following code if the first condition is true.
    print("num is less than 10")
} else if num == 10 {  // Or, if not, check the next condition...
    print("num is 10")
} else {  // If all else fails...
    print("all other conditions were false, so num is greater than 10")
}

Operadores básicos como && e || podem ser usados ​​para várias condições:

O operador lógico AND #

let num = 10
let str = "Hi"
if num == 10 && str == "Hi" {
    print("num is 10, AND str is \"Hi\"")
}

Se num == 10 fosse falso, o segundo valor não seria avaliado. Isso é conhecido como avaliação de curto-circuito.

O operador lógico OR #

if num == 10 || str == "Hi" {
    print("num is 10, or str is \"Hi\")
}

Se num == 10 for true, o segundo valor não será avaliado.

O operador lógico NOT #

if !str.isEmpty {
    print("str is not empty")
}

Operador ternário #

As condições também podem ser avaliadas em uma única linha usando o operador ternário:

Se você quisesse determinar o mínimo e o máximo de duas variáveis, você poderia usar instruções if, assim:

let a = 5
let b = 10
let min: Int

if a < b {
    min = a 
} else {
    min = b 
}

let max: Int

if a > b {
    max = a 
} else {
    max = b 
}

O operador condicional ternário pega uma condição e retorna um de dois valores, dependendo se a condição era verdadeira ou falsa. A sintaxe é a seguinte: Isso é equivalente a ter a expressão:

(<CONDITION>) ? <TRUE VALUE> : <FALSE VALUE>

O código acima pode ser reescrito usando o operador condicional ternário como abaixo:

let a = 5
let b = 10
let min = a < b ? a : b
let max = a > b ? a : b

No primeiro exemplo, a condição é a < b. Se isso for verdade, o resultado atribuído de volta a min será de a; se for falso, o resultado será o valor de b.

Nota: Como encontrar o maior ou menor de dois números é uma operação tão comum, a biblioteca padrão do Swift oferece duas funções para esse propósito: max e min.

Operador de coalescência nula #

O operador nil-coalescent <OPTIONAL> ?? <VALOR PADRÃO> desempacota o <OPCIONAL> se contiver um valor, ou retorna <VALOR PADRÃO> se for nil. <OPCIONAL> é sempre de um tipo opcional. <VALOR PADRÃO> deve corresponder ao tipo que está armazenado dentro de <OPCIONAL>.

O operador nil-coalescing é uma abreviação para o código abaixo que usa um operador ternário:

a != nil ? a! : b

isso pode ser verificado pelo código abaixo:

(a ?? b) == (a != nil ? a! : b) // ouputs true

Tempo para um exemplo

let defaultSpeed:String = "Slow"
var userEnteredSpeed:String? = nil

print(userEnteredSpeed ?? defaultSpeed) // ouputs "Slow"

userEnteredSpeed = "Fast"
print(userEnteredSpeed ?? defaultSpeed) // ouputs "Fast"
