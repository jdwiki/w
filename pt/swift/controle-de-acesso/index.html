Sintaxe #

  • Projeto de aula particular
  • let car = Car(“Ford”, model: “Escape”) //padrão interno
  • gênero enum público
  • função privada calcularMarketCap()
  • substituir a função interna setupView()
  • private(set) var area = 0
  1. Observação Básica:

Abaixo estão os três níveis de acesso do acesso mais alto (menos restritivo) ao acesso mais baixo (mais restritivo)

O acesso público permite o acesso a classes, structs, variáveis, etc. de qualquer arquivo dentro do modelo, mas mais importante fora do módulo se o arquivo externo importar o módulo que contém o código de acesso público. É popular usar o acesso público quando você cria uma estrutura.

O acesso interno permite que arquivos apenas com o módulo das entidades usem as entidades. Todas as entidades têm nível de acesso interno por padrão (com algumas exceções).

O acesso Privado impede que a entidade seja usada fora desse arquivo.

  1. Observação de Subclasse:

Uma subclasse não pode ter um acesso maior que sua superclasse.

  1. Observação do Getter e do Setter:

Se o setter da propriedade for privado, o getter será interno (que é o padrão). Além disso, você pode atribuir o nível de acesso para o getter e o setter. Esses princípios também se aplicam a subscritos

  1. Observação Geral:

Outros tipos de entidade incluem: inicializadores, protocolos, extensões, genéricos e aliases de tipo

Exemplo básico usando um struct #

No Swift 3 existem vários níveis de acesso. Este exemplo usa todos eles, exceto open:

public struct Car {
    
    public let make: String
    let model: String //Optional keyword: will automatically be "internal"
    private let fullName: String
    fileprivate var otherName: String
    
    public init(_ make: String, model: String) {
        self.make = make
        self.model = model
        self.fullName = "\(make)\(model)"
        self.otherName = "\(model) - \(make)"
    }
}

Suponha que myCar foi inicializado assim:

let myCar = Car("Apple", model: "iCar")

Car.make (público) #

print(myCar.make)

Esta impressão funcionará em qualquer lugar, incluindo alvos que importam Car.

Carro.modelo (interno) #

print(myCar.model)

Isso será compilado se o código estiver no mesmo destino que Car.

Car.otherName (fileprivate) #

print(myCar.otherName)

Isso só funcionará se o código estiver no mesmo arquivo que Car.

Car.fullName (privado) #

print(myCar.fullName)

Isso não funcionará no Swift 3. Propriedades private só podem ser acessadas dentro da mesma struct/class.

public struct Car {

    public let make: String       //public
    let model: String             //internal
    private let fullName: String! //private 

    public init(_ make: String, model model: String) {
        self.make = make
        self.model = model
        self.fullName = "\(make)\(model)"
    }
}

Se a entidade tiver vários níveis de acesso associados, o Swift procura o nível mais baixo de acesso. Se existir uma variável privada em uma classe pública, a variável ainda será considerada privada.

Exemplo de subclasse #

public class SuperClass {
    private func secretMethod() {}
}
 
internal class SubClass: SuperClass {
    override internal func secretMethod() {
        super.secretMethod()
    }
}

Exemplo de getters e setters #

struct Square {
    private(set) var area = 0

    var side: Int = 0 {
        didSet {
            area = side*side
        }
    }
}

public struct Square {
    public private(set) var area = 0
    public var side: Int = 0 {
        didSet {
            area = side*side
        }
    }
    public init() {}
}
