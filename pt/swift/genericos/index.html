O código genérico permite que você escreva funções e tipos flexíveis e reutilizáveis ​​que podem funcionar com qualquer tipo, sujeito aos requisitos que você definir. Você pode escrever um código que evite a duplicação e expresse sua intenção de maneira clara e abstrata.

Genéricos são um dos recursos mais poderosos do Swift, e grande parte da biblioteca padrão do Swift é construída com código genérico. Por exemplo, os tipos Array e Dictionary do Swift são coleções genéricas. Você pode criar um array que contém valores Int, ou um array que contém valores String, ou mesmo um array para qualquer outro tipo que possa ser criado no Swift. Da mesma forma, você pode criar um dicionário para armazenar valores de qualquer tipo especificado e não há limitações sobre qual tipo pode ser.

Fonte: Linguagem de programação Swift da Apple

O básico dos genéricos #

Genéricos são espaços reservados para tipos, permitindo que você escreva código flexível que pode ser aplicado em vários tipos. A vantagem de usar genéricos sobre Any é que eles ainda permitem que o compilador imponha uma segurança de tipo forte.

Um espaço reservado genérico é definido entre colchetes angulares <>.

Funções genéricas #

Para functions, este espaço reservado é colocado após o nome da função:

/// Escolhe uma das entradas aleatoriamente e a retorna
func pickRandom<T>(_ a:T, _ b:T) -> T {
    return arc4random_uniform(2) == 0 ? a : b
}

Neste caso, o placeholder genérico é T. Quando você chama a função, o Swift pode inferir o tipo de T para você (já que ele simplesmente age como um espaço reservado para um tipo real).

let randomOutput = pickRandom(5, 7) // returns an Int (that's either 5 or 7)

Aqui estamos passando dois inteiros para a função. Portanto Swift está inferindo T == Int – assim a assinatura da função é inferida como (Int, Int) -> Int.

Por causa da forte segurança de tipo que os genéricos oferecem – tanto os argumentos quanto o retorno da função devem ser do mesmo tipo. Portanto, o seguinte não será compilado:

struct Foo {}

let foo = Foo()

let randomOutput = pickRandom(foo, 5) // error: cannot convert value of type 'Int' to expected argument type 'Foo'

Tipos genéricos #

Para usar genéricos com classes, structs ou enums, você pode definir o espaço reservado genérico após o nome do tipo.

Barra de classe<T> {
    var baz : T
    
    init(baz:T) {
        self.baz = baz
    }
}

Este placeholder genérico exigirá um tipo quando você usar a classe Bar. Neste caso, pode ser inferido do inicializador init(baz:T).

let bar = Bar(baz: "a string") // bar's type is Bar<String>

Aqui o placeholder genérico T é inferido como sendo do tipo String, criando assim uma instância Bar<String>. Você também pode especificar o tipo explicitamente:

let bar = Bar<String>(baz: "uma string")

Quando usado com um tipo, o espaço reservado genérico fornecido manterá seu tipo durante todo o tempo de vida da instância especificada e não poderá ser alterado após a inicialização. Portanto, ao acessar a propriedade baz, ela sempre será do tipo String para esta dada instância.

let str = bar.baz // of type String

Transmitindo tipos genéricos #

Quando você passa por cima de tipos genéricos, na maioria dos casos você tem que ser explícito sobre o tipo de espaço reservado genérico que você espera. Por exemplo, como uma entrada de função:

func takeABarInt(bar:Bar<Int>) {
    ...
}

Esta função só aceitará um Bar<Int>. A tentativa de passar em uma instância Bar onde o tipo de espaço reservado genérico não é Int resultará em um erro do compilador.

Nomenclatura genérica de espaço reservado #

Os nomes genéricos de espaços reservados não se limitam apenas a letras simples. Se um determinado espaço reservado representa um conceito significativo, você deve dar a ele um nome descritivo. Por exemplo, Array do Swift tem um placeholder genérico chamado Element, que define o tipo de elemento de uma determinada instância Array.

Public struct Array<Element> : RandomAccessCollection, MutableCollection {
    ...
}

Restringindo tipos de placeholder genéricos #

É possível forçar os parâmetros de tipo de uma classe genérica para implementar um protocolo, por exemplo, [Equatable](https://developer .apple.com/library/ios/documentation/Swift/Reference/Swift_Equatable_Protocol/index.html)

class MyGenericClass<Tipo: Equatable>{
    
    var value: Type
    init(value: Type){
        self.value = value
    }
    
    func getValue() -> Type{
        return self.value
    }

    func valueEquals(anotherValue: Type) -> Bool{
        return self.value == anotherValue
    }
}

Sempre que criamos um novo MyGenericClass, o parâmetro type tem que implementar o protocolo Equatable (garantindo que o parâmetro type possa ser comparado a outra variável do mesmo tipo usando ==)

let myFloatGeneric = MyGenericClass<Double>(valor: 2,71828) // válido
let myStringGeneric = MyGenericClass<String>(value: "My String") // válido

// "Tipo [Int] não está em conformidade com o protocolo 'Equatable'"
let myInvalidGeneric = MyGenericClass<[Int]>(valor: [2])

let myIntGeneric = MyGenericClass<Int>(valor: 72)
print(myIntGeneric.valueEquals(72)) // true
print(myIntGeneric.valueEquals(-274)) // false

// "Não é possível converter o valor do tipo 'String' para o tipo de argumento esperado 'Int'"
print(myIntGeneric.valueEquals("Minha String"))

Exemplos de classes genéricas #

Uma classe genérica com o parâmetro de tipo Type

classe MyGenericClass<Tipo>{
 
    var value: Type
    init(value: Type){
        self.value = value
    }
 
    func getValue() -> Type{
        return self.value
    }
 
    func setValue(value: Type){
        self.value = value
    }
}

Agora podemos criar novos objetos usando um parâmetro de tipo

let myStringGeneric = MyGenericClass<String>(value: "My String Value")
let myIntGeneric = MyGenericClass<Int>(valor: 42)
 
print(myStringGeneric.getValue()) // "Meu valor de string"
print(myIntGeneric.getValue()) // 42
 
myStringGeneric.setValue("Outra String")
myIntGeneric.setValue(1024)
 
print(myStringGeneric.getValue()) // "Outra String"
print(myIntGeneric.getValue()) // 1024

Os genéricos também podem ser criados com vários parâmetros de tipo

classe OutraGenericClass<TypeOne, TypeTwo, TypeThree>{
 
    var value1: TypeOne
    var value2: TypeTwo
    var value3: TypeThree
    init(value1: TypeOne, value2: TypeTwo, value3: TypeThree){
        self.value1 = value1
        self.value2 = value2
        self.value3 = value3
    }
 
    func getValueOne() -> TypeOne{return self.value1}
    func getValueTwo() -> TypeTwo{return self.value2}
    func getValueThree() -> TypeThree{return self.value3}
}

E usado da mesma maneira

let myGeneric = AnotherGenericClass<String, Int, Double>(valor1: "Valor de pi", valor2: 3, valor3: 3,14159)
 
print(myGeneric.getValueOne() é String) // true
print(myGeneric.getValueTwo() é Int) // true
print(myGeneric.getValueThree() is Double) // true
print(myGeneric.getValueTwo() é String) // false
 
print(myGeneric.getValueOne()) // "Valor de pi"
print(myGeneric.getValueTwo()) // 3
print(myGeneric.getValueThree()) // 3.14159

Usando genéricos para simplificar funções de matriz #

Uma função que estende a funcionalidade da matriz criando uma função de remoção orientada a objetos.

// Need to restrict the extension to elements that can be compared.
// The `Element` is the generics name defined by Array for its item types.
// This restriction also gives us access to `index(of:_)` which is also
// defined in an Array extension with `where Element: Equatable`.
public extension Array where Element: Equatable {
    /// Removes the given object from the array.
    mutating func remove(_ element: Element) {
        if let index = self.index(of: element ) {
            self.remove(at: index)
        } else {
            fatalError("Removal error, no such element:\"\(element)\" in array.\n")
        }
    }
}

Uso

var myArray = [1,2,3]
print(myArray)

// Prints [1,2,3]

Use a função para remover um elemento sem a necessidade de um índice. Basta passar o objeto para remover.

myArray.remove(2)
print(myArray)

// Prints [1,3]

Herança de classe genérica #

As classes genéricas podem ser herdadas:

// Models
class MyFirstModel {
}

class MySecondModel: MyFirstModel {
}

// Generic classes
class MyFirstGenericClass<T: MyFirstModel> {
    
    func doSomethingWithModel(model: T) {
        // Do something here
    }
    
}

class MySecondGenericClass<T: MySecondModel>: MyFirstGenericClass<T> {
    
    override func doSomethingWithModel(model: T) {
        super.doSomethingWithModel(model)
        
        // Do more things here
    }
    
}

Use genéricos para melhorar a segurança do tipo #

Vamos pegar este exemplo sem usar genéricos

protocol JSONDecodable {
    static func from(_ json: [String: Any]) -> Any?
}

A declaração de protocolo parece boa, a menos que você realmente a use.

let myTestObject = TestObject.from(myJson) as? TestObject

Por que você tem que converter o resultado para TestObject? Por causa do tipo de retorno Any na declaração do protocolo.

Ao usar genéricos, você pode evitar esse problema que pode causar erros de tempo de execução (e não queremos tê-los!)

protocol JSONDecodable {
    associatedtype Element 
    static func from(_ json: [String: Any]) -> Element?
}

struct TestObject: JSONDecodable {
    static func from(_ json: [String: Any]) -> TestObject? {
    }
}

let testObject = TestObject.from(myJson) // testObject is now automatically `TestObject?`

Restrições de tipo avançadas #

É possível especificar várias restrições de tipo para genéricos usando a cláusula where:

func doSomething<T where T: Comparable, T: Hashable>(first: T, second: T) {
    // Access hashable function
    guard first.hashValue == second.hashValue else {
        return
    }
    // Access comparable function
    if first == second {
        print("\(first) and \(second) are equal.")
    }
}

Também é válido escrever a cláusula where após a lista de argumentos:

func doSomething<T>(first: T, second: T) where T: Comparable, T: Hashable {
    // Access hashable function
    guard first.hashValue == second.hashValue else {
        return
    }
    // Access comparable function
    if first == second {
        print("\(first) and \(second) are equal.")
    }
}

As extensões podem ser restritas a tipos que satisfaçam as condições. A função está disponível apenas para instâncias que satisfaçam as condições de tipo:

// "Element" is the generics type defined by "Array". For this example, we
// want to add a function that requires that "Element" can be compared, that
// is: it needs to adhere to the Equatable protocol.
public extension Array where Element: Equatable {
    /// Removes the given object from the array.
    mutating func remove(_ element: Element) {
        // We could also use "self.index(of: element)" here, as "index(of:_)"
        // is also defined in an extension with "where Element: Equatable".
        // For the sake of this example, explicitly make use of the Equatable.
        if let index = self.index(where: { $0 == element }) {
            self.remove(at: index)
        } else {
            fatalError("Removal error, no such element:\"\(element)\" in array.\n")
        }
    }
}
