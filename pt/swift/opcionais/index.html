“Um valor opcional contém um valor ou contém nil para indicar que um valor está faltando”

Trecho de: Apple Inc. “The Swift Programming Language (Swift 3.1 Edition).” iBooks. https://itun.es/us/k5SW7.l

Os casos de uso opcionais básicos incluem: para uma constante (let), o uso de um opcional dentro de um loop (if-let), o desempacotamento seguro de um valor opcional dentro de um método (guard-let) e como parte de loops de comutação (case-let ), padronizando para um valor se nil, usando o operador coalesce (??)

Sintaxe #

  • var opcionalNome: opcionalTipo? // declara um tipo opcional, o padrão é nil
  • var opcionalNome: opcionalTipo? = value // declara um opcional com um valor
  • var opcionalNome: opcionalTipo! // declara um opcional desempacotado implicitamente
  • opcional! // força desempacotar um opcional

Para obter mais informações sobre opcionais, consulte A linguagem de programação Swift.

Tipos de Opcionais #

Opcionais são um tipo de enumeração genérico que atua como um wrapper. Este wrapper permite que uma variável tenha um de dois estados: o valor do tipo definido pelo usuário ou nil, que representa a ausência de um valor.

Essa habilidade é particularmente importante no Swift porque um dos objetivos de design declarados da linguagem é trabalhar bem com os frameworks da Apple. Muitos (a maioria) dos frameworks da Apple utilizam nil devido à sua facilidade de uso e importância para padrões de programação e design de API dentro do Objective-C.

Em Swift, para uma variável ter um valor nil, ela deve ser opcional. Opcionais podem ser criados acrescentando um ! ou um ? ao ​​tipo de variável. Por exemplo, para tornar um Int opcional, você pode usar

var numberOne: Int! = nil
var numberTwo: Int? = nil

Os opcionais ? devem ser explicitamente desempacotados e devem ser usados ​​se você não tiver certeza se a variável terá ou não um valor ao acessá-la. Por exemplo, ao transformar uma string em um Int, o resultado é um Int? opcional, porque nil será retornado se a string não for um número válido

let str1 = "42"
let num1: Int? = Int(str1) // 42

let str2 = "Hello, World!"
let num2: Int? = Int(str2) // nil

Os opcionais ! são automaticamente desempacotados e devem somente ser usados ​​quando você tiver certeza de que a variável terá um valor quando você acessá-la. Por exemplo, uma variável global UIButton! que é inicializada em viewDidLoad()

//myButton will not be accessed until viewDidLoad is called,
//so a ! optional can be used here
var myButton: UIButton!

override func viewDidLoad(){
    self.myButton = UIButton(frame: self.view.frame)
    self.myButton.backgroundColor = UIColor.redColor()
    self.view.addSubview(self.myButton)
}

Desempacotando um opcional #

Para acessar o valor de um Optional, ele precisa ser desempacotado.

Você pode desempacotar condicionalmente um Opcional usando a ligação opcional e forçar o desempacotamento um Opcional usando o operador !.

O desempacotamento condicional pergunta efetivamente “Esta variável tem um valor?” enquanto o desempacotamento forçado diz “Esta variável tem um valor!”.

Se você forçar o desempacotamento de uma variável que é nil, seu programa lançará um inesperadamente encontrado nil enquanto desempacota uma exceção opcional e trava, então você precisa considerar cuidadosamente se usar ! é apropriado.

var text: String? = nil
var unwrapped: String = text! //crashes with "unexpectedly found nil while unwrapping an Optional value"

Para um desempacotamento seguro, você pode usar uma instrução if-let, que não lançará uma exceção ou travará se o valor agrupado for nil:

var number: Int?
if let unwrappedNumber = number {       // Has `number` been assigned a value?
    print("number: \(unwrappedNumber)") // Will not enter this line
} else {
    print("number was not assigned a value")
}

Ou, uma instrução de guarda:

var number: Int?
guard let unwrappedNumber = number else {
    return
}
print("number: \(unwrappedNumber)")

Observe que o escopo da variável unwrappedNumber está dentro da instrução if-let e fora do bloco guard.

Você pode encadear o desempacotamento de muitos opcionais, isso é útil principalmente nos casos em que seu código requer mais de uma variável para ser executado corretamente:

var firstName:String?
var lastName:String?

if let fn = firstName, let ln = lastName {
    print("\(fn) + \(ln)")//pay attention that the condition will be true only if both optionals are not nil.
}

Observe que todas as variáveis ​​devem ser desempacotadas para passar com sucesso no teste, caso contrário você não teria como determinar quais variáveis ​​foram desempacotadas e quais não foram.

Você pode encadear instruções condicionais usando seus opcionais imediatamente após o desempacotamento. Isso significa que não há instruções if - else aninhadas!

var firstName:String? = "Bob"
var myBool:Bool? = false

if let fn = firstName, fn == "Bob", let bool = myBool, !bool {
    print("firstName is bob and myBool was false!")
}

Operador de coalescência nulo #

Você pode usar o nil coalescing operator para desempacotar um valor se ele não for nulo, caso contrário, forneça um valor diferente:

func fallbackIfNil(str: String?) -> String {
    return str ?? "Fallback String"
}
print(fallbackIfNil("Hi")) // Prints "Hi"
print(fallbackIfNil(nil)) // Prints "Fallback String"

Este operador é capaz de curto-circuito, o que significa que se o operando esquerdo não for nulo, o operando direito não será avaliado:

func someExpensiveComputation() -> String { ... }

var foo : String? = "a string"
let str = foo ?? someExpensiveComputation()

Neste exemplo, como foo não é nulo, someExpensiveComputation() não será chamado.

Você também pode encadear várias instruções de união nula:

var foo : String?
var bar : String?

let baz = foo ?? bar ?? "fallback string"

Neste exemplo, baz receberá o valor desempacotado de foo se não for nulo, caso contrário, será atribuído o valor desempacotado de bar se não for nulo, caso contrário, será atribuído o valor de fallback .

Encadeamento opcional #

Você pode usar Optional Chaining para chamar um method, acessar uma propriedade ou subscript como opcional. Isso é feito colocando um ? entre a variável opcional fornecida e o membro fornecido (método, propriedade ou subscrito).

struct Foo {
    func doSomething() {
        print("Hello World!")
    }
}

var foo : Foo? = Foo()

foo?.doSomething() // prints "Hello World!" as foo is non-nil

Se foo contiver um valor, doSomething() será chamado nele. Se foo for nil, nada de ruim acontecerá - o código simplesmente falhará silenciosamente e continuará executando.

var foo : Foo? = nil

foo?.doSomething() // will not be called as foo is nil

(Este é um comportamento semelhante ao envio de mensagens para nil em Objective-C)

A razão pela qual o Optional Chaining é nomeado como tal é porque ‘opcionality’ será propagado através dos membros que você chama/acessa. O que isso significa é que os valores de retorno de quaisquer membros usados ​​com encadeamento opcional serão opcionais, independentemente de serem digitados como opcionais ou não.

struct Foo {
    var bar : Int
    func doSomething() { ... }
}

let foo : Foo? = Foo(bar: 5)
print(foo?.bar) // Optional(5)

Aqui foo?.bar está retornando um Int? mesmo que bar não seja opcional, já que foo em si é opcional.

À medida que a opcionalidade é propagada, os métodos que retornam Void retornarão Void? quando chamados com encadeamento opcional. Isso pode ser útil para determinar se o método foi chamado ou não (e, portanto, se o opcional tem um valor).

let foo : Foo? = Foo()

if foo?.doSomething() != nil {
    print("foo is non-nil, and doSomething() was called")
} else {
    print("foo is nil, therefore doSomething() wasn't called")
}

Aqui estamos comparando o valor de retorno Void? com nil para determinar se o método foi chamado (e, portanto, se foo é não-nil).

Visão geral - Por que opcionais? #

Muitas vezes ao programar é necessário fazer alguma distinção entre uma variável que tem um valor e uma que não tem. Para tipos de referência, como Ponteiros C, um valor especial como null pode ser usado para indicar que a variável não tem valor. Para tipos intrínsecos, como um inteiro, é mais difícil. Um valor indicado, como -1 pode ser usado, mas isso depende da interpretação do valor. Também elimina esse valor “especial” do uso normal.

Para resolver isso, o Swift permite que qualquer variável seja declarada como opcional. Isso é indicado pelo uso de um ? ou ! após o tipo (Veja Tipos de opcionais)

Por exemplo,

var possiblyInt: Int?

declara uma variável que pode ou não conter um valor inteiro.

O valor especial nil indica que nenhum valor está atualmente atribuído a esta variável.

possiblyInt = 5      // PossiblyInt is now 5
possiblyInt = nil    // PossiblyInt is now unassigned

nil também pode ser usado para testar um valor atribuído:

if possiblyInt != nil {
    print("possiblyInt has the value \(possiblyInt!)")
}

Observe o uso de ! na instrução print para unwrap o valor opcional.

Como exemplo de uso comum de opcionais, considere uma função que retorna um inteiro de uma string contendo dígitos; É possível que a string contenha caracteres que não sejam dígitos ou até mesmo esteja vazia.

Como pode uma função que retorna um simples Int indicar falha? Ele não pode fazer isso retornando qualquer valor específico, pois isso impediria que esse valor fosse analisado da string.

var someInt
someInt = parseInt("not an integer") // How would this function indicate failure?

No Swift, no entanto, essa função pode simplesmente retornar um Int opcional. Então a falha é indicada pelo valor de retorno de nil.

var someInt?
someInt = parseInt("not an integer")  // This function returns nil if parsing fails
if someInt == nil {
    print("That isn't a valid integer")
}
