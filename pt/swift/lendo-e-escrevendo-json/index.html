Sintaxe #

  • NSJSONSerialization.JSONObjectWithData(jsonData, options: NSJSONReadingOptions) // Retorna um objeto de jsonData. Este método lança em caso de falha.
  • NSJSONSerialization.dataWithJSONObject(jsonObject, options: NSJSONWritingOptions) // Retorna NSData de um objeto JSON. Passe NSJSONWritingOptions.PrettyPrinted nas opções para uma saída mais legível.

Serialização, codificação e decodificação JSON com a Apple Foundation e a Swift Standard Library #

A classe JSONSerialization é incorporada à estrutura Foundation da Apple.

Leia JSON #

A função JSONObjectWithData recebe NSData e retorna AnyObject. Você pode usar as? para converter o resultado para o tipo esperado.

do {
    guard let jsonData = "[\"Hello\", \"JSON\"]".dataUsingEncoding(NSUTF8StringEncoding) else {
        fatalError("couldn't encode string as UTF-8")
    }

    // Convert JSON from NSData to AnyObject
    let jsonObject = try NSJSONSerialization.JSONObjectWithData(jsonData, options: [])
    
    // Try to convert AnyObject to array of strings
    if let stringArray = jsonObject as? [String] {
        print("Got array of strings: \(stringArray.joinWithSeparator(", "))")
    }
} catch {
    print("error reading JSON: \(error)")
}

Você pode passar options: .AllowFragments em vez de options: [] para permitir a leitura de JSON quando o objeto de nível superior não for uma matriz ou dicionário.

Escrever JSON #

Chamar dataWithJSONObject converte um objeto compatível com JSON (matrizes ou dicionários aninhados com strings, números e NSNull) em NSData bruto codificado como UTF-8.

do {
    // Convert object to JSON as NSData
    let jsonData = try NSJSONSerialization.dataWithJSONObject(jsonObject, options: [])
    print("JSON data: \(jsonData)")
    
    // Convert NSData to String
    let jsonString = String(data: jsonData, encoding: NSUTF8StringEncoding)!
    print("JSON string: \(jsonString)")
} catch {
    print("error writing JSON: \(error)")
}

Você pode passar options: .PrettyPrinted em vez de options: [] para impressão bonita.

Mesmo comportamento no Swift 3, mas com uma sintaxe diferente.

do {
    guard let jsonData = "[\"Hello\", \"JSON\"]".data(using: String.Encoding.utf8) else {
        fatalError("couldn't encode string as UTF-8")
    }
    
    // Convert JSON from NSData to AnyObject
    let jsonObject = try JSONSerialization.jsonObject(with: jsonData, options: [])
    
    // Try to convert AnyObject to array of strings
    if let stringArray = jsonObject as? [String] {
        print("Got array of strings: \(stringArray.joined(separator: ", "))")
    }
} catch {
    print("error reading JSON: \(error)")
}

do {
    // Convert object to JSON as NSData
    let jsonData = try JSONSerialization.data(withJSONObject: jsonObject, options: [])
    print("JSON data: \(jsonData)")

    // Convert NSData to String
    let jsonString = String(data: jsonData, encoding: .utf8)!
    print("JSON string: \(jsonString)")
} catch {
    print("error writing JSON: \(error)")
}

Nota: No momento, o seguinte está disponível apenas no Swift 4.0 e posterior.

A partir do Swift 4.0, a biblioteca padrão do Swift inclui os protocolos Encodable e [Decodable](https://developer.apple.com /documentation/swift/decodable) para definir uma abordagem padronizada para codificação e decodificação de dados. A adoção desses protocolos permitirá implementações do Encoder e do [Decoder](https://developer.apple.com/documentation/swift /decoder) pegam seus dados e os codificam ou decodificam de e para uma representação externa, como JSON. A conformidade com o protocolo Codable combina os protocolos Codable e Decodable. Este é agora o meio recomendado para lidar com JSON em seu programa.

Codifique e decodifique automaticamente #

A maneira mais fácil de tornar um tipo codificável é declarar suas propriedades como tipos que já são Codable. Esses tipos incluem tipos de biblioteca padrão, como String, Int e Double; e tipos de Fundação como Date, Data e URL. Se as propriedades de um tipo forem codificáveis, o próprio tipo será automaticamente compatível com Codable simplesmente declarando a conformidade.

Considere o exemplo a seguir, no qual a estrutura Book está em conformidade com Codable.

struct Book: Codable {
    let title: String
    let authors: [String]
    let publicationDate: Date
}

Observe que coleções padrão como Array e Dictionary estão em conformidade com Codable se contiverem tipos codificáveis.

Ao adotar Codable, a estrutura Book agora pode ser codificada e decodificada de JSON usando as classes JSONDecoder e JSONDecoder da Apple Foundation, mesmo que Book em si não contenha código para lidar especificamente com JSON. Codificadores e decodificadores personalizados também podem ser escritos em conformidade com os protocolos Encoder e Decoder, respectivamente.

Codifique para dados JSON #

// Create an instance of Book called book
let encoder = JSONEncoder()
let data = try! encoder.encode(book) // Do not use try! in production code
print(data)

Configure encoder.outputFormatting = .prettyPrinted para facilitar a leitura. ## Decodificar a partir de dados JSON

Decodificar a partir de dados JSON #

// Retrieve JSON string from some source
let jsonData = jsonString.data(encoding: .utf8)!
let decoder = JSONDecoder()
let book = try! decoder.decode(Book.self, for: jsonData) // Do not use try! in production code
print(book)

No exemplo acima, Book.self informa ao decodificador o tipo para o qual o JSON deve ser decodificado.

Codificando ou Decodificando Exclusivamente #

Às vezes, você pode não precisar que os dados sejam codificáveis ​​e decodificáveis, como quando você precisa apenas ler dados JSON de uma API ou se seu programa envia apenas dados JSON para uma API.

Se você pretende apenas gravar dados JSON, ajuste seu tipo para Codificável.

struct Book: Encodable {
    let title: String
    let authors: [String]
    let publicationDate: Date
}

Se você pretende apenas ler dados JSON, ajuste seu tipo para Decodificável.

struct Book: Decodable {
    let title: String
    let authors: [String]
    let publicationDate: Date
}

Usando nomes de chave personalizados #

As APIs frequentemente usam convenções de nomenclatura diferentes do caso camel padrão Swift, como o caso snake. Isso pode se tornar um problema quando se trata de decodificar JSON, pois, por padrão, as chaves JSON devem se alinhar exatamente com os nomes de propriedade do seu tipo. Para lidar com esses cenários, você pode criar chaves personalizadas para seu tipo usando o protocolo CodingKey.

struct Book: Codable {
    // ...
    enum CodingKeys: String, CodingKey { 
        case title
        case authors
        case publicationDate = "publication_date"
    }
}

CodingKeys são gerados automaticamente para tipos que adotam o protocolo Codable, mas criando nossa própria implementação no exemplo acima, permitimos que nosso decodificador combine o case camel local publicationDate com o snake case publication_date como é entregue pela API.

SwiftyJSON #

SwiftyJSON é uma estrutura Swift criada para eliminar a necessidade de encadeamento opcional na serialização JSON normal.

Você pode baixá-lo aqui: https://github.com/SwiftyJSON/SwiftyJSON

Sem SwiftyJSON, seu código ficaria assim para encontrar o nome do primeiro livro em um objeto JSON:

if let jsonObject = try NSJSONSerialization.JSONObjectWithData(data, options: .AllowFragments) as? [[String: AnyObject]],
let bookName = (jsonObject[0]["book"] as? [String: AnyObject])?["name"] as? String {
    //We can now use the book name
}

No SwiftyJSON, isso é extremamente simplificado:

let json = JSON(data: data)
if let bookName = json[0]["book"]["name"].string {
    //We can now use the book name
}

Ele elimina a necessidade de verificar todos os campos, pois retornará nil se algum deles for inválido.

Para usar o SwiftyJSON, baixe a versão correta do repositório Git - existe uma ramificação para o Swift 3. Basta arrastar o “SwiftyJSON.swift” para o seu projeto e importar para a sua classe:

import SwiftyJSON

Você pode criar seu objeto JSON usando os dois inicializadores a seguir:

let jsonObject = JSON(data: dataObject)

ou

let jsonObject = JSON(jsonObject) //This could be a string in a JSON format for example

Para acessar seus dados, use subscritos:

let firstObjectInAnArray = jsonObject[0]
let nameOfFirstObject = jsonObject[0]["name"]

Você pode então analisar seu valor para um determinado tipo de dados, que retornará um valor opcional:

let nameOfFirstObject = jsonObject[0]["name"].string //This will return the name as a string

let nameOfFirstObject = jsonObject[0]["name"].double //This will return null

Você também pode compilar seus caminhos em um Array rápido:

let convolutedPath = jsonObject[0]["name"][2]["lastName"]["firstLetter"].string

É o mesmo que:

let convolutedPath = jsonObject[0, "name", 2, "lastName", "firstLetter"].string

SwiftyJSON também tem funcionalidade para imprimir seus próprios erros:

if let name = json[1337].string {
    //You can use the value - it is valid
} else {
    print(json[1337].error) // "Array[1337] is out of bounds" - You cant use the value
}

Se você precisar gravar em seu objeto JSON, poderá usar subscritos novamente:

var originalJSON:JSON = ["name": "Jack", "age": 18]
originalJSON["age"] = 25 //This changes the age to 25
originalJSON["surname"] = "Smith" //This creates a new field called "surname" and adds the value to it

Se você precisar da String original para o JSON, por exemplo, se precisar gravá-la em um arquivo, poderá obter o valor bruto:

if let string = json.rawString() { //This is a String object
    //Write the string to a file if you like
}

if let data = json.rawData() { //This is an NSData object
    //Send the data to your server if you like
}

Freddy #

Freddy é uma biblioteca de análise JSON mantida por Big Nerd Ranch. Possui três benefícios principais:

  1. Segurança de tipo: Ajuda você a trabalhar com envio e recebimento de JSON de uma forma que evita falhas de tempo de execução.

  2. Idiomático: Aproveita os genéricos, enumerações e recursos funcionais do Swift, sem documentação complicada ou operadores mágicos personalizados.

  3. Tratamento de erros: fornece informações de erro informativas para erros JSON que ocorrem com frequência.

Exemplo de dados JSON #

Vamos definir alguns dados JSON de exemplo para uso com esses exemplos.

{
  "success": true,
  "people": [
    {
      "name": "Matt Mathias",
      "age": 32,
      "spouse": true
    },
    {
      "name": "Sergeant Pepper",
      "age": 25,
      "spouse": false
    }
  ],
  "jobs": [
    "teacher",
    "judge"
  ],
  "states": {
    "Georgia": [
      30301,
      30302,
      30303
    ],
    "Wisconsin": [
      53000,
      53001
    ]
  }
}

let jsonString = "{\"success\": true, \"people\": [{\"name\": \"Matt Mathias\",\"age\": 32,\"spouse\": true},{\"name\": \"Sergeant Pepper\",\"age\": 25,\"spouse\": false}],\"jobs\": [\"teacher\",\"judge\"],\"states\": {\"Georgia\": [30301,30302,30303],\"Wisconsin\": [53000,53001]}}"
let jsonData = jsonString.dataUsingEncoding(NSUTF8StringEncoding)!

Desserializando dados brutos #

Para desserializar os dados, inicializamos um objeto JSON e acessamos uma chave específica.

do {
    let json = try JSON(data: jsonData)
    let success = try json.bool("success")
} catch {
    // do something with the error
}

Nós tentamos aqui porque acessar o json para a chave "sucesso" pode falhar – ela pode não existir, ou o valor pode não ser um booleano.

Também podemos especificar um caminho para acessar elementos aninhados na estrutura JSON. O caminho é uma lista separada por vírgulas de chaves e índices que descrevem o caminho para um valor de interesse.

do {
    let json = try JSON(data: jsonData)
    let georgiaZipCodes = try json.array("states", "Georgia")
    let firstPersonName = try json.string("people", 0, "name")
} catch {
    // do something with the error
}

Desserializando modelos diretamente #

O JSON pode ser analisado diretamente em uma classe de modelo que implementa o protocolo JSONDecodable.

public struct Person {
    public let name: String
    public let age: Int
    public let spouse: Bool
}

extension Person: JSONDecodable {
    public init(json: JSON) throws {
        name = try json.string("name")
        age = try json.int("age")
        spouse = try json.bool("spouse")
    }
}

do {
    let json = try JSON(data: jsonData)
    let people = try json.arrayOf("people", type: Person.self)
} catch {
    // do something with the error
}

Serializando dados brutos #

Qualquer valor JSON pode ser serializado diretamente para NSData.

let success = JSON.Bool(false)
let data: NSData = try success.serialize()

Serializando modelos diretamente #

Qualquer classe de modelo que implemente o protocolo JSONEncodable pode ser serializada diretamente para NSData.

extension Person: JSONEncodable {
    public func toJSON() -> JSON {
        return .Dictionary([
            "name": .String(name),
            "age": .Int(age),
            "spouse": .Bool(spouse)
        ])
    }
}

let newPerson = Person(name: "Glenn", age: 23, spouse: true)
let data: NSData = try newPerson.toJSON().serialize()

Análise simples de JSON em objetos personalizados #

Mesmo que as bibliotecas de terceiros sejam boas, uma maneira simples de analisar o JSON é fornecida por protocolos Você pode imaginar que tem um objeto Todo como

struct Todo {
    let comment: String
}

Sempre que você receber o JSON, você pode manipular o NSData simples como mostrado no outro exemplo usando o objeto NSJSONSerialization.

Depois disso, usando um protocolo simples JSONDecodable

typealias JSONDictionary = [String:AnyObject]
protocol JSONDecodable {
    associatedtype Element
    static func from(json json: JSONDictionary) -> Element?
}

E fazer sua estrutura Todo em conformidade com JSONDecodable faz o truque

extension Todo: JSONDecodable {
    static func from(json json: JSONDictionary) -> Todo? {
        guard let comment = json["comment"] as? String else { return nil }
        return Todo(comment: comment)
    }
}

Você pode tentar com este código json:

{
    "todos": [
        {
            "comment" : "The todo comment"
        }
    ]
}

Ao obtê-lo da API, você pode serializá-lo como os exemplos anteriores mostrados em uma instância AnyObject. Depois disso, você pode verificar se a instância é uma instância JSONDictionary

guard let jsonDictionary = dictionary as? JSONDictionary else { return }

A outra coisa a verificar, específica para este caso porque você tem um array de Todo no JSON, é o dicionário todos

guard let todosDictionary = jsonDictionary["todos"] as? [JSONDictionary] else { return }

Agora que você tem o array de dicionários, você pode converter cada um deles em um objeto Todo usando flatMap (ele excluirá automaticamente os valores nil do array)

let todos: [Todo] = todosDictionary.flatMap { Todo.from(json: $0) }

JSON Analisando Swift 3 #

Flecha #

Arrow é uma elegante biblioteca de análise JSON em Swift.

Ele permite analisar JSON e mapeá-lo para classes de modelo personalizadas com a ajuda de um operador <--:

identifier <-- json["id"]
name <-- json["name"]
stats <-- json["stats"]

Exemplo:

Modelo rápido

struct Profile {
    var identifier = 0
    var name = ""
    var link: NSURL?
    var weekday: WeekDay = .Monday
    var stats = Stats()
    var phoneNumbers = [PhoneNumber]()
}

arquivo JSON

{
    "id": 15678,
    "name": "John Doe",
    "link": "https://apple.com/steve",
    "weekdayInt" : 3,
    "stats": {
        "numberOfFriends": 163,
        "numberOfFans": 10987
    },
    "phoneNumbers": [{
                     "label": "house",
                     "number": "9809876545"
                     }, {
                     "label": "cell",
                     "number": "0908070656"
                     }, {
                     "label": "work",
                     "number": "0916570656"
    }]
}

Mapeamento

extension Profile: ArrowParsable {
    mutating func deserialize(json: JSON) {
        identifier <-- json["id"]
        link <-- json["link"]
        name <-- json["name"]
        weekday <-- json["weekdayInt"]
        stats <- json["stats"]
        phoneNumbers <-- json["phoneNumbers"]
    }
}

Uso

let profile = Profile()
profile.deserialize(json)

github "s4cha/Arrow"

pod 'Arrow'
use_frameworks!

Baixe o Arrow do repositório GitHub e compile o destino do Framework no projeto de exemplo. Em seguida, vincule-se a esta estrutura.
