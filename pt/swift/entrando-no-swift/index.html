jogar fora #

dump imprime o conteúdo de um objeto via reflexão (espelhamento).

Visão detalhada de um array:

let names = ["Joe", "Jane", "Jim", "Joyce"]
dump(names)

Impressões:

▿ 4 elementos - [0]: João - [1]: Jane - [2]: Jim - [3]: Joyce

Para um dicionário:

let attributes = ["foo": 10, "bar": 33, "baz": 42]
dump(attributes)

Impressões:

▿ 3 pares chave/valor ▿ [0]: (2 elementos)

  • .0: barra
  • .1: 33 ▿ [1]: (2 elementos)
  • .0: baz
  • .1: 42 ▿ [2]: (2 elementos)
  • .0: foo
  • .1: 10

dump é declarado como dump(_:name:indent:maxDepth:maxItems:).

O primeiro parâmetro não tem rótulo.

Existem outros parâmetros disponíveis, como name para definir um rótulo para o objeto que está sendo inspecionado:

dump(attributes, name: "mirroring")

Impressões:

▿ espelhamento: 3 pares chave/valor ▿ [0]: (2 elementos)

  • .0: barra
  • .1: 33 ▿ [1]: (2 elementos)
  • .0: baz
  • .1: 42 ▿ [2]: (2 elementos)
  • .0: foo
  • .1: 10

Você também pode optar por imprimir apenas um certo número de itens com maxItems:, analisar o objeto até uma certa profundidade com maxDepth: e alterar o recuo dos objetos impressos com indent:.

Impressão de depuração #

Debug Print mostra a representação de instância mais adequada para depuração.

print("Hello")
debugPrint("Hello")

let dict = ["foo": 1, "bar": 2]

print(dict)
debugPrint(dict)

Rendimentos

>>> Hello
>>> "Hello"
>>> [foo: 1, bar: 2]
>>> ["foo": 1, "bar": 2]

Essas informações extras podem ser muito importantes, por exemplo:

let wordArray = ["foo", "bar", "food, bars"]

print(wordArray)
debugPrint(wordArray)

Rendimentos

>>> [foo, bar, food, bars]
>>> ["foo", "bar", "food, bars"]

Observe como na primeira saída parece que existem 4 elementos no array ao invés de 3. Por razões como esta, é preferível ao depurar usar debugPrint

Atualizando os valores de depuração e impressão de uma classe #

struct Foo: Printable, DebugPrintable {
    var description: String {return "Clear description of the object"}
    var debugDescription: String {return "Helpful message for debugging"}
}

var foo = Foo()

print(foo)
debugPrint(foo)

>>> Clear description of the object
>>> Helpful message for debugging

print() vs dump() #

Muitos de nós começamos a depurar com um simples print(). Digamos que temos uma classe assim:

class Abc {
    let a = "aa"
    let b = "bb"
}

e temos uma instância de Abc assim:

let abc = Abc()

Quando executamos o print() na variável, a saída é

App.Abc

enquanto dump() produz

App.Abc #0
- a: "aa"
- b: "bb"

Como visto, dump() gera toda a hierarquia de classes, enquanto print() simplesmente gera o nome da classe.

Portanto, dump() é especialmente útil para depuração da interface do usuário

let view = UIView(frame: CGRect(x: 0, y: 0, width: 100, height: 100))

Com dump(view) temos:

 - <UIView: 0x108a0cde0; frame = (0 0; 100 100); layer = <CALayer: 0x159340cb0>> #0
    - super: UIResponder
      - NSObject

Enquanto print(view) obtemos:

<UIView: 0x108a0cde0; frame = (0 0; 100 100); layer = <CALayer: 0x159340cb0>>

Há mais informações sobre a classe com dump() e, portanto, é mais útil na depuração da própria classe.

imprimir vs NSLog #

No swift podemos usar as funções print() e NSLog() para imprimir algo no console Xcode.

Mas há muitas diferenças nas funções print() e NSLog(), como:

1 TimeStamp: NSLog() imprimirá o timestamp junto com a string que passamos para ele, mas print() não imprimirá o timestamp. por exemplo.

let array = [1, 2, 3, 4, 5]
print(array)
NSLog(array.description)

Resultado:

[1, 2, 3, 4, 5] 31/05/2017 13:14:38.582 Nome do projeto[2286:7473287] [1, 2, 3, 4, 5]

Ele também imprimirá ProjectName junto com o carimbo de data/hora.

2 Only String: NSLog() só aceita String como entrada, mas print() pode imprimir qualquer tipo de entrada passada para ele. por exemplo.

let array = [1, 2, 3, 4, 5]
print(array) //prints [1, 2, 3, 4, 5]
NSLog(array) //error: Cannot convert value of type [Int] to expected argument type 'String'

3 Desempenho: a função NSLog() é muito lenta em comparação com a função print().

4 Sincronização: NSLog() lida com o uso simultâneo do ambiente multi-threading e imprime a saída sem sobrepô-la. Mas print() não irá lidar com tais casos e confusão enquanto tagarela a saída.

5 Device Console: NSLog() também sai no console do dispositivo, podemos ver essa saída conectando nosso dispositivo ao Xcode. print() não imprimirá a saída no console do dispositivo.
