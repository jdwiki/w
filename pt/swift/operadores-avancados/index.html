Operadores bit a bit #

Os operadores Swift Bitwise permitem que você execute operações na forma binária de números. Você pode especificar um literal binário prefixando o número com 0b, então, por exemplo, 0b110 é equivalente ao número binário 110 (o número decimal 6). Cada 1 ou 0 é um bit no número.

Bit a bit NÃO ~:

var number: UInt8 = 0b01101100
let newNumber = ~number
// newNumber is equal to 0b01101100

Aqui, cada bit é alterado para o seu oposto. Declarar o número explicitamente como UInt8 garante que o número seja positivo (para que não tenhamos que lidar com negativos no exemplo) e que seja apenas 8 bits. Se 0b01101100 fosse uma UInt maior, haveria 0s iniciais que seriam convertidos em 1s e se tornariam significativos na inversão:

var number: UInt16 = 0b01101100
// number equals 0b0000000001101100
// the 0s are not significant
let newNumber = ~number
// newNumber equals 0b1111111110010011
// the 1s are now significant
  • 0 -> 1
  • 1 -> 0

Bit a bit AND &:

var number = 0b0110
let newNumber = number & 0b1010
// newNumber is equal to 0b0010

Aqui, um determinado bit será 1 se e somente se os números binários em ambos os lados do operador & contiverem um 1 nesse local de bit.

  • 0 e 0 -> 0
  • 0 e 1 -> 0
  • 1 e 1 -> 1

OU bit a bit |:

var number = 0b0110
let newNumber = number | 0b1000
// newNumber is equal to 0b1110

Aqui, um determinado bit será 1 se e somente se o número binário em pelo menos um lado do operador | contiver um 1 nesse local de bit.

  • 0 | 0 -> 0
  • 0 | 1 -> 1
  • 1 | 1 -> 1

XOR bit a bit (OU exclusivo) ^:

var number = 0b0110
let newNumber = number ^ 0b1010
// newNumber is equal to 0b1100

Aqui, um determinado bit será 1 se e somente se os bits nessa posição dos dois operandos forem diferentes.

  • 0 ^ 0 -> 0
  • 0 ^ 1 -> 1
  • 1 ^ 1 -> 0

Para todas as operações binárias, a ordem dos operandos não faz diferença no resultado.

Operadores personalizados #

O Swift suporta a criação de operadores personalizados. Novos operadores são declarados em nível global usando a palavra-chave operator.

A estrutura do operador é definida por três partes: posicionamento do operando, precedência e associatividade.

  1. Os modificadores prefix, infix e postfix são usados ​​para iniciar uma declaração de operador personalizada. Os modificadores prefix e postfix declaram se o operador deve estar antes ou depois, respectivamente, do valor sobre o qual ele atua. Tais operadores são urnários, como 8 e 3++ **, pois só podem atuar em um alvo. O infix declara um operador binário, que atua nos dois valores entre os quais está, como 2+3.

  2. Os operadores com maior precedência são calculados primeiro. A precedência do operador padrão é apenas superior a ?…: (um valor de 100 no Swift 2.x). A precedência dos operadores padrão do Swift pode ser encontrada aqui.

  3. Associatividade define a ordem das operações entre operadores de mesma precedência. Os operadores associativos à esquerda são calculados da esquerda para a direita (ordem de leitura, como a maioria dos operadores), enquanto os operadores associativos à direita calculam da direita para a esquerda.

A partir do Swift 3.0, pode-se definir a precedência e a associatividade em um grupo de precedência em vez do próprio operador, para que vários operadores possam compartilhar facilmente a mesma precedência sem se referir aos números crípticos. A lista de grupos de precedência padrão é mostrada abaixo.

Os operadores retornam valores com base no código de cálculo. Esse código age como uma função normal, com parâmetros especificando o tipo de entrada e a palavra-chave return especificando o valor calculado que o operador retorna.

Aqui está a definição de um operador exponencial simples, já que o Swift padrão não possui um operador exponencial.

import Foundation    

infix operator ** { associativity left precedence 170 }

func ** (num: Double, power: Double) -> Double{
    return pow(num, power)
}

O infix diz que o operador ** funciona entre dois valores, como 9**2. Como a função deixou a associatividade, 3**3**2 é calculado como (3**3)**2. A precedência de 170 é maior do que todas as operações padrão do Swift, significando que 3+2**4 calcula para 19, apesar da associatividade à esquerda de **.

import Foundation 

infix operator **: BitwiseShiftPrecedence

func ** (num: Double, power: Double) -> Double {
    return pow(num, power)
}

Em vez de especificar a precedência e a associatividade explicitamente, no Swift 3.0 poderíamos usar o grupo de precedência integrado BitwiseShiftPrecedence que fornece os valores corretos (o mesmo que <<, >>).

**: O incremento e o decremento estão obsoletos e serão removidos no Swift 3.

2: https://www.wikiod.com/pt/swift/operadores-avancados#Precedência de operadores Swift padrão

Operadores Comutativos #

Vamos adicionar um operador personalizado para multiplicar um CGSize

func *(lhs: CGFloat, rhs: CGSize) -> CGSize{
    let height = lhs*rhs.height
    let width = lhs*rhs.width
    return CGSize(width: width, height: height)
}

Agora isso funciona

let sizeA = CGSize(height:100, width:200)    
let sizeB = 1.1 * sizeA         //=> (height: 110, width: 220)

Mas se tentarmos fazer a operação inversa, obtemos um erro

let sizeC = sizeB * 20          // ERROR

Mas é simples o suficiente para adicionar:

func *(lhs: CGSize, rhs: CGFloat) -> CGSize{
    return rhs*lhs
}

Agora o operador é comutativo.

let sizeA = CGSize(height:100, width:200)    
let sizeB = sizeA * 1.1              //=> (height: 110, width: 220)

Operadores de estouro #

Estouro refere-se ao que acontece quando uma operação resultaria em um número maior ou menor do que a quantidade designada de bits para esse número pode conter.

Devido à maneira como a aritmética binária funciona, depois que um número se torna muito grande para seus bits, o número transborda para o menor número possível (para o tamanho do bit) e continua a contagem a partir daí. Da mesma forma, quando um número se torna muito pequeno, ele passa para o maior número possível (para seu tamanho de bit) e continua a contagem regressiva a partir daí.

Como esse comportamento geralmente não é desejado e pode levar a sérios problemas de segurança, os operadores aritméticos do Swift +, - e * lançarão erros quando uma operação causar um overflow ou underflow. Para permitir explicitamente overflow e underflow, use &+, &- e &*.

var almostTooLarge = Int.max
almostTooLarge + 1 // not allowed
almostTooLarge &+ 1 // allowed, but result will be the value of Int.min

Sobrecarga + para dicionários #

Como atualmente não existe uma maneira simples de combinar dicionários em Swift, pode ser útil sobrecarregar os operadores + e += para adicionar esta funcionalidade usando genéricos.

// Combines two dictionaries together. If both dictionaries contain
// the same key, the value of the right hand side dictionary is used.
func +<K, V>(lhs: [K : V], rhs: [K : V]) -> [K : V] {
    var combined = lhs
    for (key, value) in rhs {
        combined[key] = value
    }
    return combined
}

// The mutable variant of the + overload, allowing a dictionary
// to be appended to 'in-place'.
func +=<K, V>(inout lhs: [K : V], rhs: [K : V]) {
    for (key, value) in rhs {
        lhs[key] = value
    }
}

A partir do Swift 3, inout deve ser colocado antes do tipo de argumento.

func +=<K, V>(lhs: inout [K : V], rhs: [K : V]) { ... }

Exemplo de uso:

let firstDict = ["hello" : "world"]
let secondDict = ["world" : "hello"]
var thirdDict = firstDict + secondDict // ["hello": "world", "world": "hello"]

thirdDict += ["hello":"bar", "baz":"qux"] // ["hello": "bar", "baz": "qux", "world": "hello"]

Precedência de operadores Swift padrão #

Os operadores que limitam mais fortemente (precedência mais alta) são listados primeiro.

| Operadores | Grupo de precedência (≥3.0) | Precedência | Associatividade | |————|——————|—————– ———|—————| | . | | ∞ | deixei | ?, !, ++, --, [], (), {} | (pós-fixo) | | | | !, ~, +, -, ++, -- | (prefixo) | | | | ~> (swift ≤2.3) | | 255 | deixei | <<, >> | BitwiseShiftPrecedence | 160 | Nenhum | *, /, %, &, &* | MultiplicaçãoPrecedência | 150 | deixei | +, -, |, ^, &+, &- | AdiçãoPrecedência | 140 | deixei | ..., ..< | RangeFormationPrecedence | 135 | Nenhum | é, como, como?, como! | FundiçãoPrecedência | 132 | deixei | ?? | NilCoalescênciaPrecedência | 131 | certo | <, <=, >, >=, ==, !=, ===, !==, ~= | ComparaçãoPrecedência | 130 | nenhum | | && | LogicalConjunctionPrecedence | 120 | deixei | || | Disjunção LógicaPrecedência | 110 | deixei | | DefaultPrecedence* | | Nenhum | ?…: | Precedência Ternária | 100 | certo | =, +=, -=, *=, /=, %=, <<=, >>=, &=, \ |=, ^= | AtribuiçãoPrecedência | 90 | direito, atribuição | -> | FunçãoSetaPrecedência | | certo

  • O grupo de precedência DefaultPrecedence é maior que TernaryPrecedence, mas não é ordenado com o resto dos operadores. Além deste grupo, o resto das precedências são lineares.
