Use um mapa de módulo para importar cabeçalhos C #

Um module map pode simplesmente importar mymodule configurando-o para ler arquivos de cabeçalho C e fazê-los aparecer como Swift funções.

Coloque um arquivo chamado module.modulemap dentro de um diretório chamado mymodule:

Dentro do arquivo de mapa do módulo:

// mymodule/module.modulemap
module mymodule {
    header "defs.h"
}

Em seguida, importe o módulo:

// demo.swift
import mymodule
print("Empty color: \(Color())")

Use o sinalizador -I diretório para informar ao swiftc onde encontrar o módulo:

swiftc -I . demo.swift   # "-I ." means "search for modules in the current directory"

Para obter mais informações sobre a sintaxe do mapa de módulo, consulte a documentação do Clang sobre mapas de módulo.

Usando classes Objective-C do código Swift #

Se MyFramework contém classes Objective-C em seus cabeçalhos públicos (e o cabeçalho guarda-chuva), então import MyFramework é tudo o que é necessário para usá-los do Swift.

Cabeçalhos de ponte #

Um cabeçalho de ponte torna as declarações Objective-C e C adicionais visíveis para o código Swift. Ao adicionar arquivos de projeto, o Xcode pode oferecer a criação de um cabeçalho de ponte automaticamente:

Para criar um manualmente, modifique a configuração de compilação Objective-C Bridging Header:

Dentro do cabeçalho de ponte, importe os arquivos necessários para usar do código:

// MyApp-Bridging-Header.h
#import "MyClass.h"  // allows code in this module to use MyClass

Interface gerada #

Clique no botão Itens Relacionados (ou pressione ⌃1) e selecione Interface Gerada para ver a interface Swift que será gerada a partir de um cabeçalho Objective-C.

    

Especifique um cabeçalho de ponte para swiftc #

O sinalizador -import-objc-header especifica um cabeçalho para swiftc importar:

// defs.h
struct Color {
    int red, green, blue;
};

#define MAX_VALUE 255

// demo.swift
extension Color: CustomStringConvertible {  // extension on a C struct
    public var description: String {
        return "Color(red: \(red), green: \(green), blue: \(blue))"
    }
}
print("MAX_VALUE is: \(MAX_VALUE)")  // C macro becomes a constant
let color = Color(red: 0xCA, green: 0xCA, blue: 0xD0)  // C struct initializer
print("The color is \(color)")

$ swiftc demo.swift -import-objc-header defs.h && ./demo
MAX_VALUE é: 255
A cor é Color(vermelho: 202, verde: 202, azul: 208)

Interoperação refinada entre Objective-C e Swift #

Quando uma API é marcada com NS_REFINED_FOR_SWIFT, ela será prefixada com dois sublinhados (__) quando importada para o Swift:

@interface MyClass : NSObject
- (NSInteger)indexOfObject:(id)obj NS_REFINED_FOR_SWIFT;
@end

A [interface gerada](https://www.wikiod.com/pt/swift/trabalhando-com-c-e-objective-c#Usando classes Objective-C do código Swift 201605140416386495969) fica assim:

classe pública MyClass : NSObject {
    public func __indexOfObject(obj: AnyObject) -> Int
}

Agora você pode substituir a API por uma extensão mais “Swifty”. Nesse caso, podemos usar um valor de retorno opcional, filtrando [NSNotFound](https://developer.apple.com /library/mac/documentation/Cocoa/Reference/Foundation/Miscellaneous/Foundation_Constants/#//apple_ref/doc/constant_group/NSNotFound):

extensão MyClass {
    // Rather than returning NSNotFound if the object doesn't exist,
    // this "refined" API returns nil.
    func indexOfObject(obj: AnyObject) -> Int? {
        let idx = __indexOfObject(obj)
        if idx == NSNotFound { return nil }
        return idx
    }
}

// Código Swift, usando "if let" como deveria ser:
deixe meuobj = MinhaClasse()
if let idx = myobj.indexOfObject(algo) {
    // do something with idx
}

Na maioria dos casos, você pode querer restringir se um argumento para uma função Objective-C pode ser nil. Isso é feito usando a palavra-chave _Nonnull, que qualifica qualquer ponteiro ou referência de bloco:

void
doStuff(const void *const _Nonnull data, void (^_Nonnull complete)())
{
    // complex asynchronous code
}

Com isso escrito, o compilador emitirá um erro sempre que tentarmos passar nil para essa função do nosso código Swift:

doCoisas(
    nil,  // error: nil is not compatible with expected argument type 'UnsafeRawPointer'
    nil)  // error: nil is not compatible with expected argument type '() -> Void'

O oposto de _Nonnull é _Nullable, o que significa que é aceitável passar nil neste argumento. _Nullable também é o padrão; no entanto, especificá-lo explicitamente permite um código mais autodocumentado e à prova de futuro.

Para ajudar ainda mais o compilador a otimizar seu código, você também pode especificar se o bloco está escapando:

void
callNow(__attribute__((noescape)) void (^_Nonnull f)())
{
    // f is not stored anywhere
}

Com este atributo prometemos não salvar a referência do bloco e não chamar o bloco após a execução da função.

Use a biblioteca padrão C #

A interoperabilidade C do Swift permite que você use funções e tipos da biblioteca padrão C.

No Linux, a biblioteca padrão C é exposta através do módulo Glibc; nas plataformas da Apple é chamado de Darwin.

#if os(macOS) || os(iOS) || os(tvOS) || os(watchOS)
import Darwin
#elseif os(Linux)
import Glibc
#endif

// use open(), read(), and other libc features

Usando classes Swift do código Objective-C #

No mesmo módulo #

Dentro de um módulo chamado “MyModule”, o Xcode gera um cabeçalho chamado MyModule-Swift.h que expõe classes públicas do Swift para Objective-C. Importe este cabeçalho para usar as classes Swift:

// MySwiftClass.swift em MyApp
Importar Fundação

// A classe deve ser `public` para ser visível, a menos que este destino também tenha um cabeçalho de ponte
público class MySwiftClass: NSObject {
    // ...
}
// MyViewController.m em MyApp

#import "MyViewController.h"
#import "MyApp-Swift.h" // importa a interface gerada
#import <MyFramework/MyFramework-Swift.h> // ou use colchetes angulares para um destino de estrutura

@implementation MyViewController
- (void)demonstração {
    [[MySwiftClass alloc] init];  // use the Swift class
}
@end

Configurações de compilação relevantes:

  • Nome do cabeçalho da interface gerada pelo objetivo-C: controla o nome do cabeçalho Obj-C gerado.
  • Install Objective-C Compatibility Header: se o cabeçalho -Swift.h deve ser um cabeçalho público (para alvos de framework).

Em outro módulo #

Usando @import MyFramework; importa todo o módulo, incluindo interfaces Obj-C para classes Swift (se a configuração de compilação mencionada acima estiver habilitada).
