Parâmetros #

Parâmetro Detalhes
Valor a testar A variável com a qual comparar

Forneça um caso para cada valor possível de sua entrada. Use um ‘caso padrão’ para cobrir os valores de entrada restantes que você não deseja especificar. O caso padrão deve ser o último caso.

Por padrão, os Switches no Swift não continuarão verificando outros casos após a correspondência de um caso.

Interruptor e opcionais #

Alguns exemplos de casos em que o resultado é opcional.

var result: AnyObject? = someMethod()

switch result {
case nil:
    print("result is nothing")
case is String:
    print("result is a String")
case _ as Double:
    print("result is not nil, any value that is a Double")
case let myInt as Int where myInt > 0:
    print("\(myInt) value is not nil but an int and greater than 0")
case let a?:
    print("\(a) - value is unwrapped")
}

Uso Básico #

let number = 3
switch number {
case 1:
    print("One!")
case 2:
    print("Two!")
case 3:
    print("Three!")
default:
    print("Not One, Two or Three")
}

As instruções switch também funcionam com tipos de dados diferentes de inteiros. Eles funcionam com qualquer tipo de dados. Aqui está um exemplo de ativação de uma string:

let string = "Dog"
switch string {
case "Cat", "Dog":
  print("Animal is a house pet.")
default:
  print("Animal is not a house pet.")
}

Isso imprimirá o seguinte:

Animal is a house pet.

Correspondendo a um intervalo #

Um único caso em uma instrução switch pode corresponder a um intervalo de valores.

let number = 20
switch number {
case 0:
    print("Zero")
case 1..<10:
    print("Between One and Ten")
case 10..<20:
    print("Between Ten and Twenty")
case 20..<30:
    print("Between Twenty and Thirty")
default:
    print("Greater than Thirty or less than Zero")
}

Usando a instrução where em um switch #

A instrução where pode ser usada em uma correspondência de caso de comutação para adicionar critérios adicionais necessários para uma correspondência positiva. O exemplo a seguir verifica não apenas o intervalo, mas também se o número é par ou ímpar:

switch (temperature) {
      case 0...49 where temperature % 2 == 0:
        print("Cold and even")

      case 50...79 where temperature % 2 == 0:
        print("Warm and even")

      case 80...110 where temperature % 2 == 0:
        print("Hot and even")

      default:
        print("Temperature out of range or odd")
}

Correspondência parcial #

A instrução switch faz uso de correspondência parcial.

let coordinates: (x: Int, y: Int, z: Int) = (3, 2, 5)

switch (coordinates) {
case (0, 0, 0): // 1
  print("Origin")
case (_, 0, 0): // 2
  print("On the x-axis.")
case (0, _, 0): // 3
  print("On the y-axis.")
case (0, 0, _): // 4
  print("On the z-axis.")
default:        // 5
  print("Somewhere in space")
}
  1. Corresponde precisamente ao caso em que o valor é (0,0,0). Esta é a origem do espaço 3D.
  2. Corresponde a y=0, z=0 e qualquer valor de x. Isso significa que a coordenada está no eixo x.
  3. Corresponde a x=0, z=0 e qualquer valor de y. Isso significa que a coordenada está no eixo deles.
  4. Corresponde a x=0, y=0 e qualquer valor de z. Isso significa que a coordenada está no eixo z.
  5. Corresponde ao restante das coordenadas.

Nota: usar o sublinhado para significar que você não se importa com o valor.

Se você não quiser ignorar o valor, poderá usá-lo em sua instrução switch, assim:

let coordinates: (x: Int, y: Int, z: Int) = (3, 2, 5)

switch (coordinates) {
case (0, 0, 0):
  print("Origin")
case (let x, 0, 0):
  print("On the x-axis at x = \(x)")
case (0, let y, 0):
  print("On the y-axis at y = \(y)")
case (0, 0, let z):
  print("On the z-axis at z = \(z)")
case (let x, let y, let z):
  print("Somewhere in space at x = \(x), y = \(y), z = \(z)")
}

Aqui, os casos de eixo usam a sintaxe let para extrair os valores pertinentes. O código em seguida, imprime os valores usando a interpolação de string para construir a string.

Nota: você não precisa de um padrão nesta instrução switch. Isso ocorre porque o caso final é essencialmente o padrão - ele corresponde a qualquer coisa, porque não há restrições em nenhuma parte da tupla. Se a instrução switch esgotar todos os valores possíveis com seus casos, nenhum padrão será necessário.

Também podemos usar a sintaxe let-where para corresponder casos mais complexos. Por exemplo:

let coordinates: (x: Int, y: Int, z: Int) = (3, 2, 5)

switch (coordinates) {
case (let x, let y, _) where y == x:
  print("Along the y = x line.")
case (let x, let y, _) where y == x * x:
  print("Along the y = x^2 line.")
default:
break
}

Aqui, combinamos as linhas “y igual a x” e “y igual a x ao quadrado”.

Correspondência de vários valores #

Um único caso em uma instrução switch pode corresponder a vários valores.

let number = 3
switch number {
case 1, 2:
    print("One or Two!")
case 3:
    print("Three!")
case 4, 5, 6:
    print("Four, Five or Six!")
default:
    print("Not One, Two, Three, Four, Five or Six")
}

Satisfaça uma das várias restrições usando switch #

Você pode criar uma tupla e usar um switch assim:

var str: String? = "hi"
var x: Int? = 5

switch (str, x) {
case (.Some,.Some):
    print("Both have values")
case (.Some, nil):
    print("String has a value")
case (nil, .Some):
    print("Int has a value")
case (nil, nil):
    print("Neither have values")
}

Switch e Enums #

A instrução Switch funciona muito bem com valores Enum

enum CarModel {
    case Standard, Fast, VeryFast
}

let car = CarModel.Standard

switch car {
case .Standard: print("Standard")
case .Fast: print("Fast")
case .VeryFast: print("VeryFast")
}

Como fornecemos um caso para cada valor possível de car, omitimos o caso default.

Switches e tuplas #

Os switches podem ativar tuplas:

public typealias mdyTuple = (month: Int, day: Int, year: Int)

let fredsBirthday =   (month: 4, day: 3, year: 1973)




switch theMDY
{
//You can match on a literal tuple:
case (fredsBirthday):
  message = "\(date) \(prefix) the day Fred was born"

//You can match on some of the terms, and ignore others:
case (3, 15, _):
  message = "Beware the Ides of March"

//You can match on parts of a literal tuple, and copy other elements
//into a constant that you use in the body of the case:
case (bobsBirthday.month, bobsBirthday.day, let year) where year > bobsBirthday.year:
  message = "\(date) \(prefix) Bob's \(possessiveNumber(year - bobsBirthday.year))" +
    "birthday"

//You can copy one or more elements of the tuple into a constant and then
//add a where clause that further qualifies the case:
case (susansBirthday.month, susansBirthday.day, let year) 
  where year > susansBirthday.year:
  message = "\(date) \(prefix) Susan's " +
    "\(possessiveNumber(year - susansBirthday.year)) birthday"

//You can match some elements to ranges:.
case (5, 1...15, let year):
  message = "\(date) \(prefix) in the first half of May, \(year)"
}

Correspondência baseada na classe - ótimo para prepareForSegue #

Você também pode fazer um switch de instrução switch com base na class da coisa que você está ligando.

Um exemplo onde isso é útil é em prepareForSegue. Eu costumava mudar com base no identificador segue, mas isso é frágil. se você alterar seu storyboard posteriormente e renomear o identificador segue, ele quebrará seu código. Ou, se você usar segues para várias instâncias da mesma classe de controlador de exibição (mas cenas de storyboard diferentes), não poderá usar o identificador de segue para descobrir a classe do destino.

Declarações de switch Swift para o resgate.

Use a sintaxe case let var as Class do Swift, assim:

override func prepareForSegue(segue: UIStoryboardSegue, sender: AnyObject?) {
  switch segue.destinationViewController {
    case let fooViewController as FooViewController:
      fooViewController.delegate = self

    case let barViewController as BarViewController:
      barViewController.data = data

    default:
      break
  }
}

No Swift 3, o sytax mudou um pouco:

override func prepare(for segue: UIStoryboardSegue, sender: AnyObject?) {       
  switch segue.destinationViewController {
    case let fooViewController as FooViewController:
      fooViewController.delegate = self

    case let barViewController as BarViewController:
      barViewController.data = data

    default:
      break
  }
}

Alternar falhas #

Vale a pena notar que em swift, diferentemente de outras linguagens com as quais as pessoas estão familiarizadas, há uma quebra implícita no final de cada instrução case. Para seguir para o próximo caso (ou seja, executar vários casos), você precisa usar a instrução fallthrough.

switch(value) {
case 'one':
    // do operation one
    fallthrough
case 'two':
    // do this either independant, or in conjunction with first case
default:
    // default operation
}

isso é útil para coisas como fluxos.
