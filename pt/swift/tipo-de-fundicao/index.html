Sintaxe #

  • let nome = json["nome"] como? Corda ?? "" // Saída: john

  • let nome = json["nome"] como? String // Saída: Opcional("john")

  • let nome = classificar como? Int // Saída: Opcional(1)

  • let nome = classificar como? ?? 0 // Saída: 1

  • let nome = dicionário como? [String: Qualquer] ?? [:] // Saída: ["name" : "john", "subjects": ["Maths", "Science", "English", "C Language"]]

Baixando #

Uma variável pode ser reduzida para um subtipo usando os operadores de conversão de tipo as? e as!.

O operador as? tenta converter para um subtipo. Ele pode falhar, portanto, retorna um opcional.

let value: Any = "John"

let name = value as? String
print(name) // prints Optional("John")

let age = value as? Double
print(age) // prints nil

O operador as! força um lançamento. Ele não retorna um opcional, mas trava se a conversão falhar.

let value: Any = "Paul"

let name = value as! String
print(name) // prints "Paul"

let age = value as! Double // crash: "Could not cast value…"

É comum usar operadores de conversão de tipo com desempacotamento condicional:

let value: Any = "George"

if let name = value as? String {
    print(name) // prints "George"
}

if let age = value as? Double {
    print(age) // Not executed
}

Tipo de conversão em linguagem Swift #

Tipo de Fundição #

A conversão de tipo é uma maneira de verificar o tipo de uma instância ou de tratar essa instância como uma superclasse ou subclasse diferente de algum outro lugar em sua própria hierarquia de classes.

A conversão de tipos em Swift é implementada com os operadores is e as. Esses dois operadores fornecem uma maneira simples e expressiva de verificar o tipo de um valor ou converter um valor em um tipo diferente.

Downcasting #

Uma constante ou variável de um determinado tipo de classe pode realmente se referir a uma instância de uma subclasse nos bastidores. Onde você acredita que este é o caso, você pode tentar fazer downcast para o tipo de subclasse com um operador de conversão de tipo (as? ou as!).

Como o downcasting pode falhar, o operador type cast vem em duas formas diferentes. A forma condicional, as?, retorna um valor opcional do tipo para o qual você está tentando fazer downcast. A forma forçada, as!, tenta o abatimento e desdobra o resultado à força como uma única ação composta.

Use a forma condicional do operador de conversão de tipo (as?) quando não tiver certeza se o downcast será bem-sucedido. Esta forma do operador sempre retornará um valor opcional, e o valor será nil se o downcast não for possível. Isso permite que você verifique se há um downcast bem-sucedido.

Use a forma forçada do operador de conversão de tipo (as!) somente quando tiver certeza de que o downcast sempre será bem-sucedido. Essa forma do operador acionará um erro de tempo de execução se você tentar fazer downcast para um tipo de classe incorreto. Saiba mais.

Conversão de String para Int & Float: - #

     let numbers = "888.00"
     let intValue = NSString(string: numbers).integerValue
     print(intValue) // Output - 888

     
     let numbers = "888.00"
     let floatValue = NSString(string: numbers).floatValue
     print(floatValue) // Output : 888.0

Conversão de Float para String #

    let numbers = 888.00
    let floatValue = String(numbers) 
    print(floatValue) // Output : 888.0

    // Get Float value at particular decimal point 
    let numbers = 888.00
    let floatValue = String(format: "%.2f", numbers) // Here %.2f will give 2 numbers after decimal points we can use as per our need
    print(floatValue) // Output : "888.00"

Valor inteiro para String #

    let numbers = 888
    let intValue = String(numbers)
    print(intValue) // Output : "888"

Float to String value #

    let numbers = 888.00
    let floatValue = String(numbers)
    print(floatValue)

Valor flutuante opcional para String #

    let numbers: Any = 888.00
    let floatValue = String(describing: numbers)
    print(floatValue) // Output : 888.0

String opcional para valor Int #

    let hitCount = "100"
    let data :AnyObject = hitCount
    let score = Int(data as? String ?? "") ?? 0
    print(score)

Downcasting de valores de JSON #

    let json = ["name" : "john", "subjects": ["Maths", "Science", "English", "C Language"]] as [String : Any]
    let name = json["name"] as? String ?? ""
    print(name) // Output : john
    let subjects = json["subjects"] as? [String] ?? []
    print(subjects) // Output : ["Maths", "Science", "English", "C Language"]

Downcasting de valores do JSON opcional #

    let response: Any = ["name" : "john", "subjects": ["Maths", "Science", "English", "C Language"]]
    let json = response as? [String: Any] ?? [:]
    let name = json["name"] as? String ?? ""
    print(name) // Output : john
    let subjects = json["subjects"] as? [String] ?? []
    print(subjects) // Output : ["Maths", "Science", "English", "C Language"]

Gerenciar resposta JSON com condições #

    let response: Any = ["name" : "john", "subjects": ["Maths", "Science", "English", "C Language"]] //Optional Response 
    
    guard let json = response as? [String: Any] else {
        // Handle here nil value
        print("Empty Dictionary")
        // Do something here
        return
    }
    let name = json["name"] as? String ?? ""
    print(name) // Output : john
    let subjects = json["subjects"] as? [String] ?? []
    print(subjects) // Output : ["Maths", "Science", "English", "C Language"]

Gerenciar resposta nula com condição #

    let response: Any? = nil
    guard let json = response as? [String: Any] else {
        // Handle here nil value
        print("Empty Dictionary")
        // Do something here
        return
    }
    let name = json["name"] as? String ?? ""
    print(name) 
    let subjects = json["subjects"] as? [String] ?? []
    print(subjects)

Saída: Dicionário vazio #

Fundição com interruptor #

A instrução switch também pode ser usada para tentar converter em diferentes tipos:

func checkType(_ value: Any) -> String {
    switch value {

    // The `is` operator can be used to check a type
    case is Double:
        return "value is a Double"

    // The `as` operator will cast. You do not need to use `as?` in a `switch`.
    case let string as String:
        return "value is the string: \(string)"

    default:
        return "value is something else"
    }

}

checkType("Cadena")  // "value is the string: Cadena"
checkType(6.28)      // "value is a Double"
checkType(UILabel()) // "value is something else"

Upcast #

O operador as converterá para um supertipo. Como não pode falhar, não retorna um opcional.

let name = "Ringo"
let value = string as Any  // `value` is of type `Any` now

Exemplo de uso de um downcast em um parâmetro de função envolvendo subclasses #

Um downcast pode ser usado para usar o código e os dados de uma subclasse dentro de uma função tomando um parâmetro de sua superclasse.

class Rat {
    var color = "white"
}

class PetRat: Rat {
    var name = "Spot"
}

func nameOfRat(🐭: Rat) -> String {
    guard let petRat = (🐭 as? PetRat) else {
        return "No name"
    }
    
    return petRat.name
}

let noName = Rat()
let spot = PetRat()

print(nameOfRat(noName))
print(nameOfRat(spot))
