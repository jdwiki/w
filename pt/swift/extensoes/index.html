O que são extensões? #

Extensões são usados ​​para estender a funcionalidade de tipos existentes no Swift. As extensões podem adicionar subscritos, funções, inicializadores e propriedades computadas. Eles também podem tornar os tipos em conformidade com protocolos.

Suponha que você queira calcular o fatorial de um Int. Você pode adicionar uma propriedade computada em uma extensão:

extension Int {
    var factorial: Int {
        return (1..<self+1).reduce(1, combine: *)
    }
}

Então você pode acessar a propriedade como se ela tivesse sido incluída na API Int original.

let val1: Int = 10

val1.factorial  // returns 3628800

Variáveis ​​e funções #

As extensões podem conter funções e variáveis ​​get computadas/constantes. Estão no formato

extension ExtensionOf {
    //new functions and get-variables
}

Para referenciar a instância do objeto estendido, self pode ser usado, assim como poderia ser usado

Para criar uma extensão de String que adiciona uma função .length() que retorna o comprimento da string, por exemplo

extension String {
    func length() -> Int {
        return self.characters.count
    }
}

"Hello, World!".length() // 13

As extensões também podem conter variáveis ​​get. Por exemplo, adicionar uma variável .length à string que retorna o comprimento da string

extension String {
    var length: Int {
        get {
            return self.characters.count
        }
    }
}

"Hello, World!".length // 13

Inicializadores em Extensões #

As extensões podem conter inicializadores de conveniência. Por exemplo, um inicializador com falha para Int que aceita um NSString:

extension Int {
    init?(_ string: NSString) {
        self.init(string as String)  // delegate to the existing Int.init(String) initializer
    }
}

let str1: NSString = "42"
Int(str1) // 42

let str2: NSString = "abc"
Int(str2) // nil

Extensões de protocolo #

Um recurso muito útil do Swift 2.2 é ter a capacidade de estender protocolos.

Funciona praticamente como classes abstratas quando se trata de uma funcionalidade que você deseja que esteja disponível em todas as classes que implementam algum protocolo (sem ter que herdar de uma classe base comum).

protocol FooProtocol {
    func doSomething()
}

extension FooProtocol {
    func doSomething() {
        print("Hi")
    }
}

class Foo: FooProtocol {
    func myMethod() {
        doSomething() // By just implementing the protocol this method is available
    }
}

Isso também é possível usando genéricos.

Subscritos #

As extensões podem adicionar novos subscritos a um tipo existente.

Este exemplo obtém o caractere dentro de uma String usando o índice fornecido:

extension String {
    subscript(index: Int) -> Character {
        let newIndex = startIndex.advancedBy(index)
        return self[newIndex]
    }
}

var myString = "StackOverFlow"
print(myString[2]) // a
print(myString[3]) // c

extension String {
    subscript(offset: Int) -> Character {
        let newIndex = self.index(self.startIndex, offsetBy: offset)
        return self[newIndex]
    }
}

var myString = "StackOverFlow"
print(myString[2]) // a
print(myString[3]) // c

Restrições #

É possível escrever um método em um tipo genérico que seja mais restritivo usando a sentença where.

extension Array where Element: StringLiteralConvertible {
  func toUpperCase() -> [String] {
      var result = [String]()
      for value in self {
          result.append(String(value).uppercaseString)
      }
      return result
    }        
}

Exemplo de uso

let array = ["a","b","c"]
let resultado = array.toUpperCase()

O que são extensões e quando usá-las #

As extensões adicionam novas funcionalidades a uma classe, estrutura, enumeração ou tipo de protocolo existente. Isso inclui a capacidade de estender tipos para os quais você não tem acesso ao código-fonte original.

Extensões no Swift podem:

  • Adicionar propriedades computadas e propriedades de tipo computado
  • Definir métodos de instância e métodos de tipo
  • Fornecer novos inicializadores
  • Definir subscritos
  • Definir e usar novos tipos aninhados
  • Tornar um tipo existente em conformidade com um protocolo

Quando usar extensões Swift:

  • Funcionalidade adicional ao Swift
  • Funcionalidade adicional para UIKit / Foundation
  • Funcionalidade adicional sem mexer com o código de outras pessoas
  • Divisão de classes em: Dados / Funcionalidade / Delegado

Quando não usar:

  • Estenda suas próprias aulas de outro arquivo

Exemplo simples:

extension Bool {
    public mutating func toggle() -> Bool {
        self = !self
        return self
    }
}

var myBool: Bool = true
print(myBool.toggle()) // false
