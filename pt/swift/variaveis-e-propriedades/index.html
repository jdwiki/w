Criando uma variável #

Declare uma nova variável com var, seguido por um nome, tipo e valor:

var num: Int = 10

Variáveis ​​podem ter seus valores alterados:

num = 20 // num now equals 20

A menos que sejam definidos com let:

let num: Int = 10 // num cannot change

Swift infere o tipo de variável, então você nem sempre precisa declarar o tipo de variável:

let ten = 10 // num is an Int
let pi = 3.14 // pi is a Double
let floatPi: Float = 3.14 // floatPi is a Float

Os nomes de variáveis ​​não são restritos a letras e números - eles também podem conter a maioria dos outros caracteres unicode, embora existam algumas restrições

Nomes de constantes e variáveis ​​não podem conter caracteres de espaço em branco, símbolos matemáticos, setas, pontos de código Unicode de uso privado (ou inválidos) ou caracteres de desenho de linha e caixa. Nem podem começar com um número

Fonte developer.apple.com

var π: Double = 3.14159
var 🍎🍏: String = "Apples"

Observadores de propriedade #

Os observadores de propriedade respondem a alterações no valor de uma propriedade.

var myProperty = 5 {
    willSet {
        print("Will set to \(newValue). It was previously \(myProperty)")
    }
    didSet {
        print("Did set to \(myProperty). It was previously \(oldValue)")
    }
}
myProperty = 6
// prints: Will set to 6, It was previously 5
// prints: Did set to 6. It was previously 5
  • willSet é chamado antes de myProperty ser definido. O novo valor está disponível como newValue, e o valor antigo ainda está disponível como myProperty.
  • didSet é chamado depois que myProperty é definido. O valor antigo está disponível como oldValue e o novo valor agora está disponível como myProperty .

Nota: didSet e willSet não serão chamados nos seguintes casos:

  • Atribuindo um valor inicial
  • Modificando a variável dentro de seu próprio didSet ou willSet

  • Os nomes dos parâmetros para oldValue e newValue de didSet e willSet também podem ser declarados para aumentar a legibilidade:

    var myFontSize = 10 { willSet(newFontSize) { print(“Will set font to (newFontSize), it was (myFontSize)”) } didSet(oldFontSize) { print(“Did set font to (myFontSize), it was (oldFontSize)”) } }

Cuidado: Embora seja suportado declarar nomes de parâmetros setter, deve-se ter cuidado para não misturar nomes:

  • willSet(oldValue) e didSet(newValue) são totalmente legais, mas irão confundir consideravelmente os leitores do seu código.

Propriedades armazenadas preguiçosas #

Propriedades armazenadas preguiçosamente têm valores que não são calculados até serem acessados ​​pela primeira vez. Isso é útil para economia de memória quando o cálculo da variável é computacionalmente caro. Você declara uma propriedade lazy com lazy:

lazy var veryExpensiveVariable = expensiveMethod()

Muitas vezes, é atribuído a um valor de retorno de um encerramento:

lazy var veryExpensiveString = { () -> String in
    var str = expensiveStrFetch()
    str.expensiveManipulation(integer: arc4random_uniform(5))
    return str
}()

Propriedades armazenadas preguiçosas devem ser declaradas com var.

Noções básicas de propriedade #

As propriedades podem ser adicionadas a uma class ou struct (tecnicamente enums também, veja o exemplo “Computed Properties”). Estes adicionam valores que se associam a instâncias de classes/structs:

class Dog {
    var name = ""
}

No caso acima, instâncias de Dog possuem uma propriedade chamada name do tipo String. A propriedade pode ser acessada e modificada em instâncias de Dog:

let myDog = Dog()
myDog.name = "Doggy" // myDog's name is now "Doggy"

Esses tipos de propriedades são considerados propriedades armazenadas, pois armazenam algo em um objeto e afetam sua memória.

Propriedades computadas #

Diferente das propriedades armazenadas, as propriedades computadas são construídas com um getter e um setter, executando o código necessário quando acessadas e definidas. As propriedades computadas devem definir um tipo:

var pi = 3.14

class Circle {
    var radius = 0.0
    var circumference: Double {
        get {
            return pi * radius * 2
        }
        set {
            radius = newValue / pi / 2
        }
    }
}

let circle = Circle()
circle.radius = 1
print(circle.circumference) // Prints "6.28"
circle.circumference = 14
print(circle.radius) // Prints "2.229..."

Uma propriedade computada somente leitura ainda é declarada com um var:

var circumference: Double {
    get {
        return pi * radius * 2
    }
}

Propriedades computadas somente leitura podem ser reduzidas para excluir get:

var circumference: Double {
    return pi * radius * 2
}

Variáveis ​​locais e globais #

As variáveis ​​locais são definidas dentro de uma função, método ou encerramento:

func printSomething() {
    let localString = "I'm local!"
    print(localString)
}

func printSomethingAgain() {
    print(localString) // error
}

As variáveis ​​globais são definidas fora de uma função, método ou encerramento e não são definidas dentro de um tipo (pense fora de todos os colchetes). Eles podem ser usados ​​em qualquer lugar:

let globalString = "I'm global!"
print(globalString)

func useGlobalString() {
    print(globalString) // works!
}

for i in 0..<2 {
    print(globalString) // works!
}

class GlobalStringUser {
    var computeGlobalString {
        return globalString // works!
    }
}

As variáveis ​​globais são definidas lentamente (consulte o exemplo “Propriedades preguiçosas”).

Propriedades do tipo #

Propriedades de tipo são propriedades no próprio tipo, não na instância. Eles podem ser propriedades armazenadas ou computadas. Você declara uma propriedade de tipo com static:

struct Dog {
    static var noise = "Bark!"
}

print(Dog.noise) // Prints "Bark!"

Em uma classe, você pode usar a palavra-chave class em vez de static para torná-la substituível. No entanto, você só pode aplicar isso em propriedades computadas:

class Animal {
    class var noise: String {
        return "Animal noise!"
    }
}
class Pig: Animal {
    override class var noise: String {
        return "Oink oink!"
    }
}

Isso é usado frequentemente com o padrão singleton.
