Funções com parâmetros #

As funções podem receber parâmetros para que sua funcionalidade possa ser modificada. Os parâmetros são fornecidos como uma lista separada por vírgulas com seus tipos e nomes definidos.

func magicNumber(number1: Int)
{
    print("\(number1) Is the magic number")
}

Observação: A sintaxe \(number1) é básica String Interpolation e é usada para inserir o inteiro na String.

Funções com parâmetros são chamadas especificando a função pelo nome e fornecendo um valor de entrada do tipo usado na declaração da função.

magicNumber(5)
//output: "5 Is the magic number
let example: Int = 10
magicNumber(example)
//output: "10 Is the magic number"

Qualquer valor do tipo Int poderia ter sido usado.

func magicNumber(number1: Int, number2: Int)
{
    print("\(number1 + number2) Is the magic number")
}

Quando uma função usa vários parâmetros, o nome do primeiro parâmetro não é necessário para o primeiro, mas está nos parâmetros subsequentes.

let ten: Int = 10
let five: Int = 5
magicNumber(ten,number2: five)
//output: "15 Is the magic number"

Use nomes de parâmetros externos para tornar as chamadas de função mais legíveis.

func magicNumber(one number1: Int, two number2: Int)
{
    print("\(number1 + number2) Is the magic number")
}

let ten: Int = 10
let five: Int = 5
magicNumber(one: ten, two: five)

Definir o valor padrão na declaração da função permite chamar a função sem fornecer nenhum valor de entrada.

func magicNumber(one number1: Int = 5, two number2: Int = 10)
{
    print("\(number1 + number2) Is the magic number")
}

magicNumber()
//output: "15 Is the magic number"

1: https://www.wikiod.com/pt/swift/strings-e-personagens#Literais de Strings e Caracteres

Métodos #

Métodos de instância são funções que pertencem a instâncias de um tipo em Swift (uma class, struct, enumeration ou protocol). Métodos de tipo são chamados no próprio tipo.

Métodos de instância #

Métodos de instância são definidos com uma declaração func dentro da definição do tipo, ou em uma extensão.

class Counter {
    var count = 0
    func increment() {
        count += 1
    }
}

O método de instância increment() é chamado em uma instância da classe Counter:

let counter = Counter()  // create an instance of Counter class   
counter.increment()      // call the instance method on this instance

Métodos de tipo #

Os métodos de tipo são definidos com as palavras-chave static func. (Para classes, class func define um método de tipo que pode ser substituído por subclasses.)

class SomeClass {
    class func someTypeMethod() {
        // type method implementation goes here
    }
}
SomeClass.someTypeMethod()  // type method is called on the SomeClass type itself

Subscritos #

Operadores são funções #

Operadores como +, -, ?? são um tipo de função nomeada usando símbolos em vez de letras. Eles são invocados de forma diferente das funções:

  • Prefixo: -x
  • Infixo: x + y
  • Pós-fixação: x++

Você pode ler mais sobre operadores básicos e [operadores avançados](https://developer.apple. com/library/ios/documentation/Swift/Conceptual/Swift_Programming_Language/BasicOperators.html) na linguagem de programação Swift.

Parâmetros Variáveis #

Às vezes, não é possível listar o número de parâmetros que uma função pode precisar. Considere uma função sum:

func sum(_ a: Int, _ b: Int) -> Int {
    return a + b
}

Isso funciona bem para encontrar a soma de dois números, mas para encontrar a soma de três teríamos que escrever outra função:

func sum(_ a: Int, _ b: Int, _ c: Int) -> Int {
    return a + b + c
}

e um com quatro parâmetros precisaria de outro, e assim por diante. O Swift torna possível definir uma função com um número variável de parâmetros usando uma sequência de três períodos: .... Por exemplo,

func sum(_ numbers: Int...) -> Int {
    return numbers.reduce(0, combine: +)
}

Observe como o parâmetro numbers, que é variável, é unido em um único Array do tipo [Int]. Isso é verdade em geral, parâmetros variáveis ​​do tipo T... são acessíveis como um [T].

Esta função agora pode ser chamada assim:

let a = sum(1, 2) // a == 3
let b = sum(3, 4, 5, 6, 7) // b == 25

Um parâmetro variadic em Swift não precisa vir no final da lista de parâmetros, mas pode haver apenas um em cada assinatura de função.

Às vezes, é conveniente colocar um tamanho mínimo no número de parâmetros. Por exemplo, não faz sentido pegar a ‘soma’ de nenhum valor. Uma maneira fácil de impor isso é colocar alguns parâmetros obrigatórios não variádicos e, em seguida, adicionar o parâmetro variádico depois. Para ter certeza de que sum só pode ser chamado com pelo menos dois parâmetros, podemos escrever

func sum(_ n1: Int, _ n2: Int, _ numbers: Int...) -> Int {
    return numbers.reduce(n1 + n2, combine: +)
}

sum(1, 2) // ok
sum(3, 4, 5, 6, 7) // ok
sum(1) // not ok
sum() // not ok

Uso Básico #

As funções podem ser declaradas sem parâmetros ou um valor de retorno. A única informação necessária é um nome (hello neste caso).

func hello()
{
    print("Hello World")
}

Chame uma função sem parâmetros escrevendo seu nome seguido por um par de parênteses vazio.

hello()
//output: "Hello World"

Valores de retorno #

As funções podem retornar valores especificando o tipo após a lista de parâmetros.

func findHypotenuse(a: Double, b: Double) -> Double
{
    return sqrt((a * a) + (b * b))
}

let c = findHypotenuse(3, b: 5)
//c = 5.830951894845301

As funções também podem retornar vários valores usando tuplas.

func maths(number: Int) -> (times2: Int, times3: Int)
{
    let two = number * 2
    let three = number * 3
    return (two, three)
}
let resultTuple = maths(5)
//resultTuple = (10, 15)

Erros de lançamento #

Se você deseja que uma função possa lançar erros, você precisa adicionar a palavra-chave throws após os parênteses que contêm os argumentos:

func errorThrower()throws -> String {}

Quando você quiser lançar um erro, use a palavra-chave throw:

func errorThrower()throws -> String {
  if true {
    return "True"
  } else {
    // Throwing an error
    throw Error.error 
  }
}

Se você quiser chamar uma função que pode gerar um erro, você precisa usar a palavra-chave try em um bloco do:

do {
  try errorThrower()
}

Para saber mais sobre erros do Swift: https://www.wikiod.com/pt/swift/manipulacao-de-erros

Parâmetros de saída #

As funções podem modificar os parâmetros passados ​​a elas se estiverem marcadas com a palavra-chave inout. Ao passar um parâmetro inout para uma função, o chamador deve adicionar um & à variável que está sendo passada.

func updateFruit(fruit: inout Int) {
    fruit -= 1
}

var apples = 30 // Prints "There's 30 apples"
print("There's \(apples) apples")

updateFruit(fruit: &apples)

print("There's now \(apples) apples") // Prints "There's 29 apples".

Isso permite que a semântica de referência seja aplicada a tipos que normalmente teriam semântica de valor.

Sintaxe de fechamento à direita #

Quando o último parâmetro de uma função é um closure

func loadData(id: String, completion:(result: String) -> ()) {
    // ...
    completion(result:"This is the result data")
}

a função pode ser invocada usando a sintaxe de fechamento à direita

loadData("123") { result in
    print(result)
}

Funções com Fechamentos #

O uso de funções que recebem e executam closures pode ser extremamente útil para enviar um bloco de código para ser executado em outro lugar. Podemos começar permitindo que nossa função receba um encerramento opcional que (neste caso) retornará Void.

func closedFunc(block: (()->Void)? = nil) {
    print("Just beginning")

    if let block = block {
        block()
    }
}

Agora que nossa função foi definida, vamos chamá-la e passar algum código:

closedFunc() { Void in
    print("Over already")
}

Usando um trailing closure com nossa chamada de função, podemos passar o código (neste caso, print) para ser executado em algum ponto dentro de nossa função closedFunc().

O log deve imprimir:

Apenas começando

Já acabou

Um caso de uso mais específico disso poderia incluir a execução de código entre duas classes:

class ViewController: UIViewController {

    override func viewDidLoad() {
        let _  = A.init(){Void in self.action(2)}
    }

    func action(i: Int) {
        print(i)
    }
}


class A: NSObject {
    var closure : ()?

    init(closure: (()->Void)? = nil) {
        // Notice how this is executed before the  closure
        print("1")
        // Make sure closure isn't nil
        self.closure = closure?()
    }
}

O log deve imprimir:

1

2

Passando e retornando funções #

A função a seguir está retornando outra função como resultado que pode ser posteriormente atribuída a uma variável e chamada:

func jediTrainer () -> ((String, Int) -> String) {
  func train(name: String, times: Int) -> (String) {
    return "\(name) has been trained in the Force \(times) times"
  }
  return train
}

let train = jediTrainer()
train("Obi Wan", 3)

Tipos de funções #

Cada função tem seu próprio tipo de função, composto pelos tipos de parâmetro e pelo tipo de retorno da própria função. Por exemplo a seguinte função:

func sum(x: Int, y: Int) -> (result: Int) { return x + y }

tem um tipo de função de:

(Int, Int) -> (Int)

Os tipos de função podem ser usados ​​como tipos de parâmetros ou como tipos de retorno para funções de aninhamento.
