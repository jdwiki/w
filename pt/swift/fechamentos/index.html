Sintaxe #

  • var closureVar: (<parameters>) -> (<returnType>) // Como uma variável ou tipo de propriedade
  • typealias ClosureType = (<parâmetros>) -> (<returnType>)
  • { [<captureList>] (<parameters>) <throws-ness> -> <returnType> in <statements> } // Sintaxe de fechamento completa

Para obter mais informações sobre encerramentos do Swift, consulte documentação da Apple.

Noções básicas de fechamento #

Fechamentos (também conhecidos como blocos ou lambdas) são pedaços de código que podem ser armazenados e repassados ​​dentro do seu programa.

let sayHi = { print("Hello") }
// The type of sayHi is "() -> ()", aka "() -> Void"

sayHi()  // prints "Hello"

Como outras funções, closures podem aceitar argumentos e retornar resultados ou lançar erros:

let addInts = { (x: Int, y: Int) -> Int in
    return x + y
}
// The type of addInts is "(Int, Int) -> Int"

let result = addInts(1, 2)  // result is 3

let divideInts = { (x: Int, y: Int) throws -> Int in
    if y == 0 {
        throw MyErrors.DivisionByZero
    }
    return x / y
}
// The type of divideInts is "(Int, Int) throws -> Int"

Os encerramentos podem capturar valores de seu escopo:

// This function returns another function which returns an integer
func makeProducer(x: Int) -> (() -> Int) {
    let closure = { x }  // x is captured by the closure
    return closure
}

// These two function calls use the exact same code,
// but each closure has captured different values.
let three = makeProducer(3)
let four = makeProducer(4)
three()  // returns 3
four()  // returns 4

Closures podem ser passados ​​diretamente para funções:

let squares = (1...10).map({ $0 * $0 })  // returns [1, 4, 9, 16, 25, 36, 49, 64, 81, 100]
let squares = (1...10).map { $0 * $0 }

NSURLSession.sharedSession().dataTaskWithURL(myURL,
    completionHandler: { (data: NSData?, response: NSURLResponse?, error: NSError?) in
        if let data = data {
            print("Request succeeded, data: \(data)")
        } else {
            print("Request failed: \(error)")
        }
    }).resume()

Variações de sintaxe #

A sintaxe básica de fechamento é

{ [capture list] (parameters) throws-ness -> return type in body }.

Muitas dessas partes podem ser omitidas, portanto, existem várias maneiras equivalentes de escrever closures simples:

let addOne = { [] (x: Int) -> Int in return x + 1 }
let addOne = { [] (x: Int) -> Int in x + 1 }
let addOne = { (x: Int) -> Int in x + 1 }
let addOne = { x -> Int in x + 1 }
let addOne = { x in x + 1 }
let addOne = { $0 + 1 }

let addOneOrThrow = { [] (x: Int) throws -> Int in return x + 1 }
let addOneOrThrow = { [] (x: Int) throws -> Int in x + 1 }
let addOneOrThrow = { (x: Int) throws -> Int in x + 1 }
let addOneOrThrow = { x throws -> Int in x + 1 }
let addOneOrThrow = { x throws in x + 1 }

  • A lista de captura pode ser omitida se estiver vazia.

  • Os parâmetros não precisam de anotações de tipo se seus tipos puderem ser inferidos.

  • O tipo de retorno não precisa ser especificado se puder ser inferido.

  • Os parâmetros não precisam ser nomeados; em vez disso, eles podem ser referidos com $0, $1, $2, etc.

  • Se o encerramento contém uma única expressão, cujo valor deve ser retornado, a palavra-chave return pode ser omitida.

  • Se o encerramento é inferido para lançar um erro, é escrito em um contexto que espera um encerramento de lançamento, ou não lança um erro, throws pode ser omitido.

    // The closure’s type is unknown, so we have to specify the type of x and y. // The output type is inferred to be Int, because the + operator for Ints returns Int. let addInts = { (x: Int, y: Int) in x + y }

    // The closure’s type is specified, so we can omit the parameters’ type annotations. let addInts: (Int, Int) -> Int = { x, y in x + y } let addInts: (Int, Int) -> Int = { $0 + $1 }

Passando closures em funções #

As funções podem aceitar closures (ou outras funções) como parâmetros:

func foo(value: Double, block: () -> Void) { ... }
func foo(value: Double, block: Int -> Int) { ... }
func foo(value: Double, block: (Int, Int) -> String) { ... }

Sintaxe de fechamento à direita #

Se o último parâmetro de uma função for uma closure, as chaves de closure {/} podem ser escritas após a invocação da função:

foo(3.5, block: { print("Hello") })

foo(3.5) { print("Hello") }

dispatch_async(dispatch_get_main_queue(), {
    print("Hello from the main queue")
})

dispatch_async(dispatch_get_main_queue()) {
    print("Hello from the main queue")
}

Se o único argumento de uma função é um encerramento, você também pode omitir o par de parênteses () ao chamá-lo com a sintaxe de encerramento à direita:

func bar(block: () -> Void) { ... }


bar() { print("Hello") }

bar { print("Hello") }

Parâmetros @noescape #

Parâmetros de fechamento marcados como @noescape têm garantia de execução antes que a chamada da função retorne, portanto, usar self. não é necessário dentro do corpo de fechamento:

func executeNow(@noescape bloco: () -> Void) {
    // Since `block` is @noescape, it's illegal to store it to an external variable.
    // We can only call it right here.
    block()
}

func executeLater(block: () -> Void) {
    dispatch_async(dispatch_get_main_queue()) {
        // Some time in the future...
        block()
    }
}
class MyClass {
    var x = 0
    func showExamples() {
        // error: reference to property 'x' in closure requires explicit 'self.' to make capture semantics explicit
        executeLater { x = 1 }

        executeLater { self.x = 2 }  // ok, the closure explicitly captures self

        // Here "self." is not required, because executeNow() takes a @noescape block.
        executeNow { x = 3 }

        // Again, self. is not required, because map() uses @noescape.
        [1, 2, 3].map { $0 + x }
    }
}

Nota do Swift 3: #

Observe que no Swift 3, você não marca mais blocos como @noescape. Os blocos agora não escapam por padrão. No Swift 3, em vez de marcar um fechamento como sem escape, você marca um parâmetro de função que é um fechamento com escape como escape usando a palavra-chave “@escaping”.

throws e rethrows #

Fechamentos, como outras funções, podem gerar erros:

func executeNowOrIgnoreError(block: () lançamentos -> Void) {
    do {
        try block()
    } catch {
        print("error: \(error)")
    }
}

A função pode, é claro, passar o erro para seu chamador:

func executeNowOrThrow(block: () lançamentos -> Void) lançamentos {
    try block()
}

No entanto, se o bloco passado não é lançado, o chamador ainda está preso a uma função de lançamento:

// It's annoying that this requires "try", because "print()" can't throw!
try executeNowOrThrow { print("Just printing, no errors here!") }

A solução é rethrows, que designa que a função só pode lançar se seu parâmetro de fechamento lançar:

func executeNowOrRethrow(block: () lançamentos -> Void) relançamentos {
    try block()
}

// "try" não é necessário aqui, porque o bloco não pode gerar um erro.
executeNowOrRethrow { print("Nenhum erro é gerado neste encerramento") }

// Este bloco pode lançar um erro, então "try" é necessário.
tente executeNowOrRethrow { throw MyError.Example }

Muitas funções de biblioteca padrão usam rethrows, incluindo map(), filter() e indexOf().

Captura, referências fortes/fracas e retém ciclos #

class MyClass {
    func sayHi() { print("Hello") }
    deinit { print("Goodbye") }
}

Quando um closure captura um tipo de referência (uma instância de classe), ele mantém uma referência forte por padrão:

let closure: () -> Void
do {
    let obj = MyClass()
    // Captures a strong reference to `obj`: the object will be kept alive
    // as long as the closure itself is alive.
    closure = { obj.sayHi() }
    closure()  // The object is still alive; prints "Hello"
} // obj goes out of scope
closure()  // The object is still alive; prints "Hello"

A lista de capturas do closure pode ser usada para especificar uma referência fraca ou sem dono:

deixar encerramento: () -> Void
Faz {
    let obj = MyClass()
    // Captures a weak reference to `obj`: the closure will not keep the object alive;
    // the object becomes optional inside the closure.
    closure = { [weak obj] in obj?.sayHi() }
    closure()  // The object is still alive; prints "Hello"
} // obj sai do escopo e é desalocado; imprime "Adeus"
closure() // `obj` é nil de dentro da closure; isso não imprime nada.
deixar encerramento: () -> Void
Faz {
    let obj = MyClass()
    // Captures an unowned reference to `obj`: the closure will not keep the object alive;
    // the object is always assumed to be accessible while the closure is alive.
    closure = { [unowned obj] in obj.sayHi() }
    closure()  // The object is still alive; prints "Hello"
} // obj sai do escopo e é desalocado; imprime "Adeus"
closure() // crash! obj está sendo acessado após ser desalocado.

Para obter mais informações, consulte o tópico Gerenciamento de memória e [Contagem automática de referência](https://developer.apple .com/library/ios/documentation/Swift/Conceptual/Swift_Programming_Language/AutomaticReferenceCounting.html) da linguagem de programação Swift.

Reter ciclos #

Se um objeto mantém um encerramento, que também mantém uma forte referência ao objeto, este é um [ciclo de retenção](https://www.wikiod.com/pt/swift/gerenciamento-de-memoria reference-cycles-and-weak-references#t=201607211838216466797). A menos que o ciclo seja interrompido, a memória que armazena o objeto e o fechamento será vazada (nunca recuperada).

classe Jogo {
    var score = 0
    let controller: GCController
    init(controller: GCController) {
        self.controller = controller

        // BAD: the block captures self strongly, but self holds the controller
        // (and thus the block) strongly, which is a cycle.
        self.controller.controllerPausedHandler = {
            let curScore = self.score
            print("Pause button pressed; current score: \(curScore)")
        }

        // SOLUTION: use `weak self` to break the cycle.
        self.controller.controllerPausedHandler = { [weak self] in
            guard let strongSelf = self else { return }
            let curScore = strongSelf.score
            print("Pause button pressed; current score: \(curScore)")
        }
    }
}

Usando closures para codificação assíncrona #

Os encerramentos são frequentemente usados ​​para tarefas assíncronas, por exemplo, ao buscar dados de um site.

func getData(urlString: String, callback: (result: NSData?) -> Void) {

    // Turn the URL string into an NSURLRequest.
    guard let url = NSURL(string: urlString) else { return }
    let request = NSURLRequest(URL: url)
    
    // Asynchronously fetch data from the given URL.
    let task = NSURLSession.sharedSession().dataTaskWithRequest(request) {(data: NSData?, response: NSURLResponse?, error: NSError?) in

        // We now have the NSData response from the website.
        // We can get it "out" of the function by using the callback 
        // that was passed to this function as a parameter.

        callback(result: data)
    }
        
    task.resume()
}

Esta função é assíncrona, portanto, não bloqueará o encadeamento em que está sendo chamada (não congelará a interface se chamada no encadeamento principal do seu aplicativo GUI).

print("1. Going to call getData")

getData("http://www.example.com") {(result: NSData?) -> Void in

    // Called when the data from http://www.example.com has been fetched.
    print("2. Fetched data")
}

print("3. Called getData")

Como a tarefa é assíncrona, a saída geralmente será assim:

"1. Going to call getData"
"3. Called getData"
"2. Fetched data"

Porque o código dentro da closure, print("2. Fetched data"), não será chamado até que os dados do URL sejam buscados.

Fechamentos e Alias ​​de Tipo #

Um fechamento pode ser definido com um typealias. Isso fornece um espaço reservado de tipo conveniente se a mesma assinatura de fechamento for usada em vários locais. Por exemplo, retornos de chamada de solicitação de rede comuns ou manipuladores de eventos de interface do usuário são ótimos candidatos para serem “nomeados” com um alias de tipo.

public typealias ClosureType = (x: Int, y: Int) -> Int

Você pode então definir uma função usando o typealias:

função pública closureFunction(closure: ClosureType) {
    let z = closure(1, 2)
}
    
closureFunction() { (x: Int, y: Int) -> Int in return x + y }
