Literais e comentários #

Comentários #

"Comments are enclosed in double quotes. BEWARE: This is NOT a string!"
"They can span
 multiple lines."

Cordas #

'Strings are enclosed in sigle quotes.'
'Single quotes are escaped with a single quote, like this: ''.'
''''  "<--This string contains one single quote"

'Strings too can span
multiple lines'

''   "<--An empty string."

Símbolos #

#thiIsaSymbol  "Symbols are interned strings, used for method and variable names,
                and as values with fast equality checking."
#'hello world' "A symbol with a space in it"
#''            "An empty symbol, not very useful"
#+
#1             "Not the integer 1"

Personagens #

$a     "Characters are not strings. They are preceded by a $."
$A     "An uppercase character"
$      "The spacecharacter!"
$→     "An unicode character"
$1     "Not to be confused with the number 1"

Números #

Os números vêm em todas as variedades:

Inteiros: #

  • Decimal

    10

    -1

    0

    1000000000000000000000000000000000000000000

  • Hexadecimal

    16rAB1F

    16r0

    -16rFFFFFFFFFFFFFFFFFFFFFFFFFFFFFFF

  • EscaladoDecimal

    17s0

    3.14159265s8

  • Outro

    8r7731 “octal”

    2r1001 “binary”

    10r99987 “decimal again!”

Ponto flutuante #

3.14 1.2e3     "2 floating-point numbers"

Frações #

Frações não são o mesmo que números de ponto flutuante, são números exatos (sem erro de arredondamento).

4/3            "The fraction 4/3"
355/113        "A rational approximation to pi"

Matrizes #

#(#abc 123)    "A literal array with the symbol #abc and the number 123"

Matrizes de bytes #

#[1 2 3 4]       "separators are blank"
#[]              "empty ByteArray"
#[0 0 0 0 255]   "length is arbitrary"

Matrizes dinâmicas #

Matrizes dinâmicas são construídas a partir de expressões. Cada expressão dentro das chaves é avaliada como um valor diferente na matriz construída.

{self foo. 3 + 2. i * 3}   "A dynamic array built from 3 expressions"

Blocos #

[ :p | p asString ] "A code block with a parameter p.
                     Blocks are the same as lambdas in other languages"

Algumas notas: #

Observe que as matrizes literais usam qualquer tipo e número de espaços em branco como separadores

#(256 16rAB1F 3.14s2 2r1001 $A #this)
"is the same as:"
#(256
  16rAB1F
  3.14s2
  2r1001
  $A #this)

Observe também que você pode compor literais

#[255 16rFF 8r377 2r11111111]       (four times 255)

#(#[1 2 3] #('string' #symbol))     (arrays of arrays)

Há alguma “tolerância” à notação relaxada

#(symbol) = #(#symbol)              (missing # => symbol)

#('string' ($a 'a'))                (missing # => array)

Mas não aqui:

#([1 2 3]) ~= #(#[1 2 3])           (missing # => misinterpreted)

No entanto

#(true nil false)                   (pseudo variables ok)

Mas não aqui!

#(self) = #(#self)                  (missing # => symbol)

Como você pode ver, existem algumas inconsistências:

  • Enquanto pseudo variáveis ​​true, false e ​​nil são aceitas como literais dentro de arrays, as pseudo variáveis ​​self e super são interpretadas como Símbolos (usando a regra mais geral para strings não qualificadas. )

  • Embora não seja obrigatório escrever #( para iniciar um array aninhado em um array e os parênteses sejam suficientes, é obrigatório escrever #[ para iniciar um ByteArray aninhado.

Algumas delas foram retiradas de:

Envio de mensagem #

Em Smalltalk quase tudo que você faz é enviar mensagens para objetos (chamados de métodos em outras linguagens). Existem três tipos de mensagens:

Mensagens unárias:

#(1 2 3) size
"This sends the #size message to the #(1 2 3) array.
 #size is a unary message, because it takes no arguments."

Mensagens binárias:

1 + 2
"This sends the #+ message and 2 as an argument to the object 1.
 #+ is a binary message because it takes one argument (2)
 and it's composed of one or two symbol characters"

Mensagens de palavra-chave:

'Smalltalk’ allButFirst: 5.
"This sends #allButFirst: with argument 5 to the string 'Smalltalk',
 resulting in the new string 'talk'"

3 to: 10 by: 2.
"This one sends the single message #to:by:, which takes two parameters (10 and 2)
 to the number 3.
 The result is a collection with 3, 5, 7, and 9."

Várias mensagens em uma instrução são avaliadas pela ordem de precedência

unary > binary > keyword

e da esquerda para a direita.

1 + 2 * 3 " equals 9, because it evaluates left to right"
1 + (2 * 3) " but you can use parenthesis"

1 to:   #(a b c d) size   by:   5 - 4
 "is the same as:"
1 to: ( #(a b c d) size ) by: ( 5 - 4 )

Se você quiser enviar muitas mensagens para o mesmo objeto, você pode usar o operador em cascata ;(ponto e vírgula):

OrderedCollection new
  add: #abc;
  add: #def;
  add: #ghi;
  yourself.

“Isso primeiro envia a mensagem #new para a classe OrderedCollection (#new é apenas um message, not an operator). It results in a new OrderedCollection. It then sends the new collection three times the message #add (with different arguments), and the message yourself.”

Classes e métodos #

Classes e métodos geralmente são definidos no IDE Smalltalk.

Aulas #

Uma definição de classe se parece com isso no navegador:

XMLTokenizer subclass: #XMLParser
    instanceVariableNames: ''
    classVariableNames: ''
    poolDictionaries: ''
    category: 'XML-Parser'

Esta é na verdade a mensagem que o navegador enviará para você criar uma nova classe no sistema. (Neste caso é #subclass:instanceVariableNames:classVariableNames:poolDictionaries:category:, mas existem outros que também fazem novas classes).

A primeira linha mostra qual classe você está subclassificando (neste caso é XMLTokenizer) e o nome que a nova subclasse terá (#XMLParser).

As próximas três linhas são usadas para definir as variáveis ​​que a classe e suas instâncias terão.

Métodos #

Os métodos são assim no navegador:

aKeywordMethodWith: firstArgument and: secondArgument
    "Do something with an argument and return the result."

    ^firstArgument doSomethingWith: secondArgument

O ^(caret) é o operador de retorno.

** anInteger
    "Raise me to anInteger"
    | temp1 temp2 |

    temp1 := 1.
    temp2 := 1.
    1 to: anInteger do: [ :i | temp1 := temp1 * self + temp2 - i ].
    ^temp1

esta não é a maneira correta de fazer a exponenciação, mas mostra uma definição de binary message (elas são definidas como qualquer outra mensagem) e algumas variáveis ​​temporárias de método (ou métodos temporários, temp1 e temp2) mais um argumento de bloco (i).

Loops em Smalltalk #

Para este exemplo, uma Ordered Collection será usada para mostrar as diferentes mensagens que podem ser enviadas para um objeto OrderedCollection para fazer um loop sobre os elementos.
O código abaixo instanciará uma OrderedCollection vazia usando a mensagem new e preencha-o com 4 números usando a mensagem add:

anOrderedCollection := OrderedCollection new.
anOrderedCollection add: 1; add: 2; add: 3; add: 4.

Todas essas mensagens terão um bloco como parâmetro que será avaliado para cada um dos elementos dentro da coleção.

  1. do:
    Esta é a mensagem básica de enumeração. Por exemplo, se quisermos imprimir cada elemento da coleção, podemos fazer isso assim:

     anOrderedCollection do:[:each | Transcript show: each]. "Prints --> 1234"

    Each of the elements inside the collection will be defined as the user wishes using this syntax: :each This do: loop will print every element in the collection to the Transcript window.

  2. coletar:
    The collect: message allows you to do something for each item in the collection and puts the result of sua ação em uma nova coleção
    Por exemplo, se quisermos multiplicar cada elemento em nossa coleção por 2 e adicioná-lo a uma nova coleção, podemos usar a mensagem collect: assim:

     evenCollection := anOrderedCollection collect:[:each | each*2]. "#(2 4 6 8)"

  1. selecione:
    A mensagem select: permite que você crie uma subcoleção onde os itens da coleção original são selecionados com base em alguma condição ser verdadeira para eles. Por exemplo, se quisermos criar uma nova coleção de números ímpares de nossa coleção, podemos usar a mensagem select: da seguinte forma:

     oddCollection := anOrderedCollection select:[:each | each odd].

    Since each odd returns a Boolean, only the elements that make the Boolean return true will be added to oddCollection which will have #(1 3).

  2. rejeitar:
    Esta mensagem funciona de forma oposta a select: e rejeita quaisquer elementos que façam o Boolean retornar true. Ou, em outras palavras, ele selecionará quaisquer elementos que façam com que o Booleano retorne false. Por exemplo, se quiséssemos construir o mesmo oddCollection como o exemplo anterior. Podemos usar reject: assim:

     oddCollection := anOrderedCollection reject:[:each | each even].

    oddCollection will again have #(1 3) as its elements.

Estas são as quatro técnicas básicas de enumeração em Smalltalk. No entanto, sinta-se à vontade para navegar na classe Collections para mais mensagens que podem ser implementadas.
