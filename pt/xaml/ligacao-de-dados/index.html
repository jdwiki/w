Sintaxe #

  • <TextBlock Text="{Binding Title}"/>

  • <TextBlock Text="{Binding Path=Title}"/>

  • <TextBlock> <TextBlock.Text> <Binding Path="Title"/> </TextBlock.Text> </TextBlock>

Todas essas tags produzem o mesmo resultado.

String de ligação à propriedade Text #

Para alterar o conteúdo da interface do usuário em tempo de execução, você pode usar Binding. Quando a propriedade vinculada é alterada do código, ela será exibida na interface do usuário.

<TextBlock Text="{Binding Title}"/>

Para notificar a UI sobre mudanças, a propriedade deve gerar o evento PropertyChanged da interface INotifyPropertyChanged ou você pode usar Dependency Property.

O Binding está funcionando se a propriedade “Title” estiver no arquivo xaml.cs ou na classe Datacontext do XAML.

O Datacontext pode ser configurado diretamente no XAML

<Window x:Class="Application.MainWindow"
    xmlns="http://schemas.microsoft.com/winfx/2006/xaml/presentation"
    xmlns:x="http://schemas.microsoft.com/winfx/2006/xaml"
    xmlns:local="clr-namespace:Application">
<Window.DataContext>
   <local:DataContextClass/>
</Window.DataContext>

Formatação de String Bindings #

Ao fazer uma ligação de algo, por exemplo, uma data, você pode querer mostrá-la em um formato específico sem mexer com isso no código.

Para fazer isso, podemos usar a propriedade StringFormat.

aqui estão alguns exemplos:

Text="{Binding Path=ReleaseDate, StringFormat=dddd dd MMMM yyyy}"

Isso formata minha data para o seguinte:

terça-feira, 16 de agosto de 2016

Aqui está outro exemplo de temperatura.

Text="{Binding Path=Temp, StringFormat={}{0}°C}"

Isso formata para:

25°C

O básico de INotifyPropertyChanged #

Se você deseja não apenas exibir objetos estáticos, mas também fazer com que sua interface do usuário responda às alterações nos objetos correlacionados, você precisa entender o básico da interface INotifyPropertyChanged.

Assumindo que temos nossa MainWindowdefinida como

<Window x:Class="Application.MainWindow"
    xmlns="http://schemas.microsoft.com/winfx/2006/xaml/presentation"
    xmlns:x="http://schemas.microsoft.com/winfx/2006/xaml"
    xmlns:d="http://schemas.microsoft.com/expression/blend/2008"
    xmlns:vm="clr-namespace:Application.ViewModels>
    <Window.DataContext>
       <vm:MainWindowViewModel/>
    </Window.DataContext>
    <Grid>
        <TextBlock Text={Binding Path=ApplicationStateText}" />
    </Grid>
</Window>

Com nossa classe Viewmodel MainWindowViewModel definida como

namespace Application.ViewModels
{
    public class MainWindowViewModel
    {
        private string _applicationStateText;

        public string ApplicationStateText
        {
            get { return _applicationStateText; }
            set { _applicationStateText = value; }
        }
        public MainWindowViewModel() 
        { 
            ApplicationStateText = "Hello World!";
        }

    }
}

o TextBlock do nosso Aplicativo exibirá o Texto Hello World devido à sua vinculação. Se nosso ApplicationStateText for alterado durante o tempo de execução, nossa interface do usuário não será notificada sobre essa alteração.
Para implementar isso, nosso DataSource, neste caso nosso MainWindowViewModel, precisa implementar a Interface INotifyPropertyChanged. Isso fará com que nossos Bindings possam se inscrever no PropertyChangedEvent.
Tudo o que precisamos fazer é invocar o PropertyChangedEventHandler sempre que alterarmos nossa propriedade ApplicationStateText assim:

using System.ComponentModel;
using System.Runtime.CompilerServices;

namespace Application.ViewModels
{
    public class MainWindowViewModel : INotifyPropertyChanged
    {
        public event PropertyChangedEventHandler PropertyChanged;
        public void NotifyPropertyChanged( [CallerMemberName] string propertyName = null)
        {
            PropertyChanged?.Invoke(this, new PropertyChangedEventArgs(propertyName));
        }

        private string _applicationStateText;

        public string ApplicationStateText
        {
            get { return _applicationStateText; }
            set
            {
                if (_applicationStateText != value)
                {
                    _applicationStateText = value;
                    NotifyPropertyChanged();
                }
            }
        }
        public MainWindowViewModel()
        {
            ApplicationStateText = "Hello World!";
        }
    }
}

e certifique-se de que nosso Binding de TextBlock.Text realmente escuta um PropertyChangedEvent:

<Window x:Class="Application.MainWindow"
    xmlns="http://schemas.microsoft.com/winfx/2006/xaml/presentation"
    xmlns:x="http://schemas.microsoft.com/winfx/2006/xaml"
    xmlns:d="http://schemas.microsoft.com/expression/blend/2008"
    xmlns:vm="clr-namespace:Application.ViewModels">
    <Window.DataContext>
       <vm:MainWindowViewModel/>
    </Window.DataContext>
    <Grid>
        <TextBlock Text={Binding Path=ApplicationStateText, UpdateSourceTrigger=PropertyChanged }" />
    </Grid>
</Window>

Vinculando a uma coleção de objetos com INotifyPropertyChanged e INotifyCollectionChanged #

Vamos supor que você tenha um ListView que deve exibir todos os objetos User listados na propriedade Users do ViewModel onde as propriedades do objeto User podem ser atualizadas programaticamente.

<ListView ItemsSource="{Binding Path=Users}" >
    <ListView.ItemTemplate>
        <DataTemplate DataType="{x:Type models:User}">
            <StackPanel Orientation="Horizontal">
                <TextBlock Margin="5,3,15,3" 
                         Text="{Binding Id, Mode=OneWay}" />
                <TextBox Width="200"
                         Text="{Binding Name, Mode=TwoWay, UpdateSourceTrigger=PropertyChanged, Delay=450}"/>
            </StackPanel>
        </DataTemplate>
    </ListView.ItemTemplate>
</ListView>

Apesar de INotifyPropertyChanged estar implementado corretamente para o objeto User

public class User : INotifyPropertyChanged
{
    public event PropertyChangedEventHandler PropertyChanged;

    private int _id;
    private string _name;

    public int Id
    {
        get { return _id; }
        private set
        {
            if (_id == value) return;
            _id = value;
            NotifyPropertyChanged();
        }
    }
    public string Name
    {
        get { return _name; }
        set
        {
            if (_name == value) return;
            _name = value;
            NotifyPropertyChanged();
        }
    }

    public User(int id, string name)
    {
        Id = id;
        Name = name;
    }

    private void NotifyPropertyChanged([CallerMemberName] string propertyName = null)
    {
        PropertyChanged?.Invoke(this, new PropertyChangedEventArgs(propertyName));
    }
}

e para o seu objeto ViewModel

public sealed class MainWindowViewModel : INotifyPropertyChanged
{
    public event PropertyChangedEventHandler PropertyChanged;

    private List<User> _users;
    public List<User> Users
    {
        get { return _users; }
        set
        {
            if (_users == value) return;
            _users = value;
            NotifyPropertyChanged();
        }
    }
    public MainWindowViewModel()
    {
        Users = new List<User> {new User(1, "John Doe"), new User(2, "Jane Doe"), new User(3, "Foo Bar")};
    }

    private void NotifyPropertyChanged([CallerMemberName] string propertyName = null)
    {
        PropertyChanged?.Invoke(this, new PropertyChangedEventArgs(propertyName));
    }
}

sua interface do usuário não será atualizada se uma alteração em um usuário for feita programaticamente.

Isso ocorre simplesmente porque você definiu apenas INotifyPropertyChanged na instância da própria lista. Somente se você reinstanciar completamente a Lista se uma propriedade de um Elemento for alterada, sua interface do usuário será atualizada.

// DO NOT DO THIS
User[] userCache = Users.ToArray();
Users = new List<User>(userCache);

No entanto, isso é muito cansativo e inacreditavelmente ruim para o desempenho.
Se você tiver uma lista de 100.000 elementos mostrando tanto o ID quanto o nome do usuário, haverá 200.000 DataBindings em vigor, cada um deles devendo ser recriado. Isso resulta em um atraso perceptível para o usuário sempre que uma alteração é feita em qualquer coisa.

Para resolver parcialmente esse problema, você pode usar System.ComponentModel.ObservableCollection<T> em vez de List<T>:

private ObservableCollection<User> _users;
public ObservableCollection<User> Users
{
    get { return _users; }
    set
    {
        if (_users == value) return;
        _users = value;
        NotifyPropertyChanged();
    }
}

O ObservableCollection nos fornece o evento CollectionChanged e implementa o próprio INotifyPropertyChanged. De acordo com MSDN o Evento aumentará, “[..]quando um item é adicionado, removido, alterado, movido, ou toda a lista é atualizada”.
Você, no entanto, rapidamente perceberá que com o .NET 4.5.2 e anteriores, o ObservableCollection não gerará um evento CollectionChanged se uma propriedade de um elemento na coleção for alterada conforme discutido aqui.

Seguindo esta solução, podemos simplesmente implementar nossa própria TrulyObservableCollection<T> sem a restrição INotifyPropertyChanged para T ter tudo o que precisamos e expor se T implementa INotifyPropertyChanged ou não :

/*
 * Original Class by Simon @StackOverflow http://stackoverflow.com/a/5256827/3766034
 * Removal of the INPC-Constraint by Jirajha @StackOverflow 
 * according to to suggestion of nikeee @StackOverflow http://stackoverflow.com/a/10718451/3766034
 */
public sealed class TrulyObservableCollection<T> : ObservableCollection<T>
{
    private readonly bool _inpcHookup;
    public bool NotifyPropertyChangedHookup => _inpcHookup;

    public TrulyObservableCollection()
    {
        CollectionChanged += TrulyObservableCollectionChanged;
        _inpcHookup = typeof(INotifyPropertyChanged).GetTypeInfo().IsAssignableFrom(typeof(T));
    }
    public TrulyObservableCollection(IEnumerable<T> items) : this()
    {
        foreach (var item in items)
        {
            this.Add(item);
        }
    }

    private void TrulyObservableCollectionChanged(object sender, NotifyCollectionChangedEventArgs e)
    {
        if (NotifyPropertyChangedHookup && e.NewItems != null && e.NewItems.Count > 0)
        {
            foreach (INotifyPropertyChanged item in e.NewItems)
            {
                item.PropertyChanged += ItemPropertyChanged;
            }
        }
        if (NotifyPropertyChangedHookup && e.OldItems != null && e.OldItems.Count > 0)
        {
            foreach (INotifyPropertyChanged item in e.OldItems)
            {
                item.PropertyChanged -= ItemPropertyChanged;
            }
        }
    }
    private void ItemPropertyChanged(object sender, PropertyChangedEventArgs e)
    {
        var args = new NotifyCollectionChangedEventArgs(NotifyCollectionChangedAction.Replace, sender, sender, IndexOf((T)sender));
        OnCollectionChanged(args);
    }
}

e defina nossa propriedade Users como TrulyObservableCollection<User> em nosso ViewModel

private TrulyObservableCollection<string> _users;
public TrulyObservableCollection<string> Users
{
    get { return _users; }
    set
    {
        if (_users == value) return;
        _users = value;
        NotifyPropertyChanged();
    }
}

Nossa interface do usuário agora será notificada quando uma propriedade INPC de um elemento dentro da coleção for alterada sem a necessidade de recriar cada Binding.
