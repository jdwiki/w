Les tableaux typés ont été spécifiés à l’origine par le projet d’un éditeur de Khronos, puis normalisés dans ECMAScript 6 [§24](http://www. ecma-international.org/ecma-262/6.0/#sec-structured-data) et §22.2 .

Les blobs sont spécifiés par le brouillon de travail de l’API de fichier W3C.

Obtenir une représentation binaire d’un fichier image #

Cet exemple est inspiré de cette question.

Nous supposerons que vous savez comment [charger un fichier à l’aide de l’API File] (http://www.html5rocks.com/en/tutorials/file/dndfiles/).

// preliminary code to handle getting local file and finally printing to console
// the results of our function ArrayBufferToBinary().
var file = // get handle to local file.
var reader = new FileReader();
reader.onload = function(event) {
    var data = event.target.result;
    console.log(ArrayBufferToBinary(data));
};
reader.readAsArrayBuffer(file); //gets an ArrayBuffer of the file

Maintenant, nous effectuons la conversion réelle des données du fichier en 1 et 0 à l’aide d’un DataView :

function ArrayBufferToBinary(buffer) {
   // Convert an array buffer to a string bit-representation: 0 1 1 0 0 0...
   var dataView = new DataView(buffer);
   var response = "", offset = (8/8); 
   for(var i = 0; i < dataView.byteLength; i += offset) {
       response += dataView.getInt8(i).toString(2); 
   }
   return response;
}

DataViews vous permet de lire/écrire des données numériques ; getInt8 convertit les données de la position de l’octet - ici 0, la valeur transmise - dans le ArrayBuffer en représentation d’entier 8 bits signé, et toString(2) convertit l’entier 8 bits en représentation binaire format (c’est-à-dire une chaîne de 1 et de 0).

Les fichiers sont enregistrés sous forme d’octets. La valeur de décalage “magique” est obtenue en notant que nous prenons des fichiers stockés sous forme d’octets, c’est-à-dire sous forme d’entiers 8 bits, et que nous les lisons dans une représentation entière 8 bits. Si nous essayions de lire nos fichiers enregistrés par octets (c’est-à-dire 8 bits) en entiers 32 bits, nous noterions que 32/8 = 4 est le nombre d’espaces d’octets, qui est notre valeur de décalage d’octets.

Pour cette tâche, les DataView sont exagérés. Ils sont généralement utilisés dans les cas où l’endianité ou l’hétérogénéité des données sont rencontrées (par exemple, lors de la lecture de fichiers PDF, qui ont des en-têtes codés dans différentes bases et nous aimerions extraire cette valeur de manière significative). Parce que nous voulons juste une représentation textuelle, nous ne nous soucions pas de l’hétérogénéité car il n’est jamais nécessaire de

Une solution bien meilleure - et plus courte - peut être trouvée en utilisant un tableau typé UInt8Array, qui traite le ArrayBuffer entier comme composé d’entiers 8 bits non signés :

function ArrayBufferToBinary(buffer) {
    var uint8 = new Uint8Array(buffer);
    return uint8.reduce((binary, uint8) => binary + uint8.toString(2), "");
}

Conversion entre Blobs et ArrayBuffers #

JavaScript a deux manières principales de représenter les données binaires dans le navigateur. ArrayBuffers/TypedArrays contiennent des données binaires modifiables (bien que toujours de longueur fixe) que vous pouvez manipuler directement. Les objets blob contiennent des données binaires immuables accessibles uniquement via l’interface de fichier asynchrone.

Convertir un Blob en un ArrayBuffer (asynchrone) #

var blob = new Blob(["\x01\x02\x03\x04"]),
    fileReader = new FileReader(),
    array;

fileReader.onload = function() {
    array = this.result;
    console.log("Array contains", array.byteLength, "bytes.");
};

fileReader.readAsArrayBuffer(blob);

Convertir un Blob en un ArrayBuffer en utilisant une Promise (asynchrone) #

var blob = new Blob(["\x01\x02\x03\x04"]);

var arrayPromise = new Promise(function(resolve) {
    var reader = new FileReader();

    reader.onloadend = function() {
        resolve(reader.result);
    };

    reader.readAsArrayBuffer(blob);
});

arrayPromise.then(function(array) {
    console.log("Array contains", array.byteLength, "bytes.");
});

Convertit un ArrayBuffer ou un tableau typé en un Blob #

var array = new Uint8Array([0x04, 0x06, 0x07, 0x08]);

var blob = new Blob([array]);

Manipulation de ArrayBuffers avec DataViews #

Les DataViews fournissent des méthodes pour lire et écrire des valeurs individuelles à partir d’un ArrayBuffer, au lieu de voir l’ensemble comme un tableau d’un seul type. Ici, nous définissons deux octets individuellement puis nous les interprétons ensemble comme un entier non signé de 16 bits, d’abord gros-boutien puis petit-boutiste.

var buffer = new ArrayBuffer(2);
var view = new DataView(buffer);

view.setUint8(0, 0xFF);
view.setUint8(1, 0x01);

console.log(view.getUint16(0, false)); // 65281
console.log(view.getUint16(0, true));  // 511

Création d’un TypedArray à partir d’une chaîne Base64 #

var data = 
   'iVBORw0KGgoAAAANSUhEUgAAAAUAAAAFCAYAAACN' +
   'byblAAAAHElEQVQI12P4//8/w38GIAXDIBKE0DHx' +
   'gljNBAAO9TXL0Y4OHwAAAABJRU5ErkJggg==';

var characters = atob(data);

var array = new Uint8Array(characters.length);

for (var i = 0; i < characters.length; i++) {
  array[i] = characters.charCodeAt(i);
}

Utilisation de TypedArrays #

Les TypedArrays sont un ensemble de types fournissant différentes vues dans des ArrayBuffers binaires mutables de longueur fixe. Pour la plupart, ils agissent comme Arrays qui contraignent toutes les valeurs attribuées à un type numérique donné. Vous pouvez passer une instance ArrayBuffer à un constructeur TypedArray pour créer une nouvelle vue de ses données.

var buffer = new ArrayBuffer(8);
var byteView = new Uint8Array(buffer);
var floatView = new Float64Array(buffer);

console.log(byteView);  // [0, 0, 0, 0, 0, 0, 0, 0]
console.log(floatView); // [0]
byteView[0] = 0x01;
byteView[1] = 0x02;
byteView[2] = 0x04;
byteView[3] = 0x08;
console.log(floatView); // [6.64421383e-316]

Les ArrayBuffers peuvent être copiés à l’aide de la méthode .slice(...), soit directement, soit via une vue TypedArray.

var byteView2 = byteView.slice();
var floatView2 = new Float64Array(byteView2.buffer);
byteView2[6] = 0xFF;
console.log(floatView);  // [6.64421383e-316]
console.log(floatView2); // [7.06327456e-304]

Itérer dans un arrayBuffer #

Pour parcourir facilement un arrayBuffer, vous pouvez créer un itérateur simple qui implémente les méthodes “DataView” sous le capot :

var ArrayBufferCursor = function() {
  var ArrayBufferCursor = function(arrayBuffer) {
    this.dataview = new DataView(arrayBuffer, 0);
    this.size = arrayBuffer.byteLength;
    this.index = 0;
  }

  ArrayBufferCursor.prototype.next = function(type) {
    switch(type) {
      case 'Uint8':
        var result = this.dataview.getUint8(this.index);
        this.index += 1;
        return result;
      case 'Int16':
        var result = this.dataview.getInt16(this.index, true);
        this.index += 2;
        return result;
      case 'Uint16':
        var result = this.dataview.getUint16(this.index, true);
        this.index += 2;
        return result;
      case 'Int32':
        var result = this.dataview.getInt32(this.index, true);
        this.index += 4;
        return result;
      case 'Uint32':
        var result = this.dataview.getUint32(this.index, true);
        this.index += 4;
        return result;
      case 'Float':
      case 'Float32':
        var result = this.dataview.getFloat32(this.index, true);
        this.index += 4;
        return result;
      case 'Double':
      case 'Float64':
        var result = this.dataview.getFloat64(this.index, true);
        this.index += 8;
        return result;
      default:
        throw new Error("Unknown datatype");
    }
  };

  ArrayBufferCursor.prototype.hasNext = function() {
    return this.index < this.size;
  }

  return ArrayBufferCursor;
});

Vous pouvez ensuite créer un itérateur comme celui-ci :

var cursor = new ArrayBufferCursor(arrayBuffer);

Vous pouvez utiliser le hasNext pour vérifier s’il y a encore des éléments

for(;cursor.hasNext();) {
    // There's still items to process
}

Vous pouvez utiliser la méthode next pour prendre la valeur suivante :

var nextValue = cursor.next('Float');

Avec un tel itérateur, écrire votre propre analyseur pour traiter les données binaires devient assez facile.
