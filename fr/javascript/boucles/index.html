Syntaxe #

  • for (initialisation ; condition ; expression_finale) { }
  • pour (clé dans objet) { }
  • pour (variable de itérable) { }
  • tant que (condition) { }
  • faire { } tant que (condition)
  • pour chaque (variable dans objet) { } // ECMAScript pour XML

Les boucles en JavaScript aident généralement à résoudre des problèmes impliquant la répétition d’un code spécifique x fois. Disons que vous devez enregistrer un message 5 fois. Vous pourriez faire ceci :

console.log("a message");
console.log("a message");
console.log("a message");
console.log("a message");
console.log("a message");

Mais c’est juste chronophage et un peu ridicule. Et si vous aviez besoin de consigner plus de 300 messages ? Vous devez remplacer le code par une boucle “for” traditionnelle :

for(var i = 0; i < 5; i++){
    console.log("a message");
}

Boucles “pour” standard #

Utilisation standard #

for (var i = 0; i < 100; i++) {
    console.log(i);
}

Production attendue:

0 1 …
99

Déclarations multiples #

Couramment utilisé pour mettre en cache la longueur d’un tableau.

var array = ['a', 'b', 'c'];
for (var i = 0; i < array.length; i++) {
    console.log(array[i]);
}

Production attendue:

‘un’ ‘b’ ‘c’

Modification de l’incrément #

for (var i = 0; i < 100; i += 2 /* Can also be: i = i + 2 */) {
    console.log(i);
}

Production attendue:

0 2 4 …
98

Boucle décrémentée #

for (var i = 100; i >=0; i--) {
    console.log(i);
}

Production attendue:

100 99 98 …
0

Boucle “pour … de” #

const iterable = [0, 1, 2];
for (let i of iterable) {
    console.log(i);
}

Production attendue:

0 1 2

Les avantages de la boucle for…of sont :

  • Il s’agit de la syntaxe la plus concise et la plus directe à ce jour pour parcourir les éléments du tableau
  • Il évite tous les pièges du for…in
  • Contrairement à forEach(), cela fonctionne avec break, continue et return

Prise en charge de for…of dans d’autres collections #

Chaînes #

for…of traitera une chaîne comme une séquence de caractères Unicode :

const string = "abc";
for (let chr of string) {
  console.log(chr);
}

Production attendue:

un b c

Ensembles #

for…of fonctionne sur Set objects.

Noter:

  • Un objet Set éliminera les doublons.

  • Veuillez vérifier cette référence pour la prise en charge du navigateur Set().

    const names = [‘bob’, ‘alejandro’, ‘zandra’, ‘anna’, ‘bob’];

    const uniqueNames = new Set(names);

    for (let name of uniqueNames) { console.log(name); }

Production attendue:

Bob alejandro zandra Anna

Plans #

Vous pouvez également utiliser des boucles for…of pour itérer sur Maps. Cela fonctionne de la même manière que les tableaux et les ensembles, sauf que la variable d’itération stocke à la fois une clé et une valeur.

const map = new Map()
  .set('abc', 1)
  .set('def', 2)

for (const iteration of map) {
  console.log(iteration) //will log ['abc', 1] and then ['def', 2]
}

Vous pouvez utiliser [assignation déstructurante](https://www.wikiod.com/fr/javascript/mission-de-destructuration#Déstructuration des tableaux) pour capturer la clé et la valeur séparément :

const map = new Map()
  .set('abc', 1)
  .set('def', 2)

for (const [key, value] of map) {
  console.log(key + ' is mapped to ' + value)
}
/*Logs:
  abc is mapped to 1
  def is mapped to 2
*/

Objets #

les boucles for…of ne fonctionnent pas directement sur les objets ordinaires ; mais, il est possible de parcourir les propriétés d’un objet en passant à une boucle for…in ou en utilisant [Object.keys()](https://www.wikiod.com/fr/javascript/objets 736/object-keys#t=201608201852433521361):

const someObject = { name: 'Mike' };

for (let key of Object.keys(someObject)) {
  console.log(key + ": " + someObject[key]);
}

Production attendue:

nom : Mike

Boucle “pour … dans” #

Attention
for…in est destiné à itérer sur les clés d’objet, pas sur les index de tableau. Il est généralement déconseillé de l’utiliser pour parcourir un tableau. Il inclut également les propriétés du prototype, il peut donc être nécessaire de vérifier si la clé se trouve dans l’objet à l’aide de hasOwnProperty. Si des attributs de l’objet sont définis par la méthode defineProperty/defineProperties et définissez le paramètre enumerable: false, ces attributs seront inaccessibles.

var object = {"a":"foo", "b":"bar", "c":"baz"};
// `a` is inaccessible
Object.defineProperty(object , 'a', {
        enumerable: false,
});
for (var key in object) {
    if (object.hasOwnProperty(key)) {
      console.log('object.' + key + ', ' + object[key]);
    }
}

Production attendue:

objet.b, barre objet.c, baz

Boucles “while” #

Boucle While standard #

Une boucle while standard s’exécutera jusqu’à ce que la condition donnée soit fausse :

var i = 0;
while (i < 100) {
    console.log(i);
    i++;
}

Production attendue:

0 1 …
99

Boucle décrémentée #

var i = 100;
while (i > 0) {
    console.log(i);
    i--; /* equivalent to i=i-1 */
}

Production attendue:

100 99 98 …
1

Faire…tout en boucle #

Une boucle do…while s’exécutera toujours au moins une fois, que la condition soit vraie ou fausse :

var i = 101;
do {
    console.log(i);
} while (i < 100);

Production attendue:

101

“continuer” une boucle #

Poursuivre une boucle “for” #

Lorsque vous placez le mot-clé continue dans une boucle for, l’exécution saute à l’expression de mise à jour (i++ dans l’exemple) :

for (var i = 0; i < 3; i++) {
    if (i === 1) {
        continue;
    }
    console.log(i);
}

Production attendue:

0 2

Continuer une boucle While #

Lorsque vous “continuez” dans une boucle while, l’exécution saute à la condition (‘i < 3’ dans l’exemple) :

var i = 0;
while (i < 3) {
    if (i === 1) {
        i = 2;
        continue;
    }
    console.log(i);
    i++;
}

Production attendue:

0 2

Boucle “faire … tant que” #

var availableName;
do {
    availableName = getRandomName();
} while (isNameUsed(name));

Une boucle do while est garantie de s’exécuter au moins une fois car sa condition n’est vérifiée qu’à la fin d’une itération. Une boucle “while” traditionnelle peut s’exécuter zéro ou plusieurs fois car sa condition est vérifiée au début d’une itération.

Casser des boucles imbriquées spécifiques #

Nous pouvons nommer nos boucles et casser celle spécifique si nécessaire.

outerloop:
for (var i = 0;i<3;i++){
    innerloup:
    for (var j = 0;j <3; j++){
        console.log(i);
        console.log(j);
        if (j == 1){
            break outerloop;    
        }
    }
}

Production:

0
0
0
1

Casser et continuer les étiquettes #

Les instructions break et continue peuvent être suivies d’une étiquette facultative qui fonctionne comme une sorte d’instruction goto, reprend l’exécution à partir de la position référencée par l’étiquette

for(var i = 0; i < 5; i++){
  nextLoop2Iteration:
  for(var j = 0; j < 5; j++){
    if(i == j) break nextLoop2Iteration;
    console.log(i, j);
  }
}

i=0 j=0 saute le reste des j valeurs 1 0 i=1 j=1 saute le reste des j valeurs 2 0 2 1 i=2 j=2 ignore le reste des j valeurs 3 0 3 1 3 2 i=3 j=3 saute le reste des j valeurs 4 0 4 1 4 2 4 3 i=4 j=4 n’enregistre pas et les boucles sont faites

“Sortir” d’une boucle #

Sortir d’une boucle while #

var i = 0;
while(true) {
    i++;
    if(i === 42) {
        break;
    }
}
console.log(i);

Production attendue:

42

Sortir d’une boucle for #

var i;
for(i = 0; i < 100; i++) {
    if(i === 42) {
        break;
    }
}
console.log(i);

Production attendue:

42
