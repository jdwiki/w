Syntaxe #

  • “chaîne littérale”
  • ‘chaîne littérale’
  • “chaîne littérale avec ‘guillemets incompatibles’” // pas d’erreur ; les devis sont différents.
  • “chaîne littérale avec "guillemets échappés"” // pas d’erreur ; les guillemets sont échappés.
  • `chaîne de modèle ${expression}`
  • String(“a b c”) // renvoie une chaîne lorsqu’elle est appelée dans un contexte non constructeur
  • new String(“a b c”) // l’objet String, pas la primitive string

Informations de base et concaténation de chaînes #

Les chaînes en JavaScript peuvent être entourées de guillemets simples 'hello', de guillemets doubles "Hello" et (à partir de ES2015, ES6) dans des littéraux de modèle (backticks) `hello`.

var hello  = "Hello";
var world  = 'world';
var helloW = `Hello World`;              // ES2015 / ES6

Les chaînes peuvent être créées à partir d’autres types à l’aide de la fonction String().

var intString = String(32); // "32"
var booleanString = String(true); // "true"
var nullString = String(null); // "null"

Ou, toString() peut être utilisé pour convertir des nombres, des booléens ou des objets en chaînes.

var intString = (5232).toString(); // "5232"
var booleanString = (false).toString(); // "false"
var objString = ({}).toString(); // "[object Object]"

Les chaînes peuvent également être créées en utilisant la méthode String.fromCharCode.

String.fromCharCode(104,101,108,108,111) //"hello"

La création d’un objet String à l’aide du mot clé new est autorisée, mais n’est pas recommandée car il se comporte comme des objets contrairement aux chaînes primitives.

var objectString = new String("Yes, I am a String object");
typeof objectString;//"object"
typeof objectString.valueOf();//"string"

chaînes de concaténation #

La concaténation de chaînes peut être effectuée avec l’opérateur de concaténation + ou avec la méthode intégrée concat() sur le prototype d’objet String.

var foo = "Foo";
var bar = "Bar";
console.log(foo + bar);        // => "FooBar"
console.log(foo + " " + bar);  // => "Foo Bar"

foo.concat(bar)            // => "FooBar"
"a".concat("b", " ", "d")  // => "ab d"

Les chaînes peuvent être concaténées avec des variables non-chaîne mais convertiront les types des variables non-chaîne en chaînes.

var string = "string";
var number = 32;
var boolean = true;

console.log(string + number + boolean); // "string32true"

Modèles de chaînes #

Les chaînes peuvent être créées à l’aide de modèles littéraux (backticks) `hello`.

var greeting = `Hello`;

Avec les littéraux de modèle, vous pouvez effectuer une interpolation de chaîne en utilisant ${variable} à l’intérieur des littéraux de modèle :

var place = `World`;
var greet = `Hello ${place}!`

console.log(greet); // "Hello World!"

Vous pouvez utiliser String.raw pour obtenir des barres obliques inverses dans la chaîne sans modification.

`a\\b` // =  a\b
String.raw`a\\b` // = a\\b

Chaîne inversée #

La manière la plus “populaire” d’inverser une chaîne en JavaScript est le fragment de code suivant, qui est assez courant :

function reverseString(str) {
    return str.split('').reverse().join('');
}

reverseString('string');    // "gnirts"

Cependant, cela ne fonctionnera que tant que la chaîne inversée ne contient pas de paires de substitution. Les symboles astraux, c’est-à-dire les caractères en dehors du plan multilingue de base, peuvent être représentés par deux unités de code et conduiront cette technique naïve à produire des résultats erronés. De plus, les caractères avec des marques de combinaison (par exemple, le tréma) apparaîtront sur le caractère “suivant” logique au lieu de celui d’origine avec lequel il a été combiné.

'𝌆■.'.split('').reverse().join(''); //fails

Bien que la méthode fonctionne correctement pour la plupart des langages, un algorithme de respect de l’encodage vraiment précis pour l’inversion de chaîne est légèrement plus impliqué. Une de ces implémentations est une petite bibliothèque appelée [Esrever][1], qui utilise des expressions régulières pour faire correspondre les marques de combinaison et les paires de substitution afin d’effectuer parfaitement l’inversion.

Explication #

Rubrique Explication Résultat
str La chaîne d’entrée "chaîne"
String.prototype.split( délimiteur ) Divise la chaîne str en un tableau. Le paramètre "" signifie diviser entre chaque caractère. ["s","t","r","je","n","g"]
Array.prototype.reverse() Renvoie le tableau de la chaîne fractionnée avec ses éléments dans l’ordre inverse. ["g","n","i","r","t","s"]
Array.prototype.join( deliminator ) Joint les éléments du tableau dans une chaîne. Le paramètre "" signifie un délimiteur vide (c’est-à-dire que les éléments du tableau sont placés les uns à côté des autres). "gnirts"

Utilisation de l’opérateur de diffusion

function reverseString(str) {
    return [...String(str)].reverse().join('');    
}

console.log(reverseString('stackoverflow'));  // "wolfrevokcats"
console.log(reverseString(1337));             // "7331"
console.log(reverseString([1, 2, 3]));        // "3,2,1"

Fonction reverse() personnalisée

function reverse(string) {
    var strRev = "";
    for (var i = string.length - 1; i >= 0; i--) {
        strRev += string[i];
    }
    return strRev; 
}

reverse("zebra");  // "arbez"

[1] : https://github.com/mathiasbynens/esrever “Esrever”

Comparer les chaînes de manière lexicographique #

Pour comparer les chaînes par ordre alphabétique, utilisez localeCompare(). Cela renvoie une valeur négative si la chaîne de référence est lexicographiquement (par ordre alphabétique) avant la chaîne comparée (le paramètre), une valeur positive si elle vient après et une valeur de “0” si elles sont égales.

var a = "hello";
var b = "world";

console.log(a.localeCompare(b)); // -1

Les opérateurs > et < peuvent également être utilisés pour comparer des chaînes de manière lexicographique, mais ils ne peuvent pas renvoyer une valeur de zéro (ceci peut être testé avec l’opérateur d’égalité ==). Par conséquent, une forme de la fonction localeCompare() peut être écrite comme suit :

function strcmp(a, b) {
    if(a === b) {
        return 0;
    }

    if (a > b) {
        return 1;
    }

    return -1;
}

console.log(strcmp("hello", "world")); // -1
console.log(strcmp("hello", "hello")); //  0
console.log(strcmp("world", "hello")); //  1

Ceci est particulièrement utile lors de l’utilisation d’une fonction de tri qui compare en fonction du signe de la valeur de retour (telle que sort).

var arr = ["bananas", "cranberries", "apples"];
arr.sort(function(a, b) {
    return a.localeCompare(b);
});
console.log(arr); // [ "apples", "bananas", "cranberries" ]

Caractère d’accès à l’index dans la chaîne #

Utilisez charAt() pour obtenir un caractère à l’index spécifié dans la chaîne.

var string = "Hello, World!";
console.log( string.charAt(4) ); // "o"

Sinon, comme les chaînes peuvent être traitées comme des tableaux, utilisez l’index via la notation entre crochets.

var string = "Hello, World!";
console.log( string[4] ); // "o"

Pour obtenir le code de caractère du caractère à un index spécifié, utilisez charCodeAt() .

var string = "Hello, World!";
console.log( string.charCodeAt(4) ); // 111

Notez que ces méthodes sont toutes des méthodes getter (renvoyent une valeur). Les chaînes en JavaScript sont immuables. En d’autres termes, aucun d’entre eux ne peut être utilisé pour définir un caractère à une position dans la chaîne.

Échapper aux guillemets #

Si votre chaîne est entourée (c’est-à-dire) entre guillemets simples, vous devez échapper le guillemet littéral interne avec * antislash * \

var text = 'L\'albero means tree in Italian';
console.log( text ); \\ "L'albero means tree in Italian"

Idem pour les guillemets doubles :

var text = "I feel \"high\"";

Une attention particulière doit être accordée aux guillemets d’échappement si vous stockez des représentations HTML dans une chaîne, car les chaînes HTML utilisent largement les guillemets, c’est-à-dire dans les attributs :

var content = "<p class=\"special\">Hello World!</p>";        // valid String
var hello   = '<p class="special">I\'d like to say "Hi"</p>'; // valid String

Les guillemets dans les chaînes HTML peuvent également être représentés en utilisant &apos; (ou &#39;) comme guillemet simple et &quot; (ou &#34;) comme guillemets doubles.

var hi    = "<p class='special'>I'd like to say &quot;Hi&quot;</p>"; // valid String
var hello = '<p class="special">I&apos;d like to say "Hi"</p>';      // valid String

Remarque : L’utilisation de &apos; et &quot; n’écrasera pas les guillemets doubles que les navigateurs peuvent placer automatiquement sur les guillemets d’attribut. Par exemple, <p class=special> étant transformé en <p class="special">, l’utilisation de &quot; peut conduire à <p class=""special""> où \" sera être <p class="special">.

Si une chaîne a ' et ", vous pouvez envisager d’utiliser des littéraux de modèle (également appelés chaînes de modèle dans les éditions précédentes d’ES6), qui ne nécessitent pas que vous échappiez ' et ". Ceux-ci utilisent des backticks (`) au lieu de guillemets simples ou doubles.

var x = `"Escaping " and ' can become very annoying`;

Couper les espaces blancs #

Pour supprimer les espaces blancs des bords d’une chaîne, utilisez String.prototype.trim :

"    some whitespaced string  ".trim();  // "some whitespaced string"

De nombreux moteurs JavaScript, mais pas Internet Explorer, ont implémenté trimLeft non standard et les méthodes “trimRight”. Il existe une proposition, actuellement à l’étape 1 du processus, pour les méthodes standardisées trimStart et trimEnd, alias trimLeft et trimRight pour la compatibilité.

// Stage 1 proposal
"    this is me    ".trimStart();  // "this is me    "
"    this is me    ".trimEnd();  // "    this is me"

// Non-standard methods, but currently implemented by most engines
"    this is me    ".trimLeft();  // "this is me    "
"    this is me    ".trimRight();  // "    this is me"

Fractionner une chaîne dans un tableau #

Utilisez .split pour passer des chaînes à un tableau des sous-chaînes scindées :

var s = "one, two, three, four, five"
s.split(", ");  // ["one", "two", "three", "four", "five"]

Utilisez la méthode de tableau .join pour revenir à une chaîne :

s.split(", ").join("--");  // "one--two--three--four--five"

Les chaînes sont Unicode #

Toutes les chaînes JavaScript sont Unicode !

Il n’y a pas d’octets bruts ou de chaînes binaires en JavaScript. Pour gérer efficacement les données binaires, utilisez [Typed Arrays](https://www.wikiod.com/fr/javascript/donnees-binaires#Utilisation de TypedArrays).

Détection d’une chaîne #

Pour détecter si un paramètre est une chaîne primitive, utilisez typeof :

var aString = "my string";
var anInt = 5;
var anObj = {};
typeof aString === "string";   // true
typeof anInt === "string";     // false
typeof anObj === "string";     // false

Si jamais vous avez un objet String, via new String("somestr"), alors ce qui précède ne fonctionnera pas. Dans cet exemple, nous pouvons utiliser instanceof :

var aStringObj = new String("my string");
aStringObj instanceof String;    // true

Pour couvrir les deux instances, nous pouvons écrire une fonction d’assistance simple :

var isString = function(value) {
    return typeof value === "string" || value instanceof String;
};

var aString = "Primitive String";
var aStringObj = new String("String Object");
isString(aString); // true
isString(aStringObj); // true
isString({}); // false
isString(5); // false

Ou nous pouvons utiliser la fonction toString de Object. Cela peut être utile si nous devons également vérifier d’autres types, par exemple dans une instruction switch, car cette méthode prend également en charge d’autres types de données, tout comme typeof.

var pString = "Primitive String";
var oString = new String("Object Form of String");
Object.prototype.toString.call(pString);//"[object String]"
Object.prototype.toString.call(oString);//"[object String]"

Une solution plus robuste consiste à ne pas * détecter * une chaîne du tout, mais plutôt à vérifier uniquement la fonctionnalité requise. Par exemple:

var aString = "Primitive String";

// Explicit check for the String substring prototype method
if(aString.substring === String.prototype.substring) {

Sous-chaînes avec tranche #

Utilisez .slice() pour extraire les sous-chaînes à partir de deux indices :

var s = "0123456789abcdefg";
s.slice(0, 5);  // "01234"
s.slice(5, 6);  // "5"

Étant donné un index, il prendra de cet index à la fin de la chaîne :

s.slice(10);    // "abcdefg"

Fonctions de recherche et de remplacement de chaînes #

Pour rechercher une chaîne à l’intérieur d’une chaîne, il existe plusieurs fonctions :

indexOf( searchString ) et [lastIndexOf( searchString )](https ://developer.mozilla.org/en-US/docs/Web/JavaScript/Reference/Global_Objects/String/lastIndexOf) #

indexOf() renverra l’index de la première occurrence de searchString dans la chaîne. Si searchString n’est pas trouvé, alors -1 est renvoyé.

var string = "Hello, World!";
console.log( string.indexOf("o") ); // 4
console.log( string.indexOf("foo") ); // -1

De même, lastIndexOf() renverra l’index de la dernière occurrence de searchstring ou -1 s’il n’est pas trouvé.

var string = "Hello, World!";
console.log( string.lastIndexOf("o") );   // 8
console.log( string.lastIndexOf("foo") ); // -1

includes( searchString, start ) #

includes() renverra un booléen qui indique si searchString existe dans la chaîne, à partir de l’index start (par défaut à 0). C’est mieux que indexOf() si vous avez simplement besoin de tester l’existence d’une sous-chaîne.

var string = "Hello, World!";
console.log( string.includes("Hello") ); // true
console.log( string.includes("foo") );   // false

replace( regexp|substring, replacement|replaceFunction ) #

replace() renverra une chaîne contenant toutes les occurrences de sous-chaînes correspondant à RegExp regexp ou chaîne substring avec une chaîne replacement ou la valeur renvoyée de replaceFunction.

Notez que cela ne modifie pas la chaîne en place, mais renvoie la chaîne avec des remplacements.

var string = "Hello, World!";
string = string.replace( "Hello", "Bye" );
console.log( string ); // "Bye, World!"

string = string.replace( /W.{3}d/g, "Universe" );
console.log( string ); // "Bye, Universe!"

replaceFunction peut être utilisé pour des remplacements conditionnels d’objets d’expression régulière (c’est-à-dire avec une utilisation avec regexp). Les paramètres sont dans l’ordre suivant :

Paramètre Signification
correspondre la sous-chaîne qui correspond à l’expression régulière entièreg
g1, g2, g3, … les groupes correspondants dans l’expression régulière
décalage le décalage de la correspondance dans la chaîne entière
chaîne la chaîne entière

Notez que tous les paramètres sont facultatifs.

var string = "heLlo, woRlD!";
string = string.replace( /([a-zA-Z])([a-zA-Z]+)/g, function(match, g1, g2) {
    return g1.toUpperCase() + g2.toLowerCase();
}); 
console.log( string ); // "Hello, World!"

Représentations sous forme de chaîne de nombres #

JavaScript a une conversion native de Number à sa String representation pour toute base de 2 à 36.

La représentation la plus courante après decimal (base 10) est hexadecimal (base 16), mais le contenu de cette section fonctionne pour toutes les bases de la plage.

Afin de convertir un Number de décimal (base 10) en hexadécimal (base 16) Représentation String, la méthode toString peut être utilisée avec radix 16.

// base 10 Number
var b10 = 12;

// base 16 String representation
var b16 = b10.toString(16); // "c"

Si le nombre représenté est un entier, l’opération inverse peut être effectuée avec parseInt et la radix 16 à nouveau

// base 16 String representation
var b16 = 'c';

// base 10 Number
var b10 = parseInt(b16, 16); // 12

Pour convertir un nombre arbitraire (c’est-à-dire non entier) de sa représentation sous forme de chaîne en un Nombre, l’opération doit être divisée en deux parties ; la partie entière et la partie fractionnaire.

let b16 = '3.243f3e0370cdc';
// Split into integer and fraction parts
let [i16, f16] = b16.split('.');

// Calculate base 10 integer part
let i10 = parseInt(i16, 16); // 3

// Calculate the base 10 fraction part
let f10 = parseInt(f16, 16) / Math.pow(16, f16.length); // 0.14158999999999988

// Put the base 10 parts together to find the Number
let b10 = i10 + f10; // 3.14159

Remarque 1 : Soyez prudent car de petites erreurs peuvent se produire dans le résultat en raison de différences dans ce qui peut être représenté dans différentes bases. Il peut être souhaitable d’effectuer une sorte d’arrondi par la suite. Remarque 2 : Les représentations très longues de nombres peuvent également entraîner des erreurs en raison de la précision et des valeurs maximales de Numbers de l’environnement dans lequel les conversions ont lieu.

Code de caractère #

La méthode charCodeAt récupère le code de caractère Unicode d’un seul caractère :

var charCode = "µ".charCodeAt(); // The character code of the letter µ is 181

Pour obtenir le code de caractère d’un caractère dans une chaîne, la position de base 0 du caractère est transmise en tant que paramètre à charCodeAt :

var charCode = "ABCDE".charCodeAt(3); // The character code of "D" is 68

Certains symboles Unicode ne tiennent pas dans un seul caractère et nécessitent à la place deux paires de substitution UTF-16 pour être encodés. C’est le cas des codes de caractères au-delà de 216 - 1 ou 63553. Ces codes de caractères étendus ou valeurs de code point peuvent être récupérés avec codePointAt :

// The Grinning Face Emoji has code point 128512 or 0x1F600
var codePoint = "😀".codePointAt();

Chaîne en majuscule #

String.prototype.toUpperCase() :

console.log('qwerty'.toUpperCase()); // 'QWERTY'

Chaîne en minuscules #

String.prototype.toLowerCase()

console.log('QWERTY'.toLowerCase()); // 'qwerty'

Trouver l’index d’une sous-chaîne à l’intérieur d’une chaîne #

La méthode .indexOf renvoie l’index d’une sous-chaîne à l’intérieur d’une autre chaîne (si elle existe, ou -1 sinon)

'Hellow World'.indexOf('Wor');    // 7

.indexOf accepte également un argument numérique supplémentaire qui indique sur quel index la fonction doit-elle commencer à chercher

"harr dee harr dee harr".indexOf("dee", 10); // 14

Vous devez noter que .indexOf est sensible à la casse

 'Hellow World'.indexOf('WOR');    // -1

Répéter une chaîne #

Cela peut être fait en utilisant la méthode [.repeat()][1] :

"abc".repeat(3);  // Returns "abcabcabc"
"abc".repeat(0);  // Returns ""
"abc".repeat(-1); // Throws a RangeError

Dans le cas général, cela devrait être fait en utilisant un polyfill correct pour la méthode ES6 [String.prototype.repeat()][1]. Sinon, l’idiome new Array(n + 1).join(myString) peut répéter n fois la chaîne myString :

var myString = "abc";
var n = 3;

new Array(n + 1).join(myString);  // Returns "abcabcabc"

[1] : http://www.ecma-international.org/ecma-262/6.0/#sec-string.prototype.repeat
