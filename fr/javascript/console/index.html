La console de débogage d’un navigateur ou [console Web][1] est généralement utilisée par les développeurs pour identifier les erreurs, comprendre le flux d’exécution, consigner les données et à de nombreuses autres fins lors de l’exécution. Ces informations sont accessibles via l’objet [console][2].

[1] : https://developer.mozilla.org/en-US/docs/Tools/Web_Console [2] : https://developer.mozilla.org/en-US/docs/Web/API/Console

Syntaxe #

  • void console.log(obj1 [, obj2, …, objN]);
  • void console.log(msg [, sub1, …, subN]);

Paramètres #

Paramètre Descriptif
obj1 … objN Une liste d’objets JavaScript dont les représentations de chaîne sont sorties dans la console
message Une chaîne JavaScript contenant zéro ou plusieurs chaînes de substitution.
sous1 … sousN Objets JavaScript avec lesquels remplacer les chaînes de substitution dans msg.

Les informations affichées par une [console de débogage/web][1] sont rendues disponibles via les multiples [méthodes de l’objet Javascript console][2] consultables via console.dir(console). Outre la propriété console.memory, les méthodes affichées sont généralement les suivantes (extraites de la sortie de Chromium) :

  • [affirmer][3]
  • [clair][4]
  • [compter][5]
  • [débogage][6]
  • [rép][7]
  • [dirxml][8]
  • [erreur][9]

Ouvrir la console #

Dans la plupart des navigateurs actuels, la console JavaScript a été intégrée sous forme d’onglet dans les outils de développement. Les touches de raccourci répertoriées ci-dessous ouvriront les outils de développement, il peut être nécessaire de passer ensuite à l’onglet de droite.

Chrome #

Ouvrez le panneau “Console” des DevTools de Chrome :

  • Windows / Linux : n’importe laquelle des options suivantes.

    • Ctrl + Shift + J
    • Ctrl + Shift + I, then click on the “Web Console” tab or press ESC to toggle the console on and off
    • F12, then click on the “Console” tab or press ESC to toggle the console on and off

  • Mac OS : Cmd + Opt + J

#Firefox

Ouverture du panneau “Console” dans les Outils de développement de Firefox :

  • Windows / Linux : n’importe laquelle des options suivantes.

    • Ctrl + Shift + K
    • Ctrl + Shift + I, then click on the “Web Console” tab or press ESC to toggle the console on and off
    • F12, then click on the “Web Console” tab or press ESC to toggle the console on and off

  • Mac OS : Cmd + Opt + K

Edge et Internet Explorer #

Ouvrir le panneau “Console” dans F12 Developer Outils:

  • F12, puis cliquez sur l’onglet “Console”

#Safari

En ouvrant le panneau “Console” dans Web Inspector de Safari, vous devez d’abord activer le menu de développement dans les préférences de Safari

[![préférences safari][23]][23]

Ensuite, vous pouvez choisir “Développer-> Afficher la console d’erreur” dans les menus ou appuyer sur ⌘ + Option + C

Opéra #

Ouverture de la « Console » dans l’opéra :

  • Ctrl + Shift + I, puis cliquez sur l’onglet “Console”

Compatibilité #

Lors de l’utilisation ou de l’émulation d’Internet Explorer 8 ou de versions antérieures (par exemple, via l’affichage de compatibilité / le mode Entreprise), la console ne sera définie ** que ** lorsque les outils de développement sont actifs, de sorte que les instructions console.log() peuvent provoquer une exception et empêcher le code de s’exécuter. Pour atténuer cela, vous pouvez vérifier si la console est disponible avant de vous connecter :

if (typeof window.console !== 'undefined')
{
   console.log("Hello World");
}

Ou au début de votre script, vous pouvez identifier si la console est disponible et sinon, définir une fonction null pour intercepter toutes vos références et éviter les exceptions.

if (!window.console)
{ 
    console = {log: function() {}}; 
}

Notez que ce deuxième exemple arrêtera tous les journaux de la console même si la fenêtre du développeur a été ouverte.

L’utilisation de ce deuxième exemple empêchera l’utilisation d’autres fonctions telles que console.dir(obj) à moins que cela ne soit spécifiquement ajouté.

Temps de mesure - console.time() #

console.time() peut être utilisé pour mesurer la durée d’exécution d’une tâche dans votre code.

L’appel de console.time([label]) démarre une nouvelle minuterie. Lorsque console.timeEnd([label]) est appelé, le temps écoulé, en millisecondes, depuis le l’appel original .time() est calculé et enregistré. En raison de ce comportement, vous pouvez appeler .timeEnd() plusieurs fois avec la même étiquette pour enregistrer le temps écoulé depuis l’appel .time() d’origine.

Exemple 1:

console.time('response in');

alert('Click to continue');
console.timeEnd('response in');

alert('One more time');
console.timeEnd('response in');

affichera :

response in: 774.967ms
response in: 1402.199ms

Exemple 2 :

var elms = document.getElementsByTagName('*'); //select all elements on the page

console.time('Loop time');

for (var i = 0; i < 5000; i++) {
    for (var j = 0, length = elms.length; j < length; j++) {
        // nothing to do ...
    }
}

console.timeEnd('Loop time');

affichera :

Loop time: 40.716ms

Formatage de la sortie de la console #

De nombreuses [méthodes d’impression de la console][1] peuvent également gérer le [formatage de chaîne de type C][2], en utilisant des jetons % :

console.log('%s has %d points', 'Sam', 100);

Affiche Sam a 100 points.

La liste complète des spécificateurs de format en Javascript est :

Prescripteur Sortie
%s Formate la valeur sous forme de chaîne
%i ou %d Formate la valeur comme un entier
%f Formate la valeur en tant que valeur à virgule flottante
%o Formate la valeur en tant qu’élément DOM extensible
%O Formate la valeur en tant qu’objet JavaScript extensible
%c Applique les règles de style CSS à la chaîne de sortie comme spécifié par le deuxième paramètre

Style avancé #

Lorsque le spécificateur de format CSS (%c) est placé sur le côté gauche de la chaîne, la méthode d’impression accepte un deuxième paramètre avec des règles CSS qui permettent un contrôle précis du formatage de cette chaîne :

console.log('%cHello world!', 'color: blue; font-size: xx-large');

Affiche :

[![sortie][3]][3]

Il est possible d’utiliser plusieurs spécificateurs de format %c :

  • toute sous-chaîne à droite d’un %c a un paramètre correspondant dans la méthode d’impression ;

  • ce paramètre peut être une chaîne vide, s’il n’est pas nécessaire d’appliquer des règles CSS à cette même sous-chaîne ;

  • si deux spécificateurs de format %c sont trouvés, la 1ère (enfermée dans %c) et la 2ème sous-chaîne auront leurs règles définies dans la 2< sup>nd et 3rd paramètre de la méthode d’impression respectivement.

  • si trois spécificateurs de format %c sont trouvés, alors les 1er, 2ème et 3ème sous-chaînes auront leurs règles définies dans respectivement le 2ème , 3ème et 4ème paramètre, et ainsi de suite…

    console.log("%cHello %cWorld%c!!", // string to be printed “color: blue;”, // applies color formatting to the 1st substring “font-size: xx-large;”, // applies font formatting to the 2nd substring “/* no CSS rule*/” // does not apply any rule to the remaing substring );

Affiche :

[![sortie de plusieurs spécificateurs CSS][4]][4]

Utilisation de groupes pour indenter la sortie #

La sortie peut être identifiée et incluse dans un groupe réductible dans la console de débogage avec les méthodes suivantes :

  • [console.groupCollapsed()][5] : crée un groupe réduit d’entrées qui peut être développé via le bouton de divulgation afin de révéler toutes les entrées effectuées après l’appel de cette méthode ;
  • [console.group()][6] : crée un groupe étendu d’entrées qui peuvent être réduites afin de masquer les entrées après l’appel de cette méthode.

L’identification peut être supprimée pour les entrées postérieures en utilisant la méthode suivante :

  • [console.groupEnd()][7] : quitte le groupe actuel, permettant aux nouvelles entrées d’être imprimées dans le groupe parent après l’invocation de cette méthode.

Les groupes peuvent être mis en cascade pour permettre plusieurs sorties identifiées ou couches réductibles les unes dans les autres :

[![Groupe réduit][8]][8] = Groupe réduit développé => [![Groupe développé][9]][9]

Impression sur la console de débogage d’un navigateur #

La console de débogage d’un navigateur peut être utilisée pour imprimer des messages simples. Ce débogage ou console Web peut être ouvert directement dans le navigateur (touche F12 dans la plupart des navigateurs – voir Remarques ci-dessous pour plus d’informations) et la méthode log de l’objet Javascript console peut être appelée en tapant ce qui suit :

console.log('My message');

Ensuite, en appuyant sur Entrée, cela affichera Mon message dans la console de débogage.

console.log() peut être appelé avec n’importe quel nombre d’arguments et de variables disponibles dans la portée actuelle. Plusieurs arguments seront imprimés sur une seule ligne avec un petit espace entre eux.

var obj = { test: 1 };
console.log(['string'], 1, obj, window);

La méthode log affichera ce qui suit dans la console de débogage :

['string']  1  Object { test: 1 }  Window { /* truncated */ }

Outre les chaînes simples, console.log() peut gérer d’autres types, tels que des tableaux, des objets, des dates, des fonctions, etc. :

console.log([0, 3, 32, 'a string']);
console.log({ key1: 'value', key2: 'another value'});

Affiche :

Array [0, 3, 32, 'a string']
Object { key1: 'value', key2: 'another value'}

Les objets imbriqués peuvent être réduits :

console.log({ key1: 'val', key2: ['one', 'two'], key3: { a: 1, b: 2 } });

Affiche :

Object { key1: 'val', key2: Array[2], key3: Object }

Certains types tels que les objets “Date” et les “fonctions” peuvent être affichés différemment :

console.log(new Date(0));
console.log(function test(a, b) { return c; });

Affiche :

Wed Dec 31 1969 19:00:00 GMT-0500 (Eastern Standard Time)
function test(a, b) { return c; }

Autres méthodes d’impression #

En plus de la méthode log, les navigateurs modernes prennent également en charge des méthodes similaires :

  • console.info &ndash ; une petite icône informative (ⓘ) apparaît sur le côté gauche de la ou des chaînes ou des objets imprimés.

  • console.warn &ndash ; une petite icône d’avertissement (!) apparaît sur le côté gauche. Dans certains navigateurs, le fond du journal est jaune.

  • console.error &ndash ; l’icône des petits temps (⊗) apparaît sur le côté gauche. Dans certains navigateurs, l’arrière-plan du journal est rouge.

  • console.timeStamp &ndash ; affiche l’heure actuelle et une chaîne spécifiée, mais n’est pas standard :

    console.timeStamp('msg');

Affiche :

  00:00:00.001 msg

  • console.trace &ndash ; affiche la trace de la pile actuelle ou affiche la même sortie que la méthode log si elle est invoquée dans la portée globale.

    function sec() {
   first();
}
function first() {
   console.trace();
}
sec();

Affiche :

  first
  sec
  (anonymous function)

[![Fonctions de la console][1]][1]

L’image ci-dessus montre toutes les fonctions, à l’exception de timeStamp, dans Chrome version 56.

Ces méthodes se comportent de la même manière que la méthode log et, dans différentes consoles de débogage, peuvent s’afficher dans différentes couleurs ou formats.

Dans certains débogueurs, les informations sur les objets individuels peuvent être développées davantage en cliquant sur le texte imprimé ou sur un petit triangle (►) qui fait référence aux propriétés de l’objet respectif. Ces propriétés d’objet qui s’effondrent peuvent être ouvertes ou fermées dans le journal. Voir le [console.dir][2] pour plus d’informations sur ce

Inclure une trace de pile lors de la journalisation - console.trace() #

function foo() {
  console.trace('My log statement');
}

foo();

Affichera ceci dans la console :

My log statement       VM696:1
  foo                  @ VM696:1
  (anonymous function) @ (program):1

Remarque : lorsqu’il est disponible, il est également utile de savoir que la même trace de pile est accessible en tant que propriété de l’objet Error. Cela peut être utile pour le post-traitement et la collecte de commentaires automatisés.

var e = new Error('foo');
console.log(e.stack);

Comptage - console.count() #

[console.count([obj])][1] place un compteur sur la valeur de l’objet fourni en argument. Chaque fois que cette méthode est invoquée, le compteur est augmenté (à l’exception de la chaîne vide ''). Une étiquette accompagnée d’un numéro s’affiche dans la console de débogage selon le format suivant :

[label]: X

label représente la valeur de l’objet passé en argument et X représente la valeur du compteur.

La valeur d’un objet est toujours considérée, même si des variables sont fournies en arguments :

var o1 = 1, o2 = '2', o3 = "";
console.count(o1);
console.count(o2);
console.count(o3);

console.count(1);
console.count('2');
console.count('');

Affiche :

1: 1
2: 1
: 1
1: 2
2: 2
: 1

Les chaînes contenant des nombres sont converties en objets Number :

console.count(42.3);
console.count(Number('42.3'));
console.count('42.3');

Affiche :

42.3: 1
42.3: 2
42.3: 3

Les fonctions pointent toujours vers l’objet global Function :

console.count(console.constructor);
console.count(function(){});
console.count(Object);
var fn1 = function myfn(){};
console.count(fn1);
console.count(Number);

Affiche :

[object Function]: 1
[object Function]: 2
[object Function]: 3
[object Function]: 4
[object Function]: 5

Certains objets reçoivent des compteurs spécifiques associés au type d’objet auquel ils se réfèrent :

console.count(undefined);
console.count(document.Batman);
var obj;
console.count(obj);
console.count(Number(undefined));
console.count(NaN);
console.count(NaN+3);
console.count(1/0);
console.count(String(1/0));
console.count(window);
console.count(document);
console.count(console);
console.count(console.__proto__);
console.count(console.constructor.prototype);
console.count(console.__proto__.constructor.prototype);
console.count(Object.getPrototypeOf(console));
console.count(null);

Affiche :

undefined: 1
undefined: 2
undefined: 3
NaN: 1
NaN: 2
NaN: 3
Infinity: 1
Infinity: 2
[object Window]: 1
[object HTMLDocument]: 1
[object Object]: 1
[object Object]: 2
[object Object]: 3
[object Object]: 4
[object Object]: 5
null: 1

Chaîne vide ou absence d’argument #

Si aucun argument n’est fourni lors de la saisie séquentielle de la méthode count dans la console de débogage, une chaîne vide est supposée comme paramètre, c’est-à-dire :

> console.count();
  : 1
> console.count('');
  : 2
> console.count("");
  : 3

Débogage avec des assertions - console.assert() #

Écrit un message d’erreur dans la console si l’assertion est “false”. Sinon, si l’assertion est “vraie”, cela ne fait rien.

console.assert('one' === 1);

[![sortie][1]][1]

Plusieurs arguments peuvent être fournis après l’assertion - il peut s’agir de chaînes ou d’autres objets - qui ne seront imprimés que si l’assertion est “fausse” :

[![Assertion avec plusieurs objets comme paramètres][2]][2]

[console.assert][3] ne lance pas une AssertionError (sauf dans [Node.js][4]), ce qui signifie que cette méthode est incompatible avec la plupart des frameworks de test et que l’exécution du code ne s’arrêtera pas sur une affirmation ratée.

Effacement de la console - console.clear() #

Vous pouvez effacer la fenêtre de la console en utilisant la méthode console.clear(). Cela supprime tous les messages précédemment imprimés dans la console et peut imprimer un message comme “La console a été effacée” dans certains environnements.
