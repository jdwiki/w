À propos des invites utilisateur #

User Prompts sont des méthodes faisant partie de [Web Application API](https://www.w3.org/TR/ html5/webappapis.html#webappapis) utilisé pour appeler les modaux du navigateur demandant une action de l’utilisateur telle qu’une confirmation ou une entrée.

fenêtre.alerte(message)

Afficher un popup modal avec un message à l’utilisateur. Nécessite que l’utilisateur clique sur [OK] pour fermer.

alert("Hello World");

Plus d’informations ci-dessous dans “Utiliser alert()”.

booléen = window.confirm(message)

Afficher un popup modal avec le message fourni. Fournit les boutons [OK] et [Annuler] qui répondront avec une valeur booléenne true / false respectivement.

confirm("Delete this comment?");

result = window.prompt(message, defaultValue)

Afficher un popup modal avec le message fourni et un champ de saisie avec une valeur pré-remplie facultative. Renvoie comme “résultat” la valeur d’entrée fournie par l’utilisateur.

prompt("Enter your website address", "http://");

Plus d’informations ci-dessous dans “Utilisation de prompt()”.


window.print()

Ouvre un modal avec des options d’impression de document.

print();

Mode d’invite persistant #

Lors de l’utilisation de invite, un utilisateur peut toujours cliquer sur Annuler et aucune valeur ne sera renvoyée. Pour éviter les valeurs vides et les rendre plus persistantes :

<h2>Welcome <span id="name"></span>!</h2>

<script>
// Persistent Prompt modal
var userName;
while(!userName) {
  userName = prompt("Enter your name", "");
  if(!userName) {
    alert("Please, we need your name!");
  } else {
    document.getElementById("name").innerHTML = userName;
  }
}
</script>

Confirmer pour supprimer l’élément #

Une façon d’utiliser confirm() est lorsqu’une action de l’interface utilisateur apporte des modifications * destructives * à la page et est mieux accompagnée d’une ** notification ** et d’une ** confirmation de l’utilisateur ** - comme par exemple avant de supprimer un message de publication :

<div id="post-102">
  <p>I like Confirm modals.</p>
  <a data-deletepost="post-102">Delete post</a>
</div>
<div id="post-103">
  <p>That's way too cool!</p>
  <a data-deletepost="post-103">Delete post</a>
</div>

// Collect all buttons
var deleteBtn = document.querySelectorAll("[data-deletepost]");

function deleteParentPost(event) {
  event.preventDefault(); // Prevent page scroll jump on anchor click
  
  if( confirm("Really Delete this post?") ) {
    var post = document.getElementById( this.dataset.deletepost );
    post.parentNode.removeChild(post);
    // TODO: remove that post from database
  } // else, do nothing
  
}

// Assign click event to buttons
[].forEach.call(deleteBtn, function(btn) {
  btn.addEventListener("click", deleteParentPost, false);
});

Utilisation d’alerte() #

La méthode alert() de l’objet window affiche une boîte d’alerte avec un message spécifié et un bouton OK ou Annuler. Le texte de ce bouton dépend du navigateur et ne peut pas être modifié.

Syntaxe

alert("Hello world!");
// Or, alternatively...
window.alert("Hello world!");

Produit

[![Image de l’alerte][1]][1]

Une boîte d’alerte est souvent utilisée si vous voulez vous assurer que les informations parviennent à l’utilisateur.

Remarque : La boîte d’alerte détourne le focus de la fenêtre actuelle et force le navigateur à lire le message. N’abusez pas de cette méthode, car elle empêche l’utilisateur d’accéder à d’autres parties de la page tant que la boîte n’est pas fermée. Il arrête également la poursuite de l’exécution du code, jusqu’à ce que l’utilisateur clique sur OK. (en particulier, les timers qui ont été définis avec setInterval() ou setTimeout() ne cochent pas non plus). La boîte d’alerte ne fonctionne que dans les navigateurs et sa conception ne peut pas être modifiée.

Paramètre Descriptif
message Obligatoire. Spécifie le texte à afficher dans la zone d’alerte ou un objet converti en chaîne et affiché.

Valeur de retour

La fonction alert ne renvoie aucune valeur

[1] : http://i.stack.imgur.com/Ttg3s.png

Utilisation de prompt() #

L’invite affichera une boîte de dialogue à l’utilisateur demandant son entrée. Vous pouvez fournir un message qui sera placé au-dessus du champ de texte. La valeur de retour est une chaîne représentant l’entrée fournie par l’utilisateur.

var name = prompt("What's your name?");
console.log("Hello, " + name);

Vous pouvez également passer à prompt() un deuxième paramètre, qui sera affiché comme texte par défaut dans le champ de texte de l’invite.

var name = prompt('What\'s your name?', ' Name...');
console.log('Hello, ' + name);
Paramètre Descriptif
message Obligatoire. Texte à afficher au-dessus du champ de texte de l’invite.
par défaut Optionnel. Texte par défaut à afficher dans le champ de texte lorsque l’invite s’affiche.
