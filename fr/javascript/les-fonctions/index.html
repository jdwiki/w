Les fonctions en JavaScript fournissent un code organisé et réutilisable pour effectuer un ensemble d’actions. Les fonctions simplifient le processus de codage, évitent la logique redondante et facilitent le suivi du code. Cette rubrique décrit la déclaration et l’utilisation des fonctions, des arguments, des paramètres, des instructions de retour et de la portée en JavaScript.

Syntaxe #

  • exemple de fonction(x) { retour x }

  • var exemple = fonction (x) { retour x }

  • (fonction() { … })(); // Expression de fonction appelée immédiatement (IIFE)

  • var instance = new Exemple(x);

  • Méthodes

  • fn.apply(valueForThis[, arrayOfArgs])

  • fn.bind(valueForThis[, arg1[, arg2, …]])

  • fn.call(valueForThis[, arg1[, arg2, …]])

  • ES2015+ (ES6+) :

  • exemple const = x => { return x } ; // Retour explicite de la fonction flèche

  • exemple const = x => x; // Retour implicite de la fonction flèche

  • exemple const = (x, y, z) => { … } // Fonction flèche plusieurs arguments

  • (() => { … })(); // IIFE utilisant une fonction fléchée

Pour plus d’informations sur les fonctions fléchées, veuillez consulter la documentation Fonctions fléchées.

Portée de la fonction #

Lorsque vous définissez une fonction, elle crée une portée.

Tout ce qui est défini dans la fonction n’est pas accessible par code en dehors de fonction. Seul le code dans cette portée peut voir les entités définies à l’intérieur du portée.

function foo() {
  var a = 'hello';
  console.log(a);  // => 'hello'
}

console.log(a);  // reference error

Les fonctions imbriquées sont possibles en JavaScript et les mêmes règles s’appliquent.

function foo() {
  var a = 'hello';
  
  function bar() {
    var b = 'world';
    console.log(a);  // => 'hello'
    console.log(b);  // => 'world'
  }

  console.log(a);  // => 'hello'
  console.log(b);  // reference error
}

console.log(a);  // reference error
console.log(b);  // reference error

Lorsque JavaScript essaie de résoudre une référence ou une variable, il commence à rechercher dans le périmètre actuel. S’il ne trouve pas cette déclaration dans la portée actuelle, il monte une portée pour le chercher. Ce processus se répète jusqu’à ce que la déclaration soit trouvée. Si l’analyseur JavaScript atteint la portée globale et ne trouve toujours pas le référence, une erreur de référence sera renvoyée.

var a = 'hello';

function foo() {
  var b = 'world';

  function bar() {
    var c = '!!';

    console.log(a);  // => 'hello'
    console.log(b);  // => 'world'
    console.log(c);  // => '!!'
    console.log(d);  // reference error
  }
}

Ce comportement d’escalade peut également signifier qu’une référence peut « ombrager » une référence portant le même nom dans la portée externe puisqu’elle est vue en premier.

var a = 'hello';

function foo() {
  var a = 'world';

  function bar() {
    console.log(a);  // => 'world'
  }
}

La façon dont JavaScript résout la portée s’applique également au mot-clé const. Déclarer une variable avec le mot-clé const implique que vous n’êtes pas autorisé à réaffecter la valeur, mais la déclarer dans une fonction créera une nouvelle portée et avec cela une nouvelle variable.

function foo() {
  const a = true;

  function bar() {
    const a = false;  // different variable
    console.log(a);   // false
  }

  const a = false;    // SyntaxError
  a = false;          // TypeError
  console.log(a);     // true
}

Cependant, les fonctions ne sont pas les seuls blocs qui créent une portée (si vous utilisez let ou const). Les déclarations let et const ont une portée de l’instruction de bloc la plus proche. Voir [ici][1] pour une description plus détaillée.

[1] : https://www.wikiod.com/fr/javascript/portee#Différence entre var et laisser

Curry #

[Currying] [1] est la transformation d’une fonction de n arité ou arguments en une séquence de n fonctions prenant un seul argument.

Cas d’utilisation : lorsque les valeurs de certains arguments sont disponibles avant d’autres, vous pouvez utiliser le currying pour décomposer une fonction en une série de fonctions qui terminent le travail par étapes, à mesure que chaque valeur arrive. Cela peut être utile :

  • Lorsque la valeur d’un argument ne change presque jamais (par exemple, un facteur de conversion), mais que vous devez conserver la flexibilité de définir cette valeur (plutôt que de la coder en dur comme une constante).
  • Lorsque le résultat d’une fonction curry est utile avant que les autres fonctions curry ne soient exécutées.
  • Valider l’arrivée des fonctions dans une séquence précise.

Par exemple, le volume d’un prisme rectangulaire peut être expliqué par une fonction de trois facteurs : longueur (l), largeur (w) et hauteur (h):

var prism = function(l, w, h) {
    return l * w * h;
}

Une version curry de cette fonction ressemblerait à :

function prism(l) {
    return function(w) {
        return function(h) {
            return l * w * h;
        }
    }
}

// alternatively, with concise ECMAScript 6+ syntax:
var prism = l => w => h => l * w * h;

Vous pouvez appeler ces séquences de fonctions avec prism(2)(3)(5), qui devrait donner 30.

Sans quelques machines supplémentaires (comme avec les bibliothèques), le curry est d’une flexibilité syntaxique limitée en JavaScript (ES 5/6) en raison du manque d’espace réservé valeurs; ainsi, bien que vous puissiez utiliser var a = prism(2)(3) pour créer un [partiellement fonction appliquée][2], vous ne pouvez pas utiliser prism())(3)(5).

[1] : https://en.wikipedia.org/wiki/Currying [2] : https://en.wikipedia.org/wiki/Partial_application

Expressions de fonction immédiatement appelées #

Parfois, vous ne voulez pas que votre fonction soit accessible/stockée en tant que variable. Vous pouvez créer une expression de fonction appelée immédiatement (IIFE en abrégé). Ce sont essentiellement des fonctions anonymes auto-exécutables. Ils ont accès à la portée environnante, mais la fonction elle-même et toutes les variables internes seront inaccessibles de l’extérieur. Une chose importante à noter à propos de IIFE est que même si vous nommez votre fonction, IIFE ne sont pas hissés comme les fonctions standard et ne peuvent pas être appelées par le nom de fonction avec lequel elles sont déclarées.

(function() {
   alert("I've run - but can't be run again because I'm immediately invoked at runtime,
          leaving behind only the result I generate");
}());

C’est une autre façon d’écrire IIFE. Notez que la parenthèse fermante avant le point-virgule a été déplacée et placée juste après l’accolade fermante :

(function() {
   alert("This is IIFE too.");
})();

Vous pouvez facilement passer des paramètres dans un IIFE :

(function(message) {
   alert(message);
}("Hello World!"));

De plus, vous pouvez renvoyer des valeurs dans la portée environnante :

var example = (function() {
   return 42;
}());
console.log(example); // => 42

Si nécessaire, il est possible de nommer un IIFE. Bien que moins souvent vu, ce modèle présente plusieurs avantages, tels que fournir une référence qui peut être utilisée pour une récursivité et peut simplifier le débogage car le nom est inclus dans la pile d’appels.

(function namedIIFE() { 
    throw error; // We can now see the error thrown in 'namedIIFE()'
}());

Alors que l’enveloppement d’une fonction entre parenthèses est le moyen le plus courant d’indiquer à l’analyseur Javascript d’attendre une expression, aux endroits où une expression est déjà attendue, la notation peut être rendue plus concise :

var a = function() { return 42 }();
console.log(a)  // => 42

Version fléchée de la fonction immédiatement invoquée :

(() => console.log("Hello!"))(); // => Hello!

Liaison this et arguments #

Lorsque vous prenez une référence à une méthode (une propriété qui est une fonction) en JavaScript, elle ne se souvient généralement pas de l’objet auquel elle était initialement attachée. Si la méthode doit faire référence à cet objet en tant que this, elle ne pourra pas le faire et l’appeler provoquera probablement un plantage.

Vous pouvez utiliser la méthode .bind() sur une fonction pour créer un wrapper qui inclut la valeur de this et n’importe quel nombre d’arguments principaux.

var monitor = {
  threshold: 5,
  check: function(value) {
    if (value > this.threshold) {
      this.display("Value is too high!");
    }
  },
  display(message) {
    alert(message);
  }
};

monitor.check(7); // The value of `this` is implied by the method call syntax.


var badCheck = monitor.check;
badCheck(15); // The value of `this` is window object and this.threshold is undefined, so value > this.threshold is false

var check = monitor.check.bind(monitor);
check(15); // This value of `this` was explicitly bound, the function works.

var check8 = monitor.check.bind(monitor, 8);
check8(); // We also bound the argument to `8` here. It can't be re-specified.

Lorsqu’elle n’est pas en mode strict, une fonction utilise l’objet global (window dans le navigateur) comme this, à moins que la fonction ne soit appelée en tant que méthode, liée ou appelée avec la syntaxe de la méthode .call.

window.x = 12; 

function example() {
  return this.x;
}

console.log(example()); // 12

En mode strict this est undefined par défaut

window.x = 12; 
    
function example() {
  "use strict";
  return this.x;
}

console.log(example()); // Uncaught TypeError: Cannot read property 'x' of undefined(…)

Opérateur de liaison #

Le double deux-points opérateur de liaison peut être utilisé comme syntaxe abrégée pour le concept expliqué ci-dessus :

var log = console.log.bind(console); // long version
const log = ::console.log; // short version

foo.bar.call(foo); // long version
foo::bar(); // short version

foo.bar.call(foo, arg1, arg2, arg3); // long version
foo::bar(arg1, arg2, arg3); // short version

foo.bar.apply(foo, args); // long version
foo::bar(...args); // short version

Cette syntaxe vous permet d’écrire normalement, sans vous soucier de lier “this” partout.

Lier les fonctions de la console aux variables #

var log = console.log.bind(console);

Usage:

log('one', '2', 3, [4], {5: 5});

Production:

one 2 3 [4] Object {5: 5}

Pourquoi ferais-tu ça?

Un cas d’utilisation peut être lorsque vous avez un enregistreur personnalisé et que vous souhaitez décider du temps d’exécution lequel utiliser.

var logger = require('appLogger');

var log = logToServer ? logger.log : console.log.bind(console);

Fonctions nommées #

Les fonctions peuvent être nommées ou non ([fonctions anonymes][1]) :

var namedSum = function sum (a, b) { // named
    return a + b;
}

var anonSum = function (a, b) { // anonymous
    return a + b;
}

namedSum(1, 3);
anonSum(1, 3);

4 4

Mais leurs noms sont privés à leur propre portée :

var sumTwoNumbers = function sum (a, b) {
    return a + b;
}

sum(1, 3);

Erreur de référence non détectée : la somme n’est pas définie

Les fonctions nommées diffèrent des fonctions anonymes dans plusieurs scénarios :

  • Lorsque vous déboguez, le nom de la fonction apparaîtra dans la trace d’erreur/pile
  • Les fonctions nommées sont [hissées][2] alors que les fonctions anonymes ne le sont pas
  • Les fonctions nommées et les fonctions anonymes se comportent différemment lors de la gestion de la récursivité
  • Selon la version d’ECMAScript, les fonctions nommées et anonymes peuvent traiter différemment la propriété name de la fonction

Les fonctions nommées sont hissées #

Lors de l’utilisation d’une fonction anonyme, la fonction ne peut être appelée qu’après la ligne de déclaration, alors qu’une fonction nommée peut être appelée avant la déclaration. Envisager

foo();
var foo = function () { // using an anonymous function
    console.log('bar');
}

TypeError non capturé : foo n’est pas une fonction

foo();
function foo () { // using a named function
    console.log('bar');
}

barre

Fonctions nommées dans un scénario récursif #

Une fonction récursive peut être définie comme :

var say = function (times) {
    if (times > 0) {
        console.log('Hello!');

        say(times - 1);
    }
}

//you could call 'say' directly, 
//but this way just illustrates the example
var sayHelloTimes = say;

sayHelloTimes(2);

Bonjour ! Bonjour !

Et si, quelque part dans votre code, la liaison de fonction d’origine était redéfinie ?

var say = function (times) {
    if (times > 0) {
        console.log('Hello!');

        say(times - 1);
    }
}

var sayHelloTimes = say;
say = "oops";

sayHelloTimes(2);

Bonjour ! TypeError non intercepté : say n’est pas une fonction

Cela peut être résolu en utilisant une fonction nommée

// The outer variable can even have the same name as the function
// as they are contained in different scopes
var say = function say (times) {

La propriété name des fonctions #

Avant ES6, les fonctions nommées avaient leurs propriétés name définies sur leurs noms de fonction, et les fonctions anonymes avaient leurs propriétés name définies sur la chaîne vide.

var foo = function () {}
console.log(foo.name); // outputs ''

function foo () {}
console.log(foo.name); // outputs 'foo'

Post ES6, les fonctions nommées et sans nom définissent toutes deux leurs propriétés name :

var foo = function () {}
console.log(foo.name); // outputs 'foo'

function foo () {}
console.log(foo.name); // outputs 'foo'

var foo = function bar () {}
console.log(foo.name); // outputs 'bar'

Fonctions avec un nombre inconnu d’arguments (fonctions variadiques) #

Pour créer une fonction qui accepte un nombre indéterminé d’arguments, il existe deux méthodes selon votre environnement.

Chaque fois qu’une fonction est appelée, elle a un objet [arguments][1] de type Array dans sa portée, contenant tous les arguments passés à la fonction. L’indexation ou l’itération sur ceci donnera accès aux arguments, par exemple

function logSomeThings() {
    for (var i = 0; i < arguments.length; ++i) {
        console.log(arguments[i]);
    }
}

logSomeThings('hello', 'world');
// logs "hello"
// logs "world"

Notez que vous pouvez convertir arguments en un tableau réel si nécessaire ; voir : https://www.wikiod.com/fr/javascript/tableaux#Conversion d’objets de type tableau en tableaux

Depuis ES6, la fonction peut être déclarée avec son dernier paramètre en utilisant [rest operator][2] (...). Cela crée un tableau qui contient les arguments à partir de ce point

function personLogsSomeThings(person, ...msg) {
   msg.forEach(arg => {
       console.log(person, 'says', arg);
   });
}

personLogsSomeThings('John', 'hello', 'world');
// logs "John says hello"
// logs "John says world"

Les fonctions peuvent également être appelées de la même manière, la [syntaxe de diffusion][3]

const logArguments = (...args) => console.log(args)
const list = [1, 2, 3]

logArguments('a', 'b', 'c', ...list)
// output: Array [ "a", "b", "c", 1, 2, 3 ]

Cette syntaxe peut être utilisée pour insérer un nombre arbitraire d’arguments à n’importe quelle position, et peut être utilisée avec n’importe quel itérable (apply accepte uniquement les objets de type tableau).

const logArguments = (...args) => console.log(args)
function* generateNumbers() {
  yield 6
  yield 5
  yield 4
}

logArguments('a', ...generateNumbers(), ...'pqr', 'b')
// output: Array [ "a", 6, 5, 4, "p", "q", "r", "b" ]

[1] : https://developer.mozilla.org/en-US/docs/Web/JavaScript/Reference/Functions/arguments [2] : https://developer.mozilla.org/docs/Web/JavaScript/Reference/Functions/rest_parameters [3] : https://developer.mozilla.org/docs/Web/JavaScript/Reference/Operators/Spread_operator

Fonction anonyme #

Définir une fonction anonyme #

Lorsqu’une fonction est définie, vous lui donnez souvent un nom, puis vous l’invoquez en utilisant ce nom, comme ceci :

foo();

function foo(){
    // ...
}

Lorsque vous définissez une fonction de cette manière, le runtime Javascript stocke votre fonction en mémoire, puis crée une référence à cette fonction, en utilisant le nom que vous lui avez attribué. Ce nom est alors accessible dans la portée actuelle. Cela peut être un moyen très pratique de créer une fonction, mais Javascript ne vous oblige pas à attribuer un nom à une fonction. Ce qui suit est également parfaitement légal :

function() {
    // ...
}

Lorsqu’une fonction est définie sans nom, on parle de fonction anonyme. La fonction est stockée en mémoire, mais le runtime ne crée pas automatiquement une référence à celle-ci pour vous. À première vue, il peut sembler qu’une telle chose n’aurait aucune utilité, mais il existe plusieurs scénarios où les fonctions anonymes sont très pratiques.

Affectation d’une fonction anonyme à une variable #

Une utilisation très courante des fonctions anonymes consiste à les affecter à une variable :

var foo = function(){ /*...*/ };

foo();

Cette utilisation de fonctions anonymes est couverte plus en détail dans https://www.wikiod.com/fr/javascript/les-fonctions#Fonctionne comme une variable

Fournir une fonction anonyme en tant que paramètre à une autre fonction #

Certaines fonctions peuvent accepter une référence à une fonction en tant que paramètre. Celles-ci sont parfois appelées “injections de dépendances” ou “rappels”, car elles permettent à la fonction que vous appelez de “rappeler” votre code, vous donnant ainsi la possibilité de modifier le comportement de la fonction appelée. Par exemple, la fonction map de l’objet Array vous permet d’itérer sur chaque élément d’un tableau, puis de créer un nouveau tableau en appliquant une fonction de transformation à chaque élément.

var nums = [0,1,2];
var doubledNums = nums.map( function(element){ return element * 2; } ); // [0,2,4]

Il serait fastidieux, bâclé et inutile de créer une fonction nommée, ce qui encombrerait votre périmètre avec une fonction nécessaire uniquement à cet endroit et casserait le flux naturel et la lecture de votre code (un collègue devrait quitter ce code pour trouver votre fonction pour comprendre ce qui se passe).

Renvoyer une fonction anonyme à partir d’une autre fonction #

Parfois, il est utile de renvoyer une fonction comme résultat d’une autre fonction. Par exemple:

var hash = getHashFunction( 'sha1' );
var hashValue = hash( 'Secret Value' );

function getHashFunction( algorithm ){

    if ( algorithm === 'sha1' ) return function( value ){ /*...*/ };
    else if ( algorithm === 'md5' ) return function( value ){ /*...*/ };

}

Invoquer immédiatement une fonction anonyme #

Contrairement à de nombreux autres langages, la portée en Javascript est au niveau de la fonction et non au niveau du bloc. (Voir https://www.wikiod.com/fr/javascript/les-fonctions#Portée des fonctions ). Dans certains cas, cependant, il est nécessaire de créer une nouvelle portée. Par exemple, il est courant de créer une nouvelle portée lors de l’ajout de code via une balise <script>, plutôt que d’autoriser la définition de noms de variables dans la portée globale (ce qui risque d’entraîner d’autres scripts en collision avec vos noms de variables). Une méthode courante pour gérer cette situation consiste à définir une nouvelle fonction anonyme, puis à l’invoquer immédiatement, en vous cachant en toute sécurité des variables dans le cadre de la fonction anonyme et sans rendre votre code accessible à des tiers via un nom de fonction divulgué. Par exemple:

<!-- My Script -->
<script>
function initialize(){
    // foo is safely hidden within initialize, but...
    var foo = '';
}

// ...my initialize function is now accessible from global scope.
// There's a risk someone could call it again, probably by accident.
initialize();
</script>

<script>
// Using an anonymous function, and then immediately
// invoking it, hides my foo variable and guarantees
// no one else can call it a second time.
(function(){
    var foo = '';
}()) // <--- the parentheses invokes the function immediately
</script>

Fonctions anonymes auto-référentielles #

Parfois, il est utile qu’une fonction anonyme puisse se référer à elle-même. Par exemple, la fonction peut avoir besoin de s’appeler elle-même de manière récursive ou de s’ajouter des propriétés à elle-même. Si la fonction est anonyme, cependant, cela peut être très difficile car cela nécessite de connaître la variable à laquelle la fonction a été affectée. C’est la solution moins qu’idéale :

var foo = function(callAgain){
    console.log( 'Whassup?' );
    // Less then ideal... we're dependent on a variable reference...
    if (callAgain === true) foo(false);
};

foo(true);

// Console Output:
// Whassup?
// Whassup?

// Assign bar to the original function, and assign foo to another function.
var bar = foo;
foo = function(){
    console.log('Bad.')
};

bar(true);

// Console Output:
// Whassup?
// Bad.

L’intention ici était que la fonction anonyme s’appelle elle-même de manière récursive, mais lorsque la valeur de foo change, vous vous retrouvez avec un bogue potentiellement difficile à tracer.

Au lieu de cela, nous pouvons donner à la fonction anonyme une référence à elle-même en lui donnant un nom privé, comme ceci :

var foo = function myself(callAgain){
    console.log( 'Whassup?' );
    // Less then ideal... we're dependent on a variable reference...
    if (callAgain === true) myself(false);
};

foo(true);

// Console Output:
// Whassup?
// Whassup?

// Assign bar to the original function, and assign foo to another function.
var bar = foo;
foo = function(){
    console.log('Bad.')
};

bar(true);

// Console Output:
// Whassup?
// Whassup?

Notez que le nom de la fonction est limité à lui-même. Le nom n’a pas fui dans la portée externe :

myself(false); // ReferenceError: myself is not defined

Cette technique est particulièrement utile lorsqu’il s’agit de fonctions anonymes récursives en tant que paramètres de rappel :

// Calculate the fibonacci value for each number in an array:
var fib = false,
    result = [1,2,3,4,5,6,7,8].map(
        function fib(n){
            return ( n <= 2 ) ? 1 : fib( n - 1 ) + fib( n - 2 );
        });
// result = [1, 1, 2, 3, 5, 8, 13, 21]
// fib = false (the anonymous function name did not overwrite our fib variable)

Paramètres par défaut #

Avant ECMAScript 2015 (ES6), la valeur par défaut d’un paramètre pouvait être affectée de la manière suivante :

function printMsg(msg) {
  msg = typeof msg !== 'undefined' ? // if a value was provided 
        msg :                        // then, use that value in the reassignemnt
        'Default value for msg.';    // else, assign a default value
  console.log(msg);
}

ES6 a fourni une nouvelle syntaxe où la condition et la réaffectation décrites ci-dessus ne sont plus nécessaires :

function printMsg(msg='Default value for msg.') {
    console.log(msg);
}

printMsg(); // -> "Default value for msg."
printMsg(undefined); // -> "Default value for msg."
printMsg('Now my msg in different!'); // -> "Now my msg in different!"

Cela montre également que si un paramètre est manquant lorsque la fonction est invoquée, sa valeur est conservée comme “undefined”, car cela peut être confirmé en le fournissant explicitement dans l’exemple suivant (à l’aide d’une [fonction flèche][1]) :

let param_check = (p = 'str') => console.log(p + ' is of type: ' + typeof p);

param_check(); // -> "str is of type: string"
param_check(undefined); // -> "str is of type: string"

param_check(1); // -> "1 is of type: number"
param_check(this); // -> "[object Window] is of type: object"

Fonctions/variables comme valeurs par défaut et réutilisation des paramètres #

Les valeurs des paramètres par défaut ne sont pas limitées aux nombres, chaînes ou objets simples. Une fonction peut également être définie comme valeur par défaut callback = function(){} :

function foo(callback = function(){ console.log('default'); }) {
    callback();
}

foo(function (){
    console.log('custom');
});
// custom

foo();
//default

Certaines caractéristiques des opérations peuvent être effectuées via des valeurs par défaut :

  • Un paramètre précédemment déclaré peut être réutilisé comme valeur par défaut pour les valeurs des paramètres à venir.
  • Les opérations en ligne sont autorisées lors de l’attribution d’une valeur par défaut à un paramètre.
  • Les variables existant dans la même portée de la fonction déclarée peuvent être utilisées dans ses valeurs par défaut.
  • Les fonctions peuvent être invoquées afin de fournir leur valeur de retour dans une valeur par défaut.
let zero = 0;
function multiply(x) { return x * 2;}

function add(a = 1 + zero, b = a, c = b + a, d = multiply(c)) {
    console.log((a + b + c), d);
}

add(1);                // 4, 4
add(3);                // 12, 12
add(2, 7);             // 18, 18
add(1, 2, 5);          // 8, 10
add(1, 2, 5, 10);      // 8, 20

Réutilisation de la valeur de retour de la fonction dans la valeur par défaut d’un nouvel appel : #

let array = [1]; // meaningless: this will be overshadowed in the function's scope
function add(value, array = []) {
  array.push(value);
  return array;
}
add(5);         // [5]
add(6);         // [6], not [5, 6]
add(6, add(5)); // [5, 6]

arguments valeur et longueur lorsqu’il manque des paramètres dans l’invocation #

L’objet tableau [arguments][2] ne retient que les paramètres dont les valeurs ne sont pas par défaut, c’est-à-dire celles qui sont explicitement fournies lors de l’invocation de la fonction :

function foo(a = 1, b = a + 1) {
    console.info(arguments.length, arguments);
    console.log(a,b);
}

foo();        // info: 0 >> []     | log: 1, 2
foo(4);       // info: 1 >> [4]    | log: 4, 5
foo(5, 6);    // info: 2 >> [5, 6] | log: 5, 6

[1] : https://www.wikiod.com/fr/javascript/fonctions-flechees#Introduction [2] : https://www.wikiod.com/fr/javascript/fonctions-flechees#Arguments Objet

Arguments de fonction, objet “arguments”, paramètres de repos et d’étalement #

Les fonctions peuvent prendre des entrées sous la forme de variables qui peuvent être utilisées et affectées à l’intérieur de leur propre portée. La fonction suivante prend deux valeurs numériques et renvoie leur somme :

function addition (argument1, argument2){
    return argument1 + argument2; 
}

console.log(addition(2, 3)); // -> 5

objet arguments #

L’objet arguments contient tous les paramètres de la fonction qui contiennent une autre [valeur par défaut][1]. Il peut également être utilisé même si les paramètres ne sont pas explicitement déclarés :

(function() { console.log(arguments) })(0,'str', [2,{3}]) // -> [0, "str", Array[2]]

Bien que lors de l’impression d’arguments, la sortie ressemble à un tableau, il s’agit en fait d’un objet :

(function() { console.log(typeof arguments) })(); // -> object

Paramètres de repos : fonction (...parm) {} #

Dans ES6, la syntaxe ... lorsqu’elle est utilisée dans la déclaration des paramètres d’une fonction transforme la variable à sa droite en un seul objet contenant tous les paramètres restants fournis après ceux déclarés. Cela permet d’invoquer la fonction avec un nombre illimité d’arguments, qui feront partie de cette variable :

(function(a, ...b){console.log(typeof b+': '+b[0]+b[1]+b[2]) })(0,1,'2',[3],{i:4});
// -> object: 123

Paramètres de répartition : nom_fonction(...varb); #

Dans ES6, la syntaxe ... peut également être utilisée lors de l’appel d’une fonction en plaçant un objet/une variable à sa droite. Cela permet aux éléments de cet objet d’être passés dans cette fonction en tant qu’objet unique :

let nums = [2,42,-1];
console.log(...['a','b','c'], Math.max(...nums)); // -> a b c 42

[1] : https://www.wikiod.com/fr/javascript/les-fonctions#Paramètres par défaut

Passage d’arguments par référence ou valeur #

En JavaScript, tous les arguments sont passés par valeur. Lorsqu’une fonction affecte une nouvelle valeur à une variable d’argument, ce changement ne sera pas visible pour l’appelant :

var obj = {a: 2};
function myfunc(arg){
    arg = {a: 5}; // Note the assignment is to the parameter variable itself
}
myfunc(obj);
console.log(obj.a); // 2

Cependant, les modifications apportées aux propriétés (imbriquées) de ces arguments seront visibles pour l’appelant :

var obj = {a: 2};
function myfunc(arg){
    arg.a = 5; // assignment to a property of the argument
}
myfunc(obj);
console.log(obj.a); // 5

Cela peut être vu comme un appel par référence : bien qu’une fonction ne puisse pas changer l’objet de l’appelant en lui attribuant une nouvelle valeur, elle pourrait muter l’objet de l’appelant.

Comme les arguments à valeur primitive, comme les nombres ou les chaînes, sont immuables, il n’y a aucun moyen pour une fonction de les muter :

var s = 'say';
function myfunc(arg){
    arg += ' hello'; // assignment to the parameter variable itself
}
myfunc(s);
console.log(s); // 'say'

Lorsqu’une fonction veut muter un objet passé en argument, mais ne veut pas réellement muter l’objet de l’appelant, la variable d’argument doit être réaffectée :

var obj = {a: 2, b: 3};
function myfunc(arg){
    arg = Object.assign({}, arg); // assignment to argument variable, shallow copy
    arg.a = 5;
}
myfunc(obj);
console.log(obj.a); // 2

Comme alternative à la mutation sur place d’un argument, les fonctions peuvent créer une nouvelle valeur, basée sur l’argument, et la renvoyer. L’appelant peut alors l’affecter, même à la variable d’origine passée en argument :

var a = 2;
function myfunc(arg){
    arg++;
    return arg;
}
a = myfunc(a);
console.log(obj.a); // 3

Appelez et postulez #

Les fonctions ont deux méthodes intégrées qui permettent au programmeur de fournir des arguments et la variable this différemment : call et apply.

Ceci est utile, car les fonctions qui fonctionnent sur un objet (l’objet dont elles sont une propriété) peuvent être réaffectées pour fonctionner sur un autre objet compatible. De plus, les arguments peuvent être donnés en une seule fois sous forme de tableaux, similaires à l’opérateur de propagation (...) dans ES6.

let obj = {
    a: 1,
    b: 2,
    set: function (a, b) {
        this.a = a;
        this.b = b;
    }
};

obj.set(3, 7); // normal syntax
obj.set.call(obj, 3, 7); // equivalent to the above
obj.set.apply(obj, [3, 7]); // equivalent to the above; note that an array is used

console.log(obj); // prints { a: 3, b: 5 }

let myObj = {};
myObj.set(5, 4); // fails; myObj has no `set` property
obj.set.call(myObj, 5, 4); // success; `this` in set() is re-routed to myObj instead of obj
obj.set.apply(myObj, [5, 4]); // same as above; note the array

console.log(myObj); // prints { a: 3, b: 5 }

ECMAScript 5 a introduit une autre méthode appelée bind() en plus de call() et apply() pour définir explicitement this valeur de la fonction à un objet spécifique.

Il se comporte tout à fait différemment des deux autres. Le premier argument de bind() est la valeur this pour la nouvelle fonction. Tous les autres arguments représentent des paramètres nommés qui doivent être définis de manière permanente dans la nouvelle fonction.

function showName(label) {
    console.log(label + ":" + this.name);
}
var student1 = {
     name: "Ravi"
};
var student2 = {
     name: "Vinod"
};

// create a function just for student1
var showNameStudent1 = showName.bind(student1);
showNameStudent1("student1"); // outputs "student1:Ravi"

// create a function just for student2
var showNameStudent2 = showName.bind(student2, "student2");
showNameStudent2(); // outputs "student2:Vinod"

// attaching a method to an object doesn't change `this` value of that method.
student2.sayName = showNameStudent1;
student2.sayName("student2"); // outputs "student2:Ravi"

Application partielle #

Semblable au curry, l’application partielle est utilisée pour réduire le nombre d’arguments passés à une fonction. Contrairement au curry, le nombre n’a pas besoin de diminuer de un.

Exemple:

Cette fonction…

function multiplyThenAdd(a, b, c) {
    return a * b + c;
}

… peut être utilisé pour créer une autre fonction qui multipliera toujours par 2, puis ajoutera 10 à la valeur transmise ;

function reversedMultiplyThenAdd(c, b, a) {
    return a * b + c;
}

function factory(b, c) {
    return reversedMultiplyThenAdd.bind(null, c, b);
}

var multiplyTwoThenAddTen = factory(2, 10);
multiplyTwoThenAddTen(10); // 30

La partie “application” de l’application partielle consiste simplement à fixer les paramètres d’une fonction.

Composition de la fonction #

La composition de plusieurs fonctions en une seule est une pratique courante de la programmation fonctionnelle ;

la composition crée un pipeline à travers lequel nos données transiteront et seront modifiées simplement en travaillant sur la fonction-composition (tout comme assembler des morceaux d’une piste) …

vous commencez avec quelques fonctions à responsabilité unique :

 const capitalize = x => x.replace(/^\w/, m => m.toUpperCase());
 const sign = x => x + ',\nmade with love';

et créez facilement une piste de transformation :

 const formatText = compose(capitalize, sign);

 formatText('this is an example')
 //This is an example,
 //made with love

N.B. La composition est obtenue grâce à une fonction utilitaire généralement appelée “composer” comme dans notre exemple.

L’implémentation de compose est présente dans de nombreuses bibliothèques d’utilitaires JavaScript ([lodash][1], [rambda][2], etc.), mais vous pouvez également commencer avec une implémentation simple telle que :

 const compose = (...funs) =>
   x =>
   funs.reduce((ac, f) => f(ac), x);

[1] : https://lodash.com/docs#flow [2] : http://ramdajs.com/

Fonctionne comme une variable #

Une déclaration de fonction normale ressemble à ceci :

function foo(){
}

Une fonction ainsi définie est accessible de n’importe où dans son contexte par son nom. Mais parfois, il peut être utile de traiter les références de fonction comme des références d’objet. Par exemple, vous pouvez affecter un objet à une variable en fonction d’un ensemble de conditions, puis récupérer ultérieurement une propriété de l’un ou l’autre objet :

var name = 'Cameron';
var spouse;

if ( name === 'Taylor' ) spouse = { name: 'Jordan' };
else if ( name === 'Cameron' ) spouse = { name: 'Casey' };

var spouseName = spouse.name;

En JavaScript, vous pouvez faire la même chose avec les fonctions :

// Example 1
var hashAlgorithm = 'sha1';
var hash;

if ( hashAlgorithm === 'sha1' ) hash = function(value){ /*...*/ };
else if ( hashAlgorithm === 'md5' ) hash = function(value){ /*...*/ };

hash('Fred');

Dans l’exemple ci-dessus, hash est une variable normale. Une référence à une fonction lui est assignée, après quoi la fonction à laquelle elle fait référence peut être invoquée à l’aide de parenthèses, tout comme une déclaration de fonction normale.

L’exemple ci-dessus fait référence à des fonctions anonymes… des fonctions qui n’ont pas leur propre nom. Vous pouvez également utiliser des variables pour faire référence à des fonctions nommées. L’exemple ci-dessus pourrait être réécrit comme suit :

// Example 2
var hashAlgorithm = 'sha1';
var hash;

if ( hashAlgorithm === 'sha1' ) hash = sha1Hash;
else if ( hashAlgorithm === 'md5' ) hash = md5Hash;

hash('Fred');

function md5Hash(value){
    // ...
}

function sha1Hash(value){
    // ...
}

Ou, vous pouvez affecter des références de fonction à partir des propriétés d’objet :

// Example 3
var hashAlgorithms = {
    sha1: function(value) { /**/ },
    md5: function(value) { /**/ }
};

var hashAlgorithm = 'sha1';
var hash;

if ( hashAlgorithm === 'sha1' ) hash = hashAlgorithms.sha1;
else if ( hashAlgorithm === 'md5' ) hash = hashAlgorithms.md5;

hash('Fred');

Vous pouvez affecter la référence à une fonction détenue par une variable à une autre en omettant les parenthèses. Cela peut entraîner une erreur facile à commettre : tenter d’affecter la valeur de retour d’une fonction à une autre variable, mais affecter accidentellement la référence à la fonction.

// Example 4
var a = getValue;
var b = a; // b is now a reference to getValue.
var c = b(); // b is invoked, so c now holds the value returned by getValue (41)

function getValue(){
    return 41;
}

Une référence à une fonction est comme n’importe quelle autre valeur. Comme vous l’avez vu, une référence peut être affectée à une variable, et la valeur de référence de cette variable peut ensuite être affectée à d’autres variables. Vous pouvez passer des références à des fonctions comme n’importe quelle autre valeur, y compris passer une référence à une fonction comme valeur de retour d’une autre fonction. Par exemple:

// Example 5
// getHashingFunction returns a function, which is assigned
// to hash for later use:
var hash = getHashingFunction( 'sha1' );
// ...
hash('Fred');


// return the function corresponding to the given algorithmName
function getHashingFunction( algorithmName ){
    // return a reference to an anonymous function
    if (algorithmName === 'sha1') return function(value){ /**/ };
    // return a reference to a declared function
    else if (algorithmName === 'md5') return md5;
}

function md5Hash(value){
    // ...
}

Vous n’avez pas besoin d’assigner une référence de fonction à une variable pour l’invoquer. Cet exemple, basé sur l’exemple 5, appellera getHashingFunction, puis invoquera immédiatement la fonction renvoyée et transmettra sa valeur de retour à hashedValue.

// Example 6
var hashedValue = getHashingFunction( 'sha1' )( 'Fred' );

Une note sur le levage #

Gardez à l’esprit que, contrairement aux déclarations de fonctions normales, les variables qui font référence à des fonctions ne sont pas « hissées ». Dans l’exemple 2, les fonctions md5Hash et sha1Hash sont définies en bas du script, mais sont disponibles partout immédiatement. Peu importe où vous définissez une fonction, l’interpréteur la « hisse » au sommet de sa portée, la rendant immédiatement disponible. Ce n’est pas le cas pour les définitions de variables, donc un code comme celui-ci va casser :

var functionVariable;

hoistedFunction(); // works, because the function is "hoisted" to the top of its scope
functionVariable(); // error: undefined is not a function.

function hoistedFunction(){}
functionVariable = function(){};

Fonction récursive #

Une fonction récursive est simplement une fonction qui s’appellerait elle-même.

function factorial (n) {
    if (n <= 1) {
        return 1;
    }
    
    return n * factorial(n - 1);
}

La fonction ci-dessus montre un exemple de base de la façon d’effectuer une fonction récursive pour renvoyer une factorielle.

Un autre exemple serait de récupérer la somme de nombres pairs dans un tableau.

function countEvenNumbers (arr) {
    // Sentinel value.  Recursion stops on empty array.
    if (arr.length < 1) {
        return 0;
    }
    // The shift() method removes the first element from an array 
    // and returns that element. This method changes the length of the array.
    var value = arr.shift();

    // `value % 2 === 0` tests if the number is even or odd
    // If it's even we add one to the result of counting the remainder of 
    // the array.  If it's odd, we add zero to it.
    return ((value % 2 === 0) ? 1 : 0) + countEvens(arr);
}

Il est important que ces fonctions effectuent une sorte de vérification de la valeur sentinelle pour éviter les boucles infinies. Dans le premier exemple ci-dessus, lorsque n est inférieur ou égal à 1, la récursivité s’arrête, permettant au résultat de chaque appel d’être renvoyé dans la pile des appels.

Utilisation de la déclaration de retour #

L’instruction return peut être un moyen utile de créer une sortie pour une fonction. L’instruction return est particulièrement utile si vous ne savez pas encore dans quel contexte la fonction sera utilisée.

//An example function that will take a string as input and return 
//the first character of the string.

function firstChar (stringIn){
    return stringIn.charAt(0);
}

Maintenant, pour utiliser cette fonction, vous devez la mettre à la place d’une variable ailleurs dans votre code :

Utiliser le résultat de la fonction comme argument pour une autre fonction :

console.log(firstChar("Hello world"));

La sortie de la console sera :

> H

L’instruction return termine la fonction

Si nous modifions la fonction au début, nous pouvons démontrer que l’instruction return termine la fonction.

function firstChar (stringIn){
    console.log("The first action of the first char function");
    return stringIn.charAt(0);
    console.log("The last action of the first char function");
}

Exécuter cette fonction comme suit ressemblera à ceci :

console.log(firstChar("JS"));

Sortie console :

> The first action of the first char function
> J

Il n’imprimera pas le message après l’instruction return, car la fonction est maintenant terminée.

Instruction de retour sur plusieurs lignes :

En JavaScript, vous pouvez normalement diviser une ligne de code en plusieurs lignes à des fins de lisibilité ou d’organisation. Ceci est du JavaScript valide :

var
    name = "bob",
    age = 18;

Lorsque JavaScript voit une instruction incomplète comme var, il regarde la ligne suivante pour se compléter. Cependant, si vous faites la même erreur avec l’instruction return, vous n’obtiendrez pas ce que vous attendiez.

return
    "Hi, my name is "+ name + ". " +
    "I'm "+ age + " years old.";

Ce code renverra undefined car return en lui-même est une instruction complète en Javascript, il ne regardera donc pas la ligne suivante pour se compléter. Si vous devez diviser une instruction return en plusieurs lignes, placez une valeur à côté de return avant de la diviser, comme ceci.

return "Hi, my name is " + name + ". " +
    "I'm " + age + " years old.";

Récupère le nom d’un objet fonction #

ES6 :

myFunction.name

[Explication sur MDN][1]. À partir de 2015, fonctionne dans nodejs et tous les principaux navigateurs sauf IE.

ES5 :

Si vous avez une référence à la fonction, vous pouvez faire :

function functionName( func )
{
    // Match:
    // - ^          the beginning of the string
    // - function   the word 'function'
    // - \s+        at least some white space
    // - ([\w\$]+)  capture one or more valid JavaScript identifier characters
    // - \(         followed by an opening brace
    //
    var result = /^function\s+([\w\$]+)\(/.exec( func.toString() )
    
    return result ? result[1] : ''
}
