Syntaxe #

  • laissez regex = /motif/[flags]
  • let regex = new RegExp(’pattern’, [flags])
  • laissez ismatch = regex.test(’texte')
  • laisser résultats = regex.exec(’texte')

Paramètres #

Drapeaux Détails
g **global. Tous les matchs (ne pas revenir sur le premier match).
m mmultiligne. Oblige ^ & $ à correspondre au début/à la fin de chaque ligne (pas seulement au début/à la fin de la chaîne).
je **insensible. Correspondance insensible à la casse (ignore la casse de [a-zA-Z]).
tu unicode : les chaînes de modèle sont traitées comme UTF-16. Permet également aux séquences d’échappement de correspondre aux caractères Unicode.
y sticky : correspond uniquement à partir de l’index indiqué par la propriété lastIndex de cette expression régulière dans la chaîne cible (et n’essaie pas de faire correspondre à partir d’index ultérieurs).

L’objet RegExp n’est utile que si votre connaissance des expressions régulières est solide. Voir ici pour une introduction, ou voir [MDN](https://developer.mozilla.org/en-US/docs/ Web/JavaScript/Reference/Global_Objects/RegExp) pour une explication plus approfondie.

Création d’un objet RegExp #

## Création de normes

Il est recommandé d’utiliser ce formulaire uniquement lors de la création de regex à partir de variables dynamiques.

À utiliser lorsque l’expression peut changer ou que l’expression est générée par l’utilisateur.

var re = new RegExp(".*");

Avec drapeaux :

var re = new RegExp(".*", "gmi");

Avec une barre oblique inverse : (cela doit être échappé car la regex est spécifiée avec une chaîne)

var re = new RegExp("\\w*");

Initialisation statique #

À utiliser lorsque vous savez que l’expression régulière ne changera pas et que vous savez ce qu’est l’expression avant l’exécution.

var re = /.*/;

Avec drapeaux :

var re = /.*/gmi;

Avec une barre oblique inverse : (cela ne doit pas être échappé car la regex est spécifiée dans un littéral)

var re = /\w*/;

Indicateurs RegExp #

Il existe plusieurs drapeaux que vous pouvez spécifier pour modifier le comportement RegEx. Les drapeaux peuvent être ajoutés à la fin d’un littéral de regex, comme en spécifiant gi dans /test/gi, ou ils peuvent être spécifiés comme deuxième argument du constructeur RegExp, comme dans new RegExp('test ', 'gi').

g - Global. Trouve toutes les correspondances au lieu de s’arrêter après la première.

i - Ignorer la casse. /[a-z]/i est équivalent à /[a-zA-Z]/.

m - Multiligne. ^ et $ correspondent au début et à la fin de chaque ligne en traitant respectivement \n et \r comme des délimiteurs au lieu de simplement le début et la fin de la chaîne entière.

u - Unicode. Si cet indicateur n’est pas pris en charge, vous devez faire correspondre des caractères Unicode spécifiques avec \uXXXX où XXXX est la valeur du caractère en hexadécimal.

y - Trouve toutes les correspondances consécutives/adjacentes.

Vérifiez si la chaîne contient un motif à l’aide de .test () #

var re = /[a-z]+/;
if (re.test("foo")) {
    console.log("Match exists.");
}

La méthode test effectue une recherche pour voir si une expression régulière correspond à une chaîne. L’expression régulière [a-z]+ recherchera une ou plusieurs lettres minuscules. Étant donné que le modèle correspond à la chaîne, “match existe” sera enregistré dans la console.

Correspondance avec .exec() #

Correspondance à l’aide de .exec() #

RegExp.prototype.exec(string) renvoie un tableau de captures, ou null s’il n’y avait pas de correspondance.

var re = /([0-9]+)[a-z]+/;
var match = re.exec("foo123bar");

match.index est 3, l’emplacement (basé sur zéro) de la correspondance.

match[0] est la chaîne de correspondance complète.

match[1] est le texte correspondant au premier groupe capturé. match[n] serait la valeur du nième groupe capturé.

Boucle à travers les correspondances à l’aide de .exec() #

var re = /a/g;
var result;
while ((result = re.exec('barbatbaz')) !== null) {
    console.log("found '" + result[0] + "', next exec starts at index '" + re.lastIndex + "'");
}

Production attendue

trouvé ‘a’, le prochain exec commence à l’index ‘2’ trouvé ‘a’, le prochain exec commence à l’index ‘5’ trouvé ‘a’, le prochain exec commence à l’index ‘8’

Groupes RegExp #

JavaScript prend en charge plusieurs types de groupe dans ses expressions régulières, capture groups, non-capture groups et look-aheads. Actuellement, il n’y a pas de support look-behind.

Capture #

Parfois, la correspondance souhaitée dépend de son contexte. Cela signifie qu’un simple RegExp sur-trouvera le morceau de String qui vous intéresse, donc la solution est d’écrire un groupe de capture (pattern). Les données capturées peuvent ensuite être référencées comme…

  • Remplacement de la chaîne "$n" où n est le n ième groupe de capture (à partir de 1)
  • Le n ième argument dans une fonction de rappel
  • Si le RegExp n’est pas marqué g, le n+1 e élément dans un str.match Array renvoyé
  • Si le RegExp est marqué g, str.match rejette les captures, utilisez re.exec à la place

Supposons qu’il existe une String où tous les signes “+” doivent être remplacés par un espace, mais uniquement s’ils suivent une lettre. Cela signifie qu’une correspondance simple inclurait ce caractère de lettre et qu’il serait également supprimé. La capturer est la solution car cela signifie que la lettre correspondante peut être conservée.

let str = "aa+b+cc+1+2",
    re = /([a-z])\+/g;

// String replacement
str.replace(re, '$1 '); // "aa b cc 1+2"
// Function replacement
str.replace(re, (m, $1) => $1 + ' '); // "aa b cc 1+2"

Non-capture #

En utilisant la forme (?:pattern), ceux-ci fonctionnent de la même manière pour capturer des groupes, sauf qu’ils ne stockent pas le contenu du groupe après la correspondance.

Ils peuvent être particulièrement utiles si d’autres données sont capturées dont vous ne voulez pas déplacer les indices, mais que vous devez faire une correspondance de modèle avancée telle qu’un OU

let str = "aa+b+cc+1+2",
    re = /(?:\b|c)([a-z])\+/g;

str.replace(re, '$1 '); // "aa+b c 1+2"

Prévision #

Si la correspondance souhaitée repose sur quelque chose qui la suit, plutôt que de la faire correspondre et de la capturer, il est possible d’utiliser une anticipation pour la tester mais ne pas l’inclure dans la correspondance. Une anticipation positive a la forme (?=pattern), une anticipation négative (où la correspondance d’expression ne se produit que si le modèle d’anticipation ne correspond pas) a la forme (?!pattern)

let str = "aa+b+cc+1+2",
    re = /\+(?=[a-z])/g;

str.replace(re, ' '); // "aa b cc+1+2"

Remplacement de la correspondance de chaîne par une fonction de rappel #

String#replace peut avoir une fonction comme deuxième argument afin que vous puissiez fournir un remplacement basé sur une certaine logique.

"Some string Some".replace(/Some/g, (match, startIndex, wholeString) => {
  if(startIndex == 0){
    return 'Start';
  } else {
    return 'End';
  }
});
// will return Start string End

Bibliothèque de modèles à une ligne

let data = {name: 'John', surname: 'Doe'}
"My name is {surname}, {name} {surname}".replace(/(?:{(.+?)})/g, x => data[x.slice(1,-1)]);

// "My name is Doe, John Doe"

Utilisation de Regex.exec() avec des parenthèses regex pour extraire les correspondances d’une chaîne #

Parfois, vous ne voulez pas simplement remplacer ou supprimer la chaîne. Parfois, vous souhaitez extraire et traiter des correspondances. Voici un exemple de la façon dont vous manipulez les correspondances.

Qu’est-ce qu’une correspondance ? Lorsqu’une sous-chaîne compatible est trouvée pour l’ensemble de la regex dans la chaîne, la commande exec produit une correspondance. Une correspondance est un tableau composé d’abord de la sous-chaîne entière qui correspond et de toutes les parenthèses de la correspondance.

Imaginez une chaîne html :

<html>
<head></head>
<body>
  <h1>Example</h1>
  <p>Look a this great link : <a href="https://stackoverflow.com">Stackoverflow</a> http://anotherlinkoutsideatag</p>
  Copyright <a href="https://stackoverflow.com">Stackoverflow</a>
</body>

Vous voulez extraire et obtenir tous les liens à l’intérieur d’une balise “a”. Dans un premier temps, voici la regex que vous écrivez :

var re = /<a[^>]*href="https?:\/\/.*"[^>]*>[^<]*<\/a>/g;

Mais maintenant, imaginez que vous vouliez le “href” et le “ancre” de chaque lien. Et vous le voulez ensemble. Vous pouvez simplement ajouter une nouvelle expression régulière pour chaque correspondance OU vous pouvez utiliser des parenthèses :

var re = /<a[^>]*href="(https?:\/\/.*)"[^>]*>([^<]*)<\/a>/g; 
var str = '<html>\n    <head></head>\n    <body>\n        <h1>Example</h1>\n        <p>Look a this great link : <a href="https://stackoverflow.com">Stackoverflow</a> http://anotherlinkoutsideatag</p>\n\n        Copyright <a href="https://stackoverflow.com">Stackoverflow</a>\n    </body>\';\n';
var m;
var links = [];

while ((m = re.exec(str)) !== null) {
    if (m.index === re.lastIndex) {
        re.lastIndex++;
    }
    console.log(m[0]); // The all substring
    console.log(m[1]); // The href subpart
    console.log(m[2]); // The anchor subpart

    links.push({
      match : m[0],   // the entire match
      href : m[1],    // the first parenthesis => (https?:\/\/.*)
      anchor : m[2],  // the second one => ([^<]*)
    });
}

A la fin de la boucle, vous avez un tableau de lien avec anchor et href et vous pouvez l’utiliser pour écrire du markdown par exemple :

links.forEach(function(link) {
  console.log('[%s](%s)', link.anchor, link.href);
});
