Exemples d’utilisation de rappel simple #

Les rappels offrent un moyen d’étendre les fonctionnalités d’une fonction (ou d’une méthode) sans changer son code. Cette approche est souvent utilisée dans les modules (bibliothèques/plugins) dont le code n’est pas censé être modifié.

Supposons que nous ayons écrit la fonction suivante, calculant la somme d’un tableau de valeurs donné :

function foo(array) {
    var sum = 0;
    for (var i = 0; i < array.length; i++) {
        sum += array[i];
    }
    return sum;
}

Supposons maintenant que nous voulions faire quelque chose avec chaque valeur du tableau, par ex. affichez-le en utilisant alert(). Nous pourrions faire les changements appropriés dans le code de foo, comme ceci :

function foo(array) {
    var sum = 0;
    for (var i = 0; i < array.length; i++) {
        alert(array[i]);
        sum += array[i];
    }
    return sum;
}

Mais que se passe-t-il si nous décidons d’utiliser console.log au lieu de alert() ? De toute évidence, changer le code de foo, chaque fois que nous décidons de faire autre chose avec chaque valeur, n’est pas une bonne idée. C’est bien mieux d’avoir la possibilité de changer d’avis sans changer le code de foo. C’est exactement le cas d’utilisation des rappels. Nous n’avons qu’à modifier légèrement la signature et le corps de foo :

function foo(array, callback) {
    var sum = 0;
    for (var i = 0; i < array.length; i++) {
        callback(array[i]);
        sum += array[i];
    }
    return sum;
}

Et maintenant, nous pouvons changer le comportement de foo simplement en changeant ses paramètres :

var array = [];
foo(array, alert);
foo(array, function (x) {
    console.log(x);
});

Exemples avec des fonctions asynchrones #

Dans jQuery, la méthode $.getJSON() pour récupérer les données JSON est asynchrone. Par conséquent, la transmission de code dans un rappel garantit que le code est appelé * après * que le JSON est récupéré.

Syntaxe $.getJSON() :

$.getJSON( url, dataObject, successCallback );

Exemple de code $.getJSON() :

$.getJSON("foo.json", {}, function(data) {
    // data handling code
});

Ce qui suit ne fonctionnerait pas, car le code de traitement des données serait probablement appelé avant que les données ne soient réellement reçues, car la fonction $.getJSON prend une durée indéterminée et ne retient pas la pile d’appels comme il attend le JSON.

$.getJSON("foo.json", {});
// data handling code

Un autre exemple de fonction asynchrone est la fonction animate() de jQuery. Étant donné que l’exécution de l’animation prend un certain temps, il est parfois souhaitable d’exécuter du code directement après l’animation.

Syntaxe .animate() :

jQueryElement.animate( properties, duration, callback );

Par exemple, pour créer une animation de fondu après quoi l’élément disparaît complètement, le code suivant peut être exécuté. Notez l’utilisation du rappel.

elem.animate( { opacity: 0 }, 5000, function() {
    elem.hide();
} );

Cela permet de masquer l’élément juste après la fin de l’exécution de la fonction. Cela diffère de :

elem.animate( { opacity: 0 }, 5000 );
elem.hide();

car ce dernier n’attend pas que animate() (une fonction asynchrone) se termine, et donc l’élément est caché tout de suite, produisant un effet indésirable.

Qu’est-ce qu’un rappel ? #

Ceci est un appel de fonction normal :

console.log("Hello World!");

Lorsque vous appelez une fonction normale, elle fait son travail puis rend le contrôle à l’appelant.

Cependant, une fonction doit parfois rendre le contrôle à l’appelant pour faire son travail :

[1,2,3].map(function double(x) {
    return 2 * x;
});

Dans l’exemple ci-dessus, la fonction “double” est un rappel pour la fonction “map” car :

  1. La fonction ‘double’ est donnée à la fonction ‘map’ par l’appelant.
  2. La fonction map doit appeler la fonction double zéro ou plusieurs fois pour faire son travail.

Ainsi, la fonction « map » rend essentiellement le contrôle à l’appelant chaque fois qu’il appelle la fonction « double ». D’où le nom “rappel”.

Les fonctions peuvent accepter plusieurs rappels :

promise.then(function onFulfilled(value) {
    console.log("Fulfilled with value " + value);
}, function onRejected(reason) {
    console.log("Rejected with reason " + reason);
});

Ici, la fonction then accepte deux fonctions de rappel, onFulfilled et onRejected. De plus, une seule de ces deux fonctions de rappel est réellement appelée.

Ce qui est plus intéressant, c’est que la fonction then revient avant que l’un ou l’autre des rappels ne soit appelé. Par conséquent, une fonction de rappel peut être appelée même après le retour de la fonction d’origine.

Suite (synchrone et asynchrone) #

Les rappels peuvent être utilisés pour fournir du code à exécuter après la fin d’une méthode :

/**
 * @arg {Function} then continuation callback
 */
function doSomething(then) {
  console.log('Doing something');
  then();
}

// Do something, then execute callback to log 'done'
doSomething(function () {
  console.log('Done');
});

console.log('Doing something else');

// Outputs:
//   "Doing something"
//   "Done"
//   "Doing something else"

La méthode doSomething() ci-dessus s’exécute de manière synchrone avec le rappel - les blocs d’exécution jusqu’au retour de doSomething(), garantissant que le rappel est exécuté avant que l’interpréteur ne passe à autre chose.

Les rappels peuvent également être utilisés pour exécuter du code de manière asynchrone :

doSomethingAsync(then) {
  setTimeout(then, 1000);
  console.log('Doing something asynchronously');
}

doSomethingAsync(function() {
  console.log('Done');
});

console.log('Doing something else');

// Outputs:
//   "Doing something asynchronously"
//   "Doing something else"
//   "Done"

Les rappels then sont considérés comme des continuations des méthodes doSomething(). Fournir un rappel en tant que dernière instruction d’une fonction s’appelle un tail-call, qui est [optimisé par les interpréteurs ES2015](http://www.2ality .com/2015/06/tail-call-optimization.html).

Rappels et this #

Souvent, lorsque vous utilisez un rappel, vous souhaitez accéder à un contexte spécifique.

function SomeClass(msg, elem) {
  this.msg = msg;
  elem.addEventListener('click', function() {
    console.log(this.msg);  // <= will fail because "this" is undefined
  });
}

var s = new SomeClass("hello", someElement);

Solutions #

  • Utilisez lier

bind génère effectivement une nouvelle fonction qui définit this sur n’importe quel a été passé à bind puis appelle la fonction d’origine.

    function SomeClass(msg, elem) {
      this.msg = msg;
      elem.addEventListener('click', function() {
        console.log(this.msg);  
      }.bind(this));  // <=-  bind the function to `this`
    }
  • Utiliser les fonctions fléchées

Les fonctions fléchées lient automatiquement le contexte this actuel.

    function SomeClass(msg, elem) {
      this.msg = msg;
      elem.addEventListener('click',() => {   // <=-  arrow function binds `this`
        console.log(this.msg);  
      });
    }

Souvent, vous souhaitez appeler une fonction membre, en transmettant idéalement tous les arguments transmis à l’événement à la fonction.

Solutions: #

  • Utiliser la liaison

      function SomeClass(msg, elem) {
    this.msg = msg;
    elem.addEventListener('click', this.handleClick.bind(this));
  }

  SomeClass.prototype.handleClick = function(event) {
    console.log(event.type, this.msg);
  };

  • Utilisez les fonctions fléchées et l’opérateur de repos

      function SomeClass(msg, elem) {
    this.msg = msg;
    elem.addEventListener('click', (...a) => this.handleClick(...a));
  }

  SomeClass.prototype.handleClick = function(event) {
    console.log(event.type, this.msg);
  };

  • Pour les écouteurs d’événements DOM en particulier, vous pouvez implémenter l’interface EventListener

      function SomeClass(msg, elem) {
    this.msg = msg;
    elem.addEventListener('click', this);
  }

  SomeClass.prototype.handleEvent = function(event) {
    var fn = this[event.type];
    if (fn) {
       fn.apply(this, arguments);
    }
  };

  SomeClass.prototype.click = function(event) {
    console.log(this.msg);
  };

Gestion des erreurs et branchement du flux de contrôle #

Les rappels sont souvent utilisés pour assurer la gestion des erreurs. Il s’agit d’une forme de branchement de flux de contrôle, où certaines instructions ne sont exécutées que lorsqu’une erreur se produit :

const expected = true;

function compare(actual, success, failure) {
  if (actual === expected) {
    success();
  } else {
    failure();
  }
}

function onSuccess() {
  console.log('Value was expected');
}

function onFailure() {
  console.log('Value was unexpected/exceptional');
}

compare(true, onSuccess, onFailure);
compare(false, onSuccess, onFailure);

// Outputs:
//   "Value was expected"
//   "Value was unexpected/exceptional"

L’exécution du code dans compare() ci-dessus a deux branches possibles : success lorsque les valeurs attendues et réelles sont identiques, et error lorsqu’elles sont différentes. Ceci est particulièrement utile lorsque le flux de contrôle doit se ramifier après une instruction asynchrone :

function compareAsync(actual, success, failure) {
  setTimeout(function () {
    compare(actual, success, failure)
  }, 1000);
}

compareAsync(true, onSuccess, onFailure);
compareAsync(false, onSuccess, onFailure);
console.log('Doing something else');

// Outputs:
//   "Doing something else"
//   "Value was expected"
//   "Value was unexpected/exceptional"

Il convient de noter que plusieurs rappels ne doivent pas nécessairement s’exclure mutuellement : les deux méthodes peuvent être appelées. De même, le compare() pourrait être écrit avec des rappels facultatifs (en utilisant un noop comme valeur par défaut - voir Null Object pattern).

Rappel à l’aide de la fonction Flèche #

** L’utilisation de la fonction fléchée comme fonction de rappel peut réduire les lignes de code. **

La syntaxe par défaut de la fonction flèche est

() => {}

Cela peut être utilisé comme rappels

Par exemple, si nous voulons imprimer tous les éléments d’un tableau [1,2,3,4,5]

sans fonction flèche, le code ressemblera à ceci

[1,2,3,4,5].forEach(function(x){
                 console.log(x);
            }

Avec la fonction flèche, il peut être réduit à

[1,2,3,4,5].forEach(x => console.log(x));

Ici, la fonction de rappel function(x){console.log(x)} est réduite à x=>console.log(x)
