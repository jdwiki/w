La déstructuration est une technique de pattern matching qui a été récemment ajoutée à Javascript dans EcmaScript 6.

Il vous permet de lier un groupe de variables à un ensemble de valeurs correspondant lorsque leur modèle correspond au côté droit et au côté gauche de l’expression.

Syntaxe #

  • soit [x, y] = [1, 2]
  • soit [premier, … reste] = [1, 2, 3, 4]
  • soit [un, , trois] = [1, 2, 3]
  • soit [val=‘valeur par défaut’] = []
  • soit {a, b} = {a : x, b : y}
  • Soit {a : {c}} = {a : {c : ‘imbriqué’}, b : y}
  • soit {b=‘valeur par défaut’} = {a : 0}

La déstructuration est nouvelle dans la spécification ECMAScript 6 (A.K.A ES2015) et [la prise en charge du navigateur][1] peut être limitée. Le tableau suivant donne un aperçu de la première version des navigateurs prenant en charge > 75 % de la spécification.

Chrome Bord Firefox Internet Explorer Opéra safari
49 13 45 x 36 x

(Dernière mise à jour - 2016/08/18)

Déstructuration d’objets #

La déstructuration est un moyen pratique d’extraire des propriétés d’objets dans des variables.

Syntaxe de base :

let person = {
  name: 'Bob',
  age: 25
};

let { name, age } = person;

// Is equivalent to
let name = person.name; // 'Bob'
let age = person.age;   // 25

Déstructuration et renommage :

let person = {
  name: 'Bob',
  age: 25
};

let { name: firstName } = person;

// Is equivalent to
let firstName = person.name; // 'Bob'

Déstructuration avec des valeurs par défaut :

let person = {
  name: 'Bob',
  age: 25
};

let { phone = '123-456-789' } = person;

// Is equivalent to
let phone = person.hasOwnProperty('phone') ? person.phone : '123-456-789'; // '123-456-789'

Déstructuration et renommage avec des valeurs par défaut

let person = {
  name: 'Bob',
  age: 25
};

let { phone: p = '123-456-789' } = person;

// Is equivalent to
let p = person.hasOwnProperty('phone') ? person.phone : '123-456-789'; // '123-456-789'

Déstructuration des arguments de la fonction #

Extraire les propriétés d’un objet passé dans une fonction. Ce modèle simule des paramètres nommés au lieu de s’appuyer sur la position de l’argument.

let user = {
    name: 'Jill',
    age: 33,
    profession: 'Pilot'
}    

function greeting ({name, profession}) {
    console.log(`Hello, ${name} the ${profession}`)
}

greeting(user)

Cela fonctionne également pour les tableaux :

let parts = ["Hello", "World!"];

function greeting([first, second]) {
    console.log(`${first} ${second}`);
}

Déstructuration imbriquée #

Nous ne sommes pas limités à déstructurer un objet/tableau, nous pouvons déstructurer un objet/tableau imbriqué.

Déstructuration d’objets imbriqués

var obj = {
  a: {
    c: 1,
    d: 3
  },
  b: 2
};

var {
  a: {
    c: x,
    d: y
  },
  b: z
} = obj;

console.log(x, y, z);     // 1,3,2

Déstructuration de tableaux imbriqués

var arr = [1, 2, [3, 4], 5];

var [a, , [b, c], d] = arr;

console.log(a, b, c, d);      // 1 3 4 5

La déstructuration ne se limite pas à un seul motif, nous pouvons y avoir des tableaux, avec n niveaux d’imbrication. De même, nous pouvons déstructurer des tableaux avec des objets et vice-versa.

*** Tableaux dans l’objet ***

var obj = {
  a: 1,
  b: [2, 3]
};

var {
  a: x1,
  b: [x2, x3]
} = obj;

console.log(x1, x2, x3);    // 1 2 3

Objets dans les tableaux

var arr = [1, 2 , {a : 3}, 4];

var [x1, x2 , {a : x3}, x4] = arr;

console.log(x1, x2, x3, x4);

Déstructuration des tableaux #

const myArr = ['one', 'two', 'three']
const [ a, b, c ] = myArr

// a = 'one', b = 'two, c = 'three'

Nous pouvons définir la valeur par défaut dans le tableau de déstructuration, voir l’exemple de [Valeur par défaut lors de la déstructuration] (https://www.wikiod.com/fr/javascript/mission-de-destructuration#Valeur par défaut lors de la déstructuration).

Avec la déstructuration du tableau, nous pouvons facilement échanger les valeurs de 2 variables :

var a = 1;
var b = 3;

[a, b] = [b, a];
// a = 3, b = 1

Nous pouvons spécifier des emplacements vides pour ignorer les valeurs inutiles :

[a, , b] = [1, 2, 3] // a = 1, b = 3

Renommer des variables lors de la déstructuration #

La déstructuration nous permet de faire référence à une clé dans un objet, mais de la déclarer comme une variable avec un nom différent. La syntaxe ressemble à la syntaxe clé-valeur d’un objet JavaScript normal.

let user = {
  name: 'John Smith',
  id: 10,
  email: 'johns@workcorp.com',
};

let {user: userName, id: userId} = user;

console.log(userName) // John Smith
console.log(userId) // 10

Déstructuration à l’intérieur des variables #

Outre la déstructuration d’objets en arguments de fonction, vous pouvez les utiliser dans des déclarations de variables comme suit :

const person = {
  name: 'John Doe',
  age: 45,
  location: 'Paris, France',
};

let { name, age, location } = person;

console.log('I am ' + name + ', aged ' + age + ' and living in ' + location + '.');
// -> "I am John Doe aged 45 and living in Paris, France."

Comme vous pouvez le voir, trois nouvelles variables ont été créées : name, age et location et leurs valeurs ont été extraites de l’objet person si elles correspondaient aux noms de clé.

Valeur par défaut lors de la déstructuration #

Nous rencontrons souvent une situation où une propriété que nous essayons d’extraire n’existe pas dans l’objet/le tableau, ce qui entraîne une TypeError (lors de la déstructuration des objets imbriqués) ou est défini sur undefined. Lors de la déstructuration, nous pouvons définir une valeur par défaut, sur laquelle il se repliera, au cas où il ne serait pas trouvé dans l’objet.

var obj = {a : 1};
var {a : x , b : x1 = 10} = obj;
console.log(x, x1); // 1, 10
 
var arr = [];
var [a = 5, b = 10, c] = arr;
console.log(a, b, c); // 5, 10, undefined

Utilisation des paramètres de repos pour créer un tableau d’arguments #

Si jamais vous avez besoin d’un tableau composé d’arguments supplémentaires que vous pouvez ou non espérer avoir, en dehors de ceux que vous avez spécifiquement déclarés, vous pouvez utiliser le paramètre array rest dans la déclaration d’arguments comme suit :

Exemple 1, arguments facultatifs dans un tableau :

function printArgs(arg1, arg2, ...theRest) {
  console.log(arg1, arg2, theRest);
}

printArgs(1, 2, 'optional', 4, 5);
// -> "1, 2, ['optional', 4, 5]"

Exemple 2, tous les arguments sont maintenant un tableau :

function printArgs(...myArguments) {
  console.log(myArguments, Array.isArray(myArguments));
}

printArgs(1, 2, 'Arg #3');
// -> "[1, 2, 'Arg #3'] true"

La console a imprimé vrai parce que myArguments est un tableau, aussi, le ...myArguments à l’intérieur de la déclaration des arguments des paramètres convertit une liste de valeurs obtenues par la fonction (paramètres) séparées par des virgules en un tableau entièrement fonctionnel (et non un Objet de type tableau comme l’objet d’arguments natif).
