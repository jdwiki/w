Séquences d'échappement

  • Saisie de caractères spéciaux dans les chaînes et les expressions régulières
  • Types de séquences d’échappement

Sur cette page

  • Saisie de caractères spéciaux dans les chaînes et les expressions régulières
  • Types de séquences d’échappement

Tout ce qui commence par une barre oblique inverse n’est pas une séquence d’échappement. De nombreux caractères ne sont tout simplement pas utiles pour échapper des séquences et entraîneront simplement l’ignorance d’une barre oblique inverse précédente.

"\H\e\l\l\o" === "Hello" // true

D’autre part, certains caractères comme “u” et “x” provoqueront une erreur de syntaxe s’ils sont utilisés de manière incorrecte après une barre oblique inverse. Ce qui suit n’est pas un littéral de chaîne valide car il contient le préfixe de séquence d’échappement Unicode \u suivi d’un caractère qui n’est pas un chiffre hexadécimal valide ni une accolade :

"C:\Windows\System32\updatehandlers.dll" // SyntaxError

Une barre oblique inverse à la fin d’une ligne à l’intérieur d’une chaîne n’introduit pas de séquence d’échappement, mais indique la continuation de la ligne, c’est-à-dire

"contin\
uation" === "continuation" // true

Similitude avec d’autres formats #

Alors que les séquences d’échappement en JavaScript ressemblent à d’autres langages et formats, comme C++, Java, JSON, etc., il y aura souvent des différences critiques dans les détails. En cas de doute, assurez-vous de tester que votre code se comporte comme prévu et envisagez de vérifier la spécification du langage.

Saisie de caractères spéciaux dans les chaînes et les expressions régulières #

La plupart des caractères imprimables peuvent être inclus dans des littéraux de chaîne ou d’expression régulière tels quels, par ex.

var str = "ポケモン"; // a valid string
var regExp = /[Α-Ωα-ω]/; // matches any Greek letter without diacritics

Afin d’ajouter des caractères arbitraires à une chaîne ou à une expression régulière, y compris des caractères non imprimables, il faut utiliser des séquences d’échappement. Les séquences d’échappement consistent en une barre oblique inverse ("\") suivie d’un ou plusieurs autres caractères. Pour écrire une séquence d’échappement pour un caractère particulier, il faut généralement (mais pas toujours) connaître son [code de caractère] hexadécimal (https://www.wikiod.com/fr/javascript/cordes#Code de caractère =201608112028595843342).

JavaScript fournit un certain nombre de façons différentes de spécifier des séquences d’échappement, comme documenté dans les exemples de cette rubrique. Par exemple, les séquences d’échappement suivantes désignent toutes le même caractère : le saut de ligne (caractère de nouvelle ligne Unix), avec le code de caractère U+000A.

  • \n
  • \x0a
  • \u000a
  • \u{a} nouveau dans ES6, uniquement dans les chaînes
  • \012 interdit dans les littéraux de chaîne en mode strict et dans les chaînes de modèle
  • \cj uniquement dans les expressions régulières

Types de séquences d’échappement #

Séquences d’échappement à caractère unique #

Certaines séquences d’échappement consistent en une barre oblique inverse suivie d’un seul caractère.

Par exemple, dans alert("Hello\nWorld");, la séquence d’échappement \n est utilisée pour introduire une nouvelle ligne dans le paramètre de chaîne, de sorte que les mots “Hello” et “World” s’affichent sur des lignes consécutives .

Séquence d’échappement Caractère Unicode
\b (uniquement dans les chaînes, pas dans les expressions régulières) retour arrière U+0008
\t tabulation horizontale U+0009
\n saut de ligne U+000A
\v onglet vertical U+000B
\f saut de formulaire U+000C
\r retour chariot U+000D

De plus, la séquence \0, lorsqu’elle n’est pas suivie d’un chiffre compris entre 0 et 7, peut être utilisée pour échapper au caractère nul (U+0000).

Les séquences \\, \' et \" sont utilisées pour échapper le caractère qui suit la barre oblique inverse. Bien que similaires aux séquences sans échappement, où la barre oblique inverse principale est simplement ignorée (c’est-à-dire \? pour ?), elles sont explicitement traitées comme des séquences d’échappement à caractère unique à l’intérieur des chaînes conformément à la spécification.

Séquences d’échappement hexadécimales #

Les caractères avec des codes entre 0 et 255 peuvent être représentés par une séquence d’échappement où \x est suivi du code de caractère hexadécimal à 2 chiffres. Par exemple, le caractère espace insécable a le code 160 ou A0 en base 16, et il peut donc être écrit comme \xa0.

var str = "ONE\xa0LINE"; // ONE and LINE with a non-breaking space between them

Pour les chiffres hexadécimaux supérieurs à 9, les lettres a à f sont utilisées, en minuscules ou en majuscules sans distinction.

var regExp1 = /[\x00-xff]/; // matches any character between U+0000 and U+00FF
var regExp2 = /[\x00-xFF]/; // same as above

Séquences d’échappement Unicode à 4 chiffres #

Les caractères avec des codes entre 0 et 65535 (216 - 1) peuvent être représentés avec une séquence d’échappement où \u est suivi du code de caractère hexadécimal à 4 chiffres.

Par exemple, la norme Unicode définit le caractère flèche droite (“→”) avec le nombre 8594, ou 2192 au format hexadécimal. Donc, une séquence d’échappement serait \u2192.

Cela produit la chaîne “A → B”:

var str = "A \u2192 B";

Pour les chiffres hexadécimaux supérieurs à 9, les lettres a à f sont utilisées, en minuscules ou en majuscules sans distinction. Les codes hexadécimaux inférieurs à 4 chiffres doivent être complétés à gauche par des zéros : \u007A pour la lettre minuscule “z”.

Séquences d’échappement Unicode entre accolades #

ES6 étend la prise en charge d’Unicode à toute la plage de codes de 0 à 0x10FFFF. Afin d’échapper les caractères avec un code supérieur à 216 - 1, une nouvelle syntaxe pour les séquences d’échappement a été introduite :

\u{???}

Où le code entre accolades est une représentation hexadécimale de la valeur du point de code, par ex.

alert("Look! \u{1f440}"); // Look! 👀

Dans l’exemple ci-dessus, le code 1f440 est la représentation hexadécimale du code de caractère du caractère Unicode Eyes.

Notez que le code entre accolades peut contenir n’importe quel nombre de chiffres hexadécimaux, tant que la valeur ne dépasse pas 0x10FFFF. Pour les chiffres hexadécimaux supérieurs à 9, les lettres a à f sont utilisées, en minuscules ou en majuscules sans distinction.

Les séquences d’échappement Unicode avec des accolades ne fonctionnent qu’à l’intérieur des chaînes, pas à l’intérieur des expressions régulières !

Séquences d’échappement octales #

Les séquences d’échappement octales sont obsolètes depuis ES5, mais elles sont toujours prises en charge dans les expressions régulières et en mode non strict également dans les chaînes non modèles. Une séquence d’échappement octale se compose d’un, deux ou trois chiffres octaux, avec une valeur comprise entre 0 et 3778 = 255.

Par exemple, la lettre majuscule “E” a le code de caractère 69, ou 105 en base 8. Elle peut donc être représentée avec la séquence d’échappement \105 :

/\105scape/.test("Fun with Escape Sequences"); // true

En mode strict, les séquences d’échappement octales ne sont pas autorisées à l’intérieur des chaînes et produiront une erreur de syntaxe. Il convient de noter que \0, contrairement à \00 ou \000, n’est pas considéré comme une séquence d’échappement octale, et est donc toujours autorisé à l’intérieur des chaînes (même les chaînes de modèle) en mode strict.

Contrôler les séquences d’échappement #

Certaines séquences d’échappement ne sont reconnues qu’à l’intérieur des littéraux d’expressions régulières (pas dans les chaînes). Ceux-ci peuvent être utilisés pour échapper les caractères avec des codes entre 1 et 26 (U+0001–U+001A). Ils se composent d’une seule lettre A à Z (la casse ne fait aucune différence) précédée de \c. La position alphabétique de la lettre après \c détermine le code du caractère.

Par exemple, dans l’expression régulière

`/\cG/`

La lettre “G” (la 7ème lettre de l’alphabet) fait référence au caractère U+0007, et donc

`/\cG`/.test(String.fromCharCode(7)); // true
