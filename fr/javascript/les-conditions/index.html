Les expressions conditionnelles, impliquant des mots clés tels que if et else, permettent aux programmes JavaScript d’effectuer différentes actions en fonction d’une condition booléenne : vrai ou faux. Cette section couvre l’utilisation des conditions JavaScript, de la logique booléenne et des instructions ternaires.

Syntaxe #

  • si (condition) instruction ;
  • si (condition) instruction_1, instruction_2, …, instruction_n ;
  • si (état) {   déclaration }
  • si (état) {   instruction_1 ;   instruction_2 ;   …   instruction_n ; }
  • si (état) {   déclaration } autre {   déclaration }
  • si (état) {   déclaration } sinon si (condition) {   déclaration } autre {   déclaration }
  • interrupteur (expression) { cas valeur1 :   déclaration   [pause;] cas valeur2 :   déclaration   [pause;] cas valeurN :   déclaration   [pause;] défaut:   déclaration   [pause;] }
  • condition ? value_for_true : value_for_false ;

Les conditions peuvent interrompre le déroulement normal du programme en exécutant du code basé sur la valeur d’une expression. En JavaScript, cela signifie utiliser les instructions if, else if et else et les opérateurs ternaires.

Opérateurs ternaires #

Peut être utilisé pour raccourcir les opérations if/else. Ceci est pratique pour renvoyer rapidement une valeur (c’est-à-dire pour l’affecter à une autre variable).

Par exemple:

var animal = 'kitty';
var result = (animal === 'kitty') ? 'cute' : 'still nice';

Dans ce cas, result obtient la valeur ‘cute’, car la valeur de animal est ‘kitty’. Si l’animal avait une autre valeur, le résultat obtiendrait la valeur “toujours agréable”.

Comparez cela à ce que le code voudrait avec les conditions if/else.

var animal = 'kitty';
var result = '';
if (animal === 'kitty') {
    result = 'cute';
} else {
    result = 'still nice';
}

Les conditions if ou else peuvent avoir plusieurs opérations. Dans ce cas, l’opérateur renvoie le résultat de la dernière expression.

var a = 0;
var str = 'not a';
var b = '';
b = a === 0 ? (a = 1, str += ' test') : (a = 2);

Comme a était égal à 0, il devient 1 et str devient ’not a test’. L’opération qui impliquait str était la dernière, donc b reçoit le résultat de l’opération, qui est la valeur contenue dans str, c’est-à-dire ‘pas un test’.

Les opérateurs ternaires attendent toujours des conditions else, sinon vous obtiendrez une erreur de syntaxe. Comme solution de contournement, vous pouvez renvoyer un zéro quelque chose de similaire dans la branche else - cela n’a pas d’importance si vous n’utilisez pas la valeur de retour mais que vous raccourcissez (ou essayez de raccourcir) l’opération.

var a = 1;
a === 1 ? alert('Hey, it is 1!') : 0;

Comme vous le voyez, if (a === 1) alert('Hey, it is 1!'); ferait la même chose. Ce serait juste un caractère plus long, car il n’a pas besoin d’une condition obligatoire “else”. Si une condition “autre” était impliquée, la méthode ternaire serait beaucoup plus propre.

a === 1 ? alert('Hey, it is 1!') : alert('Weird, what could it be?');
if (a === 1) alert('Hey, it is 1!') else alert('Weird, what could it be?');

Les ternaires peuvent être imbriqués pour encapsuler une logique supplémentaire. Par exemple

foo ? bar ? 1 : 2 : 3

// To be clear, this is evaluated left to right 
// and can be more explicitly expressed as:

foo ? (bar ? 1 : 2) : 3

C’est la même chose que le “if/else” suivant

if (foo) {
  if (bar) {
    1
  } else {
    2
  }
} else {
  3
}

D’un point de vue stylistique, cela ne devrait être utilisé qu’avec des noms de variables courts, car les ternaires multilignes peuvent réduire considérablement la lisibilité.

Les seules instructions qui ne peuvent pas être utilisées dans les ternaires sont les instructions de contrôle. Par exemple, vous ne pouvez pas utiliser return ou break avec des ternaires. L’expression suivante sera invalide.

var animal = 'kitty';
for (var i = 0; i < 5; ++i) {
    (animal === 'kitty') ? break:console.log(i);
}

Pour les instructions de retour, les éléments suivants seraient également invalides :

var animal = 'kitty';
(animal === 'kitty') ? return 'meow' : return 'woof';

Pour faire ce qui précède correctement, vous renverriez le ternaire comme suit :

var animal = 'kitty';
return (animal === 'kitty') ? 'meow' : 'woof';

Instruction de commutation #

Les instructions Switch comparent la valeur d’une expression à 1 ou plusieurs valeurs et exécutent différentes sections de code en fonction de cette comparaison.

var value = 1;
switch (value) {
  case 1:
    console.log('I will always run');
    break;
  case 2:
    console.log('I will never run');
    break;
}

L’instruction break “interrompt” l’instruction switch et s’assure qu’aucun autre code dans l’instruction switch n’est exécuté. C’est ainsi que les sections sont définies et permettent à l’utilisateur de faire des cas “échoués”.

Avertissement : l’absence d’instruction break ou return pour chaque cas signifie que le programme continuera à évaluer le cas suivant, même si les critères du cas ne sont pas remplis !

switch (value) {
  case 1:
    console.log('I will only run if value === 1');
    // Here, the code "falls through" and will run the code under case 2
  case 2:
    console.log('I will run if value === 1 or value === 2');
    break;
  case 3:
    console.log('I will only run if value === 3');
    break;
}

Le dernier cas est le cas “par défaut”. Celui-ci fonctionnera si aucun autre match n’a été fait.

var animal = 'Lion';
switch (animal) {
  case 'Dog':
    console.log('I will not run since animal !== "Dog"');
    break;
  case 'Cat':
    console.log('I will not run since animal !== "Cat"');
    break;
  default:
    console.log('I will run since animal does not match any other case');
}

Il convient de noter qu’une expression de cas peut être n’importe quel type d’expression. Cela signifie que vous pouvez utiliser des comparaisons, des appels de fonction, etc. comme valeurs de cas.

function john() {
  return 'John';
}

function jacob() {
  return 'Jacob';
}

switch (name) {
  case john(): // Compare name with the return value of john() (name == "John")
    console.log('I will run if name === "John"');
    break;
  case 'Ja' + 'ne': // Concatenate the strings together then compare (name == "Jane")
    console.log('I will run if name === "Jane"');
    break;
  case john() + ' ' + jacob() + ' Jingleheimer Schmidt':
    console.log('His name is equal to name too!');
    break;
}

Critères inclusifs multiples pour les cas #

Étant donné que les cas “échouent” sans instruction break ou return, vous pouvez l’utiliser pour créer plusieurs critères inclusifs :

var x = "c"
switch (x) {
   case "a":
   case "b":
   case "c":
      console.log("Either a, b, or c was selected.");
      break;
   case "d":
      console.log("Only d was selected.");
      break;
   default:
      console.log("No case was matched.");
      break;  // precautionary break if case order changes
}

Si / Sinon Si / Sinon Contrôle #

Dans sa forme la plus simple, une condition “si” peut être utilisée comme ceci :

var i = 0;

if (i < 1) {
    console.log("i is smaller than 1");
}

La condition “i < 1” est évaluée, et si elle est évaluée à “vrai”, le bloc qui suit est exécuté. S’il est évalué à “faux”, le bloc est ignoré.

Une condition if peut être étendue avec un bloc else. La condition est vérifiée une fois comme ci-dessus, et si elle est évaluée à “faux”, un bloc secondaire sera exécuté (qui serait ignoré si la condition était “vraie”). Un exemple:

if (i < 1) {
    console.log("i is smaller than 1");
} else {
    console.log("i was not smaller than 1");
}

Supposons que le bloc else ne contienne rien d’autre qu’un autre bloc if (avec éventuellement un bloc else) comme ceci :

if (i < 1) {
    console.log("i is smaller than 1");
} else {
    if (i < 2) {
        console.log("i is smaller than 2");
    } else {
        console.log("none of the previous conditions was true");
    }
}

Ensuite, il y a aussi une autre façon d’écrire ceci qui réduit l’imbrication :

if (i < 1) {
    console.log("i is smaller than 1");
} else if (i < 2) {
    console.log("i is smaller than 2");
} else {
    console.log("none of the previous conditions was true");
}

Quelques notes de bas de page importantes sur les exemples ci-dessus :

  • Si une condition est évaluée à “true”, aucune autre condition de cette chaîne de blocs ne sera évaluée et tous les blocs correspondants (y compris le bloc “else”) ne seront pas exécutés.

  • Le nombre de parties else if est pratiquement illimité. Le dernier exemple ci-dessus n’en contient qu’un, mais vous pouvez en avoir autant que vous le souhaitez.

  • La condition à l’intérieur d’une instruction if peut être tout ce qui peut être contraint à une valeur booléenne, voir le sujet sur boolean logical pour plus de détails;

  • L’échelle if-else-if se termine au premier succès. Autrement dit, dans l’exemple ci-dessus, si la valeur de “i” est de 0,5, la première branche est exécutée. Si les conditions se chevauchent, le premier critère apparaissant dans le flux d’exécution est exécuté. L’autre condition, qui pourrait également être vraie, est ignorée.

  • Si vous n’avez qu’une seule déclaration, les accolades autour de cette déclaration sont techniquement facultatives, par exemple, c’est bien :

    if (i < 1) console.log("i is smaller than 1");

Et cela fonctionnera aussi :

  if (i < 1) 
      console.log("i is smaller than 1");

Si vous souhaitez exécuter plusieurs instructions à l’intérieur d’un bloc if, les accolades autour d’eux sont obligatoires. Utiliser uniquement l’indentation ne suffit pas. Par exemple, le code suivant :

  if (i < 1) 
      console.log("i is smaller than 1");
      console.log("this will run REGARDLESS of the condition"); // Warning, see text!

est équivalent à:

  if (i < 1) {
      console.log("i is smaller than 1");
  }
  console.log("this will run REGARDLESS of the condition");

[1] : https://www.wikiod.com/fr/javascript/operations-de-comparaison

Stratégie #

Un modèle de stratégie peut être utilisé en Javascript dans de nombreux cas pour remplacer une instruction switch. Il est particulièrement utile lorsque le nombre de conditions est dynamique ou très important. Il permet au code de chaque condition d’être indépendant et testable séparément.

L’objet de stratégie est un simple objet avec plusieurs fonctions, représentant chaque condition distincte. Exemple:

const AnimalSays = {
    dog () {
        return 'woof';
    },

    cat () {
        return 'meow';
    },

    lion () {
        return 'roar';
    },

    // ... other animals

    default () {
        return 'moo';
    }
};

L’objet ci-dessus peut être utilisé comme suit :

function makeAnimalSpeak (animal) {
    // Match the animal by type
    const speak = AnimalSays[animal] || AnimalSays.default;
    console.log(animal + ' says ' + speak());
}

Résultats:

makeAnimalSpeak('dog') // => 'dog says woof'
makeAnimalSpeak('cat') // => 'cat says meow'
makeAnimalSpeak('lion') // => 'lion says roar'
makeAnimalSpeak('snake') // => 'snake says moo'

Dans le dernier cas, notre fonction par défaut gère tous les animaux manquants.

Utilisation de || et && court-circuit #

Les opérateurs booléens || et && “court-circuiteront” et n’évalueront pas le deuxième paramètre si le premier est respectivement vrai ou faux. Cela peut être utilisé pour écrire des conditionnels courts comme :

var x = 10

x == 10 && alert("x is 10")
x == 10 || alert("x is not 10")
