Variables d’interprétation interactives #

Notez que ceux-ci ne fonctionnent que dans les outils de développement de certains navigateurs.

$_ vous donne la valeur de la dernière expression évaluée.

"foo"             // "foo"
$_                // "foo"

$0 fait référence à l’élément DOM actuellement sélectionné dans l’inspecteur. Donc, si <div id="foo"> est en surbrillance :

$0                      // <div id="foo">
$0.getAttribute('id')   // "foo"

$1 fait référence à l’élément précédemment sélectionné, $2 à celui sélectionné auparavant, et ainsi de suite pour $3 et $4.

Pour obtenir une collection d’éléments correspondant à un sélecteur CSS, utilisez $$(selector). Il s’agit essentiellement d’un raccourci pour document.querySelectorAll.

var images = $$('img');  // Returns an array or a nodelist of all matching elements

Points d’arrêt #

Les points d’arrêt interrompent votre programme une fois que l’exécution atteint un certain point. Vous pouvez ensuite parcourir le programme ligne par ligne, en observant son exécution et en inspectant le contenu de vos variables.

Il existe trois façons de créer des points d’arrêt.

  1. À partir du code, en utilisant l’instruction debugger;.
  2. À partir du navigateur, à l’aide des outils de développement.
  3. À partir d’un environnement de développement intégré (IDE).

Déclaration de débogage #

Vous pouvez placer une instruction debugger; n’importe où dans votre code JavaScript. Une fois que l’interpréteur JS atteint cette ligne, il arrête l’exécution du script, vous permettant d’inspecter les variables et de parcourir votre code.

Outils de développement #

La deuxième option consiste à ajouter un point d’arrêt directement dans le code à partir des outils de développement du navigateur.

Ouverture des outils de développement #

Chrome ou Firefox #

  1. Appuyez sur F12 pour ouvrir les outils de développement
  2. Basculez vers l’onglet Sources (Chrome) ou l’onglet Débogueur (Firefox)
  3. Appuyez sur Ctrl+P et tapez le nom de votre fichier JavaScript
  4. Appuyez sur Entrée pour l’ouvrir.

Internet Explorer ou Edge #

  1. Appuyez sur F12 pour ouvrir les outils de développement
  2. Passez à l’onglet Débogueur.
  3. Utilisez l’icône de dossier près du coin supérieur gauche de la fenêtre pour ouvrir un volet de sélection de fichiers ; vous pouvez y trouver votre fichier JavaScript.

###Safari

  1. Appuyez sur Commande+Option+C pour ouvrir les outils de développement
  2. Passez à l’onglet Ressources
  3. Ouvrez le dossier “Scripts” dans le panneau de gauche
  4. Sélectionnez votre fichier JavaScript.

Ajout d’un point d’arrêt à partir des outils de développement #

Une fois que votre fichier JavaScript est ouvert dans les outils de développement, vous pouvez cliquer sur un numéro de ligne pour placer un point d’arrêt. La prochaine fois que votre programme s’exécutera, il s’y arrêtera.

Remarque sur les sources minifiées : Si votre source est minifiée, vous pouvez l’imprimer (convertir en format lisible). Dans Chrome, cela se fait en cliquant sur le bouton {} dans le coin inférieur droit de la visionneuse de code source.

IDE #

Code Visual Studio (VSC) #

VSC a [prise en charge intégrée] [1] pour le débogage de JavaScript.

  1. Cliquez sur le bouton Déboguer à gauche ou Ctrl+Maj+D
  2. Si ce n’est déjà fait, créez un fichier de configuration de lancement (launch.json) en appuyant sur l’icône d’engrenage.
  3. Exécutez le code de VSC en appuyant sur le bouton de lecture vert ou appuyez sur F5.

Ajout d’un point d’arrêt dans VSC #

Cliquez à côté du numéro de ligne dans votre fichier source JavaScript pour ajouter un point d’arrêt (il sera marqué en rouge). Pour supprimer le point d’arrêt, cliquez à nouveau sur le cercle rouge.

Astuce : Vous pouvez également utiliser les points d’arrêt conditionnels dans les outils de développement du navigateur. Ceux-ci aident à éviter les pauses inutiles dans l’exécution. Exemple de scénario : vous souhaitez examiner une variable dans une boucle exactement à la 5ème itération.

Utiliser des setters et des getters pour trouver ce qui a changé une propriété #

Disons que vous avez un objet comme celui-ci :

var myObject = {
    name: 'Peter'
}

Plus loin dans votre code, vous essayez d’accéder à myObject.name et vous obtenez George au lieu de Peter. Vous commencez à vous demander qui l’a changé et où exactement il a été changé. Il existe un moyen de placer un debugger (ou quelque chose d’autre) sur chaque ensemble (chaque fois que quelqu’un fait myObject.name = 'something'):

var myObject = {
    _name: 'Peter',
    set name(name){debugger;this._name=name},
    get name(){return this._name}
}

Notez que nous avons renommé name en _name et nous allons définir un setter et un getter pour name.

set name est le setter. C’est un endroit idéal où vous pouvez placer debugger, console.trace() ou tout autre élément dont vous avez besoin pour le débogage. Le setter définira la valeur du nom dans _name. Le getter (la partie get name) lira la valeur à partir de là. Nous avons maintenant un objet entièrement fonctionnel avec une fonctionnalité de débogage.

La plupart du temps, cependant, l’objet qui est modifié n’est pas sous notre contrôle. Heureusement, nous pouvons définir des setters et des getters sur des objets existants pour les déboguer.

// First, save the name to _name, because we are going to use name for setter/getter
otherObject._name = otherObject.name;

// Create setter and getter
Object.defineProperty(otherObject, "name", {
    set: function(name) {debugger;this._name = name},
    get: function() {return this._name}
});

Consultez [setters][1] et [getters][2] sur MDN pour plus d’informations.

Prise en charge du navigateur pour les setters/getters :

[1] : https://developer.mozilla.org/en-US/docs/Web/JavaScript/Reference/Functions/set [2] : https://developer.mozilla.org/en-US/docs/Web/JavaScript/Reference/Functions/get

Utilisation de la console #

Dans de nombreux environnements, vous avez accès à un objet console global qui contient certaines méthodes de base pour communiquer avec les périphériques de sortie standard. Le plus souvent, il s’agira de la console JavaScript du navigateur (voir Chrome, [Firefox](https : //developer.mozilla.org/en-US/docs/Tools/Browser_Console), Safari et [Edge](https://developer. microsoft.com/en-us/microsoft-edge/platform/documentation/f12-devtools-guide/console/) pour plus d’informations).

// At its simplest, you can 'log' a string
console.log("Hello, World!");

// You can also log any number of comma-separated values
console.log("Hello", "World!");

// You can also use string substitution
console.log("%s %s", "Hello", "World!");

// You can also log any variable that exist in the same scope
var arr = [1, 2, 3];
console.log(arr.length, this);

Vous pouvez utiliser différentes méthodes de console pour mettre en évidence votre sortie de différentes manières. D’autres méthodes sont également utiles pour un débogage plus avancé.

Pour plus de documentation, des informations sur la compatibilité et des instructions sur la façon d’ouvrir la console de votre navigateur, consultez la rubrique Console.

Remarque : si vous devez prendre en charge IE9, supprimez console.log ou encapsulez ses appels comme suit, car console n’est pas défini tant que les outils de développement ne sont pas ouverts :

if (console) { //IE9 workaround
    console.log("test");
}

Suspendre automatiquement l’exécution #

Dans Google Chrome, vous pouvez interrompre l’exécution sans avoir à placer de points d’arrêt.

[![Pause on Exception][1]][1] Pause on Exception : Lorsque ce bouton est activé, si votre programme rencontre une exception non gérée, le programme s’interrompra comme s’il avait atteint un point d’arrêt. Le bouton se trouve près des contrôles d’exécution et est utile pour localiser les erreurs.

Vous pouvez également suspendre l’exécution lorsqu’une balise HTML (nœud DOM) est modifiée ou lorsque ses attributs sont modifiés. Pour ce faire, cliquez avec le bouton droit sur le nœud DOM dans l’onglet Éléments et sélectionnez “Arrêter sur…”.

[1] : http://i.stack.imgur.com/OoThh.png

Inspecteur des éléments #

En cliquant sur le bouton [![entrer la description de l’image ici][1]][1] Sélectionner un élément dans la page pour l’inspecter dans le coin supérieur gauche de l’onglet Éléments dans Chrome ou de l’onglet Inspecteur dans Firefox, disponible dans les outils de développement , puis en cliquant sur un élément de la page met en surbrillance l’élément et [l’assigne à la variable $0](https://www.wikiod.com/fr/javascript/debogage#Variables d’interprétation interactives =201607211329474646179).

L’inspecteur d’éléments peut être utilisé de différentes manières, par exemple :

  1. Vous pouvez vérifier si votre JS manipule DOM comme vous l’attendez,
  2. Vous pouvez plus facilement déboguer votre CSS, en voyant quelles règles affectent l’élément (Onglet Styles dans Chrome)
  3. Vous pouvez jouer avec CSS et HTML sans recharger la page.

De plus, Chrome se souvient des 5 dernières sélections dans l’onglet Éléments. $0 est la sélection actuelle, tandis que $1 est la sélection précédente. Vous pouvez monter jusqu’à $4. De cette façon, vous pouvez facilement déboguer plusieurs nœuds sans changer constamment de sélection vers eux.

Vous pouvez en savoir plus sur [Google Developers][2].

[1] : http://i.stack.imgur.com/uD8b5.png [2] : https://developers.google.com/web/tools/chrome-devtools/debug/command-line/command-line-reference#section-1

Parcourir le code #

Une fois que vous avez suspendu l’exécution sur un point d’arrêt, vous pouvez suivre l’exécution ligne par ligne pour observer ce qui se passe. [Ouvrez les outils de développement de votre navigateur] (https://www.wikiod.com/fr/javascript/debogage#Points d’arrêt) et recherchez les icônes de contrôle d’exécution. (Cet exemple utilise les icônes de Google Chrome, mais elles seront similaires dans d’autres navigateurs.)

[![Resume][1]][1] Resume : Relance l’exécution. Raccourci :F8(Chrome, Firefox)

[![Step Over][2]][2] Step Over : Exécutez la ligne de code suivante. Si cette ligne contient un appel de fonction, exécutez toute la fonction et passez à la ligne suivante, plutôt que de sauter là où la fonction est définie. Raccourci : F10(Chrome, Firefox, IE/Edge), F6(Safari)

[![Step Into][3]][3] Step Into : Exécutez la ligne de code suivante. Si cette ligne contient un appel de fonction, sautez dans la fonction et faites une pause là-bas. Raccourci : F11(Chrome, Firefox, IE/Edge), F7(Safari)

[![Step Out][4]][4] Step Out : Exécute le reste de la fonction actuelle, revient à l’endroit d’où la fonction a été appelée et s’arrête à l’instruction suivante. Raccourci : Maj + F11(Chrome, Firefox, IE/Edge), F8(Safari)

Utilisez-les en conjonction avec Call Stack, qui vous indiquera dans quelle fonction vous vous trouvez actuellement, quelle fonction a appelé cette fonction, etc.

Consultez le guide de Google sur “Comment parcourir le code” pour plus de détails et de conseils .

Pause lors de l’appel d’une fonction #

Pour les fonctions nommées (non anonymes), vous pouvez arrêter lorsque la fonction est exécutée.

debug(functionName);

La prochaine fois que la fonction functionName s’exécutera, le débogueur s’arrêtera sur sa première ligne.
