Reste / Module (%) #

L’opérateur reste/module (%) renvoie le reste après la division (entier).

console.log( 42 %  10); //  2
console.log( 42 % -10); //  2
console.log(-42 %  10); // -2
console.log(-42 % -10); // -2
console.log(-40 %  10); // -0
console.log( 40 %  10); //  0

Cet opérateur renvoie le reste lorsqu’un opérande est divisé par un second opérande. Lorsque le premier opérande est une valeur négative, la valeur de retour sera toujours négative, et vice versa pour les valeurs positives.

Dans l’exemple ci-dessus, ‘10’ peut être soustrait quatre fois de ‘42’ avant qu’il ne reste plus assez pour soustraire à nouveau sans que cela change de signe. Le reste est donc : ‘42 - 4 * 10 = 2’.

L’opérateur de reste peut être utile pour les problèmes suivants :

  1. Tester si un entier est (non) divisible par un autre nombre :

     x % 4 == 0 // true if x is divisible by 4
 x % 2 == 0 // true if x is even number
 x % 2 != 0 // true if x is odd number

Puisque 0 === -0, cela fonctionne aussi pour x <= -0.

  1. Implémenter l’incrément/décrément cyclique de la valeur dans l’intervalle [0, n).

Supposons que nous ayons besoin d’incrémenter la valeur entière de ‘0’ à (mais non compris) ’n’, de sorte que la valeur suivante après ’n-1’ devienne ‘0’. Cela peut être fait par un tel pseudocode :

var n = ...; // given n
var i = 0;
function inc() {
    i = (i + 1) % n;
}
while (true) {
    inc();
    // update something with i
}

Maintenant, généralisez le problème ci-dessus et supposons que nous devons permettre à la fois d’incrémenter et de décrémenter cette valeur de ‘0’ à (sans inclure) ’n’, de sorte que la valeur suivante après ’n-1’ devienne ‘0’ et la valeur précédente avant ‘0’ devient ’n-1’.

var n = ...; // given n
var i = 0;
function delta(d) { // d - any signed integer
    i = (i + d + n) % n; // we add n to (i+d) to ensure the sum is positive
}

Maintenant, nous pouvons appeler la fonction delta() en passant n’importe quel entier, positif et négatif, comme paramètre delta.

Utiliser le module pour obtenir la partie fractionnaire d’un nombre #

 var myNum = 10 / 4;       // 2.5
 var fraction = myNum % 1; // 0.5
 myNum = -20 / 7;          // -2.857142857142857
 fraction = myNum % 1;     // -0.857142857142857

Arrondi #

Arrondi #

Math.round() arrondira la valeur à l’entier le plus proche en utilisant half round up pour rompre les liens.

var a = Math.round(2.3);       // a is now 2  
var b = Math.round(2.7);       // b is now 3
var c = Math.round(2.5);       // c is now 3

Mais

var c = Math.round(-2.7);       // c is now -3
var c = Math.round(-2.5);       // c is now -2

Notez comment -2.5 est arrondi à -2. En effet, les valeurs intermédiaires sont toujours arrondies, c’est-à-dire qu’elles sont arrondies à l’entier avec la valeur immédiatement supérieure.

Arrondir #

Math.ceil() arrondira la valeur vers le haut.

var a = Math.ceil(2.3);        // a is now 3
var b = Math.ceil(2.7);        // b is now 3

plafond un nombre négatif arrondira vers zéro

var c = Math.ceil(-1.1);       // c is now 1

Arrondi à l’inférieur #

Math.floor() arrondira la valeur vers le bas.

var a = Math.floor(2.3);        // a is now 2
var b = Math.floor(2.7);        // b is now 2

flooring un nombre négatif l’arrondira à partir de zéro.

var c = Math.floor(-1.1);       // c is now -1

Tronquer #

Avertissement : l’utilisation d’opérateurs au niveau du bit (sauf >>>) ne s’applique qu’aux nombres compris entre -2147483649 et 2147483648.

2.3  | 0;                       // 2 (floor)
-2.3 | 0;                       // -2 (ceil)
NaN  | 0;                       // 0

Math.trunc()

Math.trunc(2.3);                // 2 (floor)
Math.trunc(-2.3);               // -2 (ceil)
Math.trunc(2147483648.1);       // 2147483648 (floor)
Math.trunc(-2147483649.1);      // -2147483649 (ceil)
Math.trunc(NaN);                // NaN

Arrondi aux décimales #

Math.floor, Math.ceil() et Math.round() peuvent être utilisés pour arrondir à un certain nombre de décimales

Pour arrondir à 2 décimales :

 var myNum = 2/3;               // 0.6666666666666666
 var multiplier = 100;
 var a = Math.round(myNum * multiplier) / multiplier;  // 0.67
 var b = Math.ceil (myNum * multiplier) / multiplier;  // 0.67
 var c = Math.floor(myNum * multiplier) / multiplier;  // 0.66

Vous pouvez également arrondir à un certain nombre de chiffres :

 var myNum = 10000/3;           // 3333.3333333333335
 var multiplier = 1/100;
 var a = Math.round(myNum * multiplier) / multiplier;  // 3300
 var b = Math.ceil (myNum * multiplier) / multiplier;  // 3400
 var c = Math.floor(myNum * multiplier) / multiplier;  // 3300

En tant que fonction plus utilisable :

 // value is the value to round
 // places if positive the number of decimal places to round to
 // places if negative the number of digits to round to
 function roundTo(value, places){
     var power = Math.pow(10, places);
     return Math.round(value * power) / power;
 }
 var myNum = 10000/3;    // 3333.3333333333335
 roundTo(myNum, 2);  // 3333.33
 roundTo(myNum, 0);  // 3333
 roundTo(myNum, -2); // 3300

Et les variantes pour ‘ceil’ et ‘floor’ :

 function ceilTo(value, places){
     var power = Math.pow(10, places);
     return Math.ceil(value * power) / power;
 }
 function floorTo(value, places){
     var power = Math.pow(10, places);
     return Math.floor(value * power) / power;
 }

Trigonométrie #

Tous les angles ci-dessous sont en radians. Un angle r en radians a pour mesure 180 * r / Math.PI en degrés.

Sinus #

Math.sin(r);

Cela renverra le sinus de r, une valeur comprise entre -1 et 1.

Math.asin(r);

Cela renverra l’arcsinus (l’inverse du sinus) de r.

Math.asinh(r)

Incrémentation (++) #

L’opérateur d’incrémentation (++) incrémente son opérande de un.

  • S’il est utilisé comme suffixe, il renvoie la valeur avant l’incrémentation.
  • S’il est utilisé comme préfixe, il renvoie la valeur après incrémentation.
//postfix
var a = 5,    // 5
    b = a++,  // 5
    c = a     // 6

Dans ce cas, ‘a’ est incrémenté après avoir défini ‘b’. Ainsi, b sera 5 et c sera 6.

//prefix
var a = 5,    // 5
    b = ++a,  // 6
    c = a     // 6

Dans ce cas, ‘a’ est incrémenté avant de définir ‘b’. Ainsi, b sera 6 et c sera 6.

Les opérateurs d’incrémentation et de décrémentation sont couramment utilisés dans les boucles “for”, par exemple :

for(var i = 0; i < 42; ++i)
{
  // do something awesome!
}

Remarquez comment la variante prefix est utilisée. Cela garantit qu’une variable temporaire n’est pas créée inutilement (pour renvoyer la valeur avant l’opération).

Opérateurs au niveau du bit #

Notez que toutes les opérations au niveau du bit opèrent sur des entiers 32 bits en passant n’importe quel opérande à la fonction interne [ToInt32][1].

Bitwise ou #

var a;
a = 0b0011 | 0b1010; // a === 0b1011
// truth table
// 1010 | (or)
// 0011  
// 1011  (result)

Au niveau du bit et #

a = 0b0011 & 0b1010; // a === 0b0010
// truth table
// 1010 & (and)
// 0011  
// 0010  (result)

Au niveau du bit pas #

a = ~0b0011; // a === 0b1100
// truth table
// 10 ~(not)
// 01  (result)

Bitwise xor (ou exclusif) #

a = 0b1010 ^ 0b0011; // a === 0b1001
// truth table
// 1010 ^ (xor)
// 0011  
// 1001  (result)

Au niveau du bit décalage gauche #

a = 0b0001 << 1; // a === 0b0010
a = 0b0001 << 2; // a === 0b0100
a = 0b0001 << 3; // a === 0b1000

Le décalage vers la gauche équivaut à un nombre entier multiplié par Math.pow(2, n). Lorsque vous effectuez des calculs sur les nombres entiers, le décalage peut améliorer considérablement la vitesse de certaines opérations mathématiques.

var n = 2;
var a = 5.4;
var result = (a << n) === Math.floor(a) * Math.pow(2,n);
// result is true
a = 5.4 << n; // 20

Bitwise right shift >> (décalage de propagation de signe) >>> (décalage à droite sans remplissage) #

a = 0b1001 >> 1; // a === 0b0100
a = 0b1001 >> 2; // a === 0b0010
a = 0b1001 >> 3; // a === 0b0001

a = 0b1001 >>> 1; // a === 0b0100
a = 0b1001 >>> 2; // a === 0b0010
a = 0b1001 >>> 3; // a === 0b0001

Une valeur négative de 32 bits a toujours le bit le plus à gauche :

a = 0b11111111111111111111111111110111 | 0;   
console.log(a); // -9
b = a >> 2;     // leftmost bit is shifted 1 to the right then new left most bit is set to on (1)
console.log(b); // -3
b = a >>> 2;    // leftmost bit is shifted 1 to the right. the new left most bit is set to off (0)
console.log(b); // 2147483643

Le résultat d’une opération >>> est toujours positif. Le résultat d’un >> est toujours du même signe que la valeur décalée.

Le décalage vers la droite sur les nombres positifs équivaut à diviser par Math.pow(2,n) et à égaliser le résultat :

a = 256.67;
n = 4;
result = (a >> n) === Math.floor( Math.floor(a) / Math.pow(2,n) );
// result is true
a = a >> n; //  16

result = (a >>> n) === Math.floor( Math.floor(a) / Math.pow(2,n) );
// result is true
a = a >>> n; //  16

Le décalage vers la droite du remplissage zéro (>>>) sur les nombres négatifs est différent. Comme JavaScript ne convertit pas en entiers non signés lors des opérations sur les bits, il n’y a pas d’équivalent opérationnel :

a = -256.67;
result = (a >>> n) === Math.floor( Math.floor(a) / Math.pow(2,n) );
// result is false

Opérateurs d’affectation au niveau du bit #

À l’exception de not (~), tous les opérateurs au niveau du bit ci-dessus peuvent être utilisés comme opérateurs d’affectation :

a |= b;   // same as: a = a | b;
a ^= b;   // same as: a = a ^ b;
a &= b;   // same as: a = a & b;
a >>= b;  // same as: a = a >> b;
a >>>= b; // same as: a = a >>> b;
a <<= b;  // same as: a = a << b;

Avertissement : Javascript utilise Big Endian pour stocker des nombres entiers. Cela ne correspondra pas toujours à l’Endian de l’appareil/du système d’exploitation. Lorsque vous utilisez des tableaux typés avec des longueurs de bits supérieures à 8 bits, vous devez vérifier si l’environnement est Little Endian ou Big Endian avant d’appliquer des opérations au niveau du bit.

Avertissement : les opérateurs binaires tels que & et | ne sont pas les mêmes que les opérateurs logiques [&& (et)](https://www.wikiod.com/fr/docs/javascript 208/boolean-logic/779/logic-operators-with-booleans#t=201606101417337061178) et [|| (ou)](https://www.wikiod.com/fr/docs/javascript208/boolean-logic/779 /opérateurs-logiques-avec-booléens#t=201606101417337061178). Ils fourniront des résultats incorrects s’ils sont utilisés comme opérateurs logiques. L’opérateur ^ n’est pas l’[opérateur de puissance (ab)](https://www.wikiod.com/fr/docs/javascript203/arithmetic-math/763/ exponentiation-math-pow-ou).

[1] : http://www.ecma-international.org/ecma-262/6.0/#sec-toint32

Ajout (+) #

L’opérateur d’addition (+) ajoute des nombres.

var a = 9,
    b = 3,
    c = a + b;

c sera maintenant 12

Cet opérande peut également être utilisé plusieurs fois dans une même affectation :

var a = 9,
    b = 3,
    c = 8,
    d = a + b + c;

d sera maintenant 20.

Les deux opérandes sont convertis en types primitifs. Ensuite, si l’un ou l’autre est une chaîne, ils sont tous deux convertis en chaînes et concaténés. Sinon, ils sont tous deux convertis en nombres et ajoutés.

null + null;      // 0
null + undefined; // NaN
null + {};        // "null[object Object]"
null + '';        // "null"

Si les opérandes sont une chaîne et un nombre, le nombre est converti en chaîne, puis ils sont concaténés, ce qui peut entraîner des résultats inattendus lorsque vous travaillez avec des chaînes qui semblent numériques.

"123" + 1;        // "1231" (not 124)

Si une valeur booléenne est donnée à la place de l’une des valeurs numériques, la valeur booléenne est convertie en un nombre (“0” pour “faux”, “1” pour “vrai”) avant que la somme ne soit calculée :

true + 1;         // 2
false + 5;        // 5
null + 1;         // 1
undefined + 1;    // NaN

Si une valeur booléenne est donnée à côté d’une valeur de chaîne, la valeur booléenne est convertie en chaîne à la place :

true + "1";        // "true1"
false + "bar";     // "falsebar"

Simuler des événements avec différentes probabilités #

Parfois, vous n’aurez peut-être besoin que de simuler un événement avec deux résultats, peut-être avec des probabilités différentes, mais vous pouvez vous retrouver dans une situation qui appelle de nombreux résultats possibles avec des probabilités différentes. Imaginons que vous vouliez simuler un événement qui a six résultats également probables. C’est assez simple.

function simulateEvent(numEvents) {
    var event = Math.floor(numEvents*Math.random());
    return event;
}

// simulate fair die
console.log("Rolled a "+(simulateEvent(6)+1));  // Rolled a 2

Cependant, vous ne voudrez peut-être pas des résultats également probables. Supposons que vous disposiez d’une liste de trois résultats représentés sous la forme d’un tableau de probabilités en pourcentages ou en multiples de probabilité. Un tel exemple pourrait être un dé pondéré. Vous pouvez réécrire la fonction précédente pour simuler un tel événement.

function simulateEvent(chances) {
    var sum = 0;
    chances.forEach(function(chance) {
        sum+=chance;
    });
    var rand = Math.random();
    var chance = 0;
    for(var i=0; i<chances.length; i++) {
        chance+=chances[i]/sum;
        if(rand<chance) {
            return i;
        }
    }
    
    // should never be reached unless sum of probabilities is less than 1
    // due to all being zero or some being negative probabilities
    return -1;
}

// simulate weighted dice where 6 is twice as likely as any other face
// using multiples of likelihood
console.log("Rolled a "+(simulateEvent([1,1,1,1,1,2])+1));  // Rolled a 1

// using probabilities
console.log("Rolled a "+(simulateEvent([1/7,1/7,1/7,1/7,1/7,2/7])+1));  // Rolled a 6

Comme vous l’avez probablement remarqué, ces fonctions renvoient un index, vous pouvez donc avoir des résultats plus descriptifs stockés dans un tableau. Voici un exemple.

var rewards = ["gold coin","silver coin","diamond","god sword"];
var likelihoods = [5,9,1,0];
// least likely to get a god sword (0/15 = 0%, never),
// most likely to get a silver coin (9/15 = 60%, more than half the time)

// simulate event, log reward
console.log("You get a "+rewards[simulateEvent(likelihoods)]);  // You get a silver coin

Pour détecter l’endian de l’appareil

var isLittleEndian = true;
(()=>{

Si le système utilise Big-Endian, les valeurs d’octets 8 bits seront

console.log(data8[2].toString(16)); // 0x33

if(isLittleEndian){
    mask = 0xFF00FF00; // littleEndian pixel channels Alpha,Blue,Green,Red
}

Aléatoire avec distribution gaussienne #

La fonction Math.random() devrait donner des nombres aléatoires qui ont un écart-type proche de 0. Lorsque vous choisissez dans un jeu de cartes ou que vous simulez un lancer de dés, c’est ce que nous voulons.

Mais dans la plupart des situations, cela est irréaliste. Dans le monde réel, le caractère aléatoire a tendance à se rassembler autour d’une valeur normale commune. Si elle est tracée sur un graphique, vous obtenez la courbe en cloche classique ou la distribution gaussienne.

Faire cela avec la fonction Math.random() est relativement simple.

var randNum = (Math.random() + Math.random()) / 2; 
var randNum = (Math.random() + Math.random() + Math.random()) / 3; 
var randNum = (Math.random() + Math.random() + Math.random() + Math.random()) / 4;

L’ajout d’une valeur aléatoire à la dernière augmente la variance des nombres aléatoires. Diviser par le nombre de fois que vous ajoutez normalise le résultat dans une plage de 0 à 1

Comme l’ajout de plus de quelques aléatoires est désordonné, une simple fonction vous permettra de sélectionner la variance que vous souhaitez.

// v is the number of times random is summed and should be over >= 1
// return a random number between 0-1 exclusive
function randomG(v){ 
    var r = 0;
    for(var i = v; i > 0; i --){
        r += Math.random();
    }
    return r / v;
}

Sin & Cos pour créer un vecteur avec une direction et une distance données #

Si vous avez un vecteur sous forme polaire (direction et distance), vous voudrez le convertir en un vecteur cartésien avec une composante x et y. Pour référence, le système de coordonnées de l’écran a des directions sous forme de points de 0 degré de gauche à droite, 90 (PI/2) pointent vers le bas de l’écran, et ainsi de suite dans le sens des aiguilles d’une montre.

var dir = 1.4536; // direction in radians
var dist = 200; // distance
var vec = {};
vec.x = Math.cos(dir) * dist; // get the x component
vec.y = Math.sin(dir) * dist; // get the y component

Vous pouvez également ignorer la distance pour créer un vecteur normalisé (1 unité de long) dans la direction de dir

var dir = 1.4536; // direction in radians
var vec = {};
vec.x = Math.cos(dir); // get the x component
vec.y = Math.sin(dir); // get the y component

Si votre système de coordonnées a y comme valeur, vous devez changer cos et sin. Dans ce cas, une direction positive est dans le sens antihoraire à partir de l’axe x.

// get the directional vector where y points up
var dir = 1.4536; // direction in radians
var vec = {};
vec.x = Math.sin(dir); // get the x component
vec.y = Math.cos(dir); // get the y component

Math.hypot #

Pour trouver la distance entre deux points, nous utilisons Pythagore pour obtenir la racine carrée de la somme du carré de la composante du vecteur entre eux.

var v1 = {x : 10, y :5};
var v2 = {x : 20, y : 10};
var x = v2.x - v1.x;
var y = v2.y - v1.y;
var distance = Math.sqrt(x * x + y * y); // 11.180339887498949

Avec ECMAScript 6 est venu Math.hypot qui fait la même chose

var v1 = {x : 10, y :5}; 
var v2 = {x : 20, y : 10}; 
var x = v2.x - v1.x; 
var y = v2.y - v1.y;
var distance = Math.hypot(x,y); // 11.180339887498949

Maintenant, vous n’avez plus besoin de maintenir les vars intermédiaires pour empêcher le code de devenir un gâchis de variables

var v1 = {x : 10, y :5};
var v2 = {x : 20, y : 10};
var distance = Math.hypot(v2.x - v1.x, v2.y - v1.y); // 11.180339887498949

Math.hypot peut prendre n’importe quel nombre de dimensions

// find distance in 3D
var v1 = {x : 10, y : 5, z : 7};

Fonctions périodiques utilisant Math.sin #

Math.sin et Math.cos sont cycliques avec une période de 2 * PI radians (360 degrés) ils produisent une onde avec une amplitude de 2 dans la plage -1 à 1.

Graphique de la fonction sinus et cosinus : (avec la permission de Wikipedia) [](https://en.wikipedia.org/wiki /Onde sinusoïdale)

Ils sont tous deux très pratiques pour de nombreux types de calculs périodiques, de la création d’ondes sonores aux animations, en passant par l’encodage et le décodage de données d’image.

Cet exemple montre comment créer une onde sinusoïdale simple avec un contrôle sur la période/fréquence, la phase, l’amplitude et le décalage.

L’unité de temps utilisée est la seconde. La forme la plus simple avec un contrôle sur la fréquence uniquement.

// time is the time in seconds when you want to get a sample
// Frequency represents the number of oscillations per second
function oscillator(time, frequency){  
    return Math.sin(time * 2 * Math.PI * frequency);
}

Dans presque tous les cas, vous souhaiterez apporter des modifications à la valeur renvoyée. Les termes communs pour les modifications

  • Phase : Le décalage en termes de fréquence depuis le début des oscillations. C’est une valeur comprise entre 0 et 1 où la valeur 0,5 fait avancer l’onde dans le temps de la moitié de sa fréquence. Une valeur de 0 ou 1 ne change rien.
  • Amplitude : La distance entre la valeur la plus basse et la valeur la plus haute pendant un cycle. Une amplitude de 1 a une plage de 2. Du point le plus bas (creux) -1 au plus haut (pic) 1. Pour une onde de fréquence 1, le pic est à 0,25 seconde et le creux à 0,75.
  • Offset : déplace toute la vague vers le haut ou vers le bas.

Pour inclure tout cela dans la fonction :

function oscillator(time, frequency = 1, amplitude = 1, phase = 0, offset = 0){
    var t = time * frequency * Math.PI * 2; // get phase at time
    t += phase * Math.PI * 2; // add the phase offset
    var v = Math.sin(t); // get the value at the calculated position in the cycle
    v *= amplitude; // set the amplitude
    v += offset; // add the offset
    return v;
}

Ou sous une forme plus compacte (et légèrement plus rapide):

function oscillator(time, frequency = 1, amplitude = 1, phase = 0, offset = 0){
    return Math.sin(time * frequency * Math.PI * 2 + phase * Math.PI * 2) * amplitude + offset; 
}

Tous les arguments sauf le temps sont facultatifs

Obtenir le maximum et le minimum #

La fonction Math.max() renvoie le plus grand de zéro ou plusieurs nombres.

Math.max(4, 12);   //  12
Math.max(-1, -15); // -1

La fonction Math.min() renvoie le plus petit de zéro ou plusieurs nombres.

Math.min(4, 12);   //  4
Math.min(-1, -15); // -15

Obtenir le maximum et le minimum d’un tableau : #

var arr = [1, 2, 3, 4, 5, 6, 7, 8, 9],
    max = Math.max.apply(Math, arr),
    min = Math.min.apply(Math, arr);

console.log(max); // Logs: 9
console.log(min); // Logs: 1

ECMAScript 6 [spread operator][1], obtenant le maximum et le minimum d’un tableau :

var arr = [1, 2, 3, 4, 5, 6, 7, 8, 9],
    max = Math.max(...arr),
    min = Math.min(...arr);

console.log(max); // Logs: 9
console.log(min); // Logs: 1

[1] : https://developer.mozilla.org/en-US/docs/Web/JavaScript/Reference/Operators/Spread_operator

Limiter le nombre à la plage min/max #

Si vous devez bloquer un nombre pour le maintenir à l’intérieur d’une limite de plage spécifique

function clamp(min, max, val) {
    return Math.min(Math.max(min, +val), max);
}

console.log(clamp(-10, 10, "4.30")); // 4.3
console.log(clamp(-10, 10, -8));     // -8
console.log(clamp(-10, 10, 12));     // 10
console.log(clamp(-10, 10, -15));    // -10

Obtenir les racines d’un nombre #

Racine carrée #

Utilisez Math.sqrt() pour trouver la racine carrée d’un nombre

Math.sqrt(16)   #=> 4

Racine cubique #

Pour trouver la racine cubique d’un nombre, utilisez la fonction Math.cbrt()

Math.cbrt(27)  #=> 3

Trouver les nièmes racines #

Pour trouver la nième racine, utilisez la fonction Math.pow() et transmettez un exposant fractionnaire.

Math.pow(64, 1/6) #=> 2
