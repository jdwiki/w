Opérateurs au niveau du bit #

Les opérateurs au niveau du bit effectuent des opérations sur les valeurs binaires des données. Ces opérateurs convertissent les opérandes en entiers 32 bits signés en complément à deux.

Conversion en entiers 32 bits #

Les nombres avec plus de 32 bits suppriment leurs bits les plus significatifs. Par exemple, l’entier suivant de plus de 32 bits est converti en un entier de 32 bits :

Before: 10100110111110100000000010000011110001000001
After:              10100000000010000011110001000001

Complément à deux #

En binaire normal, nous trouvons la valeur binaire en ajoutant les “1” en fonction de leur position en tant que puissances de 2 - Le bit le plus à droite étant “2^0” et le bit le plus à gauche étant “2^n-1” où “n” est le nombre de bits. Par exemple, en utilisant 4 bits :

// Normal Binary
// 8 4 2 1
   0 1 1 0  => 0 + 4 + 2 + 0 => 6

Le format du complément à deux signifie que la contrepartie négative du nombre (6 vs -6) est constituée de tous les bits d’un nombre inversé, plus un. Les bits inversés de 6 seraient :

// Normal binary
   0 1 1 0
// One's complement (all bits inverted)
   1 0 0 1  => -8 + 0 + 0 + 1 => -7
// Two's complement (add 1 to one's complement)
   1 0 1 0  => -8 + 0 + 2 + 0 => -6

Remarque : Ajouter plus de “1” à gauche d’un nombre binaire ne change pas sa valeur en complément à deux. Les valeurs 1010 et 1111111111010 sont toutes les deux -6.

AND au niveau du bit #

L’opération ET au niveau du bit “a & b” renvoie la valeur binaire avec un “1” où les deux opérandes binaires ont des “1” à une position spécifique et des “0” à toutes les autres positions. Par exemple:

13 & 7 => 5
// 13:     0..01101
//  7:     0..00111
//-----------------
//  5:     0..00101 (0 + 0 + 4 + 0 + 1)

Exemple concret : vérification de la parité des numéros

Au lieu de ce “chef-d’œuvre” (malheureusement trop souvent vu dans de nombreuses parties de code réel) :

function isEven(n) {
    return n % 2 == 0;
}

function isOdd(n) {
    if (isEven(n)) {
        return false;
    } else {
        return true;
    }
}

Vous pouvez vérifier la parité du nombre (entier) de manière beaucoup plus efficace et simple :

if(n & 1) {
    console.log("ODD!");
} else {
    console.log("EVEN!");
}

OR au niveau du bit #

L’opération OU au niveau du bit a | b renvoie la valeur binaire avec un 1 où les opérandes ou les deux opérandes ont des 1 dans une position spécifique, et 0 lorsque les deux valeurs ont 0 dans une position. Par exemple:

13 | 7 => 15
// 13:     0..01101
//  7:     0..00111
//-----------------
// 15:     0..01111 (0 + 8 + 4 + 2 + 1)

PAS au niveau du bit #

L’opération NOT au niveau du bit ~a retourne les bits de la valeur donnée a. Cela signifie que tous les “1” deviendront des “0” et que tous les “0” deviendront des “1”.

~13 => -14
// 13:     0..01101
//-----------------
//-14:     1..10010 (-16 + 0 + 0 + 2 + 0)

XOR au niveau du bit #

L’opération XOR au niveau du bit (exclusif ou) a ^ b place un 1 uniquement si les deux bits sont différents. Exclusif ou signifie soit l’un soit l’autre, mais pas les deux.

13 ^ 7 => 10
// 13:     0..01101
//  7:     0..00111
//-----------------
// 10:     0..01010  (0 + 8 + 0 + 2 + 0)

Exemple concret : échange de deux valeurs entières sans allocation de mémoire supplémentaire

var a = 11, b = 22;
a = a ^ b;
b = a ^ b;
a = a ^ b;
console.log("a = " + a + "; b = " + b);// a is now 22 and b is now 11

Opérateurs de décalage #

Le décalage au niveau du bit peut être considéré comme un “déplacement” des bits vers la gauche ou vers la droite, et donc la modification de la valeur des données sur lesquelles l’opération est effectuée.

Décalage à gauche #

L’opérateur de décalage vers la gauche (valeur) << (montant du décalage) décalera les bits vers la gauche de (montant du décalage) bits ; les nouveaux bits venant de la droite seront des “0”:

5 << 2 => 20
//  5:      0..000101
// 20:      0..010100 <= adds two 0's to the right

Décalage vers la droite (propagation des signes) #

L’opérateur de décalage vers la droite (valeur) >> (montant du décalage) est également connu sous le nom de “décalage vers la droite à propagation de signe” car il conserve le signe de l’opérande initial. L’opérateur de décalage vers la droite décale la “valeur” de la “quantité de décalage” spécifiée de bits vers la droite. Les bits en excès décalés vers la droite sont ignorés. Les nouveaux bits provenant de la gauche seront basés sur le signe de l’opérande initial. Si le bit le plus à gauche était ‘1’ alors les nouveaux bits seront tous ‘1’ et vice-versa pour les ‘0’.

20 >> 2 => 5
// 20:      0..010100
//  5:      0..000101 <= added two 0's from the left and chopped off 00 from the right

-5 >> 3 => -1
// -5:      1..111011
// -2:      1..111111 <= added three 1's from the left and chopped off 011 from the right

Décalage vers la droite (Remplissage zéro) #

L’opérateur de décalage vers la droite de remplissage par zéro (valeur) >>> (montant du décalage) déplacera les bits vers la droite, et les nouveaux bits seront des 0. Les “0” sont décalés depuis la gauche, et les bits en excès vers la droite sont décalés et ignorés. Cela signifie qu’il peut transformer des nombres négatifs en nombres positifs.

-30 >>> 2 => 1073741816
//       -30:      111..1100010
//1073741816:      001..1111000

Le décalage à droite à remplissage par zéro et le décalage à droite à propagation de signe donnent le même résultat pour les nombres non négatifs.
