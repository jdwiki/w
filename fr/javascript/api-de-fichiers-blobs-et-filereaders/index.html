Syntaxe #

  • lecteur = nouveau FileReader();

Paramètres #

Propriété/Méthode Descriptif
erreur Une erreur s’est produite lors de la lecture du fichier.
étatprêt Contient l’état actuel du FileReader.
résultat Contient le contenu du fichier.
onabandonne Déclenché lorsque l’opération est abandonnée.
une erreur Déclenché lorsqu’une erreur est rencontrée.
onload Déclenché lorsque le fichier est chargé.
onloadstart Déclenché lorsque l’opération de chargement du fichier a commencé.
onloadend Déclenché lorsque l’opération de chargement du fichier est terminée.
en cours Déclenché lors de la lecture d’un Blob.
abandonner() Annule l’opération en cours.
readAsArrayBuffer(blob) Démarre la lecture du fichier en tant que ArrayBuffer.
readAsDataURL(blob) Démarre la lecture du fichier en tant qu’url/uri de données.
readAsText(blob[, codage]) Démarre la lecture du fichier en tant que fichier texte. Impossible de lire les fichiers binaires. Utilisez plutôt readAsArrayBuffer.

https://www.w3.org/TR/FileAPI/

Lire le fichier en tant que chaîne #

Assurez-vous d’avoir une entrée de fichier sur votre page :

<input type="file" id="upload">

Puis en JavaScript :

document.getElementById('upload').addEventListener('change', readFileAsString)
function readFileAsString() {
    var files = this.files;
    if (files.length === 0) {
        console.log('No file is selected');
        return;
    }

    var reader = new FileReader();
    reader.onload = function(event) {
        console.log('File content:', event.target.result);
    };
    reader.readAsText(files[0]);
}

Lire le fichier en tant que dataURL #

La lecture du contenu d’un fichier dans une application Web peut être accomplie en utilisant l’API de fichier HTML5. Tout d’abord, ajoutez une entrée avec type="file" dans votre code HTML :

<input type="file" id="upload">

Ensuite, nous allons ajouter un écouteur de changement sur l’entrée de fichier. Cet exemple définit l’écouteur via JavaScript, mais il pourrait également être ajouté en tant qu’attribut sur l’élément d’entrée. Cet écouteur est déclenché chaque fois qu’un nouveau fichier est sélectionné. Dans ce rappel, nous pouvons lire le fichier qui a été sélectionné et effectuer d’autres actions (comme créer une image avec le contenu du fichier sélectionné) :

document.getElementById('upload').addEventListener('change', showImage);

function showImage(evt) {
    var files = evt.target.files;

    if (files.length === 0) {
        console.log('No files selected');
        return;
    }

    var reader = new FileReader();
    reader.onload = function(event) {
        var img = new Image();
        img.onload = function() {
            document.body.appendChild(img);
        };
        img.src = event.target.result;
    };
    reader.readAsDataURL(files[0]);
}

Découper un fichier #

La méthode blob.slice() est utilisée pour créer un nouvel objet Blob contenant les données dans la plage d’octets spécifiée du Blob source. Cette méthode est également utilisable avec les instances de File, puisque File étend Blob.

Ici, nous découpons un fichier en un nombre spécifique de blobs. Ceci est particulièrement utile dans les cas où vous devez traiter des fichiers trop volumineux pour être lus en mémoire en une seule fois. Nous pouvons ensuite lire les morceaux un par un en utilisant FileReader.

/**
* @param {File|Blob} - file to slice
* @param {Number} - chunksAmount
* @return {Array} - an array of Blobs
**/
function sliceFile(file, chunksAmount) {
  var byteIndex = 0;
  var chunks = [];
    
  for (var i = 0; i < chunksAmount; i += 1) {
    var byteEnd = Math.ceil((file.size / chunksAmount) * (i + 1));
    chunks.push(file.slice(byteIndex, byteEnd));
    byteIndex += (byteEnd - byteIndex);
  }

  return chunks;
}

Téléchargement csv côté client à l’aide de Blob #

function downloadCsv() {
  var blob = new Blob([csvString]);
  if (window.navigator.msSaveOrOpenBlob){
    window.navigator.msSaveBlob(blob, "filename.csv");
  }
  else {
    var a = window.document.createElement("a");

    a.href = window.URL.createObjectURL(blob, {
      type: "text/plain"
    });
    a.download = "filename.csv";
    document.body.appendChild(a);
    a.click();
    document.body.removeChild(a);
  }
}
var string = "a1,a2,a3";
downloadCSV(string);

Référence source ; https://github.com/mholt/PapaParse/issues/175

Sélection de plusieurs fichiers et restriction des types de fichiers #

L’API de fichier HTML5 vous permet de restreindre les types de fichiers acceptés en définissant simplement l’attribut accept sur une entrée de fichier, par exemple :

<input type="file" accept="image/jpeg">

La spécification de plusieurs types MIME séparés par une virgule (par exemple, image/jpeg,image/png) ou l’utilisation de caractères génériques (par exemple, image/* pour autoriser tous les types d’images) vous offre un moyen rapide et puissant de restreindre le type de fichiers vous souhaitez sélectionner. Voici un exemple pour autoriser n’importe quelle image ou vidéo :

<input type="file" accept="image/*,video*">

Par défaut, l’entrée de fichier permet à l’utilisateur de sélectionner un seul fichier. Si vous souhaitez activer la sélection de fichiers multiples, ajoutez simplement l’attribut “multiple” :

<input type="file" multiple>

Vous pouvez ensuite lire tous les fichiers sélectionnés via le tableau files de l’entrée de fichier. Voir [lire le fichier comme dataUrl][1]

[1] : https://www.wikiod.com/fr/javascript/api-de-fichiers-blobs-et-filereaders#Lire le fichier en tant que dataURL “Lire le fichier en tant qu’URL de données”

Récupère les propriétés du fichier #

Si vous souhaitez obtenir les propriétés du fichier (comme le nom ou la taille), vous pouvez le faire avant d’utiliser le lecteur de fichiers. Si nous avons le morceau de code html suivant :

<input type="file" id="newFile">

Vous pouvez accéder directement aux propriétés comme ceci :

document.getElementById('newFile').addEventListener('change', getFile);

function getFile(event) {
    var files = event.target.files
        , file = files[0];

    console.log('Name of the file', file.name);
    console.log('Size of the file', file.size);
}

Vous pouvez également obtenir facilement les attributs suivants : lastModified (horodatage), lastModifiedDate (date) et type (type de fichier).
