nul #

’null’ est utilisé pour représenter l’absence intentionnelle d’une valeur d’objet et est une valeur primitive. Contrairement à undefined, ce n’est pas une propriété de l’objet global.

Il est égal à “undefined” mais pas identique à celui-ci.

null == undefined; // true
null === undefined; // false

ATTENTION : Le typeof null est 'object'.

typeof null; // 'object';

Pour vérifier correctement si une valeur est “nulle”, comparez-la avec l’[opérateur d’égalité stricte][1]

var a = null;

a === null; // true

[1] : https://www.wikiod.com/fr/javascript/operations-de-comparaison

NaN #

NaN signifie “Not a Number”. Lorsqu’une fonction ou une opération mathématique en JavaScript ne peut pas renvoyer un nombre spécifique, elle renvoie la valeur “NaN” à la place.

C’est une propriété de l’objet global et une référence à [Number.NaN][1]

window.hasOwnProperty('NaN'); // true
NaN; // NaN

Peut-être déroutant, “NaN” est toujours considéré comme un nombre.

typeof NaN; // 'number'

Ne vérifiez pas NaN en utilisant l’opérateur d’égalité. Voir [isNaN](https://www.wikiod.com/fr/javascript/constantes-integrees#Test de NaN à l’aide de isNaN()) à la place.

NaN == NaN  // false
NaN === NaN // false

[1] : https://developer.mozilla.org/docs/Web/JavaScript/Reference/Global_Objects/Number/NaN

indéfini et nul #

À première vue, il peut sembler que null et undefined sont fondamentalement les mêmes, mais il existe des différences subtiles mais importantes.

undefined est l’absence de valeur dans le compilateur, car là où cela devrait être une valeur, il n’y en a pas eu, comme dans le cas d’une variable non affectée.

  • undefined est une valeur globale qui représente l’absence d’une valeur assignée.
  • typeof indéfini === 'indéfini'
  • null est un objet qui indique qu’une variable a été explicitement assignée “aucune valeur”.
  • typeof null === 'objet'

Définir une variable sur “undefined” signifie que la variable n’existe effectivement pas. Certains processus, tels que la sérialisation JSON, peuvent supprimer les propriétés “non définies” des objets. En revanche, les propriétés “null” indiquées seront conservées afin que vous puissiez transmettre explicitement le concept d’une propriété “vide”.

Les éléments suivants sont évalués à “undefined” :

  • Une variable lorsqu’elle est déclarée mais non valorisée (c’est-à-dire définie)
  •     let foo;
    console.log('is undefined?', foo === undefined);
    // is undefined? true
    ```
  • Accéder à la valeur d’un bien qui n’existe pas
  •     let foo = { a: 'a' };
    console.log('is undefined?', foo.b === undefined);
    // is undefined? true
    ```
  • La valeur de retour d’une fonction qui ne retourne pas de valeur
  •     function foo() { return; }
    console.log('is undefined?', foo() === undefined);
    // is undefined? true
    ```
  • La valeur d’un argument de fonction qui est déclaré mais a été omis de l’appel de fonction
  •     function foo(param) { 
      console.log('is undefined?', param === undefined);
    }
    foo('a');
    foo();
    // is undefined? false
    // is undefined? true
    ```

undefined est également une propriété de l’objet global window.

// Only in browsers
console.log(window.undefined); // undefined
window.hasOwnProperty('undefined'); // true

Avant ECMAScript 5, vous pouviez en fait changer la valeur de la propriété window.undefined en toute autre valeur susceptible de tout casser.

Infini et -Infini #

1 / 0; // Infinity
// Wait! WHAAAT?

Infinity est une propriété de l’objet global (donc une variable globale) qui représente l’infini mathématique. C’est une référence à Number.POSITIVE_INFINITY

Elle est supérieure à toute autre valeur, et vous pouvez l’obtenir en divisant par 0 ou en évaluant l’expression d’un nombre si grand qu’il déborde. Cela signifie en fait qu’il n’y a pas d’erreurs de division par 0 en JavaScript, il y a Infinity !

Il y a aussi -Infinity qui est l’infini mathématique négatif, et il est inférieur à toute autre valeur.

Pour obtenir -Infinity, annulez Infinity ou obtenez une référence à celui-ci dans Number.NEGATIVE_INFINITY.

- (Infinity); // -Infinity

Maintenant, amusons-nous avec des exemples :

Infinity > 123192310293; // true
-Infinity < -123192310293; // true
1 / 0; // Infinity
Math.pow(123123123, 9123192391023); // Infinity
Number.MAX_VALUE * 2; // Infinity
23 / Infinity; // 0
-Infinity; // -Infinity
-Infinity === Number.NEGATIVE_INFINITY; // true
-0; // -0 , yes there is a negative 0 in the language
0 === -0; // true
1 / -0; // -Infinity
1 / 0 === 1 / -0; // false
Infinity + Infinity; // Infinity

var a = 0, b = -0;

a === b; // true
1 / a === 1 / b; // false

// Try your own!

Constantes numériques #

Le constructeur Number a des constantes intégrées qui peuvent être utiles

Number.MAX_VALUE;          // 1.7976931348623157e+308
Number.MAX_SAFE_INTEGER;   // 9007199254740991

Number.MIN_VALUE;          // 5e-324
Number.MIN_SAFE_INTEGER;   // -9007199254740991

Number.EPSILON;            // 0.0000000000000002220446049250313

Number.POSITIVE_INFINITY;  // Infinity
Number.NEGATIVE_INFINITY;  // -Infinity

Number.NaN;                // NaN

Dans de nombreux cas, les différents opérateurs de Javascript rompront avec des valeurs en dehors de la plage de (Number.MIN_SAFE_INTEGER, Number.MAX_SAFE_INTEGER)

Notez que Number.EPSILON représente la différence entre un et le plus petit Number supérieur à un, et donc la plus petite différence possible entre deux valeurs Number différentes. L’une des raisons de l’utiliser est due à la nature de la façon dont les nombres sont stockés par JavaScript voir [Vérifier l’égalité de deux nombres](https://www.wikiod.com/fr/javascript -de-deux-nombres#t=201607291343405411584)
