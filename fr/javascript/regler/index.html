L’objet Set vous permet de stocker des valeurs uniques de tout type, qu’il s’agisse de valeurs primitives ou de références d’objet.

Les objets d’ensemble sont des collections de valeurs. Vous pouvez parcourir les éléments d’un ensemble dans l’ordre d’insertion. Une valeur dans l’ensemble ne peut apparaître qu’UNE FOIS ; il est unique dans la collection Set. Les valeurs distinctes sont discriminées à l’aide de l’algorithme de comparaison SameValueZero.

[Spécification standard sur l’ensemble] [1]

[1] : http://www.ecma-international.org/ecma-262/6.0/#sec-set-objects

Syntaxe #

  • nouvel ensemble([itérable])
  • mySet.add(valeur)
  • monEnsemble.clear()
  • mySet.delete(valeur)
  • monEnsemble.entrées()
  • mySet.forEach(rappel[, thisArg])
  • mySet.has(valeur)
  • monEnsemble.valeurs()

Paramètres #

Paramètre Détails
iterable Si un objet itérable est passé, tous ses éléments seront ajoutés au nouvel ensemble. null est traité comme indéfini.
valeur La valeur de l’élément à ajouter à l’objet Set.
rappel Fonction à exécuter pour chaque élément.
thisArg Optionnel. Valeur à utiliser comme this lors de l’exécution du rappel.

Étant donné que chaque valeur de l’ensemble doit être unique, l’égalité des valeurs sera vérifiée et n’est pas basée sur le même algorithme que celui utilisé dans l’opérateur ===. Plus précisément, pour les ensembles, +0 (qui est strictement égal à -0) et -0 sont des valeurs différentes. Cependant, cela a été modifié dans la dernière spécification ECMAScript 6. À partir de Gecko 29.0 (Firefox 29 / Thunderbird 29 / SeaMonkey 2.26) (bogue 952870) et d’un Chrome nocturne récent, +0 et -0 sont traités comme la même valeur dans les objets Set. De plus, NaN et undefined peuvent également être stockés dans un Set. NaN est considéré comme identique à NaN (même si NaN !== NaN).

Création d’un ensemble #

L’objet Set vous permet de stocker des valeurs uniques de tout type, qu’il s’agisse de valeurs primitives ou de références d’objet.

Vous pouvez pousser des éléments dans un ensemble et les itérer de la même manière qu’un tableau JavaScript simple, mais contrairement à un tableau, vous ne pouvez pas ajouter de valeur à un ensemble si la valeur existe déjà dans celui-ci.

Pour créer un nouvel ensemble :

const mySet = new Set();

Ou vous pouvez créer un ensemble à partir de n’importe quel objet itérable pour lui donner des valeurs de départ :

const arr = [1,2,3,4,4,5];
const mySet = new Set(arr);

Dans l’exemple ci-dessus, le contenu défini serait {1, 2, 3, 4, 5}. Notez que la valeur 4 n’apparaît qu’une seule fois, contrairement au tableau d’origine utilisé pour le créer.

Ajouter une valeur à un ensemble #

Pour ajouter une valeur à un Set, utilisez la méthode .add() :

mySet.add(5);

Si la valeur existe déjà dans l’ensemble, elle ne sera pas ajoutée à nouveau, car les ensembles contiennent des valeurs uniques.

Notez que la méthode .add() renvoie l’ensemble lui-même, vous pouvez donc enchaîner les appels d’ajout :

mySet.add(1).add(2).add(3);

Supprimer une valeur d’un ensemble #

Pour supprimer une valeur d’un ensemble, utilisez la méthode .delete() :

mySet.delete(some_val);

Cette fonction renverra true si la valeur existait dans l’ensemble et a été supprimée, ou false sinon.

Vérifier si une valeur existe dans un ensemble #

Pour vérifier si une valeur donnée existe dans un ensemble, utilisez la méthode .has() :

mySet.has(someVal);

Retournera true si someVal apparaît dans l’ensemble, false sinon.

Effacer un ensemble #

Vous pouvez supprimer tous les éléments d’un ensemble en utilisant la méthode .clear() :

mySet.clear();

Obtenir la longueur définie #

Vous pouvez obtenir le nombre d’éléments à l’intérieur de l’ensemble en utilisant la propriété .size

const mySet = new Set([1, 2, 2, 3]);
mySet.add(4);
mySet.size; // 4

Cette propriété, contrairement à Array.prototype.length, est en lecture seule, ce qui signifie que vous ne pouvez pas la modifier en lui affectant quelque chose :

mySet.size = 5;
mySet.size; // 4

En mode strict, il génère même une erreur :

TypeError: Cannot set property size of #<Set> which has only a getter

Conversion d’ensembles en tableaux #

Parfois, vous devrez peut-être convertir un Set en tableau, par exemple pour pouvoir utiliser les méthodes Array.prototype comme .filter(). Pour ce faire, utilisez [Array.from()](https://www.wikiod.com/fr/javascript/tableaux#Conversion d’objets de type tableau en tableaux -array#t=201608050855343146834) ou destructuring-assignment :

var mySet = new Set([1, 2, 3, 4]);
//use Array.from
const myArray = Array.from(mySet);
//use destructuring-assignment
const myArray = [...mySet];

Vous pouvez maintenant filtrer le tableau pour qu’il ne contienne que des nombres pairs et le reconvertir en Set à l’aide du constructeur Set :

mySet = new Set(myArray.filter(x => x % 2 === 0));

mySet ne contient désormais que des nombres pairs :

console.log(mySet); // Set {2, 4}

Intersection et différence dans les ensembles #

Il n’y a pas de méthodes intégrées pour l’intersection et la différence dans les ensembles, mais vous pouvez toujours y parvenir, mais en les convertissant en tableaux, en les filtrant et en les reconvertissant en ensembles :

var set1 = new Set([1, 2, 3, 4]),
    set2 = new Set([3, 4, 5, 6]);

const intersection = new Set(Array.from(set1).filter(x => set2.has(x)));//Set {3, 4}
const difference = new Set(Array.from(set1).filter(x => !set2.has(x))); //Set {1, 2}

Ensembles itératifs #

Vous pouvez utiliser une simple boucle for-of pour itérer un Set :

const mySet = new Set([1, 2, 3]);

for (const value of mySet) {
  console.log(value); // logs 1, 2 and 3
}

Lors de l’itération sur un ensemble, il renverra toujours les valeurs dans l’ordre dans lequel elles ont été ajoutées pour la première fois à l’ensemble. Par exemple:

const set = new Set([4, 5, 6])
set.add(10)
set.add(5) //5 already exists in the set
Array.from(set) //[4, 5, 6, 10]

Il existe également une méthode .forEach(), similaire à Array.prototype.forEach(). Il a deux paramètres, callback, qui seront exécutés pour chaque élément, et thisArg facultatif, qui sera utilisé comme this lors de l’exécution de callback.

callback a trois arguments. Les deux premiers arguments sont à la fois l’élément actuel de Set (par cohérence avec Array.prototype.forEach() et Map.prototype.forEach()) et le troisième argument est l’ensemble lui-même.

mySet.forEach((value, value2, set) => console.log(value)); // logs 1, 2 and 3
