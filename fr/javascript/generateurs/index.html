Les fonctions de générateur (définies par le mot-clé function*) s’exécutent comme des coroutines, générant une série de valeurs lorsqu’elles sont demandées via un itérateur.

Syntaxe #

  • fonction* nom(paramètres) { valeur de rendement ; valeur de retour }
  • générateur = nom(arguments)
  • { valeur, fait } = générateur.next (valeur)
  • { valeur, fait } = générateur.retour (valeur)
  • générateur.throw (erreur)

Les fonctions de générateur sont une fonctionnalité introduite dans le cadre de la spécification ES 2015 et ne sont pas disponibles dans tous les navigateurs. Ils sont également entièrement pris en charge dans Node.js à partir de la v6.0. Pour une liste détaillée de compatibilité des navigateurs, consultez la Documentation MDN, et pour Node, consultez le site Web node.green.

Fonctions du générateur #

Une fonction génératrice est créée avec une déclaration function*. Lorsqu’il est appelé, son corps n’est pas immédiatement exécuté. Au lieu de cela, il renvoie un objet générateur, qui peut être utilisé pour « parcourir » l’exécution de la fonction.

Une expression yield à l’intérieur du corps de la fonction définit un point auquel l’exécution peut être suspendue et reprise.

function* nums() {
    console.log('starting');  // A
    yield 1;                  // B
    console.log('yielded 1'); // C
    yield 2;                  // D
    console.log('yielded 2'); // E
    yield 3;                  // F
    console.log('yielded 3'); // G
}
var generator = nums(); // Returns the iterator. No code in nums is executed

generator.next();  // Executes lines A,B returning { value: 1, done: false }
// console: "starting"
generator.next();  // Executes lines C,D returning { value: 2, done: false }
// console: "yielded 1"
generator.next();  // Executes lines E,F returning { value: 3, done: false }
// console: "yielded 2"
generator.next();  // Executes line G returning { value: undefined, done: true }
// console: "yielded 3"

Sortie d’itération anticipée #

generator = nums();
generator.next(); // Executes lines A,B returning { value: 1, done: false }
generator.next(); // Executes lines C,D returning { value: 2, done: false }
generator.return(3); // no code is executed returns { value: 3, done: true }
// any further calls will return done = true 
generator.next(); // no code executed returns { value: undefined, done: true }

Lancer une erreur à la fonction du générateur #

function* nums() {
    try {
        yield 1;                  // A
        yield 2;                  // B
        yield 3;                  // C
    } catch (e) {
        console.log(e.message);    // D
    }
}

var generator = nums();

generator.next();  // Executes line A returning { value: 1, done: false }
generator.next();  // Executes line B returning { value: 2, done: false }
generator.throw(new Error("Error!!"));  // Executes line D returning { value: undefined, done: true}
// console: "Error!!"
generator.next();  // no code executed. returns { value: undefined, done: true }

Envoi des valeurs au générateur #

Il est possible d’envoyer une valeur au générateur en la passant à la méthode next().

function* summer() {
    let sum = 0, value;
    while (true) {
        // receive sent value
        value = yield;
        if (value === null) break;

        // aggregate values
        sum += value;
    }
    return sum;
}
let generator = summer();

// proceed until the first "yield" expression, ignoring the "value" argument
generator.next();

// from this point on, the generator aggregates values until we send "null"
generator.next(1);
generator.next(10);
generator.next(100);

// close the generator and collect the result
let sum = generator.next(null).value;  // 111

Déléguer à un autre générateur #

Depuis une fonction génératrice, le contrôle peut être délégué à une autre fonction génératrice à l’aide de yield*.

function* g1() {
  yield 2;
  yield 3;
  yield 4;
}

function* g2() {
  yield 1;
  yield* g1();
  yield 5;
}

var it = g2();

console.log(it.next()); // 1
console.log(it.next()); // 2
console.log(it.next()); // 3
console.log(it.next()); // 4
console.log(it.next()); // 5
console.log(it.next()); // undefined

Itération #

Un générateur est itérable. Il peut être bouclé avec une instruction for...of et utilisé dans d’autres constructions qui dépendent du protocole d’itération.

function* range(n) {
    for (let i = 0; i < n; ++i) {
        yield i;
    }
}

// looping
for (let n of range(10)) {
    // n takes on the values 0, 1, ... 9
}

// spread operator
let nums = [...range(3)];  // [0, 1, 2]
let max = Math.max(...range(100));  // 99

Voici un autre exemple d’utilisation du générateur pour personnaliser un objet itérable dans ES6. Ici, la fonction de générateur anonyme function * est utilisée.

let user = {
  name: "sam", totalReplies: 17, isBlocked: false
};

user[Symbol.iterator] = function *(){

  let properties = Object.keys(this);
  let count = 0;
  let isDone = false;

  for(let p of properties){
    yield this[p];
  }
};

for(let p of user){
  console.log( p );
}

Flux asynchrone avec les générateurs #

Les générateurs sont des fonctions capables de mettre en pause puis de reprendre l’exécution. Cela permet d’émuler des fonctions asynchrones à l’aide de bibliothèques externes, principalement q ou co. En gros, cela permet d’écrire des fonctions qui attendent des résultats asynchrones pour continuer :

function someAsyncResult() {
    return Promise.resolve('newValue')
}

q.spawn(function * () {
    var result = yield someAsyncResult()
    console.log(result) // 'newValue'
})

Cela permet d’écrire du code asynchrone comme s’il était synchrone. De plus, essayez d’attraper le travail sur plusieurs blocs asynchrones. Si la promesse est rejetée, l’erreur est interceptée par le catch suivant :

function asyncError() {
    return new Promise(function (resolve, reject) {
        setTimeout(function () {
            reject(new Error('Something went wrong'))
        }, 100)
    })
}

q.spawn(function * () {
    try {
        var result = yield asyncError()
    } catch (e) {
        console.error(e) // Something went wrong
    }
})

Utiliser co fonctionnerait exactement de la même manière mais avec co(function * (){...}) au lieu de q.spawn

Interface itérateur-observateur #

Un générateur est une combinaison de deux choses - un Iterator et un Observer.

Itérateur #

Un itérateur est quelque chose qui, lorsqu’il est invoqué, renvoie un “itérable”. Un itérable est quelque chose sur lequel vous pouvez itérer. Depuis ES6/ES2015, toutes les collections (Array, Map, Set, WeakMap, WeakSet) sont conformes au contrat Iterable.

Un générateur (itérateur) est un producteur. Dans l’itération, le consommateur “PULL” est la valeur du producteur.

Exemple:

function *gen() { yield 5; yield 6; }
let a = gen();

Chaque fois que vous appelez a.next(), vous tirez essentiellement la valeur de l’itérateur et mettez en pause l’exécution à yield. La prochaine fois que vous appelez a.next(), l’exécution reprend à partir de l’état précédemment interrompu.

Observateur #

Un générateur est également un observateur à l’aide duquel vous pouvez renvoyer certaines valeurs dans le générateur.

function *gen() {
  document.write('<br>observer:', yield 1);
}
var a = gen();
var i = a.next();
while(!i.done) {
  document.write('<br>iterator:', i.value);
  i = a.next(100);
}

Ici, vous pouvez voir que yield 1 est utilisé comme une expression qui prend une certaine valeur. La valeur qu’il évalue est la valeur envoyée comme argument à l’appel de fonction a.next.

Ainsi, pour la première fois i.value sera la première valeur obtenue (1), et lors de la poursuite de l’itération vers l’état suivant, nous renvoyons une valeur au générateur en utilisant a.next(100) .

Faire de l’async avec les générateurs #

Les générateurs sont largement utilisés avec la fonction spawn (de taskJS ou co), où la fonction prend un générateur et nous permet d’écrire du code asynchrone de manière synchrone. Cela ne signifie PAS que le code asynchrone est converti en code de synchronisation / exécuté de manière synchrone. Cela signifie que nous pouvons écrire du code qui ressemble à sync mais qu’en interne, il est toujours async.

La synchronisation BLOQUE ; Async est EN ATTENTE. Écrire du code qui bloque est facile. Lors du PULLing, la valeur apparaît dans la position d’affectation. Lorsque PUSHing, la valeur apparaît dans la position d’argument du rappel.

Lorsque vous utilisez des itérateurs, vous PULL la valeur du producteur. Lorsque vous utilisez des rappels, le producteur PUSH es la valeur à la position d’argument du rappel.

var i = a.next() // PULL
dosomething(..., v => {...}) // PUSH

Ici, vous tirez la valeur de a.next() et dans la seconde, v => {...} est le rappel et une valeur est PUSHed dans la position d’argument v du rappel fonction.

En utilisant ce mécanisme pull-push, nous pouvons écrire une programmation asynchrone comme celle-ci,

let delay = t => new Promise(r => setTimeout(r, t));
spawn(function*() {
  // wait for 100 ms and send 1
  let x = yield delay(100).then(() => 1);
  console.log(x); // 1

   // wait for 100 ms and send 2
  let y = yield delay(100).then(() => 2);
  console.log(y); // 2
});

Donc, en regardant le code ci-dessus, nous écrivons du code asynchrone qui semble “bloquant” (les instructions de rendement attendent 100 ms, puis continuent leur exécution), mais il est en fait “en attente”. Les propriétés pause et resume du générateur nous permettent de faire cette astuce incroyable.

Comment ça marche ? #

La fonction spawn utilise yield promise pour extraire l’état de la promesse du générateur, attend que la promesse soit résolue et renvoie la valeur résolue au générateur afin qu’il puisse la consommer.

Utilise le maintenant #

Ainsi, avec les générateurs et la fonction spawn, vous pouvez nettoyer tout votre code asynchrone dans NodeJS pour qu’il ait l’air synchrone. Cela facilitera le débogage. De plus, le code aura l’air soigné.

Cette fonctionnalité sera disponible dans les futures versions de JavaScript - en tant que “async…wait”. Mais vous pouvez les utiliser aujourd’hui dans ES2015/ES6 en utilisant la fonction spawn définie dans les bibliothèques - taskjs, co ou bluebird
