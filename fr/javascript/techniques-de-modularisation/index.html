Définition de module universel (UMD) #

Le modèle UMD (Universal Module Definition) est utilisé lorsque notre module doit être importé par un certain nombre de chargeurs de modules différents (par exemple AMD, CommonJS).

Le modèle lui-même se compose de deux parties :

  1. Un IIFE (Immediately-Invoked Function Expression) qui vérifie le chargeur de module mis en œuvre par l’utilisateur. Cela prendra deux arguments; root (une référence this à la portée globale) et factory (la fonction où nous déclarons notre module).

  2. Une fonction anonyme qui crée notre module. Ceci est passé comme deuxième argument à la partie IIFE du modèle. Cette fonction reçoit un nombre quelconque d’arguments pour spécifier les dépendances du module.

Dans l’exemple ci-dessous, nous vérifions AMD, puis CommonJS. Si aucun de ces chargeurs n’est utilisé, nous nous rabattons sur la disponibilité globale du module et de ses dépendances.

(function (root, factory) {
    if (typeof define === 'function' && define.amd) {
        // AMD. Register as an anonymous module.
        define(['exports', 'b'], factory);
    } else if (typeof exports === 'object' && typeof exports.nodeName !== 'string') {
        // CommonJS
        factory(exports, require('b'));
    } else {
        // Browser globals
        factory((root.commonJsStrict = {}), root.b);
    }
}(this, function (exports, b) {
    //use b in some fashion.

    // attach properties to the exports object to define
    // the exported module properties.
    exports.action = function () {};
}));

Expressions de fonction immédiatement appelées (IIFE) #

Les expressions de fonction appelées immédiatement peuvent être utilisées pour créer une étendue privée tout en produisant une API publique.

var Module = (function() {
  var privateData = 1;

  return {
    getPrivateData: function() {
      return privateData;
    }
  };
})();
Module.getPrivateData(); // 1
Module.privateData; // undefined

Voir le [modèle de module] (https://www.wikiod.com/fr/javascript/modeles-de-conception-de-creation#Modèles de modules et de modules révélateurs) pour plus de détails.

Définition de module asynchrone (AMD) #

AMD est un système de définition de module qui tente de résoudre certains des problèmes courants avec d’autres systèmes tels que CommonJS et les fermetures anonymes.

AMD résout ces problèmes en :

  • Enregistrement de la fonction d’usine en appelant define(), au lieu de l’exécuter immédiatement
  • Passage des dépendances sous forme de tableau de noms de modules, qui sont ensuite chargés, au lieu d’utiliser des globals
  • Exécuter la fonction d’usine uniquement une fois que toutes les dépendances ont été chargées et exécutées
  • Passer les modules dépendants comme arguments à la fonction factory

L’essentiel ici est qu’un module peut avoir une dépendance et ne pas tout bloquer en attendant qu’il se charge, sans que le développeur n’ait à écrire de code compliqué.

Voici un exemple d’AMD :

// Define a module "myModule" with two dependencies, jQuery and Lodash
define("myModule", ["jquery", "lodash"], function($, _) {
    // This publicly accessible object is our module
    // Here we use an object, but it can be of any type
    var myModule = {};

    var privateVar = "Nothing outside of this module can see me";

    var privateFn = function(param) {
        return "Here's what you said: " + param;
    };

    myModule.version = 1;

    myModule.moduleMethod = function() {
        // We can still access global variables from here, but it's better
        // if we use the passed ones
        return privateFn(windowTitle);
    };

    return myModule;
});

Les modules peuvent également ignorer le nom et être anonymes. Lorsque cela est fait, ils sont généralement chargés par nom de fichier.

define(["jquery", "lodash"], function($, _) { /* factory */ });

Ils peuvent également ignorer les dépendances :

define(function() { /* factory */ });

Certains chargeurs AMD prennent en charge la définition de modules en tant qu’objets simples :

define("myModule", { version: 1, value: "sample string" });

CommonJS - Node.js #

CommonJS est un modèle de modularisation populaire utilisé dans Node.js.

Le système CommonJS est centré sur une fonction require() qui charge d’autres modules et une propriété exports qui permet aux modules d’exporter des méthodes accessibles au public.

Voici un exemple de CommonJS, nous allons charger le module fs de Lodash et Node.js :

// Load fs and lodash, we can use them anywhere inside the module
var fs = require("fs"),
    _ = require("lodash");

var myPrivateFn = function(param) {
    return "Here's what you said: " + param;
};

// Here we export a public `myMethod` that other modules can use
exports.myMethod = function(param) {
    return myPrivateFn(param);
};

Vous pouvez également exporter une fonction en tant que module entier en utilisant module.exports :

module.exports = function() {
    return "Hello!";
};

Modules ES6 #

Dans ECMAScript 6, lors de l’utilisation de la syntaxe du module (import/export), chaque fichier devient son propre module avec un espace de noms privé. Les fonctions et variables de niveau supérieur ne polluent pas l’espace de noms global. Pour exposer des fonctions, des classes et des variables pour d’autres modules à importer, vous pouvez utiliser le mot-clé export.

Remarque : Bien qu’il s’agisse de la méthode officielle de création de modules JavaScript, elle n’est actuellement prise en charge par aucun des principaux navigateurs. Cependant, les modules ES6 sont pris en charge par de nombreux transpileurs.

export function greet(name) {
    console.log("Hello %s!", name);
}

var myMethod = function(param) {
    return "Here's what you said: " + param;
};

export {myMethod}

export class MyClass {
    test() {}
}

Utiliser les modules #

L’importation de modules est aussi simple que de spécifier leur chemin :

import greet from "mymodule.js";

greet("Bob");

Cela importe uniquement la méthode myMethod de notre fichier mymodule.js.

Il est également possible d’importer toutes les méthodes d’un module :

import * as myModule from "mymodule.js";

myModule.greet("Alice");

Vous pouvez également importer des méthodes sous un nouveau nom :

import { greet as A, myMethod as B } from "mymodule.js";

Vous trouverez plus d’informations sur les modules ES6 dans la rubrique Modules.
