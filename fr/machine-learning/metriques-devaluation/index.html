Zone sous la courbe de la caractéristique de fonctionnement du récepteur (AUROC) #

L’AUROC est l’une des métriques les plus couramment utilisées pour évaluer les performances d’un classifieur. Cette section explique comment le calculer.

AUC (Area Under the Curve) est utilisé la plupart du temps pour signifier AUROC, ce qui est une mauvaise pratique car AUC est ambigu (il peut s’agir de n’importe quelle courbe) alors qu’AUROC ne l’est pas.

Aperçu – Abréviations #

Abréviation Signification
AUROC Aire sous la courbe de la caractéristique de fonctionnement du récepteur
ASC Quartier Sous la Cure
ROC Caractéristique de fonctionnement du récepteur
TP Vrais positifs
TN Vrais négatifs
PF Faux positifs
FN Faux négatifs
TPR Taux de vrais positifs
FPR Taux de faux positifs

Interpréter l’AUROC #

L’AUROC a [plusieurs interprétations équivalentes] (http://metaoptimize.com/qa/questions/988/simple-explanation-of-area-under-the-roc-curve):

  • L’attente qu’un positif aléatoire uniformément tiré soit classé avant un négatif aléatoire uniformément tiré.
  • La proportion attendue de positifs classés avant un négatif aléatoire tiré uniformément.
  • Le taux de vrais positifs attendu si le classement est divisé juste avant un négatif aléatoire tiré uniformément.
  • La proportion attendue de négatifs classés après un positif aléatoire uniformément tiré.
  • Le taux de faux positifs attendu si le classement est divisé juste après un positif aléatoire tiré uniformément.

Calcul de l’AUROC #

Supposons que nous ayons un classificateur binaire probabiliste tel qu’une régression logistique.

Avant de présenter la courbe ROC (= courbe Receiver Operating Characteristic), il faut comprendre la notion de matrice de confusion. Lorsque nous faisons une prédiction binaire, il peut y avoir 4 types de résultats :

  • Nous prédisons 0 alors que la classe est en réalité 0 : c’est ce qu’on appelle un vrai négatif, c’est-à-dire que nous prédisons correctement que la classe est négative (0). Par exemple, un antivirus n’a pas détecté un fichier inoffensif comme un virus.
  • Nous prédisons 0 alors que la classe est en réalité 1 : c’est ce qu’on appelle un Faux Négatif, c’est-à-dire que nous prédisons à tort que la classe est négative (0). Par exemple, un antivirus n’a pas réussi à détecter un virus.
  • Nous prédisons 1 alors que la classe est en réalité 0 : c’est ce qu’on appelle un Faux Positif, c’est-à-dire que nous prédisons à tort que la classe est positive (1). Par exemple, un antivirus considère qu’un fichier inoffensif est un virus.
  • Nous prédisons 1 alors que la classe est en réalité 1 : c’est ce qu’on appelle un vrai positif, c’est-à-dire que nous prédisons correctement que la classe est positive (1). Par exemple, un antivirus a légitimement détecté un virus.

Pour obtenir la matrice de confusion, nous passons en revue toutes les prédictions faites par le modèle et comptons combien de fois chacun de ces 4 types de résultats se produit :

![entrez la description de l’image ici][1]

Dans cet exemple de matrice de confusion, parmi les 50 points de données classés, 45 sont correctement classés et les 5 sont mal classés.

Comme pour comparer deux modèles différents, il est souvent plus pratique d’avoir une seule métrique plutôt que plusieurs, nous calculons deux métriques à partir de la matrice de confusion, que nous combinerons plus tard en une seule :

  • Taux de vrais positifs (TPR), alias. sensibilité, hit rate et recall, qui est défini comme ![ formule][3]. Intuitivement, cette métrique correspond à la proportion de points de données positifs qui sont correctement considérés comme positifs, par rapport à tous les points de données positifs. En d’autres termes, plus le TPR est élevé, moins nous manquerons de points de données positifs.
  • Taux de faux positifs (FPR), alias. fall-out, qui est défini comme ![un autre][4]. Intuitivement, cette métrique correspond à la proportion de points de données négatifs considérés à tort comme positifs, par rapport à tous les points de données négatifs. En d’autres termes, plus le FPR est élevé, plus les points de données négatifs seront mal classés.

Pour combiner le FPR et le TPR en une seule métrique, nous calculons d’abord les deux anciennes métriques avec de nombreux seuils différents (par exemple ![][5]) pour la régression logistique, puis nous les traçons sur un seul graphique, avec les valeurs FPR en abscisse et les valeurs de TPR en ordonnée. La courbe résultante est appelée courbe ROC, et la métrique que nous considérons est l’AUC de cette courbe, que nous appelons AUROC.

La figure suivante montre graphiquement l’AUROC :

![entrez la description de l’image ici][2]

Dans cette figure, la zone bleue correspond à la zone sous la courbe de la caractéristique de fonctionnement du récepteur (AUROC). La ligne pointillée dans la diagonale nous présente la courbe ROC d’un prédicteur aléatoire : elle a un AUROC de 0,5. Le prédicteur aléatoire est couramment utilisé comme référence pour voir si le modèle est utile.

Matrice de confusion #

Une matrice de confusion peut être utilisée pour évaluer un classificateur, sur la base d’un ensemble de données de test pour lesquelles les vraies valeurs sont connues. C’est un outil simple, qui aide à donner un bon aperçu visuel des performances de l’algorithme utilisé.

Une matrice de confusion est représentée sous forme de tableau. Dans cet exemple, nous examinerons une matrice de confusion pour un classificateur binaire.

[![Un exemple de matrice de confusion][1]][1]

Sur le côté gauche, on peut voir la classe réelle (étiquetée comme OUI ou NON), tandis que le haut indique la classe prédite et sortie (encore une fois OUI ou NON).

Cela signifie que 50 instances de test - qui sont en fait des instances NO, ont été correctement étiquetées par le classificateur comme NO. Ceux-ci sont appelés les vrais négatifs (TN). En revanche, 100 instances OUI réelles ont été correctement étiquetées par le classificateur comme instances OUI. Ceux-ci sont appelés les vrais positifs (TP).

5 instances OUI réelles ont été mal étiquetées par le classificateur. C’est ce qu’on appelle les faux négatifs (FN). De plus, 10 instances NON ont été considérées comme des instances OUI par le classificateur, ce sont donc des faux positifs (FP).

Sur la base de ces FP,TP,FN et TN, nous pouvons tirer d’autres conclusions.

  • ** Taux de vrais positifs ** :

    • Tries to answer: When an instance is actually YES, how often does the classifier predict YES?

  • Peut être calculé comme suit : TP/# instances réelles OUI = 100/105 = 0,95

  • Taux de faux positifs :

    • Tries to answer: When an instance is actually NO, how often does the classifier predict YES?

  • Peut être calculé comme suit : FP/# instances réelles NO = 10/60 = 0,17

Courbes ROC #

Une courbe ROC (Receiver Operating Characteristic) trace le taux de TP par rapport au Le taux de FP en tant que seuil sur la confiance qu’une instance est positive est varié [![exemple ROC][1]][1]

[1] : http://i.stack.imgur.com/l9dsj.png

Algorithme de création d’une courbe ROC

  1. trier les prédictions des ensembles de tests en fonction de la confiance que l’exemple est positif

  2. parcourir la liste triée de la confiance élevée à faible

    i. locate a threshold between instances with opposite classes (keeping instances with the same confidence value on the same side of threshold)

    ii. compute TPR, FPR for instances above threshold

    iii. output (FPR, TPR) coordinate
