Implémentation d’un modèle Perceptron en C++ #

Dans cet exemple, je vais passer en revue l’implémentation du modèle perceptron en C++ afin que vous puissiez avoir une meilleure idée de son fonctionnement.

Tout d’abord, c’est une bonne pratique d’écrire un algorithme simple de ce que nous voulons faire.

Algorithme:

  1. Créez un vecteur pour les poids et initialisez-le à 0 (n’oubliez pas d’ajouter le terme de biais)
  2. Continuez à ajuster les pondérations jusqu’à ce que nous obtenions 0 erreur ou un faible nombre d’erreurs.
  3. Faites des prédictions sur des données inédites.

Après avoir écrit un algorithme super simple, écrivons maintenant certaines des fonctions dont nous aurons besoin.

  • Nous aurons besoin d’une fonction pour calculer l’entrée du réseau (e.i x * wT multipliant les entrées par les poids)
  • Une fonction en escalier pour obtenir une prédiction de 1 ou -1
  • Et une fonction qui trouve les valeurs idéales pour les poids.

Alors sans plus tarder, allons-y.

Commençons simplement en créant une classe perceptron :

class perceptron
{
public:
    
private:

};

Ajoutons maintenant les fonctions dont nous aurons besoin.

class perceptron
{
public:
    perceptron(float eta,int epochs);
    float netInput(vector<float> X);
    int predict(vector<float> X);
    void fit(vector< vector<float> > X, vector<float> y);
private:

};

Remarquez comment la fonction fit prend comme argument un vecteur de vector< float >. En effet, notre ensemble de données d’entraînement est une matrice d’entrées. Essentiellement, nous pouvons imaginer cette matrice comme un couple de vecteurs x empilés les uns sur les autres et chaque colonne de cette matrice étant une caractéristique.

Enfin, ajoutons les valeurs que notre classe doit avoir. Tels que le vecteur w pour contenir les poids, le nombre d’époques qui indique le nombre de passages que nous ferons sur l’ensemble de données d’apprentissage. Et la constante eta qui est le taux d’apprentissage dont nous multiplierons chaque mise à jour de poids afin d’accélérer la procédure d’entraînement en composant cette valeur ou si eta est trop élevé, nous pouvons composez-le pour obtenir le résultat idéal (pour la plupart des applications du perceptron, je suggérerais une valeur eta de 0,1).

class perceptron
{
public:
    perceptron(float eta,int epochs);
    float netInput(vector<float> X);
    int predict(vector<float> X);
    void fit(vector< vector<float> > X, vector<float> y);
private:
    float m_eta;
    int m_epochs;
    vector < float > m_w;
};

Maintenant avec notre jeu de classe. Il est temps d’écrire chacune des fonctions.

Nous partirons du constructeur ( perceptron(float eta,int epochs); )

perceptron::perceptron(float eta, int epochs)
{
    m_epochs = epochs; // We set the private variable m_epochs to the user selected value
    m_eta = eta; // We do the same thing for eta
}

Comme vous pouvez le voir, ce que nous allons faire est très simple. Passons donc à une autre fonction simple. La fonction de prédiction ( int predict(vector X); ). Rappelez-vous que la fonction all predict prend l’entrée nette et renvoie une valeur de 1 si netInput est supérieur à 0 et -1 sinon.

int perceptron::predict(vector<float> X)
{
    return netInput(X) > 0 ? 1 : -1; //Step Function
}

Notez que nous avons utilisé une instruction if en ligne pour nous faciliter la vie. Voici comment fonctionne l’instruction if en ligne :

condition ? if_true : sinon

Jusqu’ici tout va bien. Passons à l’implémentation de la fonction netInput ( float netInput(vector X); )

Le netInput fait ce qui suit ; multiplie le vecteur d’entrée par la transposition du vecteur de poids

*x wT

En d’autres termes, il multiplie chaque élément du vecteur d’entrée x par l’élément correspondant du vecteur de poids w, puis prend leur somme et ajoute le biais.

(x1 * w1 + x2 * w2 + … + xn * wn) + biais

biais = 1 * w0

float perceptron::netInput(vector<float> X)
{
    // Sum(Vector of weights * Input vector) + bias
    float probabilities = m_w[0]; // In this example I am adding the perceptron first
    for (int i = 0; i < X.size(); i++)
    {
        probabilities += X[i] * m_w[i + 1]; // Notice that for the weights I am counting
        // from the 2nd element since w0 is the bias and I already added it first.
    }
    return probabilities;
}

Très bien, nous avons maintenant pratiquement terminé. La dernière chose à faire est d’écrire la fonction *** fit *** qui modifie les poids.

void perceptron::fit(vector< vector<float> > X, vector<float> y)
{
    for (int i = 0; i < X[0].size() + 1; i++) // X[0].size() + 1 -> I am using +1 to add the bias term
    {
        m_w.push_back(0); // Setting each weight to 0 and making the size of the vector
        // The same as the number of features (X[0].size()) + 1 for the bias term
    }
    for (int i = 0; i < m_epochs; i++) // Iterating through each epoch
    {
        for (int j = 0; j < X.size(); j++) // Iterating though each vector in our training Matrix
        {
            float update = m_eta * (y[j] - predict(X[j])); //we calculate the change for the weights
            for (int w = 1; w < m_w.size(); w++){ m_w[w] += update * X[j][w - 1]; } // we update each weight by the update * the training sample
            m_w[0] = update; // We update the Bias term and setting it equal to the update
        }
    }
}

Donc c’était essentiellement ça. Avec seulement 3 fonctions, nous avons maintenant une classe perceptron fonctionnelle que nous pouvons utiliser pour faire des prédictions !

Le fichier de classe .cpp avec les fonctions :

perceptron::perceptron(float eta, int epochs)
{
    m_epochs = epochs;
    m_eta = eta;
}

void perceptron::fit(vector< vector<float> > X, vector<float> y)
{
    for (int i = 0; i < X[0].size() + 1; i++) // X[0].size() + 1 -> I am using +1 to add the bias term
    {
        m_w.push_back(0);
    }
    for (int i = 0; i < m_epochs; i++)
    {
        int errors = 0;
        for (int j = 0; j < X.size(); j++)
        {
            float update = m_eta * (y[j] - predict(X[j]));
            for (int w = 1; w < m_w.size(); w++){ m_w[w] += update * X[j][w - 1]; }
            m_w[0] = update;
            errors += update != 0 ? 1 : 0;
        }
        m_errors.push_back(errors);
    }
}

float perceptron::netInput(vector<float> X)
{
    // Sum(Vector of weights * Input vector) + bias
    float probabilities = m_w[0];
    for (int i = 0; i < X.size(); i++)
    {
        probabilities += X[i] * m_w[i + 1];
    }
    return probabilities;
}

int perceptron::predict(vector<float> X)
{
    return netInput(X) > 0 ? 1 : -1; //Step Function
}

void perceptron::printErrors()
{
    printVector(m_errors);
}

void perceptron::exportWeights(string filename)
{
    ofstream outFile;
    outFile.open(filename);

    for (int i = 0; i < m_w.size(); i++)
    {
        outFile << m_w[i] << endl;
    }

    outFile.close();
}

void perceptron::importWeights(string filename)
{
    ifstream inFile;
    inFile.open(filename);

    for (int i = 0; i < m_w.size(); i++)
    {
        inFile >> m_w[i];
    }
}

void perceptron::printWeights()
{
    cout << "weights: ";
    for (int i = 0; i < m_w.size(); i++)
    {
        cout << m_w[i] << " ";
    }
    cout << endl;
}

vector<float> getIrisy()
{
    vector<float> y;

Qu’est-ce qu’un perceptron exactement ? #

[![Visualisation d’un perceptron][1]][1]

À la base, un modèle perceptron est l’un des algorithmes [d’apprentissage supervisé][2] les plus simples pour la [classification binaire][3]. Il s’agit d’un type de [classificateur linéaire][4], c’est-à-dire un algorithme de classification qui fait ses prédictions sur la base d’une fonction de prédiction linéaire combinant un ensemble de poids avec le vecteur de caractéristiques. Une façon plus intuitive de penser est comme un réseau de neurones avec un seul neurone.

La façon dont cela fonctionne est très simple. Il obtient un vecteur de valeurs d’entrée x dont chaque élément est une caractéristique de notre ensemble de données.

Un exemple: #

Disons que nous voulons classer si un objet est un vélo ou une voiture. Pour les besoins de cet exemple, disons que nous voulons sélectionner 2 fonctionnalités. La hauteur et la largeur de l’objet. Dans ce cas x = [x1, x2] où x1 est la hauteur et x2 est la largeur.

Ensuite, une fois que nous avons notre vecteur d’entrée x, nous voulons multiplier chaque élément de ce vecteur par un poids. Généralement, plus la valeur du poids est élevée, plus la caractéristique est importante. Si, par exemple, nous avons utilisé couleur comme caractéristique x3 et qu’il y a un vélo rouge et une voiture rouge, le perceptron lui attribuera un poids très faible afin que la couleur n’ait pas d’impact sur le prédiction finale.

Très bien donc nous avons multiplié les 2 vecteurs x et w et nous avons récupéré un vecteur. Maintenant, nous devons additionner les éléments de ce vecteur. Une façon intelligente de le faire est au lieu de multiplier simplement x par w nous pouvons multiplier x par wT où **T ** signifie transposition. Nous pouvons imaginer la transposition d’un vecteur comme une version pivotée du vecteur. Pour plus d’informations, vous pouvez lire [la page Wikipedia][5]. Essentiellement en prenant la transposée du vecteur w nous obtenons un vecteur Nx1 au lieu d’un 1xN. Ainsi, si nous multiplions maintenant notre vecteur d’entrée de taille 1xN avec ce vecteur de poids Nx1, nous obtiendrons un vecteur 1x1 (ou simplement une valeur unique) qui sera égal à x1 * w1 + x2 * w2 + … + xn * wn. Cela fait, nous avons maintenant notre prédiction. Mais il y a une dernière chose. Cette prédiction ne sera probablement pas un simple 1 ou -1 pour pouvoir classer un nouvel échantillon. Donc, ce que nous pouvons faire, c’est simplement dire ceci : si notre prédiction est supérieure à 0, nous disons que l’échantillon appartient à la classe 1, sinon, si la prédiction est inférieure à zéro, nous disons que l’échantillon appartient à la classe -1. C’est ce qu’on appelle une [fonction échelonnée][6].

Mais comment obtenir les bonnes pondérations pour faire des prédictions correctes ? En d’autres termes, comment entraînons-nous notre modèle de perceptron ?

Eh bien, dans le cas du perceptron, nous n’avons pas besoin d’équations mathématiques fantaisistes pour *** entraîner *** notre modèle. Nos pondérations peuvent être ajustées par l’équation suivante :

Δw = eta * (y - prédiction) * x(i)

où x(i) est notre caractéristique (x1 par exemple pour le poids 1, x2 pour w2 et ainsi de suite…).

A également remarqué qu’il existe une variable appelée eta qui est le taux d’apprentissage. Vous pouvez imaginer le taux d’apprentissage comme l’ampleur que nous voulons que le changement de poids soit. Un bon taux d’apprentissage se traduit par un algorithme d’apprentissage rapide. Une valeur trop élevée de eta peut entraîner une augmentation du nombre d’erreurs à chaque époque et faire en sorte que le modèle fasse de très mauvaises prédictions et ne converge jamais. Un taux d’apprentissage trop faible peut avoir pour conséquence que le modèle mettra trop de temps à converger. (Habituellement, une bonne valeur pour définir eta pour le modèle perceptron est 0,1 mais elle peut différer d’un cas à l’autre).

Enfin, certains d’entre vous ont peut-être remarqué que la première entrée est une constante ( 1 ) et est multipliée par w0. Alors qu’est-ce que c’est exactement ? Afin d’obtenir une bonne prédiction, nous devons ajouter un biais. Et c’est exactement ce qu’est cette constante.

Pour modifier le poids du terme de biais, nous utilisons la même équation que pour les autres poids, mais dans ce cas, nous ne le multiplions pas par l’entrée (car l’entrée est une constante 1 et nous n’avons donc pas à le faire) :

Δw = eta * (y - prédiction)

Voilà donc en gros comment fonctionne un simple modèle de perceptron ! Une fois que nous avons formé nos poids, nous pouvons lui donner de nouvelles données et avoir nos prédictions.

REMARQUE: #

Le modèle Perceptron a un inconvénient important ! Il ne convergera jamais (c’est-à-dire que je trouverai les poids parfaits) si les données ne sont pas [linéairement séparables][7], ce qui signifie pouvoir séparer les 2 classes dans un espace d’entités par une ligne droite. Donc, pour éviter cela, il est recommandé d’ajouter un nombre fixe d’itérations afin que le modèle ne soit pas bloqué à ajuster des poids qui ne seront jamais parfaitement réglés.

Quel est le biais #

Un perceptron peut être vu comme une fonction qui mappe un vecteur d’entrée (valeur réelle) x à une valeur de sortie f(x) (valeur binaire) :

[![entrez la description de l’image ici][1]][1]

où w est un vecteur de poids réels et b est notre valeur de biais. Le biais est une valeur qui éloigne la limite de décision de l’origine (0,0) et qui ne dépend d’aucune valeur d’entrée.

En pensant au biais de manière spatiale, le biais modifie la position (mais pas l’orientation) de la frontière de décision. On peut voir ci-dessous un exemple de la même courbe décalée du biais :
