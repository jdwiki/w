Télécharger #

Une distribution peut être téléchargée depuis le [site Web d’Apache Struts][1]. La distribution complète contient :

  • le fichier struts2-core.jar
  • dépendances liées
  • exemples d’application
  • une copie de la documentation au format HTML
  • le code source complet.

Demande vierge

L’application Web vide dans le répertoire des applications de la distribution est conçue comme un modèle.

Il existe même un simple fichier batch dans le répertoire du code source que nous pouvons utiliser pour recompiler l’application en place.

Nous pouvons faire une copie de blank.war, le déployer dans notre conteneur et utiliser la copie éclatée comme base pour notre application.

[1] : http://struts.apache.org/downloads.html

Nouvelle application de Maven Archetype #

Pour ceux d’entre nous qui utilisent Maven comme système de construction, nous pouvons utiliser l’archétype Maven pour créer une nouvelle application.

Consultez d’abord le catalogue d’archétypes maven

mvn archetype:generate -DarchetypeCatalog=http://struts.apache.org/

vous pouvez utiliser l’un des achetypes suivants :

** L’archétype de la convention vierge (struts2-archetype-convention) **

Il a les caractéristiques suivantes :

  • Validation basée sur la convention
  • Exemples d’actions
  • Ensemble de ressources au niveau du package
  • Tests unitaires
  • Conscient de Google AppEgine

pour le télécharger via maven, exécutez la commande

mvn archetype:generate -B -DgroupId=com.mycompany.mysystem \
                            -DartifactId=myWebApp \
                            -DarchetypeGroupId=org.apache.struts \
                            -DarchetypeArtifactId=struts2-archetype-convention \
                            -DarchetypeVersion=<CURRENT_STRUTS_VERSION> \
                            -DremoteRepositories=http://struts.apache.org

L’archétype vierge (struts2-archetype-blank)

Il a les caractéristiques suivantes :

  • Configuration basée sur XML, démonstration incluant un fichier de configuration supplémentaire
  • Exemples d’actions
  • Ensemble de ressources au niveau du package
  • Validation basée sur XML
  • Tests unitaires

pour le télécharger via maven, exécutez la commande

mvn archetype:generate -B -DgroupId=com.mycompany.mysystem \
                            -DartifactId=myWebApp \
                            -DarchetypeGroupId=org.apache.struts \
                            -DarchetypeArtifactId=struts2-archetype-blank \
                            -DarchetypeVersion=<CURRENT_STRUTS_VERSION> \
                            -DremoteRepositories=http://struts.apache.org

L’archétype de départ (struts2-archetype-starter)

Il a les caractéristiques suivantes :

  • Intégration Sitemesh
  • Exemple d’action (instancié à la fois via Spring et Struts)
  • Intégration du ressort
  • Exemple de validation (niveau action et action-alias)
  • Exemple de conversion (niveau global et action)
  • Ensemble de ressources (à la fois au niveau global, au niveau de l’action et du package)

pour le télécharger via maven, exécutez la commande

mvn archetype:generate -B -DgroupId=com.mycompany.mysystem \
                            -DartifactId=myWebApp \
                            -DarchetypeGroupId=org.apache.struts \
                            -DarchetypeArtifactId=struts2-archetype-starter \
                            -DarchetypeVersion=<CURRENT_STRUTS_VERSION> \
                            -DremoteRepositories=http://struts.apache.org

** L’archétype AngularJS (struts2-archetype-angularjs) **

Il a les caractéristiques suivantes :

  • Configuration d’action basée sur la convention
  • Exemples d’actions avec résultat JSON
  • Exemple d’intégration AngularJS et Struts2

pour le télécharger via maven, exécutez la commande

mvn archetype:generate -B -DgroupId=com.mycompany.mysystem \
                            -DartifactId=myWebApp \
                            -DarchetypeGroupId=org.apache.struts \
                            -DarchetypeArtifactId=struts2-archetype-angularjs \
                            -DarchetypeVersion=<CURRENT_STRUTS_VERSION> \
                            -DremoteRepositories=http://struts.apache.org

** L’archétype vierge du portlet (struts2-archetype-portlet) **

Il a les caractéristiques suivantes :

  • Exemples de modes Afficher, Modifier et Aide
  • Formulaire simple pour les préférences en mode Édition
  • Peut être déployé en tant qu’application servlet ou portlet
  • Peut utiliser le plug-in Maven Jetty pour se déployer avec le profil intégré à Pluto (utilisez ‘mvn jetty:run -Ppluto-embedded’, puis accédez http://localhost:8080//pluto/index.jsp)

pour le télécharger via maven, exécutez la commande

mvn archetype:generate -B -DgroupId=com.mycompany.mysystem \
                            -DartifactId=myWebApp \
                            -DarchetypeGroupId=org.apache.struts \
                            -DarchetypeArtifactId=struts2-archetype-portlet \
                            -DarchetypeVersion=<CURRENT_STRUTS_VERSION> \
                            -DremoteRepositories=http://struts.apache.org

** L’archétype de la base de données de portlets (struts2-archetype-dbportlet) **

Il a les caractéristiques suivantes :

  • Utilise Spring et Hsql pour afficher une vraie requête de base de données
  • Mise en cache intégrée des résultats de la requête
  • Exemples de modes Afficher, Modifier et Aide
  • Formulaire simple pour les préférences en mode Édition
  • Peut être déployé en tant qu’application servlet ou portlet
  • Peut utiliser le plug-in Maven Jetty pour se déployer en tant qu’application Web de servlet

pour le télécharger via maven, exécutez la commande

mvn archetype:generate -B -DgroupId=com.mycompany.mysystem \
                            -DartifactId=myWebApp \
                            -DarchetypeGroupId=org.apache.struts \
                            -DarchetypeArtifactId=struts2-archetype-dbportlet \
                            -DarchetypeVersion=<CURRENT_STRUTS_VERSION> \
                            -DremoteRepositories=http://struts.apache.org

** L’archétype du plugin (struts2-archetype-plugin) **

Il a les caractéristiques suivantes :

  • Exemple de nouveau type de résultat
  • Exemple de configuration basée sur XML

pour le télécharger via maven, exécutez la commande

mvn archetype:generate -B -DgroupId=com.mycompany.mysystem \
                            -DartifactId=myPlugin \
                            -DarchetypeGroupId=org.apache.struts \
                            -DarchetypeArtifactId=struts2-archetype-plugin \
                            -DarchetypeVersion=<CURRENT_STRUTS_VERSION> \
                            -DremoteRepositories=http://struts.apache.org
