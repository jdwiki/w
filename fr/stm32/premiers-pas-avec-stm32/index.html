Première configuration avec un exemple de LED clignotante utilisant SW4STM32 et la bibliothèque HAL #

(Remarque : Il existe de nombreux IDE, chaînes d’outils et bibliothèques qui sont prêts à l’emploi avec STM32. La configuration suivante nécessite un effort minimal pour le faire fonctionner, mais ce n’est qu’une des nombreuses. N’hésitez pas à en explorer d’autres , le but de cet exemple n’est pas de forcer qui que ce soit à utiliser les outils qui seront utilisés ici.)

Installation de l’IDE #

[System Workbench for STM32][1] : IDE gratuit sur Windows, Linux et OS X. Il a été construit par [AC6][2] et disponible en téléchargement après inscription sur le [site Web de la communauté OpenSTM32][3].

L’IDE lui-même est basé sur Eclipse, mais est livré avec quelques extras pour le développement STM32 comme :

  • Chaîne d’outils Ac6 STM32 MCU GCC
  • [OpenOCD][4] et GDB (arm-none-eabi-gdb) avec des configurations de débogage générées automatiquement en fonction de la carte cible
  • Options intégrées pour programmer ou effacer la puce

Pour commencer avec STM32 avant de créer votre propre carte, il est recommandé d’expérimenter avec une [Discovery][5], une [Nucleo][5] ou une [Eval board][5], qui sont livrées avec un SWD intégré ( Serial Wire Debug) programmeur/débogueur appelé ST-Link.

Création d’un projet #

Cet exemple utilisera un [kit de découverte STM32F4][6], qui comprend un microcontrôleur STM32F407VG. (Tout autre panneau peut également être utilisé.)

  1. Ouvrez SW4STM32 et créez un nouveau projet C : Fichier → Nouveau → Projet C

  2. Donnez-lui un nom comme “STM32F4_Discovery-Blinky” et dans la liste Type de projet, choisissez le projet Executable/Ac6 STM32 MCU. Par défaut, la seule chaîne d’outils disponible est Ac6 STM32 MCU GCC. Cliquez sur Suivant.

    [![enter image description here][7]][7]

  3. L’étape suivante est Debug/Release settings, peut être ignorée maintenant en cliquant sur Next.

  4. Sélection du tableau. Les cartes existantes peuvent être sélectionnées comme dans cet exemple la découverte STM32F4 ou de nouvelles cartes personnalisées peuvent être ajoutées.

    [![enter image description here][8]][8]

  5. L’étape suivante est la configuration du micrologiciel du projet. Choisissez entre Aucun micrologiciel, Bibliothèque de périphériques standard (SPL) ou Couche d’abstraction matérielle (HAL). On se demande lequel est le plus adapté au développement, mais cette question est hors de portée dans cet exemple. Cet exemple utilisera la bibliothèque HAL car elle est actuellement prise en charge par ST Microelectronics. L’outil logiciel supplémentaire disponible pour HAL est [STM32CubeMX][9], qui est un générateur de code d’initialisation. Plusieurs exemples d’applications sont également disponibles avec les progiciels [STM32CubeFx][10] ou STM32CubeLx. Téléchargez le firmware cible s’il est manquant et il est recommandé de sélectionner les options “Ajouter des pilotes de bas niveau dans le projet” et “Comme sources dans l’application”. Enfin, cliquez sur Terminer.

    [![enter image description here][11]][11]

Application LED clignotante #

Comme ce projet a été créé avec un STM32F4 Discovery, il existe déjà plusieurs fonctions prêtes à l’emploi dans le dossier de projet /STM32F4_Discovery-Blinky/Utilities/STM32F4-Discovery/ qui peuvent être utilisées pour interfacer les périphériques du kit Discovery (accéléromètre, audio, LEDs, bouton poussoir). Dans cet exemple, les fonctions void BSP_LED_Init(Led_TypeDef Led) et void BSP_LED_Toggle(Led_TypeDef Led) seront utilisées à partir du fichier stm32f4_discovery.c pour faire clignoter la LED verte, qui est LED4. Pour décider de quelle LED il s’agit, utilisez les schémas du [Kit de découverte][12].

[![entrez la description de l’image ici][13]][13]

Les noms de broches et de ports réels sont déjà cachés par certains #define et enum, utilisez Ctrl + Click pour les suivre.

  1. Dans le main, appelez la fonction HAL_Init() qui réinitialise tous les périphériques, initialise l’interface Flash et le Systick. (Systick sera utilisé pour générer un délai pour le clignotement.)
  2. L’horloge système doit être configurée. Cela peut être fait en utilisant la [fonction de configuration d’horloge STM32CubeMX] [14] ou par le manuel de référence. Dans cet exemple, l’horloge système est alimentée par la PLL (Phase Locked Loop) interne, qui est alimentée par un oscillateur à cristal externe de 8 MHz (HSE). Les prescalers ont été réglés pour atteindre la fréquence maximale disponible, qui est de 168 MHz dans le cas du F4 Discovery.
  3. Initialisation des périphériques, dans ce cas une broche GPIO.
  4. À l’intérieur d’une boucle sans fin, appelez le basculement de la LED et la fonction HAL_Delay(). HAL_Delay() utilise le Systick et génère un délai en millisecondes.

L’ensemble du code est le suivant :

#include "stm32f4xx.h"
#include "stm32f4_discovery.h"
            
void SystemClock_Config(void);

int main(void)
{
    /* Reset of all peripherals, Initializes the Flash interface and the Systick. */
    HAL_Init();

    /* Configure the system clock */
    SystemClock_Config();

    /* Initialize one of the LED GPIO pin */
    BSP_LED_Init(LED4);

    while(1)
    {
        BSP_LED_Toggle(LED4);
        HAL_Delay(1000);       // in miliseconds
    }
}

/**
 * @brief  System Clock Configuration
 *         The system Clock is configured as follow :
 *            System Clock source            = PLL (HSE)
 *            SYSCLK(Hz)                     = 168000000
 *            HCLK(Hz)                       = 168000000
 *            AHB Prescaler                  = 1
 *            APB1 Prescaler                 = 4
 *            APB2 Prescaler                 = 2
 *            HSE Frequency(Hz)              = HSE_VALUE
 *            PLL_M                          = (HSE_VALUE/1000000u)
 *            PLL_N                          = 336
 *            PLL_P                          = 2
 *            PLL_Q                          = 7
 *            VDD(V)                         = 3.3
 *            Main regulator output voltage  = Scale1 mode
 *            Flash Latency(WS)              = 5
 * @param  None
 * @retval None
 */
void SystemClock_Config(void)
{
  RCC_ClkInitTypeDef RCC_ClkInitStruct;
  RCC_OscInitTypeDef RCC_OscInitStruct;

  // Enable Power Control clock
  __PWR_CLK_ENABLE();

  // The voltage scaling allows optimizing the power consumption when the
  // device is clocked below the maximum system frequency, to update the
  // voltage scaling value regarding system frequency refer to product
  // datasheet.
  __HAL_PWR_VOLTAGESCALING_CONFIG(PWR_REGULATOR_VOLTAGE_SCALE1);

  // Enable HSE Oscillator and activate PLL with HSE as source
  RCC_OscInitStruct.OscillatorType = RCC_OSCILLATORTYPE_HSE;
  RCC_OscInitStruct.HSEState = RCC_HSE_ON;
  RCC_OscInitStruct.PLL.PLLState = RCC_PLL_ON;
  RCC_OscInitStruct.PLL.PLLSource = RCC_PLLSOURCE_HSE;

  // This assumes the HSE_VALUE is a multiple of 1MHz. If this is not
  // your case, you have to recompute these PLL constants.
  RCC_OscInitStruct.PLL.PLLM = (HSE_VALUE/1000000u);
  RCC_OscInitStruct.PLL.PLLN = 336;
  RCC_OscInitStruct.PLL.PLLP = RCC_PLLP_DIV2;
  RCC_OscInitStruct.PLL.PLLQ = 7;
  HAL_RCC_OscConfig(&RCC_OscInitStruct);

  // Select PLL as system clock source and configure the HCLK, PCLK1 and PCLK2
  // clocks dividers
  RCC_ClkInitStruct.ClockType = (RCC_CLOCKTYPE_SYSCLK | RCC_CLOCKTYPE_HCLK
      | RCC_CLOCKTYPE_PCLK1 | RCC_CLOCKTYPE_PCLK2);
  RCC_ClkInitStruct.SYSCLKSource = RCC_SYSCLKSOURCE_PLLCLK;
  RCC_ClkInitStruct.AHBCLKDivider = RCC_SYSCLK_DIV1;
  RCC_ClkInitStruct.APB1CLKDivider = RCC_HCLK_DIV4;
  RCC_ClkInitStruct.APB2CLKDivider = RCC_HCLK_DIV2;
  HAL_RCC_ClockConfig(&RCC_ClkInitStruct, FLASH_LATENCY_5);
}

Construisez avec le marteau [![entrez la description de l’image ici][15]][15], et téléchargez l’application en cliquant avec le bouton droit sur le dossier du projet et en sélectionnant Cible → Programmer la puce… option.

Une autre façon de télécharger consiste à utiliser debug. Pour ce faire, cliquez sur la flèche à côté de l’icône de bogue [![enter image description here][16]][16] dans la barre d’outils et ouvrez le menu Debug Configuration…. Créez une nouvelle configuration Débogage Ac6 STM32 et si le champ Application C/C++ est vide, remplissez ce qui suit :

Debug\STM32F4_Discovery-Blinky.elf

D’autres paramètres de débogage tels que le fichier de configuration OpenOCD et les ports Telnet et GDB utilisés sont automatiquement générés et renseignés par le framework. Enfin, cliquez sur le bouton Déboguer.
