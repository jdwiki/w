## Transformation d’identité à l’aide de la “règle d’identité”

Cet exemple montre la base de presque toutes les transformations XSLT et le modèle de conception XSLT le plus fondamental. Produire en sortie un document XML identique au document XML source.

Document XML source :

<t>Hello, World!</t>

Transformation XSLT :

<xsl:stylesheet version="1.0"  xmlns:xsl="http://www.w3.org/1999/XSL/Transform">
 <xsl:output omit-xml-declaration="yes" indent="yes"/>
 
  <xsl:template match="@*|node()">
    <xsl:copy>
      <xsl:apply-templates select="@*|node()"/>
    </xsl:copy>
  </xsl:template>
</xsl:stylesheet>

Résultat : lors de l’application de cette transformation sur tout document XML source, la sortie est un document XML qui est identique au document XML source. Dans ce cas:

<t>Hello, World!</t>

À noter :

  1. Utiliser et remplacer la règle d’identité est le modèle de conception XSLT le plus fondamental. Cela conduit à des solutions simples, courtes et élégantes aux tâches fondamentales, telles que la suppression/insertion/renommage d’éléments, et bien plus encore.

  2. La règle/modèle d’identité est celle publiée dans la [W3C XSLT 1.0 Specification][1]

[1] : http://www.w3.org/TR/xslt#copying

Installation ou configuration #

Un processeur XSLT est nécessaire pour effectuer toute transformation XSLT. Il peut généralement être installé via le gestionnaire de paquets du système. Par exemple. dans Debian, il peut être installé avec :

sudo apt-get install xsltproc

Transformation minimale “Hellow World” #

Il s’agit d’une transformation XSLT minimale possible. Il produit la valeur de chaîne du document XML source. Le format de sortie est texte.

Document XML source :

<t>Hello, World!</t>

Transformation XSLT :

<xsl:stylesheet version="1.0" xmlns:xsl="http://www.w3.org/1999/XSL/Transform">
  <xsl:output method="text"/>
</xsl:stylesheet>

Résultat de l’application de la transformation sur le document XML source spécifié ci-dessus :

Hello, World!

À noter :

  1. Aucune déclaration <xsl:template> n’est utilisée.

  2. Le type de sortie souhaité est spécifié dans la déclaration <xsl:output>, en tant que valeur de son attribut method.

  3. Lorsqu’il n’y a pas de modèles correspondants, le processeur XSLT, en suivant les règles du modèle de traitement XSLT, applique les modèles intégrés XSLT standard, ce qui entraîne la copie dans la sortie de la concaténation de tous les nœuds de texte dans l’ordre du document. Dans ce cas simple, le document XML source n’a qu’un seul nœud de texte avec pour valeur de chaîne la chaîne “Hello, World!”.
