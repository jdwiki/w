Création d’un routeur #

Le serveur Web sert l’utilisateur en fonction de la demande envoyée par le navigateur, mais comment l’utilisateur dira au navigateur ce qu’il recherche, c’est là que nous avons besoin d’une URL. Chaque page Web sur Internet possède une URL qui peut être mise en signet, copiée, partagée et enregistrée pour référence future. Dans l’application Backbone à une seule page, tout ce que nous voyons est une vue, les vues sont formées et rendues pour afficher une page spécifique, mais que se passe-t-il si l’utilisateur veut revoir les mêmes informations. Pour ce faire, nous devons implémenter une application basée sur le routeur Backbone qui affichera les vues en fonction du nom et des paramètres de la route.

Un exemple très simple de routeur Backbone :

var UserList = Backbone.Router.extend({
   routes: {//List of URL routes with the corresponding function name which will get called when user will visit a page having URL containing this route
       "list":                "list",    // localhost:8080/#list
       "search/:name":        "search",  // localhost:8080/#search/saurav
       "search/:name/p:page": "search",  // localhost:8080/#search/kiwis/p7
       "profile/:userId":     "profile" // localhost:8080/#profile/92
   },
   list: function() {
       var userCollection = new UserCollection();
       var userCollectionView = new UserCollectionView();
       userCollection.fetch({remove : true, data:{}, success: function(){
           for(var i = 0; i < userCollection.length; i++ ){
               var userModel = userCollection.at(i);
               var userView = new UserView({ model: userModel });
               userView.render();
               userCollectionView.$el.append(userView.$el);
           }
        }});
   },
   search: function(name, page) {
       var userCollection = new UserCollection();
       var userCollectionView = new UserCollectionView();
       userCollection.fetch({remove : true, data:{pageNo: page, name: name}, success: function(){
           for(var i = 0; i < userCollection.length; i++ ){
               var userModel = userCollection.at(i);
               var userView = new UserView({ model: userModel });
               userView.render();
               userCollectionView.$el.append(userView.$el);
           }
        }});
   },
   profile: function(userId){
       var userModel = new UserModel({id: userId});
       userModel.fetch({success: function(){
             var userView = new UserView({model: userModel});
             userView.render();
       }});
   }
});
var userList = new UserList();
Backbone.history.start();

Le code ci-dessus n’est qu’un exemple de code qui montre comment vous pouvez créer un routeur Backbone et obtenir des paramètres à partir de l’URL pour afficher les vues correspondantes.

Explication du fonctionnement et du comportement du routeur ci-dessus :

Cas par URL :

  • localhost:8080/#search/saurav - la route (fonction) “search” du routeur sera déclenchée avec les paramètres name = “saurav” et page = null, maintenant la fonction userCollection.fetch() récupérera tous les utilisateurs ayant name = “saurav” du backend et il affichera les détails de chaque utilisateur un par un.

  • localhost:8080/#search/saurav/p6 - la route (fonction) “search” du routeur sera déclenchée avec les paramètres name = “saurav” et page = 6, maintenant la fonction userCollection.fetch() récupérera tous les les utilisateurs ayant le nom = “saurav” de la page 6 du backend et il affichera les détails de chaque utilisateur un par un.

  • localhost:8080/#list - la route “liste” du routeur (fonction) sera déclenchée, maintenant la fonction userCollection.fetch() récupérera tous les utilisateurs du backend et affichera les détails de chaque utilisateur un par un.

  • localhost:8080/#profile/92 - la route “profil” du routeur (fonction) sera déclenchée, nous allons créer une nouvelle instance de userModel avec id = userId c’est-à-dire 92 et nous allons récupérer les détails de l’utilisateur depuis le backend et rendre le userView avec ces données.

Un exemple facile à expérimenter :

Visitez http://backbonejs.org dans le navigateur chrome, ouvrez la console des outils de développement et collez le code ci-dessous-

var Workspace = Backbone.Router.extend({

    routes: {
       "help":                 "help",    // #help
       "search/:query":        "search",  // #search/kiwis
       "search/:query/p:page": "search"   // #search/kiwis/p7
    },

    help: function() {
       console.log("help");
    },

    search: function(query, page) {
       console.log("searched " + query + " " + page);
    }
});
var work = new Workspace();
Backbone.history.start();

Maintenant, remplacez l’URL dans le navigateur par “http://backbonejs.org/#search/kiwis/p9” et appuyez sur la touche Entrée. Cela déclenchera la route “search” (fonction) avec les paramètres query = “kiwis” et page = 9 et vous verrez une sortie dans la console du navigateur, c’est-à-dire “searched kiwis 9”.

Modification de l’itinéraire via le code :

  • Cas 1 : exécutez le code work.navigate("search/kiwis/p7", {trigger: true}); dans la console et il affichera la sortie “searched kiwis 7” mais si vous essayez de exécutez le même code avec le même paramètre alors rien ne se passera, voir le cas suivant.

  • Cas 2 : exécutez le code work.navigate("search/kiwis/p7", {trigger: false}); dans la console, il n’imprimera rien car la route ne sera pas déclenchée.

  • Cas 3 : Si vous avez besoin de recharger à nouveau la route actuelle, vous devrez exécuter ce code Backbone.history.loadUrl("search/kiwis/p7");.

  • Cas 4 : L’exécution du code work.navigate("search/kiwis/p15"); modifiera simplement l’URL mais ne déclenchera pas la route (fonction) correspondante.

  • Cas 5 : L’exécution du code work.navigate("search/kiwis/p11", {trigger: true}); modifiera l’URL et déclenchera la route.

  • Cas 6 : L’exécution du code work.navigate("search/kiwis/p17", {trigger: true, replace: true}) remplacera l’itinéraire existant par cet itinéraire. Par conséquent, cliquer sur le bouton de retour du navigateur vous amènera 2 voies de retour vers “search/kiwis/p15”.
