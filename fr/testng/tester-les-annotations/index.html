@Tester les annotations

Sur cette page

  • Syntaxe
  • Paramètres
  • Exemple rapide sur l’annotation @Test

Syntaxe #

  • @Test
  • @Test(attribut1 = valeurattribute, attribut2 = valeurattribute, etc.)

Paramètres #

Paramètre Détails
alwaysRun Si défini sur true, cette méthode de test sera toujours exécutée même si elle dépend d’une méthode qui a échoué.
dataProvider Le nom du fournisseur de données pour cette méthode de test.
dataProviderClass La classe dans laquelle rechercher le fournisseur de données. S’il n’est pas spécifié, le fournisseur de données sera recherché sur la classe de la méthode de test actuelle ou sur l’une de ses classes de base. Si cet attribut est spécifié, la méthode du fournisseur de données doit être statique sur la classe spécifiée.
dependsOnGroups La liste des groupes dont cette méthode dépend.
dependsOnMethods La liste des méthodes dont cette méthode dépend.
description La description de cette méthode.
enabled Si les méthodes de cette classe/méthode sont activées.
expectedExceptions La liste des exceptions qu’une méthode de test est censée lever. Si aucune exception ou une autre de cette liste n’est levée, ce test sera marqué comme un échec.
groups La liste des groupes auxquels appartient cette classe/méthode.
invocationCount Le nombre de fois que cette méthode doit être invoquée.
invocationTimeOut Le nombre maximum de millisecondes que ce test doit prendre pour le temps cumulé de tous les invocationcounts. Cet attribut sera ignoré si invocationCount n’est pas spécifié.
priority La priorité de cette méthode de test. Les priorités les plus basses seront programmées en premier.
successPercentage Le pourcentage de succès attendu de cette méthode
singleThreaded Si défini sur true, toutes les méthodes de cette classe de test sont garanties de s’exécuter dans le même thread, même si les tests sont actuellement exécutés avec parallel="methods". Cet attribut ne peut être utilisé qu’au niveau de la classe et sera ignoré s’il est utilisé au niveau de la méthode. Remarque : cet attribut était appelé séquentiel (désormais obsolète).
timeOut Le nombre maximum de millisecondes que ce test doit prendre.
threadPoolSize La taille du pool de threads pour cette méthode. La méthode sera appelée à partir de plusieurs threads comme spécifié par invocationCount. Remarque : cet attribut est ignoré si invocationCount n’est pas spécifié

Exemple rapide sur l’annotation @Test #

L’annotation @Test peut être appliquée à n’importe quelle classe ou méthode. Cette annotation marque une classe ou une méthode comme faisant partie du test.

  1. @Test au niveau de la méthode - marque la méthode annotée comme méthode de test
  2. @Test au niveau de la classe
    • The effect of a class level @Test annotation is to make all the public methods of the class to become test methods even if they are not annotated.
    • @Test annotation can also be repeated on a method if you want to add certain attributes.

Exemple de @Test au niveau de la méthode :

import org.testng.annotations.Test;

public class TestClass1 {
    public void notTestMethod() {
    }

    @Test
    public void testMethod() {
    }
}

Exemple de @Test au niveau de la classe :

import org.testng.annotations.Test;

@Test
public class TestClass2 {
    public void testMethod1() {
    }

    @Test
    public void testMethod2() {
    }
}
