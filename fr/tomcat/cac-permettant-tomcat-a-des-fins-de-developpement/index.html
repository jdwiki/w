Création des Keystores et configuration de Tomcat #

Cet article décrit les étapes de configuration de Tomcat pour demander des certificats CAC au client. Il est axé sur la mise en place d’un environnement de développement, de sorte que certaines fonctionnalités qui devraient être prises en compte pour la production ne sont pas ici. (Par exemple, il montre l’utilisation d’un certificat auto-signé pour https et il n’envisage pas de vérifier les certificats révoqués.)

Créer un magasin de clés pour activer les connexions HTTPS

La première étape consiste à configurer SSL sur Tomcat. Ceci est documenté sur le site Web de tomcat ici : https://tomcat.apache.org/tomcat-8.5-doc/ssl-howto.html pour être complet, les étapes pour le configurer avec un certificat auto-signé sont répertoriées ci-dessous :

Nous devons créer un fichier keystore contenant le certificat SSL pour le serveur. Le certificat est ce qui est requis pour créer une connexion https et n’a rien à voir avec le fait que le serveur demande des certificats CAC au client, mais des connexions https sont requises pour l’authentification du certificat client. Pour un environnement de développement, la création d’un certificat auto-signé est acceptable, mais elle est déconseillée pour la production. Java est fourni avec un utilitaire appelé keytool (http://docs.oracle.com/javase/8/docs/technotes/tools/windows/keytool.html) qui est utilisé pour gérer les certificats et les magasins de clés. Il peut être utilisé pour créer un certificat auto-signé et l’ajouter à un magasin de clés. Pour ce faire, vous pouvez lancer la commande suivante à partir d’une invite de commande :

keytool -genkey -alias tomcat -keyalg RSA -keystore \path\to\my\keystore -storepass change-le

Vous serez invité à fournir diverses informations, puis un fichier de magasin de clés nommé “\path\to\my\keystore” avec un mot de passe “changeit” sera créé et contiendra le certificat auto-signé généré.

Créer un truststore contenant les certificats racine du DoD

La prochaine chose qui est nécessaire est de créer un truststore qui contiendra les certificats racine du DoD. Les certificats de ce truststore seront considérés comme approuvés par tomcat et n’accepteront que les certificats clients qui ont l’un des certificats de confiance dans leur chaîne de certificats.

Pour créer le truststore, nous devons obtenir une copie des certificats racine du DoD. Pour ce faire, téléchargez « InstallRoot 5.0 » depuis http://militarycac.com/dodcerts.htm. Installez-le et exécutez-le, puis exécutez-le. Développez le volet Installer les certificats DoD et cliquez sur l’onglet Certificat : [![Écran par défaut d’InstallRoot 5.0][1]][1]

Sélectionnez ensuite les trois certificats DoD Root CA dans la liste des certificats et cliquez sur “PEM” sous le groupe d’outils d’exportation : [![Exporter les certificats racine du DoD][2]][2]

Après avoir cliqué sur le bouton d’exportation “PEM”, choisissez un emplacement vers lequel exporter les certificats et cliquez sur OK. Cela devrait avoir créé trois fichiers .cer dans le répertoire que vous avez sélectionné. Ouvrez une invite de commande et accédez à ce répertoire.

Ici, nous allons utiliser la commande keytool pour importer les certificats dans un truststore. Exécutez les commandes suivantes pour importer les trois certificats :

keytool -importcert -file DoD_Root_CA_2__0x05__DoD_Root_CA_2.cer -alias DODRoot2 -keystore truststore.jks -storepass change-le

keytool -importcert -file DoD_Root_CA_3__0x01__DoD_Root_CA_3.cer -alias DODRoot3 -keystore truststore.jks -storepass change-le

keytool -importcert -file DoD_Root_CA_4__0x01__DoD_Root_CA_4.cer -alias DODRoot4 -keystore truststore.jks -storepass change-le

Cela créera un fichier truststore.jks avec un mot de passe “changeit” dans le répertoire de travail actuel. Il contiendra les trois certificats racine du DoD, vous pouvez le voir en exécutant :

keytool -list -keystore truststore.jks

Qui devrait lister quelque chose comme :

Votre keystore contient 3 entrées

dodroot4, 23 septembre 2016, TrustedCertEntry, empreinte digitale du certificat (SHA1): B8:26:9F:25:DB:D9:37:EC:AF:D4:C3:5A:98:38:57:17:23:F2:D0:26 dodroot3, 23 septembre 2016, TrustedCertEntry, empreinte digitale du certificat (SHA1): D7:3C:A9:11:02:A2:20:4A:36:45:9E:D3:22:13:B4:67:D7:CE:97:FB dodroot2, 23 septembre 2016, TrustedCertEntry, empreinte digitale du certificat (SHA1): 8C:94:1B:34:EA:1E:A6:ED:9A:E2:BC:54:CF:68:72:52:B4:C9:B5:61

Configurez Tomcat pour utiliser le Keystore et le Truststore

Nous avons maintenant les fichiers keystore et truststore dont nous avons besoin, ensuite il faut configurer tomcat pour les utiliser. Pour ce faire, nous devons modifier le fichier /conf/server.xml. Ouvrez le fichier et ajoutez une définition de connecteur comme suit :

<Connector 
    clientAuth="true" 
    keystoreFile="path/to/keystore.jks" 
    keystorepass="changeit" 
    keystoreType="jks" 
    truststoreFile="path/to/truststore.jks" 
    truststoreType="jks" 
    truststorepass="changeit"
    maxThreads="150" 
    port="8443" 
    protocol="org.apache.coyote.http11.Http11NioProtocol" 
    scheme="https" 
    secure="true" 
    sslProtocol="TLS" 
    SSLEnabled="true" 
    />

Vous pouvez aller ici pour une définition plus approfondie de tous les attributs : http://tomcat.apache.org/tomcat-7.0-doc/config/http.html

Une fois que tout est fait, lancez Tomcat. À partir d’un ordinateur équipé d’un lecteur CAC avec un CAC inséré, accédez à l’url https://:8443/ et si tout est configuré correctement, vous devriez être invité à choisir un certificat sur la carte CAC.
