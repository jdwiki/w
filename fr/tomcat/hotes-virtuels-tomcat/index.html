Host Manager est une application Web à l’intérieur de Tomcat qui crée/supprime des hôtes virtuels dans Tomcat.

Un hôte virtuel vous permet de définir plusieurs noms d’hôte sur un seul serveur, vous pouvez donc utiliser le même serveur pour gérer les requêtes vers, par exemple, ren.myserver.com et stimpy.myserver.com.

Malheureusement, la documentation sur le côté graphique du gestionnaire d’hôtes ne semble pas exister, mais la documentation sur la configuration manuelle des hôtes virtuels dans context.xml est ici :

http://tomcat.apache.org/tomcat-7.0-doc/virtual-hosting-howto.html.

L’explication complète des paramètres Host que vous pouvez trouver ici :

http://tomcat.apache.org/tomcat-7.0-doc/config/host.html.

Application Web du gestionnaire d’hôtes Tomcat #

L’application Host Manager de Tomcat se trouve par défaut à l’adresse http://localhost:8080/host-manager, mais n’est pas accessible tant qu’un utilisateur n’en a pas reçu l’autorisation dans le fichier conf/tomcat-users.xml. Le fichier doit avoir :

  1. Un rôle de manager-gui
  2. Un utilisateur avec ce rôle

Par exemple:

<tomcat-users>
    ...
    <role rolename="manager-gui"/>
    ....
    <user username="host-admin" password="secretPassword" roles="manager-gui"/>
</tomcat-users>

Ajout d’un hôte virtuel via l’application Web Tomcat Host Manager #

Une fois que vous avez accès au gestionnaire d’hôte, l’interface graphique vous permettra d’ajouter un hôte virtuel.

Remarque : Dans Tomcat 7 et 8, l’ajout d’un hôte virtuel via l’interface graphique n’écrit pas le vhost dans les fichiers de configuration. Vous devrez modifier manuellement le fichier server.xml pour avoir le vhost disponible après un redémarrage. Voir http://tomcat.apache.org/tomcat-7.0-doc/virtual-hosting-howto.html pour plus d’informations sur la balise <Host> dans server.xml

[![Tomcat Host Manager - Ajouter un hôte virtuel][1]][1]

Au minimum, vous devez définir les champs Name et App Base. Tomcat créera alors les répertoires suivants :

{CATALINA_HOME}\conf\Catalina\{Name}
{CATALINA_HOME}\{App Base}
  • App Base sera l’endroit où les applications Web seront déployées sur l’hôte virtuel. Peut être relatif ou absolu.
  • Name est généralement le nom de domaine complet (par exemple, ren.myserver.com)
  • Alias peut être utilisé pour étendre le Name également lorsque deux adresses doivent être résolues vers le même hôte (par exemple, www.ren.myserver.com). Notez que cela doit être reflété dans les enregistrements DNS.

Les cases à cocher sont les suivantes :

  • Auto Deploy : redéploie automatiquement les applications placées dans App Base. Dangereux pour les environnements de production !
  • “Déployer au démarrage”: démarre automatiquement les applications sous App Base au démarrage de Tomcat
  • Deploy XML : détermine s’il faut analyser le fichier /META-INF/context.xml de l’application
  • Unpack WARs : décompressez les fichiers WAR placés ou téléchargés sur l’App Base, au lieu de les exécuter directement à partir du WAR.
  • Tomcat 8 Copy XML : copiez le fichier META-INF/context.xml d’une application dans l’App Base/XML Base lors du déploiement et utilisez-le exclusivement, que l’application soit mise à jour ou non. Non pertinent si Deploy XML est faux.
  • Manager App : ajoutez l’application de gestion à l’hôte virtuel (utile pour contrôler les applications que vous pourriez avoir sous ren.myserver.com)

Ajout d’un hôte virtuel à server.xml #

Une fois qu’un hôte virtuel a été ajouté via l’application Web, des répertoires existeront à :

{CATALINA_HOME}\conf\Catalina\{Name}
{CATALINA_HOME}\{App Base}

Pour conserver l’hôte virtuel après un redémarrage, le fichier server.xml doit être mis à jour avec la configuration. Un élément Host doit être ajouté à l’intérieur de l’élément Engine, semblable à ceci :

<Engine name="Catalina" ...>
  ...
  <Host name="my-virtual-app" appBase="virtualApp" autoDeploy="true" unpackWARs="true" ... />
</Engine>

Les attributs de l’élément Host doivent refléter les sélections effectuées dans l’interface graphique du gestionnaire d’hôtes (voir la [documentation de l’hôte] [1] pour plus de détails), mais peuvent être modifiés. Notez que l’option Manager App dans l’interface graphique ne correspond à aucun attribut Host.
