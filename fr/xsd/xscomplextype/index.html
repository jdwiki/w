Un xs:complexType fournit une description du contenu d’un élément XML dans le document d’instance. La définition du xs:complexType peut être faite globalement, auquel cas il a un nom et peut être réutilisé dans le schéma, ou il peut être en place et utilisé uniquement dans le contexte dans lequel il est déclaré.

Paramètres #

Attributs Descriptif
abstrait Lorsqu’il est défini sur true, le type complexe ne peut pas être utilisé directement dans un document XML d’instance via xsi:type. Il peut cependant être utilisé comme type de base pour une définition d’élément. (false par défaut) - Valable uniquement pour le niveau racine xs:complexType’s
bloc Limite les types qui peuvent être utilisés dans un document d’instance XML (par défaut à la valeur de l’attribut xs:schemas blockDefault s’il est défini, sinon par défaut à vide, les valeurs ‘#all’ | une liste de (’extension’, ’list’ , ‘union’) séparés par des espaces).
finale Limite la dérivation de types à partir de l’utilisation de ce type de certaines manières dans le schéma (par défaut à la valeur de l’attribut xs:schemas finalDefault s’il est défini, sinon par défaut à vide, les valeurs ‘#all’ | ou une liste de (’extension’, ’ list’, ‘union’) séparés par des espaces) - Valable uniquement pour le niveau racine xs:complexType’s
identifiant L’identifiant de l’élément de schéma (optionnel)
mixte Indique que l’élément XML de l’instance peut contenir un contenu mixte (valeur par défaut à false)
nom Le nom du xs:complexType - Valable uniquement pour le niveau racine xs:complexType’s
n’importe lequel Tous les autres attributs ne figurant pas dans l’espace de noms ‘http://www.w3.org/2001/XMLSchema’ sont autorisés.

Dérivant d’un xs:complexType

Lorsqu’un xs:complexType dérive d’un autre xs:complexType, il peut le faire via extension ou restriction.

  • extension - le type dérivant prend tout ce qui est défini dans la base tapez et y ajoute.
  • restriction - le type dérivant ne prend que pièces sélectionnées à partir du type de base, n’autorisant que les pièces souhaitées, aucun élément supplémentaire ne peut être ajouté.

Dérivé d’un xs:simpleType

Lorsqu’un xs:complexType dérive d’un xs:simpleType, il peut le faire via extension, auquel cas il peut ajouter des attributs au type résultant, mais pas des éléments.

Type de contenu

Conceptuellement, un xs:complexType contient du contenu simple ou complexe. Si le xs:complexType dérive d’un typé basé sur xs:anySimpleType (xs:int, xs:string etc) alors c’est simple. S’il dérive d’un xs:complexType qui contient un contenu complexe, alors il est lui-même complexe (si le xs:complexType ne dérive pas d’un type, alors il est également complexe).

Global ComplexType avec séquence et attributs #

Cet exemple montre une définition globale simple d’un complexType. La définition est considérée comme globale car elle est un enfant de xs:schema. Les types définis globalement peuvent être utilisés ailleurs dans le schéma.

Il s’agit de la forme la plus courante pour déclarer un xs:complexType global, il définit les éléments enfants à l’aide d’un xs:sequence, xs:choice ou xs:all, et possède également des attributs en option.

Remarque : étant donné qu’il s’agit d’un ensemble défini globalement, il doit avoir un nom unique dans l’ensemble de schémas.

<?xml version="1.0" encoding="utf-8" ?>
<!--Created with Liquid Studio 2017 (https://www.liquid-technologies.com)-->
<xs:schema elementFormDefault="qualified" 
           xmlns:xs="http://www.w3.org/2001/XMLSchema">
    <xs:complexType name="PersonType">
        <xs:sequence>
            <xs:element name="Forename" type="xs:string" />
            <xs:element name="Surname" type="xs:string" />
        </xs:sequence>
        <xs:attribute name="Gender">
            <xs:simpleType>
                <xs:restriction base="xs:string">
                    <xs:enumeration value="male" />
                    <xs:enumeration value="female" />
                </xs:restriction>
            </xs:simpleType>
        </xs:attribute>
    </xs:complexType>
</xs:schema>

Création d’un xs:complexType global en étendant un xs:complexType existant #

Dans cet exemple, nous créons un nouveau xs:complexType (EmployeeType) basé sur un xs:complexType (PersonType) existant.

La construction de celui-ci est un peu plus compliquée. Étant donné que la base xs:complexType (PersonType) est considérée comme complexe (plus d’informations à ce sujet ci-dessous), nous ajoutons l’élément xs:complexContent. Ensuite, parce que nous étendons PersonType, nous ajoutons l’élément <xs:extension base=“PersonType”>. Dans la balise xs:extension, nous pouvons ajouter un compositeur (xs:all/xs:choice/xs:sequence) et tout attribut supplémentaire.

<?xml version="1.0" encoding="utf-8" ?>
<!--Created with Liquid Studio 2017 (https://www.liquid-technologies.com)-->
<xs:schema elementFormDefault="qualified" 
           xmlns:xs="http://www.w3.org/2001/XMLSchema">
    <xs:complexType name="PersonType">
        <xs:sequence>
            <xs:element name="Forename" type="xs:string" />
            <xs:element name="Surname" type="xs:string" />
        </xs:sequence>
        <xs:attribute name="Gender">
            <xs:simpleType>
                <xs:restriction base="xs:string">
                    <xs:enumeration value="male" />
                    <xs:enumeration value="female" />
                </xs:restriction>
            </xs:simpleType>
        </xs:attribute>
    </xs:complexType>
    <xs:complexType name="EmployeeType">
        <xs:complexContent>
            <xs:extension base="PersonType">
                <xs:sequence>
                    <xs:element name="Salary" type="xs:decimal" />
                </xs:sequence>
                <xs:attribute name="EmployeeID" type="xs:int" use="required" />
            </xs:extension>
        </xs:complexContent>
    </xs:complexType>
</xs:schema>

Création d’un xs:complexType global en restreignant un xs:complexType existant #

C’est là que les choses se compliquent un peu. Nous restreignons maintenant un xs:complexType existant. Notre SolidStateDriveType dérive de HardDiskType mais supprime l’attribut spinUpTime et l’élément RotationSpeed.

Notez que l’approche pour traiter les attributs et les éléments est différente. Pour supprimer un attribut, vous devez le re-déclarer et définir son utilisation sur interdit. Pour les éléments, ne pas les déclarer à nouveau entraînera leur exclusion. En fait, vous devez re-déclarer tous les éléments que vous souhaitez conserver dans le nouveau type.

Concept clé pour les types restreints : Il doit être possible de charger un élément d’instance XML résultant du type restreint dans le type de base, autrement dit le type restreint doit pouvoir ’s’insérer’ dans le type de base. Ainsi, vous ne pouvez pas exclure un attribut ou un élément obligatoire, afin de l’exclure dans le type restreint, il doit être facultatif dans le type de base. Si vous modifiez les règles de type/facette d’un élément ou d’un attribut dans le type restreint, les nouvelles règles de type/facette doivent être compatibles avec le type de base, donc si le type de base était court, le type restreint pourrait être un octet, mais pas longtemps.

<?xml version="1.0" encoding="utf-8" ?>
<!--Created with Liquid Studio 2017 - Developer Bundle Edition (Trial) 15.0.2.7192 (https://www.liquid-technologies.com)-->
<xs:schema elementFormDefault="qualified" xmlns:xs="http://www.w3.org/2001/XMLSchema">
    <xs:complexType name="HardDiskType">
        <xs:sequence>
            <xs:element name="Capacity" type="xs:long" />
            <xs:element name="RotationSpeed" type="xs:int" minOccurs="0" />
        </xs:sequence>
        <xs:attribute name="name" type="xs:string" />
        <xs:attribute name="spinUpTime" type="xs:time" />
    </xs:complexType>
    <xs:complexType name="SolidStateDrive">
        <xs:complexContent>
            <xs:restriction base="HardDiskType">
                <xs:sequence>
                    <xs:element name="Capacity" type="xs:long" />
                </xs:sequence>
                <xs:attribute name="spinUpTime" use="prohibited" />
            </xs:restriction>
        </xs:complexContent>
    </xs:complexType>
</xs:schema>
