Expression S #

Une expression dans Scheme est ce qui va être exécuté. Une expression S, comme on l’appelle généralement, commence par un ( et se termine par un ). Le premier membre de l’expression est ce qui va être exécuté. Le membre suivant de l’expression sont les paramètres qui seront envoyés à l’expression lors de l’évaluation de l’expression.

Par exemple, ajouter des nombres :

(+ 1 2 3)

Dans ce cas, + est un symbole pour une fonction add qui prend plusieurs paramètres. ‘1’, ‘2’ et ‘3’ sont envoyés à la fonction ‘+’.

S-Expression peut contenir des S-Expressions en tant que paramètres, comme illustré dans l’exemple suivant :

(if (< x y)
  x
  y)

Ce qui peut être lu comme si x est inférieur à y return x sinon return y. Dans cet exemple, nous évaluons l’expression de condition, en fonction de la valeur résolue, x ou y sera renvoyé. Cela pourrait être évalué à ceci

(if #t x y)
x
(if #f x y)
y

Un exemple moins évident pour les débutants est d’avoir une S-Expression dans le cadre du premier membre d’une S-Expression. De cette façon, nous pouvons changer le comportement d’une méthode en changeant la fonction qui sera appelée sans avoir à créer des branches avec les mêmes paramètres. Voici un exemple rapide d’une expression qui ajoute ou soustrait des nombres si x est inférieur à y.

((if (< x y) + -) 
  1 2 3)

Si x est inférieur à y, l’expression sera évaluée comme :

(+ 1 2 3)
6

Par ailleurs

(- 1 2 3)
-4

Comme vous pouvez le voir, Scheme permet au programmeur de créer un morceau de code complexe tout en donnant au programmeur les outils pour empêcher la duplication de code. Dans d’autres langues, nous pourrions voir le même exemple écrit comme tel :

(si (< x y) (+ 1 2 3) (- 1 2 3))

Le problème avec cette méthode est que nous dupliquons beaucoup de code alors que la seule chose qui change est la méthode appelée. Cet exemple est assez simple mais avec plus de condition nous pourrions voir beaucoup de lignes similaires dupliquées.

Macro let simple #

Les expressions let du schéma sont en fait des macros. Ils peuvent être exprimés avec des lambdas. Un let simple pourrait ressembler à ceci :

(let ((x 1) (y 2))
  (+ x y))

Il renverra 3 car la valeur de la dernière expression du corps let est renvoyée. Comme vous pouvez le voir, une expression let exécute en fait quelque chose. Si nous traduisions cette partie de code avec des lambdas, nous obtiendrions quelque chose comme ceci :

((lambda (x y) (+ x y)) 1 2)

Ici, nous pouvons voir que nous appelons directement le lambda anonyme avec 1 et 2. Donc, le résultat dans ce cas est également 3.

Dans cet esprit, nous comprenons qu’une expression let est composée de 2 parties. Il a des paramètres et un corps comme un lambda, mais la différence est que les expressions let sont appelées juste après leur évaluation.

Pour expliquer comment une expression let fonctionne d’une vue abstraite à une vue concrète, cela ressemblerait à ceci.

(let params body ...)
(let (param1 param2 ...) body ...)
(let ((p1 val1) (p2 val2) ...) body ...)

Les paramètres sont une liste de paires de (nom valeur) à utiliser dans le corps du let.

Pourquoi utiliser l’expression let ?

Les expressions Let sont particulièrement utiles pour stocker des variables dans une méthode, tout comme les initialisations de variable dans des langages comme c. Il est favorable à l’utilisation de define car hors de l’expression let, les variables ont disparu… Utiliser un define revient en fait à ajouter une variable à l’environnement d’exécution actuel. Les variables ajoutées à l’environnement global ne peuvent pas être supprimées. Let expression peut être utilisé n’importe où en toute sécurité. Il peut également être utilisé pour masquer des variables sans toucher aux portées parentes.

Par exemple:

(let ((x 1))
  (let ((x 2) (y x))
    (display x)
    (display y))
  (display x))

Il imprimera :

2
1
1

Dans ce cas, x est défini avec 1, puis fantôme par le x dans le deuxième let avec la valeur 2. La variable y est initialisée avec la valeur x de la portée parent. Une fois l’expression interne let exécutée, elle affiche la valeur initiale de x avec 1. L’expression interne let n’a pas modifié la valeur de la portée parent.

Chaque fois que vous avez besoin d’initialiser des variables, vous devez utiliser des expressions let comme celle-ci :

(let (
  (user (get-current-user session))
  (db (get-current-db session))
  (ids (get-active-ids session))
  )
  (mark-task-completed db user ids)
  (change-last-logged db user)
  (db-commit db))

Ici, dans cet exemple, les variables sont initialisées et utilisées plusieurs fois dans le bloc de code. Et lorsque l’expression let est terminée, les variables sont automatiquement libérées car elles ne sont plus nécessaires.

Syntaxe pointée pour les paires #

Il existe une syntaxe particulière qui nous permet d’écrire la cellule cons de manière plus compacte qu’en utilisant le constructeur cons.

Un couple peut s’écrire ainsi :

'(1 . 2) == (cons 1 2)

La grande différence est que nous pouvons créer des “paires” en utilisant quote. Sinon, Scheme créerait une liste appropriée (1 . (2 . '())).

La syntaxe à points force l’expression à n’avoir que 2 membres. Chaque membre peut être de n’importe quel type, y compris des paires.

'(1 . (2 . (3 . 4)))
> (1 2 3 . 4)

Notez que la liste incorrecte doit être affichée avec un point à la fin pour montrer que le cdr de la dernière paire de la liste n’est pas la liste vide '().

Cette façon d’afficher les listes est parfois déroutante car l’expression suivante ne serait pas exprimée comme on s’y attendrait.

'((1 . 2) . ( 3 . 4))
> ((1 . 2) 3 . 4)

Étant donné que la liste ignore généralement le ., le premier argument de la liste serait (1 . 2), le deuxième argument serait 3 mais comme la liste est incorrecte, le dernier . est affiché pour montrer que le dernier élément de la liste n’est pas '(). Même pensé, les données sont affichées d’une manière différente, les données internes sont telles qu’elles ont été créées.
