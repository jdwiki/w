Des astuces:

  • N’oubliez pas de toujours vérifier le fichier /usr/ports/UPDATING avant la mise à jour. Il peut y avoir des changements importants dans les programmes que vous utilisez ou dans leur configuration qui interrompront votre configuration actuelle.

Création et installation de logiciels #

Si vous avez trouvé votre logiciel dans l’arborescence des ports, il est maintenant temps de le construire.

Construction et installation simples avec configuration manuelle #

cd /usr/ports/www/apache24
make
make install

Construction et installation simples avec configuration automatique #

cd /usr/ports/www/apache24
make BATCH=yes
make install

Obtenir l’arborescence des ports #

Snap de port #

portsnap fetch
portsnap extract

mise à jour de l’arborescence des ports avec portsnap #

portsnap update

programmer la tâche cron pour les mises à jour quotidiennes #

0 3 * * * root /usr/sbin/portsnap cron

#SVN

tête #

cd /usr/ports
svnlite checkout https://svnweb.freebsd.org/ports/head .

trimestriel #

L’équipe FreeBSD Ports gèle l’arborescence des ports tous les 3 mois. Pour obtenir cette arborescence de ports, vous pouvez utiliser des branches de ports :

cd /usr/ports
svnlite checkout https://svnweb.freebsd.org/ports/branches/2016Q4 .

Tarball (http ou ftp) #

cd /usr/ports
fetch http://ftp.freebsd.org/pub/FreeBSD/releases/amd64/11.0-RELEASE/ports.txz
tar xJvf ports.txz

#Git

git clone https://github.com/freebsd/freebsd-ports

Logiciel de recherche #

recherche par mot clé #

cd /usr/ports
make search key=apache

recherche de nom #

cd /usr/ports
make search name=apache24

Utiliser des sports #

Le site officiel des ports FreeBSD (http://freshports.org/) vous donne un bon moyen de trouver les ports et toutes les informations les concernant.

Configuration des sources logicielles #

Si vous voulez une configuration personnalisée à partir des ports, vous pouvez la configurer avant de la construire make config. Toutes les configurations de ports sont stockées dans /var/db/ports/${CATEGORY_NAME}/options en tant que makefile.

Configuration de www/apache24 #

cd /usr/ports/www/apache24
make config
make
make install

Cette configuration sera enregistrée dans /var/db/ports/www_apache24/options.

Emballage #

Emballage manuel #

Vous pouvez créer votre propre package basé sur les ports.

cd /usr/ports/www/apache24
make package BATCH=yes

Cette commande stockera votre package dans /usr/ports/packages/All.

Using poudriere #

[poudriere][1] est actuellement le constructeur de paquets officiel pour FreeBSD.

Installing poudriere #

pkg install poudriere
# or
cd /usr/ports/ports-mgmt/poudriere
make
make install

Configuring poudriere #

La configuration de poudriere est stockée dans /usr/local/etc/poudriere.conf et /usr/local/etc/poudriere.d

Deploying poudriere jail #

poudriere jail -c -j myjail

Mise à jour de la prison de poudriere #

poudriere jail -u -j myjail

Deploying poudriere ports tree #

poudriere ports -c -p myports

Mise à jour de l’arborescence des ports de poudriere #

poudriere ports -u -p myports

## Construction groupée

poudriere bulk -j myjail -p myports www/apache24

[1] : https://www.freebsd.org/cgi/man.cgi?query=poudriere&apropos=0&sektion=0&manpath=FreeBSD%20Ports%2010.3-RELEASE&arch=default&format=html
