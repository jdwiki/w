Enregistrement de l’application à utiliser avec CloudKit #

Ce dont vous avez besoin, c’est d’obtenir un fichier d’autorisations pour que l’application puisse accéder à votre iCloud et écrire des enregistrements à l’aide de CloudKit.

Suivez les étapes pour accorder l’accès à iCloud depuis votre application :

1- Sélectionnez le projet dans le Navigateur de projet, puis ouvrez l’onglet Général.

2- Dans la section Identité, définissez votre identifiant Apple de développeur dans le menu déroulant Équipe. (S’il n’est pas disponible, ajoutez-le dans le menu Xcode -> Préférences -> Comptes.

3- Accédez à l’onglet Capacités dans les propriétés du projet et activez iCloud. Ensuite, sélectionnez “Stockage clé-valeur” et “CloudKit”.

[![entrez la description de l’image ici][1]][1]

4- Assurez-vous que ces éléments sont cochés :

[![entrez la description de l’image ici][2]][2]

Si tous les éléments sont cochés, votre application est prête à utiliser CloudKit.

[1] : http://i.stack.imgur.com/vQl0o.png [2] : http://i.stack.imgur.com/j1kzA.png

Utilisation du tableau de bord CloudKit #

Tous les enregistrements créés à l’aide du code lié à CloudKit peuvent être prévisualisés, modifiés et même supprimés dans CloudKit Dashboard. Pour accéder au tableau de bord CloudKit, rendez-vous [ici][1].

Il y a plusieurs parties dans le tableau de bord :

  • Types d’enregistrement (qui seront discutés plus tard)
  • Rôles de sécurité (où vous pouvez définir les bases de données comme publiques ou privées)
  • Types d’abonnement (que votre application pourrait enregistrer pour [Apple Push Notifications (APNs)][2] pour vous avertir lorsqu’un enregistrement est modifié)

Types d’enregistrement #

Ici, vous obtenez une liste de tous les types d’enregistrements existants dans l’application. Lorsque vous ouvrez pour la première fois le tableau de bord CloudKit pour une application, il existe un type d’enregistrement appelé Utilisateurs, que vous pouvez utiliser ou simplement le supprimer et utiliser le vôtre.

Dans cette page, vous pouvez faire taper vos données manuellement. Bien sûr, dans la plupart des cas, cela ne sert à rien, car le SDK iOS peut le gérer bien mieux que le tableau de bord, mais la fonctionnalité est également là si vous préférez. Cette page est principalement utilisée pour prévisualiser les types.

[1] : https://icloud.developer.apple.com/dashboard/ [2] : https://www.wikiod.com/fr/ios/notifications-push

Enregistrement des données dans CloudKit #

Pour enregistrer la date dans CloudKit, nous devons faire :

  • Un CKRecordID (la clé de votre enregistrement unique)
  • Un CKRecord (qui inclut des données)

Créer une clé d’enregistrement #

Pour garantir que chaque nouvel identifiant d’enregistrement est unique, nous utilisons l’horodatage actuel, qui est unique. Nous obtenons l’horodatage en utilisant la méthode timeIntervalSinceReferenceDate() de NSDate. Il se présente sous la forme ###.### (# sont des nombres), dont nous utiliserons la partie entière. Pour ce faire, nous divisons la chaîne :

Rapide #

let timestamp = String(format: "%f", NSDate.timeIntervalSinceReferenceDate())
let timestampParts = timestamp.componentsSeparatedByString(".")
let recordID = CKRecordID(recordName: timestampParts[0])

Faire l’enregistrement #

Pour créer l’enregistrement, nous devons spécifier le type d’enregistrement (expliqué dans Utilisation du tableau de bord CloudKit) en tant qu’utilisateurs, l’ID en tant que chose que nous venons de créer et les données. Ici, nous ajouterons un exemple de texte, une image et la date actuelle à l’enregistrement :

Rapide #

let record = CKRecord(recordType: "Users", recordID: recordID)
record.setObject("Some Text", forKey: "text")
record.setObject(CKAsset(fileURL: someValidImageURL), forKey: "image")
record.setObject(NSDate(), forKey: "date")

Objectif c #

CKRecord *record = [[CKRecord alloc] initWithRecordType: "Users" recordID: recordID];
[record setObject: "Some Text" forKey: "text"];
[record setObject: [CKAsset assetWithFileURL: someValidImageURL] forKey: "image"];
[record setObject: [[NSDate alloc] init] forKey: "date"];

Remarque #

Ici, nous n’avons pas ajouté UIImage directement à l’enregistrement, car comme mentionné dans les remarques, le format d’image n’est pas directement pris en charge dans CloudKit, nous avons donc converti UIImage en CKAsset.

Accéder au conteneur #

Rapide #

let container = CKContainer.defaultContainer()
let database = container.privateCloudDatabase // or container.publicCloudDatabase

Enregistrement des enregistrements dans la base de données CloudKit #

Rapide #

database.saveRecord(record, completionHandler: { (_, error) -> Void in
    print(error ?? "")
})
