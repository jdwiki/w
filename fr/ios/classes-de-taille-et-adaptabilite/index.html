Pour plus de détails (Classes de taille et adaptabilité via Storyboard) sur l’utilisation de la mise en page automatique pour l’adaptabilité dans iOS, nous pouvons suivre le [lien du site du développeur Apple][1].

Nous pouvons également ajouter des contraintes programmatiquement en utilisant le langage de format visuel comme décrit [ici sur le site des développeurs Apple][2].

[1] : https://developer.apple.com/library/ios/documentation/UserExperience/Conceptual/AutolayoutPG/ [2] : https://developer.apple.com/library/prerelease/content/documentation/UserExperience/Conceptual/AutolayoutPG/ProgrammaticallyCreatingConstraints.html

Classes de taille et adaptabilité via Storyboard #

Nous pouvons ajouter de l’adaptabilité à n’importe quelle sous-classe de UIView que nous ajoutons au contrôleur de vue dans le fichier nib. Prenons un exemple d’ajout d’adaptabilité à l’aide de classes de taille à une vue.

  1. Ajoutez une vue sur le contrôleur de vue en tant que :

[![entrez la description de l’image ici][1]][1]

  1. Maintenant, nous devons épingler cette vue à sa supervue pour fixer sa taille et sa position en utilisant des contraintes comme :

[![entrez la description de l’image ici][2]][2]

  1. Nous pouvons voir les contraintes ajoutées comme suit :

[![entrez la description de l’image ici][3]][3]

Ces contraintes définissent que la vue ajoutée sera placée dans sa supervue comme

CGRect(20, 0, superview.width - 20, superview.height - 20)
  1. Pour voir l’aperçu à l’écran de ces contraintes ajoutées, nous pouvons utiliser Assistant Editor as ;

[![entrez la description de l’image ici][4]][4]

  1. Nous pouvons ajouter plus d’écran pour voir un aperçu comme :

[![entrez la description de l’image ici][5]][5]

Nous pouvons également voir l’aperçu en mode paysage en déplaçant la souris sur le nom de l’appareil et en cliquant sur le bouton de rotation comme :

[![entrez la description de l’image ici][6]][6]
