Faire défiler le contenu avec la mise en page automatique activée #

Ce projet est un exemple autonome entièrement réalisé dans Interface Builder. Vous devriez pouvoir le parcourir en 10 minutes ou moins. Ensuite, vous pouvez appliquer les concepts que vous avez appris à votre propre projet.

[![gif animé montrant le défilement horizontal][6]][6]

Ici, j’utilise simplement UIViews mais ils peuvent représenter n’importe quelle vue que vous aimez (c’est-à-dire un bouton, une étiquette, etc.). J’ai également choisi le défilement horizontal car les captures d’écran du storyboard sont plus compactes pour ce format. Les principes sont les mêmes pour le défilement vertical, cependant.

Concepts clés #

  • Le UIScrollView ne doit utiliser qu’une seule sous-vue. Il s’agit d’un ‘UIView’ qui sert de vue de contenu pour contenir tout ce que vous souhaitez faire défiler.
  • Faites en sorte que la vue de contenu et le parent de la vue de défilement aient des hauteurs égales pour le défilement horizontal. (Largeurs égales pour le défilement vertical)
  • Assurez-vous que tout le contenu défilable a une largeur définie et est épinglé de tous les côtés.

Démarrer un nouveau projet #

Il peut s’agir d’une simple application à vue unique.

Scénario #

Dans cet exemple, nous allons créer une vue de défilement horizontal. Sélectionnez le contrôleur de vue, puis choisissez Forme libre dans l’inspecteur de taille. Faites la largeur ‘1 000’ et la hauteur ‘300’. Cela nous donne juste la place sur le storyboard pour ajouter du contenu qui défilera.

[![Capture d’écran des paramètres de taille simulés][1]][1]

Ajouter une vue défilante

Ajoutez un UIScrollView et épinglez les quatre côtés à la vue racine du contrôleur de vue.

[![capture d’écran de la vue de défilement épinglée][2]][2]

Ajouter une vue de contenu

Ajoutez un UIView en tant que sous-vue à la vue de défilement. C’est la clé. N’essayez pas d’ajouter beaucoup de sous-vues à la vue de défilement. Ajoutez simplement un seul UIView. Ce sera votre vue de contenu pour les autres vues que vous souhaitez faire défiler. Épinglez la vue de contenu à la vue de défilement sur les quatre côtés.

[![Capture d’écran de l’affichage du contenu épinglé][3]][3]

Hauteurs égales

Faire défiler la taille du contenu de l’affichage #

La propriété contentSize doit être définie sur la taille du contenu défilable. Cela spécifie la taille de la zone de défilement. Le défilement est visible lorsque la zone de défilement, c’est-à-dire que contentSize est plus grand que la taille du cadre UIScrollView.

Avec mise en page automatique :

Lorsque le contenu de la vue de défilement est configuré à l’aide de la mise en page automatique, il doit être explicitement dimensionné à la fois verticalement et horizontalement et avoir les 4 bords épinglés à la vue de défilement contenante. De cette façon, le contentSize est calculé automatiquement en fonction du contenu de la vue de défilement et est également mis à jour lorsque la mise en page du contenu est modifiée.

Manuellement:

Rapide

scrollview.contentSize = CGSize(width: 640, height: 800)

Objectif c

scrollview.contentSize = CGSizeMake(640, 800);

Activer/Désactiver le défilement #

La propriété scrollEnabled stocke une valeur booléenne qui détermine si le défilement est activé ou non.
Si la valeur de cette propriété est true/YES, le défilement est activé, sinon non. La valeur par défaut est true

Rapide

scrollview.isScrollEnabled = true

Objectif c

scrollview.scrollEnabled = YES;

Zoom avant/arrière UIImageView #

Créer une instance UIScrollView

let scrollview = UIScrollView.init(frame: self.view.bounds)

Et puis définissez ces propriétés :

scrollView.minimumZoomScale = 0.1
scrollView.maximumZoomScale = 4.0
scrollView.zoomScale = 1.0
scrollview.delegate = self as? UIScrollViewDelegate

Pour zoomer et dézoomer l’image, nous devons spécifier le montant que l’utilisateur peut zoomer et dézoomer. Pour ce faire, nous définissons les valeurs des propriétés minimumZoomScale et maximumZoomScale de la vue de défilement. Les deux sont définis sur 1.0 par défaut.

Et zoomScale à 1.0 qui spécifie le facteur de zoom pour le zoom minimum et maximum.

Pour prendre en charge le zoom, nous devons définir un délégué pour votre vue de défilement. L’objet délégué doit être conforme au protocole UIScrollViewDelegate. Cette classe déléguée doit implémenter la méthode viewForZoomingInScrollView() et renvoyer la vue à zoomer.

Modifiez votre ViewController comme indiqué

class ViewController: UIViewController, UIScrollViewDelegate

Ajoutez ensuite la fonction déléguée suivante à la classe.

func viewForZoomingInScrollView(scrollView: UIScrollView) -> UIView? {
    return imageView
}

Créez maintenant une instance UIImageView #

Faire de cette variable une variable de classe

var imageView:UIImageView = UIImageView.init(image: UIImage.init(named: "someImage.jpg"))

Et puis ajoutez-le à scrollview

scrollView?.addSubview(imageView)

Détecter quand UIScrollView a fini de défiler avec des méthodes déléguées #

scrollViewDidEndDecelerating : cela indique au délégué que la vue de défilement a fini de ralentir le mouvement de défilement.

Objectif c: #

- (void)scrollViewDidEndDecelerating:(UIScrollView *)scrollView {
    [self stoppedScrolling];
}

- (void)scrollViewDidEndDragging:(UIScrollView *)scrollView willDecelerate:(BOOL)decelerate {
    if (!decelerate) {
        [self stoppedScrolling];
    }
}

- (void)stoppedScrolling {
    // done, do whatever
}

Rapide: #

func scrollViewDidEndDragging(scrollView: UIScrollView, willDecelerate decelerate: Bool) {
    if !decelerate {
        stoppedScrolling()
    }
}

func scrollViewDidEndDecelerating(scrollView: UIScrollView) {
    stoppedScrolling()
}

func stoppedScrolling() {
    // done, do whatever
}

Restreindre le sens de défilement #

Vous pouvez limiter les directions vers lesquelles l’utilisateur peut défiler à l’aide du code suivant :

func scrollViewDidScroll(_ scrollView: UIScrollView) {
    if scrollView.contentOffset.x != 0 {
        scrollView.contentOffset.x = 0
    }
}

Chaque fois que l’utilisateur fait défiler l’axe des x, le décalage de contenu de scrollView est remis à 0. Vous pouvez évidemment changer les x en ys et ainsi verrouiller la direction pour qu’elle soit uniquement horizontale.

Vous devez également vous assurer de mettre ce code dans la méthode déléguée scrollViewDidScroll(_ scrollView: UIScrollView). Sinon, vous ne le ferez pas fonctionner.

Assurez-vous également d’avoir importé le UIScrollViewDelegate dans votre déclaration de classe, comme ceci :

class ViewController: UIViewController, UIScrollViewDelegate

… et définissez le délégué de scrollView sur self dans une méthode telle que viewDidLoad(_:)

scrollView.delegate = self
