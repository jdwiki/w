UIViews personnalisés à partir de fichiers XIB

  • Éléments de câblage
  • Comment créer une UIView réutilisable personnalisée à l’aide de XIB

Sur cette page

  • Éléments de câblage
  • Comment créer une UIView réutilisable personnalisée à l’aide de XIB

[D’Apple : Création d’une vue personnalisée qui s’affiche dans Interface Builder] [1]

[1] : https://developer.apple.com/library/ios/recipes/xcode_help-IB_objects_media/Chapters/CreatingaLiveViewofaCustomObject.html

• Remarque : Gardez à l’esprit que si vous utilisez des polices “personnalisées” sophistiquées dans vos éléments XIB (tels que UILabel, UITextField, etc.), le temps de chargement initial de votre XIB sera plus long en fonction de la police choisie et de la version du système.

Éléments de câblage #

Créer un fichier XIB

Barre de menus Xcode > Fichier > Nouveau > Fichier. Sélectionnez iOS, Interface utilisateur puis “Afficher”:

[![Première étape][1]][1]

Donnez un nom à votre XIB (oui, nous faisons un exemple Pokemon 👾). N’oubliez pas de vérifier votre cible et appuyez sur “Créer”.

[![Deuxième étape][2]][2]

Concevez votre vue

Pour faciliter les choses, définissez :

  • Taille : forme libre
  • Barre d’état : Aucune
  • Barre supérieure : Aucune
  • Barre inférieure : Aucune

[![Troisième étape][3]][3]

Cliquez sur l’inspecteur de taille et redimensionnez la vue. Pour cet exemple, nous utiliserons une largeur de 321 et une hauteur de 256.

[![Quatrième étape][4]][4]

Déposez quelques éléments dans votre fichier XIB comme indiqué ci-dessous. Ici, nous allons ajouter une Vue d’image (256x256) et un Commutateur.

[![Cinquième étape][5]][5]

Ajoutez des contraintes de mise en page automatique en cliquant sur “Résoudre les problèmes de mise en page automatique” (en bas à droite) et en sélectionnant “Ajouter des contraintes manquantes” sous “Toutes les vues”.

[![Sixième étape][6]][6]

Prévisualisez les modifications que vous avez apportées en cliquant sur “Afficher l’éditeur assistant” (en haut à droite), puis sur “Aperçu”. Vous pouvez ajouter des écrans iPhone en cliquant sur le bouton “Plus”. L’aperçu devrait ressembler à ceci :

[![Septième étape][7]][7]

Sous-classe UIView

Créez la classe qui va gérer le fichier XIB. Barre de menus Xcode > Fichier > Nouveau > Fichier. Sélectionnez iOS / Source / Classe Cocoa Touch. Appuyez sur “Suivant”.

[![Huitième étape][8]][8]

Donnez un nom à la classe, qui doit être le même nom que le fichier XIB (Pokemon). Sélectionnez UIView comme type de sous-classe, puis appuyez sur “Suivant”.

[![Neuvième étape][9]][9]

Dans la fenêtre suivante, sélectionnez votre cible et cliquez sur “Créer”.

[![Dixième étape][10]][10]

Connectez Pokemon.xib à Pokemon.swift via l’attribut “Propriétaire du fichier”

Cliquez sur le fichier Pokemon.xib dans Xcode. Cliquez sur la sortie “Propriétaire du fichier”. Dans “l’inspecteur d’identité” (en haut à droite), définissez la classe sur notre fichier Pokemon.swift récemment créé.

[![Onzième étape][11]][11]

POKÉMONS !!!

Oui! Faites glisser et déposez quelques Pokémons dans votre projet pour terminer notre “infrastructure”. Ici, nous ajoutons deux fichiers PGN, 256x256, transparents.

[![Douzième étape][12]][12]

Montrez-moi déjà le code.

D’accord, d’accord. Il est temps d’ajouter du code à notre classe Pokemon.swift.

C’est en fait assez simple :

  1. Implémenter les initialiseurs requis
  2. Chargez le fichier XIB
  3. Configurez la vue qui affichera le fichier XIB
  4. Afficher la vue ci-dessus

Ajoutez le code suivant à la classe Pokemon.swift :

import UIKit

class Pokemon: UIView {
    
    // MARK: - Initializers
    
    override init(frame: CGRect) {
        super.init(frame: frame)
        setupView()
    }
    
    required init?(coder aDecoder: NSCoder) {
        super.init(coder: aDecoder)
        setupView()
    }
    
    // MARK: - Private Helper Methods
    
    // Performs the initial setup.
    private func setupView() {
        let view = viewFromNibForClass()
        view.frame = bounds

        // Auto-layout stuff.
        view.autoresizingMask = [
            UIViewAutoresizing.flexibleWidth,
            UIViewAutoresizing.flexibleHeight
        ]

        // Show the view.
        addSubview(view)
    }
    
    // Loads a XIB file into a view and returns this view.
    private func viewFromNibForClass() -> UIView {
        
        let bundle = Bundle(for: type(of: self))
        let nib = UINib(nibName: String(describing: type(of: self)), bundle: bundle)
        let view = nib.instantiate(withOwner: self, options: nil).first as! UIView
        
        /* Usage for swift < 3.x
        let bundle = NSBundle(forClass: self.dynamicType)
        let nib = UINib(nibName: String(self.dynamicType), bundle: bundle)
        let view = nib.instantiateWithOwner(self, options: nil)[0] as! UIView
        */

        return view
    }
}

@IBDesignable et @IBInspectable

En ajoutant @IBDesignable à votre classe, vous lui permettez d’être rendu en direct dans Interface Builder. En ajoutant @IBInspectable aux propriétés de votre classe, vous pouvez voir vos vues personnalisées changer dans Interface Builder dès que vous modifiez ces propriétés.

Rendons la “Vue d’image” de notre vue personnalisée “Inspectable”.

Tout d’abord, reliez la “Vue d’image” du fichier Pokemon.xib à la classe Pokemon.swift.

[![Treizième étape][13]][13]

Appelez l’outlet imageView puis ajoutez le code suivant (notez le @IBDesignable avant le nom de la classe):

@IBDesignable class Pokemon: UIView {
    
    // MARK: - Properties
    
    @IBOutlet weak var imageView: UIImageView!
    
    @IBInspectable var image: UIImage? {
        get {
            return imageView.image
        }
        set(image) {
            imageView.image = image
        }
    }

    // MARK: - Initializers
    ...

Comment créer une UIView réutilisable personnalisée à l’aide de XIB #

L’exemple suivant montre les étapes impliquées dans l’initialisation d’une vue à partir de XIB.

Ce n’est pas une opération complexe, mais des étapes précises doivent être suivies afin de le faire correctement du premier coup, en évitant les exceptions.

[Comment fonctionne loadNibNamed][1]

Les principales étapes sont :

  1. Créer un XIB
  2. Créez les classes .h et .m
  3. Définir les points de vente en .h
  4. Connectez les prises entre .h et XIB

Voir capture d’écran ci-joint :

[![Objet Label connecté à myLabel IBOutlet Variable UILabel][2]][2]

  1. Appelez loadNibNamed dans la fonction initWithCoder du fichier .m. Cela est nécessaire pour vous assurer que vous pouvez placer directement l’objet UIView dans le fichier Storyboard / Parent UIView XIB et le définir comme votre vue personnalisée. Aucun autre code d’initialisation n’est nécessaire une fois que vous avez chargé le storyboard/XIB parent. Votre vue personnalisée peut être ajoutée à d’autres vues, tout comme les autres objets de vue Objective C intégrés donnés dans XCode.
