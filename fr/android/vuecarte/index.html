Un FrameLayout avec un arrière-plan à coins arrondis et une ombre.

CardView utilise la propriété d’élévation sur Lollipop pour les ombres et revient à une implémentation d’ombre émulée personnalisée sur les anciennes plates-formes.

En raison de la nature coûteuse du découpage des coins arrondis, sur les plates-formes antérieures à Lollipop, CardView ne découpe pas ses enfants qui se croisent avec des coins arrondis. Au lieu de cela, il ajoute un rembourrage pour éviter une telle intersection (voir setPreventCornerOverlap(boolean) pour changer ce comportement).

Paramètres #

Paramètre Détails
cardBackgroundColor Couleur d’arrière-plan pour CardView.
cardCornerRadius Rayon d’angle pour CardView.
carteAltitude Altitude pour CardView.
cardMaxElevation Altitude maximale pour CardView.
cardPreventCornerOverlap Ajoutez un rembourrage à CardView sur v20 et avant pour éviter les intersections entre le contenu de la carte et les coins arrondis.
cardUseCompatPadding Ajoutez également un rembourrage dans l’API v21+ pour avoir les mêmes mesures avec les versions précédentes. Peut être une valeur booléenne, telle que “true” ou “false”.
contentPadding Rembourrage intérieur entre les bords de la carte et les enfants de la CardView.
contentPaddingBottom Rembourrage intérieur entre le bord inférieur de la carte et les enfants de la CardView.
contentPaddingLeft Rembourrage intérieur entre le bord gauche de la carte et les enfants de la CardView.
contentPaddingRight Altitude pour CardView.
carteAltitude Rembourrage intérieur entre le bord droit de la carte et les enfants de la CardView.
contentPaddingTop Rembourrage intérieur entre le bord supérieur de la carte et les enfants de la CardView.

CardView utilise une élévation réelle et des ombres dynamiques sur Lollipop (API 21) et au-dessus. Cependant, avant Lollipop, CardView retombe sur une implémentation d’ombre programmatique.

Si vous essayez de faire tenir un ImageView dans les coins arrondis d’un CardView, vous remarquerez peut-être qu’il ne semble pas correct avant Lollipop (API 21). Pour résoudre ce problème, vous devez appeler setPreventCornerOverlap(false) sur votre CardView ou ajouter app:cardPreventCornerOverlap="false" à votre mise en page.

Avant d’utiliser CardView, vous devez ajouter la dépendance de la bibliothèque de support dans le fichier build.gradle :

dependencies{
    compile 'com.android.support:cardview-v7:25.2.0'
}
  • Un certain nombre de la dernière version peut être trouvée [ici] [1] *

Premiers pas avec CardView #

CardView est membre de la bibliothèque de support Android et fournit une mise en page pour les cartes.

Pour ajouter CardView à votre projet, ajoutez la ligne suivante à vos dépendances build.gradle.

compile 'com.android.support:cardview-v7:25.1.1'
  • Un certain nombre de la dernière version peut être trouvée [ici] [1] *

Dans votre mise en page, vous pouvez ensuite ajouter les éléments suivants pour obtenir une carte.

<android.support.v7.widget.CardView
        xmlns:card_view="http://schemas.android.com/apk/res-auto"
        android:layout_width="match_parent"
        android:layout_height="wrap_content">

        <!-- one child layout containing other layouts or views -->     

</android.support.v7.widget.CardView>

Vous pouvez ensuite ajouter d’autres mises en page à l’intérieur et elles seront incluses dans une carte.

En outre, CardView peut être rempli avec n’importe quel élément d’interface utilisateur et manipulé à partir de [code][2].

<?xml version="1.0" encoding="utf-8"?>
<android.support.v7.widget.CardView
    xmlns:android="http://schemas.android.com/apk/res/android"
    xmlns:card_view="http://schemas.android.com/apk/res-auto"
    android:layout_width="match_parent"
    android:layout_height="wrap_content"
    android:id="@+id/card_view"
    android:layout_margin="5dp"
    card_view:cardBackgroundColor="#81C784"
    card_view:cardCornerRadius="12dp"
    card_view:cardElevation="3dp"
    card_view:contentPadding="4dp" >

    <RelativeLayout
        android:layout_width="match_parent"
        android:layout_height="wrap_content"
        android:padding="16dp" >

        <ImageView
            android:layout_width="100dp"
            android:layout_height="100dp"
            android:id="@+id/item_image"
            android:layout_alignParentLeft="true"
            android:layout_alignParentTop="true"
            android:layout_marginRight="16dp"
            />

        <TextView
            android:layout_width="wrap_content"
            android:layout_height="wrap_content"
            android:id="@+id/item_title"
            android:layout_toRightOf="@+id/item_image"
            android:layout_alignParentTop="true"
            android:textSize="30sp"
            />

        <TextView
            android:layout_width="wrap_content"
            android:layout_height="wrap_content"
            android:id="@+id/item_detail"
            android:layout_toRightOf="@+id/item_image"
            android:layout_below="@+id/item_title"
            />

    </RelativeLayout>
</android.support.v7.widget.CardView>

<android.support.v7.widget.CardView

Personnalisation de CardView #

CardView fournit une élévation et un rayon d’angle par défaut afin que les cartes aient une apparence cohérente sur les plates-formes.

Vous pouvez personnaliser ces valeurs par défaut à l’aide de ces attributs dans le fichier xml :

  1. L’attribut card_view:cardElevation ajoute une élévation dans CardView.
  2. L’attribut card_view:cardBackgroundColor est utilisé pour customize background color of CardView’s background(you can give any color).
  3. L’attribut card_view:cardCornerRadius est utilisé pour courber 4 bords de CardView
  4. L’attribut card_view:contentPadding ajoute un rembourrage entre la carte et children of card

Remarque : card_view est un espace de noms défini dans la vue de mise en page parente la plus élevée. xmlns:card_view=“http://schemas.android.com/apk/res-auto”

Voici un exemple :

<android.support.v7.widget.CardView
        xmlns:card_view="http://schemas.android.com/apk/res-auto"
        android:layout_width="match_parent"
        android:layout_height="wrap_content"
        card_view:cardElevation="4dp"
        card_view:cardBackgroundColor="@android:color/white"
        card_view:cardCornerRadius="8dp"
        card_view:contentPadding="16dp">

        <!-- one child layout containing other layouts or views -->     

</android.support.v7.widget.CardView>

Vous pouvez également le faire par programmation en utilisant :

card.setCardBackgroundColor(....);
card.setCardElevation(...);
card.setRadius(....);
card.setContentPadding();

Consultez le [javadoc officiel] [1] pour des propriétés supplémentaires.

[1] : https://developer.android.com/reference/android/support/v7/widget/CardView.html

Utilisation d’images comme arrière-plan dans CardView (problèmes d’appareils pré-Lollipop) #

Lorsque vous utilisez Image/Couleur comme arrière-plan dans un CardView, vous pouvez vous retrouver avec de légers rembourrages blancs (si la couleur par défaut de la carte est blanche) sur les bords. Cela se produit en raison des coins arrondis par défaut dans la vue Carte. Voici comment éviter ces marges dans les appareils pré-sucette.

Nous devons utiliser un attribut card_view:cardPreventCornerOverlap="false" dans le CardView. 1). En XML, utilisez l’extrait de code suivant.

<android.support.v7.widget.CardView
    xmlns:card_view="http://schemas.android.com/apk/res-auto"
    android:layout_width="match_parent"
    card_view:cardPreventCornerOverlap="false"
    android:layout_height="wrap_content"> 
      <ImageView
            android:id="@+id/row_wallet_redeem_img"
            android:layout_width="match_parent"
            android:layout_height="match_parent"
            android:adjustViewBounds="true"
            android:scaleType="centerCrop"
            android:src="@drawable/bg_image" />

</android.support.v7.widget.CardView>
  1. En Java comme ceci cardView.setPreventCornerOverlap(false).

Cela supprime un rembourrage indésirable sur les bords de la carte. Voici quelques exemples visuels liés à cette implémentation.

1 carte avec fond d’image en API 21 (parfaitement bien) [![Carte avec image d’arrière-plan dans l’API 21][2]][2]

2 Carte avec fond d’image dans API 19 sans attribut (remarquez les rembourrages autour de l’image) [![Carte avec fond d’image dans l’API 19 sans attribut][3]][3]

3 FIXE Carte avec image d’arrière-plan dans l’API 19 avec attribut cardView.setPreventCornerOverlap(false) (Problème maintenant corrigé) [![FIXED Carte avec image d’arrière-plan dans l’API 19 avec attribut][4]][4]

Animer la couleur d’arrière-plan de CardView avec TransitionDrawable #

public void setCardColorTran(CardView card) {
    ColorDrawable[] color = {new ColorDrawable(Color.BLUE), new ColorDrawable(Color.RED)};
    TransitionDrawable trans = new TransitionDrawable(color);
    if(Build.VERSION.SDK_INT > Build.VERSION_CODES.ICE_CREAM_SANDWICH_MR1) {
        card.setBackground(trans);
    } else {
        card.setBackgroundDrawable(trans);
    }
    trans.startTransition(5000);
}
