Installation de la console Gremlin #

Le moyen le plus simple de démarrer avec Gremlin consiste à installer la [Console Gremlin][1]. La console Gremlin est un [REPL] [2] qui permet un retour immédiat sur les résultats des traversées Gremlin.

Comme prérequis, [Java 8][3] est requis pour que la console Gremlin fonctionne. Assurez-vous qu’il est installé avant de poursuivre avec les étapes suivantes.

[Téléchargez][4] la console, décompressez-la et démarrez-la :

$ unzip apache-gremlin-console-x.y.z-bin.zip
$ cd apache-gremlin-console-x.y.z
$ bin/gremlin.sh

         \,,,/
         (o o)
-----oOOo-(3)-oOOo-----
plugin activated: tinkerpop.server
plugin activated: tinkerpop.utilities
plugin activated: tinkerpop.tinkergraph
gremlin>

Si sous Windows, il y a un fichier gremlin.bat inclus qui peut être utilisé pour démarrer la console.

Pour en savoir plus sur la console Gremlin, veuillez lire le [tutoriel][5] de TinkerPop qui traite de son utilisation plus en détail.

[1] : http://tinkerpop.apache.org/docs/current/reference/#gremlin-console [2] : https://en.wikipedia.org/wiki/Read%E2%80%93eval%E2%80%93print_loop [3] : http://www.oracle.com/technetwork/java/index.html [4] : https://www.apache.org/dyn/closer.lua/tinkerpop/3.2.0/apache-gremlin-console-3.2.0-bin.zip [5] : http://tinkerpop.apache.org/docs/current/tutorials/the-gremlin-console/

Utilisation des graphiques jouets #

Les “jouets graphiques” de TinkerPop permettent d’essayer rapidement certaines fonctionnalités de base de Gremlin. Ces graphiques sont pré-construits et fournis avec la console Gremlin. Les “graphiques de jouets” les plus couramment utilisés sont “Modern” et “The Crew”. Lorsque vous posez des questions sur StackOverflow ou sur la liste de diffusion [Gremlin Users][1], il est souvent utile de cadrer les questions dans le contexte de ces graphiques, car ils peuvent aider la communauté à répondre rapidement et facilement aux questions.

Les graphiques Modern et Crew peuvent tous deux être construits avec TinekrFactory, qui construira un TinkerGraph en mémoire avec les données préchargées :

gremlin> graph = TinkerFactory.createModern()
==>tinkergraph[vertices:6 edges:6]
gremlin> g = graph.traversal()
==>graphtraversalsource[tinkergraph[vertices:6 edges:6], standard]
gremlin> g.V()
==>v[1]
==>v[2]
==>v[3]
==>v[4]
==>v[5]
==>v[6]

Notez les conventions dans le code ci-dessus. L’instance Graph est généralement appelée “graph”, puis pour exécuter des parcours (c’est-à-dire des requêtes), on crée un TraversalSource à partir de ce Graph appelé “g”. Ensuite, la requête g.V() exécute un parcours qui obtient une liste de tous les sommets dans “g”.

Pour créer le graphique “The Crew”, qui comporte des méta/multi-propriétés, utilisez TinkerFactory.createTheCrew().

Vous trouverez plus d’informations sur l’utilisation des graphiques de jouets dans le didacticiel de TinkerPop intitulé [The Gremlin Console][2].

[1] : https://groups.google.com/forum/#!forum/gremlin-users [2] : http://tinkerpop.apache.org/docs/current/tutorials/the-gremlin-console/#toy-graphs
