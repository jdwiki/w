Au moment de la rédaction (août 2016), le sélecteur parent (&) fait toujours référence à la chaîne complète du sélecteur parent jusqu’au niveau le plus élevé. Il ne peut pas être utilisé pour sélectionner uniquement le parent immédiat ou l’ancêtre le plus racine seul.

Autrement dit, dans le code ci-dessous, &#type1 se résoudrait en #demo-container .content#type1 et pas seulement en .content ou simplement en #demo-container.

 #demo-container {
  padding: 4px;
  border: 1px solid gray;
  #heading {
    padding: 4px;
    font-size: 20px;
  }
  .content {
    padding: 2px;
    font-size: 18px;
    &#type1 {
      color: chocolate;
    }
  }
}

Sélecteur parent de base #

L’opérateur & est le sélecteur parent. Lorsqu’il est utilisé dans ou en tant que sélecteur, il est remplacé par les sélecteurs parents complets (séquence complète de sélecteurs jusqu’au niveau le plus élevé d’un bloc imbriqué) dans la sortie CSS finale.

Il est utile lors de la création de règles imbriquées nécessitant l’utilisation du sélecteur parent d’une manière différente de celle par défaut, comme la modification de l’ordre de placement du sélecteur parent ou pour le concaténer avec d’autres sélecteurs.

a {
  text-decoration: none;
  &:hover {
    text-decoration: underline;
  }
}

Les résultats dans le CSS suivant où le sélecteur parent a a été concaténé avec la règle :hover :

a {
  text-decoration: none;
}
a:hover {
  text-decoration: underline;
}

L’un des grands avantages de l’utilisation de sélecteurs parents dans la mesure du possible est la réduction de la répétition des sélecteurs.

Modification de l’ordre du sélecteur dans un bloc imbriqué #

Less permet l’utilisation du sélecteur parent (&) n’importe où dans un sélecteur complexe et permet ainsi de changer de style lorsque l’élément actuel se trouve dans un autre élément, ce qui lui donne un contexte différent :

Par exemple, dans le code ci-dessous, le sélecteur parent est placé à la fin et devient ainsi le sélecteur enfant dans le CSS compilé.

a {
  color: blue;
  .disabled-section & {
    color: grey;
  }
}

CSS compilé :

a {
  color: blue;
}
.disabled-section a {
  color: grey;
}

Sélectionnez les éléments frères qui ont la même classe sans répéter le sélecteur #

Less ne met aucune restriction sur le nombre de fois que le sélecteur parent (&) peut être utilisé dans un sélecteur complexe et donc, nous pouvons l’utiliser plus d’une fois comme dans les exemples ci-dessous pour sélectionner des éléments frères sans avoir besoin de répéter le sélecteur.

.demo {
  border: 1px solid black; /* add border to all elements with demo class */
  & + & { /* select all .demo that have another .demo sibling immediately prior */
    background: red;
  }
  & + & + & { /* select all .demo that have two .demo sibling immediately prior */
    background: chocolate;
  }
  & ~ & { /* select all .demo elements that have another .demo sibling prior */
    color: beige;
  }
}

Le code ci-dessus, une fois compilé, donnera le CSS suivant :

.demo {
  border-left: 1px solid black;
}
.demo + .demo {
  background: red;
}
.demo + .demo + .demo {
  background: chocolate;
}
.demo ~ .demo {
  color: beige;
}
