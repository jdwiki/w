Introduction #

Les mixins sont similaires à la définition et à l’appel de fonctions. Dites, si nous devons créer un style répétitif, les mixins sont pratiques. Les mixins peuvent ou non prendre des paramètres. Par exemple :

.default-round-borders {
          border-radius: 5px;
     -moz-border-radius: 5px;
  -webkit-border-radius: 5px;
}

.round-borders (@radius) {
          border-radius: @radius;
     -moz-border-radius: @radius;
  -webkit-border-radius: @radius;
}

Nous avons deux types de déclarations ci-dessus. L’un prend un paramètre et l’autre non. Voyons comment cela est utilisé quelque part :

@sky-blue: #87ceeb;
@dark-sky-blue: #baffff;

#header {
  background: @sky-blue;
  .default-round-borders;
}

.btn {
  background: @dark-sky-blue;
  .round-borders(3px);
}

Le code ci-dessus, compilé tous ensemble, donnera une sortie comme celle-ci :

#header {
  background: #87ceeb;
          border-radius: 5px;
     -moz-border-radius: 5px;
  -webkit-border-radius: 5px;
}
.btn {
  background: #baffff;
          border-radius: 3px;
     -moz-border-radius: 3px;
  -webkit-border-radius: 3px;
}

Empêcher une définition de mixin d’apparaître dans le fichier CSS compilé #

.default-settings() {
  padding: 4px;
  margin: 4px;
  font-size: 16px;
  border: 1px solid gray;
}

#demo {
  .default-settings;
}

L’exemple ci-dessus, une fois compilé, ne produirait que la sortie suivante. La définition du mixin .default-settings() ne serait pas sortie dans le fichier CSS compilé car des parenthèses sont ajoutées après.

#demo {
  padding: 4px;
  margin: 4px;
  font-size: 16px;
  border: 1px solid gray;
}

Si ces parenthèses ne sont pas attachées, le compilateur Less traitera également la définition du mixin comme un sélecteur CSS et l’imprimera donc également dans le fichier CSS de sortie.

Ajoutez !important à chaque propriété d’un mixin sans la saisir manuellement #

Lorsque nous attachons le mot-clé !important à un appel de mixin, le compilateur Less ajoute automatiquement le !important à toutes les propriétés présentes dans le mixin.

Par exemple, considérez le mixin ci-dessous :

.set-default-props() {
  margin: 4px;
  padding: 4px;
  border: 1px solid black;
  font-weight: normal;
}

Lorsque le !important est attaché à l’appel mixin comme ci-dessous,

#demo {
  .set-default-props() !important;
}

le CSS compilé ressemblerait à ceci :

#demo {
  margin: 4px !important;
  padding: 4px !important;
  border: 1px solid black !important;
  font-weight: normal !important;
}

Remarque : L’utilisation de !important est considérée comme une mauvaise pratique et ne doit être utilisée qu’en dernier recours.
