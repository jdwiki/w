Écrire une boucle for-each #

Une boucle for-each dans Less a les mêmes composants clés qu’une boucle for à l’exception des différences suivantes :

  • Une variable qui contient la liste des éléments qui doivent être itérés.
  • Une fonction extract() pour extraire chaque élément de la variable en fonction de l’index de la boucle.
  • Une fonction length() pour calculer la longueur du tableau (c’est-à-dire le nombre d’éléments dans la liste) et l’utiliser dans l’appel primaire du mixin (pour [initialization]).

Vous trouverez ci-dessous un exemple de boucle for-each écrite en Less qui parcourt chaque élément de la variable @images, crée un sélecteur #id où l’id est le même que le nom de l’élément/image et définit également l’arrière-plan propriété d’image pour cela.

@images: cat, dog, apple, orange; /* the array list of items */

.for-each-loop(@index) when (@index > 0) { /* recursive mixin call with guard - condition */
  
  @image: extract(@images, @index); /* extract function to fetch each item from the list */
  
 /* the statement */
  #@{image} {
    background-image: url("http://mysite.com/@{image}.png");
  }
  /* end of the statement */

  .for-each-loop(@index - 1); /* the next iteration's call - final-expression */
}

.for-loop(length(@images)); /* the primary call with length() function - initialization */

CSS compilé :

#orange {
  background-image: url("http://mysite.com/orange.png");
}
#apple {
  background-image: url("http://mysite.com/apple.png");
}
#dog {
  background-image: url("http://mysite.com/dog.png");
}
#cat {
  background-image: url("http://mysite.com/cat.png");
}

Écrire une boucle for simple #

L’utilisation de boucles est un excellent moyen de garder le code SEC et d’éviter les répétitions. Contrairement à Sass, il n’y a pas de directive intégrée @for ou @each dans Less pour écrire des boucles, mais elle peut toujours être écrite à l’aide de mixins récursifs. Un mixin récursif n’est rien d’autre qu’un mixin qui ne cesse de s’appeler.

Il y a quatre composants clés dans une boucle écrite en utilisant Less et ils sont les suivants :

  • Un mixin avec des expressions de garde. La garde est utilisée pour terminer la boucle lorsque les critères de sortie de la boucle sont remplis. En termes de boucle for JavaScript (for([initialization]; [condition]; [final-expression])), la garde est la [condition].
  • Un appel primaire au mixin pour exécuter la première itération. Cet appel principal au mixin peut être effectué depuis un bloc de sélecteur (si le mixin n’a pas de sélecteur enveloppant tout son contenu) ou depuis l’extérieur d’un bloc de sélecteur (si le mixin a un sélecteur enveloppant son contenu). En termes de boucle for JavaScript, cet appel principal sert de [initialization] car il définit la valeur de base de la variable de type compteur.
  • Un appel au mixin depuis lui-même pour le rendre récursif. Cet appel passe généralement une valeur incrémentée ou décrémentée de la variable compteur comme argument. Ainsi, il invoque les itérations suivantes. En termes de boucle for JS, cela fait le [final-expression] avec le prochain appel.
  • Dernier point mais non le moindre, l’autre contenu du mixin qui équivaut à la “déclaration” dans une syntaxe de boucle for typique.

Ci-dessous se trouve une simple boucle for écrite en Less qui crée plusieurs sélecteurs #img* (où * est un nombre) et définit également la propriété background-image comme image*.png.

.for-loop(@index) when (@index > 0) { /* recursive mixin with guard expression - condition */

  /* the statement */
  #img@{index} {
    background-image: url("http://mysite.com/image@{index}.png");
  }
  /* end of the statement */

  .for-loop(@index - 1); /* the next iteration's call - final-expression*/
}
.for-loop(3); /* the primary call - initialization */

CSS compilé :

#img3 {
  background-image: url("http://mysite.com/image3.png");
}
#img2 {
  background-image: url("http://mysite.com/image2.png");
}
#img1 {
  background-image: url("http://mysite.com/image1.png");
}
