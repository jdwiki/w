  • Styliser un élément en fonction de la valeur d’une variable

Syntaxe #

  • Mélanger avec les gardes
  • .mixin-name(<arguments>) lorsque <is-negation> (<ref-variable> <operator> <value>)
  • Gardes CSS
  • <selector> lorsque <is-negation> (<ref-variable> <operator> <value>)

Paramètres #

Paramètre Détails
arguments Les variables qui sont passées au mixin paramétrique. Les arguments sont facultatifs.
est-négation Ceci indique si la condition de garde est une condition “non” ou non. Par exemple when not (@type = error) signifie que le mixin sera utilisé chaque fois que la valeur de @type n’est pas error.
variable de référence Il s’agit de la variable dont la valeur détermine quelles propriétés de mixin doivent être appliquées à l’élément. Ceci est obligatoire.
opérateur Il s’agit de l’opérateur utilisé pour évaluer la condition. Cela peut être =, >, <, >=, <=. Ce n’est pas obligatoire. Lorsque l’opérateur et la valeur ne sont pas donnés, le compilateur suppose que la condition est &lt;ref-variable> = true.
valeur Valeur utilisée pour évaluer la condition. Ceci est facultatif mais devient obligatoire lorsqu’un opérateur est fourni.

Styliser un élément en fonction de la valeur d’une variable #

.set-colors(@type) when (@type = error) {
  @base-color: #d9534f;
  background: @base-color;
  color: contrast(@base-color, lighten(@base-color, 25%), darken(@base-color, 25%));
  border: 1px solid contrast(@base-color, lighten(@base-color, 25%), darken(@base-color, 25%));
}
.set-colors(@type) when (@type = info) {
  @base-color: #5bc0de;
  background: @base-color;
  color: contrast(@base-color, lighten(@base-color, 5%), darken(@base-color, 5%));
  border: 1px solid contrast(@base-color, lighten(@base-color, 5%), darken(@base-color, 5%));
}
.set-colors() {
  background: white;
  color: black;
  border: 1px solid black;
}

.error-message {
  .set-colors(error);
}
.info-message {
  .set-colors(info);
}
.default-div {
  .set-colors;
}

Dans l’exemple ci-dessus, le background, border et color sont attribués en fonction du type d’élément. Si l’élément est un texte par défaut “div”, l’arrière-plan sera blanc alors que le texte et la bordure seront noirs. S’il s’agit d’un affichage de message “d’erreur” div ou d’un affichage de message “d’information” div, les couleurs sont attribuées en fonction du type.

La sortie CSS compilée serait la suivante :

.error-message {
  background: #d9534f;
  color: #f0b9b8;
  border: 1px solid #f0b9b8;
}
.info-message {
  background: #5bc0de;
  color: #46b8da;
  border: 1px solid #46b8da;
}
.default-div {
  background: white;
  color: black;
  border: 1px solid black;
}
