Mappage plusieurs à plusieurs

  • Paramètres

Sur cette page

Un mappage ManyToMany décrit une relation entre des entités où les deux peuvent être liées à plusieurs instances l’une de l’autre, et est défini par l’annotation @ManyToMany.

Contrairement à @OneToMany où une colonne de clé étrangère dans la table de l’entité peut être utilisée, ManyToMany nécessite une table de jointure, qui mappe les entités les unes aux autres.

Paramètres #

Annotation Objectif
@TableGenerator Définit un générateur de clé primaire qui peut être référencé par son nom lorsqu’un élément générateur est spécifié pour l’annotation GeneratedValue
@GeneratedValue Fournit la spécification des stratégies de génération pour les valeurs des clés primaires. Il peut être appliqué à une propriété de clé primaire ou à un champ d’une entité ou d’une superclasse mappée en conjonction avec l’annotation Id.
@ManyToMany Spécifie la relation entre les entités Employé et Projet de sorte que de nombreux employés puissent travailler sur plusieurs projets.
mappedBy="projets" Définit une relation bidirectionnelle entre Employé et Projet
@JoinColumn Spécifie le nom de la colonne qui fera référence à l’Entité à considérer comme propriétaire de l’association
@JoinTable Spécifie la table dans la base de données qui contiendra les relations employé-projet à l’aide de clés étrangères
  • @TableGenerator et @GeneratedValue sont utilisés pour l’identification automatique création à l’aide du générateur de table jpa.
  • L’annotation @ManyToMany spécifie la relation entre Employee et Entités de projet.
  • @JoinTable spécifie le nom de la table à utiliser comme table de jointure jpa mappage plusieurs à plusieurs entre Employé et Projet en utilisant name = “employé_projet”. Ceci est fait parce qu’il n’y a aucun moyen de déterminer la propriété d’un mappage jpa plusieurs à plusieurs comme le font les tables de base de données ne pas contenir de clés étrangères pour faire référence à une autre table.
  • @JoinColumn spécifie le nom de la colonne qui fera référence au Entité à considérer comme propriétaire de l’association @inverseJoinColumn spécifie le nom du côté inverse de relation. (Vous pouvez choisir n’importe quel côté pour être considéré comme propriétaire. Assurez-vous simplement que ces côtés sont en relation). Dans notre cas nous avons Employé choisi comme propriétaire donc @JoinColumn fait référence à idemployee colonne dans la table de jointure employee_project et @InverseJoinColumn fait référence à idproject qui est l’inverse du mappage jpa plusieurs à plusieurs.
  • L’annotation @ManyToMany dans l’entité Project montre une relation inverse par conséquent, il utilise mappedBy=projects pour faire référence au champ dans Employee entité.

Un exemple complet peut être référé [ici][1]

[1] : http://www.thejavageek.com/2014/01/20/jpa-many-many-mapping/

Mappage Employé à Projet Plusieurs à Plusieurs #

Entité salariée.

package com.thejavageek.jpa.entities;

import java.util.List;

import javax.persistence.CascadeType;
import javax.persistence.Entity;
import javax.persistence.GeneratedValue;
import javax.persistence.GenerationType;
import javax.persistence.Id;
import javax.persistence.JoinColumn;
import javax.persistence.JoinTable;
import javax.persistence.ManyToMany;
import javax.persistence.TableGenerator;

@Entity
public class Employee {

    @TableGenerator(name = "employee_gen", table = "id_gen", pkColumnName = "gen_name", valueColumnName = "gen_val", allocationSize = 100)
    @Id
    @GeneratedValue(strategy = GenerationType.TABLE, generator = "employee_gen")
    private int idemployee;
    private String name;

    @ManyToMany(cascade = CascadeType.PERSIST)
    @JoinTable(name = "employee_project", joinColumns = @JoinColumn(name = "idemployee"), inverseJoinColumns = @JoinColumn(name = "idproject"))
    private List<Project> projects;

    public int getIdemployee() {
        return idemployee;
    }

    public void setIdemployee(int idemployee) {
        this.idemployee = idemployee;
    }

    public String getName() {
        return name;
    }

    public void setName(String name) {
        this.name = name;
    }

    public List<Project> getProjects() {
        return projects;
    }

    public void setProjects(List<Project> projects) {
        this.projects = projects;
    }

}

Entité du projet :

package com.thejavageek.jpa.entities;

import java.util.List;

import javax.persistence.CascadeType;
import javax.persistence.Entity;
import javax.persistence.GeneratedValue;
import javax.persistence.GenerationType;
import javax.persistence.Id;
import javax.persistence.ManyToMany;
import javax.persistence.TableGenerator;

@Entity
public class Project {

    @TableGenerator(name = "project_gen", allocationSize = 1, pkColumnName = "gen_name", valueColumnName = "gen_val", table = "id_gen")
    @Id
    @GeneratedValue(generator = "project_gen", strategy = GenerationType.TABLE)
    private int idproject;
    private String name;

    @ManyToMany(mappedBy = "projects", cascade = CascadeType.PERSIST)
    private List<Employee> employees;

    public int getIdproject() {
        return idproject;
    }

    public void setIdproject(int idproject) {
        this.idproject = idproject;
    }

    public String getName() {
        return name;
    }

    public void setName(String name) {
        this.name = name;
    }

    public List<Employee> getEmployees() {
        return employees;
    }

    public void setEmployees(List<Employee> employees) {
        this.employees = employees;
    }

}

Comment gérer la clé composée sans annotation Embeddable #

Si tu as

Role:
+-----------------------------+
| roleId | name | discription |
+-----------------------------+


Rights:
+-----------------------------+
| rightId | name | discription|
+-----------------------------+

rightrole
+------------------+
| roleId | rightId | 
+------------------+

Dans le scénario ci-dessus, la table rightrole a une clé composée et pour y accéder dans JPA, l’utilisateur doit créer une entité avec l’annotation Embeddable.

Comme ça:

L’entité pour la bonne table de rôles est :

    @Entity
    @Table(name = "rightrole")
    public class RightRole extends BaseEntity<RightRolePK> {
    
        private static final long serialVersionUID = 1L;
    
        @EmbeddedId
        protected RightRolePK rightRolePK;

    
        @JoinColumn(name = "roleID", referencedColumnName = "roleID", insertable = false, updatable = false)
        @ManyToOne(fetch = FetchType.LAZY)
        private Role role;
    
        @JoinColumn(name = "rightID", referencedColumnName = "rightID", insertable = false, updatable = false)
        @ManyToOne(fetch = FetchType.LAZY)
        private Right right;

        ......
     }


    @Embeddable
    public class RightRolePK implements Serializable {
    private static final long serialVersionUID = 1L;

      @Basic(optional = false)
      @NotNull
      @Column(nullable = false)
      private long roleID;

      @Basic(optional = false)
      @NotNull
      @Column(nullable = false)
     private long rightID;

   .....

}

L’annotation intégrable convient à un seul objet, mais cela posera un problème lors de l’insertion d’enregistrements en bloc.

Le problème est que chaque fois que l’utilisateur veut créer un nouveau role avec rights, le premier utilisateur doit store (persist) role object, puis l’utilisateur doit faire flush pour obtenir le id nouvellement généré pour le rôle. puis, l’utilisateur peut le mettre dans l’objet de l’entité rightrole.

Pour résoudre ce problème, l’utilisateur peut écrire une entité de manière légèrement différente.

L’entité pour la table des rôles est :

@Entity
@Table(name = "role")
public class Role {

    private static final long serialVersionUID = 1L;

    @Id
    @GeneratedValue(strategy = GenerationType.IDENTITY)
    @Basic(optional = false)
    @NotNull
    @Column(nullable = false)
    private Long roleID;

    
    @OneToMany(cascade = CascadeType.ALL, mappedBy = "role", fetch = FetchType.LAZY)
    private List<RightRole> rightRoleList;
 
    @ManyToMany(cascade = {CascadeType.PERSIST})
    @JoinTable(name = "rightrole",
            joinColumns = {
                @JoinColumn(name = "roleID", referencedColumnName = "ROLE_ID")},
            inverseJoinColumns = {
                @JoinColumn(name = "rightID", referencedColumnName = "RIGHT_ID")})
    private List<Right> rightList;
.......
}

L’annotation @JoinTable se chargera de l’insertion dans la table rightrole même sans entité (tant que cette table n’a que les colonnes id de role et right).

L’utilisateur peut alors simplement :

Role role = new  Role();
List<Right> rightList = new ArrayList<>();
Right right1 = new Right();
Right right2 = new Right();
rightList.add(right1);
rightList.add(right2);
role.setRightList(rightList);

Vous devez écrire @ManyToMany(cascade = {CascadeType.PERSIST}) dans inverseJoinColumns sinon vos données parent seront supprimées si l’enfant est supprimé.
