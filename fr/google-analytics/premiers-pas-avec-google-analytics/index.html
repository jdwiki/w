Embarquez avec Google Analytics #

  1. Obtenir un compte GA : If you don’t have an Analytics account, create one. If you do have an Analytics account, sign in. Both options are available at google.com/analytics

  2. Configuration d’une propriété dans votre compte Analytics : A property represents your website or app where the data gets aggregated.

  3. Créer une vue : Views let you create filtered perspectives of your data. When you create a property one view is created by default. You can create multiple views based on the requirement and filter the reports based on the reporting structure.

  4. Intégrez l’analyse dans votre site Web : accédez à la propriété > informations de suivi et obtenez le code de suivi qui ressemble à celui ci-dessous

  5. La chaîne “UA-XXXXX-Y” doit être remplacée par l’ID de propriété (également appelé “ID de suivi”) de la propriété Google Analytics que vous souhaitez suivre.

Avec ces étapes simples, votre site Web sera prêt à envoyer les pages vues à GA.

Ajouter analytics.js à votre site Web #

Ajoutez le code suivant (appelé “extrait de suivi JavaScript”) aux modèles de votre site.

Le code doit être ajouté avant la balise de fermeture et la chaîne ‘UA-XXXXX-Y’ doit être remplacée par l’[ID de propriété][1] (également appelé “ID de suivi”) de la propriété Google Analytics que vous souhaitez suivre.

<!-- Google Analytics -->
<script>
(function(i,s,o,g,r,a,m){i['GoogleAnalyticsObject']=r;i[r]=i[r]||function(){
(i[r].q=i[r].q||[]).push(arguments)},i[r].l=1*new Date();a=s.createElement(o),
m=s.getElementsByTagName(o)[0];a.async=1;a.src=g;m.parentNode.insertBefore(a,m)
})(window,document,'script','https://www.google-analytics.com/analytics.js','ga');

ga('create', 'UA-XXXXX-Y', 'auto');
ga('send', 'pageview');
</script>
<!-- End Google Analytics -->

Le code ci-dessus fait quatre choses principales :

  1. Crée un élément <script> qui commence à télécharger de manière asynchrone la bibliothèque JavaScript analytics.js à partir de https://www.google-analytics.com/analytics.js
  2. Initialise une fonction ga globale (appelée file d’attente de commandes ga()) qui vous permet de planifier l’exécution des commandes une fois que la bibliothèque analytics.js est chargée et prête à fonctionner.
  3. Ajoute une commande à la file d’attente de commandes ga() pour créer un nouvel objet de suivi pour la propriété spécifiée via le paramètre 'UA-XXXXX-Y'.
  4. Ajoute une autre commande à la file d’attente de commandes ga() pour envoyer une page vue à Google Analytics pour la page actuelle.

[1] : https://support.google.com/analytics/answer/1032385

Aperçu #

Google Analytics est utilisé pour suivre l’activité des utilisateurs sur votre site Web ou votre application mobile.

Pour configurer google-analytics sur un site Web, vous devrez obtenir un extrait de code JavaScript de Google que vous intégrerez dans l’en-tête de chaque page de votre site dont vous souhaitez suivre l’activité des utilisateurs.

Obtenez l’extrait de code sur www.google.com/analytics et, dans l’onglet Admin, sélectionnez “Créer un nouveau compte” dans le menu déroulant de la colonne du compte sur la gauche. .

Suivre les pages appelées par AJAX et le contenu non-html #

Pour suivre ce que l’on appelle les “pages vues virtuelles”, utilisez la méthode ga('send') juste après votre requête asynchrone :

Syntaxe : ga('send', 'pageview', 'chemin vers votre page virtuelle');

Exemple (lien simple) :

<a href="http://example.com/my.pdf"
   onClick="ga('send', 'pageview', '/virtual/my.pdf');">Download PDF</a>

Exemple (JQuery AJAX) :

$.ajax({
    url: '/ajax-url/file.json',
    data: {page: 4},
    success: function(data) {
        ga('send', 'pageview', '/ajax-url/file.json');
        console.log("Got response",data); 
    },
    dataType: 'json',
    method: 'GET'
});

Sources:

  • http://stackoverflow.com/questions/24199037/how-do-i-get-google-analytics-to-track-pages-called-by-ajax/24199142#24199142 - [Suivi des pages vues virtuelles – developers.google.com][1]

[1] : https://developers.google.com/analytics/devguides/collection/analyticsjs/single-page-applications#tracking_virtual_pageviews

Extrait de code de suivi asynchrone alternatif #

Alors que l’extrait de code de suivi JavaScript décrit ci-dessus garantit que le script sera chargé et exécuté de manière asynchrone sur tous les navigateurs, il présente l’inconvénient de ne pas permettre aux navigateurs modernes de précharger le script.

L’extrait de code de suivi asynchrone alternatif ci-dessous ajoute la prise en charge du préchargement, ce qui améliorera légèrement les performances des navigateurs modernes, mais peut dégrader le chargement et l’exécution synchrones sur IE 9 et les navigateurs mobiles plus anciens qui ne reconnaissent pas l’attribut de script asynchrone. N’utilisez cet extrait de suivi que si vos visiteurs utilisent principalement des navigateurs modernes pour accéder à votre site.

    <!-- Google Analytics -->
<script>
window.ga=window.ga||function(){(ga.q=ga.q||[]).push(arguments)};ga.l=+new Date;
ga('create', 'UA-XXXXX-Y', 'auto');
ga('send', 'pageview');
</script>
<script async src='https://www.google-analytics.com/analytics.js'></script>
<!-- End Google Analytics -->

Utiliser des plugins #

Les plugins sont des scripts qui améliorent la fonctionnalité d’analytics.js pour aider à mesurer l’interaction de l’utilisateur. Les plug-ins sont généralement spécifiques à un ensemble de fonctionnalités qui peuvent ne pas être requises par tous les utilisateurs de Google Analytics, telles que le commerce électronique ou le suivi inter-domaines, et ne sont donc pas inclus par défaut dans analytics.js.

Ce guide explique comment exiger et utiliser les plugins analytics.js.

La commande require prend le nom d’un plugin et l’enregistre pour l’utiliser avec la file d’attente de commandes ga(). Si le plugin accepte les options de configuration, ces options peuvent être passées comme argument final à la commande require.

Voici la signature complète de la commande require :

ga('[trackerName.]require', pluginName, [pluginOptions]);

Par exemple, voici comment vous auriez besoin du plugin Enhanced Ecommerce pour une utilisation avec le tracker par défaut :

ga('require', 'ec');

Et voici comment vous auriez besoin du plugin Display Features pour un tracker nommé “myTracker” et passer une option de configuration qui remplace la valeur par défaut du nom du cookie :

ga('myTracker.require', 'displayfeatures', {
  cookieName: 'display_features_cookie'
});

Quelles données l’extrait de suivi capture-t-il ? #

Lorsque vous ajoutez l’un de ces extraits de suivi à votre site Web, vous envoyez une page vue pour chaque page que vos utilisateurs visitent. Google Analytics traite ces données et peut en déduire un grand nombre d’informations, notamment :

Le temps total qu’un utilisateur passe sur votre site. Le temps qu’un utilisateur passe sur chaque page et dans quel ordre ces pages ont été visitées. Quels liens internes ont été cliqués (basés sur l’URL de la prochaine page vue). De plus, l’adresse IP, la chaîne de l’agent utilisateur et l’inspection initiale de la page que fait analytics.js lors de la création d’un nouveau tracker sont utilisées pour déterminer les éléments suivants :

L’emplacement géographique de l’utilisateur. Quel navigateur et quel système d’exploitation sont utilisés. Taille de l’écran et si Flash ou Java est installé. Le site référent.
