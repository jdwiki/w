Paramètres de base de l’auto-complétion et des suggestions #

Notepad++ fournit 2 types de fonctionnalités pour la saisie semi-automatique et les suggestions :

  • Auto-complétion qui lit le fichier ouvert et propose des suggestions de mots et/ou de fonctions dans le fichier
  • Suggestion avec les arguments des fonctions (spécifiques au langage)

Pour l’activer, vous devez modifier un paramètre.

  1. Allez dans Paramètres > Préférences... > Auto-complétion 2. Cochez la case “Activer la saisie semi-automatique sur chaque entrée”.
  2. Sélectionnez le type d’achèvement souhaité :
    • words only
    • name of functions only
    • both of them
  3. Facultativement, cochez “Indice de paramètre de fonction sur l’entrée” pour afficher les paramètres lors de la saisie d’un nom de fonction (cf. image ci-dessous)

[![Aide-mémoire][1]][1]

Pour l’utiliser, commencez à taper un mot ou une fonction et après le nombre de caractères que vous avez spécifié, les suggestions apparaîtront :

[![Suggestions][2]][2]

Une documentation sur l’auto-complétion est disponible ici : http://docs.notepad-plus-plus.org/index.php/Auto_Completion

Personnalisation de la suggestion de langue (indice de paramètre de fonction) #

Les conseils de paramètres peuvent être personnalisés par l’utilisateur comme indiqué dans ce lien : http://docs.notepad-plus-plus.org/index.php/Auto_Completion#How_to_create_keyword_auto-completion_definition_files

Comment créer des fichiers de définition d’auto-complétion de mots-clés

Depuis la version 5.0, Notepad ++ prend en charge ce que l’on appelle les Calltips et a introduit une nouvelle façon de stocker les données de saisie semi-automatique. Maintenant, tout est stocké au format XML, ce qui permet une extension facile des fonctionnalités. Ce faisant, les données de saisie semi-automatique et d’info-bulle sont combinées dans un seul fichier. Les anciens fichiers de texte brut .api ne sont plus utilisés par Notepad ++ et peuvent être supprimés en toute sécurité s’ils sont présents.

Vous pouvez choisir le type d’auto-complétion que vous souhaitez avoir, depuis Paramètres -> Préférences -> Auto-complétion -> Activer l'auto-complétion sur chaque entrée : mots du document actuel, fonctions de la langue actuelle, ou les deux .

Les fichiers de saisie semi-automatique se trouvent dans le dossier “plugins\APIs”, qui se trouve dans le dossier d’installation de Notepad++, le plus souvent C:\Program Files\Notepad++.

La syntaxe des fichiers de saisie semi-automatique est simple, mais a quelques règles, surtout une syntaxe correcte et un tri approprié. Si la syntaxe est incorrecte, le fichier XML ne se chargera pas et la saisie semi-automatique sera désactivée. Une description plus formelle peut être trouvée dans [Modification des fichiers d’auto-complétion][1].

Un tri incorrect (voir ci-dessous) peut entraîner un comportement erratique de la fonction de saisie semi-automatique, l’amenant à échouer sur certains mots.

Le jeu de caractères de base utilisé pour reconnaître les mots-clés est composé de lettres (c’est-à-dire a-z, A-Z, 0-9 chiffres et underscore). […]

<?xml version="1.0" encoding="Windows-1252" ?>
<NotepadPlus>
   <AutoComplete language="C++">
       <Environment ignoreCase="no" startFunc="(" stopFunc=")" paramSeparator="," terminal=";" additionalWordChar = "."/>
       <KeyWord name="abs" func="yes">
           <Overload retVal="int" descr="Returns absolute value of given integer">
               <Param name="int number" />
           </Overload>
       </KeyWord>
   </AutoComplete>
</NotepadPlus>

Un petit exemple de la façon dont le fichier XML est construit est donné ci-dessus. NotepadPlus, AutoComplete et Environment sont des éléments singleton, il ne devrait y en avoir qu’un seul de chaque, et tous devraient être présents pour l’exactitude, bien qu’il soit permis de supprimer l’élément <Environment>. Ainsi, toutes les valeurs par défaut seront celles données dans l’exemple ci-dessus.

Pour les mots-clés qui ne sont pas des fonctions, la balise Keyword se ferme automatiquement et n’a que l’attribut name. Pour indiquer qu’un mot-clé peut être affiché dans une info-bulle, ajoutez l’attribut func avec la valeur yes. Dans ce cas, la balise Keyword est un nœud et contient d’autres balises.

Ensuite, pour chaque surcharge de la fonction, un élément Overload doit être ajouté, qui spécifie le comportement et les paramètres de la fonction. Une fonction doit avoir au moins une surcharge ou elle ne sera pas affichée en tant qu’info-bulle. L’attribut retVal doit être présent et spécifie le type de la valeur de retour, mais l’attribut descr est facultatif et décrit le comportement des fonctions, comme un commentaire.

Vous pouvez ajouter des retours à la ligne dans la description si vous le souhaitez. Pour chaque paramètre pris par la fonction, un élément Param peut être ajouté. L’attribut name doit être présent et spécifie le type des paramètres et/ou tout nom de paramètre.

Dans l’élément AutoComplete, vous pouvez ajouter l’attribut language mais il n’est pas utilisé par Notepad ++, vous pouvez l’ajouter pour être complet si vous le souhaitez et vous pouvez prendre n’importe quelle chaîne que vous voulez.
