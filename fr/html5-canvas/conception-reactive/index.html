Création d’un canevas pleine page responsive #

Code de démarrage pour créer et supprimer un canevas pleine page qui répond aux événements de redimensionnement via javascript.

var canvas;    // Global canvas reference
var ctx;       // Global 2D context reference
// Creates a canvas
function createCanvas () {                
    const canvas = document.createElement("canvas"); 
    canvas.style.position = "absolute"; // Set the style 
    canvas.style.left     = "0px";      // Position in top left
    canvas.style.top      = "0px";
    canvas.style.zIndex   = 1;        
    document.body.appendChild(canvas);  // Add to document
    return canvas;
}
// Resizes canvas. Will create a canvas if it does not exist
function sizeCanvas () {                
    if (canvas === undefined) {         // Check for global canvas reference
        canvas = createCanvas();        // Create a new canvas element
        ctx = canvas.getContext("2d");  // Get the 2D context
    }
    canvas.width  = innerWidth;         // Set the canvas resolution to fill the page
    canvas.height = innerHeight;        
}
// Removes the canvas
function removeCanvas () {
    if (canvas !== undefined) {              // Make sure there is something to remove
        removeEventListener("resize", sizeCanvas); // Remove resize event
        document.body.removeChild(canvas);   // Remove the canvas from the DOM
        ctx = undefined;                     // Dereference the context
        canvas = undefined;                  // Dereference the canvas
     }
}

// Add the resize listener
addEventListener("resize", sizeCanvas); 
// Call sizeCanvas to create and set the canvas resolution
sizeCanvas();
// ctx and canvas are now available for use.

Si vous n’avez plus besoin du canevas, vous pouvez le supprimer en appelant removeCanvas()

Coordonnées de la souris après redimensionnement (ou défilement) #

Les applications de canevas dépendent souvent fortement de l’interaction de l’utilisateur avec la souris, mais lorsque la fenêtre est redimensionnée, les coordonnées d’événement de la souris sur lesquelles canevas s’appuie sont probablement modifiées car le redimensionnement entraîne le décalage du canevas dans une position différente par rapport à la fenêtre. Ainsi, la conception réactive nécessite que la position de décalage du canevas soit recalculée lorsque la fenêtre est redimensionnée - et également recalculée lorsque la fenêtre défile.

Ce code écoute les événements de redimensionnement de fenêtre et recalcule les décalages utilisés dans les gestionnaires d’événements de souris :

// variables holding the current canvas offset position
//    relative to the window
var offsetX,offsetY;

// a function to recalculate the canvas offsets
function reOffset(){
    var BB=canvas.getBoundingClientRect();
    offsetX=BB.left;
    offsetY=BB.top;        
}

// listen for window resizing (and scrolling) events
//     and then recalculate the canvas offsets
window.onscroll=function(e){ reOffset(); }
window.onresize=function(e){ reOffset(); }

// example usage of the offsets in a mouse handler
function handleMouseUp(e){
    // use offsetX & offsetY to get the correct mouse position
    mouseX=parseInt(e.clientX-offsetX);
    mouseY=parseInt(e.clientY-offsetY);
    // ...
}

Animations de toile réactives sans événements de redimensionnement. #

Les événements de redimensionnement de la fenêtre peuvent se déclencher en réponse au mouvement du périphérique d’entrée de l’utilisateur. Lorsque vous redimensionnez un canevas, il est automatiquement effacé et vous êtes obligé de restituer le contenu. Pour les animations, vous le faites à chaque image via la fonction de boucle principale appelée par requestAnimationFrame qui fait de son mieux pour maintenir le rendu synchronisé avec le matériel d’affichage.

Le problème avec l’événement de redimensionnement est que lorsque la souris est utilisée pour redimensionner la fenêtre, les événements peuvent être déclenchés plusieurs fois plus rapidement que le taux standard de 60 ips du navigateur. Lorsque l’événement de redimensionnement se termine, le tampon arrière du canevas est présenté au DOM désynchronisé avec le périphérique d’affichage, ce qui peut provoquer un cisaillement et d’autres effets négatifs. Il y a aussi beaucoup d’allocation et de libération de mémoire inutiles qui peuvent avoir un impact supplémentaire sur l’animation lorsque GC nettoie quelque temps après.

Événement de redimensionnement annulé #

Une façon courante de gérer les taux de déclenchement élevés de l’événement de redimensionnement consiste à faire rebondir l’événement de redimensionnement.

 // Assume canvas is in scope
 addEventListener.("resize", debouncedResize );

 // debounce timeout handle
 var debounceTimeoutHandle;

 // The debounce time in ms (1/1000th second)
 const DEBOUNCE_TIME = 100; 

 // Resize function 
 function debouncedResize () { 
     clearTimeout(debounceTimeoutHandle);  // Clears any pending debounce events

     // Schedule a canvas resize 
     debounceTimeoutHandle = setTimeout(resizeCanvas, DEBOUNCE_TIME);
 }

 // canvas resize function
 function resizeCanvas () { ... resize and redraw ... }

L’exemple ci-dessus retarde le redimensionnement du canevas jusqu’à 100 ms après l’événement de redimensionnement. Si, à ce moment-là, d’autres événements de redimensionnement sont déclenchés, le délai d’attente de redimensionnement existant est annulé et un nouveau programmé. Cela consomme effectivement la plupart des événements de redimensionnement.

Il a encore quelques problèmes, le plus notable étant le délai entre le redimensionnement et l’affichage du canevas redimensionné. La réduction du temps anti-rebond améliore cela, mais le redimensionnement n’est toujours pas synchronisé avec le périphérique d’affichage. Vous avez également toujours le rendu de la boucle principale de l’animation sur un canevas mal ajusté.

Plus de code peut réduire les problèmes ! Plus de code crée également ses propres nouveaux problèmes.

Simple et le meilleur redimensionnement #

Après avoir essayé de nombreuses façons différentes de lisser le redimensionnement de la toile, de l’absurde complexe à l’ignorance du problème (qui s’en soucie de toute façon ?), je me suis rabattu sur un ami fidèle.

K.I.S.S. est quelque chose dont la plupart des programmeurs devraient être conscients ((Keep It Ssimple Stupide) Le stupide se réfère à moi pour ne pas avoir pensé il y a des années. ) et il s’avère que la meilleure solution est la plus simple de toutes.

Redimensionnez simplement le canevas à partir de la boucle d’animation principale. Il reste synchronisé avec le périphérique d’affichage, il n’y a pas de rendu inutile et la gestion des ressources est au minimum possible tout en maintenant une fréquence d’images complète. Vous n’avez pas non plus besoin d’ajouter un événement de redimensionnement à la fenêtre ou des fonctions de redimensionnement supplémentaires.

Vous ajoutez le redimensionnement là où vous effaceriez normalement le canevas en vérifiant si la taille du canevas correspond à la taille de la fenêtre. Sinon, redimensionnez-le.

// Assumes canvas element is in scope as canvas

// Standard main loop function callback from requestAnimationFrame
function mainLoop(time) {

    // Check if the canvas size matches the window size
    if (canvas.width !== innerWidth || canvas.height !== innerHeight) {
        canvas.width = innerWidth;    // resize canvas
        canvas.height = innerHeight;  // also clears the canvas
    } else {
        ctx.clearRect(0, 0, canvas.width, canvas.height); // clear if not resized
    }

    // Animation code as normal.



    requestAnimationFrame(mainLoop);
}
