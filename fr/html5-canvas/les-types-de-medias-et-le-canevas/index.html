Ce sujet est de couvrir les différents types de médias et comment ils peuvent être utilisés avec le canevas dans l’interface 2D.

Les types de médias ont des catégories génériques et spécifiques au format

Types de médias

  • Animations
  • Vidéos
  • Images -Images HD
  • Image vectorielle
  • Images animées

Formats multimédias

  • Jpg/Jpeg
  • png
  • GIF -SVG -M-JPEG
  • Webm
  • Webp

Images

Il existe une grande variété de formats d’image pris en charge par les navigateurs, bien qu’aucun navigateur ne les prenne tous en charge. Si vous avez des formats d’image particuliers que vous souhaitez utiliser [Navigateurs Wiki et formats d’image pris en charge] [1] fournit un bon aperçu.

La meilleure prise en charge concerne les 3 principaux formats, “jpeg”, “png” et “gif” avec tous les principaux navigateurs prenant en charge.

JPEG

Les images JPEG conviennent mieux aux photos et aux images de type photo. Ils ne se prêtent pas bien aux graphiques, aux diagrammes et au texte. Les images JPEG ne prennent pas en charge la transparence.

Canvas peut générer des images JPEG via canvas.toDataURL et canvas.toBlob et fournit un paramètre de qualité. Comme JPEG ne prend pas en charge la transparence, tous les pixels transparents seront mélangés avec du noir pour la sortie finale JPG. L’image résultante ne sera pas une copie parfaite de la toile.

[JPEG sur wikipedia][2]

PNG

Les images PNG sont des images de la plus haute qualité et peuvent également inclure un canal alpha pour les pixels transparents. Les données d’image sont compressées mais ne produisent pas d’artefacts comme les images JPG.

En raison de la compression sans perte et de la prise en charge du canal alpha, les PNG sont utilisés pour les jeux, les images de composants d’interface utilisateur, les graphiques, les diagrammes et le texte. Lorsque vous les utilisez pour des photos et des images de type photo, leur taille de fichier peut être beaucoup plus grande que celle de JPEG. .

Le format PNG fournit également une prise en charge de l’animation bien que la prise en charge du navigateur soit limitée, et l’accès à l’animation à utiliser sur le canevas ne peut se faire que via les API et bibliothèques Javascript

Le canevas peut être utilisé pour encoder des images PNG via canvas.toDataURL et canvas.toBlob bien que le format de sortie soit limité à RGBA 32 bits compressé. Le PNG fournira une copie parfaite au pixel près de la toile.

[PNG sur wikipédia][3]

GIF

Les GIF sont utilisés pour de courtes animations, mais peuvent également être utilisés pour fournir des graphiques, des diagrammes et du texte de haute qualité comme des images. Les GIF ont une prise en charge des couleurs très limitée avec un maximum de 256 couleurs par image. Avec le traitement d’image cleaver, les images gif peuvent produire des résultats étonnamment bons, en particulier lorsqu’elles sont animées. Les gifs offrent également de la transparence bien que cela soit limité à activé ou désactivé

Comme avec PNG, les animations GIF ne sont pas directement accessibles pour une utilisation sur le canevas et vous aurez besoin d’une API ou d’une bibliothèque Javascript pour y accéder. Le GIF ne peut pas être enregistré via le canevas et nécessitera une API ou une bibliothèque pour le faire.

[GIF sur wikipédia]

Chargement et lecture de base d’une vidéo sur le canevas. #

Le canevas peut être utilisé pour afficher des vidéos provenant de diverses sources. Cet exemple montre comment charger une vidéo en tant que ressource de fichier, l’afficher et ajouter un simple clic sur la bascule lecture/pause de l’écran.

Cette question à réponse automatique de stackoverflow [Comment afficher une vidéo à l’aide de la balise de canevas HTML5] [1] montre l’exemple de code suivant en action.

Juste une image #

Une vidéo n’est qu’une image en ce qui concerne la toile. Vous pouvez le dessiner comme n’importe quelle image. La différence étant que la vidéo peut jouer et a du son.

Obtenez le canevas et la configuration de base #

// It is assumed you know how to add a canvas and correctly size it.
var canvas = document.getElementById("myCanvas"); // get the canvas from the page
var ctx = canvas.getContext("2d");
var videoContainer; // object to hold video and associated info

Création et chargement de la vidéo #

var video = document.createElement("video"); // create a video element
video.src = "urlOffVideo.webm"; 
// the video will now begin to load.
// As some additional info is needed we will place the video in a
// containing object for convenience
video.autoPlay = false; // ensure that the video does not auto play
video.loop = true; // set the video to loop.
videoContainer = {  // we will add properties as needed
     video : video,
     ready : false,   
};

Contrairement aux éléments d’images, les vidéos n’ont pas besoin d’être entièrement chargées pour être affichées sur le canevas. Les vidéos fournissent également une foule d’événements supplémentaires qui peuvent être utilisés pour surveiller l’état de la vidéo.

Dans ce cas, nous souhaitons savoir quand la vidéo est prête à être lue. oncanplay signifie qu’une partie suffisante de la vidéo a été chargée pour en lire une partie, mais qu’il n’y en a peut-être pas assez pour la lecture jusqu’à la fin.

video.oncanplay = readyToPlayVideo; // set the event to the play function that 
                                  // can be found below

Alternativement, vous pouvez utiliser oncanplaythrough qui se déclenchera lorsqu’une partie suffisante de la vidéo sera chargée afin qu’elle puisse être lue jusqu’à la fin.

video.oncanplaythrough = readyToPlayVideo; // set the event to the play function that
                                         // can be found below

N’utilisez qu’un seul des événements canPlay, pas les deux.

L’événement peut jouer (équivalent à image onload) #

function readyToPlayVideo(event){ // this is a referance to the video
    // the video may not match the canvas size so find a scale to fit
    videoContainer.scale = Math.min(
                         canvas.width / this.videoWidth, 
                         canvas.height / this.videoHeight); 
    videoContainer.ready = true;
    // the video can be played so hand it off to the display function
    requestAnimationFrame(undateCanvas);
}

Affichage #

La vidéo ne se jouera pas sur la toile. Vous devez le dessiner pour chaque nouvelle image. Comme il est difficile de connaître la fréquence d’images exacte et le moment où ils se produisent, la meilleure approche consiste à afficher la vidéo comme si elle fonctionnait à 60 images par seconde. Si la fréquence d’images est inférieure, w rend simplement la même image deux fois. Si la fréquence d’images est plus élevée, il n’y a rien à faire pour voir les images supplémentaires, nous les ignorons donc.

L’élément vidéo n’est qu’un élément d’image et peut être dessiné comme n’importe quelle image, vous pouvez mettre à l’échelle, faire pivoter, faire un panoramique de la vidéo, la mettre en miroir, la décolorer, la couper et n’afficher que des parties, la dessiner deux fois la deuxième fois avec un mode composite global pour ajouter des effets tels que lighten, screen, etc.

function updateCanvas(){
    ctx.clearRect(0,0,canvas.width,canvas.height); // Though not always needed 
                                                     // you may get bad pixels from 
                                                     // previous videos so clear to be
                                                     // safe
    // only draw if loaded and ready
    if(videoContainer !== undefined && videoContainer.ready){ 
        // find the top left of the video on the canvas
        var scale = videoContainer.scale;
        var vidH = videoContainer.video.videoHeight;
        var vidW = videoContainer.video.videoWidth;
        var top = canvas.height / 2 - (vidH /2 ) * scale;
        var left = canvas.width / 2 - (vidW /2 ) * scale;
        // now just draw the video the correct size
        ctx.drawImage(videoContainer.video, left, top, vidW * scale, vidH * scale);
        if(videoContainer.video.paused){ // if not playing show the paused screen 
            drawPayIcon();
        }
    }
    // all done for display 
    // request the next frame in 1/60th of a second
    requestAnimationFrame(updateCanvas);
}

Contrôle de pause de lecture de base #

Maintenant que la vidéo est chargée et affichée, tout ce dont nous avons besoin est le contrôle de lecture. Nous allons le faire comme un clic bascule jouer sur l’écran. Lorsque la vidéo est en cours de lecture et que l’utilisateur clique, la vidéo est mise en pause. Lorsqu’il est en pause, le clic reprend la lecture. Nous allons ajouter une fonction pour assombrir la vidéo et dessiner une icône de lecture (triangle)

function drawPayIcon(){
     ctx.fillStyle = "black";  // darken display
     ctx.globalAlpha = 0.5;
     ctx.fillRect(0,0,canvas.width,canvas.height);
     ctx.fillStyle = "#DDD"; // colour of play icon
     ctx.globalAlpha = 0.75; // partly transparent
     ctx.beginPath(); // create the path for the icon
     var size = (canvas.height / 2) * 0.5;  // the size of the icon
     ctx.moveTo(canvas.width/2 + size/2, canvas.height / 2); // start at the pointy end
     ctx.lineTo(canvas.width/2 - size/2, canvas.height / 2 + size);
     ctx.lineTo(canvas.width/2 - size/2, canvas.height / 2 - size);
     ctx.closePath();
     ctx.fill();
     ctx.globalAlpha = 1; // restore alpha
}

Maintenant, l’événement de pause de lecture #

function playPauseClick(){
     if(videoContainer !== undefined && videoContainer.ready){
          if(videoContainer.video.paused){                                 
                videoContainer.video.play();
          }else{
                videoContainer.video.pause();
          }
     }
}
// register the event
canvas.addEventListener("click",playPauseClick);

Sommaire #

La lecture d’une vidéo est très facile à l’aide de la toile, l’ajout d’effets en temps réel est également facile. Il existe cependant certaines limitations sur les formats, la façon dont vous pouvez jouer et rechercher. MDN HTMLMediaElement est l’endroit où obtenir la référence complète à l’objet vidéo.

Une fois l’image dessinée sur le canevas, vous pouvez utiliser ctx.getImageData pour accéder aux pixels qu’elle contient. Ou vous pouvez utiliser canvas.toDataURL pour prendre une photo et la télécharger. (Uniquement si la vidéo provient d’une source fiable et n’altère pas la toile).

Notez que si la vidéo a du son, la lecture jouera également le son.

image.onload = function(){

}

Créer une image

Il existe plusieurs façons de créer une image

  • nouvelle image()
  • document.createElement("img")
  • <img src = 'imageUrl' id='myImage'> Faisant partie du corps HTML et récupéré avec document.getElementById('myImage')

L’image est un HTMLImageElement

Propriété Image.src

L’image src peut être n’importe quelle URL d’image valide ou dataURL encodée. Consultez les remarques de cette rubrique pour plus d’informations sur les formats d’image et leur prise en charge.

  • image.src = "http://mon.domaine.com/images/monImage.jpg"
  • image.src = "data:image/gif;base64,R0lGODlhAQABAIAAAAUEBAAAACwAAAAAAQABAAACAkQBADs=" *

*Le dataURL est une image gif de 1 par 1 pixel contenant du noir

Remarques sur le chargement et les erreurs

L’image commencera à se charger lorsque sa propriété src sera définie. Le chargement est synchrone mais l’événement onload ne sera pas appelé tant que la fonction ou le code ne sera pas sorti/retourné.

Si vous obtenez une image de la page (par exemple document.getElementById("myImage")) et que son src est défini, il se peut qu’elle ait été chargée ou non. Vous pouvez vérifier l’état de l’image avec HTMLImageElement.complete qui sera true si complet. Cela ne signifie pas que l’image a été chargée, cela signifie qu’elle a soit

  • chargé
  • Il y avait une erreur
  • la propriété src n’a pas été définie et est égale à la chaîne vide ""

Si l’image provient d’une source non fiable et peut ne pas être accessible pour diverses raisons, elle générera un événement d’erreur. Lorsque cela se produit, l’image sera dans un état cassé. Si vous essayez ensuite de le dessiner sur la toile, il lancera l’erreur suivante

Uncaught DOMException: Failed to execute 'drawImage' on 'CanvasRenderingContext2D': The HTMLImageElement provided is in the 'broken' state.

En fournissant l’événement image.onerror = myImgErrorHandler, vous pouvez prendre les mesures appropriées pour éviter les erreurs.

## Dessiner une image svg Pour dessiner une image SVG vectorielle, l'opération n'est pas différente d'une image raster : Vous devez d'abord charger votre image SVG dans un élément HTMLImage, puis utiliser la méthode `drawImage()`. var image = new Image(); image.onload = function(){ ctx.drawImage(this, 0,0); } image.src = "someFile.SVG"; Les images SVG présentent certains avantages par rapport aux images raster, car vous ne perdrez pas en qualité, quelle que soit l'échelle à laquelle vous les dessinerez sur votre canevas. Mais attention, cela peut aussi être un peu plus lent que de dessiner une image raster. Cependant, les images SVG sont soumises à plus de restrictions que les images raster. - **Pour des raisons de sécurité, aucun contenu externe ne peut être chargé à partir d'une image SVG référencée dans un HTMLImageElement(``)** Pas de feuille de style externe, pas d'image externe référencée dans les éléments SVGImage (``), pas de filtre externe ou d'élément lié par l'attribut `xlink:href` (``) ou la méthode d'attribut funcIRI (`url()`) etc. De plus, les feuilles de style ajoutées dans le document principal n'auront aucun effet sur le document SVG une fois référencé dans un élément HTMLImage. Enfin, aucun script ne sera exécuté à l'intérieur de l'image SVG. **Solution :** Vous devrez ajouter tous les éléments externes à l'intérieur du SVG lui-même avant de faire référence à l'élément HTMLImage. (pour les images ou les polices, vous devez ajouter une version dataURI de vos ressources externes). - **L'élément racine (`
