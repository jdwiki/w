Syntaxe #

  • pour le corps de {clauses}
  • pour {clauses} corps de rendement
  • pour le corps (des clauses)
  • pour (clauses) corps de rendement

Paramètres #

Paramètre Détails
pour Mot-clé requis pour utiliser une boucle for/comprehension
clauses L’itération et les filtres sur lesquels for fonctionne.
rendement Utilisez-le si vous souhaitez créer ou “céder” une collection. L’utilisation de yield fera que le type de retour de for sera une collection au lieu de Unit.
corps Corps de l’expression for, exécuté à chaque itération.

Monadique pour les compréhensions #

Si vous avez plusieurs objets de types [monadiques][1], nous pouvons réaliser des combinaisons des valeurs en utilisant un ‘pour la compréhension’ :

for {
   x <- Option(1)
   y <- Option("b")
   z <- List(3, 4)
} {
    // Now we can use the x, y, z variables
    println(x, y, z)
    x  // the last expression is *not* the output of the block in this case!
}

// This prints
// (1, "b", 3)
// (1, "b", 4)

Le type de retour de ce bloc est Unit.

Si les objets sont du même type monadique M (par exemple Option), alors l’utilisation de yield renverra un objet de type M au lieu de Unit.

val a = for {
   x <- Option(1)
   y <- Option("b")
} yield {
    // Now we can use the x, y variables
    println(x, y)
    // whatever is at the end of the block is the output
    (7 * x, y)
}

// This prints:
// (1, "b")
// The val `a` is set:
// a: Option[(Int, String)] = Some((7,b))

Notez que le mot-clé yield ne peut pas être utilisé dans l’exemple original, où il y a un mélange de types monadiques (Option et List). Essayer de le faire produira une erreur d’incompatibilité de type au moment de la compilation.

[1] : https://www.wikiod.com/fr/scala/monades

Boucle for imbriquée #

Cela montre comment vous pouvez itérer sur plusieurs variables :

for {
  x <- 1 to 2
  y <- 'a' to 'd'
} println("(" + x + "," + y + ")")

(Notez que to ici est une méthode d’opérateur infixe qui renvoie une [plage inclusive][1]. Voir la définition [ici][2].)

Cela crée la sortie :

(1,a)
(1,b)
(1,c)
(1,d)
(2,a)
(2,b)
(2,c)
(2,d)

Notez qu’il s’agit d’une expression équivalente, utilisant des parenthèses au lieu de crochets :

for (
  x <- 1 to 2
  y <- 'a' to 'd'
) println("(" + x + "," + y + ")")

Afin d’obtenir toutes les combinaisons dans un seul vecteur, nous pouvons “rendre” le résultat et le définir sur une “val” :

val a = for {
  x <- 1 to 2
  y <- 'a' to 'd'
} yield "(%s,%s)".format(x, y)
// a: scala.collection.immutable.IndexedSeq[String] = Vector((1,a), (1,b), (1,c), (1,d), (2,a), (2,b), (2,c), (2,d))

[1] : http://www.scala-lang.org/api/2.10.3/#scala.collection.immutable.NumericRange$$Inclusive [2] : http://www.scala-lang.org/api/2.10.3/#scala.Char

Itérer dans les collections à l’aide d’une boucle For #

Ceci montre comment imprimer chaque élément d’une carte

val map = Map(1 -> "a", 2 -> "b")
for (number <- map) println(number) // prints (1,a), (2,b)
for ((key, value) <- map) println(value) // prints a, b

Ceci montre comment imprimer chaque élément d’une liste

val list = List(1,2,3)
for(number <- list) println(number) // prints 1, 2, 3

Désucrer pour comprendre #

les compréhensions “pour” en Scala ne sont que [sucre syntaxique] [1]. Ces compréhensions sont implémentées à l’aide des méthodes withFilter, foreach, flatMap et map de leurs types de sujet. Pour cette raison, seuls les types pour lesquels ces méthodes sont définies peuvent être utilisés dans une compréhension “for”.

Une compréhension pour de la forme suivante, avec des motifs pN, des générateurs gN et des conditions cN :

for(p0 <- x0 if g0; p1 <- g1 if c1) { ??? }

… supprimera le sucre des appels imbriqués en utilisant withFilter et foreach :

g0.withFilter({ case p0 => c0  case _ => false }).foreach({
  case p0 => g1.withFilter({ case p1 => c1  case _ => false }).foreach({
    case p1 => ???
  })
})

Alors qu’une expression for/yield de la forme suivante :

for(p0 <- g0 if c0; p1 <- g1 if c1) yield ???

… supprimera le sucre des appels imbriqués en utilisant withFilter et flatMap ou map :

g0.withFilter({ case p0 => c0  case _ => false }).flatMap({
  case p0 => g1.withFilter({ case p1 => c1  case _ => false }).map({
    case p1 => ???
  })
})

(Notez que map est utilisé dans la compréhension la plus profonde et flatMap est utilisé dans chaque compréhension externe.)

Une compréhension « for » peut être appliquée à tout type implémentant les méthodes requises par la représentation désucrée. Il n’y a aucune restriction sur les types de retour de ces méthodes, tant qu’elles sont composables.

[1] : https://en.wikipedia.org/wiki/Syntactic_sugar

Boucle For de base #

for (x <- 1 to 10)
  println("Iteration number " + x)

Cela illustre l’itération d’une variable, x, de 1 à 10 et de faire quelque chose avec cette valeur. Le type de retour de cette compréhension for est Unit.

Basique pour la compréhension #

Ceci illustre un filtre sur une boucle for et l’utilisation de “yield” pour créer une “compréhension de séquence”:

for ( x <- 1 to 10 if x % 2 == 0)
  yield x

La sortie pour ceci est :

scala.collection.immutable.IndexedSeq[Int] = Vector(2, 4, 6, 8, 10)

Un pour la compréhension est utile lorsque vous devez créer une nouvelle collection basée sur l’itération et ses filtres.
