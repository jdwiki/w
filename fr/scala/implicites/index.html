Syntaxe #

  • valeur implicite x : T = ???

Les classes implicites permettent d’ajouter des méthodes personnalisées aux types existants, sans avoir à modifier leur code, enrichissant ainsi les types sans avoir besoin de contrôler le code.

L’utilisation de types implicites pour enrichir une classe existante est souvent appelée modèle « enrichir ma bibliothèque ».

Restrictions sur les classes implicites

  1. Les classes implicites ne peuvent exister qu’au sein d’une autre classe, objet ou trait.
  2. Les classes implicites ne peuvent avoir qu’un seul paramètre de constructeur principal non implicite.
  3. Il ne peut pas y avoir d’autre définition d’objet, de classe, de trait ou de membre de classe dans la même portée qui porte le même nom que la classe implicite.

Classes implicites #

Les classes implicites permettent d’ajouter de nouvelles méthodes aux classes précédemment définies.

La classe String n’a pas de méthode withoutVowels. Cela peut être ajouté comme ceci:

object StringUtil {
  implicit class StringEnhancer(str: String) {
    def withoutVowels: String = str.replaceAll("[aeiou]", "")
  }
}

La classe implicite a un paramètre de constructeur unique (str) avec le type que vous souhaitez étendre (String) et contient la méthode que vous souhaitez “ajouter” au type (withoutVowels). Les méthodes nouvellement définies peuvent désormais être utilisées directement sur le type amélioré (lorsque le type amélioré est dans la portée implicite) :

import StringUtil.StringEnhancer // Brings StringEnhancer into implicit scope

println("Hello world".withoutVowels) // Hll wrld

Sous le capot, les classes implicites définissent une [conversion implicite][1] du type amélioré vers la classe implicite, comme ceci :

implicit def toStringEnhancer(str: String): StringEnhancer = new StringEnhancer(str)

Les classes implicites sont souvent définies comme [classes de valeur] [2] pour éviter de créer des objets d’exécution et donc de supprimer la surcharge d’exécution :

implicit class StringEnhancer(val str: String) extends AnyVal {
    /* conversions code here */
}

Avec la définition améliorée ci-dessus, une nouvelle instance de StringEnhancer n’a pas besoin d’être créée à chaque fois que la méthode withoutVowels est invoquée.

[1] : https://www.wikiod.com/fr/scala/implicites#Conversion implicite [2] : http://docs.scala-lang.org/overviews/core/value-classes.html

Paramètres implicites #

Les paramètres implicites peuvent être utiles si un paramètre d’un type doit être défini une fois dans la portée, puis appliqué à toutes les fonctions qui utilisent une valeur de ce type.

Un appel de fonction normal ressemble à ceci :

// import the duration methods
import scala.concurrent.duration._

// a normal method:
def doLongRunningTask(timeout: FiniteDuration): Long = timeout.toMillis

val timeout = 1.second
// timeout: scala.concurrent.duration.FiniteDuration = 1 second

// to call it
doLongRunningTask(timeout) // 1000

Maintenant, disons que nous avons des méthodes qui ont toutes une durée de temporisation, et nous voulons appeler toutes ces méthodes en utilisant la même temporisation. Nous pouvons définir timeout comme une variable implicite.

// import the duration methods
import scala.concurrent.duration._

// dummy methods that use the implicit parameter
def doLongRunningTaskA()(implicit timeout: FiniteDuration): Long = timeout.toMillis
def doLongRunningTaskB()(implicit timeout: FiniteDuration): Long = timeout.toMillis

// we define the value timeout as implicit
implicit val timeout: FiniteDuration = 1.second

// we can now call the functions without passing the timeout parameter
doLongRunningTaskA() // 1000
doLongRunningTaskB() // 1000

La façon dont cela fonctionne est que le compilateur scalac recherche une valeur dans la portée qui est marquée comme implicite et dont le type correspond à celui du paramètre implicite. S’il en trouve un, il l’appliquera comme paramètre implicite.

Notez que cela ne fonctionnera pas si vous définissez deux implicites ou plus de du même type dans le champ d’application.

Pour personnaliser le message d’erreur, utilisez l’annotation implicitNotFound sur le type :

@annotation.implicitNotFound(msg = "Select the proper implicit value for type M[${A}]!")
case class M[A](v: A) {}

def usage[O](implicit x: M[O]): O = x.v

//Does not work because no implicit value is present for type `M[Int]`
//usage[Int]   //Select the proper implicit value for type M[Int]!
implicit val first: M[Int] = M(1)
usage[Int]     //Works when `second` is not in scope
implicit val second: M[Int] = M(2)
//Does not work because more than one implicit values are present for the type `M[Int]`
//usage[Int]   //Select the proper implicit value for type M[Int]!

Un délai d’attente est un cas d’utilisation habituel pour cela, ou par exemple dans [Akka] [1], l’ActorSystem est (la plupart du temps) toujours le même, il est donc généralement passé implicitement. Un autre cas d’utilisation serait la conception de bibliothèques, le plus souvent avec des bibliothèques FP qui reposent sur des classes de types (comme [scalaz][2], [cats][3] ou [rapture][4]).

Il est généralement considéré comme une mauvaise pratique d’utiliser des paramètres implicites avec des types de base comme Int, Long, String etc. car cela créera de la confusion et rendra le code moins lisible.

[1] : http://akka.io/ [2] : https://github.com/scalaz/scalaz [3] : https://github.com/typelevel/cats [4] : https://github.com/propensive/rapture

Conversion implicite #

Une conversion implicite permet au compilateur de convertir automatiquement un objet d’un type en un autre type. Cela permet au code de traiter un objet comme un objet d’un autre type.

case class Foo(i: Int)

// without the implicit
Foo(40) + 2    // compilation-error (type mismatch)

// defines how to turn a Foo into an Int
implicit def fooToInt(foo: Foo): Int = foo.i

// now the Foo is converted to Int automatically when needed
Foo(40) + 2    // 42

La conversion est unidirectionnelle : dans ce cas, vous ne pouvez pas reconvertir 42 en Foo(42). Pour cela, une seconde conversion implicite doit être définie :

implicit def intToFoo(i: Int): Foo = Foo(i)

Notez que c’est le mécanisme par lequel une valeur flottante peut être ajoutée à une valeur entière, par exemple.

Les conversions implicites doivent être utilisées avec parcimonie car elles masquent ce qui se passe. Il est recommandé d’utiliser une conversion explicite via un appel de méthode à moins qu’il y ait un gain de lisibilité tangible à utiliser une conversion implicite.

Il n’y a pas d’impact significatif sur les performances des conversions implicites.

Scala importe automatiquement une variété de conversions implicites dans [scala.Predef][1], y compris toutes les conversions de Java vers Scala et inversement. Ceux-ci sont inclus par défaut dans toute compilation de fichiers.

[1] : http://www.scala-lang.org/api/2.12.x/scala/Predef$.html

Résolution des paramètres implicites à l’aide de ‘implicitly’ #

En supposant une liste de paramètres implicites avec plusieurs paramètres implicites :

case class Example(p1:String, p2:String)(implicit ctx1:SomeCtx1, ctx2:SomeCtx2)

Maintenant, en supposant que l’une des instances implicites n’est pas disponible (SomeCtx1) alors que toutes les autres instances implicites nécessaires sont dans la portée, pour créer une instance de la classe, une instance de SomeCtx1 doit être fournie.

Cela peut être fait tout en préservant l’autre instance implicite dans la portée à l’aide du mot-clé implicitly :

Example("something","somethingElse")(new SomeCtx1(), implicitly[SomeCtx2])

Implicites dans le REPL #

Pour afficher tous les “implicites” dans la portée lors d’une session REPL :

scala> :implicits

Pour inclure également les conversions implicites définies dans Predef.scala :

scala> :implicits -v

Si l’on a une expression et souhaite voir l’effet de toutes les règles de réécriture qui s’y appliquent (y compris les implicites) :

scala> reflect.runtime.universe.reify(expr) // No quotes. reify is a macro operating directly on code.

(Exemple:

scala> import reflect.runtime.universe._
scala> reify(Array("Alice", "Bob", "Eve").mkString(", "))
resX: Expr[String] = Expr[String](Predef.refArrayOps(Array.apply("Alice", "Bob", "Eve")(Predef.implicitly)).mkString(", "))

)
