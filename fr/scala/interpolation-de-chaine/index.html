Cette fonctionnalité existe dans Scala 2.10.0 et supérieur.

Utilisation d’une expression dans les littéraux de chaîne #

Vous pouvez utiliser des accolades pour interpoler des expressions dans des littéraux de chaîne :

def f(x: String) = x + x
val a = "A"

s"${a}"    // "A"
s"${f(a)}" // "AA"

Sans les accolades, scala n’interpolerait que l’identifiant après le $ (dans ce cas f). Puisqu’il n’y a pas de conversion implicite de f en String, il s’agit d’une exception dans cet exemple :

s"$f(a)"  // compile-time error (missing argument list for method f)

Bonjour interpolation de chaîne #

L’interpolateur s permet l’utilisation de variables dans une chaîne.

val name = "Brian"
println(s"Hello $name")

affiche “Hello Brian” sur la console lorsqu’il est exécuté.

Interpolation de chaîne formatée à l’aide de l’interpolateur f #

val num = 42d

Imprimer deux décimales pour num en utilisant f

println(f"$num%2.2f")
42.00

Imprimer num en utilisant la notation scientifique en utilisant e

println(f"$num%e")
4.200000e+01

Imprimer num en hexadécimal avec a

println(f"$num%a")
0x1.5p5

D’autres chaînes de format peuvent être trouvées sur https://docs.oracle.com/javase/6/docs/api/java/util/Formatter.html#detail

Interpolateurs de chaînes comme extracteurs #

Il est également possible d’utiliser la fonction d’interpolation de chaînes de Scala pour créer des extracteurs élaborés (matchers de modèles), comme peut-être le plus connu dans l’[API quasiquotes] [1] des macros Scala.

Étant donné que n"p0${i0}p1" se transforme en new StringContext("p0", "p1").n(i0), il n’est peut-être pas surprenant que la fonctionnalité d’extraction soit activée en fournissant une conversion implicite de StringContext à une classe avec la propriété n d’un type définissant une méthode unapply ou unapplySeq.

À titre d’exemple, considérons l’extracteur suivant qui extrait des segments de chemin en construisant une expression régulière à partir des parties StringContext. Nous pouvons ensuite déléguer la majeure partie du gros du travail à la méthode “unapplySeq” fournie par le résultat [scala.util.matching.Regex][2] :

implicit class PathExtractor(sc: StringContext) {
  object path {
    def unapplySeq(str: String): Option[Seq[String]] =
      sc.parts.map(Regex.quote).mkString("^", "([^/]+)", "$").r.unapplySeq(str)      
  }
}

"/documentation/scala/1629/string-interpolation" match {
  case path"/documentation/${topic}/${id}/${_}" => println(s"$topic, $id")
  case _ => ???
}

Notez que l’objet path pourrait également définir une méthode apply afin de se comporter également comme un interpolateur régulier.

[1] : http://docs.scala-lang.org/overviews/quasiquotes/unlifting [2] : http://www.scala-lang.org/api/current/#scala.util.matching.Regex

Interpolateurs de chaînes personnalisés #

Il est possible de définir des interpolateurs de chaîne personnalisés en plus de ceux intégrés.

my"foo${bar}baz"

Est étendu par le compilateur à :

new scala.StringContext("foo", "baz").my(bar)

scala.StringContext n’a pas de méthode my, elle peut donc être fournie par conversion implicite. Un interpolateur personnalisé avec le même comportement que l’interpolateur s intégré serait alors implémenté comme suit :

implicit class MyInterpolator(sc: StringContext) {
  def my(subs: Any*): String = {
    val pit = sc.parts.iterator
    val sit = subs.iterator
    // Note parts.length == subs.length + 1
    val sb = new java.lang.StringBuilder(pit.next())
    while(sit.hasNext) {
      sb.append(sit.next().toString)
      sb.append(pit.next())          
    }
    sb.toString
  }
}

Et l’interpolation my"foo${bar}baz" donnerait :

new MyInterpolation(new StringContext("foo", "baz")).my(bar)

Notez qu’il n’y a aucune restriction sur les arguments ou le type de retour de la fonction d’interpolation. Cela nous conduit sur un chemin sombre où la syntaxe d’interpolation peut être utilisée de manière créative pour construire des objets arbitraires, comme illustré dans l’exemple suivant :

case class Let(name: Char, value: Int)

implicit class LetInterpolator(sc: StringContext) {
  def let(value: Int): Let = Let(sc.parts(0).charAt(0), value)
}

let"a=${4}" // Let(a, 4)
let"b=${"foo"}" // error: type mismatch
let"c=" // error: not enough arguments for method let: (value: Int)Let

Interpolation de chaîne brute #

Vous pouvez utiliser l’interpolateur raw si vous souhaitez qu’une chaîne soit imprimée telle quelle et sans aucun échappement de littéraux. <pré> println(raw"Hello World In English And French\nAnglais :\tHello World\nFrançais :\t\tBonjour Le Monde")

Avec l’utilisation de l’interpolateur raw, vous devriez voir ce qui suit imprimé dans la console : <pré> Hello World en anglais et en français\nAnglais :\tHello World\nFrançais :\t\tBonjour Le Monde

Sans l’interpolateur raw, \n et \t auraient été échappés. <pré> println(“Hello World In English And French\nAnglais :\tHello World\nFrançais :\t\tBonjour Le Monde”) Impressions : <pré> Hello World en anglais et en français Anglais : Bonjour le monde French: Bonjour Le Monde
