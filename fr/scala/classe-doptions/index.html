Syntaxe #

  • la classe Some[+T](valeur : T) étend Option[T]

  • objet Aucun étend Option[Rien]

  • Option[T](valeur : T)

    Constructor to create either a Some(value) or None as appropriate for the value provided.

Utiliser l’option au lieu de Null #

En Java (et dans d’autres langages), l’utilisation de null est un moyen courant d’indiquer qu’aucune valeur n’est attachée à une variable de référence. Dans Scala, l’utilisation de Option est préférable à l’utilisation de null. Option enveloppe les valeurs qui pourraient être null.

None est une sous-classe de Option enveloppant une référence nulle. Some est une sous-classe de Option enveloppant une référence non nulle.

Envelopper une référence est facile :

val nothing = Option(null) // None
val something = Option("Aren't options cool?") // Some("Aren't options cool?")

Il s’agit d’un code typique lors de l’appel d’une bibliothèque Java qui peut renvoyer une référence nulle :

val resource = Option(JavaLib.getResource())
// if null, then resource = None
// else resource = Some(resource)

Si getResource() renvoie une valeur null, resource sera un objet None. Sinon, ce sera un objet Some(resource). La meilleure façon de gérer une Option consiste à utiliser les fonctions d’ordre supérieur disponibles dans le type Option. Par exemple, si vous voulez vérifier si votre valeur n’est pas None (similaire à vérifier si value == null), vous utiliserez la fonction isDefined :

val resource: Option[Resource] = Option(JavaLib.getResource())
if (resource.isDefined) {  // resource is `Some(_)` type
  val r: Resource = resource.get
  r.connect()
}

De même, pour rechercher une référence “nulle”, vous pouvez procéder comme suit :

val resource: Option[Resource] = Option(JavaLib.getResource())
if (resource.isEmpty) { // resource is `None` type.
  System.out.println("Resource is empty! Cannot connect.")
}

Il est préférable de traiter l’exécution conditionnelle sur la valeur encapsulée d’une Option (sans utiliser la méthode Option.get ’exceptionnelle’) en traitant Option comme une monade et en utilisant foreach :

val resource: Option[Resource] = Option(JavaLib.getResource())
resource foreach (r => r.connect())
// if r is defined, then r.connect() is run
// if r is empty, then it does nothing

Si une instance Resource est requise (par opposition à une instance Option[Resource]), vous pouvez toujours utiliser Option pour vous protéger contre les valeurs nulles. Ici, la méthode getOrElse fournit une valeur par défaut :

lazy val defaultResource = new Resource()
val resource: Resource = Option(JavaLib.getResource()).getOrElse(defaultResource)

Le code Java ne gère pas facilement Option de Scala, donc lors de la transmission de valeurs au code Java, il est bon de déballer une Option, en passant null ou une valeur par défaut raisonnable, le cas échéant :

val resource: Option[Resource] = ???
JavaLib.sendResource(resource.orNull)
JavaLib.sendResource(resource.getOrElse(defaultResource)) //

Bases #

Une Option est une structure de données qui contient soit une seule valeur, soit aucune valeur du tout. Une Option peut être considérée comme une collection de zéro ou un élément.

Option est une classe abstraite avec deux enfants : Some et None.

Some contient une seule valeur et None ne contient aucune valeur.

Option est utile dans les expressions qui utiliseraient autrement null pour représenter l’absence de valeur concrète. Cela protège contre une NullPointerException et permet la composition de nombreuses expressions qui pourraient ne pas renvoyer de valeur en utilisant des combinateurs tels que Map, FlatMap, etc.

Exemple avec la carte #

val countries = Map(
  "USA" -> "Washington",
  "UK" -> "London",
  "Germany" -> "Berlin",
  "Netherlands" -> "Amsterdam",
  "Japan" -> "Tokyo"
)
    
println(countries.get("USA")) // Some(Washington)
println(countries.get("France")) // None
println(countries.get("USA").get) // Washington
println(countries.get("France").get) // Error: NoSuchElementException
println(countries.get("USA").getOrElse("Nope")) // Washington
println(countries.get("France").getOrElse("Nope")) // Nope

Option[A] est scellé et ne peut donc pas être prolongé. Par conséquent, sa sémantique est stable et fiable.

Options en tant que collections #

Les options ont des fonctions utiles d’ordre supérieur qui peuvent être facilement comprises en affichant les options comme des collections avec zéro ou un élément - où Aucun se comporte comme la collection vide, et Some(x) se comporte comme une collection avec un élément unique, x.

val option: Option[String] = ???

option.map(_.trim) // None if option is None, Some(s.trim) if Some(s)
option.foreach(println) // prints the string if it exists, does nothing otherwise
option.forall(_.length > 4) // true if None or if Some(s) and s.length > 4
option.exists(_.length > 4) // true if Some(s) and s.length > 4
option.toList // returns an actual list

Options dans pour les compréhensions #

Les options ont une méthode flatMap. Cela signifie qu’ils peuvent être utilisés dans un pour la compréhension. De cette façon, nous pouvons soulever des fonctions régulières pour travailler sur des “options” sans avoir à les redéfinir.

val firstOption: Option[Int] = Option(1)
val secondOption: Option[Int] = Option(2)

val myResult = for {
  firstValue <- firstOption
  secondValue <- secondOption
} yield firstValue + secondValue
// myResult: Option[Int] = Some(3)

Lorsque l’une des valeurs est “Aucun”, le résultat final du calcul sera également “Aucun”.

val firstOption: Option[Int] = Option(1)
val secondOption: Option[Int] = None

val myResult = for {
  firstValue <- firstOption
  secondValue <- secondOption
} yield firstValue + secondValue
// myResult: Option[Int] = None

Remarque : ce modèle s’étend plus généralement aux concepts appelés “Monad”. (Plus d’informations devraient être disponibles sur les pages relatives aux compréhensions et Monads)

En général, il n’est pas possible de mélanger différentes monades dans un pour la compréhension. Mais comme Option peut être facilement converti en Iterable, nous pouvons facilement mélanger Options et Iterable en appelant la méthode .toIterable.

val option: Option[Int] = Option(1)
val iterable: Iterable[Int] = Iterable(2, 3, 4, 5)

// does NOT compile since we cannot mix Monads in a for comprehension
// val myResult = for {
//   optionValue <- option
//   iterableValue <- iterable
//} yield optionValue + iterableValue

// It does compile when adding a .toIterable on the option
val myResult = for {
  optionValue <- option.toIterable
  iterableValue <- iterable
} yield optionValue + iterableValue
// myResult: Iterable[Int] = List(2, 3, 4, 5)

Une petite note : si nous avions défini notre for comprehension dans l’autre sens, for comprehension serait compilé puisque notre option serait convertie implicitement. Pour cette raison, il est utile de toujours ajouter ce .toIterable (ou la fonction correspondante selon la collection que vous utilisez) pour plus de cohérence.
