Déclarer des expressions régulières #

La méthode r implicitement fournie via [scala.collection.immutable.StringOps][3] produit une instance de [scala.util.matching.Regex][1] à partir de la chaîne sujet. La syntaxe de chaîne entre guillemets triples de Scala est utile ici, car vous n’avez pas à échapper les barres obliques inverses comme vous le feriez en Java :

val r0: Regex = """(\d{4})-(\d${2})-(\d{2})""".r     // :)
val r1: Regex = "(\\d{4})-(\\d{2})-(\\d{2})".r // :(

[scala.util.matching.Regex][1] implémente une API d’expression régulière idiomatique pour Scala en tant que wrapper sur [java.util.regex.Pattern][2], et la syntaxe prise en charge est la même. Cela étant dit, la prise en charge par Scala des littéraux de chaînes multilignes rend le drapeau x considérablement plus utile, permettant les commentaires et ignorant les espaces blancs :

val dateRegex = """(?x:
   (\d{4}) # year
  -(\d{2}) # month
  -(\d{2}) # day
)""".r

Il existe une version surchargée de r, def r(names: String*): Regex qui vous permet d’attribuer des noms de groupe à vos captures de motifs. Ceci est quelque peu fragile car les noms sont dissociés des captures et ne doivent être utilisés que si l’expression régulière sera utilisée à plusieurs endroits :

"""(\d{4})-(\d{2})-(\d{2})""".r("y", "m", "d").findFirstMatchIn(str) match {
  case Some(matched) =>
    val y = matched.group("y").toInt
    val m = matched.group("m").toInt
    val d = matched.group("d").toInt
    java.time.LocalDate.of(y, m, d)
  case None => ???
}

[1] : http://www.scala-lang.org/api/current/#scala.util.matching.Regex [2] : https://docs.oracle.com/javase/7/docs/api/java/util/regex/Pattern.html [3] : http://www.scala-lang.org/api/current/#scala.collection.immutable.StringOps

Répéter la correspondance d’un motif dans une chaîne #

val re = """\((.*?)\)""".r

val str = "(The)(example)(of)(repeating)(pattern)(in)(a)(single)(string)(I)(had)(some)(trouble)(with)(once)"

re.findAllMatchIn(str).map(_.group(1)).toList
res2: List[String] = List(The, example, of, repeating, pattern, in, a, single, string, I, had, some, trouble, with, once)
