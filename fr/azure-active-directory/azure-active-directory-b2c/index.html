Azure AD B2C - Exemple d’application Angularjs (Web et mobile) #

Cet exemple illustre l’utilisation d’AD B2C pour sécuriser une application Web et mobile basée sur AngularJS.

Reportez-vous à https://github.com/NewtonJoshua/Azure-ADB2C-Angularjs-sample

Azure AD B2C #

Azure AD B2C est une solution cloud de gestion des identités pour vos applications Web et mobiles. Il s’agit d’un service mondial hautement disponible qui s’adapte à des centaines de millions d’identités.

Application Web - Hello.js #

L’implémentation de l’application Web utilise Hello.js qui effectue la gestion des identités avec Azure AD B2C . Hello.js est un SDK JavaScript côté client pour l’authentification avec les services Web OAuth2 et l’interrogation des API REST.

Application mobile - plugin ADAL #

La mise en œuvre de l’application mobile utilise ADAL Cordova Plugin Patch For B2C. Il s’agit d’une version hachée du plugin Active Directory Authentication Library (ADAL) pour les applications Apache Cordova, cordova-plugin-ms-adal qui fonctionne avec Azure AD B2C. Le plug-in original cordova-plugin-ms-adal fournit une fonctionnalité d’authentification facile à utiliser pour vos applications Apache Cordova en tirant parti d’Active Directory.

Décoder JWT #

jwtHelper de angular-jwt se chargera de vous aider à décoder le jeton (JWT) et à vérifier sa date d’expiration. Les jetons Web JSON sont une méthode ouverte et standard de l’industrie RFC 7519 pour représenter les revendications en toute sécurité entre deux parties.

1. Mise en place du projet : #

  1. Cloner ou télécharger ce dépôt

    git clone https://github.com/NewtonJoshua/Azure-ADB2C-Angularjs-sample.git

  2. Installer les dépendances

    npm install

    bower install

2. Configuration AD : #

Configuration ADAL-B2C

  1. Créer un répertoire Azure AD B2C

    Note the Domain name, it’ll be used as the tenantName.

  2. Enregistrer votre application Follow the instructions to create an application and enable both Web App and Native client. Refer Register a web application and Register a mobile/native application

    Enter the Reply URL as http://localhost:8100 or any port from wher you’ll be serving your app.

    In Application Claims, select Email Addresses too.

    Note the Application ID . It’ll be used as the clientId.

  3. [Créer une politique d’inscription ou de connexion](https://docs.microsoft.com/en-us/azure/active-directory-b2c/active-directory-b2c-reference-policies#create-a -politique d’inscription ou de connexion)

    Note the name of the policy. It’ll be used as policy.

  4. [Créer une politique de réinitialisation de mot de passe](https://docs.microsoft.com/en-us/azure/active-directory-b2c/active-directory-b2c-reference-policies#create-a-password-reset- politique)

    Note the name of the policy. It’ll be used as the password-reset-policy

3. Paramètres AD : #

Dans settings.value.js, entrez les valeurs suivantes

  • tenantName : nom de domaine de l’étape 2.1
  • clientId : identifiant de l’application de l’étape 2.2
  • politique : nom de la politique de l’étape 2.3

5. Personnalisez l’interface utilisateur Azure AD B2C #

L’écran de connexion Azure AD B2C peut être personnalisé en fonction de notre image de marque. Reportez-vous à Personnalisation de l’interface utilisateur

Dans cet exemple, nous avons deux écrans d’interface utilisateur personnalisés,

  • AD B2C Connexion et page d’inscription : adCustomPages/unified.html
  • Page de réinitialisation du mot de passe AD B2C : adCustomPages/resetpassword.html

Dans adCustomPages/unified.html, aux lignes 442 et 445, entrez votre tenantName, password-reset-policy et clientId

Les pages doivent être chargées dans un blob et leur URL doit être référencée dans les stratégies Azure AD B2C.

  • Créez un compte de stockage comme mentionné dans [Télécharger l’exemple de contenu sur Azure Blob Storage](https://docs.microsoft.com/en-us/azure/active-directory-b2c/active-directory-b2c-reference-ui -customization-helper-tool#upload-the-sample-content-to-azure-blob-storage)
  • Téléchargez les exemples de pages AD dans le conteneur et notez leur URL.
  • Pour le compte de stockage du service Blob créé, créez une règle CORS avec ‘*’ comme ORIGINES AUTORISÉES. Sélectionnez tout dans MÉTHODES AUTORISÉES. Entrez * pour les EN-TÊTES AUTORISÉS et les EN-TÊTES EXPOSÉS également.
  • Personnalisez votre politique

Maintenant, dans votre application, vous pouvez voir l’interface utilisateur personnalisée.
