Introduction #

Les fonctions de fenêtre sont utilisées pour effectuer des opérations (généralement une agrégation) sur un ensemble de lignes appelées collectivement fenêtre. Les fonctions de fenêtre fonctionnent dans Spark 1.4 ou version ultérieure. Les fonctions de fenêtre fournissent plus d’opérations que les fonctions intégrées ou UDF, telles que substr ou round (largement utilisées avant Spark 1.4). Les fonctions de fenêtre permettent aux utilisateurs de Spark SQL de calculer des résultats tels que le rang d’une ligne donnée ou une moyenne mobile sur une plage de lignes d’entrée. Ils améliorent considérablement l’expressivité des API SQL et DataFrame de Spark.

À la base, une fonction de fenêtre calcule une valeur de retour pour chaque ligne d’entrée d’une table en fonction d’un groupe de lignes, appelé Frame. Chaque ligne d’entrée peut être associée à un cadre unique. Cette caractéristique des fonctions de fenêtre les rend plus puissantes que les autres fonctions. Les types de fonctions de fenêtre sont

  • Fonctions de classement
  • Fonctions analytiques
  • Fonctions d’agrégation

Pour utiliser les fonctions de fenêtre, les utilisateurs doivent marquer qu’une fonction est utilisée comme fonction de fenêtre soit par

  • Ajout d’une clause OVER après une fonction prise en charge dans SQL, par ex. avg(revenu) OVER (...); ou
  • Appel de la méthode over sur une fonction prise en charge dans l’API DataFrame, par ex. rang().sur(...).

Cette documentation vise à démontrer certaines de ces fonctions avec des exemples. Il est supposé que le lecteur a des connaissances sur les opérations de base sur Spark DataFrame comme : ajouter une nouvelle colonne, renommer une colonne, etc.

Lecture d’un exemple d’ensemble de données :

val sampleData = Seq( ("bob","Developer",125000),("mark","Developer",108000),("carl","Tester",70000),("peter","Developer",185000),("jon","Tester",65000),("roman","Tester",82000),("simon","Developer",98000),("eric","Developer",144000),("carlos","Tester",75000),("henry","Developer",110000)).toDF("Name","Role","Salary")

Liste des déclarations d’importation requises :

import org.apache.spark.sql.expressions.Window
import org.apache.spark.sql.functions._

La première instruction importe Window Specification. Une spécification de fenêtre contient des conditions/spécifications indiquant quelles lignes doivent être incluses dans la fenêtre.

Somme cumulée #

Pour calculer la moyenne mobile du salaire des employeurs en fonction de leur rôle :

val cumSum = sampleData.withColumn("cumulativeSum", sum(sampleData("Salary"))
             .over( Window.partitionBy("Role").orderBy("Salary")))
  • orderBy() trie la colonne des salaires et calcule la somme cumulée.

Fonctions de fenêtre - Trier, Lead, Lag, Rank, Trend Analysis #

Cette rubrique montre comment utiliser des fonctions telles que withColumn, lead, lag, Level, etc. à l’aide de Spark. Spark dataframe est une couche abstraite SQL sur les fonctionnalités principales de Spark. Cela permet à l’utilisateur d’écrire du SQL sur des données distribuées. Spark SQL prend en charge les formats de fichiers hétérogènes, notamment JSON, XML, CSV, TSV, etc.

Dans ce blog, nous avons un aperçu rapide de l’utilisation de Spark SQL et des dataframes pour les cas d’utilisation courants dans le monde SQL. Par souci de simplicité, nous traiterons d’un seul fichier au format CSV. Le fichier a quatre champs, employeeID, employeeName, salaire, salaireDate

1,John,1000,01/01/2016
1,John,2000,02/01/2016
1,John,1000,03/01/2016
1,John,2000,04/01/2016
1,John,3000,05/01/2016
1,John,1000,06/01/2016

Enregistrez ce fichier sous emp.dat. Dans la première étape, nous allons créer une trame de données spark à l’aide du package CSV spark de databricks.

val sqlCont = new HiveContext(sc)
//Define a schema for file
val schema = StructType(Array(StructField("EmpId", IntegerType, false),
          StructField("EmpName", StringType, false),
          StructField("Salary", DoubleType, false),
          StructField("SalaryDate", DateType, false)))
//Apply Shema and read data to a dataframe
val myDF = sqlCont.read.format("com.databricks.spark.csv")
          .option("header", "false")
          .option("dateFormat", "MM/dd/yyyy")
          .schema(schema)
          .load("src/resources/data/employee_salary.dat")
//Show dataframe
myDF.show()

myDF est la trame de données utilisée dans l’exercice restant. Étant donné que myDF est utilisé à plusieurs reprises, il est recommandé de le conserver afin qu’il n’ait pas besoin d’être réévalué.

Obtenir et utiliser la ligne précédente (Lag)

LAG est une fonction en SQL qui est utilisée pour accéder aux valeurs de ligne précédentes dans la ligne actuelle. Ceci est utile lorsque nous avons des cas d’utilisation comme la comparaison avec la valeur précédente. Le LAG dans les dataframes Spark est disponible dans les fonctions de fenêtre

lag(Column e, int offset)
Window function: returns the value that is offset rows before the current row, and null if there is less than offset rows before the current row.


import org.apache.spark.sql.expressions.Window
//order by Salary Date to get previous salary.
//For first row we will get NULL
val window = Window.orderBy("SalaryDate")
//use lag to get previous row value for salary, 1 is the offset
val lagCol = lag(col("Salary"), 1).over(window)
myDF.withColumn("LagCol", lagCol).show()

Obtenir et utiliser la ligne suivante (Lead)

LEAD est une fonction en SQL qui est utilisée pour accéder aux valeurs de la ligne suivante dans la ligne actuelle. Ceci est utile lorsque nous avons des cas d’utilisation comme la comparaison avec la valeur suivante. LEAD dans les dataframes Spark est disponible dans les fonctions Window

lead(Column e, int offset)
Window function: returns the value that is offset rows after the current row, and null if there is less than offset rows after the current row.

Analyse de tendance avec fonctions de fenêtre Maintenant, utilisons la fonction de fenêtre LAG avec une simple analyse de tendance. Si le salaire est inférieur au mois précédent, nous le marquerons comme “DOWN”, si le salaire a augmenté, alors “UP”. Le code utilise la fonction Window pour trier par, décaler puis faire un simple if else avec WHEN.

   val window = Window.orderBy("SalaryDate")
    //Derive lag column for salary
    val laggingCol = lag(col("Salary"), 1).over(trend_window)
    //Use derived column LastSalary to find difference between current and previous row
    val salaryDifference = col("Salary") - col("LastSalary")
    //Calculate trend based on the difference
    //IF ELSE / CASE can be written using when.otherwise in spark
    val trend = when(col("SalaryDiff").isNull || col("SalaryDiff").===(0), "SAME")
    .when(col("SalaryDiff").>(0), "UP")
    .otherwise("DOWN")
    myDF.withColumn("LastSalary", laggingCol)
    .withColumn("SalaryDiff",salaryDifference)
   .withColumn("Trend", trend).show()

Moyenne mobile #

Pour calculer la moyenne mobile du salaire des employeurs en fonction de leur rôle :

val movAvg = sampleData.withColumn("movingAverage", avg(sampleData("Salary"))
             .over( Window.partitionBy("Role").rowsBetween(-1,1)) )

  • withColumn() crée une nouvelle colonne nommée movingAverage, effectuant average sur la colonne Salary

  • over() est utilisé pour définir la spécification de la fenêtre.

  • partitionBy() partitionne les données sur la colonne Role

  • rowsBetween(start, end) Cette fonction définit les lignes qui doivent être incluses dans la fenêtre. Les paramètres (start et end) prennent des entrées numériques, 0 représente la ligne actuelle, -1 est la ligne précédente, 1 est la ligne suivante et ainsi de suite. La fonction inclut toutes les lignes entre start et end, ainsi dans cet exemple trois lignes (-1,0,1) sont incluses dans la fenêtre.

      scala> movAvg.show

    +——+———+——+——————+ | Name| Role|Salary| movingAverage| +——+———+——+——————+ | bob|Developer|125000| 116500.0| | mark|Developer|108000|139333.33333333334| | peter|Developer|185000|130333.33333333333| | simon|Developer| 98000|142333.33333333334| | eric|Developer|144000|117333.33333333333| | henry|Developer|110000| 127000.0| | carl| Tester| 70000| 67500.0| | jon| Tester| 65000| 72333.33333333333| | roman| Tester| 82000| 74000.0| |carlos| Tester| 75000| 78500.0| +——+———+——+——————+

Spark ignore automatiquement les lignes précédentes et suivantes, si la ligne actuelle est respectivement la première et la dernière ligne.

Dans l’exemple ci-dessus, movingAverage de la première ligne est la moyenne de la ligne actuelle et suivante uniquement, car la ligne précédente n’existe pas. De même, la dernière ligne de la partition (c’est-à-dire la 6e ligne) est la moyenne de la ligne actuelle et précédente, car la ligne suivante n’existe pas.
