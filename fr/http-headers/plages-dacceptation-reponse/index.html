Accept-Ranges indique au client que ce serveur prendra en charge les plages pour cette ressource (fichier).

Syntaxe #

* Plages acceptées : RangeType

  • Plages d’acceptation : aucune

Paramètres #

Paramètre Description
Type de plage Ce type de plages est pris en charge. Il ne s’agit actuellement que de bytes ou none.
aucun Le serveur ne prend pas en charge les plages sur cette ressource

Accept-Ranges fait partie du système de plages. Le système de plages permet au client de ne demander qu’une partie d’un fichier au lieu d’avoir à télécharger tout le fichier.

Par exemple, si un client n’a besoin que des 100 derniers octets d’un fichier de 10 Mo, il peut demander au serveur d’envoyer uniquement les données du décalage 10485660 à 10485760.

Accept-Ranges est envoyé par le serveur pour indiquer au client s’il prend en charge les plages. Cela ne s’applique qu’à cette ressource particulière (fichier), d’autres fichiers peuvent accepter différents types de gamme.

Seules deux valeurs sont actuellement définies, bytes et none. Les valeurs bytes signifient que vous pouvez demander des plages d’octets (offset et end seront en octets). Une valeur de ’none’ signifie que le serveur ne prend pas en charge les plages.

Les clients sont libres de demander des requêtes de plage d’octets sans vérifier si le serveur prend en charge les plages.

Le client utilise Range pour demander une plage au serveur et le serveur répond avec un statut de 206 (Contenu partiel) s’il envoie la plage d’octets ou 200 (ok) s’il va envoyer le fichier entier.

Le serveur prend en charge les plages #

Requête : “http://example.com” #

<pré> OBTENIR / HTTP/1.1 Hébergeur : example.com Agent utilisateur : Mozilla/5.0 (Windows NT 6.1 ; WOW64 ; rv : 53.0) Gecko/20100101 Firefox/53.0 Accepter : text/html,application/xhtml+xml,application/xml;q=0.9,/;q=0.8 Accept-Langue : en-US,en;q=0.5 Accept-Encoding : gzip, deflate Connexion : keep-alive

Réponse: #

<pré> HTTP/1.1 200 OK Date : sam. 01 janv. 2000 01:00:00 GMT Serveur : Apache/2.4.10 (Win32) OpenSSL/1.0.1h PHP/5.4.31 Keep-Alive : délai d’attente = 5, max = 97 Connexion : Keep-Alive Type de contenu : texte/html Plages d’acceptation : octets Longueur du contenu : 500
