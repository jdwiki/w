Une analyse est une séquence consécutive de valeurs ou d’observations répétées. Pour les valeurs répétées, le “codage de longueur de plage” de R décrit de manière concise un vecteur en termes de ses plages. Envisager:

dat <- c(1, 2, 2, 2, 3, 1, 4, 4, 1, 1)

Nous avons une séquence de longueur un de 1 s ; puis une série de longueur trois de 2s ; puis une série de 3s de longueur un ; etc. Le codage de longueur d’exécution de R capture toutes les longueurs et valeurs des exécutions d’un vecteur.

#Extensions

Une exécution peut également faire référence à des observations consécutives dans des données tabulaires. Bien que R n’ait pas de moyen naturel de les coder, ils peuvent être gérés avec [rleid du package data.table (actuellement un lien sans issue)][1].

[1] : https://www.wikiod.com/fr/data-table

Identification et regroupement par runs en base R #

On pourrait vouloir regrouper leurs données par les exécutions d’une variable et effectuer une sorte d’analyse. Considérez l’ensemble de données simple suivant :

(dat <- data.frame(x = c(1, 1, 2, 2, 2, 1), y = 1:6))
#   x y
# 1 1 1
# 2 1 2
# 3 2 3
# 4 2 4
# 5 2 5
# 6 1 6

La variable ‘x’ a trois suites : une suite de longueur 2 avec la valeur 1, une suite de longueur 3 avec la valeur 2 et une suite de longueur 1 avec la valeur 1. Nous pourrions vouloir calculer la valeur moyenne de la variable ‘y’ dans chacune des exécutions de la variable “x” (ces valeurs moyennes sont 1,5, 4 et 6).

En base R, nous calculerions d’abord l’encodage de longueur de la variable x en utilisant rle :

(r <- rle(dat$x))
# Run Length Encoding
#   lengths: int [1:3] 2 3 1
#   values : num [1:3] 1 2 1

L’étape suivante consiste à calculer le numéro d’exécution de chaque ligne de notre ensemble de données. Nous savons que le nombre total d’exécutions est length(r$lengths) et que la longueur de chaque exécution est r$lengths, nous pouvons donc calculer le nombre d’exécutions de chacune de nos exécutions avec rep :

(run.id <- rep(seq_along(r$lengths), r$lengths))
# [1] 1 1 2 2 2 3

Maintenant, nous pouvons utiliser tapply pour calculer la valeur moyenne y pour chaque exécution en regroupant sur l’ID d’exécution :

data.frame(x=r$values, meanY=tapply(dat$y, run.id, mean))
#   x meanY
# 1 1   1.5
# 2 2   4.0
# 3 1   6.0

Encodage de longueur d’exécution avec rle #

Le codage de longueur de plage capture les longueurs de plages d’éléments consécutifs dans un vecteur. Prenons un exemple de vecteur :

dat <- c(1, 2, 2, 2, 3, 1, 4, 4, 1, 1)

La fonction rle extrait chaque exécution et sa longueur :

r <- rle(dat)
r
# Run Length Encoding
#   lengths: int [1:6] 1 3 1 1 2 2
#   values : num [1:6] 1 2 3 1 4 1

Les valeurs de chaque exécution sont capturées dans r$values :

r$values
# [1] 1 2 3 1 4 1

Cela montre que nous avons d’abord vu une série de 1, puis une série de 2, puis une série de 3, puis une série de 1, et ainsi de suite.

Les longueurs de chaque exécution sont capturées dans r$lengths :

r$lengths
# [1] 1 3 1 1 2 2

Nous voyons que la série initiale de 1 était de longueur 1, la série de 2 qui a suivi était de longueur 3, et ainsi de suite.

Encodage de longueur d’exécution pour compresser et décompresser les vecteurs #

Les vecteurs longs avec de longues séries de la même valeur peuvent être considérablement compressés en les stockant dans leur codage de longueur de plage (la valeur de chaque plage et le nombre de fois que cette valeur est répétée). A titre d’exemple, considérons un vecteur de longueur 10 millions avec un grand nombre de 1 et seulement un petit nombre de 0 :

set.seed(144)
dat <- sample(rep(0:1, c(1, 1e5)), 1e7, replace=TRUE)
table(dat)
#       0       1 
#     103 9999897

Le stockage de 10 millions d’entrées nécessitera un espace important, mais nous pouvons à la place créer une trame de données avec le codage de longueur d’exécution de ce vecteur :

rle.df <- with(rle(dat), data.frame(values, lengths))
dim(rle.df)
# [1] 207   2
head(rle.df)
#   values lengths
# 1      1   52818
# 2      0       1
# 3      1  219329
# 4      0       1
# 5      1  318306
# 6      0       1

À partir du codage de longueur de plage, nous voyons que les 52 818 premières valeurs du vecteur sont des 1, suivis d’un seul 0, suivis de 219 329 1 consécutifs, suivis d’un 0, et ainsi de suite. L’encodage de longueur d’exécution n’a que 207 entrées, ce qui nous oblige à stocker seulement 414 valeurs au lieu de 10 millions de valeurs. Comme rle.df est un bloc de données, il peut être stocké à l’aide de fonctions standard telles que write.csv.

La décompression d’un vecteur dans le codage de longueur d’exécution peut être accomplie de deux manières. La première méthode consiste simplement à appeler rep, en passant l’élément values de l’encodage run-length comme premier argument et l’élément lengths de l’encodage run-length comme deuxième argument :

decompressed <- rep(rle.df$values, rle.df$lengths)

Nous pouvons confirmer que nos données décompressées sont identiques à nos données d’origine :

identical(decompressed, dat)
# [1] TRUE

La deuxième méthode consiste à utiliser la fonction intégrée inverse.rle de R sur l’objet rle, par exemple :

rle.obj <- rle(dat)                            # create a rle object here
class(rle.obj)
# [1] "rle"

dat.inv <- inverse.rle(rle.obj)               # apply the inverse.rle on the rle object

Nous pouvons confirmer à nouveau que cela produit exactement le dat d’origine :

identical(dat.inv, dat)
# [1] TRUE

Identification et regroupement par exécutions dans data.table #

Le package data.table fournit un moyen pratique de regrouper par exécutions dans les données. Considérez les exemples de données suivants :

library(data.table)
(DT <- data.table(x = c(1, 1, 2, 2, 2, 1), y = 1:6))
#    x y
# 1: 1 1
# 2: 1 2
# 3: 2 3
# 4: 2 4
# 5: 2 5
# 6: 1 6

La variable ‘x’ a trois suites : une suite de longueur 2 avec la valeur 1, une suite de longueur 3 avec la valeur 2 et une suite de longueur 1 avec la valeur 1. Nous pourrions vouloir calculer la valeur moyenne de la variable ‘y’ dans chacun des passages de la variable x (ces valeurs moyennes sont 1,5, 4 et 6).

La fonction data.table rleid fournit un identifiant indiquant l’identifiant d’exécution de chaque élément d’un vecteur :

rleid(DT$x)
# [1] 1 1 2 2 2 3

On peut alors facilement regrouper sur cet ID d’exécution et résumer les données ‘y’ :

DT[,mean(y),by=.(x, rleid(x))]
#    x rleid  V1
# 1: 1     1 1.5
# 2: 2     2 4.0
# 3: 1     3 6.0
