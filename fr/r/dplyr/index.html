dplyr est une itération de plyr qui fournit des fonctions flexibles basées sur des “verbes” pour manipuler des données dans R. La dernière version de dplyr peut être téléchargée à partir du CRAN en utilisant

install.package("dplyr")

L’objet clé dans dplyr est un tbl, une représentation d’une structure de données tabulaire. Actuellement, dplyr (version 0.5.0) prend en charge :

  • trames de données
  • tableaux de données
  • SQLite
  • PostgreSQL/Redshift
  • MySQL/MariaDB
  • BigQuery
  • Monet DB
  • cubes de données avec tableaux (implémentation partielle)

Les verbes à table unique de dplyr #

dplyr introduit une grammaire de manipulation de données en R. Il fournit une interface cohérente pour travailler avec les données, peu importe où elles sont stockées : data.frame, [data.table] (https://www.wikiod.com/fr/r/datatable), ou une “base de données”. Les éléments clés de dplyr sont écrits à l’aide de Rcpp, ce qui le rend très rapide pour travailler avec des données en mémoire.

La philosophie de dplyr est d’avoir de petites fonctions qui font bien une chose. Les cinq fonctions simples (filter, arrange, select, mutate et summarise) peuvent être utilisées pour révéler de nouvelles façons de décrire les données. Lorsqu’elles sont combinées avec group_by, ces fonctions peuvent être utilisées pour calculer des statistiques récapitulatives par groupe.

Points communs de syntaxe #

Toutes ces fonctions ont une syntaxe similaire :

  • Le premier argument de toutes ces fonctions est toujours une trame de données
  • Les colonnes peuvent être référencées directement à l’aide de noms de variables simples (c’est-à-dire sans utiliser $)
  • Ces fonctions ne modifient pas les données d’origine elles-mêmes, c’est-à-dire qu’elles n’ont pas d’effets secondaires. Par conséquent, les résultats doivent toujours être enregistrés dans un objet.

Nous utiliserons l’ensemble de données intégré mtcars pour explorer les verbes de table unique de dplyr . Avant de convertir le type de mtcars en tbl_df (car cela rend l’impression plus propre), nous ajoutons les rownames de l’ensemble de données sous forme de colonne à l’aide de la fonction rownames_to_column du [tibble](https://cran.r -project.org/web/packages/tibble/index.html).

library(dplyr) # This documentation was written using version 0.5.0

mtcars_tbl <- as_data_frame(tibble::rownames_to_column(mtcars, "cars"))

# examine the structure of data

filtre #

filter aide les lignes de sous-ensemble qui correspondent à certains critères. Le premier argument est le nom de data.frame et le deuxième (et les suivants) arguments sont les critères qui filtrent les données (ces critères doivent être évalués à TRUE ou FALSE)

Sous-ensemble toutes les voitures qui ont 4 cylindres - cyl :

filter(mtcars_tbl, cyl == 4)

On peut passer plusieurs critères séparés par une virgule. Pour sous-ensemble les voitures qui ont 4 ou 6 cylindres - cyl et ont 5 vitesses - gear :

filter(mtcars_tbl, cyl == 4 | cyl == 6, gear == 5)

filter sélectionne les lignes en fonction de critères, pour sélectionner les lignes par position, utilisez slice. slice ne prend que 2 arguments : le premier est un data.frame et le second des valeurs de ligne entières.

Pour sélectionner les lignes 6 à 9 :

slice(mtcars_tbl, 6:9)

# A tibble: 4 x 12
#        cars   mpg   cyl  disp    hp  drat    wt  qsec    vs    am  gear  carb
#       <chr> <dbl> <dbl> <dbl> <dbl> <dbl> <dbl> <dbl> <dbl> <dbl> <dbl> <dbl>
#1    Valiant  18.1     6 225.0   105  2.76  3.46 20.22     1     0     3     1
#2 Duster 360  14.3     8 360.0   245  3.21  3.57 15.84     0     0     3     4
#3  Merc 240D  24.4     4 146.7    62  3.69  3.19 20.00     1     0     4     2
#4   Merc 230  22.8     4 140.8    95  3.92  3.15 22.90     1     0     4     2

Ou:

slice(mtcars_tbl, -c(1:5, 10:n()))

Cela donne le même résultat que slice(mtcars_tbl, 6:9)

n() représente le nombre d’observations dans le groupe actuel

organiser #

arrange est utilisé pour trier les données selon une ou plusieurs variables spécifiées. Tout comme le verbe précédent (et toutes les autres fonctions de dplyr), le premier argument est un data.frame, et les arguments suivants sont utilisés pour trier les données. Si plusieurs variables sont transmises, les données sont d’abord triées par la première variable, puis par la deuxième variable, et ainsi de suite.

Pour trier les données par chevaux - hp

arrange(mtcars_tbl, hp)

arrange(mtcars_tbl, desc(mpg), cyl)

résumer #

summarise calcule les statistiques récapitulatives des variables en regroupant plusieurs valeurs en une seule valeur. Il peut calculer plusieurs statistiques et nous pouvons nommer ces colonnes récapitulatives dans la même instruction.

Pour calculer la moyenne et l’écart type de mpg et disp de toutes les voitures du jeu de données :

par groupe #

selected <- select(mtcars_tbl, cars:hp, gear)

filter(

mtcars_tbl %>% 
    select(cars:hp) %>% 
    arrange(cyl, desc(mpg)) %>%
    group_by(cyl) %>% 
    filter(mpg > 20, hp > 75)

Agrégation avec l’opérateur %>% (pipe) #

Le tube (%>%) [opérateur][1] peut être utilisé en combinaison avec les fonctions dplyr. Dans cet exemple, nous utilisons le jeu de données mtcars (voir help("mtcars") pour plus d’informations) pour montrer comment résumer une trame de données et ajouter des variables aux données avec le résultat de l’application d’une fonction.

library(dplyr)
library(magrittr)
df <- mtcars
df$cars <- rownames(df) #just add the cars names to the df
df <- df[,c(ncol(df),1:(ncol(df)-1))] # and place the names in the first column

1. Résumer les données

Pour calculer les statistiques, nous utilisons summarize et les fonctions appropriées. Dans ce cas, n() est utilisé pour compter le nombre de cas.

 df %>%
  summarize(count=n(),mean_mpg = mean(mpg, na.rm = TRUE),
            min_weight = min(wt),max_weight = max(wt))

#  count mean_mpg min_weight max_weight
#1    32 20.09062      1.513      5.424

2. Calculer les statistiques par groupe

Il est possible de calculer les statistiques par groupes de données. Dans ce cas par Nombre de cylindres et Nombre de vitesses avant

df %>%
  group_by(cyl, gear) %>%
  summarize(count=n(),mean_mpg = mean(mpg, na.rm = TRUE),
            min_weight = min(wt),max_weight = max(wt))

Exemples de NSE et de variables de chaîne dans dpylr #

dplyr utilise l’évaluation non standard (NSE), c’est pourquoi nous pouvons normalement utiliser les noms de variables sans guillemets. Cependant, parfois pendant le pipeline de données, nous devons obtenir nos noms de variables à partir d’autres sources telles qu’une boîte de sélection Shiny. Dans le cas de fonctions comme select, nous pouvons simplement utiliser select_ pour utiliser une variable de chaîne pour sélectionner

variable1 <- "Sepal.Length"
variable2 <- "Sepal.Width"
iris %>%
select_(variable1, variable2) %>%
head(n=5)
#  Sepal.Length Sepal.Width
#  1          5.1         3.5
#  2          4.9         3.0
#  3          4.7         3.2
#  4          4.6         3.1
#  5          5.0         3.6

Mais si nous voulons utiliser d’autres fonctionnalités telles que résumer ou filtrer, nous devons utiliser la fonction interp du package lazyeval

variable1 <- "Sepal.Length"
variable2 <- "Sepal.Width"
variable3 <- "Species"
iris %>%
select_(variable1, variable2, variable3) %>%
group_by_(variable3) %>%
summarize_(mean1 = lazyeval::interp(~mean(var), var = as.name(variable1)), mean2 = lazyeval::interp(~mean(var), var = as.name(variable2)))
#      Species mean1 mean2
#       <fctr> <dbl> <dbl>
# 1     setosa 5.006 3.428
# 2 versicolor 5.936 2.770
# 3  virginica 6.588 2.974
