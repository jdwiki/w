Les bases de la formule #

Les fonctions statistiques dans R font un usage intensif de la notation de formule dite de Wilkinson-Rogers1 .

Lors de l’exécution de fonctions de modèle telles que lm pour les [régressions linéaires] (https://www.wikiod.com/fr/r/modeles-lineaires-regression), elles ont besoin d’une formule. Cette «formule» précise quels coefficients de régression doivent être estimés.

my_formula1 <- formula(mpg ~ wt)
class(my_formula1) 
# gives "formula"

mod1 <- lm(my_formula1, data = mtcars)
coef(mod1)
# gives (Intercept)          wt 
#         37.285126   -5.344472

Sur le côté gauche du ~ (LHS) la variable dépendante est spécifiée, tandis que le côté droit (RHS) contient les variables indépendantes. Techniquement, l’appel formula ci-dessus est redondant car l’opérateur tilde est une fonction infixe qui renvoie un objet avec une classe de formule :

form <- mpg ~ wt
class(form)
#[1] "formula"

L’avantage de la fonction formula par rapport à ~ est qu’elle permet également de spécifier un environnement d’évaluation :

form_mt <- formula(mpg ~ wt, env = mtcars)

Dans ce cas, la sortie montre qu’un coefficient de régression pour “wt” est estimé, ainsi que (par défaut) un paramètre d’interception. L’interception peut être exclue/forcée à 0 en incluant ‘0’ ou ‘-1’ dans la ‘formule’ :

coef(lm(mpg ~ 0 + wt, data = mtcars))
coef(lm(mpg ~ wt -1, data = mtcars))

Les interactions entre les variables a et b peuvent être ajoutées en incluant a:b à la formule :

 coef(lm(mpg ~ wt:vs, data = mtcars))

Comme il est (d’un point de vue statistique) généralement conseillé de ne pas avoir d’interactions dans le modèle sans les effets principaux, l’approche naïve serait d’étendre la “formule” à “a + b + a:b”. Cela fonctionne mais peut être simplifié en écrivant a*b, où l’opérateur * indique un croisement de facteurs (entre deux colonnes de facteurs) ou une multiplication lorsqu’une ou les deux colonnes sont ’numériques’ :

coef(lm(mpg ~ wt*vs, data = mtcars))

L’utilisation de la notation * élargit un terme pour inclure tous les effets d’ordre inférieur, tels que :

coef(lm(mpg ~ wt*vs*hp, data = mtcars))

donnera, en plus de l’ordonnée à l’origine, 7 coefficients de régression. Un pour l’interaction à trois voies, trois pour les interactions à deux voies et trois pour les effets principaux.

Si l’on veut, par exemple, exclure l’interaction à trois, mais conserver toutes les interactions à deux, il y a deux raccourcis. Tout d’abord, en utilisant -, nous pouvons soustraire n’importe quel terme particulier :

coef(lm(mpg ~ wt*vs*hp - wt🆚hp, data = mtcars))

Ou, nous pouvons utiliser la notation ^ pour spécifier le niveau d’interaction dont nous avons besoin :

coef(lm(mpg ~ (wt + vs + hp) ^ 2, data = mtcars))

Ces deux spécifications de formule doivent créer la même matrice de modèle.

Enfin, . est un raccourci pour utiliser toutes les variables disponibles comme effets principaux. Dans ce cas, l’argument data est utilisé pour obtenir les variables disponibles (qui ne sont pas sur le LHS). Par conséquent:

coef(lm(mpg ~ ., data = mtcars))

donne des coefficients pour l’ordonnée à l’origine et 10 variables indépendantes. Cette notation est fréquemment utilisée dans les packages d’apprentissage automatique, où l’on souhaite utiliser toutes les variables pour la prédiction ou la classification. Notez que la signification de . dépend du contexte (voir par exemple ?update.formula pour une signification différente).

  1. G.N. Wilkinson et C.E. Rogers. Journal de la Royal Statistical Society. Série C (Statistiques appliquées) Vol. 22, n° 3 (1973), p. 392-399

Créer des termes d’interaction linéaires, quadratiques et de second ordre #

y ~ . : Ici, . est interprété comme toutes les variables sauf y dans le bloc de données utilisé pour ajuster le modèle. Elle est équivalente aux combinaisons linéaires de variables prédictives. Par exemple y ~ var1 + var2 + var3+...+var15

y ~ . ^ 2 donnera tous les termes d’interaction linéaires (effets principaux) et de second ordre des variables dans le bloc de données. Cela équivaut à y ~ var1 + var2 + ...+var15 + var1:var2 + var1:var3 + var1:var4...et ainsi de suite

y ~ var1 + var2 + ...+var15 + I(var1^2) + I(var2^2) + I(var3^2)...+I(var15^2) : Ici I(var^2) indique un polynôme quadratique d’une variable dans la trame de données.

y ~ poly(var1, degré = 2) + poly(var2, degré = 2)+...poly(var15, degré = 2)

ou

y ~ poly(var1, var2, var3, ....var15, degree = 2) sera équivalent à l’expression ci-dessus.

poly(var1, degré = 2) est équivalent à var1 + I(var1^2).

Pour obtenir des polynômes cubiques, utilisez degree = 3 dans poly().

Il y a une mise en garde à utiliser poly versus I(var, 2), qui est qu’après ajustement du modèle, chacun d’eux produira des coefficients différents, mais les valeurs ajustées sont équivalentes, car elles représentent différentes paramétrisations du même modèle . Il est recommandé d’utiliser I(var, 2) plutôt que poly() pour éviter l’effet de résumé vu dans poly().

En résumé, pour obtenir des termes d’interaction linéaires, quadratiques et de second ordre, vous aurez une expression comme

y ~ .^2 + I(var1^2) + I(var2^2)+...I(var15^2)

Démo pour quatre variables :

old <- reformulate( 'y ~ x1+x2+x3+x4' )
new <- reformulate( " y ~ .^2 + I(x1^2) + I(x2^2) + I(x3^2) + I(x4^2) ")
tmp <- .Call(stats:::C_updateform, old, new)
terms.formula(tmp, simplify = TRUE )
