Syntaxe #

  • ymd_hms(…, quiet = FALSE, tz = “UTC”, locale = Sys.getlocale(“LC_TIME”))
  • maintenant(tzone = “”)
  • intervalle(début, fin, tzone = attr(début, “tzone”))
  • durée(num = NULL, units = “secondes”, …)
  • period(num = NULL, units = “seconde”, …)

install.packages("lubridate")

Pour installer la version de développement depuis Github :

library(devtools)
# dev mode allows testing of development packages in a sandbox, without interfering
# with the other packages you have installed.
dev_mode(on=T)
install_github("hadley/lubridate")
dev_mode(on=F)

Analyse des dates et des dates-heures à partir de chaînes avec lubridate #

Le package lubridate fournit des fonctions pratiques pour formater les objets date et datetime à partir de chaînes de caractères. Les fonctions sont des permutations de

Lettre Élément à analyser Base R équivalent
et année %y, %Y
m (avec y et d) mois %m, %b, %h, %B
d jour %d, %e
h heure %H, %I%p
m (avec h et s) minutes %M
s secondes %S

par exemple. ymd() pour analyser une date avec l’année suivie du mois suivi du jour, par ex. "2016-07-22" ou ymd_hms() pour analyser une date/heure dans l’ordre année, mois, jour, heures, minutes, secondes, par ex. "2016-07-22 13:04:47".

Les fonctions sont capables de reconnaître la plupart des séparateurs (tels que /, - et les espaces) sans arguments supplémentaires. Ils fonctionnent également avec des séparateurs incohérents.

Rendez-vous #

Les fonctions de date renvoient un objet de classe Date.

library(lubridate) 

mdy(c(' 07/02/2016 ', '7 / 03 / 2016', ' 7 / 4 / 16 '))
## [1] "2016-07-02" "2016-07-03" "2016-07-04"

ymd(c("20160724","2016/07/23","2016-07-25"))    # inconsistent separators
## [1] "2016-07-24" "2016-07-23" "2016-07-25"

Dateheures #

Fonctions utilitaires #

Les dates et heures peuvent être analysées à l’aide de variantes ymd_hms, notamment ymd_hm et ymd_h. Toutes les fonctions datetime peuvent accepter un argument de fuseau horaire tz semblable à celui de as.POSIXct ou strptime, mais qui par défaut est "UTC" au lieu du fuseau horaire local.

Les fonctions datetime renvoient un objet de classe POSIXct.

x <- c("20160724 130102","2016/07/23 14:02:01","2016-07-25 15:03:00")
ymd_hms(x, tz="EST")
## [1] "2016-07-24 13:01:02 EST" "2016-07-23 14:02:01 EST"
## [3] "2016-07-25 15:03:00 EST"

ymd_hms(x)
## [1] "2016-07-24 13:01:02 UTC" "2016-07-23 14:02:01 UTC"
## [3] "2016-07-25 15:03:00 UTC"

Fonctions d’analyseur #

lubridate inclut également trois fonctions pour analyser les datetimes avec une chaîne de formatage comme as.POSIXct ou strptime :

Fonction Classe de sortie Chaînes de formatage acceptées
parse_date_heure POSIXct Flexible. Acceptera le style strptime avec le style de nom de fonction datetime % ou lubridate, par exemple "ymd hms". Acceptera un vecteur de commandes pour des données hétérogènes et devinera ce qui est approprié.
parse_date_time2 POSIXct par défaut ; si lt = TRUE, POSIXlt Stricte. Accepte uniquement les jetons strptime (avec ou sans %) d’un ensemble limité.
fast_strptime POSIXlt par défaut ; si lt = FALSE, POSIXct Stricte. Accepte uniquement les jetons strptime délimités par % avec des délimiteurs (-, /, :, etc.) à partir d’un ensemble limité. 
x <- c('2016-07-22 13:04:47', '07/22/2016 1:04:47 pm')

parse_date_time(x, orders = c('mdy Imsp', 'ymd hms'))
## [1] "2016-07-22 13:04:47 UTC" "2016-07-22 13:04:47 UTC"

x <- c('2016-07-22 13:04:47', '2016-07-22 14:47:58')

parse_date_time2(x, orders = 'Ymd HMS')
## [1] "2016-07-22 13:04:47 UTC" "2016-07-22 14:47:58 UTC"

fast_strptime(x, format = '%Y-%m-%d %H:%M:%S')
## [1] "2016-07-22 13:04:47 UTC" "2016-07-22 14:47:58 UTC"

parse_date_time2 et fast_strptime utilisent un analyseur C rapide pour plus d’efficacité.

Voir ?parse_date_time pour formater les jetons.

Différence entre période et durée #

Contrairement aux durées, les périodes peuvent être utilisées pour modéliser avec précision les heures d’horloge sans savoir quand des événements tels que les secondes intercalaires, les jours intercalaires et les changements d’heure d’été se produisent.

start_2012 <- ymd_hms("2012-01-01 12:00:00")
## [1] "2012-01-01 12:00:00 UTC"

# period() considers leap year calculations.
start_2012 + period(1, "years")
## [1] "2013-01-01 12:00:00 UTC"

# Here duration() doesn't consider leap year calculations. 
start_2012 + duration(1)
## [1] "2012-12-31 12:00:00 UTC"

Instantanés #

Un instant est un moment précis dans le temps. Tout objet date-heure qui fait référence à un instant est reconnu comme un instant. Pour tester si un objet est un instant, utilisez is.instant.

library(lubridate)

today_start <- dmy_hms("22.07.2016 12:00:00", tz = "IST") # default tz="UTC"
today_start
## [1] "2016-07-22 12:00:00 IST"
is.instant(today_start)
## [1] TRUE

now_dt <- ymd_hms(now(), tz="IST")
now_dt
## [1] "2016-07-22 13:53:09 IST"
is.instant(now_dt)
## [1] TRUE

is.instant("helloworld")
## [1] FALSE
is.instant(60)
## [1] FALSE

Intervalles, durées et périodes #

Les intervalles sont le moyen le plus simple d’enregistrer des durées dans Lubrifier. Un intervalle est une période de temps qui se produit entre deux instants spécifiques.

# create interval by substracting two instants
today_start <- ymd_hms("2016-07-22 12-00-00", tz="IST")
today_start
## [1] "2016-07-22 12:00:00 IST"
today_end <- ymd_hms("2016-07-22 23-59-59", tz="IST")
today_end
## [1] "2016-07-22 23:59:59 IST"
span <- today_end - today_start
span
## Time difference of 11.99972 hours
as.interval(span, today_start)
## [1] 2016-07-22 12:00:00 IST--2016-07-22 23:59:59 IST

# create interval using interval() function
span <- interval(today_start, today_end)
[1] 2016-07-22 12:00:00 IST--2016-07-22 23:59:59 IST

Les durées mesurent le temps exact qui s’écoule entre deux instants.

duration(60, "seconds")
## [1] "60s"

duration(2, "minutes")
## [1] "120s (~2 minutes)"

Remarque : Les unités supérieures à la semaine ne sont pas utilisées en raison de leur variabilité.

Les durées peuvent être créées en utilisant dseconds, dminutes et d’autres fonctions d’aide à la durée. Exécutez ?quick_durations pour une liste complète.

dseconds(60)
## [1] "60s"

dhours(2)
## [1] "7200s (~2 hours)"

dyears(1)
## [1] "31536000s (~365 days)"

Les durées peuvent être soustraites et ajoutées aux instants pour obtenir de nouveaux instants.

today_start + dhours(5)
## [1] "2016-07-22 17:00:00 IST"

today_start + dhours(5) + dminutes(30) + dseconds(15)
## [1] "2016-07-22 17:30:15 IST"

Les durées peuvent être créées à partir d’intervalles.

as.duration(span)
[1] "43199s (~12 hours)"

Les périodes mesurent le changement d’heure qui se produit entre deux instants.

Les périodes peuvent être créées à l’aide de la fonction period ainsi que d’autres fonctions d’assistance comme seconds, hours, etc. Pour obtenir une liste complète des fonctions d’assistance de période, exécutez ?quick_periods.

period(1, "hour")
## [1] "1H 0M 0S"

hours(1)
## [1] "1H 0M 0S"

period(6, "months")
## [1] "6m 0d 0H 0M 0S"

months(6)
## [1] "6m 0d 0H 0M 0S"

years(1)
## [1] "1y 0m 0d 0H 0M 0S"

La fonction is.period peut être utilisée pour vérifier si un objet est un point.

is.period(years(1))
## [1] TRUE

is.period(dyears(1))    
## [1] FALSE

date <- now()
date
## "2016-07-22 03:42:35 IST"

year(date)
## 2016

minute(date)
## 42

wday(date, label = T, abbr = T)
# [1] Fri
# Levels: Sun < Mon < Tues < Wed < Thurs < Fri < Sat

## "2016-07-31 03:42:35 IST"

Fuseaux horaires #

with_tz renvoie une date-heure telle qu’elle apparaîtrait dans un fuseau horaire différent.

nyc_time <- now("America/New_York")
nyc_time
## [1] "2016-07-22 05:49:08 EDT"

# corresponding Europe/Moscow time
with_tz(nyc_time, tzone = "Europe/Moscow")
## [1] "2016-07-22 12:49:08 MSK"

force_tz renvoie une date-heure qui a la même heure que x dans le nouveau fuseau horaire.

nyc_time <- now("America/New_York")
nyc_time
## [1] "2016-07-22 05:49:08 EDT"

force_tz(nyc_time, tzone = "Europe/Moscow") # only timezone changes
## [1] "2016-07-22 05:49:08 MSK"

Analyse de la date et de l’heure dans lubridate #

Lubridate fournit la série de fonctions ymd() pour analyser les chaînes de caractères en dates. Les lettres y, m et d correspondent aux éléments année, mois et jour d’une date-heure.

mdy("07-21-2016")                 # Returns Date

## [1] "2016-07-21"

mdy("07-21-2016", tz = "UTC")     # Returns a vector of class POSIXt

## "2016-07-21 UTC"

dmy("21-07-2016")                 # Returns Date

## [1] "2016-07-21"

dmy(c("21.07.2016", "22.07.2016")) # Returns vector of class Date

## [1] "2016-07-21" "2016-07-22"

Arrondir les dates #

now_dt <- ymd_hms(now(), tz="IST")
now_dt
## [1] "2016-07-22 13:53:09 IST"

round_date() prend un objet date-heure et l’arrondit à la valeur entière la plus proche de l’unité de temps spécifiée.

round_date(now_dt, "minute")
## [1] "2016-07-22 13:53:00 IST"

round_date(now_dt, "hour")
## [1] "2016-07-22 14:00:00 IST"

round_date(now_dt, "year")
## [1] "2017-01-01 IST"

floor_date() prend un objet date-heure et l’arrondit à la valeur entière la plus proche de l’unité de temps spécifiée.

floor_date(now_dt, "minute")
## [1] "2016-07-22 13:53:00 IST"

floor_date(now_dt, "hour")
## [1] "2016-07-22 13:00:00 IST"

floor_date(now_dt, "year")
## [1] "2016-01-01 IST"

ceiling_date() prend un objet date-heure et l’arrondit à la valeur entière la plus proche de l’unité de temps spécifiée.

ceiling_date(now_dt, "minute")
## [1] "2016-07-22 13:54:00 IST"

ceiling_date(now_dt, "hour")
## [1] "2016-07-22 14:00:00 IST"

ceiling_date(now_dt, "year")
## [1] "2017-01-01 IST"
