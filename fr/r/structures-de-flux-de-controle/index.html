Les boucles for sont une méthode de contrôle de flux pour répéter une tâche ou un ensemble de tâches sur un domaine. La structure de base d’une boucle for est

for ( [index] in [domain]){
  [body]
}

Où

  1. [index] est un nom qui prend exactement une valeur de [domaine] à chaque itération de la boucle.
  2. [domain] est un vecteur de valeurs sur lequel itérer.
  3. [body] est l’ensemble des instructions à appliquer à chaque itération.

À titre d’exemple trivial, considérons l’utilisation d’une boucle for pour obtenir la somme cumulée d’un vecteur de valeurs.

x <- 1:4
cumulative_sum <- 0
for (i in x){
  cumulative_sum <- cumulative_sum + x[i]
}
cumulative_sum

Optimisation de la structure des boucles For #

Les boucles For peuvent être utiles pour conceptualiser et exécuter des tâches à répéter. Cependant, si elles ne sont pas construites avec soin, elles peuvent être très lentes à exécuter par rapport à l’utilisation préférée de la famille de fonctions “apply”. Néanmoins, il existe une poignée d’éléments que vous pouvez inclure dans la construction de votre boucle for pour optimiser la boucle. Dans de nombreux cas, une bonne construction de la boucle for donnera une efficacité de calcul très proche de celle d’une fonction d’application.

Une boucle for « correctement construite » s’appuie sur la structure de base et inclut une déclaration déclarant l’objet qui capturera chaque itération de la boucle. Cet objet doit avoir à la fois une classe et une longueur déclarées.

[output] <- [vector_of_length]
for ([index] in [length_safe_domain]){
  [output][index] <- [body]
}

Pour illustrer cela, écrivons une boucle pour élever au carré chaque valeur d’un vecteur numérique (il s’agit d’un exemple trivial à titre d’illustration uniquement. La manière “correcte” d’accomplir cette tâche serait x_squared <- x^2).

x <- 1:100
x_squared <- vector("numeric", length = length(x))
for (i in seq_along(x)){
  x_squared[i] <- x[i]^2
}

Encore une fois, notez que nous avons d’abord déclaré un réceptacle pour la sortie x_squared, et lui avons donné la classe “numeric” avec la même longueur que x. De plus, nous avons déclaré un “domaine de longueur sûre” en utilisant la fonction seq_along. seq_along génère un vecteur d’indices pour un objet adapté à une utilisation dans les boucles for. Bien qu’il semble intuitif d’utiliser for (i in 1:length(x)), si x a une longueur de 0, la boucle tentera d’itérer sur le domaine de 1:0, ce qui entraînera une erreur (le 0ème index est indéfini dans R).

Les objets de réceptacle et les domaines de longueur sûre sont gérés en interne par la famille de fonctions “apply” et les utilisateurs sont encouragés à adopter l’approche “apply” à la place des boucles for autant que possible. Cependant, si elle est correctement construite, une boucle for peut parfois fournir une plus grande clarté de code avec une perte d’efficacité minimale.

Vectoriser les boucles #

Les boucles for peuvent souvent être un outil utile pour conceptualiser les tâches qui doivent être accomplies dans chaque itération. Lorsque la boucle est complètement développée et conceptualisée, il peut y avoir des avantages à transformer la boucle en fonction.

Dans cet exemple, nous allons développer une boucle for pour calculer la moyenne de chaque colonne dans l’ensemble de données mtcars (encore une fois, un exemple trivial car cela pourrait être accompli via la fonction colMeans).

column_mean_loop <- vector("numeric", length(mtcars))
for (k in seq_along(mtcars)){
  column_mean_loop[k] <- mean(mtcars[[k]])
}

La boucle for peut être convertie en une fonction d’application en réécrivant le corps de la boucle en tant que fonction.

col_mean_fn <- function(x) mean(x)
column_mean_apply <- vapply(mtcars, col_mean_fn, numeric(1))

Et pour comparer les résultats :

identical(column_mean_loop, 
          unname(column_mean_apply)) #* vapply added names to the elements
                                     #* remove them for comparison

L’avantage de la forme vectorisée est que nous avons pu éliminer quelques lignes de code. Les mécanismes de détermination de la longueur et du type de l’objet de sortie et de l’itération sur un domaine de sécurité de longueur sont gérés pour nous par la fonction d’application. De plus, la fonction d’application est un peu plus rapide que la boucle. La différence de vitesse est souvent négligeable sur le plan humain selon le nombre d’itérations et la complexité du corps.

Construction optimale d’une boucle For #

Pour illustrer l’effet de la construction de la boucle good for, nous allons calculer la moyenne de chaque colonne de quatre manières différentes :

  1. Utiliser une boucle for mal optimisée
  2. Utiliser une boucle for for bien optimisée
  3. Utiliser une famille de fonctions *apply
  4. Utilisation de la fonction colMeans

Chacune de ces options sera affichée dans le code ; une comparaison du temps de calcul pour exécuter chaque option sera affichée ; et enfin une discussion des différences sera donnée.

Mal optimisé pour la boucle #

column_mean_poor <- NULL
for (i in 1:length(mtcars)){
  column_mean_poor[i] <- mean(mtcars[[i]])
}

Bien optimisé pour la boucle #

column_mean_optimal <- vector("numeric", length(mtcars))
for (i in seq_along(mtcars)){
  column_mean_optimal <- mean(mtcars[[i]])
}

Fonction vapply #

column_mean_vapply <- vapply(mtcars, mean, numeric(1))

Fonction colMeans #

column_mean_colMeans <- colMeans(mtcars)

Comparaison d’efficacité #

Les résultats de l’analyse comparative de ces quatre approches sont présentés ci-dessous (code non affiché)

Unit: microseconds
     expr     min       lq     mean   median       uq     max neval  cld
     poor 240.986 262.0820 287.1125 275.8160 307.2485 442.609   100    d
  optimal 220.313 237.4455 258.8426 247.0735 280.9130 362.469   100   c 
   vapply 107.042 109.7320 124.4715 113.4130 132.6695 202.473   100 a   
 colMeans 155.183 161.6955 180.2067 175.0045 194.2605 259.958   100  b

Notez que la boucle for optimisée a devancé la boucle for mal construite. La boucle for mal construite augmente constamment la longueur de l’objet de sortie, et à chaque changement de longueur, R réévalue la classe de l’objet.

Une partie de cette surcharge est supprimée par la boucle for optimisée en déclarant le type d’objet de sortie et sa longueur avant de démarrer la boucle.

Dans cet exemple, cependant, l’utilisation d’une fonction “vapply” double l’efficacité de calcul, en grande partie parce que nous avons dit à R que le résultat devait être numérique (si un résultat n’était pas numérique, une erreur serait renvoyée).

L’utilisation de la fonction colMeans est un peu plus lente que la fonction vapply. Cette différence est attribuable à certaines vérifications d’erreur effectuées dans colMeans et principalement à la conversion as.matrix (parce que mtcars est un data.frame) qui n’étaient pas effectuées dans la fonction vapply.

Construction de base de la boucle For #

Dans cet exemple, nous allons calculer la déviance au carré pour chaque colonne d’un bloc de données, dans ce cas le “mtcars”.

Option A : indice entier

squared_deviance <- vector("list", length(mtcars))
for (i in seq_along(mtcars)){
  squared_deviance[[i]] <- (mtcars[[i]] - mean(mtcars[[i]]))^2
}

squared_deviance est une liste de 11 éléments, comme prévu.

class(squared_deviance)
length(squared_deviance)

Option B : index des caractères

squared_deviance <- vector("list", length(mtcars))
Squared_deviance <- setNames(squared_deviance, names(mtcars))
for (k in names(mtcars)){
  squared_deviance[[k]] <- (mtcars[[k]] - mean(mtcars[[k]]))^2
}

Et si nous voulions un data.frame comme résultat ? Eh bien, il existe de nombreuses options pour transformer une liste en d’autres objets. Cependant, et peut-être le plus simple dans ce cas, sera de stocker les résultats for dans un data.frame.

squared_deviance <- mtcars #copy the original
squared_deviance[TRUE]<-NA  #replace with NA or do squared_deviance[,]<-NA
for (i in seq_along(mtcars)){
  squared_deviance[[i]] <- (mtcars[[i]] - mean(mtcars[[i]]))^2
}
dim(squared_deviance)
[1] 32 11

Le résultat sera le même événement bien que nous utilisions l’option de caractère (B).

Les autres constructions en boucle : while et repeat #

R fournit deux constructions de bouclage supplémentaires, “while” et “repeat”, qui sont généralement utilisées dans des situations où le nombre d’itérations requises est indéterminé.

La boucle “while” #

La forme générale d’une boucle “while” est la suivante,

while (condition) {
    ## do something
    ## in loop body
}

où “condition” est évaluée avant d’entrer dans le corps de la boucle. Si condition s’évalue à TRUE, le code à l’intérieur du corps de la boucle est exécuté, et ce processus se répète jusqu’à ce que condition s’évalue à FALSE (ou qu’une instruction break soit atteinte ; voir ci-dessous). Contrairement à la boucle for, si une boucle while utilise une variable pour effectuer des itérations incrémentielles, la variable doit être déclarée et initialisée à l’avance, et doit être mise à jour dans le corps de la boucle. Par exemple, les boucles suivantes accomplissent la même tâche :

for (i in 0:4) {
    cat(i, "\n")
}
# 0 
# 1 
# 2 
# 3 
# 4 

i <- 0
while (i < 5) {
    cat(i, "\n")
    i <- i + 1
}
# 0 
# 1 
# 2 
# 3 
# 4

Dans la boucle while ci-dessus, la ligne i <- i + 1 est nécessaire pour empêcher une boucle infinie.

De plus, il est possible de terminer une boucle while avec un appel à break depuis l’intérieur du corps de la boucle :

iter <- 0
while (TRUE) {
    if (runif(1) < 0.25) {
        break
    } else {
        iter <- iter + 1
    }
}
iter
#[1] 4

Dans cet exemple, condition est toujours TRUE, donc la seule façon de terminer la boucle est avec un appel à break à l’intérieur du corps. Notez que la valeur finale de iter dépendra de l’état de votre PRNG lorsque cet exemple est exécuté et devrait produire des résultats différents (essentiellement) à chaque exécution du code.

La boucle “répéter” #

La construction repeat est essentiellement la même que while (TRUE) { ## quelque chose }, et a la forme suivante :

repeat ({
    ## do something
    ## in loop body
})

Les {} supplémentaires ne sont pas nécessaires, mais les () le sont. Réécrire l’exemple précédent en utilisant repeat,

iter <- 0
repeat ({
    if (runif(1) < 0.25) {
        break
    } else {
        iter <- iter + 1
    }
})
iter
#[1] 2

En savoir plus sur “pause” #

Il est important de noter que break ne mettra fin qu’à la boucle immédiatement englobante. Autrement dit, ce qui suit est une boucle infinie :

while (TRUE) {
    while (TRUE) {
        cat("inner loop\n")
        break
    }
    cat("outer loop\n")
}

Avec un peu de créativité, cependant, il est possible de rompre entièrement à partir d’une boucle imbriquée. À titre d’exemple, considérons l’expression suivante, qui, dans son état actuel, bouclera à l’infini :

while (TRUE) {
    cat("outer loop body\n")
    while (TRUE) {
        cat("inner loop body\n")
        x <- runif(1)
        if (x < .3) {
            break
        } else {
            cat(sprintf("x is %.5f\n", x))
        }
    }
}

Une possibilité consiste à reconnaître que, contrairement à break, l’expression return a la capacité de retourner le contrôle sur plusieurs niveaux de boucles englobantes. Cependant, puisque return n’est valide que lorsqu’il est utilisé dans une fonction, nous ne pouvons pas simplement remplacer break par return() ci-dessus, mais nous devons également envelopper l’expression entière en tant que fonction anonyme :

(function() {
    while (TRUE) {
        cat("outer loop body\n")
        while (TRUE) {
            cat("inner loop body\n")
            x <- runif(1)
            if (x < .3) {
                return()
            } else {
                cat(sprintf("x is %.5f\n", x))
            }
        }
    }
})()

Alternativement, nous pouvons créer une variable factice (exit) avant l’expression et l’activer via <<- à partir de la boucle interne lorsque nous sommes prêts à terminer :

exit <- FALSE
while (TRUE) {
    cat("outer loop body\n")
    while (TRUE) {
        cat("inner loop body\n")
        x <- runif(1)
        if (x < .3) {
            exit <<- TRUE
            break
        } else {
            cat(sprintf("x is %.5f\n", x))
        }
    }
    if (exit) break
}
