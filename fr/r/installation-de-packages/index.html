Syntaxe #

  • install.packages(pkgs, lib, repos, method, destdir, dependencies, …)

Paramètres #

Paramètre Détails
pkgs vecteur de caractères des noms de packages. Si repos = NULL, un vecteur de caractères des chemins de fichiers.
lib vecteur de caractères indiquant les répertoires de la bibliothèque où installer les packages.
repos vecteur de caractères, la ou les URL de base des dépôts à utiliser, peuvent être NULL pour installer à partir de fichiers locaux
méthode méthode de téléchargement
destdir répertoire où les packages téléchargés sont stockés
dependencies logical indiquant s’il faut également installer les packages désinstallés dont ces packages dépendent/lien vers/importer/suggérer (et ainsi de suite de manière récursive). Non utilisé si repos = NULL.
… Arguments à transmettre à ‘download.file’ ou aux fonctions pour les installations binaires sur OS X et Windows.

Téléchargez et installez des packages à partir de référentiels #

Les packages sont des collections de fonctions R, de données et de code compilé dans un format bien défini. Les référentiels publics (et privés) sont utilisés pour héberger des collections de packages R. La plus grande collection de packages R est disponible auprès du CRAN.

Utilisation du CRAN #

Un package peut être installé depuis CRAN en utilisant le code suivant :

install.packages("dplyr")

Où "dplyr" est appelé vecteur de caractères.

Plusieurs packages peuvent être installés en une seule fois en utilisant la fonction de combinaison c() et en transmettant une série de vecteurs de caractères de noms de packages :

install.packages(c("dplyr", "tidyr", "ggplot2"))

Dans certains cas, install.packages peut demander un miroir CRAN ou échouer, selon la valeur de getOption("repos"). Pour éviter cela, spécifiez un miroir CRAN comme argument repos :

install.packages("dplyr", repos = "https://cloud.r-project.org/")

En utilisant l’argument repos, il est également possible d’installer à partir d’autres référentiels. Pour des informations complètes sur toutes les options disponibles, exécutez ?install.packages.

La plupart des packages nécessitent des fonctions qui ont été implémentées dans d’autres packages (par exemple, le package data.table). Afin d’installer un package (ou plusieurs packages) avec tous les packages utilisés par ce package donné, l’argument dependencies doit être défini sur TRUE):

install.packages("data.table", dependencies = TRUE)

Utiliser un bioconducteur #

Bioconductor héberge une importante collection de packages liés à la bioinformatique. Ils fournissent leur propre gestion de paquets centrée autour de la fonction biocLite :

    ## Try http:// if https:// URLs are not supported
    source("https://bioconductor.org/biocLite.R")
    biocLite()

Par défaut, cela installe un sous-ensemble de packages qui fournissent les fonctionnalités les plus couramment utilisées. Des packages spécifiques peuvent être installés en transmettant un vecteur de noms de packages. Par exemple, pour installer RImmPort à partir de Bioconductor :

    source("https://bioconductor.org/biocLite.R")
    biocLite("RImmPort")

Installer les packages depuis GitHub #

Pour installer des packages directement depuis GitHub, utilisez le package devtools :

library(devtools)
install_github("authorName/repositoryName")

Pour installer ggplot2 depuis github :

devtools::install_github("tidyverse/ggplot2")

La commande ci-dessus installera la version de ggplot2 qui correspond à la branche master. Pour installer à partir d’une branche différente d’un référentiel, utilisez l’argument ref pour fournir le nom de la branche. Par exemple, la commande suivante installera la branche dev_general du package googleway.

devtools::install_github("SymbolixAU/googleway", ref = "dev_general")

Une autre option consiste à utiliser le package ghit. Il fournit une alternative légère pour installer des packages à partir de github :

install.packages("ghit")
ghit::install_github("google/CausalImpact")

Pour installer un paquet qui se trouve dans un dépôt privé sur Github, générez un jeton d’accès personnel sur http://www.github.com/settings/tokens/ (Voir ?install_github pour la documentation sur le même). Suivez ces étapes:

  1. install.packages(c(“curl”, “httr”))

  2. config = httr :: config (ssl_verifypeer = FALSE)

  3. install.packages(“RCurl”) options(RCurlOptions = c(getOption(“RCurlOptions”),ssl.verifypeer = FALSE, ssl.verifyhost = FALSE ) )

  4. getOption(“RCurlOptions”)

    You should see the following:

    ssl.verifypeer ssl.verifyhost 

FALSE          FALSE

  5. bibliothèque (httr) set_config(config(ssl_verifypeer = 0L))

Cela évite l’erreur courante : “Le certificat homologue ne peut pas être authentifié avec des certificats CA donnés”

  1. Enfin, utilisez la commande suivante pour installer votre package de manière transparente

    install_github("username/package_name",auth_token="abc")

Vous pouvez également définir une variable d’environnement GITHUB_PAT, en utilisant

Sys.setenv(GITHUB_PAT = "access_token")
devtools::install_github("organisation/package_name")

Le PAT généré dans Github n’est visible qu’une seule fois, c’est-à-dire lors de sa création initiale, il est donc prudent d’enregistrer ce jeton dans .Rprofile. Cela est également utile si l’organisation dispose de nombreux référentiels privés.

Installer le package à partir de la source locale #

Pour installer le package à partir du fichier source local :

install.packages(path_to_source, repos = NULL, type="source")

install.packages("~/Downloads/dplyr-master.zip", repos=NULL, type="source")

Ici, path_to_source est le chemin absolu du fichier source local.

Une autre commande qui ouvre une fenêtre pour choisir les fichiers sources zip ou tar.gz téléchargés est :

install.packages(file.choose(), repos=NULL)

Une autre méthode possible consiste à utiliser le RStudio basé sur l’interface graphique :

Étape 1 : Accédez à Outils.

Étape 2 : Accédez à Installer les packages.

Étape 3 : Dans Installer depuis, définissez-le comme Fichier d’archive de package (.zip ; .tar.gz)

Étape 4 : Ensuite, Parcourir, trouvez votre fichier de package (disons crayon_1.3.1.zip) et *après un certain temps (après qu’il affiche le Chemin du package et le nom du fichier dans l’onglet Archive du package) *

Une autre façon d’installer le package R à partir d’une source locale consiste à utiliser la fonction install_local() du package devtools.

library(devtools)
install_local("~/Downloads/dplyr-master.zip")

Utilisation d’un gestionnaire de packages CLI – utilisation de base de pacman #

pacman est un simple gestionnaire de paquets pour R.

pacman permet à un utilisateur de charger de manière compacte tous les packages souhaités, en installant ceux qui manquent (et leurs dépendances), avec une seule commande, p_load. pacman ne nécessite pas que l’utilisateur tape des guillemets autour d’un nom de paquet. L’utilisation de base est la suivante :

p_load(data.table, dplyr, ggplot2)

Le seul paquet nécessitant une instruction library, require ou install.packages avec cette approche est pacman lui-même :

library(pacman)
p_load(data.table, dplyr, ggplot2)

ou, tout aussi valable :

pacman::p_load(data.table, dplyr, ggplot2)

En plus de gagner du temps en nécessitant moins de code pour gérer les packages, pacman facilite également la construction de code reproductible en installant tous les packages nécessaires si et seulement s’ils ne sont pas déjà installés.

Étant donné que vous ne savez peut-être pas si pacman est installé dans la bibliothèque d’un utilisateur qui utilisera votre code (ou par vous-même lors d’utilisations futures de votre propre code), une meilleure pratique consiste à inclure une instruction conditionnelle pour installer pacman si il n’est pas déjà chargé :

if(!(require(pacman)) install.packages("pacman")
pacman::p_load(data.table, dplyr, ggplot2)

Installer la version de développement local d’un package #

Lorsque vous travaillez sur le développement d’un package R, il est souvent nécessaire d’installer la dernière version du package. Ceci peut être réalisé en construisant d’abord une distribution source du paquet (sur la ligne de commande)

R CMD build my_package

puis [l’installer dans R][1]. Toute session R en cours d’exécution avec une version précédente du package chargée devra la recharger.

unloadNamespace("my_package")
library(my_package)

Une approche plus pratique utilise le package devtools pour simplifier le processus. Dans une session R avec le répertoire de travail défini sur le répertoire du package

devtools::install()

va construire, installer et recharger le package.

[1] : https://www.wikiod.com/fr/r/installation-de-packages#Installer le package à partir de la source locale
