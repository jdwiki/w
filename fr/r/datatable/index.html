Data.table est un package qui étend les fonctionnalités des trames de données de base R, en améliorant notamment leurs performances et leur syntaxe. Voir la zone Docs du package à https://www.wikiod.com/fr/data-table/premiers-pas-avec-datatable pour plus de détails.

Syntaxe #

  • DT[i, j, par] # DT[where, select|update|do, by]
  • DT[...][...] # chaining
  • ################# Raccourcis, fonctions spéciales et symboles spéciaux dans DT[...]
  • .()
    # in several arguments, replaces list()
    -J() # in i, replaces list()
  • :=
    # in j, a function used to add or modify columns
  • .N # in i, the total number of rows
    # in j, the number of rows in a group
  • .JE # in j, the vector of row numbers in the table (filtered by i)
  • .DAKOTA DU SUD # in j, the current subset of the data
    # selected by the .SDcols argument
  • .GRP # in j, the current index of the subset of the data
  • .PAR # in j, the list of by values for the current subset of data
  • V1, V2, … # default names for unnamed columns created in j
  • ################# Jointures à l'intérieur de DT[...]
  • DT1[DT2, on, j] # join two tables
  • je.* # special prefix on DT2’s columns after the join
  • par=.EACHI # special option available only with a join
  • DT1[!DT2, on, j] # anti-join two tables
  • DT1[DT2, on, roll, j] # join two tables, rolling on the last column in on=
  • ################# Remodeler, empiler et diviser
  • fusion (DT, id.vars, measure.vars) # transform to long format
    # for multiple columns, use measure.vars = patterns(…)
  • dcast(DT, formule) # transform to wide format
  • rlier(DT1, DT2, …) # stack enumerated data.tables
  • rbindlist(DT_list, idcol) # stack a list of data.tables
  • divisé(DT, par) # split a data.table into a list
  • ################# Quelques autres fonctions spécialisées pour data.tables
  • chevauchements # overlap joins
  • fusionner # another way of joining two tables
  • Positionner # another way of adding or modifying columns
  • fintersect, fsetdiff, funion, fsetequal, unique, dupliqué, anyDuplicated # set-theory operations with rows as elements
  • uniqueN # the number of distinct rows
  • rowidv(DT, colonnes) # row ID (1 to .N) within each group determined by cols
  • rleidv(DT, colonnes) # group ID (1 to .GRP) within each group determined by runs of cols
  • décalage(DT, n, type=c(“retard”, “avance”)) # apply a shift operator to every column
  • setorder, setcolorder, setnames, setkey, setindex, setattr # modify attributes and order by reference

Installation et assistance #

[Pour installer][1] le package data.table :

# install from CRAN
install.packages("data.table")       

# or install development version 
install.packages("data.table", type = "source", repos = "http://Rdatatable.github.io/data.table")

# and to revert from devel to CRAN, the current version must first be removed
remove.packages("data.table")
install.packages("data.table")

Le [site officiel] [2] du package contient des pages wiki fournissant de l’aide pour démarrer, ainsi que des listes de présentations et d’articles provenant du Web. Avant de poser une question - ici sur StackOverflow ou ailleurs - veuillez lire [la page d’assistance][3].

Chargement du colis #

La plupart des fonctions des exemples ci-dessus existent dans l’espace de noms data.table. Pour les utiliser, vous devrez d’abord ajouter une ligne comme library(data.table) ou utiliser leur chemin complet, comme data.table::fread au lieu de simplement fread. Pour obtenir de l’aide sur des fonctions individuelles, la syntaxe est help("fread") ou ?fread. Encore une fois, si le paquet n’est pas chargé, utilisez le nom complet comme ?data.table::fread.

[1] : https://github.com/Rdatatable/data.table/wiki/Installation [2] : https://github.com/Rdatatable/data.table/wiki [3] : https://github.com/Rdatatable/data.table/wiki/Support

Création d’un data.table #

Une data.table est une version améliorée de la classe data.frame de la base R. En tant que tel, son attribut class() est le vecteur "data.table" "data.frame" et les fonctions qui fonctionnent sur une donnée .frame fonctionnera également avec un data.table. Il existe de nombreuses façons de créer, de charger ou de contraindre un data.table.

Construire #

N’oubliez pas d’installer et d’activer le package data.table

library(data.table)

Il existe un constructeur du même nom :

DT <- data.table(
  x = letters[1:5], 
  y = 1:5, 
  z = (1:5) > 3
)
#    x y     z
# 1: a 1 FALSE
# 2: b 2 FALSE
# 3: c 3 FALSE
# 4: d 4  TRUE
# 5: e 5  TRUE

Contrairement à data.frame, data.table ne contraindra pas les chaînes aux facteurs :

sapply(DT, class)
#               x           y           z 
#     "character"   "integer"   "logical"

Lire dans #

On peut lire à partir d’un fichier texte :

dt <- fread("my_file.csv")

Contrairement à read.csv, fread lira les chaînes comme des chaînes et non comme des facteurs.

Modifier un data.frame #

Pour plus d’efficacité, data.table offre un moyen de modifier un data.frame ou une liste pour créer un data.table sur place (sans faire de copie ni modifier son emplacement mémoire):

# example data.frame
DF <- data.frame(x = letters[1:5], y = 1:5, z = (1:5) > 3)
# modification
setDT(DF)

Notez que nous n’attribuons pas <- le résultat, puisque l’objet DF a été modifié sur place. Les attributs de classe du data.frame seront conservés :

sapply(DF, class)
#         x         y         z 
#  "factor" "integer" "logical"

Contraindre l’objet à data.table #

** Si vous avez une list, data.frame ou data.table, vous devez utiliser la fonction setDT ** pour convertir en data.table car elle effectue la conversion par référence à la place de faire une copie (ce que as.data.table fait). Ceci est important si vous travaillez avec de grands ensembles de données.

Si vous avez un autre objet R (comme une matrice), vous devez utiliser as.data.table pour le contraindre à un data.table.

mat <- matrix(0, ncol = 10, nrow = 10)

DT <- as.data.table(mat)
# or
DT <- data.table(mat)

Symboles spéciaux dans data.table #

.DAKOTA DU SUD #

.SD fait référence au sous-ensemble de data.table pour chaque groupe, à l’exclusion de toutes les colonnes utilisées dans by.

.SD avec lapply peut être utilisé pour appliquer n’importe quelle fonction à plusieurs colonnes par groupe dans un data.table

Nous continuerons à utiliser le même ensemble de données intégré, mtcars :

mtcars = data.table(mtcars) # Let's not include rownames to keep things simpler

Moyenne de toutes les colonnes de l’ensemble de données par nombre de cylindres, cyl :

mtcars[ , lapply(.SD, mean), by = cyl]

#   cyl      mpg     disp        hp     drat       wt     qsec        vs        am     gear     carb
#1:   6 19.74286 183.3143 122.28571 3.585714 3.117143 17.97714 0.5714286 0.4285714 3.857143 3.428571
#2:   4 26.66364 105.1364  82.63636 4.070909 2.285727 19.13727 0.9090909 0.7272727 4.090909 1.545455
#3:   8 15.10000 353.1000 209.21429 3.229286 3.999214 16.77214 0.0000000 0.1428571 3.285714 3.500000

Outre cyl, il existe d’autres colonnes catégorielles dans l’ensemble de données telles que vs, am, gear et carb. Cela n’a pas vraiment de sens de prendre la “moyenne” de ces colonnes. Excluons donc ces colonnes. C’est là que .SDcols entre en scène.

.SDcols #

.SDcols spécifie les colonnes de data.table qui sont incluses dans .SD.

Moyenne de toutes les colonnes (colonnes continues) de l’ensemble de données par nombre de vitesses gear et nombre de cylindres, cyl, classés par gear et cyl :

# All the continuous variables in the dataset
cols_chosen <- c("mpg", "disp", "hp", "drat", "wt", "qsec")

mtcars[order(gear, cyl), lapply(.SD, mean), by = .(gear, cyl), .SDcols = cols_chosen]

#   gear cyl    mpg     disp       hp     drat       wt    qsec
#1:    3   4 21.500 120.1000  97.0000 3.700000 2.465000 20.0100
#2:    3   6 19.750 241.5000 107.5000 2.920000 3.337500 19.8300
#3:    3   8 15.050 357.6167 194.1667 3.120833 4.104083 17.1425
#4:    4   4 26.925 102.6250  76.0000 4.110000 2.378125 19.6125
#5:    4   6 19.750 163.8000 116.5000 3.910000 3.093750 17.6700
#6:    5   4 28.200 107.7000 102.0000 4.100000 1.826500 16.8000
#7:    5   6 19.700 145.0000 175.0000 3.620000 2.770000 15.5000
#8:    5   8 15.400 326.0000 299.5000 3.880000 3.370000 14.5500

Peut-être ne voulons-nous pas calculer la “moyenne” par groupes. Pour calculer la moyenne de toutes les voitures de l’ensemble de données, nous ne spécifions pas la variable “by”.

mtcars[ , lapply(.SD, mean), .SDcols = cols_chosen] 

#        mpg     disp       hp     drat      wt     qsec
#1: 20.09062 230.7219 146.6875 3.596563 3.21725 17.84875

Noter:

  • Il n’est pas nécessaire de définir cols_chosen au préalable. .SDcols peut prendre directement les noms de colonnes
  • .SDcols peut aussi prendre directement un vecteur de numéros de colonnes. Dans l’exemple ci-dessus, ce serait mtcars[ , lapply(.SD, mean), .SDcols = c(1,3:7)]

.N #

.N est un raccourci pour le nombre de lignes dans un groupe.

iris[, .(count=.N), by=Species]

#      Species count
#1:     setosa    50
#2: versicolor    50
#3:  virginica    50

Ajout et modification de colonnes #

La syntaxe DT[where, select|update|do, by] est utilisée pour travailler avec les colonnes d’un data.table.

  • La partie “où” est l’argument i
  • La partie “select|update|do” est l’argument j

Ces deux arguments sont généralement passés par position plutôt que par nom.

Notre exemple de données ci-dessous est

mtcars = data.table(mtcars, keep.rownames = TRUE)

Modification de colonnes entières #

Utilisez l’opérateur := à l’intérieur de j pour attribuer de nouvelles colonnes :

mtcars[, mpg_sq := mpg^2]

Supprimez les colonnes en définissant sur NULL :

mtcars[, mpg_sq := NULL]

Ajoutez plusieurs colonnes en utilisant le format multivarié de l’opérateur := :

mtcars[, `:=`(mpg_sq = mpg^2, wt_sqrt = sqrt(wt))]
# or 
mtcars[, c("mpg_sq", "wt_sqrt") := .(mpg^2, sqrt(wt))]

Si les colonnes sont dépendantes et doivent être définies dans l’ordre, une méthode est la suivante :

mtcars[, c("mpg_sq", "mpg2_hp") := .(temp1 <- mpg^2, temp1/hp)]

La syntaxe .() est utilisée lorsque le côté droit de LHS := RHS est une liste de colonnes.

Pour les noms de colonne déterminés dynamiquement, utilisez des parenthèses :

vn = "mpg_sq"
mtcars[, (vn) := mpg^2]

Les colonnes peuvent également être modifiées avec set, bien que cela soit rarement nécessaire :

set(mtcars, j = "hp_over_wt", v = mtcars$hp/mtcars$wt)

Modification de sous-ensembles de colonnes #

Utilisez l’argument i pour créer un sous-ensemble des lignes “où” les modifications doivent être effectuées :

mtcars[1:3, newvar := "Hello"]
# or
set(mtcars, j = "newvar", i = 1:3, v = "Hello")

Comme dans un data.frame, nous pouvons créer un sous-ensemble en utilisant des numéros de ligne ou des tests logiques. Il est également possible d’utiliser une “jointure” dans i, mais cette tâche plus compliquée est couverte dans un autre exemple.

Modification des attributs de colonne #

Les fonctions qui modifient les attributs, telles que levels<- ou names<-, remplacent en fait un objet par une copie modifiée. Même s’il n’est utilisé que sur une colonne dans un data.table, l’objet entier est copié et remplacé.

Pour modifier un objet sans copies, utilisez setnames pour changer les noms de colonne d’un data.table ou data.frame et setattr pour changer un attribut pour n’importe quel objet.

# Print a message to the console whenever the data.table is copied
tracemem(mtcars)
mtcars[, cyl2 := factor(cyl)]

# Neither of these statements copy the data.table
setnames(mtcars, old = "cyl2", new = "cyl_fac")
setattr(mtcars$cyl_fac, "levels", c("four", "six", "eight"))

# Each of these statements copies the data.table
names(mtcars)[names(mtcars) == "cyl_fac"] <- "cf"
levels(mtcars$cf) <- c("IV", "VI", "VIII")

Sachez que ces modifications sont apportées par référence, elles sont donc globales. Leur modification dans un environnement affecte l’objet dans tous les environnements.

# This function also changes the levels in the global environment
edit_levels <- function(x) setattr(x, "levels", c("low", "med", "high"))
edit_levels(mtcars$cyl_factor)

Écriture de code compatible avec data.frame et data.table #

Différences dans la syntaxe des sous-ensembles #

Une data.table est l’une des nombreuses structures de données bidimensionnelles disponibles dans R, en plus de data.frame, matrix et (2D) array. Toutes ces classes utilisent une syntaxe très similaire mais pas identique pour le sous-ensemble, le schéma A[rows, cols].

Considérez les données suivantes stockées dans une matrix, une data.frame et une data.table :

ma <- matrix(rnorm(12), nrow=4, dimnames=list(letters[1:4], c('X', 'Y', 'Z')))
df <- as.data.frame(ma)
dt <- as.data.table(ma)

ma[2:3]  #---> returns the 2nd and 3rd items, as if 'ma' were a vector (because it is!)
df[2:3]  #---> returns the 2nd and 3rd columns
dt[2:3]  #---> returns the 2nd and 3rd rows!

Si vous voulez être sûr de ce qui sera retourné, mieux vaut être explicite.

Pour obtenir des lignes spécifiques, ajoutez simplement une virgule après la plage :

ma[2:3, ]  # \
df[2:3, ]  #  }---> returns the 2nd and 3rd rows
dt[2:3, ]  # /

Mais, si vous souhaitez créer des sous-ensembles de colonnes, certains cas sont interprétés différemment. Tout trois peuvent être sous-ensembles de la même manière avec des indices entiers ou de caractères non stockés dans un variable.

ma[, 2:3]          #  \
df[, 2:3]          #   \
dt[, 2:3]          #    }---> returns the 2nd and 3rd columns
ma[, c("Y", "Z")]  #   /
df[, c("Y", "Z")]  #  /
dt[, c("Y", "Z")]  # /

Cependant, ils diffèrent pour les noms de variables sans guillemets

mycols <- 2:3
ma[, mycols]                # \
df[, mycols]                #  }---> returns the 2nd and 3rd columns
dt[, mycols, with = FALSE]  # /

dt[, mycols]                # ---> Raises an error

Dans le dernier cas, mycols est évalué comme le nom d’une colonne. Parce que dt ne peut pas trouver une colonne nommée mycols, une erreur est levée.

Remarque : Pour les versions du package data.table antérieures à 1.9.8, ce comportement était légèrement différent. Tout élément de l’index de colonne aurait été évalué en utilisant “dt” comme environnement. Ainsi, dt[, 2:3] et dt[, mycols] renverraient le vecteur 2:3. Non erreur serait générée pour le deuxième cas, car la variable “mycols” existe dans l’environnement parent.

Stratégies pour maintenir la compatibilité avec data.frame et data.table #

Il existe de nombreuses raisons d’écrire du code dont le fonctionnement est garanti avec data.frame et data.table. Peut-être êtes-vous obligé d’utiliser data.frame, ou vous devrez peut-être partager du code dont vous ne savez pas comment il sera utilisé. Il existe donc quelques stratégies principales pour y parvenir, par ordre de commodité :

  1. Utilisez une syntaxe qui se comporte de la même manière pour les deux classes.
  2. Utilisez une fonction commune qui fait la même chose que la syntaxe la plus courte.
  3. Forcez data.table à se comporter comme data.frame (ex. : appelez la méthode spécifique print.data.frame).
  4. Traitez-les comme une “liste”, ce qu’ils sont en fin de compte.
  5. Convertissez la table en data.frame avant de faire quoi que ce soit (mauvaise idée s’il s’agit d’une table énorme).
  6. Convertissez la table en data.table, si les dépendances ne sont pas un problème.

Lignes de sous-ensemble. C’est simple, utilisez simplement le sélecteur [, ], avec la virgule :

A[1:10, ]
A[A$var > 17, ]  # A[var > 17, ] just works for data.table

Colonnes de sous-ensemble. Si vous voulez une seule colonne, utilisez le sélecteur $ ou [[ ]] :

A$var
colname <- 'var'
A[[colname]]
A[[1]]

Si vous voulez une manière uniforme de saisir plus d’une colonne, il est nécessaire de faire appel un peu :

B <- `[.data.frame`(A, 2:4)

# We can give it a better name
select <- `[.data.frame`
B <- select(A, 2:4)
C <- select(A, c('foo', 'bar'))

Sous-ensemble de lignes ‘indexées’. Alors que data.frame a row.names, data.table a sa caractéristique unique key. La meilleure chose à faire est d’éviter entièrement row.names et de tirer parti des optimisations existantes dans le cas de data.table lorsque cela est possible.

B <- A[A$var != 0, ]
# or...
B <- with(A, A[var != 0, ])  # data.table will silently index A by var before subsetting

stuff <- c('a', 'c', 'f')
C <- A[match(stuff, A$name), ]  # really worse than: setkey(A); A[stuff, ]

Obtenez un tableau à 1 colonne, obtenez une ligne en tant que vecteur. C’est facile avec ce que nous avons vu jusqu’à présent :

B <- select(A, 2)    #---> a table with just the second column
C <- unlist(A[1, ])  #---> the first row as a vector (coerced if necessary)

Définition des clés dans data.table #

Oui, vous devez SETKEY avant 1.9.6

Dans le passé (avant 1.9.6), votre data.table était accéléré en définissant des colonnes comme clés de la table, en particulier pour les grandes tables. [Voir [intro vignette page 5][2] de la version de septembre 2015, où la vitesse de recherche était 544 fois meilleure.] mise en place du tableau.

library(data.table)
DT <- data.table(
  x = letters[1:5], 
  y = 5:1, 
  z = (1:5) > 3
)

#> DT
#   x y     z
#1: a 5 FALSE
#2: b 4 FALSE
#3: c 3 FALSE
#4: d 2  TRUE
#5: e 1  TRUE

Définissez votre clé avec la commande setkey. Vous pouvez avoir une clé avec plusieurs colonnes.

setkey(DT, y)

Vérifiez la clé de votre table dans tables()

tables()

> tables()
     NAME NROW NCOL MB COLS  KEY
[1,] DT      5    3  1 x,y,z y  
Total: 1MB

Notez que cela retriera vos données.

#> DT
#   x y     z
#1: e 1  TRUE
#2: d 2  TRUE
#3: c 3 FALSE
#4: b 4 FALSE
#5: a 5 FALSE

Maintenant c’est inutile

Avant la v1.9.6, vous deviez définir une clé pour certaines opérations, en particulier pour joindre des tables. Les développeurs de data.table ont accéléré et introduit une fonctionnalité "on=" qui peut remplacer la dépendance aux clés. Voir [SO réponse ici pour une discussion détaillée.] [2]

En janvier 2017, les développeurs ont écrit une [vignette autour des index secondaires][3] qui explique la syntaxe “on” et permet d’identifier d’autres colonnes pour une indexation rapide.

Création d’index secondaires ?

D’une manière similaire à key, vous pouvez setindex(DT, key.col) ou setindexv(DT, "key.col.string"), où DT est votre data.table. Supprimez tous les index avec setindex(DT, NULL).

Voir vos indices secondaires avec indices(DT).

Pourquoi des indices secondaires ?

Cela ** ne trie pas ** la table (contrairement à la clé), mais permet une indexation rapide à l’aide de la syntaxe “on”. Notez qu’il ne peut y avoir qu’une seule clé, mais vous pouvez utiliser plusieurs index secondaires, ce qui évite d’avoir à ressaisir et à reclasser la table. Cela accélérera votre sous-ensemble lors de la modification des colonnes sur lesquelles vous souhaitez créer un sous-ensemble.

Rappel, dans l’exemple ci-dessus, y était la clé de la table DT :

DT
# x y     z
# 1: e 1  TRUE
# 2: d 2  TRUE
# 3: c 3 FALSE
# 4: b 4 FALSE
# 5: a 5 FALSE

# Let us set x as index 
setindex(DT, x)

# Use indices to see what has been set
indices(DT)
# [1] "x"

# fast subset using index and not keyed column
DT["c", on ="x"]
#x y     z
#1: c 3 FALSE

# old way would have been rekeying DT from y to x, doing subset and 
# perhaps keying back to y (now we save two sorts)
# This is a toy example above but would have been more valuable with big data sets
