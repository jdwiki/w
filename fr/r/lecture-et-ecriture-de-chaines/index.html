Impression et affichage de chaînes #

R a plusieurs fonctions intégrées qui peuvent être utilisées pour imprimer ou afficher des informations, mais print et cat sont les plus basiques. Comme R est un [langage interprété][1], vous pouvez les essayer directement dans la console R :

print("Hello World")
#[1] "Hello World"
cat("Hello World\n")
#Hello World

Notez la différence d’entrée et de sortie pour les deux fonctions. (Remarque : il n’y a pas de guillemets dans la valeur de x créée avec x <- "Hello World". Ils sont ajoutés par print à l’étape de sortie.)

cat prend un ou plusieurs vecteurs de caractères comme arguments et les imprime sur la console. Si le vecteur caractère a une longueur supérieure à 1, les arguments sont séparés par un espace (par défaut) :

cat(c("hello", "world", "\n"))
#hello world

Sans le caractère de nouvelle ligne (\n), le résultat serait :

cat("Hello World")
#Hello World>

L’invite de la commande suivante apparaît immédiatement après la sortie. (Certaines consoles telles que RStudio peuvent ajouter automatiquement une nouvelle ligne aux chaînes qui ne se terminent pas par une nouvelle ligne.)

print est un exemple de fonction “générique”, ce qui signifie que la classe du premier argument passé est détectée et qu’une méthode spécifique à la classe est utilisée pour la sortie. Pour un vecteur de caractères comme "Hello World", le résultat est similaire à la sortie de cat. Cependant, la chaîne de caractères est entre guillemets et un nombre “[1]” est sorti pour indiquer le premier élément d’un vecteur de caractères (dans ce cas, le premier et unique élément) :

print("Hello World")
#[1] "Hello World"

Cette méthode d’impression par défaut est également ce que nous voyons lorsque nous demandons simplement à R d’imprimer une variable. Notez que la sortie de la saisie de s est la même que celle de l’appel de print(s) ou print("Hello World") :

s <- "Hello World"
s
#[1] "Hello World"

Ou même sans l’affecter à quoi que ce soit :

"Hello World"
#[1] "Hello World"

Si nous ajoutons une autre chaîne de caractères comme deuxième élément du vecteur (en utilisant la fonction c() pour concaténer les éléments ensemble), alors le comportement de print() semble assez différent de celui de ‘chat’ :

print(c("Hello World", "Here I am."))
#[1] "Hello World" "Here I am."

Observez que la fonction c() ne fait pas de concaténation de chaînes. (Il faut utiliser coller à cette fin.) R montre que le vecteur de caractères a deux éléments en les citant séparément. Si nous avons un vecteur suffisamment long pour s’étendre sur plusieurs lignes, R imprimera l’index de l’élément commençant chaque ligne, tout comme il imprime [1] au début de la première ligne.

c("Hello World", "Here I am!", "This next string is really long.")
#[1] "Hello World"                      "Here I am!"                      
#[3] "This next string is really long."

Le comportement particulier de print dépend de la classe de l’objet passé à la fonction.

Si nous appelons print un objet avec une classe différente, telle que “numeric” ou “logical”, les guillemets sont omis de la sortie pour indiquer que nous avons affaire à un objet qui n’est pas une classe de caractères :

print(1)
#[1] 1
print(TRUE)
#[1] TRUE

Les objets factoriels sont imprimés de la même manière que les variables de caractères, ce qui crée souvent une ambiguïté lorsque la sortie de la console est utilisée pour afficher des objets dans les corps de questions SO. Il est rare d’utiliser cat ou print sauf dans un contexte interactif. L’appel explicite de print() est particulièrement rare (sauf si vous vouliez supprimer l’apparence des guillemets ou afficher un objet renvoyé comme invisible par une fonction), car la saisie de foo sur la console est un raccourci pour print(foo). La console interactive de R est connue sous le nom de REPL, une “read-eval-print-loop”. La fonction cat est mieux enregistrée à des fins spéciales (comme écrire une sortie sur une connexion de fichier ouverte). Parfois, il est utilisé à l’intérieur des fonctions (où les appels à print() sont supprimés), mais utiliser cat() à l’intérieur d’une fonction pour générer une sortie vers la console est une mauvaise pratique. La méthode préférée est message() ou warning() pour les messages intermédiaires ; ils se comportent de la même manière que cat mais peuvent éventuellement être supprimés par l’utilisateur final. Le résultat final doit simplement être renvoyé afin que l’utilisateur puisse l’affecter pour le stocker si nécessaire.

message("hello world")
#hello world
suppressMessages(message("hello world"))

[1] : https://en.wikipedia.org/wiki/Interpreted_language

Capturer la sortie de la commande du système d’exploitation #

Fonctions qui renvoient un vecteur de caractère #

La base R a deux fonctions pour invoquer une commande système. Les deux nécessitent un paramètre supplémentaire pour capturer la sortie de la commande système.

system("top -a -b -n 1", intern = TRUE) 
system2("top", "-a -b -n 1", stdout = TRUE)

Les deux renvoient un vecteur de caractères.

  [1] "top - 08:52:03 up 70 days, 15:09,  0 users,  load average: 0.00, 0.00, 0.00"     
  [2] "Tasks: 125 total,   1 running, 124 sleeping,   0 stopped,   0 zombie"            
  [3] "Cpu(s):  0.9%us,  0.3%sy,  0.0%ni, 98.7%id,  0.1%wa,  0.0%hi,  0.0%si,  0.0%st"  
  [4] "Mem:  12194312k total,  3613292k used,  8581020k free,   216940k buffers"        
  [5] "Swap: 12582908k total,  2334156k used, 10248752k free,  1682340k cached"         
  [6] ""                                                                                
  [7] "  PID USER      PR  NI  VIRT  RES  SHR S %CPU %MEM    TIME+  COMMAND            "
  [8] "11300 root      20   0 1278m 375m 3696 S  0.0  3.2 124:40.92 trala              "
  [9] " 6093 user1     20   0 1817m 269m 1888 S  0.0  2.3  12:17.96 R                  "
 [10] " 4949 user2     20   0 1917m 214m 1888 S  0.0  1.8  11:16.73 R                  "

Pour illustration, la commande UNIX top -a -b -n 1 est utilisée. Ceci est spécifique au système d’exploitation et peut devoir être modifié pour exécuter les exemples sur votre ordinateur.

Le package devtools a une fonction pour exécuter une commande système et capturer la sortie sans paramètre supplémentaire. Il renvoie également un vecteur de caractères.

devtools::system_output("top", "-a -b -n 1")

Fonctions qui renvoient une trame de données #

La fonction fread du package data.table permet d’exécuter une commande shell et de lire la sortie comme read.table. Il retourne un data.table ou un data.frame.

fread("top -a -b -n 1", check.names = TRUE)
       PID     USER PR NI  VIRT  RES  SHR S X.CPU X.MEM     TIME.         COMMAND
  1: 11300     root 20  0 1278m 375m 3696 S     0   3.2 124:40.92           trala
  2:  6093    user1 20  0 1817m 269m 1888 S     0   2.3  12:18.56               R
  3:  4949    user2 20  0 1917m 214m 1888 S     0   1.8  11:17.33               R
  4:  7922    user3 20  0 3094m 131m 1892 S     0   1.1  21:04.95               R

Notez que fread a automatiquement sauté les 6 premières lignes d’en-tête.

Ici, le paramètre check.names = TRUE a été ajouté pour convertir %CPU, %MEN et TIME+ en noms de colonnes syntaxiquement valides.

Lecture ou écriture sur une connexion de fichier #

Nous n’avons pas toujours la liberté de lire ou d’écrire dans un chemin système local. Par exemple, si le streaming de code R map-reduce doit avoir besoin de lire et d’écrire dans la connexion au fichier. Il peut également y avoir d’autres scénarios où l’on va au-delà du système local et avec l’avènement du cloud et du big data, cela devient de plus en plus courant. L’une des façons de le faire est de suivre une séquence logique.

Établissez une connexion de fichier à lire avec la commande file() (“r” est pour le mode lecture) :

conn <- file("/path/example.data", "r") #when file is in local system
conn1 <- file("stdin", "r") #when just standard input/output for files are available

Comme cela n’établira qu’une connexion de fichier, on peut lire les données de ces connexions de fichier comme suit :

line <- readLines(conn, n=1, warn=FALSE)

Ici, nous lisons les données de la connexion au fichier conn ligne par ligne sous la forme n = 1. on peut changer la valeur de n (disons 10, 20 etc.) pour lire des blocs de données pour une lecture plus rapide (bloc de 10 ou 20 lignes lu en une seule fois). Pour lire le fichier complet en une seule fois, définissez n=-1.

Après le traitement des données ou dire l’exécution du modèle ; on peut réécrire les résultats dans la connexion de fichier en utilisant de nombreuses commandes différentes telles que writeLines(), cat() etc. qui sont capables d’écrire dans une connexion de fichier. Cependant, toutes ces commandes tireront parti de la connexion de fichier établie pour l’écriture. Cela pourrait être fait en utilisant la commande file() comme :

conn2 <- file("/path/result.data", "w") #when file is in local system
conn3 <- file("stdout", "w") #when just standard input/output for files are available

Ensuite, écrivez les données comme suit :

writeLines("text",conn2, sep = "\n")
