Étant donné un objet R, nous pouvons exiger une analyse séparée pour une ou plusieurs parties des données qu’il contient. Le processus d’obtention de ces parties de données à partir d’un objet donné est appelé “sous-ensemble”.

Valeurs manquantes :

Les valeurs manquantes (NAs) utilisées dans le sous-ensemble avec [ renvoient NA depuis un index NA

sélectionne un élément inconnu et renvoie donc NA dans l’élément correspondant.

Le type “par défaut” de NA est “logique” (typeof(NA)) ce qui signifie que, comme tout vecteur “logique” utilisé dans le sous-ensemble, sera recyclé pour correspondre à la longueur de l’objet sous-ensemble . Donc x[NA] est équivalent à x[as.logical(NA)] qui est équivalent à x[rep_len(as.logical(NA), length(x))] et, par conséquent, il renvoie une valeur manquante (NA) pour chaque élément de x. Par exemple:

x <- 1:3
x[NA]
## [1] NA NA NA

Lors de l’indexation avec “numérique”/“entier”, NA sélectionne un seul élément NA (pour chaque NA dans l’index):

x[as.integer(NA)]
## [1] NA

x[c(NA, 1, NA, NA)]
## [1] NA  1 NA NA

Sous-ensemble hors limites :

L’opérateur [, avec un argument passé, autorise les indices qui sont > longueur(x) et renvoie NA pour les vecteurs atomiques ou NULL pour les vecteurs génériques. En revanche, avec [[ et lorsque [ reçoit plus d’arguments (c’est-à-dire un sous-ensemble d’objets hors limites avec length(dim(x)) > 2), une erreur est renvoyée :

(1:3)[10]
## [1] NA
(1:3)[[10]]
## Error in (1:3)[[10]] : subscript out of bounds
as.matrix(1:3)[10]
## [1] NA
as.matrix(1:3)[, 10]
## Error in as.matrix(1:3)[, 10] : subscript out of bounds
list(1, 2, 3)[10]
## [[1]]
## NULL
list(1, 2, 3)[[10]]
## Error in list(1, 2, 3)[[10]] : subscript out of bounds

Le comportement est le même lors d’un sous-ensemble avec des vecteurs “character”, qui ne correspondent pas non plus dans l’attribut “names” de l’objet :

c(a = 1, b = 2)["c"]
## <NA> 
##   NA 
list(a = 1, b = 2)["c"]
## <NA>
## NULL

Sujets d’aide:

Voir ?Extraire pour plus d’informations.

Trames de données #

Le sous-ensemble d’un bloc de données dans un bloc de données plus petit peut être accompli de la même manière que le sous-ensemble d’une liste.

> df3 <- data.frame(x = 1:3, y = c("a", "b", "c"), stringsAsFactors = FALSE)

> df3
##   x y
## 1 1 a
## 2 2 b
## 3 3 c

> df3[1]   # Subset a variable by number
##   x
## 1 1
## 2 2
## 3 3

> df3["x"]   # Subset a variable by name 
##   x
## 1 1
## 2 2
## 3 3

> is.data.frame(df3[1])
## TRUE

> is.list(df3[1])
## TRUE

Le sous-ensemble d’une trame de données dans un vecteur de colonne peut être réalisé à l’aide de doubles crochets [[ ]] ou de l’opérateur de signe dollar $.

> df3[[2]]    # Subset a variable by number using [[ ]]
## [1] "a" "b" "c"

> df3[["y"]]  # Subset a variable by name using [[ ]]
## [1] "a" "b" "c"

> df3$x    # Subset a variable by name using $
## [1] 1 2 3

> typeof(df3$x)
## "integer"

> is.vector(df3$x)
## TRUE

Définir une donnée sous forme de matrice bidimensionnelle peut être accompli en utilisant les termes i et j.

> df3[1, 2]    # Subset row and column by number
## [1] "a"

> df3[1, "y"]  # Subset row by number and column by name
## [1] "a"

> df3[2, ]     # Subset entire row by number  
##   x y
## 2 2 b

> df3[ , 1]    # Subset all first variables 
## [1] 1 2 3

> df3[ , 1, drop = FALSE]
##   x
## 1 1
## 2 2
## 3 3

Remarque : le sous-ensemble par j (colonne) seul simplifie le propre type de la variable, mais le sous-ensemble par i seul renvoie un data.frame, car les différentes variables peuvent avoir des types et des classes différents. Définir le paramètre drop sur FALSE conserve la trame de données.

> is.vector(df3[, 2])
## TRUE

> is.data.frame(df3[2, ])
## TRUE

> is.data.frame(df3[, 2, drop = FALSE])
## TRUE

Vecteurs atomiques #

Les vecteurs atomiques (qui excluent les listes et les expressions, qui sont également des vecteurs) sont sous-ensembles à l’aide de l’opérateur [ :

# create an example vector
v1 <- c("a", "b", "c", "d")

# select the third element
v1[3]
## [1] "c"

L’opérateur [ peut également prendre un vecteur comme argument. Par exemple, pour sélectionner les premier et troisième éléments :

v1 <- c("a", "b", "c", "d")

v1[c(1, 3)]
## [1] "a" "c"

Parfois, nous pouvons avoir besoin d’omettre une valeur particulière du vecteur. Ceci peut être réalisé en utilisant un signe négatif (-) avant l’index de cette valeur. Par exemple, pour omettre d’omettre la première valeur de v1, utilisez v1[-1]. Cela peut être étendu à plus d’une valeur de manière simple. Par exemple, v1[-c(1,3)].

> v1[-1]
[1] "b" "c" "d"
> v1[-c(1,3)]
[1] "b" "d"

A certaines occasions, on aimerait savoir, notamment, lorsque la longueur du vecteur est grande, indice d’une valeur particulière, s’il existe :

> v1=="c"
[1] FALSE FALSE  TRUE FALSE
> which(v1=="c")
[1] 3

Si le vecteur atomique a des noms (un attribut names), il peut être sous-ensemble en utilisant un vecteur de caractères de noms :

v <- 1:3
names(v) <- c("one", "two", "three")

v
##  one   two three 
##    1     2     3 

v["two"]
## two 
##   2

L’opérateur [[ peut également être utilisé pour indexer des vecteurs atomiques, avec des différences en ce sens qu’il accepte un vecteur d’indexation d’une longueur de un et supprime tous les noms présents :

v[[c(1, 2)]]
## Error in v[[c(1, 2)]] : 
##  attempt to select more than one element in vectorIndex

v[["two"]]
## [1] 2

Les vecteurs peuvent également être sous-ensembles à l’aide d’un vecteur logique. Contrairement au sous-ensemble avec des vecteurs numériques et de caractères, le vecteur logique utilisé pour le sous-ensemble doit être égal à la longueur du vecteur dont les éléments sont extraits, donc si un vecteur logique ‘y’ est utilisé pour sous-ensemble ‘x’, c’est-à-dire ‘x [y], si length(y) < length(x)alorsysera recyclé pour correspondre àlength(x)` :

v[c(TRUE, FALSE, TRUE)]
##  one three 
##    1     3 

v[c(FALSE, TRUE)]  # recycled to 'c(FALSE, TRUE, FALSE)'
## two 
##   2 

v[TRUE]   # recycled to 'c(TRUE, TRUE, TRUE)'
##  one   two three 
##    1     2     3 

v[FALSE]   # handy to discard elements but save the vector's type and basic structure
## named integer(0)

Matrices #

Pour chaque dimension d’un objet, l’opérateur [ prend un argument. Les vecteurs ont une dimension et prennent un argument. Les matrices et les blocs de données ont deux dimensions et prennent deux arguments, donnés par [i, j] où i est la ligne et j est la colonne. L’indexation commence à 1.

## a sample matrix
mat <- matrix(1:6, nrow = 2, dimnames = list(c("row1", "row2"), c("col1", "col2", "col3")))

mat
#      col1 col2 col3
# row1    1    3    5
# row2    2    4    6

mat[i,j] est l’élément de la i-ème ligne, j-ème colonne de la matrice mat. Par exemple, une valeur i de 2 et une valeur j de 1 donnent le nombre dans la deuxième ligne et la première colonne de la matrice. L’omission de i ou j renvoie toutes les valeurs de cette dimension.

mat[ , 3]
## row1 row2 
##    5    6 

mat[1, ]
# col1 col2 col3 
#    1    3    5

Lorsque la matrice a des noms de lignes ou de colonnes (non obligatoires), ceux-ci peuvent être utilisés pour le sous-ensemble :

mat[ , 'col1']
# row1 row2 
#    1    2

Par défaut, le résultat d’un sous-ensemble sera simplifié si possible. Si le sous-ensemble n’a qu’une seule dimension, comme dans les exemples ci-dessus, le résultat sera un vecteur unidimensionnel plutôt qu’une matrice bidimensionnelle. Cette valeur par défaut peut être remplacée par l’argument drop = FALSE à [ :

## This selects the first row as a vector
class(mat[1, ])
# [1] "integer"

## Whereas this selects the first row as a 1x3 matrix:
class(mat[1, , drop = F])
# [1] "matrix"

Bien sûr, les dimensions ne peuvent pas être supprimées si la sélection elle-même a deux dimensions :

mat[1:2, 2:3]  ## A 2x2 matrix
#      col2 col3
# row1    3    5
# row2    4    6

Sélection des entrées individuelles de la matrice par leurs positions #

Il est également possible d’utiliser une matrice Nx2 pour sélectionner N éléments individuels à partir d’une matrice (comme le fonctionnement d’un système de coordonnées). Si vous vouliez extraire, dans un vecteur, les entrées d’une matrice dans le (1ère ligne, 1ère colonne), (1ère ligne, 3ème colonne), (2ème ligne, 3ème colonne), (2ème ligne, 1ère colonne) cela peut être fait facilement en créant une matrice d’index avec ces coordonnées et en l’utilisant pour créer un sous-ensemble de la matrice :

mat
#      col1 col2 col3
# row1    1    3    5
# row2    2    4    6

ind = rbind(c(1, 1), c(1, 3), c(2, 3), c(2, 1))
ind
#      [,1] [,2]
# [1,]    1    1
# [2,]    1    3
# [3,]    2    3
# [4,]    2    1

mat[ind]
# [1] 1 5 6 2

Dans l’exemple ci-dessus, la 1ère colonne de la matrice ind fait référence aux lignes dans mat, la 2ème colonne de ind fait référence aux colonnes dans mat.

Listes #

Une liste peut être sous-ensemble avec [ :

l1 <- list(c(1, 2, 3), 'two' = c("a", "b", "c"), list(10, 20))
l1
## [[1]]
## [1] 1 2 3
## 
## $two
## [1] "a" "b" "c"
##
## [[3]]
## [[3]][[1]]
## [1] 10
##
## [[3]][[2]]
## [1] 20

l1[1]
## [[1]]
## [1] 1 2 3

l1['two']
## $two
## [1] "a" "b" "c"

l1[[2]]
## [1] "a" "b" "c"

l1[['two']]
## [1] "a" "b" "c"

Notez que le résultat de l1[2] est toujours une liste, car l’opérateur [ sélectionne les éléments d’une liste, renvoyant une liste plus petite. L’opérateur [[ extrait les éléments de la liste, renvoyant un objet du type de l’élément de la liste.

Les éléments peuvent être indexés par un numéro ou une chaîne de caractères du nom (s’il existe). Plusieurs éléments peuvent être sélectionnés avec [ en passant un vecteur de nombres ou des chaînes de noms. L’indexation avec un vecteur de longueur > 1 dans [ et [[ renvoie une “liste” avec les éléments spécifiés et un sous-ensemble récursif (si disponible), respectivement :

l1[c(3, 1)]
## [[1]]
## [[1]][[1]]
## [1] 10
## 
## [[1]][[2]]
## [1] 20
## 
## 
## [[2]]
## [1] 1 2 3

Par rapport à:

l1[[c(3, 1)]]
## [1] 10

ce qui équivaut à :

l1[[3]][[1]]
## [1] 10

L’opérateur $ vous permet de sélectionner les éléments de la liste uniquement par leur nom, mais contrairement à [ et [[, ne nécessite pas de guillemets. En tant qu’opérateur infixe, $ ne peut prendre qu’un seul nom :

l1$two
## [1] "a" "b" "c"

De plus, l’opérateur $ permet une correspondance partielle par défaut :

l1$t
## [1] "a" "b" "c"

contrairement à [[ où il faut spécifier si la correspondance partielle est autorisée :

l1[["t"]]
## NULL
l1[["t", exact = FALSE]]
## [1] "a" "b" "c"

En définissant options(warnPartialMatchDollar = TRUE), un “avertissement” est donné lorsqu’une correspondance partielle se produit avec $ :

l1$t
## [1] "a" "b" "c"
## Warning message:
## In l1$t : partial match of 't' to 'two'

Autres objets #

Les opérateurs [ et [[ sont des fonctions primitives génériques. Cela signifie que tout object dans R (en particulier isTRUE(is.object(x)) –c’est-à-dire a un attribut “class” explicite) peut avoir son propre comportement spécifié lorsqu’il est sous-ensemble ; c’est-à-dire qu’il a ses propres méthodes pour [ et/ou [[.

Par exemple, c’est le cas avec les objets “data.frame” (is.object(iris)) où les méthodes [.data.frame et [[.data.frame sont définies et sont conçues pour exposer sous-ensemble de type “matrice” et de type “liste”. En forçant une erreur lors du sous-ensemble d’un “data.frame”, nous voyons qu’en fait, une fonction [.data.frame a été appelée alors que nous -venions- d’utiliser [.

iris[invalidArgument, ]
## Error in `[.data.frame`(iris, invalidArgument, ) : 
##   object 'invalidArgument' not found

Sans plus de détails sur le sujet en cours, un exemple de méthode [ :

x = structure(1:5, class = "myClass")
x[c(3, 2, 4)]
## [1] 3 2 4
'[.myClass' = function(x, i) cat(sprintf("We'd expect '%s[%s]' to be returned but this a custom `[` method and should have a `?[.myClass` help page for its behaviour\n", deparse(substitute(x)), deparse(substitute(i))))

x[c(3, 2, 4)]
## We'd expect 'x[c(3, 2, 4)]' to be returned but this a custom `[` method and should have a `?[.myClass` help page for its behaviour
## NULL

Nous pouvons surmonter la méthode dispatching de [ en utilisant l’équivalent non générique .subset (et .subset2 pour [[). Ceci est particulièrement utile et efficace lors de la programmation de nos propres “classes” et que vous souhaitez éviter les solutions de contournement (comme unclass (x)) lors du calcul efficace sur nos “classes” (en évitant l’envoi de méthodes et la copie d’objets):

.subset(x, c(3, 2, 4))
## [1] 3 2 4

Opérations matricielles élémentaires #

Soient A et B deux matrices de même dimension. Les opérateurs +,-,/,*,^, lorsqu’ils sont utilisés avec des matrices de même dimension, effectuent les opérations requises sur les éléments correspondants des matrices et renvoient une nouvelle matrice de même dimension. Ces opérations sont généralement appelées opérations élément par élément.

Opérateur A sur B Signification
+ A + B Addition des éléments correspondants de A et B
- A - B Soustrait les éléments de B des éléments correspondants de A
/ A/B Divise les éléments de A par les éléments correspondants de B
* A * B Multiplie les éléments de A par les éléments correspondants de B
^ A^(-1) Par exemple, donne une matrice dont les éléments sont réciproques de A

Pour une “vraie” multiplication matricielle, comme dans Algèbre linéaire, utilisez %*%. Par exemple, la multiplication de A par B est : A %*% B. Les exigences dimensionnelles sont que le ncol() de A soit le même que nrow() de B
