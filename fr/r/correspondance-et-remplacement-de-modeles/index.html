Cette rubrique couvre les modèles de chaîne correspondants, ainsi que leur extraction ou leur remplacement. Pour plus de détails sur la définition de modèles compliqués, voir [Expressions régulières][1].

[1] : https://www.wikiod.com/fr/r/expressions-regulieres-regex

Syntaxe #

  • grep(“requête”, “sujet”, arguments_optionnels)

  • grpl(“requête”, “sujet”, arguments_optionnels)

  • gsub("(groupe1)(groupe2)", “\\groupe#”, “sujet”)

Différences avec d’autres langues #

Les symboles [regex][1] échappés (comme \1) doivent être échappés une deuxième fois (comme \\1), non seulement dans l’argument pattern, mais aussi dans le replacement à sub et gsub.

Par défaut, le modèle pour toutes les commandes (grep, sub, regexpr) n’est pas une expression régulière compatible Perl (PCRE), donc certaines choses comme les contournements ne sont pas prises en charge. Cependant, chaque fonction accepte un argument perl=TRUE pour les activer. Voir la rubrique [R Regular Expressions topic][2] pour plus de détails.

Forfaits spécialisés #

  • [chaîne][3]
  • corde

[1] : https://www.wikiod.com/fr/regex [2] : https://www.wikiod.com/fr/r/expressions-regulieres-regex [3] : https://www.wikiod.com/fr/r/manipulation-de-chaines-avec-le-paquet-stringi

Trouver des correspondances #

# example data
test_sentences <- c("The quick brown fox", "jumps over the lazy dog")

Y a-t-il une correspondance ? #

grepl() est utilisé pour vérifier si un mot ou une expression régulière existe dans une chaîne ou un vecteur de caractères. La fonction renvoie un vecteur VRAI/FAUX (ou “Booléen”).

Notez que nous pouvons vérifier chaque chaîne pour le mot “renard” et recevoir un vecteur booléen en retour.

grepl("fox", test_sentences)
#[1]  TRUE FALSE

Emplacements correspondants #

grep prend une chaîne de caractères et une expression régulière. Il renvoie un vecteur numérique d’index. Cela renverra quelle phrase contient le mot “renard”.

grep("fox", test_sentences)
#[1] 1

Valeurs correspondantes #

Pour sélectionner des phrases qui correspondent à un modèle :

# each of the following lines does the job:
test_sentences[grep("fox", test_sentences)]
test_sentences[grepl("fox", test_sentences)]
grep("fox", test_sentences, value = TRUE)
# [1] "The quick brown fox"

Détails #

Étant donné que le modèle "fox" est juste un mot, plutôt qu’une expression régulière, nous pourrions améliorer les performances (avec grep ou grepl) en spécifiant fixed = TRUE.

grep("fox", test_sentences, fixed = TRUE)
#[1] 1

Pour sélectionner des phrases qui ne correspondent pas à un motif, on peut utiliser grep avec invert = TRUE ; ou suivez les règles de [sous-ensemble][1] avec -grep(...) ou !grepl(...).

Dans grepl(pattern, x) et grep(pattern, x), le paramètre x est [vectorisé][2], le paramètre pattern ne l’est pas. Par conséquent, vous ne pouvez pas les utiliser directement pour faire correspondre pattern[1] à x[1], pattern[2] à x[2], etc.

Résumé des matchs #

Après avoir effectué par ex. la commande grepl, peut-être souhaitez-vous avoir un aperçu du nombre de correspondances où TRUE ou FALSE. Ceci est utile par ex. en cas de grands ensembles de données. Pour ce faire, exécutez la commande summary :

# example data
test_sentences <- c("The quick brown fox", "jumps over the lazy dog") 

# find matches
matches <- grepl("fox", test_sentences)

# overview
summary(matches)

[1] : https://www.wikiod.com/fr/r/sous-ensemble [2] : https://www.wikiod.com/fr/r/bonnes-pratiques-de-vectorisation-du-code-r

Correspondance unique et globale. #

Lorsque vous travaillez avec des expressions régulières, un modificateur pour PCRE est g pour une correspondance globale.

Dans R, les fonctions de correspondance et de remplacement ont deux versions : première correspondance et correspondance globale :

  • sub(pattern,replacement,text) remplacera la première occurrence de pattern par remplacement dans le texte

  • gsub(pattern,replacement,text) fera la même chose que sub mais pour chaque occurrence de pattern

  • regexpr(pattern,text) renverra la position de correspondance pour la première instance de pattern

  • gregexpr(pattern,text) renverra toutes les correspondances.

Quelques données aléatoires :

set.seed(123)
teststring <- paste0(sample(letters,20),collapse="")

# teststring
#[1] "htjuwakqxzpgrsbncvyo"

Voyons comment cela fonctionne si nous voulons remplacer les voyelles par autre chose :

sub("[aeiouy]"," ** HERE WAS A VOWEL** ",teststring)
#[1] "htj ** HERE WAS A VOWEL** wakqxzpgrsbncvyo"

gsub("[aeiouy]"," ** HERE WAS A VOWEL** ",teststring)
#[1] "htj ** HERE WAS A VOWEL** w ** HERE WAS A VOWEL** kqxzpgrsbncv ** HERE WAS A VOWEL**  ** HERE WAS A VOWEL** "

Voyons maintenant comment trouver une consonne immédiatement suivie d’une ou plusieurs voyelles :

regexpr("[^aeiou][aeiou]+",teststring)
#[1] 3
#attr(,"match.length")
#[1] 2
#attr(,"useBytes")
#[1] TRUE

Nous avons une correspondance sur la position 3 de la chaîne de longueur 2, c’est-à-dire : ju

Maintenant, si nous voulons obtenir toutes les correspondances :

gregexpr("[^aeiou][aeiou]+",teststring)
#[[1]]
#[1]  3  5 19
#attr(,"match.length")
#[1] 2 2 2
#attr(,"useBytes")
#[1] TRUE

Tout cela est vraiment génial, mais cela ne donne que des positions d’utilisation de correspondance et ce n’est pas si facile d’obtenir ce qui correspond, et voici regmatches son seul but est d’extraire la chaîne correspondante de regexpr, mais il a une syntaxe différente.

Enregistrons nos correspondances dans une variable, puis extrayons-les de la chaîne d’origine :

matches <- gregexpr("[^aeiou][aeiou]+",teststring)
regmatches(teststring,matches)
#[[1]]
#[1] "ju" "wa" "yo"

Cela peut sembler étrange de ne pas avoir de raccourci, mais cela permet l’extraction d’une autre chaîne par les correspondances de notre premier (pensez à comparer deux longs vecteurs où vous savez qu’il y a un modèle commun pour le premier mais pas pour le second, cela permet un comparaison facile) :

teststring2 <- "this is another string to match against"
regmatches(teststring2,matches)
#[[1]]
#[1] "is" " i" "ri"

Attention : par défaut, le modèle n’est pas une expression régulière compatible Perl, certaines choses comme les contournements ne sont pas pris en charge, mais chaque fonction présentée ici permet l’argument perl=TRUE pour les activer.

Faire des substitutions #

# example data
test_sentences <- c("The quick brown fox quickly", "jumps over the lazy dog")

Rendons le renard brun rouge :

sub("brown","red", test_sentences)
#[1] "The quick red fox quickly"       "jumps over the lazy dog"

Maintenant, faisons agir le renard "rapide" "rapidement". Cela ne le fera pas :

sub("quick", "fast", test_sentences)
#[1] "The fast red fox quickly"       "jumps over the lazy dog"

sub ne fait que le premier remplacement disponible, nous avons besoin de gsub pour [remplacement global] [1] :

gsub("quick", "fast", test_sentences)
#[1] "The fast red fox fastly"       "jumps over the lazy dog"

Voir https://www.wikiod.com/fr/r/modification-de-chaines-par-substitution pour plus d’exemples.

[1] : https://www.wikiod.com/fr/r/correspondance-et-remplacement-de-modeles

## Trouvez des correspondances dans des ensembles de données volumineuses Dans le cas d’ensembles de données volumineux, l’appel de grepl("fox", test_sentences) ne fonctionne pas bien. Les grands ensembles de données sont par ex. sites Web explorés ou millions de Tweets, etc.

La première accélération est l’utilisation de l’option perl = TRUE. L’option fixed = TRUE est encore plus rapide. Un exemple complet serait :

# example data
test_sentences <- c("The quick brown fox", "jumps over the lazy dog")

grepl("fox", test_sentences, perl = TRUE)
#[1]  TRUE FALSE

En cas de fouille de texte, c’est souvent un corpus qui est utilisé. Un corpus ne peut pas être utilisé directement avec grepl. Par conséquent, considérez cette fonction :

searchCorpus <- function(corpus, pattern) {
  return(tm_index(corpus, FUN = function(x) {
    grepl(pattern, x, ignore.case = TRUE, perl = TRUE)
  }))
}
