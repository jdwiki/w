Syntaxe #

  • lm(formula, data, subset, weights, na.action, method = “qr”, model = TRUE, x = FALSE, y = FALSE, qr = TRUE, singular.ok = TRUE, contrasts = NULL, offset, . ..)

Paramètres #

Paramètre Signification
formule une formule en notation Wilkinson-Rogers ; response ~ ... où ... contient des termes correspondant aux variables de l’environnement ou du bloc de données spécifié par l’argument data
données bloc de données contenant les variables de réponse et de prédiction
sous-ensemble un vecteur spécifiant un sous-ensemble d’observations à utiliser : peut être exprimé sous la forme d’une déclaration logique en termes de variables dans data
poids pondérations analytiques (voir la section Poids ci-dessus)
na.action comment gérer les valeurs manquantes (NA) : voir ?na.action
méthode comment effectuer le montage. Seuls les choix sont "qr" ou "model.frame" (ce dernier renvoie le cadre du modèle sans ajuster le modèle, identique à la spécification de model=TRUE)
modèle s’il faut stocker le cadre du modèle dans l’objet ajusté
x s’il faut stocker la matrice de modèle dans l’objet ajusté
y s’il faut stocker la réponse du modèle dans l’objet ajusté
QR s’il faut stocker la décomposition QR dans l’objet ajusté
singulier.ok s’il faut autoriser les ajustements singuliers, les modèles avec des prédicteurs colinéaires (un sous-ensemble des coefficients sera automatiquement défini sur NA dans ce cas
contrastes une liste de contrastes à utiliser pour des facteurs particuliers dans le modèle ; voir l’argument contrasts.arg de ?model.matrix.default. Les contrastes peuvent également être définis avec options() (voir l’argument contrasts) ou en affectant les attributs contrast d’un facteur (voir ?contrasts)
décalage utilisé pour spécifier un composant connu a priori dans le modèle. Peut également être spécifié dans le cadre de la formule. Voir ?model.offset
… arguments supplémentaires à transmettre aux fonctions d’ajustement de niveau inférieur (lm.fit() ou lm.wfit())

Régression linéaire sur le jeu de données mtcars #

Le mtcars intégré data frame contient des informations sur 32 voitures, y compris leur poids, leur consommation de carburant (en miles par gallon), vitesse, etc. (Pour en savoir plus sur le jeu de données, utilisez help(mtcars)).

Si nous sommes intéressés par la relation entre l’efficacité énergétique (mpg) et le poids (wt), nous pouvons commencer à tracer ces variables avec :

plot(mpg ~ wt, data = mtcars, col=2)

Les tracés montrent une relation (linéaire) !. Ensuite, si nous voulons effectuer une régression linéaire pour déterminer les coefficients d’un modèle linéaire, nous utiliserons la fonction lm :

fit <- lm(mpg ~ wt, data = mtcars)

Le ~ ici signifie “expliqué par”, donc la formule mpg ~ wt signifie que nous prédisons mpg comme expliqué par wt. Le moyen le plus utile d’afficher la sortie consiste à :

summary(fit)

Ce qui donne la sortie :

Celui-ci fournit des informations sur :

  • la pente estimée de chaque coefficient (wt et l’ordonnée à l’origine), ce qui suggère que la prédiction la mieux adaptée de mpg est 37,2851 + (-5,3445) * wt
  • La valeur p de chaque coefficient, ce qui suggère que l’ordonnée à l’origine et le poids ne sont probablement pas dus au hasard
  • Estimations globales de l’ajustement telles que R^2 et R^2 ajusté, qui montrent dans quelle mesure la variation en “mpg” est expliquée par le modèle

Nous pourrions ajouter une ligne à notre premier graphique pour afficher le mpg prédit :

abline(fit,col=3,lwd=2)

Il est également possible d’ajouter l’équation à ce tracé. Tout d’abord, obtenez les coefficients avec coef. Ensuite, en utilisant paste0, nous combinons les coefficients avec les variables appropriées et +/-, pour construire l’équation. Enfin, nous l’ajoutons au tracé en utilisant mtext :

bs <- round(coef(fit), 3) 
lmlab <- paste0("mpg = ", bs[1],
             ifelse(sign(bs[2])==1, " + ", " - "), abs(bs[2]), " wt ")
mtext(lmlab, 3, line=-2)

Pondération #

Parfois, nous voulons que le modèle donne plus de poids à certains points de données ou exemples qu’à d’autres. Ceci est possible en spécifiant le poids des données d’entrée lors de l’apprentissage du modèle. Il existe généralement deux types de scénarios dans lesquels nous pourrions utiliser des poids non uniformes sur les exemples :

  • Pondérations analytiques : reflètent les différents niveaux de précision des différentes observations. Par exemple, si vous analysez des données où chaque observation est la moyenne des résultats d’une zone géographique, le poids analytique est proportionnel à l’inverse de la variance estimée. Utile lorsqu’il s’agit de moyennes dans les données en fournissant un poids proportionnel compte tenu du nombre d’observations. Source

  • Poids d’échantillonnage (Inverse Probability Weights - IPW) : technique statistique de calcul de statistiques standardisées à une population différente de celle dans laquelle les données ont été collectées. Les plans d’étude avec une population d’échantillonnage disparate et une population d’inférence cible (population cible) sont couramment appliqués. Utile lorsque vous traitez des données qui ont des valeurs manquantes. Source

La fonction lm() effectue une pondération analytique. Pour les poids d’échantillonnage, le package survey est utilisé pour créer un objet de conception d’enquête et exécuter svyglm(). Par défaut, le package “survey” utilise des poids d’échantillonnage. (REMARQUE : lm() et svyglm() avec la famille gaussian() produiront tous les mêmes estimations ponctuelles, car ils résolvent tous les deux les coefficients en minimisant les moindres carrés pondérés. Ils diffèrent dans la façon dont les erreurs standard sont calculés.)

Utilisation de la fonction ‘predict’ #

Une fois qu’un modèle est construit, “prédire” est la fonction principale à tester avec de nouvelles données. Notre exemple utilisera le jeu de données intégré mtcars pour régresser en miles par gallon par rapport au déplacement :

my_mdl <- lm(mpg ~ disp, data=mtcars)
my_mdl

Call:
lm(formula = mpg ~ disp, data = mtcars)

Coefficients:
(Intercept)         disp  
   29.59985     -0.04122

Si j’avais une nouvelle source de données avec déplacement, je pourrais voir les miles estimés par gallon.

set.seed(1234)
newdata <- sample(mtcars$disp, 5)
newdata
[1] 258.0  71.1  75.7 145.0 400.0

newdf <- data.frame(disp=newdata)
predict(my_mdl, newdf)
       1        2        3        4        5 
18.96635 26.66946 26.47987 23.62366 13.11381

La partie la plus importante du processus consiste à créer un nouveau bloc de données avec les mêmes noms de colonne que les données d’origine. Dans ce cas, les données d’origine avaient une colonne intitulée “disp”, j’étais sûr d’appeler les nouvelles données du même nom.

Mise en garde

Examinons quelques pièges courants :

  1. ne pas utiliser de data.frame dans le nouvel objet :

    predict(my_mdl, newdata)
Error in eval(predvars, data, env) : 
   numeric 'envir' arg not of length one

  2. ne pas utiliser les mêmes noms dans le nouveau bloc de données :

    newdf2 <- data.frame(newdata)
predict(my_mdl, newdf2)
Error in eval(expr, envir, enclos) : object 'disp' not found

Précision

Pour vérifier l’exactitude de la prédiction, vous aurez besoin des valeurs y réelles des nouvelles données. Dans cet exemple, newdf aura besoin d’une colonne pour ‘mpg’ et ‘disp’.

newdf <- data.frame(mpg=mtcars$mpg[1:10], disp=mtcars$disp[1:10])

Vérification de la non-linéarité avec la régression polynomiale #

Parfois, lorsque nous travaillons avec une régression linéaire, nous devons vérifier la non-linéarité des données. Une façon de procéder consiste à ajuster un modèle polynomial et à vérifier s’il s’adapte mieux aux données qu’un modèle linéaire. Il existe d’autres raisons, telles que théoriques, qui indiquent d’ajuster un modèle quadratique ou d’ordre supérieur, car on pense que la relation des variables est de nature intrinsèquement polynomiale.

Ajustons un modèle quadratique pour le jeu de données mtcars. Pour un modèle linéaire, voir https://www.wikiod.com/fr/r/modeles-lineaires-regression#Régression linéaire sur le jeu de données mtcars

Nous faisons d’abord un nuage de points des variables mpg (Miles/gallon), disp (Déplacement (cu.in.)) et wt (Poids (1000 lbs)). La relation entre mpg et disp semble non linéaire.

plot(mtcars[,c("mpg","disp","wt")])

[![entrez la description de l’image ici][1]][1]

Un ajustement linéaire montrera que “disp” n’est pas significatif.

fit0 = lm(mpg ~ wt+disp, mtcars)
summary(fit0)

#            Estimate Std. Error t value Pr(>|t|)    
#(Intercept) 34.96055    2.16454  16.151 4.91e-16 ***
#wt          -3.35082    1.16413  -2.878  0.00743 ** 
#disp        -0.01773    0.00919  -1.929  0.06362 .  
#---
#Signif. codes:  0 ‘***’ 0.001 ‘**’ 0.01 ‘*’ 0.05 ‘.’ 0.1 ‘ ’ 1
#Residual standard error: 2.917 on 29 degrees of freedom
#Multiple R-squared:  0.7809,    Adjusted R-squared:  0.7658

Ensuite, pour obtenir le résultat d’un modèle quadratique, nous avons ajouté I(disp^2). Le nouveau modèle apparaît mieux lorsque l’on regarde R ^ 2 et toutes les variables sont significatives.

fit1 = lm(mpg ~ wt+disp+I(disp^2), mtcars)
summary(fit1)

# Coefficients:
#              Estimate Std. Error t value Pr(>|t|)    
#(Intercept) 41.4019837  2.4266906  17.061  2.5e-16 ***
#wt          -3.4179165  0.9545642  -3.581 0.001278 ** 
#disp        -0.0823950  0.0182460  -4.516 0.000104 ***
#I(disp^2)    0.0001277  0.0000328   3.892 0.000561 ***
#---
#Signif. codes:  0 ‘***’ 0.001 ‘**’ 0.01 ‘*’ 0.05 ‘.’ 0.1 ‘ ’ 1
#Residual standard error: 2.391 on 28 degrees of freedom
#Multiple R-squared:  0.8578,    Adjusted R-squared:  0.8426

Comme nous avons trois variables, le modèle ajusté est une surface représentée par :

mpg = 41.4020-3.4179*wt-0.0824*disp+0.0001277*disp^2

Une autre façon de spécifier la régression polynomiale consiste à utiliser poly avec le paramètre raw=TRUE, sinon les polynômes orthogonaux seront pris en compte (voir help(ploy) pour plus d’informations). On obtient le même résultat en utilisant :

summary(lm(mpg ~ wt+poly(disp, 2, raw=TRUE),mtcars))

Tracer la régression (base) #

En continuant sur l’exemple mtcars, voici un moyen simple de produire un tracé de votre régression linéaire potentiellement adapté à la publication.

Ajustez d’abord le modèle linéaire et

fit <- lm(mpg ~ wt, data = mtcars)

Tracez ensuite les deux variables d’intérêt et ajoutez la droite de régression dans le domaine de définition :

plot(mtcars$wt,mtcars$mpg,pch=18, xlab = 'wt',ylab = 'mpg')
lines(c(min(mtcars$wt),max(mtcars$wt)),
as.numeric(predict(fit, data.frame(wt=c(min(mtcars$wt),max(mtcars$wt))))))

Presque là! La dernière étape consiste à ajouter au tracé, l’équation de régression, le rcarré ainsi que le coefficient de corrélation. Ceci est fait en utilisant la fonction vector :

rp = vector('expression',3)
rp[1] = substitute(expression(italic(y) == MYOTHERVALUE3 + MYOTHERVALUE4 %*% x), 
          list(MYOTHERVALUE3 = format(fit$coefficients[1], digits = 2),
                        MYOTHERVALUE4 = format(fit$coefficients[2], digits = 2)))[2]
rp[2] = substitute(expression(italic(R)^2 == MYVALUE), 
             list(MYVALUE = format(summary(fit)$adj.r.squared,dig=3)))[2]
rp[3] = substitute(expression(Pearson-R == MYOTHERVALUE2), 
             list(MYOTHERVALUE2 = format(cor(mtcars$wt,mtcars$mpg), digits = 2)))[2]

legend("topright", legend = rp, bty = 'n')

Notez que vous pouvez ajouter tout autre paramètre tel que le RMSE en adaptant la fonction vectorielle. Imaginez que vous vouliez une légende avec 10 éléments. La définition du vecteur serait la suivante :

rp = vector('expression',10)

et vous devrez définir r[1]…. à r[10]

Évaluation de la qualité #

Après avoir construit un modèle de régression, il est important de vérifier le résultat et de décider si le modèle est approprié et fonctionne bien avec les données disponibles. Cela peut être fait en examinant le tracé des résidus ainsi que d’autres tracés de diagnostic.

# fit the model
fit <- lm(mpg ~ wt, data = mtcars)
# 
par(mfrow=c(2,1))
# plot model object
plot(fit, which =1:2)

[![entrez la description de l’image ici][1]][1]

[1] : http://i.stack.imgur.com/tHoK5.jpg

Ces tracés vérifient deux hypothèses qui ont été faites lors de la construction du modèle :

  1. Que la valeur attendue de la variable prédite (dans ce cas mpg) est donnée par une combinaison linéaire des prédicteurs (dans ce cas wt). Nous nous attendons à ce que cette estimation soit impartiale. Les résidus doivent donc être centrés autour de la moyenne de toutes les valeurs des prédicteurs. Dans ce cas, nous voyons que les résidus ont tendance à être positifs aux extrémités et négatifs au milieu, suggérant une relation non linéaire entre les variables.
  2. Que la variable prédite réelle est normalement distribuée autour de son estimation. Ainsi, les résidus doivent être normalement distribués. Pour les données distribuées normalement, les points d’un tracé Q-Q normal doivent se trouver sur ou près de la diagonale. Il y a une certaine quantité de biais aux extrémités ici.
