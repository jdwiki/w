Une fonction de la famille *apply est une abstraction d’une boucle for. Par rapport aux boucles for, les fonctions *apply présentent les avantages suivants :

  1. Nécessite moins de code pour écrire.
  2. N’a pas de compteur d’itérations.
  3. N’utilise pas de variables temporaires pour stocker les résultats intermédiaires.

Cependant, les boucles for sont plus générales et peuvent nous donner plus de contrôle permettant de réaliser des calculs complexes qui ne sont pas toujours triviaux à faire en utilisant les fonctions *apply.

La relation entre les boucles for et les fonctions *apply est expliquée dans la [documentation des boucles for][1].

Membres de la famille *apply #

La famille de fonctions « *apply » contient plusieurs variantes du même principe qui diffèrent principalement en fonction du type de sortie qu’elles renvoient.

fonction Entrée Sortie
appliquer matrice, data.frame ou tableau vecteur ou matrice (selon la longueur de chaque élément renvoyé)
sapply vecteur ou ’liste' vecteur ou matrice (selon la longueur de chaque élément renvoyé)
appliquer vecteur ou ’liste' liste
vappliquer vecteur ou `liste vecteur ou matrice (selon la longueur de chaque élément renvoyé) de la classe désignée par l’utilisateur
mapply plusieurs vecteurs, listes ou une combinaison liste

Voir “Exemples” pour voir comment chacune de ces fonctions est utilisée.

[1] : https://www.wikiod.com/fr/r/structures-de-flux-de-controle

Utilisation des fonctionnalités intégrées #

Fonctions intégrées : lapply(), sapply() et mapply() #

R est livré avec des fonctions intégrées, dont les plus connues sont peut-être la famille de fonctions apply. Voici une description de certaines des fonctions d’application les plus courantes :

  • lapply() = prend une liste comme argument et applique la fonction spécifiée à la liste.
  • sapply() = identique à lapply() mais tente de simplifier la sortie en un vecteur ou une matrice.
    • vapply() = a variant of sapply() in which the output object’s type must be specified.
  • mapply() = comme lapply() mais peut passer plusieurs vecteurs en entrée de la fonction spécifiée. Peut être simplifié comme sapply().
    • Map() is an alias to mapply() with SIMPLIFY = FALSE.

lapply() #

lapply() peut être utilisé avec deux itérations différentes :

  • appliquer(variable, FUN)

  • lapply(seq_along(variable), FUN)

    Two ways of finding the mean of x #

    set.seed(1) df <- data.frame(x = rnorm(25), y = rnorm(25)) lapply(df, mean) lapply(seq_along(df), function(x) mean(df[[x]))

sapply() #

sapply() tentera de résoudre sa sortie en vecteur ou en matrice.

# Two examples to show the different outputs of sapply()
sapply(letters, print)  ## produces a vector
x <- list(a = 1:10, beta = exp(-3:3), logic = c(TRUE,FALSE,FALSE,TRUE))
sapply(x, quantile)  ## produces a matrix

mapply() #

mapply() fonctionne un peu comme lapply() sauf qu’il peut prendre plusieurs vecteurs en entrée (d’où le m pour multivarié).

mapply(sum, 1:5, 10:6, 3) # 3 will be "recycled" by mapply

[1] : http://adv-r.had.co.nz/Functionals.html “Fonctionnels”

Chargement de fichiers en masse #

pour un grand nombre de fichiers qui peuvent devoir être traités dans un processus similaire et avec des noms de fichiers bien structurés.

il faut d’abord créer un vecteur des noms de fichiers auxquels accéder, il y a plusieurs options pour cela :

  • Création manuelle du vecteur avec paste0()

      files <- paste0("file_", 1:100, ".rds")

  • L’utilisation de list.files() avec un terme de recherche regex pour le type de fichier nécessite la connaissance des expressions régulières ([regex][1]) si d’autres fichiers du même type se trouvent dans le répertoire.

      files <- list.files("./", pattern = "\\.rds$", full.names = TRUE)

où ‘X’ est un vecteur d’une partie du format de nommage des fichiers utilisé.

lapply affichera chaque réponse comme élément d’une liste.

readRDS est spécifique aux fichiers .rds et changera en fonction de l’application du processus.

my_file_list <- lapply(files, readRDS)

Ce n’est pas nécessairement plus rapide qu’une boucle for de testing mais permet à tous les fichiers d’être un élément d’une liste sans les affecter explicitement.

Enfin, nous devons souvent charger plusieurs packages à la fois. Cette astuce peut le faire assez facilement en appliquant library() à toutes les bibliothèques que nous souhaitons importer :

lapply(c("jsonlite","stringr","igraph"),library,character.only=TRUE)

[1] : https://www.wikiod.com/fr/regex/premiers-pas-avec-les-expressions-regulieres

Combinaison de plusieurs data.frames (lapply, mapply) #

Dans cet exercice, nous allons générer quatre modèles de régression linéaire bootstrap et combiner les résumés de ces modèles en une seule base de données.

library(broom)

#* Create the bootstrap data sets
BootData <- lapply(1:4,
                   function(i) mtcars[sample(1:nrow(mtcars),
                                             size = nrow(mtcars),
                                             replace = TRUE), ])

#* Fit the models
Models <- lapply(BootData,
                 function(BD) lm(mpg ~ qsec + wt + factor(am),
                                 data = BD))

#* Tidy the output into a data.frame
Tidied <- lapply(Models,
                 tidy)

#* Give each element in the Tidied list a name
Tidied <- setNames(Tidied, paste0("Boot", seq_along(Tidied)))

À ce stade, nous pouvons adopter deux approches pour insérer les noms dans le data.frame.

#* Insert the element name into the summary with `lapply`
#* Requires passing the names attribute to `lapply` and referencing `Tidied` within
#* the applied function.
Described_lapply <- 
 lapply(names(Tidied),
        function(nm) cbind(nm, Tidied[[nm]]))

Combined_lapply <- do.call("rbind", Described_lapply)

#* Insert the element name into the summary with `mapply`
#* Allows us to pass the names and the elements as separate arguments.
Described_mapply <- 
 mapply(
  function(nm, dframe) cbind(nm, dframe),
  names(Tidied),
  Tidied,
  SIMPLIFY = FALSE)

Combined_mapply <- do.call("rbind", Described_mapply)

Si vous êtes un fan des pipes de style magrittr, vous pouvez accomplir toute la tâche en une seule chaîne (bien qu’il ne soit peut-être pas prudent de le faire si vous avez besoin de l’un des objets intermédiaires, tels que les objets modèles eux-mêmes):

library(magrittr)
library(broom)
Combined <- lapply(1:4,
                   function(i) mtcars[sample(1:nrow(mtcars),
                                             size = nrow(mtcars),
                                             replace = TRUE), ]) %>%
 lapply(function(BD) lm( mpg ~ qsec + wt + factor(am), data = BD)) %>%
 lapply(tidy) %>%
 setNames(paste0("Boot", seq_along(.))) %>%
 mapply(function(nm, dframe) cbind(nm, dframe),
        nm = names(.),
        dframe = .,
        SIMPLIFY = FALSE) %>%
 do.call("rbind", .)

Utilisation de fonctionnalités définies par l’utilisateur #

Fonctions définies par l’utilisateur #

Les utilisateurs peuvent créer leurs propres fonctionnalités à différents degrés de complexité. Les exemples suivants sont tirés de [Functionals][1] de Hadley Wickham :

randomise <- function(f) f(runif(1e3))
        
lapply2 <- function(x, f, ...) {
    out <- vector("list", length(x))
    for (i in seq_along(x)) {
        out[[i]] <- f(x[[i]], ...)
    }
    out
}

Dans le premier cas, randomise accepte un seul argument f et l’appelle sur un échantillon de variables aléatoires uniformes. Pour démontrer l’équivalence, nous appelons set.seed ci-dessous :

set.seed(123)
randomise(mean)
#[1] 0.4972778
    
set.seed(123)
mean(runif(1e3))
#[1] 0.4972778


set.seed(123)
randomise(max)
#[1] 0.9994045

set.seed(123)
max(runif(1e3))
#[1] 0.9994045

Le deuxième exemple est une réimplémentation de base :: lapply, qui utilise des fonctions pour appliquer une opération (f) à chaque élément d’une liste (x). Le paramètre ... permet à l’utilisateur de passer des arguments supplémentaires à f, comme l’option na.rm dans la fonction mean :

lapply(list(c(1, 3, 5), c(2, NA, 6)), mean)
# [[1]]
# [1] 3
# 
# [[2]]
# [1] NA

lapply2(list(c(1, 3, 5), c(2, NA, 6)), mean)
# [[1]]
# [1] 3
# 
# [[2]]
# [1] NA


lapply(list(c(1, 3, 5), c(2, NA, 6)), mean, na.rm = TRUE)
# [[1]]
# [1] 3
# 
# [[2]]
# [1] 4

lapply2(list(c(1, 3, 5), c(2, NA, 6)), mean, na.rm = TRUE)
# [[1]]

Utiliser des fonctions anonymes avec apply #

apply est utilisé pour évaluer une fonction (peut-être anonyme) sur les marges d’un tableau ou d’une matrice.

Utilisons le jeu de données iris pour illustrer cette idée. L’ensemble de données “iris” contient des mesures de 150 fleurs de 3 espèces. Voyons comment cet ensemble de données est structuré :

> head(iris)

  Sepal.Length Sepal.Width Petal.Length Petal.Width Species
1         5.1          3.5          1.4         0.2  setosa
2         4.9          3.0          1.4         0.2  setosa
3         4.7          3.2          1.3         0.2  setosa
4         4.6          3.1          1.5         0.2  setosa
5         5.0          3.6          1.4         0.2  setosa
6         5.4          3.9          1.7         0.4  setosa

Maintenant, imaginez que vous vouliez connaître la moyenne de chaque de ces variables. Une façon de résoudre ce problème pourrait être d’utiliser une boucle for, mais les programmeurs R préféreront souvent utiliser apply (pour les raisons, voir Remarques):

> apply(iris[1:4], 2, mean)

Sepal.Length  Sepal.Width Petal.Length  Petal.Width 
    5.843333     3.057333     3.758000     1.199333
  • Dans le premier paramètre, nous avons sous-ensemble iris pour n’inclure que les 4 premières colonnes, car mean ne fonctionne que sur des données numériques.
  • La valeur du deuxième paramètre de 2 indique que nous voulons travailler uniquement sur les colonnes (le deuxième indice du tableau r×c); ‘1’ donnerait la ligne signifie.

De la même manière, nous pouvons calculer des valeurs plus significatives :

# standard deviation
apply(iris[1:4], 2, sd)
# variance
apply(iris[1:4], 2, var)

Caveat : R a quelques fonctions intégrées qui sont meilleures pour calculer les sommes et les moyennes des colonnes et des lignes : [colMeans et rowMeans](http://stat.ethz.ch/R-manual/R-patched/library /base/html/colSums.html).

Maintenant, effectuons une tâche différente et plus significative : calculons la moyenne uniquement pour les valeurs supérieures à “0,5”. Pour cela, nous allons créer notre propre fonction mean.

> our.mean.function <- function(x) { mean(x[x > 0.5]) }
> apply(iris[1:4], 2, our.mean.function)

Sepal.Length  Sepal.Width Petal.Length  Petal.Width 
    5.843333     3.057333     3.758000     1.665347

(Notez la différence dans la moyenne de Petal.Width)

Mais que se passe-t-il si nous ne voulons pas utiliser cette fonction dans le reste de notre code ? Ensuite, nous pouvons utiliser une fonction anonyme et écrire notre code comme ceci :

apply(iris[1:4], 2, function(x) { mean(x[x > 0.5]) })

Ainsi, comme nous l’avons vu, nous pouvons utiliser apply pour exécuter la même opération sur des colonnes ou des lignes d’un ensemble de données en utilisant une seule ligne.

Mise en garde : Étant donné que apply renvoie des types de sortie très différents en fonction de la longueur des résultats de la fonction spécifiée, il se peut que ce ne soit pas le meilleur choix dans les cas où vous ne travaillez pas de manière interactive. Certaines des autres fonctions de la famille *apply sont un peu plus prévisibles (voir Remarques).
