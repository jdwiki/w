Accéder à l’éditeur Visual Basic d’Outlook, insérer votre premier module et renommer ce module.

Connaissances préalables attendues : Vous êtes un utilisateur d’Outlook.

Avec Outlook 2003, vous pouvez immédiatement sélectionner Visual Basic Editor. Avec les versions ultérieures, vous devez ajouter l’onglet Développeur avant de pouvoir sélectionner Visual Basic Editor.

1.3 Premiers pas avec Visual Basic Editor #

Les images de cette section proviennent toutes d’Outlook 2016, mais elles pourraient provenir d’Outlook 2003. Outlook VBA a peut-être changé au fil des ans, mais à mes yeux, l’éditeur VBA ne l’a pas été. Quelle que soit la version que vous avez, vous verrez quelque chose comme :

[![Visual Basic Editor avec Explorateur de projets non développé][1]][1]

Au-dessus, il y a un “+” contre “Project1”. Si vous avez un « + », cliquez dessus, puis sur le « + » contre « Objets Microsoft Outlook » pour obtenir :

[![Visual Basic Editor avec l’Explorateur de projets étendu][2]][2]

La fenêtre Propriétés peut ne pas être présente ou, si elle est présente, peut être positionnée ailleurs dans la fenêtre de l’éditeur VB. Nous n’en avons pas besoin pour le moment. Vous pouvez le fermer en cliquant sur la croix et utiliser F4 pour le rendre à nouveau visible à tout moment. Je ne l’ai normalement pas visible car je n’ai pas besoin d’accéder aux propriétés la plupart du temps et ma liste d’explorateurs de projets occupe la majeure partie du côté gauche. Je vous suggère de le garder visible jusqu’à ce qu’il devienne une nuisance.

Si vous cliquez sur ThisOutlookSession, soit la zone grise deviendra blanche, soit, comme dans l’image ci-dessous, une fenêtre de code apparaîtra dans la zone grise :

[![Cette zone de code de session Outlook][3]][3]

Vous pouvez taper n’importe quel code dans cette fenêtre de code. Cependant, les routines d’événement (dont il est question à la fin de ce didacticiel) doivent être saisies dans cette fenêtre de code. Je vous recommande de réserver la zone de code ThisOutlookSession pour les routines d’événement.

Au lieu de cela, cliquez sur Insérer pour obtenir :

[![Menu déroulant Insérer][4]][4]

Cliquez sur Module pour ajouter un module :

[![Nouveau module][5]][5]

Mon nouveau module s’appelle “Module1”. Si votre version d’Outlook n’est pas une version anglaise, votre module aura un nom équivalent dans votre langue. Vous pouvez ajouter d’autres modules qui seront nommés “Module2”, “Module3” et ainsi de suite.

Si je crée un classeur Excel, pour lequel je n’ai besoin que d’un module, je peux laisser le nom “Module1”. Mais avec Outlook, toutes mes macros doivent aller ici donc j’ai plein de modules. Au fil des ans, j’ai écrit de nombreuses routines que je réutilise à plusieurs reprises. J’ai un module pour les routines générales VBA, un autre pour les routines d’accès à Excel, un autre pour les routines Outlook VBA, puis un module par tâche Outlook que j’exécute avec des macros. Si vous regardez la fenêtre Propriétés, vous verrez que la seule propriété d’un module est son nom. Cliquez sur le “Module1” contre “Nom” et vous pouvez le remplacer par n’importe quel nom valide (commence par une lettre, contient uniquement des lettres et des chiffres, etc.). Vous obtenez des erreurs étranges si un module et une procédure ont le même nom donc je commence tous mes noms de module par “Mod” et je n’utilise pas ce préfixe pour mes procédures. Pourquoi ne pas renommer ce module “ModIntro” ou similaire prêt pour la prochaine partie de ce tutoriel ?

Ces zones de code et comme les zones de saisie de données de n’importe quel éditeur. Cliquez sur la zone de code pour la sélectionner et tapez votre code ou collez-y du code copié ailleurs, comme dans la section suivante de ce didacticiel.

1.1 Accéder à l’éditeur Visual Basic d’Outlook 2003 #

Toutes les images proviennent des versions britanniques d’Outlook. Je sais que certains noms sont traduits dans la langue locale pour d’autres versions et je suppose que la plupart des noms des onglets sont traduits. La séquence d’onglets est probablement inchangée dans les versions non anglaises. Alternativement, vous devrez regarder vos onglets et décider lequel équivaudrait, par exemple, à “Outils”.

Avec Outlook 2003 ouvert, le haut de la fenêtre peut ressembler à :

[![Fenêtre principale d’Outlook 2003][1]][1]

Cliquez sur Outils et déplacez le curseur vers Macros pour voir :

[![entrez la description de l’image ici][2]][2]

Déplacez le curseur vers la droite puis vers le bas et cliquez sur Visual Basic Editor. Vous pouvez également quitter les sélections et cliquer sur Alt+F11.

[1] : https://i.stack.imgur.com/brDHk.png [2] : https://i.stack.imgur.com/0riBk.png

1.2 Accès à Visual Basic Editor dans Outlook 2007 et versions ultérieures #

Toutes les images de cette section proviennent de la version britannique d’Outlook 2016. Je sais que certains noms sont traduits dans la langue locale pour d’autres versions et je suppose que la plupart des noms des onglets sont traduits. La séquence d’onglets est probablement inchangée dans les versions non anglaises. Alternativement, vous devrez regarder vos onglets et décider lequel équivaudrait, par exemple, à “Outils”.

Les fenêtres Outlook 2010 sont formatées différemment mais sont essentiellement identiques. Je comprends que les autres versions sont également essentiellement identiques à Outlook 2016.

Le haut de la fenêtre principale pourrait ressembler à :

[![Fenêtre principale pour Outlook 2016 avant l’ajout de la balise Developer][1]][1]

Cliquez sur Fichier, sur la gauche, pour obtenir ce qui suit sur la gauche de la fenêtre :

[![Menu Fichier][2]][2]

Cliquez sur Options, vers le bas, pour obtenir ce qui suit sur la gauche de la fenêtre :

[![Menu Options][3]][3]

Cliquez sur Personnaliser le ruban, à mi-chemin. pour obtenir ce qui suit à droite de la fenêtre :

[![Menu principal des balises][4]][4]

Cliquez sur la case à côté de “Développeur”, près du bas, pour obtenir une coche, puis cliquez sur OK, en bas. La fenêtre principale réapparaîtra mais aura changé pour :

[![Fenêtre principale d’Outlook 2016 avec l’onglet Développeur ajouté][5]][5]

Cliquez sur le nouvel onglet Développeur pour obtenir :

[![Fenêtre principale d’Outlook 2016 avec l’onglet Développeur cliqué][6]][6]

Cliquez sur Visual Basic, sur la gauche, pour sélectionner l’éditeur Visual Basic.
