Création d’une application Ruby on Rails #

Cet exemple suppose que Ruby et Ruby on Rails ont déjà été correctement installés. Si ce n’est pas le cas, vous pouvez trouver comment procéder [ici][1].

Ouvrez une ligne de commande ou un terminal. Pour générer une nouvelle application rails, utilisez la commande [rails new][2] suivie du nom de votre application :

$ rails new my_app

Si vous souhaitez créer votre application Rails avec une version Rails spécifique, vous pouvez le spécifier au moment de la génération de l’application. Pour ce faire, utilisez rails _version_ new suivi du nom de l’application :

$ rails _4.2.0_ new my_app

Cela créera une application Rails appelée MyApp dans un répertoire my_app et installera les dépendances gem déjà mentionnées dans Gemfile en utilisant bundle install.

Pour basculer vers le répertoire de votre application nouvellement créée, utilisez la commande cd, qui signifie change directory.

$ cd my_app

Le répertoire my_app contient un certain nombre de fichiers et de dossiers générés automatiquement qui constituent la structure d’une application Rails. Voici une liste des fichiers et dossiers créés par défaut :

Fichier/Dossier Objectif
app/ Contient les contrôleurs, les modèles, les vues, les assistants, les expéditeurs et les actifs de votre application.
bin/ Contient le script rails qui démarre votre application et peut contenir d’autres scripts que vous utilisez pour configurer, mettre à jour, déployer ou exécuter votre application.
config/ Configurez les itinéraires, la base de données et plus encore de votre application.
config.ru Configuration en rack pour les serveurs en rack utilisés pour démarrer l’application.
db/ Contient votre schéma de base de données actuel, ainsi que les migrations de base de données.
Gemfile Gemfile.lock Ces fichiers vous permettent de spécifier quelles dépendances gem sont nécessaires pour votre application Rails. Ces fichiers sont utilisés par le gem Bundler.
lib/ Modules étendus pour votre application.
log/ Fichiers journaux des applications.
publique/ Le seul dossier vu par le monde tel quel. Contient des fichiers statiques et des ressources compilées.
Rakefile Ce fichier localise et charge les tâches pouvant être exécutées à partir de la ligne de commande. Les définitions de tâches sont définies dans tous les composants de Rails.
LISEZMOI.md Il s’agit d’un bref manuel d’instructions pour votre application. Vous devez modifier ce fichier pour indiquer aux autres ce que fait votre application, comment la configurer, etc.
tester/ Tests unitaires, montages et autres appareils de test.
temp/ Fichiers temporaires (comme les fichiers cache et pid).
vendeur/ Une place pour tout code tiers. Dans une application Rails typique, cela inclut les gemmes vendues.

Vous devez maintenant créer une base de données à partir de votre fichier database.yml :

rake db:create
# OR
rails db:create

rake db:create

Maintenant que nous avons créé la base de données, nous devons exécuter des migrations pour configurer les tables :

rake db:migrate
# OR
rails db:migrate

rake db:migrate

Pour démarrer l’application, nous devons lancer le serveur :

$ rails server
# OR
$ rails s

Par défaut, les rails démarreront l’application sur le port 3000. Pour démarrer l’application avec un numéro de port différent, nous devons lancer le serveur comme,

$ rails s -p 3010

Si vous accédez à http://localhost:3000 dans votre navigateur, vous verrez une page d’accueil Rails indiquant que votre application est en cours d’exécution.

S’il génère une erreur, il peut y avoir plusieurs problèmes possibles :

  • Il y a un problème avec le config/database.yml
  • Vous avez des dépendances dans votre Gemfile qui n’ont pas été installées.
  • Vous avez des migrations en attente. Exécutez rails db:migrate
  • Si vous passez à la migration précédente rails db:rollback

Si cela génère toujours une erreur, vous devriez vérifier votre config/database.yml

[1] : https://www.wikiod.com/fr/ruby-on-rails/premiers-pas-avec-ruby-on-rails#Installation des rails [2] : http://guides.rubyonrails.org/command_line.html#rails-new

Créez une nouvelle application Rails avec votre choix de base de données et incluant l’outil de test RSpec #

Rails utilise sqlite3 comme base de données par défaut, mais vous pouvez générer une nouvelle application rails avec une base de données de votre choix. Ajoutez simplement l’option -d suivie du nom de la base de données.

$ rails new MyApp -T -d postgresql

Voici une liste (non exhaustive) des options de base de données disponibles :

-mysql

  • oracle
  • postgresql
  • sqlite3
  • base avant
  • ibm_db
  • serveur SQL
  • jdbcmysql
  • jdbcsqlite3
  • jdbcpostgresql
  • jdbc

La commande -T indique de sauter l’installation de minitest. Pour installer une suite de tests alternative comme [RSpec][1], éditez le Gemfile et ajoutez

group :development, :test do
  gem 'rspec-rails', 
end

Lancez ensuite la commande suivante depuis la console :

rails generate rspec:install

[1] : http://rspec.info/

Générer un contrôleur #

Pour générer un contrôleur (par exemple Posts), accédez au répertoire de votre projet à partir d’une ligne de commande ou d’un terminal, et exécutez :

$ rails generate controller Posts

Vous pouvez raccourcir ce code en remplaçant generate par g, par exemple :

$ rails g controller Posts

Si vous ouvrez le fichier app/controllers/posts_controller.rb nouvellement généré, vous verrez un contrôleur sans action :

class PostsController < ApplicationController
    # empty
end

Il est possible de créer des méthodes par défaut pour le contrôleur en transmettant les arguments du nom du contrôleur.

$ rails g controller ControllerName method1 method2

Pour créer un contrôleur dans un module, spécifiez le nom du contrôleur sous la forme d’un chemin tel que parent_module/controller_name. Par exemple:

$ rails generate controller CreditCards open debit credit close
# OR
$ rails g controller CreditCards open debit credit close

Cela générera les fichiers suivants :

Controller: app/controllers/credit_cards_controller.rb
Test:       test/controllers/credit_cards_controller_test.rb
Views:      app/views/credit_cards/debit.html.erb [...etc]
Helper:     app/helpers/credit_cards_helper.rb

Un contrôleur est simplement une classe définie pour hériter de ApplicationController.

C’est à l’intérieur de cette classe que vous définirez les méthodes qui deviendront les actions de ce contrôleur.

Installation des rails #

Installation de rails sur Ubuntu

Sur un Ubuntu propre, l’installation de Rails devrait être simple

Mise à niveau des packages Ubuntu

sudo apt-get update
sudo apt-get upgrade

Installer les dépendances Ruby et Rails

sudo apt-get install git-core curl zlib1g-dev build-essential libssl-dev libreadline-dev libyaml-dev libsqlite3-dev sqlite3 libxml2-dev libxslt1-dev libcurl4-openssl-dev python-software-properties libffi-dev

Installation du gestionnaire de version ruby. Dans ce cas, le plus simple est d’utiliser rbenv

git clone https://github.com/rbenv/rbenv.git ~/.rbenv
echo 'export PATH="$HOME/.rbenv/bin:$PATH"' >> ~/.bashrc
echo 'eval "$(rbenv init -)"' >> ~/.bashrc

Installation de Ruby Build

git clone https://github.com/rbenv/ruby-build.git ~/.rbenv/plugins/ruby-build
echo 'export PATH="$HOME/.rbenv/plugins/ruby-build/bin:$PATH"' >> ~/.bashrc

Redémarrer le shell

exec $SHELL

Installer rubis

rbenv install 2.3.1
rbenv global 2.3.1
rbenv rehash

Installation des rails

gem install rails

Installation de Rails sous Windows

Étape 1 : Installation de Ruby

Nous avons besoin du langage de programmation Ruby installé. Nous pouvons utiliser une version précompilée de Ruby appelée RubyInstaller.

  • Téléchargez et exécutez Ruby Installer à partir de [rubyinstaller.org][1].
  • Exécutez le programme d’installation. Cochez “Ajouter des exécutables Ruby à votre PATH”, puis installez.
  • Pour accéder à Ruby, allez dans le menu Windows, cliquez sur Tous les programmes, faites défiler jusqu’à Ruby et cliquez sur “Démarrer l’invite de commande avec Ruby”. Un terminal d’invite de commande s’ouvrira. Si vous tapez ruby -v et appuyez sur Entrée, vous devriez voir le numéro de version de Ruby que vous avez installé.

Étape 2 : Kit de développement Ruby

Après avoir installé Ruby, nous pouvons essayer d’installer Rails. Mais certaines des bibliothèques dont Rails dépend ont besoin d’outils de construction pour être compilées, et Windows ne dispose pas de ces outils par défaut. Vous pouvez l’identifier si vous voyez une erreur lors de la tentative d’installation de Rails Gem :: InstallError: La gemme native '[gem name]' nécessite des outils de construction installés. Pour résoudre ce problème, nous devons installer le kit de développement Ruby.

  • Téléchargez le [DevKit][2]
  • Exécutez le programme d’installation.
  • Nous devons spécifier un dossier dans lequel nous allons installer définitivement le DevKit. Je recommande de l’installer à la racine de votre disque dur, à C:\RubyDevKit. (N’utilisez pas d’espaces dans le nom du répertoire.)

Nous devons maintenant mettre les outils DevKit à la disposition de Ruby.

  • Dans votre invite de commande, accédez au répertoire DevKit. cd C:\RubyDevKit ou le répertoire dans lequel vous l’avez installé.
  • Nous devons exécuter un script Ruby pour initialiser la configuration de DevKit. Taper ruby dk.rb init. Nous allons maintenant dire à ce même script d’ajouter le DevKit à notre installation Ruby. Tapez ruby dk.rb install.

Le DevKit devrait maintenant être disponible pour vos outils Ruby à utiliser lors de l’installation de nouvelles bibliothèques.

Étape 3 : *** Rails ***

Nous pouvons maintenant installer Rails. Rails est un joyau Ruby. Dans votre invite de commande, tapez :

rails d'installation gemme

Une fois que vous aurez appuyé sur Entrée, le programme gem téléchargera et installera cette version de la gemme Rails, ainsi que toutes les autres gemmes dont Rails dépend.

Étape 4 : Node.js

Certaines bibliothèques dont dépend Rails nécessitent l’installation d’un runtime JavaScript. Installons Node.js pour que ces bibliothèques fonctionnent correctement.

  • Téléchargez le programme d’installation de Node.js depuis [ici][3].
  • Une fois le téléchargement terminé, visitez votre dossier de téléchargements et exécutez le programme d’installation node-v4.4.7.pkg.
  • Lisez l’intégralité du contrat de licence, acceptez les termes et cliquez sur Suivant dans le reste de l’assistant, en laissant tout par défaut.
  • Une fenêtre peut apparaître vous demandant si vous souhaitez autoriser l’application à apporter des modifications à votre ordinateur. Cliquez sur “Oui”.
  • Une fois l’installation terminée, vous devrez redémarrer votre ordinateur pour que Rails puisse accéder à Node.js.

Une fois votre ordinateur redémarré, n’oubliez pas d’aller dans le menu Windows, cliquez sur “Tous les programmes”, faites défiler jusqu’à Ruby et cliquez sur “Démarrer l’invite de commande avec Ruby”.

[1] : http://rubyinstaller.org/ [2] : http://rubyinstaller.org/downloads/ [3] : https://nodejs.org/en/download/

Créez une nouvelle application Rails avec un adaptateur de base de données non standard #

Rails est livré par défaut avec ActiveRecord, un ORM (Object Relational Mapping) dérivé du modèle avec le [même nom][1].

En tant qu’ORM, il est conçu pour gérer le mappage relationnel, et plus précisément en gérant les requêtes SQL pour vous, d’où la limitation aux bases de données SQL uniquement.

Cependant, vous pouvez toujours créer une application Rails avec un autre système de gestion de base de données :

  1. créez simplement votre application sans enregistrement actif
$ rails app new MyApp --skip-active-record
  1. ajoutez votre propre système de gestion de base de données dans Gemfile
gem 'mongoid', '~> 5.0'
  1. bundle install et suivez les étapes d’installation à partir de la base de données souhaitée.

Dans cet exemple, mongoid est un mappage d’objet pour MongoDB et - comme beaucoup d’autres gemmes de base de données conçues pour les rails - il hérite également de ActiveModel de la même manière que ActiveRecord, qui fournit une interface commune pour de nombreuses fonctionnalités telles que comme les validations, les rappels, les traductions, etc.

Les autres adaptateurs de base de données incluent, mais ne sont pas limités à :

  • dossiers de données
  • rails de suite

[1] : https://en.wikipedia.org/wiki/Active_record_pattern

Création d’API Rails en JSON #

Cet exemple suppose que vous avez de l’expérience dans la création d’applications Rails.

Pour créer une application uniquement API dans Rails 5, exécutez

rails new name-of-app --api

Ajouter active_model_serializers dans Gemfile

gem 'active_model_serializers'

installer le bundle dans le terminal

bundle install

Définissez l’adaptateur ActiveModelSerializer pour utiliser : json_api

# config/initializers/active_model_serializer.rb
ActiveModelSerializers.config.adapter = :json_api
Mime::Type.register "application/json", :json, %w( text/x-json application/jsonrequest application/vnd.api+json )

Générer un nouvel échafaudage pour votre ressource

rails generate scaffold Task name:string description:text

Cela générera les fichiers suivants :

Contrôleur : app/controllers/tasks_controller.rb

Test:       test/models/task_test.rb
            test/controllers/tasks_controller_test.rb
Routes:     resources :tasks added in routes.rb
Migration:  db/migrate/_create_tasks.rb
Model:      app/models/task.rb
Serializer: app/serializers/task_serializer.rb
Controller: app/controllers/tasks_controller.rb

Générer une ressource avec des échafaudages #

Sur guides.rubyonrails.org :

Au lieu de générer directement un modèle . . . installons un échafaudage. Un échafaudage dans Rails est un ensemble complet de modèles, une migration de base de données pour ce modèle, un contrôleur pour le manipuler, des vues pour afficher et manipuler les données et une suite de tests pour chacun des éléments ci-dessus.

Voici un exemple d’échafaudage d’une ressource appelée “Tâche” avec un nom de chaîne et une description textuelle :

rails generate scaffold Task name:string description:text

Cela générera les fichiers suivants :

Controller: app/controllers/tasks_controller.rb
Test:       test/models/task_test.rb
            test/controllers/tasks_controller_test.rb
Routes:     resources :tasks added in routes.rb
Views:      app/views/tasks
            app/views/tasks/index.html.erb
            app/views/tasks/edit.html.erb
            app/views/tasks/show.html.erb
            app/views/tasks/new.html.erb
            app/views/tasks/_form.html.erb
Helper:     app/helpers/tasks_helper.rb
JS:         app/assets/javascripts/tasks.coffee 
CSS:        app/assets/stylesheets/tasks.scss
            app/assets/stylesheets/scaffolds.scss

exemple pour supprimer les fichiers générés par scaffold pour la ressource appelée Task

rails destroy scaffold Task
