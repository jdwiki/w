Environnements dans Rails #

## Configuration de la base de données La configuration de la base de données d’un projet rails se trouve dans un fichier config/database.yml. Si vous créez un projet à l’aide de la commande rails new et que vous ne spécifiez pas de moteur de base de données à utiliser, rails utilise sqlite comme base de données par défaut. Un fichier database.yml typique avec la configuration par défaut ressemblera à ce qui suit.

# SQLite version 3.x
#   gem install sqlite3
#
#   Ensure the SQLite 3 gem is defined in your Gemfile
#   gem 'sqlite3'
#
default: &default
  adapter: sqlite3
  pool: 5
  timeout: 5000

development:
  <<: *default
  database: db/development.sqlite3

# Warning: The database defined as "test" will be erased and
# re-generated from your development database when you run "rake".
# Do not set this db to the same as development or production.
test:
  <<: *default
  database: db/test.sqlite3

production:
  <<: *default
  database: db/production.sqlite3

Si vous souhaitez modifier la base de données par défaut lors de la création d’un nouveau projet, vous pouvez spécifier la base de données : rails new hello_world --database=mysql

Configuration générale des rails #

Les options de configuration suivantes doivent être appelées sur un objet Rails::Railtie

  • config.after_initialize : Prend un bloc qui sera exécuté après que rails ait initialisé l’application.
  • config.asset_host : cela définit l’hôte pour les actifs. Ceci est utile lors de l’utilisation d’un Réseau de diffusion de contenu. Ceci est un raccourci pour config.action_controller.asset_host
  • config.autoload_once_paths : cette option accepte un tableau de chemins où Rails charge automatiquement les constantes. La valeur par défaut est un tableau vide
  • config.autoload_paths : Ceci accepte un tableau de chemins où Rails charge automatiquement les constantes. Il s’agit par défaut de tous les répertoires sous ‘app’
  • config.cache_classes : détermine si les classes et les modules doivent être rechargés à chaque requête. En mode développement, la valeur par défaut est “false” et dans les modes de production et de test, la valeur par défaut est “true”.
  • config.action_view.cache_template_loading : Ceci détermine si les modèles doivent être rechargés à chaque requête. Il utilise par défaut le paramètre config.cache_classes
  • config.beginning_of_week : Ceci définit le début de semaine par défaut. Il nécessite un symbole de jour de semaine valide (: lundi)
  • config.cache_store : choisissez le magasin de cache à utiliser. Les options incluent :file_store, :memory_store, mem_cache_store ou null_store.
  • config.colorize_logging : ceci contrôle si les informations de journalisation sont colorées
  • config.eager_load : charge rapidement tous les fichiers enregistrés
  • config.encoding : spécifie l’encodage de l’application. La valeur par défaut est UTF-8
  • config.log_level : Définit la verbosité du Rails Logger. La valeur par défaut est :debug dans tous les environnements.
  • config.middleware : utilisez ceci pour configurer le middleware de l’application
  • config.time_zone : Cela définit le fuseau horaire par défaut de l’application.

Configuration des actifs #

Les options de configuration suivantes peuvent être utilisées pour configurer les actifs

  • config.assets.enabled : détermine si le pipeline d’actifs est activé. Cette valeur par défaut est true
  • config.assets.raise_runtime_errors : cela active la vérification des erreurs d’exécution. C’est utile pour le “mode développement”
  • config.assets.compress : permet de compresser les actifs. En mode production, cette valeur par défaut est true
  • config.assets.js_compressor : spécifie le compresseur JS à utiliser. Les options incluent :closure, :uglifier et :yui
  • config.assets.paths : spécifie les chemins dans lesquels rechercher les actifs.
  • config.assets.precompile : vous permet de choisir des ressources supplémentaires à précompiler lorsque rake assets:precompile est exécuté
  • config.assets.digest : cette option permet l’utilisation des empreintes digitales “MD-5” dans les noms d’actifs. Il est vrai par défaut en mode développement
  • config.assets.compile : Active/désactive la compilation “Sprockets” en direct en mode production

Configuration des générateurs #

Rails vous permet de configurer les générateurs utilisés lors de l’exécution des commandes rails generate. Cette méthode, config.generators prend un bloc

config.generators do |g|
  g.orm :active_record
  g.test_framework :test_unit
end

Voici quelques-unes des options

Options Descriptif Défaut
actifs Crée des actifs lors de la génération d’échafaudage vrai
force_pluriel Autorise les noms de modèles pluralisés faux
aide Détermine s’il faut générer des assistants vrai
outil_intégration Spécifiez l’outil d’intégration test_unit
moteur_javascript Configure le moteur JS :js
ressource_route Génère une route de ressource vrai
stylesheet_engine Configure le moteur de feuille de style :cs
échafaudage_stylesheet Crée CSS sur l’échafaudage vrai
test_framework Spécifiez le cadre de test Mini-test
template_engine Configure le moteur de modèle :erb
