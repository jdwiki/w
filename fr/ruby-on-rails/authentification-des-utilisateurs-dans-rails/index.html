Authentification des utilisateurs dans Rails

Devise est un joyau très puissant, il vous permet de vous inscrire, de vous connecter et de vous déconnecter juste après l’installation. De plus, l’utilisateur peut ajouter des authentifications et des restrictions à ses applications. Devise est également livré avec ses propres vues, si l’utilisateur souhaite l’utiliser. Un utilisateur peut également personnaliser les formulaires d’inscription et de connexion en fonction de ses besoins et de ses exigences. Il convient de noter que Devise vous recommande d’implémenter votre propre connexion si vous débutez avec les rails.

Au moment de générer des configurations de dispositif à l’aide de rails generate dispositif:install, dispositif listera un tas d’instructions sur le terminal à suivre.

Si vous avez déjà un modèle USER, l’exécution de cette commande rails generate devise USER ajoutera les colonnes nécessaires à votre modèle USER existant.

Utilisez cette méthode d’assistance before_action : authentication_user! en haut de votre contrôleur pour vérifier si user est connecté ou non. sinon, ils seront redirigés vers la page de connexion.

Authentification avec Devise #

Ajoutez gem au Gemfile :

gemme 'conception'

Exécutez ensuite la commande bundle install.

Utilisez la commande $ rails generate devise:install pour générer le fichier de configuration requis.

Configurer les options d’URL par défaut pour l’expéditeur Devise dans chaque environnement Dans l’environnement de développement, ajoutez cette ligne :

config.action_mailer.default_url_options = { host: 'localhost', port: 3000 }

à votre config/environments/development.rb

de même en production, éditez le fichier config/environments/production.rb et ajoutez

config.action_mailer.default_url_options = { host: 'your-site-url'}

Créez ensuite un modèle en utilisant : $ rails generate devise USER Où USER est le nom de la classe pour laquelle vous souhaitez implémenter l’authentification.

Enfin, lancez : rake db:migrate et vous êtes prêt.

Affichages personnalisés #

Si vous avez besoin de configurer vos vues, vous pouvez utiliser le générateur rails generate concevoir:views qui copiera toutes les vues dans votre application. Ensuite, vous pouvez les modifier comme vous le souhaitez.

Si vous avez plus d’un modèle Devise dans votre application (par exemple Utilisateur et Admin), vous remarquerez que Devise utilise les mêmes vues pour tous les modèles. Devise offre un moyen simple de personnaliser les vues. Définissez config.scoped_views = true dans le fichier config/initializers/devise.rb.

Vous pouvez également utiliser le générateur pour créer des vues étendues : rails generate concevoir:views users

Si vous souhaitez générer uniquement quelques ensembles de vues, comme celles du module enregistrable et confirmable, utilisez le drapeau -v : rails generate devise:views -v registrations confirmations

Filtres et assistants de contrôleur de conception #

Pour configurer un contrôleur avec l’authentification de l’utilisateur à l’aide de devise, ajoutez ceci before_action : (en supposant que votre modèle de devise est « Utilisateur ») :

action_avant :utilisateur_authentique !

Pour vérifier si un utilisateur est connecté, utilisez l’assistant suivant :

user_signed_in ?

Pour l’utilisateur actuellement connecté, utilisez cet assistant :

utilisateur_actuel

Vous pouvez accéder à la session pour ce champ d’application :

session_utilisateur

  • Notez que si votre modèle Devise s’appelle Member au lieu de User, remplacez user ci-dessus par member

Omniauth #

Choisissez d’abord votre stratégie d’authentification et ajoutez-la à votre Gemfile. Vous pouvez trouver une liste de stratégies ici : https://github.com/intridea/omniauth/wiki/List-of-Strategies

gem 'omniauth-github', :github => 'intridea/omniauth-github'
gem 'omniauth-openid', :github => 'intridea/omniauth-openid'

Vous pouvez ajouter ceci à votre middleware rails comme suit :

Rails.application.config.middleware.use OmniAuth::Builder do
  require 'openid/store/filesystem' 
  provider :github, ENV['GITHUB_KEY'], ENV['GITHUB_SECRET']
  provider :openid, :store => OpenID::Store::Filesystem.new('/tmp')
end

Par défaut, OmniAuth ajoutera /auth/:provider à vos itinéraires et vous pouvez commencer par utiliser ces chemins.

Par défaut, en cas d’échec, omniauth redirigera vers /auth/failure

has_secure_password #

Créer un modèle utilisateur #

les rails génèrent le modèle Email de l'utilisateur :chaîne password_digest:string

Ajouter le module has_secure_password au modèle utilisateur #

class User < ActiveRecord::Base
  has_secure_password
end

Vous pouvez maintenant créer un nouvel utilisateur avec un mot de passe

user = User.new email: 'bob@bob.com', password: 'Password1', password_confirmation: 'Password1'

Vérifier le mot de passe avec la méthode d’authentification

user.authenticate('somepassword')

has_secure_token #

Créer un modèle utilisateur

# Schema: User(token:string, auth_token:string)
class User < ActiveRecord::Base
  has_secure_token
  has_secure_token :auth_token
end

Désormais, lorsque vous créez un nouvel utilisateur, un jeton et auth_token sont automatiquement générés

user = User.new
user.save
user.token # => "pX27zsMN2ViQKta1bGfLmVJE"
user.auth_token # => "77TMHrHJFvFDwodq8w7Ev2m7"

Vous pouvez mettre à jour les jetons en utilisant regenerate_token et regenerate_auth_token

user.regenerate_token # => true
user.regenerate_auth_token # => true
