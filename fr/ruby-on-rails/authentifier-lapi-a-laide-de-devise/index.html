Authentifier l'API à l'aide de Devise

  • Commencer
  • Jeton d’authentification

Sur cette page

  • Commencer
  • Jeton d’authentification

Devise est une solution d’authentification pour Rails. Avant d’aller plus loin, je voudrais ajouter une note rapide sur l’API. Ainsi, l’API ne gère pas les sessions (est sans état), ce qui signifie une réponse après votre demande, et ne nécessite ensuite aucune attention supplémentaire, ce qui signifie qu’aucun état précédent ou futur n’est requis pour que le système fonctionne, donc chaque fois que nous demandons au serveur besoin de transmettre les détails d’authentification avec toutes les API et doit indiquer à Devise de ne pas stocker les détails d’authentification.

Commencer #

Donc, nous allons d’abord créer un projet de rails et un dispositif de configuration

créer une application ferroviaire

rails new devise_example

maintenant ajouter un devis à la liste des gemmes

vous pouvez trouver un fichier nommé ‘Gemfile’ à la racine du projet rails

Ensuite, lancez bundle install

Ensuite, vous devez lancer le générateur :

rails generate devise:install

Maintenant, sur la console, vous pouvez trouver quelques instructions, suivez-les simplement.

Générer un modèle de conception

rails generate devise MODEL

Puis lancez rake db:migrate

Pour plus de détails, rendez-vous sur : [Devise Gem][1]

Jeton d’authentification #

Le jeton d’authentification est utilisé pour authentifier un utilisateur avec un jeton unique, donc Avant de poursuivre avec la logique, nous devons d’abord ajouter le champ auth_token à un modèle Devise

Ainsi,

rails g migration add_authentication_token_to_users

class AddAuthenticationTokenToUsers < ActiveRecord::Migration
  def change
    add_column :users, :auth_token, :string, default: ""
    add_index :users, :auth_token, unique: true
  end
end

Puis lancez rake db:migrate

Maintenant, nous sommes tous prêts à faire l’authentification en utilisant auth_token

Dans app/controllers/application_controllers.rb

D’abord cette ligne

respond_to :html, :json

cela aidera l’application rails à répondre à la fois avec html et json

Alors

protect_from_forgery with: :null

changera ce :null car nous ne traitons pas de sessions.

maintenant, nous allons ajouter une méthode d’authentification dans application_controller

Ainsi, par défaut, Devise utilise l’e-mail comme champ unique, nous pouvons également utiliser des champs personnalisés. Dans ce cas, nous nous authentifierons à l’aide de user_email et auth_token.

  before_filter do
    user_email = params[:user_email].presence
    user       = user_email && User.find_by_email(user_email)

    if user && Devise.secure_compare(user.authentication_token, params[:auth_token])
      sign_in user, store: false
    end
  end

Remarque : le code ci-dessus est purement basé sur votre logique. J’essaie simplement d’expliquer l’exemple de travail.

À la ligne 6 dans le code ci-dessus, vous pouvez voir que j’ai défini store: false qui empêchera de créer une session sur chaque requête, nous avons donc obtenu une re sans état

[1] : https://github.com/plataformatec/devise#getting-started
