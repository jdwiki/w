La directive d’application #

La directive Application définit les attributs spécifiques à l’application. Il est fourni en haut du fichier global.aspx. La syntaxe de base de la directive Application est :

<%@ Application Language="C#" %>

Les attributs de la directive Application sont :

Attributs Descriptif
Hérite Nom de la classe dont hériter.
Descriptif Description textuelle de l’application. Les analyseurs et les compilateurs ignorent cela.
Langue Le langage utilisé dans les blocs de code.

La directive de contrôle #

La directive control est utilisée avec les contrôles utilisateur et apparaît dans les fichiers de contrôle utilisateur (.ascx).

La syntaxe de base de la directive Control est :

<%@ Control Language="C#"  EnableViewState="false" %>

Les attributs de la directive Control sont :

Attributs Descriptif
AutoEventWireup La valeur booléenne qui active ou désactive l’association automatique des événements aux gestionnaires.
NomClasse Nom de fichier du contrôle.
Débogage La valeur booléenne qui active ou désactive la compilation avec les symboles de débogage.
Descriptif La description textuelle de la page de contrôle, ignorée par le compilateur.
EnableViewState La valeur booléenne qui indique si l’état d’affichage est conservé dans les demandes de page.
explicite Pour le langage VB, indique au compilateur d’utiliser le mode explicite de l’option.
Hérite La classe dont hérite la page de contrôle.
Langue Le langage pour le code et le script.
Src Nom de fichier de la classe code-behind.
Strict Pour le langage VB, indique au compilateur d’utiliser l’option mode strict.

La directive “Mise en œuvre” #

La directive Implement indique que la page Web, la page maître ou la page de contrôle utilisateur doit implémenter l’interface de framework .Net spécifiée.

La syntaxe de base de la directive implements est :

<%@ Implements  Interface="interface_name" %>

La directive sur l’importation #

La directive Import importe un espace de noms dans une page Web, la page de contrôle utilisateur de l’application. Si la directive Import est spécifiée dans le fichier global.asax, elle est appliquée à l’ensemble de l’application. S’il se trouve dans une page de la page de contrôle de l’utilisateur, il est appliqué à cette page ou à ce contrôle.

La syntaxe de base pour la directive d’importation est :

<%@ namespace="System.Drawing" %>

La directive maîtresse #

La directive Master spécifie un fichier de page comme étant la page principale.

La syntaxe de base de l’exemple de directive MasterPage est :

<%@ MasterPage Language="C#"  AutoEventWireup="true"  CodeFile="SiteMater.master.cs" Inherits="SiteMaster"  %>

La directive MasterType #

La directive MasterType attribue un nom de classe à la propriété Master d’une page, pour la rendre fortement typée.

La syntaxe de base de la directive MasterType est :

<%@ MasterType attribute="value"[attribute="value" ...]  %>

La directive sur les pages #

La directive Page définit les attributs spécifiques au fichier de page pour l’analyseur de page et le compilateur.

La syntaxe de base de la directive Page est :

<%@ Page Language="C#"  AutoEventWireup="true" CodeFile="Default.aspx.cs"  Inherits="_Default"  Trace="true" %>

Les attributs de la directive Page sont :

Attributs Descriptif
AutoEventWireup La valeur booléenne qui active ou désactive les événements de page qui sont automatiquement liés aux méthodes ; par exemple, Page_Load.
Tampon La valeur booléenne qui active ou désactive la mise en mémoire tampon des réponses HTTP.
NomClasse Nom de classe de la page.
ClientCible Le navigateur pour lequel les contrôles du serveur doivent restituer le contenu.
CodeFichier Le nom du code derrière le fichier.
Débogage La valeur booléenne qui active ou désactive la compilation avec des symboles de débogage.
Descriptif La description textuelle de la page, ignorée par l’analyseur.
ActiverSessionState Il active, désactive ou rend l’état de la session en lecture seule.
EnableViewState La valeur booléenne qui active ou désactive l’état d’affichage dans les demandes de page.
PageErreur URL de redirection si une exception de page non gérée se produit.
Hérite Le nom du code derrière ou d’une autre classe.
Langue Le langage de programmation pour le code.
Src Le nom de fichier du code derrière la classe.
tracer Il active ou désactive le traçage.
TraceMode Il indique comment les messages de trace sont affichés et triés par heure ou par catégorie.
Transaction Il indique si les transactions sont prises en charge.
ValiderRequête La valeur booléenne qui indique si toutes les données d’entrée sont validées par rapport à une liste de valeurs codées en dur.

La directive OutputCache #

La directive OutputCache contrôle les stratégies de mise en cache de sortie d’une page Web ou d’un contrôle utilisateur.

La syntaxe de base de la directive OutputCache est :

<%@ OutputCache Duration="15" VaryByParam="None"  %>
