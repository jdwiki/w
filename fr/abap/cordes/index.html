Modèles de chaînes #

Les modèles de chaînes sont un moyen pratique de mélanger des chaînes littérales avec des valeurs de variables :

WRITE |Hello, { lv_name }, nice to meet you!|.

Il peut également formater des choses comme les dates. Pour utiliser le format de date de l’utilisateur connecté :

WRITE |The order was completed on { lv_date DATE = USER } and can not be changed|.

Les appels et expressions de méthode fonctionnelle sont pris en charge :

WRITE |Your token is { to_upper( lv_token ) }|.
WRITE |Version is: { cond #( when lv_date < sy-datum then 'out of date' else 'up to date' ) }|.

Attention ! La mise en œuvre directe de résultats temporaires (comme des appels de méthode) dans des modèles de chaîne peut entraîner d’énormes problèmes de performances (en savoir plus à ce sujet [ici](https://blogs.sap.com/2016/08/ 15/performance-trap-in-string-concatenations/)). Bien que son utilisation dans des instructions rarement exécutées soit acceptable, cela ralentit rapidement votre programme en boucles.

Littéraux #

ABAP propose trois opérateurs différents pour déclarer des variables de type chaîne ou char

Symboles Type interne Longueur Nom
‘…’ C 1-255 caractères littéraux de champ de texte
`…` CString 0-255 caractères littéraux de chaîne de texte
|…| CString 0-255 caractères littéraux de modèle

Notez que la plage de longueurs s’applique uniquement aux valeurs codées en dur. En interne, les variables CString ont une longueur arbitraire tandis que les variables de type C ont toujours une longueur fixe.

Concaténer des chaînes #

Les variables de type chaîne et char peuvent être concaténées à l’aide de la commande ABAP CONCATENATE. Une variable supplémentaire pour stocker les résultats est nécessaire.

Exemple:

CONCATENATE var1 var2 var3 INTO result.
"result now contains the values of var1, var2 & var3 stringed together without spaces

Sténographie

Les nouvelles versions d’ABAP offrent une variante très courte de concaténation utilisant && (opérateur de chaînage).

DATA(lw_result) = `Sum: ` && lw_sum.

Attention ! Il convient de noter que l’utilisation de résultats temporaires en combinaison avec l’opérateur de chaînage à l’intérieur des boucles peut entraîner d’énormes problèmes de performances en raison du nombre croissant d’instructions de copie (en savoir plus à ce sujet [ici](https://blogs. sap.com/2016/08/15/performance-trap-in-string-concatenations/)).
