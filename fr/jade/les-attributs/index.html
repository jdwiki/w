&les attributs #

Prononcée “et attributs”, la syntaxe “&attributs” peut être utilisée pour décomposer un objet en attributs d’un élément. #Code : div#foo(data-bar="foo")&attributs({'data-foo' : 'bar'})

Résultat: #

<div id="foo" data-bar="foo" data-foo="bar"></div>

L’objet n’a pas besoin d’être un littéral d’objet. Il peut également s’agir simplement d’une variable qui a un objet comme valeur (voir aussi Mixin Attributes)

#Code :

- var attributes = {'data-foo': 'bar'};
div#foo(data-bar="foo")&attributes(attributes)

Résultat: #

<div id="foo" data-bar="foo" data-foo="bar"></div>

Attributs normaux #

Balise HTML : #

Les attributs de balise ressemblent à html, mais leurs valeurs ne sont que du JavaScript normal.

a(href='google.com') Google
a(class='button', href='google.com') Google

Résultat: #

<a href="google.com">Google</a><a href="google.com" class="button">Google</a>

#Variable : Toutes les expressions JavaScript normales fonctionnent également :

- var authenticated = true
body(class=authenticated ? 'authed' : 'anon')

Résultat: #

<body class="authed"></body>

De nombreux attributs #

Si vous avez de nombreux attributs, vous pouvez également les répartir sur plusieurs lignes :

input(
  type='checkbox'
  name='agreement'
  checked
)

Résultat: #

<input type="checkbox" name="accord" vérifié="coché"/>

Attributs sans échappement #

Par défaut, tous les attributs sont échappés (en remplaçant les caractères spéciaux par des séquences d’échappement) pour empêcher les attaques telles que les scripts intersites. Si vous avez besoin d’utiliser des caractères spéciaux, vous pouvez utiliser != au lieu de =.

#Code :

div(escaped="<code>")
div(unescaped!="<code>")

Résultat: #

<div escaped="&lt;code&gt;"></div>
<div unescaped="<code>"></div>

Attributs booléens #

Les attributs booléens sont reflétés par Jade et acceptent les booléens, c’est-à-dire “true” ou “false”. Lorsqu’aucune valeur n’est spécifiée, true est supposé. #Code :

input(type='checkbox', checked)
input(type='checkbox', checked=true)
input(type='checkbox', checked=false)
input(type='checkbox', checked=true.toString())

Résultat: #

<input type="checkbox" checked="checked"/>
<input type="checkbox" checked="checked"/>
<input type="checkbox"/>
<input type="checkbox" checked="true"/>

Si le doctype est html, jade sait ne pas refléter l’attribut et utilise le style concis (compris par tous les navigateurs).

#Code :

doctype html
input(type='checkbox', checked)
input(type='checkbox', checked=true)
input(type='checkbox', checked=false)
input(type='checkbox', checked=true && 'checked')

Résultat: #

<!DOCTYPE html>
<input type="checkbox" checked>
<input type="checkbox" checked>
<input type="checkbox">
<input type="checkbox" checked="checked">

Attributs de style #

L’attribut style peut être une chaîne (comme n’importe quel attribut normal) mais il peut aussi être un objet, ce qui est pratique lorsque des parties du style sont générées par JavaScript. #Code : a(style={couleur : 'rouge', arrière-plan : 'vert'})

Résultat: #

<a style="color:red;background:green"></a>

Attributs de classe #

L’attribut class peut être une chaîne (comme n’importe quel attribut normal) mais il peut également s’agir d’un tableau de noms de classes, ce qui est pratique lorsqu’il est généré à partir de JavaScript. #Code :

- var classes = ['foo', 'bar', 'baz']
a(class=classes)
//- the class attribute may also be repeated to merge arrays
a.bing(class=classes class=['bing'])

Résultat: #

<a class="foo bar baz"></a><a class="bing foo bar baz bing"></a>

Il peut également s’agir d’un objet mappant des noms de classe à des valeurs vraies ou fausses, ce qui est utile pour appliquer des classes conditionnelles #Code :

- var currentUrl = '/about'
a(class={active: currentUrl === '/'} href='/') Home
a(class={active: currentUrl === '/about'} href='/about'

Résultat: #

<a href="/">Accueil</a><a href="/about" class="active">À propos</a>

Littéral de classe #

Les classes peuvent être définies à l’aide d’une syntaxe .classname : #Code : a.bouton

Résultat: #

<a class="bouton"></a>

Étant donné que les div sont un choix de balise si courant, il s’agit de la valeur par défaut si vous omettez le nom de la balise : #Code : .contenu

Résultat: #

<div class="content"></div>

ID Littéral #

Les identifiants peuvent être définis à l’aide d’une syntaxe #idname : #Code : a#lien-principal

Résultat: #

<a id="main-link"></a>

Étant donné que les div sont un choix de balise si courant, il s’agit de la valeur par défaut si vous omettez le nom de la balise : #Code : #contenu

Résultat: #

<div id="content"></div>
