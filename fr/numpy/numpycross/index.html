Syntaxe #

  • numpy.cross(a, b) # produit croisé de a et b (ou vecteurs dans a et b)
  • numpy.cross(a, b, axisa=-1) #produit croisé de vecteurs dans a avec b, s.t. les vecteurs dans a sont disposés le long de l’axe axisa
  • numpy.cross(a, b, axisa=-1, axisb=-1, axisc=-1) # produits croisés de vecteurs dans a et b, vecteurs de sortie disposés le long de l’axe spécifié par *axisc *
  • numpy.cross(a, b, axis=None) # produits croisés de vecteurs en a et b, vecteurs en a, b, et en sortie disposés le long de l’axe axis

Paramètres #

Colonne Colonne
un,b Dans l’utilisation la plus simple, ‘a’ et ‘b’ sont deux vecteurs à 2 ou 3 éléments. Ils peuvent également être des tableaux de vecteurs (c’est-à-dire des matrices à deux dimensions). Si a est un tableau et ‘b’ est un vecteur, cross(a,b) renvoie un tableau dont les éléments sont les produits croisés de chaque vecteur de a avec le vecteur b. Le b est un tableau et a est un vecteur unique, cross(a,b) renvoie un tableau dont les éléments sont les produits croisés de a avec chaque vecteur dans b. a et b peuvent tous deux être des tableaux s’ils ont la même forme. Dans ce cas, cross(a,b) renvoie cross(a[0],b[0]), cross(a[1], b[1]), ...
axisa/b Si a est un tableau, il peut avoir des vecteurs disposés sur l’axe variant le plus rapidement, l’axe variant le plus lentement ou quelque chose entre les deux. axisa indique à cross() comment les vecteurs sont disposés dans a. Par défaut, il prend la valeur de l’axe variant le plus lentement. axisb fonctionne de la même manière avec l’entrée b. Si la sortie de cross() va être un tableau, les vecteurs de sortie peuvent être disposés sur différents axes de tableau ; axisc indique à cross comment disposer les vecteurs dans son tableau de sortie. Par défaut, axisc indique l’axe variant le plus lentement.
axe Un paramètre pratique qui définit axisa, axisb et axisc tous à la même valeur si vous le souhaitez. Si axis et l’un des autres paramètres sont présents dans l’appel, la valeur de axis remplacera les autres valeurs.

Produit croisé de deux vecteurs #

Numpy fournit une fonction [cross][1] pour calculer les produits vectoriels croisés. Le produit croisé des vecteurs [1, 0, 0] et [0, 1, 0] est [0, 0, 1]. Numpy nous dit :

>>> a = np.array([1, 0, 0])
>>> b = np.array([0, 1, 0])
>>> np.cross(a, b)
array([0, 0, 1])

comme prévu.

Alors que les produits croisés ne sont normalement définis que pour les vecteurs tridimensionnels. Cependant, l’un ou l’autre des arguments de la fonction Numpy peut être deux vecteurs d’éléments. Si le vecteur c est donné comme [c1, c2], Numpy attribue zéro à la troisième dimension : [c1, c2, 0]. Alors,

>>> c = np.array([0, 2])
>>> np.cross(a, c)
array([0, 0, 2])

Contrairement à dot qui existe à la fois en tant que [fonction Numpy][2] et [méthode de ndarray][3], cross n’existe qu’en tant que fonction autonome :

>>> a.cross(b)
Traceback (most recent call last):
  File "<stdin>", line 1, in <module>
AttributeError: 'numpy.ndarray' object has no attribute 'cross'

[1] : https://docs.scipy.org/doc/numpy/reference/generated/numpy.cross.html [2] : https://docs.scipy.org/doc/numpy/reference/generated/numpy.dot.html [3] : https://docs.scipy.org/doc/numpy/reference/generated/numpy.ndarray.dot.html

Plusieurs produits croisés avec un seul appel #

L’une ou l’autre des entrées peut être un tableau de vecteurs à 3 (ou 2) éléments.

>>> a=np.array([[1,0,0],[0,1,0],[0,0,1]])
>>> b=np.array([1,0,0])
>>> np.cross(a,b)
array([[ 0,  0,  0],
       [ 0,  0, -1],
       [ 0,  1,  0]])

Le résultat dans ce cas est array([np.cross(a[0],b), np.cross(a[1],b), np.cross(a[2],b)])

b peut également être un tableau de vecteurs à 3 (ou 2) éléments, mais il doit avoir la même forme que a. Sinon, le calcul échoue avec une erreur “incompatibilité de forme”. Alors on peut avoir

>>> b=np.array([[0,0,1],[1,0,0],[0,1,0]])
>>> np.cross(a,b)
array([[ 0, -1,  0],
       [ 0,  0, -1],
       [-1,  0,  0]])

et maintenant le résultat est array([np.cross(a[0],b[0]), np.cross(a[1],b[1]), np.cross(a[2],b[ 2])])

Plus de flexibilité avec plusieurs produits croisés #

Dans nos deux derniers exemples, numpy a supposé que a[0,:] était le premier vecteur, a[1,:] le second et a[2,:] le troisième. Numpy.cross a un argument facultatif axisa qui nous permet de spécifier quel axe définit les vecteurs. Alors,

>>> a=np.array([[1,1,1],[0,1,0],[1,0,-1]])
>>> b=np.array([0,0,1])
>>> np.cross(a,b)
array([[ 1, -1,  0],
       [ 1,  0,  0],
       [ 0, -1,  0]])
>>> np.cross(a,b,axisa=0)
array([[ 0, -1,  0],
       [ 1, -1,  0],
       [ 0, -1,  0]])
>>> np.cross(a,b,axisa=1)
array([[ 1, -1,  0],
       [ 1,  0,  0],
       [ 0, -1,  0]])

Le résultat axisa=1 et le résultat par défaut sont tous les deux (np.cross([1,1,1],b), np.cross([0,1,0],b), np.cross([ 1,0,-1],b)). Par défaut, axisa indique toujours le dernier axe (variant le plus lentement) du tableau. Le résultat axisa=0 est (np.cross([1,0,1],b), np.cross([1,1,0],b), np.cross([1,0,- 1],b)).

Un paramètre facultatif similaire, axisb, remplit la même fonction pour l’entrée b, s’il s’agit également d’un tableau à 2 dimensions.

Les paramètres axisa et axisb indiquent à numpy comment distribuer les données d’entrée. Un troisième paramètre, axisc indique à numpy comment distribuer la sortie si ‘a’ ou ‘b’ est multidimensionnel. En utilisant les mêmes entrées a et b que ci-dessus, nous obtenons

>>> np.cross(a,b,1)
array([[ 1, -1,  0],
       [ 1,  0,  0],
       [ 0, -1,  0]])
>>> np.cross(a,b,1,axisc=0)
array([[ 1,  1,  0],
       [-1,  0, -1],
       [ 0,  0,  0]])
>>> np.cross(a,b,1,axisc=1)
array([[ 1, -1,  0],
       [ 1,  0,  0],
       [ 0, -1,  0]])

Ainsi, axisc=1 et la valeur par défaut axisc donnent tous deux le même résultat, c’est-à-dire que les éléments de chaque vecteur sont contigus dans l’index mobile le plus rapide du tableau de sortie. axisc est par défaut le dernier axe du tableau. axisc=0 distribue les éléments de chaque vecteur sur la dimension variable la plus lente du tableau.

Si vous voulez que axisa, axisb et axisc aient tous la même valeur, vous n’avez pas besoin de définir les trois paramètres. Vous pouvez définir un quatrième paramètre, “axe”, sur la valeur unique nécessaire et les trois autres paramètres seront automatiquement définis. axis remplace axisa, axisb ou axisc si l’un d’entre eux est présent dans l’appel de fonction.
