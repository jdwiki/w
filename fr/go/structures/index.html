Les structures sont des ensembles de diverses variables regroupées. La structure elle-même n’est qu’un package contenant des variables et les rendant facilement accessibles.

Contrairement à C, les structures de Go peuvent avoir des méthodes qui leur sont attachées. Il leur permet également d’implémenter des interfaces. Cela rend les structures de Go similaires aux objets, mais il leur manque (probablement intentionnellement) certaines fonctionnalités majeures connues dans les langages orientés objet comme l’héritage.

Méthodes #

Les méthodes de structure sont très similaires aux fonctions :

type User struct {
    name string
}

func (u User) Name() string {
    return u.name
}

func (u *User) SetName(newName string) {
    u.name = newName
}

La seule différence est l’ajout de la méthode receiver. Il peut être déclaré soit comme une instance du type, soit comme un pointeur vers une instance du type. Étant donné que SetName() modifie l’instance, le récepteur doit être un pointeur afin d’effectuer un changement permanent dans l’instance.

Par exemple:

package main

import "fmt"

type User struct {
    name string
}

func (u User) Name() string {
    return u.name
}

func (u *User) SetName(newName string) {
    u.name = newName
}

func main() {
    var me User

    me.SetName("Slim Shady")
    fmt.Println("My name is", me.Name())
}

Structure anonyme #

Il est possible de créer une structure anonyme :

data := struct {
    Number int 
    Text   string
} { 
    42, 
    "Hello world!",
}

Exemple complet :

package main

import (
    "fmt"
)

func main() {
    data := struct {Number int; Text string}{42, "Hello world!"} // anonymous struct
    fmt.Printf("%+v\n", data)
}

[jouez-y sur le terrain de jeu] (https://play.golang.org/p/atpNnP5wE_)

Composition et intégration #

La composition offre une alternative à l’héritage. Une structure peut inclure un autre type par son nom dans sa déclaration :

type Request struct {
    Resource string
}

type AuthenticatedRequest struct {
    Request
    Username, Password string
}

Dans l’exemple ci-dessus, AuthenticatedRequest contiendra quatre membres publics : Resource, Request, Username et Password.

Les structures composites peuvent être instanciées et utilisées de la même manière que les structures normales :

func main() {
    ar := new(AuthenticatedRequest)
    ar.Resource = "example.com/request"
    ar.Username = "bob"
    ar.Password = "P@ssw0rd"
    fmt.Printf("%#v", ar)
}

[jouez-le sur le terrain de jeu] (https://play.golang.org/p/MfBhvhNMa-)

Intégration #

Dans l’exemple précédent, Request est un champ intégré. La composition peut également être obtenue en incorporant un type différent. Ceci est utile, par exemple, pour décorer un Struct avec plus de fonctionnalités. Par exemple, en continuant avec l’exemple Resource, nous voulons une fonction qui formate le contenu du champ Resource pour le préfixer avec http:// ou https://. Nous avons deux options : créer les nouvelles méthodes sur AuthenticatedRequest ou les intégrer à partir d’une structure différente :

type ResourceFormatter struct {}

func(r *ResourceFormatter) FormatHTTP(resource string) string {
    return fmt.Sprintf("http://%s", resource)
}
func(r *ResourceFormatter) FormatHTTPS(resource string) string {
    return fmt.Sprintf("https://%s", resource)
}


type AuthenticatedRequest struct {
    Request
    Username, Password string
    ResourceFormatter
}

Et maintenant, la fonction principale pourrait faire ce qui suit :

func main() {
    ar := new(AuthenticatedRequest)
    ar.Resource = "www.example.com/request"
    ar.Username = "bob"
    ar.Password = "P@ssw0rd"

    println(ar.FormatHTTP(ar.Resource))
    println(ar.FormatHTTPS(ar.Resource))

    fmt.Printf("%#v", ar)
}

Regardez que le AuthenticatedRequest qui a une structure intégrée ResourceFormatter.

Mais l’inconvénient est que vous ne pouvez pas accéder aux objets en dehors de votre composition. Ainsi, ResourceFormatter ne peut pas accéder aux membres de AuthenticatedRequest.

[jouez-le sur le terrain de jeu] (https://play.golang.org/p/Ngl3D8UW5I)

## Champs exportés ou non exportés (privés ou publics) Les champs de structure dont les noms commencent par une lettre majuscule sont exportés. Tous les autres noms ne sont pas exportés.

type Account struct {
    UserID      int    // exported
    accessToken string // unexported
}

Les champs non exportés ne sont accessibles que par code dans le même package. Ainsi, si vous accédez à un champ à partir d’un package différent, son nom doit commencer par une lettre majuscule.

package main

import "bank"

func main() {
    var x = &bank.Account{
        UserID: 1,          // this works fine
        accessToken: "one", // this does not work, since accessToken is unexported
    }
}

Cependant, à partir du package bank, vous pouvez accéder à la fois à UserId et à accessToken sans problème.

Le package bank pourrait être implémenté comme ceci :

package bank

type Account struct {
    UserID int
    accessToken string
}

func ProcessUser(u *Account) {    
    u.accessToken = doSomething(u) // ProcessUser() can access u.accessToken because 
                                   // it's defined in the same package
}

Déclaration de base #

Une structure de base est déclarée comme suit :

type User struct {
    FirstName, LastName string
    Email               string
    Age                 int
}

Chaque valeur est appelée un champ. Les champs sont généralement écrits un par ligne, le nom du champ précédant son type. Des champs consécutifs du même type peuvent être combinés, comme FirstName et LastName dans l’exemple ci-dessus.

Mots clés #

Les champs de structure peuvent être associés à des balises. Ces balises peuvent être lues par le package reflect pour obtenir des informations personnalisées spécifiées sur un champ par le développeur.

struct Account {
    Username      string `json:"username"`
    DisplayName   string `json:"display_name"`
    FavoriteColor string `json:"favorite_color,omitempty"`
}

Dans l’exemple ci-dessus, les balises sont utilisées pour modifier les noms de clé utilisés par le package encoding/json lors du marshaling ou du démarshaling JSON.

Bien que la balise puisse être n’importe quelle valeur de chaîne, il est recommandé d’utiliser des paires key:"value" séparées par des espaces :

struct StructName {
    FieldName int `package1:"customdata,moredata" package2:"info"`
}

Les balises struct utilisées avec les packages encoding/xml et encoding/json sont utilisées dans toute la bibliothèque standard.

Littéraux de structure #

Une valeur d’un type struct peut être écrite à l’aide d’un * littéral struct * qui spécifie les valeurs de ses champs.

type Point struct { X, Y int }
p := Point{1, 2}

L’exemple ci-dessus spécifie chaque champ dans le bon ordre. Ce qui n’est pas utile, car les programmeurs doivent se souvenir des champs exacts dans l’ordre. Le plus souvent, une structure peut être initialisée en répertoriant certains ou tous les noms de champs et leurs valeurs correspondantes.

anim := gif.GIF{LoopCount: nframes}

Les champs omis sont définis sur la valeur zéro pour leur type.

Remarque : Les deux formes ne peuvent pas être mélangées dans le même littéral.

Faire des copies de structure. #

Une structure peut simplement être copiée à l’aide de l’affectation.

type T struct {
    I int
    S string
}

// initialize a struct
t := T{1, "one"}

// make struct copy
u := t // u has its field values equal to t

if u == t { // true
    fmt.Println("u and t are equal") // Prints: "u and t are equal"
}

Dans le cas ci-dessus, 't' et ‘u’ sont maintenant des objets séparés (valeurs struct).

Étant donné que ‘T’ ne contient aucun type de référence (tranches, carte, canaux) comme champs, ’t’ et ‘u’ ci-dessus peuvent être modifiés sans s’affecter mutuellement.

fmt.Printf("t.I = %d, u.I = %d\n", t.I, u.I) // t.I = 100, u.I = 1

Cependant, si ‘T’ contient un type de référence, par exemple :

type T struct {
    I  int
    S  string
    xs []int // a slice is a reference type
}

Ensuite, une simple copie par affectation copierait également la valeur du champ de type tranche dans le nouvel objet. Cela se traduirait par deux objets différents faisant référence au même objet tranche.

// initialize a struct
t := T{I: 1, S: "one", xs: []int{1, 2, 3}}

// make struct copy
u := t // u has its field values equal to t

Étant donné que u et t font tous deux référence à la même tranche via leur champ xs, la mise à jour d’une valeur dans la tranche d’un objet refléterait le changement dans l’autre.

// update a slice field in u
u.xs[1] = 500

fmt.Printf("t.xs = %d, u.xs = %d\n", t.xs, u.xs)
// t.xs = [1 500 3], u.xs = [1 500 3]

Par conséquent, des précautions supplémentaires doivent être prises pour s’assurer que cette propriété de type de référence ne produit pas de comportement involontaire.

Pour copier les objets ci-dessus par exemple, une copie explicite du champ tranche pourrait être effectuée :

// explicitly initialize u's slice field
u.xs = make([]int, len(t.xs))
// copy the slice values over from t
copy(u.xs, t.xs)

// updating slice value in u will not affect t
u.xs[1] = 500

fmt.Printf("t.xs = %d, u.xs = %d\n", t.xs, u.xs)
// t.xs = [1 2 3], u.xs = [1 500 3]

Structure vide #

Une structure est une séquence d’éléments nommés, appelés champs, chacun ayant un nom et un type. La structure vide n’a pas de champs, comme cette structure vide anonyme :

var s struct{}

Ou comme ce type de structure vide nommé :

type T struct{}

La chose intéressante à propos de la structure vide est que sa taille est nulle (essayez [The Go Playground][2]) :

fmt.Println(unsafe.Sizeof(s))

Ceci imprime ‘0’, donc la structure vide elle-même ne prend pas de mémoire. c’est donc une bonne option pour quitter le canal, comme (essayez [The Go Playground][1]):

package main

import (
    "fmt"
    "time"
)

func main() {
    done := make(chan struct{})
    go func() {
        time.Sleep(1 * time.Second)
        close(done)
    }()

    fmt.Println("Wait...")
    <-done
    fmt.Println("done.")
}
