Syntaxe #

  • tapez CancelFunc fonction()
  • func Background() Contexte
  • func TODO() Contexte
  • func WithCancel(parent Context) (ctx Context, cancel CancelFunc)
  • func WithDeadline (Contexte parent, heure limite. Heure) (Context, CancelFunc)
  • func WithTimeout(parent Context, timeout time.Duration) (Context, CancelFunc)
  • func WithValue(parent Context, key interface{}, val interface{})

Le package context (dans Go 1.7) ou le package golang.org/x/net/context (Pre 1.7) est une interface pour créer des contextes qui peuvent être utilisés pour transporter des valeurs et des délais de demande à travers les limites de l’API et entre services, ainsi qu’une implémentation simple de ladite interface.

de côté : le mot « contexte » est vaguement utilisé pour désigner l’arbre entier ou des feuilles individuelles de l’arbre, par ex. les valeurs réelles de context.Context.

A un niveau élevé, un contexte est un arbre. De nouvelles feuilles sont ajoutées à l’arbre lorsqu’elles sont construites (un context.Context avec une valeur parent), et les feuilles ne sont jamais supprimées de l’arbre. Tout contexte a accès à toutes les valeurs au-dessus de lui (l’accès aux données ne circule que vers le haut), et si un contexte est annulé, ses enfants sont également annulés (les signaux d’annulation se propagent vers le bas). Le signal d’annulation est mis en œuvre au moyen d’une fonction qui renvoie un canal qui sera fermé (lisible) lorsque le contexte sera annulé ; cela fait des contextes un moyen très efficace d’implémenter le modèle de concurrence de pipeline et d’annulation, ou les délais d’attente.

Par convention, les fonctions qui prennent un contexte ont pour premier argument ctx context.Context. Bien qu’il ne s’agisse que d’une convention, celle-ci doit être suivie car de nombreux outils d’analyse statique recherchent spécifiquement cet argument. Étant donné que Context est une interface, il est également possible de transformer des données contextuelles existantes (valeurs transmises tout au long d’une chaîne d’appel de requête) en un contexte Go normal et de les utiliser de manière rétrocompatible en implémentant simplement quelques méthodes. De plus, les contextes sont sûrs pour un accès simultané, vous pouvez donc les utiliser à partir de nombreuses goroutines (qu’elles s’exécutent sur des threads parallèles ou en tant que coroutines simultanées) sans crainte.

  • https://blog.golang.org/context

Arbre de contexte représenté sous la forme d’un graphe orienté #

Une arborescence de contexte simple (contenant des valeurs communes pouvant être étendues à la demande et incluses dans un contexte) construite à partir du code Go comme suit :

// Pseudo-Go
ctx := context.WithValue(
    context.WithDeadline(
        context.WithValue(context.Background(), sidKey, sid),
        time.Now().Add(30 * time.Minute),
    ),
    ridKey, rid,
)
trCtx := trace.NewContext(ctx, tr)
logCtx := myRequestLogging.NewContext(ctx, myRequestLogging.NewLogger())

Est un arbre qui peut être représenté comme un graphe orienté qui ressemble à ceci :

[![Contexte représenté sous forme de graphe orienté][1]][1]

Chaque contexte enfant a accès aux valeurs de ses contextes parents, de sorte que l’accès aux données circule vers le haut dans l’arbre (représenté par des bords noirs). Les signaux d’annulation par contre descendent dans l’arborescence (si un contexte est annulé, tous ses enfants sont également annulés). Le flux de signal d’annulation est représenté par les bords gris.

[1] : http://i.stack.imgur.com/R0CED.png

Utiliser un contexte pour annuler le travail #

Le passage d’un contexte avec un délai d’attente (ou avec une fonction d’annulation) à une fonction de longue durée peut être utilisé pour annuler le fonctionnement des fonctions :

ctx, _ := context.WithTimeout(context.Background(), 200*time.Millisecond)
for {
    select {
    case <-ctx.Done():
        return ctx.Err()
    default:
        // Do an iteration of some long running work here!
    }
}
