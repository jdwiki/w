Téléchargement Go #

Visitez la [Liste des téléchargements][1] et trouvez l’archive adaptée à votre système d’exploitation. Les noms de ces téléchargements peuvent être un peu cryptiques pour les nouveaux utilisateurs.

Les noms sont au format go[version].[système d’exploitation]-[architecture].[archive]

Pour la version, vous voulez choisir la plus récente disponible. Celles-ci devraient être les premières options que vous voyez.

Pour le système d’exploitation, cela est assez explicite, sauf pour les utilisateurs de Mac, où le système d’exploitation s’appelle “darwin”. Ceci est nommé d’après la [partie open-source du système d’exploitation utilisée par les ordinateurs Mac] [2].

Si vous utilisez une machine 64 bits (ce qui est le plus courant sur les ordinateurs modernes), la partie “architecture” du nom de fichier doit être “amd64”. Pour les machines 32 bits, ce sera “386”. Si vous êtes sur un appareil ARM comme un Raspberry Pi, vous aurez besoin de “armv6l”.

Pour la partie “archive”, les utilisateurs Mac et Windows ont deux options car Go fournit des programmes d’installation pour ces plates-formes. Pour Mac, vous voulez probablement “pkg”. Pour Windows, vous voulez probablement “msi”.

Ainsi, par exemple, si je suis sur une machine Windows 64 bits et que je veux télécharger Go 1.6.3, le téléchargement que je veux s’appellera :

go1.6.3.windows-amd64.msi

Extraction des fichiers téléchargés #

Maintenant que nous avons téléchargé une archive Go, nous devons l’extraire quelque part.

Mac et Windows #

Étant donné que des programmes d’installation sont fournis pour ces plates-formes, l’installation est facile. Exécutez simplement le programme d’installation et acceptez les valeurs par défaut.

##Linux

Il n’y a pas d’installateur pour Linux, donc un peu plus de travail est nécessaire. Vous devriez avoir téléchargé un fichier avec le suffixe “.tar.gz”. Il s’agit d’un fichier d’archive, similaire à un fichier “.zip”. Nous devons l’extraire. Nous allons extraire les fichiers Go vers /usr/local car c’est l’emplacement recommandé.

Ouvrez un terminal et changez de répertoire à l’endroit où vous avez téléchargé l’archive. C’est probablement dans Téléchargements. Si ce n’est pas le cas, remplacez le répertoire dans la commande suivante de manière appropriée.

Téléchargements de CD

Maintenant, exécutez ce qui suit pour extraire l’archive dans /usr/local, en remplaçant [filename] par le nom du fichier que vous avez téléchargé.

tar -C /usr/local -xzf [nom de fichier].tar.gz

Définition des variables d’environnement #

Il reste une étape à franchir avant d’être prêt à commencer à développer. Nous devons définir des variables d’environnement, qui sont des informations que les utilisateurs peuvent modifier pour donner aux programmes une meilleure idée de la configuration de l’utilisateur.

Les fenêtres #

Vous devez définir le GOPATH, qui est le dossier dans lequel vous allez travailler avec Go.

Vous pouvez définir des variables d’environnement via le bouton “Variables d’environnement” de l’onglet “Avancé” du panneau de configuration “Système”. Certaines versions de Windows fournissent ce panneau de configuration via l’option “Paramètres système avancés” dans le panneau de configuration “Système”.

Le nom de votre nouvelle variable d’environnement doit être “GOPATH”. La valeur doit être le chemin complet vers un répertoire dans lequel vous allez développer du code Go. Un dossier appelé “go” dans votre répertoire utilisateur est un bon choix.

Mac #

Vous devez définir le GOPATH, qui est le dossier dans lequel vous allez travailler avec Go.

Modifiez un fichier texte nommé “.bash_profile”, qui devrait se trouver dans votre répertoire utilisateur, et ajoutez la nouvelle ligne suivante à la fin, en remplaçant [zone de travail] par un chemin complet vers un répertoire dans lequel vous souhaitez travailler. Si “.bash_profile” n’existe pas, créez-le. Un dossier appelé “go” dans votre répertoire utilisateur est un bon choix.

exporter GOPATH=[zone de travail]

##Linux

Parce que Linux n’a pas d’installateur, cela nécessite un peu plus de travail. Nous devons montrer au terminal où se trouvent le compilateur Go et d’autres outils, et nous devons définir le GOPATH, qui est un dossier dans lequel vous allez travailler avec Go.

Modifiez un fichier texte nommé “.profile”, qui doit se trouver dans votre répertoire utilisateur, et ajoutez la ligne suivante à la fin, en remplaçant [work area] par un chemin complet vers un répertoire dans lequel vous souhaitez travailler avec Go. Si “.profile” n’existe pas, créez-le. Un dossier appelé “go” dans votre répertoire utilisateur est un bon choix.

Ensuite, sur une autre nouvelle ligne, ajoutez ce qui suit à votre fichier “.profile”.

export PATH=$PATH:/usr/local/go/bin

Achevé! #

Si les outils Go ne sont toujours pas disponibles dans le terminal, essayez de fermer cette fenêtre et d’ouvrir une nouvelle fenêtre de terminal.

[1] : https://golang.org/dl/ [2] : https://en.wikipedia.org/wiki/Darwin_(système_d’exploitation)

Exemple .profile ou .bash_profile #

# This is an example of a .profile or .bash_profile for Linux and Mac systems
export GOPATH=/home/user/go
export PATH=$PATH:/usr/local/go/bin
