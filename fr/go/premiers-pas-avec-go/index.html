Bonjour le monde! #

Placez le code suivant dans un nom de fichier “hello.go” :

package main

import "fmt"

func main() {
    fmt.Println("Hello, 世界")
}

Terrain de jeu

Lorsque Go est [installé correctement] [1], ce programme peut être compilé et exécuté comme ceci :

go run hello.go

Production: #

Hello, 世界

Une fois que vous êtes satisfait du code, il peut être compilé en un exécutable en exécutant :

go build hello.go

Cela créera un fichier exécutable adapté à votre système d’exploitation dans le répertoire actuel, que vous pourrez ensuite exécuter avec la commande suivante :

Linux, OSX et autres systèmes de type Unix

./hello

Les fenêtres

hello.exe

Remarque : Les caractères chinois sont importants car ils démontrent que les chaînes Go sont stockées sous forme de tranches d’octets en lecture seule.

[1] : https://golang.org/dl/

Configuration de l’environnement #

Si Go n’est pas préinstallé sur votre système, vous pouvez vous rendre sur https://golang.org/dl/ et choisir votre plate-forme pour télécharger et installer Go.

Pour configurer un environnement de développement Go de base, seules quelques-unes des nombreuses variables d’environnement qui affectent le comportement de l’outil “go” (voir : [Liste des variables d’environnement Go] [1] pour une liste complète) doivent être définies (généralement dans le fichier ~/.profile de votre shell, ou équivalent sur les systèmes d’exploitation de type Unix).

GOPATH #

Comme la variable d’environnement système PATH, le chemin Go est une liste délimitée par :(; sous Windows) de répertoires où Go recherchera les packages. L’outil go get téléchargera également les packages dans le premier répertoire de cette liste.

Le GOPATH est l’endroit où Go configurera les dossiers associés bin, pkg et src nécessaires pour l’espace de travail :

  • src — emplacement des fichiers sources : .go, .c, .g, .s
  • pkg — a compilé les fichiers .a
  • bin — contient des fichiers exécutables construits par Go

À partir de Go 1.8, la variable d’environnement “GOPATH” aura une [valeur par défaut][2] si elle n’est pas définie. Sa valeur par défaut est $HOME/go sous Unix/Linux et %USERPROFILE%/go sous Windows.

Certains outils supposent que GOPATH contiendra un seul répertoire.

GOBIN #

Le répertoire bin où go install et go get placera les binaires après la construction des packages main. Généralement, ceci est défini quelque part sur le système PATH afin que les binaires installés puissent être exécutés et découverts facilement.

GOROOT #

Il s’agit de l’emplacement de votre installation Go. Il est utilisé pour trouver les bibliothèques standard. Il est très rare d’avoir à définir cette variable car Go intègre le chemin de construction dans la chaîne d’outils. Le réglage GOROOT est nécessaire si le répertoire d’installation diffère du répertoire de construction (ou de la valeur définie lors de la construction).

[1] : https://www.wikiod.com/fr/go/premiers-pas-avec-go#Liste des variables d’environnement Go [2] : https://golang.org/doc/go1.8#gopath

FizzBuzz #

Un autre exemple de programmes de style “Hello World” est [FizzBuzz][1]. Ceci est un exemple d’implémentation de FizzBuzz. Go très idiomatique en jeu ici.

package main

// Simple fizzbuzz implementation

import "fmt"

func main() {
    for i := 1; i <= 100; i++ {
        s := ""       
        if i % 3 == 0 {
            s += "Fizz"
        }
        if i % 5 == 0 {
            s += "Buzz"
        }
        if s != "" {
            fmt.Println(s) 
        } else {
            fmt.Println(i) 
        }
    }
}

Liste des variables d’environnement Go #

Les variables d’environnement qui affectent l’outil go peuvent être visualisées via la commande go env [var ...] :

$ go env
GOARCH="amd64"
GOBIN="/home/yourname/bin"
GOEXE=""
GOHOSTARCH="amd64"
GOHOSTOS="linux"
GOOS="linux"
GOPATH="/home/yourname"
GORACE=""
GOROOT="/usr/lib/go"
GOTOOLDIR="/usr/lib/go/pkg/tool/linux_amd64"
CC="gcc"
GOGCCFLAGS="-fPIC -m64 -pthread -fmessage-length=0 -fdebug-prefix-map=/tmp/go-build059426571=/tmp/go-build -gno-record-gcc-switches"
CXX="g++"
CGO_ENABLED="1"

Par défaut, il imprime la liste sous forme de script shell ; cependant, si un ou plusieurs noms de variable sont donnés comme arguments, il imprime la valeur de chaque variable nommée.

$go env GOOS GOPATH
linux
/home/yourname

Accéder à la documentation hors ligne #

Pour une documentation complète, exécutez la commande :

godoc -http=:<port-number>

Pour une visite guidée de Go (fortement recommandé pour les débutants dans la langue) :

go tool tour

Les deux commandes ci-dessus démarreront les serveurs Web avec une documentation similaire à ce qui se trouve en ligne ici et ici respectivement.

Pour une vérification de référence rapide à partir de la ligne de commande, par exemple pour fmt.Print :

godoc cmd/fmt Print
# or
go doc fmt Print

L’aide générale est également disponible à partir de la ligne de commande :

go help [command]

Courir Go en ligne #

Le terrain de jeu Go #

Un outil Go peu connu est The Go Playground. Si l’on veut expérimenter avec Go sans le télécharger, on peut facilement le faire simplement en . . .

  1. Visiter le Playground dans leur navigateur Web
  2. Entrer son code
  3. Cliquez sur “Exécuter”

Partage de votre code #

Le Go Playground dispose également d’outils de partage ; si un utilisateur appuie sur le bouton “Partager”, un lien (comme celui-ci) sera généré et pourra être envoyé à d’autres personnes pour tester et modifier.
