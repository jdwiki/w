Les fonctions de Go fournissent un code organisé et réutilisable pour effectuer un ensemble d’actions. Les fonctions simplifient le processus de codage, évitent la logique redondante et facilitent le suivi du code. Cette rubrique décrit la déclaration et l’utilisation des fonctions, des arguments, des paramètres, des instructions de retour et des portées dans Go.

Syntaxe #

  • func() // type de fonction sans arguments et sans valeur de retour
  • func(x int) int // accepte un entier et renvoie un entier
  • func(a, b int, z float32) bool // accepte 2 entiers, un flottant et renvoie un booléen
  • func(prefix string, values ​​…int) // fonction “variadique” qui accepte une chaîne et un ou plusieurs entiers
  • func() (int, bool) // fonction renvoyant deux valeurs
  • func(a, b int, z float64, opt …interface{}) (success bool) // accepte 2 entiers, un flottant et un ou plusieurs nombres d’interfaces et renvoie une valeur booléenne nommée (qui est déjà initialisée à l’intérieur de fonction)

Fonctions littérales et fermetures #

Une fonction littérale simple, affichant Hello! sur stdout :

package main

import "fmt"

func main() {
    func(){
        fmt.Println("Hello!")
    }()
}

[jouez-le sur le terrain de jeu] (https://play.golang.org/p/upOAwpOaue)

Une fonction littérale, affichant l’argument str sur stdout :

package main

import "fmt"

func main() {
    func(str string) {
        fmt.Println(str)
    }("Hello!")
}

[jouez-le sur le terrain de jeu] (https://play.golang.org/p/jz-5wpEkY2)

Une fonction littérale, fermant la variable str :

package main

import "fmt"

func main() {
    str := "Hello!"
    func() {
        fmt.Println(str)
    }()
}

[jouez-le sur le terrain de jeu] (https://play.golang.org/p/j6ZgyAna7l)

Il est possible d’affecter une fonction littérale à une variable :

package main

import (
    "fmt"
)

func main() {
    str := "Hello!"
    anon := func() {
        fmt.Println(str)
    }
    anon()
}

[jouez-le sur le terrain de jeu] (https://play.golang.org/p/Ick7RmdTFb)

Déclaration de base #

Une fonction simple qui n’accepte aucun paramètre et ne renvoie aucune valeur :

func SayHello() {
    fmt.Println("Hello!")
}

Valeurs de retour nommées #

Les valeurs de retour peuvent être affectées à une variable locale. Une instruction return vide peut alors être utilisée pour renvoyer leurs valeurs actuelles. C’est ce qu’on appelle un retour “nu”. Les instructions de retour nues ne doivent être utilisées que dans les fonctions courtes car elles nuisent à la lisibilité dans les fonctions plus longues :

func Inverse(v float32) (reciprocal float32) {
    if v == 0 {
        return
    }
    reciprocal = 1 / v
    return
}

[jouez-le sur le terrain de jeu] (https://play.golang.org/p/dS_bGmP6W0)

//A function can also return multiple values
func split(sum int) (x, y int) {
    x = sum * 4 / 9
    y = sum - x
    return
}

[jouez-le sur le terrain de jeu] (https://play.golang.org/p/upOAwpOaue)

Deux choses importantes doivent être notées :

  • Les parenthèses autour des valeurs de retour sont obligatoires.
  • Une instruction return vide doit toujours être fournie.

Fonctions variadiques #

Une fonction variadique peut être appelée avec n’importe quel nombre d’arguments finissants. Ces éléments sont stockés dans une tranche.

package main

import "fmt"

func variadic(strs ...string) {
     // strs is a slice of string
     for i, str := range strs {
         fmt.Printf("%d: %s\n", i, str)
     }
}

func main() {
     variadic("Hello", "Goodbye")
     variadic("Str1", "Str2", "Str3")
}

[jouez-le sur le terrain de jeu] (https://play.golang.org/p/rnzg1yK_Va)

Vous pouvez également donner une tranche à une fonction variadique, avec ... :

func main() {
     strs := []string {"Str1", "Str2", "Str3"}

     variadic(strs...)
}

[jouez-y sur le terrain de jeu] (https://play.golang.org/p/gl4L5R9v8_)

Valeurs de retour #

Une fonction peut renvoyer une ou plusieurs valeurs à l’appelant :

func AddAndMultiply(a, b int) (int, int) {
    return a+b, a*b
}

La deuxième valeur de retour peut également être l’erreur var :

import errors

func Divide(dividend, divisor int) (int, error) {
    if divisor == 0 {
        return 0, errors.New("Division by zero forbidden")
    }
    return dividend / divisor, nil
}

Deux choses importantes doivent être notées :

  • La parenthèse peut être omise pour une seule valeur de retour.
  • Chaque instruction return doit fournir une valeur pour toutes les valeurs de retour déclarées.

Paramètres #

Une fonction peut éventuellement déclarer un ensemble de paramètres :

func SayHelloToMe(firstName, lastName string, age int) {
    fmt.Printf("Hello, %s %s!\n", firstName, lastName)
    fmt.Printf("You are %d", age)
}

Notez que le type de firstName est omis car il est identique à lastName.
