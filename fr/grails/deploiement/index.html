Jar exécutable #

L’un des moyens les plus simples de déployer Grails 3.x consiste à créer un fichier jar exécutable qui intègre un conteneur de servlet (Tomcat, Undertow, etc.) avec l’application.

Modifiez build.gradle :

// Remove or comment out the war plugin:
// apply plugin:"war"

// Enable the executable jar:
springBoot {
    executable = true
}

// Optional: Customize the jar properties:
//  https://docs.gradle.org/current/dsl/org.gradle.api.tasks.bundling.Jar.html
jar {
    archiveName('myapp.jar')
}

Construire en utilisant ./gradlew assemble

Le jar résultant est maintenant une application entièrement exécutable que vous pouvez démarrer :

$ head build/libs/myapp.jar
#!/bin/bash
#
#    .   ____          _            __ _ _
#   /\\ / ___'_ __ _ _(_)_ __  __ _ \ \ \ \
#  ( ( )\___ | '_ | '_| | '_ \/ _` | \ \ \ \
#   \\/  ___)| |_)| | | | | || (_| |  ) ) ) )
#    '  |____| .__|_| |_|_| |_\__, | / / / /
#   =========|_|==============|___/=/_/_/_/
#   :: Spring Boot Startup Script ::
#

Vous pouvez le démarrer comme vous le feriez normalement pour n’importe quelle application en ligne de commande :

$ ./build/libs/myapp.jar
Grails application running at http://localhost:8080 in environment: production

Il se comporte également comme un service init :

$ ln -s /opt/myapp/myapp.jar /etc/init.d/myapp
$ service myapp [start|stop|status|restart]

Une documentation détaillée se trouve sous la documentation spring-boot : http://docs.spring.io/spring-boot/docs/current/reference/html/deployment-install.html

Création de fichiers de guerre #

Lorsque nous écrivons une application Web dans Grails, pour déployer l’application, nous avons besoin d’un fichier “war” qui doit être placé dans le conteneur de servlet (Tomcat, etc.).

Allez d’abord dans le répertoire du projet :

 cd to_project_directory

  1. Création du fichier war depuis l’invite de commande :

    grails war

  2. Il est toujours recommandé de nettoyer votre application avant la création de la guerre

Application de nettoyage depuis l’invite de commande :

grails clean

En combinant les deux étapes ci-dessus en une seule, vous obtiendrez

grails clean && grails war

Vous pouvez également spécifier l’environnement dans lequel vous souhaitez créer le fichier war.

grails [environment] war

Où [environment] peut prendre les valeurs suivantes : dev, prod ou test par exemple.

Contrairement aux autres commandes, la commande war s’exécute par défaut dans l’environnement de production au lieu du développement.
