Syntaxe #

  • /* toutes les versions */
  • pour ([expression] ; [expression] ; [expression]) une_instruction
  • pour ([expression] ; [expression] ; [expression]) { zero or several statements }
  • while (expression) one_statement
  • while (expression) { zéro ou plusieurs instructions }
  • faire one_statement while (expression);
  • do { une ou plusieurs instructions } while (expression);
  • // depuis C99 en plus du formulaire ci-dessus
  • for (declaration; [expression]; [expression]) one_statement;
  • pour (déclaration ; [expression] ; [expression]) { zero or several statements }

Les instructions/boucles d’itération se répartissent en deux catégories :

  • déclaration/boucles d’itération contrôlées par la tête
  • déclaration/boucles d’itération contrôlées au pied

Instruction/boucles d’itération contrôlées par la tête #

for ([<expression>]; [<expression>]; [<expression>]) <statement>
while (<expression>) <statement>

for ([declaration expression]; [expression] [; [expression]]) statement

Instruction/boucles d’itération contrôlées au pied #

do <statement> while (<expression>);

Boucle for #

Afin d’exécuter à nouveau un bloc de code, des boucles entrent en jeu. La boucle for est à utiliser lorsqu’un bloc de code doit être exécuté un nombre fixe de fois. Par exemple, pour remplir un tableau de taille n avec les entrées de l’utilisateur, nous devons exécuter scanf() n fois.

#include <stddef.h>          // for size_t

int array[10];               // array of 10 int

for (size_t i = 0; i < 10; i++)    // i starts at 0 and finishes with 9
{
   scanf("%d", &array[i]);
}

De cette manière, l’appel de la fonction scanf() est exécuté n fois (10 fois dans notre exemple), mais n’est écrit qu’une seule fois.

Ici, la variable “i” est l’indice de boucle, et il vaut mieux la déclarer telle qu’elle est présentée. Le type size_t (type de taille) doit être utilisé pour tout ce qui compte ou boucle dans les objets de données.

Cette façon de déclarer les variables à l’intérieur du for n’est disponible que pour les compilateurs qui ont été mis à jour vers la norme C99. Si, pour une raison quelconque, vous êtes toujours bloqué avec un compilateur plus ancien, vous pouvez déclarer l’index de boucle avant la boucle for :

#include <stddef.h>        /* for size_t */
size_t i;
int array[10];             /* array of 10 int */

for (i = 0; i < 10; i++)       /* i starts at 0 and finishes at 9 */
{
   scanf("%d", &array[i]);
}

Boucle tant que #

Une boucle while est utilisée pour exécuter un morceau de code alors qu’une condition est vraie. La boucle while est à utiliser lorsqu’un bloc de code doit être exécuté un nombre variable de fois. Par exemple, le code affiché obtient l’entrée de l’utilisateur, tant que l’utilisateur insère des nombres qui ne sont pas “0”. Si l’utilisateur insère 0, la condition while n’est plus vraie, donc l’exécution quittera la boucle et continuera avec tout code suivant :

int num = 1;

while (num != 0)
{
    scanf("%d", &num);
}

Boucle Do-While #

Contrairement aux boucles for et while, les boucles do-while vérifient la véracité de la condition à la fin de la boucle, ce qui signifie que le bloc do s’exécutera une fois, puis vérifiera la condition du while en bas du bloc. Cela signifie qu’une boucle “do-while” sera toujours exécutée au moins une fois.

Par exemple, cette boucle do-while obtiendra des nombres de l’utilisateur, jusqu’à ce que la somme de ces valeurs soit supérieure ou égale à 50 :

int num, sum;
num = sum = 0;

do 
{
  scanf("%d", &num);
  sum += num;

} while (sum < 50);

Les boucles “do-while” sont relativement rares dans la plupart des styles de programmation.

Structure et flux de contrôle dans une boucle for #

for ([declaration-or-expression]; [expression2]; [expression3])
{
    /* body of the loop */
}

Dans une boucle for, la condition de boucle a trois expressions, toutes facultatives.

  • La première expression, declaration-or-expression, initialise la boucle. Il est exécuté exactement une fois au début de la boucle.

Il peut s’agir soit d’une déclaration et de l’initialisation d’une variable de boucle, soit d’une expression générale. S’il s’agit d’une déclaration, la portée de la variable déclarée est restreinte par l’instruction for.

Les versions historiques de C n’autorisaient qu’une expression, ici, et la déclaration d’une variable de boucle devait être placée avant le for.

  • La deuxième expression, expression2, est la condition de test. Il est d’abord exécuté après l’initialisation. Si la condition est “vraie”, alors le contrôle entre dans le corps de la boucle. Sinon, il se déplace vers l’extérieur du corps de la boucle à la fin de la boucle. Par la suite, cette condition est vérifiée après chaque exécution du corps ainsi que l’instruction de mise à jour. Lorsque “true”, le contrôle revient au début du corps de la boucle. La condition est généralement destinée à vérifier le nombre d’exécutions du corps de la boucle. C’est le principal moyen de sortir d’une boucle, l’autre étant d’utiliser [instructions de saut][1].
  • La troisième expression, expression3, est l’instruction de mise à jour. Il est exécuté après chaque exécution du corps de la boucle. Il est souvent utilisé pour incrémenter une variable qui compte le nombre de fois où le corps de la boucle a été exécuté, et cette variable est appelée iterator.

Chaque instance d’exécution du corps de la boucle est appelée une itération.

Exemple:

for(int i = 0; i < 10 ; i++)
{
    printf("%d", i);
}

La sortie est :

0123456789

Dans l’exemple ci-dessus, le premier i = 0 est exécuté, initialisant i. Ensuite, la condition “i < 10” est vérifiée, qui est évaluée comme étant “vraie”. Le contrôle entre dans le corps de la boucle et la valeur de “i” est imprimée. Ensuite, la commande passe à “i++”, mettant à jour la valeur de “i” de 0 à 1. Ensuite, la condition est à nouveau vérifiée et le processus continue. Cela continue jusqu’à ce que la valeur de ‘i’ devienne 10. Ensuite, la condition ‘i < 10’ évalue ‘false’, après quoi le contrôle sort de la boucle.

[1] : https://www.wikiod.com/fr/c/instructions-de-saut

Boucles infinies #

Une boucle est dite boucle infinie si le contrôle entre mais ne quitte jamais le corps de la boucle. Cela se produit lorsque la condition de test de la boucle n’est jamais évaluée à “faux”.

Exemple:

for (int i = 0; i >= 0; )
{
    /* body of the loop where i is not changed*/
}

Dans l’exemple ci-dessus, la variable “i”, l’itérateur, est initialisée à 0. La condition de test est initialement “vraie”. Cependant, i n’est modifié nulle part dans le corps et l’expression de mise à jour est vide. Par conséquent, “i” restera 0 et la condition de test ne sera jamais évaluée comme “fausse”, conduisant à une boucle infinie.

En supposant qu’il n’y a pas d’[instructions de saut,][1] une autre façon de former une boucle infinie consiste à maintenir explicitement la condition vraie :

while (true)
{
    /* body of the loop */
}

Dans une boucle for, l’instruction de condition est facultative. Dans ce cas, la condition est toujours “vraie” dans le vide, conduisant à une boucle infinie.

for (;;)
{
    /* body of the loop */
}

Cependant, dans certains cas, la condition peut être maintenue “vraie” intentionnellement, avec l’intention de quitter la boucle à l’aide d’une [instruction de saut][1] telle que “break”.

while (true)
{
    /* statements */
    if (condition)
    {
         /* more statements */
         break;
    }
}

[1] : https://www.wikiod.com/fr/c/instructions-de-saut

Déroulement de la boucle et appareil de Duff #

Parfois, la boucle directe ne peut pas être entièrement contenue dans le corps de la boucle. En effet, la boucle doit être amorcée par certaines instructions B. Ensuite, l’itération commence par quelques instructions A, qui sont ensuite suivies par B à nouveau avant de boucler.

do_B();
while (condition) {
    do_A();
    do_B();
}

Pour éviter les problèmes potentiels de copier/coller avec la répétition de B deux fois dans le code, [Duff’s Device][1] pourrait être appliqué pour démarrer la boucle à partir du milieu du corps while, en utilisant une [instruction switch ][2] et tomber par le comportement.

switch (true) while (condition) {
case false: do_A(); /* FALL THROUGH */
default:    do_B(); /* FALL THROUGH */
}

Le dispositif de Duff a en fait été inventé pour implémenter le déroulement de boucle. Imaginez l’application d’un masque à un bloc de mémoire, où “n” est un type intégral signé avec une valeur positive.

do {
    *ptr++ ^= mask;
} while (--n > 0);

Si n était toujours divisible par 4, vous pourriez dérouler cela facilement comme suit :

do {
    *ptr++ ^= mask;
    *ptr++ ^= mask;
    *ptr++ ^= mask;
    *ptr++ ^= mask;
} while ((n -= 4) > 0);

Mais, avec Duff’s Device, le code peut suivre cet idiome déroulant qui saute au bon endroit au milieu de la boucle si ’n’ n’est pas divisible par 4.

switch (n % 4) do {
case 0: *ptr++ ^= mask; /* FALL THROUGH */
case 3: *ptr++ ^= mask; /* FALL THROUGH */
case 2: *ptr++ ^= mask; /* FALL THROUGH */
case 1: *ptr++ ^= mask; /* FALL THROUGH */
} while ((n -= 4) > 0);

Ce type de déroulement manuel est rarement requis avec les compilateurs modernes, car le moteur d’optimisation du compilateur peut dérouler des boucles au nom du programmeur.
