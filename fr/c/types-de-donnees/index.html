Types entiers à largeur fixe (depuis C99) #

L’en-tête <stdint.h> fournit plusieurs définitions de type entier à largeur fixe. Ces types sont facultatifs et ne sont fournis que si la plate-forme a un type entier de la largeur correspondante, et si le type signé correspondant a une représentation en complément à deux des valeurs négatives.

Voir la section des remarques pour des conseils d’utilisation des types à largeur fixe.

/* commonly used types include */
uint32_t u32 = 32; /* exactly 32-bits wide */

uint8_t u8 = 255;  /* exactly 8-bits wide */

int64_t i64 = -65  /* exactly 64 bit in two's complement representation */

Interprétation des déclarations #

Une particularité syntaxique distinctive de C est que les déclarations reflètent l’utilisation de l’objet déclaré comme il le serait dans une expression normale.

L’ensemble d’opérateurs suivant avec une priorité et une associativité identiques est réutilisé dans les déclarateurs, à savoir :

  • l’opérateur unaire de “déréférencement” * qui dénote un pointeur ;
  • l’opérateur binaire “abonnement tableau” [] qui dénote un tableau ;
  • l’opérateur “appel de fonction” (1+n)-aire () qui dénote une fonction ;
  • les parenthèses de regroupement () qui remplacent la priorité et l’associativité du reste des opérateurs listés.

Les trois opérateurs ci-dessus ont la priorité et l’associativité suivantes :

Opérateur Priorité relative Associativité
[] (abonnement au tableau) 1 De gauche à droite
() (appel de fonction) 1 De gauche à droite
* (déréférencement) 2 De droite à gauche

Lors de l’interprétation des déclarations, il faut partir de l’identifiant vers l’extérieur et appliquer les opérateurs adjacents dans le bon ordre conformément au tableau ci-dessus. Chaque application d’un opérateur peut être remplacée par les mots anglais suivants :

| expression | Interprétation | | ——————– | ————————————————– - | | chose[X] | un tableau de taille X de… | | chose(t1, t2, t3) | une fonction prenant t1, t2, t3 et retournant… | | *chose | un pointeur vers… |

Il s’ensuit que le début de l’interprétation anglaise commencera toujours par l’identificateur et se terminera par le type qui se trouve à gauche de la déclaration.

Exemples

char *names[20];

[] a priorité sur *, donc l’interprétation est : names est un tableau de taille 20 d’un pointeur vers char.

char (*place)[10];

En cas d’utilisation de parenthèses pour remplacer la priorité, le * est appliqué en premier : place est un pointeur vers un tableau de taille 10 de char.

int fn(long, short);

Il n’y a pas de priorité à prendre en compte ici : fn est une fonction prenant long, short et renvoyant int.

int *fn(void);

Le () est appliqué en premier : fn est une fonction prenant void et renvoyant un pointeur vers int.

int (*fp)(void);

Remplacer la priorité de () : fp est un pointeur vers une fonction prenant void et renvoyant int.

int arr[5][8];

Les tableaux multidimensionnels ne font pas exception à la règle ; les opérateurs [] sont appliqués dans l’ordre de gauche à droite selon l’associativité dans le tableau : arr est un tableau de taille 5 d’un tableau de taille 8 de int.

int **ptr;

Les deux opérateurs de déréférencement ont la même priorité, de sorte que l’associativité prend effet. Les opérateurs sont appliqués dans l’ordre de droite à gauche : ptr est un pointeur vers un pointeur vers un int.

Déclarations multiples

La virgule peut être utilisée comme séparateur (*n'agissant pas* comme l'opérateur virgule) afin de délimiter plusieurs déclarations au sein d'une même instruction. La déclaration suivante contient cinq déclarations : 
int fn(void), *ptr, (*fp)(int), arr[10][20], num;

Les objets déclarés dans l’exemple ci-dessus sont :

  • fn : une fonction prenant void et renvoyant int ;
  • ptr : un pointeur vers un int ;
  • fp : un pointeur vers une fonction prenant int et renvoyant int ;
  • arr : un tableau de taille 10 d’un tableau de taille 20 de int ;
  • num'' : entier''

Interprétation alternative

Étant donné que les déclarations reflètent l’utilisation, une déclaration peut également être interprétée en termes d’opérateurs pouvant être appliqués sur l’objet et le type résultant final de cette expression. Le type qui se trouve sur le côté gauche est le résultat final obtenu après l’application de tous les opérateurs.

/*
 * Subscripting "arr" and dereferencing it yields a "char" result.
 * Particularly: *arr[5] is of type "char".
 */
char *arr[20];

/*
 * Calling "fn" yields an "int" result.
 * Particularly: fn('b') is of type "int".
 */
int fn(char);

/*
 * Dereferencing "fp" and then calling it yields an "int" result.
 * Particularly: (*fp)() is of type "int".
 */
int (*fp)(void);

/*
 * Subscripting "strings" twice and dereferencing it yields a "char" result.
 * Particularly: *strings[5][15] is of type "char"
 */
char *strings[10][20];

Types entiers et constantes #

Les entiers signés peuvent être de ces types (le int après short, ou long est facultatif):

signed char c = 127; /* required to be 1 byte, see remarks for further information. */
signed short int si = 32767; /* required to be at least 16 bits. */
signed int i = 32767; /* required to be at least 16 bits */
signed long int li = 2147483647; /* required to be at least 32 bits. */

signed long long int li = 2147483647; /* required to be at least 64 bits */

Chacun de ces types d’entiers signés a une version non signée.

unsigned int i = 65535;
unsigned short = 2767;
unsigned char = 255;

Pour tous les types sauf char, la version signed est supposée si la partie signed ou unsigned est omise. Le type char constitue un troisième type de caractère, différent de signed char et unsigned char et la signature (ou non) dépend de la plate-forme.

Différents types de constantes entières (appelées * littéraux * dans le jargon C) peuvent être écrites dans différentes bases et différentes largeurs, en fonction de leur préfixe ou suffixe.

/* the following variables are initialized to the same value: */
int d = 42;   /* decimal constant (base10) */
int o = 052;  /* octal constant (base8) */
int x = 0xaf; /* hexadecimal constants (base16) */
int X = 0XAf; /* (letters 'a' through 'f' (case insensitive) represent 10 through 15) */

Les constantes décimales sont toujours “signées”. Les constantes hexadécimales commencent par ‘0x’ ou ‘0X’ et les constantes octales commencent juste par ‘0’. Les deux derniers sont « signés » ou « non signés » selon que la valeur correspond ou non au type signé.

/* suffixes to describe width and signedness : */
long int i = 0x32; /* no suffix represent int, or long int */
unsigned int ui = 65535u; /* u or U represent unsigned int, or long int */
long int li = 65536l; /* l or L represent long int */

Sans suffixe, la constante a le premier type qui correspond à sa valeur, c’est-à-dire une constante décimale supérieure à INT_MAX est de type long si possible, ou long long sinon.

Le fichier d’en-tête <limits.h> décrit les limites des nombres entiers comme suit. Leurs valeurs définies par la mise en œuvre doivent être égales ou supérieures en amplitude (valeur absolue) à celles indiquées ci-dessous, avec le même signe.

Si la valeur d’un objet de type char sign-extend lorsqu’il est utilisé dans une expression, la valeur de CHAR_MIN doit être la même que celle de SCHAR_MIN et la valeur de CHAR_MAX doit être la même que celle de SCHAR_MAX . Si la valeur d’un objet de type char ne s’étend pas de signe lorsqu’il est utilisé dans une expression, la valeur de CHAR_MIN doit être 0 et la valeur de CHAR_MAX doit être la même que celle de UCHAR_MAX.

La norme C99 a ajouté un nouvel en-tête, <stdint.h>, qui contient des définitions pour les entiers à largeur fixe. Voir l’exemple d’entier à largeur fixe pour une explication plus détaillée.

Constantes à virgule flottante #

Le langage C a trois types de virgule flottante réels obligatoires, float, double et long double.

float f = 0.314f;        /* suffix f or F denotes type float */
double d = 0.314;        /* no suffix denotes double */
long double ld = 0.314l; /* suffix l or L denotes long double */

/* the different parts of a floating point definition are optional */
double x = 1.; /* valid, fractional part is optional */
double y = .1; /* valid, whole-number part is optional */

/* they can also defined in scientific notation */
double sd = 1.2e3; /* decimal fraction 1.2 is scaled by 10^3, that is 1200.0 */

L’en-tête <float.h> définit diverses limites pour les opérations en virgule flottante.

L’arithmétique à virgule flottante est définie par l’implémentation. Cependant, la plupart des plates-formes modernes (arm, x86, x86_64, MIPS) utilisent des opérations en virgule flottante IEEE 754.

C a également trois types de virgule flottante complexes facultatifs qui sont dérivés de ce qui précède.

Littéraux de chaîne #

Un littéral de chaîne en C est une séquence de caractères, terminée par un zéro littéral.

char* str = "hello, world"; /* string literal */

/* string literals can be used to initialize arrays */
char a1[] = "abc"; /* a1 is char[4] holding {'a','b','c','\0'} */
char a2[4] = "abc"; /* same as a1 */
char a3[3] = "abc"; /* a1 is char[3] holding {'a','b','c'}, missing the '\0' */

Les littéraux de chaîne ne sont ** pas modifiables ** (et peuvent en fait être placés dans une mémoire en lecture seule telle que .rodata). Tenter de modifier leurs valeurs entraîne un comportement indéfini.

char* s = "foobar";
s[0] = 'F'; /* undefined behaviour */

/* it's good practice to denote string literals as such, by using `const` */
char const* s1 = "foobar";
s1[0] = 'F'; /* compiler error! */

Plusieurs littéraux de chaîne sont concaténés au moment de la compilation, ce qui signifie que vous pouvez écrire des constructions comme celles-ci.

/* only two narrow or two wide string literals may be concatenated */
char* s = "Hello, " "World";

/* since C99, more than two can be concatenated */
/* concatenation is implementation defined */
char* s1 = "Hello" ", " "World";

/* common usages are concatenations of format strings */
char* fmt = "%" PRId16; /* PRId16 macro since C99 */

Les littéraux de chaîne, comme les constantes de caractères, prennent en charge différents jeux de caractères.

/* normal string literal, of type char[] */
char* s1 = "abc";

/* wide character string literal, of type wchar_t[] */
wchar_t* s2 = L"abc";

/* UTF-8 string literal, of type char[] */
char* s3 = u8"abc";

/* 16-bit wide string literal, of type char16_t[] */
char16_t* s4 = u"abc";

/* 32-bit wide string literal, of type char32_t[] */
char32_t* s5 = U"abc";
