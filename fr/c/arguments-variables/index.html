Arguments variables

Les arguments variables sont utilisés par les fonctions de la famille printf (printf, fprintf, etc.) et d’autres pour permettre à une fonction d’être appelée avec un nombre différent d’arguments à chaque fois, d’où le nom varargs.

Pour implémenter des fonctions à l’aide de la fonction d’arguments variables, utilisez #include <stdarg.h>.

Pour appeler des fonctions qui prennent un nombre variable d’arguments, assurez-vous qu’il existe un prototype complet avec les points de suspension dans la portée : void err_exit(const char *format, ...); par exemple.

Syntaxe #

  • void va_start(va_list ap, last); /* Démarre le traitement des arguments variadiques ; last est le dernier paramètre de la fonction avant les points de suspension (“…”) */
  • type va_arg(va_list ap, type); /* Récupère le prochain argument variadique dans la liste ; assurez-vous de passer le bon type promu */
  • vide va_end(va_list ap); /* Fin du traitement des arguments */
  • annuler va_copy(va_list dst, va_list src); /* C99 ou version ultérieure : copie la liste des arguments, c’est-à-dire la position actuelle dans le traitement des arguments, dans une autre liste (par exemple, pour passer plusieurs fois les arguments) */

Paramètres #

Paramètre Détails
va_list ap  pointeur d’argument, position courante dans la liste des arguments variadiques
dernier nom du dernier argument de la fonction non variadique, afin que le compilateur trouve le bon endroit pour commencer à traiter les arguments variadiques ; ne peut pas être déclaré comme une variable register, une fonction ou un type de tableau
type Type promu de l’argument variadique à lire (par exemple int pour un argument short int)
va_list src pointeur d’argument courant à copier
va_list dst nouvelle liste d’arguments à remplir

Les fonctions va_start, va_arg, va_end et va_copy sont en fait des macros.

Assurez-vous de toujours appeler va_start en premier, et une seule fois, et d’appeler va_end en dernier, et une seule fois, et à chaque point de sortie de la fonction. Ne pas le faire * peut * fonctionner sur * votre * système mais n’est sûrement ** pas ** portable et invite donc les bogues.

Prenez soin de déclarer votre fonction correctement, c’est-à-dire avec un prototype, et faites attention aux restrictions sur le dernier argument non variadique (pas register, pas une fonction ou un type de tableau). Il n’est pas possible de déclarer une fonction qui ne prend que des arguments variadiques, car au moins un argument non variadique est nécessaire pour pouvoir démarrer le traitement des arguments.

Lors de l’appel de va_arg, vous devez demander le type d’argument promu, c’est-à-dire :

  • short est promu en int (et unsigned short est également promu en int sauf si sizeof(unsigned short) == sizeof(int), auquel cas il est promu en unsigned int) .
  • float est promu à double.
  • signed char est promu en int ; unsigned char est également promu en int sauf si sizeof(unsigned char) == sizeof(int), ce qui est rarement le cas.
  • char est généralement promu en int.
  • Les types C99 comme uint8_t ou int16_t sont promus de la même manière.

Le traitement des arguments variadiques historiques (c’est-à-dire K&R) est déclaré dans <varargs.h> mais ne doit pas être utilisé car il est obsolète. Le traitement standard des arguments variadiques (celui décrit ici et déclaré dans <stdarg.h>) a été introduit en C89 ; la macro va_copy a été introduite dans C99 mais fournie par de nombreux compilateurs avant cela.

Utilisation d’un argument count explicite pour déterminer la longueur de va_list #

Avec toute fonction variadique, la fonction doit savoir comment interpréter la liste des arguments variables. Avec les fonctions printf() ou scanf(), la chaîne de format indique à la fonction à quoi s’attendre.

La technique la plus simple consiste à passer un décompte explicite des autres arguments (qui sont normalement tous du même type). Ceci est démontré dans la fonction variadique du code ci-dessous qui calcule la somme d’une série d’entiers, où il peut y avoir n’importe quel nombre d’entiers mais ce nombre est spécifié comme argument avant la liste d’arguments variables.

#include <stdio.h>
#include <stdarg.h>

/* first arg is the number of following int args to sum. */
int sum(int n, ...) {
    int sum = 0;
    va_list it; /* hold information about the variadic argument list. */

    va_start(it, n); /* start variadic argument processing */
    while (n--)
      sum += va_arg(it, int); /* get and sum the next variadic argument */
    va_end(it); /* end variadic argument processing */

    return sum;
}

int main(void)
{
    printf("%d\n", sum(5, 1, 2, 3, 4, 5)); /* prints 15 */
    printf("%d\n", sum(10, 5, 9, 2, 5, 111, 6666, 42, 1, 43, -6218)); /* prints 666 */
    return 0;
}

Utilisation des valeurs de terminaison pour déterminer la fin de va_list #

Avec toute fonction variadique, la fonction doit savoir comment interpréter la liste des arguments variables. L’approche “traditionnelle” (illustrée par printf) consiste à spécifier le nombre d’arguments à l’avance. Cependant, ce n’est pas toujours une bonne idée :

/* First argument specifies the number of parameters; the remainder are also int */
extern int sum(int n, ...);

/* But it's far from obvious from the code. */
sum(5, 2, 1, 4, 3, 6)

/* What happens if i.e. one argument is removed later on? */
sum(5, 2, 1, 3, 6)  /* Disaster */

Parfois, il est plus robuste d’ajouter un terminateur explicite, illustré par le POSIX execlp() fonction. Voici une autre fonction pour calculer la somme d’une série de nombres “doubles”:

#include <stdarg.h>
#include <stdio.h>
#include <math.h>

/* Sums args up until the terminator NAN */
double sum (double x, ...) {
  double sum = 0;
  va_list va;

  va_start(va, x);
  for (; !isnan(x); x = va_arg(va, double)) {
    sum += x;
  }
  va_end(va);

  return sum;
}

int main (void) {
  printf("%g\n", sum(5., 2., 1., 4., 3., 6., NAN));
  printf("%g\n", sum(1, 0.5, 0.25, 0.125, 0.0625, 0.03125, NAN));
}

Bonnes valeurs de terminateur :

  • entier (supposé être entièrement positif ou non négatif) — ‘0’ ou ‘-1’
  • types à virgule flottante — NAN
  • types de pointeurs — NULL
  • types d’énumérateurs - une valeur spéciale

Implémenter des fonctions avec une interface de type printf() #

Une utilisation courante des listes d’arguments de longueur variable consiste à implémenter des fonctions qui sont une mince enveloppe autour de la famille de fonctions printf(). Un tel exemple est un ensemble de fonctions de rapport d’erreurs.

errmsg.h

#ifndef ERRMSG_H_INCLUDED
#define ERRMSG_H_INCLUDED

#include <stdarg.h>
#include <stdnoreturn.h>    // C11

void verrmsg(int errnum, const char *fmt, va_list ap);
noreturn void errmsg(int exitcode, int errnum, const char *fmt, ...);
void warnmsg(int errnum, const char *fmt, ...);

#endif

Ceci est un exemple simple; ces packages peuvent être très élaborés. Normalement, les programmeurs utiliseront soit errmsg() ou warnmsg(), qui eux-mêmes utilisent verrmsg() en interne. Si quelqu’un a besoin d’en faire plus, la fonction exposée verrmsg() sera utile. Vous pouvez éviter de l’exposer jusqu’à ce que vous en ayez besoin (YAGNI - vous n’en aurez pas besoin), mais le besoin finira par survenir ( tu vas en avoir besoin - YAGNI).

errmsg.c

Ce code doit uniquement transmettre les arguments variadiques à la fonction vfprintf() pour la sortie vers l’erreur standard. Il signale également le message d’erreur système correspondant au numéro d’erreur système (errno) transmis aux fonctions.

#include "errmsg.h"
#include <stdio.h>
#include <stdlib.h>
#include <string.h>

void
verrmsg(int errnum, const char *fmt, va_list ap)
{
    if (fmt)
        vfprintf(stderr, fmt, ap);
    if (errnum != 0)
        fprintf(stderr, ": %s", strerror(errnum));
    putc('\n', stderr);
}

void
errmsg(int exitcode, int errnum, const char *fmt, ...)
{
    va_list ap;
    va_start(ap, fmt);
    verrmsg(errnum, fmt, ap);
    va_end(ap);
    exit(exitcode);
}

void
warnmsg(int errnum, const char *fmt, ...)
{
    va_list ap;
    va_start(ap, fmt);
    verrmsg(errnum, fmt, ap);
    va_end(ap);
}

Utilisation de errmsg.h

Vous pouvez maintenant utiliser ces fonctions comme suit :

#include "errmsg.h"
#include <errno.h>
#include <fcntl.h>
#include <stdio.h>
#include <stdlib.h>
#include <unistd.h>

int main(int argc, char **argv)
{
    char buffer[BUFSIZ];
    int fd;
    if (argc != 2)
    {
        fprintf(stderr, "Usage: %s filename\n", argv[0]);
        exit(EXIT_FAILURE);
    }
    const char *filename = argv[1];

    if ((fd = open(filename, O_RDONLY)) == -1)
        errmsg(EXIT_FAILURE, errno, "cannot open %s", filename);
    if (read(fd, buffer, sizeof(buffer)) != sizeof(buffer))
        errmsg(EXIT_FAILURE, errno, "cannot read %zu bytes from %s", sizeof(buffer), filename);
    if (close(fd) == -1)
        warnmsg(errno, "cannot close %s", filename);
    /* continue the program */
    return 0;
}

Si soit le open() ou read() les appels système échouent, l’erreur est écrite dans l’erreur standard et le le programme se termine avec le code de sortie 1. Si la fermer() l’appel système échoue, l’erreur est simplement imprimée comme un message d’avertissement, et le programme continue.

Vérification de l’utilisation correcte des formats printf()

Si vous utilisez GCC (le compilateur GNU C, qui fait partie du GNU Compiler Collection), ou en utilisant Clang, alors vous pouvez avoir le compilateur vérifiez que les arguments que vous passez aux fonctions de message d’erreur correspondent ce que printf() attend. Étant donné que tous les compilateurs ne prennent pas en charge l’extension, elle doit être compilée conditionnellement, ce qui est un peu fastidieux. Cependant, la protection qu’il offre en vaut la peine.

Tout d’abord, nous devons savoir comment détecter que le compilateur est GCC ou Clang émulant GCC. La réponse est que GCC définit __GNUC__ pour l’indiquer.

Voir [fonction commune attributs] (https://gcc.gnu.org/onlinedocs/gcc-7.1.0/gcc/Common-Function-Attributes.html#Common-Function-Attributes) pour plus d’informations sur les attributs - en particulier le “format” attribut.

Réécrit errmsg.h

#ifndef ERRMSG_H_INCLUDED
#define ERRMSG_H_INCLUDED

#include <stdarg.h>
#include <stdnoreturn.h>    // C11

#if !defined(PRINTFLIKE)
#if defined(__GNUC__)
#define PRINTFLIKE(n,m) __attribute__((format(printf,n,m)))
#else
#define PRINTFLIKE(n,m) /* If only */
#endif /* __GNUC__ */
#endif /* PRINTFLIKE */

void verrmsg(int errnum, const char *fmt, va_list ap);
void noreturn errmsg(int exitcode, int errnum, const char *fmt, ...)
        PRINTFLIKE(3, 4);
void warnmsg(int errnum, const char *fmt, ...)
        PRINTFLIKE(2, 3);

#endif

Maintenant, si vous faites une erreur comme :

errmsg(EXIT_FAILURE, errno, "Failed to open file '%d' for reading", filename);

(où le %d devrait être %s), alors le compilateur se plaindra :

$ gcc -O3 -g -std=c11 -Wall -Wextra -Werror -Wmissing-prototypes -Wstrict-prototypes \
>     -Wold-style-definition -c erruse.c
erruse.c: In function ‘main’:
erruse.c:20:64: error: format ‘%d’ expects argument of type ‘int’, but argument 4 has type ‘const char *’ [-Werror=format=]
         errmsg(EXIT_FAILURE, errno, "Failed to open file '%d' for reading", filename);
                                                           ~^
                                                           %s
cc1: all warnings being treated as errors
$

Utilisation d’une chaîne de format #

L’utilisation d’une chaîne de format fournit des informations sur le nombre et le type attendus des arguments variadiques suivants de manière à éviter le besoin d’un argument de comptage explicite ou d’une valeur de terminaison.

L’exemple ci-dessous montre une fonction a qui encapsule la fonction standard printf(), permettant uniquement l’utilisation d’arguments variadiques du type char, int et double (au format décimal à virgule flottante). Ici, comme avec printf(), le premier argument de la fonction d’emballage est la chaîne de format. Lorsque la chaîne de format est analysée, la fonction est capable de déterminer s’il existe un autre argument variadique attendu et quel doit être son type.

#include <stdio.h>
#include <stdarg.h>

int simple_printf(const char *format, ...)
{
    va_list ap; /* hold information about the variadic argument list. */
    int printed = 0; /* count of printed characters */

    va_start(ap, format); /* start variadic argument processing */

    while (*format != '\0') /* read format string until string terminator */
    {
        int f = 0;

        if (*format == '%')
        {
            ++format;
            switch(*format)
            {
                case 'c' :
                    f = printf("%d", va_arg(ap, int)); /* print next variadic argument, note type promotion from char to int */
                    break;
                case 'd' :
                    f = printf("%d", va_arg(ap, int)); /* print next variadic argument */
                    break;

                case 'f' :
                    f = printf("%f", va_arg(ap, double)); /* print next variadic argument */
                    break;
                default :
                    f = -1; /* invalid format specifier */
                    break;
            }
        }
        else
        {
            f = printf("%c", *format); /* print any other characters */
        }

        if (f < 0) /* check for errors */
        {
            printed = f;
            break;
        }
        else
        {
            printed += f;
        }
        ++format; /* move on to next character in string */
    }

    va_end(ap); /* end variadic argument processing */

    return printed;
}

int main (int argc, char *argv[])
{
    int x = 40;
    int y = 0;

    y = simple_printf("There are %d characters in this sentence", x);
    simple_printf("\n%d were printed\n", y);
}
