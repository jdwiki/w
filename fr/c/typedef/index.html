Le mécanisme typedef permet la création d’alias pour d’autres types. Il ne crée pas de nouveaux types. Les gens utilisent souvent typedef pour améliorer la portabilité du code, pour donner des alias aux types de structure ou d’union, ou pour créer des alias pour les types de fonction (ou de pointeur de fonction).

Dans la norme C, typedef est classé comme une ‘classe de stockage’ pour plus de commodité ; il se produit syntaxiquement là où des classes de stockage telles que static ou extern pourraient apparaître.

Syntaxe #

  • typedef nom_existant nom_alias ;

** Inconvénients de Typedef **

typedef pourrait conduire à la pollution de l’espace de noms dans les grands programmes C.

** Inconvénients des structures Typedef **

De plus, les structures typedef’ sans nom de balise sont une cause majeure d’imposition inutile de relations d’ordre entre les fichiers d’en-tête.

Envisager:

#ifndef FOO_H
#define FOO_H 1

#define FOO_DEF (0xDEADBABE)

struct bar; /* forward declaration, defined in bar.h*/

struct foo {
    struct bar *bar;
};

#endif

Avec une telle définition, sans utiliser typedefs, il est possible pour une unité de compilation d’inclure foo.h pour obtenir la définition FOO_DEF. S’il n’essaie pas de déréférencer le membre bar de la structure foo, il n’y aura pas besoin d’inclure le fichier bar.h.

Typedef contre #define

#define est une directive de préprocesseur C qui est également utilisée pour définir les alias pour divers types de données similaires à typedef mais avec les différences suivantes :

  • typedef se limite à donner des noms symboliques aux types uniquement, alors que #define peut également être utilisé pour définir des alias pour les valeurs.

  • L’interprétation typedef est effectuée par le compilateur tandis que les instructions #define sont traitées par le pré-processeur.

  • Notez que #define cptr char * suivi de cptr a, b; ne fait pas la même chose que typedef char *cptr; suivi de cptr a, b;. Avec le #define, b est une simple variable char, mais c’est aussi un pointeur avec le typedef.

Typedef pour les structures et les unions #

Vous pouvez donner des noms d’alias à un struct :

typedef struct Person {
    char name[32];
    int age;
} Person;

Person person;

Par rapport à la manière traditionnelle de déclarer des structures, les programmeurs n’auraient pas besoin d’avoir “struct” à chaque fois qu’ils déclarent une instance de cette structure.

Notez que le nom Person (par opposition à struct Person) n’est pas défini avant le dernier point-virgule. Ainsi pour les listes chaînées et les arborescences devant contenir un pointeur vers le même type de structure, il faut utiliser soit :

typedef struct Person {
    char name[32];
    int age;
    struct Person *next;
} Person;

ou:

typedef struct Person Person;

struct Person {
    char name[32];
    int age;
    Person *next;
};

L’utilisation d’un typedef pour un type union est très similaire.

typedef union Float Float;

union Float
{
    float f;
    char  b[sizeof(float)];
};

Une structure similaire à celle-ci peut être utilisée pour analyser les octets qui composent une valeur “float”.

Utilisations simples de Typedef #

Pour donner des noms courts à un type de données #

À la place de:

long long int foo;
struct mystructure object;

on peut utiliser

/* write once */
typedef long long ll;
typedef struct mystructure mystruct;

/* use whenever needed */
ll foo;
mystruct object;

Cela réduit la quantité de frappe nécessaire si le type est utilisé plusieurs fois dans le programme.

Amélioration de la portabilité #

Les attributs des types de données varient selon les différentes architectures. Par exemple, un int peut être un type à 2 octets dans une implémentation et un type à 4 octets dans une autre. Supposons qu’un programme doive utiliser un type de 4 octets pour s’exécuter correctement.

Dans une implémentation, laissez la taille de int être de 2 octets et celle de long de 4 octets. Dans un autre, laissez la taille de int être de 4 octets et celle de long de 8 octets. Si le programme est écrit en utilisant la deuxième implémentation,

/* program expecting a 4 byte integer */
int foo; /* need to hold 4 bytes to work */
/* some code involving many more ints */

Pour que le programme s’exécute dans la première implémentation, toutes les déclarations int devront être changées en long.

/* program now needs long */
long foo; /*need to hold 4 bytes to work */
/* some code involving many more longs - lot to be changed */

Pour éviter cela, on peut utiliser typedef

/* program expecting a 4 byte integer */
typedef int myint; /* need to declare once - only one line to modify if needed */
myint foo; /* need to hold 4 bytes to work */
/* some code involving many more myints */

Ensuite, seule l’instruction typedef devrait être modifiée à chaque fois, au lieu d’examiner l’ensemble du programme.

L’en-tête <stdint.h> et l’en-tête associé <inttypes.h> définissent des noms de type standard (en utilisant typedef) pour des entiers de différentes tailles, et ces noms sont souvent le meilleur choix dans le code moderne qui doit être corrigé entiers de taille. Par exemple, uint8_t est un type entier 8 bits non signé ; int64_t est un type entier 64 bits signé. Le type uintptr_t est un type entier non signé suffisamment grand pour contenir n’importe quel pointeur vers un objet. Ces types sont théoriquement facultatifs, mais il est rare qu’ils ne soient pas disponibles. Il existe des variantes comme uint_least16_t (le plus petit type d’entier non signé avec au moins 16 bits) et int_fast32_t (le type d’entier signé le plus rapide avec au moins 32 bits). De plus, intmax_t et uintmax_t sont les plus grands types d’entiers pris en charge par l’implémentation. Ces types sont obligatoires.

Pour spécifier un usage ou pour améliorer la lisibilité #

Si un ensemble de données a un but particulier, on peut utiliser typedef pour lui donner un nom significatif. De plus, si la propriété des données change de telle sorte que le type de base doit changer, seule l’instruction typedef devrait être modifiée, au lieu d’examiner l’ensemble du programme.

Typedef pour les pointeurs de fonction #

Nous pouvons utiliser typedef pour simplifier l’utilisation des pointeurs de fonction. Imaginez que nous ayons des fonctions, ayant toutes la même signature, qui utilisent leur argument pour imprimer quelque chose de différentes manières :

#include<stdio.h>

void print_to_n(int n)
{
    for (int i = 1; i <= n; ++i)
        printf("%d\n", i);
}

void print_n(int n)
{
    printf("%d\n, n);
}

Nous pouvons maintenant utiliser un typedef pour créer un type de pointeur de fonction nommé appelé imprimante :

typedef void (*printer_t)(int);

Cela crée un type, nommé printer_t pour un pointeur vers une fonction qui prend un seul argument int et ne renvoie rien, qui correspond à la signature des fonctions que nous avons ci-dessus. Pour l’utiliser, nous créons une variable du type créé et lui affectons un pointeur sur l’une des fonctions en question :

printer_t p = &print_to_n;
void (*p)(int) = &print_to_n; // This would be required without the type

Ensuite, pour appeler la fonction pointée par la variable de pointeur de fonction :

p(5);           // Prints 1 2 3 4 5 on separate lines
(*p)(5);        // So does this

Ainsi, le typedef permet une syntaxe plus simple lorsqu’il s’agit de pointeurs de fonction. Cela devient plus évident lorsque des pointeurs de fonction sont utilisés dans des situations plus complexes, telles que des arguments de fonctions.

Si vous utilisez une fonction qui prend un pointeur de fonction comme paramètre sans type de pointeur de fonction défini, la définition de la fonction serait,

void foo (void (*printer)(int), int y){
    //code
    printer(y);
    //code
}

Cependant, avec le typedef c’est:

void foo (printer_t printer, int y){
    //code
    printer(y);
    //code
}

De même, les fonctions peuvent renvoyer des pointeurs de fonction et encore une fois, l’utilisation d’un typedef peut simplifier la syntaxe lors de cette opération.

Un exemple classique est la fonction signal de <signal.h>. La déclaration correspondante (de la norme C) est :

void (*signal(int sig, void (*func)(int)))(int);

C’est une fonction qui prend deux arguments - un int et un pointeur vers une fonction qui prend un int comme argument et ne renvoie rien - et qui renvoie un pointeur vers la fonction comme son deuxième argument.

Si nous avons défini un type SigCatcher comme alias pour le pointeur vers le type de fonction :

typedef void (*SigCatcher)(int);

alors nous pourrions déclarer signal() en utilisant :

SigCatcher signal(int sig, SigCatcher func);

Dans l’ensemble, c’est plus facile à comprendre (même si la norme C n’a pas choisi de définir un type pour faire le travail). La fonction signal prend deux arguments, un int et un SigCatcher, et elle renvoie un SigCatcher — où un SigCatcher est un pointeur vers une fonction qui prend un argument int et ne renvoie rien.

Bien que l’utilisation de noms typedef pour les types de pointeur vers des fonctions facilite la vie, cela peut également prêter à confusion pour les autres qui maintiendront votre code plus tard, donc utilisez-le avec prudence et avec une documentation appropriée.
