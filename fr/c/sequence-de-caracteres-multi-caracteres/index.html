Séquence de caractères multi-caractères

  • Trigraphes
  • Digrammes

Sur cette page

  • Trigraphes
  • Digrammes

Tous les préprocesseurs ne prennent pas en charge le traitement des séquences de trigraphes. Certains compilateurs offrent une option ou un commutateur supplémentaire pour les traiter. D’autres utilisent un programme séparé pour convertir les trigraphes.

Le compilateur GCC ne les reconnaît pas à moins que vous ne le lui demandiez explicitement (utilisez -trigraphs pour les activer ; utilisez -Wtrigraphs, partie de -Wall, pour obtenir des avertissements sur les trigraphes).

Comme la plupart des plates-formes utilisées aujourd’hui prennent en charge la gamme complète de caractères uniques utilisés en C, les digraphes sont préférés aux trigraphes, mais l’utilisation de séquences de caractères à plusieurs caractères est généralement déconseillée.

Aussi, méfiez-vous de l’utilisation accidentelle de trigraphes (puts("Que s'est-il passé ??!!");, par exemple).

Trigraphes #

Les symboles [ ] { } ^ \ | ~ # sont fréquemment utilisés dans les programmes C, mais à la fin des années 1980, il y avait des jeux de codes en cours d’utilisation (variantes ISO 646, par exemple, dans les pays scandinaves) où les positions des caractères ASCII pour ceux-ci étaient utilisées pour les variantes de caractères de la langue nationale (par ex. £ pour # au Royaume-Uni ; Æ Å æ å ø Ø pour { } { } | \ au Danemark ; il n’y avait pas de ~ dans EBCDIC). Cela signifiait qu’il était difficile d’écrire du code C sur des machines qui utilisaient ces ensembles.

Pour résoudre ce problème, la norme C a suggéré l’utilisation de combinaisons de trois caractères pour produire un seul caractère appelé trigraphe. Un trigraphe est une séquence de trois caractères dont les deux premiers sont des points d’interrogation.

Voici un exemple simple qui utilise des séquences de trigraphes au lieu de #, { et } :

??=include <stdio.h>

int main()
??<
    printf("Hello World!\n");
??>

Cela sera modifié par le préprocesseur C en remplaçant les trigraphes par leurs équivalents à un seul caractère comme si le code avait été écrit :

#include <stdio.h>

int main()
{
    printf("Hello World!\n");
}
Trigraphe Équivalent
??= #
??/ \
??' ^
??( [
??) ]
??! |
??< {
??> }
??- ~

Notez que les trigraphes sont problématiques car, par exemple, ??/ est une barre oblique inverse et peut affecter la signification des lignes de continuation dans les commentaires, et doivent être reconnus à l’intérieur des chaînes et des caractères littéraux (par exemple, '??/??/' est un caractère unique, une barre oblique inverse).

Digrammes #

En 1994, des alternatives plus lisibles à cinq des trigraphes ont été fournies. Ceux-ci n’utilisent que deux caractères et sont appelés digraphes. Contrairement aux trigraphes, les digraphes sont des jetons. Si un digraphe apparaît dans un autre jeton (par exemple, des littéraux de chaîne ou des constantes de caractères), il ne sera pas traité comme un digraphe, mais restera tel quel.

Ce qui suit montre la différence avant et après le traitement de la séquence de digraphes.

#include <stdio.h>

int main()
<%
    printf("Hello %> World!\n"); /* Note that the string contains a digraph */
%>

Qui sera traité de la même manière que :

#include <stdio.h>

int main()
{
    printf("Hello %> World!\n"); /* Note the unchanged digraph within the string. */
}
Digraphe Équivalent
< : [
:> ]
<% {
%> }
% : #
