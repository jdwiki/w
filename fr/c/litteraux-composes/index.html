Littéraux composés

Sur cette page

Syntaxe #

  • (type){ initializer-list }

La norme C dit en C11-§6.5.2.5/3 :

Une expression postfixée qui consiste en un nom de type entre parenthèses suivi d’une liste d’initialiseurs entre accolades est un littéral composé. Il fournit un objet sans nom dont la valeur est donnée par la liste d’initialisation.99)

et la note de bas de page 99 dit :

Notez que cela diffère d’une expression cast. Par exemple, un cast spécifie une conversion en types scalaires ou void uniquement, et le résultat d’une expression de cast n’est pas une lvalue.

Notez que:

Les littéraux de chaîne et les littéraux composés avec des types const-qualifiés n’ont pas besoin de désigner des objets distincts.101)

101) Cela permet aux implémentations de partager le stockage des littéraux de chaîne et des littéraux composés constants avec des représentations identiques ou qui se chevauchent.

Un exemple est donné dans la norme : C11-§6.5.2.5/13 :

Comme les littéraux de chaîne, les littéraux composés const-qualifiés peuvent être placés dans une mémoire en lecture seule et peuvent même être partagés. Par exemple,

(const char []){“abc”} == “abc” peut donner 1 si le stockage des littéraux est partagé.

Définition/Initialisation des littéraux composés #

Un littéral composé est un objet sans nom qui est créé dans la portée où est défini. Le concept a été introduit pour la première fois dans la norme C99. Un exemple de littéral composé est

Exemples de la norme C, C11-§6.5.2.5/9 : #

entier *p = (entier [2]){ 2, 4 } ;

‘p’ est initialisé à l’adresse du premier élément d’un tableau sans nom de deux entiers.

Le littéral composé est une lvalue. La durée de stockage de l’objet sans nom est soit statique (si le littéral apparaît au niveau du fichier), soit automatique (si le littéral apparaît au niveau du bloc), et dans ce dernier cas, la durée de vie de l’objet se termine lorsque le contrôle quitte le bloc englobant.

vide f(vide) { entier p ; /…/ p = (entier [2]){ p } ; /…/ } p se voit attribuer l’adresse du premier élément d’un tableau de deux entiers, le premier ayant la valeur précédemment pointée par p et le second, zéro.[…]

Ici ‘p’ reste valide jusqu’à la fin du bloc.

Littéral composé avec des désignateurs #

(également à partir de C11)

  struct point {
    unsigned x;
    unsigned y;
  };

  extern void drawline(struct point, struct point);

 // used somewhere like this
 drawline((struct point){.x=1, .y=1}, (struct point){.x=3, .y=4});

Une fonction fictive drawline reçoit deux arguments de type struct pointe'. Le premier a pour coordonnées x == 1ety == 1, alors que le la seconde a x == 3ety == 4`

Littéral composé sans spécifier la longueur du tableau #

int *p = (int []){ 1, 2, 3};

Dans ce cas, la taille du tableau n’est pas spécifiée, elle sera déterminée par la longueur de l’initialiseur.

Littéral composé ayant une longueur d’initialiseur inférieure à la taille de tableau spécifiée #

int *p = (int [10]){1, 2, 3};

le reste des éléments du littéral composé sera implicitement initialisé à “0”.

Littéral composé en lecture seule #

Notez qu’un littéral composé est une lvalue et donc ses éléments peuvent être modifiables. Un littéral composé en lecture seule peut être spécifié en utilisant le qualificatif const comme (const int[]){1,2}.

Littéral composé contenant des expressions arbitraires #

A l’intérieur d’une fonction, un littéral composé, comme pour toute initialisation depuis C99, peut avoir des expressions arbitraires.

void foo()
{
    int *p;
    int i = 2; j = 5;
    /*...*/
    p = (int [2]){ i+j, i*j };
    /*...*/
}
