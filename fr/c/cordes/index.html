En C, une chaîne n’est pas un type intrinsèque. Une chaîne C est la convention pour avoir un tableau unidimensionnel de caractères qui se termine par un caractère nul, par un '\0'.

Cela signifie qu’une chaîne C avec un contenu de "abc" aura quatre caractères 'a', 'b', 'c' et '\0'.

Voir l’exemple [introduction de base aux chaînes][1].

[1] : https://www.wikiod.com/fr/c/cordes#Introduction de base aux chaînes

Syntaxe #

  • char str1[] = “Bonjour, tout le monde !” ; /* modifiable */
  • char str2[14] = “Bonjour, monde !” ; /* modifiable */
  • char* str3 = “Bonjour, monde!”; /* Non modifiable*/

Tokénisation : strtok(), strtok_r() et strtok_s() #

La fonction strtok décompose une chaîne en chaînes plus petites, ou jetons, à l’aide d’un ensemble de délimiteurs.

#include <stdio.h>
#include <string.h>

int main(void)
{
    int toknum = 0;
    char src[] = "Hello,, world!";
    const char delimiters[] = ", !";
    char *token = strtok(src, delimiters);
    while (token != NULL)
    {
        printf("%d: [%s]\n", ++toknum, token);
        token = strtok(NULL, delimiters);
    }
    /* source is now "Hello\0, world\0\0" */
}

Production:

1: [Hello]
2: [world]

La chaîne de délimiteurs peut contenir un ou plusieurs délimiteurs et différentes chaînes de délimiteurs peuvent être utilisées avec chaque appel à strtok.

Les appels à strtok pour continuer à segmenter la même chaîne source ne doivent pas transmettre à nouveau la chaîne source, mais à la place transmettre NULL comme premier argument. Si la même chaîne source est transmise, le premier jeton sera à la place re-tokenisé. Autrement dit, étant donné les mêmes délimiteurs, strtok renverrait simplement le premier jeton à nouveau.

Notez que comme strtok n’alloue pas de nouvelle mémoire pour les jetons, il modifie la chaîne source. Autrement dit, dans l’exemple ci-dessus, la chaîne src sera manipulée pour produire les jetons référencés par le pointeur renvoyé par les appels à strtok. Cela signifie que la chaîne source ne peut pas être const (elle ne peut donc pas être une chaîne littérale). Cela signifie également que l’identité de l’octet de délimitation est perdue (c’est-à-dire que dans l’exemple, “,” et “!” sont effectivement supprimés de la chaîne source et vous ne pouvez pas dire quel caractère délimiteur correspond).

Notez également que plusieurs délimiteurs consécutifs dans la chaîne source sont traités comme un seul ; dans l’exemple, la deuxième virgule est ignorée.

strtok n’est ni thread-safe ni ré-entrant car il utilise un tampon statique lors de l’analyse. Cela signifie que si une fonction appelle strtok, aucune fonction qu’elle appelle pendant qu’elle utilise strtok ne peut également utiliser strtok, et elle ne peut être appelée par aucune fonction qui utilise elle-même strtok.

Voici un exemple qui illustre les problèmes causés par le fait que strtok n’est pas réentrant :

char src[] = "1.2,3.5,4.2";
char *first = strtok(src, ","); 

do 
{
    char *part;
    /* Nested calls to strtok do not work as desired */
    printf("[%s]\n", first);
    part = strtok(first, ".");
    while (part != NULL)
    {
        printf(" [%s]\n", part);
        part = strtok(NULL, ".");
    }
} while ((first = strtok(NULL, ",")) != NULL);

Production:

[1.2]
 [1]
 [2]

L’opération attendue est que la boucle do while externe doit créer trois jetons constitués de chaque chaîne de nombres décimaux ("1.2", "3.5", "4.2"), pour chacun desquels le strtok les appels pour la boucle interne doivent la diviser en chaînes de chiffres distinctes ("1", "2", "3", "5", "4", "2").

Cependant, comme strtok n’est pas ré-entrant, cela ne se produit pas. Au lieu de cela, le premier strtok crée correctement le jeton “1.2\0”, et la boucle interne crée correctement les jetons "1" et "2". Mais alors le strtok dans la boucle externe est à la fin de la chaîne utilisée par la boucle interne et renvoie NULL immédiatement. Les deuxième et troisième sous-chaînes du tableau src ne sont pas du tout analysées.

Les bibliothèques C standard ne contiennent pas de version thread-safe ou ré-entrante, mais d’autres le font, comme POSIX’ strtok_r. Notez que sur MSVC, l’équivalent de strtok, strtok_s est thread-safe.

C11 a une partie optionnelle, l’annexe K, qui offre une version thread-safe et ré-entrante nommée strtok_s. Vous pouvez tester la fonctionnalité avec __STDC_LIB_EXT1__. Cette partie facultative n’est pas largement prise en charge.

La fonction strtok_s diffère de la fonction POSIX strtok_r en protégeant contre le stockage en dehors de la chaîne en cours de tokenisation et en vérifiant les contraintes d’exécution. Sur les programmes correctement écrits, cependant, strtok_s et strtok_r se comportent de la même manière.

L’utilisation de strtok_s avec l’exemple donne maintenant la réponse correcte, comme ceci :

/* you have to announce that you want to use Annex K */ 
#define __STDC_WANT_LIB_EXT1__ 1
#include <string.h>

#ifndef __STDC_LIB_EXT1__
# error "we need strtok_s from Annex K"
#endif

char src[] = "1.2,3.5,4.2";  
char *next = NULL;
char *first = strtok_s(src, ",", &next);

do 
{
    char *part;
    char *posn;

    printf("[%s]\n", first);
    part = strtok_s(first, ".", &posn);
    while (part != NULL)
    {
        printf(" [%s]\n", part);
        part = strtok_s(NULL, ".", &posn);
    }
} 
while ((first = strtok_s(NULL, ",", &next)) != NULL);

Et la sortie sera :

[1.2]
 [1]
 [2]
[3.5]
 [3]
 [5]
[4.2]
 [4]
 [2]

Littéraux de chaîne #

Les littéraux de chaîne représentent des tableaux [static-duration][1] terminés par null de char. Parce qu’ils ont une durée de stockage statique, un littéral de chaîne ou un pointeur vers le même tableau sous-jacent peut être utilisé en toute sécurité de plusieurs manières qu’un pointeur vers un tableau automatique ne peut pas. Par exemple, renvoyer un littéral de chaîne à partir d’une fonction a un comportement bien défini :

const char *get_hello() {
    return "Hello, World!";  /* safe */
}

Pour des raisons historiques, les éléments du tableau correspondant à un littéral de chaîne ne sont pas formellement const. Néanmoins, toute tentative de les modifier a [comportement indéfini][2]. En règle générale, un programme qui tente de modifier le tableau correspondant à un littéral de chaîne se bloque ou fonctionne mal.

char *foo = "hello";
foo[0] = 'y';  /* Undefined behavior - BAD! */

Lorsqu’un pointeur pointe vers un littéral de chaîne - ou lorsqu’il peut parfois le faire - il est conseillé de déclarer le référent de ce pointeur “const” pour éviter d’engager accidentellement un tel comportement indéfini.

const char *foo = "hello";
/* GOOD: can't modify the string pointed to by foo */

D’un autre côté, un pointeur vers ou dans le tableau sous-jacent d’un littéral de chaîne n’est pas lui-même intrinsèquement spécial ; sa valeur peut librement être modifiée pour pointer vers autre chose :

char *foo = "hello";
foo = "World!"; /* OK - we're just changing what foo points to */

De plus, bien que les initialiseurs pour les tableaux “char” puissent avoir la même forme que les littéraux de chaîne, l’utilisation d’un tel initialiseur ne confère pas les caractéristiques d’un littéral de chaîne au tableau initialisé. L’initialiseur désigne simplement la longueur et le contenu initial du tableau. En particulier, les éléments sont modifiables s’ils ne sont pas explicitement déclarés ‘const’ :

char foo[] = "hello";
foo[0] = 'y';  /* OK! */

[1] : https://www.wikiod.com/fr/c/classes-de-stockage#statique [2] : https://www.wikiod.com/fr/c/comportement-indefini

Calculer la longueur : strlen() #

#include <stdio.h>
#include <stdlib.h>
#include <string.h>

int main(int argc, char **argv) 
{
    /* Exit if no second argument is found. */
    if (argc != 2) 
    {
        puts("Argument missing.");
        return EXIT_FAILURE;
    }

    size_t len = strlen(argv[1]);
    printf("The length of the second argument is %zu.\n", len);

    return EXIT_SUCCESS;
}

Ce programme calcule la longueur de son deuxième argument d’entrée et stocke le résultat dans len. Il imprime ensuite cette longueur sur le terminal. Par exemple, lorsqu’il est exécuté avec les paramètres program_name "Hello, world!", le programme affichera The length of the second argument is 13. car la chaîne Hello, world! comporte 13 caractères.

strlen compte tous les octets depuis le début de la chaîne jusqu’au caractère NUL de fin, '\0', mais sans l’inclure. En tant que tel, il ne peut être utilisé que lorsque la chaîne est garantie d’être terminée par NUL.

Gardez également à l’esprit que si la chaîne contient des caractères Unicode, strlen ne vous dira pas combien de caractères se trouvent dans la chaîne (puisque certains caractères peuvent avoir plusieurs octets de long). Dans de tels cas, vous devez compter vous-même les caractères (c’est-à-dire, les unités de code). Considérez la sortie de l’exemple suivant :

#include <stdio.h>
#include <stdlib.h>
#include <string.h>

int main(void) 
{
    char asciiString[50] = "Hello world!";
    char utf8String[50] = "Γειά σου Κόσμε!"; /* "Hello World!" in Greek */

    printf("asciiString has %zu bytes in the array\n", sizeof(asciiString));
    printf("utf8String has %zu bytes in the array\n", sizeof(utf8String));
    printf("\"%s\" is %zu bytes\n", asciiString, strlen(asciiString));
    printf("\"%s\" is %zu bytes\n", utf8String, strlen(utf8String));
}

Production:

asciiString has 50 bytes in the array
utf8String has 50 bytes in the array
"Hello world!" is 12 bytes
"Γειά σου Κόσμε!" is 27 bytes

Introduction de base aux chaînes #

En C, une chaîne est une séquence de caractères terminée par un caractère nul (’\0’).

Nous pouvons créer des chaînes en utilisant des littéraux de chaîne, qui sont des séquences de caractères entourées de guillemets doubles ; par exemple, prenez la chaîne littérale "hello world". Les littéraux de chaîne sont automatiquement terminés par null.

Nous pouvons créer des chaînes en utilisant plusieurs méthodes. Par exemple, nous pouvons déclarer un char * et l’initialiser pour pointer vers le premier caractère d’une chaîne :

char * string = "hello world";

Lors de l’initialisation d’un char * à une constante de chaîne comme ci-dessus, la chaîne elle-même est généralement allouée en données en lecture seule ; string est un pointeur vers le premier élément du tableau, qui est le caractère 'h'.

Étant donné que le littéral de chaîne est alloué dans la mémoire en lecture seule, il n’est pas modifiable1. Toute tentative de modification conduira à [undefined behavior][1], il est donc préférable d’ajouter const pour obtenir une erreur de compilation comme celle-ci

char const * string = "hello world";

Il a un effet similaire2 comme

char const string_arr[] = "hello world";

Pour créer une chaîne modifiable, vous pouvez déclarer un tableau de caractères et initialiser son contenu à l’aide d’un littéral de chaîne, comme ceci :

char modifiable_string[] = "hello world";

Cela équivaut à ce qui suit :

char modifiable_string[] = {'h', 'e', 'l', 'l', 'o', ' ', 'w', 'o', 'r', 'l', 'd', '\0'};

Étant donné que la deuxième version utilise un initialiseur entre accolades, la chaîne n’est pas automatiquement terminée par un caractère nul à moins qu’un caractère '\0' ne soit inclus explicitement dans le tableau de caractères, généralement comme dernier élément.

1 Non modifiable implique que les caractères de la chaîne littérale ne peuvent pas être modifiés, mais rappelez-vous que le pointeur chaîne peut être modifié (peut pointer ailleurs ou peut être incrémenté ou décrémenté).

2 Les deux chaînes ont un effet similaire dans le sens où les caractères des deux chaînes ne peuvent pas être modifiés. Il convient de noter que string est un pointeur vers char et c’est une modifiable l-value donc elle peut être incrémentée ou pointer vers un autre emplacement alors que le tableau string_arr est une l-value non modifiable, il ne peut pas être modifié.

[1] : https://www.wikiod.com/fr/c/comportement-indefini

Copie de chaînes #

Les affectations de pointeur ne copient pas les chaînes #

Vous pouvez utiliser l’opérateur = pour copier des entiers, mais vous ne pouvez pas utiliser l’opérateur = pour copier des chaînes en C. Les chaînes en C sont représentées comme des tableaux de caractères avec un caractère nul de fin, donc l’utilisation de l’opérateur = ne sauvegardera que l’adresse (pointeur) d’une chaîne.

#include <stdio.h>

int main(void) {
    int a = 10, b;
    char c[] = "abc", *d;

    b = a; /* Integer is copied */
    a = 20; /* Modifying a leaves b unchanged - b is a 'deep copy' of a */
    printf("%d %d\n", a, b); /* "20 10" will be printed */

    d = c; 
    /* Only copies the address of the string - 
    there is still only one string stored in memory */
    
    c[1] = 'x';
    /* Modifies the original string - d[1] = 'x' will do exactly the same thing */

    printf("%s %s\n", c, d); /* "axc axc" will be printed */

    return 0;
}

L’exemple ci-dessus a été compilé parce que nous avons utilisé char *d plutôt que char d[3]. L’utilisation de ce dernier provoquerait une erreur de compilation. Vous ne pouvez pas affecter à des tableaux en C.

#include <stdio.h>

int main(void) {
    char a[] = "abc";
    char b[8];

    b = a; /* compile error */
    printf("%s\n", b);

    return 0;
}

Copier des chaînes à l’aide de fonctions standard #

strcpy() #

Pour copier réellement des chaînes, la fonction strcpy() est disponible dans string.h. Un espace suffisant doit être alloué à la destination avant la copie.

#include <stdio.h>
#include <string.h>

int main(void) {
    char a[] = "abc";
    char b[8];

    strcpy(b, a); /* think "b special equals a" */
    printf("%s\n", b); /* "abc" will be printed */

    return 0;
}

snprintf() #

Pour éviter le dépassement de mémoire tampon, snprintf() peut être utilisé. Ce n’est pas la meilleure solution en termes de performances puisqu’elle doit analyser la chaîne de modèle, mais c’est la seule fonction sans limite de mémoire tampon pour copier des chaînes facilement disponible dans la bibliothèque standard, qui peut être utilisée sans aucune étape supplémentaire.

#include <stdio.h>
#include <string.h>

int main(void) {
    char a[] = "012345678901234567890";
    char b[8];

#if 0
    strcpy(b, a); /* causes buffer overrun (undefined behavior), so do not execute this here! */
#endif

    snprintf(b, sizeof(b), "%s", a); /* does not cause buffer overrun */
    printf("%s\n", b); /* "0123456" will be printed */

    return 0;
}

strncat() #

Une deuxième option, avec de meilleures performances, consiste à utiliser strncat() (une version de contrôle de débordement de tampon de strcat() ) - il prend un troisième argument qui lui indique le nombre maximum d’octets à copier :

char dest[32];

dest[0] = '\0';
strncat(dest, source, sizeof(dest) - 1);
    /* copies up to the first (sizeof(dest) - 1) elements of source into dest,
    then puts a \0 on the end of dest */

Notez que cette formulation utilise sizeof(dest) - 1 ; c’est crucial car strncat() ajoute toujours un octet nul (bon), mais ne le compte pas dans la taille de la chaîne (une cause de confusion et d’écrasements de tampon).

Notez également que l’alternative - concaténer après une chaîne non vide - est encore plus difficile. Envisager:

char dst[24] = "Clownfish: ";
char src[] = "Marvin and Nemo";
size_t len = strlen(dst);

strncat(dst, src, sizeof(dst) - len - 1);
printf("%zu: [%s]\n", strlen(dst), dst);

La sortie est :

23: [Clownfish: Marvin and N]

Notez, cependant, que la taille spécifiée comme longueur n’était pas la taille du tableau de destination, mais la quantité d’espace qui y restait, sans compter l’octet nul du terminal. Cela peut causer de gros problèmes d’écrasement. C’est aussi un peu de gaspillage; pour spécifier correctement l’argument de longueur, vous connaissez la longueur des données dans la destination, vous pouvez donc à la place spécifier l’adresse de l’octet nul à la fin du contenu existant, évitant ainsi à strncat() de le réanalyser :

    strcpy(dst, "Clownfish: ");
    assert(len < sizeof(dst) - 1);
    strncat(dst + len, src, sizeof(dst) - len - 1);
    printf("%zu: [%s]\n", strlen(dst), dst);

Cela produit la même sortie qu’auparavant, mais strncat() n’a pas à parcourir le contenu existant de dst avant de commencer la copie.

strncpy() #

La dernière option est la fonction strncpy(). Bien que vous puissiez penser qu’il devrait venir en premier, c’est une fonction plutôt trompeuse qui a deux pièges principaux :

  1. Si la copie via strncpy() atteint la limite de mémoire tampon, un caractère nul de fin ne sera pas écrit.
  2. strncpy() remplit toujours complètement la destination, avec des octets nuls si nécessaire.

(Cette implémentation originale est historique et [était initialement destinée à gérer les noms de fichiers UNIX][1])

La seule façon correcte de l’utiliser est d’assurer manuellement la terminaison nulle :

strncpy(b, a, sizeof(b)); /* the third parameter is destination buffer size */
b[sizeof(b)/sizeof(*b) - 1] = '\0'; /* terminate the string */
printf("%s\n", b); /* "0123456" will be printed */

Même dans ce cas, si vous avez un gros tampon, il devient très inefficace d’utiliser strncpy() à cause du remplissage nul supplémentaire.

[1] : https://blogs.msdn.microsoft.com/oldnewthing/20050107-00/?p=36773/

Itération sur les caractères d’une chaîne #

Si nous connaissons la longueur de la chaîne, nous pouvons utiliser une boucle for pour parcourir ses caractères :

char * string = "hello world"; /* This 11 chars long, excluding the 0-terminator. */
size_t i = 0;
for (; i < 11; i++) {
    printf("%c\n", string[i]);    /* Print each character of the string. */
}

Alternativement, nous pouvons utiliser la fonction standard strlen() pour obtenir la longueur d’une chaîne si nous ne savons pas quelle est la chaîne :

size_t length = strlen(string);
size_t i = 0; 
for (; i < length; i++) {
    printf("%c\n", string[i]);    /* Print each character of the string. */
}

Enfin, nous pouvons tirer parti du fait que les chaînes en C sont garanties à terminaison nulle (ce que nous avons déjà fait en le passant à strlen() dans l’exemple précédent ;-)). Nous pouvons itérer sur le tableau quelle que soit sa taille et arrêter l’itération une fois que nous atteignons un caractère nul :

size_t i = 0;
while (string[i] != '\0') {       /* Stop looping when we reach the null-character. */
    printf("%c\n", string[i]);    /* Print each character of the string. */
    i++;
}

Création de tableaux de chaînes #

Un tableau de chaînes peut signifier plusieurs choses :

  1. Un tableau dont les éléments sont char *s
  2. Un tableau dont les éléments sont des tableaux de chars

Nous pouvons créer un tableau de pointeurs de caractères comme ceci :

char * string_array[] = {
    "foo",
    "bar",
    "baz"
};

N’oubliez pas : lorsque nous attribuons des littéraux de chaîne à char *, les chaînes elles-mêmes sont allouées en mémoire en lecture seule. Cependant, le tableau string_array est alloué en mémoire lecture/écriture. Cela signifie que nous pouvons modifier les pointeurs dans le tableau, mais nous ne pouvons pas modifier les chaînes vers lesquelles ils pointent.

En C, le paramètre principal de argv (le tableau des arguments de ligne de commande passés lors de l’exécution du programme) est un tableau de char * : char * argv[].

Nous pouvons également créer des tableaux de tableaux de caractères. Comme les chaînes sont des tableaux de caractères, un tableau de chaînes est simplement un tableau dont les éléments sont des tableaux de caractères :

char modifiable_string_array_literals[][4] = {
    "foo",
    "bar",
    "baz"
};

Cela équivaut à :

char modifiable_string_array[][4] = {
    {'f', 'o', 'o', '\0'},
    {'b', 'a', 'r', '\0'},
    {'b', 'a', 'z', '\0'}
};

Notez que nous spécifions 4 comme taille de la deuxième dimension du tableau ; chacune des chaînes de notre tableau est en fait de 4 octets puisque nous devons inclure le caractère de terminaison nul.

Convertir les chaînes en nombre : atoi(), atof() (dangereux, ne les utilisez pas) #

Avertissement : Les fonctions atoi, atol, atoll et atof sont intrinsèquement dangereuses, car : [Si la valeur du résultat ne peut pas être représentée, le comportement est indéfini.](http://www. open-std.org/jtc1/sc22/wg14/www/docs/n1124.pdf) (7.20.1p1)

#include <stdio.h>
#include <stdlib.h>

int main(int argc, char** argv)
{
    int val;
    if (argc < 2)
    {
        printf("Usage: %s <integer>\n", argv[0]);
        return 0;
    }

    val = atoi(argv[1]);

    printf("String value = %s, Int value = %d\n", argv[1], val);

    return 0;
}

Lorsque la chaîne à convertir est un entier décimal valide compris dans la plage, la fonction fonctionne :

$ ./atoi 100
String value = 100, Int value = 100
$ ./atoi 200
String value = 200, Int value = 200

Pour les chaînes qui commencent par un nombre, suivi d’autre chose, seul le nombre initial est analysé :

$ ./atoi 0x200
0
$ ./atoi 0123x300
123

Dans tous les autres cas, le comportement est indéfini :

$ ./atoi hello
Formatting the hard disk...

En raison des ambiguïtés ci-dessus et de ce comportement indéfini, la famille de fonctions “atoi” ne doit jamais être utilisée.

  • Pour convertir en long int, utilisez strtol() au lieu de atol().
  • Pour convertir en double, utilisez strtod() au lieu de atof().
  • Pour convertir en long long int, utilisez strtoll() au lieu de atoll().

Écrire des données formatées dans une chaîne

int sprintf ( char * str, const char * format, ... );

utilisez la fonction sprintf pour écrire des données flottantes dans la chaîne.

#include <stdio.h>
int main ()
{
  char buffer [50];
  double PI = 3.1415926;
  sprintf (buffer, "PI = %.7f", PI);
  printf ("%s\n",buffer);

}

Lire les données formatées à partir de la chaîne

int sscanf ( const char * s, const char * format, ...);

utilisez la fonction sscanf pour analyser les données formatées.

#include <stdio.h>
int main ()
{
  char sentence []="date : 06-06-2012";
  char str [50];
  int year;
  int month;
  int day;
  sscanf (sentence,"%s : %2d-%2d-%4d", str, &day, &month, &year);
  printf ("%s -> %02d-%02d-%4d\n",str, day, month, year);
  return 0;
}

Copie et concaténation : strcpy(), strcat() #

#include <stdio.h>
#include <string.h>

int main(void)
{
  /* Always ensure that your string is large enough to contain the characters
   * and a terminating NUL character ('\0')!
   */
  char mystring[10];

  /* Copy "foo" into `mystring`, until a NUL character is encountered. */
  strcpy(mystring, "foo");
  printf("%s\n", mystring);

  /* At this point, we used 4 chars of `mystring`, the 3 characters of "foo",
   * and the NUL terminating byte.
   */

  /* Append "bar" to `mystring`. */
  strcat(mystring, "bar");
  printf("%s\n", mystring);

  /* We now use 7 characters of `mystring`: "foo" requires 3, "bar" requires 3
   * and there is a terminating NUL character ('\0') at the end.
   */

  /* Copy "bar" into `mystring`, overwriting the former contents. */
  strcpy(mystring, "bar");
  printf("%s\n", mystring);

  return 0;
}

Les sorties:

foo
foobar
bar

Si vous ajoutez ou copiez à partir d’une chaîne existante, assurez-vous qu’elle se termine par NUL !

Les littéraux de chaîne (par exemple, "foo") seront toujours terminés par NUL par le compilateur.

Comparaison : strcmp(), strncmp(), strcasecmp(), strncasecmp() #

Les fonctions strcase* ne sont pas du C standard, mais une extension POSIX.

La fonction strcmp compare lexicographiquement deux tableaux de caractères à terminaison nulle. Les fonctions renvoient une valeur négative si le premier argument apparaît avant le second dans l’ordre lexicographique, zéro si elles sont égales ou positives si le premier argument apparaît après le second dans l’ordre lexicographique.

#include <stdio.h>
#include <string.h>

void compare(char const *lhs, char const *rhs)
{
    int result = strcmp(lhs, rhs); // compute comparison once
    if (result < 0) {
        printf("%s comes before %s\n", lhs, rhs);
    } else if (result == 0) {
        printf("%s equals %s\n", lhs, rhs);
    } else { // last case: result > 0
        printf("%s comes after %s\n", lhs, rhs);
    }
}

int main(void)
{
    compare("BBB", "BBB");
    compare("BBB", "CCCCC");
    compare("BBB", "AAAAAA");
    return 0;
}

Les sorties:

BBB equals BBB
BBB comes before CCCCC
BBB comes after AAAAAA

Comme strcmp, la fonction strcasecmp compare également lexicographiquement ses arguments après avoir traduit chaque caractère en son correspondant en minuscule :

#include <stdio.h>
#include <string.h>

void compare(char const *lhs, char const *rhs)
{
    int result = strcasecmp(lhs, rhs); // compute case-insensitive comparison once
    if (result < 0) {
        printf("%s comes before %s\n", lhs, rhs);
    } else if (result == 0) {
        printf("%s equals %s\n", lhs, rhs);
    } else { // last case: result > 0
        printf("%s comes after %s\n", lhs, rhs);
    }
}

int main(void)
{
    compare("BBB", "bBB");
    compare("BBB", "ccCCC");
    compare("BBB", "aaaaaa");
    return 0;
}

Les sorties:

BBB equals bBB
BBB comes before ccCCC
BBB comes after aaaaaa

strncmp et strncasecmp comparent au plus n caractères :

#include <stdio.h>
#include <string.h>

void compare(char const *lhs, char const *rhs, int n)
{
    int result = strncmp(lhs, rhs, n); // compute comparison once
    if (result < 0) {
        printf("%s comes before %s\n", lhs, rhs);
    } else if (result == 0) {
        printf("%s equals %s\n", lhs, rhs);
    } else { // last case: result > 0
        printf("%s comes after %s\n", lhs, rhs);
    }
}

int main(void)
{
    compare("BBB", "Bb", 1);
    compare("BBB", "Bb", 2);
    compare("BBB", "Bb", 3);
    return 0;
}

Les sorties:

BBB equals Bb
BBB comes before Bb
BBB comes before Bb

strspn et strcspn #

Convertir en toute sécurité des chaînes en nombre : fonctions strtoX #

Depuis C99, la bibliothèque C dispose d’un ensemble de fonctions de conversion sûres qui interprètent une chaîne comme un nombre. Leurs noms sont de la forme strtoX, où X est l’un des l, ul, d, etc. pour déterminer le type cible de la conversion

double strtod(char const* p, char** endptr);
long double strtold(char const* p, char** endptr);

Ils permettent de vérifier qu’une conversion a eu un dépassement supérieur ou inférieur :

double ret = strtod(argv[1], 0); /* attempt conversion */

/* check the conversion result. */
if ((ret == HUGE_VAL || ret == -HUGE_VAL) && errno == ERANGE) 
    return;  /* numeric overflow in in string */
else if (ret == HUGE_VAL && errno == ERANGE) 
    return; /* numeric underflow in in string */

/* At this point we know that everything went fine so ret may be used */

Si la chaîne ne contient en fait aucun nombre, cette utilisation de strtod renvoie 0.0.

Si cela n’est pas satisfaisant, le paramètre supplémentaire “endptr” peut être utilisé. C’est un pointeur vers un pointeur qui sera pointé à la fin du nombre détecté dans la chaîne. S’il est défini sur ‘0’, comme ci-dessus, ou ‘NULL’, il est simplement ignoré.

Ce paramètre endptr indique s’il y a eu une conversion réussie et si oui, où le numéro s’est terminé :

char *check = 0;
double ret = strtod(argv[1], &check); /* attempt conversion */

/* check the conversion result. */
if (argv[1] == check) 
    return; /* No number was detected in string */
else if ((ret == HUGE_VAL || ret == -HUGE_VAL) && errno == ERANGE) 
    return; /* numeric overflow in in string */
else if (ret == HUGE_VAL && errno == ERANGE) 
    return; /* numeric underflow in in string */

/* At this point we know that everything went fine so ret may be used */

Il existe des fonctions analogues pour convertir vers les types entiers plus larges :

long strtol(char const* p, char** endptr, int nbase);
long long strtoll(char const* p, char** endptr, int nbase);
unsigned long strtoul(char const* p, char** endptr, int nbase);
unsigned long long strtoull(char const* p, char** endptr, int nbase);

Ces fonctions ont un troisième paramètre nbase qui contient le nombre base dans laquelle le nombre est écrit.

long a = strtol("101",   0, 2 ); /* a = 5L */
long b = strtol("101",   0, 8 ); /* b = 65L */
long c = strtol("101",   0, 10); /* c = 101L */
long d = strtol("101",   0, 16); /* d = 257L */
long e = strtol("101",   0, 0 ); /* e = 101L */
long f = strtol("0101",  0, 0 ); /* f = 65L */
long g = strtol("0x101", 0, 0 ); /* g = 257L */

La valeur spéciale 0 pour nbase signifie que la chaîne est interprétée de la même manière que les littéraux numériques sont interprétés dans un programme C : un préfixe de 0x correspond à une représentation hexadécimale, sinon un 0 de tête est octal et tous les autres nombres sont considérés comme décimaux.

Ainsi, la manière la plus pratique d’interpréter un argument de ligne de commande comme un nombre serait

int main(int argc, char* argv[] {
    if (argc < 1)
        return EXIT_FAILURE; /* No number given. */

    /* use strtoull because size_t may be wide */
    size_t mySize = strtoull(argv[1], 0, 0);

    /* then check conversion results. */

     ...

    return EXIT_SUCCESS;
}

Cela signifie que le programme peut être appelé avec un paramètre en octal, décimal ou hexadécimal.

strstr #

/* finds the next instance of needle in haystack 
   zbpos: the zero-based position to begin searching from
   haystack: the string to search in
   needle: the string that must be found
   returns the next match of `needle` in `haystack`, or -1 if not found
*/
int findnext(int zbpos, const char *haystack, const char *needle)
{
    char *p; 

    if (((p = strstr(haystack + zbpos, needle)) != NULL)
        return p - haystack;

    return -1;
}

strstr recherche dans le haystack (premier) argument la chaîne pointée par needle. S’il est trouvé, strstr renvoie l’adresse de l’occurrence. S’il ne trouve pas needle, il renvoie NULL. Nous utilisons zbpos pour ne pas retrouver la même aiguille encore et encore. Afin d’ignorer la première instance, nous ajoutons un décalage de zbpos. Un clone du Bloc-notes peut appeler findnext comme ceci, afin d’implémenter son dialogue “Find Next”:

/*
    Called when the user clicks "Find Next"
    doc: The text of the document to search
    findwhat: The string to find
*/
void onfindnext(const char *doc, const char *findwhat)
{
    static int i;

    if ((i = findnext(i, doc, findwhat)) != -1)
        /* select the text starting from i and ending at i + strlen(findwhat) */
    else
        /* display a message box saying "end of search" */
}
