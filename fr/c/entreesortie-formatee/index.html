Spécificateurs de conversion pour l’impression #

Spécificateur de conversion Type d’argumentation Descriptif
je, d entier imprime décimal
u entier non signé imprime décimal
o entier non signé imprime en octal
x entier non signé imprime en hexadécimal, en minuscules
‘X’ entier non signé imprime en hexadécimal, majuscule
f doubler les impressions flottent avec une précision par défaut de 6, si aucune précision n’est donnée (minuscules utilisées pour les nombres spéciaux nan et inf ou infinity)
F doubler les impressions flottent avec une précision par défaut de 6, si aucune précision n’est donnée (majuscules utilisées pour les nombres spéciaux NAN et INF ou INFINITY)
e doubler imprime flotter avec une précision par défaut de 6, si aucune précision n’est donnée, en utilisant la notation scientifique utilisant mantisse/exposant ; exposant minuscule et nombres spéciaux
E doubler imprime flotter avec une précision par défaut de 6, si aucune précision n’est donnée, en utilisant la notation scientifique utilisant mantisse/exposant ; exposant majuscule et nombres spéciaux
g doubler utilise f ou e [voir ci-dessous]
G doubler utilise F ou E [voir ci-dessous]
a doubler imprime en hexadécimal, en minuscules
A doubler imprime en hexadécimal, majuscule
c char imprime un seul caractère
s caractère* imprime une chaîne de caractères jusqu’à un terminateur NUL, ou tronqué à la longueur donnée par précision, si spécifié
p vide* imprime la valeur du pointeur void ; un pointeur non void doit être explicitement converti (“cast”) en void* ; pointeur vers un objet uniquement, pas un pointeur de fonction
% n/a imprime le caractère %
n entier * écrivez le nombre d’octets imprimés jusqu’ici dans le int pointé.

Notez que des modificateurs de longueur peuvent être appliqués à %n (par exemple, %hhn indique que a suivant le spécificateur de conversion n s’applique à un pointeur vers un signed char argument, selon ISO/IEC 9899:2011 §7.21.6.1 ¶7).

Notez que les conversions en virgule flottante s’appliquent aux types float et double en raison des règles de promotion par défaut — §6.5.2.2 Appels de fonction, ¶7 La notation des points de suspension dans un déclarateur de prototype de fonction provoque conversion du type d’argument pour s’arrêter après le dernier paramètre déclaré. L’argument par défaut les promotions sont effectuées sur les arguments de fin.) Ainsi, les fonctions telles que printf() ne reçoivent que des valeurs double, même si la variable référencée est de type float.

Avec les formats g et G, le choix entre la notation e et f (ou E et F) est documenté dans le standard C et dans la spécification POSIX pour printf() :

Le double argument représentant un nombre à virgule flottante doit être converti dans le style f ou e (ou dans le style F ou E dans le cas d’un spécificateur de conversion G), selon le valeur convertie et la précision. Soit P égal à la précision si non nulle, 6 si la précision est omise ou 1 si la précision est nulle. Alors, si une conversion avec le style ‘E’ aurait un exposant de ‘X’ :

  • Si P > X >= -4, la conversion doit être de style f (ou F) et de précision P - (X+1).
  • Sinon, la conversion se fera avec le style e (ou E) et la précision P - 1.

Enfin, à moins que le drapeau « # » ne soit utilisé, tous les zéros à droite doivent être supprimés de la partie fractionnaire du résultat et le caractère décimal doit être supprimé s’il ne reste aucune partie fractionnaire.

Impression de la valeur d’un pointeur vers un objet #

Pour imprimer la valeur d’un pointeur vers un objet (par opposition à un pointeur de fonction), utilisez le spécificateur de conversion p. Il est défini pour n’afficher que les pointeurs void, donc pour afficher la valeur d’un pointeur non void, il doit être explicitement converti (“casted*”) en void*.

#include <stdlib.h> /* for EXIT_SUCCESS */
#include <stdio.h>  /* for printf() */

int main(void)
{
  int i;
  int * p = &i;

  printf("The address of i is %p.\n", (void*) p);

  return EXIT_SUCCESS;
}

Utilisation de <inttypes.h> et uintptr_t #

Une autre façon d’imprimer des pointeurs dans C99 ou une version ultérieure utilise le type uintptr_t et les macros de <inttypes.h> :

#include <inttypes.h> /* for uintptr_t and PRIXPTR */
#include <stdio.h>    /* for printf() */

int main(void)
{
  int  i;
  int *p = &i;

  printf("The address of i is 0x%" PRIXPTR ".\n", (uintptr_t)p);

  return 0;
}

En théorie, il se peut qu’il n’y ait pas de type entier pouvant contenir un pointeur converti en entier (donc le type uintptr_t peut ne pas exister). En pratique, ça existe. Les pointeurs vers des fonctions n’ont pas besoin d’être convertibles en type uintptr_t - bien qu’ils soient le plus souvent convertibles.

Si le type uintptr_t existe, le type intptr_t existe également. Cependant, il n’est pas clair pourquoi vous voudriez traiter les adresses comme des entiers signés.

Historique pré-standard : #

Avant C89 à l’époque de K&R-C, il n’y avait pas de type void* (ni d’en-tête <stdlib.h>, ni de prototypes, et donc pas de notation int main(void)), donc le pointeur était converti en long unsigned int et imprimé à l’aide du modificateur de longueur/spécificateur de conversion lx.

** L’exemple ci-dessous est juste à titre informatif. De nos jours, il s’agit d’un code invalide, qui pourrait très bien provoquer le tristement célèbre [Comportement indéfini][1].**

#include <stdio.h> /* optional in pre-standard C - for printf() */

int main()
{
  int  i;
  int *p = &i;

  printf("The address of i is 0x%lx.\n", (long unsigned) p);

  return 0;
}

[1] : https://www.wikiod.com/fr/c/comportement-indefini

Impression de la différence des valeurs de deux pointeurs vers un objet #

[La soustraction des valeurs de deux pointeurs][1] à un objet donne un entier signé *1. Il serait donc imprimé en utilisant * au moins * le spécificateur de conversion d.

Pour s’assurer qu’il existe un type suffisamment large pour contenir une telle “différence de pointeur”, puisque C99 <stddef.h> définit le type ptrdiff_t. Pour imprimer un ptrdiff_t, utilisez le modificateur de longueur t.

#include <stdlib.h> /* for EXIT_SUCCESS */
#include <stdio.h> /* for printf() */
#include <stddef.h> /* for ptrdiff_t */


int main(void)
{
  int a[2];
  int * p1 = &a[0], * p2 = &a[1];
  ptrdiff_t pd = p2 - p1;

  printf("p1 = %p\n", (void*) p1);
  printf("p2 = %p\n", (void*) p2);
  printf("p2 - p1 = %td\n", pd);

  return EXIT_SUCCESS;
}

Le résultat pourrait ressembler à ceci :

p1 = 0x7fff6679f430
p2 = 0x7fff6679f434
p2 - p1 = 1

Veuillez noter que la valeur résultante de la différence est mise à l’échelle par la taille du type vers lequel pointent les pointeurs soustraits, un “int” ici. La taille d’un int pour cet exemple est 4.

*1Si les deux pointeurs à soustraire ne pointent pas vers le même objet, le comportement est indéfini.

[1] : https://www.wikiod.com/fr/c/les-operateurs#Arithmétique du pointeur

La fonction printf() #

Accessible via notamment <stdio.h>, la fonction printf() est le principal outil utilisé pour imprimer du texte sur la console en C.

printf("Hello world!");
// Hello world!

Les tableaux de caractères normaux et non formatés peuvent être imprimés par eux-mêmes en les plaçant directement entre les parenthèses.

printf("%d is the answer to life, the universe, and everything.", 42);
// 42 is the answer to life, the universe, and everything.

int x = 3;
char y = 'Z';
char* z = "Example";
printf("Int: %d, Char: %c, String: %s", x, y, z);
// Int: 3, Char: Z, String: Example

Alternativement, les entiers, les nombres à virgule flottante, les caractères, etc. peuvent être imprimés en utilisant le caractère d’échappement %, suivi d’un caractère ou d’une séquence de caractères indiquant le format, connu sous le nom de * spécificateur de format *.

Tous les arguments supplémentaires de la fonction printf() sont séparés par des virgules, et ces arguments doivent être dans le même ordre que les spécificateurs de format. Les arguments supplémentaires sont ignorés, tandis que les arguments mal typés ou l’absence d’arguments entraîneront des erreurs ou un comportement indéfini. Chaque argument peut être une valeur littérale ou une variable.

Après une exécution réussie, le nombre de caractères imprimés est renvoyé avec le type int. Sinon, un échec renvoie une valeur négative.

Drapeaux de format d’impression #

La norme C (C11 et C99 également) définit les drapeaux suivants pour printf() :

Drapeau Conversions Sens
- tout Le résultat de la conversion doit être justifié à gauche dans le champ. La conversion est justifiée à droite si ce drapeau n’est pas spécifié.
+ numérique signé Le résultat d’une conversion signée doit toujours commencer par un signe ( ‘+’ ou ‘-’ ). La conversion ne doit commencer par un signe que lorsqu’une valeur négative est convertie si ce fanion n’est pas spécifié.
<espace> numérique signé Si le premier caractère d’une conversion signée n’est pas un signe ou si une conversion signée n’aboutit à aucun caractère, un <espace> doit être préfixé au résultat. Cela signifie que si les drapeaux <espace> et ‘+’ apparaissent tous les deux, le drapeau <espace> doit être ignoré.
# tout Spécifie que la valeur doit être convertie en une forme alternative. Pour la conversion o, il doit augmenter la précision, si et seulement si nécessaire, pour forcer le premier chiffre du résultat à être un zéro (si la valeur et la précision sont toutes les deux 0, un seul 0 est imprimé). Pour les spécificateurs de conversion x ou X, un résultat différent de zéro doit avoir 0x (ou 0X) comme préfixe. Pour les spécificateurs de conversion a, A, e, E, f, F, g et G, le résultat doit toujours contenir un caractère de base, même si aucun chiffre ne suit le caractère de base. Sans cet indicateur, un caractère de base apparaît dans le résultat de ces conversions uniquement si un chiffre le suit. Pour les spécificateurs de conversion g et G, les zéros à la fin ne doivent pas être supprimés du résultat comme ils le sont normalement. Pour les autres spécificateurs de conversion, le comportement n’est pas défini.
‘0’ numérique Pour les spécificateurs de conversion d, i, o, u, x, X, a, A, e, E, f, F, g et G, les zéros non significatifs (après toute indication de signe ou de base) sont utilisés pour remplir le champ largeur plutôt que d’effectuer un remplissage d’espace, sauf lors de la conversion d’un infini ou d’un NaN. Si les drapeaux ‘0’ et ‘-’ apparaissent tous les deux, le drapeau ‘0’ est ignoré. Pour les spécificateurs de conversion d, i, o, u, x et X, si une précision est spécifiée, le drapeau ‘0’ doit être ignoré. ⌦ Si les drapeaux ‘0’ et <apostrophe> apparaissent tous les deux, les caractères de regroupement sont insérés avant le remplissage par zéro. Pour les autres conversions, le comportement est indéfini. ⌫

Ces indicateurs sont également pris en charge par Microsoft avec les mêmes significations.

La spécification POSIX pour printf() ajoute :

Drapeau Conversions Sens
' je, d, u, f, F, g, G La partie entière du résultat d’une conversion décimale doit être formatée avec des milliers de caractères de groupement. Pour les autres conversions, le comportement est indéfini. Le caractère de regroupement non monétaire est utilisé.

Modificateurs de longueur #

Les normes C99 et C11 spécifient les modificateurs de longueur suivants pour printf() ; leurs significations sont :

| Modificateur | Modifie | S’applique à | |———-|———————|———– ———————————————–| | hh | d, je, o, u, x ou X | car, car signé ou car non signé | h | d, je, o, u, x ou X | short int ou unsigned short int | l | d, je, o, u, x ou X | long int ou unsigned long int | l | a, A, e, E, f, F, g ou G | double (pour compatibilité avec scanf() ; non défini dans C90) | ll | d, je, o, u, x ou X | long long int ou unsigned long long int | j | d, je, o, u, x ou X | intmax_t ou uintmax_t | z | d, je, o, u, x ou X | size_t ou le type signé correspondant (ssize_t dans POSIX) | t | d, je, o, u, x ou X | ptrdiff_t ou le type entier non signé correspondant | L | a, A, e, E, f, F, g ou G | “long double”

Si un modificateur de longueur apparaît avec un spécificateur de conversion autre que celui spécifié ci-dessus, le comportement est indéfini.

Microsoft spécifie certains modificateurs de longueur différents et ne prend explicitement pas en charge hh, j, z ou t .

| Modificateur | Modifie | S’applique à | |———-|———————|———– ———————————————–| | I32 | d, je, o, x ou X | __int32 | I32 | o, u, x ou X | non signé __int32 | I64 | d, je, o, x ou X | __int64 | I64 | o, u, x ou X | non signé __int64 | je | d, je, o, x ou X | ptrdiff_t (c’est-à-dire, __int32 sur les plates-formes 32 bits, __int64 sur les plates-formes 64 bits) | je | o, u, x ou X | size_t (c’est-à-dire, unsigned __int32 sur les plates-formes 32 bits, unsigned __int64 sur les plates-formes 64 bits) | l ou L | a, A, e, E, f, g ou G | long double (Dans Visual C++, bien que long double soit un type distinct, il a la même représentation interne que double.) | l ou w | c ou C | Caractère large avec les fonctions printf et wprintf. (Un spécificateur de type lc, lC, wc ou wC est synonyme de C dans les fonctions printf et de c dans les fonctions wprintf.) | l ou w | s, S ou Z | Chaîne de caractères larges avec les fonctions printf et wprintf. (Un spécificateur de type ls, lS, ws ou wS est synonyme de S dans les fonctions printf et de s dans les fonctions wprintf.)

Notez que les spécificateurs de conversion C, S et Z et les modificateurs de longueur I, I32, I64 et w sont des extensions Microsoft. Traiter l comme un modificateur pour long double plutôt que double est différent de la norme, bien que vous ayez du mal à repérer la différence à moins que long double ait une représentation différente de double.
