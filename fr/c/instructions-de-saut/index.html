Syntaxe #

  • valeur de retour ; /* Renvoie à partir de la fonction actuelle. val peut être une valeur de n’importe quel type qui est convertie en type de retour de la fonction. */
  • revenir; /* Renvoie à partir de la fonction void actuelle. */
  • Pause; /* Saute inconditionnellement au-delà de la fin (“sort”) d’une instruction d’itération (boucle) ou hors de l’instruction switch la plus interne. */
  • Continuez; /* Saute inconditionnellement au début d’une déclaration d’itération (boucle). */
  • aller à LBL ; /* Saute à l’étiquette LBL. */
  • LBL : instruction /* toute instruction dans la même fonction. */

Ce sont les sauts qui sont intégrés en C au moyen de mots clés.

C a également une autre construction de saut, long jump, qui est spécifiée avec un type de données, jmp_buf, et des appels de bibliothèque C, setjmp et longjmp.

Utilisation du retour #

Renvoyer une valeur #

Un cas couramment utilisé : revenir de main()

#include <stdlib.h> /* for EXIT_xxx macros */

int main(int argc, char ** argv)
{
  if (2 < argc)
  {
    return EXIT_FAILURE; /* The code expects one argument: 
                            leave immediately skipping the rest of the function's code */
  }

  /* Do stuff. */

  return EXIT_SUCCESS;
}

Notes complémentaires:

  1. Pour une fonction dont le type de retour est void (n’incluant pas void * ou les types associés), l’instruction return ne doit pas avoir d’expression associée ; c’est-à-dire que la seule instruction de retour autorisée serait return;.

  2. Pour une fonction ayant un type de retour non “void”, l’instruction “return” ne doit pas apparaître sans expression.

  3. Pour main() (et uniquement pour main()), une instruction explicit return n’est pas requise (dans C99 ou version ultérieure). Si l’exécution atteint le } de fin, une valeur implicite de 0 est renvoyée. Certaines personnes pensent que l’omission de ce “retour” est une mauvaise pratique ; d’autres suggèrent activement de le laisser de côté.

Ne retournant rien #

Retour d’une fonction void

void log(const char * message_to_log)
{
  if (NULL == message_to_log)
  {
    return; /* Nothing to log, go home NOW, skip the logging. */
  }

  fprintf(stderr, "%s:%d %s\n", __FILE__, _LINE__, message_to_log);

  return; /* Optional, as this function does not return a value. */
}

Utilisation de goto pour sortir des boucles imbriquées #

Sauter hors des boucles imbriquées nécessiterait généralement l’utilisation d’une variable booléenne avec une vérification de cette variable dans les boucles. Supposons que nous itérons sur i et j, cela pourrait ressembler à ceci

size_t i,j;
for (i = 0; i < myValue && !breakout_condition; ++i) {
    for (j = 0; j < mySecondValue && !breakout_condition; ++j) {
        ... /* Do something, maybe modifying breakout_condition */
            /* When breakout_condition == true the loops end */
     }
}

Mais le langage C propose la clause goto, qui peut être utile dans ce cas. En l’utilisant avec une étiquette déclarée après les boucles, nous pouvons facilement sortir des boucles.

size_t i,j;
for (i = 0; i < myValue; ++i) {
    for (j = 0; j < mySecondValue; ++j) {
        ...
        if(breakout_condition) 
          goto final;
    }
}
final:

Cependant, souvent, lorsque ce besoin se présente, un «retour» pourrait être mieux utilisé à la place. Cette construction est également considérée comme “non structurée” dans la théorie de la programmation structurelle.

Une autre situation où goto pourrait être utile est pour sauter à un gestionnaire d’erreur :

ptr = malloc(N *  x);
if(!ptr)
  goto out_of_memory;

/* normal processing */
free(ptr);
return SUCCESS;

out_of_memory:
 free(ptr); /* harmless, and necessary if we have further errors */
 return FAILURE;

L’utilisation de goto sépare le flux d’erreur du flux de contrôle normal du programme. Il est cependant également considéré comme “non structuré” au sens technique.

Utiliser pause et continuer #

“Continuer” immédiatement la lecture en cas d’entrée invalide ou “interrompre” à la demande de l’utilisateur ou à la fin du fichier :

#include <stdlib.h> /* for EXIT_xxx macros */
#include <stdio.h>  /* for printf() and getchar() */
#include <ctype.h> /* for isdigit() */

void flush_input_stream(FILE * fp);


int main(void)
{
  int sum = 0;
  printf("Enter digits to be summed up or 0 to exit:\n");

  do
  {
    int c = getchar();
    if (EOF == c)
    {
      printf("Read 'end-of-file', exiting!\n");

      break;
    }

    if ('\n' != c)
    {
      flush_input_stream(stdin);
    }

    if (!isdigit(c))
    {
      printf("%c is not a digit! Start over!\n", c);

      continue;
    }

    if ('0' == c)
    {
      printf("Exit requested.\n");

      break;
    }

    sum += c - '0';

    printf("The current sum is %d.\n", sum);
  } while (1);

  return EXIT_SUCCESS;
}

void flush_input_stream(FILE * fp)
{
  size_t i = 0;
  int c;
  while ((c = fgetc(fp)) != '\n' && c != EOF) /* Pull all until and including the next new-line. */
  {
    ++i;
  }

  if (0 != i)
  {
    fprintf(stderr, "Flushed %zu characters from input.\n", i);
  }
}
