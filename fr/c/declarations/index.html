La déclaration d’identifiant faisant référence à un objet ou à une fonction est souvent appelée simplement une déclaration d’objet ou de fonction.

Appel d’une fonction depuis un autre fichier C #

foo.h

#ifndef FOO_DOT_H    /* This is an "include guard" */
#define FOO_DOT_H    /* prevents the file from being included twice. */
                     /* Including a header file twice causes all kinds */
                     /* of interesting problems.*/

/**
 * This is a function declaration.
 * It tells the compiler that the function exists somewhere.
 */
void foo(int id, char *name);

#endif /* FOO_DOT_H */

foo.c

#include "foo.h"    /* Always include the header file that declares something
                     * in the C file that defines it. This makes sure that the
                     * declaration and definition are always in-sync.  Put this
                     * header first in foo.c to ensure the header is self-contained.
                     */
#include <stdio.h>
                       
/**
 * This is the function definition.
 * It is the actual body of the function which was declared elsewhere.
 */
void foo(int id, char *name)
{
    fprintf(stderr, "foo(%d, \"%s\");\n", id, name);
    /* This will print how foo was called to stderr - standard error.
     * e.g., foo(42, "Hi!") will print `foo(42, "Hi!")`
     */
}

principal c

#include "foo.h"

int main(void)
{
    foo(42, "bar");
    return 0;
}

** Compiler et lier **

Premièrement, nous compilons à la fois foo.c et main.c en fichiers objets. Ici, nous utilisons le compilateur gcc, votre compilateur peut avoir un nom différent et avoir besoin d’autres options.

$ gcc -Wall -c foo.c
$ gcc -Wall -c main.c

Maintenant, nous les lions ensemble pour produire notre exécutable final :

$ gcc -o testprogram foo.o main.o

Utilisation d’une variable globale #

Introduction #

Voici des exemples de déclarations :

int a; /* declaring single identifier of type int */

La déclaration ci-dessus déclare un identifiant unique nommé a qui fait référence à un objet de type int.

int a1, b1; /* declaring 2 identifiers of type int */

La deuxième déclaration déclare 2 identifiants nommés a1 et b1 qui font référence à d’autres objets mais avec le même type int.

Fondamentalement, la façon dont cela fonctionne est la suivante - vous mettez d’abord un [type] [1], puis vous écrivez une ou plusieurs expressions séparées par des virgules (,) (** qui ne seront pas évaluées à ce stade - et qui devraient autrement être appelés déclarants dans ce contexte**). En écrivant de telles expressions, vous êtes autorisé à appliquer uniquement les opérateurs d’indirection (*), d’appel de fonction (( )) ou d’indice (ou d’indexation de tableau - [ ]) sur un identifiant (vous ne pouvez pas non plus utiliser d’opérateur opérateurs du tout). L’identifiant utilisé n’a pas besoin d’être visible dans la portée courante. Quelques exemples:

/* 1 */ int /* 2 */ (*z) /* 3 */ , /* 4 */ *x , /* 5 */ **c /* 6 */ ;
# Descriptif
1 Le nom du type entier.
2 Expression non évaluée appliquant l’indirection à un identifiant z.
3 Nous avons une virgule indiquant qu’une autre expression suivra dans la même déclaration.
4 Expression non évaluée appliquant l’indirection à un autre identifiant x.
5 Expression non évaluée appliquant l’indirection à la valeur de l’expression (*c).
6 Fin de déclaration.

Notez qu’aucun des identifiants ci-dessus n’était visible avant cette déclaration et que les expressions utilisées ne seraient donc pas valides avant.

Après chacune de ces expressions, l’identificateur utilisé dans celle-ci est introduit dans la portée actuelle. (Si l’identifiant lui a attribué un lien, il peut également être redéclaré avec le même type de lien afin que les deux identifiants se réfèrent au même objet ou à la même fonction)

De plus, le signe de l’opérateur égal (=) peut être utilisé pour l’initialisation. Si une expression non évaluée (déclarateur) est suivie de = à l’intérieur de la déclaration, nous disons que l’identifiant introduit est également initialisé. Après le signe = nous pouvons mettre encore une fois une expression, mais cette fois elle sera évaluée et sa valeur sera utilisée comme initiale pour l’objet déclaré.

Exemples:

int l = 90; /* the same as: */

int l; l = 90; /* if it the declaration of l was in block scope */

int c = 2, b[c]; /* ok, equivalent to: */

int c = 2; int b[c];

Plus tard dans votre code, vous êtes autorisé à écrire exactement la même expression à partir de la partie déclaration de l’identifiant nouvellement introduit, vous donnant un objet du type spécifié au début de la déclaration, en supposant que vous avez attribué des valeurs valides à tous les accès objets sur le chemin. Exemples:

void f()
{
    int b2; /* you should be able to write later in your code b2 
            which will directly refer to the integer object
            that b2 identifies */
    
    b2 = 2; /* assign a value to b2 */
    
    printf("%d", b2); /*ok - should print 2*/

    int *b3; /* you should be able to write later in your code *b3 */

    b3 = &b2; /* assign valid pointer value to b3 */

    printf("%d", *b3); /* ok - should print 2 */

    int **b4; /* you should be able to write later in your code **b4 */

    b4 = &b3;

    printf("%d", **b4); /* ok - should print 2 */

    void (*p)(); /* you should be able to write later in your code (*p)() */

    p = &f; /* assign a valid pointer value */

    (*p)(); /* ok - calls function f by retrieving the
            pointer value inside p -    p
            and dereferencing it -      *p
            resulting in a function
            which is then called -      (*p)() -

            it is not *p() because else first the () operator is 
            applied to p and then the resulting void object is
            dereferenced which is not what we want here */
}

La déclaration de b3 spécifie que vous pouvez potentiellement utiliser la valeur b3 comme moyen d’accéder à un objet entier.

Bien sûr, pour appliquer l’indirection (’*’) à ‘b3’, vous devez également y stocker une valeur appropriée (voir [pointeurs][2] pour plus d’informations). Vous devez également d’abord stocker une certaine valeur dans un objet avant d’essayer de le récupérer (vous pouvez en savoir plus sur ce problème [ici][3]). Nous avons fait tout cela dans les exemples ci-dessus.

int a3(); /* you should be able to call a3 */

Celui-ci indique au compilateur que vous allez essayer d’appeler a3. Dans ce cas, “a3” fait référence à une fonction au lieu d’un objet. Une différence entre objet et fonction est que les fonctions auront toujours une sorte de lien. Exemples:

void f1()
{
    {
        int f2(); /* 1 refers to some function f2 */
    }
    
    {
        int f2(); /* refers to the exact same function f2 as (1) */
    }
}

Dans l’exemple ci-dessus, les 2 déclarations font référence à la même fonction f2, alors que si elles déclaraient des objets alors dans ce contexte (ayant 2 portées de bloc différentes), elles auraient été 2 objets distincts différents.

int (*a3)(); /* you should be able to apply indirection to `a3` and then call it */

Maintenant, cela peut sembler compliqué, mais si vous connaissez la priorité des opérateurs, vous n’aurez aucun problème à lire la déclaration ci-dessus. Les parenthèses sont nécessaires car l’opérateur * a moins de priorité que celui ( ).

Dans le cas de l’utilisation de l’opérateur d’indice, l’expression résultante ne serait pas réellement valide après la déclaration car l’index utilisé (la valeur à l’intérieur de [ et ]) sera toujours 1 au-dessus de la valeur maximale autorisée pour cette objet/fonction.

int a4[5]; /* here a4 shouldn't be accessed using the index 5 later on */

Mais il devrait être accessible par tous les autres index inférieurs à 5. Exemples :

a4[0], a4[1]; a4[4];

a4[5] donnera UB. Plus d’informations sur les tableaux peuvent être trouvées [ici][4].

int (*a5)[5](); /* here a4 could be applied indirection
                indexed up to (but not including) 5
                and called */

Malheureusement pour nous, bien que syntaxiquement possible, la déclaration de a5 est interdite par la norme actuelle.

[1] : https://www.wikiod.com/fr/c/types-de-donnees [2] : https://www.wikiod.com/fr/c/pointeurs [3] : https://www.wikiod.com/fr/c [4] : https://www.wikiod.com/fr/c/tableaux

Utilisation de constantes globales #

Typedef #

Les typedefs sont des déclarations qui ont le mot-clé typedef devant et avant le type. Par exemple.:

typedef int (*(*t0)())[5];

(vous pouvez techniquement mettre le typedef après le type aussi - comme ceci int typedef (*(*t0)())[5]; mais cela est déconseillé)

Les déclarations ci-dessus déclarent un identifiant pour un nom de typedef. Vous pouvez l’utiliser comme ceci après :

t0 pf;

Ce qui aura le même effet que d’écrire :

int (*(*pf)())[5];

Comme vous pouvez le voir, le nom du typedef “enregistre” la déclaration en tant que type à utiliser ultérieurement pour d’autres déclarations. De cette façon, vous pouvez enregistrer certaines frappes. De plus, comme la déclaration utilisant typedef est toujours une déclaration, vous n’êtes pas limité uniquement par l’exemple ci-dessus :

t0 (*pf1);

Est le même que:

int (*(**pf1)())[5];

Utilisation de la règle droite-gauche ou spirale pour déchiffrer la déclaration C #

La règle “droite-gauche” est une règle tout à fait régulière pour déchiffrer les déclarations C. Cela peut également être utile pour les créer.

Lisez les symboles tels que vous les rencontrez dans la déclaration…

*   as "pointer to"          - always on the left side
[]  as "array of"            - always on the right side
()  as "function returning"  - always on the right side

Comment appliquer la règle

ÉTAPE 1

Trouvez l’identifiant. C’est votre point de départ. Alors dites-vous, “l’identifiant est”. Vous avez commencé votre déclaration.

ÉTAPE 2

Regardez les symboles à droite de l’identifiant. Si, par exemple, vous y trouvez (), alors vous savez qu’il s’agit de la déclaration d’une fonction. Vous auriez alors “l’identifiant est la fonction qui retourne”. Ou si vous y trouviez un [], vous diriez “l’identifiant est un tableau de”. Continuez à droite jusqu’à ce que vous manquiez de symboles OU appuyez sur une parenthèse droite ). (Si vous appuyez sur une parenthèse gauche (, c’est le début d’un symbole (), même s’il y a des choses entre les parenthèses. Plus d’informations ci-dessous.)

ÉTAPE 3

Regardez les symboles à gauche de l’identifiant. Si ce n’est pas l’un de nos symboles ci-dessus (par exemple, quelque chose comme “int”), dites-le simplement. Sinon, traduisez-le en anglais en utilisant le tableau ci-dessus. Continuez à gauche jusqu’à ce que vous manquiez de symboles OU appuyez sur une parenthèse gauche (.

Répétez maintenant les étapes 2 et 3 jusqu’à ce que vous ayez formé votre déclaration.

Voici quelques exemples:

int *p[];

Tout d’abord, trouvez l’identifiant :

int *p[];
     ^

“p est”

Maintenant, déplacez-vous vers la droite jusqu’à ce qu’il n’y ait plus de symboles ou que la parenthèse droite soit atteinte.

int *p[];
      ^^

“p est un tableau de”

Vous ne pouvez plus vous déplacer vers la droite (hors symboles), alors déplacez-vous vers la gauche et trouvez :

int *p[];
    ^

“p est un tableau de pointeur vers”

Continuez à gauche et trouvez :

int *p[];
^^^

“p est un tableau de pointeur vers int”.

(ou “p est un tableau où chaque élément est de type pointeur vers int”)

Un autre exemple:

int *(*func())();

Trouvez l’identifiant.

int *(*func())();
       ^^^^

“fonction est”

Déplacer vers la droite.

int *(*func())();
           ^^

“func est une fonction qui retourne”

Je ne peux plus me déplacer vers la droite à cause de la parenthèse droite, alors déplacez-vous vers la gauche.

int *(*func())();
      ^

“func est une fonction renvoyant le pointeur vers”

Je ne peux plus aller à gauche à cause de la parenthèse gauche, alors continuez à droite.

int *(*func())();
              ^^

“func est une fonction renvoyant un pointeur vers une fonction renvoyant”

Je ne peux plus aller à droite car nous n’avons plus de symboles, alors allez à gauche.

int *(*func())();
    ^

“func est une fonction renvoyant un pointeur vers une fonction renvoyant un pointeur vers”

Et enfin, continuez à gauche, car il n’y a plus rien à droite.

int *(*func())();
^^^

“func est une fonction renvoyant un pointeur vers une fonction renvoyant un pointeur vers int”.

Comme vous pouvez le voir, cette règle peut être très utile. Vous pouvez également l’utiliser pour vérifier votre santé mentale pendant que vous créez des déclarations et pour vous indiquer où placer le symbole suivant et si des parenthèses sont nécessaires.

Certaines déclarations semblent beaucoup plus compliquées qu’elles ne le sont en raison de la taille des tableaux et des listes d’arguments sous forme de prototype. Si vous voyez [3], cela se lit comme “tableau (taille 3) de…”. Si vous voyez (char *, int) qui est lu comme * “la fonction attend (char *, int) et renvoie …” *.

En voici une amusante :

int (*(*fun_one)(char *,double))[9][20];

Je ne passerai pas par chacune des étapes pour déchiffrer celle-ci.

*“fun_one est un pointeur vers la fonction attendant (char ,double) et retournant le pointeur vers le tableau (taille 9) du tableau (taille 20) de int.”

Comme vous pouvez le voir, ce n’est pas aussi compliqué si vous vous débarrassez des tailles de tableau et listes d’arguments :

int (*(*fun_one)())[][];

Vous pouvez le déchiffrer de cette façon, puis mettre les tailles de tableau et les listes d’arguments plus tard.

Quelques mots de conclusion:

Il est tout à fait possible de faire des déclarations illégales en utilisant cette règle, donc une certaine connaissance de ce qui est légal en C est nécessaire. Par exemple, si ce qui précède avait été :

int *((*fun_one)())[][];

il aurait lu “fun_one est un pointeur vers une fonction renvoyant un tableau de tableau de pointeur vers int”. Puisqu’une fonction ne peut pas retourner un tableau, mais seulement un pointeur vers un tableau, cette déclaration est illégale.

Les combinaisons illégales incluent :

[]() - cannot have an array of functions
()() - cannot have a function that returns a function
()[] - cannot have a function that returns an array

Dans tous les cas ci-dessus, vous auriez besoin d’un ensemble de parenthèses pour lier un symbole * à gauche entre ces symboles de droite () et [] afin que la déclaration soit légale.

Voici quelques exemples supplémentaires :

Juridique

int i;               an int
int *p;              an int pointer (ptr to an int)
int a[];             an array of ints
int f();             a function returning an int
int **pp;            a pointer to an int pointer (ptr to a ptr to an int)
int (*pa)[];         a pointer to an array of ints
int (*pf)();         a pointer to a function returning an int
int *ap[];           an array of int pointers (array of ptrs to ints)
int aa[][];          an array of arrays of ints
int *fp();           a function returning an int pointer
int ***ppp;          a pointer to a pointer to an int pointer
int (**ppa)[];       a pointer to a pointer to an array of ints
int (**ppf)();       a pointer to a pointer to a function returning an int
int *(*pap)[];       a pointer to an array of int pointers
int (*paa)[][];      a pointer to an array of arrays of ints
int *(*pfp)();       a pointer to a function returning an int pointer
int **app[];         an array of pointers to int pointers
int (*apa[])[];      an array of pointers to arrays of ints
int (*apf[])();      an array of pointers to functions returning an int
int *aap[][];        an array of arrays of int pointers
int aaa[][][];       an array of arrays of arrays of int
int **fpp();         a function returning a pointer to an int pointer
int (*fpa())[];      a function returning a pointer to an array of ints
int (*fpf())();      a function returning a pointer to a function returning an int

Illégal

int af[]();          an array of functions returning an int
int fa()[];          a function returning an array of ints
int ff()();          a function returning a function returning an int
int (*pfa)()[];      a pointer to a function returning an array of ints
int aaf[][]();       an array of arrays of functions returning an int
int (*paf)[]();      a pointer to a an array of functions returning an int
int (*pff)()();      a pointer to a function returning a function returning an int
int *afp[]();        an array of functions returning int pointers
int afa[]()[];       an array of functions returning an array of ints
int aff[]()();       an array of functions returning functions returning an int
int *fap()[];        a function returning an array of int pointers
int faa()[][];       a function returning an array of arrays of ints
int faf()[]();       a function returning an array of functions returning an int
int *ffp()();        a function returning a function returning an int pointer
