Tutoriel itext

Si vous regardez la création de PDF, vous trouverez deux approches différentes :

  • Les graphistes utilisent des applications de bureau telles qu’Adobe Acrobat ou Adobe InDesign pour créer un document dans un processus manuel ou semi-manuel.
  • Dans un autre contexte, les documents PDF sont créés par programmation, en utilisant une API pour produire des PDF directement à partir d’applications logicielles, sans - ou avec une intervention humaine minimale. Parfois, le document est d’abord créé dans un format intermédiaire (par exemple XML, HTML,…), puis converti en PDF.

Ces différentes approches exigent des produits logiciels différents.

Il en va de même pour la manipulation de PDF.

  • Vous pouvez mettre à jour un PDF manuellement dans des outils tels qu’Adobe Acrobat,
  • Il existe également des outils qui permettent de remplir automatiquement des formulaires à partir des informations d’une base de données.

iText est un outil qui se concentre sur le côté automatisation des choses.

Qu’est-ce qu’iText ? #

iText est un SDK qui a été développé pour permettre aux développeurs de faire ce qui suit (et bien plus encore) :

  • Générer des documents et des rapports basés sur les données d’un fichier XML ou d’une base de données
  • Créez des cartes et des livres en exploitant de nombreuses fonctionnalités interactives disponibles en PDF
  • Ajoutez des signets, des numéros de page, des filigranes et d’autres fonctionnalités aux documents PDF existants
  • Diviser ou concaténer des pages à partir de fichiers PDF existants
  • Remplir des formulaires interactifs
  • Signer numériquement des documents PDF
  • Servir des documents PDF générés ou manipulés dynamiquement vers un navigateur Web

iText n’est pas un outil d’utilisateur final. Vous devez intégrer iText dans vos propres applications afin de pouvoir automatiser le processus de création et de manipulation de PDF.

Quand utiliser iText ? #

En règle générale, iText est utilisé dans les projets qui répondent à l’une des exigences suivantes :

  • Le contenu n’est pas disponible à l’avance : il est calculé en fonction des entrées de l’utilisateur ou des informations de la base de données en temps réel.
  • Les fichiers PDF ne peuvent pas être produits manuellement en raison de l’énorme volume de contenu : un grand nombre de pages ou de documents.
  • Les documents doivent être créés en mode sans assistance, dans un processus par lots.
  • Le contenu doit être personnalisé ou personnalisé ; par exemple, le nom de l’utilisateur final doit être tamponné sur un certain nombre de pages.

Vous rencontrerez souvent ces exigences dans les applications Web, où le contenu doit être servi dynamiquement à un navigateur. Normalement, vous fourniriez ces informations sous forme de HTML, mais pour certains documents, le PDF est préféré au HTML pour une meilleure qualité d’impression, pour une présentation identique sur une variété de plates-formes, pour des raisons de sécurité, pour se conformer à des normes spécifiques de l’industrie (telles que comme PAdES, PDF/A ou PDF/UA), ou pour réduire la taille du fichier.
