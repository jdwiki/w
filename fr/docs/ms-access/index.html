Tutoriel ms-access

  • Excuses pour le mur de texte. Les futures modifications ajouteront probablement des captures d’écran et d’autres éléments visuels.*

Les requêtes dans Microsoft Access peuvent être créées par l’une des quatre méthodes

  1. Utilisation d’un générateur d’assistant de requête étape par étape via l’interface graphique qui vous posera une série de questions sur les données que vous souhaitez afficher et sur la manière dont les différents éléments de données sont liés les uns aux autres.
  2. En utilisant l’interface graphique en mode Création, dans laquelle vous sélectionnez des tables et des champs spécifiques avec votre souris. L’ordre visuel des différents champs peut être spécifié en faisant glisser les colonnes pertinentes dans le panneau inférieur et des propriétés supplémentaires pour chaque champ peuvent être spécifiées dans le panneau Propriétés (à droite).
  3. En passant de la vue de conception à la vue SQL et en spécifiant le code de requête SQL brut. Dans la plupart des cas, vous pouvez librement basculer entre le mode Conception et le mode SQL - Access incorporera les modifications que vous avez apportées d’un mode à l’autre. La syntaxe d’Access SQL est similaire, mais pas identique, à celle utilisée dans MySQL, PostgreSQL, Oracle ou MS SQL Server (tSQL).
  4. Utilisation de la programmation Visual Basic pour Applications accessible via le groupe Macro du ruban Outils de base de données (Access 2007+). La manipulation de la base de données se produit via les bibliothèques ADO ou DAO et utilise la même syntaxe que SQL View dans l’application principale, à l’exception que divers caractères spéciaux doivent être “échappés”. Les requêtes créées via cette méthode ne sont pas accessibles directement depuis le volet de navigation mais doivent être placées dans une fonction ou une sous-procédure et soit déclenchées par d’autres éléments (par exemple par un bouton dans un formulaire) via des macros, soit directement exécutées dans l’interface graphique VBA.

La modification d’une valeur d’enregistrement à partir d’une requête en mode feuille de données entraînera une modification de la valeur d’enregistrement sous-jacente, en supposant que le champ de requête n’est pas une agrégation ou une concaténation de plusieurs sources d’informations.

Les formulaires et les rapports peuvent être utilisés pour afficher des informations à partir de requêtes dans un formulaire alternatif à une simple vue « Feuille de données » qui ressemble à une feuille de calcul de style Excel. Les formulaires sont destinés à l’affichage à l’écran, tandis que les rapports sont destinés à ceux imprimés sur papier.
