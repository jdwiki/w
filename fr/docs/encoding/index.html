Tutoriel d'encodage

Qu’est-ce qu’un encodage et comment ça marche ?

Un ordinateur ne peut pas stocker de lettres ou quoi que ce soit d’autre - il stocke des bits. Le bit peut être 0 ou 1 (“oui”/“non”, “vrai”/“faux” - ces formats sont donc appelés binaires). Pour utiliser ces bits, certaines règles sont nécessaires, pour convertir les bits en un certain contenu. Ces règles sont appelées * encodages *, où des séquences de bits 1/0 représentent certains caractères. Une séquence de 8 bits est appelée byte.

Les encodages fonctionnent comme des tableaux, où chaque caractère est lié à un octet spécifique. Pour coder quelque chose en codage ASCII, il faut suivre les entrées de droite à gauche, en recherchant les bits liés aux caractères. Pour décoder une chaîne de bits en caractères, on substitue des bits aux lettres de gauche à droite.

Les octets peuvent être représentés dans différents formats : par exemple, 10011111 en binaire est 237 en octal, 159 en décimal et 9F en hexadécimal.

Quelle est la différence entre les différents encodages ?

Le premier codage de caractères comme ASCII de l’ère pré-8 bits n’utilisait que 7 bits sur 8. ASCII a été utilisé pour coder la langue anglaise avec les 26 lettres en minuscules et majuscules, des chiffres et de nombreux signes de ponctuation. L’ASCII ne pouvait pas couvrir les autres langues européennes avec toutes les lettres ö-ß-é-å - des encodages ont donc été développés qui utilisaient le 8ème bit d’un octet pour couvrir 128 autres caractères.

Mais un octet ne suffit pas pour représenter des langues de plus de 256 caractères - par exemple le chinois. L’utilisation de deux octets (16 bits) permet d’encoder 65 536 valeurs distinctes. Des codages tels que BIG-5 séparent une chaîne de bits en blocs de 16 bits (2 octets) pour coder les caractères. Les encodages multi-octets ont l’avantage d’être économes en espace, mais l’inconvénient que les opérations telles que la recherche de sous-chaînes, les comparaisons, etc. doivent toutes décoder les caractères en points de code Unicode avant que de telles opérations puissent être effectuées (il existe cependant des raccourcis ).

Un autre type d’encodage est celui avec un nombre variable d’octets par caractère - comme les normes UTF. Ces normes ont une certaine taille d’unité, qui pour [UTF-8][1] est de 8 bits, pour UTF-16 de 16 bits et pour UTF-32 de 32 bits. Et puis la norme définit certains des bits comme des drapeaux : s’ils sont définis, alors l’unité suivante dans une séquence d’unités doit être considérée comme faisant partie du même caractère. S’ils ne sont pas définis, cette unité ne représente qu’un seul caractère (par exemple, l’anglais n’occupe qu’un seul octet, et c’est pourquoi le codage ASCII correspond entièrement à UTF-8).

Qu’est-ce que l’Unicode ?

[Unicode] [2] s’il s’agit d’un énorme jeu de caractères (disant d’une manière plus compréhensible - un tableau) avec 1 114 112 points de code, chacun d’eux représente une lettre, un symbole ou un autre caractère spécifique. En utilisant Unicode, vous pouvez écrire un document qui contient théoriquement n’importe quelle langue utilisée par les gens.

Unicode n’est pas un encodage - c’est un ensemble de points de code. Et il existe plusieurs façons d’encoder les points de code Unicode en bits - comme UTF-8, -16 et -32.
