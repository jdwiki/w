Qu’est-ce que wxPython #

En termes simples, wxPython est un ensemble de liaisons à la bibliothèque [wxWidgets][1] C++ Cross Platform GUI.

Ok qu’est-ce que wxWidgets #

La bibliothèque wxWidgets fournit un ensemble gratuit, gratuit et open source d’abstractions pour les différents éléments de l’interface graphique afin que les contrôles natifs soient toujours utilisés, le cas échéant, en conservant l’apparence, la convivialité et la vitesse natives. En tant que tel, il fournit une abstraction pour la création d’interface graphique et un certain nombre d’autres utilitaires dans une plate-forme qui permet aux développeurs de créer des applications pour Windows, Mac OS X, Linux et d’autres plates-formes à l’aide d’une seule base de code. wxWidgets a été lancé en 1992 et vous pouvez voir un historique détaillé [ici][2]. La bibliothèque wxWidgets est distribuée sous la licence wxWindows, qui est basée sur la L-GPL mais avec une clause d’exception. La clause d’exception vous permet de lier votre application dynamiquement ou statiquement à wxWidgets sans l’obligation de distribuer la source pour votre propre application. En d’autres termes, vous pouvez utiliser wxWidgets pour des projets gratuits ou commerciaux, sans frais*. La licence vous encourage à rendre les améliorations que vous apportez à la bibliothèque wxWidgets elle-même.*

Les points forts, notez que wxWidgets comprend des centaines de classes pour le développement d’applications multiplateformes :

  • Disposition des fenêtres à l’aide de calibreurs
  • Contextes de périphérique (avec stylos, pinceaux et polices)
  • Système complet de gestion des événements
  • Visionneuse d’aide HTML
  • Lecture audio et vidéo
  • Prise en charge d’Unicode et de l’internationalisation
  • Architecture de document/vue
  • Architecture d’impression
  • Prises
  • Multithreading
  • Manipulation de fichiers et de répertoires
  • Aide en ligne et contextuelle
  • Rendu HTML
  • Conteneurs de base
  • Chargement, enregistrement, dessin et manipulation d’images
  • Bibliothèque date-heure et minuteries
  • La gestion des erreurs
  • Presse-papiers et glisser-déposer

Notez que certaines de ces installations, *par ex. threading, * ne sont pas réellement liés à l’interface graphique, mais fournissent une abstraction multiplateforme utile de sorte que, dans le cas du threading par exemple, un ensemble de code d’application fonctionnera sur n’importe quelle plate-forme prise en charge.

Pendant de nombreuses années, la bibliothèque wxWidgets a produit 4 versions distinctes, * en plus des versions de débogage * à partir d’un ensemble de code source, de bibliothèques statiques et dynamiques conçues à la fois pour ASCII et Unicode. Il est généralement disponible pré-construit dans les variantes les plus courantes et sous forme de code source à construire avec les différentes options pour l’environnement cible et avec la chaîne d’outils C++ des développeurs avec de nombreuses chaînes d’outils prises en charge.

Les liaisons python pour cette bibliothèque et certains ajouts forment wxPython.

Retour à Qu’est-ce que wxPython, (que me donne-t-il) ? #

wxPython donne à un développeur un moyen de bénéficier d’une bibliothèque d’interface graphique multiplateforme, avec une licence claire, tout en offrant les avantages de Python. Comme wxWidgets et Python, wxPython est gratuit, gratuit et open source, et disponible pour être utilisé et distribué dans des projets gratuits et commerciaux * sans qu’il en résulte l’obligation de distribuer votre code source *.

  • Suite graphique complète comprenant, (mais sans s’y limiter) :
  • Windows (y compris Windows MDI)
  • Assistants
  • Cadres et mini-cadres
  • Dialogues, standard, avancés et personnalisés
  • Livres, arbres, grilles et commandes d’affichage des données
  • Jauges, Sliders, Spinners, Animations, Presse-papiers, Drag & Drop
  • Prise en charge de la visionneuse HTML, PDF et image
  • Les composants de l’interface graphique peuvent être positionnés de manière absolue, mais il est fortement recommandé d’utiliser une disposition basée sur le dimensionneur qui prend en charge le dimensionnement automatique, etc.
  • Multiplateforme - Prend en charge les interfaces graphiques pour Windows, OS-X et Linux avec une seule base de code * sans instructions conditionnelles dans votre code *
  • Vitesse native, look & feel.
  • Prototypage rapide, test et débogage - * rappelez-vous que c’est python *
  • Exécutez et modifiez des échantillons d’à peu près tout dans le package de démonstration.
  • Licence claire pour une utilisation gratuite même dans les produits commerciaux.
  • Si nécessaire, votre interface graphique python peut être refactorisée ultérieurement en une interface graphique C++ wxWidgets, car elle l’utilise déjà.
  • Grande communauté d’utilisateurs et de développeurs active et utile à la fois sur [StackOverflow] [14] et [listes de diffusion] [15].

Notez que là où python lui-même fournit un mécanisme multiplateforme pour implémenter les fonctions utilitaires de wxWidgets, threading étant à nouveau un bon exemple, il est intentionnellement omis de wxPython.

wxPython propose également une très grande suite de démonstrations pouvant être exécutées, testées et modifiées à partir du package Documents and Demo.

Saveurs de wxPython #

ASCII contre Unicode : #

Pendant de nombreuses années, * comme avec wxWidgets *, les développeurs ont dû choisir entre les versions ASCII et Unicode, ainsi qu’une version pour leur version spécifique de python ainsi que les options 32/64 bits. Depuis environ wxPython 2.8.9, la version ASCII uniquement de wxPython a été abandonnée, de sorte que le support Unicode est toujours disponible.

Classique contre Phoenix : #

Depuis wxPython 3.0.0, il existe la version publiée “Classic” de wxPython et une version Phoenix actuellement inédite. La version classique a tendance à être en retard sur les versions wxWidgets des mêmes numéros et le package de documentation est le C++ - il est disponible en téléchargement pour différentes plates-formes, (voir [Installation of Classic] [3]), dans le cas de Windows en tant que programme d’installation exécutable. Les liaisons Phoenix, étant en grande partie générées automatiquement, devraient suivre de plus près les versions de wxWidgets et inclure également une documentation spécifique à wxPython - elle peut être construite à partir de la source ou des versions nocturnes * car les roues * peuvent être obtenues à l’aide de **pip **, (voir [Installation de Phénix][4]).

Dans wxPython mais pas wxWidgets #

wxPython étend la bibliothèque wxWidgets avec un certain nombre de fonctionnalités, les suivantes ne sont que quelques-unes, qui ne sont pas disponibles dans wxWidgets :

  • Programmeurs Éditeurs & Shells : [crust][5], [crustslices][6], [AlaCart & AlaMode][7], [AlaModeTest][8]
  • [Interprète][9] & [magie][10]
  • Inspection - cela vous permet de lancer une fenêtre pour parcourir tous les composants de l’interface graphique de vos applications.
  • Un ensemble complet de démos
