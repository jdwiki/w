Construisez et installez Erlang/OTP sur FreeBSD #

Les exemples suivants montrent 3 méthodes principales pour installer Erlang/OTP sur FreeBSD.

Méthode 1 - Paquet binaire pré-construit #

Utilisez pkg pour installer le package binaire prédéfini :

$ pkg install erlang

Testez votre nouvelle installation :

$ erl
Erlang/OTP 18 [erts-7.3.1] [source] [64-bit] [smp:2:2] [async-threads:10] [hipe] [kernel-poll:false]

Eshell V7.3.1  (abort with ^G)

Méthode 2 - Construire et installer à l’aide de la collection de ports (recommandé) #

Construisez et installez le port comme d’habitude :

$ make -C /usr/ports/lang/erlang install clean

Testez votre nouvelle installation :

$ erl
Erlang/OTP 18 [erts-7.3.1] [source] [64-bit] [smp:2:2] [async-threads:10] [hipe] [kernel-poll:false]

Eshell V7.3.1  (abort with ^G)

Cela récupérera l’archive de la version sur le site officiel, appliquera quelques correctifs si nécessaire, compilera la version et l’installera. Évidemment, cela prendra du temps.

Méthode 3 - Construire et installer à partir de l’archive de la version #

Remarque : la construction manuelle de la version fonctionne, mais l’utilisation des deux méthodes ci-dessus doit être préférée, car la collection de ports intègre des correctifs qui rendent la version plus conviviale pour FreeBSD.

Téléchargez le fichier de version :

$ fetch 'http://erlang.org/download/otp_src_18.3.tar.gz'

Vérifiez que sa somme MD5 est correcte :

$ fetch 'http://erlang.org/download/MD5'
MD5                                           100% of   24 kB  266 kBps 00m00s

$ grep otp_src_18.3.tar.gz MD5
MD5(otp_src_18.3.tar.gz)= 7e4ff32f97c36fb3dab736f8d481830b

$ md5 otp_src_18.3.tar.gz
MD5 (otp_src_18.3.tar.gz) = 7e4ff32f97c36fb3dab736f8d481830b

Extrayez l’archive :

$ tar xzf otp_src_18.3.tar.gz

Configurez :

$ ./configure --disable-hipe

Si vous voulez construire Erlang avec HiPe, vous devrez appliquer les correctifs de la collection de ports.

Construire:

$ gmake

Installer:

$ gmake install

Testez votre nouvelle installation :

$ erl
Erlang/OTP 18 [erts-7.3] [source] [64-bit] [smp:2:2] [async-threads:10] [kernel-poll:false]

Eshell V7.3  (abort with ^G)

Construire et installer en utilisant kerl #

kerl est un outil qui vous aide à créer et à installer des versions d’Erlang/OTP.

Installez curl :

$ make -C /usr/ports/ftp/curl install clean

Téléchargement mec :

$ fetch 'https://raw.githubusercontent.com/kerl/kerl/master/kerl'
$ chmod +x kerl

Mettez à jour la liste des versions disponibles :

$ ./kerl update releases
The available releases are:
R10B-0 R10B-10 R10B-1a R10B-2 R10B-3 R10B-4 R10B-5 R10B-6 R10B-7 R10B-8 R10B-9 R11B-0 R11B-1 R11B-2 R11B-3 R11B-4 R11B-5 R12B-0 R12B-1 R12B-2 R12B-3 R12B-4 R12B-5 R13A R13B01 R13B02-1 R13B02 R13B03 R13B04 R13B R14A R14B01 R14B02 R14B03 R14B04 R14B R14B_erts-5.8.1.1 R15B01 R15B02 R15B02_with_MSVCR100_installer_fix R15B03-1 R15B03 R15B R16A_RELEASE_CANDIDATE R16B01 R16B02 R16B03-1 R16B03 R16B 17.0-rc1 17.0-rc2 17.0 17.1 17.3 17.4 17.5 18.0 18.1 18.2 18.2.1 18.3 19.0

Créez la version requise :

$ ./kerl build 18.3 erlang-18.3

Vérifiez que la compilation est présente dans la liste des compilations :

$ ./kerl list builds
18.3,erlang-18.3

Installez la version quelque part :

$ ./kerl install erlang-18.3 ./erlang-18.3

Sourcez le fichier activate si vous utilisez bash ou fish shell. Si vous exécutez un cshell, ajoutez le répertoire build bin au PATH :

$ setenv PATH "/some/where/erlang-18.3/bin/:$PATH"

Testez votre nouvelle installation :

$ which erl
/some/where/erlang-18.3/bin//erl

$ erl
Erlang/OTP 18 [erts-7.3] [source] [64-bit] [smp:2:2] [async-threads:10] [hipe] [kernel-poll:false]

Eshell V7.3  (abort with ^G)

Autres versions #

Si vous souhaitez créer une autre version d’Erlang/OTP, recherchez les autres ports dans la collection :

  • lang/erlang-runtime15
  • lang/erlang-runtime16
  • lang/erlang-runtime17
  • lang/erlang-runtime18

Construire et installer Erlang/OTP sur Ubuntu #

Les exemples suivants montrent deux méthodes principales pour installer Erlang/OTP sur Ubuntu.

Méthode 1 - Paquet binaire pré-construit #

Exécutez simplement cette commande et elle téléchargera et installera la dernière version stable d’Erlang à partir de [Erlang Solutions] (https://www.erlang-solutions.com/resources/download.html).

$ sudo apt-get install erlang

Méthode 2 - Construire et installer à partir de la source #

Téléchargez le fichier tar :

$ wget http://erlang.org/download/otp_src_19.0.tar.gz

Extrayez le fichier tar :

$ tar -zxf otp_src_19.0.tar.gz

Entrez le répertoire extrait et définissez ERL_TOP comme chemin actuel :

$ cd otp_src_19.0
$ export ERL_TOP=`pwd`

Maintenant, avant de configurer la construction, vous voulez vous assurer que vous disposez de toutes les dépendances dont vous avez besoin pour installer Erlang :

Dépendances de base :

$ sudo apt-get install autoconf libncurses-dev build-essential

Dépendances d’autres applications

Candidature Installation de dépendance
HiPE $ sudo apt-get install m4
ODBC $ sudo apt-get install unixodbc-dev
OpenSSL $ sudo apt-get install libssl-dev
wxWidgets $ sudo apt-get install libwxgtk3.0-dev libglu-dev
Documents $ sudo apt-get install fop xsltproc
Orber et autres projets C++ $ sudo apt-get install g++
jinterface $ sudo apt-get install default-jdk

Configurez et construisez : Vous pouvez définir vos propres options ou la laisser vide pour exécuter la configuration par défaut. Configuration avancée et compilation pour Erlang/OTP.

$ ./configure [ options ]
$ make

Test de la construction :

$ make release_tests
$ cd release/tests/test_server
$ $ERL_TOP/bin/erl -s ts install -s ts smoke_test batch -s init stop

Après avoir exécuté ces commandes, ouvrez $ERL_TOP/release/tests/test_server/index.html avec votre navigateur Web et vérifiez que vous n’avez aucun échec. Si tous les tests réussissent, nous pouvons continuer l’installation.

Installation :

$ make install

Construisez et installez Erlang/OTP sur OpenBSD #

Erlang sur OpenBSD est actuellement cassé sur les architectures alpha, sparc et hppa.

Méthode 1 - Paquet binaire pré-construit #

OpenBSD vous permet de choisir la version que vous souhaitez installer sur votre système :

######################################################################
# free-choice:
######################################################################
$ pkg_add erlang
# a       0: <None>
#   1: erlang-16b.03p10v0
#   2: erlang-17.5p6v0
#   3: erlang-18.1p1v0
#   4: erlang-19.0v0

######################################################################
# manual-choice:
######################################################################
pkg_add erlang%${version}
# example: 
pkg_add erlang%19

OpenBSD peut prendre en charge plusieurs versions d’Erlang. Pour faciliter l’utilisation, chaque binaire est installé en version Erlang dans son nom. Donc, si vous avez installé erlang-19.0v0, votre binaire erl sera erl19.

Si vous souhaitez utiliser erl, vous pouvez créer un lien symbolique :

ln -s /usr/local/bin/erl19 /usr/local/bin/erl

ou créez un alias dans votre fichier de configuration shell ou dans le fichier .profile :

echo 'alias erl="erl19"' >> ~/.profile
# or
echo 'alias erl="erl19"' >> ~/.shrc

Vous pouvez maintenant exécuter erl :

erl19
# or if you have an alias or symlink
erl
# Erlang/OTP 19 [erts-8.0] [source] [async-threads:10] [kernel-poll:false]
# 
# Eshell V8.0  (abort with ^G)

Méthode 2 - Construire et installer à l’aide de ports #

 RELEASE=OPENBSD_$(uname -r  | sed 's/\./_/g')
 cd /usr
 cvs -qz3 -danoncvs@anoncvs.openbsd.org:/cvs co -r${RELEASE}
 cd /usr/ports/lang/erlang
 ls -p 
 # 16/ 17/ 18/ 19/  CVS/  Makefile  Makefile.inc  erlang.port.mk
 cd 19
 make && make install

Méthode 3 - Construire à partir de la source #

La compilation à partir de la source nécessite des packages supplémentaires :

  • “git”
  • gmake
  • autoconf-2.59
pkg_add git gmake autoconf%2.59
git clone https://github.com/erlang/otp.git
cd otp
AUTOCONF_VERSION="2.59" ./build_build all
