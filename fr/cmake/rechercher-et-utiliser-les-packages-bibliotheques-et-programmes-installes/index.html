Rechercher et utiliser les packages, bibliothèques et programmes installés

Syntaxe #

  • find_package(pkgname [version] [EXACT] [QUIET] [REQUIRED])
  • inclure (FindPkgConfig)
  • pkg_search_module(prefix [REQUIRED] [QUIET] pkgname [otherpkg…])
  • pkg_check_modules(prefix [REQUIRED] [QUIET] pkgname [otherpkg…])

Paramètres #

Paramètre Détails
version (facultatif) Version minimale du package définie par un numéro majeur et éventuellement un numéro mineur, patch et tweak, au format major.minor.patch.tweak
EXACT (optionnel) Spécifiez que la version spécifiée dans version est la version exacte à trouver
OBLIGATOIRE (facultatif) Génère automatiquement une erreur et arrête le processus si le paquet n’est pas trouvé
SILENCIEUX (optionnel) La fonction n’enverra aucun message à la sortie standard

  • La méthode find_package est compatible sur toutes les plates-formes, tandis que la méthode pkg-config n’est disponible que sur les plates-formes de type Unix, telles que Linux et OSX.

  • Une description complète des nombreux paramètres et options de find_package peut être trouvée dans le manuel.

  • Même s’il est possible de spécifier de nombreux paramètres optionnels tels que la version du package, tous les modules Find n’utilisent pas correctement tous ces paramètres. Si un comportement indéfini se produit, il peut être nécessaire de trouver le module dans le chemin d’installation de CMake et de corriger ou de comprendre son comportement.

Utilisez les modules find_package et Find.cmake #

La méthode par défaut pour rechercher les packages installés avec CMake consiste à utiliser la fonction find_package conjointement avec un fichier Find<package>.cmake. Le but du fichier est de définir les règles de recherche pour le package et de définir différentes variables, telles que <package>_FOUND, <package>_INCLUDE_DIRS et <package>_LIBRARIES.

De nombreux fichiers Find<package>.cmake sont déjà définis par défaut dans CMake. Cependant, s’il n’y a pas de fichier pour le package dont vous avez besoin, vous pouvez toujours écrire le vôtre et le mettre dans ${CMAKE_SOURCE_DIR}/cmake/modules (ou tout autre répertoire si CMAKE_MODULE_PATH a été remplacé)

Une liste des modules par défaut peut être trouvée dans le manuel (v3.6). Il est essentiel de vérifier le manuel en fonction de la version de CMake utilisée dans le projet, sinon il pourrait y avoir des modules manquants. Il est également possible de trouver les modules installés avec cmake --help-module-list.

Il y a un bel exemple pour un FindSDL2.cmake sur Github

Voici un CMakeLists.txt de base qui nécessiterait SDL2 :

cmake_minimum_required(2.8 FATAL_ERROR)
project("SDL2Test")

set(CMAKE_MODULE_PATH "${CMAKE_MODULE_PATH} ${CMAKE_SOURCE_DIR}/cmake/modules")
find_package(SDL2 REQUIRED)

include_directories(${SDL2_INCLUDE_DIRS})
add_executable(${PROJECT_NAME} main.c)
target_link_libraries(${PROJECT_NAME} ${SDL2_LIBRARIES})

Utilisez pkg_search_module et pkg_check_modules #

Sur les systèmes d’exploitation de type Unix, il est possible d’utiliser le programme pkg-config pour trouver et configurer les packages qui fournissent un fichier <package>.pc.

Pour utiliser pkg-config, il est nécessaire d’appeler include(FindPkgConfig) dans un CMakeLists.txt. Ensuite, il y a 2 fonctions possibles :

  • pkg_search_module, qui recherche le paquet et utilise le premier disponible.
  • pkg_check_modules, qui vérifie tous les packages correspondants.

Voici un CMakeLists.txt de base qui utilise pkg-config pour trouver SDL2 avec une version supérieure ou égale à 2.0.1 :

cmake_minimum_required(2.8 FATAL_ERROR)
project("SDL2Test")

include(FindPkgConfig)
pkg_search_module(SDL2 REQUIRED sdl2>=2.0.1)

include_directories(${SDL2_INCLUDE_DIRS})
add_executable(${PROJECT_NAME} main.c)
target_link_libraries(${PROJECT_NAME} ${SDL2_LIBRARIES})
