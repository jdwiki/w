Installation ou configuration #

OrientDB est disponible en deux éditions :

  • Community Edition est publié en tant que projet open source sous la [licence Apache 2](http://www.apache.org/licenses/LICENSE- 2.0.html). Cette licence permet une utilisation gratuite et illimitée pour les projets open source et commerciaux.

  • Enterprise Edition est un logiciel commercial construit sur la Community Edition. Enterprise est développé par la même équipe qui a développé le moteur OrientDB. Il sert d’extension de l’édition communautaire, fournissant des fonctionnalités d’entreprise, telles que :

    • Non-Stop Backup and Restore
    • Scheduled FULL and Incremental Backups
    • Query Profiler
    • Distributed Clustering configuration
    • Metrics Recording
    • Live Monitoring with configurable Alerts

L’édition communautaire est disponible sous forme de package binaire à télécharger ou sous forme de code source sur GitHub. La licence Enterprise Edition est incluse avec les achats Support.

Conditions préalables

Les deux éditions d’OrientDB s’exécutent sur n’importe quel système d’exploitation qui implémente la machine virtuelle Java (JVM). Voici quelques exemples :

  • Linux, toutes distributions, y compris ARM (Raspberry Pi, etc.)
  • Mac OS X
  • Microsoft Windows, à partir de 95/NT et supérieur -Solaris -HP-UX -IBM AIX

OrientDB nécessite Java, version 1.7 ou supérieure.

Remarque : dans les conteneurs OSGi, OrientDB utilise une implémentation ConcurrentLinkedHashMap fournie par concurrentlinkedhashmap pour créer le cache basé sur LRU. Cette bibliothèque utilise activement le package sun.misc qui n’est généralement pas exposé en tant que package système. Pour surmonter cette limitation, vous devez ajouter la propriété org.osgi.framework.system.packages.extra avec la valeur sun.misc à votre liste de propriétés de framework.

Cela peut être aussi simple que de passer un argument à la VM démarrant la plateforme :

$ java -Dorg.osgi.framework.system.packages.extra=sun.misc

Utiliser Docker #

Si Docker est installé sur votre ordinateur, c’est le moyen le plus simple d’exécuter OrientDB. A partir de la ligne de commande tapez :

$ docker run -d –-name orientdb -p 2424:2424 -p 2480:2480
   -e ORIENTDB_ROOT_PASSWORD=root orientdb:latest

Où au lieu de “root”, tapez le mot de passe root que vous souhaitez utiliser.

Installation binaire #

OrientDB fournit un package binaire pré-compilé pour installer la base de données sur votre système. Selon votre système d’exploitation, il s’agit d’un package tar ou compressé qui contient tous les fichiers pertinents dont vous avez besoin pour exécuter OrientDB. Pour les installations de bureau, accédez à OrientDB Downloads et sélectionnez le package qui convient le mieux à votre système.

Sur les installations serveur, vous pouvez utiliser l’utilitaire wget :

$ wget https://orientdb.com/download.php?file=orientdb-community-2.2.0.tar.gz

Que vous utilisiez votre navigateur Web ou wget, décompressez ou extrayez le fichier téléchargé dans un répertoire pratique pour votre utilisation (par exemple, /opt/orientdb/ sous Linux). Cela crée un répertoire appelé orientdb-community-2.2.0 avec les fichiers et scripts pertinents, dont vous aurez besoin pour exécuter OrientDB sur votre système.

Installation du code source #

En plus de télécharger les packages binaires, vous avez également la possibilité de compiler OrientDB à partir du code source de Community Edition, disponible sur GitHub. Ce processus nécessite que vous installiez Git et Apache Maven sur votre système.

Pour compiler OrientDB à partir du code source, clonez le référentiel Community Edition, puis exécutez Maven (mvn) dans le répertoire nouvellement créé :

$ git clone https://github.com/orientechnologies/orientdb
$ git checkout develop
$ cd orientdb
$ mvn clean install

Il est possible de sauter des tests :

$ mvn clean install -DskipTests

La branche develop contient du code pour la prochaine version d’OrientDB. Les versions stables sont étiquetées sur la branche master. Pour chaque version maintenue, OrientDB a sa propre branche “hotfix”. Au moment de la rédaction de ces notes, l’état des branches est :

  • développement : travail en cours pour la prochaine version 3.0.x (3.0.x-SNAPSHOT)
  • 2.2.x : correctif pour la prochaine version stable 2.2.x (2.2.x-SNAPSHOT)
  • 2.1.x : correctif pour la prochaine version stable 2.1.x (2.1.x-SNAPSHOT)
  • 2.0.x : correctif pour la prochaine version stable 2.0.x (2.0.x-SNAPSHOT)
  • la dernière balise sur le maître est 2.2.0

Le processus de construction installe tous les jars dans le référentiel maven local et crée des archives sous le module distribution dans le répertoire target. Au moment de la rédaction, la construction à partir de la branche 2.1.x donnait :

$ls -l distribution/target/
total 199920
    1088 26 Jan 09:57 archive-tmp
     102 26 Jan 09:57 databases
     102 26 Jan 09:57 orientdb-community-3.0.0-SNAPSHOT.dir
48814386 26 Jan 09:57 orientdb-community-3.0.0-SNAPSHOT.tar.gz
53542231 26 Jan 09:58 orientdb-community-3.0.0-SNAPSHOT.zip
$

Le répertoire orientdb-community-3.0.0-SNAPSHOT.dir contient la distribution OrientDB non compressée. Jetez un œil à Contribute to OrientDB si vous voulez être impliqué.

Chaque package de distribution contient un exemple de base de données appelé GratefulDeadConcerts. Il est possible de générer la base de données :

$ cd distribution/
$ mvn  prepare-package

$ ls target/databases/

Autorisations de mise à jour #

Pour les systèmes d’exploitation Linux, Mac OS X et UNIX, vous devez modifier les autorisations sur certains fichiers après la compilation à partir de la source.

$ chmod 755 bin/*.sh
$ chmod -R 777 config

Ces commandes mettent à jour les autorisations d’exécution sur les fichiers du répertoire config/ et les scripts shell dans bin/, garantissant que vous pouvez exécuter les scripts ou les programmes que vous avez compilés.

Tâches de post-installation #

Pour les utilisateurs de bureau installant le binaire, OrientDB est maintenant installé et peut être exécuté via des scripts shell trouvés dans le répertoire bin du package de l’installation. Pour les serveurs, vous devez suivre certaines étapes supplémentaires afin de gérer le serveur de base de données pour OrientDB en tant que service. La procédure pour cela varie en fonction de votre système d’exploitation.

  • Installer en tant que service sur Unix, Linux et Mac OS X
  • Installer en tant que service sur Microsoft Windows

Mise à jour #

Lorsque vient le temps de mettre à niveau vers une version plus récente d’OrientDB, les méthodes varient en fonction de la façon dont vous avez choisi de l’installer en premier lieu. Si vous avez installé à partir de téléchargements binaires, répétez le processus de téléchargement ci-dessus et mettez à jour tous les liens symboliques ou raccourcis pour pointer vers le nouveau répertoire.

Pour les systèmes où OrientDB a été construit à partir de la source, déroulez le dernier code source et compilez à partir de la source.

$ git pull origin master
$ mvn clean install

Gardez à l’esprit que lorsque vous créez à partir de la source, vous pouvez changer de branche pour créer différentes versions d’OrientDB à l’aide de Git. Par exemple,

$ git checkout 2.2.x
$ mvn clean install

construit la branche 2.2.x, au lieu de master.

Construire un seul jar exécutable avec OrientDB #

OrientDB pour les composants internes tels que les moteurs, les opérateurs, les usines utilise Java SPI [Service Provider Interface] (https://docs.oracle.com/javase/tutorial/ext/basics/spi.html). Cela signifie que les jars d’OrientDB sont livrés avec des fichiers dans META-INF/services qui contiennent l’implémentation des composants. Gardez à l’esprit que lors de la construction d’un seul jar exécutable, vous devez concaténer le contenu des fichiers portant le même nom dans différents orientdb-*.jar . Si vous utilisez Maven Shade Plugin, vous pouvez utiliser [Service Resource Transformer](https://maven.apache.org/plugins/ maven-shade-plugin/examples/resource-transformers.html#ServicesResourceTransformer) pour le faire.
