Configuration de l’application ExtJS 6 #

Premiers pas avec Ext JS 6.

Étapes préalables :

  1. Téléchargez [Sencha Cmd 6][1] et installez-le avec la commande sencha définie dans environment path variables.
  2. Téléchargez la version d’essai de [Sencha SDK][2] (ext-6.2.1.zip) et décompressez-la. (à l’emplacement D:\ext-6.2.1)

Le système est prêt à créer l’application Extjs 6.

Créer un exemple d’application (World) avec la structure de dossiers standard Extjs 6

Il existe trois types d’application :

  1. Universel (fonctionne sur le Web mobile Sencha touch et le Web de bureau Sencha Extjs)

  2. Classique (fonctionne sur le Web de bureauSencha Extjs)

  3. Moderne (fonctionne sur le Web mobile ‘Sencha touch’)

Ouvrez CMD et exécutez la commande ci-dessous pour générer l’application “World” :

Application universelle : sencha -sdk D:\ext-6.2.1 génère l'application World D:\World

Application classique : sencha -sdk D:\ext-6.2.1 génère l'application --classic World D:\World

Application moderne : sencha -sdk D:\ext-6.2.1 génère l'application --modern World D:\World

Cette commande respective générera la structure de dossiers de l’application à l’emplacement respectif.

Accédez à cd D:/World et exécutez la commande :

regarder l’application sencha

L’application sera chargée sur :

http://localhost:1841/

L’application “World” est accessible depuis le navigateur.

  • Le chemin de l’application peut varier selon les besoins.

Afin de créer une application extjs pour l’utilisation du code de production,

création d’application sencha

Cela créera du code extjs compressé et obfusqué prêt pour une utilisation en production dans [APPLICATION_HOME]/build/production. Le répertoire de construction contient également d’autres dossiers pour le code de développement et de test.

[1] : https://www.sencha.com/products/extjs/cmd-download/ [2] : https://www.sencha.com/products/extjs/evaluate/
