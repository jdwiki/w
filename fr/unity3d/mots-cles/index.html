Une balise est une chaîne qui peut être appliquée pour marquer les types GameObject. De cette manière, il est plus facile d’identifier des objets GameObject particuliers via le code.

Une balise peut être appliquée à un ou plusieurs objets de jeu, mais un objet de jeu n’aura toujours qu’une seule balise. Par défaut, la balise “Untagged” est utilisée pour représenter un GameObject qui n’a pas été tagué intentionnellement.

Création et application de balises #

Les balises sont généralement appliquées via l’éditeur ; cependant, vous pouvez également appliquer des balises via un script. Toute balise personnalisée doit être créée via la fenêtre * Balises et calques * avant d’être appliquée à un objet de jeu.

Définition des balises dans l’éditeur #

Avec un ou plusieurs objets de jeu sélectionnés, vous pouvez sélectionner une balise dans l’inspecteur. Les objets de jeu porteront toujours une seule balise ; par défaut, les objets du jeu seront étiquetés comme “Untagged”. Vous pouvez également passer à la fenêtre Balises et calques, en sélectionnant “Ajouter une balise…" ; cependant, il est important de noter que cela ne vous amène qu’à la fenêtre Tags & Layers. Toute balise que vous créez ne s’appliquera pas automatiquement à l’objet du jeu.

[![Le menu déroulant des balises se trouve directement sous le nom des objets du jeu dans la fenêtre de l’inspecteur.][1]][1]

Définir des balises via un script #

Vous pouvez modifier directement une balise d’objets de jeu via le code. Il est important de noter que vous devez fournir une balise de la liste des balises actuelles ; si vous fournissez une balise qui n’a pas encore été créée, cela entraînera une erreur.

Comme détaillé dans d’autres exemples, l’utilisation d’une série de variables de “chaîne statique” au lieu d’écrire manuellement chaque balise peut garantir la cohérence et la fiabilité.

Le script suivant montre comment nous pouvons modifier une série de balises d’objets de jeu, en utilisant des références de “chaîne statique” pour assurer la cohérence. Notez l’hypothèse que chaque chaîne statique représente une balise qui a déjà été créée dans la fenêtre Tags & Layers.

using UnityEngine;

public class Tagging : MonoBehaviour
{
    static string tagUntagged = "Untagged";
    static string tagPlayer = "Player";
    static string tagEnemy = "Enemy";

    /// <summary>Represents the player character. This game object should 
    /// be linked up via the inspector.</summary>
    public GameObject player;
    /// <summary>Represents all the enemy characters. All enemies should 
    /// be added to the array via the inspector.</summary>
    public GameObject[] enemy;

    void Start ()
    {
        // We ensure that the game object this script is attached to
        // is left untagged by using the default "Untagged" tag.
        gameObject.tag = tagUntagged;

        // We ensure the player has the player tag.
        player.tag = tagUntagged;

        // We loop through the enemy array to ensure they are all tagged.
        for(int i = 0; i < enemy.Length; i++)
        {
            enemy[i].tag = tagEnemy;
        }
    }
}

Création de balises personnalisées #

Que vous définissiez des balises via l’inspecteur ou via un script, les balises doivent être déclarées via la fenêtre Tags & Layers avant utilisation. Vous pouvez accéder à cette fenêtre en sélectionnant “Ajouter des balises…” dans un menu déroulant de balises d’objets de jeu. Vous pouvez également trouver la fenêtre sous Édition > Paramètres du projet > Balises et calques.

[![Les balises peuvent contenir des espaces et des signes de ponctuation.][2]][2]

Sélectionnez simplement le bouton +, entrez le nom souhaité et sélectionnez Enregistrer pour créer une balise. La sélection du bouton - supprimera la balise actuellement en surbrillance. Notez que de cette manière, la balise sera immédiatement affichée comme "(Removed)”, et sera complètement supprimée lors du prochain rechargement du projet.

La sélection de l’engrenage/du rouage en haut à droite de la fenêtre vous permettra de réinitialiser toutes les options personnalisées. Cela supprimera immédiatement toutes les balises personnalisées, ainsi que tout calque personnalisé que vous pourriez avoir sous “Trier les calques” et “Calques”.

[1] : https://i.stack.imgur.com/JWzCP.png “Par défaut, les objets du jeu peuvent être étiquetés comme Untagged, Respawn, Finish, EditorOnly, MainCamera, Player ou GameController. Vous pouvez également passer aux Tags & Calques pour créer des balises personnalisées.” [2] : https://i.stack.imgur.com/NAVSg.png “Depuis la fenêtre Balises et calques, nous pouvons ajouter et supprimer des balises personnalisées”

Recherche de GameObjects par tag : #

Les balises facilitent particulièrement la localisation d’objets de jeu spécifiques. Nous pouvons rechercher un seul objet de jeu ou en rechercher plusieurs.

Trouver un seul GameObject #

Nous pouvons utiliser la fonction statique GameObject.FindGameObjectWithTag(string tag) pour rechercher des objets de jeu individuels. Il est important de noter que, de cette manière, les objets du jeu ne sont pas interrogés dans un ordre particulier. Si vous recherchez une balise utilisée sur plusieurs objets de jeu dans la scène, cette fonction ne sera pas en mesure de garantir quel objet de jeu est renvoyé. En tant que tel, il est plus approprié lorsque nous savons qu’un seul * objet de jeu * utilise une telle balise, ou lorsque nous ne nous inquiétons pas de l’instance exacte de GameObject qui est renvoyée.

///<summary>We create a static string to allow us consistency.</summary>
string playerTag = "Player"

///<summary>We can now use the tag to reference our player GameObject.</summary>
GameObject player = GameObject.FindGameObjectWithTag(playerTag);

Recherche d’un tableau d’instances GameObject #

Nous pouvons utiliser la fonction statique GameObject.FindGameObjectsWithTag(string tag) pour rechercher tous les objets de jeu qui utilisent une balise particulière. Ceci est utile lorsque nous voulons parcourir un groupe d’objets de jeu particuliers. Cela peut également être utile si nous voulons trouver un objet de jeu * unique *, mais que nous pouvons avoir * plusieurs * objets de jeu utilisant la même balise. Comme nous ne pouvons pas garantir l’instance exacte renvoyée par GameObject.FindGameObjectWithTag(string tag), nous devons plutôt récupérer un tableau de toutes les instances potentielles de GameObject avec GameObject.FindGameObjectsWithTag(string tag), et analyser plus en détail le tableau résultant pour trouver l’instance que nous recherchons.

///<summary>We create a static string to allow us consistency.</summary>
string enemyTag = "Enemy";

///<summary>We can now use the tag to create an array of all enemy GameObjects.</summary>
GameObject[] enemies = GameObject.FindGameObjectsWithTag(enemyTag );

// We can now freely iterate through our array of enemies
foreach(GameObject enemy in enemies)
{
    // Do something to each enemy (link up a reference, check for damage, etc.)
}

Comparer les balises #

Lors de la comparaison de deux GameObjects par balises, il convient de noter que les éléments suivants entraîneraient une surcharge du Garbage Collector car une chaîne est créée à chaque fois :

if (go.Tag == "myTag")
{ 
    //Stuff 
}

Lorsque vous effectuez ces comparaisons dans Update() et d’autres rappels réguliers de Unity (ou une boucle), vous devez utiliser cette méthode sans allocation de tas :

if (go.CompareTag("myTag")
{ 
    //Stuff 
}

De plus, il est plus facile de conserver vos balises dans une classe statique.

public static class Tags
{
    public const string Player = "Player";
    public const string MyCustomTag = "MyCustomTag";
}

Ensuite, vous pouvez comparer en toute sécurité

if (go.CompareTag(Tags.MyCustomTag)
{ 
    //Stuff 
}

de cette façon, vos chaînes de balises sont générées au moment de la compilation et vous limitez les implications des fautes d’orthographe.

Tout comme conserver les balises dans une classe statique, il est également possible de les stocker dans une énumération :

public enum Tags
{
    Player, Ennemies, MyCustomTag;
}

et ensuite vous pouvez le comparer en utilisant la méthode enum toString() :

if (go.CompareTag(Tags.MyCustomTag.toString())
{ 
    //Stuff 
}
