Corps rigides #

Aperçu #

Le composant Rigidbody donne à un GameObject une * présence physique * dans la scène en ce sens qu’il est capable de répondre aux forces. Vous pouvez appliquer des forces directement au GameObject ou lui permettre de réagir à des forces externes telles que la gravité ou un autre Rigidbody qui le frappe.

Ajout d’un composant Rigidbody #

Vous pouvez ajouter un Rigidbody en cliquant sur Composant > Physique > Rigidbody

Déplacer un objet Rigidbody #

Il est recommandé que si vous appliquez un Rigidbody à un GameObject, vous utilisez des forces ou un couple pour le déplacer plutôt que de manipuler sa transformation. Utilisez les méthodes AddForce() ou AddTorque() pour cela :

// Add a force to the order of myForce in the forward direction of the Transform.
GetComponent<Rigidbody>().AddForce(transform.forward * myForce);

// Add torque about the Y axis to the order of myTurn.
GetComponent<Rigidbody>().AddTorque(transform.up * torque * myTurn);

Masse #

Vous pouvez modifier la masse d’un Rigidbody GameObject pour affecter la façon dont il réagit avec d’autres Rigidbodies et forces. Une masse plus élevée signifie que le GameObject aura plus d’influence sur les autres GameObjects basés sur la physique et nécessitera une plus grande force pour se déplacer. Les objets de masse différente tomberont à la même vitesse s’ils ont les mêmes valeurs de traînée. Pour modifier la masse dans le code :

GetComponent<Rigidbody>().mass = 1000;

Glisser #

Plus la valeur de traînée est élevée, plus un objet ralentira en se déplaçant. Pensez-y comme une force opposée. Pour modifier le glisser dans le code :

GetComponent<Rigidbody>().drag = 10;

estCinématique #

Si vous marquez un Rigidbody comme Kinematic, il ne peut pas être affecté par d’autres forces mais peut toujours affecter d’autres GameObjects. Pour modifier dans le code :

GetComponent<Rigidbody>().isKinematic = true;

Contraintes #

Il est également possible d’ajouter des contraintes à chaque axe pour figer la position ou la rotation du Rigidbody dans l’espace local. La valeur par défaut est RigidbodyConstraints.None comme indiqué ici :

[![entrez la description de l’image ici][1]][1]

Un exemple de contraintes dans le code :

// Freeze rotation on all axes.
GetComponent<Rigidbody>().constraints = RigidbodyConstraints.FreezeRotation 

// Freeze position on all axes.
GetComponent<Rigidbody>().constraints = RigidbodyConstraints.FreezePosition 

// Freeze rotation and motion an all axes.
GetComponent<Rigidbody>().constraints = RigidbodyConstraints.FreezeAll

Vous pouvez utiliser l’opérateur OR au niveau du bit | pour combiner plusieurs contraintes comme ceci :

// Allow rotation on X and Y axes and motion on Y and Z axes.
GetComponent<Rigidbody>().constraints = RigidbodyConstraints.FreezePositionZ | 
    RigidbodyConstraints.FreezeRotationX;

Collisions #

Si vous voulez qu’un GameObject avec un Rigidbody dessus réponde aux collisions, vous devrez également lui ajouter un collisionneur. Les types de collisionneurs sont :

  • Collisionneur de boîtes
  • Collisionneur de sphère
  • Collisionneur de capsules
  • Collisionneur de roue
  • Collisionneur de maille

Si vous appliquez plus d’un collisionneur à un GameObject, nous l’appelons un collisionneur composé.

Vous pouvez transformer un collisionneur en Trigger afin d’utiliser les méthodes OnTriggerEnter(), OnTriggerStay() et OnTriggerExit(). Un collisionneur déclencheur ne réagira pas physiquement aux collisions, d’autres GameObjects le traverseront simplement. Ils sont utiles pour détecter quand un autre GameObject se trouve dans une certaine zone ou non, par exemple, lors de la collecte d’un élément, nous pouvons vouloir simplement le parcourir mais détecter quand cela se produit.

[1] : http://i.stack.imgur.com/e6PwA.png

Gravité dans un corps rigide #

La propriété useGravity d’un RigidBody contrôle si la gravité l’affecte ou non. S’il est défini sur “false”, le “RigidBody” se comportera comme s’il se trouvait dans l’espace (sans qu’une force constante ne lui soit appliquée dans une certaine direction).

GetComponent<RigidBody>().useGravity = false;

Il est très utile dans les situations où vous avez besoin de toutes les autres propriétés de RigidBody à l’exception du mouvement contrôlé par la gravité.

Lorsqu’il est activé, le RigidBody sera affecté par une force gravitationnelle, configurée sous Physics Settings :

[![entrez la description de l’image ici][1]][1]

La gravité est définie en unités mondiales par seconde au carré et est saisie ici sous la forme d’un vecteur tridimensionnel : ce qui signifie qu’avec les paramètres de l’exemple d’image, tous les “RigidBodies” avec la propriété “useGravity” définie sur “True” subiront une force de 9,81 unités mondiales par seconde par seconde vers le bas (comme les valeurs Y négatives dans le système de coordonnées d’Unity pointent vers le bas).

[1] : https://i.stack.imgur.com/nozwu.png
