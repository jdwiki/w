Syntaxe #

  • Public static GameObject Find(string name);
  • GameObject public statique FindGameObjectWithTag (balise de chaîne);
  • public static GameObject [] FindGameObjectsWithTag (balise de chaîne);
  • Objet statique public FindObjectOfType (Type type);
  • public static Object[] FindObjectsOfType(Type type);

Quelle méthode utiliser #

Soyez prudent lorsque vous recherchez des GameObjects au moment de l’exécution, car cela peut consommer beaucoup de ressources. Notamment : n’exécutez pas FindObjectOfType ou Find dans Update, FixedUpdate ou plus généralement dans une méthode appelée une ou plusieurs fois par frame.

  • Appelez les méthodes d’exécution FindObjectOfType et Find uniquement lorsque cela est nécessaire
  • FindGameObjectWithTag a de très bonnes performances par rapport aux autres méthodes basées sur des chaînes. Unity garde des onglets séparés sur les objets étiquetés et les interroge au lieu de la scène entière.
  • Pour les GameObjects “statiques” (tels que les éléments d’interface utilisateur et les préfabriqués) créés dans l’éditeur, utilisez [référence GameObject sérialisable][1] dans l’éditeur
  • Gardez vos listes de GameObjects dans des listes ou des tableaux que vous gérez vous-même
  • En général, si vous instanciez beaucoup de GameObjects du même type, jetez un œil à [Object Pooling][2]
  • Mettez en cache vos résultats de recherche pour éviter d’exécuter encore et encore les méthodes de recherche coûteuses.

Approfondir #

Outre les méthodes fournies avec Unity, il est relativement facile de concevoir vos propres méthodes de recherche et de collecte.

  • Dans le cas de FindObjectsOfType(), vous pouvez faire en sorte que vos scripts conservent une liste d’eux-mêmes dans une collection statique. Il est beaucoup plus rapide d’itérer une liste prête d’objets que de rechercher et d’inspecter des objets de la scène.

  • Ou créez un script qui stocke leurs instances dans un “Dictionnaire” basé sur une chaîne, et vous disposez d’un système de marquage simple que vous pouvez développer.

[1] : https://www.wikiod.com/fr/unity3d/trouver-et-collecter-des-gameobjects#Inséré dans les scripts en mode édition [2] : https://www.wikiod.com/fr/unity3d/regroupement-dobjets#Pool d’objets

Recherche par nom de GameObject #

var go = GameObject.Find("NameOfTheObject");
Avantages Les inconvénients
Facile à utiliser Les performances se dégradent avec le nombre d’objets de jeu dans la scène
Les chaînes sont des * références faibles * et suspectes d’erreurs de l’utilisateur

Recherche par les balises de GameObject #

var go = GameObject.FindGameObjectWithTag("Player");
Avantages Les inconvénients
Possibilité de rechercher à la fois des objets uniques et des groupes entiers Les chaînes sont des références faibles et suspectes d’erreurs de l’utilisateur.
Relativement rapide et efficace Le code n’est pas portable car les balises sont codées en dur dans les scripts.

Recherche de GameObjects par des scripts MonoBehaviour #

ExampleScript script = GameObject.FindObjectOfType<ExampleScript>();
GameObject go = script.gameObject;

FindObjectOfType() renvoie null si aucun n’est trouvé.

Avantages Les inconvénients
Fortement typé Les performances se dégradent en fonction du nombre d’objets de jeu nécessaires pour évaluer
Possibilité de rechercher à la fois des objets uniques et des groupes entiers

Inséré dans les scripts en mode édition #

[SerializeField]
GameObject[] gameObjects;
Avantages Les inconvénients
Grandes performances La collection d’objets est statique
Code portatif Ne peut faire référence qu’aux GameObjects de la même scène

Trouver GameObjects par nom à partir d’objets enfants #

Transform tr = GetComponent<Transform>().Find("NameOfTheObject");
GameObject go = tr.gameObject;

Find renvoie null si aucun n’est trouvé

Avantages Les inconvénients
Champ de recherche limité et bien défini Les chaînes sont des références faibles
