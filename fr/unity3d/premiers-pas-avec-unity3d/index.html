Installation ou configuration #

Aperçu #

Unity fonctionne sur Windows et Mac. Il existe également une version Linux alpha disponible.

Il existe 4 plans de paiement différents pour Unity :

  1. Personnel - Gratuit (voir ci-dessous)
  2. Plus - 35 USD par mois et par siège (voir ci-dessous)
  3. Pro - 125 USD par mois et par siège - Après avoir souscrit au plan Pro pendant 24 mois consécutifs, vous avez la possibilité d’arrêter de vous abonner et de conserver la version dont vous disposez.
  4. Entreprise - [Contactez Unity pour plus d’informations][1]

Selon le CLUF : les entreprises ou les entités constituées qui ont réalisé un chiffre d’affaires supérieur à 100 000 USD au cours de leur dernier exercice doivent utiliser Unity Plus (ou une licence supérieure) ; au-delà de 200 000 USD, ils doivent utiliser Unity Pro (ou Enterprise).

Installation #

  1. Téléchargez [l’assistant de téléchargement Unity][2].

  2. Exécutez l’assistant et choisissez les modules que vous souhaitez télécharger et installer, tels que l’éditeur Unity, l’IDE MonoDevelop, la documentation et les modules de construction de plate-forme souhaités.

Si vous avez une version plus ancienne, vous pouvez mettre à jour vers la dernière version stable.

Si vous souhaitez installer Unity sans l’assistant de téléchargement Unity, vous pouvez obtenir les programmes d’installation de composants à partir de [Notes de version Unity 5.5.1][3].

Installation de plusieurs versions de Unity #

Il est souvent nécessaire d’installer plusieurs versions de Unity en même temps. Faire cela:

  • Sous Windows, remplacez le répertoire d’installation par défaut par un dossier vide que vous avez précédemment créé, tel que Unity 5.3.1f1.

  • Sur Mac, le programme d’installation s’installera toujours dans /Applications/Unity. Renommez ce dossier pour votre installation existante (par exemple en /Applications/Unity5.3.1f1) avant d’exécuter le programme d’installation pour la version différente.

  • Vous pouvez maintenir Alt lors du lancement de Unity pour le forcer à vous laisser choisir un projet à ouvrir. Sinon, le dernier projet chargé tentera de se charger (si disponible) et il peut vous inviter à mettre à jour un projet que vous ne souhaitez pas mettre à jour.

[1] : https://store.unity.com/contact?type=sales [2] : https://unity3d.com/get-unity/download [3] : https://unity3d.com/unity/whats-new/unity-5.5.1

Éditeur et code de base #

Disposition #

L’éditeur de base Unity ressemblera à ci-dessous. Les fonctionnalités de base de certaines fenêtres/onglets par défaut sont décrites dans l’image.

[![entrez la description de l’image ici][1]][1]

Mise en page Linux #

Il y a une petite différence dans la disposition du menu de la version Linux, comme la capture d’écran ci-dessous, [![entrez la description de l’image ici][2]][2]

Utilisation de base #

Créez un GameObject vide en faisant un clic droit dans la fenêtre Hierarchy et sélectionnez Create Empty. Créez un nouveau script en cliquant avec le bouton droit de la souris dans la fenêtre de projet et sélectionnez Create > C# Script. Renommez-le au besoin.

Lorsque le GameObject vide est sélectionné dans la fenêtre Hiérarchie, faites glisser et déposez le script nouvellement créé dans la fenêtre Inspecteur. Maintenant, le script est attaché à l’objet dans la fenêtre Hiérarchie. Ouvrez le script avec l’IDE MonoDevelop par défaut ou selon vos préférences.

Script de base #

Le code de base ressemblera à ci-dessous, à l’exception de la ligne Debug.Log("hello world!!");.

using UnityEngine;
using System.Collections;

public class BasicCode : MonoBehaviour {

    // Use this for initialization
    void Start () {
        Debug.Log("hello world!!");
    }
    
    // Update is called once per frame
    void Update () {
    
    }
}

Ajoutez la ligne Debug.Log("hello world!!"); dans la méthode void Start(). Enregistrez le script et revenez à l’éditeur. Lancez-le en appuyant sur Play en haut de l’éditeur.

Le résultat devrait être comme ci-dessous dans la fenêtre de la console :

[![entrez la description de l’image ici][3]][3]

[1] : http://i.stack.imgur.com/HDm82.png [2] : http://i.stack.imgur.com/He5k7.png [3] : http://i.stack.imgur.com/A6TWZ.png

Mises en page de l’éditeur #

Vous pouvez enregistrer la disposition de vos onglets et fenêtres pour uniformiser votre environnement de travail.

Le menu des mises en page se trouve dans le coin supérieur droit de Unity Editor :

[![mises en page][1]][1]

Unity est livré avec 5 dispositions par défaut (2 par 3, 4 Split, Default, Tall, Wide) (marqué d’un 1). Dans l’image ci-dessus, outre les mises en page par défaut, il existe également une mise en page personnalisée en haut.

Vous pouvez ajouter vos propres mises en page en cliquant sur le bouton “Enregistrer la mise en page…” dans le menu (marqué d’un 2) :

[![enregistrer la mise en page][2]][2]

Vous pouvez également supprimer n’importe quelle mise en page en cliquant sur le bouton “Supprimer la mise en page…” dans le menu (marqué d’un 2) :

[![supprimer la mise en page][3]][3]

Le bouton “Revert Factory Settings…” supprime toutes les mises en page personnalisées et restaure les mises en page par défaut (marquées d’un 2).

[1] : http://i.stack.imgur.com/k4UO6.png [2] : http://i.stack.imgur.com/PyJWn.png [3] : http://i.stack.imgur.com/BxrhM.png

Personnalisation de votre espace de travail #

Vous pouvez personnaliser votre disposition des vues en cliquant-glissant l’onglet de n’importe quelle vue vers l’un des nombreux emplacements. Déposer un onglet dans la zone d’onglets d’une fenêtre existante ajoutera l’onglet à côté de tous les onglets existants. Alternativement, déposer un onglet dans n’importe quelle zone Dock ajoutera la vue dans une nouvelle fenêtre. [![entrez la description de l’image ici][1]][1]

Les onglets peuvent également être détachés de la fenêtre de l’éditeur principal et organisés dans leurs propres fenêtres d’éditeur flottantes. Les fenêtres flottantes peuvent contenir des arrangements de vues et d’onglets, tout comme la fenêtre de l’éditeur principal. [![entrez la description de l’image ici][2]][2]

Lorsque vous avez créé une mise en page d’éditeur, vous pouvez enregistrer la mise en page et la restaurer à tout moment. [Reportez-vous à cet exemple pour les mises en page de l’éditeur][3]. [![entrez la description de l’image ici][4]][4]

À tout moment, vous pouvez cliquer avec le bouton droit sur l’onglet de n’importe quelle vue pour afficher des options supplémentaires comme Agrandir ou ajouter un nouvel onglet à la même fenêtre. [![entrez la description de l’image ici][5]][5]
