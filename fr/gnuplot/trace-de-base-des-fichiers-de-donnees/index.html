L’une des principales fonctionnalités utiles de gnuplot est la possibilité de tracer des fichiers de données. Tracer un fichier de données est vraiment simple avec gnuplot, en fait, une fois que vous avez ouvert le logiciel depuis le terminal, il suffit de taper la commande plot 'file' pour obtenir un tracé automatique.

Tout d’abord, avant de tracer, vous devez être sûr d’être sous le même répertoire où se trouve le fichier de données, sinon vous finirez par recevoir un warning.

Syntaxe #

  • tracer datafile en utilisant column_expression avec style

Tracer un seul fichier de données #

La commande par défaut gnuplot plot (également uniquement p) trace l’ensemble de données avec des colonnes, sous la forme du fichier data_set.dat ci-dessous.

# Prototype of a gnuplot data set
# data_set.dat
# X -   X^2 -    2*X -    Random 
0       0        0        5
1       1        2        15
1.4142  2        2.8284   1
2       4        4        30
3       9        6        26.46
3.1415  9.8696   6.2832   39.11
4       16       8        20
4.5627  20.8182  9.1254   17 
5.0     25.0     10.0     25.50
6       36       12       0.908

Comme vous pouvez le voir, vous pouvez écrire dans votre ensemble de données en notation à virgule flottante. Maintenant tout est prêt pour faire le tracé des données : en tapant uniquement

plot "data_set.dat"

gnuplot produira un graphique dans votre destination de “sortie”. Les paramètres par défaut utiliseront les deux premières colonnes de votre fichier de données, respectivement x et y. Pour spécifier les colonnes à tracer, utilisez le spécificateur using

plot "data_set.dat" using 2:4

ce qui signifie “tracer le fichier en utilisant la colonne 2 comme X et la colonne 4 comme Y”. Dans le cas où votre ensemble de données est un fichier tridimensionnel, utilisez simplement splot et ajoutez la colonne z

splot "data_set.dat" using 1:2:3

Il existe également différents styles (voir la documentation de gnuplot ou [Selecting a plotting style][1] for more infos) pour tracer les points. Comme dit précédemment, le style par défaut est point

plot "data_set.dat" using 1:4 with point

qui tracera la même chose que si vous ne tapez pas with point. Un style utile pour le traçage des données est linespoint qui est, évidemment, “lignes + points”. PAR EXEMPLE.:

plot "data_set.dat" using 1:4 with linespoint
# the abbreviated form is completely equivalent:
# p "data_set.dat" u 1:4 w lp

[![point de ligne][2]][2]

Formes d’itération de tracé

Dans le cas où vous avez plus de colonnes et que vous souhaitez toutes les tracer dans le même graphique, passez simplement à la fonction plot n’importe quel argument que vous préférez, en les séparant par un , :

p "data_set.dat" u 1:2 w lp,\
    "data_set.dat" u 1:3 w lp,\
    "data_set.dat" u 1:4 w lp

Quoi qu’il en soit, il peut parfois y avoir trop de colonnes à écrire une par une. Dans ce cas, la boucle d’itération “for” est très utile :

p for [col = 2:4] "data_set.dat" using 1:col w lp

ce qui donne la sortie

[![Tracé itéré][3]][3]

En bref, l’itération for incrémente la variable dans la boucle, dans ce cas col, avec un pas décidé (si non spécifié = 1). Par exemple for [i = 0:6:2] incrémentera i de 0 à 6 en 2 étapes : i = 0, 2, 4, 6. Toutes les valeurs (début, arrêt et incrément) sont converties en valeurs entières.

*Grille

La grille est souvent utile lors du traçage d’un ensemble de données. Pour ajouter un type de grille

set grid

Tracer plusieurs fichiers de données #

Première méthode - Concaténation de chaînes #

La méthode la plus simple pour tracer plusieurs fichiers de données consiste à insérer une boucle “for” dans la commande “plot” de gnuplot. En supposant que vous ayez N fichiers nommés séquentiellement, * c’est-à-dire *

file_1.dat
file_2.dat
file_3.dat
...
file_N.dat

Exécution de la commande

plot for[i = 1:N] "file_".i.".dat"

tracera tous les fichiers entre file_1.dat et file_N.dat dans le même graphique.

Exemple avec trois fichiers de données

Tableau des jeux de données

Axes X fichier Axe Y_1.dat Fichier Axe Y_2.dat Axe Y file_3.dat
1 1 1 1
2 2 4 2
3 3 9 6
4 4 16 24
5 5 25 120

Commandes

set terminal postscript color noenhanced ##setting the term
set output "multiple_files.ps"

set key center ##legend placement

plot [1:5][1:120] \
    for [i = 1:3] "file_".i.".dat" \
    pointsize 1.3 linecolor i+4 \
    title "file\_".i.".dat" \
    with linespoint

La boucle commence par for [i = 1:3] "file_".i.".dat" et exécute la commande plot jusqu’à ce qu’elle atteigne i = 3. Le .i. est le nombre concaténé.

title "file\_".i.".dat" a été écrit avec le \ afin que le symbole _ dans le nom des fichiers apparaisse comme un underscore plutôt qu’un indice, et le spécificateur “noenhanced” est fondamental pour obtenir ce résultat.

Le résultat final est présenté ci-dessous

[![L’exemple de tracé][1]][1]

Deuxième méthode - Utilisation de la fonction sprintf #

Un autre chemin possible à suivre consiste à utiliser la fonction sprintf qui fonctionne fondamentalement le même [que le langage C sprintf](https://www.wikiod.com/fr/c/cordes#lecture/écriture de données au format chaîne). La bonne syntaxe, à partir de la documentation gnuplot 5.1 est

sprintf("format", x, y, ...)

Un bref exemple clarifiera tous les doutes.

file_name(n) = sprintf("file_%d.dat", n)
plot for[i = 1:N] file_name(i) title file_name(i)
