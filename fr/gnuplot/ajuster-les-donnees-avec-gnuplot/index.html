La commande fit peut ajuster une fonction définie par l’utilisateur à un ensemble de points de données (x,y) ou (x,y,z), en utilisant une implémentation des moindres carrés non linéaires (NLLS) Algorithme de Marquardt-Levenberg.

Toute variable définie par l’utilisateur apparaissant dans le corps de la fonction peut servir de paramètre d’ajustement, mais le type de retour de la fonction doit être réel.

Syntaxe #

  • fit [xrange][yrange] function “datafile” using modifier via parameter_file

Paramètres #

Paramètres Détail
Paramètres d’ajustement a, b, c et toute lettre qui n’avait pas été utilisée précédemment Utilisez des lettres pour représenter les paramètres qui seront utilisés pour ajuster une fonction. Par exemple : f(x) = a * exp(b * x) + c, g(x,y) = a*x**2 + b*y**2 + c*x*y
Paramètres du fichier start.par Au lieu d’utiliser des paramètres non initialisés (le Marquardt-Levenberg s’initialisera automatiquement pour vous a=b=c=...=1), vous pouvez les mettre dans un fichier start.par et les appelez avec dans la section parameter_file. Par exemple : fit f(x) 'data.dat' u 1:2 via 'start.par'. Un exemple pour le fichier start.par est montré ci-dessous

Brève introduction #

fit est utilisé pour trouver un ensemble de paramètres qui correspond le mieux à vos données à votre fonction définie par l’utilisateur. L’ajustement est jugé sur la base de la somme des différences au carré ou «résidus» (SSR) entre les points de données d’entrée et les valeurs de fonction, évaluées aux mêmes endroits. Cette quantité est souvent appelée « chicarré » (c’est-à-dire la lettre grecque chi, à la puissance 2). L’algorithme tente de minimiser le SSR, ou plus précisément le WSSR, car les résidus sont “pondérés” par les erreurs de données d’entrée (ou 1,0) avant d’être mis au carré. (Idem)

Le fichier fit.log

Après chaque étape d’itération, une information détaillée est donnée sur l’état de l’ajustement à la fois sur l’écran et sur un fichier journal appelé fit.log. Ce fichier ne sera jamais effacé mais toujours ajouté afin que l’historique de l’ajustement ne soit pas perdu.

Ajustement des données avec des erreurs #

Il peut y avoir jusqu’à 12 variables indépendantes, il y a toujours 1 variable dépendante et n’importe quel nombre de paramètres peut être ajusté. Facultativement, des estimations d’erreur peuvent être saisies pour pondérer les points de données. (T. Williams, C. Kelley - gnuplot 5.0, un programme de traçage interactif)

Si vous avez un ensemble de données et que vous souhaitez adapter si la commande est très simple et naturelle :

fit f(x) "data_set.dat" using 1:2 via par1, par2, par3

où à la place f(x) pourrait être aussi f(x, y). Dans le cas où vous avez également des estimations d’erreurs de données, ajoutez simplement le {y | xy | z}errors ({ | } représentent les choix possibles) dans l’option modifier (voir Syntaxe). Par exemple

fit f(x) "data_set.dat" using 1:2:3 yerrors via par1, par2, par3

où le {y | xy | L'option z}errors nécessite respectivement 1 (y), 2 (xy), 1 (z) colonne qui spécifie la valeur de l’estimation de l’erreur.

** Ajustement exponentiel avec xyerrors d’un fichier **

Les estimations d’erreur de données sont utilisées pour calculer le poids relatif de chaque point de données lors de la détermination de la somme pondérée des carrés des résidus, du WSSR ou du chicarré. Ils peuvent affecter les estimations des paramètres, car ils déterminent l’influence de l’écart de chaque point de données par rapport à la fonction ajustée sur les valeurs finales. Certaines des informations de sortie d’ajustement, y compris les estimations d’erreur de paramètre, sont plus significatives si des estimations d’erreur de données précises ont été fournies. (Ibidem)

Nous allons prendre un exemple de jeu de données measured.dat, composé de 4 colonnes : les coordonnées de l’axe des x (Temperature (K)), les coordonnées de l’axe des y (Pressure (kPa) ), les estimations d’erreur x (T_err (K)) et les estimations d’erreur y (P_err (kPa)).

Maintenant, composez simplement le prototype de la fonction qui, d’après la théorie, devrait approximer nos données. Dans ce cas:

Z = 0.001
f(x) = W * exp(x * Z)

où nous avons initialisé le paramètre Z car sinon l’évaluation de la fonction exponentielle exp(x * Z) donne des valeurs énormes, ce qui conduit à (virgule flottante) Infinity et NaN dans l’algorithme d’ajustement de Marquardt-Levenberg, généralement vous ne le feriez pas besoin d’initialiser les variables - regardez ici, si vous voulez en savoir plus sur Marquardt-Levenberg.

Il est temps d’adapter les données!

fit f(x) "measured.dat" u 1:2:3:4 xyerrors via W, Z

Tracé avec ajustement de measured.dat L’utilisation de la commande with xyerrorbars affichera les estimations d’erreurs sur le x et sur le y. set grid placera une grille en pointillés sur les tics majeurs. [![Ajustement exponentiel][1]][1]

Dans le cas où les estimations d’erreur ne sont pas disponibles ou sans importance, il est également possible d’ajuster les données sans le `{y | xy | Option de montage z}errors :

fit f(x) "measured.dat" u 1:2 via W, Z

Dans ce cas, les xyerrorbars devaient également être évités.

Exemple de fichier “start.par” #

Si vous chargez vos paramètres d’ajustement à partir d’un fichier, vous devez y déclarer tous les paramètres que vous allez utiliser et, si nécessaire, les initialiser.

## Start parameters for the fit of data.dat

m  = -0.0005           
q  = -0.0005           
d  = 1.02              
Tc = 45.0
g_d = 1.0
b  = 0.01002

Fit : interpolation linéaire de base d’un ensemble de données #

L’utilisation de base de l’ajustement est mieux expliquée par un exemple simple :

f(x) = a + bx + cx**2 fit [-234:320][0:200] f(x) ‘measured.dat’ en utilisant 1:2 sauter 4 via a,b,c plot ‘measured.dat’ u 1:2, f(x) Des plages peuvent être spécifiées pour filtrer les données utilisées dans l’ajustement. Les points de données hors plage sont ignorés. (T. Williams, C. Kelley - gnuplot 5.0, un programme de traçage interactif)

L’interpolation linéaire (ajustement avec une ligne) est le moyen le plus simple d’ajuster un ensemble de données. Supposons que vous ayez un fichier de données où la croissance de votre quantité y est linéaire, vous pouvez utiliser

[…] polynômes linéaires pour construire de nouveaux points de données dans la plage d’un ensemble discret de points de données connus. (de Wikipedia, Interpolation linéaire)

Exemple avec un polynôme de première année #

Nous allons travailler avec l’ensemble de données suivant, appelé house_price.dat, qui comprend les mètres carrés d’une maison dans une certaine ville et son prix en 1000 $.

Ajustons ces paramètres avec gnuplot La commande elle-même est très simple, comme vous pouvez le remarquer à partir de la syntaxe, définissez simplement votre prototype d’ajustement, puis utilisez la commande fit pour obtenir le résultat :

## m, q will be our fitting parameters
f(x) = m * x + q       
fit f(x) 'data_set.dat' using 1:2 via m, q

Mais il pourrait être intéressant d’utiliser également les paramètres obtenus dans l’intrigue elle-même. Le code ci-dessous correspondra au fichier house_price.dat, puis tracera les paramètres m et q pour obtenir la meilleure approximation de la courbe de l’ensemble de données. Une fois que vous avez les paramètres, vous pouvez calculer la valeur y, dans ce cas le prix de la maison, à partir de n’importe quelle x-vaule donnée (mètres carrés de la maison) en remplaçant simplement dans la formule

y = m * x + q

la “valeur x” appropriée. Commentons le code.

0. Définition du terme

set term pos col
set out 'house_price_fit.ps'

1. Administration ordinaire pour embellir le graphique

set title 'Linear Regression Example Scatterplot'
set ylabel 'House price (k$ = $1000)'
set xlabel 'Square meters (m^2)'
set style line 1 ps 1.5 pt 7 lc 'red'
set style line 2 lw 1.5 lc 'blue'

set grid
set key bottom center box height 1.4

set xrange [0:450]
set yrange [0:]

**2. Le bon ajustement **

Pour cela, nous n’aurons qu’à taper les commandes :

f(x) = m * x + q
fit f(x) 'house_price.dat' via m, q

3. Enregistrement des valeurs m et q dans une chaîne et traçage

Ici, nous utilisons la fonction sprintf pour préparer l’étiquette (encadrée dans le rectangle de l'objet) dans laquelle nous allons imprimer le résultat de l’ajustement. Enfin, nous traçons l’ensemble du graphique.

mq_value = sprintf("Parameters values\nm = %f k$/m^2\nq = %f k$", m, q)
set object 1 rect from 90,725 to 200, 650 fc rgb "white" 
set label 1 at 100,700 mq_value


p 'house_price.dat' ls 1 t 'House price', f(x) ls 2 t 'Linear regression'
set out

La sortie ressemblera à ceci.
