Syntaxe #

  • images docker [OPTIONS] [DÉPÔT[:TAG]]
  • docker inspect [OPTIONS] CONTENEUR|IMAGE [CONTENANT|IMAGE…]
  • docker pull [OPTIONS] NOM[:TAG|@DIGEST]
  • docker rmi [OPTIONS] IMAGE [IMAGE…]
  • balise docker [OPTIONS] IMAGE[:TAG] [REGISTRYHOST/][USERNAME/]NAME[:TAG]

Liste des images téléchargées localement #

$ docker images
REPOSITORY          TAG                 IMAGE ID            CREATED             SIZE
hello-world         latest              693bce725149        6 days ago          967 B
postgres            9.5                 0f3af79d8673        10 weeks ago        265.7 MB
postgres            latest              0f3af79d8673        10 weeks ago        265.7 MB

Récupération d’une image depuis Docker Hub #

Habituellement, les images sont extraites automatiquement de Docker Hub. Docker tentera d’extraire toute image de Docker Hub qui n’existe pas déjà sur l’hôte Docker. Par exemple, l’utilisation de docker run ubuntu lorsque l’image ubuntu n’est pas déjà sur l’hôte Docker obligera Docker à lancer une extraction de la dernière image ubuntu. Il est possible d’extraire une image séparément en utilisant “docker pull” pour récupérer ou mettre à jour manuellement une image à partir de Docker Hub.

docker pull ubuntu
docker pull ubuntu:14.04

Des options supplémentaires pour extraire d’un registre d’images différent ou extraire une version spécifique d’une image existent. L’indication d’un autre registre se fait à l’aide du nom complet de l’image et de la version facultative. Par exemple, la commande suivante tentera d’extraire l’image ubuntu:14.04 du registre registry.example.com :

docker pull registry.example.com/username/ubuntu:14.04

Images de référence #

Les commandes Docker qui prennent le nom d’une image acceptent quatre formes différentes :

Taper Exemple
Identifiant court 693bce725149
Nom hello-world (par défaut la balise :latest)
Nom+tag hello-world:dernier
Résumé hello-world@sha256:e52be8ffeeb1f374f440893189cd32f44cb166650e7ab185fa7735b7dc48d619

Remarque : Vous ne pouvez faire référence à une image par son résumé que si cette image a été extraite à l’origine à l’aide de ce résumé. Pour voir le résumé d’une image (si disponible), lancez docker images --digests.

Suppression d’images #

La commande docker rmi est utilisée pour supprimer des images :

docker rmi <image name>

Le nom complet de l’image doit être utilisé pour supprimer une image. À moins que l’image n’ait été balisée pour supprimer le nom du registre, il doit être spécifié. Par exemple:

docker rmi registry.example.com/username/myAppImage:1.3.5

Il est également possible de supprimer des images par leur ID à la place :

docker rmi 693bce725149

Pour plus de commodité, il est possible de supprimer des images par leur ID d’image en spécifiant uniquement les premiers caractères de l’ID d’image, tant que la sous-chaîne spécifiée n’est pas ambiguë :

docker rmi 693

Remarque : Les images peuvent être supprimées même si des conteneurs existants utilisent cette image ; docker rmi “démarque” simplement l’image.

Si aucun conteneur n’utilise une image, celle-ci est récupérée. Si un conteneur utilise une image, l’image sera ramassée une fois que tous les conteneurs l’utilisant seront supprimés. Par exemple:

$ docker ps -a
CONTAINER ID        IMAGE               COMMAND             CREATED                  STATUS                     PORTS               NAMES
5483657ee07b        hello-world         "/hello"            Less than a second ago   Exited (0) 2 seconds ago                       small_elion

$ docker rmi hello-world
Untagged: hello-world:latest

$ docker ps -a
CONTAINER ID        IMAGE               COMMAND             CREATED                  STATUS                      PORTS               NAMES
5483657ee07b        693bce725149        "/hello"            Less than a second ago   Exited (0) 12 seconds ago                       small_elion

Supprimer toutes les images sans conteneurs démarrés

Pour supprimer toutes les images locales qui n’ont pas de conteneurs démarrés, vous pouvez fournir une liste des images en tant que paramètre :

docker rmi $(docker images -qa)

Supprimer toutes les images

Si vous souhaitez supprimer des images, qu’elles aient ou non un conteneur démarré, utilisez l’indicateur de force (-f):

docker rmi -f $(docker images -qa)

Supprimer les images pendantes

Si une image n’est pas taguée et n’est utilisée par aucun conteneur, elle est " pendante " et peut être supprimée comme ceci :

docker images -q --no-trunc -f dangling=true | xargs -r docker rmi

Rechercher des images dans Docker Hub #

Vous pouvez rechercher des images dans Docker Hub à l’aide de la commande search :

docker search <term>

Par exemple:

$ docker search nginx
NAME                      DESCRIPTION                                     STARS     OFFICIAL   AUTOMATED

jwilder/nginx-proxy       Automated Nginx reverse proxy for docker c...   717                  [OK]

## Inspection des images

docker inspect <image>

La sortie est au format JSON. Vous pouvez utiliser l’utilitaire de ligne de commande jq pour analyser et imprimer uniquement les clés souhaitées.

docker inspect <image> | jq -r '.[0].Author'

La commande ci-dessus affichera le nom de l’auteur des images.

Marquage d’images #

Le balisage d’une image est utile pour garder une trace des différentes versions de l’image :

docker tag ubuntu:latest registry.example.com/username/ubuntu:latest

Autre exemple de marquage :

docker tag myApp:1.4.2 myApp:latest
docker tag myApp:1.4.2 registry.example.com/company/myApp:1.4.2

Enregistrer et charger des images Docker #

docker save -o ubuntu.latest.tar ubuntu:latest

Cette commande enregistrera l’image ubuntu:latest en tant qu’archive tarball dans le répertoire courant avec le nom ubuntu.latest.tar. Cette archive tarball peut ensuite être déplacée vers un autre hôte, par exemple en utilisant rsync, ou archivée dans le stockage.

Une fois l’archive tar déplacée, la commande suivante créera une image à partir du fichier :

docker load -i /tmp/ubuntu.latest.tar

Il est maintenant possible de créer des conteneurs à partir de l’image ubuntu:latest comme d’habitude.
