Registre privésécurisé Docker avec API v2

Un registre Docker privé et sécurisé au lieu d’un Docker Hub. Des compétences de base en docker sont requises.

Paramètres #

Commande Explication
sudo docker run -p 5000:5000 Démarrez un conteneur docker et liez le port 5000 du conteneur au port 5000 de la machine physique.
–registre de noms Nom du conteneur (utilisé pour améliorer la lisibilité de “docker ps”).
-v ‘pwd’/certs:/certs Liez CURRENT_DIR/certs de la machine physique sur /certs du conteneur (comme un “dossier partagé”).
-e REGISTRY_HTTP_TLS_CERTIFICATE=/certs/server.crt Nous précisons que le registre doit utiliser le fichier /certs/server.crt pour démarrer. (variable d’environnement)
-e REGISTRY_HTTP_TLS_KEY=/certs/server.key Idem pour la clé RSA (server.key).
-v /root/images:/var/lib/registry/ Si vous souhaitez enregistrer toutes vos images de registre, vous devez le faire sur la machine physique. Ici, nous sauvegardons toutes les images sur /root/images sur la machine physique. Si vous faites cela, vous pouvez arrêter et redémarrer le registre sans perdre aucune image.
registre :2 On précise qu’on aimerait extraire l’image du registre du docker hub (ou localement), et on ajoute « 2 » car on veut installer la version 2 du registre.

[Comment installer un docker-engine (appelé client sur ce tutoriel)][1]

[Comment générer un certificat auto-signé SSL][2]

[1] : https://docs.docker.com/engine/installation/ [2] : http://www.akadia.com/services/ssh_test_certificate.html

Génération de certificats #

Générer une clé privée RSA : openssl genrsa -des3 -out server.key 4096

Openssl devrait demander une phrase de passe à cette étape. Notez que nous n’utiliserons que le certificat pour la communication et l’authentification, sans phrase de passe. Utilisez simplement 123456 par exemple.

Générez la demande de signature de certificat : openssl req -new -key server.key -out server.csr

Cette étape est importante car il vous sera demandé des informations sur les certificats. L’information la plus importante est le « nom commun », c’est-à-dire le nom de domaine, qui doit être utilisé pour la communication entre le registre Docker privé et toutes les autres machines. Exemple : mondomaine.com

Supprimer la phrase de passe de la clé privée RSA : cp server.key server.key.org && openssl rsa -in server.key.org -out server.key

Comme je l’ai dit, nous allons nous concentrer sur le certificat sans phrase de passe. Soyez donc prudent avec tous les fichiers de votre clé (.key,.csr,.crt) et conservez-les en lieu sûr.

Générer le certificat auto-signé : openssl x509 -req -days 365 -in server.csr -signkey server.key -out server.crt

Vous avez maintenant deux fichiers essentiels, server.key et server.crt, nécessaires à l’authentification du registre privé.

Exécuter le registre avec un certificat auto-signé #

Pour exécuter le registre privé (en toute sécurité), vous devez générer un certificat auto-signé, vous pouvez vous référer à l’exemple précédent pour le générer.

Pour mon exemple, j’ai mis server.key et server.crt dans /root/certs

Avant d’exécuter la commande docker, vous devez être placé (utilisez cd) dans le répertoire contenant le dossier * certs *. Si ce n’est pas le cas et que vous essayez d’exécuter la commande, vous recevrez une erreur du type

level=fatal msg=“open /certs/server.crt : aucun fichier ou répertoire de ce type”

Lorsque vous êtes (cd /root dans mon exemple), vous pouvez essentiellement démarrer le registre sécurisé/privé en utilisant : sudo docker run -p 5000:5000 –restart=always –name register -v &#96 ;pwd`/certs:/certs -e REGISTRY_HTTP_TLS_CERTIFICATE=/certs/server.crt -e REGISTRY_HTTP_TLS_KEY=/certs/server.key -v /root/Documents:/var/lib/registry/registry:2
Des explications sur la commande sont disponibles dans la partie Paramètres.

Tirer ou pousser depuis un client docker #

Lorsque vous obtenez un registre de travail en cours d’exécution, vous pouvez extraire ou pousser des images dessus. Pour cela, vous avez besoin du fichier server.crt dans un dossier spécial sur votre client docker. Le certificat vous permet de vous authentifier auprès du registre, puis de chiffrer la communication.

Copiez server.crt de la machine de registre dans /etc/docker/certs.d/mydomain.com:5000/ sur votre machine cliente. Et renommez-le ensuite en ca-certificates.crt : mv /etc/docker/certs.d/mydomain.com:5000/server.crt /etc/docker/certs.d/mydomain.com:5000/ca- certificats.crt

À ce stade, vous pouvez extraire ou transférer des images depuis votre registre privé :
PULL : docker pull mydomain.com:5000/nginx ou
APPUYEZ :

  1. Obtenez une image officielle de hub.docker.com : docker pull nginx
  2. Taguez cette image avant de la pousser dans le registre privé : docker tag IMAGE_ID mydomain.com:5000/nginx (utilisez docker images pour obtenir l’IMAGE_ID)
  3. Poussez l’image vers le registre : docker push mydomain.com:5000/nginx
