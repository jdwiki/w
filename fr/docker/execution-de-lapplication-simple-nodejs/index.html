Exécution d’une application Basic Node.js dans un conteneur #

L’exemple dont je vais parler suppose que vous disposez d’une installation Docker qui fonctionne sur votre système et d’une compréhension de base de l’utilisation de Node.js . Si vous savez comment vous devez travailler avec Docker , il devrait être évident que le framework Node.js n’a pas besoin d’être installé sur votre système, nous utiliserons plutôt la “dernière” version de l’image “node” disponible sur Docker. Par conséquent, si nécessaire, vous pouvez télécharger l’image au préalable avec la commande “docker pull node”. (La commande “extrait” automatiquement la dernière version de l’image “nœud” de docker.)

  1. Procédez à la création d’un répertoire où résideraient tous vos fichiers d’application de travail. Créez un fichier package.json dans ce répertoire qui décrit votre application ainsi que les dépendances. Votre fichier package.json devrait ressembler à ceci :

     {
   "name": "docker_web_app",
   "version": "1.0.0",
   "description": "Node.js on Docker",
   "author": "First Last <first.last@example.com>",
   "main": "server.js",
   "scripts": {
     "start": "node server.js"
   },
   "dependencies": {
     "express": "^4.13.3"
   }
 }

  1. Si nous devons travailler avec Node.js, nous créons généralement un fichier server qui définit une application Web. Dans ce cas, nous utilisons le framework Express.js (à partir de la version 4.13.3). Un fichier server.js de base ressemblerait à ceci :

     var express = require('express');
 var PORT = 8080;
 var app = express();
 app.get('/', function (req, res) {
   res.send('Hello world\n');
 });

 app.listen(PORT);
 console.log('Running on http://localhost:' + PORT);

  2. Pour ceux qui connaissent Docker, vous auriez rencontré un Dockerfile. Un Dockerfile est un fichier texte qui contient toutes les commandes nécessaires pour créer une image personnalisée adaptée à votre application.

Créez un fichier texte vide nommé “Dockerfile” dans le répertoire courant. La méthode pour en créer un est simple dans Windows. Sous Linux, vous pouvez exécuter touch Dockerfile dans le répertoire contenant tous les fichiers nécessaires à votre application. Ouvrez le Dockerfile avec n’importe quel éditeur de texte et ajoutez les lignes suivantes :

FROM node:latest
RUN mkdir -p /usr/src/my_first_app
WORKDIR /usr/src/my_first_app
COPY package.json /usr/src/my_first_app/
RUN npm install
COPY . /usr/src/my_first_app
EXPOSE 8080

  • FROM node:latest indique au démon Docker à partir de quelle image nous voulons construire. Dans ce cas, nous utilisons la “dernière” version du “nœud” d’image Docker officiel disponible sur le [Docker Hub][1].

  • Dans cette image, nous procédons à la création d’un répertoire de travail contenant tous les fichiers requis et nous demandons au démon de définir ce répertoire comme répertoire de travail souhaité pour notre application. Pour cela nous ajoutons

      RUN mkdir -p /usr/src/my_first_app
  WORKDIR /usr/src/my_first_app

  • Nous procédons ensuite à l’installation des dépendances de l’application en déplaçant d’abord le fichier package.json (qui spécifie les informations sur l’application, y compris les dépendances) vers le répertoire de travail /usr/src/my_first_app dans l’image. Nous faisons cela en

      COPY package.json /usr/src/my_first_app/
  RUN npm install

  • Nous tapons ensuite COPY . /usr/src/my_first_app pour ajouter tous les fichiers d’application et le code source au répertoire de travail dans l’image.

  • Nous utilisons ensuite la directive EXPOSE pour demander au démon de rendre visible le port 8080 du conteneur résultant (via un mappage conteneur-hôte) puisque l’application se lie au port 8080.

  • Dans la dernière étape, nous demandons au démon d’exécuter la commande node server.js à l’intérieur de l’image en exécutant la commande de base npm start. Nous utilisons pour cela la directive CMD, qui prend les commandes comme arguments.

      CMD [ "npm", "start" ]

  1. Nous créons ensuite un fichier .dockerignore dans le même répertoire que le Dockerfile pour empêcher notre copie de node_modules et les journaux utilisés par notre installation système Node.js d’être copiés sur l’image Docker. Le fichier .dockerignore doit avoir le contenu suivant :

     node_modules
 npm-debug.log

  2. Construisez votre image #

Accédez au répertoire contenant le “Dockerfile” et exécutez la commande suivante pour créer l’image Docker. Le drapeau -t vous permet de baliser votre image afin qu’elle soit plus facile à retrouver plus tard à l’aide de la commande docker images :

    $ docker build -t <your username>/node-web-app .

Votre image sera maintenant répertoriée par Docker. Affichez les images à l’aide de la commande ci-dessous :

$ docker images

REPOSITORY                      TAG        ID              CREATED
node                            latest     539c0211cd76    10 minutes ago
<your username>/node-web-app    latest     d64d3505b0d2    1 minute ago

  1. Exécution de l’image #

Nous pouvons maintenant exécuter l’image que nous venons de créer en utilisant le contenu de l’application, l’image de base node et le Dockerfile. Nous procédons maintenant à l’exécution de notre image <votre nom d'utilisateur>/node-web-app nouvellement créée. Fournir le commutateur -d à la commande docker run exécute le conteneur en mode détaché, de sorte que le conteneur s’exécute en arrière-plan. Le drapeau -p redirige un port public vers un port privé à l’intérieur du conteneur. Exécutez l’image que vous avez précédemment créée à l’aide de cette commande :

$ docker run -p 49160:8080 -d <your username>/node-web-app

  1. Imprimez la sortie de votre application en exécutant “docker ps” sur votre terminal. La sortie devrait ressembler à ceci.

     CONTAINER ID        IMAGE                         COMMAND             CREATED             STATUS              PORTS                     NAMES
 7b701693b294      <your username>/node-web-app   "npm start"         20 minutes ago       Up 48 seconds       0.0.0.0:49160->8080/tcp   loving_goldstine

Obtenez la sortie de l’application en saisissant docker logs <CONTAINER ID>. Dans ce cas, il s’agit de docker logs 7b701693b294.

Sortie : En cours d'exécution sur http://localhost:8080

  1. À partir de la sortie docker ps, le mappage de port obtenu est 0.0.0.0:49160->8080/tcp. Par conséquent, Docker a mappé le port 8080 à l’intérieur du conteneur sur le port 49160 sur la machine hôte. Dans le navigateur, nous pouvons maintenant entrer localhost:49160.

Nous pouvons également appeler notre application en utilisant curl :

$ curl -i localhost:49160

HTTP/1.1 200 OK
X-Powered-By: Express
Content-Type: text/html; charset=utf-8
Content-Length: 12
Date: Sun, 08 Jan 2017 14:00:12 GMT
Connection: keep-alive

Hello world
