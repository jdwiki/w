Syntaxe #

  • docker inspect [OPTIONS] CONTENEUR|IMAGE [CONTENANT|IMAGE…]

Obtenir des informations spécifiques d’un conteneur #

Vous pouvez obtenir une information spécifique à partir d’un conteneur en exécutant :

docker inspect -f '<format>' <container>

Par exemple, vous pouvez obtenir les paramètres réseau en exécutant :

docker inspect -f '{{ .NetworkSettings }}' <container>

Vous pouvez également obtenir uniquement l’adresse IP :

docker inspect -f '{{ .NetworkSettings.IPAddress }}' <container>

Le paramètre -f signifie format et recevra un Go Template en entrée pour formater ce qui est attendu, mais cela n’apportera pas un beau retour, alors essayez :

docker inspect -f '{{ json .NetworkSettings }}' {{containerIdOrName}}

le mot-clé json apportera le retour sous forme de JSON.

Donc pour finir, une petite astuce consiste à utiliser python pour formater la sortie JSON :

docker inspect -f '{{ json .NetworkSettings }}' <container> | python -mjson.tool

Et le tour est joué, vous pouvez interroger n’importe quoi sur le docker inspect et le rendre joli dans votre terminal.

Il est également possible d’utiliser un utilitaire appelé “jq” afin d’aider à traiter la sortie de la commande docker inspect.

docker inspect -f '{{ json .NetworkSettings }}' aa1 | jq [.Gateway]

La commande ci-dessus renverra la sortie suivante :

[
  "172.17.0.1"
]

Cette sortie est en fait une liste contenant un élément. Parfois, docker inspect affiche une liste de plusieurs éléments, et vous pouvez faire référence à un élément spécifique. Par exemple, si Config.Env contient plusieurs éléments, vous pouvez vous référer au premier élément de cette liste en utilisant index :

docker inspect --format '{{ index (index .Config.Env) 0 }}' <container>

Le premier élément est indexé à zéro, ce qui signifie que le second élément de cette liste est à l’index ‘1’ :

docker inspect --format '{{ index (index .Config.Env) 1 }}' <container>

En utilisant len il est possible d’obtenir le nombre d’éléments de la liste :

docker inspect --format ‘{{ len .Config.Env }}’ <container>

Et en utilisant des nombres négatifs, il est possible de se référer au dernier élément de la liste :

docker inspect –format “{{ index .Config.Cmd $[$(docker inspect –format ‘{{ len .Config.Cmd }}’ <container>)-1]}}” <container>

Certaines informations docker inspect se présentent sous la forme d’un dictionnaire de clé: valeur, voici un extrait d’un docker inspectd’un conteneur en cours d’exécution jess/spotify

"Configuration": { "Hostname": "8255f4804dde", "Domainname": "", "User": "spotify", "AttachStdin": false, "AttachStdout": false, "AttachStderr": false, "Tty": false, "OpenStdin": false, "StdinOnce": false, "Env": [ "DISPLAY=unix:0", "PATH=/usr/local/sbin:/usr/local/bin:/usr/sbin:/usr/bin:/sbin:/bin", "HOME=/home/spotify" ], "Cmd": [ "-stylesheet=/home/spotify/spotify-override.css" ], "Image": "jess/spotify", "Volumes": null, "WorkingDir": "/home/spotify", "Entrypoint": [ "spotify" ], "OnBuild": null, "Labels": {} },

donc j’obtiens les valeurs de toute la section Config

docker inspecter -f '{{.Config}}' 825

{8255f4804dde spotify faux faux faux carte[] faux faux faux [DISPLAY=unix:0 PATH=/usr/local/sbin:/usr/local/bin:/usr/sbin:/usr/bin:/sbin:/bin HOME=/home/spotify] [-stylesheet=/home/spotify/spotify-override.css] false jess/spotify map[] /home/spotify [spotify] false [] map[] }

mais aussi un seul champ, comme la valeur de Config.Image

docker inspect -f '{{index (.Config) "Image" }}' 825

jess/spotify

ou Config.Cmd

docker inspect -f '{{.Config.Cmd}}' 825

[-stylesheet=/home/spotify/spotify-override.css]

Obtenir des informations sur le conteneur #

Pour obtenir toutes les informations d’un conteneur, vous pouvez exécuter :

docker inspect <container>

Déboguer les journaux du conteneur à l’aide de docker inspect #

Examen de stdout/stderr d’un conteneur en cours d’exécution #

docker logs --follow <containerid>

Cela termine la sortie du conteneur en cours d’exécution. Ceci est utile si vous n’avez pas configuré de pilote de journalisation sur le démon docker.
