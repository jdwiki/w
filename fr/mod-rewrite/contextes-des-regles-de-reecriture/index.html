Contextes des règles de réécriture

  • Réécrire les règles dans le contexte de l’hôte virtuel

Cette rubrique décrit les deux contextes dans lesquels RewriteRule peut être utilisé. Dans les exemples omettant RewriteEngine on, il est supposé que cette directive s’est produite avant cet exemple.

Réécrire les règles dans un contexte par répertoire #

Le contexte par répertoire fait partie du fichier de configuration statique entre les balises <Directory> et </Directory>. L’intégralité du contenu des fichiers de configuration dynamiques se trouve dans le contexte par répertoire du dossier dans lequel réside le .htaccess.

Les RewriteRule dans le contexte de chaque répertoire correspondent à la partie d’une URL après le protocole, le nom d’hôte, le port et le préfixe du répertoire dans lequel ils résident, et avant la chaîne de requête.

Dans le fichier de configuration statique

Lorsque la règle suivante est utilisée sur l’url http://example.com/foo?id=1, la regex dans le premier argument de RewriteRule est comparée à foo. Le protocole (http), le nom d’hôte (example.com) et le préfixe de ce répertoire (/) sont supprimés. À l’autre extrémité, la chaîne de requête (?id=1) est également supprimée.

<Directory "/">
  RewriteRule ^foo$ bar [L]
</Directory>

Dans l’exemple suivant, en utilisant l’URL http://example.com/topic/15-my-topic-name, le premier argument de RewriteRule serait mis en correspondance avec topic/15-my-topic-name :

<Directory "/topic/">
  RewriteRule ^topic/([0-9]+)-[^/]*/?$ topics.php?id=$1 [L]
</Directory>

Dans le fichier de configuration dynamique

Lorsque la règle suivante est placée dans un fichier .htaccess qui se trouve dans le dossier www-root, puis utilisée sur l’url http://example.com/foo?id=1, la première regex est comparée à ` fou'.

RewriteRule ^foo$ bar [L]

Dans un contexte par répertoire, l’url correspondante jamais commence par un /. Dans un tel contexte, une directive commençant par RewriteRule ^/ ne correspondra jamais à rien.

Réécrire les règles dans le contexte de l’hôte virtuel #

Le contexte de l’hôte virtuel fait partie du fichier de configuration statique entre les balises <VirtualHost> et </VirtualHost>.

RewriteRule dans le contexte de l’hôte virtuel correspond à la partie d’une URL après le protocole, le nom d’hôte et le port, et avant la chaîne de requête.

Lorsque la règle suivante est utilisée pour l’URL http://example.com/foo?id=1, l’expression régulière dans le premier argument de RewriteRule est comparée à /foo.

<VirtualHost 1.2.3.4:80>
  ServerName example.com

  RewriteEngine on
  RewriteRule ^/foo$ /bar [L]
</VirtualHost>
