Syntaxe #

  • dimensionnement de la boîte : paramètre ;

Paramètres #

Paramètre Détail
content-box La largeur et la hauteur de l’élément n’incluent que la zone de contenu.
padding-box La largeur et la hauteur de l’élément incluent le contenu et le rembourrage.
border-box La largeur et la hauteur de l’élément incluent le contenu, le rembourrage et la bordure.
initial Définit le modèle de boîte à son état par défaut.
inherit Hérite du modèle de boîte de l’élément parent.

À propos de la boîte de rembourrage #

Cette valeur n’a été implémentée que par Firefox et ne doit donc pas être utilisée. Il a été supprimé dans Firefox version 50.0.

Qu’est-ce que le modèle Box ? #

Les bords #

Le navigateur crée un rectangle pour chaque élément du document HTML. Le modèle de boîte décrit comment le remplissage, la bordure et la marge sont ajoutés au contenu pour créer ce rectangle.

![Modèle de boîte CSS][2]

Diagramme de [CSS2.2 Working Draft][1]

Le périmètre de chacune des quatre zones est appelé un bord. Chaque bord définit une boîte.

  • Le rectangle le plus à l’intérieur est la zone de contenu. La largeur et la hauteur de celui-ci dépendent du contenu rendu de l’élément (texte, images et tout élément enfant qu’il peut avoir).
  • Vient ensuite la boîte de remplissage, telle que définie par la propriété [padding][6]. S’il n’y a pas de largeur de padding définie, le bord de remplissage est égal au bord du contenu.
  • Ensuite, nous avons la boîte de bordure, telle que définie par la propriété [border][7]. S’il n’y a pas de largeur de border définie, le bord de la bordure est égal au bord du remplissage.
  • Le rectangle le plus à l’extérieur est la boîte de marge, telle que définie par la propriété [margin][8]. Si aucune largeur de “marge” n’est définie, le bord de la marge est égal au bord de la bordure.

Exemple #

div {
    border: 5px solid red;
    margin: 50px;
    padding: 20px;
}

Ce CSS stylise tous les éléments div pour qu’ils aient une bordure supérieure, droite, inférieure et gauche de 5px de largeur ; une marge supérieure, droite, inférieure et gauche de 50px ; et un rembourrage en haut, à droite, en bas et à gauche de 20px. En ignorant le contenu, notre boîte générée ressemblera à ceci :

![Modèle de boîte pour l’exemple ci-dessus][3]

Capture d’écran du panneau Styles d’éléments de Google Chrome

  • Comme il n’y a pas de contenu, la zone de contenu (la boîte bleue au milieu) n’a ni hauteur ni largeur (0px par 0px).
  • La zone de rembourrage par défaut a la même taille que la zone de contenu, plus la largeur de 20 pixels sur les quatre bords que nous définissons ci-dessus avec la propriété padding (40px sur 40px).
  • La boîte de bordure a la même taille que la boîte de remplissage, plus la largeur de 5 pixels que nous définissons ci-dessus avec la propriété border (50px par 50px).
  • Enfin, la boîte de marge a la même taille que la boîte de bordure, plus la largeur de 50 pixels que nous définissons ci-dessus avec la propriété margin (donnant à notre élément une taille totale de 150px sur 150px).

Donnons maintenant à notre élément un frère avec le même style. Le navigateur examine le modèle de boîte des deux éléments pour déterminer où, par rapport au contenu de l’élément précédent, le nouvel élément doit être positionné :

![Deux éléments identiques côte à côte][4]

Le contenu de chacun des éléments est séparé par un espace de 150 pixels, mais les cases des deux éléments se touchent.

Si nous modifions ensuite notre premier élément pour qu’il n’ait pas de marge droite, le bord de la marge droite serait dans la même position que le bord de la bordure droite, et nos deux éléments ressembleraient maintenant à ceci :

![Premier élément sans marge droite][5]

dimensionnement de la boîte #

Le modèle de boîte par défaut (content-box) peut être contre-intuitif, car la width / height d’un élément ne représentera pas sa largeur ou sa hauteur réelle à l’écran dès que vous commencerez à ajouter stylespaddingetborder`` à l’élément.

L’exemple suivant illustre ce problème potentiel avec content-box :

textarea {
    width: 100%;
    padding: 3px;
    box-sizing: content-box; /* default value */
}

Étant donné que le rembourrage sera ajouté à la largeur de la zone de texte, l’élément résultant est une zone de texte plus large que 100 %.

Heureusement, CSS nous permet de changer le modèle de boîte avec la propriété box-sizing pour un élément. Trois valeurs différentes sont disponibles pour la propriété :

  • content-box : le modèle de boîte commun - la largeur et la hauteur n’incluent que le contenu, pas le rembourrage ou la bordure

  • padding-box : la largeur et la hauteur incluent le contenu et le rembourrage, mais pas la bordure

  • border-box : La largeur et la hauteur incluent le contenu, le rembourrage ainsi que la bordure

[![entrez la description de l’image ici][1]][1]

Pour résoudre le problème de textarea ci-dessus, vous pouvez simplement changer la propriété box-sizing en padding-box ou border-box. border-box est le plus couramment utilisé.

textarea {
    width: 100%;
    padding: 3px;
    box-sizing: border-box;
}

Pour appliquer un modèle de boîte spécifique à chaque élément de la page, utilisez l’extrait de code suivant :

html {
    box-sizing: border-box;
}

*, *:before, *:after {
    box-sizing: inherit;
}

Dans ce codage, box-sizing:border-box; n’est pas directement appliqué à *, vous pouvez donc facilement écraser cette propriété sur des éléments individuels.
