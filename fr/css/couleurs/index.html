Syntaxe #

  • couleur : #rgb
  • couleur : #rrggbb
  • couleur : rgb[a](<rouge>, <vert>, <bleu>[, <alpha>])
  • couleur : hsl[a](<hue>, <saturation%>, <lightness%>[, <alpha>])
  • couleur : [colorkeyword](https://www.wikiod.com/fr/css/couleurs#Mots clés de couleur) /* vert, bleu, jaune, orange, rouge, ..etc */

couleurcourante #

currentColor renvoie la valeur de couleur calculée de l’élément actuel.

Utiliser dans le même élément #

Ici, currentColor est évalué en rouge puisque la propriété color est définie sur red :

div {
   color: red;     
   border: 5px solid currentColor;
   box-shadow: 0 0 5px currentColor;
}

Dans ce cas, spécifier currentColor pour la bordure est très probablement redondant car l’omettre devrait produire des résultats identiques. Utilisez uniquement currentColor à l’intérieur de la propriété border dans le même élément s’il serait écrasé autrement en raison d’un [plus spécifique] (https://www.wikiod.com/fr/css/cascade-et-specificite#Calcul de la spécificité du sélecteur -specificity#t=201607221033325129018) sélecteur.

Puisqu’il s’agit de la couleur calculée, la bordure sera verte dans l’exemple suivant car la deuxième règle remplace la première :

div {
   color: blue;
   border: 3px solid currentColor; 
   color: green;
}

Hérité de l’élément parent #

La couleur du parent est héritée, ici currentColor est évalué à ‘blue’, ce qui rend la couleur de bordure de l’élément enfant bleue.

.parent-class {
    color: blue;
}

.parent-class .child-class {
    border-color: currentColor;
}

currentColor peut également être utilisé par d’autres règles qui normalement n’hériteraient pas de la propriété color, comme background-color. L’exemple ci-dessous montre les enfants utilisant la couleur définie dans le parent comme arrière-plan :

.parent-class {
    color: blue;
}

.parent-class .child-class {
    background-color: currentColor;
}

Mots clés de couleur #

La plupart des navigateurs prennent en charge l’utilisation de mots-clés de couleur pour spécifier une couleur. Par exemple, pour définir la couleur d’un élément sur bleu, utilisez le mot-clé blue :

.some-class {
    color: blue;
}

Les mots-clés CSS ne sont pas sensibles à la casse : ‘blue’, ‘Blue’ et ‘BLUE’ donneront tous ‘#0000FF’.

Valeur hexadécimale #

Arrière plan #

Les couleurs CSS peuvent également être représentées sous la forme d’un triplet hexadécimal, où les membres représentent les composants rouge, vert et bleu d’une couleur. Chacune de ces valeurs représente un nombre compris entre « 00 » et « FF », ou entre « 0 » et « 255 » en notation décimale. Des valeurs hexadécimales majuscules et/ou minuscules peuvent être utilisées (c’est-à-dire #3fc = #3FC = #33ffCC). Le navigateur interprète #369 comme #336699. Si ce n’est pas ce que vous vouliez mais que vous vouliez plutôt “#306090”, vous devez le spécifier explicitement.

Le nombre total de couleurs pouvant être représentées avec la notation hexadécimale est de 256 ^ 3 ou 16 777 216.

Syntaxe #

color: #rrggbb;
color: #rgb
Valeur Descriptif
rr 00 - FF pour la quantité de rouge
gg 00 - FF pour la quantité de vert
bb 00 - FF pour la quantité de bleu 
.some-class {
    /* This is equivalent to using the color keyword 'blue' */
    color: #0000FF;
}

.also-blue {
    /* If you want to specify each range value with a single number, you can!
       This is equivalent to '#0000FF' (and 'blue') */
    color: #00F;
}

La notation hexadécimale est utilisée pour spécifier les valeurs de couleur dans le format de couleur RVB, conformément aux [‘Valeurs de couleur numériques’ du W3C](https://www.w3. org/TR/css3-color/#numerical).

Il existe de nombreux outils disponibles sur Internet pour rechercher des valeurs de couleur hexadécimales (ou simplement hexadécimales).

Recherchez « palette de couleurs hexadécimale » ou « sélecteur de couleurs hexadécimale » avec votre navigateur Web préféré pour trouver un tas d’options !

Les valeurs hexadécimales commencent toujours par un signe dièse (#), comportent jusqu’à six “chiffres” et ne sont pas sensibles à la casse : c’est-à-dire qu’elles ne se soucient pas des majuscules. #FFC125 et #ffc125 sont de la même couleur.

rgb() Notation #

RVB est un modèle de couleur additif qui représente les couleurs comme des mélanges de lumière rouge, verte et bleue. Essentiellement, la représentation RVB est l’équivalent décimal de la notation hexadécimale. En hexadécimal, chaque nombre varie de 00 à FF, ce qui équivaut à 0 à 255 en décimal et à 0 % à 100 % en pourcentage.

.some-class {
    /* Scalar RGB, equivalent to 'blue'*/
    color: rgb(0, 0, 255);
}

.also-blue {
    /* Percentile RGB values*/
    color: rgb(0%, 0%, 100%);
}

Syntaxe #

rgb(<red>, <green>, <blue>)
Valeur Descriptif
<rouge> un entier de 0 à 255 ou un pourcentage de 0 à 100 %
<vert> un entier de 0 à 255 ou un pourcentage de 0 à 100 %
<bleu> un entier de 0 à 255 ou un pourcentage de 0 à 100 %

rgba() Notation #

Similaire à la [notation rgb()](https://www.wikiod.com/fr/css/couleurs#Notation RVB()), mais avec une valeur alpha (opacité) supplémentaire.

.red {
    /* Opaque red */
    color: rgba(255, 0, 0, 1);
}

.red-50p {
    /* Half-translucent red. */
    color: rgba(255, 0, 0, .5);
}

Syntaxe #

rgba(<red>, <green>, <blue>, <alpha>);
Valeur Descriptif
<rouge> un entier de 0 à 255 ou un pourcentage de 0 à 100 %
<vert> un entier de 0 à 255 ou un pourcentage de 0 à 100 %
<bleu> un entier de 0 à 255 ou un pourcentage de 0 à 100 %
<alpha> un nombre compris entre 0 et 1, où 0,0 est entièrement transparent et 1,0 est entièrement opaque

hsl() Notation #

HSL signifie teinte (“quelle couleur”), saturation (“combien de couleur”) et luminosité (“combien de blanc”).

La teinte est représentée sous la forme d’un angle de 0° à 360° (sans unités), tandis que la saturation et la luminosité sont représentées sous forme de pourcentages.

p {
    color: hsl(240, 100%, 50%); /* Blue */
}

[![La roue chromatique HSL][1]][1]

Syntaxe #

color: hsl(<hue>, <saturation>%, <lightness>%);
Valeur Descriptif
<teinte> spécifié en degrés autour de la roue chromatique (sans unités), où 0° est rouge, 60° est jaune, 120° est vert, 180° est cyan, 240° est bleu, 300° est magenta et 360° est rouge
<saturation> spécifié en pourcentage où 0 % est entièrement désaturé (niveaux de gris) et 100 % est entièrement saturé (couleurs vives)
<légèreté> spécifié en pourcentage où 0 % est entièrement noir et 100 % est entièrement blanc

Remarques #

  • Une saturation de 0 % produit toujours une couleur en niveaux de gris ; changer la teinte n’a aucun effet.
  • Une luminosité de 0 % produit toujours du noir et 100 % produit toujours du blanc ; changer la teinte ou la saturation n’a aucun effet.

[1] : https://upload.wikimedia.org/wikipedia/commons/c/cb/HSL_color_solid_cylinder_alpha_lowgamma.png

Notation hsla() #

Similaire à la [notation hsl()](https://www.wikiod.com/fr/css/couleurs#Notation hsl()), mais avec une valeur alpha (opacité) ajoutée.

hsla(240, 100%, 50%, 0)     /* transparent */
hsla(240, 100%, 50%, 0.5)   /* half-translucent blue */
hsla(240, 100%, 50%, 1)     /* fully opaque blue */

Syntaxe #

hsla(<hue>, <saturation>%, <lightness>%, <alpha>);
Valeur Descriptif
<teinte> spécifié en degrés autour de la roue chromatique (sans unités), où 0° est rouge, 60° est jaune, 120° est vert, 180° est cyan, 240° est bleu, 300° est magenta et 360° est rouge
<saturation> pourcentage où 0 % est entièrement désaturé (niveaux de gris) et 100 % est entièrement saturé (couleurs vives)
<légèreté> pourcentage où 0 % est entièrement noir et 100 % est entièrement blanc
<alpha> un nombre de 0 à 1 où 0 est entièrement transparent et 1 est entièrement opaque
