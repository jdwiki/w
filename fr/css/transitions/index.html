Syntaxe #

  • transition : [transition-property] [transition-duration] [transition-timing-function] [transition-delay] ;

Paramètres #

Paramètre Détails
propriété de transition La propriété CSS spécifique dont le changement de valeur doit être transféré (ou) all, si toutes les propriétés transitoires doivent être transitionné.
durée de transition La durée (ou période) en secondes (s) ou millisecondes (ms) pendant laquelle la transition doit avoir lieu.
fonction de temporisation de transition Une fonction qui décrit comment les valeurs intermédiaires pendant la transition sont calculées. Les valeurs couramment utilisées sont ease, ease-in, ease-out, ease-in-out, linear, cubic-bezier(), steps(). Vous trouverez plus d’informations sur les différentes fonctions de synchronisation dans les spécifications W3C.
délai de transition Le temps qui doit s’être écoulé avant que la transition puisse commencer. Peut être spécifié en secondes (s) ou en millisecondes (ms)

Certains navigateurs plus anciens ne prennent en charge que les propriétés [vendor-prefixed][1] transition :

  • -webkit : Chrome 25-, Safari 6-, Safari et Chrome pour iOS 6.1-, Android 4.3- Navigateur, Blackberry Browser 7-, UC Browser 9.9- pour Android.
  • -moz : Firefox 15-.
  • -o : Opéra 11.5-, Opera Mobile 12-.

Exemple:

-webkit-transition: all 1s;
-moz-transition: all 1s;
-o-transition: all 1s;
transition: all 1s;

[1] : http://caniuse.com/#search=transitions

Raccourci de transition #

CSS

div{
    width: 150px;
    height:150px;
    background-color: red;
    transition: background-color 1s;
}
div:hover{
    background-color: green;
}

HTML

<div></div>

Cet exemple changera la couleur d’arrière-plan lorsque la div est survolée, le changement de couleur d’arrière-plan durera 1 seconde.

cube-bézier #

La fonction cubic-bezier est une fonction de synchronisation de transition qui est souvent utilisée pour des transitions personnalisées et fluides.

transition-timing-function: cubic-bezier(0.1, 0.7, 1.0, 0.1);

La fonction prend quatre paramètres :

cubique-bézier(P1_x, P1_y, P2_x, P2_y)

[![cubique-bézier][1]][1]

Ces paramètres seront mappés sur des points faisant partie d’une [courbe de Bézier][3] :

[![courbe-de-bézier][2]][2]

Pour les courbes de Bézier CSS, P0 et P3 sont toujours au même endroit. P0 est en (0,0) et P3 est en (1,1), ce qui signifie que les paramètres passés à la fonction cubique-bézier ne peuvent être qu’entre 0 et 1.

Si vous passez des paramètres qui ne sont pas dans cet intervalle, la fonction sera par défaut une transition linéaire.

Étant donné que cubic-bezier est la transition la plus flexible en CSS, vous pouvez traduire toutes les autres fonctions de synchronisation de transition en fonctions cubic-bezier :

linéaire : cubique-bézier(0,0,1,1)

ease-in : cubic-bezier(0.42, 0.0, 1.0, 1.0)

ease-out: cubic-bezier(0.0, 0.0, 0.58, 1.0)

ease-in-out : cubic-bezier(0.42, 0.0, 0.58, 1.0)

[1] : http://i.stack.imgur.com/4jlAK.png [2] : http://i.stack.imgur.com/FabHY.png [3] : https://en.wikipedia.org/wiki/B%C3%A9zier_curve#Cubic_B.C3.A9zier_curves

Transition (manuscrit) #

CSS #

div {
  height: 100px;
  width: 100px;
  border: 1px solid;
  transition-property: height, width;
  transition-duration: 1s, 500ms;
  transition-timing-function: linear;
  transition-delay: 0s, 1s;
}
div:hover {
  height: 200px;
  width: 200px;
}

HTML #

<div></div>
  • transition-property : spécifie les propriétés CSS auxquelles l’effet de transition est destiné. Dans ce cas, la div s’étendra à la fois horizontalement et verticalement lorsqu’elle est survolée.
  • transition-duration : spécifie la durée d’exécution d’une transition. Dans l’exemple ci-dessus, les transitions de hauteur et de largeur prendront respectivement 1 seconde et 500 millisecondes.
  • transition-timing-function : spécifie la courbe de vitesse de l’effet de transition. Une valeur linéaire indique que la transition aura la même vitesse du début à la fin.
  • transition-delay : spécifie le temps nécessaire pour attendre avant que l’effet de transition ne démarre. Dans ce cas, la hauteur commencera la transition immédiatement, tandis que la largeur attendra 1 seconde.
