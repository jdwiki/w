Syntaxe #

  • @media [not|only] mediatype and (media feature) { /* Règles CSS à appliquer */ }

Paramètres #

Paramètre Détails
type de média (Facultatif) Il s’agit du type de média. Il peut s’agir de n’importe quoi dans la plage de “tout” à “écran”.
‘pas’ (Facultatif) N’applique pas le CSS pour ce type de média particulier et s’applique à tout le reste.
fonctionnalité multimédia Logique pour identifier le cas d’utilisation de CSS. Options décrites ci-dessous.
Fonctionnalité multimédia Détails
rapport d'aspect Décrit le format d’image de la zone d’affichage ciblée du périphérique de sortie.
couleur Indique le nombre de bits par composant de couleur du périphérique de sortie. Si le périphérique n’est pas un périphérique couleur, cette valeur est zéro.
index-couleur Indique le nombre d’entrées dans la table de correspondance des couleurs pour le périphérique de sortie.
grille Détermine si le périphérique de sortie est un périphérique de grille ou un périphérique bitmap.
hauteur La fonction de média de hauteur décrit la hauteur de la surface de rendu du périphérique de sortie.
largeur-max CSS ne s’appliquera pas sur une largeur d’écran plus large que celle spécifiée.
largeur min CSS ne s’appliquera pas sur une largeur d’écran plus étroite que celle spécifiée.
hauteur-max CSS ne s’appliquera pas sur une hauteur d’écran supérieure à celle spécifiée.
hauteur-min CSS ne s’appliquera pas sur une hauteur d’écran inférieure à celle spécifiée.
monochrome Indique le nombre de bits par pixel sur un périphérique monochrome (niveaux de gris).
orientation CSS ne s’affichera que si l’appareil utilise l’orientation spécifiée. Voir les remarques pour plus de détails.
résolution Indique la résolution (densité de pixels) du périphérique de sortie.
scanner Décrit le processus de balayage des périphériques de sortie de télévision.
largeur La fonction largeur du support décrit la largeur de la surface de rendu du périphérique de sortie (telle que la largeur de la fenêtre du document ou la largeur de la zone de page sur une imprimante).
Fonctionnalités obsolètes Détails
device-aspect-ratio Déprécié Le CSS ne s’affichera que sur les appareils dont le rapport hauteur/largeur correspond au rapport spécifié. Il s’agit d’une fonctionnalité “obsolète” et son fonctionnement n’est pas garanti.
max-device-width Obsolète Identique à max-width mais mesure la largeur physique de l’écran, plutôt que la largeur d’affichage du navigateur.
min-device-width Obsolète Identique à min-width mais mesure la largeur physique de l’écran, plutôt que la largeur d’affichage du navigateur.
hauteur-max-device Obsolète Identique à max-height mais mesure la largeur physique de l’écran, plutôt que la largeur d’affichage du navigateur.
min-device-height Obsolète Identique à min-height mais mesure la largeur physique de l’écran, plutôt que la largeur d’affichage du navigateur.

Les requêtes multimédias sont prises en charge dans tous les navigateurs modernes, y compris Chrome, Firefox, Opera et Internet Explorer 9 et versions ultérieures.

Il est important de noter que la fonctionnalité multimédia “orientation” n’est pas limitée aux appareils mobiles. Il est basé sur la largeur et la hauteur de la fenêtre (et non sur la fenêtre ou les appareils).

Le mode paysage est lorsque la largeur de la fenêtre est supérieure à la hauteur de la fenêtre.

Mode Portrait est lorsque la hauteur de la fenêtre est supérieure à la largeur de la fenêtre.

Cela se traduit généralement par un moniteur de bureau en mode paysage, mais il peut parfois s’agir d’un portrait.

Dans la plupart des cas, les appareils mobiles indiqueront leur résolution et non leur taille réelle en pixels, qui peut différer en raison de la densité des pixels. Pour les forcer à indiquer leur taille réelle en pixels, ajoutez ce qui suit dans votre balise “head” :

<meta name="viewport" content="width=device-width, initial-scale=1">

Exemple de base #

@media screen and (min-width: 720px) {
    body {
        background-color: skyblue;
    }
}

La requête multimédia ci-dessus spécifie deux conditions :

  1. La page doit être visualisée sur un écran normal (pas une page imprimée, un projecteur, etc.).
  2. La largeur de la fenêtre d’affichage de l’utilisateur doit être d’au moins 720 pixels.

Si ces conditions sont remplies, les styles à l’intérieur de la requête média seront actifs et la couleur d’arrière-plan de la page sera bleu ciel.

Les requêtes média sont appliquées dynamiquement. Si, lors du chargement de la page, les conditions spécifiées dans la requête multimédia sont remplies, le CSS sera appliqué, mais sera immédiatement désactivé si les conditions cessent d’être remplies. Inversement, si les conditions ne sont initialement pas remplies, le CSS ne sera pas appliqué tant que les conditions spécifiées ne seront pas remplies.

Dans notre exemple, si la largeur de la fenêtre d’affichage de l’utilisateur est initialement supérieure à 720 pixels, mais que l’utilisateur réduit la largeur du navigateur, la couleur d’arrière-plan cessera d’être bleu ciel dès que l’utilisateur aura redimensionné la fenêtre d’affichage à moins de 720 pixels dans largeur.

type de support #

Les requêtes média ont un paramètre mediatype facultatif. Ce paramètre est placé directement après la déclaration @media (@media mediatype), par exemple :

@media print {
    html {
        background-color: white;
    }
}

Le code CSS ci-dessus donnera à l’élément DOM HTML une couleur de fond blanche lors de l’impression.

Le paramètre mediatype a un préfixe facultatif not ou only qui appliquera les styles à tout sauf au type de média spécifié ou uniquement au type de média spécifié, respectivement. Par exemple, l’exemple de code suivant appliquera le style à tous les types de média, à l’exception de “print”.

@media not print {
    html {
        background-color: green;
    }
}

Et de la même manière, pour ne l’afficher que sur l’écran, cela peut être utilisé :

@media only screen {
    .fadeInEffects {
        display: block;
    }
}

La liste de mediatype peut être mieux comprise avec le tableau suivant :

Type de média Descriptif
tout Appliquer à tous les appareils
écran Ordinateurs par défaut
imprimer Imprimantes en général. Utilisé pour styliser les versions imprimées des sites Web
portable PDA, téléphones portables et appareils portables avec un petit écran
projection Pour une présentation projetée, par exemple des projecteurs
aural Systèmes de parole
braille Appareils tactiles braille
en relief Imprimantes braille paginé
tv Appareils de type télévision
tty Appareils avec une grille de caractères à pas fixe. Terminaux, portables.

Terminologie et structure #

Les requêtes média permettent d’appliquer des règles CSS basées sur le type d’appareil/média (par exemple, écran, impression ou ordinateur de poche) appelé type de média, des aspects supplémentaires de l’appareil sont décrits avec fonctionnalités multimédia telles que la disponibilité de la couleur ou des dimensions de la fenêtre.

Structure générale d’une requête média #

@media [...] {
    /* One or more CSS rules to apply when the query is satisfied */
}

Une requête média contenant un type de média #

@media print {
    /* One or more CSS rules to apply when the query is satisfied */
}

Une requête média contenant un type de média et une fonctionnalité média #

@media screen and (max-width: 600px) {
    /* One or more CSS rules to apply when the query is satisfied */
}

Une requête média contenant une fonctionnalité média (et un type de média implicite de “tous”) #

@media (orientation: portrait) {        
    /* One or more CSS rules to apply when the query is satisfied */
}

Largeur vs fenêtre d’affichage #

Lorsque nous utilisons la “largeur” ​​avec les requêtes multimédias, il est important de définir correctement la balise meta. La balise meta de base ressemble à ceci et doit être insérée dans la balise <head>.

<meta name="viewport" content="width=device-width,initial-scale=1">

Pourquoi est-ce important ?

Selon la définition de MDN, la “largeur” ​​est

La fonction width media décrit la largeur de la surface de rendu du périphérique de sortie (comme la largeur de la fenêtre de document ou la largeur de la zone de page sur une imprimante).

Qu’est-ce que cela signifie?

View-port est la largeur de l’appareil lui-même. Si la résolution de votre écran indique que la résolution est de 1280 x 720, la largeur de votre fenêtre d’affichage est de “1280px”.

Le plus souvent, de nombreux appareils allouent une quantité de pixels différente pour afficher un pixel. Par exemple, l’iPhone 6 Plus a une résolution de 1242 x 2208. Mais la largeur et la hauteur réelles de la fenêtre sont de 414 x 736. Cela signifie que 3 pixels sont utilisés pour créer 1 pixel.

Mais si vous n’avez pas correctement défini la balise meta, il essaiera d’afficher votre page Web avec sa résolution native, ce qui se traduira par une vue agrandie (textes et images plus petits).

Media Queries pour les écrans Retina et non Retina #

Bien que cela ne fonctionne que pour les navigateurs basés sur WebKit, ceci est utile :

/* ----------- Non-Retina Screens ----------- */
@media screen 
  and (min-width: 1200px) 
  and (max-width: 1600px) 
  and (-webkit-min-device-pixel-ratio: 1) { 
}

/* ----------- Retina Screens ----------- */
@media screen 
  and (min-width: 1200px) 
  and (max-width: 1600px) 
  and (-webkit-min-device-pixel-ratio: 2)
  and (min-resolution: 192dpi) { 
}

Informations d’arrière-plan

Il existe deux types de pixels à l’écran. L’un est les pixels logiques et l’autre les pixels physiques. Généralement, les pixels physiques restent toujours les mêmes, car ils sont les mêmes pour tous les périphériques d’affichage. Les pixels logiques changent en fonction de la résolution des appareils pour afficher des pixels de meilleure qualité. Le ratio de pixels du périphérique est le rapport entre les pixels physiques et les pixels logiques. Par exemple, le MacBook Pro Retina, l’iPhone 4 et les versions ultérieures signalent un rapport de pixels de l’appareil de 2, car la résolution linéaire physique est le double de la résolution logique.

La raison pour laquelle cela ne fonctionne qu’avec les navigateurs basés sur WebKit est la suivante :

  • Le préfixe fournisseur -webkit- avant la règle.
  • Cela n’a pas été implémenté dans des moteurs autres que WebKit et Blink.

Utilisation des Media Queries pour cibler différentes tailles d’écran #

Souvent, la conception Web réactive implique des requêtes multimédias, qui sont des blocs CSS qui ne sont exécutés que si une condition est satisfaite. Ceci est utile pour la conception Web réactive, car vous pouvez utiliser des requêtes multimédias pour spécifier différents styles CSS pour la version mobile de votre site Web par rapport à la version de bureau.

@media only screen and (min-width: 300px) and (max-width: 767px) {
    .site-title {
        font-size: 80%;
    }

    /* Styles in this block are only applied if the screen size is atleast 300px wide, but no more than 767px */
}

@media only screen and (min-width: 768px) and (max-width: 1023px) {
    .site-title {
        font-size: 90%;
    }

    /* Styles in this block are only applied if the screen size is atleast 768px wide, but no more than 1023px */
}

@media only screen and (min-width: 1024px) {
    .site-title {
        font-size: 120%;
    }

    /* Styles in this block are only applied if the screen size is over 1024px wide. */
}

Utiliser sur la balise de lien #

<link rel="stylesheet" media="min-width: 600px" href="example.css" />

Cette feuille de style est toujours téléchargée mais n’est appliquée que sur les appareils dont la largeur d’écran est supérieure à 600 pixels.

Requêtes média et IE8 #

[Les requêtes multimédias][1] ne sont pas du tout prises en charge dans IE8 et les versions antérieures.

Une solution de contournement basée sur Javascript #

Pour ajouter la prise en charge d’IE8, vous pouvez utiliser l’une des nombreuses solutions JS. Par exemple, [Respond][2] peut être ajouté pour ajouter la prise en charge des requêtes multimédias pour IE8 uniquement avec le code suivant :

<!--[if lt IE 9]>
<script 
   src="respond.min.js">
</script>
<![endif]-->

[CSS Mediaqueries][3] est une autre bibliothèque qui fait la même chose. Le code pour ajouter cette bibliothèque à votre HTML serait identique :

<!--[if lt IE 9]>
<script 
   src="css3-mediaqueries.js">
</script>
<![endif]-->

L’alternative #

Si vous n’aimez pas une solution basée sur JS, vous devriez également envisager d’ajouter une feuille de style IE<9 uniquement dans laquelle vous ajustez votre style spécifique à IE<9. Pour cela, vous devez ajouter le code HTML suivant à votre code :

<!--[if lt IE 9]>
<link rel="stylesheet" type="text/css" media="all" href="style-ielt9.css"/>
<![endif]-->

Noter :

Techniquement, c’est une autre alternative : utiliser [hacks CSS][4] pour cibler IE<9. Il a le même impact qu’une feuille de style IE<9 uniquement, mais vous n’avez pas besoin d’une feuille de style séparée pour cela. Je ne recommande pas cette option, cependant, car ils produisent du code CSS invalide (ce qui n’est qu’une des nombreuses raisons pour lesquelles l’utilisation de hacks CSS est généralement mal vue aujourd’hui).

[1] : http://www.brianhadaway.com/responsive-web-design-using-css3-media-queries/ [2] : https://github.com/scottjehl/Respond [3] : https://code.google.com/p/css3-mediaqueries-js/ [4] : http://browserhacks.com/
