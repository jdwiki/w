La propriété d’affichage #

La propriété CSS display est fondamentale pour contrôler la mise en page et le flux d’un document HTML. La plupart des éléments ont une valeur display par défaut de block ou inline (bien que certains éléments aient d’autres valeurs par défaut).

En ligne #

Un élément inline n’occupe que la largeur nécessaire. Il s’empile horizontalement avec d’autres éléments du même type et ne peut pas contenir d’autres éléments non en ligne.

<span>This is some <b>bolded</b> text!</span>

[![][1]][1]

Comme démontré ci-dessus, deux éléments inline, <span> et <b>, sont en ligne (d’où le nom) et ne rompent pas le flux du texte.

Bloquer #

Un élément block occupe la largeur maximale disponible de son élément parent. Il commence par une nouvelle ligne et, contrairement aux éléments “en ligne”, il ne limite pas le type d’éléments qu’il peut contenir.

<div>Hello world!</div><div>This is an example!</div>

[![][2]][2]

L’élément div est au niveau du bloc par défaut, et comme indiqué ci-dessus, les deux éléments block sont empilés verticalement et, contrairement aux éléments inline, le flux du texte se brise.

Bloc en ligne #

La valeur inline-block nous donne le meilleur des deux mondes : elle mélange l’élément avec le flux du texte tout en nous permettant d’utiliser padding, margin, height et des propriétés similaires qui n’ont aucun effet visible sur les éléments “en ligne”.

Les éléments avec cette valeur d’affichage agissent comme s’ils étaient du texte normal et, par conséquent, sont affectés par les règles contrôlant le flux de texte telles que text-align. Par défaut, ils sont également réduits à la taille la plus petite possible pour s’adapter à leur contenu.

<!--Inline: unordered list-->
<style>
li {
    display : inline;
    background : lightblue;
    padding:10px;

    border-width:2px;
    border-color:black;
    border-style:solid;
    }
</style>    

<ul>
<li>First Element </li>
<li>Second Element </li>
<li>Third Element </li>
</ul>

rien #

Un élément qui reçoit la valeur none pour sa propriété display ne sera pas affiché du tout.

Par exemple, créons un élément div dont l’identifiant est myDiv :

<div id="myDiv"></div>

Cela peut maintenant être marqué comme n’étant pas affiché par la règle CSS suivante :

#myDiv {
    display: none;
}

Lorsqu’un élément a été défini sur display:none;, le navigateur ignore toutes les autres propriétés de mise en page pour cet élément spécifique (à la fois position et float). Aucune boîte ne sera rendue pour cet élément et son existence en html n’affecte pas la position des éléments suivants.

Notez que cela est différent de définir la propriété visibility sur hidden. Définir visibility: hidden; pour un élément n’afficherait pas l’élément sur la page, mais l’élément occuperait toujours l’espace dans le processus de rendu comme s’il était visible. Cela affectera donc la façon dont les éléments suivants sont affichés sur la page.

La valeur “none” de la propriété display est couramment utilisée avec JavaScript pour afficher ou masquer des éléments à volonté, éliminant ainsi le besoin de les supprimer et de les recréer.

Pour obtenir l’ancienne structure de table en utilisant div #

Il s’agit de la structure normale des tableaux HTML

<style>
    table {
        width: 100%;
    }
</style>

<table>
  <tr>
    <td>
        I'm a table
    </td>
  </tr>
</table>

Vous pouvez faire la même implémentation comme celle-ci

<style>
    .table-div {
        display: table;
    }
    .table-row-div {
        display: table-row;
    }
    .table-cell-div {
        display: table-cell;
    }
</style>

<div class="table-div>
  <div class="table-row-div>
    <div class="table-cell-div>
      I behave like a table now
    </div>
  </div>
</div>
