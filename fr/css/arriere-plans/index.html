Avec CSS, vous pouvez définir des couleurs, des dégradés et des images comme arrière-plan d’un élément.

Il est possible de spécifier diverses combinaisons d’images, de couleurs et de dégradés, et d’ajuster la taille, le positionnement et la répétition (entre autres) de ceux-ci.

Syntaxe #

  • couleur de fond : couleur | transparent | initiale | hériter;
  • image de fond : url | aucun | initiale | hériter;
  • position d’arrière-plan : valeur ;
  • taille d’arrière-plan : <bg-size> [<bg-size>]
  • <taille-bg> : automatique | longueur | couverture | contenir | initiale | hériter;
  • répétition d’arrière-plan : répéter | répéter-x | répéter-y | sans répétition | initiale | hériter;
  • background-origin : padding-box | border-box | boîte de contenu | initiale | hériter;
  • clip d’arrière-plan : border-box | boîte de rembourrage | boîte de contenu | initiale | hériter;
  • pièce jointe en arrière-plan : faites défiler | fixe | locale | initiale | hériter;
  • arrière-plan : couleur-bg position de l’image-bg / taille-bg répétition-bg origine-bg clip-bg attachement-bg initiale | hériter;
  • Les dégradés CSS3 ne fonctionneront pas sur les versions d’Internet Explorer inférieures à 10.

Couleur de l’arrière plan #

La propriété background-color définit la couleur d’arrière-plan d’un élément à l’aide d’une valeur de couleur ou de mots-clés, tels que transparent, inherit ou initial.

  • transparent, spécifie que la couleur de fond doit être transparente. C’est la valeur par défaut.

  • inherit, hérite cette propriété de son élément parent.

  • initial, définit cette propriété à sa valeur par défaut.

Ceci peut être appliqué à tous les éléments, et ::first-letter/::first-line [pseudo-elements][1].

Les couleurs en CSS peuvent être spécifiées par [différentes méthodes][2].

Noms des couleurs #

CSS

div {
  background-color: red;  /* red */
}

HTML

<div>This will have a red background</div>
  • L’exemple utilisé ci-dessus est l’une des nombreuses façons dont CSS doit représenter une seule couleur.

Codes de couleur hexadécimaux #

Le code hexadécimal est utilisé pour désigner les composants RVB d’une couleur en notation hexadécimale en base 16. #ff0000, par exemple, est rouge vif, où la composante rouge de la couleur est de 256 bits (ff) et les parties verte et bleue correspondantes de la couleur sont 0 (00).

Si les deux valeurs de chacune des trois paires RVB (R, G et B) sont identiques, le code de couleur peut être raccourci en trois caractères (le premier chiffre de chaque paire). #ff0000 peut être raccourci en #f00, et #ffffff peut être raccourci en #fff.

La notation hexadécimale est insensible à la casse.

body {
  background-color: #de1205; /* red */
}

.main {
  background-color: #00f; /* blue */
}

RVB / RVBa #

Une autre façon de déclarer une couleur est d’utiliser RGB ou RGBa.

RVB signifie Rouge, Vert et Bleu, et nécessite trois valeurs distinctes entre 0 et 255, mises entre parenthèses, qui correspondent aux valeurs de couleur décimales pour respectivement le rouge, le vert et le bleu.

RGBa vous permet d’ajouter un paramètre alpha supplémentaire entre 0,0 et 1,0 pour définir l’opacité.

header {
  background-color: rgb(0, 0, 0); /* black */
}

footer {
  background-color: rgba(0, 0, 0, 0.5); /* black with 50% opacity */
}

HSL / HSLa #

Une autre façon de déclarer une couleur consiste à utiliser HSL ou HSLa et est similaire à RGB et RGBa.

HSL signifie teinte, saturation et luminosité, et est aussi souvent appelé HLS :

  • La teinte est un degré sur la roue chromatique (de 0 à 360).
  • La saturation est un pourcentage compris entre 0% et 100%.
  • La légèreté est aussi un pourcentage compris entre 0% et 100%.

HSLa vous permet d’ajouter un paramètre alpha supplémentaire entre 0,0 et 1,0 pour définir l’opacité.

li a {
  background-color: hsl(120, 100%, 50%); /* green */
}

#p1 {
  background-color: hsla(120, 100%, 50%, .3); /* green with 30% opacity */
}

Interaction avec l’image d’arrière-plan #

Les déclarations suivantes sont toutes équivalentes :

body {
  background: red;
  background-image: url(partiallytransparentimage.png);
}

body {
  background-color: red;
  background-image: url(partiallytransparentimage.png);
}

body {
  background-image: url(partiallytransparentimage.png);
  background-color: red;
}

body {
  background: red url(partiallytransparentimage.png);
}

Ils conduiront tous à l’affichage de la couleur rouge sous l’image, où les parties de l’image sont transparentes, ou l’image ne s’affiche pas (peut-être à cause de background-repeat).

Notez que ce qui suit n’est pas équivalent :

body {
  background-image: url(partiallytransparentimage.png);
  background: red;
}

Ici, la valeur de background remplace votre background-image.

Pour plus d’informations sur la propriété background, voir https://www.wikiod.com/fr/css/arriere-plans#Raccourci d’arrière-plan

[1] : https://www.wikiod.com/fr/css/pseudo-elements [2] : https://www.wikiod.com/fr/css/couleurs

Dégradés d’arrière-plan #

Les dégradés sont de nouveaux types d’images, ajoutés dans CSS3. En tant qu’image, les dégradés sont définis avec la propriété [background-image](https://www.wikiod.com/fr/css/arriere-plans#Image de fond) ou le raccourci background.

Il existe deux types de fonctions de gradient, linéaire et radiale. Chaque type a une variante non répétitive et une variante répétitive :

  • dégradé linéaire()
  • repeating-linear-gradient()
  • dégradé radial()
  • repeating-radial-gradient()

gradient linéaire() #

Un linear-gradient a la syntaxe suivante

background: linear-gradient( <direction>?, <color-stop-1>, <color-stop-2>, ...);
Valeur Signification
<direction> Il peut s’agir d’un argument tel que “vers le haut”, “vers le bas”, “vers la droite” ou “vers la gauche” ; ou un angle comme 0deg, 90deg… . L’angle commence de haut en bas et tourne dans le sens des aiguilles d’une montre. Peut être spécifié dans deg, grad, rad, ou turn . S’il est omis, le dégradé s’écoule de haut en bas
<liste-stop-couleur> Liste de couleurs, éventuellement suivie chacune d’un pourcentage ou d’une longueur pour l’afficher. Par exemple, jaune 10%, rgba(0,0,0,.5) 40px, #fff 100%…

Par exemple, cela crée un dégradé linéaire qui part de la droite et passe du rouge au bleu

.linear-gradient {
  background: linear-gradient(to left, red, blue); /* you can also use 270deg */
}

Vous pouvez créer un dégradé “diagonal” en déclarant une position de départ horizontale et verticale.

.diagonal-linear-gradient {
  background: linear-gradient(to left top, red, yellow 10%);
}

Il est possible de spécifier n’importe quel nombre d’arrêts de couleur dans un dégradé en les séparant par des virgules. Les exemples suivants créeront un dégradé avec 8 arrêts de couleur

.linear-gradient-rainbow {
  background: linear-gradient(to left, red, orange, yellow, green, blue, indigo, violet)
}

dégradé radial() #

.radial-gradient-simple {
  background: radial-gradient(red, blue);
}

.radial-gradient {
  background: radial-gradient(circle farthest-corner at top left, red, blue);
}
Valeur Signification
cercle Forme de dégradé. Les valeurs sont circle ou ellipse, la valeur par défaut est ellipse.
coin le plus éloigné Mots-clés décrivant la taille de la forme finale. Les valeurs sont côté le plus proche, côté le plus éloigné, coin le plus proche, coin le plus éloigné
en haut à gauche Définit la position du centre du dégradé, de la même manière que background-position.

Dégradés répétitifs #

Les fonctions de gradient répétitives prennent les mêmes arguments que les exemples ci-dessus, mais placent le dégradé en mosaïque sur l’arrière-plan de l’élément.

.bullseye {
  background: repeating-radial-gradient(red, red 10%, white 10%, white 20%);
}
.warning {
  background: repeating-linear-gradient(-45deg, yellow, yellow 10%, black 10%, black 20% );
}
Valeur Signification
-45deg Unité d’angle. L’angle commence de haut en bas et tourne dans le sens des aiguilles d’une montre. Peut être spécifié dans deg, grad, rad, ou turn .
à gauche Direction du dégradé, la valeur par défaut est “vers le bas”. Syntaxe : to [y-axis(top OR bottom)] [x-axis(left OR right)] c’est-à-dire to top right
jaune 10% Couleur, éventuellement suivie d’un pourcentage ou d’une longueur pour l’afficher. Répété deux fois ou plus.

Notez que les codes de couleur HEX, RGB, RGBa, HSL et HSLa peuvent être utilisés à la place des noms de couleur. Les noms de couleurs ont été utilisés à des fins d’illustration. Notez également que la syntaxe du dégradé radial est beaucoup plus complexe que celle du dégradé linéaire, et une version simplifiée est présentée ici. Pour une explication complète et les spécifications, consultez la MDN Docs

Image de fond #

La propriété background-image est utilisée pour spécifier une image d’arrière-plan à appliquer à tous les éléments correspondants. Par défaut, cette image est en mosaïque pour couvrir tout l’élément, à l’exclusion de la marge.

.myClass {
  background-image: url('/path/to/image.jpg');
}

Pour utiliser plusieurs images comme background-image, définissez url() séparé par des virgules

.myClass {
  background-image: url('/path/to/image.jpg'),
                    url('/path/to/image2.jpg');
}

Les images s’empileront selon leur ordre avec la première image déclarée au-dessus des autres et ainsi de suite.

Valeur | Résultat —————————-|——————— ———– url('/chemin/vers/image.jpg') | Spécifiez le(s) chemin(s) de l’image d’arrière-plan ou une ressource d’image spécifiée avec le schéma d’URI de données (les apostrophes peuvent être omises), séparez les multiples par des virgules aucun | Pas d’image de fond initiale | Valeur par défaut hériter | Hériter de la valeur du parent

Plus de CSS pour l’image d’arrière-plan

Les attributs suivants sont très utiles et presque essentiels aussi.

background-size:     xpx ypx | x% y%;
background-repeat:   no-repeat | repeat | repeat-x | repeat-y;
background-position: left offset (px/%) right offset (px/%) | center center | left top | right bottom;

Raccourci d’arrière-plan #

La propriété background peut être utilisée pour définir une ou plusieurs propriétés liées à l’arrière-plan :

Valeur Descriptif CSS Ver.
image-de-fond Image de fond à utiliser 1+
couleur de fond Couleur de fond à appliquer 1+
position d'arrière-plan Position de l’image de fond 1+
taille d'arrière-plan Taille de l’image de fond 3+
répétition d'arrière-plan Comment répéter l’image d’arrière-plan 1+
origine-arrière-plan Comment l’arrière-plan est positionné (ignoré lorsque background-attachment est fixed) 3+
fond-clip Comment l’arrière-plan est peint par rapport à content-box, border-box ou padding-box 3+
pièce jointe d'arrière-plan Comment se comporte l’image d’arrière-plan, qu’elle défile avec son bloc contenant ou qu’elle ait une position fixe dans la fenêtre d’affichage 1+
initiale Définit la propriété sur la valeur par défaut 3+
hériter Hérite la valeur de la propriété du parent 2+

L’ordre des valeurs n’a pas d’importance et chaque valeur est facultative

Syntaxe #

La syntaxe de la déclaration abrégée d’arrière-plan est :

background: [<background-image>] [<background-color>]  [<background-position>]/[<background-size>] [<background-repeat>] [<background-origin>] [<background-clip>] [<background-attachment>] [<initial|inherit>];

Exemples #

background: red;

Définissez simplement un background-color avec la valeur red.

background: border-box red;

Définir un background-clip sur border-box et un background-color sur rouge.

background: no-repeat center url("somepng.jpg");

Définit un background-repeat à no-repeat, background-origin au centre et un background-image à une image.

background: url('pattern.png') green;

Dans cet exemple, le background-color de l’élément serait défini sur green avec pattern.png, s’il est disponible, superposé sur la couleur, répété aussi souvent que nécessaire pour remplir l’élément. Si pattern.png inclut une transparence, la couleur green sera visible derrière.

background: #000000 url("picture.png") top left / 600px auto no-repeat;

Dans cet exemple, nous avons un arrière-plan noir avec une image ‘picture.png’ en haut, l’image ne se répète dans aucun des deux axes et est positionnée dans le coin supérieur gauche. Le / après la position permet d’inclure la taille de l’image d’arrière-plan qui, dans ce cas, est définie sur 600px largeur et auto pour la hauteur. Cet exemple pourrait bien fonctionner avec une image caractéristique qui peut se fondre dans une couleur unie.

REMARQUE : L’utilisation de la propriété abrégée d’arrière-plan réinitialise toutes les valeurs de propriété d’arrière-plan précédemment définies, même si aucune valeur n’est donnée. Si vous souhaitez uniquement modifier une valeur de propriété d’arrière-plan précédemment définie, utilisez plutôt une propriété longue.

Taille d’arrière-plan #

Aperçu général #

La propriété [background-size][1] permet de contrôler la mise à l’échelle de background-image. Il prend jusqu’à deux valeurs, qui déterminent l’échelle/taille de l’image résultante dans le sens vertical et horizontal. Si la propriété est manquante, elle est réputée “auto” à la fois pour “width” et “height”.

auto conservera le rapport d’aspect de l’image, s’il peut être déterminé. La hauteur est facultative et peut être considérée comme “auto”. Par conséquent, sur une image de 256 px × 256 px, tous les paramètres de taille d’arrière-plan suivants donneraient une image d’une hauteur et d’une largeur de 50 px :

background-size: 50px;
background-size: 50px auto; /* same as above */
background-size: auto 50px;
background-size: 50px 50px;

Donc, si nous commençons avec l’image suivante (qui a la taille mentionnée de 256 px × 256 px),

[![notre innocente image 256x256][2]][2]

nous allons nous retrouver avec un 50 px × 50 px sur l’écran de l’utilisateur, contenu dans le fond de notre élément :

[![le minuscule 50px] [3]][3]

On peut également utiliser des valeurs en pourcentage pour mettre à l’échelle l’image par rapport à l’élément. L’exemple suivant donnerait une image dessinée de 200 px × 133 px :

#withbackground {
    background-image: url(to/some/background.png);

    background-size: 100% 66%;
    
    width: 200px;
    height: 200px;

    padding: 0;
    margin: 0;
}

[![plus de format d’image :/][4]][4]

Le comportement dépend de [background-origin][5].

Garder le rapport d’aspect #

Le dernier exemple de la section précédente a perdu son format d’image d’origine. Le cercle est devenu une ellipse, le carré un rectangle, le triangle un autre triangle.

L’approche de la longueur ou du pourcentage n’est pas suffisamment flexible pour conserver le rapport d’aspect à tout moment. auto n’aide pas, car vous ne savez peut-être pas quelle dimension de votre élément sera la plus grande. Cependant, pour couvrir complètement certaines zones avec une image (et un rapport d’aspect correct) ou pour contenir une image avec un rapport d’aspect correct complètement dans une zone d’arrière-plan, les valeurs contain et cover fournissent la fonctionnalité supplémentaire.

Eggsplanation pour contain et cover #

Désolé pour le mauvais jeu de mots, mais nous allons utiliser une photo du jour par Biswarup Ganguly pour la démonstration. Disons qu’il s’agit de votre écran et que la zone grise est en dehors de votre écran visible. Pour la démonstration, nous allons supposer un ratio de 16 × 9.

![écran][6]

Nous voulons utiliser l’image du jour susmentionnée comme arrière-plan. Cependant, nous avons recadré l’image à 4x3 pour une raison quelconque. Nous pourrions définir la propriété background-size sur une longueur fixe, mais nous nous concentrerons sur contain et cover. Notez que je suppose également que nous n’avons pas modifié la largeur et/ou la hauteur de body.

contient #

contenir Mettez l’image à l’échelle, tout en préservant son rapport d’aspect intrinsèque (le cas échéant), à la plus grande taille de sorte que sa largeur et sa hauteur puissent tenir à l’intérieur de la zone de positionnement de l’arrière-plan.

Cela garantit que l’image d’arrière-plan est toujours complètement contenue dans la zone de positionnement d’arrière-plan, cependant, il pourrait y avoir un espace vide rempli avec votre background-color dans ce cas :

![contient][7]

couverture #

couverture Mettez l’image à l’échelle, tout en préservant son rapport d’aspect intrinsèque (le cas échéant), à la plus petite taille de sorte que sa largeur et sa hauteur puissent couvrir complètement la zone de positionnement de l’arrière-plan.

Cela garantit que l’image d’arrière-plan couvre tout. Il n’y aura pas de “couleur d’arrière-plan” visible, cependant, selon le rapport de l’écran, une grande partie de votre image pourrait être coupée :

![couverture][8]

Position d’arrière-plan #

La propriété background-position est utilisée pour spécifier la position de départ d’une image d’arrière-plan ou d’un dégradé

.myClass {
  background-image: url('path/to/image.jpg');
  background-position: 50% 50%;
}

La position est définie à l’aide d’une coordonnée X et Y et doit être définie à l’aide de l’une des unités utilisées dans CSS.

Unité Descriptif
valeur% valeur% Un pourcentage pour le décalage horizontal est relatif à (largeur de la zone de positionnement d’arrière-plan - largeur de l’image d’arrière-plan).
Un pourcentage pour le décalage vertical est relatif à (hauteur de la zone de positionnement d’arrière-plan - hauteur de l’image d’arrière-plan)
La taille de l’image est la taille donnée par background-size.
valeurpx valeurpx Décale l’image d’arrière-plan d’une longueur donnée en pixels par rapport au coin supérieur gauche de la zone de positionnement d’arrière-plan

Les unités en CSS peuvent être spécifiées par différentes méthodes (voir [ici][1]).

Propriétés de position d’arrière-plan à la main #

En plus de la propriété abrégée ci-dessus, on peut également utiliser les propriétés d’arrière-plan détaillées « background-position-x » et « background-position-y ». Ceux-ci vous permettent de contrôler les positions x ou y séparément.

REMARQUE : Ceci est pris en charge par tous les navigateurs sauf Firefox (versions 31-48) [2]. Firefox 49, qui sortira en septembre 2016, prendra en charge ces propriétés. Jusque-là, il y a un hack Firefox dans cette réponse Stack Overflow.

La propriété background-origin #

La propriété background-origin spécifie où l’image d’arrière-plan est positionnée.

Remarque : Si la propriété background-attachment est définie sur fixed, cette propriété n’a aucun effet.

Valeur par défaut : padding-box

Valeurs possibles:

  • padding-box - La position est relative à la boîte de remplissage
  • border-box - La position est relative à la bordure de la boîte
  • content-box - La position est relative à la boîte de contenu
  • “initiale”
  • ‘hériter’

Pièce jointe en arrière-plan #

La propriété background-attachment définit si une image d’arrière-plan est fixe ou défile avec le reste de la page.

body { 
  background-image: url('img.jpg');
  background-attachment: fixed;
}
Valeur La description
faire défiler L’arrière-plan défile avec l’élément. C’est la valeur par défaut.
fixe L’arrière-plan est fixe par rapport à la fenêtre.
locale L’arrière-plan défile avec le contenu de l’élément.
initiale Définit cette propriété à sa valeur par défaut.
hériter Hérite cette propriété de son élément parent.

Exemples #

pièce jointe d’arrière-plan : faites défiler #

Le comportement par défaut, lorsque le corps défile, l’arrière-plan défile avec :

body {
  background-image: url('image.jpg');
  background-attachment: scroll;
}

pièce jointe en arrière-plan : corrigée #

L’image d’arrière-plan sera fixe et ne bougera pas lors du défilement du corps :

body {
  background-image: url('image.jpg');
  background-attachment: fixed;
}

pièce jointe en arrière-plan : local #

L’image d’arrière-plan de la div défile lorsque le contenu de la div défile.

div {
  background-image: url('image.jpg');
  background-attachment: local;
}

Image d’arrière-plan multiple #

En CSS3, nous pouvons empiler plusieurs arrière-plans dans le même élément.

#mydiv {
  background-image: url(img_1.png), /* top image */
                    url(img_2.png), /* middle image */
                    url(img_3.png); /* bottom image */
  background-position: right bottom,
                       left top,
                       right top;
  background-repeat: no-repeat,
                     repeat,
                     no-repeat;
}

Les images seront empilées les unes sur les autres avec le premier arrière-plan en haut et le dernier arrière-plan à l’arrière. img_1 sera en haut, img_2 et img_3 seront en bas.

Nous pouvons également utiliser la propriété abrégée d’arrière-plan pour cela :

#mydiv {
  background: url(img_1.png) right bottom no-repeat,
              url(img_2.png) left top repeat,
              url(img_3.png) right top no-repeat;
}

Nous pouvons également empiler des images et des dégradés :

#mydiv {
  background: url(image.png) right bottom no-repeat,
              linear-gradient(to bottom, #fff 0%,#000 100%);
}

Clip d’arrière-plan #

Définition et utilisation : la propriété background-clip spécifie la zone de peinture de l’arrière-plan.

Valeur par défaut : border-box

Valeurs

  • border-box est la valeur par défaut. Cela permet à l’arrière-plan de s’étendre jusqu’au bord extérieur de la bordure de l’élément.

  • content-box coupe l’arrière-plan au bord de la boîte de contenu.

Couleur de fond avec opacité #

Si vous définissez opacity sur un élément, cela affectera tous ses éléments enfants. Pour définir une opacité uniquement sur le fond d’un élément, vous devrez utiliser des couleurs RGBA. L’exemple suivant aura un fond noir avec une opacité de 0,6.

/* Fallback for web browsers that don't support RGBa */
background-color: rgb(0, 0, 0);

/* RGBa with 0.6 opacity */
background-color: rgba(0, 0, 0, 0.6);

/* For IE 5.5 - 7*/
filter: progid:DXImageTransform.Microsoft.gradient(startColorstr=#99000000, endColorstr=#99000000);

/* For IE 8*/
-ms-filter: "progid:DXImageTransform.Microsoft.gradient(startColorstr=#99000000, endColorstr=#99000000)";
