Les sélecteurs CSS identifient des éléments HTML spécifiques comme cibles pour les styles CSS. Cette rubrique explique comment les sélecteurs CSS ciblent les éléments HTML. Les sélecteurs utilisent un large éventail de plus de 50 méthodes de sélection proposées par le langage CSS, notamment des éléments, des classes, des identifiants, des pseudo-éléments et des pseudo-classes, ainsi que des modèles.

Syntaxe #

  • #identifiant

  • .nom du cours

  • :pseudo-nom de classe

  • ::nom du pseudo-élément

  • [attr] /* a l’attribut attr. */

  • [attr="value"] /* a l’attribut attr, et sa valeur est exactement “value”. */

  • [attr~="value"] /* a l’attribut attr, et sa valeur, lorsqu’elle est divisée sur whitespace, contient “value”. */

  • [attr|="value"] /* a l’attribut attr, et sa valeur est exactement “value”, ou sa valeur commence par “value-” . */

  • [attr^="value"] /* a l’attribut attr, et sa valeur commence par “value”. */

  • [attr$="value"] /* a l’attribut attr, et sa valeur se termine par “value”. */

  • [attr*="value"] /* a l’attribut attr, et sa valeur contient “value”. */

  • nom-élément

  • *​

  • Parfois, vous verrez des doubles-points (::) au lieu d’un seul (:). C’est un moyen de séparer les [pseudo-classes][1] des [pseudo-éléments][2].

  • Les anciens navigateurs, comme Internet Explorer 8, prennent uniquement en charge un seul deux-points (:) pour définir des pseudo-éléments.

  • Contrairement aux pseudo-classes, un seul pseudo-élément peut être utilisé par sélecteur, s’il est présent il doit apparaître après la séquence de sélecteurs simples qui représente les sujets du sélecteur (une future version de la [spécification W3C][2] pourra permettre plusieurs pseudo-éléments par sélecteur).

[1] : https://www.w3.org/TR/selectors/#pseudo-classes [2] : https://www.w3.org/TR/selectors/#pseudo-elements

Sélecteurs d’attributs #

Aperçu #

Les sélecteurs d’attributs peuvent être utilisés avec différents types d’opérateurs qui modifient les critères de sélection en conséquence. Ils sélectionnent un élément en utilisant la présence d’un attribut ou d’une valeur d’attribut donné.

Sélecteur(1) Élément apparié Sélectionne des éléments… Version CSS
[attr] <attribut div> Avec l’attribut attr 2
[attr='val'] <div attr="val"> Où l’attribut attr a la valeur val 2
[attr~='val'] <div attr="val val2 val3"> Où val apparaît dans la
liste séparée par des espaces de attr		 2
[attr^='val'] <div attr="val1 val2"> Où la valeur de attr commence par val 3
[attr$='val'] <div attr="sth aval"> Où la valeur de attr se termine par val 3
[attr*='val'] <div attr="quelquefois"> Où attr contient val n’importe où 3
[attr|=‘val’] <div attr="val-sth etc"> Où la valeur de attr est exactement val,
ou commence par val et immédiatement
suivi de - (U+002D)		 2
[attr=‘val’ i] <div attr=“val”> Où attr a la valeur val,
ignorant la casse des lettres de val.		 4(2)

Remarques:

  1. La valeur de l’attribut peut être entourée de guillemets simples ou de guillemets doubles. L’absence de guillemets peut également fonctionner, mais elle n’est pas valide selon la norme CSS et est déconseillée.

  2. Il n’y a pas de spécification CSS4 unique et intégrée, car elle est divisée en modules distincts. Cependant, il existe des modules de “niveau 4”. [Voir la prise en charge du navigateur][1].

Détails #

[attribut] #

Sélectionne les éléments avec l’attribut donné.

div[data-color] {
  color: red;
}

<div data-color="red">This will be red</div>
<div data-color="green">This will be red</div>
<div data-background="red">This will NOT be red</div>

Démo en direct sur JSBin

[attribut="valeur"] #

Sélectionne les éléments avec l’attribut et la valeur donnés.

div[data-color="red"] {
  color: red;
}

<div data-color="red">This will be red</div>
<div data-color="green">This will NOT be red</div>
<div data-color="blue">This will NOT be red</div>

Démo en direct sur JSBin

[attribut*="valeur"] #

Sélectionne les éléments avec l’attribut et la valeur donnés où l’attribut donné contient la valeur donnée n’importe où (en tant que sous-chaîne).

[class*="foo"] {
  color: red;
}

<div class="foo-123">This will be red</div>
<div class="foo123">This will be red</div>
<div class="bar123foo">This will be red</div>
<div class="barfooo123">This will be red</div>
<div class="barfo0">This will NOT be red</div>

Démo en direct sur JSBin

[attribut~="valeur"] #

Sélectionne les éléments avec l’attribut et la valeur donnés où la valeur donnée apparaît dans une liste séparée par des espaces.

[class~="color-red"] {
  color: red;
}

<div class="color-red foo-bar the-div">This will be red</div>
<div class="color-blue foo-bar the-div">This will NOT be red</div>

Démo en direct sur JSBin

[attribut^="valeur"] #

Sélectionne les éléments avec l’attribut et la valeur donnés où l’attribut donné commence par la valeur.

[class^="foo-"] {
  color: red;
}

<div class="foo-123">This will be red</div>
<div class="foo-234">This will be red</div>
<div class="bar-123">This will NOT be red</div>

Démo en direct sur JSBin

[attribut$="valeur"] #

Sélectionne les éléments avec l’attribut et la valeur donnés où l’attribut donné se termine par la valeur donnée.

[class$="file"] {
  color: red;
}

<div class="foobar-file">This will be red</div>
<div class="foobar-file">This will be red</div>
<div class="foobar-input">This will NOT be red</div>

Démo en direct sur JSBin

[attribut|="valeur"] #

Sélectionne des éléments avec un attribut et une valeur donnés où la valeur de l’attribut est exactement la valeur donnée ou est exactement la valeur donnée suivie de - (U+002D)

[lang|="EN"] {
  color: red;
}

<div lang="EN-us">This will be red</div>
<div lang="EN-gb">This will be red</div>
<div lang="PT-pt">This will NOT be red</div>

Démo en direct sur JSBin

[attribut="valeur" i] #

Sélectionne des éléments avec un attribut et une valeur donnés où la valeur de l’attribut peut être représentée par Value, VALUE, vAlUe ou toute autre possibilité insensible à la casse.

[lang="EN" i] {
  color: red;
}

<div lang="EN">This will be red</div>
<div lang="en">This will be red</div>
<div lang="PT">This will NOT be red</div>

Démo en direct sur JSBin

Spécificité des sélecteurs d’attributs #

0-1-0 #

Identique au sélecteur de classe et à la pseudoclasse.

*[type=checkbox] // 0-1-0

Notez que cela signifie qu’un sélecteur d’attribut peut être utilisé pour sélectionner un élément par son ID à un niveau de spécificité inférieur à celui s’il était sélectionné avec un [sélecteur d’ID][2] : [id="my-ID"] cibles le même élément que #my-ID mais avec une spécificité moindre.

Voir la Section Syntaxe pour plus de détails.

[1] : http://caniuse.com/#feat=css-case-insensible [2] : https://www.wikiod.com/fr/css/selecteurs#Sélecteurs d’ID

Sélecteurs de base #

Sélecteur Descriptif
* Sélecteur universel (tous éléments)
div Sélecteur de balises (tous les éléments <div>)
.blue Sélecteur de classe (tous les éléments avec la classe blue)
.bleu.rouge Tous les éléments avec la classe blue et red (un type de sélecteur composé)
#titre Sélecteur d’ID (l’élément avec l’attribut “id” défini sur headline)
:pseudo-classe [Tous les éléments avec pseudo-classe][2]
::pseudo-élément [Élément qui correspond à un pseudo-élément][5]
:lang(fr) Élément qui correspond à la déclaration :lang, par exemple <lang span="en">
div > p sélecteur enfant

Remarque : La valeur d’un ID doit être unique dans une page Web. C’est une violation de la norme HTML d’utiliser la valeur d’un ID plus d’une fois dans la même arborescence de documents .

Une liste complète des sélecteurs est disponible dans la spécification CSS Selectors Level 3.

[1] : https://www.w3.org/TR/selectors [2] : https://www.wikiod.com/fr/css/selecteurs#Pseudo-classes [3] : https://www.wikiod.com/fr/css/selecteurs#Sélecteurs d’attributs [4] : https://www.wikiod.com/fr/css/selecteurs#Combinateurs [5] : https://www.wikiod.com/fr/css/pseudo-elements

Combinateurs #

Aperçu #

Sélecteur Descriptif
étendue div Sélecteur descendant (tous les <span> descendants d’un <div>)
div > étendue Sélecteur d’enfant (tous les <span> qui sont un enfant direct d’un <div>)
a ~ étendue Sélecteur de frère général (tous les <span> qui sont frères après un <a>)
a + étendue Sélecteur de frères et sœurs adjacents (tous les <span> qui se trouvent immédiatement après un <a>)

Remarque : Les sélecteurs frères ciblent les éléments qui les suivent dans le document source. CSS, de par sa nature (il cascade), ne peut pas cibler les éléments précédents ou parents. Cependant, en utilisant la propriété flex order, [un sélecteur frère précédent peut être simulé sur un support visuel] (http://stackoverflow.com/a/36118012/3597276).

Combinateur descendant : selector selector #

Un combinateur descendant, représenté par au moins un espace ( ), sélectionne les éléments qui sont un descendant de l’élément défini. Ce combinateur sélectionne tous les descendants de l’élément (à partir des éléments enfants vers le bas).

div p {
  color:red;
}

<div>
  <p>My text is red</p>
  <section>
    <p>My text is red</p>
  </section>
</div>

<p>My text is not red</p>

Démo en direct sur JSBin

Dans l’exemple ci-dessus, les deux premiers éléments <p> sont sélectionnés puisqu’ils sont tous deux descendants de <div>.

Combinateur enfant : selector > selector #

Le combinateur enfant (>) est utilisé pour sélectionner des éléments qui sont des enfants, ou des descendants directs, de l’élément spécifié.

div > p {
  color:red;
}

<div>
  <p>My text is red</p>
  <section>
    <p>My text is not red</p>
  </section>
</div>

Démo en direct sur JSBin

Le CSS ci-dessus ne sélectionne que le premier élément <p>, car c’est le seul paragraphe qui descend directement d’un <div>.

Le deuxième élément <p> n’est pas sélectionné car il n’est pas un enfant direct de <div>.

Combinateur de frères et sœurs adjacents : selector + selector #

Le combinateur frère adjacent (+) sélectionne un élément frère qui suit immédiatement un élément spécifié.

p + p {
  color:red;
}

<p>My text is not red</p>
<p>My text is red</p>
<p>My text is red</p>
<hr>
<p>My text is not red</p>

Démo en direct sur JSBin

L’exemple ci-dessus ne sélectionne que les éléments <p> qui sont directement précédés d’un autre élément <p>.

Combinateur général des frères et sœurs : selector ~ selector #

Le combinateur général de frères et sœurs (~) sélectionne tous les frères et sœurs qui suivent l’élément spécifié.

p ~ p {
  color:red;
}

<p>My text is not red</p>
<p>My text is red</p>
<hr>
<h1>And now a title</h1>
<p>My text is red</p>

Démo en direct sur JSBin

L’exemple ci-dessus sélectionne tous les éléments <p> qui sont précédés d’un autre élément <p>, qu’ils soient ou non immédiatement adjacents.

Pseudo-classes #

Les [pseudo-classes][1] sont des mots-clés qui permettent une sélection basée sur des informations qui se trouvent en dehors de l’arborescence du document ou qui ne peuvent pas être exprimées par d’autres sélecteurs ou combinateurs. Ces informations peuvent être associées à un certain état (pseudo-classes [state][2] et [dynamic][3]), à des localisations (pseudo-classes [structural][4] et [target][5]), à négations de la première (pseudo-classe [négation][6]) ou aux langues (pseudo-classe [lang][7]). Les exemples incluent si un lien a été suivi ou non (:visited), la souris est sur un élément (:hover), une case à cocher est cochée (:checked), etc.

Syntaxe #

selector:pseudo-class { 
    property: value;
}

Liste des pseudo-classes : #

Nom Descriptif
[:active][8] S’applique à tout élément activé (c’est-à-dire cliqué) par l’utilisateur.
[:tout][9] Vous permet de créer des ensembles de sélecteurs associés en créant des groupes auxquels les éléments
inclus correspondront. C’est une alternative à la répétition d’un sélecteur entier.
[:cible][10] Sélectionne l’élément #news actuellement actif (cliqué sur une URL
contenant ce nom d’ancre)
[:vérifié][11] S’applique aux éléments de radio, de case à cocher ou d’option qui sont cochés
ou basculés dans un état “on”.
[:default][12] Représente n’importe quel élément d’interface utilisateur qui est la valeur par défaut parmi un groupe d’
éléments similaires.
[:désactivé][13] S’applique à tout élément d’interface utilisateur qui est dans un état désactivé.
[:vide][14] S’applique à tout élément qui n’a pas d’enfant.
[:activé][13] S’applique à tout élément d’interface utilisateur qui est dans un état activé.
[:première][15] Utilisé en conjonction avec la règle @page, cela sélectionne la première page d’un
document imprimé.
[:premier-enfant][16] Représente tout élément qui est le premier élément enfant de son parent.
[:first-of-type][17] S’applique lorsqu’un élément est le premier du type d’élément sélectionné
à l’intérieur de son parent. Cela peut ou non être le premier enfant.
[:focus][8] S’applique à tout élément qui a le focus de l’utilisateur. Cela peut être donné par le clavier de l’utilisateur, les événements de la souris ou d’autres formes d’entrée.
[:focus-within][18] Peut être utilisé pour mettre en surbrillance une section entière lorsqu’un élément à l’intérieur est ciblé. Il correspond à tout élément auquel la pseudo-classe :focus correspond ou qui a un descendant focalisé.
[:plein écran][19] S’applique à tout élément affiché en mode plein écran. Il sélectionne toute la pile
d’éléments et pas seulement l’élément de niveau supérieur.
[:survoler][8] S’applique à tout élément survolé par le dispositif de pointage de l’utilisateur, mais
non activé.
[:indéterminé][20] Applique les éléments d’interface utilisateur radio ou case à cocher qui ne sont ni cochés ni
décochés, mais qui sont dans un état indéterminé. Cela peut être dû à l’attribut d’un élément ou à une manipulation du DOM.
[:in-range][21] La pseudo-classe CSS :in-range correspond lorsqu’un élément a
son attribut de valeur dans les limites de plage spécifiées pour cet élément.
Elle permet à la page de donner un retour sur la valeur actuellement définie< br /> l’utilisation de l’élément est à l’intérieur des limites de la plage.
[:invalide][22] S’applique aux éléments <input> dont les valeurs sont invalides selon
le type spécifié dans l’attribut type=.
[:lang][23] S’applique à tout élément qui enveloppe l’élément <body> avec un attribut lang= correctement
désigné. Pour que la pseudo-classe soit valide, elle doit
contenir un code de langue valide à deux ou trois lettres.
[:dernier-enfant][24] Représente tout élément qui est le dernier élément enfant de son parent.
[:dernier-de-type][25] S’applique lorsqu’un élément est le dernier du type d’élément sélectionné à l’intérieur
de son parent. Cela peut ou non être le dernier enfant.
[:gauche][26] Utilisé en conjonction avec la règle @page, cela sélectionne toutes les
pages gauches d’un document imprimé.
[:lien][27] S’applique à tous les liens qui n’ont pas été visités par l’utilisateur.
[:pas()][28] S’applique à tous les éléments qui ne correspondent pas à la valeur transmise à
(:not(p) ou :not(.class-name) par exemple. Sa valeur doit être< br />valide et ne peut contenir qu’un seul sélecteur. Cependant, vous pouvez enchaîner plusieurs sélecteurs :not ensemble.
[:nth-enfant][29] S’applique lorsqu’un élément est le n-ième élément de son parent, où n
peut être un entier, une expression mathématique (par exemple n+3) ou les mots clés
odd ou “pair”.
[:nth-de-type][30] S’applique lorsqu’un élément est le n-ième élément de son parent du
même type d’élément, où n peut être un entier, une expression mathématique
(par exemple n+3) ou les mots-clés “impair” ou “pair”.
[:unique-enfant][31] La pseudo-classe CSS :only-child représente tout élément
qui est le seul enfant de son parent. C’est la même chose que
:first-child:last-child ou :nth-child(1):nth-last-child(1),
mais avec une spécificité moindre.
[:facultatif][32] La pseudo-classe CSS :optional représente tout élément
pour lequel l’attribut required n’est pas défini. Cela permet
aux formulaires d’indiquer facilement les champs facultatifs et de les styliser en conséquence.
[:hors plage][33] La pseudo-classe CSS :out-of-range correspond lorsqu’un élément a son attribut de valeur en dehors des limites de plage spécifiées pour cet élément.
Elle permet à la page de donner un retour indiquant que la valeur actuellement défini à l’aide de l’élément
est en dehors des limites de la plage. Une valeur peut être en dehors d’une plage si elle est
plus petite ou plus grande que les valeurs maximales et minimales définies.
[:placeholder-shown][34] Expérimental. S’applique à tout élément de formulaire affichant actuellement un texte d’espace réservé.
[:lecture seule][35] S’applique à tout élément non modifiable par l’utilisateur.
[:lecture-écriture][36] S’applique à tout élément modifiable par un utilisateur, tel que les éléments <input>.
[:right][37] Utilisé en conjonction avec la règle @page, cela sélectionne toutes les bonnes pages dans un
document imprimé.
[:root][38] correspond à l’élément racine d’un arbre représentant le document.
[:portée][39] La pseudo-classe CSS correspond aux éléments qui constituent un
point de référence pour les sélecteurs.
[:cible][10] Sélectionne l’élément #news actuellement actif (cliqué sur une URL
contenant ce nom d’ancre)
[:visité][27] S’applique à tous les liens qui ont été visités par l’utilisateur.

La pseudo-classe :visited ne peut plus être utilisée pour la plupart des styles dans de nombreux navigateurs modernes car c’est une faille de sécurité. Voir ce [lien][40] pour référence.

Sélecteurs de nom de classe #

Le sélecteur de nom de classe sélectionne tous les éléments avec le nom de classe ciblé. Par exemple, le nom de classe .warning sélectionnerait l’élément <div> suivant :

<div class="warning">
    <p>This would be some warning copy.</p>
</div>

Vous pouvez également combiner des noms de classe pour cibler des éléments plus spécifiquement. Construisons sur l’exemple ci-dessus pour présenter une sélection de classe plus compliquée.

CSS

.important {
    color: orange;
}
.warning {
    color: blue;
}
.warning.important {
    color: red;
}

HTML

<div class="warning">
    <p>This would be some warning copy.</p>
</div>

<div class="important warning">
    <p class="important">This is some really important warning copy.</p>
</div>

Dans cet exemple, tous les éléments avec la classe .warning auront une couleur de texte bleue, les éléments avec la classe .important auront une couleur de texte orange, et tous les éléments qui ont les deux .important et Le nom de classe .warning aura une couleur de texte rouge.

Notez que dans le CSS, la déclaration .warning.important n’avait pas d’espace entre les deux noms de classe. Cela signifie qu’il ne trouvera que les éléments qui contiennent à la fois les noms de classe warning et important dans leur attribut class. Ces noms de classe peuvent être dans n’importe quel ordre sur l’élément.

Si un espace était inclus entre les deux classes dans la déclaration CSS, il ne sélectionnerait que les éléments qui ont des éléments parents avec des noms de classe .warning et des éléments enfants avec des noms de classe .important.

Sélectionnez l’élément en utilisant son ID sans la haute spécificité du sélecteur d’ID #

Cette astuce vous aide à sélectionner un élément en utilisant l’ID comme valeur pour un sélecteur d’attribut afin d’éviter la haute spécificité du sélecteur d’ID.

HTML :

<div id="element">...</div>

CSS

#element { ... } /* High specificity will override many selectors */

[id="element"] { ... } /* Low specificity, can be overridden easily */

Booléen global avec case à cocher : coché et ~ (combinateur général de frères et sœurs) #

Avec le sélecteur ~, vous pouvez facilement implémenter un booléen global accessible sans utiliser JavaScript.

Ajouter un booléen comme case à cocher #

Au tout début de votre document, ajoutez autant de booléens que vous le souhaitez avec un id unique et l’ensemble d’attributs hidden :

<input type="checkbox" id="sidebarShown" hidden />
<input type="checkbox" id="darkThemeUsed" hidden />

<!-- here begins actual content, for example: -->
<div id="container">
    <div id="sidebar">
        <!-- Menu, Search, ... -->
    </div>

    <!-- Some more content ... -->
</div>

<div id="footer">
    <!-- ... -->
</div>

Changer la valeur du booléen #

Vous pouvez basculer le booléen en ajoutant un label avec l’attribut for défini :

<label for="sidebarShown">Show/Hide the sidebar!</label>

Accéder à la valeur booléenne avec CSS #

Le sélecteur normal (comme .color-red) spécifie les propriétés par défaut. Ils peuvent être remplacés en suivant les sélecteurs true / false :

/* true: */
<checkbox>:checked ~ [sibling of checkbox & parent of target] <target>

/* false: */
<checkbox>:not(:checked) ~ [sibling of checkbox & parent of target] <target>

Notez que <checkbox>, [sibling ...] et <target> doivent être remplacés par les sélecteurs appropriés. [sibling ...] peut être un sélecteur spécifique, (souvent si vous êtes paresseux) simplement * ou rien si la cible est déjà un frère de la case à cocher.

Des exemples pour la structure HTML ci-dessus seraient :

#sidebarShown:checked ~ #container #sidebar {
    margin-left: 300px;
}

#darkThemeUsed:checked ~ #container,
#darkThemeUsed:checked ~ #footer {
    background: #333;
}

Sélecteurs d’ID #

Les sélecteurs d’ID sélectionnent les éléments DOM avec l’ID ciblé. Pour sélectionner un élément par un ID spécifique dans CSS, le préfixe # est utilisé.

Par exemple, l’élément HTML div suivant…

<div id="exampleID">
    <p>Example</p>
</div>

…peut être sélectionné par #exampleID dans CSS comme indiqué ci-dessous :

#exampleID {
    width: 20px;
}

Remarque : les spécifications HTML n’autorisent pas plusieurs éléments avec le même ID

L’exemple de sélecteur de pseudo-classe :only-child #

La pseudo-classe :only-child CSS représente tout élément qui est le seul enfant de son parent.

HTML :

<div>
  <p>This paragraph is the only child of the div, it will have the color blue</p>
</div>

<div>
  <p>This paragraph is one of the two children of the div</p>
  <p>This paragraph is one of the two children of its parent</p>
</div>

CSS :

p:only-child {
  color: blue;
}

L’exemple ci-dessus sélectionne l’élément <p> qui est l’enfant unique de son parent, dans ce cas un <div>.
