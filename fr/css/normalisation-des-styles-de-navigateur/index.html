Normalisation des styles de navigateur

  • normaliser.css

Chaque navigateur possède un ensemble de styles CSS par défaut qu’il utilise pour rendre les éléments. Ces styles par défaut peuvent ne pas être cohérents d’un navigateur à l’autre car : les spécifications de langage ne sont pas claires, les styles de base doivent donc être interprétés, les navigateurs peuvent ne pas suivre les spécifications données ou les navigateurs peuvent ne pas avoir de styles par défaut pour les nouveaux éléments HTML. Par conséquent, les utilisateurs peuvent vouloir normaliser les styles par défaut sur autant de navigateurs que possible.

La réinitialisation de Meyer, bien qu’efficace, apporte les mêmes modifications à presque tous les éléments couramment utilisés. Cela a pour résultat de polluer les fenêtres de l’inspecteur du navigateur Web avec les mêmes styles appliqués encore et encore, et crée plus de travail pour le navigateur (plus de règles à appliquer à plus d’éléments). La technique Normaliser, en revanche, est beaucoup plus ciblée et moins large. Cela simplifie le travail de la part du navigateur et réduit l’encombrement des outils d’inspection du navigateur.

normaliser.css #

Les navigateurs ont un ensemble de styles CSS par défaut qu’ils utilisent pour le rendu des éléments. Certains de ces styles peuvent même être personnalisés à l’aide des paramètres du navigateur pour modifier les définitions de police et de taille par défaut, par exemple. Les styles contiennent la définition des éléments censés être au niveau du bloc ou en ligne, entre autres choses.

Étant donné que ces styles par défaut bénéficient d’une certaine marge de manœuvre par les spécifications de langage et que les navigateurs peuvent ne pas suivre correctement les spécifications, ils peuvent différer d’un navigateur à l’autre.

C’est là que normalize.css entre en jeu. Il remplace les incohérences les plus courantes et corrige les bogues connus.

Qu’est ce que ça fait #

  • Préserve les valeurs par défaut utiles, contrairement à de nombreuses réinitialisations CSS.
  • Normalise les styles pour un large éventail d’éléments.
  • Corrige les bogues et les incohérences courantes du navigateur.
  • Améliore la convivialité avec des modifications subtiles.
  • Explique ce que fait le code à l’aide de commentaires détaillés.

Ainsi, en incluant normalize.css dans votre projet, votre conception se ressemblera et sera plus cohérente sur différents navigateurs.

Différence avec reset.css #

Vous avez peut-être entendu parler de reset.css. Quelle est la différence entre les deux ?

Alors que normalize.css assure la cohérence en définissant différentes propriétés sur des valeurs par défaut unifiées, reset.css assure la cohérence en supprimant tous les styles de base qu’un navigateur peut appliquer. Bien que cela puisse sembler une bonne idée au début, cela signifie en fait que vous devez écrire toutes les règles vous-même, ce qui va à l’encontre d’une norme solide.

Approches et exemples #

Les réinitialisations CSS adoptent des approches distinctes pour les paramètres par défaut du navigateur. Le CSS de réinitialisation d’Eric Meyer existe depuis un certain temps. Son approche annule de nombreux éléments du navigateur connus pour causer des problèmes dès le départ. Ce qui suit provient de sa version (v2.0 | 20110126) CSS Reset.

html, body, div, span, applet, object, iframe,
h1, h2, h3, h4, h5, h6, p, blockquote, pre,
a, abbr, acronym, address, big, cite, code,
del, dfn, em, img, ins, kbd, q, s, samp,
small, strike, strong, sub, sup, tt, var,
b, u, i, center,
dl, dt, dd, ol, ul, li,
fieldset, form, label, legend,
table, caption, tbody, tfoot, thead, tr, th, td,
article, aside, canvas, details, embed, 
figure, figcaption, footer, header, hgroup, 
menu, nav, output, ruby, section, summary,
time, mark, audio, video {
    margin: 0;
    padding: 0;
    border: 0;
    font-size: 100%;
    font: inherit;
    vertical-align: baseline;
}

[CSS de réinitialisation d’Eric Meyer] [1]

Normaliser CSS d’autre part et traite bon nombre d’entre eux séparément. Voici un exemple de la version (v4.2.0) du code.

/**
 * 1. Change the default font family in all browsers (opinionated).
 * 2. Correct the line height in all browsers.
 * 3. Prevent adjustments of font size after orientation changes in IE and iOS.
 */

/* Document
   ========================================================================== */

html {
  font-family: sans-serif; /* 1 */
  line-height: 1.15; /* 2 */
  -ms-text-size-adjust: 100%; /* 3 */
  -webkit-text-size-adjust: 100%; /* 3 */
}

/* Sections
   ========================================================================== */

/**
 * Remove the margin in all browsers (opinionated).
 */

body {
  margin: 0;
}

/**
 * Add the correct display in IE 9-.
 */

article,
aside,
footer,
header,
nav,
section {
  display: block;
}

/**
 * Correct the font size and margin on `h1` elements within `section` and
 * `article` contexts in Chrome, Firefox, and Safari.
 */

h1 {
  font-size: 2em;
  margin: 0.67em 0;
}
