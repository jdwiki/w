La spécificité CSS vise à promouvoir la concision du code en permettant à un auteur de définir des règles générales de formatage pour un large ensemble d’éléments, puis de les remplacer pour un certain sous-ensemble.

Calcul de la spécificité du sélecteur #

Chaque sélecteur CSS individuel a sa propre valeur de spécificité. Chaque sélecteur dans une séquence augmente la spécificité globale de la séquence. Les sélecteurs appartiennent à l’un des trois groupes de spécificité différents : A, B et c. Lorsque plusieurs séquences de sélecteurs sélectionnent un élément donné, le navigateur utilise les styles appliqués par la séquence avec la spécificité globale la plus élevée.

Groupe Composé de Exemples
A sélecteurs d’identifiant #foo
B sélecteurs de classes
sélecteurs d’attributs
pseudo-classes		 .bar
[title], [colspan="2"]
:hover, :nth-child(2)
c sélecteurs de type
pseudo-éléments		 div, li
::avant, ::première-lettre

Le groupe A est le plus spécifique, suivi du groupe B, puis enfin du groupe c.

Le sélecteur universel (*) et les combinateurs (comme > et ~) n’ont aucune spécificité.

Exemple 1 : Spécificité de différentes séquences de sélecteurs #

#foo #baz {}      /* a=2, b=0, c=0 */

#foo.bar {}       /* a=1, b=1, c=0 */

#foo {}           /* a=1, b=0, c=0 */

.bar:hover {}     /* a=0, b=2, c=0 */

div.bar {}        /* a=0, b=1, c=1 */

:hover {}         /* a=0, b=1, c=0 */

[title] {}        /* a=0, b=1, c=0 */

.bar {}           /* a=0, b=1, c=0 */

div ul + li {}    /* a=0, b=0, c=3 */

p::after {}       /* a=0, b=0, c=2 */

*::before {}      /* a=0, b=0, c=1 */

::before {}       /* a=0, b=0, c=1 */

div {}            /* a=0, b=0, c=1 */

* {}              /* a=0, b=0, c=0 */

Exemple 2 : Comment la spécificité est utilisée par le navigateur #

Imaginez l’implémentation CSS suivante :

#foo {
  color: blue;
}

.bar {
  color: red;
  background: black;
}

Ici, nous avons un sélecteur d’ID qui déclare color comme blue, et un sélecteur de classe qui déclare color comme red et background comme black.

Un élément avec un ID de #foo et une classe de .bar sera sélectionné par les deux déclarations. Les sélecteurs d’ID ont une spécificité de groupe A et les sélecteurs de classe ont une spécificité de groupe B. Un sélecteur d’ID l’emporte sur n’importe quel nombre de sélecteurs de classe. Pour cette raison, color:blue; du sélecteur #foo et background:black; du sélecteur .bar seront appliqués à l’élément. La plus grande spécificité du sélecteur d’ID fera que le navigateur ignorera la déclaration de couleur du sélecteur .bar.

Imaginez maintenant une implémentation CSS différente :

.bar {
  color: red;
  background: black;
}

.baz {
  background: white;
}

Ici, nous avons deux sélecteurs de classe ; dont l’un déclare color comme red et background comme black, et l’autre déclare background comme white.

Un élément avec à la fois les classes .bar et .baz sera affecté par ces deux déclarations, cependant le problème que nous avons maintenant est que .bar et .baz ont une spécificité de groupe B identique . La nature en cascade de CSS résout cela pour nous : comme .baz est défini après .bar, notre élément se termine par la rouge color de .bar mais le white background de .baz.

Exemple 3 : Comment manipuler la spécificité #

Le dernier extrait de l’exemple 2 ci-dessus peut être manipulé pour s’assurer que la déclaration color de notre sélecteur de classe .bar est utilisée à la place de celle du sélecteur de classe .baz.

.bar {}        /* a=0, b=1, c=0 */
.baz {}        /* a=0, b=1, c=0 */

Le moyen le plus courant d’y parvenir serait de découvrir quels autres sélecteurs peuvent être appliqués à la séquence de sélecteurs .bar. Par exemple, si la classe .bar n’était appliquée qu’aux éléments span, nous pourrions modifier le sélecteur .bar en span.bar. Cela lui donnerait une nouvelle spécificité de groupe C, qui remplacerait l’absence de celle-ci dans le sélecteur “.baz” :

span.bar {}    /* a=0, b=1, c=1 */
.baz {}        /* a=0, b=1, c=0 */

Cependant, il n’est pas toujours possible de trouver un autre sélecteur commun partagé par tout élément utilisant la classe .bar. Pour cette raison, CSS nous permet de dupliquer les sélecteurs pour augmenter la spécificité. Au lieu de simplement .bar, nous pouvons utiliser .bar.bar à la place (Voir The grammar of Selectors, W3C Recommendation) . Cela sélectionne toujours n’importe quel élément avec une classe de .bar, mais a maintenant le double de la spécificité du groupe B :

.bar.bar {}    /* a=0, b=2, c=0 */
.baz {}        /* a=0, b=1, c=0 */

!important et déclarations de style en ligne #

Le drapeau !important sur une déclaration de style et les styles déclarés par l’attribut HTML style sont considérés comme ayant une plus grande spécificité que n’importe quel sélecteur. Si elles existent, la déclaration de style qu’elles affectent annulera les autres déclarations quelle que soit leur spécificité. C’est-à-dire, à moins que vous n’ayez plus d’une déclaration contenant un indicateur !important pour la même propriété qui s’applique au même élément. Ensuite, les règles de spécificité normales s’appliqueront à ces propriétés en référence les unes aux autres.

Parce qu’ils remplacent complètement la spécificité, l’utilisation de !important est mal vue dans la plupart des cas d’utilisation. Il faut l’utiliser le moins possible. Pour garder le code CSS efficace et maintenable à long terme, il est presque toujours préférable d’augmenter la spécificité du sélecteur environnant plutôt que d’utiliser !important.

L’une de ces rares exceptions où !important n’est pas mal vu, c’est lors de l’implémentation de classes d’assistance génériques comme une classe .hidden ou .background-yellow qui sont censées toujours remplacer une ou plusieurs propriétés partout où elles sont rencontrées. Et même alors, vous devez savoir ce que vous faites. La dernière chose que vous souhaitez, lorsque vous écrivez du CSS maintenable, est d’avoir des drapeaux « !important » dans votre CSS.

Une note finale #

Une idée fausse courante à propos de la spécificité CSS est que les valeurs des groupes A, B et c doivent être combinées les unes avec les autres (a=1, b=5, c=1 => 151). Ce n’est pas le cas. Si tel était le cas, avoir 20 d’un sélecteur de groupe B ou c serait suffisant pour remplacer un seul sélecteur de groupe A ou B respectivement. Les trois groupes doivent être considérés comme des niveaux individuels de spécificité. La spécificité ne peut pas être représentée par une valeur unique.

Lors de la création de votre feuille de style CSS, vous devez maintenir la spécificité la plus faible possible. Si vous devez rendre la spécificité un peu plus élevée pour écraser une autre méthode, rendez-la plus élevée mais aussi basse que possible pour la rendre plus élevée. Vous ne devriez pas avoir besoin d’un sélecteur comme celui-ci :

body.page header.container nav div#main-nav li a {}

Cela rend les modifications futures plus difficiles et pollue cette page CSS.

Vous pouvez calculer la spécificité de votre sélecteur [ici][2]

[2] : http://specificity.keegan.st/

La déclaration !importante #

La déclaration !important est utilisée pour remplacer la spécificité habituelle dans une feuille de style en donnant une priorité plus élevée à une règle. Son utilisation est : propriété : valeur !important;

#mydiv {
    font-weight: bold !important;    /* This property won't be overridden
                                       by the rule below */
}

#outerdiv #mydiv {
    font-weight: normal;            /* #mydiv font-weight won't be set to normal
                                    even if it has a higher specificity because
                                    of the !important declaration above */
}

Il est fortement recommandé d’éviter l’utilisation de !important (sauf en cas d’absolue nécessité), car cela perturbera le flux naturel des règles CSS, ce qui peut apporter de l’incertitude dans votre feuille de style. Il est également important de noter que lorsque plusieurs déclarations  !important sont appliquées à la même règle sur un certain élément, celle avec la spécificité la plus élevée sera celle appliquée.

Voici quelques exemples où l’utilisation de la déclaration !important peut être justifiée :

  • Si vos règles ne doivent pas être remplacées par un style en ligne de l’élément qui est écrit dans l’attribut style de l’élément html.
  • Pour donner à l’utilisateur plus de contrôle sur l’accessibilité du Web, comme augmenter ou diminuer la taille de la taille de la police, en remplaçant le style de l’auteur en utilisant !important.
  • Pour tester et déboguer à l’aide de l’élément inspect.

Voir également:

  • [W3C - 6 Affectation de valeurs de propriété, mise en cascade et héritage – 6.4.2 !règles importantes][1]

[1] : https://www.w3.org/TR/CSS22/cascade.html#important-rules

Cascade #

La cascade et la spécificité sont utilisées ensemble pour déterminer la valeur finale d’une propriété de style CSS. Ils définissent également les mécanismes de résolution des conflits dans les ensembles de règles CSS.

Ordre de chargement CSS #

Les styles sont lus à partir des sources suivantes, dans cet ordre :

  1. Feuille de style de l’agent utilisateur (les styles fournis par le fournisseur du navigateur)
  2. Feuille de style utilisateur (Le style supplémentaire qu’un utilisateur a défini sur son navigateur)
  3. Feuille de style de l’auteur (Auteur signifie ici le créateur de la page Web/du site Web)
  • Peut-être un ou plusieurs fichiers .css
  • Dans l’élément <style> du document HTML
  1. Styles en ligne (dans l’attribut style sur un élément HTML)

Le navigateur recherchera le(s) style(s) correspondant(s) lors du rendu d’un élément.

Comment les conflits sont-ils résolus ? #

Lorsqu’un seul ensemble de règles CSS essaie de définir un style pour un élément, il n’y a pas de conflit et cet ensemble de règles est utilisé.

Lorsque plusieurs ensembles de règles sont trouvés avec des paramètres en conflit, d’abord les règles de spécificité, puis les règles de cascade sont utilisées pour déterminer le style à utiliser.

Exemple 1 - Règles de spécificité #

.mystyle { color: blue; }  /* specificity: 0, 0, 1, 0 */
div { color: red; }        /* specificity: 0, 0, 0, 1 */

<div class="mystyle">Hello World</div>

De quelle couleur sera le texte ? (survolez pour voir la réponse)

 ! bleu

D’abord, les règles de spécificité sont appliquées, et celle qui a la spécificité la plus élevée “gagne”.

Exemple 2 - Règles en cascade avec des sélecteurs identiques #

Fichier css externe

.class {
  background: #FFF;
}

Css interne (dans le fichier HTML)

<style>
.class {
  background: #000;
}
<style>

Dans ce cas, où vous avez des sélecteurs identiques, la cascade se déclenche et détermine que le dernier chargé “gagne”.

Exemple 3 - Règles en cascade après les règles de spécificité #

body > .mystyle { background-color: blue; }   /* specificity: 0, 0, 1, 1 */
.otherstyle > div { background-color: red; }  /* specificity: 0, 0, 1, 1 */

<body class="otherstyle">
  <div class="mystyle">Hello World</div>
</body>

De quelle couleur sera le fond ?

!rouge

Après avoir appliqué les règles de spécificité, il y a toujours un conflit entre le bleu et le rouge, donc les règles en cascade sont appliquées en plus des règles de spécificité. La cascade examine l’ordre de chargement des règles, que ce soit dans le même fichier .css ou dans la collection de sources de style. Le dernier chargé remplace tous les précédents. Dans ce cas, la règle .otherstyle > div “gagne”.

Une note finale #

  • La spécificité du sélecteur prévaut toujours.
  • Liens de rupture d’ordre de feuille de style.
  • Les styles en ligne l’emportent sur tout.

Exemple de spécificité plus complexe #

div {
    font-size: 7px;
    border: 3px dotted pink;
    background-color: yellow;
    color: purple;
}

body.mystyle > div.myotherstyle {
    font-size: 11px;
    background-color: green;
}

#elmnt1 {
    font-size: 24px;
    border-color: red;
}

.mystyle .myotherstyle {
    font-size: 16px;
    background-color: black;
    color: red;
}

<body class="mystyle">
    <div id="elmnt1" class="myotherstyle">
        Hello, world!
    </div>
</body>

Quelles seront les bordures, les couleurs et les tailles de police du texte ?

taille de police:

 ! font-size: 24;, puisque l’ensemble de règles #elmnt1 a la spécificité la plus élevée pour le <div> en question, chaque propriété ici est définie.

frontière:

 ! Bordure : 3 pixels en pointillés rouges ;. La couleur de bordure red est tirée de l’ensemble de règles #elmnt1, car elle a la spécificité la plus élevée. Les autres propriétés de la bordure, border-thickness et border-style proviennent de l’ensemble de règles div.

Couleur de l’arrière plan:

 ! couleur de fond : vert ;. La background-color est définie dans les ensembles de règles div, body.mystyle > div.myotherstyle et .mystyle .myotherstyle. Les spécificités sont (0, 0, 1) vs (0, 2, 2) vs (0, 2, 0), donc celle du milieu “gagne”.

Couleur:

 ! couleur : rouge ;. La couleur est définie dans les ensembles de règles div et .mystyle .myotherstyle. Ce dernier a la spécificité la plus élevée de (0, 2, 0) et “gagne”.
