Syntaxe #

  • Détourage
  • chemin du clip : <clip-source> | [ <forme_de_base> || <clip-geometry-box> ] | rien
  • Masquage
  • image-masque : [ aucune | <mask-reference> ]#
  • mode masque : [ <mode masque> ]#
  • mask-repeat : [ <repeat-style ]#
  • position du masque : [ <position> ]#
  • mask-clip : [ <geometry-box> | pas de clip ]#
  • mask-origin : [ <geometry-box> ]#
  • taille du masque : [ <bg-size> ]#
  • mask-composite : [ <compositing-operator> ]#
  • masque : [ <mask-reference> <masking-mode> ? || <position> [ / <taille-bg> ] ? || <style de répétition> || <boîte-géométrie> || [ <boîte-géométrie> | pas de clip ] || <opérateur de composition> ]#

Paramètres #

Paramètre Détails
clip-source Une URL qui peut pointer vers un élément SVG en ligne (ou) un élément SVG dans un fichier externe qui contient la définition du chemin du clip.
forme de base Fait référence à un élément parmi inset(), circle(), ellipse() ou polygon(). En utilisant l’une de ces fonctions, le chemin de détourage est défini. Ces fonctions de forme fonctionnent exactement de la même manière que dans Shapes for Floats
clip-géométrie-boîte Cela peut avoir une valeur parmi content-box, padding-box, border-box, margin-box, fill-box, stroke-box, view-box. Lorsque cela est fourni sans aucune valeur pour <basic-shape>, les bords de la boîte correspondante sont utilisés comme chemin de détourage. Lorsqu’il est utilisé avec une <basic-shape>, il agit comme la boîte de référence pour la forme.
masque-référence Il peut s’agir de “none” ou d’une image ou d’une URL de référence vers une source d’image de masque.
style de répétition Cela spécifie comment le masque doit être répété ou carrelé dans les axes X et Y. Les valeurs prises en charge sont repeat-x, repeat-y, repeat, space, round, no-repeat.
mode masque Peut être ‘alpha’ ou ’luminance’ ou ‘auto’ et indique si le masque doit être traité comme un masque alpha ou un masque de luminance. Si aucune valeur n’est fournie et que la référence de masque est une image directe, elle serait considérée comme un masque alpha (ou) si la référence de masque est une URL, elle serait considérée comme un masque de luminance.
poste Cela spécifie la position de chaque calque de masque et est similaire dans son comportement à la propriété background-position. La valeur peut être fournie en syntaxe à 1 valeur (comme top, 10%) ou en syntaxe à 2 valeurs (comme top right, 50% 50%).
boîte-géométrie Ceci précise la case sur laquelle le masque doit être clippé (zone de peinture du masque) ou la case qui doit servir de référence à l’origine du masque (zone de positionnement du masque) selon la propriété. La liste des valeurs possibles est content-box, padding-box, border-box, margin-box, fill-box, stroke-box, view-box. Une explication détaillée du fonctionnement de chacune de ces valeurs est disponible dans la Spécification W3C.
taille-bg Cela représente la taille de chaque couche d’image de masque et a la même syntaxe que background-size. La valeur peut être longueur ou pourcentage ou automatique ou couvrir ou contenir. La longueur, le pourcentage et l’auto peuvent être fournis en tant que valeur unique ou en tant que valeur unique pour chaque axe.
opérateur de compositing Cela peut être n’importe lequel parmi ajouter, soustraire, exclure, multiplier par couche et définit le type d’opération de composition qui doit être utilisée pour cette couche avec celles en dessous. Une explication détaillée de chaque valeur est disponible dans les Spécifications W3C.

CSS Clipping and Masking sont des concepts très nouveaux et la prise en charge de ces propriétés par le navigateur est donc assez faible.

Masques : #

Au moment de la rédaction (juillet 2016), Chrome, Safari et Opera prennent en charge ces propriétés avec le préfixe -webkit-.

Firefox ne nécessite pas de préfixes mais il ne prend en charge les masques que lorsqu’ils sont utilisés avec des éléments SVG mask. Pour les éléments mask SVG intégrés, la syntaxe est mask: url(#msk) tandis que pour l’utilisation d’éléments mask dans un fichier SVG externe, la syntaxe est mask: url('yourfilepath/yourfilename.svg#msk') . #msk dans les deux cas fait référence à l’id de l’élément mask auquel il est fait référence. Comme indiqué dans cette réponse, à l’heure actuelle, Firefox ne prend en charge aucun paramètre autre que mask-reference dans la propriété mask.

Internet Explorer (et Edge) n’offre pas encore de support pour cette propriété.

La propriété mask-mode n’est actuellement prise en charge par aucun navigateur avec ou sans préfixes.

Clip-path : #

Au moment de l’écriture (juillet 2016), Chrome, Safari et Opera prennent en charge clip-path lorsque le chemin est créé à l’aide de formes de base (comme circle, polygon) ou la syntaxe url (#clipper) avec SVG en ligne. Ils ne prennent pas en charge le découpage basé sur des formes faisant partie de fichiers SVG externes. De plus, ils nécessitent la présence du préfixe -webkit.

Firefox ne prend en charge que la syntaxe url() pour clip-path alors qu’Internet Explorer (et Edge) n’offre aucune prise en charge.

Masque simple qui fait passer une image du solide au transparent #

CSS #

div {
  height: 200px;
  width: 200px;
  background: url(http://lorempixel.com/200/200/nature/1);
  mask-image: linear-gradient(to right, white, transparent); 
}

HTML #

<div></div>

Dans l’exemple ci-dessus, il y a un élément avec une image en arrière-plan. Le masque appliqué sur l’image (à l’aide de CSS) donne l’impression qu’elle s’estompe de gauche à droite.

Le masquage est obtenu en utilisant un “dégradé linéaire” qui va du blanc (à gauche) au transparent (à droite) comme masque. Comme il s’agit d’un masque alpha, l’image devient transparente là où le masque est transparent.

Détourage (Cercle) #

CSS : #

div{
  width: 200px;
  height: 200px;
  background: teal;
  clip-path: circle(30% at 50% 50%); /* refer remarks before usage */
}

HTML #

<div></div>

Cet exemple montre comment découper une div dans un cercle. L’élément est découpé dans un cercle dont le rayon est de 30% basé sur les dimensions de la boîte de référence avec son point central au centre de la boîte de référence. Ici, puisqu’aucune <clip-geometry-box> (autrement dit, une boîte de référence) n’est fournie, la border-box de l’élément sera utilisée comme boîte de référence.

La forme du cercle doit avoir un rayon et un centre avec les coordonnées (x,y) :

circle(radius at x y)

[Voir l’exemple][1]

Détourage (Polygone) #

CSS : #

div{
  width:200px;
  height:200px;
  background:teal;
  clip-path: polygon(0 0, 0 100%, 100% 50%); /* refer remarks before usage */
}

HTML : #

<div></div>

Dans l’exemple ci-dessus, un chemin de détourage polygonal est utilisé pour découper l’élément carré (200 x 200) en forme de triangle. La forme de sortie est un triangle car le chemin commence à (c’est-à-dire que les premières coordonnées sont à) 0 0 - qui est le coin supérieur gauche de la boîte, puis va à 0 100% - qui est en bas à gauche coin de la boîte et enfin à 100 % 50 % qui n’est rien d’autre que le point médian droit de la boîte. Ces chemins se ferment automatiquement (c’est-à-dire que le point de départ sera le point d’arrivée) et la forme finale est donc celle d’un triangle.

Cela peut également être utilisé sur un élément avec une image ou un dégradé en arrière-plan.

Utilisation de masques pour découper un trou au milieu d’une image #

CSS #

div {
  width: 200px;
  height: 200px;
  background: url(http://lorempixel.com/200/200/abstract/6);
  mask-image: radial-gradient(circle farthest-side at center, transparent 49%, white 50%); /* check remarks before using */
}

HTML #

Dans l’exemple ci-dessus, un cercle transparent est créé au centre à l’aide du “gradient radial” et celui-ci est ensuite utilisé comme masque pour produire l’effet d’un cercle découpé à partir du centre d’une image.

Utiliser des masques pour créer des images aux formes irrégulières #

CSS #

div { /* check remarks before usage */
  height: 200px;
  width: 400px;
  background-image: url(http://lorempixel.com/400/200/nature/4);
  mask-image: linear-gradient(to top right, transparent 49.5%, white 50.5%), linear-gradient(to top left, transparent 49.5%, white 50.5%), linear-gradient(white, white);
  mask-size: 75% 25%, 25% 25%, 100% 75%;
  mask-position: bottom left, bottom right, top left;
  mask-repeat: no-repeat;
}

HTML #

<div></div>

Dans l’exemple ci-dessus, trois images à « dégradés linéaires » (qui, lorsqu’elles sont placées dans leurs positions appropriées, couvriraient 100 % x 100 % de la taille du conteneur) sont utilisées comme masques pour produire une coupe triangulaire transparente au bas de l’image.
