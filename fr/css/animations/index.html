Syntaxe #

  • transition : <propriété> <durée> <fonction de synchronisation> <délai> ;
  • @keyframes <identifiant>
  • [ [ de | à | <pourcentage> ] [, de | à | <pourcentage> ]* bloc ]*

Paramètres #

Transition  
Paramètre Détails
propriété Soit la propriété CSS sur laquelle effectuer la transition, soit all, qui spécifie toutes les propriétés pouvant faire l’objet d’une transition.
durée Temps de transition, en secondes ou en millisecondes.
fonction de temporisation Spécifie une fonction pour définir comment les valeurs intermédiaires des propriétés sont calculées. Les valeurs courantes sont ease, linear et step-end. Consultez le [easing function cheat-sheet][1] pour en savoir plus.
retard Durée, en secondes ou millisecondes, à attendre avant de jouer l’animation.
   
@images clés  
[ de | à | <pourcentage> ] Vous pouvez soit spécifier une durée définie avec une valeur en pourcentage, soit deux valeurs en pourcentage, c’est-à-dire “10 %, 20 %”, pour une période de temps où les attributs définis de l’image clé sont définis.
bloquer N’importe quelle quantité d’attributs CSS pour l’image clé.

[1] : http://easings.net/

Animations avec images clés #

Pour les animations CSS en plusieurs étapes, vous pouvez créer CSS @keyframes. Les images clés vous permettent de définir plusieurs points d’animation, appelés image clé, pour définir des animations plus complexes.

Exemple de base #

Dans cet exemple, nous allons créer une animation d’arrière-plan de base qui alterne entre toutes les couleurs.

@keyframes rainbow-background {
         0%     { background-color: #ff0000; }
     8.333%     { background-color: #ff8000; }
    16.667%     { background-color: #ffff00; }
    25.000%     { background-color: #80ff00; }
    33.333%     { background-color: #00ff00; }
    41.667%     { background-color: #00ff80; }
    50.000%     { background-color: #00ffff; }
    58.333%     { background-color: #0080ff; }
    66.667%     { background-color: #0000ff; }
    75.000%     { background-color: #8000ff; }
    83.333%     { background-color: #ff00ff; }
    91.667%     { background-color: #ff0080; }
    100.00%     { background-color: #ff0000; }
}

.RainbowBackground {
    animation: rainbow-background 5s infinite;
}

[Afficher le résultat][1]

Il y a quelques choses différentes à noter ici. Tout d’abord, la véritable syntaxe @keyframes.

@keyframes rainbow-background{

Cela définit le nom de l’animation sur rainbow-background.

0%     { background-color: #ff0000; }

Il s’agit de la définition d’une image clé dans l’animation. La première partie, le 0% dans le cas, définit où se trouve l’image clé pendant l’animation. Le “0 %” implique qu’il s’agit de 0 % de la durée totale de l’animation depuis le début.

L’animation effectuera automatiquement une transition entre les images clés. Ainsi, en définissant la couleur d’arrière-plan suivante sur 8,333%, l’animation prendra en douceur 8,333% du temps pour passer d’une image clé à l’autre.

.RainbowBackground {
    animation: rainbow-background 5s infinite;
}

Ce code attache notre animation à tous les éléments qui ont la classe .RainbowBackground.

La propriété animation réelle prend les arguments suivants.

  • animation-name : Le nom de notre animation. Dans ce cas, “arc-en-ciel”
  • animation-duration : durée de l’animation, dans ce cas 5 secondes.
  • animation-iteration-count (facultatif) : le nombre de fois que l’animation sera bouclée. Dans ce cas, l’animation se poursuivra indéfiniment. Par défaut, l’animation jouera une fois.
  • animation-delay (facultatif) : spécifie le temps d’attente avant le démarrage de l’animation. Sa valeur par défaut est de 0 seconde et peut prendre des valeurs négatives. Par exemple, -2s démarrerait l’animation 2 secondes après le début de sa boucle.
  • animation-timing-function (Facultatif) : Spécifie la courbe de vitesse de l’animation. La valeur par défaut est “ease”, où l’animation démarre lentement, s’accélère et se termine lentement.

Dans cet exemple particulier, les images clés 0% et 100% spécifient { background-color: #ff0000; }. Partout où deux ou plusieurs images clés partagent un état, on peut les spécifier dans une seule instruction. Dans ce cas, les deux lignes 0% et 100% pourraient être remplacées par cette seule ligne :

0%, 100%     { background-color: #ff0000; }

Compatibilité entre navigateurs #

Pour les anciens navigateurs basés sur WebKit, vous devrez utiliser le préfixe du fournisseur à la fois dans la déclaration @keyframes et dans la propriété animation, comme ceci :

@-webkit-keyframes{}

-webkit-animation: ...

Animations avec la propriété de transition #

Utile pour les animations simples, la propriété CSS transition permet aux propriétés CSS basées sur des nombres de s’animer entre les états.

Exemple #

.Example{
    height: 100px;
    background: #fff;
}

.Example:hover{
    height: 120px;
    background: #ff0000;
}

[Afficher le résultat][1]

Par défaut, survoler un élément avec la classe .Example ferait immédiatement passer la hauteur de l’élément à 120px et sa couleur d’arrière-plan au rouge (#ff0000).

En ajoutant la propriété transition, nous pouvons faire en sorte que ces changements se produisent au fil du temps :

.Example{
    ...
    transition: all 400ms ease;
}

[Afficher le résultat][2]

La valeur all applique la transition à toutes les propriétés compatibles (basées sur des nombres). Tout nom de propriété compatible (tel que height ou top) peut être remplacé par ce mot-clé.

400ms spécifie la durée de la transition. Dans ce cas, le changement de hauteur de l’élément prendra 400 millisecondes.

Enfin, la valeur « ease » est la fonction d’animation, qui détermine la façon dont l’animation est jouée. “facilité” signifie commencer lentement, accélérer, puis finir lentement à nouveau. Les autres valeurs sont linear, ease-out et ease-in.

Compatibilité entre navigateurs #

La propriété transition est généralement bien prise en charge par tous les principaux navigateurs, à l’exception d’IE 9. Pour les versions antérieures de Firefox et des navigateurs basés sur Webkit, utilisez les préfixes de fournisseur comme suit :

.Example{
    transition:         all 400ms ease;
    -moz-transition:    all 400ms ease;
    -webkit-transition: all 400ms ease;
}

Remarque : La propriété transition peut animer des changements entre deux valeurs numériques, quelle que soit l’unité. Il peut également passer d’une unité à l’autre, comme “100px” à “50vh”. Cependant, il ne peut pas passer d’un nombre à une valeur par défaut ou automatique, comme la transition de la hauteur d’un élément de 100px à auto.

[1] : https://jsfiddle.net/0kcm6rwo/ [2] : https://jsfiddle.net/v2j4ggue/1/

Exemples de syntaxe #

Notre premier exemple de syntaxe montre la propriété abrégée d’animation en utilisant toutes les propriétés/paramètres disponibles :

  animation: 3s         ease-in           1s      2                 reverse     both        paused       slidein;
  /*         duration | timing-function | delay | iteration-count | direction | fill-mode | play-state | name   */

Notre second exemple est un peu plus simple, et montre que certaines propriétés peuvent être omises :

  animation: 3s         linear            1s      slidein;
  /*         duration | timing-function | delay | name   */

Notre troisième exemple montre la déclaration la plus minimale. Notez que le nom de l’animation et la durée de l’animation doivent être déclarés :

  animation: 3s         slidein;
  /*         duration | name */

Il convient également de mentionner que lors de l’utilisation du raccourci d’animation, l’ordre des propriétés fait une différence. Évidemment, le navigateur peut confondre votre durée avec votre retard.

Si la brièveté n’est pas votre truc, vous pouvez également ignorer la propriété abrégée et écrire chaque propriété individuellement :

animation-duration: 3s;
animation-timing-function: ease-in;
animation-delay: 1s;
animation-iteration-count: 2;
animation-direction: reverse;
animation-fill-mode: both;
animation-play-state: paused;
animation-name: slidein;

Augmentation des performances d’animation à l’aide de l’attribut will-change #

Lors de la création d’animations et d’autres actions gourmandes en GPU, il est important de comprendre l’attribut “will-change”.

Les images clés CSS et la propriété transition utilisent l’accélération GPU. Les performances sont augmentées en déchargeant les calculs sur le GPU de l’appareil. Cela se fait en créant des couches de peinture (parties de la page rendues individuellement) qui sont déchargées sur le GPU pour être calculées. La propriété will-change indique au navigateur ce qui sera animé, permettant au navigateur de créer des zones de peinture plus petites, augmentant ainsi les performances.

La propriété will-change accepte une liste de propriétés séparées par des virgules à animer. Par exemple, si vous envisagez de transformer un objet et de modifier son opacité, vous devez spécifier :

.Example{
    ...
    will-change: transform, opacity;
}

Remarque : Utilisez will-change avec parcimonie. La définition de will-change pour chaque élément d’une page peut entraîner des problèmes de performances, car le navigateur peut tenter de créer des couches de peinture pour chaque élément, ce qui augmente considérablement la quantité de traitement effectuée par le GPU.
