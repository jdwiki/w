Exemple de code #

Vous trouverez d’autres exemples de code [ici][azure-servicebus-messaging-samples].

[azure-servicebus-messaging-samples] : https://github.com/Azure-Samples/azure-servicebus-messaging-samples

1. Créer un espace de noms à l’aide du portail Azure #

  1. Connectez-vous au portail Azure Classic.

  2. Dans le volet de navigation de gauche du portail, cliquez sur Service Bus.

  3. Dans le volet inférieur du portail, cliquez sur Créer.

    ![Select Create][select-create]

  4. Dans la boîte de dialogue Ajouter un nouvel espace de noms, saisissez un nom d’espace de noms. Le système vérifie immédiatement si le nom est disponible.

    ![Namespace name][namespace-name]

  5. Après vous être assuré que le nom de l’espace de noms est disponible, choisissez le pays ou la région dans lequel votre espace de noms doit être hébergé.

  6. Laissez les autres champs de la boîte de dialogue avec leurs valeurs par défaut (Messagerie et Niveau standard), puis cliquez sur la coche OK. Le système crée maintenant votre espace de noms et l’active. Vous devrez peut-être attendre plusieurs minutes pendant que le système provisionne des ressources pour votre compte.

    ![Created successfully][created-successfully]

###Obtenir les identifiants

  1. Dans le volet de navigation de gauche, cliquez sur le nœud Service Bus pour afficher la liste des espaces de noms disponibles :

    ![Select service bus][select-service-bus]

  2. Sélectionnez l’espace de noms que vous venez de créer dans la liste affichée :

    ![Select namespace][select-namespace]

  3. Cliquez sur Informations de connexion.

    ![Connection information][connection-information]

  4. Dans le volet Accéder aux informations de connexion, recherchez la chaîne de connexion qui contient la clé SAS et le nom de la clé.

    ![Access connection information][access-connection-information]

  5. Notez la clé ou copiez-la dans le presse-papiers.

2. Créez une file d’attente à l’aide du portail Azure #

  1. Connectez-vous au [portail classique Azure][portail-classique].

  2. Dans le volet de navigation de gauche du portail, cliquez sur Service Bus.

  3. Sélectionnez l’espace de noms dans lequel vous souhaitez créer la file d’attente. Dans ce cas, il s’agit de “mytestns1”.

    ![Selecting a namespace][select-namespace]

  4. Sélectionnez Files d’attente.

    ![Select Queues][select-queue]

  5. Sélectionnez Nouveau dans le coin inférieur gauche, puis sélectionnez Création rapide.

    ![Select New][select-new]

  6. Entrez le Nom de la file d’attente et assurez-vous que l’espace de noms approprié est sélectionné.

  7. Sélectionnez Créer une nouvelle file d’attente.

    ![Create a queue][create-queue]

3. Envoyer des messages à la file d’attente #

Afin d’envoyer des messages à la file d’attente, nous allons écrire une application console C# à l’aide de Visual Studio.

Créer une application console #

  1. Lancez Visual Studio et créez une nouvelle application console.

Ajouter le package Service Bus NuGet #

  1. Faites un clic droit sur le projet nouvellement créé et sélectionnez Gérer les packages NuGet.

  2. Cliquez sur l’onglet Parcourir, puis recherchez « Microsoft Azure Service Bus » et sélectionnez l’élément Microsoft Azure Service Bus. Cliquez sur Installer pour terminer l’installation, puis fermez cette boîte de dialogue.

    ![Select a NuGet package][nuget-pkg]

Écrivez du code pour envoyer un message à la file d’attente #

  1. Ajoutez l’instruction using suivante en haut du fichier Program.cs.

    using Microsoft.ServiceBus.Messaging;

  2. Ajoutez le code suivant à la méthode Main, définissez la variable connectionString comme chaîne de connexion obtenue lors de la création de l’espace de noms et définissez queueName comme nom de file d’attente utilisé lors de la création de la file d’attente. .

    var connectionString = "<Your connection string>";
var queueName = "<Your queue name>";

var client = QueueClient.CreateFromConnectionString(connectionString, queueName);
var message = new BrokeredMessage("This is a test message!");
client.Send(message);

    Here is what your Program.cs should look like.

    using System;
using Microsoft.ServiceBus.Messaging;

namespace GettingStartedWithQueues
{
    class Program
    {
        static void Main(string[] args)
        {
            var connectionString = "<Your connection string>";
            var queueName = "<Your queue name>";

            var client = QueueClient.CreateFromConnectionString(connectionString, queueName);
            var message = new BrokeredMessage("This is a test message!");

            client.Send(message);
        }
    }
}

  3. Exécutez le programme et vérifiez le portail Azure Classic. Notez que la valeur Longueur de la file d’attente doit maintenant être 1.

    ![Queue length][queue-length-send]

[nuget-pkg] : https://raw.githubusercontent.com/Azure/azure-content/master/articles/service-bus/media/service-bus-dotnet-get-started-with-queues/nuget-package. png [queue-length-send] : https://raw.githubusercontent.com/Azure/azure-content/master/articles/service-bus/media/service-bus-dotnet-get-started-with-queues/queue- longueur-envoyer.png

4. Recevoir les messages de la file d’attente #

  1. Créez une nouvelle application console et ajoutez une référence au package Service Bus NuGet, similaire à l’application d’envoi ci-dessus.

  2. Ajoutez l’instruction using suivante en haut du fichier Program.cs.

    using Microsoft.ServiceBus.Messaging;

  3. Ajoutez le code suivant à la méthode Main, définissez la variable connectionString comme chaîne de connexion obtenue lors de la création de l’espace de noms, et définissez queueName comme nom de file d’attente que vous avez utilisé lors de la création de l’espace de noms. file d’attente.

    var connectionString = "";
var queueName = "samplequeue";

var client = QueueClient.CreateFromConnectionString(connectionString, queueName);

client.OnMessage(message =>
{
  Console.WriteLine(String.Format("Message body: {0}", message.GetBody<String>()));
  Console.WriteLine(String.Format("Message id: {0}", message.MessageId));
});

Console.ReadLine();

    Here is what your Program.cs file should look like:

    using System;
using Microsoft.ServiceBus.Messaging;

namespace GettingStartedWithQueues
{
  class Program
  {
    static void Main(string[] args)
    {
      var connectionString = "";
      var queueName = "samplequeue";

      var client = QueueClient.CreateFromConnectionString(connectionString, queueName);

      client.OnMessage(message =>
      {
        Console.WriteLine(String.Format("Message body: {0}", message.GetBody<String>()));
        Console.WriteLine(String.Format("Message id: {0}", message.MessageId));
      });

      Console.ReadLine();
    }
  }
}

  4. Exécutez le programme et vérifiez le portail. Notez que la valeur Longueur de la file d’attente doit maintenant être de 0.

    ![Queue length][queue-length-receive]

Toutes nos félicitations! Vous avez maintenant créé une file d’attente, envoyé un message et reçu un message.
