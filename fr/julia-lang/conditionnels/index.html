Syntaxe #

  • si cond ; corps; fin
  • si cond ; corps; autre; corps; fin
  • si cond ; corps; sinon cond ; corps; autre; fin
  • si cond ; corps; sinon cond ; corps; fin -cond ? si vrai : iffalse
  • cond && si vrai
  • cond || si faux
  • ifelse(cond, iftrue, iffalse)

Tous les opérateurs et fonctions conditionnels impliquent l’utilisation de conditions booléennes (“true” ou “false”). Dans Julia, le type des booléens est Bool. Contrairement à d’autres langages, d’autres types de nombres (comme “1” ou “0”), des chaînes, des tableaux, etc. ne peuvent pas être utilisés directement dans les conditions.

En règle générale, on utilise soit des fonctions de prédicat (fonctions qui renvoient un “Bool”), soit des [opérateurs de comparaison] [1] dans la condition d’un opérateur ou d’une fonction conditionnels.

[1] : https://www.wikiod.com/fr/julia-lang/comparaisons

Opérateurs de court-circuit : && et || #

Pour brancher #

Les opérateurs conditionnels de court-circuit && et || peuvent être utilisés comme remplacements légers pour les constructions suivantes :

  • x && y est équivalent à x ? y : x
  • x || y est équivalent à x ? x : y

Une utilisation des opérateurs de court-circuit est un moyen plus concis de tester une condition et d’effectuer une certaine action en fonction de cette condition. Par exemple, le code suivant utilise l’opérateur && pour générer une erreur si l’argument x est négatif :

function mysqrt(x)
    x < 0 && throw(DomainError("x is negative"))
    x ^ 0.5
end

L’opérateur || peut également être utilisé pour la vérification des erreurs, sauf qu’il déclenche l’erreur à moins que une condition soit vérifiée, au lieu de si la condition est vérifiée :

function halve(x::Integer)
    iseven(x) || throw(DomainError("cannot halve an odd number"))
    x ÷ 2
end

Une autre application utile consiste à fournir une valeur par défaut à un objet, uniquement s’il n’a pas été défini auparavant :

isdefined(:x) || (x = NEW_VALUE)

Ici, cela vérifie si le symbole x est défini (c’est-à-dire s’il y a une valeur assignée à l’objet ‘x’). Si c’est le cas, rien ne se passe. Mais si ce n’est pas le cas, x se verra attribuer NEW_VALUE. Notez que cet exemple ne fonctionnera qu’au niveau supérieur.

Dans des conditions #

Les opérateurs sont également utiles car ils permettent de tester deux conditions, dont la seconde n’est évaluée qu’en fonction du résultat de la première condition. Extrait de la [documentation] Julia (http://docs.julialang.org/en/release-0.4/manual/control-flow/#man-short-circuit-evaluation) :

Dans l’expression a && b, la sous-expression b n’est évaluée que si a est évalué à true

Dans l’expression a || b, la sous-expression b n’est évaluée que si a est évalué à false

Ainsi, alors que a & b et a && b donneront true si a et b sont true, leur comportement si a est false est différent.

Par exemple, supposons que nous souhaitions vérifier si un objet est un nombre positif, alors qu’il est possible qu’il ne soit même pas un nombre. Considérez les différences entre ces deux tentatives d’implémentation :

CheckPositive1(x) = (typeof(x)<:Number) & (x > 0) ? true : false
CheckPositive2(x) = (typeof(x)<:Number) && (x > 0) ? true : false

CheckPositive1("a")
CheckPositive2("a")

CheckPositive1() produira une erreur si un type non numérique lui est fourni comme argument. C’est parce qu’il évalue les deux expressions, quel que soit le résultat de la première, et la seconde expression produira une erreur lorsqu’on essaiera de l’évaluer pour un type non numérique.

CheckPositive2(), cependant, renverra false (plutôt qu’une erreur) si un type non numérique lui est fourni, car la deuxième expression n’est évaluée que si la première est true.

Plusieurs opérateurs de court-circuit peuvent être enchaînés. Par exemple.:

1 > 0 && 2 > 0 && 3 > 5

La fonction ifelse #

shift(x) = ifelse(x > 10, x + 1, x - 1)

Usage:

julia> shift(10)
9

julia> shift(11)
12

julia> shift(-1)
-2

La fonction ifelse évaluera les deux branches, même celle qui n’est pas sélectionnée. Cela peut être utile soit lorsque les branches ont des effets secondaires qui doivent être évalués, soit parce que cela peut être plus rapide si les deux branches elles-mêmes sont bon marché.

expression si… sinon #

Le conditionnel le plus courant dans Julia est l’expression if…else. Par exemple, ci-dessous, nous implémentons l’algorithme euclidien pour calculer le plus grand diviseur commun en utilisant une condition pour gérer le cas de base :

mygcd(a, b) = if a == 0
    abs(b)
else
    mygcd(b % a, a)
end

La forme if…else dans Julia est en fait une expression et a une valeur ; la valeur est l’expression en position de queue (c’est-à-dire la dernière expression) sur la branche qui est prise. Considérez l’exemple d’entrée suivant :

julia> mygcd(0, -10)
10

Ici, ‘a’ est ‘0’ et ‘b’ est ‘-10’. La condition a == 0 est true, donc la première branche est prise. La valeur renvoyée est abs(b) qui est 10.

julia> mygcd(2, 3)
1

Ici, ‘a’ est ‘2’ et ‘b’ est ‘3’. La condition a == 0 est fausse, donc la deuxième branche est prise, et nous calculons mygcd(b % a, a), qui est mygcd(3 % 2, 2). L’opérateur ‘%’ renvoie le reste lorsque ‘3’ est divisé par ‘2’, dans ce cas ‘1’. Ainsi, nous calculons mygcd(1, 2), et cette fois a est 1 et b est 2. Encore une fois, a == 0 est faux, donc la deuxième branche est prise, et nous calculons mygcd(b % a, a), qui est mygcd(0, 1). Cette fois, a == 0 enfin et donc abs(b) est renvoyé, ce qui donne le résultat 1.

instruction if…else #

name = readline()
if startswith(name, "A")
    println("Your name begins with A.")
else
    println("Your name does not begin with A.")
end

Toute expression, telle que l’expression if…else, peut être placée en position d’instruction. Cela ignore sa valeur mais exécute toujours l’expression pour ses effets secondaires.

si instruction #

Comme toute autre expression, la valeur de retour d’une expression if…else peut être ignorée (et donc rejetée). Ceci n’est généralement utile que lorsque le corps de l’expression a des effets secondaires, tels que l’écriture dans un fichier, la mutation de variables ou l’impression à l’écran.

De plus, la branche else d’une expression if…else est facultative. Par exemple, nous pouvons écrire le code suivant pour afficher à l’écran uniquement si une condition particulière est remplie :

second = Dates.second(now())
if iseven(second)
    println("The current second, $second, is even.")
end

Dans l’exemple ci-dessus, nous utilisons les fonctions [heure et date][1] pour obtenir la seconde actuelle ; par exemple, s’il est actuellement 10:55:27, la variable second contiendra 27. Si ce nombre est pair, alors une ligne sera imprimée à l’écran. Sinon, rien ne sera fait.

[1] : https://www.wikiod.com/fr/julia-lang/temps

Opérateur conditionnel ternaire #

pushunique!(A, x) = x in A ? A : push!(A, x)

L’opérateur conditionnel ternaire est une expression moins verbeuse if…else.

La syntaxe est précisément :

[condition] ? [execute if true] : [execute if false]

Dans cet exemple, nous ajoutons x à la collection A uniquement si x n’est pas déjà dans A. Sinon, nous laissons simplement A inchangé.

if instruction avec plusieurs branches #

d = Dates.dayofweek(now())
if d == 7
    println("It is Sunday!")
elseif d == 6
    println("It is Saturday!")
elseif d == 5
    println("Almost the weekend!")
else
    println("Not the weekend yet...")
end

N’importe quel nombre de branches elseif peut être utilisé avec une instruction if, éventuellement avec ou sans une branche else finale. Les conditions ultérieures ne seront évaluées que si toutes les conditions antérieures se sont avérées “fausses”.
