Syntaxe #

  • Pkg.add (paquet)
  • Pkg.checkout(package, branch=“master”)
  • Pkg.clone(url)
  • Pkg.dir (paquet)
  • Pkg.pin (paquet, version)
  • Pkg.rm(paquet)

Paramètres #

Paramètre Détails
Pkg.add(paquet) Téléchargez et installez le package enregistré donné.
Pkg.checkout(paquet, branche) Découvrez la branche donnée pour le package enregistré donné. branch est facultatif et vaut par défaut "master".
Pkg.clone(url) Clonez le référentiel Git à l’URL donnée en tant que package.
Pkg.dir(paquet) Obtenir l’emplacement sur le disque pour le package donné.
Pkg.pin(paquet, version) Force le paquet à rester à la version donnée. version est facultatif et correspond par défaut à la version actuelle du package.
Pkg.rm(paquet) Supprimez le package donné de la liste des packages requis.

Installer, utiliser et supprimer un package enregistré #

Après avoir trouvé un package Julia officiel, il est simple de télécharger et d’installer le package. Tout d’abord, il est recommandé d’actualiser la copie locale de METADATA :

julia> Pkg.update()

Cela garantira que vous obtenez les dernières versions de tous les packages.

Supposons que le package que nous souhaitons installer s’appelle [Currencies.jl][1]. La commande à exécuter pour installer ce package serait :

julia> Pkg.add("Currencies")

Cette commande installera non seulement le package lui-même, mais également toutes ses dépendances.

Si l’installation réussit, vous pouvez [tester que le package fonctionne correctement][2] :

julia> Pkg.test("Currencies")

Ensuite, pour utiliser le package, utilisez

julia> using Currencies

et procédez comme décrit dans la documentation du package, généralement liée ou incluse dans son fichier README.md.

Pour désinstaller un package qui n’est plus nécessaire, utilisez la fonction Pkg.rm :

julia> Pkg.rm("Currencies")

Notez que cela ne supprimera peut-être pas réellement le répertoire du package ; à la place, il marquera simplement le paquet comme n’étant plus nécessaire. Souvent, c’est parfaitement bien - cela vous fera gagner du temps au cas où vous auriez à nouveau besoin du paquet à l’avenir. Mais si nécessaire, pour supprimer physiquement le paquet, appelez la fonction rm, puis appelez Pkg.resolve :

julia> rm(Pkg.dir("Currencies"); recursive=true)

julia> Pkg.resolve()

[1] : https://github.com/JuliaFinance/Currencies.jl [2] : https://www.wikiod.com/fr/julia-lang/tests-unitaires#Tester un paquet

Consultez une branche ou une version différente #

Parfois, la dernière version balisée d’un paquet est boguée ou manque certaines fonctionnalités requises. Les utilisateurs avancés peuvent souhaiter mettre à jour vers la dernière version de développement d’un package (parfois appelé “maître”, nommé d’après le nom habituel d’une [branche][1] de développement dans Git). Les avantages de ceci incluent :

  • Les développeurs qui contribuent à un package doivent contribuer à la dernière version de développement.
  • La dernière version de développement peut avoir des fonctionnalités utiles, des corrections de bogues ou des améliorations de performances.
  • Les utilisateurs signalant un bogue peuvent souhaiter vérifier si un bogue se produit sur la dernière version de développement.

Cependant, l’exécution de la dernière version de développement présente de nombreux inconvénients :

  • La dernière version de développement peut être mal testée et avoir de sérieux bugs.
  • La dernière version de développement peut changer fréquemment, cassant votre code.

Pour extraire la dernière branche de développement d’un package nommé [JSON.jl][2], par exemple, utilisez

Pkg.checkout("JSON")

Pour extraire une branche ou un tag différent (non nommé “maître”), utilisez

Pkg.checkout("JSON", "v0.6.0")

Toutefois, si la balise représente une version, il est généralement préférable d’utiliser

Pkg.pin("JSON", v"0.6.0")

Notez qu’un littéral de version est utilisé ici, pas une chaîne simple. La version Pkg.pin informe le gestionnaire de paquets de la contrainte de version, permettant au gestionnaire de paquets d’offrir des commentaires sur les problèmes qu’elle pourrait causer.

Pour revenir à la dernière version taguée,

Pkg.free("JSON")

[1] : https://www.wikiod.com/fr/git/tortuegit#Ramification [2] : https://github.com/JuliaLang/JSON.jl

Installer un package non enregistré #

Certains packages expérimentaux ne sont pas inclus dans le référentiel de packages METADATA. Ces packages peuvent être installés en clonant directement leurs référentiels Git. Notez qu’il peut y avoir des dépendances de packages non enregistrés qui ne sont pas eux-mêmes enregistrés ; ces dépendances ne peuvent pas être résolues par le gestionnaire de packages et doivent être résolues manuellement. Par exemple, pour installer le package non enregistré OhMyREPL.jl :

Pkg.clone("https://github.com/KristofferC/Tokenize.jl")
Pkg.clone("https://github.com/KristofferC/OhMyREPL.jl")

Ensuite, comme d’habitude, utilisez using pour utiliser le package :

using OhMyREPL
