Syntaxe #

  • f(n) = …
  • fonction f(n) … fin -n::Type
  • x -> …
  • f(n) faire … fin

Outre les fonctions génériques (qui sont les plus courantes), il existe également des fonctions intégrées. Ces fonctions incluent is, isa, typeof, throw et des fonctions similaires. Les fonctions intégrées sont généralement implémentées en C au lieu de Julia, elles ne peuvent donc pas être spécialisées sur les types d’arguments pour la distribution.

## Introduction à la répartition Nous pouvons utiliser la syntaxe :: pour répartir sur le [type][1] d’un argument.

describe(n::Integer) = "integer $n"
describe(n::AbstractFloat) = "floating point $n"

Usage:

julia> describe(10)
"integer 10"

julia> describe(1.0)
"floating point 1.0"

Contrairement à de nombreux langages, qui fournissent généralement une répartition multiple statique ou une répartition unique dynamique, Julia a une répartition multiple dynamique complète. Autrement dit, les fonctions peuvent être spécialisées pour plus d’un argument. Cela s’avère pratique lors de la définition de méthodes spécialisées pour les opérations sur certains types et de méthodes de secours pour d’autres types.

describe(n::Integer, m::Integer) = "integers n=$n and m=$m"
describe(n, m::Integer) = "only m=$m is an integer"
describe(n::Integer, m) = "only n=$n is an integer"

Usage:

julia> describe(10, 'x')
"only n=10 is an integer"

julia> describe('x', 10)
"only m=10 is an integer"

julia> describe(10, 10)
"integers n=10 and m=10"

Arguments facultatifs #

Julia permet aux fonctions de prendre des arguments optionnels. Dans les coulisses, cela est implémenté comme un autre cas particulier d’envoi multiple. Par exemple, résolvons le problème populaire [Fizz Buzz] [2]. Par défaut, nous le ferons pour les nombres dans la plage 1:10, mais nous autoriserons une valeur différente si nécessaire. Nous autoriserons également l’utilisation d’expressions différentes pour « Fizz » ou « Buzz ».

function fizzbuzz(xs=1:10, fizz="Fizz", buzz="Buzz")
    for i in xs
        if i % 15 == 0
            println(fizz, buzz)
        elseif i % 3 == 0
            println(fizz)
        elseif i % 5 == 0
            println(buzz)
        else
            println(i)
        end
    end
end

Si nous inspectons fizzbuzz dans le REPL, il est indiqué qu’il existe quatre méthodes. Une méthode a été créée pour chaque combinaison d’arguments autorisée.

julia> fizzbuzz
fizzbuzz (generic function with 4 methods)

julia> methods(fizzbuzz)
# 4 methods for generic function "fizzbuzz":
fizzbuzz() at REPL[96]:2
fizzbuzz(xs) at REPL[96]:2
fizzbuzz(xs, fizz) at REPL[96]:2
fizzbuzz(xs, fizz, buzz) at REPL[96]:2

Nous pouvons vérifier que nos valeurs par défaut sont utilisées lorsqu’aucun paramètre n’est fourni :

julia> fizzbuzz()
1
2
Fizz
4
Buzz
Fizz
7
8
Fizz
Buzz

mais que les paramètres optionnels sont acceptés et respectés si nous les fournissons :

julia> fizzbuzz(5:8, "fuzz", "bizz")
bizz
fuzz
7
8

Envoi paramétrique #

Il arrive souvent qu’une fonction doive être distribuée sur des types paramétriques, tels que [Vector{T}][3] ou Dict{K,V}, mais les paramètres de type ne sont pas fixes. Ce cas peut être traité en utilisant la répartition paramétrique :

julia> foo{T<:Number}(xs::Vector{T}) = @show xs .+ 1
foo (generic function with 1 method)

julia> foo(xs::Vector) = @show xs
foo (generic function with 2 methods)

julia> foo([1, 2, 3])
xs .+ 1 = [2,3,4]
3-element Array{Int64,1}:
 2
 3
 4

julia> foo([1.0, 2.0, 3.0])
xs .+ 1 = [2.0,3.0,4.0]
3-element Array{Float64,1}:
 2.0
 3.0
 4.0

julia> foo(["x", "y", "z"])
xs = String["x","y","z"]
3-element Array{String,1}:
 "x"
 "y"
 "z"

On peut être tenté d’écrire simplement xs::Vector{Number}. Mais cela ne fonctionne que pour les objets dont le type est explicitement Vector{Number} :

julia> isa(Number[1, 2], Vector{Number})
true

julia> isa(Int[1, 2], Vector{Number})
false

Cela est dû à [l’invariance paramétrique][4] : l’objet Int[1, 2] n’est pas un Vector{Number}, car il ne peut contenir que des Ints, alors qu’un Vector{Number} devrait pouvoir contenir n’importe quel type de nombres.

Écrire du code générique #

Dispatch est une fonctionnalité incroyablement puissante, mais il est souvent préférable d’écrire du code générique qui fonctionne pour tous les types, au lieu de spécialiser le code pour chaque type. L’écriture de code générique évite la duplication de code.

Par exemple, voici le code pour calculer la somme des carrés d’un vecteur d’entiers :

function sumsq(v::Vector{Int})
    s = 0
    for x in v
        s += x ^ 2
    end
    s
end

Mais ce code ne fonctionne que pour un vecteur de Ints. Cela ne fonctionnera pas sur un UnitRange :

julia> sumsq(1:10)
ERROR: MethodError: no method matching sumsq(::UnitRange{Int64})
Closest candidates are:
  sumsq(::Array{Int64,1}) at REPL[8]:2

Cela ne fonctionnera pas sur un Vector{Float64} :

julia> sumsq([1.0, 2.0])
ERROR: MethodError: no method matching sumsq(::Array{Float64,1})
Closest candidates are:
  sumsq(::Array{Int64,1}) at REPL[8]:2

Une meilleure façon d’écrire cette fonction sumsq devrait être

function sumsq(v::AbstractVector)
    s = zero(eltype(v))
    for x in v
        s += x ^ 2
    end
    s
end

Cela fonctionnera sur les deux cas énumérés ci-dessus. Mais il y a certaines collections dont nous pourrions vouloir additionner les carrés qui ne sont pas du tout des vecteurs, en aucun cas. Par exemple,

julia> sumsq(take(countfrom(1), 100))
ERROR: MethodError: no method matching sumsq(::Base.Take{Base.Count{Int64}})
Closest candidates are:
  sumsq(::Array{Int64,1}) at REPL[8]:2
  sumsq(::AbstractArray{T,1}) at REPL[11]:2

montre que nous ne pouvons pas additionner les carrés d’un [lazy iterable][5].

Une implémentation encore plus générique est simplement

function sumsq(v)
    s = zero(eltype(v))
    for x in v
        s += x ^ 2
    end
    s
end

Ce qui marche dans tous les cas :

julia> sumsq(take(countfrom(1), 100))
338350

C’est le code Julia le plus idiomatique, et il peut gérer toutes sortes de situations. Dans certains autres langages, la suppression des annotations de type peut affecter les performances, mais ce n’est pas le cas dans Julia ; seule la [stabilité du type][6] est importante pour les performances.

[1] : https://www.wikiod.com/fr/julia-lang/les-types [2] : https://www.wikiod.com/fr/julia-lang/pour-les-boucles#Fizz-Buzz [3] : https://www.wikiod.com/fr/julia-lang/tableaux#Vecteurs [4] : https://en.wikipedia.org/wiki/Covariance_and_contravariance_(computer_science)#Arrays [5] : https://www.wikiod.com/fr/julia-lang/iterables#Combiner des itérables paresseux [6] : https://www.wikiod.com/fr/julia-lang/stabilite-des-types

Mettez un nombre au carré #

Voici la syntaxe la plus simple pour définir une fonction :

square(n) = n * n

Pour appeler une fonction, utilisez des parenthèses (sans espace entre les deux) :

julia> square(10)
100

Les fonctions sont des objets dans Julia, et nous pouvons les afficher dans le [REPL][1] comme avec n’importe quel autre objet :

julia> square
square (generic function with 1 method)

Toutes les fonctions Julia sont génériques (autrement appelées [polymorphes] [2]) par défaut. Notre fonction “carré” fonctionne aussi bien avec des valeurs à virgule flottante :

julia> square(2.5)
6.25

…ou encore [matrices][3] :

julia> square([2 4
               2 1])
2×2 Array{Int64,2}:
 12  12
  6   9

[1] : https://www.wikiod.com/fr/julia-lang/repl [2] : https://en.wikipedia.org/wiki/Polymorphism_(science_informatique) [3] : https://www.wikiod.com/fr/julia-lang/tableaux

Fonctions récursives #

Récursivité simple #

En utilisant la récursivité et l’[opérateur conditionnel ternaire][1], nous pouvons créer une implémentation alternative de la fonction “factorielle” intégrée :

myfactorial(n) = n == 0 ? 1 : n * myfactorial(n - 1)

Usage:

julia> myfactorial(10)
3628800

Travailler avec des arbres #

Les fonctions récursives sont souvent plus utiles sur les structures de données, en particulier les structures de données arborescentes. Puisque les [expressions][2] dans Julia sont des structures arborescentes, la récursivité peut être très utile pour la [métaprogrammation][3]. Par exemple, la fonction ci-dessous rassemble un ensemble de toutes les têtes utilisées dans une expression.

heads(ex::Expr) = reduce(∪, Set((ex.head,)), (heads(a) for a in ex.args))
heads(::Any) = Set{Symbol}()

Nous pouvons vérifier que notre fonction fonctionne comme prévu :

julia> heads(:(7 + 4x > 1 > A[0]))
Set(Symbol[:comparison,:ref,:call])

Cette fonction est compacte et utilise une variété de techniques plus avancées, telles que reduce [fonction d’ordre supérieur][4], le type de données Set et les expressions de générateur.

[1] : https://www.wikiod.com/fr/julia-lang/conditionnels#Opérateur conditionnel ternaire [2] : https://www.wikiod.com/fr/julia-lang/expressions [3] : https://www.wikiod.com/fr/julia-lang/metaprogrammation [4] : https://www.wikiod.com/fr/julia-lang/fonctions-dordre-superieur

Factoriel impératif #

Une syntaxe longue est disponible pour définir les fonctions multilignes. Cela peut être utile lorsque nous utilisons des structures impératives telles que des boucles. L’expression en position terminale est renvoyée. Par exemple, la fonction ci-dessous utilise une boucle [for][1] pour calculer la [factorielle][2] d’un entier n :

function myfactorial(n)
    fact = one(n)
    for m in 1:n
        fact *= m
    end
    fact
end

Usage:

julia> myfactorial(10)
3628800

Dans les fonctions plus longues, il est courant de voir l’instruction return utilisée. L’instruction return n’est pas nécessaire en position de queue, mais elle est encore parfois utilisée pour plus de clarté. Par exemple, une autre façon d’écrire la fonction ci-dessus serait

function myfactorial(n)
    fact = one(n)
    for m in 1:n
        fact *= m
    end
    return fact
end

dont le comportement est identique à celui de la fonction ci-dessus.

[1] : https://www.wikiod.com/fr/julia-lang/pour-les-boucles [2] : https://en.wikipedia.org/wiki/Factorial

Fonctions anonymes #

Syntaxe des flèches #

Des fonctions anonymes peuvent être créées en utilisant la syntaxe ->. Ceci est utile pour transmettre des fonctions à des [fonctions d’ordre supérieur][1], telles que la fonction [map][2]. La fonction ci-dessous calcule le carré de chaque nombre dans un [tableau][3] ‘A’.

squareall(A) = map(x -> x ^ 2, A)

Un exemple d’utilisation de cette fonction :

julia> squareall(1:10)
10-element Array{Int64,1}:
   1
   4
   9
  16
  25
  36
  49
  64
  81
 100

Syntaxe multiligne #

Des fonctions anonymes multilignes peuvent être créées en utilisant la syntaxe function. Par exemple, l’exemple suivant calcule les [factorielles][4] des premiers “n” nombres, mais en utilisant une fonction anonyme au lieu de la “factorielle” intégrée.

julia> map(function (n)
               product = one(n)
               for i in 1:n
                   product *= i
               end
               product
           end, 1:10)
10-element Array{Int64,1}:
       1
       2
       6
      24
     120
     720
    5040
   40320
  362880
 3628800

Faire une syntaxe de bloc #

Parce qu’il est si courant de passer une fonction anonyme comme premier argument d’une fonction, il existe une syntaxe de bloc do. La syntaxe

map(A) do x
    x ^ 2
end

est équivalent à

map(x -> x ^ 2, A)

mais le premier peut être plus clair dans de nombreuses situations, en particulier si beaucoup de calculs sont effectués dans la fonction anonyme. La syntaxe de bloc do est particulièrement utile pour [file input and output][5] pour des raisons de gestion des ressources.
