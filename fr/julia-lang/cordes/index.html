Syntaxe #

  • “[chaîne de caractères]”
  • ‘[Valeur scalaire Unicode]’
  • graphèmes([chaîne])

Paramètres #

Paramètre Détails
Pour sprint(f, xs...)
f Une fonction qui prend un objet IO comme premier argument.
xs Zéro ou plusieurs arguments restants à passer à f.

Bonjour le monde! #

Les chaînes dans Julia sont délimitées à l’aide du symbole " :

julia> mystring = "Hello, World!"
"Hello, World!"

Notez que contrairement à d’autres langages, le symbole ' ne peut pas être utilisé à la place. ' définit un character literal ; il s’agit d’un type de données “Char” et ne stockera qu’une seule valeur scalaire Unicode :

julia> 'c'
'c'

julia> 'character'
ERROR: syntax: invalid character literal

On peut extraire les valeurs scalaires unicode d’une chaîne en itérant dessus avec une boucle [for][1] :

julia> for c in "Hello, World!"
           println(c)
       end
H
e
l
l
o
,
 
W
o
r
l
d
!

[1] : https://www.wikiod.com/fr/julia-lang/pour-les-boucles

Graphèmes #

Le type “Char” de Julia représente une valeur scalaire Unicode, qui ne correspond que dans certains cas à ce que les humains perçoivent comme un “caractère”. Par exemple, une représentation du caractère é, comme dans résumé, est en fait une combinaison de deux valeurs scalaires Unicode :

julia> collect("é")
2-element Array{Char,1}:
 'e'
 '́'

Les descriptions Unicode de ces points de code sont “LATIN SMALL LETTER E” et “COMBINING ACUTE ACCENT”. Ensemble, ils définissent un seul caractère “humain”, qui est un terme Unicode appelé [graphème] (http://www.unicode.org/glossary/#grapheme). Plus précisément, l’annexe 29 d’Unicode motive la définition d’un grappe de graphèmes car :

Il est important de reconnaître que ce que l’utilisateur considère comme un « caractère » - une unité de base d’un système d’écriture pour une langue - peut ne pas être simplement un seul point de code Unicode. Au lieu de cela, cette unité de base peut être composée de plusieurs points de code Unicode. Pour éviter toute ambiguïté avec l’utilisation informatique du terme caractère, on parle de caractère perçu par l’utilisateur. Par exemple, “G” + accent aigu est un caractère perçu par l’utilisateur : les utilisateurs le considèrent comme un caractère unique, mais il est en fait représenté par deux points de code Unicode. Ces caractères perçus par l’utilisateur sont approximés par ce qu’on appelle un groupe de graphèmes, qui peut être déterminé par programmation.

Julia fournit la fonction “graphèmes” pour itérer sur les clusters de graphèmes dans une chaîne :

julia> for c in graphemes("résumé")
           println(c)
       end
r
é
s
u
m
é

Notez comment le résultat, en imprimant chaque caractère sur sa propre ligne, est meilleur que si nous avions itéré sur les valeurs scalaires Unicode :

julia> for c in "résumé"
           println(c)
       end
r
e

s
u
m
e

En règle générale, lorsque vous travaillez avec des caractères dans un sens perçu par l’utilisateur, il est plus utile de traiter des clusters de graphèmes qu’avec des valeurs scalaires Unicode. Par exemple, supposons que nous voulions écrire une fonction pour calculer la longueur d’un seul mot. Une solution naïve serait d’utiliser

julia> wordlength(word) = length(word)
wordlength (generic function with 1 method)

Nous notons que le résultat est contre-intuitif lorsque le mot comprend des groupes de graphèmes constitués de plusieurs points de code :

julia> wordlength("résumé")
8

Lorsque nous utilisons la définition la plus correcte, en utilisant la fonction graphèmes, nous obtenons le résultat attendu :

julia> wordlength(word) = length(graphemes(word))
wordlength (generic function with 1 method)

julia> wordlength("résumé")
6

Convertir les types numériques en chaînes #

Il existe de nombreuses façons de convertir des types numériques en chaînes dans Julia :

julia> a = 123
123

julia> string(a)
"123"

julia> println(a)
123

La fonction string() peut également prendre plus d’arguments :

julia> string(a, "b")
"123b"

Vous pouvez également insérer (c’est-à-dire interpoler) des entiers (et certains autres types) dans des chaînes en utilisant $ :

julia> MyString = "my integer is $a"
"my integer is 123"

Conseil de performance : Les méthodes ci-dessus peuvent parfois être très pratiques. Mais, si vous allez effectuer de très nombreuses opérations de ce type et que vous êtes préoccupé par la vitesse d’exécution de votre code, le [guide des performances] de Julia (http://docs.julialang.org/en/release-0.4/manual/performance- tips/#avoid-string-interpolation-for-i-o) recommande contre cela, et plutôt en faveur des méthodes ci-dessous :

Vous pouvez fournir plusieurs arguments à print() et println() qui fonctionneront sur eux exactement comme string() fonctionne sur plusieurs arguments :

julia> println(a, "b")
123b

Ou, lors de l’écriture dans un fichier, vous pouvez utiliser de la même manière, par ex.

open("/path/to/MyFile.txt", "w") do file
    println(file, a, "b", 13)
end

ou

file = open("/path/to/MyFile.txt", "a")
println(file, a, "b", 13)
close(file)

Celles-ci sont plus rapides car elles évitent d’avoir à former d’abord une chaîne à partir de morceaux donnés, puis à la sortir (soit sur l’affichage de la console, soit dans un fichier) et à la place de sortir séquentiellement les différents morceaux.

Crédits : réponse basée sur la question SO [Quelle est la meilleure façon de convertir un Int en chaîne dans Julia ?] (http://stackoverflow.com/questions/38676573/whats-the-best-way-to-convert-an- int-to-a-string-in-julia) avec la réponse de Michael Ohlrogge et la contribution de Fengyang Wang

Interpolation de chaîne (insérer la valeur définie par la variable dans la chaîne) #

En Julia, comme dans de nombreux autres langages, il est possible d’interpoler en insérant des valeurs définies par des variables dans des chaînes. Pour un exemple simple :

n = 2
julia> MyString = "there are $n ducks"
"there are 2 ducks"

Nous pouvons utiliser d’autres types que numériques, par ex.

Result = false
julia> println("test results is $Result")
test results is false

Vous pouvez avoir plusieurs interpolations dans une chaîne donnée :

MySubStr = "a32"
MyNum = 123.31
println("$MySubStr  ,   $MyNum")

Conseil de performance L’interpolation est assez pratique. Mais, si vous allez le faire plusieurs fois très rapidement, ce n’est pas le plus efficace. Au lieu de cela, voir [Convert numeric types to strings](https://www.wikiod.com/fr/julia-lang/cordes#Convertir des types numériques en chaînes) pour des suggestions lorsque les performances sont un problème.

Utilisation de sprint pour créer des chaînes avec des fonctions IO #

Les chaînes peuvent être créées à partir de fonctions qui fonctionnent avec des objets IO en utilisant la fonction sprint. Par exemple, la fonction code_llvm accepte un objet IO comme premier argument. Généralement, il est utilisé comme

julia> code_llvm(STDOUT, *, (Int, Int))

define i64 @"jlsys_*_46115"(i64, i64) #0 {
top:
  %2 = mul i64 %1, %0
  ret i64 %2
}

Supposons que nous voulions plutôt cette sortie sous forme de chaîne. Ensuite, nous pouvons simplement faire

julia> sprint(code_llvm, *, (Int, Int))
"\ndefine i64 @\"jlsys_*_46115\"(i64, i64) #0 {\ntop:\n  %2 = mul i64 %1, %0\n  ret i64 %2\n}\n"

julia> println(ans)

define i64 @"jlsys_*_46115"(i64, i64) #0 {
top:
  %2 = mul i64 %1, %0
  ret i64 %2
}

La conversion des résultats de fonctions “interactives” comme code_llvm en chaînes peut être utile pour une analyse automatisée, telle que [tester][1] si le code généré peut avoir régressé.

La fonction sprint est une [fonction d’ordre supérieur] [2] qui prend la fonction opérant sur les objets IO comme premier argument. Dans les coulisses, il crée un IOBuffer dans la RAM, appelle la fonction donnée et prend les données du tampon dans un objet String.

[1] : https://www.wikiod.com/fr/julia-lang/tests-unitaires [2] : https://www.wikiod.com/fr/julia-lang/fonctions-dordre-superieur
