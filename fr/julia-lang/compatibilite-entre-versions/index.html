Syntaxe #

  • en utilisant Compat
  • Chaîne de compatibilité
  • Compat.UTF8String
  • @compat f.(x, y)

Il est parfois très difficile d’obtenir une nouvelle syntaxe pour bien jouer avec plusieurs versions. Comme Julia est toujours en cours de développement actif, il est souvent utile de simplement abandonner la prise en charge des anciennes versions et de cibler uniquement les plus récentes.

Numéros de version #

Julia a une implémentation intégrée de [version sémantique] (http://semver.org/) exposée via le type VersionNumber.

Pour construire un VersionNumber en tant que littéral, la [macro de chaîne] @v_str peut être utilisée :

julia> vers = v"1.2.0"
v"1.2.0"

Alternativement, on peut appeler le constructeur VersionNumber ; notez que le constructeur accepte jusqu’à cinq arguments, mais tous sauf le premier sont facultatifs.

julia> vers2 = VersionNumber(1, 1)
v"1.1.0"

Les numéros de version peuvent être comparés à l’aide d’[opérateurs de comparaison][2], et peuvent donc être triés :

julia> vers2 < vers
true

julia> v"1" < v"0"
false

julia> sort([v"1.0.0", v"1.0.0-dev.100", v"1.0.1"])
3-element Array{VersionNumber,1}:
 v"1.0.0-dev.100"
 v"1.0.0"
 v"1.0.1"

Les numéros de version sont utilisés à plusieurs endroits dans Julia. Par exemple, la constante VERSION est un VersionNumber :

julia> VERSION
v"0.5.0"

Ceci est couramment utilisé pour l’évaluation conditionnelle du code, selon la version de Julia. Par exemple, pour exécuter un code différent sur v0.4 et v0.5, on peut faire

if VERSION < v"0.5"
    println("v0.5 prerelease, v0.4 or older")
else
    println("v0.5 or newer")
end

Chaque [package][3] installé est également associé à un numéro de version actuelle :

julia> Pkg.installed("StatsBase")
v"0.9.0"

[1] : https://www.wikiod.com/fr/julia-lang/macros-de-chaine [2] : https://www.wikiod.com/fr/julia-lang/comparaisons [3] : https://www.wikiod.com/fr/julia-lang/paquets

Utilisation de Compat.jl #

Le package Compat.jl permet d’utiliser certaines nouvelles fonctionnalités et syntaxes de Julia avec les anciennes versions de Julia. Ses fonctionnalités sont documentées sur son README, mais un résumé des applications utiles est donné ci-dessous.

Type de chaîne unifiée #

Dans Julia v0.4, il y avait de nombreux types différents de [chaînes][1]. Ce système était considéré comme trop complexe et déroutant, donc dans Julia v0.5, il ne reste que le type String. Compat permet d’utiliser le type et le constructeur String sur la version 0.4, sous le nom Compat.String. Par exemple, ce code v0.5

buf = IOBuffer()
println(buf, "Hello World!")
String(buf)  # "Hello World!\n"

peut être directement traduit en ce code, qui fonctionne à la fois sur la v0.5 et la v0.4 :

using Compat
buf = IOBuffer()
println(buf, "Hello World!")
Compat.String(buf)  # "Hello World!\n"

Notez qu’il y a quelques mises en garde.

  • Sur la v0.4, Compat.String est typaliasé à ByteString, qui est Union{ASCIIString, UTF8String}. Ainsi, les types avec des champs String ne seront pas stables. Dans ces situations, Compat.UTF8String est conseillé, car cela signifiera String sur v0.5 et UTF8String sur v0.4, qui sont tous deux des types concrets.

  • Il faut faire attention à utiliser Compat.String ou import Compat: String, car String lui-même a une signification sur la v0.4 : c’est un alias obsolète pour AbstractString. Un signe que String a été accidentellement utilisé au lieu de Compat.String est si à tout moment, les avertissements suivants apparaissent :

    WARNING: Base.String is deprecated, use AbstractString instead. likely near no file:0 WARNING: Base.String is deprecated, use AbstractString instead. likely near no file:0

Syntaxe de diffusion compacte #

Julia v0.5 introduit le sucre syntaxique pour broadcast. La syntaxe

f.(x, y)

est abaissé à broadcast(f, x, y). Des exemples d’utilisation de cette syntaxe incluent sin.([1, 2, 3]) pour prendre le sinus de plusieurs nombres à la fois.

En v0.5, la syntaxe peut être utilisée directement :

julia> sin.([1.0, 2.0, 3.0])
3-element Array{Float64,1}:
 0.841471
 0.909297
 0.14112

Cependant, si nous essayons la même chose sur la v0.4, nous obtenons une erreur :

julia> sin.([1.0, 2.0, 3.0])
ERROR: TypeError: getfield: expected Symbol, got Array{Float64,1}

Heureusement, Compat rend également cette nouvelle syntaxe utilisable à partir de la v0.4. Encore une fois, nous ajoutons using Compat. Cette fois, nous entourons l’expression avec la macro @compat :

julia> using Compat

julia> @compat sin.([1.0, 2.0, 3.0])
3-element Array{Float64,1}:
 0.841471
 0.909297
 0.14112
