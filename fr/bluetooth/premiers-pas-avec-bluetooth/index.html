Profils #

La spécification Bluetooth contient plusieurs spécifications de profil. Un profil décrit comment utiliser et implémenter une fonction.

Ils peuvent dépendre les uns des autres, voici une disposition de base des dépendances de profil les plus courantes

[![entrez la description de l’image ici][1]][1]

Tous les profils peuvent être trouvés sur [BT SIG][2], sachez que différentes versions peuvent contenir différentes fonctionnalités. Notez également que certains profils contiennent plusieurs catégories, celles-ci sont parfois facultatives, assurez-vous donc que votre appareil prend en charge la catégorie en question. Voici quelques-uns des profils de smartphone les plus courants et leurs spécifications

A2DP - Profil de distribution audio avancé
Le profil de distribution audio avancé (A2DP) définit les protocoles et les procédures qui réalisent la distribution de contenu audio de haute qualité en mono ou stéréo sur les canaux ACL. Un cas d’utilisation typique est la diffusion de contenu musical depuis un lecteur de musique stéréo vers des écouteurs ou des haut-parleurs. Les données audio sont compressées dans un format approprié pour une utilisation efficace de la bande passante limitée.
Dépendances : GAVDP, GAP

AVRCP - Profil de télécommande audio/vidéo
Le profil de télécommande audio/vidéo (AVRCP) définit les fonctionnalités et les procédures requises pour assurer l’interopérabilité entre les appareils Bluetooth avec des fonctions de contrôle audio/vidéo dans les scénarios de distribution audio/vidéo. Ce profil adopte le modèle de périphérique AV/C et le format de commande pour les messages de contrôle, et ces messages sont transportés par le protocole de transport de contrôle audio/vidéo (AVCTP). Dans ce profil, le contrôleur traduit l’action détectée de l’utilisateur en signal de contrôle A/V, puis le transmet à un appareil Bluetooth distant. En plus de cela, le profil utilise des extensions spécifiques à Bluetooth pour prendre en charge le transfert de métadonnées liées au contenu à transférer entre les appareils Bluetooth. La télécommande décrite dans ce profil est conçue spécifiquement pour le contrôle A/V.
Dépendances : GAP

HFP - Profil mains libres
Ce document définit les protocoles et les procédures qui doivent être utilisés par les appareils mettant en œuvre le profil mains libres. Les exemples les plus courants de tels dispositifs sont les unités mains libres embarquées utilisées avec les téléphones cellulaires ou les casques sans fil portables. Le profil définit comment deux appareils prenant en charge le profil mains libres doivent interagir l’un avec l’autre sur une base point à point. Une mise en œuvre du profil mains libres permet généralement à un casque ou à une unité mains libres intégrée de se connecter, sans fil, à un téléphone cellulaire dans le but d’agir en tant que mécanisme d’entrée et de sortie audio du téléphone cellulaire et de permettre aux fonctions de téléphonie typiques d’être effectué sans accès au téléphone réel.
Dépendances : SPP, GAP

HSP - Profil du casque
Ce profil de casque définit les protocoles et les procédures qui doivent être utilisés par les appareils nécessitant une connexion audio en duplex intégral associée à des commandes de contrôle d’appareil minimales. Les exemples les plus courants de tels appareils sont les casques, les ordinateurs personnels, les PDA et les téléphones cellulaires, bien que la plupart des téléphones cellulaires préfèrent utiliser un profil plus avancé tel que le profil mains libres. Le casque peut être connecté sans fil afin d’agir comme mécanisme d’entrée et de sortie audio de l’appareil, fournissant un son en duplex intégral.
Dépendances : SPP, GAP

PBAP - Profil d’accès au répertoire
Le profil d’accès à l’annuaire téléphonique (PBAP) définit les protocoles et les procédures qui doivent être utilisés par les appareils pour la récupération des objets de l’annuaire téléphonique. Il est basé sur un modèle d’interaction client-serveur dans lequel le périphérique client extrait les objets de l’annuaire téléphonique du périphérique serveur. Ce profil est spécialement conçu pour le cas d’utilisation mains libres (c’est-à-dire mis en œuvre en combinaison avec le “profil mains libres” ou le “profil d’accès SIM”). Il offre de nombreuses fonctionnalités qui permettent une gestion avancée des objets de l’annuaire téléphonique, selon les besoins de l’environnement automobile. En particulier, il est beaucoup plus riche que le profil de poussée d’objet (qui pourrait être utilisé pour pousser une entrée d’annuaire téléphonique au format vCard d’un appareil à un autre). Ce profil peut également être appliqué à d’autres cas d’utilisation où un appareil client doit extraire des objets d’annuaire téléphonique d’un appareil serveur. Notez cependant que ce profil ne permet que la consultation des objets de l’annuaire (lecture seule). Il n’est pas possible de modifier le contenu de l’objet d’origine de l’annuaire téléphonique (lecture/écriture).
Dépendances : GOEP, SPP, GAP
