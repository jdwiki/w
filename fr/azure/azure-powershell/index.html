Connectez-vous à Azure #

Mode classique (gestion des services) :

Add-AzureAccount

Cela vous authentifiera à l’aide d’Azure Active Directory et PowerShell obtient un jeton d’accès qui expire après environ 12 heures. Vous devez donc répéter l’authentification après 12 heures.

Une alternative consiste à exécuter l’applet de commande suivante :

Get-AzurePublishSettingsFile

Cela ouvre une fenêtre de navigateur dans laquelle vous pouvez télécharger un fichier de paramètres de publication. Ce fichier contient un certificat qui permet à PowerShell de s’authentifier. Ensuite, vous pouvez importer votre fichier de paramètres de publication en utilisant ce qui suit :

Import-AzurePublishSettingsFile

N’oubliez pas que le fichier de paramètres de publication contient un certificat avec des privilèges d’administrateur effectifs dans votre abonnement. Gardez-le en lieu sûr ou supprimez-le après l’avoir utilisé.

Gestionnaire de ressources

Côté Resource Manager, nous ne pouvons utiliser l’authentification Azure Active Directory qu’avec les jetons d’accès de 12 heures. Il existe actuellement deux commandes alternatives que vous pouvez utiliser :

Login-AzureRmAccount
Add-AzureRmAccount

Sélection de l’abonnement #

Lorsque vous avez plusieurs abonnements sous votre compte Azure ; il est important que vous sélectionniez celui sur lequel vous souhaitez opérer (et que vous l’utilisiez par défaut) ; pour éviter que des accidents n’arrivent aux ressources sur le mauvais abonnement.

Mode classique

Set-AzureSubscription
Select-AzureSubscription

Gestionnaire de ressources

Select-AzureRmSubscription

Information d’abonnement

Les commandes ci-dessus vous demandent de spécifier des informations (par exemple, l’ID d’abonnement) pour identifier l’abonnement vers lequel vous souhaitez basculer. Pour répertorier ces informations pour les abonnements auxquels vous avez accès, exécutez cette commande :

Get-AzureSubscription

Mode classique vs mode ARM #

Lorsque vous travaillez avec Azure à l’aide de PowerShell, vous devez être conscient de 2 manières différentes et vous verrez une grande partie de la documentation Microsoft faisant référence aux deux :

“Mode classique (Gestion des services)”

Il s’agit de l’ancienne méthode d’exploitation et de gestion d’Azure. Il existe encore certains services dans Azure qui ne peuvent être gérés qu’en mode classique, même si de plus en plus de services évoluent vers le nouveau mode ARM.

Pour lister les modules installés sur vos machines fonctionnant en mode classique vous pouvez procéder comme suit :

Get-Module -ListAvailable Azure.*

“Gestionnaire de ressources (ARM)”

C’est la nouvelle façon de gérer Azure (Basé sur les API REST fournies). La plupart des services que vous pouviez gérer dans Azure à partir du mode classique de Powershell peuvent désormais être gérés à l’aide du nouveau mode, à quelques exceptions près. Cela devrait être la méthode préférée de gestion de vos ressources Azure, sauf si certains services ne sont pas encore pris en charge en mode ARM.

Pour lister les modules installés sur votre machine contenant les commandes pour fonctionner en mode ARM se trouvent sous vous pouvez faire ce qui suit :

Get-Module -ListAvailable AzureRM*

Obtenir la version actuelle d’Azure PowerShell #

Pour déterminer la version d’Azure PowerShell que vous avez installée, exécutez la commande suivante :

Get-Module -ListAvailable -Name Azure -Refresh

Cette commande renvoie la version installée même si vous n’avez pas chargé le module Azure PowerShell dans votre session PowerShell actuelle.

## Manipuler les ressources Azure Les applets de commande Azure vous permettent d’effectuer certaines des mêmes actions sur les ressources Azure via PowerShell que vous utiliseriez avec du code C# ou le portail Azure.

Par exemple, ces étapes vous permettent de télécharger le contenu d’un blob Azure dans un répertoire local :

New-Item -Path .\myblob -ItemType Directory
$context = New-AzureStorageContext -StorageAccountName MyAccountName -StorageAccountKey {key from the Azure portal}
$blob = Get-AzureStorageBlob -Container MyContainerName -Context $context
$blob | Get-AzureStorageBlobContent -Destination .\myblob\

Gestion des gestionnaires de trafic #

Avec Azure PowerShell, vous pouvez obtenir certaines fonctionnalités actuellement indisponibles sur Azure Portal, comme :

  • Reconfigurez tous les terminaux de Traffic Manager en même temps
  • Adressez d’autres services via Azure ResourceId au lieu du nom de domaine, vous n’avez donc pas besoin de définir manuellement l’emplacement pour Azure Endpoints

Conditions préalables #

Pour commencer, vous devez [vous connecter](https://www.wikiod.com/fr/azure/azure-powershell#Connectez-vous à Azure) et [sélectionner un abonnement RM](http://stackoverflow.com/ documentation/azure/3961/azure-powershell/13825/selecting-subscription).

Obtenir le profil TrafficManager #

Les opérations avec les Traffic Managers via PowerShell se font en trois étapes :

  1. Obtenez le profil TM :
    $profile = Get-AzureRmTrafficManagerProfile -ResourceGroupName my-resource-group -Name my-traffic-manager
    Ou créez un nouveau comme [dans cet article](https://docs. microsoft.com/en-us/powershell/resourcemanager/azurerm.trafficmanager/v2.2.0/new-azurermtrafficmanagerprofile).
  2. Explorez et modifiez le profil TM
    Vérifiez les champs $profile et $profile.Endpoints pour voir la configuration de chaque point de terminaison.
  3. Enregistrez les modifications via Set-AzureRmTrafficManagerProfile -TrafficManagerProfile $profile.

Modifier les points de terminaison #

Tous les points de terminaison actuels sont stockés dans la liste $profile.Endpoints, vous pouvez donc les modifier directement par index
$profile.Endpoints[0].Weight = 100
ou par nom
$profile .Endpoints | ?{ $_.Name -eq 'mon-endpoint' } | %{ $_.Poids = 100 }

Pour effacer tous les points de terminaison, utilisez
$profile.Endpoints.Clear()

Pour supprimer un endpoint particulier, utilisez
Remove-AzureRmTrafficManagerEndpointConfig -TrafficManagerProfile $profile -EndpointName 'my-endpoint'

Pour ajouter un nouveau point de terminaison, utilisez
Add-AzureRmTrafficManagerEndpointConfig -TrafficManagerProfile $profile -EndpointName "my-endpoint" -Type AzureEndpoints -TargetResourceId "/subscriptions/00000000-0000-0000-0000-000000000000/resourceGroups/my-resource-group/providers/Microsoft.ClassicCompute/domainNames/ my-azure-service" -EndpointStatus Enabled -Weight 100

Comme vous pouvez le voir, dans le dernier cas, nous avons adressé notre service azur via ResourceId plutôt que le nom de domaine.

Gardez à l’esprit #

Vos modifications apportées à TM et à ses points de terminaison ne sont pas appliquées tant que vous n’appelez pas Set-AzureRmTrafficManagerProfile -TrafficManagerProfile $profile. Cela vous permet de reconfigurer entièrement TM en une seule opération.

Traffic Manager est une implémentation de DNS et l’adresse IP donnée aux clients a un certain temps à vivre (alias TTL, vous pouvez voir sa durée en secondes dans le $ champ profile.Ttl). Ainsi, après avoir reconfiguré TM, certains clients continueront d’utiliser les anciens points de terminaison qu’ils ont mis en cache jusqu’à l’expiration de cette durée de vie.
