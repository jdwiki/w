Paramètres #

Paramètre Détails
État de la réponse 401 si le serveur d’origine requiert une authentification, [407](https://tools.ietf.org/html/rfc7235 #section-3.2) si un proxy intermédiaire requiert une authentification
En-têtes de réponse WWW-Authenticate par le serveur d’origine, [Proxy-Authenticate](https://tools.ietf.org/html /rfc7235#section-4.3) par un proxy intermédiaire
En-têtes de requête Authorization pour l’autorisation contre un serveur d’origine, [Proxy-Authorization](https://tools.ietf.org/html /rfc7235#section-4.4) contre un proxy intermédiaire
Schéma d’authentification Basic pour l’authentification de base, mais d’autres tels que Digest et SPNEGO peuvent être utilisés. Voir le Registre des schémas d’authentification HTTP.
Royaume Un nom de l’espace protégé sur le serveur ; un serveur peut avoir plusieurs de ces espaces, chacun avec un nom et des mécanismes d’authentification distincts.
Identifiants Pour Basic : nom d’utilisateur et mot de passe séparés par deux-points, encodés en base64 ; par exemple, username:password codé en base64 est dXNlcm5hbWU6cGFzc3dvcmQ=

L’authentification de base est définie dans RFC2617. Il peut être utilisé pour s’authentifier sur le serveur d’origine après avoir reçu un 401 Unauthorized ainsi que sur un serveur proxy après un 407 (Proxy Authentication Required). Dans les informations d’identification (décodées), le mot de passe commence après le premier deux-points. Par conséquent, le nom d’utilisateur ne peut pas contenir de deux-points, contrairement au mot de passe.

Authentification de base HTTP #

L’authentification de base HTTP fournit un mécanisme simple d’authentification. Les informations d’identification sont envoyées en texte brut et ne sont donc pas sécurisées par défaut. Une authentification réussie se déroule comme suit.

Le client demande une page dont l’accès est restreint :

GET /secret

Le serveur répond avec le code d’état 401 Unauthorized et demande au client de s’authentifier :

401 Unauthorized
WWW-Authenticate: Basic realm="Secret Page"

Le client envoie l’en-tête Authorization. Les identifiants sont username:password codés en base64 :

GET /secret
Authorization: Basic dXNlcm5hbWU6cGFzc3dvcmQ=

Le serveur accepte les informations d’identification et répond avec le contenu de la page :

HTTP/1.1 200 OK
