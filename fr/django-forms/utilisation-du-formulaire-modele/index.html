Django [ModelForm][1] permet la création d’une classe Form à partir d’un modèle Django.

[1] : https://docs.djangoproject.com/en/1.10/topics/forms/modelforms/#modelform

Rendre les champs non modifiables #

Django 1.9 a ajouté l’attribut [Field.disabled][1] :

L’argument booléen disabled, lorsqu’il est défini sur True, désactive un champ de formulaire en utilisant l’attribut HTML désactivé afin qu’il ne soit pas modifiable par utilisateurs. Même si un utilisateur falsifie la valeur du champ soumis au serveur, elle sera ignorée au profit de la valeur issue du formulaire données initiales.

Et donc il vous suffit de faire :

MyChangeForm(ModelForm): 

    def __init__(self, *args, **kwargs): 
        super(MyChangeForm, self).__init__(*args, **kwargs)                       
        self.fields['<field_to_disable>'].disabled = True

Et créer le formulaire dont vous avez besoin :

MyChangeForm(initial={'<field_to_disable>': "something"})

Avant la version 1.9 vous deviez :

class MyChangeForm(ModelForm):
    def __init__(self, *args, **kwargs):
        super(ItemForm, self).__init__(*args, **kwargs)
        instance = getattr(self, 'instance', None)
    if instance and instance.id:
        self.fields['<field_to_disable>'].required = False
        self.fields['<field_to_disable>'].widget.attrs['disabled'] = True

def clean_<field_to_disable>(self):
    # As shown in the above answer.
    instance = getattr(self, 'instance', None)
    if instance:
        return instance.<field_to_disable>
    else:
        return self.cleaned_data.get('<field_to_disable>', None)

Et créer le formulaire dont vous avez besoin :

MyChangeForm(instance=MyChange.objects.get_or_create(<field_to_disable>="something"))

Cet exemple était basé sur [cette question][2].

[1] : https://docs.djangoproject.com/en/1.9/ref/forms/fields/#disabled [2] : http://stackoverflow.com/questions/324477/in-a-django-form-how-do-i-make-a-field-readonly-or-disabled-so-that-it-cannot

Utilisation du formulaire de modèle Django avec la vue basée sur les classes Django. #

Django Model Form avec la vue [Django Class Based][1] est un moyen classique de créer des pages pour effectuer rapidement des opérations de création/mise à jour dans l’application Django. Dans le formulaire, nous pouvons mettre des méthodes pour exécuter des tâches. C’est une façon plus propre de mettre des tâches dans des formulaires plutôt que de mettre des vues/modèles.

Pour donner un exemple utilisant Django Model Form, nous devons d’abord définir notre modèle.

class MyModel(models.Model):
   name = models.CharField(
          verbose_name = 'Name',
          max_length = 255)

Créons maintenant un formulaire en utilisant ce modèle :

class MyModelForm(forms.ModelForm):

    class Meta:
        model = MyModel
        fields = '__all__'

Ajoutons une méthode pour y imprimer hello world.

class MyModelForm(forms.ModelForm):
    class Meta:
        model = MyModel
        fields = '__all__'
    
    def print_hello_world(self):
         print('Hello World')

Créons un modèle pour afficher le formulaire :

<form method="post" action="">
    {% csrf_token %}
<ul>
    {{ form.as_p }}
</ul>
<input type="submit" value="Submit Form"/>
</form>

Nous allons maintenant utiliser ce formulaire dans trois vues différentes qui vont respectivement créer et mettre à jour les tâches.

from django.views.generic.edit import CreateView, UpdateView
from myapp.models import MyModel

class MyModelCreate(CreateView):
    model = MyModel
    fields = ['name']
    form_class = MyModelForm
    template_name = 'my_template.html'

    def form_valid(self, form):
        # This method is called when valid form data has been POSTed.
        # It should return an HttpResponse.
        form.print_hello_world() # This method will print hello world in console
        return super(MyModelCreate, self).form_valid(form)
    

class MyModelUpdate(UpdateView):
    model = MyModel
    fields = ['name']
    form_class = MyModelForm
    template_name = 'my_template.html'

Créons maintenant une URL pour accéder à ces vues.

from django.conf.urls import url
from myapp.views import MyModelCreate, MyModelUpdate

urlpatterns = [
    # ...
    url(r'mymodel/add/$', MyModelCreate.as_view(), name='author-add'),
    url(r'mymodel/(?P<pk>[0-9]+)/$', MyModelUpdate.as_view(), name='author-update')
]

Bon, notre travail a été fait. Nous pouvons accéder à l’url : localhost:8000/mymodel/add pour créer une entrée dans le modèle. Accédez également à localhost:8000/mymodel/1 pour mettre à jour cette entrée.
