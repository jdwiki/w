Création de bibliothèques de balises personnalisées à l'aide de JSP 2.0 (sans classes Java)

  • Un exemple basique d’utilisation d’une balise personnalisée sans aucun code Java

Sur cette page

  • Un exemple basique d’utilisation d’une balise personnalisée sans aucun code Java

Afin d’utiliser des balises personnalisées dans JSP, nous avions l’habitude d’avoir des fichiers TLD, ainsi qu’une classe Java pour définir les balises personnalisées, mais après les spécifications JSP 2.0, la même chose peut être obtenue, sans avoir besoin de fichiers de code source, ce qui facilite à maintenir et il peut également être créé par quelqu’un ayant une connaissance de base des balises JSP sans avoir besoin de code java.

Habituellement, pour ajouter du contenu statique à une page HTML, nous utilisons soit

lien balise rel=“importer” (en HTML5) ou jQuery .load("")

pour inclure tout contenu externe. En cas de contenu dynamique, en utilisant JSP, nous utilisons

jsp : inclure

pour inclure un tel contenu. Mais pour les cas, lorsque nous voulons ajouter du contenu personnalisé, qui peut interagir avec le contenu existant, nous utilisons des balises JSP personnalisées. Elles nous permettent de générer du contenu personnalisé, basé sur les paramètres de l’appelant et également de renvoyer des valeurs traitées dans la balise personnalisée vers l’appelant.

Un exemple basique d’utilisation d’une balise personnalisée sans aucun code Java #

Afin de créer une balise personnalisée, nous devons suivre quelques étapes :

  1. Créez un fichier de balises, en définissant les attributs utilisés par celui-ci et toutes les variables qui seront utilisées par la balise

    a. The attributes need to have a name,their type and and required field with a boolean value

b. The variables will be defined with a name along with a certain scope-

NESTED(Available in tag body),

AT_BEGIN(Within tag till end of scope) and 

AT_END(From end of tag till end of scope)

     <%@ attribute name="name" required="true" type="java.lang.String" description="Name of User"  %>
     <%@ attribute name="role" required="true" type="java.lang.String" description="Role of User" %>
     <%@ variable name-given="passBack" scope="AT_BEGIN"%>
     <%@taglib uri="http://java.sun.com/jsp/jstl/core" prefix="c"%>

     <h1>${name}</h1>
     <h2>${role}</h2>

     <%String backToCaller="Back to caller";%>
     <c:set var="passBack" value="Pass back successful"/>

  2. Définissez l’entrée pour importer la balise, où tagdir sera le répertoire contenant le fichier de balises avec l’extension .tag

     <%@ taglib prefix="tags" tagdir="/WEB-INF/tags" %>

  3. Enfin, utilisez la balise, en transmettant les attributs requis, conformément à la définition de la balise. Ici, customTag est le nom du fichier de balises, ainsi que le préfixe tel que défini

     <tags:customTag name="Hello Tag!!" role="I am the boss here"/>

  4. La variable définie à partir de la balise peut être récupérée dans le jsp appelant à l’aide du langage d’expression

    Hello ${passBack}
