Installation ou configuration #

Bienvenue dans le monde des graphiques. Si vous avez des données connectées, vous aurez peut-être besoin de l’un des types de graphiques pour modéliser ces modèles. Il y a plusieurs choses qui peuvent être faites avec les graphiques comme la cartographie des modèles de trafic, la gestion des réseaux de distribution d’eau, l’analyse des médias sociaux, etc… Au fond, nous devons être en mesure de créer des nœuds et des bords avec des données associées à chacun.

NetworkX est une bibliothèque traitant de la création/importation/exportation/manipulation/algorithmes/traçage de bases de données de graphes. Vous pouvez commencer à utiliser plusieurs types de graphiques de réseau. Par exemple : Facebook pourrait placer ses utilisateurs dans un simple graphique()

import networkx as nx
facebook = nx.Graph()  
facebook.add_node('you')
facebook.add_node('me')
if both_agree_to_be_friends('you','me'):
    facebook.add_edge('you','me') #order isn't important here.

Facebook utiliserait un Graph() normal car il n’y a rien de spécial à propos du bord entre les nœuds. Cet exemple Facebook ne peut avoir qu’un seul bord (amitié) entre les nœuds.

Un autre type de graphique serait un graphique orienté. Twitter utiliserait un graphique dirigé car les nœuds ont une direction. Sur Twitter, je peux vous suivre mais vous n’êtes pas obligé de me suivre. Nous pourrions donc représenter cela avec ce code :

import networkx as nx
twitter = nx.DiGraph()
twitter.add_node('you')
twitter.add_node('me')
twitter.add_edge('me','you') #order is important here.

C’est ainsi que Twitter pourrait configurer « moi » pour suivre « vous », mais pas l’inverse.

Il existe également MultiGraphs () et MultiDiGraphs () au cas où vous voudriez plus d’un bord entre deux nœuds. Ces quatre types couvrent une grande variété de problèmes qui peuvent être représentés avec des graphes. Si vous le souhaitez, vous pouvez ajouter un dictionnaire de données aux nœuds et aux arêtes. La [documentation sur ce module][1] est incroyable. Chaque algorithme est bien étudié et bien implémenté. J’espère que vous apprécierez de travailler avec.

Instructions d’installation: Les instructions détaillées sur l’installation de NetworkX sont disponibles [ici][2].

Comme avec tout autre package python, NetworkX peut être installé à l’aide de pip, Miniconda/Anaconda et à partir du code source.

Installation avec pip

pip install networkx

Une tentative sera faite pour trouver et installer une version appropriée de NetworkX qui correspond à votre système d’exploitation et à la version de Python.

Pour utiliser pip, vous devez avoir installé les outils de configuration.

Si vous souhaitez installer la version de développement à partir de GitHub, utilisez la commande

pip install git://github.com/networkx/networkx.git#egg=networkx

Miniconda et Anaconda utilisent conda pour l’installation/les mises à jour logicielles.

NetworkX est [actuellement installé][2] avec [Anaconda][3]. [Miniconda][3] n’est pas fourni avec NetworkX par défaut.

Vous pouvez mettre à jour/installer NetworkX vers la dernière version avec :

conda install networkx

ou si vous souhaitez mettre à jour l’installation de NetworkX, alors

conda update networkx

Installation depuis la source

Archive du fichier source

  1. Téléchargez la source à partir de https://pypi.python.org/pypi/networkx/ or get the [latest version][3].
  2. Déballez et changez directory to the source directory (it should have the files README.txt and setup.py).
  3. Exécutez python setup.py install pour compiler et install
  4. (Facultatif) Exécutez nosestests pour exécuter les tests si vous avez nose installed.

Installation depuis GitHub

  1. Clonez le référentiel NetworkX (voir https://github.com/networkx/networkx/ pour les options)

    git clone https://github.com/networkx/networkx.git

  2. Changez de répertoire pour NetworkX

  3. Exécutez python setup.py install pour compiler et installer

  4. (Facultatif) Exécutez nosetests pour exécuter les tests si nez est installé.

Si vous n’êtes pas autorisé à installer le logiciel sur votre système, vous pouvez l’installer dans un autre répertoire en utilisant les indicateurs --user, --prefix ou --home pour setup.py.

Conditions Pour utiliser NetworkX, vous avez besoin de Python 2.7, 3.3 ou version ultérieure

Forfaits optionnels

  1. NumPy : Fournit une représentation matricielle des graphiques et est utilisé dans certains algorithmes de graphes pour les calculs matriciels hautes performances. (http://scipy.org/Download)
  2. SciPy : Fournit une représentation matricielle clairsemée de graphiques et de nombreux outils scientifiques numériques. (http://scipy.org/Download)
  3. Matplotlib : Fournit un dessin flexible de graphiques. (http://matplotlib.sourceforge.net/)
  4. GraphViz en conjonction avec PyGraphviz (http://pygraphviz.github.io) ou pydotplus (https://github.com/carlos-jenkins/pydotplus) : fournit des algorithmes de dessin et de mise en page de graphes. (http://graphviz.org)
  5. PyYAML : Requis pour la lecture et l’écriture du format YAML. (http://pyyaml.org)

[3] : http://conda.pydata.org/miniconda.html [2] : https://networkx.github.io/documentation/development/install.html

Programme de base pour afficher les nœuds dans matplotlib en utilisant networkx #

import networkx as nx  # importing networkx package
import matplotlib.pyplot as plt # importing matplotlib package and pyplot is for displaying the graph on canvas 
b=nx.Graph()
b.add_node('helloworld')
b.add_node(1)
b.add_node(2)
'''Node can be called by any python-hashable obj like string,number etc'''
nx.draw(b)  #draws the networkx graph containing nodes which are declared till before
plt.show()  # displays the networkx graph on matplotlib canvas

Clarification supplémentaire :

nx.draw(b,nodelist=[1,'helloworld']) #displays the particular nodes which are given by nodelist only 
nx.draw_networkx(b,nodelist=[1,'helloworld']) #displays the node along with its name given by us i.e 1, hello  respectively
