A - Mise en route #

En un mot, notre élément personnalisé devrait

  • Avoir une fonctionnalité de recherche intégrée, qui recherche et marque des lieux sur la carte.
  • Accepter Un attribut appelé location-array , qui est une liste de lieux
  • Avoir un Listener , pour écouter un événement appelé “google-map-ready”. Cet événement est déclenché, lorsque l’élément est chargé.
  • Cet écouteur doit parcourir le tableau d’emplacement et attribuer l’élément suivant dans votre tableau d’emplacement en tant que “requête de recherche” actuelle
  • Avoir une méthode appelée cache
  • Cette méthode met en cache la latitude et la longitude de chaque lieu dans le tableau de localisation, dès que la recherche renvoie un résultat
  • Avoir un modèle Dom-repeat, qui parcourt les résultats mis en cache et dépose un marqueur à chaque emplacement

Ainsi, le but de notre élément personnalisé signifie essentiellement, si vous passez un tableau d’emplacements comme celui-ci :

<g-map location-array='["Norvège","Suède"]'></g-map>

L’élément personnalisé

  • Ne devrait pas seulement marquer la Norvège et la Suède sur la carte, mais,
  • Devrait également marquer toutes les recherches ultérieures sur la carte - c’est-à-dire, si vous recherchez Boston, après que la carte a marqué la Norvège et la Suède, Boston devrait également avoir une épingle sur la carte

Remarque : #

Obtenir une clé API à partir de l’API javascript de Google est assez simple.

Suivez simplement le lien ci-dessous, pour générer votre clé personnelle par projet.

[Clé API Google][2]

À présent, Étant donné que la clé API doit être utilisée, nous avons deux choix.

  • Laisser l’utilisateur le passer comme attribut
  • Avoir une clé API par défaut configurée

Maintenant, j’opterais pour le premier, simplement parce que toute clé API a ses limites imposées. c’est-à-dire qu’il ne peut être utilisé qu’un certain nombre de fois. Ainsi, une clé configurée et livrée avec l’élément que nous développons pourrait bientôt s’épuiser.

Quiconque souhaite utiliser l’élément personnalisé peut générer une nouvelle clé et la transmettre en tant qu’attribut.

ainsi:

<g-map api-key="AIzaSyBLc_T17p91u6ujSpThe2H3nh_8nG2p6FQ"></g-map>

Nous transmettons donc la clé api en tant qu’attribut et dans notre élément personnalisé, nous le lions à l’attribut api-key de l’élément Polymer.

ainsi:

<google-map api-key=[[apiKey]]></google-map>

Donc une fois qu’on passe la clé API, et qu’on rafraichit, on ne voit toujours pas la carte !

Remarque : #

Google maps, besoin d’un peu de css pour le rendu. Nous avons besoin de la hauteur de l’ensemble de cartes explicitement.

Alors, créez un fichier css dans le répertoire css, dites app.css et ajoutez ces lignes

google-map{
 
min-hauteur : 30vmax ;
}

C - Ajout de la capacité de recherche à notre élément personnalisé #

Pour rechercher un lieu, nous utilisons l’élément puissant fourni par Polymer, appelé google-map-search .

Tout ce que vous avez à faire, c’est de passer

  • un objet cartographique et
  • une chaîne de requête à l’élément comme ceci:
<google-map-search map=[[map]] query=[[query]]></google-map-search>

Comment passe-t-on l’objet map ? D’où obtenons-nous cela?

Simple!

Lorsque vous appelez l’élément google-map, liez la carte de propriétés de votre élément personnalisé à la carte de propriétés de l’élément .

Donc, nous invoquons l’élément google-map comme ceci :

<google-map api-key="whatever key you generated for google's javascript API" map={{map}}></google-map>

Remarque : #

Nous effectuons une liaison de données bidirectionnelle pour la carte ci-dessus, lors de l’appel de google-map polymère.

Il est représenté par les accolades {{}} au lieu d’un sens unique reliure indiquée par des accolades[[]].

Quand on fait une liaison bidirectionnelle,

  • Chaque fois que la propriété de carte de notre élément personnalisé g-map change, l’élément google-map est notifié

et,

  • Chaque fois que la propriété de carte de google-map change, nous sommes avertis dans notre g-map d’élément personnalisé.

Ainsi, avec la liaison de données bidirectionnelle en place, chaque fois que l’objet cartographique change dans google-map, notre élément personnalisé est mis à jour avec les modifications.

et nous passons l’objet de la carte en tant qu’attribut à l’élément google-map-search, de sorte que tous les résultats de la recherche sont marqués dans la carte actuelle rendue.

À quoi ressemble le DOM de notre élément personnalisé en ce moment ?

<template id="google-map-custom-element">
 
<google-map-search map=[[map]] query=[[query]]></google-map-search>
<google-map api-key=[[apiKey]] map={{map}}></google-map>
</template>

Nous n’avons jamais ajouté les propriétés de carte et de requête pour notre élément personnalisé !

Ajouter les propriétés dans l’appel d’enregistrement à Polymer

Polymer({
 
    is:"g-map",
 
    properties:{
 
        map:{
 
            type: Object
 
        },
 
        query:{
 
            type:String,
 
            value:""
 
        }
    }
 
});

Maintenant que nous avons enregistré les propriétés, notons que nous

  • Obtiennent la carte actuellement rendue en tant qu’objet - via la liaison de données et la stockent en tant que propriété locale également appelée carte,
  • Avoir une requête de propriété, transmise à google-map-search

Acceptation de l’entrée de l’utilisateur en tant que requête de recherche #

Mais qui nous donne la requête à rechercher ? Se tromper! à l’heure actuelle? PERSONNE.

C’est précisément ce que nous faisons ensuite.

Ajoutons un formulaire de recherche, que l’utilisateur qui rend la carte utiliserait pour saisir une requête. Nous avons besoin d’entrées pour le champ de recherche et nous utilisons l’entrée de fer de Polymer.

Ajouter un formulaire de recherche #

Inclure les éléments de fer nécessaires au début du fichier g-map.html

<link rel=import href=../bower_components/iron-input/iron-input.html>
<link rel=import href=../bower_components/iron-icon/iron-icon.html>
<link rel=import href=../bower_components/iron-icons/iron-icons.html>
<link rel=import href=../bower_components/iron-icons/maps-icons.html>

Nous voulons le formulaire de recherche, dans le coin supérieur gauche, nous ajoutons donc une entrée de fer avec position absolue, et quelques CSS pour l’épingler là.

Donc, enveloppez le DOM pour notre élément personnalisé, dans une div appelée “map-container”, et ajoutez une div enfant appelée “search_form”.

<template id="google-map-custom-element">
    <div class="map-container">
         <google-map-search map="[[map]]" query="[[query]]"></google-map-search>
         <google-map api-key="[[apiKey]]"></google-map>
         <div class="search_form">
             <iron-icon icon="icons:search" on-tap=__search></iron-icon>
             <input is="iron-input" placeholder="Search Places" type=text name="search" bind-value="{{query}}">
         </div>
     </div>
 </template>

Ajoutez quelques CSS de base pour épingler le formulaire de recherche que nous avons écrit en haut à droite de la carte

.search_form{
    position: absolute;
    top:10px;
    right:10px;
}
.search_form iron-icon{
    position: absolute;
    right: 1px;
    top:1px;
    cursor: pointer;
}

Et puis, quand on rafraîchit, on a ça. Le formulaire de recherche en haut à droite

[![entrez la description de l’image ici][1]][1]

Il ne nous reste plus qu’à fournir une entrée à l’élément google-map-search via la propriété query de notre élément personnalisé.

Ainsi, nous lions notre propriété “requête”, à l’entrée de fer dans le formulaire de recherche. ainsi:

<input is="iron-input" placeholder="Search" type=text name="search" bind-value="{{query}}">

Remarque : #

Nous lions la requête de propriété de notre élément personnalisé à l’entrée de recherche en utilisant,

attribut “bind-value” d’un apport en fer.

Ainsi, bind-value={{query}} dans l’entrée de fer ci-dessus, s’assure que tout modifications apportées au texte à l’intérieur de l’entrée de fer, est notifiée à la propriété de la requête.

Avec notre configuration actuelle,

Pour chaque alphabet saisi par un utilisateur dans le champ de recherche, une recherche est effectuée, qui ralentit la carte, et consomme la limite sur notre clé API.

Pourquoi ?

Parce que la requête de propriété de notre élément personnalisé est liée au fer entrée, et chaque lettre tapée dans l’entrée de fer, change la valeur de « query », qui affecte alors le html dans

<google-map-search map="[[map]]" query="[[query]]">

provoquer une recherche

Notre solution est simple ! Liez simplement l’attribut “query” de l’élément google-map-search à une propriété distincte.

Disons que nous créons une propriété “searchQuery” pour notre élément personnalisé.

Ajoutez ceci, sous la propriété “query” dans l’appel Polymer

searchQuery :{
 
    type:String
 
}

Ensuite, modifiez la liaison dans l’appel de l’élément google-map-search dans le DOM de notre élément personnalisé

ainsi:

<google-map-search map="[[map]]" query="[[searchQuery]]">

Enfin, notez que nous avons ajouté une icône de fer (icône de recherche) à notre formulaire de recherche ?

<iron-icon icon="icons:search" on-tap=search></iron-icon>

Ajoutons cette fonction de recherche à notre élément personnalisé. Il est appelé en appuyant sur l’icône de recherche.

Dans notre méthode de recherche, nous attribuons tout ce qui se trouve dans le champ de recherche à l’attribut “requête” de google-map-search !

Ajoutez cette fonction, après l’objet « propriétés » dans l’appel de polymère pour l’enregistrement.

//paste this after, the properties object
 
search: function() {
this.query = this.searchQuery;
}

Très beau! Donc, maintenant, une recherche ne se produit qu’en appuyant sur l’icône de recherche.

Afficher les résultats de la recherche #

Nous avons

  • Rendu une carte
  • Épinglé un formulaire de recherche sur la carte
  • Écrit la fonctionnalité de recherche sur le robinet de l’icône de recherche

Maintenant, nous devons afficher les résultats.

Pour cela, nous devons lier les résultats de l’élément google-map-search à une propriété locale.

Alors, créons une propriété pour notre g-map d’élément personnalisé, et nommons-la résultats duh !

Dans l’appel d’enregistrement Polymer, ajoutez les résultats de propriété de type objet dans la déclaration de propriétés

results:{
 
type:Object
 
}

Et liez la propriété de résultats de google-map-search à la propriété de résultats locaux que vous avez définie ci-dessus.

<google-map-search results={{results}} map=[[map]] query=[[query]]></google-map-search>

Avec cela en place, rafraîchissons la page et recherchons starbucks sur la carte.

[![entrez la description de l’image ici][2]][2]

Rien ne s’est passé!

Eh bien ! nous venons de lier les résultats de google-map-search à la propriété results . Avons-nous écrit quelque chose pour les afficher sur la carte ? NON!

Nous devons le faire.

Ainsi, nous savons que l’élément Polymer google-map-search renvoie un tableau de marqueurs en tant que résultats.

Nous devons parcourir le tableau et placer des marqueurs sur la carte.

Donc, nous écrivons un modèle dom-repeat, dans notre élément , pour afficher les marqueurs pour chaque résultat

ainsi:

<google-map api-key=[[apiKey]] map=[[map]]>
 
    <template is="dom-repeat" items="{{results}}" as="marker">
        <google-map-marker latitude="{{marker.latitude}}" longitude="{{marker.longitude}}">
            <h2>{{marker.name}}</h2>
            <span>{{marker.formatted_address}}</span>
        </google-map-marker>
    </template>
 
</google-map>

Et maintenant, lors de l’actualisation, nous voyons ceci

[![entrez la description de l’image ici][3]][3]

[1] : https://i.stack.imgur.com/X9gYH.jpg [2] : https://i.stack.imgur.com/xG6EV.png [3] : https://i.stack.imgur.com/yCHBc.jpg

Très beau!

Nous avons ajouté avec succès la fonctionnalité de recherche à notre élément personnalisé.

Tout ce que nous devons faire, c’est

écrivez ceci dans notre index.html

<g-map api-key="Whatever API key you generated"></g-map>

Et vous obtenez une carte sur laquelle vous pouvez chercher !

D - Mise en cache des résultats de recherche #

Notre objectif était cependant de

  • faire une liste de lieux
  • marquez-les tous sur la carte et,
  • marquez tout autre lieu de notre choix, en utilisant le champ de recherche

Nous avons un champ de recherche, où l’utilisateur peut rechercher une requête à la fois. Afin de répondre à plusieurs requêtes, nous devons mettre en cache les résultats de chaque requête, avant d’afficher les marqueurs et de les épingler sur la carte.

Alors, ajoutons

  • Une propriété dit des emplacements qui accepte un tableau d’emplacements
  • Un cache de propriété qui stocke les résultats de chaque requête dans le tableau d’emplacement passé à notre élément personnalisé
cache:{
 
type:Object
 
},
 
locations:{
 
type:Object
 
}

Ensuite, nous écrivons une fonction pour mettre en cache chaque résultat de recherche , résultant d’une recherche de chaque lieu dans le tableau d’emplacement.

C’est assez simple !

Nous rassemblons déjà les résultats dans une propriété appelée “résultats”, que nous parcourons ensuite pour afficher les marqueurs.

Tout ce que nous devons faire maintenant, c’est pousser chaque résultat de recherche dans un tableau, et notre cache est terminé !

Nous devons utiliser un événement pour déclencher la mise en cache. Cet événement est appelé “on-google-map-search-result” et il est déclenché une fois que l’élément google-map-search a terminé une recherche.

Ainsi, “on-google-map-search-result”, nous appelons une fonction, disons “cacher”, pour mettre en cache les résultats de la recherche actuelle dans une propriété appelée cache.

<google-map-search map=[[map]] query=[[searchQuery]] on-google-map-search-results=cacher>

Remarque : #

Normalement, pour pousser un élément dans un tableau, on utilise le push vanille méthode

ex : j’ai un nouveau lieu à ajouter à ma liste de lieux que je souhaite marquer, dites New York.

Si le tableau auquel je veux l’ajouter s’appelle locations, j’écrirais

locations.push('New York');

Cependant, dans notre cas, nous devons poussez n’importe quel résultat de google-map-search dans un tableau appelé cache.Then nous devons parcourir le cache avec les derniers résultats inclus.Non le cache existant.

Maintenant, nous utilisons le modèle dom-repeat pour parcourir les résultats et placer des marqueurs sur la carte.

Cependant, si nous passons un tableau d’emplacement comme ceci :

<g-map locations=["Iceland","Argentina","London"]></g-map>

Ensuite, la propriété results de notre élément personnalisé sera écrasée pour chaque élément de ce tableau.

Par conséquent, nous appelons l’événement on-google-map-search-results de la méthode cacher et poussons le résultat actuel dans la propriété cache.

Cela ne peut pas être une poussée de vanille comme dans la note ci-dessus.

Nous devons faire savoir au répéteur DOM que notre cache a été écrasé à chaque nouvelle recherche.

Donc, nous utilisons une poussée spéciale fournie par Polymer, comme ceci :

cacher: function() {
this.push('cache', this.results);
}

Ajoutez cette fonction, après la fonction de recherche que nous avons écrite plus tôt.

La syntaxe implique que le cache de propriété doit être muté, en ajoutant la valeur actuelle des résultats de la propriété et que tous les répéteurs dom utilisant la propriété de cache pour boucler doivent être informés de la mutation.

Enfin, nous modifions le modèle dom-repeat pour faire une boucle dans le cache, au lieu des résultats

Remarque : #

cache est un tableau de tableaux.

chaque élément du tableau de cache, est un tableau de résultats

Donc, notre répéteur dom, sera comme ça:

<template id="resultList" is="dom-repeat" items="[[cache]]">
 
    <template is="dom-repeat" items="[[item]]" as="marker">
 
        <google-map-marker latitude="{{marker.latitude}}" longitude="{{marker.longitude}}">
            <h2>{{marker.name}}</h2>
            <span>{{marker.formatted_address}}</span>
        </google-map-marker>
    </template>
</template>

Très beau! Nous avons l’élément personnalisé tout codé et prêt à être utilisé !

Testons-le en passant un tableau d’emplacements à partir de index.html d’accord ?

<g-map
 
api-key="AIzaSyBLc_T17p91u6ujSpThe2H3nh_8nG2p6FQ"
 
locations='["Boston","NewYork","California","Pennsylvania"]'>
</g-map>

et on y va !

Le polymère Google Map personnalisé complet avec cache intégré #

<link rel=import href="../bower_components/google-map/google-map.html">
<link rel=import href="../bower_components/google-map/google-map-marker.html">
<link rel=import href="../bower_components/google-map/google-map-search.html">
<link rel=import href="../bower_components/google-map/google-map-directions.html">
<link rel=import href="../bower_components/iron-input/iron-input.html">
<link rel=import href="../bower_components/iron-icon/iron-icon.html">
<link rel=import href="../bower_components/iron-icons/iron-icons.html">
<link rel=import href="../bower_components/iron-icons/maps-icons.html">
<dom-module id="g-map">
    <template id="google-map">
        <div class="map-container">
            <google-map-search map="[[map]]" query="[[query]]" results="{{results}}" on-google-map-search-results=cacher>
            </google-map-search>
            <google-map api-key="[[apiKey]]" map="{{map}}" on-google-map-ready="_onMapResolve" fit-to-markers disable-zoom single-info-window>
                <template id="resultList" is="dom-repeat" items="[[cache]]">
 
                    <template is="dom-repeat" items="[[item]]" as="marker">
 
                        <google-map-marker latitude="{{marker.latitude}}" longitude="{{marker.longitude}}">
                            <h2>{{marker.name}}</h2>
                            <span>{{marker.formatted_address}}</span>
                        </google-map-marker>
                    </template>
                </template>
            </google-map>
            <div class="search_form">
                <p>
                    <iron-icon icon=icons:search on-tap=search></iron-icon>
                    <input is=iron-input placeholder="Search" type=text name=start bind-value={{searchQuery}}>
                </p>
            </div>
        </div>
    </template>
    <script>
        Polymer({
            is: "g-map",
            properties: {
                apiKey: {
                    type: String,
                    value: "AIzaSyBLc_T17p91u6ujSpThe2H3nh_8nG2p6FQ"
                },
                map: {
                    type: Object
                },
                query: {
                    type: String,
                    value: ""
                },
                locations: {
                    type: Array,
                    value: []
                },
                cache: {
                    type: Array,
                    value: []
                },
                results: {
                    type: Array,
                    value: function() {
                            return [];
                        }
                    }
            },
            _onMapResolve: function() {
                var search = document.querySelector("google-map-search");
                this.locations.forEach(function(location) {
                    search.query = location;
                })
            },
            search: function() {
                this.query = this.searchQuery;
            },
            cacher: function() {
                this.push('cache', this.results);
            }
        })
    </script>
</dom-module>
