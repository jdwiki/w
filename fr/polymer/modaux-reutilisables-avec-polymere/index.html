Modaux #

Donc, vous voulez ajouter la conception matérielle à votre portefeuille d’affaires ou de carrière, hein ? Vous ne pouvez tout simplement pas résister à l’utilisation d’un Modal ? qui apparaît en un clic sur ces cartes nettes que vous avez l’intention de concevoir, pour chacun de vos projets/produits !

Disons que pour chaque projet que vous avez réalisé ou pour chaque produit que vous avez conçu, vous avez besoin d’une carte de matériau. Chaque carte de ce type doit avoir

  • Une icône, qui fait apparaître une boîte d’informations, qui répertorie toutes les fonctionnalités de
  • votre produit ou projet, en détail. Une icône “partager”, qui partage
  • votre produit ou projet sur les réseaux sociaux Une icône “fork this”, qui ouvre la page github de votre produit/projet

Avec le html natif, et en supposant que vous utilisiez bootstrap ou toute autre disposition de boîte flexible, vous auriez besoin de

  • Écrivez un wrapper div pour chaque nouveau projet avec une ligne de classe
  • Enveloppez votre image de héros pour chaque projet dans une colonne de 12 colonnes de large
  • Ecrire une autre ligne
  • Enveloppez chaque lien (info | partager | fork) dans une colonne de 4 colonnes de large
  • Injectez ces deux rangées dans un récipient

Pour commencer. Une fois que vous avez fait ce qui précède pour tous vos projets, vous devez travailler sur la création de la fenêtre contextuelle requise pour chaque projet. Ainsi,

  • Vous téléchargez des tonnes de javascript et de css (en personnalisant bootstrap pour la fonctionnalité modale)
  • Vous écrivez un Modal avec un identifiant unique, pour différencier les informations de chaque projet (modal-P1 pour le projet 1, modal-P2 pour le projet 2 et ainsi de suite)
  • Vous composez ensuite le html nécessaire, pour afficher les informations de chaque projet, dans son Modal correspondant.

Quels sont nos objectifs ? #

Juste pour réitérer, visuellement, si vous avez deux projets dans votre portfolio, le but est d’avoir des cartes comme ça :

Nous avons donc deux préoccupations pour CHAQUE projet

  • L’image du héros
  • La barre d’outils avec des informations (pop up a Modal), des liens de partage et de fourche

Tout d’abord, abordons ce qui se passe lorsqu’un utilisateur clique sur le lien d’information dans la barre sous chaque projet. Nous avons besoin d’un Modal pour apparaître, avec plus d’informations sur le projet.

Alors, comment configurez-vous exactement un Modal avec bootstrap ?

  • Vous écrivez ceci pour le déclencher, pour chaque projet

  • Vous écrivez ensuite le corps du Modal comme ceci :
<div id="myProject1" class="modal fade" role="dialog">

                 <button type="button" class="close" data-dismiss="modal"></button>
                 <h4 class="modal-title">Project Title</h4>
             </div>

                 <p>Some text in the modal.</p>
             </div>
             <div class="modal-footer">
                 <button type="button" class="btn btn-default" data-dismiss="modal">Close</button>
             </div>
         </div>
     </div>
</div>

et selon le contenu et la longueur de votre description de projet, vous vous perdez en fait un peu en surveillant l’ouverture et la fermeture d’une soupe div ci-dessus dans le corps du Modal !

De plus, ce balisage est carrément fastidieux à composer CHAQUE FOIS que vous avez besoin d’un modal.

Phew! Et ça s’est terminé là ? Non. Vous devez toujours écrire pour chaque projet,

Barre d’outils réutilisable avec modal | Partager | Icônes de fourche - Codage de l’élément #

Jusqu’à présent, nous avons défini à quel point il est facile d’écrire un élément personnalisé qui cache le code HTML désordonné derrière lui et donne à l’utilisateur un balisage facile à écrire et bref.

Il est temps de le coder !

Notre élément personnalisé, pour afficher la barre sous l’image du héros devrait

  • Accepter un lien à partager
  • Accepter un Lien vers le Repo à bifurquer
  • Acceptez un modalId pour différencier les autres Modals.
  • Accepter un titre pour le modal

Enregistrez l’élément personnalisé de la barre d’outils avec Modal | Partager | Icônes de fourche

Enregistrons notre élément personnalisé, qui accepte la liste de propriétés ci-dessus.

<script>
Polymer({
    is:"tool-bar",
    properties:{
        link2Share:{
            type:String,
            value:""
        },
        link2Fork:{
            type:String,
            value:""
        },
        modalId:{
            type:String,
            value:""
        },
       title:{
            type:String,
            value:"myModal"
        }
    }
});
</script>

Très beau! Ensuite, ajoutez le Html dont nous avons besoin.

Remarque : utilisation et grammaire de l’icône de fer #

Si nous avons besoin d’utiliser l’icône d’information pour notre Modal Popup, nous écrivons le balisage comme suit :

<iron-icon icon="icons:info"></iron-icon>

Si nous voulez l’icône de partage de réseau social, puis l’attribut d’icône ci-dessus, devient

icon=”social:partager”

Donc, la signification de la grammaire, j’ai besoin de l’ensemble d’icônes de “SOCIAL” et je veux l’icône PARTAGER de cet ensemble.

Vous pouvez également utiliser font-awesome , auquel cas vous pouvez utiliser,

choisissez ce qui vous convient le mieux.

Une fois les inclusions faites, nous écrivons le modèle de notre élément personnalisé. Nous utiliserons des classeurs à sens unique pour les données dans le balisage de notre élément personnalisé.

Ainsi, le plan html pour notre élément personnalisé est :

<dom-module id="tool-bar">
 
    <template id="tool-box">
 
        <ul class="flex-wrap toolbar">
            <li>
                <iron-icon icon="icons:info" id="anchor-for-[[modalId]]" onclick="clickHandler(event)"></iron-icon>
            </li>
            <li>
                <a href="https://plus.google.com/share?url=[[link2Share]]" target="_blank" ><iron-icon icon="social:share"></iron-icon></a>
            </li>
            <li>
                <a href="[[link2Fork]]" target="_blank"><i class="fa fa-2x fa-code-fork" aria-hidden=true></i></a> 
            </li>
        </ul>
 
    </template>
 
</dom-module>

Cool! Ainsi, la barre d’outils a son modèle ul , qui répertorie les trois liens

  • icône info – Modal Popup
  • partager cette icône - partager sur google
  • Fork cette icône - Fork le dépôt

De plus, nous avons lié les attributs écrits par l’utilisateur aux propriétés respectives.

Remarque : Rendre l’ID modal unique #

Nous avons traité l’unicité de l’icône d’information, en spécifiant l’identifiant attribut comme ça

id="ancre-pour-[[modalId]]" onclick="clickHandler(event)"></iron-icon>

Ainsi, chaque projet aura un identifiant d’ancrage unique.

c’est-à-dire

Si l’utilisateur passe “project-cats” comme modal-id, dans l’appel à notre élément personnalisé , l’identifiant de l’ancre serait « ancre-pour-projet-chats »

Si l’utilisateur ajoute un autre projet et passe “project-puppy” comme attribut modal-id, l’identifiant de l’ancre serait anchor-for-project-puppy

et ainsi de suite.

Mais comment s’assurer que chaque icône d’information est associée à son propre modal ?

L’« icône de fer » pour info, doit se voir attribuer un Attribut “data-dialog”, qui doit également être unique.

L’attribut data-dialog est similaire à l’attribut data-target dans le modal bootstrap. Il doit être unique pour chaque nouveau projet.

Nous laissons à l’utilisateur le soin de maintenir l’unicité. L’utilisateur ne peut donc pas ont le même attribut modal-id pour différents appels <tool-bar>.

Ainsi, comme l’indique la note ci-dessus, nous devons mapper l’icône d’informations de chaque projet sur une boîte de dialogue de données.

Ajoutons cette fonctionnalité.

Nous utilisons la méthode ready.It est très similaire à celle de Jquery

$.prêt();

.Il est invoqué lorsque l’élément personnalisé a fini de se charger.

Ajoutez cette fonction, après l’objet properties, dans l’appel Polymer pour l’enregistrement.

ready: function() {
    document.querySelector("#anchor-for-" + this.modalId).setAttribute('data-dialog', this.modalId);
}

Très cool! Alors maintenant nous avons

  • Un élément personnalisé enregistré en tant qu’élément Polymère -Modalité | Partager | Icônes de fourche ajoutées à l’élément personnalisé

Un Modal est attaché à l’info iron-icon, avec l’id “anchor-for-[Whatever ispassed as modal-id]”. C’est courtoisie la méthode prête de notre élément.

Donc, jusqu’à présent, écrire

<tool-bar
link2-share="http://www.myportfolio.com/project-neighbourhood-map"
link2-fork="https://github.com/me/project-neighbourhood-map"
modal-id="project-neighbourhood-map">
</tool-bar>

[![entrez la description de l’image ici][2]][2]

[1] : http://i.stack.imgur.com/y8sJp.jpg [2] : http://i.stack.imgur.com/2osBg.jpg

et notre objectif était d’envelopper cette description complète du projet, dans la balise <tool-bar> sur la page des utilisateurs.

Vous vous souvenez de Shadow DOM ? Nous avons besoin d’un conteneur dans notre élément personnalisé, qui fournit le shadow DOM pour le contenu de notre Modal. Le contenu lui-même est enveloppé dans une balise <content> à l’intérieur du shadowDOM de l’élément personnalisé.

Pour le conteneur qui héberge la description du Projet, nous utilisons l’élément Polymer “paper-dialog”. Paper-dialog, est assigné comme id, quel que soit ce qui est passé comme modal-id dans la déclaration sur la page de l’utilisateur.

Donc, ajoutez ce balisage, après le ul dans le DOM de l’élément personnalisé de la barre d’outils

        <iron-icon icon="icons:power-settings-new" dialog-dismiss></iron-icon>
    </div>
    <h2 class="text-center text-capitalize">[[title]]</h2>
    <paper-dialog-scrollable>
        <div class="container-fluid">
            <div class="row flex-wrap info">
                <content></content>
            </div>
        </div>
    </paper-dialog-scrollable>
</paper-dialog>

Remarque : Paper-dialog-scrollable #

Nous utilisons Paper-dialog-scrollable, pour envelopper la description de notre projet, donc que, Toutes les longues descriptions, ne débordent pas.

Le dernier élément consiste à ajouter la fonctionnalité onclick lorsque le modal doit s’ouvrir, en cliquant sur l’icône d’information.

Lorsqu’un clic se produit, comme c’est normal, un événement est déclenché, que nous transmettons à notre fonction clickHandler comme ceci :

<iron-icon icon="icons:info" id="anchor-for-[[modalId]]" onclick="clickHandler(event)"></iron-icon>

Donc, une fois que la fonction clickHandler reçoit l’événement qui lui est transmis,

  • Il doit rassembler, quel élément a déclenché le clic,
  • Il doit rassembler l’attribut data-dialog de cet élément pour déterminer quel modal ouvrir (rappelez-vous que chaque projet a son propre identifiant modal ?)
  • Il doit ensuite appeler la méthode open, pour exactement ce modal

Donc, la fonction est comme ceci:

Ajoutez ceci, avant l’appel d’enregistrement de l’élément. Ceci ne fait pas partie de l’élément, mais est étendu à l’intérieur de l’élément.

function clickHandler(e) {
    var button = e.target;
    while (!button.hasAttribute('data-dialog') && button !== document.body) {
        button = button.parentElement;
    }
 
    if (!button.hasAttribute('data-dialog')) {
        return;
    }
 
    var id = button.getAttribute('data-dialog');
    var dialog = document.getElementById(id);
    if (dialog) {
        dialog.open();
    }
}

Donc, avec cela, nous avons terminé le codage de l’élément personnalisé, <tool-bar>.

             },
