Une fois votre projet terminé, vous vous demandez comment allez-vous télécharger ces 100 importations HTML sur votre serveur Web et même si vous le faites, combien d’heures votre site va prendre pour se charger pour un seul client. Dans cette rubrique, vous verrez comment convertir le désordre du développement en fichiers html et js uniques raffinés.

Syntaxe #

  • npm install : enregistre les packages à l’aide du gestionnaire de packages Node.js
  • npm install -g : enregistre les packages de npm en tant que global (utile pour les packages d’interface de ligne de commande)
  • cd : définit le focus de la ligne de commande sur une bibliothèque spécifique dans laquelle les processus peuvent être effectués
  • vulcaniser : réduit toutes les importations HTML dans un seul fichier
  • Crisper : convertit Inline JS dans un fichier HTML en un fichier JS externe

Installation de tous les outils nécessaires #

Exécutez la commande suivante une par une :

npm install -g vulcanize crisper

Utilisation #

  • Vulcaniser : écrase tous les fichiers d’importation HTML en un seul fichier
  • Crisper : extrait le js en ligne dans son propre fichier

Remarque : les utilisateurs d’Ubuntu peuvent avoir besoin de préfixer la commande ci-dessus avec sudo.

Mettre tous ensemble #

Mettez toutes les importations html de vos fichiers dans un seul fichier elements.html. Ne vous inquiétez pas si un fichier est importé plus d’une fois, il sera réduit à une seule importation.

Exécution de la vulcanisation et de la netteté #

sur votre fichier elements.html, exécutez les commandes suivantes :

cd PATH/TO/IMPORTFILE/
vulcanize elements.html -o elements.vulc.html --strip-comments --inline-css --inline-js
crisper --source elements.vulc.html --html build.html --js build.js

Vulcaniser le code source récupéré de toutes les importations, puis remplacer les importations par leur code source.
Crisper a extrait tous les fichiers js du fichier elements.vulc.html, les a placés dans un seul fichier build.js, a défini une balise script faisant référence au fichier build.js dans le fichier build.html

Minification des fichiers #

  • Ouvrez [minificateur HTML] (https://kangax.github.io/html-minifier/)

  • Ouvrez build.html

  • Copier tout son code

  • Dans la première zone de texte du Minfier HTML, collez le code que vous avez copié depuis build.html

  • Cliquez sur le bouton Réduire

  • Dans la deuxième zone de texte, le code minifié apparaîtra. Bien reçu

  • Créez un fichier build.min.html et collez-y tout votre code copié

  • Ouvrez JSCompress

  • Sélectionnez le 2ème onglet, c’est-à-dire Télécharger des fichiers JavaScript

  • Cliquez sur “Choisir les fichiers”

  • Sélectionnez le fichier build.js

  • Cliquez sur le bouton Compresser

  • Téléchargez le fichier sous build.js dans le même répertoire que build.min.html

importation de build.min.html #

Supprimez toutes les importations précédentes des fichiers HTML à partir desquels vous avez copié-collé les importations. Remplacer les importations par

<link rel="import" href="PATH/TO/IMPORTFILE/build.min.html">
