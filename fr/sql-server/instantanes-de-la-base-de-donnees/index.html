Instantanés de la base de données

  • Créer un instantané de base de données
  • Restaurer un instantané de base de données
  • SUPPRIMER l’instantané

Sur cette page

  • Créer un instantané de base de données
  • Restaurer un instantané de base de données
  • SUPPRIMER l’instantané

Un instantané de base de données est une vue statique en lecture seule d’une base de données SQL Server qui est cohérente sur le plan transactionnel avec la base de données source au moment de la création de l’instantané.

Un instantané de base de données réside toujours sur la même instance de serveur que sa base de données source. Lorsque la base de données source est mise à jour, l’instantané de la base de données est mis à jour.

Un instantané diffère d’une sauvegarde car le processus de création d’instantané est instantané et l’instantané n’occupe de l’espace que lorsque les modifications apportées à la base de données source sont appliquées. Une sauvegarde, quant à elle, stocke une copie complète des données au moment de la création de la sauvegarde.

De plus, un instantané donne une copie instantanée en lecture seule de la base de données, tandis qu’une sauvegarde doit être restaurée sur un serveur pour être lisible (et une fois restaurée, elle peut également être écrite)

Les instantanés de base de données ne sont disponibles que dans les éditions Enterprise et Developer.

Créer un instantané de base de données #

Un instantané de base de données est une vue statique en lecture seule d’une base de données SQL Server (la base de données source). Il est similaire à la sauvegarde, mais il est disponible comme n’importe quelle autre base de données afin que le client puisse interroger la base de données d’instantanés.

CREATE DATABASE MyDatabase_morning -- name of the snapshot
ON (
     NAME=MyDatabase_data, -- logical name of the data file of the source database
     FILENAME='C:\SnapShots\MySnapshot_Data.ss' -- snapshot file; 
) 
AS SNAPSHOT OF MyDatabase; -- name of source database

Vous pouvez également créer un instantané de la base de données avec plusieurs fichiers :

CREATE DATABASE MyMultiFileDBSnapshot ON
    (NAME=MyMultiFileDb_ft, FILENAME='C:\SnapShots\MyMultiFileDb_ft.ss'),
    (NAME=MyMultiFileDb_sys, FILENAME='C:\SnapShots\MyMultiFileDb_sys.ss'),
    (NAME=MyMultiFileDb_data, FILENAME='C:\SnapShots\MyMultiFileDb_data.ss'),
    (NAME=MyMultiFileDb_indx, FILENAME='C:\SnapShots\MyMultiFileDb_indx.ss')
AS SNAPSHOT OF MultiFileDb;

Restaurer un instantané de base de données #

Si les données d’une base de données source sont endommagées ou si des données erronées sont écrites dans la base de données, dans certains cas, la restauration de la base de données vers un instantané de base de données antérieur aux dommages peut constituer une alternative appropriée à la restauration de la base de données à partir d’une sauvegarde.

RESTORE DATABASE MYDATABASE FROM DATABASE_SNAPSHOT='MyDatabase_morning';

Avertissement : Cela supprimera toutes les modifications apportées à la base de données source depuis la prise de l’instantané !

SUPPRIMER l’instantané #

Vous pouvez supprimer des instantanés existants de la base de données à l’aide de l’instruction DELETE DATABASE :

DROP DATABASE Mydatabase_morning

Dans cette instruction, vous devez référencer le nom de l’instantané de la base de données.
