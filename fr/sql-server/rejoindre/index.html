Dans Structured Query Language (SQL), un JOIN est une méthode de liaison de deux tables de données dans une seule requête, permettant à la base de données de renvoyer un ensemble contenant des données des deux tables à la fois, ou d’utiliser les données d’une table à utiliser comme un Filtrez sur le second tableau. Il existe plusieurs types de JOIN définis dans la norme ANSI SQL.

Jointure interne #

La “jointure interne” renvoie uniquement les enregistrements/lignes qui correspondent/existent dans les deux tables en fonction d’une ou plusieurs conditions (spécifiées à l’aide du mot-clé “ON”). C’est le type de jointure le plus courant. La syntaxe générale de la “jointure interne” est la suivante :

SELECT * 
FROM table_1
INNER JOIN table_2
  ON table_1.column_name = table_2.column_name

Il peut également être simplifié en tant que “JOIN” :

SELECT * 
FROM table_1
JOIN table_2
  ON table_1.column_name = table_2.column_name

Exemple

/* Sample data. */
DECLARE @Animal table (
    AnimalId Int IDENTITY,
    Animal Varchar(20)
);

DECLARE @AnimalSound table (
    AnimalSoundId Int IDENTITY,
    AnimalId Int,
    Sound Varchar(20)
);

INSERT INTO @Animal (Animal) VALUES ('Dog');
INSERT INTO @Animal (Animal) VALUES ('Cat');
INSERT INTO @Animal (Animal) VALUES ('Elephant');

INSERT INTO @AnimalSound (AnimalId, Sound) VALUES (1, 'Barks');
INSERT INTO @AnimalSound (AnimalId, Sound) VALUES (2, 'Meows');
INSERT INTO @AnimalSound (AnimalId, Sound) VALUES (3, 'Trumpets');
/* Sample data prepared. */

SELECT 
    * 
FROM 
    @Animal
    JOIN @AnimalSound
        ON @Animal.AnimalId = @AnimalSound.AnimalId;

AnimalId    Animal               AnimalSoundId AnimalId    Sound
----------- -------------------- ------------- ----------- --------------------
1           Dog                  1             1           Barks
2           Cat                  2             2           Meows
3           Elephant             3             3           Trumpets

Utilisation d’une jointure interne avec une jointure externe gauche (remplacer par n’existe pas)

Cette requête renverra les données de la table 1 où les champs correspondant à la table2 avec une clé et les données ne figurant pas dans la table 1 lors de la comparaison avec la table2 avec une condition et une clé

select * 
  from Table1 t1
    inner join Table2 t2 on t1.ID_Column = t2.ID_Column 
    left  join Table3 t3 on t1.ID_Column = t3.ID_Column 
  where t2.column_name = column_value 
    and t3.ID_Column is null 
  order by t1.column_name;

Jointure externe #

Jointure externe gauche

LEFT JOIN renvoie toutes les lignes de la table de gauche, mises en correspondance avec les lignes de la table de droite où les conditions de la clause ON sont remplies. Les lignes dans lesquelles la clause ON n’est pas remplie ont NULL dans toutes les colonnes de la table de droite. La syntaxe d’un LEFT JOIN est :

SELECT * FROM table_1 AS t1
LEFT JOIN table_2 AS t2 ON t1.ID_Column = t2.ID_Column

Jointure externe droite

RIGHT JOIN renvoie toutes les lignes de la table de droite, mises en correspondance avec les lignes de la table de gauche où les conditions de la clause ON sont remplies. Les lignes dans lesquelles la clause ON n’est pas remplie ont NULL dans toutes les colonnes de la table de gauche. La syntaxe d’un RIGHT JOIN est :

SELECT * FROM table_1 AS t1
RIGHT JOIN table_2 AS t2 ON t1.ID_Column = t2.ID_Column

Jointure externe complète

FULL JOIN combine LEFT JOIN et RIGHT JOIN. Toutes les lignes sont renvoyées des deux tables, que les conditions de la clause “ON” soient remplies ou non. Les lignes qui ne satisfont pas la clause ON sont renvoyées avec NULL dans toutes les colonnes de la table opposée (c’est-à-dire que pour une ligne dans la table de gauche, toutes les colonnes de la table de droite contiendront NULL, et vice versa ). La syntaxe d’un FULL JOIN est :

SELECT * FROM table_1 AS t1
FULL JOIN table_2 AS t2 ON t1.ID_Column = t2.ID_Column

Exemples

/* Sample test data. */
DECLARE @Animal table (
    AnimalId Int IDENTITY,
    Animal Varchar(20)
);

DECLARE @AnimalSound table (
    AnimalSoundId Int IDENTITY,
    AnimalId Int,
    Sound Varchar(20)
);

INSERT INTO @Animal (Animal) VALUES ('Dog');
INSERT INTO @Animal (Animal) VALUES ('Cat');
INSERT INTO @Animal (Animal) VALUES ('Elephant');
INSERT INTO @Animal (Animal) VALUES ('Frog');

INSERT INTO @AnimalSound (AnimalId, Sound) VALUES (1, 'Barks');
INSERT INTO @AnimalSound (AnimalId, Sound) VALUES (2, 'Meows');
INSERT INTO @AnimalSound (AnimalId, Sound) VALUES (3, 'Trumpet');
INSERT INTO @AnimalSound (AnimalId, Sound) VALUES (5, 'Roars');
/* Sample data prepared. */

JOINTURE EXTERNE GAUCHE

SELECT * 
FROM @Animal As t1 
LEFT JOIN @AnimalSound As t2 ON t1.AnimalId = t2.AnimalId;

Résultats pour LEFT JOIN

AnimalId    Animal               AnimalSoundId AnimalId    Sound
----------- -------------------- ------------- ----------- --------------------
1           Dog                  1             1           Barks
2           Cat                  2             2           Meows
3           Elephant             3             3           Trumpet
4           Frog                 NULL          NULL        NULL

JOINTURE EXTÉRIEURE DROITE

SELECT * 
FROM @Animal As t1 
RIGHT JOIN @AnimalSound As t2 ON t1.AnimalId = t2.AnimalId;

Résultats pour RIGHT JOIN

AnimalId    Animal               AnimalSoundId AnimalId    Sound
----------- -------------------- ------------- ----------- --------------------
1           Dog                  1             1           Barks
2           Cat                  2             2           Meows
3           Elephant             3             3           Trumpet
NULL        NULL                 4             5           Roars

JOINTURE EXTÉRIEURE COMPLÈTE

FROM @Animal As t1

----------- -------------------- ------------- ----------- --------------------

3           Elephant             3             3           Trumpet

NULL        NULL                 4             5           Roars

Utilisation de Join dans une mise à jour #

Les jointures peuvent également être utilisées dans une instruction UPDATE :

CREATE TABLE Users (
    UserId int NOT NULL,
    AccountId int NOT NULL,
    RealName nvarchar(200) NOT NULL
)

CREATE TABLE Preferences (
    UserId int NOT NULL,
    SomeSetting bit NOT NULL
)

Mettez à jour la colonne SomeSetting de la table Preferences en filtrant par un prédicat sur la table Users comme suit :

UPDATE p
SET p.SomeSetting = 1
FROM Users u
JOIN Preferences p ON u.UserId = p.UserId
WHERE u.AccountId = 1234

p est un alias pour Preferences défini dans la clause FROM de l’instruction. Seules les lignes avec un AccountId correspondant de la table Users seront mises à jour.

Mettre à jour avec les instructions de jointure externe gauche

Update t 
SET  t.Column1=100
FROM Table1 t LEFT JOIN Table12 t2 
ON t2.ID=t.ID

** Mettre à jour les tables avec jointure interne et fonction d’agrégation **

UPDATE t1
SET t1.field1 = t2.field2Sum
FROM table1 t1
INNER JOIN (select field3, sum(field2) as field2Sum
from table2
group by field3) as t2
on t2.field3 = t1.field3

Join sur une sous-requête #

La jointure sur une sous-requête est souvent utilisée lorsque vous souhaitez obtenir des données agrégées (telles que Count, Avg, Max ou Min) à partir d’une table enfant/détails et les afficher avec les enregistrements de la table parent/en-tête. Par exemple, vous souhaiterez peut-être récupérer la première/la première ligne enfant en fonction de la date ou de l’identifiant, ou peut-être souhaitez-vous un nombre de toutes les lignes enfants ou une moyenne.

Cet exemple utilise des alias qui facilitent la lecture des requêtes lorsque plusieurs tables sont impliquées. Dans ce cas, nous récupérons toutes les lignes de la table parent Purchase Orders et récupérons uniquement la dernière (ou la plus récente) ligne enfant de la table enfant PurchaseOrderLineItems. Cet exemple suppose que la table enfant utilise des ID numériques incrémentiels.

SELECT po.Id, po.PODate, po.VendorName, po.Status, item.ItemNo, 
  item.Description, item.Cost, item.Price
FROM PurchaseOrders po
LEFT JOIN 
     (
       SELECT l.PurchaseOrderId, l.ItemNo, l.Description, l.Cost, l.Price, Max(l.id) as Id 
       FROM PurchaseOrderLineItems l
       GROUP BY l.PurchaseOrderId, l.ItemNo, l.Description, l.Cost, l.Price
     ) AS item ON item.PurchaseOrderId = po.Id

Jointure croisée #

“Une jointure croisée” est une jointure cartésienne, c’est-à-dire un produit cartésien des deux tables. Cette jointure ne nécessite aucune condition pour joindre deux tables. Chaque ligne du tableau de gauche sera jointe à chaque ligne du tableau de droite. Syntaxe pour une jointure croisée :

SELECT * FROM table_1
CROSS JOIN table_2

Exemple:

/* Sample data. */
DECLARE @Animal table (
    AnimalId Int IDENTITY,
    Animal Varchar(20)
);

DECLARE @AnimalSound table (
    AnimalSoundId Int IDENTITY,
    AnimalId Int,
    Sound Varchar(20)
);

INSERT INTO @Animal (Animal) VALUES ('Dog');
INSERT INTO @Animal (Animal) VALUES ('Cat');
INSERT INTO @Animal (Animal) VALUES ('Elephant');

INSERT INTO @AnimalSound (AnimalId, Sound) VALUES (1, 'Barks');
INSERT INTO @AnimalSound (AnimalId, Sound) VALUES (2, 'Meows');
INSERT INTO @AnimalSound (AnimalId, Sound) VALUES (3, 'Trumpet');
/* Sample data prepared. */

SELECT 
    * 
FROM 
    @Animal 
    CROSS JOIN @AnimalSound;

Résultats:

AnimalId    Animal               AnimalSoundId AnimalId    Sound
----------- -------------------- ------------- ----------- --------------------
1           Dog                  1             1           Barks
2           Cat                  1             1           Barks
3           Elephant             1             1           Barks
1           Dog                  2             2           Meows
2           Cat                  2             2           Meows
3           Elephant             2             2           Meows
1           Dog                  3             3           Trumpet
2           Cat                  3             3           Trumpet
3           Elephant             3             3           Trumpet

Notez qu’il existe d’autres façons d’appliquer un CROSS JOIN. Il s’agit d’une jointure “à l’ancienne” (obsolète depuis ANSI SQL-92) sans condition, ce qui entraîne une jointure croisée/cartésienne :

SELECT * 
FROM @Animal, @AnimalSound;

Cette syntaxe fonctionne également en raison d’une condition de jointure “toujours vraie”, mais n’est pas recommandée et doit être évitée, en faveur de la syntaxe explicite “CROSS JOIN”, pour des raisons de lisibilité.

SELECT * 
FROM 
    @Animal 
    JOIN @AnimalSound 
        ON 1=1

Rejoindre soi-même #

Une table peut être jointe sur elle-même dans ce que l’on appelle une auto-jointure, combinant des enregistrements de la table avec d’autres enregistrements de la même table. Les jointures automatiques sont généralement utilisées dans les requêtes où une hiérarchie dans les colonnes de la table est définie.

Considérez les exemples de données dans un tableau appelé “Employés” :

identifiant Nom Boss_ID
1 Bob 3
2 Jim 1
3 Sam 2

Le Boss_ID de chaque employé correspond à l’ID d’un autre employé. Pour récupérer une liste d’employés avec le nom de leur patron respectif, la table peut être jointe sur elle-même à l’aide de ce mappage. Notez que joindre une table de cette manière nécessite l’utilisation d’un alias (“Bosses” dans ce cas) sur la deuxième référence à la table pour se distinguer de la table d’origine.

SELECT Employees.Name,
    Bosses.Name AS Boss
FROM Employees
INNER JOIN Employees AS Bosses 
    ON Employees.Boss_ID = Bosses.ID

L’exécution de cette requête affichera les résultats suivants :

Nom Chef
Bob Sam
Jim Bob
Sam Jim

Transformation accidentelle d’une jointure externe en jointure interne #

Les jointures externes renvoient toutes les lignes d’une ou des deux tables, ainsi que les lignes correspondantes.

Table People
PersonID FirstName
       1 Alice
       2 Bob
       3 Eve

Table Scores
PersonID Subject Score
       1 Math    100
       2 Math     54
       2 Science  98

A gauche en rejoignant les tables :

Select * from People a
left join Scores b
on a.PersonID = b.PersonID

Retour:

PersonID FirstName PersonID Subject Score
       1 Alice            1 Math    100
       2 Bob              2 Math     54
       2 Bob              2 Science  98
       3 Eve           NULL NULL   NULL

Si vous vouliez renvoyer toutes les personnes, avec les scores mathématiques applicables, une erreur courante consiste à écrire :

Select * from People a
left join Scores b
on a.PersonID = b.PersonID
where Subject = 'Math'

Cela supprimerait Eve de vos résultats, en plus de supprimer le score scientifique de Bob, car Subject est NULL pour elle.

La syntaxe correcte pour supprimer les enregistrements non mathématiques tout en conservant tous les individus dans la table “Personnes” serait :

Select * from People a
left join Scores b
on a.PersonID = b.PersonID
and b.Subject = 'Math'

Supprimer en utilisant Joindre #

Les jointures peuvent également être utilisées dans une instruction DELETE. Soit un schéma comme suit :

CREATE TABLE Users (
    UserId int NOT NULL,
    AccountId int NOT NULL,
    RealName nvarchar(200) NOT NULL
)

CREATE TABLE Preferences (
    UserId int NOT NULL,
    SomeSetting bit NOT NULL
)

Nous pouvons supprimer des lignes de la table Preferences, en filtrant par un prédicat sur la table Users comme suit :

DELETE p
FROM Users u
INNER JOIN Preferences p ON u.UserId = p.UserId
WHERE u.AccountId = 1234

Ici, p est un alias pour Preferences défini dans la clause FROM de la déclaration et nous ne supprimons que les lignes qui ont un AccountId correspondant de la table Users.
