Formatage de la date et de l’heure à l’aide de FORMAT #

Vous pouvez utiliser la nouvelle fonction : FORMAT().

En utilisant cela, vous pouvez transformer vos champs DATETIME en votre propre format VARCHAR personnalisé.

Exemple

DECLARE @Date DATETIME = '2016-09-05 00:01:02.333'

SELECT FORMAT(@Date, N'dddd, MMMM dd, yyyy hh:mm:ss tt')

lundi 05 septembre 2016 00:01:02

DATEADD pour ajouter et soustraire des périodes de temps #

Syntaxe générale :

DATEADD (datepart , number , datetime_expr)

Pour ajouter une mesure de temps, le “nombre” doit être positif. Pour soustraire une mesure de temps, le “nombre” doit être négatif.

Exemples

DECLARE @now DATETIME2 = GETDATE();
SELECT @now;                        --2016-07-21 14:39:46.4170000
SELECT DATEADD(YEAR, 1, @now)       --2017-07-21 14:39:46.4170000
SELECT DATEADD(QUARTER, 1, @now)    --2016-10-21 14:39:46.4170000
SELECT DATEADD(WEEK, 1, @now)       --2016-07-28 14:39:46.4170000
SELECT DATEADD(DAY, 1, @now)        --2016-07-22 14:39:46.4170000
SELECT DATEADD(HOUR, 1, @now)       --2016-07-21 15:39:46.4170000
SELECT DATEADD(MINUTE, 1, @now)     --2016-07-21 14:40:46.4170000
SELECT DATEADD(SECOND, 1, @now)     --2016-07-21 14:39:47.4170000
SELECT DATEADD(MILLISECOND, 1, @now)--2016-07-21 14:39:46.4180000

REMARQUE : DATEADD accepte également les abréviations dans le paramètre datepart. L’utilisation de ces abréviations est généralement déconseillée car elles peuvent prêter à confusion (m vs mi, ww vs w, etc.).

Récupère la DateHeure actuelle #

Les fonctions intégrées [GETDATE][1] et [GETUTCDATE][2] renvoient chacune la date et l’heure actuelles sans décalage de fuseau horaire.

La valeur de retour des deux fonctions est basée sur le système d’exploitation de l’ordinateur sur lequel l’instance de SQL Server s’exécute.

La valeur de retour de GETDATE représente l’heure actuelle dans le même fuseau horaire que le système d’exploitation. La valeur de retour de GETUTCDATE représente l’heure UTC actuelle.

L’une ou l’autre fonction peut être incluse dans la clause SELECT d’une requête ou dans le cadre d’une expression booléenne dans la clause WHERE.

Exemples:

-- example query that selects the current time in both the server time zone and UTC
SELECT GETDATE() as SystemDateTime, GETUTCDATE() as UTCDateTime

-- example query records with EventDate in the past.
SELECT * FROM MyEvents WHERE EventDate < GETDATE()

[1] : https://msdn.microsoft.com/en-us/library/ms188383.aspx [2] : https://msdn.microsoft.com/en-us/library/ms178635.aspx

Il existe quelques autres fonctions intégrées qui renvoient différentes variations de la date-heure actuelle :

SELECT 
    GETDATE(),          --2016-07-21 14:27:37.447
    GETUTCDATE(),       --2016-07-21 18:27:37.447
    CURRENT_TIMESTAMP,  --2016-07-21 14:27:37.447
    SYSDATETIME(),      --2016-07-21 14:27:37.4485768
    SYSDATETIMEOFFSET(),--2016-07-21 14:27:37.4485768 -04:00
    SYSUTCDATETIME()    --2016-07-21 18:27:37.4485768

Obtenir le dernier jour d’un mois #

En utilisant les fonctions DATEADD et DATEDIFF, il est possible de retourner la dernière date d’un mois.

SELECT DATEADD(d, -1, DATEADD(m, DATEDIFF(m, 0, '2016-09-23') + 1, 0))
-- 2016-09-30 00:00:00.000

La fonction EOMONTH fournit un moyen plus concis de renvoyer la dernière date d’un mois et dispose d’un paramètre facultatif pour décaler le mois.

SELECT EOMONTH('2016-07-21')        --2016-07-31
SELECT EOMONTH('2016-07-21', 4)     --2016-11-30
SELECT EOMONTH('2016-07-21', -5)    --2016-02-29

Créer une fonction pour calculer l’âge d’une personne à une date précise #

Cette fonction prendra 2 paramètres datetime, la date de naissance et une date pour vérifier l’âge à

  CREATE FUNCTION [dbo].[Calc_Age]
    (
    @DOB datetime , @calcDate datetime 
    )
    RETURNS int
    AS
   BEGIN
declare @age int

IF (@calcDate < @DOB  )
RETURN -1

-- If a DOB is supplied after the comparison date, then return -1
SELECT @age = YEAR(@calcDate) - YEAR(@DOB) +
  CASE WHEN DATEADD(year,YEAR(@calcDate) - YEAR(@DOB)
  ,@DOB) > @calcDate THEN -1 ELSE 0 END
    

RETURN @age
    

END

SELECT  dbo.Calc_Age('2000-01-01',Getdate())

DATEDIFF pour le calcul des différences de période de temps #

Syntaxe générale :

DATEDIFF (datepart, datetime_expr1, datetime_expr2)

Il renverra un nombre positif si datetime_expr est dans le passé par rapport à datetime_expr2, et un nombre négatif sinon.

Exemples

DECLARE @now DATETIME2 = GETDATE();
DECLARE @oneYearAgo DATETIME2 = DATEADD(YEAR, -1, @now);
SELECT @now                                    --2016-07-21 14:49:50.9800000
SELECT @oneYearAgo                             --2015-07-21 14:49:50.9800000
SELECT DATEDIFF(YEAR, @oneYearAgo, @now)       --1
SELECT DATEDIFF(QUARTER, @oneYearAgo, @now)    --4
SELECT DATEDIFF(WEEK, @oneYearAgo, @now)       --52
SELECT DATEDIFF(DAY, @oneYearAgo, @now)        --366
SELECT DATEDIFF(HOUR, @oneYearAgo, @now)       --8784
SELECT DATEDIFF(MINUTE, @oneYearAgo, @now)     --527040
SELECT DATEDIFF(SECOND, @oneYearAgo, @now)     --31622400

REMARQUE : DATEDIFF accepte également les abréviations dans le paramètre datepart. L’utilisation de ces abréviations est généralement déconseillée car elles peuvent prêter à confusion (m vs mi, ww vs w, etc.).

DATEDIFF peut également être utilisé pour déterminer le décalage entre UTC et l’heure locale de SQL Server. L’instruction suivante peut être utilisée pour calculer le décalage entre l’heure UTC et l’heure locale (y compris le fuseau horaire).

select  DATEDIFF(hh, getutcdate(), getdate()) as 'CentralTimeOffset'

DATEPART & DATENAME #

DATEPART renvoie la datepart spécifiée de l’expression datetime spécifiée sous la forme d’une valeur numérique.

DATENAME renvoie une chaîne de caractères qui représente la datepart spécifiée de la date spécifiée. En pratique, DATENAME est surtout utile pour obtenir le nom du mois ou du jour de la semaine.

Il existe également des fonctions abrégées pour obtenir l’année, le mois ou le jour d’une expression datetime, qui se comportent comme DATEPART avec leurs unités datepart respectives.

Syntaxe:

DATEPART ( datepart , datetime_expr )
DATENAME ( datepart , datetime_expr )
DAY ( datetime_expr )
MONTH ( datetime_expr )
YEAR ( datetime_expr )

Exemples:

DECLARE @now DATETIME2 = GETDATE();
SELECT @now                       --2016-07-21 15:05:33.8370000
SELECT DATEPART(YEAR, @now)       --2016
SELECT DATEPART(QUARTER, @now)    --3
SELECT DATEPART(WEEK, @now)       --30
SELECT DATEPART(HOUR, @now)       --15
SELECT DATEPART(MINUTE, @now)     --5
SELECT DATEPART(SECOND, @now)     --33
-- Differences between DATEPART and DATENAME:
SELECT DATEPART(MONTH, @now)      --7
SELECT DATENAME(MONTH, @now)      --July
SELECT DATEPART(WEEKDAY, @now)    --5
SELECT DATENAME(WEEKDAY, @now)    --Thursday
--shorthand functions
SELECT DAY(@now)    --21
SELECT MONTH(@now)  --7
SELECT YEAR(@now)   --2016

REMARQUE : DATEPART et DATENAME acceptent également les abréviations dans le paramètre datepart. L’utilisation de ces abréviations est généralement déconseillée car elles peuvent prêter à confusion (m vs mi, ww vs w, etc.).

Renvoie uniquement la date à partir d’un DateTime #

Il existe plusieurs façons de renvoyer une date à partir d’un objet DateTime

  1. SÉLECTIONNER CONVERT(Date, GETDATE())
  2. SELECT DATEADD(dd, 0, DATEDIFF(dd, 0, GETDATE())) renvoie 2016-07-21 00:00:00.000
  3. SÉLECTIONNER CAST(GETDATE() COMME DATE)
  4. SÉLECTIONNER CONVERT(CHAR(10),GETDATE(),111)
  5. SELECT FORMAT(GETDATE(), 'aaaa-MM-jj')

Notez que les options 4 et 5 renvoient une chaîne, pas une date.

OBJET DE DATE CROSS PLATEFORME #

Dans Transact SQL, vous pouvez définir un objet comme Date (ou DateTime) en utilisant la fonction [DATEFROMPARTS][1] (ou [DATETIMEFROMPARTS][1]) comme suit :

 DECLARE @myDate DATE=DATEFROMPARTS(1988,11,28) 
 DECLARE @someMoment DATETIME=DATEFROMPARTS(1988,11,28,10,30,50,123)

Les paramètres que vous fournissez sont Année, Mois, Jour pour la fonction DATEFROMPARTS et, pour la fonction DATETIMEFROMPARTS, vous devrez fournir l’année, le mois, le jour, l’heure, les minutes, les secondes et les millisecondes.

Ces méthodes sont utiles et valent la peine d’être utilisées car l’utilisation de la chaîne simple pour créer une date (ou datetime) peut échouer en fonction de la région de la machine hôte, de l’emplacement ou des paramètres de format de date.
