Le mot clé Drop peut être utilisé avec divers objets SQL, cette rubrique fournit des exemples rapides d’utilisation différente avec des objets de base de données.

  • [PROCÉDURE DE SUPPRESSION (Transact-SQL)][2]

  • [SUPPRIMER LA BASE DE DONNÉES (Transact-SQL)][3]

[1] : https://msdn.microsoft.com/en-us/library/ms173790.aspx [2] : https://msdn.microsoft.com/en-us/library/ms174969.aspx [3] : https://msdn.microsoft.com/en-us/library/ms178613.aspx

Supprimer les tables temporaires #

Dans SQL Server, nous avons 2 types de tables temporaires :

  1. ##GlobalTempTable est un type de table temporaire qui est transparente entre toutes les sessions de l’utilisateur.
  2. Onglet temporaire #LocalTempTable - c’est un type de table temporaire qui n’existe que dans la portée actuelle (uniquement dans le processus réel - vous pouvez obtenir l’identifiant de votre processus actuel par SELECT @@SPID)

Le processus de suppression des tables temporaires est le même que pour une table normale :

DROP TABLE [ database_name . [ schema_name ] . | schema_name . ] table_name

AVANT SQL Server 2016 :

IF(OBJECT_ID('tempdb..#TempTable') is not null)
    DROP TABLE #TempTable;

SQL Server 2016 :

DROP TABLE IF EXISTS #TempTable

Supprimer les tableaux #

La commande DROP TABLE supprime les définitions de table et toutes les données, index, déclencheurs, contraintes et autorisations associées.

Avant de supprimer une table, vous devez vérifier s’il existe des objets (vues, procédures stockées, autres tables) qui font référence à la table.

Vous ne pouvez pas supprimer une table référencée par une autre table par FOREIGN KEY. Vous devez d’abord déposer la FOREIGN KEY qui y fait référence.

Vous pouvez supprimer une table référencée par une vue ou une procédure stockée, mais après la suppression de la table, la vue ou la procédure stockée n’est plus utilisable.

La syntaxe

DROP TABLE [ IF EXISTS ] [ database_name . [ schema_name ] . | schema_name . ]
table_name [ ,...n ] [ ; ]
  • IF EXISTS - Supprimer la table uniquement si elle existe
  • database_name - Spécifiez le nom de la base de données où la table est contenue
  • schema_name - Spécifiez le nom du schéma où se trouve la table
  • table_name - Spécifiez le nom de la table à supprimer

Exemples

Supprimer la table portant le nom TABLE_1 de la base de données actuelle et du schéma par défaut dbo

DROP TABLE Table_1;

Supprimez la table avec TABLE_1 de la base de données HR et du schéma par défaut dbo

DROP TABLE HR.Table_1;

Supprimez la table avec TABLE_1 de la base de données HR et du schéma external

DROP TABLE HR.external.TABLE_1;

Supprimer les bases de données #

La commande DROP DATABASE supprime un catalogue de base de données, quel que soit son état (hors ligne, en lecture seule, suspect, etc.), de l’instance SQL Server actuelle.

Une base de données ne peut pas être supprimée si des instantanés de base de données lui sont associés, car les instantanés de base de données doivent être supprimés en premier.

Une suppression de base de données supprime tous les fichiers du disque physique (sauf s’il est hors ligne) utilisés par la base de données, sauf si vous utilisez la procédure stockée ‘sp_detach_db’.

Une suppression d’instantané de base de données supprime l’instantané de l’instance SQL Server et supprime les fichiers physiques également utilisés par celle-ci.

Une base de données supprimée ne peut être recréée qu’en restaurant une sauvegarde (pas non plus à partir d’un instantané de base de données).

La syntaxe

DROP DATABASE [ IF EXISTS ] { database_name | database_snapshot_name } [ ,...n ] [;]
  • IF EXISTS - Supprimer la table uniquement si elle existe
  • database_name - Spécifie le nom de la base de données à supprimer
  • database_snapshot_name - Spécifie l’instantané de la base de données à supprimer

Exemples

Supprimer une seule base de données ;

DROP DATABASE Database1;

Suppression de plusieurs bases de données

DROP DATABASE Database1, Database2;

Suppression d’un instantané

DROP DATABASE Database1_snapshot17;

Suppression si la base de données existe

DROP DATABASE IF EXISTS Database1;
