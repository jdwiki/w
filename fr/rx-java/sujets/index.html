Syntaxe #

  • Sujet<T, R> sujet = AsyncSubject.create(); // AsyncSubject par défaut
  • Sujet<T, R> sujet = BehaviorSubject.create(); // Sujet de comportement par défaut
  • Sujet<T, R> sujet = PublishSubject.create(); // Objet de publication par défaut
  • Sujet<T, R> sujet = ReplaySubject.create(); // ReplaySubject par défaut
  • mySafeSubject = new SerializedSubject(unSafeSubject); // Convertit un unsafeSubject en un safeSubject - généralement pour les sujets multi-threads

PublierObjet #

PublishSubject émet vers un Observer uniquement les éléments qui sont émis par la source Observable après l’heure de l’abonnement.

Un exemple simple de PublishSubject :

Observable<Long> clock = Observable.interval(500, TimeUnit.MILLISECONDS);
Subject<Long, Long> subjectLong = PublishSubject.create();

clock.subscribe(subjectLong);

System.out.println("sub1 subscribing...");
subjectLong.subscribe(l -> System.out.println("sub1 -> " + l));
Thread.sleep(3000);
System.out.println("sub2 subscribing...");
subjectLong.subscribe(l -> System.out.println("sub2 -> " + l));
Thread.sleep(5000);

Production:

sub1 subscribing...
sub1 -> 0
sub1 -> 1
sub2 subscribing...
sub1 -> 2
sub2 -> 2
sub1 -> 3
sub2 -> 3

Dans l’exemple ci-dessus, un PublishSubject s’abonne à un Observable qui agit comme une horloge et émet des éléments (Long) toutes les 500 millisecondes. Comme on le voit dans la sortie, le PublishSubject transmet les valeurs qu’il obtient de la source (clock) à ses abonnés (sub1 et sub2).

Un PublishSubject peut commencer à émettre des éléments dès sa création, sans aucun observateur, ce qui risque de faire perdre un ou plusieurs éléments jusqu’à ce qu’un observateur puisse s’inscrire.

createClock(); // 3 lines moved for brevity. same as above example

Thread.sleep(5000); // introduces a delay before first subscribe

sub1andsub2(); // 6 lines moved for brevity. same as above example

Production:

sub1 subscribing...
sub1 -> 10
sub1 -> 11
sub2 subscribing...
sub1 -> 12
sub2 -> 12
sub1 -> 13
sub2 -> 13

Notez que sub1 émet des valeurs à partir de 10. Le délai de 5 secondes introduit a causé une perte d’articles. Ceux-ci ne peuvent pas être reproduits. Cela fait essentiellement de PublishSubject un Hot Observable.

Notez également que si un observateur s’abonne à PublishSubject après avoir émis n éléments, ces n éléments ne peuvent pas être reproduits pour cet observateur.

Ci-dessous, le diagramme en marbre de PublishSubject

[![entrez la description de l’image ici][1]][1]

Le PublishSubject émet des éléments à tous ceux qui se sont abonnés, à tout moment avant que le onCompleted de la source Observable ne soit appelé.

Si la source Observable se termine par une erreur, le PublishSubject n’émettra aucun élément aux observateurs suivants, mais transmettra simplement la notification d’erreur de la source Observable.

[![entrez la description de l’image ici][2]][2]

Cas d’utilisation
Supposons que vous souhaitiez créer une application qui surveillera les cours des actions d’une certaine entreprise et les transmettra à tous les clients qui en feront la demande.

/* Dummy stock prices */
Observable<Integer> prices = Observable.just(11, 12, 14, 11, 10, 12, 15, 11, 10);

/* Your server */
PublishSubject<Integer> watcher = PublishSubject.create();
/* subscribe to listen to stock price changes and push to observers/clients */
prices.subscribe(watcher);

/* Client application */
stockWatcher = getWatcherInstance(); // gets subject
Subscription steve = stockWatcher.subscribe(i -> System.out.println("steve watching " + i));
Thread.sleep(1000);
System.out.println("steve stops watching");
steve.unsubscribe();

Dans l’exemple de cas d’utilisation ci-dessus, PublishSubject agit comme un pont pour transmettre les valeurs de votre serveur à tous les clients qui s’abonnent à votre watcher.

Matières de base #

Un Subject dans RxJava est une classe qui est à la fois un Observable et un Observer. Cela signifie essentiellement qu’il peut agir comme un “Observable” et transmettre des entrées aux abonnés et comme un “Observer” pour obtenir des entrées d’un autre Observable.

Subject<String, String> subject = PublishSubject.create(); 
subject.subscribe(System.out::print);
subject.onNext("Hello, World!");

Les impressions ci-dessus “Hello, World!” pour consoler en utilisant Sujets.

Explication

  1. La première ligne de code définit un nouveau Subject de type PublishSubject

     Subject<String, String> subject = PublishSubject.create();
     |     |       |       |                 |
  subject<input, output>  name   = default publish subject

  2. La deuxième ligne s’abonne au sujet, montrant le comportement “Observateur”.

     subject.subscribe(System.out::print);

    This enables the Subject to take inputs like a regular subscriber

  3. La troisième ligne appelle la méthode onNext du sujet, montrant le comportement Observable.

     subject.onNext("Hello, World!");

    This enables the Subject to give inputs to all subscribing to it.

Les types

Un Sujet (dans RxJava) peut appartenir à l’un de ces quatre types :

  • AsyncSubject
  • ComportementSujet
  • PublierSujet
  • ReplaySujet

De plus, un Subject peut être de type SerializedSubject. Ce type garantit que le “sujet” ne viole pas le contrat observable (qui spécifie que tous les appels doivent être sérialisés)
