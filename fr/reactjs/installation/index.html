Configuration simple #

Paramétrage des dossiers #

Cet exemple suppose que le code se trouve dans src/ et que la sortie est placée dans out/. En tant que tel, la structure du dossier devrait ressembler à quelque chose comme

example/
|-- src/
|   |-- index.js
|   `-- ...
|-- out/
`-- package.json

Mise en place des packages #

En supposant un environnement de configuration npm, nous devons d’abord configurer babel afin de transpiler le code React en code conforme es5.

$npm install --save-dev babel-core babel-loader babel-preset-es2015 babel-preset-react

La commande ci-dessus demandera à npm d’installer les bibliothèques principales de babel ainsi que le module de chargement à utiliser avec webpack. Nous installons également les préréglages es6 et réagissons pour que babel comprenne le code des modules JSX et es6. (Plus d’informations sur les préréglages peuvent être trouvées ici [Préréglages Babel][1])

$npm i -D webpack

Cette commande installera webpack en tant que dépendance de développement. (i est le raccourci pour install et -D le raccourci pour –save-dev)

Vous pouvez également installer des packages Webpack supplémentaires (tels que des chargeurs supplémentaires ou l’extension webpack-dev-server)

Enfin, nous aurons besoin du code de réaction réel

$npm i -D réagir réagir-dom

Mise en place de webpack #

Avec la configuration des dépendances, nous aurons besoin d’un fichier webpack.config.js pour dire à webpack quoi faire

simple webpack.config.js :

var path = require('path');

module.exports = {
  entry: './src/index.js',
  output: {
    path: path.resolve(__dirname, 'out'),
    filename: 'bundle.js'
  },
  module: {
    loaders: [
      {
        test: /\.js$/,
        exclude: /(node_modules)/,
        loader: 'babel-loader',
        query: {
          presets: ['es2015', 'react']
        }
      }
    ]
  }
};

Ce fichier indique à webpack de commencer par le fichier index.js (supposé être dans src/ ) et convertissez-le en un seul fichier bundle.js dans le répertoire “out”.

Le bloc module indique à Webpack de tester tous les fichiers rencontrés par rapport à l’expression régulière et, s’ils correspondent, appellera le chargeur spécifié. (babel-loader dans ce cas) De plus, l’expression régulière exclude indique à Webpack d’ignorer ce chargeur spécial pour tous les modules du dossier node_modules, ce qui permet d’accélérer le processus de transpilation. Enfin, l’option query indique à webpack quels paramètres transmettre à babel et est utilisée pour transmettre les préréglages que nous avons installés précédemment.

Test de la configuration #

Il ne reste plus qu’à créer le fichier src/index.js et à essayer de compresser l’application

src/index.js :

'use strict'

import React from 'react'
import { render } from 'react-dom'

const App = () => {
  return <h1>Hello world!</h1>
}

render(
  <App />,
  document. getElementById('app')
)

Ce fichier afficherait normalement un simple en-tête <h1>Hello world!</h1> dans la balise html avec l’id ‘app’, mais pour l’instant, il devrait suffire de transpiler le code une fois.

$./node_modules/.bin/webpack . Exécutera la version installée localement de webpack (utilisez $webpack si vous avez installé webpack globalement avec -g)

Cela devrait créer le fichier out/bundle.js avec le code transpilé à l’intérieur et conclut l’exemple.

[1] : http://babeljs.io/docs/plugins/

Utilisation de webpack-dev-server #

Installer #

Après avoir configuré un projet simple pour utiliser webpack, babel et réagir en émettant $npm i -g webpack-dev-server installera le serveur http de développement pour un développement plus rapide.

Modification de webpack.config.js #

var path = require('path');

module.exports = {
  entry: './src/index.js',
  output: {
    path: path.resolve(__dirname, 'out'),
    publicPath: '/public/',
    filename: 'bundle.js'
  },
  module: {
    loaders: [
      {
        test: /\.js$/,
        exclude: /(node_modules)/,
        loader: 'babel',
        query: {
          presets: ['es2015', 'react']
        }
      }
    ]
  },
  devServer: {
    contentBase: path.resolve(__dirname, 'public'),
    hot: true
  }
};

Les modifications sont en

  • output.publicPath qui définit un chemin à partir duquel notre bundle sera servi (voir [fichiers de configuration Webpack] [1] pour plus d’informations)

  • devServer

    • contentBase the base path to serve static files from (for example index.html)
    • hot sets the webpack-dev-server to hot reload when changes get made to files on disk

Et enfin, nous avons juste besoin d’un simple index.html pour tester notre application.

index.html :

<!DOCTYPE html>
<html lang="en">
  <head>
    <meta charset="utf-8">
    <title>React Sandbox</title>
  </head>
  <body>

    <div id="app" />

    <script src="public/bundle.js"></script>
  </body>
</html>

Avec cette configuration, l’exécution de $webpack-dev-server devrait démarrer un serveur http local sur le port 8080 et lors de la connexion, une page contenant un <h1>Hello world!</h1> devrait s’afficher.

[1] : https://github.com/webpack/docs/wiki/Configuration#outputpublicpath
