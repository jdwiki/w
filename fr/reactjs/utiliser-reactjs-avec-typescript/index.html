Composant ReactJS écrit en Typescript #

En fait, vous pouvez utiliser les composants de ReactJS dans Typescript comme dans l’exemple de Facebook. Remplacez simplement l’extension du fichier ‘jsx’ par ’tsx’ :

//helloMessage.tsx:
var HelloMessage = React.createClass({
  render: function() {
    return <div>Hello {this.props.name}</div>;
  }
});
ReactDOM.render(<HelloMessage name="John" />, mountNode);

Mais pour tirer pleinement parti de la fonctionnalité principale de Typescript (vérification de type statique), deux choses doivent être faites :

1) convertir l’exemple React.createClass en classe ES6 :

//helloMessage.tsx:
class HelloMessage extends React.Component {
  render() {
    return <div>Hello {this.props.name}</div>;
  }
}
ReactDOM.render(<HelloMessage name="John" />, mountNode);

2) ajoutez ensuite les interfaces Props et State :

interface IHelloMessageProps {
    name:string;
}

interface IHelloMessageState {
  //empty in our case
}

class HelloMessage extends React.Component<IHelloMessageProps, IHelloMessageState> {
  constructor(){
    super();
  }  
  render() {
    return <div>Hello {this.props.name}</div>;
  }
}
ReactDOM.render(<HelloMessage name="Sebastian" />, mountNode);

Désormais, Typescript affichera une erreur si le programmeur oublie de passer les accessoires. Ou s’ils ont ajouté des accessoires qui ne sont pas définis dans l’interface.

Installation et configuration #

Pour utiliser TypeScript avec React dans un projet de nœud, vous devez d’abord avoir un répertoire de projet initialisé avec npm. Pour initialiser le répertoire avec npm init

Installation via npm ou yarn

Vous pouvez installer React à l’aide de npm en procédant comme suit :

npm install --save react react-dom

Facebook a publié son propre gestionnaire de packages nommé Yarn, qui peut également être utilisé pour installer React. Après avoir installé Yarn, il vous suffit d’exécuter cette commande :

yarn add react react-dom

Vous pouvez ensuite utiliser React dans votre projet exactement de la même manière que si vous aviez installé React via npm.

Installation des définitions de type de réaction dans Typescript 2.0+

Pour compiler votre code à l’aide de typescript, ajoutez/installez des fichiers de définition de type à l’aide de npm ou yarn.

npm install --save-dev @types/react @types/react-dom

ou, en utilisant du fil

yarn add --dev @types/react @types/react-dom

Installation des définitions de type de réaction dans les anciennes versions de Typescript

Vous devez utiliser un package séparé appelé [tsd][1]

tsd install react react-dom --save

Ajout ou modification de la configuration Typescript

Pour utiliser [JSX][2], un langage mixant javascript avec html/xml, il faut changer la configuration du compilateur typescript. Dans le fichier de configuration dactylographié du projet (généralement nommé tsconfig.json), vous devrez ajouter l’option JSX comme :

"compilerOptions": {
    "jsx": "react"
},

Cette option de compilateur indique essentiellement au compilateur de typescript de traduire les balises JSX dans le code en appels de fonction javascript.

Pour éviter que le compilateur dactylographié ne convertisse JSX en appels de fonction javascript simples, utilisez

"compilerOptions": {
    "jsx": "preserve"
},

[1] : https://www.npmjs.com/package/tsd [2] : https://www.typescriptlang.org/docs/handbook/jsx.html

Composants React sans état dans Typescript #

Les composants React qui sont de pures fonctions de leurs accessoires et ne nécessitent aucun état interne peuvent être écrits en tant que fonctions JavaScript au lieu d’utiliser la syntaxe de classe standard, comme :

import React from 'react'

const HelloWorld = (props) => (
    <h1>Hello, {props.name}!</h1>
);

La même chose peut être obtenue dans Typescript en utilisant la classe React.SFC :

import * as React from 'react';

class GreeterProps {
   name: string
}

const Greeter : React.SFC<GreeterProps> = props =>
    <h1>Hello, {props.name}!</h1>;

Notez que le nom React.SFC est un alias pour React.StatelessComponent Donc, l’un ou l’autre peut être utilisé.

Composants sans état et sans propriété #

Le composant de réaction le plus simple sans état et sans propriétés peut être écrit comme suit :

import * as React from 'react';

const Greeter = () => <span>Hello, World!</span>

Ce composant, cependant, ne peut pas accéder à this.props puisque le typescript ne peut pas dire s’il s’agit d’un composant de réaction. Pour accéder à ses accessoires, utilisez :

import * as React from 'react';

const Greeter: React.SFC<{}> = props => () => <span>Hello, World!</span>

Même si le composant n’a pas de propriétés explicitement définies, il peut désormais accéder à props.children puisque tous les composants ont intrinsèquement des enfants.

Une autre bonne utilisation similaire des composants sans état et sans propriété est la création de modèles de page simples. Ce qui suit est un exemple simple de composant Page, en supposant qu’il existe déjà des composants hypothétiques Container, NavTop et NavBottom dans le projet :

import * as React from 'react';

const Page: React.SFC<{}> = props => () => 
    <Container>
        <NavTop />
        {props.children}
        <NavBottom />
    </Container>

const LoginPage: React.SFC<{}> = props => () =>
    <Page>
        Login Pass: <input type="password" />
    </Page>

Dans cet exemple, le composant Page peut ensuite être utilisé par n’importe quelle autre page réelle comme modèle de base.
