Communication entre composants

Il existe au total 3 cas de communication entre les composants React :

  • Cas 1 : Communication parent-enfant
  • Cas 2 : Communication enfant-parent
  • Cas 3 : communication de composants non liés (tout composant à tout composant)

Composants enfant à parent #

En renvoyant les données au parent, pour ce faire, nous transmettons simplement une fonction en tant qu’accessoire du composant parent au composant enfant, et le composant enfant appelle cette fonction.

Dans cet exemple, nous allons modifier l’état Parent en transmettant une fonction au composant Child et en appelant cette fonction à l’intérieur du composant Child.

import React from 'react';

class Parent extends React.Component {
    constructor(props) {
        super(props);
        this.state = { count: 0 };

        this.outputEvent = this.outputEvent.bind(this);
    }
    outputEvent(event) {
        // the event context comes from the Child
        this.setState({ count: this.state.count++ });
    }

    render() {
        const variable = 5;
        return (
            <div>
                Count: { this.state.count }
                <Child clickHandler={this.outputEvent} />
            </div>
        );
    }
}

class Child extends React.Component {
    render() {
        return (
            <button onClick={this.props.clickHandler}>
                Add One More
            </button>
        );
    }
}

export default Parent;

Notez que la méthode outputEvent du parent (qui modifie l’état du parent) est invoquée par l’événement onClick du bouton de l’enfant.

Composants non liés #

La seule façon si vos composants n’ont pas de relation parent-enfant (ou sont liés mais trop éloignés, comme un arrière-arrière-petit-fils) est d’avoir une sorte de signal auquel un composant s’abonne et l’autre écrit.

Ce sont les 2 opérations de base de tout système d’événement : s’abonner/écouter à un événement à notifier, et envoyer/déclencher/publier/envoyer un événement à notifier à ceux qui le souhaitent.

Il y a au moins 3 modèles pour le faire. Vous pouvez trouver une [comparaison ici][1].

Voici un bref résumé :

  • Modèle 1 : Event Emitter/Target/Dispatcher : les auditeurs doivent référencer la source pour s’abonner.

  • pour s’abonner : otherObject.addEventListener('click', () => { alert('click!'); });

  • pour expédier : this.dispatchEvent('click');

  • Modèle 2 : Publier/S’abonner : vous n’avez pas besoin d’une référence spécifique à la source qui déclenche l’événement, il existe un objet global accessible partout qui gère tous les événements.

  • pour s’abonner : globalBroadcaster.subscribe('click', () => { alert('click!'); });

  • pour envoyer : globalBroadcaster.publish('click');

  • Modèle 3 : Signaux : similaire à Event Emitter/Target/Dispatcher mais vous n’utilisez pas de chaînes aléatoires ici. Chaque objet susceptible d’émettre des événements doit avoir une propriété spécifique portant ce nom. De cette façon, vous savez exactement quels événements un objet peut émettre.

  • pour s’abonner : otherObject.clicked.add( () => { alert('click'); });

  • pour expédier : this.clicked.dispatch();

[1] : https://github.com/millermedeiros/js-signals/wiki/Comparison-between-different-Observer-Pattern-implementations

Composants parent à enfant #

C’est le cas le plus simple en fait, très naturel dans le monde React et il y a de fortes chances que vous l’utilisiez déjà.

Vous pouvez transmettre les accessoires aux composants enfants. Dans cet exemple, message est le prop que nous transmettons au composant enfant, le nom message est choisi arbitrairement, vous pouvez le nommer comme vous le souhaitez.

import React from 'react';

class Parent extends React.Component {
    render() {
        const variable = 5;
        return (
            <div>
                <Child message="message for child" />
                <Child message={variable} />
            </div>
        );
    }
}

class Child extends React.Component {
    render() {
        return <h1>{this.props.message}</h1>
    }
}

export default Parent;

Ici, le composant <Parent /> rend deux composants <Child />, passant message for child dans le premier composant et 5 dans le second.

En résumé, vous avez un composant (parent) qui en rend un autre (enfant) et lui transmet des accessoires.
