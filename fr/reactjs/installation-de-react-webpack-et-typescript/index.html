Pour obtenir la coloration syntaxique dans votre éditeur (par exemple, VS Code), vous devrez télécharger les informations de frappe pour les modules que vous utilisez dans votre projet.

Supposons, par exemple, que vous utilisiez React et ReactDOM dans votre projet et que vous souhaitiez obtenir la mise en évidence et Intellisense pour eux. Vous devrez ajouter les types à votre projet à l’aide de cette commande :

npm install --save @types/react @types/react-dom

Votre éditeur devrait maintenant détecter automatiquement ces informations de frappe et vous fournir la saisie semi-automatique et Intellisense pour ces modules.

webpack.config.js #

module.exports = {
    entry: './src/index',
    output: {
        path: __dirname + '/build',
        filename: 'bundle.js'
    },
    module: {
        rules: [{
            test: /\.tsx?$/,
            loader: 'ts-loader',
            exclude: /node_modules/
        }]
    },
    resolve: {
        extensions: ['.ts', '.tsx']
    }
};

Les principaux composants sont (en plus des propriétés standard entry, output et autres propriétés webpack):

Le chargeur #

Pour cela, vous devez créer une règle qui teste les extensions de fichier .ts et .tsx, spécifiez ts-loader comme chargeur.

Résoudre les extensions TS #

Vous devez également ajouter les extensions .ts et .tsx dans le tableau resolve, sinon Webpack ne les verra pas.

tsconfig.json #

Il s’agit d’un tsconfig minimal pour vous permettre d’être opérationnel.

{
    "include": [
        "src/*"
    ],
    "compilerOptions": {
        "target": "es5",
        "jsx": "react",
        "allowSyntheticDefaultImports": true
    }
}

Passons en revue les propriétés une par une :

inclure #

Il s’agit d’un tableau de code source. Ici, nous n’avons qu’une seule entrée, src/*, qui spécifie que tout ce qui se trouve dans le répertoire src doit être inclus dans la compilation.

compilerOptions.target #

Spécifie que nous voulons compiler vers la cible ES5

compilerOptions.jsx #

Définir ceci sur true fera que TypeScript compilera automatiquement votre syntaxe tsx de <div /> à React.createElement("div").

compilerOptions.allowSyntheticDefaultImports #

Propriété pratique qui vous permettra d’importer des modules de nœud comme s’il s’agissait de modules ES6, donc au lieu de faire

import * as React from 'react'
const { Component } = React

tu peux juste faire

import React, { Component } from 'react'

sans aucune erreur vous indiquant que React n’a pas d’exportation par défaut.

Mon premier composant #

import React, { Component } from 'react';
import ReactDOM from 'react-dom';


interface AppProps {
    name: string;
}
interface AppState {
    words: string[];
}

class App extends Component<AppProps, AppState> {
    constructor() {
        super();
        this.state = {
            words: ['foo', 'bar']
        };
    }

    render() {
        const { name } = this.props;
        return (<h1>Hello {name}!</h1>);
    }
}

const root = document.getElementById('root');
ReactDOM.render(<App name="Foo Bar" />, root);

Lorsque vous utilisez TypeScript avec React, une fois que vous avez téléchargé les définitions de type React DefinitelyTyped (npm install --save @types/react), chaque composant vous demandera d’ajouter des annotations de type.

Tu fais ça comme ça :

class App extends Component<AppProps, AppState> { }

où AppProps et AppState sont respectivement des interfaces (ou des alias de type) pour les accessoires et l’état de vos composants.
