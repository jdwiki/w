Mesure des performances avec ReactJS #

Vous ne pouvez pas améliorer quelque chose que vous ne pouvez pas mesurer. Pour améliorer les performances des composants React, vous devez pouvoir les mesurer. ReactJS fournit des outils * addon * pour mesurer les performances. Importez le module react-addons-perf pour mesurer les performances

import Perf from 'react-addons-perf' // ES6
var Perf = require('react-addons-perf') // ES5 with npm
var Perf = React.addons.Perf; // ES5 with react-with-addons.js

Vous pouvez utiliser les méthodes ci-dessous à partir du module “Perf” importé :

  • Perf.printInclusive()
  • Perf.printExclusive()
  • Perf.printWasted()
  • Perf.printOperations()
  • Perf.printDOM()

Le plus important dont vous aurez besoin la plupart du temps est Perf.printWasted() qui vous donne la représentation tabulaire du temps perdu de votre composant individuel

[![entrez la description de l’image ici][1]][1]

Vous pouvez noter la colonne Temps perdu dans le tableau et améliorer les performances du composant à l’aide de la section Conseils et astuces ci-dessus

Reportez-vous au [React Official Guide][2] et à l’excellent article de [Benchling Engg. sur les performances de réaction][3]

[1] : https://i.stack.imgur.com/rCKLY.png [2] : https://facebook.github.io/react/docs/perf.html [3] : http://benchling.engineering/deep-dive-react-perf-debugging/

Algorithme diff de React #

Générer le nombre minimum d’opérations pour transformer un arbre en un autre a une complexité de l’ordre de O(n^3) où n est le nombre de nœuds dans l’arbre. React s’appuie sur deux hypothèses pour résoudre ce problème en un temps linéaire - O(n)

  1. Deux composants de la même classe généreront des arbres similaires et deux composants de classes différentes généreront des arbres différents.

  2. Il est possible de fournir une clé unique pour les éléments qui est stable sur différents rendus.

Afin de décider si deux nœuds sont différents, React différencie 3 cas

  1. Deux nœuds sont différents s’ils ont des types différents.
  • par exemple, <div>...</div> est différent de <span>...</span>
  1. Chaque fois que deux nœuds ont des clés différentes
  • par exemple, <div key="1">...</div> est différent de <div key="2">...</div>

De plus, ce qui suit est crucial et extrêmement important à comprendre si vous souhaitez optimiser les performances

S’ils [deux nœuds] ne sont pas du même type, React ne va pas essayez même de faire correspondre ce qu’ils rendent. Il va juste supprimer le premier du DOM et insérez le second.

Voici pourquoi

Il est très peu probable qu’un élément

génère un DOM qui va ressembler à ce qu’un générerait. À la place de passer du temps à essayer de faire correspondre ces deux structures, React juste reconstruit l’arbre à partir de zéro.

Les bases - DOM HTML vs DOM virtuel #

HTML DOM est cher

Chaque page Web est représentée en interne sous la forme d’une arborescence d’objets. Cette représentation est appelée Document Object Model. De plus, il s’agit d’une interface indépendante de la langue qui permet aux langages de programmation (tels que JavaScript) d’accéder aux éléments HTML.

Autrement dit

Le DOM HTML est une norme sur la façon d’obtenir, de modifier, d’ajouter ou de supprimer du code HTML éléments.

Cependant, ces opérations DOM sont extrêmement coûteuses.

** DOM virtuel est une solution **

L’équipe de React a donc eu l’idée d’abstraire le DOM HTML et de créer son propre DOM virtuel afin de calculer le nombre minimum d’opérations que nous devons appliquer sur le DOM HTML pour reproduire l’état actuel de notre application.

** Le DOM virtuel permet de gagner du temps grâce aux modifications inutiles du DOM. **

De quelle façon précisément?

À chaque instant, React a l’état de l’application représenté sous la forme d’un “DOM virtuel”. Chaque fois que l’état de l’application change, voici les étapes que React effectue afin d’optimiser les performances

  1. Générer un nouveau DOM virtuel qui représente le nouvel état de notre application

  2. Comparez l’ancien DOM virtuel (qui représente le DOM HTML actuel) et le nouveau DOM virtuel

  3. Basé sur 2. trouver le nombre minimum d’opérations pour transformer l’ancien DOM virtuel (qui représente le DOM HTML actuel) dans le nouveau DOM virtuel

  • pour en savoir plus à ce sujet - lisez l’algorithme Diff de React
  1. Une fois ces opérations trouvées, elles sont mappées dans leurs opérations HTML DOM équivalentes
  • rappelez-vous, le DOM virtuel n’est qu’une abstraction du DOM HTML et il existe une relation isomorphe entre eux
  1. Désormais, le nombre minimum d’opérations qui ont été trouvées et transférées vers leurs opérations HTML DOM équivalentes sont désormais appliquées directement sur le HTML DOM de l’application, ce qui permet de gagner du temps en modifiant inutilement le HTML DOM.

Noter: Les opérations appliquées sur le DOM virtuel sont bon marché, car le DOM virtuel est un objet JavaScript.

Conseils & Astuces #

Lorsque deux nœuds ne sont pas du même type, React n’essaie pas de les faire correspondre - il supprime simplement le premier nœud du DOM et insère le second. C’est pourquoi le premier conseil dit

  1. Si vous vous voyez alterner entre deux classes de composants avec une sortie très similaire, vous voudrez peut-être en faire la même classe.

  1. Utilisez shouldComponentUpdate pour empêcher le composant de se restituer, si vous savez qu’il ne va pas changer, par exemple

    shouldComponentUpdate: function(nextProps, nextState) { return nextProps.id !== this.props.id; }
