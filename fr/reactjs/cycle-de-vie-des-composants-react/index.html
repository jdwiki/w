Les méthodes de cycle de vie doivent être utilisées pour exécuter du code et interagir avec votre composant à différents moments de la vie des composants. Ces méthodes sont basées sur un composant Montage, Mise à jour et Démontage.

Création de composants #

Lorsqu’un composant React est créé, un certain nombre de fonctions sont appelées :

  • Si vous utilisez React.createClass (ES5), 5 fonctions définies par l’utilisateur sont appelées
  • Si vous utilisez class Component étend React.Component (ES6), 3 fonctions définies par l’utilisateur sont appelées

getDefaultProps() (ES5 uniquement) #

Il s’agit de la première méthode appelée.

Les valeurs prop renvoyées par cette fonction seront utilisées par défaut si elles ne sont pas définies lors de l’instanciation du composant.

Dans l’exemple suivant, this.props.name sera défini par défaut sur Bob s’il n’est pas spécifié autrement :

getDefaultProps() {
  return {
    initialCount: 0,
    name: 'Bob'
  };
}

getInitialState() (ES5 uniquement) #

Il s’agit de la deuxième méthode appelée.

La valeur de retour de getInitialState() définit l’état initial du composant React. Le framework React appellera cette fonction et attribuera la valeur de retour à this.state.

Dans l’exemple suivant, this.state.count sera initialisé avec la valeur de this.props.initialCount :

getInitialState() {
  return {
    count : this.props.initialCount
  };
}

composantWillMount() (ES5 et ES6) #

C’est la troisième méthode appelée.

Cette fonction peut être utilisée pour apporter des modifications finales au composant avant qu’il ne soit ajouté au DOM.

componentWillMount() {
  ...
}

rendre() (ES5 et ES6) #

Il s’agit de la quatrième méthode appelée.

La fonction render() doit être une pure fonction de l’état et des accessoires du composant. Il renvoie un seul élément qui représente le composant pendant le processus de rendu et doit être soit une représentation d’un composant DOM natif (par exemple <p />) ou un composant composite. Si rien ne doit être rendu, il peut renvoyer null ou undefined.

Cette fonction sera rappelée après toute modification des accessoires ou de l’état du composant.

render() {
  return (
    <div>
      Hello, {this.props.name}!
    </div>
  );
}

componentDidMount() (ES5 et ES6) #

Il s’agit de la cinquième méthode appelée.

Le composant a été monté et vous pouvez maintenant accéder aux nœuds DOM du composant, par ex. via refs.

Cette méthode doit être utilisée pour :

  • Préparation des minuteries

  • Récupération des données

  • Ajout d’écouteurs d’événements

  • Manipulation des éléments DOM

    componentDidMount() { … }

Syntaxe ES6 #

Si le composant est défini à l’aide de la syntaxe de classe ES6, les fonctions getDefaultProps() et getInitialState() ne peuvent pas être utilisées.

Au lieu de cela, nous déclarons notre defaultProps comme une propriété statique sur la classe, et déclarons la forme de l’état et l’état initial dans le constructeur de notre classe. Ceux-ci sont tous deux définis sur l’instance de la classe au moment de la construction, avant que toute autre fonction de cycle de vie React ne soit appelée.

L’exemple suivant illustre cette approche alternative :

class MyReactClass extends React.Component {
  constructor(props){
    super(props);

    this.state = {
      count: this.props.initialCount
    };
  }
  
  upCount() {
    this.setState((prevState) => ({
      count: prevState.count + 1
    }));
  }
  
  render() {
    return (
      <div>
        Hello, {this.props.name}!<br />
        You clicked the button {this.state.count} times.<br />
        <button onClick={this.upCount}>Click here!</button>
      </div>
    );
  }
}

MyReactClass.defaultProps = {
  name: 'Bob',
  initialCount: 0
};

Remplacement de getDefaultProps() #

Les valeurs par défaut des props du composant sont spécifiées en définissant la propriété defaultProps de la classe :

MyReactClass.defaultProps = {
  name: 'Bob',
  initialCount: 0
};

Remplacement de getInitialState() #

La manière idiomatique de configurer l’état initial du composant consiste à définir this.state dans le constructeur :

constructor(props){
  super(props);

  this.state = {
    count: this.props.initialCount
  };
}

Suppression de composants #

componentWillUnmount() #

Cette méthode est appelée avant qu’un composant ne soit démonté du DOM.

C’est un bon endroit pour effectuer des opérations de nettoyage comme :

  • Suppression des écouteurs d’événements.

  • Effacement des minuteries.

  • Prises d’arrêt.

  • Nettoyage des états redux.

    componentWillUnmount(){ … }

Un exemple de suppression de l’écouteur d’événement attaché dans componentWillUnMount

import React, { Component } from 'react';

export default class SideMenu extends Component {

  constructor(props) {
    super(props);
    this.state = {
        ...
      };
    this.openMenu = this.openMenu.bind(this);
    this.closeMenu = this.closeMenu.bind(this);
  }

  componentDidMount() {
    document.addEventListener("click", this.closeMenu);
  }

  componentWillUnmount() {
    document.removeEventListener("click", this.closeMenu);
  }

  openMenu() {
    ...
  }

  closeMenu() {
    ...
  }

  render() {
    return (
      <div>
        <a
          href      = "javascript:void(0)"
          className = "closebtn"
          onClick   = {this.closeMenu}
        >
          ×
        </a>
        <div>
          Some other structure
        </div>
      </div>
    );
  }
}

Mise à jour des composants #

composantWillReceiveProps(nextProps) #

Il s’agit de la première fonction appelée lors des changements de propriétés.

Lorsque les propriétés du composant changent, React appellera cette fonction avec les nouvelles propriétés. Vous pouvez accéder aux anciens props avec this.props et aux nouveaux props avec nextProps.

Avec ces variables, vous pouvez effectuer des opérations de comparaison entre les anciens et les nouveaux accessoires, ou appeler la fonction en raison d’un changement de propriété, etc.

componentWillReceiveProps(nextProps){
  if (nextProps.initialCount && nextProps.initialCount > this.state.count){
    this.setState({
      count : nextProps.initialCount
    });
  }
}

shouldComponentUpdate(nextProps, nextState) #

Il s’agit de la deuxième fonction appelée sur les changements de propriétés et la première sur les changements d’état.

Par défaut, si un autre composant / votre composant modifie une propriété / un état de votre composant, React affichera une nouvelle version de votre composant. Dans ce cas, cette fonction renvoie toujours true.

Vous pouvez surcharger cette fonction et choisir plus précisément si votre composant doit être mis à jour ou non.

Cette fonction est principalement utilisée pour l’optimisation.

Si la fonction renvoie false, le pipeline de mise à jour s’arrête immédiatement.

componentShouldUpdate(nextProps, nextState){
  return this.props.name !== nextProps.name ||
    this.state.count !== nextState.count;
}

componentWillUpdate(nextProps, nextState) #

Cette fonction fonctionne comme componentWillMount(). Les modifications ne sont pas dans DOM, vous pouvez donc effectuer certaines modifications juste avant que la mise à jour ne soit effectuée.

/!\ : vous ne pouvez pas utiliser this.setState().

componentWillUpdate(nextProps, nextState){}

rendre() #

Il y a quelques changements, alors refaites le rendu du composant.

componentDidUpdate(prevProps, prevState) #

Même chose que componentDidMount() : DOM est actualisé, vous pouvez donc faire un peu de travail sur le DOM ici.

componentDidUpdate(prevProps, prevState){}

Appel de la méthode du cycle de vie dans différents états #

Cet exemple sert de complément à d’autres exemples qui expliquent comment utiliser les méthodes de cycle de vie et quand la méthode sera appelée.

Cet exemple résume quelles méthodes (componentWillMount, componentWillReceiveProps, etc.) seront appelées et dans quelle séquence sera différente pour un composant dans différents états :

Lorsqu’un composant est initialisé :

  1. getDefaultProps
  2. getInitialState
  3. composantWillMount
  4. rendu
  5. composantDidMount

Lorsqu’un composant a changé d’état :

  1. shouldComponentUpdate
  2. componentWillUpdate
  3. rendu
  4. componentDidUpdate

Lorsqu’un composant a des accessoires modifiés :

  1. componentWillReceiveProps
  2. shouldComponentUpdate
  3. componentWillUpdate
  4. rendu
  5. componentDidUpdate

Lorsqu’un composant est en cours de démontage :

  1. composantWillUnmount

Conteneur de composants réactifs #

Lors de la création d’une application React, il est souvent souhaitable de diviser les composants en fonction de leur responsabilité principale, en composants de présentation et de conteneur. Les composants de présentation ne concernent que l’affichage des données - ils peuvent être considérés et sont souvent implémentés comme des fonctions qui convertissent un modèle en vue. En règle générale, ils ne maintiennent aucun état interne. Les composants de conteneur sont concernés par la gestion des données. Cela peut être fait en interne via leur propre état ou en agissant en tant qu’intermédiaires avec une bibliothèque de gestion d’état telle que Redux. Le composant de conteneur n’affichera pas directement les données, mais les transmettra à un composant de présentation.

// Container component
import React, { Component } from 'react';
import Api from 'path/to/api';

class CommentsListContainer extends Component {
    constructor() {
        super();
        // Set initial state
        this.state = { comments: [] }
    }

    componentDidMount() {
        // Make API call and update state with returned comments
        Api.getComments().then(comments => this.setState({ comments }));
    }

    render() {
        // Pass our state comments to the presentational component
        return (
            <CommentsList comments={this.state.comments} />;
        );
    }
}

// Presentational Component
const CommentsList = ({ comments }) => (
    <div>
        {comments.map(comment => (
            <div>{comment}</div>
        )}
    </div>
);

CommentsList.propTypes = {
    comments: React.PropTypes.arrayOf(React.PropTypes.string)
}
