Configurer Codecommit pour la ligne de commande git #

AWS Codecommit peut être utilisé comme stockage pour les référentiels GIT privés. La configuration implique quelques étapes, en supposant que vous disposez déjà d’un compte AWS valide.

  1. Inscrivez-vous à [AWS Codecommit][1]. Actuellement, seule la région “us-east-1” est disponible.

  2. Créez un [utilisateur IAM][2] qui aura accès aux référentiels, par exemple codecommit-user

  3. Attachez le rôle d’autorisation AWSCodeCommitFullAccess à cet utilisateur

  4. Créez une nouvelle « clé d’accès » pour cet utilisateur et notez « ID de la clé » et « code secret »

  5. Maintenant, allez-y et créez un [profil de configuration AWS][3] sur votre machine locale

    $ aws configure –profile codecommit-user

Dans l’étape suivante, nous associons la commande aws à git en tant qu’assistant d’identification avec les commandes suivantes :

$ git config --global credential.helper \
    '!aws --profile codecommit-user codecommit credential-helper $@'
$ git config --global credential.UseHttpPath true

Vous pouvez vérifier ou modifier cette configuration par la suite :

$ git config --global --edit

Vous devriez noter une section :

[credential]
    helper = !aws --profile codecommit-user codecommit credential-helper $@
    UseHttpPath = true

Vous pouvez maintenant utiliser git depuis la ligne de commande comme d’habitude.

[1] : https://console.aws.amazon.com/codecommit/home?region=us-east-1#/repository/list [2] : https://console.aws.amazon.com/iam/home#users [3] : https://www.wikiod.com/fr/aws-cli/premiers-pas-avec-aws-cli#Création d’un nouveau profil

Utiliser SourceTree avec AWS Codecommit #

Atlassian [SourceTree][1] est un outil visuel pour Mac et Windows permettant de gérer les référentiels de code source. Cela peut être utilisé avec Codecommit en tant que référentiel distant, mais il faut ajouter une option de configuration supplémentaire au référentiel local dans SourceTree pour pouvoir se connecter avec codecommit.

Tout d’abord, configurez Codecommit pour git local.

En supposant que vous ayez un référentiel git local que vous souhaitez pousser vers codecommit, suivez simplement ces étapes :

  1. Connectez-vous à AWS Codecommit à l’aide de la [console Web][2].

  2. Créez un nouveau référentiel, par exemple my-project

  3. Copiez l’URL HTTPS, elle devrait ressembler à https://git-codecommit.us-east-1.amazonaws.com/v1/repos/my-project

  4. Maintenant, dans SourceTree, ouvrez le panneau Paramètres / Télécommandes

  5. Ajoutez une nouvelle télécommande avec le nom : origin et Url / Path : le lien que vous avez copié auparavant

  6. Enfin, ouvrez l’option Modifier le fichier de configuration et ajoutez l’extrait suivant :

    [credential] helper = /usr/local/bin/aws –profile codecommit-user codecommit credential-helper $@ UseHttpPath = true

[![entrez la description de l’image ici][3]][3]

Une fois enregistré, le fichier de configuration devrait ressembler à ceci :

[core]
    repositoryformatversion = 0
    filemode = true
    bare = false
    logallrefupdates = true
    ignorecase = true
    precomposeunicode = true
[branch "master"]
    remote = origin
    merge = refs/heads/master
[remote "origin"]
    url = https://git-codecommit.us-east-1.amazonaws.com/v1/repos/digitaloffice.nu
    fetch = +refs/heads/*:refs/remotes/origin/*
[credential]
    helper = /usr/local/bin/aws --profile codecommit-user codecommit credential-helper $@
    UseHttpPath = true

Remarque : ceci est basé sur la configuration OS-X. Faites particulièrement attention au chemin pour aws (qui est /usr/local/bin/aws dans ce cas) et sera très certainement différent sous d’autres configurations Unix ou Windows.
