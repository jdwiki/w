Il existe deux types de procédures dans Progress ABL : les procédures internes et les prototypes de procédures qui sont des façades vers les dll Windows ou les procédures de bibliothèque partagée Unix/Linux.

Comme pour les fonctions, les procédures ne peuvent pas être imbriquées. Vous ne pouvez pas imbriquer des fonctions dans des procédures et vice versa.

Une procédure est appelée avec l’instruction RUN.

Syntaxe #

  • Nom de la procédure RUN. //Exécute une procédure appelée nomprocédure.

  • EXÉCUTER proc1 (ENTRÉE “BONJOUR”). // Entre la chaîne HELLO dans proc1

  • RUN proc2(INPUT var1, output var2). //Entre var1 et sortir var2 vers/depuis proc2

  • RUN proc3(input “name = ‘joe’”, OUTPUT TABLE ttResult). // Entre name=joe et sort les enregistrements dans une table

  • PROCEDURE proc : // Déclare une procédure nommée proc

  • PROCÉDURE DE FIN. // Termine la procédure en cours

Une procédure interne de base #

Contrairement aux fonctions, il n’est pas nécessaire de déclarer une procédure en avant. Il peut être placé n’importe où dans votre code, avant ou après que vous l’ayez appelé en utilisant RUN.

RUN proc.

//Procedure starts here
PROCEDURE proc:

//Procedure ends here
END PROCEDURE.

Le nom de la procédure est suivi d’un signe deux-points indiquant qu’il s’agit du début d’un bloc. Le bloc se termine par END PROCEDURE. (mais cela peut être remplacé simplement par END.).

Paramètres d’ENTRÉE et de SORTIE #

Une procédure peut avoir des paramètres de différents types : entrée, sortie, entrée-sortie (bidirectionnelle) et aussi certains types spéciaux comme les tables temporaires et les ensembles de données).

Dans l’instruction run, il est facultatif de déclarer INPUT (c’est considéré par défaut) - toutes les autres directions doivent être spécifiquement déclarées.

Une procédure prenant deux nombres entiers en entrée et produisant un nombre décimal.

PROCEDURE divideAbyB:
    DEFINE INPUT  PARAMETER piA       AS INTEGER     NO-UNDO.
    DEFINE INPUT  PARAMETER piB       AS INTEGER     NO-UNDO.
    DEFINE OUTPUT PARAMETER pdeResult AS DECIMAL     NO-UNDO.

    pdeResult = piA / piB.

END PROCEDURE.

DEFINE VARIABLE de AS DECIMAL     NO-UNDO.

RUN divideAbyB(10, 2, OUTPUT de).

DISPLAY de. //5.00

Les paramètres sont totalement facultatifs. Vous pouvez mélanger et assortir comme vous le souhaitez. L’ordre des paramètres dépend de vous, mais il est pratique de commencer par l’entrée et de terminer par la sortie - vous devez les mettre dans le bon ordre dans l’instruction run et le mélange des directions peut être ennuyeux.

Récursivité - voir récursivité #

La récursivité est facile - “EXÉCUTER” la procédure elle-même depuis l’intérieur de la procédure. Cependant, si vous récurez trop loin, la pile manquera d’espace.

Une procédure de calcul de la factorielle.

PROCEDURE factorial:
    DEFINE INPUT  PARAMETER piNum AS INTEGER     NO-UNDO.
    DEFINE OUTPUT PARAMETER piFac AS INTEGER     NO-UNDO.

    DEFINE VARIABLE iFac AS INTEGER     NO-UNDO.

    IF piNum = 1 THEN DO:
        pifac = 1.
    END.
    ELSE DO:
        RUN factorial(piNum - 1, OUTPUT iFac).
        piFac = piNum * iFac.
    END.

END PROCEDURE.

DEFINE VARIABLE f AS INTEGER     NO-UNDO.

RUN factorial(7, OUTPUT f).

DISPLAY f.

Portée #

La procédure a sa propre portée. La portée extérieure « saignera » dans la procédure, mais pas l’inverse.

DEFINE VARIABLE i AS INTEGER     NO-UNDO INIT 1.
DEFINE VARIABLE j AS INTEGER     NO-UNDO.

PROCEDURE p:

    MESSAGE i VIEW-AS ALERT-BOX. // 1
    MESSAGE j VIEW-AS ALERT-BOX. // 0

    j = 2.

END PROCEDURE.

RUN p.

MESSAGE i VIEW-AS ALERT-BOX. // 1
MESSAGE j VIEW-AS ALERT-BOX. // 2

Déclarer une variable à l’intérieur d’une procédure qui a le même nom qu’un paramètre à l’extérieur ne créera qu’une variable locale.

DEFINE VARIABLE i AS INTEGER     NO-UNDO INIT 1.
DEFINE VARIABLE j AS INTEGER     NO-UNDO.

PROCEDURE p:
    
    DEFINE VARIABLE i AS INTEGER     NO-UNDO INIT 5.

    MESSAGE i VIEW-AS ALERT-BOX. // 5
    MESSAGE j VIEW-AS ALERT-BOX. // 0

    j = 2.

END PROCEDURE.

RUN p.

MESSAGE i VIEW-AS ALERT-BOX. // 1
MESSAGE j VIEW-AS ALERT-BOX. // 2

Toute variable créée à l’intérieur d’une procédure n’est accessible qu’à cette procédure.

Cela générera une erreur de compilation :

PROCEDURE p:
    
    DEFINE VARIABLE i AS INTEGER     NO-UNDO INIT 5.

END PROCEDURE.

RUN p.

MESSAGE i VIEW-AS ALERT-BOX. // Unknown Field or Variable name i - error 201
