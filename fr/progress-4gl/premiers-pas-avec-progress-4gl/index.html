FizzBuzz #

Un autre exemple de programmes de style “Hello World” est [FizzBuzz][1].

DEFINE VARIABLE i AS INTEGER     NO-UNDO.
DEFINE VARIABLE cOut AS CHARACTER   NO-UNDO.

DO i = 1 TO 100:

    /* Dividable by 3: fizz */
    IF i MODULO 3 = 0 THEN
        cOut = "Fizz".
    /* Dividable by 5: buzz */
    ELSE IF i MODULO 5 = 0 THEN
        cOut = "Buzz".
    /* Otherwise just the number */
    ELSE 
        cOut = STRING(i).
    
    /* Display the output */
    DISPLAY cOut WITH FRAME x1 20 DOWN.
    /* Move the display position in the frame down 1 */
    DOWN WITH FRAME x1.
END.

[1] : https://fr.wikipedia.org/wiki/Fizz_buzz

Installation ou configuration #

Avancement de l’installation

Téléchargez votre distribution depuis Progress. Si vous voulez une licence de démonstration, vous devez les contacter. Assurez-vous de télécharger un fichier tar 64 bits et non un fichier tar 32 bits (sauf si vous utilisez une machine 32 bits).

Les fenêtres

Le téléchargement sera une archive zip. Décompressez-le et exécutez simplement setup.exe. L’installation sera graphique mais sinon exactement comme celle décrite ci-dessous.

Linux/Unix/HP-UX, etc.

Placez le fichier tar sur votre système Progress. Disons que vous l’avez dans votre répertoire personnel :

/home/user/PROGRESSFILENAME.tar

Extrayez-le :

cd /home/user
tar xvf PROGRESSFILENAME.tar

Il va créer un répertoire nommé

proinst

Changez de répertoire vers une autre destination et créez-y un répertoire temporaire. Par exemple:

cd /tmp
mkdir proinst116
cd proinst116

Une fois l’installation terminée, ce répertoire contiendra des informations sur l’installation ainsi que des fichiers que vous pourrez enregistrer et utiliser pour de futures répétitions automatiques de la même installation.

Exécutez maintenant le script d’installation (nommé “proinst” dans le répertoire “proinst”) :

/home/user/proinst/proinst

Installation silencieuse

L’installation a stocké un fichier nommé /usr/dlc/install/response.ini (ou votre répertoire d’installation). Ce fichier peut être utilisé pour répéter exactement la même installation dans une installation “silencieuse” qui peut être scriptée et exécutée sans aucune interaction.

Pour exécuter une installation silencieuse, faites simplement :

/path-to-proinst/proinst -b /path-to-response-file/response.ini -l /path-to-store-log/silent.log

Bonjour le monde! #

Une fois que vous avez démarré l’éditeur Progress de votre choix (il y a plusieurs options), écrivez simplement :

DISPLAY "Hello, World!".

Et exécutez en appuyant sur la touche ou l’élément de menu correspondant :

Sous Windows dans AppBuilder : F1 (Compiler -> Exécuter)

Sous Linux/Unix dans l’éditeur L4G : F2 (ou ctrl+X) (Compiler -> Exécuter)

Sous Windows dans Developer Studio : alt+shift+X, suivi de G (Run -> Run As Progress Application OpenEdge)

Configuration de l’environnement #

Linux/Unix

Une fois que vous avez installé Progress, il est très facile à exécuter.

Vous n’avez besoin que de quelques variables d’environnement. Le répertoire où Progress a été installé (par défaut /usr/dlc mais peut être autre chose) doit être dans la variable DLC

DLC=/usr/dlc

Et vous voudrez peut-être aussi le sous-répertoire "bin" de DLC dans votre PATH :

PATH=$PATH:$DLC/bin

Maintenant, vous êtes prêt !

Il y a aussi un script installé appelé proenv qui fera cela (et un peu plus) pour vous. Son emplacement par défaut est /usr/dlc/bin/proenv.

Quelques utilitaires :

showcfg

Cela listera tous vos produits installés.

pro

Cela lancera “l’éditeur de procédure” où vous pourrez éditer et exécuter vos programmes.

pro program.p

Ouvrera program.p pour modification s’il existe. Sinon une erreur s’affichera.

pro -p program.p

Cela exécutera “program.p”. Si un fichier compilé (program.r) est présent, il sera exécuté, sinon il sera temporairement compilé et ensuite exécuté. Le fichier compilé ne sera pas enregistré.

Création de la base de données de démonstration “sports2000” à partir de la ligne de commande #

Cela montre comment créer la base de données de démonstration utilisée dans une grande partie de la documentation de Progress : sports2000.

Cela suppose que vous avez installé les produits Progress avec au moins un type de licence de base de données.

Exécutez le script/fichier bat proenv qui vous donnera une invite avec toutes les variables d’environnement définies.

Créer un répertoire.

Cet exemple est pour Windows. La gestion des répertoires, etc. peut être différente dans un autre système d’exploitation.

proenv> cd \
proenv> mkdir db
proenv> cd db
proenv> mkdir sports2000
proenv> cd sports2000

Créer une base de données sports2000 en utilisant “prodb”

proenv> prodb mySportsDb sports2000

Syntaxe de prodb :

prodb nom-de-la-nouvelle-base-de-données nom-et-chemin-de-la-base-de-données-source

Cela créera une base de données appelée “mySportsDb” dans le répertoire courant. Cette base de données est une copie exacte de la base de données sports2000 fournie avec l’installation de Progress. Étant donné que la base de données source sports2000 se trouve dans le répertoire d’installation de Progress, vous n’avez pas besoin de spécifier le chemin.

Fichiers de programme #

Le code Progress ABL est normalement stocké dans des fichiers avec une fin différente en fonction de ce qu’ils contiennent. Les terminaisons sont facultatives mais plutôt un standard de facto :

Extension de nom de fichier Contient
.p Un programme Progress. Peut contenir plusieurs procédures internes, fonctions, etc.
.je Inclure le fichier à inclure dans d’autres fichiers
.w Fichier contenant une représentation graphique d’une fenêtre ou d’un dialogue, basé sur WinForm.
.r Le résultat compilé de tout fichier contenant Progress 4GL. Appelé r-code.
.cls Une classe orientée objet de progression
.wrx Un conteneur pour les données ActiveX chaque fois que nécessaire (généré par compilation dans “AppBuilder”).

Pour exécuter un fichier programme dans Progress 4GL, l’instruction RUN est utilisée :

RUN program.p. //Will run program.p without parameters.
RUN program.w (INPUT true). //Will run program.w with input parameter set to true.

RUN program. //Will run program.r if present otherwise internal procedure "program".

Pour inclure un autre fichier dans un programme Progress, la directive {} est utilisée :

{program.i} //Includes program.i in the current program

Exécuter sports2000 en tant que service #

Une fois la base de données sports2000 installée, il est temps de l’exécuter en tant que serveur autonome (et non de s’y connecter en tant que fichier).

Démarrez proenv (proenv dans le menu de démarrage sous Windows ou /usr/install-directory/bin/proenv sous Linux/Unix).

Connexion et déconnexion

Une fois votre base de données opérationnelle, vous pouvez vous y connecter dans votre éditeur de progression :

CONNECT mySportsDb -H localhost -S 9999.

ou

CONNECT "-db mySportsDb -H localhost -S 9999".

Si vous obtenez un message d’erreur, vous avez soit des informations erronées dans la commande, soit la base de données n’est pas opérationnelle. Vous pouvez également avoir un pare-feu logiciel ou similaire interférant.

Vous pouvez consulter le fichier journal de la base de données (mySportsDb.lg dans cet exemple) pour tout indice.

La déconnexion est tout aussi simple :

DISCONNECT mySportDb.

ou

DISCONNECT "mySportsDb".

Arrêter la base de données (ou déconnecter les utilisateurs

Pour fermer la base de données, vous pouvez exécuter la commande proshut à partir de proenv :

proenv> proshut mySportsDb
OpenEdge Release 11.6 as of Fri Oct 16 19:01:51 EDT 2015
usr    pid    time of login           user id     Type  tty                  Limbo?
 24   7044 Wed Feb 01 20:22:57 2017   xyz         REMC  XYZ-PC               no
            1  Disconnect a User
            2  Unconditional Shutdown
            3  Emergency Shutdown (Kill All)
            x  Exit
  1. Utilisez ‘1’ pour déconnecter des utilisateurs spécifiques.
  2. Utilisez ‘2’ pour fermer la base de données. Remarque : aucune question posée, l’arrêt démarre directement !
  3. Utilisez 3 uniquement si vous ne pouvez pas supprimer la base de données d’une autre manière. Cela pourrait corrompre vos données.
  4. Utilisez x pour quitter l’utilitaire proshut.

Vous pouvez également arrêter la base de données directement depuis la ligne de commande :

proenv>proshut mySportsDb -by

Ou déconnectez un utilisateur de la ligne de commande (en supposant que vous connaissiez son numéro d’utilisateur, usr dans la liste ci-dessus) :

proenv>proshut mySportsDb -C disconnect 24
