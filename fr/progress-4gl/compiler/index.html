Compile le code de progression appelé “r-code” et est normalement enregistré dans un fichier avec l’extension .r. Il existe plusieurs façons de compiler : en utilisant l’instruction “COMPILE” ou sous Linux ou AppBuilder : le compilateur d’application intégré. Developer Studio (l’environnement Eclipse) a une compilation intégrée dans son processus de construction.

Vous devez avoir installé 4GL Development ou OpenEdge Studio pour compiler les programmes 4GL qui mettent à jour la base de données.

Syntaxe #

  • COMPILE programme.p SAVE. //Compile program.p et enregistre son r-code
  • COMPILER LA VALEUR (var) ENREGISTRER. //Compile le nom enregistré dans la variable “var” et enregistre son r-code
  • COMPILE prog.p XREF prog.xref LISTING prog.list. // Compilez prog.p et créez des xref et des fichiers de liste. N’enregistrez pas le r-code.
  • COMPILE program.p SAVE NO-ERROR. //Compilez program.p, enregistrez le r-code et supprimez les erreurs pour arrêter l’exécution.

Compilateur d’applications #

Constructeur d’applications Windows

Dans Windows Appbuilder, le compilateur d’applications se trouve dans le menu Outils. [![Windows AppBuilder][1]][1]

Éditeur de procédure (Linux - pro ou Windows pro.exe

Dans l’éditeur de procédure (à la fois Linux et Windows), le compilateur se trouve dans le menu Outils. [![Linux Procedure Editor Editor][2]][2]

** Compilateur d’applications **

[![Compilateur d’applications Windows][3]][1] [![Compilateur d’applications Linux][4]][3]

Quel que soit le système d’exploitation, la fonctionnalité du compilateur est la même. Vous pouvez ajouter des répertoires et/ou des fichiers et les compiler.

Paramètres principaux (plus ci-dessous):

  • Enregistrer le nouveau fichier .r. Si cette case n’est pas cochée, les fichiers seront simplement compilés mais pas enregistrés. Utile pour le suivi des erreurs par exemple.
  • Regardez dans les sous-répertoires. Sinon, des sous-répertoires devront être ajoutés.
  • Supprimer les anciens fichiers .r. Remplacez l’ancien fichier .r.
  • Onlu Compile si aucun fichier .r. Compile uniquement les fichiers non compilés.

Option :

  • Propath - vous montre le propath et vous permet de sélectionner les répertoires à compiler à partir de celui-ci.
  • Ajouter - vous permet de saisir un répertoire ou un fichier.
  • Modifier - vous permet de modifier une entrée existante.
  • Supprimer - Supprime une entrée.
  • Démarrer la compilation - Démarre le compilateur. Raccourci : F2

Les choix du menu principal :

  • Fichier -> Quitter : Quitte le compilateur
  • Compiler -> Démarrer la compilation : Démarre le compilateur. Raccourci : F2
  • Outils -> Accès aux autres outils
  • Option -> Compilateur… : Paramètres, voir ci-dessous.
  • Aide -> Aide OpenEdge (Windows uniquement). Aide en ligne. Raccourci : F1

Réglages

[![Paramètres][5]][5]

  • Spécification de fichier par défaut : extensions de nom de fichier à compiler
  • Fichier journal des messages : fichier dans lequel enregistrer les messages, les avertissements et les erreurs.
  • Enregistrer dans : où stocker le fichier .r. Si vide, le même répertoire que le code.
  • Langues : pour les traductions. Non couvert ici.
  • V6Frame : Ancien et inutile…
  • Steam-IO : si vous souhaitez imprimer la sortie du compilateur. Préférablement pas.
  • Fichier de liste : si vous souhaitez que le compilateur crée un fichier de liste. Utile pour le débogage
  • Ajouter : ajouter au fichier de liste existant. Sinon, écrasez.
  • Largeur de page + longueur : format du fichier de liste.
  • Fichier Xref : si vous voulez que le compilateur crée un XREF. Utile pour le débogage, la vérification de l’utilisation de l’index, etc.
  • Format XML : si la xréf du compilateur doit être un xml. Sinon, texte “en clair”.
  • Ajouter : ajouter au fichier xref existant. Sinon, écrasez.
  • Fichier de débogage : fichier de liste du débogueur.
  • Clé de cryptage : si le fichier source est crypté à l’aide de xcode, insérez la clé ici.
  • Minimize R-code Size : Supprimez certaines informations de débogage pour garder le r-code petit.
  • Générer MD-5 : Principalement pour la compilation WebClient.

Utilisation de base

  1. Lancez le compilateur
  2. Ajouter un chemin (s’il n’a pas déjà été enregistré lors de la dernière session)
  3. Appuyez sur F2 pour compiler.
  4. Observez les éventuelles erreurs.
  5. Quitter

Instruction COMPILE #

L’instruction compile vous permet de compiler des programmes dans Progress ABL :

Utilisation de base :

COMPILE hello-world.p SAVE.

Avec une variable :

DEFINE VARIABLE prog AS CHARACTER   NO-UNDO.

prog = "hello.p".

COMPILE VALUE(prog) SAVE.

Il existe plusieurs options pour l’instruction COMPILE :

SAVE indique que le code .r doit être enregistré pour une utilisation future.

COMPILE hello-world.p SAVE.

SAVE INTO dir OR SAVE INTO VALUE(dir-variable) enregistre le r-code dans le répertoire spécifié :

COMPILE hello-world.p SAVE INTO /usr/sources.

Fichier LISTE. Crée un fichier de liste contenant des informations de débogage concernant les blocs, les inclusions, etc.

COMPILE program.p SAVE LISTING c:\temp\listing.txt.

La liste a quelques options pour ajouter des fichiers, la taille de la page et la largeur de la page :

APPEND PAGE-SIZE num PAGE-WIDTH num

XREF xreffile enregistrera un fichier xref du compilateur contenant des informations sur l’utilisation des chaînes et des index, etc. Vous pouvez également APPEND celui-ci.

COMPILE checkFile.p SAVE XREF c:\directory\xref-file.txt.

XREF-XML xreffile-or-dir fera la même chose que XREF mais enregistre le fichier au format xml à la place. Si vous utilisez un répertoire, le fichier xref sera nommé nomprogramme.xref.xml.

COMPILE file.p SAVE XREF c:\temp\.

NO-ERROR supprimera toute erreur d’arrêt de votre programme.

COMPILE program SAVE NO-ERROR.

DEBUG-LIST file génère un fichier de débogage avec des numéros de ligne.

COMPILE checkFile.p SAVE DEBUG-LIST c:\temp\debug.txt.

Le fichier PREPROCESS traduira d’abord tous les préprocesseurs, puis créera un nouveau fichier .p avec le code avant la compilation.

 COMPILE checkFile.p SAVE PREPROCESS c:\temp\PREPROC.txt.

Clé XCODE compilera un code source crypté avec clé comme clé. Vous ne pouvez pas utiliser XCODE avec les options XREF, XREF-XML, STRING-XREF ou LISTING ensemble.

COMPILE program.p SAVE XCODE myKey.

Vous pouvez combiner plusieurs options :

COMPILE prog.p SAVE INTO /usr/r-code XREF /usr/xrefs/xref.txt APPEND LISTING /usr/listings.txt APPEND NO-ERROR.

Descripteur du système COMPILER #

Le descripteur système COMPILER vous permet de consulter les informations concernant une compilation récente.

En supposant que ok-program.p est un programme sans erreur ni avertissement :

COMPILE ok-program.p SAVE NO-ERROR.

DEFINE VARIABLE iError AS INTEGER     NO-UNDO.

MESSAGE  
    "Errors: "   COMPILER:ERROR SKIP
    "Warnings: " COMPILER:WARNING SKIP
    "Messages: " COMPILER:NUM-MESSAGES
    VIEW-AS ALERT-BOX INFORMATION.

Cela produira :

[![Aucune erreur][1]][1]

Compilation d’un programme avec un avertissement :

/* program-with-warning.p */
DEFINE VARIABLE c AS CHARACTER   NO-UNDO.
DEFINE VARIABLE i AS INTEGER     NO-UNDO.

c = "hello".
DISPLAY c.
//This RETURN makes the program exit here and the code below unreachable.
RETURN.

IF TRUE THEN DO:
    i = 10. 
END.

Compilation du programme :

COMPILE program-with-warning.p SAVE.

DEFINE VARIABLE iError AS INTEGER     NO-UNDO.

MESSAGE  
    "Errors: "   COMPILER:ERROR SKIP
    "Warnings: " COMPILER:WARNING SKIP
    "Messages: " COMPILER:NUM-MESSAGES
    VIEW-AS ALERT-BOX INFORMATION.

DO iError = 1 TO COMPILER:NUM-MESSAGES:
    DISPLAY 
        COMPILER:GET-FILE-NAME(iError)    LABEL "Filename" FORMAT "x(20)"
        COMPILER:GET-MESSAGE(iError)      LABEL "Message"  FORMAT "x(50)"
        COMPILER:GET-NUMBER(iError)       LABEL "Msg#"  
        COMPILER:GET-ERROR-COLUMN(iError) LABEL "Column"
        COMPILER:GET-ERROR-ROW(iError)    LABEL "Row"
            WITH FRAME fr1 SIDE-LABELS 1 COLUMNS.
END.

Compilation d’un programme avec une erreur

DEFINE VARIABLE c AS CHARACTER   NO-UNDO.
DEFINE VARIABLE i AS INTEGER     NO-UNDO.

c = "hello".
DISPLAY c.
//Casting should be required below...
IF TRUE THEN DO:
    i = c.
END.

Compilation du programme :

//Use no-errors to supress any error messages from interrupting us.
COMPILE c:\temp\program-with-error.p SAVE NO-ERROR.

DEFINE VARIABLE iError AS INTEGER     NO-UNDO.

MESSAGE  
    "Errors: "   COMPILER:ERROR SKIP
    "Warnings: " COMPILER:WARNING SKIP
    "Messages: " COMPILER:NUM-MESSAGES
    VIEW-AS ALERT-BOX INFORMATION.

DO iError = 1 TO COMPILER:NUM-MESSAGES:
    DISPLAY 
        COMPILER:GET-FILE-NAME(iError)    LABEL "Filename" FORMAT "x(20)"
        COMPILER:GET-MESSAGE(iError)      LABEL "Message"  FORMAT "x(50)"
        COMPILER:GET-NUMBER(iError)       LABEL "Msg#"  
        COMPILER:GET-ERROR-COLUMN(iError) LABEL "Column"
        COMPILER:GET-ERROR-ROW(iError)    LABEL "Row"
            WITH FRAME fr1 SIDE-LABELS 1 COLUMNS 20 DOWN.

    DOWN WITH FRAME fr1.
END.
