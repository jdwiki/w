Opérateurs prédéfinis #

Opérateurs prédéfinis selon ISO/IEC 13211-1 et 13211-2 :

Priorité Taper Opérateur(s) Utiliser
1200 xfx :- -->
1200 effet :- ?- Directive, requête
1100 xfy ;
1050 xfy ->
1000 xfy ','
900 fy \+
700 xfx = \\= Unification des termes
700 xfx == \\== @< @=< @> @>= Comparaison de termes
700 xfx =..
700 xfx est =:= =\= < > =< >= Évaluation et comparaison arithmétiques
600 xfy : Qualification des modules
500 yfx + - /\ \/
400 yfx * / div mod // rem << >>
200 xfx ** Puissance flottante
200 xfy ^ Quantification variable, puissance entière
200 fy + - \ Identité arithmétique, négation ; complément au niveau du bit

Déclaration de l’opérateur #

Dans Prolog, les opérateurs personnalisés peuvent être définis à l’aide de op/3 :

op(+Précédence, +Type, :Opérateur)

  • Déclare Operator comme étant un opérateur d’un Type avec une Priorité. L’opérateur peut également être une liste de noms, auquel cas tous les éléments de la liste sont déclarés être des opérateurs identiques.

  • La priorité est un entier compris entre 0 et 1200, où 0 supprime la déclaration.

  • Le type est l’un des suivants : xf, yf, xfx, xfy, yfx, fy ou fx où f indique la position du foncteur et x et y indique les positions des arguments. y désigne un terme avec une précédence inférieure ou égale à la précédence du foncteur, alors que x dénote une précédence strictement inférieure.

  • Préfixe : fx , fy

  • Infix : xfx (non associatif), xfy (associatif à droite), yfx (associatif à gauche)

  • Suffixe : xf , yf

Exemple d’utilisation :

:- op(900, xf, is_true).

X_0 is_true :-
  X_0.

Exemple de requête :

?- dif(X, a) is_true.
dif(X, a).

Ordre des termes #

Deux termes peuvent être comparés via la commande standard :

variables @< nombres @< atomes @< chaînes @< structures @< listes

Remarques:

  • Les structures se comparent alphabétiquement par foncteur d’abord, puis par arité et enfin par la comparaison de chaque argument.

  • Les listes se comparent d’abord par longueur, puis par chaque élément.

Opérateur de commande Réussit si
X @< Y X est inférieur à Y dans l’ordre standard
X @> Y X est supérieur à Y dans l’ordre standard
X @=< Y X est inférieur ou égal à Y dans l’ordre standard
X @>= Y X est supérieur ou égal à Y dans l’ordre standard

Exemples de requêtes :

?- alpha @< beta.
true.

?- alpha(1) @< beta.
false.

?- alpha(X) @< alpha(1).
true.

?- alpha(X) @=< alpha(Y).
true.

?- alpha(X) @> alpha(Y).
false.

?- compound(z) @< compound(inner(a)).
true.

Égalité des termes #

Opérateur d’égalité Réussit si
X = Y X peut être unifié avec Y
X \= Y X ne peut pas être unifié avec Y
X == Y X et Y sont identiques (c’est-à-dire qu’ils s’unifient sans qu’aucune liaison de variable ne se produise)
X \== Y X et Y ne sont pas identiques
X =:= Y X et Y sont arithmétiquement égaux
X =\= Y X et Y ne sont pas arithmétiquement égaux
