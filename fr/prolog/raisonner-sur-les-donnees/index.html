Une nouvelle section intitulée Structures de données a vu le jour où des explications sur certaines structures + quelques exemples simples de création sont fournis. Pour que son contenu reste concis et épuré, il ne doit contenir aucune documentation sur la manipulation des données.

Par conséquent, cette section a été renommée “Raisonnement sur les données” avec pour objectif la généralisation du raisonnement sur les données en Prolog. Cela pourrait inclure des sujets allant de «l’inférence descendante» à «l’exploration de listes», ainsi que de nombreux autres. En raison de sa large généralisation, des sous-sections claires doivent être faites !

Récursivité #

Prolog n’a pas d’itération, mais toutes les itérations peuvent être réécrites en utilisant la récursivité. La récursivité apparaît lorsqu’un prédicat contient un but qui se réfère à lui-même. Lors de l’écriture de tels prédicats en Prolog, un motif récursif standard comporte toujours au moins deux parties :

  • Clause de base (non récursive) : généralement, la ou les règles de cas de base représenteront le ou les exemples les plus petits possibles du problème que vous essayez de résoudre : une liste sans membre ou un seul membre, ou si vous travaillez avec une structure arborescente, il peut s’agir d’un arbre vide, ou d’un arbre avec un seul nœud, etc. Il décrit de manière non récursive la base du processus récursif.

  • Clause récursive (continue) : contient toute logique requise, y compris un appel à elle-même, poursuivant la récursivité.

A titre d’exemple, nous définirons le prédicat bien connu append/3. Vu de manière déclarative, append(L1,L2,L3) est valable lorsque la liste L3 est le résultat de l’ajout des listes L1 et L2. Lorsque nous essayons de comprendre le sens déclaratif d’un prédicat, nous essayons de décrire les solutions pour lesquelles le prédicat est valable. La difficulté ici consiste à essayer d’éviter tout détail récurrent étape par étape tout en gardant à l’esprit le comportement procédural que le prédicat doit présenter.

% Base case
append([],L,L).

% Recursive clause
append([X|L1],L2,[X|L3]) :- append(L1,L2,L3).

Le cas de base déclare de manière déclarative “tout L ajouté à la liste vide est L”, notez que cela ne dit rien sur le fait que L soit vide - ou même qu’il s’agisse d’une liste (rappelez-vous, dans Prolog, tout se résume à des termes):

?- append(X,some_term(a,b),Z).
X = [],
Z = some_term(a, b).

Pour décrire la règle récursive, bien que Prolog exécute les règles de gauche à droite, nous omettons la tête pendant une seconde et regardons d’abord le corps - en lisant la règle de droite à gauche :

<pré> append([X|L1],L2,[X|L3]) :- append(L1,L2,L3).

Maintenant, nous disons que si le corps tient : “en supposant que append(L1,L2,L3) tient”

<pré> append([X|L1],L2,[X|L3]) :- append(L1,L2,L3).

Il en va de même pour la tête : “alors il en va de même pour append([X|L1],L2,[X|L3])”

En clair, cela se traduit simplement par :

En supposant que L3 est la concaténation de L1 et L2, alors [X suivi de L3] est aussi la concaténation de [X suivi de L1] et L2.

Dans un exemple pratique :

« En supposant que [1,2,3] est la concaténation de [1] et [2,3], alors [a,1,2,3] est aussi la concaténation de [a,1] and [2,3].”

Examinons maintenant quelques requêtes :

C’est toujours une bonne idée de tester initialement votre prédicat avec la requête la plus générale plutôt que de lui fournir un cas de test de scénario spécifique. Pensez-y : en raison de l’unification de Prolog, nous ne sommes pas obligés de fournir des données de test, nous lui donnons juste des variables libres !

?- append(L1,L2,L3).
L1 = [],
L2 = L3 ;                                   % Answer #1
L1 = [_G1162],
L3 = [_G1162|L2] ;                          % Answer #2
L1 = [_G1162, _G1168],
L3 = [_G1162, _G1168|L2] ;                  % Answer #3
L1 = [_G1162, _G1168, _G1174],
L3 = [_G1162, _G1168, _G1174|L2] ;          % Answer #4
...

Remplaçons la notation de type variable libre _G1162 par des lettres alphabétiques pour obtenir un meilleur aperçu :

?- append(L1,L2,L3).
L1 = [],
L2 = L3 ;                                   % Answer #1
L1 = [_A],
L3 = [_A|L2] ;                              % Answer #2
L1 = [_A, _B],
L3 = [_A, _B|L2] ;                          % Answer #3
L1 = [_A, _B, _C],
L3 = [_A, _B, _C|L2] ;                      % Answer #4
...

Dans la première réponse, le cas de base a été mis en correspondance avec le modèle et Prolog a instancié L1 à la liste vide et unifié L2 et L3 prouvant que L3 est la concaténation de la liste vide et de L2.

À la réponse #2, par retour en arrière chronologique, la clause récursive entre en jeu et Prolog essaie de prouver qu’un élément dans la tête de L1 concaténé avec L2 est L3 avec ce même élément dans sa tête de liste. Pour ce faire, une nouvelle variable libre _A est unifiée avec la tête de L1 et L3 s’avère maintenant être [_A|L2].

Un nouvel appel récursif est effectué, maintenant avec L1 = [_A]. Une fois de plus, Prolog essaie de prouver qu’un élément placé dans la tête de L1, concaténé avec L2 est L3 avec ce même élément dans sa tête. Notez que _A est déjà la tête de L1, ce qui correspond parfaitement à la règle, donc maintenant, par récursivité, Prolog met _A devant une nouvelle variable libre et nous obtenons L1 = [_A,_B] et L3 = [_A,_B|L2]

Nous voyons clairement le motif récursif se répéter et pouvons facilement voir que, par exemple, le résultat de la 100e étape de la récursivité ressemblerait à :

L1 = [X1,X2,..,X99],
L3 = [X1,X2,..,X99|L2]

Remarque : comme c’est typique pour un bon code Prolog, la définition récursive de append/3 nous offre non seulement la possibilité de vérifier si une liste est la concaténation de deux autres listes, elle génère également toutes les réponses possibles satisfaisant les relations logiques avec des listes entièrement ou partiellement instanciées.

Accéder aux listes #

Membre

member/2 a la signature member(?Elem, ?List) et indique true si Elem est membre de List. Ce prédicat peut être utilisé pour accéder à des variables dans une liste, où différentes solutions sont récupérées par retour en arrière.

Exemples de requêtes :

?- member(X, [1,2,3]).
X = 1 ;
X = 2 ;
X = 3.

?- member(X,[Y]).
X = Y.

?- member(X,Y).
Y = [X|_G969] ;
Y = [_G968, X|_G972] ;
Y = [_G968, _G971, X|_G975] ;
Y = [_G968, _G971, _G974, X|_G978]
...

Correspondance de modèle

Lorsque les index auxquels vous devez accéder sont petits, le pattern matching peut être une bonne solution, par exemple :

third([_,_,X|_], X).
fourth([_,_,_,X|_], X).
