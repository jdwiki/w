Introduction à Access SQL #

Lorsque vous utilisez Access, vous pouvez récupérer des données à l’aide de requêtes. Ces requêtes sont construites à l’aide du langage SQL (Structured Query Language). Il est important de comprendre SQL, car cela peut aider à créer des requêtes meilleures et plus utiles.

Lors de la création de requêtes dans Access, vous pouvez basculer vers “SQL View”. Un exemple de requête “select” est présenté ici :

Requêtes d’union (fusion) #

Lorsque vous souhaitez combiner les résultats de plusieurs tables ou requêtes avec des champs similaires dans un seul ensemble de données résultant sans effectuer de jointure relationnelle (c’est-à-dire que vous souhaitez répertorier un ensemble de données immédiatement après l’autre), vous utiliserez une requête “UNION”. Cependant, il est à noter que ces requêtes doivent être créées manuellement dans SQL View.

La syntaxe d’une requête “UNION” est

SELECT
  floatingpoint_field AS floatptfld,
  text_field
FROM first_table
  UNION
SELECT
  integer_field,
  decimal_field
FROM a_saved_query
  UNION
SELECT
  1.0,
  "hi there Jack"

et renverra un ensemble de données à deux champs avec des noms de champs (colonnes) : floatptfld et text_field

Il est essentiel que les types de données (et les styles de données) des champs de tableaux fusionnés ultérieurement soient compatibles avec la première requête de la série. En d’autres termes, si la première requête SELECT génère un nombre pour la première colonne, la deuxième requête doit également renvoyer un nombre dans la première colonne. En plus de faire correspondre les types de champs dans l’ordre, les instructions SELECT doivent renvoyer le même nombre de champs. Les noms des champs de la feuille de données résultante sont hérités de la première définition de table.

La requête suivante ne serait PAS légale, car le texte ne peut pas être transformé en données décimales et les nombres à virgule flottante ne peuvent pas non plus être convertis en nombres entiers (sans troncature ou arrondi explicite ni conversion de type).

SELECT
  integer_field AS this_really_wont_turn_out_well,
  decimal_field
FROM a_saved_query
  UNION
SELECT
  floatingpoint_field,
  text_field
FROM first_table

La fonction COUNT() #

Vous pouvez utiliser la fonction COUNT() pour renvoyer le nombre d’enregistrements correspondant à une requête. Le tableau « Employé » suivant contient les numéros d’identification des employés et le numéro d’identification de leur responsable associé.

Employé_ID Manager_ID
12 37
22 37
37 63
42 45
45 63
57 45
59 45
63

Une instruction COUNT() peut être utilisée pour savoir combien d’employés ont un responsable spécifique :

SELECT COUNT(*) AS CNT FROM Employees WHERE Employee.Manager_ID = 37;

Retour

CNT
2

La fonction peut également être combinée dans des requêtes plus complexes. Pour savoir combien d’employés sont directement supervisés par une personne déterminée, on peut appliquer ce qui suit :

SELECT T1.Employee_ID,
    (SELECT COUNT(*) AS CNT FROM Employees AS T2 WHERE T2.Manager_ID =
        T1.Employee_ID) AS Supervised_Count
FROM Employees AS T1;

Retour:

Employé_ID Nombre_supervisé
12 0
22 0
37 2
42 0
45 3
57 0
59 0
63 2

La documentation MSDN peut être trouvée [ici][1].
