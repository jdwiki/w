Migration de <1.10 vers 1.10 et plus

Datatables 1.10.x est la dernière version à ce jour. Bien qu’il soit rétrocompatible avec les versions précédentes (1.9 etc.), il est fortement conseillé d’utiliser la dernière version qui renvoie directement un objet datatable api.

Un autre changement majeur, qui est le plus visible, est le passage de la notation hongroise à la [camelcase](https://en.wikipedia.org/wiki/ Affaire de chameau)

Syntaxe #

  • Le seul changement de syntaxe majeur est l’utilisation de camelCase partout au lieu de la notation hongroise.
  • Un guide plus détaillé peut être trouvé ici qui implique la conversion des paramètres en <1.10 vers 1.10+

Pour plus de détails, essayez de visiter les pages suivantes :

  1. Mise à niveau vers Datatables 1.10
  2. Changelog Datatables 1.10
  3. [Convertir les noms de paramètres de 1.9 à 1.10] (https://datatables.net/upgrade/1.10-convert)
  4. Utilisation de l’API Datatable

Initialisation de Datatable 1.10+ #

Auparavant, les tables de données étaient initialisées comme suit :

var oTable = $("#selector").dataTable();

Cela servait à retourner un objet jQuery qui serait stocké dans la variable oTable. Et puis pour accéder à l’api pour modifier les propriétés de la table, nous avons dû initialiser l’api différemment, comme indiqué ci-dessous :

var api = oTable.api()
//r
var api = new $.fn.dataTable.api("#selector")

Maintenant, l’initialisation est modifiée comme suit :

var table = $("#selector").DataTable();

Notez que cela renvoie une instance datatable api et que vous pouvez directement utiliser la variable table pour manipuler les propriétés de la table de données.

Fonctionnalités Non disponible dans Datatables 1.10+ #

Les 3 fonctionnalités suivantes (qui étaient obsolètes en 1.9) ne sont plus disponibles en 1.10, ce sont :

  1. fnRender : selon le développeur :

L’ancienne option fnRender fournissait une méthode de manipulation d’une cellule il a été créé. cependant, il a été fourni avec une liste déroutante de options comme arguments, et nécessitait une structure particulière dans DataTables en interne qui ont causé des problèmes de performances. Enlèvement de fnRender a conduit à une amélioration significative des performances de DataTables avec de grands ensembles de données et la possibilité de fournir des objets instances de DataTables en tant qu’objets de source de données (par exemple Knockout objets observables).

Des alternatives à fnRender sont disponibles en tant que columns.render et columns .createdCell

  1. bScrollInfinite : selon le développeur :

La capacité intégrée de DataTables 1.9 à afficher un défilement infini la grille via l’option bScrollInfinite a été supprimée en raison de la incohérences qu’il a causées dans l’API. La suppression a également aidé simplement le code interne de manière significative.

Une extension qui porte le nom de Scroller est disponible comme alternative.

  1. Enregistrement d’état basé sur les cookies :

La sauvegarde de l’état basée sur les cookies a été remplacée par la sauvegarde basée sur localStorage enregistrement de l’état dans DataTables 1.10. Cookie’s, avec leur limite de 4 Kio, étaient très limité, et encouru une pénalité de performance puisqu’ils faisaient partie de chaque requête HTTP. localStorage est beaucoup plus rapide et plus flexible, et est utilisé comme stockage par défaut pour les informations d’état dans DataTables 1.10.

  1. Contrôle de pagination “two_button” :

DataTables 1.10 a considérablement amélioré les contrôles de pagination de DataTables (voir pagingType), ce qui a pour conséquence que l’ancienne forme de pagination intégrée à deux boutons a été supprimé.

Ils ont pris soin des personnes qui souhaitent toujours utiliser la méthode de pagination two_button en fournissant un fichier javascript supplémentaire appelé two_button.js. L’utilisation est la suivante :

Incluez simplement ce fichier dans votre document, après avoir chargé DataTables mais avant d’avoir initialisé votre table et la pagination à deux boutons sera restaurée exactement telle qu’elle était dans la version 1.9 (y compris les noms de classe, etc.).
