Installation à partir des sources dans un système Debian/Ubuntu #

Instructions détaillées sur la configuration ou l’installation d’openerp dans Debian/Ubuntu.

Pour installer à partir du code source, nous avons besoin de Python 2.7, Git et d’une base de données PostgreSQL :

$ sudo apt-get install git python-pip python2.7-dev -y
$ sudo apt-get install postgresql -y
$ sudo su -c "createuser -s $(whoami)" postgres

Ensuite, nous avons besoin de certaines dépendances système, requises par les packages Python utilisés par Odoo :

$ sudo apt-get install libxml2-dev libxslt1-dev \
    libevent-dev libsasl2-dev libpq-dev \
    libpng12-dev libjpeg-dev \
    node-less node-clean-css -y

Les dépendances Python sont déclarées dans le fichier requirements.txt :

$ wget https://raw.githubusercontent.com/odoo/odoo/master/requirements.txt
$ sudo -H pip install --upgrade pip
$ sudo -H pip install -r requirements.txt

Vous pouvez maintenant obtenir le code source de GitHub et commencer à créer votre première instance Odoo :

$ git clone https://github.com/odoo/odoo.git -b 9.0 --depth=1
$ ./odoo/odoo.py -d myodoo

Odoo sur AWS #

Noter: #

Le chargement d’Odoo sur un conteneur AWS EC2 nécessite un compte AWS #

Le chargement d’Odoo sur une instance AWS EC2 peut se faire en un seul clic, il suffit d’aller ici ou recherchez “Odoo AWS” dans Google.

Cela peut prendre un certain temps, mais une fois que c’est prêt, vous devrez faire deux choses :

  1. Obtenez votre mot de passe
  2. Trouvez votre nom d’utilisateur

Obtenir votre mot de passe #

Dans votre tableau de bord Amazonaws EC2, sélectionnez votre nouvelle instance Odoo et cliquez sur le bouton “Actions” en haut. Choisissez : Actions > Paramètres de l’instance > Obtenir le journal système. Faites défiler le journal système jusqu’à ce que vous trouviez la partie qui vous indique Réglage du mot de passe de l’application Bitnami sur xxxxxxxxx. Cela ressemblera à ceci :

[![entrez la description de l’image ici][1]][1]

Assurez-vous de changer votre mot de passe une fois que vous êtes connecté au portail d’applications #

Obtenir votre nom d’utilisateur #

Votre nom d’utilisateur peut être trouvé à partir de la page sur laquelle vous avez téléchargé l’image de la machine Odoo dans la section intitulée Instructions d’utilisation. Cependant, l’e-mail par défaut est généralement user@example.com ou admin@example.com

Après l’installation #

Une fois Odoo installé, il ajoutera également son propre groupe de sécurité à votre compte et appliquera ce groupe de sécurité à lui-même. Par défaut, ce groupe de sécurité autorise le trafic entrant suivant

Ports Protocole La source
80 TCP 0.0.0.0/0
22 TCP 0.0.0.0/0
443 TCP 0.0.0.0/0

Cela permettra des connexions HTTP et HTTPS de base ainsi que des connexions SSH et SFTP. Si vous souhaitez modifier ces paramètres, vous pouvez cliquer sur le groupe de sécurité et modifier les règles sur la balise inboud OU vous pouvez ajouter un nouveau groupe de sécurité en cliquant sur Actions > Mise en réseau > Modifier les groupes de sécurité dans l’EC2 Tableau de bord

lien pour odoo sur aws : https://aws.amazon.com/marketplace/pp/B00QBWCCQ6/ref_=_mkt_ste_menu?nc2=h_l3_ba #

Gérer les instances de serveur #

Une fois Odoo installé, nous devons créer une instance de serveur. Une instance de serveur est un service Odoo écoutant sur un port spécifique, 8060, par défaut, et utilisant une base de données pour stocker des données.

La commande minimale pour démarrer une instance Odoo en utilisant la base de données mydb :

$ ./odoo.py -d mydb

Si la base de données n’existe pas encore, Odoo essaiera automatiquement de la créer, puis créera automatiquement toutes les tables de base de données dont il a besoin. Si l’utilisateur qui démarre le service n’a pas les privilèges pour créer des bases de données (une bonne pratique de sécurité), nous devons utiliser la commande PostgreSQL createdb pour le créer. préalablement.

Vous trouverez ci-dessous un exemple des options les plus couramment utilisées lorsque vous travaillez avec Odoo :

$ ./odoo.py -d mydb --db-filter=^mydb$ --addons-path=./addons,../myaddons --xmlrpc-port=8080

Dépôts nocturnes d’Odoo #

Odoo fournit des programmes d’installation officiels, pour différents types de systèmes d’exploitation et versions Openerp/Odoo, sur le [dépôt Nightly][1]

Option 1 : installation manuelle à partir du package d’installation prédéfini (Windows, Linux) : #

Téléchargez le programme d’installation/paquet de la version de votre choix pour votre système (Windows (.exe), Debian/Ubuntu (.deb), Redhat/CentOS(.rpm)) depuis le ** dépôt nocturne d’Odoo ** : http://nightly .odoo.com/ et installez-le.

Option 2 : Configurez l’entrée du référentiel pour obtenir d’autres mises à jour à l’aide du système de gestion des packages de système d’exploitation standard (Linux uniquement) : #

  • Mise en place du dépôt nocturne Debian [détails][2]
    wget -O - https://nightly.odoo.com/odoo.key | apt-key add -
    echo "deb http://nightly.odoo.com/8.0/nightly/deb/ ./" >> /etc/apt/sources.list
    apt-get update && apt-get install odoo
  • Mise en place du dépôt nocturne RedHat [détails][3]
    yum-config-manager --add-repo=https://nightly.odoo.com/8.0/nightly/rpm/odoo.repo
    yum update && yum install odoo

Noter: #

Des exemples sont montrés Odoo version 8.0 liens de référentiel, adaptez ces liens pour installer une version différente en changeant /8.0/ dans les URL de référentiel ci-dessus en /7.0/ pour Openerp v7 ou /9.0/ pour Odoo v9, selon votre version de choix

[1] : http://nightly.odoo.com/ [2] : https://www.odoo.com/documentation/8.0/setup/install.html#deb [3] : https://www.odoo.com/documentation/8.0/setup/install.html#rpm
