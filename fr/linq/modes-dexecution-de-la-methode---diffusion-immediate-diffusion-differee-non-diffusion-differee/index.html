Exécution différée vs exécution immédiate #

Certaines méthodes LINQ renvoient un objet de requête. Cet objet ne contient pas les résultats de la requête ; à la place, il contient toutes les informations nécessaires pour générer ces résultats :

var list = new List<int>() {1, 2, 3, 4, 5};
var query = list.Select(x => {
    Console.Write($"{x} ");
    return x;
});

La requête contient un appel à Console.Write, mais rien n’a été envoyé à la console. C’est parce que la requête n’a pas encore été exécutée, et donc la fonction passée à Select n’a jamais été évaluée. C’est ce qu’on appelle exécution différée – l’exécution de la requête est retardée jusqu’à plus tard indiquer.

D’autres méthodes LINQ forcent une exécution immédiate de la requête ; ces méthodes exécutent la requête et génèrent ses valeurs :

var newList = query.ToList();

À ce stade, la fonction transmise à Select sera évaluée pour chaque valeur de la liste d’origine, et les éléments suivants seront affichés sur la console :

1 2 3 4 5

Généralement, les méthodes LINQ qui renvoient une seule valeur (telle que Max ou Count), ou qui renvoient un objet qui contient réellement les valeurs (telles que ToList ou ToDictionary) s’exécutent immédiatement.

Les méthodes qui renvoient un IEnumerable<T> ou IQueryable<T> renvoient l’objet de requête et permettent de reporter l’exécution à un point ultérieur.

Si une méthode LINQ particulière force une requête à s’exécuter immédiatement ou non, peut être trouvée sur MSDN – C#, ou [VB. NET] (https://msdn.microsoft.com/en-us/library/mt692840.aspx).

Mode streaming (évaluation paresseuse) vs mode non-streaming (évaluation impatiente) #

Parmi les méthodes LINQ qui utilisent l’exécution différée, certaines nécessitent qu’une seule valeur soit évaluée à la fois. Le code suivant :

var lst = new List<int>() {3, 5, 1, 2};
var streamingQuery = lst.Select(x => {
    Console.WriteLine(x);
    return x;
});
foreach (var i in streamingQuery) {
    Console.WriteLine($"foreach iteration value: {i}");
}

affichera :

3 valeur d’itération foreach : 3 5 valeur d’itération foreach : 5 1 valeur d’itération pour chaque : 1 2 valeur d’itération foreach : 2

car la fonction passée à Select est évaluée à chaque itération de foreach. Ceci est connu sous le nom de mode de diffusion en continu ou évaluation paresseuse.

D’autres méthodes LINQ – opérateurs de tri et de regroupement – exigent que toutes les valeurs soient évaluées, avant de pouvoir renvoyer toute valeur :

var nonStreamingQuery = lst.OrderBy(x => {
    Console.WriteLine(x);
    return x;
});
foreach (var i in nonStreamingQuery) {
    Console.WriteLine($"foreach iteration value: {i}");
}

affichera :

3 5 1 2 valeur d’itération pour chaque : 1 valeur d’itération foreach : 2 valeur d’itération foreach : 3 valeur d’itération foreach : 5

Dans ce cas, étant donné que les valeurs doivent être générées dans foreach dans l’ordre croissant, tous les éléments doivent d’abord être évalués, afin de déterminer lequel est le plus petit, et lequel est le plus petit suivant, et ainsi de suite. Ceci est connu sous le nom de mode non-streaming ou évaluation rapide .

Si une méthode LINQ particulière utilise le mode streaming ou non, peut être trouvée sur MSDN – C#, ou VB.NET .

Avantages de l’exécution différée - création de requêtes #

L’exécution différée permet de combiner différentes opérations pour construire la requête finale, avant d’évaluer les valeurs :

var list = new List<int>() {1,1,2,3,5,8};
var query = list.Select(x => x + 1);

Si nous exécutons la requête à ce stade :

foreach (var x in query) {
    Console.Write($"{x} ");
}

nous obtiendrions la sortie suivante :

2 2 3 4 6 9

Mais nous pouvons modifier la requête en ajoutant plus d’opérateurs :

Console.WriteLine();
query = query.Where(x => x % 2 == 0);
query = query.Select(x => x * 10);

foreach (var x in query) {
    Console.Write($"{x} ");
}

Production:

20 20 40 60

Avantages de l’exécution différée - interrogation des données actuelles #

Avec l’exécution différée, si les données à interroger sont modifiées, l’objet de requête utilise les données au moment de l’exécution, et non au moment de la définition.

var data = new List<int>() {2, 4, 6, 8};
var query = data.Select(x => x * x);

Si nous exécutons la requête à ce stade avec une méthode immédiate ou foreach, la requête fonctionnera sur la liste des nombres pairs.

Cependant, si nous modifions les valeurs dans la liste :

data.Clear();
data.AddRange(new [] {1, 3, 5, 7, 9});

ou même si nous attribuons une nouvelle liste à data :

data = new List<int>() {1, 3, 5, 7, 9};

puis exécutez la requête, la requête fonctionnera sur la nouvelle valeur de data :

foreach (var x in query) {
    Console.Write($"{x} ");
}

et affichera ce qui suit :

1 9 25 49 81
