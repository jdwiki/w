Syntaxe #

  • ember déployer la production // déployer l’environnement de production
  • ember deploy staging // déployer l’environnement de staging
  • ember deploy development // déploie un environnement de développement non compressé et minifié

Ember-Cli est un outil puissant qui vient avec beaucoup d’autres pour nous aider à déployer plus rapidement et plus facilement. Tout ce dont vous avez besoin pour installer [Ember-Cli-Deploy][1] et utiliser ember deploy.

Ember CLI Deploy structure le déploiement de votre application à l’aide d’un pipeline, qui se compose de plusieurs crochets de pipeline. Ces normes les crochets sont la base d’un riche écosystème de plugins que vous pouvez composer pour créer un processus de déploiement adapté à votre application.

Comme Ember-Cli-Deploy est un module complémentaire d’Ember, vous pouvez donc l’installer facilement avec ember install ember-cli-deploy. Il existe deux autres modules complémentaires utiles qui rendent notre construction et notre compression fiables pendant le déploiement.

Exécutez simplement les commandes suivantes :

# Install the Build plugin, which builds your app during deployment
ember install ember-cli-deploy-build

# Gzip our files
ember install ember-cli-deploy-gzip

Cependant, si vous voulez maximiser vos avantages en utilisant ember deploy, il est plus probable que votre application Ember ait différents environnements et que vous déployiez la version de production, de préproduction ou de développement de votre application avec la configuration appropriée.

Les plates-formes que vous pouvez déployer maintenant sont :

-Heroku

  • Azur -AWS S3
  • Base de feu
  • Grappe CouchDB

Veuillez vous référer à la section d’exemple pour voir comment vous pouvez déployer.

[1] : http://ember-cli-deploy.com/

##AWS S3 Pour procéder au déploiement sur S3, nous allons installer ces plugins :

  • [ember-cli-deploy-s3-index][1] : il télécharge le fichier index.html sur S3 avec les informations de révision et l’active
  • [ember-cli-deploy-s3][2] : il télécharge les actifs (js, css et autres fichiers multimédias) sur S3

Comme il s’agit d’un addon Ember, vous pouvez facilement l’installer en exécutant les commandes suivantes

ember install ember-cli-deploy-s3-index
ember install ember-cli-deploy-s3

Tout ce dont vous avez besoin après cela est de configurer le fichier deploy.js qui devrait se trouver dans le dossier /config :

// config/deploy.js

module.exports = function(deployTarget) {

  var ENV = {
    build: {
      environment: deployTarget
    },
    'revision-data': {
      type: 'git-commit'
    },
    's3-index': {
      accessKeyId: process.env['S3_ACCESS_KEY'],
      secretAccessKey: process.env['S3_SECRET_ACCESS_KEY'],
      bucket: "your-app-deployment-bucket",
      region: "YOUR REGISION",
      allowOverwrite: true // if you want to overwrite index file if not change it to false
    },
    's3': {
      accessKeyId: process.env['S3_ACCESS_KEY'],
      secretAccessKey: process.env['S3_SECRET_ACCESS_KEY'],
      bucket: "your-app-deployment-bucket",
      region: "YOUR REGISION",
    }
  };

  return ENV;

};

Notez que nous avons utilisé les variables d’environnement dans notre configuration. Si vous avez besoin de lire la configuration à partir d’un fichier, il est également possible de renvoyer une promesse qui se résout avec l’objet “ENV”. Voici un exemple pour définir différents environnements dans le fichier deploy.js :

if (deployTarget === 'development') {
    ENV.build.environment = 'development';
    // configure other plugins for development deploy target here
  }

  if (deployTarget === 'staging') {
    ENV.build.environment = 'production';
    // configure other plugins for staging deploy target here
  }

  if (deployTarget === 'production') {
    ENV.build.environment = 'production';
    // configure other plugins for production deploy target here
  }

Enfin, vous pouvez facilement exécuter la commande suivante pour déployer

ember deploy [YOUR APP ENVIRONMENT] //e.g-> ember deploy production or ember deploy staging

si vous aimez voir les détails, vous pouvez exécuter :

ember deploy production --verbose --activate=true

[1] : https://github.com/ember-cli-deploy/ember-cli-deploy-s3-index [2] : https://github.com/ember-cli-deploy/ember-cli-deploy-s3

Heroku #

Vous devez d’abord avoir installé [Heroku Toolbelt][1]. Avoir un compte dans Heroku et l’installation de ember-cli-deploy sont obligatoires.

Création d’une nouvelle instance Heroku à partir du répertoire parent d’une application Ember CLI :

$ heroku create --buildpack https://github.com/tonycoco/heroku-buildpack-ember-cli.git

$ git push heroku master

[1] : http://%20https://toolbelt.heroku.com/

Azur #

Tout d’abord, il est nécessaire d’installer le module de Microsoft [ember-cli-azure-deploy][1]. Vous devez être dans le répertoire racine de votre application.

npm install --save-dev -g ember-cli-azure-deploy
azure-deploy init

Si vous utilisez le gestionnaire de packages Yarn, vous pouvez simplement l’installer en :

yarn global add ember-cli-azure-deploy
azure-deploy init

Cela créera un deploy.sh dans le dossier racine de votre projet, permettant à Azure de suivre un ensemble d’instructions - y compris l’installation de tous les modules de nœud requis, l’exécution de ember build et le déploiement du dossier dist/ résultant sur le de votre site Web. wwwroot.

[1] : https://github.com/felixrieseberg/ember-cli-azure-deploy

Firebase #

Tout d’abord, vous devez installer [Firebase tools][1]. Exécutez simplement les commandes ci-dessous :

Gestionnaire de paquets Npm

npm install -g firebase-tools

ou gestionnaire de paquets de fil

yarn add firebase-tools

Pour configurer votre application afin qu’elle soit prête à être déployée, vous devez exécuter ce qui suit dans le répertoire racine de votre application :

firebase init

enfin, en exécutant la commande suivante, vous pouvez déployer votre application

firebase deploy

[1] : http://%20https://github.com/firebase/firebase-tools
