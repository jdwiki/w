Conditions #

Perl prend en charge de nombreux types d’instructions conditionnelles (instructions basées sur des résultats booléens). Les instructions conditionnelles les plus courantes sont if-else, à moins que et les instructions ternaires. Les instructions données sont introduites comme une construction de type commutateur à partir de langages dérivés du C et sont disponibles dans les versions Perl 5.10 et supérieures.

Instructions Si-Sinon #

La structure de base d’une instruction if ressemble à ceci :

if (EXPR) BLOCK
if (EXPR) BLOCK else BLOCK
if (EXPR) BLOCK elsif (EXPR) BLOCK ...
if (EXPR) BLOCK elsif (EXPR) BLOCK ... else BLOCK

Pour les instructions if simples, le if peut précéder ou succéder au code à exécuter.

$number = 7;
if ($number > 4) { print "$number is greater than four!"; }

# Can also be written this way
print "$number is greater than four!" if $number > 4;

Boucles #

Perl prend en charge de nombreux types de constructions de boucles : for/foreach, while/do-while et until.

@numbers = 1..42;
for (my $i=0; $i <= $#numbers; $i++) {
    print "$numbers[$i]\n";
}

#Can also be written as
foreach my $num (@numbers) {
    print "$num\n";
}

La boucle while évalue la condition avant d’exécuter le bloc associé. Ainsi, parfois le bloc n’est jamais exécuté. Par exemple, le code suivant ne serait jamais exécuté si le filehandle $fh était le filehandle d’un fichier vide, ou s’il était déjà épuisé avant la condition.

while (my $line = readline $fh) {
    say $line;
}

Les boucles do/while et do/until, quant à elles, évaluent la condition after à chaque fois que le bloc est exécuté. Ainsi, une boucle do/while ou do/until est toujours exécutée au moins une fois.

my $greeting_count = 0;
do {
    say "Hello";
    $greeting_count++;
} until ( $greeting_count > 1)

# Hello
