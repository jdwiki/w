Gestion des exceptions

  • évaluer et mourir

Sur cette page

évaluer et mourir #

C’est le moyen intégré de gérer les “exceptions” sans s’appuyer sur des bibliothèques tierces telles que [Try::Tiny][1].

my $ret;

eval {
  $ret = some_function_that_might_die();
  1;
} or do {
  my $eval_error = $@ || "Zombie error!";
  handle_error($eval_error);
};

# use $ret

Nous “abusons” du fait que die a une fausse valeur de retour, et la valeur de retour du bloc de code global est la valeur de la dernière expression dans le bloc de code :

  • si $ret est assigné avec succès, alors l’expression 1; est la dernière chose qui se passe dans le bloc de code eval. Le bloc de code eval a donc une valeur vraie, donc le bloc or do ne s’exécute pas.
  • si some_function_that_might_die() fait die, alors la dernière chose qui se produit dans le bloc de code eval est le die. Le bloc de code eval a donc une valeur fausse et le bloc or do s’exécute.
  • La première chose que vous devez faire dans le bloc or do est de lire $@. Cette variable globale contiendra tout argument passé à die. Le `|| La garde “Zombie Error” est populaire, mais inutile dans le cas général.

Ceci est important à comprendre car certains codes n’échouent pas en appelant die, mais la même structure peut être utilisée malgré tout. Considérez une fonction de base de données qui renvoie :

  • le nombre de lignes affectées en cas de succès
  • '0 mais vrai' si la requête a réussi mais qu’aucune ligne n’a été affectée
  • 0 si la requête n’a pas abouti.

Dans ce cas, vous pouvez toujours utiliser le même idiome, mais vous devez sauter le 1; final, et cette fonction doit être la dernière chose dans l’eval. Quelque chose comme ça:

eval {
  my $value = My::Database::retrieve($my_thing); # dies on fail
  $value->set_status("Completed");
  $value->set_completed_timestamp(time());
  $value->update(); # returns false value on fail
} or do { # handles both the die and the 0 return value
  my $eval_error = $@ || "Zombie error!";
  handle_error($eval_error);
};

[1] : http://p3rl.org/Try::Tiny
