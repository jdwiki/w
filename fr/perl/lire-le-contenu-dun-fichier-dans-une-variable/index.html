Chemin :: Minuscule #

Utilisation de l’idiome de [The Manual Way] (https://www.wikiod.com/fr/perl/lire-le-contenu-dun-fichier-dans-une-variable#La méthode manuelle) plusieurs fois dans un script devient rapidement fastidieux, vous pouvez donc essayer un module.

use Path::Tiny;
my $contents = path($filename)->slurp;

Vous pouvez passer une option binmode si vous avez besoin de contrôler les encodages de fichiers, les fins de ligne, etc. - voir man perlio :

my $contents = path($filename)->slurp( {binmode => ":encoding(UTF-8)"} );

Path::Tiny a également beaucoup d’autres fonctions pour gérer les fichiers afin qu’il puisse être un bon choix.

La méthode manuelle #

open my $fh, '<', $filename
    or die "Could not open $filename for reading: $!";
my $contents = do { local $/; <$fh> };

Après avoir ouvert le fichier (lisez man perlio si vous voulez lire des encodages de fichiers spécifiques au lieu d’octets bruts), l’astuce se trouve dans le bloc do : <$fh>, le descripteur de fichier dans un opérateur diamant, renvoie un seul enregistrement du fichier. La variable “séparateur d’enregistrement d’entrée” $/ spécifie ce qu’est un “enregistrement” - par défaut, il est défini sur un caractère de nouvelle ligne, donc “un enregistrement” signifie “une seule ligne”. Comme $/ est une variable globale, local fait deux choses : il crée une copie locale temporaire de $/ qui disparaîtra à la fin du bloc, et lui donne la (non-)valeur undef (la “valeur” que Perl donne aux variables non initialisées). Lorsque le séparateur d’enregistrement d’entrée a cette (non-) valeur, l’opérateur en losange renverra le fichier entier. (Il considère le fichier entier comme une seule ligne.)

En utilisant do, vous pouvez même contourner l’ouverture manuelle d’un fichier. Pour la lecture répétée de fichiers,

sub readfile { do { local(@ARGV,$/) = $_[0]; <> } }
my $content = readfile($filename);

peut être utilisé. Ici, une autre variable globale (@ARGV) est localisée pour simuler le même processus utilisé lors du démarrage d’un script perl avec des paramètres. $/ est toujours undef, puisque le tableau devant lui “mange” tous les arguments entrants. Ensuite, l’opérateur losange <> fournit à nouveau un enregistrement défini par $/ (le fichier entier) et revient du bloc do, qui à son tour revient du sous.

Le sous-marin n’a pas de gestion explicite des erreurs, ce qui est une mauvaise pratique ! Si une erreur se produit lors de la lecture du fichier, vous recevrez undef comme valeur de retour, par opposition à une chaîne vide provenant d’un fichier vide.

Un autre inconvénient du dernier code est le fait que vous ne pouvez pas utiliser PerlIO pour différents encodages de fichiers - vous obtenez toujours des octets bruts.

Fichier ::Slurp #

Ne l’utilisez pas. Bien qu’il existe depuis longtemps et qu’il soit toujours le module que la plupart des programmeurs suggéreront, [il est cassé et ne sera probablement pas corrigé] (http://blogs.perl.org/users/leon_timmermans/2015/08/fileslurp -is-broken-and-wrong.html).

Fichier ::Slurper #

Il s’agit d’un module minimaliste qui transforme uniquement les fichiers en variables, rien d’autre.

use File::Slurper 'read_text';
my $contents = read_text($filename);

read_text() prend deux paramètres facultatifs pour spécifier l’encodage du fichier et si les fins de ligne doivent être traduites entre les normes unixish LF ou DOSish CRLF :

my $contents = read_text($filename, 'UTF-8', 1);

Transférer un fichier dans une variable tableau #

open(my $fh, '<', "/some/path") or die $!;
my @ary = <$fh>;

Lorsqu’il est évalué dans un contexte de liste, l’opérateur losange renvoie une liste composée de toutes les lignes du fichier (dans ce cas, l’affectation du résultat à un tableau fournit un contexte de liste). Le terminateur de ligne est conservé et peut être supprimé en mordant :

chomp(@ary); #removes line terminators from all the array elements.

Fichier Slurp en une seule ligne #

Le séparateur d’enregistrement d’entrée peut être spécifié avec le commutateur -0 (* zéro *, pas * majuscule O *). Il prend un nombre octal ou hexadécimal comme valeur. Toute valeur 0400 ou supérieure forcera Perl à aspirer les fichiers, mais par convention, la valeur utilisée à cette fin est 0777.

perl -0777 -e 'my $file = <>; print length($file)' input.txt

Pour aller plus loin avec le minimalisme, spécifier le commutateur -n oblige Perl à lire automatiquement chaque ligne (dans notre cas, le fichier entier) dans la variable $_.

perl -0777 -ne 'print length($_)' input.txt
