Ces exemples présentent des exemples concrets d’utilisation des webhooks PayPal pour fournir une surveillance des événements pour votre application et vos paiements.

Test des Webhooks Sandbox avec ngrok et Express (Node) #

Dans cet exemple, nous allons examiner le test des notifications de webhook dans le bac à sable, en utilisant [ngrok][1] pour fournir un tunnel pour notre écouteur Node HTTP, exécuté sur localhost, vers Internet. Pour cet exemple, nous allons utiliser Node pour configurer des webhooks de notification pour les événements de paiement (tels qu’un paiement en cours), puis configurer le serveur pour qu’il écoute les messages HTTP POST entrants à partir des événements du webhook.

Il y a quelques étapes que nous allons suivre ici pour y arriver :

  1. Configurez un serveur simple pour écouter le trafic POST entrant à partir des webhooks, qui sera la notification de PayPal, et commencez à écouter sur localhost.
  2. Utilisez ensuite ngrok pour fournir un tunnel de localhost à Internet afin que PayPal puisse publier une notification.
  3. Enfin, abonnez-vous à notre application (sur la base des informations d’identification fournies) aux événements de webhook que nous souhaitons suivre, en fournissant l’URI ngrok public de l’étape 2.

Création d’un écouteur Webhooks

La première chose que nous devons faire est de créer l’écouteur. La raison pour laquelle nous commençons avec l’écouteur est que nous avons besoin de l’URL en direct ngrok à fournir aux webhooks lorsque nous les créons ou les mettons à jour.

var bodyParser = require('body-parser'),
    http = require('http'),
    app = require('express')();

app.use(bodyParser.json());

app.post('/', function(req, res){
    console.log(JSON.stringify(req.body));
});

//create server
http.createServer(app).listen(3001, function () {
   console.log('Server started: Listening on port 3001');
});

Notre écouteur est un itinéraire simple utilisant Express. Nous écoutons tout trafic POST entrant, puis crachons le corps POST vers la console. Nous pouvons l’utiliser pour faire ce que nous voulons avec l’auditeur quand il arrive.

Lorsque nous créons le serveur HTTP à la fin, nous le configurons pour écouter sur le port 3001 de l’hôte local. Exécutez ce script maintenant pour commencer à écouter le trafic.

Utiliser ngrok pour exposer l’auditeur à Internet

Avec l’écouteur configuré sur localhost:3001, notre prochaine tâche consiste à exposer ce script à Internet, afin que le trafic puisse lui être envoyé, ce qui est le travail de ngrok.

Exécutez la commande suivante depuis une fenêtre de terminal :

ngrok http 3001

Cela lancera le processus de fourniture d’un tunnel en direct pour localhost sur le port 3001 et fournira les informations suivantes une fois exécuté :

[![entrez la description de l’image ici][2]][2]

Comme nous pouvons le voir, l’adresse en direct que nous pouvons utiliser pour pointer le webhook PayPal vers notre écouteur en cours d’exécution sur localhost est http(s)://055b3480.ngrok.io. C’est tout ce que nous devons savoir pour configurer l’écouteur.

S’abonner aux notifications

Notre dernière étape consiste à créer des webhooks pour notre application, qui créeront des notifications lorsque certains événements se produisent avec des paiements, des remboursements, etc. sur notre application. Nous n’avons besoin de créer ces webhooks qu’une seule fois pour les lier à l’application, ils n’ont donc pas besoin d’être exécutés à chaque fois que vous souhaitez les utiliser.

Nous avons d’abord configuré l’environnement PayPal en ajoutant l’exigence du SDK PayPal Node, notre ID client / secret de création d’une application, puis en configurant l’environnement pour le bac à sable.

var paypal = require('paypal-rest-sdk');

var clientId = 'YOUR APPLICATION CLIENT ID';
var secret = 'YOUR APPLICATION SECRET';

paypal.configure({
  'mode': 'sandbox', //sandbox or live
  'client_id': clientId,
  'client_secret': secret
});

Ensuite, nous configurons la structure JSON pour nos webhooks. webhooks contient deux informations, l’url à laquelle tous les événements webhook doivent être envoyés et les event_types auxquels nous voulons nous abonner.

Dans le cas de cet exemple, l’URL est définie sur notre URL en direct ngrok, et les événements que nous écoutons sont des cas où les paiements sont effectués ou refusés.

Pour une liste complète des événements potentiels, consultez https://developer.paypal.com/docs/integration/direct/rest-webhooks-overview/#event-type-support.

Enfin, nous passons l’objet webhooks dans l’appel pour créer les webhooks, notification.webhook.create. En cas de succès, PayPal enverra des notifications au point de terminaison que nous avons spécifié, qui s’exécute sur localhost.

var webhooks = {
    "url": "https://436e4d13.ngrok.io",
    "event_types": [{
        "name": "PAYMENT.SALE.COMPLETED"
    },{
        "name": "PAYMENT.SALE.DENIED"
    }
]};

paypal.notification.webhook.create(webhooks, function (err, webhook) {
    if (err) {
        console.log(err.response);
        throw error;
    } else {
        console.log("Create webhook Response");
        console.log(webhook);
    }
});

Une fois que nous émettons un paiement à l’aide de ces informations d’identification d’application, des informations sur l’état du paiement seront envoyées au point de terminaison que nous avons configuré.

Mise à jour d’un Webhook avec une nouvelle URL (exemple de nœud) #

Cet exemple vous montrera comment mettre à jour une URL de transfert de webhook existante (où les notifications doivent être postées). Pour exécuter cela, vous devez avoir l’identifiant fourni par PayPal lorsque vous avez créé vos webhooks pour la première fois.

Tout d’abord, ajoutez le SDK PayPal et configurez l’environnement (sandbox ci-dessous).

var paypal = require('paypal-rest-sdk');

var clientId = 'YOUR APPLICATION CLIENT ID';
var secret = 'YOUR APPLICATION SECRET';

paypal.configure({
    'mode': 'sandbox', //sandbox or live
    'client_id': clientId,
    'client_secret': secret
});

Ensuite, configurez la structure JSON et les détails du webhook. Attribuez d’abord l’ID de votre webhook à webhookId. Ensuite, dans webhookUpdate, spécifiez une opération de remplacement, définissez le path sur /url pour spécifier une mise à jour de cette ressource et fournissez la nouvelle URL pour la remplacer sous value.

var webhookId = "YOUR WEBHOOK ID";
var webhookUpdate = [{
    "op": "replace",
    "path": "/url",
    "value": "https://64fb54a2.ngrok.io"
}];

Enfin, appelez notification.webhook.replace(...), en transmettant webhookId et webhookUpdate.

paypal.notification.webhook.replace(webhookId, webhookUpdate, function (err, res) { if (err) { console.log(err); throw err; } else { console.log(JSON.stringify(res)); } });

Si tout réussit, un objet similaire au suivant doit être renvoyé par PayPal et, dans le cas de cet exemple, affiché dans le terminal avec les informations nouvellement mises à jour.
