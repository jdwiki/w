Paramètres #

Paramètre Détails
identifiant client Votre identifiant client d’application PayPal (identifiants OAuth 2)
liens Objet de référence simple pour tous les liens de retour HATEOAS de PayPal
identifiant de paiement L’identifiant du paiement renvoyé par PayPal afin de finaliser le paiement
identifiantpayeur L’identifiant du payeur renvoyé par PayPal afin de finaliser le paiement
paypal Référence du SDK PayPal Node
payReq Objet JSON contenant les informations de paiement pour la transaction
demande L’objet de requête du serveur request
res L’objet de réponse de la requête du serveur
secrets Votre secret d’application PayPal (identifiants OAuth 2)

Ces exemples expliquent comment traiter un paiement via PayPal à l’aide des SDK PayPal. Ce sont des exemples de demande simples qui décrivent le processus en plusieurs étapes pour autoriser cette option de paiement.

Exemple de serveur Node Express #

Dans cet exemple, nous allons configurer une intégration de serveur Express pour afficher comment traiter un paiement avec PayPal, à l’aide du SDK PayPal Node. Nous utiliserons une structure JSON statique pour les détails de paiement par souci de brièveté.

Il y a trois étapes générales que nous suivrons lors de la création des fonctions pour gérer le paiement PayPal :

  1. Nous créons un objet JSON contenant le paiement que nous avons l’intention de traiter via PayPal. Nous l’envoyons ensuite à PayPal pour obtenir un lien vers lequel rediriger l’utilisateur afin de confirmer le paiement.
  2. Ensuite, nous redirigeons l’utilisateur vers PayPal pour confirmer le paiement. Une fois confirmé, PayPal redirige l’utilisateur vers notre application.
  3. Une fois revenu à l’application, nous effectuons le paiement au nom de l’utilisateur.

En décomposant cela en une simple application Node, nous commençons par obtenir le SDK PayPal Node de NPM :

npm install paypal-rest-sdk

Ensuite, nous configurons la configuration et les packages de l’application.

var http = require('http'),
    paypal = require('paypal-rest-sdk'),
    bodyParser = require('body-parser'),
    app = require('express')();

var client_id = 'YOUR APPLICATION CLIENT ID';
var secret = 'YOUR APPLICATION SECRET';

//allow parsing of JSON bodies
app.use(bodyParser.json());

//configure for sandbox environment
paypal.configure({
    'mode': 'sandbox', //sandbox or live
    'client_id': client_id,
    'client_secret': secret
});

Nous avons besoin de quatre conditions pour cette application :

  1. Le package HTTP pour notre serveur.
  2. Le package SDK PayPal Node.
  3. Le package bodyParser pour travailler avec des corps encodés JSON.
  4. Le framework Express pour notre serveur.

Les quelques lignes suivantes configurent des variables pour l’ID client et le secret qui ont été obtenus lors de [la création d’une application][1]. Nous configurons ensuite bodyParser pour autoriser les corps encodés JSON, puis configurons notre application à l’aide des détails de l’application et spécifions l’environnement dans lequel nous travaillons (live pour la production ou sandbox pour les tests).

Créons maintenant la route pour créer une demande de paiement avec PayPal.

app.get('/create', function(req, res){
    //build PayPal payment request
    var payReq = JSON.stringify({
        'intent':'sale',
        'redirect_urls':{
            'return_url':'http://localhost:3000/process',
            'cancel_url':'http://localhost:3000/cancel'
        },
        'payer':{
            'payment_method':'paypal'
        },
        'transactions':[{
            'amount':{
                'total':'7.47',
                'currency':'USD'
            },
            'description':'This is the payment transaction description.'
        }]
    });

    paypal.payment.create(payReq, function(error, payment){
        if(error){
            console.error(error);
        } else {
            //capture HATEOAS links
            var links = {};
            payment.links.forEach(function(linkObj){
                links[linkObj.rel] = {
                    'href': linkObj.href,
                    'method': linkObj.method
                };
            })
        
            //if redirect url present, redirect user
            if (links.hasOwnProperty('approval_url')){
                res.redirect(links['approval_url'].href);
            } else {
                console.error('no redirect URI present');
            }
        }
    });
});

La première chose que nous faisons est de configurer l’objet JSON de demande de paiement, qui contient les informations que nous devons fournir à PayPal pour créer le paiement. Nous définissons l’‘intention’ sur ‘vente’, spécifions les URL de redirection (où PayPal doit rediriger l’utilisateur après avoir confirmé/annulé le paiement), ajoutons un ‘payment_method’ de ‘paypal’ pour signaler que nous ferons un PayPal paiement, puis spécifiez les informations de transaction pour que le payeur confirme.

Nous appelons ensuite payment.create(...), en transmettant notre objet payReq. Cela enverra la demande de création de paiement à PayPal. Une fois que cela revient et réussit, nous pouvons parcourir les liens [HATEOAS][2] fournis dans l’objet de retour pour extraire l’URL vers laquelle nous devons rediriger l’utilisateur, qui est étiquetée sous approval_url.

Le format des liens HATEOAS peut rendre le code de référence fragile s’il est utilisé directement, nous parcourons donc tous les liens fournis et les plaçons dans un meilleur objet de référence pour une protection future contre les modifications. Si approval_url est alors trouvé dans cet objet, nous redirigeons l’utilisateur.

À ce stade, l’utilisateur est redirigé vers PayPal pour confirmer le paiement. Une fois qu’ils le font, ils sont redirigés vers le return_url que nous avons spécifié dans la fonction createPayment(...).

Nous devons maintenant fournir un itinéraire pour gérer ce retour, afin de terminer le paiement.

app.get('/process', function(req, res){
    var paymentId = req.query.paymentId;
    var payerId = { 'payer_id': req.query.PayerID };

    paypal.payment.execute(paymentId, payerId, function(error, payment){
        if(error){
            console.error(error);
        } else {
            if (payment.state == 'approved'){ 
                res.send('payment completed successfully');
            } else {
                res.send('payment not successful');
            }
        }
    });
});

Lorsque l’utilisateur reviendra dans votre application, trois paramètres de chaîne de requête seront également envoyés : paymentId, PayerID et token. Nous n’avons qu’à nous occuper des deux premiers.

Nous extrayons les paramètres, et plaçons le PayerID dans un objet simple pour les besoins de l’étape d’exécution du paiement. Ensuite, un appel est effectué à payment.execute(...), en transmettant ces deux paramètres, afin de terminer le paiement.

Une fois cette demande effectuée, nous vérifions si le paiement a été effectué avec succès en vérifiant si payment.state est défini sur approved. Si tel est le cas, nous pouvons stocker ce dont nous avons besoin à partir de l’objet de paiement renvoyé.

Notre dernière étape consiste à initialiser notre serveur et à écouter le trafic arrivant sur les routes que nous avons spécifiées.

//create server
http.createServer(app).listen(3000, function () {
   console.log('Server started: Listening on port 3000');
});

Une fois le serveur initialisé, aller sur http://localhost:3000/create initialise le processus de paiement.
