Liste liée XOR #

Une liste liée XOR est également appelée liste liée à mémoire efficace. C’est une autre forme de liste doublement liée. Cela dépend fortement de la porte logique XOR et de ses propriétés.

Pourquoi est-ce appelé la liste chaînée à mémoire efficace ?

C’est ce qu’on appelle ainsi car cela utilise moins de mémoire qu’une liste traditionnelle à double lien.

Est-ce différent d’une liste doublement liée ?

**Oui c’est le cas.

Une liste doublement liée stocke deux pointeurs, qui pointent vers le nœud suivant et le nœud précédent. Fondamentalement, si vous voulez revenir en arrière, vous allez à l’adresse pointée par le pointeur “retour”. Si vous voulez avancer, vous allez à l’adresse pointée par le pointeur next. C’est comme:

[![entrez la description de l’image ici][1]][1]

Une liste liée efficace en mémoire, ou à savoir la liste liée XOR n’a qu’un seul pointeur au lieu de deux. Ceci stocke l’adresse précédente (addr (prev)) XOR (^) l’adresse suivante (addr (next)). Lorsque vous souhaitez passer au nœud suivant, vous effectuez certains calculs et trouvez l’adresse du nœud suivant. C’est la même chose pour aller au nœud précédent. C’est comme:

[![entrez la description de l’image ici][2]][2]

Comment ça marche ?

Pour comprendre le fonctionnement de la liste chaînée, vous devez connaître les propriétés de XOR (^) :

|-------------|------------|------------|
|    Name     |   Formula  |    Result  |
|-------------|------------|------------|
| Commutative |    A ^ B   |    B ^ A   |
|-------------|------------|------------|
| Associative | A ^ (B ^ C)| (A ^ B) ^ C|
|-------------|------------|------------|
| None (1)    |    A ^ 0   |     A      |
|-------------|------------|------------|
| None (2)    |    A ^ A   |     0      |
|-------------|------------|------------|
| None (3)    | (A ^ B) ^ A|     B      |
|-------------|------------|------------|

Maintenant, laissons cela de côté et voyons ce que chaque nœud stocke.

Le premier nœud, ou le head, stocke 0 ^ addr (next) car il n’y a pas de nœud ou d’adresse précédent. Cela ressemble à ceci.

Ensuite, le deuxième nœud stocke addr (prev) ^ addr (next). Cela ressemble à ceci.

La queue de la liste n’a pas de nœud suivant, elle stocke donc addr (prev) ^ 0. Cela ressemble à ceci.

Passage de la tête au nœud suivant

Disons que vous êtes maintenant au premier nœud, ou au nœud A. Vous voulez maintenant vous déplacer au nœud B. Voici la formule pour le faire :

Address of Next Node = Address of Previous Node ^ pointer in the current Node

Ce serait donc :

addr (next) = addr (prev) ^ (0 ^ addr (next))

Comme il s’agit de la tête, l’adresse précédente serait simplement 0, donc :

addr (next) = 0 ^ (0 ^ addr (next))

Nous pouvons supprimer les parenthèses :

addr (next) = 0 ^ 0 addr (next)

En utilisant la propriété none (2), nous pouvons dire que 0 ^ 0 sera toujours 0 :

addr (next) = 0 ^ addr (next)

En utilisant la propriété none (1), nous pouvons le simplifier en :

addr (next) = addr (next)

Vous avez l’adresse du nœud suivant !

Passer d’un nœud au nœud suivant

Disons maintenant que nous sommes dans un nœud intermédiaire, qui a un nœud précédent et suivant.

Appliquons la formule :

Address of Next Node = Address of Previous Node ^ pointer in the current Node

Remplacez maintenant les valeurs :

addr (next) = addr (prev) ^ (addr (prev) ^ addr (next))

Supprimer les parenthèses :

addr (next) = addr (prev) ^ addr (prev) ^ addr (next)

En utilisant la propriété none (2), nous pouvons simplifier :

addr (next) = 0 ^ addr (next)

En utilisant la propriété none (1), nous pouvons simplifier :

addr (next) = addr (next)

Et vous l’obtenez!

Passer d’un nœud au nœud dans lequel vous vous trouviez précédemment

Si vous ne comprenez pas le titre, cela signifie essentiellement que si vous étiez au nœud X et que vous êtes maintenant passé au nœud Y, vous souhaitez revenir au nœud visité précédemment, ou essentiellement au nœud X.

Ce n’est pas une tâche fastidieuse. Rappelez-vous que j’avais mentionné ci-dessus, que vous stockez l’adresse à laquelle vous étiez dans une variable temporaire. Ainsi, l’adresse du nœud que vous souhaitez visiter se trouve dans une variable :

addr (prev) = temp_addr

Passer d’un nœud au nœud précédent

Ce n’est pas la même chose que mentionné ci-dessus. Je veux dire que vous étiez au nœud Z, maintenant vous êtes au nœud Y et que vous voulez aller au nœud X.

C’est presque la même chose que de passer d’un nœud au nœud suivant. C’est juste que c’est l’inverse. Lorsque vous écrivez un programme, vous utiliserez les mêmes étapes que celles que j’avais mentionnées dans le passage d’un nœud au nœud suivant, juste que vous trouvez l’élément précédent dans la liste plutôt que de trouver l’élément suivant.

Exemple de code en C

/* C/C++ Implementation of Memory efficient Doubly Linked List */
#include <stdio.h>
#include <stdlib.h>
 
// Node structure of a memory efficient doubly linked list
struct node
{
    int data;
    struct node* npx;  /* XOR of next and previous node */
};
 
/* returns XORed value of the node addresses */
struct node* XOR (struct node *a, struct node *b)
{
    return (struct node*) ((unsigned int) (a) ^ (unsigned int) (b));
}
 
/* Insert a node at the begining of the XORed linked list and makes the
   newly inserted node as head */
void insert(struct node **head_ref, int data)
{
    // Allocate memory for new node
    struct node *new_node  = (struct node *) malloc (sizeof (struct node) );
    new_node->data = data;
 
    /* Since new node is being inserted at the begining, npx of new node
       will always be XOR of current head and NULL */
    new_node->npx = XOR(*head_ref, NULL);
 
    /* If linked list is not empty, then npx of current head node will be XOR 
       of new node and node next to current head */
    if (*head_ref != NULL)
    {
        // *(head_ref)->npx is XOR of NULL and next. So if we do XOR of 
        // it with NULL, we get next
        struct node* next = XOR((*head_ref)->npx,  NULL);
        (*head_ref)->npx = XOR(new_node, next);
    }
 
    // Change head
    *head_ref = new_node;
}
 
// prints contents of doubly linked list in forward direction
void printList (struct node *head)
{
    struct node *curr = head;
    struct node *prev = NULL;
    struct node *next;
 
    printf ("Following are the nodes of Linked List: \n");
 
    while (curr != NULL)
    {
        // print current node
        printf ("%d ", curr->data);
 
        // get address of next node: curr->npx is next^prev, so curr->npx^prev
        // will be next^prev^prev which is next
        next = XOR (prev, curr->npx);
 
        // update prev and curr for next iteration
        prev = curr;
        curr = next;
    }
}
 
// Driver program to test above functions
int main ()
{
    /* Create following Doubly Linked List
       head-->40<-->30<-->20<-->10   */
    struct node *head = NULL;
    insert(&head, 10);
    insert(&head, 20);
    insert(&head, 30);
    insert(&head, 40);
 
    // print the created list
    printList (head);
 
    return (0);
}

Le code ci-dessus remplit deux fonctions de base : insertion et transversal.

  • Insertion : Ceci est effectué par la fonction insert. Cela insère un nouveau nœud au début. Lorsque cette fonction est appelée, elle place le nouveau nœud comme nœud principal et le nœud principal précédent comme deuxième nœud.

  • Parcours : Ceci est effectué par la fonction printList. Il passe simplement par chaque nœud et imprime la valeur.

Notez que XOR des pointeurs n’est pas défini par la norme C/C++. Ainsi, l’implémentation ci-dessus peut ne pas fonctionner sur toutes les plates-formes.

Introduction aux listes liées #

Une liste liée est une collection linéaire d’éléments de données, appelés nœuds, qui sont liés à d’autres nœuds au moyen d’un « pointeur ». Vous trouverez ci-dessous une liste à liens simples avec une référence principale.

         ┌─────────┬─────────┐   ┌─────────┬─────────┐         
 HEAD ──▶│  data   │"pointer"│──▶│  data   │"pointer"│──▶ null 
         └─────────┴─────────┘   └─────────┴─────────┘

Il existe de nombreux types de listes liées, y compris [singly](https://www.wikiod.com/fr/data-structures/liste-liee#Liste liée individuellement) et [doublement](http:// stackoverflow.com/documentation/data-structures/2967/linked-list/23473/doubly-linked-list) listes liées et listes liées circulaires.

Avantages

  • Les listes chaînées sont une structure de données dynamique, qui peut s’agrandir et se rétrécir, allouant et désallouant de la mémoire pendant l’exécution du programme.

  • Les opérations d’insertion et de suppression de nœuds sont facilement mises en œuvre dans une liste chaînée.

  • Les structures de données linéaires telles que les piles et les files d’attente sont facilement implémentées avec une liste chaînée.

  • Les listes chaînées peuvent réduire le temps d’accès et peuvent s’étendre en temps réel sans surcharge de mémoire.

## Liste chaînée simple Les listes liées individuellement sont un type de [liste liée] (https://www.wikiod.com/fr/data-structures/liste-liee#Introduction aux listes liées). Les nœuds d’une liste liée individuellement n’ont qu’un seul “pointeur” vers un autre nœud, généralement “suivant”. C’est ce qu’on appelle une liste chaînée simple car chaque nœud n’a qu’un seul “pointeur” vers un autre nœud. Une liste à liens simples peut avoir une référence de début et/ou de fin. L’avantage d’avoir une référence de queue sont les cas getFromBack, addToBack et removeFromBack, qui deviennent O(1).

 HEAD ──▶│  data   │"pointer"│──▶│  data   │"pointer"│──▶ null 
         └─────────┴────△────┘   └─────────┴─────────┘

## Liste de sauts Les listes de sauts sont des listes liées qui vous permettent de passer au nœud approprié. Il s’agit d’une méthode beaucoup plus rapide qu’une liste simple liée normale. Il s’agit essentiellement d’une [liste liée singulièrement] (https://www.wikiod.com/fr/data-structures/liste-liee#Liste liée individuellement) mais les pointeurs ne vont pas d’un nœud au nœud suivant , mais en sautant quelques nœuds. D’où le nom “Skip List”.

Est-ce différent d’une liste à lien unique ?

**Oui c’est le cas.

Une liste chaînée simple est une liste dont chaque nœud pointe vers le nœud suivant. Une représentation graphique d’une liste simplement chaînée ressemble à :

[![entrez la description de l’image ici][1]][1]

Une liste de sauts est une liste avec chaque nœud pointant vers un nœud qui pourrait ou non être après lui. Une représentation graphique d’une liste de sauts est :

[![entrez la description de l’image ici][2]][2]

Comment ça marche ?

La liste de sauts est simple. Disons que nous voulons accéder au nœud 3 dans l’image ci-dessus. Nous ne pouvons pas prendre la route allant de la tête au quatrième nœud, car c’est après le troisième nœud. On passe donc de la tête au deuxième nœud, puis au troisième.

Liste doublement liée #

Les listes doublement liées sont un type de [liste liée] (https://www.wikiod.com/fr/data-structures/liste-liee#Introduction aux listes liées). Les nœuds d’une liste doublement liée ont deux “pointeurs” vers d’autres nœuds, “suivant” et “précédent”. C’est ce qu’on appelle une liste chaînée double car chaque nœud n’a que deux “pointeurs” vers d’autres nœuds. Une liste doublement liée peut avoir un pointeur de début et/ou de fin.

         ┌─────────┬─────────┬─────────┐   ┌─────────┬─────────┬─────────┐         
 null ◀──┤previous │  data   │  next   │◀──┤previous │  data   │  next   │         
         │"pointer"│         │"pointer"│──▶│"pointer"│         │"pointer"│──▶ null 
         └─────────┴─────────┴─────────┘   └─────────┴─────────┴─────────┘         
                          ▲                     △                   △              
                     HEAD │                     │     DOUBLE        │              
                                                └──── REFERENCE ────┘

Les listes à double liaison sont moins efficaces en termes d’espace que les listes à liaison simple ; cependant, pour certaines opérations, ils offrent des gains de temps significatifs. Un exemple simple est la fonction get, qui, pour une liste doublement liée avec une référence d’en-tête et de fin, commencera à partir de l’en-tête ou de la fin en fonction de l’index de l’élément à obtenir. Pour une liste à n éléments, pour obtenir l’élément indexé n/2 + i, une liste chaînée simple avec des références de tête/queue doit traverser n/2 + i nœuds, car elle ne peut pas “revenir en arrière” à partir de queue. Une liste doublement chaînée avec des références de tête/queue n’a qu’à traverser n/2 - i nœuds, car elle peut “revenir en arrière” à partir de la queue, parcourant la liste dans l’ordre inverse.

Une implémentation de base pour une liste à liens simples en Java - qui peut ajouter des valeurs entières à la fin de la liste, supprimer la première valeur rencontrée de la liste, renvoyer un tableau de valeurs à un instant donné et déterminer si une valeur donnée est présente dans la liste.

    private Node next;
    private int value;
    
    public Node(int value) {
        this.value=value;
    }

    public Node getNext() {
        return next;
    }

    public void setNext(Node next) {
        this.next = next;
    }

    public int getValue() {
        return value;
    }
    
    
}

SinglyLinkedList.java

package com.example.so.ds;


/**
 * <p> Basic single-linked-list implementation </p>
 * @since 20161008
 * @author Ravindra HV
 * @version 0.2
 */
public class SinglyLinkedList {
    
    private Node head;
    private volatile int size;
    
    public int getSize() {
        return size;
    }
    
    public synchronized void append(int value) {
        
        Node entry = new Node(value);
        if(head == null) {
            head=entry;
        }
        else {
            Node temp=head;
            while( temp.getNext() != null) {
                temp=temp.getNext();
            }
            temp.setNext(entry);
        }
        
        size++;
    }
    
    
    public synchronized boolean removeFirst(int value) {
        boolean result = false;
        
        if( head == null ) { // or size is zero..
            // no action
        }
        else if( head.getValue() == value ) {
            head = head.getNext();
            result = true;
        }
        else {

            Node previous = head;
            Node temp = head.getNext();
            while( (temp != null) && (temp.getValue() != value) ) {
                previous = temp;
                temp = temp.getNext();
            }
            
            if((temp != null) && (temp.getValue() == value)) { // if temp is null then not found..
                previous.setNext( temp.getNext() );
                result = true;
            }

        }
        
        if(result) {
            size--;
        }
        
        return result;
        
    }
    
    public synchronized int[] snapshot() {
        Node temp=head;
        int[] result = new int[size];
        for(int i=0;i<size;i++) {
            result[i]=temp.getValue();
            temp = temp.getNext();
        }
        return result;
    }
    
    public synchronized boolean contains(int value) {
        boolean result = false;
        Node temp = head;
        
        while(temp!=null) {
            if(temp.getValue() == value) {
                result=true;
                break;
            }
            temp=temp.getNext();
        }
        return result;
    }
    
}

package com.example.so.ds;

import java.util.Arrays;

/**
 * 
 * <p> Test-case for single-linked list</p>

public class TestSinglyLinkedList {

    /**
     * @param args
     */
    public static void main(String[] args) {
        
        SinglyLinkedList singleLinkedList = new SinglyLinkedList();
        
        int loop = 11;
        Random random = new Random();
        
        for(int i=0;i<loop;i++) {
            
            int value = random.nextInt(loop);
            singleLinkedList.append(value);

            System.out.println();
            System.out.println("Append :"+value);
            System.out.println(Arrays.toString(singleLinkedList.snapshot()));
            System.out.println(singleLinkedList.getSize());
            System.out.println();
        }
        
        for(int i=0;i<loop;i++) {
            int value = random.nextInt(loop);
            boolean contains = singleLinkedList.contains(value);
            singleLinkedList.removeFirst(value);
            
            System.out.println();
            System.out.println("Contains :"+contains);
            System.out.println("RemoveFirst :"+value);
            System.out.println(Arrays.toString(singleLinkedList.snapshot()));
            System.out.println(singleLinkedList.getSize());
            System.out.println();
        }
        

    }

}
