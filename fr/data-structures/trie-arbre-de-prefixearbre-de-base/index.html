Introduction à l’essai #

Vous êtes-vous déjà demandé comment fonctionnent les moteurs de recherche ? Comment Google affiche-t-il des millions de résultats devant vous en quelques millisecondes ? Comment une énorme base de données située à des milliers de kilomètres de vous trouve-t-elle les informations que vous recherchez et vous les renvoie-t-elle ? La raison derrière cela n’est pas possible uniquement en utilisant un Internet plus rapide et des super-ordinateurs. Certains algorithmes de recherche et structures de données fascinants fonctionnent derrière. L’un d’eux est Trie.

Trie, également appelé arbre numérique et parfois arbre de base ou arbre de préfixes (car ils peuvent être recherchés par préfixes), est une sorte d’arbre de recherche, une structure de données arborescente ordonnée qui est utilisée pour stocker un ensemble dynamique ou un tableau associatif où les clés sont généralement des chaînes. C’est l’une de ces structures de données qui peuvent être facilement mises en œuvre. Disons que vous avez une énorme base de données de millions de mots. Vous pouvez utiliser trie pour stocker ces informations et la complexité de la recherche de ces mots dépend uniquement de la longueur du mot que nous recherchons. Cela signifie que cela ne dépend pas de la taille de notre base de données. N’est-ce pas incroyable?

Supposons que nous ayons un dictionnaire avec ces mots :

algo
algea
also
tom
to

Nous voulons stocker ce dictionnaire en mémoire de manière à pouvoir trouver facilement le mot que nous recherchons. L’une des méthodes consisterait à trier les mots de manière lexicographique - comment les dictionnaires réels stockent les mots. Ensuite, nous pouvons trouver le mot en faisant une recherche binaire. Une autre façon consiste à utiliser Prefix Tree ou Trie, en bref. Le mot ‘Trie’ vient du mot Retrieval. Ici, préfixe désigne le préfixe de chaîne qui peut être défini comme ceci : Toutes les sous-chaînes qui peuvent être créées à partir du début d’une chaîne sont appelées préfixe. Par exemple : ‘c’, ‘ca’, ‘cat’ sont tous le préfixe de la chaîne ‘cat’.

Revenons maintenant à Trie. Comme son nom l’indique, nous allons créer un arbre. Au début, nous avons un arbre vide avec juste la racine :

[![Racine][1]][1]

Nous allons insérer le mot ‘algo’. Nous allons créer un nouveau nœud et nommer le bord entre ces deux nœuds ‘a’. À partir du nouveau nœud, nous allons créer une autre arête nommée ’l’ et la connecter à un autre nœud. De cette façon, nous allons créer deux nouvelles arêtes pour ‘g’ et ‘o’. Notez que nous ne stockons aucune information dans les nœuds. Pour l’instant, nous n’envisagerons de créer de nouvelles arêtes qu’à partir de ces nœuds. A partir de maintenant, appelons une arête nommée ‘x’ - edge-x

[![insérer le mot algo][2]][2]

Maintenant, nous voulons ajouter le mot ‘algea’. Nous avons besoin d’un edge-a de la racine, que nous avons déjà. Nous n’avons donc pas besoin d’ajouter de nouveaux bords. De même, nous avons un bord de ‘a’ à ’l’ et de ’l’ à ‘g’. Cela signifie que ‘alg’ est déjà dans le trie. Nous n’ajouterons que edge-e et edge-a.

[![Insérer le mot algue][3]][3]

Nous ajouterons le mot ‘aussi’. Nous avons le préfixe ‘al’ de la racine. Nous n’ajouterons que ‘so’ avec.

[![insérer le mot aussi][4]][4]

Ajoutons ’tom’. Cette fois, nous créons un nouveau bord à partir de la racine car nous n’avons aucun préfixe de tom créé auparavant.

[![Insérer le mot tom][5]][5]

Maintenant, comment devrions-nous ajouter ‘à’ ? ’to’ est complètement un préfixe de ’tom’, nous n’avons donc pas besoin d’ajouter d’arête. Ce que nous pouvons faire, c’est mettre des marques de fin dans certains nœuds. Nous mettrons des marques de fin dans les nœuds où au moins un mot est complété. Le cercle vert indique la marque de fin. L’essai ressemblera à :

[![Insérer le mot à][6]][6]

Vous pouvez facilement comprendre pourquoi nous avons ajouté les marques de fin. Nous pouvons déterminer les mots stockés dans trie. Les caractères seront sur les bords et les nœuds contiendront les marques de fin.

Maintenant, vous pourriez vous demander, quel est le but de stocker des mots comme celui-ci ? Disons qu’on vous demande de trouver le mot ‘alice’ dans le dictionnaire. Vous traverserez le trie. Vous vérifierez s’il existe un edge-a à partir de la racine. Vérifiez ensuite à partir de ‘a’, s’il y a un edge-l. Après cela, vous ne trouverez aucun edge-i, vous pouvez donc conclure que le mot alice n’existe pas dans le dictionnaire.

Si on vous demande de trouver le mot ‘alg’ dans le dictionnaire, vous traverserez root->a, a->l, l->g , mais vous ne trouverez pas de nœud vert à la fin. Donc le mot n’existe pas dans le dictionnaire. Si vous recherchez ’tom’, vous vous retrouverez dans un nœud vert, ce qui signifie que le mot existe dans le dictionnaire.

Complexité:

Le nombre maximum d’étapes nécessaires pour rechercher un mot dans un trie est la longueur du mot que nous recherchons. La complexité est O(longueur). La complexité de l’insertion est la même. La quantité de mémoire nécessaire pour implémenter trie dépend de l’implémentation. Nous verrons une implémentation dans un autre exemple où nous pouvons stocker 106 caractères (pas des mots, des lettres) dans un trie.

Utilisation de Trie :

  • Pour insérer, supprimer et rechercher un mot dans le dictionnaire.
  • Pour savoir si une chaîne est un préfixe d’une autre chaîne.
  • Pour savoir combien de chaînes ont un préfixe commun.
  • Suggestion de noms de contact dans nos téléphones en fonction du préfixe que nous entrons.
  • Découverte de la « sous-chaîne commune la plus longue » de deux chaînes.
  • Connaître la longueur du ‘préfixe commun’ pour deux mots en utilisant ’le plus long ancêtre commun’

Implémentation de Trie #

Avant de lire cet exemple, il est fortement recommandé de lire [Introduction to Trie](https://www.wikiod.com/fr/data-structures/trie-arbre-de-prefixearbre-de-base#Introduction à essayer -trie) d’abord.

L’un des moyens les plus simples d’implémenter Trie consiste à utiliser une liste chaînée.

Nœud:

Les nœuds seront constitués de :

  1. Variable pour la marque finale.
  2. Tableau de pointeurs vers le nœud suivant.

La variable End-Mark indiquera simplement s’il s’agit d’un point final ou non. Le tableau de pointeurs indiquera toutes les arêtes possibles pouvant être créées. Pour les alphabets anglais, la taille du tableau sera de 26, car un maximum de 26 arêtes peuvent être créées à partir d’un seul nœud. Au tout début, chaque valeur du tableau de pointeurs sera NULL et la variable de marque de fin sera fausse.

Node:
Boolean Endmark
Node *next[26]
Node()
    endmark = false
    for i from 0 to 25
        next[i] := NULL
    endfor
endNode

Chaque élément du tableau next[] pointe vers un autre nœud. next[0] pointe vers le nœud partageant edge-a, next[1] pointe vers le nœud partageant edge-b et ainsi de suite. Nous avons défini le constructeur de Node pour initialiser Endmark comme faux et mettre NULL dans toutes les valeurs du tableau de pointeurs next[].

Pour créer Trie, nous devons d’abord instancier root. Ensuite, à partir de la racine, nous créerons d’autres nœuds pour stocker les informations.

Insertion:

Procedure Insert(S, root):    // s is the string that needs to be inserted in our dictionary
curr := root
for i from 1 to S.length
    id := S[i] - 'a'
    if curr -> next[id] == NULL
        curr -> next[id] = Node()
    curr := curr -> next[id]
endfor
curr -> endmark = true

Ici, nous travaillons avec a-z. Donc, pour convertir leurs valeurs ASCII en 0-25, nous leur soustrayons la valeur ASCII de ‘a’. Nous allons vérifier si le nœud actuel (curr) a un avantage pour le caractère en question. Si c’est le cas, nous passons au nœud suivant en utilisant ce bord, si ce n’est pas le cas, nous créons un nouveau nœud. À la fin, nous changeons la marque de fin en true.

Recherche:

Procedure Search(S, root)      // S is the string we are searching
curr := root
for i from 1 to S.length
    id := S[i] - 'a'
    if curr -> next[id] == NULL
        Return false
    curr := curr -> id
Return curr -> endmark

Le processus est similaire à l’insertion. À la fin, nous renvoyons le endmark. Donc, si la marque de fin est vraie, cela signifie que le mot existe dans le dictionnaire. Si c’est faux, alors le mot n’existe pas dans le dictionnaire.

C’était notre implémentation principale. Nous pouvons maintenant insérer n’importe quel mot dans trie et le rechercher.

Effacement:

Parfois, nous aurons besoin d’effacer les mots qui ne seront plus utilisés pour économiser de la mémoire. Pour cela, nous devons supprimer les mots inutilisés :

Procedure Delete(curr):       //curr represents the current node to be deleted
for i from 0 to 25
    if curr -> next[i] is not equal to NULL
        delete(curr -> next[i])
    delete(curr)               //free the memory the pointer is pointing to

Cette fonction va à tous les nœuds enfants de curr, les supprime, puis curr est supprimé pour libérer de la mémoire. Si nous appelons delete(root), cela supprimera tout Trie.

Trie peut également être implémenté en utilisant Arrays.
