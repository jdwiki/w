## Introduction à la pile La pile est une structure de données LIFO (dernier entré, premier sorti), c’est-à-dire que l’élément le plus récent (ou “dernier entré”) ajouté à la pile sera le premier élément supprimé (“premier sorti”).

Prenons l’exemple des livres dans une boîte. Un seul livre peut être ajouté ou retiré de la boîte à la fois, et il ne peut être ajouté et retiré que par le haut.

Maintenant, la boîte avec les deux premiers livres ressemble à ceci : <pré> |——–| | livre 2 | ◀──── dessus de la boîte |——–| | livre 1 | ◀──── fond de boîte |——–| Si on ajoute le tome 3, ce sera en haut. Le reste des livres, qui ont été ajoutés avant le livre 3, restera en dessous : <pré> |——–| | livre 3 | ◀──── dessus de la boîte |——–| | livre 2 | |——–| | livre 1 | ◀──── fond de boîte |——–| La boîte est comme une pile et les livres sont des éléments de données. Ajouter un livre à la boîte est l’opération push tandis que retirer/obtenir un livre du haut de la boîte est l’opération pop. Si nous effectuons l’opération pop, nous obtiendrons le livre 3 et la boîte reviendra à ce qu’elle était avant que nous ayons poussé le livre 3. Cela signifie que le dernier élément (ou le plus récent) que nous avons mis était le premier élément que nous avons sorti (LIFO). Afin d’obtenir le livre 1, nous devons effectuer deux autres pops : un pour supprimer le livre 2 et le second pour obtenir le livre 1.

Implémentation de la pile à l’aide d’un tableau. Pour cela, nous avons besoin d’un index pointant vers l’emplacement supérieur (tos). Chaque fois qu’un élément est poussé dans la pile, le tos s’incrémente de un et chaque fois qu’une opération pop est effectuée, l’index pointant vers le haut de la pile (tos) est décrémenté de un.

POUSSER:

Avant l’opération PUSH <pré> tos |
i1 i2 i3 i4
—– —– —– —– —–
void push(dataElment item){
stack[top]=item; //item = i4
top++; 
}

Après l’opération PUSH La pile a un pointeur vers le haut de la pile. Chaque fois que l’opération push est appelée, elle place la valeur en haut et la met à jour. <pré> tos – Top of the stack tos |
i1 i2 i3
—– —– —– —–
POPULAIRE :
L’opération pop supprime le contenu du haut de la pile et met à jour la valeur de tos en décrémentant de 1

Avant l’opération pop : <pré> tos |
i1 i2 i3 i4
—– —– —– —– —– 
dataElment pop(){
    dataElment value = stack[tos--];
    return value;
}

Après l’opération pop : <pré> tos |
i1 i2 i3 i4
—– —– —– —–
When a push operation is performed it overwrites i4.

Utiliser des piles pour trouver des palindromes #

Un palindrome est un mot qui peut se lire dans les deux sens, comme « kayak ».

Cet exemple montrera comment trouver si un mot donné est un palindrome en utilisant Python3.

Tout d’abord, nous devons transformer une chaîne en pile (nous utiliserons des tableaux comme piles).

str = "string"
stk = [c for c in str] #this makes an array: ["s", "t", "r", "i", "n", "g"]
stk.append("s") #adds a letter to the array
stk.pop() #pops the last element

Maintenant, nous devons inverser le mot.

def invert(str):
    stk = [c for c in str]
    stk2 = []
    while len(stk) > 0:
        stk2.append(stk.pop())
    #now, let's turn stk2 into a string
    str2 = ""
    for c in stk2:
        str2 += c
    return str2

Nous pouvons maintenant comparer le mot et la forme inversée.

def palindrome(str):
    return str == invert(str)

Implémentation de la pile à l’aide d’un tableau et d’une liste liée #

Implémentation de la baie

#include<stdio.h>

#define MAX 100000                    //Global variables
int top = -1;
int a[MAX];

void push(int x){
    
    if(top == MAX-1){                         // Check whether stack is full
        printf("Stack Overflow\n");
        return;
    }
    
    a[++top] = x;                            // Otherwise increment top and insert x
}

void pop(){
    if(top == -1){                                     // if top = -1, empty stack
        printf("Empty stack\n");
        return;
    }
    top--;        
}

void print(){                         //printing stack values
    
    for(int i=0;i <= top;i++){
        printf("%d ",a[i]);        
    }        
    printf("\n");        
}

void topValue(){                        // Method can print the top value of a stack
    printf("%d",a[top]);
}
int main(){
        
    push(5);
    push(20);
    push(15);
    print();
    pop();
    print();
    push(35);
    print();
    topValue();    
    
}

Mise en œuvre de la liste liée

#include<stdio.h>
#include<malloc.h>

 struct node{
    int data;
    node* link;
};
node* top = NULL;

void push(int data){
    
    node* temp = (struct node*)malloc(sizeof(struct node*));
    temp->data = data;
    temp->link = top;
    top = temp;
}

void pop(){
    
    if(top == NULL) return;
    node* temp = top;
    top = top->link;
    free(temp);
}
void print(){
    
    node* t = top;
    while(t != NULL){
        printf("%d ",t->data);
        t=t->link;
    }
    printf("\n");
}

void topValue(){
    printf("%d ",top->data);
}
int main(){
        
    push(5);
    push(20);
    push(15);
    print();
    pop();
    print();
    push(35);
    print();            
    topValue();    
}

Vérification des parenthèses équilibrées #

Un crochet est considéré comme l’un des caractères suivants : (, ), {, }, [ ou ].

Deux crochets sont considérés comme une paire appariée si un crochet ouvrant (c’est-à-dire (, [ ou {) apparaît à gauche d’un crochet fermant (c’est-à-dire ), ] ou }) du même type. Il existe trois types de paires de crochets correspondants : [], {} et ().

Une paire de parenthèses correspondante n’est pas équilibrée si l’ensemble de parenthèses qu’elle contient ne correspond pas. Par exemple, {[(])} n’est pas équilibré car le contenu entre { et } n’est pas équilibré. La paire de crochets contient un seul crochet ouvrant déséquilibré, (, et la paire de parenthèses contient un seul crochet fermant déséquilibré, ].

Selon cette logique, nous disons qu’une séquence de parenthèses est considérée comme équilibrée si les conditions suivantes sont remplies :

Il ne contient pas de crochets inégalés.

Le sous-ensemble de parenthèses enfermé dans les limites d’une paire appariée de parenthèses est également une paire de parenthèses appariées.

Algorithme:

  1. Déclarez une pile (dites “pile”).
  2. Parcourez maintenant l’entrée de chaîne.
  • a) Si le caractère actuel est une parenthèse de départ (c’est-à-dire ( ou { ou [ ) alors poussez-le pour empiler.
  • b) Si le caractère actuel est une parenthèse fermante (c’est-à-dire ) ou } ou ]) puis pop de la pile. Si le caractère sauté est le correspondant parenthèse de départ puis bien sinon les parenthèses ne sont pas équilibrées.
  • c) Si le caractère actuel est un crochet fermant (c’est-à-dire ) ou } ou ]) et que la pile est vide, alors les parenthèses ne sont pas équilibrées.
  1. Après un parcours complet, s’il reste une parenthèse de départ dans la pile, la chaîne n’est pas “équilibrée”, sinon nous avons une chaîne “équilibrée”.
