Les résultats d’une requête SELECT peuvent être regroupés par une ou plusieurs colonnes à l’aide de l’instruction GROUP BY : tous les résultats ayant la même valeur dans les colonnes regroupées sont agrégés ensemble. Cela génère un tableau de résultats partiels, au lieu d’un seul résultat. GROUP BY peut être utilisé conjointement avec des fonctions d’agrégation à l’aide de l’instruction HAVING pour définir la manière dont les colonnes non groupées sont agrégées.

Syntaxe #

  • PAR GROUPE {       expression_colonne     | ROLLUP ( <group_by_expression> [ ,…n ] )     | CUBE ( <group_by_expression> [ ,…n ] )     | JEUX DE GROUPEMENT ( [ ,…n ] )     | () – calcule le total général } [ ,…n ]

  • <group_by_expression> ::=       expression_colonne     | ( expression_colonne [ ,…n ] )

  • <grouping_set> ::=       () – calcule le total général     | <grouping_set_item>     | ( <grouping_set_item> [ ,…n ] )

  • <grouping_set_item> ::=       <group_by_expression>     | ROLLUP ( <group_by_expression> [ ,…n ] )     | CUBE ( <group_by_expression> [ ,…n ] )

Exemple de base de GROUP BY #

Cela pourrait être plus facile si vous considérez GROUP BY comme “pour chaque” pour des raisons d’explication. La requête ci-dessous :

SELECT EmpID, SUM (MonthlySalary) 
FROM Employee
GROUP BY EmpID

Est en train de dire:

“Donnez-moi la somme de MonthlySalary pour chaque EmpID”

Donc, si votre tableau ressemblait à ceci :

+-----+-------------+
|EmpID|MonthlySalary|
+-----+-------------+
|1    |200          |
+-----+-------------+
|2    |300          |
+-----+-------------+

Résultat:

+-+---+
|1|200|
+-+---+
|2|300|
+-+---+

La somme ne semblerait rien faire car la somme d’un nombre est ce nombre. Par contre si ça ressemblait à ça :

+-----+-------------+
|EmpID|MonthlySalary|
+-----+-------------+
|1    |200          |
+-----+-------------+
|1    |300          |
+-----+-------------+
|2    |300          |
+-----+-------------+

Résultat:

+-+---+
|1|500|
+-+---+
|2|300|
+-+---+

Alors ce serait parce qu’il y a deux EmpID 1 à additionner.

UTILISEZ GROUP BY pour COMPTER le nombre de lignes pour chaque entrée unique dans une colonne donnée #

Supposons que vous souhaitiez générer des décomptes ou des sous-totaux pour une valeur donnée dans une colonne.

Compte tenu de ce tableau, “Westerosians”:

Nom GreatHouseAllegience
Arya dur
Boucles d’oreilles Lannister
Myrcelle Lannister
Yara Greyjoy
Catelyn dur
Sansa dur

Sans GROUP BY, COUNT renverra simplement un nombre total de lignes :

SELECT Count(*) Number_of_Westerosians
FROM Westerosians

Retour…

Nombre_de_Westerosiens
6

Mais en ajoutant GROUP BY, nous pouvons COMPTER les utilisateurs pour chaque valeur dans une colonne donnée, pour renvoyer le nombre de personnes dans une Grande Maison donnée, disons :

SELECT GreatHouseAllegience House, Count(*) Number_of_Westerosians
FROM Westerosians
GROUP BY GreatHouseAllegience

Retour…

Maison Nombre_de_Westerosiens
dur 3
Greyjoy 1
Lannister 2

Il est courant de combiner GROUP BY avec ORDER BY pour trier les résultats par catégorie la plus grande ou la plus petite :

SELECT GreatHouseAllegience House, Count(*) Number_of_Westerosians
FROM Westerosians
GROUP BY GreatHouseAllegience
ORDER BY Number_of_Westerosians Desc

Retour…

Maison Nombre_de_Westerosiens
dur 3
Lannister 2
Greyjoy 1

Filtrer les résultats GROUP BY à l’aide d’une clause HAVING #

Une clause HAVING filtre les résultats d’une expression GROUP BY. Remarque : Les exemples suivants utilisent l’exemple de base de données [Library][1].

Exemples:

Renvoie tous les auteurs qui ont écrit plus d’un livre ([live example][3]).

SELECT
  a.Id,
  a.Name,
  COUNT(*) BooksWritten
FROM BooksAuthors ba
  INNER JOIN Authors a ON a.id = ba.authorid
GROUP BY
  a.Id,
  a.Name
HAVING COUNT(*) > 1    -- equals to HAVING BooksWritten > 1
;

Renvoie tous les livres qui ont plus de trois auteurs ([live example][4]).

SELECT
  b.Id,
  b.Title,
  COUNT(*) NumberOfAuthors
FROM BooksAuthors ba
  INNER JOIN Books b ON b.id = ba.bookid
GROUP BY
  b.Id,
  b.Title
HAVING COUNT(*) > 3    -- equals to HAVING NumberOfAuthors > 3
;

[1] : https://www.wikiod.com/fr/sql/exemples-de-bases-de-donnees-et-de-tables#Base de données de la bibliothèque [3] : http://sqlfiddle.com/#!9/7c06f/7 [4] : http://sqlfiddle.com/#!9/7c06f/9

Agrégation ROLAP (Data Mining) #

La description #

La norme SQL fournit deux opérateurs d’agrégation supplémentaires. Ceux-ci utilisent la valeur polymorphe “ALL” pour désigner l’ensemble de toutes les valeurs qu’un attribut peut prendre. Les deux opérateurs sont :

  • avec cube de données qu’il fournit toutes les combinaisons possibles que les attributs d’argument de la clause.
  • with roll up qu’il fournit les agrégats obtenus en considérant les attributs dans l’ordre de gauche à droite par rapport à la façon dont ils sont répertoriés dans l’argument de la clause.

Versions standard SQL prenant en charge ces fonctionnalités : 1999,2003,2006,2008,2011.

Exemples #

Considérez ce tableau :

Nourriture Marque Montant_total
Coller Marque1 100
Pâtes Marque2 250
Pizzas Marque2 300

Avec cubes #

select Food,Brand,Total_amount
from Table
group by Food,Brand,Total_amount with cube
Nourriture Marque Montant_total
Coller Marque1 100
Pâtes Marque2 250
Pâtes TOUT 350
Pizzas Marque2 300
Pizzas TOUT 300
TOUT Marque1 100
TOUT Marque2 550
TOUT TOUT 650

Avec enroulement #

select Food,Brand,Total_amount
from Table
group by Food,Brand,Total_amount with roll up
Nourriture Marque Montant_total
Coller Marque1 100
Pâtes Marque2 250
Pizzas Marque2 300
Pâtes TOUT 350
Pizzas TOUT 300
TOUT TOUT 650
