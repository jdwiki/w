Utilisez ORDER BY avec TOP pour renvoyer les x premières lignes en fonction de la valeur d’une colonne #

Dans cet exemple, nous pouvons utiliser GROUP BY non seulement pour déterminer le tri des lignes renvoyées, mais également quelles lignes sont renvoyées, puisque nous utilisons TOP pour limiter le jeu de résultats.

Disons que nous voulons renvoyer les 5 utilisateurs les plus réputés d’un site de questions-réponses populaire sans nom.

Sans COMMANDER PAR

Cette requête renvoie les 5 premières lignes classées par défaut, qui dans ce cas est “Id”, la première colonne du tableau (même s’il ne s’agit pas d’une colonne affichée dans les résultats).

SELECT TOP 5 DisplayName, Reputation
FROM Users

Retour…

NomAffichage Réputation
Communauté 1
Geoff Dalgas 12567
Jarrod Dixon 11739
Jeff Atwood 37628
Joël Spolsky 25784

Avec COMMANDER PAR

SELECT TOP 5 DisplayName, Reputation
FROM Users
ORDER BY Reputation desc

Retour…

NomAffichage Réputation
JonSkeet 865023
Darin Dimitrov 661741
BalusC 650237
Hans Passant 625870
Mark Gravel 601636

Remarques

Certaines versions de SQL (telles que MySQL) utilisent une clause LIMIT à la fin d’un SELECT, au lieu de TOP au début, par exemple :

SELECT DisplayName, Reputation
FROM Users
ORDER BY Reputation DESC
LIMIT 5

Tri par numéro de colonne (au lieu du nom) #

Vous pouvez utiliser le numéro d’une colonne (où la colonne la plus à gauche est ‘1’) pour indiquer sur quelle colonne baser le tri, au lieu de décrire la colonne par son nom.

Avantage : Si vous pensez qu’il est probable que vous modifiez les noms de colonne ultérieurement, cela ne cassera pas ce code.

Inconvénients : Cela réduira généralement la lisibilité de la requête (la signification de “ORDER BY Reputation” est immédiatement claire, tandis que “ORDER BY 14” nécessite un certain comptage, probablement avec un doigt sur l’écran.)

Cette requête trie le résultat en fonction des informations dans la position de colonne relative “3” de l’instruction select au lieu du nom de colonne “Réputation”.

SELECT DisplayName, JoinDate, Reputation
FROM Users
ORDER BY 3
NomAffichage JoinDate Réputation
Communauté 2008-09-15 1
Jarrod Dixon 2008-10-03 11739
Geoff Dalgas 2008-10-03 12567
Joël Spolsky 2008-09-16 25784
Jeff Atwood 2008-09-16 37628

Tri sur plusieurs colonnes #

SELECT DisplayName, JoinDate, Reputation
FROM Users
ORDER BY JoinDate, Reputation
NomAffichage JoinDate Réputation
Communauté 2008-09-15 1
Jeff Atwood 2008-09-16 25784
Joël Spolsky 2008-09-16 37628
Jarrod Dixon 2008-10-03 11739
Geoff Dalgas 2008-10-03 12567

Trier par alias #

En raison de l’ordre logique de traitement des requêtes, les alias peuvent être utilisés dans l’ordre par.

SELECT DisplayName, JoinDate as jd, Reputation as rep
FROM Users
ORDER BY jd, rep

Et peut utiliser l’ordre relatif des colonnes dans l’instruction select. Considérez le même exemple que ci-dessus et au lieu d’utiliser un alias, utilisez l’ordre relatif comme pour le nom d’affichage, c’est 1 , pour Jd c’est 2 et ainsi de suite

SELECT DisplayName, JoinDate as jd, Reputation as rep
FROM Users
ORDER BY 2, 3

Ordre de tri personnalisé #

Pour trier ce tableau « Employé » par département, vous devez utiliser « ORDER BY Department ». Cependant, si vous voulez un ordre de tri différent qui n’est pas alphabétique, vous devez mapper les valeurs Department en différentes valeurs triées correctement ; cela peut être fait avec une expression CASE :

Nom Département
Hassan informatique
Yusuf RH
Hilary RH
Jo informatique
Joyeux RH
Ken Comptable 
SELECT *
FROM Employee
ORDER BY CASE Department
         WHEN 'HR'         THEN 1
         WHEN 'Accountant' THEN 2
         ELSE                   3
         END;
