Syntaxe #

  • Wild Card avec % : SELECT * FROM [table] WHERE [column_name] Like ‘%Évaluer%’

Wild Card avec _ : SELECT * FROM [table] WHERE [column_name] J’aime ‘V_n%’

Wild Card avec [charlist] : SELECT * FROM [table] WHERE [column_name] Like ‘V[abc]n%’

La condition LIKE dans la clause WHERE est utilisée pour rechercher les valeurs de colonne qui correspondent au modèle donné. Les motifs sont formés à l’aide des deux caractères génériques suivants

  • % (Symbole de pourcentage) - Utilisé pour représenter zéro ou plusieurs caractères
  • _ (Underscore) - Utilisé pour représenter un seul caractère

Correspond au modèle ouvert #

Le caractère générique % ajouté au début ou à la fin (ou aux deux) d’une chaîne permettra à 0 ou plus de n’importe quel caractère avant le début ou après la fin du motif de correspondre.

L’utilisation de ‘%’ au milieu permettra à 0 ou plusieurs caractères entre les deux parties du modèle de correspondre.

Nous allons utiliser cette table Employés :

Correspondance d’un seul caractère #

Pour élargir les sélections d’une instruction de langage de requête structuré (SQL-SELECT), des caractères génériques, le signe de pourcentage (%) et le trait de soulignement (_), peuvent être utilisés.

Le caractère _ (trait de soulignement) peut être utilisé comme caractère générique pour n’importe quel caractère unique dans une correspondance de modèle.

Trouvez tous les employés dont le Fname commence par ‘j’ et se termine par ’n’ et a exactement 3 caractères dans Fname.

SELECT * FROM Employees WHERE FName LIKE 'j_n'

Le caractère _ (trait de soulignement) peut également être utilisé plus d’une fois comme caractère générique pour correspondre aux modèles.

Par exemple, ce modèle correspondrait à “jon”, “jan”, “jen”, etc.

Ces noms ne seront pas affichés “jn”,“john”,“jordan”, “justin”, “jason”, “julian”, “jillian”, “joann” car dans notre requête un trait de soulignement est utilisé et il peut sauter exactement un caractère, donc le résultat doit être de 3 caractères Fname.

Par exemple, ce modèle correspondrait à “LaSt”, “LoSt”, “HaLt”, etc.

SELECT * FROM Employees WHERE FName LIKE '_A_T'

Instruction ESCAPE dans la requête LIKE #

Si vous implémentez une recherche de texte en tant que requête LIKE, vous le faites généralement comme ceci :

SELECT * 
FROM T_Whatever 
WHERE SomeField LIKE CONCAT('%', @in_SearchText, '%')

Cependant, (mis à part le fait que vous ne devriez pas nécessairement utiliser LIKE lorsque vous pouvez utiliser la recherche en texte intégral), cela crée un problème lorsque quelqu’un saisit du texte comme “50%” ou “a_b”.

Ainsi (au lieu de passer à la recherche en texte intégral), vous pouvez résoudre ce problème en utilisant l’instruction d’échappement LIKE :

SELECT * 
FROM T_Whatever 
WHERE SomeField LIKE CONCAT('%', @in_SearchText, '%') ESCAPE '\'

Cela signifie que \ sera désormais traité comme un caractère ESCAPE. Cela signifie que vous pouvez maintenant simplement ajouter \ à chaque caractère de la chaîne que vous recherchez, et les résultats commenceront à être corrects, même lorsque l’utilisateur saisira un caractère spécial comme % ou _.

par exemple.

string stringToSearch = "abc_def 50%";
string newString = "";
foreach(char c in stringToSearch) 
     newString += @"\" + c;
 
sqlCmd.Parameters.Add("@in_SearchText", newString); 
// instead of sqlCmd.Parameters.Add("@in_SearchText", stringToSearch);

Noter: L’algorithme ci-dessus est uniquement à des fins de démonstration. Cela ne fonctionnera pas dans les cas où 1 graphème se compose de plusieurs caractères (utf-8). par exemple. string stringToSearch = "Les Mise\u0301ables"; Vous devrez le faire pour chaque graphème, pas pour chaque caractère. Vous ne devez pas utiliser l’algorithme ci-dessus si vous avez affaire à des langues asiatiques/est-asiatiques/sud-asiatiques. Ou plutôt, si vous voulez commencer par un code correct, vous devez simplement le faire pour chaque graphemeCluster.

Voir également https://stackoverflow.com/questions/1009689/reversestring-a-c-sharp-interview-question/36312251#36312251

Correspondance par plage ou ensemble #

Correspond à n’importe quel caractère unique dans la plage spécifiée (par exemple : [a-f]) ou ensemble (par exemple : [abcdef]).

Ce modèle de plage correspondrait à “gary” mais pas à “mary”:

SELECT * FROM Employees WHERE FName LIKE '[a-g]ary'

Ce modèle d’ensemble correspondrait à “mary” mais pas à “gary”:

SELECT * FROM Employees WHERE Fname LIKE '[lmnop]ary'

La plage ou l’ensemble peut également être inversé en ajoutant le caret ^ avant la plage ou l’ensemble :

Ce modèle de plage ne correspondrait pas à “gary”, mais correspondra à “mary”:

SELECT * FROM Employees WHERE FName LIKE '[^a-g]ary'

Ce modèle d’ensemble ne correspondrait pas à “marie”, mais correspondra à “gary”:

SELECT * FROM Employees WHERE Fname LIKE '[^lmnop]ary'

Rechercher une plage de caractères #

L’instruction suivante correspond à tous les enregistrements dont le FName commence par une lettre de A à F de la table [Employés][1].

SELECT * FROM Employees WHERE FName LIKE '[A-F]%'

[1] : https://www.wikiod.com/fr/sql/exemples-de-bases-de-donnees-et-de-tables#Base de données des ateliers automobiles

Caractères génériques #

les caractères génériques sont utilisés avec l’opérateur SQL LIKE. Les caractères génériques SQL sont utilisés pour rechercher des données dans une table.

Les caractères génériques en SQL sont : %, _, [charlist], [^charlist]

% - Un substitut pour zéro ou plusieurs caractères

   Eg:  //selects all customers with a City starting with "Lo"
        SELECT * FROM Customers
        WHERE City LIKE 'Lo%';

       //selects all customers with a City containing the pattern "es"
      SELECT * FROM Customers
       WHERE City LIKE '%es%';

***** - Un substitut pour un seul caractère

Eg://selects all customers with a City starting with any character, followed by "erlin"
SELECT * FROM Customers
WHERE City LIKE '_erlin';

[charlist] - Ensembles et plages de caractères à faire correspondre

Eg://selects all customers with a City starting with "a", "d", or "l"
SELECT * FROM Customers
WHERE City LIKE '[adl]%';

//selects all customers with a City starting with "a", "d", or "l"
SELECT * FROM Customers
WHERE City LIKE '[a-c]%';

[^charlist] - Correspond uniquement à un caractère NON spécifié entre parenthèses

Eg://selects all customers with a City starting with a character that is not "a", "p", or "l"
SELECT * FROM Customers
WHERE City LIKE '[^apl]%';

or

SELECT * FROM Customers
WHERE City NOT LIKE '[apl]%' and city like '_%';

Associez N’IMPORTE QUEL à TOUS #

Correspond à n’importe quel :
Doit correspondre à au moins une chaîne. Dans cet exemple, le type de produit doit être “électronique”, “livres” ou “vidéo”.

SELECT *
FROM   purchase_table
WHERE  product_type LIKE ANY ('electronics', 'books', 'video');

Match all (doit répondre à toutes les exigences).
Dans cet exemple, ‘royaume-uni’ et ’londres’ et ‘route de l’est’ (variantes comprises) doivent correspondre.

SELECT *
FROM   customer_table
WHERE  full_address LIKE ALL ('%united kingdom%', '%london%', '%eastern road%');

Sélection négative :
Utilisez ALL pour exclure tous les éléments.
Cet exemple donne tous les résultats où le type de produit n’est ni “électronique”, ni “livres”, ni “vidéo”.

SELECT *
FROM   customer_table
WHERE  product_type NOT LIKE ALL ('electronics', 'books', 'video');
