Syntaxe #

  • SELECT nom_colonne FROM nom_table WHERE nom_colonne valeur de l’opérateur
  • SELECT nom_colonne, fonction_agrégat (nom_colonne) FROM nom_table GROUP BY nom_colonne HAVING valeur de l’opérateur de la fonction_agrégé(nom_colonne)

Utilisez BETWEEN pour filtrer les résultats #

Les exemples suivants utilisent les exemples de bases de données [Item Sales][1] et [Customers][2].

Remarque : L’opérateur BETWEEN est inclus.

Utilisation de l’opérateur BETWEEN avec des nombres :

SELECT * From ItemSales
WHERE Quantity BETWEEN 10 AND 17

Cette requête renverra tous les enregistrements “ItemSales” dont la quantité est supérieure ou égale à 10 et inférieure ou égale à 17. Les résultats ressembleront à :

identifiant Date de vente ID d’article Quantité Prix
1 2013-07-01 100 10 34,5
4 2013-07-23 100 15 34,5
5 2013-07-24 145 10 34,5

Utilisation de l’opérateur BETWEEN avec des valeurs de date :

SELECT * From ItemSales
WHERE SaleDate BETWEEN '2013-07-11' AND '2013-05-24'

Cette requête renverra tous les enregistrements “ItemSales” avec une “SaleDate” supérieure ou égale au 11 juillet 2013 et inférieure ou égale au 24 mai 2013.

identifiant Date de vente ID d’article Quantité Prix
3 2013-07-11 100 20 34,5
4 2013-07-23 100 15 34,5
5 2013-07-24 145 10 34,5

Lorsque vous comparez des valeurs datetime au lieu de dates, vous devrez peut-être convertir les valeurs datetime en valeurs date, ou ajouter ou soustraire 24 heures pour obtenir les résultats corrects.

Utilisation de l’opérateur BETWEEN avec des valeurs textuelles :

SELECT Id, FName, LName FROM Customers
WHERE LName BETWEEN 'D' AND 'L';

Exemple en direct : [violon SQL] [3]

Cette requête renverra tous les clients dont le nom se situe alphabétiquement entre les lettres « D » et « L ». Dans ce cas, les clients #1 et #3 seront retournés. Le client #2, dont le nom commence par un ‘M’ ne sera pas inclus.

identifiant FName LName
1 Guillaume Jones
3 Richard Davis

[1] : https://www.wikiod.com/fr/sql/exemples-de-bases-de-donnees-et-de-tables [2] : https://www.wikiod.com/fr/sql/exemples-de-bases-de-donnees-et-de-tables [3] : http://sqlfiddle.com/#!9/76b9b/2

Utiliser HAVING avec des fonctions d’agrégation #

Contrairement à la clause WHERE, HAVING peut être utilisé avec des fonctions d’agrégation.

Une fonction d’agrégation est une fonction dans laquelle les valeurs de plusieurs lignes sont regroupées en entrée sur certains critères pour former une valeur unique de signification ou de mesure plus significative ([Wikipedia][1]).

Les fonctions d’agrégation courantes incluent COUNT(), SUM(), MIN() et MAX().

Cet exemple utilise la Car Table des exemples de bases de données.

SELECT CustomerId, COUNT(Id) AS [Number of Cars]
FROM Cars
GROUP BY CustomerId
HAVING COUNT(Id) > 1

Cette requête renverra le CustomerId et le Number of Cars count de tout client qui a plus d’une voiture. Dans ce cas, le seul client qui a plus d’une voiture est le client #1.

Les résultats ressembleront à :

ID client Nombre de voitures
1 2

[1] : https://en.wikipedia.org/wiki/Aggregate_function

Clause WHERE avec des valeurs NULL/NOT NULL #

SELECT *
FROM Employees
WHERE ManagerId IS NULL

Cette instruction renverra tous les enregistrements [Employee][1] où la valeur de la colonne ManagerId est NULL.

Le résultat sera :

Id    FName    LName    PhoneNumber    ManagerId    DepartmentId
1     James    Smith    1234567890     NULL         1

SELECT *
FROM Employees
WHERE ManagerId IS NOT NULL

Cette instruction renverra tous les enregistrements [Employee][1] où la valeur de ManagerId n’est pas NULL.

Le résultat sera :

Id    FName       LName     PhoneNumber    ManagerId    DepartmentId
2     John        Johnson   2468101214     1            1
3     Michael     Williams  1357911131     1            2
4     Johnathon   Smith     1212121212     2            1

Remarque : La même requête ne renverra pas de résultats si vous modifiez la clause WHERE en WHERE ManagerId = NULL ou WHERE ManagerId <> NULL.

[1] : https://www.wikiod.com/fr/sql/exemples-de-bases-de-donnees-et-de-tables#Base de données des ateliers automobiles

La clause WHERE ne renvoie que les lignes qui correspondent à ses critères #

Steam a une section de jeux à moins de 10 $ sur sa page de magasin. Quelque part au cœur de leurs systèmes, il y a probablement une requête qui ressemble à :

SELECT * 
FROM Items 
WHERE Price < 10

Égalité #

ET et OU #

Utilisez IN pour renvoyer des lignes avec une valeur contenue dans une liste #

Cet exemple utilise la Car Table des exemples de bases de données.

SELECT *
FROM Cars
WHERE TotalCost IN (100, 200, 300)

Cette requête renverra la voiture n°2 qui coûte 200 et la voiture n°3 qui coûte 100. Notez que cela équivaut à utiliser plusieurs clauses avec OR, par exemple :

SELECT *
FROM Cars
WHERE TotalCost = 100 OR TotalCost = 200 OR TotalCost = 300

Utilisez LIKE pour trouver les chaînes et sous-chaînes correspondantes #

Voir documentation complète sur l’opérateur LIKE.

Cet exemple utilise la [table des employés](https://www.wikiod.com/fr/sql/exemples-de-bases-de-donnees-et-de-tables#Base de données des ateliers automobiles) des exemples de bases de données.

SELECT *
FROM Employees
WHERE FName LIKE 'John'

Cette requête renverra uniquement l’employé n° 1 dont le prénom correspond exactement à ‘John’.

SELECT *
FROM Employees
WHERE FName like 'John%'

L’ajout de % vous permet de rechercher une sous-chaîne :

  • John% - renverra tout employé dont le nom commence par ‘John’, suivi de n’importe quel nombre de caractères
  • % John - renverra tout employé dont le nom se termine par ‘John’, précédé de n’importe quel nombre de caractères
  • % John% - renverra tout employé dont le nom contient ‘John’ n’importe où dans la valeur

Dans ce cas, la requête renverra l’employé #2 dont le nom est ‘John’ ainsi que l’employé #4 dont le nom est ‘Johnathon’.

Utilisez HAVING pour vérifier plusieurs conditions dans un groupe #

Tableau des commandes

ID client ID produit Quantité Prix ​​
1 2 5 100
1 3 2 200
1 4 1 500
2 1 4 50
3 5 6 700

Pour vérifier les clients qui ont commandé à la fois - ProductID 2 et 3, HAVING peut être utilisé

 select customerId
 from orders
 where productID in (2,3)
 group by customerId
 having count(distinct productID) = 2

Valeur de retour :

IDclient
1

La requête sélectionne uniquement les enregistrements avec les productIDs dans les questions et avec la clause HAVING vérifie les groupes ayant 2 productIds et pas un seul.

Une autre possibilité serait

 select customerId
 from orders
 group by customerId
 having sum(case when productID = 2 then 1 else 0 end) > 0
    and sum(case when productID = 3 then 1 else 0 end) > 0

Cette requête sélectionne uniquement les groupes ayant au moins un enregistrement avec productID 2 et au moins un avec productID 3.

Où EXISTE #

Sélectionne les enregistrements dans TableName qui ont des enregistrements correspondants dans TableName1.

SELECT * FROM TableName t WHERE EXISTS (
    SELECT 1 FROM TableName1 t1 where t.Id = t1.Id)
