UNION UNION TOUS

  • Syntaxe
  • Requête de base UNION ALL
  • Explication simple et exemple

Sur cette page

  • Syntaxe
  • Requête de base UNION ALL
  • Explication simple et exemple

Le mot-clé UNION dans SQL est utilisé pour combiner les résultats de l’instruction SELECT sans aucun doublon.

Pour utiliser UNION et combiner les résultats, les deux instructions SELECT doivent avoir le même nombre de colonnes avec le même type de données dans le même ordre, mais la longueur de la colonne peut être différente.

Syntaxe #

  • SELECT colonne_1 [, colonne_2 ] FROM table_1 [, table_2 ] [condition WHERE] UNION | UNION TOUS SELECT colonne_1 [, colonne_2 ] FROM table_1 [, table_2 ] [condition WHERE]

Les clauses UNION et UNION ALL combinent l’ensemble de résultats de deux ou plusieurs instructions SELECT structurées de manière identique en un seul résultat / tableau.

Le nombre de colonnes et les types de colonnes pour chaque requête doivent correspondre pour qu’un UNION / UNION ALL fonctionne.

La différence entre une requête UNION et une requête UNION ALL est que la clause UNION supprimera toutes les lignes en double dans le résultat là où UNION ALL ne le fera pas.

Cette suppression distincte d’enregistrements peut ralentir considérablement les requêtes même s’il n’y a pas de lignes distinctes à supprimer à cause de cela si vous savez qu’il n’y aura pas de doublons (ou ne vous en souciez pas), toujours par défaut “UNION ALL” pour une requête plus optimisée.

Requête de base UNION ALL #

CREATE TABLE HR_EMPLOYEES
(
    PersonID int,
    LastName VARCHAR(30),
    FirstName VARCHAR(30),
    Position VARCHAR(30)
);

CREATE TABLE FINANCE_EMPLOYEES
(
    PersonID INT,
    LastName VARCHAR(30),
    FirstName VARCHAR(30),
    Position VARCHAR(30)
);

Disons que nous voulons extraire les noms de tous les “managers” de nos départements.

En utilisant un “UNION”, nous pouvons obtenir tous les employés des départements des ressources humaines et des finances, qui occupent le “poste” de “manager”.

SELECT 
    FirstName, LastName   
FROM 
    HR_EMPLOYEES  
WHERE 
    Position = 'manager'  
UNION ALL  
SELECT 
    FirstName, LastName  
FROM 
    FINANCE_EMPLOYEES  
WHERE 
    Position = 'manager'

L’instruction UNION supprime les lignes en double des résultats de la requête. Puisqu’il est possible d’avoir des personnes ayant le même nom et la même position dans les deux départements, nous utilisons UNION ALL, afin de ne pas supprimer les doublons.

Si vous souhaitez utiliser un alias pour chaque colonne de sortie, vous pouvez simplement les placer dans la première instruction select, comme suit :

SELECT 
    FirstName as 'First Name', LastName as 'Last Name'
FROM 
    HR_EMPLOYEES  
WHERE 
    Position = 'manager'  
UNION ALL  
SELECT 
    FirstName, LastName  
FROM 
    FINANCE_EMPLOYEES  
WHERE 
    Position = 'manager'

Explication simple et exemple #

En termes simples :

  • UNION rejoint 2 ensembles de résultats tout en supprimant les doublons de l’ensemble de résultats
  • UNION ALL rejoint 2 ensembles de résultats sans tenter de supprimer les doublons

Une erreur que beaucoup de gens commettent est d’utiliser un “UNION” alors qu’ils n’ont pas besoin de supprimer les doublons. Le coût de performance supplémentaire par rapport à de grands ensembles de résultats peut être très important.

Quand vous pourriez avoir besoin de UNION

Supposons que vous deviez filtrer une table par rapport à 2 attributs différents et que vous ayez créé des index non clusterisés distincts pour chaque colonne. Une UNION vous permet de tirer parti des deux index tout en empêchant les doublons.

SELECT C1, C2, C3 FROM Table1 WHERE C1 = @Param1
UNION
SELECT C1, C2, C3 FROM Table1 WHERE C2 = @Param2

Cela simplifie votre réglage des performances puisque seuls des index simples sont nécessaires pour effectuer ces requêtes de manière optimale. Vous pourrez peut-être même vous débrouiller avec un peu moins d’index non clusterisés, ce qui améliore également les performances globales d’écriture sur la table source.

Quand vous pourriez avoir besoin de UNION ALL

Supposons que vous ayez toujours besoin de filtrer une table par rapport à 2 attributs, mais que vous n’ayez pas besoin de filtrer les enregistrements en double (soit parce que cela n’a pas d’importance, soit parce que vos données ne produiraient aucun doublon lors de l’union en raison de la conception de votre modèle de données).

SELECT C1 FROM Table1
UNION ALL
SELECT C1 FROM Table2

Ceci est particulièrement utile lors de la création de vues qui joignent des données conçues pour être partitionnées physiquement sur plusieurs tables (peut-être pour des raisons de performances, mais qui souhaitent toujours cumuler des enregistrements). Étant donné que les données sont déjà fractionnées, le fait que le moteur de base de données supprime les doublons n’ajoute aucune valeur et ajoute simplement du temps de traitement supplémentaire aux requêtes.
