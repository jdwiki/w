L’instruction SELECT est au cœur de la plupart des requêtes SQL. Il définit quel ensemble de résultats doit être renvoyé par la requête et est presque toujours utilisé en conjonction avec la clause FROM, qui définit quelle(s) partie(s) de la base de données doit être interrogée.

Syntaxe #

  • SÉLECTIONNER [DISTINCT] [colonne1] [, [colonne2] … ] DE [tableau] [ Condition OÙ ] [ GROUPER PAR [colonne1] [, [colonne2] … ]

[ AYANT [colonne1] [, [colonne2] … ]

[ ORDRE PAR ASC | DESC ]

SELECT détermine les données des colonnes à renvoyer et dans quel ordre À PARTIR d’une table donnée (étant donné qu’elles correspondent spécifiquement aux autres exigences de votre requête - où et ayant des filtres et des jointures).

SELECT Name, SerialNumber
FROM ArmyInfo

renverra uniquement les résultats des colonnes Name et Serial Number, mais pas de la colonne appelée Rank, par exemple

SELECT *
FROM ArmyInfo

indique que toutes les colonnes seront renvoyées. Cependant, veuillez noter qu’il est déconseillé de sélectionner SELECT * car vous renvoyez littéralement toutes les colonnes d’une table.

Utilisation du caractère générique pour sélectionner toutes les colonnes d’une requête. #

Considérez une base de données avec les deux tables suivantes.

Tableau des employés :

identifiant FName LName ID de service
1 Jacques forgeron 3
2 Jean Johnson 4

Tableau des départements :

identifiant Nom
1 Ventes
2 Commercialisation
3 Finances
4 informatique

Instruction de sélection simple #

* est le caractère générique utilisé pour sélectionner toutes les colonnes disponibles dans une table.

Lorsqu’il est utilisé comme substitut des noms de colonnes explicites, il renvoie toutes les colonnes de toutes les tables qu’une requête sélectionne “FROM”. Cet effet s’applique à toutes les tables auxquelles la requête accède via ses clauses JOIN.

Considérez la requête suivante :

SELECT * FROM Employees

Il renverra tous les champs de toutes les lignes de la table “Employés” :

identifiant FName LName ID de service
1 Jacques forgeron 3
2 Jean Johnson 4

Notation par points #

Pour sélectionner toutes les valeurs d’un tableau spécifique, le caractère générique peut être appliqué au tableau avec la notation par points.

Considérez la requête suivante :

SELECT 
    Employees.*, 
    Departments.Name
FROM 
    Employees
JOIN 
    Departments 
    ON Departments.Id = Employees.DeptId

Cela renverra un ensemble de données avec tous les champs de la table Employee, suivi uniquement du champ Name dans la table Departments :

identifiant FName LName ID de service Nom
1 Jacques forgeron 3 Finances
2 Jean Johnson 4 informatique

Mises en garde contre l’utilisation

Il est généralement conseillé d’éviter d’utiliser * dans le code de production dans la mesure du possible, car cela peut entraîner un certain nombre de problèmes potentiels, notamment :

  1. E/S excessives, charge réseau, utilisation de la mémoire, etc., en raison du fait que le moteur de base de données lit les données qui ne sont pas nécessaires et les transmet au code frontal. Ceci est particulièrement préoccupant lorsqu’il peut y avoir de grands champs tels que ceux utilisés pour stocker de longues notes ou des fichiers joints.
  2. Charge d’E/S excessive supplémentaire si la base de données doit spouler les résultats internes sur le disque dans le cadre du traitement d’une requête plus complexe que SELECT <columns> FROM <table>.
  3. Traitement supplémentaire (et/ou encore plus d’E/S) si certaines des colonnes inutiles sont :
    • computed columns in databases that support them
    • in the case of selecting from a view, columns from a table/view that the query optimiser could otherwise optimise out
  4. Le risque d’erreurs inattendues si des colonnes sont ajoutées ultérieurement aux tables et aux vues, ce qui entraîne des noms de colonne ambigus. Par exemple, SELECT * FROM commandes JOIN personnes ON personnes.id = commandes.personid ORDER BY displayname - si une colonne appelée displayname est ajoutée à la table des commandes pour permettre aux utilisateurs de donner à leurs commandes des noms significatifs pour référence future, alors le le nom de la colonne apparaîtra deux fois dans la sortie, de sorte que la clause ORDER BY sera ambiguë, ce qui peut provoquer des erreurs (“nom de colonne ambigu” dans les versions récentes de MS SQL Server), et si ce n’est pas dans cet exemple, votre code d’application pourrait commencer à afficher l’ordre name où le nom de la personne est destiné car la nouvelle colonne est la première de ce nom renvoyé, et ainsi de suite.

Quand pouvez-vous utiliser *, en gardant à l’esprit l’avertissement ci-dessus ? #

Bien qu’il soit préférable d’éviter dans le code de production, l’utilisation de * convient comme raccourci lors de l’exécution de requêtes manuelles sur la base de données à des fins d’investigation ou de prototype.

Parfois, les décisions de conception de votre application la rendent inévitable (dans de telles circonstances, préférez tablealias.* plutôt que * lorsque cela est possible).

Lorsque vous utilisez EXISTS, comme SELECT A.col1, A.Col2 FROM A WHERE EXISTS (SELECT * FROM B where A.ID = B.A_ID), nous ne renvoyons aucune donnée de B. Ainsi, une jointure est inutile, et le moteur sait qu’aucune valeur de B ne doit être renvoyée, donc aucun impact sur les performances pour l’utilisation de *. De même, COUNT(*) convient car il ne renvoie aucune des colonnes, il n’a donc besoin que de lire et de traiter celles qui sont utilisées à des fins de filtrage.

SELECT Utilisation des alias de colonne #

Les alias de colonne sont principalement utilisés pour raccourcir le code et rendre les noms de colonne plus lisibles.

Le code devient plus court car les noms de table longs et l’identification inutile des colonnes (par exemple, il peut y avoir 2 ID dans la table, mais un seul est utilisé dans l’instruction) peuvent être évités. Avec les [alias de table][1], cela vous permet d’utiliser des noms descriptifs plus longs dans la structure de votre base de données tout en gardant les requêtes sur cette structure concises.

De plus, ils sont parfois nécessaires, par exemple dans les vues, pour nommer les sorties calculées.

Toutes les versions de SQL #

Les alias peuvent être créés dans toutes les versions de SQL en utilisant des guillemets doubles (").

SELECT 
    FName AS "First Name", 
    MName AS "Middle Name",
    LName AS "Last Name"
FROM Employees

Différentes versions de SQL #

Vous pouvez utiliser des guillemets simples ('), des guillemets doubles (") et des crochets ([]) pour créer un alias dans Microsoft SQL Server.

SELECT 
    FName AS "First Name", 
    MName AS 'Middle Name',
    LName AS [Last Name]
FROM Employees

Les deux entraîneront :

Prénom Deuxième prénom Nom de famille
Jacques Jean Forgeron
Jean Jacques Johnson
Michel Marcus Williams

Cette instruction renverra les colonnes FName et LName avec un nom donné (un alias). Ceci est réalisé en utilisant l’opérateur AS suivi de l’alias, ou en écrivant simplement alias directement après le nom de la colonne. Cela signifie que la requête suivante a le même résultat que la précédente.

SELECT 
    FName "First Name", 
    MName "Middle Name",
    LName "Last Name"
FROM Employees
Prénom Deuxième prénom Nom de famille
Jacques Jean Forgeron
Jean Jacques Johnson
Michel Marcus Williams

Cependant, la version explicite (c’est-à-dire utilisant l’opérateur AS) est plus lisible.

Si l’alias a un seul mot qui n’est pas un mot réservé, nous pouvons l’écrire sans guillemets simples, guillemets doubles ou crochets :

SELECT 
    FName AS FirstName, 
    LName AS LastName
FROM Employees
Prénom Nom de famille
Jacques Forgeron
Jean Johnson
Michel Williams

Une autre variante disponible dans MS SQL Server, entre autres, est <alias> = <column-or-calculation>, par exemple :

SELECT FullName = FirstName + ' ' + LastName, 
       Addr1    = FullStreetAddress,
       Addr2    = TownName
FROM CustomerDetails

ce qui équivaut à :

SELECT FirstName + ' ' + LastName As FullName
       FullStreetAddress          As Addr1,
       TownName                   As Addr2
FROM CustomerDetails

Les deux entraîneront :

Nomcomplet Adr1 Adr2
James Smith 123 AnyStreet VilleVille
Jean Johnson 668 MaRoute N’importe quelle ville
Michel Williams 999, promenade haut de gamme Williamsburg

Certains trouvent que l’utilisation de = au lieu de As est plus facile à lire, bien que beaucoup déconseillent ce format, principalement parce qu’il n’est pas standard et n’est donc pas largement pris en charge par toutes les bases de données. Cela peut prêter à confusion avec d’autres utilisations du caractère =.

Toutes les versions de SQL #

De plus, si vous avez besoin d’utiliser des mots réservés, vous pouvez utiliser des crochets ou des guillemets pour vous échapper :

SELECT
    FName as "SELECT",
    MName as "FROM",
    LName as "WHERE"
FROM Employees

Différentes versions de SQL #

De même, vous pouvez échapper des mots-clés dans MSSQL avec toutes les différentes approches :

SELECT 
    FName AS "SELECT", 
    MName AS 'FROM',
    LName AS [WHERE]
FROM Employees
CHOISIR DE OÙ
Jacques Jean Forgeron
Jean Jacques Johnson
Michel Marcus Williams

De plus, un alias de colonne peut être utilisé dans n’importe laquelle des clauses finales de la même requête, comme un ORDER BY :

SELECT 
    FName AS FirstName, 
    LName AS LastName
FROM 
    Employees 
ORDER BY 
    LastName DESC

Cependant, vous ne pouvez pas utiliser

SELECT
    FName AS SELECT,
    LName AS FROM
FROM 
    Employees
ORDER BY 
    LastName DESC

Créer un alias à partir de ces mots réservés (SELECT et FROM).

Cela entraînera de nombreuses erreurs lors de l’exécution.

[1] : https://www.wikiod.com/fr/sql/selectionner#Sélection avec alias de table

## Sélectionner des colonnes individuelles SELECT PhoneNumber, Email, PreferredContact FROM Customers

Cette instruction renverra les colonnes PhoneNumber, Email et PreferredContact de toutes les lignes de la table [Customers][1]. De plus, les colonnes seront renvoyées dans l’ordre dans lequel elles apparaissent dans la clause SELECT.

Le résultat sera :

Numéro de téléphone E-mail Contact préféré
3347927472 william.jones@exemple.com TÉLÉPHONER
2137921892 dmiller@exemple.net E-MAIL
NUL richard0123@exemple.com E-MAIL

Si plusieurs tables sont jointes, vous pouvez sélectionner des colonnes à partir de tables spécifiques en spécifiant le nom de la table avant le nom de la colonne : [table_name].[column_name]

SELECT 
    Customers.PhoneNumber, 
    Customers.Email, 
    Customers.PreferredContact,
    Orders.Id AS OrderId
FROM 
    Customers
LEFT JOIN 
    Orders ON Orders.CustomerId = Customers.Id

*AS OrderId signifie que le champ Id de la table Orders sera renvoyé sous la forme d’une colonne nommée OrderId. Voir [selecting with column alias](https://www.wikiod.com/fr/sql/selectionner#SELECT Utilisation des alias de colonne) pour plus d’informations.

Pour éviter d’utiliser des noms de table longs, vous pouvez utiliser des alias de table. Cela atténue la peine d’écrire des noms de table longs pour chaque champ que vous sélectionnez dans les jointures. Si vous effectuez une auto-jointure (une jointure entre deux instances de la même table), vous devez utiliser des alias de table pour distinguer vos tables. Nous pouvons écrire un alias de table comme Customers c ou Customers AS c. Ici, c fonctionne comme un alias pour Customers et nous pouvons sélectionner disons Email comme ceci : c.Email.

SELECT 
    c.PhoneNumber, 
    c.Email, 
    c.PreferredContact,
    o.Id AS OrderId
FROM 
    Customers c
LEFT JOIN 
    Orders o ON o.CustomerId = c.Id

[1] : https://www.wikiod.com/fr/sql/exemples-de-bases-de-donnees-et-de-tables

Sélection du nombre spécifié d’enregistrements #

La norme SQL 2008 définit la clause “FETCH FIRST” pour limiter le nombre d’enregistrements renvoyés.

SELECT Id, ProductName, UnitPrice, Package 
FROM Product 
ORDER BY UnitPrice DESC
FETCH FIRST 10 ROWS ONLY

Cette norme n’est prise en charge que dans les versions récentes de certains RDMS. Une syntaxe non standard spécifique au fournisseur est fournie dans d’autres systèmes. Progress OpenEdge 11.x prend également en charge la syntaxe FETCH FIRST <n> ROWS ONLY.

De plus, OFFSET <m> ROWS avant FETCH FIRST <n> ROWS ONLY permet de sauter lignes avant de récupérer lignes.

SELECT Id, ProductName, UnitPrice, Package 
FROM Product 
ORDER BY UnitPrice DESC
OFFSET 5 ROWS
FETCH FIRST 10 ROWS ONLY

La requête suivante est prise en charge dans [SQL Server][1] et MS Access :

SELECT TOP 10 Id, ProductName, UnitPrice, Package
FROM Product
ORDER BY UnitPrice DESC

Pour faire la même chose dans [MySQL][2] ou PostgreSQL, le mot-clé LIMIT doit être utilisé :

SELECT Id, ProductName, UnitPrice, Package
FROM Product
ORDER BY UnitPrice DESC
LIMIT 10

Dans Oracle, la même chose peut être faite avec ROWNUM :

SELECT Id, ProductName, UnitPrice, Package
FROM Product
WHERE ROWNUM <= 10
ORDER BY UnitPrice DESC

Résultats : 10 enregistrements.

Id    ProductName               UnitPrice             Package
38    Côte de Blaye             263.50                12 - 75 cl bottles
29    Thüringer Rostbratwurst   123.79                50 bags x 30 sausgs.
9    Mishi Kobe Niku            97.00                 18 - 500 g pkgs.
20    Sir Rodney's Marmalade    81.00                 30 gift boxes
18    Carnarvon Tigers          62.50                 16 kg pkg.
59    Raclette Courdavault      55.00                 5 kg pkg.
51    Manjimup Dried Apples     53.00                 50 - 300 g pkgs.
62    Tarte au sucre            49.30                 48 pies
43    Ipoh Coffee               46.00                 16 - 500 g tins
28    Rössle Sauerkraut         45.60                 25 - 825 g cans

Nuances du vendeur :

Il est important de noter que le TOP dans Microsoft SQL fonctionne après la clause WHERE et renverra le nombre spécifié de résultats s’ils existent n’importe où dans la table, tandis que ROWNUM fonctionne dans le cadre de la clause WHERE donc si d’autres conditions n’existent pas dans le nombre de lignes spécifié au début du tableau, vous n’obtiendrez aucun résultat alors qu’il pourrait y en avoir d’autres à trouver.

[1] : https://www.wikiod.com/fr/sql-server/limiter-lensemble-de-resultats [2] : https://www.wikiod.com/fr/mysql/limite-et-decalage

Sélection avec condition #

La syntaxe de base de SELECT avec la clause WHERE est :

SELECT column1, column2, columnN
FROM table_name
WHERE [condition]

La [condition] peut être n’importe quelle expression SQL, spécifiée à l’aide d’opérateurs de comparaison ou logiques tels que >, <, =, <>, >=, <=, LIKE, NOT, IN, BETWEEN, etc.

L’instruction suivante renvoie toutes les colonnes de la table ‘Cars’ où la colonne d’état est ‘READY’ :

SELECT * FROM Cars WHERE status = 'READY'

Voir WHERE and HAVING pour plus d’exemples.

Sélection avec CASE #

Lorsque les résultats doivent avoir une logique appliquée «à la volée», on peut utiliser l’instruction CASE pour l’implémenter.

SELECT CASE WHEN Col1 < 50 THEN 'under' ELSE 'over' END threshold
FROM TableName

peut également être enchaîné

SELECT 
    CASE WHEN Col1 < 50 THEN 'under' 
         WHEN Col1 > 50 AND Col1 <100 THEN 'between' 
         ELSE 'over' 
    END threshold
FROM TableName

on peut aussi avoir CASE dans une autre instruction CASE

SELECT 
    CASE WHEN Col1 < 50 THEN 'under' 
         ELSE 
            CASE WHEN Col1 > 50 AND Col1 <100 THEN Col1 
            ELSE 'over' END 
    END threshold
FROM TableName

Sélectionnez les colonnes nommées d’après des mots-clés réservés #

Lorsqu’un nom de colonne correspond à un mot-clé réservé, le SQL standard exige que vous le mettiez entre guillemets :

SELECT 
    "ORDER",
    ID 
FROM ORDERS

Notez que cela rend le nom de la colonne sensible à la casse.

Certains SGBD ont des façons propriétaires de citer les noms. Par exemple, SQL Server utilise des crochets à cette fin :

SELECT 
    [Order],
    ID 
FROM ORDERS

tandis que MySQL (et MariaDB) utilisent par défaut des backticks :

SELECT 
    `Order`,
    id 
FROM orders

Sélection avec alias de table #

SELECT e.Fname, e.LName 
FROM Employees e

La table Employés reçoit l’alias ’e’ directement après le nom de la table. Cela permet de supprimer l’ambiguïté dans les scénarios où plusieurs tables ont le même nom de champ et vous devez être précis quant à la table à partir de laquelle vous souhaitez renvoyer des données.

SELECT e.Fname, e.LName, m.Fname AS ManagerFirstName 
FROM Employees e 
    JOIN Managers m ON e.ManagerId = m.Id

Notez qu’une fois que vous avez défini un alias, vous ne pouvez plus utiliser le nom de table canonique. c’est à dire.,

SELECT e.Fname, Employees.LName, m.Fname AS ManagerFirstName 
FROM Employees e 
JOIN Managers m ON e.ManagerId = m.Id

renverrait une erreur.

Il convient de noter que les alias de table - plus formellement des “variables de plage” - ont été introduits dans le langage SQL pour résoudre le problème des colonnes en double causées par INNER JOIN. La norme SQL de 1992 a corrigé ce défaut de conception antérieur en introduisant NATURAL JOIN (implémenté dans mySQL, PostgreSQL et Oracle mais pas encore dans SQL Server), dont le résultat n’a jamais de noms de colonne en double. L’exemple ci-dessus est intéressant en ce que les tables sont jointes sur des colonnes de noms différents (Id et ManagerId) mais ne sont pas censées être jointes sur les colonnes de même nom (LName, FName), nécessitant le renommage des colonnes à effectuer avant la jointure :

SELECT Fname, LName, ManagerFirstName 
FROM Employees
     NATURAL JOIN
     ( SELECT Id AS ManagerId, Fname AS ManagerFirstName
       FROM Managers ) m;

Notez que bien qu’une variable alias/plage doive être déclarée pour la table dérivée (sinon SQL génère une erreur), il n’est jamais logique de l’utiliser réellement dans la requête.

SELECT * FROM Employees ORDER BY LName DESC

Ou

 SELECT * FROM Employees ORDER BY LName ASC

Cette instruction modifie le sens du tri.

On peut également spécifier plusieurs colonnes de tri. Par exemple:

SELECT * FROM Employees ORDER BY LName ASC, FName ASC

Cet exemple triera d’abord les résultats par LName puis, pour les enregistrements qui ont le même LName, triera par FName. Cela vous donnera un résultat similaire à ce que vous trouveriez dans un annuaire téléphonique.

Afin d’éviter de retaper le nom de la colonne dans la clause ORDER BY, il est possible d’utiliser à la place le numéro de la colonne. Notez que les numéros de colonne commencent à partir de 1.

SELECT Id, FName, LName, PhoneNumber FROM Employees ORDER BY 3

Vous pouvez également intégrer une instruction CASE dans la clause ORDER BY.

SELECT Id, FName, LName, PhoneNumber FROM Employees ORDER BY CASE WHEN LName='Jones` THEN 0 ELSE 1 END ASC

Cela triera vos résultats pour avoir tous les enregistrements avec le LName de “Jones” en haut.

Sélection avec les fonctions d’agrégation #

Moyen #

La fonction d’agrégation AVG() renverra la moyenne des valeurs sélectionnées. #

SELECT AVG(Salary) FROM Employees

Les fonctions d’agrégation peuvent également être combinées avec la clause where. #

SELECT AVG(Salary) FROM Employees where DepartmentId = 1

Les fonctions d’agrégation peuvent également être combinées avec la clause group by. #

Si l’employé est classé dans plusieurs départements et que nous voulons trouver le salaire moyen pour chaque département, nous pouvons utiliser la requête suivante.

SELECT AVG(Salary) FROM Employees GROUP BY DepartmentId

Le minimum #

La fonction d’agrégation MIN() renverra le minimum de valeurs sélectionnées. #

SELECT MIN(Salary) FROM Employees

#Maximum

La fonction d’agrégation MAX() renverra le maximum de valeurs sélectionnées. #

SELECT MAX(Salary) FROM Employees

Compter #

La fonction d’agrégation COUNT() renverra le nombre de valeurs sélectionnées. #

SELECT Count(*) FROM Employees

Il peut également être combiné avec des conditions where pour obtenir le nombre de lignes qui satisfont à des conditions spécifiques. #

SELECT Count(*) FROM Employees where ManagerId IS NOT NULL

Des colonnes spécifiques peuvent également être spécifiées pour obtenir le nombre de valeurs dans la colonne. Notez que les valeurs NULL ne sont pas comptées. #

Select Count(ManagerId) from Employees

Count peut également être combiné avec le mot-clé distinct pour un nombre distinct. #

Select Count(DISTINCT DepartmentId) from Employees

Somme #

La fonction d’agrégation SUM() renvoie la somme des valeurs sélectionnées pour toutes les lignes. #

SELECT SUM(Salary) FROM Employees

Sélectionner sans verrouiller la table #

Parfois, lorsque les tables sont utilisées principalement (ou uniquement) pour les lectures, l’indexation n’aide plus et chaque petit bit compte, on peut utiliser des sélections sans LOCK pour améliorer les performances.

serveur SQL

SELECT * FROM TableName WITH (nolock)

MySQL

SET SESSION TRANSACTION ISOLATION LEVEL READ UNCOMMITTED;
SELECT * FROM TableName;
SET SESSION TRANSACTION ISOLATION LEVEL REPEATABLE READ;

Oracle

SET TRANSACTION ISOLATION LEVEL READ UNCOMMITTED;
SELECT * FROM TableName;

DB2

SELECT * FROM TableName WITH UR;

où UR signifie “lecture non validée”.

S’il est utilisé sur une table comportant des modifications d’enregistrement en cours, les résultats peuvent être imprévisibles.

Sélectionnez avec la condition de plusieurs valeurs de la colonne #

SELECT * FROM Cars  WHERE status IN ( 'Waiting', 'Working' )

Ceci est sémantiquement équivalent à

SELECT * FROM Cars  WHERE ( status = 'Waiting' OR status = 'Working' )

c’est-à-dire que value IN ( <value list> ) est un raccourci pour la disjonction (logique OR).

Obtenir un résultat agrégé pour les groupes de lignes #

Comptage des lignes en fonction d’une valeur de colonne spécifique :

SELECT category, COUNT(*) AS item_count
FROM item
GROUP BY category;

Obtenir le revenu moyen par département :

SELECT department, AVG(income)
FROM employees
GROUP BY department;

L’important est de sélectionner uniquement les colonnes spécifiées dans la clause GROUP BY ou utilisées avec [aggregate functions](http://stackoverflow .com/documentation/sql/222/select/6470/selecting-with-aggregate-functions).

La clause WHERE peut également être utilisée avec GROUP BY, mais WHERE filtre les enregistrements * avant * que tout regroupement soit effectué :

SELECT department, AVG(income)
FROM employees
WHERE department <> 'ACCOUNTING'
GROUP BY department;

Si vous avez besoin de filtrer les résultats une fois le regroupement effectué, par exemple, pour ne voir que les départements dont le revenu moyen est supérieur à 1000, vous devez utiliser le [HAVING](https://www.wikiod.com/fr/docs/sql /636/filter-results-using-where-and-having):

SELECT department, AVG(income)
FROM employees
WHERE department <> 'ACCOUNTING'
GROUP BY department
HAVING avg(income) > 1000;

## Sélectionner des lignes dans plusieurs tables SELECT * FROM table1, table2

SELECT
    table1.column1,
    table1.column2,
    table2.column1
FROM
    table1,
    table2

C’est ce qu’on appelle le produit croisé en SQL, c’est la même chose que le produit croisé dans les ensembles

Ces instructions renvoient les colonnes sélectionnées à partir de plusieurs tables dans une requête.

Il n’y a pas de relation spécifique entre les colonnes renvoyées par chaque table.

Sélection avec null #

SELECT Name FROM Customers WHERE PhoneNumber IS NULL

La sélection avec des valeurs nulles prend une syntaxe différente. N’utilisez pas =, utilisez plutôt IS NULL ou IS NOT NULL.
