Syntaxe #

  • Function([DISTINCT] expression) -DISTINCT est un paramètre facultatif
  • AVG (expression [ ALL | DISTINCT ] )
  • COUNT( { [ALL | DISTINCT ] expression ] | * } )
  • GROUPING(<expression_colonne>)
  • MAX ( expression [ TOUT | DISTINCT ] )
  • MIN (expression [ TOUT | DISTINCT ] )
  • SOMME ( [ ALL | DISTINCT ] expression )
  • VAR ( expression [ TOUT | DISTINCT ] ) OVER ( [ partition_by_clause ] order_by_clause )
  • VARP ( expression [ TOUT | DISTINCT ] ) OVER ( [ partition_by_clause ] order_by_clause
  • STDEV ( expression [ TOUT | DISTINCT ] ) OVER ( [ partition_by_clause ] order_by_clause )
  • STDEVP ( expression [ TOUT | DISTINCT ] ) OVER ( [ partition_by_clause ] order_by_clause )

Dans la gestion de base de données, une fonction d’agrégation est une fonction dans laquelle les valeurs de plusieurs lignes sont regroupées en entrée sur certains critères pour former une valeur unique de signification ou de mesure plus significative, telle qu’un ensemble, un sac ou une liste.

MIN        returns the smallest value in a given column
MAX        returns the largest value in a given column
SUM        returns the sum of the numeric values in a given column
AVG        returns the average value of a given column
COUNT      returns the total number of values in a given column
COUNT(*)   returns the number of rows in a table
GROUPING   Is a column or an expression that contains a column in a GROUP BY clause.
STDEV      returns the statistical standard deviation of all values in the specified expression.
STDEVP     returns the statistical standard deviation for the population for all values in the specified expression.
VAR        returns the statistical variance of all values in the specified expression. may be followed by the OVER clause.
VARP       returns the statistical variance for the population for all values in the specified expression.

Les fonctions d’agrégation sont utilisées pour calculer par rapport à une “colonne renvoyée de données numériques” de votre instruction SELECT. Ils résument essentiellement les résultats d’une colonne particulière de données sélectionnées. - SQLCourse2.com

Toutes les fonctions d’agrégation ignorent les valeurs NULL.

Agrégation conditionnelle #

Tableau des paiements

Client Type_de_paiement Montant

Pierre Crédit 300
Jean Crédit 1000
Jean Débit 500 
select customer, 
       sum(case when payment_type = 'credit' then amount else 0 end) as credit,
       sum(case when payment_type = 'debit' then amount else 0 end) as debit
from payments
group by customer

Résultat:

Pierre 400 0
Jean 1000 500 
select customer, 
       sum(case when payment_type = 'credit' then 1 else 0 end) as credit_transaction_count,
       sum(case when payment_type = 'debit' then 1 else 0 end) as debit_transaction_count
from payments
group by customer

Résultat:

Client credit_transaction_count debit_transaction_count
Pierre 2 0
Jean 1 1

Concaténation de listes #

Crédit partiel à [cette] [1] SO réponse.

La concaténation de liste agrège une colonne ou une expression en combinant les valeurs en une seule chaîne pour chaque groupe. Une chaîne pour délimiter chaque valeur (soit un blanc, soit une virgule en cas d’omission) et l’ordre des valeurs dans le résultat peuvent être spécifiés. Bien qu’il ne fasse pas partie de la norme SQL, tous les principaux fournisseurs de bases de données relationnelles le prennent en charge à leur manière.

#MySQL

SELECT ColumnA
     , GROUP_CONCAT(ColumnB ORDER BY ColumnB SEPARATOR ',') AS ColumnBs
  FROM TableName
 GROUP BY ColumnA
 ORDER BY ColumnA;

#Oracle & DB2

SELECT ColumnA
     , LISTAGG(ColumnB, ',') WITHIN GROUP (ORDER BY ColumnB) AS ColumnBs
  FROM TableName
 GROUP BY ColumnA
 ORDER BY ColumnA;

PostgreSQL #

SELECT ColumnA
     , STRING_AGG(ColumnB, ',' ORDER BY ColumnB) AS ColumnBs
  FROM TableName
 GROUP BY ColumnA
 ORDER BY ColumnA;

Serveur SQL #

SQL Server 2016 et versions antérieures #

(CTE inclus pour encourager le [principe DRY][2])

  WITH CTE_TableName AS (
       SELECT ColumnA, ColumnB
         FROM TableName)
SELECT t0.ColumnA
     , STUFF((
       SELECT ',' + t1.ColumnB
         FROM CTE_TableName t1
        WHERE t1.ColumnA = t0.ColumnA
        ORDER BY t1.ColumnB
          FOR XML PATH('')), 1, 1, '') AS ColumnBs
  FROM CTE_TableName t0
 GROUP BY t0.ColumnA
 ORDER BY ColumnA;

SQL Server 2017 et SQL Azure #

SELECT ColumnA
     , STRING_AGG(ColumnB, ',') WITHIN GROUP (ORDER BY ColumnB) AS ColumnBs
  FROM TableName
 GROUP BY ColumnA
 ORDER BY ColumnA;

#SQLite

sans commander :

SELECT ColumnA
     , GROUP_CONCAT(ColumnB, ',') AS ColumnBs
  FROM TableName
 GROUP BY ColumnA
 ORDER BY ColumnA;

la commande nécessite une sous-requête ou CTE :

  WITH CTE_TableName AS (
       SELECT ColumnA, ColumnB
         FROM TableName
        ORDER BY ColumnA, ColumnB)
SELECT ColumnA
     , GROUP_CONCAT(ColumnB, ',') AS ColumnBs
  FROM CTE_TableName
 GROUP BY ColumnA
 ORDER BY ColumnA;

[1] : http://stackoverflow.com/a/19348687/4896952 [2] : https://en.wikipedia.org/wiki/Don%27t_repeat_yourself

SOMME #

La fonction Sum additionne la valeur de toutes les lignes du groupe. Si la clause group by est omise, alors additionne toutes les lignes.

select sum(salary) TotalSalary
from employees;
Salaire total
2500 
select DepartmentId, sum(salary) TotalSalary
from employees
group by DepartmentId;
ID de service Salaire total
1 2000
2 500

AVG() #

La fonction d’agrégation AVG() renvoie la moyenne d’une expression donnée, généralement des valeurs numériques dans une colonne. Supposons que nous ayons un tableau contenant le calcul annuel de la population dans les villes du monde. Les records pour New York ressemblent à ceux ci-dessous :

TABLEAU EXEMPLE #

nom_ville population année
New York 8 550 405 2015
New York … …
New York 8 000 906 2005

Pour sélectionner la population moyenne de la ville de New York, États-Unis à partir d’un tableau contenant les noms de villes, les mesures de la population et les années de mesure des dix dernières années :

REQUÊTE #

select city_name, AVG(population) avg_population
from city_population
where city_name = 'NEW YORK CITY';

Remarquez que l’année de mesure est absente de la requête puisque la population est moyennée dans le temps.

RÉSULTATS #

nom_ville avg_population
New York 8 250 754

Remarque : La fonction AVG() convertira les valeurs en types numériques. Ceci est particulièrement important à garder à l’esprit lorsque vous travaillez avec des dates.

Compter #

Vous pouvez compter le nombre de lignes :

SELECT count(*) TotalRows
FROM employees;
TotalRows
4

Ou comptez les employés par département :

SELECT DepartmentId, count(*) NumEmployees
FROM employees
GROUP BY DepartmentId;
ID de service NombreEmployés
1 3
2 1

Vous pouvez compter sur une colonne/expression avec l’effet qui ne comptera pas les valeurs NULL :

SELECT count(ManagerId) mgr
FROM EMPLOYEES;
directeur
3

(Il y a une colonne managerID de valeur nulle)

Vous pouvez également utiliser DISTINCT dans une autre fonction telle que COUNT pour rechercher uniquement les membres DISTINCT de l’ensemble sur lesquels effectuer l’opération.

Par exemple:

 SELECT COUNT(ContinentCode) AllCount
 ,      COUNT(DISTINCT ContinentCode) SingleCount
 FROM Countries;

Retournera des valeurs différentes. Le SingleCount ne comptera qu’une seule fois les continents individuels, tandis que le AllCount inclura les doublons.

Max #

Trouvez la valeur maximale de la colonne :

select max(age) from employee;

L’exemple ci-dessus renverra la plus grande valeur pour la colonne “âge” de la table “employé”.

Syntaxe:

SELECT MAX(column_name) FROM table_name;

Min #

Trouvez la plus petite valeur de la colonne :

 select min(age) from employee;

L’exemple ci-dessus renverra la plus petite valeur pour la colonne “âge” de la table “employé”.

Syntaxe:

 SELECT MIN(column_name) FROM table_name;
