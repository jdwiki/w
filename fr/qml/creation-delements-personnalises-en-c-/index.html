Création d’éléments personnalisés en C++ #

QML est venu avec un riche ensemble d’éléments visuels. En utilisant uniquement QML, nous pouvons créer des applications complexes avec ces éléments. Il est également très facile de créer votre propre élément basé sur un ensemble d’éléments standard tels que rectangle, bouton, image, etc. De plus, nous pouvons utiliser des éléments tels que Canvas pour créer des éléments avec une peinture personnalisée. Il semblerait que nous puissions construire une variété d’applications en QML uniquement, sans toucher aux capacités de C++. Et c’est vrai, mais parfois nous aimerions rendre notre application plus rapide ou nous voulons l’étendre avec la puissance de Qt ou ajouter des opportunités qui ne sont pas disponibles dans QML. Et il y a certainement une telle possibilité dans QML. Fondamentalement, QtQuick utilise Scene Graph pour peindre son contenu avec un moteur de rendu hautes performances basé sur OpenGL. Pour implémenter notre propre élément visuel, nous pouvons utiliser 2 façons :

  1. La méthode traditionnelle pour Qt en utilisant [QPainter][1] ([QQuickPaintedItem][2]).
  2. La méthode QML courante utilisant [QQuickItem][3] et la fonctionnalité OpenGL.

Il est possible que la première méthode semble plus facile, mais il convient de considérer qu’elle est également plus lente que la première puisque QtQuick peint le contenu de l’élément sur une surface, puis insérez-le dans le graphe scénique afin que le rendu soit une opération en deux étapes. Ainsi, l’utilisation directe de l’API de graphe de scène est toujours beaucoup plus rapide.

Afin d’explorer les deux méthodes de plus près, créons notre propre élément qui n’existe définitivement pas dans QML, par exemple un triangle.

Déclaration de classe

class QQuickCustomItem : public QQuickItem
{
    Q_OBJECT
    Q_PROPERTY(QColor color READ color WRITE setColor NOTIFY colorChanged)
public:
    QQuickCustomItem(QQuickItem *parent = Q_NULLPTR);

protected:
    QSGNode *updatePaintNode(QSGNode *oldNode, UpdatePaintNodeData *updatePaintNodeData);

    QColor color() const;
    void setColor(const QColor &color);

private:
    QColor m_color;
    bool m_needUpdate;

signals:
    void colorChanged();
};

Nous ajoutons la macro [Q_OBJECT][4] pour travailler avec les signaux. Nous ajoutons également une propriété personnalisée pour spécifier la couleur de notre rectangle. Pour que cela fonctionne, tout ce dont nous avons besoin est de réimplémenter la fonction virtuelle [QQuiclItem::updatePaintNode()][5].

Mise en œuvre de la classe.

Tout d’abord, nous définissons un constructeur.

QQuickCustomItem::QQuickCustomItem(QQuickItem *parent) :
    QQuickItem(parent),
    m_color(Qt::red),
    m_needUpdate(true)
{
    setFlag(QQuickItem::ItemHasContents);
}

Veuillez noter que l’appel de la fonction [setFlag()][6] est obligatoire sinon votre objet ne sera pas ajouté au graphe de scène. Ensuite, nous définissons une fonction pour la douleur.

QSGNode *QQuickCustomItem::updatePaintNode(QSGNode *oldNode, QQuickItem::UpdatePaintNodeData *updatePaintNodeData)
{
    Q_UNUSED(updatePaintNodeData)
    QSGGeometryNode *root = static_cast<QSGGeometryNode *>(oldNode);

    if(!root) {
        root = new QSGGeometryNode;
        QSGGeometry *geometry = new QSGGeometry(QSGGeometry::defaultAttributes_Point2D(), 3);
        geometry->setDrawingMode(GL_TRIANGLE_FAN);
        geometry->vertexDataAsPoint2D()[0].set(width() / 2, 0);
        geometry->vertexDataAsPoint2D()[1].set(width(), height());
        geometry->vertexDataAsPoint2D()[2].set(0, height());

        root->setGeometry(geometry);
        root->setFlag(QSGNode::OwnsGeometry);
        root->setFlag(QSGNode::OwnsMaterial);
    }

    if(m_needUpdate) {
        QSGFlatColorMaterial *material = new QSGFlatColorMaterial;
        material->setColor(m_color);
        root->setMaterial(material);
        m_needUpdate = false;
    }

    return root;
}

Lors du premier appel à la fonction, notre nœud n’est pas encore créé, donc oldNode sera NULL. Nous créons donc le nœud et lui attribuons une géométrie et un matériau. Ici, nous utilisons GL_TRIANGLE_FAN pour notre géométrie afin de peindre un rectangle solide. Ce point est le même que dans OpenGL. Par exemple, pour dessiner un cadre en triangle, nous pouvons changer le code en :

geometry->setDrawingMode(GL_LINE_LOOP);
geometry->setLineWidth(5);

Vous pouvez vous référer au manuel OpenGL pour vérifier d’autres formes. Donc, il ne reste plus qu’à définir setter/getter pour notre propriété :

QColor QQuickCustomItem::color() const
{
    return m_color;
}

void QQuickCustomItem::setColor(const QColor &color)
{
    if(m_color != color) {
        m_color = color;
        m_needUpdate = true;
        update();
        colorChanged();
    }
}

Maintenant, il n’y a qu’un petit détail pour que cela fonctionne. Nous devons informer QtQuick du nouvel élément. Par exemple, vous pouvez ajouter ce code à votre main.cpp :

qmlRegisterType<QQuickCustomItem>("stackoverflow.qml", 1, 0, "Triangle");

Et voici notre fichier de test QML :

import QtQuick 2.7
import QtQuick.Window 2.0
import stackoverflow.qml 1.0

Window {
    width: 800
    height: 800
    visible: true

    Rectangle {
        width: 200
        height: 200
        anchors.centerIn: parent
        color: "lightgrey"

        Triangle {
            id: rect
            width: 200
            height: 200
            transformOrigin: Item.Top
            color: "green"
            onColorChanged: console.log("color was changed");
            PropertyAnimation on rotation {
                from: 0
                to: 360
                duration: 5000
                loops: Animation.Infinite
            }
        }
    }
    Timer {
        interval: 1000
        repeat: true
        running: true
        onTriggered: rect.color = Qt.rgba(Math.random(),Math.random(),Math.random(),1);
    }
}

Comme vous le voyez, notre élément se comporte comme tous les autres éléments QML. Créons maintenant le même élément en utilisant [QPainter][1] :

Tout ce dont nous avons besoin est de remplacer

QSGNode *updatePaintNode(QSGNode *oldNode, UpdatePaintNodeData *updatePaintNodeData);

avec

void paint(QPainter *painter);

et, bien sûr, hériter de notre classe de QQuickPaintedItem au lieu de QQuickItem. Voici notre fonction peinture :

void QQuickCustomItem::paint(QPainter *painter)
{
    QPainterPath path;
    path.moveTo(width() / 2, 0);
    path.lineTo(width(), height());
    path.lineTo(0, height());
    path.lineTo(width() / 2, 0);
    painter->fillPath(path, m_color);
}

Tout le reste reste inchangé.
