Programmation avec PLpgSQL

PL/pgSQL est le langage de programmation intégré de PostgreSQL pour écrire des fonctions qui s’exécutent dans la base de données elle-même, appelées procédures stockées dans d’autres bases de données. Il étend SQL avec des boucles, des conditions et des types de retour. Bien que sa syntaxe puisse sembler étrange à de nombreux développeurs, elle est beaucoup plus rapide que tout ce qui s’exécute sur le serveur d’applications car la surcharge de connexion à la base de données est éliminée, ce qui est particulièrement utile lorsque vous auriez autrement besoin d’exécuter une requête, d’attendre le résultat, et soumettre une autre requête.

Bien que de nombreux autres langages procéduraux existent pour PostgreSQL, tels que PL/Python, PL/Perl et PLV8, PL/pgSQL est un point de départ courant pour les développeurs qui souhaitent écrire leur première fonction PostgreSQL car sa syntaxe repose sur SQL. Il est également similaire à PL/SQL, le langage procédural natif d’Oracle, de sorte que tout développeur familiarisé avec PL/SQL trouvera le langage familier, et tout développeur qui a l’intention de développer des applications Oracle à l’avenir mais qui souhaite commencer avec une base de données gratuite peut faire la transition de PL/pgSQL à PL/SQL avec une relative facilité.

Il convient de souligner que d’autres langages procéduraux existent et que PL/pgSQL ne leur est pas nécessairement supérieur en aucune façon, y compris en termes de vitesse, mais des exemples en PL/pgSQL peuvent servir de point de référence commun pour d’autres langages utilisés pour écrire des fonctions PostgreSQL. PL/pgSQL a le plus de tutoriels et de livres de tous les PL et peut être un tremplin pour apprendre les langages avec moins de documentation.

Fonction PL/pgSQL de base #

Une simple fonction PL/pgSQL :

CREATE FUNCTION active_subscribers() RETURNS bigint AS $$
DECLARE
    -- variable for the following BEGIN ... END block
    subscribers integer;
BEGIN
    -- SELECT must always be used with INTO
    SELECT COUNT(user_id) INTO subscribers FROM users WHERE subscribed;
    -- function result
    RETURN subscribers;
EXCEPTION
    -- return NULL if table "users" does not exist
    WHEN undefined_table
    THEN RETURN NULL;
END;
$$ LANGUAGE plpgsql;

Cela aurait pu être réalisé avec juste l’instruction SQL mais démontre la structure de base d’une fonction.

Pour exécuter la fonction faites :

select active_subscribers();

Syntaxe PL/pgSQL #

CREATE [OR REPLACE] FUNCTION functionName (someParameter 'parameterType') 
RETURNS 'DATATYPE'
AS $_block_name_$
DECLARE
    --declare something
BEGIN
    --do something
    --return something
END;
$_block_name_$
LANGUAGE plpgsql;

Bloc RETOURS #

Options de retour dans une fonction PL/pgSQL :

  • Datatype [Liste de tous les types de données][1]
  • Table(nom_colonne type_colonne, ...)
  • Setof 'Datatype' ou 'table_column'

[1] : https://www.postgresql.org/docs/9.6/static/datatype.html

exceptions personnalisées #

création de l’exception personnalisée ‘P2222’ :

create or replace function s164() returns void as
$$
begin
raise exception using message = 'S 164', detail = 'D 164', hint = 'H 164', errcode = 'P2222';
end;
$$ language plpgsql
;

créer une exception personnalisée sans attribuer errm :

create or replace function s165() returns void as
$$
begin
raise exception '%','nothing specified';
end;
$$ language plpgsql
;

appel:

t=# do
$$
declare
 _t text;
begin
  perform s165();
  exception when SQLSTATE 'P0001' then raise info '%','state P0001 caught: '||SQLERRM;
  perform s164();

end;
$$
;
INFO:  state P0001 caught: nothing specified
ERROR:  S 164
DETAIL:  D 164
HINT:  H 164
CONTEXT:  SQL statement "SELECT s164()"
PL/pgSQL function inline_code_block line 7 at PERFORM

ici personnalisé P0001 traité, et P2222, non, interrompant l’exécution.

Il est également très logique de conserver un tableau des exceptions, comme ici : http://stackoverflow.com/a/2700312/5315974
