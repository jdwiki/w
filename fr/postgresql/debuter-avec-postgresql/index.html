Installer PostgreSQL sous Windows #

Bien qu’il soit recommandé d’utiliser un système d’exploitation basé sur Unix (ex. Linux ou BSD) comme serveur de production, vous pouvez facilement installer PostgreSQL sur Windows (espérons-le uniquement en tant que serveur de développement).

Téléchargez les fichiers binaires d’installation de Windows à partir d’EnterpriseDB : http://www.enterprisedb.com/products-services-training/pgdownload Il s’agit d’une société tierce créée par les principaux contributeurs au projet PostgreSQL qui ont optimisé les fichiers binaires pour Windows.

Sélectionnez la dernière version stable (non bêta) (9.5.3 au moment de la rédaction). Vous voudrez probablement le package Win x86-64, mais si vous utilisez une version 32 bits de Windows, ce qui est courant sur les ordinateurs plus anciens, sélectionnez plutôt Win x86-32.

Remarque : Le basculement entre les versions bêta et stable implique des tâches complexes telles que le vidage et la restauration. La mise à niveau dans la version bêta ou stable ne nécessite qu’un redémarrage du service.

Vous pouvez vérifier si votre version de Windows est 32 ou 64 bits en allant dans Panneau de configuration -> Système et sécurité -> Système -> Type de système, qui indiquera “Système d’exploitation ##-bit”. C’est le chemin pour Windows 7, il peut être légèrement différent sur d’autres versions de Windows.

Dans le programme d’installation, sélectionnez les packages que vous souhaitez utiliser. Par exemple:

  • pgAdmin ( https://www.pgadmin.org ) est une interface graphique gratuite pour gérer votre base de données et je le recommande vivement. Dans la version 9.6, cela sera installé par défaut.
  • PostGIS ( http://postgis.net ) fournit des fonctionnalités d’analyse géospatiale sur les coordonnées GPS, les distances, etc. très populaires parmi les développeurs SIG.
  • Le package de langue fournit les bibliothèques requises pour les langages procéduraux officiellement pris en charge PL/Python, PL/Perl et PL/Tcl.
  • D’autres packages comme pgAgent, pgBouncer et Slony sont utiles pour les serveurs de production plus importants, uniquement vérifiés si nécessaire.

Tous ces packages optionnels peuvent être installés ultérieurement via “Application Stack Builder”.

Remarque : Il existe également d’autres langages non officiellement pris en charge tels que [PL/V8][3], [PL/Lua][4] PL/Java disponibles.

Ouvrez pgAdmin et connectez-vous à votre serveur en double-cliquant sur son nom, ex. “PostgreSQL 9.5 (hôte local : 5432).

A partir de là, vous pouvez suivre des guides tels que l’excellent livre PostgreSQL: Up and Running, 2nd Edition ( http://shop.oreilly.com/product/0636920032144.do ).

Facultatif : Type de démarrage manuel du service

PostgreSQL s’exécute en tant que service en arrière-plan, ce qui est légèrement différent de la plupart des programmes. Ceci est courant pour les bases de données et les serveurs Web. Son type de démarrage par défaut est Automatique, ce qui signifie qu’il fonctionnera toujours sans aucune intervention de votre part.

Pourquoi voudriez-vous contrôler manuellement le service PostgreSQL ? Si vous utilisez votre PC comme serveur de développement de temps en temps et que vous l’utilisez également pour jouer à des jeux vidéo par exemple, PostegreSQL pourrait ralentir un peu votre système pendant son exécution.

Pourquoi ne voudriez-vous pas un contrôle manuel ? Démarrer et arrêter le service peut être un problème si vous le faites souvent.

Si vous ne remarquez aucune différence de vitesse et préférez éviter les tracas, laissez son type de démarrage sur Automatique et ignorez le reste de ce guide. Autrement…

Allez dans Panneau de configuration -> Système et sécurité -> Outils d’administration.

Sélectionnez “Services” dans la liste, faites un clic droit sur son icône et sélectionnez Envoyer vers -> Bureau pour créer une icône de bureau pour un accès plus pratique.

Fermez la fenêtre Outils d’administration puis lancez les Services à partir de l’icône du bureau que vous venez de créer.

Faites défiler vers le bas jusqu’à ce que vous voyiez un service avec un nom comme postgresql-x##-9.# (ex. “postgresql-x64-9.5”).

Faites un clic droit sur le service postgres, sélectionnez Propriétés -> Type de démarrage -> Manuel -> Appliquer -> OK. Vous pouvez le remettre en mode automatique tout aussi facilement.

Si vous voyez d’autres services liés à PostgreSQL dans la liste tels que “pgbouncer” ou “PostgreSQL Scheduling Agent - pgAgent”, vous pouvez également modifier leur type de démarrage en manuel car ils ne sont pas très utiles si PostgreSQL n’est pas en cours d’exécution. Bien que cela signifie plus de tracas chaque fois que vous démarrez et arrêtez, c’est à vous de décider. Ils n’utilisent pas autant de ressources que PostgreSQL lui-même et peuvent ne pas avoir d’impact notable sur les performances de votre système.

Si le service est en cours d’exécution, son statut indiquera Démarré, sinon il ne s’exécute pas.

Pour le démarrer, cliquez avec le bouton droit de la souris et sélectionnez Démarrer. Une invite de chargement s’affichera et devrait disparaître d’elle-même peu de temps après. Si cela vous donne une erreur, essayez une deuxième fois. Si cela ne fonctionne pas, il y a eu un problème avec l’installation, peut-être parce que vous avez modifié certains paramètres dans Windows, la plupart des gens ne changent pas, donc trouver le problème peut nécessiter des recherches.

Pour arrêter postgres, cliquez avec le bouton droit sur le service et sélectionnez Arrêter.

Si vous obtenez une erreur lors de la tentative de connexion à votre base de données, vérifiez les services pour vous assurer qu’il fonctionne.

Pour d’autres détails très spécifiques sur l’installation d’EDB PostgreSQL, par ex. la version d’exécution de python dans le pack de langue officiel d’une version spécifique de PostgreSQL, reportez-vous toujours au [guide d’installation officiel d’EBD][5] , changez la version en lien avec la version majeure de votre installateur.

[1] : https://www.enterprisedb.com/docs/en/9.5/instguide/EDB_Postgres_Advanced_Server_Installation_Guide.1.49.html# “versions linguistiques” [2] : https://www.enterprisedb.com/docs/en/9.6/instguide/EDB_Postgres_Advanced_Server_Installation_Guide.1.49.html# “version linguistique” [3] : http://www.postgresonline.com/journal/archives/360-PLV8-binaries-for-PostgreSQL-9.5-windows-both-32-bit-and-64-bit.html “PLV8” [4] : https://github.com/pllua/pllua “PL/Lua” [5] : https://www.enterprisedb.com/docs/en/9.6/instguide/toc.html

Installer PostgreSQL à partir de la source sur Linux #

Dépendances :

  • Version GNU Make > 3.80
  • un compilateur C ISO/ ANSI (par exemple gcc)
  • un extracteur type tar ou gzip
  • zlib-devel
  • readline-devel ou libedit-devel

Sources: [Lien vers la dernière source (9.6.3)][1]

Vous pouvez maintenant extraire les fichiers sources :

tar -xzvf postgresql-9.6.3.tar.gz

Il existe un grand nombre d’options différentes pour la configuration de PostgreSQL :

[Lien complet vers la procédure d’installation complète][2]

Petite liste des options disponibles :

  • --prefix=PATH chemin pour tous les fichiers
  • --exec-prefix=PATH chemin pour le fichier dépendant de l’architecture
  • --bindir=PATH chemin pour les programmes exécutables
  • --sysconfdir=PATH chemin pour les fichiers de configuration
  • --with-pgport=NUMBER spécifiez un port pour votre serveur
  • --with-perl ajoute le support perl
  • --with-python ajoute le support de python
  • --with-openssl ajoute le support d’openssl
  • --with-ldap ajoute le support LDAP
  • --with-blocksize=BLOCKSIZE définit la taille de la page en Ko
    • BLOCKSIZE must a power of 2 and between 1 and 32
  • --with-wal-segsize=SEGSIZE définit la taille du segment WAL en Mo
    • SEGSIZE must be a power of 2 between 1 and 64

Allez dans le nouveau dossier créé et exécutez le script cofigure avec les options souhaitées :

./configure --exec=/usr/local/pgsql

Exécutez make pour créer les fichiers d’objets

Exécutez make install pour installer PostgreSQL à partir des fichiers construits

Exécutez make clean pour ranger

Pour l’extension changez le répertoire cd contrib, lancez make et make install

[1] : https://ftp.postgresql.org/pub/source/v9.6.3/postgresql-9.6.3.tar.gz [2] : https://www.postgresql.org/docs/9.6/static/install-procedure.html

Installation sous GNU+Linux #

Sur la plupart des systèmes d’exploitation GNU+Linux, PostgreSQL peut facilement être installé à l’aide du gestionnaire de packages du système d’exploitation.

Famille Red Hat #

Les référentiels peuvent être trouvés ici : https://yum.postgresql.org/repopackages.php

Téléchargez le référentiel sur la machine locale avec la commande

yum -y install https://download.postgresql.org/pub/repos/yum/X.X/redhat/rhel-7-x86_64/pgdg-redhatXX-X.X-X.noarch.rpm

Voir les forfaits disponibles :

yum list available | grep postgres*

Les forfaits nécessaires sont : postgresqlXX postgresqlXX-server postgresqlXX-libs postgresqlXX-contrib

Ceux-ci sont installés avec la commande suivante : yum -y install postgresqlXX postgresqlXX-server postgresqlXX-libs postgresqlXX-contrib

Une fois installé, vous devrez démarrer le service de base de données en tant que propriétaire du service (la valeur par défaut est postgres). Cela se fait avec la commande pg_ctl.

sudo -su postgres
./usr/pgsql-X.X/bin/pg_ctl -D /var/lib/pgsql/X.X/data start

Pour accéder à la base de données en CLI, entrez psql

Famille Debian #

Sur les systèmes d’exploitation [Debian et dérivés][2], saisissez :

sudo apt-get install postgresql

Cela installera le package du serveur PostgreSQL, à la version par défaut proposée par les référentiels de packages du système d’exploitation.

Si la version installée par défaut n’est pas celle que vous souhaitez, vous pouvez utiliser le gestionnaire de packages pour rechercher des versions spécifiques qui peuvent être proposées simultanément.

Vous pouvez également utiliser le référentiel Yum fourni par le projet PostgreSQL (connu sous le nom de [PGDG][3]) pour obtenir une version différente. Cela peut autoriser des versions qui ne sont pas encore proposées par les référentiels de packages du système d’exploitation.

[1] : https://www.postgresql.org/download/linux/redhat/ [2] : https://www.postgresql.org/download/linux/ubuntu/ [3] : http://yum.postgresql.org/repopackages.php

Comment installer PostgreSQL via MacPorts sur OSX #

Pour installer PostgreSQL sur OSX, vous devez savoir quelles versions sont actuellement prises en charge.

Utilisez cette commande pour voir quelles versions vous avez disponibles.

sudo port list | grep "^postgresql[[:digit:]]\{2\}[[:space:]]"

Vous devriez obtenir une liste qui ressemble à ceci :

postgresql80                   @8.0.26         databases/postgresql80
postgresql81                   @8.1.23         databases/postgresql81
postgresql82                   @8.2.23         databases/postgresql82
postgresql83                   @8.3.23         databases/postgresql83
postgresql84                   @8.4.22         databases/postgresql84
postgresql90                   @9.0.23         databases/postgresql90
postgresql91                   @9.1.22         databases/postgresql91
postgresql92                   @9.2.17         databases/postgresql92
postgresql93                   @9.3.13         databases/postgresql93
postgresql94                   @9.4.8          databases/postgresql94
postgresql95                   @9.5.3          databases/postgresql95
postgresql96                   @9.6beta2       databases/postgresql96

Dans cet exemple, la version la plus récente de PostgreSQL prise en charge dans la version 9.6, nous allons donc l’installer.

sudo port install postgresql96-server postgresql96

Vous verrez un journal d’installation comme celui-ci :

--->  Computing dependencies for postgresql96-server
--->  Dependencies to be installed: postgresql96
--->  Fetching archive for postgresql96
--->  Attempting to fetch postgresql96-9.6beta2_0.darwin_15.x86_64.tbz2 from         https://packages.macports.org/postgresql96
--->  Attempting to fetch postgresql96-9.6beta2_0.darwin_15.x86_64.tbz2.rmd160 from https://packages.macports.org/postgresql96
--->  Installing postgresql96 @9.6beta2_0
--->  Activating postgresql96 @9.6beta2_0

To use the postgresql server, install the postgresql96-server port

--->  Cleaning postgresql96
--->  Fetching archive for postgresql96-server
--->  Attempting to fetch postgresql96-server-9.6beta2_0.darwin_15.x86_64.tbz2 from https://packages.macports.org/postgresql96-server
--->  Attempting to fetch postgresql96-server-9.6beta2_0.darwin_15.x86_64.tbz2.rmd160 from https://packages.macports.org/postgresql96-server
--->  Installing postgresql96-server @9.6beta2_0
--->  Activating postgresql96-server @9.6beta2_0

To create a database instance, after install do
 sudo mkdir -p /opt/local/var/db/postgresql96/defaultdb
 sudo chown postgres:postgres /opt/local/var/db/postgresql96/defaultdb
 sudo su postgres -c '/opt/local/lib/postgresql96/bin/initdb -D /opt/local/var/db/postgresql96/defaultdb'

--->  Cleaning postgresql96-server
--->  Computing dependencies for postgresql96
--->  Cleaning postgresql96
--->  Updating database of binaries
--->  Scanning binaries for linking errors
--->  No broken files found.

Le journal fournit des instructions sur le reste des étapes d’installation, nous le faisons donc ensuite.

sudo mkdir -p /opt/local/var/db/postgresql96/defaultdb
sudo chown postgres:postgres /opt/local/var/db/postgresql96/defaultdb
sudo su postgres -c '/opt/local/lib/postgresql96/bin/initdb -D /opt/local/var/db/postgresql96/defaultdb'

Maintenant, nous démarrons le serveur :

sudo port load -w postgresql96-server

Vérifiez que nous pouvons nous connecter au serveur :

su postgres -c psql

Vous verrez une invite de postgres :

psql (9.6.1)
Type "help" for help.

postgres=#

Ici, vous pouvez taper une requête pour voir que le serveur est en cours d’exécution.

postgres=#SELECT setting FROM pg_settings WHERE name='data_directory';

Et voir la réponse :

                setting
------------------------------------------
/opt/local/var/db/postgresql96/defaultdb
(1 row)
postgres=#

Tapez \q pour quitter :

postgres=#\q

Et vous serez de retour à l’invite de votre shell.

Toutes nos félicitations! Vous avez maintenant une instance PostgreSQL en cours d’exécution sur OS/X.

Installer postgresql avec brew sur Mac #

Homebrew s’appelle ‘le gestionnaire de paquets manquant pour macOS’. Il peut être utilisé pour créer et installer des applications et des bibliothèques. Une fois [installé][1], vous pouvez utiliser la commande brew pour installer PostgreSQL et ses dépendances comme suit :

brew update
brew install postgresql

Homebrew installe généralement la dernière version stable. Si vous en avez besoin d’une autre, brew search postgresql listera les versions disponibles. Si vous avez besoin de PostgreSQL construit avec des options particulières, alors brew info postgresql listera les options prises en charge. Si vous avez besoin d’une option de construction non prise en charge, vous devrez peut-être faire la construction vous-même, mais vous pouvez toujours utiliser Homebrew pour installer les dépendances communes.

Démarrez le serveur :

brew services start postgresql

Ouvrez l’invite PostgreSQL

psql

Si psql se plaint qu’il n’y a pas de base de données correspondante pour votre utilisateur, exécutez createdb.

[1] : http://brew.sh

Postgres.app pour Mac OSX #

Un outil extrêmement simple pour installer PostgreSQL sur un Mac est disponible en téléchargeant Postgres.app.

Vous pouvez modifier les préférences pour que PostgreSQL s’exécute en arrière-plan ou uniquement lorsque l’application est en cours d’exécution.
