Syntaxe #

  • curl -X<VERB> ‘<PROTOCOL>://<HOST>:<PORT>/<PATH>?<QUERY_STRING>’ -d ‘<BODY>’

  • Où:

  • VERB : la méthode ou le verbe HTTP approprié : GET, POST, PUT, HEAD ou DELETE

  • PROTOCOLE : Soit http, soit https (si vous avez un proxy https devant Elasticsearch.)

  • HOST : Le nom d’hôte de n’importe quel nœud de votre cluster Elasticsearch, ou localhost pour un nœud sur votre machine locale.

  • PORT : le port exécutant le service HTTP Elasticsearch, dont la valeur par défaut est 9200.

  • PATH : API Endpoint (par exemple _count renverra le nombre de documents dans le cluster). Le chemin peut contenir plusieurs composants, tels que _cluster/stats ou _nodes/stats/jvm

  • QUERY_STRING : tous les paramètres de chaîne de requête facultatifs (par exemple, ?pretty imprimera la réponse JSON pour la rendre plus facile à lire.)

- BODY : un corps de requête encodé en JSON (si la requête en a besoin).

  • Référence : [Parler à Elasticsearch : Elasticsearch Docs][1]

[1] : https://www.elastic.co/guide/en/elasticsearch/guide/current/_talking_to_elasticsearch.html#_talking_to_elasticsearch

Curl Commande pour compter le nombre de documents dans le cluster #

curl -XGET 'http://www.example.com:9200/myIndexName/_count?pretty'

Production:

{
  "count" : 90,
  "_shards" : {
    "total" : 6,
    "successful" : 6,
    "failed" : 0
  }
}

L’index contient 90 documents.

Lien de référence : [ici][1]

[1] : https://www.elastic.co/guide/en/elasticsearch/guide/current/_talking_to_elasticsearch.html#_talking_to_elasticsearch

Récupérer un document par identifiant #

curl -XGET 'http://www.example.com:9200/myIndexName/myTypeName/1'

Production:

{
    "_index" : "myIndexName",
    "_type" : "myTypeName",
    "_id" : "1",
    "_version" : 1,
    "found": true,
    "_source" : {
        "user" : "mrunal",
        "postDate" : "2016-07-25T15:48:12",
        "message" : "This is test document!"
    }
}

Lien de référence : [ici][1]

[1] : https://www.elastic.co/guide/en/elasticsearch/reference/current/docs-get.html

Créer un index #

curl -XPUT 'www.example.com:9200/myIndexName?pretty'

Production:

{
  "acknowledged" : true
}

curl 'www.example.com:9200/_cat/indices?v'

Supprimer un index #

curl -XDELETE 'http://www.example.com:9200/myIndexName?pretty'

production:

{
  "acknowledged" : true
}

Lien de référence : [ici][1]

[1] : https://www.elastic.co/guide/en/elasticsearch/reference/1.4/_delete_an_index.html

Liste tous les documents dans un index #

curl -XGET http://www.example.com:9200/myIndexName/_search?pretty=true&q=*:*

Cela utilise l’API Search et renverra toutes les entrées sous l’index myIndexName.
