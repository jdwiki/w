Créer un document simple #

Cet extrait très simple de XQuery peut être exécuté dans QueryConsole en utilisant la base de données “Documents” intégrée comme bac à sable. Chaque morceau de l’extrait de code a un commentaire pour expliquer ce que signifie la ligne de code suivante.

xquery version "1.0-ml";
(: Let's first insert a simple document to get started :)
(: You need a URI- the location where the document is found in the database :)
let $uri := "/stuff/mysimpledocument.xml"
(: Documents need content. This is a simple XML node :)
let $doc := 
  <my-document>
    <body>Very simple example</body>
  </my-document>
(: Permissions are a big topic. For now, we'll use the default permissions. :)
let $permissions := xdmp:default-permissions()
(: Document collections are optional. One or more can be specified :)
(: Adding a collection for further examples that will use it. :)
let $collections := "simple-example"
(: Now we're just going to insert this document in the database :)
let $insert := xdmp:document-insert($uri,$doc,$permissions,$collections)
return <message>Document saved to {$uri}</message>

Lorsque ceci est exécuté, la console renvoie

<message>Document saved to /stuff/mysimpledocument.xml</message>

Maintenant que le document existe, nous pouvons tester des opérations supplémentaires…

Lire/accéder à notre exemple de document #

Si nous connaissons l’URI souhaitée du document que nous recherchons :

fn:doc("/stuff/mysimpledocument.xml")

Renvoie le document complet de la base de données, en utilisant l’URI pour le localiser.

Puisqu’il s’agit de XQuery, nous pouvons utiliser XPath pour trouver le document lorsque nous connaissons la structure, mais pas l’URI :

/my-document

Renvoie l’élément “mon-document” et son contenu…

Mettre à jour le document simple #

Nous allons maintenant ajouter quelques nœuds XML supplémentaires à l’élément “my-document” et mettre à jour le document. L’extrait contient à nouveau des commentaires pour expliquer ce qui se passe.

xquery version "1.0-ml";
(: We are preserving the same URI as we used originally :)
let $uri := "/stuff/mysimpledocument.xml"
(: Need to get the existing contents so we can append to those :)
let $orig-content := fn:doc($uri)/my-document
(: Documents need content. This is a simple XML node :)
let $new-content := 
  <notes>
    <note>Anything can be changed</note>
    <note>New content is added in this example, but we could replace it all too</note>
  </notes>
(: Now to build the new xml. There's lots of ways to do this :)
(: line 17 inserts the original contents into this new node construct :)
(: line 18 inserts the $new-content after the original contents :)
let $new-doc-content :=
  <my-document>
    {$orig-content/node()}
    {$new-content}
  </my-document>
(: Leave permissions untouched... :)
let $permissions := xdmp:document-get-permissions($uri)
(: Leave collections untouched... :)
let $collections := xdmp:document-get-collections($uri)
(: Now we're just going to insert this document in the database :)
let $insert := xdmp:document-insert($uri,$new-doc-content,$permissions,$collections)
return <message>Document {$uri} updated</message>

Lorsqu’il est exécuté, cela renvoie le message

<message>Document /stuff/mysimpledocument.xml updated</message>

Exécutez maintenant les différentes commandes de lecture ci-dessus pour voir le contenu mis à jour. Ça devrait ressembler à ça:

<my-document>
    <body>Very simple example</body>
    <notes>
        <note>Anything can be changed</note>
        <note>New content is added in this example, but we could replace it all too</note>
    </notes>
</my-document>

Bonus : exemple de recherche simple (une autre façon de lire) #

MarkLogic est avant tout un moteur de recherche, utilisons donc deux méthodes différentes pour rechercher ce document.

Utilisation de search:search()

Cela donne un aperçu de l’utilisation de search:search() pour développer des applications de recherche. Cette bibliothèque fournit des résultats de recherche de type Google et accélérera probablement votre développement d’outils de recherche simples. Plus d’informations et une plongée plus profonde [peuvent être trouvées ici][1].

xquery version "1.0-ml";
import module namespace search = "http://marklogic.com/appservices/search"
     at "/MarkLogic/appservices/search/search.xqy";
(: What is search without a keyword? :)
let $term := "very simple"
return search:search($term)

Le résultat semble un peu déroutant, mais vous pouvez voir qu’il renvoie un résultat, notre exemple de document.

Utilisation de cts:search()

Des situations de recherche plus avancées peuvent nécessiter des capacités de recherche plus granulaires. C’est juste pour aiguiser votre appétit pour ce qui est disponible dans la recherche. Des informations plus détaillées [se trouvent ici][2].

xquery version "1.0-ml";
(: What is search without a keyword? :)
let $term := "very simple"
(: Complex queries can be made from individual cts queries. Here, we just have one simple query :)
let $query := cts:word-query($term,"case-insensitive")
(: Return the documents that match the query :)
return cts:search(fn:doc(),$query)

C’est un exemple incroyablement simple. BTW, si nous voulons récupérer l’URI pour les documents correspondants, au lieu des documents eux-mêmes, nous pouvons changer la dernière ligne de cet extrait en-

return 
  for $result in cts:search(fn:doc(),$query)
    return fn:base-uri($result)

[1] : http://docs.marklogic.com/search [2] : http://docs.marklogic.com/guide/search-dev

Supprimer - Dernier point, mais non le moindre #

Pour compléter des exemples simples d’opérations CRUD, nous présentons les exemples suivants. Faites toujours très attention lorsque vous supprimez des documents.

(: When we know the URI, we can delete it very easily :)
let $uri := "/stuff/mysimpledocument.xml"
return xdmp:document-delete($uri)

ou simplifié :

xdmp:document-delete("/stuff/mysimpledocument.xml")

Vous pouvez certainement utiliser XPath pour trouver le document, en obtenir l’URI, puis le supprimer avec quelque chose comme ceci, mais le danger est que tous les documents renvoyés par les expressions XPath seront supprimés. Pas toujours une bonne chose.

(: Use caution when using XPath to select target docs to delete :)
for $doc in /my-document
return xdmp:document-delete(fn:base-uri($doc))

Vous voulez supprimer tous les documents ? Cela le fera, mais soyez très prudent, vous savez sur quelle base de données votre code s’exécutera.

for $doc in fn:doc()
return xdmp:document-delete(fn:base-uri($doc))
