Définir l’espace de noms d’URL pour une application réutilisable (Django 1.9+) #

Configurez l’URLconf de votre application pour qu’elle utilise automatiquement un espace de noms d’URL en définissant l’attribut app_name :

# In <myapp>/urls.py
from django.conf.urls import url

from .views import overview

app_name = 'myapp'
urlpatterns = [
    url(r'^$', overview, name='overview'),
]

Cela définira l’espace de noms de l’application sur 'myapp' lorsqu’il est inclus dans l’URLconf racine >. L’utilisateur de votre application réutilisable n’a pas besoin d’effectuer de configuration autre que d’inclure vos URL :

# In <myproject>/urls.py
from django.conf.urls import include, url

urlpatterns = [
    url(r'^myapp/', include('myapp.urls')),
]

Votre application réutilisable peut désormais inverser les URL à l’aide de l’espace de noms de l’application :

>>> from django.urls import reverse
>>> reverse('myapp:overview')
'/myapp/overview/'

L’URLconf racine peut toujours définir un espace de noms d’instance avec le paramètre namespace :

# In <myproject>/urls.py
urlpatterns = [
    url(r'^myapp/', include('myapp.urls', namespace='mynamespace')),
]

L’espace de noms d’application et l’espace de noms d’instance peuvent être utilisés pour inverser les URL :

>>> from django.urls import reverse
>>> reverse('myapp:overview')
'/myapp/overview/'
>>> reverse('mynamespace:overview')
'/myapp/overview/'

L’espace de noms de l’instance est par défaut l’espace de noms de l’application s’il n’est pas explicitement défini.

Comment Django gère une requête #

Django gère une requête en routant le chemin de l’URL entrante vers une fonction de vue. La fonction d’affichage est chargée de renvoyer une réponse au client qui a fait la demande. Différentes URL sont généralement gérées par différentes fonctions d’affichage. Pour acheminer la requête vers une fonction de vue spécifique, Django regarde votre configuration d’URL (ou URLconf en abrégé). Le modèle de projet par défaut définit l’URLconf dans <myproject>/urls.py.

Votre URLconf doit être un module python qui définit un attribut nommé urlpatterns, qui est une liste de [django.conf.urls.url()](https://docs.djangoproject.com/en/1.9/ref /urls/#django.conf.urls.url). Chaque instance url() doit au minimum définir une expression régulière (une expression régulière) pour correspondre à l’URL, et une cible, qui est soit une fonction d’affichage, soit une URLconf différente. Si un modèle d’URL cible une fonction d’affichage, il est judicieux de lui donner un nom pour référencer facilement le modèle ultérieurement.

Prenons un exemple basique :

# In <myproject>/urls.py

from django.conf.urls import url

from myapp.views import home, about, blog_detail

urlpatterns = [
    url(r'^$', home, name='home'),
    url(r'^about/$', about, name='about'),
    url(r'^blog/(?P<id>\d+)/$', blog_detail, name='blog-detail'),
]

Cette URLconf définit trois modèles d’URL, ciblant tous une vue : home, about et blog-detail.

  • url(r'^$', maison, nom='maison'),

L’expression régulière contient une ancre de début ‘^’, immédiatement suivie d’une ancre de fin ‘$’. Ce modèle correspondra aux requêtes dont le chemin de l’URL est une chaîne vide et les acheminera vers la vue home définie dans myapp.views.

  • url(r'^about/$', about, name='about'),

Cette expression régulière contient une ancre de début, suivie de la chaîne littérale “about/” et de l’ancre de fin. Cela correspondra à l’URL /about/ et l’acheminera vers la vue about. Étant donné que chaque URL non vide commence par un /, Django coupe commodément la première barre oblique pour vous.

  • url(r'^blog/(?P<id>\d+)/$', blog_detail, name='blog-detail'),

Cette regex est un peu plus complexe. Il définit l’ancre de début et la chaîne littérale blog/, comme le modèle précédent. La partie suivante, (?P<id>\d+), est appelée un groupe de capture. Un groupe de capture, comme son nom l’indique, capture une partie de la chaîne, et Django passe la chaîne capturée comme argument à la fonction d’affichage.

La syntaxe d’un groupe de capture est (?P<nom>motif). name définit le nom du groupe, qui est aussi le nom que Django utilise pour passer l’argument à la vue. Le modèle définit quels caractères correspondent au groupe.

Dans ce cas, le nom est id, donc la fonction blog_detail doit accepter un paramètre nommé id. Le motif est \d+. \d signifie que le modèle ne correspond qu’aux caractères numériques. + signifie que le modèle doit correspondre à un ou plusieurs caractères.

Quelques modèles courants :

Motif Utilisé pour Matchs
\d+ identifiant Un ou plusieurs caractères numériques
[\w-]+ limace Un ou plusieurs caractères alphanumériques, traits de soulignement ou tirets
[0-9]{4} année (longue) Quatre nombres, de zéro à neuf
[0-9]{2} année (court)
mois
jour du mois		 Deux nombres, de zéro à neuf
[^/]+ segment de chemin Tout sauf une barre oblique

Le groupe de capture dans le modèle blog-detail est suivi d’un littéral / et de l’ancre de fin.

Les URL valides incluent :

  • /blog/1/ # passe id='1'
  • /blog/42/ # passe id='42'

Les URL non valides sont par exemple :

  • /blog/a/ # 'a' ne correspond pas à '\d'
  • /blog// # aucun caractère dans le groupe de capture ne correspond à '+'

Django traite chaque modèle d’URL dans le même ordre qu’ils sont définis dans urlpatterns. Ceci est important si plusieurs modèles peuvent correspondre à la même URL. Par exemple:

urlpatterns = [
    url(r'blog/(?P<slug>[\w-]+)/$', blog_detail, name='blog-detail'),
    url(r'blog/overview/$', blog_overview, name='blog-overview'),
]

Dans l’URLconf ci-dessus, le deuxième modèle n’est pas accessible. Le modèle correspondrait à l’URL /blog/overview/, mais au lieu d’appeler la vue blog_overview, l’URL correspondra d’abord au modèle blog-detail et appellera la vue blog_detail avec un argument `slug=’ vue d’ensemble’’.

Pour vous assurer que l’URL /blog/overview/ est acheminée vers la vue blog_overview, le modèle doit être placé au-dessus du modèle blog-detail :

urlpatterns = [
    url(r'blog/overview/$', blog_overview, name='blog-overview'),
    url(r'blog/(?P<slug>[\w-]+)/$', blog_detail, name='blog-detail'),
]
