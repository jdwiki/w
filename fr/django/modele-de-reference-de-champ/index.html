Paramètres #

Paramètre Détails
nul Si vrai, les valeurs vides peuvent être stockées comme “null” dans la base de données
vide Si vrai, alors le champ ne sera pas requis dans les formulaires. Si les champs sont laissés vides, Django utilisera la valeur de champ par défaut.
choix Un itérable d’itérables à 2 éléments à utiliser comme choix pour ce champ. S’il est défini, le champ est rendu sous forme de liste déroulante dans l’administrateur. [('m', 'Homme'),('f','Femme'),('z','Préférer ne pas divulguer')]. Pour regrouper les options, imbriquez simplement les valeurs : [('Video Source',((1,'YouTube'),(2,'Facebook')),('Audio Source',((3, 'Soundcloud'), (4, 'Spotify'))]
db_column Par défaut, django utilise le nom du champ pour la colonne de la base de données. Utilisez ceci pour fournir un nom personnalisé
db_index Si True, un index sera créé sur ce champ dans la base de données
db_tablespace Tablespace à utiliser pour l’index de ce champ. Ce champ n’est utilisé que si le moteur de base de données le prend en charge, sinon il est ignoré.
par défaut La valeur par défaut de ce champ. Peut être une valeur ou un objet appelable. Pour les valeurs par défaut mutables (une liste, un ensemble, un dictionnaire), vous devez utiliser un callable. En raison de la compatibilité avec les migrations, vous ne pouvez pas utiliser les lambdas.
modifiable Si False, le champ n’est pas affiché dans l’administrateur du modèle ni dans aucun ModelForm. La valeur par défaut est “Vrai”.
messages_erreur Utilisé pour personnaliser les messages d’erreur par défaut affichés pour ce champ. La valeur est un dictionnaire, les clés représentant l’erreur et la valeur étant le message. Les clés par défaut (pour les messages d’erreur) sont null, blank, invalid, invalid_choice, unique et unique_for_date ; des messages d’erreur supplémentaires peuvent être définis par des champs personnalisés.
texte_aide Texte à afficher avec le champ, pour aider les utilisateurs. HTML est autorisé.
on_delete Lorsqu’un objet référencé par une clé étrangère est supprimé, Django émule le comportement de la contrainte SQL spécifiée par l’argument on_delete. Il s’agit du deuxième argument positionnel pour les champs “ForeignKey” et “OneToOneField”. Les autres champs n’ont pas cet argument.
clé_primaire Si True, ce champ sera la clé primaire. Django ajoute automatiquement une clé primaire ; cela n’est donc nécessaire que si vous souhaitez créer une clé primaire personnalisée. Vous ne pouvez avoir qu’une seule clé primaire par modèle.
uniques Si True, des erreurs sont générées si des valeurs en double sont saisies pour ce champ. Il s’agit d’une restriction au niveau de la base de données, et pas simplement d’un bloc d’interface utilisateur.
unique_pour_date Définissez la valeur sur DateField ou DateTimeField, et des erreurs seront générées s’il existe des valeurs en double pour la même date ou date-heure.
unique_pour_mois Semblable à unique_for_date, sauf que les vérifications sont limitées pour le mois.
unique_pour_année Semblable à unique_for_date, sauf que les vérifications sont limitées à l’année.
nom_verbeux Un nom convivial pour le champ, utilisé par django à divers endroits (comme la création d’étiquettes dans les formulaires d’administration et de modèle).
validateurs Une liste de validateurs pour ce champ.
  • Vous pouvez écrire vos propres champs si vous le jugez nécessaire
  • Vous pouvez remplacer les fonctions de la classe de modèle de base, le plus souvent la fonction save()

Champs numériques #

Des exemples de champs numériques sont donnés :

Champ automatique

Entier auto-incrémenté généralement utilisé pour les clés primaires.

from django.db import models

class MyModel(models.Model):
    pk = models.AutoField()

Chaque modèle obtient un champ de clé primaire (appelé id) par défaut. Par conséquent, il n’est pas nécessaire de dupliquer un champ id dans le modèle pour les besoins d’une clé primaire.

ChampGrosEntier

Un entier correspondant aux nombres compris entre -9223372036854775808 et 9223372036854775807 (8 octets).

from django.db import models

class MyModel(models.Model):
    number_of_seconds = models.BigIntegerField()

Champ entier

Le IntegerField est utilisé pour stocker des valeurs entières de -2147483648 à 2147483647 (4 Bytes).

from django.db import models

class Food(models.Model):
    name = models.CharField(max_length=255)
    calorie = models.IntegerField(default=0)

Le paramètre default n’est pas obligatoire. Mais il est utile de définir une valeur par défaut.

Champ d’entier positif

Comme un IntegerField, mais doit être positif ou nul (0). Le PositiveIntegerField est utilisé pour stocker des valeurs entières de 0 à 2147483647 (4 Bytes). Cela peut être utile au champ qui devrait être sémantiquement positif. Par exemple, si vous enregistrez des aliments avec leurs calories, cela ne devrait pas être négatif. Ce champ empêchera les valeurs négatives via ses validations.

from django.db import models

class Food(models.Model):
    name = models.CharField(max_length=255)
    calorie = models.PositiveIntegerField(default=0)

Le paramètre default n’est pas obligatoire. Mais il est utile de définir une valeur par défaut.

Champ de petit nombre entier

Le SmallIntegerField est utilisé pour stocker des valeurs entières de -32768 à 32767 (2 Bytes). Ce champ est utile pour les valeurs qui ne sont pas extrêmes.

from django.db import models

class Place(models.Model):
    name = models.CharField(max_length=255)
    temperature = models.SmallIntegerField(null=True)

Champ de petit nombre entier positif

Le SmallIntegerField est utilisé pour stocker des valeurs entières de 0 à 32767 (2 Bytes). Tout comme SmallIntegerField, ce champ est utile pour les valeurs moins élevées et doit être sémantiquement positif. Par exemple, il peut stocker l’âge qui ne peut pas être négatif.

from django.db import models

class Staff(models.Model):
    first_name = models.CharField(max_length=255)
    last_name = models.CharField(max_length=255)
    age = models.PositiveSmallIntegerField(null=True)

Outre PositiveSmallIntegerField est utile pour les choix, c’est la façon Djangoic d’implémenter Enum :

from django.db import models
from django.utils.translation import gettext as _

APPLICATION_NEW = 1
APPLICATION_RECEIVED = 2
APPLICATION_APPROVED = 3
APPLICATION_REJECTED = 4

APLICATION_CHOICES = (
    (APPLICATION_NEW, _('New')),
    (APPLICATION_RECEIVED, _('Received')),
    (APPLICATION_APPROVED, _('Approved')),
    (APPLICATION_REJECTED, _('Rejected')),
)

class JobApplication(models.Model):
    first_name = models.CharField(max_length=255)
    last_name = models.CharField(max_length=255)
    status = models.PositiveSmallIntegerField(
        choices=APLICATION_CHOICES, 
        default=APPLICATION_NEW
    )
    ...

La définition des choix en variables de classe ou variables de module selon la situation est une bonne façon de les utiliser. Si les choix sont passés au champ sans noms conviviaux, cela créera de la confusion.

Champ décimal

Un nombre décimal à précision fixe, représenté en Python par une instance Decimal. Contrairement à IntegerField et ses dérivés, ce champ a 2 arguments obligatoires :

  1. DecimalField.max_digits : Le nombre maximum de chiffres autorisés dans le nombre. Notez que ce nombre doit être supérieur ou égal à decimal_places.
  2. DecimalField.decimal_places : Le nombre de décimales à stocker avec le nombre.

Si vous souhaitez stocker des nombres jusqu’à 99 avec 3 décimales, vous devez utiliser max_digits=5 et decimal_places=3 :

class Place(models.Model):
    name = models.CharField(max_length=255)
    atmospheric_pressure = models.DecimalField(max_digits=5, decimal_places=3)

Clé étrangère #

Le champ ForeignKey est utilisé pour créer une relation “plusieurs à un” entre les modèles. Contrairement à la plupart des autres champs, les arguments de position ne sont pas nécessaires. L’exemple suivant illustre la relation entre la voiture et le propriétaire :

from django.db import models

class Person(models.Model):
    GENDER_FEMALE = 'F'
    GENDER_MALE = 'M'

    GENDER_CHOICES = (
        (GENDER_FEMALE, 'Female'),
        (GENDER_MALE, 'Male'),
    )

    first_name = models.CharField(max_length=100)
    last_name = models.CharField(max_length=100)
    gender = models.CharField(max_length=1, choices=GENDER_CHOICES)
    age = models.SmallIntegerField()


class Car(model.Model)
    owner = models.ForeignKey('Person')
    plate = models.CharField(max_length=15)
    brand = models.CharField(max_length=50)
    model = models.CharField(max_length=50)
    color = models.CharField(max_length=50)

Le premier argument du champ est la classe à laquelle le modèle est lié. Le deuxième argument positionnel est l’argument on_delete. Dans les versions actuelles, cet argument n’est pas requis, mais il le sera dans Django 2.0. La fonctionnalité par défaut de l’argument est illustrée comme suit :

class Car(model.Model)
    owner = models.ForeignKey('Person', on_delete=models.CASCADE)
    ...

Cela entraînera la suppression des objets Voiture du modèle lorsque son propriétaire sera supprimé du modèle Personne. Il s’agit de la fonctionnalité par défaut.

class Car(model.Model)
    owner = models.ForeignKey('Person', on_delete=models.PROTECT)
    ...

Cela empêchera les objets Personne d’être supprimés s’ils sont liés à au moins un objet Voiture. Tous les objets Voiture qui font référence à un objet Personne doivent être supprimés en premier. Et puis l’objet personne peut être supprimé.

CharField #

Le CharField est utilisé pour stocker des longueurs de texte définies. Dans l’exemple ci-dessous, jusqu’à 128 caractères de texte peuvent être stockés dans le champ. La saisie d’une chaîne plus longue entraînera la génération d’une erreur de validation.

from django.db import models

class MyModel(models.Model):
    name = models.CharField(max_length=128, blank=True)

ChampDateHeure #

DateTimeField est utilisé pour stocker les valeurs de date et d’heure.

class MyModel(models.Model):
    start_time = models.DateFimeField(null=True, blank=True)
    created_on = models.DateTimeField(auto_now_add=True)
    updated_on = models.DateTimeField(auto_now=True)

Un DateTimeField a deux paramètres facultatifs :

  • auto_now_add définit la valeur du champ sur la date et l’heure actuelles lorsque l’objet est créé.

  • auto_now définit la valeur du champ sur la date et l’heure actuelles à chaque fois que le champ est enregistré.

Ces options et le paramètre default s’excluent mutuellement.

ChampBinaire #

Il s’agit d’un champ spécialisé, utilisé pour stocker des données binaires. Il n’accepte que les octets. Les données sont sérialisées en base64 lors du stockage.

Comme il stocke des données binaires, ce champ ne peut pas être utilisé dans un filtre.

from django.db import models

class MyModel(models.Model):
    my_binary_data = models.BinaryField()
