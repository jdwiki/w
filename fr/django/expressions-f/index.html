Expressions F()

Une expression F() est un moyen pour Django d’utiliser un objet Python pour faire référence à la valeur d’un champ de modèle ou d’une colonne annotée dans la base de données sans avoir à extraire la valeur dans la mémoire Python. Cela permet aux développeurs d’éviter certaines conditions de concurrence et de filtrer les résultats en fonction des valeurs de champ du modèle.

Syntaxe #

  • depuis django.db.models import F

Éviter les conditions de course #

Voir [cette question Q&R][1] si vous ne savez pas quelles sont les conditions de course.

[1] : http://stackoverflow.com/q/34510/1529346

Le code suivant peut être soumis à des conditions de concurrence :

article = Article.objects.get(pk=69)
article.views_count += 1
article.save()

Si views_count est égal à 1337, cela entraînera une telle requête :

UPDATE app_article SET views_count = 1338 WHERE id=69

Si deux clients accèdent à cet article en même temps, ce qui peut arriver est que la seconde requête HTTP exécute Article.objects.get(pk=69) avant que la première exécute article.save(). Ainsi, les deux requêtes auront views_count = 1337, incrémentez-le et enregistrez views_count = 1338 dans la base de données, alors qu’il devrait en fait être 1339.

Pour résoudre ce problème, utilisez une expression F() :

article = Article.objects.get(pk=69)
article.views_count = F('views_count') + 1
article.save()

Ceci, d’autre part, se traduira par une telle requête:

UPDATE app_article SET views_count = views_count + 1 WHERE id=69

Mise à jour de l’ensemble de requêtes en masse #

Supposons que nous voulions supprimer 2 votes positifs de tous les articles de l’auteur avec l’identifiant 51.
Faire cela uniquement avec Python exécuterait des requêtes N (N étant le nombre d’articles dans le jeu de requêtes):

for article in Article.objects.filter(author_id=51):
    article.upvotes -= 2
    article.save()
    # Note that there is a race condition here but this is not the focus
    # of this example.

Et si au lieu d’extraire tous les articles dans Python, de les parcourir en boucle, de diminuer les votes positifs et de sauvegarder chaque article mis à jour dans la base de données, il y avait un autre moyen ?
L’utilisation d’une expression F() permet de le faire en une seule requête :

Article.objects.filter(author_id=51).update(upvotes=F('upvotes') - 2)

Qui peut être traduit dans la requête SQL suivante :

UPDATE app_article SET upvotes = upvotes - 2 WHERE author_id = 51

Pourquoi est-ce mieux ?

  • Au lieu que Python fasse le travail, nous passons la charge dans la base de données qui est affinée pour faire de telles requêtes.
  • Réduit efficacement le nombre de requêtes de base de données nécessaires pour obtenir le résultat souhaité.

Exécuter des opérations arithmétiques entre les champs #

Les expressions F() peuvent être utilisées pour exécuter des opérations arithmétiques (+, -, * etc.) parmi les champs du modèle, afin de définir une recherche/connexion algébrique entre eux.

  • Soit modèle :

     class MyModel(models.Model):
     int_1 = models.IntegerField()
     int_2 = models.IntegerField()

  • Supposons maintenant que nous voulons récupérer tous les objets de la table MyModel dont les champs int_1 et int_2 satisfont cette équation : int_1 + int_2 >= 5. En utilisant [annotate()][1] et filter() nous obtenons :

     result = MyModel.objects.annotate(
              diff=F(int_1) + F(int_2)
          ).filter(diff__gte=5)

    result now contains all of the aforementioned objects.

Bien que l’exemple utilise des champs “Integer”, cette méthode fonctionnera sur tous les champs sur lesquels une opération arithmétique peut être appliquée.

[1] : https://www.wikiod.com/fr/django/agregations-de-modeles
