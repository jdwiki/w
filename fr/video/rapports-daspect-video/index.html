Les rapports d’aspect sont souvent exprimés sous la forme d’un rapport largeur:hauteur qui est souvent - mais pas toujours - simplifié et parfois aussi sous la forme d’un simple nombre entier à virgule flottante.

Tous les rapports d’aspect suivants ont la même valeur exprimée de différentes manières :

  • 1280:720
  • 16:9
  • 1.77777777777777777777777777778

Format d’affichage d’affichage (DAR) #

[![entrez la description de l’image ici][2]][2]

Ceci est une capture d’écran d’une vidéo en cours de lecture. Vous voyez une vidéo 16:9 normale comme vous vous attendez à voir dans n’importe quelle solution vidéo moderne. Ceci - le rapport d’aspect que le spectateur voit - est ce qu’on appelle le rapport d’aspect d’affichage ou DAR.

D’après les paramètres illustrés, nous voyons que DAR = 1280:720 = 16:9 = 1,777777777777777777777777777778.

[2] : http://i.stack.imgur.com/f8cA4.jpg

Format d’image (PAR) #

En interne, toutes les vidéos sont une simple série d’images. Jetons un coup d’œil à une telle image.

[![entrez la description de l’image ici][1]][1]

Ça a l’air bizarre, non ? En effet. Les images qui composent une vidéo peuvent avoir un rapport d’aspect différent du DAR, le plus souvent pour des raisons algorithmiques (par exemple, seules les tailles multiples de 16 peuvent être compressées par l’algorithme choisi). C’est ce qu’on appelle le format d’image ou PAR.

Dans cet exemple, nous avons les dimensions de l’image exactement inversées entre la forme affichée (16:9) et l’image réelle, donc le PAR est de 9:16. Normalement, les différences sont plus petites mais cet exemple est exagéré pour plus de clarté.

D’après les paramètres illustrés, nous voyons que PAR = 720:1280 = 9:16 = 0,5625

[1] : http://i.stack.imgur.com/MDFeG.jpg

Exemple de format d’image (SAR) #

Comme l’indique l’exemple rapport d’aspect d’image, les vidéos sont des séries d’images qui n’ont pas nécessairement le même rapport d’aspect que le résultat final à afficher à l’utilisateur.

Alors, comment passer de ces images étirées à la sortie normalement affichée ? Vous avez besoin d’un facteur d’étirement ! Ce facteur d’étirement est appliqué à l’image afin de l’amener au format correct pour l’affichage. Il s’agit du sample aspect ratio ou SAR.

[![entrez la description de l’image ici][1]][1]

Le facteur d’étirement est souvent exprimé sous la forme d’un rapport de deux nombres entiers. Vous le calculez comme SAR = PAR / DAR.

D’après les paramètres illustrés, nous voyons que SAR = 9:16 / 16:9 = (9/16)/(16/9) = 81/256 = 3,1604938271604938271604938271605

[1] : http://i.stack.imgur.com/fm4a4.png

Pixel aspect ratio #

Ceci est un autre nom pour sample aspect ratio et doit être évité, car l’acronyme naturel (PAR) entre en conflit avec picture aspect ratio.
