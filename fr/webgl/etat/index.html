Textures #

Les unités de texture sont des états globaux. S’ils étaient implémentés en JavaScript, ils ressembleraient à ceci

// pseudo code
gl = {
  activeTextureUnit: 0,
  textureUnits: [
    { TEXTURE_2D: ?, TEXTURE_CUBE_MAP: ? },
    { TEXTURE_2D: ?, TEXTURE_CUBE_MAP: ? },
    { TEXTURE_2D: ?, TEXTURE_CUBE_MAP: ? },
    { TEXTURE_2D: ?, TEXTURE_CUBE_MAP: ? },
    { TEXTURE_2D: ?, TEXTURE_CUBE_MAP: ? },
    { TEXTURE_2D: ?, TEXTURE_CUBE_MAP: ? },
    { TEXTURE_2D: ?, TEXTURE_CUBE_MAP: ? },
    { TEXTURE_2D: ?, TEXTURE_CUBE_MAP: ? },
    { TEXTURE_2D: ?, TEXTURE_CUBE_MAP: ? },
  ],
};

Vous pouvez choisir quelle unité indexer avec gl.activeTexture.

// pseudo code
gl.activeTexture = function(textureUnit) {
  gl.activeTextureUnit = textureUnit - gl.TEXTURE0;
};

L’appel de gl.bindTexture lie une texture à l’unité de texture active comme ceci

// pseudo code
gl.bindTexture = function(target, texture) {
  var textureUnit = gl.textureUnits[gl.activeTextureUnit];
  textureUnit[target] = texture;
}

Lorsque vous avez un programme de shader qui utilise des textures, vous devez indiquer à ce programme de shader à quelles unités de texture vous avez lié les textures. Par exemple, si vous avez un shader comme celui-ci

uniform sampler2D diffuse;
uniform sampler2D normalMap;
uniform samplerCube environmentMap;

...

Pour vous devez interroger les emplacements uniformes

var diffuseUniformLocation = gl.getUniformLocation(someProgram, "diffuse");
var normalMapUniformLocation = gl.getUniformLocation(someProgram, "normalMap");
var environmmentMapUniformLocation = gl.getUniformLocation(someProgram,
                                                           "environmentMap");

Ensuite, après avoir fait de votre programme de shader le * programme actuel *

gl.useProgram(someProgram);

Vous devez ensuite indiquer au shader sur quelles unités de texture vous avez mis/allez mettre les textures. Par exemple

var diffuseTextureUnit = 3;
var normalMapTextureUnit = 5;
var environmentMapTextureUnit = 2;

gl.uniform1i(diffuseUniformLocation, diffuseTextureUnit);
gl.uniform1i(normalMapUniformLocation, normalMapTextureUnit);
gl.uniform1i(environmentMapUniformLocation, environmentMapTextureUnit);

Maintenant, vous avez indiqué au shader quelles unités vous avez utilisées/allez utiliser. Les unités de texture que vous décidez d’utiliser dépendent entièrement de vous.

Pour lier réellement des textures à des unités de texture, vous feriez quelque chose comme ça

gl.activeTexture(gl.TEXTURE0 + diffuseTextureUnit);
gl.bindTexture(gl.TEXTURE_2D, diffuseTexture);
gl.activeTexture(gl.TEXTURE0 + normalMapTextureUnit);
gl.bindTexture(gl.TEXTURE_2D, normalMapTexture);
gl.activeTexture(gl.TEXTURE0 + environmentMapTextureUnit);
gl.bindTexture(gl.TEXTURE_CUBE_MAP, environmentMapTexture);

Pour les exemples WebGL très simples qui n’utilisent qu’une seule texture, il est courant de ne jamais appeler gl.activeTexture car il s’agit par défaut de l’unité de texture #0. Il est également courant de ne pas appeler gl.uniform1i car les uniformes sont par défaut à 0, donc le programme de shader utilisera par défaut l’unité de texture #0 pour toutes les textures.

Toutes les autres fonctions de texture fonctionnent également à partir de la texture active et des cibles d’unité de texture. Par exemple, gl.texImage2D pourrait ressembler à ceci

gl.texImage2D = function(target, level, internalFormat, width, height, 
                         border, format, type, data) {
   var textureUnit = gl.textureUnits[gl.activeTextureUnit];
   var texture = textureUnit[target];

   // Now that we've looked up the texture form the activeTextureUnit and
   // the target we can effect a specific texture
   ...
};

Les attributs #

Les attributs sont d’état global (*). S’ils étaient implémentés en JavaScript, ils ressembleraient à ceci

 // pseudo code
 gl = {
   ARRAY_BUFFER: null,
   vertexArray: {
     attributes: [
       { enable: ?, type: ?, size: ?, normalize: ?, stride: ?, offset: ?, buffer: ? },
       { enable: ?, type: ?, size: ?, normalize: ?, stride: ?, offset: ?, buffer: ? },
       { enable: ?, type: ?, size: ?, normalize: ?, stride: ?, offset: ?, buffer: ? },
       { enable: ?, type: ?, size: ?, normalize: ?, stride: ?, offset: ?, buffer: ? },
       { enable: ?, type: ?, size: ?, normalize: ?, stride: ?, offset: ?, buffer: ? },
       { enable: ?, type: ?, size: ?, normalize: ?, stride: ?, offset: ?, buffer: ? },
       { enable: ?, type: ?, size: ?, normalize: ?, stride: ?, offset: ?, buffer: ? },
       { enable: ?, type: ?, size: ?, normalize: ?, stride: ?, offset: ?, buffer: ? },
     ],
     ELEMENT_ARRAY_BUFFER: null,
   },
 }

Comme vous pouvez le voir ci-dessus, il y a des attributs “8” et ils sont un état global.

Lorsque vous appelez gl.enableVertexAttribArray(location) ou gl.disableVertexAttribArray vous pouvez y penser comme ceci

// pseudo code
gl.enableVertexAttribArray = function(location) {
  gl.vertexArray.attributes[location].enable = true;
};

gl.disableVertexAttribArray = function(location) {
  gl.vertexArray.attributes[location].enable = false;
};

En d’autres termes, “location” fait directement référence à l’index d’un attribut.

De même, gl.vertexAttribPointer serait implémenté quelque chose comme ça

// pseudo code
gl.vertexAttribPointer = function(location, size, type, normalize, stride, offset) {
  var attrib = gl.vertexArray.attributes[location];
  attrib.size = size;
  attrib.type = type;
  attrib.normalize = normalize;
  attrib.stride = stride ? stride : sizeof(type) * size;
  attrib.offset = offset;
  attrib.buffer = gl.ARRAY_BUFFER;  // !!!! <-----
};

Notez que attrib.buffer est défini sur ce que le gl.ARRAY_BUFFER actuel est défini. gl.ARRAY_BUFFER est défini en appelant gl.bindBuffer(gl.ARRAY_BUFFER, someBuffer).

Donc, ensuite, nous avons des vertex shaders. Dans vertex shader, vous déclarez des attributs. Exemple

attribute vec4 position;
attribute vec2 texcoord;
attribute vec3 normal;

...

void main() {
  ...
}

Lorsque vous liez un vertex shader avec un fragment shader en appelant gl.linkProgram(someProgram) WebGL (le pilote/GPU/navigateur) décident eux-mêmes quel index/emplacement utiliser pour chaque attribut. Vous n’avez aucune idée de ceux qu’ils vont choisir. C’est le navigateur/pilote/GPU. Donc, vous devez lui demander quel attribut avez-vous utilisé pour position, texcoord et normal ?. Pour ce faire, appelez gl.getAttribLocation

var positionLoc = gl.getAttribLocation(program, "position");
var texcoordLoc = gl.getAttribLocation(program, "texcoord");
var normalLoc = gl.getAttribLocation(program, "normal");

Disons positionLoc = 5. Cela signifie que lorsque le vertex shader s’exécute (lorsque vous appelez gl.drawArrays ou gl.drawElements), le vertex shader s’attend à ce que vous ayez l’attribut de configuration 5 avec le bon type, size, offset, stride , tampon etc.

Notez qu’AVANT de lier le programme, vous pouvez choisir les emplacements en appelant gl.bindAttribLoction(program, location, nameOfAttribute). Exemple:

// Tell `gl.linkProgram` to assign `position` to use attribute #7
gl.bindAttribLocation(program, 7, "position");

état complet de l’attribut #

Ce qui manque dans la description ci-dessus, c’est que chaque attribut a également une valeur par défaut. Il est omis ci-dessus car il est peu courant de l’utiliser.

attributes: [
   { enable: ?, type: ?, size: ?, normalize: ?, stride: ?, offset: ?, buffer: ?
     value: [0, 0, 0, 1], },
   { enable: ?, type: ?, size: ?, normalize: ?, stride: ?, offset: ?, buffer: ?
     value: [0, 0, 0, 1], },
   ..

Vous pouvez définir la valeur avec les différentes fonctions gl.vertexAttribXXX. La value est utilisée lorsque enable est false. Lorsque enable est vrai, les données de l’attribut sont extraites du buffer attribué.

Objets de tableau de sommets #

WebGL a une extension, OES_vertex_array_object

Dans le diagramme ci-dessus, OES_vertex_array_object vous permet de créer et de remplacer le vertexArray. Autrement dit

var vao = ext.createVertexArrayOES();

crée l’objet que vous voyez attaché à gl.vertexArray dans le pseudo-code ci-dessus. L’appel de ext.bindVertexArrayOES(vao) affecte votre objet de tableau de sommets créé comme tableau de sommets actuel.

// pseudo code
ext.bindVertexArrayOES = function(vao) {
  gl.vertexArray = vao;
}

Cela vous permet de définir tous les attributs et ELEMENT_ARRAY_BUFFER dans le VAO actuel de sorte que lorsque vous souhaitez dessiner, il s’agit d’un appel à ext.bindVertexArrayOES où, sans l’extension, il s’agirait d’un appel à la fois gl.bindBuffer gl.vertexAttribPointer (et éventuellement gl.enableVertexAttribArray) par attribut.

Uniformes #

Les uniformes sont * selon l’état du programme *. Chaque programme shader a son propre état uniforme et ses propres emplacements.
