Utilisation d'Arduino avec Atmel Studio 7

  • Atmel Studio 7 exemple d’esquisse importée

Sur cette page

  • Atmel Studio 7 exemple d’esquisse importée

Installer #

  • Téléchargez et installez Atmel Studio 7 depuis [ici][1].
  • Achetez un débogueur. Vous pouvez vous débrouiller avec un programmeur ISP, mais si vous voulez des capacités de débogage, ce qui est l’un des grands avantages de l’utilisation d’Atmel Studio, vous aurez besoin d’un débogueur. Je recommande [Atmel ICE][2], car il offre des capacités de débogage pour les arduinos basés sur AVR (comme Uno, pro mini, etc.) et les Arduinos basés sur ARM, tels que Zero et Due. Si vous avez un budget limité, vous pouvez [l’obtenir][3] sans le boîtier en plastique et veillez à ne pas le [choquer][4].

Connexions #

  • Pour l’Uno, utilisez le [câble ICSP à 6 broches] [5]. Branchez un côté dans l’Uno comme indiqué. Branchez l’autre côté dans le port AVR du débogueur.

[![entrez la description de l’image ici][6]][6]

Pour l’Arduino Pro Mini, utilisez le [câble mini squid][7] comme indiqué, en connectant à nouveau l’autre côté du port AVR du débogueur.

[![entrez la description de l’image ici][8]][8]

Considérations de débogage #

Pour le débogage avec l’Uno, vous devrez couper la trace d’activation de la réinitialisation (vous pouvez toujours la souder pour l’utiliser avec l’IDE Arduino) :

[![entrez la description de l’image ici][9]][9]

À l’aide du Pro Mini, si vous avez l’intention de connecter le port série à votre ordinateur à l’aide d’une carte FTDI, ne connectez pas la ligne DTR, car cela interférera avec l’interface Serial Wire Debug (SWD) d’Atmel. Je connecte simplement l’alimentation, la masse, Tx et Rx comme indiqué ci-dessous. Rx et Tx sur Arduino vont à Tx et Rx, respectivement sur la carte FTDI. Certaines cartes FTDI sont étiquetées différemment, donc si le port série ne fonctionne pas, échangez Rx et Tx.

[![entrez la description de l’image ici][10]][10]

Vous devrez alimenter séparément l’Arduino car le débogueur ne l’alimentera pas. Cela peut être fait sur le Pro Mini via la carte FTDI comme indiqué ci-dessus, ou avec un câble USB ou un adaptateur secteur sur l’Uno.

Configuration du logiciel #

Branchez l’Atmel ICE sur votre ordinateur, démarrez Atmel Studio et vous pouvez maintenant importer un projet Arduino existant.

Dans Atmel Studio, allez dans Fichier -> Nouveau -> Projet et sélectionnez “Créer un projet à partir d’un croquis Arduino”. Remplissez les options, y compris les menus déroulants de la carte et de l’appareil.

Allez dans Project -> yourProjectName Properties, cliquez sur Tool, sélectionnez Atmel ICE sous debugger/programmer et debugWire sous interface. Allez dans Débogage -> Démarrer le débogage et arrêtez. Vous devriez voir un avertissement et être invité à régler le fusible DWEN. Choisissez OK, débranchez l’Arduino de l’alimentation et rebranchez-le. Vous pouvez arrêter le débogage en cliquant sur le bouton carré rouge et commencer en cliquant sur le bouton triangle vert. Pour remettre l’Arduino dans un état où il peut être utilisé dans l’IDE Arduino, pendant que vous déboguez, choisissez Debug -> disable debugWIRE and close.

Notez que toutes les fonctions que vous ajoutez doivent également inclure un prototype de fonction (la boucle et la configuration n’en ont pas besoin). Vous pouvez voir celles ajoutées par Atmel Studio en haut de l’esquisse s’il y avait des fonctions lorsque vous avez importé votre projet dans Atmel Studio (voir exemple de code par exemple).

La prise en charge de C++11 est activée par défaut dans Arduino 1.6.6 et supérieur. Cela fournit plus de fonctionnalités du langage C++ et son activation peut augmenter la compatibilité avec le système Arduinio. Pour activer C++11 dans Atmel Studio 7, faites un clic droit sur votre fichier de projet, sélectionnez les propriétés, cliquez sur ToolChain à gauche, cliquez sur Divers sous AVR/GNU C++ Compiler et mettez ```-std=c++11` `` dans le champ Autres indicateurs.

Pour inclure des bibliothèques dans votre esquisse #

Copiez le fichier de bibliothèque .cpp dans C:\Users\YourUserName\Documents\Atmel Studio\7.0\YourSolutionName\YourProjectName\ArduinoCore\src\core, puis dans Atmel Studio, ouvrez la fenêtre Solution Explorer cliquez avec le bouton droit sur le dossier Arduino Core/src/core, choisissez ajouter -> élément existant et choisissez le fichier que vous avez ajouté. Faites de même avec le fichier de bibliothèque .h et le dossier YourProjectName/Dependancies.

Pour ajouter la fenêtre du terminal #

Vous pouvez toujours ouvrir l’IDE Android et utiliser cette fenêtre série (sélectionnez simplement le bon port série), mais pour ajouter une fenêtre série intégrée à Atmel Studio, accédez à Outils -> Extensions et mises à jour, cliquez sur Téléchargements disponibles et recherchez Terminal Window ou Terminal pour Atmel Studio et installez-le. Une fois installé, allez dans Affichage -> Fenêtre du terminal.

Avantages #

La programmation d’Arduino avec un IDE moder comme Atmel Studio 7 vous offre de nombreux avantages par rapport à l’IDE Arduino, y compris le débogage, l’auto-complétion, le saut à la définition et à la déclaration, la navigation avant/arrière, les signets et les options de refactoring pour n’en nommer que quelques-uns.

Vous pouvez configurer les raccourcis clavier en allant dans Outils -> Options -> Environnement -> Clavier. Certains qui accélèrent vraiment le développement sont :

  • Edit.CommentSelection, Edit.UncommentSelection
  • View.NavigateForward, View.NavigateBackward
  • Edit.MoveSelectedLinesUp, Edit.MoveSelectedLinesDown
  • Edit.GoToDefinition

Atmel Studio 7 exemple d’esquisse importée #

Ceci est un exemple de ce à quoi ressemble une simple esquisse Arduino après avoir été importée dans Atmel Studio. Atmel Studio a ajouté les sections générées automatiquement en haut. Le reste est identique au code original Arduino. Si vous développez le projet ArduinoCore qui a été créé et regardez dans le dossier src -> core, vous trouverez main.cpp, le point d’entrée du programme. Là, vous pouvez voir l’appel à la fonction de configuration Arduino et une boucle for sans fin qui appelle la fonction de boucle Arduino encore et encore.

/* Begining of Auto generated code by Atmel studio */
#include <Arduino.h>
/* End of auto generated code by Atmel studio */


// Beginning of Auto generated function prototypes by Atmel Studio
void printA();
// End of Auto generated function prototypes by Atmel Studio

void setup() {
  Serial.begin(9600);
}

void loop() {
    printA();
}

void printA() {
    Serial.println("A");
}
