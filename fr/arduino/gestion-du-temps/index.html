Syntaxe #

  • long millis non signé()

  • micros longs non signés()

  • délai d’annulation (longs millisecondes non signés)

  • void delayMicroseconds (microsecondes longues non signées)

  • Voir [l’en-tête elapsedMillis][1] pour les constructeurs et les opérateurs de cette classe. Bref:

  • elapsedMillis elapsedMillisObject ; crée un objet pour garder une trace du temps depuis sa création ou depuis un autre point explicitement défini dans le temps

  • elapsedMillisObject = 0 ; * réinitialiser le temps suivi par l’objet à “depuis maintenant” *

  • deltaT long non signé = elapsedMillisObject ; nous permet de regarder le temps suivi

  • elapsedMillisObject += et -= * ceux-ci fonctionnent comme prévu *

[1] : https://github.com/pfeerick/elapsedMillis/blob/master/elapsedMillis.h “en-tête elapsedMillis”

Code bloquant ou non bloquant #

Pour des croquis très simples, écrire du code de blocage en utilisant delay() et delayMicroseconds() peut être approprié. Lorsque les choses deviennent plus complexes, l’utilisation de ces fonctions peut présenter certains inconvénients. Certains d’entre eux sont :

  • Perte de temps CPU : des esquisses plus complexes peuvent avoir besoin du CPU pour autre chose en attendant la fin d’une période de clignotement des LED.
  • retards inattendus : lorsque delay() est appelé dans des sous-programmes qui ne sont évidemment pas appelés, par exemple dans les bibliothèques que vous incluez.
  • les événements manquants qui se produisent pendant le délai et ne sont pas gérés par un gestionnaire d’interruption, par exemple les pressions sur les boutons interrogés : un bouton peut être enfoncé pendant 100 ms, mais cela peut être masqué par un delay(500).

Détails d’implémentation #

millis() s’appuie généralement sur une minuterie matérielle qui s’exécute à une vitesse bien supérieure à 1 kHz. Lorsque millis() est appelé, l’implémentation renvoie une valeur, mais vous ne savez pas quel âge elle a réellement. Il est possible que la milliseconde “actuelle” vienne de commencer ou qu’elle se termine juste après cet appel de fonction. Cela signifie que, lors du calcul de la différence entre deux résultats de millis(), vous pouvez vous tromper de presque zéro à presque une milliseconde. Utilisez micros() si une plus grande précision est nécessaire.

L’examen du code source de elapsedMillis révèle qu’il utilise effectivement millis () en interne pour comparer deux points dans le temps, il souffre donc également de cet effet. Encore une fois, il existe l’alternative “elapsedMicros” pour une plus grande précision, à partir de la même bibliothèque.

Blinky non bloquant avec millis() #

C’est très proche de [un exemple de la documentation arduino] [1] :

// set constants for blinking the built-in LED at 1 Hz
#define OUTPIN LED_BUILTIN
#define PERIOD 500  // this is in milliseconds

int ledState = LOW;

// millis() returns an unsigned long so we'll use that to keep track of time
unsigned long lastTime = 0;

void setup() {
  // set the digital pin as output:
  pinMode(OUTPIN, OUTPUT);
}

void loop() {
  unsigned long now = millis();
  if (now - lastTime >= PERIOD) // this will be true every PERIOD milliseconds
  {
    lastTime = now;
    if (ledState == LOW)
    {
      ledState = HIGH;
    }
    else
    {
      ledState = LOW;
    }
    digitalWrite(OUTPIN, ledState);
  }

  // now there's lots of time to do other stuff here
}

Utiliser millis() de cette manière - pour chronométrer les opérations de manière non bloquante - est quelque chose qui est nécessaire assez fréquemment, alors pensez à utiliser la bibliothèque elapsedMillis pour cela.

[1] : https://www.arduino.cc/en/Tutorial/BlinkWithoutDelay

Blinky non bloquant avec la bibliothèque (et la classe) elapsedMillis #

La [bibliothèque elapsedMillis][1] fournit une classe portant le même nom qui garde une trace du temps qui s’est écoulé depuis qu’elle a été créée ou définie sur une certaine valeur :

#include <elapsedMillis.h>

#define OUTPIN LED_BUILTIN
#define PERIOD 500

elapsedMillis ledTime;

bool ledState = false;

void setup() 
{                
  // initialize the digital pin as an output.
  pinMode(OUTPIN, OUTPUT);     
}

void loop()
{
    if (ledTime >= PERIOD) 
    {                
        ledState = !ledState;
        digitalWrite(OUTPIN, ledState);
        ledTime = 0;
    }
    // do other stuff here
}

Vous pouvez voir dans l’exemple que l’objet ledTime est assigné à zéro lorsque la broche LED a été basculée. Cela n’est peut-être pas surprenant à première vue, mais cela a un effet si des choses qui prennent plus de temps se produisent :

Considérez une situation où la comparaison entre ledTime et PERIOD est effectuée après 750 millisecondes. Ensuite, régler ledTime sur zéro signifie que toutes les opérations de basculement suivantes seront de 250 ms “en retard”. Si, au contraire, PERIOD était soustrait de ledTime, la LED verrait une courte période puis continuerait à clignoter comme si rien ne s’était passé.

[1] : http://playground.arduino.cc/Code/ElapsedMillis

blocage de blinky avec delay() #

L’une des manières les plus simples de faire clignoter une LED est la suivante : allumez-la, attendez un peu, éteignez-la, attendez encore et répétez à l’infini :

// set constants for blinking the built-in LED at 1 Hz
#define OUTPIN LED_BUILTIN
#define PERIOD 500

void setup()
{
  pinMode(OUTPIN, OUTPUT);      // sets the digital pin as output
}

void loop()
{
  digitalWrite(OUTPIN, HIGH);   // sets the pin on
  delayMicroseconds(PERIOD);        // pauses for 500 miliseconds      
  digitalWrite(OUTPIN, LOW);    // sets the pin off
  delayMicroseconds(PERIOD);        // pauses for 500 milliseconds

  // doing other time-consuming stuff here will skew the blinking
}

Cependant, attendre comme dans l’exemple ci-dessus gaspille des cycles CPU, car il reste simplement là dans une boucle attendant qu’un certain moment passe. C’est ce que les méthodes non bloquantes, utilisant millis() ou elapsedMillis, font mieux - dans le sens où elles ne brûlent pas autant les capacités du matériel.

Mesurer le temps que quelque chose a pris, en utilisant elapsedMillis et elapsedMicros #

#include <elapsedMillis.h>

void setup() {
  Serial.begin(115200);
  elapsedMillis msTimer;
  elapsedMicros usTimer;

  long int dt = 500;
  delay(dt);

  long int us = usTimer;
  long int ms = msTimer;

  Serial.print("delay(");Serial.print(dt);Serial.println(") took");
  Serial.print(us);Serial.println(" us, or");
  Serial.print(ms);Serial.println(" ms");
}

void loop() {
}

Dans cet exemple, un objet elapsedMillis et un objet elapsedMicros sont utilisés pour mesurer le temps que quelque chose a pris, en les créant juste avant l’exécution de l’expression que nous voulons chronométrer, et en obtenant leurs valeurs par la suite. Ils afficheront des résultats légèrement différents, mais le résultat en millisecondes ne sera pas décalé de plus d’une milliseconde.

Plus d’une tâche sans délai() #

Si vous avez plusieurs tâches à exécuter de manière répétée à des intervalles différents, utilisez cet exemple comme point de départ :

unsigned long intervals[] = {250,2000}; //this defines the interval for each task in milliseconds
unsigned long last[] = {0,0};           //this records the last executed time for each task

void setup() {
  pinMode(LED_BUILTIN, OUTPUT); //set the built-it led pin as output
  Serial.begin(115200);         //initialize serial
}

void loop() {
  unsigned long now = millis();
  if(now-last[0]>=intervals[0]){ last[0]=now; firstTask(); }
  if(now-last[1]>=intervals[1]){ last[1]=now; secondTask(); }
  
  //do other things here
}

void firstTask(){
  //let's toggle the built-in led
  digitalWrite(LED_BUILTIN, digitalRead(LED_BUILTIN)?0:1);
}

void secondTask(){
  //say hello
  Serial.println("hello from secondTask()");
}

Pour ajouter une autre tâche à exécuter toutes les 15 secondes, étendez les variables intervals et last :

unsigned long intervals[] = {250,2000,15000};
unsigned long last[] = {0,0,0};

Ajoutez ensuite une instruction if pour exécuter la nouvelle tâche. Dans cet exemple, je l’ai nommé thirdTask.

if(now-last[2]>=intervals[2]){ last[2]=now; thirdTask(); }

Déclarez enfin la fonction :

void thirdTask(){
  //your code here
}
