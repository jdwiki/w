Utilisez Option<‘T> sur les valeurs nulles #

Dans les langages de programmation fonctionnels tels que F#, les valeurs null sont considérées style potentiellement nocif et médiocre (non idiomatique).

Considérez ce code C# :

string x = SomeFunction ();
int    l = x.Length;

x.Length lancera si x est null ajoutons une protection :

string x = SomeFunction ();
int    l = x != null ? x.Length : 0;

Ou:

string x = SomeFunction () ?? "";
int    l = x.Length;

Ou:

string x = SomeFunction ();
int    l = x?.Length;

Dans les valeurs idiomatiques F # null ne sont pas utilisées, notre code ressemble donc à ceci :

let x = SomeFunction ()
let l = x.Length

Cependant, il est parfois nécessaire de représenter des valeurs vides ou invalides. Alors nous pouvons utiliser Option<'T> :

let SomeFunction () : string option = ...

SomeFunction renvoie soit Some string value ou None. Nous extrayons le valeur string utilisant la correspondance de modèle

let v =
  match SomeFunction () with
  | Some x  -> x.Length
  | None    -> 0

La raison pour laquelle ce code est moins fragile que :

string x = SomeFunction ();
int    l = x.Length;

C’est parce que nous ne pouvons pas appeler Length sur une option de chaîne. Nous devons extraire le chaîne en utilisant la correspondance de modèle et ce faisant, nous sommes assurés que le La valeur string peut être utilisée en toute sécurité.

Le module d’option permet la programmation orientée ferroviaire #

La gestion des erreurs est importante mais peut transformer un algorithme élégant en gâchis. Programmation orientée ferroviaire (ROP) est utilisé pour rendre la gestion des erreurs élégante et composable.

Considérez la fonction simple f :

let tryParse s =
  let b, v = System.Int32.TryParse s
  if b then Some v else None

let f (g : string option) : float option =
  match g with
  | None    -> None
  | Some s  ->
    match tryParse s with           // Parses string to int
    | None              -> None
    | Some v when v < 0 -> None     // Checks that int is greater than 0
    | Some v -> v |> float |> Some  // Maps int to float

Le but de f est d’analyser la valeur d’entrée string (s’il y a Some) dans un ‘int’. Si le int est supérieur à 0 nous le transformons en un float. Dans tout d’autres cas, nous renflouer avec Aucun.

Bien qu’une fonction extrêmement simple, la “correspondance” imbriquée diminue la lisibilité significativement.

ROP observe que nous avons deux types de chemins d’exécution dans notre programme

  1. Happy path - Calculera éventuellement la valeur “Some”
  2. Chemin d’erreur - Tous les autres chemins produisent “Aucun”

Comme les chemins d’erreur sont plus fréquents, ils ont tendance à prendre le dessus sur le code. Nous aimerions que le code du chemin heureux soit le chemin de code le plus visible.

Une fonction équivalente g utilisant ROP pourrait ressembler à ceci :

let g (v : string option) : float option =
  v
  |> Option.bind    tryParse  // Parses string to int
  |> Option.filter  ((<) 0)   // Checks that int is greater than 0
  |> Option.map     float     // Maps int to float

Cela ressemble beaucoup à la façon dont nous avons tendance à traiter les listes et les séquences en F#.

On peut voir une Option<'T> comme une List<'T> qui ne peut contenir que l’élément 0 ou 1 où Option.bind se comporte comme List.pick (conceptuellement Option.bind mappe préférable de List.collect mais List.pick pourrait être plus facile à comprendre).

bind, filter et map gère les chemins d’erreur et g ne contient que le happy code de chemin.

Toutes les fonctions qui acceptent directement Option<_> et renvoient Option<_> sont directement composable avec |> et >>.

ROP augmente donc la lisibilité et la composabilité.

Définition de l’option #

Une Option est une union discriminée avec deux cas, None ou Some.

type Option<'T> = Some of 'T | None

Utilisation des types d’options de C# #

Ce n’est pas une bonne idée d’exposer les types Option au code C#, car C# n’a aucun moyen de les gérer. Les options sont soit d’introduire FSharp.Core comme dépendance dans votre projet C# (ce que vous auriez à faire si vous utilisez une bibliothèque F# non conçue pour l’interopérabilité avec C#), ou de changer ` Aucune valeur à “null”.

Pré-Fa# 4.0 #

Pour ce faire, créez votre propre fonction de conversion :

let OptionToObject opt =
    match opt with
    | Some x -> x 
    | None -> null

Pour les types de valeur, vous devrez recourir à leur mise en boîte ou à l’aide de System.Nullable.

let OptionToNullable x = 
    match x with 
    | Some i -> System.Nullable i
    | None -> System.Nullable ()

Fa# 4.0 #

Dans F # 4.0, les fonctions ofObj, toObj, ofNullable et toNullable ont été introduites dans le module Option. Dans F# interactif, ils peuvent être utilisés comme suit :

let l1 = [ Some 1 ; None ; Some 2]
let l2 = l1 |> List.map Option.toNullable;;

// val l1 : int option list = [Some 1; null; Some 2]
// val l2 : System.Nullable<int> list = [1; null; 2]

let l3 = l2 |> List.map Option.ofNullable;;
// val l3 : int option list = [Some 1; null; Some 2]

// Equality
l1 = l2 // fails to compile: different types
l1 = l3 // true

Notez que None se compile en null en interne. Cependant, en ce qui concerne F #, il s’agit d’un “Aucun”.

let lA = [Some "a"; None; Some "b"]
let lB = lA |> List.map Option.toObj

// val lA : string option list = [Some "a"; null; Some "b"]
// val lB : string list = ["a"; null; "b"]

let lC = lB |> List.map Option.ofObj
// val lC : string option list = [Some "a"; null; Some "b"]

// Equality
lA = lB // fails to compile: different types
lA = lC // true
