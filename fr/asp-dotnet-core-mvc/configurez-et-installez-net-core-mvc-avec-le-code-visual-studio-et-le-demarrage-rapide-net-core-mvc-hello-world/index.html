Cet article donne des idées sur la configuration et l’installation du noyau Asp.Net avec le code Visual Studio. Créez également un modèle et un débogage MVC de base.

Étape 1 - installation du code Visual studio.

Étape 2 - Configuration de .Net core et C#.

Cet article est sur le point de configurer à partir de zéro avec le code open source de Visual Studio et de créer et de déboguer des applications mvc de base .net core.

  • L’emplacement du fichier utilisé ci-dessus est modifié selon les utilisateurs, aucune contrainte.
  • Besoin d’Internet pour télécharger les configurations.

Étape 2 - Configuration de .Net core et C#. #

Après avoir installé le code Visual Studio, configurez .net core et C#.

  • Configurez C # en fonction de la place de marché. URL de référence : [C# market place .net core][1]

[![entrez la description de l’image ici][2]][2]

  1. Lancez le code Visual Studio.
  2. Appuyez sur [ctrl + P]
  3. Collez “ext install csharp” ceci et cliquez.

Une fois les étapes ci-dessus effectuées, l’extension C # est disponible dans VS Code.

Étape 3 - Créer un modèle MVC de base. #


  • Créez votre nouveau dossier de projet et ouvrez-le dans l’invite de commande Windows avec l’emplacement du dossier de projet.

[![entrez la description de l’image ici][1]][1]

  • Tapez “dotnet new -t web” et cliquez. C’est pour créer un nouveau modèle mvc.

[![entrez la description de l’image ici][2]][2]

  • Une fois terminé. ALLEZ à l’emplacement du projet et voyez que le projet mvc de base a été créé.

[![entrez la description de l’image ici][3]][3]

  • Tapez ensuite “dotnet restore” et cliquez. Cela permet de restaurer tous les packages à partir du fichier project.json.

[![entrez la description de l’image ici][4]][4]

[![entrez la description de l’image ici][5]][5]

  • Lancez maintenant VScode et ouvrez le dossier du projet.

[![entrez la description de l’image ici][6]][6]

[![entrez la description de l’image ici][7]][7]

Étape 4 - Exécutez et déboguez l’application. #

Ouvrez le dossier du projet dans VScode.

Exemple ici, je suis en train de définir un point d’arrêt dans le contrôleur domestique.

[![entrez la description de l’image ici][1]][1]

Cliquez maintenant sur l’option de débogage.

[![entrez la description de l’image ici][2]][2]

ajoutez la configuration de débogage comme ci-dessous. Assurez-vous que .Net core Launch (web) est sélectionné.

[![entrez la description de l’image ici][3]][3]

[![entrez la description de l’image ici][4]][4]

Vous pouvez voir que le point d’arrêt atteindra , une fois que vous aurez commencé le débogage en appuyant sur l’icône d’exécution.

Ensuite, continuez. La page Web sera affichée dans le navigateur comme ci-dessous. [![entrez la description de l’image ici][5]][5]

La page Web semble cassée.

Appuyez sur “F12” ou ouvrez l’outil de développement.

Vous pouvez voir quelques erreurs dans la console.

[![entrez la description de l’image ici][6]][6]

Quelques fichiers bootstrap et jquery n’ont pas été chargés.

[Trouvez un script et des fichiers CSS par Ctrl+shift+f dans le code VS et entrez le nom du fichier manqué et recherchez.]

Corrigez cela en ajoutant des scripts avec cdn ou l’emplacement exact du fichier dans le fichier de mise en page.

[![entrez la description de l’image ici][7]][7]

Actualisez maintenant la page et regardez.
