  • Créer un module à partir de zéro (Hello World)

Les modules existent pour être étendus. Vous ne pouvez pas modifier les fichiers the app/code/ sans interdire toute mise à jour future. Au lieu de cela, nous ajoutons un module au répertoire app/code/local (le répertoire local peut être manquant, si c’est le cas, il doit être créé manuellement. Ceci est courant dans les versions ultérieures de Magento) pour ajouter des fonctionnalités locales personnalisées.

Tous les fichiers de configuration du module commencent par une balise <config>. Le nouveau module est déclaré à l’intérieur de la balise <modules>. Nous appellerons un module nommé YOUR_COMPANY_HelloWorld, donc les balises <YOUR_COMPANY_HelloWorld> sont utilisées. Ici, nous définissons la version (toute première = 0.0.1)

Une liste des événements XML peut être trouvée à : http://www.magentocommerce.com/wiki/5_-_modules_and_development/reference/module_config.xml

Si vous rencontrez des problèmes, consultez : http://coding.smashingmagazine.com/2012/11/30/introducing-magento-layout/

Créer un module à partir de zéro (Hello World) #

Le développement de modules personnalisés Magento est au cœur de tout développement Magento ou projet Magento, car à tout moment, vous pouvez souhaiter intégrer votre propre fonctionnalité/module dans votre projet Magento existant.

La première chose que les développeurs doivent désactiver est le cache système. Sinon, le développement deviendra pénible car tout changement nécessitera une chasse d’eau. Depuis le panneau d’administration de magento : accédez à Système > Gestion du cache > Tout sélectionner > Actions : Désactiver. Utilisez le guide suivant pour créer un nouveau module :

  • Créez un dossier dans app/code/local/ - conventions de nommage généralement prenez le nom de votre entreprise, par ex. app/code/local/<VOTRE_ENTREPRISE>.

  • Nous avons maintenant un emplacement personnalisé pour nos modules. Créez un autre répertoire, appelez-le quelque chose en rapport avec le type de module que vous souhaitez créer, par ex. app/code/local/<YOUR_COMPANY>/HelloWorld/ - “HelloWorld” est ce que j’appellerai ce module.

  • Ce répertoire a besoin d’un config.xml pour que Magento le reconnaisse comme un nouveau module. Créez un autre dossier nommé etc. Suivi d’un fichier xml appelé config.xml. Le répertoire devrait ressembler à app/code/local/<YOUR_COMPANY/HelloWorld/etc/config.xml Voici la structure du fichier xml :

     <?xml version="1.0" encoding="UTF-8" ?>
 <config>
    <modules>
       <YOUR_COMPANY_HelloWorld>
          <version>0.0.1</version>
       </YOUR_COMPANY_HelloWorld>
    </modules>
 </config>

  • Ensuite, les modules doivent être déclarés à Magento. Passez à app/etc/modules. Créez un autre document XML et donnez-lui les noms choisis de vos balises : YOUR_COMPANY_HelloWorld dans mon cas. Dans ce document écrivez :

     <?xml version="1.0" encoding="UTF-8"?>
 <config>
    <modules>
       <YOUR_COMPANY_HelloWorld>   
          <!-- Whether our module is active: true or false -->
          <active>true</active>
          <!-- Which code pool to use: core, community or local -->
          <codePool>local</codePool>
       </YOUR_COMPANY_HelloWorld>
    </modules>
 </config>

  • Encore une fois, les balises config et module sont utilisées pour déclarer un nouveau module à Magento. Active est la valeur par défaut accessible dans le Panneau d'administration sous Système > Configuration > Avancé. codePool indique à Magento dans quel répertoire chercher. local dans notre cas

  • Ce module est maintenant mis en place, donc le Modèle de notre structure MVC. vous devriez pouvoir voir votre nouveau module dans le panneau d’administration sous “Système > Configuration > Avancé”. Cependant, il ne fait rien pour le moment ! Vous devrez revenir à notre fichier config.xml et définir les éléments XML.

  • Suite avec le tutoriel; Nous utiliserons certains de ces éléments XML pour créer des classes et manipuler toutes les pages de l’interface de notre site. De retour au fichier config.xml, écrivez ce qui suit sous la balise </modules> :

     <global>
    <!-- adding a new block definition -->
    <blocks>
        <!-- A unique short name for our block files -->
        <helloworld>
           <!-- the location of our modules block -->
           <class>YOUR_COMPANY_HelloWorld_Block</class>
        </helloworld>
    </blocks>
 </global>
 <!-- We are making changes to the frontend -->
 <frontend>
    <!-- We are making changes to the layout of the front end -->
    <layout>
       <!-- we are adding a new update file -->
       <updates>
          <!-- Creating the name of our update and linking it the module -->
          <helloworld module="YOUR_COMPANY_HelloWorld">
              <!-- The name of the layout file we are adding -->
              <file>helloworld.xml</file>
          </helloworld>
      </updates>
   </layout>
 </frontend>

  • Comme vous pouvez le voir, nous étendons constamment plutôt que de manipuler les fichiers principaux. La balise helloworld est en minuscule car elle pointe vers un Handle, et pour la continuité, nous la nommerons aussi fidèlement que possible. Nous le lions ensuite au module YOUR_COMPANY_HelloWorld.

  • Nous changeons la mise en page. Par conséquent, nous devons créer ce Handle dans le répertoire de mise en page. passez à app/design/frontend/base/default/layout. Nous avons dit au module de chercher le fichier helloworld.xml. Il faut donc le créer dans ce répertoire. Qu’est-ce que tu attends. Fais le!! et remplissez-le avec:

    <?xml version="1.0" encoding="UTF-8"?>
<!-- All Layout files begin with this code -->
<layout>
   <!-- this is the layout handle. Default is handled on all pages. We want this module to execute on all pages -->
   <default>
      <!-- This is the block we want to bolt onto the existing before_body_end block -->
      <reference name="before_body_end">
          <!-- We define our new block and template to be added to before_body_end -->
          <block name="helloworld_footer" template="helloworld/footer.phtml" type="helloworld/footer"/>
      </reference>
   </default>
</layout>

  • Maintenant, ceux d’entre vous qui ont une petite expérience de Magento, ou qui ont lu d’autres didacticiels Magento remarquables, peuvent être haletants du fait que nous apportons des modifications à la base/par défaut, car c’est là que se trouvent les fichiers principaux de Magento. Cependant, nous ne modifions aucun fichier ici, nous en créons de nouveaux, et de plus nous préfixons notre nom de fichier avec “helloworld”, il y a donc très peu de chances que cela entre en conflit avec d’autres modules ou cause des problèmes avec la mise à niveau de Magento dans le avenir. Jours heureux!

  • Parce que nous voulons affecter toutes les pages, nous utilisons la balise par défaut et la référençons au bloc structurel before_body_end. Cela jouera le rôle de l’action et déclenchera la section View de notre structure MVC.

  • Nous comprenons maintenant que nous nous attaquons au bloc before_body_end. et le lier à notre bloc personnalisé. C’est ce qu’on appelle une référence et c’est un crochet. Nous ne l’accrochons actuellement à rien d’existant, nous devons donc créer les fichiers nécessaires.

  • Dans helloworld.xml, nous avons indiqué dans le modèle un footer.phtml. Passez à app/design/frontend/base/default/template et créez un répertoire helloworld.

  • Dans ce répertoire, créez le fichier footer.phtml et remplissez-le avec HTML, ce tutoriel écrit simplement ceci pour afficher certaines fonctionnalités PHP liées à notre fichier PHTML :

     <p>Hello Everyone! Todays date is <?php echo $this->getDate() ?></p>

  • Nous devons maintenant créer notre propre objet bloc pour coupler le modèle avec notre fonctionnalité de bloc. Créez le répertoire app/code/local/YOUR_COMPANY/HelloWorld/Block/ et créez le fichier Footer.php à l’intérieur de celui-ci. Cela a été référencé dans notre zeta_layout.xml dans le type “helloworld/footer”. Remplissez ce fichier avec :

     <?php
 class YOUR_COMPANY_HelloWorld_Block_Footer extends Mage_Core_Block_Template {
    public function getDate() {
       $date = date('Y-m-d');
       return urlencode($date);
    }
 }
 ?>

  • C’est la fonctionnalité qui remplira notre appel getDate() que nous avons appelé à partir de notre fichier .phtml. Nous étendons le Mage_Core_Block_Template.

  • Cette fonctionnalité est maintenant terminée. Testez cela en vous rendant sur votre page d’accueil où vous devriez voir votre module dans le pied de page de chaque page !
