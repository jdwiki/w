JEÛNE #

Le format de fichier FASTA est utilisé pour représenter une ou plusieurs séquences de nucléotides ou d’acides aminés sous la forme d’une chaîne continue de caractères. Les séquences sont annotées avec une ligne de commentaire, qui commence par le caractère >, qui précède chaque séquence. La ligne de commentaire est généralement formatée de manière uniforme, dictée par la base de données source de la séquence ou le logiciel de génération. Par exemple:

>gi|62241013|ref|NP_001014431.1| RAC-alpha serine/threonine-protein kinase [Homo sapiens]
MSDVAIVKEGWLHKRGEYIKTWRPRYFLLKNDGTFIGYKERPQDVDQREAPLNNFSVAQCQLMKTERPRP
NTFIIRCLQWTTVIERTFHVETPEEREEWTTAIQTVADGLKKQEEEEMDFRSGSPSDNSGAEEMEVSLAK
PKHRVTMNEFEYLKLLGKGTFGKVILVKEKATGRYYAMKILKKEVIVAKDEVAHTLTENRVLQNSRHPFL
TALKYSFQTHDRLCFVMEYANGGELFFHLSRERVFSEDRARFYGAEIVSALDYLHSEKNVVYRDLKLENL
MLDKDGHIKITDFGLCKEGIKDGATMKTFCGTPEYLAPEVLEDNDYGRAVDWWGLGVVMYEMMCGRLPFY
NQDHEKLFELILMEEIRFPRTLGPEAKSLLSGLLKKDPKQRLGGGSEDAKEIMQHRFFAGIVWQHVYEKK
LSPPFKPQVTSETDTRYFDEEFTAQMITITPPDQDDSMECVDSERRPHFPQFSYSASGTA

L’exemple ci-dessus illustre la séquence d’acides aminés d’une isoforme des gènes AKT1 humains, extraite de la base de données de protéines NCBI. La ligne d’en-tête spécifie que cette séquence peut être identifiée avec le [GI ID] (http://www.ncbi.nlm.nih.gov/genbank/sequenceids/) “62241013” et l’ID de transcrit protéique “NP_001014431.1”. Cette protéine est appelée “RAC-alpha sérine/thréonine-protéine kinase” et est dérivée de l’espèce “Homo sapiens”.

Format d’annotation de mutation (MAF) #

Le [format de fichier MAF] (https://wiki.nci.nih.gov/display/TCGA/Mutation+Annotation+Format+(MAF)+Specification) est un format de fichier texte délimité par des tabulations destiné à décrire les mutations somatiques de l’ADN détectées dans résultats de séquençage et est distinct du type de fichier Format d’alignement multiple, qui est destiné à représenter des séquences de nucléotides alignées. Les en-têtes et l’ordre des colonnes peuvent parfois varier entre les fichiers de différentes sources, mais les noms et les ordres des colonnes, tels que définis dans la spécification, sont les suivants :

Tumor_Sample_Barcode
Matched_Norm_Sample_Barcode
Match_Norm_Seq_Allele1
Match_Norm_Seq_Allele2
Tumor_Validation_Allele1
Tumor_Validation_Allele2
Match_Norm_Validation_Allele1
Match_Norm_Validation_Allele2
Verification_Status4

De nombreux fichiers MAF, tels que ceux disponibles auprès du TCGA, contiennent également des colonnes supplémentaires développant l’annotation de variante. Ces colonnes peuvent inclure des identifiants de transcrit de nucléotide de référence pour les gènes correspondants, des changements représentatifs de codons ou d’acides aminés, des mesures de contrôle qualité, des statistiques de population, etc.

GCT #

Le format de fichier GCT est un format de fichier texte délimité par des tabulations utilisé pour décrire l’expression génique traitée ou les données ARNi, généralement dérivé de l’analyse des puces à ADN. Ces données sont organisées avec un seul gène ou sonde annoté par ligne et un seul échantillon de puce par colonne (au-delà des colonnes d’annotation). Par exemple:

#1.2            
22215    2        
Name    Description    Tumor_One    Normal_One
1007_s_at    DDR1    -0.214548    -0.18069
1053_at    RFC2    0.868853    -1.330921
117_at    HSPA6    1.124814    0.933021
121_at    PAX8    -0.825381    0.102078
1255_g_at    GUCA1A    -0.734896    -0.184104
1294_at    UBE1L    -0.366741    -1.209838

Dans cet exemple, la première ligne spécifie la version de la spécification du fichier GCT, qui dans ce cas est “1.2”. La deuxième ligne spécifie le nombre de lignes de données (‘22215’) et le nombre d’échantillons (‘2’). La ligne d’en-tête spécifie deux colonnes d’annotation (‘Name’ pour les identifiants de l’ensemble de sondes à puce et ‘Description’ pour les symboles de gène couverts par l’ensemble de sondes) et les noms des échantillons testés (‘Tumor_One’ et ‘Normal_One’). Chaque ligne de données au-delà de l’en-tête répertorie un seul identifiant d’ensemble de sondes (dans ce cas, les ensembles de sondes de puce génétique Affymetrix), son symbole de gène correspondant (le cas échéant) et les valeurs normalisées pour chaque échantillon. Les valeurs des données d’échantillon varient en fonction du type de test et des méthodes de normalisation, mais sont généralement des valeurs numériques à virgule flottante signées.

Écriture de séquence au format fasta #

Ceci est un exemple de fonction python pour l’écriture de séquences au format fasta.

Paramètres:

  • filename(String) - Un nom de fichier pour écrire la séquence au format fasta.

  • seq(String) - Une séquence d’ADN ou d’ARN.

  • id(String) - L’ID de la séquence donnée.

  • desc(String) - Une courte description de la séquence donnée.

    import math

      def save_fsta(filename,seq,id,desc):
      fo = open(filename+'.fa',"a")
      header= str(id)+' <'+desc+'> \n'
      fo.write(header)
      count=math.floor(len(seq)/80+1)
      iteration = range(count)
      for i in iteration:
          fo.write(seq[80*(i):80*(i+1)]+'\n')        
      fo.write('\n \n')
      fo.close()

Une autre façon consiste à utiliser textwrap

import textwrap

def save_fasta(filename,seq, id, desc):
    filename+='.fa'
    with open(filename, 'w') as f:
        f.write('>'+id+' <'+desc+'>\n');
        text = textwrap.wrap(seq,80);
        for x in text:
            f.write(x+'\n');
