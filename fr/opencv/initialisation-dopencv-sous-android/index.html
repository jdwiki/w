Initialisation asynchrone #

L’utilisation de l’initialisation asynchrone est une méthode recommandée pour le développement d’applications. Il utilise [OpenCV Manager][1] pour accéder aux bibliothèques OpenCV installées en externe dans le système cible.

Extrait de code implémentant l’initialisation asynchrone :

public class MainActivity extends Activity implements CvCameraViewListener2 {

    private BaseLoaderCallback mLoaderCallback = new BaseLoaderCallback(this) {
        @Override
        public void onManagerConnected(int status) {
            switch(status) {
                case LoaderCallbackInterface.SUCCESS:
                    Log.i(TAG,"OpenCV Manager Connected");
                    //from now onwards, you can use OpenCV API
                    Mat m = new Mat(5, 10, CvType.CV_8UC1, new Scalar(0));
                    break;
                case LoaderCallbackInterface.INIT_FAILED:
                    Log.i(TAG,"Init Failed");
                    break;
                case LoaderCallbackInterface.INSTALL_CANCELED:
                    Log.i(TAG,"Install Cancelled");
                    break;
                case LoaderCallbackInterface.INCOMPATIBLE_MANAGER_VERSION:
                    Log.i(TAG,"Incompatible Version");
                    break;
                case LoaderCallbackInterface.MARKET_ERROR:
                    Log.i(TAG,"Market Error");
                    break;
                default:
                    Log.i(TAG,"OpenCV Manager Install");
                    super.onManagerConnected(status);
                    break;
            }
        }
    };

    @Override
    public void onResume() {
        super.onResume();
        OpenCVLoader.initAsync(OpenCVLoader.OPENCV_VERSION_3_1_0, this, mLoaderCallback);
    }

    ...
}

Dans ce cas, notre application fonctionne avec OpenCV Manager de manière asynchrone. Le rappel OnManagerConnected sera appelé dans le thread de l’interface utilisateur à la fin de l’initialisation.

Veuillez noter qu’il n’est pas autorisé d’utiliser des appels OpenCV ou de charger Les bibliothèques natives dépendantes d’OpenCV avant d’invoquer ce rappel. Chargez votre propres bibliothèques natives qui dépendent d’OpenCV après le succès d’OpenCV initialisation.

L’implémentation par défaut de BaseLoaderCallback traite le contexte de l’application comme Activity et appelle la méthode Activity.finish() pour quitter en cas d’échec de l’initialisation. Pour remplacer ce comportement, vous devez remplacer la méthode finish() de la classe BaseLoaderCallback et implémenter votre propre méthode de finalisation.

Gestionnaire OpenCV #

OpenCV Manager est un service Android destiné à gérer les fichiers binaires de la bibliothèque OpenCV sur les appareils des utilisateurs finaux. Il permet de partager les bibliothèques dynamiques OpenCV entre les applications sur le même appareil.

Le gestionnaire offre les avantages suivants :

  • Moins d’utilisation de la mémoire (environ 40 Mo). Toutes les applications utilisent les mêmes fichiers binaires du service et ne conservent pas les bibliothèques natives à l’intérieur d’elles-mêmes.
  • Optimisations spécifiques au matériel pour toutes les plates-formes prises en charge.
  • Source de bibliothèque OpenCV de confiance. Tous les packages avec OpenCV sont publiés sur le marché Google Play.
  • Mises à jour régulières et corrections de bugs.

Le seul inconvénient est que l’utilisateur est invité à télécharger une application supplémentaire, de sorte que l’expérience utilisateur diminue légèrement.

Plus d’informations : [Android OpenCV Manager][2]

Mise à jour le 18/10/16 : Il y a un bug dans la version OpenCV Manager distribuée sur [Play Store][1] (mis à jour le 21/09/15). Cela n’affecte que la version OpenCV 3.1.0. Lorsque vous exécutez certains Fonctions OpenCV, vous obtenez une erreur SIGSEGV. La version distribuée avec Android SDK fonctionne bien (OpenCV-android-sdk/apk/OpenCV_3.1.0_Manager_3.10_{platform}.apk). Il peut être téléchargé à partir du [site Web OpenCV][3]. Plus d’infos : [Numéro #6247][4].

[1] : https://play.google.com/store/apps/details?id=org.opencv.engine [2] : http://docs.opencv.org/3.0-beta/platforms/android/service/doc/index.html [3] : http://opencv.org/downloads.html [4] : https://github.com/opencv/opencv/issues/6247

Initialisation statique #

Selon cette approche, tous les fichiers binaires OpenCV sont inclus dans votre package d’application. Il est principalement conçu à des fins de développement et de débogage. Cette approche est obsolète pour le code de production, l’initialisation asynchrone est recommandée.

Si votre projet d’application n’a pas de partie JNI, copiez simplement les bibliothèques natives OpenCV correspondantes de OpenCV-3.1.0-android-sdk/sdk/native/libs dans le répertoire de votre projet dans le dossier app/src/main/ jniLibs.

Dans le cas du projet d’application avec une partie JNI, au lieu de copier manuellement les bibliothèques, vous devez modifier votre fichier Android.mk : ajoutez les deux lignes de code suivantes après "include $(CLEAR_VARS)" et avant "include path_to_OpenCV-3.1.0-android-sdk/sdk/native/jni/OpenCV.mk" :

OPENCV_CAMERA_MODULES:=on
OPENCV_INSTALL_MODULES:=on

Le résultat devrait ressembler à ce qui suit :

include $(CLEAR_VARS)
# OpenCV
OPENCV_CAMERA_MODULES:=on
OPENCV_INSTALL_MODULES:=on
include ../../sdk/native/jni/OpenCV.mk

Après cela, les bibliothèques OpenCV seront copiées dans le dossier jniLibs de votre application lors de la construction JNI.

La dernière étape de l’activation d’OpenCV dans votre application est le code d’initialisation Java avant d’appeler l’API OpenCV. Cela peut être fait, par exemple, dans la section statique de la classe Activity :

static {
    if (!OpenCVLoader.initDebug()) {
        // Handle initialization error
    }
}

Si votre application inclut d’autres bibliothèques natives dépendantes d’OpenCV, vous devez les charger après l’initialisation d’OpenCV :

static {
    if (!OpenCVLoader.initDebug()) {
        // Handle initialization error
    } else {
        System.loadLibrary("my_jni_lib1");
        System.loadLibrary("my_jni_lib2");
    }
}

Remarque : La méthode [initDebug()][1] est obsolète pour le code de production. Il est conçu à des fins expérimentales et de développement local uniquement. Si vous souhaitez publier votre application, utilisez une approche avec initialisation asynchrone.

[1] : http://docs.opencv.org/3.0-beta/platforms/android/service/doc/JavaHelper.html#boolean-initdebug
