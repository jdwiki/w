Syntaxe #

  • CascadeClassifier cascade = new CascadeClassifier(“cascade.xml”); // Crée un classificateur en cascade à partir de cascade.xml
  • Image Mat = Imgcodecs.imread(“image.png”); // Convertit image.png en un objet Mat (Matrix)
  • Détections MatOfRect = new MatOfRect(); // Crée un fichier MatOfRect (Matrix of Rectangles) vide, utilisé comme sortie pour nos classes de détection
  • détections.toArray(); // Renvoie un tableau d’objets Rect qui peuvent être itérés
  • Imgproc.rectangle(image, new Point(rect.x, rect.y), new Point(rect.x + rect.width, rect.y + rect.height), new Scalar(0, 255, 0)); // Dessine un rectangle aux contours verts depuis les emplacements x et y du premier point jusqu’aux emplacements x et y du deuxième point sur l’objet Mat “image”. “rect” est un objet Rect, généralement fourni par detections.toArray(). Utilise la classe Point d’OpenCV.
  • Imgcodecs.imwrite(“output.png”, image); // Écrit l’objet Mat modifié “image” dans le “output.png”
  • CascadeClassifier.detectMultiScale(image, détections); // Détecte tout objet dans l’objet Mat “image” et affiche les détections dans l’objet MatOfRect “detections”
  • CascadeClassifier.detectMultiScale(image, détections, scaleFactor, minNeighbors, flags, minSize, maxSize); // Effectue une détection avec des paramètres supplémentaires. Voir les détails ci-dessous.
  • Imgproc.ellipse(image, center, axes, 0, 0, 360, new Scalar(255, 0, 255), thickness, lineType, 0); // Dessine une ellipse sur l’image au point center. Utilise la classe Point d’OpenCV.

Paramètres #

Paramètre Détails
facteur d’échelle De combien la taille de l’image est réduite à chaque échelle d’image. Par défaut = ‘1.1’
minVoisins Combien de voisins un rectangle candidat doit avoir avant de le sélectionner comme objet détecté. Par défaut = ‘4’
drapeaux Drapeaux hérités. Dans la plupart des cas, il doit être défini sur “0”. Par défaut = ‘0’
taille min Taille minimale qu’un rectangle candidat peut avoir. Cela utilise la classe Size d’OpenCV. Peut être utilisé pour réduire le temps de détection et l’utilisation du processeur, ainsi que pour réduire les faux positifs.
taillemax Taille maximale qu’un rectangle candidat peut avoir. Cela utilise la classe Size d’OpenCV. Peut être utilisé pour réduire le temps de détection et l’utilisation du processeur, ainsi que pour réduire les faux positifs.
axes Utilise la classe Size d’OpenCV. Définit la largeur et la hauteur de l’ellipse.
épaisseur Épaisseur de la ligne, en pixels.
typeligne A divers paramètres. 0 est la ligne pleine, 8 est pour une ligne 8-connectée, 4 est pour une ligne 4-connectée, et CV_AA est pour une ligne anticrénelée. Par défaut = ‘8’

Obtention d’une image statique, détection d’éléments dessus et sortie des résultats. #

Veuillez noter que cet exemple utilise OpenCV 3.1.

import org.opencv.core.Mat;
import org.opencv.core.MatOfRect;
import org.opencv.core.Point;
import org.opencv.core.Rect;
import org.opencv.core.Scalar;
import org.opencv.imgcodecs.Imgcodecs;
import org.opencv.imgproc.Imgproc;
import org.opencv.objdetect.CascadeClassifier;

public class Classifier {
    private CascadeClassifier diceCascade = new
        CascadeClassifier("res/newMethod/diceCascade.xml");
    private Mat image;
    private String loc = "path/to/image.png";
    private String output = "path/to/output.png";

    public void detImg() {
    
        Mat image = Imgcodecs.imread(loc); // Reads the image
    
        MatOfRect diceDetections = new MatOfRect(); // Output container
        diceCascade.detectMultiScale(image, diceDetections); // Performs the detection
    
        // Draw a bounding box around each detection.
        for (Rect rect : diceDetections.toArray()) {
            Imgproc.rectangle(image, new Point(rect.x, rect.y),
                new Point(rect.x + rect.width, rect.y + rect.height),
                new Scalar(0, 255, 0));
        }
    
        // Save the visualized detection.
        Imgcodecs.imwrite(output, image);
    
    }
}

Le Rect[] renvoyé par diceDetections.toArray() peut être itéré. Chaque Rect à l’intérieur du tableau aura quatre propriétés principales : x, y, width et height. x et y définissent la position en haut à gauche du rectangle, et width et height renvoient un int de la largeur et de la hauteur du rectangle. Ceci est utilisé lors du dessin de rectangles sur des images. Les paramètres minimaux requis pour la fonction Imgproc.rectangle sont les suivants :

Imgproc.rectangle(Mat image, Point start, Point end, Scalar color);

Les deux “Point” sont utilisés pour les positions du coin supérieur gauche et du coin inférieur droit. Ces positions sont toutes deux absolues par rapport à l’image fournie comme premier paramètre, et non l’une par rapport à l’autre. Ainsi, vous devez ajouter à la fois la position x ou y du rectangle en plus de la largeur ou de la hauteur pour définir correctement le point de fin.

Notez que la classe Point utilisée dans ces paramètres n’est pas la classe Point de la bibliothèque standard de Java. Vous devez importer la classe Point d’OpenCV à la place !

Détection d’images à partir d’un appareil vidéo #

Cet exemple présente la classe VideoCapture, où nous l’utilisons pour prendre une image d’une webcam et l’enregistrer dans une image.

import org.opencv.core.Mat;
import org.opencv.core.MatOfRect;
import org.opencv.core.Point;
import org.opencv.core.Rect;
import org.opencv.core.Scalar;
import org.opencv.imgcodecs.Imgcodecs;
import org.opencv.imgproc.Imgproc;
import org.opencv.objdetect.CascadeClassifier;
import org.opencv.videoio.VideoCapture;

public class Classifier {
    private CascadeClassifier diceCascade = new
        CascadeClassifier("res/newMethod/diceCascade.xml");
    private Mat image;
    private String loc = "path/to/image.png";
    private String output = "path/to/output.png";
    private VideoCapture vc = new VideoCapture();

    public void detImg() {
        vc.open(0); // Opens the video stream

        Mat image = new Mat(); // Creates an empty matrix
        vc.read(image); // Reads the image from the video stream and
            writes it to the image matrix.
    
        MatOfRect diceDetections = new MatOfRect(); // Output container
        diceCascade.detectMultiScale(image, diceDetections); // Performs the detection
    
        // Draw a bounding box around each detection.
        for (Rect rect : diceDetections.toArray()) {
            Imgproc.rectangle(image, new Point(rect.x, rect.y),
                new Point(rect.x + rect.width, rect.y + rect.height),
                new Scalar(0, 255, 0));
        }
    
        // Save the visualized detection.
        Imgcodecs.imwrite(output, image);

        vc.release(); // Closes the stream.
    
    }
}

Conversion d’un objet Mat en un objet BufferedImage #

Cet exemple de Daniel Baggio a été tiré directement de cette réponse StackExchange, mais a été republié pour plus de visibilité.

Cette classe prend un objet Mat et renvoie l’objet BufferedImage utilisé par les bibliothèques javax.swing. Ceci peut être utilisé par un objet Graphics pour dessiner l’image.

private BufferedImage toBufferedImage(Mat m) {
    if (!m.empty()) {
        int type = BufferedImage.TYPE_BYTE_GRAY;
        if (m.channels() > 1) {
            type = BufferedImage.TYPE_3BYTE_BGR;
        }
        int bufferSize = m.channels() * m.cols() * m.rows();
        byte[] b = new byte[bufferSize];
        m.get(0, 0, b); // get all the pixels
        BufferedImage image = new BufferedImage(m.cols(), m.rows(), type);
        final byte[] targetPixels = ((DataBufferByte) image.getRaster().getDataBuffer()).getData();
        System.arraycopy(b, 0, targetPixels, 0, b.length);
        return image;
    }
    
    return null;
}

## Détections dans les détections Cet exemple utilise Dice et les points noirs sur les dés (les pépins) comme objet. Comme l’exemple est assez long, il est essentiel d’expliquer d’abord certains concepts clés pour comprendre l’exemple.

Comprendre le premier exemple, “Obtenir une image statique, détecter des éléments dessus et afficher les résultats”. est essentiel pour comprendre cet exemple, en particulier comment OpenCV dessine des rectangles.

Jetez un oeil à l’image suivante:

[![entrez la description de l’image ici][1]][1]

Nous utiliserons la méthode de sous-imagerie, où nous utilisons une zone détectée comme base pour appliquer plus de détections. Ceci n’est possible que si un objet sera toujours dans un autre objet que nous pouvons détecter, comme nos pips sur nos dés. Cette méthode présente plusieurs avantages :

  • Au lieu de numériser l’intégralité de l’image, nous n’avons qu’à numériser la zone où nous savons que l’objet se trouvera.
  • Supprime toute possibilité de faux positifs en dehors de la zone de détection.

Pour ce faire, nous appliquons d’abord un balayage de classificateur en cascade sur l’ensemble de l’image pour nous donner un objet “MatOfRect” contenant nos grands objets (les dés, dans ce cas). Nous parcourons ensuite le tableau Rect[] donné par la fonction toArray() à partir de l’objet MatOfRect. Cet objet Rect est utilisé pour créer un objet Mat temporaire qui est “recadré” selon les propriétés de l’objet Rect (x, y, largeur, hauteur) à partir de l’image d’origine, où nous pouvons ensuite effectuer des détections sur l’objet temporaire “Mat”. En d’autres termes, nous disons au classificateur de n’effectuer que des détections sur les parties de dés de l’image à la place, et nous spécifions la position de chaque dé en utilisant les objets “Rect” que nous avons obtenus en effectuant une détection sur l’image entière.

Cependant, les objets Rect (pips) ont leurs propriétés relatives à leurs dés, et non à l’image elle-même. Pour résoudre ce problème, lorsque nous voulons dessiner des rectangles sur l’image réelle montrant les emplacements des pips, nous ajoutons à la fois dice.x et dice.y au début Point.

import org.opencv.core.Mat;
import org.opencv.core.MatOfRect;
import org.opencv.core.Point;
import org.opencv.core.Rect;
import org.opencv.core.Scalar;
import org.opencv.core.Size;
import org.opencv.imgcodecs.Imgcodecs;
import org.opencv.imgproc.Imgproc;
import org.opencv.objdetect.CascadeClassifier;
import org.opencv.videoio.VideoCapture;

public class Classifier {

    private CascadeClassifier diceCascade =
        new CascadeClassifier("res/newMethod/diceCascade.xml");
    private CascadeClassifier pipCascade =
        new CascadeClassifier("res/newMethod/pipCascade6.xml");
    private VideoCapture vc = new VideoCapture();
    private Mat image;

    public void openVC(int index) {
        vc.open(index);
    }

    public void closeVC() {
        vc.close();
    }

    public Mat getNextImage() {
        image = new Mat();
        vc.read(image); // Sets the matrix to the current livestream frame.
        
        MatOfRect diceDetections = new MatOfRect(); // Output container
    
        // See syntax for explainations on addition parameters
        diceCascade.detectMultiScale(image, diceDetections, 1.1, 4, 0, new Size(20, 20),
            new Size(38, 38));
        
        // Iterates for every Dice ROI
        for (int i = 0; i < diceDetections.toArray().length; i++) {
            Rect diceRect = diceDetections.toArray()[i];
            
            // Draws rectangles around our detected ROI
            Point startingPoint = new Point(diceRect.x, diceRect.y);
            Point endingPoint = new Point(diceRect.x + diceRect.width,
                diceRect.y + diceRect.height);
            Imgproc.rectangle(image, startingPoint, endingPoint, new Scalar(255, 255, 0));
            
            MatOfRect pipDetections = new MatOfRect();
            
            pipCascade.detectMultiScale(image.submat(diceRect), pipDetections, 1.01, 4, 0,
                new Size(2, 2), new Size(10, 10));
            
            // Gets the number of detected pips and draws a cricle around the ROI
            for (int y = 0; y < pipDetections.toArray().length; y++) {
                // Provides the relative position of the pips to the dice ROI
                Rect pipRect = pipDetections.toArray()[y];
                
                // See syntax explaination
                // Draws a circle around our pips
                Point center = new Point(diceRect.x + pipRect.x + pipRect.width / 2,
                    diceRect.y + pipRect.y + pipRect.height / 2);
                Imgproc.ellipse(image, center, new Size(pipRect.width / 2, pipRect.height / 2),
                     0, 0, 360, new Scalar(255, 0, 255), 1, 0, 0);
            }
        }
        
        return image;
    }
}

La fonction getNextImage() renvoie un objet Mat, qui, en conjonction avec les autres exemples publiés, peut être appelé en permanence et peut être converti en un BufferImage, pour fournir un flux en direct affichant les détections.
