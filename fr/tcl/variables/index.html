Syntaxe #

  • définir varName ?value? – désactivé ?-pas de plainte ? , ?varName varName varName?

  • met $varName

  • met [set varName]

  • variable varName

  • global varName ?varName varName ?

  • Les paramètres entre ?…? tels que ?varName? représentent des arguments optionnels pour une commande Tcl.

  • Documentation : [global][1], [upvar][2]

[1] : http://www.tcl.tk/man/tcl/TclCmd/global.htm [2] : http://www.tcl.tk/man/tcl/TclCmd/upvar.htm

Affectation de valeurs aux variables #

La commande set est utilisée pour assigner des valeurs en Tcl. Lorsqu’il est appelé avec deux arguments de la manière suivante,

% set tempVar "This is a string."
This is a string.

il place le deuxième argument (“Ceci est une chaîne.”) dans l’espace mémoire référencé par le premier argument (tempVar). set renvoie toujours le contenu de la variable nommée dans le premier argument. Dans l’exemple ci-dessus, set renverrait “Ceci est une chaîne”. sans les guillemets.

  • Si value est spécifié, alors le contenu de la variable varName sont définis égaux à valeur.
  • Si varName se compose uniquement de caractères alphanumériques, et sans parenthèses, c’est un scalaire variable.
  • Si varName a la forme varName(index), c’est un membre d’un tableau associatif.

Notez que le nom de la variable n’est pas limité à l’alphabet latin, il peut consister en n’importe quelle combinaison de caractères unicode (par exemple arménien) :

% set տուն house
house
% puts ${տուն}
house

Portée #

set alpha 1

proc myproc {} {
    puts $alpha
}

myproc

Ce code ne fonctionne pas car les deux alphas sont dans des portées différentes.

La commande set alpha 1 crée une variable dans la portée globale (ce qui en fait une variable globale).

La commande puts $alpha est exécutée dans une portée qui est créée lorsque la commande myproc s’exécute.

Les deux périmètres sont distincts. Cela signifie que lorsque puts $alpha essaie de rechercher le nom alpha, il ne trouve aucune variable de ce type.

Nous pouvons cependant résoudre ce problème :

proc myproc {} {
    global alpha beta
    puts $alpha
}

Dans ce cas, deux variables globales, alpha et beta, sont liées à des variables d’alias (portant le même nom) dans la portée de la procédure. La lecture à partir des variables d’alias récupère la valeur dans les variables globales, et leur écriture modifie les valeurs dans les variables globales.

Plus généralement, la commande upvar crée des alias vers des variables de n’importe laquelle des portées précédentes. Il peut être utilisé avec la portée globale (#0):

proc myproc {} {
    upvar #0 alpha alpha beta b
    puts $alpha
}

Les alias peuvent porter le même nom que la variable liée à (alpha) ou un autre nom (beta / b).

Si nous appelons myproc à partir de la portée globale, cette variante fonctionne également :

proc myproc {} {
    upvar 1 alpha alpha beta b
    puts $alpha
}

Le numéro de portée “1” signifie “la portée précédente” ou “la portée de l’appelant”.

À moins que vous ne sachiez vraiment ce que vous faites, #0, 0 et 1 sont les seules étendues qui ont du sens à utiliser avec upvar. (upvar 0 crée un alias local pour une variable locale, pas strictement une opération de portée.)

Certains autres langages définissent la portée par des accolades et laissent le code s’exécutant dans chaque portée voir tous les noms dans les portées environnantes. En Tcl, une seule portée est créée lorsqu’une procédure est appelée, et seuls ses propres noms sont visibles. Si une procédure appelle une autre procédure, sa portée est empilée au-dessus de la portée précédente, et ainsi de suite. Cela signifie que contrairement aux langages de style C qui n’ont qu’une portée globale et une portée locale (avec des sous-portées), chaque portée agit comme une portée englobante (mais pas immédiatement visible) pour toute portée qu’elle a ouverte. Lorsqu’une procédure revient, sa portée est détruite.

Documentation: global, upvar

Invoquer un ensemble avec un argument #

set peut également être appelé avec un seul argument. Lorsqu’il est appelé avec un seul argument, il renvoie le contenu de cet argument.

% set x 235
235
% set x
235

Suppression de variable(s) #

La commande unset est utilisée pour supprimer une ou plusieurs variables.

unset ?-nocomplain? ?--? ?name name name name?

  • Chaque name est un nom de variable spécifié de n’importe quelle manière acceptable pour la commande set.

  • Si un nom fait référence à un élément d’un tableau, cet élément est supprimé sans affecter le reste du tableau.

  • Si un nom consiste en un nom de tableau sans index entre parenthèses, alors le tableau entier est supprimé.

  • Si -nocomplain est donné comme premier argument, alors toutes les erreurs possibles sont supprimées de la sortie de la commande.

  • L’option – indique la fin des options et doit être utilisée si vous souhaitez supprimer une variable portant le même nom que l’une des options.

    % set x 235 235 % set x 235 % unset x % set x can’t read “x”: no such variable

Impression de la valeur d’une variable #

Pour imprimer la valeur d’une variable telle que,

set tempVar "This is a string."

L’argument dans l’instruction puts est précédé d’un signe $, qui indique à Tcl d’utiliser la valeur de la variable.

% set tempVar "This is a string."
This is a string.
% puts $tempVar
This is a string.

Variables d’espace de noms #

La commande variable garantit qu’une variable d’espace de noms donnée est créée. Tant qu’une valeur ne lui est pas affectée, la valeur de la variable est indéfinie :

namespace eval mynamespace {
    variable alpha
    set alpha 0
}

La variable est accessible depuis l’extérieur de l’espace de noms (de n’importe où, en fait) en lui attachant le nom de l’espace de noms :

set ::mynamespace::alpha

L’accès peut être simplifié au sein d’une procédure en utilisant à nouveau la commande variable :

proc ::mynamespace::myproc {} {
    variable alpha
    set alpha
}

Cela crée un alias local pour la variable d’espace de noms.

Pour une procédure définie dans un autre espace de noms, le nom de la variable doit contenir l’espace de noms dans l’invocation de variable :

proc myproc {} {
    variable ::mynamespace::alpha
    set alpha
}
