Syntaxe #

  • si expr1 ?alors? corps1 elseif expr2 ?alors? corps2 … ?autrement ? ?corpsN?
  • pour démarrer le test du corps suivant
  • tandis que le corps d’essai
  • foreach varlist1 liste1 ?varlist2 liste2 …? corps

Documentation: break, pour, foreach, if, switch, uplevel, while

si / tant que / pour #

si expr1 ?alors? body1 elseif expr2 ?alors? corps2 … ?autrement ? ?corpsN?

exprN est une expression qui donne une valeur booléenne. bodyN est une liste de commandes.

set i 5
if {$i < 10} {
  puts {hello world}
} elseif {$i < 70} {
  puts {enjoy world}
} else {
  puts {goodbye world}
}

pour démarrer le test du corps suivant

start, next et body sont des listes de commandes. test est une expression qui correspond à une valeur booléenne.

La commande break sortira de la boucle. La commande continue passera à la prochaine itération de la boucle.

L’usage courant est :

for {set i 0} {$i < 5} {incr i} {
  puts "$i: hello world"
}

Puisque start et next sont des listes de commandes, n’importe quelle commande peut être présente.

for {set i 0; set j 5} {$i < 5} {incr i; incr j -1} {
  puts "i:$i j:$j"
}

tandis que tester le corps

Le test est une expression qui s’évalue en une valeur booléenne. Tant que test est vrai, body est exécuté.

set x 0
while {$x < 5} {
  puts "hello world"
  incr x
}

La commande break sortira de la boucle. La commande continue passera à la prochaine itération de la boucle.

set lineCount 0
while {[gets stdin line] >= 0} {
  puts "[incr lineCount]: $line"
  if { $line eq "exit" } {
    break
  }  
}

Itération de liste : foreach #

foreach varlist1 list1 ?varlist2 list2 …? corps

foreach est une structure de contrôle puissante qui permet de boucler sur une liste ou plusieurs listes.

set alpha [list a b c d e f]
foreach {key} $alpha {
   puts "key: $key"
}

Plusieurs noms de variables peuvent être spécifiés.

set alphaindexes [list a 1 b 2 c 3 d 4 e 5 f 6]
foreach {key num} $alphaindexes {
   puts "key:$key num:$num"
}

Plusieurs listes peuvent être itérées en même temps.

set alpha [list a b c d e f]
set indexes [list 1 2 3 4 5 6]
foreach {key} $alpha {idx} $indexes {
   puts "key: $key idx:$idx"
}

Ajout d’une nouvelle structure de contrôle à Tcl #

En Tcl, une structure de contrôle est fondamentalement juste une autre commande. C’est une implémentation possible d’une structure de contrôle do ... while / do ... until.

proc do {body keyword expression} {
    uplevel 1 $body
    switch $keyword {
        while {uplevel 1 [list while $expression $body]}
        until {uplevel 1 [list while !($expression) $body]}
        default {
            return -code error "unknown keyword \"$keyword\": must be until or while"
        }
    }
}

Le point commun aux deux types de boucles do est que le script nommé body sera toujours exécuté au moins une fois, nous le faisons donc tout de suite. L’invocation uplevel 1 $body signifie “exécuter le script au niveau de la pile de l’appelant”. De cette façon, toutes les variables utilisées par le script seront visibles et tous les résultats produits resteront au niveau de l’appelant. Le script sélectionne ensuite, en fonction du paramètre keyword, s’il faut itérer pendant qu’une condition est vraie ou jusqu’à ce qu’elle soit fausse, ce qui revient à itérer pendant que la négation logique de la condition est vraie. Si un mot-clé inattendu est donné, un message d’erreur est généré.
