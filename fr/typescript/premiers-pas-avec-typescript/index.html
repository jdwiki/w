Installation et configuration #

Arrière plan #

TypeScript est un sur-ensemble typé de JavaScript qui se compile directement en code JavaScript. Les fichiers TypeScript utilisent généralement l’extension .ts. De nombreux IDE prennent en charge TypeScript sans aucune autre configuration requise, mais TypeScript peut également être compilé avec le package TypeScript Node.JS à partir de la ligne de commande.

IDE #

Visual Studio #

  • Visual Studio 2015 inclut TypeScript.
  • Visual Studio 2013 Update 2 ou version ultérieure inclut TypeScript, ou vous pouvez télécharger TypeScript pour les versions antérieures.

## Code Visual Studio

  • Visual Studio Code (vscode) fournit la saisie semi-automatique contextuelle ainsi que des outils de refactorisation et de débogage pour TypeScript. vscode est lui-même implémenté dans TypeScript. Disponible pour Mac OS X, Windows et Linux.

Tempête Web #

  • WebStorm 2016.2 est livré avec TypeScript et un compilateur intégré. [Webstorm n’est pas gratuit]

IDÉE IntelliJ #

  • IntelliJ IDEA 2016.2 prend en charge Typescript et un compilateur via un [plugin](https://www.jetbrains.com/help/idea/2016.2 /typescript-support.html) maintenu par l’équipe Jetbrains. [IntelliJ n’est pas gratuit]

Atom & atom-typescript #

  • Atom prend en charge TypeScript avec le package atom-typescript.

Texte sublime #

  • Sublime Text prend en charge TypeScript avec le package typescript.

Installation de l’interface de ligne de commande #

Installer Node.js #

Installez le package npm globalement #

Vous pouvez installer TypeScript globalement pour y avoir accès depuis n’importe quel répertoire.

npm install -g typescript

ou

Installez le paquet npm localement #

Vous pouvez installer TypeScript localement et enregistrer dans package.json pour limiter à un répertoire.

npm install typescript --save-dev

Canaux d’installation #

Vous pouvez installer à partir de :

  • Canal stable : npm install typescript
  • Canal bêta : npm install typescript@beta
  • Canal de développement : npm install typescript@next

Compilation du code TypeScript #

La commande de compilation tsc est fournie avec typescript, qui peut être utilisé pour compiler du code.

tsc my-code.ts

Cela crée un fichier my-code.js.

Compiler en utilisant tsconfig.json #

Vous pouvez également fournir des options de compilation qui accompagnent votre code via un fichier tsconfig.json. Pour démarrer un nouveau projet TypeScript, cd dans le répertoire racine de votre projet dans une fenêtre de terminal et exécutez tsc --init. Cette commande générera un fichier tsconfig.json avec des options de configuration minimales, similaires à celles ci-dessous.

{
    "compilerOptions": {
        "module": "commonjs",
        "target": "es5",
        "noImplicitAny": false,
        "sourceMap": false,
        "pretty": true
    },
    "exclude": [
        "node_modules"
    ]
}

Avec un fichier tsconfig.json placé à la racine de votre projet TypeScript, vous pouvez utiliser la commande tsc pour exécuter la compilation.

Syntaxe de base #

TypeScript est un sur-ensemble typé de JavaScript, ce qui signifie que tout le code JavaScript est un code TypeScript valide. TypeScript ajoute de nombreuses nouvelles fonctionnalités en plus de cela.

TypeScript fait de JavaScript un langage fortement typé et orienté objet, semblable à C # et Java. Cela signifie que le code TypeScript a tendance à être plus facile à utiliser pour les grands projets et que le code a tendance à être plus facile à comprendre et à gérer. Le typage fort signifie également que le langage peut (et est) précompilé et que les variables ne peuvent pas se voir attribuer des valeurs qui sont hors de leur plage déclarée. Par exemple, lorsqu’une variable TypeScript est déclarée comme un nombre, vous ne pouvez pas lui attribuer une valeur de texte.

Ce typage fort et cette orientation objet rendent TypeScript plus facile à déboguer et à maintenir, et ce sont deux des points les plus faibles du JavaScript standard.

Déclarations de types #

Vous pouvez ajouter des déclarations de type aux variables, aux paramètres de fonction et aux types de retour de fonction. Le type est écrit après deux-points après le nom de la variable, comme ceci : var num : nombre = 5 ; Le compilateur vérifiera ensuite les types (si possible) lors de la compilation et signalera les erreurs de type.

var num: number = 5;
num = "this is a string";  // error: Type 'string' is not assignable to type 'number'.

Les types de base sont :

  • number (à la fois des nombres entiers et des nombres à virgule flottante)

  • “chaîne”

  • ‘booléen’

  • ‘Tableau’. Vous pouvez spécifier les types des éléments d’un tableau. Il existe deux manières équivalentes de définir des types de tableau : Array<T> et T[]. Par exemple: - number[] - array of numbers - Array<string> - array of strings

  • Tuples. Les tuples ont un nombre fixe d’éléments avec des types spécifiques. - [boolean, string] - tuple where the first element is a boolean and the second is a string. - [number, number, number] - tuple of three numbers.

  • {} - objet, vous pouvez définir ses propriétés ou son indexeur

    • {name: string, age: number} - object with name and age attributes
    • {[key: string]: number} - a dictionary of numbers indexed by string

  • enum - { Rouge = 0, Bleu, Vert } - énumération mappée sur des nombres

  • Fonction. Vous spécifiez des types pour les paramètres et la valeur de retour :

    • (param: number) => string - function taking one number parameter returning string
    • () => number - function with no parameters returning an number.
    • (a: string, b?: boolean) => void - function taking a string and optionally a boolean with no return value.

  • any - Autorise n’importe quel type. Les expressions impliquant any ne sont pas vérifiées.

  • void - représente “rien”, peut être utilisé comme valeur de retour de fonction. Seuls null et undefined font partie du type void.

  • “jamais”

    • let foo: never; -As the type of variables under type guards that are never true.
    • function error(message: string): never { throw new Error(message); } - Comme type de retour des fonctions qui ne reviennent jamais.

  • null - saisissez la valeur null. null fait implicitement partie de chaque type, à moins que les vérifications NULL strictes ne soient activées.

Casting #

Vous pouvez effectuer un casting explicite à travers des crochets angulaires, par exemple :

var derived: MyInterface;
(<ImplementingClass>derived).someSpecificMethod();

Cet exemple montre une classe derived qui est traitée par le compilateur comme MyInterface. Sans le transtypage sur la deuxième ligne, le compilateur lèverait une exception car il ne comprend pas someSpecificMethod(), mais le transtypage via <ImplementingClass>derived suggère au compilateur quoi faire.

Une autre façon de caster dans Typescript consiste à utiliser le mot-clé as :

var derived: MyInterface;
(derived as ImplementingClass).someSpecificMethod();

Depuis Typescript 1.6, la valeur par défaut utilise le mot-clé as, car l’utilisation de <> est ambiguë dans les fichiers .jsx. Ceci est mentionné dans la documentation officielle de Typescript.

Des classes #

Les classes peuvent être définies et utilisées dans le code TypeScript. Pour en savoir plus sur les classes, consultez la page de documentation des classes.

Bonjour le monde #

class Greeter {
    greeting: string;

    constructor(message: string) {
        this.greeting = message;
    }
    greet(): string {
        return this.greeting;
    }
};

let greeter = new Greeter("Hello, world!");
console.log(greeter.greet());

Ici, nous avons une classe, Greeter, qui a un constructor et une méthode greet. Nous pouvons construire une instance de la classe en utilisant le mot clé new et transmettre une chaîne que nous voulons que la méthode greet affiche sur la console. L’instance de notre classe Greeter est stockée dans la variable greeter que nous appelons ensuite la méthode greet.

TypeScript REPL dans Node.js #

Pour utiliser TypeScript REPL dans Node.js, vous pouvez utiliser [paquet tsun][1]

Installez-le globalement avec

npm install -g tsun

et exécutez dans votre terminal ou invite de commande avec la commande tsun

Exemple d’utilisation :

$ tsun
TSUN : TypeScript Upgraded Node
type in TypeScript expression to evaluate
type :help for commands in repl
$ function multiply(x, y) {
..return x * y;
..}
undefined
$ multiply(3, 4)
12

[1] : https://www.npmjs.com/package/tsun

Exécution de TypeScript à l’aide de ts-node #

[ts-node][1] est un package npm qui permet à l’utilisateur d’exécuter directement des fichiers dactylographiés, sans avoir besoin de précompilation à l’aide de tsc. Il fournit également [REPL][2].

Installez ts-node globalement en utilisant

npm install -g ts-node

ts-node n’inclut pas le compilateur TypeScript, vous devrez donc peut-être l’installer.

npm install -g typescript

Exécution du script

Pour exécuter un script nommé main.ts, exécutez

ts-node main.ts

// main.ts
console.log("Hello world");

Exemple d’utilisation

$ ts-node main.ts
Hello world

Repl en cours d’exécution

Pour exécuter REPL, exécutez la commande ts-node

Exemple d’utilisation

$ ts-node
> const sum = (a, b): number => a + b;
undefined
> sum(2, 2)
4
> .exit

Pour quitter REPL, utilisez la commande .exit ou appuyez deux fois sur CTRL + C.

[1] : https://www.npmjs.com/package/ts-node [2] : https://en.wikipedia.org/wiki/Read%E2%80%93eval%E2%80%93print_loop
