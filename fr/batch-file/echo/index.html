echo peut être utilisé pour contrôler et produire une sortie.

Syntaxe #

  • ÉCHO [MARCHE | À L’ARRÊT]
  • Message ÉCHO
  • ÉCHO(message
  • ÉCHO(

Paramètres #

| Paramètre | Détails | | ——— | ————————————————– —– | | SUR | DÉSACTIVÉ | Peut être ON ou OFF (insensible à la casse) | | message | Toute chaîne (sauf ON ou OFF lorsqu’elle est utilisée sans () |

  • echo. affichera également une chaîne vide. Cependant, c’est plus lent que echo( car echo. recherchera un fichier nommé “echo”. La commande ne fonctionnera que si ce fichier n’existe pas, mais cette vérification la ralentit.
  • echo: se comportera exactement comme echo(, sauf si message ressemble à un chemin de fichier, par exemple echo:foo\..\test.bat. Dans ce cas, l’interpréteur verra echo:foo comme nom de dossier, supprimez echo:foo\..\ (car il semble juste entrer dans le répertoire echo:foo puis le quitter à nouveau) puis exécutez test.bat, ce qui n’est pas le comportement souhaité.

Affichage des messages #

Pour afficher “Some Text”, utilisez la commande :

echo Some Text

Cela affichera la chaîne “Some Text” suivie d’une nouvelle ligne.

Pour afficher les chaînes On et Off (insensible à la casse) ou la chaîne vide, utilisez un ( au lieu d’un espace :

echo(ON
echo(
echo(off

Cela affichera :

ACTIVÉ

éteint

Il est également courant d’utiliser echo. pour afficher une ligne vide, mais veuillez consulter les remarques pour savoir pourquoi ce n’est pas la meilleure idée.

Pour afficher du texte sans inclure de CR/LF, utilisez la commande suivante :

<nul set/p=Some Text

Cette commande tentera de définir la variable appelée la chaîne vide sur l’entrée utilisateur à la suite d’une invite. Le fichier nul est redirigé vers la commande avec <nul, donc la commande abandonnera dès qu’elle essaiera de le lire et seule la chaîne d’invite sera laissée. Étant donné que l’utilisateur n’a jamais tapé une nouvelle ligne, il n’y a pas de saut de ligne.

Sortie d’écho vers le fichier #

Façons de créer un fichier avec la commande echo :

echo. > example.bat (creates an empty file called "example.bat")

echo message > example.bat (creates example.bat containing "message")
echo message >> example.bat (adds "message" to a new line in example.bat)
(echo message) >> example.bat (same as above, just another way to write it)

Sortie vers le chemin

Un petit problème que vous pourriez rencontrer en faisant ceci :

echo Hello how are you? > C:\Users\Ben Tearzz\Desktop\example.bat

(This will NOT make a file on the Desktop, and might show an error message)

Mais alors comment fait-on ? Eh bien, c’est en fait extrêmement simple … Lorsque vous tapez un chemin ou un nom de fichier dont le nom contient un espace, n’oubliez pas d’utiliser des “guillemets”

echo Hello how are you? > "C:\Users\Ben Tearzz\Desktop\example.bat"
(This will make a file on MY Desktop)

Mais que se passe-t-il si vous voulez créer un fichier qui génère un nouveau fichier ?

echo message > file1.bat > example.bat

(This is NOT going to output:
"message > file1.bat" to example.bat

Alors, comment faisons-nous cela?

echo message ^> file1.bat > example.bat

(This will output: 
"message > file1.bat" to example.bat

Il en va de même pour d’autres choses en lot

Si vous n’avez pas appris ce que sont les variables et les instructions, vous ne comprendrez probablement pas ce qui suit : [cliquez ici pour en savoir plus sur les variables][1] | [cliquez ici pour en savoir plus sur les instructions “if”][2]

Variable :

set example="text"
echo %example% > file.bat
(This will output "text" to file.bat)

si nous ne voulons pas qu’il affiche du “texte” mais simplement %example% alors écrivez :

echo ^%example^% > file.bat
(This will output "%example%" to file.bat)

déclarations d’autre:

else = ||
if ^%example^%=="Hello" echo True || echo False > file.bat

(This will output:
if %example%=="Hello" echo True

pour sortir toute la ligne on écrit :

if ^%example^%=="Hello" echo True ^|^| echo False > file.bat

This will output:
if %example%=="Hello" echo True || echo False

Si la variable est égale à “Hello” alors elle dira “True”, sinon elle dira “False”

[1] : https://www.wikiod.com/fr/batch-file/variables-dans-les-fichiers-batch [2] : https://www.wikiod.com/fr/batch-file/si-les-declarations

@Écho off #

@echo off empêche l’affichage de l’invite et du contenu du fichier de commandes, de sorte que seule la sortie est visible. Le @ masque également la sortie de la commande echo off.

Réglage de l’écho #

Le paramètre d’écho détermine si l’écho des commandes est activé ou désactivé. Voici à quoi ressemble un exemple de programme avec la commande echoing on (par défaut) :

C:\Windows\System32>echo Hello, World!
Hello, World!

C:\Windows\System32>where explorer
C:\Windows\System32\explorer.exe

C:\Windows\System32>exit

Voici à quoi cela ressemble avec echo off :

Hello, World!
C:\Windows\System32\explorer.exe

Obtention et mise en place #

Pour obtenir (afficher) le paramètre d’écho, utilisez “echo” sans paramètre :

> echo
ECHO is on.

Pour définir le paramètre d’écho, utilisez echo avec on ou off :

> echo off

> echo
ECHO is off.

> echo on

> echo
ECHO is on.

Notez qu’avec ces exemples, l’invite a été représentée par > . Lors de la modification du paramètre d’écho, l’invite apparaîtra et disparaîtra, mais cela donne des exemples peu clairs.

Notez qu’il est possible d’empêcher l’écho d’une commande même lorsque l’écho est activé, en plaçant un caractère @ avant la commande.

Echo affiche tout littéralement #

Les devis seront affichés tels quels :

echo "Some Text"

“Quelque texte”

Les jetons de commentaire sont ignorés :

echo Hello World REM this is not a comment because it is being echoed!

Hello World REM ceci n’est pas un commentaire car il est en écho !

Cependant, echo affichera toujours des variables - voir [Variable Usage] (https://www.wikiod.com/fr/batch-file/variables-dans-les-fichiers-batch#Usage) - et des caractères spéciaux toujours en vigueur :

echo hello && echo world

bonjour monde

Activer l’écho à l’intérieur des crochets #

Dans l’exemple suivant, “echo on” prendra effet une fois que la fin du contexte entre parenthèses sera atteinte :

@echo off
(
    echo on
    echo ##
)
echo $$

Pour “activer” l’écho dans un contexte entre crochets (y compris les commandes FOR et IF), vous pouvez utiliser [FOR /f macro][1] :

@echo off
setlocal

:: echo on macro should followed by the command you want to execute with echo turned on
set "echo_on=echo on&for %%. in (.) do"

(
  %echo_on% echo ###
)
