Hybrides batch et VBS

  • Exécuter VBS avec des fichiers temporaires
  • Incorporer le code vbscript dans le fichier batch sans utiliser de fichiers temporaires

Sur cette page

  • Exécuter VBS avec des fichiers temporaires
  • Incorporer le code vbscript dans le fichier batch sans utiliser de fichiers temporaires

Batch est capable de fonctionner avec la fonctionnalité VBS, ce qui augmente encore sa fiabilité. Par exemple, VBS peut gérer les décimales, les espaces et certaines autres opérations avancées qui ne peuvent pas être effectuées dans batch. Est également capable de travailler avec des objets WMI et ActiveX.

Exécuter VBS avec des fichiers temporaires #

La méthode à l’ancienne pour exécuter un autre script à partir de batch consiste à faire écho au script dans un autre emplacement, puis à l’exécuter.

Cette méthode peut être représentée comme ceci :

@echo off
rem VBS below
    echo your vbs > TempVBS.vbs
    echo other vbs>>TempVBS.vbs
rem End of VBS

cscript //nologo TempVBS.vbs
del /f /s /q TempVBS.vbs

La méthode ci-dessus nécessiterait beaucoup de echo (vbs) >> TempVBS.vbs, alors voici un moyen de la raccourcir. (codé par Aacini)

@echo off
setlocal

rem Get the number of the "<resource>" line
for /F "delims=:" %%a in ('findstr /N "<resource>" "%~F0"') do set "start=%%a"

rem Skip such number of lines and show the rest of this file
(for /F "usebackq skip=%start% delims=" %%a in ("%~F0") do echo %%a) > Program.vbs

cscript //nologo Program.vbs
del /f /s /q Program.vbs
exit /b

<resource>
your vbs
another line of vbs

La dernière méthode consiste à utiliser streams. Un fichier peut avoir quelques flux. Et chaque flux peut contenir des informations différentes.

@echo off

    echo vbs >%0:stream1
    rem This command redirect "vbs" into the stream1 of this script, then we can call it later

cscript %0:stream1 //nologo
   rem if you like, you can clear the stream1 of this file by:
   type nul>%0:stream1

Incorporer le code vbscript dans le fichier batch sans utiliser de fichiers temporaires #

Voici un exemple avec la technique(hack) inventée par l’utilisateur des forums [dostips][1] Liviu :

@echo off
echo Printed by CMD.EXE
cscript //nologo "%~f0?.wsf" //job:JS //job:VBS

exit /b %errorlevel%

----END OF BATCH CODE---
<package>
  <job id="JS"> 
    <script language="VBScript">
    
        WScript.Echo("Printed by VBScript"):
        
    </script>
  </job>
  <job id="VBS"> 
    <script language="JScript">
    
        WScript.Echo("Printed by JScript");
        
    </script>
  </job>
</package>

Comme l’exécution du fichier wsf avec [windows script host][2] est sensible à l’extension, vous pouvez exécuter un fichier avec n’importe quelle extension en ajoutant ?.wsf à la fin du fichier (qui est le cœur du hack). Alors que l’exemple de Liviu est probablement plus robuste, le code ci-dessus est une version plus simplifiée. Comme wsh ne se soucie pas beaucoup des choses en dehors du nœud <package>, vous n’êtes pas obligé de tout mettre dans les commentaires xml. Bien qu’il faille faire attention aux symboles de redirection (< et >)

[1] : http://www.dostips.com/forum/viewtopic.php?t=5543 [2] : https://en.wikipedia.org/wiki/Windows_Script_File
