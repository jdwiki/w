Substitution de variables #

Contrairement à d’autres langages de programmation, dans un fichier batch, une variable est remplacée par sa valeur réelle ** avant ** que le script batch ne soit exécuté. En d’autres termes, la substitution est effectuée lorsque le script est lu en mémoire par le processeur de commandes, et non lorsque le script est exécuté ultérieurement.

Cela permet l’utilisation de variables comme commandes dans le script, et dans le cadre d’autres noms de variables dans le script, etc. Le “script” dans ce contexte étant une ligne - ou un bloc - de code, entouré de crochets : () .

Mais ce comportement signifie que vous ne pouvez pas modifier la valeur d’une variable à l’intérieur d’un bloc !

SET VAR=Hello
FOR /L %%a in (1,1,2) do (
    ECHO %VAR%
    SET VAR=Goodbye
)

va imprimer

Hello
Hello

puisque (comme vous le voyez, lorsque vous regardez le script s’exécuter dans la fenêtre de commande), il est évalué à :

SET VAR=Hello
FOR /L %%a in (1,1,2) do (
    echo Hello
    SET VAR=Goodbye
)

Dans l’exemple ci-dessus, la commande ECHO est évaluée comme Hello lorsque le script est lu en mémoire, donc le script affichera Hello pour toujours, quel que soit le nombre de passages effectués dans le script.

La façon d’obtenir le comportement variable plus “traditionnel” (de la variable étant développée pendant que le script est en cours d’exécution) est d’activer “l’expansion retardée”. Cela implique d’ajouter cette commande dans le script avant l’instruction de boucle (généralement une boucle FOR, dans un script batch) et d’utiliser un point d’exclamation (!) au lieu d’un signe de pourcentage (%) dans le nom de la variable :

setlocal enabledelayedexpansion 
SET VAR=Hello
FOR /L %%a in (1,1,2) do (
    echo !VAR!
    SET VAR=Goodbye
)
endlocal

va imprimer

Hello
Goodbye

La syntaxe %%a in (1,1,2) provoque l’exécution de la boucle 2 fois : à la première occasion, la variable porte sa valeur initiale de ‘Hello’, mais au second passage dans la boucle - après avoir exécuté la deuxième instruction SET comme dernière action de la 1ère passe - elle a été remplacée par la valeur révisée ‘Au revoir’.

Substitution de variable avancée

Maintenant, une technique avancée. L’utilisation de la commande CALL permet au processeur de commandes par lots de développer une variable située sur la même ligne du script. Cela peut fournir une expansion à plusieurs niveaux, par l’utilisation répétée d’appels et de modificateurs.

Ceci est utile, par exemple, dans une boucle FOR. Comme dans l’exemple suivant, où nous avons une liste numérotée de variables :

"c:\MesFichiers\test1.txt" "c:\MesFichiers\test2.txt" "c:\MesFichiers\test3.txt"

Nous pouvons y parvenir en utilisant la boucle FOR suivante :

setlocal enabledelayedexpansion
for %%x in (%*) do (
    set /a "i+=1"
    call set path!i!=%%~!i!
    call echo %%path!i!%%
)
endlocal

Production:

c:\MyFiles\test1.txt
c:\MyFiles\test2.txt
c:\MyFiles\test3.txt

Notez que la variable !i! est d’abord étendue à sa valeur initiale, 1, puis la variable résultante, %1, est étendue à sa valeur réelle de c:\MyFiles\test1.txt. C’est la double expansion de la variable i. Sur la ligne suivante, i est à nouveau développé deux fois, en utilisant la commande CALL ECHO avec le préfixe variable %%, puis imprimé à l’écran (c’est-à-dire affiché à l’écran).

A chaque passage successif dans la boucle, le nombre initial est augmenté de 1 (du fait du code ‘i+=1’). Ainsi, il passe à ‘2’ au 2ème passage dans la boucle et à ‘3’ au 3ème passage. Ainsi, la chaîne renvoyée à l’écran se modifie à chaque passage.

Déclaration #

Pour créer une variable simple et l’affecter à une valeur ou à une chaîne, utilisez la commande “SET” :

SET var=10

Ici, le code déclare une nouvelle variable var avec une valeur de 10. Par défaut, toutes les variables sont stockées en interne sous forme de chaînes ; cela signifie que la valeur 10 n’est pas différente de foo1234 ou Hello, World!

Notes sur les guillemets #

Les guillemets utilisés seront inclus dans la valeur de la variable :

SET var="new value"             <-- %var% == '"new value"'

Espaces dans les variables #

Le langage batch considère les espaces comme des parties acceptables des noms de variables. Par exemple, set var = 10 donnera une variable appelée var qui contient la valeur 10 (notez l’espace supplémentaire à droite de var et à gauche de dix).

Utiliser des guillemets pour éliminer les espaces #

Afin d’éviter les espaces, utilisez des guillemets autour de l’intégralité du devoir ; le nom et la valeur de la variable. Cela empêche également les espaces de fin accidentels à la fin de la ligne (le caractère ␣ désigne un espace):

SET␣var=my␣new␣value␣           <-- '%var%' == 'my new value '
SET␣"var=my␣new␣value"␣         <-- '%var%' == 'my new value'

Utilisez également des guillemets lorsque vous joignez plusieurs instructions avec & ou | - sinon, placez le symbole directement après la fin de la valeur de la variable :

SET var=val & goto :next        <-- '%var%' == 'val '
SET "var=val" & goto :next      <-- '%var%' == 'val'
SET var=val& goto :next         <-- '%var%' == 'val'

Déclarer plusieurs variables #

Lorsque plusieurs variables sont définies au début du lot, une forme de définition courte peut être utilisée en employant une chaîne [remplacement][1].

@echo off
set "vars=_A=good,_B=,_E=bad,_F=,_G=ugly,_C=,_H=,_I=,_J=,_K=,_L=,_D=6
set "%vars:,=" & set "%"

for /f %%l in ('set _') do echo %%l
exit /b

_A=good
_D=6
_E=bad
_G=ugly

Notez que dans l’exemple ci-dessus, les variables sont nativement triées par ordre alphabétique, lorsqu’elles sont imprimées à l’écran.

[1] : http://ss64.com/nt/syntax-replace.html

Utilisation #

echo %var%

Ce code fera écho à la valeur de var

Si setLocal EnableDelayedExpansion est utilisé, ce qui suit renverra la valeur de var (l’expression standard %var% ne fonctionnera pas dans ce contexte).

echo !var!

Dans les fichiers de commandes, les variables peuvent être utilisées dans n’importe quel contexte, y compris en tant que parties de commandes ou parties d’autres variables. Vous ne pouvez pas appeler une variable avant de la définir.

Utiliser des variables comme commandes :

set var=echo
%var% This will be echoed

Utiliser des variables dans d’autres variables :

set var=part1
set %var%part2=Hello
echo %part1part2%

Utilisation d’une variable comme tableau #

Il est possible de créer un ensemble de variables pouvant agir de la même manière qu’un tableau (bien qu’il ne s’agisse pas d’un objet tableau réel) en utilisant des espaces dans l’instruction SET :

@echo off
SET var=A "foo bar" 123
for %%a in (%var%) do (
    echo %%a
)
echo Get the variable directly: %var%

Résultat:

A
"foo bar"
123
Get the variable directly: A "foo bar" 123

Il est également possible de déclarer votre variable à l’aide d’index afin de récupérer des informations spécifiques. Cela créera plusieurs variables, avec l’illusion d’un tableau :

@echo off
setlocal enabledelayedexpansion
SET var[0]=A
SET var[1]=foo bar
SET var[2]=123
for %%a in (0,1,2) do (
    echo !var[%%a]!
)
echo Get one of the variables directly: %var[1]%

Résultat:

A
foo bar
123
Get one of the variables directly: foo bar

Notez que dans l’exemple ci-dessus, vous ne pouvez pas référencer var sans indiquer quel est l’index souhaité, car var n’existe pas en soi. Cet exemple utilise également setlocal enabledelayedexpansion en conjonction avec les points d’exclamation à !var[%%a]!. Vous pouvez afficher plus d’informations à ce sujet dans la [Documentation sur la portée de la substitution de variable] [1].

[1] : https://www.wikiod.com/fr/batch-file/variables-dans-les-fichiers-batch#Substitution variable

Définir des variables à partir d’une entrée #

En utilisant le commutateur /p avec la commande SET, vous pouvez définir des variables à partir d’une entrée.

Cette entrée peut être une entrée utilisateur (clavier) :

echo Enter your name : 
set /p name=
echo Your name is %name%

Qui peut être simplifié comme ceci :

set /p name=Enter your name :
echo Your name is %name%

Ou vous pouvez obtenir l’entrée d’un fichier :

set /p name=< file.txt

dans ce cas, vous obtiendrez la valeur de la première ligne de file.txt

Obtenir la valeur de différentes lignes dans un fichier :

(
   set /p line1=
   set /p line2=
   set /p line3=

) < file.txt

Opérations sur les variables #

set var=10
set /a var=%var%+10
echo %var%

La valeur finale de var est 20.

La deuxième ligne ne fonctionne pas dans un bloc de commande utilisé par exemple sur une condition IF ou sur une boucle FOR car une expansion retardée serait nécessaire au lieu d’une expansion de variable d’environnement standard.

Voici une autre meilleure façon de travailler également dans un bloc de commande :

set var=10
set /A var+=10
echo %var%

L’environnement d’invite de commande prend en charge les valeurs entières 32 bits signées :

  • addition + et +=
  • soustraction - et -=
  • multiplication * et *=
  • division / et /=
  • division de module % et %=
  • AND au niveau du bit &
  • OR au niveau du bit |
  • au niveau du bit PAS ~
  • XOR au niveau du bit ^
  • décalage au niveau du bit vers la gauche <<
  • décalage au niveau du bit vers la droite >>
  • NON logique ‘!’
  • unaire moins -
  • regroupement avec ( et )

L’interpréteur de commandes Windows ne prend pas en charge les valeurs entières 64 bits ou les valeurs à virgule flottante dans les expressions arithmétiques.

Remarque : L’opérateur % doit être écrit dans un fichier de commandes sous la forme %% pour être interprété comme un opérateur.

Dans une fenêtre d’invite de commande, l’exécution de la ligne de commande set /A Value=8 % 3 attribue la valeur 2 à la variable d’environnement Value et génère en outre 2.

Dans un fichier batch, il faut écrire set /A Value=8 %% 3 pour attribuer la valeur 2 à la variable d’environnement Value et rien n’est sorti respectivement écrit pour gérer STDOUT (sortie standard). Une ligne set /A Value=8 % 3 dans un fichier batch entraînerait le message d’erreur Missing operator lors de l’exécution du fichier batch.

L’environnement requiert le commutateur /A pour les opérations arithmétiques uniquement, pas pour les variables de chaîne ordinaires.

Chaque chaîne dans l’expression arithmétique après set /A, qu’il s’agisse d’un nombre ou d’un opérateur, est automatiquement interprétée comme le nom d’une variable d’environnement.

Pour cette raison, référencer la valeur d’une variable avec %variable% ou avec !variable! n’est pas nécessaire lorsque le nom de la variable se compose uniquement de caractères de mot (0-9A-Za-z_) avec le premier caractère n’étant pas un chiffre ce qui est particulièrement utile dans un bloc de commande commençant par ( et se terminant par un ) correspondant.

Les nombres sont convertis de chaîne en entier avec la fonction C/C++ strtol avec “base” étant zéro, ce qui signifie une détermination automatique de la base qui peut facilement entraîner des imprévus résultats.

Exemple:

set Divided=11
set Divisor=3

set /A Quotient=Divided / Divisor
set /A Remainder=Divided %% Divisor

echo %Divided% / %Divisor% = %Quotient%
echo %Divided% %% %Divisor% = %Remainder%

set HexValue1=0x14
set HexValue2=0x0A
set /A Result=(HexValue1 + HexValue2) * -3

echo (%HexValue1% + %HexValue2%) * -3 = (20 + 10) * -3 = %Result%

set /A Result%%=7
echo -90 %%= 7 = %Result%

set OctalValue=020
set DecimalValue=12
set /A Result=OctalValue - DecimalValue

echo %OctalValue% - %DecimalValue% = 16 - 12 = %Result%

La sortie de cet exemple est :

11 / 3 = 3
11 % 3 = 2
(0x14 + 0x0A) * -3 = (20 + 10) * -3 = -90
-90 %= 7 = -6
020 - 12 = 16 - 12 = 4

Les variables non définies lors de l’évaluation de l’expression arithmétique sont remplacées par la valeur 0.
