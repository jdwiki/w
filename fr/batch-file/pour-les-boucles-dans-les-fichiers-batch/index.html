Syntaxe #

  • pour /l %%p dans (startNumber, incrément, endNumber) faites la commande
  • for /f %%p in (filename) do command
  • pour /f %%p in (“textStrings”) faites la commande
  • for /f %%p in (‘command’) do command
  • for /r drive:\path %%p in (set) do command
  • pour /d %%p dans (répertoire) faire la commande

La commande for accepte les options lorsque le drapeau /f est utilisé. Voici une liste d’options pouvant être utilisées :

  • delims=x Caractère(s) délimiteur(s) pour séparer les jetons

  • skip=n Nombre de lignes à ignorer au début du fichier et des chaînes de texte

  • eol=; Caractère au début de chaque ligne pour indiquer un commentaire

  • tokens=n Éléments numérotés à lire à partir de chaque ligne ou chaîne à traiter

  • usebackq Utiliser un autre style de guillemets :

    Use double quotes for long file names in “files”

    Use single quotes for ’textStrings’

    Use back quotes for `command`

Visite récursive des répertoires dans une arborescence de répertoires #

La commande for /r peut être utilisée pour visiter de manière récursive tous les répertoires d’une arborescence de répertoires et exécuter une commande.

@echo off
rem start at the top of the tree to visit and loop though each directory
for /r %%a in (.) do (
  rem enter the directory
  pushd %%a
  echo In directory:
  cd
  rem leave the directory
  popd
  )

Remarques:

  • for /r - Boucle dans les fichiers (sous-dossiers Recurse).
  • pushd - Modifiez le répertoire/dossier actuel et stockez le dossier/chemin précédent pour une utilisation par la commande POPD.
  • popd - Remplacez le répertoire par le chemin/dossier le plus récemment stocké par la commande PUSHD.

Boucle sur chaque ligne d’un ensemble de fichiers #

Ce qui suit fera écho à chaque ligne du fichier C:\scripts\testFile.txt. Les lignes vides ne seront pas traitées.

for /F "tokens=*" %%A in (C:\scripts\testFile.txt) do (
  echo %%A
  rem do other stuff here
  )

Un exemple plus avancé montre comment les données dérivées d’une boucle FOR à partir d’un ensemble de fichiers restreints peuvent être utilisées pour rediriger l’exécution par lots, tout en enregistrant le contenu recherché dans un fichier :

@echo off
setlocal enabledelayedexpansion

for /f %%i in ('dir "%temp%\test*.log" /o:-d /t:w /b') do (
    set "last=%temp%\%%i"
    type !last! | find /n /i "Completed" >nul 2>&1 >> %temp%\Completed.log ^
     && (echo Found in log %%i & goto :end) || (echo Not found in log %%i & set "result=1"))

:: add user tasks code here
if defined result echo Performing user tasks...

:end    
echo All tasks completed
exit /b

Notez combien de temps les chaînes de commande sont divisées en plusieurs lignes de code et les groupes de commandes sont séparés par des parenthèses.

Renommer tous les fichiers du répertoire courant #

Ce qui suit utilise une variable avec une boucle for pour renommer un groupe de fichiers.

SetLocal EnableDelayedExpansion

for %%j in (*.*) do (
  set filename=%%~nj
  set filename=!filename:old=new!
  set filename=Prefix !filename!
  set filename=!filename! Suffix
  ren "%%j" "!filename!%%~xj"
  )

En définissant le nom de la variable %%j et en l’associant à tous les fichiers courants (*.*), nous pouvons utiliser la variable dans une boucle for pour représenter chaque fichier dans le répertoire courant.

Chaque itération (ou passage) dans la boucle traite ainsi un fichier différent du groupe défini (qui pourrait également être n’importe quel groupe, par exemple *.jpg ou *.txt).

Dans le premier exemple, nous remplaçons text : la chaîne de texte “old” est remplacée par la chaîne de texte “new” (si, mais seulement si, le texte “old” est présent dans le nom du fichier).

Dans le deuxième exemple, nous ajoutons du texte : le texte “Prefix " est ajouté au début du nom de fichier. Ceci n’est pas une substitution. Cette modification sera appliquée à tous les fichiers du groupe.

Dans le troisième exemple, nous ajoutons à nouveau du texte : le texte " Suffixe” est ajouté à la fin du nom de fichier. Encore une fois, ce n’est pas une substitution. Cette modification sera appliquée à tous les fichiers du groupe.

La dernière ligne gère en fait le changement de nom.

Itération #

for /L %%A in (1,2,40) do echo %%A

Cette ligne itérera de 1 à 39, augmentant de 2 à chaque fois.

Le premier paramètre, “1”, est le numéro de départ.

Le deuxième paramètre, “2”, est l’incrément.

Le troisième paramètre, “40”, est le maximum.
