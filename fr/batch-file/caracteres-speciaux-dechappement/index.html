Dans toutes les versions cmd.exe et DOS, certains caractères sont réservés à un usage spécifique (par exemple, la redirection de commande). Cette rubrique explique comment utiliser les caractères spéciaux sans problème.

Échappez en utilisant caret(^) #

La plupart des caractères spéciaux peuvent être échappés à l’aide du caret(^). Jetez un oeil à l’exemple suivant.

echo > Hi
echo ^> Hi

Cette première commande n’affichera pas > Hi car > est un caractère spécial, ce qui signifie rediriger la sortie vers un fichier. Dans ce cas, le fichier est nommé “Salut”

Cependant, dans la deuxième commande, > Hi serait affiché sans aucun problème car le caret ( ^) indique au > d’arrêter de fonctionner en tant que commande “rediriger la sortie vers le fichier”, maintenant > est juste un caractère normal .

Voici une liste de caractères spéciaux qui peuvent être échappés (pris et modifiés à partir de la page de Rob van der Woude)

Caractère Résultat échappé Remarques
^ ^^
& ^&
< ^<
> ^>
| ^|
\ ^\
! ^^ ! Requis uniquement lorsque DelayedExpansion est activé

Echapper au caret #

Les carets peuvent être empilés pour échapper aux autres carets, considérons l’exemple suivant.

Entrée Sortie
^& &
^^^& ^&
^^^^^& ^^&

Remarque : Les carets en gras sont échappés.

Problème de sécurité #

Un peu hors sujet ici, mais c’est très important! Un échappement indésirable à la fin du fichier pourrait provoquer une fuite de mémoire !

any-invalid-command-you-like-here ^

Cette commande entraînerait une fuite de toute la mémoire, rendant le système complètement inutilisable ! Voir [ici][1] pour plus d’informations.

[1] : https://stackoverflow.com/questions/15466298/simple-caret-at-end-of-windows-batch-file-consumes-all-memory

FIND et FINDSTR Caractères spéciaux #

Dans find et findstr, certains caractères spéciaux nécessitent une certaine prudence.

TROUVER #

Il n’y a qu’un seul caractère à échapper - " guillemet. Pour l’échapper, ajoutez simplement un autre guillemet à côté. Ainsi " devient "". Assez simple.

FINDSTR #

Findstr est livré avec de nombreux caractères à échapper, alors soyez très prudent. En utilisant \, nous pouvons échapper les caractères spéciaux. Voici une liste de caractères spéciaux à échapper

Caractère Résultat échappé
\ \\
[ \[
] \]
" \"
. \.
* \*
? \ ?

FOR /F Caractères spéciaux #

POUR /F #

Dans une instruction FOR /F, certains caractères doivent être échappés, ici une liste (extraite et modifiée de la page de Rob van der Woude)

Caractère Résultat échappé Remarques
' ^' Nécessaire uniquement entre parenthèses FOR /F, sauf si usebackq est spécifié.
` ^` Nécessaire uniquement entre parenthèses FOR /F, lorsque usebackq est spécifié
, ^, ┒
; ^ ; ┃
= ^= ┣ Doit être échappé entre parenthèses FOR /F, même s’il est entre guillemets
( ^( ┃
) ^) ┙

Caractères spéciaux supplémentaires #

Voici une liste d’autre(s) caractère(s) spécial(s), qui nécessite(nt)/peut avoir besoin d’être échappé, mais qui n’est pas mentionné ci-dessus.

Caractère Résultat échappé Remarques
% %%
[LF] ^[LF] Cette astuce est mentionnée par Mark Stang dans le groupe de nouvelles alt.msdos.batch.

S’échapper par le pipeline #

Lorsqu’il y a une expression avec un tube, le cmd démarre deux threads des deux côtés du tube et l’expression est analysée deux fois (pour chaque côté du tube), les carets doivent donc être doublés.

Sur le côté gauche:

echo ^^^&|more

Sur le côté droit:

break|echo ^^^&
