Vous pouvez ajouter des variables de départ à la fonction en ajoutant <paramètre> à son étiquette. Ces variables de départ sont accessibles avec %n où n est le numéro de la variable de départ (%1 pour la première, %2 pour la seconde. Cette méthode %n fonctionne pour %1 - %9. Pour paramètre 10 - 255, vous devrez utiliser la commande [Shift][1]). Par exemple:

:function <var1> <var2>

Une fois que vous avez utilisé call :function param1 param2, param1 est accessible avec %1 et param2 avec %2. Remarque : le <paramètre> n’est pas strictement nécessaire, mais il aide à la lisibilité.

Une astuce intéressante qui est utile lorsque de nombreuses variables volent est d’utiliser setlocal et endlocal en tandem avec %n. setlocal et endlocal font essentiellement de la fonction sa propre instance distincte de l’invite de commande, les variables qui y sont définies ne restent que lorsqu’elles sont dans le cadre.

Si vous utilisez setlocal et endlocal, et que vous renvoyez des valeurs globales, utilisez ceci.

endlocal & set var=variable

Cela définit la valeur globale var sur variable. Vous pouvez les enchaîner pour plusieurs variables.

endlocal & set var=variable & set var2=variable number 2

Cela définit la variable globale var sur variable et la valeur globale var2 sur variable number 2. Étant donné que le code dans les blocs de code est également exécuté simultanément, vous pouvez également le faire.

if "%var%"=="" (
    endlocal
    set %~2=10
)

Mais, vous __ ne pouvez pas __ le faire.

if "%var%"=="" (
    set %~2=10
    endlocal
)

[1] : https://www.wikiod.com/fr/batch-file/arguments-de-ligne-de-commande-du-fichier-batch

Fonctions anonymes dans les fichiers batch #

La technique des fonctions [anonymes] [1] utilise le fait que la commande CALL utilise en interne GOTO lorsque le sous-programme est appelé et abuse de l’impression des messages d’aide [avec une double expansion variable] [2] :

@echo off
setlocal 
set "anonymous=/?"

call :%%anonymous%% a b c 3>&1 >nul

if "%0" == ":%anonymous%" (
  echo(
  echo Anonymous call:
  echo %%1=%1 %%2=%2 %%3=%3
  exit /b 0
)>&3

Vous ne pouvez appeler une fonction anonyme que si elle est définie après le CALL (ou après avoir terminé le contexte des parenthèses si le CALL est exécuté entre parenthèses). Il ne peut pas être appelé à partir d’un [script extérieur][3] , mais il s’agit d’un appel de fonction plus lent que la normale.

[1] : http://www.dostips.com/forum/viewtopic.php?f=3&t=3803&start=15#p42446 [2] : https://stackoverflow.com/questions/31987023/why-call-prints-the-goto-help-message-in-this-scriptand-why-command-after-that [3] : https://stackoverflow.com/a/30170342/388389

Fonction simple #

call :FunctionX
rem More code...

:FunctionX
rem Some code here.
goto :eof

C’est une fonction très simple. Les fonctions sont des commandes intégrées au programme qui exécutent plusieurs commandes à la fois. Les fonctions sont créées en créant une étiquette et en y mettant du code, et une fois que c’est fait, vous ajoutez un goto :eof ou exit /b <ErrorlevelYou'dLike> qui retourne là où il a été invoqué. Les fonctions sont appelées avec call :functionname adparams.

Fonction avec paramètres #

call :tohex 14 result
rem More code...

:tohex <innum> <outvar>
set dec=%1
set outvar=%~2
rem %n and %~n are functionally identical, but %~n is slightly safer.
goto :eof

Cela prend les paramètres supplémentaires de call comme si la fonction était un fichier Batch séparé. Remarque : le <paramètre> n’est pas nécessaire, mais il aide à la lisibilité.

Fonction utilisant setlocal et endlocal #

set var1=123456789
set var2=abcdef
call :specialvars
echo %var1%, %var2%
rem More code...

:specialvars
setlocal
set var1=987654321
set var2=fedcba
endlocal
goto :eof

À l’intérieur de la section setlocal , section endlocal, les variables sont séparées des variables de l’appelant, d’où la raison pour laquelle %var1% et %var2% n’ont pas été modifiés.

Les combiner tous #

set importantvar=importantstuff
call :stuff 123 var1
rem More code...

:stuff <arg1> <arg2>
setlocal
set importantvar=%~1
echo Writing some stuff into %~2!
endlocal
set %~2=some stuff
setlocal
set importantvar=junk
endlocal
goto :eof

Cela utilise la fonction de base, setlocal et endlocal et des arguments pour créer une petite fonction étrange.

Appel de fonctions depuis un autre fichier batch #

Prenons le fichier suivant appelé library.cmd :

@echo off

echo -/-/- Batch Functions Library -/-/-

:function1
    echo argument1 - %1
    goto :eof

Pour exécuter uniquement :function1 sans le code du reste du fichier, vous devez mettre une étiquette :function1 dans l’appelant bat et l’utiliser comme ceci :

@echo off

call :function1 ###
exit /b %errorlevel%

:function1
    library.bat %*

la sortie sera (le code en dehors de la fonction dans library.cmd n’est pas exécuté):

argument1 - ###

Pour plus d’informations, consultez [ceci][1].
