AlchimieLangage

  • Appel combiné : utilisez plusieurs fonctions dans un seul appel d’API (Node.js)

Sur cette page

  • Appel combiné : utilisez plusieurs fonctions dans un seul appel d’API (Node.js)

AlchemyLanguage est une collection de méthodes d’analyse de texte qui fournissent un aperçu plus approfondi de votre texte ou de votre contenu HTML. Consultez la rubrique [Getting Started][1] pour savoir comment démarrer avec AlchemyLanguage et d’autres services Watson. Pour plus de détails et d’exemples sur AlchemyLanguage, consultez la [référence API][2] et la [documentation][3].

Limites de taille #

  • Contenu HTML avant nettoyage du texte : 600 Ko
  • Texte source, après nettoyage du texte : 50 Ko
  • Appels utilisant des modèles personnalisés : 5 Ko

Support linguistique #

Pour voir quelles langues sont prises en charge pour chaque fonction, reportez-vous à l’entrée de chaque fonction dans la [référence API][4].

Détection de la langue #

Par défaut, AlchemyLanguage détecte automatiquement la langue de votre texte source. Vous pouvez spécifier manuellement la langue de votre contenu avec le paramètre de requête language. (par exemple, langue=espagnol)

Nettoyage de texte #

Lorsque vous utilisez une fonction HTML ou URL de l’API, AlchemyLanguage nettoie le contenu pour préparer le texte source pour l’analyse. Le paramètre sourceText vous permet de personnaliser le processus de nettoyage avec les options suivantes :

  • cleaned_or_raw (par défaut) - Supprime les éléments du site Web tels que les liens, les publicités, etc. Si le nettoyage échoue, le texte brut de la page Web est utilisé
  • cleaned– Supprime les éléments du site Web tels que les liens, les publicités, etc.
  • raw - Utilise le texte brut de la page Web sans nettoyage
  • cquery – Utilise la requête de contraintes visuelles que vous spécifiez dans le paramètre cquery. Voir la [documentation][5] pour plus de détails sur les requêtes de contraintes visuelles.
  • xpath – Utilise la requête XPath que vous spécifiez dans le paramètre xpath
  • xpath_or_raw – Utilise les résultats d’une requête XPath, revenant au texte brut si la requête XPath ne renvoie rien
  • cleaned_and_xpath – Utilise les résultats d’une requête XPath sur le texte d’une page Web nettoyée

Appel combiné : utilisez plusieurs fonctions dans un seul appel d’API (Node.js) #

La méthode Combined Call vous permet d’utiliser plusieurs fonctions AlchemyLanguage dans une seule requête. Cet exemple utilise un appel combiné pour obtenir des entités et des mots-clés à partir du site Web d’IBM et renvoie des informations sur les sentiments pour chaque résultat.

Cet exemple nécessite [informations d’identification du service AlchemyLanguage][1] et [Node.js][2].

  1. Utilisez une interface de ligne de commande pour installer le [SDK Watson Developer Cloud Node.js][3] :

    $ npm install watson-developer-cloud

  2. Enregistrez le code suivant dans un fichier app.js dans le même répertoire. Assurez-vous de remplacer API_KEY par votre clé AlchemyAPI :

var AlchemyLanguageV1 = require('watson-developer-cloud/alchemy-language/v1');
var alchemy_language = AlchemyLanguageV1({
  api_key: 'API_KEY'
})

var parameters = {
  extract: 'entities,keywords',
  sentiment: 1, 
  url: 'https://www.ibm.com/us-en/'
};

alchemy_language.combined(parameters, function (err, response) {
  if (err)
    console.log('error:', err);
  else
    console.log(JSON.stringify(response, null, 2));
});

  1. Exécutez l’application :

    $ node app.js

## Analyse des sentiments : obtenez des informations sur les sentiments pour des phrases spécifiques dans le texte (Node.js) La fonction Sentiment ciblé d’AlchemyLanguage peut rechercher dans votre contenu des phrases cibles et renvoyer des informations sur le sentiment pour chaque résultat.

Cet exemple nécessite [informations d’identification du service AlchemyLanguage][1] et [Node.js][2]

  1. Utilisez une interface de ligne de commande pour installer le [SDK Watson Developer Cloud Node.js][3] :

    $ npm install watson-developer-cloud

  2. Enregistrez le code suivant dans un fichier app.js dans le même répertoire. Assurez-vous de remplacer API_KEY par votre clé AlchemyAPI :

var AlchemyLanguageV1 = require('watson-developer-cloud/alchemy-language/v1');
var alchemy_language = new AlchemyLanguageV1({
  api_key: 'API_KEY'
})

var parameters = {
  text: 'Grapes are the best! I hate peaches.',
  targets: [
    'grapes',
    'peaches'
  ]
};

alchemy_language.sentiment(parameters, function (err, response) {
  if (err)
    console.log('error:', err);
  else
    console.log(JSON.stringify(response, null, 2));
});

  1. Exécutez l’application :

    $ node app.js

## Concepts : identifiez les concepts d’une page Web (Node.js) AlchemyLanguage peut détecter les concepts généraux référencés dans votre contenu. Le service renvoie des liens [Données liées][1] pour chaque concept et une URL vers un site Web pertinent lorsque cela est possible.

Cet exemple nécessite [informations d’identification du service AlchemyLanguage][2] et [Node.js][3]

  1. Utilisez une interface de ligne de commande pour installer le [Watson Developer Cloud Node.js SDK][4] :

    $ npm install watson-developer-cloud

  2. Enregistrez le code suivant dans un fichier app.js dans le même répertoire. Assurez-vous de remplacer API_KEY par votre clé AlchemyAPI.

var AlchemyLanguageV1 = require('watson-developer-cloud/alchemy-language/v1');
var alchemy_language = new AlchemyLanguageV1({
  api_key: 'API_KEY'
})

var parameters = {
  url: 'http://www.cnn.com'
};

alchemy_language.concepts(parameters, function (err, response) {
  if (err)
    console.log('error:', err);
  else
    console.log(JSON.stringify(response, null, 2));
});

  1. Exécutez l’application :

    $ node app.js
