si, si / alors, si / sinon, à moins que, opérateur ternaire #

L’instance la plus élémentaire d’une construction “if” évalue une condition et exécute du code en fonction du résultat de la condition. Si la condition renvoie “true”, le code contenu dans la condition est exécuté.

counter = 10
if counter is 10
  console.log 'This will be executed!'

La construction if peut être enrichie d’une instruction else. Le code dans l’instruction else sera exécuté chaque fois que la condition if n’est pas remplie.

counter = 9
if counter is 10
  console.log 'This will not be executed...'
else
  console.log '... but this one will!'

Les constructions if peuvent être enchaînées en utilisant else, sans aucune limitation sur le nombre pouvant être enchaînés. La première condition qui renvoie “true” exécutera son code et arrêtera la vérification : aucune condition en dessous de ce point ne sera évaluée par la suite, et aucun bloc de code de ces conditions ne sera exécuté.

if counter is 10
  console.log 'I counted to 10'
else if counter is 9
  console.log 'I counted to 9'
else if counter < 7
  console.log 'Not to 7 yet'
else
  console.log 'I lost count'

La forme opposée de “si” est “à moins que”. Contrairement à if, unless ne s’exécutera que si la condition renvoie false.

counter = 10
unless counter is 10
  console.log 'This will not be executed!

Les instructions if peuvent être placées sur une seule ligne, mais dans ce cas, le mot-clé then est requis.

if counter is 10 then console.log 'Counter is 10'

Une syntaxe alternative est le Ruby-like :

console.log 'Counter is 10' if counter is 10

Les deux derniers blocs de code sont équivalents.

L’opérateur ternaire est une compression d’une construction “if / then / else” et peut être utilisé lors de l’attribution de valeurs à des variables. La valeur finale attribuée à la variable sera celle définie après le then lorsque la condition if est remplie. Sinon, la valeur après else sera attribuée.

outcome = if counter is 10 then 'Done counting!' else 'Still counting'

Changer #

TL ; DR : Les instructions switch de CoffeeScript utilisent when pour chaque cas et else pour le cas par défaut. Ils utilisent “then” pour les cas d’une seule ligne et des virgules pour plusieurs cas avec un seul résultat. Ils interdisent intentionnellement le fallthrough et n’ont donc pas besoin d’une “pause” explicite (puisqu’elle est toujours là implicitement). Une instruction switch peut être utilisée comme une expression retournable et assignable.

Les instructions switch de CoffeeScript sont une sorte d’instruction de contrôle qui vous permet d’effectuer différentes actions en fonction d’une valeur. Elles ressemblent aux instructions “if”, mais là où une instruction “if” effectue généralement l’une des deux actions selon que quelque chose est “vrai” ou “faux”, les instructions “switch” effectuent une action parmi n’importe quel nombre en fonction de la valeur de n’importe quelle expression - une chaîne, un nombre ou quoi que ce soit d’autre.

CoffeeScript switch commence par le mot clé switch suivi de l’expression à activer. Ensuite, chaque cas est représenté par le mot-clé “quand” suivi de la valeur de ce cas.

switch name
  when "Alice"
    # Code here will run when name is Alice
    callAlice()
  when "Bob"
    # Code here will run when name is Bob
    giveBobSandwich()

Il existe également une syntaxe abrégée pour chaque cas sur une ligne, en utilisant le mot-clé then au lieu d’un retour à la ligne :

livesLeft = 2
switch livesLeft
  when 3 then fullHealth()
  when 2 then healthAt 2
  when 1 then healthAt 1
  when 0 then playerDie()

Vous pouvez mélanger et assortir les deux formats selon vos besoins :

livesLeft = 2
switch livesLeft
  when 3 then fullHealth()
  when 2 then healthAt 2
  when 1
    healthAt 1
    alert "Warning! Health low!"
  when 0 then playerDie()

Bien que les éléments les plus courants à activer soient une variable (comme dans l’exemple précédent) ou le résultat d’une fonction, vous pouvez activer n’importe quelle expression de votre choix :

indexOfAnswer = 0
switch indexOfAnswer + 1
  when 1 then console.log "The answer is the 1st item"
  when 2 then console.log "The answer is the 2nd item"
  when 3 then console.log "The answer is the 3rd item"

Vous pouvez également avoir plusieurs cas menant à la même action :

switch password
  when "password", "123456", "letmein" then console.log "Wrong!"
  when "openpoppyseed" then console.log "Close, but no cigar."
  when "opensesame" then console.log "You got it!"

Une fonctionnalité très utile est un cas par défaut ou fourre-tout, qui ne s’exécutera que si aucun des autres critères n’est rempli. CoffeeScript le signifie avec le mot-clé else :

switch password
  when "password", "123456", "letmein" then console.log "Wrong!"
  when "openpoppyseed" then console.log "Close, but no cigar."
  when "opensesame" then console.log "You got it!"
  else console.log "Not even close..."

(Notez que vous n’avez pas besoin du mot clé then pour le cas else car il n’y a pas de condition.)

Voici maintenant un exemple de toutes les fonctionnalités de switch en action !

switch day
  when "Mon" then go work
  when "Tue" then go relax
  when "Thu" then go iceFishing
  when "Fri", "Sat"
    if day is bingoDay
      go bingo
      go dancing
  when "Sun" then go church
  else go work

Vous pouvez également faire en sorte que la condition d’un cas soit une expression :

switch fullName
  when myFullName() then alert "Doppelgänger detected"
  when presidentFirstName + " " + presidentLastName
    alert "Get down Mr. president!"
    callSecretService()
  when "Joey Bonzo" then alert "Joey Bonzo everybody"

Les instructions switch de CoffeeScript ont également un trait unique : elles peuvent renvoyer des valeurs comme une fonction. Si vous affectez une variable à une instruction switch, elle sera affectée quel que soit le retour de l’instruction.

address = switch company
  when "Apple" then "One Infinite Loop"
  when "Google" then "1600 Amphitheatre Parkway"
  when "ACME"
    if isReal
      "31918 Hayman St"
    else
      "Unknown desert location"
  else lookUpAddress company

(N’oubliez pas que la dernière instruction d’un bloc est renvoyée implicitement. Vous pouvez également utiliser le mot-clé return manuellement.)

Les instructions switch peuvent également être utilisées sans expression de contrôle, ce qui les transforme en une alternative plus propre aux chaînes if/else.

score = 76
grade = switch
  when score < 60 then 'F'
  when score < 70 then 'D'
  when score < 80 then 'C'
  when score < 90 then 'B'
  else 'A'

(Ceci est fonctionnellement équivalent à grade = switch true car le premier cas évalué à true correspondra. Cependant, puisque chaque cas implicitement breaks à la fin, seul le premier cas correspondant sera exécuté.)
