Comment créer un projet de test unitaire et des tests unitaires et comment exécuter les tests unitaires et l’outil de couverture de code.

Dans ce guide, le framework MSTest standard sera utilisé et l’outil standard d’analyse de la couverture du code disponible dans Visual Studio.

Le guide a été écrit pour Visual Studio 2015, il est donc possible que certaines choses soient différentes dans d’autres versions.

Création d’un projet de test unitaire #

  • Ouvrir le projet C#
  • Clic droit sur la solution -> Ajouter -> Nouveau projet…
  • (Figure 1)

[![entrez la description de l’image ici][Figure 1]][Figure 1]

  • Allez dans Installé -> Visual C# -> Test
  • Cliquez sur Projet de test unitaire
  • Donnez-lui un nom et cliquez sur OK
  • (Figure 2)

[![entrez la description de l’image ici][Figure 2]][Figure 2]

  • Le projet de test unitaire est ajouté à la solution
  • (Figure 3)

[![entrez la description de l’image ici][Figure 3]][Figure 3]

[Illustration 1] : https://i.stack.imgur.com/GSo4k.png [Figure 2] : https://i.stack.imgur.com/BIKJd.png [Illustration 3] : https://i.stack.imgur.com/0sWwU.png

Ajout de la référence à l’application que vous souhaitez tester

  • Dans le projet de test unitaire, ajoutez une référence au projet que vous souhaitez tester
  • Clic droit sur Références -> Ajouter une référence…
  • (Figure 3)

[![entrez la description de l’image ici][Figure 3]][Figure 3]

  • Sélectionnez le projet que vous souhaitez tester
  • Allez dans Projets -> Solution
  • Cochez la case du projet que vous souhaitez tester -> cliquez sur OK
  • (Figure 4)

[![entrez la description de l’image ici][Figure 4]][Figure 4]

[Illustration 3] : https://i.stack.imgur.com/JCevC.png [Figure 4] : https://i.stack.imgur.com/HfIYf.png

Deux méthodes pour créer des tests unitaires #

Méthode 1 #

  • Accédez à votre classe de test unitaire dans le projet de test unitaire

  • Rédiger un test unitaire

    [Testclass] public class UnitTest1 { [TestMethod] public void TestMethod1() { //Arrange ApplicationToTest.Calc ClassCalc = new ApplicationToTest.Calc(); int expectedResult = 5;

          //Act
      int result = ClassCalc.Sum(2,3);

      //Assert
      Assert.AreEqual(expectedResult, result);
  }

    }

Méthode 2 #

  • Allez la méthode que vous voulez tester
  • Clic droit sur la méthode -> Créer des tests unitaires
  • (Figure 4)

[![entrez la description de l’image ici][Figure 4]][Figure 4]

  • Définir le cadre de test sur MSTest
  • Définissez Test Project sur le nom de votre projet de test unitaire
  • Définissez Fichier de sortie sur le nom de la classe des tests unitaires
  • Définissez le code pour la méthode de test sur l’une des options répertoriées que vous préférez
  • Les autres options peuvent être modifiées mais ce n’est pas nécessaire

(Astuce : si vous n’avez pas encore créé de projet de tests unitaires, vous pouvez toujours utiliser cette option. Définissez simplement Projet de test sur et Fichier de sortie sur . Il créera le projet de test unitaire et cela ajoutera la référence du projet au projet de test unitaire)

  • (Figure 5)

[![entrez la description de l’image ici][Figure 5]][Figure 5]

  • Comme vous le voyez ci-dessous, il crée la base du test unitaire que vous devez remplir
  • (Figure 6)

[![entrez la description de l’image ici][Figure 6]][Figure 6]

[Illustration 4] : https://i.stack.imgur.com/8KUK5.png [Illustration 5] : https://i.stack.imgur.com/7xGvF.png [Illustration 6] : https://i.stack.imgur.com/MnzQ9.png

Exécution de tests unitaires dans Visual Studio #

  • Pour voir vos tests unitaires allez dans Test -> Windows -> Test Explorer
  • (Figure 1)

[![entrez la description de l’image ici][Figure 1]][Figure 1]

  • Cela ouvrira un aperçu de tous les tests de l’application
  • (Figure 2)

[![entrez la description de l’image ici][Figure 2]][Figure 2]

  • Dans la figure ci-dessus, vous pouvez voir que l’exemple a un test unitaire et qu’il n’a pas encore été exécuté
  • Vous pouvez double-cliquer sur un test pour accéder au code où le test unitaire est défini
  • Vous pouvez exécuter des tests simples ou multiples avec les boutons Exécuter tout ou Exécuter…
  • Vous pouvez également exécuter des tests et modifier les paramètres à partir du menu Test (Figure 1)

[Figure 1] : https://i.stack.imgur.com/C8EKC.png [Figure 2] : https://i.stack.imgur.com/jKoUO.png

Exécution d’une analyse de couverture de code dans Visual Studio #

  • Pour voir vos tests unitaires, allez dans Test -> Windows -> Code Coverage Results
  • (Figure 1)

[![entrez la description de l’image ici][Figure 1]][Figure 1]

  • Cela ouvrira la fenêtre suivante
  • (Figure 2)

[![entrez la description de l’image ici][Figure 2]][Figure 2]

  • La fenêtre est maintenant vide
  • Allez dans le menu Test -> Analyser la couverture du code
  • (Figure 3)

[![entrez la description de l’image ici][Figure 3]][Figure 3]

  • Les tests seront désormais également exécutés (voir les résultats dans l’explorateur de tests)
  • Les résultats seront affichés dans un tableau dans lequel vous pourrez voir quelles classes et méthodes sont couvertes par les tests unitaires et lesquelles ne le sont pas
  • (Figure 4)

[![entrez la description de l’image ici][Figure 4]][Figure 4]
