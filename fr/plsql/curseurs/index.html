Syntaxe #

  • Le curseur cursor_name est votre_instruction_select
  • Le curseur cursor_name (param TYPE) est your_select_statement_using_param
  • FOR x in (your_select_statement) BOUCLE …

Les curseurs déclarés sont difficiles à utiliser, et vous devriez préférer les boucles FOR dans la plupart des cas. Ce qui est très intéressant dans les curseurs par rapport aux simples boucles FOR, c’est qu’on peut les paramétrer.

Il vaut mieux éviter de faire des boucles avec PL/SQL et des curseurs au lieu d’utiliser Oracle SQL de toute façon. Cependant, pour les personnes habituées au langage procédural, cela peut être beaucoup plus facile à comprendre.

Si vous souhaitez vérifier si un enregistrement existe, puis faire différentes choses selon que l’enregistrement existe ou non, il est logique d’[utiliser les instructions MERGE] (https://www.wikiod.com/fr/oracle/differentes-facons-de-mettre-a-jour-les-enregistrements /different-ways-to-update-records/14689/update-using-merge#t=20160803063310475484) dans des requêtes SQL ORACLE pures au lieu d’utiliser des boucles de curseur. (Veuillez noter que MERGE n’est disponible que dans les versions d’Oracle>= 9i).

Curseur “FOR loop” paramétré #

DECLARE
  CURSOR c_emp_to_be_raised(p_sal emp.sal%TYPE) IS 
    SELECT * FROM emp WHERE  sal < p_sal;
BEGIN
  FOR cRowEmp IN c_emp_to_be_raised(1000) LOOP
    dbms_Output.Put_Line(cRowEmp .eName ||' ' ||cRowEmp.sal||'... should be raised ;)');
  END LOOP;
END;
/

Curseur “FOR loop” implicite #

BEGIN
  FOR x IN (SELECT * FROM emp WHERE sal < 100) LOOP
    dbms_Output.Put_Line(x.eName ||' '||x.sal||'... should REALLY be raised :D');
  END LOOP;
END;
/
  • Le premier avantage est qu’il n’y a pas de déclaration fastidieuse à faire (pensez à cet horrible “CURSEUR” que vous aviez dans les versions précédentes)
  • le deuxième avantage est que vous construisez d’abord votre requête de sélection, puis lorsque vous avez ce que vous voulez, vous pouvez immédiatement accéder aux champs de votre requête (x.<myfield>) dans votre boucle PL/SQL
  • La boucle ouvre le curseur et récupère un enregistrement à la fois pour chaque boucle. A la fin de la boucle, le curseur est fermé.
  • Les curseurs implicites sont plus rapides car le travail de l’interpréteur augmente à mesure que le code s’allonge. Moins il y a de code, moins l’interpréteur a de travail à faire.

Manipulation d’un CURSEUR #

  • Déclarez le curseur pour parcourir une liste d’enregistrements
  • Ouvrez-le
  • Récupérer l’enregistrement actuel dans les variables (ceci incrémente la position)
  • Utilisez %notfound pour détecter la fin de la liste
  • N’oubliez pas de fermer le curseur pour limiter la consommation de ressources dans le contexte actuel

–

DECLARE
  CURSOR curCols IS -- select column name and type from a given table
         SELECT column_name, data_type FROM all_tab_columns where table_name='MY_TABLE';
  v_tab_column all_tab_columns.column_name%TYPE;
  v_data_type all_tab_columns.data_type%TYPE;
  v_ INTEGER := 1;
BEGIN
  OPEN curCols;
  LOOP
     FETCH curCols INTO v_tab_column, v_data_type;
     IF curCols%notfound OR v_ > 2000 THEN
       EXIT;
     END IF;
     dbms_output.put_line(v_||':Column '||v_tab_column||' is of '|| v_data_type||' Type.');
     v_:= v_ + 1;
  END LOOP;

  -- Close in any case
  IF curCols%ISOPEN THEN  
     CLOSE curCols;
  END IF;
END;
/

Travailler avec SYS_REFCURSOR #

SYS_REFCURSOR peut être utilisé comme type de retour lorsque vous avez besoin de gérer facilement une liste renvoyée non pas à partir d’une table, mais plus spécifiquement à partir d’une fonction :

fonction renvoyant un curseur #

CREATE OR REPLACE FUNCTION list_of (required_type_in IN VARCHAR2)
   RETURN SYS_REFCURSOR
IS
   v_ SYS_REFCURSOR;
BEGIN
   CASE required_type_in
      WHEN 'CATS'
      THEN
         OPEN v_ FOR
           SELECT nickname FROM (
                select 'minou' nickname from dual
      union all select 'minâ'           from dual
      union all select 'minon'          from dual         
           );
      WHEN 'DOGS'
      THEN
         OPEN v_ FOR
              SELECT dog_call FROM (
                select 'bill'   dog_call from dual
      union all select 'nestor'          from dual
      union all select 'raoul'           from dual         
           );
   END CASE;
   -- Whit this use, you must not close the cursor.
   RETURN v_;
END list_of;
/

et comment l’utiliser : #

DECLARE
   v_names   SYS_REFCURSOR;
   v_        VARCHAR2 (32767);
BEGIN
   v_names := list_of('CATS');
   LOOP
      FETCH v_names INTO v_;
      EXIT WHEN v_names%NOTFOUND;
      DBMS_OUTPUT.put_line(v_);
   END LOOP;
   -- here you close it
   CLOSE v_names;
END;
/
