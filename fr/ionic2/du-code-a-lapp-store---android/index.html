Vous trouverez des instructions étape par étape sur la façon de préparer et de télécharger l’application ionique de production sur Google Play.

Prêt pour la production #

Création d’un projet d’application

Lors de la création d’une application Android prête pour l’App Store, il est important lors de l’utilisation de ionic start, nous ajoutons les drapeaux --appname|-a et --id|-i qui est utilisé par Google Play pour identifier votre application à partir d’autres applications.

Si vous démarrez un nouveau projet d’application mobile, vous pouvez utiliser l’exemple cli ci-dessous.

$ ionic start --v2 -a "App Example" -i "com.example.app" -t "tabs"

1. Fichier de configuration de l’application

si vous souhaitez définir ces informations dans une application existante, vous pouvez modifier config.xml. Je recommande à ceux qui ont utilisé la commande ci-dessus de modifier également config.xml.

Confirmez/modifiez les attributs widget id, name, description et author.

Exemple:

<?xml version="1.0" encoding="UTF-8" standalone="yes"?>
<widget id="com.example.app" version="1.0.0" xmlns="http://www.w3.org/ns/widgets" xmlns:cdv="http://cordova.apache.org/ns/1.0">
  <name>Example App</name>
  <description>Example app for stackoverflow users</description>
  <author email="admin@example.com" href="http://example.com/">Your name or team</author>
  ...
</widget>

2. icône et écran de démarrage

Les types de fichiers pris en charge par les icônes et les images de démarrage sont png, psd ou ai et doivent avoir un nom de fichier correspondant à “icon” ou “splash” et placé sous le répertoire des ressources à la racine de votre projet. Les dimensions minimales de l’image de l’icône doivent être de 192 x 192 px et ne doivent pas avoir de coins arrondis. et l’écran de démarrage est beaucoup plus compliqué, alors cliquez ici pour en savoir plus. Néanmoins, les dimensions minimales doivent être de 2208x2208 px.

si vous avez un fichier icône à générer, utilisez cette commande ionic resources --icon si vous avez un fichier splash à générer, utilisez cette commande ionic resources --splash

3. Application de production de bâtiments

Avant de créer votre application de production, supprimez toutes les données de journal sensibles.

Pour créer une version avec toutes les optimisations par défaut en place, utilisez la balise –release & –prod

construction ionique android --release --prod

Pour une liste complète des optimisations disponibles, vous pouvez visiter le [@ionic/app-scripts repository][1]

4. Créer une clé privée

Maintenant, nous devons signer l’APK non signé (android-release-unsigned.apk) et exécuter un utilitaire d’alignement dessus pour l’optimiser et le préparer pour l’App Store. Si vous avez déjà une clé de signature, ignorez ces étapes et utilisez celle-ci à la place.

Ensuite, localisez votre fichier APK non signé android-release-unsigned.apk dans le répertoire du projet/platforms/android/build/outputs/apk/ et utilisez Commande keytools qui sera utilisée pour signer notre fichier apk. Vous pouvez utiliser l’exemple ci-dessous :

$ keytool -genkey -v -keystore my-release-key.keystore -alias androidKey -keyalg RSA -keysize 2048 -validity 10000

vous pouvez trouver my-release-key.keystore dans votre répertoire actuel.

Générons notre clé privée à l’aide de la commande keytool fournie avec le JDK. Si cet outil n’est pas trouvé, reportez-vous au guide d’installation :

Vous serez d’abord invité à créer un mot de passe pour le magasin de clés. Ensuite, répondez au reste des questions des bons outils et quand tout est fait, vous devriez avoir un fichier appelé my-release-key.keystore créé dans le répertoire courant.

Remarque : Assurez-vous de sauvegarder ce fichier dans un endroit sûr, si vous le perdez, vous ne pourrez pas soumettre de mises à jour pour votre application !

5. Signer l’APK

Pour signer l’APK non signé, exécutez l’outil jarsigner qui est également inclus dans le JDK :

$ jarsigner -verbose -sigalg SHA1withRSA -digestalg SHA1 -keystore my-release-key.keystore HelloWorld-release-unsigned.apk alias_name

Cela signe l’apk en place. Enfin, nous devons exécuter l’outil d’alignement zip pour optimiser l’APK. L’outil zipalign se trouve dans /path/to/Android/sdk/build-tools/VERSION/zipalign.

$ zipalign -v 4 HelloWorld-release-unsigned.apk HelloWorld.apk

Nous avons maintenant notre fichier binaire de version finale appelé HelloWorld.apk et nous pouvons le publier sur le Google Play Store pour que tout le monde puisse en profiter !

** Publiez votre application sur Google Play Store. ** Maintenant que notre APK de version est prêt pour le Google Play Store, nous pouvons créer une liste Play Store et télécharger notre APK. Pour commencer, vous devrez visiter la console développeur de Google Play Store et créer un nouveau compte développeur. Il vous en coûtera 25 $ de frais uniques.

Une fois que vous avez un compte de développeur, vous pouvez cliquer sur “Publier une application Android sur Google Play” et suivre les instructions à l’écran.

[1] : https://github.com/driftyco/ionic-app-scripts
