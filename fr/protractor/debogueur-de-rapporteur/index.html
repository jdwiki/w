Syntaxe #

  • navigateur.pause()

Cette section explique comment nous pouvons déboguer les tests de rapporteur.

Utilisation du navigateur.pause() #

La méthode pause() est l’une des solutions les plus simples que Protractor vous propose pour déboguer le code. Pour l’utiliser, vous devez l’ajouter dans votre code à l’endroit où vous souhaitez suspendre l’exécution. Une fois l’exécution en pause :

  1. Vous pouvez utiliser C (type C) pour avancer. Soyez prudent lorsque vous l’utilisez, vous devez écrire cette commande sans délai car vous pourriez obtenir une erreur de délai d’attente de votre bibliothèque d’assertions si vous avez retardé pour appuyer sur c.

  2. Tapez repl pour entrer en mode interactif. Le mode interactif est utilisé pour envoyer des commandes de navigateur directement pour ouvrir une instance de navigateur. Par exemple, en mode interactif, vous pouvez émettre une commande comme celle-ci :

    element(by.css(’#username’)).getText() NoSuchElementError: No element found using locator: by.username("#username")

Remarquez que la sortie de la commande ci-dessus apparaît directement là-bas, ce qui vous permet de connaître l’exactitude de votre commande.

Remarque : Si vous avez ouvert les outils de développement Chrome, vous devez les fermer avant de poursuivre le test, car ChromeDriver ne peut pas fonctionner lorsque les outils de développement sont ouverts.

  1. Quittez le mode débogage en utilisant CTRL + C, vous pouvez vous sortir du mode débogage en utilisant la commande classique CTRL + C.

    it(‘should pause when we use pause method’, function () { browser.get(’/index.html’);

     var username = element(by.model('username'));
 username.sendKeys('username');
 browser.pause();

 var password = element(by.model('password'));
 password.sendKeys('password');
 browser.pause();

    });

  2. Appuyez sur d pour passer à l’instruction de débogage suivante

Utilisation de browser.debugger() #

Vous pouvez utiliser browser.debugger() pour arrêter l’exécution. Vous pouvez l’insérer n’importe où dans votre code et il arrêtera l’exécution après cette ligne jusqu’à ce que vous ne commandiez pas de continuer.

Remarque : Pour exécuter les tests en mode débogueur, vous devez émettre une commande comme celle-ci :

    `protractor debug <configuration.file.js>`

Entrez c pour démarrer l’exécution et continuer après le point d’arrêt ou entrez la commande next. La commande suivante passe à la ligne suivante dans le flux de contrôle.

Le débogueur utilisé dans Protractor utilise node debugger et interrompt l’exécution de manière asynchrone. Par exemple, dans le code ci-dessous, le browser.debugger() sera appelé lorsque username.sendKeys('username') a été exécuté.

Remarque : Étant donné qu’il s’agit de tâches asynchrones, vous devez augmenter le délai d’expiration par défaut de vos spécifications, sinon une exception de délai d’expiration par défaut serait générée !

it('should pause when we use pause method', function () {
browser.get('/index.html');

var username = element(by.model('username'));
username.sendKeys('username');
browser.debugger();

var password = element(by.model('password'));
password.sendKeys('password');
});

On peut entrer dans le mode repl en entrant la commande-

debug > repl
> element(by.model('abc')).sendKeys('xyz');

Cela exécutera la commande sendKeys comme tâche suivante, puis entrera à nouveau dans le débogueur.

On peut changer le Port no. ils veulent déboguer leurs scripts en passant simplement le port à la méthode du débogueur-

browser.debugger(4545); //will start the debugger in port 4545

La méthode debugger() injecte un côté client de Protractor au navigateur et vous pouvez exécuter quelques commandes dans la console du navigateur afin de récupérer les éléments. L’un des exemples d’utilisation du script côté client est :

window.clientSideScripts.findInputs('nom d'utilisateur');
