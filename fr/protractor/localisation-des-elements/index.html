Pour pouvoir interagir avec une page, vous devez indiquer à Protractor exactement quel élément rechercher. La base utilisée pour sélectionner les éléments sont les localisateurs. Protractor, en plus d’inclure les sélecteurs génériques Selenium, possède également des localisateurs spécifiques à Angular qui sont plus robustes et persistants aux changements. Cependant, parfois, même dans une application Angular, des localisateurs réguliers doivent être utilisés.

Paramètres #

Paramètre Détail
sélecteur Une chaîne qui spécifie la valeur du sélecteur (dépend du localisateur)

Localisateurs spécifiques au rapporteur (pour les applications basées sur Angular) #

Ces localisateurs doivent être utilisés en priorité lorsque cela est possible, car ils sont plus persistants face aux modifications d’une application que les localisateurs basés sur css ou xpath, qui peuvent facilement se casser.

Localisateur de liaison #

Syntaxe

by.binding('bind value')

Exemple #

Voir

<span>{{user.password}}</span>
<span ng-bind="user.email"></span>

Localisateur

by.binding('user.password')
by.binding('user.email')

Prend également en charge les correspondances partielles

by.binding('email')

Localisateur de liaison exacte #

Semblable à binding, sauf que les correspondances partielles ne sont pas autorisées.

Syntaxe

by.exactBinding('exact bind value')

Exemple #

Voir

<span>{{user.password}}</span>

Localisateur

by.exactBinding('user.password')
by.exactBinding('password') // Will not work

Localisateur de modèle #

Sélectionne un élément avec une [directive de modèle angulaire][1]

Syntaxe

by.model('model value')

Exemple #

Voir

<input ng-model="user.username">

Localisateur

by.model('user.username')

Localisateur de texte de bouton #

Sélectionne un bouton en fonction de son texte. Ne doit être utilisé que si le texte du bouton ne devrait pas changer souvent.

Syntaxe

by.buttonText('button text')

Exemple #

Voir

<button>Sign In</button>

Localisateur

by.buttonText('Sign In')

Localisateur de texte de bouton partiel #

Semblable à buttonText, mais autorise les correspondances partielles. Ne doit être utilisé que si le texte du bouton ne devrait pas changer souvent.

Syntaxe

by.partialButtonText('partial button text')

Exemple

Voir

<button>Register an account</button>

Localisateur

by.partialButtonText('Register')

Localisateur de répéteur #

Sélectionne un élément avec une [Directive de répéteur angulaire][2]

Syntaxe

by.repeater('repeater value')

Exemple #

Voir

<tbody ng-repeat="review in reviews">
    <tr>Movie was good</tr>
    <tr>Movie was ok</tr>
    <tr>Movie was bad</tr>
</tbody>

Localisateur

by.repeater('review in reviews')

Prend également en charge les correspondances partielles

by.repeater('reviews')

Localisateur de répéteur exact #

Similaire à “répéteur”, mais n’autorise pas les correspondances partielles

Syntaxe

by.exactRepeater('exact repeater value')

Exemple #

Voir

<tbody ng-repeat="review in reviews">
    <tr>Movie was good</tr>
    <tr>Movie was ok</tr>
    <tr>Movie was bad</tr>
</tbody>

Localisateur

by.exactRepeater('review in reviews')
by.exactRepeater('reviews') // Won't work

CSS et localisateur de texte #

Un localisateur CSS étendu où vous pouvez également spécifier le contenu textuel de l’élément.

Syntaxe

by.cssContainingText('css selector', 'text of css element')

Exemple #

Voir

<ul>
    <li class="users">Mike</li>
    <li class="users">Rebecca</li>
</ul>

Localisateur

by.cssContainingText('.users', 'Rebecca') // Will return the second li only

Localisateur d’options #

Sélectionne un élément avec une [directive d’options angulaires] [3]

Syntaxe

by.options('options value')

Exemple #

Voir

<select ng-options="country.name for c in countries">
    <option>Canada</option>
    <option>United States</option>
    <option>Mexico</option>
</select>

Localisateur

by.options('country.name for c in countries')

Localisateur CSS profond #

Localisateur CSS qui s’étend dans le [DOM fantôme][4]

Syntaxe

by.deepCss('css selector')

Exemple #

Voir

<div>
    <span id="outerspan">
    <"shadow tree">
        <span id="span1"></span>
        <"shadow tree">
            <span id="span2"></span>
        </>
    </>
</div>

Localisateur

by.deepCss('span') // Will select every span element

[1] : https://docs.angularjs.org/api/ng/directive/ngModel [2] : https://docs.angularjs.org/api/ng/directive/ngRepeat [3] : https://docs.angularjs.org/api/ng/directive/ngOptions [4] : https://glazkov.com/2011/01/14/what-the-heck-is-shadow-dom/

Bases du localisateur #

Les localisateurs en eux-mêmes ne renvoient pas un élément avec lequel il est possible d’interagir dans Protractor, ce sont simplement des instructions qui indiquent à Protractor comment trouver l’élément.

Pour accéder à l’élément lui-même, utilisez cette syntaxe :

element(locator);
element.all(locator);

Remarque : le ou les éléments ne sont pas réellement accessibles tant qu’une action n’est pas effectuée dessus - c’est-à-dire que Protractor n’ira réellement récupérer l’élément que lorsqu’une action telle que getText() est appelée sur l’élément.

Si vous souhaitez sélectionner un seul élément à l’aide d’un localisateur, utilisez element. Si votre localisateur pointe vers plusieurs éléments, element renverra le premier trouvé. element renvoie un ElementFinder.

Si vous souhaitez sélectionner plusieurs éléments à l’aide d’un localisateur, element.all renverra tous les éléments trouvés. element.all renvoie un ElementArrayFinder, et chaque élément du tableau peut être accédé en utilisant différentes méthodes - par exemple, la fonction map.

element.all(locator).map(function(singleElement) {
        return singleElement.getText();
    }
});

** Localisateurs de chaînage **

Vous pouvez enchaîner plusieurs localisateurs pour sélectionner un élément dans une application complexe. Vous ne pouvez pas enchaîner directement les objets locator, vous devez enchaîner ElementFinders :

element(by.repeater('movie in movies').element(by.linkText('Watch Frozen on Netflix')

Il n’y a pas de limite au nombre de chaînes que vous pouvez utiliser ; à la fin, vous recevrez toujours un seul ElementFinder ou ElementArrayFinder, selon vos localisateurs.
