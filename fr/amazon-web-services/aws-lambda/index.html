AWS Lambda est un service qui vous permet d’exécuter du code back-end sans avoir besoin de provisionner ou de gérer des serveurs. AWS Lambda s’occupe de la mise à l’échelle et de la haute disponibilité. Le coût dépend directement de la fréquence et de la durée d’exécution de votre code.

Vous trouverez des exemples de création et de déploiement de fonctions AWS Lambda dans différentes langues.

  • Le code AWS Lambda doit être écrit sans état. Bien que l’instance d’un lambda puisse être conservée et réutilisée, vous ne devez jamais vous attendre à cela.

Projet Java Gradle de base #

Pour déployer du code Java sur AWS Lambda, vous devez créer un fichier zip de distribution contenant toutes les dépendances nécessaires pendant l’exécution. Exemple de projet dans Gradle :

apply plugin: 'java'

repositories {
    mavenCentral()
}

dependencies {
    compile 'com.amazonaws:aws-lambda-java-core:1.1.0'
}

task buildZip(type: Zip) {
    from compileJava
    from processResources
    into('lib') {
        from configurations.runtime
    }
}

build.dependsOn buildZip

L’exécution de gradle build créera un fichier zip avec toutes les dépendances fournies avec votre code, prêt à être déployé.

Code Lambda de base en Java #

Un lambda a besoin d’un gestionnaire, qui servira de point d’entrée à votre application. Chaque gestionnaire doit implémenter l’interface RequestHandler<I, O> où I est le type d’entrée et O est le type de sortie. Le type d’entrée est ensuite transmis à la méthode handleRequest() et la méthode renvoie le type de sortie.

Un exemple simple pourrait ressembler à ceci :

package com.example.lambda;

import com.amazonaws.services.lambda.runtime.Context;
import com.amazonaws.services.lambda.runtime.RequestHandler;

public class HelloNumber implements RequestHandler<Integer, String> {

    @Override
    public String handleRequest(Integer input, Context context) {
        return "You sent " + input;
    }
}

Code Lambda de base en JavaScript (NodeJs) #

Un lambda a besoin d’un gestionnaire, qui servira de point d’entrée à votre application. Dans le cas le plus simple, vous obtenez votre entrée à partir du context et transmettez le résultat à l’aide de la fonction callback() :

exports.handler = (événement, contexte, rappel) => { callback(null, ‘You sent ’ + event.number); } ;

Création d’AWS Lambda à l’aide de l’interface graphique Web (Java) #

Pour créer un lambda Java à l’aide de l’interface graphique, assurez-vous d’abord que votre zip de distribution est prêt. Vous aurez également besoin d’un compte AWS capable de créer des lambdas.

  • Basculez vers AWS Lambda, fermez l’écran d’introduction (ou lisez la documentation de démarrage) et appuyez sur le bouton “Créer une fonction Lambda”.
  • Sélectionnez un plan approprié. Un blueprint est un exemple de modèle qui peut vous aider à créer votre lambda. Pour l’instant, commencez par une Fonction vide.
  • Pour l’instant, ne vous inquiétez pas des déclencheurs et ignorez cet écran.
  • Nommez votre fonction et téléchargez le fichier zip.
  • Sur le même écran de configuration, saisissez votre nom de gestionnaire en tant que classe Java complète avec package, éventuellement avec la méthode qui gérera les requêtes. Si vous avez utilisé l’interface Java de l’exemple Basic Lambda Code, le nom de la classe suffit.
  • Sur le même écran, sélectionnez “Créer un nouveau rôle à partir de modèles” et nommez votre rôle. Le rôle est utilisé pour donner à votre lambda l’accès à d’autres ressources AWS. Dans ce cas, laissez-le vide car nous n’avons pas besoin d’autres ressources.
  • Sélectionnez la mémoire disponible la plus faible (128 Mo). La mémoire influence également le coût. Enregistrez votre fonction.

Test du code Lambda de base #

Si vous avez réussi à déployer Lambda, vous pouvez également le tester directement dans l’interface graphique. Lorsque vous cliquez sur le bouton bleu “Tester” pour la première fois, il vous présente la création d’un nouvel événement de test.

Si vous testez le code Java à partir de l’exemple Basic Lambda code in Java, supprimez tout le corps et ajoutez simplement un nombre « 1 ».

Si vous testez le code Javascript à partir de l’exemple Basic Lambda code in Javascript (Node), créez un json qui ressemble à ceci :

{
    "number": "1"
}

Exécutez le test. Vous devriez voir le résultat :

“Vous avez envoyé 1”

Ajout du déclencheur AWS API Gateway #

La façon la plus courante d’invoquer un Lambda consiste probablement à utiliser API Gateway, qui mappe un appel REST au code lambda. Vous pouvez ajouter plusieurs déclencheurs à votre Lambda à tout moment, y compris lors de la création d’un nouveau lambda.

Si vous connaissez API Gateway, vous pouvez associer n’importe quelle méthode à un lambda déployé. Créez simplement une nouvelle méthode avec le type d’intégration défini sur la fonction Lambda et pointez vers votre lambda. Vous pouvez mapper les entrées REST sur les entrées lambda et les sorties sur les sorties REST.

À FAIRE : Décidez si vous souhaitez décrire API Gateway ici ou faire référence à une autre partie de la documentation.
