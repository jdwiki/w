Déployer une image de conteneur Docker à l'aide d'ECS

Avant de pouvoir ajouter des instances ECS à un cluster, vous devez d’abord accéder à la console de gestion EC2 et créer des instances « ecs-optimized » avec un rôle IAM auquel la stratégie « AmazonEC2ContainerServiceforEC2Role » est attachée.

  1. Accédez à votre [Tableau de bord EC2][1] et cliquez sur le bouton « Lancer l’instance ».

  2. Sous “Community AMIs”, recherchez “ecs-optimized” et sélectionnez celle qui correspond le mieux aux besoins de votre projet. Tout fonctionnera. Cliquez sur Suivant.

  3. Lorsque vous accédez à Configure Instance Details, cliquez sur le create new IAM role link et créez un nouveau rôle appelé ecsInstanceRole.

  4. Attachez la stratégie AmazonEC2ContainerServiceforEC2Role à ce rôle.

  5. Par défaut, votre instance de conteneur se lance dans votre cluster “default”. Si vous souhaitez lancer votre propre cluster au lieu de celui par défaut, choisissez la liste “Détails avancés” et collez le script suivant dans le champ “Données utilisateur”, en remplaçant “votre_nom_de_cluster” par le nom de votre cluster.

    #!/bin/bash echo ECS_CLUSTER=your_cluster_name » /etc/ecs/ecs.config

  6. Ensuite, terminez la configuration de votre instance ECS.

REMARQUE : si vous créez un serveur Web, vous souhaiterez créer un “groupe de sécurité” pour autoriser l’accès au port 80.

  1. Créez un référentiel : aws ecr create-repository --repository-name example-repository
  2. Authentifiez votre client Docker auprès de votre registre : `aws ecr get-login –region us-east-1 | merde
  3. Créez votre image Docker : docker build -t example-image .
  4. Marquez votre image afin de pouvoir pousser l’image vers ce référentiel : docker tag example-image:latest example-namespace/example-image:latest
  5. Transférez cette image vers votre référentiel AWS nouvellement créé : docker push example-namespace/example-image:latest
  6. Enregistrez une définition de tâche ECS : aws ecs register-task-definition --cli-input-json example-task.json
  7. Exécutez la tâche : aws ecs run-task --task-definition example-task

[1] : https://console.aws.amazon.com/ec2/

Déployer un exemple d’application sur le service AWS ECS comme preuve de concept #

Suivez les étapes suivantes pour essayer un exemple d’application sur le service AWS ECS en tant que preuve de concept.

  1. Connectez-vous à la console de gestion AWS et accédez à AWS service catalog -> Compute -> Ec2

  2. Créez une machine virtuelle (instance EC2) à l’aide du système d’exploitation amazon linux 64 bits, que nous utiliserons pour configurer docker, git, l’outil d’agent AWS ECS et d’autres outils. Nous utiliserons également la même machine virtuelle en tant que nœud dans le cluster ECS pour déployer des applications basées sur des conteneurs. Suivez les étapes ci-dessous pour créer une machine virtuelle. a) Suivez les étapes habituelles pour créer une instance EC2, accordez une attention particulière aux étapes suivantes lors de la création de l’instance EC2. b) Sélectionnez un rôle IAM avec le moins d’autorisations suivantes : AmazonEC2ContainerServiceforEC2Role c) Assurez-vous que Java est installé sur la machine virtuelle [![entrez la description de l’image ici][1]][1]

  3. Installation de docker [exécuter les commandes ci-dessous] mettez d’abord à jour le référentiel de packages yum

    sudo yum update –y maintenant pour installer docker exécutez yum install

    sudo yum install -y docker

  4. Démarrer le service Docker

    sudo service docker start

  5. Ajoutez l’utilisateur ec2 au groupe docker afin de pouvoir exécuter les commandes Docker sans utiliser sudo.

    sudo usermod -a -G docker ec2-user

  6. Déconnectez-vous de l’EC2 et reconnectez-vous pour récupérer les nouvelles autorisations du groupe Docker.

  7. Vérifiez que l’utilisateur ec2 peut exécuter des commandes Docker sans sudo.

    docker info

  8. Installer Git

    sudo yum install -y git

  9. Clonez l’exemple d’application PHP sur l’instance EC2 à partir de git. Nous utiliserons cette application pour notre POC.

    git clone https://github.com/awslabs/ecs-demo-php-simple-app

    cd ecs-demo-php-simple-app

vérifier que Dockerfile existe en listant le contenu du répertoire

ls
  1. Accédez au catalogue de services AWS -> Calcul -> Service de conteneur Ec2
  2. Cliquez sur Commencer [![entrez la description de l’image ici][2]][2]
  3. Cliquez sur annuler [![entrez la description de l’image ici][3]][3]
  4. Cliquez sur les référentiels dans le menu Référentiels à gauche [![entrez la description de l’image ici][4]][4]
  5. Cliquez sur Commencer

[![entrez la description de l’image ici][5]][5] 15. Entrez le nom du référentiel et cliquez sur suivant [![entrez la description de l’image ici][6]][6] 16. Configurer les outils EC2

aws configure

fournir l’ID de clé d’accès AWS, la clé d’accès secrète, le nom de la région par défaut selon votre compte 17. Construire, étiqueter et pousser l’image Docker a) Récupérez la commande docker login que vous pouvez utiliser pour authentifier votre client Docker auprès de votre registre :

aws ecr get-login --region us-east-1

b) Exécutez la commande return comme sortie de l’étape précédente 18. Créez l’image Docker à partir de votre Dockerfile. (Rappelez-vous l’étape 9, où vous avez téléchargé un exemple d’application docker) un)

docker build -t amazon-ecs-sample .

(Notez que le “.” signifie le répertoire actuel) b) Exécutez des images docker pour vérifier que l’image a été créée correctement et que le nom de l’image contient un référentiel dans lequel vous pouvez transférer vos modifications vers l’image docker

docker images

[![entrez la description de l’image ici][7]][7]

c) Exécutez l’image nouvellement construite. L’option -p 80:80 mappe le port 80 exposé sur le conteneur au port 80 sur le système hôte (instance Ec2 dans ce cas).

docker run -p 80:80 amazon-ecs-sample

Ignorez l’avertissement « apache2 : Impossible de déterminer de manière fiable le nom de domaine complet du serveur, en utilisant 172.17.0.2 pour ServerName » 19. Essayez d’accéder à la page Web de l’exemple d’application sur le navigateur, assurez-vous que le port 80 est ouvert dans les groupes de sécurité associés à l’instance

http://<ec2-instance-dns-address>

[![entrez la description de l’image ici][8]][8]

  1. Appuyez sur la touche ctrl + c, cela arrêtera l’image docker. L’exemple d’application ne doit pas être accessible.

  2. Maintenant, après avoir vérifié avec succès notre exemple d’application docker, nous allons essayer de configurer un cluster pour exécuter l’exemple d’application automatiquement. De plus, à des fins de démonstration, nous essaierons d’utiliser l’instance ec2 existante en tant que nœud dans le cluster. Ceci peut être réalisé en installant un programme agent sur l’instance ec2.

  3. Installation de l’agent de conteneur Amazon ECS sur l’instance ec2 un)

    sudo yum install -y ecs-init

b) Redémarrez le démon docker

sudo service docker restart

c) Démarrez la tâche de démarrage ecs-init

sudo start ecs

d) (Facultatif) Vous pouvez vérifier que l’agent est en cours d’exécution et afficher des informations sur votre nouvelle instance de conteneur avec l’API d’introspection de l’agent. Assurez-vous que le port 51678 est ouvert dans le groupe de sécurité.

curl http://localhost:51678/v1/metadata

  1. Accédez au catalogue de services AWS -> Compute -> Ec2 Container Service -> Cluster et vérifiez qu’un cluster par défaut est créé [![entrez la description de l’image ici][9]][9]

  2. Nous procédons maintenant à la création d’un groupe de tâches et à l’ajout de notre image docker en tant que tâche à exécuter sur le cluster a) Examinez le fichier simple-app-task-def.json dans le dossier ecs-demo-php-simple-app. b) Modifiez le simple-app-task-def.json et réduisez la mémoire, afin qu’elle puisse s’exécuter sur une instance éligible au niveau gratuit (je suppose que l’on utilise une instance ec2 éligible au niveau gratuit pour ce POC, sinon pas besoin de réduire la mémoire limite) c) Mettre à jour memory=250 dans toutes les occurrences du fichier simple-app-task-def.json d) Enregistrez une définition de tâche avec le fichier simple-app-task-def.json.

    aws ecs register-task-definition –cli-input-json file://simple-app-task-def.json

e) Accédez à la définition de tâche dans la page de service de conteneur ec2, vous trouverez la définition de tâche enregistrée

f) Utilisez la commande AWS CLI suivante pour exécuter une tâche avec la définition de tâche console-sample-app.

aws ecs run-task --task-definition console-sample-app

g) Ouvrez l’exemple d’application Web dans le navigateur, il doit être accessible (reportez-vous à l’étape 19)

example-task.json #

{
  "family": "example-task",
  "containerDefinitions": [
    {
        "environment": [],
        "name": "example-container",
        "image": "example-namespace/example-image:latest",
        "cpu": 10,
        "memory": 500,
        "portMappings": [
            {
                "containerPort": 8080,
                "hostPort": 80
            }
        ],
        "entryPoint": [],
        "essential": true
    }
  ]
}
