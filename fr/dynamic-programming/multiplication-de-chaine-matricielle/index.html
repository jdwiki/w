Multiplication de chaîne matricielle

  • Solution récursive

Sur cette page

Solution récursive #

Multiplication de chaîne matricielle est un problème d’optimisation qui peut être résolu à l’aide de la programmation dynamique. Étant donné une séquence de matrices, le but est de trouver la manière la plus efficace de multiplier ces matrices. Le problème n’est pas réellement d’effectuer les multiplications, mais simplement de décider de la séquence des multiplications matricielles impliquées.

Disons que nous avons deux matrices A1 et A2 de dimension m * n et p * q . D’après les règles de la multiplication matricielle, nous savons que,

  1. Nous pouvons multiplier A1 et A2 si et seulement si n = p. Cela signifie que le nombre de colonnes de A1 doit être égal au nombre de lignes de A2.
  2. Si la première condition est satisfaite et que nous multiplions A1 et A2, nous obtiendrons une nouvelle matrice, appelons il A3 de dimension m * q.
  3. Pour multiplier A1 et A2, nous devons faire quelques multiplications d’échelle. Le nombre total de multiplications de scaler que nous devons effectuer est (m * n * q) ou (m * p * q ).
  4. A1 * A2 n’est pas égal à A2 * A 1.

Si nous avons trois matrices A1, A2 et A3 de dimension ** m** * n, n * p et p * q respectivement, alors A4 = A1 * A2 * A3 aura une dimension de m * q. Nous pouvons maintenant effectuer cette multiplication matricielle de deux manières :

  • Nous multiplions d’abord A1 et A2, puis avec le résultat nous multiplions *A3 *. C’est-à-dire : (A1 * A2) * A3.
  • Nous multiplions d’abord A2 et A3, puis avec le résultat nous multiplions *A1 *. C’est-à-dire : A1 * (A2 * A3).

Vous pouvez remarquer que l’ordre de multiplication reste le même, c’est-à-dire que nous ne multiplions pas (A1 * A3) * ** A2** car il n’est peut-être pas valide. On change seulement la parenthèse pour multiplier un ensemble avant de le multiplier avec le reste. La manière dont nous plaçons ces parenthèses est importante. Pourquoi? Disons que la dimension de 3 matrices A1, A2 et A3 est 10 * 100, 100 * 5, 5 * 50. Alors,

  1. Pour (A1 * A2) * A3, le nombre total des multiplications du détartreur sont : (10 * 100 * 5) + (10 * 5 * 50) = 7500 fois.
  2. Pour A1 * (A2 * A3), le nombre total des multiplications du scaler sont : (100 * 5 * 50) + (10 * 100 * 50) = 75000 fois.

Pour le 2ème type, le nombre de multiplication du scaler est 10 fois le nombre du 1er type ! Donc, si vous pouvez trouver un moyen de trouver l’orientation correcte des parenthèses nécessaires pour minimiser la multiplication totale du scaler, cela réduirait à la fois le temps et la mémoire nécessaires à la multiplication matricielle. C’est là que la multiplication de chaînes matricielles devient utile. Ici, nous ne nous intéresserons pas à la multiplication réelle des matrices, nous ne découvrirons que l’ordre correct des parenthèses afin que le nombre de multiplications du scaler soit minimisé. Nous aurons des matrices A1, A2, A3 …. …. An et nous découvrirons le nombre minimum de multiplications de scaler nécessaires pour les multiplier. Nous supposerons que les dimensions données sont valides, c’est-à-dire qu’elles satisfont notre première exigence pour la multiplication matricielle.

Nous utiliserons la méthode diviser pour régner pour résoudre ce problème. La programmation dynamique est nécessaire en raison de sous-problèmes communs. Par exemple : pour n = 5, nous avons 5 matrices A1, *A2 *, A3, A4 et A5. Nous voulons connaître le nombre minimum de multiplications nécessaires pour effectuer cette multiplication matricielle A1 * A2 * A 3 * A4 * A5. Parmi les nombreuses façons, concentrons-nous sur une : (A1 * A2) * (A3 sub> * A4 * A5).

Pour celui-ci, nous trouverons Agauche = A1 * A2 . Adroit = A3 * A4 * A5< /sous>. Ensuite, nous découvrirons Aréponse = Agauche * Adroite. Le nombre total de multiplications de mise à l’échelle nécessaires pour trouver Aréponse : = Le nombre total de multiplications de mise à l’échelle nécessaires pour déterminer Agauche + Le total nombre de multiplications du scaler nécessaires pour déterminer Aright + Le nombre total de multiplications du scaler nécessaires pour déterminer Aleft * A à droite.

Le dernier terme, Le nombre total de multiplications nécessaires pour déterminer Agauche * Adroite peut être écrit comme : Le nombre de lignes dans Agauche * le nombre de colonnes dans Agauche * le nombre de colonnes dans *Adroite *. (Selon la 2ème règle de multiplication matricielle)

Mais nous pourrions également définir la parenthèse d’autres manières. Par exemple:

  • A1 * (A2 * A3 * A 4 * A5)
  • (A1 * A2 * A3) * (A< sous>4 * A5)
  • (A1 * A2 * A3 * A 4) * A5 etc.

Pour chacun des cas possibles, nous déterminerons le nombre de multiplications nécessaires pour trouver Agauche et Adroite, puis pour Agauche * Adroite. Si vous avez une idée générale de la récursivité, vous avez déjà compris comment effectuer cette tâche. Notre algorithme sera :

- Set parenthesis in all possible ways.
- Recursively solve for the smaller parts.
- Find out the total number of scaler multiplication by merging left and right.
- Of all possible ways, choose the best one.

Pourquoi est-ce un problème de programmation dynamique ? Pour déterminer (A1 * A2 * A3), si vous avez déjà calculé (A1 * A2), il sera utile dans ce cas.

Pour déterminer l’état de cette récursivité, nous pouvons voir que pour résoudre chaque cas, nous aurons besoin de connaître la gamme de matrices avec lesquelles nous travaillons. Nous aurons donc besoin d’un begin et d’un end. Comme nous utilisons diviser pour régner, notre cas de base aura moins de 2 matrices (begin >= end), où nous n’avons pas du tout besoin de multiplier. Nous aurons 2 tableaux : ligne et colonne. row[i] et column[i] stockeront le nombre de lignes et de colonnes pour la matrice Ai. Nous aurons un tableau dp[n][n] pour stocker les valeurs déjà calculées et l’initialiser avec -1, où -1 représente la valeur n’a pas été calculée encore. dp[i][j] représente le nombre de multiplications du scaler nécessaires pour multiplier Ai, *Ai+1 *, …..,Aj inclus. Le pseudo-code ressemblera à :

Procedure matrixChain(begin, end):
if begin >= end
    Return 0
else if dp[begin][end] is not equal to -1
    Return dp[begin][end]
end if
answer := infinity
for mid from begin to end
    operation_for_left := matrixChain(begin, mid)
    operation_for_right := matrixChain(mid+1, right)
    operation_for_left_and_right := row[begin] * column[mid] * column[end]
    total := operation_for_left + operation_for_right + operation_for_left_and_right
    answer := min(total, answer)
end for
dp[begin][end] := answer
Return dp[begin][end]

Complexité:

La valeur de begin et end peut aller de 1 à n. Il existe n2 états différents. Pour chaque état, la boucle à l’intérieur s’exécutera n fois. Complexité temporelle totale : O(n³) et complexité mémoire : O(n²).
