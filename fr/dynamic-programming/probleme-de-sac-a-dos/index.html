Problème de sac à dos

  • 0-1 Problème de sac à dos

Sur cette page

  • 0-1 Problème de sac à dos

Le problème du sac à dos ou du sac à dos est un problème d’optimisation combinatoire. Étant donné un ensemble d’éléments, chacun avec un poids et une valeur, déterminez le nombre de chaque élément à inclure dans une collection de sorte que le poids total soit inférieur ou égal à une limite donnée et que la valeur totale soit aussi grande que possible. Il tire son nom du problème rencontré par quelqu’un qui est contraint par un sac à dos de taille fixe et doit le remplir avec les objets les plus précieux.

Le problème se pose souvent dans l’allocation des ressources où il y a des contraintes financières et est étudié dans des domaines tels que combinatorics, computer science.org/wiki/Computer_science), théorie de la complexité, cryptographie, mathématiques appliquées et daily fantasy sports.

Le problème du sac à dos est étudié depuis plus d’un siècle, les premiers travaux remontant à 1897. Le nom “problème du sac à dos” remonte aux premiers travaux du mathématicien [Tobias Dantzig](https://en.wikipedia.org /wiki/Tobias_Dantzig) (1884-1956) et fait référence au problème courant d’emballer vos objets les plus précieux ou utiles sans surcharger vos bagages.

0-1 Problème de sac à dos #

Supposons qu’on vous demande, étant donné le poids total que vous pouvez transporter sur votre sac à dos et certains articles avec leur poids et leur valeur, comment pouvez-vous prendre ces articles de telle sorte que la somme de leurs valeurs soit maximale, mais la somme de leurs poids ne ne dépasse pas le poids total que vous pouvez porter ? Le 0-1 indique que vous choisissez l’article ou que vous ne le faites pas. Nous avons également une quantité de chaque article. Cela signifie que vous ne pouvez pas diviser l’élément. S’il ne s’agissait pas d’un problème de sac à dos 0-1, cela signifie que si vous pouviez diviser les éléments, il existe une solution gourmande, appelée problème de sac à dos fractionnaire.

Concentrons-nous, pour l’instant, sur notre problème actuel. Par exemple, disons que nous avons une capacité de sac à dos de 7. Nous avons 4 articles. Leurs poids et valeurs sont :

+----------+---+---+---+---+
|   Item   | 1 | 2 | 3 | 4 |
+----------+---+---+---+---+
|  Weight  | 1 | 3 | 4 | 5 |
+----------+---+---+---+---+
|   Value  | 1 | 4 | 5 | 7 |
+----------+---+---+---+---+

Une méthode de force brute consisterait à prendre toutes les combinaisons possibles d’éléments. Ensuite, nous pouvons calculer leurs poids totaux et exclure ceux qui dépassent la capacité de notre sac à dos et trouver celui qui nous donne la valeur maximale. Pour 4 éléments, nous devrons vérifier (4! - 1) = 23 combinaisons possibles (excluant une sans aucun élément). C’est assez encombrant lorsque le nombre d’éléments augmente. Ici, quelques aspects que nous pouvons remarquer, c’est-à-dire :

  • Nous pouvons prendre des objets moins importants et calculer la valeur maximale que nous pouvons obtenir en utilisant ces objets et les combiner. Notre problème peut donc être divisé en sous-problèmes.
  • Si nous calculons les combinaisons pour l’élément {1,2}, nous pouvons l’utiliser lorsque nous calculons {1, 2, 3}.
  • Si nous minimisons le poids et maximisons la valeur, nous pouvons trouver notre solution optimale.

Pour ces raisons, nous allons utiliser la programmation dynamique pour résoudre notre problème. Notre stratégie sera la suivante : chaque fois qu’un nouvel article arrive, nous vérifions si nous pouvons choisir l’article ou non, et encore une fois, nous sélectionnons les articles qui nous donnent le maximum de valeur. Maintenant, si nous choisissons l’article, notre valeur sera la valeur de l’article, plus la valeur que nous pouvons obtenir en soustrayant la valeur de notre capacité et le maximum que nous pouvons obtenir pour ce poids restant. Si nous ne choisissons pas l’article, nous choisirons les articles qui nous donnent une valeur maximale sans inclure l’article. Essayons de le comprendre avec notre exemple :

Nous allons prendre un tableau 2D table, où le nombre de colonnes sera la valeur maximale que nous pouvons obtenir en prenant les éléments + 1. Et le nombre de lignes sera le même que le nombre d’éléments. Notre tableau ressemblera à :

+-------+--------+---+---+---+---+---+---+---+---+
| Value | Weight | 0 | 1 | 2 | 3 | 4 | 5 | 6 | 7 |
+-------+--------+---+---+---+---+---+---+---+---+
|   1   |    1   | 0 |   |   |   |   |   |   |   |
+-------+--------+---+---+---+---+---+---+---+---+
|   4   |    3   | 0 |   |   |   |   |   |   |   |
+-------+--------+---+---+---+---+---+---+---+---+
|   5   |    4   | 0 |   |   |   |   |   |   |   |
+-------+--------+---+---+---+---+---+---+---+---+
|   7   |    5   | 0 |   |   |   |   |   |   |   |
+-------+--------+---+---+---+---+---+---+---+---+

Nous avons incorporé le poids et la valeur de chaque article au tableau pour notre commodité. N’oubliez pas qu’elles ne font pas partie du tableau, elles sont uniquement à des fins de calcul, vous n’avez pas besoin de stocker ces valeurs dans le tableau table.

Notre première colonne est remplie de 0. Cela signifie que si notre capacité maximale est de 0, quel que soit l’article que nous avons, puisque nous ne pouvons pas choisir d’articles, notre valeur maximale sera de 0. Commençons par Table[0][1]. Lorsque nous remplissons Table[1][1], nous nous demandons si notre capacité maximale était de 1 et si nous n’avions que le premier élément, quelle serait notre valeur maximale ? Le mieux que nous puissions faire est 1, en choisissant l’article. Pour Table[0][2], cela signifie que si notre capacité maximale est de 2 et que nous n’avons que le premier élément, la valeur maximale que nous pouvons obtenir est de 1. Ce sera la même chose pour Table[0][3], Table[0][4], Table[0][5], Table [0][6] et tableau[0][7]. C’est parce que nous n’avons qu’un seul élément, ce qui nous donne une valeur 1. Puisque nous n’avons que 1 quantité de chaque article, peu importe comment nous augmentons la capacité de notre sac à dos, à partir d’un article, 1 est la meilleure valeur que nous puissions faire. Ainsi, notre tableau ressemblera à :

+-------+--------+---+---+---+---+---+---+---+---+
| Value | Weight | 0 | 1 | 2 | 3 | 4 | 5 | 6 | 7 |
+-------+--------+---+---+---+---+---+---+---+---+
|   1   |    1   | 0 | 1 | 1 | 1 | 1 | 1 | 1 | 1 |
+-------+--------+---+---+---+---+---+---+---+---+
|   4   |    3   | 0 |   |   |   |   |   |   |   |
+-------+--------+---+---+---+---+---+---+---+---+
|   5   |    4   | 0 |   |   |   |   |   |   |   |
+-------+--------+---+---+---+---+---+---+---+---+
|   7   |    5   | 0 |   |   |   |   |   |   |   |
+-------+--------+---+---+---+---+---+---+---+---+

Ensuite, pour Table[1][1], nous nous demandons si nous avions les articles 1 et 2 et si la capacité maximale de notre sac à dos était de **1 **, quelle est la valeur maximale que nous pouvons obtenir ? Si nous prenons à la fois les articles 1 et 2, le poids total sera de 4, ce qui dépassera la capacité actuelle de notre sac à dos. L’élément 2 ne peut donc pas être sélectionné. Maintenant, que pouvons-nous faire de mieux sans prendre l’objet 2 ? La valeur à partir du haut, c’est-à-dire Table[0][1] qui contient la valeur maximale que nous pouvons obtenir si nous avions une capacité maximale 1 et nous n’avons pas choisi le deuxième élément. Pour Table[1][2], puisque 2 est inférieur à weight[2], c’est-à-dire le poids du deuxième article, nous ne pouvons pas prendre l’article . Nous pouvons donc établir que, si le poids de l’article actuel est supérieur à notre capacité maximale, nous ne pouvons pas prendre cet article. Dans ce cas, nous prendrons simplement la valeur du haut, qui représente la valeur maximale que nous pouvons prendre en excluant l’élément.

if weight[i] > j
    Table[i][j] := Table[i-1][j]
end if

Maintenant, pour Table[1][3] puisque notre capacité maximale est égale à notre poids actuel, nous avons deux choix.

  • Nous choisissons l’objet et ajoutons sa valeur à la valeur maximale que nous pouvons obtenir des autres objets restants après avoir pris cet objet.
  • Nous pouvons exclure cet article.

Parmi les deux choix, nous choisirons celui dont nous pouvons tirer le maximum de valeur. Si nous sélectionnons l’élément, nous obtenons : valeur pour cet élément + valeur maximale du reste des éléments après avoir pris cet élément = 4 + 0 = 4. Nous obtenons 4 (valeur de l’article) de notre tableau poids et le 0 (valeur maximale que nous pouvons obtenir du reste des articles après avoir pris cet article) vient en allant **1 ** pas au-dessus et ** 3 ** pas en arrière. C’est-à-dire Table[0][0]. Pourquoi? Si nous prenons l’article, notre capacité restante sera de 3 - 3 = 0 et l’article restant sera le premier article. Eh bien, si vous vous souvenez, Table[0][0] stocke la valeur maximale que nous pouvons obtenir si notre capacité était de 0 et que nous n’avions que le premier élément. Maintenant, si nous ne sélectionnons pas l’élément, la valeur maximale que nous pouvons obtenir provient de l’étape 1 ci-dessus, c’est-à-dire 1. Maintenant, nous prenons le maximum de ces deux valeurs (4, 1) et définissons Table[1][2] = 4. Pour Table[1][4], depuis 4, la capacité maximale du sac à dos est supérieure à 3, le poids de notre article actuel, nous avons à nouveau deux options. Nous prenons max(Poids[2] + Table[0][4-Poids[2]], Table[0][4]) = max (Poids[2] + Tableau[0][1], Tableau[0][4]) = max(4 + 1, 1) = 5.

if weight[i] <= j
    w := weight[i]
    Table[i][j] := max(w + Table[i][j-w], Table[i-1][j])
end if

En utilisant ces deux formules, nous pouvons remplir tout le tableau Table. Notre tableau ressemblera à :

+-------+--------+---+---+---+---+---+---+---+---+
| Value | Weight | 0 | 1 | 2 | 3 | 4 | 5 | 6 | 7 |
+-------+--------+---+---+---+---+---+---+---+---+
|   1   |    1   | 0 | 1 | 1 | 1 | 1 | 1 | 1 | 1 |
+-------+--------+---+---+---+---+---+---+---+---+
|   4   |    3   | 0 | 1 | 1 | 4 | 5 | 5 | 5 | 5 |
+-------+--------+---+---+---+---+---+---+---+---+
|   5   |    4   | 0 | 1 | 1 | 4 | 5 | 6 | 6 | 9 |
+-------+--------+---+---+---+---+---+---+---+---+
|   7   |    5   | 0 | 1 | 1 | 4 | 5 | 7 | 8 | 9 |
+-------+--------+---+---+---+---+---+---+---+---+

Ici, la dernière valeur que nous avons insérée dans notre tableau, Table[3][7] contient notre valeur maximale requise. Il s’agit de la valeur maximale que nous pouvons obtenir si nous avions 4 articles et que notre capacité maximale du sac à dos était de 7.

Ici une chose qu’il faut retenir c’est que, même pour la première rangée, le poids peut être supérieur à la contenance du sac à dos. Nous devrons conserver une autre contrainte pour vérifier la valeur lors du remplissage de la première ligne. Ou nous pouvons simplement prendre une autre ligne et définir toutes les valeurs de la première ligne sur 0. Le pseudo-code ressemblerait à :

Procedure Knapsack(Weight, Value, maxCapacity):
n := Item.size - 1
Table[n+1][maxCapacity+1]
for i from 0 to n
    Table[i][0] := 0
end for
for j from 1 to maxCapacity
    if j >= Weight[0]
        Table[0][j] := Weight[0]
    else
        Table[0][j] := 0
    end if
end for
for i from 1 to n
    for j from 1 to maxCapacity
        if Weight[i] >= j                                            //can't pick the item
            Table[i][j] := Table[i-1][j]        
        else                                                        //can pick the item
            w := Weight[i]
            Table[i][j] := max(w + Table[i-1][j-w], Table[i-1][j])
        end if
    end for
end for
Return Table[n][maxCapacity]

La complexité temporelle de cet algorithme est O(n*maxCapacity), où n est le nombre d’éléments et maxCapacity est la capacité maximale de notre sac à dos.

Jusqu’à présent, nous avons trouvé la valeur maximale que nous pouvons obtenir de notre exemple. Une question demeure. Quels sont les articles réels? Nous allons retracer les valeurs dans notre tableau Table pour découvrir les éléments que nous avons pris. Nous suivrons deux stratégies :

  • Pour tout élément, si la valeur provient de la position ci-dessus, nous n’avons pas pris l’élément actuel. Nous allons 1 étape au-dessus.
  • Si la valeur ne provient pas de la position ci-dessus, nous avons pris l’élément. Nous allons donc 1 étape au-dessus et x étapes en arrière où x est le poids de l’élément actuel.

Le pseudo-code sera :

Procedure printItems(Table, maxCapacity, Value):
i := Item.size - 1
j := maxCapacity
while j is not equal to 0
    if Table[i][j] is equal to Table[i-1][j]
        i := i - 1
    else
        Print: i
        j := j - weight[i]
        i := i - 1
    end if
end while
