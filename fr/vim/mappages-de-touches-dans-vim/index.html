La mise à jour des mappages de touches Vim vous permet de résoudre deux types de problèmes : réaffecter des raccourcis clavier à des lettres plus faciles à mémoriser ou accessibles, et créer des raccourcis clavier pour des fonctions qui n’en ont pas. Ici, vous découvrirez les différentes manières de [re] mapper les raccourcis clavier et le contexte auquel ils s’appliquent * (c’est-à-dire les modes vim) *

Cartographie de base #

carte Aperçu #

Une séquence de touches peut être remappée sur une autre séquence de touches à l’aide de l’une des variantes “map”.

Par exemple, la “carte” typique suivante quittera le mode Insertion lorsque vous appuyez sur jk en séquence rapide :

:inoremap jk <Esc>

opérateur de carte #

Il existe plusieurs variantes de :map pour différents modes.

Commandes | Modes ——————————–|—————– ———————— :map, :noremap, :unmap | Mode normal, visuel et opérateur en attente :map!, :noremap!, :unmap! | Mode insertion et ligne de commande :nmap, :nnoremap, :nunmap | Mode normal :imap, :inoremap, :iunmap | Mode Insérer et Remplacer :vmap, :vnoremap, :vunmap | Mode visuel et sélection :xmap, :xnoremap, :xunmap | Mode visuel :smap, :snoremap, :sunmap | Sélectionnez le mode :cmap, :cnoremap, :cunmap | Mode ligne de commande :omap, :onoremap, :ounmap | Mode opérateur en attente

Habituellement, vous devriez utiliser les variantes :noremap; cela rend le mappage insensible au remappage et à la récursivité.

commande de carte #

  • Vous pouvez afficher tous les mappages en utilisant :map (ou l’une des variantes ci-dessus).
  • Pour afficher le mappage actuel pour une séquence de touches spécifique, utilisez :map <key> où <key> est une séquence de touches
  • Les touches spéciales comme Esc sont mappées en utilisant la notation spéciale <>, comme <Esc>. Pour la liste complète des codes clés, voir http://vimdoc.sourceforge.net/htmldoc/intro.html#keycodes
  • :nmapclear - Efface toutes les cartes en mode normal
  • :nunmap - Démapper une carte en mode normal
  • Vous pouvez configurer le temps maximum entre les touches d’une séquence en modifiant les variables timeout et ttimeout

Exemples #

  • imap jk <Esc> : taper jk en mode insertion vous ramènera au mode normal
  • nnoremap tt :tabnew<CR> : taper tt en mode normal ouvrira une nouvelle page d’onglet
  • nnoremap <C-j> <C-w>j : taper <C-j> en mode normal vous fera sauter à la fenêtre en dessous et à gauche
  • vmap <C-c> \cc : taper <C-c> en mode visuel exécutera \cc (commande NERDCommenter pour commenter la ligne). Comme cela repose sur un mappage de plugin, vous ne pouvez pas utiliser :vnoremap ici !

Combinaison de touches de chef de carte #

La clé principale peut être utilisée pour créer un mappage avec une liaison de clé qui peut être remplacée par l’utilisateur final.

Le leader est la clé \ par défaut. Afin de le remplacer, l’utilisateur final devrait exécuter :let g:mapleader='somekey(s)' avant de définir le mappage.

Dans un scénario typique, le mapleader est défini dans le .vimrc, et les plugins utilisent <Leader> dans la partie keybinding de leurs mappages pour les personnaliser.

Dans le plugin, nous définirions des mappages avec :

:nnoremap <Leader>a somecomplexaction

Cela mappera l’action somecomplexaction à la combinaison de touches \+a.

L’action a sans le leader ne change pas.

Il est également possible d’utiliser <Plug>Mappings pour laisser plus de place à la personnalisation des raccourcis clavier des plugins.

Illustration du mappage de base (raccourcis pratiques). #

Dans la plupart des éditeurs de texte, le raccourci standard pour enregistrer le document courant est Ctrl+S (ou Cmd+S sur macOS).

Vim n’a pas cette fonctionnalité par défaut mais cela peut être mappé pour faciliter les choses. L’ajout des lignes suivantes dans le fichier .vimrc fera l’affaire.

nnoremap <c-s> :w<CR>
inoremap <c-s> <c-o>:w<CR>

La commande nnoremap mappe Ctrl+s à la commande :w (écrire le contenu actuel dans le fichier) tandis que la commande inoremap mappe le Ctrl</ kbd>+S à la commande :w et revient au mode d’insertion (<c-o> passe en mode normal pour une commande et revient ensuite au mode d’insertion, sans modifier la position du curseur quelle autre des solutions comme <esc>:w<cr>a ne peuvent pas garantir).

De la même manière,

" This is commented, as Ctrl+Z is used in terminal emulators to suspend the ongoing program/process.
" nnoremap <c-z> :u<CR>

" Thus, Ctrl+Z can be used in Insert mode
inoremap <c-z> <c-o>:u<CR>

" Enable Ctrl+C for copying selected text in Visual mode
vnoremap <c-c> <c-o>:y<CR>

PS : Cependant, il convient de noter que Ctrl+S peut ne pas fonctionner comme prévu lors de l’utilisation de ssh (ou PuTTY). La solution à ce problème n’entre pas dans le cadre de ce document, mais peut être trouvée [ici][1].

[1] : https://raamdev.com/2007/recovering-from-ctrls-in-putty/
