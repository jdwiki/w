Copier, Couper et Coller #

Dans Vim, ces opérations sont gérées différemment de ce à quoi vous pourriez être habitué dans presque tous les autres éditeurs ou traitements de texte modernes (Ctrl-C, Ctrl-X, Ctrl-V). Pour comprendre, il faut connaître un peu les registres et les motions.

Remarque : cette section ne couvrira pas la copie et la découpe en mode visuel ni l’extraction de plage, car elles dépassent le cadre du mode normal et de l’édition de base.

Registres #

Vim utilise le concept de registers pour gérer le déplacement de texte dans le programme lui-même. Windows a un seul presse-papiers à cet effet, qui est analogue à un seul registre dans Vim. Lorsque vous copiez, coupez et collez dans Vim, il existe des moyens d’utiliser un flux de travail d’édition tout aussi simple (où vous n’avez pas à penser aux registres), mais il existe également des possibilités beaucoup plus complexes.

Un registre est ciblé pour l’entrée/sortie d’une commande en préfixant la commande avec " et un nom en lettre minuscule.

Mouvements #

Un mouvement dans Vim est une commande qui déplace la position du curseur ailleurs. Lorsque vous copiez, coupez et collez en mode normal, les possibilités de sélection de texte pour le mouvement ne sont limitées que par votre connaissance des mouvements. Quelques-uns seront illustrés ci-dessous.

Copier et couper #

Les commandes de base sur lesquelles les opérations de copie et de coupe sont construites sont y (‘yank’, pour copier) et d (‘delete’, pour couper). Vous verrez les similitudes dans le tableau suivant.

| Commande | : | Descriptif | | | —— | —— | —— | | y{mouvement} | | Copiez (“yank”) le texte indiqué par le mouvement dans le registre par défaut | aa | | Copiez la ligne actuelle dans le registre par défaut, linewise | Oui | | Copiez la ligne courante dans le registre par défaut (synonyme de yy) | “ayiw | | Copie le mot sur lequel se trouve le curseur dans le registre ‘a’ | 20"byy | | Copie vingt lignes, en commençant par le curseur, dans le registre ‘b’ | d{mouvement} | | Couper (“supprimer”) le texte indiqué par le mouvement dans le registre par défaut | jj | | Coupez la ligne actuelle dans le registre par défaut, linewise | D | | Couper du curseur à la fin de la ligne dans le registre par défaut (PAS un synonyme de dd) | “adiw | | Couper le mot sur lequel se trouve le curseur dans le registre ‘a’ | 20"bdd | | Coupez vingt lignes, en commençant par le curseur, dans le registre ‘b’

Remarque : lorsque quelque chose est copié ou coupé linewise, le comportement de collage illustré ci-dessous placera le texte soit avant soit après la line actuelle (plutôt que le curseur). Des exemples suivent pour clarifier.

Coller #

Il existe plusieurs façons de coller dans Vim, selon ce que vous essayez d’accomplir.

| Commande | : | Descriptif | | | —— | —— | —— | | p | | Collez ce qui se trouve dans le registre par défaut après le curseur | P | | Collez ce qui se trouve dans le registre par défaut avant le curseur | “ap | | Coller le contenu du registre ‘a’ après le curseur | “cP | | Coller le contenu du registre ‘c’ avant le curseur

Alors, comment puis-je effectuer un copier-coller vraiment simple ?

Si j’ai le texte suivant :

1 This line should be second
2 This line should be first

Je peux faire le copier-coller le plus simple en plaçant mon curseur quelque part sur la ligne 1 et en tapant ddp. Voici les résultats:

1 This line should be first
2 This line should be second

Qu’est-il arrivé? dd ‘Coupe’ la première ligne (linewise) dans le registre par défaut - qui ne contiendra qu’une seule chose à la fois, comme le presse-papiers de Windows - et p colle la ligne après l’actuel, qui vient de changer à cause de la commande dd.

Voici un exemple pas tout à fait aussi simple. J’ai besoin de déplacer quelques mots. (Ceci est artificiel et inutile, mais vous pouvez appliquer ce principe à de plus gros morceaux de code.)

1 These words order out are of

Je peux répéter w pour arriver au ‘o’ devant ‘order’ (ou b si je viens de le taper et de réaliser mon erreur).

Puis “adaw pour mettre ‘ordre’ dans le registre ‘a’.

Puis w pour arriver au ‘a’ dans ‘are’.

Suite à cela, je taperais “bdaw pour mettre ‘are ’ dans le registre ‘b’. Maintenant, j’ai ceci affiché :

1 These words out of

Pour être clair, maintenant ‘order ’ est dans le registre ‘a’ et ‘are ’ est dans le registre ‘b’, comme deux presse-papiers séparés.

Pour arranger les mots correctement, je tape b pour arriver au ‘o’ dans ‘out’, puis “bP pour mettre ‘are ’ du registre ‘b’ dans devant ‘out’ :

1 These words are out of

Maintenant, je tape A pour arriver à la fin de la ligne, suivi de EspaceEsc (en supposant qu’il n’y avait pas d’espace après ‘de’) et “ap pour mettre ‘ordre’ à sa place.

1 These words are out of order

Annuler et rétablir de base #

annuler #

| Commande | : | Descriptif | | | —— | —— | —— | | u |u,undo | Annuler la modification la plus récente | 5u | | Annuler les cinq modifications les plus récentes (utilisez n’importe quel nombre)

Veuillez noter que dans Vim, la “modification la plus récente” varie en fonction du mode dans lequel vous vous trouvez. Si vous entrez en mode Insertion (i) et tapez un paragraphe entier avant de revenir en mode Normal ( Échap), ce paragraphe entier est considéré comme la modification la plus récente.

Refaire #

| Commande | : | Descriptif | | | —— | —— | —— | | Ctrl-R |red,redo | Rétablir la modification annulée la plus récente | 2Ctrl-R| | Rétablir les deux modifications annulées les plus récentes (utilisez n’importe quel nombre)

Il existe une autre façon d’annuler et de rétablir les modifications dans Vim qui est gérée un peu différemment. Lorsque vous annulez une modification avec u, vous parcourez les nœuds sur une “arborescence” de vos modifications, et appuyez sur Ctrl-R pour parcourir ces nœuds dans l’ordre. (L’arbre d’annulation est un sujet distinct et est trop complexe pour être traité ici.)

Vous pouvez également utiliser U (c’est-à-dire en majuscule) pour supprimer toutes les dernières modifications sur une seule ligne (la ligne où vos dernières modifications ont été apportées). Cela ne traverse pas les nœuds de l’arbre de la même manière que u. L’utilisation de U compte en fait comme un changement lui-même - un autre nœud forward sur l’arborescence - de sorte que si vous appuyez sur U une deuxième fois immédiatement après la première, il agira comme une commande Redo .

Chacun a ses utilisations, mais u /  :undo devrait couvrir les cas les plus simples.

Introduction - Note rapide sur le mode normal #

En mode normal, les commandes peuvent être saisies par des combinaisons de touches directes (en tapant u pour annuler la dernière modification, par exemple). Ces commandes ont souvent des équivalents en mode ’ex’, accessibles en tapant deux-points :, qui vous déposent dans un tampon d’une seule ligne au bas de la fenêtre Vim.

En mode ’ex’, après avoir tapé les deux-points, vous tapez un nom de commande ou son abréviation suivi de Entrée pour exécuter la commande. Ainsi,  :undoEnter accomplit la même chose que de taper directement u en mode normal.

Vous pouvez voir que les commandes directes seront souvent plus rapides (une fois apprises) que les commandes “ex” pour une édition simple, mais pour être complet, dans la mesure du possible dans la documentation qui suit, si les deux sont disponibles, les deux seront affichés.

La plupart de ces commandes peuvent également être précédées d’un count en préfixant ou en intercalant un nombre - taper 3dd en mode normal, par exemple, supprime trois lignes (en commençant par la position actuelle du curseur).

Répétez le dernier changement #

La commande Répéter, exécutée avec la touche point ou point (.), est plus utile qu’il n’y paraît. Une fois appris, vous vous retrouverez à l’utiliser souvent.

| Commande | : | Descriptif | | | —— | —— | —— | | . | | Répéter le dernier changement | 10. | | Répétez le dernier changement 10 fois

Alors, pour un exemple très simple, si vous modifiez la ligne 1 en tapant iIEsc, avec le résultat suivant :

1 I made a mistake
2  made a mistake
3  made a mistake

Votre curseur sera à la position 1 de la ligne 1, et tout ce que vous avez à faire pour fixer les deux lignes suivantes est d’appuyer deux fois sur j. - c’est-à-dire j pour descendre une ligne et . pour répéter le dernier changement, qui était l’ajout du I. Pas besoin de revenir deux fois en mode Insertion pour corriger ces lignes.

Il devient beaucoup plus puissant lorsqu’il est utilisé pour répéter [macros][1].

[1] : https://www.wikiod.com/fr/vim/macros

Achèvement #

La saisie semi-automatique peut être utilisée pour faire correspondre les mots utilisés dans un document. Lors de la saisie d’un mot, Ctrlp ou Ctrln correspondra aux mots similaires précédents ou suivants dans le document.

Cela peut même être combiné avec le mode Ctrl-X pour compléter des lignes entières. Par exemple, tapez quelque chose comme :

This is an example sentence.

puis passez à la ligne suivante et commencez à taper la même phrase :

Thi

puis appuyez sur Ctrlp qui se traduira par :

This

Maintenant toujours en mode insertion, appuyez sur Ctrlx Ctrlp et le mot suivant sera complété résultant en :

This is

Continuez à appuyer sur Ctrlx Ctrlp jusqu’à ce que la ligne entière soit terminée.

Si vous savez que vous voulez compléter une ligne entière, tapez ceci :

This is an example sentence.

puis sur la ligne suivante tapez :

Thi

et appuyez sur x Ctrll pour terminer la ligne.

Si la complétion en cours est un nom de fichier, Ctrlx Ctrlf peut être utilisé pour compléter ce répertoire. Taper:

~/Deskt

puis appuyez sur Ctrlx Ctrlf et :

~/Desktop

sera terminé (si à cet endroit). Ctrlx Ctrlf peut ensuite être utilisé à plusieurs reprises pour lister les fichiers dans le Bureau.
