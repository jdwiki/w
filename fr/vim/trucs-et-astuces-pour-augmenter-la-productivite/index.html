Syntaxe #

- : définir le nombre relatif

  • : numéro de série
  • :set nonnombre / :set neuf
  • :pwd

Ce rechargement automatique ne se produira que si vous éditez votre vimrc dans une version complète de vim qui prend en charge autocmd.

Activer les numéros de ligne relatifs #

Pour supprimer certaines lignes de texte lorsque vous ne connaissez pas le nombre exact de lignes à supprimer, vous essayez 10dd , 5dd , 3dd jusqu’à ce que vous supprimiez toutes les lignes.

Les numéros de ligne relatifs résolvent ce problème, supposons que nous ayons un fichier contenant :

sometimes, you see a block of 
text. You want to remove 
it but you 
cannot directly get the 
exact number of 
lines to delete 
so you try 
10d , 5d 
3d until 
you
remove all the block.

Entrer en mode NORMAL : Esc

Maintenant, exécutez :set relativenumber. Une fois terminé, le fichier s’affichera comme suit :

<pré> 3 parfois, vous voyez un bloc de 2 texte. Vous souhaitez supprimer 1 mais toi 0 ne peut pas obtenir directement le 1 nombre exact de 2 lignes à supprimer 3 donc vous essayez 4 10j , 5j 5 3j jusqu’à 6 vous 7 supprimer tout le bloc.

où 0 est le numéro de ligne de la ligne actuelle et il affiche également le numéro de ligne réel devant le nombre relatif, vous n’avez donc plus besoin d’estimer le nombre de lignes de la ligne actuelle pour couper ou supprimer ou pire compter eux un par un.

Vous pouvez maintenant exécuter votre commande habituelle comme 6dd et vous êtes sûr du nombre de lignes.

Vous pouvez également utiliser la forme abrégée de la même commande :set rnu pour activer les nombres relatifs et :set nornu pour les désactiver.

Si vous avez également :set number ou si :set number est déjà activé, vous obtiendrez le numéro de ligne de la ligne sur laquelle se trouve le curseur.

<pré> 3 parfois, vous voyez un bloc de 2 texte. Vous souhaitez supprimer 1 mais toi 4 ne peut pas obtenir directement le 1 nombre exact de 2 lignes à supprimer 3 donc vous essayez 4 10j , 5j 5 3j jusqu’à 6 vous 7 supprimer tout le bloc.

Mappages pour quitter le mode Insertion #

Beaucoup d’utilisateurs de Vim trouvent le Esc trop difficile à atteindre et finissent par trouver un autre mappage facile à atteindre depuis la ligne d’accueil. Notez que Ctrl-[ peut être équivalent à Esc sur un clavier anglais, et est beaucoup plus facile à atteindre.

jk #

inoremap jk <ESC>

Celui-ci est vraiment facile à déclencher ; écrasez simplement vos deux premiers doigts sur la ligne de départ en même temps. Il est également difficile de déclencher accidentellement car “jk” n’apparaît jamais dans aucun mot anglais, et si vous êtes en mode normal, cela ne fait rien. Si vous ne tapez pas trop « blackjack », envisagez également d’ajouter inoremap kj <ESC> afin que vous n’ayez pas à vous soucier de la synchronisation des deux touches.

Verrouillage des majuscules #

##Linux Sous Linux, vous pouvez utiliser xmodmap pour que Caps Lock fasse la même chose que Esc. Mets ça dans un fichier :

!! No clear Lock 
clear lock
!! make caps lock an escape key
keycode 0x42 = Escape

Ensuite, exécutez fichier xmodmap. Cela remappe Caps Lock à Esc.

Sous Windows, vous pouvez utiliser [SharpKey] ou [AutoHotkey].

Si vous avez macOS 10.12.1 ou une version ultérieure, vous pouvez reconfigurer le verrouillage des majuscules sur Échap à l’aide des Préférences Système. Sélectionnez Clavier, accédez à l’onglet Clavier et cliquez sur Touches de modification.

Utilisation de la fonctionnalité de complétion de chemin dans Vim #

C’est très courant, vous mémorisez un chemin vers un fichier ou un dossier, vous ouvrez Vim et essayez d’écrire ce que vous venez de mémoriser, mais vous n’êtes pas sûr à 100% que ce soit correct, alors vous fermez l’éditeur et recommencez.

Lorsque vous voulez la fonctionnalité de complétion de chemin et que vous avez un fichier /home/ubuntu/my_folder/my_file et que vous modifiez un autre fichier faisant référence au chemin du précédent :

Entrer en mode insertion : insérer ou faites-le comme vous le souhaitez. Ensuite, écrivez /h. Lorsque le curseur est sous h, appuyez sur Ctrlx puis Ctrlf pour que l’éditeur le complète jusqu’à /home/ car le modèle /h est unique

Maintenant, supposons que vous ayez deux dossiers dans /home/ubuntu/ appelés my_folder_1 my_folder_2

et vous voulez le chemin /home/ubuntu/my_folder_2

comme d’habitude:

Entrer en mode insertion

écrivez /h et appuyez sur Ctrlx puis Ctrlf . Maintenant vous avez /home/ Ajoutez ensuite u après /home/ et appuyez sur Ctrlx puis Ctrlf . Maintenant, vous avez /home/ubuntu/ car ce chemin est unique. Maintenant, écrivez my après /home/ubuntu/ et appuyez sur Ctrlx puis Ctrlf . L’éditeur complétera votre mot jusqu’à my_folder_ et vous verrez l’arborescence des répertoires, utilisez donc les touches fléchées pour choisir celui que vous voulez.

Affichage des numéros de ligne #

Pour afficher les numéros de ligne à partir de la vue par défaut, entrez

:set number

Pour masquer les numéros de ligne

:set nonumber

Il existe également un raccourci pour ci-dessus. ’nu’ est identique à ’nombre'.

:set nonu

Pour masquer les numéros de ligne, nous pouvons également utiliser

:set nu!

Écrivez un fichier si vous oubliez de sudo avant de démarrer vim #

Cette commande enregistrera le fichier ouvert avec les droits sudo

:w !sudo tee % >/dev/null

Vous pouvez également mapper w !! pour écrire un fichier en tant que root

:cnoremap w!! w !sudo tee % >/dev/null

Macros “jetables” rapides #

Ajoutez ceci à votre vimrc :

nnoremap Q @q

Pour démarrer l’enregistrement de la macro “jetable”, utilisez qq. Pour terminer l’enregistrement, appuyez sur q (en mode normal pour les deux).

Pour exécuter la macro enregistrée, utilisez Q.

Ceci est utile pour les macros que vous devez répéter plusieurs fois de suite mais qui ne seront probablement pas réutilisées par la suite.

Comment connaître le répertoire et/ou le chemin du fichier que vous éditez #

Pour connaître le chemin du répertoire dans lequel se trouve votre fichier, vous pouvez utiliser :

Échap pour entrer en mode commande

:pwd

Cela imprimera le chemin d’accès au répertoire en bas de l’éditeur, comme ceci

I'm a ninja
~
~
~
~
~
/home/ubuntu/myfolder                                                1,5         All

Maintenant, si vous voulez connaître le nom du fichier que vous éditez relativement au répertoire de travail vim, vous pouvez utiliser :

Échap pour entrer en mode commande CTRLG

I'm a ninja
~
~
~
~
~
"myfile"[Modified][New file] 1 line --100%--                         1,5         All

Enfin, pour obtenir le chemin absolu du fichier que vous modifiez, utilisez

Esc pour entrer en mode commande,

1 puis CTRLG

I'm a ninja
~
~
~
~
~
"~/myfolder/myfile"[Modified][New file] 1 line --100%--              1,5         All

Recharge automatiquement vimrc lors de l’enregistrement #

Pour recharger automatiquement vimrc lors de l’enregistrement, ajoutez ce qui suit à votre vimrc :

if has('autocmd') " ignore this section if your vim does not support autocommands
    augroup reload_vimrc
        autocmd!
        autocmd! BufWritePost $MYVIMRC,$MYGVIMRC nested source %
    augroup END
endif

puis pour la dernière fois, tapez :

:so $MYVIMRC

La prochaine fois que vous enregistrerez votre vimrc, il sera automatiquement rechargé.

nested est utile si vous utilisez vim-airline. Le processus de chargement de la compagnie aérienne déclenche certaines autocommandes, mais puisque vous êtes en train d’exécuter une autocommande, elles sont ignorées. nested permet de déclencher des autocommandes imbriquées et permet à la compagnie aérienne de se charger correctement.

Achèvement de la ligne de commande #

définissez wildmenu pour activer les suggestions de complétion pour la ligne de commande. Exécutez ce qui suit

set wildmenu
set wildmode=list:longest,full

Maintenant, si vous dites, :color tab,

Tu auras

256-jungle Benokai BlackSea C64 CandyPaper Chasing_Logic ChocolateLiquor :couleur 0x7A69_dark

Recherche dans un bloc fonction #

Pour rechercher le texte foo dans un bloc {} entourant le curseur, utilisez la commande suivante (<ESC> - touche d’échappement, <CR> - touche d’entrée) :

vi{<ESC>/\%Vfoo<CR>

maintenant vous pouvez sauter entre les correspondances dans le bloc en appuyant sur n et p. Si l’option hlsearch est activée, toutes les correspondances seront mises en surbrillance. \%V est une partie spéciale du motif de recherche, qui indique à vim de rechercher uniquement dans la zone sélectionnée visuellement. Vous pouvez également faire un mappage comme celui-ci :

:vnoremap g/ <ESC>/\%V

Après cela, la commande ci-dessus est raccourcie comme suit :

vi{g/foo<CR>

Une autre astuce utile consiste à imprimer toutes les lignes contenant le motif :

vi{
:'<,'>g/foo/#

La plage '<,'> est insérée automatiquement.

Voir :help range et :help :g.

Copier, déplacer ou supprimer la ligne trouvée #

De nombreux utilisateurs se retrouvent dans une situation où ils souhaitent simplement copier, déplacer ou supprimer une ligne rapidement et revenir là où ils se trouvaient.

Habituellement, si vous souhaitez déplacer une ligne contenant le mot “lot” sous la ligne actuelle, vous devez taper quelque chose comme :

/lot<Esc>dd<C-o>p

Mais pour augmenter la productivité, vous pouvez utiliser ce raccourci dans ces cas :

" It's recommended to turn on incremental search when doing so
set incsearch

" copy the found line
cnoremap $t <CR>:t''<CR>
" move the found line
cnoremap $m <CR>:m''<CR>
" delete the found line
cnoremap $d <CR>:d<CR>``

Donc une recherche comme celle-ci :

/lot$m

déplacerait la ligne contenant “lot” sous la ligne sur laquelle se trouvait votre curseur lorsque vous avez lancé la recherche.
