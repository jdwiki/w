Un plugin est un script ou un ensemble de scripts qui modifie le comportement par défaut de Vim, soit en ajoutant des fonctionnalités non existantes, soit en étendant des fonctionnalités existantes.

Les “fonctionnalités non existantes” souvent ajoutées incluent :

  • commenter,
  • détection d’indentation,
  • l’auto-complétion,
  • correspondance floue,
  • prise en charge d’une langue spécifique,
  • etc.

Les “fonctionnalités existantes” souvent étendues incluent :

  • omni-complétion,
  • objets-texte et mouvements,
  • tirer et mettre,
  • ligne d’état,
  • rechercher et remplacer,
  • changement de tampon/fenêtre/page d’onglet,
  • pliant,
  • etc.

Lapin #

Vundle est un gestionnaire de plugins pour Vim.

Installer Vundle #

(Les détails complets de l’installation peuvent être trouvés dans Vundle Quick Start)

  1. Installez Git et clonez Vundle dans ~/.vim/bundle/Vundle.vim.

  2. Configurez les plugins en ajoutant ce qui suit en haut de votre .vimrc, en ajoutant ou en supprimant des plugins si nécessaire (les plugins de la liste sont simplement à des fins d’illustration)

     set nocompatible              " be iMproved, required
 filetype off                  " required

 " set the runtime path to include Vundle and initialize
 set rtp+=~/.vim/bundle/Vundle.vim
 call vundle#begin()
 " alternatively, pass a path where Vundle should install plugins
 "call vundle#begin('~/some/path/here')

 " let Vundle manage Vundle, required
 Plugin 'VundleVim/Vundle.vim'

 " All of your Plugins must be added before the following line
 call vundle#end()            " required
 filetype plugin indent on    " required
 " To ignore plugin indent changes, instead use:
 "filetype plugin on

 "place non-Plugin stuff after this line

  3. Installez les plugins : en lançant Vim et en exécutant :PluginInstall.

Formats de plug-ins pris en charge #

Voici des exemples de différents formats pris en charge. Gardez les commandes du plugin entre vundle#begin et vundle#end.

Emplacement du plugin Utilisation
plugin sur GitHub Plugin 'tpope/vim-fugitif'
plugin de http://vim-scripts.org/vim/scripts.html Plug-in 'L9'
Plugin Git non hébergé sur GitHub Plug-in 'git://git.wincent.com/command-t.git'
git repos sur votre machine locale (c’est-à-dire lorsque vous travaillez sur votre propre plugin) Plugin 'file:///home/gmarik/path/to/plugin'
Le script vim de démarrage se trouve dans un sous-répertoire de ce référentiel appelé vim. Transmettez le chemin pour définir correctement le chemin d’exécution. Plug-in 'rstacruz/sparkup', {'rtp': 'vim/'}
Installez L9 et évitez un conflit de nommage si vous avez déjà installé une version différente ailleurs. Plugin 'ascenator/L9', {'name': 'newL9'}

Travailler sur un compte partagé, par exemple, sur le nœud principal du cluster peut soulever des problèmes au niveau de l’utilisation du disque par le répertoire .vim. Il existe quelques packages qui occupent une quantité considérable d’espace disque, par exemple YCM. Veuillez donc choisir judicieusement votre répertoire de plug-ins Vundle, et il est très facile de le faire en définissant rtp. Et aussi si vous envisagez d’installer un plugin vim, ne faites pas directement git clone dans le répertoire bundle. Utilisez la voie Vundle.

Agent pathogène #

[vim-pathogen][1] est un gestionnaire de runtimepath créé par Tim Pope pour faciliter l’installation de plugins et de fichiers d’exécution dans leurs propres répertoires privés.

Installation du pathogène #

  1. Mettez pathogen dans ~/.vim/bundle (ici avec Git, mais ce n’est pas obligatoire):

    git clone https://github.com/tpope/vim-pathogen.git

  2. Ajoutez les lignes suivantes en haut de votre .vimrc :

     " enable vim-pathogen
 runtime bundle/vim-pathogen/autoload/pathogen.vim
 execute pathogen#infect()
  • la directive runtime spécifie le chemin vers le script de chargement automatique de vim-pathogen ;
  • execute pathogen#infect() le lance.

Une fois lancé, Pathogen lancera automatiquement un balayage dans les dossiers de ~/.vim/bundle et chargera le plugin à partir de chacun d’eux.

Utiliser un agent pathogène #

  1. Placez le répertoire de niveau supérieur de votre plugin dans ~/.vim/bundle/ pour le rendre disponible la prochaine fois que vous démarrerez Vim.
  2. Lancez :Helptags pour indexer la documentation de votre nouveau plugin.

Avantages #

  • Chaque plugin réside dans son propre répertoire sous ~/.vim/bundle/.
  • Votre .vimrc reste propre de la configuration nécessaire pour charger les plugins.

L’effort nécessaire pour “gérer” un plugin est ainsi réduit à :

  • mettez son répertoire de niveau supérieur sous ~/.vim/bundle/ pour l’installer,
  • remplacer son répertoire de niveau supérieur pour le mettre à jour,
  • supprimez son répertoire de niveau supérieur pour le désinstaller.

La façon dont vous effectuez ces trois actions (manuellement, via un outil d’automatisation, avec Git/Svn/Hg/whatever…) dépend entièrement de vous.

[1] : https://github.com/tpope/vim-pathogen

Comment fonctionnent les plugins #

Un plugin peut se présenter comme un seul fichier contenant 30 lignes de vimscript ou comme 20 Mo de vimscript/python/ruby/whatever divisé en plusieurs fichiers dans une douzaine de répertoires qui dépendent d’un certain nombre d’outils externes.

Le premier est évidemment facile à installer et à gérer, mais le second pourrait mettre en pause tout un défi.

Le principe #

L’option 'runtimepath' indique à Vim où chercher les scripts d’exécution. La valeur par défaut oblige Vim à rechercher des scripts dans les répertoires suivants dans l’ordre :

  • sur les systèmes de type UNIX
  • $HOME/.vim/
  • $VIM/vimfiles/
  • $VIMRUNTIME/
  • $VIM/vimfiles/after/
  • $HOME/.vim/after/
  • sous Windows
  • $HOME/vimfiles/
  • $VIM/vimfiles/
  • $VIMRUNTIME/
  • $VIM/vimfiles/after/
  • $HOME/vimfiles/after/

Parmi les répertoires ci-dessus, installez uniquement les plugins dans ceux en gras. Les autres causeront de l’instabilité sans raison valable. Installer un plugin revient à placer chacun de ses composants dans le bon répertoire sous $HOME/.vim/ ou $HOME/vimfiles/.

La méthode manuelle #

Plugin de fichier unique #

Placez le fichier sous $HOME/.vim/plugin ou $HOME/vimfiles/plugin

Cela sourcerait le plugin au démarrage de Vim. Désormais, l’utilisateur peut utiliser tout ce qui y est défini. Si le plugin doit cependant être activé, l’utilisateur doit soit exécuter la commande lui-même chaque fois qu’il veut l’utiliser, soit ajouter la commande à .vimrc

Paquet #

Un bundle est une structure de répertoires utilisée par le plugin. Il se compose de tous les fichiers du plugin sous les sous-répertoires appropriés.

Pour installer un tel plugin, les sous-répertoires doivent être fusionnés avec leurs homologues dans $HOME/.vim/plugin. Cette approche conduit cependant à mélanger les fichiers de différents plugins dans les mêmes répertoires et pourrait éventuellement conduire à des problèmes d’espace de noms.

Une autre approche consiste à copier le répertoire entier dans $HOME/.vim/bundle.

Lorsque vous utilisez cette approche, il devrait y avoir au moins un fichier .vim sous le répertoire $HOME/.vim/bundle/autoload. Ces fichiers seraient sourcés par vim au démarrage.

Remarque : Selon le système d’exploitation de l’utilisateur, le préfixe de tous les chemins peut être $HOME/vimfiles. Pour plus de détails, voir [Comment fonctionnent les plugins] [1]

[1] : https://www.wikiod.com/fr/vim/extension-de-vim#Comment fonctionnent les plugins
