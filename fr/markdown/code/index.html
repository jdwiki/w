Syntaxe #

  • inline : `Code entre backticks` ou <code>Code entre ces balises HTML</code>
  • multiline : ____Code qui est indenté sur chaque ligne ou <pre><code>Plusieurs lignes de code entre ces balises HTML</code></pre>

Code en ligne #

Markdown prend en charge l’ajout de code en ligne “comme celui-ci”, obtenu en enveloppant le texte dans des backticks :

`code ici`

Alternativement, vous pouvez placer votre code en ligne entre les balises HTML <code> et </code>.

Considérez le code de démarquage suivant :

`This` is an inline code block! <code>This</code> is one too!

Cela produirait la sortie suivante :

“Ceci” est un bloc de code en ligne ! Ceci en est un aussi !

Si vous avez besoin d’inclure un backtick dans le code en ligne, vous pouvez utiliser plusieurs backticks pour commencer et terminer le bloc de code en ligne, comme ceci :

``code containing a backtick (`) character``

Cela produirait la sortie suivante :

code contenant un caractère backtick (`)

Utilisez \ pour échapper les backticks. Par exemple:

\`a\`

sera rendu comme

`a`

Coloration syntaxique (StackExchange) #

Sur les sites StackExchange, les extraits de code peuvent fournir une coloration syntaxique facultative. Sur des sites comme Stack Overflow, la langue par défaut est dérivée des balises utilisées dans la question associée (le cas échéant). De plus, le langage de coloration syntaxique d’un extrait de code peut également être [défini en ajoutant un commentaire HTML au corps du texte][1].

<!-- language: lang-vb -->

    Sub ShowVB()
    Dim i As Long
    For i = 1 To 2
        If i = 3 Then
            MsgBox "How did that happen?"
        End If
    Next
    End Sub

Un tel commentaire changera le langage de coloration syntaxique pour tous les extraits de code suivants, ce qui peut être plutôt utile, en particulier lorsque plusieurs langues sont impliquées dans un article.

Ce qui précède sera rendu avec la mise en surbrillance Visual Basic comme :

Sub ShowVB()
Dim i As Long
For i = 1 To 2
    If i = 3 Then
        MsgBox "How did that happen?"
    End If
Next
End Sub

[1] : http://meta.stackexchange.com/questions/184108/what-is-syntax-highlighting-and-how-does-it-work/184109#184109

Blocs de code clôturé #

Certains analyseurs permettent de désigner le code en ajoutant trois backticks avant et après une section de code.

```
<p><em>This</em> is an HTML example!</p>
```

En option, de nombreux analyseurs permettent d’ajouter la coloration syntaxique en spécifiant le langage du code immédiatement après le premier jeu de backticks :

```html
<p><em>This</em> is an HTML example!</p>
```

Résultat:

<p><em>This</em> is an HTML example!</p>

Blocs de code indentés à l’intérieur des listes #

Lors de l’ajout de blocs de code indentés dans une liste, vous avez d’abord besoin d’une ligne vide, puis d’indenter davantage le code. Différentes saveurs de Markdown ont des règles différentes pour cela.

  1. StackExchange exige que le code soit indenté de 8 caractères au lieu des 4 habituels. (Espaces remplacés par * pour plus de clarté) :

     1.*Listitem1
 2.*Listitem2

 ********code here
 3.*Listitem3

  2. Les spécifications telles que CommonMark exigent que le bloc de code soit indenté de 4 caractères à partir du début du texte de l’élément de liste. (Espaces remplacés par * pour plus de clarté) :

     1.****Listitem1
 2.****Listitem2

 **********code here
 3.****Listitem3

Blocs de code indentés #

Vous pouvez créer des extraits de code multilignes en indentant chaque ligne avec au moins quatre espaces ou une tabulation :

    #include <stdio.h>

    int main() {
      printf("Hello World!\n");
      return 0;
    }
