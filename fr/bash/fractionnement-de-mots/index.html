Syntaxe #

  • Définissez IFS sur la nouvelle ligne : IFS=$’\n'
  • Définissez IFS sur nullstring : IFS=
  • Définissez IFS sur / caractère : IFS=/

Paramètres #

Paramètre Détails
IFS Séparateur de champ interne
-x Afficher les commandes et leurs arguments au fur et à mesure de leur exécution (option Shell)
  • Le fractionnement de mots n’est pas effectué pendant les affectations, par exemple newvar=$var
  • Le fractionnement de mots n’est pas effectué dans la construction [[ ... ]]
  • Utilisez des guillemets doubles sur les variables pour éviter le fractionnement des mots

Mauvais effets de la division des mots #

$ a='I am a string with spaces'
$ [ $a = $a ] || echo "didn't match"
bash: [: too many arguments
didn't match

[ $a = $a ] était interprété comme [ je suis une chaîne avec des espaces = je suis une chaîne avec des espaces ]. [ est la commande test pour laquelle Je suis une chaîne avec des espaces n’est pas un seul argument, mais plutôt 6 arguments !!

$ [ $a = something ] || echo "didn't match"
bash: [: too many arguments
didn't match

[ $a = quelque chose ] était interprété comme [ Je suis une chaîne avec des espaces = quelque chose ]

$ [ $(grep . file) = 'something' ]
bash: [: too many arguments

La commande grep renvoie une chaîne multiligne avec des espaces, vous pouvez donc imaginer combien d’arguments il y a… :D

Voir [quoi, quand et pourquoi](https://www.wikiod.com/fr/bash/fractionnement-de-mots#Quoi, quand et pourquoi ?) pour les bases.

Fractionnement avec IFS #

Pour être plus clair, créons un script nommé showarg :

#!/usr/bin/env bash
printf "%d args:" $#
printf " <%s>" "$@"
echo

Voyons maintenant les différences :

$ var="This is an example"
$ showarg $var
4 args: <This> <is> <an> <example>

$var est divisé en 4 arguments. IFS est composé d’espaces blancs et donc le fractionnement des mots s’est produit dans les espaces

$ var="This/is/an/example"
$ showarg $var
1 args: <This/is/an/example>

Dans le mot ci-dessus, le fractionnement n’a pas eu lieu car les caractères IFS n’ont pas été trouvés.

Maintenant, définissons IFS=/

$ IFS=/
$ var="This/is/an/example"
$ showarg $var
4 args: <This> <is> <an> <example>

Le $var est divisé en 4 arguments et non en un seul argument.

Quoi, quand et pourquoi ? #

Lorsque le shell effectue une expansion de paramètre, une substitution de commande, une expansion de variable ou arithmétique, il recherche les limites de mots dans le résultat. Si une limite de mot est trouvée, le résultat est divisé en plusieurs mots à cette position. La frontière du mot est définie par une variable shell IFS (Internal Field Separator). Les valeurs par défaut pour IFS sont l’espace, la tabulation et le retour à la ligne, c’est-à-dire que le fractionnement des mots se produira sur ces trois caractères d’espace blanc s’il n’est pas explicitement empêché.

set -x
var='I am
a
multiline string'
fun() {
    echo "-$1-"
    echo "*$2*"
    echo ".$3."
}
fun $var

Dans l’exemple ci-dessus, voici comment la fonction “fun” est exécutée :

fun I am a multiline string

$var est divisé en 5 arguments, seuls I, am et a seront imprimés.

IFS et fractionnement de mots #

  • Voir [quoi, quand et pourquoi] (https://www.wikiod.com/fr/bash/fractionnement-de-mots#Quoi, quand et pourquoi ?) si vous ne connaissez pas le affiliation de l’IFS au word splitting*

définissons l’IFS sur le caractère d’espacement uniquement :

set -x
var='I am
a
multiline string'
IFS=' '
fun() {
    echo "-$1-"
    echo "*$2*"
    echo ".$3."
}
fun $var

Cette fois, le fractionnement des mots ne fonctionnera que sur les espaces. La fonction fun sera exécutée comme ceci :

fun I 'am
a
multiline' string

$var est divisé en 3 arguments. I, am\na\nmultiline et string seront imprimés

Définissons l’IFS sur nouvelle ligne uniquement :

IFS=$'\n'
...

Maintenant, le fun sera exécuté comme :

fun 'I am' a 'multiline string'

$var est divisé en 3 arguments. Je suis, a, chaîne multiligne seront imprimés

Voyons ce qui se passe si nous définissons IFS sur nullstring :

IFS=
...

Cette fois, le “fun” sera exécuté comme ceci :

fun 'I am
a
multiline string'

$var n’est pas divisé, c’est-à-dire qu’il est resté un seul argument.

** Vous pouvez empêcher le fractionnement des mots en définissant l’IFS sur nullstring **

Un moyen général d’empêcher la séparation des mots consiste à utiliser des guillemets doubles :

fun "$var"

empêchera la division des mots dans tous les cas évoqués ci-dessus, c’est-à-dire que la fonction “fun” sera exécutée avec un seul argument.

Utilité du fractionnement de mots #

Il existe certains cas où le fractionnement de mots peut être utile :

Remplir le tableau :

arr=($(grep -o '[0-9]\+' file))

Cela remplira arr avec toutes les valeurs numériques trouvées dans file

Parcours de mots séparés par des espaces :

words='foo bar baz'
for w in $words;do
    echo "W: $w"
done

Production:

W: foo
W: bar
W: baz

Passer des paramètres séparés par des espaces qui ne contiennent pas d’espaces blancs :

packs='apache2 php php-mbstring php-mysql'
sudo apt-get install $packs

ou

packs='
apache2
php
php-mbstring
php-mysql
'
sudo apt-get install $packs

Cela installera les packages. Si vous doublez le $packs entre guillemets, cela générera une erreur.

Unquoetd $packs envoie tous les noms de paquets séparés par des espaces en tant qu’arguments à apt-get, tandis que la citation enverra la chaîne $packs en tant qu’argument unique, puis apt-get essaiera d’installer un package nommé apache2 php php-mbstring php-mysql (pour le premier) qui n’existe évidemment pas

Voir [quoi, quand et pourquoi](https://www.wikiod.com/fr/bash/fractionnement-de-mots#Quoi, quand et pourquoi ?) pour les bases.
