Exécuter la commande avec le document ici #

ssh -p 21 example@example.com <<EOF
  echo 'printing pwd'
  echo "\$(pwd)"
  ls -a
  find '*.txt'
EOF

$ est échappé car nous ne voulons pas qu’il soit développé par le shell actuel, c’est-à-dire que $(pwd) doit être exécuté sur le shell distant.

Autrement:

ssh -p 21 example@example.com <<'EOF'
  echo 'printing pwd'
  echo "$(pwd)"
  ls -a
  find '*.txt'
EOF

Remarque : L’EOF de fermeture devrait être au début de la ligne (pas d’espaces avant). Si l’indentation est requise, des tabulations peuvent être utilisées si vous commencez votre heredoc par <<-. Voir les [Documents d’indentation ici](https://www.wikiod.com/fr/bash/ici-les-documents-et-ici-les-chaines#Indenter ici les documents) et [Limit Strings](http:// stackoverflow.com/documentation/bash/655/here-documents-and-here-strings/12587/limit-strings) exemples pour plus d’informations.

Indentation ici des documents #

Vous pouvez indenter le texte à l’intérieur de ces documents avec des tabulations, vous devez utiliser l’opérateur de redirection <<- au lieu de << :

$ cat <<- EOF
    This is some content indented with tabs `\t`.
    You cannot indent with spaces you __have__ to use tabs.
    Bash will remove empty space before these lines.
    __Note__: Be sure to replace spaces with tabs when copying this example.
EOF

This is some content indented with tabs _\t_.
You cannot indent with spaces you __have__ to use tabs.
Bash will remove empty space before these lines.
__Note__: Be sure to replace spaces with tabs when copying this example.

Un cas d’utilisation pratique de ceci (comme mentionné dans man bash) est dans les scripts shell, par exemple :

if cond; then
    cat <<- EOF
    hello
    there
    EOF
fi

Il est d’usage d’indenter les lignes dans les blocs de code comme dans cette instruction “if”, pour une meilleure lisibilité. Sans la syntaxe de l’opérateur <<-, nous serions obligés d’écrire le code ci-dessus comme ceci :

if cond; then
    cat << EOF
hello
there
EOF
fi

C’est très désagréable à lire, et c’est bien pire dans un script réaliste plus complexe.

Ici les chaînes #

Vous pouvez alimenter une commande en utilisant ici des chaînes comme celle-ci :

$ awk '{print $2}' <<< "hello world - how are you?"
world

$ awk '{print $1}' <<< "hello how are you
> she is fine"
hello
she

Vous pouvez également alimenter une boucle while avec une chaîne ici :

$ while IFS=" " read -r word1 word2 rest
> do
> echo "$word1"
> done <<< "hello how are you - i am fine"
hello

Créer un fichier #

Une utilisation classique d’ici documents est de créer un fichier en tapant son contenu :

cat > fruits.txt << EOF
apple
orange
lemon
EOF

Le document ici est les lignes entre << EOF et EOF.

Ce document ici devient l’entrée de la commande cat. La commande cat affiche simplement son entrée, et en utilisant l’opérateur de redirection de sortie > nous redirigeons vers un fichier fruits.txt.

Par conséquent, le fichier fruits.txt contiendra les lignes :

apple
orange
lemon

Les règles habituelles de redirection de sortie s’appliquent : si fruits.txt n’existait pas auparavant, il sera créé. S’il existait auparavant, il sera tronqué.

Exécuter plusieurs commandes avec sudo #

sudo -s <<EOF
  a='var'
  echo 'Running serveral commands with sudo'
  mktemp -d
  echo "\$a"
EOF
  • $a doit être échappé pour éviter qu’il ne soit étendu par le shell actuel

Ou

sudo -s <<'EOF'
  a='var'
  echo 'Running serveral commands with sudo'
  mktemp -d
  echo "$a"
EOF

Limiter les chaînes #

Un heredoc utilise le limitstring pour déterminer quand arrêter de consommer l’entrée. La chaîne limite de terminaison doit

  • Soyez au début d’une ligne.
  • Soyez le seul texte sur la ligne Remarque : Si vous utilisez <<-, la chaîne limite peut être précédée de tabulations \t

Corriger:

cat <<limitstring
line 1
line 2
limitstring

Cela affichera :

ligne 1 ligne 2

Utilisation incorrecte :

cat <<limitstring
line 1
line 2
 limitstring

Étant donné que limitstring sur la dernière ligne n’est pas exactement au début de la ligne, le shell continuera d’attendre d’autres entrées jusqu’à ce qu’il voie une ligne commençant par limitstring et ne contenant rien d’autre. Ce n’est qu’alors qu’il arrêtera d’attendre une entrée et passera le document ici à la commande cat.

Notez que lorsque vous préfixez la chaîne limite initiale avec un trait d’union, tous les onglets au début de la ligne sont supprimés avant l’analyse, de sorte que les données et la chaîne limite peuvent être indentées avec des onglets (pour faciliter la lecture dans les scripts shell).

cat <<-limitstring
        line 1    has a tab each before the words line and has
            line 2 has two leading tabs
        limitstring

produira

la ligne 1 a une tabulation avant la ligne de mots et a la ligne 2 a deux tabulations principales

avec les onglets principaux (mais pas les onglets internes) supprimés.
