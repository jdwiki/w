Syntaxe #

  • [heure [-p]] [!] commande1 [ | ou |& commande2 ] …

Un pipeline est une séquence de commandes simples séparées par l’un des opérateurs de contrôle | ou |& ([source][1]).

| connecte la sortie de command1 à l’entrée de command2.

|& connecte la sortie standard et l’erreur standard de command1 à l’entrée standard de command2.

[1] : https://www.gnu.org/software/bash/manual/html_node/Pipelines.html

Utiliser |& #

|& connecte la sortie standard et l’erreur standard de la première commande à la seconde tandis que | ne connecte que la sortie standard de la première commande à la seconde commande.

Dans cet exemple, la page est téléchargée via curl. avec l’option -v curl écrit des informations sur stderr y compris , la page téléchargée est écrite sur stdout. Le titre de la page peut être trouvé entre <title> et </title>.

curl -vs 'http://www.google.com/' |& awk '/Host:/{print} /<title>/{match($0,/<title>(.*)<\/title>/,a);print a[1]}'

La sortie est :

> Host: www.google.com
Google

Mais avec |, beaucoup plus d’informations seront imprimées, c’est-à-dire celles qui sont envoyées à stderr car seule stdout est redirigée vers la commande suivante. Dans cet exemple, toutes les lignes sauf la dernière (Google) ont été envoyées à stderr par curl :

Afficher tous les processus paginés #

ps -e | less

ps -e affiche tous les processus, sa sortie est connectée à l’entrée de more via |, less pagine les résultats.

Modifier la sortie continue d’une commande #

~$ ping -c 1 google.com # unmodified output
PING google.com (16.58.209.174) 56(84) bytes of data.
64 bytes from wk-in-f100.1e100.net (16.58.209.174): icmp_seq=1 ttl=53 time=47.4 ms
~$ ping google.com | grep -o '^[0-9]\+[^()]\+' # modified output
64 bytes from wk-in-f100.1e100.net 
64 bytes from wk-in-f100.1e100.net 
64 bytes from wk-in-f100.1e100.net 
64 bytes from wk-in-f100.1e100.net 
64 bytes from wk-in-f100.1e100.net 
64 bytes from wk-in-f100.1e100.net 
64 bytes from wk-in-f100.1e100.net 
64 bytes from wk-in-f100.1e100.net 
64 bytes from wk-in-f100.1e100.net 
64 bytes from wk-in-f100.1e100.net 
...

Le tube (|) connecte le stdout de ping au stdin de grep, qui le traite immédiatement. Certaines autres commandes telles que sed mettent par défaut en mémoire tampon leur stdin, ce qui signifie qu’elles doivent recevoir suffisamment de données avant d’imprimer quoi que ce soit, ce qui peut entraîner des retards dans le traitement ultérieur.
