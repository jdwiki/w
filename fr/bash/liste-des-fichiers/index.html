Syntaxe #

  • ls [OPTION]… [FICHIER]…

Paramètres #

Options Descriptif
-a, --tout Liste toutes les entrées, y compris celles qui commencent par un point
-A, --presque-tout Lister toutes les entrées sauf . et ..
-c Trier les fichiers par heure de changement
-d, --répertoire Lister les entrées du répertoire
-h, --lisible par l'homme Afficher les tailles dans un format lisible par l’homme (c’est-à-dire ‘K’, ‘M’)
-H Comme ci-dessus uniquement avec des puissances de 1000 au lieu de 1024
-l Afficher le contenu au format longue liste
-o Format de liste longue sans informations de groupe
-r, --reverse Afficher le contenu dans l’ordre inverse
-s, --taille Taille d’impression de chaque fichier en blocs
-S Trier par taille de fichier
--sort=MOT Trier le contenu par mot. (c’est-à-dire taille, version, statut)
-t Trier par heure de modification
-u Trier par heure du dernier accès
-v Trier par version
-1 Lister un fichier par ligne

Lister les fichiers dans un format de liste longue #

L’option -l de la commande ls imprime le contenu d’un répertoire spécifié dans un format de liste longue. Si aucun répertoire n’est spécifié, alors, par défaut, le contenu du répertoire courant est répertorié.

ls -l /etc

Exemple de sortie :

total 1204
drwxr-xr-x  3 root root    4096 Apr 21 03:44 acpi
-rw-r--r--  1 root root    3028 Apr 21 03:38 adduser.conf
drwxr-xr-x  2 root root    4096 Jun 11 20:42 alternatives
...

La sortie affiche d’abord “total”, qui indique la taille totale en blocs de tous les fichiers du répertoire répertorié. Il affiche ensuite huit colonnes d’informations pour chaque fichier dans le répertoire répertorié. Vous trouverez ci-dessous les détails de chaque colonne de la sortie :

N° de colonne Exemple La description
1.1 d Type de fichier (voir tableau ci-dessous)
1.2 rwxr-xr-x Chaîne d’autorisation
2 3 Nombre de liens durs
3 root Nom du propriétaire
4 root Groupe propriétaire
5 4096 Taille du fichier en octets
6 21 avril 03:44 Heure de modification
7 acpi Nom de fichier

Type de fichier #

Le type de fichier peut être l’un des caractères suivants.

Caractère Type de fichier
- Fichier normal
b Bloquer le fichier spécial
c Fichier spécial personnage
C Fichier hautes performances (“données contiguës”)
d Répertoire
D Porte (fichier IPC spécial dans Solaris 2.5+ uniquement)
l Lien symbolique
M Fichier hors ligne (“migré”) (Cray DMF)
n Fichier spécial réseau (HP-UX)
p FIFO (tube nommé)
P Port (fichier système spécial dans Solaris 10+ uniquement)
s Prise
? Un autre type de fichier

Lister les dix fichiers les plus récemment modifiés #

Ce qui suit répertorie jusqu’à dix des fichiers les plus récemment modifiés dans le répertoire actuel, en utilisant un format de liste longue (-l) et triés par heure (-t).

ls -lt | head

Lister tous les fichiers, y compris les fichiers de points #

Un dotfile est un fichier dont le nom commence par un .. Ceux-ci sont normalement masqués par ls et ne sont pas répertoriés, sauf sur demande.

Par exemple, la sortie suivante de ls :

$ ls
bin  pki

L’option -a ou --all listera tous les fichiers, y compris les dotfiles.

$ ls -a
.   .ansible       .bash_logout   .bashrc  .lesshst  .puppetlabs  .viminfo
..  .bash_history  .bash_profile  bin      pki       .ssh

L’option -A ou --almost-all listera tous les fichiers, y compris les dotfiles, mais ne listera pas implicitement . et ... Notez que . est le répertoire courant et .. est le répertoire parent.

$ ls -A 
.ansible       .bash_logout   .bashrc  .lesshst  .puppetlabs  .viminfo
.bash_history  .bash_profile  bin      pki       .ssh

Lister les fichiers #

La commande ls répertorie le contenu d’un répertoire spécifié, excluant les fichiers de points. Si aucun répertoire n’est spécifié, alors, par défaut, le contenu du répertoire courant est répertorié.

Les fichiers répertoriés sont triés par ordre alphabétique, par défaut, et alignés en colonnes s’ils ne tiennent pas sur une seule ligne.

$ ls
apt  configs  Documents  Fonts    Music      Programming  Templates  workspace
bin  Desktop  eclipse    git      Pictures   Public       Videos

Lister les fichiers sans utiliser ls #

Utilisez [l’extension du nom de fichier] du shell Bash (https://www.gnu.org/software/bash/manual/bashref.html#Filename-Expansion) et [l’extension de l’accolade] (https://www.gnu.org/software /bash/manual/bashref.html#Brace-Expansion) pour obtenir les noms de fichiers :

# display the files and directories that are in the current directory
printf "%s\n" *

# display only the directories in the current directory
printf "%s\n" */

# display only (some) image files
printf "%s\n" *.{gif,jpg,png}

Pour capturer une liste de fichiers dans une variable à traiter, il est généralement recommandé d’utiliser un tableau bash :

files=( * )

# iterate over them
for file in "${files[@]}"; do
    echo "$file"
done

Lister les fichiers dans un format arborescent #

La commande tree répertorie le contenu d’un répertoire spécifié dans un format arborescent. Si aucun répertoire n’est spécifié, alors, par défaut, le contenu du répertoire courant est répertorié.

Exemple de sortie :

$ tree /tmp
/tmp
├── 5037
├── adb.log
└── evince-20965
    └── image.FPWTJY.png

Utilisez l’option -L de la commande tree pour limiter la profondeur d’affichage et l’option -d pour ne lister que les répertoires.

Exemple de sortie :

$ tree -L 1 -d /tmp
/tmp
└── evince-20965

Liste les fichiers triés par taille #

L’option -S de la commande ls trie les fichiers par ordre décroissant de taille de fichier.

$ ls -l -S ./Fruits
total 444
-rw-rw-rw- 1 root root 295303 Jul 28 19:19 apples.jpg
-rw-rw-rw- 1 root root 102283 Jul 28 19:19 kiwis.jpg
-rw-rw-rw- 1 root root  50197 Jul 28 19:19 bananas.jpg

Lorsqu’il est utilisé avec l’option -r, l’ordre de tri est inversé.

$ ls -l -S -r /Fruits
total 444
-rw-rw-rw- 1 root root  50197 Jul 28 19:19 bananas.jpg
-rw-rw-rw- 1 root root 102283 Jul 28 19:19 kiwis.jpg
-rw-rw-rw- 1 root root 295303 Jul 28 19:19 apples.jpg
