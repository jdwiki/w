Un aperçu des variables internes de Bash, où, comment et quand les utiliser.

Les variables internes de Bash en un coup d’œil #

variables Détails
$* / $@ Paramètres positionnels de fonction/script (arguments). Développez comme suit :

$* et $@ sont identiques à $1 $2 ... (notez qu’il est généralement inutile de les laisser sans guillemets)
"$* " est identique à "$1 $2 ..." 1
"$@" est identique à "$1" "$2" ...
1. Les arguments sont séparés par le premier caractère de $IFS, qui ne doit pas nécessairement être un espace.
$# Nombre de paramètres positionnels passés au script ou à la fonction
$! ID de processus de la dernière commande (la plus à droite pour les pipelines) dans le travail le plus récemment mis en arrière-plan (notez qu’il n’est pas nécessairement le même que l’ID de groupe de processus du travail lorsque le contrôle des travaux est activé)
$$ ID du processus qui a exécuté bash
$? État de sortie de la dernière commande
$n Paramètres positionnels, où n=1, 2, 3, …, 9
${n} Paramètres positionnels (comme ci-dessus), mais n peut être > 9
$0 Dans les scripts, chemin avec lequel le script a été appelé ; avec bash -c 'printf "%s\n" "$0"' name args' : name (le premier argument après le script en ligne), sinon, le argv[0] que bash a reçu.
$_ Dernier champ de la dernière commande
$IFS Séparateur de champ interne
$CHEMIN Variable d’environnement PATH utilisée pour rechercher des exécutables
$OLDPWD Répertoire de travail précédent
$PWD Répertoire de travail actuel
$FUNCNAME Tableau de noms de fonctions dans la pile des appels d’exécution
$BASH_SOURCE Tableau contenant les chemins source des éléments du tableau FUNCNAME. Peut être utilisé pour obtenir le chemin du script.
$BASH_ALIASES Tableau associatif contenant tous les alias actuellement définis
$BASH_REMATCH Tableau des correspondances de la dernière correspondance de regex
$BASH_VERSION Chaîne de version bash
$BASH_VERSINFO Un tableau de 6 éléments avec les informations de version de Bash
$BASH Chemin absolu vers le shell Bash en cours d’exécution lui-même (déterminé de manière heuristique par bash basé sur argv[0] et la valeur de $PATH ; peut être erroné dans les cas extrêmes)
$BASH_SOUS-SHELL Niveau de sous-shell bash
$UID Réel (inefficace s’il est différent) ID utilisateur du processus exécutant bash
$PS1 Invite de ligne de commande principale ; voir [Utilisation des variables PS*][1]
$PS2 Invite de ligne de commande secondaire (utilisée pour une entrée supplémentaire)
$PS3 Invite de ligne de commande tertiaire (utilisée dans la boucle de sélection)
$PS4 Invite de ligne de commande quaternaire (utilisée pour ajouter des informations avec une sortie détaillée)
$RANDOM Un entier pseudo-aléatoire compris entre 0 et 32767
$RÉPONSE Variable utilisée par read par défaut lorsqu’aucune variable n’est spécifiée. Également utilisé par select pour renvoyer la valeur fournie par l’utilisateur
$PIPESTATUS Variable de tableau contenant les valeurs d’état de sortie de chaque commande dans le pipeline de premier plan le plus récemment exécuté.

L’affectation de variable ne doit pas avoir d’espace avant et après. a=123 et non a = 123. Ce dernier (un signe égal entouré d’espaces) signifie isolément exécuter la commande a avec les arguments = et 123, bien qu’il soit également vu dans l’opérateur de comparaison de chaînes (qui syntaxiquement est un argument de [ ou [[ ou le test que vous utilisez).

[1] : https://www.wikiod.com/fr/bash/gestion-de-linvite-systeme

## $@ "$@" s’étend à tous les arguments de la ligne de commande sous forme de mots séparés. Il est différent de "$*", qui s’étend à tous les arguments en un seul mot.

"$@" est particulièrement utile pour [faire une boucle][2] à travers les arguments et gérer les arguments avec des espaces.

Considérez que nous sommes dans un script que nous avons invoqué avec deux arguments, comme ceci :

$ ./script.sh "␣1␣2␣" "␣3␣␣4␣"

Les variables $* ou $@ se développeront en $1␣$2, qui à leur tour se développeront en 1␣2␣3␣4 d’où la boucle ci-dessous :

for var in $*; do # same for var in $@; do
    echo \<"$var"\>
done

imprimera pour les deux

<1>
<2>
<3>
<4>

Tandis que "$*" sera développé en "$1␣$2" qui à son tour se développera en "␣1␣2␣␣␣3␣␣4␣" et donc la boucle :

for var in "$*"; do
    echo \<"$var"\>   
done

n’invoquera echo qu’une seule fois et imprimera

<␣1␣2␣␣␣3␣␣4␣>

Et enfin "$@" se développera en "$1" "$2", qui se développera en "␣1␣2␣" "␣3␣␣4␣" et donc la boucle

for var in "$@"; do
    echo \<"$var"\>
done

va imprimer

<␣1␣2␣>
<␣3␣␣4␣>

préservant ainsi à la fois l’espacement interne des arguments et la séparation des arguments. Notez que la construction pour var dans "$@" ; do ... est si courant et si idiomatique qu’il s’agit de la valeur par défaut d’une boucle for et peut être raccourci en `for var; faire …'.

[1] : https://www.wikiod.com/fr/bash/tableaux [2] : https://www.wikiod.com/fr/bash/structures-de-controle#Boucler sur un tableau [3] : https://www.wikiod.com/fr/bash/tableaux

$# #

Pour obtenir le nombre d’arguments de ligne de commande ou de paramètres positionnels, tapez :

#!/bin/bash
echo "$#"

Lorsqu’il est exécuté avec trois arguments, l’exemple ci-dessus aboutira à la sortie :

~> $ ./testscript.sh firstarg secondarg thirdarg
3

$FUNCNAME #

Pour obtenir le nom de la fonction actuelle - tapez :

my_function()
{
    echo "This function is $FUNCNAME"    # This will output "This function is my_function"
}

Cette instruction ne renverra rien si vous la saisissez en dehors de la fonction :

my_function

echo "This function is $FUNCNAME"    # This will output "This function is"

$ACCUEIL #

Le répertoire personnel de l’utilisateur

~> $ echo $HOME
/home/user

$IFS #

Contient la chaîne de séparateur de champ interne que bash utilise pour diviser les chaînes lors de la boucle, etc. La valeur par défaut est les caractères d’espace blanc : \n (nouvelle ligne), \t (tabulation) et espace. Changer ceci en quelque chose d’autre vous permet de diviser les chaînes en utilisant différents caractères :

IFS=","
INPUTSTR="a,b,c,d"
for field in ${INPUTSTR}; do
    echo $field
done

La sortie de ce qui précède est :

a
b
c
d

Remarques:

  • Ceci est responsable du phénomène connu sous le nom de [word splitting](https://www.wikiod.com/fr/bash/fractionnement-de-mots#Quoi, quand et pourquoi ?).

## $OLDPWD OLDPWD (OLDPrintWorkingDirectory) contient le répertoire avant la dernière commande cd :

~> $ cd directory
directory> $ echo $OLDPWD
/home/user

## $PWD PWD (PrintWorkingDirectory) Le répertoire de travail actuel dans lequel vous vous trouvez actuellement :

~> $ echo $PWD
/home/user
~> $ cd directory
directory> $ echo $PWD
/home/user/directory

$1 $2 $3 etc… #

Paramètres positionnels transmis au script depuis la ligne de commande ou une fonction :

#!/bin/bash
# $n is the n'th positional parameter
echo "$1"
echo "$2"
echo "$3"

La sortie de ce qui précède est :

~> $ ./testscript.sh firstarg secondarg thirdarg
firstarg
secondarg
thirdarg

Si le nombre d’arguments de position est supérieur à neuf, des accolades doivent être utilisées.

#  "set -- " sets positional parameters
set -- 1 2 3 4 5 6 7 8 nine ten eleven twelve
# the following line will output 10 not 1 as the value of $1 the digit 1
# will be concatenated with the following 0
echo $10   # outputs 1
echo ${10} # outputs ten
# to show this clearly:
set -- arg{1..12}
echo $10 
echo ${10}

## $* Retournera tous les paramètres positionnels dans une seule chaîne.

testscript.sh :

#!/bin/bash
echo "$*"

Exécutez le script avec plusieurs arguments :

./testscript.sh firstarg secondarg thirdarg

Production:

firstarg secondarg thirdarg

## $ ! L’ID de processus (pid) de la dernière tâche exécutée en arrière-plan :

~> $ ls &
testfile1 testfile2
[1]+  Done                    ls
~> $ echo $!
21715

## $ ? L’état de sortie de la dernière fonction ou commande exécutée. Habituellement, 0 signifie OK, tout le reste indique un échec :

~> $ ls *.blah;echo $?
ls: cannot access *.blah: No such file or directory
2
~> $ ls;echo $?
testfile1 testfile2
0

$$ #

L’ID de processus (pid) du processus actuel :

~> $ echo $$
13246

## $HISTSIZE Le nombre maximum de commandes mémorisées :

~> $ echo $HISTSIZE
1000

## $BASHPID ID de processus (pid) de l’instance actuelle de Bash. Ce n’est pas la même chose que la variable $$, mais cela donne souvent le même résultat. Ceci est nouveau dans Bash 4 et ne fonctionne pas dans Bash 3.

~> $ echo "\$\$ pid = $$  BASHPID = $BASHPID"
$$ pid = 9265  BASHPID = 9265

$BASH_ENV #

Une variable d’environnement pointant vers le fichier de démarrage de Bash qui est lu lorsqu’un script est invoqué.

$BASH_VERSINFO #

Un tableau contenant les informations de version complètes divisées en éléments, beaucoup plus pratique que $BASH_VERSION si vous recherchez uniquement la version majeure :

~> $ for ((i=0; i<=5; i++)); do echo "BASH_VERSINFO[$i] = ${BASH_VERSINFO[$i]}"; done
BASH_VERSINFO[0] = 3
BASH_VERSINFO[1] = 2
BASH_VERSINFO[2] = 25
BASH_VERSINFO[3] = 1
BASH_VERSINFO[4] = release
BASH_VERSINFO[5] = x86_64-redhat-linux-gnu

$BASH_VERSION #

Affiche la version de bash en cours d’exécution, cela vous permet de décider si vous pouvez utiliser des fonctionnalités avancées :

~> $ echo $BASH_VERSION
4.1.2(1)-release

## $EDITEUR L’éditeur par défaut qui sera invoqué par tous les scripts ou programmes, généralement vi ou emacs.

~> $ echo $EDITOR
vi

$NOMHÔTE #

Nom d’hôte attribué au système lors du démarrage.

~> $ echo $HOSTNAME
mybox.mydomain.com

$TYPEHÔTE #

Cette variable identifie le matériel, elle peut être utile pour déterminer quels binaires exécuter :

~> $ echo $HOSTTYPE
x86_64

$MACHTYPE #

Semblable à $ HOSTTYPE ci-dessus, cela inclut également des informations sur le système d’exploitation ainsi que sur le matériel

~> $ echo $MACHTYPE
x86_64-redhat-linux-gnu

## $OSTYPE Renvoie des informations sur le type de système d’exploitation exécuté sur la machine, par exemple.

~> $ echo $OSTYPE
linux-gnu

$CHEMIN #

Le chemin de recherche pour trouver des binaires pour les commandes. Les exemples courants incluent /usr/bin et /usr/local/bin.

Lorsqu’un utilisateur ou un script tente d’exécuter une commande, les chemins dans $PATH sont recherchés afin de trouver un fichier correspondant avec l’autorisation d’exécution.

Les répertoires dans $PATH sont séparés par un caractère :.

~> $ echo "$PATH"
/usr/kerberos/bin:/usr/local/bin:/bin:/usr/bin

Ainsi, par exemple, étant donné le $PATH ci-dessus, si vous tapez lss à l’invite, le shell recherchera /usr/kerberos/bin/lss, puis /usr/local/bin/lss , puis /bin/lss, puis /usr/bin/lss, dans cet ordre, avant de conclure qu’une telle commande n’existe pas.

$PPID #

L’ID de processus (pid) du parent du script ou du shell, c’est-à-dire le processus qui a appelé le script ou le shell actuel.

~> $ echo $$
13016
~> $ echo $PPID
13015

## $SECONDES Le nombre de secondes d’exécution d’un script. Cela peut devenir assez volumineux s’il est affiché dans le shell :

~> $ echo $SECONDS
98834

$SHELLOPTS #

Une liste en lecture seule des options bash est fournie au démarrage pour contrôler son comportement :

~> $ echo $SHELLOPTS
braceexpand:emacs:hashall:histexpand:history:interactive-comments:monitor

## $_ Affiche le dernier champ de la dernière commande exécutée, utile pour obtenir quelque chose à transmettre à une autre commande :

 ~> $ ls *.sh;echo $_
testscript1.sh  testscript2.sh
testscript2.sh
Il donne le chemin du script s'il est utilisé avant toute autre commande :

test.sh :

#!/bin/bash
echo "$_"

Production:

~> $ ./test.sh # running test.sh
./test.sh

Remarque : Ce n’est pas un moyen infaillible d’obtenir le chemin du script

$ALÉATOIRE #

Chaque fois que ce paramètre est référencé, un entier aléatoire compris entre 0 et 32767 est généré. Attribuer une valeur à cette variable amorce le générateur de nombres aléatoires ([source][1]).

~> $ echo $RANDOM 
27119
~> $ echo $RANDOM 
1349

[1] : https://www.gnu.org/software/bash/manual/bashref.html#Bash-Variables

## $GROUPES Un tableau contenant le nombre de groupes dans lesquels l’utilisateur se trouve :

#!/usr/bin/env bash
echo You are assigned to the following groups:
for group in ${GROUPS[@]}; do
   IFS=: read -r name dummy number members < <(getent group $group )
   printf "name: %-10s number: %-15s members: %s\n" "$name" "$number" "$members" 
done

$Linge #

Affiche le numéro de ligne dans le script actuel. Principalement utile lors du débogage de scripts.

#!/bin/bash
# this is line 2
echo something  # this is line 3
echo $LINENO # Will output 4

## $SHLVL Lorsque la commande bash est exécutée, un nouveau shell est ouvert. La variable d’environnement $SHLVL contient le nombre de niveaux de shell sur lesquels le shell current s’exécute.

Dans une nouvelle fenêtre de terminal, l’exécution de la commande suivante produira des résultats différents en fonction de la distribution Linux utilisée.

echo $SHLVL

En utilisant Fedora 25, la sortie est “3”. Cela indique que lors de l’ouverture d’un nouveau shell, une commande bash initiale s’exécute et exécute une tâche. La commande bash initiale exécute un processus enfant (une autre commande bash) qui, à son tour, exécute une commande bash finale pour ouvrir le nouveau shell. Lorsque le nouveau shell s’ouvre, il s’exécute en tant que processus enfant de 2 autres processus shell, d’où la sortie de “3”.

Dans l’exemple suivant (étant donné que l’utilisateur exécute Fedora 25), la sortie de $SHLVL dans un nouveau shell sera définie sur “3”. Au fur et à mesure que chaque commande bash est exécutée, $SHLVL est incrémenté de un.

~> $ echo $SHLVL
3
~> $ bash
~> $ echo $SHLVL
4
~> $ bash
~> $ echo $SHLVL
5

On peut voir que l’exécution de la commande ‘bash’ (ou l’exécution d’un script bash) ouvre un nouveau shell. En comparaison, le sourcing d’un script exécute le code dans le shell actuel.

test1.sh

#!/usr/bin/env bash
echo "Hello from test1.sh. My shell level is $SHLVL"
