iOS 10 - Changements dans la demande d'autorisations de l'appareil photo, de la bibliothèque de photos provoquant le blocage de l'application

Dans iOS 10, Apple étend la portée du contrôle de la confidentialité. L’utilisateur doit demander l’autorisation d’utilisation avant d’accéder aux données privées de l’utilisateur dans la clé suivante : Calendrier, Contact, Rappel, Photo, Partage Bluetooth, Microphone, Appareil photo, Emplacement, Heath, HomeKit, MediaLibrary, Mouvement, CallKit, Reconnaissance vocale, SiriKit, TV L’utilisateur du fournisseur doit déclarer dans le fichier info.plist pour accéder à toutes les données privées.

IOS 10 Autorisation requise pour accéder à l’appareil photo, à la photothèque …. #

iOS 10 nécessite désormais l’autorisation de l’utilisateur pour accéder à la bibliothèque multimédia, aux photos, Appareil photo et autre matériel comme ceux-ci. La solution pour cela est d’ajouter leurs clés dans info.plist avec une description pour l’utilisateur indiquant comment nous sommes en utilisant leurs données, iOS avait déjà besoin d’autorisations pour accéder au microphone, à l’appareil photo et à la bibliothèque multimédia plus tôt (iOS6, iOS7), mais depuis iOS10, les applications planteront si vous ne fournissez pas la description de la raison pour laquelle vous demandez l’autorisation.

** Il existe une liste de toutes les clés Cocoa que vous pouvez spécifier dans votre fichier Info.plist **

Photo :

Key       :  Privacy - Photo Library Usage Description    
Value   :  $(PRODUCT_NAME) photo use

Microphone :

Key        :  Privacy - Microphone Usage Description    
Value    :  $(PRODUCT_NAME) microphone use

Caméra :

Key       :  Privacy - Camera Usage Description   
Value   :  $(PRODUCT_NAME) camera use

https://developer.apple.com/library/content/documentation/General/Reference/InfoPlistKeyReference/Articles/CocoaKeys.html][1]

[1] : https://developer.apple.com/library/content/documentation/General/Reference/InfoPlistKeyReference/Articles/CocoaKeys.html

Autorisations nécessaires pour les clés de confidentialité dans iOS 10 #

Calendrier : Clé : Confidentialité - Calendriers Utilisation Description Valeur : événements d’agenda $(PRODUCT_NAME)

Rappel : Clé : Confidentialité - Rappels Utilisation Description Valeur : utilisation du rappel $(PRODUCT_NAME)

Contact : Clé : Confidentialité - Contacts Utilisation Description Valeur : utilisation du contact $(PRODUCT_NAME)

Photo : Clé : Confidentialité - Description de l’utilisation de la photothèque Valeur : utilisation de la photo $(PRODUCT_NAME)

Partage Bluetooth : Clé : Confidentialité - Description de l’utilisation des périphériques Bluetooth Valeur : $(PRODUCT_NAME) Utilisation du périphérique Bluetooth

Microphone : Légende : Confidentialité - Description de l’utilisation du microphone Valeur : Utilisation du microphone de $(PRODUCT_NAME)

Caméra : Clé : Confidentialité - Description de l’utilisation de la caméra Valeur : $(PRODUCT_NAME) utilisation de l’appareil photo

Emplacement : Clé : Confidentialité - Emplacement Toujours Utilisation Description Valeur : utilisation de l’emplacement $(PRODUCT_NAME)

Clé : Confidentialité - Emplacement lors de l’utilisation Description de l’utilisation Valeur : utilisation de l’emplacement $(PRODUCT_NAME)

Heath : Clé : Confidentialité - Description de l’utilisation du partage de santé Valeur : utilisation de la part d’intégrité de $(PRODUCT_NAME)

Clé : Confidentialité - Description de l’utilisation de la mise à jour de l’état de santé Valeur : utilisation de la mise à jour d’intégrité de $(PRODUCT_NAME)

HomeKit : Clé : Confidentialité - Description de l’utilisation de HomeKit Valeur : $(PRODUCT_NAME) utilisation du kit pour la maison

Médiathèque : Clé : Confidentialité - Description de l’utilisation de la médiathèque Valeur : utilisation de la médiathèque de $(PRODUCT_NAME)

Mouvement : Clé : Confidentialité - Motion Utilisation Description Valeur : utilisation de mouvement $(PRODUCT_NAME)

Reconnaissance de la parole : Clé : Confidentialité - Utilisation de la reconnaissance vocale Description Valeur : utilisation vocale de $(PRODUCT_NAME)

SiriKit : Clé : Confidentialité - Description de l’utilisation de Siri Valeur : $(PRODUCT_NAME) siri utilise

Fournisseur de télévision : Clé : Confidentialité - Description de l’utilisation du fournisseur de télévision Valeur : utilisation de $(PRODUCT_NAME) tvProvider

Si vous ne fournissez pas la clé de confidentialité dans Info.plist, l’application crash. Les logs sont comme ça :

L’application s’est plantée car elle a tenté d’accéder à des informations confidentielles données sans description d’utilisation. Le fichier Info.plist de l’application doit contenir un Clé NSCalendarUsageDescription avec une valeur de chaîne expliquant à l’utilisateur comment l’utilisateur de l’application comment l’application utilise ces données.
