Création de polices

Les polices SVG ne sont plus directement prises en charge par les navigateurs. Pourtant, ils sont très pratiques pour générer par programme des polices telles que des polices de symboles ou des polices de codes à barres.

Il existe de nombreux outils qui vous permettent de convertir des polices svg vers n’importe quel autre format de police.

Voici une liste d’outils que vous pouvez utiliser avec les polices SVG.

Convertisseurs #

  • https://github.com/fontello/svg2ttf

une police simple #

Un exemple simple d’une police svg. Quelques éléments à noter ici :

  • le système de coordonnées des glyphes est en opposition au système de coordonnées habituel en svg. l’axe y pointe vers le haut. Le point 0,0 est dans le coin inférieur droit.

  • Tous les chemins d’une police doivent être tracés dans le sens inverse des aiguilles d’une montre.

  • Dans la plupart des outils, seul l’attribut d de l’élément glyphe est pris en charge. Les éléments enfants ne fonctionneront pas, bien qu’ils soient techniquement autorisés.

Si vous avez des glyphes plus larges ou plus étroits, modifiez simplement horiz-adv-x sur l’élément glyphe lui-même.

<glyph unicode="a" horiz-adv-x="512" d="M0 0 H1000 L500 1000z M200 200 L500 800 L800 200z" />

sélection de polices #

la propriété unicode est utilisée pour une sélection ultérieure de glyphes. Vous pouvez utiliser des lettres simples ou des points de code unicode ainsi que des ligatures (combinaison de lettres ou points de code unicode)

  • unicode="abc"
  • unicode="&#97;&#98;"
  • unicode="ab&#97;&#98;"
  • unicode="a"
  • unicode="&#98;"

Les glyphes sont toujours sélectionnés par première correspondance, donc ayez toutes les ligatures avant chaque caractère.

les points de code unicode peuvent être écrits en notation décimale &#123; ou en notation hexadécimale &#x1f.

montée, descente et ligne de base #

la propriété units-per-em est l’une des propriétés de police les plus importantes. Il est utilisé pour donner un sens à n’importe quelle valeur de n’importe quelle autre propriété.

la spécification de police CSS2 a une belle [définition du carré em] [1] :

Certaines valeurs, telles que les métriques de largeur, sont exprimées en unités relatives à un carré abstrait dont la hauteur est la distance prévue entre les lignes de type dans la même taille de type

la ligne de base par défaut est à 0 dans le carré em. pour le calcul de la hauteur de ligne et à des fins d’alignement, les deux propriétés montée et descente sont de la plus haute importance.

L’ascension est la distance maximale entre la ligne de base et le point le plus élevé de votre plus grand glyphe. Habituellement, c’est 1em, donc la valeur que vous avez donnée pour les unités par em.

La descente est la distance maximale entre la ligne de base et le point le plus bas de n’importe quel glyphe de votre police.

Voici une police avec des glyphes représentant une ligne au point le plus bas et le plus haut ainsi qu’à la ligne de base.

<svg xmlns="http://www.w3.org/2000/svg" viewBox="0 0 1000 1000">
  <font id = "myFont"
        horiz-adv-x   = "1000"
        vert-origin-x = "0"
        vert-origin-y = "0" >
    <font-face font-family  = "myFont"
               font-weight  = "normal"
               units-per-em = "1000"
               descent="500"
               ascent="1000">
      <font-face-src>
        <font-face-name name="myFont"/>
      </font-face-src>
    </font-face>`
    <glyph unicode = "a" d = "M0 900h1000v100h-1000z" />
    <glyph unicode = "b" d = "M0 0h1000v100h-1000z" />
    <glyph unicode = "c" d = "M0 -500h1000v100h-1000z" />
  </font>
</svg>

L’ascension et la descente sont utilisées pour déterminer la hauteur de ligne. Les unités par em et la ligne de base sont utilisées pour déterminer la position et la taille par rapport aux autres polices utilisées.

[1] : https://www.w3.org/TR/2008/REC-CSS2-20080411/fonts.html#emsq
