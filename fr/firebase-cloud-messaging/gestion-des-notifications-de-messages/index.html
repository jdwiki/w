Gestion des messages avec l’application en arrière-plan ou supprimée #

Firebase gère les notifications différemment lorsque l’application est en arrière-plan (processus tué) et lorsqu’elle est au premier plan (active).

Lorsque votre application est en arrière-plan, les messages de notification sont affichés dans la barre d’état système et onMessageReceived n’est pas appelé. Pour les messages de notification avec une charge utile de données, le message de notification s’affiche dans la barre d’état système et les données incluses dans le message de notification peuvent être récupérées à partir de l’intention lancée lorsque l’utilisateur appuie sur la notification. [[1]]

Si l’application est en arrière-plan, le service déclenchera la notification par défaut avec le title et le body dans la notification et, comme mentionné, la méthode onMessageReceived ne le fera pas. être déclencheur. Au lieu de cela, le clic ouvrira l’activité du Manifest.xml marqué par :

<intent-filter>
        <action android:name="android.intent.action.MAIN"/>
        <category android:name="android.intent.category.LAUNCHER"/>
</intent-filter>

À partir de maintenant, vous pouvez obtenir vos données dans l’intention de cette activité :

<pré> si (getIntent() != null && getIntent().getExtras() != null) { String customString = (String) getIntent().getExtras().get(“myStringData”); Integer customInteger = (Integer) getIntent().getExtras().get(“myIntData”); }

Dans ce cas, la définition de l’icône et de la couleur d’arrière-plan de l’icône peut être effectuée à partir de Manifest.xml[[2]] :

<meta-data
        android:name="com.google.firebase.messaging.default_notification_icon"
        android:resource="@drawable/your_drawable_icon" />

<meta-data
        android:name="com.google.firebase.messaging.default_notification_color"
        android:resource="@color/your_color" />
