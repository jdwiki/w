Configuration de Dojo avec dojoConfig

  • Configuration du chargeur
  • Charger DojoConfig

Sur cette page

  • Configuration du chargeur
  • Charger DojoConfig

L’objet dojoConfig (anciennement djConfig) vous permet de définir des options et un comportement par défaut pour divers aspects de la boîte à outils dojo. Des exemples expliqueront ce qui est possible et comment vous pouvez utiliser dojoConfig dans votre code.

Notez que dojoConfig est défini dans un bloc de script avant le chargement de dojo.js. Ceci est d’une importance primordiale - si inversé, les propriétés de configuration seront ignorées.

Configuration du chargeur #

Dojo a reçu un nouveau chargeur dans Dojo 1.7 pour s’adapter au nouveau format de module AMD de la boîte à outils. Ce nouveau chargeur a ajouté quelques nouvelles options de configuration qui sont cruciales pour définir les packages, les cartes, etc. Pour plus de détails sur le chargeur, consultez le didacticiel sur l’utilisation avancée d’AMD. Les paramètres de configuration importants du chargeur incluent :

baseUrl : l’URL de base ajoutée à un identifiant de module lorsque le convertir en chemin ou en URL.

baseUrl: "/js"

packages : un tableau d’objets qui fournissent le nom du package et emplacement :

packages: [{
    name: "myapp",
    location: "/js/myapp"
}]

map : vous permet de mapper les chemins dans les identifiants de module vers différents chemins :

map: {
    dijit16: {
        dojo: "dojo16"
    }
}

chemins : une carte des fragments d’ID de module vers les chemins de fichier :

var dojoConfig = {
    packages: [
        "package1",
        "package2"
    ],
    paths: {
        package1: "../lib/package1",
        package2: "/js/package2"
    }
};

    // ...is equivalent to:
var dojoConfig = {
    packages: [
        { name: "package1", location: "../lib/package1" },
        { name: "package2", location: "/js/package2" }
    ]
};

async : définit si le noyau Dojo doit être chargé de manière asynchrone. Valeurs peut être vrai, faux ou legacyAsync, ce qui met le chargeur en permanence en mode interdomaine hérité.

async: true

parseOnLoad : si vrai, analyse la page avec dojo/parser lorsque le DOM et toutes les dépendances initiales (y compris celles du fichier dojoConfig.deps tableau) ont été chargés.

parseOnLoad: true

It is recommended that parseOnLoad be left at false (it defaults to false, so you can simply omit this property), and that developers explicitly require dojo/parser and call parser.parse().

deps : Un tableau de chemins de ressources qui doivent se charger immédiatement une fois Dojo a chargé :

deps: ["dojo/parser"]

callback : Le callback à exécuter une fois les deps récupérés :

callback: function(parser) {
    // Use the resources provided here
}

waitSeconds : temps d’attente avant de signaler le délai de chargement pour un module ; par défaut à 0 (attendre indéfiniment):

waitSeconds: 5

cacheBust : si vrai, ajoute l’heure en tant que chaîne de requête à chaque module URL pour éviter la mise en cache du module :

cacheBust: true

Créons maintenant une démonstration simple qui utilise les paramètres de base. Un scénario très courant consiste à utiliser Dojo Toolkit de CDN avec des modules locaux.

Disons que nous utilisons Google CDN avec des modules dans l’espace /documentation/tutorials/1.10/dojo_config/demo :

<!-- Configure Dojo first -->
<script>
    dojoConfig = {
        has: {
            "dojo-firebug": true,
            "dojo-debug-messages": true
        },
        // Don't attempt to parse the page for widgets
        parseOnLoad: false,
        packages: [
            // Any references to a "demo" resource should load modules locally, *not* from CDN
            {
                name: "demo",
                location: "/documentation/tutorials/1.10/dojo_config/demo"
            }
        ],
        // Timeout after 10 seconds
        waitSeconds: 10,
        map: {
            // Instead of having to type "dojo/domReady!", we just want "ready!" instead
            "*": {
                ready: "dojo/domReady"
            }
        },
        // Get "fresh" resources
        cacheBust: true
    };
</script>

<!-- Load Dojo, Dijit, and DojoX resources from Google CDN -->
<script src="//ajax.googleapis.com/ajax/libs/dojo/1.10.4/dojo/dojo.js"></script>

<!-- Load a "demo" module -->

<script>
    require(["demo/AuthoredDialog", "dojo/parser", "ready!"], function(AuthoredDialog, parser) {
        // Parse the page
        parser.parse();

        // Do something with demo/AuthoredDialog...
    });
</script>

En utilisant la configuration des packages, nous avons fait en sorte que toutes les références à demo/* pointent vers notre répertoire local /documentation/tutorials/1.10/dojo_config/demo/, tout en autorisant toutes les références à dojo, dijit, et dojox pour venir de Google CDN. Si le package de démonstration n’avait pas été défini, la demande de demo/AuthoredDialog aurait été envoyée à //ajax.googleapis.com/ajax/libs/dojo/1.10.4/dojo/demo/AuthoredDialog.js. Nous avons également utilisé des alias, en associant ready à dojo/domReady.

Charger DojoConfig #

Dans l’exemple ci-dessous, nous créons un objet javascript global “dojoConfig” qui contiendra toutes les valeurs de configuration.

Remarque : dojoConfig est défini dans un bloc de script avant le chargement de dojo.js. Ceci est d’une importance primordiale - si inversé, les propriétés de configuration seront ignorées.

<script>
    dojoConfig= {
        has: {
            "dojo-firebug": true
        },
        parseOnLoad: false,
        foo: "bar",
        async: true
    };
</script>
<script src="//ajax.googleapis.com/ajax/libs/dojo/1.10.4/dojo/dojo.js"></script>

<script>
// Require the registry, parser, Dialog, and wait for domReady
require(["dijit/registry", "dojo/parser", "dojo/json", "dojo/_base/config", "dijit/Dialog", "dojo/domReady!"]
, function(registry, parser, JSON, config) {
    // Explicitly parse the page
    parser.parse();
    // Find the dialog
    var dialog = registry.byId("dialog");
    // Set the content equal to what dojo.config is
    dialog.set("content", "<pre>" + JSON.stringify(config, null, "\t") + "```");
    // Show the dialog
    dialog.show();
});
</script>

<!-- and later in the page -->
<div id="dialog" data-dojo-type="dijit/Dialog" data-dojo-props="title: 'dojoConfig / dojo/_base/config'"></div>
