Accéder à Visual Basic Editor dans Microsoft Office #

Vous pouvez ouvrir l’éditeur VB dans n’importe quelle application Microsoft Office en appuyant sur Alt+F11 ou en allant dans l’onglet Développeur et en cliquant sur le bouton “Visual Basic”. Si vous ne voyez pas l’onglet Développeur dans le ruban, vérifiez s’il est activé.

Par défaut, l’onglet Développeur est désactivé. Pour activer l’onglet Développeur, allez dans Fichier -> Options, sélectionnez Personnaliser le ruban dans la liste de gauche. Dans l’arborescence de droite “Personnaliser le ruban”, recherchez l’élément de l’arborescence Développeur et cochez la case Développeur sur cochée. Cliquez sur OK pour fermer la boîte de dialogue Options.

[![Menu Options de fichier][1]][1]

L’onglet Développeur est maintenant visible dans le ruban sur lequel vous pouvez cliquer sur “Visual Basic” pour ouvrir l’éditeur Visual Basic. Vous pouvez également cliquer sur “Afficher le code” pour afficher directement le volet de code de l’élément actuellement actif, par ex. Feuille de calcul, graphique, forme.

[![Onglet Développeur][2]][2]

[![VBE][3]][3]

Vous pouvez utiliser VBA pour automatiser presque toutes les actions pouvant être effectuées de manière interactive (manuellement) et également fournir des fonctionnalités qui ne sont pas disponibles dans Microsoft Office. VBA peut créer un document, y ajouter du texte, le formater, le modifier et l’enregistrer, le tout sans intervention humaine.

[1] : http://i.stack.imgur.com/8WoiR.png [2] : http://i.stack.imgur.com/388eU.png [3] : http://i.stack.imgur.com/azT5a.png

Premier module et Hello World #

Pour commencer à coder en premier lieu, vous devez cliquer avec le bouton droit sur votre projet VBA dans la liste de gauche et ajouter un nouveau module. Votre premier code Hello-World pourrait ressembler à ceci :

Sub HelloWorld()
    MsgBox "Hello, World!"
End Sub

Pour le tester, appuyez sur le bouton Play dans votre barre d’outils ou appuyez simplement sur la touche F5. Toutes nos félicitations! Vous avez créé votre premier module VBA.

Débogage #

Le débogage est un moyen très puissant d’examiner de plus près et de corriger le code qui ne fonctionne pas correctement (ou qui ne fonctionne pas).

Exécuter le code étape par étape #

La première chose que vous devez faire pendant le débogage est d’arrêter le code à des endroits spécifiques et puis exécutez-le ligne par ligne pour voir si cela se produit comme prévu.

  • Point d’arrêt (F9, Debug - Basculer le point d’arrêt) : vous pouvez ajouter un point d’arrêt à n’importe quelle ligne exécutée (par exemple, pas aux déclarations), lorsque l’exécution atteint ce point, elle s’arrête et donne le contrôle à l’utilisateur.
  • Vous pouvez également ajouter le mot-clé “Stop” à une ligne vide pour que le code s’arrête à cet emplacement lors de l’exécution. Ceci est utile si, par exemple, avant les lignes de déclaration auxquelles vous ne pouvez pas ajouter de point d’arrêt avec F9
  • Step into (F8, Debug - Step into) : exécute une seule ligne de code, si c’est un appel d’une sous/fonction définie par l’utilisateur, alors c’est exécuté ligne par ligne.
  • Step over (Shift+F8, Debug - Step over) : exécute une ligne de code, n’entre pas dans les sous-fonctions/fonctions définies par l’utilisateur.
  • Sortir (Ctrl+Maj+F8, Debug - Sortir) : Quitter la sous/fonction actuelle (exécuter le code jusqu’à sa fin).
  • Exécuter jusqu’au curseur (Ctrl+F8, Debug - Exécuter jusqu’au curseur) : exécutez le code jusqu’à atteindre la ligne avec le curseur.
  • Vous pouvez utiliser Debug.Print pour imprimer des lignes dans la fenêtre immédiate lors de l’exécution. Vous pouvez également utiliser Debug.? comme raccourci pour Debug.Print

Vitrine des montres #

L’exécution du code ligne par ligne n’est que la première étape, nous avons besoin de connaître plus de détails et un outil pour cela est la fenêtre de surveillance (View - Watch window), ici vous pouvez voir les valeurs des expressions définies. Pour ajouter une variable à la fenêtre de surveillance, soit :

  • Faites un clic droit dessus puis sélectionnez “Ajouter une montre”.
  • Faites un clic droit dans la fenêtre de la montre, sélectionnez “Ajouter une montre”.
  • Allez dans Debug - Ajouter une montre.

Lorsque vous ajoutez une nouvelle expression, vous pouvez choisir si vous voulez simplement voir sa valeur, ou également interrompre l’exécution du code lorsqu’elle est vraie ou lorsque sa valeur change.

Fenêtre immédiate #

La fenêtre immédiate vous permet d’exécuter du code arbitraire ou d’imprimer des éléments en les faisant précéder du mot clé Print ou d’un simple point d’interrogation “?”

Quelques exemples:

  • ? ActiveSheet.Name - renvoie le nom de la feuille active
  • Print ActiveSheet.Name - renvoie le nom de la feuille active
  • ? foo - renvoie la valeur de foo*
  • x = 10 fixe x à 10*

* Obtenir/Définir les valeurs des variables via la fenêtre immédiate ne peut être effectué que pendant l’exécution

Bonnes pratiques de débogage #

Chaque fois que votre code ne fonctionne pas comme prévu, la première chose à faire est de le relire attentivement, à la recherche d’erreurs.

Si cela ne vous aide pas, commencez à le déboguer ; pour les procédures courtes, il peut être efficace de simplement l’exécuter ligne par ligne, pour les plus longues, vous devrez probablement définir des points d’arrêt ou des pauses sur les expressions surveillées, le but ici est de trouver la ligne qui ne fonctionne pas comme prévu.

Une fois que vous avez la ligne qui donne le résultat incorrect, mais que la raison n’est pas encore claire, essayez de simplifier les expressions ou de remplacer les variables par des constantes, cela peut aider à comprendre si la valeur des variables est erronée.

Si vous ne parvenez toujours pas à le résoudre et demandez de l’aide :

  • Incluez la plus petite partie possible de votre code pour comprendre votre problème
  • Si le problème n’est pas lié à la valeur des variables, alors remplacez-les par des constantes. (donc, au lieu de Sheets(a*b*c+d^2).Range(addressOfRange) écrivez Sheets(4).Range("A2"))
  • Décrivez quelle ligne donne le mauvais comportement, et de quoi il s’agit (erreur, mauvais résultat…)
