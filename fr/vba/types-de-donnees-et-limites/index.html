Une variante #

Dim Value As Variant    'Explicit
Dim Value               'Implicit

Un Variant est un type de données COM utilisé pour stocker et échanger des valeurs de types arbitraires, et tout autre type dans VBA peut être affecté à un Variant. Les variables déclarées sans type explicite spécifié par As [Type] sont par défaut Variant.

Les variantes sont stockées en mémoire sous la forme d’une structure VARIANT qui consiste en un descripteur de type d’octet (VARTYPE) suivi de 6 octets réservés puis d’une zone de données de 8 octets. Pour les types numériques (y compris Date et Boolean), la valeur sous-jacente est stockée dans le Variant lui-même. Pour tous les autres types, la zone de données contient un pointeur vers la valeur sous-jacente.

[![entrez la description de l’image ici][1]][1]

Le type sous-jacent d’un Variant peut être déterminé soit avec la fonction VarType() qui renvoie la valeur numérique stockée dans le descripteur de type, soit avec la fonction TypeName() qui renvoie la représentation sous forme de chaîne :

Dim Example As Variant
Example = 42
Debug.Print VarType(Example)    'Prints 2 (VT_I2)
Debug.Print TypeName(Example)   'Prints "Integer"
Example = "Some text"
Debug.Print VarType(Example)    'Prints 8 (VT_BSTR)
Debug.Print TypeName(Example)   'Prints "String"

Étant donné que les variantes peuvent stocker des valeurs de n’importe quel type, les affectations de littéraux sans [indications de type] (https://www.wikiod.com/fr/vba/declaration-de-variables#Conseils de saisie) seront implicitement converties à une variante du type approprié selon le tableau ci-dessous. Les littéraux avec des indications de type seront convertis en un Variant du type indiqué.

Valeur Type résultant
Valeurs de chaîne Chaîne
Nombres à virgule non flottante dans la plage d’entiers Entier
Nombres à virgule non flottante à longue portée longue
Nombres à virgule non flottante en dehors de Long range Double
Tous les nombres à virgule flottante Double

Remarque : À moins qu’il n’y ait une raison spécifique d’utiliser une variante (c’est-à-dire un itérateur dans une boucle For Each ou une exigence d’API), le type doit généralement être évité pour les tâches de routine pour les raisons suivantes :

  • Ils ne sont pas sécurisés, ce qui augmente le risque d’erreurs d’exécution. Par exemple, un Variant contenant une valeur Integer se transformera silencieusement en Long au lieu de déborder.
  • Ils introduisent une surcharge de traitement en exigeant au moins un déréférencement de pointeur supplémentaire.
  • La mémoire requise pour une variante est toujours *** au moins *** 8 octets supérieur à celui nécessaire pour stocker le type sous-jacent.

La fonction de conversion à convertir en Variant est CVar().

[1] : http://i.stack.imgur.com/RFvAd.png

Booléen #

Dim Value As Boolean

Un booléen est utilisé pour stocker des valeurs qui peuvent être représentées par True ou False. En interne, le type de données est stocké sous la forme d’une valeur de 16 bits, 0 représentant False et toute autre valeur représentant True.

Il convient de noter que lorsqu’un booléen est converti en un type numérique, tous les bits sont définis sur 1. Il en résulte une représentation interne de -1 pour les types signés et la valeur maximale pour un type non signé (Byte).

Dim Example As Boolean
Example = True
Debug.Print CInt(Example)  'Prints -1
Debug.Print CBool(42)      'Prints True
Debug.Print CByte(True)    'Prints 255

La fonction de conversion à convertir en booléen est CBool(). Même s’il est représenté en interne sous la forme d’un nombre de 16 bits, la conversion en booléen à partir de valeurs en dehors de cette plage est à l’abri du débordement, bien qu’il définisse tous les 16 bits sur 1 :

Dim Example As Boolean
Example = CBool(2 ^ 17)
Debug.Print CInt(Example)   'Prints -1
Debug.Print CByte(Example)  'Prints 255

Chaîne de caractères #

Une chaîne représente une séquence de caractères et se présente sous deux formes :

Longueur variable #

Dim Value As String

Une chaîne de longueur variable permet l’ajout et la troncature et est stockée en mémoire en tant que COM BSTR. Il s’agit d’un entier non signé de 4 octets qui stocke la longueur de la chaîne en octets suivi des données de la chaîne elle-même sous forme de caractères larges (2 octets par caractère) et se termine par 2 octets nuls. Ainsi, la longueur de chaîne maximale pouvant être gérée par VBA est de 2 147 483 647 caractères.

Le pointeur interne vers la structure (récupérable par la fonction StrPtr()) pointe vers l’emplacement mémoire des données, pas le préfixe de longueur. Cela signifie qu’une chaîne VBA peut être transmise directement aux fonctions API qui nécessitent un pointeur vers un tableau de caractères.

Étant donné que la longueur peut changer, VBA réalloue de la mémoire pour une chaîne * chaque fois que la variable est affectée à *, ce qui peut imposer des pénalités de performances pour les procédures qui les modifient à plusieurs reprises.

Longueur fixe #

Dim Value As String * 1024    'Declares a fixed length string of 1024 characters.

Les chaînes de longueur fixe se voient allouer 2 octets pour chaque caractère et sont stockées en mémoire sous la forme d’un simple tableau d’octets. Une fois allouée, la longueur de la chaîne est immuable. Ils ne sont pas terminés par null en mémoire, donc une chaîne qui remplit la mémoire allouée avec des caractères non nuls ne convient pas pour être transmise aux fonctions API qui attendent une chaîne terminée par null.

Les chaînes de longueur fixe comportent une limitation d’index héritée de 16 bits, elles ne peuvent donc contenir que 65 535 caractères. Tenter d’attribuer une valeur plus longue que l’espace mémoire disponible n’entraînera pas d’erreur d’exécution - à la place, la valeur résultante sera simplement tronquée :

Dim Foobar As String * 5
Foobar = "Foo" & "bar"
Debug.Print Foobar          'Prints "Fooba"

La fonction de conversion à convertir en une chaîne de l’un ou l’autre type est CStr ().

Octet #

Dim Value As Byte

Un octet est un type de données non signé de 8 bits. Il peut représenter des nombres entiers entre 0 et 255 et toute tentative de stockage d’une valeur en dehors de cette plage entraînera erreur d’exécution 6 : Overflow . Byte est le seul type intrinsèque non signé disponible dans VBA.

La fonction de conversion à convertir en octet est CByte(). Pour les conversions à partir de types à virgule flottante, le résultat est arrondi à la valeur entière la plus proche avec 0,5 arrondi vers le haut.

** Tableaux d’octets et chaînes **

Les chaînes et les tableaux d’octets peuvent être remplacés par une simple affectation (aucune fonction de conversion n’est nécessaire).

Par exemple:

Sub ByteToStringAndBack()

Dim str As String
str = "Hello, World!"

Dim byt() As Byte
byt = str

Debug.Print byt(0)  ' 72

Dim str2 As String
str2 = byt

Debug.Print str2    ' Hello, World!

End Sub

Afin de pouvoir encoder des caractères [Unicode][1], chaque caractère de la chaîne occupe deux octets dans le tableau, l’octet le moins significatif en premier. Par exemple:

Sub UnicodeExample()

Dim str As String
str = ChrW(&H2123) & "."  ' Versicle character and a dot

Dim byt() As Byte
byt = str

Debug.Print byt(0), byt(1), byt(2), byt(3)  ' Prints: 35,33,46,0

End Sub

[1] : http://unicode.org/

Devise #

Dim Value As Currency

Une devise est un type de données à virgule flottante 64 bits signé similaire à un Double, mais mis à l’échelle de 10 000 à donner une plus grande précision aux 4 chiffres à droite de la virgule décimale. Une variable Currency peut stocker des valeurs comprises entre -922 337 203 685 477,5808 et 922 337 203 685 477,5807, ce qui lui confère la plus grande capacité de tous les types intrinsèques dans une application 32 bits. Comme le nom du type de données l’indique, il est recommandé d’utiliser ce type de données lors de la représentation de calculs monétaires, car la mise à l’échelle permet d’éviter les erreurs d’arrondi.

La fonction de conversion à convertir en devise est CCur().

Décimal #

Dim Value As Variant
Value = CDec(1.234)

'Set Value to the smallest possible Decimal value
Value = CDec("0.0000000000000000000000000001")

Le type de données Decimal est uniquement disponible en tant que sous-type de Variant, vous devez donc déclarer toute variable devant contenir un Decimal en tant que Variant et ensuite attribuer un Decimal valeur à l’aide de la fonction CDec. Le mot clé Decimal est un mot réservé (ce qui suggère que VBA allait éventuellement ajouter un support de première classe pour le type), donc Decimal ne peut pas être utilisé comme nom de variable ou de procédure.

Le type Decimal nécessite 14 octets de mémoire (en plus des octets requis par le Variant parent) et peut stocker des nombres avec jusqu’à 28 décimales. Pour les nombres sans décimales, la plage de valeurs autorisées est de -79 228 162 514 264 337 593 543 950 335 à +79 228 162 514 264 337 593 543 950 335 inclus. Pour les nombres avec un maximum de 28 décimales, la plage de valeurs autorisées est de -7,9228162514264337593543950335 à +7,9228162514264337593543950335 inclus.

Entier #

Dim Value As Integer

Un entier est un type de données 16 bits signé. Il peut stocker des nombres entiers dans la plage de -32 768 à 32 767 et tenter de stocker une valeur en dehors de cette plage entraînera l’erreur d’exécution 6 : débordement.

Les nombres entiers sont stockés en mémoire sous forme de valeurs [little-endian] (https://en.wikipedia.org/wiki/Endianness) avec des négatifs représentés sous la forme d’un [complément à deux] (https://en.wikipedia.org/wiki /Deux). %27s_complement).

Notez qu’en général, il est préférable d’utiliser un Long plutôt qu’un Integer à moins que le type plus petit est membre d’un type ou doit (soit par une convention d’appel d’API ou une autre raison) être de 2 octets. Dans la plupart des cas, VBA traite les entiers comme 32 bits en interne, il n’y a donc généralement aucun avantage à utiliser le type le plus petit. De plus, il y a une pénalité de performance encourue chaque fois qu’un type Integer est utilisé car il est silencieusement converti en Long.

La fonction de conversion à convertir en entier est CInt(). Pour les conversions à partir de types à virgule flottante, le résultat est arrondi à la valeur entière la plus proche avec 0,5 arrondi vers le haut.

Long #

Dim Value As Long

Un Long est un type de données 32 bits signé. Il peut stocker des nombres entiers dans la plage de -2 147 483 648 à 2 147 483 647 et tenter de stocker une valeur en dehors de cette plage entraînera l’erreur d’exécution 6 : Débordement.

Les longueurs sont stockées en mémoire sous forme de valeurs little-endian avec des valeurs négatives représentées sous la forme d’un [complément à deux](https://en.wikipedia.org/wiki/Two %27s_complément).

Notez qu’étant donné qu’un Long correspond à la largeur d’un pointeur dans un système d’exploitation 32 bits, les Longs sont couramment utilisés pour stocker et transmettre des pointeurs vers et depuis les fonctions API.

La fonction de conversion à convertir en Long est CLng(). Pour les conversions à partir de types à virgule flottante, le résultat est arrondi à la valeur entière la plus proche avec 0,5 arrondi vers le haut.

Seul #

Dim Value As Single

Un Single est un type de données à virgule flottante 32 bits signé. Il est stocké en interne à l’aide d’une mémoire little-endian IEEE 754 disposition. En tant que tel, il n’y a pas de plage fixe de valeurs pouvant être représentées par le type de données - ce qui est limité, c’est la précision de la valeur stockée. Un Single peut stocker une valeur entier valeurs dans la plage de -16 777 216 à 16 777 216 sans perte de précision. La précision des nombres à virgule flottante dépend de l’exposant.

Un Single débordera s’il lui est attribué une valeur supérieure à environ 2128. Il ne débordera pas d’exposants négatifs, bien que la précision utilisable soit discutable avant que la limite supérieure ne soit atteinte.

Comme avec tous les nombres à virgule flottante, des précautions doivent être prises lors des comparaisons d’égalité. La meilleure pratique consiste à inclure une valeur delta appropriée à la précision requise.

La fonction de conversion à convertir en Single est CSng().

Double #

Dim Value As Double

Un Double est un type de données à virgule flottante 64 bits signé. Comme le Single, il est stocké en interne à l’aide d’un [little-endian](https://en .wikipedia.org/wiki/Endianness) IEEE 754 et les mêmes précautions concernant la précision doivent être prises. Un Double peut stocker des valeurs entières dans la plage de -9 007 199 254 740 992 à 9 007 199 254 740 992 sans perte de précision. La précision des nombres à virgule flottante dépend de l’exposant.

Un Double débordera s’il lui est attribué une valeur supérieure à environ 21024. Il ne débordera pas d’exposants négatifs, bien que la précision utilisable soit discutable avant que la limite supérieure ne soit atteinte.

La fonction de conversion à convertir en Double est CDbl().

Date #

Dim Value As Date

Un type Date est représenté en interne sous la forme d’un type de données à virgule flottante 64 bits signé avec la valeur à gauche de la décimale représentant le nombre de jours à partir de la date d’époque du 30 décembreth 1899 (bien que voir le note ci-dessous). La valeur à droite de la décimale représente l’heure sous la forme d’un jour fractionnaire. Ainsi, un entier Date aurait un composant horaire de 12:00:00AM et x.5 aurait un composant horaire de 12:00:00PM.

Les valeurs valides pour les dates sont comprises entre le 1er janvierer 100 et le 31 décembreer 9999. Étant donné qu’un double a une plage plus large, il est possible de dépasser une date en attribuant des valeurs en dehors de cette gamme.

En tant que tel, il peut être utilisé de manière interchangeable avec un Double pour les calculs de date :

Dim MyDate As Double
MyDate = 0                                  'Epoch date.
Debug.Print Format$(MyDate, "yyyy-mm-dd")   'Prints 1899-12-30.
MyDate = MyDate + 365
Debug.Print Format$(MyDate, "yyyy-mm-dd")   'Prints 1900-12-30.

La fonction de conversion à convertir en date est CDate(), qui accepte toute représentation date/heure de chaîne de type numérique. Il est important de noter que les représentations sous forme de chaîne des dates seront converties en fonction des paramètres régionaux actuellement utilisés. Par conséquent, les transtypages directs doivent être évités si le code est censé être portable.

LongLong #

Dim Value As LongLong

Un LongLong est un type de données 64 bits signé et n’est disponible que dans les applications 64 bits. Il n’est pas disponible dans les applications 32 bits exécutées sur des systèmes d’exploitation 64 bits. Il peut stocker des valeurs entières dans la plage de -9 223 372 036 854 775 808 à 9 223 372 036 854 775 807 et tenter de stocker une valeur en dehors de cette plage entraînera l’erreur d’exécution 6 : Débordement.

Les LongLongs sont stockés en mémoire sous forme de valeurs little-endian avec des négatifs représentés comme un [complément à deux](https://en.wikipedia.org/wiki/Two %27s_complément).

Le type de données LongLong a été introduit dans le cadre de la prise en charge du système d’exploitation 64 bits de VBA. Dans les applications 64 bits, cette valeur peut être utilisée pour stocker et transmettre des pointeurs vers des API 64 bits.

La fonction de conversion à convertir en LongLong est CLngLng(). Pour les conversions à partir de types à virgule flottante, le résultat est arrondi à la valeur entière la plus proche avec 0,5 arrondi vers le haut.

LongPtr #

Dim Value As LongPtr

Le LongPtr a été introduit dans VBA afin de prendre en charge les plates-formes 64 bits. Sur un système 32 bits, il est traité comme Long et sur les systèmes 64 bits, il est traité comme un LongLong.

Son utilisation principale est de fournir un moyen portable de stocker et de transmettre des pointeurs sur les deux architectures (voir https://www.wikiod.com/fr/vba/compilation-conditionnelle#Modification du comportement du code au moment de la compilation #t=20160724185822285365).

Bien qu’il soit traité par le système d’exploitation comme une adresse mémoire lorsqu’il est utilisé dans les appels d’API, il convient de noter que VBA le traite comme un type signé (et donc sujet à un débordement non signé vers signé). Pour cette raison, toute arithmétique de pointeur effectuée à l’aide de LongPtrs ne doit pas utiliser de comparaisons > ou <. Cette “bizarre” permet également que l’ajout de décalages simples pointant vers des adresses valides en mémoire puisse provoquer des erreurs de débordement, il faut donc être prudent lorsque l’on travaille avec des pointeurs dans VBA.

La fonction de conversion à convertir en LongPtr est CLngPtr(). Pour les conversions à partir de types à virgule flottante, le résultat est arrondi à la valeur entière la plus proche avec 0,5 arrondi vers le haut (bien qu’il s’agisse généralement d’une adresse mémoire, l’utiliser comme cible d’affectation pour un calcul en virgule flottante est au mieux dangereux).
