Syntaxe #

  • Option optionName [valeur]
  • Option explicite
  • Option Comparer {Texte | Binaire | Base de données}
  • Option module privé
  • Base d’options {0 | 1}

Paramètres #

Option Détail
Explicit Exiger une déclaration de variable dans le module dans lequel elle est spécifiée (idéalement toutes) ; avec cette option spécifiée, l’utilisation d’une variable non déclarée (/mispelled) devient une erreur de compilation.
Comparer Texte Rend les comparaisons de chaînes du module insensibles à la casse, en fonction des paramètres régionaux du système, en donnant la priorité à l’équivalence alphabétique (par exemple, “a” = “A”).
Comparer Binaire Mode de comparaison de chaînes par défaut. Rend les comparaisons de chaînes du module sensibles à la casse, en comparant les chaînes à l’aide de la représentation binaire/valeur numérique de chaque caractère (par exemple, ASCII).
Comparer Base de données (MS-Access uniquement) Fait fonctionner les comparaisons de chaînes du module comme elles le feraient dans une instruction SQL.
Module privé Empêche l’accès au membre “Public” du module depuis l’extérieur du projet dans lequel réside le module, masquant ainsi les procédures de l’application hôte (c’est-à-dire qu’elles ne peuvent pas être utilisées comme macros ou fonctions définies par l’utilisateur).
Option Base 0 Paramètres par défaut. Définit la limite inférieure du tableau implicite à 0 dans un module. Lorsqu’un tableau est déclaré sans valeur limite inférieure explicite, ‘0’ sera utilisé.
Option Base 1 Définit la limite inférieure du tableau implicite à 1 dans un module. Lorsqu’un tableau est déclaré sans valeur limite inférieure explicite, ‘1’ sera utilisé.

Il est beaucoup plus facile de contrôler les limites des tableaux en déclarant explicitement les limites plutôt que de laisser le compilateur se rabattre sur une déclaration Option Base {0|1}. Cela peut être fait comme ceci :

Dim myStringsA(0 To 5) As String '// This has 6 elements (0 - 5)
Dim myStringsB(1 To 5) As String '// This has 5 elements (1 - 5)
Dim myStringsC(6 To 9) As String '// This has 3 elements (6 - 9)

Option explicite #

Il est recommandé de toujours utiliser Option Explicit dans VBA car cela oblige le développeur à déclarer toutes ses variables avant utilisation. Cela présente également d’autres avantages, tels que la capitalisation automatique pour les noms de variables déclarés et IntelliSense.

Option Explicit

Sub OptionExplicit()
    Dim a As Integer
    a = 5
    b = 10 '// Causes compile error as 'b' is not declared
End Sub

La définition de Exiger une déclaration de variable dans la page de propriétés Outils ► Options ► Éditeur du VBE placera l’Option Explicit en haut de chaque feuille de code nouvellement créée.

[![require_variable_declaration][1]][1]

Cela évitera les erreurs de codage stupides comme les fautes d’orthographe et vous incitera à utiliser le type de variable correct dans la déclaration de variable. (D’autres exemples sont donnés sur <https://www.wikiod.com/fr/excel-vba/meilleures-pratiques-vba#Utilisez TOUJOURS “Option Explicit”>.)

[1] : http://i.stack.imgur.com/C29RO.png

Base d’options {0 | 1} #

Option Base est utilisé pour déclarer la limite inférieure par défaut des éléments array. Il est déclaré au niveau du module et n’est valable que pour le module en cours.

Par défaut (et donc si aucune Option Base n’est spécifiée), la Base est 0. Ce qui signifie que le premier élément de tout tableau déclaré dans le module a un index de 0.

Si Option Base 1 est spécifié, le premier élément du tableau a l’index 1

Exemple en Base 0 : #

Option Base 0

Sub BaseZero()

    Dim myStrings As Variant
    
    ' Create an array out of the Variant, having 3 fruits elements
    myStrings = Array("Apple", "Orange", "Peach")
    
    Debug.Print LBound(myStrings) ' This Prints "0"
    Debug.Print UBound(myStrings) ' This print "2", because we have 3 elements beginning at 0 -> 0,1,2
            
    For i = 0 To UBound(myStrings)
    
        Debug.Print myStrings(i) ' This will print "Apple", then "Orange", then "Peach"
    
    Next i

End Sub

Même exemple avec la base 1 #

Option Base 1

Sub BaseOne()

    Dim myStrings As Variant
    
    ' Create an array out of the Variant, having 3 fruits elements
    myStrings = Array("Apple", "Orange", "Peach")
    
    Debug.Print LBound(myStrings) ' This Prints "1"
    Debug.Print UBound(myStrings) ' This print "3", because we have 3 elements beginning at 1 -> 1,2,3
            
    For i = 0 To UBound(myStrings)
    
        Debug.Print myStrings(i) ' This triggers an error 9 "Subscript out of range"
    
    Next i

End Sub

Le deuxième exemple a généré un [Indice hors limites (Erreur 9)] [1] à la première étape de la boucle car une tentative d’accès à l’index 0 du tableau a été faite, et cet index n’existe pas car le module est déclaré avec Base 1

Le bon code avec Base 1 est : #

    For i = 1 To UBound(myStrings)
    
        Debug.Print myStrings(i) ' This will print "Apple", then "Orange", then "Peach"
    
    Next i

Il convient de noter que la fonction Split toujours crée un tableau avec un index d’élément de base zéro, indépendamment de tout ` Réglage de la base d’options. Des exemples d’utilisation de la fonction Split peuvent être trouvés [ici](https://www.wikiod.com/fr/vba/tableaux#Utilisation de Split pour créer un tableau à partir d’une chaîne -from-a-string#t=201608041103181263837)

Fonction de fractionnement
Renvoie un tableau unidimensionnel de base zéro contenant un nombre spécifié de sous-chaînes.

Dans Excel, les propriétés Range.Value et Range.Formula pour une plage multicellulaire renvoient toujours un tableau Variant 2D de base 1.

De même, dans ADO, la méthode Recordset.GetRows renvoie toujours un tableau 2D de base 1.

Une « meilleure pratique » recommandée consiste à toujours utiliser LBound et [UBound](https://msdn.microsoft.com/ en-us/library/office/gg278658.aspx) pour déterminer l’étendue d’un tableau.

'for single dimensioned array
Debug.Print LBound(arr) & ":" & UBound(arr)
Dim i As Long
For i = LBound(arr) To UBound(arr)
    Debug.Print arr(i)
Next i

'for two dimensioned array
Debug.Print LBound(arr, 1) & ":" & UBound(arr, 1)
Debug.Print LBound(arr, 2) & ":" & UBound(arr, 2)
Dim i As long, j As Long
For i = LBound(arr, 1) To UBound(arr, 1)
    For j = LBound(arr, 2) To UBound(arr, 2)
         Debug.Print arr(i, j)
    Next j
Next i

L’option Base 1 doit être en haut de chaque module de code où un tableau est créé ou redimensionné si les tableaux doivent être créés de manière cohérente avec une limite inférieure de 1.

[1] : https://msdn.microsoft.com/en-us/library/aa264519.aspx

Option Comparer {Binaire | Texte | Base de données} #

Option Comparer Binaire #

La comparaison binaire effectue toutes les vérifications de l’égalité des chaînes dans un module/une casse de classe sensible. Techniquement, avec cette option, les comparaisons de chaînes sont effectuées en utilisant l’ordre de tri des représentations binaires de chaque caractère.

UNE < B < E < Z < une < b < e < z

Si aucune option Compare n’est spécifiée dans un module, Binary est utilisé par défaut.

Option Compare Binary

Sub CompareBinary()

    Dim foo As String
    Dim bar As String
    
    '// Case sensitive
    foo = "abc"
    bar = "ABC"
    
    Debug.Print (foo = bar) '// Prints "False"
    
    '// Still differentiates accented characters
    foo = "ábc"
    bar = "abc"
    
    Debug.Print (foo = bar) '// Prints "False"
    
    '// "b" (Chr 98) is greater than "a" (Chr 97)
    foo = "a"
    bar = "b"
    
    Debug.Print (bar > foo) '// Prints "True"
    
    '// "b" (Chr 98) is NOT greater than "á" (Chr 225)
    foo = "á"
    bar = "b"
    
    Debug.Print (bar > foo) '// Prints "False"

End Sub

Option Comparer le texte #

Option Compare Text fait que toutes les comparaisons de chaînes dans un module/classe utilisent une comparaison insensible à la casse.

(A | a) < (B | b) < (Z | z)

Option Compare Text

Sub CompareText()

    Dim foo As String
    Dim bar As String
    
    '// Case insensitivity
    foo = "abc"
    bar = "ABC"
    
    Debug.Print (foo = bar) '// Prints "True"
    
    '// Still differentiates accented characters
    foo = "ábc"
    bar = "abc"
    
    Debug.Print (foo = bar) '// Prints "False"
    
    '// "b" still comes after "a" or "á"
    foo = "á"
    bar = "b"
    
    Debug.Print (bar > foo) '// Prints "True"

End Sub

Option Comparer la base de données #

La base de données de comparaison d’options n’est disponible que dans MS Access. Il définit le module/la classe pour qu’il utilise les paramètres actuels de la base de données afin de déterminer s’il faut utiliser le mode texte ou binaire.

Remarque : L’utilisation de ce paramètre est déconseillée à moins que le module ne soit utilisé pour écrire des UDF Access personnalisées (fonctions définies par l’utilisateur) qui doivent traiter les comparaisons de texte de la même manière que les requêtes SQL dans cette base de données.
