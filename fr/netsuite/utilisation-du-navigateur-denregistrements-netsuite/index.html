Utilisation du navigateur d’enregistrements NetSuite #

Le Navigateur d’enregistrements définit le schéma pour tous les types d’enregistrements inscriptibles ; c’est un outil de référence extrêmement critique pour chaque développeur SuiteScript. Lorsque vous avez besoin de savoir comment référencer un champ particulier sur un type d’enregistrement spécifique dans votre script, le Navigateur d’enregistrements est votre guide.

Lien direct

Autre schéma #

Vous pouvez également remarquer des onglets en haut du Records Browser pour Schema Browser et Connect Browser. Ceux-ci sont très similaires au Navigateur d’enregistrements, mais pour différentes API NetSuite.

Le Schema Browser fournit le schéma de l’API de services Web basée sur SOAP, tandis que le Connect Browser fournit le schéma du connecteur ODBC.

Naviguer dans le navigateur d’enregistrements #

Vous parcourez d’abord le Navigateur d’enregistrements par type d’enregistrement, c’est-à-dire “Commande client”, “Facture”, “Employé”. Il n’y a pas de fonction de recherche dans le Records Browser, donc toute la navigation se fait manuellement. Les types d’enregistrement sont organisés par ordre alphabétique, de sorte que vous cliquez d’abord sur la première lettre du type d’enregistrement qui vous intéresse, puis sélectionnez le type d’enregistrement lui-même à gauche.

Par exemple, si vous vouliez voir le schéma pour le type d’enregistrement Subsidary, vous devez d’abord cliquer sur S en haut, puis sur Subsidiary à gauche.

Lecture du schéma #

Chaque schéma vous fournit une énorme quantité d’informations sur chaque type d’enregistrement. Il est important de savoir décomposer toutes ces informations.

En haut du schéma se trouve le nom du type d’enregistrement suivi de l’ID interne du type d’enregistrement ; cet ID interne est la référence programmatique pour le type d’enregistrement. Le schéma est ensuite divisé en plusieurs sections :

  • Champs : la section Champs répertorie les détails de tous les champs corps de l’enregistrement. Les champs décrits ici peuvent être utilisés lorsque vous travaillez avec l’enregistrement actuellement en contexte ou avec une référence directe à un objet d’enregistrement.
  • Sous-listes : la section Sous-listes affiche toutes les sous-listes de l’enregistrement et chaque colonne inscriptible de chaque sous-liste. Les champs de cette section s’appliquent à nouveau lorsque vous travaillez avec l’enregistrement actuellement en contexte ou avec une référence directe à un objet d’enregistrement.
  • Onglets : la section Onglets décrit tous les sous-onglets natifs du type d’enregistrement.
  • Rechercher des jointures : la section Rechercher des jointures décrit tous les enregistrements associés à travers lesquels vous pouvez créer des jointures dans vos recherches de ce type d’enregistrement.
  • Filtres de recherche : la section Filtres de recherche décrit tous les champs disponibles en tant que filtre de recherche pour ce type d’enregistrement. L’ID interne lors de l’utilisation d’un champ spécifique comme filtre de recherche ne correspond pas toujours à son ID interne comme champ de corps.
  • Colonnes de recherche : la section Colonnes de recherche décrit tous les champs disponibles en tant que colonne de recherche pour ce type d’enregistrement. L’ID interne lors de l’utilisation d’un champ spécifique en tant que colonne de recherche ne correspond pas toujours à son ID interne en tant que champ de corps.
  • Types de transformation : la section Types de transformation décrit tous les types d’enregistrement dans lesquels celui-ci peut être transformé à l’aide de l’API de transformation d’enregistrement.

Trouver un champ #

Comme indiqué précédemment, il n’y a pas de fonction de recherche intégrée au Records Browser. Une fois que vous avez navigué jusqu’au type d’enregistrement approprié, si vous ne connaissez pas déjà l’ID interne d’un champ particulier, le moyen le plus simple de le trouver est d’utiliser la fonction Rechercher de votre navigateur (généralement CTRL+F) pour localiser le champ en son nom dans l’interface utilisateur.

Champs obligatoires #

La colonne Obligatoire du schéma indique si ce champ est obligatoire pour sauvegarder l’enregistrement. Si cette colonne indique “true”, vous devrez fournir une valeur pour ce champ lors de l’enregistrement de tout enregistrement de ce type.

nlapiSubmitField et édition en ligne #

La colonne nlapiSubmitField est un élément essentiel à comprendre. Cette colonne indique si le champ est disponible pour l’édition en ligne. Si nlapiSubmitField est true, alors le champ peut être modifié en ligne. Cela a un impact considérable sur la façon dont ce champ est géré lorsque vous essayez d’utiliser les fonctions nlapiSubmitField ou record.submitFields dans vos scripts.

Lorsque cette colonne est “true”, vous pouvez utiliser en toute sécurité les API Submit Fields pour mettre à jour ce champ en ligne. Lorsqu’il est “faux”, vous pouvez toujours utiliser ces fonctions pour mettre à jour le champ, mais ce qui se passe réellement dans les coulisses change considérablement.

Lorsque nlapiSubmitField est false pour un champ particulier et que vous utilisez l’une des API Submit Fields dessus, le moteur de script en arrière-plan effectuera en fait un chargement complet de l’enregistrement, mettra à jour le champ et soumettra la modification. à la base de données. Le résultat final est le même, mais comme l’intégralité de l’enregistrement est chargée et enregistrée, votre script utilisera en fait beaucoup plus de gouvernance que prévu et prendra plus de temps à s’exécuter.

Vous pouvez en savoir plus à ce sujet sur la page d’aide intitulée “Conséquences de l’utilisation de nlapiSubmitField sur les champs modifiables non en ligne”.
