Installation et configuration #

Arrière plan #

============

[WebStorm][1] est un environnement de développement intégré léger mais puissant (IDE) parfaitement équipé pour le développement complexe côté client et le développement côté serveur, il est multiplateforme et fonctionne sur Windows, Mac OS X et Linux.

WebStorm offre une prise en charge avancée de JavaScript, HTML, CSS et de leurs successeurs modernes, ainsi que des frameworks tels que [AngularJS][2] ou [React][3], * débogage* et intégration avec le VCS et divers outils de développement Web. en outre, il fournit un environnement de développement local configuré et prêt à l’emploi, y compris la prise en charge de [Node.js][4], [Meteor][5], [CoffeeScript][6], [TypeScript][7], [Dart ][8], [Sass][9], et plus encore.


Exigences matérielles #

=======================

Afin d’exécuter WebStorm en douceur, la configuration matérielle suivante est requise :

  • 1 Go de RAM minimum, 2 Go de RAM recommandés
  • Résolution d’écran minimale de 1024x768

Configuration requise #

=====================

Pour les versions précédentes de WebStorm avant 2016, Java doit être installé sur la machine afin d’exécuter WebStorm, à partir de WebStorm 2016 et supérieur, JRE 1.8 est fourni avec des distributions pour toutes les plates-formes. Vous n’avez pas besoin d’installer Java sur votre machine pour exécuter WebStorm.

Pour le système d’exploitation (OS), les éléments suivants sont requis :

  • Microsoft Windows 10/8/7/Vista/2003/XP (y compris 64 bits)
  • OS X 10.5 ou supérieur, y compris 10.10 (seul OS X 64 bits est pris en charge)
  • OS Linux 64 bits (bureau KDE, GNOME ou Unity DE)

#Installation de Windows :

  1. Téléchargez WebStorm à partir de la page [Télécharger][10].
  2. Exécutez le fichier WebStorm-*.exe qui démarre l’assistant d’installation. ( le * signifie la version actuelle téléchargée )
  3. Suivez toutes les étapes suggérées par l’assistant. Portez une attention particulière aux options d’installation correspondantes.

#Installation d’OS X :

  1. Téléchargez le fichier image disque OS X WebStorm-*.dmg à partir de la page [Télécharger][11]. ( le * signifie la version actuelle téléchargée )
  2. Double-cliquez sur le fichier image de disque OS X WebStorm-*.dmg téléchargé pour le monter.
  3. Copiez WebStorm dans votre dossier Applications.

#Installation Linux :

  1. Téléchargez le fichier WebStorm-*.tar.gz à partir de la page [Télécharger][12]. ( le * signifie la version actuelle téléchargée )

  2. Décompressez le fichier WebStorm-*.tar.gz dans un répertoire vide à l’aide de la commande suivante :

    tar xfz WebStorm-*.tar.gz

  3. Étant donné que l’exécution de WebStorm à partir de l’endroit où vous avez téléchargé le fichier peut être gênante, il est recommandé de déplacer le dossier d’archive extrait ou décompressé vers l’emplacement souhaité à l’aide de la commande mv dans l’un des formats suivants :

    `mv <path to extracted archive folder> <new archive folder>`

    Example #

    • mv /downloads/WebStorm-* my/desired/location
    • mv <path to WebStorm-*.tar.gz>

  4. Basculez vers le répertoire bin dans le nouvel emplacement :

    cd <new archive folder>/WebStorm-*/bin

    Example #

    • cd /WebStorm-*/bin

  5. Exécutez webstorm.sh à partir du sous-répertoire bin.

Créer un lanceur de ligne de commande #

Un lanceur de ligne de commande est un outil de gestion qui permet d’ouvrir WebStorm à l’aide de la ligne de commande. Il peut facilement être créé en utilisant les menus

Tools > Create Command-line Launcher...

[![Capture d’écran de l’option “Créer un lanceur de ligne de commande…” dans le menu][1]][1]

Après avoir sélectionné l’option, l’invite “Créer un script de lancement” vous sera présentée pour un emplacement et un nom du lanceur de ligne de commande. Cet emplacement doit se trouver dans $PATH afin qu’il puisse être appelé dans l’invite de commande/le terminal.

La valeur par défaut est /usr/local/bin/webstorm où /usr/local/bin est le chemin vers lequel le lanceur sera ajouté et webstorm est le nom du fichier. Le nom du fichier sera également utilisé comme commande pour invoquer et lancer WebStorm, il peut le changer, mais pour cet exemple nous supposerons qu’il s’agit de la valeur par défaut de “webstorm”

[![Capture d’écran de l’invite “Créer un script de lancement”][2]][2]

Une fois cela fait, vous pouvez ouvrir un terminal ou une invite de commande et utiliser la commande “webstorm” pour lancer WebStorm avec un répertoire spécifique. Dans l’exemple ci-dessous, WebStorm sera ouvert avec le répertoire ~/Workspace/my-project à la racine du projet.

> webstorm ~/Workspace/my-project

Astuce : Utilisez webstorm . pour ouvrir le répertoire courant.
