La récursivité est principalement disponible dans les saveurs compatibles Perl, telles que :

  • Perle
  • PCRE
  • Oniguruma
  • Augmenter

Recurse dans un sous-modèle #

Vous pouvez revenir en arrière dans un sous-modèle en utilisant les constructions suivantes (selon le type), en supposant que n est un numéro de groupe de capture et name le nom d’un groupe de capture.

  • (?n)
  • \g<n>
  • \g'0'
  • (?&nom)
  • \g<nom>
  • \g'nom'
  • (?P>nom)

Le modèle suivant :

 \[(?<angle><(?&angle)*+>)*\]

Correspondra à du texte tel que : [<<><>><>] - crochets angulaires bien équilibrés entre crochets. La récursivité est souvent utilisée pour la correspondance de constructions équilibrées.

Les récursions sont atomiques (PCRE) #

Dans PCRE, il n’y a pas de rétrolien après la première correspondance pour une récursivité. Alors

(?(DEFINE)(aaa|aa|a))(?1)ab

ne correspond pas

aab

car après avoir trouvé ‘aa’ dans la récursivité, il n’essaie plus jamais de faire correspondre uniquement ‘a’.

Récurser tout le motif #

La construction (?R) est équivalente à (?0) (ou \g<0>) - elle vous permet de parcourir tout le motif :

<(?>[^<>]+|(?R))+>

Cela correspondra à des chevrons correctement équilibrés avec tout texte entre les crochets, comme <a<b>c<d>e>.

Définitions des sous-modèles #

La construction (?(DEFINE)…) vous permet de définir des sous-modèles auxquels vous pourrez vous référer ultérieurement par récursivité. Lorsqu’il est rencontré dans le modèle, il ne sera pas comparé.

Ce groupe doit contenir des définitions de sous-modèles nommés, qui ne seront accessibles que par récursivité. Vous pouvez définir les grammaires comme suit :

(?x) # ignore pattern whitespace
(?(DEFINE)
  (?<string> ".*?" )
  (?<number> \d+ )
  (?<value>
    \s* (?:
        (?&string)
      | (?&number)
      | (?&list)
    ) \s*
  )
  (?<list> \[ (?&value) (?: , (?&value) )* \] )
)
^(?&value)$

Ce modèle validera un texte comme celui-ci :

[42, "abc", ["foo", "bar"], 10]

Notez qu’une liste peut contenir une ou plusieurs valeurs et qu’une valeur peut elle-même être une liste.

Références de groupe relatives #

Les sous-modèles peuvent être référencés avec leur numéro de groupe relatif :

  • (?-1) reviendra dans le groupe précédent
  • (?+1) reviendra dans le groupe suivant

Également utilisable avec la syntaxe \g<N>.

Références arrière dans les récursions (PCRE) #

Dans PCRE, les groupes appariés utilisés pour les références arrière avant une récursivité sont conservés dans la récursivité. Mais après la récursivité, ils se sont tous réinitialisés à ce qu’ils étaient avant d’y entrer. En d’autres termes, les groupes appariés dans la récursivité sont tous oubliés.

Par exemple:

(?J)(?(DEFINE)(\g{a}(?<a>b)\g{a}))(?<a>a)\g{a}(?1)\g{a}
