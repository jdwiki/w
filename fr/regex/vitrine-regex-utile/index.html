Faire correspondre une adresse e-mail #

Faire correspondre une adresse e-mail dans une chaîne est une tâche difficile, car spécification le définissant, la RFC2822, est complexe, ce qui le rend difficile à implémenter en tant que regex. Pour plus de détails pourquoi c’est ce n’est pas une bonne idée de faire correspondre un email avec une regex, veuillez vous référer au exemple d’antipattern [quand ne pas utiliser une regex : pour la correspondance e-mails] (https://www.wikiod.com/fr/regex/quand-vous-ne-devriez-pas-utiliser-dexpressions-regulieres numéro-ou-numéro-de-carte-de-crédit-ou#t=201607301200162540574). Le meilleur conseil à noter sur cette page est d’utiliser un examen par les pairs et largement bibliothèque dans votre langue préférée pour l’implémenter.

Valider un format d’adresse email #

Lorsque vous devez valider rapidement une entrée pour vous assurer qu’elle ressemble à un e-mail, la meilleure option est de rester simple :

^\S{1,}@\S{2,}\.\S{2,}$

Cette expression régulière vérifiera que l’adresse e-mail est une séquence non séparée par des espaces de caractères de longueur supérieure à un, suivi d’un @, suivi de deux séquences de caractères non espaces de longueur deux ou plus séparés par un .. Il n’est pas parfait et peut valider des adresses invalides (selon le format), mais surtout, il n’invalide pas les adresses valides.

Vérifiez que l’adresse existe #

Le seul moyen fiable de vérifier qu’un e-mail est valide est de vérifier son existence. Auparavant, la commande SMTP “VRFY” avait été conçue à cet effet, mais malheureusement, après [avoir été abusé par des spammeurs, il n’est plus disponible] (http://stackoverflow.com/a/566121/1290438).

Donc, la seule façon qu’il vous reste pour vérifier que le courrier est valide et existe est de effectivement envoyer un e-mail à cette adresse.

D’énormes alternatives Regex #

Cependant, il n’est pas impossible de valider une adresse e-mail à l’aide d’une regex. Les seuls problèmes est que plus ces regex seront proches de la spécification, plus elles seront grandes et par conséquent, ils sont incroyablement difficiles à lire et à maintenir. Ci-dessous, vous trouverez un exemple de ces regex plus précises qui sont utilisées dans certaines bibliothèques.

⚠️ Les regex suivantes sont données à des fins de documentation et d’apprentissage, copier coller les dans votre code est une mauvaise idée. Au lieu de cela, utilisez cette bibliothèque directement, afin que vous puissiez compter sur code en amont et développeurs pairs pour maintenir votre code d’analyse d’e-mails à jour et maintenu.

Module de correspondance d’adresse Perl #

Les meilleurs exemples de telles regex se trouvent dans certaines bibliothèques standard de langages. Par exemple, il y en a un du RFC::RFC822::Address module dans la bibliothèque Perl qui essaie d’être aussi précise que possible selon la RFC. Pour votre curiosité, vous pouvez trouver une version de cette regex à cette URL, qui a été généré à partir de la grammaire, et si vous êtes tenté de le copier coller, voici la citation de l’auteur de la regex :

“Je ne maintiens pas l’expression régulière [liée]. Il peut y avoir des bogues qui ont déjà été corrigés dans le module Perl.”

.Module de correspondance d’adresses réseau #

Une autre variante, plus courte, est celle utilisée par la bibliothèque standard .Net dans le Module “EmailAddressAttribute” :

^((([a-z]|\d|[!#\$%&'\*\+\-\/=\?\^_`{\|}~]|[\u00A0-\uD7FF\uF900-\uFDCF\uFDF0-\uFFEF])+(\.([a-z]|\d|[!#\$%&'\*\+\-\/=\?\^_`{\|}~]|[\u00A0-\uD7FF\uF900-\uFDCF\uFDF0-\uFFEF])+)*)|((\x22)((((\x20|\x09)*(\x0d\x0a))?(\x20|\x09)+)?(([\x01-\x08\x0b\x0c\x0e-\x1f\x7f]|\x21|[\x23-\x5b]|[\x5d-\x7e]|[\u00A0-\uD7FF\uF900-\uFDCF\uFDF0-\uFFEF])|(\\([\x01-\x09\x0b\x0c\x0d-\x7f]|[\u00A0-\uD7FF\uF900-\uFDCF\uFDF0-\uFFEF]))))*(((\x20|\x09)*(\x0d\x0a))?(\x20|\x09)+)?(\x22)))@((([a-z]|\d|[\u00A0-\uD7FF\uF900-\uFDCF\uFDF0-\uFFEF])|(([a-z]|\d|[\u00A0-\uD7FF\uF900-\uFDCF\uFDF0-\uFFEF])([a-z]|\d|-|\.|_|~|[\u00A0-\uD7FF\uF900-\uFDCF\uFDF0-\uFFEF])*([a-z]|\d|[\u00A0-\uD7FF\uF900-\uFDCF\uFDF0-\uFFEF])))\.)+(([a-z]|[\u00A0-\uD7FF\uF900-\uFDCF\uFDF0-\uFFEF])|(([a-z]|[\u00A0-\uD7FF\uF900-\uFDCF\uFDF0-\uFFEF])([a-z]|\d|-|\.|_|~|[\u00A0-\uD7FF\uF900-\uFDCF\uFDF0-\uFFEF])*([a-z]|[\u00A0-\uD7FF\uF900-\uFDCF\uFDF0-\uFFEF])))\.?$

Mais même s’il est plus court, il reste trop gros pour être lisible et facilement maintenable.

Module de correspondance d’adresse Ruby #

En ruby, une composition de regex est utilisée dans le module rfc822 pour faire correspondre une adresse. C’est une bonne idée, car si des bogues sont trouvés, il sera plus facile d’identifier la partie regex à modifier et répare le.

Faire correspondre une date #

Vous devez vous rappeler que regex a été conçu pour faire correspondre une date (ou non). Dire qu’une date est valide est une lutte beaucoup plus compliquée, car cela nécessitera beaucoup de gestion des exceptions (voir conditions d’année bissextile).

Commençons par faire correspondre le mois (1 - 12) avec un 0 initial facultatif :

0?[1-9]|1[0-2]

Pour faire correspondre le jour, également avec un 0 initial facultatif :

0?[1-9]|[12][0-9]|3[01]

Et pour correspondre à l’année (supposons simplement la plage 1900 - 2999):

(?:19|20)[0-9]{2}

Le séparateur peut être un espace, un tiret, une barre oblique, un vide, etc. N’hésitez pas à ajouter tout ce qui, selon vous, peut être utilisé comme séparateur :

[-\\/ ]?

Maintenant, vous concaténez le tout et obtenez:

(0?[1-9]|1[0-2])[-\\/ ]?(0?[1-9]|[12][0-9]|3[01])[-/ ]?(?:19|20)[0-9]{2} // MMDDYYYY
(0?[1-9]|[12][0-9]|3[01])[-\\/ ]?(0?[1-9]|1[0-2])[-/ ]?(?:19|20)[0-9]{2} // DDMMYYYY
(?:19|20)[0-9]{2}[-\\/ ]?(0?[1-9]|1[0-2])[-/ ]?(0?[1-9]|[12][0-9]|3[01]) // YYYYMMDD

Si vous voulez être un peu plus pédant, vous pouvez utiliser une référence arrière pour être sûr que les deux séparateurs seront les mêmes :

(0?[1-9]|1[0-2])([-\\/ ]?)(0?[1-9]|[12][0-9]|3[01])\2(?:19|20)[0-9]{2} // MMDDYYYY
                                                 ^ refer to [-/ ]
(0?[1-9]|[12][0-9]|3[01])([-\\/ ]?)(0?[1-9]|1[0-2])\2(?:19|20)[0-9]{2} // DDMMYYYY
(?:19|20)[0-9]{2}([-\\/ ]?)(0?[1-9]|1[0-2])\2(0?[1-9]|[12][0-9]|3[01]) // YYYYMMDD

Faire correspondre un numéro de téléphone #

Voici comment faire correspondre un code de préfixe (un + ou (00), puis un nombre de 1 à 1939, avec un espace facultatif) : Cela ne recherche pas un préfixe valide mais quelque chose qui pourrait être un préfixe. Voir la liste complète des préfixes

(?:00|\+)?[0-9]{4}

Ensuite, comme la longueur totale du numéro de téléphone est au maximum de 15, nous pouvons rechercher jusqu’à 14 chiffres : Au moins 1 chiffre est utilisé pour le préfixe

[0-9]{1,14}

Les nombres peuvent contenir des espaces, des points ou des tirets et peuvent être groupés par 2 ou 3.

(?:[ .-][0-9]{3}){1,5}

Avec le préfixe facultatif :

(?:(?:00|\+)?[0-9]{4})?(?:[ .-][0-9]{3}){1,5}

Si vous souhaitez faire correspondre un format de pays spécifique, vous pouvez utiliser cette requête de recherche et ajouter le pays, la question a certainement déjà été a demandé.

Faire correspondre une adresse IP #

IPv4

Pour correspondre au format d’adresse IPv4, vous devez vérifier les nombres [0-9]{1,3} trois fois {3} séparés par des points \. et se terminant par un autre nombre.

^(?:[0-9]{1,3}\.){3}[0-9]{1,3}$

Cette expression régulière est trop simple - si vous voulez qu’elle soit précise, vous devez vérifier que les nombres sont compris entre ‘0’ et ‘255’, l’expression régulière ci-dessus acceptant ‘444’ dans n’importe quelle position. Vous voulez vérifier 250-255 avec 25[0-5], ou toute autre valeur 200 2[0-4][0-9], ou toute valeur 100 ou moins avec [01] ?[ 0-9][0-9]. Vous voulez vérifier qu’il est suivi d’un point \. trois fois {3} puis une fois sans point.

^(?:(?:25[0-5]|2[0-4][0-9]|[01]?[0-9][0-9]?)\.){3}(?:25[0-5]|2[0-4][0-9]|[01]?[0-9][0-9]?)$

IPv6

Les adresses IPv6 prennent la forme de 8 mots hexadécimaux de 16 bits délimités par le caractère deux-points (:). Dans ce cas, nous vérifions 7 mots suivis de deux-points, suivis d’un qui ne l’est pas. Si un mot a des zéros non significatifs, ils peuvent être tronqués, ce qui signifie que chaque mot peut contenir entre 1 et 4 chiffres hexadécimaux.

^(?:[0-9a-fA-F]{1,4}:){7}[0-9a-fA-F]{1,4}$

Ceci est toutefois insuffisant. Comme les adresses IPv6 peuvent devenir assez “verbeuses”, la norme spécifie que les mots contenant uniquement des zéros peuvent être remplacés par ::. Cela ne peut être fait qu’une seule fois dans une adresse (pour n’importe où entre 1 et 7 mots consécutifs), car cela serait autrement indéterminé. Cela produit un certain nombre de variations (plutôt désagréables):

^::(?:[0-9a-fA-F]{1,4}:){0,6}[0-9a-fA-F]{1,4}$
^[0-9a-fA-F]{1,4}::(?:[0-9a-fA-F]{1,4}:){0,5}[0-9a-fA-F]{1,4}$
^[0-9a-fA-F]{1,4}:[0-9a-fA-F]{1,4}::(?:[0-9a-fA-F]{1,4}:){0,4}[0-9a-fA-F]{1,4}$
^(?:[0-9a-fA-F]{1,4}:){0,2}[0-9a-fA-F]{1,4}::(?:[0-9a-fA-F]{1,4}:){0,3}[0-9a-fA-F]{1,4}$
^(?:[0-9a-fA-F]{1,4}:){0,3}[0-9a-fA-F]{1,4}::(?:[0-9a-fA-F]{1,4}:){0,2}[0-9a-fA-F]{1,4}$
^(?:[0-9a-fA-F]{1,4}:){0,4}[0-9a-fA-F]{1,4}::(?:[0-9a-fA-F]{1,4}:)?[0-9a-fA-F]{1,4}$
^(?:[0-9a-fA-F]{1,4}:){0,5}[0-9a-fA-F]{1,4}::[0-9a-fA-F]{1,4}$
^(?:[0-9a-fA-F]{1,4}:){0,6}[0-9a-fA-F]{1,4}::$

Maintenant, en mettant tout cela ensemble (en utilisant l’alternance), on obtient :

^(?:[0-9a-fA-F]{1,4}:){7}[0-9a-fA-F]{1,4}$|
^::(?:[0-9a-fA-F]{1,4}:){0,6}[0-9a-fA-F]{1,4}$|
^[0-9a-fA-F]{1,4}::(?:[0-9a-fA-F]{1,4}:){0,5}[0-9a-fA-F]{1,4}$|
^[0-9a-fA-F]{1,4}:[0-9a-fA-F]{1,4}::(?:[0-9a-fA-F]{1,4}:){0,4}[0-9a-fA-F]{1,4}$|
^(?:[0-9a-fA-F]{1,4}:){0,2}[0-9a-fA-F]{1,4}::(?:[0-9a-fA-F]{1,4}:){0,3}[0-9a-fA-F]{1,4}$|
^(?:[0-9a-fA-F]{1,4}:){0,3}[0-9a-fA-F]{1,4}::(?:[0-9a-fA-F]{1,4}:){0,2}[0-9a-fA-F]{1,4}$|
^(?:[0-9a-fA-F]{1,4}:){0,4}[0-9a-fA-F]{1,4}::(?:[0-9a-fA-F]{1,4}:)?[0-9a-fA-F]{1,4}$|
^(?:[0-9a-fA-F]{1,4}:){0,5}[0-9a-fA-F]{1,4}::[0-9a-fA-F]{1,4}$|
^(?:[0-9a-fA-F]{1,4}:){0,6}[0-9a-fA-F]{1,4}::$

Assurez-vous de l’écrire en mode multiligne et avec une pile de commentaires afin que celui qui est inévitablement chargé de comprendre ce que cela signifie ne vous poursuive pas avec un objet contondant.

Valider une chaîne horaire de 12h et 24h #

Pour un format horaire de 12 heures, on peut utiliser :

^(?:0?[0-9]|1[0-2])[-:][0-5][0-9]\s*[ap]m$

Où

  • (?:0?[0-9]|1[0-2]) est l’heure
  • [-:] est le séparateur, qui peut être ajusté selon vos besoins
  • [0-5][0-9] est la minute
  • \s*[ap]m suivi de n’importe quel nombre de caractères d’espacement, et am ou pm

Si vous avez besoin des secondes :

^(?:0?[0-9]|1[0-2])[-:][0-5][0-9][-:][0-5][0-9]\s*[ap]m$

Pour un format d’heure de 24h :

^(?:[01][0-9]|2[0-3])[-:h][0-5][0-9]$

Où:

  • (?:[01][0-9]|2[0-3]) est l’heure
  • [-:h] le séparateur, qui peut être ajusté selon vos besoins
  • [0-5][0-9] est la minute

Avec les secondes :

^(?:[01][0-9]|2[0-3])[-:h][0-5][0-9][-:m][0-5][0-9]$

Où [-:m] est un second séparateur, remplaçant le h pour les heures par un m pour les minutes, et [0-5][0-9] est le second.

Correspond au code postal du Royaume-Uni #

Regex pour correspondre à [codes postaux au Royaume-Uni] [1]

Le format est le suivant, où A signifie une lettre et 9 un chiffre :

Formater Couverture Exemple
Cellule Cellule
AA9A 9AA Zone de code postal WC ; EC1–EC4, NW1W, SE1P, SW1 EC1A 1BB
A9A 9AA E1W, N1C, N1P W1A 0AX
A9 9AA, A99 9AA B, E, G, L, M, N, S, W M1 1AE, B33 8TH
AA9 9AA, AA99 9AA Tous les autres codes postaux CR2 6XH, DN55 1PT 
(GIR 0AA)|((([A-Z-[QVX]][0-9][0-9]?)|(([A-Z-[QVX]][A-Z-[IJZ]][0-9][0-9]?)|(([A-Z-[QVX]][0-9][A-HJKPSTUW])|([A-Z-[QVX]][A-Z-[IJZ]][0-9][ABEHMNPRVWXY])))) [0-9][A-Z-[CIKMOV]]{2})

Où première partie :

 (GIR 0AA)|((([A-Z-[QVX]][0-9][0-9]?)|(([A-Z-[QVX]][A-Z-[IJZ]][0-9][0-9]?)|(([A-Z-[QVX]][0-9][A-HJKPSTUW])|([A-Z-[QVX]][A-Z-[IJZ]][0-9][ABEHMNPRVWXY]))))

Deuxième:

[0-9][A-Z-[CIKMOV]]{2})
