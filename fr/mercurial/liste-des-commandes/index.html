Commandes pour préparer les commits #

  • add : ajoute les fichiers spécifiés au prochain commit
  • addremove : ajoutez tous les nouveaux fichiers, supprimez tous les fichiers manquants
  • backout : effet inverse du jeu de modifications précédent
  • commit, ci : validez les fichiers spécifiés ou toutes les modifications en suspens
  • copy, cp : marque les fichiers comme copiés pour le prochain commit
  • forget : oublie les fichiers spécifiés lors du prochain commit
  • merge : fusionne une autre révision dans le répertoire de travail
  • remove, rm : supprime les fichiers spécifiés au prochain commit
  • renommer, déplacer, mv : renommer les fichiers ; équivalent de copier + supprimer
  • Résoudre : rétablir les fusions ou définir/afficher l’état de fusion des fichiers
  • revert : restaure les fichiers à leur état fraîchement extrait

Inspecter l’historique #

  • annoter, blâmer : afficher les informations sur le jeu de modifications par ligne pour chaque fichier
  • bisect : recherche de subdivision des changesets
  • cat : affiche la révision actuelle ou donnée des fichiers
  • diff : référentiel diff (ou fichiers sélectionnés)
  • grep : recherche un modèle dans les fichiers et révisions spécifiés
  • log, history : affiche l’historique des révisions de l’ensemble du référentiel ou des fichiers

Échanger des ensembles de modifications avec des dépôts distants #

  • archive : crée une archive non versionnée d’une révision du référentiel
  • bundle : créer un fichier changegroup
  • clone : faire une copie d’un référentiel existant
  • export : vider l’en-tête et les diffs pour un ou plusieurs ensembles de modifications
  • greffer : copier les modifications des autres branches sur la branche actuelle
  • entrant : affiche les nouveaux ensembles de modifications trouvés dans la source
  • import, patch : importez un ensemble ordonné de patchs
  • init : crée un nouveau dépôt dans le répertoire donné
  • sortant : affiche les ensembles de modifications introuvables dans la destination
  • phase : définir ou afficher le nom de la phase actuelle
  • pull : extrait les modifications de la source spécifiée
  • push : envoie les modifications vers la destination spécifiée
  • recover : annuler une transaction interrompue
  • rollback : annuler la dernière transaction (DANGEREUX) (OBSOLÈTE)
  • servir : démarrer le serveur Web autonome
  • unbundle : appliquez un ou plusieurs fichiers changegroup

Statut : où en êtes-vous ? #

  • signets, signet : créez un nouveau signet ou répertoriez les signets existants
  • branch : définir ou afficher le nom de la branche actuelle
  • branches : répertorie les branches nommées du référentiel
  • config, showconfig, debugconfig : affiche les paramètres de configuration combinés de tous les fichiers hgrc
  • fichiers : répertorie les fichiers suivis
  • help : affiche l’aide pour un sujet donné ou un aperçu de l’aide
  • identifier, id : identifier le répertoire de travail ou la révision spécifiée
  • incoming, in : affiche les nouveaux ensembles de modifications trouvés dans la source
  • localiser : localiser les fichiers correspondant à des modèles spécifiques (OBSOLÈTE)
  • manifest : affiche la révision actuelle ou donnée du manifeste du projet
  • sortant, sortant : affiche les ensembles de modifications introuvables dans la destination
  • parents : affiche les parents du répertoire de travail ou de la révision (OBSOLÈTE)
  • chemins : affiche les alias pour les référentiels distants
  • phase : définir ou afficher le nom de la phase actuelle
  • root : affiche la racine (en haut) du répertoire de travail actuel
  • status, st : affiche les fichiers modifiés dans le répertoire de travail
  • summary, sum : résume l’état du répertoire de travail
  • tags : répertorie les balises du référentiel
  • astuce : affiche la révision de l’astuce (obsolète)
  • verify : vérifie l’intégrité du dépôt
  • version : version de sortie et informations de copyright

Workflow : branches, balises et déplacements #

  • signets, signet : créez un nouveau signet ou répertoriez les signets existants
  • branch : définir ou afficher le nom de la branche actuelle
  • tag : ajoutez un ou plusieurs tags pour la révision actuelle ou donnée
  • update, up, checkout, co : mettre à jour le répertoire de travail (ou changer de révision)
