Système de coordonnées #

Dans un système de coordonnées mathématique normal, le point x=0, y=0 se trouve dans le coin inférieur gauche du graphique. Mais dans le système de coordonnées SVG, ce point (0,0) se trouve dans le coin supérieur gauche de la ’toile’, il est un peu similaire à CSS lorsque vous spécifiez la position absolue/fixe et utilisez haut et gauche pour contrôler le point exact de l’élément.

Il est essentiel de garder à l’esprit que lorsque y augmente dans SVG, les formes se déplacent vers le bas.

Disons que nous allons créer un nuage de points avec chaque point correspondant à une valeur x et une valeur y. Pour mettre à l’échelle la valeur, nous devons définir le domaine et la plage comme ceci :

d3.svg.scale()
  .range([0, height])
  .domain([0,max])

Cependant, si vous conservez simplement les paramètres comme celui-ci, les points seront basés sur le bord horizontal supérieur au lieu de la ligne horizontale inférieure comme prévu.

La bonne chose à propos de d3 est que vous pouvez facilement changer cela par un simple ajustement dans le paramètre de domaine :

d3.scale.linear()
  .range([height, 0])
  .domain([0, max])

Avec le code ci-dessus, le point zéro du domaine correspond à la hauteur du SVG, qui est la ligne inférieure du graphique aux yeux du spectateur, tandis que la valeur maximale des données source correspondra au point zéro du SVG système de coordonnées, qui est la valeur maximale pour les téléspectateurs.

L’élément #

<rect> représente un rectangle, à part les propriétés esthétiques comme le trait et le remplissage, le rectangle doit être défini par l’emplacement et la taille.

Quant à l’emplacement, il est déterminé par les attributs x et y. L’emplacement est relatif au parent du rectangle. Et si vous ne spécifiez pas l’attribut x ou y, la valeur par défaut sera 0 par rapport à l’élément parent.

Après avoir spécifié l’emplacement, ou plutôt le ‘point de départ’ du rect, la prochaine chose est de spécifier la taille, ce qui est essentiel si vous voulez réellement dessiner qch sur la toile, c’est-à-dire si vous ne le faites pas spécifiez les attributs de taille ou la valeur est définie sur 0, vous ne verrez rien sur le canevas.

Cas : Diagramme à barres

Continuez avec le premier scénario, les axes y, mais cette fois, essayons de dessiner un graphique à barres.

En supposant que le paramètre d’échelle y est le même, l’axe y est également correctement défini, la seule différence entre le nuage de points et ce graphique à barres est que nous devons spécifier la largeur et la hauteur, en particulier la hauteur. Pour être plus précis, nous avons déjà le “point de départ”, le reste consiste à utiliser des choses comme pour la hauteur :

.attr("height", function(d){
  return (height - yScale(d.value))
})

L’élément

L’élément <svg> est l’élément racine, ou le canevas car nous dessinons des graphiques dessus.

Les éléments SVG peuvent être imbriqués les uns dans les autres, et de cette manière, les formes SVG peuvent être regroupées, tandis que toutes les formes imbriquées dans un élément

Une chose mérite d’être mentionnée, c’est que nous ne pouvons pas imbriquer dans un autre , cela ne fonctionnera pas.

L’élément #

SVG ne prend actuellement pas en charge les sauts de ligne automatiques ou le retour à la ligne, c’est là que vient à la rescousse. L’élément positionne les nouvelles lignes de texte par rapport à la ligne de texte précédente. Et en utilisant dx ou dy dans chacune de ces étendues, nous pouvons positionner le mot par rapport au mot qui le précède.

Cas : Annotation sur les axes

Par exemple, lorsque nous voulons ajouter une annotation sur y Axe :

svg.append("g")
  .attr("class", "y axis")
  .call(yAxis)
.append("text")
  .attr("transform", "rotate(-90)")
  .attr("y", 6)
  .attr("dy", ".71em")
  .style("text-anchor", "end")
  .text("Temperature (ºF)");

Ajout correct d’un élément SVG #

C’est une erreur relativement courante : vous avez créé un élément rect, dans un graphique à barres par exemple, et vous souhaitez ajouter une étiquette de texte (disons, la valeur de cette barre). Ainsi, en utilisant la même variable que vous avez utilisée pour ajouter le rect et définir sa position x et y, vous ajoutez votre élément text. Très logique, pensez-vous. Mais cela ne fonctionnera pas.

Comment se produit cette erreur ?

Voyons un exemple concret, un code très basique pour créer un histogramme ([fiddle here][1]) :

var data = [210, 36, 322, 59, 123, 350, 290];

var width = 400, height = 300;

var svg = d3.select("body")
    .append("svg")
    .attr("width", width)
    .attr("height", height);

var bars = svg.selectAll(".myBars")
    .data(data)
    .enter()
    .append("rect");

bars.attr("x", 10)
    .attr("y", function(d,i){ return 10 + i*40})
    .attr("width", function(d){ return d})
    .attr("height", 30);

Ce qui nous donne ce résultat :

[![entrez la description de l’image ici][2]][2]

Mais vous souhaitez ajouter des éléments de texte, peut-être une simple valeur à chaque barre. Alors, tu fais ça :

bars.append("text")
   .attr("x", 10)
   .attr("y", function(d,i){ return 10 + i*40})
   .text(function(d){ return d});

Et voilà : rien ne se passe ! Si vous en doutez, [voici le violon][3].

“Mais je vois l’étiquette !”

Si vous inspectez le SVG créé par ce dernier code, vous verrez ceci :

[![entrez la description de l’image ici][4]][4]

Et à ce stade, beaucoup de gens disent : “Mais je vois la balise de texte, elle est ajoutée !”. Oui, c’est le cas, mais cela ne signifie pas que cela fonctionnera. Vous pouvez ajouter n’importe quoi ! Voir ceci, par exemple :

svg.append("crazyTag");

Cela vous donnera ce résultat :

<svg>
    <crazyTag></crazyTag>
</svg>

Mais vous ne vous attendez à aucun résultat simplement parce que la balise est là, n’est-ce pas ?

Ajouter les éléments SVG dans le bon sens

Découvrez quels éléments SVG peuvent contenir des enfants en lisant les [spécifications][5]. Dans notre dernier exemple, le code ne fonctionne pas car les éléments rect ne peuvent pas contenir d’éléments text. Alors, comment afficher nos textes ?

Créez une autre variable et ajoutez le “texte” au SVG :

var texts = svg.selectAll(".myTexts")
    .data(data)
    .enter()
    .append("text");

texts.attr("x", function(d){ return d + 16})
    .attr("y", function(d,i){ return 30 + i*40})
    .text(function(d){ return d});

Et voici le résultat :

SVG : l’ordre de dessin #

C’est quelque chose qui peut être frustrant : vous faites une visualisation en utilisant D3.js mais le rectangle que vous voulez en haut est caché derrière un autre rectangle, ou la ligne que vous aviez prévu d’être derrière un cercle est en fait au-dessus. Vous essayez de résoudre ce problème en utilisant le z-index dans votre CSS, mais cela ne fonctionne pas (dans SVG 1.1).

L’explication est simple : dans un SVG, l’ordre des éléments définit l’ordre de la “peinture”, et l’ordre de la peinture définit qui va en haut.

Les éléments d’un fragment de document SVG ont un ordre de dessin implicite, les premiers éléments du fragment de document SVG étant “peints” en premier. Les éléments suivants sont peints au-dessus des éléments précédemment peints.

Donc, supposons que nous ayons ce SVG :

<svg width="400" height=200>
    <circle cy="100" cx="80" r="60" fill="blue"></circle>
    <circle cy="100" cx="160" r="60" fill="yellow"></circle>
    <circle cy="100" cx="240" r="60" fill="red"></circle>
    <circle cy="100" cx="320" r="60" fill="green" z-index="-1"></circle>
</svg>

Il a quatre cercles. Le cercle bleu est le premier “peint”, il sera donc en dessous de tous les autres. Ensuite, nous avons le jaune, puis le rouge et enfin le vert. Le vert est le dernier, et il sera en haut.

Voici à quoi ça ressemble:

Changer l’ordre des éléments SVG avec D3

Alors, est-il possible de changer l’ordre des éléments ? Puis-je faire le cercle rouge devant le cercle vert ?

Oui. La première approche que vous devez avoir à l’esprit est l’ordre des lignes dans votre code : dessinez d’abord les éléments de l’arrière-plan, et plus tard dans le code les éléments du premier plan.

Mais nous pouvons changer dynamiquement l’ordre des éléments, même après qu’ils aient été peints. Il existe plusieurs fonctions JavaScript simples que vous pouvez écrire pour ce faire, mais D3 a déjà 2 fonctionnalités intéressantes, selection.raise() et selection.lower().

Selon l’API :

selection.raise() : réinsère chaque élément sélectionné, dans l’ordre, en tant que dernier enfant de son parent. selection.lower() : réinsère chaque élément sélectionné, dans l’ordre, en tant que premier enfant de son parent.

Donc, pour montrer comment manipuler l’ordre des éléments dans notre SVG précédent, voici un tout petit code :

d3.selectAll("circle").on("mouseover", function(){
    d3.select(this).raise(); 
});

Qu’est ce que ça fait? Il sélectionne tous les cercles et, lorsque l’utilisateur survole un cercle, il sélectionne ce cercle particulier et le place au premier plan. Très simple!
