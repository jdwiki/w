Le rôle des espaces réservés dans les sélections “enter” #

Qu’est-ce qu’une sélection d’entrée ?

En D3.js, lorsque l’on lie des données à des éléments DOM, trois situations sont possibles :

  1. Le nombre d’éléments et le nombre de points de données sont les mêmes ;
  2. Il y a plus d’éléments que de points de données ;
  3. Il y a plus de points de données que d’éléments ;

Dans la situation #3, tous les points de données sans élément DOM correspondant appartiennent à la sélection enter. Ainsi, dans D3.js, les sélections enter sont des sélections qui, après avoir joint des éléments aux données, contiennent toutes les données qui ne correspondent à aucun élément DOM. Si nous utilisons une fonction append dans une sélection enter, D3 créera de nouveaux éléments liant ces données pour nous.

Il s’agit d’un diagramme de Venn expliquant les situations possibles concernant le nombre de points de données/nombre d’éléments DOM :

[![entrez la description de l’image ici][1]][1]

Comme nous pouvons le voir, la sélection enter est la zone bleue à gauche : points de données sans éléments DOM correspondants.

La structure de la sélection d’entrée

Typiquement, une sélection enter a ces 4 étapes :

  1. selectAll : sélectionnez des éléments dans le DOM ;
  2. data : compte et analyse les données ;
  3. enter : en comparant la sélection avec les données, crée de nouveaux éléments ;
  4. append : ajoute les éléments réels dans le DOM ;

Ceci est un exemple très basique (regardez les 4 étapes dans le var divs):

var data = [40, 80, 150, 160, 230, 260];

var body = d3.select("body");

var divs = body.selectAll("div")
    .data(data)
    .enter()
    .append("div");

divs.style("width", function(d) { return d + "px"; })
    .attr("class", "divchart")
    .text(function(d) { return d; });

Et voici le résultat ([jsfiddle ici][2]):

[![entrez la description de l’image ici][3]][3]

Notez que, dans ce cas, nous avons utilisé selectAll("div") comme première ligne dans notre variable de sélection “enter”. Nous avons un ensemble de données avec 6 valeurs et D3 a créé 6 divs pour nous.

Le rôle des espaces réservés

Mais supposons que nous ayons déjà une div dans notre document, quelque chose comme <div>Ceci est mon graphique</div> en haut. Dans ce cas, lorsque nous écrivons :

body.selectAll("div")

nous sélectionnons cette div existante. Ainsi, notre sélection d’entrée n’aura que 5 données sans éléments correspondants. Par exemple, [dans ce jsfiddle][4], où il y a déjà un div dans le HTML (“Ceci est mon graphique”), voici le résultat :

[![entrez la description de l’image ici][5]][5]

On ne voit plus la valeur “40” : notre première “barre” a disparu, et la raison en est que notre sélection “enter” ne comporte plus que 5 éléments.

Ce que nous devons comprendre ici, c’est que dans la première ligne de notre variable de sélection d’entrée, selectAll("div"), ces divs ne sont que des espaces réservés. Nous n’avons pas à sélectionner tous les divs si nous ajoutons divs, ou tous les circle si nous ajoutons circle. Nous pouvons sélectionner différentes choses. Et, si nous ne prévoyons pas d’avoir une sélection “mise à jour” ou “sortie”, nous pouvons sélectionner n’importe quoi :

var divs = body.selectAll(".foo")//this class doesn't exist, and never will!
    .data(data)
    .enter()
    .append("div");

En procédant ainsi, nous sélectionnons tous les “.foo”. Ici, “foo” est une classe qui non seulement n’existe pas, mais en plus elle n’est jamais créée ailleurs dans le code ! Mais ce n’est pas grave, ce n’est qu’un espace réservé. La logique est celle-ci :

Si dans votre sélection “entrer” vous sélectionnez quelque chose qui n’existe pas, votre sélection “entrer” contiendra toujours toutes vos données.

Maintenant, en sélectionnant .foo, notre sélection “enter” a 6 éléments, même si nous avons déjà un div dans le document :

[![entrez la description de l’image ici][6]][6]

Et voici le [jsfiddle correspondant][7].

Sélectionner null

De loin, la meilleure façon de garantir que vous ne sélectionnez rien est de sélectionner “null”. Non seulement cela, mais cette alternative est bien plus rapide que toute autre.

Ainsi, pour une sélection d’entrée, il suffit de faire :

selection.selectAll(null)
    .data(data)
    .enter()
    .append(element);

Voici un violon de démonstration : https://jsfiddle.net/gerardofurtado/th6s160p/

Conclusion

Lorsque vous traitez avec des sélections “entrer”, faites très attention à ne pas sélectionner quelque chose qui existe déjà. Vous pouvez utiliser n’importe quoi dans votre selectAll, même des choses qui n’existent pas et n’existeront jamais (si vous ne prévoyez pas d’avoir une sélection “mise à jour” ou “sortie”).

Utiliser “this” avec une fonction fléchée #

La plupart des fonctions dans D3.js acceptent une fonction anonyme comme argument. Les exemples courants sont .attr, .style, .text, .on et .data, mais la liste est bien plus longue que cela.

Dans de tels cas, la fonction anonyme est évaluée pour chaque élément sélectionné, dans l’ordre, étant passé :

  1. La donnée courante (d)
  2. L’indice courant (i)
  3. Le groupe actuel (nœuds)
  4. this comme élément DOM actuel.

La donnée, l’index et le groupe courant sont passés en arguments, les fameux premier, deuxième et troisième argument dans D3.js (dont les paramètres sont traditionnellement nommés d, i et p dans D3 v3.x). Pour utiliser this, cependant, il n’est pas nécessaire d’utiliser d’argument :

.on("mouseover", function(){
    d3.select(this);
});

Le code ci-dessus sélectionnera “this” lorsque la souris survolera l’élément. Vérifiez qu’il fonctionne dans ce violon : https://jsfiddle.net/y5fwgopx/

La fonction flèche

En tant que nouvelle syntaxe ES6, une fonction de flèche a une syntaxe plus courte par rapport à l’expression de fonction. Cependant, pour un programmeur D3 qui utilise constamment “this”, il y a un écueil : une fonction fléchée ne crée pas son propre contexte “this”. Cela signifie que, dans une fonction fléchée, “this” a sa signification d’origine à partir du contexte englobant.

Cela peut être utile dans plusieurs circonstances, mais c’est un problème pour un codeur habitué à utiliser “this” en D3. Par exemple, en utilisant le même exemple dans le violon ci-dessus, cela ne fonctionnera pas :

.on("mouseover", ()=>{
    d3.select(this);
});

Si vous en doutez, voici le violon : https://jsfiddle.net/tfxLsv9u/

Eh bien, ce n’est pas un gros problème : on peut simplement utiliser une expression de fonction régulière à l’ancienne en cas de besoin. Mais que se passe-t-il si vous voulez écrire tout votre code en utilisant des fonctions fléchées ? Est-il possible d’avoir un code avec des fonctions fléchées et d’utiliser toujours correctement “this” dans D3 ?

Les deuxième et troisième arguments combinés

La réponse est oui, car “ceci” est identique à “nœuds[i]”. L’indice est en fait présent partout dans l’API D3, lorsqu’il décrit ceci :

…avec this comme élément DOM courant (nodes[i])

L’explication est simple : puisque nodes est le groupe actuel d’éléments dans le DOM et i est l’index de chaque élément, nodes[i] fait référence à l’élément DOM actuel lui-même. C’est-à-dire “ceci”.

Ainsi, on peut utiliser :

.on("mouseover", (d, i, nodes) => {
    d3.select(nodes[i]);
});

Et voici le violon correspondant : https://jsfiddle.net/2p2ux38s/

Différents sélecteurs #

Vous pouvez sélectionner des éléments avec différents sélecteurs :

  • par balise : "div"
  • par classe : ".class"
  • par identifiant : "#id"
  • par attribut : "[color=blue]"
  • sélecteurs multiples (OU) : "div1, div2, class1"
  • sélecteurs multiples (ET) : "div1 div2 class1"

Sélection de base et modifications #

Si vous êtes familier avec la syntaxe jQuery et Sizzle, les sélections d3 ne devraient pas être très différentes. d3 imite l’API W3C Selectors pour faciliter l’interaction avec les éléments.

Pour un exemple de base, pour sélectionner tous les <p> et ajouter un changement à chacun d’eux :

d3.selectAll('p')
  .attr('class','textClass') 
  .style('color', 'white');

En un mot, c’est relativement la même chose que de faire dans jQuery

$('p')
  .attr('class','textClass') 
  .css('color, 'white')

Généralement, vous commencerez par une seule sélection à votre conteneur div pour ajouter un élément SVG qui sera affecté à une variable (le plus souvent appelée svg).

var svg = d3.select('#divID').append('svg');

De là, nous pouvons appeler svg pour faire nos sous-sélections d’objets multiples (même s’ils n’existent pas encore).

svg.selectAll('path')
