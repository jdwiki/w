Syntaxe #

  • Commandes Hadoop v1 : hadoop fs -<commande>

  • Commandes Hadoop v2 : hdfs dfs -<commande>

Commandes Hadoop v1 #

1. Imprimez la version Hadoop #

hadoop version

2. Lister le contenu du répertoire racine dans HDFS #

#

hadoop fs -ls /

3. Indiquez la quantité d’espace utilisé et #

disponible sur le système de fichiers actuellement monté #

#

hadoop fs -df hdfs:/

4. Comptez le nombre de répertoires, de fichiers et d’octets sous #

les chemins qui correspondent au modèle de fichier spécifié #

#

hadoop fs -count hdfs:/

5. Exécutez un utilitaire de vérification du système de fichiers DFS #

#

hadoop fsck – /

6. Exécutez un utilitaire d’équilibrage de cluster #

#

hadoop balancer

7. Créez un nouveau répertoire nommé “hadoop” sous le #

Répertoire /user/training dans HDFS. Depuis que tu es #

actuellement connecté avec l’ID utilisateur “formation”, #

/user/training est votre répertoire personnel dans HDFS. #

#

hadoop fs -mkdir /user/training/hadoop

8. Ajoutez un exemple de fichier texte à partir du répertoire local #

nommé “data” dans le nouveau répertoire que vous avez créé dans HDFS #

lors de l’étape précédente. #

#

hadoop fs -put data/sample.txt /user/training/hadoop

9. Lister le contenu de ce nouveau répertoire dans HDFS. #

#

hadoop fs -ls /user/training/hadoop

10. Ajoutez l’intégralité du répertoire local appelé “retail” au #

Répertoire /user/training dans HDFS. #

#

hadoop fs -put data/retail /user/training/hadoop

11. Puisque /user/training est votre répertoire personnel dans HDFS, #

toute commande qui n’a pas de chemin absolu est #

interprété comme relatif à ce répertoire. Le suivant #

La commande # listera donc votre répertoire personnel, et

devrait afficher les éléments que vous venez d’y ajouter. #

#

hadoop fs -ls

12. Voyez combien d’espace ce répertoire occupe dans HDFS. #

#

hadoop fs -du -s -h hadoop/retail

13. Supprimez un fichier « clients » du répertoire « retail ». #

#

hadoop fs -rm hadoop/retail/customers

14. Assurez-vous que ce fichier n’est plus dans HDFS. #

#

hadoop fs -ls hadoop/retail/customers

15. Supprimez tous les fichiers du répertoire “retail” en utilisant un caractère générique. #

#

hadoop fs -rm hadoop/retail/*

#16. Pour vider la corbeille

#

hadoop fs -expunge

17. Enfin, supprimez tout le répertoire retail et tous #

de son contenu dans HDFS. #

#

hadoop fs -rm -r hadoop/retail

18. Lister à nouveau le répertoire hadoop #

#

hadoop fs -ls hadoop

19. Ajoutez le fichier achats.txt depuis le répertoire local #

nommé “/home/training/” dans le répertoire hadoop que vous avez créé dans HDFS #

#

hadoop fs -copyFromLocal /home/training/purchases.txt hadoop/

20. Pour visualiser le contenu de votre fichier texte achats.txt #

qui est présent dans votre répertoire hadoop. #

#

hadoop fs -cat hadoop/purchases.txt

21. Ajoutez le fichier Purchases.txt du répertoire “hadoop” qui est présent dans le répertoire HDFS #

au répertoire « data » qui est présent dans votre répertoire local #

#

hadoop fs -copyToLocal hadoop/purchases.txt /home/training/data

22. cp est utilisé pour copier des fichiers entre des répertoires présents dans HDFS #

#

hadoop fs -cp /user/training/*.txt /user/training/hadoop

23. La commande ‘-get’ peut être utilisée alternativement à la commande ‘-copyToLocal’ #

#

hadoop fs -get hadoop/sample.txt /home/training/

24. Affichez le dernier kilo-octet du fichier “purchases.txt” sur stdout. #

#

hadoop fs -tail hadoop/purchases.txt

25. Les autorisations de fichier par défaut sont 666 dans HDFS #

Utilisez la commande ‘-chmod’ pour modifier les autorisations d’un fichier #

#

hadoop fs -ls hadoop/purchases.txt
sudo -u hdfs hadoop fs -chmod 600 hadoop/purchases.txt

26. Les noms par défaut du propriétaire et du groupe sont formation, formation #

Utilisez ‘-chown’ pour changer simultanément le nom du propriétaire et le nom du groupe #

#

hadoop fs -ls hadoop/purchases.txt
sudo -u hdfs hadoop fs -chown root:root hadoop/purchases.txt

27. Le nom par défaut du groupe est la formation #

Utilisez la commande ‘-chgrp’ pour changer le nom du groupe #

#

hadoop fs -ls hadoop/purchases.txt
sudo -u hdfs hadoop fs -chgrp training hadoop/purchases.txt

28. Déplacer un répertoire d’un emplacement à un autre #

#

hadoop fs -mv hadoop apache_hadoop

29. Le facteur de réplication par défaut dans un fichier est 3. #

Utilisez la commande ‘-setrep’ pour modifier le facteur de réplication d’un fichier #

#

hadoop fs -setrep -w 2 apache_hadoop/sample.txt

30. Copier un répertoire d’un nœud du cluster à un autre #

Utilisez la commande ‘-distcp’ pour copier, #

-option d’écrasement pour écraser dans un fichier existant #

-update commande pour synchroniser les deux répertoires #

#

hadoop fs -distcp hdfs://namenodeA/apache_hadoop hdfs://namenodeB/hadoop

31. Commande pour faire en sorte que le nœud de nom quitte le mode sans échec #

#

hadoop fs -expunge
sudo -u hdfs hdfs dfsadmin -safemode leave

32. Lister toutes les commandes shell du système de fichiers hadoop #

#

hadoop fs

33. Obtenez les valeurs de quota hdfs et le nombre actuel de noms et d’octets utilisés. #

#

hadoop fs -count -q [-h] [-v] <directory>...<directory>

34. Enfin et surtout, demandez toujours de l’aide ! #

#

hadoop fs -help

Commandes Hadoop v2 #

appendToFile : Ajoute un seul src ou plusieurs src du système de fichiers local au système de fichiers de destination. Lit également l’entrée de stdin et l’ajoute au système de fichiers de destination. Gardez le comme -

 hdfs dfs -appendToFile [localfile1 localfile2 ..] [/HDFS/FILE/PATH..]

cat : Copie les chemins source vers stdout.

 hdfs dfs -cat URI [URI …]

chgrp : Modifie l’association de groupe de fichiers. Avec -R, effectue le changement de manière récursive via la structure de répertoires. L’utilisateur doit être le propriétaire du fichier ou le superutilisateur.

 hdfs dfs -chgrp [-R] GROUP URI [URI …]

chmod : Modifie les autorisations des fichiers. Avec -R, effectue le changement de manière récursive via la structure de répertoires. L’utilisateur doit être le propriétaire du fichier ou le superutilisateur

 hdfs dfs -chmod [-R] <MODE[,MODE]... | OCTALMODE> URI [URI …]

chown : Change le propriétaire des fichiers. Avec -R, effectue le changement de manière récursive via la structure de répertoires. L’utilisateur doit être le superutilisateur.

 hdfs dfs -chown [-R] [OWNER][:[GROUP]] URI [URI ]

copyFromLocal: Fonctionne de la même manière que la commande put, sauf que la source est limitée à une référence de fichier local.

 hdfs dfs -copyFromLocal <localsrc> URI

copyToLocal : Fonctionne de la même manière que la commande get, sauf que la destination est limitée à une référence de fichier local.

 hdfs dfs -copyToLocal [-ignorecrc] [-crc] URI <localdst>

count : compte le nombre de répertoires, de fichiers et d’octets sous les chemins qui correspondent au modèle de fichier spécifié.

 hdfs dfs -count [-q] [-h] <paths>

cp : Copie un ou plusieurs fichiers d’une source spécifiée vers une destination spécifiée. Si vous spécifiez plusieurs sources, la destination spécifiée doit être un répertoire.

 hdfs dfs -cp URI [URI …] <dest>

du: Affiche la taille du fichier spécifié ou la taille des fichiers et répertoires contenus dans le répertoire spécifié. Si vous spécifiez l’option -s, affiche un récapitulatif global des tailles de fichier plutôt que des tailles de fichier individuelles. Si vous spécifiez l’option -h, formate les tailles de fichier d’une manière “lisible par l’homme”.

 hdfs dfs -du [-s] [-h] URI [URI …]

dus : Affiche un résumé des tailles de fichiers ; équivalent à hdfs dfs -du –s.

 hdfs dfs -dus <args>

effacer : vide la corbeille. Lorsque vous supprimez un fichier, il n’est pas immédiatement supprimé de HDFS, mais est renommé en un fichier dans le répertoire /trash. Tant que le fichier y reste, vous pouvez le restaurer si vous changez d’avis, bien que seule la dernière copie du fichier supprimé puisse être restaurée.

 hdfs dfs –expunge

get : copie les fichiers dans le système de fichiers local. Les fichiers qui échouent à un contrôle de redondance cyclique (CRC) peuvent toujours être copiés si vous spécifiez l’option -ignorecrc. Le CRC est une technique courante de détection des erreurs de transmission de données. Les fichiers de somme de contrôle CRC ont l’extension .crc et sont utilisés pour vérifier l’intégrité des données d’un autre fichier. Ces fichiers sont copiés si vous spécifiez l’option -crc.

 hdfs dfs -get [-ignorecrc] [-crc] <src> <localdst>

getmerge : concatène les fichiers dans src et écrit le résultat dans le fichier de destination local spécifié. Pour ajouter un caractère de saut de ligne à la fin de chaque fichier, spécifiez l’option addnl.

 hdfs dfs -getmerge <src> <localdst> [addnl]

ls : Renvoie des statistiques pour les fichiers ou répertoires spécifiés.

 hdfs dfs -ls <args>

lsr : Sert de version récursive de ls ; similaire à la commande Unix ls -R.

 hdfs dfs -lsr <args>

mkdir : Crée des répertoires sur un ou plusieurs chemins spécifiés. Son comportement est similaire à la commande Unix mkdir -p, qui crée tous les répertoires menant au répertoire spécifié s’ils n’existent pas déjà.

 hdfs dfs -mkdir <paths>

moveFromLocal: Fonctionne de la même manière que la commande put, sauf que la source est supprimée après sa copie.

 hdfs dfs -moveFromLocal <localsrc> <dest>

mv : Déplace un ou plusieurs fichiers d’une source spécifiée vers une destination spécifiée. Si vous spécifiez plusieurs sources, la destination spécifiée doit être un répertoire. Le déplacement de fichiers entre les systèmes de fichiers n’est pas autorisé.

 hdfs dfs -mv URI [URI …] <dest>

put : Copie les fichiers du système de fichiers local vers le système de fichiers de destination. Cette commande peut également lire les entrées de stdin et écrire dans le système de fichiers de destination.

 hdfs dfs -put <localsrc> ... <dest>

rm : Supprime un ou plusieurs fichiers spécifiés. Cette commande ne supprime pas les répertoires ou fichiers vides. Pour contourner la corbeille (si elle est activée) et supprimer immédiatement les fichiers spécifiés, spécifiez l’option -skipTrash.

 hdfs dfs -rm [-skipTrash] URI [URI …]

rm r: Sert de version récursive de –rm.

 hdfs dfs -rm -r [-skipTrash] URI [URI …]

setrep : Modifie le facteur de réplication pour un fichier ou un répertoire spécifié. Avec -R, effectue le changement de manière récursive via la structure de répertoires.

 hdfs dfs -setrep <rep> [-R] <path>

stat: Affiche des informations sur le chemin spécifié.

 hdfs dfs -stat URI [URI …]

tail : Affiche le dernier kilo-octet d’un fichier spécifié sur la sortie standard. La syntaxe prend en charge l’option Unix -f, qui permet de surveiller le fichier spécifié. Au fur et à mesure que de nouvelles lignes sont ajoutées au fichier par un autre processus, tail met à jour l’affichage.

 hdfs dfs -tail [-f] URI

test : Renvoie les attributs du fichier ou du répertoire spécifié. Spécifie -e pour déterminer si le fichier ou le répertoire existe ; -z pour déterminer si le fichier ou le répertoire est vide ; et -d pour déterminer si l’URI est un répertoire.

 hdfs dfs -test -[ezd] URI

text : Affiche un fichier source spécifié au format texte. Les formats de fichier d’entrée valides sont zip et TextRecordInputStream.

 hdfs dfs -text <src>

touchz : Crée un nouveau fichier vide de taille 0 dans le chemin spécifié.

 hdfs dfs -touchz <path>
