Conditions de test #

Pour utiliser un saut conditionnel, une condition doit être testée. ** Tester une condition ** ici se réfère uniquement à l’acte de vérifier les drapeaux, le saut réel est décrit sous [Sauts conditionnels] (https://www.wikiod.com/fr/x86/flux-de-controle#Sauts conditionnels -sauts).

x86 teste les conditions en s’appuyant sur le registre EFLAGS, qui contient un ensemble d’indicateurs que chaque instruction peut potentiellement définir.

Les instructions arithmétiques, comme sub ou add, et les instructions logiques, comme xor ou and, “définissent les drapeaux”. Cela signifie que les drapeaux CF, OF, SF, ZF, AF, PF sont modifiés par ces instructions. Cependant, toute instruction est autorisée à modifier les drapeaux, par exemple cmpxchg modifie le ZF.

Vérifiez toujours la référence de l’instruction pour savoir quels drapeaux sont modifiés par une instruction spécifique.

x86 a un ensemble de * sauts conditionnels *, mentionnés précédemment, qui sautent si et seulement si certains drapeaux sont définis ou certains sont clairs ou les deux.

Drapeaux #

Les opérations arithmétiques et logiques sont très utiles pour définir les drapeaux. Par exemple après un sub eax, ebx, pour l’instant contenant des valeurs non signées, nous avons :

Drapeau Lorsqu’il est défini Lorsqu’il est clair
ZF Lorsque le résultat est zéro.
EAX - EBX = 0 ⇒ EAX = EBX		 Lorsque le résultat est non nul.
EAX - EBX ≠ 0 ⇒ EAX ≠ EBX
CF Lorsque le résultat devait être reporté pour le MSb.
EAX - EBX < 0 ⇒ EAX < EBX		 Lorsque le résultat n’a pas besoin d’être reporté pour le MSb.
EAX - EBX ≮ 0 ⇒ EAX ≮ EBX
SF Lorsque le résultat MSb est défini. Lorsque le résultat MSb n’est pas défini.
DE Lorsqu’un débordement signé s’est produit. Lorsqu’un débordement signé ne s’est pas produit.
PF Lorsque le nombre de bits définis dans l’octet le moins significatif du résultat est pair. Lorsque le nombre de bits définis dans l’octet le moins significatif du résultat est impair.
AF Lorsque le chiffre BCD inférieur a généré un report.
C’est le report du bit 4.		 Lorsque le chiffre BCD inférieur n’a pas généré de retenue.
C’est la retenue du bit 4.

Essais non destructifs #

Les instructions sub et and modifient leur opérande de destination et nécessiteraient deux copies supplémentaires (sauvegarde et restauration) pour conserver la destination non modifiée.

Pour effectuer un test non destructif, il existe les instructions cmp et test. Ils sont identiques à leur homologue destructif sauf que le résultat de l’opération est ignoré et que seuls les drapeaux sont enregistrés.

Destructeur Non destructif
sous cmp
et tester 
test eax, eax             ;and eax, eax
                          ;ZF = 1 iff EAX is zero

test eax, 03h             ;and eax, 03h
                          ;ZF = 1 if both bit[1:0] are clear
                          ;ZF = 0 if at least one of bit[1:0] is set

cmp eax, 241d             ;sub eax, 241d
                          ;ZF = 1 iff EAX is 241
                          ;CF = 1 iff EAX < 241

Tests signés et non signés #

Le processeur ne donne aucune signification particulière aux valeurs d’enregistrement1, le signe est une construction de programmeur. ** Il n’y a pas de différence lors du test des valeurs signées et non signées. ** Le processeur calcule suffisamment d’indicateurs pour tester les relations arithmétiques habituelles (égal, inférieur à, supérieur à, etc.) à la fois si les opérandes devaient être considérés comme signés et non signés.

1 Bien qu’il comporte certaines instructions qui n’ont de sens qu’avec des formats spécifiques, comme le complément à deux. Il s’agit de rendre le code plus efficace car l’implémentation de l’algorithme dans un logiciel nécessiterait beaucoup de code.

Sauts conditionnels #

En fonction de l’état des drapeaux, la CPU peut exécuter ou ignorer un saut. Une instruction qui effectue un saut basé sur les drapeaux porte le nom générique de Jcc - Jump on Condition Code1.

Synonymes et terminologie #

Afin d’améliorer la lisibilité du code d’assemblage, Intel a défini plusieurs synonymes pour le même code de condition. Par exemple, jae, jnb et jnc sont tous le même code de condition CF = 0.

Bien que le nom de l’instruction puisse donner une indication très forte sur le moment de l’utiliser ou non, la seule approche significative consiste à reconnaître les drapeaux qui doivent être testés et puis à choisir les instructions de manière appropriée. Intel a cependant donné aux instructions des noms parfaitement logiques lorsqu’elles sont utilisées après une instruction “cmp”. Pour les besoins de cette discussion, “cmp” sera supposé avoir défini les drapeaux avant un saut conditionnel.

Égalité #

Les opérandes sont égaux ssi ZF a été défini, ils diffèrent sinon. Pour tester l’égalité, nous avons besoin de ZF = 1.

je a_label           ;Jump if operands are equal
jz a_label           ;Jump if zero (Synonym)

jne a_label          ;Jump if operands are NOT equal
jnz a_label          ;Jump if not zero (Synonym)
Instruction Drapeaux
est, jz ZF = 1
jne, jnz ZF = 0

Plus grand que #

Pour les opérandes non signés, la destination est supérieure à la source si la retenue n’était pas nécessaire, c’est-à-dire si CF = 0. Lorsque CF = 0, il est possible que les opérandes soient égaux, tester ZF lèvera l’ambiguïté.

jae a_label      ;Jump if above or equal (>=)
jnc a_label      ;Jump if not carry (Synonym)
jnb a_label      ;Jump if not below (Synonym)

ja a_label       ;Jump if above  (>)
jnbe a_label     ;Jump if not below and not equal (Synonym)
Instruction Drapeaux
, jae, jnc, jnb FC = 0
ja, jnbe CF = 0, ZF = 0

Pour les opérandes signés, nous devons vérifier que SF = 0, sauf s’il y a eu un débordement signé, auquel cas le SF résultant est inversé. Puisque OF = 0 si aucun débordement signé ne s’est produit et 1 sinon, nous devons vérifier que SF = OF.

ZF peut être utilisé pour implémenter un test strict/non strict.

jge a_label      ;Jump if greater or equal (>=)
jnl a_label      ;Jump if not less (Synonym)

jg a_label       ;Jump if greater (>)
jnle a_label     ;Jump if not less and not equal (Synonym)
Instruction Drapeaux
jge, jnl SF = DE
jg, jnle SF = OF, ZF = 0

Moins que #

Ceux-ci utilisent les conditions inversées ci-dessus.

jbe a_label      ;Jump if below or equal (<=)
jna a_label      ;Jump if not above (Synonym)

jb a_label       ;Jump if below (<)
jc a_label       ;Jump if carry (Synonym)
jnae a_label     ;Jump if not above and not equal (Synonym)

;SIGNED 

jle a_label      ;Jump if less or equal (<=)
jng a_label      ;Jump if not greater (Synonym)

jl a_label       ;Jump if less (<)
jnge a_label     ;Jump if not greater and not equal (Synonym)
Instruction Drapeaux
, jbe, jna CF = 1 ou ZF = 1
jb, jc, jnae CF = 1
jle, jng SF != OF ou ZF = 1
jl, jnge SF != DE

Drapeaux spécifiques #

Chaque drapeau peut être testé individuellement avec j<flag_name> où flag_name ne contient pas le F final (par exemple CF → C, PF → P).

Les codes restants non couverts auparavant sont :

Instruction Drapeau
js FS = 1
jns FS = 0
``oui'' DE = 1
jno DE = 0
jp, jpe (e = pair) FP = 1
jnp, jpo (o = impair) PF = 0

Un autre saut conditionnel (supplémentaire) #

Un saut conditionnel x86 spécial ne teste pas l’indicateur. Au lieu de cela, il teste la valeur du registre cx ou ecx (basé sur le mode d’adresse CPU actuel étant de 16 ou 32 bits), et le saut est exécuté lorsque le registre contient zéro.

Cette instruction a été conçue pour la validation du registre counter (cx/ecx) avant les instructions de type rep, ou avant les boucles loop.

jcxz  a_label   ; jump if cx (16b mode) or ecx (32b mode) is zero
jecxz a_label   ; synonym of jcxz (recommended in source code for 32b target)
Instruction S’inscrire (pas signaler)
jcxz, jecxz cx = 0 (mode 16b)
jcxz, jecxz ecx = 0 (mode 32b)

1 Ou quelque chose comme ça.

a_label #

Dans les exemples ci-dessus, a_label est la destination cible du processeur lorsque la condition testée est “vraie”. Lorsque la condition testée est “fausse”, le CPU continuera sur la prochaine instruction suivant le saut conditionnel.

Synonymes #

Il existe des synonymes d’instructions qui peuvent être utilisés pour améliorer la lisibilité du code. Par exemple ja et jnbe (Saut non inférieur ni égal) sont la même instruction.

Codes compagnons non signés signés #

Opération Non signé Signé
> ‘oui’ jg
>= jae jge
< jb jl
, <= jbe jle
= ’est' ’est'
≠, !=, <> ``jne'' ``jne''

Sauts inconditionnels #

jmp a_label                      ;Jump to a_label
jmp bx                           ;Jump to address in BX
jmp WORD [aPointer]              ;Jump to address in aPointer
jmp 7c0h:0000h                   ;Jump to segment 7c0h and offset 0000h
jmp FAR WORD [aFarPointer]       ;Jump to segment:offset in aFarPointer

Relatif près des sauts #

jmp a_label est :

  • à proximité Il ne spécifie que la partie offset de l’adresse logique de destination. Le segment est supposé être “CS”.
  • relatif La sémantique de l’instruction est saute rel octets vers l’avant1 à partir de l’adresse de l’instruction suivante ou IP = IP + rel.

L’instruction est codée sous la forme EB <rel8> ou EB <rel16/32>, l’assembleur choisissant la forme la plus appropriée, préférant généralement une forme plus courte. Le remplacement par assembleur est possible, par exemple avec NASM jmp SHORT a_label, jmp WORD a_label et jmp DWORD a_label génèrent les trois formes possibles.

## Sauts proches indirects absolus ##

jmp bx et jmp WORD [aPointer] sont :

  • à proximité Ils ne précisent que la partie offset de l’adresse logique de destination. Le segment est supposé être “CS”.
  • absolue indirecte La sémantique des instructions est sauter à l’adresse dans reg ou mem ou IP = reg, IP = mem.

L’instruction est codée comme FF/4, pour la mémoire indirecte la taille de l’opérande est déterminée comme pour tout autre accès mémoire.

Sauts lointains absolus #

jmp 7c0h:0000h est :

  • loin Il spécifie les deux parties de l’adresse logique : le segment et le décalage.

  • absolu La sémantique de l’instruction est saut à l’adresse segment:offset ou CS = segment, IP = offset.

L’instruction est codée sous la forme EA <imm32/48> en fonction de la taille du code. Il est possible de choisir entre les deux formes dans certains assembleurs, par exemple avec NASM jmp 7c0h : WORD 0000h et jmp 7c0h : DWORD 0000h génèrent la première et la seconde forme.

Sauts lointains indirects absolus #

jmp FAR WORD [aFarPointer] est :

  • loin Il spécifie les deux parties de l’adresse logique : le segment et le décalage.

  • Indirect absolu La sémantique de l’instruction est sauter au segment:offset stocké dans mem2 ou CS = mem[23:16/32], IP = [15/31:0] .

L’instruction est codée en tant que FF /5, la taille de l’opérande peut être contrôlée avec les spécificateurs de taille. Dans NASM, un peu non intuitif, ce sont jmp FAR WORD [aFarPointer] pour un opérande 16:16 et jmp FAR DWORD [aFarPointer] pour un opérande 16:32.

Sauts manquants #

  • presque absolu Peut être émulé avec un saut presque indirect.

      mov bx, target            ;BX = absolute address of target
  jmp bx

  • ** parent éloigné ** Cela n’a aucun sens ou est trop étroit d’utilisation de toute façon.

1 Le complément à deux est utilisé pour spécifier un décalage signé et ainsi sauter en arrière. 2 Qui peut être un seg16:off16 ou un seg16:off32, de tailles 16:16 et 16:32.
