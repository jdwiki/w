La famille x86 existe depuis longtemps, et en tant que telle, de nombreuses astuces et techniques ont été découvertes et développées qui sont de notoriété publique - ou peut-être pas si publiques.

La plupart de ces astuces tirent parti du fait que de nombreuses instructions font effectivement la même chose - mais différentes versions sont plus rapides, ou économisent de la mémoire, ou n’affectent pas les drapeaux.

Voici un certain nombre de trucs qui ont été découverts. Chacun a ses avantages et ses inconvénients, il doit donc être répertorié.

En cas de doute, vous pouvez toujours vous référer au assez complet [Intel 64 and IA-32 Architectures Optimization Reference Manual][1], qui est une excellente ressource de la société derrière l’architecture x86 elle-même.

[1] : http://www.intel.com/content/dam/www/public/us/en/documents/manuals/64-ia-32-architectures-optimization-manual.pdf

Remise à zéro d’un registre #

Le moyen le plus évident de mettre à zéro un registre est de mettre “MOV” dans un “0”, par exemple :

B8 00 00 00 00    MOV eax, 0

Notez qu’il s’agit d’une instruction de 5 octets.

Si vous êtes prêt à écraser les drapeaux (MOV n’affecte jamais les drapeaux), vous pouvez utiliser l’instruction XOR pour bitwise-XOR le registre avec lui-même :

33 C0             XOR eax, eax

Cette instruction ne nécessite que 2 octets et [s’exécute plus rapidement sur tous les processeurs] (http://stackoverflow.com/questions/33666617/what-is-the-best-way-to-set-a-register-to-zero-in -x86-assembly-xor-mov-or-and/33668295#33668295).

Tester un registre pour 0 #

Arrière plan #

Pour savoir si un registre contient un zéro, la technique naïve consiste à faire ceci :

    cmp   eax, 0

Mais si vous regardez l’opcode pour cela, vous obtenez ceci :

83 F8 00      cmp   eax, 0

Utilisez test #

    test   eax, eax      ; Equal to zero?

Examinez l’opcode que vous obtenez :

85 c0         test   eax, eax

Avantages #

  • Seulement deux octets !

Les inconvénients #

  • Opaque pour un lecteur non familiarisé avec la technique

Vous pouvez également consulter la [Q&A Question on this technique][1].

[1] : http://stackoverflow.com/questions/33721204/test-whether-a-register-is-zero-with-cmp-reg-0-vs-or-reg-reg

Déplacement du drapeau Carry dans un registre #

Arrière plan #

Si le drapeau Carry (C) contient une valeur que vous souhaitez mettre dans un registre, la méthode naïve consiste à faire quelque chose comme ceci :

    mov  al, 1
    jc   NotZero
    mov  al, 0
NotZero:

Utilisez ‘sbb’ #

Un moyen plus direct, évitant le saut, est d’utiliser “Soustraire avec Emprunter”:

    sbb  al,al    ; Move Carry to al

Si ‘C’ est égal à zéro, alors ‘al’ sera égal à zéro. Sinon, ce sera 0xFF (-1). Si vous avez besoin qu’il soit 0x01, ajoutez :

    and  al, 0x01 ; Mask down to 1 or 0

Avantages #

  • A peu près la même taille
  • Deux ou une instruction de moins
  • Pas de saut coûteux

Les inconvénients #

  • C’est opaque pour un lecteur peu familier avec la technique
  • Il modifie d’autres drapeaux

Appels système Linux avec moins de gonflement #

Sous Linux 32 bits, les appels système sont généralement effectués à l’aide de l’instruction sysenter (je dis généralement parce que les programmes plus anciens utilisent le désormais obsolète int 0x80) cependant, cela peut prendre beaucoup de place dans un programme et il existe donc des moyens qu’on peut couper les coins ronds pour raccourcir et accélérer les choses. Il s’agit généralement de la disposition d’un appel système sur Linux 32 bits :

mov eax, <System call number>
mov ebx, <Argument 1> ;If applicable
mov ecx, <Argument 2> ;If applicable
mov edx, <Argument 3> ;If applicable
push <label to jump to after the syscall>
push ecx
push edx
push ebp
mov ebp, esp
sysenter

C’est énorme, n’est-ce pas ! Mais il y a quelques astuces que nous pouvons tirer pour éviter ce gâchis. La première consiste à fixer ebp à la valeur de esp diminuée de la taille de 3 registres 32 bits, soit 12 octets. C’est très bien tant que vous êtes d’accord pour écraser ebp, edx et ecx avec des ordures (comme lorsque vous déplacerez une valeur dans ces registres directement après de toute façon), nous pouvons le faire en utilisant l’instruction LEA afin que nous n’ayons pas besoin affecter la valeur de l’ESP lui-même.

mov eax, <System call number>
mov ebx, <Argument 1>
mov ecx, <Argument 2>
mov edx, <Argument 3>
push <label to jump to after the syscall>
lea ebp, [esp-12]
sysenter

Cependant, nous n’avons pas fini, si l’appel système est sys_exit, nous pouvons nous en tirer sans rien pousser du tout dans la pile !

mov eax, 1
xor ebx, ebx ;Set the exit status to 0
mov ebp, esp
sysenter

Multiplier par 3 ou 5 #

Arrière plan #

Pour obtenir le produit d’un registre et d’une constante et le stocker dans un autre registre, la méthode naïve consiste à procéder comme suit :

    imul ecx, 3      ; Set ecx to 5 times its previous value
    imul edx, eax, 5 ; Store 5 times the contend of eax in edx

Utilisez ceci #

Les multiplications sont des opérations coûteuses. Il est plus rapide d’utiliser une combinaison de décalages et d’ajouts. Pour le cas particulier de la multiplication du contenu d’un registre 32 ou 64 bits qui n’est pas esp ou rsp par 3 ou 5, vous pouvez utiliser l’instruction lea. Celui-ci utilise le circuit de calcul d’adresse pour calculer rapidement le produit.

    lea ecx, [2*ecx+ecx] ; Load 2*ecx+ecx = 3*ecx into ecx
    lea edx, [4*edx+edx] ; Load 4*edx+edx = 5*edx into edx

De nombreux assembleurs comprendront également

    lea ecx, [3*ecx]
    lea edx, [5*edx]

Pour tous les multiplicandes possibles autres que ebp ou rbp, la longueur d’instruction résultante est la même qu’avec l’utilisation de imul.

Avantages #

  • S’exécute beaucoup plus rapidement

Les inconvénients #

  • Si votre multiplicande est ebp ou rbp, il leur faut un octet de plus en utilisant imul
  • Plus à taper si votre assembleur ne supporte pas les raccourcis
  • Opaque pour un lecteur non familiarisé avec la technique
