Scripts Web

Sur cette page

  • Bonjour le monde Web Script

Les scripts Web sont des modules fonctionnels dans Alfresco, qui peuvent simplement afficher certaines informations ou également faire certaines choses à l’intérieur d’Alfresco (par exemple, exécuter un flux de travail, travailler avec des fichiers, des utilisateurs, des groupes ou d’autres entités).

Chaque webscript a deux parties principales - le code (.js, .java) et le modèle Freemaker (.ftl)

Webscript peut également avoir un fichier .properties supplémentaire avec des chaînes de texte utilisées. Les pièces sont appariées dans le fichier context.xml (logique du framework Spring).

Il est lié à n’importe quelle URL décrite dans ce fichier.

Bonjour le monde Web Script #

Faisons un bonjour le script web mondial. Les scripts Web ont un descripteur, un contrôleur et, éventuellement, une vue. Ces fichiers doivent suivre une convention de dénomination.

Ce descripteur est nommé helloworld.get.desc.xml.

<webscript>
  <shortname>Hello World</shortname>
  <description>Hello world web script</description>
  <url>/example/helloworld?name={nameArgument}</url>
  <authentication>user</authentication>  
</webscript>

Vous pouvez voir que le descripteur déclare que ce script Web sera mappé sur une URL, “/example/helloworld”, et qu’il nécessite une authentification de l’utilisateur. Le descripteur déclare également un argument appelé nom.

Voici le contrôleur. Il s’appelle helloworld.get.js.

model.foo = "bar";

Ce contrôleur est écrit en JavaScript mais les contrôleurs peuvent aussi être écrits en Java. Avec un peu plus de travail, vous pouvez également écrire des contrôleurs dans d’autres langages.

Ce contrôleur ne fait pas grand chose. Il ajoute simplement une nouvelle variable au modèle appelée “foo” et lui donne une valeur de “bar”.

Votre contrôleur a accès à une variété de variables de portée racine qui sont toutes documentées dans la [documentation officielle][1].

Enfin, regardons la vue. Il s’appelle helloworld.get.html.ftl

<html>
  <body>
    <p>Hello, ${args.name!"name not specified"}!</p>
    <p>Foo: ${foo}</p>
  </body>
</html>

Vous pouvez voir d’après le nom que cette vue est implémentée en tant que modèle Freemarker et génère du HTML. Cette vue récupère la valeur de “foo” du modèle et elle récupère également l’argument de nom qui a été transmis au script Web. Si un argument de nom n’est pas spécifié, le modèle fournit un texte par défaut.

Si vous vouliez produire XML ou JSON à la place, vous pouvez simplement changer le nom, puis mettre à jour votre implémentation de modèle en conséquence.

Déploiement

Les scripts Web peuvent être déployés dans le chemin de classe ou téléchargés dans le référentiel. Par exemple, pour déployer ce script Web en le téléchargeant dans le référentiel, procédez comme suit :

  1. Téléchargez ces trois fichiers dans Data Dictionary/Web Scripts Extensions
  2. Actualisez les scripts Web en accédant à http://localhost:8080/alfresco/s/index et en cliquant sur “Actualiser les scripts Web”.
  3. Accédez au script Web en allant sur http://localhost:8080/alfresco/s/example/helloworld?name=Jeff

[1] : http://docs.alfresco.com/community/references/API-JS-rootscoped.html

Folder Maker : un script Web qui gère le POST #

Le script Web Hello World gère les méthodes HTTP GET. Mais que se passe-t-il si vous souhaitez créer des données sur le serveur ? Pour cela, votre script Web doit gérer POST.

Voici un exemple simple qui crée de nouveaux dossiers dans Company Home. Il est invoqué en effectuant un appel POST avec un corps JSON qui ressemble à :

{'name':'testfolder'}

Vous pouvez éventuellement ajouter un titre ou une description au dossier en les transmettant dans le corps, comme :

{
  'name':'testfolder',
  'title':'test title',
  'description':'test description'
}

Le descripteur s’appelle foldermaker.post.desc.xml :

<webscript>
  <shortname>Folder Maker</shortname>
  <description>Creates folders</description>
  <family>Examples</family>
  <url>/example/folders</url>
  <format default="json"></format>
  <authentication>user</authentication>
</webscript>

L’élément facultatif “famille” est un moyen pratique de regrouper les scripts Web dans l’index des scripts Web. L’élément “format” déclare que ce script web renvoie JSON.

Le contrôleur s’appelle foldermaker.post.json.js :

var name = title = desc = null;

var name = json.get('name');

try {
  title = json.get('title');
} catch (err) {}

try {
  desc = json.get('description');
} catch (err) {}

var folder = companyhome.createFolder(name);

var needsSave = false;

if (title != null) {
  folder.properties['cm:title'] = title;
  needsSave = true;
}

if (desc != null) {
  folder.properties['cm:description'] = desc;
  needsSave = true;
}

if (needsSave) {
  folder.save();
}

model.id = folder.nodeRef.toString();
model.name = name;
model.title = title;
model.description = desc;

Notez que ce contrôleur a “json” dans son nom. Cela indique à Alfresco d’attendre une charge utile JSON. Alfresco analysera automatiquement le JSON et le placera dans une variable racine appelée “json”.

Le contrôleur récupère le nom, le titre et la description du JSON et crée le dossier à l’aide de la variable de portée racine appelée “companyhome”.

Si un titre ou une description est passé dans ces propriétés, elles sont enregistrées.

Les valeurs transmises ainsi que la référence de nœud du nouveau dossier sont définies sur le modèle avant de passer le contrôle à la vue.

La vue est nommée foldermaker.post.json.ftl :

{
  <#if title??>
  "title": "${title}",
  </#if>
  <#if description??>
  "description": "${description}",
  </#if>
  "id": "${id}",
  "name": "${name}"
}

Ce marqueur libre renvoie simplement les valeurs définies sur le modèle en tant que JSON. Le titre et la description peuvent ne pas toujours être présents, donc la vue utilise la vérification null Freemarker intégrée dans une instruction if pour éviter de les renvoyer s’ils n’étaient pas définis.

N’importe quel client HTTP peut être utilisé pour tester ce script Web. Voici à quoi cela ressemblerait en utilisant curl :

jpotts$ curl -uadmin:admin -H "content-type: application/json" -X POST "http://localhost:8080/alfresco/s/example/folders" -d "{'name':'testfolder','title':'test title', 'description':'test desc'}"
{
  "title": "test title",
  "description": "test desc"
  "id": "workspace://SpacesStore/cc26a12f-306b-41f1-a859-668f11fc2a54",
  "name": "testfolder"
}

Notez que curl transmet les informations d’identification d’authentification de base. Dans ce cas, il utilise “admin”. Étant donné que cet exemple crée des éléments dans Company Home, vous devez utiliser un utilisateur disposant des autorisations appropriées pour le faire.

Ce script Web n’a pas de véritable vérification d’erreur. Si vous ne transmettez pas un nom ou si vous transmettez un nom qui a déjà été utilisé, vous verrez une erreur.
