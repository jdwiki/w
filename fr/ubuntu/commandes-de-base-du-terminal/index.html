Comment extraire les fichiers tar.gz/bz2/tbz :

Si votre extension de fichier est .tar.gz (ou .tgz) utilisez cette commande

 tar xvzf file.tar.gz

  • x : cela indique à tar d’extraire les fichiers.

  • v : Cette option listera tous les fichiers un par un dans l’archive. Le “v” signifie “verbeux”.

  • z : L’option z est très importante et indique à la commande tar de décompressez le fichier (gzip).

  • f : cette option indique à tar que vous allez lui donner un nom de fichier travailler avec.

** Si votre extension de fichier est .tar.bz2 (ou .tbz)**

La principale différence entre ces deux options est que l’option z a été remplacée par l’option j.

  • j : Ceci décompressera un fichier bzip2.

!! #

(lu comme bangbang) est un raccourci pour répéter la dernière commande entrée dans la console. Il est particulièrement utile d’exécuter la commande précédente avec quelques modifications

adduser tom

adduser : seul root peut ajouter un utilisateur ou un groupe au système.

Oh snap, et maintenant? Eh bien, vous pouvez retaper la commande avec sudo devant ou vous pouvez essayer

sudo !!

équivalent à sudo “commande précédente entrée”

Ceci est particulièrement utile si la commande que vous venez de taper est particulièrement longue.

Il peut également être utilisé pour modifier une partie des commandes précédentes

cd path/to/wrong/directory

Avec

!!:s/wrong/right/

Ça ira

cd path/to/right/directory

Modification du mot de passe de l’utilisateur actuel #

Pour changer le mot de passe de l’utilisateur actuel, tapez simplement :

sudo passwd

Il vous demandera d’entrer votre mot de passe actuel :

[sudo] password for <user>:

Et ensuite, il vous sera demandé d’entrer un nouveau mot de passe :

Enter new UNIX password:

Et enfin il vous sera demandé de ressaisir votre mot de passe :

Retype new UNIX password:

Remarque : [Par défaut][1], les touches sur lesquelles vous appuyez à une invite de mot de passe de ligne de commande ne s’affichent pas du tout. Ils sont cependant toujours enregistrés.

[1] : https://coderwall.com/p/3vywiw/make-password-asterisks-visible-in-your-terminal

Liste les fichiers et dossiers #

Pour lister les fichiers et dossiers dans le répertoire courant, nous utilisons la commande ls :

user@host:/$ ls
bin  boot  cdrom  dev  etc  home  initrd.img  lib  lib64  lost+found  
media  mnt  opt  proc  root  run  sbin  srv  sys  tmp  usr  var  vmlinuz

ls imprime la structure des dossiers en vue simple, avec un code couleur par type. Les couleurs par défaut d’Ubuntu pour ls sont :

bleu pour les répertoires, vert pour les fichiers exécutables, bleu ciel pour les fichiers liés, jaune sur fond noir pour les appareils, rose pour les fichiers image et * rouge* pour les fichiers d’archive.

Un autre raccourci pour ls -la est ll. Cependant, ce n’est pas une commande intégrée. C’est plutôt un “alias” courant dans les systèmes Ubuntu, en entier son “ls -laF”. L’alias vous donnera la même sortie que ls -la, mais avec une barre oblique supplémentaire (/) à la fin de chaque dossier, pour vous aider à identifier plus facilement le dossier.

L’alias ll peut être visualisé en entier en tapant alias ll. Comme illustré ci-dessous. Si l’alias n’est pas défini, la commande génère une erreur.

vagrant@host ~ ->>
08:05 AM Mon Sep 12$ alias ll
 alias ll='ls -alF'

Ajout d’un nouvel utilisateur #

La commande adduser ajoute un utilisateur au système. Pour ajouter un nouveau type d’utilisateur :

sudo adduser <user_name>

Exemple:

sudo adduser tom

Après avoir tapé la commande ci-dessus, vous serez invité à entrer des détails sur le nouvel utilisateur, tels que le nouveau mot de passe, le nom complet de l’utilisateur, etc.

Vous trouverez ci-dessous les informations que l’utilisateur sera invité à remplir afin d’ajouter un nouvel utilisateur :

Enter new UNIX password:
Retype new UNIX password:
passwd: password updated successfully
Changing the user information for tom
Enter the new value, or press ENTER for the default
    Full Name []: Test User
    Room Number []:
    Work Phone []:
    Home Phone []:
    Other []:
Is the information correct? [Y/n] y

Redémarrez Ubuntu #

Vous pouvez redémarrer Ubuntu à partir de la ligne de commande. Vous trouverez ci-dessous un exemple de redémarrage immédiat d’ubuntu.

 sudo reboot

Vous devez avoir le privilège sudo pour utiliser cette commande.

Les autres commandes avec les mêmes résultats sont sudo shutdown -r now et sudo init 6.

Installer un nouveau logiciel #

APT et APT-GET #

Le moyen le plus simple et le plus rapide consiste à utiliser la commande apt-get. Cette commande peut être considérée comme de niveau inférieur et “back-end”, et prend en charge d’autres outils basés sur APT. Il n’y a pas de chargeurs sophistiqués, seulement des informations de base sur la progression. C’est le moyen le plus rapide d’installer des applications.

Usage:

sudo apt-get install déluge openssh-server

Cette commande installera deux nouvelles applications : deluge et openssh-server. Vous pouvez installer autant d’applications que vous le souhaitez dans une seule ligne de commandes.

La manière “fantaisiste” du même processus est wits apt :

sudo apt install déluge openssh-server

Le résultat est le même, mais l’interaction avec les utilisateurs est différente de la commande précédente. apt est conçu pour les utilisateurs finaux (humains) et sa sortie peut être modifiée entre les versions.

Les deux commandes géreront automatiquement les dépendances.

DPKG #

dpkg signifie paquet Debian. Il s’agit d’un programme d’installation de base de bas niveau pour Debian et d’autres dérivés de Debian. dpkg a de nombreuses options, mais nous sommes intéressés par -i, qui signifie install.

sudo dpkg -i google-chrome-stable.deb

Vous pouvez installer les packages .deb avec dpkg.

Si vous obtenez une erreur lors de l’installation du paquet .deb, dans la plupart des cas, vous n’avez pas de dépendances. Pour vous débarrasser de cette erreur et installer correctement l’application, exécutez sudo apt-get -f install sans aucun autre paramètre. Cela recherchera les dépendances et les installera avant .deb. L’installation se poursuivra et l’application sera installée.

Lecture d’un fichier texte #

En utilisant Ubuntu, vous avez différentes façons de lire un fichier texte, toutes similaires mais utiles dans un contexte différent.

chat

C’est la manière la plus simple de lire un fichier texte ; il affiche simplement le contenu du fichier dans le terminal. Attention : si le fichier est volumineux, le processus d’impression peut prendre un certain temps ! Si vous avez besoin de l’arrêter, vous pouvez toujours appuyer sur CTRL + C. Notez que si vous devez naviguer dans le document, vous devez faire défiler la sortie du terminal.

cat file_name.txt

Suite

Une version améliorée de cat. Si votre fichier est plus long que l’affichage du terminal, vous pouvez simplement taper

more file_name.txt

et vous aurez un affichage de texte défilant vers le bas, dans lequel vous pourrez vous déplacer vers le bas en appuyant sur ENTER.

moins

Il s’agit de la commande more avec quelques améliorations, et c’est généralement un meilleur choix que cat pour lire des documents moyens à volumineux. Il ouvre le fichier les montrant depuis le début, permettant de faire défiler vers le haut/bas/droite/gauche à l’aide des flèches.

less file_name.txt

Une fois le document ouvert, vous pouvez saisir des commandes pour activer certaines fonctionnalités utiles, telles que :

  • q : ferme immédiatement le fichier ouvert.
  • /word : recherche ‘mot’ dans le document. En appuyant sur ’n’, vous pouvez accéder à l’occurrence suivante de ‘mot’.
  • ENTER : défile vers le bas d’une seule ligne.
  • r : redessine le contenu du fichier, s’il change pendant la lecture.

C’est le meilleur choix pour la lecture de documents moyens à volumineux.

queue

Ce logiciel affiche uniquement la dernière partie du fichier. C’est utile si vous n’avez besoin de lire que quelques lignes à la fin d’un très gros document.

tail file_name.txt

La commande ci-dessus affichera les 10 dernières lignes (par défaut) du fichier. Pour lire les 3 dernières lignes, il faut écrire :

tail -3 file_name.txt

Il existe un autre cas d’utilisation où cette commande est extrêmement utile. Imaginez avoir un document vide, qui se remplit pendant que vous le regardez ; si vous voulez voir les nouvelles lignes en temps réel pendant qu’elles sont écrites dans le fichier sans le rouvrir, ouvrez simplement le fichier avec l’option -f. C’est vraiment utile si vous regardez des journaux, par exemple.

tail -f file_name.txt

C’est le meilleur choix pour lire des documents en pleine croissance.

tête

Cette commande fait le contraire de tail. Par exemple, la commande suivante affichera les 15 premières lignes du fichier file_name.txt.

head -15 file_name.txt

queue

Il s’agit d’une alternative à tail -f filename. Il suit les modifications de fichier au fur et à mesure qu’elles se produisent et vous montre la sortie.

vigueur

Certains d’entre nous aiment vi, d’autres aiment vim. Ce n’est pas seulement pour lire des fichiers, vous pouvez aussi les éditer ! Voyons maintenant seulement quelques fonctionnalités concernant la lecture de documents. Veuillez noter que vim propose la coloration syntaxique.

vim file_name.txt

Une fois le fichier ouvert, soyez prudent ! Ne commencez pas à taper, ou vous allez tout gâcher ! En fait, même si vous pouvez voir le curseur, vous devez appuyer sur ‘i’ pour commencer à taper et ‘ESC’ après avoir fini de taper. Au fait, je vais maintenant vous montrer quelques commandes utiles qui concernent la lecture (et non l’écriture) :

  • : : vous devez taper deux-points avant d’insérer chacune des commandes suivantes !
  • q! : sortie du fichier sans demander de confirmation. C’est la même chose que q si vous n’avez pas modifié le texte.
  • /word : recherche de ‘word’ dans le document.
  • 230 : va à la ligne ‘230’.

Astuce : un raccourci pour insérer deux-points, puis taper « wq ! » pour écrire des modifications dans le fichier et quitter sans demander de confirmation, vous pouvez maintenir « SHIFT » enfoncé et appuyer deux fois sur « z ».

C’est le meilleur choix pour lire les fichiers de code.

Recherche le nom exact d’un paquet pour apt-get #

Si vous savez que vous avez besoin d’un paquet <packet>, mais que vous ne connaissez pas le nom exact <packet-exact-name>, au lieu de chercher avec Google, essayez ce qui suit :

sudo apt-get update
apt-cache search <packet>

Cela vous renverra une liste de noms de paquets avec une description. Une fois que vous avez identifié le nom exact du paquet <packet-exact-name>, installez-le normalement :

sudo apt-get install <packet-exact-name>

Limiter la sortie aux lignes avec la chaîne souhaitée #

Si vous exécutez une commande qui renvoie des centaines de lignes, mais que votre intérêt se porte uniquement sur les lignes contenant le mot <mot>, vous devez absolument utiliser grep ! Par exemple, essayez d’exécuter :

ifconfig -a

Vérifiez la sortie, qui peut n’être que de quelques lignes ou assez longue, si vous avez un serveur avec plusieurs interfaces réseau. Pour afficher (par exemple) uniquement les lignes contenant ‘HWaddr’, essayez d’utiliser grep :

ifconfig -a | grep HWaddr

Maintenant, la sortie devrait être beaucoup plus courte ! De manière générale, vous pouvez utiliser grep de la manière suivante :

<command-with-output> | grep <word>

Compressez les fichiers et les dossiers à l’aide de la commande tar #

Utilisez la commande tar (tape archive) pour compresser vos fichiers et dossiers. Cela revient à créer des fichiers .ZIP dans l’environnement Windows.

Syntax: tar -zcvf <output tar file> <source file> 

Example: tar -zcvf outputfile.tar.gz source file

Voici ce que signifient réellement ces commutateurs :

-c : créer une archive.
-z : Compressez l’archive avec gzip.
-v : affiche la progression dans le terminal lors de la création de l’archive, également appelé mode “verbeux”. Le v est toujours facultatif dans ces commandes, mais il est utile.
-f : Permet de spécifier le nom de fichier de l’archive.
