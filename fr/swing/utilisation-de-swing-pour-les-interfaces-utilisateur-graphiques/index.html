Quitter l’application à la fermeture de la fenêtre #

Il est facile d’oublier de quitter l’application lorsque la fenêtre est fermée. N’oubliez pas d’ajouter la ligne suivante.

frame.setDefaultCloseOperation(WindowConstants.EXIT_ON_CLOSE); //Quit the application when the JFrame is closed

Création d’une fenêtre vide (JFrame) #

Création du JFrame #

Créer une fenêtre est facile. Il vous suffit de créer un JFrame.

JFrame frame = new JFrame();

Intituler la fenêtre #

Vous pouvez donner un titre à votre fenêtre. Vous pouvez le faire en passant une chaîne lors de la création du JFrame, ou en appelant frame.setTitle(String title).

JFrame frame = new JFrame("Super Awesome Window Title!");
//OR
frame.setTitle("Super Awesome Window Title!");

Définition de la taille de la fenêtre #

La fenêtre sera la plus petite possible une fois créée. Pour l’agrandir, vous pouvez définir sa taille explicitement :

frame.setSize(512, 256);

Ou vous pouvez avoir la taille du cadre lui-même en fonction de la taille de son contenu avec la méthode pack().

frame.pack();

Les méthodes setSize() et pack() s’excluent mutuellement, utilisez donc l’une ou l’autre.

Que faire lors de la fermeture de la fenêtre #

Notez que l’application ne se fermera pas lorsque la fenêtre sera fermée. Vous pouvez quitter l’application après la fermeture de la fenêtre en demandant à JFrame de le faire.

frame.setDefaultCloseOperation(WindowConstants.EXIT_ON_CLOSE);

Alternativement, vous pouvez dire à la fenêtre de faire autre chose lorsqu’elle est fermée.

WindowConstants.DISPOSE_ON_CLOSE //Get rid of the window
WindowConstants.EXIT_ON_CLOSE //Quit the application
WindowConstants.DO_NOTHING_ON_CLOSE //Don't even close the window
WindowConstants.HIDE_ON_CLOSE //Hides the window - This is the default action

Création d’un volet de contenu #

Une étape facultative consiste à créer un volet de contenu pour votre fenêtre. Ce n’est pas nécessaire, mais si vous voulez le faire, créez un JPanel et appelez frame.setContentPane(Component component).

JPanel pane = new JPanel();
frame.setContentPane(pane);

Affichage de la fenêtre #

Après l’avoir créé, vous voudrez créer vos composants, puis afficher la fenêtre. L’affichage de la fenêtre se fait en tant que tel.

frame.setVisible(true);

Exemple #

Pour ceux d’entre vous qui aiment copier et coller, voici un exemple de code.

JFrame frame = new JFrame("Super Awesome Window Title!"); //Create the JFrame and give it a title
frame.setSize(512, 256); //512 x 256px size
frame.setDefaultCloseOperation(WindowConstants.EXIT_ON_CLOSE); //Quit the application when the JFrame is closed

JPanel pane = new JPanel(); //Create the content pane
frame.setContentPane(pane); //Set the content pane

frame.setVisible(true); //Show the window

Ajout de composants #

Un composant est une sorte d’élément d’interface utilisateur, tel qu’un bouton ou un champ de texte.

Création d’un composant #

La création de composants est presque identique à la création d’une fenêtre. Au lieu de créer un JFrame cependant, vous créez ce composant. Par exemple, pour créer un JButton, procédez comme suit.

JButton button = new JButton();

De nombreux composants peuvent avoir des paramètres qui leur sont transmis lors de leur création. Par exemple, un bouton peut recevoir du texte à afficher.

JButton button = new JButton("Super Amazing Button!");

Si vous ne souhaitez pas créer de bouton, une liste des composants communs peut être trouvée dans un autre exemple sur cette page.

Les paramètres qui peuvent leur être transmis varient d’un composant à l’autre. Un bon moyen de vérifier ce qu’ils peuvent accepter est de consulter les paramètres de votre IDE (si vous en utilisez un). Les raccourcis par défaut sont répertoriés ci-dessous.

  • IntelliJ IDEA - Windows/Linux : CTRL + P
  • IntelliJ IDEA - OS X / macOS : CMD + P
  • Éclipse : CTRL + MAJ + Espace
  • NetBeans : CTRL + P

[![Exemple d’affichage des informations sur les paramètres de méthode dans IntelliJ IDEA][1]][1]

Affichage du composant #

Une fois qu’un composant a été créé, vous définissez généralement ses paramètres. Après cela, vous devez le mettre quelque part, comme sur votre JFrame, ou sur votre volet de contenu si vous en avez créé un.

frame.add(button); //Add to your JFrame
//OR
pane.add(button); //Add to your content pane
//OR
myComponent.add(button); //Add to whatever

Exemple #

Voici un exemple de création d’une fenêtre, de définition d’un volet de contenu et d’ajout d’un bouton.

JFrame frame = new JFrame("Super Awesome Window Title!"); //Create the JFrame and give it a title
frame.setSize(512, 256); //512 x 256px size
frame.setDefaultCloseOperation(WindowConstants.EXIT_ON_CLOSE); //Quit the application when the JFrame is closed

JPanel pane = new JPanel(); //Create the content pane
frame.setContentPane(pane); //Set the content pane

JButton button = new JButton("Super Amazing Button!"); //Create the button
pane.add(button); //Add the button to the content pane

frame.setVisible(true); //Show the window

Définition des paramètres pour les composants #

Les composants ont divers paramètres qui peuvent être définis pour eux. Ils varient d’un composant à l’autre, donc un bon moyen de voir quels paramètres peuvent être définis pour les composants est de commencer à taper componentName.set, et de laisser la saisie semi-automatique de votre IDE (si vous utilisez un IDE) suggérer des méthodes. Le raccourci par défaut dans de nombreux IDE, s’il ne s’affiche pas automatiquement, est CTRL + Espace.

[![Autocomplete dans IntelliJ IDEA][1]][1]

Création d’interfaces utilisateur interactives #

Avoir un bouton là-bas, c’est bien beau, mais à quoi ça sert si cliquer dessus ne fait rien ? ActionListeners sont utilisés pour indiquer à votre bouton ou à un autre composant de faire quelque chose lorsqu’il est activé.

L’ajout de ActionListeners se fait en tant que tel.

buttonA.addActionListener(new ActionListener() {
    @Override
    public void actionPerformed(ActionEvent e) {
        //Code goes here...
        System.out.println("You clicked the button!");
    }
});

Ou, si vous utilisez Java 8 ou supérieur…

buttonA.addActionListener(e -> {
    //Code
    System.out.println("You clicked the button!");
});

Exemple (Java 8 et supérieur) #

JFrame frame = new JFrame("Super Awesome Window Title!"); //Create the JFrame and give it a title
frame.setSize(512, 256); //512 x 256px size
frame.setDefaultCloseOperation(WindowConstants.EXIT_ON_CLOSE); //Quit the application when the JFrame is closed

JPanel pane = new JPanel(); //Create a pane to house all content
frame.setContentPane(pane);

JButton button = new JButton("Click me - I know you want to.");
button.addActionListener(e -> {
    //Code goes here
    System.out.println("You clicked me! Ouch.");
});
pane.add(buttonA);

frame.setVisible(true); //Show the window

Organisation de la disposition des composants #

L’ajout de composants les uns après les autres donne une interface utilisateur difficile à utiliser, car les composants sont tous quelque part. Les composants sont classés de haut en bas, chaque composant dans une “ligne” distincte.

Pour remédier à cela et vous offrir, en tant que développeur, la possibilité de disposer facilement les composants, Swing a LayoutManagers.

Ces LayoutManagers sont couverts plus en détail dans Introduction aux Layout Managers ainsi que les rubriques distinctes du gestionnaire de mise en page :

  • [Disposition GridBag] [2]
