Pour utiliser [std::bitset][1], vous devrez inclure l’en-tête [<bitset>][2].

#include <bitset>

std::bitset surcharge toutes les fonctions de l’opérateur pour permettre la même utilisation que la gestion de style c des bitsets.

Références

  • [Bit Twiddling Hacks] [3]

[1] : http://en.cppreference.com/w/cpp/utility/bitset [2] : http://en.cppreference.com/w/cpp/header/bitset [3] : https://graphics.stanford.edu/~seander/bithacks.html

Basculer un peu #

Manipulation de bits de style C #

Un bit peut être basculé à l’aide de l’opérateur XOR (^).

// Bit x will be the opposite value of what it is currently
number ^= 1LL << x;

Utilisation de std::bitset #

std::bitset<4> num(std::string("0100"));
num.flip(2); // num is now 0000
num.flip(0); // num is now 0001
num.flip();  // num is now 1110 (flips all bits)

Je vérifie un peu #

Manipulation de bits de style C #

La valeur du bit peut être obtenue en décalant le nombre vers la droite x fois, puis en effectuant un ET bit à bit (&) dessus :

(number >> x) & 1LL;  // 1 if the 'x'th bit of 'number' is set, 0 otherwise

L’opération de décalage vers la droite peut être mise en œuvre sous la forme d’un décalage arithmétique (signé) ou d’un décalage logique (non signé). Si number dans l’expression number >> x a un type signé et une valeur négative, la valeur résultante est définie par l’implémentation.

Si nous avons besoin de la valeur de ce bit directement sur place, nous pourrions plutôt décaler le masque vers la gauche :

(number & (1LL << x));  // (1 << x) if the 'x'th bit of 'number' is set, 0 otherwise

L’un ou l’autre peut être utilisé comme conditionnel, puisque toutes les valeurs non nulles sont considérées comme vraies.

Utilisation de std::bitset #

std::bitset<4> num(std::string("0010"));
bool bit_val = num.test(1);  // bit_val value is set to true;

Définir tous les bits #

Manipulation de bits de style C #

x = -1; // -1 == 1111 1111 ... 1111b

(Voir ici pour une explication de pourquoi cela fonctionne et est en fait la meilleure approche.)

Utilisation de std::bitset #

std::bitset<10> x;
x.set(); // Sets all bits to '1'

Supprimer le bit défini le plus à droite #

Manipulation de bits de style C #

template <typename T>
T rightmostSetBitRemoved(T n)
{
    // static_assert(std::is_integral<T>::value && !std::is_signed<T>::value, "type should be unsigned"); // For c++11 and later
    return n & (n - 1);
}

Explication

  • si n vaut zéro, on a 0 & 0xFF..FF qui vaut zéro
  • sinon n peut être écrit 0bxxxxxx10..00 et n - 1 est 0bxxxxxx011..11, donc n & (n - 1) est 0bxxxxxx000..00.

Réglage un peu #

Manipulation de bits de style C #

Un bit peut être défini à l’aide de l’opérateur OR au niveau du bit (|).

// Bit x will be set
number |= 1LL << x;

Utilisation de std::bitset #

set(x) ou set(x,true) - définit le bit à la position x sur 1.

std::bitset<5> num(std::string("01100"));
num.set(0);      // num is now 01101
num.set(2);      // num is still 01101
num.set(4,true); // num is now 11110

Effacer un peu #

Manipulation de bits de style C #

Un bit peut être effacé à l’aide de l’opérateur ET au niveau du bit (&).

// Bit x will be cleared
number &= ~(1LL << x);

Utilisation de std::bitset #

reset(x) ou set(x,false) - efface le bit à la position x.

std::bitset<5> num(std::string("01100"));
num.reset(2);     // num is now 01000
num.reset(0);     // num is still 01000
num.set(3,false); // num is now 00000

Changer le nième bit en x #

Manipulation de bits de style C #

// Bit n will be set if x is 1 and cleared if x is 0.
number ^= (-x ^ number) & (1LL << n);

Utilisation de std::bitset #

set(n,val) - définit le bit n à la valeur val.

std::bitset<5> num(std::string("00100"));
num.set(0,true);  // num is now 00101
num.set(2,false); // num is now 00001

Jeu de bits de comptage #

Le décompte de la population d’une chaîne de bits est souvent nécessaire en cryptographie et dans d’autres applications et le problème a été largement étudié.

La méthode naïve nécessite une itération par bit :

unsigned value = 1234;
unsigned bits = 0;  // accumulates the total number of bits set in `n`

for (bits = 0; value; value >>= 1)
  bits += value & 1;

Une bonne astuce (basée sur https://www.wikiod.com/fr/docs/c%2b%2b/3016/bit-manipulation/17299/remove-rightmost-set-bit ) est :

unsigned bits = 0;  // accumulates the total number of bits set in `n`

for (; value; ++bits)
  value &= value - 1;

Il passe par autant d’itérations qu’il y a de bits définis, donc c’est bien quand on s’attend à ce que value ait peu de bits non nuls.

La méthode a été proposée pour la première fois par Peter Wegner (dans [CACM][1] 3 / 322 - 1960) et elle est bien connue puisqu’elle apparaît dans C Programming Language de Brian W. Kernighan et Dennis M. Ritchie.

Cela nécessite 12 opérations arithmétiques, dont l’une est un multication :

unsigned popcount(std::uint64_t x)
{
  const std::uint64_t m1  = 0x5555555555555555;  // binary: 0101...
  const std::uint64_t m2  = 0x3333333333333333;  // binary: 00110011..
  const std::uint64_t m4  = 0x0f0f0f0f0f0f0f0f;  // binary: 0000111100001111

  x -= (x >> 1) & m1;             // put count of each 2 bits into those 2 bits
  x = (x & m2) + ((x >> 2) & m2); // put count of each 4 bits into those 4 bits 
  x = (x + (x >> 4)) & m4;        // put count of each 8 bits into those 8 bits 
  return (x * h01) >> 56;  // left 8 bits of x + (x<<8) + (x<<16) + (x<<24) + ... 
}

Ce type d’implémentation a le meilleur comportement dans le pire des cas (voir [Poids de Hamming] [2] pour plus de détails).

De nombreux processeurs ont une instruction spécifique (comme popcnt de x86) et le compilateur peut offrir une fonction intégrée spécifique (** non standard **). Par exemple. avec g++ il y a :

int __builtin_popcount (unsigned x);

[1] : http://cacm.acm.org/ [2] : https://en.wikipedia.org/wiki/Hamming_weight

Vérifie si un entier est une puissance de 2 #

L’astuce n & (n - 1) (voir https://www.wikiod.com/fr/docs/c%2b%2b/3016/bit-manipulation/17299/remove-rightmost-set-bit) est également utile pour savoir si un entier est une puissance de 2 :

bool power_of_2 = n && !(n & (n - 1));

Notez que sans la première partie de la vérification (n &&), 0 est incorrectement considéré comme une puissance de 2.

Application de manipulation de bits : minuscule à majuscule #

L’une des nombreuses applications de la manipulation de bits consiste à convertir une lettre minuscule en majuscule ou vice versa en choisissant un masque et une opération de bit appropriée. Par exemple, la lettre a a cette représentation binaire 01(1)00001 tandis que sa contrepartie majuscule a 01(0)00001. Ils diffèrent uniquement par le bit entre parenthèses. Dans ce cas, la conversion de la lettre a de minuscule en majuscule définit essentiellement le bit entre parenthèses sur un. Pour ce faire, nous procédons comme suit :

/****************************************
convert small letter to captial letter.
========================================
     a: 01100001
  mask: 11011111  <-- (0xDF)  11(0)11111
      :---------
a&mask: 01000001  <-- A letter
*****************************************/

Le code pour convertir une lettre en lettre A est

#include <cstdio>

int main()
{
    char op1 = 'a';  // "a" letter (i.e. small case)
    int mask = 0xDF; // choosing a proper mask

    printf("a (AND) mask = A\n");
    printf("%c   &   0xDF = %c\n", op1, op1 & mask);
    
    return 0;
}

Le résultat est

$ g++ main.cpp -o test1
$ ./test1
a (AND) mask = A
a   &   0xDF = A
