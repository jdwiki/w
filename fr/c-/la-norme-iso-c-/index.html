En 1998, il y a eu une première publication de la norme faisant du C++ un langage standardisé en interne. À partir de ce moment, C++ a évolué, aboutissant à différents dialectes de C++. Sur cette page, vous trouverez un aperçu de toutes les différentes normes et de leurs modifications par rapport à la version précédente. Les détails sur la façon d’utiliser ces fonctionnalités sont décrits sur des pages plus spécialisées.

Lorsque C++ est mentionné, une référence est souvent faite au “Standard”. Mais quelle est exactement cette norme ?

C++ a une longue histoire. Commencé comme un petit projet par Bjarne Stroustrup au sein des Bell Labs, au début des années 90, il était devenu très populaire. Plusieurs entreprises créaient des compilateurs C++ afin que les utilisateurs puissent exécuter leurs compilateurs C++ sur une large gamme d’ordinateurs. Mais pour faciliter cela, tous ces compilateurs concurrents devraient partager une même définition du langage.

À ce moment-là, le langage C avait été standardisé avec succès. Cela signifie qu’une description formelle de la langue a été écrite. Cela a été soumis à l’American National Standards Institute (ANSI), qui a ouvert le document pour examen et l’a ensuite publié en 1989. Un an plus tard, l’Organisation internationale de normalisation (parce qu’elle aurait des acronymes différents dans différentes langues, ils ont choisi une forme , ISO, dérivé du grec isos, signifiant égal.) a adopté la norme américaine comme norme internationale.

Pour C++, il était clair dès le début qu’il y avait un intérêt international. Un groupe de travail au sein de l’ISO a été lancé (connu sous le nom de WG21, au sein du sous-comité 22). Ce groupe de travail a rédigé une première norme vers 1995. Mais comme nous, les programmeurs, le savons, il n’y a rien de plus dangereux pour une livraison planifiée que des fonctionnalités de dernière minute, et cela est également arrivé à C++. En 1995, une nouvelle bibliothèque cool nommée STL a fait surface, et les personnes travaillant dans WG21 ont décidé d’ajouter une version allégée au projet de norme C++. Naturellement, cela a fait manquer les délais et ce n’est que 3 ans plus tard que le document est devenu définitif. L’ISO est une organisation très formelle, c’est pourquoi la norme C++ a été baptisée du nom peu commercialisable d’ISO/IEC 14882. Comme les normes peuvent être mises à jour, cette version exacte est devenue 14882:1998.

Et en effet, il y avait une demande de mise à jour de la norme. Le Standard est un document très épais, qui vise à décrire exactement comment les compilateurs C++ doivent fonctionner. Même une légère ambiguïté peut valoir la peine d’être corrigée, donc en 2003, une mise à jour a été publiée en tant que 14882: 2003. Cependant, cela n’a ajouté aucune fonctionnalité à C++ ; les nouvelles fonctionnalités étaient prévues pour la deuxième mise à jour.

De manière informelle, cette deuxième mise à jour était connue sous le nom de C++0x, car on ne savait pas si cela prendrait jusqu’en 2008 ou 2009. Eh bien, cette version a également subi un léger retard, c’est pourquoi elle est devenue 14882:2011.

Heureusement, WG21 a décidé de ne pas laisser cela se reproduire. C++11 a été bien accueilli et a suscité un regain d’intérêt pour C++. Alors, pour garder le rythme, la troisième mise à jour est passée de la planification à la publication en 3 ans, pour devenir 14882:2014.

Le travail ne s’est pas arrêté là non plus. La norme C++17 a été proposée et les travaux pour C++20 ont commencé.

Brouillons de travail actuels #

Toutes les normes ISO publiées sont disponibles à la vente dans la boutique ISO ( http://www.iso.org ). Les versions préliminaires des normes C++ sont toutefois accessibles gratuitement au public.

Les différentes versions de la norme :

  • À venir (Parfois appelé C++20 ou C++2a) : Brouillon de travail actuel (version HTML)
  • Proposé (Parfois appelé C++17 ou C++1z) : [Projet de travail de mars 2017 N4659](http://www.open-std.org/jtc1/sc22/wg21/docs/papers/2017/n4659 .pdf).
  • C++14 (parfois appelé C++1y) : ébauche de travail de novembre 2014 N4296
  • C++11 (parfois appelé C++0x) : ébauche de travail N3242 de février 2011 -C++03 -C++98

C++17 #

La norme C++17 est complète et a été proposée pour la normalisation. Dans les compilateurs avec prise en charge expérimentale de ces fonctionnalités, il est généralement appelé C++1z.

C++14 #

La norme C++14 est souvent qualifiée de correction de bogue pour C++11. Il ne contient qu’une liste limitée de modifications dont la plupart sont des extensions des nouvelles fonctionnalités de C++11. Vous trouverez ci-dessous un aperçu des modifications telles qu’elles ont été regroupées sur la FAQ isocpp avec des liens vers une documentation plus détaillée.

Extensions de langue ================================================= ===============

  • Littéraux binaires
  • Déduction généralisée de type retour
  • decltype (auto)
  • [Captures lambda généralisées][1]
  • [Lambdas génériques][2]
  • Modèles variables
  • constexpr étendu
  • [L’attribut [[obsolète]]][3]
  • [Séparateurs de chiffres][4]

Extensions de bibliothèque ================================================= =============

  • Verrouillage partagé
  • Littéraux définis par l’utilisateur pour les types std ::
  • [std::make_unique][5]
  • Transformation de type _t alias
  • [Adressage des tuples par type][6] (par exemple get<string>(t))
  • [Foncteurs d’opérateurs transparents][7] (par exemple, greater<>(x))
  • [std::quoted][8]

Obsolète / Supprimé ================================================= =============

  • [std::gets][9] est obsolète en C++11 et supprimé de C++14
  • [std::random_shuffle][10] est obsolète

C++98 #

C++98 est la première version standardisée de C++. Comme il a été développé comme une extension de C, de nombreuses fonctionnalités qui distinguent C++ de C sont ajoutées.

Extensions de langue (par rapport à C89/C90) #

  • Classes, classes dérivées, fonctions membres virtuelles, fonctions membres const
  • Surcharge de fonction, Surcharge d’opérateur
  • Commentaires sur une seule ligne (a été introduit dans le langage C avec la norme C99)
  • Références
  • nouveau et supprimer
  • type booléen (a été introduit dans le langage C avec la norme C99)
  • modèles
  • espaces de noms
  • exceptions
  • moulages spécifiques

Extensions de bibliothèque #

  • La bibliothèque de modèles standard

C++03 #

La norme C++03 traite principalement des rapports de défauts de la norme C++98. En dehors de ces défauts, il n’ajoute qu’une seule nouveauté.

Extensions de langue #

  • [Initalisation de la valeur][1]

C++20 #

C++20 est le futur standard de C++, actuellement en développement, basé sur le standard C++17. Sa progression peut être suivie sur le site Web officiel de l’ISO cpp.

Les fonctionnalités suivantes sont simplement ce qui a été accepté pour la prochaine version du standard C++, prévue pour 2020.

Extensions de langue ================================================= ===============

Aucune extension de langue n’a été acceptée pour le moment.

Extensions de bibliothèque ================================================= =============

Aucune extension de bibliothèque n’a été acceptée pour le moment.
