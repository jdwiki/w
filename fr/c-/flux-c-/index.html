Le constructeur par défaut de std::istream_iterator construit un itérateur qui représente la fin du flux. Ainsi, std::copy(std::istream_iterator<int>(ifs), std::istream_iterator<int>(), .... signifie copier de la position actuelle dans ifs jusqu’à la fin.

Flux de chaînes #

Impression de collections avec iostream #

Impression de base #

std::ostream_iterator permet d’imprimer le contenu d’un conteneur STL dans n’importe quel flux de sortie sans boucles explicites. Le deuxième argument du constructeur std::ostream_iterator définit le délimiteur. Par exemple, le code suivant :

std::vector<int> v = {1,2,3,4};
std::copy(v.begin(), v.end(), std::ostream_iterator<int>(std::cout, " ! "));

va imprimer

1 ! 2 ! 3 ! 4 !

Conversion de type implicite #

std::ostream_iterator permet de convertir implicitement le type de contenu du conteneur. Par exemple, réglons std::cout pour imprimer des valeurs à virgule flottante avec 3 chiffres après la virgule :

std::cout << std::setprecision(3);
std::fixed(std::cout);

et instancier std::ostream_iterator avec float, tandis que les valeurs contenues restent int :

std::vector<int> v = {1,2,3,4};
std::copy(v.begin(), v.end(), std::ostream_iterator<float>(std::cout, " ! "));

donc le code ci-dessus donne

1.000 ! 2.000 ! 3.000 ! 4.000 !

malgré std :: vector contient ints.

Génération et transformation #

Les fonctions std::generate, std::generate_n et std::transform fournissent un outil très puissant pour la manipulation de données à la volée. Par exemple, ayant un vecteur :

std::vector<int> v = {1,2,3,4,8,16};

nous pouvons facilement imprimer la valeur booléenne de l’instruction “x est pair” pour chaque élément :

std::boolalpha(std::cout); // print booleans alphabetically
std::transform(v.begin(), v.end(), std::ostream_iterator<bool>(std::cout, " "),
[](int val) {
    return (val % 2) == 0;
});

ou imprimez l’élément au carré :

std::transform(v.begin(), v.end(), std::ostream_iterator<int>(std::cout, " "),
[](int val) {
    return val * val;
});

Impression de N nombres aléatoires délimités par des espaces :

const int N = 10;
std::generate_n(std::ostream_iterator<int>(std::cout, " "), N, std::rand);

Tableaux #

Comme dans la section sur la lecture des fichiers texte, presque toutes ces considérations peuvent être appliquées aux tableaux natifs. Par exemple, imprimons des valeurs au carré à partir d’un tableau natif :

int v[] = {1,2,3,4,8,16};
std::transform(v, std::end(v), std::ostream_iterator<int>(std::cout, " "),
[](int val) {
    return val * val;
});

Lire un fichier jusqu’à la fin #

Lire un fichier texte ligne par ligne #

Une bonne façon de lire un fichier texte ligne par ligne jusqu’à la fin n’est généralement pas claire dans la documentation ifstream. Considérons quelques erreurs courantes commises par les programmeurs C++ débutants et une manière appropriée de lire le fichier.

Lignes sans caractères d’espacement #

Par souci de simplicité, supposons que chaque ligne du fichier ne contienne aucun symbole d’espacement.

ifstream a operator bool(), qui renvoie vrai lorsqu’un flux n’a pas d’erreurs et est prêt à être lu. De plus, ifstream::operator >> renvoie une référence au flux lui-même, afin que nous puissions lire et vérifier EOF (ainsi que les erreurs) sur une seule ligne avec une syntaxe très élégante :

std::ifstream ifs("1.txt");
std::string s;
while(ifs >> s) {
    std::cout << s << std::endl;
}

Lignes avec caractères d’espacement #

ifstream::operator >> lit le flux jusqu’à ce qu’un caractère d’espacement se produise, donc le code ci-dessus imprimera les mots d’une ligne sur des lignes séparées. Pour tout lire jusqu’à la fin de la ligne, utilisez std::getline au lieu de ifstream::operator >>. getline renvoie la référence au thread avec lequel il a travaillé, donc la même syntaxe est disponible :

while(std::getline(ifs, s)) {
    std::cout << s << std::endl;
}

Évidemment, std::getline devrait également être utilisé pour lire un fichier d’une seule ligne jusqu’à la fin.

Lire un fichier dans un tampon en une seule fois #

Enfin, lisons le fichier du début à la fin sans s’arrêter à aucun caractère, y compris les espaces et les retours à la ligne. Si nous savons que la taille exacte du fichier ou la limite supérieure de la longueur est acceptable, nous pouvons redimensionner la chaîne, puis lire :

s.resize(100);
std::copy(std::istreambuf_iterator<char>(ifs), std::istreambuf_iterator<char>(),
    s.begin());

Sinon, nous devons insérer chaque caractère à la fin de la chaîne, donc std::back_inserter est ce dont nous avons besoin :

std::copy(std::istreambuf_iterator<char>(ifs), std::istreambuf_iterator<char>(),
    std::back_inserter(s));

Alternativement, il est possible d’initialiser une collection avec des données de flux, en utilisant un constructeur avec des arguments de plage d’itérateur :

std::vector v(std::istreambuf_iterator<char>(ifs),
    std::istreambuf_iterator<char>());

Notez que ces exemples sont également applicables si ifs est ouvert en tant que fichier binaire :

std::ifstream ifs("1.txt", std::ios::binary);

Copie de flux #

Un fichier peut être copié dans un autre fichier avec des flux et des itérateurs :

std::ofstream ofs("out.file");
std::copy(std::istreambuf_iterator<char>(ifs), std::istreambuf_iterator<char>(),
    std::ostream_iterator<char>(ofs));
ofs.close();

ou redirigé vers tout autre type de flux avec une interface compatible. Par exemple, le flux réseau Boost.Asio :

boost::asio::ip::tcp::iostream stream;
stream.connect("example.com", "http");
std::copy(std::istreambuf_iterator<char>(ifs), std::istreambuf_iterator<char>(),
    std::ostream_iterator<char>(stream));
stream.close();

Tableaux #

Comme les itérateurs peuvent être considérés comme une généralisation des pointeurs, les conteneurs STL dans les exemples ci-dessus peuvent être remplacés par des tableaux natifs. Voici comment analyser des nombres dans un tableau :

int arr[100];
std::copy(std::istream_iterator<char>(ifs), std::istream_iterator<char>(), arr);

Méfiez-vous du dépassement de mémoire tampon, car les tableaux ne peuvent pas être redimensionnés à la volée après leur allocation. Par exemple, si le code ci-dessus est alimenté avec un fichier contenant plus de 100 nombres entiers, il tentera d’écrire en dehors du tableau et se heurtera à un comportement indéfini.

Analyse des fichiers #

Analyser des fichiers dans des conteneurs STL #

Les istream_iterators sont très utiles pour lire des séquences de nombres ou d’autres données analysables dans des conteneurs STL sans boucles explicites dans le code.

Utilisation d’une taille de conteneur explicite :

std::vector<int> v(100);
std::copy(std::istream_iterator<int>(ifs), std::istream_iterator<int>(),
    v.begin());

ou en insérant un itérateur :

std::vector<int> v;
std::copy(std::istream_iterator<int>(ifs), std::istream_iterator<int>(),
    std::back_inserter(v));

Notez que les nombres dans le fichier d’entrée peuvent être divisés par n’importe quel nombre de caractères d’espacement et de retour à la ligne.

Analyser des tables de texte hétérogènes #

Comme istream::operator>> lit le texte jusqu’à un symbole d’espacement, il peut être utilisé dans la condition while pour analyser des tables de données complexes. Par exemple, si nous avons un fichier avec deux nombres réels suivis d’une chaîne (sans espaces) sur chaque ligne :

1.12 3.14 foo
2.1 2.2 barr

il peut être analysé comme ceci:

std::string s;
double a, b;
while(ifs >> a >> b >> s) {
    std::cout << a << " " << b << " " << s << std::endl;
}

Transformation #

Toute fonction de manipulation de plage peut être utilisée avec les plages std::istream_iterator. L’un d’eux est std::transform, qui permet de traiter les données à la volée. Par exemple, lisons des valeurs entières, multiplions-les par 3,14 et stockons le résultat dans un conteneur à virgule flottante :

std::vector<double> v(100);
std::transform(std::istream_iterator<int>(ifs), std::istream_iterator<int>(),
v.begin(),
[](int val) {
    return val * 3.14;
});
