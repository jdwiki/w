Cas #

Introduit une étiquette case d’une instruction switch. L’opérande doit être une expression constante et correspondre à la condition de commutation dans le type. Lorsque l’instruction switch est exécutée, elle saute à l’étiquette de cas avec l’opérande égal à la condition, le cas échéant.

char c = getchar();
bool confirmed;
switch (c) {
  case 'y':
    confirmed = true;
    break;
  case 'n':
    confirmed = false;
    break;
  default:
    std::cout << "invalid response!\n";
    abort();
}

changer #

Selon la norme C++,

L’instruction switch entraîne le transfert du contrôle vers l’une des instructions en fonction de la valeur d’un état.

Le mot clé switch est suivi d’une condition entre parenthèses et d’un bloc, qui peut contenir des étiquettes case et une étiquette facultative default. Lorsque l’instruction switch est exécutée, le contrôle sera transféré soit à une étiquette case avec une valeur correspondant à celle de la condition, le cas échéant, soit à l’étiquette default, s’il y en a une.

La condition doit être une expression ou une déclaration, qui a un type entier ou énumération, ou un type classe avec une fonction de conversion en type entier ou énumération.

char c = getchar();
bool confirmed;
switch (c) {
  case 'y':
    confirmed = true;
    break;
  case 'n':
    confirmed = false;
    break;
  default:
    std::cout << "invalid response!\n";
    abort();
}

attraper #

Le mot-clé catch introduit un gestionnaire d’exceptions, c’est-à-dire un bloc dans lequel le contrôle sera transféré lorsqu’une exception de type compatible est lancée. Le mot clé catch est suivi d’une déclaration d’exception entre parenthèses, dont la forme est similaire à celle d’une déclaration de paramètre de fonction : le nom du paramètre peut être omis et les points de suspension ... sont autorisés, ce qui correspond à n’importe quel type. Le gestionnaire d’exceptions ne gérera l’exception que si sa déclaration est compatible avec le type de l’exception. Pour plus de détails, voir attraper les exceptions.

try {
    std::vector<int> v(N);
    // do something
} catch (const std::bad_alloc&) {
    std::cout << "failed to allocate memory for vector!" << std::endl;
} catch (const std::runtime_error& e) {
    std::cout << "runtime error: " << e.what() << std::endl;
} catch (...) {
    std::cout << "unexpected exception!" << std::endl;
    throw;
}

défaut #

Dans une instruction switch, introduit une étiquette qui sera sautée si la valeur de la condition n’est égale à aucune des valeurs des étiquettes de cas.

char c = getchar();
bool confirmed;
switch (c) {
  case 'y':
    confirmed = true;
    break;
  case 'n':
    confirmed = false;
    break;
  default:
    std::cout << "invalid response!\n";
    abort();
}

Définit un constructeur par défaut, un constructeur de copie, un constructeur de déplacement, un destructeur, un opérateur d’affectation de copie ou un opérateur d’affectation de déplacement pour avoir son comportement par défaut.

class Base {
    // ...
    // we want to be able to delete derived classes through Base*,
    // but have the usual behaviour for Base's destructor.
    virtual ~Base() = default;
};

si #

Introduit une instruction if. Le mot-clé “if” doit être suivi d’une condition entre parenthèses, qui peut être soit une expression, soit une déclaration. Si la condition est véridique, la sous-instruction après la condition sera exécutée.

int x;
std::cout << "Please enter a positive number." << std::endl;
std::cin >> x;
if (x <= 0) {
    std::cout << "You didn't enter a positive number!" << std::endl;
    abort();
}

autre #

La première sous-instruction d’une instruction if peut être suivie du mot-clé “else”. La sous-instruction après le mot clé else sera exécutée lorsque la condition est fausse (c’est-à-dire lorsque la première sous-instruction n’est pas exécutée).

int x;
std::cin >> x;
if (x%2 == 0) {
    std::cout << "The number is even\n";
} else {
    std::cout << "The number is odd\n";
}

aller à #

Saute à une instruction étiquetée, qui doit être située dans la fonction actuelle.

bool f(int arg) {
    bool result = false;
    hWidget widget = get_widget(arg);
    if (!g()) {
        // we can't continue, but must do cleanup still
        goto end;
    }
    // ...
    result = true;
  end:
    release_widget(widget);
    return result;
}

revenir #

Renvoie le contrôle d’une fonction à son appelant.

Si return a un opérande, l’opérande est converti dans le type de retour de la fonction et la valeur convertie est renvoyée à l’appelant.

int f() {
    return 42;
}
int x = f(); // x is 42
int g() {
    return 3.14;
}
int y = g(); // y is 3

Si return n’a pas d’opérande, la fonction doit avoir le type de retour void. En tant que cas particulier, une fonction renvoyant void peut également renvoyer une expression si l’expression a le type void.

void f(int x) {
    if (x < 0) return;
    std::cout << sqrt(x);
}
int g() { return 42; }
void h() {
    return f(); // calls f, then returns
    return g(); // ill-formed
}

Lorsque main revient, std::exit est appelé implicitement avec la valeur de retour, et la valeur est ainsi renvoyée à l’environnement d’exécution. (Cependant, revenir de main détruit les variables locales automatiques, contrairement à l’appel direct de std::exit.)

int main(int argc, char** argv) {
    if (argc < 2) {
        std::cout << "Missing argument\n";
        return EXIT_FAILURE; // equivalent to: exit(EXIT_FAILURE);
    }
}

jeter #

  1. Lorsque throw se produit dans une expression avec un opérande, son effet est de lever une [exception] (https://www.wikiod.com/fr/docs/c%2b%2b/1354/exceptions), qui est une copie de l’opérande.

    void print_asterisks(int count) {
    if (count < 0) {
        throw std::invalid_argument("count cannot be negative!");
    }
    while (count--) { putchar('*'); }
}

  2. Lorsque throw se produit dans une expression sans opérande, son effet est de [relancer l’exception actuelle] (https://www.wikiod.com/fr/docs/c%2b%2b/1354/exceptions/5574/rethrow-propagate -exception). S’il n’y a pas d’exception en cours, std::terminate est appelé.

    try {
    // something risky
} catch (const std::bad_alloc&) {
    std::cerr << "out of memory" << std::endl;
} catch (...) {
    std::cerr << "unexpected exception" << std::endl;
    // hope the caller knows how to handle this exception
    throw;
}

  3. Lorsque throw se produit dans un déclarateur de fonction, il introduit une spécification d’exception dynamique, qui répertorie les types d’exceptions que la fonction est autorisée à propager.

    // this function might propagate a std::runtime_error,
// but not, say, a std::logic_error
void risky() throw(std::runtime_error);
// this function can't propagate any exceptions
void safe() throw();

Les spécifications d’exception dynamiques sont obsolètes à partir de C++11.

Notez que les deux premières utilisations de “throw” listées ci-dessus constituent des expressions plutôt que des déclarations. (Le type d’une expression throw est void.) Cela permet de les imbriquer dans des expressions, comme ceci :

unsigned int predecessor(unsigned int x) {
    return (x > 0) ? (x - 1) : (throw std::invalid_argument("0 has no predecessor"));
}

essayer #

Le mot clé try est suivi d’un bloc, ou d’une liste d’initialisation de constructeur, puis d’un bloc (voir [ici](https://www.wikiod.com/fr/docs/c%2b%2b/1354/exceptions/6692/function -try-blocks-in-constructor)). Le bloc try est suivi d’un ou plusieurs catch blocks. Si une exception se propage hors du bloc try, chacun des blocs catch correspondants après le bloc try a la possibilité de gérer l’exception , si les types correspondent.

std::vector<int> v(N);     // if an exception is thrown here,
                           // it will not be caught by the following catch block
try {
    std::vector<int> v(N); // if an exception is thrown here,
                           // it will be caught by the following catch block
    // do something with v
} catch (const std::bad_alloc&) {
    // handle bad_alloc exceptions from the try block
}
