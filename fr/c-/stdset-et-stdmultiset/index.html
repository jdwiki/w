set est un type de conteneur dont les éléments sont triés et uniques. multiset est similaire, mais, dans le cas de multiset, plusieurs éléments peuvent avoir la même valeur.

Différents styles de C++ ont été utilisés dans ces exemples. Faites attention que si vous utilisez un compilateur C++98 ; une partie de ce code peut ne pas être utilisable.

Changer le tri par défaut d’un ensemble #

set et multiset ont des méthodes de comparaison par défaut, mais dans certains cas, vous devrez peut-être les surcharger.

Imaginons que nous stockions des valeurs de chaîne dans un ensemble, mais nous savons que ces chaînes ne contiennent que des valeurs numériques. Par défaut, le tri sera une comparaison de chaînes lexicographiques, donc l’ordre ne correspondra pas au tri numérique. Si vous souhaitez appliquer un tri équivalent à ce que vous auriez avec des valeurs int, vous avez besoin d’un foncteur pour surcharger la méthode compare :

#include <iostream>
#include <set>
#include <stdlib.h> 

struct custom_compare final
{
    bool operator() (const std::string& left, const std::string& right) const
    {
        int nLeft = atoi(left.c_str());
        int nRight = atoi(right.c_str());
        return nLeft < nRight;
    }
};

int main ()
{
    std::set<std::string> sut({"1", "2", "5", "23", "6", "290"});
  
    std::cout << "### Default sort on std::set<std::string> :" << std::endl;
    for (auto &&data: sut)
        std::cout << data << std::endl;
  
    std::set<std::string, custom_compare> sut_custom({"1", "2", "5", "23", "6", "290"},
                                                     custom_compare{}); //< Compare object optional as its default constructible.
  
    std::cout << std::endl << "### Custom sort on set :" << std::endl;
    for (auto &&data : sut_custom)
        std::cout << data << std::endl;
  
    auto compare_via_lambda = [](auto &&lhs, auto &&rhs){ return lhs > rhs; };
    using set_via_lambda = std::set<std::string, decltype(compare_via_lambda)>;
    set_via_lambda sut_reverse_via_lambda({"1", "2", "5", "23", "6", "290"},
                                          compare_via_lambda);
  
    std::cout << std::endl << "### Lambda sort on set :" << std::endl;
    for (auto &&data : sut_reverse_via_lambda)
        std::cout << data << std::endl;

    return 0;
}

La sortie sera :

1
2
23
290
5
6
### Custom sort on set :
1
2
5
6
23
290  

### Lambda sort on set :

5
290
23
2
1

Dans l’exemple ci-dessus, on peut trouver 3 façons différentes d’ajouter des opérations de comparaison au std::set, chacune d’elles est utile dans son propre contexte.

Tri par défaut #

Cela utilisera l’opérateur de comparaison de la clé (premier argument du modèle). Souvent, la clé fournira déjà une bonne valeur par défaut pour la fonction std::less<T>. Sauf si cette fonction est spécialisée, elle utilise l’opérateur< de l’objet. Ceci est particulièrement utile lorsqu’un autre code essaie également d’utiliser un certain ordre, car cela permet une cohérence sur l’ensemble de la base de code.

Écrire le code de cette façon réduira l’effort de mise à jour de votre code lorsque la clé change est l’API, comme : une classe contenant 2 membres qui se transforme en une classe contenant 3 membres. En mettant à jour operator< dans la classe, toutes les occurrences seront mises à jour.

Comme vous pouvez vous y attendre, l’utilisation du tri par défaut est une valeur par défaut raisonnable.

Tri personnalisé #

L’ajout d’un tri personnalisé via un objet avec un opérateur de comparaison est souvent utilisé lorsque la comparaison par défaut n’est pas conforme. Dans l’exemple ci-dessus, c’est parce que les chaînes font référence à des entiers. Dans d’autres cas, il est souvent utilisé lorsque vous souhaitez comparer des pointeurs (intelligents) en fonction de l’objet auquel ils se réfèrent ou parce que vous avez besoin de contraintes différentes pour la comparaison (exemple : comparer std::pair par la valeur de first).

Lors de la création d’un opérateur de comparaison, il doit s’agir d’un tri stable. Si le résultat de l’opérateur de comparaison change après l’insertion, vous aurez un comportement indéfini. Comme bonne pratique, votre opérateur de comparaison ne devrait utiliser que les données constantes (membres const, fonctions const …).

Comme dans l’exemple ci-dessus, vous rencontrerez souvent des classes sans membres comme opérateurs de comparaison. Cela se traduit par des constructeurs par défaut et des constructeurs de copie. Le constructeur par défaut vous permet d’omettre l’instance au moment de la construction et le constructeur de copie est requis car l’ensemble prend une copie de l’opérateur de comparaison.

Tri lambda #

Lambdas sont un moyen plus court d’écrire des objets de fonction. Cela permet d’écrire l’opérateur de comparaison sur moins de lignes, ce qui rend le code global plus lisible.

L’inconvénient de l’utilisation de lambdas est que chaque lambda obtient un type spécifique au moment de la compilation, donc decltype(lambda) sera différent pour chaque compilation de la même unité de compilation (fichier cpp) que sur plusieurs unités de compilation (lorsqu’elles sont incluses via En tête de fichier). Pour cette raison, il est recommandé d’utiliser des objets de fonction comme opérateur de comparaison lorsqu’ils sont utilisés dans des fichiers d’en-tête.

Cette construction est souvent rencontrée lorsqu’un std::set est utilisé dans la portée locale d’une fonction à la place, tandis que l’objet fonction est préféré lorsqu’il est utilisé comme arguments de fonction ou membres de classe.

Autres options de tri #

Comme l’opérateur de comparaison de std :: set est un argument de modèle, tous les [objets appelables] (https://www.wikiod.com/fr/docs/c%2b%2b/6073/callable-objects) peuvent être utilisés comme opérateur de comparaison et les exemples ci-dessus ne sont que des cas particuliers. Les seules restrictions que ces objets appelables ont sont :

  • Ils doivent être constructibles par copie
  • Elles doivent être appelables avec 2 arguments du type de la clé. (les conversions implicites sont autorisées, bien que non recommandées car elles peuvent nuire aux performances)

Supprimer des valeurs d’un ensemble #

La méthode la plus évidente, si vous souhaitez simplement réinitialiser votre set/multiset sur un vide, consiste à utiliser clear :

  std::set<int> sut;
  sut.insert(10);
  sut.insert(15);
  sut.insert(22);
  sut.insert(3); 
  sut.clear(); //size of sut is 0

Ensuite, la méthode “effacer” peut être utilisée. Il offre quelques possibilités semblant quelque peu équivalentes à l’insertion :

std::set<int> sut;
std::set<int>::iterator it;
          
sut.insert(10);
sut.insert(15);
sut.insert(22);
sut.insert(3); 
sut.insert(30);
sut.insert(33);
sut.insert(45);
    
// Basic deletion
sut.erase(3);
    
// Using iterator
it = sut.find(22);
sut.erase(it);
          
// Deleting a range of values
it = sut.find(33);
sut.erase(it, sut.end());
    
std::cout << std::endl << "Set under test contains:" << std::endl;
for (it = sut.begin(); it != sut.end(); ++it)
{
  std::cout << *it << std::endl;
}

La sortie sera :

Set under test contains:                                                                                                                                                                                                                                                                                                  
10                                                                                                                                                                                                                                                                                                                        
15                                                                                                                                                                                                                                                                                                                        
30

Toutes ces méthodes s’appliquent également à multiset. Veuillez noter que si vous demandez la suppression d’un élément d’un multiset et qu’il est présent plusieurs fois, toutes les valeurs équivalentes seront supprimées.

Insertion de valeurs dans un ensemble #

Trois méthodes d’insertion différentes peuvent être utilisées avec les ensembles.

  • Tout d’abord, une simple insertion de la valeur. Cette méthode renvoie une paire permettant à l’appelant de vérifier si l’insertion s’est réellement produite.

  • Deuxièmement, une insertion en donnant une indication de l’endroit où la valeur sera insérée. L’objectif est d’optimiser le temps d’insertion dans un tel cas, mais savoir où insérer une valeur n’est pas courant. Soyez prudent dans ce cas; la façon de donner un indice diffère selon les versions du compilateur.

  • Enfin, vous pouvez insérer une plage de valeurs en donnant un pointeur de début et un pointeur de fin. Celui de départ sera inclus dans l’insertion, celui de fin est exclu.

    #include #include

    int main () { std::set sut; std::set::iterator it; std::pair<std::set::iterator,bool> ret;

    // Basic insert
sut.insert(7);
sut.insert(5);
sut.insert(12);

ret = sut.insert(23);

Insertion de valeurs dans un multiset #

Toutes les méthodes d’insertion à partir d’ensembles s’appliquent également aux multiensembles. Néanmoins, une autre possibilité existe, qui fournit un initializer_list :

auto il = { 7, 5, 12 };
std::multiset<int> msut;
msut.insert(il);

Recherche de valeurs dans set et multiset #

Il existe plusieurs manières de rechercher une valeur donnée dans std::set ou dans std::multiset :

Pour obtenir l’itérateur de la première occurrence d’une clé, la fonction find() peut être utilisée. Elle renvoie end() si la clé n’existe pas.

  std::set<int> sut;
  sut.insert(10);
  sut.insert(15);
  sut.insert(22);
  sut.insert(3); // contains 3, 10, 15, 22    

  auto itS = sut.find(10); // the value is found, so *itS == 10
  itS = sut.find(555); // the value is not found, so itS == sut.end()   

  std::multiset<int> msut;
  sut.insert(10);
  sut.insert(15);
  sut.insert(22);
  sut.insert(15);
  sut.insert(3); // contains 3, 10, 15, 15, 22  

  auto itMS = msut.find(10);

Une autre méthode consiste à utiliser la fonction count(), qui compte le nombre de valeurs correspondantes trouvées dans le set/multiset (dans le cas d’un set, la valeur de retour ne peut être que 0 ou 1). En utilisant les mêmes valeurs que ci-dessus, nous aurons :

int result = sut.count(10); // result == 1
result = sut.count(555); // result == 0

result = msut.count(10); // result == 1
result = msut.count(15); // result == 2

Dans le cas de std::multiset, plusieurs éléments peuvent avoir la même valeur. Pour obtenir cette plage, la fonction equal_range() peut être utilisée. Il renvoie std::pair ayant respectivement la limite inférieure (inclusive) et la limite supérieure (exclusive) de l’itérateur. Si la clé n’existe pas, les deux itérateurs pointent vers la valeur supérieure la plus proche (basée sur la méthode de comparaison utilisée pour trier le multiset donné).

auto eqr = msut.equal_range(15);
auto st = eqr.first; // point to first element '15'
auto en = eqr.second; // point to element '22'

eqr = msut.equal_range(9); // both eqr.first and eqr.second point to element '10'
