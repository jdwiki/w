Utilisé pour éviter les collisions de noms lors de l’utilisation de plusieurs bibliothèques, un espace de noms est un préfixe déclaratif pour les fonctions, les classes, les types, etc.

Syntaxe #

  • espace de noms identifiant(opt) { declaration-seq }
  • espace de noms en ligne identifier(opt) { declaration-seq } /* depuis C++11 */
  • inline(opt) namespace attribute-specifier-seq identifier(opt) { declaration-seq } /* depuis C++17 */
  • namespace enclosing-namespace-specifier :: identifier { declaration-seq } /* depuis C++17 */
  • namespace identifiant = qualified-namespace-specifier ;
  • using namespace nested-name-specifier(opt) namespace-name ;
  • attribute-specifier-seq using namespace nested-name-specifier(opt) namespace-name ; /* depuis C++11 */

Le mot-clé namespace a trois significations différentes selon le contexte :

  1. Lorsqu’il est suivi d’un nom facultatif et d’une séquence de déclarations entre accolades, il [définit un nouvel espace de noms] (https://www.wikiod.com/fr/docs/c%2b%2b/495/namespaces) ou [étend un existant namespace] (https://www.wikiod.com/fr/docs/c%2b%2b/495/namespaces/1623/extending-namespaces) avec ces déclarations. Si le nom est omis, l’espace de noms est un espace de noms sans nom.

  2. Lorsqu’il est suivi d’un nom et d’un signe égal, il déclare un alias d’espace de noms.

  3. Lorsqu’il est précédé de “using” et suivi d’un nom d’espace de noms, il forme une [* directive using *] (https://www.wikiod.com/fr/docs/c%2b%2b/495/namespaces/1624/using-namespaces #t=201608060458446751916), qui permet de trouver des noms dans l’espace de noms donné par une recherche de nom non qualifié (mais ne redéclare pas ces noms dans la portée actuelle). Une using-directive ne peut pas se produire dans la portée de la classe.

using namespace std; est déconseillé. Pourquoi? Parce que namespace std est énorme ! Cela signifie qu’il y a de fortes chances que les noms entrent en collision :

//Really bad!
using namespace std;

//Calculates p^e and outputs it to std::cout
void pow(double p, double e) { /*...*/ }

//Calls pow
pow(5.0, 2.0); //Error! There is already a pow function in namespace std with the same signature,
               //so the call is ambiguous

Que sont les espaces de noms ? #

Un espace de noms C++ est une collection d’entités C++ (fonctions, classes, variables), dont les noms sont préfixés par le nom de l’espace de noms. Lors de l’écriture de code dans un espace de noms, les entités nommées appartenant à cet espace de noms n’ont pas besoin d’être précédées du nom de l’espace de noms, mais les entités en dehors de celui-ci doivent utiliser le nom complet. Le nom complet a le format <espace de noms>::<entité>. Exemple:

namespace Example
{
  const int test = 5;

  const int test2 = test + 12; //Works within `Example` namespace
}

const int test3 = test + 3; //Fails; `test` not found outside of namespace.

const int test3 = Example::test + 3; //Works; fully qualified name used.

Les espaces de noms sont utiles pour regrouper des définitions associées. Prenez l’analogie d’un centre commercial. Généralement, un centre commercial est divisé en plusieurs magasins, chaque magasin vendant des articles d’une catégorie spécifique. Un magasin peut vendre des appareils électroniques, tandis qu’un autre magasin peut vendre des chaussures. Ces séparations logiques dans les types de magasins aident les acheteurs à trouver les articles qu’ils recherchent. Les espaces de noms aident les programmeurs C++, comme les acheteurs, à trouver les fonctions, les classes et les variables qu’ils recherchent en les organisant de manière logique. Exemple:

namespace Electronics
{
    int TotalStock;
    class Headphones
    {
        // Description of a Headphone (color, brand, model number, etc.)
    };
    class Television
    {
        // Description of a Television (color, brand, model number, etc.)
    };
}

namespace Shoes
{
    int TotalStock;
    class Sandal
    {
        // Description of a Sandal (color, brand, model number, etc.)
    };
    class Slipper
    {
        // Description of a Slipper (color, brand, model number, etc.)
    };
}

Il existe un seul espace de noms prédéfini, qui est l’espace de noms global qui n’a pas de nom, mais peut être désigné par ::. Exemple:

void bar() {
    // defined in global namespace
}
namespace foo {
    void bar() {
        // defined in namespace foo
    }
    void barbar() {
        bar();   // calls foo::bar()
        ::bar(); // calls bar() defined in global namespace
    }
}

Recherche dépendante de l’argument #

Lors de l’appel d’une fonction sans qualificateur d’espace de noms explicite, le compilateur peut choisir d’appeler une fonction dans un espace de noms si l’un des types de paramètres de cette fonction se trouve également dans cet espace de noms. C’est ce qu’on appelle “Argument Dependent Lookup”, ou ADL :

namespace Test
{
  int call(int i);

  class SomeClass {...};

  int call_too(const SomeClass &data);
}

call(5); //Fails. Not a qualified function name.

Test::SomeClass data;

call_too(data); //Succeeds

call échoue car aucun de ses types de paramètres ne provient de l’espace de noms Test. call_too fonctionne car SomeClass est membre de Test et par conséquent, il se qualifie pour les règles ADL.

Quand l’ADL ne se produit-elle pas #

ADL ne se produit pas si une recherche non qualifiée normale trouve un membre de classe, une fonction qui a été déclarée à la portée du bloc ou quelque chose qui n’est pas de type fonction. Par exemple:

void foo();
namespace N {
    struct X {};
    void foo(X ) { std::cout << '1'; }
    void qux(X ) { std::cout << '2'; }
}

struct C {
    void foo() {}
    void bar() {
        foo(N::X{}); // error: ADL is disabled and C::foo() takes no arguments
    }
};

void bar() {
    extern void foo(); // redeclares ::foo
    foo(N::X{});       // error: ADL is disabled and ::foo() doesn't take any arguments
}

int qux;

void baz() {
    qux(N::X{}); // error: variable declaration disables ADL for "qux"
}

Étendre les espaces de noms #

Une fonctionnalité utile des espaces de noms est que vous pouvez les développer (y ajouter des membres).

namespace Foo
{
    void bar() {}
}

//some other stuff

namespace Foo
{
    void bar2() {}
}

En utilisant la directive #

Le mot-clé [‘using’][1] a trois saveurs. Combiné avec le mot-clé ’namespace’, vous écrivez une ‘directive using’ :

Si vous ne voulez pas écrire Foo :: devant chaque élément de l’espace de noms Foo, vous pouvez utiliser using namespace Foo; pour importer chaque élément de Foo.

namespace Foo
{
    void bar() {}
    void baz() {}
}

//Have to use Foo::bar()
Foo::bar();

//Import Foo
using namespace Foo;
bar(); //OK
baz(); //OK

Il est également possible d’importer des entités sélectionnées dans un espace de noms plutôt que l’ensemble de l’espace de noms :

using Foo::bar;
bar(); //OK, was specifically imported
baz(); // Not OK, was not imported

Un mot d’avertissement : “l’utilisation d’espaces de noms” dans les fichiers d’en-tête est considérée comme un mauvais style dans la plupart des cas. Si cela est fait, l’espace de noms est importé dans chaque fichier qui inclut l’en-tête. Puisqu’il n’y a aucun moyen de “désutiliser” un espace de noms, cela peut entraîner une pollution de l’espace de noms (symboles plus nombreux ou inattendus dans l’espace de noms global) ou, pire, des conflits. Voir cet exemple pour une illustration du problème :

/***** foo.h *****/
namespace Foo
{
    class C;
}

/***** bar.h *****/
namespace Bar
{
    class C;
}

/***** baz.h *****/
#include "foo.h"
using namespace Foo;

/***** main.cpp *****/
#include "bar.h"
#include "baz.h"

using namespace Bar;
C c; // error: Ambiguity between Bar::C and Foo::C

Une using-directive ne peut pas se produire dans la portée de la classe.

[1] : https://www.wikiod.com/fr/docs/c%2B%2B/4891/keywords/19096/using#t=20160909070552352037

Création d’espaces de noms #

La création d’un espace de noms est très simple :

//Creates namespace foo
namespace Foo
{
    //Declares function bar in namespace foo
    void bar() {}
}

Pour appeler bar, vous devez d’abord spécifier l’espace de noms, suivi de l’opérateur de résolution de portée :: :

Foo::bar();

Il est permis de créer un espace de noms dans un autre, par exemple :

namespace A
{
    namespace B
    {
        namespace C
        {
            void bar() {}
        }
    }
}

Le code ci-dessus pourrait être simplifié comme suit :

namespace A::B::C
{
    void bar() {}
}

Espaces de noms sans nom/anonymes #

Un espace de noms sans nom peut être utilisé pour s’assurer que les noms ont un lien interne (ne peut être référencé que par l’unité de traduction actuelle). Un tel espace de noms est défini de la même manière que tout autre espace de noms, mais sans le nom :

namespace {
    int foo = 42;
}

foo n’est visible que dans l’unité de traduction dans laquelle il apparaît.

Il est recommandé de ne jamais utiliser d’espaces de noms sans nom dans les fichiers d’en-tête car cela donne une version du contenu pour chaque unité de traduction dans laquelle il est inclus. Ceci est particulièrement important si vous définissez des variables globales non constantes.

// foo.h
namespace {
    std::string globalString;
}

// 1.cpp
#include "foo.h" //< Generates unnamed_namespace{1.cpp}::globalString ...

globalString = "Initialize";

// 2.cpp
#include "foo.h" //< Generates unnamed_namespace{2.cpp}::globalString ...

std::cout << globalString; //< Will always print the empty string

Espaces de noms imbriqués compacts #

namespace a {
  namespace b {
    template<class T>
    struct qualifies : std::false_type {};
  }
}

namespace other {
  struct bob {};
}

namespace a::b {
  template<>
  struct qualifies<::other::bob> : std::true_type {};
}

Vous pouvez entrer les espaces de noms a et b en une seule étape avec namespace a::b à partir de C++17.

Espace de noms en ligne #

inline namespace inclut le contenu de l’espace de noms en ligne dans l’espace de noms englobant, donc

namespace Outer
{
    inline namespace Inner
    {
        void foo();
    }
}

est le plus souvent équivalent à

namespace Outer
{

    namespace Inner
    {
        void foo();
    }

    using Inner::foo;
}

mais les éléments de Outer::Inner:: et ceux associés à Outer:: sont identiques.

Donc la suite est équivalente

Outer::foo();
Outer::Inner::foo();

L’alternative using namespace Inner; ne serait pas équivalente pour certaines parties délicates comme la spécialisation de modèle :

Pour

#include <outer.h> // See below

class MyCustomType;
namespace Outer
{
    template <>
    void foo<MyCustomType>() { std::cout << "Specialization"; }
}

  • L’espace de noms en ligne permet la spécialisation de Outer::foo

    // outer.h
// include guard omitted for simplification

namespace Outer
{
    inline namespace Inner
    {
        template <typename T>
        void foo() { std::cout << "Generic"; }
    }
}

  • Alors que using namespace ne permet pas la spécialisation de Outer::foo

    // outer.h
// include guard omitted for simplification

namespace Outer
{
    namespace Inner
    {
        template <typename T>
        void foo() { std::cout << "Generic"; }
    }
    using namespace Inner;
    // Specialization of `Outer::foo` is not possible
    // it should be `Outer::Inner::foo`.
}

L’espace de noms en ligne est un moyen de permettre à plusieurs versions de cohabiter et de choisir par défaut la version “en ligne”

namespace MyNamespace
{
    // Inline the last version
    inline namespace Version2
    {
        void foo(); // New version
        void bar();
    }

    namespace Version1 // The old one
    {
        void foo();
    }

}

Et avec l’usage

MyNamespace::Version1::foo(); // old version
MyNamespace::Version2::foo(); // new version
MyNamespace::foo();           // default version : MyNamespace::Version1::foo();

Alias ​​d’espace de noms #

Un espace de noms peut recevoir un alias (c’est-à-dire un autre nom pour le même espace de noms) en utilisant la syntaxe namespace identifier =. Les membres de l’espace de noms alias sont accessibles en les qualifiant avec le nom de l’alias. Dans l’exemple suivant, l’espace de noms imbriqué AReallyLongName::AnotherReallyLongName n’est pas pratique à taper, donc la fonction qux déclare localement un alias N. Les membres de cet espace de noms sont alors accessibles simplement en utilisant N::.

namespace AReallyLongName {
    namespace AnotherReallyLongName {
        int foo();
        int bar();
        void baz(int x, int y);
    }
}
void qux() {
    namespace N = AReallyLongName::AnotherReallyLongName;
    N::baz(N::foo(), N::bar());
}
