Exigences de Team Foundation Server 2015 #

Team Foundation Server (TFS) peut être installé de deux manières d’infrastructure de base, le déploiement de serveur unique et le déploiement multi-serveur. La différence entre ces deux déploiements réside dans l’emplacement de la couche Application (TFS) et de la couche Données (MS SQL).

Il est considéré comme une bonne pratique d’installer TFS dans un déploiement multi-serveur.

Avant de pouvoir installer TFS 2015, vous devez vous assurer que vous avez satisfait aux exigences matérielles et logicielles.

  • Selon la documentation Microsoft, TFS 2015 prend en charge le matériel 32 bits, mais il est fortement recommandé de l’installer sur 64 bits, car TFS 2017 ** UNIQUEMENT ** prend en charge 64 bits. *

Déploiement sur un seul serveur :

  • 1 processeur double cœur
  • 4 à 8 Go de RAM

Déploiement multi-serveur :

  • Niveau d’application
  • 1 processeur double cœur
  • 8 à 16 Go de RAM (ou plus)
  • Stockage performant tel qu’un SSD
  • Niveau de données
  • 1 - 2 Processeur Quad-Core
  • 8 à 16 Go de RAM (ou plus)
  • Stockage performant tel qu’un SSD

Prise en charge des versions du système d’exploitation :

  • Serveur Windows 2016
  • Windows Server 2012 R2 (Essentials, Standard, Centre de données)
  • Windows Server 2012 (Essentials, Standard, Datacenter)
  • Windows Server 2008 R2 (minimum SP1) (Standard, Entreprise, Centre de données)

Prise en charge des versions de Microsoft SQL Server :

TFS prend en charge les éditions Express, Standard et Enterprise, mais il est recommandé d’utiliser Standard au strict minimum.

  • TFS 2015 Mise à jour 3
  • Serveur SQL 2016
  • Serveur SQL 2014
  • SQL Server 2012 (minimum SP1)
  • TFS 2015 (* Nous vous recommandons d’obtenir la mise à jour 3 dès que possible *)
  • Serveur SQL 2014
  • SQL Server 2012 (minimum SP1)

Voici les composants requis de Microsoft SQL :

  • Requis pour TFS
  • Services de moteur de base de données
  • Extractions de texte intégral et sémantique pour la recherche
  • Nécessaire pour les rapports
  • Services de rapports – natifs
  • Services d’analyse

Prise en charge des versions de Microsoft SharePoint :

  • SharePoint 2013 (Fondation, Standard, Entreprise)
  • SharePoint 2010 (Fondation, Standard, Entreprise)
