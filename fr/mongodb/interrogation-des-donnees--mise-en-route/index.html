Exemples de requêtes de base

Trouver() #

récupérer tous les documents d’une collection

db.collection.find({});

récupérer des documents dans une collection en utilisant une condition (similaire à WHERE dans MYSQL)

db.collection.find({key: value}); 
example
  db.users.find({email:"sample@email.com"});

récupérer des documents dans une collection à l’aide de conditions booléennes (opérateurs de requête)

//AND
db.collection.find( {
    $and: [
     { key: value }, { key: value } 
    ] 
})
//OR
db.collection.find( {
    $or: [
     { key: value }, { key: value } 
    ] 
})
//NOT
db.inventory.find( { key: { $not: value } } )

plus d’opérations booléennes et d’exemples peuvent être trouvés [ici][1]

REMARQUE : find() continuera à chercher dans la collection même si une correspondance de document a été trouvée, il est donc inefficace lorsqu’il est utilisé dans une grande collection, cependant en modélisant soigneusement vos données et/ou en utilisant des index, vous pouvez augmenter l’efficacité de find()

[1] : https://docs.mongodb.com/manual/reference/operator/query/#query-and-projection-operators

FindOne() #

db.collection.findOne({});

la fonctionnalité de requête est similaire à find() mais cela mettra fin à l’exécution dès qu’il trouvera un document correspondant à sa condition, s’il est utilisé avec un objet vide, il récupérera le premier document et le renverra. [documentation de l’API mongodb findOne()][1]

[1] : http://mongodb.github.io/node-mongodb-native/2.0/api/Collection.html#findOne

limiter, ignorer, trier et compter les résultats de la méthode find() #

Semblable aux méthodes d’agrégation également par la méthode find() vous avez la possibilité de limiter, ignorer, trier et compter les résultats. Disons que nous avons la collection suivante :

db.test.insertMany([
    {name:"Any", age:"21", status:"busy"}, 
    {name:"Tony", age:"25", status:"busy"}, 
    {name:"Bobby", age:"28", status:"online"}, 
    {name:"Sonny", age:"28", status:"away"}, 
    {name:"Cher", age:"20", status:"online"}
])

Pour répertorier la collection :

db.test.find({})

Reviendra :

{ "_id" : ObjectId("592516d7fbd5b591f53237b0"), "name" : "Any", "age" : "21", "status" : "busy" }
{ "_id" : ObjectId("592516d7fbd5b591f53237b1"), "name" : "Tony", "age" : "25", "status" : "busy" }
{ "_id" : ObjectId("592516d7fbd5b591f53237b2"), "name" : "Bobby", "age" : "28", "status" : "online" }
{ "_id" : ObjectId("592516d7fbd5b591f53237b3"), "name" : "Sonny", "age" : "28", "status" : "away" }
{ "_id" : ObjectId("592516d7fbd5b591f53237b4"), "name" : "Cher", "age" : "20", "status" : "online" }

Pour ignorer les 3 premiers documents :

db.test.find({}).skip(3)

Reviendra :

{ "_id" : ObjectId("592516d7fbd5b591f53237b3"), "name" : "Sonny", "age" : "28", "status" : "away" }
{ "_id" : ObjectId("592516d7fbd5b591f53237b4"), "name" : "Cher", "age" : "20", "status" : "online" }

Pour trier par ordre décroissant par nom de champ :

db.test.find({}).sort({ "name" : -1})

Reviendra :

{ "_id" : ObjectId("592516d7fbd5b591f53237b1"), "name" : "Tony", "age" : "25", "status" : "busy" }
{ "_id" : ObjectId("592516d7fbd5b591f53237b3"), "name" : "Sonny", "age" : "28", "status" : "away" }
{ "_id" : ObjectId("592516d7fbd5b591f53237b4"), "name" : "Cher", "age" : "20", "status" : "online" }
{ "_id" : ObjectId("592516d7fbd5b591f53237b2"), "name" : "Bobby", "age" : "28", "status" : "online" }
{ "_id" : ObjectId("592516d7fbd5b591f53237b0"), "name" : "Any", "age" : "21", "status" : "busy" }

Si vous voulez trier par ordre croissant, remplacez simplement -1 par 1

Pour compter les résultats :

db.test.find({}).count()

Reviendra :

5

Des combinaisons de ces méthodes sont également autorisées. Par exemple, obtenez 2 documents de la collection triée par ordre décroissant en sautant le premier :

db.test.find({}).sort({ "name" : -1}).skip(1).limit(2)

Reviendra :

{ "_id" : ObjectId("592516d7fbd5b591f53237b3"), "name" : "Sonny", "age" : "28", "status" : "away" }
{ "_id" : ObjectId("592516d7fbd5b591f53237b4"), "name" : "Cher", "age" : "20", "status" : "online" }

Méthode Find() avec Projection #

Dans MongoDB, la projection signifie sélectionner uniquement les données nécessaires plutôt que de sélectionner l’ensemble des données d’un document.

La syntaxe de base de la méthode find() avec projection est la suivante

> db.COLLECTION_NAME.find({},{KEY:1});

Si vous souhaitez afficher tous les documents sans le champ d’âge, la commande est la suivante

> db.people.find({},{age:0});

Si vous souhaitez afficher uniquement le champ âge, la commande est la suivante

> db.people.find({},{age:1});

Remarque : Le champ _id est toujours affiché lors de l’exécution de la méthode find(). Si vous ne voulez pas ce champ, vous devez le définir sur 0.

> db.people.find({},{name:1,_id:0});

Remarque : ‘1’ est utilisé pour afficher le champ tandis que ‘0’ est utilisé pour masquer les champs.
