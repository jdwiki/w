Créer une base de données SQL Windows Azure à partir du portail #

Comme pour tout dans Windows Azure, vous devez avoir un compte Windows Azure et un abonnement Azure. Une fois que vous avez les deux, accédez à https://portal.azure.com. À partir de là, vous pouvez ajouter de nouvelles ressources à votre abonnement Azure.

[![Nouvelle ressource][1]][1]

Cliquez sur Nouveau dans le menu de gauche. Une nouvelle lame sera ajoutée à droite de votre menu. Dans ce menu, choisissez Bases de données et une autre lame sera ajoutée à votre menu.

[![Lame Bases de données][2]][2]

Choisissez Base de données SQL, puis les panneaux “Nouveau” et “Bases de données” seront supprimés et un nouveau panneau Base de données SQL sera ajouté à votre menu.

[![Créer un panneau de base de données SQL Windows Azure][3]][3]

De là, vous devez choisir un nom de base de données pour donner cette nouvelle base de données. Vous devez également sélectionner l’abonnement auquel ajouter cette base de données, si vous avez accès à plusieurs abonnements. C’est là que la facture de votre base de données sera enregistrée.

Ensuite, si vous avez déjà défini des groupes de ressources pour votre abonnement, vous pouvez choisir de réutiliser l’un de ces groupes de ressources ou vous pouvez en créer un nouveau. Considérez ces groupes de ressources comme des collections de ressources dans Windows Azure qui partagent un cycle de vie commun. Toutes les ressources de ce groupe sont toutes créées et détruites dans le même laps de temps. Ce concept de groupes de ressources vous aidera à gérer vos ressources plus facilement à l’avenir.

Une fois que vous avez sélectionné le groupe de ressources le plus adapté aux besoins de votre entreprise, vous pouvez choisir la source de votre nouvelle base de données.

  1. Vous pouvez choisir de créer une base de données vierge, cela vous donne le plus de flexibilité avec votre nouvelle base de données.
  2. Vous pouvez également choisir de créer votre base de données à partir d’un exemple de base de données. Actuellement, les deux exemples d’options sont AdventureWorks, la base de données de démonstration OLTP (transactionnelle) de Microsoft.
  3. La dernière option consiste à choisir de créer votre base de données à partir d’une sauvegarde. Cette option est idéale pour ceux qui créent plusieurs copies d’une même base de données. Avec cette option, vous pouvez choisir de restaurer à partir d’une sauvegarde automatique effectuée sur une base de données SQL Windows Azure. Cela peut être utile si vous devez effectuer une restauration “partielle”.

Après avoir sélectionné l’option de source la plus appropriée aux besoins de votre entreprise, vous pouvez choisir de créer cette nouvelle base de données sur un serveur de base de données SQL Windows Azure existant (si vous en avez un) ou de créer un nouveau serveur. Si vous créez un nouveau serveur, le nom que vous donnez à ce nouveau serveur doit être unique dans tout Windows Azure. Vous devrez sélectionner l’alias de ce serveur. Le nom complet de ce serveur sera .database.windows.net.

Une fois que vous avez sélectionné l’option de serveur qui correspond le mieux aux besoins de votre entreprise, vous serez invité à définir un niveau de tarification. Si vous cliquez sur la sélection du niveau de tarification, vous pouvez voir les SLA et les prix actuels pour les différents niveaux de tarification.

Voici les bases des niveaux de tarification. Les bases de données de niveau B seront les plus lentes et auront les restrictions les plus sévères en termes de taille et de fonctionnalités, mais seront les moins chères. Les bases de données de niveau P seront les offres les plus rapides et les plus riches en fonctionnalités, mais leur prix en tiendra compte. Le niveau S est au milieu.

La dernière option à laquelle vous êtes confronté lors de la création d’une nouvelle base de données est le classement. Cela contrôle la façon dont les données seront comparées et triées. Si vous n’avez pas de bonne raison commerciale de changer l’option par défaut, je vous suggère de laisser cela seul. Une fois que vous avez effectué toutes vos sélections, appuyez sur Créer et une tâche en arrière-plan sera lancée pour créer votre nouvelle base de données.

Vous recevrez un avis dans le portail Windows Azure une fois que votre base de données sera prête. Afin de se connecter à cette base de données depuis l’extérieur de Windows Azure (comme votre poste de travail), des étapes supplémentaires seront nécessaires. Consultez la documentation StackOverflow pour la gestion des paramètres de pare-feu de la base de données SQL Windows Azure.

Créer une base de données SQL Windows Azure avec PowerShell #

Avant de commencer, assurez-vous que la dernière version d’[Azure PowerShell][1] est installée. Une fois installé, démarrez une session Azure PowerShell à partir de votre machine. Tout d’abord, vous devez vous connecter et vous authentifier auprès de Windows Azure.

Add-AzureRmAccount

Vous recevrez une boîte de dialogue vous demandant vos informations d’identification Azure. Saisissez-les, puis cliquez sur Connexion. Ensuite, vous devrez identifier à quel abonnement vous souhaitez ajouter cette nouvelle base de données SQL. Dans PowerShell, il est plus facile d’identifier votre abonnement par Globally Unique Identifier (GUID), plutôt que par son nom.

Pour trouver votre GUID d’abonnement, accédez à https://portal.azure.com et cliquez sur la vignette des abonnements à partir de la page d’accueil du portail.

[![entrez la description de l’image ici][2]][2]

Le panneau de la page de destination sera remplacé par votre panneau Abonnements. À partir de là, vous pouvez voir tous les abonnements auxquels vous avez accès dans votre contexte Windows Azure Active Directory actuel. Conservez une copie de ces GUID pour les futurs scripts PowerShell. Maintenant que vous disposez du GUID de votre abonnement, vous pouvez définir la portée de votre session PowerShell sur cet abonnement.

Set-AzureRmContext -SubscriptionId '<your subscription's GUID>'

Maintenant, si vous n’avez pas de groupe de ressources existant auquel ajouter cette nouvelle base de données SQL, vous devrez en créer un. Au minimum, votre groupe de ressources a besoin d’un nom et d’un emplacement. Les emplacements sont les différents centres de données qui peuvent héberger vos ressources Azure.

Pour obtenir une liste des centres de données Azure capables d’héberger votre base de données SQL, exécutez la commande suivante.

(Get-AzureRmLocation | Where-Object { $_.Providers -eq "Microsoft.Sql" }).Location

Notez que tous les emplacements sont en minuscules. Maintenant, pour créer un nouveau groupe de ressources, utilisez la commande suivante.

New-AzureRmResourceGroup 
     -Name '<new_resource_group>' `
     -Location '<data_center>'

Ensuite, vous avez besoin d’un serveur pour héberger votre base de données SQL. Si vous n’avez pas encore de serveur que vous souhaitez utiliser, créez-en un maintenant. Les serveurs nécessitent un nom de groupe de ressources, un nom de serveur et un emplacement.

New-AzureRmSqlServer `
     -Location '<data_center>' `
     -ResourceGroupName '<new_resource_group>' `
     -ServerName '<sql_server_name>'

Vous serez invité à entrer un nom d’utilisateur et un mot de passe administratifs. Ce sera le compte “sa” ou administrateur système de votre nouveau serveur SQL.

Maintenant que vous avez un groupe de ressources et un nom de serveur SQL, vous êtes prêt à créer la base de données elle-même. Les deux dernières sélections à faire sont l’édition et le niveau de service.

Pour l’édition, vous pouvez choisir Par défaut, Aucun, Premium, De base, Standard, DataWarehouse ou Gratuit. Pour le niveau de service, vous avez beaucoup plus de choix. Pour les débutants, restez fidèles à ceux dont vous pouvez voir les prix sur le calculateur de prix Azure. Pour les utilisateurs plus avancés, consultez l’applet de commande PowerShell[Get-AzureRmSqlServerServiceObjective][3].

La commande suivante créera votre base de données SQL.

New-AzureRmSqlDatabase `
     -DatabaseName '<database_name>' `
     -Edition 'basic' `
     -ResourceGroupName '<new_resource_group>' `
     -RequestedServiceObjectiveName 'basic' `
     -ServerName '<sql_server_name>'
