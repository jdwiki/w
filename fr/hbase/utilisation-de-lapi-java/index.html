Syntaxe #

  • HBaseConfiguration.create(); //Créer un fichier de configuration

  • Configuration.set (clé de chaîne, valeur de chaîne) ; //Ajouter une clé à la configuration

  • ConnectionFactory.createConnection (configuration HBaseConfiguration) ; // Se connecte à HBase

  • Connexion.getAdmin(); //Instancier un nouvel administrateur

  • new HTableDescriptor(Table.valueOf(String tableName));; //Créer un descripteur de table

  • HTableDescriptor.addFamily(new HColumnDescriptor(String familyName)); //Ajouter une famille au descripteur de table

  • Admin.createTable (descripteur HTableDescriptor) ; //Créer une table comme décrit dans le descripteur

  • Admin.deleteTable(TableName.valueOf(String tableName)); //Suppression d’un tableau

  • Connection.getTable(TableName.valueOf(String tableName)); //Obtenir un objet tableau

  • nouveau Get(Bytes.toBytes(String row_key)); //Créer un nouveau Get

  • table.get(Get get) //Renvoie un résultat

  • nouveau Put(String row_key); //Créer un nouveau put

  • table.put(Put put); //Insère la ou les lignes

  • nouveau scan(); //Créer une nouvelle analyse

  • table.getScanner (analyse par balayage) ; //Renvoie un ResultScanner

  • new Delete(Bytes.toBytes(String row_key)); //Créer une nouvelle suppression

  • table.delete(Supprimer supprimer); //Suppression d’une ligne du tableau

Paramètres #

Paramètre Valeurs possibles
CompareOp CompareOp.EQUAL , CompareOp.GREATER , CompareOp.GREATER_OR_EQUAL , CompareOp.LESS , CompareOp.LESS_OR_EQUAL , CompareOp.NOT_EQUAL , CompareOp.NO_OP (aucune opération)

Cette rubrique présente divers exemples d’utilisation de l’API Java pour HBase. Dans cette rubrique, vous apprendrez à créer et supprimer une table, à insérer, interroger et supprimer des lignes d’une table mais aussi à utiliser les filtres Scans.

Vous remarquerez que de nombreuses méthodes de cette API prennent des octets comme paramètres, par exemple le nom de la colonne ColumnFamily, cela est dû à l’implémentation de HBase. À des fins d’optimisation, au lieu de stocker les valeurs sous forme de chaîne, d’entier ou autre, il stocke une liste d’octets, c’est pourquoi vous devez analyser toutes ces valeurs en tant qu’octets. Pour ce faire, la méthode la plus simple consiste à utiliser Bytes.toBytes(something).

N’hésitez pas à signaler si vous constatez une erreur ou un malentendu.

Interroger HBase, Get, Put, Delete et Scans #

Dans HBase, vous pouvez utiliser 4 types d’opérations

  • Get : récupère une ligne
  • Put : insère une ou plusieurs ligne(s)
  • Supprimer : supprimer une ligne
  • Scan : récupère plusieurs lignes

Si vous souhaitez simplement récupérer une ligne, étant donné sa row_key, vous pouvez utiliser l’objet Get :

Get get = new Get(Bytes.toBytes("my_row_key"));
Table table = this.connection.getTable(TableName.valueOf("myTable"));
Result r = table.get(get);
byte[] value = r.getValue(Bytes.toBytes(columnFamily), Bytes.toBytes("myColumn"));
String valueStr = Bytes.toString(value);
System.out.println("Get result :" + valueStr);

Ici, nous obtenons uniquement la valeur de la colonne que nous voulons, si vous voulez récupérer toute la colonne, utilisez l’attribut rawCell de l’objet Get :

Get get = new Get(Bytes.toBytes(rowKey));
Table table = this.connection.getTable(TableName.valueOf(tableName));
Result r = table.get(get);
System.out.println("GET result :");
    for (Cell c : r.rawCells()) {
        System.out.println("Family : " + new String(CellUtil.cloneFamily(c)));
        System.out.println("Column Qualifier : " + new String(CellUtil.cloneQualifier(c)));
        System.out.println("Value : " + new String(CellUtil.cloneValue(c)));
        System.out.println("----------");
    }

Bon, on peut maintenant récupérer les données de notre tableau, ligne par ligne, mais comment en mettre ? Vous utilisez l’objet Put :

Put put = new Put("my_row_key");
put.addColumn(Bytes.toBytes("myFamily"), Bytes.toBytes("myColumn"), Bytes.toBytes("awesomeValue");
//Add as many columns as you want

Table table = connection.getTable(TableName.valueOf("myTable");
table.put(put);

NB : Table.put peut aussi prendre en paramètre une liste de puts, ce qui est, quand on veut ajouter beaucoup de lignes, bien plus efficace que put par put.

Bon maintenant, je peux mettre des lignes et en récupérer de ma HBase, mais que se passe-t-il si je veux obtenir plusieurs lignes et si je ne connais pas mes row_keys ?

Capitaine ici ! Vous pouvez utiliser l’objet Scan :

Une analyse regarde essentiellement toutes les lignes et les récupère, vous pouvez y ajouter plusieurs paramètres, tels que des filtres et une ligne de début/fin, mais nous verrons cela dans [un autre exemple] (https://www.wikiod.com/fr/hbase/utilisation-de-lapi-java /using-the-java-api/15541/using-the-scan-filters#t=201607281342145028956).

Si vous souhaitez analyser toutes les valeurs de colonne de votre table, pour une colonne donnée, utilisez les lignes suivantes :

Table table = this.connection.getTable(TableName.valueOf("myTable"));
Scan scan = new Scan();
scan.addColumn(Bytes.toBytes("myFamily"), Bytes.toBytes("myColumn"));
ResultScanner rs = table.getScanner(scan);
    try {
        for (Result r = rs.next(); r != null; r = rs.next()) {
            byte[] value = r.getValue(Bytes.toBytes("myFamily"), Bytes.toBytes("myCOlumn"));
            String valueStr = Bytes.toString(value);
            System.out.println("row key "+new String(r.getRow()));
            System.out.println("Scan result :" + valueStr);
        }
    } finally {
        rs.close(); // always close the ResultScanner!
   }

Je tiens vraiment à insister sur le fait que il faut toujours fermer le ResultScanner (même chose que n’importe quel ResultSet d’une base de données d’ailleurs)

Presque fini ! Voyons maintenant comment supprimer une ligne. Vous avez un objet Delete pour cela :

Table table = this.connection.getTable(TableName.valueOf("myTable"));
Delete d = new Delete(Bytes.toBytes("my_weird_key"));
table.delete(d);
System.out.prinln("Row " + row_key + " from table " + tableName + " deleted");

Une dernière chose : avant d’exécuter l’une des opérations, vérifiez toujours que la table existe, sinon vous obtiendrez une exception.

C’est tout pour l’instant, vous pouvez gérer vos données dans HBase avec cet exemple.

Création et suppression de tables #

Dans HBase, les données sont stockées dans des tables avec des colonnes. Les colonnes sont regroupées en familles de colonnes, qui peuvent être par exemple “personnelles” ou “professionnelles”, chacune d’elles contenant des informations spécifiques.

Pour créer une table, vous devez utiliser l’Objet Admin, créez-le en utilisant :

Admin admin = connection.getAdmin();

Une fois que vous avez cet administrateur, vous pouvez commencer à créer des tableaux. Assurez-vous d’abord que cette table n’existe pas déjà avec la ligne

admin.tableExists(TableName.valueOf("myTable);

Cette méthode renverra true si la table existe. Lorsque vous avez coché cette case, vous pouvez créer votre tableau en utilisant les lignes

HTableDescriptor descriptor = new HTableDescriptor(TableName.valueOf("myTable"));
descriptor.addFamily(new HColumnDescriptor("myFamily"));
admin.createTable(descriptor);

Vous devez définir au moins une famille pour la table, et le livre de référence HBase recommande de ne pas dépasser 3 familles de colonnes, sinon vous perdrez des performances.

Toutes nos félicitations ! Votre tableau est créé !

Si vous devez supprimer votre table, vous pouvez utiliser

this.admin.disableTable(TableName.valueOf(tableName));
this.admin.deleteTable(TableName.valueOf(tableName));

Assurez-vous de toujours désactiver la table en premier !

Vous savez maintenant comment gérer les tables dans HBase.

Utilisation des filtres d’analyse #

Fondamentalement, l’objet Scan récupère toutes les lignes de la table, mais que se passe-t-il si vous souhaitez récupérer uniquement les lignes où la valeur d’une colonne donnée est égale à quelque chose ? Laissez-moi vous présenter les Filtres, ils fonctionnent comme le WHERE en SQL.

Avant de commencer à utiliser les filtres, si vous savez comment vos row_keys sont stockées, vous pouvez définir une ligne de début et une ligne de fin pour votre Scan, ce qui optimisera votre requête.

Dans HBase, les row_keys sont stockées dans l’ordre lexicographique, mais vous pouvez toujours utiliser le salage pour changer la façon dont il est stocké, je n’expliquerai pas le salage dans ce sujet, cela prendrait trop de temps et ce n’est pas le sujet.

Revenons à nos limites de ligne, vous avez deux méthodes à utiliser pour définir la ligne de début et de fin

Scan scan = new Scan();
scan.setStartRow(Bytes.toBytes("row_10"));
scan.setStopRow(Bytes.toBytes("row_42"));

Cela changera le comportement de votre scanner pour récupérer toutes les lignes entre “row_10” et “row_42”.

NB : Comme dans la plupart des méthodes “sub” (par exemple substring), le startRow est inclusif et le stopRow est exclusif.

Maintenant que nous pouvons lier notre Scan, nous devons maintenant ajouter quelques filtres à nos scans, il y en a beaucoup, mais nous allons voir ici les plus importants.

  • Si vous souhaitez récupérer toutes les lignes ayant une row_key commençant par un motif donné

Utilisez le RowPrefixFilter :

Scan scan = new Scan();
scan.setRowPrefixFilter(Bytes.toBytes("hello"));

Avec ce code, votre scan ne récupérera que les lignes ayant un row_key commençant par “hello”.

  • Si vous souhaitez récupérer toutes les lignes où la valeur d’une colonne donnée est égale à quelque chose

Utilisez le SingleColumnValueFilter :

Scan scan = new Scan();
SingleColumnValueFilter filter = new SingleColumnValueFilter(Bytes.toBytes("myFamily"),Bytes.toBytes("myColumn"), CompareOp.EQUAL, Bytes.toBytes("42"));
scan.setFilter(filter);

Avec ce code, vous obtiendrez toutes les lignes où la valeur de la colonne myColumn est égale à 42. Vous avez différentes valeurs pour CompareOp qui sont expliquées dans la section Paramètres.

-Bien, mais que se passe-t-il si je veux utiliser des expressions régulières

Utilisez le filtre RegexStringComparator :

Scan scan = new Scan();
RegexStringComparator comparator = new RegexStringComparator(".hello.");
SingleColumnValueFilter filter = new SingleColumnValueFilter(Bytes.toBytes("myFamily"),Bytes.toBytes("myColumn"), CompareOp.EQUAL, comparator);
scan.setFilter(filter);

Et vous obtiendrez toutes les lignes où la colonne myColumn contient hello.

Veuillez également noter que la méthode Scan.setFilter() peut également prendre une liste de Filter comme paramètres

Connexion à HBase #

Si vous souhaitez vous connecter à un serveur HBase, vous devez d’abord vous assurer que l’IP du serveur est dans votre fichier /etc/hosts par exemple ajouter la ligne

255.255.255.255    hbase

Ensuite, vous pouvez utiliser l’API Java pour vous connecter à zookeeper, vous n’avez qu’à spécifier le port client et l’adresse du zookeeper

Configuration config =  HBaseConfiguration.create();
config.set("hbase.zookeeper.quorum", "hbase");
config.set("hbase.zookeeper.property.clientPort","2181");

Après avoir configuré la connexion, vous pouvez la tester à l’aide de

HBaseAdmin.checkHBaseAvailable(config);

Si vous rencontrez un problème avec votre configuration HBase, une exception sera levée.

Enfin pour se connecter au serveur, il suffit d’utiliser

Connection connection = ConnectionFactory.createConnection(config);
