Installation de HBase en mode autonome #

HBase Standalone est un mode qui vous permet de vous débarrasser de HDFS et de tester HBase avant de déployer dans un cluster, Il n’est pas orienté production.

L’installation de HBase en mode autonome est extrêmement simple. Vous devez d’abord télécharger l’archive HBase nommée hbase-X.X.X-bin.tar.gz disponible sur l’un des [miroirs apache][1].

Une fois que vous avez fait cela, exécutez cette commande shell

tar xzvf hbase-X.X.X-bin.tar.gz

Il exportera l’archive dans votre répertoire, vous pourrez la mettre où vous voulez.

Maintenant, allez dans le répertoire HBase que vous avez exporté et éditez le fichier conf/hbase-env.sh

cd hbase-X.X.X
vi -o conf/hbase-env.xml

Dans ce fichier, décommentez la ligne et modifiez le chemin de JAVA_HOME

JAVA_HOME=/usr    #The directory must contain bin/java

Presque là ! maintenant éditez le fichier conf/hbase-sitexml et mettez les lignes suivantes

<configuration>
  <property>
    <name>hbase.rootdir</name>
    <value>file:///home/user/hbase</value>
  </property>
  <property>
    <name>hbase.zookeeper.property.dataDir</name>
    <value>/home/user/zookeeper</value>
  </property>
</configuration>

Vous pouvez placer ces répertoires où vous voulez, assurez-vous simplement de vous en souvenir si vous voulez vérifier les journaux, etc.

Votre HBase est maintenant prêt à fonctionner ! Exécutez simplement la commande

bin/start-hbase.sh

et si vous voulez arrêter HBase

bin/stop-hbase.sh

Maintenant, votre HBase est lancé sur votre hôte local et vous pouvez y accéder (en utilisant l’API Java ou le shell HBase). Pour exécuter le shell HBase, utilisez

bin/hbase shell

Amusez-vous avec HBase !

[1] : http://www.apache.org/dyn/closer.cgi/hbase/
