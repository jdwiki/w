Installation et configuration sous Windows et Linux #

Télécharger Qt pour la version Linux Open Source

Allez sur https://www.qt.io/download-open-source/ et cliquez sur Télécharger maintenant, assurez-vous que vous téléchargez le programme d’installation de Qt pour Linux.

[![Page de téléchargement open source de Qt Linux][3]][3]

Un fichier avec le nom qt-unified-linux-x-online.run sera téléchargé, puis ajoutez l’autorisation exec

chmod +x qt-unified-linux-x-online.run

N’oubliez pas de changer ‘x’ pour la version actuelle du programme d’installation. Lancez ensuite le programme d’installation

./qt-unified-linux-x-online.run

Télécharger Qt pour la version open source de Windows

Allez sur https://www.qt.io/download-open-source/. La capture d’écran suivante montre la page de téléchargement sous Windows :

[![Page de téléchargement de Qt Windows open source][1]][1]

Ce que vous devez faire maintenant dépend de l’IDE que vous allez utiliser. Si vous allez utiliser Qt Creator, qui est inclus dans le programme d’installation, cliquez simplement sur Télécharger maintenant et lancez l’exécutable.

Si vous allez utiliser Qt dans Visual Studio, normalement le bouton Télécharger maintenant devrait également fonctionner. Assurez-vous que le fichier téléchargé s’appelle qt-opensource-windows-x86-msvc2015_64-x.x.x.exe ou qt-opensource-windows-x86-msvc2015_32-x.x.x.exe (où x.x.x est la version de Qt, par exemple 5.7.0). Si ce n’est pas le cas, cliquez sur Afficher tous les téléchargements et sélectionnez l’une des quatre premières options sous Hôte Windows.

Si vous comptez utiliser Qt dans Code :: Blocks, cliquez sur Afficher tous les téléchargements et sélectionnez Qt x.x.x pour Windows 32 bits (MinGW x.x.x, 1,2 Go) sous Windows Host.

Une fois que vous avez téléchargé le fichier d’installation approprié, lancez l’exécutable et suivez les instructions ci-dessous. Notez que vous devez être administrateur pour installer Qt. Si vous n’êtes pas administrateur, vous pouvez trouver plusieurs solutions alternatives [ici][2].

Installez Qt dans n’importe quel système d’exploitation

Une fois que vous avez téléchargé Qt et ouvert le programme d’installation, la procédure d’installation est la même pour tous les systèmes d’exploitation, bien que les captures d’écran puissent être un peu différentes. Les captures d’écran fournies ici proviennent de Linux.

Connectez-vous avec un compte Qt existant ou créez-en un nouveau :

[![Installateur Qt][4]][4]

Sélectionnez un chemin pour installer les bibliothèques et outils Qt

[![chemin d’installation][5]][5]

Sélectionnez la version de la bibliothèque et les fonctionnalités souhaitées

[![Caractéristiques de Qt][6]][6]

Une fois le téléchargement terminé et l’installation terminée, accédez au répertoire d’installation de Qt et lancez Qt Creator ou exécutez-le directement depuis la ligne de commande.

[![entrez la description de l’image ici][7]][7]

Bonjour le monde #

Dans cet exemple, nous créons et affichons simplement un bouton poussoir dans un cadre de fenêtre sur le bureau. Le bouton poussoir portera l’étiquette “Hello world!”

Cela représente le programme Qt le plus simple possible.

Tout d’abord, nous avons besoin d’un fichier de projet :

helloworld.pro

QT       += core gui

greaterThan(QT_MAJOR_VERSION, 4): QT += widgets

TARGET = helloworld
TEMPLATE = app

SOURCES += main.cpp
  • QT est utilisé pour indiquer quelles bibliothèques (modules Qt) sont utilisées dans ce projet. Puisque notre première application est une petite interface graphique, nous aurons besoin de QtCore et QtGui. Comme Qt5 sépare QtWidgets de QtGui, nous devons ajouter la ligne greaterThan afin de le compiler avec Qt5.
  • TARGET est le nom de l’application ou de la bibliothèque.
  • TEMPLATE décrit le type à construire. Il peut s’agir d’une application (app), d’une bibliothèque (lib) ou simplement de sous-répertoires (subdirs).
  • SOURCES est une liste de fichiers de code source à utiliser lors de la construction du projet.

Nous avons également besoin du main.cpp contenant une application Qt :

main.cpp

#include <QApplication>
#include <QPushButton>

int main(int argc, char *argv[])
{
    QApplication a(argc, argv);

    QPushButton button ("Hello world!");
    button.show();

    return a.exec(); // .exec starts QApplication and related GUI, this line starts 'event loop'    
}
  • Objet QApplication. Cet objet gère les ressources à l’échelle de l’application et est nécessaire pour exécuter tout programme Qt doté d’une interface graphique. Il a besoin de argv et args car Qt accepte quelques arguments de ligne de commande. Lors de l’appel de a.exec(), la boucle d’événements Qt est lancée.
  • Objet QPushButton. Le bouton poussoir avec l’étiquette Hello world!. La ligne suivante, button.show(), affiche le bouton poussoir à l’écran dans son propre cadre de fenêtre.

Enfin, pour exécuter l’application, ouvrez une invite de commande et entrez le répertoire dans lequel se trouve le fichier .cpp du programme. Tapez les commandes shell suivantes pour générer le programme.

qmake -project
qmake
make

Application de base avec QtCreator et QtDesigner #

QtCreator est, à l’heure actuelle, le meilleur outil pour créer une application Qt. Dans cet exemple, nous allons voir comment créer une application Qt simple qui gère un bouton et écrit du texte.

Pour créer une nouvelle application cliquez sur File->New File or Project :

[![entrez la description de l’image ici][1]][1]

Ensuite, choisissez l’application Projets-> Application-> Qt Widgets

[![entrez la description de l’image ici][2]][2]

Ensuite, vous pouvez choisir le nom et le chemin de votre projet :

[![entrez la description de l’image ici][3]][3]

Ensuite, vous pouvez choisir les kits que vous utiliserez. Si vous n’avez pas de kit, QtCreator créera un kit avec votre version de Qt et le compilateur principal de votre ordinateur. Si vous n’avez pas de compilateur, vous pouvez en installer un. Sous Windows : installez Visual Studio. Sous Linux/Mac : installez g++ ou clang++.

[![entrez la description de l’image ici][4]][4]

Ensuite vous pouvez choisir le nom de votre classe de fenêtre principale, la classe héritée, le nom du fichier correspondant à votre classe de fenêtre principale. Si vous apprenez ou testez Qt, vous n’avez pas vraiment besoin de les changer.

[![entrez la description de l’image ici][5]][5]

La dernière étape peut être de choisir un sous-projet de ce projet et d’ajouter un contrôle de version tel que git et svn. Encore une fois, si ce n’est que pour les tests, vous n’avez pas besoin de les modifier.

[![entrez la description de l’image ici][6]][6]

Cliquez ensuite sur le bouton Terminer. Maintenant, vous devriez être ici :

[![entrez la description de l’image ici][7]][7]

C’est la base de votre application. si vous l’exécutez maintenant en cliquant sur Build->Run ou ctrl+R (par défaut), vous verrez une fenêtre vide.

Nous allons maintenant ajouter un texte et un bouton. pour ce faire, nous utiliserons Qt Designer. Double-cliquez sur mainwindow.ui Alors maintenant, vous devriez voir : (si ce n’est pas le cas et que vous voyez un fichier xml, cliquez sur le bouton Conception à gauche)

[![entrez la description de l’image ici][8]][8]

Ici Qt Designer ! Semble assez compliqué. Mais une fois qu’on s’y est habitué, c’est vraiment génial. Nous allons ajouter du texte et un bouton. Sur la gauche, il y a la liste des articles. Vous pouvez cliquer dessus et glisser-déposer les objets. Cliquez sur le bouton poussoir et déposez-le dans la fenêtre. Recherchez ensuite le Label, et faites de même (vous avez un filtre en haut à gauche où vous pouvez écrire l’objet que vous recherchez).

Vous devriez avoir quelque chose comme ça maintenant :

[![entrez la description de l’image ici][9]][9]

En double-cliquant sur l’objet, vous pouvez modifier le texte dessus. Ou vous pouvez voir en bas à droite les propriétés de l’objet que vous êtes maintenant et trouver la propriété text. Ici, vous pouvez également modifier le nom.

Maintenant, si vous enregistrez et exécutez (mieux vaut cliquer sur le bouton d’édition puis enregistrer à nouveau pour être sûr que vos modifications ont été enregistrées), vous obtenez :

[![entrez la description de l’image ici][10]][10]

Hein? Pourquoi mon étiquette et mon bouton sont comme ça quand je cours ? C’est parce qu’il n’y a pas de mise en page dans notre objet central. Au fait, si vous redimensionnez votre fenêtre principale, vous pouvez voir que les objets gardent leur place. Donc, pour le réparer, nous allons ajouter une mise en page. Disons une disposition verticale. Faites donc glisser et déposez une mise en page verticale à partir de la liste d’objets à gauche. Vous devriez maintenant voir :

[![entrez la description de l’image ici][11]][11]

Une mise en page flottante.

Alors maintenant, faites un clic droit sur la fenêtre principale, n’importe où sauf sur l’étiquette et le bouton. c\Cliquez sur Disposition->Disposer verticalement. Vous devriez maintenant voir que vos objets sont alignés verticalement dans votre fenêtre. Alors maintenant, déplacez (avec glisser-déposer à nouveau) votre étiquette et votre bouton dans la mise en page. maintenant vous devriez obtenir :

[![entrez la description de l’image ici][12]][12]

Dans votre concepteur. Et si vous lancez votre application :

[![entrez la description de l’image ici][13]][13]

Ici, vous pouvez voir votre application avec l’étiquette et le bouton. Et si vous redimensionnez votre fenêtre, l’étiquette et le bouton se redimensionnent également.

Mais notre bouton ne fait toujours rien. Nous pouvons le changer de 2 manières différentes. La première consiste à connecter le bouton avec une méthode que nous avons créée. Nous pouvons le faire avec le nom de méthode connect. Revenons donc à notre code et allons à mainwindow.cpp ajoutez maintenant :

connect(ui->pushButton, SIGNAL(clicked(bool)), this, SLOT(whenButtonIsClicked()));

Dans le constructeur de votre MainWindow APRES le ui->setupUI(this); qui initialise le ui.

Ensuite, nous pouvons créer le MainWindow::whenButtonIsClicked() dans notre classe .cpp qui pourrait changer le texte de l’étiquette comme ça :

void MainWindow::whenButtonIsClicked()
{
    ui->label->setText("the button has been clicked !");
}

Et dans notre mainwindow.h, nous devons ajouter :

public slots:
    void whenButtonIsClicked();

Les créneaux publics signifient que cette méthode peut être appelée lorsqu’un signal est reçu. connect relie le signal lorsque l’on clique sur le bouton et une méthode à appeler.

Alors maintenant, si nous lançons notre application et cliquons sur le bouton, nous obtenons :

[![entrez la description de l’image ici][14]][14]

Ce qui signifie que notre connexion fonctionne. Mais avec Qt Designer, nous avons un moyen encore plus simple de le faire. Si vous voulez faire dans l’autre sens, supprimez la connexion pour déconnecter le bouton (car nous le connecterons différemment), revenez à mainwindow.ui et faites un clic droit sur le bouton. Cliquez sur Go to slot… , sélectionnez clicked() et appuyez sur ok.

[![entrez la description de l’image ici][15]][15]

Ensuite, vous devriez être déplacé vers cette fonction :

void MainWindow::on_pushButton_clicked()
{
}

C’est la fonction qui sera appelée lorsque vous cliquerez sur le bouton. Vous pouvez donc ajouter

ui->label->setText("it's even easier !");

Dans ça. Allez dans mainwindow.h pour enregistrer la modification (lorsque vous faites un aller à l’emplacement, cela crée une méthode liée au signal que vous avez demandé. Il définit la fonction dans le .h mais ne l’enregistre pas. Donc vous devriez aller dans le fichier et enregistrez-le).

Et maintenant, lorsque vous exécutez votre application et que vous appuyez sur le bouton, vous pouvez voir le nouveau message (si vous voyez toujours l’ancien, c’est que vous n’avez pas supprimé la connexion).

[![entrez la description de l’image ici][16]][16]

Nous pouvons également ajouter un int, double, etc dans notre label grâce au QVariant qui est une classe géniale qui peut convertir beaucoup de choses en beaucoup d’autres choses. Donc, à gauche, ajoutez un int qui augmente lorsque nous appuyons sur le bouton.

Donc le .h :

#ifndef MAINWINDOW_H
#define MAINWINDOW_H

#include <QMainWindow>

namespace Ui {
class MainWindow;
}

class MainWindow : public QMainWindow
{
    Q_OBJECT

public:
    explicit MainWindow(QWidget *parent = 0);
    ~MainWindow();

public slots:
    void whenButtonIsClicked();

private slots:
    void on_pushButton_clicked();

private:
    Ui::MainWindow  *ui;
    double          _smallCounter;
};

#endif // MAINWINDOW_H

Le .cpp :

#include "mainwindow.h"
#include "ui_mainwindow.h"

MainWindow::MainWindow(QWidget *parent) :
    QMainWindow(parent),
    ui(new Ui::MainWindow)
{
    ui->setupUi(this);
//    connect(ui->pushButton, SIGNAL(clicked(bool)), this, SLOT(whenButtonIsClicked()));
    _smallCounter = 0.0f;
}

MainWindow::~MainWindow()
{
    delete ui;
}

void MainWindow::whenButtonIsClicked()
{
    ui->label->setText("the button has been clicked !");
}

void MainWindow::on_pushButton_clicked()
{
    _smallCounter += 0.5f;
    ui->label->setText("it's even easier ! " + QVariant(_smallCounter).toString());
}

Et maintenant, nous pouvons enregistrer et exécuter à nouveau. Chaque fois que vous cliquez sur le bouton, il affiche “c’est encore plus simple !” avec la valeur de _smallCounter. Vous devriez donc avoir quelque chose comme :

[![entrez la description de l’image ici][17]][17]
