Les sections sont une fonctionnalité qui permet aux propriétaires de magasins d’ajouter, de modifier, de supprimer et de réorganiser facilement le contenu d’une page. Il existe 2 types de sections : dynamiques et fixes. Les sections dynamiques peuvent être réordonnées dans leur intégralité avec d’autres sections de la page (page d’accueil). Les sections fixes ne peuvent pas être réorganisées, mais le contenu qu’elles contiennent peut toujours être ajouté, modifié, supprimé et réorganisé.

Choses à retenir: #

  • Le schéma d’une section est limité à cette section particulière. Cela signifie que vous pouvez donner à vos variables des noms relativement simples sans vous soucier de leur conflit avec les noms de variables d’autres sections de la même page.

Davantage de ressources: #

  • https://help.shopify.com/themes/development/theme-editor/sections
  • https://help.shopify.com/themes/development/theme-editor/settings-schema

Inclure une section sur une page #

L’inclusion d’une section statique sur une page est similaire à l’inclusion d’un extrait de code. La seule différence est qu’au lieu d’utiliser le mot « inclure », vous utilisez le mot « section ».

{% section 'section-file-name' %}

L’utilisation de cette balise “codera en dur” la section dans votre thème à l’endroit où vous avez mis le code et ne pourra pas être déplacée ou supprimée de la page.

Pour utiliser les sections dynamiques, vous devez utiliser le contrôle “preset” au bas des paramètres de schéma pour cette section.

{% schema %}
  {
    "presets": [
      {
        "category": "Custom Content",
        "name": "Text",
        "settings": {
          "heading": "Hello World"
        },
        "blocks": [
          {
            "type": "text",
            "settings": {
              "content": "Once upon a time..."
            }
          }
        ]
      }
    ]
  }
{% endschema %}

En utilisant des préréglages, cela vous permettra de déplacer et d’ajouter votre section autour de votre page selon vos besoins. Il vous permet également de remplir la section avec du contenu factice afin que vous puissiez voir exactement à quoi ressemblera la section sur votre page sans avoir à l’ajouter à la page.

Dans le schéma du fichier #

Pour les sections, au lieu d’utiliser un gros fichier JSON (comme settings_schema.json, ils conservent plutôt leur schéma dans le fichier de la section qui l’utilise. Pour ce faire, {% schema %} {% endschema %} sont utilisés et JSON est placé entre eux. À partir de là, le format est similaire à settings_schema.json.

{% schema %}
  {
    "name": "Header Banner",
    "settings": [
      {
        "type": "checkbox",
        "id": "banner_enable",
        "label": "Enable Banner",
        "default": false
      },
      {
        "type": "color",
        "id": "banner_color",
        "label": "Banner Background Color",
        "default": "#000000"
      },
      {
        "type": "color",
        "id": "text_color",
        "label": "Banner Text Color",
        "default": "#ffffff"
      },
      {
        "type": "text",
        "id": "banner_text",
        "label": "Banner Text",
        "default": "Welcome to my Section"
      }
    ]
  }
{% endschema %}

Pour l’utiliser dans votre code HTML/Liquid, vous tireriez de section.settings. Quelque chose comme:

{% if section.settings.banner_enable %}
  <div class="banner">
    <p>{{ section.settings.banner_text }}</p>
  </div>
{% endif %}

Utilisation de blocs pour les éléments récurrents #

Les blocs sont l’une des choses les plus utiles qui accompagnent les sections. Les blocs sont essentiellement un plan pour quelque chose qui peut être créé un nombre illimité de fois. L’un des meilleurs exemples est les diapositives d’un curseur. Un bloc est un élément de niveau supérieur dans le schéma, ce qui signifie qu’il se trouve à côté d’éléments tels que name et settings, mais pas à l’intérieur d’eux. Dans les blocs se trouvent essentiellement des mini-schémas.

{% schema %}
  {
    "name": "Slider",
    "max_blocks": 6,
    "blocks": [
      {
        "type": "slide",
        "name": "Slide",
        "limit": 4,
        "settings": [
          {
            "id": "image",
            "type": "image_picker",
            "label": "Image"
          }
        ]
      }
    ]
  }
{% endschema %}

Notez que la diapositive a également un type de “diapositive”. Vous pouvez donner à vos blocs leur propre type qui n’est pas l’un des types fournis par Shopify.

Pour l’utiliser, vous feriez simplement une boucle sur chaque bloc de votre code HTML/Liquid.

<div class="slider">
  {% for block in section.blocks %}
    <img src="{{ block.settings.image | img_url: 'master' }}" alt="" />
  {% endfor %}
</div>

Utilisation de JavaScript et de feuilles de style dans les sections #

Chaque section peut contenir du JavaScript et des feuilles de style, ces deux langages sont utilisés dans les balises liquides : {% javascript %}{% endjavascript %} & {% stylesheet %}{% endstylesheet %}.

Lorsque vous placez du code à l’intérieur de ces deux balises, Shopify compile chaque élément dans shopify_compiled.js & shopify_compiled.css. Cela permet une plus grande lisibilité dans le code de chaque section, car vous n’avez pas à parcourir un long document pour trouver chaque élément.

Si vous ne souhaitez pas utiliser CSS, vous pouvez également utiliser SCSS en utilisant {% stylesheet 'scss' %} à la place.

Il y a cependant des inconvénients à les utiliser. Liquid ne peut être utilisé dans aucun d’entre eux, tout comme il ne peut pas être utilisé dans les fichiers normaux *.js, *.css, *.scss sans l’extension .liquid.

De plus, lorsque vous utilisez la balise de feuille de style SCSS, il ne s’agit pas d’une feuille de style globale, donc toutes les variables définies dans le document .scss de votre thème ne seront pas accessibles et devront être redéfinies.

Exemples d’utilisation

JavaScript

{% javascript %}
    $( "p" ).click(function() {
      $( this ).slideUp();
    });
{% endjavascript %}

CSS

{% stylesheet %}
    .container {
        width: 100%;
    }
    .container p {
        color: #ff0000;
    }
{% endstylesheet %}

TOUPET

{% stylesheet 'scss' %}
    $red: #ff0000;
    .container {
        width: 100%;
        p {
            color: $red;
        }
    }
{% endstylesheet %}
