Travailler avec angular-cli Générer des composants, des directives, des tuyaux, des services, etc.

L’outil angular-cli peut vous aider à échafauder différentes parties d’une application angulaire (composants, directives, tuyaux, services, classes, gardes, interfaces, énumérations et modules).

Syntaxe #

  • ng génère [composant | directive | services | tuyau | classe | énumération | interface | garde | module] [nom] [drapeaux…]
  • ng g [c | d | s | p | cl | e | je | g | m] [nom] [drapeaux…]

Paramètres #

Paramètre La description
composant ou c Utilisé pour générer un composant
directive ou d Utilisé pour générer des directives
service ou s Utilisé pour générer des services
tuyau ou p Utilisé pour générer des tuyaux
classe ou cl Utilisé pour générer des classes
énumération ou e Utilisé pour générer des énumérations
interfaces ou i Utilisé pour générer des interfaces
garde ou g Utilisé pour générer des gardes
module ou m Utilisé pour générer des modules
--flat ou -f Utilisé pour activer/désactiver la création de répertoire
--inline-template ou -it Utilisé pour activer/désactiver les modèles HTML en ligne dans les composants
--inline-style ou -is Utilisé pour activer/désactiver les styles en ligne dans les composants
--prefix ou -p Utilisé pour désactiver ou modifier le préfixe
--spec ou -s Utilisé pour activer/désactiver la création de fichiers .spec
--skip-import Utilisé pour ignorer l’import du module
--app ou -a Utilisé pour spécifier le nom de l’application à utiliser
--module ou -m Utilisé pour spécifier le module déclarant
--view-encapsulation ou -ve Utilisé pour spécifier la stratégie d’encapsulation de la vue dans les composants
--change-detection ou -cd Utilisé pour spécifier la stratégie de détection des changements dans les composants
--routing ou -r Utilisé pour spécifier si le fichier du module de routage doit être généré

Utilisation de “Générer une commande” #

Vous pouvez utiliser la commande ng generate ou ng g pour générer des blocs de construction angulaires (composants, services, tuyaux, etc.).

Vous pouvez trouver tous les plans possibles dans le tableau ci-dessous :

Échafaudage Utilisation Raccourci
Composant de générer le composant nom-composant de g c nom-composant
directive ng generate directive nom-directive ng g d nom_directive
Tuyau ng génère le tuyau nom-du-tuyau ng g p nom-tuyau
Services ng génère le service nom-service nom-service ng g s
Classe ng génère la classe nom-classe ng g cl nom-classe
Garde ng génère garde nom-garde ng g g nom-garde
Interface ng génère l'interface nom-interface ng g i nom-interface
Énumération ng génère enum enum-name ng g e enum-name
module du module de génération nom-module de g m nom-module

Ainsi, par exemple, si vous exécutez ng generate component user-list - angular-cli :

  • créez le répertoire user-list dans le dossier src/app ou le dossier où vous avez exécuté la commande.
  • dans ce répertoire générer 4 fichiers (user-list.component.ts, user-list.component.html, user-list.component.css et user-list.component.spec.ts)
  • ajouter user-list comme déclaration dans le décorateur @NgModule du module le plus proche.

Génération de services #

Pour ajouter un service avec le nom UserService, exécutez :

  $ ng g s user

  installing service
    create src/app/user.service.spec.ts
    create src/app/user.service.ts
  • Pour empêcher la création de fichiers .spec, ajoutez le drapeau --spec false ou -sp false
  $ ng g s user --spec false
  
  installing service
    create src/app/user.service.ts
  • Pour activer la création de dossier, ajoutez le drapeau --flat false ou -f false
  $ ng g s user --flat false

  installing service
    create src/app/user/user.service.spec.ts
    create src/app/user/user.service.ts

Vous pouvez également combiner les drapeaux énumérés ci-dessus. Par exemple, pour créer uniquement le fichier user.service.ts dans le dossier user sans le fichier .spec, utilisez la commande suivante.

  $ ng g s user -f false -sp false

  installing service
    create src/app/user/user.service.ts

Tous les indicateurs “générer un service” :

Descriptif Drapeau Raccourci Valeur par défaut
Activer la création de dossier --plat faux -f faux “vrai”
Empêcher la création de fichiers .spec --spec faux -sp faux “vrai”

Génération de modules #

Pour ajouter un module appelé GuestModule, exécutez :

  $ ng g m guest

  installing module
    create src/app/guest/guest.module.ts
  • Pour activer la création de fichiers .spec, ajoutez le drapeau --spec ou -sp
  $ ng g m guest --spec

  installing module
    create src/app/guest/guest.module.spec.ts
    create src/app/guest/guest.module.ts
  • Pour activer le routage, ajoutez le drapeau --routing ou -r
  $ ng g m guest --routing

  installing module
    create src/app/guest/guest-routing.module.ts
    create src/app/guest/guest.module.ts

Vous pouvez également combiner les drapeaux énumérés ci-dessus. Par exemple, pour créer un module avec routage et spécifications, utilisez la commande suivante.

  $ ng g m guest -sp -r
  
  installing module
    create src/app/guest/guest-routing.module.ts
    create src/app/guest/guest.module.spec.ts
    create src/app/guest/guest.module.ts

Tous les indicateurs generate module :

Descriptif Drapeau Raccourci Valeur par défaut
Activer la création de fichiers .spec --spec -sp “faux”
Activer le routage --routing -r “faux”

Génération de composants #

Pour ajouter un composant avec un sélecteur [prefix]-user-list, exécutez :

  $ ng g c user-list
  
  installing component
    create src/app/user-list/user-list.component.css
    create src/app/user-list/user-list.component.html
    create src/app/user-list/user-list.component.spec.ts
    create src/app/user-list/user-list.component.ts
    update src/app/app.module.ts

préfixe empêche les collisions de noms d’éléments avec des composants d’autres applications et avec des éléments HTML natifs. Ainsi, par exemple, si le préfixe est app - le composant généré aura le sélecteur app-user-list.

  • Pour empêcher l’utilisation du préfixe, ajoutez le drapeau --prefix false ou -p false
  $ ng g c user-list --prefix false

import { Component } from '@angular/core';

@Component({
  selector: 'user-list',
  templateUrl: './user-list.component.html',
  styleUrls: ['./user-list.component.css']
})
export class UserListComponent {}
  • Pour empêcher la création de fichiers .spec, ajoutez le drapeau --spec false ou -sp false
  $ ng g c user-list --spec false

  installing component
    create src/app/user-list/user-list.component.css
    create src/app/user-list/user-list.component.html
    create src/app/user-list/user-list.component.ts
    update src/app/app.module.ts
  • Pour utiliser des modèles html intégrés au lieu de modèles externes, ajoutez le drapeau --inline-template ou -it
  $ ng g c user-list --inline-template

  installing component
    create src/app/user-list/user-list.component.css
    create src/app/user-list/user-list.component.spec.ts
    create src/app/user-list/user-list.component.ts
    update src/app/app.module.ts
  • Pour utiliser des styles en ligne au lieu de styles externes, ajoutez le drapeau --inline-style ou -is
  $ ng g c user-list --inline-style

  installing component
    create src/app/user-list/user-list.component.html
    create src/app/user-list/user-list.component.spec.ts
    create src/app/user-list/user-list.component.ts
    update src/app/app.module.ts
  • Pour empêcher la création de dossier, ajoutez le drapeau --flat ou -f
  $ ng g c user-list --flat
  
  installing component
    create src/app/user-list.component.css
    create src/app/user-list.component.html
    create src/app/user-list.component.spec.ts
    create src/app/user-list.component.ts
    update src/app/app.module.ts

Vous pouvez également combiner les drapeaux énumérés ci-dessus. Par exemple, pour créer uniquement le fichier .component.ts sans les fichiers et dossiers .css, .html, .spec, utilisez la commande suivante.

  $ ng g c user-list -f -it -is -sp false
  
  installing component
    create src/app/user-list.component.ts
    update src/app/app.module.ts

Tous les indicateurs generate component :

Descriptif Drapeau Raccourci Valeur par défaut
Empêcher la création de dossier --plat -f “faux”
Empêcher l’utilisation du préfixe --préfixe faux -p faux “vrai”
Empêcher la création de fichiers .spec --spec faux -sp faux “vrai”
Activer les modèles HTML intégrés --modèle-en-ligne -il “faux”
Activer les styles en ligne --style-en-ligne -est “faux”
