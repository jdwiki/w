Sur cette page

  • Exemple d’un WebBrowser dans un BusyIndicator

Cela vous permet de mettre un navigateur Web dans votre application WPF.

Un point clé à noter, qui n’est pas évident d’après la documentation, et vous pourriez passer des années sans le savoir, c’est qu’il se comporte par défaut comme InternetExplorer7, plutôt que comme votre installation InternetExplorer la plus récente (voir https://weblog. west-wind.com/posts/2011/may/21/web-browser-control-specifying-the-ie-version ).

Cela ne peut pas être résolu en définissant une propriété sur le contrôle ; vous devez soit modifier les pages affichées en ajoutant une balise Meta HTML, soit en appliquant un paramètre de registre (!). (Les détails des deux approches sont sur le lien ci-dessus.)

Par exemple, ce comportement de conception bizarre peut vous amener à recevoir un message indiquant “Erreur de script”/“Une erreur s’est produite dans le script de cette page”. Googler cette erreur peut vous faire penser que la solution consiste à essayer de supprimer l’erreur, plutôt que de comprendre le problème réel et d’appliquer la bonne solution.

Exemple d’un WebBrowser dans un BusyIndicator #

Sachez que le contrôle WebBrowser n’est pas compatible avec votre définition XAML et se rend au-dessus d’autres choses. Par exemple, si vous le mettez à l’intérieur d’un BusyIndicator qui a été marqué comme étant occupé, il s’affichera toujours au-dessus de ce contrôle. La solution consiste à lier la visibilité du WebBrowser à la valeur utilisée par BusyIndicator et à utiliser un convertisseur pour inverser le booléen et le convertir en une visibilité. Par exemple:

    <telerik:RadBusyIndicator IsBusy="{Binding IsBusy}">
        <WebBrowser Visibility="{Binding IsBusy, Converter={StaticResource InvertBooleanToVisibilityConverter}}"/>
    </telerik:RadBusyIndicator>
