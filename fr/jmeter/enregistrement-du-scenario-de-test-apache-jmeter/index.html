L’enregistrement de scénarios de test est l’un des moyens les plus pratiques de créer des scripts de test. En effet, les enregistrements de test vous permettent d’imiter des flux de travail utilisateur réalistes, au lieu d’avoir à créer manuellement un script de test. Les enregistrements capturent toutes les demandes du navigateur à l’application Web, puis créent automatiquement un fichier jmx qui peut être exécuté dans des tests de performances. En utilisant la fonctionnalité d’enregistrement/lecture de JMeter ou des outils tiers comme BlazeMeter et BadBoy, les testeurs peuvent rendre leur travail 3 fois plus rapide.

Enregistrement de script avec JMeter Proxy Recorder #

JMeter vous permet également de configurer manuellement votre espace de travail. C’est plus complexe, mais vous pouvez adapter les scripts à vos besoins exacts.

Configurez le proxy de votre navigateur

  1. Configurez votre navigateur, comme décrit au chapitre 1.

Configurer JMeter

La branche « WorkBench » peut être utilisée comme espace de travail temporaire pour créer des scripts. N’oubliez pas que les entrées ajoutées à cette section ne seront pas enregistrées avec le plan de test. Par conséquent, si vous souhaitez réutiliser la même configuration d’enregistrement à l’avenir, vous devrez la copier et la coller dans la section “Plan de test”.

  1. Ajoutez “Recording Controller” à “WorkBench”: Faites un clic droit sur “WorkBench” -> “Add” -> “Logic Controller” -> “Recording Controller”

[![entrez la description de l’image ici][1]][1]

  1. Ajoutez “HTTP(S) Test Script Recorder” au même “WorkBench”: Faites un clic droit sur “WorkBench” -> “Add” -> “Non-Test Elements” -> “HTTP(S) Test Script Recorder”

[![entrez la description de l’image ici][2]][2]

  1. Sur la page de configuration “HTTP(S) Test Script Recorder” dans “Global Settings: Port”, vous devez mettre le même port que celui spécifié dans la configuration du proxy de votre navigateur, par exemple 8080.

  2. Si vous souhaitez exclure des requêtes vers des ressources spécifiques, vous pouvez utiliser la section « Modèles d’URL à exclure ». Cela peut être utile si vous souhaitez inclure uniquement les types de contenu que vous souhaitez demander (par exemple, *.html, *.php, etc.) ou pour exclure les types de contenu que vous ne souhaitez pas demander (par exemple, *.jpg, * .png, *.js, etc.).

Quand l’utiliserions-nous ? Par exemple, lors de l’enregistrement d’un script qui appelle une application tierce ou lors du test d’un script côté serveur, vous ne souhaiterez peut-être pas télécharger les actifs car ils pourraient encombrer vos tests et consommer de la bande passante, ou vous souhaiterez peut-être enregistrer certaines demandes de un certain chemin.

Les modèles d’exclusion les plus courants sont : “..png”, “..jpg”, “..gif”, “..css”, “..js”. Vous pouvez également combiner différents modèles ensemble. Ce modèle combiné devrait éliminer toutes les requêtes redondantes qui pourraient vous distraire des requêtes importantes : “..(bmp|css|js|gif|ico|jpe?g|png|swf|woff)”

[![entrez la description de l’image ici][3]][3]

  1. D’autre part, vous souhaiterez peut-être imiter le comportement complet du navigateur et inclure le chargement de toutes les ressources. Dans ce cas, il n’est pas nécessaire d’exclure les modèles d’URL. Gardez à l’esprit que le navigateur télécharge toutes les ressources intégrées à partir de la page demandée et qu’il dispose d’un mécanisme de mise en cache, ce qui peut affecter les performances des résultats.

Dans ce cas, il est recommandé de télécharger toutes les ressources embarquées dans le script : Faites un clic droit sur “Plan de test” -> “Ajouter” -> “Elément de configuration” -> “Paramètres par défaut de la requête HTTP” -> “Avancé” -> cochez la case “Récupérer toutes les ressources intégrées”.

  1. Pour que JMeter se comporte davantage comme un vrai navigateur, il est recommandé d’ajouter le “HTTP Cache Manager”, qui vous permet de simuler la fonctionnalité de mise en cache du navigateur dans vos tests de performances. Faites un clic droit sur “Plan de test” -> “Ajouter” -> “Elément de configuration” -> “Gestionnaire de cache HTTP”.

  2. Cliquez maintenant sur le bouton “Démarrer”, qui se trouve en bas de la page “Enregistreur de script de test HTTP (S)”, et parcourez le flux de travail de l’application Web que vous souhaitez tester. Lorsque vous revenez à JMeter, vous devriez voir toutes les requêtes capturées à partir de votre navigateur sous le “contrôleur d’enregistrement”.

[![entrez la description de l’image ici][4]][4]

Enregistrement de script avec la fonctionnalité de modèle JMeter #

Dans la version 2.10, [JMeter][1] a introduit un mécanisme qui vous fait gagner du temps lors de la création de scripts - JMeter Templates. Ces modèles sont des squelettes qui peuvent être réutilisés comme base pour vos nouveaux scripts.

JMeter a déjà plusieurs modèles disponibles avec des descriptions détaillées, et vous pouvez également ajouter les vôtres. Les modèles ont toute la configuration et tous les éléments requis pour enregistrer vos scripts de performances à partir de zéro.

Voici comment utiliser la fonctionnalité de modèle JMeter :

Configurer JMeter

  1. Ouvrez JMeter

  2. Sélectionnez le modèle pour l’enregistrement du script : Fichier -> Modèles… -> Sélectionner un modèle -> Enregistrement -> Créer JMeter ajoutera les éléments pertinents à l’arbre de test.

[![entrez la description de l’image ici][2]][2]

Configurez le proxy de votre navigateur

Pour utiliser JMeter Recorder, vous devez configurer votre navigateur pour envoyer toutes les requêtes via proxy. N’importe quel navigateur peut être utilisé pour ces besoins, bien qu’il puisse y avoir des différences entre les emplacements des configurations des navigateurs, qui sont spécifiques au navigateur et peuvent varier selon le système d’exploitation.

  1. Pour configurer votre navigateur :

    Chrome: Menu button -> Settings -> Show advanced settings… -> Network -> Change proxy settings

    Safari: Preferences -> Advanced -> Proxies -> Change Settings…

    Firefox: Menu button -> Preferences -> Advanced -> Network -> Connection -> Settings..

  2. Par exemple, vous pouvez utiliser l’hôte local 127.0.0.1. Remplacez le port par le port de l’enregistreur de script HTTP(S).

[![entrez la description de l’image ici][3]][3]

Si vous rencontrez des problèmes lors de cette étape, vérifiez que vous n’avez pas de plug-ins tiers capables de gérer les paramètres de proxy de votre navigateur. Si vous le faites, comme Hola VPN par exemple, les paramètres de proxy ne seront pas disponibles dans le menu de votre navigateur.

  1. Cliquez sur le bouton “Démarrer”, qui se trouve en bas de la page “HTTP(S) Test Script Recorder”, et parcourez le flux de travail de l’application Web que vous souhaitez tester. Lorsque vous revenez à JMeter, vous devriez voir toutes les requêtes capturées depuis votre navigateur.

[1] : http://jmeter.apache.org/ [2] : https://i.stack.imgur.com/RyZce.png [3] : https://i.stack.imgur.com/9Mb0l.png

Enregistrement de scripts de performance pour les appareils mobiles #

JMeter peut également être utilisé pour enregistrer des tests de performances mobiles. L’enregistrement de scripts mobiles est très similaire à l’enregistrement de scripts d’applications Web.

Configurer JMeter

  1. Configurez « JMeter Templates » comme spécifié au chapitre 1.

Configurez votre téléphone mobile

Une fois la configuration JMeter préparée, y compris l’élément JMeter “HTTP (S) Test Script Recording” démarré sur un port spécifié, vous pouvez configurer votre téléphone mobile pour envoyer une demande à l’application Web que vous testez via le proxy JMeter.

  1. IOS :

  • Réglage -> Wi-Fi

  • Cliquez sur le réseau connecté

  • Allez dans la section de configuration “PROXY HTTP”

  • Cliquez sur l’onglet “Manuel”

  • Définissez l’adresse IP de l’ordinateur sur lequel l’application JMeter est exécutée sous “Serveur”

  • Définissez le port spécifié dans « HTTP(S) Test Script Recording » sous “Port”

    Android:

  • Réglage -> Wi-Fi

  • Cliquez longuement sur le réseau connecté et cliquez sur l’option “Modifier le réseau”

  • Cliquez sur la case à cocher “Options avancées”

  • Réglez l’option “Proxy” sur “Manuel”

  • Définissez le “Nom d’hôte proxy” comme adresse IP de votre ordinateur et “Port proxy” comme spécifié dans la configuration “Enregistrement de script de test HTTP (S)” sous “Port”

  • Cliquez sur “Enregistrer”

  1. Vous pouvez maintenant lancer l’exécution de l’application sur votre appareil mobile. Les demandes seront enregistrées sur JMeter.

Enregistrement du trafic HTTPS #

Si votre application Web utilise le cryptage SSL, vous devez capturer le trafic HTTPS au lieu de HTTP. Pour enregistrer le trafic HTTPS avec JMeter, vous devez configurer les certificats SSL.

Configurez votre proxy SSL

  1. Assurez-vous que le proxy SSL est configuré de la même manière que le proxy HTTP :

[![entrez la description de l’image ici][1]][1]

Configurer JMeter 2. Démarrez l’enregistrement du script en utilisant la fonction “Modèle d’enregistrement JMeter” comme expliqué dans l’exemple “Enregistrement de script avec la fonction de modèle JMeter”.

  1. Après avoir ouvert l’application Web, vous verrez un message concernant une connexion non sécurisée. Pour continuer, il vous suffit d’accepter le certificat factice JMeter :
  • Cliquez sur “Avancé”
  • Cliquez sur “Ajouter une exception…”
  • Décochez “Stocker cette exception en permanence”
  • Cliquez sur “Confirmer l’exception de sécurité”

[![entrez la description de l’image ici][2]][2]

[![entrez la description de l’image ici][3]][3]

  1. Si vous voyez le message “Ce site fournit une identification valide et vérifiée. Il n’est pas nécessaire d’ajouter une exception. message d’avertissement, vous devez effacer l’historique du navigateur pour votre application, y compris les cookies, le cache, les données de site Web hors ligne. Ensuite, recommencez les mêmes étapes.

Cette approche fonctionne également pour l’enregistrement de scripts mobiles, car le certificat JMeter doit être installé uniquement sur l’hôte utilisé pour exécuter JMeter.

Enregistrement de script avec l’extension Chrome BlazeMeter #

Jusqu’à présent, nous avons couvert les méthodes de base pour enregistrer des scénarios de test. Mais l’un des moyens les plus rapides et les plus simples d’enregistrer vos scripts de performance, qui est également gratuit, consiste à utiliser l’extension [BlazeMeter Recorder Chrome][1]. Ces enregistrements peuvent être exécutés dans JMeter ou dans BlazeMeter.

La raison pour laquelle l’extension est si utile est qu’elle vous permet d’enregistrer des scripts de performance à partir de votre navigateur sans avoir à configurer votre proxy.

[![entrez la description de l’image ici][2]][2]

Pour créer un nouveau script de performances :

  1. Ouvrez l’enregistreur depuis votre Chrome
  2. Entrez un nom de test dans le champ supérieur
  3. Lancez l’enregistrement en cliquant sur le bouton d’enregistrement, en forme de cercle, et effectuez les actions Web que vous souhaitez enregistrer. Toutes vos demandes seront capturées. L’extension Blazemeter Chrome prend également en charge l’enregistrement du trafic HTTPS.
  4. Une fois l’enregistrement terminé, cliquez sur le bouton d’arrêt, en forme de carré. Vous pouvez également mettre en pause votre enregistrement puis le reprendre, ainsi que le modifier, au format .jmx ou JSON, ou dans le cloud.
  5. Exportez votre enregistrement - pour exécuter le test dans JMeter, exportez au format .jmx en cliquant sur le bouton .jmx. Pour exécuter le test dans BlazeMeter, cliquez sur “jouer”.

Pour plus d’informations, voir [ici][3].

[1] : https://chrome.google.com/webstore/detail/blazemeter-the-load-testi/mbopgmdnpcbohhpnfglgohlbhfongabi?hl=fr [2] : https://i.stack.imgur.com/YSAZ1.png [3] : https://guide.blazemeter.com/hc/en-us/articles/206732849-Chrome-Extension-Record

Enregistrement de script avec BadBoy #

Un autre outil d’enregistrement tiers utile est BadBoy. Cependant, cela ne fonctionne que pour le système d’exploitation Windows.

Pour créer un nouveau script de performances :

  1. Installez BadBoy [ici][1]

  2. Entrez l’URL testée dans la barre d’adresse [![entrez la description de l’image ici][2]][2]

  3. Appuyez sur le bouton d’enregistrement, en forme de cercle rouge et effectuez les actions que vous souhaitez capturer.

  4. Exportez votre script vers JMeter - Fichier -> Exporter vers JMeter

Pour plus d’informations, voir [ici][3].

L’utilisation d’un enregistreur de scripts de performances est un excellent moyen d’éviter les tâches de routine tout en obtenant les meilleurs scripts de test. Après l’enregistrement, configurez le test sur le nombre d’utilisateurs virtuels que vous souhaitez tester ainsi que des configurations de test supplémentaires, exécutez votre test et analysez les résultats pour identifier les erreurs et les goulots d’étranglement et caractériser les tendances qui vous montrent la santé de votre système.
