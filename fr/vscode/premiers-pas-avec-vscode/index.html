Installation ou configuration #

Sous Windows #

  • [Téléchargez le programme d’installation de Visual Studio Code][1] pour Windows.
  • Une fois téléchargé, lancez le programme d’installation (VSCodeSetup-version.exe). Cela ne prendra qu’une minute.

Par défaut, VS Code est installé sous C:\Program Files (x86)\Microsoft VS Code pour une machine 64 bits.

Remarque : .NET Framework 4.5.2 est requis pour VS Code. Si vous utilisez Windows 7, assurez-vous que .NET Framework 4.5.2 est installé.

Conseil : Le programme d’installation ajoutera éventuellement Visual Studio Code à votre %PATH%, de sorte que depuis la console, vous pouvez taper “code”. pour ouvrir VS Code sur ce dossier. Vous devrez redémarrer votre console après l’installation pour que la modification de la variable d’environnement %PATH% prenne effet.

Sur Mac #

  • [Télécharger Visual Studio Code][2] pour Mac.
  • Double-cliquez sur l’archive téléchargée pour développer le contenu.
  • Faites glisser Visual Studio Code.app vers le dossier Applications, le rendant disponible dans le Launchpad.
  • Ajoutez VS Code à votre Dock en cliquant avec le bouton droit sur l’icône et en choisissant Options, Conserver dans le Dock.

Vous pouvez également exécuter VS Code depuis le terminal en tapant “code” après l’avoir ajouté au chemin :

  • Lancer VS Code.
  • Ouvrez la palette de commandes (Ctrl+Maj+P) et tapez « commande shell » pour trouver la commande Shell : installez la commande « code » dans la commande PATH.

Redémarrez le terminal pour que la nouvelle valeur $PATH prenne effet. Vous pourrez taper ‘code .’ dans n’importe quel dossier pour commencer à modifier les fichiers de ce dossier.

Remarque : si vous avez toujours l’ancien alias de code dans votre .bash_profile (ou équivalent) d’une version antérieure de VS Code, supprimez-le et remplacez-le en exécutant la commande Shell : Installer la commande ‘code’ dans la commande PATH.

Sous Linux #

Distributions basées sur Debian et Ubuntu #

Le moyen le plus simple d’installer les distributions basées sur Debian/Ubuntu est de [télécharger][3] et d’installer le package .deb (64 bits) soit via le centre logiciel graphique s’il est disponible, soit via la ligne de commande avec :

sudo dpkg -i <file>.deb
sudo apt-get install -f # Install dependencies

L’installation du package .deb installera automatiquement le référentiel apt et la clé de signature pour activer la mise à jour automatique à l’aide du mécanisme système normal. Notez que les binaires 32 bits et .tar.gz sont également disponibles sur la page de téléchargement.

Le référentiel et la clé peuvent également être installés manuellement avec le script suivant :

curl https://packages.microsoft.com/keys/microsoft.asc | gpg --dearmor > microsoft.gpg
sudo mv microsoft.gpg /etc/apt/trusted.gpg.d/microsoft.gpg
sudo sh -c 'echo "deb [arch=amd64] https://packages.microsoft.com/repos/vscode stable main" > /etc/apt/sources.list.d/vscode.list'

Ensuite, mettez à jour le cache du package et installez le package en utilisant :

sudo apt-get update
sudo apt-get install code # or code-insiders for insiders build

Distributions basées sur RHEL, Fedora et CentOS #

Nous expédions actuellement le code VS 64 bits stable dans un référentiel yum, le script suivant installera la clé et le référentiel :

sudo rpm --import https://packages.microsoft.com/keys/microsoft.asc
sudo sh -c 'echo -e "[code]\nname=Visual Studio Code\nbaseurl=https://packages.microsoft.com/yumrepos/vscode\nenabled=1\ngpgcheck=1\ngpgkey=https://packages.microsoft.com/keys/microsoft.asc" > /etc/yum.repos.d/vscode.repo'

Ensuite, mettez à jour le cache du package et installez le package à l’aide de dnf (Fedora 22 et supérieur) :

dnf check-update
sudo dnf install code

Ou sur les anciennes versions utilisant yum :

yum check-update
sudo yum install code

Distributions basées sur openSUSE et SLE #

Le référentiel yum ci-dessus fonctionne également pour les systèmes basés sur openSUSE et SLE, le script suivant installera la clé et le référentiel :

sudo rpm --import https://packages.microsoft.com/keys/microsoft.asc
sudo sh -c 'echo -e "[code]\nname=Visual Studio Code\nbaseurl=https://packages.microsoft.com/yumrepos/vscode\nenabled=1\ntype=rpm-md\ngpgcheck=1\ngpgkey=https://packages.microsoft.com/keys/microsoft.asc" > /etc/zypp/repos.d/vscode.repo'

Ensuite, mettez à jour le cache du package et installez le package en utilisant :

sudo zypper refresh
sudo zypper install code

Paquet AUR pour Arch Linux #

Il existe un [package Arch User Repository (AUR)][4] géré par la communauté pour VS Code.

Installation manuelle du package .rpm Le package .rpm (64 bits) peut également être téléchargé et installé manuellement, mais la mise à jour automatique ne fonctionnera que si le référentiel ci-dessus est installé. Une fois téléchargé, il peut être installé à l’aide de votre gestionnaire de packages, par exemple avec dnf :

sudo dnf install <file>.rpm

Notez que les binaires 32 bits et .tar.gz sont également disponibles sur la [page de téléchargement][5].

Premiers pas (C++) : HelloWorld.cpp #

Le premier programme que l’on écrit généralement dans n’importe quel langage est le script “hello world”. Cet exemple montre comment écrire ce programme et le déboguer à l’aide de Visual Studio Code (je ferai désormais référence à Visual Studio Code en tant que VS Code).

Créer le projet

L’étape 1 consistera à créer un nouveau projet. Cela peut être fait de plusieurs façons. La première façon est directement depuis l’interface utilisateur.

  1. Ouvrez le programme VS Code. Vous serez accueilli avec l’écran de bienvenue standard (notez que les images sont prises lorsque vous travaillez sur un Mac, mais elles doivent être similaires à votre installation) : [![Écran d’accueil du code VS][1]][1]

[1] : https://i.stack.imgur.com/j3KLo.png

  1. Dans le menu Démarrer, sélectionnez Nouveau fichier. Cela ouvrira une nouvelle fenêtre d’édition où nous pourrons commencer à construire notre script. Allez-y et enregistrez ce fichier (vous pouvez utiliser le menu Fichier > Enregistrer pour le faire). Pour cet exemple nous appellerons le fichier HelloWorld.cpp et le placerons dans un nouveau répertoire que nous appellerons VSC_HelloWorld/.

  2. Écrivez le programme. Cela devrait être assez simple, mais n’hésitez pas à copier ce qui suit dans le fichier :

    #include

    int main() { std::cout « “Hello world!” « std::endl; return 0; }

Exécuter le code

Ensuite, nous voulons exécuter le script et vérifier sa sortie. Il y a un certain nombre de façons de le faire. Le plus simple est d’ouvrir un terminal et de naviguer dans le répertoire que nous avons créé. Vous pouvez maintenant compiler le script et l’exécuter avec gcc en tapant :

$ g++ HelloWorld.cpp -o helloworld
$ ./helloworld
Hello World!

Youpi, le programme a fonctionné ! Mais ce n’est pas vraiment ce que nous voulons. Ce serait bien mieux si nous pouvions exécuter le programme à partir de VSCode lui-même. Nous avons de la chance quand même ! VSCode a un terminal intégré auquel nous pouvons accéder via le menu “View” > “Integrated Terminal”. Cela ouvrira un terminal dans la moitié inférieure de la fenêtre à partir de laquelle vous pourrez accéder au répertoire VSC_HelloWorld et exécuter les commandes ci-dessus.

Généralement, nous le faisons en exécutant une Tâche d’exécution. Dans le menu, sélectionnez “Tâches” > “Exécuter la tâche…”. Vous remarquerez que vous obtenez une petite fenêtre contextuelle près du haut de la fenêtre avec un message d’erreur (quelque chose du genre

Premier programme (C++) : Hello World.cpp #

Cet exemple vous présente les fonctionnalités de base de VS Code en démontrant comment écrire un programme “hello world” en C++. Avant de continuer, assurez-vous que l’extension “ms-vscode.cpptools” est installée.

** Initialiser le projet **

La première étape consiste à créer un nouveau projet. Pour ce faire, chargez le programme VS Code. Vous devriez être accueilli par l’écran de bienvenue typique : [![entrez la description de l’image ici][1]][1]

Pour créer le premier programme, sélectionnez “Démarrer” > “Nouveau fichier” depuis l’écran d’accueil. Cela ouvrira une nouvelle fenêtre de fichier. Allez-y et enregistrez le fichier ("Fichier" > “Enregistrer”) dans un nouveau répertoire. Vous pouvez nommer le répertoire comme vous le souhaitez, mais cet exemple appellera le répertoire “VSC_HelloWorld” et le fichier “HelloWorld.cpp”.

Maintenant, écrivez le programme réel (n’hésitez pas à copier le texte ci-dessous):

#include <iostream>

int main()
{
    // Output the hello world text
    std::cout << "Hello world!" << std::endl;
    return 0;
}

Super! Vous remarquerez également que parce que vous avez installé l’extension “ms-vscode.cpptools”, vous avez également une jolie mise en évidence du code. Passons maintenant à l’exécution du code.

Exécuter le script (de base)

Nous pouvons exécuter “HelloWorld.cpp” à partir de VS Code lui-même. La façon la plus simple d’exécuter un tel programme est d’ouvrir le terminal intégré ("View" > “Integrated Terminal”). Cela ouvre une fenêtre de terminal dans la partie inférieure de la vue. De l’intérieur de ce terminal, nous pouvons naviguer vers notre répertoire créé, construire et exécuter le script que nous avons écrit. [![entrez la description de l’image ici][2]][2] Ici, nous avons utilisé les commandes suivantes pour compiler et exécuter le code :

$ g++ HelloWorld.cpp -o hellowold
$ ./hellowold

Notez que nous obtenons la sortie attendue “Hello World!”.

Exécution du script (légèrement plus avancé)

Génial, mais nous pouvons également utiliser VS Code directement pour créer et exécuter le code. Pour cela, nous devons d’abord transformer le répertoire “VSC_HelloWorld” en espace de travail. Cela peut être fait par :

  1. Ouvrir le menu Explorer (l’élément le plus haut dans le menu vertical à l’extrême gauche)
  2. Sélectionnez le bouton Ouvrir le dossier
  3. Sélectionnez le répertoire “VSC_HelloWorld” dans lequel nous avons travaillé. [![entrez la description de l’image ici][3]][3] Remarque : Si vous ouvrez un répertoire dans VS Code (en utilisant “Fichier” > “Ouvrir…” par exemple), vous serez déjà dans un espace de travail.

Le menu Explorer affiche maintenant le contenu du répertoire.

Ensuite, nous voulons définir les tâches réelles que nous voulons que VS Code exécute. Pour ce faire, sélectionnez “Tâches” > “Configurer la tâche de génération par défaut”. Dans le menu déroulant, sélectionnez “Autre”. Cela ouvre un nouveau fichier appelé “tasks.json” qui contient des valeurs par défaut pour une tâche. Nous devons changer ces valeurs. Mettez à jour ce fichier pour qu’il contienne les éléments suivants et enregistrez-le :

{
    "version": "2.0.0",
    "tasks": [
        {
            "taskName": "build",
            "type": "shell",
            "command": "g++ HelloWorld.cpp -o helloworld"
        },
        {
            "taskName": "run",
            "type": "shell",
            "command": "${workspaceRoot}/helloworld"
        }
    ]
}

Notez que ce qui précède crée également un répertoire caché .vscode dans notre répertoire de travail. C’est là que VS Code place les fichiers de configuration, y compris tous les fichiers de paramètres spécifiques au projet. Vous pouvez en savoir plus sur Tasks ici.

Dans l’exemple ci-dessus, ${workspaceRoot} fait référence au répertoire de niveau supérieur de notre espace de travail, qui est notre répertoire “VSC_HelloWorld”. Maintenant, pour construire le projet depuis l’intérieur de la méthode, sélectionnez “Tasks” > “Run Build Task…” et sélectionnez notre tâche “build” créée et “Continue sans analyser la tâche output” dans les menus déroulants qui s’affichent. Ensuite, nous pouvons exécuter l’exécutable en utilisant “Tasks” > “Run Task…” et en sélectionnant la tâche “run” que nous avons créée. Si le terminal intégré est ouvert, vous remarquerez que le message “Hello World!” le texte y sera imprimé.

Il est possible que le terminal se ferme avant que vous puissiez voir la sortie. Si cela se produit, vous pouvez insérer une ligne de code comme celle-ci int i; std::cin >> i; juste avant l’instruction return à la fin de la fonction main(). Vous pouvez ensuite terminer le script en tapant n’importe quel nombre et en appuyant sur <Entrée>.

Et c’est tout! Vous pouvez maintenant commencer à écrire et à exécuter vos scripts C++ à partir de VS Code.
