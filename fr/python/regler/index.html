Syntaxe #

  • empty_set = set() # initialise un ensemble vide
  • literal_set = {‘foo’, ‘bar’, ‘baz’} # construit un ensemble avec 3 chaînes à l’intérieur
  • set_from_list = set([‘foo’, ‘bar’, ‘baz’]) # appelle la fonction set pour un nouvel ensemble
  • set_from_iter = set(x for x in range(30)) # utilise des itérables arbitraires pour créer un ensemble
  • set_from_iter = {x for x in [random.randint(0,10) for i in range(10)]} # notation alternative

Les ensembles sont non ordonnés et ont un temps de recherche très rapide (amorti O(1) si vous voulez être technique). Il est idéal à utiliser lorsque vous avez une collection de choses, que l’ordre n’a pas d’importance et que vous chercherez beaucoup d’articles par leur nom. S’il est plus logique de rechercher des éléments par un numéro d’index, envisagez plutôt d’utiliser une liste. Si l’ordre est important, envisagez également une liste.

Les ensembles sont mutables et ne peuvent donc pas être hachés, vous ne pouvez donc pas les utiliser comme clés de dictionnaire ou les mettre dans d’autres ensembles, ou n’importe où ailleurs qui nécessite des types hachables. Dans de tels cas, vous pouvez utiliser un frozenset immuable.

Les éléments d’un ensemble doivent être hashable. Cela signifie qu’ils ont une méthode __hash__ correcte, cohérente avec __eq__. En général, les types mutables tels que list ou set ne sont pas hachables et ne peuvent pas être placés dans un ensemble. Si vous rencontrez ce problème, pensez à utiliser dict et des clés immuables.

Opérations sur les ensembles #

avec d’autres ensembles

# Intersection    
{1, 2, 3, 4, 5}.intersection({3, 4, 5, 6})  # {3, 4, 5}
{1, 2, 3, 4, 5} & {3, 4, 5, 6}              # {3, 4, 5}

# Union
{1, 2, 3, 4, 5}.union({3, 4, 5, 6})  # {1, 2, 3, 4, 5, 6}
{1, 2, 3, 4, 5} | {3, 4, 5, 6}       # {1, 2, 3, 4, 5, 6}

# Difference
{1, 2, 3, 4}.difference({2, 3, 5})  # {1, 4}
{1, 2, 3, 4} - {2, 3, 5}            # {1, 4}

# Symmetric difference with
{1, 2, 3, 4}.symmetric_difference({2, 3, 5})  # {1, 4, 5}
{1, 2, 3, 4} ^ {2, 3, 5}                      # {1, 4, 5}

# Superset check
{1, 2}.issuperset({1, 2, 3})  # False
{1, 2} >= {1, 2, 3}           # False

# Subset check
{1, 2}.issubset({1, 2, 3})  # True
{1, 2} <= {1, 2, 3}         # True

# Disjoint check
{1, 2}.isdisjoint({3, 4})  # True
{1, 2}.isdisjoint({1, 4})  # False

avec des éléments simples

# Existence check
2 in {1,2,3}      # True
4 in {1,2,3}      # False
4 not in {1,2,3}  # True

# Add and Remove
s = {1,2,3}
s.add(4)        # s == {1,2,3,4}

s.discard(3)    # s == {1,2,4}
s.discard(5)    # s == {1,2,4}

s.remove(2)     # s == {1,4}
s.remove(2)     # KeyError!

Les opérations d’ensemble renvoient de nouveaux ensembles, mais ont les versions sur place correspondantes :

méthode opération sur place méthode en place
union s |= t mise à jour
intersection s &= t mise à jour_intersection
différence s -= t difference_update
différence_symétrique s ^= t mise à jour_différence_symétrique

Par exemple:

s = {1, 2}
s.update({3, 4})   # s == {1, 2, 3, 4}

Récupère les éléments uniques d’une liste #

Disons que vous avez une liste de restaurants - peut-être que vous l’avez lue à partir d’un fichier. Vous vous souciez des restaurants uniques de la liste. La meilleure façon d’obtenir les éléments uniques d’une liste est de la transformer en un ensemble :

restaurants = ["McDonald's", "Burger King", "McDonald's", "Chicken Chicken"]
unique_restaurants = set(restaurants)
print(unique_restaurants)
# prints {'Chicken Chicken', "McDonald's", 'Burger King'}

Notez que l’ensemble n’est pas dans le même ordre que la liste d’origine ; c’est parce que les ensembles sont non ordonnés, tout comme les dict.

Cela peut facilement être retransformé en une List avec la fonction list intégrée de Python, donnant une autre liste qui est la même liste que l’originale mais sans doublons :

list(unique_restaurants)
# ['Chicken Chicken', "McDonald's", 'Burger King']

Il est également courant de voir ceci comme une seule ligne :

# Removes all duplicates and returns another list
list(set(restaurants))

Désormais, toutes les opérations pouvant être effectuées sur la liste d’origine peuvent être effectuées à nouveau.

Ensemble d’ensembles #

{{1,2}, {3,4}}

mène à:

TypeError: unhashable type: 'set'

À la place, utilisez frozenset :

{frozenset({1, 2}), frozenset({3, 4})}

Définir les opérations à l’aide de méthodes et de fonctions intégrées #

Nous définissons deux ensembles a et b

>>> a = {1, 2, 2, 3, 4}
>>> b = {3, 3, 4, 4, 5}

REMARQUE : {1} crée un ensemble d’un élément, mais {} crée un dict vide. La bonne façon de créer un ensemble vide est set().

Intersection #

a.intersection(b) renvoie un nouvel ensemble avec des éléments présents à la fois dans a et b

>>> a.intersection(b)
{3, 4}

Syndicat #

a.union(b) renvoie un nouvel ensemble avec des éléments présents dans a et b

>>> a.union(b)
{1, 2, 3, 4, 5}

Différence #

a.difference(b) renvoie un nouvel ensemble avec des éléments présents dans a mais pas dans b

>>> a.difference(b)
{1, 2}
>>> b.difference(a)
{5}

Différence symétrique #

a.symmetric_difference(b) renvoie un nouvel ensemble avec des éléments présents dans a ou b mais pas dans les deux

>>> a.symmetric_difference(b)
{1, 2, 5}
>>> b.symmetric_difference(a)
{1, 2, 5}

REMARQUE : a.symmetric_difference(b) == b.symmetric_difference(a)

Sous-ensemble et sur-ensemble #

c.issubset(a) teste si chaque élément de c est dans a.

a.issuperset(c) teste si chaque élément de c est dans a.

>>> c = {1, 2}
>>> c.issubset(a)
True
>>> a.issuperset(c)
True

Ces dernières opérations ont des opérateurs équivalents comme indiqué ci-dessous :

Méthode Opérateur
a.intersection(b) a & b
a.union(b) a | b
a.différence(b) a - b
a.symmetric_difference(b) a ^ b
a.issubset(b) a <= b
a.superset(b) a >= b

Ensembles disjoints #

Les ensembles a et d sont disjoints si aucun élément de a n’est également dans d et vice versa.

>>> d = {5, 6}
>>> a.isdisjoint(b) # {2, 3, 4} are in both sets
False
>>> a.isdisjoint(d)
True

# This is an equivalent check, but less efficient
>>> len(a & d) == 0
True

# This is even less efficient
>>> a & d == set()
True

Tester l’adhésion #

Le mot-clé intégré in recherche les occurrences

>>> 1 in a
True
>>> 6 in a
False

Longueur #

La fonction intégrée len() renvoie le nombre d’éléments dans l’ensemble

>>> len(a)
4
>>> len(b)
3

Ensembles contre multiensembles #

Les ensembles sont des collections non ordonnées d’éléments distincts. Mais parfois, nous voulons travailler avec des collections non ordonnées d’éléments qui ne sont pas nécessairement distincts et garder une trace des multiplicités des éléments.

Considérez cet exemple :

>>> setA = {'a','b','b','c'}
>>> setA
set(['a', 'c', 'b'])

En sauvegardant les chaînes 'a', 'b', 'b', 'c' dans une structure de données définie, nous avons perdu l’information sur le fait que 'b' apparaît deux fois. Bien sûr, enregistrer les éléments dans une liste conserverait ces informations

>>> listA = ['a','b','b','c']
>>> listA
['a', 'b', 'b', 'c']

mais une structure de données de liste introduit un ordre supplémentaire inutile qui ralentira nos calculs.

Pour implémenter des multisets, Python fournit la classe Counter du module collections (à partir de la version 2.7):

>>> from collections import Counter
>>> counterA = Counter(['a','b','b','c'])
>>> counterA
Counter({'b': 2, 'a': 1, 'c': 1})

Counter est un dictionnaire dans lequel les éléments sont stockés sous forme de clés de dictionnaire et leurs nombres sont stockés sous forme de valeurs de dictionnaire. Et comme tous les dictionnaires, c’est une collection non ordonnée.
