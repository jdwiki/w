Les fonctions en Python fournissent un code organisé, réutilisable et modulaire pour effectuer un ensemble d’actions spécifiques. Les fonctions simplifient le processus de codage, évitent la logique redondante et rendent le code plus facile à suivre. Cette rubrique décrit la déclaration et l’utilisation des fonctions dans Python.

Python a de nombreuses fonctions intégrées comme print(), input(), len(). Outre les fonctions intégrées, vous pouvez également créer vos propres fonctions pour effectuer des tâches plus spécifiques. Celles-ci sont appelées fonctions définies par l’utilisateur.

Syntaxe #

  • def nom_fonction(arg1, … argN, *args, kw1, kw2=default, …, **kwargs) : instructions
  • lambda arg1, … argN, *args, kw1, kw2=default, …, **kwargs : expression

Paramètres #

Paramètre Détails
arg1, …, argN Arguments réguliers
*arguments Arguments positionnels sans nom
kw1, …, kwN Arguments de mot-clé uniquement
**kwargs Le reste des arguments de mots-clés

5 choses de base que vous pouvez faire avec les fonctions :

  • Affecter des fonctions aux variables

    def f():
  print(20)
y = f
y()
# Output: 20

  • Définir des fonctions dans d’autres fonctions (https://www.wikiod.com/fr/python/les-fonctions#Fonctions imbriquées )

    def f(a, b, y):
    def inner_add(a, b):      # inner_add is hidden from outer code
        return a + b
    return inner_add(a, b)**y

  • Les fonctions peuvent retourner d’autres fonctions

    def f(y):
    def nth_power(x):
        return x ** y
    return nth_power    # returns a function

squareOf = f(2)         # function that returns the square of a number           
cubeOf = f(3)           # function that returns the cube of a number
squareOf(3)             # Output: 9
cubeOf(2)               # Output: 8

  • Les fonctions peuvent être transmises en tant que paramètres à d’autres fonctions

    def a(x, y):
    print(x, y)
def b(fun, str):        # b has two arguments: a function and a string 
    fun('Hello', str)
b(a, 'Sophia')           # Output: Hello Sophia

  • Les fonctions internes ont accès à la portée englobante (https://www.wikiod.com/fr/python/les-fonctions#Fermeture )

    def outer_fun(name):
    def inner_fun():     # the variable name is available to the inner function
        return "Hello "+ name + "!"
    return inner_fun
greet = outer_fun("Sophia")
print(greet())            # Output: Hello Sophia!

Définir une fonction avec un nombre arbitraire d’arguments #

Nombre arbitraire d’arguments positionnels : #

Définir une fonction capable de prendre un nombre arbitraire d’arguments peut être fait en préfixant l’un des arguments avec un *

def func(*args):
    # args will be a tuple containing all values that are passed in
    for i in args:
        print(i)

func(1, 2, 3)  # Calling it with 3 arguments
# Out: 1
#      2
#      3

list_of_arg_values = [1, 2, 3]
func(*list_of_arg_values)  # Calling it with list of values, * expands the list
# Out: 1
#      2
#      3 

func()  # Calling it without arguments
# No Output

Vous ne pouvez pas fournir une valeur par défaut pour args, par exemple func(*args=[1, 2, 3]) générera une erreur de syntaxe (ne compilera même pas).

Vous ne pouvez pas les fournir par leur nom lors de l’appel de la fonction, par exemple func(*args=[1, 2, 3]) déclenchera une TypeError.

Mais si vous avez déjà vos arguments dans un tableau (ou tout autre Iterable), vous pouvez invoquer votre fonction comme ceci : func(*my_stuff).

Ces arguments (*args) sont accessibles par index, par exemple args[0] renverra le premier argument

Nombre arbitraire d’arguments de mots-clés #

Vous pouvez prendre un nombre arbitraire d’arguments avec un nom en définissant un argument dans la définition avec deux * devant :

def func(**kwargs):
    # kwargs will be a dictionary containing the names as keys and the values as values
    for name, value in kwargs.items():
        print(name, value)

func(value1=1, value2=2, value3=3)   # Calling it with 3 arguments
# Out: value1 1
#      value2 2
#      value3 3

func()                               # Calling it without arguments
# No Out put

my_dict = {'foo': 1, 'bar': 2}
func(**my_dict)                      # Calling it with a dictionary
# Out: foo 1
#      bar 2

Vous ne pouvez pas fournir ces noms sans, par exemple func(1, 2, 3) déclenchera une TypeError.

kwargs est un dictionnaire python natif simple. Par exemple, args['value1'] donnera la valeur de l’argument value1. Assurez-vous de vérifier au préalable qu’il existe un tel argument ou une KeyError sera levée.

Avertissement #

Vous pouvez les mélanger avec d’autres arguments facultatifs et obligatoires, mais l’ordre dans la définition est important.

Les arguments positionnels/mots clés viennent en premier. (arguments obligatoires). Viennent ensuite les arguments arbitraires *arg. (Optionnel). Viennent ensuite les arguments mot clé uniquement. (Obligatoire). Enfin, le mot-clé arbitraire **kwargs arrive. (Optionnel).

#       |-positional-|-optional-|---keyword-only--|-optional-|
def func(arg1, arg2=10 , *args, kwarg1, kwarg2=2, **kwargs):
     pass
  • arg1 doit être donné, sinon une TypeError est levée. Il peut être donné en tant qu’argument positionnel (func(10)) ou mot-clé (func(arg1=10)).
  • kwarg1 doit également être donné, mais il ne peut être fourni qu’en tant qu’argument mot-clé : func(kwarg1=10).
  • arg2 et kwarg2 sont facultatifs. Si la valeur doit être modifiée, les mêmes règles que pour arg1 (positionnel ou mot-clé) et kwarg1 (uniquement mot-clé) s’appliquent.
  • *args capture des paramètres de position supplémentaires. Mais notez que arg1 et arg2 doivent être fournis en tant qu’arguments positionnels pour passer des arguments à *args : func(1, 1, 1, 1).
  • **kwargs capture tous les paramètres de mots-clés supplémentaires. Dans ce cas, tout paramètre autre que arg1, arg2, kwarg1 ou kwarg2. Par exemple : func(kwarg3=10).
  • En Python 3, vous pouvez utiliser * seul pour indiquer que tous les arguments suivants doivent être spécifiés en tant que mots-clés. Par exemple, la fonction math.isclose dans Python 3.5 et supérieur est définie à l’aide de def math.isclose (a, b, *, rel_tol=1e-09, abs_tol=0.0), ce qui signifie que les deux premiers arguments peuvent être fournis positionnellement, mais les troisième et quatrième paramètres facultatifs ne peuvent être fournis que sous forme d’arguments de mots clés.

Python 2.x ne prend pas en charge les paramètres de mots clés uniquement. Ce comportement peut être émulé avec kwargs :

def func(arg1, arg2=10, **kwargs):
    try:
        kwarg1 = kwargs.pop("kwarg1")
    except KeyError:
        raise TypeError("missing required keyword-only argument: 'kwarg1'")

    kwarg2 = kwargs.pop("kwarg2", 2)
    # function body ...

Remarque sur la dénomination #

La convention de nommer les arguments positionnels facultatifs args et les arguments de mots clés facultatifs kwargs est juste une convention que vous ** pouvez ** utiliser les noms que vous aimez ** mais ** il est utile de suivre la convention afin que les autres sachent ce que vous faites, * ou même vous-même plus tard *, alors faites-le s’il vous plaît.

Remarque sur l’unicité #

Toute fonction peut être définie avec aucun ou un *args et aucun ou un **kwargs mais pas avec plus d’un de chaque. De plus, *args doit être le dernier argument positionnel et **kwargs doit être le dernier paramètre. Tenter d’utiliser plus d’un des deux résultera en une exception d’erreur de syntaxe.

Remarque sur les fonctions d’imbrication avec des arguments facultatifs #

Il est possible d’imbriquer de telles fonctions et la convention habituelle consiste à supprimer les éléments que le code a déjà gérés ** mais ** si vous transmettez les paramètres, vous devez transmettre des arguments positionnels facultatifs avec un préfixe * et un mot-clé facultatif args avec un préfixe **, sinon args doit être passé en tant que liste ou tuple et kwargs en tant que dictionnaire unique. par exemple.:

def fn(**kwargs):
    print(kwargs)
    f1(**kwargs)

def f1(**kwargs):
    print(len(kwargs))

fn(a=1, b=2)
# Out:
# {'a': 1, 'b': 2}
# 2

Définir et appeler des fonctions simples #

L’utilisation de l’instruction def est le moyen le plus courant de définir une fonction en python. Cette instruction est ce qu’on appelle une instruction composée à clause unique avec la syntaxe suivante :

def function_name(parameters):
    statement(s)

function_name est connu comme l’identifiant de la fonction. Puisqu’une définition de fonction est une instruction exécutable, son exécution lie le nom de la fonction à l’objet fonction qui peut être appelé ultérieurement à l’aide de l’identifiant.

parameters est une liste facultative d’identificateurs liés aux valeurs fournies en tant qu’arguments lorsque la fonction est appelée. Une fonction peut avoir un nombre arbitraire d’arguments séparés par des virgules.

instruction(s) - également connu sous le nom de corps de la fonction - sont une séquence non vide d’instructions exécutées chaque fois que la fonction est appelée. Cela signifie qu’un corps de fonction ne peut pas être vide, comme n’importe quel [bloc indenté](https://www.wikiod.com/fr/python/debuter-avec-le-langage-python#Indentation de bloc).

Voici un exemple de définition de fonction simple dont le but est d’afficher “Bonjour” à chaque fois qu’elle est appelée :

def greet():
    print("Hello")

Appelons maintenant la fonction définie greet() :

greet()
# Out: Hello

C’est un autre exemple de définition de fonction qui prend un seul argument et affiche la valeur transmise à chaque fois que la fonction est appelée :

def greet_two(greeting):
    print(greeting)

Après cela, la fonction greet_two() doit être appelée avec un argument :

greet_two("Howdy")
# Out: Howdy

Vous pouvez également donner une valeur par défaut à cet argument de fonction :

def greet_two(greeting="Howdy"):
    print(greeting)

Vous pouvez maintenant appeler la fonction sans donner de valeur :

greet_two()
# Out: Howdy

Vous remarquerez que contrairement à de nombreux autres langages, vous n’avez pas besoin de déclarer explicitement un type de retour de la fonction. Les fonctions Python peuvent renvoyer des valeurs de n’importe quel type via le mot clé return. Une fonction peut renvoyer n’importe quel nombre de types différents !

def many_types(x):
    if x < 0:
        return "Hello!"
    else:
        return 0

print(many_types(1))
print(many_types(-1))

# Output:
0
Hello!

Tant que cela est géré correctement par l’appelant, il s’agit de code Python parfaitement valide.

Une fonction qui atteint la fin de l’exécution sans instruction return renverra toujours None :

def do_nothing():
    pass

print(do_nothing())
# Out: None

Comme mentionné précédemment, une définition de fonction doit avoir un corps de fonction, une séquence non vide d’instructions. Par conséquent, l’instruction pass est utilisée comme corps de fonction, qui est une opération nulle - lorsqu’elle est exécutée, rien ne se passe. Il fait ce que cela veut dire, il saute. Il est utile comme espace réservé lorsqu’une instruction est requise syntaxiquement, mais qu’aucun code ne doit être exécuté.

Fonctions Lambda (en ligne/anonymes) #

Le mot-clé lambda crée une fonction en ligne qui contient une seule expression. La valeur de cette expression est ce que la fonction renvoie lorsqu’elle est invoquée.

Considérez la fonction :

def greeting():
    return "Hello"

qui, lorsqu’il est appelé:

print(greeting())

impressions :

Hello

Cela peut être écrit comme une fonction lambda comme suit :

greet_me = lambda: "Hello"

Voir la note au bas de cette section concernant l’affectation des lambdas aux variables. Généralement, ne le faites pas.

Cela crée une fonction en ligne avec le nom greet_me qui renvoie Hello. Notez que vous n’écrivez pas return lors de la création d’une fonction avec lambda. La valeur après : est automatiquement renvoyée.

Une fois assignée à une variable, elle peut être utilisée comme une fonction normale :

print(greet_me())

impressions :

Hello

lambda peut aussi prendre des arguments :

strip_and_upper_case = lambda s: s.strip().upper()

strip_and_upper_case("  Hello   ")

renvoie la chaîne :

HELLO

Ils peuvent également prendre un nombre arbitraire d’arguments / arguments de mots-clés, comme les fonctions normales.

greeting = lambda x, *args, **kwargs: print(x, args, kwargs)
greeting('hello', 'world', world='world')

impressions :

hello ('world',) {'world': 'world'}

lambdas sont couramment utilisés pour les fonctions courtes qui sont pratiques à définir au point où elles sont appelées (généralement avec sorted, filter et map).

Par exemple, cette ligne trie une liste de chaînes en ignorant leur casse et en ignorant les espaces au début et à la fin :

sorted( [" foo ", "    bAR", "BaZ    "], key=lambda s: s.strip().upper())
# Out:
# ['    bAR', 'BaZ    ', ' foo ']

Trier la liste en ignorant simplement les espaces :

sorted( [" foo ", "    bAR", "BaZ    "], key=lambda s: s.strip())
# Out:
# ['BaZ    ', '    bAR', ' foo ']

Exemples avec map :

sorted( map( lambda s: s.strip().upper(), [" foo ", "    bAR", "BaZ    "]))
# Out:
# ['BAR', 'BAZ', 'FOO']

sorted( map( lambda s: s.strip(), [" foo ", "    bAR", "BaZ    "]))
# Out:
# ['BaZ', 'bAR', 'foo']

Exemples avec des listes numériques :

my_list = [3, -4, -2, 5, 1, 7]
sorted( my_list, key=lambda x: abs(x))
# Out:
# [1, -2, 3, -4, 5, 7]

list( filter( lambda x: x>0, my_list))
# Out:
# [3, 5, 1, 7]

list( map( lambda x: abs(x), my_list))
# Out:
[3, 4, 2, 5, 1, 7]

On peut appeler d’autres fonctions (avec/sans arguments) depuis l’intérieur d’une fonction lambda.

def foo(msg):
    print(msg)

greet = lambda x = "hello world": foo(x)
greet()

impressions :

hello world

Ceci est utile car lambda ne peut contenir qu’une seule expression et en utilisant une fonction subsidiaire, on peut exécuter plusieurs instructions.

REMARQUE

Gardez à l’esprit que [PEP-8][1] (le guide de style Python officiel) ne recommande pas d’attribuer des lambdas aux variables (comme nous l’avons fait dans les deux premiers exemples) :

Utilisez toujours une instruction def au lieu d’une instruction d’affectation qui lie une expression lambda directement à un identifiant.

Oui :

def f(x) : retourne 2*x Non :

f = lambda x : 2*x La première forme signifie que le nom de l’objet fonction résultant est spécifiquement f au lieu du générique <lambda>. Ceci est plus utile pour les traces et les représentations de chaînes en général. L’utilisation de l’instruction d’affectation élimine le seul avantage qu’une expression lambda peut offrir par rapport à une instruction def explicite (c’est-à-dire qu’elle peut être intégrée dans une expression plus grande).

[1] : https://www.python.org/dev/peps/pep-0008/#programming-recommendations

Définir une fonction avec des arguments optionnels #

Des arguments facultatifs peuvent être définis en attribuant (à l’aide de =) une valeur par défaut au nom de l’argument :

def make(action='nothing'):
    return action

L’appel de cette fonction est possible de 3 manières différentes :

make("fun")
# Out: fun

make(action="sleep")
# Out: sleep

# The argument is optional so the function will use the default value if the argument is 
# not passed in.
make()   
# Out: nothing

Avertissement #

Les types mutables (list, dict, set, etc.) doivent être traités avec précaution lorsqu’ils sont donnés comme attribut default. Toute mutation de l’argument par défaut le changera définitivement. Voir [Définir une fonction avec des arguments mutables facultatifs][1].

[1] : https://www.wikiod.com/fr/python/les-fonctions#Définition d’une fonction avec des arguments mutables facultatifs

Définition d’une fonction avec des arguments mutables facultatifs #

Il y a un problème lors de l’utilisation d’arguments facultatifs avec un type par défaut mutable (décrit dans [Définir une fonction avec des arguments facultatifs](https://www.wikiod.com/fr/python/les-fonctions /defining-a-function-with-optional-arguments)), ce qui peut potentiellement conduire à un comportement inattendu.

Explication #

Ce problème survient parce que les arguments par défaut d’une fonction sont initialisés une fois, au moment où la fonction est définie, et pas (comme beaucoup d’autres langages) lorsque la fonction est appelée. Les valeurs par défaut sont stockées dans la variable membre __defaults__ de l’objet fonction.

def f(a, b=42, c=[]):
    pass

print(f.__defaults__)
# Out: (42, [])

Pour les types immuables (voir [Passage d’arguments et mutabilité][1]), ce n’est pas un problème car il n’y a aucun moyen de muter la variable ; il ne peut être réaffecté qu’en laissant la valeur d’origine inchangée. Par conséquent, les suivants sont garantis d’avoir la même valeur par défaut. Cependant, pour un type mutable, la valeur d’origine peut muter, en faisant des appels à ses différentes fonctions membres. Par conséquent, il n’est pas garanti que les appels successifs à la fonction aient la valeur par défaut initiale.

def append(elem, to=[]):
    to.append(elem)      # This call to append() mutates the default variable "to"
    return to

append(1)
# Out: [1]

append(2)  # Appends it to the internally stored list
# Out: [1, 2]

append(3, [])  # Using a new created list gives the expected result
# Out: [3]

# Calling it again without argument will append to the internally stored list again
append(4)   
# Out: [1, 2, 4]

Remarque : Certains IDE comme PyCharm émettent un avertissement lorsqu’un type mutable est spécifié comme attribut par défaut.

La solution #

Si vous voulez vous assurer que l’argument par défaut est toujours celui que vous spécifiez dans la définition de la fonction, la solution consiste à toujours utiliser un type immuable comme argument par défaut.

Un idiome courant pour y parvenir lorsqu’un type mutable est nécessaire par défaut consiste à utiliser Aucun (immuable) comme argument par défaut, puis à attribuer la valeur par défaut réelle à la variable d’argument si elle est égale à Aucun.

def append(elem, to=None):
    if to is None:
        to = []

    to.append(elem)
    return to

[1] : https://www.wikiod.com/fr/python/les-fonctions#Passage d’arguments et mutabilité

Passage d’arguments et mutabilité #

Tout d’abord, un peu de terminologie :

  • argument (paramètre réel) : la variable réelle passée à une fonction ;
  • paramètre (paramètre formel) : la variable de réception utilisée dans une fonction.

En Python, les arguments sont passés par assignment (contrairement aux autres langages, où les arguments peuvent être passés par valeur/référence/pointeur).

  • La mutation d’un paramètre entraînera la mutation de l’argument (si le type de l’argument est mutable).

    def foo(x):        # here x is the parameter
    x[0] = 9       # This mutates the list labelled by both x and y
    print(x)

y = [4, 5, 6]
foo(y)             # call foo with y as argument
# Out: [9, 5, 6]   # list labelled by x has been mutated
print(y)           
# Out: [9, 5, 6]   # list labelled by y has been mutated too

  • La réaffectation du paramètre ne réaffecte pas l’argument.

    def foo(x):        # here x is the parameter, when we call foo(y) we assign y to x
    x[0] = 9       # This mutates the list labelled by both x and y
    x = [1, 2, 3]  # x is now labeling a different list (y is unaffected)
    x[2] = 8       # This mutates x's list, not y's list

y = [4, 5, 6]      # y is the argument, x is the parameter
foo(y)             # Pretend that we wrote "x = y", then go to line 1
y
# Out: [9, 5, 6]

En Python, nous n’attribuons pas vraiment de valeurs aux variables, à la place nous lions (c’est-à-dire affectons, attachons) des variables (considérées comme des noms) aux objets.

  • Immuable : Entiers, chaînes, tuples, etc. Toutes les opérations font des copies.

  • Mutable : Listes, dictionnaires, ensembles, etc. Les opérations peuvent ou non muter.

    x = [3, 1, 9] y = x x.append(5) # Mutates the list labelled by x and y, both x and y are bound to [3, 1, 9] x.sort() # Mutates the list labelled by x and y (in-place sorting) x = x + [4] # Does not mutate the list (makes a copy for x only, not y) z = x # z is x ([1, 3, 9, 4]) x += [6] # Mutates the list labelled by both x and z (uses the extend function). x = sorted(x) # Does not mutate the list (makes a copy for x only). x

    Out: [1, 3, 4, 5, 6, 9] #

    y

    Out: [1, 3, 5, 9] #

    z

    Out: [1, 3, 5, 9, 4, 6] #

Renvoyer les valeurs des fonctions #

Les fonctions peuvent “renvoyer” une valeur que vous pouvez utiliser directement :

def give_me_five():
    return 5

print(give_me_five())  # Print the returned value
# Out: 5

ou enregistrez la valeur pour une utilisation ultérieure :

num = give_me_five()
print(num)             # Print the saved returned value
# Out: 5

ou utilisez la valeur pour toutes les opérations :

print(give_me_five() + 10)
# Out: 15

Si return est rencontré dans la fonction, la fonction sera quittée immédiatement et les opérations suivantes ne seront pas évaluées :

def give_me_another_five():
    return 5
    print('This statement will not be printed. Ever.')

print(give_me_another_five())
# Out: 5

Vous pouvez également “retourner” plusieurs valeurs (sous la forme d’un tuple) :

def give_me_two_fives():
    return 5, 5  # Returns two 5

first, second = give_me_two_fives()
print(first)
# Out: 5
print(second)
# Out: 5

Une fonction avec l’instruction no return renvoie implicitement None. De même, une fonction avec une instruction return, mais sans valeur de retour ni variable renvoie None.

Fermeture #

Les fermetures en Python sont créées par des appels de fonction. Ici, l’appel à makeInc crée une liaison pour x qui est référencée dans la fonction inc. Chaque appel à makeInc crée une nouvelle instance de cette fonction, mais chaque instance a un lien vers une liaison différente de x.

def makeInc(x):
  def inc(y):
     # x is "attached" in the definition of inc
     return y + x

  return inc

incOne = makeInc(1)
incFive = makeInc(5)

incOne(5) # returns 6
incFive(5) # returns 10

Notez que, bien que dans une fermeture normale, la fonction fermée hérite entièrement de toutes les variables de son environnement englobant, dans cette construction, la fonction fermée n’a qu’un accès en lecture aux variables héritées mais ne peut pas leur faire d’affectations

def makeInc(x):
  def inc(y):
     # incrementing x is not allowed
     x += y  
     return x

  return inc

incOne = makeInc(1)
incOne(5) # UnboundLocalError: local variable 'x' referenced before assignment

Python 3 propose l’instruction nonlocal (https://www.wikiod.com/fr/python/portee-variable-et-liaison#Variables non locales ) pour réaliser une fermeture complète avec des fonctions imbriquées.

def makeInc(x):
  def inc(y):
     nonlocal x
     # now assigning a value to x is allowed
     x += y  
     return x

  return inc

incOne = makeInc(1)
incOne(5) # returns 6

Forcer l’utilisation de paramètres nommés #

Tous les paramètres spécifiés après le premier astérisque dans la signature de la fonction sont des mots-clés uniquement.

def f(*a, b):
    pass

f(1, 2, 3)
# TypeError: f() missing 1 required keyword-only argument: 'b'

En Python 3, il est possible de mettre un seul astérisque dans la signature de la fonction pour s’assurer que les arguments restants ne peuvent être passés qu’à l’aide d’arguments de mots clés.

def f(a, b, *, c):
    pass

f(1, 2, 3)
# TypeError: f() takes 2 positional arguments but 3 were given
f(1, 2, c=3)
# No error

Définir une fonction avec des arguments #

Les arguments sont définis entre parenthèses après le nom de la fonction :

def divide(dividend, divisor):  # The names of the function and its arguments
    # The arguments are available by name in the body of the function
    print(dividend / divisor)

Le nom de la fonction et sa liste d’arguments sont appelés la signature de la fonction. Chaque argument nommé est effectivement une variable locale de la fonction.

Lors de l’appel de la fonction, donnez des valeurs aux arguments en les listant dans l’ordre

divide(10, 2)
# output: 5

ou spécifiez-les dans n’importe quel ordre en utilisant les noms de la définition de fonction :

divide(divisor=2, dividend=10)
# output: 5

Fonctions récursives #

Une fonction récursive est une fonction qui s’appelle elle-même dans sa définition. Par exemple la fonction mathématique, factorielle, définie par factorial(n) = n*(n-1)*(n-2)*...*3*2*1. peut être programmé comme

def factorial(n):
    #n here should be an integer
    if n == 0:
        return 1
    else:
        return n*factorial(n-1)

les sorties ici sont:

factorial(0)
#out 1
factorial(1)
#out 1
factorial(2)
#out 2
factorial(3)
#out 6

comme prévu. Notez que cette fonction est récursive car la seconde return factorial(n-1), où la fonction s’appelle elle-même dans sa définition.

Certaines fonctions récursives peuvent être implémentées en utilisant [lambda][1], la fonction factorielle utilisant lambda ressemblerait à ceci :

factorial = lambda n: 1 if n == 0 else n*factorial(n-1)

La fonction affiche la même chose que ci-dessus.

[1] : https://www.wikiod.com/fr/python/les-fonctions#Fonctions Lambda (en ligne/anonymes)

Limite de récursivité #

Il existe une limite à la profondeur de la récursivité possible, qui dépend de l’implémentation Python. Lorsque la limite est atteinte, une exception RuntimeError est levée :

def cursing(depth):
  try:
    cursing(depth + 1) # actually, re-cursing
  except RuntimeError as RE:
    print('I recursed {} times!'.format(depth))

cursing(0)
# Out: I recursed 1083 times!

Il est possible de changer la limite de profondeur de récursion en utilisant sys.setrecursionlimit(limit) et de vérifier cette limite avec sys.getrecursionlimit().

sys.setrecursionlimit(2000)
cursing(0)
# Out: I recursed 1997 times!

À partir de Python 3.5, l’exception est une RecursionError, dérivée de RuntimeError.

Fonctions imbriquées #

Les fonctions en python sont des objets de première classe. Ils peuvent être définis dans n’importe quelle portée

def fibonacci(n):
    def step(a,b):
        return b, a+b
    a, b = 0, 1
    for i in range(n):
        a, b = step(a, b)
    return a

Les fonctions capturent leur portée englobante et peuvent être transmises comme n’importe quel autre type d’objet

def make_adder(n):
    def adder(x):
        return n + x
    return adder
add5 = make_adder(5)
add6 = make_adder(6)
add5(10)
#Out: 15
add6(10)
#Out: 16

def repeatedly_apply(func, n, x):
    for i in range(n):
        x = func(x)
    return x

repeatedly_apply(add5, 5, 1)
#Out: 26

Définir une fonction avec plusieurs arguments #

On peut donner à une fonction autant d’arguments que l’on veut, les seules règles fixes sont que chaque nom d’argument doit être unique et que les arguments optionnels doivent être après ceux qui ne sont pas optionnels :

def func(value1, value2, optionalvalue=10):
    return '{0} {1} {2}'.format(value1, value2, optionalvalue1)

Lors de l’appel de la fonction, vous pouvez soit donner chaque mot-clé sans le nom, mais l’ordre compte :

print(func(1, 'a', 100))
# Out: 1 a 100

print(func('abc', 14))
# abc 14 10

Ou combinez en donnant les arguments avec nom et sans. Ensuite, ceux qui ont un nom doivent suivre ceux qui n’en ont pas mais l’ordre de ceux qui ont un nom n’a pas d’importance :

print(func('This', optionalvalue='StackOverflow Documentation', value2='is'))
# Out: This is StackOverflow Documentation
