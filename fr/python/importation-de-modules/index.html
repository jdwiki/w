Syntaxe #

  • importer nom_du_module
  • importer module_name.submodule_name
  • depuis l’importation de nom_du_module *
  • from nom_module import nom_sous-module [, nom_classe, nom_fonction, …etc]
  • depuis nom_du_module importez un_nom en tant que nouveau_nom
  • from nom_module.nom_sous-module import nom_classe [, nom_fonction, …etc]

L’importation d’un module obligera Python à évaluer tout le code de niveau supérieur de ce module afin qu’il apprenne toutes les fonctions, classes et variables que contient le module. Lorsque vous souhaitez qu’un de vos modules soit importé ailleurs, soyez prudent avec votre code de niveau supérieur et encapsulez-le dans if __name__ == '__main__': si vous ne voulez pas qu’il soit exécuté lorsque le module obtient importé.

La variable spéciale all #

Les modules peuvent avoir une variable spéciale nommée __all__ pour limiter les variables importées lors de l’utilisation de from mymodule import *.

Soit le module suivant :

# mymodule.py

__all__ = ['imported_by_star']

imported_by_star = 42
not_imported_by_star = 21

Seul imported_by_star est importé lors de l’utilisation de from mymodule import * :

>>> from mymodule import *
>>> imported_by_star
42
>>> not_imported_by_star
Traceback (most recent call last):
  File "<stdin>", line 1, in <module>
NameError: name 'not_imported_by_star' is not defined

Cependant, not_imported_by_star peut être importé explicitement :

>>> from mymodule import not_imported_by_star
>>> not_imported_by_star
21

Importation d’un module #

Utilisez l’instruction import :

>>> import random
>>> print(random.randint(1, 10))
4

import module importera un module et vous permettra ensuite de référencer ses objets – valeurs, fonctions et classes, par exemple – en utilisant la syntaxe module.name. Dans l’exemple ci-dessus, le module random est importé, qui contient la fonction randint. Ainsi, en important random, vous pouvez appeler randint avec random.randint.

Vous pouvez importer un module et lui attribuer un nom différent :

>>> import random as rn
>>> print(rn.randint(1, 10))
4

Si votre fichier python main.py se trouve dans le même dossier que custom.py. Vous pouvez l’importer comme ceci :

import custom

Il est aussi possible d’importer une fonction depuis un module :

>>> from math import sin
>>> sin(1)
0.8414709848078965

Pour importer des fonctions spécifiques plus profondément dans un module, l’opérateur point peut être utilisé uniquement sur le côté gauche du mot-clé import :

from urllib.request import urlopen

En python, nous avons deux façons d’appeler une fonction depuis le niveau supérieur. L’un est « importer » et l’autre est « de ». Nous devrions utiliser import lorsque nous avons une possibilité de collision de noms. Supposons que nous ayons le fichier hello.py et les fichiers world.py ayant la même fonction nommée function. Ensuite, l’instruction “import” fonctionnera bien.

from hello import function
from world import function

function() #world's function will be invoked. Not hello's

En général, “import” vous fournira un espace de noms.

import hello
import world

hello.function() # exclusively hello's function will be invoked 
world.function() # exclusively world's function will be invoked

Mais si vous êtes assez sûr, dans l’ensemble de votre projet, il n’y a aucun moyen d’avoir le même nom de fonction, vous devez utiliser l’instruction from

Plusieurs importations peuvent être effectuées sur la même ligne :

>>> # Multiple modules
>>> import time, sockets, random
>>> # Multiple functions
>>> from math import sin, cos, tan
>>> # Multiple constants
>>> from math import pi, e

>>> print(pi)
3.141592653589793
>>> print(cos(45))
0.5253219888177297
>>> print(time.time())
1482807222.7240417

Les mots clés et la syntaxe indiqués ci-dessus peuvent également être utilisés en combinaison :

>>> from urllib.request import urlopen as geturl, pathname2url as path2url, getproxies
>>> from math import factorial as fact, gamma, atan as arctan
>>> import random.randint, time, sys

>>> print(time.time())
1482807222.7240417
>>> print(arctan(60))
1.554131203080956
>>> filepath = "/dogs/jumping poodle (december).png"
>>> print(path2url(filepath))
/dogs/jumping%20poodle%20%28december%29.png

Importer des modules à partir d’un emplacement arbitraire du système de fichiers #

Si vous souhaitez importer un module qui n’existe pas déjà en tant que module intégré dans la [Python Standard Library] [1] ni en tant que package secondaire, vous pouvez le faire en ajoutant le chemin d’accès au répertoire où votre module se trouve dans [sys.path][2]. Cela peut être utile lorsque plusieurs environnements python existent sur un hôte.

import sys
sys.path.append("/path/to/directory/containing/your/module")
import mymodule

Il est important que vous ajoutiez le chemin vers le répertoire dans lequel se trouve mymodule, et non le chemin vers le module lui-même.

[1] : https://docs.python.org/3/library/ [2] : https://docs.python.org/3/library/sys.html#sys.path

Importer tous les noms d’un module #

from module_name import *

par exemple:

from math import *
sqrt(2)    # instead of math.sqrt(2)
ceil(2.7)  # instead of math.ceil(2.7)

Cela importera tous les noms définis dans le module math dans l’espace de noms global, à l’exception des noms commençant par un trait de soulignement (ce qui indique que l’auteur estime qu’il s’agit d’un usage interne uniquement).

Attention : Si une fonction portant le même nom a déjà été définie ou importée, elle sera écrasée. Importer presque toujours uniquement des noms spécifiques from math import sqrt, ceil est la méthode recommandée :

def sqrt(num):
    print("I don't know what's the square root of {}.".format(num))

sqrt(4)
# Output: I don't know what's the square root of 4.

from math import * 
sqrt(4)
# Output: 2.0

Les importations favorites ne sont autorisées qu’au niveau du module. Les tentatives de les exécuter dans les définitions de classe ou de fonction entraînent une SyntaxError.

def f():
    from math import *

et

class A:
    from math import *

les deux échouent avec:

SyntaxError: import * only allowed at module level

Importation programmatique #

Pour importer un module via un appel de fonction, utilisez le module importlib (inclus dans Python à partir de la version 2.7) :

import importlib
random = importlib.import_module("random")

La fonction importlib.import_module() importera également directement le sous-module d’un package :

collections_abc = importlib.import_module("collections.abc")

Pour les anciennes versions de Python, utilisez le module imp.

Utilisez les fonctions imp.find_module et imp.load_module pour effectuer une importation par programme.

Tiré de [documentation standard de la bibliothèque][1]

import imp, sys
def import_module(name):
    fp, pathname, description = imp.find_module(name)
    try:
        return imp.load_module(name, fp, pathname, description)
    finally:
        if fp:
            fp.close()

N’utilisez PAS __import__() pour importer des modules par programmation ! Il y a des détails subtils impliquant sys.modules, l’argument fromlist, etc.

[1] : https://docs.python.org/2/library/imp.html#examples

Règles PEP8 pour les importations #

Quelques directives de style PEP8 recommandées pour les importations :

  1. Les importations doivent figurer sur des lignes distinctes :

     from math import sqrt, ceil      # Not recommended
 from math import sqrt            # Recommended
 from math import ceil

  2. Commandez les importations comme suit en haut du module :

    • Standard library imports
    • Related third party imports
    • Local application/library specific imports

  3. Les importations de caractères génériques doivent être évitées car elles entraînent une confusion dans les noms dans l’espace de noms actuel. Si vous faites from module import *, il peut être difficile de savoir si un nom spécifique dans votre code provient de module ou non. Ceci est doublement vrai si vous avez plusieurs instructions de type from module import *.

  4. Éviter d’utiliser les importations relatives ; utilisez plutôt des importations explicites.

Importer des noms spécifiques depuis un module #

Au lieu d’importer le module complet, vous pouvez importer uniquement les noms spécifiés :

from random import randint # Syntax "from MODULENAME import NAME1[, NAME2[, ...]]"
print(randint(1, 10))      # Out: 5

from random est nécessaire, car l’interpréteur python doit savoir à partir de quelle ressource il doit importer une fonction ou une classe et import randint spécifie la fonction ou la classe elle-même.

Un autre exemple ci-dessous (similaire à celui ci-dessus):

from math import pi
print(pi)                  # Out: 3.14159265359

L’exemple suivant générera une erreur, car nous n’avons pas importé de module :

random.randrange(1, 10)    # works only if "import random" has been run before

Les sorties:

NameError: name 'random' is not defined

L’interpréteur python ne comprend pas ce que vous entendez par “aléatoire”. Il doit être déclaré en ajoutant “import random” à l’exemple :

import random
random.randrange(1, 10)

Importation de sous-modules #

from module.submodule import function

Ceci importe function de module.submodule.

Fonction import() #

La fonction __import__() peut être utilisée pour importer des modules dont le nom n’est connu qu’au moment de l’exécution

if user_input == "os":
    os = __import__("os")

# equivalent to import os

Cette fonction peut également être utilisée pour spécifier le chemin du fichier vers un module

mod = __import__(r"C:/path/to/file/anywhere/on/computer/module.py")

Réimporter un module #

Lorsque vous utilisez l’interpréteur interactif, vous souhaiterez peut-être recharger un module. Cela peut être utile si vous éditez un module et souhaitez importer la version la plus récente, ou si vous avez patché un élément d’un module existant et souhaitez annuler vos modifications.

Notez que vous ** ne pouvez pas ** simplement “importer” à nouveau le module pour revenir en arrière :

import math
math.pi = 3
print(math.pi)    # 3
import math
print(math.pi)    # 3

En effet, l’interpréteur enregistre chaque module que vous importez. Et lorsque vous essayez de réimporter un module, l’interpréteur le voit dans le registre et ne fait rien. Ainsi, la manière la plus difficile de réimporter consiste à utiliser import après avoir supprimé l’élément correspondant du registre :

print(math.pi)    # 3
import sys
if 'math' in sys.modules:  # Is the ``math`` module in the register?
    del sys.modules['math']  # If so, remove it.
import math
print(math.pi)    # 3.141592653589793

Mais il y a plus un moyen simple et direct.

##Python2 #

Utilisez la fonction “recharger” :

import math
math.pi = 3
print(math.pi)    # 3
reload(math)
print(math.pi)    # 3.141592653589793

##Python 3 #

La fonction reload a été déplacée vers importlib :

import math
math.pi = 3
print(math.pi)    # 3
from importlib import reload
reload(math)
print(math.pi)    # 3.141592653589793
