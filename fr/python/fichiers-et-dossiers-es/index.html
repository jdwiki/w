Lorsqu’il s’agit de stocker, de lire ou de communiquer des données, travailler avec les fichiers d’un système d’exploitation est à la fois nécessaire et facile avec Python. Contrairement à d’autres langages où l’entrée et la sortie de fichiers nécessitent des objets de lecture et d’écriture complexes, Python simplifie le processus en ne nécessitant que des commandes pour ouvrir, lire/écrire et fermer le fichier. Cette rubrique explique comment Python peut s’interfacer avec des fichiers sur le système d’exploitation.

Syntaxe #

  • file_object = open(filename [, access_mode][, buffering])

Paramètres #

Paramètre Détails
nom de fichier le chemin d’accès à votre fichier ou, si le fichier se trouve dans le répertoire de travail, le nom de votre fichier
mode_accès une valeur de chaîne qui détermine comment le fichier est ouvert
mise en mémoire tampon une valeur entière utilisée pour la mise en mémoire tampon de ligne facultative

Éviter l’enfer de l’encodage multiplateforme #

Lors de l’utilisation de open() intégré à Python, il est recommandé de toujours passer l’argument encoding, si vous souhaitez que votre code soit exécuté sur plusieurs plates-formes. La raison en est que l’encodage par défaut d’un système diffère d’une plate-forme à l’autre.

Alors que les systèmes linux utilisent effectivement utf-8 par défaut, ce n’est pas nécessairement vrai pour MAC et Windows.

Pour vérifier l’encodage par défaut d’un système, essayez ceci :

import sys
sys.getdefaultencoding()

depuis n’importe quel interpréteur Python.

Par conséquent, il est sage de toujours spécifier un encodage, pour vous assurer que les chaînes avec lesquelles vous travaillez sont encodées comme vous le pensez, garantissant ainsi la compatibilité multiplateforme.

with open('somefile.txt', 'r', encoding='UTF-8') as f:
    for line in f:
        print(line)

Modes de fichier #

Il existe différents modes avec lesquels vous pouvez ouvrir un fichier, spécifiés par le paramètre mode. Ceux-ci inclus:

  • 'r' - mode de lecture. Le défaut. Il vous permet uniquement de lire le fichier, de ne pas le modifier. Lorsque vous utilisez ce mode, le fichier doit exister.

  • 'w' - mode d’écriture. Il créera un nouveau fichier s’il n’existe pas, sinon il effacera le fichier et vous permettra d’y écrire.

  • 'a' - mode d’ajout. Il écrira des données à la fin du fichier. Il n’efface pas le fichier, et le fichier doit exister pour ce mode.

  • 'rb' - mode de lecture en binaire. Ceci est similaire à r sauf que la lecture est forcée en mode binaire. C’est aussi un choix par défaut.

  • 'r+' - mode lecture plus mode écriture en même temps. Cela vous permet de lire et d’écrire dans des fichiers en même temps sans avoir à utiliser r et w.

  • 'rb+' - mode lecture et écriture en binaire. Identique à r+ sauf que les données sont en binaire

  • 'wb' - mode d’écriture en binaire. Identique à w sauf que les données sont en binaire.

  • 'w+' - mode d’écriture et de lecture. Exactement la même chose que r+ mais si le fichier n’existe pas, un nouveau est créé. Sinon, le fichier est écrasé.

  • 'wb+' - mode d’écriture et de lecture en mode binaire. Identique à w+ mais les données sont en binaire.

  • 'ab' - ajout en mode binaire. Similaire à ‘a’ sauf que les données sont en binaire.

  • 'a+' - mode d’ajout et de lecture. Semblable à w+ car il créera un nouveau fichier si le fichier n’existe pas. Sinon, le pointeur de fichier est à la fin du fichier s’il existe.

  • 'ab+' - mode d’ajout et de lecture en binaire. Identique à a+ sauf que les données sont en binaire.

      with open(filename, 'r') as f:
      f.read()
  with open(filename, 'w') as f:
      f.write(filedata)
  with open(filename, 'a') as f:
      f.write('\n' + newdata)

Python 3 a ajouté un nouveau mode de création exclusive afin que vous ne tronquiez ou n’écrasiez pas accidentellement un fichier existant.

  • 'x' - ouvert pour la création exclusive, déclenchera FileExistsError si le fichier existe déjà
  • 'xb' - ouvert pour un mode d’écriture de création exclusif en binaire. Identique à x sauf que les données sont en binaire.

Lire un fichier ligne par ligne #

La manière la plus simple de parcourir un fichier ligne par ligne :

with open('myfile.txt', 'r') as fp:
    for line in fp:
        print(line)

readline() permet un contrôle plus granulaire sur l’itération ligne par ligne. L’exemple ci-dessous est équivalent à celui ci-dessus :

with open('myfile.txt', 'r') as fp:
    while True:
        cur_line = fp.readline()
        # If the result is an empty string
        if cur_line == '':
            # We have reached the end of the file
            break
        print(cur_line)

L’utilisation conjointe de l’itérateur de boucle for et de readline() est considérée comme une mauvaise pratique.

Plus communément, la méthode readlines() est utilisée pour stocker une collection itérable des lignes du fichier :

with open("myfile.txt", "r") as fp:
    lines = fp.readlines()
for i in range(len(lines)):
    print("Line " + str(i) + ": " + line)

Cela imprimerait ce qui suit:

Ligne 0 : bonjour

Ligne 1 : monde

Itérer les fichiers (récursivement) #

Pour itérer tous les fichiers, y compris dans les sous-répertoires, utilisez os.walk :

import os
for root, folders, files in os.walk(root_dir):
    for filename in files:
        print root, filename

root_dir peut être “.” pour démarrer à partir du répertoire courant, ou tout autre chemin à partir duquel démarrer.

Si vous souhaitez également obtenir des informations sur le fichier, vous pouvez utiliser la méthode plus efficace os.scandir comme suit :

for entry in os.scandir(path):
   if not entry.name.startswith('.') and entry.is_file():
       print(entry.name)

Obtenir le contenu complet d’un fichier #

La méthode préférée d’E/S de fichier consiste à utiliser le mot-clé with. Cela garantira que le descripteur de fichier est fermé une fois la lecture ou l’écriture terminée.

with open('myfile.txt') as in_file:
    content = in_file.read()

print(content)

ou, pour gérer la fermeture manuelle du fichier, vous pouvez renoncer à with et simplement appeler close vous-même :

in_file = open('myfile.txt', 'r')
content = in_file.read()
print(content)
in_file.close()

Gardez à l’esprit que sans utiliser d’instruction with, vous pourriez accidentellement garder le fichier ouvert au cas où une exception inattendue surviendrait comme ceci :

in_file = open('myfile.txt', 'r')
raise Exception("oops")
in_file.close()  # This will never be called

with open('myfile.txt', 'w') as f:
    f.write("Line 1")
    f.write("Line 2")
    f.write("Line 3")
    f.write("Line 4")

Si vous ouvrez myfile.txt, vous verrez que son contenu est :

Python n’ajoute pas automatiquement de sauts de ligne, vous devez le faire manuellement :

with open('myfile.txt', 'w') as f:
    f.write("Line 1\n")
    f.write("Line 2\n")
    f.write("Line 3\n")
    f.write("Line 4\n")

Ligne 1 Ligne 2 Ligne 3 Ligne 4

N’utilisez pas os.linesep comme fin de ligne lors de l’écriture de fichiers ouverts en mode texte (par défaut) ; utilisez \n à la place.

Si vous souhaitez spécifier un encodage, vous ajoutez simplement le paramètre encoding à la fonction open :

with open('my_file.txt', 'w', encoding='utf-8') as f:
    f.write('utf-8 text')

Il est également possible d’utiliser l’instruction print pour écrire dans un fichier. Les mécanismes sont différents dans Python 2 et Python 3, mais le concept est le même en ce sens que vous pouvez prendre la sortie qui serait allée à l’écran et l’envoyer dans un fichier à la place.

with open('fred.txt', 'w') as outfile:
    s = "I'm Not Dead Yet!"
    print(s) # writes to stdout
    print(s, file = outfile) # writes to outfile

    #Note: it is possible to specify the file parameter AND write to the screen
    #by making sure file ends up with a None value either directly or via a variable
    myfile = None
    print(s, file = myfile) # writes to stdout
    print(s, file = None)   # writes to stdout

En Python 2, vous auriez fait quelque chose comme

outfile = open('fred.txt', 'w')
s = "I'm Not Dead Yet!"
print s   # writes to stdout
print >> outfile, s   # writes to outfile

Contrairement à l’utilisation de la fonction d’écriture, la fonction d’impression ajoute automatiquement des sauts de ligne.

Vérifier si un fichier ou un chemin existe #

Utilisez le style de codage EAFP et “essayez” de l’ouvrir.

import errno

try:
    with open(path) as f:
        # File exists
except IOError as e:
    # Raise the exception if it is not ENOENT (No such file or directory)
    if e.errno != errno.ENOENT:
        raise
    # No such file or directory

Cela évitera également les conditions de concurrence si un autre processus supprimait le fichier entre la vérification et son utilisation. Cette condition de concurrence peut se produire dans les cas suivants :

  • En utilisant le module os :

    import os
os.path.isfile('/path/to/some/file.txt')

  • Utilisation de pathlib :

    import pathlib
path = pathlib.Path('/path/to/some/file.txt')
if path.is_file():
    ...

Pour vérifier si un chemin donné existe ou non, vous pouvez suivre la procédure EAFP ci-dessus, ou vérifier explicitement le chemin :

import os
path = "/home/myFiles/directory1"

if os.path.exists(path):
    ## Do stuff

Accès aléatoire aux fichiers à l’aide de mmap #

L’utilisation du module mmap permet à l’utilisateur d’accéder de manière aléatoire à des emplacements dans un fichier en mappant le fichier dans la mémoire. Il s’agit d’une alternative à l’utilisation des opérations de fichiers normales.

import mmap

with open('filename.ext', 'r') as fd:
    # 0: map the whole file
    mm = mmap.mmap(fd.fileno(), 0)

    # print characters at indices 5 through 10
    print mm[5:10]

    # print the line starting from mm's current position
    print mm.readline()

    # write a character to the 5th index
    mm[5] = 'a'

    # return mm's position to the beginning of the file
    mm.seek(0)

    # close the mmap object
    mm.close()

Lire un fichier entre une plage de lignes #

Supposons donc que vous souhaitiez parcourir uniquement certaines lignes spécifiques d’un fichier

Vous pouvez utiliser itertools pour cela

import itertools

with open('myfile.txt', 'r') as f:
    for line in itertools.islice(f, 12, 30):
        # do something here

Cela lira les lignes 13 à 20 comme dans python l’indexation commence à partir de 0. Donc la ligne numéro 1 est indexée comme 0

Comme vous pouvez également lire quelques lignes supplémentaires en utilisant le mot-clé next() ici.

Et lorsque vous utilisez l’objet fichier en tant qu’itérable, veuillez ne pas utiliser l’instruction readline() ici car les deux techniques de parcours d’un fichier ne doivent pas être mélangées

Vérifier si un fichier est vide #

>>> import os
>>> os.stat(path_to_file).st_size == 0

ou

>>> import os    
>>> os.path.getsize(path_to_file) > 0

Cependant, les deux lèveront une exception si le fichier n’existe pas. Pour éviter d’avoir à attraper une telle erreur, procédez comme suit :

import os
def is_empty_file(fpath):  
    return os.path.isfile(fpath) and os.path.getsize(fpath) > 0

qui renverra une valeur bool.
