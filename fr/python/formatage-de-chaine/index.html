Lors du stockage et de la transformation de données visibles par les humains, le formatage des chaînes peut devenir très important. Python offre une grande variété de méthodes de formatage de chaîne décrites dans cette rubrique.

Syntaxe #

  • “{}".format(42) ==> “42”

  • “{0}".format(42) ==> “42”

  • “{0:.2f}".format(42) ==> “42.00”

  • “{0:.0f}".format(42.1234) ==> “42”

  • “{answer}".format(no_answer=41, answer=42) ==> “42”

  • “{answer:.2f}".format(no_answer=41, answer=42) ==> “42.00”

  • “{[clé]}".format({‘clé’ : ‘valeur’}) ==> “valeur”

  • “{[1]}".format([‘zéro’, ‘un’, ‘deux’]) ==> “un”

  • “{réponse} = {réponse}".format(réponse=42) ==> “42 = 42”

  • ’ ‘.join([‘pile’, ‘débordement’]) ==> “débordement de pile”

  • Devrait consulter PyFormat.info pour une introduction/explication très approfondie et douce de son fonctionnement.

Principes de base du formatage de chaînes #

foo = 1
bar = 'bar'
baz = 3.14

Vous pouvez utiliser str.format pour formater la sortie. Les paires de parenthèses sont remplacées par des arguments dans l’ordre dans lequel les arguments sont passés :

print('{}, {} and {}'.format(foo, bar, baz))
# Out: "1, bar and 3.14"

Les index peuvent également être spécifiés à l’intérieur des crochets. Les nombres correspondent aux index des arguments passés à la fonction str.format (basé sur 0).

print('{0}, {1}, {2}, and {1}'.format(foo, bar, baz))
# Out: "1, bar, 3.14, and bar"
print('{0}, {1}, {2}, and {3}'.format(foo, bar, baz))
# Out: index out of range error

Les arguments nommés peuvent également être utilisés :

print("X value is: {x_val}. Y value is: {y_val}.".format(x_val=2, y_val=3))
# Out: "X value is: 2. Y value is: 3."

Les attributs d’objet peuvent être référencés lorsqu’ils sont passés dans str.format :

class AssignValue(object):
    def __init__(self, value):
        self.value = value
my_value = AssignValue(6)
print('My value is: {0.value}'.format(my_value))  # "0" is optional
# Out: "My value is: 6"

Les clés de dictionnaire peuvent également être utilisées :

my_dict = {'key': 6, 'other_key': 7}
print("My other key is: {0[other_key]}".format(my_dict))  # "0" is optional
# Out: "My other key is: 7"

La même chose s’applique aux indices de liste et de tuple :

my_list = ['zero', 'one', 'two']
print("2nd element is: {0[2]}".format(my_list))  # "0" is optional
# Out: "2nd element is: two"

Remarque : En plus de str.format, Python fournit également l’opérateur modulo %–également appelé string formatting ou interpolation operator (voir [PEP 3101][1])–pour formater les chaînes. str.format est un successeur de % et offre une plus grande flexibilité, par exemple en facilitant les substitutions multiples.

En plus des index d’arguments, vous pouvez également inclure une spécification de format à l’intérieur des accolades. Il s’agit d’une expression qui suit des règles spéciales et doit être précédée de deux-points (:). Voir les [docs][2] pour une description complète de la spécification du format. Un exemple de spécification de format est la directive d’alignement :~^20 (^ signifie alignement central, largeur totale 20, remplissage avec le caractère ~) :

'{:~^20}'.format('centered')
# Out: '~~~~~~centered~~~~~~'

format autorise un comportement impossible avec %, par exemple la répétition d’arguments :

t = (12, 45, 22222, 103, 6)
print '{0} {2} {1} {2} {3} {2} {4} {2}'.format(*t)    
# Out: 12 22222 45 22222 103 22222 6 22222

Comme format est une fonction, il peut être utilisé comme argument dans d’autres fonctions :

number_list = [12,45,78]
print map('the number is {}'.format, number_list)
# Out: ['the number is 12', 'the number is 45', 'the number is 78']   
    

from datetime import datetime,timedelta
    
once_upon_a_time = datetime(2010, 7, 1, 12, 0, 0)
delta = timedelta(days=13, hours=8,  minutes=20)
    
gen = (once_upon_a_time + x * delta for x in xrange(5))
    
print '\n'.join(map('{:%Y-%m-%d %H:%M:%S}'.format, gen))
#Out: 2010-07-01 12:00:00
#     2010-07-14 20:20:00
#     2010-07-28 04:40:00
#     2010-08-10 13:00:00
#     2010-08-23 21:20:00

[1] : https://www.python.org/dev/peps/pep-3101/ [2] : https://docs.python.org/2/library/string.html#format-specification-mini-language

Alignement et rembourrage #

La méthode format() peut être utilisée pour modifier l’alignement de la chaîne. Vous devez le faire avec une expression de format de la forme :[fill_char][align_operator][width] où align_operator est l’un des suivants :

  • < force le champ à être aligné à gauche dans width.
  • > force le champ à être aligné à droite dans width.
  • ^ force le champ à être centré dans width.
  • = force le rembourrage à être placé après le signe (types numériques uniquement).

fill_char (si omis, la valeur par défaut est un espace) est le caractère utilisé pour le remplissage.

'{:~<9s}, World'.format('Hello')
# 'Hello~~~~, World'

'{:~>9s}, World'.format('Hello')
# '~~~~Hello, World'

'{:~^9s}'.format('Hello')
# '~~Hello~~'

'{:0=6d}'.format(-123)
# '-00123'

Remarque : vous pouvez obtenir les mêmes résultats en utilisant les fonctions de chaîne ljust(), rjust(), center(), zfill(), mais ces fonctions sont obsolètes depuis la version 2.5.

Littéraux de format (chaîne f) #

Les chaînes de format littéral ont été introduites dans PEP 498 (Python3.6 et versions ultérieures), vous permettant d’ajouter “f” au début d’un littéral de chaîne pour lui appliquer efficacement .format avec toutes les variables de la portée actuelle.

>>> foo = 'bar'
>>> f'Foo is {foo}'
'Foo is bar'

Cela fonctionne également avec des chaînes de format plus avancées, y compris l’alignement et la notation par points.

>>> f'{foo:^7s}'
'  bar  '

Remarque : Le f'' ne désigne pas un type particulier comme b'' pour bytes ou u'' pour unicode en python2. La mise en forme est immédiatement appliquée, d’où un brassage normal.

Les chaînes de format peuvent également être imbriquées :

>>> price = 478.23
>>> f"{f'${price:0.2f}':*>20s}"
'*************$478.23'

Les expressions d’une f-string sont évaluées dans l’ordre de gauche à droite. Ceci n’est détectable que si les expressions ont des effets secondaires :

>>> def fn(l, incr):
...    result = l[0]
...    l[0] += incr
...    return result
...
>>> lst = [0]
>>> f'{fn(lst,2)} {fn(lst,3)}'
'0 2'
>>> f'{fn(lst,2)} {fn(lst,3)}'
'5 7'
>>> lst
[10]

Format flottant #

>>> '{0:.0f}'.format(42.12345)
'42'

>>> '{0:.1f}'.format(42.12345)
'42.1'

>>> '{0:.3f}'.format(42.12345)
'42.123'

>>> '{0:.5f}'.format(42.12345)
'42.12345'

>>> '{0:.7f}'.format(42.12345)
'42.1234500'

Même prise pour les autres modes de référencement :

>>> '{:.3f}'.format(42.12345)
'42.123'

>>> '{answer:.3f}'.format(answer=42.12345)
'42.123'

Les nombres à virgule flottante peuvent également être formatés en [notation scientifique][1] ou en pourcentage :

>>> '{0:.3e}'.format(42.12345)
'4.212e+01'

>>> '{0:.0%}'.format(42.12345)
'4212%'

Vous pouvez également combiner les notations {0} et {nom}. Ceci est particulièrement utile lorsque vous souhaitez arrondir toutes les variables à un nombre prédéfini de décimales avec 1 déclaration :

>>> s = 'Hello'
>>> a, b, c = 1.12345, 2.34567, 34.5678
>>> digits = 2

>>> '{0}! {1:.{n}f}, {2:.{n}f}, {3:.{n}f}'.format(s, a, b, c, n=digits)
'Hello! 1.12, 2.35, 34.57'

[1] : https://en.wikipedia.org/wiki/Scientific_notation

Formatage de chaîne avec datetime #

N’importe quelle classe peut configurer sa propre syntaxe de formatage de chaîne via la méthode __format__. Un type de la bibliothèque Python standard qui en fait un usage pratique est le type datetime, où l’on peut utiliser des codes de formatage de type strftime directement dans str.format :

>>> from datetime import datetime
>>> 'North America: {dt:%m/%d/%Y}.  ISO: {dt:%Y-%m-%d}.'.format(dt=datetime.now())
'North America: 07/21/2016.  ISO: 2016-07-21.'

Une liste complète de la liste des formateurs datetime peut être trouvée dans la [documentation officielle][1].

[1] : https://docs.python.org/3/library/datetime.html#strftime-and-strptime-behavior

Espaces réservés nommés #

Les chaînes de format peuvent contenir des espaces réservés nommés qui sont interpolés à l’aide d’arguments de mot-clé pour format.

Utilisation d’un dictionnaire (Python 2.x) #

>>> data = {'first': 'Hodor', 'last': 'Hodor!'}
>>> '{first} {last}'.format(**data)
'Hodor Hodor!'

Utilisation d’un dictionnaire (Python 3.2+) #

>>> '{first} {last}'.format_map(data)
'Hodor Hodor!'

str.format_map permet d’utiliser des dictionnaires sans avoir à les décompresser au préalable. De plus, la classe de data (qui peut être un type personnalisé) est utilisée à la place d’un dict nouvellement rempli.

Sans dictionnaire : #

>>> '{first} {last}'.format(first='Hodor', last='Hodor!')
'Hodor Hodor!'

Formater avec Getitem et Getattr #

Toute structure de données prenant en charge __getitem__ peut avoir sa structure imbriquée formatée :

person = {'first': 'Arthur', 'last': 'Dent'} 
'{p[first]} {p[last]}'.format(p=person) 
# 'Arthur Dent'

Les attributs d’objet sont accessibles à l’aide de getattr() :

class Person(object):
    first = 'Zaphod'
    last = 'Beeblebrox'

'{p.first} {p.last}'.format(p=Person())
# 'Zaphod Beeblebrox'

Formatage des valeurs numériques #

La méthode .format() peut interpréter un nombre dans différents formats, tels que :

>>> '{:c}'.format(65)    # Unicode character
'A'

>>> '{:d}'.format(0x0a)  # base 10
'10'

>>> '{:n}'.format(0x0a)  # base 10 using current locale for separators
'10'

Formater des entiers dans différentes bases (hex, oct, binaire)

>>> '{0:x}'.format(10) # base 16, lowercase - Hexadecimal
'a'

>>> '{0:X}'.format(10) # base 16, uppercase - Hexadecimal
'A'

>>> '{:o}'.format(10) # base 8 - Octal
'12'

>>> '{:b}'.format(10) # base 2 - Binary
'1010'

>>> '{0:#b}, {0:#o}, {0:#x}'.format(42) # With prefix
'0b101010, 0o52, 0x2a'

>>> '8 bit: {0:08b}; Three bytes: {0:06x}'.format(42) # Add zero padding
'8 bit: 00101010; Three bytes: 00002a'

Utilisez le formatage pour convertir un tuple flottant RVB en une chaîne hexadécimale de couleur :

>>> r, g, b = (1.0, 0.4, 0.0)
>>> '#{:02X}{:02X}{:02X}'.format(int(255 * r), int(255 * g), int(255 * b))
'#FF6600'

Seuls les entiers peuvent être convertis :

>>> '{:x}'.format(42.0)
Traceback (most recent call last):
  File "<stdin>", line 1, in <module>
ValueError: Unknown format code 'x' for object of type 'float'

Formatage imbriqué #

Certains formats peuvent accepter des paramètres supplémentaires, tels que la largeur de la chaîne formatée ou l’alignement :

>>> '{:.>10}'.format('foo')
'.......foo'

Ceux-ci peuvent également être fournis en tant que paramètres de format en imbriquant plus de {} dans le {} :

>>> '{:.>{}}'.format('foo', 10)
'.......foo'
'{:{}{}{}}'.format('foo', '*', '^', 15)
'******foo******'

Dans ce dernier exemple, la chaîne de format '{:{}{}{}}' est modifiée en '{:*^15}' (c’est-à-dire “center and pad with * to total length of 15”) avant en l’appliquant à la chaîne réelle 'foo' à formater de cette façon.

Cela peut être utile dans les cas où les paramètres ne sont pas connus à l’avance, par exemple lors de l’alignement de données tabulaires :

>>> data = ["a", "bbbbbbb", "ccc"]
>>> m = max(map(len, data))
>>> for d in data:
...     print('{:>{}}'.format(d, m))
      a
bbbbbbb
    ccc

Chaînes de remplissage et de troncature, combinées #

Supposons que vous souhaitiez imprimer des variables dans une colonne de 3 caractères.

Remarque : doubler { et } les échappe.

s = """

pad
{{:3}}             :{a:3}:

truncate
{{:.3}}            :{e:.3}:

combined
{{:>3.3}}          :{a:>3.3}:
{{:3.3}}           :{a:3.3}:
{{:3.3}}           :{c:3.3}:
{{:3.3}}           :{e:3.3}:
"""

print (s.format(a="1"*1, c="3"*3, e="5"*5))

Production:

Formatage personnalisé pour une classe #

Noter:

Tout ce qui suit s’applique à la méthode str.format, ainsi qu’au fonction “formater”. Dans le texte ci-dessous, les deux sont interchangeables.

Pour chaque valeur transmise à la fonction format, Python recherche une méthode __format__ pour cet argument. Votre propre classe personnalisée peut donc avoir sa propre méthode __format__ pour déterminer comment la fonction format affichera et formatera votre classe et ses attributs.

Ceci est différent de la méthode __str__, car dans la méthode __format__, vous pouvez prendre en compte le langage de formatage, y compris l’alignement, la largeur de champ, etc., et même (si vous le souhaitez) implémenter vos propres spécificateurs de format et votre propre formatage extensions de langage.[1]

object.__format__(self, format_spec)

Par exemple :

# Example in Python 2 - but can be easily applied to Python 3

class Example(object):
    def __init__(self,a,b,c):
        self.a, self.b, self.c = a,b,c

    def __format__(self, format_spec):
        """ Implement special semantics for the 's' format specifier """
        # Reject anything that isn't an s
        if format_spec[-1] != 's':
            raise ValueError('{} format specifier not understood for this object', format_spec[:-1])

        # Output in this example will be (<a>,<b>,<c>)
        raw = "(" + ",".join([str(self.a), str(self.b), str(self.c)]) + ")"
        # Honor the format language by using the inbuilt string format
        # Since we know the original format_spec ends in an 's' 
        # we can take advantage of the str.format method with a 
        # string argument we constructed above
        return "{r:{f}}".format( r=raw, f=format_spec )

inst = Example(1,2,3)
print "{0:>20s}".format( inst )
# out :              (1,2,3)
# Note how the right align and field width of 20 has been honored.

Noter:

Si votre classe personnalisée n’a pas de méthode __format__ personnalisée et un l’instance de la classe est passée à la fonction format, Python2 utilisera toujours la valeur de retour de la méthode __str__ ou __repr__ méthode pour déterminer ce qu’il faut imprimer (et si ni l’un ni l’autre n’existe alors le repr par défaut sera utilisé), et vous devrez utiliser le format s spécificateur pour formater ceci. Avec Python3, pour réussir votre cours personnalisé à la fonction format, vous devrez définir la méthode __format__ sur votre classe personnalisée.
