Les objets String sont immuables, ce qui signifie qu’ils ne peuvent pas être modifiés sur place comme le peut une liste. Pour cette raison, les méthodes du type intégré str renvoient toujours un nouvel objet str, qui contient le résultat de l’appel de la méthode.

Changer la capitalisation d’une chaîne #

Le type de chaîne de Python fournit de nombreuses fonctions qui agissent sur la capitalisation d’une chaîne. Ceux-ci inclus :

  • str.casefold
  • str.supérieur
  • str.lower
  • str.capitalize
  • str.title
  • str.swapcase

Avec les chaînes Unicode (la valeur par défaut dans Python 3), ces opérations ne sont pas des mappages 1:1 ou réversibles. La plupart de ces opérations sont destinées à des fins d’affichage, plutôt que de normalisation.

str.casefold() #

str.casefold crée une chaîne en minuscules adaptée aux comparaisons insensibles à la casse. Ceci est plus agressif que str.lower et peut modifier des chaînes qui sont déjà en minuscules ou provoquer une augmentation de la longueur des chaînes, et n’est pas destiné à des fins d’affichage.

"XßΣ".casefold()
# 'xssσ'

"XßΣ".lower()
# 'xßς'

Les transformations qui ont lieu sous le pliage de dossier sont définies par le Consortium Unicode dans le fichier CaseFolding.txt sur leur site Web.

str.upper() #

str.upper prend chaque caractère d’une chaîne et le convertit en son équivalent majuscule, par exemple :

"This is a 'string'.".upper()
# "THIS IS A 'STRING'."

str.lower() #

str.lower fait le contraire ; il prend chaque caractère d’une chaîne et le convertit en son équivalent minuscule :

"This IS a 'string'.".lower()
# "this is a 'string'."

str.capitalize() #

str.capitalize renvoie une version en majuscule de la chaîne, c’est-à-dire qu’elle met le premier caractère en majuscule et le reste en minuscule :

"this Is A 'String'.".capitalize() # Capitalizes the first character and lowercases all others
# "This is a 'string'."

str.title() #

str.title renvoie la version en casse du titre de la chaîne, c’est-à-dire que chaque lettre au début d’un mot est en majuscule et toutes les autres en minuscules :

"this Is a 'String'".title()
# "This Is A 'String'"

str.swapcase() #

str.swapcase renvoie un nouvel objet chaîne dans lequel tous les caractères minuscules sont remplacés par des majuscules et tous les caractères majuscules par des minuscules :

"this iS A STRiNG".swapcase() #Swaps case of each character
# "THIS Is a strIng"

Utilisation en tant que méthodes de classe str #

Il convient de noter que ces méthodes peuvent être appelées soit sur des objets chaîne (comme indiqué ci-dessus), soit en tant que méthode de classe de la classe str (avec un appel explicite à str.upper, etc.)

str.upper("This is a 'string'")
# "THIS IS A 'STRING'"

Ceci est particulièrement utile lors de l’application de l’une de ces méthodes à plusieurs chaînes à la fois, par exemple une fonction [map][1].

map(str.upper,["These","are","some","'strings'"])
# ['THESE', 'ARE', 'SOME', "'STRINGS'"]

[1] : https://www.wikiod.com/fr/python/fonction-de-carte

str.translate : traduction des caractères dans une chaîne #

Python prend en charge une méthode translate sur le type str qui vous permet de spécifier la table de traduction (utilisée pour les remplacements) ainsi que les caractères qui doivent être supprimés dans le processus.

str.translate(table[, deletechars])
Paramètre Descriptif
tableau C’est une table de recherche qui définit le mappage d’un caractère à un autre.
deletechars Une liste de caractères qui doivent être supprimés de la chaîne.

La méthode maketrans (str.maketrans en Python 3 et string.maketrans en Python 2) permet de générer une table de traduction.

>>> translation_table = str.maketrans("aeiou", "12345")
>>> my_string = "This is a string!"
>>> translated = my_string.translate(translation_table)
'Th3s 3s 1 str3ng!'

La méthode translate renvoie une chaîne qui est une copie traduite de la chaîne d’origine.

Vous pouvez définir l’argument table sur None si vous avez seulement besoin de supprimer des caractères.

>>> 'this syntax is very useful'.translate(None, 'aeiou')
'ths syntx s vry sfl'

str.format et f-strings : formater les valeurs dans une chaîne #

Python fournit des fonctionnalités d’interpolation et de formatage de chaînes via la fonction str.format, introduite dans la version 2.6 et les f-strings introduites dans la version 3.6.

Étant donné les variables suivantes :

i = 10
f = 1.5
s = "foo"
l = ['a', 1, 2]
d = {'a': 1, 2: 'foo'}

Les énoncés suivants sont tous équivalents

"10 1.5 foo ['a', 1, 2] {'a': 1, 2: 'foo'}"

>>> "{} {} {} {} {}".format(i, f, s, l, d)

>>> str.format("{} {} {} {} {}", i, f, s, l, d)

>>> "{0} {1} {2} {3} {4}".format(i, f, s, l, d)

>>> "{0:d} {1:0.1f} {2} {3!r} {4!r}".format(i, f, s, l, d)

>>> "{i:d} {f:0.1f} {s} {l!r} {d!r}".format(i=i, f=f, s=s, l=l, d=d)

>>> f"{i} {f} {s} {l} {d}"

>>> f"{i:d} {f:0.1f} {s} {l!r} {d!r}"

Pour référence, Python prend également en charge les qualificatifs de style C pour le formatage des chaînes. Les exemples ci-dessous sont équivalents à ceux ci-dessus, mais les versions str.format sont préférées en raison des avantages en termes de flexibilité, de cohérence de notation et d’extensibilité :

"%d %0.1f %s %r %r" % (i, f, s, l, d)

"%(i)d %(f)0.1f %(s)s %(l)r %(d)r" % dict(i=i, f=f, s=s, l=l, d=d)

Les accolades utilisées pour l’interpolation dans str.format peuvent également être numérotées pour réduire la duplication lors du formatage des chaînes. Par exemple, les éléments suivants sont équivalents :

"I am from Australia. I love cupcakes from Australia!"

>>> "I am from {}. I love cupcakes from {}!".format("Australia", "Australia")

>>> "I am from {0}. I love cupcakes from {0}!".format("Australia")

Alors que la documentation officielle de python est, comme d’habitude, assez complète, pyformat.info a un grand nombre d’exemples avec des explications détaillées.

De plus, les caractères { et } peuvent être échappés en utilisant des crochets doubles :

"{'a': 5, 'b': 6}"

>>> "{{'{}': {}, '{}': {}}}".format("a", 5, "b", 6)

>>> f"{{'{'a'}': {5}, '{'b'}': {6}}"

Voir String Formatting pour plus d’informations. str.format() a été proposé dans [PEP 3101][1] et f-strings dans [PEP 498][2].

[1] : https://www.python.org/dev/peps/pep-3101/ [2] : https://www.python.org/dev/peps/pep-0498/

Fractionner une chaîne basée sur un délimiteur en une liste de chaînes #

str.split(sep=Aucun, maxsplit=-1) #

str.split prend une chaîne et renvoie une liste de sous-chaînes de la chaîne d’origine. Le comportement diffère selon que l’argument sep est fourni ou omis.

Si sep n’est pas fourni, ou est None, alors le fractionnement a lieu partout où il y a des espaces. Cependant, les espaces blancs de début et de fin sont ignorés et plusieurs caractères d’espacement consécutifs sont traités de la même manière qu’un seul caractère d’espacement :

>>> "This is a sentence.".split()
['This', 'is', 'a', 'sentence.']

>>> " This is    a sentence.  ".split()
['This', 'is', 'a', 'sentence.']

>>> "            ".split()
[]

Le paramètre sep peut être utilisé pour définir une chaîne de délimitation. La chaîne d’origine est fractionnée là où la chaîne de délimitation apparaît et le délimiteur lui-même est ignoré. Plusieurs délimiteurs consécutifs ne sont pas traités comme une seule occurrence, mais entraînent plutôt la création de chaînes vides.

>>> "This is a sentence.".split(' ')
['This', 'is', 'a', 'sentence.']

>>> "Earth,Stars,Sun,Moon".split(',')
['Earth', 'Stars', 'Sun', 'Moon']

>>> " This is    a sentence.  ".split(' ')
['', 'This', 'is', '', '', '', 'a', 'sentence.', '', '']

>>> "This is a sentence.".split('e')
['This is a s', 'nt', 'nc', '.']

>>> "This is a sentence.".split('en')
['This is a s', 't', 'ce.']

La valeur par défaut est de fractionner à chaque occurrence du délimiteur, cependant le paramètre maxsplit limite le nombre de fractionnements qui se produisent. La valeur par défaut de -1 signifie aucune limite :

>>> "This is a sentence.".split('e', maxsplit=0)
['This is a sentence.']

>>> "This is a sentence.".split('e', maxsplit=1)
['This is a s', 'ntence.']

>>> "This is a sentence.".split('e', maxsplit=2)
['This is a s', 'nt', 'nce.']

>>> "This is a sentence.".split('e', maxsplit=-1)
['This is a s', 'nt', 'nc', '.']

str.rsplit(sep=Aucun, maxsplit=-1) #

str.rsplit (“right split”) diffère de str.split (“left split”) lorsque maxsplit est spécifié. Le fractionnement commence à la fin de la chaîne plutôt qu’au début :

>>> "This is a sentence.".rsplit('e', maxsplit=1)
['This is a sentenc', '.']

>>> "This is a sentence.".rsplit('e', maxsplit=2)
['This is a sent', 'nc', '.']

Remarque : Python spécifie le nombre maximal de splits effectués, tandis que la plupart des autres langages de programmation spécifient le nombre maximal de sous-chaînes créées. Cela peut créer de la confusion lors du portage ou de la comparaison de code.

Remplacer toutes les occurrences d’une sous-chaîne par une autre sous-chaîne #

Le type str de Python a également une méthode pour remplacer les occurrences d’une sous-chaîne par une autre sous-chaîne dans une chaîne donnée. Pour les cas plus exigeants, on peut utiliser [re.sub][1].

str.replace(old, new[, count]) : #

str.replace prend deux arguments old et new contenant la sous-chaîne old qui doit être remplacée par la sous-chaîne new. L’argument optionnel count spécifie le nombre de remplacements à effectuer :

Par exemple, pour remplacer 'foo' par 'spam' dans la chaîne suivante, nous pouvons appeler str.replace avec old = 'foo' et new = 'spam' :

>>> "Make sure to foo your sentence.".replace('foo', 'spam')
"Make sure to spam your sentence."

Si la chaîne donnée contient plusieurs exemples qui correspondent à l’argument old, toutes les occurrences sont remplacées par la valeur fournie dans new :

>>> "It can foo multiple examples of foo if you want.".replace('foo', 'spam')
"It can spam multiple examples of spam if you want."

à moins, bien sûr, que nous fournissions une valeur pour count. Dans ce cas, les occurrences de count vont être remplacées :

>>> """It can foo multiple examples of foo if you want, \
... or you can limit the foo with the third argument.""".replace('foo', 'spam', 1)
'It can spam multiple examples of foo if you want, or you can limit the foo with the third argument.'

[1] : https://www.wikiod.com/fr/python/expressions-regulieres-regex

Tester la composition d’une chaîne #

Le type str de Python propose également un certain nombre de méthodes qui peuvent être utilisées pour évaluer le contenu d’une chaîne. Ce sont str.isalpha, str.isdigit, str.isalnum, str.isspace. La capitalisation peut être testée avec str.isupper, str.islower et str.istitle.

str.isalpha #

str.isalpha ne prend aucun argument et renvoie True si tous les caractères d’une chaîne donnée sont alphabétiques, par exemple :

>>> "Hello World".isalpha()  # contains a space
False
>>> "Hello2World".isalpha()  # contains a number
False
>>> "HelloWorld!".isalpha()  # contains punctuation
False
>>> "HelloWorld".isalpha()
True

En tant que cas limite, la chaîne vide est évaluée à False lorsqu’elle est utilisée avec "".isalpha().

str.isupper, str.islower, str.istitle #

Ces méthodes testent la casse dans une chaîne donnée.

str.isupper est une méthode qui renvoie True si tous les caractères d’une chaîne donnée sont en majuscules et False sinon.

>>> "HeLLO WORLD".isupper()
False
>>> "HELLO WORLD".isupper()
True
>>> "".isupper()
False

Inversement, str.islower est une méthode qui renvoie True si tous les caractères d’une chaîne donnée sont en minuscules et False sinon.

>>> "Hello world".islower()
False
>>> "hello world".islower()
True
>>> "".islower()
False

str.istitle renvoie True si la chaîne donnée est en casse de titre ; c’est-à-dire que chaque mot commence par un caractère majuscule suivi de caractères minuscules.

>>> "hello world".istitle()
False
>>> "Hello world".istitle()
False
>>> "Hello World".istitle()
True
>>> "".istitle()
False

str.isdecimal, str.isdigit, str.isnumeric #

str.isdecimal renvoie si la chaîne est une séquence de chiffres décimaux, appropriée pour représenter un nombre décimal.

str.isdigit inclut des chiffres qui ne sont pas sous une forme appropriée pour représenter un nombre décimal nombre, comme les chiffres en exposant.

str.isnumeric inclut toutes les valeurs numériques, même s’il ne s’agit pas de chiffres, telles que les valeurs en dehors de la plage 0-9.

            isdecimal    isdigit   isnumeric

12345        True        True       True
១2߃໔5        True        True       True
①²³🄅₅       False       True       True
⑩⒓          False       False      True
Five         False       False      False

Les chaînes d’octets (bytes en Python 3, str en Python 2) ne prennent en charge que isdigit, qui ne vérifie que les chiffres ASCII de base.

Comme avec str.isalpha, la chaîne vide est évaluée à False.

str.isalnum #

Il s’agit d’une combinaison de str.isalpha et str.isnumeric, plus précisément, il est évalué à True si tous les caractères de la chaîne donnée sont ** alphanumériques **, c’est-à-dire qu’ils se composent de caractères alphabétiques * ou * numériques :

>>> "Hello2World".isalnum()
True
>>> "HelloWorld".isalnum()
True
>>> "2016".isalnum()
True
>>> "Hello World".isalnum()  # contains whitespace
False

str.isspace #

Prend la valeur “True” si la chaîne ne contient que des caractères d’espacement.

>>> "\t\r\n".isspace()
True
>>> " ".isspace()
True

Parfois, une chaîne semble “vide” mais nous ne savons pas si c’est parce qu’elle ne contient que des espaces ou aucun caractère du tout

>>> "".isspace()
False

Pour couvrir ce cas, nous avons besoin d’un test supplémentaire

>>> my_str = ''
>>> my_str.isspace()
False
>>> my_str.isspace() or not my_str
True

Mais le moyen le plus court de tester si une chaîne est vide ou contient simplement des caractères d’espacement est d’utiliser [strip][1](sans arguments, il supprime tous les caractères d’espacement de début et de fin)

>>> not my_str.strip()
True

[1] : https://www.wikiod.com/fr/python/methodes-de-chaine#Suppression des caractères de début/fin indésirables d’une chaîne

Suppression des caractères de début/fin indésirables d’une chaîne #

Trois méthodes sont fournies qui offrent la possibilité de supprimer les caractères de début et de fin d’une chaîne : str.strip, str.rstrip et str.lstrip. Les trois méthodes ont la même signature et toutes les trois renvoient un nouvel objet chaîne avec les caractères indésirables supprimés.

str.strip([chars]) #

str.strip agit sur une chaîne donnée et supprime (supprime) tous les caractères de début ou de fin contenus dans l’argument chars ; si chars n’est pas fourni ou vaut None, tous les espaces blancs sont supprimés par défaut. Par exemple:

>>> "    a line with leading and trailing space     ".strip() 
'a line with leading and trailing space'

Si chars est fourni, tous les caractères qu’il contient sont supprimés de la chaîne, qui est renvoyée. Par exemple:

>>> ">>> a Python prompt".strip('> ')  # strips '>' character and space character 
'a Python prompt'

str.rstrip([chars]) et str.lstrip([chars]) #

Ces méthodes ont une sémantique et des arguments similaires avec str.strip(), leur différence réside dans la direction à partir de laquelle elles commencent. str.rstrip() commence à partir de la fin de la chaîne tandis que str.lstrip() se divise à partir du début de la chaîne.

Par exemple, en utilisant str.rstrip :

>>> "     spacious string      ".rstrip()
'     spacious string'

Tandis que, en utilisant str.lstrip :

>>> "     spacious string      ".rstrip()
'spacious string      '

Inverser une chaîne #

Une chaîne peut être inversée à l’aide de la fonction intégrée reversed(), qui prend une chaîne et renvoie un itérateur dans l’ordre inverse.

>>> reversed('hello')
<reversed object at 0x0000000000000000>
>>> [char for char in reversed('hello')]
['o', 'l', 'l', 'e', 'h']

reversed() peut être enveloppé dans un appel à [''.join() pour créer une chaîne][1] à partir de l’itérateur.

>>> ''.join(reversed('hello'))
'olleh'

Bien que l’utilisation de reversed() puisse être plus lisible pour les utilisateurs Python non initiés, l’utilisation de [slicing][2] étendu avec un pas de -1 est plus rapide et plus concis. Ici, essayez de l’implémenter en tant que fonction :

>>> def reversed_string(main_string):
...     return main_string[::-1]
...
>>> reversed_string('hello')
'olleh'

[1] : https://www.wikiod.com/fr/python/methodes-de-chaine#Joindre une liste de chaînes en une seule chaîne [2] : https://www.wikiod.com/fr/python/indexation-et-decoupage#Tranchage de base

Joindre une liste de chaînes en une seule chaîne #

Une chaîne peut être utilisée comme séparateur pour joindre une liste de chaînes dans un chaîne unique en utilisant la méthode join(). Par exemple, vous pouvez créer une chaîne où chaque élément d’une liste est séparé par un espace.

>>> " ".join(["once","upon","a","time"])
"once upon a time"

L’exemple suivant sépare les éléments de chaîne par trois traits d’union.

>>> "---".join(["once", "upon", "a", "time"])
"once---upon---a---time"

Chaîne Contient #

Python rend extrêmement intuitif la vérification si une chaîne contient une sous-chaîne donnée. Utilisez simplement l’opérateur “in”:

>>> "foo" in "foo.baz.bar"
True

Remarque : le test d’une chaîne vide aboutira toujours à “True” :

>>> "" in "test"
True

Compter le nombre de fois qu’une sous-chaîne apparaît dans une chaîne #

Une méthode est disponible pour compter le nombre d’occurrences d’une sous-chaîne dans une autre chaîne, str.count.

str.count(sous[, début[, fin]]) #

str.count renvoie un int indiquant le nombre d’occurrences sans chevauchement de la sous-chaîne sub dans une autre chaîne. Les arguments optionnels start et end indiquent le début et la fin de la recherche. Par défaut, start = 0 et end = len(str), ce qui signifie que la chaîne entière sera recherchée :

>>> s = "She sells seashells by the seashore."
>>> s.count("sh")
2
>>> s.count("se")
3
>>> s.count("sea")
2
>>> s.count("seashells")
1

En spécifiant une valeur différente pour start, end nous pouvons obtenir une recherche et un comptage plus localisés, par exemple, si start est égal à 13 l’appel à :

>>> s.count("sea", start)
1

est équivalent à:

>>> t = s[start:]
>>> t.count("sea")
1

Comparaisons de chaînes insensibles à la casse #

Comparer une chaîne de manière insensible à la casse semble être quelque chose de trivial, mais ce n’est pas le cas. Cette section ne considère que les chaînes Unicode (la valeur par défaut dans Python 3). Notez que Python 2 peut avoir des faiblesses subtiles par rapport à Python 3 - la gestion unicode de ce dernier est beaucoup plus complète.

La première chose à noter est que les conversions de suppression de casse en unicode ne sont pas triviales. Il y a du texte pour lequel text.lower() != text.upper().lower(), tel que "ß" :

>>> "ß".lower()
'ß'

>>> "ß".upper().lower()
'ss'

Mais disons que vous vouliez comparer sans cas "BUSSE" et "Buße". Heck, vous voulez probablement aussi comparer "BUSSE" et "BUẞE" égaux - c’est la nouvelle forme de capital. La méthode recommandée consiste à utiliser casefold :

>>> help(str.casefold)
"""
Help on method_descriptor:

casefold(...)
      S.casefold() -> str
    
     Return a version of S suitable for caseless comparisons.
"""

Ne vous contentez pas d’utiliser “inférieur”. Si casefold n’est pas disponible, faire .upper().lower() aide (mais seulement un peu).

Ensuite, vous devriez considérer les accents. Si votre rendu de police est bon, vous pensez probablement "ê" == "ê" - mais ce n’est pas le cas :

>>> "ê" == "ê"
False

C’est parce qu’ils sont en fait

>>> import unicodedata

>>> [unicodedata.name(char) for char in "ê"]
['LATIN SMALL LETTER E WITH CIRCUMFLEX']

>>> [unicodedata.name(char) for char in "ê"]
['LATIN SMALL LETTER E', 'COMBINING CIRCUMFLEX ACCENT']

La façon la plus simple de gérer cela est unicodedata.normalize. Vous souhaitez probablement utiliser la normalisation NFKD, mais n’hésitez pas à consulter la documentation. Ensuite on fait

>>> unicodedata.normalize("NFKD", "ê") == unicodedata.normalize("NFKD", "ê")
True

Pour finir, ici cela s’exprime en fonctions :

import unicodedata

def normalize_caseless(text):
    return unicodedata.normalize("NFKD", text.casefold())

def caseless_equal(left, right):
    return normalize_caseless(left) == normalize_caseless(right)

Tester les caractères de début et de fin d’une chaîne #

Afin de tester le début et la fin d’une chaîne donnée en Python, on peut utiliser les méthodes str.startswith() et str.endswith().

str.startswith(prefix[, start[, end]]) #

Comme son nom l’indique, str.startswith est utilisé pour tester si une chaîne donnée commence par les caractères donnés dans prefix.

>>> s = "This is a test string"
>>> s.startswith("T")
True
>>> s.startswith("Thi")
True
>>> s.startswith("thi")  
False

Les arguments optionnels start et end spécifient les points de début et de fin à partir desquels le test commencera et se terminera. Dans l’exemple suivant, en spécifiant une valeur de départ de ‘2’, notre chaîne sera recherchée à partir de la position ‘2’ et après :

>>> s.startswith("is", 2)
True

Cela donne True puisque s[2] == 'i' et s[3] == 's'.

Vous pouvez également utiliser un tuple pour vérifier s’il commence par l’un des ensembles de chaînes

>>> s.startswith(('This', 'That'))
True
>>> s.startswith(('ab', 'bc'))
False

str.endswith(prefix[, start[, end]]) #

str.endswith est exactement similaire à str.startswith à la seule différence qu’il recherche les caractères de fin et non les caractères de début. Par exemple, pour tester si une chaîne se termine par un point, on pourrait écrire :

>>> s = "this ends in a full stop."
>>> s.endswith('.')
True
>>> s.endswith('!')
False

comme avec startswith, plusieurs caractères peuvent être utilisés comme séquence de fin :

>>> s.endswith('stop.')
True
>>> s.endswith('Stop.')
False

Vous pouvez également utiliser un tuple pour vérifier s’il se termine par l’un des ensembles de chaînes

>>> s.endswith(('.', 'something'))
True
>>> s.endswith(('ab', 'bc'))
False

Justifier les chaînes #

Python fournit des fonctions pour justifier les chaînes, permettant le remplissage de texte pour faciliter l’alignement des différentes chaînes.

Voici un exemple de str.ljust et str.rjust :

interstates_lengths = {
    5: (1381, 2222),
    19: (63, 102),
    40: (2555, 4112),
    93: (189,305),
}
for road, length in interstates_lengths.items():
    miles,kms = length
    print('{} -> {} mi. ({} km.)'.format(str(road).rjust(4), str(miles).ljust(4), str(kms).ljust(4)))

  40 -> 2555 mi. (4112 km.)
  19 -> 63   mi. (102  km.)
   5 -> 1381 mi. (2222 km.)
  93 -> 189  mi. (305  km.)

ljust et rjust sont très similaires. Les deux ont un paramètre width et un paramètre facultatif fillchar. Toute chaîne créée par ces fonctions est au moins aussi longue que le paramètre width qui a été transmis à la fonction. Si la chaîne est déjà plus longue que width, elle n’est pas tronquée. L’argument fillchar, qui est par défaut le caractère d’espacement ' ' doit être un caractère unique, pas une chaîne multicaractères.

La fonction ljust remplit la fin de la chaîne sur laquelle elle est appelée avec le fillchar jusqu’à ce qu’elle ait une longueur de width caractères. La fonction rjust remplit le début de la chaîne de la même manière. Par conséquent, le l et le r dans les noms de ces fonctions font référence au côté où la chaîne d’origine, pas le fillchar, est positionnée dans la chaîne de sortie.

Conversion entre les données str ou bytes et les caractères unicode #

Le contenu des fichiers et des messages réseau peut représenter des caractères codés. Ils doivent souvent être convertis en unicode pour un affichage correct.

Dans Python 2, vous devrez peut-être convertir les données str en caractères Unicode. La valeur par défaut ('', "", etc.) est une chaîne ASCII, avec toutes les valeurs en dehors de la plage ASCII affichées sous forme de valeurs échappées. Les chaînes Unicode sont u'' (ou u"", etc.).

# You get "© abc" encoded in UTF-8 from a file, network, or other data source

s = '\xc2\xa9 abc'  # s is a byte array, not a string of characters
                    # Doesn't know the original was UTF-8
                    # Default form of string literals in Python 2
s[0]                # '\xc2' - meaningless byte (without context such as an encoding)
type(s)             # str - even though it's not a useful one w/o having a known encoding

u = s.decode('utf-8')  # u'\xa9 abc'
                       # Now we have a Unicode string, which can be read as UTF-8 and printed properly
                       # In Python 2, Unicode string literals need a leading u
                       # str.decode converts a string which may contain escaped bytes to a Unicode string
u[0]                # u'\xa9' - Unicode Character 'COPYRIGHT SIGN' (U+00A9) '©'
type(u)             # unicode

u.encode('utf-8')   # '\xc2\xa9 abc'
                    # unicode.encode produces a string with escaped bytes for non-ASCII characters

Dans Python 3, vous devrez peut-être convertir des tableaux d’octets (appelés « littéraux d’octets ») en chaînes de caractères Unicode. La valeur par défaut est maintenant une chaîne Unicode et les littéraux de chaîne d’octets doivent maintenant être entrés sous la forme b'', b"", etc. Un littéral d’octet renverra True à isinstance(some_val, byte), en supposant some_val comme une chaîne pouvant être encodée en octets.

# You get from file or network "© abc" encoded in UTF-8

s = b'\xc2\xa9 abc' # s is a byte array, not characters
                    # In Python 3, the default string literal is Unicode; byte array literals need a leading b
s[0]                # b'\xc2' - meaningless byte (without context such as an encoding)
type(s)             # bytes - now that byte arrays are explicit, Python can show that.

u = s.decode('utf-8')  # '© abc' on a Unicode terminal
                       # bytes.decode converts a byte array to a string (which will, in Python 3, be Unicode)
u[0]                # '\u00a9' - Unicode Character 'COPYRIGHT SIGN' (U+00A9) '©'
type(u)             # str
                    # The default string literal in Python 3 is UTF-8 Unicode

u.encode('utf-8')   # b'\xc2\xa9 abc'
                    # str.encode produces a byte array, showing ASCII-range bytes as unescaped characters.
