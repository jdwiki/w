  • Itérer deux ou plusieurs listes simultanément dans la compréhension de la liste

Une compréhension de liste est un outil syntaxique permettant de créer des listes de manière naturelle et concise, comme illustré dans le code suivant pour faire une liste de carrés des nombres de 1 à 10 :

[i ** 2 pour i dans la plage (1,11)]

Le mannequin i d’une liste existante range est utilisé pour créer un nouveau motif d’élément. Il est utilisé là où une boucle for serait nécessaire dans des langages moins expressifs.

Syntaxe #

  • [i for i in range(10)] # compréhension de base des listes
  • [i for i in xrange(10)] # compréhension de base de la liste avec un objet générateur en python 2.x
  • [i for i in range(20) if i % 2 == 0] # with filter
  • [x + y pour x dans [1, 2, 3] pour y dans [3, 4, 5]] # boucles imbriquées
  • [i if i > 6 else 0 for i in range(10)] # expression ternaire
  • [i if i > 4 else 0 for i in range(20) if i % 2 == 0] # avec filtre et expression ternaire
  • [[x + y for x in [1, 2, 3]] for y in [3, 4, 5]] # compréhension de liste imbriquée

Les compréhensions de liste ont été décrites dans PEP 202 et introduites dans Python 2.0.

Compréhensions de listes conditionnelles #

Étant donné une [list comprehension][1], vous pouvez ajouter une ou plusieurs conditions “if” pour filtrer les valeurs.

[<expression> for <element> in <iterable> if <condition>]

Pour chaque <element> dans <iterable> ; si <condition> est évalué à True, ajoutez <expression> (généralement une fonction de <element>) à la liste renvoyée.

Par exemple, cela peut être utilisé pour extraire uniquement les nombres pairs d’une séquence d’entiers :

[x for x in range(10) if x % 2 == 0]
# Out: [0, 2, 4, 6, 8]

[Démo en direct] (http://coliru.stacked-crooked.com/a/8ea85b7b87f57876)

Le code ci-dessus est équivalent à :

even_numbers = [] 
for x in range(10):
    if x % 2 == 0:
        even_numbers.append(x)

print(even_numbers)
# Out: [0, 2, 4, 6, 8]

De plus, une compréhension de liste conditionnelle de la forme [e for x in y if c] (où e et c sont des expressions en termes de x) est équivalente à list(filter(lambda x: c , map(lambda x : e, y))).

Bien qu’il fournisse le même résultat, faites attention au fait que le premier exemple est presque 2 fois plus rapide que le second. Pour ceux qui sont curieux, this est une belle explication de la raison pour laquelle .

Notez que ceci est assez différent de l’expression conditionnelle ... if ... else ... (parfois appelée [expression ternaire][2]) que vous pouvez utiliser pour la partie <expression> de la compréhension de la liste. Considérez l’exemple suivant :

[x if x % 2 == 0 else None for x in range(10)]
# Out: [0, None, 2, None, 4, None, 6, None, 8, None]

[Démo en direct] (http://coliru.stacked-crooked.com/a/38edffb6f855e3fc)

Ici, l’expression conditionnelle n’est pas un filtre, mais plutôt un opérateur déterminant la valeur à utiliser pour les éléments de la liste :

<value-if-condition-is-true> if <condition> else <value-if-condition-is-false>

Cela devient plus évident si vous le combinez avec d’autres opérateurs :

[2 * (x if x % 2 == 0 else -1) + 1 for x in range(10)]
# Out: [1, -1, 5, -1, 9, -1, 13, -1, 17, -1]

[Démo en direct] (http://coliru.stacked-crooked.com/a/59802eec8ad5deb8)

Si vous utilisez Python 2.7, xrange peut être meilleur que range pour plusieurs raisons, comme décrit dans la [documentation xrange][3].

[2 * (x if x % 2 == 0 else -1) + 1 for x in xrange(10)]
# Out: [1, -1, 5, -1, 9, -1, 13, -1, 17, -1]

Le code ci-dessus est équivalent à :

numbers = []
for x in range(10):
    if x % 2 == 0:
        temp = x
    else:
        temp = -1
    numbers.append(2 * temp + 1)
print(numbers)
# Out: [1, -1, 5, -1, 9, -1, 13, -1, 17, -1]

On peut combiner des expressions ternaires et des conditions if. L’opérateur ternaire travaille sur le résultat filtré :

[x if x > 2 else '*' for x in range(10) if x % 2 == 0]
# Out: ['*', '*', 4, 6, 8]

La même chose n’aurait pas pu être obtenue uniquement par l’opérateur ternaire :

[x if (x > 2 and x % 2 == 0) else '*' for x in range(10)]
# Out:['*', '*', '*', '*', 4, '*', 6, '*', 8, '*']

Voir aussi : Filters, qui fournissent souvent une alternative suffisante aux compréhensions de listes conditionnelles.

Liste des compréhensions avec des boucles imbriquées #

[List Comprehensions][1] peut utiliser des boucles for imbriquées. Vous pouvez coder n’importe quel nombre de boucles for imbriquées dans une compréhension de liste, et chaque boucle for peut avoir un test if associé facultatif. Ce faisant, l’ordre des constructions for est le même que lors de l’écriture d’une série d’instructions for imbriquées. La structure générale des compréhensions de liste ressemble à ceci :

[ expression for target1 in iterable1 [if condition1]
             for target2 in iterable2 [if condition2]...
             for targetN in iterableN [if conditionN] ]

Par exemple, le code suivant aplatit une liste de listes à l’aide de plusieurs instructions “for” :

data = [[1, 2], [3, 4], [5, 6]]
output = []
for each_list in data:
    for element in each_list:
        output.append(element)
print(output)
# Out: [1, 2, 3, 4, 5, 6]

peut être écrit de manière équivalente sous la forme d’une compréhension de liste avec plusieurs constructions for :

data = [[1, 2], [3, 4], [5, 6]]
output = [element for each_list in data for element in each_list]
print(output)
# Out: [1, 2, 3, 4, 5, 6]

[Démo en direct][2]

Dans la forme développée et dans la compréhension de la liste, la boucle externe (première instruction for) vient en premier.

En plus d’être plus compacte, la compréhension imbriquée est également nettement plus rapide.

In [1]: data = [[1,2],[3,4],[5,6]]
In [2]: def f():
   ...:     output=[]
   ...:     for each_list in data:
   ...:         for element in each_list:
   ...:             output.append(element)
   ...:     return output
In [3]: timeit f()
1000000 loops, best of 3: 1.37 µs per loop
In [4]: timeit [inner for outer in data for inner in outer]
1000000 loops, best of 3: 632 ns per loop

La surcharge pour l’appel de fonction ci-dessus est d’environ 140ns.

Les “if” en ligne sont imbriqués de la même manière et peuvent apparaître à n’importe quelle position après le premier “for” :

data = [[1], [2, 3], [4, 5]]
output = [element for each_list in data
                if len(each_list) == 2
                for element in each_list
                if element != 5]
print(output)
# Out: [2, 3, 4]

[Démo en direct][3]

Pour des raisons de lisibilité, cependant, vous devriez envisager d’utiliser les for-loops traditionnels. Cela est particulièrement vrai lorsque l’imbrication a plus de 2 niveaux de profondeur et/ou que la logique de compréhension est trop complexe. la compréhension de plusieurs listes de boucles imbriquées peut être sujette aux erreurs ou donner un résultat inattendu.

Filtre de refactorisation et carte pour lister les compréhensions #

Les fonctions filter ou map devraient souvent être remplacées par [list comprehensions][1]. Guido Van Rossum le décrit bien dans une [lettre ouverte en 2005][2] :

filter(P, S) est presque toujours écrit plus clairement comme [x for x in S if P(x)], et cela a l’énorme avantage que les utilisations les plus courantes impliquent des prédicats qui sont des comparaisons, par ex. x==42, et définir un lambda pour cela nécessite juste beaucoup plus d’efforts pour le lecteur (en plus, le lambda est plus lent que la compréhension de la liste). Encore plus pour map(F, S) qui devient [F(x) for x in S]. Bien sûr, dans de nombreux cas, vous pourrez utiliser des expressions de générateur à la place.

Les lignes de code suivantes sont considérées comme “non pythonique” et généreront des erreurs dans de nombreux linters python.

filter(lambda x: x % 2 == 0, range(10)) # even numbers < 10
map(lambda x: 2*x, range(10)) # multiply each number by two
reduce(lambda x,y: x+y, range(10)) # sum of all elements in list

En prenant ce que nous avons appris de la citation précédente, nous pouvons décomposer ces expressions filter et map en leurs compréhensions de liste équivalentes ; supprimant également les fonctions * lambda * de chacun - rendant le code plus lisible dans le processus.

# Filter:
# P(x) = x % 2 == 0
# S = range(10)
[x for x in range(10) if x % 2 == 0]

# Map
# F(x) = 2*x
# S = range(10)
[2*x for x in range(10)]

La lisibilité devient encore plus évidente lorsqu’il s’agit de fonctions de chaînage. Lorsque, pour des raisons de lisibilité, les résultats d’une carte ou d’une fonction de filtrage doivent être transmis comme résultat à la suivante ; avec des cas simples, ceux-ci peuvent être remplacés par une compréhension de liste unique. De plus, nous pouvons facilement dire à partir de la compréhension de la liste quel est le résultat de notre processus, où il y a plus de charge cognitive lors du raisonnement sur le processus enchaîné Map & Filter.

# Map & Filter
filtered = filter(lambda x: x % 2 == 0, range(10))
results = map(lambda x: 2*x, filtered)

# List comprehension
results = [2*x for x in range(10) if x % 2 == 0]

Refactoring - Référence rapide #

  • Carte

    map(F, S) == [F(x) for x in S]

  • Filtre

    filter(P, S) == [x for x in S if P(x)]

où F et P sont des fonctions qui transforment respectivement les valeurs d’entrée et renvoient un bool

Compréhensions de listes imbriquées #

Les compréhensions de liste imbriquées, contrairement aux compréhensions de liste avec des boucles imbriquées, sont des compréhensions de liste au sein d’une compréhension de liste. L’expression initiale peut être n’importe quelle expression arbitraire, y compris une autre liste en compréhension.

#List Comprehension with nested loop
[x + y for x in [1, 2, 3] for y in [3, 4, 5]]
#Out: [4, 5, 6, 5, 6, 7, 6, 7, 8]

#Nested List Comprehension
[[x + y for x in [1, 2, 3]] for y in [3, 4, 5]]
#Out: [[4, 5, 6], [5, 6, 7], [6, 7, 8]]

L’exemple Nested équivaut à

l = []
for y in [3, 4, 5]:
    temp = []
    for x in [1, 2, 3]:
        temp.append(x + y)
    l.append(temp)

Un exemple où une compréhension imbriquée peut être utilisée pour transposer une matrice.

matrix = [[1,2,3],
          [4,5,6],
          [7,8,9]] 

[[row[i] for row in matrix] for i in range(len(matrix))]
# [[1, 4, 7], [2, 5, 8], [3, 6, 9]]

Comme les boucles for imbriquées, il n’y a pas de limite à la profondeur des compréhensions qui peuvent être imbriquées.

[[[i + j + k for k in 'cd'] for j in 'ab'] for i in '12']
# Out: [[['1ac', '1ad'], ['1bc', '1bd']], [['2ac', '2ad'], ['2bc', '2bd']]]

Itérer deux ou plusieurs listes simultanément dans la compréhension de la liste #

Pour itérer plus de deux listes simultanément dans list comprehension, on peut utiliser [zip()][1] comme :

>>> list_1 = [1, 2, 3 , 4]
>>> list_2 = ['a', 'b', 'c', 'd']
>>> list_3 = ['6', '7', '8', '9']

# Two lists
>>> [(i, j) for i, j in zip(list_1, list_2)]
[(1, 'a'), (2, 'b'), (3, 'c'), (4, 'd')]

# Three lists
>>> [(i, j, k) for i, j, k in zip(list_1, list_2, list_3)]
[(1, 'a', '6'), (2, 'b', '7'), (3, 'c', '8'), (4, 'd', '9')]

# so on ...
