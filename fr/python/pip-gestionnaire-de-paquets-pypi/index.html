pip est le gestionnaire de packages le plus utilisé pour Python Package Index, installé par défaut avec les versions récentes de Python.

Syntaxe #

  • pip <commande> [options] où <commande> est l’un des :
  • installer
    • Install packages
  • désinstaller
    • Uninstall packages
  • gel
    • Output installed packages in requirements format
  • liste
    • List installed packages
  • Afficher
    • Show information about installed packages
  • chercher
    • Search PyPI for packages
  • la roue
    • Build wheels from your requirements
  • Zip *: français
    • Zip individual packages (deprecated)
  • décompresser
    • Unzip individual packages (deprecated)
  • paquet
    • Create pybundles (deprecated)
  • aider
    • Show help for commands

Parfois, pip effectuera une compilation manuelle du code natif. Sous Linux, Python choisira automatiquement un compilateur C disponible sur votre système. Reportez-vous au tableau ci-dessous pour connaître la version requise de Visual Studio/Visual C++ sous Windows (les versions plus récentes ne fonctionneront pas.).

version python Version Visual Studio Version Visual C++
2.6 - 3.2 Visual Studio 2008 [Visuel C++ 9.0][1]
3.3 - 3.4 Visual Studio 2010 Visual C++ 10.0
3.5 Visual Studio 2015 Visual C++ 14.0

[Source : wiki.python.org][2]

[1] : https://www.microsoft.com/en-us/download/details.aspx?id=44266 [2] : https://wiki.python.org/moin/WindowsCompilers

Installer les packages #

Pour installer la dernière version d’un package nommé “SomePackage” :

$ pip install SomePackage

Pour installer une version spécifique d’un package :

$ pip install SomePackage==1.0.4

Pour spécifier une version minimale à installer pour un package :

$ pip install SomePackage>=1.0.4

Si les commandes affichent une erreur d’autorisation refusée sous Linux/Unix, utilisez sudo avec les commandes

Installer à partir des fichiers d’exigences #

$ pip install -r requirements.txt

Chaque ligne du fichier requirements indique quelque chose à installer, et comme arguments pour pip install, les détails sur le format des fichiers sont ici : [Requirements File Format][1].

Après avoir installé le package, vous pouvez le vérifier à l’aide de la commande freeze :

$ pip freeze

Forfaits de mise à niveau #

Fonctionnement

$ pip install --upgrade SomePackage

mettra à jour le package “SomePackage” et toutes ses dépendances. En outre, pip supprime automatiquement l’ancienne version du package avant la mise à niveau.

Pour mettre à jour pip lui-même, faites

$ pip install --upgrade pip

sous Unix ou

$ python -m pip install --upgrade pip

sur les machines Windows.

Désinstaller les packages #

Pour désinstaller un package :

$ pip uninstall SomePackage

Mise à jour de tous les packages obsolètes sous Linux #

pip ne contient pas actuellement d’indicateur permettant à un utilisateur de mettre à jour tous les packages obsolètes en une seule fois. Cependant, cela peut être accompli en regroupant les commandes dans un environnement Linux :

pip list --outdated --local | grep -v '^\-e' | cut -d = -f 1  | xargs -n1 pip install -U

Cette commande prend tous les packages dans le virtualenv local et vérifie s’ils sont obsolètes. À partir de cette liste, il obtient le nom du package, puis le dirige vers une commande pip install -U. À la fin de ce processus, tous les packages locaux doivent être mis à jour.

Mise à jour de tous les packages obsolètes sous Windows #

pip ne contient pas actuellement d’indicateur permettant à un utilisateur de mettre à jour tous les packages obsolètes en une seule fois. Cependant, cela peut être accompli en regroupant les commandes dans un environnement Windows :

for /F "delims= " %i in ('pip list --outdated --local') do pip install -U %i

Cette commande prend tous les packages dans le virtualenv local et vérifie s’ils sont obsolètes. À partir de cette liste, il obtient le nom du package, puis le dirige vers une commande pip install -U. À la fin de ce processus, tous les packages locaux doivent être mis à jour.

Créez un fichier requirements.txt de tous les packages du système #

pip aide à créer des fichiers requirements.txt en fournissant l’option [freeze][1].

pip freeze > requirements.txt

Cela enregistrera une liste de tous les packages et de leur version installés sur le système dans un fichier nommé “requirements.txt” dans le dossier actuel.

[1] : https://pip.pypa.io/en/stable/reference/pip_freeze/

Créer un fichier requirements.txt de packages uniquement dans le virtualenv actuel #

pip aide à créer des fichiers requirements.txt en fournissant l’option [freeze][1].

pip freeze --local > requirements.txt

Le paramètre [--local][1] affichera uniquement une liste de packages et de versions installés localement sur un virtualenv. Les packages globaux ne seront pas répertoriés.

[1] : https://pip.pypa.io/en/stable/reference/pip_freeze/#cmdoption-l

Utiliser une certaine version de Python avec pip #

Si vous avez à la fois Python 3 et Python 2 installés, vous pouvez spécifier la version de Python que vous souhaitez que pip utilise. Ceci est utile lorsque les packages ne prennent en charge que Python 2 ou 3 ou lorsque vous souhaitez tester avec les deux.

Si vous souhaitez installer des packages pour Python 2, exécutez soit :

pip install [package]

ou:

pip2 install [package]

Si vous souhaitez installer des packages pour Python 3, procédez comme suit :

pip3 install [package]

Vous pouvez également invoquer l’installation d’un package sur une installation python spécifique avec :

\path\to\that\python.exe -m pip install some_package # on Windows OR
/usr/bin/python25 -m pip install some_package # on OS-X/Linux

Sur les plates-formes OS-X/Linux/Unix, il est important de faire la distinction entre la version système de python (que la mise à niveau rend votre système inutilisable) et la ou les versions utilisateur de python. Vous pouvez, selon ce que vous essayez de mettre à niveau, avoir besoin de préfixer ces commandes avec sudo et d’entrer un mot de passe.

De même, sous Windows, certaines installations python, en particulier celles qui font partie d’un autre package, peuvent se retrouver installées dans des répertoires système - ceux que vous devrez mettre à niveau à partir d’une fenêtre de commande en mode administrateur - si vous trouvez qu’il semble que vous deviez c’est une très bonne idée de vérifier quelle installation python vous essayez de mettre à niveau avec une commande telle que python -c"import sys;print(sys.path);" ou py -3.5 - c"import sys;print(sys.path);" vous pouvez également vérifier quel pip vous essayez d’exécuter avec pip --version

Sous Windows, si vous avez à la fois python 2 et python 3 installés, et sur votre chemin et que votre python 3 est supérieur à 3.4, vous aurez probablement aussi le lanceur python py sur votre chemin système. Vous pouvez alors faire des trucs comme :

py -3 -m pip install -U some_package # Install/Upgrade some_package to the latest python 3
py -3.3 -m pip install -U some_package # Install/Upgrade some_package to python 3.3 if present
py -2 -m pip install -U some_package # Install/Upgrade some_package to the latest python 2 - 64 bit if present
py -2.7-32 -m pip install -U some_package # Install/Upgrade some_package to python 2.7 - 32 bit if present

Si vous exécutez et maintenez plusieurs versions de python, je vous recommande fortement de lire sur les python virtualenv ou venv [environnements virtuels] [1] qui vous permettent d’isoler à la fois la version de python et les packages présents.

[1] : https://virtualenv.pypa.io/en/stable/

Installation de packages pas encore sur pip as wheels #

De nombreux packages en python pur ne sont pas encore disponibles sur le Python Package Index en tant que roues, mais s’installent toujours correctement. Cependant, certains packages sous Windows génèrent la redoutable erreur vcvarsall.bat introuvable.

Le problème est que le package que vous essayez d’installer contient une extension C ou C++ et n’est pas actuellement disponible en tant que roue pré-construite à partir de l’index du package python, pypi, et sous Windows vous n’avez pas l’outil chaîne nécessaire pour construire de tels objets.

La réponse la plus simple est d’aller sur l’excellent site de [Christoph Gohlke][1] et de localiser la version appropriée des bibliothèques dont vous avez besoin. Par approprié dans le nom du package, un -cpNN**-** doit correspondre à votre version de python, c’est-à-dire que si vous utilisez Windows 32 bits python même sur win64, le nom doit inclure -win32- et si vous utilisez le python 64 bits, il doit inclure -win_amd64-, puis la version de python doit correspondre, c’est-à-dire que pour Python 34, le nom de fichier doit inclure *-cp *34-, etc. c’est essentiellement la magie que pip fait pour vous sur le site pypi.

Alternativement, vous devez obtenir le kit de développement Windows approprié pour la version de python que vous utilisez, les en-têtes de toute bibliothèque avec laquelle le package que vous essayez de créer des interfaces, éventuellement les en-têtes python pour la version de python, etc.

Python 2.7 utilisait Visual Studio 2008, Python 3.3 et 3.4 utilisaient Visual Studio 2010 et Python 3.5+ utilise Visual Studio 2015.

  • Installez “[Visual C++ Compiler Package for Python 2.7][2]”, qui est disponible sur le site Web de Microsoft ** ou **
  • Installez “[Windows SDK for Windows 7 and .NET Framework 4][3]” (v7.1), qui est disponible sur le site Web de Microsoft ou
  • Installez [Visual Studio 2015 Community Edition][4], (ou toute version ultérieure, lors de leur sortie), en vous assurant de sélectionner les options pour installer la prise en charge C et C++ n’est plus la valeur par défaut -* On me dit que le téléchargement et l’installation peuvent prendre jusqu’à * *** 8 heures ***, alors assurez-vous ** que ces options sont définies du premier coup.

** Ensuite ** vous * devrez peut-être * localiser les fichiers d’en-tête, * à la révision correspondante * pour toutes les bibliothèques auxquelles votre package souhaité est lié et les télécharger aux emplacements appropriés.

Enfin, vous pouvez laisser pip faire votre construction - bien sûr, si le paquet a des dépendances que vous n’avez pas encore, vous devrez peut-être également trouver les fichiers d’en-tête correspondants.

Alternatives : Il vaut également la peine de rechercher, à la fois sur pypi ou sur le site de Christop, toute version légèrement antérieure du package que vous recherchez, soit en python pur, soit pré-construite pour votre plate-forme et votre version de python et éventuellement les utiliser, s’ils sont trouvés, jusqu’à ce que votre colis soit disponible. De même, si vous utilisez la toute dernière version de python, vous constaterez peut-être qu’il faut un peu de temps aux responsables du paquet pour rattraper son retard, donc pour les projets qui ont vraiment ** besoin ** d’un paquet spécifique, vous devrez peut-être utiliser un python légèrement plus ancien pour le moment. Vous pouvez également consulter le site source des packages pour voir s’il existe une version fourchue disponible pré-construite ou en python pur et rechercher des packages alternatifs qui fournissent les fonctionnalités dont vous avez besoin mais qui sont disponibles - un exemple qui vient à l’esprit est le [Oreiller][5], maintenu activement, en remplacement de [PIL][6] actuellement non mis à jour depuis 6 ans et non disponible pour python 3.

Après-propos, j’encourage toute personne rencontrant ce problème à accéder au gestionnaire de bogues du paquet et à ajouter, ou à créer s’il n’y en a pas déjà un, un ticket poliment demandant que les responsables du paquet fournissez une roue sur pypi pour votre combinaison spécifique de plate-forme et de python, si cela est fait, les choses s’amélioreront normalement avec le temps, certains responsables de paquets ne se rendent pas compte qu’ils ont manqué une combinaison donnée que les gens peuvent utiliser.

Remarque sur l’installation des versions préliminaires #

Pip suit les règles de [Semantic Versioning][7] et préfère par défaut les packages publiés aux pré-versions. Donc, si un paquet donné a été publié en tant que V0.98 et qu’il existe également une version candidate V1.0-rc1, le comportement par défaut de pip install sera d’installer V0.98 - si vous le souhaitez installez la release candidate, il est conseillé de tester d’abord dans un environnement virtuel, vous pouvez l’activer avec --pip install --pre package-name ou --pip install --pre --upgrade nom-du-paquet. Dans de nombreux cas, les pré-versions ou les versions candidates peuvent ne pas avoir de roues conçues pour toutes les combinaisons de plates-formes et de versions, vous êtes donc plus susceptible de rencontrer les problèmes ci-dessus.

Remarque sur l’installation des versions de développement #

Vous pouvez également utiliser pip pour installer des versions de développement de packages à partir de github et d’autres emplacements, car un tel code est en flux, il est très peu probable qu’il ait des roues construites pour cela, donc tout package impur nécessitera la présence des outils de construction, et ils peuvent être cassé à tout moment, l’utilisateur est donc fortement encouragé à n’installer de tels packages que dans un environnement virtuel.

Trois options existent pour de telles installations :

  1. Téléchargez un instantané compressé, la plupart des systèmes de contrôle de version en ligne ont la possibilité de télécharger un instantané compressé du code. Cela peut être téléchargé manuellement puis installé avec pip install path/to/downloaded/file notez que pour la plupart des formats de compression, pip gérera la décompression dans une zone de cache, etc.
  2. Laissez pip gérer le téléchargement et l’installation pour vous avec : pip install URL/of/package/repository - vous devrez peut-être également utiliser --trusted-host, --client-cert et /ou --proxy flags pour que cela fonctionne correctement, en particulier dans un environnement d’entreprise. par exemple:
