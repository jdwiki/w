Syntaxe #

  • != - N’est pas égal à

  • == - Est égal à

  • > - supérieur à

  • < - inférieur à

  • >= - supérieur ou égal à

  • <= - inférieur ou égal à

  • is - teste si les objets sont exactement le même objet

  • n’est pas = teste si les objets ne sont pas exactement le même objet

Paramètres #

Paramètre Détails
x Premier article à comparer
y Deuxième élément à comparer

Comparaisons de chaînes #

Vous pouvez comparer plusieurs éléments avec plusieurs opérateurs de comparaison avec la comparaison en chaîne. Par exemple

x > y > z

est juste une forme abrégée de :

x > y and y > z

Cela sera évalué à True uniquement si les deux comparaisons sont True.

La forme générale est

a OP b OP c OP d ...

Où OP représente l’une des opérations de comparaison multiples que vous pouvez utiliser, et les lettres représentent des expressions valides arbitraires.

Notez que 0 != 1 != 0 est évalué à True, même si 0 != 0 est False. Contrairement à la notation mathématique courante dans laquelle x != y != z signifie que x, y et z ont des valeurs différentes. L’enchaînement des opérations == a le sens naturel dans la plupart des cas, puisque l’égalité est généralement transitive.

Style #

Il n’y a pas de limite théorique au nombre d’éléments et d’opérations de comparaison que vous utilisez tant que vous avez la syntaxe appropriée :

1 > -1 < 2 > 0.5 < 100 != 24

Ce qui précède renvoie “True” si chaque comparaison renvoie “True”. Cependant, utiliser un chaînage alambiqué n’est pas un bon style. Un bon chaînage sera “directionnel”, pas plus compliqué que

1 > x > -4 > y != 8

Effets secondaires #

Dès qu’une comparaison renvoie “False”, l’expression prend immédiatement la valeur “False”, en ignorant toutes les comparaisons restantes.

Notez que l’expression exp dans a > exp > b ne sera évaluée qu’une seule fois, alors que dans le cas de

a > exp and exp > b

exp sera calculé deux fois si a > exp est vrai.

Comparaison par is vs == #

Un écueil courant est de confondre les opérateurs de comparaison d’égalité is et ==.

a == b compare la valeur de a et b.

a est b comparera les identités de a et b.

Pour illustrer:

a = 'Python is fun!'
b = 'Python is fun!'
a == b # returns True
a is b # returns False

a = [1, 2, 3, 4, 5]
b = a      # b references a
a == b     # True
a is b     # True
b = a[:]   # b now references a copy of a
a == b     # True
a is b     # False [!!]

Fondamentalement, is peut être considéré comme un raccourci pour id(a) == id(b).

Au-delà de cela, il y a des bizarreries de l’environnement d’exécution qui compliquent encore les choses. Les chaînes courtes et les petits entiers renverront True par rapport à is, car la machine Python tente d’utiliser moins de mémoire pour des objets identiques.

a = 'short'
b = 'short'
c = 5
d = 5
a is b # True
c is d # True

Mais les chaînes plus longues et les entiers plus grands seront stockés séparément.

a = 'not so short'
b = 'not so short'
c = 1000
d = 1000
a is b # False
c is d # False

Vous devez utiliser is pour tester None :

if myvar is not None:
    # not None
    pass
if myvar is None:
    # None
    pass

Une utilisation de “est” est de tester une “sentinelle” (c’est-à-dire un objet unique).

sentinel = object()
def myfunc(var=sentinel):
    if var is sentinel:
        # value wasn’t provided
        pass
    else:
        # value was provided
        pass

Supérieur ou inférieur à #

x > y
x < y

Ces opérateurs comparent deux types de valeurs, ce sont les opérateurs inférieur à et supérieur à. Pour les nombres, cela compare simplement les valeurs numériques pour voir laquelle est la plus grande :

12 > 4
# True
12 < 4
# False
1 < 4
# True

Pour les chaînes, elles seront comparées lexicographiquement, ce qui est similaire à l’ordre alphabétique mais pas tout à fait le même.

"alpha" < "beta"
# True
"gamma" > "beta"
# True
"gamma" < "OMEGA"
# False

Dans ces comparaisons, les lettres minuscules sont considérées comme “plus grandes que” les majuscules, c’est pourquoi "gamma" < "OMEGA" est faux. S’ils étaient tous en majuscules, cela renverrait le résultat attendu du classement alphabétique :

"GAMMA" < "OMEGA"
# True

Chaque type définit son calcul différemment avec les opérateurs < et >, vous devez donc rechercher ce que signifient les opérateurs avec un type donné avant de l’utiliser.

Pas égal à #

x != y

Cela renvoie True si x et y ne sont pas égaux et sinon renvoie False.

12 != 1
# True
12 != '12'
# True
'12' != '12'
# False

Égal à #

x == y

Cette expression évalue si x et y sont la même valeur et renvoie le résultat sous la forme d’une valeur booléenne. Généralement, le type et la valeur doivent correspondre, donc l’int 12 n’est pas le même que la chaîne '12'.

12 == 12
# True
12 == 1
# False
'12' == '12'
# True
'spam' == 'spam'
# True
'spam' == 'spam '
# False
'12' == 12
# False

Notez que chaque type doit définir une fonction qui sera utilisée pour évaluer si deux valeurs sont identiques. Pour les types intégrés, ces fonctions se comportent comme prévu et évaluent simplement les choses en fonction de la même valeur. Cependant, les types personnalisés peuvent définir les tests d’égalité comme ils le souhaitent, y compris toujours renvoyer “True” ou toujours “False”.

Comparer des objets #

Afin de comparer l’égalité des classes personnalisées, vous pouvez remplacer == et != en définissant les méthodes __eq__ et __ne__. Vous pouvez également remplacer __lt__ (<), __le__ (<=), __gt__ (>) et __ge__ (>). Notez que vous n’avez besoin de remplacer que deux méthodes de comparaison, et Python peut gérer le reste (== est identique à not < et not >, etc.)

class Foo(object):
    def __init__(self, item):
        self.my_item = item
    def __eq__(self, other):
        return self.my_item == other.my_item
    
a = Foo(5)
b = Foo(5)
a == b     # True
a != b     # False
a is b     # False

Notez que cette comparaison simple suppose que “autre” (l’objet auquel est comparé) est le même type d’objet. La comparaison avec un autre type génère une erreur :

class Bar(object):
    def __init__(self, item):
        self.other_item = item
    def __eq__(self, other):
        return self.other_item == other.other_item
    def __ne__(self, other):
        return self.other_item != other.other_item
    
c = Bar(5)
a == c    # throws AttributeError: 'Foo' object has no attribute 'other_item'

La vérification de isinstance() ou similaire aidera à éviter cela (si vous le souhaitez).

Common Gotcha : Python n’applique pas la saisie #

Dans de nombreuses autres langues, si vous exécutez ce qui suit (exemple Java)

if("asgdsrf" == 0) {
    //do stuff
}

… vous obtiendrez une erreur. Vous ne pouvez pas simplement comparer des chaînes à des entiers comme ça. En Python, il s’agit d’une déclaration parfaitement légale - elle se résoudra simplement en “False”.

Un piège commun est le suivant

myVariable = "1"
if 1 == myVariable:
    #do stuff

Cette comparaison sera évaluée à “False” sans erreur, à chaque fois, cachant potentiellement un bogue ou brisant une condition.
