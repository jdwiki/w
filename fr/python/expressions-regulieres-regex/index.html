Python rend les expressions régulières disponibles via le module re.

Les expressions régulières sont des combinaisons de caractères qui sont interprétées comme des règles pour faire correspondre les sous-chaînes. Par exemple, l’expression 'amount\D+\d+' correspondra à toute chaîne composée du mot amount suivi d’un nombre entier, séparés par un ou plusieurs non-chiffres, tels que : amount=100, `amount est 3’, ‘montant est égal à : 33’, etc.

Syntaxe #

  • Expressions régulières directes

  • re.match(pattern, string, flag=0) # Out : correspond au modèle au début de la chaîne ou Aucun

  • re.search(pattern, string, flag=0) # Out : match pattern inside string ou None

  • re.findall(pattern, string, flag=0) # Out : liste de toutes les correspondances de pattern dans string ou []

  • re.finditer(pattern, string, flag=0) # Out : identique à re.findall, mais renvoie l’objet itérateur

  • re.sub(pattern, replacement, string, flag=0) # Out : chaîne avec remplacement (chaîne ou fonction) à la place du modèle

  • Expressions régulières précompilées

  • precompiled_pattern = re.compile(pattern, flag=0)

  • precompiled_pattern.match(string) # Out : correspondance au début de la chaîne ou None

  • precompiled_pattern.search(string) # Out : correspond n’importe où dans la chaîne ou Aucun

  • precompiled_pattern.findall(string) # Out : liste de toutes les sous-chaînes correspondantes

  • precompiled_pattern.sub(string/pattern/function, string) # Out : chaîne remplacée

Correspondant au début d’une chaîne #

Le premier argument de re.match() est l’expression régulière, le second est la chaîne à rechercher :

import re

pattern = r"123"
string = "123zzb"

re.match(pattern, string)
# Out: <_sre.SRE_Match object; span=(0, 3), match='123'>

match = re.match(pattern, string)

match.group()
# Out: '123'

Vous remarquerez peut-être que la variable de modèle est une chaîne préfixée par r, ce qui indique que la chaîne est un * littéral de chaîne brute *.

Un littéral de chaîne brut a une syntaxe légèrement différente d’un littéral de chaîne, à savoir une barre oblique inverse \ dans un littéral de chaîne brut signifie “juste une barre oblique inverse” et il n’est pas nécessaire de doubler les contrecoups pour échapper aux “séquences d’échappement” telles que les retours à la ligne ( \n), les tabulations (\t), les retours arrière (\), les sauts de page (\r), etc. Dans les littéraux de chaîne normaux, chaque barre oblique inverse doit être doublée pour éviter d’être considérée comme le début d’une séquence d’échappement.

Par conséquent, r"\n" est une chaîne de 2 caractères : \ et n. Les modèles Regex utilisent également des barres obliques inverses, par ex. \d fait référence à n’importe quel caractère numérique. Nous pouvons éviter d’avoir à double échapper nos chaînes ("\\d") en utilisant des chaînes brutes (r"\d").

Par exemple:

string = "\\t123zzb" # here the backslash is escaped, so there's no tab, just '\' and 't'
pattern = "\\t123"   # this will match \t (escaping the backslash) followed by 123
re.match(pattern, string).group()   # no match
re.match(pattern, "\t123zzb").group()  # matches '\t123'

pattern = r"\\t123"  
re.match(pattern, string).group()   # matches '\\t123'

La correspondance est effectuée à partir du début de la chaîne uniquement. Si vous voulez trouver une correspondance n’importe où, utilisez [re.search][1] à la place :

match = re.match(r"(123)", "a123zzb")

match is None
# Out: True

match = re.search(r"(123)", "a123zzb")

match.group()
# Out: '123'

[1] : https://www.wikiod.com/fr/python/expressions-regulieres-regex#Recherche

Recherche #

pattern = r"(your base)"
sentence = "All your base are belong to us."

match = re.search(pattern, sentence)
match.group(1)
# Out: 'your base'

match = re.search(r"(belong.*)", sentence)
match.group(1)
# Out: 'belong to us.'

La recherche est effectuée n’importe où dans la chaîne contrairement à re.match. Vous pouvez également utiliser re.findall.

Vous pouvez également rechercher au début de la chaîne (utilisez ^),

match = re.search(r"^123", "123zzb")
match.group(0)
# Out: '123'

match = re.search(r"^123", "a123zzb")
match is None
# Out: True

à la fin de la chaîne (utilisez $),

match = re.search(r"123$", "zzb123")
match.group(0)
# Out: '123'

match = re.search(r"123$", "123zzb")
match is None
# Out: True

ou les deux (utilisez à la fois ^ et $):

match = re.search(r"^123$", "123")
match.group(0)
# Out: '123'

Modèles précompilés #

import re

precompiled_pattern = re.compile(r"(\d+)")
matches = precompiled_pattern.search("The answer is 41!")
matches.group(1)
# Out: 41

matches = precompiled_pattern.search("Or was it 42?")
matches.group(1)
# Out: 42

Compiler un pattern permet de le réutiliser ultérieurement dans un programme. Cependant, notez que Python met en cache les expressions récemment utilisées ([docs][1], SO answer), donc * “les programmes qui n’utilisent que quelques expressions régulières à la fois, vous n’avez pas à vous soucier de compiler des expressions régulières”*.

import re

precompiled_pattern = re.compile(r"(.*\d+)")
matches = precompiled_pattern.match("The answer is 41!")
print(matches.group(1))
# Out: The answer is 41

matches = precompiled_pattern.match("Or was it 42?")
print(matches.group(1))
# Out: Or was it 42

Il peut être utilisé avec re.match().

[1] : https://docs.python.org/3/library/re.html#re.compile

Drapeaux #

Pour certains cas particuliers, nous devons modifier le comportement de l’expression régulière, cela se fait à l’aide de drapeaux. Les drapeaux peuvent être définis de deux manières, via le mot clé flags ou directement dans l’expression.

Mot-clé de drapeaux #

Ci-dessous un exemple pour re.search mais cela fonctionne pour la plupart des fonctions du module re.

m = re.search("b", "ABC")  
m is None
# Out: True

m = re.search("b", "ABC", flags=re.IGNORECASE)
m.group()
# Out: 'B'

m = re.search("a.b", "A\nBC", flags=re.IGNORECASE) 
m is None
# Out: True

m = re.search("a.b", "A\nBC", flags=re.IGNORECASE|re.DOTALL) 
m.group()
# Out: 'A\nB'

Drapeaux communs

Drapeau Brève description
re.IGNORECASE, re.I Fait que le motif ignore la casse
re.DOTALL, re.S Fait en sorte que . corresponde à tout, y compris les retours à la ligne
re.MULTILINE, re.M Fait correspondre ^ au début d’une ligne et $ à la fin d’une ligne
re.DEBUG Active les informations de débogage

Pour la liste complète de tous les drapeaux disponibles, consultez les [docs][1]

Drapeaux en ligne #

À partir de la [docs] [2] :

(?iLmsux) (Une ou plusieurs lettres de l’ensemble ‘i’, ‘L’, ’m’, ’s’, ‘u’, ‘x’.)

Le groupe correspond à la chaîne vide ; les lettres définissent les drapeaux correspondants : re.I (ignorer la casse), re.L (dépendant des paramètres régionaux), re.M (multiligne), re.S (le point correspond à tous), re.U (dépendant d’Unicode) et re.X (verbeux), pour toute l’expression régulière. Ceci est utile si vous souhaitez inclure les drapeaux dans l’expression régulière, au lieu de passer un argument de drapeau à la fonction re.compile().

Notez que le drapeau (?x) change la façon dont l’expression est analysée. Il doit être utilisé en premier dans la chaîne d’expression ou après un ou plusieurs caractères d’espacement. S’il y a des caractères non blancs avant le drapeau, les résultats sont indéfinis.

[1] : https://docs.python.org/2/library/re.html#module-contents [2] : https://docs.python.org/2/library/re.html#regular-expression-syntax

Remplacement #

Des remplacements peuvent être effectués sur des chaînes en utilisant re.sub.

Remplacement des chaînes #

re.sub(r"t[0-9][0-9]", "foo", "my name t13 is t44 what t99 ever t44")
# Out: 'my name foo is foo what foo ever foo'

Utilisation des références de groupe #

Les remplacements avec un petit nombre de groupes peuvent être effectués comme suit :

re.sub(r"t([0-9])([0-9])", r"t\2\1", "t13 t19 t81 t25")
# Out: 't31 t91 t18 t52'

Cependant, si vous créez un ID de groupe comme ‘10’, cela ne fonctionne pas: ‘\10’ est lu comme ’numéro d’identification 1 suivi de 0’. Vous devez donc être plus précis et utiliser la notation \g<i> :

re.sub(r"t([0-9])([0-9])", r"t\g<2>\g<1>", "t13 t19 t81 t25")
# Out: 't31 t91 t18 t52'

Utiliser une fonction de remplacement #

items = ["zero", "one", "two"]
re.sub(r"a\[([0-3])\]", lambda match: items[int(match.group(1))], "Items: a[0], a[1], something, a[2]")
# Out: 'Items: zero, one, something, two'

## Rechercher toutes les correspondances qui ne se chevauchent pas

re.findall(r"[0-9]{2,3}", "some 1 text 12 is 945 here 4445588899")
# Out: ['12', '945', '444', '558', '889']

Notez que le r avant "[0-9]{2,3}" indique à python d’interpréter la chaîne telle quelle ; sous forme de chaîne “brute”.

Vous pouvez également utiliser re.finditer() qui fonctionne de la même manière que re.findall() mais renvoie un itérateur avec des objets SRE_Match au lieu d’une liste de chaînes :

results = re.finditer(r"([0-9]{2,3})", "some 1 text 12 is 945 here 4445588899")
print(results)
# Out: <callable-iterator object at 0x105245890>
for result in results:
     print(result.group(0))
''' Out:
12
945
444
558
889
'''

Vérification des caractères autorisés #

Si vous voulez vérifier qu’une chaîne ne contient qu’un certain ensemble de caractères, dans ce cas a-z, A-Z et 0-9, vous pouvez le faire comme ceci,

import re

def is_allowed(string):
    characherRegex = re.compile(r'[^a-zA-Z0-9.]')
    string = characherRegex.search(string)
    return not bool(string)
    
print (is_allowed("abyzABYZ0099")) 
# Out: 'True'

print (is_allowed("#*@#$%^")) 
# Out: 'False'

Vous pouvez également adapter la ligne d’expression de [^a-zA-Z0-9.] à [^a-z0-9.], pour interdire les majuscules par exemple.

Crédit partiel : http://stackoverflow.com/a/1325265/2697955

Fractionner une chaîne à l’aide d’expressions régulières #

Vous pouvez également utiliser des expressions régulières pour diviser une chaîne. Par exemple,

import re
data = re.split(r'\s+', 'James 94 Samantha 417 Scarlett 74')
print( data )
# Output: ['James', '94', 'Samantha', '417', 'Scarlett', '74']

Regroupement #

Le regroupement se fait avec des parenthèses. L’appel de group() renvoie une chaîne formée des sous-groupes correspondants entre parenthèses.

match.group() # Group without argument returns the entire match found
# Out: '123'
match.group(0) # Specifying 0 gives the same result as specifying no argument
# Out: '123'

Des arguments peuvent également être fournis à group() pour récupérer un sous-groupe particulier.

À partir de la [docs] [1] :

S’il y a un seul argument, le résultat est une seule chaîne ; s’il y a plusieurs arguments, le résultat est un tuple avec un élément par argument.

Appeler groups(), d’autre part, renvoie une liste de tuples contenant les sous-groupes.

sentence = "This is a phone number 672-123-456-9910"
pattern = r".*(phone).*?([\d-]+)"

match = re.match(pattern, sentence)

match.groups()   # The entire match as a list of tuples of the paranthesized subgroups
# Out: ('phone', '672-123-456-9910')

m.group()        # The entire match as a string
# Out: 'This is a phone number 672-123-456-9910'

m.group(0)       # The entire match as a string
# Out: 'This is a phone number 672-123-456-9910'

m.group(1)       # The first parenthesized subgroup.
# Out: 'phone'

m.group(2)       # The second parenthesized subgroup.
# Out: '672-123-456-9910'

m.group(1, 2)    # Multiple arguments give us a tuple.
# Out: ('phone', '672-123-456-9910')

groupes nommés #

match = re.search(r'My name is (?P<name>[A-Za-z ]+)', 'My name is John Smith')
match.group('name')
# Out: 'John Smith'

match.group(1)
# Out: 'John Smith'

Crée un groupe de capture qui peut être référencé par nom ainsi que par index.

Groupes non capturants #

L’utilisation de (?:) crée un groupe, mais le groupe n’est pas capturé. Cela signifie que vous pouvez l’utiliser en groupe, mais cela ne polluera pas votre “espace de groupe”.

re.match(r'(\d+)(\+(\d+))?', '11+22').groups()
# Out: ('11', '+22', '22')

re.match(r'(\d+)(?:\+(\d+))?', '11+22').groups()
# Out: ('11', '22')

Cet exemple correspond à 11+22 ou 11, mais pas à 11+. C’est parce que le signe + et le deuxième terme sont regroupés. Par contre, le signe + n’est pas capturé.

[1] : https://docs.python.org/2/library/re.html#re.MatchObject.group

Caractères spéciaux d’échappement #

Les caractères spéciaux (comme les crochets de classe de caractères [ et ] ci-dessous) ne correspondent pas littéralement :

match = re.search(r'[b]', 'a[b]c')
match.group()
# Out: 'b'

En échappant les caractères spéciaux, ils peuvent être mis en correspondance littéralement :

match = re.search(r'\[b\]', 'a[b]c')
match.group()
# Out: '[b]'

La fonction re.escape() peut être utilisée pour faire cela pour vous :

re.escape('a[b]c')
# Out: 'a\\[b\\]c'
match = re.search(re.escape('a[b]c'), 'a[b]c')
match.group()
# Out: 'a[b]c'

La fonction re.escape() échappe tous les caractères spéciaux, elle est donc utile si vous composez une expression régulière basée sur l’entrée de l’utilisateur :

username = 'A.C.'  # suppose this came from the user
re.findall(r'Hi {}!'.format(username), 'Hi A.C.! Hi ABCD!')
# Out: ['Hi A.C.!', 'Hi ABCD!']
re.findall(r'Hi {}!'.format(re.escape(username)), 'Hi A.C.! Hi ABCD!')
# Out: ['Hi A.C.!']

Itérer sur les correspondances en utilisant re.finditer #

Vous pouvez utiliser re.finditer pour parcourir toutes les correspondances d’une chaîne. Cela vous donne (par rapport à re.findall des informations supplémentaires, telles que des informations sur l’emplacement de la correspondance dans la chaîne (index):

import re
text = 'You can try to find an ant in this string'
pattern = 'an?\w' # find 'an' either with or without a following word character

for match in re.finditer(pattern, text):
    # Start index of match (integer)
    sStart = match.start()

    # Final index of match (integer)
    sEnd = match.end()

    # Complete match (string)
    sGroup = match.group()

    # Print match
    print('Match "{}" found at: [{},{}]'.format(sGroup, sStart,sEnd))

Résultat:

Match "an" found at: [5,7]
Match "an" found at: [20,22]
Match "ant" found at: [23,26]

Faire correspondre une expression uniquement à des emplacements spécifiques #

Souvent, vous souhaitez faire correspondre une expression uniquement à des endroits spécifiques (en les laissant intacts dans d’autres, c’est-à-dire). Considérez la phrase suivante :

An apple a day keeps the doctor away (I eat an apple everyday).

Ici, la “pomme” se produit deux fois, ce qui peut être résolu avec les soi-disant * verbes de contrôle de retour en arrière * qui sont pris en charge par le nouveau module regex . L’idée est :

forget_this | or this | and this as well | (but keep this)

Avec notre exemple de pomme, ce serait :

import regex as re
string = "An apple a day keeps the doctor away (I eat an apple everyday)."
rx = re.compile(r'''
    \([^()]*\) (*SKIP)(*FAIL)  # match anything in parentheses and "throw it away"
    |                          # or
    apple                      # match an apple
    ''', re.VERBOSE)
apples = rx.findall(string)
print(apples)
# only one

Cela correspond à “apple” uniquement lorsqu’il peut être trouvé en dehors des parenthèses. Voici comment ça fonctionne:

  • En regardant de gauche à droite, le moteur regex consomme tout à gauche, le (*SKIP) agit comme une “affirmation toujours vraie”. Ensuite, il échoue correctement sur (*FAIL) et revient en arrière.
  • Maintenant, il arrive au point de (*SKIP) de droite à gauche (c’est-à-dire en revenant en arrière) où il est interdit d’aller plus loin vers la gauche. Au lieu de cela, le moteur est invité à jeter tout ce qui se trouve à gauche et à sauter au point où le (*SKIP) a été invoqué.
