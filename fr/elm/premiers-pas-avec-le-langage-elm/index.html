Installation #

Pour commencer le développement avec Elm, vous devez installer un ensemble d’outils appelé [elm-platform][1].

Il comprend : [elm-make][2], [elm-reactor][3], [elm-repl][4] et [elm-package][5].

Tous ces outils sont disponibles via CLI, c’est-à-dire que vous pouvez les utiliser depuis votre terminal.

Choisissez l’une des méthodes suivantes pour installer Elm :

Utilisation du programme d’installation #

Téléchargez le programme d’installation à partir du [site officiel] [6] et suivez l’assistant d’installation.

Utiliser npm #

Vous pouvez utiliser [Node Package Manager][7] pour installer la plate-forme Elm.

Installation globale :

$ npm install elm -g

Mise en place locale :

$ npm install elm

Les outils de la plateforme Elm installés localement sont accessibles via :

$ ./node_modules/.bin/elm-repl  # launch elm-repl from local node_modules/

Utilisation de l’homebrew #

$ brew install elm

Basculer entre les versions avec elm-use #

Installer elm-use

$ npm install -g elm-use

Basculer vers une version plus ancienne ou plus récente d’elm

$ elm-use 0.18  // or whatever version you want to use

Lectures complémentaires #

Apprenez à [Initialiser et compiler][8] votre premier projet.

[1] : https://github.com/elm-lang/elm-platform [2] : https://github.com/elm-lang/elm-make [3] : https://github.com/elm-lang/elm-reactor [4] : https://github.com/elm-lang/elm-repl [5] : https://github.com/elm-lang/elm-package [6] : http://elm-lang.org/install [7] : https://www.npmjs.com/ [8] : https://www.wikiod.com/fr/elm/premiers-pas-avec-le-langage-elm#Initialiser et compiler

Initialiser et compiler #

Vous devriez avoir la plate-forme Elm installée sur votre ordinateur, le tutoriel suivant est écrit avec l’hypothèse que vous êtes familier avec le terminal.

Initialisation #

Créez un dossier et accédez-y avec votre terminal :

$ mkdir elm-app
$ cd elm-app/

Initialisez le projet Elm et installez les dépendances principales :

$ elm-package install -y

Les dossiers elm-package.json et elm-stuff doivent apparaître dans votre projet.

Créez le point d’entrée de votre application “Main.elm” et collez-y l’exemple [Hello World][1].

Construire le projet #

Pour créer votre premier projet, exécutez :

$ elm-make Main.elm

Cela produira index.html avec le fichier Main.elm (et toutes les dépendances) compilé en JavaScript et intégré dans le HTML. Essayez de l’ouvrir dans votre navigateur !

Si cela échoue avec l’erreur Je ne peux pas trouver le module 'Html'. cela signifie que vous n’utilisez pas la dernière version d’Elm. Vous pouvez résoudre le problème soit en mettant à jour Elm et en refaisant la première étape, soit avec la commande suivante :

$ elm-package install elm-lang/html -y

Si vous avez votre propre fichier index.html (par exemple, lorsque vous travaillez avec des ports), vous pouvez également compiler vos fichiers Elm dans un fichier JavaScript :

$ elm-make Main.elm --output=elm.js

Plus d’informations dans l’exemple [Embedding into HTML][2].

[1] : https://www.wikiod.com/fr/elm/premiers-pas-avec-le-langage-elm#Bonjour le monde [2] : https://www.wikiod.com/fr/elm/premiers-pas-avec-le-langage-elm#Intégration dans HTML

Guide de style et format orme #

Le guide de style officiel se trouve sur la page d’accueil et concerne généralement :

  • lisibilité (au lieu de compacité)
  • facilité de modification -diffs propres

Cela signifie que, par exemple, ceci :

homeDirectory : String
homeDirectory =
  "/root/files"


evaluate : Boolean -> Bool
evaluate boolean =
  case boolean of
    Literal bool ->
        bool

    Not b ->
        not (evaluate b)

    And b b' ->
        evaluate b && evaluate b'

    Or b b' ->
        evaluate b || evaluate b'

est considéré comme meilleur que :

homeDirectory = "/root/files"

eval boolean = case boolean of
    Literal bool -> bool
    Not b        -> not (eval b)
    And b b'     -> eval b && eval b'
    Or b b'      -> eval b || eval b'

L’outil elm-format vous aide en formatant votre code source pour vous automatiquement (généralement lors de l’enregistrement), dans la même veine que le langage Go gofmt . Encore une fois, la valeur sous-jacente est d’avoir un style cohérent et d’économiser des arguments et des guerres de flammes sur divers problèmes comme tabulations contre espaces ou longueur d’indentation.

Vous pouvez installer elm-format en suivant les instructions sur le [dépôt Github](https://github.com/avh4/elm- format). Ensuite, configurez votre éditeur pour formater les fichiers Elm automatiquement ou exécutez elm-format FILE_OR_DIR --yes manuellement.

Intégration dans HTML #

Il existe trois possibilités pour insérer du code Elm dans une page HTML existante.

Intégrer dans la balise body #

En supposant que vous ayez compilé l’exemple [Hello World][1] dans le fichier elm.js, vous pouvez laisser Elm reprendre la balise <body> comme suit :

<!DOCTYPE html>
<html>
    <body>
        <script src="elm.js"></script>
        <script>
          Elm.Main.fullscreen()
        </script>
    </body>
</html>

AVERTISSEMENT : Parfois, certaines extensions chromées interfèrent avec <body>, ce qui peut entraîner l’interruption de la production de votre application. Il est recommandé de toujours intégrer dans une div spécifique. Plus d’infos ici.

Intégrer dans une Div (ou un autre nœud DOM) #

Alternativement, en fournissant un élément HTML concret, le code Elm peut être exécuté dans cet élément de page spécifique :

<!DOCTYPE html>
<html>
    <head>
        <title>Hello World</title>
    </head>
    <body>
        <div id='app'></div>
        <script src="elm.js"></script>
        <script>
            Elm.Main.embed(document.getElementById('app'))
        </script>
    </body>
</html>

Intégrer en tant que travailleur Web (pas d’interface utilisateur) #

Le code Elm peut également être démarré en tant que travailleur et communiquer via [ports][2] :

<!DOCTYPE html>
<html>
    <head>
        <title>Hello Worker</title>
    </head>
    <body>
        <script src="elm.js"></script>
        <script>
            var app = Elm.Main.worker();
            app.ports.fromElmToJS.subscribe(function(world) {
                console.log(world)
            });
            app.ports.fromJSToElm.send('hello');
        </script>
    </body>
</html>

[1] : https://www.wikiod.com/fr/elm/premiers-pas-avec-le-langage-elm#Bonjour le monde [2] : https://www.wikiod.com/fr/elm/ports-interoperabilite-js

REPL #

Une bonne façon d’en savoir plus sur Elm est d’essayer d’écrire quelques expressions dans la REPL (Read-Eval-Print Loop). Ouvrez une console dans votre dossier elm-app (que vous avez créé lors de la phase [Initialize and build][1]) et essayez ce qui suit :

$ elm repl
---- elm-repl 0.17.1 -----------------------------------------------------------
 :help for help, :exit to exit, more at <https://github.com/elm-lang/elm-repl>
--------------------------------------------------------------------------------
> 2 + 2
4 : number
> \x -> x
<function> : a -> a
> (\x -> x + x)
<function> : number -> number
> (\x -> x + x) 2
4 : number
>

elm-repl est en fait un outil assez puissant. Supposons que vous créiez un fichier Test.elm dans votre dossier elm-app avec le code suivant :

module Test exposing (..)


a = 1


b = "Hello"

Maintenant, vous revenez à votre REPL (qui est resté ouvert) et tapez :

import Test exposing (..)
> a
1 : number
> b
"Hello" : String
>

Encore plus impressionnant, si vous ajoutez une nouvelle définition à votre fichier Test.elm, comme

s = """
Hello,
Goodbye.
"""

Enregistrez votre fichier, revenez une fois de plus à votre REPL, et sans réimporter “Test”, la nouvelle définition est disponible immédiatement :

> s
"\nHello,\nGoodbye.\n" : String
>

C’est très pratique lorsque vous voulez écrire des expressions qui s’étendent sur plusieurs lignes. C’est aussi très utile pour tester rapidement les fonctions que vous venez de définir. Ajoutez ce qui suit à votre fichier :

f x =
  x + x * x

Enregistrez et revenez au REPL :

> f
<function> : number -> number
> f 2
6 : number
> f 4
20 : number
>

Chaque fois que vous modifiez et enregistrez un fichier que vous avez importé, et que vous revenez au REPL et essayez de faire quoi que ce soit, le fichier complet est recompilé. Par conséquent, il vous informera de toute erreur dans votre code. Ajoute ça:

c = 2 ++ 2

Essayez ça :

> 0
-- TYPE MISMATCH -------------------------------------------------- ././Test.elm

The left argument of (++) is causing a type mismatch.

22|     2 ++ 2
        ^
(++) is expecting the left argument to be a:

    appendable

But the left argument is:

    number

Hint: Only strings, text, and lists are appendable.


>

Pour conclure cette introduction au REPL, ajoutons que elm-repl connaît également les packages que vous avez installés avec elm package install. Par exemple:

> import Html.App
> Html.App.beginnerProgram
<function>
    : { model : a, update : b -> a -> a, view : a -> Html.Html b }
      -> Platform.Program Basics.Never
>

[1] : https://www.wikiod.com/fr/elm/premiers-pas-avec-le-langage-elm#Initialiser et compiler

Serveur de construction local (Elm Reactor) #

Elm Reactor est l’outil indispensable pour le prototypage de votre application.

Veuillez noter que vous ne pourrez pas compiler Main.elm avec Elm Reactor, si vous utilisez [Http.App.programWithFlags][1] ou [Ports][2]

L’exécution d’elm-reactor dans un répertoire de projets démarrera un serveur Web avec un explorateur de projets, qui vous permettra de compiler chaque composant séparé.

Toutes les modifications que vous apportez à votre code sont mises à jour lorsque vous rechargez la page.

$ elm-reactor                     # launch elm-reactor on localhost:8000
$ elm-reactor -a=0.0.0.0 -p=3000  # launch elm-reactor on 0.0.0.0:3000
