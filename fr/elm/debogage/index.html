Syntaxe #

  • Debug.log “tag” anyValue

Debug.log prend deux paramètres, une String pour étiqueter la sortie de débogage dans la console (afin que vous sachiez d’où elle vient / à quoi correspond le message), et une valeur de n’importe quel type. Debug.log exécute l’effet secondaire de la journalisation de la balise et de la valeur dans la console JavaScript, puis renvoie la valeur. L’implémentation dans JS pourrait ressembler à :

function log (tag, value){
    console.log(tag, value);
    return value
}

JavaScript a des conversions implicites, donc value n’a pas besoin d’être explicitement converti en String pour que le code ci-dessus fonctionne. Cependant, les types Elm doivent être explicitement convertis en String, et le code natif pour [Debug.log](https://github.com/elm-lang/core/blob/4.0.3/src/Native /Debug.js#L7) le montre en action.

Enregistrer une valeur sans interrompre les calculs #

Le deuxième argument de Debug.log est toujours renvoyé, vous pouvez donc écrire du code comme celui-ci et cela fonctionnerait simplement :

update : Msg -> Model -> (Model, Cmd Msg)
update msg model =
    case Debug.log "The Message" msg of
        Something ->
            ...

Remplacer case msg of par case Debug.log "The Message" msg of entraînera l’enregistrement du message actuel dans la console à chaque fois que la fonction de mise à jour est appelée, mais ne change rien d’autre.

Transférer un Debug.log #

Au moment de l’exécution, ce qui suit afficherait une liste d’URL dans votre console et continuerait le calcul

payload =
    [{url:..., title:...}, {url=..., title=...}]

main = 
    payload
        |> List.map .url -- only takes the url
        |> Debug.log " My list of URLs" -- pass the url list to Debug.log and return it
        |> doSomething -- do something with the url list

Débogueur voyageant dans le temps #

Au moment de la rédaction (juillet 2016), [elm-reactor][1] a été temporairement dépouillé de sa fonctionnalité de voyage dans le temps. Il est cependant possible de l’obtenir en utilisant le package [jinjor/elm-time-travel][2].

Son utilisation reflète les fonctions program* des modules [Html.App][3] ou [Navigation][4], par exemple au lieu de :

import Html.App

main =
    Html.App.program
        { init = init
        , update = update
        , view = view
        , subscriptions = subscriptions
        }

tu écrirais :

import TimeTravel.Html.App

main =
    TimeTravel.Html.App.program
        { init = init
        , update = update
        , view = view
        , subscriptions = subscriptions
        }

(Bien sûr, après avoir installé le paquet avec elm-package.)

L’interface de votre application change en conséquence, [voir l’une des démos][5].

Depuis la version 0.18.0, vous pouvez simplement compiler votre programme avec le drapeau --debug et obtenir [le débogage du voyage dans le temps] [6] sans effort supplémentaire.

[1] : https://github.com/elm-lang/elm-reactor#note-about-time-travel [2] : http://package.elm-lang.org/packages/jinjor/elm-time-travel/latest/ [3] : https://www.wikiod.com/fr/elm/larchitecture-de-lorme [4] : http://package.elm-lang.org/packages/elm-lang/navigation/1.0.0/Navigation [5] : http://jinjor.github.io/elm-time-travel/ [6] : http://elm-lang.org/blog/the-perfect-bug-report “le rapport de bug parfait”

Debug. Crash #

case thing of
    Cat ->
        meow
    Bike ->
        ride
    Sandwich ->
        eat
    _ ->
        Debug.crash "Not yet implemented"

Vous pouvez utiliser [Debug.crash][1] lorsque vous voulez que le programme échoue, généralement utilisé lorsque vous êtes en train d’implémenter une expression case. Il n’est pas recommandé d’utiliser Debug.crash au lieu d’utiliser un type Maybe ou Result pour les entrées inattendues, mais généralement uniquement au cours du développement (c’est-à-dire que vous ne publierez généralement pas de code Elm qui utilise Debug. crash).

Debug.crash prend une valeur String, le message d’erreur à afficher en cas de plantage. Notez qu’Elm affichera également le nom du module et la ligne du crash, et si le crash est dans une expression case, il indiquera la valeur du case.

[1] : http://package.elm-lang.org/packages/elm-lang/core/latest/Debug#crash
