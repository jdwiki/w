Syntaxe #

  • – définir une fonction sans arguments ressemble à la simple définition d’une valeur language = “orme”
  • – appeler une fonction sans argument en indiquant son nom Langue
  • – les paramètres sont séparés par des espaces et suivent le nom de la fonction ajouter x y = x + y
  • – appeler une fonction de la même manière ajouter 5 2
  • – appliquer partiellement une fonction en ne fournissant que certains de ses paramètres incrément = ajouter 1
  • – utilisez l’opérateur |> pour passer l’expression de gauche à la fonction de droite dix = 9 |> incrément
  • – le <| l’opérateur passe l’expression de droite à la fonction de gauche incrément <| ajouter 5 4
  • – enchaîner/composer deux fonctions avec l’opérateur » backsYell = String.reverse » String.toUpper
  • – le « fonctionne de la même manière dans le sens inverse backsYell = String.toUpper « String.reverse
  • – une fonction avec un nom non alphanumérique entre parenthèses crée un nouvel opérateur (#) x y = x * y dix = 5 # 2
  • – tout opérateur infixe devient une fonction normale lorsque vous l’enveloppez entre parenthèses dix = (+) 5 5
  • – des annotations de type facultatives apparaissent au-dessus des déclarations de fonction isTen : Int -> Bool isTen n = si n == 10 alors Vrai sinon Faux

Application partielle #

L’application partielle signifie appeler une fonction avec moins d’arguments qu’elle n’en a et enregistrer le résultat sous une autre fonction (qui attend le reste des arguments).

multiplyBy: Int -> Int -> Int    
multiplyBy x y =
    x * y


multiplyByTwo : Int -> Int  -- one Int has disappeared! we now know what x is.
multiplyByTwo =
    multiplyBy 2


> multiplyByTwo 2
4 : Int

> multiplyByTwo 4
8 : Int

En tant que sidenote académique, Elm peut le faire à cause de [curry] [1] dans les coulisses.

[1] : https://en.wikipedia.org/wiki/Currying

Variables locales #

Il est possible de définir des variables locales à l’intérieur d’une fonction pour

  • réduire la répétition du code
  • donner un nom aux sous-expressions
  • réduire le nombre d’arguments passés.

La construction pour cela est let ... in ....

bigNumbers =
    let
        allNumbers =
            [1..100]

        isBig number =
            number > 95
    in
        List.filter isBig allNumbers

> bigNumbers
[96,97,98,99,100] : List number

> allNumbers
-- error, doesn't know what allNumbers is!

L’ordre des définitions dans la première partie de let n’a pas d’importance !

outOfOrder =
    let
        x =
            y + 1  -- the compiler can handle this

        y =
            100
    in
        x + y

> outOfOrder
201 : number

Aperçu #

La syntaxe d’application de fonction dans Elm n’utilise pas de parenthèses ni de virgules, et est plutôt sensible aux espaces.

Pour définir une fonction, spécifiez son nom multiplyByTwo et ses arguments x, toute opération après le signe égal = est ce qui est renvoyé d’une fonction.

multiplyByTwo x =
    x * 2

Pour appeler une fonction, indiquez son nom et ses arguments :

multiplyByTwo 2  -- 4

Notez qu’une syntaxe telle que multiplyByTwo(2) n’est pas nécessaire (même si le compilateur ne se plaint pas). Les parenthèses ne servent qu’à résoudre la priorité :

> multiplyByTwo multiplyByTwo 2
-- error, thinks it's getting two arguments, but it only needs one

> multiplyByTwo (multiplyByTwo 2)
4 : number

> multiplyByTwo 2 + 2
6 : number
-- same as (multiplyByTwo 2) + 2

> multiplyByTwo (2 + 2)
8 : number

Expressions lambda #

Elm a une syntaxe spéciale pour les expressions lambda ou les fonctions anonymes :

\arguments -> returnedValue

Par exemple, comme on le voit dans List.filter :

> List.filter (\num -> num > 1) [1,2,3]
[2,3] : List number

Plus en profondeur, une barre oblique inversée, \, est utilisée pour marquer le début de l’expression lambda, et la flèche, ->, est utilisée pour délimiter les arguments du corps de la fonction. S’il y a plus d’arguments, ils sont séparés par un espace :

normalFunction x y = x + y
-- is equivalent to
lambdaFunction = \x y -> x + y

> normalFunction 1 2
3 : number

> lambdaFunction 1 2
3 : number

Évaluation stricte et différée #

Dans elm, la valeur d’une fonction est calculée lorsque le dernier argument est appliqué. Dans l’exemple ci-dessous, le diagnostic de log sera imprimé lorsque f est invoqué avec 3 arguments ou qu’une forme curry de f est appliquée avec le dernier argument.

import String
import Debug exposing (log)

f a b c = String.join "," (log "Diagnostic" [a,b,c]) -- <function> : String -> String -> String -> String

f2 = f "a1" "b2" -- <function> : String -> String

f "A" "B" "C"
-- Diagnostic: ["A","B","C"]
"A,B,C" : String

f2 "c3"
-- Diagnostic: ["a1","b2","c3"]
"a1,b2,c3" : String

Parfois, vous souhaiterez empêcher l’application immédiate d’une fonction. Une utilisation typique dans elm est Lazy.lazy qui fournit une abstraction pour contrôler quand les fonctions sont appliqués.

lazy : (() -> a) -> Lazy a

Les calculs paresseux prennent une fonction d’un argument de type () ou Unit. Le type d’unité est classiquement le type d’un argument d’espace réservé. Dans une liste d’arguments, l’argument correspondant est spécifié sous la forme _, indiquant que la valeur n’est pas utilisée. La valeur unitaire dans elm est spécifiée par le symbole spécial () qui peut conceptuellement représenter un tuple vide ou un trou. Cela ressemble à la liste d’arguments vide en C, Javascript et d’autres langages qui utilisent des parenthèses pour les appels de fonction, mais c’est une valeur ordinaire.

Dans notre exemple, f peut être protégé contre une évaluation immédiate avec un lambda :

doit f = f () -- <function> : (() -> a) -> a
whatToDo = \_ -> f "a" "b" "c" -- <function> : a -> String
-- f is not evaluated yet

doit whatToDo
-- Diagnostic: ["a","b","c"]
"a,b,c" : String

L’évaluation de la fonction est retardée chaque fois qu’une fonction est partiellement appliquée.

defer a f = \_ -> f a -- <function> : a -> (a -> b) -> c -> b

delayF = f "a" "b" |> defer "c" -- <function> : a -> String

doit delayF
-- Diagnostic: ["a","b","c"]
"a,b,c" : String

Elm a une fonction always, qui ne peut pas être utilisée pour retarder l’évaluation. Étant donné que elm évalue tous les arguments de la fonction, que le résultat de l’application de la fonction soit utilisé ou non, envelopper une application de fonction dans always ne causera pas de retard, car f est entièrement appliqué en tant que paramètre à always.

alwaysF = always (f "a" "b" "c") -- <function> : a -> String
-- Diagnostic: ["a","b","c"] -- Evaluation wasn't delayed.

Opérateurs infixes et notation infixe #

Elm permet la définition d’opérateurs d’infixes personnalisés.

Les opérateurs infixes sont définis à l’aide de parenthèses autour du nom d’une fonction.

Considérez cet exemple d’opérateur infixe pour la construction Tuples 1 => True -- (1, True) :

(=>) : a -> b -> ( a, b )
(=>) a b =
    ( a, b )

La plupart des fonctions d’Elm sont définies en notation préfixée.

Appliquez n’importe quelle fonction en utilisant la notation infixe en spécifiant le premier argument avant le nom de la fonction entouré d’accents graves :

import List exposing (append)


append [1,1,2] [3,5,8]   -- [1,1,2,3,5,8]
[1,1,2] `append` [3,5,8] -- [1,1,2,3,5,8]
