Qu’est-ce que Selenium WebDriver ? #

Selenium est un ensemble d’outils conçus pour automatiser les navigateurs. Il est couramment utilisé pour les tests d’applications Web sur plusieurs plates-formes. Il existe quelques outils disponibles sous l’égide de Selenium, tels que Selenium WebDriver (ex-Selenium RC), Selenium IDE et Selenium Grid.

WebDriver est une interface de contrôle à distance qui vous permet de manipuler les éléments [DOM][1] dans les pages Web, ainsi que de commander le comportement des agents utilisateurs. Cette interface fournit un [wire protocol][2] indépendant du langage qui a été implémenté pour diverses plates-formes telles que :

  • [GeckoDriver][3] (Mozilla Firefox)
  • [ChromeDriver][4] (Google Chrome)
  • [SafariDriver] [5] (Apple Safari)
  • [Pilote InternetExplorer][6] (MS InternetExplorer)
  • [MicrosoftWebDriver ou EdgeDriver][7] (MS Edge)
  • [OperaChromiumDriver][8] (navigateur Opera)

ainsi que d’autres implémentations :

  • EventFiringWebDriver
  • HtmlUnitDriver -PhantomJSDriver
  • Pilote Web à distance

Selenium WebDriver est l’un des outils Selenium qui fournit des API orientées objet dans une variété de langages pour permettre plus de contrôle et l’application de pratiques de développement logiciel standard. Pour simuler avec précision la manière dont un utilisateur interagira avec une application Web, il utilise des “événements natifs au niveau du système d’exploitation” par opposition aux “événements JavaScript synthétisés”.

Liens à consulter :

  • http://www.seleniumhq.org/
  • http://www.aosabook.org/en/selenium.html
  • https://www.w3.org/TR/webdriver/

Installation ou configuration #

Pour commencer à utiliser WebDriver, vous devrez obtenir le pilote approprié sur le site Selenium : [Téléchargements Selenium HQ][1]. À partir de là, vous devez télécharger le pilote correspondant au(x) navigateur(s) et/ou plateforme(s) sur lesquels vous essayez d’exécuter WebDriver, par ex. si vous testiez dans Chrome, le site Selenium vous dirigera vers :

https://sites.google.com/a/chromium.org/chromedriver/

Pour télécharger chromedriver.exe.

Enfin, avant de pouvoir utiliser WebDriver, vous devrez télécharger les liaisons de langage appropriées, par ex. si vous utilisez C #, vous pouvez accéder au téléchargement à partir de la page de téléchargement de Selenium HQ pour obtenir les fichiers .dll requis ou, alternativement, les télécharger sous forme de packages dans Visual Studio via le gestionnaire de packages NuGet.

Les fichiers requis doivent maintenant être téléchargés, pour plus d’informations sur la façon de commencer à utiliser WebDriver, reportez-vous à l’autre documentation selenium-webdriver.

Pour Visual Studio [NuGet] #

Le moyen le plus simple d’installer Selenium WebDriver consiste à utiliser un gestionnaire de packages NuGet.

Dans votre projet, cliquez avec le bouton droit sur “Références”, puis cliquez sur “Gérer les packages NuGet” comme indiqué :

[![Fenêtre Visual Studio][2]][2]

Ensuite, tapez dans le champ de recherche “webdriver”. Vous devriez alors voir quelque chose comme ceci :

[![Fenêtre NuGet][3]][3]

Installez « Selenium.WebDriver » et « Selenium.Support » (le package de support comprend des ressources supplémentaires, telles que [Wait][4]) en cliquant sur le bouton Installer sur le côté droit.

Ensuite, vous pouvez installer vos WebDrivers que vous souhaitez utiliser, comme l’un de ceux-ci :

  • Selenium.WebDriver.ChromeDriver (Google Chrome)

  • [PhantomJS][5] (sans tête)

Installation ou configuration pour Java #

Afin d’écrire des tests en utilisant Selenium Webdriver et Java comme langage de programmation, vous devrez télécharger les fichiers JAR de Selenium Webdriver à partir du site Web de Selenium.

Il existe plusieurs façons de configurer un projet Java pour le pilote Web Selenium, l’une des plus simples étant d’utiliser Maven. Maven télécharge les liaisons Java requises pour le pilote Web Selenium, y compris toutes les dépendances. L’autre méthode consiste à télécharger les fichiers JAR et à les importer dans votre projet.

Étapes pour configurer le projet Selenium Webdriver à l’aide de Maven :

  1. Installez maven sur la boîte de fenêtres suivant ce document : https://maven.apache.org/install.html

  2. Créez un dossier avec le nom selenium-learing

  3. Créez un fichier dans le dossier ci-dessus en utilisant n’importe quel éditeur de texte avec le nom pom.xml

  4. Copiez le contenu ci-dessous dans pom.xml

     <project xmlns="http://maven.apache.org/POM/4.0.0"
      xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
      xsi:schemaLocation="http://maven.apache.org/POM/4.0.0 http://maven.apache.org/xsd/maven-4.0.0.xsd">
         <modelVersion>4.0.0</modelVersion>
         <groupId>SeleniumLearning</groupId>
         <artifactId>SeleniumLearning</artifactId>
         <version>1.0</version>
         <dependencies>
             <dependency>
                 <groupId>org.seleniumhq.selenium</groupId>
                 <artifactId>selenium-learning</artifactId>
                 <version>3.0.0-beta1</version>
             </dependency>
         </dependencies>
 </project>

Remarque : Assurez-vous que la version que vous avez spécifiée ci-dessus est la dernière. Vous pouvez vérifier la dernière version à partir d’ici : http://docs.seleniumhq.org/download/maven.jsp

  1. À l’aide de la ligne de commande, exécutez la commande ci-dessous dans le répertoire du projet.

    mvn clean install

La commande ci-dessus téléchargera toutes les dépendances requises et les ajoutera ensuite au projet. 6. Écrivez la commande ci-dessous pour générer un projet eclipse que vous pouvez importer dans l’IDE Eclipse.

`mvn eclipse:eclipse`
  1. Pour importer le projet dans eclipse ide, vous pouvez suivre les étapes ci-dessous

Ouvrir Elipse -> Fichier -> Importer -> Général -> Projet existant dans l’espace de travail -> Suivant -> Parcourir -> Localiser le dossier contenant pom.xml -> Ok -> Terminer

Installez le plugin m2eclipse en faisant un clic droit sur votre projet et sélectionnez Maven -> Enable Dependency Management.

Étapes pour configurer le projet Selenium Webdriver à l’aide de fichiers Jar

  1. Créez un nouveau projet dans Eclipse en suivant les étapes ci-dessous.

Ouvrir Elipse -> Fichier -> Nouveau -> Projet Java -> Indiquer un nom (selenium-learning) -> Terminer

  1. Téléchargez les fichiers jar depuis http://www.seleniumhq.org/download/. Vous devez télécharger Selenium Standalone Server et Selenium Client & WebDriver Language Bindings. Étant donné que ce document parle de Java, vous devez donc télécharger uniquement le fichier jar de la section Java. Jetez un oeil dans la capture d’écran ci-jointe.

[![entrez la description de l’image ici][1]][1]

Remarque : Selenium Standalone Server n’est requis que si vous souhaitez utiliser un serveur distant pour exécuter les tests. Étant donné que ce document est tout au-dessus de la mise en place du projet, il est donc préférable d’avoir tout en place.

  1. Les pots seront téléchargés dans un fichier zip, décompressez-les. Vous devriez pouvoir voir .jar directement.

[1] : http://i.stack.imgur.com/kPQwu.png

  1. Dans Eclipse, faites un clic droit sur le projet que vous avez créé à l’étape 1 et suivez les étapes ci-dessous.

Propriétés -> Chemin de compilation Java -> Sélectionnez l’onglet Bibliothèques -> Cliquez sur Ajouter des fichiers JAR externes -> Localisez le dossier JAR décompressé que vous avez téléchargé ci-dessus -> Sélectionnez tous les fichiers JAR du dossier lib -> Cliquez sur OK -> Cliquez à nouveau sur Ajouter Jars externes -> Localisez le même dossier décompressé -> Sélectionnez le jar qui se trouve en dehors du dossier lib (client-combined-3.0.0-beta1-nodeps.jar) -> Ok

De même, ajoutez le Selenium Standalone Server en suivant l’étape ci-dessus.

  1. Vous pouvez maintenant commencer à écrire du code sélénium dans votre projet.

PS : la documentation ci-dessus est basée sur la version bêta de sélénium-3.0.0, de sorte que les noms des fichiers jar spécifiés peuvent changer avec la version.
