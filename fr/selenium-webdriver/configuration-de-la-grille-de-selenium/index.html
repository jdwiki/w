Selenium Grid est un framework pour exécuter des tests répartis sur une gamme d’appareils de test. Il est utilisé pour tester les applications Web. Il est possible d’écrire des tests dans différents langages de programmation populaires, notamment C#, Groovy, Java, Perl, PHP, Python et Ruby. Les tests peuvent être exécutés sur une gamme de navigateurs Web sur des plates-formes telles que Windows, Linux et OS X.

Il s’agit d’un logiciel open source, publié sous la licence Apache 2.0 : les développeurs Web peuvent le télécharger et l’utiliser sans frais.

Syntaxe #

  • pour exécuter le fichier jar, voici la syntaxe de chaque fichier jar
  • java -jar <jar-file-full-name>.jar -<vos paramètres s'il y en a>

Paramètres #

Paramètres Détails
rôle Est ce qui indique au sélénium qu’il était hub ou node
port C’est pour spécifier quel port le hub ou node doit écouter.
hub Ce paramètre est utilisé dans node pour spécifier l’url du hub
nom_navigateur Il a été utilisé dans node pour spécifier le nom du navigateur comme firefox, chrome, internet explorer
maxInstances C’est là que l’instance du navigateur est spécifiée, par exemple. 5 signifie qu’il y aura 5 instances du navigateur spécifié par l’utilisateur qui seront présentes.
nodeConfig Un fichier de configuration Json pour le nœud. Vous pouvez spécifier le rôle, le port, etc. ici
hubConfig Un fichier de configuration Json pour le nœud. Vous pouvez spécifier le rôle, le port, le nombre maximal d’instances, etc. ici

Création d’un hub et d’un nœud Selenium Grid #

Création d’un hub #

Une configuration rapide pour une configuration de concentrateur et de nœud dans une grille de sélénium. Pour plus d’informations, voir : [Documents Grid 2][1]

Conditions #

Pour configurer un hub de grille, vous avez besoin du flux :

  • [Selenium-server-standalone-.jar][2]

Création du hub #

Pour créer un hub, vous devez exécuter le serveur Selenium.

  1. Téléchargez Selenium-server-standalone-.jar
  2. Ouvrez votre terminal et accédez au dossier où se trouve Selenium-server-standalone-.jar
  3. Exécutez la commande suivante :
    1. For default configuration java -jar selenium-server-standalone-<Version>.jar -role hub
    2. For Json configuration java -jar selenium-server-standalone-<Version>.jar -role hub -hubConfig hubConfig.json
  4. Ouvrez http://localhost:4444/ vous verrez un message suivant

[![entrez la description de l’image ici][3]][3]

En cliquant sur console -> View config pour afficher la configuration des détails du hub.

Création d’un nœud #

Conditions #

Pour configurer un hub de grille, vous avez besoin du flux :

  • Selenium-server-standalone-.jar
  • Webdrivers
    • [Chrome driver][4]
    • [FireFox driver][5]
    • [Microsoft Edge driver][6]
  • Navigateurs
    • [Chrome][7]
    • [FireFox][8]
    • [Microsoft Edge][9] (Windows 10)

Création du nœud #

Maintenant, pour créer des nœuds pour le concentrateur

  1. Téléchargez Selenium-server-standalone-.jar
  2. Téléchargez les navigateurs dans lesquels vous souhaitez tester
  3. Téléchargez les pilotes pour les navigateurs que vous souhaitez tester dans
  4. Ouvrez un nouveau terminal et accédez à l’emplacement du fichier jar du serveur Selenium
  5. Exécutez la commande suivante :
    1. for default configuration java -jar selenium-server-standalone-<VERSION NUMBER>.jar -role node
    2. For Json configuration java -jar selenium-server-standalone-<Version>.jar -role node -nodeConfig nodeConfig.json
  6. Allez maintenant sur http://localhost:4444/grid/console pour afficher les détails du nœud

[1] : https://github.com/SeleniumHQ/selenium/wiki/Grid2 “Documents Grid2” [2] : http://selenium-release.storage.googleapis.com/index.html “selenium-server-standalone-version.jar” [3] : http://i.stack.imgur.com/GzPVL.png [4] : https://sites.google.com/a/chromium.org/chromedriver/downloads “Pilote Chrome” [5] : https://github.com/mozilla/geckodriver/releases “pilote Firefox” [6] : https://developer.microsoft.com/en-us/microsoft-edge/tools/webdriver/ “Pilote Microsoft Edge” [7] : https://www.google.com/chrome/browser/desktop/index.html “Chrome” [8] : https://www.mozilla.org/nl/firefox/new/ “FireFox’” [9] : https://www.microsoft.com/nl-nl/windows/microsoft-edge “Microsoft Edge”

Code Java pour Selenium Grid #

String hubUrl = "http://localhost:4444/wd/hub"       
DesiredCapabilities capability = DesiredCapabilities.firefox(); //or which browser you want
RemoteWebDriver driver = new RemoteWebDriver(hubUrl, capability);
