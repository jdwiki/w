Exceptions dans Selenium-WebDriver

  • Exceptions Python

Sur cette page

  • Exceptions Python

Il existe un certain nombre d’exceptions qui peuvent être générées lors de l’utilisation d’un pilote Web. Les exemples ci-dessous sont destinés à donner une idée de ce qu’ils signifient.

Exceptions Python #

[Documentation sur les exceptions au sélénium][1]

ElementNotInteractableException : Levé lorsqu’un élément est présent dans le DOM mais que les interactions avec cet élément toucheront un autre élément en raison de l’ordre de peinture

  • ElementNotSelectableException : Levé lors d’une tentative de sélection d’un élément non sélectionnable. Exemples d’éléments non sélectionnables :
    • script
  • ElementNotVisibleException : Levé lorsqu’un élément est présent sur le DOM, mais qu’il n’est pas visible et qu’il est donc impossible d’interagir avec lui. Le plus souvent rencontré lorsque vous essayez de cliquer ou de lire le texte d’un élément qui est masqué.
  • ErrorInResponseException : Levé lorsqu’une erreur s’est produite côté serveur. Cela peut se produire lors de la communication avec l’extension firefox ou le serveur de pilote distant.
  • ImeActivationFailedException : Levée lorsque l’activation d’un moteur IME a échoué.
  • ImeNotAvailableException : Levée lorsque la prise en charge IME n’est pas disponible. Cette exception est levée pour chaque appel de méthode lié à IME si la prise en charge IME n’est pas disponible sur la machine.
  • InvalidArgumentException : Les arguments passés à une commande sont soit invalides, soit malformés.
  • InvalidCookieDomainException : Lève une tentative d’ajout d’un cookie sous un domaine différent de celui de l’URL actuelle.
  • InvalidElementStateException : Levée lorsqu’une action entraînerait un état invalide pour un élément. Sous-classes :
    • ElementNotInteractableException
    • ElementNotSelectableException
    • ElementNotVisibleException
  • InvalidSelectorException : Levée lorsque le sélecteur utilisé pour rechercher un élément ne renvoie pas de WebElement. Actuellement, cela ne se produit que lorsque le sélecteur est une expression xpath et qu’il est syntaxiquement invalide (c’est-à-dire qu’il ne s’agit pas d’une expression xpath) ou que l’expression ne sélectionne pas WebElements (par exemple, “count(//input)”).
  • InvalidSwitchToTargetException : Levée lorsque la cible du cadre ou de la fenêtre à basculer n’existe pas.
  • MoveTargetOutOfBoundsException : Levée lorsque la cible fournie à la méthode ActionsChains move() est invalide, c’est-à-dire hors du document.
  • NoAlertPresentException : Levé lors du passage à aucune alerte présentée. Cela peut être dû à l’appel d’une opération sur la classe Alert() lorsqu’une alerte n’est pas encore à l’écran.
  • NoSuchAttributeException : Levé lorsque l’attribut de l’élément est introuvable. Vous voudrez peut-être vérifier si l’attribut existe dans le navigateur particulier que vous testez. Certains navigateurs peuvent avoir des noms de propriété différents pour la même propriété. (.innerText d’IE8 contre Firefox .textContent)
  • NoSuchElementException : Levée lorsque l’élément est introuvable. Si vous rencontrez cette exception, vous pouvez vérifier ce qui suit :
    • Check your selector used in your find_by…
    • Element may not yet be on the screen at the time of the find operation, (webpage is still loading) see selenium.webdriver.support.wait.WebDriverWait() for how to write a wait wrapper to wait for an element to appear.
  • NoSuchFrameException : Levé lorsque la cible de l’image à changer n’existe pas.
  • NoSuchWindowException : Levée lorsque la fenêtre cible à changer n’existe pas. Pour trouver l’ensemble actuel de poignées de fenêtre actives, vous pouvez obtenir une liste des poignées de fenêtre actives de la manière suivante :
    print driver.window_handles
  • ExceptionRemoteDriverServer :
  • StaleElementReferenceException : Levée lorsqu’une référence à un élément est désormais “périmée”. Stale signifie que l’élément n’apparaît plus sur le DOM de la page. Les causes possibles de StaleElementReferenceException incluent, mais sans s’y limiter :
    • You are no longer on the same page, or the page may have refreshed since the element was located.
    • The element may have been removed and re-added to the screen, since it was located. Such as an element being relocated. This can happen typically with a javascript framework when values are updated and the node is rebuilt.
    • Element may have been inside an iframe or another context which was refreshed.
  • TimeoutException : Levée lorsqu’une commande ne se termine pas dans un délai suffisant.
  • UnableToSetCookieException : Levé lorsqu’un pilote ne parvient pas à définir un cookie.
  • UnexpectedAlertPresentException : Lève une alerte inattendue. Généralement déclenché lorsqu’un modal attendu bloque le formulaire du pilote Web en exécutant d’autres commandes.
  • UnexpectedTagNameException : Levée lorsqu’une classe de support n’a pas obtenu un élément Web attendu.
  • WebDriverException : Exception de base du pilote Web. Toutes les exceptions de pilote Web utilisent WebDriverException ou InvalidStateException comme classe parent.

[1] : https://seleniumhq.github.io/selenium/docs/api/py/common/selenium.common.exceptions.html
