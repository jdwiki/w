Démarrage rapide avec l'API Python des utilisateurs, l'authentification App Engine

  • Gestionnaire de page principale [views.py]
  • Routage d’application [urls.py]
  • Html, exemple frontal d’utilisation de l’API Users [index.html]

Sur cette page

  • Gestionnaire de page principale [views.py]
  • Routage d’application [urls.py]
  • Html, exemple frontal d’utilisation de l’API Users [index.html]

L’utilisation de l’API Users est un moyen très simple et flexible d’effectuer l’authentification dans App Engine, mais assurez-vous que votre application les cas ne nécessitent pas plus d’éléments pour l’environnement d’authentification.

Remarque : Si vous avez besoin de plus d’informations sur la structure traditionnelle d’une application App Engine, veuillez consulter ces [infos](https://cloud.google.com/appengine/docs/standard/python/getting-started/ création-livre d’or).

L’API Utilisateurs permet :

  • Détecter si l’utilisateur actuel s’est connecté.
  • Rediriger l’utilisateur vers la page de connexion appropriée pour se connecter.
  • Demandez à l’utilisateur de votre application de créer un nouveau compte Google s’il n’en a pas déjà un.

Référence et plus de détails

Éléments importants dans la vue :

Importer:

from google.appengine.api import users

Objet utilisateur et méthodes :

user = users.get_current_user()

Remarque : l’implémentation de jinja2 est facultative, mais l’article sert à expliquer complètement le flux de travail.

Gestionnaire de page principale [views.py] #

Importations générales, en utilisant jinja2 pour remplir les modèles en htmls.

import jinja2
import webapp2

Importation importante pour utiliser l’API Users :

from google.appengine.api import users

Configuration de l’environnement Jinja : [dans l’exemple, la technologie sélectionnée pour remplir les informations dans le frontend]

JINJA_ENVIRONMENT = jinja2.Environment(
       loader=jinja2.FileSystemLoader(os.path.dirname(__file__)),
       extensions=['jinja2.ext.autoescape'],
       autoescape=True)

Manipulateur de béton :

class MainPage(webapp2.RequestHandler):
   def get(self):

       user = users.get_current_user()
       if user:
           url = users.create_logout_url(self.request.uri)

Vous pouvez inclure plus de logique ici pour les utilisateurs

       else:
           url = users.create_login_url(self.request.uri)

Modèles pour transmettre des informations à l’aide de jinja2. Pour cet exemple, l’objet utilisateur et la chaîne d’URL.

       template_values = {
           'user': user,
           'url': url,
       }

       JINJA_ENVIRONMENT.add_extension('jinja2.ext.do')

Utilisation de l’exemple index.html. [page html traditionnelle]

       template = JINJA_ENVIRONMENT.get_template('index.html')
       self.response.write(template.render(template_values))

Routage d’application [urls.py] #

J’ai utilisé pour cet exemple webapp2 pour couvrir le routage.

from webapp2_extras.routes import RedirectRoute as Route

Importer depuis des vues :

from views import MainPage

MainPage est le gestionnaire défini dans la racine “/”:

urlpatterns = [
   Route('/', MainPage),
]

Html, exemple frontal d’utilisation de l’API Users [index.html] #

Extrait simple de index.html :

    <div class="sign-in">

        {% if user %}

[En passant l’url, nous avons la possibilité de déconnecter l’utilisateur]

            <a href="{{ url|safe }}">LOG OUT</a>

[Vous pouvez inclure ici les opérations pour l’utilisateur authentifié]

        {% else %}

[En passant l’url, nous avons la possibilité de connecter l’utilisateur]

            <a href="{{ url|safe }}">SIGN IN</a>

        {% endif %}
    </div>

Ceci est un exemple simple de la façon dont l’opération est utilisée sur la page index.html.
