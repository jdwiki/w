Cette section donne un aperçu de ce qu’est laravel-5.3 et pourquoi un développeur peut vouloir l’utiliser.

Il devrait également mentionner tous les sujets importants dans laravel-5.3 et établir un lien avec les sujets connexes. Étant donné que la documentation de laravel-5.3 est nouvelle, vous devrez peut-être créer des versions initiales de ces sujets connexes.

Hello World Example (Basic) et avec l’utilisation d’une vue #

L’exemple de base Ouvrez le fichier routes/web.php et collez le code suivant dans le fichier :

Route::get('helloworld', function () {
    return '<h1>Hello World</h1>';
});

ici ‘helloworld’ agira comme le nom de la page à laquelle vous souhaitez accéder,

et si vous ne souhaitez pas créer de fichier blade et que vous souhaitez toujours accéder directement à la page alors vous pouvez utiliser le routage laravel de cette façon

tapez maintenant localhost/helloworld dans la barre d’adresse du navigateur et vous pouvez accéder à la page affichant Hello World.

La prochaine étape.
Vous avez donc appris à créer un Hello World très simple ! page en renvoyant une phrase hello world. Mais nous pouvons le rendre un peu plus agréable!

Étape 1.
Nous allons recommencer à partir de notre fichier routes/web.php maintenant au lieu d’utiliser le code ci-dessus, nous utiliserons le code suivant :

Route::get('helloworld', function() {
    return view('helloworld');
});

La valeur de retour cette fois n’est pas simplement un simple texte helloworld mais une vue. Une vue dans Laravel est simplement un nouveau fichier. Ce fichier “helloworld” contient le HTML et peut-être même plus tard du PHP du texte Helloworld.

Étape 2.
Maintenant que nous avons ajusté notre route pour appeler une vue, nous allons créer la vue. Laravel fonctionne avec les fichiers blade.php dans les vues. Donc, dans ce cas, notre route s’appelle helloworld. Notre vue s’appellera donc helloworld.blade.php

Nous allons créer le nouveau fichier dans le répertoire resources/views et nous l’appellerons helloworld.blade.php

Nous allons maintenant ouvrir ce nouveau fichier et le modifier en créant notre phrase Hello World. Nous pouvons ajouter plusieurs façons différentes d’obtenir notre phrase comme dans l’exemple ci-dessous.

<html>
    <body>
        <h1> Hello World! </h1>

        <?php
            echo "Hello PHP World!";
        ?>

    </body>
</html>

allez maintenant dans votre navigateur et tapez à nouveau votre route comme dans l’exemple de base : localhost/helloworld vous verrez votre nouvelle vue créée avec tout le contenu !

Serveur de développement local #

Si PHP est installé localement et que vous souhaitez utiliser le serveur de développement intégré de PHP pour servir votre application, vous pouvez utiliser la commande Artisan serve. Cette commande démarrera un serveur de développement à http://localhost:8000 :

php artisan serve

Bien sûr, des options de développement local plus robustes sont disponibles via [Homestead][1] et [Valet][2].

Il est également possible d’utiliser un port personnalisé, quelque chose comme 8080. Vous pouvez le faire avec l’option --port.

php artisan serve --port=8080

Si vous avez un domaine local dans votre fichier hosts, vous pouvez définir le nom d’hôte. Cela peut être fait par l’option --host.

php artisan serve --host=example.dev

Vous pouvez également exécuter sur un hôte et un port personnalisés, cela peut être fait par la commande suivante.

php artisan serve --host=example.dev --port=8080

[1] : https://laravel.com/docs/5.3/homestead [2] : https://laravel.com/docs/5.3/valet

Configuration requise pour le serveur #

Le framework Laravel a quelques exigences système. Bien sûr, toutes ces exigences sont satisfaites par la machine virtuelle [Laravel Homestead][1], il est donc fortement recommandé d’utiliser Homestead comme environnement de développement Laravel local.

Cependant, si vous n’utilisez pas Homestead, vous devrez vous assurer que votre serveur répond aux exigences suivantes :

  • PHP >= 5.6.4 -Extension PHP OpenSSL
  • Extension PHP AOP
  • Extension PHP Mbstring
  • Extension PHP Tokenizer -Extension PHP XML

[1] : https://laravel.com/docs/5.3/homestead

Installer Laravel #

Conditions: Vous avez besoin de PHP >= 5.6.4 et Composer installés sur votre machine. Vous pouvez vérifier la version des deux en utilisant la commande : Pour PHP :

php -v

Sortie comme ceci :

PHP 7.0.9 (cli) (built: Aug 26 2016 06:17:04) ( NTS )
Copyright (c) 1997-2016 The PHP Group
Zend Engine v3.0.0, Copyright (c) 1998-2016 Zend Technologies

Pour Compositeur Vous pouvez exécuter la commande sur votre terminal/CMD :

composer --version

Sortie comme ceci :

composer version 1.2.1 2016-09-12 11:27:19

Laravel utilise [Composer][1] pour gérer ses dépendances. Donc, avant d’utiliser Laravel, assurez-vous que Composer est installé sur votre machine.

Par l’intermédiaire de l’installateur Laravel #

Tout d’abord, téléchargez le programme d’installation de Laravel à l’aide de Composer :

composer global require "laravel/installer"

Assurez-vous de placer le répertoire $HOME/.composer/vendor/bin (ou le répertoire équivalent pour votre système d’exploitation) dans votre $PATH afin que l’exécutable laravel puisse être localisé par votre système.

Une fois installée, la commande laravel new créera une nouvelle installation Laravel dans le répertoire que vous spécifiez. Par exemple, laravel new blog créera un répertoire nommé blog contenant une nouvelle installation de Laravel avec toutes les dépendances de Laravel déjà installées :

laravel new blog

Via Composer Créer un projet #

Alternativement, vous pouvez également installer Laravel en exécutant la commande Composer create-project dans votre terminal :

composer create-project --prefer-dist laravel/laravel blog

Installer #

Une fois l’installation de Laravel terminée, vous devrez définir des “autorisations” pour les dossiers de stockage et Bootstrap.

Remarque : La configuration des “autorisations” est l’un des processus les plus importants pour terminer lors de l’installation de Laravel.

Serveur de développement local

Si PHP est installé localement et que vous souhaitez utiliser le serveur de développement intégré de PHP pour servir votre application, vous pouvez utiliser la commande Artisan serve. Cette commande démarrera un serveur de développement à http://localhost:8000 :

php artisan serve

Ouvrez l’URL de requête de votre navigateur http://localhost:8000

[1] : https://getcomposer.org/

Exemple Hello World (de base) #

Ouvrez le fichier d’itinéraires. Collez le code suivant dans :

Route::get('helloworld', function () {
    return '<h1>Hello World</h1>';
});

après être allé sur la route http://localhost/helloworld, il affiche Hello World.

Le fichier de routes se trouve /routes/web.php

Configuration du serveur Web pour les jolies URL #

Si vous avez installé Laravel via Composer ou le programme d'installation de Laravel, vous aurez besoin de la configuration ci-dessous.

Configuration pour Apache Laravel inclut un fichier public/.htaccess qui est utilisé pour fournir des URL sans le contrôleur frontal index.php dans le chemin. Avant de servir Laravel avec Apache, assurez-vous d’avoir activé le module mod_rewrite afin que le fichier .htaccess soit honoré par le serveur.

Si le fichier .htaccess fourni avec Laravel ne fonctionne pas avec votre installation Apache, essayez cette alternative :

Options +FollowSymLinks
RewriteEngine On

RewriteCond %{REQUEST_FILENAME} !-d
RewriteCond %{REQUEST_FILENAME} !-f
RewriteRule ^ index.php [L]

Configuration pour Nginx Si vous utilisez Nginx, la directive suivante dans la configuration de votre site dirigera toutes les requêtes vers le contrôleur frontal index.php :

location / {
    try_files $uri $uri/ /index.php?$query_string;
}

Bien sûr, lorsque vous utilisez [Homestead][1] ou [Valet][2], de jolies URL seront automatiquement configurées.

[1] : https://laravel.com/docs/5.4/homestead [2] : https://laravel.com/docs/5.4/valet
