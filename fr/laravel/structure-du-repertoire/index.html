Changer le répertoire d’application par défaut #

Il existe des cas d’utilisation où vous voudrez peut-être renommer votre répertoire d’applications en autre chose. Dans Laravel4, vous pouvez simplement modifier une entrée de configuration, voici une façon de le faire dans Laravel5.

Dans cet exemple, nous allons renommer le répertoire app en src.

Remplacer la classe d’application #

Le nom de répertoire app est codé en dur dans la classe Application principale, il doit donc être remplacé. Créez un nouveau fichier Application.php. Je préfère garder le mien dans le répertoire src (celui par lequel nous remplacerons app), mais vous pouvez le placer ailleurs.

Voici à quoi devrait ressembler la classe remplacée. Si vous voulez un nom différent, remplacez simplement la chaîne src par autre chose.

namespace App;

class Application extends \Illuminate\Foundation\Application
{
    /**
     * @inheritdoc
     */
    public function path($path = '')
    {
        return $this->basePath . DIRECTORY_SEPARATOR . 'src' . ($path ? DIRECTORY_SEPARATOR . $path : $path);
    }
}

Enregistrez le fichier. Nous en avons fini avec ça.

Appel de la nouvelle classe #

Ouvrez bootstrap/app.php et localisez

$app = new Illuminate\Foundation\Application(
    realpath(__DIR__.'/../')
);

Nous allons le remplacer par celui-ci

$app = new App\Application(
    realpath(__DIR__.'/../')
);

Compositeur #

Ouvrez votre fichier composer.json et modifiez le chargement automatique pour qu’il corresponde à votre nouvel emplacement

"psr-4": {
    "App\\": "src/"
}

Et enfin, dans la ligne de commande, exécutez composer dump-autoload et votre application devrait être servie à partir du répertoire src.

Changer le répertoire des contrôleurs #

si nous voulons changer le répertoire Controllers nous avons besoin de :

  1. Déplacez et/ou renommez le répertoire Controllers par défaut où vous le souhaitez. Par exemple, de app/Http/Controllers à app/Controllers

  2. Mettez à jour tous les espaces de noms des fichiers à l’intérieur du dossier Controllers, en les faisant adhérer au nouveau chemin, en respectant le PSR-4 spécifique.

  3. Modifiez l’espace de noms appliqué au fichier routes.php en modifiant app\Providers\RouteServiceProvider.php et modifiez ceci :

protected $namespace = 'App\Http\Controllers';

pour ça:

protected $namespace = 'App\Controllers';
