Les files d’attente permettent à votre application de réserver des éléments de travail qui prennent du temps à être traités par un processus d’arrière-plan.

Cas d’utilisation #

Par exemple, si vous envoyez un e-mail à un client après avoir démarré une tâche, il est préférable de rediriger immédiatement l’utilisateur vers la page suivante tout en mettant en file d’attente l’e-mail à envoyer en arrière-plan. Cela accélérera le temps de chargement de la page suivante, car l’envoi d’un e-mail peut parfois prendre plusieurs secondes ou plus.

Un autre exemple serait la mise à jour d’un système d’inventaire après qu’un client ait passé sa commande. Plutôt que d’attendre que les appels d’API se terminent, ce qui peut prendre plusieurs secondes, vous pouvez immédiatement rediriger l’utilisateur vers la page de réussite du paiement tout en mettant en file d’attente les appels d’API en arrière-plan.

Configuration du pilote de file d’attente #

Chacun des pilotes de file d’attente de Laravel est configuré à partir du fichier config/queue.php. Un pilote de file d’attente est le gestionnaire permettant de gérer l’exécution d’un travail en file d’attente, d’identifier si les travaux ont réussi ou échoué et de relancer le travail s’il est configuré pour le faire.

Prêt à l’emploi, Laravel prend en charge les pilotes de file d’attente suivants :

sync #

Sync, ou synchrone, est le pilote de file d’attente par défaut qui exécute une tâche en file d’attente dans votre processus existant. Lorsque ce pilote est activé, vous n’avez effectivement aucune file d’attente car le travail en file d’attente s’exécute immédiatement. Ceci est utile à des fins locales ou de test, mais clairement déconseillé pour la production car cela supprime les avantages en termes de performances de la configuration de votre file d’attente.

base de données #

Ce pilote stocke les travaux en file d’attente dans la base de données. Avant d’activer ce pilote, vous devrez créer des tables de base de données pour stocker vos tâches en file d’attente et ayant échoué :

php artisan queue:table
php artisan migrate

carrés #

Ce pilote de file d’attente utilise [Amazon’s Simple Queue Service][1] pour gérer les tâches en file d’attente. Avant d’activer cette tâche, vous devez installer le package composer suivant : aws/aws-sdk-php ~3.0

Notez également que si vous prévoyez d’utiliser des délais pour les tâches en file d’attente, Amazon SQS ne prend en charge qu’un délai maximum de 15 minutes.

fer #

Ces pilotes de file d’attente utilisent [Iron][2] pour gérer les tâches en file d’attente.

redis #

Ce pilote de file d’attente utilise une instance de [Redis][3] pour gérer les tâches en file d’attente. Avant d’utiliser ce pilote de file d’attente, vous devrez configurer une copie de Redis et installer la dépendance composer suivante : predis/predis ~1.0

haricot magique #

Ce pilote de file d’attente utilise une instance de [Beanstalk][4] pour gérer les tâches en file d’attente. Avant d’utiliser ce pilote de file d’attente, vous devrez configurer une copie de Beanstalk et installer la dépendance composer suivante : pda/pheanstalk ~3.0

null #

Spécifier null comme pilote de file d’attente supprimera tous les travaux en file d’attente.

[1] : https://aws.amazon.com/sqs/ [2] : https://www.iron.io/ [3] : http://redis.io/ [4] : http://kr.github.io/beanstalkd/
