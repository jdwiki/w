Laravel prend en charge le moteur de modélisation Blade prêt à l’emploi. Le moteur de modèles Blade nous permet de créer des modèles principaux et des modèles enfants chargeant le contenu à partir de modèles principaux. Nous pouvons avoir des variables, des boucles et des instructions conditionnelles à l’intérieur du fichier blade.

Vues : Introduction #

Les vues, dans un modèle MVC, contiennent la logique sur comment pour présenter les données à l’utilisateur. Dans une application Web, ils sont généralement utilisés pour générer la sortie HTML qui est renvoyée aux utilisateurs avec chaque réponse. Par défaut, les vues de Laravel sont stockées dans le répertoire resources/views.

Une vue peut être appelée à l’aide de la fonction d’assistance view :

view(string $path, array $data = [])

Le premier paramètre de l’assistant est le chemin d’accès à un fichier de vue et le second paramètre est un tableau facultatif de données à transmettre à la vue.

Par conséquent, pour appeler resources/views/example.php, vous utiliserez :

view('example');

Afficher les fichiers dans les sous-dossiers du répertoire resources/views, tels que resources/views/parts/header/navigation.php, peut être appelé en utilisant la notation par points : view('parts.header.navigation');

Dans un fichier de vue, tel que resources/views/example.php, vous êtes libre d’inclure HTML et PHP ensemble :

<html>
    <head>
        <title>Hello world!</title>
    </head>
    <body>
        <h1>Welcome!</h1>
        <p>Your name is: <?php echo $name; ?></p>
    </body>
</html>

Dans l’exemple précédent (qui n’utilise aucune syntaxe spécifique à Blade), nous sortons la variable $name. Pour passer cette valeur à notre vue, nous passerions un tableau de valeurs lors de l’appel de l’assistant de vue :

view('example', ['name' => $name]);

ou alternativement, utilisez l’assistant compact(). Dans ce cas, la chaîne passée à compact() correspond au nom de la variable que nous voulons passer à la vue.

view('example', compact('name'));

CONVENTION DE NOMMAGE POUR LES VARIABLES DE LAME

Lors du renvoi des données à la vue. Vous pouvez utiliser underscore pour plusieurs mots variable mais avec - laravel donne une erreur.

Comme celui-ci donnera une erreur (notez trait d'union ( - ) dans le user-address

view('example',['user-address' => 'Some Address']);

La bonne façon de procéder sera

view('example', ['user_address' => 'Some Address']);

Héritage de la mise en page #

Une mise en page est un fichier de vue, qui est étendu par d’autres vues qui injectent des blocs de code dans leur parent. Par exemple:

parent.blade.php :

<html>
    <head>
        <style type='text/css'>
        @yield('styling')
        </style>
    </head>
    <body>
        <div class='main'>
        @yield('main-content')
        </div>
    </body>
</html>

enfant.blade.php :

@extends('parent')

@section('styling')
.main {
    color: red;
}
@stop

@section('main-content')
This is child page!
@stop

autrepage.blade.php :

@extends('parent')

@section('styling')
.main {
    color: blue;
}
@stop

@section('main-content')
This is another page!
@stop

Ici, vous voyez deux exemples de pages enfants, qui étendent chacune le parent. Les pages enfants définissent une @section, qui est insérée dans le parent à l’instruction @yield appropriée.

Ainsi, la vue rendue par View::make('child') dira “This is child page!” en rouge, tandis que View::make('otherpage') produira le même html, sauf avec le texte “Ceci est une autre page !” en bleu à la place.

Il est courant de séparer les fichiers de vue, par ex. avoir un dossier de mise en page spécifiquement pour les fichiers de mise en page et un dossier séparé pour les différentes vues individuelles spécifiques.

Les mises en page sont destinées à appliquer le code qui doit apparaître sur chaque page, par ex. ajouter une barre latérale ou un en-tête, sans avoir à écrire tout le passe-partout html dans chaque vue individuelle.

Les vues peuvent être étendues à plusieurs reprises - c’est-à-dire que page3 peut @extend('page2') et page2 peut @extend('page1').

La commande extend utilise la même syntaxe que celle utilisée pour View::make et @include, donc le fichier layouts/main/page.blade.php est accessible en tant que layouts.main.page.

Structures de contrôle #

Blade fournit une syntaxe pratique pour les structures de contrôle PHP courantes.

Chacune des structures de contrôle commence par @[structure] et se termine par @[endstructure]. Notez que dans les balises, nous tapons simplement du code HTML normal et incluons des variables avec la syntaxe Blade.

Conditionnels #

Déclarations “Si” #

@if ($i > 10)
    <p>{{ $i }} is large.</p>
@elseif ($i == 10)
    <p>{{ $i }} is ten.</p>
@else
    <p>{{ $i }} is small.</p>
@endif

Déclarations ‘Sauf si’ #

(Syntaxe courte pour ‘si non’.)

@unless ($user->hasName())
    <p>A user has no name.</p>
@endunless

Boucles #

Boucle “Tant que” #

@while (true)
    <p>I'm looping forever.</p>
@endwhile

Boucle ‘Foreach’ #

@foreach ($users as $id => $name)
    <p>User {{ $name }} has ID {{ $id }}.</p>
@endforeach

Boucle ‘Forel’ #

(Identique à la boucle ‘foreach’, mais ajoute une directive spéciale @empty, qui est exécutée lorsque l’expression de tableau itérée est vide, comme moyen d’afficher le contenu par défaut.)

@forelse($posts as $post)
    <p>{{ $post }} is the post content.</p>
@empty
    <p>There are no posts.</p>
@endforelse

Dans les boucles, une variable spéciale $loop sera disponible, contenant des informations sur l’état de la boucle :

Propriété La description
$boucle->index L’index de l’itération de la boucle en cours (commence à 0).
$boucle->itération L’itération de la boucle en cours (commence à 1).
$boucle->restant Les itérations de boucle restantes.
$boucle->compter Le nombre total d’éléments dans le tableau en cours d’itération.
$boucle->première S’il s’agit de la première itération de la boucle.
$boucle->dernier S’il s’agit de la dernière itération de la boucle.
$boucle->profondeur Le niveau d’imbrication de la boucle en cours.
$boucle->parent Dans une boucle imbriquée, la variable de boucle du parent.

Exemple:

@foreach ($users as $user)
  @foreach ($user->posts as $post)
        @if ($loop->parent->first)
            This is first iteration of the parent loop.
        @endif
    @endforeach
@endforeach

Depuis Laravel 5.2.22, on peut aussi utiliser les directives @continue et @break

Propriété La description
@continuer Arrêtez l’itération en cours et démarrez la suivante.
@pause Arrête la boucle en cours.

Exemple :

@foreach ($users as $user)
    @continue ($user->id == 2)
        <p>{{ $user->id }} {{ $user->name }}</p>
    @break ($user->id == 4)
@endforeach

Ensuite (en supposant que plus de 5 utilisateurs sont triés par ID et qu’aucun ID ne manque), la page s’affichera

1 Dave
3 John
4 William

Faire écho aux expressions PHP #

Toute expression PHP entre accolades doubles {{ $variable }} sera renvoyée en ‘écho’ après avoir été exécutée via la fonction d’assistance [e] [1]. (Ainsi, les caractères spéciaux html (<, >, ", ', &) sont remplacés en toute sécurité pour les entités html correspondantes.) (L’expression PHP doit correspondre à une chaîne, sinon une exception sera levée.)

Faire écho à une variable #

{{ $variable }}

Faire écho à un élément dans un tableau #

{{ $array["key"] }}

Faire écho à une propriété d’objet #

{{ $object->property }}

Faire écho au résultat d’un appel de fonction #

{{ strtolower($variable) }}

Vérification de l’existence #

Normalement, en PHP, pour vérifier si une variable est définie et l’imprimer, vous feriez

  • Avant PHP 7

  • Après PHP 7 (en utilisant le “Null coalescing operator” )

L’opérateur de lame “ou” facilite cette opération :

{{ $variable or 'Default' }}

Échos bruts #

Comme mentionné, la syntaxe régulière des doubles accolades {{ }} est filtrée par la fonction htmlspecialchars de PHP, pour des raisons de sécurité (empêchant l’injection malveillante de HTML dans la vue). Si vous souhaitez contourner ce comportement, par exemple si vous essayez de générer un bloc de contenu HTML résultant d’une expression PHP, utilisez la syntaxe suivante :

{!! $myHtmlString !!}

Notez qu’il est considéré comme une bonne pratique d’utiliser la syntaxe standard {{ }} pour échapper vos données, sauf en cas d’absolue nécessité. De plus, lorsque vous faites écho à un contenu non fiable (c’est-à-dire un contenu fourni par les utilisateurs de votre site), vous devez éviter d’utiliser le {!! !!} syntaxe.

[1] : https://laravel.com/docs/5.3/helpers#method-e

Inclure les vues partielles #

Avec Blade, vous pouvez également inclure des vues partielles (appelées “partiels”) directement dans une page comme ceci :

@include('includes.info', ['title' => 'Information Station'])

Le code ci-dessus inclura la vue dans ‘views/includes/info.blade.php’. Il passera également une variable $title ayant pour valeur ‘Information Station’.

En général, une page incluse aura accès à toutes les variables auxquelles la page appelante a accès. Par exemple, si nous avons :

{{$user}} // Outputs 'abc123'
@include('includes.info')

Et ‘includes/info.blade.php’ a ce qui suit :

<p>{{$user}} is the current user.</p>

Ensuite, la page affichera :

abc123
abc123 is the current user.

Inclure chacun

Parfois, vous souhaiterez combiner une instruction include avec une instruction foreach et accéder aux variables à partir de la boucle foreach dans l’include. Dans ce cas, utilisez la directive @each de Blade :

@each('includes.job', $jobs, 'job')

Le premier paramètre est la page à inclure. Le deuxième paramètre est le tableau sur lequel itérer. Le troisième paramètre est la variable affectée aux éléments du tableau. L’énoncé ci-dessus équivaut à :

@foreach($jobs as $job)
    @include('includes.job', ['job' => $job])
@endforeach

Vous pouvez également passer un quatrième argument facultatif à la directive @each pour spécifier la vue à afficher lorsque le tableau est vide.

@each('includes.job', $jobs, 'job', 'includes.jobsEmpty')

Exécute du code PHP arbitraire #

Bien qu’il ne soit peut-être pas approprié de faire une telle chose dans une vue si vous avez l’intention de séparer strictement les préoccupations, la directive php Blade permet d’exécuter du code PHP, par exemple, de définir une variable :

@php($varName = 'Enter content ')

(pareil que:)

@php
    $varName = 'Enter content ';
@endphp

plus tard:

{{ $varName }}
