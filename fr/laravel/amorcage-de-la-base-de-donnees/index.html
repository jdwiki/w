Exécution d’un semoir #

Vous pouvez ajouter votre nouveau Seeder à la classe DatabaseSeeder.

    /**
     * Run the database seeds.
     *
     * @return void
     */
    public function run()
    {
        $this->call(UserTableSeeder::class);
    }

Pour exécuter un seeder de base de données, utilisez la commande Artisan

php artisan db:seed

Cela exécutera la classe DatabaseSeeder. Vous pouvez également choisir d’utiliser l’option --class= pour spécifier manuellement le seeder à exécuter.

*Notez que vous devrez peut-être exécuter composer dumpautoload si votre classe Seeder est introuvable. Cela se produit généralement si vous créez manuellement une classe seeder au lieu d’utiliser la commande artisan.

Créer une graine #

Les graines de base de données sont stockées dans le répertoire /database/seeds. Vous pouvez créer une graine à l’aide d’une commande Artisan.

php artisan make:seed UserTableSeeder

Vous pouvez également créer une nouvelle classe qui étend Illuminate\Database\Seeder. La classe doit être une fonction publique nommée run().

Insertion de données à l’aide d’un seeder #

Vous pouvez référencer des modèles dans un seeder.

use DB;
use App\Models\User;

class UserTableSeeder extends Illuminate\Database\Seeder{

    public function run(){
        # Remove all existing entrie
        DB::table('users')->delete() ;
        User::create([
            'name' => 'Admin',
            'email' => 'admin@example.com',
            'password' => Hash::make('password')
        ]);

    }
}

Insertion de données avec une Model Factory #

Vous souhaiterez peut-être utiliser des usines modèles dans vos graines. Cela créera 3 nouveaux utilisateurs.

use App\Models\User;

class UserTableSeeder extends Illuminate\Database\Seeder{

    public function run(){
        factory(User::class)->times(3)->create();    
    }
}

Vous pouvez également définir des champs spécifiques sur votre amorçage comme un mot de passe, par exemple. Cela créera 3 utilisateurs avec le même mot de passe.

factory(User::class)->times(3)->create(['password' => '123456']);

Amorçage avec MySQL Dump #

Suivez l’exemple précédent de création d’une graine. Cet exemple utilise un vidage MySQL pour amorcer une table dans la base de données du projet. La table doit être créée avant l’ensemencement.

<?php

use Illuminate\Database\Seeder;

class UserTableSeeder extends Seeder
{

    /**
     * Run the database seeds.
     *
     * @return void
     */
     public function run()
     {
         $sql = file_get_contents(database_path() . '/seeds/users.sql');
    
         DB::statement($sql);
      }
}

Notre $ sql va être le contenu de notre vidage users.sql. Le vidage doit avoir une instruction INSERT INTO. Ce sera à vous de décider où vous stockez vos décharges. Dans l’exemple ci-dessus, il est stocké dans le répertoire du projet \database\seeds. En utilisant la fonction d’assistance de laravel database_path() et en ajoutant le répertoire et le nom de fichier du vidage.

INSERT INTO `users` (`id`, `name`, `email`, `password`, `remember_token`, `created_at`, `updated_at`) VALUES
(1, 'Jane', 'janeDoe@fakemail.com', 'superSecret', NULL, '2016-07-21 00:00:00', '2016-07-21 00:00:00'),
(2, 'John', 'johnny@fakemail.com', 'sup3rS3cr3t', NULL, '2016-07-21 00:00:00', '2016-07-21 00:00:00');

DB::statement($sql) exécutera les insertions une fois le Seeder exécuté. Comme dans les exemples précédents, vous pouvez placer UserTableSeeder dans la classe DatabaseSeeder fournie par laravel :

<?php

use Illuminate\Database\Seeder;

class DatabaseSeeder extends Seeder
{
    /**
     * Run the database seeds.
     *
     * @return void
     */
    public function run()
    {
        $this->call(UserTableSeeder::class);
    }
}

et exécuté à partir de la CLI dans le répertoire du projet php artisan db:seed. Ou vous pouvez exécuter le Seeder pour une seule classe en utilisant php artisan db:seed --class=UsersTableSeeder

Utilisation de faker et ModelFactories pour générer des graines #

1) FAÇON SIMPLE DE BASE

Les applications basées sur une base de données ont souvent besoin de données pré-ensemencées dans le système à des fins de test et de démonstration.

Pour créer de telles données, créez d’abord la classe seeder

ProductTableSeeder

use Faker\Factory as Faker;
use App\Product;

class ProductTableSeeder extends DatabaseSeeder {

public function run()
{
    $faker = $this->getFaker();

    for ($i = 0; $i < 10; $i++)
    {
        $name =         $faker->word;
        $image =        $faker->imageUrl;
       
        Modelname::create([
            'name' => $name,
            'image' => $image,
        ]);
      }
    }
 }

Pour appeler un pouvoir exécuter une classe seeder, vous devez l’appeler depuis la classe DatabaseSeeder, simplement en passant le nom du seeder que vous souhaitez exécuter :

utilisez Illuminate\Database\Seeder ;

class DatabaseSeeder extends Seeder {

    protected $faker;

    public function getFaker() {
        if (empty($this->faker)) {
            $faker = Faker\Factory::create();
            $faker->addProvider(new Faker\Provider\Base($faker));
            $faker->addProvider(new Faker\Provider\Lorem($faker));
        }
        return $this->faker = $faker;
    }
    public function run() {
        $this->call(ProductTableSeeder::class);
    }
}

N’oubliez pas d’exécuter $ composer dump-autoload après avoir créé le Seeder, car ils ne sont pas automatiquement chargés automatiquement par composer (sauf si vous avez créé le seeder par la commande artisan $ php artisan make:seeder Name)

Vous êtes maintenant prêt à semer en exécutant cette commande artisanale php artisan db:seed

2) UTILISATION D’usines Modèles

Tout d’abord, vous devez définir un ensemble d’attributs par défaut pour chaque modèle dans App/database/factories/ModelFactory.php

En prenant un modèle utilisateur comme exemple, voici à quoi ressemble une ModelFactory

$factory->define(App\User::class, function (Faker\Generator $faker) {
    return [
        'name' => $faker->name,
        'email' => $faker->email,
        'password' => bcrypt(str_random(10)),
        'remember_token' => str_random(10),
    ];
});

Créez maintenant un seeder de table php artisan make:seeder UsersTableSeeder

Et ajoutez ceci

public function run()
{
    factory(App\User::class, 100)->create()
}

puis ajoutez ceci au DatabaseSeeder

public function run()
{
    $this->call(UsersTableSeeder::class);
}

Cela amorcera la table avec 100 enregistrements.
