L’ensemencement de la base de données vous permet d’insérer des données, des données de test générales dans votre base de données. Par défaut, il existe une classe DatabaseSeeder sous database/seeds.

Faire fonctionner les semoirs peut être fait avec

php artisan db:seed

Ou si vous ne souhaitez traiter qu’une seule classe

php artisan db:seed --class=TestSeederClass

Comme pour toutes les commandes artisanales, vous avez accès à un large éventail de méthodes qui peuvent être trouvées dans la [documentation api][1]

[1] : https://laravel.com/api/5.3/Illuminate/Console/Command.html

Création d’un semoir #

Pour créer des seeders, vous pouvez utiliser la commande Artisan make:seeder. Tous les seeders générés seront placés dans le répertoire database/seeds.

$ php artisan make:seeder MoviesTableSeeder

Les seeders générés contiendront une méthode : run. Vous pouvez insérer des données dans votre base de données dans cette méthode.

<?php

use Illuminate\Database\Seeder;
use Illuminate\Database\Eloquent\Model;

class MoviesTableSeeder extends Seeder
{
    /**
     * Run the database seeds.
     *
     * @return void
     */
    public function run()
    {
        App\Movie::create([
            'name' => 'A New Hope',
            'year' => '1977'
        ]);

        App\Movie::create([
            'name' => 'The Empire Strikes Back',
            'year' => '1980'
        ]);
    }
}

Vous souhaiterez généralement appeler tous vos seeders [à l’intérieur de la classe DatabaseSeeder] (https://www.wikiod.com/fr/laravel/semis#Appeler d’autres seeders).

Une fois que vous avez fini d’écrire les seeders, utilisez la commande db:seed. Cela exécutera la fonction run de DatabaseSeeder.

$ php artisan db:seed

Vous pouvez également spécifier d’exécuter une classe de départ spécifique à exécuter individuellement à l’aide de l’option --class.

$ php artisan db:seed --class=UserSeeder

Si vous souhaitez annuler et réexécuter toutes les migrations, puis réamorcer :

$ php artisan migrate:refresh --seed

La commande migrate:refresh --seed est un raccourci vers ces 3 commandes :

$ php artisan migrate:reset # annuler toutes les migrations
$ php artisan migrate # exécution des migrations
$ php artisan db:seed # lancer les seeders

Insertion de données #

Il existe plusieurs manières d’insérer des données :

Utilisation de la façade DB #

public function run()
{
    DB::table('users')
        ->insert([
            'name' => 'Taylor',
            'age'  => 21
        ]);
}

Via l’instanciation d’un modèle #

public function run()
{
    $user = new User;
    $user->name = 'Taylor';
    $user->save();
}

Utilisation de la méthode de création #

public function run()
{
    User::create([
        'name' => 'Taylor',
        'age'  => 21
    ]);
}

Utilisation de l’usine #

public function run()
{
    factory(App\User::class, 10)->create();
}

Seeding && suppression des anciennes données et réinitialisation de l’auto-incrémentation #

public function run()
{
    DB::table('users')->delete();
    DB::unprepared('ALTER TABLE users AUTO_INCREMENT=1;');
    factory(App\User::class, 200)->create();
}

Voir l’exemple Persisting pour plus d’informations sur l’insertion/la mise à jour des données.

Appeler d’autres seeders #

Dans votre classe DatabaseSeeder, vous pouvez appeler d’autres seeders

$this->call(TestSeeder::class)

Cela vous permet de garder un fichier où vous pouvez facilement trouver vos semoirs. Gardez à l’esprit que vous devez faire attention à l’ordre de vos appels concernant les contraintes de clé étrangère. Vous ne pouvez pas référencer une table qui n’existe pas encore.

Réensemencement en toute sécurité #

Vous voudrez peut-être réamorcer votre base de données sans affecter vos graines précédemment créées. Pour cela, vous pouvez utiliser [firstOrCreate][1] dans votre seeder :

EmployeeType::firstOrCreate([
    'type' => 'manager',
]);

Ensuite, vous pouvez ensemencer la base de données :

php artisan db:seed

Plus tard, si vous souhaitez ajouter un autre type d’employé, vous pouvez simplement ajouter ce nouveau dans le même fichier :

EmployeeType::firstOrCreate([
    'type' => 'manager',
]);
EmployeeType::firstOrCreate([
    'type' => 'secretary',
]);

Et réamorcez votre base de données sans problème :

php artisan db:seed

Remarquez que lors du premier appel, vous récupérez l’enregistrement mais ne faites rien avec.

[1] : https://laravel.com/api/5.2/Illuminate/Database/Eloquent/Builder.html#method_firstOrCreate
