Introduction #

Artisan est un utilitaire qui peut vous aider à effectuer des tâches répétitives spécifiques avec des commandes bash. Il couvre de nombreuses tâches, notamment : travailler avec la base de données [migrations][1] et [seeding][2], vider le cache, créer les fichiers nécessaires à la configuration de l’authentification, faire de nouveaux contrôleurs, modèles, classes d’événements, et bien plus encore.

Artisan est le nom de l’interface de ligne de commande incluse avec Laravel. Il fournit un certain nombre de commandes utiles pour votre utilisation lors du développement de votre application.

Pour afficher une liste de toutes les commandes Artisan disponibles, vous pouvez utiliser la commande list :

php artisan list

Pour en savoir plus sur toute commande disponible, faites simplement précéder son nom du mot-clé help :

php artisan help [command-name]

[1] : https://www.wikiod.com/fr/laravel/migrations-de-bases-de-donnees#Migrations [2] : https://www.wikiod.com/fr/laravel/semis

Liste toutes les routes enregistrées filtrées par plusieurs méthodes #

php artisan route:list --method=GET --method=POST

Cela inclura toutes les routes qui acceptent les méthodes GET et POST simultanément.

Création et enregistrement d’une nouvelle commande artisanale #

Vous pouvez créer de nouvelles commandes via

php artisan make:commande [nom de la commande]

Cela créera donc la classe de commande [commandName] dans le répertoire app/Console/Commands.

à l’intérieur de cette classe, vous trouverez les variables protected $signature et protected $description, elles représentent le nom et la description de votre commande qui sera utilisé pour décrire votre commande.

après avoir créé la commande, vous pouvez enregistrer votre commande dans la classe app/Console/Kernel.php où vous trouverez la propriété commands.

vous pouvez donc ajouter votre commande dans le tableau $command comme :

protected $commands = [
    Commands\[commandName]::class
];

puis je peux utiliser ma commande via la console.

donc comme exemple j’ai nommé ma commande comme

protected $signature = 'test:command';

Donc, chaque fois que je vais courir

php artisan test:command

il appellera la méthode handle à l’intérieur de la classe ayant la signature test: command.

Exécution des commandes Laravel Artisan à l’aide de code PHP #

Vous pouvez également utiliser les commandes Laravel Artisan à partir de vos routes ou de vos contrôleurs.

Pour exécuter une commande à l’aide de code PHP :

Artisan::call('command-name');

Par exemple,

Artisan::call('db:seed');
