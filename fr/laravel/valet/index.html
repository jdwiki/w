Valet est un environnement de développement sur mesure pour macOS. Cela élimine le besoin de machines virtuelles, Homestead ou Vagrant. Plus besoin de mettre constamment à jour votre fichier /etc/hosts. Vous pouvez même partager publiquement vos sites en utilisant des tunnels locaux.

Laravel Valet rend tous les sites disponibles sur un domaine *.dev en liant les noms de dossier aux noms de domaine.

Syntaxe #

  • commande valet [options] [arguments]

Paramètres #

Paramètre Ensemble de valeurs
commande [domaine](https://www.wikiod.com/fr/laravel/valet#Domaine voiturier), fetch-share-url, oublier, aider, installer, [lien](http://stackoverflow.com /documentation/laravel/1906/valet/6218/valet-link#t=201607272250304319463), [liens](https://www.wikiod.com/fr/laravel/valet#Liens voiturier) , list, logs, on-latest-version, open, [park](https://www.wikiod.com/fr/laravel/valet#Service de voiturier), paths, restart, secure, démarrer, arrêter, désinstaller, dissocier, non sécurisé, qui
options -h, –help, -q, –quiet, -V, –version, –ansi, –no-ansi, -n, –no-interaction, -v, -vv, -vvv,- -verbeux
arguments (facultatif)

Étant donné que Valet pour Linux et Windows ne sont pas officiels, il n’y aura pas de support en dehors de leurs référentiels Github respectifs.

Lien voiturier #

Cette commande est utile si vous souhaitez servir un seul site dans un annuaire et non l’intégralité de l’annuaire.

cd ~/Projects/my-blog/
valet link awesome-blog

Valet créera un lien symbolique dans ~/.valet/Sites qui pointe vers votre répertoire de travail actuel. Après avoir exécuté la commande de lien, vous pouvez accéder au site dans votre navigateur à l’adresse http://awesome-blog.dev.

Pour voir une liste de tous vos répertoires liés, exécutez la commande [valet links][1]. Vous pouvez utiliser valet unlink awesome-blog pour détruire le lien symbolique.

[1] : https://www.wikiod.com/fr/laravel/valet#Liens voiturier

Liens de voiturier #

Cette commande affichera tous les liens Valet enregistrés que vous avez créés et leurs chemins de fichiers correspondants sur votre ordinateur.

Commande:

valet links

Exemple de sortie :

...
site1 -> /path/to/site/one
site2 -> /path/to/site/two
...

Remarque 1 : Vous pouvez exécuter cette commande depuis n’importe où, pas seulement depuis un dossier lié.

Remarque 2 : Les sites seront répertoriés sans la fin .dev mais vous utiliserez toujours site1.dev pour accéder à votre application depuis le navigateur.

Service voiturier #

cd ~/Projects
valet park

Cette commande enregistrera votre répertoire de travail actuel en tant que chemin que Valet doit rechercher pour les sites. Désormais, tout projet Laravel que vous créez dans votre répertoire “parqué” sera automatiquement servi en utilisant la convention http://folder-name.dev.

Installation #

IMPORTANT !! Valet est un outil conçu uniquement pour macOS.

Conditions préalables

  • Valet utilise le port HTTP de votre machine locale (port 80), par conséquent, vous ne pourrez pas l’utiliser si Apache ou Nginx sont installés et exécutés sur la même machine.
  • Le gestionnaire de paquets non officiel de macOS [Homebrew][1] est requis pour utiliser correctement Valet.
  • Assurez-vous que Homebrew est mis à jour vers la dernière version en exécutant “brew update” dans le terminal.

Installation

  • Installez PHP 7.1 en utilisant Homebrew via brew install homebrew/php/php71.
  • Installez Valet avec Composer via composer global require laravel/valet.
  • Ajoutez le répertoire ~/.composer/vendor/bin au “PATH” de votre système s’il n’y est pas déjà.
  • Exécutez la commande valet install.

Après l’installation Au cours du processus d’installation, Valet a installé DnsMasq. Il a également enregistré le démon de Valet pour qu’il se lance automatiquement au démarrage de votre système, vous n’avez donc pas besoin d’exécuter valet start ou valet install chaque fois que vous redémarrez votre machine.

[1] : https://brew.sh

Domaine voiturier #

Cette commande vous permet de modifier ou d’afficher le TLD (domaine de premier niveau) utilisé pour lier des domaines à votre ordinateur local.

Obtenir le TLD actuel

$ valet domain
> dev

Définir le TLD

$ valet domain local
> Your Valet domain has been updated to [local].

Installation (Linux) #

IMPORTANT!! Valet est un outil conçu pour macOS, la version ci-dessous est portée pour Linux OS.

Conditions préalables

  • Ne pas installer valet en tant que root ou en utilisant la commande sudo.
  • Valet utilise le port HTTP de votre machine locale (port 80), par conséquent, vous ne pourrez pas l’utiliser si Apache ou Nginx sont installés et exécutés sur la même machine.
  • Une version à jour de composer est nécessaire pour installer et exécuter Valet.

Installation

  • Exécutez composer global require cpriego/valet-linux pour installer Valet globalement.
  • Exécutez la commande valet install pour terminer l’installation.

Après l’installation

Au cours du processus d’installation, Valet a installé DnsMasq. Il a également enregistré le démon de Valet pour qu’il se lance automatiquement au démarrage de votre système, vous n’avez donc pas besoin d’exécuter valet start ou valet install chaque fois que vous redémarrez votre machine.

La [Documentation officielle peut être trouvée ici][1].

[1] : https://github.com/cpriego/valet-linux/wiki
