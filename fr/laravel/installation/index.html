Installation #

Les applications Laravel sont installées et gérées avec [Composer][1], un gestionnaire de dépendances PHP populaire. Il existe deux façons de créer une nouvelle application Laravel.

Via Compositeur #

$ composer create-project laravel/laravel [foldername]

Ou

$ composer create-project --prefer-dist laravel/laravel [foldername]

Remplacez [nom du dossier] par le nom du répertoire dans lequel vous souhaitez installer votre nouvelle application Laravel. Il ne doit pas exister avant l’installation. Vous devrez peut-être également ajouter l’exécutable Composer à votre chemin système.

Si vous souhaitez créer un projet Laravel en utilisant une version spécifique du framework, vous pouvez fournir un modèle de version, sinon votre projet utilisera la dernière version disponible.

Si vous vouliez créer un projet dans Laravel 5.2 par exemple, vous exécuteriez :

$ composer create-project --prefer-dist laravel/laravel 5.2.*

Pourquoi –prefer-dist

Il y a deux manières de télécharger un paquet : source et dist. Pour les versions stables, Composer utilisera le dist par défaut. La source est un référentiel de contrôle de version. Si --prefer-source est activé, Composer s’installera à partir de la source s’il y en a une.

--prefer-dist est l’opposé de --prefer-source et indique à Composer d’installer à partir de dist si possible. Cela peut accélérer considérablement les installations sur les serveurs de build et dans d’autres cas d’utilisation où vous n’exécutez généralement pas les mises à jour du fournisseur. Cela permet également d’éviter les problèmes avec Git si vous n’avez pas une configuration appropriée.

Via le programme d’installation de Laravel #

Laravel fournit un utilitaire de ligne de commande utile pour créer rapidement des applications Laravel. Tout d’abord, installez le programme d’installation :

$ composer global require laravel/installer

Vous devez vous assurer que le dossier des binaires Composer se trouve dans votre variable $PATH pour exécuter le programme d’installation de Laravel.

Tout d’abord, regardez s’il est déjà dans votre variable $PATH

echo $PATH

Si tout est correct, la sortie devrait contenir quelque chose comme ceci :

Users/votrenom d'utilisateur/.composer/vendor/bin

Si ce n’est pas le cas, modifiez votre .bashrc ou, si vous utilisez ZSH, votre .zshrc afin qu’il contienne le chemin vers le répertoire de votre fournisseur Composer.

Une fois installée, cette commande créera une nouvelle installation Laravel dans le répertoire que vous spécifiez.

laravel new [foldername]

Vous pouvez également utiliser . (un point) à la place de [nomdudossier] pour créer le projet dans le répertoire de travail courant sans créer de sous-répertoire.

Lancer l’application #

Laravel est livré avec un serveur Web basé sur PHP qui peut être démarré en exécutant

$ php artisan serve

Par défaut, le serveur HTTP utilisera le port 8000, mais si le port est déjà utilisé ou si vous souhaitez exécuter plusieurs applications Laravel à la fois, vous pouvez utiliser le drapeau --port pour spécifier un port différent :

$ php artisan serve --port=8080

Le serveur HTTP utilisera localhost comme domaine par défaut pour exécuter l’application, mais vous pouvez utiliser le drapeau --host pour spécifier une adresse différente :

$ php artisan serve --host=192.168.0.100 --port=8080

Utiliser un autre serveur #

Si vous préférez utiliser un autre logiciel de serveur Web, certains fichiers de configuration vous sont fournis dans le répertoire “public” de votre projet ; .htaccess pour Apache et web.config pour ASP.NET. Pour d’autres logiciels tels que NGINX, vous pouvez convertir les configurations Apache à l’aide de divers outils en ligne.

Le framework a besoin que l’utilisateur du serveur Web ait des autorisations en écriture sur les répertoires suivants :

  • /stockage
  • /bootstrap/cache

Sur les systèmes d’exploitation * nix, cela peut être réalisé en

chown -R www-data:www-data storage bootstrap/cache
chmod -R ug+rwx storage bootstrap/cache

(où www-data est le nom et le groupe de l’utilisateur du serveur Web)

Le serveur Web de votre choix doit être configuré pour servir le contenu du répertoire /public de votre projet, ce qui se fait généralement en le définissant comme racine du document. Le reste de votre projet ne doit pas être accessible via votre serveur Web.

Si vous avez tout configuré correctement, la navigation vers l’URL de votre site Web devrait afficher la page de destination par défaut de Laravel.

[1] : https://getcomposer.org/

Conditions #

Le framework Laravel a les exigences suivantes :

  • PHP >= 5.6.4 *Extension PHP XML
  • Extension PHP AOP
  • Extension PHP OpenSSL
  • Extension PHP Mbstring
  • Extension PHP Tokenizer
  • PHP >= 5.5.9
  • Extension PHP AOP
  • Laravel 5.1 est la première version de Laravel à prendre en charge PHP 7.0.
  • PHP >= 5.4, PHP < 7
  • Extension PHP OpenSSL
  • Extension PHP Tokenizer
  • Extension PHP Mbstring
  • Extension PHP JSON (uniquement sur PHP 5.5)

*PHP >= 5.4

  • Extension PHP Mbstring
  • Extension PHP JSON (uniquement sur PHP 5.5)

Exemple Hello World (de base) #

Ouvrez le fichier d’itinéraires. Collez le code suivant dans :

Route::get('helloworld', function () {
    return '<h1>Hello World</h1>';
});

après être allé sur la route localhost/helloworld, il affiche Hello World.

Le fichier routes se trouve :

Pour le Web

routes/web.php

Pour les API

routes/api.php

app/Http/routes.php

app/routes.php

Exemple Hello World (utilisation du contrôleur et de la vue) #

  1. Créez une application Laravel :

     $ composer create-project laravel/laravel hello-world

  2. Naviguez jusqu’au dossier du projet, par ex.

     $ cd C:\xampp\htdocs\hello-world

  3. Créez un contrôleur :

     $ php artisan make:controller HelloController --resource

Cela créera le fichier app/Http/Controllers/HelloController.php. L’option --resource générera des méthodes CRUD pour le contrôleur, par ex. indexer, créer, afficher, mettre à jour.

  1. Enregistrez une route vers la méthode index de HelloController. Ajoutez cette ligne à app/Http/routes.php (version 5.0 à 5.2) ou routes/web.php (version 5.3) :
    Route::get('hello', 'HelloController@index');

Pour voir vos routes nouvellement ajoutées, vous pouvez exécuter $ php artisan route:list

  1. Créez un modèle Blade dans le répertoire views :

    resources/views/hello.blade.php:

     <h1>Hello world!</h1>

  2. Maintenant, nous indiquons à la méthode index d’afficher le modèle hello.blade.php :

    app/Http/Controllers/HelloController.php

    <?php
    
    namespace App\Http\Controllers;
    
    use Illuminate\Http\Request;
    
    use App\Http\Requests;
    
    class HelloController extends Controller
    {
        /**
         * Display a listing of the resource.
         *
         * @return \Illuminate\Http\Response
         */
        public function index()
        {
            return view('hello');
        }
    
        // ... other resources are listed below the index one above

Vous pouvez diffuser votre application à l’aide de la commande PHP Artisan suivante : php artisan serve ; il vous montrera l’adresse à laquelle vous pouvez accéder à votre application (habituellement à http://localhost:8000 par défaut).

Alternativement, vous pouvez vous rendre directement à l’emplacement approprié dans votre navigateur ; dans le cas où vous utilisez un serveur comme XAMPP (soit : http://localhost/hello-world/public/hello si vous avez installé votre instance Laravel, hello-world, directement dans votre xampp/htdocs répertoire comme dans : après avoir exécuté l’étape 1 de ce Hello Word depuis votre interface de ligne de commande, en pointant sur votre répertoire xampp/htdocs).

Installation avec LaraDock (Laravel Homestead pour Docker) #

LaraDock est un environnement de développement similaire à Laravel Homestead mais pour Docker au lieu de Vagrant. https://github.com/LaraDock/laradock

#Mise en place

*Nécessite Git et Docker

Clonez le dépôt LaraDock :

A. Si vous avez déjà un projet Laravel, clonez ce référentiel sur votre répertoire racine Laravel :

git submodule add https://github.com/LaraDock/laradock.git

B. Si vous n’avez pas de projet Laravel et que vous souhaitez installer Laravel depuis Docker, clonez ce référentiel n’importe où sur votre machine :

git clone https://github.com/LaraDock/laradock.git

Utilisation de base #

  1. Exécutez les conteneurs : (assurez-vous d’être dans le dossier laradock avant d’exécuter les commandes docker-compose).

Exemple : Exécuter NGINX et MySQL : docker-compose up -d nginx mysql

Il existe une liste de conteneurs disponibles que vous pouvez sélectionner pour créer vos propres combinaisons.

nginx, hhvm, php-fpm, mysql, redis, postgres, mariadb, neo4j, mongo, apache2, caddy, memcached, beanstalkd, beanstalkd-console, espace de travail

  1. Entrez dans le conteneur Workspace, pour exécuter des commandes telles que (Artisan, Composer, PHPUnit, Gulp, …).

    docker-compose exec workspace bash

  2. Si vous n’avez pas encore installé de projet Laravel, suivez l’étape pour installer Laravel à partir d’un conteneur Docker.

un. Entrez dans le conteneur Workspace.

b. Installez Laravel. composer create-project laravel/laravel my-cool-app "5.3.*"

  1. Modifiez les configurations Laravel. Ouvrez le fichier .env de votre Laravel et définissez le DB_HOST sur votre mysql :

    DB_HOST=mysql

  2. Ouvrez votre navigateur et visitez votre adresse d’hôte local.
