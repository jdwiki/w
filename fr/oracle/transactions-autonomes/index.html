Sur cette page

  • Utilisation d’une transaction autonome pour la journalisation des erreurs

Les cas d’utilisation typiques pour les transactions autonomes sont.

  1. Pour créer tout type de framework de journalisation comme le framework de journalisation des erreurs expliqué dans l’exemple ci-dessus.
  2. Pour auditer les opérations DML dans les déclencheurs sur les tables, quel que soit le statut final de la transaction (COMMIT ou ROLLBACK).

Utilisation d’une transaction autonome pour la journalisation des erreurs #

La procédure suivante est une procédure générique qui sera utilisée pour consigner toutes les erreurs d’une application dans une table de journal des erreurs commune.

CREATE OR REPLACE PROCEDURE log_errors
(
  p_calling_program IN VARCHAR2,
  p_error_code IN INTEGER,
  p_error_description IN VARCHAR2
)
IS
  PRAGMA AUTONOMOUS_TRANSACTION;
BEGIN
  INSERT INTO error_log
  VALUES
  (
  p_calling_program,
  p_error_code,
  p_error_description,
  SYSDATE,
  USER
  );
  COMMIT;
END log_errors;

Le bloc PLSQL anonyme suivant montre comment appeler la procédure log_errors.

BEGIN
   DELETE FROM dept WHERE deptno = 10;
EXCEPTION
   WHEN OTHERS THEN
      log_errors('Delete dept',sqlcode, sqlerrm);
      RAISE;
END;

SELECT * FROM error_log;
