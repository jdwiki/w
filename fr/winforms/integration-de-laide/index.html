Vous pouvez fournir de l’aide pour les formulaires et les contrôles dans une application Windows Forms de différentes manières. Vous pouvez afficher une aide contextuelle, ouvrir un fichier CHM ou une URL. Vous pouvez afficher une aide contextuelle pour les formulaires, les contrôles et les boîtes de dialogue.

Composant HelpProvider #

Vous pouvez configurer un composant HelpProvider pour fournir une aide contextuelle pour le composant. Ainsi, lorsque l’utilisateur appuie sur la touche F1 ou sur le bouton Aide du formulaire, vous pouvez automatiquement :

  • Afficher une fenêtre contextuelle d’aide contextuelle pour les contrôles
  • Ouvrir un fichier CHM en fonction du contexte (Afficher la table des matières, Afficher un mot clé ou un index, afficher un sujet)
  • Accédez à une URL à l’aide du navigateur par défaut

Classe d’aide #

Vous pouvez utiliser la classe Help dans le code pour fournir ces types d’aide :

  • Afficher une fenêtre contextuelle d’aide pour un contrôle
  • Ouvrir un fichier CHM en fonction du contexte (Afficher la table des matières, Afficher un mot clé ou un index, afficher un sujet)
  • Accédez à une URL à l’aide du navigateur par défaut

Événement HelpRequested #

Vous pouvez gérer l’événement HelpRequested des objets Control ou Formulaire pour effectuer des actions personnalisées lorsque l’utilisateur appuie sur F1 ou clique sur le bouton Aide du formulaire.

Bouton d’aide du formulaire #

Vous pouvez configurer le “Formulaire” pour afficher le bouton Aide sur la barre de titre. De cette façon, si l’utilisateur clique sur le bouton Aide, le curseur se transformera en un curseur ? et après avoir cliqué sur n’importe quel point, toute aide contextuelle associée au contrôle utilisant HelpProvider sera affichée.

Bouton d’aide de MessgeBox et CommonDialogs #

Vous pouvez fournir de l’aide pour MessageBox, OpenFileDialog, SaveDialog et ColorDialog en utilisant le bouton Aide des composants.

Composant d’info-bulle #

Vous pouvez utiliser le composant ToolTip pour afficher un texte d’aide lorsque l’utilisateur pointe sur les contrôles. Un ToolTip peut être associé à n’importe quel contrôle.

Noter

Utilisation de HelpProvider et de la classe Help Vous pouvez afficher des fichiers d’aide compilés (.chm) ou des fichiers HTML au format d’aide HTML. Les fichiers d’aide compilés fournissent une table des matières, un index, une capacité de recherche et des liens de mots clés dans les pages. Les raccourcis ne fonctionnent que dans les fichiers d’aide compilés. Vous pouvez générer des fichiers HTML Help 1.x à l’aide de HTML Help Workshop. Pour plus d’informations sur l’aide HTML, consultez “HTML Help Workshop” et d’autres rubriques d’aide HTML sur Microsoft HTML Help .

Afficher le fichier d’aide #

La Help Class encapsule le moteur HTML Help 1.0. Vous pouvez utiliser l’objet Help pour afficher des fichiers d’aide compilés (.chm) ou des fichiers HTML au format d’aide HTML. Les fichiers d’aide compilés fournissent une table des matières, un index, une recherche et des liens de mots clés dans les pages. Les raccourcis ne fonctionnent que dans les fichiers d’aide compilés. Vous pouvez générer des fichiers HTML Help 1.x avec un outil gratuit de Microsft appelé HTML Help Workshop.

Un moyen simple d’afficher un fichier d’aide compilé dans une seconde fenêtre :

C#

Help.ShowHelp(this, helpProviderMain.HelpNamespace);

VB.NET

Help.ShowHelp(Me, hlpProviderMain.HelpNamespace)

Gestion de l’événement HelpRequested des contrôles et du formulaire #

Lorsqu’un utilisateur appuie sur F1 sur un contrôle ou clique sur le bouton Aide du formulaire (?), puis clique sur un contrôle, [HelpRequested](https://msdn L’événement .microsoft.com/en-us/library/system.windows.forms.control.helprequested(v=vs.110).aspx) sera déclenché.

Vous pouvez gérer cet événement pour fournir une action personnalisée lorsque l’utilisateur demande de l’aide pour les contrôles ou le formulaire.

Le HelpRequested prend en charge le mécanisme de bulle. Il se déclenche pour votre contrôle actif et si vous ne gérez pas l’événement et ne définissez pas [Handled](https://msdn.microsoft.com/en-us/library/system.windows.forms.helpeventargs.handled( v=vs.110).aspx) propriété de son événement arg sur true, puis il remonte jusqu’à la hiérarchie de contrôle parent jusqu’au formulaire.

Par exemple, si vous gérez l’événement HelpRequested du formulaire comme ci-dessous, alors lorsque vous appuyez sur F1 une boîte de message apparaîtra et affichera le nom du contrôle actif, mais pour textBox1 il affichera un autre message:

private void Form1_HelpRequested(object sender, HelpEventArgs hlpevent)
{
    var c = this.ActiveControl;
    if(c!=null)
        MessageBox.Show(c.Name);
}
private void textBox1_HelpRequested(object sender, HelpEventArgs hlpevent)
{
    hlpevent.Handled = true;
    MessageBox.Show("Help request handled and will not bubble up");
}

Vous pouvez effectuer toute autre action personnalisée, comme utiliser la navigation vers une URL ou afficher un fichier CHM à l’aide de [Help](https://msdn.microsoft.com/en-us/library/system.windows.forms.help(v =vs.110).aspx) classe.

Créer un bouton d’aide personnalisé qui agit comme un bouton d’aide de formulaire standard #

Si vous avez un Form avec MinimizeBox et MaximizeBox définis sur true, vous ne pouvez pas afficher le bouton Aide sur la barre de titre de Form et perdrez la fonction de cliquer sur le bouton d’aide pour le convertir en curseur d’aide pour pouvoir cliquer sur les contrôles pour afficher l’aide.

Vous pouvez faire en sorte qu’un élément de menu sur MenuStrip agisse comme un bouton d’aide standard. Pour ce faire, ajoutez un MenuStrip au formulaire et ajoutez-y un ToolStripMenuItem, puis gérez l’événement Click de l’élément :

private const int WM_SYSCOMMAND = 0x0112;
private const int SC_CONTEXTHELP = 0xF180;
[System.Runtime.InteropServices.DllImport("user32.dll")]
static extern IntPtr SendMessage(IntPtr hWnd, int Msg, int wParam, int lParam);
private void helpToolStripMenuItem_Click(object sender, EventArgs e)
{
    SendMessage(this.Handle, WM_SYSCOMMAND, SC_CONTEXTHELP, 0);
}

Remarque : Si vous souhaitez le faire à l’aide d’un Button, vous devez également définir button1.Capture = false; avant d’envoyer le message. Mais ce n’est pas nécessaire pour un ToolStripMenuItem.

Ensuite, lorsque vous cliquez sur le menu d’aide, le curseur sera remplacé par le curseur " ?" et agira comme lorsque vous cliquez sur le bouton d’aide standard :

Afficher l’aide pour MessageBox #

Vous pouvez fournir de l’aide pour la boîte de message de différentes manières. Vous pouvez configurer une MessageBox pour afficher ou non un bouton Aide. Vous pouvez également configurer MessageBox de manière à ce que lorsque l’utilisateur demande de l’aide en cliquant sur le bouton Aide ou en appuyant sur F1, il affiche un fichier CHM ou navigue vers une URL ou effectue une action personnalisée. Voici quelques exemples dans ce sujet.

Dans tous les exemples ci-dessous, la MessageBox ressemblerait à ceci :

[![MessageBox avec bouton d’aide][1]][1]

Afficher un fichier CHM et naviguer jusqu’à un mot-clé (index) #

MessageBox.Show("Some Message", "Title", MessageBoxButtons.YesNoCancel,
    MessageBoxIcon.Question, MessageBoxDefaultButton.Button3, 0,
    "help.chm", HelpNavigator.KeywordIndex, "SomeKeyword");

Afficher un fichier CHM et accéder à un sujet #

MessageBox.Show("Some Message", "Title", MessageBoxButtons.YesNoCancel,
    MessageBoxIcon.Question, MessageBoxDefaultButton.Button3, 0,
    "help.chm", HelpNavigator.Topic, "/SomePath/SomePage.html");

Afficher un fichier CHM et parcourir la première page d’aide dans la table des matières #

MessageBox.Show("Some Message", "Title", MessageBoxButtons.YesNoCancel,
    MessageBoxIcon.Question, MessageBoxDefaultButton.Button3, 0,
    "help.chm");

Ouvrez le navigateur par défaut et accédez à une URL #

MessageBox.Show("Some Message", "Title", MessageBoxButtons.YesNoCancel,
    MessageBoxIcon.Question, MessageBoxDefaultButton.Button3, 0,
    "http://example.com");

Effectuez une action personnalisée lorsque vous appuyez sur le bouton Aide ou sur la touche F1 #

Dans ce cas, vous devez gérer l’événement HelpRequested du parent de MessageBox et effectuez une opération personnalisée :

private void Form1_HelpRequested(object sender, HelpEventArgs hlpevent)
{
    // Perform custom action, for example show a custom help form
    var f = new Form();  
    f.ShowDialog();
}

Ensuite, vous pouvez afficher la MessageBox avec le bouton Aide :

MessageBox.Show("Some Message", "Title", MessageBoxButtons.YesNoCancel,
    MessageBoxIcon.Question, MessageBoxDefaultButton.Button3, 0, true);

Ou affichez-le sans le bouton Aide :

MessageBox.Show("Some Message", "Title", MessageBoxButtons.YesNoCancel,
    MessageBoxIcon.Question, MessageBoxDefaultButton.Button3, 0, false);

[1] : http://i.stack.imgur.com/GahF4.png

Afficher l’aide pour CommonDialogs #

Vous pouvez fournir de l’aide pour OpenFileDialog, SaveFileDialog et ColorDialog. Pour ce faire, définissez la propriété ShowHelp de la boîte de dialogue sur true et gérez [HelpRequest](https://msdn.microsoft.com/en-us/library/system.windows.forms.commondialog.helprequest(v= vs.110).aspx) événement pour la boîte de dialogue :

void openFileDialog1_HelpRequest(object sender, EventArgs e)
{
    //Perform custom action
    Help.ShowHelp(this, "Http://example.com");
}

Noter

  • L’événement ne sera déclenché que si vous définissez ShowHelp sur true.
  • L’événement ne sera déclenché qu’en cliquant sur le bouton “Aide” et ne sera pas déclenché à l’aide de la touche F1.

Dans l’image ci-dessous, vous pouvez voir un “OpenFileDialog” avec un bouton d’aide :

[![Ouvrir la boîte de dialogue de fichier avec le bouton d’aide][1]][1]

[1] : http://i.stack.imgur.com/mcvGL.png

Afficher l’aide à l’aide de la classe d’aide #

Vous pouvez utiliser la classe Help dans le code pour fournir ces types d’aide :

  • Afficher une fenêtre contextuelle d’aide pour un contrôle
  • Ouvrir un fichier CHM en fonction du contexte (Afficher la table des matières, Afficher un mot clé ou un index, afficher un sujet)
  • Accédez à une URL à l’aide du navigateur par défaut

Afficher la fenêtre contextuelle d’aide #

Vous pouvez utiliser Help.ShowPopup pour afficher une aide fenêtre contextuelle :

private void control_MouseClick(object sender, MouseEventArgs e)
{
    var c = (Control)sender;
    var help = "Lorem ipsum dolor sit amet, consectetur adipiscing elit, " +
               "sed do eiusmod tempor incididunt ut labore et dolore magna aliqua."
    if (c != null)
        Help.ShowPopup(c, "Lorem ipsum dolor sit amet.", c.PointToScreen(e.Location));
}

Il affichera une telle aide contextuelle à l’emplacement du pointeur de votre souris :

[![entrez la description de l’image ici][1]][1]

Afficher le fichier d’aide CHM #

Vous pouvez utiliser différentes surcharges de la méthode Help.ShowHelp, pour afficher un fichier CHM et accéder à un mot-clé, un sujet, un index ou une table des matières :

Afficher la table des matières de l’aide #

Help.ShowHelp(this, "Help.chm");

Afficher l’aide pour un mot clé spécifique (index) #

Help.ShowHelp(this, "Help.chm", HelpNavigator.Index, "SomeKeyword");

Afficher l’aide pour un sujet spécifique #

Help.ShowHelp(this, "Help.chm", HelpNavigator.Topic, "/SomePath/SomePage.html");

Afficher l’URL #

Vous pouvez afficher n’importe quelle URL dans le navigateur par défaut en utilisant la méthode “ShowHelp” :

Help.ShowHelp(this, "Http://example.com");

[1] : http://i.stack.imgur.com/UrZtU.png

Afficher le bouton d’aide sur la barre de titre du formulaire #

Vous pouvez afficher un bouton d’aide dans la barre de titre d’un “formulaire”. Pour ce faire, vous devez :

  1. Définissez la propriété HelpButton du formulaire sur true .
  2. Définissez MinimizeBox et MaximizeBox sur false.

Ensuite, un bouton d’aide apparaîtra sur la barre de titre de Form :

[![entrez la description de l’image ici][1]][1]

De plus, lorsque vous cliquez sur le bouton Aide, le curseur sera remplacé par un curseur ? :

[![entrez la description de l’image ici][2]][2]

Ensuite, si vous cliquez sur un Control ou Form, l’événement HelpRequested sera déclenché et également si vous avez configuré un HelpProvider, l’aide pour le contrôle sera affichée à l’aide de HelpProvider.

[1] : http://i.stack.imgur.com/otusY.png [2] : http://i.stack.imgur.com/HS79o.png

Gestion de l’événement HelpButtonClicked du formulaire #

Vous pouvez détecter quand un utilisateur a cliqué sur un HelpButton sur la barre de titre du formulaire en gérant [HelpButtonClicked](https://msdn.microsoft.com/en-us/library/system.windows.forms.form. bouton d’aide cliqué (v=vs.110).aspx). Vous pouvez laisser l’événement continuer ou l’annuler en définissant la propriété Cancel de ses arguments d’événement sur true.

private void Form1_HelpButtonClicked(object sender, CancelEventArgs e)
{
    e.Cancel = true;
    //Perform some custom action
    MessageBox.Show("Some Custom Help");
}
