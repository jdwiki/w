Utilisation des outils de développement pour déboguer votre extension #

Une rallonge chromée est séparée en 4 parties maximum :

  • la page de fond
  • la page contextuelle
  • un ou plusieurs scripts de contenu
  • la page des options

Chaque partie, puisqu’elles sont intrinsèquement séparées, nécessite un débogage individuel.

**Gardez à l’esprit que ces pages sont séparées, ce qui signifie que les variables ne sont pas directement partagées entre elles et qu’un ** console.log() *dans l’une de ces pages ne sera visible dans les journaux d’aucune autre partie. *

Utilisation des outils de développement Chrome :

Les extensions Chrome sont déboguées de la même manière que les autres applications Web et pages Web. Le débogage se fait le plus souvent avec l’utilisation de l’inspecteur devtools de chrome ouvert en utilisant le raccourci clavier pour windows et macs respectivement : ctrl+shift+i et cmd+shift+i ou en faisant un clic droit sur la page et en sélectionnant inspecter.

À partir de l’inspecteur, un développeur peut vérifier les éléments html et la façon dont css les affecte, ou utiliser la console pour inspecter les valeurs des variables javascript et lire les sorties de n’importe quel console.log() que le ou les développeurs ont configuré.

Vous trouverez plus d’informations sur l’utilisation de l’inspecteur sur [Chrome Devtools][1].

Inspection de la fenêtre contextuelle, de la page d’options et d’autres pages accessibles à l’aide de chrome://…..yourExtensionId…/:

La page contextuelle et la page d’options sont accessibles simplement en les inspectant lorsqu’elles sont ouvertes.

Les pages html supplémentaires qui font partie de l’extension, mais qui ne sont ni la fenêtre contextuelle ni la page d’options sont également déboguées de la même manière.

Inspection de la page d’arrière-plan :

Pour accéder à votre page d’arrière-plan, vous devez d’abord accéder à la page d’extension Chrome à l’adresse [chrome://extensions/][2]. Assurez-vous que la coche ‘Mode développeur’ est activée. [![entrez la description de l’image ici][3]][3]

Cliquez ensuite sur votre script d’arrière-plan à côté de “Inspecter les vues” pour inspecter votre page d’arrière-plan. [![entrez la description de l’image ici][4]][4]

Inspection des scripts de contenu :

Les scripts de contenu s’exécutent parallèlement aux sites Web dans lesquels ils ont été insérés. Vous pouvez inspecter le script de contenu en inspectant d’abord le site Web où le script de contenu est inséré. Dans la console, vous pourrez voir tous les console.log() produits par votre extension, mais vous ne pourrez pas modifier ou inspecter les variables du script de contenu. [![inspection de la page supérieure (parent)][5]][5]

Pour résoudre ce problème, vous devez cliquer sur le menu déroulant qui est généralement défini sur 'top' et sélectionner votre extension dans la liste des extensions. [![inspection du script de contenu][6]][6]

De là, vous aurez accès aux variables de votre extension.
