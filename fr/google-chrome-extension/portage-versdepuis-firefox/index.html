Si vous utilisez une version Firefox antérieure à 48, vous aurez également besoin d’un clé supplémentaire dans manifest.json appelée applications :

"applications": {
  "gecko": {
    "id": "borderify@example.com",
    "strict_min_version": "42.0",
    "strict_max_version": "50.*",
    "update_url": "https://example.com/updates.json"
  }
}

[candidatures][1]

Noter:

[Signature d’extension][2] :

Avec la sortie de Firefox 48, la signature d’extension ne peut plus être désactivé dans les builds de version et de canal bêta en utilisant une préférence. Comme indiqué lors de l’annonce de la signature de l’extension, nous publions des versions spécialisées qui prennent en charge cette préférence afin que les développeurs puissent continuer à tester par rapport au code que les versions bêta et de version sont généré à partir de.

État des “extensions Web” :

Les WebExtensions sont actuellement dans un état alpha expérimental. De Firefox 46, vous pouvez publier des WebExtensions pour les utilisateurs de Firefox, tout comme tout autre module complémentaire. Nous visons une première version stable dans Firefox 48.

MISE À JOUR : Firefox 48 publié le 02.08.2016.

Portage via WebExtensions #

Avant de parler de portage des extensions Firefox depuis/vers, il faut savoir ce qu’est WebExtensions.

WebExtensions - est une plateforme qui représente une API pour créer des extensions Firefox.

Il utilise la même architecture d’extension que Chromium, par conséquent, cette API est compatible à bien des égards avec l’API de Google Chrome et Opera (Opera basé sur Chromium). Dans de nombreux cas, les extensions développées pour ces navigateurs fonctionneront dans Firefox avec quelques modifications ou même sans elles du tout.

MDN [recommande][1] d’utiliser WebExtension pour les nouvelles extensions :

À l’avenir, WebExtensions sera le moyen recommandé pour développer Les modules complémentaires de Firefox et d’autres systèmes seront obsolètes.

Compte tenu de ce qui précède, si vous souhaitez porter des extensions vers Firefox, vous devez savoir comment l’extension a été écrite.

Les extensions pour Firefox peuvent être basées sur WebExtension, Add-on SDK ou XUL.

Extensions compatibles basées sur WebExtension #

Lors de l’utilisation de WebExtension, il faut parcourir la liste des [incompatibilités][2], car certaines fonctions sont entièrement ou partiellement prises en charge, c’est-à-dire qu’il faut vérifier son manifest.json.

Cela permet également d’utiliser le même [namespace][3] :

À l’heure actuelle, toutes les API sont accessibles via l’espace de noms chrome.. Lorsque nous commençons à ajouter nos propres API, nous prévoyons de les ajouter au navigateur. espace de noms. Les développeurs pourront utiliser la détection de fonctionnalités pour déterminer si une API est disponible dans le navigateur.*.

Une extension simple qui peut fonctionner dans Firefox et Google Chrome

manifest.json :

{
  "manifest_version": 2,
  
  "name": "StackMirror",
  
  "version": "1.0",
  
  "description": "Mirror reflection of StackOverflow sites",
  
  "icons": {
    "48": "icon/myIcon-48.png"
  },
  
  "page_action": {
    "default_icon": "icon/myIcon-48.png"
  },
  
  "background": {
    "scripts"   : ["js/background/script.js"],
    "persistent": false
  },
  
  "permissions": ["tabs", "*://*.stackoverflow.com/*"]
}

Script “arrière-plan” :

function startScript(tabId, changeInfo, tab) {

    if (tab.url.indexOf("stackoverflow.com") > -1) {

        chrome.tabs.executeScript(tabId, 
            
            {code: 'document.body.style.transform = "scaleX(-1)";'}, function () {

            if (!chrome.runtime.lastError) {

                chrome.pageAction.show(tabId);
            }
        });
    }
}

chrome.tabs.onUpdated.addListener(startScript);

Compressez le projet en tant que fichier zip standard, mais avec les extensions .xpi.

Ouvrez la page about:addons, accessible via Menu > Add-ons.
