Cette rubrique se compose d’une grande variété de trucs et astuces utiles découverts par les utilisateurs SO grâce à leur expérience dans le codage. Ce sont souvent des exemples de moyens de contourner les frustrations courantes ou des façons d’utiliser Excel de manière plus “intelligente”.

Utilisation de feuilles xlVeryHidden #

Les feuilles de calcul dans Excel ont trois options pour la propriété “Visible”. Ces options sont représentées par des constantes dans l’énumération xlSheetVisibility et sont les suivantes :

  1. Valeur xlVisible ou xlSheetVisible : -1 (valeur par défaut pour les nouvelles feuilles)
  2. Valeur xlHidden ou xlSheetHidden : 0
  3. xlVeryHidden valeur xlSheetVeryHidden : 2

Les feuilles visibles représentent la visibilité par défaut des feuilles. Ils sont visibles dans la barre d’onglets de la feuille et peuvent être librement sélectionnés et visualisés. Les feuilles masquées sont masquées dans la barre d’onglets des feuilles et ne peuvent donc pas être sélectionnées. Cependant, les feuilles masquées peuvent être affichées dans la fenêtre Excel en cliquant avec le bouton droit sur les onglets de la feuille et en sélectionnant “Afficher”

Les feuilles très masquées, en revanche, sont uniquement accessibles via Visual Basic Editor. Cela en fait un outil incroyablement utile pour stocker des données sur des instances d’Excel ainsi que pour stocker des données qui devraient être cachées aux utilisateurs finaux. Les feuilles sont accessibles par référence nommée dans le code VBA, permettant une utilisation facile des données stockées.

Pour modifier manuellement la propriété .Visible d’une feuille de calcul en xlSheetVeryHidden, ouvrez la fenêtre Propriétés du VBE (F4), sélectionnez la feuille de calcul que vous souhaitez modifier et utilisez la liste déroulante de la treizième ligne pour effectuer votre sélection.

[![worksheet_properties_window_visible][1]][1]

Pour modifier la propriété .Visible d’une feuille de calcul en xlSheetVeryHidden¹ dans le code, accédez de la même manière à la propriété .Visible et attribuez une nouvelle valeur.

with Sheet3
    .Visible = xlSheetVeryHidden
end with

¹ xlVeryHidden et xlSheetVeryHidden renvoient une valeur numérique de 2 (ils sont interchangeables).

Utilisation de chaînes avec des délimiteurs à la place de tableaux dynamiques #

L’utilisation de tableaux dynamiques dans VBA peut être assez fastidieuse et chronophage sur de très grands ensembles de données. Lors du stockage de types de données simples dans un tableau dynamique (chaînes, nombres, booléens, etc.), on peut éviter les instructions “ReDim Preserve” requises pour les tableaux dynamiques dans VBA en utilisant la fonction “Split()” avec des procédures de chaîne intelligentes. Par exemple, nous examinerons une boucle qui ajoute une série de valeurs d’une plage à une chaîne en fonction de certaines conditions, puis utilise cette chaîne pour remplir les valeurs d’un ListBox.

Private Sub UserForm_Initialize()

Dim Count As Long, DataString As String, Delimiter As String

For Count = 1 To ActiveSheet.UsedRows.Count
    If ActiveSheet.Range("A" & Count).Value <> "Your Condition" Then
        RowString = RowString & Delimiter & ActiveSheet.Range("A" & Count).Value
        Delimiter = "><" 'By setting the delimiter here in the loop, you prevent an extra occurance of the delimiter within the string
    End If
Next Count

ListBox1.List = Split(DataString, Delimiter)

End Sub

La chaîne Delimiter elle-même peut être définie sur n’importe quelle valeur, mais il est prudent de choisir une valeur qui n’apparaîtra pas naturellement dans l’ensemble. Supposons, par exemple, que vous traitiez une colonne de dates. Dans ce cas, utiliser ., - ou / serait imprudent comme délimiteurs, car les dates pourraient être formatées pour utiliser n’importe lequel d’entre eux, générant plus de points de données que prévu.

Remarque : Il existe des limites à l’utilisation de cette méthode (à savoir la longueur maximale des chaînes), elle doit donc être utilisée avec prudence dans le cas d’ensembles de données très volumineux. Ce n’est pas nécessairement la méthode la plus rapide ou la plus efficace pour créer des tableaux dynamiques dans VBA, mais c’est une alternative viable.

Feuille de calcul .Name, .Index ou .CodeName #

Nous savons que les “meilleures pratiques” dictent qu’un objet de plage doit avoir sa feuille de calcul parent explicitement référencée. Une feuille de calcul peut être référencée par sa propriété .Name, sa propriété numérique .Index ou sa propriété .CodeName mais un utilisateur peut réorganiser la file d’attente de la feuille de calcul en faisant simplement glisser un onglet de nom ou renommer la feuille de calcul avec un double-clic sur le même onglet et certains taper dans un classeur non protégé.

Considérez une feuille de calcul standard à trois. Vous avez renommé les trois feuilles de calcul lundi, mardi et mercredi dans cet ordre et codé les sous-procédures VBA qui y font référence. Maintenant, considérez qu’un utilisateur arrive et décide que lundi appartient à la fin de la file d’attente de la feuille de calcul, puis un autre arrive et décide que les noms de feuille de calcul semblent mieux en français. Vous avez maintenant un classeur avec une file d’attente d’onglets de nom de feuille de calcul qui ressemble à ce qui suit.

[![worksheet_tab_queue][1]][1]

Si vous aviez utilisé l’une des méthodes de référence de feuille de calcul suivantes, votre code serait maintenant cassé.

'reference worksheet by .Name
with worksheets("Monday")
    'operation code here; for example:
    .Range(.Cells(2, "A"), .Cells(.Rows.Count, "A").End(xlUp)) = 1
end with

'reference worksheet by ordinal .Index
with worksheets(1)
    'operation code here; for example:
    .Range(.Cells(2, "A"), .Cells(.Rows.Count, "A").End(xlUp)) = 1
end with

La commande d’origine et le nom de la feuille de calcul d’origine ont été compromis. Cependant, si vous aviez utilisé la propriété .CodeName de la feuille de calcul, votre sous-procédure serait toujours opérationnelle

with Sheet1
    'operation code here; for example:
    .Range(.Cells(2, "A"), .Cells(.Rows.Count, "A").End(xlUp)) = 1
end with

L’image suivante montre la fenêtre Projet VBA ([Ctrl]+R) qui répertorie les feuilles de calcul par .CodeName puis par .Name (entre parenthèses). L’ordre dans lequel ils sont affichés ne change pas ; l’ordinal .Index est pris par l’ordre dans lequel ils sont affichés dans la file d’attente des onglets de nom dans la fenêtre de la feuille de calcul.

[![worksheet_project_window][2]][2]

Bien qu’il soit rare de renommer un .CodeName, ce n’est pas impossible. Ouvrez simplement la fenêtre Propriétés du VBE ([F4]).

[![worksheet_properties_window][3]][3]

La feuille de calcul .CodeName se trouve dans la première ligne. Le nom de la feuille de calcul est dans le dixième. Les deux sont modifiables.

## Événement de double-clic pour les formes Excel Par défaut, les formes dans Excel n’ont pas de méthode spécifique pour gérer les clics simples ou doubles, contenant uniquement la propriété “OnAction” pour vous permettre de gérer les clics. Cependant, il peut y avoir des cas où votre code vous oblige à agir différemment (ou exclusivement) lors d’un double-clic. La sous-routine suivante peut être ajoutée à votre projet VBA et, lorsqu’elle est définie comme routine “OnAction” pour votre forme, vous permet d’agir sur les doubles clics.

Public Const DOUBLECLICK_WAIT as Double = 0.25 'Modify to adjust click delay
Public LastClickObj As String, LastClickTime As Date

Sub ShapeDoubleClick()
    
    If LastClickObj = "" Then
        LastClickObj = Application.Caller
        LastClickTime = CDbl(Timer)
    Else
        If CDbl(Timer) - LastClickTime > DOUBLECLICK_WAIT Then
            LastClickObj = Application.Caller
            LastClickTime = CDbl(Timer)
        Else
            If LastClickObj = Application.Caller Then
                'Your desired Double Click code here
                LastClickObj = ""
            Else
                LastClickObj = Application.Caller
                LastClickTime = CDbl(Timer)
            End If
        End If
    End If
    
End Sub

Cette routine fera en sorte que la forme ignore fonctionnellement le premier clic, exécutant uniquement le code souhaité au deuxième clic dans le laps de temps spécifié.

Boîte de dialogue Ouvrir un fichier - Plusieurs fichiers #

Cette sous-routine est un exemple rapide sur la façon de permettre à un utilisateur de sélectionner plusieurs fichiers, puis de faire quelque chose avec ces chemins de fichiers, comme obtenir les noms de fichiers et les envoyer à la console via debug.print.

Option Explicit

Sub OpenMultipleFiles()
    Dim fd As FileDialog
    Dim fileChosen As Integer
    Dim i As Integer
    Dim basename As String
    Dim fso As Variant
    Set fso = CreateObject("Scripting.FileSystemObject")
    Set fd = Application.FileDialog(msoFileDialogFilePicker)
    basename = fso.getBaseName(ActiveWorkbook.Name)
    fd.InitialFileName = ActiveWorkbook.Path ' Set Default Location to the Active Workbook Path
    fd.InitialView = msoFileDialogViewList
    fd.AllowMultiSelect = True
    
    fileChosen = fd.Show
    If fileChosen = -1 Then
        'open each of the files chosen
        For i = 1 To fd.SelectedItems.Count
            Debug.Print (fd.SelectedItems(i))
            Dim fileName As String
            ' do something with the files.
            fileName = fso.getFileName(fd.SelectedItems(i))
            Debug.Print (fileName)
        Next i
    End If

End Sub
