Fenêtre immédiate #

Si vous souhaitez tester une ligne de code de macro sans avoir à exécuter un sous-programme entier, vous pouvez taper des commandes directement dans la fenêtre immédiate et appuyer sur “ENTRER” pour exécuter la ligne.

Pour tester la sortie d’une ligne, vous pouvez la faire précéder d’un point d’interrogation ? pour imprimer directement dans la fenêtre immédiate. Alternativement, vous pouvez également utiliser la commande print pour imprimer la sortie.

Dans Visual Basic Editor, appuyez sur CTRL + G pour ouvrir la fenêtre Exécution. Pour renommer votre feuille actuellement sélectionnée en “ExampleSheet”, tapez ce qui suit dans la fenêtre immédiate et appuyez sur “ENTRER”

   ActiveSheet.Name = "ExampleSheet"

Pour imprimer le nom de la feuille actuellement sélectionnée directement dans la fenêtre Exécution

? ActiveSheet.Name
ExampleSheet

Cette méthode peut être très utile pour tester la fonctionnalité des fonctions intégrées ou définies par l’utilisateur avant de les implémenter dans le code. L’exemple ci-dessous montre comment la fenêtre immédiate peut être utilisée pour tester la sortie d’une fonction ou d’une série de fonctions pour confirmer un résultat attendu.

'In this example, the Immediate Window was used to confirm that a series of Left and Right 
'string methods would return the desired string

'expected output: "value"
print Left(Right("1111value1111",9),5) ' <---- written code here, ENTER pressed
value                                  ' <---- output

La fenêtre immédiate peut également être utilisée pour définir ou réinitialiser l’application, le classeur ou d’autres propriétés nécessaires. Cela peut être utile si vous avez Application.EnableEvents = False dans une sous-routine qui génère une erreur de manière inattendue, provoquant sa fermeture sans réinitialiser la valeur à True (ce qui peut entraîner une fonctionnalité frustrante et inattendue. Dans ce cas, les commandes peut être saisi directement dans la fenêtre Exécution et exécuté :

? Application.EnableEvents       ' <---- Testing the current state of "EnableEvents"
False                            ' <---- Output
Application.EnableEvents = True  ' <---- Resetting the property value to True
? Application.EnableEvents       ' <---- Testing the current state of "EnableEvents"
True                             ' <---- Output

Pour des techniques de débogage plus avancées, un deux-points : peut être utilisé comme séparateur de ligne. Cela peut être utilisé pour les expressions multilignes telles que les boucles dans l’exemple ci-dessous.

x = Split("a,b,c",","): For i = LBound(x,1) to UBound(x,1): Debug.Print x(i): Next i '<----Input this and press enter
a '<----Output
b '<----Output
c '<----Output

Débogage.Imprimer #

Pour imprimer une liste des descriptions des codes d’erreur dans la fenêtre immédiate, transmettez-la à la fonction “Debug.Print” :

Private Sub ListErrCodes()
    Debug.Print "List Error Code Descriptions"
    For i = 0 To 65535
        e = Error(i)
        If e <> "Application-defined or object-defined error" Then Debug.Print i & ": " & e
    Next i
End Sub

Vous pouvez afficher la fenêtre Exécution en :

  • Sélection Vue | IFenêtre immédiate à partir de la barre de menu
  • En utilisant le raccourci clavier Ctrl-G

Utilisez la minuterie pour trouver les goulots d’étranglement dans les performances #

La première étape de l’optimisation de la vitesse consiste à trouver les sections de code les plus lentes. La fonction VBA Timer renvoie le nombre de secondes écoulées depuis minuit avec une précision de 1/256e de seconde (3,90625 millisecondes) sur les PC Windows. Les fonctions VBA Now et Time ne sont précises qu’à la seconde près.

Dim start As Double       ' Timer returns Single, but converting to Double to avoid 
start = Timer             ' scientific notation like 3.90625E-03 in the Immediate window
' ... part of the code
Debug.Print Timer - start; "seconds in part 1" 

start = Timer
' ... another part of the code
Debug.Print Timer - start; "seconds in part 2"

Fenêtre Locales du débogueur #

La fenêtre Variables locales permet d’accéder facilement à la valeur actuelle des variables et des objets dans le cadre de la fonction ou de la sous-routine que vous exécutez. C’est un outil essentiel pour déboguer votre code et parcourir les modifications afin de trouver des problèmes. Il vous permet également d’explorer des propriétés dont vous ignoriez peut-être l’existence.

Prenons l’exemple suivant,

Option Explicit
Sub LocalsWindowExample()
    Dim findMeInLocals As Integer
    Dim findMEInLocals2 As Range
    
    findMeInLocals = 1
    Set findMEInLocals2 = ActiveWorkbook.Sheets(1).Range("A1")
End Sub

Dans l’éditeur VBA, cliquez sur Affichage -> Fenêtre Locals

[![Où se trouve le menu des locaux dans la fenêtre][1]][1]

Ensuite, en parcourant le code à l’aide de F8 après avoir cliqué à l’intérieur de la sous-routine, nous nous sommes arrêtés avant d’attribuer findMeinLocals. Ci-dessous, vous pouvez voir que la valeur est 0 — et c’est ce qui serait utilisé si vous ne lui attribuiez jamais de valeur. L’objet de plage est ‘Rien’.

[![A cette étape les variables ne sont rien…][2]][2]

[![Montrer que les variables ne sont rien.][3]][3]

Si nous nous arrêtons juste avant la fin du sous-programme, nous pouvons voir les valeurs finales des variables.

[![Définissez un point de débogage… si vous le souhaitez][4]][4]

Nous pouvons voir findMeInLocals avec une valeur de 1 et un type Integer, et FindMeInLocals2 avec un type Range/Range. Si nous cliquons sur le signe +, nous pouvons développer l’objet et voir ses propriétés, telles que le nombre ou la colonne.

Arrêt #

La commande Stop mettra l’exécution en pause lorsqu’elle sera appelée. À partir de là, le processus peut être repris ou être exécuté étape par étape.

Sub Test()
    Dim TestVar as String
    TestVar = "Hello World"
    Stop                    'Sub will be executed to this point and then wait for the user
    MsgBox TestVar
End Sub

Ajouter un point d’arrêt à votre code #

Vous pouvez facilement ajouter un point d’arrêt à votre code en cliquant sur la colonne grise à gauche de la ligne de votre code VBA où vous souhaitez arrêter l’exécution. Un point rouge apparaît dans la colonne et le code du point d’arrêt est également surligné en rouge.

Vous pouvez ajouter plusieurs points d’arrêt dans votre code et la reprise de l’exécution s’effectue en appuyant sur l’icône “jouer” dans votre barre de menus. Tout le code ne peut pas être un point d’arrêt car les lignes de définition de variable, la première ou la dernière ligne d’une procédure et les lignes de commentaire ne peuvent pas être sélectionnées comme point d’arrêt.
