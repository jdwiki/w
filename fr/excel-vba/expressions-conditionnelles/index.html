L’instruction Si #

L’instruction de contrôle “If” permet d’exécuter un code différent en fonction de l’évaluation d’une instruction conditionnelle (booléenne). Une instruction conditionnelle est une instruction qui prend la valeur “True” ou “False”, par ex. x > 2.

Il existe trois modèles qui peuvent être utilisés lors de l’implémentation d’une instruction “If”, qui sont décrits ci-dessous. Notez qu’une évaluation conditionnelle “Si” est toujours suivie d’un “Alors”.

**1. Évaluer une instruction conditionnelle If et faire quelque chose si c’est True **

Instruction If sur une seule ligne

C’est le moyen le plus court d’utiliser un If et il est utile lorsqu’une seule déclaration doit être effectuée lors d’une évaluation True. Lorsque vous utilisez cette syntaxe, tout le code doit être sur une seule ligne. N’incluez pas End If à la fin de la ligne.

If [Some condition is True] Then [Do something]

Bloc “Si”

Si plusieurs lignes de code doivent être exécutées lors d’une évaluation “True”, un bloc “If” peut être utilisé.

If [Some condition is True] Then
  [Do some things]
End If

Notez que, si un bloc If multiligne est utilisé, un End If correspondant est requis.

**2. Évaluer une instruction conditionnelle If, faire une chose si c’est True et faire autre chose si c’est False **

Instruction If, Else sur une seule ligne

Cela peut être utilisé si une déclaration doit être effectuée lors d’une évaluation “Vrai” et une déclaration différente doit être effectuée lors d’une évaluation “Faux”. Soyez prudent lorsque vous utilisez cette syntaxe, car il est souvent moins clair pour les lecteurs qu’il existe une instruction “Else”. Lorsque vous utilisez cette syntaxe, tout le code doit être sur une seule ligne. N’incluez pas End If à la fin de la ligne.

If [Some condition is True] Then [Do something] Else [Do something else]

Bloc If, Else

Utilisez un bloc If, Else pour ajouter de la clarté à votre code, ou si plusieurs lignes de code doivent être exécutées sous une évaluation True ou False.

If [Some condition is True] Then
   [Do some things]
Else
   [Do some other things]
End If

Notez que, si un bloc If multiligne est utilisé, un End If correspondant est requis.

**3. Évaluer de nombreuses instructions conditionnelles, lorsque les instructions précédentes sont toutes “Faux”, et faire quelque chose de différent pour chacune **

Ce modèle est l’utilisation la plus générale de “Si” et serait utilisé lorsqu’il existe de nombreuses conditions qui ne se chevauchent pas et qui nécessitent un traitement différent. Contrairement aux deux premiers modèles, ce cas nécessite l’utilisation d’un bloc If, même si une seule ligne de code sera exécutée pour chaque condition.

**Bloc If, ElseIf, ..., Else **

Au lieu d’avoir à créer plusieurs blocs “If” les uns sous les autres, un “ElseIf” peut être utilisé pour évaluer une condition supplémentaire. Le ElseIf n’est évalué que si une évaluation If précédente est False.

If [Some condition is True] Then
   [Do some thing(s)]
ElseIf [Some other condition is True] Then
   [Do some different thing(s)]
Else    'Everything above has evaluated to False
   [Do some other thing(s)]
End If

Autant d’instructions de contrôle ElseIf peuvent être incluses entre un If et un End If selon les besoins. Une instruction de contrôle Else n’est pas requise lors de l’utilisation de ElseIf (bien qu’elle soit recommandée), mais si elle est incluse, elle doit être l’instruction de contrôle finale avant End If.
