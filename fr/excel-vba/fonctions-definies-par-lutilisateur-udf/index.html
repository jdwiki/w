Syntaxe #

  1. Function functionName(argumentVariable As dataType, argumentVariable2 As dataType, Optional argumentVariable3 As dataType) As functionReturnDataType Déclaration de base d’une fonction. Chaque fonction a besoin d’un nom, mais elle n’a pas besoin de prendre d’arguments. Il peut prendre 0 argument ou un nombre donné d’arguments. Vous pouvez également déclarer un argument comme facultatif (ce qui signifie que cela n’a pas d’importance si vous le fournissez lors de l’appel de la fonction). Il est recommandé de fournir le type de variable pour chaque argument et, de même, de renvoyer le type de données que la fonction elle-même va renvoyer.

  2. nomFonction = theVariableOrValueBeingReturned Si vous venez d’autres langages de programmation, vous êtes peut-être habitué au mot-clé “Retour”. Ceci n’est pas utilisé dans VBA - à la place, nous utilisons le nom de la fonction. Vous pouvez le définir sur le contenu d’une variable ou sur une valeur fournie directement. Notez que si vous avez défini un type de données pour le retour de la fonction, la variable ou les données que vous fournissez cette fois doivent être de ce type de données.

  3. Fin de fonction Obligatoire. Signifie la fin du bloc de code Function et doit donc être à la fin. Le VBE le fournit généralement automatiquement lorsque vous créez une nouvelle fonction.

Une fonction définie par l’utilisateur (ou UDF) fait référence à une fonction spécifique à une tâche qui a été créée par l’utilisateur. Elle peut être appelée en tant que fonction de feuille de calcul (ex : =SUM(...)) ou utilisée pour renvoyer une valeur à un processus en cours d’exécution dans une procédure Sub. Une UDF renvoie une valeur, généralement à partir des informations qui lui sont transmises sous la forme d’un ou de plusieurs paramètres.

Il peut être créé par :

  1. en utilisant VBA.
  2. en utilisant l’API Excel C - En créant un XLL qui exporte les fonctions compilées vers Excel.
  3. en utilisant l’interface COM.

Autoriser les références de colonne complètes sans pénalité #

Il est plus facile d’implémenter certaines UDF sur la feuille de calcul si des références de colonne complètes peuvent être transmises en tant que paramètres. Cependant, en raison de la nature explicite du codage, toute boucle impliquant ces plages peut traiter des centaines de milliers de cellules complètement vides. Cela réduit votre projet VBA (et votre classeur) à un gâchis gelé pendant que des non-valeurs inutiles sont traitées.

La boucle dans les cellules d’une feuille de calcul est l’une des méthodes les plus lentes pour accomplir une tâche, mais elle est parfois inévitable. Réduire le travail effectué à ce qui est réellement nécessaire est parfaitement logique.

La solution consiste à tronquer la colonne complète ou les références de ligne complètes à la propriété Worksheet.UsedRange avec la méthode Intersect. L’exemple suivant répliquera vaguement la fonction SUMIF native d’une feuille de calcul afin que criteria_range soit également redimensionné pour s’adapter à sum_range puisque chaque valeur de sum_range doit être accompagnée d’une valeur dans criteria_range.

Le Application.Caller pour un UDF utilisé sur une feuille de calcul est la cellule dans laquelle il réside. La propriété .Parent de la cellule est la feuille de calcul. Ceci sera utilisé pour définir le .UsedRange.

Dans une feuille de code Module :

Option Explicit

Function udfMySumIf(rngA As Range, rngB As Range, _
                    Optional crit As Variant = "yes")
    Dim c As Long, ttl As Double
    
    With Application.Caller.Parent
        Set rngA = Intersect(rngA, .UsedRange)
        Set rngB = rngB.Resize(rngA.Rows.Count, rngA.Columns.Count)
    End With
    
    For c = 1 To rngA.Cells.Count
        If IsNumeric(rngA.Cells(c).Value2) Then
            If LCase(rngB(c).Value2) = LCase(crit) Then
                ttl = ttl + rngA.Cells(c).Value2
            End If
        End If
    Next c
    
    udfMySumIf = ttl

End Function

Syntaxe :
        =udfMySumIf(*sum_range*, *criteria_range*, [*criteria*])

Compter les valeurs uniques dans la plage #

Function countUnique(r As range) As Long
    'Application.Volatile False ' optional
    Set r = Intersect(r, r.Worksheet.UsedRange) ' optional if you pass entire rows or columns to the function
    Dim c As New Collection, v
    On Error Resume Next   ' to ignore the Run-time error 457: "This key is already associated with an element of this collection".
    For Each v In r.Value  ' remove .Value for ranges with more than one Areas
        c.Add 0, v & ""
    Next
    c.Remove "" ' optional to exclude blank values from the count
    countUnique = c.Count
End Function
