Package Control est un gestionnaire de packages complet qui permet de découvrir, d’installer, de mettre à jour et de supprimer des packages pour Sublime Text.

Package Control est le gestionnaire de packages Sublime Text. Il comprend une liste de plus de 2 500 packages disponibles pour l’installation, et les utilisateurs peuvent ajouter eux-mêmes n’importe quel référentiel GitHub ou BitBucket. Une fois installés, les packages sont mis à jour automatiquement.

L’exemple de code crée le dossier Installed Packages pour vous (si nécessaire), puis y télécharge le Package Control.sublime-package. Le téléchargement sera effectué via HTTP au lieu de HTTPS en raison des limitations de la bibliothèque standard Python, mais le fichier sera validé à l’aide de SHA-256.

Installation du contrôle des packages #

Si vous utilisez Sublime Text 3, le moyen le plus simple d’installer Package Control est de sélectionner “Installer Package Control” dans le menu “Outils” ou dans la “Palette de commandes” et Sublime Text l’installera automatiquement.

Installation manuelle : #

Pour Sublime Text 2, les anciennes versions de Sublime Text 3, ou si vous avez un problème lié au serveur proxy, alors Package Control peut être installé à l’aide d’un script Python qui doit être collé dans la console Sublime Text.

  • Ouvrez la [Page Web d’installation de Package Control][1] dans votre navigateur Web.
  • Cliquez sur la version Sublime Text dont vous avez besoin.
  • Copiez le script Python dans votre presse-papiers.
  • Ouvrez la console Sublime Text en sélectionnant Afficher la console dans le menu Affichage, ou en utilisant les touches de raccourci Ctrl+`, ou en sélectionnant Console : Afficher dans la Palette de commandes.
  • Collez le script Python dans la console et appuyez sur la touche “Entrée” pour exécuter le script.
  • L’installation prendra quelques secondes, mais une fois l’installation terminée, un nouveau tampon Sublime Text s’affichera avec des informations sur Package Control.

Vous pouvez à nouveau accéder au [Site Web de contrôle des packages][2] et choisir les packages que vous souhaitez installer.

[1] : https://packagecontrol.io/installation [2] : https://packagecontrol.io

Personnalisation du texte sublime #

Une fois que vous avez installé le contrôle des paquets, il est très facile d’installer n’importe quel * plugin *, * thème *, * schéma de couleurs *, * syntaxe * que vous voulez !

  • plugin : effectuer une action (compiler votre code less en css par exemple)
  • thème : changez tout l’habillage du texte sublime (onglets, barre latérale, palette de commandes, etc.)
  • jeu de couleurs : changez la couleur de votre code
  • syntaxe : définir comment le code doit être mis en surbrillance.

Comment trouver un colis #

Vous pouvez rechercher des packages sur ce site Web : [packagecontrol.io][1], ou simplement rechercher des mots clés lorsque vous décidez que vous devez installer un package (voir ci-dessous).

Une fois que j’ai choisi le package que je veux installer, comment puis-je l’installer ? #

À partir de ST, ouvrez la palette de commandes et tapez “install package” jusqu’à ce que “package control : install package” soit mis en surbrillance, puis appuyez sur Entrée.

Attendez un peu, et une liste avec tous les packages disponibles apparaîtra. Tapez le nom de celui que vous souhaitez installer, et enter. Fait!

Remarque : Vous pouvez redémarrer le texte sublime en toute sécurité après avoir installé un package. Mais cela devient de moins en moins utile (le texte sublime s’améliore, donc vous n’en avez presque plus besoin maintenant). Donc, redémarrez-le simplement si quelque chose est bizarre, pas seulement parce que je l’ai dit.

[1] : https://packagecontrol.io

Installation d’un package non répertorié #

Par package non répertorié, j’entends un package qui n’est pas (encore) disponible via Package Control. Donc, vous ne pouvez pas le trouver dans [packagecontrol.io][].

MAIS, vous pouvez toujours l’installer à l’aide de Package Control, vous bénéficierez donc de tous les avantages. Par exemple, ils seront automatiquement mis à jour, tout comme une installation “normale”.

Et c’est vraiment facile.

  • Trouvez votre référentiel de packages (il sera souvent sur GitHub, mais vous pouvez en utiliser un BitBucket) et copiez l’URL.
  • Recherchez Package Control : Ajouter un référentiel dans la palette de commandes (ctrl+shift+p)
  • Collez l’URL
  • Appuyez sur Entrée

Et maintenant, c’est comme si ce package était sur le canal par défaut (un canal est une simple liste de packages. Le canal par défaut est celui que vous obtenez… par défaut 😃).

  • Recherchez Package Control : Install Package dans la palette de commandes (ctrl+shift+p)
  • (vous devrez peut-être attendre quelques secondes) Recherchez le package que vous souhaitez installer
  • Appuyez sur enter !

C’est ça! Plutôt cool, hein ? Donc, si vous êtes un développeur de packages, n’hésitez pas à leur faire savoir qu’ils peuvent le faire de cette façon (toujours mieux que le git clone et git pull 😉).

Note : N’hésitez pas à copier/coller ce texte dans votre readme, ou à adapter votre propre version !

[packagecontrol.io] : https://packagecontrol.io
