Chaque fois que votre application rencontre une erreur, Heroku renverra une page d’erreur standard avec le code d’état HTTP 503. Cependant, pour vous aider à déboguer l’erreur sous-jacente, la plateforme ajoutera également des informations d’erreur personnalisées à vos journaux. Chaque type d’erreur reçoit son propre code d’erreur, toutes les erreurs HTTP commençant par la lettre H et toutes les erreurs d’exécution commençant par R. Les erreurs de journalisation commencent par L.

H13 - Connexion fermée sans réponse #

Cette erreur est générée lorsqu’un processus de votre dyno Web accepte une connexion, mais ferme ensuite le socket sans rien y écrire.

2010-10-06T21:51:37-07:00 heroku[router]: at=error code=H13 desc="Connection closed without response" method=GET path="/" host=myapp.herokuapp.com fwd=17.17.17.17 dyno=web.1 connect=3030ms service=9767ms status=503 bytes=0

Un exemple où cela peut se produire est lorsqu’un serveur Web Unicorn est configuré avec un délai d’expiration inférieur à 30 secondes et qu’une demande n’a pas été traitée par un travailleur avant que le délai d’expiration ne se produise. Dans ce cas, Unicorn ferme la connexion avant que des données ne soient écrites, ce qui entraîne un H13.

H10 - L’application a planté #

Un dyno Web en panne ou un délai d’attente de démarrage sur le dyno Web présentera cette erreur.

2010-10-06T21:51:04-07:00 heroku[web.1]: State changed from down to starting
2010-10-06T21:51:07-07:00 app[web.1]: Starting process with command: `bundle exec rails server -p 22020`
2010-10-06T21:51:09-07:00 app[web.1]: >> Using rails adapter
2010-10-06T21:51:09-07:00 app[web.1]: Missing the Rails 2.3.5 gem. Please `gem install -v=2.3.5 rails`, update your RAILS_GEM_VERSION setting in config/environment.rb for the Rails version you do have installed, or comment out RAILS_GEM_VERSION to use the latest version installed.
2010-10-06T21:51:10-07:00 heroku[web.1]: Process exited
2010-10-06T21:51:12-07:00 heroku[router]: at=error code=H10 desc="App crashed" method=GET path="/" host=myapp.herokuapp.com fwd=17.17.17.17 dyno= connect= service= status=503 bytes=

H11 - Carnet de commandes trop important #

Lorsque les requêtes HTTP arrivent plus vite que votre application ne peut les traiter, elles peuvent constituer un arriéré important sur un certain nombre de routeurs. Lorsque le backlog sur un routeur particulier dépasse un seuil, le routeur détermine que votre application ne suit pas son volume de requêtes entrantes. Vous verrez une erreur H11 pour chaque demande entrante tant que le backlog dépasse cette taille. La valeur exacte de ce seuil peut changer en fonction de divers facteurs, tels que le nombre de dynos dans votre application, le temps de réponse pour les demandes individuelles et le volume normal de demandes de votre application.

2010-10-06T21:51:07-07:00 heroku[router]: at=error code=H11 desc="Backlog too deep" method=GET path="/" host=myapp.herokuapp.com fwd=17.17.17.17 dyno= connect= service= status=503 bytes=

La solution consiste à augmenter le débit de votre application en ajoutant plus de dynos, en ajustant votre base de données (par exemple, en ajoutant un index) ou en rendant le code lui-même plus rapide. Comme toujours, l’augmentation des performances est très spécifique à l’application et nécessite un profilage.

H12 - Délai d’expiration de la demande #

Une requête HTTP a pris plus de 30 secondes pour se terminer. Dans l’exemple ci-dessous, une application Rails prend 37 secondes pour rendre la page ; le routeur HTTP renvoie un 503 avant que Rails ne termine son cycle de requête, mais le processus Rails continue et le message d’achèvement s’affiche après le message du routeur.

2010-10-06T21:51:07-07:00 app[web.2]: Processing PostController#list (for 75.36.147.245 at 2010-10-06 21:51:07) [GET]
2010-10-06T21:51:08-07:00 app[web.2]: Rendering template within layouts/application
2010-10-06T21:51:19-07:00 app[web.2]: Rendering post/list
2010-10-06T21:51:37-07:00 heroku[router]: at=error code=H12 desc="Request timeout" method=GET path="/" host=myapp.herokuapp.com fwd=17.17.17.17 dyno=web.1 connect=6ms service=30001ms status=503 bytes=0
2010-10-06T21:51:42-07:00 app[web.2]: Completed in 37000ms (View: 27, DB: 21) | 200 OK [http://myapp.heroku.com/]

Cette limite de 30 secondes est mesurée par le routeur et inclut tout le temps passé dans le dyno, y compris la file d’attente de connexion entrante du noyau et l’application elle-même.

H14 - Aucun dynos Web en cours d’exécution #

Ceci est probablement le résultat de la mise à l’échelle de vos dynos Web jusqu’à 0 dynos. Pour résoudre ce problème, redimensionnez vos dynos Web à 1 ou plusieurs dynos :

$ heroku ps:scale web=1

Utilisez la commande heroku ps pour déterminer l’état de vos dynos Web.

2010-10-06T21:51:37-07:00 heroku[router]: at=error code=H14 desc="No web processes running" method=GET path="/" host=myapp.herokuapp.com fwd=17.17.17.17 dyno= connect= service= status=503 bytes=

H15 - Connexion inactive #

Le dyno n’a pas envoyé de réponse complète et a été arrêté en raison de 55 secondes d’inactivité. Par exemple, la réponse indiquait un Content-Length de 50 octets qui n’ont pas été envoyés à temps.

2010-10-06T21:51:37-07:00 heroku[router]: at=error code=H15 desc="Idle connection" method=GET path="/" host=myapp.herokuapp.com fwd=17.17.17.17 dyno=web.1 connect=1ms service=55449ms status=503 bytes=18
