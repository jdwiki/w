Disposition de la grille

Lorsqu’une instance de GridLayout est définie sur un Composite (ou une sous-classe de Composite), tous les contrôles enfants seront disposés dans un modèle de grille. Lorsqu’il y a plusieurs colonnes, la grille est remplie de gauche à droite et de haut en bas.

En plus de spécifier le nombre de colonnes, vous pouvez éventuellement ajuster les marges autour de la grille, ainsi que l’espacement entre les cellules de la grille via diverses variables membres.

Syntaxe #

  • Disposition de la grille()
  • GridLayout(int numColumns, boolean makeColumnsEqualWidth)

Paramètres #

Paramètre Détails
numColonnes Le nombre de colonnes dans la grille
makeColumnsEqualWidth Si oui ou non les colonnes de la mise en page doivent avoir la même largeur

Lors de l’utilisation d’un GridLayout sur un Composite (ou une sous-classe de Composite), les contrôles enfants doivent définir leurs données de mise en page sur une instance unique de [GridData](http://help.eclipse .org/kepler/index.jsp?topic=%2Forg.eclipse.platform.doc.isv%2Freference%2Fapi%2Forg%2Feclipse%2Fswt%2Flayout%2FGridData.html) pour dicter comment l’enfant doit être affiché dans le parent :

// ...
final Shell shell = new Shell(display);
shell.setLayout(new GridLayout());

final Composite child = new Composite(shell, SWT.NONE);
child.setLayoutData(new GridData(SWT.FILL, SWT.FILL, true, true));
// ...

Si vous ne définissez pas un objet GridData sur les contrôles enfants, l’enfant peut ne pas être positionné comme souhaité. De plus, si l’enfant est un Composite (ou une sous-classe de Composite), l’enfant et ses enfants peuvent ne pas être visibles du tout.

Si les mauvaises données de mise en page sont utilisées (par exemple, FormData au lieu de GridData), le résultat sera une ClassCastException à l’exécution :

Exception dans le fil “principal” java.lang.ClassCastException : org.eclipse.swt.layout.FormData ne peut pas être converti en org.eclipse.swt.layout.GridData

Seule colonne #

Dans cet exemple, nous utilisons le constructeur [GridLayout()][1] par défaut sans arguments pour créer une mise en page avec une seule colonne.

public class SingleColumnGridLayoutExample {

    private final Display display;
    private final Shell shell;

    public SingleColumnGridLayoutExample() {
        display = new Display();
        shell = new Shell(display);

        // Create the layout and apply to the Shell
        shell.setLayout(new GridLayout());

        // Add the child controls to the Shell - in this case, in a single column
        final Button buttonA = new Button(shell, SWT.PUSH);
        buttonA.setLayoutData(new GridData(SWT.FILL, SWT.FILL, true, true));
        buttonA.setText("Button A");

        final Button buttonB = new Button(shell, SWT.PUSH);
        buttonB.setLayoutData(new GridData(SWT.FILL, SWT.FILL, true, true));
        buttonB.setText("Button B");

        final Button buttonC = new Button(shell, SWT.PUSH);
        buttonC.setLayoutData(new GridData(SWT.FILL, SWT.FILL, true, true));
        buttonC.setText("Button C");

        final Button buttonD = new Button(shell, SWT.PUSH);
        buttonD.setLayoutData(new GridData(SWT.FILL, SWT.FILL, true, true));
        buttonD.setText("Button D");
    }

    public void run() {
        shell.pack();
        shell.open();

        while (!shell.isDisposed()) {
            if (!display.readAndDispatch()) {
                display.sleep();
            }
        }
        display.dispose();
    }

    public static void main(final String... args) {
        new SingleColumnGridLayoutExample().run();
    }

}

Multi-colonne #

Semblable à l’exemple de colonne unique ci-dessus, si nous utilisons à la place le constructeur [GridLayout(int, boolean)][1], nous pouvons créer une mise en page avec plusieurs colonnes.

Dans ce cas, nous créons deux colonnes, chacune ayant la même largeur.

public class MultiColumnGridLayoutExample {

    private final Display display;
    private final Shell shell;

    public MultiColumnGridLayoutExample() {
        display = new Display();
        shell = new Shell(display);

        // Create the layout and apply to the Shell
        shell.setLayout(new GridLayout(2, true));

        // Add the child controls to the Shell - in this case, in two columns
        final Button buttonA = new Button(shell, SWT.PUSH);
        buttonA.setLayoutData(new GridData(SWT.FILL, SWT.FILL, true, true));
        buttonA.setText("Button A");
        
        final Button buttonB = new Button(shell, SWT.PUSH);
        buttonB.setLayoutData(new GridData(SWT.FILL, SWT.FILL, true, true));
        buttonB.setText("Button B");
        
        final Button buttonC = new Button(shell, SWT.PUSH);
        buttonC.setLayoutData(new GridData(SWT.FILL, SWT.FILL, true, true));
        buttonC.setText("Button C");
        
        final Button buttonD = new Button(shell, SWT.PUSH);
        buttonD.setLayoutData(new GridData(SWT.FILL, SWT.FILL, true, true));
        buttonD.setText("Button D");
    }

    public void run() {
        shell.pack();
        shell.open();

        while (!shell.isDisposed()) {
            if (!display.readAndDispatch()) {
                display.sleep();
            }
        }
        display.dispose();
    }

    public static void main(final String... args) {
        new MultiColumnGridLayoutExample().run();
    }

}

Espacement et marges non par défaut #

En exploitant certaines des variables membres de l’instance GridLayout, nous pouvons modifier les marges autour de la mise en page et l’espacement entre les cellules. Dans cet exemple, nous définissons les éléments suivants :

  1. verticalSpacing = 0 - Définit l’espacement vertical entre les cellules sur 0px.
  2. horizontalSpacing = 20 - Définit l’espacement horizontal entre les cellules sur 20px.
  3. marginWidth = 10 - Définit les marges gauche et droite de la mise en page sur 10px.

Remarque : Nous ne modifions pas la marginHeight, elle reste donc à la valeur par défaut 5px.

public class GridLayoutExample {

    private final Display display;
    private final Shell shell;

    public GridLayoutExample() {
        display = new Display();
        shell = new Shell(display);

        // Create a layout with two columns of equal width
        final GridLayout shellLayout = new GridLayout(2, true);
        shellLayout.verticalSpacing = 0; // Vertical spacing between cells
        shellLayout.horizontalSpacing = 20; // Horizontal spacing between cells
        shellLayout.marginWidth = 10; // Horizontal margin around the layout
        shell.setLayout(shellLayout);

        final Button buttonA = new Button(shell, SWT.PUSH);
        buttonA.setLayoutData(new GridData(SWT.FILL, SWT.FILL, true, true));
        buttonA.setText("Button A");

        final Button buttonB = new Button(shell, SWT.PUSH);
        buttonB.setLayoutData(new GridData(SWT.FILL, SWT.FILL, true, true));
        buttonB.setText("Button B");

        final Button buttonC = new Button(shell, SWT.PUSH);
        buttonC.setLayoutData(new GridData(SWT.FILL, SWT.FILL, true, true));
        buttonC.setText("Button C");

        final Button buttonD = new Button(shell, SWT.PUSH);
        buttonD.setLayoutData(new GridData(SWT.FILL, SWT.FILL, true, true));
        buttonD.setText("Button D");
    }

    public void run() {
        shell.pack();
        shell.open();

        while (!shell.isDisposed()) {
            if (!display.readAndDispatch()) {
                display.sleep();
            }
        }
        display.dispose();
    }

    public static void main(final String... args) {
        new GridLayoutExample().run();
    }

}
