Bot Framework prend en charge les “Recognizers”. Un module de reconnaissance est utilisé pour reconnaître ce qu’il faut faire chaque fois qu’un utilisateur envoie un message au bot. Par conséquent, vous pouvez concevoir votre bot pour qu’il reconnaisse les intentions en fonction de l’entrée de l’utilisateur. Le module de reconnaissance peut être utilisé avec l’API LUIS afin d’ajouter une compréhension du langage naturel pour le bot.

Syntaxe #

  • var recognizer = new builder.LUISRecognizer(‘L’URL de votre modèle’);

  • var intents = new builder.IntentDialog({recognizers : [recognizer]});

Initialisation et ajout de LUISRecognizer #

Une fois que vous avez créé un nouveau projet avec le modèle de base fourni dans l’introduction, vous devriez pouvoir ajouter un LUISRecognizer comme ceci -

var model = ''    // Your LUIS Endpoint link comes here
var recognizer = new builder.LuisRecognizer(model);

Désormais, recognizer est un LUISRecognizer et peut transmettre des intentions en fonction de votre modèle LUIS défini. Vous pouvez ajouter le “recognizer” à vos intentions en

var intents = new builder.IntentDialog({recognizers: [recognizer]});

Votre bot est désormais capable de gérer les intentions de LUIS. Toutes les intentions nommées sur LUIS peuvent être détectées en utilisant la propriété matches de la classe IntentDialog. Par exemple, une intention nommée “hi” est définie dans le modèle LUIS, pour reconnaître l’intention sur le bot,

intents.matches('hi', function(session) {
    session.send("Hey :-)");
});

Définir un modèle LUIS avec des intentions #

La création d’un modèle LUIS nécessite peu ou pas d’expérience en programmation. Cependant, vous devez vous familiariser avec 2 termes importants qui seront largement utilisés.

  1. Intents - C’est ainsi que vous identifiez les fonctions qui doivent être exécutées lorsque l’utilisateur tape quelque chose. Par exemple - Une intention nommée Hi identifiera une fonction qui doit être exécutée chaque fois que l’utilisateur envoie “Hi”. Les intentions sont nommées de manière unique dans votre programme/modèle.
  2. Entités - Celles-ci identifient les noms dans une déclaration. Par exemple - “Définir une alarme pour 13h00”, ici 13h00 est une entité qui doit être reconnue par le chat-bot pour définir une alarme.

Remarque : Les images du site Web ne sont pas fournies en tant que mon changement frontal, mais le concept de base reste le même.

Pour créer un nouveau modèle, accédez à LUIS.ai et connectez-vous avec votre compte Microsoft pour accéder à la page de création d’application. Où un projet vierge peut être créé.

Définir les intentions :

Les intentions peuvent être définies dans l’onglet “Intents”. Ils identifient la fonction que vous devez exécuter lorsque l’utilisateur saisit quoi que ce soit.

Toutes les applications ont une intention “Aucune” par défaut, qui est activée chaque fois que l’entrée de l’utilisateur ne correspond à aucune autre intention.

Pour définir une intention,

  1. Donnez-lui un nom unique correspondant à la fonction que vous souhaitez exécuter.
  2. Une fois la dénomination terminée, vous devez ajouter des “énoncés” à l’intention. Les énoncés sont ce que vous voulez que l’utilisateur envoie afin d’activer l’intention que vous définissez. Essayez de fournir autant d’“énoncés” différents que possible afin que LUIS associe correctement les “intentions” et les “énoncés”.
  3. Entraînez votre modèle LUIS en cliquant sur le bouton “Entraîner” dans l’onglet “Entraîner et tester”. Après la formation, l’application peut être testée dans le panneau ci-dessous.
  4. Enfin, publiez votre application dans l’onglet “Publier l’application”. Vous devriez maintenant obtenir une URL de point de terminaison qui doit être insérée lors de la définition de “LUISRecognizer” dans votre code de bot.

Ajout d’entités au modèle LUIS #

Une entité est l’information que votre bot extrait d’un énoncé particulier conformément à une intention.

Par exemple, laissez “Mon nom est John Doe” appartenir à une intention appelée “introduction”. Pour que votre bot comprenne et extraie le nom “John Doe” de la phrase, vous devez définir une entité qui le fait. Vous pouvez nommer l’entité comme vous le souhaitez, mais il est préférable de la nommer comme quelque chose se rapportant à ce qu’elle extrait. Dans notre exemple, nous pouvons appeler notre entité name.

Les entités peuvent être réutilisées entre différentes intentions, pour extraire différentes choses. Le meilleur principe serait donc de créer une entité qui extrait uniquement le type de données et de l’utiliser dans différentes intentions. Par conséquent, dans notre exemple ci-dessus, dire “Réserver un vol sur Emirates” appartient à l’intention “réservation”, alors la même entité, “nom”, peut être utilisée pour extraire le nom du vol “emirates”.

Vous devez garder à l’esprit deux choses avant de continuer à définir des entités -

  1. Les entités doivent être uniques par énoncé dans une intention. Une entité ne peut pas être utilisée deux fois dans le même énoncé.
  2. LUIS est insensible à la casse. Cela implique que tout ce qui est extrait et reçu via l’extraction d’entités sera en minuscules. Donc, extraire des données sensibles à la casse via des entités est probablement une mauvaise idée.

Ajout d’entités prédéfinies

Les entités pré-construites sont, comme leur nom l’indique, pré-construites, c’est-à-dire qu’elles sont déjà configurées pour extraire un type particulier de données dans l’intention à laquelle elles sont ajoutées. Un exemple peut être l’entité “nombre” qui extrait les nombres de l’intention à laquelle elle est affectée. Les nombres peuvent être numériques ou alphabétiques comme ‘10’ ou ‘dix’.

Pour une liste complète de toutes les entités prédéfinies, vous pouvez visiter [Liste des entités prédéfinies] [1].

Pour ajouter des entités prédéfinies,

  1. Allez dans l’onglet “entités”.
  2. Cliquez sur “Ajouter des entités prédéfinies” et sélectionnez l’entité que vous souhaitez ajouter au modèle et appuyez sur Enregistrer.

Ajout d’entités personnalisées Les entités personnalisées sont de 4 types, 1. Simple : une entité simple extrait une donnée particulière, “nom” dans les exemples ci-dessus est une entité simple. 2. Hiérarchique : Une entité parent avec des entités enfants (sous-types) qui dépendent du parent. 3. Composite : Un groupe de 2 ou plusieurs entités indépendantes ensemble. 4. Liste : une entité qui ne reconnaît que les mots d’une liste donnée.

Définition d’entités simples

  1. Allez dans l’onglet “entités”.
  2. Cliquez sur “Ajouter des entités personnalisées”
  3. Nommez votre entité, cochez le type d’entité requis et cliquez sur “Enregistrer”.

Tous les autres types d’entités peuvent être ajoutés de la même manière en changeant simplement le “Type d’entité” en l’un des types ci-dessus. Dans les types d’entités hiérarchiques et composites, vous devrez également donner des noms aux enfants avec le nom de l’entité parent. La définition des entités List est un peu différente du reste.

Définition des entités de liste

Après avoir suivi les étapes ci-dessus pour créer une List Entity en définissant le Entity Type comme List, vous serez dirigé vers la page de détails de l’entité que vous venez de définir.

  1. Définissez une valeur canonique. Il s’agit d’une valeur standard que le bot recevra lorsque l’utilisateur saisira l’un des synonymes.
  2. Définissez des synonymes à la valeur canonique. Ils seront convertis à la valeur canonique lorsqu’ils seront rencontrés par l’entité.

Vous pouvez également importer des listes entières en utilisant un tableau d’objets JSON, de la forme :

[
    {
        "canonicalForm": "Hey",
        "list": [
            "Howdy",
            "Hi"
         ]
    },
    .
    .
    .
]

Associer une entité à une intention

Les entités “pré-construites” et “liste” ont déjà un ensemble de valeurs définies qui peuvent être extraites de tous les énoncés, cependant, les énoncés “simples”, “hiérarchiques” et “composites” doivent être formés pour récupérer des valeurs.

Cela peut être fait par

  1. Accédez à l’onglet “Intents” et choisissez l’intention à laquelle vous souhaitez ajouter l’entité.
  2. Ajoutez un énoncé avec une valeur fictive que vous souhaitez extraire. Supposons que vous pouvez ajouter “Je m’appelle John Doe” comme énoncé.
  3. Cliquez et faites glisser la souris sur les mots que vous souhaitez extraire de l’entité. Vous devrez mettre en surbrillance john doe dans l’exemple ci-dessus.
  4. Une liste déroulante s’ouvrira avec une liste de toutes les entités disponibles dans votre projet. Sélectionnez celui qui vous convient comme bon vous semble. Name sera l’entité sélectionnée dans l’exemple ci-dessus.
  5. Ajoutez plus d’énoncés avec différentes valeurs fictives à chaque fois et toutes les structures possibles auxquelles vous pouvez penser.
  6. Entraînez et publiez votre modèle LUIS.
