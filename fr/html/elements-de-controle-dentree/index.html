Élément clé des systèmes Web interactifs, les balises d’entrée sont des éléments HTML conçus pour prendre une forme spécifique d’entrée des utilisateurs. Différents types d’éléments d’entrée peuvent réguler les données saisies pour s’adapter à un format spécifié et assurer la sécurité de la saisie du mot de passe.

Syntaxe #

  • <type d'entrée="" name="" value="">

Paramètres #

Paramètre Détails
classe Indique la classe de l’entrée
identifiant Indique l’ID de l’entrée
taper Identifie le type de contrôle d’entrée à afficher. Les valeurs acceptables sont hidden, text, tel, url, email, password, date, time, number, range, color, checkbox , radio, fichier, soumettre, image, réinitialiser et bouton. Par défaut, text s’il n’est pas spécifié, si la valeur n’est pas valide ou si le navigateur ne prend pas en charge le type spécifié.
nom Indique le nom de l’entrée
désactivé Valeur booléenne indiquant que l’entrée doit être désactivée. Les contrôles désactivés ne peuvent pas être modifiés, ne sont pas envoyés lors de la soumission du formulaire et ne peuvent pas recevoir le focus.
vérifié Lorsque la valeur de l’attribut type est radio ou case à cocher, la présence de cet attribut booléen indique que le champ est sélectionné par défaut ; sinon, il est ignoré.
multiples HTML5 Indique que plusieurs fichiers ou valeurs peuvent être passés (s’applique uniquement aux entrées de type file et email )
espace réservé HTML5 Une indication à l’utilisateur de ce qui peut être saisi dans le contrôle . Le texte de l’espace réservé ne doit pas contenir de retours chariot ni de sauts de ligne
saisie semi-automatique HTML5 Indique si la valeur du champ peut être complétée automatiquement par le navigateur.
lecture seule Valeur booléenne indiquant que l’entrée n’est pas modifiable. Les contrôles en lecture seule sont toujours envoyés lors de la soumission du formulaire, mais ne recevront pas le focus. HTML5 : Cet attribut est ignoré lorsque la valeur de l’attribut type est définie sur hidden, range, color, checkbox, radio, file ou button.
requis HTML5 Indique qu’une valeur doit être présente ou que l’élément doit être coché pour que le formulaire soit soumis
alt Un texte alternatif pour les images, au cas où elles ne seraient pas affichées.
mise au point automatique L’élément <input> devrait avoir le focus lors du chargement de la page.
valeur Spécifie la valeur de l’élément <input>.
étape L’attribut step spécifie les intervalles de nombres légaux. Il fonctionne avec les types d’entrée suivants : number, range, date, datetime-local, month, time et week.

Comme avec les autres éléments vides HTML5, <input> se ferme automatiquement et peut être écrit <input />. HTML5 ne nécessite pas cette barre oblique.

Voici les types d’entrée valides en HTML :

  • bouton
  • case à cocher
  • file
  • caché
  • image
  • [password](https://www.wikiod.com/fr/html/elements-de-controle-dentree#Mot de passe)
  • [radio](https://www.wikiod.com/fr/html/elements-de-controle-dentree#Case à cocher et boutons radio)
  • reset
  • soumettre
  • text (valeur par défaut)

Voici les nouveaux types d’entrée introduits dans le cadre de la norme HTML 5. Certains de ces types ne sont pas pris en charge par tous les navigateurs Web. Dans le cas où un type n’est pas pris en charge, l’élément d’entrée sera par défaut le type text.

  • color
  • date
  • datetime (obsolète et obsolète)
  • datetime-local
  • email
  • mois
  • number
  • range
  • “rechercher”
  • tel
  • time
  • url
  • [week](https://www.wikiod.com/fr/html/elements-de-controle-dentree#La semaine)

Pour vérifier quels navigateurs prennent en charge quels types, vous pouvez aller sur [caniuse.com][c].

[c] : http://caniuse.com/#search=input%20type

Case à cocher et boutons radio #

Aperçu #

Les cases à cocher et les boutons radio sont écrits avec la balise HTML <input>, et leur comportement est défini dans la [spécification HTML][1].

La case à cocher ou le bouton radio le plus simple est un élément <input> avec un attribut type de checkbox ou radio, respectivement :

<input type="checkbox">
<input type="radio">

Un seul élément de case à cocher autonome est utilisé pour une seule option binaire telle qu’une question oui ou non. Les cases à cocher sont indépendantes, ce qui signifie que l’utilisateur peut sélectionner autant de choix qu’il le souhaite dans un groupe de cases à cocher. En d’autres termes, cocher une case ne décoche pas les autres cases du groupe de cases à cocher.

Les boutons radio viennent généralement en groupes (s’ils ne sont pas regroupés avec un autre bouton radio, vous vouliez probablement utiliser une case à cocher à la place) identifiés en utilisant le même attribut name sur tous les boutons de ce groupe. La sélection des boutons radio est mutuellement exclusive, ce qui signifie que l’utilisateur ne peut sélectionner qu’un seul choix parmi un groupe de boutons radio. Lorsqu’un bouton radio est coché, tout autre bouton radio portant le même “nom” qui était précédemment coché devient décoché.

Exemple:

<input type="radio" name="color" id="red" value="#F00">
<input type="radio" name="color" id="green" value="#0F0">
<input type="radio" name="color" id="blue" value="#00F">

Lorsqu’ils sont affichés, les boutons radio apparaissent sous la forme d’un cercle (décoché) ou d’un cercle rempli (coché). Les cases à cocher apparaissent sous la forme d’un carré (non coché) ou d’un carré plein (coché). Selon le navigateur et le système d’exploitation, le carré a parfois des coins arrondis.

Les attributs #

les cases à cocher et les boutons radio ont un certain nombre d’attributs pour contrôler leur comportement :

valeur #

Comme tout autre élément d’entrée, l’attribut value spécifie la valeur de chaîne à associer au bouton en cas de soumission de formulaire. Cependant, les cases à cocher et les boutons radio sont spéciaux en ce sens que lorsque la valeur est omise, elle est par défaut “on” lorsqu’elle est soumise, plutôt que d’envoyer une valeur vide. L’attribut value n’est pas reflété dans l’apparence du bouton.

vérifié #

L’attribut checked spécifie l’état initial d’une case à cocher ou d’un bouton radio. Ceci est un attribut booléen et peut être omis.

Chacune de ces méthodes est valide et équivalente pour définir un bouton radio coché :

<input checked>
<input checked="">
<input checked="checked">
<input checked="ChEcKeD">

L’absence de l’attribut checked est la seule syntaxe valide pour un bouton non coché :

<input type="radio">
<input type="checkbox">

Lors de la réinitialisation d’un <form>, les cases à cocher et les boutons radio reviennent à l’état de leur attribut checked.

Accessibilité #

Étiquettes #

Pour donner un contexte aux boutons et montrer aux utilisateurs à quoi sert chaque bouton, chacun d’eux doit avoir une étiquette. Cela peut être fait en utilisant un élément <label> pour envelopper le bouton. De plus, cela rend l’étiquette cliquable, vous sélectionnez donc le bouton correspondant.

Exemple:

<label>
  <input type="radio" name="color" value="#F00">
  Red
</label>

ou avec un élément <label> avec un attribut for défini sur l’attribut id du bouton :

<input type="checkbox" name="color" value="#F00" id="red">
<label for="red">Red</label>

Groupes de boutons #

Étant donné que chaque bouton radio affecte les autres dans le groupe, il est courant de fournir une étiquette ou un contexte pour l’ensemble du groupe de boutons radio.

Pour fournir une étiquette pour l’ensemble du groupe, les boutons radio doivent être inclus dans un élément <fieldset> avec un élément <legend> à l’intérieur.

Exemple:

<fieldset>
  <legend>Theme color:</legend>
  <p>
    <input type="radio" name="color" id="red" value="#F00">
    <label for="red">Red</label>
  </p>
  <p>
    <input type="radio" name="color" id="green" value="#0F0">
    <label for="green">Green</label>
  </p>
  <p>
    <input type="radio" name="color" id="blue" value="#00F">
    <label for="blue">Blue</label>
  </p>
</fieldset>

Les cases à cocher peuvent également être regroupées de la même manière, avec un ensemble de champs et une légende identifiant le groupe de cases à cocher associées. Cependant, gardez à l’esprit que les cases à cocher ne doivent pas partager le même nom car elles ne s’excluent pas mutuellement. Cela entraînera la soumission par le formulaire de plusieurs valeurs pour la même clé et tous les langages côté serveur ne gèrent pas cela de la même manière (comportement indéfini). Chaque case à cocher doit avoir un nom unique ou utiliser un ensemble de crochets ([]) pour indiquer que le formulaire doit soumettre un tableau de valeurs pour cette clé. La méthode que vous choisissez doit dépendre de la façon dont vous envisagez de gérer les données de formulaire côté client ou côté serveur. Vous devez également garder la légende courte, car certaines combinaisons de navigateurs et de lecteurs d’écran lisent la légende avant chaque champ de saisie dans l’ensemble de champs.

[1] : https://html.spec.whatwg.org/multipage/forms.html#radio-button-state-(type=radio)

Validation des entrées #

La validation de l’entrée HTML est effectuée automatiquement par le navigateur en fonction d’attributs spéciaux sur l’élément d’entrée. Il pourrait remplacer partiellement ou complètement la validation des entrées JavaScript. Ce type de validation peut être contourné par l’utilisateur via des requêtes HTTP spécialement conçues, de sorte qu’il ne remplace pas la validation des entrées côté serveur. La validation ne se produit que lors de la tentative de soumission du formulaire, donc toutes les entrées restreintes doivent être à l’intérieur d’un formulaire pour que la validation se produise (sauf si vous utilisez JavaScript). Gardez à l’esprit que les entrées désactivées ou en lecture seule ne déclencheront pas de validation.

Certains types d’entrée plus récents (comme email, url, tel, date et bien d’autres) sont automatiquement validés et ne nécessitent pas vos propres contraintes de validation.

Obligatoire #

Utilisez l’attribut required pour indiquer qu’un champ doit être rempli afin de réussir la validation.

<input required>

Longueur minimale / maximale #

Utilisez les attributs minlength et maxlength pour indiquer les exigences de longueur. La plupart des navigateurs empêcheront l’utilisateur de saisir plus de caractères max dans la zone, ce qui l’empêchera de rendre sa saisie invalide avant même qu’il ne tente de la soumettre.

<input minlength="3">
<input maxlength="15">
<input minlength="3" maxlength="15">

Spécifier une plage #

Utilisez les attributs min et max pour restreindre la plage de nombres qu’un utilisateur peut saisir dans une entrée de type number ou range

Marks: <input type="number" size="6" name="marks" min="0" max="100" />
Subject Feedback: <input type="range" size="2" name="feedback" min="1" max="5" />

Faire correspondre un motif #

Pour plus de contrôle, utilisez l’attribut pattern pour spécifier toute expression régulière qui doit correspondre pour réussir la validation. Vous pouvez également spécifier un title, qui est inclus dans le message de validation si le champ ne passe pas.

<input pattern="\d*" title="Numbers only, please.">

Voici le message affiché dans Google Chrome version 51 lors de la tentative d’envoi du formulaire avec une valeur non valide dans ce champ :

[![Veuillez correspondre au format demandé. Chiffres uniquement, s’il vous plaît.][1]][1]

Tous les navigateurs n’affichent pas un message pour les modèles non valides, bien qu’il existe une prise en charge complète parmi les navigateurs modernes les plus utilisés.

Vérifiez le dernier support sur [CanIUse][2] et implémentez-le en conséquence.

Accepter le type de fichier #

Pour les champs de saisie de type “fichier”, il est possible d’accepter uniquement certains types de fichiers, tels que des vidéos, des images, des fichiers audio, des extensions de fichiers spécifiques ou certains [types de média][3]. Par exemple:

<input type="file" accept="image/*" title="Only images are allowed">

Plusieurs valeurs peuvent être spécifiées avec une virgule, par exemple :

<input type="file" accept="image/*,.rar,application/zip">

Remarque : L’ajout de l’attribut novalidate à l’élément form ou de l’attribut formnovalidate au bouton d’envoi empêche la validation des éléments de formulaire. Par exemple:

<form>
    <input type="text" name="name" required>
    <input type="email" name="email" required>
    <input pattern="\d*" name="number" required>

    <input type="submit" value="Publish"> <!-- form will be validated -->
    <input type="submit" value="Save" formnovalidate> <!-- form will NOT be validated -->
</form>

Le formulaire contient des champs obligatoires pour “publier” le brouillon, mais pas pour “enregistrer” le brouillon.

[1] : http://i.stack.imgur.com/GoCmo.png [2] : http://caniuse.com/#feat=input-pattern [3] : http://www.iana.org/assignments/media-types/media-types.xhtml

Texte #

Le type d’entrée le plus basique et l’entrée par défaut si aucun “type” n’est spécifié. Ce type d’entrée définit un champ de texte sur une seule ligne avec des sauts de ligne automatiquement supprimés de la valeur d’entrée. Tous les autres caractères peuvent y être saisis. Les éléments <input> sont utilisés dans un élément <form> pour déclarer des contrôles d’entrée qui permettent aux utilisateurs d’entrer des données.

Syntaxe

<input type="text">

ou (sans spécifier de type, en utilisant l’attribut par défaut) :

<input>

La largeur par défaut d’une entrée de champ de texte est de 20 caractères. Cela peut être changé en spécifiant une valeur pour l’attribut size comme ceci :

<input type="text" size="50">

L’attribut size est nettement différent de la définition d’une largeur avec CSS. L’utilisation d’une largeur définit une valeur spécifique (en nombre de pixels, pourcentage de l’élément parent, etc.) que l’entrée doit toujours être large. L’utilisation de size calcule la quantité de largeur à allouer en fonction de la police utilisée et de la largeur normale des caractères.

Remarque : L’utilisation de l’attribut “taille” ne limite pas de manière inhérente le nombre de caractères pouvant être saisis dans la zone, mais uniquement la largeur d’affichage de la zone. Pour limiter la longueur, voir [Input Validation](https://www.wikiod.com/fr/html/elements-de-controle-dentree#Validation des entrées).

Un champ de saisie n’autorise qu’une seule ligne de texte. Si vous avez besoin d’une entrée de texte multiligne pour une quantité importante de texte, utilisez un élément <textarea> à la place.

Couleur #

<input type="color" name="favcolor" value="#ff0000">

Dans les navigateurs compatibles, l’élément d’entrée avec un attribut de type dont la valeur est color crée un contrôle semblable à un bouton, avec une couleur égale à la valeur de l’attribut color (par défaut, noir si la valeur n’est pas spécifiée ou est un hexadécimal invalide format).

[![entrez la description de l’image ici][1]][1]

Cliquer sur ce bouton ouvre le widget de couleur du système d’exploitation, qui permet à l’utilisateur de sélectionner une couleur.

[![entrez la description de l’image ici][2]][2]

La solution de rechange pour les navigateurs qui ne prennent pas en charge ce type d’entrée est un [input type=text][3] normal.

[![entrez la description de l’image ici][4]][4]

[1] : http://i.stack.imgur.com/aIvlb.png [2] : http://i.stack.imgur.com/cCZxo.jpg [3] : https://www.wikiod.com/fr/html/elements-de-controle-dentree#Texte [4] : http://i.stack.imgur.com/togs5.png [5] : http://caniuse.com/#search=color

Mot de passe #

<input type="password" name="password">

L’élément d’entrée avec un attribut de type dont la valeur est password crée un champ de texte sur une seule ligne similaire à [input type=text][1], sauf que le texte n’est pas affiché lorsque l’utilisateur le saisit.

<input type="password" name="password" placeholder="Password">

Le texte de l’espace réservé s’affiche en texte brut et est automatiquement écrasé lorsqu’un utilisateur commence à taper.

[![Exemple de champ de saisie du mot de passe][2]][2]

Remarque : Certains navigateurs et systèmes modifient le comportement par défaut du champ de mot de passe pour afficher également le dernier caractère saisi pendant une courte durée, comme ceci :

[![Exemple du dernier caractère affiché][3]][3]

Dossier #

<input type="file" name="fileSubmission">

Les entrées de fichier permettent aux utilisateurs de sélectionner un fichier à partir de leur système de fichiers local pour l’utiliser avec la page en cours. S’ils sont utilisés en conjonction avec un élément form, ils peuvent être utilisés pour permettre aux utilisateurs de télécharger des fichiers sur un serveur (pour plus d’informations, voir [Télécharger des fichiers] [1]).

L’exemple suivant permet aux utilisateurs d’utiliser l’entrée file pour sélectionner un fichier dans leur système de fichiers et de télécharger ce fichier dans un script sur le serveur nommé upload_file.php.

<form action="upload_file.php" method="post" enctype="multipart/form-data">
    Select file to upload:
    <input type="file" name="fileSubmission" id="fileSubmission">
    <input type="submit" value="Upload your file" name="submit">
</form>

Plusieurs fichiers

En ajoutant l’attribut multiple, l’utilisateur pourra sélectionner plus d’un fichier :

<input type="file" name="fileSubmission" id="fileSubmission" multiple>

Accepter les fichiers

L’attribut Accept spécifie les types de fichiers que l’utilisateur peut sélectionner. Par exemple. .png, .gif, .jpeg.

<input type="file" name="fileSubmission" accept="image/x-png,image/gif,image/jpeg" />

Soumettre #

<input type="submit" value="Submit">

Une entrée de soumission crée un bouton qui soumet le formulaire dans lequel il se trouve lorsqu’il est cliqué.

Vous pouvez également utiliser l’élément <button> si vous avez besoin d’un bouton d’envoi qui peut être plus facilement stylé ou contenir d’autres éléments :

<button type="submit">
  <img src="submit-icon.jpg" /> Submit
</button>

Réinitialiser #

<input type="reset" value="Reset">

Une entrée de type reset crée un bouton qui, lorsqu’il est cliqué, réinitialise toutes les entrées du formulaire dans lequel il se trouve à leur état par défaut.

  • Le texte dans un champ de saisie sera réinitialisé à vide ou à sa valeur par défaut (spécifiée à l’aide de l’attribut value).
  • Toutes les options d’un menu de sélection seront désélectionnées à moins qu’elles n’aient l’attribut selected.
  • Toutes les cases à cocher et les cases d’option seront désélectionnées à moins qu’elles n’aient l’attribut “coché”.

Remarque : Un bouton de réinitialisation doit être à l’intérieur ou attaché à (via l’attribut form) un élément <form> pour avoir un effet. Le bouton ne réinitialisera que les éléments de ce formulaire.

Bouton #

<input type="button" value="Button Text">

Les boutons peuvent être utilisés pour déclencher des actions sur la page, sans soumettre le formulaire. Vous pouvez également utiliser l’élément <bouton> si vous avez besoin d’un bouton qui peut être stylé plus facilement ou qui contient d’autres éléments :

<button type="button">Button Text</button>

Les boutons sont généralement utilisés avec un événement “onclick”:

<input type="button" onclick="alert('hello world!')" value="Click Me">

ou

<button type="button" onclick="alert('hello world!')">Click Me</button>

Les attributs #

[nom] #

Le “nom” du bouton, qui est soumis avec les données du formulaire.

[type] #

Le “type” du bouton.

Les valeurs possibles sont :

submit : le bouton soumet les données du formulaire au serveur. Il s’agit de la valeur par défaut si l’attribut n’est pas spécifié ou si l’attribut est modifié dynamiquement en une valeur vide ou invalide.

reset : Le bouton réinitialise tous les contrôles à leurs valeurs initiales.

button : Le bouton n’a pas de comportement par défaut. Il peut avoir des scripts côté client associés aux événements de l’élément, qui sont déclenchés lorsque les événements se produisent.

menu : Le bouton ouvre un menu contextuel défini via son élément

désigné.

[valeur] #

La valeur initiale du bouton.

## Attributs supplémentaires pour les boutons d’envoi

Attribut La description
formulaire Spécifie l’ID du formulaire auquel appartient le bouton.
Si aucun n’est spécifié, il appartiendra à son élément de formulaire ancêtre (s’il existe).
formaction Spécifie où envoyer les données du formulaire
lorsque le formulaire est soumis à l’aide de ce bouton.
formenctype Spécifie comment les données du formulaire doivent être encodées
lors de leur soumission au serveur à l’aide de ce bouton.
Ne peut être utilisé qu’avec formmethod="post".
formmethod Spécifie la méthode HTTP à utiliser (POST ou GET)
lors de l’envoi de données de formulaire à l’aide de ce bouton.
formnovalidate Spécifie que les données du formulaire ne doivent pas être validées lors de la soumission.
formtarget Indique où afficher la réponse reçue
après avoir soumis le formulaire à l’aide de ce bouton.

Caché #

<input type="hidden" name="inputName" value="inputValue">

Une entrée masquée ne sera pas visible pour l’utilisateur, mais sa valeur sera néanmoins envoyée au serveur lorsque le formulaire sera soumis.

Tél #

<input type="tel" value="+8400000000">

L’élément d’entrée avec un attribut de type dont la valeur est “tel” représente un contrôle d’édition en texte brut d’une ligne pour entrer un numéro de téléphone.

Numéro #

<input type="number" value="0" name="quantity">

L’élément Input avec un attribut de type dont la valeur est number représente un contrôle précis pour définir la valeur de l’élément sur une chaîne représentant un nombre.

Veuillez noter que ce champ ne garantit pas d’avoir un numéro correct. Il permet simplement tous les symboles qui pourraient être utilisés dans n’importe quel nombre réel, par exemple l’utilisateur pourra entrer une valeur comme e1e-,0.

E-mail #

Le <input type="email"> est utilisé pour les champs de saisie qui doivent contenir une adresse e-mail.

<form>
  <label>E-mail: <label>
  <input type="email" name="email">
</form>

L’adresse e-mail peut être automatiquement validée lorsqu’elle est soumise en fonction de la prise en charge du navigateur.

Intervalle #

<input type="range" min="" max="" step="" />

Un contrôle pour saisir un nombre dont la valeur exacte n’a pas d’importance.

Attribut Descriptif Valeur par défaut
minute Valeur minimale pour la plage 0
maximum Valeur maximale pour la plage 100
étape Montant à augmenter à chaque incrément. 1

Image #

<input type="image" src="img.png" alt="image_name" height="50px" width="50px"/>

Une image. Vous devez utiliser l’attribut src pour définir la source de l’image et l’attribut alt pour définir un texte alternatif. Vous pouvez utiliser les attributs height et width pour définir la taille de l’image en pixels.

La semaine #

<input type="week" />

En fonction de la prise en charge du navigateur, un contrôle s’affichera pour entrer un numéro de semaine-année et un numéro de semaine sans fuseau horaire.

Chercher #

La recherche par type d’entrée est utilisée pour la recherche textuelle. Il ajoutera un symbole de loupe à côté de l’espace pour le texte sur la plupart des navigateurs

<input type="search" name="googlesearch">

URL #

<input type="url" name="Homepage">

Ceci est utilisé pour les champs de saisie qui doivent contenir une adresse URL.

Selon la prise en charge du navigateur, le champ url peut être automatiquement validé lorsqu’il est soumis.

Certains smartphones reconnaissent le type “url” et ajoutent “.com” au clavier pour correspondre à la saisie de l’url.

DateHeure-Local #

<input type="datetime-local" />

En fonction de la prise en charge du navigateur, un sélecteur de date et d’heure apparaîtra à l’écran pour vous permettre de choisir une date et une heure.

Mois #

<input type="month" />

En fonction de la prise en charge du navigateur, un contrôle s’affichera pour choisir le mois.

Temps #

<input type="time" />

L’entrée time marque cet élément comme acceptant une chaîne représentant une heure. Le format est défini dans la [RFC 3339][1] et doit être un temps partiel tel que

19:04:39
08:20:39.04

Actuellement, toutes les versions d’Edge, Chrome, Opera et Chrome pour Android prennent en charge type=“time”. Les nouvelles versions du navigateur Android, en particulier 4.4 et plus, le prennent en charge. Safari pour iOS offre une prise en charge partielle, ne prenant pas en charge les attributs min, max et step.

[1] : https://tools.ietf.org/html/rfc3339

Date #

<input type="date" />

Un sélecteur de date apparaîtra à l’écran pour vous permettre de choisir une date. Ceci n’est pas pris en charge dans Firefox ou Internet Explorer.

DateHeure (Global) #

L’élément d’entrée avec un attribut de type dont la valeur est “datetime” représente un contrôle pour définir la valeur de l’élément sur une chaîne représentant une date et heure globales (avec des informations de fuseau horaire).

<fieldset>
   <p><label>Meeting time: <input type=datetime name="meeting.start"></label>
</fieldset>

  • désactivé

  • taper
  • Saisie automatique
