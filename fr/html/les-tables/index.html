L’élément HTML <table> permet aux auteurs Web d’afficher des données tabulaires (telles que du texte, des images, des liens, d’autres tableaux, etc.) dans un tableau à deux dimensions avec des lignes et des colonnes de cellules.

Syntaxe #

  • <tableau></table>
  • <thead></thead>
  • <tbody></tbody>
  • <tfoot></tfoot>
  • <tr></tr>
  • <th></th>
  • <td></td>

Les différents éléments de table et leurs attributs de contenu définissent ensemble le modèle de table. L’élément <table> est l’élément conteneur pour les modèles de table/données tabulaires. Les tableaux ont des lignes, des colonnes et des cellules données par leurs descendants. Les lignes et les colonnes forment une grille ; les cellules d’un tableau doivent couvrir complètement cette grille sans se chevaucher. La liste ci-dessous décrit les différents éléments du modèle de table :

  • <table> - L’élément conteneur pour les modèles de table/données tabulaires. <table> représente des données à plusieurs dimensions sous la forme d’un tableau.
  • <caption> - Légende ou titre du tableau (comme une figcaption à une figure)
  • <col> - Une colonne (un élément sans contenu)
  • <colgroup> - Un regroupement de colonnes
  • <thead> - En-tête de tableau (un seul)
  • <tbody> - Corps du tableau / contenu (plusieurs sont acceptables)
  • <tfoot> - Pied de page du tableau (un seul)
  • <tr> - Ligne du tableau
  • <th> - Cellule d’en-tête de tableau
  • <td> - Cellule de données du tableau

Sémantiquement, les tables sont destinées à contenir des données tabulaires. Vous pouvez y voir un moyen d’afficher et de décrire des données qui auraient du sens dans une feuille de calcul (colonnes et lignes).

L’utilisation de tableaux pour la mise en page n’est pas recommandée. Utilisez plutôt les règles CSS pour la mise en page et le formatage, y compris display: table.

Une exception notable généralement affichée dans l’industrie concernant l’utilisation de la mise en page <table> concerne le courrier électronique HTML : certains clients de messagerie, y compris Outlook, sont revenus à des moteurs de rendu plus anciens après que Microsoft a perdu son monopole contre l’UE. Pour que Microsoft ne fasse pas d’IE une partie du système d’exploitation, ils ont simplement restauré le moteur de rendu d’Outlook vers une version antérieure de Trident. Cette restauration ne prend tout simplement pas en charge les technologies Web modernes. Par conséquent, l’utilisation de mises en page basées sur <table> pour les e-mails HTML est le seul moyen d’assurer la compatibilité entre navigateurs/plates-formes/clients.

## Tableau simple

Heading 1/Column 1 Heading 2/Column 2
Row 1 Data Column 1 Row 1 Data Column 2
Row 2 Data Column 1 Row 2 Data Column 2

Cela rendra un <table> composé de trois lignes au total (<tr>) : une ligne de cellules d’en-tête (<th>) et deux lignes de cellules de contenu (<td>). Les éléments <th> sont des en-têtes tabulaires et les éléments <td> sont des données tabulaires. Vous pouvez mettre ce que vous voulez dans un <td> ou <th>.

Heading 1/Column 1 Heading 2/Column 2 Row 1 Data Column 1 Row 1 Data Column 2 Row 2 Data Column 1 Row 2 Data Column 2

Étendre des colonnes ou des lignes #

Les cellules du tableau peuvent s’étendre sur plusieurs colonnes ou lignes à l’aide des attributs “colspan” et “rowspan”. Ces attributs peuvent être appliqués aux éléments <th> et <td>.

<table>
    <tr>
        <td>row 1 col 1</td>
        <td>row 1 col 2</td>
        <td>row 1 col 3</td>
    </tr>
    <tr>
        <td colspan="3">This second row spans all three columns</td>
    </tr>
    <tr>
        <td rowspan="2">This cell spans two rows</td>
        <td>row 3 col 2</td>
        <td>row 3 col 3</td>
    </tr>
    <tr>
        <td>row 4 col 2</td>
        <td>row 4 col 3</td>
    </tr>

</table>

Aura pour résultat [![entrez la description de l’image ici][1]][1]

Notez que vous ne devez pas concevoir un tableau dans lequel les lignes et les colonnes se chevauchent, car il s’agit d’un code HTML non valide et le résultat est géré différemment par les différents navigateurs Web.

rowspan = Un entier non négatif qui spécifie le nombre de lignes couvertes par une cellule. La valeur par défaut de cet attribut est un (1). Une valeur de zéro (0) signifie que la cellule s’étendra de la ligne actuelle jusqu’à la dernière ligne du tableau (<thead>, <tbody> ou <tfoot>).

colspan = Un entier non négatif qui spécifie le nombre de colonnes couvertes par la cellule actuelle. La valeur par défaut de cet attribut est un (1). Une valeur de zéro (0) signifie que la cellule s’étendra de la colonne actuelle à la dernière colonne du groupe de colonnes <colgroup> dans lequel la cellule est définie.

[1] : http://i.stack.imgur.com/voMBJ.jpg

Groupes de colonnes #

Parfois, vous souhaiterez peut-être appliquer un style à une colonne ou à un groupe de colonnes. Ou à des fins sémantiques, vous pouvez regrouper les colonnes. Pour ce faire, utilisez les éléments <colgroup> et <col>.

La balise facultative <colgroup> vous permet de regrouper des colonnes. Les éléments <colgroup> doivent être des éléments enfants d’une <table> et doivent venir après tout élément <caption> et avant tout contenu de table (par exemple, <tr>, <thead>, < tbody>, etc.).

<table>
    <colgroup span="2"></colgroup>
    <colgroup span="2"></colgroup>
    ...
</table>

La balise facultative <col> vous permet de référencer des colonnes individuelles ou une plage de colonnes sans appliquer de regroupement logique. Les éléments <col> sont facultatifs, mais s’ils sont présents, ils doivent être à l’intérieur d’un élément <colgroup>.

<table>
    <colgroup>
        <col id="MySpecialColumn" />
        <col />
    </colgroup>
    <colgroup>
        <col class="CoolColumn" />
        <col class="NeatColumn" span="2" />
    </colgroup>
    ...
</table>

Les styles CSS suivants peuvent être appliqués aux éléments <colgroup> et <col> :

  • ‘frontière’

  • “arrière-plan”

  • “largeur”

  • “visibilité”

  • display (comme dans display: none)

    • display: none; will actually remove the columns from the display, causing the table to render as if those cells don’t exist

Pour plus d’informations, voir [HTML5 Tabular data][1].

[1] : https://www.w3.org/TR/html5/tabular-data.html

Tableau avec le titre, le corps, le pied et la légende #

HTML fournit également les tables avec les éléments <thead>, <tbody>, <tfoot> et <caption>. Ces éléments supplémentaires sont utiles pour ajouter une valeur sémantique à vos tableaux et pour fournir un emplacement pour un style CSS distinct.

Lors de l’impression d’un tableau qui ne tient pas sur une page (papier), la plupart des navigateurs répètent le contenu de <thead> sur chaque page.

Il y a un ordre spécifique qui doit être respecté, et nous devons être conscients que tous les éléments ne se mettent pas en place comme on pourrait s’y attendre. L’exemple suivant montre comment nos 4 éléments doivent être placés.

<table>
 <caption>Table Title</caption> <!--| caption is the first child of table |-->
  <thead> <!--======================| thead is after caption |-->
    <tr>
      <th>Header content 1</th> 
      <th>Header content 2</th>
    </tr>
  </thead>

  <tbody> <!--======================| tbody is after thead |-->
    <tr>
      <td>Body content 1</td>
      <td>Body content 2</td>
    </tr>
  </tbody>
  
  <tfoot><!--| tfoot can be placed before or after tbody, but not in a group of tbody. |-->         <!--| Regardless where tfoot is in markup, it's rendered at the bottom. |-->                                                                                                                                                                                                                                                                                               
    <tr>
      <td>Footer content 1</td>
      <td>Footer content 2</td>
    </tr>
  </tfoot>

</table>

Les résultats de l’exemple suivant sont démontrés deux fois : le premier tableau n’a aucun style, le deuxième tableau a quelques propriétés CSS appliquées : background-color, color et border*. Les styles sont fournis à titre indicatif et ne constituent pas un aspect essentiel du sujet traité.

Étendue de la rubrique #

Les éléments th sont très couramment utilisés pour indiquer les en-têtes des lignes et des colonnes d’un tableau, comme ceci :

<table>
    <thead>
        <tr>
            <td></td>
            <th>Column Heading 1</th>
            <th>Column Heading 2</th>
        </tr>
    </thead>
    <tbody>
        <tr>
            <th>Row Heading 1</th>
            <td></td>
            <td></td>
        </tr>
        <tr>
            <th>Row Heading 2</th>
            <td></td>
            <td></td>
        </tr>
    </tbody>
</table>

Cela peut être amélioré pour l’accessibilité en utilisant l’attribut scope. L’exemple ci-dessus serait modifié comme suit :

<table>
    <thead>
        <tr>
            <td></td>
            <th scope="col">Column Heading 1</th>
            <th scope="col">Column Heading 2</th>
        </tr>
    </thead>
    <tbody>
        <tr>
            <th scope="row">Row Heading 1</th>
            <td></td>
            <td></td>
        </tr>
        <tr>
            <th scope="row">Row Heading 1</th>
            <td></td>
            <td></td>
        </tr>
    </tbody>
</table>

scope est connu comme un attribut énuméré, ce qui signifie qu’il peut avoir une valeur à partir d’un ensemble spécifique de valeurs possibles. Cet ensemble comprend :

  • ‘col’
  • ’ligne'
  • colgroup
  • groupe de lignes
