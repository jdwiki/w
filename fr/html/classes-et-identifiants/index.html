Les classes et les identifiants facilitent le référencement des éléments HTML à partir de scripts et de feuilles de style. L’attribut class peut être utilisé sur une ou plusieurs balises et est utilisé par CSS pour le style. Cependant, les identifiants sont destinés à faire référence à un seul élément, ce qui signifie que le même identifiant ne doit jamais être utilisé deux fois. Les identifiants sont généralement utilisés avec JavaScript et les liens de documents internes, et sont déconseillés en CSS. Cette rubrique contient des explications et des exemples utiles concernant l’utilisation correcte des attributs de classe et d’ID en HTML.

Syntaxe #

  • classe=“classe1 classe2 classe3”
  • id=“identifiant unique”

Paramètres #

Paramètre Détails
classe Indique la Classe de l’élément (non unique)
identifiant Indique l’ID de l’élément (unique dans le même contexte)
  • class et id sont des attributs globaux, et peuvent donc être assignés à n’importe quel élément HTML.
  • Les noms de classe doivent commencer par une lettre (A-Z ou a-z) et peuvent être suivis de lettres, de chiffres, de traits d’union et de traits de soulignement.
  • Dans HTML5, les attributs class et id peuvent être utilisés sur n’importe quel élément. Dans HTML 4.0.1, ils étaient interdits aux <base>, <head>, <html>, <meta>, <param>, <script>, balises <style> et <title>.
  • Un élément peut avoir une ou plusieurs classes. Les classes sont séparées par des espaces et ne peuvent pas contenir d’espaces elles-mêmes.
  • Un élément ne peut avoir qu’un seul ID et il doit être unique dans son contexte (c’est-à-dire une page Web). Les ID ne peuvent pas non plus contenir d’espaces eux-mêmes.

Donner une classe à un élément #

Les classes sont des identifiants pour les éléments auxquels elles sont affectées. Utilisez l’attribut class pour attribuer une classe à un élément.

<div class="example-class"></div>

Pour affecter plusieurs classes à un élément, séparez les noms de classe par des espaces.

<div class="class1 class2"></div>

Utilisation des classes en CSS #

Les classes peuvent être utilisées pour styliser certains éléments sans changer tous les éléments de ce type. Par exemple, ces deux éléments span peuvent avoir des styles complètement différents :

<span></span>
<span class="special"></span>

Des classes du même nom peuvent être attribuées à n’importe quel nombre d’éléments sur une page et ils recevront tous le style associé à cette classe. Ce sera toujours vrai sauf si vous spécifiez l’élément dans le CSS.

Par exemple, nous avons deux éléments, tous deux avec la classe highlight :

<div class="highlight">Lorem ipsum</div>
<span class="highlight">Lorem ipsum</span>

Si notre CSS est comme ci-dessous, alors la couleur verte sera appliquée au texte dans les deux éléments :

.highlight { color: green; }

Cependant, si nous voulons uniquement cibler les div avec la classe highlight, nous pouvons ajouter une spécificité comme ci-dessous :

div.highlight { color: green; }

Néanmoins, lors du style avec CSS, il est généralement recommandé de n’utiliser que des classes (par exemple .highlight) plutôt que des éléments avec des classes (par exemple div.highlight).

Comme avec tout autre sélecteur, les classes peuvent être imbriquées :

.main .highlight { color: red; } /* Descendant combinator */ 
.footer > .highlight { color: blue; } /* Child combinator */

Vous pouvez également chaîner le sélecteur de classe pour sélectionner uniquement les éléments qui ont une combinaison de plusieurs classes. Par exemple, s’il s’agit de notre HTML :

<div class="special left menu">This text will be pink</div>

Et nous voulons colorer ce morceau de texte spécifique en rose, nous pouvons faire ce qui suit dans notre CSS :

.special.left.menu { color: pink; }

Donner un ID à un élément #

L’attribut ID d’un élément est un identifiant qui doit être unique dans tout le document. Son but est d’identifier de manière unique l’élément lors de la liaison (à l’aide d’une ancre), du script ou du style (avec CSS).

<div id="example-id"></div>

Vous ne devez pas avoir deux éléments avec le même ID dans le même document, même si les attributs sont attachés à deux types d’éléments différents. Par exemple, le code suivant est incorrect :

<div id="example-id"></div>
<span id="example-id"></span>

Les navigateurs feront de leur mieux pour restituer ce code, mais un comportement inattendu peut se produire lors du style avec CSS ou de l’ajout de fonctionnalités avec JavaScript.

Pour référencer des éléments par leur ID dans CSS, préfixez l’ID avec #.

#example-id { color: green; }

Pour accéder à un élément avec un ID sur une page donnée, ajoutez # au nom de l’élément dans l’URL.

http://example.com/about#example-id

Cette fonctionnalité est prise en charge par la plupart des navigateurs et ne nécessite pas de code JavaScript ou CSS supplémentaire pour fonctionner.

## Valeurs acceptables

Pour une pièce d’identité #

Les seules restrictions sur la valeur d’un id sont :

  1. il doit être unique dans le document
  2. il ne doit contenir aucun espace
  3. il doit contenir au moins un caractère

Ainsi, la valeur peut être composée de tous les chiffres, d’un seul chiffre, de caractères de ponctuation uniquement, d’inclure des caractères spéciaux, etc. Juste pas d’espace.

Ceux-ci sont donc valables :

<div id="container"> ... </div>
<div id="999"> ... </div>
<div id="#%LV-||"> ... </div>
<div id="____V"> ... </div>
<div id="⌘⌥"> ... </div>
<div id="♥"> ... </div>
<div id="{}"> ... </div>
<div id="©"> ... </div>
<div id="♤₩¤☆€~¥"> ... </div>

Ceci n’est pas valide :

<div id=" "> ... </div>

Ceci est également invalide, lorsqu’il est inclus dans le même document :

<div id="results"> ... </div>
<div id="results"> ... </div>

Une valeur id doit commencer par une lettre, qui ne peut alors être suivie que de :

  • lettres (A-Z/a-z)
  • chiffres (0-9)
  • traits d’union ("-")
  • traits de soulignement ("_")
  • deux-points (":")
  • points (".")

En se référant au premier groupe d’exemples dans la section HTML5 ci-dessus, un seul est valide :

<div id="container"> ... </div>

Ceux-ci sont également valables :

<div id="sampletext"> ... </div>
<div id="sample-text"> ... </div>
<div id="sample_text"> ... </div>
<div id="sample:text"> ... </div>
<div id="sample.text"> ... </div>

Encore une fois, s’il ne commence pas par une lettre (majuscule ou minuscule), il n’est pas valide.

Pour une classe #

Les règles pour les classes sont essentiellement les mêmes que pour un id. La différence est que les valeurs de “classe” n’ont pas besoin d’être uniques dans le document.

En se référant aux exemples ci-dessus, bien que cela ne soit pas valable dans le même document :

<div id="results"> ... </div>
<div id="results"> ... </div>

C’est tout à fait correct :

<div class="results"> ... </div>
<div class="results"> ... </div>

Remarque importante : comment les valeurs d’ID et de classe sont traitées en dehors du HTML #

Gardez à l’esprit que les règles et les exemples ci-dessus s’appliquent dans le contexte du HTML.

L’utilisation de chiffres, de ponctuation ou de caractères spéciaux dans la valeur d’un id ou d’une class peut causer des problèmes dans d’autres contextes, tels que CSS, JavaScript et les expressions régulières.

Par exemple, bien que l’id suivant soit valide en HTML5 :

<div id="9lions"> ... </div>

… c’est invalide en CSS :

4.1.3 Caractères et casse

En CSS, les identifiants (y compris les noms d’éléments, les classes et les ID dans sélecteurs) ne peut contenir que les caractères [a-zA-Z0-9] et ISO 10646 caractères U+00A0 et plus, plus le trait d’union (-) et le trait de soulignement (_); ils ne peuvent pas commencer par un chiffre, deux traits d’union ou un trait d’union suivi d’un chiffre. (soulignement ajouté)

Dans la plupart des cas, vous pourrez peut-être échapper des caractères dans des contextes où ils ont des restrictions ou une signification particulière.

Références W3C #

  • 3.2.5.1 L’attribut id
  • 3.2.5.7 L’attribut class
  • 6.2 Types de base SGML

Problèmes liés aux identifiants en double #

Avoir plus d’un élément avec le même ID est un problème difficile à résoudre. L’analyseur HTML essaiera généralement de rendre la page dans tous les cas. Habituellement, aucune erreur ne se produit. Mais le rythme pourrait aboutir à une page Web qui se comporte mal.

Dans cet exemple :

<div id="aDiv">a</div>
<div id="aDiv">b</div>

Les sélecteurs CSS fonctionnent toujours

#aDiv {
    color: red;
}

Mais JavaScript ne parvient pas à gérer les deux éléments :

var html = document.getElementById("aDiv").innerHTML;

Dans ce cas, la variable html ne porte que le premier contenu div ("a").
