Les balises META dans les documents HTML fournissent des informations utiles sur le document, notamment une description, des mots-clés, l’auteur, les dates de modification et environ 90 autres champs. Cette rubrique traite de l’utilisation et de l’objectif de ces balises.

Syntaxe #

  • <meta name="nom des métadonnées" content="value">
  • <meta http-equiv="directive pragma" content="value">
  • <meta charset="étiquette d'encodage">

La balise meta est une balise HTML utilisée pour définir les métadonnées du document HTML. Les balises META doivent aller dans l’élément head. Une page peut avoir n’importe quel nombre de balises meta.

La balise méta “keywords” n’est généralement pas utilisée par les robots. La plupart des moteurs de recherche déterminent quels mots clés correspondent au contenu des pages Web. Cela étant dit, rien ne dit que vous ne devriez plus inclure la balise meta keywords.

Les métadonnées d’une page sont principalement utilisées par le navigateur (comme la mise à l’échelle d’un document) et les robots d’exploration du Web utilisés par les moteurs de recherche (Google, Yahoo!, Bing).

La spécification donne un certain nombre de noms de métadonnées standardisés à utiliser avec <nom de méta> et pragma de métadonnées standardisées directives à utiliser avec <meta http-equiv>. Cependant, de nombreux services sur Internet (crawlers Web, outils de création, services de partage sur les réseaux sociaux, etc.) utilisent le formulaire <meta name> comme point d’extension générique pour les métadonnées. Certains d’entre eux sont répertoriés [sur la page wiki de la spécification] (https://wiki.whatwg.org/wiki/MetaExtensions).

Informations sur la page #

nom-application #

Donner le nom de l’application Web que la page représente.

<meta name="application-name" content="OpenStreetMap">

S’il ne s’agit pas d’une application Web, la balise META application-name ne doit pas être utilisée.

auteur #

Définissez l’auteur de la page :

<meta name="author" content="Your Name">

Un seul nom peut être donné.

description #

Définissez la description de la page :

<meta name="description" content="Page Description">

La balise meta “description” peut être utilisée par divers moteurs de recherche lors de l’indexation de votre page Web à des fins de recherche. Habituellement, la description contenue dans la balise META est le court résumé qui apparaît sous le titre principal de la page/du site Web dans les résultats des moteurs de recherche. Google n’utilise généralement que les 20 à 25 premiers mots de votre description.

générateur #

<meta name="generator" content="HTML Generator 1.42">

Identifie l’un des progiciels utilisés pour générer le document. À n’utiliser que pour les pages où le balisage est généré automatiquement.

mots clés #

Définissez des mots-clés pour les moteurs de recherche (séparés par des virgules) :

<meta name="keywords" content="Keyword1, Keyword2">

La balise meta keywords est parfois utilisée par les moteurs de recherche pour connaître la requête de recherche pertinente pour votre page Web. En règle générale, c’est probablement une bonne idée de ne pas ajouter trop de mots, car la plupart des moteurs de recherche qui utilisent cette balise méta pour l’indexation n’indexeront que les ~20 premiers mots. Assurez-vous de mettre les mots-clés les plus importants en premier.

Encodage de caractère #

L’attribut charset spécifie le codage de caractères pour le document HTML et doit être un codage de caractères valide (les exemples incluent windows-1252, ISO-8859-2, Shift_JIS et UTF-8). UTF-8 (Unicode) est le plus largement utilisé et doit être utilisé pour tout nouveau projet.

<meta charset="UTF-8">

<meta charset="ISO-8859-1">

Tous les navigateurs ont toujours reconnu la forme <meta charset>, mais si, pour une raison quelconque, vous avez besoin que votre page soit valide HTML 4.01, vous pouvez utiliser ce qui suit à la place :

<meta http-equiv="content-type" content="text/html; charset=UTF-8">

<meta http-equiv="content-type" content="text/html; charset=ISO-8859-1">

Voir également la [Norme d’encodage][1], pour afficher toutes les étiquettes d’encodage de caractères disponibles que les navigateurs reconnaissent.

[1] : https://encoding.spec.whatwg.org/#names-and-labels

Robots #

L’attribut robots, pris en charge par plusieurs moteurs de recherche majeurs, contrôle si les moteurs de recherche sont autorisés à indexer une page ou non et s’ils doivent ou non suivre les liens d’une page.

<meta name="robots" content="noindex">

Cet exemple indique à tous les moteurs de recherche de ne pas afficher la page dans les résultats de recherche. Les autres valeurs autorisées sont :

Valeur/Directive Sens
tout Par défaut. Équivalent à index, follow. Voir note ci-dessous.
pas d'index Ne pas indexer la page du tout.
pas de suivi Ne suivez pas les liens sur cette page
suivre Les liens sur la page peuvent être suivis. Voir note ci-dessous.
aucun Équivalent à “noindex, nofollow”.
pas d'archive Ne rendez pas disponible une version en cache de cette page dans les résultats de recherche.
nocache Synonyme de noarchive utilisé par certains bots comme Bing.
nosnippet Ne pas afficher un extrait de cette page dans les résultats de recherche.
noodp N’utilisez pas les métadonnées de cette page du projet Open Directory pour les titres ou les extraits dans les résultats de recherche.
notranslate Ne proposez pas de traductions de cette page dans les résultats de recherche.
noimageindex Ne pas indexer les images sur cette page.
unavailable_after [date/heure RFC-850] Ne pas afficher cette page dans les résultats de recherche après la date/heure spécifiée. La date/heure doit être spécifiée au format RFC 850.

Remarque : Définir explicitement index et/ou follow, bien que des valeurs valides, n’est pas nécessaire car pratiquement tous les moteurs de recherche supposeront qu’ils sont autorisés à le faire s’ils ne sont pas explicitement empêchés de le faire. Semblable au fonctionnement du fichier robots.txt, les moteurs de recherche ne recherchent généralement que les choses qu’ils ne sont pas autorisés à faire. Indiquer uniquement des choses qu’un moteur de recherche n’est pas autorisé à faire empêche également d’indiquer accidentellement des contraires (tels que index, ..., noindex) que tous les moteurs de recherche ne traiteront pas de la même manière.

Des médias sociaux #

Open Graph est une norme pour les métadonnées qui étend les informations normales contenues dans le balisage principal d’un site. Cela permet aux sites Web tels que Facebook d’afficher des informations plus détaillées et plus riches sur un site Web dans un format structuré. Ces informations sont ensuite automatiquement affichées lorsque les utilisateurs partagent des liens vers des sites Web contenant des métadonnées OG sur Facebook.

Facebook / Graphique ouvert #

<meta property="fb:app_id" content="123456789">
<meta property="og:url" content="https://example.com/page.html">
<meta property="og:type" content="website">
<meta property="og:title" content="Content Title">
<meta property="og:image" content="https://example.com/image.jpg">
<meta property="og:description" content="Description Here">
<meta property="og:site_name" content="Site Name">
<meta property="og:locale" content="en_US">
<meta property="article:author" content="">
<!-- Facebook: https://developers.facebook.com/docs/sharing/webmasters#markup -->
<!-- Open Graph: http://ogp.me/ -->
  • [Facebook Open Graph Markup] (https://developers.facebook.com/docs/sharing/webmasters#markup)
  • [Protocole Open Graph] (http://ogp.me/)

Facebook / Articles instantanés #

<meta charset="utf-8">
<meta property="op:markup_version" content="v1.0">

<!-- The URL of the web version of your article -->
<link rel="canonical" href="http://example.com/article.html">

<!-- The style to be used for this article -->
<meta property="fb:article_style" content="myarticlestyle">
  • [Facebook Instant Articles : Création d’articles] (https://developers.facebook.com/docs/instant-articles/guides/articlecreate)
  • [Articles instantanés : Référence de format] (https://developers.facebook.com/docs/instant-articles/reference)

Twitter utilise son propre balisage pour les métadonnées. Ces métadonnées sont utilisées comme informations pour contrôler la façon dont les tweets sont affichés lorsqu’ils contiennent un lien vers le site.

Twitter #

<meta name="twitter:card" content="summary">
<meta name="twitter:site" content="@site_account">
<meta name="twitter:creator" content="@individual_account">
<meta name="twitter:url" content="https://example.com/page.html">
<meta name="twitter:title" content="Content Title">
<meta name="twitter:description" content="Content description less than 200 characters">
<meta name="twitter:image" content="https://example.com/image.jpg">
  • [Twitter Cards : Guide de démarrage] (https://dev.twitter.com/cards/getting-started)
  • Twitter Card Validator

### Google+/Schema.org

<link href="https://plus.google.com/+YourPage" rel="publisher">
<meta itemprop="name" content="Content Title">
<meta itemprop="description" content="Content description less than 200 characters">
<meta itemprop="image" content="https://example.com/image.jpg">

Contrôle de la disposition mobile #

Les sites courants optimisés pour les mobiles utilisent la balise <meta name="viewport"> comme ceci :

<meta name="viewport" content="width=device-width, initial-scale=1">

L’élément viewport donne au navigateur des instructions sur la façon de contrôler les dimensions et la mise à l’échelle de la page en fonction de l’appareil que vous utilisez.

Dans l’exemple ci-dessus, content="width=device-width signifie que le navigateur affichera la largeur de la page à la largeur de son propre écran. Donc, si cet écran fait 480px de large, la fenêtre du navigateur sera 480px de large. initial-scale=1 indique que le zoom initial (qui est de 1 dans ce cas, signifie qu’il ne zoome pas).

Vous trouverez ci-dessous les attributs pris en charge par cette balise :

Attribut Descriptif
largeur La largeur de la fenêtre d’affichage virtuelle de l’appareil.
Valeurs1 : device-width ou la largeur réelle en pixels, comme 480
hauteur La hauteur de la fenêtre d’affichage virtuelle de l’appareil.
Valeurs2 : device-height ou la largeur réelle en pixels, comme 600
échelle initiale Le zoom initial lorsque la page est chargée. 1.0 ne zoome pas.
échelle minimale Le montant minimum que le visiteur peut zoomer sur la page. 1.0 ne zoome pas.
échelle maximale Le montant maximum que le visiteur peut zoomer sur la page. 1.0 ne zoome pas.
évolutif par l'utilisateur Permet à l’appareil d’effectuer un zoom avant et arrière. Les valeurs sont “oui” ou “non”. S’il est défini sur non, l’utilisateur ne peut pas zoomer sur la page Web. La valeur par défaut est oui. Les paramètres du navigateur peuvent ignorer cette règle.

Remarques:

1 La propriété width peut être spécifiée en pixels (width=600) ou en device-width (width=device-width) qui représente la largeur physique de l’écran de l’appareil.

2 De même, la propriété height peut être spécifiée en pixels (height=600) ou en device-height (height=device-height) qui représente la hauteur de l’écran de l’appareil.

Actualisation automatique #

Pour actualiser la page toutes les cinq secondes, ajoutez cet élément meta dans l’élément head :

<meta http-equiv="refresh" content="5">

ATTENTION ! Bien qu’il s’agisse d’une commande valide, il est recommandé de ne pas l’utiliser en raison de ses effets négatifs sur l’expérience utilisateur. L’actualisation trop fréquente de la page peut entraîner une absence de réponse et un défilement vers le haut de la page. Si certaines informations sur la page doivent être mises à jour en permanence, il existe de bien meilleures façons de le faire en actualisant uniquement une partie de la page.

Reconnaissance du numéro de téléphone #

Les plates-formes mobiles telles qu’iOS reconnaissent automatiquement les numéros de téléphone et les transforment en liens “tel:”. Bien que la fonctionnalité soit très pratique, le système détecte parfois les codes ISBN et d’autres numéros comme numéros de téléphone.

Pour que Safari mobile et certains autres navigateurs mobiles basés sur WebKit désactivent la reconnaissance et le formatage automatiques des numéros de téléphone, vous avez besoin de cette balise META :

<meta name="format-detection" content="telephone=no">

Redirection automatique #

Parfois, votre page Web a besoin d’une redirection automatique.

Par exemple, pour rediriger vers example.com après 5 secondes :

<meta http-equiv="refresh" content="5;url=https://www.example.com/" />

Cette ligne vous enverra au site Web désigné (dans ce cas, example.com après 5 secondes.

Si vous avez besoin de modifier le délai avant une redirection, il suffit de changer le nombre juste avant votre ;url= pour modifier le délai.

Application Web #

Vous pouvez configurer votre application Web ou votre site Web pour qu’une icône de raccourci d’application soit ajoutée à l’écran d’accueil d’un appareil et que l’application se lance en “mode application” plein écran à l’aide de l’élément de menu [“Ajouter à l’écran d’accueil”][1] de Chrome pour Android. .

Les balises méta ci-dessous ouvriront l’application Web en mode plein écran (sans barre d’adresse).

AndroidChrome

<meta name="mobile-web-app-capable" content="yes">

iOS

<meta name="apple-mobile-web-app-capable" content="yes">


Vous pouvez également définir la couleur de la barre d’état et de la barre d’adresse dans la balise META.

AndroidChrome

<meta name="theme-color" content="black">

iOS

<meta name="apple-mobile-web-app-status-bar-style" content="black">

[1] : https://developer.chrome.com/multidevice/android/installtohomescreen
