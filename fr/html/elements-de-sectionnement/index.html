Les normes HTML5 ne répertorient pas l’élément principal en tant qu’élément de sectionnement.

Élément de navigation #

L’élément <nav> est principalement destiné à être utilisé pour les sections qui contiennent des blocs de navigation principaux pour le site Web, cela peut inclure des liens vers d’autres parties de la page Web (par exemple, des ancres pour une table des matières) ou d’autres pages entièrement.

éléments en ligne #

Ce qui suit affichera un ensemble d’hyperliens en ligne.

<nav>
    <a href="https://google.com">Google</a>
    <a href="https://www.yahoo.com">Yahoo!</a>
    <a href="https://www.bing.com">Bing</a>
</nav>

Utilisez les éléments de la liste si nécessaire #

Si le contenu représente une liste d’éléments, utilisez un élément de liste pour l’afficher et améliorer l’expérience utilisateur.

Notez le role="navigation", plus à ce sujet ci-dessous.

<nav role="navigation">
    <ul>
        <li><a href="https://google.com">Google</a></li>
        <li><a href="https://www.yahoo.com">Yahoo!</a></li>
        <li><a href="https://www.bing.com">Bing</a></li>
    </ul>
</nav>

Évitez les utilisations inutiles #

Les éléments <footer> peuvent avoir une liste de liens vers d’autres parties du site (FAQ, T&C, etc.). L’élément footer seul est suffisant dans ce cas, vous n’avez pas * besoin * d’envelopper davantage vos liens avec un élément <nav> dans le <footer>.

<!-- the <nav> is not required in the <footer> -->
<footer>
    <nav>
        <a href="#">...</a>
    </nav>
</footer>

<!-- The footer alone is sufficient -->
<footer>
    <a href="#">...</a>
</footer>

Remarques :

  • Les descendants de [<main> element][main-el] ne sont pas autorisés dans un <nav>

L’ajout d’un role="navigation" [ARIA role][w3-aria] à l’élément <nav> est conseillé pour aider les agents utilisateurs qui ne prennent pas en charge HTML5 et également pour fournir plus de contexte à ceux qui le font.

Lecteurs d’écran : (logiciel permettant aux utilisateurs aveugles ou malvoyants de naviguer sur le site)

Les agents utilisateurs comme les lecteurs d’écran interpréteront l’élément <nav> différemment selon leurs besoins.

  • Cela pourrait donner à l’élément <nav> une priorité plus élevée lors du rendu de la page
  • Cela pourrait retarder le rendu de l’élément
  • Il pourrait adapter la page d’une manière spécifique pour l’adapter aux besoins de l’utilisateur exemple : agrandissez les liens de texte dans les éléments <nav> pour une personne malvoyante.

[Cliquez ici pour lire la spécification HTML5 officielle pour l’élément <nav>][html5-spec]

[html5-spec] : https://www.w3.org/TR/html5/sections.html#the-nav-element [main-el] : http://docs-beta.stackexchange.com/documentation/html/311/sectioning-elements/1093/main-element#t=201607161859442758976 [w3-aria] : https://www.w3.org/TR/html-aria/ [aria-navigation] : https://www.w3.org/TR/html5/dom.html#index-aria-navigation [présentation-aria] : https://www.w3.org/TR/html5/dom.html#index-présentation-aria

Élément d’article #

L’élément <article> contient du contenu autonome comme des articles, des articles de blog, des commentaires d’utilisateurs ou un widget interactif qui pourrait être distribué en dehors du contexte de la page, par exemple par RSS.

  • Lorsque les éléments d’article sont imbriqués, le contenu du nœud d’article interne doit être lié à l’élément d’article externe.

Un blog (section) avec plusieurs articles (article) et commentaires (article) peut ressembler à ceci.

<section>
    <!-- Each individual blog post is an <article> -->
    <article>
        <header>
            <h1>Blog Post</h1>
            <time datetime="2016-03-13">13th March 2016</time>
        </header>

        <p>The article element represents a self contained article or document.</p>
        <p>The section element represents a grouping of content.</p>

        <section>
            <h2>Comments <small>relating to "Blog Post"</small></h2>

            <!-- Related comment is also a self-contained article -->
            <article id="user-comment-1">
                <p>Excellent!</p>
                <footer><p>...</p><time>...</time></footer>
            </article>
        </section>
    </article>

    <!-- ./repeat: <article> -->

</section>

<!-- Content unrelated to the blog or posts should be outside the section. -->
<footer>
    <p>This content should be unrelated to the blog.</p>
</footer>

Évitez les utilisations inutiles #

Lorsque le contenu principal de la page (hors en-têtes, pieds de page, barres de navigation, etc.) est simplement un groupe d’éléments. Vous pouvez omettre <article> en faveur de l’élément <main>.

<article>
    <p>This doesn't make sense, this article has no real `context`.</p>
</article>

Au lieu de cela, remplacez l’article par un élément [<main>][main-el] pour indiquer qu’il s’agit du contenu “principal” de cette page.

<main>
    <p>I'm the main content, I don't need to belong to an article.</p>
</main>

Si vous utilisez un autre élément, assurez-vous de spécifier le [<main> rôle ARIA][aria-main] pour une interprétation et un rendu corrects sur plusieurs appareils et navigateurs non HTML5.

<section role="main">
    <p>This section is the main content of this page.</p>
</section>

Remarques :

  • Les descendants de [<main> element][main-el] ne sont pas autorisés dans un <article>

[Cliquez ici pour lire la spécification HTML5 officielle pour l’élément <article>][html5-spec]

[html5-spec] : https://www.w3.org/TR/html5/sections.html#the-article-element [main-el] : http://docs-beta.stackexchange.com/documentation/html/311/sectioning-elements/1093/main-element#t=201607161859442758976 [w3-aria] : https://www.w3.org/TR/html-aria/ [aria-article] : https://www.w3.org/TR/html5/dom.html#index-aria-article [application-aria] : https://www.w3.org/TR/html5/dom.html#index-application-aria [document-aria] : https://www.w3.org/TR/html5/dom.html#index-document-aria [aria-main] : https://www.w3.org/TR/html5/dom.html#index-aria-main

Élément principal #

L’élément <main> contient le contenu principal de votre page Web. Ce contenu est unique à la page individuelle et ne doit pas apparaître ailleurs sur le site. Le contenu répétitif comme les en-têtes, les pieds de page, la navigation, les logos, etc., est placé à l’extérieur de l’élément.

  • L’élément <main> ne doit jamais être utilisé au plus une fois sur une seule page.
  • L’élément <main> ne doit pas être inclus en tant que descendant d’un élément article, aside, footer, header ou nav.

Dans l’exemple suivant, nous affichons un article de blog unique (et des informations connexes telles que des références et des commentaires).

<body>
    <header>
        <nav>...</nav>
    </header>

    <main>
        <h1>Individual Blog Post</h1>
        <p>An introduction for the post.</p>

        <article>
            <h2>References</h2>
            <p>...</p>
        </article>

        <article>
            <h2>Comments</h2> ...
        </article>
    </main>

    <footer>...</footer>
</body>

  • Le billet de blog est contenu dans l’élément <main> pour indiquer qu’il s’agit du contenu principal de cette page (et donc unique sur le site Web).

  • Les balises <header> et <footer> sont frères de l’élément <main>.

Remarques :

La spécification HTML5 reconnaît l’élément <main> comme un élément grouping, et non comme un élément sectioning.

  • [Attributs du rôle ARIA][w3-aria] : [main][aria-main] (par défaut), [presentation][aria-presentation]

L’ajout d’un attribut role="main" [ARIA role][aria-main] à d’autres éléments destinés à être utilisés comme contenu principal est conseillé pour aider les agents utilisateurs qui ne prennent pas en charge HTML5 et également pour fournir plus de contexte pour ceux qui le font.

L’élément <main> par défaut a le rôle principal, et n’a donc pas besoin d’être fourni.

[Cliquez ici pour lire la spécification HTML5 officielle pour l’élément <main>][html5-spec]

[w3-aria] : https://www.w3.org/TR/html-aria/ [aria-main] : https://www.w3.org/TR/html5/dom.html#index-aria-main [présentation-aria] : https://www.w3.org/TR/html5/dom.html#index-présentation-aria [html5-spec] : https://www.w3.org/TR/html5/grouping-content.html#the-main-element

Élément de section #

L’élément <section> représente une section générique pour regrouper le contenu par thème. En règle générale, chaque section doit pouvoir être identifiée par un élément d’en-tête en tant qu’enfant de la “section”.

  • Vous pouvez utiliser l’élément <section> dans un <article> et vice-versa.
  • Chaque section doit avoir un thème (un élément d’en-tête identifiant cette région)
  • N’utilisez pas l’élément <section> comme un ‘conteneur’ de style général. Si vous avez besoin d’un conteneur pour appliquer le style, utilisez plutôt un <div>.

Dans l’exemple suivant, nous affichons un article de blog unique avec plusieurs chapitres, chaque chapitre est une section (un ensemble de contenu regroupé par thème, qui peut être identifié par les éléments d’en-tête de chaque section).

<article>
    <header>
        <h2>Blog Post</h2>
    </header>
    <p>An introduction for the post.</p>
    <section>
        <h3>Chapter 1</h3>
        <p>...</p>
    </section>
    <section>
        <h3>Chapter 2</h3>
        <p>...</p>
    </section>
    <section>
        <h3>Comments</h3> ...
    </section>
</article>

Remarques :

Les développeurs doivent utiliser l’élément article lorsqu’il est judicieux de syndiquer le contenu de l’élément.

[Cliquez ici pour lire la spécification HTML5 officielle pour l’élément <main>][html5-spec]

[html5-spec] : https://www.w3.org/TR/html5/grouping-content.html#the-section-element [w3-aria] : https://www.w3.org/TR/html-aria/ [région-aria] : https://www.w3.org/TR/html5/dom.html#index-aria-main

Élément d’en-tête #

L’élément <header> représente le contenu d’introduction pour son contenu de sectionnement ancêtre le plus proche ou son élément racine de sectionnement. Un <header> contient généralement un groupe d’aides d’introduction ou de navigation.

Remarque : L’élément d’en-tête ne sectionne pas le contenu ; il n’introduit pas de nouvelle section.

Exemples: #

<header>
  <p>Welcome to...</p>
  <h1>Voidwars!</h1>
</header>

Dans cet exemple, le [<article>][1] a un <header>.

<article>
  <header>
    <h1>Flexbox: The definitive guide</h1>
  </header>
  <p>The guide about Flexbox was supposed to be here, but it turned out Wes wasn’t a Flexbox expert either.</p>
</article>

[Proposition de recommandation du W3C][2]

[1] : https://www.wikiod.com/fr/html/elements-de-sectionnement#Élément d’article [2] : https://www.w3.org/TR/html51/sections.html#the-header-element

Élément de pied de page #

L’élément <footer> contient la partie pied de page de la page.

Voici un exemple d’élément <footer> contenant la balise de paragraphe p.

<footer>
    <p>All rights reserved</p>
</footer>
