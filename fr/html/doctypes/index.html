Doctypes - abréviation de “type de document” - aide les navigateurs à comprendre la version de HTML dans laquelle le document est écrit pour une meilleure interprétabilité. Les déclarations Doctype ne sont pas des balises HTML et appartiennent tout en haut d’un document. Cette rubrique explique la structure et la déclaration de divers doctypes en HTML.

Syntaxe #

  • <!DOCTYPE [chaîne spécifique à la version]>

La déclaration <!DOCTYPE> n’est pas une balise HTML. Il est utilisé pour spécifier la version de HTML utilisée par le document. C’est ce qu’on appelle la déclaration de type de document (DTD).

La déclaration <!DOCTYPE> n’est PAS sensible à la casse. Pour vérifier si le code HTML de vos pages Web est valide, accédez au [service de validation du W3C][1].

  • Certaines anciennes versions d’IE ne prennent pas en charge certaines balises HTML à moins qu’un doctype approprié ne soit disponible.
  • Il est vital qu’un doctype soit déclaré pour s’assurer que le navigateur n’utilise pas le mode quirks. Plus d’infos sur MDN.

[1] : http://validator.w3.org/

Ajout du Doctype #

La déclaration <!DOCTYPE> doit toujours être incluse en haut du document HTML, avant la balise <html>.

Voir https://www.wikiod.com/fr/html/doctypes#Doctype HTML 5 pour plus de détails sur le HTML 5 Doctype.

<!DOCTYPE html>

Voir https://www.wikiod.com/fr/html/doctypes#Doctypes HTML 4.01 pour plus de détails sur la façon dont ces types diffèrent les uns des autres.

Stricte

<!DOCTYPE HTML PUBLIC "-//W3C//DTD HTML 4.01//EN" "http://www.w3.org/TR/html4/strict.dtd">

De transition

<!DOCTYPE HTML PUBLIC "-//W3C//DTD HTML 4.01 Transitional//EN" "http://www.w3.org/TR/html4/loose.dtd">

Ensemble de cadres

<!DOCTYPE HTML PUBLIC "-//W3C//DTD HTML 4.01 Frameset//EN" "http://www.w3.org/TR/html4/frameset.dtd">

HTML 5 Doctype #

HTML5 n’est pas basé sur SGML et ne nécessite donc pas de référence à une DTD.

Déclaration de Doctype HTML 5 :

<!DOCTYPE html>

Insensibilité à la casse #

Selon la spécification W3.org HTML 5 DOCTYPE:

Un DOCTYPE doit être composé des éléments suivants, dans cet ordre :

  1. Une chaîne qui est une correspondance ASCII insensible à la casse pour la chaîne "<!DOCTYPE".

par conséquent, les DOCTYPE suivants sont également valides :

<!doctype html>
<!dOCtyPe html>
<!DocTYpe html>

Cet article SO aborde le sujet en détail : http://stackoverflow.com/questions/7020961/uppercase-or-lowercase-doctype

Doctypes HTML 4.01 #

La spécification HTML 4.01 fournit plusieurs types différents de doctypes qui permettent de spécifier différents types d’éléments dans le document.

HTML 4.01 Strict #

<!DOCTYPE HTML PUBLIC "-//W3C//DTD HTML 4.01//EN" "http://www.w3.org/TR/html4/strict.dtd">

Inclut tous les éléments et attributs HTML, mais ** n’inclut pas les éléments de présentation ou obsolètes ** et les ** jeux de cadres ne sont pas autorisés **.

HTML 4.01 Transitionnel #

<!DOCTYPE HTML PUBLIC "-//W3C//DTD HTML 4.01 Transitional//EN" "http://www.w3.org/TR/html4/loose.dtd">

Inclut tous les éléments et attributs HTML, ainsi que les éléments de présentation et obsolètes, mais les jeux de cadres ne sont pas autorisés.

Jeu de cadres HTML 4.01 #

<!DOCTYPE HTML PUBLIC "-//W3C//DTD HTML 4.01 Frameset//EN" "http://www.w3.org/TR/html4/frameset.dtd">

Inclut tous les éléments et attributs HTML, les éléments de présentation et obsolètes. Les jeux de cadres sont autorisés.

Anciens Doctypes #

###HTML 3.2

<!DOCTYPE HTML PUBLIC "-//W3C//DTD HTML 3.2 Final//EN">

HTML 3.2 est bien pris en charge par la plupart des navigateurs utilisés. Cependant, HTML 3.2 a une prise en charge limitée des feuilles de style et aucune prise en charge des fonctionnalités HTML 4 telles que les cadres et l’internationalisation.

###HTML 2.0

<!DOCTYPE HTML PUBLIC "-//IETF//DTD HTML//EN">

HTML 2.0 est largement pris en charge par les navigateurs, mais ne prend pas en charge les tableaux, les cadres et l’internationalisation, ainsi que de nombreux éléments et attributs de présentation couramment utilisés.
