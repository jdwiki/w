Syntaxe #

  • <img src="" alt="">

Paramètres #

Paramètres Détails
src Spécifie l’URL de l’image
srcset Images à utiliser dans différentes situations (par exemple, écrans haute résolution, petits moniteurs, etc.)
tailles Tailles d’image entre les points d’arrêt
origine croisée Comment l’élément gère les requêtes crossorigin
usemap Nom de l’image cliquable à utiliser
ismap Indique si l’image est une image cliquable côté serveur
alt Texte alternatif qui devrait être affiché si pour une raison quelconque l’image ne pouvait pas être affichée
largeur Spécifie la largeur de l’image (facultatif)
hauteur Spécifie la hauteur de l’image (facultatif)

Création d’une image #

Pour ajouter une image à une page, utilisez la balise image.

Les balises d’image (img) n’ont pas de balises de fermeture. Les deux principaux attributs que vous donnez à la balise img sont src, la source de l’image et alt, qui est un texte alternatif décrivant l’image.

<img src="images/hello.png" alt="Hello World">

Vous pouvez également obtenir des images à partir d’une URL Web :

<img src="https://i.stack.imgur.com/ALgZi.jpg?s=48&g=1" alt="StackOverflow user Caleb Kleveter">

Remarque : Les images ne sont pas techniquement insérées dans une page HTML, les images sont liées à des pages HTML. La balise <img> crée un espace de maintien pour l’image référencée.

Il est également possible d’intégrer des images directement à l’intérieur de la page en utilisant base64 :

<img src="data:image/png;base64,iVBOR..." alt="Hello World">

Astuce : Pour lier une image à un autre document, imbriquez simplement la balise <img> dans les balises <a>.

Choisir le texte alternatif #

Le texte alternatif est utilisé par les lecteurs d’écran pour les utilisateurs malvoyants et par les moteurs de recherche. Il est donc important d’écrire un bon texte alternatif pour vos images.

Le texte doit être correct même si vous remplacez l’image par son attribut alt. Par exemple:

<!-- Incorrect -->
<img src="anonymous.png" alt="Anonymous user avatar"/> An anonymous user wrote:
<blockquote>Lorem ipsum dolor sed.</blockquote>
<a href="https://google.com/"><img src="edit.png" alt="Edit icon"/></a> /
<a href="https://google.com/"><img src="delete.png" alt="Delete icon"/></a>

Sans les images, cela ressemblerait à :

Avatar d’utilisateur anonyme Un utilisateur anonyme a écrit :

C'est chiant.
Icône Modifier / Supprimer l'icône

Pour corriger cela :

  • Supprimez le texte alternatif de l’avatar. Cette image ajoute des informations pour les utilisateurs voyants (une icône facilement identifiable pour indiquer que l’utilisateur est anonyme), mais ces informations sont déjà disponibles dans le texte1.
  • Supprimez “l’icône” du texte alternatif pour les icônes. Savoir qu’il s’agirait d’une icône si elle était là n’aide pas à transmettre son véritable objectif.
<!-- Correct -->
<img src="anonymous.png" alt=""/> An anonymous user wrote:
<blockquote>Lorem ipsum dolor sed.</blockquote>
<a href="https://google.com/"><img src="edit.png" alt="Edit"/></a> /
<a href="https://google.com/"><img src="delete.png" alt="Delete"/></a>

Un utilisateur anonyme a écrit :

C'est chiant.
Modifier / Supprimer

Notes de bas de page

1 Il existe une différence sémantique entre inclure un attribut alt vide et l’exclure complètement. Un attribut alt vide indique que l’image n’est pas un élément clé du contenu (comme c’est le cas dans ce cas - c’est juste une image additive qui n’est pas nécessaire pour comprendre le reste) et peut donc être omise du rendu. Cependant, l’absence d’attribut alt indique que l’image est un élément clé du contenu et qu’il n’y a tout simplement pas d’équivalent textuel disponible pour le rendu.

Image réactive utilisant l’attribut srcset #

Utilisation de srcset avec des tailles #

<img sizes="(min-width: 1200px) 580px,
      (min-width: 640px) 48vw,
      98vw"
    srcset="img/hello-300.jpg 300w,
      img/hello-600.jpg 600w,
      img/hello-900.jpg 900w,
      img/hello-1200.jpg 1200w"
    src="img/hello-900.jpg" alt="hello">

Les “tailles” sont comme des requêtes multimédias, décrivant l’espace occupé par l’image dans la fenêtre d’affichage.

  • si la fenêtre d’affichage est supérieure à 1200 pixels, l’image mesure exactement 580 pixels (par exemple, notre contenu est centré dans un conteneur d’une largeur maximale de 1200 pixels. L’image en prend la moitié moins les marges).
  • si la fenêtre d’affichage est comprise entre 640px et 1200px, l’image prend 48 % de la fenêtre d’affichage (par exemple, l’image s’adapte à notre page et prend la moitié de la largeur de la fenêtre d’affichage moins les marges).
  • si la fenêtre d’affichage est d’une autre taille, dans notre cas moins de 640px, l’image prend 98% de la fenêtre d’affichage (par exemple, l’image s’adapte à notre page et prend toute la largeur de la fenêtre d’affichage moins les marges). La condition du support doit être omise pour le dernier article.

srcset indique simplement au navigateur quelles images sont disponibles et quelles sont leurs tailles.

  • img/hello-300.jpg fait 300px de large,
  • img/hello-600.jpg fait 600px de large,
  • img/hello-900.jpg fait 900px de large,
  • img/hello-1200.jpg fait 1200px de large

src est toujours une source d’image obligatoire. En cas d’utilisation avec srcset, src servira d’image de secours au cas où le navigateur ne prend pas en charge srcset.

Utilisation de srcset sans tailles #

<img src="img/hello-300.jpg" alt="hello"
     srcset="img/hello-300.jpg 1x, 
       img/hello-600.jpg 2x,
       img/hello-1200.jpg 3x">

srcset fournit la liste des images disponibles, avec le descripteur x du rapport périphérique-pixel.

  • si le rapport appareil-pixel est de 1, utilisez img/hello-300.jpg
  • si le rapport appareil-pixel est de 2, utilisez img/hello-600.jpg
  • si le rapport appareil-pixel est de 3, utilisez img/hello-1200.jpg

src est toujours une source d’image obligatoire. En cas d’utilisation avec srcset, src servira d’image de secours au cas où le navigateur ne prend pas en charge srcset.

## Image réactive utilisant un élément d’image Code

<picture>
  <source media="(min-width: 600px)" srcset="large_image.jpg">
  <source media="(min-width: 450px)" srcset="small_image.jpg">
  <img src="default_image.jpg" style="width:auto;">
</picture>

Usage

Pour afficher différentes images sous différentes largeurs d’écran, vous devez inclure toutes les images utilisant la balise source dans une balise image, comme indiqué dans l’exemple ci-dessus.

Résultat

  • Sur les écrans avec une largeur d’écran> 600px, il affiche large_image.jpg
  • Sur les écrans avec une largeur d’écran> 450px, il affiche small_image.jpg
  • Sur les écrans avec une autre largeur d’écran, il affiche default_image.jpg

Largeur et hauteur de l’image #

Remarque : Les attributs width et height ne sont pas obsolètes sur les images et ne l’ont jamais été. Leur utilisation a cependant été rendue beaucoup plus stricte.

Les dimensions d’une image peuvent être spécifiées à l’aide des attributs width et height de la balise image :

<img src="images/200x200-img.png" width="200" height="200" alt="A 200x200 image">

En spécifiant la largeur et la hauteur d’une image, votre structure donne au navigateur une indication sur la façon dont la page doit être mise en page même si vous ne spécifiez que la taille réelle de l’image. Si les dimensions de l’image ne sont pas spécifiées, le navigateur devra recalculer la mise en page de la page après le chargement de l’image, ce qui peut faire “sauter” la page pendant son chargement.

Vous pouvez donner à l’image une largeur et une hauteur en nombre de pixels CSS ou en pourcentage des dimensions réelles de l’image.

Ces exemples sont tous valables :

<img src="images/50x50-img.png" width="50" height="50" alt="A 50x50 image">
<img src="images/50x50-img.png" width="200" height="200" alt="A 200x200 image">
<img src="images/200x200-img.png" width="50" height="50" alt="A 50x50 image">
<img src="images/200x200-img.png" width="50%" height="50%" alt="A 100x100 image">

La largeur et la hauteur de l’image doivent être spécifiées en pixels CSS ; une valeur en pourcentage n’est plus une valeur valide. De plus, si les deux attributs sont spécifiés, ils doivent correspondre à l’une des trois formules qui conservent les proportions . Bien que valides, vous ne devez pas utiliser les attributs width et height pour étirer une image à une taille plus grande.

Ces exemples sont valables :

<img src="images/50x50-img.png" width="50" height="50" alt="A 50x50 image">
<img src="images/200x200-img.png" width="50" height="50" alt="A 50x50 image">
<img src="images/50x50-img.png" width="0" height="0" alt="A hidden tracking image">

Cet exemple n’est pas recommandé :

<img src="images/50x50-img.png" width="200" height="200" alt="A 200x200 image">

Ces exemples ne sont pas valides :

<img src="images/200x200-img.png" width="50%" height="50%" alt="A 100x100 image">
<img src="images/200x200-img.png" width="1" height="200" alt="A 1x200 image">
