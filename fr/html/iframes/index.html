Paramètres #

Attribut Détails
nom Définit le nom de l’élément, à utiliser avec une balise a pour changer le src de l’iframe.
largeur Définit la largeur de l’élément en pixels.
hauteur Définit la hauteur de l’élément en pixels.
src Spécifie la page qui sera affichée dans le cadre.
srcdoc Spécifie le contenu qui sera affiché dans le cadre, en supposant que le navigateur le prend en charge. Le contenu doit être du HTML valide.
bac à sable Lorsqu’il est défini, le contenu de l’iframe est traité comme provenant d’une origine unique et les fonctionnalités, y compris les scripts, les plugins, les formulaires et les popups, seront désactivées. Les restrictions peuvent être assouplies de manière sélective en ajoutant une liste de valeurs séparées par des espaces. Voir le tableau dans Remarques pour les valeurs possibles.
allowfullscreen S’il faut autoriser le contenu de l’iframe à utiliser requestFullscreen()

Un iframe est utilisé pour incorporer un autre document dans le document HTML actuel.

Vous POUVEZ utiliser des iframes pour afficher :

  • d’autres pages HTML sur le même domaine ;
  • d’autres pages HTML sur un autre domaine (voir ci-dessous - Politique de même origine) ;
  • Documents PDF (bien qu’IE puisse avoir quelques problèmes, [Cette question SO][1] pourrait aider);

Vous DEVEZ utiliser une iframe en dernier recours, car elle a des problèmes avec les signets et la navigation, et il y a toujours de meilleures options autres qu’une iframe. [Cette question SO][2] devrait vous aider à mieux comprendre les hauts et les bas des iframes.

Politique de même origine #

Certains sites ne peuvent pas être affichés à l’aide d’un iframe, car ils appliquent une politique appelée [Same-origin policy][3]. Cela signifie que le site sur lequel se trouve l’iframe doit être sur le même domaine que celui à afficher.

Cette politique s’applique également à la manipulation de contenu qui vit à l’intérieur d’un iFrame. Si l’iFrame accède au contenu d’un domaine différent, vous ne pourrez pas accéder ou manipuler le contenu à l’intérieur d’un iFrame.

L’élément iframe sur [W3C][4]

Attribut sandbox #

L’attribut sandbox, lorsqu’il est défini, ajoute des restrictions supplémentaires à l’iframe. Une liste de jetons séparés par des espaces peut être utilisée pour assouplir ces restrictions.

Valeur Détails
allow-forms Permet de soumettre des formulaires.
allow-pointer-lock Active l’API de pointeur JavaScript.
allow-popups Les popups peuvent être créés en utilisant window.open ou <a target="_blank"
autoriser-même-origine Le document iframe utilise sa véritable origine au lieu d’en recevoir une unique. S’il est utilisé avec allow-scripts, le document iframe peut supprimer tout sandboxing s’il provient de la même origine que le document parent.
allow-scripts Active les scripts. Le document iframe et le document parent peuvent communiquer entre eux à l’aide de l’API postMessage(). S’il est utilisé avec allow-same-origin, le document iframe peut supprimer tout sandboxing s’il provient de la même origine que le document parent.
allow-top-navigation Permet au contenu de l’iframe de modifier l’emplacement du document de niveau supérieur.

[1] : http://stackoverflow.com/questions/12974115/how-to-open-a-pdf-file-in-an-iframe [2] : http://stackoverflow.com/questions/362730/are-iframes-considered-bad-practice [3] : https://en.wikipedia.org/wiki/Same-origin_policy [4] : https://www.w3.org/TR/html5/embedded-content-0.html#the-iframe-element

## Sandbox Ce qui suit intègre une page Web non approuvée avec toutes les restrictions activées

<iframe sandbox src="http://example.com/"></iframe>

Pour permettre à la page d’exécuter des scripts et de soumettre des formulaires, ajoutez allow-scripts et allow-forms à l’attribut sandbox.

<iframe sandbox="allow-scripts allow-forms" src="http://example.com/"></iframe>

S’il existe un contenu non fiable (tel que des commentaires d’utilisateurs) sur le même domaine que la page Web parente, un iframe peut être utilisé pour désactiver les scripts tout en permettant au document parent d’interagir avec son contenu à l’aide de JavaScript.

<iframe sandbox="allow-same-origin allow-top-navigation" src="http://example.com/untrusted/comments/page2">

Le document parent peut ajouter des écouteurs d’événement et redimensionner l’IFrame pour l’adapter à son contenu. Ceci, avec allow-top-navigation, peut donner l’impression que l’iframe en bac à sable fait partie du document parent.

Ce bac à sable ne remplace pas la désinfection des entrées, mais peut être utilisé dans le cadre d’une stratégie de défense en profondeur.

Sachez également que ce bac à sable peut être contourné par un attaquant convainquant un utilisateur de visiter directement la source de l’iframe. L’en-tête HTTP Content Security Policy peut être utilisé pour atténuer cette attaque.

Réglage de la taille du cadre #

L’IFrame peut être redimensionnée à l’aide des attributs width et height, où les valeurs sont représentées en pixels (HTML 4.01 autorise les valeurs en pourcentage, mais HTML 5 n’autorise que les valeurs en pixels CSS).

<iframe src="base.html" width="800" height="600"></iframe>

Principes de base d’un cadre en ligne #

Le terme “IFrame” signifie Inline Frame. Il peut être utilisé pour inclure une autre page dans votre page. Cela donnera un petit cadre qui montrera le contenu exact de base.html.

<iframe src="base.html"></iframe>

Utilisation de l’attribut “srcdoc” #

L’attribut srcdoc peut être utilisé (au lieu de l’attribut src) pour spécifier le contenu exact de l’iframe en tant que document HTML complet. Cela donnera un IFrame avec le texte “Les IFrames sont cool!”

<iframe srcdoc="<p>IFrames are cool!</p>"></iframe>

Si l’attribut srcdoc n’est pas pris en charge par le navigateur, l’IFrame utilisera à la place l’attribut src, mais si les attributs src et srcdoc sont présents et pris en charge par le navigateur, srcdoc est prioritaire.

<iframe srcdoc="<p>Iframes are cool!</p>" src="base.html"></iframe>

Dans l’exemple ci-dessus, si le navigateur ne prend pas en charge l’attribut srcdoc, il affichera à la place le contenu de la page base.html.

Utiliser des ancres avec des IFrames #

Normalement, un changement de page Web dans une Iframe est initié à partir de l’Iframe, par exemple, en cliquant sur un lien à l’intérieur de l’Iframe. Cependant, il est possible de modifier le contenu d’une IFrame depuis l’extérieur de l’IFrame. Vous pouvez utiliser une balise d’ancrage dont l’attribut href est défini sur l’URL souhaitée et dont l’attribut target est défini sur l’attribut name de l’iframe.

<iframe src="webpage.html" name="myIframe"></iframe>
<a href="different_webpage.html" target="myIframe">Change the Iframe content to different_webpage.html</a>
