Syntaxe #

  • <element data-custom-name="unevaleur">

Paramètres #

Valeur Descriptif
une valeur Spécifie la valeur de l’attribut (sous forme de chaîne)

## Utilisation des attributs de données Les attributs HTML5 data-* offrent un moyen pratique de stocker des données dans des éléments HTML. Les données stockées peuvent être lues ou modifiées à l’aide de JavaScript

<div data-submitted="yes" class="user_profile">
  … some content …
</div>

  • La structure de l’attribut de données est data-*, c’est-à-dire que le nom de l’attribut de données vient après la partie data-. A l’aide de ce nom, l’attribut est accessible.

  • Les données au format chaîne (y compris json) peuvent être stockées à l’aide de l’attribut data-*.

Prise en charge des anciens navigateurs #

Les attributs de données ont été introduits dans HTML5 qui est pris en charge par tous les navigateurs modernes, mais les anciens navigateurs antérieurs à HTML5 ne reconnaissent pas les attributs de données.

Cependant, dans les spécifications HTML, les attributs qui ne sont pas reconnus par le navigateur doivent être laissés seuls et le navigateur les ignorera simplement lors du rendu de la page.

Les développeurs Web ont utilisé ce fait pour créer des attributs non standard qui sont des attributs ne faisant pas partie des spécifications HTML. Par exemple, l’attribut value dans la ligne ci-dessous est considéré comme un attribut non standard car les spécifications de la balise <img> n’ont pas d’attribut value et ce n’est pas un attribut global :

<img src="sample.jpg" value="test" />

Cela signifie que bien que les attributs de données ne soient pas pris en charge dans les anciens navigateurs, ils fonctionnent toujours et vous pouvez les définir et les récupérer en utilisant les mêmes méthodes JavaScript génériques setAttribute et getAttribute, mais vous ne pouvez pas utiliser la nouvelle propriété dataset qui est uniquement pris en charge dans les navigateurs modernes.
