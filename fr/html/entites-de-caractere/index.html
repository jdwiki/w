Caractères spéciaux communs #

Certains caractères peuvent être réservés au HTML et ne peuvent pas être utilisés directement car ils peuvent obstruer les codes HTML réels. Par exemple, essayer d’afficher les crochets angulaires gauche et droit (<>) dans le code source peut entraîner des résultats inattendus dans la sortie. De même, les espaces blancs tels qu’ils sont écrits dans le code source peuvent ne pas s’afficher comme prévu dans le code HTML de sortie. Certains, comme ☎, ne sont pas disponibles dans le jeu de caractères ASCII.

A cet effet, des entités de caractères sont créées. Ceux-ci sont de la forme &entity_name; ou &entity_number;. Voici quelques-unes des entités HTML disponibles.

Personnage Descriptif Nom de l’entité Numéro d’entité
“&nbsp ;” espace insécable &nbsp; &#160;
“<” moins de &lt; &#60;
“>” supérieur à &gt; &#62;
“&amp ;” esperluette &amp; &#38;
« &mdash ; » em tiret &mdash; &#8212;
“–” en tiret &ndash; &#8211;
“&copie;” droit d’auteur &copie; &# 169;
“®” marque déposée &reg; &#174;
“&Commerce;” marque déposée &trade; &#8482;
“&téléphoner;” téléphone &phone; &#9742;

Ainsi, écrire

&copie; 2016 Stack Exchange Inc.

le code HTML suivant est utilisé :

<b>&copy; 2016 Stack Exchange Inc.</b>

Entités de caractère en HTML #

De nombreux symboles et caractères spéciaux sont nécessaires lors du développement d’une page Web en html, mais comme nous le savons, l’utilisation directe de caractères peut parfois interférer avec le code html réel qui a certains caractères réservés et certains caractères non disponibles sur le clavier. Ainsi, pour éviter le conflit et en même temps pouvoir utiliser différents symboles dans notre code, w3 org nous fournit des ’entités de caractère'.

Les entités de caractère sont prédéfinies avec ‘Entity Name’ - &entity_name; et ‘Numéro d’entité’ - &entity_number; nous devons donc utiliser l’un des deux pour que le symbole requis soit rendu sur notre page.

La liste de quelques entités de personnage peut être trouvée sur https://dev.w3.org/html5/html-author/charref

Un exemple simple avec l’utilisation de l’entité caractère pour ’loupe’ :

<input type="text" placeholder="  &#128269; Search"/>
