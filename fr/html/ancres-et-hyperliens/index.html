Les balises d’ancrage sont couramment utilisées pour lier des pages Web distinctes, mais elles peuvent également être utilisées pour créer des liens entre différents endroits dans un même document, souvent dans la table des matières ou même lancer des applications externes. Cette rubrique explique l’implémentation et l’application des balises d’ancrage HTML dans divers rôles.

Syntaxe #

  • <a href="URL ou ancre">Texte du lien</a>

Paramètres #

Paramètre Détails
href Spécifie l’adresse de destination. Il peut s’agir d’une URL absolue ou relative, ou du “nom” d’une ancre. Une URL absolue est l’URL complète d’un site Web comme http://example.com/. Une URL relative pointe vers un autre répertoire et/ou document à l’intérieur du même site Web, par ex. /about-us/ pointe vers le répertoire « about-us » à l’intérieur du répertoire racine (/). Lorsqu’ils pointent vers un autre répertoire sans spécifier explicitement le document, les serveurs Web renvoient généralement le document “index.html” à l’intérieur de ce répertoire.
hreflang Spécifie la langue de la ressource liée par l’attribut href (qui doit être présent avec celui-ci). Utilisez les valeurs de langue de BCP 47 pour HTML5 et RFC 1766 pour HTML 4.
rel Spécifie la relation entre le document actuel et le document lié. Pour HTML5, les valeurs doivent être définies dans la spécification ou [enregistrées dans le wiki Microformats](http ://microformats.org/wiki/existing-rel-values#HTML5_link_type_extensions).
cible Spécifie où ouvrir le lien, par ex. dans un nouvel onglet ou une nouvelle fenêtre. Les valeurs possibles sont _blank, _self, _parent, _top et framename (obsolète). Forcer un tel comportement n’est pas recommandé car cela viole le contrôle de l’utilisateur sur un site Web.
titre Spécifie des informations supplémentaires sur un lien. Les informations sont le plus souvent affichées sous forme de texte d’info-bulle lorsque le curseur se déplace sur le lien. Cet attribut n’est pas limité aux liens, il peut être utilisé sur presque toutes les balises HTML.
télécharger Spécifie que la cible sera téléchargée lorsqu’un utilisateur cliquera sur le lien hypertexte. La valeur de l’attribut sera le nom du fichier téléchargé. Il n’y a aucune restriction sur les valeurs autorisées, et le navigateur détectera automatiquement l’extension de fichier correcte et l’ajoutera au fichier (.img, .pdf, etc.). Si la valeur est omise, le nom de fichier d’origine est utilisé.

Lien vers un autre site #

C’est l’utilisation de base de l’élément [<a> (anchor element)][1] :

<a href="http://example.com/">Link to example.com</a>

Il crée un lien hypertexte vers l’URL http://example.com/ comme spécifié par l’attribut href (référence hypertexte), avec le texte d’ancrage “Lien vers exemple.com”. Cela ressemblerait à quelque chose comme ceci :

[Lien vers example.com][2]

Pour indiquer que ce lien mène à un site Web externe, vous pouvez utiliser le type de lien “externe” :

<a href="http://example.com/" rel="external">example site</a>

Vous pouvez créer un lien vers un site qui utilise un protocole autre que HTTP. Par exemple, pour créer un lien vers un site FTP, vous pouvez faire,

<a href="ftp://example.com/">This could be a link to a FTP site</a>

Dans ce cas, la différence est que cette balise d’ancrage demande au navigateur de l’utilisateur de se connecter à “example.com” à l’aide du protocole de transfert de fichiers (FTP) plutôt que du protocole de transfert hypertexte (HTTP).

Il peut s’agir d’un lien vers un site FTP

[1] : https://developer.mozilla.org/en-US/docs/Web/HTML/Element/a [2] : http://exemple.com

Lien vers une ancre #

Les ancres peuvent être utilisées pour accéder à des balises spécifiques sur une page HTML. La balise <a> peut pointer vers n’importe quel élément qui a un attribut id. Pour en savoir plus sur les identifiants, consultez la [documentation sur les classes et les identifiants][1]. Les ancres sont principalement utilisées pour accéder à une sous-section d’une page et sont utilisées conjointement avec les balises d’en-tête.

Supposons que vous ayez créé une page (page1.html) sur de nombreux sujets :

<h2>First topic</h2>
<p>Content about the first topic</p>
<h2>Second topic</h2>
<p>Content about the second topic</p>

Une fois que vous avez plusieurs sections, vous pouvez créer une table des matières en haut de la page avec des liens rapides (ou des signets) vers des sections spécifiques.

Si vous avez donné un attribut id à vos sujets, vous pourriez alors créer un lien vers eux

<h2 id="Topic1">First topic</h2>
<p>Content about the first topic</p>
<h2 id="Topic2">Second topic</h2>
<p>Content about the second topic</p>

Vous pouvez maintenant utiliser l’ancre dans votre table des matières :

<h1>Table of Contents</h1>
    <a href='#Topic1'>Click to jump to the First Topic</a>
    <a href='#Topic2'>Click to jump to the Second Topic</a>

Ces ancres sont également attachées à la page Web sur laquelle elles se trouvent (page1.html). Ainsi, vous pouvez créer un lien sur le site d’une page à l’autre en faisant référence à la page * et * au nom de l’ancre.

 Remember, you can always <a href="page1.html#Topic1">look back in the First Topic</a> for supporting information.

[1] : https://www.wikiod.com/fr/html/classes-et-identifiants

Lien vers une page du même site #

Vous pouvez utiliser un [chemin relatif][1] pour créer un lien vers des pages du même site Web.

<a href="/example">Text Here</a>

L’exemple ci-dessus irait dans le fichier example dans le répertoire racine (/) du serveur.

Si ce lien était sur http://example.com, les deux liens suivants amèneraient l’utilisateur au même emplacement

<a href="/page">Text Here</a>
<a href="http://example.com/page">Text Here</a>

Les deux éléments ci-dessus iraient dans le fichier page dans le répertoire racine de example.com.

[1] : https://en.wikipedia.org/wiki/Path_(informatique)#Absolute_and_relative_paths

Lien qui compose un numéro #

Si la valeur de l’attribut href commence par tel:, votre appareil composera le numéro lorsque vous cliquerez dessus. Cela fonctionne sur les appareils mobiles ou sur les ordinateurs/tablettes exécutant des logiciels - comme Skype ou FaceTime - qui peuvent passer des appels téléphoniques.

<a href="tel:11234567890">Call us</a>

La plupart des appareils et des programmes invitent l’utilisateur d’une manière ou d’une autre à confirmer le numéro qu’il est sur le point de composer.

Ouvrir le lien dans un nouvel onglet/une nouvelle fenêtre #

<a href="example.com" target="_blank">Text Here</a>

L’attribut target spécifie où ouvrir le lien. En le définissant sur _blank, vous indiquez au navigateur de l’ouvrir dans un nouvel onglet ou une nouvelle fenêtre (selon les préférences de l’utilisateur).

AVERTISSEMENT DE VULNÉRABILITÉ DE SÉCURITÉ !

L’utilisation de target="_blank" donne au site d’ouverture un accès partiel à l’objet window.opener via JavaScript, ce qui permet à cette page d’accéder ensuite et de modifier le window.opener.location de votre page et potentiellement rediriger les utilisateurs vers des sites malveillants ou de phishing.

Chaque fois que vous l’utilisez pour des pages que vous ne contrôlez pas, ajoutez rel="noopener" à votre lien pour empêcher l’envoi de l’objet window.opener avec la requête.

Actuellement, Firefox ne prend pas en charge noopener, vous devrez donc utiliser rel="noopener noreferrer" pour un effet maximal.

Lien qui exécute JavaScript #

Utilisez simplement le protocole javascript: pour exécuter le texte en tant que JavaScript au lieu de l’ouvrir comme un lien normal :

<a href="javascript:myFunction();">Run Code</a>

Vous pouvez également obtenir la même chose en utilisant l’attribut onclick :

<a href="#" onclick="myFunction(); return false;">Run Code</a>

Le return false; est nécessaire pour empêcher votre page de défiler vers le haut lorsque le lien vers # est cliqué. Assurez-vous d’inclure tout le code que vous souhaitez exécuter avant, car le retour arrêtera l’exécution de code supplémentaire.

A noter également, vous pouvez inclure un point d’exclamation ! après le hashtag afin d’empêcher la page de défiler vers le haut. Cela fonctionne car tout slug invalide empêchera le lien de défiler partout sur la page, car il ne pourra pas localiser l’élément auquel il fait référence (un élément avec id="!"). Vous pouvez également utiliser n’importe quel slug invalide (tel que #scrollsNowhere) pour obtenir le même effet. Dans ce cas, return false; n’est pas obligatoire :

<a href="#!" onclick="myFunction();">Run Code</a>

** Devriez-vous utiliser tout cela ? **

La réponse est presque certainement non. Exécuter JavaScript en ligne avec l’élément comme celui-ci est une assez mauvaise pratique. Envisagez d’utiliser des solutions JavaScript pures qui recherchent l’élément dans la page et lui lient une fonction à la place. https://www.wikiod.com/fr/javascript

Considérez également si cet élément est vraiment un bouton au lieu d’un lien. Si c’est le cas, vous devez utiliser <bouton>.

Lien qui exécute le client de messagerie #

Utilisation de base

Si la valeur de l’attribut href commence par mailto:, il essaiera d’ouvrir un client de messagerie en cliquant :

<a href="mailto:example@example.com">Send email</a>

Cela placera l’adresse e-mail example@example.com comme destinataire de l’e-mail nouvellement créé.

Cc et Cci

Vous pouvez également ajouter des adresses pour les destinataires cc ou bcc en utilisant la syntaxe suivante :

<a href="mailto:example@example.com?cc=john@example.com&bcc=jane@example.com">Send email</a>

Objet et corps du texte

Vous pouvez également renseigner l’objet et le corps du nouvel e-mail :

<a href="mailto:example@example.com?subject=Example+subject&body=Message+text">Send email</a>

Ces valeurs doivent être [URL encodées] (https://en.wikipedia.org/wiki/Percent-encoding).

Cliquer sur un lien avec mailto: essaiera d’ouvrir le client de messagerie par défaut spécifié par votre système d’exploitation ou il vous demandera de choisir le client que vous souhaitez utiliser. Toutes les options spécifiées après l’adresse du destinataire ne sont pas prises en charge dans tous les clients de messagerie.
