Exécuter Parse Server localement (exemple le plus rapide) #

$ npm install -g parse-server mongodb-runner
$ mongodb-runner start
$ parse-server --appId APPLICATION_ID --masterKey MASTER_KEY --databaseURI mongodb://localhost/test

Vous pouvez utiliser n’importe quelle chaîne arbitraire comme ID d’application et clé principale. Ceux-ci seront utilisés par vos clients pour s’authentifier auprès du serveur d’analyse.

C’est ça! Vous exécutez maintenant une version autonome de Parse Server sur votre ordinateur, avec également une base de données Mongo hébergée localement.

Enregistrer votre premier objet #

Maintenant que vous exécutez Parse Server, il est temps d’enregistrer votre premier objet. Nous utiliserons l’API REST, mais vous pouvez facilement faire la même chose en utilisant l’un des [SDK d’analyse](https://parseplatform.github.io /#sdks). Exécutez ce qui suit :

curl -X POST \
-H "X-Parse-Application-Id: APPLICATION_ID" \
-H "Content-Type: application/json" \
-d '{"score":1337,"playerName":"Sean Plott","cheatMode":false}' \
http://localhost:1337/parse/classes/GameScore

Vous devriez obtenir une réponse semblable à celle-ci :

{
  "objectId": "2ntvSpRGIK",
  "createdAt": "2016-03-11T23:51:48.050Z"
}

Serveur d’analyse + Express #

Vous pouvez également créer une instance de Parse Server et la monter sur un site Web Express nouveau ou existant :

var express = require('express');
var ParseServer = require('parse-server').ParseServer;
var app = express();

var api = new ParseServer({
  databaseURI: 'mongodb://localhost:27017/dev', // Connection string for your MongoDB database
  cloud: '/home/myApp/cloud/main.js', // Absolute path to your Cloud Code
  appId: 'myAppId',
  masterKey: 'myMasterKey', // Keep this key secret!
  fileKey: 'optionalFileKey',
  serverURL: 'http://localhost:1337/parse' // Don't forget to change to https if needed
});

// Serve the Parse API on the /parse URL prefix
app.use('/parse', api);

app.listen(1337, function() {
  console.log('parse-server-example running on port 1337.');
});
