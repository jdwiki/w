Les informats et les formats sont utilisés pour dire à SAS comment lire et écrire les données respectivement. Les informats sont couramment utilisés dans une étape de données lors de la lecture de données à partir d’un fichier externe. Les informats sont rarement utilisés dans les PROC. Les formats sont couramment utilisés dans les étapes de données et les PROC.

Les formats SAS convertissent les valeurs numériques ou alphanumériques en valeurs alphanumériques. Un format peut soit être appliqué en utilisant une instruction format ou put, qui change la façon dont une valeur est affichée, ou en utilisant la fonction put pour stocker la valeur formatée dans une nouvelle variable.

Il existe quatre catégories de formats :

  • Caractère - demande à SAS d’écrire des valeurs de données de caractères à partir de variables de caractère.
  • Date et heure - ordonne à SAS d’écrire des données des valeurs de variables qui représentent des dates, des heures et des datetimes.
  • ISO 8601 - demande à SAS d’écrire les valeurs de date, d’heure et de date-heure à l’aide de la norme ISO 8601.
  • Numérique - demande à SAS d’écrire des valeurs de données numériques à partir de variables numériques.

Les formats prennent généralement la forme <formatname><w>.<d>;, w étant la largeur (y compris les décimales et le point), d étant le nombre de décimales.

Formats de date courants (appliqués aux valeurs de date SAS) :

  • date9. par ex. 02AUG2016
  • ddmmyyn8. par ex. 02082016
  • jjmmaa8. par ex. 02/08/16
  • aammjj10. par ex. 20160802
  • année4. par ex. 2016

Formats numériques courants (appliqués aux nombres) :

  • comma11.0 par ex. 1 234 567
  • comma12.2 par ex. 1 234 567,00
  • dollar11.2 par ex. 5 789,12 $
  • nlmnlgbp11.2 par ex. 2 468,02 £

Autres formats :

  • $hex8., convertit la chaîne en hexadécimal
  • $upcase., convertit la chaîne en majuscule
  • $quote., mettez une chaîne entre guillemets

Une liste complète des formats peut être trouvée ici > https://support.sas.com/documentation/cdl/en/lrdict/64316/HTML/default/viewer.htm#a001263753.htm

Utilisation de l’instruction de format #

L’instruction format applique le format donné à la variable spécifiée à des fins d’affichage uniquement, c’est-à-dire que la valeur sous-jacente ne change pas.

<pré> exemple de données1 ; Date = ‘02AUG2016’d ; /* stocké sous forme de date SAS, c’est-à-dire un nombre */ Date2 = ‘31AUG2016’d ; format Date mona7. Date2 aammjj8. ; Cours ;

Rendez-vous Date2
AOÛT2016 20160831

Utilisation de l’instruction de format pour regrouper les données #

Vous pouvez appliquer des formats dans une procédure, par ex. pour changer les groupements dans un proc summary ou proc freq.

Regroupement des dates SAS

<pré> exemple de données2 ; faire Date = ‘01JUN2016’dt à ‘31AUG2016’dt ; Days = 1 ; output ; fin ; Cours ;

/* Résumer par année et mois / proc summary data=example2 nway ; date de classe ; était des jours ; sortie out=example2_sum (drop=TYPE FREQ) sum= ; format Date yymmn6. ; / par exemple. 201606 */ Cours ;

Rendez-vous Jours
201606 30
201607 31
201608 31

<pré> /* Résumer par mois et année / proc summary data=example2 nway ; date de classe ; était des jours ; sortie out=example2_sum2 (drop=TYPE FREQ) sum= ; format Date mona7. ; / par exemple. JUIN2016 */ Cours ;

Rendez-vous Jours
JUIN2016 30
NOËL2016 31
AOÛT2016 31

L’avantage d’utiliser un format est que l’ordre de tri naturel est conservé.

En utilisant sashelp.class comme exemple, supposons que vous vouliez comparer la fréquence de la première lettre de chaque nom. Vous pouvez utiliser la fonction substr() pour trouver la première lettre et exécuter un proc freq sur la nouvelle variable. Alternativement, vous pouvez appliquer le format $1. à la variable Name : <pré> proc freq data=sashelp.class ; nom de la table ; format Nom $1. ; Cours ;

Nom COMPTER
Un 7
B 4
C 2
etc

Formats personnalisés #

Les formats personnalisés, également appelés formats définis par l’utilisateur, peuvent être créés et utilisés comme n’importe quel autre format par défaut.

/*Create new character format for state variables*/
PROC FORMAT;
VALUE $statef      'CA' = 'California'
                   'MA' = 'Massachusetts'
                   'NY' = 'New York';

/*Once created, you can use your custom format in PROC and DATA steps*/
PROC PRINT DATA=table;
FORMAT state-var $statef.;
RUN;

La variable state-var sera imprimée selon le nouveau format. Par exemple, la valeur 'CA' sera imprimée comme 'California'. Si une valeur n’a pas été formatée, telle que 'CT', alors cette valeur sera imprimée telle qu’elle apparaît dans l’ensemble de données.

Utilisation d’informats pour lire des données #

Les informats sont utilisés pour indiquer à SAS comment lire les données et sont identifiés par une déclaration “informat”.

data test;
 infile test.csv;
 informat     id $6.
              date mmddyy10.
              cost comma10.2
 ;
 input @1 id
       @7 date
       @20 cost
 ;
run;

Les informats et les formats peuvent également être utilisés ensemble pour lire les données et les écrire dans un format différent, comme avec la variable de salaire ci-dessous :

DATA workers;
  informat first last $16.;
  informat salary 12.1;
  informat birthdate 8.;
  input 
    first $ 
    last $ 
    birthdate 
    salary;
  format salary dollar10.;
datalines;
John Smith 19810505 54998.5
Jane Doe 19950925 45884.5
Frank James 19600222 70000.5
Jamie Love 19630530 292000.5
;
run;
