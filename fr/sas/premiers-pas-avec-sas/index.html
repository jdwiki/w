Installation ou configuration #

SAS peut être exécuté dans le modèle client-serveur, en utilisant soit le client lourd Enterprise Guide ou le client léger SAS Studio (activé sur le Web), ou en mode « serveur local » où un système SAS entièrement fonctionnel est présent sur une machine locale (Windows ou bureau ou serveur Unix/Linux fonctionnant en mode interactif) et exécutez soit en mode Display Manager (le client local) soit via l’un des clients client-serveur ci-dessus (connecté au serveur installé localement).

L’installation de SAS est généralement effectuée par un administrateur SAS, qui installera le logiciel à partir d’un dépôt de logiciels personnalisé pour le site (et souvent fourni directement par SAS Institute).

Pour apprendre SAS, il existe également la version gratuite SAS University Edition, qui peut être installée gratuitement à des fins éducatives par n’importe qui sur un ordinateur Windows, Mac ou Unix/Linux. Il est disponible directement auprès de SAS, actuellement sur la page SAS University Edition, soit en exécutant une instance AWS (sur le niveau gratuit) ou en téléchargeant une machine virtuelle localement. Voir le guide d’installation sur SAS.com pour les instructions à jour, ou ci-dessous pour la version actuelle (juillet 2016) des instructions.

Pour l’installer localement, vous devez d’abord télécharger et installer Oracle Virtualbox 5.0 (Windows/Mac/Linux). Téléchargez ensuite la plus récente image disque SAS University Edition, qui fait environ 2 Go et nécessite la configuration d’un profil SAS.com.

Une fois que vous avez fait cela, vous devez configurer la machine virtuelle dans VirtualBox. Importez la VM SAS en tant qu’appliance (“Importer l’appliance” dans VirtualBox). Créez un dossier que SAS utilisera comme stockage local (afin que vous puissiez placer les fichiers dans un emplacement que SAS puisse les voir) et configurez-le en tant que dossier partagé dans la boîte de dialogue des paramètres de la machine. Configurez-le pour le montage automatique.

Ensuite, démarrez la machine virtuelle SAS, et une fois qu’elle est démarrée, vous pouvez vous connecter via votre navigateur Web, en vous connectant à http://localhost:10080/ si vous avez utilisé les paramètres par défaut.

Si vous rencontrez des problèmes, les Forums de la communauté SAS - Analytics U sont les forums des fournisseurs pour obtenir de l’aide ou demandez à un question sur [Stack Overflow] (https://stackoverflow.com).

Présentation de Base SAS #

SAS est un système intégré de solutions logicielles qui vous permet d’effectuer les tâches suivantes :

  • Saisie, récupération et gestion des données
  • rédaction de rapports et conception graphique
  • analyse statistique et mathématique
  • prévision d’activité et aide à la décision
  • recherche opérationnelle et gestion de projet
  • développement d’applications

La façon dont vous utilisez SAS dépend de ce que vous voulez accomplir. Certaines personnes utilisent de nombreuses fonctionnalités du Système SAS, et d’autres n’en utilisent que quelques-unes.

Bonjour le monde! #

En raison de la structure de SAS, il existe trois façons principales de créer “Hello World!” exemples:

  1. Dans une étape de données pour placer un message dans le journal SAS (_null_ indique qu’aucun jeu de données de sortie ne doit être créé) :

    data null; put “Hell” “o World!”; run;

  2. Dans une étape de données pour stocker “Hello World!” dans une variable (foo indique qu’un ensemble de données de sortie appelé foo doit être créé qui a) contient un seul enregistrement et b) contient une seule variable : bar, qui a une valeur de Hello World !):

    data foo ; bar=“Hello” ; put bar= “World!”; run ;

  3. Via le langage SAS Macro (en ‘code ouvert’ en dehors de toute étape de données). & identifie un appel à une variable de macro et . identifie la fin de la variable (si un caractère d’espace blanc n’est pas souhaité) :

    %let foo=Hello; %put &foo.o World!;

  4. Hybride : Utilisation d’une macro-variable dans une étape de données :

    %let foo=Hello;

    data null ; put “&foo World!”; run ;

## Architecture de serveur SAS Aperçu : Il existe généralement deux types de déploiements SAS :

  1. Installation SAS Foundation uniquement (BASE SAS). Ceci est généralement installé sur un PC. Il n’exécute aucun logiciel serveur.

  2. Déploiement planifié SAS pour leur architecture de serveur qui installera l’environnement de serveur SAS avec éventuellement tout logiciel client SAS.

Lequel de ceux-ci vous avez sera indiqué dans votre e-mail de commande de logiciel SAS en indiquant la planification ou la non-planification. Si vous effectuez une installation planifiée, vous aurez besoin d’un fichier de plan pour votre commande qui commence par votre topologie.

[Note d’installation 44320 : Utilisation des plans de déploiement lors d’une installation SAS®][1]

Architecture de serveur SAS

L’environnement du serveur SAS est divisé en 3 niveaux différents :

  1. Serveur(s) de métadonnées SAS - Le serveur de métadonnées SAS est responsable de la gestion de l’environnement du serveur SAS, y compris les bibliothèques, les utilisateurs et la configuration du serveur.

2. Serveur(s) d’application SAS - Le serveur d’application SAS est principalement un serveur de calcul à partir duquel vos clients lancent généralement des travaux.

3. SAS Middle Tier(s) = Le SAS Middle Tier est principalement votre niveau Web qui exécute vos applications Web.

4. Niveau client - Le niveau client correspond aux applications clientes de vos utilisateurs qu’ils utilisent pour se connecter à l’environnement, telles que SAS Enterprise Guide.

[Mémoire 363-2011| Comprendre l’anatomie d’un déploiement SAS® : qu’y a-t-il dans My Server Soup ? Mark Schneider, Donna Bennett et Connie Robison, SAS Institute Inc., Cary, Caroline du Nord][2]

Topologie :

Le niveau de métadonnées SAS, le niveau de serveur d’application SAS et le niveau intermédiaire SAS peuvent être installés sur un seul serveur de machine ou répartis sur plusieurs serveurs. Ceci est déterminé par le fichier de plan que vous avez, il doit répondre à la topologie souhaitée pour votre déploiement.

En règle générale, la plupart, sinon la totalité, du niveau client sont des applications Windows, de sorte que le niveau client se trouve sur les postes de travail de vos utilisateurs SAS. En option, ils pourraient également être installés sur le ou les serveurs s’ils sont basés sur Windows.

Gestion des versions #
