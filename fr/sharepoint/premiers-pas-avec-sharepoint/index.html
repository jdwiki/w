Installation de SharePoint 2016 pour une batterie de serveurs unique #

Introduction #

SharePoint 2016 est la version 16 de la famille de produits SharePoint. Il a été publié le 4 mai 2016. Cet exemple couvre l’installation de SharePoint 2016 à l’aide de la configuration de batterie de serveurs unique. Cette configuration couvre les bases de la configuration d’une batterie de serveurs SharePoint sans avoir besoin de plusieurs serveurs. Notez que les scénarios couverts par une batterie de serveurs unique sont généralement limités aux scénarios de développement et de très petite production.

Conditions #

Avant d’installer SharePoint, l’environnement de base doit être configuré. SharePoint stocke des documents ainsi que des métadonnées, des journaux, des applications personnalisées, des personnalisations et bien plus encore. Assurez-vous que vous disposez de suffisamment d’espace disque et de RAM disponible au-delà des exigences de base.

  • 4 cœurs sur un processeur compatible 64 bits
  • 12 - 24 Go de RAM (selon le test ou le déploiement de la production)
  • Disque dur de 80 Go pour le système
  • Disque dur de 100 Go comme deuxième disque
  • Serveur avec Windows Server 2012 R2 64 bits ou aperçu technique “Threshold”
  • SQL Server 2014 ou SQL Server 2016
  • .NET Framework 4.5.2 ou .NET Framework 4.6
  • Comptes d’ordinateurs joints à un domaine et de services agricoles délégués

Tous les autres prérequis peuvent être installés manuellement ou à l’aide du programme d’installation des prérequis SharePoint inclus avec l’installation de SharePoint.

Installation #

  • Exécutez le programme d’installation des prérequis ; il peut demander de redémarrer le serveur avant de continuer
  • Exécutez Setup.exe à partir de l’installation de SharePoint
  • Entrez la clé de licence
  • Acceptez le contrat de licence
  • Sélectionnez “Complete” dans l’onglet Type de serveur
  • L’installation devrait se terminer avec succès
  • Sur la page complète, laissez la case cochée à côté de l’assistant Exécuter la configuration du produit et cliquez sur Fermer

Configuration #

Si vous continuez à partir de l’étape précédente, l’assistant de configuration du produit SharePoint 2016 devrait s’ouvrir automatiquement. si la boîte n’apparaît pas ou si vous exécutez la configuration ultérieurement, ouvrez l’assistant de configuration en accédant à Démarrer -> Produits SharePoint 2016 -> Assistant de configuration du produit SharePoint 2016.

  • Cliquez sur suivant sur la page d’accueil
  • Une boîte de dialogue modale apparaîtra indiquant que certains services peuvent être redémarrés pendant la configuration ; rien n’a encore été installé, alors cliquez sur oui
  • Ajouter le serveur de base de données pour la ferme
    • Enter the name of the machine running SQL Server; in this case, it is the local machine
    • Enter the name of the Configuration database or keep the default name SharePoint_Config
    • Enter the username of the domain service user who will be accessing the database (in the form of DOMAIN\user) *Enter the password for the domain user
    • Click next when done
  • Entrez le mot de passe de la ferme ; cela sera utilisé lors de l’ajout de serveurs supplémentaires à la nouvelle batterie
  • Sélectionnez le rôle Single Server Farm
  • Configurez l’application Web de l’administration centrale (d’où SharePoint sera géré par les administrateurs de la batterie), sélectionnez le numéro de port et sélectionnez le type de fédération d’authentification (NTLM ou Negotate (Kerberose))
  • Passez en revue les paramètres sur les dernières pages et apportez les modifications nécessaires
  • Lorsque vous êtes prêt, lancez la configuration qui peut prendre quelques minutes
  • Une fois terminé, vous ouvrirez l’assistant qui vous permettra d’ouvrir le site de l’administration centrale
  • En cas d’échec, vous pouvez rechercher les journaux dans le dossier %COMMONPROGRAMFILES%\Microsoft Shared\Web Server Extensions\16\LOG

Configuration de la ferme #

Une fois l’application Web centrale, la base de données de configuration et l’administrateur central configurés, vous serez prêt à configurer la batterie de serveurs pour une utilisation par les utilisateurs ou le développement. Vous pouvez mettre en signet l’emplacement du site de l’administration centrale ou y accéder via un raccourci au même emplacement que l’assistant de configuration du produit.

  • Si vous démarrez la configuration plus tard, cliquez sur Quick Launch -> Configuration Wizards -> Farm Configuration Wizard
  • Si vous démarrez l’assistant à partir de l’étape d’installation, cliquez sur Démarrer l’assistant
  • Choisissez si vous souhaitez faire partie du programme d’amélioration client en cliquant sur Oui ou Non
  • Sur la page de configuration de la ferme, sélectionnez le compte de domaine qui exécutera les services d’arrière-plan sur la ferme
    • While this account may be the same as the database account, they may also be different for separation of roles and privileges
    • Enter the account as DOMAIN\user
  • Validez les services que vous souhaitez mettre à disposition sur la ferme sur la page Services
  • Créer la première collection de sites sur la batterie (cette étape peut être ignorée et effectuée ultérieurement)
    • Enter the site collection’s title, description, web address (usually the first site is at the server root), and the template
    • Most things can be changed (title, description) can be changed easily, but others like the web URL may take much more work to change; the template can also not be easily rolled back, but SharePoint allows a large amount of customizations that allows you to take any base template and convert the style and layout of the site
  • Lorsque vous avez terminé la configuration, cliquez sur Terminer

La batterie de serveurs et la première collection de sites sont maintenant configurées pour être utilisées.

Créer un composant WebPart avec SharePoint Framework #

[dev.office.com/sharepoint][1] est un excellent endroit pour démarrer avec SharePoint Framework.

SharePoint Framework est une approche moderne côté client du développement SharePoint initialement destinée à SharePoint Online dans Office 365. Les composants Web créés avec SharePoint Framework sont un nouveau type de composants Web et peuvent être mis à disposition pour être ajoutés à la fois aux pages SharePoint existantes et nouvelles pages SharePoint.

Il existe un excellent exemple hello world pour ce processus hébergé sur [Build your first SharePoint client-side web part (Hello World part 1)][2]. Tous les exemples sur dev.office.com sont disponibles pour les contributions de la communauté via github.

Les étapes de base de Hello World dans SharePoint Framework sont :

  1. Générez le squelette du projet avec [the Yeoman SharePoint Generator][3].

    yo @microsoft/SharePoint

  2. Editez le code généré dans l’éditeur de votre choix. La prise en charge de [Visual Studio Code][4] est solide sur toutes les plates-formes.

  3. Prévisualisez le composant WebPart à l’aide de gulp et du SharePoint Workbench local

    gulp serve

  4. Aperçu dans votre environnement SharePoint Online

Allez à l’URL suivante : ‘https://votre-site-sharepoint/_layouts/workbench.aspx’

[1] : https://dev.office.com/sharepoint [2] : https://dev.office.com/sharepoint/docs/spfx/web-parts/get-started/build-a-hello-world-web-part [3] : https://www.npmjs.com/package/@microsoft/generator-sharepoint [4] : https://code.visualstudio.com/

Journaux et journalisation ULS SharePoint #

Le service de journalisation unifiée SharePoint (ULS) fournit des fonctionnalités de support et de débogage pour les opérations et les développeurs. Comprendre comment lire les journaux est une première étape importante pour résoudre les problèmes.

Outillage #

Microsoft fournit [ULS Viewer] [1] pour faciliter la lecture des anciens journaux et des journaux en cours d’écriture pendant l’exécution de la batterie. Il peut également filtrer et appliquer un formatage aux journaux pour aider à affiner un problème.

Identifiant de corrélation #

Pour isoler un problème, il est utile de ne regarder qu’un identifiant de corrélation particulier. Chaque identifiant de corrélation est associé à une demande ou à une action de bout en bout du système (telle qu’un time jobber). S’il y a un problème avec une page Web en cours de rendu, localiser la demande dans les journaux ULS et l’isoler de l’ID de corrélation spécifique supprime tout le bruit des autres journaux, ce qui aide à identifier le problème.

Ajouter SPMonitoredScope à mon code #

Une façon de comprendre l’ajout d’une journalisation et d’une certaine surveillance des performances consiste à ajouter SPMonitoredScope à votre code.

using (new SPMonitoredScope("Feature Monitor"))
{
    // My code here 
}

Ce code enregistrera le début et la fin de vos requêtes ainsi que certaines données de performance. La création de votre propre moniteur personnalisé qui implémente ISPScopedPerformanceMonitor vous permet de définir le niveau de trace ou le temps d’exécution maximal pour un ensemble de code.

[1] : https://www.microsoft.com/en-us/download/details.aspx?id=44020
