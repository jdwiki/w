gestionnaire d’accès pubnub côté serveur #

PubNub Access Manager (PAM) étend le cadre de sécurité existant de PubNub en permettant aux développeurs de créer et d’appliquer un accès sécurisé aux canaux sur l’ensemble du réseau en temps réel de PubNub.

Access Manager vous permet de gérer des autorisations granulaires pour vos applications et flux de données en temps réel, de créer plusieurs niveaux d’autorisation, d’accorder et de révoquer l’accès et d’auditer l’accès des utilisateurs.

Pour utiliser Access Manager, vous devez activer Access Manager dans le tableau de bord d’administration. Une fois que vous avez activé Access Manager, vous devez accorder des autorisations avant que des données puissent être envoyées ou reçues.

PAM Server side Configuration

Pour que le côté client fonctionne correctement, le côté serveur doit d’abord émettre les autorisations appropriées pour un canal PAM donné ou une combinaison de groupe de canaux et de jeton d’authentification.

pour accorder ces autorisations, vous devez initialiser l’instance pubnub au moins avec vos clés d’abonnement et secrètes.

Exemple :

Étape 1. Configurer Pubnub : -

PNConfiguration pnConfiguration = new PNConfiguration();
        pnConfiguration.setSubscribeKey(SUBSCRIBE_KEY);
        pnConfiguration.setPublishKey(PUBLISH_KEY);;
        pnConfiguration.setSecretKey(SECRET_KEY);
        pnConfiguration.setSecure(true);
        pnConfiguration.setLogVerbosity(PNLogVerbosity.BODY);

Étape 2. Initialiser PubNub avec pnConfiguration

PubNub pubnub = new PubNub(pnConfiguration);

L’opération PAM se produit à trois niveaux

1. A global level (no auth key, and no channel/channel group is defined) 

2. A channel/channel group level (only a channel/channel group is defined) 

3. A channel/channel group and key level (where both the channel/channel     group and key are defined)

À tous ces niveaux, nous pouvons accorder, révoquer et auditer les autorisations. Ici, nous faisons la même chose au niveau du canal/groupe de canaux et de la clé d’authentification.

PAM  Grant

nous pouvons accorder une autorisation de lecture/écriture à auth_key sur des canaux ou des groupes de canaux spécifiques

Exemple:

Synchrone :

try {

    pubnub.grant().authKeys(Arrays.asList("auth1,auth2"))
    .channels(Arrays.asList("channel1,channel2")).read(true).write(true    ).ttl(0).sync();

} catch (PubNubException e) {
    e.printStackTrace();
}

De manière asynchrone :

pubNub.grant().channels(channels).authKeys(Arrays.asList(authKey)).read(true).write(true).manage(false).ttl(0) .async(nouveau PNCallback() {

@Override
public void onResponse(PNAccessManagerGrantResult result,
     PNStatus status) {
}});

PAM REVOKE : nous pouvons révoquer une autorisation à auth_key d’un canal ou de groupes de canaux spécifiques.

Syntaxe pour révoquer l’autorisation identique à accorder . Nous avons juste besoin de changer l’autorisation true en false.

try {

    pubnub.grant().authKeys(Arrays.asList("auth1,auth2"))
    .channels(Arrays.asList("channel1,channel2")).read(false).write( false    ).ttl(0).sync();

} catch (PubNubException e) {
    e.printStackTrace();
}

PAM Audit : nous pouvons auditer une autorisation donnée sur un canal/groupe de canaux spécifique ou sur une auth_key donnée sur un canal ou un groupe de canaux spécifique

Exemple:

pubnub.audit().channel("mycha").authKeys(Arrays.asList("a1")).async(new PNCallback<PNAccessManagerAuditResult>(){

    @Override
    public void onResponse(PNAccessManagerAuditResult result,
            PNStatus status) {
                
            }
    });

PAM Ajouter des chaînes dans des groupes : nous pouvons également ajouter des chaînes dans des groupes de chaînes

Exemple:

pubnub.addChannelsToChannelGroup().channelGroup("my_channel").channels(Arrays.asList("my_channel5"))
    .async(new PNCallback<PNChannelGroupsAddChannelResult>() {

            @Override
            public void onResponse(PNChannelGroupsAddChannelResult                                                                                                                                             result,PNStatus status) {
                
                
            }
});

Problème d'authentification côté client (403 Interdit):

S’il y a une erreur lors de l’exécution des opérations PAM, vous pouvez recevoir une erreur 403. Si c’est le cas, assurez-vous d’avoir défini la clé secrète correcte et que l’horloge de l’ordinateur émetteur est synchronisée avec NTP.

NTP  Setup

Network Time Protocol (NTP) est un protocole utilisé pour synchroniser les horloges des ordinateurs dans un réseau d’ordinateurs. NTP utilise le temps universel coordonné (UTC) pour synchroniser les heures d’horloge de l’ordinateur à une milliseconde, et parfois à une fraction de milliseconde.

Ici, nous devons scyn l’heure du serveur avec pubnub. Suivez l’étape pour le faire

Étape 1 Installation NTP

$ sudo apt-get update
$ sudo apt-get install ntp

Étape 2 Modifier ntp.conf

Replace these four with pubnub server 

server 0.ubuntu.pool.ntp.org 
server 1.ubuntu.pool.ntp.org 
server 2.ubuntu.pool.ntp.org 
server 3.ubuntu.pool.ntp.org

à

server 0.pubsub.pubnub.com
server 1.pubsub.pubnub.com
server 2.pubsub.pubnub.com
server 3.pubsub.pubnub.com

Étape 3 Redémarrez le service NTP

$ redémarrage du service sudo ntp

Subvention de gestion de groupe de canaux génériques - Java SDK v4 #

Lorsqu’il s’agit d’ajouter/supprimer des chaînes à/de vos groupes de chaînes, vous devez avoir l’autorisation “gérer” pour ces groupes de chaînes. Mais vous ne devez jamais accorder aux clients la permission de “gérer” les groupes de canaux auxquels ils s’abonneront. S’ils le faisaient, ils pourraient ajouter n’importe quelle chaîne de leur choix à leur groupe de chaînes et avoir instantanément un accès en lecture à cette chaîne.

C’est pourquoi votre serveur doit être la seule entité à disposer de l’autorisation “gérer”. Mais votre serveur devra avoir l’autorisation “gérer” pour chaque groupe de canaux afin qu’il puisse ajouter/supprimer des canaux vers/des groupes de canaux au nom de tous les clients.

Mais accorder “gérer” à chaque groupe de canaux peut être un peu fastidieux. Au lieu de cela, vous pouvez accorder “gérer” à tous les groupes de canaux (existants et à créer) dans une seule autorisation * générique *.

// init PubNub instance using PNConfiguration with the secret-key
PNConfiguration pnConfiguration = new PNConfiguration();
pnConfiguration.setSubscribeKey("my_subkey")
pnConfiguration.setPublishKey("my_pubkey");
// secret key allows server to `grant` permissions
pnConfiguration.setSecretKey("my_secretkey");
pnConfiguration.setSecure(true);
// set the the server's auth key
pnConfiguration.setAuthKey("server_authkey");
PubNub pubnub = new PubNub(pnConfiguration);

// grant read and manage using the channel group wildcard - ":" 
// with forever ttl (0) 
pubNub.grant()
    .channelGroups(Arrays.asList(":")) // colon (:) is channel group wildcard
    .manage(true) // add/remove channels to/from channel groups
    .read(true) // in case server needs to subscribe or do here-now on channel groups
    .ttl(0) // 0 = forever grant
    .async(new PNCallback<PNAccessManagerGrantResult>() {
        @Override
        public void onResponse(PNAccessManagerGrantResult result, PNStatus status) {
            // check status for success or failure of grant
        }
    });

À partir de là, votre serveur pourra ajouter/supprimer des canaux à/de n’importe quel groupe de canaux créé par votre application.
