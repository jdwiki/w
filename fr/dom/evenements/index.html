Paramètres #

Paramètre Descriptif
type String définit le nom de l’événement à écouter.
auditeur Function se déclenche lorsque l’événement se produit.
options Boolean pour définir la capture, si Object vous pouvez définir les propriétés suivantes dessus, notez que l’option object est faiblement prise en charge.
1. capturer Un booléen qui indique que les événements de ce type seront distribués à l’écouteur enregistré avant d’être distribués à n’importe quel EventTarget en dessous dans l’arborescence DOM.
2. une fois Un booléen indiquant que l’écouteur doit être appelé au plus une fois après avoir été ajouté. Si c’est vrai, l’écouteur sera automatiquement supprimé lorsqu’il sera appelé.
3. passif Un booléen indiquant que l’écouteur n’appellera jamais preventDefault(). Si c’est le cas, l’agent utilisateur doit l’ignorer et générer un avertissement de console.

Origine des événements

[![Les événements ne commencent pas à l’endroit où vous déclenchez l’événement.][1]][1]

Les événements ne commencent pas à l’endroit où vous déclenchez l’événement (un bouton par exemple).

Au lieu de cela

Il touche chaque élément sur son chemin et il informe chaque élément qu’un événement se produit. Les événements remontent également après avoir atteint leur destination, informant à nouveau les éléments de son apparition.

Capturer et bouillonner

Comme nous l’avons appris, les événements commencent par le haut de l’arborescence DOM, informent chaque nœud de son chemin jusqu’à sa destination, puis remontent lorsqu’il atteint sa destination, informant également chaque élément qu’il touche en montant de son occurrence.

Les événements qui descendent dans l’arborescence DOM sont dans la phase de capture, les événements qui remontent dans l’arborescence DOM sont dans la phase de bouillonnement.

Par défaut, les événements sont écoutés dans la phase de bouillonnement. Pour changer cela, vous pouvez spécifier la phase à laquelle l’événement est écouté en spécifiant le troisième paramètre dans la fonction addEventListener. (exemple de code dans la section capture)

Bullage et capture d’événements #

Les événements déclenchés sur les éléments DOM n’affectent pas seulement l’élément qu’ils ciblent. N’importe lequel des ancêtres de la cible dans le DOM peut également avoir une chance de réagir à l’événement. Considérez le document suivant :

<!DOCTYPE html>
<html>
<head>
<meta charset="utf-8" />
</head>
<body>
  <p id="paragraph">
    <span id="text">Hello World</span>
  </p>
</body>
</html>

Si nous ajoutons simplement des écouteurs à chaque élément sans aucune option, déclenchons alors un clic sur la durée…

document.body.addEventListener('click', function(event) {
  console.log("Body clicked!");
});
window.paragraph.addEventListener('click', function(event) {
  console.log("Paragraph clicked!");
});
window.text.addEventListener('click', function(event) {
  console.log("Text clicked!");
});

window.text.click();

…alors l’événement bouillonnera à travers chaque ancêtre, déclenchant chaque gestionnaire de clic sur le chemin :

Text clicked!
Paragraph clicked!
Body clicked!

Si vous souhaitez que l’un de vos gestionnaires empêche l’événement de déclencher d’autres gestionnaires, il peut appeler la méthode event.stopPropagation(). Par exemple, si nous remplaçons notre deuxième gestionnaire d’événements par ceci :

window.paragraph.addEventListener('click', function(event) {
  console.log("Paragraph clicked, and that's it!");
  event.stopPropagation();
});

Nous verrions la sortie suivante, avec le gestionnaire click de body jamais déclenché :

Text clicked!
Paragraph clicked, and that's it!

Enfin, nous avons la possibilité d’ajouter des écouteurs d’événements qui se déclenchent lors de “capture” au lieu de bouillonner. Avant qu’un événement ne jaillisse d’un élément via ses ancêtres, il est d’abord “capturé” jusqu’à l’élément via ses ancêtres. Un écouteur de capture est ajouté en spécifiant true ou {capture : true} comme troisième argument facultatif à addEventListener. Si nous ajoutons les écouteurs suivants à notre premier exemple ci-dessus :

document.body.addEventListener('click', function(event) {
  console.log("Body click captured!");
}, true);
window.paragraph.addEventListener('click', function(event) {
  console.log("Paragraph click captured!");
}, true);
window.text.addEventListener('click', function(event) {
  console.log("Text click captured!");
}, true);

Nous obtiendrons la sortie suivante :

Body click captured!
Paragraph click captured!
Text click captured!
Text clicked!
Paragraph clicked!
Body clicked!

Par défaut, les événements sont écoutés dans la phase de bouillonnement. Pour changer cela, vous pouvez spécifier la phase à laquelle l’événement est écouté en spécifiant le troisième paramètre dans la fonction addEventListener. (Pour en savoir plus sur la capture et le bouillonnement, consultez * remarques *)

element.addEventListener(eventName, eventHandler, useCapture)

useCapture : true signifie écouter l’événement lorsqu’il descend dans l’arborescence DOM. false signifie écouter l’événement pendant qu’il remonte l’arborescence DOM.

window.addEventListener("click", function(){alert('1: on bubble')}, false);
window.addEventListener("click", function(){alert('2: on capture')}, true);

Les boîtes d’alerte apparaîtront dans cet ordre :

  • 2 : à la capture
  • 1 : sur bulle

Cas d’utilisation réels

Capture Event sera envoyé avant Bubble Event, vous pouvez donc vous assurer qu’un événement est écouté en premier si vous l’écoutez dans sa phase de capture.

si vous écoutez un événement click sur un élément parent et un autre sur son enfant, vous pouvez écouter l’enfant en premier ou le parent en premier, selon la façon dont vous modifiez le paramètre useCapture.

[![en bouillonnant, l’événement enfant est appelé en premier, en capture, le parent en premier][1]][1]

en bouillonnement, l’événement enfant est appelé en premier, en capture, le parent en premier

HTML :

<div id="parent">
   <div id="child"></div>
</div>

JavaScript :

child.addEventListener('click', function(e) {
   alert('child clicked!');
});

parent.addEventListener('click', function(e) {
   alert('parent clicked!');
}, true);

Définir true sur le parent eventListener déclenchera d’abord l’écouteur parent.

Combiné avec e.stopPropagation(), vous pouvez empêcher l’événement de déclencher l’écouteur d’événement enfant / ou le parent. (plus à ce sujet dans l’exemple suivant)

[1] : http://i.stack.imgur.com/nabvw.png

Introduction #

Définition:

En informatique, un événement est une action ou un événement reconnu par les logiciels pouvant être manipulés par le logiciel. Les événements informatiques peuvent être générés ou déclenchés par le système, par l’utilisateur ou par d’autres moyens. [Source de la définition][1]

[![entrez la description de l’image ici][2]][2]

Les événements HTML sont des “choses” qui arrivent aux éléments HTML. JavaScript peut “réagir” à ces événements. via Event Listeners. De plus, des événements personnalisés peuvent être déclenchés à l’aide de dispatchEvent. Mais ceci n’est qu’une introduction, alors commençons !

Écouteur d’événements de base

Pour écouter les événements, vous appelez target.addEventListener(type, listener);

function loadImage() {
  console.log('image code here!');
}
var myButton = document.querySelector('#my-button');
myButton.addEventListener('click', loadImage);

Cela déclenchera loadImage chaque fois que my-button est cliqué.

Les écouteurs d’événement peuvent être attachés à n’importe quel nœud de l’arborescence DOM. pour voir la liste complète de tous les événements déclenchés nativement dans le navigateur : rendez-vous ici [Lien MDN pour la liste complète des événements][3]

[1] : https://en.wikipedia.org/wiki/Event_(informatique) [2] : http://i.stack.imgur.com/59MvU.png [3] : https://developer.mozilla.org/en-US/docs/Web/Events

Objet d’événement #

Pour accéder à l’objet événement, incluez un paramètre “événement” dans la fonction de rappel de l’écouteur d’événement :

var foo = document.getElementById("foo");
foo.addEventListener("click", onClick);

function onClick(event) {
  // the `event` parameter is the event object
  // e.g. `event.type` would be "click" in this case
};

e.stopPropagation();

HTML :

<div id="parent">
   <div id="child"></div>
</div>

JavaScript :

var parent = document.querySelector('#parent');
var child = document.querySelector('#child');

child.addEventListener('click', function(e) {
   e.stopPropagation();
   alert('child clicked!');
});

parent.addEventListener('click', function(e) {
   alert('parent clicked!');
});

étant donné que l’enfant arrête la propagation de l’événement et que les événements sont écoutés pendant la phase de bouillonnement, cliquer sur l’enfant ne fera que déclencher l’enfant. sans arrêter la propagation, les deux événements seront déclenchés.


e.preventDefault();

La méthode event.preventDefault() arrête l’action par défaut d’un élément.

Par exemple:

  • Empêcher un bouton d’envoi de soumettre un formulaire

  • Empêcher un lien de suivre l’URL

    var allAnchorTags = document.querySelector(‘a’);

    allAnchorTags.addEventListener(‘click’, function(e){ e.preventDefault(); console.log(‘anchor tags are useless now! evil laugh’); });

e.target contre e.currentTarget

e.currentTarget Identifie la cible actuelle de l’événement, lorsque l’événement traverse les DOM. Il fait toujours référence à l’élément que le gestionnaire d’événements a été attaché à par opposition à event.target qui identifie l’élément sur où l’événement s’est produit.

autrement dit

e.target renverra ce qui déclenche le déclenchement du répartiteur d’événements

e.currentTarget renverra ce à quoi vous avez assigné votre écouteur.

HTML :

<body>
   <button id="my-button"></button>
</body>

JavaScript :

var body = document.body;
body.addEventListener( 'click', function(e) {
    console.log('e.target', e.target);
    console.log('e.currentTarget', e.currentTarget);
});

si vous cliquez sur mon-bouton,

  • e.target sera mon-bouton
  • e.currentTarget sera body

Suppression des écouteurs d’événements #

La méthode removeEventListener() supprime les gestionnaires d’événements qui ont été attachés avec la méthode addEventListener() :

element.removeEventListener("mousemove", myFunction);

Tout (nom d’événement, fonction et options) dans removeEventListener doit correspondre à celui défini lors de l’ajout de l’écouteur d’événement à l’élément.

.lier avec removeListener

l’utilisation de .bind sur la fonction lors de l’ajout d’un écouteur d’événement empêchera la fonction d’être supprimée, pour supprimer réellement l’eventListener, vous pouvez écrire :

function onEvent() {
   console.log(this.name);
}

var bindingOnEvent = onEvent.bind(this);

document.addEventListener('click', bindingOnEvent);

...

document.removeEventListener('click', bindingOnEvent);

écouter un événement une seule fois

Jusqu’à ce que l’option “une fois” soit largement prise en charge, nous devons supprimer manuellement l’écouteur pair une fois que l’événement est déclenché pour la première fois.

Cette petite aide nous aidera à atteindre cet objectif :

Object.prototype.listenOnce = Object.prototype.listenOnce ||
  function listenOnce(eventName, eventHandler, options) {
      var target = this;
      target.addEventListener(eventName, function(e) {
          eventHandler(e);
          target.removeEventListener(eventName, eventHandler, options);
      }, options);
  }

var target = document.querySelector('#parent');
target.listenOnce("click", clickFunction, false);

*Ce n’est pas une bonne pratique d’attacher des fonctions au prototype d’objet, vous pouvez donc supprimer la première ligne de ce code et y ajouter une cible en tant que premier paramètre.

Délégation d’événement #

## Déclencher des événements personnalisés L’API CustomEvent permet aux développeurs de créer des événements personnalisés et de les déclencher sur les nœuds DOM, en transmettant des données en cours de route.

event = new CustomEvent(typeArg, customEventInit);

typeArg - DOMString représentant le nom de l’événement.

customEventInit - est un paramètre facultatif (qui sera transmis sous la forme e dans l’exemple suivant).

Vous pouvez joindre eventListeners à document ou à tout élément HTML.

Une fois l’événement personnalisé ajouté et lié à l’élément (ou au document), vous pouvez le déclencher manuellement à partir de javascript.

document.addEventListener("event-name", function(e) {
  console.log(e.detail); // logs custom object passed from the event.
});

var event = new CustomEvent("event-name", { "param-name": "param-value" });
document.dispatchEvent(event);
