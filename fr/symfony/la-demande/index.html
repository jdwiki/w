La classe Request de Symfony est une représentation orientée objet de la requête HTTP. Il contient des informations telles que l’URL, la chaîne de requête, les fichiers téléchargés, les cookies et d’autres en-têtes provenant du navigateur.

Syntaxe #

  • $request->getPathInfo(); // renvoie le chemin (partie locale de l’URL) qui est demandé (mais sans la chaîne de requête). C’est à dire. lors de la visite de https://example.com/foo/bar?key=value, cela contiendra /foo/bar
  • $request->query->get(‘id’); // renvoie un paramètre de chaîne de requête ($_GET)
  • $request->query->get(‘id’, 1); // renvoie un paramètre de chaîne de requête avec une valeur par défaut
  • $request->request->get(’nom’); // renvoie une variable de corps de requête ($_POST)
  • $request->files->get(‘attachment’); // renvoie une instance de UploadedFile identifiée par “attachment”
  • $request->cookies->get(‘PHPSESSID’); // renvoie la valeur d’un cookie ($_COOKIE)
  • $request->headers->get(‘content_type’); // renvoie un en-tête de requête HTTP
  • $request->getMethod(); // renvoie la méthode de requête HTTP (GET, POST, PUT, DELETE, etc.)
  • $request->getLanguages(); // renvoie un tableau de langues que le client accepte

Obtenir un paramètre de chaîne de requête #

Disons que nous voulons construire une liste paginée de produits, où le numéro de la page est passé en tant que paramètre de chaîne de requête. Par exemple, pour récupérer la 3ème page, vous iriez à :

http://example.com/products?page=3

La requête HTTP brute ressemblerait à ceci :

GET /products?page=3 HTTP/1.1
Host: example.com
Accept: text/html
User-Agent: Mozilla/5.0 (Macintosh)

Pour obtenir le numéro de page de l’objet de requête, vous pouvez accéder à la propriété query :

$page = $request->query->get('page'); // 3

Dans le cas d’un paramètre page, vous souhaiterez probablement passer une valeur par défaut au cas où le paramètre de chaîne de requête n’est pas défini :

$page = $request->query->get('page', 1);

Cela signifie que lorsque quelqu’un visite http://example.com/products (notez l’absence de la chaîne de requête), la variable $page contiendra la valeur par défaut 1.

Création d’un objet Request à partir de variables globales #

PHP expose un certain nombre de soi-disant variables globales qui contiennent des informations sur la requête HTTP, telles que [$_POST][1], [$_GET][2], [$_FILES][3 ], [$_SESSION][4], etc. La classe Request contient une méthode statique createFromGlobals() afin d’instancier un objet de requête basé sur ces variables :

use Symfony\Component\HttpFoundation\Request;

$request = Request::createFromGlobals();

Lorsque vous utilisez le framework Symfony, vous ne devez pas instancier l’objet de requête vous-même. Au lieu de cela, vous devez utiliser l’objet qui est instancié lorsque le framework est démarré dans app.php / app_dev.php. Par exemple en [tapez en indiquant l’objet de requête dans votre contrôleur] [5].

[1] : https://secure.php.net/manual/en/reserved.variables.post.php [2] : https://secure.php.net/manual/en/reserved.variables.get.php [3] : https://secure.php.net/manual/en/reserved.variables.files.php [4] : https://secure.php.net/manual/en/reserved.variables.session.php [5] : https://www.wikiod.com/fr/symfony/controleurs#Utilisation des données de l’objet Request

Accéder à une variable POST #

Pour obtenir le contenu d’un formulaire soumis avec method="post", utilisez la propriété post :

$name = $request->request->get('name');

Accéder au contenu d’un cookie #

$id = $request->cookies->get('PHPSESSID');

Cela renverra la valeur du cookie ‘PHPSESSID’ envoyé par le navigateur.
