Éléments de l’interface utilisateur #

L’interface utilisateur d’Emacs utilise des termes qui ont été inventés tôt et peuvent être troublants pour les utilisateurs habitués à une terminologie plus moderne.

[![Terminologie Emacs][1]][1]

Cadre #

Dans Emacs, ce qu’on appelle autrement une fenêtre (la zone d’affichage utilisée par un programme) s’appelle un cadre. Emacs commence à utiliser un cadre, bien que des cadres supplémentaires puissent être créés en utilisant C-x 5.

Fenêtre #

Un cadre contient une ou plusieurs fenêtres (autrement appelées volets), chacune affichant le contenu d’un tampon. Chaque frame commence généralement par une seule window, mais des fenêtres supplémentaires peuvent être créées en divisant celles existantes ; soit horizontalement avec C-x 2 soit verticalement avec C-x 3. Voir aussi https://www.wikiod.com/fr/emacs/raccourcis-clavier-de-base#Fenêtres ou châssis multiples

Amortir #

Le terme tampon fait référence au contenu affiché dans une fenêtre. Un tel contenu peut refléter le contenu d’un fichier dans le système de fichiers (ou peut-être une version mise à jour qui n’a pas encore été enregistrée sur le disque), mais plus généralement, il peut s’agir de n’importe quel type de texte.

Ligne de mode #

Au bas de chaque fenêtre se trouve une ligne de mode, qui décrit synthétiquement le tampon affiché dans la fenêtre.

Barre d’outils #

De manière similaire à de nombreux autres logiciels, une barre d’outils peut être affichée en haut de chaque frame. Son contenu peut varier selon le type de tampon en cours d’édition.

Mini-tampon #

Un minibuffer, généralement affiché en bas de chaque frame, permet d’interagir avec Emacs. Chaque fois qu’une commande demande une entrée utilisateur, elle est demandée dans le minibuffer. Inversement, les messages affichés à la vue de l’utilisateur y sont imprimés.

[1] : http://i.stack.imgur.com/hIeNf.png

Point, marque et région #

Emacs utilise les termes point, marque et région pour fournir plus de précision sur le texte sélectionné et la position du curseur. En comprenant ces termes, cela vous aidera à comprendre et à utiliser d’autres opérations et fonctions.

Le point est l’endroit dans un tampon où l’édition (c’est-à-dire l’insertion) est en cours, et est généralement indiqué par un curseur. La marque est un marqueur placé n’importe où dans le tampon à l’aide de commandes telles que set-mark-command (C-SPC) ou exchange-point-and-mark ( C-x C-x). La région est la zone entre le point et la marque, et de nombreuses commandes opèrent sur la région pour par exemple supprimer, vérifier l’orthographe, mettre en retrait ou la compiler.

Lorsque vous cliquez avec votre souris sur un emplacement dans un tampon, vous voyez le point. Lorsque vous sélectionnez du texte, vous définissez la région (le texte sélectionné) et la marque (au début de votre sélection).

Fichiers et tampons #

Dans Emacs, fichier a la même signification que dans le système d’exploitation et est utilisé pour le stockage permanent des données. Un tampon est la représentation interne d’un fichier en cours d’édition. Les fichiers peuvent être lus dans des tampons en utilisant C-x C-f, et les tampons peuvent être écrits dans des fichiers en utilisant C-x C-s (enregistrer le fichier à son emplacement actuel) ou C-x C-w</ kbd> (écrire le fichier à un emplacement différent, en le demandant - l’équivalent de Enregistrer sous).

Tuer et tirer #

Killing et yanking correspondent plus ou moins à ce qu’on appelle habituellement “couper” et “coller”.

Tuer #

tuer signifie supprimer du texte et le copier dans le kill-ring (qui pourrait être vu comme une sorte de “presse-papiers” dans la terminologie “couper-coller”). Le kill ring est ainsi nommé parce qu’il stocke plusieurs morceaux de texte tué, qui peuvent ensuite être consultés dans un ordre cyclique.

Il existe diverses commandes qui tuent un mot (M-d), le reste de la ligne (C-k) ou des blocs de texte plus grands (comme la région actuellement sélectionnée : C-w).

D’autres commandes existent, qui enregistrent le texte dans le kill ring, sans réellement le tuer (d’une manière similaire à la “copie” dans les terminologies modernes). Par exemple, M-w, qui agit sur la région actuellement sélectionnée.

Coup sec #

Les entrées du kill ring peuvent ensuite être récupérées dans un tampon. On peut généralement extraire le texte le plus récemment tué en utilisant par exemple C-y (ce qui est similaire à l’opération “coller” dans une terminologie plus moderne). Mais d’autres commandes peuvent accéder et extraire les anciennes entrées du kill ring.

Modes #

Mode majeur #

Emacs peut adapter son comportement au type spécifique de texte édité dans un buffer. L’ensemble des personnalisations spécifiques d’Emacs pour un type de texte particulier est appelé un “mode majeur”. Chaque tampon a exactement un mode majeur selon son type de contenu.

Les modes majeurs peuvent modifier la signification de certaines touches, définir des règles de mise en évidence de la syntaxe ou d’indentation et installer de nouvelles liaisons de touches (commençant généralement par C-c) pour les commandes spécifiques au mode. Emacs est livré avec un large éventail de modes majeurs, répartis en trois catégories principales :

  • prise en charge du texte (par exemple, les langages de balisage),
  • prise en charge des langages de programmation,
  • applications au sein d’emacs (par exemple dired, gnus, …). Les tampons utilisant ce dernier groupe de modes majeurs ne sont généralement pas associés à des fichiers, mais servent plutôt d’interface utilisateur.

Mode mineur #

Les modes mineurs sont des fonctionnalités optionnelles qui peuvent être activées et désactivées. Les modes mineurs peuvent être activés pour des tampons spécifiques (modes tampons locaux) ou tous les tampons (modes globaux). Contrairement aux modes majeurs, n’importe quel nombre de modes mineurs peut être activé pour un tampon donné.

Emacs fournit de nombreux modes mineurs. Voici quelques exemples :

  • Mode de remplissage automatique pour envelopper automatiquement les lignes de texte au fur et à mesure que vous tapez.
    • Mode Flyspell * pour mettre en évidence les fautes d’orthographe au fur et à mesure que vous tapez.
    • Mode ligne visuelle * pour envelopper de longues lignes pour s’adapter à l’écran.
  • Mode de marque transitoire pour mettre en surbrillance la région actuelle.
