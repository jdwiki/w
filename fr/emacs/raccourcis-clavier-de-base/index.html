Quitter Emacs #

Vous pouvez quitter Emacs avec le raccourci clavier suivant :

C-x C-c

Où C est la touche control.

Suspendre Emacs

Vous pouvez suspendre Emacs en utilisant le raccourci clavier suivant :

C-z

Cela vous ramène à votre coquille. Si vous souhaitez reprendre votre session emacs, entrez fg dans votre terminal.

Abandonner la commande en cours #

Vous arriverez souvent dans un état où vous avez une séquence de commandes partiellement tapée en cours, mais vous voulez l’abandonner. Vous pouvez l’abandonner avec l’une des combinaisons de touches suivantes :

C-g

ÉchapÉchapÉchap

Tampons #

  • Exemple de liste tampon

    CRM Buffer                Size  Mode              Filename[/Process]
. * .emacs                3294  Emacs-Lisp        ~/.emacs
 %  *Help*                 101  Help
    search.c             86055  C                 ~/cvs/emacs/src/search.c
 %  src                  20959  Dired by name     ~/cvs/emacs/src/
  * *mail*                  42  Mail
 %  HELLO                 1607  Fundamental       ~/cvs/emacs/etc/HELLO
 %  NEWS                481184  Outline           ~/cvs/emacs/etc/NEWS
    *scratch*              191  Lisp Interaction
  * *
Messages*            1554  Messages

    The first field of a line indicates:

    • ‘.’ the buffer is current.
    • ‘%’ a read-only buffer.
    • ‘*’ the buffer is modified.

  • Sélectionnez le tampon. Vous pouvez sélectionner n’importe quel tampon ouvert avec le raccourci clavier suivant :

C-x b

Vous serez invité à entrer le nom du tampon vers lequel vous souhaitez basculer.

  • Liste des tampons :

    C-x C-b

  • Save-some-buffer, donnant le choix du tampon à sauvegarder ou non :

    C-x s

  • Tuez un tampon :

    C-x k

  • Opérations sur les tampons marqués :

    <kbd>S</kbd>  Save the marked buffers                                                                  

<kbd>A</kbd>  View the marked buffers in this frame.                                                   

<kbd>H</kbd>  View the marked buffers in another frame.                                                

<kbd>V</kbd> Revert the marked buffers.                                                               

<kbd>T</kbd> Toggle read-only state of marked buffers.                                                

<kbd>D</kbd> Kill the marked buffers.                                                                 

<kbd>M-s a C-s</kbd> Do incremental search in the marked buffers.                                     

<kbd>M-s a C-M-s</kbd> Isearch for regexp in the marked buffers.                                      

<kbd>U</kbd> Replace by regexp in each of the marked buffers.                                                                                                                  

<kbd>Q</kbd> Query replace in each of the marked buffers.                                             

<kbd>I</kbd> As above, with a regular expression.                                                     

<kbd>P</kbd> Print the marked buffers.                                                                

<kbd>O</kbd> List lines in all marked buffers which match a given regexp (like the function `occur').                                                                                    

<kbd>X</kbd> Pipe the contents of the marked buffers to a shell command.                                                   

<kbd>N</kbd> Replace the contents of the marked buffers with the output of a shell command.  

<kbd>!</kbd> Run a shell command with the buffer's file as an argument.                                                             

<kbd>E</kbd> Evaluate a form in each of the marked buffers. This is a very flexible command.  For example, if you want to make all of the marked buffers read only, try using (read-only-mode 1) as the input form.                                  

<kbd>W</kbd> - As above, but view each buffer while the form is evaluated.                                           

<kbd>k</kbd> - Remove the marked lines from the *Ibuffer* buffer, but don't kill the associated buffer.                                      

<kbd>x</kbd> - Kill all buffers marked for deletion.

  • Save-some-buffer, donnant le choix du tampon à sauvegarder ou non :

    C-x s

  • Passer au tampon suivant :

C-x DROITE

  • Passer au tampon précédent :

C-x GAUCHE

La gestion des fichiers #

  • Réenregistrer le fichier ouvert sous le même nom de fichier (Save):

    C-x C-s

  • Écrire en tant que “nom de fichier” (Enregistrer sous) :

    C-x C-w filename

Le nouveau nom de fichier sera demandé dans le mini-tampon.

  • Créer un nouveau fichier ou charger un fichier existant (Nouveau / Charger) :

    C-x C-f filename

    With the mnemonic here for f meaning file. You will be prompted for a file path in the minibuffer.

  • Visitez le fichier alternatif

    C-x C-f

    If the file does not exist yet, you will be prompted the path of the file to create in the minibuffer.

Mouvement du curseur (point) #

En plus des mouvements du curseur à l’aide des touches fléchées, Début, Fin, Page vers le haut et Page vers le bas, emacs définit un certain nombre de frappes qui peuvent déplacer le curseur sur des morceaux de texte plus petits ou plus grands :

Par caractère :

  • Caractère inversé : C-b
  • Caractère avant : C-f

Par mot

  • Mot inversé : M-b (c’est-à-dire Alt b, ou Meta b)
  • Mot vers l’avant : M-f

Par ligne :

  • Début de la ligne courante : C-a
  • Début du premier caractère (non-espace) de la ligne actuelle : M-m
  • Fin de la ligne courante : C-e
  • Ligne précédente : C-p
  • Ligne suivante : C-n

Tampon entier :

  • Début de buffer : M-«/kbd>
  • Fin de buffer : M->

Par ‘bloc’, selon le contexte (mode) :

Raccourcis clavier typiques :

  • Phrase/instruction à l’envers : M-a
  • Phrase/instruction avant : M-e
  • Début de fonction : M-C-a
  • Fin de fonction : M-C-e

Arguments de préfixe

Afin de déplacer plusieurs “étapes” à la fois, les commandes de mouvement peuvent recevoir un argument de préfixe en appuyant sur ESC ou C-u et un nombre avant les frappes répertoriées. Pour C-u, le nombre est facultatif et par défaut à 4. Ex. ESC 3 C-n déplace 3 lignes vers le bas, tandis que C-u M-f déplace 4 mots vers l’avant.

Notation des raccourcis clavier #

La documentation d’Emacs utilise une notation cohérente pour toutes les liaisons de touches, qui est expliquée ici :

Accords clés #

Un “accord clé” est obtenu en appuyant simultanément sur deux touches ou plus. Les accords clés sont indiqués en séparant toutes les clés par des tirets (-). Ils impliquent généralement des touches de modification, qui sont placées devant :

  • C- : contrôle ;
  • S- : décalage ;
  • M- : alt (le “M” signifie “Meta” pour des raisons historiques).

D’autres clés sont simplement désignées par leur nom, comme :

  • a : la touche a ;
  • gauche : la touche fléchée vers la gauche ;
  • SPC : la touche espace ;
  • RET : la clé de retour.

Des exemples d’accords clés incluent donc:

  • C-a : appuyer simultanément sur control et a ;
  • S-droite : appuyer simultanément sur shift et droite ;
  • C-M-a : en appuyant simultanément sur control, alt et a.

Séquences de touches #

Les “séquences de touches” sont des séquences de touches (ou accords de touches), qui doivent être tapées les unes après les autres. Ils sont désignés en séparant toutes les notations de clé (ou d’accord) par un espace.

Les exemples comprennent:

  • C-x b : appuyer simultanément sur control et x, puis les relâcher et appuyer sur b ;
  • C-x C-f : appuyer simultanément sur control et x, puis relâcher x et appuyer sur f (puisque les deux accords impliquent le modificateur control, il n’est pas nécessaire de le relâcher).

Utiliser ESC au lieu de Alt #

Les accords clés utilisant le modificateur Alt peuvent également être saisis sous forme de séquence de touches commençant par ESC. Cela peut être utile lors de l’utilisation d’Emacs sur une connexion à distance qui ne transmet pas les accords de touche Alt, ou lorsque ces combinaisons de touches sont capturées e.g par un gestionnaire de fenêtres.

Exemple:

M-x peut être saisi sous la forme ESC x.

Décrire les raccourcis clavier dans les fichiers Lisp d’Emacs #

La même notation décrite ici peut être utilisée lors de la définition des raccourcis clavier dans les fichiers Lisp d’Emacs.

Exemple:

(global-set-key (kbd “C-x C-b”) ‘buffer-menu) lie la séquence de touches C-x C-b à la commande buffer-menu

Fenêtres ou cadres multiples #

“Fenêtre” dans Emacs fait référence à ce que l’on pourrait autrement appeler un “volet” ou une “division d’écran”. Certaines commandes de manipulation de fenêtre incluent :

  • Fractionner la fenêtre courante horizontalement : C-x 2
  • Diviser la fenêtre actuelle verticalement : C-x 3
  • Sélectionnez la fenêtre suivante : C-x o
  • Fermer la fenêtre en cours : C-x 0
  • Fermer toutes les autres fenêtres, sauf celle en cours : C-x 1

Un “cadre” dans Emacs est ce qu’on pourrait autrement appeler une “fenêtre”. Les cadres sont manipulés à l’aide de ces commandes :

  • Créer un nouveau cadre : C-x 5 2
  • Supprimer la trame actuelle : C-x 5 0
  • Supprimer les autres cadres : C-x 5 1

Le changement de fenêtre peut être réalisé à l’aide de

  • S-gauche, S-droite, S-haut, S-bas (c’est-à-dire Shift en combinaison avec une touche fléchée) pour passer à la fenêtre voisine dans une direction, ou
  • C-x o pour passer à la fenêtre suivante.

Région - Couper, Copier, Coller #

  • Définir la marque à l’emplacement du curseur :

    C-space or C-@

  • Tuer la région (Couper):

    C-w

  • Copier la région dans le kill ring :

    M-w or Esc-w

  • Yank (Coller) tué le plus récemment :

    C-y

  • Yank (Coller) suivant dernier tué :

    M-y or Esc-y

Tuer #

kill est la commande utilisée par Emacs pour la suppression de texte. La commande kill est analogue à la commande cut de Windows. Diverses commandes existent qui “tuent” un mot (M-d), le reste de la ligne (C-k) ou des blocs de texte plus grands. Le texte supprimé est ajouté au kill-ring, à partir duquel il peut ensuite être yanked.

Sélectionner et couper (tuer) #

Tuer et tirer Semblable à la fonction “sélectionner et couper” de Windows, nous avons ici C-spc. La liaison de touches C-spc lancera la sélection, l’utilisateur peut déplacer la marque à l’aide des touches fléchées ou d’une autre commande pour effectuer une sélection. Une fois la sélection terminée - appuyez sur C-w pour tuer le texte sélectionné

Quelques commandes de base qui peuvent être utilisées comme référence rapide pour la commande kill (tirées du tutoriel Emacs) <pré> M-DEL Kill the word immediately before the cursor M-d Kill the next word after the cursor

C-k         Kill from the cursor position to end of line
M-k         Kill to the end of the current sentence

Coup sec #

yank décrit l’insertion de texte précédemment supprimé, par exemple en utilisant C-y qui copie le texte le plus récemment supprimé. La commande Yank est analogue à la commande paste de Windows.

Yank texte tué précédemment #

Nous savons que la commande kill ajoute le texte tué à un kill-ring. Pour récupérer le texte supprimé du kill-ring, utilisez la commande M-y à plusieurs reprises jusqu’à ce que le texte souhaité soit extrait.

(Remarque : pour que la touche M-y fonctionne, la commande précédente doit être un YANK sinon cela ne fonctionnerait pas)

Annuler #

Pour annuler quelque chose que vous venez de faire :

C-_ ou C-x u ou C-/

Rechercher et remplacer #

Dans Emacs, l’outil de recherche de base (“I-Search”) vous permet de rechercher après ou avant l’emplacement de votre curseur.

  • Pour rechercher untexte après l’emplacement de votre curseur (search-forward), appuyez sur C-s untexte. Si vous voulez aller à la prochaine occurrence de sometext, appuyez simplement sur C-s à nouveau (et ainsi de suite pour les prochaines occurrences). Lorsque le curseur atterrit au bon endroit, appuyez sur Entrée pour quitter l’invite de recherche.

  • Pour rechercher avant l’emplacement de votre curseur (search-backward), utilisez C-r de la même manière que vous avez utilisé auparavant.

  • Pour passer de search-backward à search-forward, appuyez 2 fois sur C-s. Et appuyez 2 fois sur C-r pour “rechercher en arrière” lorsque vous êtes dans l’invite “rechercher en avant”.

  • Rechercher et remplacer :

    M-% (or Esc-% ) oldtext Enter newtext Enter

    • Confirm: y
    • Skip: n
    • Quit: q
    • Replace all: !

Cas #

  • Mot en majuscule : M-c

  • Convertir un mot en majuscule : M-u

  • Convertir un mot en minuscule : M-l
