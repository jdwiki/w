Syntaxe #

  • TableauNouveau(dimension, estSynchronisé)

Paramètres #

Nom Descriptif
Dimension Nombre de dimensions dans le nouveau tableau : 1, 2 ou 3
est synchronisé Lorsque false, crée un tableau non synchronisé. Lorsque true, la fonction renvoie un tableau synchronisé.

Dans un tableau synchronisé, plus de deux threads ne peuvent pas accéder simultanément au tableau. Les autres threads doivent attendre que le thread actif termine son travail, ce qui se traduit par des performances significatives.

Dans la version 2016 de ColdFusion, vous pouvez utiliser un tableau non synchronisé et permettre à plusieurs threads d’accéder simultanément au même objet de tableau.

Création de tableaux #

Créer des tableaux explicitement en utilisant ArrayNew()

Déclarez un tableau avec la fonction ArrayNew. Spécifiez le nombre de dimensions comme argument.

  • ArrayNew(dimension) crée un tableau de 1 à 3 dimensions.
  • Les baies ColdFusion se développent dynamiquement à mesure que des données sont ajoutées.
  • ArrayNew() renvoie un tableau.

Histoire #

Introduit dans ColdFusion MX 6

Déclaration #

CFML

<!--- One Dimension Array--->
<cfset oneDimensionArray = ArrayNew(1)>

CFScriptComment Notez qu’à l’intérieur d’une fonction, vous devez utiliser la portée var. Les versions antérieures de CF exigeaient que la portée var soit la première chose dans une fonction ; les versions ultérieures l’autorisent n’importe où dans une fonction.

<cfscript>
    oneDimensionArray = ArrayNew(1);

    public void function myFunc() { 
        var oneDimensionArray = ArrayNew(1);
    } 
</cfscript>

Après avoir créé le tableau, ajoutez des éléments en utilisant les index d’éléments. L’indice Coldfusion Array commence à 1 :

CFML

<cfset oneDimensionArray[1] = 1>
<cfset oneDimensionArray[2] = 'one'>
<cfset oneDimensionArray[3] = '1'>

CFScriptComment

<cfscript>
    oneDimensionArray[1] = 1;
    oneDimensionArray[2] = 'one';
    oneDimensionArray[3] = '1';
</cfscript>

Utilisation de ArrayAppend() #

Vous pouvez ajouter des éléments à un tableau en utilisant la fonction ArrayAppend(array, value).

<cfscript>
    ArrayAppend(oneDimensionArray, 1);
    ArrayAppend(oneDimensionArray, 'one');
    ArrayAppend(oneDimensionArray, '1');
</cfscript>

Affichez le contenu du tableau à l’aide de <cfdump> :

<cfdump var="#oneDimensionArray#">

CFML

<!--- Two Dimension Array--->
<cfset twoDimensionArray = ArrayNew(2)>
<cfset twoDimensionArray[1][1] = 1>
<cfset twoDimensionArray[1][2] = 2>
<cfset twoDimensionArray[2][1] = 3>
<cfset twoDimensionArray[2][2] = 4>

CFScriptComment

<cfscript>
    twoDimensionArray = ArrayNew(2);
    
    twoDimensionArray[1][1] = 1;
    twoDimensionArray[1][2] = 2;
    twoDimensionArray[2][1] = 3;
    twoDimensionArray[2][2] = 4;
</cfscript>

Afficher le contenu du tableau à l’aide de <cfdump>

<cfdump var="#twoDimensionArray#">

Création implicite d’un tableau 1D #

Lors de la création implicite d’un tableau, les crochets ([]) entourent le contenu du tableau avec des séparateurs virgules.

<cfset oneDimensionArrayImplicit = [ 1 ,'one','1' ]>

Cette instruction est équivalente aux quatre instructions utilisées pour créer le oneDimensionArray ci-dessus. Le résultat est le même en utilisant :

<cfdump var="#oneDimensionArrayImplicit#">

Créer un tableau 2D implicitement #

<cfset twoDimensionArrayImplicit = [[ 1 , 2 ],[ 3 , 4 ],[ 5 , 6 ]]>

Ou:

<cfset firstElement = ["1", "2"]> 
<cfset secondElement= ["3", "4"]> 
<cfset thirdElement = ["5", "6"]> 
<cfset twoDimensionArrayImplicit = [firstElement , secondElement, thirdElement]>

Sortie du contenu à l’aide de

<cfdump var="#twoDimensionArrayImplicit#">

[![Tableau implicite à deux dimensions][3]][3]

Déclaration explicite alternative

Vous pouvez également déclarer 1 Dimension Array comme

<cfset oneDimensionArray = []>

<cfscript>
    oneDimensionArray = [];
</cfscript>

Cette déclaration est la même que celle de l’utilisation de ArrayNew(1).

Mais si vous essayez de déclarer 2 Dimension Array comme

<cfset twoDimensionArray =[][]>    //Invalid CFML construct

une erreur se produira lors du traitement de cette demande.

La déclaration suivante traitera la demande :

<cfset twoDimensionArray =[]>

mais la variable twoDimensionArray ne sera pas réellement un tableau dans un tableau (ou un tableau à 2 dimensions). Il contient en fait une structure dans Array.

<cfset twoDimensionArray =[]>
<cfset twoDimensionArray[1][1] = 1>
<cfset twoDimensionArray[1][2] = 2>
<cfset twoDimensionArray[2][1] = 3>
<cfset twoDimensionArray[2][2] = 4>

<cfdump var="#twoDimensionArray#">

Tableau dans CFScript #

<cfscript>
    oneDimensionArray = ArrayNew(1);
    oneDimensionArray[1] = 1;
    oneDimensionArray[2] = 'one';
    oneDimensionArray[3] = '1';
</cfscript>

<cfif IsDefined("oneDimensionArray")>
    <cfdump var="#oneDimensionArray#">
</cfif>

Résultat:

[![Tableau dans CFSCRIPT][1]][1]

De plus, nous pouvons déclarer un tableau à une dimension comme :

oneDimensionArray = [];

Alternativement, CF a introduit WriteDump() de CF9 comme une fonction équivalente à la balise <cfdump> qui peut être utilisée dans <cfscript>.

<cfscript>
    WriteDump(oneDimensionArray);
</cfscript>

De même, pour un tableau à 2 dimensions : #

<cfscript>
    twoDimensionArray = ArrayNew(2);
    twoDimensionArray[1][1] = 1;
    twoDimensionArray[1][2] = 2;
    twoDimensionArray[2][1] = 3;
    twoDimensionArray[2][2] = 4;
</cfscript>
<cfdump var="#twoDimensionArray#">

Informations générales #

Tout d’abord, quelques informations générales sur le comportement des tableaux dans Coldfusion par rapport à d’autres langages de programmation.

  • Les tableaux ne peuvent avoir que des index numériques (si vous voulez avoir un index de chaîne, utilisez plutôt structs)
  • Les tableaux commencent à l’index [1]
  • Les tableaux peuvent avoir une ou plusieurs dimensions
