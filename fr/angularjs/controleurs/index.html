Syntaxe #

  • <htmlElement of-controller=“controllerName”> … </htmlElement>
  • <script> app.controller(‘controllerName’, controllerFunction); </script>

Votre premier contrôleur #

Un contrôleur est une structure de base utilisée dans Angular pour préserver la portée et gérer certaines actions au sein d’une page. Chaque contrôleur est couplé à une vue HTML.

Vous trouverez ci-dessous un passe-partout de base pour une application angulaire :

<!DOCTYPE html>

<html lang="en" ng-app='MyFirstApp'>
    <head>
        <title>My First App</title>

        <!-- angular source -->
        <script src="https://code.angularjs.org/1.5.3/angular.min.js"></script>
        
        <!-- Your custom controller code -->
        <script src="js/controllers.js"></script>
    </head>
    <body>
        <div ng-controller="MyController as mc">
            <h1>{{ mc.title }}</h1>
            <p>{{ mc.description }}</p>
            <button ng-click="mc.clicked()">
                Click Me!
            </button>
        </div>
    </body>
</html>

Il y a quelques points à noter ici :

<html ng-app='MyFirstApp'>

Définir le nom de l’application avec ng-app vous permet d’accéder à l’application dans un fichier Javascript externe, qui sera traité ci-dessous.

<script src="js/controllers.js"></script>

Nous aurons besoin d’un fichier Javascript dans lequel vous définissez vos contrôleurs et leurs actions/données.

<div ng-controller="MyController as mc">

L’attribut ng-controller définit le contrôleur pour cet élément DOM et tous les éléments qui sont des enfants (récursivement) en dessous.

Vous pouvez avoir plusieurs du même contrôleur (dans ce cas, MyController) en disant ... as mc, nous donnons à cette instance du contrôleur un alias.

<h1>{{ mc.title }}</h1>

La notation {{ ... }} est une expression angulaire. Dans ce cas, cela définira le texte intérieur de cet élément <h1> quelle que soit la valeur de mc.title.

Remarque : Angular utilise une liaison de données bidirectionnelle, ce qui signifie que quelle que soit la façon dont vous mettez à jour la valeur mc.title, elle sera reflétée à la fois dans le contrôleur et dans la page.

Notez également que les expressions angulaires ne doivent pas faire référence à un contrôleur. Une expression angulaire peut être aussi simple que {{ 1 + 2 }} ou {{ "Hello " + "World" }}.

<button ng-click="mc.clicked()">

ng-click est une directive angulaire, dans ce cas liant l’événement click pour que le bouton déclenche la fonction clicked() de l’instance MyController.

Avec ces choses à l’esprit, écrivons une implémentation du contrôleur MyController. Avec l’exemple ci-dessus, vous écririez ce code dans js/controller.js.

Tout d’abord, vous devrez instancier l’application Angular dans votre Javascript.

var app = angular.module("MyFirstApp", []);

Notez que le nom que nous transmettons ici est le même que le nom que vous avez défini dans votre HTML avec la directive ng-app.

Maintenant que nous avons l’objet app, nous pouvons l’utiliser pour créer des contrôleurs.

app.controller('MyController', function(){
    var ctrl = this;

    ctrl.title = "My First Angular App";
    ctrl.description = "This is my first Angular app!";

    ctrl.clicked = function(){
        alert("MyController.clicked()");
    };
});

Remarque : Pour tout ce que nous voulons faire partie de l’instance de contrôleur, nous utilisons le mot-clé “this”.

C’est tout ce qui est nécessaire pour construire un contrôleur simple.

Création de contrôleurs, minification sécurisée #

Il existe plusieurs façons de protéger la création de votre contrôleur contre la minification.

La première est appelée annotation de tableau en ligne. Il ressemble à ce qui suit :

var app = angular.module('app');
app.controller('sampleController', ['$scope', '$http', function(a, b){
    //logic here
}]);

Le deuxième paramètre de la méthode du contrôleur peut accepter un tableau de dépendances. Comme vous pouvez le voir, j’ai défini $scope et $http qui devraient correspondre aux paramètres de la fonction du contrôleur dans laquelle a sera le $scope, et b serait $http . Notez que le dernier élément du tableau doit être la fonction de votre contrôleur.

La seconde option utilise la propriété $inject. Il ressemble à ce qui suit :

var app = angular.module('app');
app.controller('sampleController', sampleController);
sampleController.$inject = ['$scope', '$http'];
function sampleController(a, b) {
    //logic here
}

Cela fait la même chose que l’annotation de tableau en ligne mais fournit un style différent pour ceux qui préfèrent une option à l’autre.

L’ordre des dépendances injectées est important #

Lors de l’injection de dépendances à l’aide de la forme de tableau, assurez-vous que la liste des dépendances correspond à sa liste d’arguments correspondante transmise à la fonction de contrôleur.

Notez que dans l’exemple suivant, $scope et $http sont inversés. Cela causera un problème dans le code.

// Intentional Bug: injected dependencies are reversed which will cause a problem
app.controller('sampleController', ['$scope', '$http',function($http, $scope) {
    $http.get('sample.json');
}]);

Utilisation de ControllerAs dans Angular JS #

Dans Angular, $scope est le ciment entre le contrôleur et la vue qui aide à tous nos besoins de liaison de données. Controller As est un autre moyen de lier le contrôleur et la vue et il est généralement recommandé de l’utiliser. Fondamentalement, ce sont les deux constructions de contrôleur dans Angular (c’est-à-dire $scope et Controller As).

Différentes façons d’utiliser Controller As sont -

controllerAs View Syntaxe

<div ng-controller="CustomerController as customer">
    {{ customer.name }}
</div>

controllerAs Contrôleur Syntaxe

function CustomerController() {
    this.name = {};
    this.sendMessage = function() { };
}

contrôleurAs avec vm

function CustomerController() {
    /*jshint validthis: true */
    var vm = this;
    vm.name = {};
    vm.sendMessage = function() { };
}

controllerAs est du sucre syntaxique sur $scope. Vous pouvez toujours vous lier à la vue et toujours accéder aux méthodes $scope. L’utilisation de controllerAs est l’une des meilleures pratiques suggérées par l’équipe de base angulaire. Il y a plusieurs raisons à cela, peu d’entre elles sont -

  • $scope expose les membres du contrôleur à la vue via un objet intermédiaire. En définissant this.*, nous pouvons exposer uniquement ce que nous voulons exposer du contrôleur à la vue. Il suit également la manière standard d’utiliser JavaScript.

  • en utilisant la syntaxe controllerAs, nous avons un code plus lisible et la propriété parent est accessible en utilisant le nom d’alias du contrôleur parent au lieu d’utiliser la syntaxe $parent.

  • Il favorise l’utilisation de la liaison à un objet “pointillé” dans la vue (par exemple client.nom au lieu de nom), qui est plus contextuel, plus facile à lire et évite tout problème de référence qui peut survenir sans “pointillé”.

  • Aide à éviter d’utiliser des appels $ parent dans les vues avec des contrôleurs imbriqués.

  • Utilisez une variable de capture pour cela lorsque vous utilisez la syntaxe controllerAs. Choisissez un nom de variable cohérent tel que vm, qui signifie ViewModel. Parce que le mot-clé this est contextuel et lorsqu’il est utilisé dans une fonction à l’intérieur d’un contrôleur peut changer son contexte. Capturer le contexte de ceci évite de rencontrer ce problème.

REMARQUE : en utilisant la syntaxe controllerAs ajouter à la référence de portée actuelle au contrôleur actuel, de sorte qu’il soit disponible en tant que champ

<div ng-controller="Controller as vm>...</div>

vm est disponible en tant que $scope.vm.

Création de contrôleurs angulaires sécurisés pour la minification #

Pour créer des contrôleurs angulaires sécurisés pour la minification, vous allez modifier les paramètres de la fonction controller.

Le deuxième argument de la fonction module.controller doit être passé à un tableau, où le dernier paramètre est la fonction du contrôleur, et chaque paramètre avant cela est le nom de chaque valeur injectée.

Ceci est différent du paradigme normal; qui prend la fonction de contrôleur avec les arguments injectés.

Donné:

var app = angular.module('myApp');

Le contrôleur devrait ressembler à ceci :

app.controller('ctrlInject', 
    [
        /* Injected Parameters */
        '$Injectable1', 
        '$Injectable2', 
        /* Controller Function */
        function($injectable1Instance, $injectable2Instance) {
            /* Controller Content */
        }
    ]
);

Remarque : Les noms des paramètres injectés ne sont pas tenus de correspondre, mais ils seront liés dans l’ordre.

Cela réduira à quelque chose de similaire à ceci:

var a=angular.module('myApp');a.controller('ctrlInject',['$Injectable1','$Injectable2',function(b,c){/* Controller Content */}]);

Le processus de minification remplacera chaque instance de app par a, chaque instance de $Injectable1Instance par b et chaque instance de $Injectable2Instance par c.

Création de contrôleurs #

angular
    .module('app')
    .controller('SampleController', SampleController)

SampleController.$inject = ['$log', '$scope'];
function SampleController($log, $scope){
    $log.debug('*****SampleController******');
    
    /* Your code below */
}

Remarque : Le .$inject s’assurera que vos dépendances ne seront pas brouillées après la minification. Assurez-vous également qu’il est en ordre avec la fonction nommée.

Contrôleurs imbriqués #

Les contrôleurs imbriqués enchaînent également le $scope. La modification d’une variable $scope dans le contrôleur imbriqué modifie la même variable $scope dans le contrôleur parent.

.controller('parentController', function ($scope) {
    $scope.parentVariable = "I'm the parent";
});

.controller('childController', function ($scope) {
    $scope.childVariable = "I'm the child";

    $scope.childFunction = function () {
        $scope.parentVariable = "I'm overriding you";
    };
});

Essayons maintenant de gérer les deux, imbriqués.

<body ng-controller="parentController">
    What controller am I? {{parentVariable}}
    <div ng-controller="childController">
        What controller am I? {{childVariable}}
        <button ng-click="childFunction()"> Click me to override! </button>
    </div>
</body>

Les contrôleurs imbriqués peuvent avoir leurs avantages, mais une chose doit être gardée à l’esprit lorsque vous le faites. L’appel de la directive ngController crée une nouvelle instance du contrôleur - ce qui peut souvent créer de la confusion et des résultats inattendus.
