Expressions angulaires - Texte vs nombre #

Cet exemple montre comment les expressions angulaires sont évaluées lors de l’utilisation de type="text" et type="number" pour l’élément d’entrée. Considérez le contrôleur et la vue suivants :

Manette

 var app = angular.module('app', []);

 app.controller('ctrl', function($scope) {
     $scope.textInput = {
         value: '5'
     };     
     $scope.numberInput = {
         value: 5
     };
 });

Voir

<div ng-app="app" ng-controller="ctrl">
    <input type="text" ng-model="textInput.value">
    {{ textInput.value + 5 }}
    <input type="number" ng-model="numberInput.value">
    {{ numberInput.value + 5 }}
</div>
  • Lors de l’utilisation de + dans une expression liée à l’entrée text, l’opérateur concaténera les chaînes (premier exemple), en affichant 55 à l’écran *.
  • Lors de l’utilisation de + dans une expression liée à l’entrée number, l’opérateur renvoie la somme des nombres (deuxième exemple), en affichant 10 à l’écran *.

* - C’est-à-dire jusqu’à ce que l’utilisateur modifie la valeur dans le champ de saisie, après quoi l’affichage changera en conséquence.

[Exemple de travail] (https://jsfiddle.net/fkfd9tar/)

si #

[ng-if][1] est une directive similaire à [ng-show][2] mais insère ou supprime l’élément du DOM au lieu de simplement le masquer. Angular 1.1.5 a introduit la directive ng-If. Vous pouvez utiliser la directive ng-if au-dessus des versions 1.1.5. Ceci est utile car Angular ne traitera pas les résumés des éléments à l’intérieur d’un ng-if supprimé, ce qui réduira la charge de travail d’Angular, en particulier pour les liaisons de données complexes.

Contrairement à ng-show, la directive ng-if crée une portée enfant qui utilise l’héritage prototypique. Cela signifie que la définition d’une valeur primitive sur la portée enfant ne s’appliquera pas au parent. Pour définir une primitive sur la portée parent, la propriété $parent sur la portée enfant devra être utilisée.

Javascript #

angular.module('MyApp', []);

angular.module('MyApp').controller('myController', ['$scope', '$window', function myController($scope, $window) {
    $scope.currentUser= $window.localStorage.getItem('userName');
}]);

Voir #

<div ng-controller="myController">
    <div ng-if="currentUser">
        Hello, {{currentUser}}
    </div>
    <div ng-if="!currentUser">
        <a href="/login">Log In</a>
        <a href="/register">Register</a>
    </div>
</div>

DOM si currentUser n’est pas indéfini #

<div ng-controller="myController">
    <div ng-if="currentUser">
        Hello, {{currentUser}}
    </div>
    <!-- ng-if: !currentUser -->
</div>

DOM si currentUser n’est pas défini #

<div ng-controller="myController">
    <!-- ng-if: currentUser -->
    <div ng-if="!currentUser">
        <a href="/login">Log In</a>
        <a href="/register">Register</a>
    </div>
</div>

[Exemple de travail] (https://jsfiddle.net/beekalam/4wwq1a3w/)

Promesse de fonction #

La directive ngIf accepte également les fonctions, qui nécessitent logiquement de renvoyer true ou false.

<div ng-if="myFunction()">
    <span>Span text</span>
</div>

Le texte span n’apparaîtra que si la fonction renvoie true.

$scope.myFunction = function() {
    var result = false;
    // Code to determine the boolean value of result 
    return result;
};

Comme toute expression angulaire, la fonction accepte tout type de variables.

[1] : https://docs.angularjs.org/api/ng/directive/ngIf [2] : https://docs.angularjs.org/api/ng/directive/ngShow

ngRepeat #

ng-repeat est une directive intégrée dans Angular qui vous permet d’itérer un tableau ou un objet et vous donne la possibilité de répéter un élément une fois pour chaque élément de la collection.

répète le tableau

<ul>
    <li ng-repeat="item in itemCollection">
       {{item.Name}}
    </li>
</ul>

Où :
item = article individuel de la collection
itemCollection = Le tableau que vous itérez


ng-répéter un objet

<ul>
    <li ng-repeat="(key, value) in myObject">
       {{key}} : {{value}}
    </li>
</ul>

Où :
clé = le nom de la propriété
valeur = la valeur de la propriété
myObject = l’objet que vous itérez


** filtrez votre ng-repeat par entrée utilisateur **

<input type="text" ng-model="searchText">
<ul>
    <li ng-repeat="string in stringArray | filter:searchText">
       {{string}}
    </li>
</ul>

Où :
searchText = le texte par lequel l’utilisateur souhaite filtrer la liste
stringArray = un tableau de chaînes, par ex. ['chaîne', 'tableau']

Vous pouvez également afficher ou référencer les éléments filtrés ailleurs en attribuant à la sortie du filtre un alias avec as aliasName, comme ceci :

<input type="text" ng-model="searchText">
<ul>
    <li ng-repeat="string in stringArray | filter:searchText as filteredStrings">
       {{string}}
    </li>
</ul>
<p>There are {{filteredStrings.length}} matching results</p>

ng-repeat-start et ng-repeat-end

Pour répéter plusieurs éléments DOM en définissant un point de départ et un point de fin, vous pouvez utiliser les directives ng-repeat-start et ng-repeat-end.

<ul>
    <li ng-repeat-start="item in [{a: 1, b: 2}, {a: 3, b:4}]">
        {{item.a}}
    </li>
    <li ng-repeat-end>
        {{item.b}}
    </li>
</ul>

Il est important de toujours fermer ng-repeat-start avec ng-repeat-end.

Variables

ng-repeat expose également ces variables à l’intérieur de l’expression

variables Taper Détails
$index Numéro Égal à l’index de l’itération actuelle ($index===0 sera évalué à true au premier élément itéré ; voir $first)
$premier Booléen Prend la valeur true au premier élément itéré
$dernier booléen Prend la valeur true au dernier élément itéré
$milieu booléen Est évalué à true si l’élément est entre $first et $last
$pair booléen Est évalué à vrai à une itération paire (équivalent à $index%2===0)
$odd booléen Renvoie vrai à une itération impaire (équivalent à $index%2===1)

Considérations relatives aux performances

Le rendu de ngRepeat peut devenir lent, en particulier lors de l’utilisation de grandes collections.

Si les objets de la collection ont une propriété d’identifiant, vous devez toujours “suivre par” l’identifiant au lieu de l’objet entier, qui est la fonctionnalité par défaut. Si aucun identifiant n’est présent, vous pouvez toujours utiliser le $index intégré.

<div ng-repeat="item in itemCollection track by item.id">
<div ng-repeat="item in itemCollection track by $index">

Portée de ngRepeat

ngRepeat créera toujours une portée enfant isolée, il faut donc faire attention si la portée parent doit être accessible à l’intérieur de la répétition.

Voici un exemple simple montrant comment vous pouvez définir une valeur dans votre portée parent à partir d’un événement de clic à l’intérieur de ngRepeat.

scope val:  {{val}}<br/>
ctrlAs val: {{ctrl.val}}
<ul>
    <li ng-repeat="item in itemCollection">
        <a href="#" ng-click="$parent.val=item.value; ctrl.val=item.value;">
            {{item.label}} {{item.value}}
        </a>
    </li>
</ul>

$scope.val = 0;
this.val = 0;

$scope.itemCollection = [{
    id: 0,
    value: 4.99,
    label: 'Football'
},
{
    id: 1,
    value: 6.99,
    label: 'Baseball'
},
{
    id: 2,
    value: 9.99,
    label: 'Basketball'
}];

S’il n’y avait que val = item.value à ng-click, il ne mettra pas à jour le val dans la portée parent en raison de la portée isolée. C’est pourquoi la portée parent est accessible avec la référence $parent ou avec la syntaxe controllerAs (par exemple, ng-controller="mainController as ctrl").

Imbriqué par répétition

Vous pouvez également utiliser ng-repeat imbriqué.

<div ng-repeat="values in test">
    <div ng-repeat="i in values">
      [{{$parent.$index}},{{$index}}] {{i}}
    </div>
</div>

var app = angular.module("myApp", []);
app.controller("ctrl", function($scope) {
  $scope.test = [
    ['a', 'b', 'c'],
    ['d', 'e', 'f']
  ];
});

Ici, pour accéder à l’index du parent ng-repeat à l’intérieur de l’enfant ng-repeat, vous pouvez utiliser $parent.$index.

ngCloak #

La directive ngCloak est utilisée pour empêcher le modèle HTML angulaire d’être brièvement affiché par le navigateur dans sa version brute (non compilée) formulaire pendant le chargement de votre application. - [Afficher la source][2]

HTML

<div ng-cloak>
  <h1>Hello {{ name }}</h1>
</div>

ngCloak peut être appliqué à l’élément body, mais le préféré l’utilisation est d’appliquer plusieurs directives ngCloak à de petites portions du page pour permettre un rendu progressif de la vue du navigateur.

La directive ngCloak n’a pas de paramètres.

Voir aussi : [Prévenir le scintillement][1]

[1] : http://www.ng-newsletter.com/25-days-of-angular/day-3 [2] : https://docs.angularjs.org/api/ng/directive/ngCloak

of-model-options #

ng-model-options permet de changer le comportement par défaut de ng-model, cette directive permet d’enregistrer des événements qui se déclencheront lorsque le ng-model est mis à jour et d’attacher un effet anti-rebond.

Cette directive accepte une expression qui évaluera un objet de définition ou une référence à une valeur de portée.

Exemple:

<input type="text" ng-model="myValue" ng-model-options="{'debounce': 500}">

L’exemple ci-dessus attachera un effet anti-rebond de 500 millisecondes sur myValue, ce qui entraînera la mise à jour du modèle 500 ms après que l’utilisateur a fini de taper sur input (c’est-à-dire lorsque myValue a terminé la mise à jour).

Propriétés d’objet disponibles

  1. updateOn : spécifie quel événement doit être lié à l’entrée

    ng-model-options="{ updateOn: 'blur'}" // will update on blur

  2. debounce : spécifie un délai de quelques millisecondes vers la mise à jour du modèle

    ng-model-options="{'debounce': 500}" // will update the model after 1/2 second

  3. allowInvalid : un indicateur booléen autorisant une valeur invalide pour le modèle, contournant la validation du formulaire par défaut, par défaut ces valeurs seraient traitées comme undefined.

  4. getterSetter : un indicateur booléen indiquant s’il faut traiter le ng-model comme une fonction getter/setter au lieu d’une simple valeur de modèle. La fonction s’exécutera alors et renverra la valeur du modèle.

Exemple:

   <input type="text" ng-model="myFunc" ng-model-options="{'getterSetter': true}">

   $scope.myFunc = function() {return "value";}
  1. timezone : définit le fuseau horaire pour le modèle si l’entrée est de la date ou time. les types

ngInclude #

ng-include vous permet de déléguer le contrôle d’une partie de la page à un contrôleur spécifique. Vous voudrez peut-être le faire parce que la complexité de ce composant devient telle que vous souhaitez encapsuler toute la logique dans un contrôleur dédié.

Un exemple est :

  <div ng-include
       src="'/gridview'"
       ng-controller='gridController as gc'>
  </div>

Notez que /gridview devra être servi par le serveur Web en tant qu’URL distincte et légitime.

Notez également que l’attribut src accepte une expression angulaire. Cela pourrait être une variable ou un appel de fonction par exemple ou, comme dans cet exemple, une constante de chaîne. Dans ce cas, vous devez vous assurer de envelopper l’URL source entre guillemets simples, afin qu’elle soit évaluée comme une constante de chaîne. C’est une source courante de confusion.

Dans le /gridview html, vous pouvez vous référer au gridController comme s’il était enroulé autour de la page, par exemple :

<div class="row">
  <button type="button" class="btn btn-default" ng-click="gc.doSomething()"></button>
</div>

ngCopie #

La directive ngCopy spécifie le comportement à exécuter sur un événement de copie.

Empêcher un utilisateur de copier des données #

<p ng-copy="blockCopy($event)">This paragraph cannot be copied</p>

Dans le contrôleur

$scope.blockCopy = function(event) {
    event.preventDefault();
    console.log("Copying won't work");
}

ngClic #

La directive ng-click attache un événement click à un élément DOM.

La directive ng-click vous permet de spécifier un comportement personnalisé lorsqu’un élément du DOM est cliqué.

C’est utile lorsque vous souhaitez attacher des événements de clic sur des boutons et les gérer sur votre contrôleur.

Cette directive accepte une expression avec l’objet d’événements disponible en tant que $event

HTML

<input ng-click="onClick($event)">Click me</input>

Manette

.controller("ctrl", function($scope) {   
    $scope.onClick = function(evt) {
        console.debug("Hello click event: %o ",evt);
    } 
})

HTML

<button ng-click="count = count + 1" ng-init="count=0">
  Increment
</button>
<span>
  count: {{count}}
</span>

HTML

<button ng-click="count()" ng-init="count=0">
  Increment
</button>
<span>
  count: {{count}}
</span>

Manette

...

$scope.count = function(){
    $scope.count = $scope.count + 1;
}
...

Lorsque le bouton est cliqué, une invocation de la fonction onClick affichera “Hello click event” suivi de l’objet événement.

ngColler #

La directive ngPaste spécifie le comportement personnalisé à exécuter lorsqu’un utilisateur colle du contenu

<input ng-paste="paste=true" ng-init="paste=false" placeholder='paste here'>
pasted: {{paste}}

ngOptions #

ngOptions est une directive qui simplifie la création d’une liste déroulante html pour la sélection d’un élément dans un tableau qui sera stocké dans un modèle. L’attribut ngOptions est utilisé pour générer dynamiquement une liste d’éléments <option> pour l’élément <select> en utilisant le tableau ou l’objet obtenu en évaluant l’expression de compréhension ngOptions.

Avec ng-options, le balisage peut être réduit à une simple balise select et la directive créera le même select :

<select ng-model="selectedFruitNgOptions" 
        ng-options="curFruit as curFruit.label for curFruit in fruit">
</select>

Il existe une autre façon de créer des options select en utilisant ng-repeat, mais il n’est pas recommandé d’utiliser ng-repeat car il est principalement utilisé à des fins générales comme le forEach juste pour boucler. Alors que ng-options est spécifiquement destiné à créer des options de balise select.

L’exemple ci-dessus utilisant ng-repeat serait

<select ng-model="selectedFruit">
    <option ng-repeat="curFruit in fruit" value="{{curFruit}}">
        {{curFruit.label}}
    </option>
</select>

EXEMPLE COMPLET

Voyons l’exemple ci-dessus en détail avec quelques variantes.

Modèle de données pour l’exemple :

$scope.fruit = [
    { label: "Apples", value: 4, id: 2 },

ngModel #

Avec ng-model, vous pouvez lier une variable à n’importe quel type de champ de saisie. Vous pouvez afficher la variable en utilisant des accolades doubles, par exemple {{myAge}}.

<input type="text" ng-model="myName">
<p>{{myName}}</p>

Au fur et à mesure que vous tapez dans le champ de saisie ou que vous le modifiez de quelque manière que ce soit, vous verrez la valeur dans le paragraphe se mettre à jour instantanément.

La variable ng-model, dans ce cas, sera disponible dans votre contrôleur en tant que $scope.myName. Si vous utilisez la syntaxe controllerAs :

<div ng-controller="myCtrl as mc">
    <input type="text" ng-model="mc.myName">
    <p>{{mc.myName}}</p>
</div>

Vous devrez faire référence à la portée du contrôleur en ajoutant l’alias du contrôleur défini dans l’attribut ng-controller à la variable ng-model. De cette façon, vous n’aurez pas besoin d’injecter $scope dans votre contrôleur pour référencer votre variable ng-model, la variable sera disponible en tant que this.myName dans la fonction de votre contrôleur.

ngClass #

Supposons que vous ayez besoin d’afficher le statut d’un utilisateur et que vous ayez plusieurs classes CSS possibles qui pourraient être utilisées. Angular permet de choisir très facilement parmi une liste de plusieurs classes possibles qui vous permettent de spécifier une liste d’objets incluant des conditions. Angular est capable d’utiliser la bonne classe en fonction de la véracité des conditions.

Votre objet doit contenir des paires clé/valeur. La clé est un nom de classe qui sera appliqué lorsque la valeur (conditionnelle) sera évaluée à true.

<style>
    .active { background-color: green; color: white; }
    .inactive { background-color: gray; color: white; }
    .adminUser { font-weight: bold; color: yellow; }
    .regularUser { color: white; }
</style>

<span ng-class="{ 
    active: user.active, 
    inactive: !user.active, 
    adminUser: user.level === 1, 
    regularUser: user.level === 2 
}">John Smith</span>

Angular vérifiera l’objet $scope.user pour voir le statut actif et le numéro de niveau. En fonction des valeurs de ces variables, Angular appliquera le style correspondant au <span>.

ngSrc #

L’utilisation d’un balisage angulaire comme {{hash}} dans un attribut src ne fonctionne pas correctement. Le navigateur récupérera l’URL avec le texte littéral {{hash}} jusqu’à ce qu’Angular remplace l’expression à l’intérieur de {{hash}}. La directive ng-src remplace l’attribut src d’origine pour l’élément de balise d’image et résout le problème

<div ng-init="pic = 'pic_angular.jpg'">
    <h1>Angular</h1>
    <img ng-src="{{pic}}">
</div>

ngListe #

La directive ng-list est utilisée pour convertir une chaîne délimitée d’une entrée de texte en un tableau de chaînes ou vice versa.

La directive ng-list utilise un délimiteur par défaut de ", " (espace virgule).

Vous pouvez définir le délimiteur manuellement en attribuant à ng-list un délimiteur comme celui-ci ng-list="; ".

Dans ce cas, le délimiteur est défini sur un point-virgule suivi d’un espace.

Par défaut, ng-list a un attribut ng-trim qui est défini sur true. ng-trim lorsqu’il est faux, respectera l’espace blanc dans votre délimiteur. Par défaut, ng-list ne prend pas en compte les espaces blancs à moins que vous ne définissiez ng-trim="false".

Exemple:

angular.module('test', [])
  .controller('ngListExample', ['$scope', function($scope) {
    $scope.list = ['angular', 'is', 'cool!'];
}]);

Un délimiteur de client est défini sur  ; . Et le modèle de la zone de saisie est défini sur le tableau qui a été créé sur l’oscilloscope.

  <body ng-app="test" ng-controller="ngListExample">
    <input ng-model="list" ng-list="; " ng-trim="false">
  </body>

La zone de saisie s’affichera avec le contenu : angular ; est; sympa !

ngShow et ngHide #

La directive ng-show affiche ou masque l’élément HTML selon que l’expression qui lui est transmise est vraie ou fausse. Si la valeur de l’expression est fausse, elle sera masquée. Si c’est vrai, ça se verra.

La directive ng-hide est similaire. Cependant, si la valeur est fausse, l’élément HTML s’affichera. Lorsque l’expression est véridique, elle la cachera.

[Exemple JSBin de travail][1]

Manette:

var app = angular.module('app', []);
  
angular.module('app')
  .controller('ExampleController', ExampleController);
 
function ExampleController() {
  
  var vm = this;
  
  //Binding the username to HTML element
  vm.username = '';
  
  //A taken username
  vm.taken_username = 'StackOverflow';
  
}

Voir

<section ng-controller="ExampleController as main">
    
    <p>Enter Password</p>
    <input ng-model="main.username" type="text">
    
    <hr>
    
    <!-- Will always show as long as StackOverflow is not typed in -->
    <!-- The expression is always true when it is not StackOverflow -->
    <div style="color:green;" ng-show="main.username != main.taken_username">
      Your username is free to use!
    </div>
    
    <!-- Will only show when StackOverflow is typed in -->
    <!-- The expression value becomes falsy -->
    <div style="color:red;" ng-hide="main.username != main.taken_username">
      Your username is taken!
    </div>
    
    <p>Enter 'StackOverflow' in username field to show ngHide directive.</p>
    
 </section>

ngDblclick #

La directive ng-dblclick est utile lorsque vous souhaitez lier un événement de double-clic dans vos éléments DOM.

Cette directive accepte une expression

HTML

<input type="number" ng-model="num = num + 1" ng-init="num=0">

<button ng-dblclick="num++">Double click me</button>

Dans l’exemple ci-dessus, la valeur maintenue à l’entrée sera incrémentée lorsque le bouton sera double-cliqué.

ngModèle #

La directive ng-pattern accepte une expression qui correspond à un modèle d’expression régulière et utilise ce modèle pour valider une entrée textuelle.

Exemple:

Disons que nous voulons qu’un élément <input> devienne valide lorsque sa valeur (ng-model) est une adresse IP valide.

Modèle:

<input type="text" ng-model="ipAddr" ng-pattern="ipRegex" name="ip" required>

Manette:

$scope.ipRegex = /\b(?:(?:25[0-5]|2[0-4][0-9]|[01]?[0-9][0-9]?)\.){3}(?:25[0-5]|2[0-4][0-9]|[01]?[0-9][0-9]?)\b/;

ngRef #

ngHref est utilisé à la place de l’attribut href, si nous avons une expression angulaire à l’intérieur de la valeur href. La directive ngHref remplace l’attribut href d’origine d’une balise html en utilisant l’attribut href tel que la balise , la balise etc.

La directive ngHref garantit que le lien n’est pas rompu même si l’utilisateur clique sur le lien avant qu’AngularJS n’ait évalué le code.

ngMouseenter et ngMouseleave #

Les directives ng-mouseenter et ng-mouseleave sont utiles pour exécuter des événements et appliquer un style CSS lorsque vous survolez dans ou hors de vos éléments DOM.

La directive ng-mouseenter exécute une expression sur un événement d’entrée de souris (lorsque l’utilisateur entre le pointeur de sa souris sur l’élément DOM dans lequel réside cette directive)

HTML

<div ng-mouseenter="applyStyle = true" ng-class="{'active': applyStyle}">

Dans l’exemple ci-dessus, lorsque l’utilisateur pointe sa souris sur la div, applyStyle se transforme en true, qui à son tour applique la classe CSS .active à la ng-class.

La directive ng-mouseleave exécute une expression sur un événement de sortie de la souris (lorsque l’utilisateur éloigne le curseur de la souris de l’élément DOM dans lequel cette directive réside)

HTML

<div ng-mouseenter="applyStyle = true" ng-mouseleaver="applyStyle = false" ng-class="{'active': applyStyle}">

En réutilisant le premier exemple, maintenant, lorsque l’utilisateur éloigne le pointeur de la souris de la div, la classe .active est supprimée.

ngDisabled #

Cette directive est utile pour limiter les événements d’entrée en fonction de certaines conditions existantes.

La directive ng-disabled accepte une expression qui doit être évaluée à une valeur de vérité ou de fausseté.

ng-disabled est utilisé pour appliquer conditionnellement l’attribut disabled sur un élément input.

HTML

<input type="text" ng-model="vm.name">

<button ng-disabled="vm.name.length===0" ng-click="vm.submitMe">Submit</button>

vm.name.length===0 est évalué à true si la longueur de input est 0, ce qui désactive le bouton, empêchant l’utilisateur de déclencher l’événement click de ng-click

ngRequired #

Le ng-required ajoute ou supprime l’attribut de validation required sur un élément, qui à son tour activera et désactivera la clé de validation require pour input.

Il est utilisé pour définir éventuellement si un élément input doit avoir une valeur non vide. La directive est utile lors de la conception de la validation sur des formulaires HTML complexes.

HTML

<input type="checkbox" ng-model="someBooleanValue">
<input type="text" ng-model="username" ng-required="someBooleanValue">

ngValeur #

Principalement utilisé sous ng-repeat ngValue est utile lors de la génération dynamique de listes de boutons radio à l’aide de ngRepeat

<script>
   angular.module('valueExample', [])

     </label>

 </form>
