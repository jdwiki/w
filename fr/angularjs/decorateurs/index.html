Syntaxe #

  • décorateur(nom, décorateur);

Decorator est une fonction qui permet à un [service][1], [factory][2], [directive][3] ou [filtre][4] à modifier avant son utilisation. Le décorateur est utilisé pour remplacer ou modifier le comportement du service. La valeur de retour de la fonction décorateur peut être le service d’origine, ou un nouveau service qui remplace, ou enveloppe et délègue, le service d’origine.

Toute décoration doit être effectuée dans la phase config de l’application angulaire en injectant $provide et en utilisant sa fonction $provide.decorator.

La fonction décorateur a un objet $delegate injecté pour fournir accès au service qui correspond au sélecteur dans le décorateur. Cette $delegate sera le service que vous décorez. La valeur de retour de la fonction assurée au décorateur tiendra lieu de prestation, directive, ou filtre en cours de décoration.

Il ne faut envisager d’utiliser le décorateur que si toute autre approche n’est pas appropriée ou s’avère trop fastidieuse. Si une grande application utilise le même service et qu’une partie modifie le comportement du service, il est facile de créer de la confusion et/ou des bogues dans le processus.

Le cas d’utilisation typique serait lorsque vous avez une dépendance tierce que vous ne pouvez pas mettre à niveau mais dont vous avez besoin pour qu’elle fonctionne un peu différemment ou que vous l’étendiez.

[1] : https://www.wikiod.com/fr/angularjs/fournisseurs#Service [2] : https://www.wikiod.com/fr/angularjs/fournisseurs#Usine [3] : https://www.wikiod.com/fr/angularjs/directives-personnalisees [4] : https://www.wikiod.com/fr/angularjs/filtres

Service de décoration, usine #

Vous trouverez ci-dessous un exemple de décorateur de service, remplaçant la date “null” renvoyée par le service.

angular.module('app', [])
  .config(function($provide) {
    $provide.decorator('myService', function($delegate) {
      $delegate.getDate = function() { // override with actual date object
        return new Date();
      };
      return $delegate;
    });
  })
  .service('myService', function() {
    this.getDate = function() {
      return null; // w/o decoration we'll be returning null
    };
  })
  .controller('myController', function(myService) {
    var vm = this;
    vm.date = myService.getDate();
  });

<body ng-controller="myController as vm">
  <div ng-bind="vm.date | date:'fullDate'"></div>
</body>

Directive de décoration #

Les directives peuvent être décorées comme des services et nous pouvons modifier ou remplacer n’importe laquelle de ses fonctionnalités. Notez que la directive elle-même est accessible à la position 0 dans le tableau $delegate et que le paramètre name dans le décorateur doit inclure le suffixe Directive (sensible à la casse).

Ainsi, si la directive s’appelle myDate, on peut y accéder en utilisant myDateDirective en utilisant $delegate[0].

Vous trouverez ci-dessous un exemple simple où la directive indique l’heure actuelle. Nous allons le décorer pour mettre à jour l’heure actuelle par intervalles d’une seconde. Sans décoration, il affichera toujours la même heure.

<body>
  <my-date></my-date>
</body>

angular.module('app', [])
  .config(function($provide) {
    $provide.decorator('myDateDirective', function($delegate, $interval) {
      var directive = $delegate[0]; // access directive

      directive.compile = function() { // modify compile fn
        return function(scope) {
          directive.link.apply(this, arguments);
          $interval(function() {
            scope.date = new Date(); // update date every second
          }, 1000);
        };
      };
      
      return $delegate;
    });
  })
  .directive('myDate', function() {
    return {
      restrict: 'E',
      template: '<span>Current time is {{ date | date:\'MM:ss\' }}</span>',
      link: function(scope) {
        scope.date = new Date(); // get current date
      }
    };
  });

Décorer le filtre #

Lors de la décoration des filtres, le paramètre de nom doit inclure le suffixe “Filtre” (sensible à la casse). Si le filtre est appelé repeat, le paramètre de décorateur est repeatFilter. Ci-dessous, nous allons décorer un filtre personnalisé qui répète une chaîne donnée n fois afin que le résultat soit inversé. Vous pouvez également décorer les filtres intégrés d’angular de la même manière, bien que cela ne soit pas recommandé car cela peut affecter la fonctionnalité du framework.

<body>
  <div ng-bind="'i can haz cheeseburger ' | repeat:2"></div>
</body>

angular.module('app', [])
  .config(function($provide) {
    $provide.decorator('repeatFilter', function($delegate) {
      return function reverse(input, count) {
        // reverse repeated string
        return ($delegate(input, count)).split('').reverse().join(''); 
      };
    });
  })
  .filter('repeat', function() {
    return function(input, count) {
      // repeat string n times
      return (input || '').repeat(count || 1);
    };
  });
