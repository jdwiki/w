Syntaxe #

  • myApp.controller(‘MyController’, function($scope) { … }); // code non minifié
  • myApp.controller(‘MonContrôleur’, [’$scope’, function($scope) { … }]); // prend en charge la minification
  • fonction MonContrôleur(){}

MonContrôleur.$inject = [’$scope’] ;

monApp.controller(‘MonContrôleur’, MonContrôleur); // $injecter une annotation

  • $injecteur.get(‘injectable’); //injection dynamique/d’exécution

Les fournisseurs ne peuvent pas être injectés dans les blocs run.

Les services ou les valeurs ne peuvent pas être injectés dans les blocs config.

Assurez-vous d’annoter vos injections afin que votre code ne se brise pas lors de la minification.

Injections dynamiques #

Il existe également une option pour demander dynamiquement des composants. Vous pouvez le faire en utilisant le service $injector :

myModule.controller('myController', ['$injector', function($injector) {
    var myService = $injector.get('myService');
}]);

Remarque : bien que cette méthode puisse être utilisée pour éviter le problème de dépendance circulaire qui pourrait casser votre application, il n’est pas considéré comme une bonne pratique de contourner le problème en l’utilisant. Une dépendance circulaire indique généralement qu’il y a un défaut dans l’architecture de votre application, et vous devez plutôt le résoudre.

Charger dynamiquement le service AngularJS en JavaScript vanille #

Vous pouvez charger des services AngularJS en JavaScript vanille à l’aide de la méthode AngularJS injector(). Chaque élément jqLite récupéré en appelant angular.element() a une méthode injector() qui peut être utilisée pour récupérer l’injecteur.

var service;
var serviceName = 'myService';

var ngAppElement = angular.element(document.querySelector('[ng-app],[data-ng-app]') || document);
var injector = ngAppElement.injector();

if(injector && injector.has(serviceNameToInject)) {
    service = injector.get(serviceNameToInject);
}

Dans l’exemple ci-dessus, nous essayons de récupérer l’élément jqLite contenant la racine de l’application AngularJS (ngAppElement). Pour ce faire, nous utilisons la méthode angular.element(), recherchant un élément DOM contenant l’attribut ng-app ou data-ng-app ou, s’il n’existe pas, nous revenons à document élément. Nous utilisons ngAppElement pour récupérer l’instance de l’injecteur (avec ngAppElement.injector()). L’instance d’injecteur est utilisée pour vérifier si le service à injecter existe (avec injector.has()) puis pour charger le service (avec injector.get()) dans la variable service.

Injections #

L’exemple le plus simple d’une injection dans une application Angular - injecter $scope à un Controller Angular :

angular.module('myModule', [])
.controller('myController', ['$scope', function($scope) {
    $scope.members = ['Alice', 'Bob'];
    ...
}])

Ce qui précède illustre une injection d’un $scope dans un controller, mais c’est la même chose si vous injectez un module dans un autre. Le processus est le même.

Le système d’Angular est chargé de résoudre les dépendances pour vous. Si vous créez un service par exemple, vous pouvez le lister comme dans l’exemple ci-dessus et il sera disponible pour vous.

Vous pouvez utiliser DI - Dependency Injection - partout où vous définissez un composant.

Notez que dans l’exemple ci-dessus, nous utilisons ce qu’on appelle “Inline Array Annotation”. Cela signifie que nous écrivons explicitement sous forme de chaînes les noms de nos dépendances. Nous le faisons pour empêcher l’application de se casser lorsque le code est minifié pour la production. La minification de code modifie les noms des variables (mais pas les chaînes), ce qui interrompt l’injection. En utilisant des chaînes, Angular sait quelles dépendances nous voulons.

Très important - l’ordre des noms de chaînes doit être le même que celui des paramètres de la fonction.

Il existe des outils qui automatisent ce processus et s’en occupent pour vous.

Annotation de la propriété $inject #

De manière équivalente, nous pouvons utiliser l’annotation de propriété $inject pour obtenir la même chose que ci-dessus :

var MyController = function($scope) {
  // ...
}
MyController.$inject = ['$scope'];
myModule.controller('MyController', MyController);
