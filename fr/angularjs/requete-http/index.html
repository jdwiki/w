Utilisation de $http dans un contrôleur #

Le service $http est une fonction qui génère une requête HTTP et renvoie une promesse.

Utilisation générale

// Simple GET request example:
$http({
  method: 'GET',
  url: '/someUrl'
}).then(function successCallback(response) {
    // this callback will be called asynchronously
    // when the response is available
  }, function errorCallback(response) {
    // called asynchronously if an error occurs
    // or server returns response with an error status.
  });

Utilisation à l’intérieur du contrôleur

appName.controller('controllerName',
    ['$http', function($http){

    // Simple GET request example:
    $http({
        method: 'GET',
        url: '/someUrl'
    }).then(function successCallback(response) {
        // this callback will be called asynchronously
        // when the response is available
    }, function errorCallback(response) {
        // called asynchronously if an error occurs
        // or server returns response with an error status.
    });
}])

Méthodes de raccourci

Le service $http a également des méthodes de raccourci. Lisez à propos des méthodes http ici

Syntaxe

$http.get('/someUrl', config).then(successCallback, errorCallback);
$http.post('/someUrl', data, config).then(successCallback, errorCallback);

Méthodes raccourcies

  • $http.get
  • $http.tête
  • $http.post
  • $http.put
  • $http.delete
  • $http.jsonp
  • $http.patch

Utilisation de la requête $http dans un service #

Les requêtes HTTP sont largement utilisées à plusieurs reprises dans toutes les applications Web. Il est donc judicieux d’écrire une méthode pour chaque requête courante, puis de l’utiliser à plusieurs endroits dans l’application.

Créer un httpRequestsService.js

httpRequestsService.js

appName.service('httpRequestsService', function($q, $http){

    return {
        // function that performs a basic get request
        getName: function(){
            // make sure $http is injected
            return $http.get("/someAPI/names")
                .then(function(response) {
                    // return the result as a promise
                    return response;
                }, function(response) {
                    // defer the promise
                    return $q.reject(response.data);
                });
        },

        // add functions for other requests made by your app
        addName: function(){
            // some code...
        }
    }
})

Le service ci-dessus effectuera une requête get à l’intérieur du service. Cela sera disponible pour tout contrôleur où le service a été injecté.

Exemple d’utilisation

appName.controller('controllerName',
    ['httpRequestsService', function(httpRequestsService){

        // we injected httpRequestsService service on this controller
        // that made the getName() function available to use.
        httpRequestsService.getName()
            .then(function(response){
                // success
            }, function(error){
                // do something with the error
            })
    }])

En utilisant cette approche, nous pouvons désormais utiliser httpRequestsService.js à tout moment et dans n’importe quel contrôleur.

Timing d’une requête $http #

Les requêtes $http nécessitent un temps qui varie selon le serveur, certaines peuvent prendre quelques millisecondes, et certaines peuvent prendre jusqu’à quelques secondes. Souvent, le temps nécessaire pour récupérer les données d’une requête est critique. En supposant que la valeur de réponse est un tableau de noms, considérez l’exemple suivant :

Incorrect

$scope.names = [];

$http({
    method: 'GET',
    url: '/someURL'
}).then(function successCallback(response) {
            $scope.names = response.data;
        },
        function errorCallback(response) {
            alert(response.status);
        });

alert("The first name is: " + $scope.names[0]);

Accéder à $scope.names[0] juste en dessous de la requête $http génère souvent une erreur - cette ligne de code s’exécute avant que la réponse ne soit reçue du serveur.

Corriger

$scope.names = [];

$scope.$watch('names', function(newVal, oldVal) {
    if(!(newVal.length == 0)) {
        alert("The first name is: " + $scope.names[0]);
    }
});

$http({
    method: 'GET',
    url: '/someURL'
}).then(function successCallback(response) {
            $scope.names = response.data;
        },
        function errorCallback(response) {
            alert(response.status);
        });

En utilisant le service [$watch][1], nous accédons au tableau $scope.names uniquement lorsque la réponse est reçue. Lors de l’initialisation, la fonction est appelée même si $scope.names a été initialisé auparavant, il est donc nécessaire de vérifier si newVal.length est différent de 0. Soyez conscient - toute modification apportée à $scope.names déclenchera la fonction de surveillance.

[1] : https://www.wikiod.com/fr/angularjs/procedure-pas-a-pas-de-la-boucle-de-digestion#$digest et $watch
