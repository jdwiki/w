Que sont les registres ?

Le processeur peut fonctionner sur des valeurs numériques (nombres), mais celles-ci doivent d’abord être stockées quelque part. Les données sont stockées principalement en mémoire, ou à l’intérieur de l’opcode d’instruction (qui est généralement stocké également en mémoire), ou dans une mémoire spéciale sur puce placée directement dans le processeur, appelée registre.

Pour travailler avec la valeur dans le registre, vous n’avez pas besoin de l’adresser par adresse, mais des “noms” mnémoniques spéciaux sont utilisés, comme par exemple ax sur x86, ou A sur Z80, ou r0 sur ARM.

Certains processeurs sont construits d’une manière où presque tous les registres sont égaux et peuvent être utilisés à toutes fins (souvent groupe de processeurs RISC), d’autres ont une spécialisation distincte, alors que seuls certains registres peuvent être utilisés pour l’arithmétique (“accumulateur” sur premiers processeurs) et d’autres registres pour l’adressage mémoire uniquement, etc.

Cette construction utilisant la mémoire directement sur la puce du processeur a une énorme implication sur les performances, l’ajout de deux nombres à partir des registres le stockant dans le registre est généralement effectué dans les plus brefs délais par ce processeur (exemple sur le processeur ARM : ADD r2, r0, r1 sets r2 à la valeur (r0 + r1), dans un seul cycle de processeur).

Au contraire, lorsque l’un des opérandes fait référence à un emplacement mémoire, le processeur peut caler pendant un certain temps, attendant que la valeur arrive de la puce mémoire (sur x86, cela peut aller de zéro attente de valeurs dans le cache L0 à des centaines de Le processeur effectue un cycle lorsque la valeur n’est dans aucun cache et doit être lue directement à partir de la puce mémoire DRAM).

Ainsi, lorsque le programmeur crée un code de traitement de données, il souhaite généralement disposer de toutes les données pendant le traitement dans des registres pour obtenir les meilleures performances. Si ce n’est pas possible et que des lectures/écritures en mémoire sont nécessaires, celles-ci doivent être minimisées et former un modèle qui coopère avec l’architecture des caches/mémoire de la plate-forme particulière.

La taille native du registre en bits est souvent utilisée pour regrouper les processeurs, comme Z80 étant “processeur 8 bits”, et 80386 étant “processeur 32 bits” - bien que ce groupement soit rarement clair. Par exemple, Z80 fonctionne également avec des paires de registres, formant une valeur 16 bits native, et le processeur 32 bits 80686 a des instructions MMX pour fonctionner avec des registres 64 bits de manière native.

Registres x64 #

L’architecture x64 est l’évolution de l’ancienne architecture x86, elle a gardé la compatibilité avec son prédécesseur (les registres x86 sont toujours disponibles) mais elle a aussi introduit de nouvelles fonctionnalités :

  • Les registres ont maintenant une capacité de 64 bits ;
  • Il existe 8 registres plus généraux ;
  • Les registres de segments sont forcés à 0 en mode 64 bits ;
  • Les 32, 16 et 8 bits inférieurs de chaque registre sont maintenant disponibles.

Usage général

S’inscrire Nom Sous-registres (bits)
RAX Accumulateur EAX(32), AX(16), AH(8), AL(8)
, RBX base EBX(32), BX(16), BH(8), BL(8)
RCX Comptoir ECX(32), CX(16), CH(8), CL(8)
RDX Données EDX(32), DX(16), DH(8), DL(8)
RSE source ESI (32), SI (16), SL (8)
RDI destination EDI(32), DI(16), DL(8)
RBP Pointeur de base EBP(32), BP(16), BPL(8)
RER Pointeur de pile ESP(32), SP(16), SPL(8)
R8-R15 Nouveaux registres R8D-R15D(32), R8W-R15W(16), R8B-R15B(8)

Noter

Les suffixes utilisés pour adresser les bits inférieurs des nouveaux registres signifient :

  • octet B, 8 bits ;
  • mot W, 16 bits ;
  • D double mot, 32 bits.

Registres Zilog Z80 #

Registres : 8 bits : A, B, C, D, E, H, L, F, I, R, 16 bits : SP , PC, IX, IY et les ombres de certains registres 8b : A', B', C', D', E', H' , L' et F'.

La plupart des registres 8 bits peuvent également être utilisés par paires en tant que registres 16 bits : AF, BC, DE et HL.

SP est pointeur de pile, marquant le bas de la mémoire de pile (utilisé par les instructions PUSH/POP/CALL/RET). PC est compteur de programme, pointant vers l’instruction en cours d’exécution. I est registre d’interruption, fournissant l’octet de poids fort de l’adresse de la table vectorielle pour le mode d’interruption IM 2. R est registre de rafraîchissement, il s’incrémente à chaque fois que le CPU récupère un opcode (ou un préfixe d’opcode). Certaines instructions non officielles existent sur certains processeurs Z80 pour manipuler les parties 8 bits de IX en tant que IXH:IXL et IY en tant que IYH:IYL.

Aucune instruction ne peut accéder directement aux variantes d’ombre, l’instruction EX AF, AF' permutera entre AF et AF', et l’instruction EXX permutera BC,DE,HL avec BC ',DE',HL'.

Chargement de valeur dans un registre :

    ; from other register
    LD   I,A        ; copies value in A into I (8 bit)
    LD   BC,HL      ; copies value in HL into BC (16 bit)
    ; directly with value encoded in instruction machine code
    LD   B,d8       ; 8b value d8 into B
    LD   DE,d16     ; 16b value d16 into DE
    ; from a memory (ROM/RAM)
    LD   A,(HL)     ; value from memory addressed by HL into A
    LD   A,(a16)    ; value from memory with address a16 into A
    LD   HL,(a16)   ; 16b value from memory with address a16 into HL
    POP  IX         ; 16b value popped from stack into IX
    LD   A,(IY+a8)  ; IX and IY allows addressing with 8b offset
    ; from I/O port (for writing value at I/O port use "OUT")
    IN   A,(C)      ; reads I/O port C, value goes to A

Les combinaisons correctes d’opérandes source et destination possibles sont limitées (par exemple, LD H,(a16) n’existe pas).

Stocker une valeur dans une mémoire :

    LD   (HL),D     ; value D stored into memory addressed by HL
    LD   (a16),A    ; value A into memory with address a16
    LD   (a16),HL   ; value HL into 16b of memory with address a16
    LD   (IX+a8),d8 ; value d8 into memory at address IX+a8
    LD   (IY+a8),B  ; value B into memory at address IY+a8
    ; specials ;)
    PUSH DE         ; 16b value DE pushed to stack
    CALL a16        ; while primarily used for execution branching
      ; it also stores next instruction address into stack

Registres x86 #

Dans le monde 32 bits, les registres à usage général se répartissent en trois classes générales : les registres à usage général 16 bits, les registres étendus 32 bits registres à usage général et les moitiés de registre 8 bits. Ces trois classes ne représentent pas du tout trois ensembles de registres entièrement distincts. Le 16 bits et Les registres 8 bits sont en fait des noms de régions à l’intérieur des registres 32 bits. S’inscrire la croissance de la famille de processeurs x86 s’est produite en étendant les registres existants dans processeurs plus anciens

Il existe huit registres à usage général 16 bits : AX, BX, CX, DX, BP, SI, DI, et SP; et vous pouvez y placer n’importe quelle valeur qui peut être exprimée en 16 bits ou moins.

Quand Intel a étendu l’architecture x86 à 32 bits en 1986, il a doublé la taille des huit registres et leur a donné de nouveaux noms en préfixant un E devant chaque nom de registre, résultant en EAX, EBX, ECX, EDX, EBP, ESI, EDI et ESP.

Avec x86_64 est venu un autre doublement de la taille des registres, ainsi que l’ajout de quelques nouveaux registres. Ces registres ont une largeur de 64 bits et sont nommés (barre oblique utilisée pour indiquer un autre nom de registre) : RAX/r0, RBX/r3, RCX/r1, RDX/r2, RBP/r5, RSI/r6, RDI/r7, RSP/r4 , R8, R9, R10, R11, R12, R13, R14, R15.

Bien que les registres à usage général puissent techniquement être utilisés pour n’importe quoi, chaque registre a également un objectif alternatif/principal :

  • AX (accumulateur) est utilisé dans les opérations arithmétiques.
  • CX (compteur) est utilisé dans les instructions de décalage et de rotation, et utilisé pour les boucles.
  • DX (données) est utilisé dans les opérations arithmétiques et d’E/S.
  • BX (base) utilisé comme pointeur vers des données (en particulier comme décalage vers le registre de segment DS en mode segmenté).
  • SP (pile) pointe vers le haut de la pile.
  • BP (stack base) pointe vers la base de la pile.
  • SI (source) pointe vers une source en mémoire pour les opérations de flux (par exemple, lodsb).
  • DI (destination) pointe vers une destination en mémoire pour les opérations de flux (par exemple, stosb).

Les registres de segment, utilisés en mode segmenté, pointent vers différents segments en mémoire. Chaque registre de segment de 16 bits donne une vue sur 64k (16 bits) de données. Une fois qu’un registre de segment a été défini pour pointer vers un bloc de mémoire, les registres (tels que BX, SI et DI) peuvent être utilisés comme décalages vers le registre de segment afin que des emplacements spécifiques dans l’espace 64k puissent être accédé.

Les six registres de segments et leurs utilisations sont :

S’inscrire Nom complet Descriptif
SS Segment de pile Points à la pile
CS Segment de code Utilisé par le CPU pour récupérer le code
DS Segment de données Registre par défaut pour les opérations MOV
ES Segment supplémentaire Segment de données supplémentaire
FS Segment supplémentaire Segment de données supplémentaire
GS Segment supplémentaire Segment de données supplémentaire
