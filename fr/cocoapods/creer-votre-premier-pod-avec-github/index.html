Création de votre premier pod avec Github #

** Dépôt ouvert **

  1. Créer nouveau dépôt (Mon conseil, ajoutez d’abord des pods sur un dépôt temporaire avant de les ajouter à votre projet d’origine)
  2. Remplissez le nom du référentiel avec votre nom de dépôt. J’utiliserai “myrepo” pour ce didacticiel et j’utiliserai les symboles “<” et “>” pour les parties que vous devriez changer pour les vôtres.
  3. Cliquez sur “Initialiser ce référentiel avec un README”
  4. Ajoutez une licence. Je vais utiliser “MIT” pour ce tutoriel.

** Tirez votre dépôt vers le local **

  1. Ouvrez Terminal.
  2. Allez où votre dépôt est cd </your/path/<myrepo> ou créez-le avec cd </your/path>. mkdir <mondépôt>. cd <mondépôt>
  3. Ajoutez git à votre dossier avec git init
  4. Liez votre référentiel avec git remote add origin https://github.com/<username>/<myrepo>.git. (Vous pouvez vérifier si cela fonctionne avec git remote -v (pas git remove -v))
  5. Tirez votre repo de github avec git pull
  6. Tapez : git pull origin master
  7. Tapez git commit

Ajouter des cocopodes

  1. Installez les cocopods : “sudo gem install cocopods”. Écrivez pod --help pour vérifier si cela fonctionne. (Sauter si vous avez déjà des cacaopodes)
  2. Créez <myrepo>.podspec avec touch <myrepo>.podspec
  3. Ouvrez <myrepo>.podspec avec un éditeur. Utilisez Xcode ou Sublime Text.
  4. Collez ce qui suit. Vous pouvez consulter cet exemple. Pour plus de détails, consultez la documentation
Pod ::Spec.new faire |s|
s.name = "<mondépôt>"
s.version = "0.1"
s.summary = "<Mon résumé>"
s.description = "<DOIT ÊTRE PLUS LONG QUE RÉSUMÉ>"
s.homepage = "https://github.com/<username>/<myrepo>"
s.licence = 'MY'
s.author = { "<username>" => "<useremail>" }
s.source = { :git => "https://github.com/<username>/<myrepo>.git", :tag => s.version.to_s }
s.platform = :ios, '8.0'
s.requires_arc = true

# Si plus d'un fichier source : https://guides.cocoapods.org/syntax/podspec.html#source_files
s.source_files = '<mondépôt>.swift'

fin
  1. Écrivez pod lib lint <myrepo>.podspec au terminal. Si vous avez des erreurs, écrivez pod lib lint <myrepo>.podspec --verbose. Résolvez vos erreurs (la résolution de vos erreurs peut prendre un certain temps ne peut pas vous aider ici).
  2. Ajoutez de nouvelles modifications à git avec git add . puis validez avec git commit -m "Added pods"
  3. Poussez vos modifications vers votre référentiel distant avec git push
  4. Créez une nouvelle version https://github.com// Cliquez sur Releases -> Cliquez sur New Version
  5. Écrivez “0.1” dans Tag Version. (Facultatif, vous pouvez remplir le “Titre de la version”). Assurez-vous que la version est identique à s.version dans <myrepo>.podspec
  6. Écrivez pod trunk register name@example.org 'Your Name' --description='macbook pro'
  7. Après votre nouvelle version, ouvrez le terminal et écrivez pod trunk push <myrepo>.podspec. Cela changera les CocoaPods Specs qui vous permettront d’écrire pod '<myrepo>' dans votre podfile
