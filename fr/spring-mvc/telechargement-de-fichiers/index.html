Syntaxe #

  • @RequestPart(String, String, boolean)

Paramètres #

Paramètre Détails
@RequestPart Cette annotation spécifie qu’un paramètre doit être mappé à une partie de demande donnée. Le nom de la partie doit correspondre au nom du paramètre de la méthode, sauf si vous choisissez de le fournir comme argument à @RequestPart. Si le nom de la partie n’est pas exprimable en tant que nom Java (par exemple, 123), vous pouvez utiliser l’attribut value de @RequestPart pour spécifier le nom réel. par exemple. @RequestPart("123") Chaîne _123.

Si vous utilisez une ancienne version de Java (avant 1.7) ou si vous compilez * sans * informations de débogage, alors Java remplacera le nom du paramètre par arg0, arg1, etc., ce qui empêchera Spring d’être capable de les faire correspondre avec les noms de pièces. Si tel est le cas, vous devrez définir le nom de la pièce dans l’annotation @RequestPart, comme documenté dans Paramètres.

Téléchargement d’un seul fichier #

Pour recevoir un fichier téléchargé via un HTTP Post, vous devez procéder comme suit :

@RequestMapping(
    value = "...",
    method = RequestMethod.POST,
    consumes = MediaType.MULTIPART_FORM_DATA_VALUE
)
public Object uploadFile(
    @RequestPart MultipartFile file
) {
    String fileName = file.getOriginalFilename();
    InputStream inputStream = file.getInputStream();
    String contentType = file.getContentType();
    .
    .
    .
}

Notez que le nom du paramètre @RequestPart doit correspondre au nom de la partie dans la requête.

Au format HTML :

 <form action="/..." enctype="multipart/form-data" method="post">
        <input type="file" name="file">
    </form>

Au format HTML (Spring TagLibs) :

<form action="/..." enctype="multipart/form-data" method="post">
    <form:input type="file" path="file">
</form>

En tant que requête HTTP brute :

POST /... HTTP/1.1
Host: ...
Content-Type: multipart/form-data; boundary=----------287032381131322

------------287032381131322
Content-Disposition: form-data; name="file"; filename="r.gif"
Content-Type: image/gif

GIF87a.............,...........D..;
------------287032381131322--

Cette demande signifierait ce qui suit :

fileName == "r.gif"
contentType == "image/gif"

Au printemps MVC

Besoin d’ajouter le bean mentionné pour accéder à la fonctionnalité en plusieurs parties 

    <!-- max size of file in memory (in bytes) -->
    <property name="maxInMemorySize" value="1048576" /> <!-- 1MB -->

</bean>

Regrouper une partie dans un objet #

Si vous souhaitez convertir le contenu d’une partie en un objet de domaine (par exemple, un User ou Account ou Address), alors le processus est très simple :

Il est possible de télécharger plusieurs parties, chacune avec un nom différent. Pour chaque nom de pièce, vous aurez besoin d’un paramètre annoté avec @RequestPart, dont le nom correspond au nom de la pièce.

Pour recevoir un fichier téléchargé via un HTTP Post, vous devez procéder comme suit :

@RequestMapping(
    value = "...",
    method = RequestMethod.POST,
    consumes = MediaType.MULTIPART_FORM_DATA_VALUE
)
public Object uploadFile(
    @RequestPart Address address,
) {
    .
    .
    .
}

En tant que requête HTTP brute :

POST /... HTTP/1.1
Host: ...
Content-Type: multipart/form-data; boundary=----------287032381131322

------------287032381131322
Content-Disposition: form-data; name="address"; filename="address.json"
Content-Type: application/json

{"houseNumber": "10/A", "streetName": "Dumbldore Road", "town": "Hogsmede"}
------------287032381131322--

Les choses les plus importantes sont :

  • Le nom de la pièce doit correspondre au nom de la variable.
  • Le Content-Type de la pièce doit être celui que Spring serait capable de gérer si vous l’aviez envoyé comme une demande régulière. Autrement dit, si vous pouviez effectuer un POST sur un point de terminaison avec un Content-Type de foo/bar, et que Spring est capable de le transformer en objet, il pourra également marshaler une partie dans un objet.
  • Vous devez pouvoir définir le Content-Type de la partie. Si vous ne pouvez pas, cette approche ne fonctionnera pas - Spring n’essaiera pas de deviner le Content-Type de la pièce.

Téléchargement de plusieurs fichiers #

Pour recevoir plusieurs fichiers téléchargés via une seule publication HTTP, vous devez procéder comme suit :

@RequestMapping(
    value = "...",
    method = RequestMethod.POST,
    consumes = MediaType.MULTIPART_FORM_DATA_VALUE
)
public Object uploadFile(
    @RequestPart MultipartFile[] files
) {
    for (file : files) {
        String fileName = file.getOriginalFilename();
        InputStream inputStream = file.getInputStream();
        String contentType = file.getContentType();
        .
        .
        .
    }
}

Notez que le nom du paramètre @RequestPart doit correspondre au nom de la partie dans la requête.

Au format HTML :

<form action="/..." enctype="multipart/form-data" method="post">
    <input type="file" name="files">
    <input type="file" name="files">
</form>

En tant que requête HTTP brute :

POST /... HTTP/1.1
Host: ...
Content-Type: multipart/form-data; boundary=----------287032381131322

------------287032381131322
Content-Disposition: form-data; name="files"; filename="r.gif"
Content-Type: image/gif

GIF87a.............,...........D..;
------------287032381131322
Content-Disposition: form-data; name="files"; filename="banana.jpeg"
Content-Type: image/jpeg

GIF87a.............,...........D..;
------------287032381131322--

Cette demande signifierait ce qui suit :

files[0].getOriginalFilename() == "r.gif"
files[0].getContentType() == "image/gif"
files[1].getOriginalFilename() == "r.jpeg"
files[1].getContentType() == "image/jpeg"

Il est possible de télécharger plusieurs parties, chacune avec un nom différent. Pour chaque nom de pièce, vous aurez besoin d’un paramètre annoté avec @RequestPart, dont le nom correspond au nom de la pièce.

Pour recevoir un fichier téléchargé via un HTTP Post, vous devez procéder comme suit :

@RequestMapping(
    value = "...",
    method = RequestMethod.POST,
    consumes = MediaType.MULTIPART_FORM_DATA_VALUE
)
public Object uploadFile(
    @RequestPart MultipartFile profilePicture,
    @RequestPart MultipartFile companyLogo,
) {
    .
    .
    .
}

Au format HTML :

<form action="/..." enctype="multipart/form-data" method="post">
    <input type="file" name="profilePicture">
    <input type="file" name="companyLogo">
</form>
