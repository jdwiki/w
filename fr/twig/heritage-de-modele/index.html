Utiliser un modèle de base #

base.twig.html #

<!DOCTYPE html>
<html>
    <head>
        <title>{{ title | default('Hello World') }}</title>
        <link rel="stylesheet" type="text/css" href="theme.css">
        {% block css %}
        {% endblock %}
    </head>
    <body>
    {% block body %}
        <nav>
        {% block navigation %}
            <a href="#">Link</a>
            <a href="#">Link</a>
            <a href="#">Link</a>
        {% endblock navigation %}
        </nav>
        <section id="container">
            <section id="content">
            {% block content %}
            <p>
            Lorem ipsum dolor sit amet.
            </p>
        </section>
        {% endblock content %}
    </section>
    {% endblock body %}
    </body>
</html>

page.brindille.html

{% extends base.twig.html %}
{% block navigation %}
<a href="page2.html">Page 2</a>
<a href="page3.html">Page 3</a>
<a href="page4.html">Page 4</a>
{% endblock %}
{% block content %}
This is my first page
{% endblock content %}

Modifier le contenu d’un modèle inclus #

article.brindille.html #

<article>
    <h1>{{ article.title }}</h1>
    {% block content %}
    <p>{{ article.content }}</p>
    {% endblock %}
</article>

articles.brindille.html #

{# use default template for article #}
{% for article in articles %}
    {% include "article.twig.html" %}
{% endfor %}

{# use custom template for article #}
{% for article in articles %}
    {% embed "article.twig.html" %}
        {% block content %}
            <img src="{{ article.image }}" alt="{{ article.title }}" />
            {{ parent() }}
        {% endblock %}
    {% endembed %}
{% endfor %}

Héritage des variables #

parent.html.twig #

<!DOCTYPE html>
<html>
    <head>
        <title>{{ title_variable | default('Normal Page') }}</title>
    </head>
    <body>
         <h1>This is the {{ title_variable }} Page</h1>    
    </body>
</html>

enfant.html.brindille #

{% extends "parent.html.twig" %}
{% set title_variable = "Child" %}

Comparaison de l’inclusion, de l’extension, de l’utilisation, de la macro, de l’intégration #

Il existe différents types d’héritage et de réutilisation de code dans Twig :

Comprendre #

L’objectif principal est la réutilisation du code. Pensez à utiliser header.html.twig & footer.html.twig dans base.html.twig comme exemple.

header.html.twig

<nav>
   <div>Homepage</div>
   <div>About</div>
</nav>

base.html.twig

{% include 'header.html.twig' %}
<main>{% block main %}{% endblock %}</main>

Étend #

L’objectif principal est l’héritage vertical. Envisagez d’étendre base.html.twig à l’intérieur de homepage.html.twig et about.html.twig par exemple.

base.html.twig

{% include 'header.html.twig' %}
<main>{% block main %}{% endblock %}</main>

page d’accueil.html.twig

{% extends 'base.html.twig' %}

{% block main %}
<p>You are at the homepage</p>
{% endblock %}

about.html.twig

{% extends 'base.html.twig' %}

{% block main %}
<p>You are at the about page</p>
{% endblock %}

Utilisation #

L’objectif principal est la réutilisation horizontale. Pensez à utiliser sidebar.product.html.twig dans single.product.html.twig (étend product.layout.html.twig) et single.service.html.twig (étend ‘service.layout. html.page’). (c’est comme les macros, mais pour les blocs)

sidebar.html.twig

<aside>{% block sidebar %}{% endblock %}</aside>

single.product.html.twig

{% extends 'product.layout.html.twig' %}

{% use 'sidebar.html.twig' %}
{% block main %}
<p>You are at the product page for product number 123</p>
{% endblock %}

single.service.html.brindille

{% extends 'service.layout.html.twig' %}

{% use 'sidebar.html.twig' %}
{% block main %}
<p>You are at the service page for service number 456</p>
{% endblock %}

#Macro L’objectif principal est d’avoir un balisage réutilisable dans de nombreux modèles avec des variables. Considérez une fonction qui obtient des variables et génère un balisage.

form.html.twig

{% entrée macro (nom, valeur, type) %} <input type="{{ type|default(’text’) }}" name="{{ name }}" value="{{ value|e }}" }}" /> {% endmacro %} #

profil.service.html.twig

{% import "forms.html.twig" as forms %}

<div>{{ forms.input('username') }}</div>

Intégrer #

L’objectif principal est le ** remplacement de bloc **. Il a des fonctionnalités à la fois “Utiliser” et “Inclure” ensemble. Envisagez d’intégrer pagination.html.twig dans product.table.html.twig & service.table.html.twig.

pagination.html.twig

<div>
    <div>{% block first %}{% endblock %}</div>
    {% for i in (min + 1)..(max - 1) %}
        <div>{{ i }}</div>
    {% endfor %}
    <div>{% block last %}{% endblock %}</div>
</div>

product.table.html.twig

{% set min, max = 1, products.itemPerPage %}

{% embed 'pagination.html.twig' %}
    {% block first %}First Product Page{% endblock %}
    {% block last %}Last Product Page{% endblock %}
{% endembed %}

service.table.html.twig

{% set min, max = 1, services.itemPerPage %}

{% embed 'pagination.html.twig' %}
    {% block first %}First Service Page{% endblock %}
    {% block last %}Last Service Page{% endblock %}
{% endembed %}

Veuillez noter que le fichier intégré (pagination.html.twig ici) a accès au contexte actuel (variables min, max ici). Vous pouvez également transmettre des variables supplémentaires au fichier intégré :

pagination.html.twig

<p>{{ count }} items</p>
<div>
    <div>{% block first %}{% endblock %}</div>
    {% for i in (min + 1)..(max - 1) %}
        <div>{{ i }}</div>
    {% endfor %}
    <div>{% block last %}{% endblock %}</div>
</div>

product.table.html.twig

{% set min, max = 1, products|length %}

{% embed 'pagination.html.twig' with {'count': products|length } %}
    {% block first %}First Product Page{% endblock %}
    {% block last %}Last Product Page{% endblock %}
{% endembed %}
