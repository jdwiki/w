Explication des constructions syntaxiques de base des modèles

Accéder aux variables #

Dans les modèles Twig, les variables sont accessibles en utilisant la notation des accolades doubles {{ variableName }}.

Exemple de base de l’accueil d’un utilisateur

<!DOCTYPE html>
<html>
  <body>
    <span>Hello {{ name }}</span>
  </body>
</html>

Accéder aux éléments du tableau #

Twig en tant que paramètre peut recevoir un tableau. Pour accéder à un élément spécifique du tableau, vous pouvez utiliser la notation de support d’accès au tableau php habituelle {{ array[key] }}.

Exemple précédent modifié pour utiliser array comme paramètre

<!DOCTYPE html>
<html>
  <body>
    <span>Hello {{ user['name'] }}</span>
  </body>
</html>

Accéder aux propriétés de l’objet #

Les objets peuvent également être passés en tant que paramètre au modèle. La notation ‘Point’ (.) est utilisée pour accéder aux propriétés d’objet spécifiques {{ object.propertyName }}.

Même exemple avec objet en paramètre

<!DOCTYPE html>
<html>
  <body>
    <span>Hello {{ user.name }}</span>
  </body>
</html>

Filtres #

Les variables peuvent être modifiées à l’aide de filtres. Pour appliquer le filtre à la variable, suivez le nom de la variable avec le tube | et le nom du filtre :

{{ variable|filterName }}

Par exemple, pour afficher la valeur de la variable en majuscule, utilisez construct.

{{ variable|upper }}

Les filtres peuvent être paramétrés. Les paramètres de filtre sont passés entre parenthèses sous la forme d’une liste séparée par des virgules (,) :

{{ variable|filterName(param1, param2, ...) }}

Pour arrondir le nombre à une précision donnée, nous pouvons utiliser le filtre “round”, il accepte jusqu’à 2 paramètres. Le premier spécifie la précision (par défaut : 0), le second la méthode d’arrondi (par défaut : common).

Pour arrondir le nombre à 1 décimale en utilisant la méthode courante, vous pouvez utiliser {{ number|round(1, 'common') }} ou {{ number|round(1) }} car common est la méthode par défaut.

Les filtres peuvent également être utilisés sur les objets intégrés et les variables de tableau :

{{ array['key'] | upper }} {{ object.text | upper }}

Les filtres peuvent également être concaténés :

{% set array = "3,1,2"|split(',') %}

{{ array | sort | first }}

La liste des filtres de base est disponible [ici][1]

[1] : http://twig.sensiolabs.org/doc/filters/index.html

Blocs conditionnels #

Certaines parties du modèle peuvent être affichées de manière conditionnelle. L’instruction If est utilisée à cette fin. C’est similaire à l’instruction if dans les langages de programmation. Le contenu du bloc est exécuté/affiché si une expression est évaluée à “true”.

{% if enabled == false %}
    Disabled
{% endif %}

Disabled sera affiché uniquement lorsque enabled sera égal à false.

Plusieurs branches peuvent être créées en utilisant elseif et else.

{% if temperature < 10 %}
    It's cold
{% elseif temperature < 18 %}
    It's chilly
{% elseif temperature < 24 %}
    It's warm
{% elseif temperature < 32 %}
    It's hot
{% else %}
    It's very hot
{% endif %}

Pour la boucle #

Les boucles For peuvent être très utiles dans TWIG, permettant la création de pages Web dynamiques.

Supposons que nous créons un simple tableau de nombres :

{% set array = "3,1,2" %}

Nous pouvons ensuite parcourir le tableau et imprimer ce que nous voulons. Toutes les données dans le bloc de tableau seront sorties en fonction de la quantité de données dans le tableau. Cet exemple imprimerait trois éléments h1 avec les données du tableau concaténées.

{% for current in array %}
    <h1>This is number {{ current }} in the array </h1>
{% endear %}

Notez que {{ current }} a été utilisé pour accéder aux données et non une version de array

Voir ce travail : https://twigfiddle.com/mxwkea/2

Un autre exemple pourrait être l’utilisation d’objets. Par exemple, disons que nous avons un objet Entity avec le champ ’name’ avec ses getters et setters pertinents. Si un certain nombre de ces entités sont également stockées dans le tableau, elles sont accessibles comme n’importe quel autre objet dans TWIG :

{% for currentObject in ArrayOfObjects %}
    {{ currentObject.name }}
{% endfor %}

Voir ce travail avec les données JSON : https://twigfiddle.com/mxwkea

Opérateur ternaire (abréviation If-Then-Else) et opérateur de coalescence nulle #

L’opérateur ternaire (?:) #

La prise en charge de l’opérateur ternaire étendu a été ajoutée dans Twig 1.12.0.

{{ foo ? 'yes' : 'no' }}

Évalue :

if foo echo yes else echo no

{{ foo ?: 'no' }}

ou

{{ foo ? foo : 'no' }}

Évalue :

si ‘foo’ l’écho, sinon ’non’

{{ foo ? 'yes' }}

ou

{{ foo ? 'yes' : '' }}

Évalue :

si foo echo yes sinon echo rien

L’opérateur de fusion nulle (??:) #

{{ foo ?? 'no' }}

Évalue :

Renvoie la valeur de foo s’il est défini et non nul, no sinon

Gestion des espaces blancs #

Pour supprimer les espaces (espaces, tabulations, retours à la ligne…) entre les balises HTML, utilisez la balise spaceless :

{% spaceless %}

    <div>
        <span>foo bar </span>
    </div>
{% endspaceless %}
{# produces output <div><strong>foo bar </strong></div> #}

Si vous avez besoin de supprimer les espaces au niveau de chaque balise, utilisez le modificateur de contrôle des espaces blancs, c’est-à-dire le trait d’union (-). En l’utilisant, vous pouvez couper les espaces blancs de début et/ou de fin :

{% set value = 'foo bar' %}

<span>  {{- value }}  </span> 
{# produces '<span>foo bar  </span>' #}

<span>  {{ value -}}  </span> 
{# produces '<span>  foo bar</span>' #}

<span>  {{- value -}}  </span> 
{# produces '<span>foo bar</span>' #}

<span {%- if true  %} class="foo"{% endif %}>
{# produces '<span class="foo">' #}

<span {%- if false %} class="foo"{% endif %}>
{# produces '<span>' #}
