Qu’est-ce que l’introspection ?

L’introspection consiste à regarder à l’intérieur pour connaître l’intérieur. C’est une simple définition de l’introspection.

En programmation et Ruby en général… l’introspection est la capacité de regarder un objet, une classe… au moment de l’exécution pour connaître celui-ci.

s = "Hello"  # s is a string

Oui, les s ont-ils un nom de méthode supérieur. Vous voulez obtenir la version majuscule de s ? Essayons:

irb(main):044:0> s.respond_to?(:upper)
=> false

On dirait que non, la bonne méthode est upcase permet de vérifier :

irb(main):047:0*
irb(main):048:0* s.respond_to?(:upcase)
=> true

Introspection de classe #

Voici la définition de la classe

class A
   def a; end
end

module B
   def b; end
end

class C < A
   include B
   def c; end
end

Quelles sont les méthodes d’instance de C ?

C.instance_methods # [:c, :b, :a, :to_json, :instance_of?...]

Quelles sont les méthodes d’instance qui déclarent uniquement sur C ?

C.instance_methods(false) # [:c]

Quels sont les ancêtres de la classe C ?

C.ancestors # [C, B, A, Objet,...]

Superclasse de ‘C’ ?

C.superclasse # A
