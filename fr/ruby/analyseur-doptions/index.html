OptionParser peut être utilisé pour analyser les options de ligne de commande de ARGV.

Options de ligne de commande obligatoires et facultatives #

Il est relativement facile d’analyser la ligne de commande à la main si vous ne cherchez rien de trop complexe :

# Naive error checking
abort('Usage: ' + $0 + ' site id ...') unless ARGV.length >= 2

# First item (site) is mandatory
site = ARGV.shift

ARGV.each do | id |
  # Do something interesting with each of the ids
end

Mais lorsque vos options commenceront à se compliquer, vous devrez probablement utiliser un analyseur d’options tel que, eh bien, OptionParser :

require 'optparse'

# The actual options will be stored in this hash
options = {}

# Set up the options you are looking for
optparse = OptionParser.new do |opts|
  opts.banner = "Usage: #{$0} -s NAME id ..."

  opts.on("-s", "--site NAME", "Site name") do |s|
    options[:site] = s
  end

  opts.on( '-h', '--help', 'Display this screen' ) do
    puts opts
    exit
  end
end

# The parse! method also removes any options it finds from ARGV.
optparse.parse!

Il existe également une analyse non destructive, mais elle est beaucoup moins utile si vous prévoyez d’utiliser le reste de ce qui se trouve dans ARGV.

La classe OptionParser n’a aucun moyen d’appliquer des arguments obligatoires (comme --site dans ce cas). Cependant, vous pouvez effectuer votre propre vérification après avoir exécuté parse! :

# Slightly more sophisticated error checking
if options[:site].nil? or ARGV.length == 0
  abort(optparse.help)
end

Pour un gestionnaire d’options obligatoires plus générique, voir cette réponse. Au cas où ce ne serait pas clair, toutes les options sont facultatives, sauf si vous faites tout votre possible pour les rendre obligatoires.

Les valeurs par défaut #

Avec OptionsParser, il est vraiment facile de configurer des valeurs par défaut. Pré-remplissez simplement le hachage dans lequel vous stockez les options :

options = {
  :directory => ENV['HOME']
}

Lorsque vous définissez l’analyseur, il écrasera la valeur par défaut si un utilisateur fournit une valeur :

OptionParser.new do |opts|
  opts.on("-d", "--directory HOME", "Directory to use") do |d|
    options[:directory] = d
  end
end

Descriptions longues #

Parfois, votre description peut être assez longue. Par exemple, irb -h répertorie les arguments qui se lisent comme suit :

  --context-mode n  Set n[0-3] to method to create Binding Object,
                    when new workspace was created

Il n’est pas immédiatement clair comment soutenir cela. La plupart des solutions nécessitent un ajustement pour aligner l’indentation de la deuxième ligne et des lignes suivantes sur la première. Heureusement, la méthode “on” prend en charge plusieurs lignes de description en les ajoutant en tant qu’arguments distincts :

  opts.on("--context-mode n",
          "Set n[0-3] to method to create Binding Object,",
          "when new workspace was created") do |n|
    optons[:context_mode] = n
  end

Vous pouvez ajouter autant de lignes de description que vous le souhaitez pour expliquer complètement l’option.
