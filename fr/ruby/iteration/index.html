Chaque #

Ruby a de nombreux types d’énumérateurs, mais le premier type d’énumérateur et le plus simple pour commencer est each. Nous afficherons pair ou odd pour chaque nombre entre 1 et 10 pour montrer comment each fonctionne.

Fondamentalement, il existe deux façons de passer les soi-disant “blocs”. Un block est un morceau de code transmis qui sera exécuté par la méthode appelée. La méthode each prend un block qu’elle appelle pour chaque élément de la collection d’objets sur laquelle elle a été appelée.

Il existe deux manières de passer un bloc à une méthode :

Méthode 1 : en ligne #

(1..10).each { |i| puts i.even? ? 'even' : 'odd' }

C’est une manière très compressée et ruby de résoudre ce problème. Décomposons cela morceau par morceau.

  1. (1..10) est une plage de 1 à 10 inclus. Si nous voulions qu’il soit exclusif de ‘1’ à ‘10’, nous écririons ‘(1…10)’.
  2. .each est un énumérateur qui énumère sur chaque élément de l’objet sur lequel il agit. Dans ce cas, il agit sur “chaque” nombre de la plage.
  3. { |i| met i.even? ? 'even' : 'odd' } est le bloc de l’instruction each, qui elle-même peut être décomposée davantage.
    1. |i| this means that each element in the range is represented within the block by the identifier i.
    2. puts is an output method in Ruby that has an automatic line break after each time it prints. (We can use print if we don’t want the automatic line break)
    3. i.even? checks if i is even. We could have also used i % 2 == 0; however, it is preferable to use built in methods.
    4. ? "even" : "odd" this is ruby’s ternary operator. The way a ternary operator is constructed is expression ? a : b. This is short for
    if expression
  a
else
  b
end

Pour un code de plus d’une ligne, le “bloc” doit être passé en tant que “bloc multiligne”.

Méthode 2 : multiligne #

(1..10).each do |i|
  if i.even?
    puts 'even'
  else
    puts 'odd'
  end
end

Dans un bloc multiligne le do remplace la parenthèse ouvrante et end remplace la parenthèse fermante du style inline.

Ruby prend également en charge reverse_each. Il itérera le tableau en arrière.

@arr = [1,2,3,4]
puts @arr.inspect # output is [1,2,3,4]

print "Reversed array elements["
@arr.reverse_each do |val|
        print " #{val} " # output is 4 3 2 1
end
print "]\n"

Itérer sur des objets complexes #

Tableaux

Vous pouvez itérer sur des tableaux imbriqués :

[[1, 2], [3, 4]].each { |(a, b)| p "a: #{ a }", "b: #{ b }" }

La syntaxe suivante est également autorisée :

[[1, 2], [3, 4]].each { |a, b| "a: #{ a }", "b: #{ b }" }

Produira:

"a: 1"
"b: 2"
"a: 3"
"b: 4"

Hachages

Vous pouvez parcourir les paires clé-valeur :

{a: 1, b: 2, c: 3}.each { |pair| p "pair: #{ pair }" }

Produira:

"pair: [:a, 1]"
"pair: [:b, 2]"
"pair: [:c, 3]"

Vous pouvez itérer simultanément sur les clés et les valeurs :

{a: 1, b: 2, c: 3}.each { |(k, v)| p "k: #{ k }", "v: #{ k }" }

Produira:

"k: a"
"v: a"
"k: b"
"v: b"
"k: c"
"v: c"

Pour l’itérateur #

Cela itère de 4 à 13 (inclus).

for i in 4..13
    puts "this is #{i}.th number"
end

Nous pouvons également itérer sur des tableaux en utilisant pour

names = ['Siva', 'Charan', 'Naresh', 'Manish']

for name in names
    puts name
end

Itération avec index #

Parfois, vous souhaitez connaître la position (** index **) de l’élément actuel lors de l’itération sur un énumérateur. Dans ce but, Ruby fournit la méthode with_index. Elle peut être appliquée à tous les enquêteurs. Fondamentalement, en ajoutant with_index à une énumération, vous pouvez énumérer cette énumération. L’index est passé à un bloc en tant que deuxième argument.

[2,3,4].map.with_index { |e, i| puts "Element of array number #{i} => #{e}" }
#Element of array number 0 => 2
#Element of array number 1 => 3
#Element of array number 2 => 4
#=> [nil, nil, nil]

with_index a un argument optionnel - le premier index qui est 0 par défaut :

[2,3,4].map.with_index(1) { |e, i| puts "Element of array number #{i} => #{e}" }
#Element of array number 1 => 2
#Element of array number 2 => 3
#Element of array number 3 => 4
#=> [nil, nil, nil]

Il existe une méthode spécifique each_with_index. La seule différence entre lui et each.with_index est que vous ne pouvez pas lui passer d’argument, donc le premier index est toujours 0.

[2,3,4].each_with_index { |e, i| puts "Element of array number #{i} => #{e}" }
#Element of array number 0 => 2
#Element of array number 1 => 3
#Element of array number 2 => 4
#=> [2, 3, 4]

Carte #

Renvoie l’objet modifié, mais l’objet d’origine reste tel qu’il était. Par exemple:

arr = [1, 2, 3]
arr.map { |i| i + 1 } # => [2, 3, 4]
arr # => [1, 2, 3]

map! modifie l’objet d’origine :

arr = [1, 2, 3]
arr.map! { |i| i + 1 } # => [2, 3, 4]
arr # => [2, 3, 4]

Remarque : vous pouvez également utiliser collect pour faire la même chose.
