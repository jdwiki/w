si, elsif, sinon et fin #

Ruby propose les expressions if et else attendues pour la logique de branchement, terminées par le mot-clé end :

# Simulate flipping a coin
result = [:heads, :tails].sample

if result == :heads
  puts 'The coin-toss came up "heads"'
else
  puts 'The coin-toss came up "tails"'
end

En Ruby, les instructions if sont des expressions qui évaluent une valeur, et le résultat peut être assigné à une variable :

status = if age < 18
           :minor
         else
           :adult
         end

Ruby propose également des opérateurs ternaires de style C ([voir ici pour plus de détails][1]) qui peuvent être exprimés comme suit :

some_statement ? if_true : if_false

Cela signifie que l’exemple ci-dessus utilisant if-else peut également être écrit comme

status = age < 18 ? :minor : :adult

De plus, Ruby propose le mot-clé elsif qui accepte une expression pour activer une logique de branchement supplémentaire :

label = if shirt_size == :s
          'small'
        elsif shirt_size == :m
          'medium'
        elsif shirt_size == :l
          'large'
        else
          'unknown size'
        end

Si aucune des conditions d’une chaîne if/elsif n’est vraie et qu’il n’y a pas de clause else, alors l’expression est évaluée à nil. Cela peut être utile dans l’interpolation de chaîne, puisque nil.to_s est la chaîne vide :

"user#{'s' if @users.size != 1}"

[1] : https://www.wikiod.com/fr/ruby/flux-de-controle#Opérateur ternaire

Déclaration de cas #

Ruby utilise le mot-clé case pour les instructions switch.

Selon les Ruby Docs :

Les déclarations de cas consistent en une condition facultative, qui se trouve dans position d’un argument de case et zéro ou plusieurs clauses when. La première clause « quand » pour correspondre à la condition (ou pour évaluer Vérité booléenne, si la condition est nulle) “gagne”, et sa strophe de code est exécuté. La valeur de l’instruction case est la valeur de la clause quand réussie, ou nil s’il n’y a pas une telle clause.

Une instruction case peut se terminer par une clause else. Chaque “quand” un instruction peut avoir plusieurs valeurs candidates, séparées par des virgules.

Exemple:

case x
when 1,2,3
  puts "1, 2, or 3"
when 10
  puts "10"
else
  puts "Some other number"
end

Version plus courte :

case x
when 1,2,3 then puts "1, 2, or 3"
when 10 then puts "10"
else puts "Some other number"
end

La valeur de la clause case est mise en correspondance avec chaque clause when en utilisant la méthode === (et non ==). Par conséquent, il peut être utilisé avec une variété de différents types d’objets.

Une instruction case peut être utilisée avec Ranges :

case 17
when 13..19
  puts "teenager"
end

Une instruction case peut être utilisée avec une Regexp :

case "google"
when /oo/
  puts "word contains oo"
end

Une instruction case peut être utilisée avec un Proc ou un lambda :

case 44
when -> (n) { n.even? or n < 0 }
  puts "even or less than zero"
end

Une instruction case peut être utilisée avec Classes :

case x
when Integer
  puts "It's an integer"
when String
  puts "It's a string"
end

En implémentant la méthode ===, vous pouvez créer vos propres classes de correspondance :

class Empty
  def self.===(object)
    !object or "" == object
  end
end

case ""
when Empty
  puts "name was empty"
else
  puts "name is not empty"
end

Une instruction case peut être utilisée sans valeur à comparer :

case
when ENV['A'] == 'Y'
  puts 'A'
when ENV['B'] == 'Y'
  puts 'B'
else
  puts 'Neither A nor B'
end

Une instruction case a une valeur, vous pouvez donc l’utiliser comme argument de méthode ou dans une affectation :

description = case 16
              when 13..19 then "teenager"
              else ""
              end

Valeurs du vrai et du faux #

Dans Ruby, il y a exactement deux valeurs qui sont considérées comme “fausses” et renverront fausses lorsqu’elles seront testées comme condition pour une expression if. Elles sont:

  • “néant”
  • booléen “faux”

Toutes les autres valeurs sont considérées comme “véridiques”, y compris :

  • 0 - zéro numérique (entier ou autre)
  • "" - Chaînes vides
  • "\n" - Chaînes contenant uniquement des espaces
  • [] - Tableaux vides
  • {} - Hachages vides

Prenons par exemple le code suivant :

def check_truthy(var_name, var)
  is_truthy = var ? "truthy" : "falsy"
  puts "#{var_name} is #{is_truthy}"
end

check_truthy("false", false)
check_truthy("nil", nil)
check_truthy("0", 0)
check_truthy("empty string", "")
check_truthy("\\n", "\n")
check_truthy("empty array", [])
check_truthy("empty hash", {})

Sortira :

false is falsy
nil is falsy
0 is truthy
empty string is truthy
\n is truthy
empty array is truthy
empty hash is truthy

Inline si/sauf si #

Un modèle courant consiste à utiliser un if ou unless en ligne ou à la fin :

puts "x is less than 5" if x < 5

C’est ce qu’on appelle un modificateur conditionnel, et c’est un moyen pratique d’ajouter un code de garde simple et des retours anticipés :

def save_to_file(data, filename)
  raise "no filename given" if filename.empty?
  return false unless data.valid?

  File.write(filename, data)
end

Il n’est pas possible d’ajouter une clause “else” à ces modificateurs. De plus, il n’est généralement pas recommandé d’utiliser des modificateurs conditionnels à l’intérieur de la logique principale. Pour un code complexe, il convient d’utiliser à la place les if, elsif, else normaux.

tant que, jusqu’à #

Une boucle “while” exécute le bloc tant que la condition donnée est remplie :

i = 0
while i < 5
  puts "Iteration ##{i}"
  i +=1
end

Une boucle “jusqu’à” exécute le bloc alors que la condition est fausse :

i = 0
until i == 5
  puts "Iteration ##{i}"
  i +=1
end

Or-Equals/Opérateur d’affectation conditionnelle (||=) #

Ruby a un opérateur or-equals qui permet d’attribuer une valeur à une variable si et seulement si cette variable est évaluée à ’nil’ ou ‘false’.

 ||= # this is the operator that achieves this.

cet opérateur avec les doubles tuyaux représentant le ou et le signe égal représentant l’affectation d’une valeur. Vous pensez peut-être que cela représente quelque chose comme ceci :

 x = x || y

cet exemple ci-dessus n’est pas correct. L’opérateur or-equal représente en fait ceci :

 x || x = y

Si x est évalué à nil ou false alors x reçoit la valeur de y, et reste inchangé sinon.

Voici un cas d’utilisation pratique de l’opérateur or-equals. Imaginez que vous ayez une partie de votre code qui est censée envoyer un e-mail à un utilisateur. Que faites-vous si pour une raison quelconque, il n’y a pas d’e-mail pour cet utilisateur. Vous pourriez écrire quelque chose comme ceci :

 if user_email.nil?
    user_email = "error@yourapp.com"
 end

En utilisant l’opérateur or-equals, nous pouvons couper tout ce morceau de code, offrant un contrôle et des fonctionnalités propres et clairs.

 user_email ||= "error@yourapp.com"

Dans les cas où “false” est une valeur valide, il faut veiller à ne pas la remplacer accidentellement :

has_been_run = false
has_been_run ||= true
#=> true

has_been_run = false
has_been_run = true if has_been_run.nil?
#=> false

Opérateur Flip-Flop #

L’opérateur de bascule “..” est utilisé entre deux conditions dans une instruction conditionnelle :

(1..5).select do |e|
  e if (e == 2) .. (e == 4)
end
# => [2, 3, 4]

La condition est évaluée à false jusqu’à que la première partie devienne true. Ensuite, il est évalué à “true” jusqu’à ce que la deuxième partie devienne “true”. Après cela, il repasse à “faux”.

Cet exemple illustre ce qui est sélectionné :

[1, 2, 2, 3, 4, 4, 5].select do |e|
  e if (e == 2) .. (e == 4)
end
# => [2, 2, 3, 4]

L’opérateur de bascule ne fonctionne qu’à l’intérieur de ifs (y compris “à moins que”) et de l’opérateur ternaire. Sinon, il est considéré comme l’opérateur de plage.

(1..5).select do |e|
  (e == 2) .. (e == 4)
end
# => ArgumentError: bad value for range

Il peut passer de “faux” à “vrai” et inversement plusieurs fois :

((1..5).to_a * 2).select do |e|
  e if (e == 2) .. (e == 4)
end
# => [2, 3, 4, 2, 3, 4]

sauf si #

Une déclaration courante est si !(une condition). Ruby propose l’alternative de l’instruction “sauf si”.

La structure est exactement la même qu’une instruction if, sauf que la condition est négative. De plus, l’instruction unless ne prend pas en charge elsif, mais elle prend en charge else :

# Prints not inclusive
unless 'hellow'.include?('all')
  puts 'not inclusive'
end

lancer, rattraper #

Contrairement à de nombreux autres langages de programmation, les mots clés throw et catch ne sont pas liés à la gestion des exceptions dans Ruby.

En Ruby, throw et catch agissent un peu comme des étiquettes dans d’autres langages. Ils sont utilisés pour modifier le flux de contrôle, mais ne sont pas liés à un concept “d’erreur” comme le sont les exceptions.

catch(:out) do
  catch(:nested) do
    puts "nested"
  end

  puts "before"
  throw :out
  puts "will not be executed"
end
puts "after"
# prints "nested", "before", "after"

Opérateur ternaire #

Ruby a un opérateur ternaire ( ?:), qui renvoie l’une des deux valeurs selon qu’une condition est évaluée comme véridique :

conditional ? value_if_truthy : value_if_falsy

value = true
value ? "true" : "false"
#=> "true"

value = false
value ? "true" : "false"
#=> "false"

c’est la même chose que d’écrire si a alors b sinon c fin, bien que le ternaire soit préféré

Exemples:

puts (if 1 then 2 else 3 end) # => 2

puts 1 ? 2 : 3                # => 2

x = if 1 then 2 else 3 end
puts x                        # => 2

Contrôle de boucle avec break, next et redo #

Le flux d’exécution d’un bloc Ruby peut être contrôlé avec les instructions break, next et redo.

pause #

L’instruction break quittera le bloc immédiatement. Toutes les instructions restantes dans le bloc seront ignorées et l’itération se terminera :

actions = %w(run jump swim exit macarena)
index = 0

while index < actions.length
  action = actions[index]

  break if action == "exit"

  index += 1
  puts "Currently doing this action: #{action}"
end

# Currently doing this action: run
# Currently doing this action: jump
# Currently doing this action: swim

suivant #

L’instruction next reviendra immédiatement en haut du bloc et passera à l’itération suivante. Toutes les instructions restantes dans le bloc seront ignorées :

actions = %w(run jump swim rest macarena)
index = 0

while index < actions.length
  action = actions[index]
  index += 1

  next if action == "rest"

  puts "Currently doing this action: #{action}"
end

# Currently doing this action: run
# Currently doing this action: jump
# Currently doing this action: swim
# Currently doing this action: macarena

refaire #

L’instruction redo reviendra immédiatement en haut du bloc et réessayera la même itération. Toutes les instructions restantes dans le bloc seront ignorées :

actions = %w(run jump swim sleep macarena)
index = 0
repeat_count = 0

while index < actions.length
  action = actions[index]
  puts "Currently doing this action: #{action}"

  if action == "sleep"
    repeat_count += 1
    redo if repeat_count < 3
  end

  index += 1
end

# Currently doing this action: run
# Currently doing this action: jump
# Currently doing this action: swim
# Currently doing this action: sleep
# Currently doing this action: sleep
# Currently doing this action: sleep
# Currently doing this action: macarena

## Itération “énumérable” En plus des boucles, ces instructions fonctionnent avec les méthodes d’itération Enumerable, telles que each et map :

[1, 2, 3].each do |item|
  next if item.even?
  puts "Item: #{item}"
end

# Item: 1
# Item: 3

Bloquer les valeurs de résultat #

Dans les instructions break et next, une valeur peut être fournie et sera utilisée comme valeur de résultat de bloc :

even_value = for value in [1, 2, 3]
  break value if value.even?
end

puts "The first even value is: #{even_value}"

# The first even value is: 2

return vs. next : retour non local dans un bloc #

Considérez cet extrait cassé :

def foo
  bar = [1, 2, 3, 4].map do |x|
    return 0 if x.even?
    x
  end
  puts 'baz'
  bar
end
foo # => 0

On pourrait s’attendre à ce que return produise une valeur pour le tableau de résultats de blocs de map. Ainsi, la valeur de retour de foo serait [1, 0, 3, 0]. Au lieu de cela, return renvoie une valeur de la méthode foo. Notez que baz n’est pas affiché, ce qui signifie que l’exécution n’a jamais atteint cette ligne.

next avec une valeur fait l’affaire. Il agit comme un “retour” au niveau du bloc.

def foo
  bar = [1, 2, 3, 4].map do |x|
    next 0 if x.even?
    x
  end
  puts 'baz'
  bar
end
foo # baz
    # => [1, 0, 3, 0]

En l’absence de return, la valeur renvoyée par le bloc est la valeur de sa dernière expression.

Flux de contrôle avec des instructions logiques #

Bien que cela puisse sembler contre-intuitif, vous pouvez utiliser des opérateurs logiques pour déterminer si une instruction est exécutée ou non. Par exemple:

File.exist?(filename) or STDERR.puts "#{filename} does not exist!"

Cela vérifiera si le fichier existe et n’imprimera le message d’erreur que si ce n’est pas le cas. L’instruction or est paresseuse, ce qui signifie qu’elle s’arrêtera de s’exécuter une fois qu’elle sera sûre que sa valeur est vraie ou fausse. Dès que le premier terme est trouvé vrai, il n’est pas nécessaire de vérifier la valeur de l’autre terme. Mais si le premier terme est faux, il doit vérifier le second terme.

Une utilisation courante consiste à définir une valeur par défaut :

glass = glass or 'full' # Optimist!

Cela définit la valeur de glass sur ‘full’ si elle n’est pas déjà définie. Plus concis, vous pouvez utiliser la version symbolique de or :

glass ||= 'empty' # Pessimist.

Il est également possible d’exécuter la deuxième instruction uniquement si la première est fausse :

File.exist?(filename) and puts "#{filename} found!"

Encore une fois, et est paresseux, il n’exécutera donc la deuxième instruction que si nécessaire pour arriver à une valeur.

L’opérateur “ou” a une priorité inférieure à “et”. De même, || a une priorité inférieure à &&. Les formes de symboles ont une priorité plus élevée que les formes de mots. C’est pratique à savoir quand on veut mélanger cette technique avec le devoir :

a = 1 and b = 2
#=> a==1
#=> b==2

a = 1 && b = 2; puts a, b
#=> a==2
#=> b==2

Notez que le Ruby Style Guide recommande :

Les mots clés and et or sont interdits. La lisibilité ajoutée minimale ne vaut tout simplement pas la forte probabilité d’introduire des bogues subtils. Pour les expressions booléennes, utilisez toujours && et || à la place. Pour le contrôle de flux, utilisez if et unless ; && et || sont également acceptables mais moins clairs.

début, fin #

Le bloc begin est une structure de contrôle qui regroupe plusieurs instructions.

begin
  a = 7
  b = 6
  a * b
end

Un bloc begin renverra la valeur de la dernière instruction du bloc. L’exemple suivant renverra ‘3’.

begin
  1
  2
  3
end

Le bloc begin est utile pour l’affectation conditionnelle à l’aide de l’opérateur ||= où plusieurs instructions peuvent être nécessaires pour renvoyer un résultat.

circumference ||=
  begin
    radius = 7
    tau = Math::PI * 2
    tau * radius
  end

Il peut également être combiné avec d’autres structures de blocs telles que “rescue”, “ensure”, “while”, “if”, “unless”, etc. pour fournir un meilleur contrôle du déroulement du programme.

Les blocs Begin ne sont pas des blocs de code, comme { ... } ou do ... end ; ils ne peuvent pas être passés à des fonctions.
