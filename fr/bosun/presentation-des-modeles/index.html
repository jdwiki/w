Syntaxe #

  • #Voir https://golang.org/pkg/text/template/ pour Go Template Action et Function syntaxe
  • expression = état d’alerte {{.Last.Status}} et une variable {{.Eval .Alert.Vars.q | printf “%.2f”}}
  • expression = `Utiliser des backticks pour s’étendre
  • plusieurs lignes avec des sauts de ligne
  • dans le fichier de configuration Bosun`
  • modèle nom {
  • sujet = expression
  • corps = expression
  • }

Les modèles Bosun sont basés sur le package [Go html/template][1] et peuvent être partagés entre plusieurs alertes, mais un seul modèle est utilisé pour afficher toutes les [Notifications Bosun][2] pour cette alerte. Les alertes référencent le modèle à utiliser via la directive template et spécifient les notifications à utiliser via les directives warnNotification et critNotification (peut avoir plusieurs notifications warn/crit définies pour chaque alerte).

Les modèles sont affichés lorsqu’une instance d’alerte est déclenchée et peuvent :

  • Utiliser les variables définies dans l’alerte pour afficher du texte ou des graphiques
  • Utilisez [Go Template Actions and Functions][3] comme if, range et, not, index et printf
  • Accédez aux métadonnées Bosun pour afficher des détails supplémentaires sur un système
  • Accéder à d’autres [Variables et fonctions du modèle Bosun] [4]
  • Extraire des informations d’autres systèmes via [HTTPGet et HTTPGetJSON][5]
  • Utilisez les styles Images, HTML et CSS pour les notifications enrichies (CSS peut être intégré pour une meilleure prise en charge des e-mails)

L’objet du modèle sera affiché sous forme d’en-têtes sur le tableau de bord, comme ligne d’objet des notifications par e-mail et comme contenu par défaut des notifications HTTP POST. Le corps du modèle s’affiche lorsqu’une instance d’alerte est développée et en tant que corps des notifications par e-mail.

[1] : https://golang.org/pkg/html/template/ [2] : https://www.wikiod.com/fr/bosun/presentation-des-notifications [3] : https://golang.org/pkg/text/template/ [4] : https://bosun.org/configuration#template [5] : https://www.wikiod.com/fr/bosun/modeles-httpget-et-httpgetjson

Alerte et modèle de mémoire faible #

Les modèles peuvent être prévisualisés et modifiés à l’aide de l’onglet Éditeur de règles dans Bosun. Utilisez les liens Aller à pour sélectionner l’alerte que vous souhaitez modifier, puis vous pouvez utiliser le bouton modèle à côté de macro pour basculer entre les sections d’alerte et de modèle de la configuration. Si une alerte a plusieurs instances, vous pouvez utiliser host=xxx,name=xxx dans la section Template Group pour spécifier pour quel jeu de balises vous souhaitez voir le modèle rendu.

template os.low.memory {
    subject = {{.Last.Status}}: Low Memory: {{.Eval .Alert.Vars.q | printf "%.0f"}}% Free Memory on {{.Group.host}} ({{.Eval .Alert.Vars.free | bytes }} Free of {{.Eval .Alert.Vars.total | bytes }} Total)

    body = `
    <p><a href="{{.Ack}}">Acknowledge</a> | <a href="{{.Rule}}">View Alert in Bosun's Rule Editor</a></p>
    <p><strong>Alert Key: </strong>{{printf "%s%s" .Alert.Name  .Group }}</p>
    <p><strong>Incident: </strong><a href="{{.Incident}}">#{{.Last.IncidentId}}</a></p>
    <p><strong>Notes: </strong>{{html .Alert.Vars.notes}}</p>

    <strong>Graph</strong>
    <div>{{.Graph .Alert.Vars.graph .Alert.Vars.graph_unit}}</div>
    `
}

notification sample.notification {
    email = alerts@example.com
}

alert os.low.memory {
    template = os.low.memory
    $notes = Alerts when less than 5% free, or less than 500MB (when total > 2GB). In Linux, Buffers and Cache are considered "Free Memory".

    $default_time = "2m"
    $host = wildcard(*)
    $graph = q("avg:300s-avg:os.mem.percent_free{host=$host}", "1d", "")
    $graph_unit = Percent Free Memory (Including Buffers and Cache)
    $q = avg(q("avg:os.mem.percent_free{host=$host}", $default_time, ""))
    $total = last(q("sum:os.mem.total{host=$host}", $default_time, ""))
    $free = last(q("sum:os.mem.free{host=$host}", $default_time, ""))

    #Warn when less than 5% free or total > 2GB and free < 500MB
    warn = $q < 5 || ($total > 2147483648 && $free < 524288000)
    #Crit when less than 0.5% free
    crit = $q <= .5
    critNotification = sample.notification
}

Après avoir testé l’alerte sur la page Éditeur de règles, vous pouvez utiliser l’onglet Résultats pour voir les calculs, Modèle pour voir la notification d’alerte rendue et Chronologie pour voir tous les incidents d’alerte (uniquement lorsque De et *À * les dates sont précisées).

Modèles intégrés et styles CSS #

Vous pouvez intégrer un autre corps de modèle dans votre modèle via {{template "mysharedtemplate" .}} pour réutiliser les composants partagés. Voici un exemple qui crée un modèle d’en-tête qui peut être réutilisé en haut de tous les autres corps de modèle. Il utilise également CSS pour styliser la sortie afin qu’elle soit plus facile à lire. Notez que tous les blocs <style>...</style> seront convertis en CSS inline sur chaque élément afin que les clients de messagerie comme Gmail restituent correctement la sortie.

template header {
    body = `
    <style>
    td, th {
        padding-right: 10px;
    }
    a.rightpad {
        padding-right: 10px;
    }
    </style>
    <p style="font-weight: bold; text-decoration: underline;">
        <a class="rightpad" href="{{.Ack}}">Acknowledge</a>
        <a class="rightpad" href="{{.Rule}}">View Alert in Bosun's Rule Editor</a>
        {{if .Group.host}}
            <a class="rightpad" href="https://opserver/dashboard/node?node={{.Group.host}}">View {{.Group.host}} in Opserver</a>
            <a href="http://kibana/app/kibana?#/discover?_g=(refreshInterval:(display:Off,pause:!f,value:0),time:(from:now-15m,mode:quick,to:now))&_a=(columns:!(_source),index:%5Blogstash-%5DYYYY.MM.DD,interval:auto,query:(query_string:(analyze_wildcard:!t,query:'logsource:{{.Group.host}}')),sort:!('@timestamp',desc))">View {{.Group.host}} in Kibana</a>
        {{end}}
    </p>
    <table>
        <tr>
            <td><strong>Key: </strong></td>
            <td>{{printf "%s%s" .Alert.Name  .Group }}</td>
        </tr>
        <tr>
            <td><strong>Incident: </strong></td>
            <td><a href="{{.Incident}}">#{{.Last.IncidentId}}</a></td>
        </tr>
    </table>
    <br/>
    {{if .Alert.Vars.notes}}
        <p><strong>Notes:</strong> {{html .Alert.Vars.notes}}</p>
    {{end}}

    <p><strong>Tags</strong>
    <table>
        {{range $k, $v := .Group}}
            {{if eq $k "host"}}
                <tr><td>{{$k}}</td><td><a href="{{$.HostView $v}}">{{$v}}</a></td></tr>
            {{else}}
                <tr><td>{{$k}}</td><td>{{$v}}</td></tr>
            {{end}}
        {{end}}
    </table></p>
    `
}

Après quoi, vous pouvez ajouter commencer vos modèles avec body = `{{template "header" .}} pour obtenir la sortie suivante en haut :

Modèle générique avec graphiques facultatifs #

Il est souvent plus rapide d’utiliser un modèle générique lors de la première création d’une nouvelle alerte et de ne spécialiser le modèle que lorsque vous avez besoin d’afficher plus d’informations. Le modèle suivant affichera un sujet avec une valeur numérique, une mise en forme personnalisée et une chaîne de description, puis un corps avec jusqu’à deux graphiques. Si aucune variable de graphique n’est spécifiée, il listera à la place les calculs utilisés dans l’alerte. Le modèle générique utilise également le nom de l’alerte pour générer le sujet (en remplaçant les points par des espaces) et vérifie l’existence de variables avant de les utiliser pour éviter les erreurs.

#See Embedded Templates and CSS Styles example for header template
template header { ... } 

template computation {
    body = `
    <p><strong>Computation</strong>
    <table>
        {{range .Computations}}
            <tr><td><a href="{{$.Expr .Text}}">{{.Text}}</a></td><td>{{.Value}}</td></tr>
        {{end}}
    </table></p>`
}

template generic_template {
    subject = {{.Last.Status}}: {{replace .Alert.Name "." " " -1}}: {{if .Alert.Vars.value}}{{if .Alert.Vars.value_format}}{{.Eval .Alert.Vars.value | printf .Alert.Vars.value_format}}{{else}}{{.Eval .Alert.Vars.value | printf "%.1f"}}{{end}}{{end}}{{if .Alert.Vars.value_string}}{{.Alert.Vars.value_string}}{{end}}{{if .Group.host}} on {{.Group.host}}{{end}}
    
    body = `{{template "header" .}}

    {{if or .Alert.Vars.generic_graph .Alert.Vars.generic_graph_all}}
        <strong>Graph</strong>
        {{if and .Alert.Vars.graph_unit .Alert.Vars.generic_graph}}
            <div>{{.Graph .Alert.Vars.generic_graph .Alert.Vars.graph_unit}}</div>
        {{else if .Alert.Vars.generic_graph}}
            <div>{{.Graph .Alert.Vars.generic_graph}}</div>
        {{end}}
        {{if and .Alert.Vars.graph_unit2 .Alert.Vars.generic_graph2}}
            <div>{{.Graph .Alert.Vars.generic_graph2 .Alert.Vars.graph_unit2}}</div>
        {{else if .Alert.Vars.generic_graph2}}
            <div>{{.Graph .Alert.Vars.generic_graph2}}</div>
        {{end}}
        {{if and .Alert.Vars.generic_graph_all .Alert.Vars.graph_unit}}
            <div>{{.GraphAll .Alert.Vars.generic_graph_all .Alert.Vars.graph_unit}}</div>
        {{else if .Alert.Vars.generic_graph_all}}
            <div>{{.GraphAll .Alert.Vars.generic_graph_all}}</div>
        {{end}}
        {{if and .Alert.Vars.generic_graph_all2 .Alert.Vars.graph_unit2}}
            <div>{{.GraphAll .Alert.Vars.generic_graph_all2 .Alert.Vars.graph_unit2}}</div>
        {{else if .Alert.Vars.generic_graph_all2}}
            <div>{{.GraphAll .Alert.Vars.generic_graph_all2}}</div>
        {{end}}
    {{else}}
        {{template "computation" .}}
    {{end}}`
}


alert puppet.last.run {
    template = generic_template
    $timethreshold = 60
    $timegraph = 24h
    $notes = Checks if puppet has not run in at least ${timethreshold} minutes. Doesn't include hosts which have puppet disabled.
    
    $generic_graph = q("sum:300s-max:puppet.last_run{host=*}", "$timegraph", "") / 60
    $graph_unit = Minutes since Last Puppet Run
    $generic_graph2 = q("sum:300s-max:puppet.disabled{host=*}", "$timegraph", "")
    $graph_unit2 = Puppet Disabled=1 Enabled=0
    
    $value = last(q("sum:puppet.last_run{host=*}", "6h", "")) / 60
    $value_format = It has been %.0f
    $value_string = ` minutes since last run`
    $disabled = max(q("sum:puppet.disabled{host=*}", "60m", ""))
    warn = ($value > $timethreshold) && ! $disabled
    warnNotification = default
    runEvery = 15
}

Ce qui produira un sujet comme “avertissement : dernière exécution de la marionnette : 62 minutes se sont écoulées depuis la dernière exécution sur co-lb04” et inclura un graphique de last_run et désactivé pour cet hôte. Si vous souhaitez représenter graphiquement tous les résultats d’une requête au lieu des seuls ensembles de balises correspondants, vous pouvez utiliser $generic_graph_all et $generic_graph_all2 comme noms de variable.
