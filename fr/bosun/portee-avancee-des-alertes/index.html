Comprendre la fonction de transposition : t() #

Aperçu #

La fonction de transposition est l’une des fonctions les plus puissantes de Bosun, mais sa compréhension demande également des efforts. Il est puissant car il nous permet d’alerter à des niveaux différents de la structure des balises des données sous-jacentes.

Transposer modifie la portée de votre alerte. Cela vous permet de porter les éléments dans des collections plus importantes. Ainsi, par exemple, si vous avez des requêtes qui renvoient une portée de host, cluster et que vous souhaitez alerter en fonction de la santé du cluster et non des hôtes individuels, la transposition peut être utilisée pour ce faire.

Qu’est-ce que la portée ? La portée est la liste des clés de balise qui composent votre résultat final. Par exemple:

  • Si la portée est host, vous obtenez des résultats par hôte dans vos alertes.
  • Si votre champ d’application est vide (pas de clés de balise), vous ne pouvez éventuellement recevoir qu’une seule alerte.
  • Si votre champ d’application est “hôte, iface”, vous pouvez obtenir des alertes pour chaque interface sur chaque hôte dans le résultat.

Ainsi, les alertes que nous recevons sont liées aux balises des données. La fonction de transposition nous permet d’alerter à différentes portées autres que la structure de balise métrique. Nous pouvons donc interroger les éléments qui aboutissent à host, cluster mais alerter sur une portée cluster.

Décomposer la fonction #

La signature de la fonction de transposition est :

t(numberSet, group string) seriesSet

Il faut donc un numberSet, un scope alias groupe pour le résultat, et renvoie un seriesSet

Qu’est-ce que c’est ? #

Set, numberSets et seriesSets #

Les résultats de nombreuses fonctions dans bosun sont des ensembles, généralement un numberSet ou un seriesSet. L’ensemble complet du résultat partage la même balise keys. Et chaque élément de l’ensemble est unique à la valeur de chaque clé correspondante. Si la valeur de chaque élément de l’ensemble est une série (timestamp:value,timestamp:value) alors nous avons un seriesSet. Si la valeur de chaque élément n’est qu’un nombre, alors nous avons un numberSet.

La viande de celui-ci #

Transpose prend un numberSet et renvoie un seriesSet avec une portée plus grande (moins de clés de balise). Le seriesSet résultant est un peu étrange car l’index n’est pas le temps comme c’est le cas habituel d’un seriesSet, donc la valeur de temps n’est plus le temps et n’est qu’un numéro d’index. Elle doit donc être ignorée.

Nous finissons donc par transposer les éléments d’ensemble en valeurs de l’ensemble résultant, où le type d’ensemble résultant (un seriesSet) peut contenir plusieurs valeurs :

[![illustration de t][1]][1]

Passons en revue un exemple : #

# This returns a seriesSet of a scope of host,cluster
$connByHostCluster = q("sum:rate{counter,,1}:elastic.http.total_opened{cluster=StackExchangeNetwork|LogStash,host=ny-*}", "1h", "")

[![entrez la description de l’image ici][2]][2]

# Turn each item in the set into a numberSet by reducing it via average
$avgConnByHostCluster = avg($connByHostCluster)

[![entrez la description de l’image ici][3]][3]

# Transpose to new scope 
$clusterScope = t($avgConnByHostCluster, "cluster")

[![entrez la description de l’image ici][4]][4]

Vous pouvez maintenant faire des choses intéressantes avec chaque élément qui représente le cluster. Par exemple, vous pouvez faire sum($clusterScope > 5) (notez que $clusterScope est un seriesSet) pour obtenir le nombre d’éléments dans le cluster où chaque élément a un taux supérieur à cinq. Vous pouvez alors alerter si le nombre est supérieur à une certaine valeur. Par exemple, vous pouvez également utiliser len($clusterScope) pour obtenir le nombre d’hôtes dans chaque cluster et alerter sur le nombre d’hôtes au-dessus du seuil par rapport au nombre d’hôtes dans le cluster.
