Syntaxe #

  1. SELECT expression1, expression2, … expression_n,
  2. fonction_agrégation (expression)
  3. DES tableaux
  4. [OÙ conditions]
  5. GROUP BY expression1, expression2, … expression_n ;

Paramètres #

Paramètre DÉTAILS
expression1, expression2, … expression_n Les expressions qui ne sont pas encapsulées dans une fonction d’agrégat et doivent être incluses dans la clause GROUP BY.
aggregate_function Une fonction telle que les fonctions SUM, COUNT, MIN, MAX ou AVG.
tables les tables dont vous souhaitez récupérer les enregistrements. Il doit y avoir au moins une table répertoriée dans la clause FROM.
Conditions OÙ Facultatif. Les conditions qui doivent être remplies pour que les enregistrements soient sélectionnés.

La clause MySQL GROUP BY est utilisée dans une instruction SELECT pour collecter des données sur plusieurs enregistrements et regrouper les résultats par une ou plusieurs colonnes.

Son comportement est régi en partie par la valeur de [la variable ONLY_FULL_GROUP_BY][1]. Lorsque cette option est activée, les instructions SELECT qui regroupent par n’importe quelle colonne ne figurant pas dans la sortie renvoient une erreur. ([Il s’agit de la valeur par défaut à partir de 5.7.5][2].) Définir ou ne pas définir cette variable peut causer des problèmes aux utilisateurs naïfs ou aux utilisateurs habitués à d’autres SGBD.

[1] : http://dev.mysql.com/doc/refman/5.7/en/sql-mode.html#sqlmode_only_full_group_by [2] : http://dev.mysql.com/doc/refman/5.7/en/group-by-handling.html

GROUP BY en utilisant HAVING #

SELECT department, COUNT(*) AS "Man_Power"
FROM employees
GROUP BY department
HAVING COUNT(*) >= 10;

L’utilisation de GROUP BY ... HAVING pour filtrer des enregistrements agrégés est analogue à l’utilisation de SELECT ... WHERE pour filtrer des enregistrements individuels.

Vous pouvez également dire HAVING Man_Power >= 10 car HAVING comprend les “alias”.

Grouper en utilisant Group Concat #

[Group Concat][1] est utilisé dans MySQL pour obtenir des valeurs concaténées d’expressions avec plus d’un résultat par colonne. Cela signifie qu’il y a plusieurs lignes à sélectionner pour une colonne, telles que Name(1):Score(*)

Nom Note
Adam A+
Adam A-
Adam B
Adam C+
Facture D-
Jean A- 
SELECT Name, GROUP_CONCAT(Score ORDER BY Score desc SEPERATOR ' ') AS Grades
FROM   Grade
GROUP BY Name

Résultats:

+------+------------+ 
| Name | Grades     | 
+------+------------+ 
| Adam | C+ B A- A+ | 
| Bill | D-         | 
| John | A-         | 
+------+------------+

[1] : http://dev.mysql.com/doc/refman/5.7/en/group-by-functions.html#function_group-concat

Grouper en utilisant la fonction MIN #

Supposons une table d’employés dans laquelle chaque ligne est un employé qui a un “nom”, un “service” et un “salaire”.

SELECT department, MIN(salary) AS "Lowest salary"
FROM employees
GROUP BY department;

Cela vous indiquerait quel département contient l’employé avec le salaire le plus bas et quel est ce salaire. Trouver le “nom” de l’employé avec le salaire le plus bas dans chaque département est un problème différent, au-delà de la portée de cet exemple. Voir “groupwise max”.

  • COMPTER

Renvoie le nombre de lignes qui satisfont à un critère spécifique dans la clause “WHERE”.

Ex. : Nombre de commandes pour chaque client.

SELECT customer, COUNT(*) as orders
FROM orders
GROUP BY customer
ORDER BY customer

Résultat:

+----------+--------+
| customer | orders |
+----------+--------+
| Bob      |      3 |
| Carly    |      2 |
| Fred     |      2 |
| Jenny    |      1 |
| Tess     |      2 |
+----------+--------+
  • SOMME

Renvoie la somme de la colonne sélectionnée.

Ex. : Somme du total et des éléments pour chaque client.

SELECT customer, SUM(total) as sum_total, SUM(items) as sum_items
FROM orders
GROUP BY customer
ORDER BY customer

Résultat:

+----------+-----------+-----------+
| customer | sum_total | sum_items |
+----------+-----------+-----------+
| Bob      |      2100 |        31 |
| Carly    |      1800 |        15 |
| Fred     |       600 |         3 |
| Jenny    |       200 |         2 |
| Tess     |     14000 |        58 |
+----------+-----------+-----------+
  • MOY

Renvoie la valeur moyenne d’une colonne de valeur numérique.

Par exemple : Valeur moyenne des commandes pour chaque client.

SELECT customer, AVG(total) as avg_total
FROM orders
GROUP BY customer
ORDER BY customer

Résultat:

+----------+-----------+
| customer | avg_total |
+----------+-----------+
| Bob      |       700 |
| Carly    |       900 |
| Fred     |       300 |
| Jenny    |       200 |
| Tess     |      7000 |
+----------+-----------+
  • MAX

Renvoie la valeur la plus élevée d’une certaine colonne ou expression.

Par exemple : total de commande le plus élevé pour chaque client.

SELECT customer, MAX(total) as max_total
FROM orders
GROUP BY customer
ORDER BY customer

Résultat:

+----------+-----------+
| customer | max_total |
+----------+-----------+
| Bob      |      1300 |
| Carly    |      1000 |
| Fred     |       500 |
| Jenny    |       200 |
| Tess     |     11500 |
+----------+-----------+
  • MIN

Renvoie la valeur la plus faible d’une certaine colonne ou expression.

Par exemple : le total de commande le plus bas pour chaque client.

SELECT customer, MIN(total) as min_total
FROM orders
GROUP BY customer
ORDER BY customer

Résultat:

+----------+-----------+
| customer | min_total |
+----------+-----------+
| Bob      |       300 |
| Carly    |       800 |
| Fred     |       100 |
| Jenny    |       200 |
| Tess     |      2500 |
+----------+-----------+

GROUPER EN UTILISANT LA FONCTION SOMME #

SELECT product, SUM(quantity) AS "Total quantity"
FROM order_details
GROUP BY product;

Grouper en utilisant la fonction COUNT #

SELECT department, COUNT(*) AS "Man_Power"
FROM employees
GROUP BY department;
