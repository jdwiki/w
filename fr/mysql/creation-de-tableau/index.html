Syntaxe #

  • CREATE TABLE nom_table ( nom_colonne1 type_données(taille), nom_colonne2 type_données(taille), nom_colonne3 type_données(taille), …. ); // Création de table de base

  • CREATE TABLE nom_table [SI NON EXISTE] ( nom_colonne1 type_données(taille), nom_colonne2 type_données(taille), nom_colonne3 type_données(taille), …. ); // Création de table vérifiant l’existant

  • CREATE [TEMPORARY] TABLE nom_table [SI NON EXISTE] ( nom_colonne1 type_données(taille), nom_colonne2 type_données(taille), nom_colonne3 type_données(taille), …. ); // Création de table temporaire

  • CREATE TABLE new_tbl [AS] SELECT * FROM orig_tbl ; // Création de table depuis SELECT

L’instruction CREATE TABLE doit se terminer par une spécification ENGINE :

CREATE TABLE table_name ( column_definitions ) ENGINE=engine;

Certaines options sont :

  • InnoDB : (valeur par défaut depuis la version 5.5.5) C’est un moteur transation-safe (conforme ACID). Il dispose de la validation et de l’annulation des transactions, ainsi que de capacités de récupération sur incident et de verrouillage au niveau des lignes.
  • MyISAM : (Par défaut avant la version 5.5.5) C’est un moteur simple et rapide. Il ne prend pas en charge les transactions, ni les clés étrangères, mais il est utile pour l’entreposage de données.
  • Mémoire : stocke toutes les données dans la RAM pour des opérations extrêmement rapides, mais la date de la table sera perdue au redémarrage de la base de données.

Plus d’options de moteur ici.

Création de table avec clé primaire #

CREATE TABLE Person (
    PersonID     INT UNSIGNED NOT NULL,
    LastName     VARCHAR(66) NOT NULL,
    FirstName    VARCHAR(66),
    Address      VARCHAR(255),
    City         VARCHAR(66),
    PRIMARY KEY (PersonID)
);

Une clé primaire est un identifiant “NOT NULL” unique ou multi-colonnes qui identifie de manière unique une ligne d’une table. Un [index][1] est créé, et s’il n’est pas explicitement déclaré comme NOT NULL, MySQL le déclarera silencieusement et implicitement.

Une table ne peut avoir qu’une seule CLÉ PRIMAIRE, et il est recommandé d’en avoir une pour chaque table. InnoDB en créera automatiquement un en son absence (comme on le voit dans la [documentation MySQL] [2]) bien que ce soit moins souhaitable.

Souvent, un AUTO_INCREMENT INT également connu sous le nom de “clé de substitution”, est utilisé pour l’optimisation de l’index fin et les relations avec d’autres tables. Cette valeur augmentera (normalement) de 1 chaque fois qu’un nouvel enregistrement est ajouté, à partir d’une valeur par défaut de 1.

Cependant, malgré son nom, son but n’est pas de garantir que les valeurs sont incrémentales, mais simplement qu’elles sont séquentielles et uniques.

Une valeur INT auto-incrémentée ne sera pas réinitialisée à sa valeur de départ par défaut si toutes les lignes de la table sont supprimées, sauf si la table est tronquée à l’aide de l’instruction [TRUNCATE TABLE][3].

Définir une colonne comme clé primaire (définition en ligne) #

Si la clé primaire se compose d’une seule colonne, la clause “PRIMARY KEY” peut être placée en ligne avec la définition de la colonne :

CREATE TABLE Person (
    PersonID     INT UNSIGNED NOT NULL PRIMARY KEY,
    LastName     VARCHAR(66) NOT NULL,
    FirstName    VARCHAR(66),
    Address      VARCHAR(255),
    City         VARCHAR(66)
);

Cette forme de commande est plus courte et plus facile à lire.

Définir une clé primaire à plusieurs colonnes #

Il est également possible de définir une clé primaire comprenant plus d’une colonne. Cela peut être fait par ex. sur la table enfant d’une relation de clé étrangère. Une clé primaire multi-colonnes est définie en listant les colonnes participantes dans une clause PRIMARY KEY distincte. La syntaxe en ligne n’est pas autorisée ici, car une seule colonne peut être déclarée “PRIMARY KEY” en ligne. Par exemple:

CREATE TABLE invoice_line_items (
    LineNum      SMALLINT UNSIGNED NOT NULL,
    InvoiceNum   INT UNSIGNED NOT NULL,
    -- Other columns go here
    PRIMARY KEY (InvoiceNum, LineNum),
    FOREIGN KEY (InvoiceNum) REFERENCES -- references to an attribute of a table
);

Notez que les colonnes de la clé primaire devraient être spécifiées dans l’ordre de tri logique, qui peut être différent de l’ordre dans lequel les colonnes ont été définies, comme dans l’exemple ci-dessus.

Les index plus volumineux nécessitent plus d’espace disque, de mémoire et d’E/S. Par conséquent, les clés doivent être aussi petites que possible (surtout en ce qui concerne les clés composées). Dans InnoDB, chaque ‘index secondaire’ inclut une copie des colonnes de la ‘CLE PRIMAIRE’.

[1] : https://www.wikiod.com/fr/mysql/index-et-cles [2] : https://dev.mysql.com/doc/refman/5.7/en/create-table.html [3] : https://www.wikiod.com/fr/sql/tronquer [4] : https://www.wikiod.com/fr/mysql/creation-de-tableau#Création de table avec clé étrangère

Création de table de base #

L’instruction CREATE TABLE est utilisée pour créer une table dans une base de données MySQL.

CREATE TABLE Person (
    `PersonID`      INTEGER NOT NULL PRIMARY KEY,
    `LastName`      VARCHAR(80),
    `FirstName`     VARCHAR(80),
    `Address`       TEXT,
    `City`          VARCHAR(100)
) Engine=InnoDB;

Chaque définition de champ doit avoir :

  1. Nom de champ : Un nom de champ valide. Assurez-vous d’encadrer les noms entre `-chars. Cela garantit que vous pouvez utiliser par exemple des caractères d’espacement dans le nom du champ.
  2. Type de données [Longueur] : si le champ est CHAR ou VARCHAR, il est obligatoire de spécifier une longueur de champ.
  3. Attributs NULL | NOT NULL : si NOT NULL est spécifié, toute tentative de stockage d’une valeur NULL dans ce champ échouera.
  4. En savoir plus sur les types de données et leurs attributs [ici][1].

Engine=... est un paramètre facultatif utilisé pour spécifier le moteur de stockage de la table. Si aucun moteur de stockage n’est spécifié, la table sera créée à l’aide du moteur de stockage de table par défaut du serveur (généralement InnoDB ou MyISAM).

Réglage des valeurs par défaut #

De plus, lorsque cela a du sens, vous pouvez définir une valeur par défaut pour chaque champ en utilisant DEFAULT :

CREATE TABLE Address (
    `AddressID`   INTEGER NOT NULL PRIMARY KEY,
    `Street`      VARCHAR(80),
    `City`        VARCHAR(80),
    `Country`     VARCHAR(80) DEFAULT "United States",
    `Active`      BOOLEAN DEFAULT 1,
) Engine=InnoDB;

Si pendant les insertions aucune Street n’est spécifiée, ce champ sera NULL lors de la récupération. Lorsqu’aucun “Pays” n’est spécifié lors de l’insertion, la valeur par défaut est “États-Unis”.

Vous pouvez définir des valeurs par défaut pour tous les types de colonnes, [sauf][2] pour les champs BLOB, TEXT, GEOMETRY et JSON.

[1] : http://dev.mysql.com/doc/refman/5.7/en/data-types.html [2] : http://dev.mysql.com/doc/refman/5.7/en/data-type-defaults.html

Création de table avec clé étrangère #

CREATE TABLE Account (
    AccountID     INT UNSIGNED NOT NULL,
    AccountNo     INT UNSIGNED NOT NULL,
    PersonID    INT UNSIGNED,
    PRIMARY KEY (AccountID),
    FOREIGN KEY (PersonID) REFERENCES Person (PersonID)
) ENGINE=InnoDB;

Clé étrangère : Une clé étrangère (FK) est soit une colonne unique, soit un composé de plusieurs colonnes de colonnes, dans une table de référencement. L’existence de ce FK est confirmée dans la table référencée. Il est fortement recommandé que la clé de table référencée confirmant le FK soit une clé primaire, mais cela n’est pas appliqué. Il est utilisé comme une recherche rapide dans le référencé où il n’a pas besoin d’être unique, et peut en fait être un index le plus à gauche.

Les relations de clé étrangère impliquent une table parent qui contient les valeurs de données centrales et une table enfant avec des valeurs identiques pointant vers son parent. La clause FOREIGN KEY est spécifiée dans la table enfant. Les tables parent et enfant doivent utiliser le même moteur de stockage. Il ne doit pas s’agir de tables [TEMPORARY][1].

Les colonnes correspondantes dans la clé étrangère et la clé référencée doivent avoir des types de données similaires. La taille et le signe des types entiers doivent être identiques. La longueur des types de chaîne n’a pas besoin d’être la même. Pour les colonnes de chaînes non binaires (caractères), le jeu de caractères et le classement doivent être identiques.

Remarque : les contraintes de clé étrangère sont prises en charge par le moteur de stockage InnoDB (pas MyISAM ou MEMORY). Les configurations de base de données utilisant d’autres moteurs accepteront cette instruction CREATE TABLE mais ne respecteront pas les contraintes de clé étrangère. (Bien que les nouvelles versions de MySQL utilisent par défaut InnoDB, mais c’est une bonne pratique d’être explicite.)

[1] : https://dev.mysql.com/doc/refman/5.7/en/create-table.html

Afficher la structure du tableau #

Si vous souhaitez voir les informations de schéma de votre table, vous pouvez utiliser l’une des options suivantes :

SHOW CREATE TABLE child; -- Option 1

CREATE TABLE `child` (
  `id` int(11) NOT NULL AUTO_INCREMENT,
  `fullName` varchar(100) NOT NULL,
  `myParent` int(11) NOT NULL,
  PRIMARY KEY (`id`),
  KEY `mommy_daddy` (`myParent`),
  CONSTRAINT `mommy_daddy` FOREIGN KEY (`myParent`) REFERENCES `parent` (`id`) 
      ON DELETE CASCADE ON UPDATE CASCADE
) ENGINE=InnoDB DEFAULT CHARSET=utf8;

S’il est utilisé depuis l’outil de ligne de commande mysql, c’est moins verbeux :

SHOW CREATE TABLE child \G

Une manière moins descriptive d’afficher la structure du tableau :

mysql> CREATE TABLE Tab1(id int, name varchar(30));
Query OK, 0 rows affected (0.03 sec)

mysql> DESCRIBE Tab1; -- Option 2 

+-------+-------------+------+-----+---------+-------+ 
| Field | Type        | Null | Key | Default | Extra |  
+-------+-------------+------+-----+---------+-------+ 
| id    | int(11)     | YES  |     | NULL    |       | 
| name  | varchar(30) | YES  |     | NULL    |       |  
+-------+-------------+------+-----+---------+-------+

DESCRIBE et DESC donnent le même résultat.

Pour voir DESCRIBE exécuté sur toutes les tables d’une base de données à la fois, consultez cet [Exemple][1].

[1] : http://stackoverflow.com/a/38679580

Cloner une table existante #

Une table peut être répliquée comme suit :

CREATE TABLE ClonedPersons LIKE Persons;

La nouvelle table aura exactement la même structure que la table d’origine, y compris les index et les attributs de colonne.

Outre la création manuelle d’un tableau, il est également possible de créer un tableau en sélectionnant des données dans un autre tableau :

CREATE TABLE ClonedPersons SELECT * FROM Persons;

Vous pouvez utiliser n’importe laquelle des fonctionnalités normales d’une instruction SELECT pour modifier les données au fur et à mesure :

CREATE TABLE ModifiedPersons
SELECT PersonID, FirstName + LastName AS FullName FROM Persons
WHERE LastName IS NOT NULL;

Les clés primaires et les index ne seront pas conservés lors de la création de tables à partir de SELECT. Vous devez les redéclarer :

CREATE TABLE ModifiedPersons (PRIMARY KEY (PersonID))
SELECT PersonID, FirstName + LastName AS FullName FROM Persons
WHERE LastName IS NOT NULL;

CRÉER UNE TABLE À PARTIR DE LA SÉLECTION #

Vous pouvez créer une table à partir d’une autre en ajoutant une instruction SELECT à la fin de l’instruction CREATE TABLE :

CREATE TABLE stack (
    id_user INT,
    username VARCHAR(30),
    password VARCHAR(30)
);

Créer une table dans la même base de données :

-- create a table from another table in the same database with all attributes
CREATE TABLE stack2 AS SELECT * FROM stack;

-- create a table from another table in the same database with some attributes
CREATE TABLE stack3 AS SELECT username, password FROM stack;

Créer des tables à partir de différentes bases de données :

-- create a table from another table from another database with all attributes
CREATE TABLE stack2 AS SELECT * FROM second_db.stack;

-- create a table from another table from another database with some attributes
CREATE TABLE stack3 AS SELECT username, password FROM second_db.stack;

N.B

Pour créer une table identique à une autre table qui existe dans une autre base de données, vous devez spécifier le nom de la base de données comme ceci :

FROM NAME_DATABASE.name_table

Table créée avec une colonne d’horodatage pour afficher la dernière mise à jour #

La colonne TIMESTAMP indiquera quand la ligne a été mise à jour pour la dernière fois.

CREATE TABLE `TestLastUpdate` (
    `ID` INT NULL,
    `Name` VARCHAR(50) NULL,
    `Address` VARCHAR(50) NULL,
    `LastUpdate` TIMESTAMP NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP
)
COMMENT='Last Update'
;
