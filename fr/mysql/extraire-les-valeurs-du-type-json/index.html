Extraire les valeurs du type JSON

  • Syntaxe
  • Paramètres
  • Lire la valeur du tableau JSON
  • Opérateurs d’extraction JSON

Sur cette page

  • Syntaxe
  • Paramètres
  • Lire la valeur du tableau JSON
  • Opérateurs d’extraction JSON

MySQL 5.7.8+ prend en charge le type JSON natif. Bien que vous ayez différentes façons de créer des objets json, vous pouvez également accéder aux membres et les lire de différentes manières.

La fonction principale est JSON_EXTRACT, donc les opérateurs -> et ->> sont plus conviviaux.

Syntaxe #

  • JSON_EXTRACT(json_doc,chemin[,…])
  • JSON_EXTRACT(json_doc,chemin)
  • JSON_EXTRACT(json_doc,chemin1,chemin2)

Paramètres #

Paramètre Description
json_doc document JSON valide
chemin chemin des membres

Mentionné dans MySQL 5.7 Reference Manual

  • Plusieurs valeurs correspondantes par argument(s) de chemin

S’il est possible que ces arguments renvoient plusieurs valeurs, les valeurs correspondantes sont automatiquement enveloppées sous forme de tableau, dans l’ordre correspondant aux chemins qui les ont produites. Sinon, la valeur de retour est la seule valeur correspondante.

- NULL Résultat lorsque :

  • tout argument est NULL
  • chemin non adapté

Renvoie NULL si un argument est NULL ou si aucun chemin ne localise une valeur dans le document.

Lire la valeur du tableau JSON #

Créez la variable @myjson en tant que type JSON (en savoir plus) :

 SET @myjson = CAST('["A","B",{"id":1,"label":"C"}]' as JSON) ;

SÉLECTIONNER certains membres !

 SELECT
   JSON_EXTRACT( @myjson , '$[1]' ) ,
   JSON_EXTRACT( @myjson , '$[*].label') ,
   JSON_EXTRACT( @myjson , '$[1].*' ) ,
   JSON_EXTRACT( @myjson , '$[2].*')
 ;
 -- result values:
 '\"B\"', '[\"C\"]', NULL, '[1, \"C\"]'
 -- visually:
 "B", ["C"], NULL, [1, "C"]

Opérateurs d’extraction JSON #

Extrayez path par les opérateurs -> ou ->>, tandis que ->> est une valeur UNQUOTED :

 SELECT
   myjson_col->>'$[1]' , myjson_col->'$[1]' ,
   myjson_col->>'$[*].label' ,
   myjson_col->>'$[1].*' ,
   myjson_col->>'$[2].*'
 FROM tablename ;
  -- visuall:
     B, "B" , ["C"], NULL, [1, "C"]
  --^^^ ^^^

Donc col->>path est égal à JSON_UNQUOTE(JSON_EXTRACT(col,path)) :

Comme pour ->, l’opérateur -» est toujours développé dans la sortie de EXPLAIN, comme le montre l’exemple suivant :

mysql> EXPLAIN SELECT c-»’$.name’ AS nom -> FROM jemp WHERE g > 2\G *************************** 1ère rangée ******************* ******** identifiant : 1 select_type : SIMPLE tableau : jemp partitions : NULL type : gamme clés_possibles : je clé : je key_len : 5 réf : NUL rangs : 2 filtré : 100,00 Extra : Utiliser où 1 ligne en set, 1 avertissement (0.00 sec)

mysql> AFFICHER LES AVERTISSEMENTS\G *************************** 1ère rangée ******************* ******** Niveau : Remarque Code : 1003 Message : /* select#1 */ select json_unquote(json_extract(jtest.jemp.c,’$.name’)) AS name from jtest.jemp où (jtest.jemp.g > 2) 1 ligne dans le jeu (0,00 sec) En savoir plus sur inline path extract(+)
