Syntaxe #

  • CRÉER [DEFINER = { user | CURRENT_USER }] TRIGGER trigger_name trigger_time trigger_event ON tbl_name FOR EACH ROW [trigger_order] trigger_body
  • trigger_time : { AVANT | APRÈS }
  • événement_déclencheur : { INSÉRER | MISE À JOUR | EFFACER }
  • trigger_order : { SUIT | PRECEDES } autre_nom_déclencheur

Deux points doivent attirer votre attention si vous utilisez déjà des triggers sur d’autres DB :

POUR CHAQUE LIGNE #

FOR EACH ROW est une partie obligatoire de la syntaxe

Vous ne pouvez pas créer un déclencheur instruction (une fois par requête) comme le fait Oracle. C’est plus un problème lié aux performances qu’une véritable fonctionnalité manquante

CRÉER OU REMPLACER LE DÉCLENCHEUR #

Le CREATE OR REPLACE n’est pas pris en charge par MySQL

MySQL n’autorise pas cette syntaxe, vous devez plutôt utiliser ce qui suit :

DELIMITER $$

DROP TRIGGER IF EXISTS myTrigger;
$$
CREATE TRIGGER myTrigger
-- ...

$$
DELIMITER ;

Attention, ceci n’est pas une transaction atomique :

  • vous perdrez l’ancien déclencheur si le CREATE échoue
  • sur une charge importante, d’autres opérations peuvent se produire entre le DROP et le CREATE, utilisez un LOCK TABLES myTable WRITE; d’abord pour éviter l’incohérence des données et UNLOCK TABLES; après le CREATE pour libérer la table

Déclencheur de base #

Créer un tableau

mysql> CREATE TABLE account (acct_num INT, amount DECIMAL(10,2));
Query OK, 0 rows affected (0.03 sec)

Créer un déclencheur

mysql> CREATE TRIGGER ins_sum BEFORE INSERT ON account
    -> FOR EACH ROW SET @sum = @sum + NEW.amount;
Query OK, 0 rows affected (0.06 sec)

L’instruction CREATE TRIGGER crée un déclencheur nommé ins_sum qui est associé à la table des comptes. Il comprend également des clauses qui spécifient l’heure d’action du déclencheur, l’événement déclencheur et ce qu’il faut faire lorsque le déclencheur s’active.

Insérer la valeur

Pour utiliser le déclencheur, définissez la variable d’accumulateur (@sum) sur zéro, exécutez une instruction INSERT, puis voyez quelle valeur a la variable par la suite :

mysql> SET @sum = 0;
mysql> INSERT INTO account VALUES(137,14.98),(141,1937.50),(97,-100.00);
mysql> SELECT @sum AS 'Total amount inserted';
+-----------------------+
| Total amount inserted |
+-----------------------+
| 1852.48               |
+-----------------------+

Dans ce cas, la valeur de @sum après l’exécution de l’instruction INSERT est 14,98 + 1937,50 - 100, soit 1852,48.

** Déclencheur de chute **

mysql> DROP TRIGGER test.ins_sum;

Si vous supprimez une table, tous les déclencheurs de la table sont également supprimés.

Types de déclencheurs #

Horaire #

Il existe deux modificateurs de temps d’action de déclenchement :

  • Le déclencheur BEFORE s’active avant l’exécution de la requête,
  • APRÈS déclencher le feu après le changement.

Evénement déclencheur #

Il existe trois événements auxquels les déclencheurs peuvent être associés :

  • INSÉRER
  • MISE À JOUR
  • ‘SUPPRIMER’

Exemple de déclencheur avant insertion #

DELIMITER $$

CREATE TRIGGER insert_date 
    BEFORE INSERT ON stack
    FOR EACH ROW
BEGIN
        -- set the insert_date field in the request before the insert
    SET NEW.insert_date = NOW();
END;

$$
DELIMITER ;

Exemple de déclencheur avant la mise à jour #

DELIMITER $$

CREATE TRIGGER update_date 
    BEFORE UPDATE ON stack
    FOR EACH ROW
BEGIN
        -- set the update_date field in the request before the update
    SET NEW.update_date = NOW();
END;

$$
DELIMITER ;

Exemple de déclencheur après suppression #

DELIMITER $$

CREATE TRIGGER deletion_date 
    AFTER DELETE ON stack
    FOR EACH ROW
BEGIN
        -- add a log entry after a successful delete
    INSERT INTO log_action(stack_id, deleted_date) VALUES(OLD.id, NOW());
END;

$$
DELIMITER ;
