MySQL, sur la plupart des machines, utilise [l’arithmétique à virgule flottante IEEE 754 64 bits] [1] pour ses calculs.

Dans les contextes entiers, il utilise l’arithmétique entière.

  • RAND() n’est pas un générateur de nombres aléatoires parfait. Il est principalement utilisé pour générer rapidement des nombres pseudo-aléatoires

[1] : https://en.wikipedia.org/wiki/IEEE_floating_point

Opérateurs arithmétiques #

MySQL fournit les opérateurs arithmétiques suivants

Opérateur Nom Exemple
+ Ajout SELECT 3+5; -> 8
SELECT 3.5+2.5; -> 6.0
SELECT 3.5+2; -> 5.5
- Soustraction SELECT 3-5; -> -2
* Multiplication SELECT 3 * 5; -> 15
/ Section SELECT 20 / 4; -> 5
SELECT 355 / 113; -> 3.1416
SELECT 10.0 / 0; -> NULL
DIV Division entière SELECT 5 DIV 2; -> 2
% ou MOD Modulo SELECT 7 % 3; -> 1
SELECT 15 MOD 4 -> 3
SELECT 15 MOD -4 -> 3
SELECT -15 MOD 4 -> -3
SELECT -15 MOD -4 -> -3
SELECT 3 MOD 2.5 -> 0.5

BIGINT #

Si les nombres de votre arithmétique sont tous des entiers, MySQL utilise le type de données entier BIGINT (64 bits signé) pour faire son travail. Par exemple:

sélectionner (1024 * 1024 * 1024 * 1024 *1024 * 1024) + 1 -> 1 152 921 504 606 846 977

et

select (1024 * 1024 * 1024 * 1024 *1024 * 1024 * 1024 -> BIGINT erreur hors plage

DOUBLE #

Si des nombres dans votre arithmétique sont fractionnaires, MySQL utilise [arithmétique à virgule flottante IEEE 754 64 bits] [1]. Vous devez être prudent lorsque vous utilisez l’arithmétique à virgule flottante, car de nombreux [nombres à virgule flottante sont, par nature, des approximations plutôt que des valeurs exactes][2].

[1] : https://en.wikipedia.org/wiki/IEEE_floating_point [2] : http://dev.mysql.com/doc/refman/5.7/en/problems-with-float.html

Constantes mathématiques #

##Pi

Ce qui suit renvoie la valeur de PI formatée à 6 décimales. La valeur réelle est bonne à “DOUBLE” ;

SELECT PI();    -> 3.141593

Trigonométrie (SIN, COS) #

Les angles sont en radians et non en degrés. Tous les calculs sont effectués en [virgule flottante IEEE 754 64 bits][1]. Tous les calculs en virgule flottante sont sujets à de petites erreurs, appelées [machine ε (epsilon)][2], évitez donc d’essayer de les comparer pour l’égalité. Il n’y a aucun moyen d’éviter ces erreurs lors de l’utilisation de la virgule flottante ; ils sont intégrés à la technologie.

Si vous utilisez des valeurs DECIMAL dans les calculs trigonométriques, elles sont implicitement converties en virgule flottante, puis de nouveau en décimales.

Sinus #

Renvoie le sinus d’un nombre X exprimé en radians

SELECT SIN(PI()); -> 1.2246063538224e-16

Cosinus #

Renvoie le cosinus de X lorsque X est donné en radians

SELECT COS(PI()); -> -1

Tangente #

Renvoie la tangente d’un nombre X exprimé en radians. Notez que le résultat est très proche de zéro, mais pas exactement zéro. Ceci est un exemple de machine ε.

SELECT TAN(PI());   -> -1.2246063538224e-16

Arc Cosinus (cosinus inverse) #

Renvoie l’arc cosinus de X si X est compris entre -1 et 1

SELECT ACOS(1);    -> 0
SELECT ACOS(1.01); -> NULL

Arc sinus (sinus inverse) #

Renvoie l’arc sinus de X si X est compris entre -1 et 1

SELECT ASIN(0.2); -> 0.20135792079033

Arc Tangente (tangente inverse) #

ATAN(x) renvoie l’arc tangente d’un seul nombre.

SELECT ATAN(2); -> 1.1071487177941

ATAN2(X, Y) renvoie l’arc tangente des deux variables X et Y. Il est similaire au calcul de l’arc tangente de Y / X. Mais il est numériquement plus robuste : t fonctionne correctement lorsque X est proche de zéro, et les signes des deux arguments sont utilisés pour déterminer le quadrant du résultat.

La meilleure pratique suggère d’écrire des formules pour utiliser ATAN2() plutôt que ATAN() dans la mesure du possible.

 ATAN2(1,1);    -> 0.7853981633974483 (45 degrees)
 ATAN2(1,-1);   -> 2.356194490192345  (135 degrees)
 ATAN2(0, -1);  -> PI  (180 degrees)  don't try ATAN(-1 / 0)... it won't work

Cotangente #

Renvoie la cotangente de X

SELECT COT(12); -> -1.5726734063977

Conversion #

SELECT RADIANS(90) -> 1.5707963267948966
SELECT SIN(RADIANS(90)) -> 1
SELECT DEGREES(1), DEGREES(PI()) -> 57.29577951308232, 180

[1] : https://en.wikipedia.org/wiki/Double-precision_floating-point_format [2] : https://en.wikipedia.org/wiki/Machine_epsilon

Arrondi (ROUND, FLOOR, CEIL) #

Arrondir un nombre décimal à une valeur entière #

Pour les valeurs numériques exactes (par exemple, DECIMAL) : si la première décimale d’un nombre est 5 ou plus, cette fonction arrondira un nombre à l’entier suivant loin de zéro. Si cette décimale est 4 ou moins, cette fonction arrondira à la valeur entière suivante * la plus proche de zéro *.

SELECT ROUND(4.51) -> 5
SELECT ROUND(4.49) -> 4
SELECT ROUND(-4.51) -> -5

Pour les valeurs numériques approximatives (par exemple, DOUBLE) : le résultat de la fonction ROUND() dépend de la bibliothèque C ; sur de nombreux systèmes, cela signifie que ROUND() utilise la règle arrondir au pair le plus proche :

SELECT ROUND(45e-1) -> 4  -- The nearest even value is 4
SELECT ROUND(55e-1) -> 6  -- The nearest even value is 6

Arrondir un nombre #

Pour arrondir un nombre, utilisez la fonction CEIL() ou CEILING()

SELECT CEIL(1.23)    -> 2
SELECT CEILING(4.83) -> 5

Arrondissez un nombre #

Pour arrondir un nombre, utilisez la fonction FLOOR()

SELECT FLOOR(1.99) -> 1

FLOOR et CEIL vont vers / s’éloignent de -l’infini :

SELECT FLOOR(-1.01), CEIL(-1.01) -> -2 and -1
SELECT FLOOR(-1.99), CEIL(-1.99) -> -2 and -1

Arrondit un nombre décimal à un nombre spécifié de décimales. #

SELECT ROUND(1234.987, 2) -> 1234.99
SELECT ROUND(1234.987, -2) -> 1200

La discussion entre haut et bas et “5” s’applique également.

Élever un nombre à une puissance (POW) #

Pour élever un nombre x à une puissance y, utilisez les fonctions POW() ou POWER()

SELECT POW(2,2); => 4
SELECT POW(4,2); => 16

Racine carrée (SQRT) #

Utilisez la fonction SQRT(). Si le nombre est négatif, NULL sera renvoyé

SELECT SQRT(16); -> 4
SELECT SQRT(-3); -> NULL

Nombres aléatoires (RAND) #

Générer un nombre aléatoire #

Pour générer un nombre à virgule flottante pseudo-aléatoire entre ‘0’ et ‘1’, utilisez la fonction ‘RAND()’

Supposons que vous ayez la requête suivante

SELECT i, RAND() FROM t;

Cela renverra quelque chose comme ça

je ARAND()
1 0.6191438870682
2 0.93845168309142
3 0.83482678498591

Nombre aléatoire dans une plage #

Pour générer un nombre aléatoire dans la plage a <= n <= b, vous pouvez utiliser la formule suivante

FLOOR(a + RAND() * (b - a + 1))

Par exemple, cela générera un nombre aléatoire entre 7 et 12

SELECT FLOOR(7 + (RAND() * 6));

Un moyen simple de renvoyer aléatoirement les lignes d’un tableau :

SELECT * FROM tbl ORDER BY RAND();

Ce sont des nombres pseudo-aléatoires.

Le générateur de nombres pseudo-aléatoires de MySQL n’est pas cryptographiquement sécurisé. Autrement dit, si vous utilisez MySQL pour générer des nombres aléatoires à utiliser comme secrets, un adversaire déterminé qui sait que vous avez utilisé MySQL pourra deviner vos secrets plus facilement que vous ne le pensez.

Valeur absolue et signe (ABS, SIGN) #

Renvoie la valeur absolue d’un nombre

SELECT ABS(2);   -> 2
SELECT ABS(-46); -> 46

Le “signe” d’un nombre le compare à 0.

signe Résultat Exemple
-1 n < 0 SÉLECTIONNER SIGNE(42); -> 1
0 n = 0 SÉLECTIONNER SIGNE(0); -> 0
1 n > 0 SÉLECTIONNER SIGNE(-3); -> -1 
SELECT SIGN(-423421); -> -1
