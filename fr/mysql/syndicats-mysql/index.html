Syntaxe #

  • SELECT nom(s) de colonne(s) FROM table1 UNION SELECT nom(s) de colonne(s) FROM table2 ;
  • SELECT nom(s) de colonne(s) FROM table1 UNION TOUS SELECT nom_colonne(s) FROM table2 ;
  • SELECT nom(s) de colonne(s) FROM table1 WHERE col_name=“XYZ” UNION TOUS SELECT nom_colonne(s) FROM table2 WHERE col_name=“XYZ” ;

“UNION DISTINCT” est identique à “UNION” ; il est plus lent que UNION ALL en raison d’une passe de déduplication. Une bonne pratique consiste à toujours épeler “DISTINCT” ou “TOUS”, signalant ainsi que vous avez réfléchi à ce qu’il faut faire.

Opérateur syndical #

L’opérateur UNION est utilisé pour combiner le jeu de résultats (uniquement des valeurs distinctes) de deux ou plusieurs instructions SELECT.

Requête : (Pour sélectionner toutes les villes différentes (uniquement des valeurs distinctes) des tables “Clients” et “Fournisseurs”)

SELECT City FROM Customers
UNION
SELECT City FROM Suppliers
ORDER BY City;

Résultat:

Number of Records: 10

City
------
Aachen
Albuquerque
Anchorage
Annecy
Barcelona
Barquisimeto
Bend
Bergamo
Berlin
Bern

Union TOUS #

UNION ALL pour sélectionner toutes les villes (valeurs en double également) des tables “Clients” et “Fournisseurs”.

Requête:

SELECT City FROM Customers
UNION ALL
SELECT City FROM Suppliers
ORDER BY City;

Résultat:

Number of Records: 12

City
-------
Aachen
Albuquerque
Anchorage
Ann Arbor
Annecy
Barcelona
Barquisimeto
Bend
Bergamo
Berlin
Berlin
Bern

UNION TOUT AVEC OÙ #

UNION ALL pour sélectionner toutes les villes allemandes (valeurs en double également) dans les tables “Clients” et “Fournisseurs”. Ici, Country="Germany" doit être spécifié dans la clause where.

Requête:

SELECT City, Country FROM Customers
WHERE Country='Germany'
UNION ALL
SELECT City, Country FROM Suppliers
WHERE Country='Germany'
ORDER BY City;
