Syntaxe #

  • git remote [-v | --verbeux]
  • git remote add [-t <branch>] [-m <master>] [-f] [--[no-]tags] [--mirror=<fetch|push>]<nom> <url>
  • git remote rename <ancien> <nouveau>
  • git remote remove <nom>
  • git remote set-head <nom> (-a | --auto | -d | --delete | <branche>)
  • git remote set-branches [--add] <nom> <branche>...
  • git remote set-url [--push] <name> <newurl> [<oldurl>]
  • git remote set-url --add [--push] <nom> <nouvelleurl>
  • git remote set-url --delete [--push] <nom> <url>
  • git remote [-v | --verbose] affiche [-n] <nom>...
  • git remote prune [-n | --dry-run] <nom>...
  • git remote [-v | --verbose] mise à jour [-p | --prune] [(<groupe> | <distant>)...]
  • git remote show <nom>

Paramètres #

Paramètre Détails
-v, –verbose Exécutez verbalement.
-m <maître> Définit head sur la branche <master> de la télécommande
–mirror=extraire Les références ne seront pas stockées dans l’espace de noms refs/remotes, mais seront reflétées dans le dépôt local
–mirror=pousser git push se comportera comme si –mirror était passé
–no-tags git fetch <nom> n’importe pas les balises du référentiel distant
-t <branche> Spécifie la télécommande à suivre uniquement <branch>
-f git fetch <nom> est exécuté immédiatement après la configuration de la télécommande
–tags git fetch <nom> importe chaque balise du référentiel distant
-a, –auto Le HEAD de la référence symbolique est défini sur la même branche que le HEAD
-d, –delete Toutes les références répertoriées sont supprimées du référentiel distant
–add Ajoute <name> à la liste des branches actuellement suivies (set-branches)
–add Au lieu de changer une URL, une nouvelle URL est ajoutée (set-url)
–tout Poussez toutes les branches.
–delete Toutes les URL correspondant à <url> sont supprimées. (ensemble-url)
–pousser Les URL push sont manipulées au lieu de récupérer les URL
-n Les têtes distantes ne sont pas interrogées en premier avec git ls-remote <nom>, les informations mises en cache sont utilisées à la place
–dry-run signalez les branches qui seront taillées, mais ne les taillez pas réellement
–prune Supprimer les branches distantes qui n’ont pas d’équivalent local

Afficher les référentiels distants #

Pour lister tous les référentiels distants configurés, utilisez git remote.

Il affiche le nom abrégé (alias) de chaque poignée distante que vous avez configurée.

$ git remote
premium
premiumPro
origin

Pour afficher des informations plus détaillées, le drapeau --verbose ou -v peut être utilisé. La sortie inclura l’URL et le type de la télécommande (“push” ou “pull”) :

$ git remote -v
premiumPro    https://github.com/user/CatClickerPro.git (fetch)
premiumPro    https://github.com/user/CatClickerPro.git (push)
premium    https://github.com/user/CatClicker.git (fetch)
premium    https://github.com/user/CatClicker.git (push)
origin    https://github.com/ud/starter.git (fetch)
origin    https://github.com/ud/starter.git (push)

Modifier l’URL distante de votre référentiel Git #

Vous pouvez effectuer cette opération si le référentiel distant est migré. La commande pour changer l’URL distante est :

git remote set-url

Il prend 2 arguments : un nom distant existant (origine, amont) et l’url.

Vérifiez votre URL distante actuelle :

git remote -v
origin    https://bitbucket.com/develop/myrepo.git (fetch)
origin    https://bitbucket.com/develop/myrepo.git (push)

Modifiez votre URL distante :

git remote set-url origin https://localserver/develop/myrepo.git

Vérifiez à nouveau votre URL distante :

git remote -v
origin    https://localserver/develop/myrepo.git (fetch)
origin    https://localserver/develop/myrepo.git (push)

Supprimer un référentiel distant #

Supprimez la télécommande nommée <nom>. Toutes les branches de suivi à distance et les paramètres de configuration de la télécommande sont supprimés.

Pour supprimer un dépôt distant dev :

git remote rm dev

Ajouter un référentiel distant #

Pour ajouter une télécommande, utilisez git remote add à la racine de votre référentiel local.

Pour ajouter un référentiel Git distant <url> en tant que nom abrégé facile <nom> utilisez

git remote add <name> <url>

La commande git fetch <name> peut ensuite être utilisée pour créer et mettre à jour les branches de suivi à distance <name>/<branch>.

Afficher plus d’informations sur le référentiel distant #

Vous pouvez afficher plus d’informations sur un référentiel distant par git remote show <remote repository alias>

git remote show origin

résultat:

remote origin
Fetch URL:  https://localserver/develop/myrepo.git
Push  URL:  https://localserver/develop/myrepo.git
HEAD branch: master
Remote branches:
  master      tracked
Local branches configured for 'git pull':
  master      merges with remote master
Local refs configured for 'git push':
  master      pushes to master      (up to date)

Renommer un référentiel distant #

Renommez la télécommande nommée <old> en <new>. Toutes les branches de suivi à distance et les paramètres de configuration de la télécommande sont mis à jour.

Pour renommer une branche distante dev en dev1 :

git remote rename dev dev1
