Syntaxe #

  • branche git [--set-upstream | --track | --no-track] [-l] [-f] <nom de la branche> [<point de départ>]
  • git branch (--set-upstream-to=<upstream> | -u <upstream>) [<branchname>]
  • git branch --unset-upstream [<nom de la branche>]
  • git branch (-m | -M) [<anciennebranche>] <nouvellebranche>
  • git branch (-d | -D) [-r] <nom de la branche>…​
  • git branch --edit-description [<nom de la branche>]
  • branche git [--color[=<quand>] | --no-color] [-r | -a] [--list] [-v [--abbrev=<longueur> | --no-abbrev]] [--column[=<options>] | --no-column] [(--merged | --no-merged | --contains) [<commit>]] [--sort=<key>] [--points-at <object>] [<pattern >…​]

Paramètres #

Paramètre Détails
-d, –delete Supprimer une branche. La branche doit être entièrement fusionnée dans sa branche amont, ou dans HEAD si aucun amont n’a été défini avec --track ou --set-upstream
-D Raccourci pour --delete --force
-m, –move Déplacer/renommer une branche et le reflog correspondant
-M Raccourci pour --move --force
-r, –remotes Lister ou supprimer (si utilisé avec -d) les branches de suivi à distance
-a, –all Répertoriez à la fois les succursales de suivi à distance et les succursales locales
–liste Activez le mode liste. git branch <pattern> essayerait de créer une branche, utilisez git branch --list <pattern> pour lister les branches correspondantes
–set-amont Si la branche spécifiée n’existe pas encore ou si --force a été donné, agit exactement comme --track. Sinon, configurez la configuration comme le ferait –track lors de la création de la branche, sauf que l’endroit où pointe la branche n’est pas modifié

Chaque dépôt git a une ou plusieurs branches. Une branche est une référence nommée au HEAD d’une séquence de commits.

Un dépôt git a une branche current (indiquée par un * dans la liste des noms de branches imprimée par la commande git branch). Chaque fois que vous créez un nouveau commit avec la commande git commit, votre nouveau commit devient le HEAD de la branche actuelle, et le HEAD précédent devient le parent du nouveau commit.

Une nouvelle branche aura le même HEAD que la branche à partir de laquelle elle a été créée jusqu’à ce que quelque chose soit validé dans la nouvelle branche.

Créer et vérifier de nouvelles branches #

Pour créer une nouvelle branche, tout en restant sur la branche courante, utilisez :

git branch <name>

En règle générale, le nom de la branche ne doit pas contenir d’espaces et est soumis à d’autres spécifications répertoriées [ici] (http://stackoverflow.com/questions/3651860/which-characters-are-illegal-within-a-branch-name). Pour basculer vers une branche existante :

git checkout <name>

Pour créer une nouvelle branche et y basculer :

git checkout -b <name>

Pour créer une branche à un point autre que le dernier commit de la branche actuelle (également appelé HEAD), utilisez l’une de ces commandes :

git branch <name> [<start-point>]
git checkout -b <name> [<start-point>]

Le <start-point> peut être n’importe quelle révision connue de git (par exemple, un autre nom de branche, commit SHA ou une référence symbolique telle que HEAD ou un nom de la balise) :

git checkout -b <name> some_other_branch
git checkout -b <name> af295
git checkout -b <name> HEAD~5
git checkout -b <name> v1.0.5

Pour créer une branche à partir d’une [branche distante][1] (la valeur par défaut <remote_name> est origin) :

git branch <name> <remote_name>/<branch_name>
git checkout -b <name> <remote_name>/<branch_name>

Si un nom de branche donné n’est trouvé que sur une seule télécommande, vous pouvez simplement utiliser

git checkout -b <branch_name>

qui équivaut à

git checkout -b <branch_name> <remote_name>/<branch_name>

Parfois, vous devrez peut-être déplacer plusieurs de vos commits récents vers une nouvelle branche. Ceci peut être réalisé en se ramifiant et en “revenant en arrière”, comme ceci :

git branch <new_name>
git reset --hard HEAD~2 # Go back 2 commits, you will lose uncommitted work.
git checkout <new_name>

Voici une explication illustrative de cette technique :

 Initial state       After git branch <new_name>    After git reset --hard HEAD~2
                             newBranch                        newBranch
                                 ↓                                ↓
A-B-C-D-E (HEAD)         A-B-C-D-E (HEAD)                 A-B-C-D-E (HEAD)
        ↑                        ↑                            ↑
      master                   master                       master

[1] : https://www.wikiod.com/fr/git/travailler-avec-des-telecommandes

Liste des succursales #

Git fournit plusieurs commandes pour lister les branches. Toutes les commandes utilisent la fonction de git branch, qui fournira une liste de certaines branches, en fonction des options mises sur la ligne de commande. Git indiquera si possible la branche actuellement sélectionnée avec une étoile à côté.

Objectif Commande
Liste des succursales locales branche git
Lister les branches locales en mode verbeux git branch -v
Liste des succursales distantes et locales git branch -a OU git branch --all
Lister les branches distantes et locales (verbeux) git branch -av
Liste des succursales distantes git branch -r
Lister les branches distantes avec le dernier commit git branch -rv
Liste des succursales fusionnées git branch --merged
Liste des succursales non fusionnées git branch --no-merged
Lister les branches contenant commit git branch --contains [<commit>]

Remarques:

  • Ajout d’un v supplémentaire à -v par ex. $ git branch -avv ou $ git branch -vv affichera également le nom de la branche en amont.
  • Les succursales affichées en rouge sont des succursales distantes

Supprimer une branche distante #

Pour supprimer une branche sur le référentiel distant origin, vous pouvez utiliser pour Git version 1.5.0 et plus récente

git push origin :<branchName>

et à partir de la version 1.7.0 de Git, vous pouvez supprimer une branche distante en utilisant

git push origin --delete <branchName>

Pour supprimer une branche locale de suivi à distance :

git branch --delete --remotes <remote>/<branch>
git branch -dr <remote>/<branch> # Shorter

git fetch <remote> --prune # Delete multiple obsolete tracking branches
git fetch <remote> -p      # Shorter

Pour supprimer une branche localement. Notez que cela ne supprimera pas la branche si elle contient des modifications non fusionnées :

git branch -d <branchName>

Pour supprimer une branche, même si elle contient des modifications non fusionnées :

git branch -D <branchName>

Basculement rapide vers la branche précédente #

Vous pouvez passer rapidement à la branche précédente en utilisant

git checkout -

Découvrez une nouvelle succursale en suivant une succursale distante #

Il existe trois façons de créer une nouvelle branche “fonctionnalité” qui suit la branche distante “origine/fonctionnalité” :

  • git checkout --track -b feature origin/feature,
  • git checkout -t origine/fonctionnalité,
  • git checkout feature - en supposant qu’il n’y a pas de branche feature locale et qu’il n’y a qu’une seule télécommande avec la branche feature.

Pour configurer l’amont pour suivre la branche distante - tapez :

  • git branch --set-upstream-to=<distant>/<branch> <branch>
  • git branch -u <distant>/<branche> <branche>

où:

  • <remote> peut être : origin, develop ou celui créé par l’utilisateur,
  • <branch> est la branche de l’utilisateur à suivre à distance.

Pour vérifier quelles succursales distantes vos succursales locales suivent :

  • branche git -vv

Supprimer une branche localement #

$ git branch -d dev

Supprime la branche nommée dev si ses modifications sont fusionnées avec une autre branche et ne seront pas perdues. Si la branche dev contient des modifications qui n’ont pas encore été fusionnées et qui seraient perdues, git branch -d échouera :

$ git branch -d dev
error: The branch 'dev' is not fully merged.
If you are sure you want to delete it, run 'git branch -D dev'.

Selon le message d’avertissement, vous pouvez forcer la suppression de la branche (et perdre toutes les modifications non fusionnées dans cette branche) en utilisant le drapeau -D :

$ git branch -D dev

Créer une branche orpheline (c’est-à-dire une branche sans engagement parent) #

git checkout --orphan new-orphan-branch

Le premier commit fait sur cette nouvelle branche n’aura pas de parents et sera la racine d’une nouvelle histoire totalement déconnectée de toutes les autres branches et commits.

Renommer une branche #

Renommez la branche que vous avez extraite :

git branch -m new_branch_name

Renommer une autre branche :

git branch -m branch_you_want_to_rename new_branch_name

Pousser la branche vers la télécommande #

Utilisez pour pousser les commits effectués sur votre branche locale vers un référentiel distant.

La commande git push prend deux arguments :

- Un nom distant, par exemple “origine”

  • Un nom de branche, par exemple, ‘maître’

Par exemple:

git push  <REMOTENAME> <BRANCHNAME>

Par exemple, vous exécutez généralement git push origin master pour pousser vos modifications locales vers votre référentiel en ligne.

L’utilisation de -u (abréviation de --set-upstream) configurera les informations de suivi pendant le push.

git push -u <REMOTENAME> <BRANCHNAME>

Par défaut, git pousse la branche locale vers une branche distante portant le même nom. Par exemple, si vous avez une branche locale appelée new-feature, si vous poussez la branche locale, elle créera également une branche distante new-feature. Si vous souhaitez utiliser un nom différent pour la branche distante, ajoutez le nom distant après le nom de la branche locale, séparé par : :

git push <REMOTENAME> <LOCALBRANCHNAME>:<REMOTEBRANCHNAME>

Recherche dans les succursales #

Pour répertorier les branches locales contenant un commit ou une balise spécifique

git branch --contains <commit>

Pour répertorier les branches locales et distantes contenant un commit ou une balise spécifique

git branch -a --contains <commit>

Déplacez la branche actuelle HEAD vers un commit arbitraire #

Une branche n’est qu’un pointeur vers un commit, vous pouvez donc la déplacer librement. Pour faire en sorte que la branche fasse référence au commit aabbcc, lancez la commande

git reset --hard aabbcc

Veuillez noter que cela écrasera le commit actuel de votre branche et, par conséquent, tout son historique. Vous pourriez perdre du travail en exécutant cette commande. Si tel est le cas, vous pouvez utiliser le [reflog][1] pour récupérer les commits perdus. Il peut être conseillé d’effectuer cette commande sur une nouvelle branche au lieu de votre branche actuelle.

Cependant, cette commande peut être particulièrement utile lors d’un changement de base ou d’autres modifications importantes de l’historique.

[1] : https://www.wikiod.com/fr/git/reflog—restauration-des-commits-non-affiches-dans-le-journal-git

Écraser le fichier unique dans le répertoire de travail actuel avec le même d’une autre branche #

Le fichier extrait écrasera les modifications non encore validées que vous avez apportées à ce fichier.

Cette commande extraira le fichier file.example (qui se trouve dans le répertoire path/to/) et écrasera toutes les modifications que vous pourriez avoir apportées à ce fichier.

git checkout some-branch path/to/file

some-branch peut être n’importe quoi tree-ish connu de git (voir Revision Selection et gitrevisions pour plus de détails)

Vous devez ajouter -- avant le chemin si votre fichier peut être confondu avec un fichier (facultatif sinon). Aucune autre option ne peut être fournie après le --.

git checkout some-branch -- some-file

Le deuxième “some-file” est un fichier dans cet exemple.
