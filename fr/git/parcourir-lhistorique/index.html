Syntaxe #

  • git log [options] [plage de révision] [[–] chemin…]

Paramètres #

Paramètre           Explication
-q, –quiet Silencieux, supprime la sortie diff
–source Affiche la source du commit
–use-mailmap Utiliser le fichier de carte de messagerie (modifie les informations de l’utilisateur pour l’engagement de l’utilisateur)
–decorate[=…] Options de décoration
–L <n,m:fichier> Afficher le journal pour une plage spécifique de lignes dans un fichier, en comptant à partir de 1. Commence à partir de la ligne n, va à la ligne m. Montre également diff.
–show-signature Afficher les signatures des commits signés
-i, –regexp-ignore-case Faites correspondre les modèles de limitation d’expression régulière sans tenir compte de la casse des lettres

Références et documentation à jour : [documentation officielle de git-log][1]

[1] : https://git-scm.com/docs/git-log

Journal plus joli #

Pour voir le journal dans une structure graphique plus jolie, utilisez :

git log --decorate --oneline --graph

exemple de sortie :

* e0c1cea (HEAD -> maint, tag: v2.9.3, origin/maint) Git 2.9.3
*   9b601ea Merge branch 'jk/difftool-in-subdir' into maint
|\  
| * 32b8c58 difftool: use Git::* functions instead of passing around state
| * 98f917e difftool: avoid $GIT_DIR and $GIT_WORK_TREE
| * 9ec26e7 difftool: fix argument handling in subdirs
* |   f4fd627 Merge branch 'jk/reset-ident-time-per-commit' into maint
...

Comme il s’agit d’une commande assez volumineuse, vous pouvez lui attribuer un alias :

git config --global alias.lol "log --decorate --oneline --graph"

Pour utiliser la version d’alias :

# history of current branch :
git lol

# combined history of active branch (HEAD), develop and origin/master branches :
git lol HEAD develop origin/master

# combined history of everything in your repo :
git lol --all

Journal Git “normal” #

git log

affichera tous vos commits avec l’auteur et le hachage. Cela sera affiché sur plusieurs lignes par commit. (Si vous souhaitez afficher une seule ligne par commit, regardez [onelineing][1]). Utilisez la touche q pour quitter le journal.

Par défaut, sans arguments, git log liste les commits effectués dans ce référentiel dans l’ordre chronologique inverse, c’est-à-dire le plus récent les commits apparaissent en premier. Comme vous pouvez le voir, cette commande répertorie chaque commit avec sa somme de contrôle SHA-1, le nom et l’email de l’auteur, la date écrit, et le message de validation. - source

Si vous souhaitez limiter votre commande au dernier n commits log, vous pouvez simplement passer un paramètre. Par exemple, si vous souhaitez répertorier les 2 derniers journaux de commits

git log -2

Journal en ligne #

git log --oneline

affichera tous vos commits avec seulement la première partie du hachage et le message de commit. Chaque commit sera sur une seule ligne, comme le suggère le drapeau oneline.

L’option oneline imprime chaque commit sur une seule ligne, ce qui est utile si vous regardez beaucoup de commits. - source

Exemple (du référentiel Free Code Camp, avec la même section de code de l’autre exemple) :

Si vous souhaitez limiter votre commande au dernier n commits log, vous pouvez simplement passer un paramètre. Par exemple, si vous souhaitez répertorier les 2 derniers journaux de commits

git log -2 --oneline

Coloriser les journaux #

git log --graph --pretty=format:'%C(red)%h%Creset -%C(yellow)%d%Creset %s %C(green)(%cr) %C(yellow)<%an>%Creset'

L’option format vous permet de spécifier votre propre format de sortie de journal :

Paramètre Détails
%C(nom_couleur) l’option colore la sortie qui la suit
%h ou %H raccourcit le hash de commit (utilisez %H pour le hash complet)
%Creset réinitialise la couleur à la couleur par défaut du terminal
%d noms de référence
%s sujet [message de validation]
%cr date de validation, par rapport à la date actuelle
%an nom de l’auteur

Recherche de journaux #

git log -S"#define SAMPLES"

Recherche ajout ou suppression d’une chaîne spécifique ou de la chaîne correspondante fournie REGEXP. Dans ce cas, nous recherchons l’ajout/la suppression de la chaîne #define SAMPLES. Par exemple:

+#define SAMPLES  100000

ou

-#define SAMPLES  100000

git log -G"#define SAMPLES"

Recherche les modifications dans les lignes contenant une chaîne spécifique ou la chaîne correspondante fournie REGEXP. Par exemple:

-#define SAMPLES  100000
+#define SAMPLES  100000000

Liste toutes les contributions regroupées par nom d’auteur #

git shortlog résume git log et regroupe par auteur

Si aucun paramètre n’est donné, une liste de tous les commits effectués par committer sera affichée dans l’ordre chronologique.

$ git shortlog
Committer 1 (<number_of_commits>):
    Commit Message 1
    Commit Message 2
    ...
Committer 2 (<number_of_commits>):
    Commit Message 1
    Commit Message 2
    ...

Pour voir simplement le nombre de commits et supprimer la description du commit, passez l’option summary :

-s

--résumé

$ git shortlog -s
<number_of_commits> Committer 1
<number_of_commits> Committer 2

Pour trier la sortie par nombre de commits plutôt que par ordre alphabétique par nom de committer, passez l’option numérotée :

-n

--numéroté

Pour ajouter l’e-mail d’un committer, ajoutez l’option email :

-e

--courriel

Une option de format personnalisé peut également être fournie si vous souhaitez afficher des informations autres que le sujet du commit :

--format

Cela peut être n’importe quelle chaîne acceptée par l’option --format de git log.

Voir [Coloriser les journaux][1] ci-dessus pour plus d’informations à ce sujet.

[1] : https://www.wikiod.com/fr/git/parcourir-lhistorique#Coloriser les journaux

Recherche de la chaîne de validation dans le journal git #

Recherche de git log en utilisant une chaîne dans log :

git log [options] --grep "search_string"

Exemple:

git log --all --grep "removed file"

Recherche la chaîne fichier supprimé dans tous les journaux dans toutes les branches.

À partir de git 2.4+, la recherche peut être inversée en utilisant l’option --invert-grep.

Exemple:

git log --grep="add file" --invert-grep

Affichera tous les commits qui ne contiennent pas add file.

Filtrer les journaux #

git log --after '3 days ago'

Les dates spécifiques fonctionnent aussi :

git log --after 2016-05-01

Comme avec d’autres commandes et drapeaux qui acceptent un paramètre de date, le format de date autorisé est celui pris en charge par la date GNU (très flexible).

Un alias pour --after est --since.

Des drapeaux existent aussi pour l’inverse : --before et --until.

Vous pouvez également filtrer les journaux par “auteur”. par exemple.

git log --author=author
