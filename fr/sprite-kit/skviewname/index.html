Paramètres #

Paramètre Détails
montreFPS Affiche un décompte de la fréquence d’images actuelle en images par seconde dans la vue.
afficheNodeCount Afficher un décompte du nombre actuel de SKNodes affichés dans la vue.
montrePhysique Affichez une représentation visuelle des SKPhysicsBodys dans la vue.
montreChamps Affichez une image représentant les effets des champs physiques dans la vue.
afficheDrawCount Affiche le nombre de passes de dessin nécessaires pour rendre la vue.
montreQuadCount Affiche le nombre de rectangles requis pour rendre la vue.

Un SKView est une sous-classe de UIView qui est utilisée pour présenter les animations SpriteKit 2D.

Un SKView peut être ajouté à Interface Builder ou par programmation de la même manière que les UIViews “normaux”. Le contenu de SpriteKit est ensuite présenté dans le SKView dans un SKScene.

Voir aussi [SKView Class Reference][1] de la documentation Apple.

[1] : https://developer.apple.com/library/ios/documentation/SpriteKit/Reference/SKView/

Créer un SKView plein écran à l’aide d’Interface Builder #

Un cas d’utilisation typique de SpriteKit est celui où SKView remplit tout l’écran.

Pour ce faire dans Interface Builder de Xcode, créez d’abord un ViewController normal, puis sélectionnez la vue contenue et changez sa Class de UIView à SKView :

[![entrez la description de l’image ici][1]][1]

Dans le code du View Controller, dans la méthode viewDidLoad, saisissez un lien vers ce SKView en utilisant self.view :

En Swift :

guard let skView = self.view as? SKView else {
    // Handle error
    return
}

(La déclaration de garde ici protège contre l’erreur théorique selon laquelle la vue n’est pas une SKView.)

Vous pouvez ensuite l’utiliser pour effectuer d’autres opérations telles que la présentation d’une SKScene :

En Swift :

skView.presentScene(scene)

[1] : http://i.stack.imgur.com/wOgg1.png

Créez un petit SKView avec d’autres contrôles à l’aide d’Interface Builder #

Un SKView n’a pas besoin de remplir tout l’écran et peut partager l’espace avec d’autres contrôles de l’interface utilisateur. Vous pouvez même afficher plusieurs SKView à la fois si vous le souhaitez.

Pour créer un SKView plus petit parmi d’autres contrôles avec Interface Builder, créez d’abord un ViewController normal, puis faites glisser et déposez une nouvelle vue sur le contrôleur de vue :

[![Glisser et déposer une nouvelle vue][1]][1]

Il peut être utile de définir la couleur de cette vue sur autre chose que le blanc (ici, le noir est utilisé) afin qu’elle puisse être vue plus clairement dans Interface Builder (cette couleur ne sera pas affichée sur l’application finale). Ajoutez d’autres contrôles (un UIView, deux boutons et une étiquette sont présentés ici à titre d’exemples) et utilisez les contraintes comme d’habitude pour les disposer sur l’écran :

[![Changer de couleur, ajouter des contrôles, ajouter des contraintes][2]][2]

Sélectionnez ensuite la vue que vous souhaitez utiliser comme SKView et changez sa classe en SKView :

[![Changer la classe en SKView][3]][3]

Ensuite, à l’aide de l’assistant d’édition, faites glisser le contrôle de ce SKView vers votre code et créez un Outlet :

[![Créer un point de vente][4]][4]

Utilisez cette prise pour présenter votre SKScene.

En Swift :

skView.presentScene(scene)

Résultat (basé sur l’exemple [Hello World][5]) :

[![entrez la description de l’image ici][6]][6]

Affichage des informations de débogage #

La fréquence d’images actuelle (en FPS, images par seconde) et le nombre total de SKNodes dans la scène (nodeCount, chaque sprite est un SKNode mais les autres objets de la scène sont également des SKNodes) peuvent être affichés dans le coin inférieur droit de la vue .

Ceux-ci peuvent être utiles lorsqu’ils sont activés (définis sur true) pour le débogage et l’optimisation de votre code, mais doivent être désactivés (définis sur false) avant de soumettre l’application à l’AppStore.

En Swift :

skView.showsFPS = true
skView.showsNodeCount = true
