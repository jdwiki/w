Installation ou configuration #

Obtenir zsh #

zsh est disponible sur de nombreuses plates-formes de type UNIX via leurs systèmes de gestion de packages intégrés. Sur les distributions Debian et Ubuntu Linux, zsh est disponible dans les référentiels de packages par défaut et peut être installé à l’aide de :

$ sudo apt-get install zsh
# or, on newer Ubuntu distributions
$ sudo apt install zsh

Sur les distributions basées sur RPM, zsh est également souvent disponible dans les archives de paquets par défaut et peut être installé en utilisant :

$ yum install zsh

Sur Fedora 22 et versions ultérieures :

$ dnf install zsh

Sur les systèmes BSD, zsh peut être installé à l’aide de pkg :

$ pkg install zsh

Sur OpenBSD, zsh peut être installé en utilisant pkg_add :

$ pkg_add zsh

Sur Arch Linux, zsh peut être installé à l’aide de pacman :

$ pacman -S zsh

Sur openSUSE, zsh peut être installé à l’aide de zypper :

$ zypper install zsh

Sur les systèmes exécutant macOS (OS X), zsh est déjà installé par défaut, bien qu’il ne soit pas défini comme shell par défaut. Vous pouvez également installer des versions plus récentes via Homebrew :

$ brew install zsh

Alternativement, le code source de zsh peut être obtenu sur le site officiel.

À partir de là, le shell peut être démarré en tapant zsh à l’invite.

Faire de zsh votre shell par défaut #

Sur la plupart des systèmes Linux et BSD, zsh peut être défini comme shell par défaut pour un utilisateur utilisant la commande chsh :

$ chsh -s shell [username]

Où

  • username est un vrai nom d’utilisateur (par défaut, l’utilisateur actuel s’il est omis)
  • shell est le chemin vers le binaire zsh. Le chemin doit être répertorié dans le fichier /etc/shells, qui contient une liste des shells autorisés à être utilisés avec chsh. Si zsh n’y figure pas - par exemple parce que vous l’avez compilé et installé à partir des sources - vous devrez d’abord ajouter une ligne avec le chemin absolu vers zsh. Vous pouvez obtenir ce chemin avec which zsh (à condition qu’il soit installé dans un répertoire répertorié dans PATH)

Pour voir les modifications, déconnectez-vous une fois et connectez-vous. Ouvrez l’émulateur de terminal et utilisez

`echo $SHELL`

S’il affiche /bin/zsh, vous avez réussi à changer le shell par défaut en zsh.

Configuration #

Au démarrage de Zsh, il sourcera les fichiers suivants dans cet ordre par défaut :

  1. /etc/zsh/zshenv Utilisé pour définir les variables d’environnement à l’échelle du système ; il ne doit pas contenir de commandes produisant une sortie ou supposant que le shell est attaché à un tty. Ce fichier sera toujours sourcé, cela ne peut pas être remplacé.

  2. $ZDOTDIR/.zshenv Utilisé pour définir les variables d’environnement de l’utilisateur ; il ne doit pas contenir de commandes produisant une sortie ou supposant que le shell est attaché à un tty. Ce fichier sera toujours sourcé.

  3. /etc/zsh/zprofile Utilisé pour exécuter des commandes au démarrage, sera sourcé au démarrage en tant que shell de connexion.

Notez que sur Arch Linux, par défaut, il contient une ligne qui source le /etc/profile.

/etc/profile Ce fichier doit provenir de tous les shells compatibles Bourne lors de la connexion : il configure $PATH et d’autres variables d’environnement et paramètres spécifiques à l’application (/etc/profile.d/*.sh) lors de la connexion. connexion.

  1. $ZDOTDIR/.zprofile Utilisé pour exécuter les commandes de l’utilisateur au démarrage, sera sourcé au démarrage en tant que shell de connexion.

  2. /etc/zsh/zshrc Utilisé pour définir la configuration du shell interactif et exécuter des commandes, sera sourcé lors du démarrage en tant que shell interactif.

  3. $ZDOTDIR/.zshrc Utilisé pour définir la configuration du shell interactif de l’utilisateur et exécuter des commandes, sera sourcé lors du démarrage en tant que shell interactif.

  4. /etc/zsh/zlogin Utilisé pour exécuter des commandes à la fin de la progression initiale, sera sourcé au démarrage en tant que shell de connexion.

  5. $ZDOTDIR/.zlogin Utilisé pour exécuter les commandes de l’utilisateur à la fin de la progression initiale, sera sourcé au démarrage en tant que shell de connexion.

  6. $ZDOTDIR/.zlogout Sera sourcé à la sortie d’un shell de connexion.

  7. /etc/zsh/zlogout Sera sourcé à la sortie d’un shell de connexion.

Si $ZDOTDIR n’est pas défini, $HOME est utilisé à la place.

Pour un usage personnel général, il est courant de modifier le fichier .zshrc de l’utilisateur pour les préférences personnelles

Alias #

Pour aliaser une commande dans votre fichier ~/.zshrc, vous pouvez utiliser la syntaxe suivante :

alias [alias-name]="[command-to-execute]"

Par exemple, il est courant d’exécuter la commande ls -a. Vous pouvez alias cette commande comme la comme tel :

alias la="ls -a"

Après avoir rechargé le fichier ~/.zshrc, vous pourrez taper la et ls -a sera exécuté.

Alias ​​de répertoire #

Il est courant d’avoir certains dossiers dans lesquels vous vous connectez souvent. Si tel est le cas, vous pouvez créer des alias vers ces répertoires pour faciliter leur accès. Par exemple, ce qui suit sera un alias pour le dossier Dropbox :

alias db="cd ~/Dropbox"

vous permettant d’entrer db et de changer de répertoire en ~/Dropbox.

Recharger la configuration ZSH #

zsh charge la configuration à partir du fichier ~/.zshrc au démarrage. Si vous apportez des modifications à ce fichier, vous pouvez soit redémarrer zsh, soit exécuter la commande suivante pour recharger la configuration.

. ~/.zshrc

Vous pouvez créer un alias pour cette commande utile dans votre ~/.zshrc comme ceci :

alias reload=". ~/.zshrc"
