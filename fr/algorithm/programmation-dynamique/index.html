La programmation dynamique est un concept largement utilisé et souvent utilisé pour l’optimisation. Il s’agit de simplifier un problème compliqué en le décomposant en sous-problèmes plus simples d’une manière récursive, généralement une approche ascendante. Il y a deux attributs clés qu’un problème doit avoir pour que la programmation dynamique soit applicable “Sous-structure optimale” et “Sous-problèmes superposés”. Pour réaliser son optimisation, la programmation dynamique utilise un concept appelé Mémorisation

La programmation dynamique est une amélioration de Brute Force, voir [cet exemple][1] pour comprendre comment on peut obtenir une solution de programmation dynamique de Brute Force.

Une solution de programmation dynamique a 2 exigences principales :

  1. Problèmes de chevauchement

  2. Sous-structure optimale

*** Sous-problèmes qui se chevauchent *** signifie que les résultats de versions plus petites du problème sont réutilisés plusieurs fois afin d’arriver à la solution au problème d’origine

Sous-structure optimale signifie qu’il existe une méthode de calcul d’un problème à partir de ses sous-problèmes.

Une solution de programmation dynamique comporte 2 composants principaux, l’État et la Transition

L’État fait référence à un sous-problème du problème d’origine.

La Transition est la méthode pour résoudre un problème basé sur ses sous-problèmes

Le temps pris par une solution de programmation dynamique peut être calculé comme No. d'États * Temps de transition. Ainsi, si une solution a N^2 états et que la transition est O(N), alors la solution prendrait environ O(N^3) temps.

[1] : https://www.wikiod.com/fr/algorithm/vendeur-ambulant

Modifier la distance #

L’énoncé du problème est comme si on nous donnait deux chaînes str1 et str2, alors combien d’opérations minimales peuvent être effectuées sur le str1 qu’il est converti en str2.

Implémentation en Java

public class EditDistance {

public static void main(String[] args) {
    // TODO Auto-generated method stub
    String str1 = "march";
    String str2 = "cart";
    
    EditDistance ed = new EditDistance();
    System.out.println(ed.getMinConversions(str1, str2));
}

public int getMinConversions(String str1, String str2){
    int dp[][] = new int[str1.length()+1][str2.length()+1];
    for(int i=0;i<=str1.length();i++){
        for(int j=0;j<=str2.length();j++){
            if(i==0)
                dp[i][j] = j;
            else if(j==0)
                dp[i][j] = i;
            else if(str1.charAt(i-1) == str2.charAt(j-1))
                dp[i][j] = dp[i-1][j-1];
            else{
                dp[i][j] = 1 + Math.min(dp[i-1][j], Math.min(dp[i][j-1], dp[i-1][j-1]));
            }
        }
    }
    return dp[str1.length()][str2.length()];
}

}

Production

‘3’

Algorithme de planification pondérée des tâches #

L’algorithme de planification pondérée des tâches peut également être appelé algorithme de sélection d’activité pondérée.

Le problème est que, étant donné certains travaux avec leur heure de début et leur heure de fin, et un profit que vous réalisez lorsque vous terminez le travail, quel est le profit maximum que vous pouvez réaliser étant donné que deux travaux ne peuvent pas être exécutés en parallèle ?

Celui-ci ressemble à la sélection d’activité utilisant l’algorithme Greedy, mais il y a une torsion supplémentaire. Autrement dit, au lieu de maximiser le nombre de travaux terminés, nous nous concentrons sur la réalisation du profit maximum. Le nombre de travaux effectués n’a pas d’importance ici.

Regardons un exemple :

+-------------------------+---------+---------+---------+---------+---------+---------+
|          Name           |    A    |    B    |    C    |    D    |    E    |    F    |
+-------------------------+---------+---------+---------+---------+---------+---------+
|(Start Time, Finish Time)|  (2,5)  |  (6,7)  |  (7,9)  |  (1,3)  |  (5,8)  |  (4,6)  |
+-------------------------+---------+---------+---------+---------+---------+---------+
|         Profit          |    6    |    4    |    2    |    5    |    11   |    5    |
+-------------------------+---------+---------+---------+---------+---------+---------+

Les travaux sont désignés par un nom, leur heure de début et de fin et leur profit. Après quelques itérations, nous pouvons savoir si nous effectuons Job-A et Job-E, nous pouvons obtenir le profit maximum de 17. Maintenant, comment le savoir à l’aide d’un algorithme ?

La première chose que nous faisons est de trier les travaux en fonction de leur heure de fin dans un ordre non décroissant. Pourquoi faisons-nous cela? C’est parce que si nous sélectionnons un travail qui prend moins de temps à terminer, nous laissons le plus de temps pour choisir d’autres travaux. Nous avons:

+-------------------------+---------+---------+---------+---------+---------+---------+
|          Name           |    D    |    A    |    F    |    B    |    E    |    C    |
+-------------------------+---------+---------+---------+---------+---------+---------+
|(Start Time, Finish Time)|  (1,3)  |  (2,5)  |  (4,6)  |  (6,7)  |  (5,8)  |  (7,9)  |
+-------------------------+---------+---------+---------+---------+---------+---------+
|         Profit          |    5    |    6    |    5    |    4    |    11   |    2    |
+-------------------------+---------+---------+---------+---------+---------+---------+

Nous aurons un tableau temporaire supplémentaire Acc_Prof de taille n (ici, n indique le nombre total de travaux). Cela contiendra le profit maximum accumulé de l’exécution des travaux. Vous ne comprenez pas ? Attendez et regardez. Nous allons initialiser les valeurs du tableau avec le profit de chaque jobs. Cela signifie que Acc_Prof[i] détiendra d’abord le bénéfice de l’exécution du i-ème travail.

+-------------------------+---------+---------+---------+---------+---------+---------+
|         Acc_Prof        |    5    |    6    |    5    |    4    |    11   |    2    |
+-------------------------+---------+---------+---------+---------+---------+---------+

Désignons maintenant la position 2 par i, et la position 1 sera désignée par j. Notre stratégie sera d’itérer j de 1 à i-1 et après chaque itération, nous incrémenterons i de 1, jusqu’à ce que i devienne ** n+1**.

                               j        i

+-------------------------+---------+---------+---------+---------+---------+---------+

+-------------------------+---------+---------+---------+---------+---------+---------+
|         Acc_Prof        |    5    |    6    |    5    |    4    |    11   |    2    |
+-------------------------+---------+---------+---------+---------+---------+---------+

Nous vérifions si Job[i] et Job[j] se chevauchent, c’est-à-dire si l’heure de fin de Job[j] est supérieure à Job[i ]’s start time, alors ces deux tâches ne peuvent pas être effectuées ensemble. Cependant, s’ils ne se chevauchent pas, nous vérifierons si Acc_Prof[j] + Profit[i] > Acc_Prof[i]. Si tel est le cas, nous mettrons à jour Acc_Prof[i] = Acc_Prof[j] + Profit[i]. C’est-à-dire:

if Job[j].finish_time <= Job[i].start_time
    if Acc_Prof[j] + Profit[i] > Acc_Prof[i]
        Acc_Prof[i] = Acc_Prof[j] + Profit[i]
    endif
endif

Ici Acc_Prof[j] + Profit[i] représente le profit cumulé de l’exécution de ces deux tâches ensemble. Vérifions-le pour notre exemple :

Ici Job[j] chevauche Job[i]. Donc, cela ne peut pas être fait ensemble. Puisque notre j est égal à i-1, nous incrémentons la valeur de i à i+1 soit 3. Et on fait j = 1.

Le montant total de profit que nous pouvons réaliser en choisissant ces deux emplois est : Acc_Prof[j] + Profit[i] = 5 + 5 = 10 qui est supérieur à Acc_Prof[i]. Nous mettons donc à jour Acc_Prof[i] = 10. Nous incrémentons également j de 1. Nous obtenons,

* Quelques étapes ont été sautées pour raccourcir le document.

Si nous parcourons le tableau Acc_Prof, nous pouvons déterminer que le profit maximum est de 17 ! Le pseudo-code :

Procedure WeightedJobScheduling(Job)
sort Job according to finish time in non-decreasing order
for i -> 2 to n
    for j -> 1 to i-1
        if Job[j].finish_time <= Job[i].start_time
            if Acc_Prof[j] + Profit[i] > Acc_Prof[i]
                Acc_Prof[i] = Acc_Prof[j] + Profit[i]
            endif
        endif
    endfor
endfor

maxProfit = 0
for i -> 1 to n
    if maxProfit < Acc_Prof[i]
        maxProfit = Acc_Prof[i]
return maxProfit

La complexité du remplissage du tableau Acc_Prof est O(n2). La traversée du tableau prend O(n). La complexité totale de cet algorithme est donc O(n2).

Maintenant, si nous voulons savoir quels travaux ont été effectués pour obtenir le profit maximum, nous devons parcourir le tableau dans l’ordre inverse et si le Acc_Prof correspond au maxProfit, nous pousserons le **name ** du travail dans une pile et soustrayez Profit de ce travail de maxProfit. Nous ferons cela jusqu’à ce que notre maxProfit > 0 ou que nous atteignions le point de départ du tableau Acc_Prof. Le pseudo-code ressemblera à :

Procedure FindingPerformedJobs(Job, Acc_Prof, maxProfit):
S = stack()
for i -> n down to 0 and maxProfit > 0
    if maxProfit is equal to Acc_Prof[i]
        S.push(Job[i].name
        maxProfit = maxProfit - Job[i].profit
    endif
endfor

La complexité de cette procédure est : O(n).

Une chose à retenir, s’il y a plusieurs horaires de travail qui peuvent nous donner un profit maximum, nous ne pouvons trouver qu’un seul horaire de travail via cette procédure.

Sous-séquence commune la plus longue #

Si on nous donne les deux chaînes, nous devons trouver la plus longue sous-séquence commune présente dans les deux.

Exemple

LCS pour les séquences d’entrée “ABCDGH” et “AEDFHR” est “ADH” de longueur 3.

LCS pour les séquences d’entrée “AGGTAB” et “GXTXAYB” est “GTAB” de longueur 4.

Implémentation en Java

public class LCS {

    public static void main(String[] args) {
        // TODO Auto-generated method stub
        String str1 = "AGGTAB";
        String str2 = "GXTXAYB";
        LCS obj = new LCS();
        System.out.println(obj.lcs(str1, str2, str1.length(), str2.length()));
        System.out.println(obj.lcs2(str1, str2));
    }
    
    //Recursive function
    public int lcs(String str1, String str2, int m, int n){
        if(m==0 || n==0)
            return 0;
        if(str1.charAt(m-1) == str2.charAt(n-1))
            return 1 + lcs(str1, str2, m-1, n-1);
        else
            return Math.max(lcs(str1, str2, m-1, n), lcs(str1, str2, m, n-1));
    }
    
    //Iterative function
    public int lcs2(String str1, String str2){
        int lcs[][] = new int[str1.length()+1][str2.length()+1];
        
        for(int i=0;i<=str1.length();i++){
            for(int j=0;j<=str2.length();j++){
                if(i==0 || j== 0){
                    lcs[i][j] = 0;
                }
                else if(str1.charAt(i-1) == str2.charAt(j-1)){
                    lcs[i][j] = 1 + lcs[i-1][j-1]; 
                }else{
                    lcs[i][j] = Math.max(lcs[i-1][j], lcs[i][j-1]);
                }
            }
        }
        
        return lcs[str1.length()][str2.length()];
    }

** Sous-problèmes qui se chevauchent **

Ici fib(0), fib(1) et fib(3) sont les sous-problèmes qui se chevauchent. fib(0) se répète 3 fois, fib(1) se répète 5 fois et fib(3) se répète 2 fois fois.

Mise en œuvre

public int fib(int n){
        int f[] = new int[n+1];
        f[0]=0;f[1]=1;
        for(int i=2;i<=n;i++){
            f[i]=f[i-1]+f[i-2];
        }
        return f[n];
    }

Sous-chaîne commune la plus longue #

Étant donné 2 chaînes str1 et str2, nous devons trouver la longueur de la plus longue sous-chaîne commune entre elles.

Exemples

Entrée : X = “abcdxyz”, y = “xyzabcd” Sortie : 4

La plus longue sous-chaîne commune est “abcd” et est de longueur 4.

Entrée : X = “zxabcdezy”, y = “yzabcdezx” Sortie : 6

La sous-chaîne commune la plus longue est “abcdez” et est de longueur 6.

Implémentation en Java

public int getLongestCommonSubstring(String str1,String str2){
        int arr[][] = new int[str2.length()+1][str1.length()+1];
        int max = Integer.MIN_VALUE;
        for(int i=1;i<=str2.length();i++){
            for(int j=1;j<=str1.length();j++){
                if(str1.charAt(j-1) == str2.charAt(i-1)){
                    arr[i][j] = arr[i-1][j-1]+1;
                    if(arr[i][j]>max)
                        max = arr[i][j];
                }
                else
                    arr[i][j] = 0;
            }
        }
        return max;
    }

Problème de sac à dos #

Sac à dos 0-1

Le problème du sac à dos est un problème lorsqu’on lui donne un ensemble d’éléments, chacun avec un poids, une valeur et exactement 1 copie, déterminez le ou les éléments à inclure dans une collection afin que le poids total soit inférieur ou égal à égale à une limite donnée et la valeur totale est aussi grande que possible.

Exemple C++ : #

Mise en œuvre:

int knapsack(vector<int> &value, vector<int> &weight, int N, int C){
    int dp[C+1];                               
    for (int i = 1; i <= C; ++i){
        dp[i] = -100000000;                   
    }
    dp[0] = 0;
    for (int i = 0; i < N; ++i){
        for (int j = C; j >= weight[i]; --j){
            dp[j] = max(dp[j],dp[j-weight[i]]+value[i]);
        }
    }
    return dp[C];
}

Test:

3 5
5 2
2 1
3 2

Production:

3

Cela signifie que la valeur maximale pouvant être atteinte est 3, ce qui est obtenu en choisissant (2,1) et (3,2).

Sac à dos illimité

Le problème du sac à dos illimité est un problème qui, étant donné un ensemble d’éléments, chacun avec un poids, une valeur et des copies infinies, détermine le nombre de chaque élément à inclure dans une collection de sorte que le poids total soit inférieur ou égal à une limite donnée et la valeur totale est aussi grande que possible.

Python(2.7.11) Exemple : #

Mise en œuvre:

def unbounded_knapsack(w, v, c): # weight, value and capactiy
    m = [0]
    for r in range(1, c+1):
        val = m[r-1]
        for i, wi in enumerate(w):
            if wi > r:
                continue
            val = max(val, v[i] + m[r-wi])
        m.append(val)
    return m[c] # return the maximum value can be achieved

La complexité de cette implémentation est O(nC), où n est le nombre d’éléments.

Test:

w = [2, 3, 4, 5, 6]
v = [2, 4, 6, 8, 9]

print unbounded_knapsack(w, v, 13)

Production:

20

Cela signifie que la valeur maximale pouvant être atteinte est de 20, ce qui est obtenu en choisissant (5, 8), (5, 8) et (3, 4).
