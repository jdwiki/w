Introduction à l’algorithme de Prim #

Disons que nous avons 8 maisons. Nous voulons installer des lignes téléphoniques entre ces maisons. Le bord entre les maisons représente le coût de la mise en ligne entre deux maisons.

[![Exemple de graphique][1]][1]

Notre tâche est de mettre en place des lignes de manière à ce que toutes les maisons soient connectées et que le coût de mise en place de l’ensemble de la connexion soit minimal. Maintenant, comment pouvons-nous le savoir? Nous pouvons utiliser l’algorithme de Prim.

L’algorithme de Prim est un algorithme gourmand qui trouve un arbre couvrant minimum pour un graphe non orienté pondéré. Cela signifie qu’il trouve un sous-ensemble des arêtes qui forme un arbre qui inclut chaque nœud, où le poids total de toutes les arêtes de l’arbre est minimisé. L’algorithme a été développé en 1930 par le mathématicien tchèque Vojtěch Jarník et plus tard redécouvert et republié par l’informaticien [Robert Clay Prim](https://en.wikipedia. org/wiki/Robert_C._Prim) en 1957 et Edsger Wybe Dijkstra en 1959. Il est également connu sous le nom algorithme DJP, ** Algorithme de Jarnik**, algorithme Prim-Jarnik ou algorithme Prim-Dijsktra.

Voyons d’abord les termes techniques. Si nous créons un graphe, S en utilisant certains nœuds et arêtes d’un graphe non orienté G, alors S est appelé un sous-graphe du graphe G. Maintenant S sera appelé un Spanning Tree si et seulement si :

  • Il contient tous les nœuds de G.
  • C’est un arbre, cela signifie qu’il n’y a pas de cycle et que tous les nœuds sont connectés.
  • Il y a (n-1) arêtes dans l’arbre, où n est le nombre de nœuds dans G.

Il peut y avoir plusieurs Spanning Tree d’un graphe. Le Minimum Spanning Tree d’un graphe pondéré non orienté est un arbre, tel que la somme des poids des arêtes soit minimale. Nous allons maintenant utiliser l’algorithme de Prim pour trouver l’arbre couvrant minimum, c’est-à-dire comment configurer les lignes téléphoniques dans notre exemple de graphique de manière à ce que le coût de configuration soit minimum.

Dans un premier temps, nous sélectionnerons un nœud source. Disons que node-1 est notre source. Nous allons maintenant ajouter l’arête du nœud-1 qui a le coût minimum à notre sous-graphe. Ici, nous marquons les arêtes qui se trouvent dans le sous-graphe en utilisant la couleur bleu. Ici 1-5 est notre avantage souhaité. [![1-5 sélectionné][2]][2]

Maintenant, nous considérons tous les bords de node-1 et node-5 et prenons le minimum. Puisque 1-5 est déjà marqué, nous prenons 1-2. [![1-2 sélectionné][3]][3]

Cette fois, nous considérons nœud-1, nœud-2 et nœud-5 et prenons l’arête minimale qui est 5-4. [![5-4 sélectionné][4]][4]

La prochaine étape est importante. À partir de nœud-1, nœud-2, nœud-5 et nœud-4, l’arête minimale est 2-4. Mais si nous sélectionnons celui-là, cela créera un cycle dans notre sous-graphe. C’est parce que nœud-2 et nœud-4 sont déjà dans notre sous-graphe. Donc, prendre l’avantage 2-4 ne nous profite pas. Nous sélectionnerons les arêtes de manière à ajouter un nouveau nœud dans notre sous-graphe. Nous sélectionnons donc l’arête 4-8. [![4-8 sélectionnés][5]][5]

Si nous continuons ainsi, nous sélectionnerons les arêtes 8-6, 6-7 et 4-3. Notre sous-graphe ressemblera à : [![reste des arêtes sélectionnées][6]][6]

Ceci est notre sous-graphe souhaité, qui nous donnera l’arbre couvrant minimum. Si nous supprimons les arêtes que nous n’avons pas sélectionnées, nous obtiendrons : [![Arbre couvrant minimal][7]][7]

Il s’agit de notre minimum spanning tree (MST). Ainsi, le coût d’établissement des connexions téléphoniques est : 4 + 2 + 5 + 11 + 9 + 2 + **1 ** = 34. Et l’ensemble des maisons et leurs connexions sont indiqués dans le graphique. Il peut y avoir plusieurs MST d’un graphique. Cela dépend du nœud source que nous choisissons.

Le pseudo-code de l’algorithme est donné ci-dessous :

Procedure PrimsMST(Graph):     // here Graph is a non-empty connected weighted graph
Vnew[] = {x}                   // New subgraph Vnew with source node x
Enew[] = {}
while Vnew is not equal to V
    u -> a node from Vnew
    v -> a node that is not in Vnew such that edge u-v has the minimum cost
                               // if two nodes have same weight, pick any of them
    add v to Vnew
    add edge (u, v) to Enew
end while
Return Vnew and Enew

Complexité:

La complexité temporelle de l’approche naïve ci-dessus est O(V²). Il utilise une matrice de contiguïté. Nous pouvons réduire la complexité en utilisant la file d’attente prioritaire. Lorsque nous ajoutons un nouveau nœud à Vnew, nous pouvons ajouter ses bords adjacents dans la file d’attente prioritaire. Ensuite, extrayez-en le bord pondéré minimum. Alors la complexité sera : O(ElogE), où E est le nombre d’arêtes. Encore une fois, un tas binaire peut être construit pour réduire la complexité à ** O (ElogV) **.

Le pseudo-code utilisant Priority Queue est donné ci-dessous :

Procedure MSTPrim(Graph, source):
for each u in V
    key[u] := inf
    parent[u] := NULL
end for
key[source] := 0
Q = Priority_Queue()
Q = V
while Q is not empty
    u -> Q.pop
    for each v adjacent to i
        if v belongs to Q and Edge(u,v) < key[v]    // here Edge(u, v) represents
                                                    // cost of edge(u, v)
            parent[v] := u
            key[v] := Edge(u, v)
        end if
    end for
end while

Ici, key[] stocke le coût minimum de traversée de node-v. parent[] est utilisé pour stocker le nœud parent. Il est utile pour parcourir et imprimer l’arbre.

Voici un programme simple en Java :

import java.util.*;

public class Graph
{
   private static int infinite = 9999999;
   int[][]  LinkCost;
   int      NNodes;
   Graph(int[][] mat)
   {
      int i, j;
      NNodes = mat.length;
      LinkCost = new int[NNodes][NNodes];
      for ( i=0; i < NNodes; i++)
      {
         for ( j=0; j < NNodes; j++)
         {
            LinkCost[i][j] = mat[i][j];
            if ( LinkCost[i][j] == 0 )
               LinkCost[i][j] = infinite;
         }
      }
      for ( i=0; i < NNodes; i++)
      {
         for ( j=0; j < NNodes; j++)
            if ( LinkCost[i][j] < infinite )
               System.out.print( " " + LinkCost[i][j] + " " );
            else
               System.out.print(" * " );
         System.out.println();
      }
   }
   public int unReached(boolean[] r)
   {
      boolean done = true;
      for ( int i = 0; i < r.length; i++ )
         if ( r[i] == false )
            return i;
      return -1;
   }
   public void Prim( )
   {
      int i, j, k, x, y;
      boolean[] Reached = new boolean[NNodes];
      int[] predNode = new int[NNodes];
      Reached[0] = true;
      for ( k = 1; k < NNodes; k++ )
      {
         Reached[k] = false;
      }
      predNode[0] = 0;
      printReachSet( Reached );
      for (k = 1; k < NNodes; k++)
      {
         x = y = 0;
         for ( i = 0; i < NNodes; i++ )
            for ( j = 0; j < NNodes; j++ )
            {
                if ( Reached[i] && !Reached[j] &&
                     LinkCost[i][j] < LinkCost[x][y] )
                {
                   x = i;
                   y = j;
                }
            }
         System.out.println("Min cost edge: (" +
                                + x + "," +
                                + y + ")" +
                                "cost = " + LinkCost[x][y]);
         predNode[y] = x;
         Reached[y] = true;
         printReachSet( Reached );
         System.out.println();
      }
      int[] a= predNode;
   for ( i = 0; i < NNodes; i++ )
          System.out.println( a[i] + " --> " + i );
   }
   void printReachSet(boolean[] Reached )
   {
      System.out.print("ReachSet = ");
      for (int i = 0; i < Reached.length; i++ )
         if ( Reached[i] )
           System.out.print( i + " ");
      //System.out.println();
   }
 public static void main(String[] args)
   {
      int[][] conn = {{0,3,0,2,0,0,0,0,4},  // 0
                      {3,0,0,0,0,0,0,4,0},  // 1
                      {0,0,0,6,0,1,0,2,0},  // 2
                      {2,0,6,0,1,0,0,0,0},  // 3
                      {0,0,0,1,0,0,0,0,8},  // 4
                      {0,0,1,0,0,0,8,0,0},  // 5
                      {0,0,0,0,0,8,0,0,0},  // 6
                      {0,4,2,0,0,0,0,0,0},  // 7
                      {4,0,0,0,8,0,0,0,0}   // 8
                     };
      Graph G = new Graph(conn);
      G.Prim();
   }
}

Compilez le code ci-dessus en utilisant javac Graph.java
