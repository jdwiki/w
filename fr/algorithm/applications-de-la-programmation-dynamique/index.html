Applications de la programmation dynamique

  • Définitions
  • Nombres de Fibonacci
  • Remarques

Sur cette page

  • Définitions
  • Nombres de Fibonacci
  • Remarques

L’idée de base derrière la programmation dynamique consiste à décomposer un problème complexe en plusieurs petits problèmes simples qui se répètent. Si vous pouvez identifier un sous-problème simple qui est calculé à plusieurs reprises, il y a de fortes chances qu’il existe une approche de programmation dynamique du problème.

Comme ce sujet est intitulé * Applications de la programmation dynamique *, il se concentrera davantage sur les applications plutôt que sur le processus de création d’algorithmes de programmation dynamique.

Définitions #

Memoization - une technique d’optimisation utilisée principalement pour accélérer les programmes informatiques en stockant les résultats des appels de fonction coûteux et en renvoyant le résultat mis en cache lorsque les mêmes entrées se reproduisent .

Programmation dynamique - une méthode pour résoudre un problème complexe en le décomposant en une collection de sous-problèmes plus simples, en résolvant chacun de ces sous-problèmes une seule fois et en stockant leurs solutions.

Nombres de Fibonacci #

Les nombres de Fibonacci sont un sujet primordial pour la programmation dynamique car l’approche récursive traditionnelle fait beaucoup de calculs répétés. Dans ces exemples, j’utiliserai le cas de base f(0) = f(1) = 1.

Voici un exemple d’arbre récursif pour fibonacci(4), notez les calculs répétés :[![arbre récursif de Fibonacci][1]][1]

Programmation non dynamique O(2^n) Complexité d’exécution, O(n) Complexité de la pile

def fibonacci(n):
    if n < 2:
        return 1
    return fibonacci(n-1) + fibonacci(n-2)

C’est la manière la plus intuitive d’écrire le problème. Au plus l’espace de la pile sera O(n) lorsque vous descendez la première branche récursive en faisant des appels à fibonacci(n-1) jusqu’à ce que vous atteigniez le cas de base n < 2.

La preuve de la complexité d’exécution O(2^n) qui peut être vue ici : https://stackoverflow.com/questions/360748/computational-complexity-of-fibonacci-sequence. Le point principal à noter est que le temps d’exécution est exponentiel, ce qui signifie que le temps d’exécution doublera pour chaque terme suivant, fibonacci (15) prendra deux fois plus longtemps que fibonacci (14).

Mémoisé Complexité d’exécution O(n), Complexité d’espace O(n), Complexité de pile O(n)

memo = []
memo.append(1) # f(1) = 1
memo.append(1) # f(2) = 1

def fibonacci(n):
    if len(memo) > n:
        return memo[n]

    result = fibonacci(n-1) + fibonacci(n-2)
    memo.append(result) # f(n) = f(n-1) + f(n-2)
    return result

Avec l’approche mémoïsée, nous introduisons un tableau qui peut être considéré comme tous les appels de fonction précédents. L’emplacement memo[n] est le résultat de l’appel de fonction fibonacci(n). Cela nous permet d’échanger la complexité spatiale de O(n) contre un temps d’exécution O(n) car nous n’avons plus besoin de calculer les appels de fonction en double.

Programmation dynamique itérative Complexité d’exécution O(n), Complexité d’espace O(n), Pas de pile récursive

def fibonacci(n):
    memo = [1,1] # f(0) = 1, f(1) = 1

    for i in range(2, n+1):
         memo.append(memo[i-1] + memo[i-2])

    return memo[n]

Si nous décomposons le problème en ses éléments de base, vous remarquerez que pour calculer fibonacci(n) nous avons besoin de fibonacci(n-1) et fibonacci(n-2). Nous pouvons également remarquer que notre cas de base apparaîtra à la fin de cet arbre récursif, comme indiqué ci-dessus.

Avec ces informations, il est maintenant logique de calculer la solution à rebours, en commençant par les cas de base et en remontant. Maintenant, pour calculer fibonacci(n), nous calculons d’abord tous les nombres de fibonacci jusqu’à n.

Ce principal avantage ici est que nous avons maintenant éliminé la pile récursive tout en conservant le temps d’exécution O(n). Malheureusement, nous avons toujours une complexité d’espace O(n) mais cela peut également être modifié.

Programmation dynamique itérative avancée Complexité d’exécution O(n), Complexité d’espace O(1), Pas de pile récursive

def fibonacci(n):
    memo = [1,1] # f(1) = 1, f(2) = 1

    for i in range (2, n):
        memo[i%2] = memo[0] + memo[1]

    return memo[n%2]

Comme indiqué ci-dessus, l’approche de programmation dynamique itérative part des cas de base et fonctionne jusqu’au résultat final. L’observation clé à faire pour arriver à la complexité de l’espace à O(1) (constante) est la même observation que nous avons faite pour la pile récursive - nous n’avons besoin que de fibonacci(n-1) et fibonacci(n -2) pour construire fibonacci(n). Cela signifie que nous n’avons qu’à enregistrer les résultats pour fibonacci(n-1) et fibonacci(n-2) à tout moment de notre itération.

Pour stocker ces 2 derniers résultats, j’utilise un tableau de taille 2 et retourne simplement l’index que j’attribue en utilisant i % 2 qui alternera comme suit : `0, 1, 0, 1, 0, 1, .. ., je % 2'.

J’additionne les deux indices du tableau ensemble car nous savons que l’addition est commutative (5 + 6 = 11 et 6 + 5 == 11). Le résultat est alors attribué au plus ancien des deux spots (noté ‘i % 2’). Le résultat final est alors stocké à la position n%2

Remarques #

  • Il est important de noter qu’il peut parfois être préférable de proposer une solution itérative mémorisée pour les fonctions qui effectuent de grandes calculs à plusieurs reprises car vous constituerez un cache de la réponse à les appels de fonction et les appels suivants peuvent être O(1) s’il a déjà été calculé.
