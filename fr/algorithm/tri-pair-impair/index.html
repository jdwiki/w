Impair-Pair Trier les informations de base #

Un [tri pair-impair] [1] ou tri par brique est un algorithme de tri simple, qui est développé pour être utilisé sur des processeurs parallèles avec une interconnexion locale. Cela fonctionne en comparant toutes les paires indexées paires/impaires d’éléments adjacents dans la liste et, si une paire est dans le mauvais ordre, les éléments sont inversés. L’étape suivante le répète pour les paires indexées paires/impaires. Ensuite, il alterne entre les étapes paires/impaires et paires/impaires jusqu’à ce que la liste soit triée.

Pseudo-code pour le tri pair-impair :

if n>2 then
    1. apply odd-even merge(n/2) recursively to the even subsequence a0, a2, ..., an-2 and to the odd subsequence a1, a3, , ..., an-1
    2. comparison [i : i+1] for all i element {1, 3, 5, 7, ..., n-3}
else
    comparison [0 : 1]

Mise en œuvre:

J’ai utilisé le langage C # pour implémenter l’algorithme de tri pair-impair.

public class OddEvenSort
{
    private static void SortOddEven(int[] input, int n)
    {
        var sort = false;

        while (!sort)
        {
            sort = true;
            for (var i = 1; i < n - 1; i += 2)
            {
                if (input[i] <= input[i + 1]) continue;
                var temp = input[i];
                input[i] = input[i + 1];
                input[i + 1] = temp;
                sort = false;
            }
            for (var i = 0; i < n - 1; i += 2)
            {
                if (input[i] <= input[i + 1]) continue;
                var temp = input[i];
                input[i] = input[i + 1];
                input[i + 1] = temp;
                sort = false;
            }
        }
    }

    public static int[] Main(int[] input)
    {
        SortOddEven(input, input.Length);
        return input;
    }
}

Espace auxiliaire : O(n)
Complexité temporelle : O(n)
