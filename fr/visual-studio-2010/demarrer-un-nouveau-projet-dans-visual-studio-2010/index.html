Démarrage d’un nouveau projet dans Visual Studio 2010 #

Pour démarrer un nouveau projet dans Visual Studio 2010 :

  • Cliquez sur le menu Fichier
  • Sélectionnez Projet

[![entrez la description de l’image ici][1]][1]

Ensuite, nous avons répertorié les types de projets et les langues disponibles :

  • Sélectionnez les langues disponibles dans le panneau le plus à gauche [Ci-dessous, nous avons des langues comme Visual C #, VB, F # etc.]
  • Une fois que nous avons sélectionné les langues à partir des modèles installés, nous avons répertorié les types de projets/applications.
  • Sélectionnez les types de projet, donnez un nom à votre projet et cliquez sur OK.

[![entrez la description de l’image ici][2]][2]

Quelques exemples courants de types de projets :

Applications Windows Forms :

Projet de création d’application avec l’interface utilisateur Windows Forms.

Applications WPF :

Application cliente Windows Presentation Foundation.

Applications consoles :

Applications en ligne de commande. Aucune interface utilisateur graphique (GUI).

ASP.NET :

Applications d’interface utilisateur Web, c’est-à-dire. des applications Web

Applications Silverlight :

Applications Internet riches utilisant Silverlight

Ouvrir un projet existant #

  • Ouvrez Visual Studio et sélectionnez Menu Fichier -> Ouvrir -> Projet

[![entrez la description de l’image ici][1]][1]

  • Ensuite, sélectionnez le fichier de solution dans le dossier du projet

[![entrez la description de l’image ici][2]][2]

  • Les fichiers de projet seront affichés dans Solution Explorer.

[1] : https://i.stack.imgur.com/gwL5v.png [2] : https://i.stack.imgur.com/STLS4.png

Microsoft Visual Studion 2010 Microsoft® Visual C#® Liaisons de touches par défaut #

Commandes de fichier

  • Fichier.NouveauProjet :

CTRL + MAJ + N

Affiche la boîte de dialogue Nouveau projet.

  • Fichier.OuvrirProjet :

CTRL + MAJ + O

Affiche la boîte de dialogue Ouvrir un projet, où les projets existants peuvent être ajouté à la solution.

  • Projet.AjouterClass :

MAJ + ALT + C

Affiche la boîte de dialogue Ajouter un nouvel élément et sélectionne le modèle de classe comme par défaut.

  • Projet.AddExistingItem :

MAJ + ALT + A

Affiche la boîte de dialogue Ajouter un élément existant, dans laquelle les fichiers existants être ajouté au projet en cours.

  • Projet.AddNewItem :

CTRL + MAJ + A

Affiche la boîte de dialogue Ajouter un nouvel élément, dans laquelle un nouveau fichier peut être ajouté à le projet en cours.

Montage

  • Modifier.Masquer la sélection :

CTRL + M , CTRL + H

Réduit la sélection actuelle à une zone de contour.

- Edit.ToggleAllOutlining :

CTRL + M , L

Bascule toutes les régions de contour précédemment réduites entre les régions réduites et états étendus.

- Edit.ToggleOutliningExpansion :

CTRL + M , M

Bascule la région réduite actuellement sélectionnée entre la région réduite et état expansé.

- Edit.StopOutlining :

CTRL + M , P

Supprime toutes les informations de contour de l’ensemble du document.

- Edit.CommentSelection :

CTRL + K , C ou CTRL + E , C

Insère // au début de la ligne courante ou à chaque ligne du sélection en cours.

Edit.UncommentSelectionEdit.UncommentSelection CTRL + K, U ou CTRL + E, U Supprime le // au début de la ligne courante ou chaque ligne du sélection actuelle.

  • Modifier.FormatDocument :

CTRL + K, D ou CTRL + E, D

Formate le document courant en fonction de l’indentation et du code paramètres de formatage spécifiés dans le volet Formatage sous : Outils | Options | Éditeur de texte | C#.

  • Modifier.SélectionFormat :

CTRL + K, F ou CTRL + E, F

Formate la sélection courante selon l’indentation et le code paramètres de formatage spécifiés dans le volet Formatage sous Outils | Options | Éditeur de texte | C#.

- Modifier.Insérer un extrait :

CTRL + K, X

Affiche le sélecteur d’extrait de code. L’extrait de code sélectionné sera inséré à la position du caret.

Modifier.SurroundWith :

CTRL + K, S

Affiche le sélecteur d’extrait de code. L’extrait de code sélectionné sera enveloppant le texte sélectionné.

  • Modifier.InvokeSnippetFromShortcut :

TAB

Insère l’extrait de code développé à partir du nom du raccourci.

  • Edit.CycleClipboardRing :

CTRL +MAJ +V

Colle le texte de l’anneau du Presse-papiers à l’emplacement du curseur dans le fichier. L’utilisation ultérieure de la touche de raccourci parcourt les éléments du Anneau presse-papiers.

  • Modifier.Remplacer :

CTRL + H

Affiche les options de remplacement dans l’onglet Remplacement rapide de la Boîte de dialogue Remplacer.

  • Modifier.RemplacerDansFichiers :

CTRL + MAJ + H

Affiche les options de remplacement dans l’onglet Remplacer dans les fichiers de la et la boîte de dialogue Remplacer.

Commandes de fenêtre :

  • Afficher.ClassView :

CTRL +W, C

Affiche la fenêtre d’affichage des classes.

  • Afficher.CodeDefinitionWindow :

CTRL + W, D

Affiche la fenêtre Définition du code.

  • Afficher.ErrorList :

CTRL + W, E

Affiche la fenêtre Liste des erreurs.

  • Afficher.ObjectBrowser :

CTRL + W, J

Affiche l’Explorateur d’objets.

  • Afficher.Sortie :

CTRL + W, O

Affiche la fenêtre de sortie, où les messages d’état peuvent être visualisés à temps d’exécution.

  • Afficher.FenêtrePropriétés :

CTRL + W, P

Affiche la fenêtre Propriétés, qui répertorie les propriétés de conception et les événements pour l’élément actuellement sélectionné.

  • Afficher.Explorateur de solutions :

CTRL + W, S

Affiche l’Explorateur de solutions, qui répertorie les projets et les ﬁchiers du résolution actuelle.

  • Afficher.Liste des tâches :

CTRL + W, T

Affiche la fenêtre Liste des tâches, qui affiche les tâches personnalisées, les commentaires, raccourcis, avertissements et messages d’erreur.

  • Afficher.Boîte à outils :

CTRL + W, X

Affiche la boîte à outils, qui contient des contrôles qui peuvent être inclus ou utilisé avec votre code.

  • Afficher.Explorateur de serveur :

CTRL + W, L

Affiche l’explorateur de serveur, qui vous permet de visualiser et de manipuler la base de données serveurs, journaux d’événements, files d’attente de messages, services Web et autres services système.

  • Fenêtre.FermerFenêtreOutils :

MAJ + ESC

Ferme la fenêtre d’outils actuelle.

  • Données.AfficherSourcesDonnées :

MAJ + ALT + D

Affiche la fenêtre Sources de données.

  • Fenêtre.FermerDocumentWindow :

CTRL + F4

Ferme l’onglet en cours

Commande de débogage :

  • Débogage.Autos

CTRL+D, A

Affiche la fenêtre Autos, qui affiche les variables utilisées dans le ligne de code courante et la ligne de code précédente.

  • Débogage.CallStack :

CTRL+D, C

Affiche la fenêtre Call Stack, qui affiche une liste de tous les méthodes ou cadres de pile pour le thread d’exécution en cours.

  • Débogage.Immédiat :

CTRL+D, I

Affiche la fenêtre Exécution, où les expressions peuvent être évaluées.

  • Debug.Locals :

CTRL+D, L

Affiche la fenêtre Locals, qui affiche les variables locales et leurs valeurs pour chaque méthode dans le cadre de pile actuel.

  • Débogage.QuickWatch :

CTRL+D, Q

Affiche la boîte de dialogue QuickWatch contenant la valeur actuelle du expression sélectionnée.

  • Déboguer.Démarrer :

F5

Lance l’application sous le débogueur en fonction des paramètres du projet de démarrage. En mode Break, invoquer cette commande exécutera l’application jusqu’au prochain point d’arrêt.

  • Debug.StartWithoutDebugging :

CTRL+F5

Lance l’application sans appeler le débogueur.

  • Debug.StepInto :

F11

Exécute le code une instruction à la fois, après l’exécution dans appels de méthode.

  • Debug.StepIntoSpeciﬁc :

MAJ+ALT+F11

Invite à sélectionner la sous-expression à parcourir, quel que soit le réglage des options d’outils pour ignorer les propriétés et les opérateurs.

  • Débogage.StepOut :

MAJ+F11

Exécute les lignes restantes d’une méthode dans laquelle le courant le point d’exécution est localisé.

  • Débogage.StepOver :

F10

Exécute la ligne de code suivante, mais ne suit pas l’exécution tous les appels de méthode.

  • Déboguer.Arrêter le débogage :

MAJ+F5

Arrête l’exécution de l’application en cours sous le débogueur.

  • Debug.ToggleBreakpoint :

F9

Définit ou supprime un point d’arrêt sur la ligne actuelle.

  • Débogage.Regarder :

CTRL+D, W

Affiche la fenêtre Watch, qui affiche les valeurs des variables ou expressions de surveillance.

  • Debug.EnableBreakpoint :

CTRL+F9

Bascule le point d’arrêt entre désactivé et activé.

  • Test.RunAllTestsInSolution :

CTRL+R, A

Exécute tous les tests unitaires de la solution.

Construire des commandes :

  • Construire.ConstruireSolution :

F6 ou CTRL+MAJ+B

Construit tous les projets de la solution.

  • Construire.BuildSelection :

MAJ+F6

Construit le projet sélectionné et ses dépendances.
