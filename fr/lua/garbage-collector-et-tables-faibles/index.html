Garbage collector et tables faibles

  • Syntaxe
  • Paramètres
  • Tableaux faibles

Sur cette page

  • Syntaxe
  • Paramètres
  • Tableaux faibles

Syntaxe #

  1. collectgarbage(gcrule [, gcdata]) – collecte les ordures à l’aide de gcrule
  2. setmetatable(tab, {__mode =faiblemode}) - définit le mode faible de l’onglet sur mode faible

Paramètres #

paramètre détails
gcrule & gcdata Action vers gc (garbage collector) : "stop" (arrêter la collecte), "restart" (recommencer la collecte), "collect" ou nil (récupérer toutes les ordures), "step" ( faire une étape de collecte), "count" (renvoie le nombre de mémoire utilisée en Ko), "setpause" et les données sont un nombre compris entre 0% et 100% (définit le paramètre de pause de gc), " setstepmul" et les données sont un nombre compris entre 0% et 100 (définissez "stepmul" pour gc).
mode faible Type de table faible : "k" (uniquement les clés faibles), "v" (uniquement les valeurs faibles), "vk" (les clés et les valeurs faibles)

Tableaux faibles #

local t1, t2, t3, t4 = {}, {}, {}, {} -- Create 4 tables
local maintab = {t1, t2} -- Regular table, strong references to t1 and t2
local weaktab = setmetatable({t1, t2, t3, t4}, {__mode = 'v'}) -- table with weak references.

t1, t2, t3, t4 = nil, nil, nil, nil -- No more "strong" references to t3 and t4
print(#maintab, #weaktab) --> 2 4

collectgarbage() -- Destroy t3 and t4 and delete weak links to them.
print(#maintab, #weaktab) --> 2 2
