Les booléens, la vérité et la fausseté sont simples en Lua. Réviser:

  1. Il existe un type booléen avec exactement deux valeurs : “true” et “false”.
  2. Dans un contexte conditionnel (if, elseif, while, until), un booléen n’est pas nécessaire. N’importe quelle expression peut être utilisée.
  3. Dans un contexte conditionnel, “faux” et “nil” comptent comme faux, et tout le reste compte comme vrai.
  4. Bien que 3 implique déjà ceci : si vous venez d’autres langages, rappelez-vous que 0 et la chaîne vide comptent comme vrai dans les contextes conditionnels en Lua.

Le type booléen #

Booléens et autres valeurs #

Lorsqu’il s’agit de lua, il est important de faire la différence entre les valeurs booléennes “true” et “false” et les valeurs évaluées comme vraies ou fausses.

Il n’y a que deux valeurs dans lua qui sont évaluées comme fausses : ’nil’ et ‘false’, tandis que tout le reste, y compris le ‘0’ numérique, est évalué comme vrai.

Quelques exemples de ce que cela signifie :

if 0 then print("0 is true") end --> this will print "true"
if (2 == 3) then print("true") else print("false") end --> this prints "false"
if (2 == 3) == false then print("true") end --> this prints "true"
if (2 == 3) == nil then else print("false") end
--> prints false, because even if nil and false both evaluate to false,
--> they are still different things.

Opérations logiques #

Les opérateurs logiques en lua ne renvoient pas nécessairement des valeurs booléennes :

et renverra la deuxième valeur si la première valeur est vraie ;

or renvoie la deuxième valeur si la première valeur est fausse ;

Cela permet de simuler l’opérateur ternaire, comme dans d’autres langages :

local var = false and 20 or 30 --> returns 30
local var = true and 20 or 30 --> returns 20
-- in C: false ? 20 : 30

Cela peut également être utilisé pour initialiser des tables si elles n’existent pas

tab = tab or {} -- if tab already exists, nothing happens

ou pour éviter d’utiliser des instructions if, rendant le code plus facile à lire

print(debug and "there has been an error") -- prints "false" line if debug is false
debug and print("there has been an error") -- does nothing if debug is false
-- as you can see, the second way is preferable, because it does not output
-- anything if the condition is not met, but it is still possible.
-- also, note that the second expression returns false if debug is false,
-- and whatever print() returns if debug is true (in this case, print returns nil)

Vérifier si des variables sont définies #

On peut aussi facilement vérifier si une variable existe (si elle est définie), puisque les variables inexistantes renvoient ’nil’, qui s’évalue à faux.

local tab_1, tab_2 = {}
if tab_1 then print("table 1 exists") end --> prints "table 1 exists"
if tab_2 then print("table 2 exists") end --> prints nothing

Le seul cas où cela ne s’applique pas est lorsqu’une variable stocke la valeur “false”, auquel cas elle existe techniquement mais est toujours évaluée à false. Pour cette raison, c’est une mauvaise conception de créer des fonctions qui renvoient ‘false’ et ’nil’ en fonction de l’état ou de l’entrée. On peut quand même vérifier si on a un ’nil’ ou un ‘false’ :

if nil == nil then print("A nil is present") else print("A nil is not present") end
if false == nil then print("A nil is present") else print("A nil is not present") end
-- The output of these calls are:
-- A nil is present!
-- A nil is not present

Contextes conditionnels #

Les contextes conditionnels en Lua (if, elseif, while, until) ne nécessitent pas de booléen. Comme beaucoup de langages, n’importe quelle valeur Lua peut apparaître dans une condition. Les règles d’évaluation sont simples :

  1. “false” et “nil” sont considérés comme faux.

  2. Tout le reste compte comme vrai.

    if 1 then
  print("Numbers work.")
end
if 0 then
  print("Even 0 is true")
end

if "strings work" then
  print("Strings work.")
end
if "" then
  print("Even the empty string is true.")
end

Opérateurs logiques #

En Lua, les booléens peuvent être manipulés via des opérateurs logiques. Ces opérateurs incluent not, and et or.

Dans des expressions simples, les résultats sont assez simples :

print(not true) --> false
print(not false) --> true
print(true or false) --> true
print(false and true) --> false

Ordre de préséance #

L’ordre de priorité est similaire aux opérateurs mathématiques unaires -, * et + :

  • “pas”
  • puis ’et’
  • puis ‘ou’

Cela peut conduire à des expressions complexes :

print(true and false or not false and not true)
print( (true and false) or ((not false) and (not true)) )
    --> these are equivalent, and both evaluate to false

Évaluation raccourcie #

Les opérateurs “et” et “ou” ne peuvent être évalués qu’à l’aide du premier opérande, à condition que le second ne soit pas nécessaire :

function a()
    print("a() was called")
    return true
end

function b()
    print("b() was called")
    return false
end

print(a() or b())
    --> a() was called
    --> true
    --  nothing else
print(b() and a())
    --> b() was called
    --> false
    --  nothing else
print(a() and b())
    --> a() was called
    --> b() was called
    --> false

Idiomatique [opérateur conditionnel][1] #

En raison de la priorité des opérateurs logiques, de la capacité d’évaluation par raccourci et de l’évaluation des valeurs non “false” et non “nil” comme “true”, un opérateur conditionnel idiomatique est disponible en Lua :

function a()
    print("a() was called")
    return false
end
function b()
    print("b() was called")
    return true
end
function c()
    print("c() was called")
    return 7
end

print(a() and b() or c())
    --> a() was called
    --> c() was called
    --> 7
    
print(b() and c() or a())
    --> b() was called
    --> c() was called
    --> 7

De plus, en raison de la nature de la structure x et a ou b, a ne sera jamais renvoyé s’il est évalué à false, cette condition retournera alors toujours b, peu importe ce que x est .

print(true and false or 1)  -- outputs 1

Tables de vérité #

Les opérateurs logiques en Lua ne “renvoyent” pas un booléen, mais un de leurs arguments. En utilisant nil pour false et des nombres pour true, voici comment ils se comportent.

print(nil and nil)       -- nil
print(nil and 2)         -- nil
print(1 and nil)         -- nil
print(1 and 2)           -- 2

print(nil or nil)        -- nil
print(nil or 2)          -- 2
print(1 or nil)          -- 1
print(1 or 2)            -- 1

Comme vous pouvez le voir, Lua renverra toujours la première valeur qui fait échouer ou réussir la vérification. Voici les tables de vérité qui le montrent.

  x  |  y  || and            x  |  y  || or
------------------         ------------------
false|false||  x           false|false||  y   
false|true ||  x           false|true ||  y   
true |false||  y           true |false||  x   
true |true ||  y           true |true ||  x

Pour ceux qui en ont besoin, voici deux fonctions représentant ces opérateurs logiques.

function exampleAnd(value1, value2)
  if value1 then
    return value2
  end
  return value1
end

function exampleOr(value1, value2)
  if value1 then
    return value1
  end
  return value2
end

Emulation d’un opérateur ternaire avec les opérateurs logiques ’et’ ‘ou’. #

En lua, les opérateurs logiques “et” et “ou” renvoient l’un des opérandes comme résultat au lieu d’un résultat booléen. En conséquence, ce mécanisme peut être exploité pour émuler le comportement de l’opérateur ternaire bien que lua n’ait pas d’opérateur ternaire “réel” dans le langage.

Syntaxe #

condition et truthy_expr ou falsey_expr

Utilisation dans l’affectation/initialisation de variables #

local drink = (fruit == "apple") and "apple juice" or "water"

Utilisation dans le constructeur de table #

local menu =
{
  meal  = vegan and "carrot" or "steak",
  drink = vegan and "tea"    or "chicken soup"
}

Utiliser comme argument de fonction #

print(age > 18 and "beer" or "fruit punch")

Utiliser dans l’instruction de retour #

function get_gradestring(student)
  return student.grade > 60 and "pass" or "fail"
end

Mise en garde #

Il existe des situations où ce mécanisme n’a pas le comportement souhaité. Considérez ce cas

local var = true and false or "should not happen"

Dans un ‘vrai’ opérateur ternaire, la valeur attendue de ‘var’ est ‘false’. En lua, cependant, l’évaluation « et » « échoue » parce que le deuxième opérande est faux. En conséquence, var se termine par ne devrait pas arriver à la place.

Deux solutions de contournement possibles à ce problème, refactorisez cette expression afin que l’opérande du milieu ne soit pas faux. par exemple.

local var = not true and "should not happen" or false

ou alternativement, utilisez la construction classique if then else.
