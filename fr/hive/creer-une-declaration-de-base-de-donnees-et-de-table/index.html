Syntaxe #

  • CREATE [TEMPORARY] [EXTERNAL] TABLE [IF NOT EXISTS] [db_name.]table_name

    [(col_name data_type [COMMENT col_comment], ...)]

[COMMENT tableau_commentaire] [PARTITIOND BY (col_name data_type [COMMENT col_comment], …)] [CLUSTERED BY (col_name, col_name, …) [SORTED BY (col_name [ASC|DESC], …)] INTO num_buckets BUCKETS] [SKEWED BY (col_name, col_name, …) – (Remarque : disponible dans Hive 0.10.0 et versions ultérieures)] ON ((col_value, col_value, …), (col_value, col_value, …), …) [STORED AS DIRECTORIES] [ [FORMAT DE LIGNE row_format] [STOCKÉ COMME file_format] | STORED BY ‘storage.handler.class.name’ [WITH SERDEPROPERTIES (…)] ] [LOCATION hdfs_path] [TBLPROPERTIES (nom_propriété=valeur_propriété, …)] [AS select_statement] ;

  • CREATE [TEMPORARY] [EXTERNAL] TABLE [IF NOT EXISTS] [db_name.]table_name COMME nom_table_ou_vue_existante [LOCATION hdfs_path] ;
  • type_données : type_primitif, type_tableau, type_carte, type_struct, type_union
  • type_primitif : TINYINT, SMALLINT, INT , BIGINT, BOOLEAN, FLOAT, DOUBLE, STRING, BINARY, TIMESTAMP, DECIMAL, DECIMAL (précision, échelle), DATE, VARCHAR, CHAR
  • type_tableau : TABLEAU < type_données >
  • type_carte : MAP < type_primitif, type_données >
  • struct_type : STRUCT < col_name : data_type [COMMENT col_comment], …>
  • type_union : UNIONTYPE < type_données, type_données, … >
  • row_format : DELIMITED [FIELDS TERMINATED BY char [ESCAPED BY char]] [COLLECTION ITEMS TERMINATED BY char] [MAP KEYS TERMINATED BY char] [LINES TERMINATED BY char] [NULL DEFINED AS char]
    , SERDE nom_serde [AVEC SERDEPROPERTIES (nom_propriété=valeur_propriété, nom_propriété=valeur_propriété, …)]
  • format de fichier: : FICHIER SEQUENCE , TEXTFILE , RCFILE , ORC , PARQUET , AVRO , INPUTFORMAT input_format_classname OUTPUTFORMAT output_format_classname
  • CREATE (DATABASE|SCHEMA) [IF NOT EXISTS] nom_base_données [COMMENT base de données_commentaire] [LOCATION hdfs_path] [AVEC DBPROPERTIES (nom_propriété=valeur_propriété, …)] ;

Lorsque vous travaillez avec des tables et des bases de données dans HIVE. Les points ci-dessous peuvent être utiles.

  • Nous pouvons changer de base de données en utilisant use database; command
  • Pour connaître la base de données de travail actuelle, nous pouvons utiliser SELECT current_database()
  • Pour voir le DDL utilisé pour l’instruction de création de table, nous pouvons utiliser SHOW CREATE TABLE tablename
  • Pour voir toutes les colonnes de la table, utilisez DESCRIBE tablename pour afficher des détails étendus comme l’emplacement serde utilisé et d’autres DESCRIBE FORMATTED tablename. DESCRIBE peut également être abrégé en DESC.

Créer un tableau #

Création d’une table gérée avec partition et stockée en tant que fichier de séquence. Le format de données dans les fichiers est supposé être délimité par des champs par Ctrl-A (^A) et délimité par des lignes par une nouvelle ligne. Le tableau ci-dessous est créé dans le répertoire de l’entrepôt de ruche spécifié dans la valeur de la clé hive.metastore.warehouse.dir dans le fichier de configuration Hive hive-site.xml.

CREATE TABLE view
(time INT, 
id BIGINT,
url STRING, 
referrer_url STRING,
add STRING COMMENT 'IP of the User')
COMMENT 'This is view table'
PARTITIONED BY(date STRING, region STRING)
ROW FORMAT DELIMITED
FIELDS TERMINATED BY '\001'
STORED AS SEQUENCEFILE;

Création d’une table externe avec des partitions et stockée sous forme de fichier de séquence. Le format de données dans les fichiers est supposé être délimité par des champs par ctrl-A et délimité par des lignes par retour à la ligne. Le tableau ci-dessous est créé à l’emplacement spécifié et il est pratique lorsque nous avons déjà des données. L’un des avantages de l’utilisation d’une table externe est que nous pouvons supprimer la table sans supprimer les données. Par exemple, si nous créons une table et réalisons que le schéma est erroné, nous pouvons supprimer la table en toute sécurité et recréer avec le nouveau schéma sans nous soucier des données. L’autre avantage est que si nous utilisons d’autres outils comme pig sur les mêmes fichiers, nous pouvons continuer à les utiliser même après avoir supprimé la table.

CREATE EXTERNAL TABLE view
(time INT, 
id BIGINT,
url STRING, 
referrer_url STRING,
add STRING COMMENT 'IP of the User')
COMMENT 'This is view table'
PARTITIONED BY(date STRING, region STRING)
ROW FORMAT DELIMITED
FIELDS TERMINATED BY '\001'
STORED AS SEQUENCEFILE
LOCATION '<hdfs_location>';

Création d’une table à l’aide d’une requête de sélection et remplissage des résultats de la requête, these statements are known as CTAS(Create Table As Select).

Il y a deux parties dans CTAS, la partie SELECT peut être n’importe quelle instruction SELECT prise en charge par HiveQL. La partie CREATE du CTAS prend le schéma résultant de la partie SELECT et crée la table cible avec d’autres propriétés de table telles que le SerDe et le format de stockage.

CTAS a ces restrictions :

  • La table cible ne peut pas être une table partitionnée.

  • La table cible ne peut pas être une table externe.

  • La table cible ne peut pas être une table de regroupement de listes.

    CREATE TABLE new_key_value_store ROW FORMAT SERDE “org.apache.hadoop.hive.serde2.columnar.ColumnarSerDe” STORED AS RCFile AS SELECT * FROM page_view SORT BY url, add;

Créer une table comme :

La forme LIKE de CREATE TABLE vous permet de copier exactement une définition de table existante (sans copier ses données). Contrairement à CTAS, l’instruction ci-dessous crée une nouvelle table dont la définition correspond exactement à la table existante dans tous les détails autres que le nom de la table. La nouvelle table ne contient aucune ligne.

CREATE TABLE empty_page_views
LIKE page_views;

Créer une base de données #

Création d’une base de données dans un emplacement particulier. Si nous ne spécifions aucun emplacement pour la base de données, elle est créée dans le répertoire de l’entrepôt.

CREATE DATABASE IF NOT EXISTS db_name 
COMMENT 'TEST DATABASE'
LOCATION /PATH/HDFS/DATABASE/;

Intégration HIVE_HBASE #

L’intégration Hive-Hbase est prise en charge depuis les versions ci-dessous. Ruche : 0.11.0 HBase : 0.94.2 Hadoop : 0.20.2

CREATE TABLE hbase_hive
(id string,
 col1 string,
 col2 string,
 col3 int) 
STORED BY 'org.apache.hadoop.hive.hbase.HBaseStorageHandler'
WITH SERDEPROPERTIES 
("hbase.columns.mapping" = ":key,cf1:col1,cf1:col2,cf1:col3")
TBLPROPERTIES ("hbase.table.name" = "hive_hbase");

Remarque : la 1ère colonne doit être la colonne clé.

Créer une table en utilisant les propriétés de table existantes. #

CREATE TABLE new_table_name LIKE existing_table_name;

Création de la table Hive ACID. #

Les tables ACID sont prises en charge depuis la version 0.14 de Hive. Le tableau ci-dessous prend en charge UPDATE/DELETE/INSERT

Ci-dessous, les changements de configuration requis dans hive-site.xml.

 hive.support.concurrency = true
 hive.enforce.bucketing = true
 hive.exec.dynamic.partition.mode = nonstrict
 hive.txn.manager =org.apache.hadoop.hive.ql.lockmgr.DbTxnManager
 hive.compactor.initiator.on = true
 hive.compactor.worker.threads = 1

Actuellement, seul le format de fichier orc est pris en charge.

Instruction de création de table.

 create table Sample_Table(
 col1 Int,
 col2 String,
 col3 String) 
 clustered by (col3) into 3 buckets 
 stored as orc 
 TBLPROPERTIES ('transactional'='true');
