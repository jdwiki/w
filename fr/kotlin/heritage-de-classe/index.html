Tout langage de programmation orienté objet possède une certaine forme d’héritage de classe. Permettez-moi de réviser:

Imaginez que vous deviez programmer un bouquet de fruits : Pommes, Oranges et Poires. Ils diffèrent tous par leur taille, leur forme et leur couleur, c’est pourquoi nous avons des classes différentes.

Mais disons que leurs différences n’ont pas d’importance une seconde et que vous voulez juste un “Fruit”, peu importe lequel exactement ? Quel type de retour aurait getFruit() ?

La réponse est la classe “Fruit”. Nous créons une nouvelle classe et en faisons hériter tous les fruits !

Syntaxe #

  • ouvrir {Classe de base}
  • classe {Classe dérivée} : {Classe de base}({Init Arguments})
  • remplacer {Définition de fonction}
  • {DC-Object} est {Classe de base} == vrai

Paramètres #

Paramètre Détails
Classe de base Classe héritée de
Classe dérivée Classe qui hérite de la classe de base
Init Arguments Arguments transmis au constructeur de la classe de base
Définition de fonction Fonction dans la classe dérivée qui a un code différent de celui de la classe de base
DC-Object “Derived Class-Object” Objet qui a le type de la classe dérivée

Notions de base : le mot-clé “open” #

Dans Kotlin, les classes sont finales par défaut, ce qui signifie qu’elles ne peuvent pas être héritées.

Pour autoriser l’héritage sur une classe, utilisez le mot clé open.

open class Thing {
    // I can now be extended!
}

Remarque : les classes abstraites, les classes scellées et les interfaces être “ouvert” par défaut.

Héritage des champs d’une classe #

Définition de la classe de base : #

open class BaseClass {
    val x = 10
}

Définition de la classe dérivée : #

class DerivedClass: BaseClass() {
    fun foo() {
        println("x is equal to " + x)
    }
}

Utilisation de la sous-classe : #

fun main(args: Array<String>) {
    val derivedClass = DerivedClass()
    derivedClass.foo() // prints: 'x is equal to 10'
}

Héritage des méthodes d’une classe #

Définition de la classe de base : #

open class Person {
    fun jump() {
        println("Jumping...")
    }
}

Définition de la classe dérivée : #

class Ninja: Person() {
    fun sneak() {
        println("Sneaking around...")
    }
}

Le Ninja a accès à toutes les méthodes en personne #

fun main(args: Array<String>) {
    val ninja = Ninja()
    ninja.jump()  // prints: 'Jumping...'
    ninja.sneak() // prints: 'Sneaking around...'
}

Remplacer les propriétés et les méthodes #

Propriétés de remplacement (à la fois en lecture seule et modifiables) : #

abstract class Car {
    abstract val name: String;
    open var speed: Int = 0;
}

class BrokenCar(override val name: String) : Car() {
    override var speed: Int
        get() = 0
        set(value) {
            throw UnsupportedOperationException("The car is bloken")
        }
}

fun main(args: Array<String>) {
    val car: Car = BrokenCar("Lada")
    car.speed = 10
}

Remplacer les méthodes : #

interface Ship {
    fun sail()
    fun sink()
}

object Titanic : Ship {

    var canSail = true

    override fun sail() {
        sink()
    }

    override fun sink() {
        canSail = false
    }
}
