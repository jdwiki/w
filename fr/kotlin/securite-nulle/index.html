Casts intelligents #

Si le compilateur peut déduire qu’un objet ne peut pas être nul à un certain point, vous n’avez plus besoin d’utiliser les opérateurs spéciaux :

var string: String? = "Hello!"
print(string.length)  // Compile error
if(string != null) {
    // The compiler now knows that string can't be null
    print(string.length)  // It works now!
}

Remarque : Le compilateur ne vous permettra pas de convertir intelligemment les variables mutables qui pourraient potentiellement être modifiées entre la vérification nulle et l’utilisation prévue.

Si une variable est accessible depuis l’extérieur de la portée du bloc actuel (parce qu’elle est membre d’un objet non local, par exemple), vous devez créer une nouvelle référence locale que vous pouvez ensuite convertir et utiliser intelligemment.

## affirmation Les suffixes !! ignorent la possibilité de nullité et renvoient une version non nulle de ce type. KotlinNullPointerException sera lancé si l’objet est un null.

val message: String? = null
println(message!!) //KotlinNullPointerException thrown, app crashes

Types Nullable et Non-Nullable #

Les types normaux, comme String, ne sont pas nullables. Pour qu’ils puissent contenir des valeurs nulles, vous devez les indiquer explicitement en plaçant un ? derrière eux : String?

var string        : String = "Hello World!"
var nullableString: String? = null

string = nullableString    // Compiler error: Can't assign nullable to non-nullable type.
nullableString = string    // This will work however!

Élimine les valeurs nulles d’un Iterable et d’un tableau #

Parfois, nous devons changer le type de Collection<T?> en Collections<T>. Dans ce cas, filterNotNull est notre solution.

val a: List<Int?> = listOf(1, 2, 3, null)
val b: List<Int> = a.filterNotNull()

Null Coalescing / Opérateur Elvis #

Parfois, il est souhaitable d’évaluer une expression nullable d’une manière if-else. L’opérateur elvis, ?:, peut être utilisé dans Kotlin pour une telle situation.

Par exemple:

val value: String = data?.first() ?: "Nothing here."

L’expression ci-dessus renvoie "Rien ici" si data?.first() ou data lui-même donne une valeur null sinon le résultat de data?.first().

Il est également possible de lever des exceptions en utilisant la même syntaxe pour interrompre l’exécution du code.

val value: String = data?.second() 
    ?: throw IllegalArgumentException("Value can't be null!")

Rappel : les exceptions NullPointerExceptions peuvent être levées à l’aide de l’[opérateur d’assertion][1] (par exemple, data !!.second() !!)

[1] : https://www.wikiod.com/fr/kotlin/securite-nulle#Affirmation

Opérateur d’appel sécurisé #

Pour accéder aux fonctions et aux propriétés de types nullables, vous devez utiliser des opérateurs spéciaux.

Le premier, ?., vous donne la propriété ou la fonction à laquelle vous essayez d’accéder, ou il vous donne null si l’objet est null :

val string: String? = "Hello World!"
print(string.length)   // Compile error: Can't directly access property of nullable type.
print(string?.length)  // Will print the string's length, or "null" if the string is null.

Idiom : appeler plusieurs méthodes sur le même objet vérifié par null #

Une manière élégante d’appeler plusieurs méthodes d’un objet à valeur nulle consiste à utiliser [apply][1] de Kotlin comme ceci :

obj?.apply { 
    foo()
    bar()
}

Cela appellera foo et bar sur obj (qui est this dans le bloc apply) uniquement si obj n’est pas nul, en sautant le bloc entier sinon.

Pour amener une variable nullable dans la portée en tant que référence non nullable sans en faire le récepteur implicite des appels de fonction et de propriété, vous pouvez utiliser [let][2] au lieu de apply :

nullable?.let { notnull ->
    notnull.foo()
    notnull.bar()
}

notnull pourrait être nommé n’importe quoi, ou même omis et utilisé via [le paramètre lambda implicite it.][3]

[1] : https://kotlinlang.org/api/latest/jvm/stdlib/kotlin/apply.html [2] : https://kotlinlang.org/api/latest/jvm/stdlib/kotlin/let.html [3] : https://www.wikiod.com/fr/kotlin/les-fonctions#Fonctions lambda

Opérateur Elvis (?:) #

Dans Kotlin, nous pouvons déclarer une variable pouvant contenir une “référence nulle”. Supposons que nous ayons une référence nullable a, nous pouvons dire “si a n’est pas nul, utilisez-le, sinon utilisez une valeur non nulle x”

var a: String? = "Nullable String Value"

Maintenant, ‘a’ peut être nul. Ainsi, lorsque nous devons accéder à la valeur de a, nous devons effectuer un contrôle de sécurité, qu’il contienne ou non une valeur. Nous pouvons effectuer ce contrôle de sécurité par une instruction conventionnelle “if…else”.

val b: Int = if (a != null) a.length else -1

Mais voici l’opérateur avancé Elvis(Operator Elvis : ?:). Ci-dessus if...else peut être exprimé avec l’opérateur Elvis comme ci-dessous :

val b = a?.length ?: -1

Si l’expression à gauche de ?: (ici : a?.length) n’est pas nulle, l’opérateur elvis la renvoie, sinon il renvoie l’expression à droite (ici : -1). L’expression du côté droit est évaluée uniquement si le côté gauche est nul.
