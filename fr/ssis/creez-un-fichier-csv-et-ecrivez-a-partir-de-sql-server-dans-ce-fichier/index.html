Créez un fichier CSV et écrivez à partir de SQL Server dans ce fichier

  • Étapes pour importer des données

Sur cette page

  • Étapes pour importer des données

Le guide aide à comprendre comment importer des données de la table du serveur SQL vers un fichier CSV/txt.

  1. Cliquez avec le bouton droit sur la tâche de flux de données et sélectionnez la propriété. Les propriétés DefaultBufferMaxRows et DefaultBufferSize peuvent être modifiées pour améliorer les performances de chargement des données.
  2. Plusieurs tâches de flux de données peuvent être exécutées en parallèle pour de meilleures performances.
  3. Chaque tâche a deux flux de réussite et d’échec. Il est important de gérer le flux de pannes pour rendre le package plus robuste.
  4. Dans la tâche de flux de données, cliquez avec le bouton droit sur la flèche bleue et sélectionnez Activer la visionneuse de données pour vérifier le flux de données au moment de l’exécution.
  5. Si une colonne est supprimée dans la source ou la destination, vérifiez quelle colonne est supprimée. Dans le flux de données, cliquez avec le bouton droit sur la flèche bleue et sélectionnez Résoudre les références, dans la nouvelle fenêtre, nous pouvons voir les colonnes de sortie non mappées (à gauche) et les colonnes d’entrée non mappées (à droite).

Étapes pour importer des données #

Voici ce qui est nécessaire pour remplir cet objectif.

  1. Tâche de flux de données : Dans cette tâche, nous allons effectuer une importation de données.
  2. Source OLE DB : Pour sélectionner la source de données, c’est-à-dire la table de base de données du serveur SQL.
  3. Flat File Destination : Destination dans laquelle nous voulons charger les données.

Pas

  1. Faites glisser et déposez une * tâche de flux de données * de la boîte à outils SSIS à partir de la section Favoris. [![entrez la description de l’image ici][1]][1]
  2. Double-cliquez sur la * tâche de flux de données * dans le flux de contrôle, cela nous amènera au flux de données.
  3. Faites glisser et déposez une source OLE DB, par défaut une croix apparaîtra dessus, cela signifie qu’elle n’est pas configurée avec une connexion. Double-cliquez sur la tâche OLE DB Source, cliquez sur Nouveau. [![entrez la description de l’image ici][2]][2]
  4. Dans la fenêtre Configurer le gestionnaire de connexion OLE DB, cliquez sur Nouveau. Maintenant, dans la fenêtre du gestionnaire de connexion, sélectionnez le nom du serveur auquel vous souhaitez vous connecter. Sélectionnez Authentification Windows si votre serveur se trouve sur votre machine, sinon Utilisez l’authentification SQL Server et entrez le nom d’utilisateur et le mot de passe. Cliquez sur Tester la connexion en bas à gauche pour vérifier la validité de l’identifiant saisi. Cliquez sur OK puis à nouveau sur OK. [![entrez la description de l’image ici][3]][3]
  5. Dans l’éditeur de source OLE DB, sélectionnez le nom de la table ou de la vue et cliquez sur Aperçu pour vérifier les données. Cliquez sur Fermer puis OK. [![entrez la description de l’image ici][4]][4]
  6. Faites glisser et déposez une tâche * Destination de fichier plat * de la boîte à outils SSIS sous la section autres destinations. Connectez la source OLE DB à la destination de fichier plat.
  7. Double-cliquez sur Destination de fichier plat, cliquez sur Nouveau pour ouvrir la fenêtre Format de fichier plat. Sélectionnez Délimité si vous souhaitez spécifier le séparateur, le qualificateur de texte, la fin de ligne, etc. Cliquez sur OK. [![entrez la description de l’image ici][5]][5]
  8. Dans l’éditeur Flat File Connection Manager, cliquez sur le bouton Parcourir, sélectionnez le chemin d’accès au fichier et entrez le nom du fichier, puis cliquez sur Ouvrir. Même si nous n’avons sélectionné aucun fichier, nous venons d’entrer le nom du fichier, le fichier sera créé.
  9. Sélectionnez maintenant la page de code, le qualificateur de texte, etc. N’oubliez pas de cocher la case nom de colonne dans la première ligne de données. Sur le côté gauche, sélectionnez Colonnes ici, vous pouvez spécifier le séparateur de données comme une virgule ou un tuyau (|). Cliquez sur OK. [![entrez la description de l’image ici][6]][6]
  10. Dans l’éditeur de destination de fichier plat, écraser les données dans le fichier est sélectionné, mettez-le à jour en fonction des besoins. Sur la gauche, sélectionnez Mappages et vérifiez si les colonnes sont correctement mappées. Cliquez sur OK.
  11. Dans l’Explorateur de solutions, cliquez avec le bouton droit sur le nom du package et exécutez-le pour vérifier.
