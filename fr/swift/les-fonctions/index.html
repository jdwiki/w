Fonctions avec paramètres #

Les fonctions peuvent prendre des paramètres afin que leur fonctionnalité puisse être modifiée. Les paramètres sont donnés sous forme de liste séparée par des virgules avec leurs types et noms définis.

func magicNumber(number1: Int)
{
    print("\(number1) Is the magic number")
}

Remarque : La syntaxe \(number1) est basique [String Interpolation][1] et est utilisée pour insérer l’entier dans la chaîne.

Les fonctions avec paramètres sont appelées en spécifiant la fonction par son nom et en fournissant une valeur d’entrée du type utilisé dans la déclaration de la fonction.

magicNumber(5)
//output: "5 Is the magic number
let example: Int = 10
magicNumber(example)
//output: "10 Is the magic number"

N’importe quelle valeur de type Int aurait pu être utilisée.

func magicNumber(number1: Int, number2: Int)
{
    print("\(number1 + number2) Is the magic number")
}

Lorsqu’une fonction utilise plusieurs paramètres, le nom du premier paramètre n’est pas requis pour le premier mais figure sur les paramètres suivants.

let ten: Int = 10
let five: Int = 5
magicNumber(ten,number2: five)
//output: "15 Is the magic number"

Utilisez des noms de paramètres externes pour rendre les appels de fonction plus lisibles.

func magicNumber(one number1: Int, two number2: Int)
{
    print("\(number1 + number2) Is the magic number")
}

let ten: Int = 10
let five: Int = 5
magicNumber(one: ten, two: five)

La définition de la valeur par défaut dans la déclaration de la fonction vous permet d’appeler la fonction sans donner de valeurs d’entrée.

func magicNumber(one number1: Int = 5, two number2: Int = 10)
{
    print("\(number1 + number2) Is the magic number")
}

magicNumber()
//output: "15 Is the magic number"

[1] : https://www.wikiod.com/fr/swift/chaines-et-caracteres#Littéraux de chaînes et de caractères

Méthodes #

Les méthodes d’instance sont des fonctions qui appartiennent aux instances d’un type dans Swift (une [classe][1], [struct][2], [énumération][3] ou [protocol][4]). Les méthodes de type sont appelées sur un type lui-même.

## Méthodes d’instance

Les méthodes d’instance sont définies avec une déclaration func à l’intérieur de la définition du type, ou dans une [extension][5].

class Counter {
    var count = 0
    func increment() {
        count += 1
    }
}

La méthode d’instance increment() est appelée sur une instance de la classe Counter :

let counter = Counter()  // create an instance of Counter class   
counter.increment()      // call the instance method on this instance

Typer les méthodes #

Les méthodes de type sont définies avec les mots clés static func. (Pour les classes, class func définit une méthode de type qui peut être remplacée par des sous-classes.)

class SomeClass {
    class func someTypeMethod() {
        // type method implementation goes here
    }
}
SomeClass.someTypeMethod()  // type method is called on the SomeClass type itself

Indices #

Les opérateurs sont des fonctions #

[Opérateurs] (https://www.wikiod.com/fr/swift/operateurs-avances) tels que +, -, ?? sont une sorte de fonction nommée en utilisant des symboles plutôt que des lettres. Elles sont appelées différemment des fonctions :

  • Préfixe : -x
  • Infixe : x + y - Suffixe : x++

Vous pouvez en savoir plus sur les opérateurs de base et les [opérateurs avancés](https://developer.apple. com/library/ios/documentation/Swift/Conceptual/Swift_Programming_Language/BasicOperators.html) dans le langage de programmation Swift.

Paramètres variadiques #

Parfois, il n’est pas possible de lister le nombre de paramètres dont une fonction pourrait avoir besoin. Prenons une fonction sum :

func sum(_ a: Int, _ b: Int) -> Int {
    return a + b
}

Cela fonctionne bien pour trouver la somme de deux nombres, mais pour trouver la somme de trois, nous devrions écrire une autre fonction :

func sum(_ a: Int, _ b: Int, _ c: Int) -> Int {
    return a + b + c
}

et un avec quatre paramètres en aurait besoin d’un autre, et ainsi de suite. Swift permet de définir une fonction avec un nombre variable de paramètres utilisant une séquence de trois périodes : ‘…’. Par exemple,

func sum(_ numbers: Int...) -> Int {
    return numbers.reduce(0, combine: +)
}

Remarquez comment le paramètre numbers, qui est variadique, est fusionné en un seul Array de type [Int]. Ceci est vrai en général, les paramètres variadiques de type ‘T…’ sont accessibles sous la forme d’un ‘[T]’.

Cette fonction peut maintenant être appelée comme ceci :

let a = sum(1, 2) // a == 3
let b = sum(3, 4, 5, 6, 7) // b == 25

Un paramètre variadique dans Swift ne doit pas nécessairement figurer à la fin de la liste des paramètres, mais il ne peut y en avoir qu’un dans chaque signature de fonction.

Parfois, il est pratique de mettre une taille minimale sur le nombre de paramètres. Par exemple, cela n’a pas vraiment de sens de prendre la “somme” d’aucune valeur. Un moyen simple d’appliquer cela consiste à mettre des paramètres requis non variadiques, puis à ajouter le paramètre variadique après. Pour être sûr que sum ne peut être appelé qu’avec au moins deux paramètres, nous pouvons écrire

func sum(_ n1: Int, _ n2: Int, _ numbers: Int...) -> Int {
    return numbers.reduce(n1 + n2, combine: +)
}

sum(1, 2) // ok
sum(3, 4, 5, 6, 7) // ok
sum(1) // not ok
sum() // not ok

Utilisation de base #

Les fonctions peuvent être déclarées sans paramètre ni valeur de retour. La seule information requise est un nom (“bonjour” dans ce cas).

func hello()
{
    print("Hello World")
}

Appelez une fonction sans paramètre en écrivant son nom suivi d’une paire vide de parenthèses.

hello()
//output: "Hello World"

Valeurs renvoyées #

Les fonctions peuvent renvoyer des valeurs en spécifiant le type après la liste des paramètres.

func findHypotenuse(a: Double, b: Double) -> Double
{
    return sqrt((a * a) + (b * b))
}

let c = findHypotenuse(3, b: 5)
//c = 5.830951894845301

Les fonctions peuvent également renvoyer plusieurs valeurs à l’aide de tuples.

func maths(number: Int) -> (times2: Int, times3: Int)
{
    let two = number * 2
    let three = number * 3
    return (two, three)
}
let resultTuple = maths(5)
//resultTuple = (10, 15)

Lancer des erreurs #

Si vous voulez qu’une fonction soit capable de générer des erreurs, vous devez ajouter le mot-clé “throws” après les parenthèses contenant les arguments :

func errorThrower()throws -> String {}

Lorsque vous souhaitez générer une erreur, utilisez le mot-clé “throw” :

func errorThrower()throws -> String {
  if true {
    return "True"
  } else {
    // Throwing an error
    throw Error.error 
  }
}

Si vous voulez appeler une fonction qui peut générer une erreur, vous devez utiliser le mot clé try dans un bloc do :

do {
  try errorThrower()
}

Pour en savoir plus sur les erreurs Swift : https://www.wikiod.com/fr/swift/la-gestion-des-erreurs

Paramètres d’entrée #

Les fonctions peuvent modifier les paramètres qui leur sont passés s’ils sont marqués avec le mot clé inout. Lors de la transmission d’un paramètre “inout” à une fonction, l’appelant doit ajouter un “&” à la variable transmise.

func updateFruit(fruit: inout Int) {
    fruit -= 1
}

var apples = 30 // Prints "There's 30 apples"
print("There's \(apples) apples")

updateFruit(fruit: &apples)

print("There's now \(apples) apples") // Prints "There's 29 apples".

Cela permet d’appliquer la sémantique de référence aux types qui auraient normalement une sémantique de valeur.

Syntaxe de fermeture de fin #

Lorsque le dernier paramètre d’une fonction est une fermeture

func loadData(id: String, completion:(result: String) -> ()) {
    // ...
    completion(result:"This is the result data")
}

la fonction peut être invoquée à l’aide de la syntaxe de fermeture de fin

loadData("123") { result in
    print(result)
}

Fonctions avec des fermetures #

L’utilisation de fonctions qui acceptent et exécutent des fermetures peut être extrêmement utile pour envoyer un bloc de code à exécuter ailleurs. Nous pouvons commencer par autoriser notre fonction à prendre une fermeture facultative qui (dans ce cas) renverra Void.

func closedFunc(block: (()->Void)? = nil) {
    print("Just beginning")

    if let block = block {
        block()
    }
}

Maintenant que notre fonction a été définie, appelons-la et transmettons du code :

closedFunc() { Void in
    print("Over already")
}

En utilisant une fermeture finale avec notre appel de fonction, nous pouvons transmettre du code (dans ce cas, print) à exécuter à un moment donné dans notre fonction closedFunc().

class ViewController: UIViewController {

    override func viewDidLoad() {
        let _  = A.init(){Void in self.action(2)}
    }

    func action(i: Int) {
        print(i)
    }

class A: NSObject {
    var closure : ()?

    init(closure: (()->Void)? = nil) {
        // Notice how this is executed before the  closure
        print("1")
        // Make sure closure isn't nil

    }

Fonctions de passage et de retour #

La fonction suivante renvoie une autre fonction comme résultat qui peut ensuite être affectée à une variable et appelée :

func jediTrainer () -> ((String, Int) -> String) {
  func train(name: String, times: Int) -> (String) {
    return "\(name) has been trained in the Force \(times) times"
  }
  return train
}

let train = jediTrainer()
train("Obi Wan", 3)

Types de fonctions #

Chaque fonction a son propre type de fonction, composé des types de paramètres et du type de retour de la fonction elle-même. Par exemple la fonction suivante :

func sum(x: Int, y: Int) -> (result: Int) { return x + y }

a un type de fonction de :

(Int, Int) -> (Int)

Les types de fonctions peuvent ainsi être utilisés comme types de paramètres ou comme types de retour pour les fonctions imbriquées.
