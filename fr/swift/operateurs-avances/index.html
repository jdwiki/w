Opérateurs au niveau du bit #

Les opérateurs Swift Bitwise vous permettent d’effectuer des opérations sur la forme binaire des nombres. Vous pouvez spécifier un littéral binaire en préfixant le nombre avec 0b, ainsi par exemple 0b110 est équivalent au nombre binaire 110 (le nombre décimal 6). Chaque 1 ou 0 est un bit dans le nombre.

Au niveau du bit NOT ~ :

var number: UInt8 = 0b01101100
let newNumber = ~number
// newNumber is equal to 0b01101100

Ici, chaque bit est remplacé par son opposé. Déclarer explicitement le nombre UInt8 garantit que le nombre est positif (afin que nous n’ayons pas à traiter les négatifs dans l’exemple) et qu’il ne comporte que 8 bits. Si 0b01101100 était un UInt plus grand, il y aurait des 0 en tête qui seraient convertis en 1 et deviendraient significatifs lors de l’inversion :

var number: UInt16 = 0b01101100
// number equals 0b0000000001101100
// the 0s are not significant
let newNumber = ~number
// newNumber equals 0b1111111110010011
// the 1s are now significant
  • 0 -> 1
  • 1 -> 0

AND au niveau du bit ‘&’ :

var number = 0b0110
let newNumber = number & 0b1010
// newNumber is equal to 0b0010

Ici, un bit donné sera 1 si et seulement si les nombres binaires des deux côtés de l’opérateur & contenaient un 1 à cet emplacement de bit.

  • 0 & 0 -> 0
  • 0 & 1 -> 0
  • 1 & 1 -> 1

OR au niveau du bit | :

var number = 0b0110
let newNumber = number | 0b1000
// newNumber is equal to 0b1110

Ici, un bit donné sera 1 si et seulement si le nombre binaire d’au moins un côté de l’opérateur | contenait un 1 à cet emplacement de bit.

  • 0 | 0 -> 0
  • 0 | 1 -> 1
  • 1 | 1 -> 1

XOR au niveau du bit (OU exclusif) ^ :

var number = 0b0110
let newNumber = number ^ 0b1010
// newNumber is equal to 0b1100

Ici, un bit donné sera 1 si et seulement si les bits à cette position des deux opérandes sont différents.

  • 0 ^ 0 -> 0
  • 0 ^ 1 -> 1
  • 1 ^ 1 -> 0

Pour toutes les opérations binaires, l’ordre des opérandes ne fait aucune différence sur le résultat.

Opérateurs personnalisés #

Swift prend en charge la création d’opérateurs personnalisés. Les nouveaux opérateurs sont déclarés au niveau global à l’aide du mot clé operator.

La structure de l’opérateur est définie par trois parties : le placement des opérandes, la priorité et l’associativité.

  1. Les modificateurs prefix, infix et postfix sont utilisés pour démarrer une déclaration d’opérateur personnalisé. Les modificateurs prefix et postfix déclarent si l’opérateur doit être avant ou après, respectivement, la valeur sur laquelle il agit. De tels opérateurs sont urnaires, comme 8 et 3++ [**][1], puisqu’ils ne peuvent agir que sur une cible. L’‘infixe’ déclare un opérateur binaire, qui agit sur les deux valeurs entre lesquelles il se trouve, comme ‘2+3’.

  2. Les opérateurs avec une priorité plus élevée sont calculés en premier. La priorité de l’opérateur par défaut est juste supérieure à ?…: (une valeur de 100 dans Swift 2.x). La priorité des opérateurs Swift standard peut être trouvée [ici][2].

  3. Associativité définit l’ordre des opérations entre opérateurs de même priorité. Les opérateurs associatifs de gauche sont calculés de gauche à droite (ordre de lecture, comme la plupart des opérateurs), tandis que les opérateurs associatifs de droite calculent de droite à gauche.

À partir de Swift 3.0, on définirait la priorité et l’associativité dans un groupe de priorité au lieu de l’opérateur lui-même, de sorte que plusieurs opérateurs puissent facilement partager la même priorité sans se référer aux numéros cryptiques. La liste des groupes de priorité standard est indiquée [ci-dessous] [2].

Les opérateurs renvoient des valeurs basées sur le code de calcul. Ce code agit comme une fonction normale, avec des paramètres spécifiant le type d’entrée et le mot-clé “return” spécifiant la valeur calculée que l’opérateur renvoie.

Voici la définition d’un opérateur exponentiel simple, puisque Swift standard n’a pas d’opérateur exponentiel.

import Foundation    

infix operator ** { associativity left precedence 170 }

func ** (num: Double, power: Double) -> Double{
    return pow(num, power)
}

L’“infixe” indique que l’opérateur “” fonctionne entre deux valeurs, telles que “92”. Étant donné que la fonction a laissé l’associativité, ‘332’ est calculé comme ‘(33)2’. La priorité de ‘170’ est supérieure à toutes les opérations Swift standard, ce qui signifie que ‘3+24’ calcule à ‘19’, malgré l’associativité à gauche de ‘’.

import Foundation 

infix operator **: BitwiseShiftPrecedence

func ** (num: Double, power: Double) -> Double {
    return pow(num, power)
}

Au lieu de spécifier explicitement la priorité et l’associativité, sur Swift 3.0, nous pourrions utiliser le groupe de priorité intégré BitwiseShiftPrecedence qui donne les valeurs correctes (identiques à <<, >>).

** : L’incrémentation et la décrémentation sont obsolètes et seront supprimées dans Swift 3.

[1] : https://github.com/apple/swift-evolution/blob/master/proposals/0004-remove-pre-post-inc-decrement.md [2] : https://www.wikiod.com/fr/swift/operateurs-avances#Priorité des opérateurs Swift standard

Opérateurs commutatifs #

Ajoutons un opérateur personnalisé pour multiplier un CGSize

func *(lhs: CGFloat, rhs: CGSize) -> CGSize{
    let height = lhs*rhs.height
    let width = lhs*rhs.width
    return CGSize(width: width, height: height)
}

Maintenant cela fonctionne

let sizeA = CGSize(height:100, width:200)    
let sizeB = 1.1 * sizeA         //=> (height: 110, width: 220)

Mais si nous essayons de faire l’opération en sens inverse, nous obtenons une erreur

let sizeC = sizeB * 20          // ERROR

Mais c’est assez simple pour ajouter:

func *(lhs: CGSize, rhs: CGFloat) -> CGSize{
    return rhs*lhs
}

Or l’opérateur est commutatif.

let sizeA = CGSize(height:100, width:200)    
let sizeB = sizeA * 1.1              //=> (height: 110, width: 220)

Opérateurs de débordement #

Le débordement fait référence à ce qui se passe lorsqu’une opération aboutit à un nombre supérieur ou inférieur à la quantité de bits désignée pour ce nombre.

En raison du fonctionnement de l’arithmétique binaire, lorsqu’un nombre devient trop grand pour ses bits, le nombre déborde jusqu’au plus petit nombre possible (pour la taille de bit) et continue ensuite à compter à partir de là. De même, lorsqu’un nombre devient trop petit, il dépasse jusqu’au plus grand nombre possible (pour sa taille en bits) et continue à décompter à partir de là.

Étant donné que ce comportement n’est pas souvent souhaité et peut entraîner de graves problèmes de sécurité, les opérateurs arithmétiques Swift +, - et * génèrent des erreurs lorsqu’une opération provoque un débordement ou un sous-dépassement. Pour autoriser explicitement le débordement et le sous-dépassement, utilisez plutôt &+, &- et &*.

var almostTooLarge = Int.max
almostTooLarge + 1 // not allowed
almostTooLarge &+ 1 // allowed, but result will be the value of Int.min

Surcharge + pour les dictionnaires #

Comme il n’existe actuellement aucun moyen simple de combiner des dictionnaires dans Swift, il peut être utile de [surcharger][1] les opérateurs + et += afin d’ajouter cette fonctionnalité à l’aide de [génériques][2].

// Combines two dictionaries together. If both dictionaries contain
// the same key, the value of the right hand side dictionary is used.
func +<K, V>(lhs: [K : V], rhs: [K : V]) -> [K : V] {
    var combined = lhs
    for (key, value) in rhs {
        combined[key] = value
    }
    return combined
}

// The mutable variant of the + overload, allowing a dictionary
// to be appended to 'in-place'.
func +=<K, V>(inout lhs: [K : V], rhs: [K : V]) {
    for (key, value) in rhs {
        lhs[key] = value
    }
}

Depuis Swift 3, inout doit être placé avant le type d’argument.

func +=<K, V>(lhs: inout [K : V], rhs: [K : V]) { ... }

Exemple d’utilisation :

let firstDict = ["hello" : "world"]
let secondDict = ["world" : "hello"]
var thirdDict = firstDict + secondDict // ["hello": "world", "world": "hello"]

thirdDict += ["hello":"bar", "baz":"qux"] // ["hello": "bar", "baz": "qux", "world": "hello"]

[1] : https://developer.apple.com/library/ios/documentation/Swift/Conceptual/Swift_Programming_Language/AdvancedOperators.html#//apple_ref/doc/uid/TP40014097-CH27-ID42 [2] : https://www.wikiod.com/fr/swift/generiques

Priorité des opérateurs Swift standard #

Les opérateurs qui se limitent plus étroitement (priorité plus élevée) sont répertoriés en premier.

| Opérateurs | Groupe de priorité (≥3.0) | Priorité | Associativité | |————|——————|—————– ———|—————| | . | | &fin; | la gauche | ?, !, ++, --, [], (), {} | (suffixe) | | | | !, ~, +, -, ++, -- | (préfixe) | | | | ~> (swift ≤2.3) | | 255 | la gauche | <<, >> | BitwiseShiftPrecedence | 160 | rien | *, /, %, &, &* | MultiplicationPrécédence | 150 | la gauche | +, -, |, ^, &+, &- | AdditionPrécédence | 140 | la gauche | ..., ..< | RangeFormationPrecedence | 135 | rien | est, comme, comme?, comme! | CastingPrécédence | 132 | la gauche | ?? | NilCoalescingPrecedence | 131 | droit | <, <=, >, >=, ==, !=, ===, !==, ~= | ComparaisonPrécédence | 130 | aucun | | && | LogicalConjunctionPrecedence | 120 | la gauche | || | LogicalDisjunctionPrecedence | 110 | la gauche | | Précédence par défaut* | | rien | ?…: | TernairePrécédence | 100 | droit | =, +=, -=, *=, /=, %=, <<=, >>=, &=, \ |=, ^= | AffectationPrécédence | 90 | à droite, affectation | -> | FunctionArrowPrecedence | | droit

  • Le groupe de priorité DefaultPrecedence est supérieur à TernaryPrecedence, mais n’est pas ordonné avec le reste des opérateurs. En dehors de ce groupe, le reste des priorités est linéaire.
