Syntaxe #

  • pour une constante dans la séquence { instructions }
  • pour une constante en séquence où condition { instructions }
  • pour la variable var dans la séquence { instructions }
  • pour _ en séquence { instructions }
  • pour le cas let constante dans la séquence { instructions }
  • pour le cas let constante dans la séquence où condition {instructions}
  • pour la variable case var dans la séquence {instructions}
  • tant que la condition { instructions }
  • répéter { instructions } tant que la condition
  • sequence.forEach(body: (Element) throws -> Void)

Boucle for-in #

La boucle for-in vous permet d’itérer sur n’importe quelle séquence.

Itérer sur une plage #

Vous pouvez itérer sur des plages semi-ouvertes et fermées :

for i in 0..<3 {
    print(i)
}

for i in 0...2 {
    print(i)
}

// Both print:
// 0
// 1
// 2

Itérer sur un tableau ou un ensemble #

let names = ["James", "Emily", "Miles"]

for name in names {
   print(name)
}

// James
// Emily
// Miles

Si vous avez besoin de l’index de chaque élément du tableau, vous pouvez utiliser la méthode [enumerate()][1] sur [SequenceType][2].

for (index, name) in names.enumerate() {
   print("The index of \(name) is \(index).")
}

// The index of James is 0.
// The index of Emily is 1.
// The index of Miles is 2.

enumerate() renvoie une séquence paresseuse contenant des paires d’éléments avec des Int consécutifs, commençant à 0. Par conséquent, avec les tableaux, ces nombres correspondront à l’index donné de chaque élément - mais cela peut ne pas être le cas avec d’autres types de collectes.

Dans Swift 3, enumerate() a été renommé [enumerated()][3] :

for (index, name) in names.enumerated() {
   print("The index of \(name) is \(index).")
}

Itérer sur un dictionnaire #

let ages = ["James": 29, "Emily": 24]

for (name, age) in ages {
    print(name, "is", age, "years old.")
}

// Emily is 24 years old.
// James is 29 years old.

Itérer en sens inverse #

Vous pouvez utiliser la méthode [reverse()][4] sur [SequenceType][2] afin d’itérer sur n’importe quelle séquence en sens inverse :

for i in (0..<3).reverse() {
    print(i)
}

for i in (0...2).reverse() {
    print(i)
}

// Both print:
// 2
// 1
// 0

let names = ["James", "Emily", "Miles"]

for name in names.reverse() {
    print(name)
}

// Miles
// Emily
// James

Dans Swift 3, reverse() a été renommé [reversed()][5] :

for i in (0..<3).reversed() {
    print(i)
}

Itérer sur des plages avec une foulée personnalisée #

En utilisant les méthodes [stride(_:_:)][7] sur [Strideable][8], vous pouvez itérer sur une plage avec une foulée personnalisée :

for i in 4.stride(to: 0, by: -2) {
    print(i)
}

// 4
// 2

for i in 4.stride(through: 0, by: -2) {
    print(i)
}

// 4
// 2
// 0

Dans Swift 3, les méthodes stride(_:_:) sur Stridable ont été remplacées par les fonctions globales [stride(_:_:_:)][9] :

for i in stride(from: 4, to: 0, by: -2) {
    print(i)
}

for i in stride(from: 4, through: 0, by: -2) {
    print(i)
}

[1] : http://swiftdoc.org/v2.2/protocol/SequenceType/#func–enumerate [2] : http://swiftdoc.org/v2.2/protocol/SequenceType/ [3] : http://swiftdoc.org/v3.0/protocol/Sequence/#func–enumerated [4] : http://swiftdoc.org/v2.2/protocol/SequenceType/#func–reverse [5] : http://swiftdoc.org/v3.0/protocol/Sequence/#func–reversed [6] : http://swiftdoc.org/v3.0/protocol/Sequence/ [7] : http://swiftdoc.org/v2.2/protocol/Strideable/#func–stride-through_by_ [8] : http://swiftdoc.org/v2.2/protocol/Strideable/ [9] : http://swiftdoc.org/v3.0/func/stride/#func-stride-t_-strideable-from_-t-to_-t-by_-t-stride

Boucle de répétition #

Semblable à la boucle while, seule l’instruction de contrôle est évaluée après la boucle. Par conséquent, la boucle s’exécutera toujours au moins une fois.

var i: Int = 0

repeat {
   print(i)
   i += 1
} while i < 3

// 0
// 1
// 2

Boucle for-in avec filtrage #

  1. Clause where

En ajoutant une clause “where”, vous pouvez restreindre les itérations à celles qui satisfont la condition donnée.

for i in 0..<5 where i % 2 == 0 {
    print(i)
}

// 0
// 2
// 4


let names = ["James", "Emily", "Miles"]

for name in names where name.characters.contains("s") {
    print(name)
}

// James
// Miles
  1. Clause cas

C’est utile lorsque vous devez parcourir uniquement les valeurs qui correspondent à un modèle :

let points = [(5, 0), (31, 0), (5, 31)]
for case (_, 0) in points {
    print("point on x-axis")
}

//point on x-axis
//point on x-axis

Vous pouvez également filtrer les valeurs facultatives et les déballer si nécessaire en ajoutant la marque ? après la constante de liaison :

let optionalNumbers = [31, 5, nil]
for case let number? in optionalNumbers {
    print(number)
}

//31    
//5

Type de séquence pour chaque bloc #

Un type conforme au protocole SequenceType peut parcourir ses éléments dans une fermeture :

collection.forEach { print($0) }

La même chose pourrait également être faite avec un paramètre nommé :

collection.forEach { item in
    print(item)
}

*Remarque : les instructions de flux de contrôle (telles que break ou continue) ne peuvent pas être utilisées dans ces blocs. Un retour peut être appelé et, s’il est appelé, renverra immédiatement le bloc pour l’itération en cours (un peu comme le ferait un continue). La prochaine itération s’exécutera alors.

let arr = [1,2,3,4]

arr.forEach {
    
    // blocks for 3 and 4 will still be called
    if $0 == 2 {
        return
    }
}

boucle while #

Une boucle “while” s’exécutera tant que la condition est vraie.

var count = 1

while count < 10 {
    print("This is the \(count) run of the loop")
    count += 1
}

Casser une boucle #

Une boucle s’exécutera tant que sa condition reste vraie, mais vous pouvez l’arrêter manuellement à l’aide du mot-clé break. Par exemple:

var peopleArray = ["John", "Nicole", "Thomas", "Richard", "Brian", "Novak", "Vick", "Amanda", "Sonya"]
var positionOfNovak = 0

for person in peopleArray {
    if person == "Novak" { break }
    positionOfNovak += 1
}

print("Novak is the element located on position [\(positionOfNovak)] in peopleArray.")
//prints out: Novak is the element located on position 5 in peopleArray. (which is true)
