décharger #

dump imprime le contenu d’un objet par réflexion (mirroring).

Vue détaillée d’un tableau :

let names = ["Joe", "Jane", "Jim", "Joyce"]
dump(names)

Impressions :

▿ 4 éléments - [0] : Joe - [1] : Jeanne - [2] : Jim - [3] : Joyce

Pour un dictionnaire :

let attributes = ["foo": 10, "bar": 33, "baz": 42]
dump(attributes)

Impressions :

▿ 3 paires clé/valeur ▿ [0] : (2 éléments)

  • .0 : barre
  • .1 : 33 ▿ [1] : (2 éléments)
  • .0 : baz
  • .1 : 42 ▿ [2] : (2 éléments)
  • .0 : foo
  • .1 : 10

dump est déclaré comme dump(_:name:indent:maxDepth:maxItems :).

Le premier paramètre n’a pas d’étiquette.

D’autres paramètres sont disponibles, comme name pour définir une étiquette pour l’objet inspecté :

dump(attributes, name: "mirroring")

Impressions :

▿ miroir : 3 paires clé/valeur ▿ [0] : (2 éléments)

  • .0 : barre
  • .1 : 33 ▿ [1] : (2 éléments)
  • .0 : baz
  • .1 : 42 ▿ [2] : (2 éléments)
  • .0 : foo
  • .1 : 10

Vous pouvez également choisir de n’imprimer qu’un certain nombre d’éléments avec maxItems:, d’analyser l’objet jusqu’à une certaine profondeur avec maxDepth: et de modifier l’indentation des objets imprimés avec indent:.

Impression de débogage #

Debug Print affiche la représentation d’instance la plus appropriée pour le débogage.

print("Hello")
debugPrint("Hello")

let dict = ["foo": 1, "bar": 2]

print(dict)
debugPrint(dict)

Rendements

>>> Hello
>>> "Hello"
>>> [foo: 1, bar: 2]
>>> ["foo": 1, "bar": 2]

Ces informations supplémentaires peuvent être très importantes, par exemple :

let wordArray = ["foo", "bar", "food, bars"]

print(wordArray)
debugPrint(wordArray)

Rendements

>>> [foo, bar, food, bars]
>>> ["foo", "bar", "food, bars"]

Remarquez comment dans la première sortie, il apparaît qu’il y a 4 éléments dans le tableau au lieu de 3. Pour des raisons comme celle-ci, il est préférable lors du débogage d’utiliser debugPrint

Mise à jour d’un débogage de classes et des valeurs d’impression #

struct Foo: Printable, DebugPrintable {
    var description: String {return "Clear description of the object"}
    var debugDescription: String {return "Helpful message for debugging"}
}

var foo = Foo()

print(foo)
debugPrint(foo)

>>> Clear description of the object
>>> Helpful message for debugging

print() contre dump() #

Beaucoup d’entre nous commencent le débogage avec un simple print(). Disons que nous avons une telle classe:

class Abc {
    let a = "aa"
    let b = "bb"
}

et nous avons une instance de Abc comme suit :

let abc = Abc()

Lorsque nous exécutons le print() sur la variable, la sortie est

App.Abc

alors que dump() affiche

App.Abc #0
- a: "aa"
- b: "bb"

Comme on le voit, dump() affiche toute la hiérarchie des classes, tandis que print() affiche simplement le nom de la classe.

Par conséquent, dump() est particulièrement utile pour le débogage de l’interface utilisateur

let view = UIView(frame: CGRect(x: 0, y: 0, width: 100, height: 100))

Avec dump(view) nous obtenons :

 - <UIView: 0x108a0cde0; frame = (0 0; 100 100); layer = <CALayer: 0x159340cb0>> #0
    - super: UIResponder
      - NSObject

Alors que print(view) nous obtenons :

<UIView: 0x108a0cde0; frame = (0 0; 100 100); layer = <CALayer: 0x159340cb0>>

Il y a plus d’informations sur la classe avec dump(), et c’est donc plus utile pour déboguer la classe elle-même.

impression contre NSLog #

Dans Swift, nous pouvons utiliser les fonctions print() et NSLog() pour imprimer quelque chose sur la console Xcode.

Mais il existe de nombreuses différences entre les fonctions print() et NSLog(), telles que :

1 TimeStamp : NSLog() imprimera l’horodatage avec la chaîne que nous lui avons transmise, mais print() n’imprimera pas l’horodatage. par exemple.

let array = [1, 2, 3, 4, 5]
print(array)
NSLog(array.description)

Production:

[1, 2, 3, 4, 5] 2017-05-31 13:14:38.582 ProjetName[2286:7473287] [1, 2, 3, 4, 5]

Il imprimera également ProjectName avec l’horodatage.

2 Uniquement String : NSLog() ne prend que String comme entrée, mais print() peut imprimer n’importe quel type d’entrée qui lui est transmis. par exemple.

let array = [1, 2, 3, 4, 5]
print(array) //prints [1, 2, 3, 4, 5]
NSLog(array) //error: Cannot convert value of type [Int] to expected argument type 'String'

3 Performance : La fonction NSLog() est très lente par rapport à la fonction print().

4 Synchronisation : NSLog() gère l’utilisation simultanée à partir d’un environnement multi-threading et imprime la sortie sans la chevaucher. Mais print() ne gérera pas de tels cas et des confusions lors de l’évaluation de la sortie.

5 Console de l’appareil : Sorties NSLog() sur la console de l’appareil également, nous pouvons voir cette sortie en connectant notre appareil à Xcode. print() n’imprimera pas la sortie sur la console de l’appareil.
