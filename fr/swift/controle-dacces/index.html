Syntaxe #

  • projet de classe privée
  • let car = Car(“Ford”, model: “Escape”) //default internal
  • genre d’énumération publique
  • fonction privée calculateMarketCap()
  • remplace la fonction interne setupView()
  • zone var privée (ensemble) = 0
  1. Remarque de base :

Vous trouverez ci-dessous les trois niveaux d’accès, de l’accès le plus élevé (le moins restrictif) à l’accès le plus bas (le plus restrictif)

L’accès public permet d’accéder aux classes, structures, variables, etc. à partir de n’importe quel fichier du modèle, mais surtout en dehors du module si le fichier externe importe le module contenant le code d’accès public. Il est courant d’utiliser l’accès public lorsque vous créez un framework.

L’accès interne permet aux fichiers uniquement avec le module des entités d’utiliser les entités. Toutes les entités ont un niveau d’accès interne par défaut (à quelques exceptions près).

L’accès privé empêche l’utilisation de l’entité en dehors de ce fichier.

  1. Remarque de sous-classement :

Une sous-classe ne peut pas avoir un accès supérieur à sa super-classe.

  1. *** Getter & Setter Remarque : ***

Si le setter de la propriété est private, le getter est interne (ce qui est la valeur par défaut). Vous pouvez également attribuer un niveau d’accès à la fois au getter et au setter. Ces principes s’appliquent également aux indices

  1. *** Remarque générale : ***

Les autres types d’entités incluent : les initialiseurs, les protocoles, les extensions, les génériques et les alias de type

Exemple de base utilisant une structure #

Dans Swift 3, il existe plusieurs niveaux d’accès. Cet exemple les utilise tous sauf open :

public struct Car {
    
    public let make: String
    let model: String //Optional keyword: will automatically be "internal"
    private let fullName: String
    fileprivate var otherName: String
    
    public init(_ make: String, model: String) {
        self.make = make
        self.model = model
        self.fullName = "\(make)\(model)"
        self.otherName = "\(model) - \(make)"
    }
}

Supposons que myCar a été initialisé comme ceci :

let myCar = Car("Apple", model: "iCar")

Car.make (public) #

print(myCar.make)

Cette impression fonctionnera partout, y compris les cibles qui importent “Car”.

Car.model (interne) #

print(myCar.model)

Cela compilera si le code est dans la même cible que Car.

Car.otherName (fichierprivé) #

print(myCar.otherName)

Cela ne fonctionnera que si le code est dans le même fichier que Car.

Car.fullName (privé) #

print(myCar.fullName)

Cela ne fonctionnera pas dans Swift 3. Les propriétés private ne sont accessibles qu’au sein de la même struct/class.

public struct Car {

    public let make: String       //public
    let model: String             //internal
    private let fullName: String! //private 

    public init(_ make: String, model model: String) {
        self.make = make
        self.model = model
        self.fullName = "\(make)\(model)"
    }
}

Si l’entité a plusieurs niveaux d’accès associés, Swift recherche le niveau d’accès le plus bas. Si une variable privée existe dans une classe publique, la variable sera toujours considérée comme privée.

Exemple de sous-classement #

public class SuperClass {
    private func secretMethod() {}
}
 
internal class SubClass: SuperClass {
    override internal func secretMethod() {
        super.secretMethod()
    }
}

Getters et Setters Exemple #

struct Square {
    private(set) var area = 0

    var side: Int = 0 {
        didSet {
            area = side*side
        }
    }
}

public struct Square {
    public private(set) var area = 0
    public var side: Int = 0 {
        didSet {
            area = side*side
        }
    }
    public init() {}
}
