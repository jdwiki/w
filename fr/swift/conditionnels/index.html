Les expressions conditionnelles, impliquant des mots clés tels que if, else if et else, permettent aux programmes Swift d’effectuer différentes actions en fonction d’une condition booléenne : Vrai ou Faux. Cette section couvre l’utilisation des conditions Swift, de la logique booléenne et des instructions ternaires.

Pour plus d’informations sur les instructions conditionnelles, consultez Le langage de programmation Swift.

Liaison facultative et clauses “where” #

Optionals doivent être déballés avant de pouvoir être utilisés dans la plupart des expressions. if let est une liaison facultative, qui réussit si la valeur facultative était pas nil :

let num: Int? = 10 // or: let num: Int? = nil

if let unwrappedNum = num {
    // num has type Int?; unwrappedNum has type Int
    print("num was not nil: \(unwrappedNum + 1)")
} else {
    print("num was nil")
}

Vous pouvez réutiliser le même nom pour la nouvelle variable liée, en masquant l’original :

// num originally has type Int?
if let num = num {
    // num has type Int inside this block
}

Combinez plusieurs liaisons facultatives avec des virgules (,) :

if let unwrappedNum = num, let unwrappedStr = str {
    // Do something with unwrappedNum & unwrappedStr
} else if let unwrappedNum = num {
    // Do something with unwrappedNum
} else {
    // num was nil
}

Appliquez d’autres contraintes après la liaison facultative à l’aide d’une clause where :

if let unwrappedNum = num where unwrappedNum % 2 == 0 {
    print("num is non-nil, and it's an even number")
}

Si vous vous sentez aventureux, entrelacez n’importe quel nombre de liaisons facultatives et de clauses “where”:

if let num = num                           // num must be non-nil
    where num % 2 == 1,                    // num must be odd
    let str = str,                         // str must be non-nil
    let firstChar = str.characters.first   // str must also be non-empty
    where firstChar != "x"                 // the first character must not be "x"
{
    // all bindings & conditions succeeded!
}

Dans Swift 3, les clauses where ont été remplacées ([SE-0099][1]) : utilisez simplement un autre , pour séparer les liaisons facultatives et les conditions booléennes.

if let unwrappedNum = num, unwrappedNum % 2 == 0 {
    print("num is non-nil, and it's an even number")
}

si let num = num, // num doit être non nul
    num % 2 == 1,                           // num must be odd
    let str = str,                          // str must be non-nil
    let firstChar = str.characters.first,   // str must also be non-empty
    firstChar != "x"                        // the first character must not be "x"
{
    // all bindings & conditions succeeded!
}

[1] : https://github.com/apple/swift-evolution/blob/master/proposals/0099-conditionclauses.md

Utilisation de la garde #

Guard vérifie une condition, et si elle est fausse, il entre dans la branche. Les branches de contrôle de garde doivent quitter leur bloc englobant via return, break ou continue (le cas échéant); ne pas le faire entraîne une erreur du compilateur. Cela a l’avantage que lorsqu’un guard est écrit, il n’est pas possible de laisser le flux continuer accidentellement (comme cela serait possible avec un if).

L’utilisation de gardes peut aider à maintenir des niveaux d’imbrication bas, ce qui améliore généralement la lisibilité du code .

func printNum(num: Int) {
    guard num == 10 else {
        print("num is not 10")
        return
    }
    print("num is 10")
}

Guard peut également vérifier s’il y a une valeur dans un [facultatif][1], puis le déballer dans la portée externe :

func printOptionalNum(num: Int?) {
    guard let unwrappedNum = num else {
        print("num does not exist")
        return
    }
    print(unwrappedNum)
}

Guard peut combiner le déballage [facultatif][1] et la vérification de l’état à l’aide du mot-clé « où » :

func printOptionalNum(num: Int?) {
guard let unwrappedNum = num, unwrappedNum == 10 else {
    print("num does not exist or is not 10")
    return
}
print(unwrappedNum)
}

[1] : https://www.wikiod.com/fr/swift/options

Conditionnelles de base : instructions if #

Une instruction if vérifie si une condition Bool est true :

let num = 10

if num == 10 {
    // Code inside this block only executes if the condition was true.
    print("num is 10")
}

let condition = num == 10   // condition's type is Bool
if condition {
    print("num is 10")
}

Les instructions if acceptent les blocs else if et else, qui peuvent tester des conditions alternatives et fournir une solution de secours :

let num = 10
if num < 10 {  // Execute the following code if the first condition is true.
    print("num is less than 10")
} else if num == 10 {  // Or, if not, check the next condition...
    print("num is 10")
} else {  // If all else fails...
    print("all other conditions were false, so num is greater than 10")
}

Les opérateurs de base tels que && et || peuvent être utilisés pour plusieurs conditions :

L’opérateur logique ET #

let num = 10
let str = "Hi"
if num == 10 && str == "Hi" {
    print("num is 10, AND str is \"Hi\"")
}

Si num == 10 était faux, la deuxième valeur ne serait pas évaluée. C’est ce qu’on appelle l’évaluation des courts-circuits.

L’opérateur logique OU #

if num == 10 || str == "Hi" {
    print("num is 10, or str is \"Hi\")
}

Si num == 10 est vrai, la seconde valeur ne sera pas évaluée.

L’opérateur logique NON #

if !str.isEmpty {
    print("str is not empty")
}

[1] : https://www.wikiod.com/fr/swift/options

Opérateur ternaire #

Les conditions peuvent également être évaluées sur une seule ligne à l’aide de l’opérateur ternaire :

Si vous vouliez déterminer le minimum et le maximum de deux variables, vous pourriez utiliser des instructions if, comme ceci :

let a = 5
let b = 10
let min: Int

if a < b {
    min = a 
} else {
    min = b 
}

let max: Int

if a > b {
    max = a 
} else {
    max = b 
}

L’opérateur conditionnel ternaire prend une condition et renvoie l’une des deux valeurs, selon que la condition était vraie ou fausse. La syntaxe est la suivante : Cela équivaut à avoir l’expression :

(<CONDITION>) ? <TRUE VALUE> : <FALSE VALUE>

Le code ci-dessus peut être réécrit en utilisant l’opérateur conditionnel ternaire comme ci-dessous :

let a = 5
let b = 10
let min = a < b ? a : b
let max = a > b ? a : b

Dans le premier exemple, la condition est a < b. Si c’est vrai, le résultat assigné à min sera de a ; si c’est faux, le résultat sera la valeur de b.

Remarque : Parce que trouver le plus grand ou le plus petit de deux nombres est une opération si courante, la bibliothèque standard Swift fournit deux fonctions à cette fin : max et min.

Opérateur de coalescence nulle #

L’opérateur de coalescence nulle <FACULTATIF> ?? <DEFAULT VALUE> déballe le <OPTIONAL> s’il contient une valeur, ou renvoie <DEFAULT VALUE> si est nil. <OPTIONAL> est toujours d’un type facultatif. <DEFAULT VALUE> doit correspondre au type stocké dans <OPTIONAL>.

L’opérateur de coalescence nulle est un raccourci pour le code ci-dessous qui utilise un opérateur ternaire :

a != nil ? a! : b

cela peut être vérifié par le code ci-dessous:

(a ?? b) == (a != nil ? a! : b) // ouputs true

Le temps d’un exemple

let defaultSpeed:String = "Slow"
var userEnteredSpeed:String? = nil

print(userEnteredSpeed ?? defaultSpeed) // ouputs "Slow"

userEnteredSpeed = "Fast"
print(userEnteredSpeed ?? defaultSpeed) // ouputs "Fast"
