« Une valeur facultative contient une valeur ou contient nil pour indiquer qu’une valeur est manquante »

Extrait de : Apple Inc. “Le langage de programmation Swift (édition Swift 3.1)”. iBooks. https://itun.es/us/k5SW7.l

Les cas d’utilisation facultatifs de base incluent : pour une constante (let), l’utilisation d’un facultatif dans une boucle (if-let), le déballage en toute sécurité d’une valeur facultative dans une méthode (guard-let) et dans le cadre de boucles de commutation (case-let ), par défaut à une valeur si nulle, en utilisant l’opérateur de coalescence (??)

Syntaxe #

  • var nomfacultatif : typefacultatif ? // déclare un type optionnel, par défaut à nil
  • var nomfacultatif : typefacultatif ? = valeur // déclare une option avec une valeur
  • var nomfacultatif : typefacultatif ! // déclarer une option implicitement déballée
  • optionnel! // force le déballage d’une option

Pour plus d’informations sur les options, voir [Le langage de programmation Swift][1].

[1] : https://developer.apple.com/library/ios/documentation/Swift/Conceptual/Swift_Programming_Language/TheBasics.html#//apple_ref/doc/uid/TP40014097-CH5-ID330

Types d’Options #

Les options sont un type d’énumération générique qui agit comme un wrapper. Ce wrapper permet à une variable d’avoir l’un des deux états suivants : la valeur du type défini par l’utilisateur ou “nil”, qui représente l’absence de valeur.

Cette capacité est particulièrement importante dans Swift car l’un des objectifs de conception déclarés du langage est de bien fonctionner avec les frameworks d’Apple. Beaucoup (la plupart) des frameworks d’Apple utilisent “nil” en raison de sa facilité d’utilisation et de son importance pour les modèles de programmation et la conception d’API dans Objective-C.

Dans Swift, pour qu’une variable ait une valeur “nil”, elle doit être facultative. Les options peuvent être créées en ajoutant un ! ou un ? au type de variable. Par exemple, pour rendre un Int facultatif, vous pouvez utiliser

var numberOne: Int! = nil
var numberTwo: Int? = nil

Les options ? doivent être explicitement déballées et doivent être utilisées si vous n’êtes pas certain que la variable aura une valeur lorsque vous y accéderez. Par exemple, lorsque vous transformez une chaîne en Int, le résultat est un Int? facultatif, car nil sera renvoyé si la chaîne n’est pas un nombre valide

let str1 = "42"
let num1: Int? = Int(str1) // 42

let str2 = "Hello, World!"
let num2: Int? = Int(str2) // nil

Les options ! sont automatiquement déballées et ne doivent être *utilisées que lorsque vous êtes certain que la variable aura une valeur lorsque vous y accéderez. Par exemple, une variable globale UIButton! qui est initialisée dans viewDidLoad()

//myButton will not be accessed until viewDidLoad is called,
//so a ! optional can be used here
var myButton: UIButton!

override func viewDidLoad(){
    self.myButton = UIButton(frame: self.view.frame)
    self.myButton.backgroundColor = UIColor.redColor()
    self.view.addSubview(self.myButton)
}

Déballage d’un facultatif #

Pour accéder à la valeur d’un Optional, il doit être déballé.

Vous pouvez déballer conditionnellement un Facultatif en utilisant une liaison facultative et forcer le déballage d’un Facultatif en utilisant l’opérateur !.

Le déballage conditionnel demande effectivement “Cette variable a-t-elle une valeur?” tandis que le déballage forcé dit “Cette variable a une valeur!”.

Si vous forcez le déballage d’une variable qui est nil, votre programme lancera un nil trouvé de manière inattendue lors du déballage d’une exception facultative et se bloquera, vous devez donc examiner attentivement si l’utilisation de ! est appropriée.

var text: String? = nil
var unwrapped: String = text! //crashes with "unexpectedly found nil while unwrapping an Optional value"

Pour un déballage en toute sécurité, vous pouvez utiliser une instruction if-let, qui ne lèvera pas d’exception ou ne plantera pas si la valeur encapsulée est nil :

var number: Int?
if let unwrappedNumber = number {       // Has `number` been assigned a value?
    print("number: \(unwrappedNumber)") // Will not enter this line
} else {
    print("number was not assigned a value")
}

Ou, [une déclaration de garde] [1] :

var number: Int?
guard let unwrappedNumber = number else {
    return
}
print("number: \(unwrappedNumber)")

Notez que la portée de la variable unwrappedNumber est à l’intérieur de l’instruction if-let et à l’extérieur du bloc guard.

Vous pouvez chaîner le déballage de nombreuses options, ceci est principalement utile dans les cas où votre code nécessite plus de variables pour s’exécuter correctement :

var firstName:String?
var lastName:String?

if let fn = firstName, let ln = lastName {
    print("\(fn) + \(ln)")//pay attention that the condition will be true only if both optionals are not nil.
}

Notez que toutes les variables doivent être désencapsulées pour réussir le test, sinon vous n’auriez aucun moyen de déterminer quelles variables ont été désencapsulées et lesquelles ne l’ont pas été.

Vous pouvez enchaîner des instructions conditionnelles en utilisant vos options immédiatement après leur déballage. Cela signifie qu’il n’y a pas d’instructions if - else imbriquées !

var firstName:String? = "Bob"
var myBool:Bool? = false

if let fn = firstName, fn == "Bob", let bool = myBool, !bool {
    print("firstName is bob and myBool was false!")
}

[1] : https://www.wikiod.com/fr/swift/conditionnels#Utilisation de la garde

Opérateur de coalescence nulle #

Vous pouvez utiliser [nil coalescing operator][1] pour déballer une valeur si elle est non nulle, sinon fournissez une valeur différente :

func fallbackIfNil(str: String?) -> String {
    return str ?? "Fallback String"
}
print(fallbackIfNil("Hi")) // Prints "Hi"
print(fallbackIfNil(nil)) // Prints "Fallback String"

Cet opérateur est capable de [court-circuiter][2], c’est-à-dire que si l’opérande de gauche est non nul, l’opérande de droite ne sera pas évalué :

func someExpensiveComputation() -> String { ... }

var foo : String? = "a string"
let str = foo ?? someExpensiveComputation()

Dans cet exemple, comme foo est non nul, someExpensiveComputation() ne sera pas appelé.

Vous pouvez également enchaîner plusieurs instructions de coalescence nulle :

var foo : String?
var bar : String?

let baz = foo ?? bar ?? "fallback string"

Dans cet exemple, baz se verra attribuer la valeur non encapsulée de foo si elle est non nulle, sinon elle se verra attribuer la valeur non encapsulée de bar si elle est non nulle, sinon elle se verra attribuer la valeur de repli .

[1] : https://developer.apple.com/library/ios/documentation/Swift/Conceptual/Swift_Programming_Language/BasicOperators.html#//apple_ref/doc/uid/TP40014097-CH6-ID72 [2] : https://en.wikipedia.org/wiki/Short-circuit_evaluation

Chaînage optionnel #

Vous pouvez utiliser [Chaînage facultatif][1] pour appeler une [méthode][2], accéder à une [propriété][3] ou [indice][4] un facultatif. Cela se fait en plaçant un ? entre la variable facultative donnée et le membre donné (méthode, propriété ou indice).

struct Foo {
    func doSomething() {
        print("Hello World!")
    }
}

var foo : Foo? = Foo()

foo?.doSomething() // prints "Hello World!" as foo is non-nil

Si foo contient une valeur, doSomething() sera appelé dessus. Si foo est nil, alors rien de grave ne se produira - le code échouera simplement en silence et continuera à s’exécuter.

var foo : Foo? = nil

foo?.doSomething() // will not be called as foo is nil

(Ceci est similaire à l’envoi de messages à nil en Objective-C)

La raison pour laquelle le chaînage facultatif est nommé en tant que tel est que le « caractère facultatif » sera propagé à travers les membres que vous appelez/ accédez. Cela signifie que les valeurs de retour de tous les membres utilisés avec un chaînage facultatif seront facultatives, qu’elles soient typées comme facultatives ou non.

struct Foo {
    var bar : Int
    func doSomething() { ... }
}

let foo : Foo? = Foo(bar: 5)
print(foo?.bar) // Optional(5)

Ici, foo?.bar renvoie un Int? même si bar n’est pas facultatif, car foo lui-même est facultatif.

Au fur et à mesure que l’optionalité est propagée, les méthodes renvoyant Void renverront Void? lorsqu’elles sont appelées avec un chaînage facultatif. Cela peut être utile pour déterminer si la méthode a été appelée ou non (et donc si l’option a une valeur).

let foo : Foo? = Foo()

if foo?.doSomething() != nil {
    print("foo is non-nil, and doSomething() was called")
} else {
    print("foo is nil, therefore doSomething() wasn't called")
}

Ici, nous comparons la valeur de retour Void? avec nil afin de déterminer si la méthode a été appelée (et donc si foo est non-nil).

[1] : https://developer.apple.com/library/ios/documentation/Swift/Conceptual/Swift_Programming_Language/OptionalChaining.html#//apple_ref/doc/uid/TP40014097-CH21-ID245 [2] : https://developer.apple.com/library/ios/documentation/Swift/Conceptual/Swift_Programming_Language/Methods.html#//apple_ref/doc/uid/TP40014097-CH15-ID234 [3] : https://developer.apple.com/library/ios/documentation/Swift/Conceptual/Swift_Programming_Language/Properties.html#//apple_ref/doc/uid/TP40014097-CH14-ID254 [4] : https://developer.apple.com/library/ios/documentation/Swift/Conceptual/Swift_Programming_Language/Subscripts.html#//apple_ref/doc/uid/TP40014097-CH16-ID305

Présentation - Pourquoi des options ? #

Souvent, lors de la programmation, il est nécessaire de faire une distinction entre une variable qui a une valeur et une autre qui n’en a pas. Pour les types de référence, tels que les pointeurs C, une valeur spéciale telle que “null” peut être utilisée pour indiquer que la variable n’a pas de valeur. Pour les types intrinsèques, comme un entier, c’est plus difficile. Une valeur désignée, telle que -1 peut être utilisée, mais cela dépend de l’interprétation de la valeur. Il élimine également cette valeur “spéciale” de l’utilisation normale.

Pour résoudre ce problème, Swift permet à n’importe quelle variable d’être déclarée en option. Ceci est indiqué par l’utilisation d’un ? ou ! après le type (Voir [Types d’options][1])

Par exemple,

var possiblyInt: Int?

déclare une variable qui peut contenir ou non une valeur entière.

La valeur spéciale “nil” indique qu’aucune valeur n’est actuellement affectée à cette variable.

possiblyInt = 5      // PossiblyInt is now 5
possiblyInt = nil    // PossiblyInt is now unassigned

’nil’ peut également être utilisé pour tester une valeur assignée :

if possiblyInt != nil {
    print("possiblyInt has the value \(possiblyInt!)")
}

Notez l’utilisation de ! dans l’instruction print pour [unwrap][2] la valeur facultative.

Comme exemple d’utilisation courante des options, considérons une fonction qui renvoie un entier à partir d’une chaîne contenant des chiffres ; Il est possible que la chaîne contienne des caractères non numériques ou même qu’elle soit vide.

Comment une fonction qui renvoie un simple “Int” peut-elle indiquer un échec ? Il ne peut pas le faire en renvoyant une valeur spécifique car cela empêcherait cette valeur d’être analysée à partir de la chaîne.

var someInt
someInt = parseInt("not an integer") // How would this function indicate failure?

Dans Swift, cependant, cette fonction peut simplement renvoyer un Int facultatif. Ensuite, l’échec est indiqué par la valeur de retour ’nil'.

var someInt?
someInt = parseInt("not an integer")  // This function returns nil if parsing fails
if someInt == nil {
    print("That isn't a valid integer")
}
