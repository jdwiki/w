Les structures sont des types valeur #

Contrairement aux classes, qui sont passées par référence, les structures sont passées par copie :

first = "Hello"
second = first
first += " World!"
// first == "Hello World!"
// second == "Hello"

String est une structure, elle est donc copiée lors de l’affectation.

Les structures ne peuvent pas non plus être comparées à l’aide de l’opérateur d’identité :

window0 === window1 // works because a window is a class instance
"hello" === "hello" // error: binary operator '===' cannot be applied to two 'String' operands

Toutes les deux instances de structure sont considérées comme identiques si elles sont comparables.

Collectivement, ces traits qui différencient les structures des classes sont ce qui fait que les structures valorisent les types.

Bases des structures #

struct Repository {
    let identifier: Int
    let name: String
    var description: String?
}

Cela définit une structure Repository avec trois propriétés stockées, un entier identifier, une chaîne name et une chaîne facultative description. identifier et name sont des constantes, car ils ont été déclarés à l’aide du mot-clé let. Une fois définis lors de l’initialisation, ils ne peuvent pas être modifiés. La description est une variable. La modifier met à jour la valeur de la structure.

Les types de structure reçoivent automatiquement un initialiseur membre s’ils ne définissent aucun de leurs propres initialiseurs personnalisés. La structure reçoit un initialiseur membre même si elle a stocké des propriétés qui n’ont pas de valeurs par défaut.

Repository contient trois propriétés stockées dont seule description a une valeur par défaut (nil). De plus, il ne définit aucun initialiseur propre, il reçoit donc gratuitement un initialiseur membre:

let newRepository = Repository(identifier: 0, name: "New Repository", description: "Brand New Repository")

Mutation d’une structure #

Une méthode d’une structure qui modifie la valeur de la structure elle-même doit être préfixée avec le mot-clé mutating

struct Counter {
    private var value = 0
    
    mutating func next() {
        value += 1
    }
}

Quand pouvez-vous utiliser des méthodes de mutation #

Les méthodes mutating ne sont disponibles que sur les valeurs de structure à l’intérieur des variables.

var counter = Counter()
counter.next()

Quand vous ne pouvez PAS utiliser les méthodes de mutation #

D’autre part, les méthodes de “mutation” ne sont PAS disponibles sur les valeurs de structure à l’intérieur des constantes

let counter = Counter()
counter.next()
//  error: cannot use mutating member on immutable value: 'counter' is a 'let' constant

Les structures ne peuvent pas hériter #

Contrairement aux classes, les structures ne peuvent pas hériter :

class MyView: NSView { }  // works

struct MyInt: Int { } // error: inheritance from non-protocol type 'Int'

Les structures peuvent cependant adopter des protocoles :

struct Vector: Hashable { ... }  // works

Accéder aux membres de la structure #

Dans Swift, les structures utilisent une simple “syntaxe à points” pour accéder à leurs membres.

Par exemple:

struct DeliveryRange {
  var range: Double
  let center: Location
}
let storeLocation = Location(latitude: 44.9871,
                             longitude: -93.2758)
var pizzaRange = DeliveryRange(range: 200,
                               center: storeLocation)

Vous pouvez accéder (imprimer) à la gamme comme ceci :

    print(pizzaRange.range) // 200

Vous pouvez même accéder aux membres des membres en utilisant la syntaxe à points :

 print(pizzaRange.center.latitude) // 44.9871

De la même manière que vous pouvez lire des valeurs avec la syntaxe à points, vous pouvez également les affecter.

pizzaRange.range = 250
