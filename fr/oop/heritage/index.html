Héritage - Définition #

L’héritage est l’un des principaux concepts de la Programmation Orientée Objet (POO). En utilisant l’héritage, nous pouvons modéliser correctement un problème et réduire le nombre de lignes à écrire. Voyons l’héritage en utilisant un exemple populaire.

Considérez que vous devez modéliser le règne animal (le règne animal simplifié, bien sûr. Biologistes, pardonnez-moi) en utilisant la POO. Il existe de nombreuses espèces d’animaux, certaines ont des caractéristiques uniques, tandis que d’autres partagent les mêmes caractéristiques.

Il existe de grandes familles d’animaux. Disons, Mammifères, Reptiles.

Ensuite, nous avons des enfants de ces familles. À titre d’exemple,

  • “Chat”, “Chien” et “Lion” sont des mammifères.
  • ‘Cobra’ et ‘Python’ sont des reptiles.

Chaque animal partage certaines caractéristiques de base comme “manger”, “boire”, “bouger”. Par conséquent, nous pouvons dire que nous pouvons avoir un parent appelé “Animal” dont ils peuvent hériter de ces fonctionnalités de base.

Ensuite, ces familles partagent également certaines caractéristiques. Par exemple, les reptiles utilisent crawling pour se déplacer. Chaque mammifère est « nourri de lait » aux premiers stades de sa vie.

Ensuite, il y a des caractéristiques uniques pour chaque animal.

Demandez-vous si nous devons créer ces espèces animales séparément. Nous devons écrire le même code encore et encore dans chaque espèce animale. Au lieu de cela, nous utilisons l’héritage. Nous pouvons modéliser le règne animal comme suit :

  • Nous pouvons avoir un objet parent appelé Animal, qui possède les caractéristiques de base de tous les animaux.
  • Objets Mammal et Reptile (bien sûr, les autres familles d’animaux aussi) avec leurs caractéristiques communes tout en héritant des caractéristiques de base de l’objet parent, Animal.
  • Objets d’espèces animales : Cat et Dog héritent de l’objet Mammal, Cobra et Python héritent de l’objet Reptile, etc.

Sous cette forme, nous pouvons réduire le code que nous écrivons, car nous n’avons pas besoin de définir les caractéristiques de base des animaux dans chaque espèce animale, car nous pouvons les définir dans l’objet Animal puis en hériter. Il en va de même pour les familles d’animaux.

Exemple d’héritage - Considérez ci-dessous deux classes #

Classe de professeur :

class Teacher {
       private String name;
       private double salary;
       private String subject;
       public Teacher (String tname)  {
           name = tname;
       }
       public String getName()  {
           return name;
       }
       private double getSalary()  {
           return salary;
       }
       private String  getSubject()  {
            return  subject;
       }
    }

Classe OfficeStaff :

class  OfficeStaff{
   private String name;
   private double salary;
   private String dept;
   public OfficeStaff (String sname)  {
      name = sname;
   }
   public String getName()  {
       return name;
   }
   private double  getSalary()  {
       return salary;
   }
   private String  getDept ()  {
       return dept;
   }
}
  1. Les deux classes partagent peu de propriétés et de méthodes communes. Donc répétition de code.
  2. Créer une classe qui contient les méthodes et propriétés communes.
  3. Les classes Teacher et OfficeStaff peuvent hériter de toutes les propriétés et méthodes communes de la classe Employee ci-dessous.

Classe d’employé :

class Employee{
   private String name;
   private double salary;
   public Employee(String ename){
      name=ename;
   }
   public String getName(){
      return name;
   }
   private double getSalary(){
      return salary;
   } 
}
  1. Ajoutez-lui des méthodes et des propriétés individuelles Une fois que nous avons créé une super classe qui définit les attributs communs à un ensemble d’objets, elle peut être utilisée pour créer n’importe quel nombre de sous-classes plus spécifiques
  2. Toutes les classes similaires comme Ingénieur, Principal peuvent être générées en tant que sous-classes de la classe Employé.
  3. La classe parente est appelée super classe et la classe héritée est la sous-classe
  4. Une sous-classe est la version spécialisée d’une super classe - Elle hérite de toutes les variables d’instance et méthodes définies par la super classe et ajoute ses propres éléments uniques.
  5. Bien qu’une sous-classe comprenne tous les membres de sa super classe, elle ne peut pas accéder aux membres de la super classe qui ont été déclarés comme privés.
  6. Une variable de référence d’une super classe peut être affectée à une référence à n’importe quelle sous-classe dérivée de cette super classe c’est-à-dire Employé emp = new Teacher();
