Paramètres #

paramètre détails
e / modifier indique à Maya que vous souhaitez modifier la valeur d’une propriété existante
q / requête indique à Maya que vous souhaitez obtenir la valeur d’une propriété existante

Maya est livré avec une boîte à outils d’interface utilisateur assez complète qui comprend des fenêtres, des mises en page et une variété de contrôles. Ceci est implémenté à l’aide du framework QT en C++, mais exposé aux utilisateurs MEL et Python via le jeu de commandes Maya par défaut.

QT #

Les utilisateurs avancés peuvent étendre l’interface utilisateur Maya en utilisant C++ ou Python. Les versions Maya de 2012 à 2016 utilisent Pyside et QT4 ; Maya 2017 utilise Pyside2 et QT5. Plus de détails ici

Remarque : Une référence plus ancienne sur le Web fait référence à la boîte à outils de l’interface graphique Maya en tant que “ELF” ; c’est toujours le nom correct mais il est rarement utilisé.

Fonctions de rappel #

De nombreux widgets incluent des événements qui peuvent déclencher des fonctions de rappel lorsque l’utilisateur interagit avec le widget. Par exemple, lorsqu’un bouton est enfoncé, une case cochée ou une liste déroulante choisie, vous pouvez déclencher une fonction.

Le drapeau exact associé à ces événements dépend du widget, mais un rappel typique ressemblerait à ceci :

 def callback_fn(_ignore):
     print "button pressed"

 button = cmds.button(label='press me', command = callback_fn)

Appuyer sur le bouton affichera “bouton enfoncé” dans la fenêtre d’écoute. La plupart des widgets déclenchent des arguments lorsque leurs rappels s’activent - le bouton' par exemple inclut toujours une valeur booléenne - vous devrez donc vous assurer que le gestionnaire de rappel a la bonne signature pour aller avec le widget que vous utilisez. C'est pourquoi callback_fn()` prend un argument même s’il n’en a pas besoin.

Affectation de rappel #

Maya prend en charge deux manières différentes d’attacher des fonctions de rappel :

   # this works, but is not a great idea
   cmds.button(label = 'string reference', command = 'string_name_of_function')
   # use this form whenever possible
   cmds.button(label = 'direct function reference', command = callback_fn)

Dans le premier exemple, le rappel est affecté par une valeur de chaîne. Maya trouvera le rappel dans la portée globale de Python - qui est généralement difficile d’accès lors de l’écriture de code correctement organisé. Les rappels de nom de chaîne sont également plus lents à résoudre. Le deuxième exemple transmet la fonction Python réelle au rappel - cette forme est préférée car elle est plus rapide et, si vous n’avez pas réussi à fournir une fonction valide à la fonction de rappel, vous saurez quand l’interface utilisateur est créée au lieu de quand les widgets de l’interface utilisateur sont réellement utilisés.

Si vous souhaitez transmettre une valeur d’argument à une fonction de rappel, vous pouvez utiliser un lambda, un closure ou un functools.partial lier les arguments au rappel.

Utilisation de partial : #

 from functools import partial
 ....
 def callback_fn(myValue, _ignore):  # _ignore swallows the original button argument
     print myValue

 button = cmds.button(label='press me', command = partial(callback_fn, "fooo"))

Utilisation de lambda : #

 def callback_fn(myValue):
     print myValue

 button = cmds.button(label='press me', command = lambda _ignore: callback_fn("fooo"))
 # here the lambda needs to handle the button argument

Utilisation des fermetures #

b = cmds.button(label = 'press me')
# by defining this function when `b` exists, we can use it later
# without storing it explicitly
def get_button_label(*_):
    print "label of button", b, " is ", cmds.button(b, q=True, l=True)
cmds.button(b, e=True, c=get_button_label)

En savoir plus sur les noms de rappel de chaîne par rapport à la fonction de rappel [ici] (https://theodox.github.io/2014/maya_callbacks_cheat_sheet#.WPqom4Qs5xU.link)

Exemple d’interface utilisateur de base [Python] #

La boîte à outils Maya GUI crée une variété d’éléments d’interface utilisateur sous une forme simple et impérative. Il existe des commandes de base pour créer et modifier des widgets GUI ; les widgets sont identifiés par un nom de chaîne unique.

Toutes les commandes de l’interface graphique prennent la même forme de base : vous fournissez un type de commande et le nom de chaîne de l’objet sur lequel vous souhaitez travailler ou créer, ainsi que des drapeaux qui spécifient l’apparence ou le comportement du widget. Ainsi, par exemple, pour créer un bouton, vous utiliserez :

 cmds.button('my_button', label = 'my label')

Cela créera un nouveau bouton gui. Pour éditer le bouton, vous utiliserez la même commande avec le drapeau edit (la version courte est juste e). Vous pouvez donc modifier le libellé du bouton comme ceci :

cmds.button('my_button', e=True, label = 'a different label')

et vous pouvez interroger la valeur actuelle d’une propriété avec le drapeau query ou q :

cmds.button(`my button`, q=True, label=True)
# 'a different label'

Nommage des widgets #

Lorsque vous créez un nouveau widget avec une commande d’interface utilisateur, vous pouvez fournir le nom que vous souhaitez que le nouveau widget obtienne. Cependant, ce n’est pas garanti : Maya donnera au bouton le nom que vous avez demandé – si vous lui avez donné un caractère qu’il ne reconnaît pas ou s’il existe déjà un widget avec le même nom, vous pouvez le récupérer un nom différent. C’est toujours une bonne pratique de saisir le nom d’un nouveau widget lorsqu’il est créé pour éviter les surprises :

 my_button = cmds.button('my_button') 
 cmds.button(my_button, e=True, label = "a new label")

Création d’une fenêtre #

# create a window with a button that closes the window when clicked
window = cmds.window(title='example window')       # create the window
layout = cmds.columnLayout(adjustableColumn=True)  # add a vertical layout

def close_window(*_):
    cmds.deleteUI(window)                          # deletes the window above

button = cmds.button(label= 'press to close", command = close_window)

# show the window
cmds.showWindow(window)

Lambdas et boucles #

Les lambdas sont un raccourci utile pour connecter des comportements aux éléments de l’interface graphique.

b = cmds.button("make a cube", command = lambda _: cmds.polyCube())

Cependant, en raison de la façon dont Python capture les variables à l’intérieur des lambdas, vous pouvez obtenir des résultats inattendus si vous liez des commandes à l’aide de lambdas dans une boucle. Par exemple, ceci ressemble à des boutons qui créent des sphères de différentes tailles :

# warning: doesn't work like it looks!
for n in range(5):
    b = cmds.button("sphere size %i" % n, command = lambda _: cmds.polySphere(radius=n))

Les boutons seront étiquetés correctement mais utiliseront tous le même rayon (4) car les lambdas captureront tous cette valeur lorsque la boucle se fermera. TLDR : Si vous générez des rappels à l’intérieur d’une boucle, utilisez functools.partial ou une autre méthode pour capturer des valeurs - les lambdas ne fonctionnent pas pour cette application.
