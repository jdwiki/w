Création de votre premier modèle #

  • Sélectionnez le fichier .xcdatamodeld. Vous remarquerez que vous n’avez pas entités. Vous devrez en créer un vous-même. En bas de Xcode, vous remarquerez un bouton indiquant “Ajouter une entité”, cliquez dessus et vous aurez une nouvelle entité avec laquelle travailler sur le projet.

[![Modèle de données][3]][3]

  • Dans cette étape, il y a quelques points à mentionner. La première est que vous avez changé le nom de votre entité ici, l’entité reçoit le nom “Personne”. Sous la section des attributs, vous ajoutez les attributs à votre modèle. Ce bouton est un plus situé en bas de la section. Vous pouvez ajouter tous les attributs pertinents pour votre application. Par exemple, si vous créez une application de répertoire de contacts, vous n’avez pas besoin de faire en sorte que le modèle d’une personne contienne un attribut de Arms avec le type Boolean. Vous devez le garder lié à votre application. Par exemple, ajoutez un attribut telephone de type Integer ou si vous aimez String. Vous avez le choix entre plusieurs options de types allant de “Booléens” à “Dates” et plus encore.

[![Création de votre modèle][4]][4]

  • Une fois que vous êtes satisfait de votre modèle, vous pouvez créer la sous-classe NSManagedObject.

[![Créer NSManagedObjectSubclass][5]][5]

[![Sélection du modèle de données][6]][6]

[![Sélection de l’entité à gérer][7]][7]

  • Ici, nous pouvons voir deux choses importantes. Tout d’abord, cette personne (identique au nom de votre entité) est une sous-classe de NSManagedObject. Ceci est important pour votre Core Data Stack. Deuxièmement, vous devez créer deux fichiers, une classe Person.swift et une extension de Person appelée Person+CoreDataProperites.swift.

  • Tous les ajouts doivent être effectués dans Person.swift, car si jamais vous modifiez votre modèle et relancez le générateur de classe, il écrasera tout dans Person+CoreDataProperties.swift.
