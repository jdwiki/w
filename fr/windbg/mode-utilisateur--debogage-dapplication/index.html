Commandes importantes #

Documenter votre travail #

Rappelez-vous ce que vous avez fait et conservez les sorties longues qui ne peuvent pas être conservées dans le tampon de WinDbg. Il est toujours bon d’avoir un journal disponible pour reproduire les étapes de débogage, par ex. pour poser des questions sur Stack Overflow.

Commande Objectif
.logopen créer un fichier journal
.logclose fermer le fichier journal
.dump enregistrer le fichier de vidage sur incident (instantané de la session de débogage en cours)

Travailler avec des symboles #

Sans ou avec des symboles incorrects, vous pouvez recevoir des informations erronées et être induit en erreur. Assurez-vous de bien connaître ces commandes avant de commencer à travailler dans WinDbg. Voir aussi Comment configurer des symboles dans WinDbg.

Commande Objectif
.symfix définir ou ajouter des symboles au chemin officiel des symboles Microsoft
.sympath définir ou ajouter des symboles propres ou tiers
.reload recharger les symboles
.sympt définir les options de gestion des symboles
!sym chargement du symbole de contrôle
x examiner les symboles
ln liste les symboles les plus proches

Analyse des collisions #

Découvrez ce qui s’est passé (dans les vidages sur incident) et comment gérer les événements (dans le débogage en direct).

Commande Objectif
.exr afficher l’enregistrement d’exception
.lastevent afficher le dernier événement
sx définir la gestion des exceptions
!analyser analyser un crash ou un blocage
!avrf vérificateur d’applications

L’environnement #

Vérifiez le nom du processus et les informations de version.

Commande Objectif
| (tuyau) informations sur le processus
lm liste des modules

Threads, piles d’appels, registres et mémoire #

Inspectez les détails.

Commande Objectif
~ liste de fils
r registres
k pile d’appels
d* mémoire d’affichage
e* modifier la mémoire
s mémoire de recherche
.formats convertir entre les formats numériques
? évaluer l’expression
u* démonter
a assembler
!adresse informations sur la mémoire

Contrôler la cible #

Dans le débogage en direct, prenez le contrôle de l’exécution.

Commande Objectif
g aller / continuer
gu monter
p une seule étape
t trace (registres d’étape et de sortie uniques)
bp définir un point d’arrêt
bl liste des points d’arrêt

Travailler avec des extensions #

Les extensions peuvent fournir des avantages et des améliorations significatifs.

Commande Objectif
.load charger l’extension (chemin complet)
.loadby charger l’extension relative au module
.chaîne afficher les extensions chargées
.unload décharger l’extension

Arrêter le débogage #

Commande Objectif
q quitter et mettre fin à l’application
qd se détacher et quitter

Attacher et détacher #

Commande Objectif
.tlist liste de processus
.attachez attacher au processus
.create créer un processus et attacher
.childdbg définir le comportement de débogage du processus enfant
.detach se détacher d’un processus
.kill tuer un processus
.restart redémarrer le processus

Comportement de WinDbg #

Commande Objectif
.prefer_dml définir la gestion du langage de balisage du débogueur
.effmach changer le bitness

Commandes d’utilisabilité #

Commande Objectif
.cmdtree Charge un fichier texte avec des commandes prédéfinies dans une fenêtre séparée

Obtenir de l’aide #

Commande Objectif
.hh Affiche le manuel d’aide pour les commandes WinDbg

Créer une fenêtre de commande personnalisée dans Windbg #

La commande .cmdtree permet d’ouvrir un fichier .txt avec des commandes prédéfinies que vous pouvez simplement double-cliquer pour exécuter.

Comment créer un fichier de commandes

Créer le fichier à l’aide de ce modèle

windbg ANSI Command Tree 1.0    
title {"Window title"}
body
{"Group Heading"}
 {"Name of command to display"} {"command"}
 {"Name of command to display"} {"command"}
{"Group Heading"}
 {"Name of command to display"} {"command"}

** Choses à prendre en compte **

  1. Le format du modèle doit être suivi avec précision pour ouvrir le fichier dans Windbg.
  2. La nouvelle ligne est requise après chaque {Group Heading}.
  3. Chaque paire {Nom de la commande à afficher} {commande} doit être sur une seule ligne et doit être suivie d’une nouvelle ligne.

Exemple de fichier de commandes personnalisé

windbg ANSI Command Tree 1.0    
title {"Your title goes here"}
body
{"Basic commands"}
 {"Show CLR Version"} {"lmv m clr"}
 {"Load SOS from CLR"} {".loadby sos clr "}
{"Symbols"}
 {"Load my symbols"} {".sympath+ "c:\DebugSymbols" ; .reload"}

Comment ouvrir l’interface utilisateur de commande à partir de la fenêtre de commande

Exécutez .cmdtree <chemin de votre fichier .txt> pour ouvrir la fenêtre. Vous verrez une fenêtre comme celle-ci
