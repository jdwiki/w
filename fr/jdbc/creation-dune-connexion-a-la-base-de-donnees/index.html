Syntaxe #

  • DB_URL = “jdbc:DBMS://DB_HOST:DB_PORT/DB_NAME”

  • SGBD : Data Base Driver Manager, cela peut être n’importe quel SGBD (mysql, oracle, postgresql, sqlite, …), exemple de mysql : “com.mysql.jdbc.Driver”

  • DB_HOST : votre hébergeur de base de données, l’adresse IP de votre base de données exemple : 10.6.0.1, la valeur par défaut est localhost ou 127.0.0.1

  • DB_PORT : port de la base de données, chaque SGBD a un port par défaut, exemple mysql=3306, postegesql=5432

  • DB_NAME : le nom de votre base de données

  • Pour vous connecter, vous devez obtenir une référence à l’objet de classe,

  • Class.forName(DRIVER);

  • Et pour vous connecter à la base de données, vous devez créer une connexion

  • java.sql.Connection avec = DriverManager.getConnection(DB_URL, DB_USER_NAME, DB_PASSWORD );

  • DB_USER_NAME : le nom d’utilisateur de votre base de données

  • DB_PASSWORD : le mot de passe de votre base de données

Création d’une connexion à MySQL #

Pour vous connecter à MySQL, vous devez utiliser le pilote MySQL Connector/J. Vous pouvez le télécharger depuis http://dev.mysql.com/downloads/connector/j/ ou vous pouvez utiliser Maven :

<dependency>
    <groupId>mysql</groupId>
    <artifactId>mysql-connector-java</artifactId>
    <version>5.1.39</version>
</dependency>

L’[URL JDBC de base pour MySQL][1] est :

jdbc:mysql://<hostname>[:<port>]/<database>[?<propertyName>=<propertyValue>[&<propertyName>=<propertyValue>]...]

Où:

Clé Descriptif Exemple
<nom d'hôte> Nom d’hôte du serveur MySQL localhost
<port> Port du serveur MySQL (optionnel, par défaut : 3306) 3306
<base de données> Nom de la base de données foobar
<Nompropriété> Nom d’une [propriété de connexion][1] useCompression
<valeurpropriété> Valeur d’une propriété de connexion vrai

L’URL prise en charge est plus complexe que celle indiquée ci-dessus, mais cela suffit pour la plupart des besoins “normaux”.

Pour vous connecter, utilisez :

try (Connection connection = DriverManager.getConnection(
        "jdbc:mysql://localhost/foobardb", "peter", "nicepassword")) {
    // do something with connection
}

Pour les anciennes versions de Java/JDBC :

// Load the MySQL Connector/J driver
Class.forName("com.mysql.jdbc.Driver");

Connection connection = DriverManager.getConnection(
        "jdbc:mysql://localhost/foobardb", "peter", "nicepassword");
try {
    // do something with connection
} finally {
  // explicitly close connection
  connection.close();
}

[1] : http://dev.mysql.com/doc/connector-j/5.1/en/connector-j-reference-configuration-properties.html

Connexion à une base de données Microsoft Access avec UCanAccess #

UCanAccess est un pilote Java JDBC pur qui nous permet de lire et d’écrire dans des bases de données Access sans utiliser ODBC. Il utilise deux autres packages, Jackcess et HSQLDB, pour effectuer ces tâches.

Une fois qu’il a été configuré*, nous pouvons travailler avec des données dans des fichiers .accdb et .mdb en utilisant un code comme celui-ci :

import java.sql.*;

Connection conn=DriverManager.getConnection("jdbc:ucanaccess://C:/__tmp/test/zzz.accdb");
Statement s = conn.createStatement();
ResultSet rs = s.executeQuery("SELECT [LastName] FROM [Clients]");
while (rs.next()) {
    System.out.println(rs.getString(1));
}

*Pour plus de détails, consultez la question suivante :

http://stackoverflow.com/q/21955256/2144390

Création d’une connexion à l’aide de java.sql.DriverManager #

Pour vous connecter à l’aide de [java.sql.DriverManager][1], vous avez besoin d’une URL JDBC pour vous connecter à votre base de données. Les URL JDBC sont spécifiques à la base de données, mais elles sont toutes de la forme

jdbc:<subprotocol>:<subname>

Où <subprotocol> identifie le pilote ou la base de données (par exemple postgresql, mysql, firebirdsql, etc.) et <subname> est spécifique au sous-protocole.

Vous devez vérifier la documentation de votre base de données et du pilote JDBC pour le sous-protocole d’URL spécifique et le format de votre pilote.

Un exemple simple pour créer une connexion à une base de données avec l’url jdbc:somedb://localhost/foobar :

try (Connection connection = DriverManager.getConnection(
        "jdbc:somedb://localhost/foobar", "anna", "supersecretpassword")) {
    // do something with connection
}

Nous utilisons ici un [try-with-resources][3] afin que la connexion soit automatiquement fermée lorsque nous en avons terminé, même si des exceptions se produisent.

Sur Java 6 (JDBC 4.0) et versions antérieures, l’essai avec ressources n’est pas disponible. Dans ces versions, vous devez utiliser un bloc finally pour fermer explicitement une connexion :

Connection connection = DriverManager.getConnection(
        "jdbc:somedb://localhost/foobar", "anna", "supersecretpassword");
try {
    // do something with connection
} finally {
    // explicitly close connection
    connection.close();
}

JDBC 4.0 (Java 6) a introduit le concept de chargement automatique du pilote. Si vous utilisez Java 5 ou une version antérieure, ou un ancien pilote JDBC qui n’implémente pas la prise en charge de JDBC 4, vous devrez charger explicitement le ou les pilotes :

Class.forName("org.example.somedb.jdbc.Driver");

Cette ligne doit apparaître (au moins) une fois dans votre programme, avant toute connexion.

Même en Java 6 et supérieur avec un JDBC 4.0, il peut être nécessaire de charger explicitement un pilote : par exemple dans les applications Web lorsque le pilote n’est pas chargé dans le conteneur, mais dans le cadre de l’application Web.

Vous pouvez également fournir un objet “Propriétés” pour vous connecter :

Properties props = new Properties();
props.setProperty("user", "anna");
props.setProperty("password", "supersecretpassword");
// other, database specific, properties
try (Connection connection = DriverManager.getConnection(
        "jdbc:somedb://localhost/foobar", props)) {
    // do something with connection
}

Ou même sans propriétés, par exemple si la base de données n’a pas besoin de nom d’utilisateur et de mot de passe :

try (Connection connection = DriverManager.getConnection(
        "jdbc:somedb://localhost/foobar")) {
    // do something with connection
}

[1] : https://docs.oracle.com/javase/8/docs/api/java/sql/DriverManager.html [3] : https://www.wikiod.com/fr/java/exceptions-et-gestion-des-exceptions#L’instruction try-with-resources

Connexion Oracle JDBC #

Chauffeur: #

  • 12c R1
  • 11g R2

(Remarque : le pilote n’est pas inclus dans Maven Central !)

Initialisation de la classe de pilote : #

    Class.forName("oracle.jdbc.driver.OracleDriver");

URL de connexion #

Ancien format, avec SID

"jdbc:oracle:thin:@<hostname>:<port>:<SID>"

Format plus récent, avec le nom du service

"jdbc:oracle:thin:@//<hostname>:<port>/<servicename>"

Tnsnames comme entrée

"jdbc:oracle:thin:@(DESCRIPTION=(ADDRESS=(PROTOCOL=TCPS)(HOST=<hostname>)(PORT=<port>))"
    +"(CONNECT_DATA=(SERVICE_NAME=<servicename>)))"

Chaîne de connexion du cluster RAC pour le basculement

"jdbc:oracle:thin:@(DESCRIPTION=(ADDRESS_LIST=(LOAD_BALANCE=OFF)(FAILOVER=ON)"
    +"(ADDRESS=(PROTOCOL=TCP)(HOST=<hostname1>)(PORT=<port1>))"
    +"(ADDRESS=(PROTOCOL=TCP)(HOST=<hostname2>)(PORT=<port2>)))"
    +"(CONNECT_DATA=SERVICE_NAME=<servicename>)(SERVER=DEDICATED)))"

Exemple #

connection = DriverManager.getConnection("jdbc:oracle:thin:@localhost:1521:orcl", "HR", "HRPASS");
