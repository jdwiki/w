jQuery gère les événements en interne via la fonction [addEventListener][1]. Cela signifie qu’il est parfaitement légal d’avoir plus d’une fonction liée au même événement pour le même élément DOM.

[1] : https://developer.mozilla.org/en-US/docs/Web/API/EventTarget/addEventListener

Événements délégués #

Commençons par l’exemple. Voici un exemple HTML très simple.

Exemple HTML

<html>
    <head>
    </head>
    <body>
        <ul>
            <li>
                <a href="some_url/">Link 1</a>
            </li>
            <li>
                <a href="some_url/">Link 2</a>
            </li>
            <li>
                <a href="some_url/">Link 3</a>
            </li>
        </ul>
    </body>
</html>

Le problème

Maintenant, dans cet exemple, nous voulons ajouter un écouteur d’événement à tous les éléments <a>. Le problème est que la liste dans cet exemple est dynamique. Les éléments <li> sont ajoutés et supprimés au fil du temps. Cependant, la page ne s’actualise pas entre les modifications, ce qui nous permettrait d’utiliser de simples écouteurs d’événements de clic pour les objets de lien (c’est-à-dire $('a').click()).

Le problème que nous avons est de savoir comment ajouter des événements aux éléments <a> qui vont et viennent.

Informations générales - Propagation des événements

Les événements délégués ne sont possibles qu’en raison de la propagation des événements (souvent appelée diffusion d’événements). Chaque fois qu’un événement est déclenché, il remontera jusqu’à la racine du document. Ils délèguent la gestion d’un événement à un élément ancêtre immuable, d’où le nom d’événements “délégués”.

Ainsi, dans l’exemple ci-dessus, cliquer sur le lien de l’élément <a> déclenchera l’événement ‘click’ dans ces éléments dans cet ordre :

  • un
  • li -ul
  • corps -html
  • racine du document

Solution

Sachant ce que fait le bouillonnement d’événements, nous pouvons attraper l’un des événements recherchés qui se propagent dans notre code HTML.

Un bon endroit pour l’attraper dans cet exemple est l’élément <ul>, car cet élément n’est pas dynamique :

$('ul').on('click', 'a', function () {
  console.log(this.href); // jQuery binds the event function to the targeted DOM element
                          // this way `this` refers to the anchor and not to the list
  // Whatever you want to do when link is clicked
});

En haut :

  • Nous avons ‘ul’ qui est le destinataire de cet écouteur d’événement
  • Le premier paramètre (“clic”) définit les événements que nous essayons de détecter.
  • Le deuxième paramètre (‘a’) est utilisé pour déclarer d’où l’événement doit provenir (de tous les éléments enfants sous le destinataire de cet écouteur d’événement, ul).
  • Enfin, le troisième paramètre est le code qui est exécuté si les exigences des premier et deuxième paramètres sont remplies.

En détail le fonctionnement de la solution

  1. L’utilisateur clique sur l’élément <a>
  2. Cela déclenche l’événement de clic sur l’élément <a>.
  3. L’événement commence à remonter vers la racine du document.
  4. L’événement se propage d’abord à l’élément <li> puis à l’élément <ul>.
  5. L’écouteur d’événement est exécuté car l’élément <ul> a l’écouteur d’événement attaché.
  6. L’écouteur d’événement détecte d’abord l’événement déclencheur. L’événement bouillonnant est ‘clic’ et l’auditeur a ‘clic’, c’est une passe.
  7. L’écouteur vérifie les tentatives de faire correspondre le deuxième paramètre (‘a’) à chaque élément de la chaîne de bulles. Comme le dernier élément de la chaîne est un ‘a’, cela correspond au filtre et c’est aussi une réussite.
  8. Le code du troisième paramètre est exécuté en utilisant l’élément correspondant car il s’agit de “this”. Si la fonction n’inclut pas d’appel à stopPropagation(), l’événement continuera à se propager vers le haut vers la racine (document).

Remarque : Si un ancêtre non changeant approprié n’est pas disponible/pratique, vous devez utiliser document. Par habitude, n’utilisez pas 'body' pour les raisons suivantes :

  • body a un bogue, lié au style, qui peut signifier que les événements de la souris ne lui parviennent pas. Cela dépend du navigateur et peut se produire lorsque la hauteur de corps calculée est de 0 (par exemple, lorsque tous les éléments enfants ont des positions absolues). Les événements de souris bouillonnent toujours vers document.
  • document toujours existe dans votre script, vous pouvez donc attacher des gestionnaires délégués à document en dehors d’un gestionnaire prêt pour DOM et être certain qu’ils fonctionneront toujours.

Attacher et détacher des gestionnaires d’événements #

Attacher un gestionnaire d’événements #

Depuis la version [1.7][1], jQuery a l’API d’événement .on(). De cette façon, tout [événement javascript standard][2] ou événement personnalisé peut être lié à l’élément jQuery actuellement sélectionné. Il existe des raccourcis tels que .click(), mais .on() vous donne plus d’options.

HTML #

<button id="foo">bar</button>

jQuery #

$( "#foo" ).on( "click", function() {
  console.log( $( this ).text() ); //bar
});

Détacher un gestionnaire d’événements #

Naturellement, vous avez également la possibilité de détacher des événements de vos objets jQuery. Pour ce faire, utilisez .off( events [, selector ] [, handler ] ).

HTML #

<button id="hello">hello</button>

jQuery #

$('#hello').on('click', function(){
    console.log('hello world!');
    $(this).off();
});

Lorsque vous cliquez sur le bouton $(this) fera référence à l’objet jQuery actuel et en supprimera tous les gestionnaires d’événements attachés. Vous pouvez également spécifier le gestionnaire d’événements à supprimer.

jQuery #

$('#hello').on('click', function(){
    console.log('hello world!');
    $(this).off('click');
});

$('#hello').on('mouseenter', function(){
    console.log('you are about to click');
});

Dans ce cas, l’événement mouseenter fonctionnera toujours après avoir cliqué.

[1] : http://api.jquery.com/category/version/1.7/ [2] : https://developer.mozilla.org/en-US/docs/Web/Events

Evénement de chargement de document .load() #

Si vous voulez que votre script attende qu’une certaine ressource soit chargée, comme une image ou un PDF, vous pouvez utiliser .load(), qui est un raccourci pour le raccourci pour .on( "load", handler).

HTML

<img src="image.jpeg" alt="image" id="image">

jQuery

$( "#image" ).load(function() {
  // run script
});

Événements pour répéter des éléments sans utiliser d’ID #

Problème

Il existe une série d’éléments répétitifs dans la page sur lesquels vous devez savoir sur lequel un événement s’est produit pour faire quelque chose avec cette instance spécifique.

La solution

  • Donner à tous les éléments communs une classe commune
  • Appliquer un écouteur d’événement à une classe. this à l’intérieur du gestionnaire d’événements est l’élément de sélection correspondant sur lequel l’événement s’est produit
  • Traverse vers le conteneur externe le plus répétitif pour cette instance en commençant par this
  • Utilisez find() dans ce conteneur pour isoler d’autres éléments spécifiques à cette instance

HTML

<div class="item-wrapper" data-item_id="346">
   <div class="item"><span class="person">Fred</span></div>
   <div class="item-toolbar">
      <button class="delete">Delete</button>
   </div>   
</div>
<div class="item-wrapper" data-item_id="393">
   <div clss="item"><span class="person">Wilma</span></div>
   <div class="item-toolbar">
      <button class="delete">Delete</button>
   </div>   
</div>

jQuery

$(function() {
  $('.delete').on('click', function() {
    // "this" is element event occured on
    var $btn = $(this);
    // traverse to wrapper container
    var $itemWrap = $btn.closest('.item-wrapper');
    // look within wrapper to get person for this button instance
    var person = $itemWrap.find('.person').text();
    // send delete to server and remove from page on success of ajax
    $.post('url/string', { id: $itemWrap.data('item_id')}).done(function(response) {
      $itemWrap.remove()
    }).fail(function() {
      alert('Ooops, not deleted at server');
    });
  });
});

événement d’origine #

Parfois, certaines propriétés ne sont pas disponibles dans l’événement jQuery. Pour accéder aux propriétés sous-jacentes, utilisez Event.originalEvent

Obtenir la direction de défilement #

$(document).on("wheel",function(e){
    console.log(e.originalEvent.deltaY)
    // Returns a value between -100 and 100 depending on the direction you are scrolling
})

Activation et désactivation d’événements spécifiques via jQuery. (auditeurs nommés) #

Parfois, vous souhaitez désactiver tous les écouteurs précédemment enregistrés.

//Adding a normal click handler
$(document).on("click",function(){
    console.log("Document Clicked 1")
});
//Adding another click handler
$(document).on("click",function(){
    console.log("Document Clicked 2")
});
//Removing all registered handlers.
$(document).off("click")

Un problème avec cette méthode est que TOUS les écouteurs liés à document par d’autres plugins, etc. seraient également supprimés.

Plus souvent qu’autrement, nous voulons détacher tous les auditeurs attachés uniquement par nous.

Pour ce faire, nous pouvons lier des auditeurs nommés comme,

//Add named event listener.
$(document).on("click.mymodule",function(){
    console.log("Document Clicked 1")
});
$(document).on("click.mymodule",function(){
    console.log("Document Clicked 2")
});

//Remove named event listener.
$(document).off("click.mymodule");

Cela garantit que tout autre écouteur de clic n’est pas modifié par inadvertance.
