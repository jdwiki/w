Dans le cadre du mouvement [Serverless][1], AWS a récemment annoncé la prise en charge de C# pour les fonctions AWS Lambda. Dans cet article, je vais vous montrer comment développer, installer et appeler une simple fonction C# AWS Lambda à l’aide de Visual Studio 2015.

Avant de commencer cet exemple, accédez d’abord à la section Remarques de ce document et assurez-vous que tous les composants requis sont installés.

[1] : https://en.wikipedia.org/wiki/Serverless_computing

Installer les composants requis #

Au moment d’écrire ces lignes (janvier 2017), les composants suivants devront être installés sur votre machine dans cet ordre. La machine que j’ai utilisée avait Windows 10 Home installé.

  1. [Mise à jour 3 de Visual Studio 2015] [2]
  2. [AWS Toolkit pour Visual Studio 2015][3] - version 1.11.2.0
  3. [Outils Microsoft .NET Core (Aperçu 2)][4] - version 14.1.21111.0

Chacun de ces composants est activement développé et publié, alors vérifiez les liens et les versions et mettez à jour ce document si nécessaire.

Chacun de ces éléments [peut prendre un certain temps à installer] [1], alors soyez patient, faites-le un à la fois pour minimiser les erreurs.

Enfin, redémarrer Windows avant de commencer le développement est triste, mais toujours une bonne idée après avoir effectué des installations majeures d’outils de développement sur une machine Windows.

[1] : http://stackoverflow.com/questions/28788574/visual-studio-2015-installer-hangs-during-install [2] : https://www.visualstudio.com/vs/community/ [3] : https://aws.amazon.com/visualstudio/ [4] : https://go.microsoft.com/fwlink/?LinkId=827546

Étape 1. Créer une nouvelle solution dans Visual Studio #

  1. Ouvrez Visual Studio et sélectionnez Fichier -> Nouveau projet
  2. Sélectionnez Projet AWS Lambda avec tests (.NET Core) [![entrez la description de l’image ici][1]][1]
  3. Ensuite, l’écran Select Blueprint s’affiche. Sélectionnez Fonction vide et cliquez sur le bouton Terminer : [![entrez la description de l’image ici][2]][2]
  4. Accédez à Outils -> Gestionnaire de packages NuGet -> Console du gestionnaire de packages.
  5. Dans la fenêtre de la console, saisissez [Install-Package Amazon.Lambda.Core][3]
  6. Cliquez avec le bouton droit sur votre projet dans l’explorateur de solutions et sélectionnez Restaurer les packages. Il s’agit de la dernière étape de préparation avant de commencer à écrire du code.

[![entrez la description de l’image ici][4]][4]

[1] : https://i.stack.imgur.com/WnOPj.png [2] : https://i.stack.imgur.com/WOATD.png [3] : https://www.nuget.org/packages/Amazon.Lambda.Core/ [4] : https://i.stack.imgur.com/Dkkpx.png

Étape 2. Ajoutez et modifiez le code dans votre projet #

  1. Ouvrez Function.cs et remplacez le code de la classe par ce qui suit :

     public class Function
 {

 /// <summary>
 /// A simple function that takes a birth date and returns Age in years
 /// </summary>
 /// <param name="input"></param>
 /// <returns>Age is years</returns>
 /// 
 [LambdaSerializer(typeof(SimpleSerializer))]
 public string FunctionHandler(Dictionary<string, int> input)
 {
     var defaultMessage = "Age could not be determined.";

     var birthDate = new DateTime(input["year"], input["month"], input["day"]);
     var ageInYears = DateTime.Today.Year - birthDate.Year;
     if (birthDate.DayOfYear > DateTime.Today.DayOfYear)
         ageInYears--;

     defaultMessage = $"Age in years: {ageInYears}";

     return defaultMessage;
 }
 }

  3. Dans le projet de test, modifiez la ligne 23 de FunctionTest.cs comme suit :

             var upperCase = function.FunctionHandler(null);

  4. Générez votre solution - vous ne devriez avoir aucune erreur de génération.

Étape 3 - Installez votre fonction Lambda dans AWS #

  1. Cliquez avec le bouton droit sur votre projet et sélectionnez Publier sur AWS Lambda…

  2. L’écran Télécharger sur AWS Lambda s’affiche. Assurez-vous que la bonne région est sélectionnée. Conservez toutes les valeurs par défaut. Entrez ensuite uniquement le nom de la fonction : AWSLambdaFunctionAgeInYears, puis cliquez sur suivant. [![entrez la description de l’image ici][1]][1]

  3. Sur la page suivante, sélectionnez AWSLambdaRole pour le champ Nom du rôle. Cliquez sur Télécharger et la fonction devrait télécharger sans erreur. [![entrez la description de l’image ici][2]][2]

[1] : https://i.stack.imgur.com/pAt9w.png [2] : https://i.stack.imgur.com/O4DTI.png

Étape 4. Appelez votre fonction AWS Lambda depuis Visual Studio #

  1. Après l’étape 3 ci-dessus, Visual Studio ouvrira la fenêtre “Afficher la fonction” avec votre fonction chargée.

  2. En bas à gauche de cet écran, saisissez le json suivant dans la zone Sample Input :

    { “month”: “10”, “day”: “28”, “year”: “1979” }

[![entrez la description de l’image ici][1]][1]

  1. Dernière étape : Cliquez sur le bouton vert Appeler. La fonction s’exécutera sur AWS et la réponse sera affichée dans la fenêtre Réponse en bas à droite.
