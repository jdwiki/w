Pages GitHub #

GitHub offre un hébergement illimité pour les utilisateurs ou les organisations et le site du projet. Les fichiers Jekyll et statiques sont disponibles.

Voici les étapes pour héberger votre blog Jekyll sur Github.

Installer #

Site d’utilisateurs ou d’organisations #

  1. Créez un référentiel nommé username.github.io, où username est votre nom d’utilisateur (ou le nom de votre organisation) sur GitHub.
  2. Clonez le référentiel sur votre ordinateur : 
    $ git clone https://github.com/username/username.github.io
  3. Entrez dans le dossier du projet, bootstrap, concevez et déboguez votre site : 
    $ cd username.github.io
$ bundle install
$ bundle exec jekyll serve
  4. Validez et transférez le référentiel : 
    $ git add --all
$ git commit -m "Initial commit"
$ git push -u origin master

Vous devriez maintenant pouvoir accéder à username.github.io pour voir votre blog.

Site du projet #

Le site de projet peut être activé dans chaque référentiel, y compris les référentiels privés.

  1. Activez le site du projet.

    Go to Settings-GitHub Pages-Sources, choose a source to switch on GitHub Pages for the repository.

  2. Site de construction

    You may build a Jekyll site from scratch or use Theme Chooser to find a theme for your project site.

  3. Modifier le contenu

  4. Engagez-vous

Vous devriez maintenant pouvoir accéder à username.github.io/your-project pour voir le site de votre projet.

Domaines personnalisés #

  1. Ouvrez Paramètres->Pages GitHub->Domaine personnalisé, ajoutez votre domaine personnalisé.
  2. Créez un fichier CNAME : 
    $ cd username.github.io
$ echo "example.com" > CNAME
  3. S’engager et pousser 
    $ git commit -m "Add CNAME" CNAME
$ git push -u origin master

Restrictions #

Plugins #

Jekyll dispose d’un système de plugins avec des crochets qui vous permettent de créer du contenu généré personnalisé spécifique à votre site. Cependant, GitHub Pages n’autorise qu’une liste blanche de plugins pour des raisons de sécurité.

Voici la liste blanche :

  • Plan du site Jekyll
  • Balise SEO Jekyll
  • métadonnées github
  • Flux Jekyll
  • Redirection Jekyll depuis
  • Yemojis
  • Mentions Jekyll

Pour éviter l’incohérence avec les pages GitHub, vous pouvez utiliser --safe pour servir en local.

Vous pouvez toujours utiliser tous les plugins en publiant votre site généré sur les pages GitHub, en convertissant le site localement et en poussant les fichiers statiques générés vers votre référentiel GitHub au lieu des fichiers source Jekyll.

Moteur de démarquage #

Depuis le 01/05/2016, GitHub Pages ne prend en charge que kramdown en tant que moteur Markdown.

Voir https://github.com/blog/2100-github-pages-now-faster-and-simpler-with-jekyll-3-0 pour plus de détails.

Sources #

GitHub vous permet de définir les sources Jekyll sur la branche master, la branche gh-pages ou le dossier /docs dans la branche master.

Un tutoriel complet est disponible sur https://pages.github.com/

Machine locale #

À des fins de test, vous pouvez héberger votre blog sur votre ordinateur local. Après avoir configuré et apporté des modifications, Jekyll peut héberger le blog sur http://localhost:4000. Sur la ligne de commande dans le répertoire racine du projet, exécutez :

$ bundle exec jekyll serve

La partie bundle exec est facultative, mais si vous utilisez Bundler, elle garantit que les dépendances de la gemme sont à jour. Pour une boucle Edit-Build-Test plus rapide, utilisez l’option --draft pour créer des articles à partir du répertoire _drafts :

$ bundle exec jekyll serve --draft --detach

L’utilisation de --detach place le processus en arrière-plan afin que l’invite de commande puisse être utilisée pour autre chose.

(À partir de la version 2.4, l’option --watch est activée par défaut. Si, par hasard, vous avez une version plus ancienne, vous devrez ajouter cette option afin que les modifications soient surveillées.)

Vous pouvez également définir le répertoire de destination sur un répertoire sur un autre serveur Web tel qu’Apache, nginx ou Lighttpd :

$ jekyll build --destination /path/to/web_server/root

Hébergement CloudCannon (et CMS) #

CloudCannon propose un hébergement et un CMS pour les applications Jekyll. Voici les étapes pour héberger votre application Jekyll sur CloudCannon (http://cloudcannon.com).

Sans contrôle de version :

  • Créez votre blog localement en utilisant des fichiers locaux et jekyll serve
  • Créez un compte CloudCannon et créez un nouveau site
  • Faites glisser votre site vers le “Navigateur de fichiers” dans CloudCannon

Avec contrôle de version :

  • Créer un dépôt sur Github ou Bitbucket
  • Créez votre blog localement en utilisant des fichiers locaux et jekyll serve
  • Créez un compte CloudCannon et créez un nouveau site
  • Connectez votre référentiel Github ou Bitbucket

Notez que CloudCannon n’est pas entièrement gratuit. Dans le forfait gratuit, vous pouvez utiliser CloudCannon comme CMS graphique, mais vous aurez besoin d’un hébergement externe. Dans le plan payant, l’hébergement est également inclus.
