Erreurs attendues lors de la génération de composants Angular 2 dans le projet .NET Core (version 0.8.3) #

Lors de la génération de nouveaux composants Angular 2 dans un projet .NET Core, vous pouvez rencontrer les erreurs suivantes (à partir de la version 0.8.3) :

Error locating module for declaration
        SilentError: No module files found

OU

No app module found. Please add your new Class to your component. 
        Identical ClientApp/app/app.module.ts

[LA SOLUTION]

  1. Renommez app.module.client.ts en app.client.module.ts

  2. Ouvrez app.client.module.ts : faites précéder la déclaration de 3 points “…” et placez la déclaration entre parenthèses.

    For example: [...sharedConfig.declarations, <MyComponent>]

  3. Ouvrez boot-client.ts : mettez à jour votre importation pour utiliser le nouveau app.client.module reference.

    For example: import { AppModule } from './app/app.client.module';

  4. Essayez maintenant de générer le nouveau composant : ng g component my-component

[EXPLICATION]

Angular CLI recherche un fichier nommé app.module.ts dans votre projet et essaie de trouver une référence pour la propriété declarations pour importer le composant. Cela devrait être un tableau (comme l’est le sharedConfig.declarations), mais les modifications ne sont pas appliquées
