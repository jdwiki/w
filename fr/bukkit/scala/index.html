Comment implémenter les plugins Bukkit dans le langage de programmation Scala

Configuration du projet (Scala Eclipse) #

Créer un projet en scala est très similaire à en créer un en java. Voici à quoi devrait ressembler la classe d’entrée :

package com.example.myplugin; //{$TopLevelDomain}.{$Domain}.{$PluginName}

import org.bukkit.plugin.java.JavaPlugin
import org.bukkit.command.CommandSender
import org.bukkit.command.Command

class PluginName extends JavaPlugin {

  override def onEnable() {

  }

  override def onDisable() {

  }

  override def onCommand(sender: CommandSender, cmd: Command, label: String, args: Array[String]): Boolean = {

    false
  }

}

Tout d’abord, assurez-vous d’avoir installé la dernière version de Scala située ici : https://www.scala-lang.org/download/

Ensuite, vous voudrez télécharger Scala Eclipse, disponible ici : http://scala-ide.org/ et extrayez le téléchargement dans un dossier de votre choix.

Une fois ceux-ci installés, ouvrez simplement Scala Eclipse.

Enfin, pour que votre plugin fonctionne - vous devez avoir une sorte de plugin d’exécution pour charger la bibliothèque scala pour vous, j’utilise celui-ci : https://dev.bukkit.org/projects/scala-loader (placez ceci jar dans votre dossier plugins comme n’importe quel autre plugin)

À partir de maintenant, le processus est presque identique à Java :

  1. Appuyez sur Alt + Maj + N -> cliquez sur Projet Scala
  2. Faites un clic droit sur votre projet - cliquez sur “Propriétés”
  3. Cliquez sur Java Build Path, puis cliquez sur l’onglet Libraries
  4. Cliquez sur “Ajouter des fichiers Jar externes” et sélectionnez votre fichier jar spigot-api
  5. Cliquez sur “Appliquer” puis sur “OK”

Pour la configuration du projet, vous souhaiterez créer un package afin :

Faites un clic droit sur le projet -> Nouveau -> Package

Nommez-le comme vous le souhaitez, généralement : com.yourdomain.pluginname

Dans ce package, créez une classe Scala et nommez-la comme vous le souhaitez, généralement : PluginName

Faites en sorte que la classe extend JavaPlugin et remplace les fonctions fournies pour une configuration de base, comme indiqué ci-dessus.

Enfin, faites un clic droit sur le dossier appelé “src” et sélectionnez Nouveau fichier. Nommez le fichier plugin.yml (PAS le nom de votre plugin, mais explicitement plugin.yml) et ouvrez-le.

Une implémentation de base devrait ressembler à ceci :

name: PluginName
main: com.example.pluginname.PluginName
version: 0.1

Et voila! Une fois que vous avez fini d’écrire votre plugin, cliquez sur Fichier -> Exporter -> Java -> Jar file -> Sélectionnez votre projet et spécifiez le dossier du plugin de votre serveur comme destination -> cliquez sur Terminer

En règle générale, vous pouvez simplement recharger votre serveur pour voir les modifications après l’exportation, mais ** certains plugins se cassent lors du rechargement, alors soyez prudent ! ** Je conseille toujours de ** redémarrer ** le serveur, sauf si vous savez que le rechargement ne cassera pas d’autres plugins .
