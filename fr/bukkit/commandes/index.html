Gestion d’une commande #

Pour gérer une commande, vous devez disposer d’une classe qui implémente l’interface CommandExecutor. La classe JavaPlugin (la classe principale de votre plugin) implémente déjà cela.

Lors de l’implémentation de l’interface CommandExecutor, la méthode suivante doit être implémentée :

public boolean onCommand(CommandSender sender, Command cmd, String label, String[] args) {
//Handle your command in here
return true;      ///Should return false if you want to show the usage

}

L’expéditeur est celui qui a envoyé la commande. Il peut s’agir d’un lecteur ou de la console.

CMD est la commande que vous écoutez, comme déclaré dans plugin.yml. A ne pas confondre avec l’étiquette.

label est l’alias utilisé pour exécuter cette commande, c’est ce que l’expéditeur tape après la barre oblique.

et enfin, args sont les arguments que l’expéditeur peut avoir utilisés pour envoyer votre commande.

Une commande possible pourrait aller comme

/dire Kerooker Salut, Kerooker !

Tell serait votre étiquette, et peut également être défini comme votre commande si vous l’avez dit dans plugin.yml ;

‘Kerooker’, ‘Salut,’ , ‘Kerooker !’ sont vos arguments 0, 1 et 2, respectivement

En retour, vous voudrez probablement toujours retourner true lorsque vous vous attendiez à ce que tout se passe de cette façon. Vous devez renvoyer false si vous souhaitez montrer à l’expéditeur l’utilisation de la commande définie dans votre plugin.yml

Une simple commande set GameMode ( /gm ) #

Cet exemple montre un exemple très basique de la façon d’utiliser onCommand. Je ne suggère pas de traiter vos commandes directement dans onCommand, mais cela fait l’affaire pour ce cas simple.

Dans cet exemple, nous essayons de définir le mode de jeu du joueur.

La première chose que nous devons faire est de nous assurer que l’expéditeur n’est pas un ConsoleCommandSender, car nous ne pouvons pas définir le mode de jeu d’une console. Ceci est fait avec (sender instanceof Player).

Ensuite, nous voulons que le joueur tape /gm CREATIVE (ou n’importe quel autre mode de jeu) donc nous devons vérifier 2 choses :

  1. assurez-vous qu’ils passent en 1 argument (CREATIVE)
  2. assurez-vous que leur commande était “gm”

Nous avons effectué ces vérifications avec : args.length == 1 && label.equalsIgnoreCase(“gm”)

Maintenant, nous savons avec certitude que le joueur a tapé “/gm x”.

La prochaine chose que nous devons faire est de transformer args[0] en un objet GameMode afin que nous puissions l’appliquer au joueur. Cela peut être fait avec GameMode.valueOf(String) Cependant, selon la documentation d’énumération Java, si une chaîne est transmise à valueOf() qui ne correspond pas à une énumération, elle lèvera une IllegalArgumentException - nous devons donc nous assurer de l’attraper.

Une fois que nous avons le mode de jeu, nous pouvons continuer et simplement utiliser p.setGameMode(gm) et le mode de jeu du joueur changera. Dans le cas où nous avons détecté une exception, nous imprimons simplement une déclaration et renvoyons false.

@Override
public boolean onCommand(CommandSender sender, Command command, String label, String[] args) {
    if (sender instanceof Player) { 
        final Player p = (Player) sender;

        if (args.length == 1 && label.equalsIgnoreCase("gm")) {
            try {
                GameMode gm = GameMode.valueOf(args[0]);
                p.setGameMode(gm);
                p.sendMessage(ChatColor.GREEN + "Your gamemode has been set to: " + gm.toString());
                return true;
            } catch (IllegalArgumentException e) {
                p.sendMessage(ChatColor.RED + "Invalid gamemode option!");
                return false;
            }

        }
    }
    return false;
}

Commande pas dans la classe principale #

Si vous avez beaucoup de commandes, vous ne devriez pas toutes les mettre dans la classe principale.

  1. Créez une nouvelle classe et faites-la implémenter CommandExecutor

  2. Ajoutez ce qui suit à la classe :

     @Override 
 public boolean onCommand(CommandSender sender, Command cmd, String label, String[] args) {

 }

  3. Dans votre classe principale, ajoutez onEnable (remplacez commandName par le nom de la commande et CommandExecutor par le nom de la classe):

     getCommand("commandName").setExecutor(new CommandExecutor());
