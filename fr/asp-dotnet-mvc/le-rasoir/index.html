Qu’est-ce que le rasoir ?

Razor est une syntaxe de balisage qui vous permet d’intégrer du code basé sur un serveur (Visual Basic et C#) dans des pages Web.

Le code basé sur le serveur peut créer du contenu Web dynamique à la volée, tandis qu’une page Web est écrite dans le navigateur. Lorsqu’une page Web est appelée, le serveur exécute le code basé sur le serveur à l’intérieur de la page avant de renvoyer la page au navigateur. En s’exécutant sur le serveur, le code peut effectuer des tâches complexes, comme l’accès aux bases de données.

Syntaxe #

  • @{ … }
  • @Nom de variable
  • @(Nom de variable)
  • @pour(…){ }
  • @(expression explicite)
  • @* commentaires *@

ASP.NET Razor inclut des moteurs d’affichage pour C # et VB.

Le moteur de vue C# traite les fichiers avec une extension .cshtml, tandis que le moteur de vue VB fonctionne avec les fichiers .vbhtml.

Syntaxe de base #

Le code Razor peut être inséré n’importe où dans le code HTML. Les blocs de code Razor sont entourés de @{ ... }. La variable et les fonctions en ligne commencent par @. Le code à l’intérieur des parenthèses Razor suit les règles normales C# ou VB.

Énoncé sur une seule ligne :

@{ var firstNumber = 1; }

Bloc de code multiligne :

@{
    var secondNumber = 2;
    var total = firstNumber + secondNumber;
}

Utilisation d’une variable en ligne :

<h1>The total count is @total</h1>

Utilisation explicite d’une variable en ligne :

<h2>Item@(item.Id)</h2>

Pour cet exemple particulier, nous ne pourrons pas utiliser la syntaxe implicite car Item@item.Id ressemble à un e-mail et sera rendu comme tel par Razor.

Joindre le code dans les instructions de flux de contrôle :

<h1>Start with some HTML code</h1>

@for (int i = 0; i < total; i++){
    Console.Write(i);
}

<p>Mix in some HTML code for fun!</p>
<p>Add a second paragraph.</p>

@if (total > 3)
{
    Console.Write("The total is greater than 3");
}
else
{
    Console.Write("The total is less than 3");
}

Cette même syntaxe serait utilisée pour toutes les instructions telles que for, foreach, while, if, switch, etc.

Ajout de code à l’intérieur du code :

@if (total > 3)
{
    if(total == 10)
    {
        Console.Write("The total is 10")
    }
}

Notez que vous n’avez pas besoin de saisir le @ au second if. Après le code, vous pouvez simplement taper un autre code derrière le code existant.

Si vous souhaitez ajouter du code après un élément HTML, vous ** devez ** saisir un @.

Ajoutez des commentaires #

Razor a sa propre syntaxe de commentaire qui commence par @* et se termine par *@.

Commentaire en ligne :

<h1>Comments can be @*hi!*@ inline</h1>

Commentaire sur plusieurs lignes :

@* Comments can spread
   over multiple
   lines *@

Commentaire HTML

Vous pouvez également utiliser la syntaxe de commentaire HTML normale commençant par <!-- et se terminant par --> dans les vues Razor. Mais contrairement aux autres commentaires, le code Razor à l’intérieur d’un commentaire HTML est toujours exécuté normalement.

@{
    var hello = "Hello World!"; 
} 
<!-- @hello -->

L’exemple ci-dessus produit la sortie HTML suivante :

<!-- Hello World! -->

Commentaires dans un bloc de code :

@{
    // This is a comment
    var Input = "test";
}

Ajout d’un attribut personnalisé avec - (trait d’union) dans le nom #

Si vous devez ajouter un attribut via un rasoir qui a un - (trait d’union) dans le nom, vous ne pouvez pas simplement le faire

@Html.DropDownListFor(m => m.Id, Model.Values, new { @data-placeholder = "whatever" })

il ne compilera pas. Les attributs data-* sont valides et courants dans html5 pour ajouter des valeurs supplémentaires aux éléments.

Cependant, ce qui suit fonctionnera

@Html.DropDownListFor(m => m.Id, Model.Values, new { @data_placeholder = "whatever" })

puisque “_” est remplacé par “-” lors du rendu.

Cela fonctionne bien car les traits de soulignement ne sont pas acceptables dans les noms d’attributs en HTML.

Créez des classes et des méthodes en ligne à l’aide de @functions #

L’utilisation du mot-clé Razor @functions donne la possibilité d’introduire des classes et des méthodes pour une utilisation en ligne dans un fichier Razor :

@functions
{
    string GetCssClass(Status status)
    {
        switch (status)
        {
            case Status.Success:
                return "alert-success";
            case Status.Info:
                return "alert-info";
            case Status.Warning:
                return "alert-warning";
            case Status.Danger:
            default:
                return "alert-danger";
         }
     }
}


<label class="alert @GetCssClass(status)"></label>

La même chose peut être faite pour les classes :

@functions
{
    class Helpers
    {
        //implementation
    }
}

Afficher le code HTML dans le bloc de code Razor #

À l’intérieur d’un bloc de code Razor, le navigateur ne reconnaîtra le code HTML que si le code est échappé.

Utilisez @: pour une seule ligne :

@foreach(int number in Model.Numbers)
{
    @:<h1>Hello, I am a header!</h1>
}

Utilisez <text> ... </text> pour Multi-ligne :

@{
    var number = 1;

    <text>
        Hello, I am text
        <br / >
        Hello, I am more text!
    </text>
}

Notez que Razor, lorsqu’il se trouve à l’intérieur d’un bloc de code, comprendra les balises HTML. Par conséquent, l’ajout de la balise text autour des balises HTML n’est pas nécessaire (bien que toujours correct), comme :

@{
    var number = 1;
    <text>
        <div>
            Hello, I am text
            <br / >
            Hello, I am more text!
        </div>
    </text>
}

Échapper au caractère @ #

Dans de nombreux cas, l’analyseur Razor est suffisamment intelligent pour déterminer quand le signe @ est censé être utilisé dans le cadre du code, par opposition à faire partie de quelque chose comme une adresse e-mail. Dans l’exemple ci-dessous, il n’est pas nécessaire d’échapper le signe “@” :

<p>Reach out to us at contact@mail.com</p>

Cependant, dans certains cas, l’utilisation du signe @ est plus ambiguë et il doit être explicitement échappé avec @@, comme dans l’exemple ci-dessous :

<p>Join us @@ Stack Overflow!</p>

Alternativement, nous pouvons utiliser un caractère @ encodé en HTML

<p>Join us &#64; Stack Overflow!</p>

Partager @helpers à travers les vues #

@Helpers pourrait être partagé entre les vues.

Ils doivent être créés dans le dossier App_Code

[![entrez la description de l’image ici][1]][1]

[1] : http://i.stack.imgur.com/EMu2v.png

@helper CreatePrimaryBootstrapButton(string label)
{
    <button type="button" class="btn btn-primary">@label</button>
}

//call

@MenuHelpers.CreatePrimaryBootstrapButton("my button")

Les variables globales @Url et @Html ne sont pas disponibles par défaut dans le @Helper défini dans App_code. Vous pouvez les ajouter comme suit (pour chaque .cshtml dans votre dossier App_code)

@*  Make @Html and @Url available *@
@functions
{
    private new static HtmlHelper<object> Html
    {
        get { return ((WebViewPage)CurrentPage).Html; }
    }

    private static UrlHelper Url
    {
        get { return ((WebViewPage)CurrentPage).Url; }
    }
 }

Modèles d’éditeur #

Les modèles d’éditeur sont un bon moyen de réutiliser le code Razor. Vous pouvez définir des modèles d’éditeur en tant que vues partielles Razor, puis les utiliser dans d’autres vues.

Les modèles d’éditeur existent généralement dans le dossier Views/Shared/EditorTemplates/, bien qu’ils puissent également être enregistrés dans le dossier Views/ControllerName/EditorTemplates/. Le nom de la vue est généralement le nom de l’objet pour lequel vous souhaitez utiliser le modèle, comme <type>.cshtml.

Voici un modèle d’éditeur simple pour DateTime :

@model DateTime
<div>
   <span>
      @Html.TextBox("", Model.ToShortDateString(), new { data_date_picker="true" })
   </span>
</div>

Enregistrez le fichier sous Views/Shared/EditorTemplate/DateTime.cshtml.

Ensuite, utilisez [EditorFor][1] pour appeler ce code de modèle dans une autre vue :

@Html.EditorFor(m => m.CreatedDate)

Il existe également un attribut UIHint pour spécifier le nom du fichier :

public class UiHintExampleClass
{
    [UIHint("PhoneNumber")]
    public string Phone { get; set; }
}

Définissez ce modèle de numéro de téléphone dans Views/Shared/EditorTemplates/PhoneNumber.cshtml.

Les modèles d’éditeur peuvent également être définis pour les types personnalisés.

Voici un type personnalisé appelé SubModel :

public class SubModel
{
    public Guid Id { get; set;} 
    public string FirstName { get; set; }
    public string LastName { get; set; }
}

public class Model
{
    public Guid Id { get; set; }
    public DateTime Created {get; set; }
    
    public SubModel SubModel{get; set; }
}

Voici l’EditorTemplate pour SubModel :

@model SubModel
<div class="form-group">
    @Html.LabelFor(m => m.FirstName)
    @Html.TextBoxFor(m => m.FirstName)
</div>
<div class="form-group">
    @Html.LabelFor(m => m.LastName)
    @Html.TextBoxFor(m => m.LastName)
</div>

Désormais, la vue du modèle devient simplement :

@model Model
@Html.EditorFor(m => m.CreatedDate)
@Html.EditorFor(m => m.SubModel, new { @Prefix = "New"}) 
@* the second argument is how you can pass viewdata to your editor template*@

[1] : https://msdn.microsoft.com/en-us/library/system.web.mvc.html.editorextensions.editorfor(v=vs.118).aspx

Transmettre le contenu de Razor à un @helper #

Envoyez une partie Razor à un @helper, par exemple une div HTML.

@helper WrapInBox(Func<Object, HelperResult> content)
{
    <div class="box">@content(null) </div>
}

//call 
@WrapInBox(@<div>
                I'm a inner div
            </div>)
