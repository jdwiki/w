Si construction #

La construction if (appelée instruction IF de bloc dans FORTRAN 77) est commune à de nombreux langages de programmation. Il exécute conditionnellement un bloc de code lorsqu’une expression logique est évaluée à true.

[name:] IF (expr) THEN
    block
[ELSE IF (expr) THEN [name]
    block]
[ELSE [name]
   block]
END IF [name]

où,

  • name - le nom de la construction if (optionnel)
  • expr - une expression logique scalaire entre parenthèses
  • bloc - une séquence de zéro ou plusieurs instructions ou constructions

Un nom de construction au début d’une instruction if then doit avoir la même valeur que le nom de construction à l’instruction end if, et il doit être unique pour l’unité de portée actuelle.

Dans les instructions if, les (in)égalités et les expressions logiques évaluant une instruction peuvent être utilisées avec les opérateurs suivants :

.LT.  which is <   ! less than
.LE.           <=  ! less than or equal
.GT.           >   ! greater than
.GE.           >=  ! greater than or equal
.EQ.           =   ! equal
.NE.           /=  ! not equal
.AND.              ! logical and
.OR.               ! logical or
.NOT.              ! negation

Exemples:

! simplest form of if construct
if (a > b) then
    c =  b / 2
end if
!equivalent example with alternate syntax
if(a.gt.b)then
   c=b/2
endif

! named if construct
circle: if (r >= 0) then
    l = 2 * pi * r
end if circle

! complex example with nested if construct
block: if (a < e) then
    if (abs(c - e) <= d) then
        a = a * c
    else
        a = a * d
    end if
else
    a = a * e
end if block

Une utilisation historique de la construction if est dans ce qu’on appelle une instruction “arithmétique si”. Comme cela peut être remplacé par des constructions plus modernes, cependant, il n’est pas couvert ici. Plus de détails peuvent être trouvés [ici][1].

[1] : https://www.wikiod.com/fr/fortran/alternatives-modernes-aux-caracteristiques-historiques#Arithmétique if instruction

Construction SELECT CASE #

Une construction select case exécute conditionnellement un bloc de constructions ou d’instructions en fonction de la valeur d’une expression scalaire dans une instruction select case. Cette construction de contrôle peut être considérée comme un remplacement du « goto » calculé.

[name:] SELECT CASE (expr)
[CASE (case-value [, case-value] ...) [name]
   block]...
[CASE DEFAULT [name]
   block]
END SELECT [name]

où,

  • name - le nom de la construction select case (optionnel)
  • expr - une expression scalaire de type entier, logique ou caractère (entre parenthèses)
  • case-value - une ou plusieurs expressions scalaires entières, logiques ou d’initialisation de caractères entre parenthèses
  • bloc - une séquence de zéro ou plusieurs instructions ou constructions

Exemples:

! simplest form of select case construct
select case(i)
case(:-1)
    s = -1
case(0)
    s = 0
case(1:)
    s = 1
case default
    print "Something strange is happened"
end select

Dans cet exemple, la valeur de cas (:-1) est une plage de valeurs correspondant à toutes les valeurs inférieures à zéro, (0) correspond à des zéros et (1:) correspond à toutes les valeurs supérieures à zéro, La section default implique si d’autres sections n’ont pas été exécutées.

Bloquer la construction DO #

Une construction “do” est une construction en boucle qui a un certain nombre d’itérations régies par un contrôle de boucle

integer i
do i=1, 5
  print *, i
end do
print *, i

Dans le formulaire ci-dessus, la variable de boucle i passe 5 fois dans la boucle, prenant les valeurs 1 à 5 à tour de rôle. Une fois la construction terminée, la variable de boucle a la valeur 6, c’est-à-dire que la variable de boucle est incrémentée une fois de plus après l’achèvement de la boucle.

Plus généralement, la construction de boucle do peut être comprise comme suit

integer i, first, last, step
do i=first, last, step
end do

La boucle commence par i avec la valeur first, incrémentant chaque itération de step jusqu’à ce que i soit supérieur à last (ou inférieur à last si la taille du pas est négative).

Il est important de noter que depuis Fortran 95, la variable de boucle et les expressions de contrôle de boucle doivent être des entiers.

Une itération peut se terminer prématurément avec l’instruction “cycle”

do i=1, 5
  if (i==4) cycle
end do

et l’ensemble de la construction peut cesser son exécution avec l’instruction exit

do i=1, 5
  if (i==4) exit
end do
print *, i

Les constructions do peuvent être nommées :

do_name: do i=1, 5
end do do_name

ce qui est particulièrement utile lorsqu’il y a des constructions “do” imbriquées

do1: do i=1, 5
  do j=1,6
    if (j==3) cycle        ! This cycles the j construct
    if (j==4) cycle        ! This cycles the j construct
    if (i+j==7) cycle do1  ! This cycles the i construct
    if (i*j==15) exit do1  ! This exits the i construct
  end do
end do1

Les constructions do peuvent également avoir un contrôle de boucle indéterminé, soit “pour toujours”, soit jusqu’à ce qu’une condition donnée soit remplie

integer :: i=0
do
  i=i+1
  if (i==5) exit
end do

ou

integer :: i=0
do while (i<6)
  i=i+1
end do

Cela permet également une boucle do infinie via une instruction .true.

print *,'forever'
do while(.true.)
  print *,'and ever'
end do

Une construction do peut également laisser l’ordre des itérations indéterminé

do concurrent (i=1:5)
end do

en notant que la forme du contrôle de boucle est la même que dans un contrôle forall.

Il existe diverses restrictions sur les instructions pouvant être exécutées dans la plage d’une construction “do concurrent” qui sont conçues pour garantir qu’il n’y a pas de dépendances de données entre les itérations de la construction. Cette indication explicite par le programmeur peut permettre une plus grande optimisation (y compris la parallélisation) par le compilateur qui peut être difficile à déterminer autrement.

Les variables “privées” dans une interaction peuvent être réalisées en utilisant une construction block dans le do concurrent :

do concurrent (i=1:5, j=2:7)
  block
    real tempval  ! This is independent across iterations
  end block
end do

Une autre forme de la construction de bloc do utilise une instruction continue étiquetée au lieu d’une end do :

    do 100, i=1, 5
100 continue

Il est même possible d’imbriquer de telles constructions avec une instruction de terminaison partagée

    do 100, i=1,5
    do 100, j=1,5
100 continue

Ces deux formes, et surtout la seconde (qui est obsolète), sont généralement à éviter dans un souci de clarté.

Enfin, il existe également une construction “do” non bloc. Ceci est également considéré comme obsolète et est [décrit ailleurs] (https://www.wikiod.com/fr/fortran/alternatives-modernes-aux-caracteristiques-historiques#Constructions DO non-blocs 201607242013447800136), ainsi que des méthodes de restructuration en tant que construction de bloc “do”.

WHERE construire #

La construction where, disponible à partir de Fortran90 représente une construction do masquée. L’instruction de masquage suit les mêmes règles que l’instruction if, mais est appliquée à tous les éléments du tableau donné. L’utilisation de where permet d’effectuer des opérations sur un tableau (ou plusieurs tableaux de même taille), dont les éléments satisfont à une certaine règle. Ceci peut être utilisé pour simplifier les opérations simultanées sur plusieurs variables.

Syntaxe:

[name]: where (mask)
    block
[elsewhere (mask)
    block]
[elsewhere
    block]
end where [name]

Ici,

  • nom - est le nom donné au bloc (si nommé)
  • mask - est une expression logique appliquée à tous les éléments
  • bloc - série de commandes à exécuter

Exemples:

! Example variables
real:: A(5),B(5),C(5)
A = 0.0
B = 1.0
C = [0.0, 4.0, 5.0, 10.0, 0.0]

! Simple where construct use
where (C/=0)
    A=B/C
elsewhere
    A=0.0
end

! Named where construct
Block: where (C/=0)
    A=B/C
elsewhere
    A=0.0
end where Block
