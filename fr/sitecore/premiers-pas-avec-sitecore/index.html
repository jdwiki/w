Utilisation du gestionnaire SIM pour l’installation #

Sitecore Instance Manager est un outil open source utilisé pour gérer le parc local d’instances Sitecore. Vous pouvez installer, localiser, entretenir, réinstaller ou supprimer des produits Sitecore. Il vous aide également à installer votre instance sitecore avec tous les packages et modules sitecore et la seule chose que vous devez faire est de définir le dossier du référentiel local où tous les fichiers d’installation sont conservés. Voir la capture d’écran ci-dessous pour les paramètres. [![entrez la description de l’image ici][1]][1]

Il dispose également d’une API et d’un moteur de plug-in afin que vous puissiez l’étendre pour tous vos besoins.

Pour l’installation, cliquez [ici][2]

Pour plus de détails sur la carte SIM, veuillez cliquer [ici][3]

Installation ou configuration #

Fichier exécutable

Téléchargez le fichier .exe depuis http://dev.sitecore.net/ et double-cliquez pour démarrer. Ce .exe fera tout pour vous - attacher des bases de données, modifier le fichier hôte et définir les autorisations de dossier. Le seul inconvénient est qu’il laisse une entrée dans le registre sous [HKEY_LOCAL_MACHINE\SOFTWARE\Sitecore CMS] et rend votre instance disponible sous “Ajout/Suppression de programmes”, ce qui est trompeur. Sitecore n’est qu’une application ASP.NET, pas une application de bureau.

Installation manuelle

Vous pouvez choisir d’installer et de configurer Sitecore manuellement en téléchargeant le zip racine du site. Vous êtes responsable d’attacher des bases de données et de vous assurer que les autorisations sont correctement configurées. Vous souhaiterez peut-être utiliser cette méthode dans des environnements nécessitant des étapes d’installation personnalisées (les bases de données se trouvent sur un autre serveur, des autorisations personnalisées sont nécessaires). Suivez le guide d’installation (disponible sur http://dev.sitecore.net/ ; assurez-vous de lire le guide de votre version spécifique) et - si vous installez en production - le Guide de renforcement de la sécurité.

Une fois que vous deviendrez un développeur Sitecore certifié, vous pourrez télécharger la dernière version de Sitecore et tous les modules associés à partir de http://dev.sitecore.net/. Sur dev.sitecore.net, il y a toujours deux formats différents disponibles au téléchargement - la racine complète du site ou un fichier exécutable. Vous trouverez ci-dessous une liste des manières d’installer Sitecore.

Pas à pas

  1. Téléchargez la version de Sitecore que vous voulez à partir du [site Web de Sitecore][1] - c’est correct si vous préférez la méthode d’installation automatique, mais les étapes suivantes couvrent le processus manuel
  2. Décompressez les archives dans un dossier sur votre ordinateur. Vous verrez les sous-dossiers suivants :
  • Données - C’est là que Sitecore conserve un certain nombre d’archives liées à l’instance actuellement installée, telles que les journaux, les index, les packages et les fichiers de cache
  • Bases de données - Fichiers de base de données de SQL Server et Oracle
  • Site Web - Tout ce qui sera accessible depuis le navigateur Web.
  1. Ouvrez IIS et créez un nouveau site Web avec un lien vers le dossier “Site Web”
  2. Définissez le pool d’applications de votre site Web pour utiliser le .NET Framework vers la version 4.5
  3. Ajustez le chemin de votre DataFolder (sur le web.config) - pour Dev, vous pouvez copier le dossier Data dans le dossier Website
  4. Copiez le fichier license.xml dans le dossier Data
  5. Dans les propriétés du dossier Site Web, décochez la case “lecture seule”. Dans Dev, vous pouvez donner des autorisations complètes à l’utilisateur du pool d’applications. En production, vous devez suivre les instructions contenues ([ici][2]), sous le point “4.2 Configuration des autorisations de dossier et de registre”.
  6. Installez toutes les bases de données dans le dossier “Database” et modifiez les configurations dans le fichier ConnectionStrings.config, situé dans le dossier Website/App_Config
  7. Toujours dans ConnectionStrings.config, configurez les informations de connexion pour toutes les entrées MongoDB (celles avec connectionString=“mongodb:…”)
  8. Si votre site est sur localhost, pointez votre navigateur sur http://localhost/sitecore. Si vous voyez un écran de connexion, vous avez installé Sitecore avec succès.
