Vérification des arguments #

Condition d’argument #

Cette méthode vérifie si l’argument spécifié est vrai. Il prend également le nom de l’argument qui est consigné si la condition échoue.

Assert.ArgumentCondition(pageIndex >= 0, "pageIndex", "Value must be greater than or equal to zero.");

ArgumentNotNull #

Cette méthode garantit que l’argument passé n’est pas nul. Il existe deux signatures pour cette méthode, la première prend un objet et un nom de paramètre et effectue une simple vérification nulle.

Assert.ArgumentNotNull(item, "item");

ArgumentNotNullOrEmpty #

Ceci est similaire à la méthode ArgumentNotNull, mais vérifiera également si l’objet est vide. Il existe trois variantes de cette méthode. La première variante prend un ID Sitecore et un nom d’argument, et vérifie si l’ID est nul.

var nullId = new new ID("{00000000-0000-0000-0000-000000000000}");

// Both of these calls will result in an exception
Assert.ArgumentNotNullOrEmpty((ID)null, "null");
Assert.ArgumentNotNullOrEmpty(nullId, nameof(nullId));

La deuxième méthode ajoute une vérification pour voir si la chaîne donnée est nulle ou vide.

// Both of these calls will result in an exception
Assert.ArgumentNotNullOrEmpty((string)null, "null");
Assert.ArgumentNotNullOrEmpty("", nameof(emptyString));

Assurez-vous que l’élément est en mode édition #

Vérifie si l’“élément” transmis est en mode d’édition. Sinon, il lève une EditingNotAllowedException.

Assert.IsEditing(Sitecore.Context.Item);

Assurez-vous que deux valeurs sont égales #

Compare deux valeurs pour l’égalité. Il ne peut comparer que des chaînes, des entiers et des valeurs booléennes.

Assert.AreEqual(documentElement.LocalName, "xamlControls", "Xaml files must have a root node named 'xamlControls'.");

S’assurer qu’une valeur est vraie ou fausse #

Pour affirmer qu’une valeur est vraie ou fausse, :

Assert.IsFalse(Settings.DoBadThings, "Bad things should not happen, disable DoBadThings.");
Assert.IsTrue(magicNumber =< 42, "The magic number is greater than 42!");

Vous pouvez également passer des paramètres de formatage pour le message d’exception

Assert.IsFalse(myValue > 5, "The value should not be greater than 5, it's currently {0}", myValue);

ResultNotNull #

ResultNotNull() vérifie si l’objet transmis n’est pas nul. Si l’objet et le message ne sont pas nuls, il renverra simplement l’objet qui a été transmis, sinon il lancera InvalidOperationException.

return Assert.ResultNotNull(this.Database.GetItem(this.ItemRootId), string.Concat("Root item not found. ID: ", this.ItemRootId));

Objet requis #

Cela vérifie si l’objet donné est nul, puis lève RequiredObjectIsNullException si c’est le cas.

Assert.Required(parameter, "parameter is required.");

Chèques nuls/vides #

Est non nulle #

Il s’agit d’une méthode très simple et populaire à utiliser pour vérifier si un élément n’est pas nul. Il vérifie simplement l’objet qui est transmis pour voir s’il est nul.

Assert.IsNotNull(database, type, "Name: {0}", item);

IsNotNullOrEmpty #

C’est la même chose que IsNotNull ci-dessus, mais fonctionne sur des valeurs de chaîne au lieu d’objets.

Assert.IsNotNullOrEmpty(propertyName, "user");

EstNull #

Il s’agit simplement d’un inverse de la méthode IsNotNull(). Cette méthode affirme que l’objet est nul.

Assert.IsNull(this.StandardValues, "A Standard Values item has already been created for this template ");

## Assertions de sécurité

CanRunApplication #

Pour vérifier si l’utilisateur est autorisé à exécuter l’application donnée. Sinon, AccessDeniedException est levée.

Assert.CanRunApplication("WebEdit");

A accès #

HasAccess vérifiera si le paramètre donné est vrai, sinon une AccessDeniedException sera levée.

Assert.HasAccess(Context.User.IsAdministrator, "Only administrators can create new domains");
