Syntaxe #

  • NSLog(@“texte à enregistrer”); // Journal de texte de base
  • NSLog(@“data : %f - %.2f”, myFloat, anotherFloat); // Journalisation du texte incluant les nombres flottants.
  • NSLog(@“données : %i”, monEntier) ; // Texte de journalisation comprenant un nombre entier.
  • NSLog(@“data : %@”, myStringOrObject); // Texte de journalisation faisant référence à une autre chaîne ou à tout objet dérivé de NSObject.

Pour la journalisation de divers types d’objets et de types de données, reportez-vous à : [Objective-C, Format Specificers][1]

[1] : https://www.wikiod.com/fr/objective-c/specificateurs-de-format

Journalisation #

NSLog(@"Log Message!");
NSLog(@"NSString value: %@", stringValue);
NSLog(@"Integer value: %d", intValue);

Le premier argument de NSLog est un NSString contenant le format du message de journal. Les autres paramètres sont utilisés comme valeurs à substituer à la place des spécificateurs de format.

Le formatage fonctionne exactement de la même manière que printf, à l’exception du spécificateur de format supplémentaire %@ pour un objet Objective-C arbitraire. Cette:

NSLog(@"%@", object);

est équivalent à:

NSLog(@"%s", [object description].UTF8String);

Format de sortie NSLog #

NSLog(@"NSLog message");

Le message qui est imprimé en appelant NSLog a le format suivant lorsqu’il est affiché dans Console.app :

Rendez-vous Temps Nom du programme ID de processus Identifiant du fil Message
2016-07-16 08:58:04.681 test [46259 : 1244773] Message NSLog

Enregistrement des valeurs variables #

Vous ne devriez pas appeler NSLog sans une chaîne de format littérale comme celle-ci :

NSLog(variable);    // Dangerous code!

Si la variable n’est pas une NSString, le programme plantera, car NSLog attend une NSString.

Si la variable est un NSString, cela fonctionnera à moins que votre chaîne ne contienne un %. NSLog analysera la séquence % en tant que spécificateur de format, puis lira une valeur parasite dans la pile, provoquant un plantage ou même [l’exécution de code arbitraire] (https://en.wikipedia.org/wiki/Uncontrolled_format_string).

Au lieu de cela, faites toujours du premier argument un spécificateur de format, comme ceci :

NSLog(@"%@", anObjectVariable);
NSLog(@"%d", anIntegerVariable);

Suppression des instructions de journal des versions de version #

Les messages imprimés à partir de NSLog sont affichés sur Console.app même dans la version de version de votre application, ce qui n’a pas de sens pour les impressions qui ne sont utiles que pour le débogage. Pour résoudre ce problème, vous pouvez utiliser cette macro pour la journalisation du débogage au lieu de “NSLog”.

#ifdef DEBUG
#define DLog(...) NSLog(__VA_ARGS__)
#else
#define DLog(...)
#endif

Utiliser:

NSString *value = @"value 1";
DLog(@"value = %@", value);
// little known fact: programmers look for job postings in Console.app
NSLog(@"We're hiring!");

Dans les versions de débogage, DLog appellera NSLog. Dans les versions de version, DLog ne fera rien.

Le message vide n’est pas imprimé #

Lorsque NSLog est invité à imprimer une chaîne vide, il omet complètement le journal.

NSString *name = @"";
NSLog(@"%@", name);  // Resolves to @""

Le code ci-dessus n’imprimera rien.

Il est recommandé de préfixer les journaux avec des étiquettes :

NSString *name = @"";
NSLog(@"Name: %@", name);  // Resolves to @"Name: "

Le code ci-dessus imprimera :

2016-07-21 14:20:28.623 App[87711:6153103] Name:

Utilisation de __FUNCTION __ #

NSLog(@"%s %@",__FUNCTION__, @"etc etc");

Insère le nom de la classe et de la méthode dans le résultat :

2016-07-22 12:51:30.099 loggingExample[18132:2971471] -[ViewController viewDidLoad] etc etc

NSLog contre printf #

NSLog(@"NSLog message");
printf("printf message\n");

Production:

2016-07-16 08:58:04.681 test[46259:1244773] NSLog message
printf message

NSLog affiche la date, l’heure, le nom du processus, l’ID du processus et l’ID du thread en plus du message de journal. printf affiche simplement le message.

NSLog nécessite un NSString et ajoute automatiquement une nouvelle ligne à la fin. printf nécessite une chaîne C et n’ajoute pas automatiquement une nouvelle ligne.

NSLog envoie la sortie à stderr, printf envoie la sortie à stdout.

Certains spécificateurs de format dans printf vs NSLog sont différents. Par exemple, lors de l’inclusion d’une chaîne imbriquée, les différences suivantes se produisent :

NSLog(@"My string: %@", (NSString *)myString);
printf("My string: %s", [(NSString *)myString UTF8String]);

Journalisation des métadonnées NSLog #

NSLog(@"%s %d %s, yourVariable: %@", __FILE__, __LINE__, __PRETTY_FUNCTION__, yourVariable);

Enregistre le fichier, le numéro de ligne et les données de fonction avec toutes les variables que vous souhaitez enregistrer. Cela peut rendre les lignes de journal beaucoup plus longues, en particulier avec des noms de fichiers et de méthodes verbeux, mais cela peut aider à accélérer les diagnostics d’erreur.

Vous pouvez également l’envelopper dans une macro (stockez-la dans un singleton ou là où vous en aurez le plus besoin) ;

#define ALog(fmt, ...) NSLog((@"%s [Line %d] " fmt), __PRETTY_FUNCTION__, __LINE__, ##__VA_ARGS__);

Ensuite, lorsque vous souhaitez vous connecter, appelez simplement

ALog(@"name: %@", firstName);

Ce qui vous donnera quelque chose comme;

 -[AppDelegate application:didFinishLaunchingWithOptions:] [Line 27] name: John

Type NSLog et BOOL #

Il n’y a pas de spécificateur de format pour imprimer le type booléen à l’aide de NSLog. Une façon d’imprimer une valeur booléenne consiste à la convertir en chaîne.

BOOL boolValue = YES;
NSLog(@"Bool value %@", boolValue ? @"YES" : @"NO");

Production:

2016-07-30 22:53:18.269 Test[4445:64129] Bool value YES

Une autre façon d’imprimer une valeur booléenne consiste à la convertir en entier, obtenant une sortie binaire (1=oui, 0=non).

BOOL boolValue = YES;
NSLog(@"Bool value %i", boolValue);

Production:

2016-07-30 22:53:18.269 Test[4445:64129] Bool value 1

Journalisation par ajout à un fichier #

NSLog est bon, mais vous pouvez également vous connecter en ajoutant à un fichier à la place, en utilisant un code comme :

NSFileHandle* fh = [NSFileHandle fileHandleForWritingAtPath:path];
if ( !fh ) {
    [[NSFileManager defaultManager] createFileAtPath:path contents:nil attributes:nil];
    fh = [NSFileHandle fileHandleForWritingAtPath:path];
}
if ( fh ) {
    @try {
        [fh seekToEndOfFile];
        [fh writeData:[self dataUsingEncoding:enc]];
    }
    @catch (...) {
    }
    [fh closeFile];
}
