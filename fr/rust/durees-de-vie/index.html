Syntaxe #

  • fonction fn<‘a>(x : &‘a Type)

  • struct Struct<‘a> { x : &‘a Type }

  • enum Enum<‘a> { Variante(&‘a Type) }

  • impl<‘a> Struct<‘a> { fn x<‘a>(&self) -> &‘a Type { self.x } }

  • impl<‘a> Trait<‘a> pour Type

  • impl<‘a> Trait pour Type<‘a>

  • fonction fn<F>(f : F) où pour<'a> F : FnOnce(&'a Type)

  • struct Struct<F> where for<'a> F: FnOnce(&'a Type) { x: F }

  • enum Enum<F> where for<'a> F: FnOnce(&'a Type) { Variant(F) }

  • impl<F> Struct<F> where for<'a> F: FnOnce(&'a Type) { fn x(&self) -> &F { &self.x } }

  • Toutes les références dans Rust ont une durée de vie, même si elles ne sont pas explicitement annotées. Le compilateur est capable d’attribuer implicitement des durées de vie.

  • La durée de vie 'statique est attribuée aux références qui sont stockées dans le binaire du programme et sera valide tout au long de son exécution. Cette durée de vie est notamment attribuée aux littéraux de chaîne, qui ont le type &'static str.

Limites des traits de rang supérieur #

fn copy_if<F>(slice: &[i32], pred: F) -> Vec<i32>
    where for<'a> F: Fn(&'a i32) -> bool
{
    let mut result = vec![];
    for &element in slice {
        if pred(&element) {
            result.push(element);
        }
    }
    result
}

Cela spécifie que la référence sur i32 dans la limite de trait Fn peut avoir n’importe quelle durée de vie.

Ce qui suit ne fonctionne pas :

fn wrong_copy_if<'a, F>(slice: &[i32], pred: F) -> Vec<i32>
    where F: Fn(&'a i32) -> bool
{                                   // <----------------+
    let mut result = vec![];        //       'a scope   |
    for &element in slice {         // <--------+       |
        if pred(&element) {         //          |       |
            result.push(element);   // element's|       |
        }                           //   scope  |       |
    }                               // <--------+       |
    result                          //                  |
}                                   // <----------------+

Le compilateur donne l’erreur suivante :

error: `element` does not live long enough
if pred(&element) {         //          |       |
         ^~~~~~~

car la variable locale element ne vit pas aussi longtemps que la durée de vie 'a (comme nous pouvons le voir dans les commentaires du code).

La durée de vie ne peut pas être déclarée au niveau de la fonction, car nous avons besoin d’une autre durée de vie. C’est pourquoi nous avons utilisé for<'a>: pour spécifier que la référence peut être valide pour n’importe quelle durée de vie (donc une durée de vie plus petite peut être utilisée).

Les limites de trait de rang supérieur peuvent également être utilisées sur les structures :

struct Window<F>
    where for<'a> F: FnOnce(&'a Window<F>)
{
    on_close: F,
}

ainsi que sur d’autres articles.

Les limites de trait de rang supérieur sont le plus souvent utilisées avec les traits “Fn*”.

Pour ces exemples, l’élision de durée de vie fonctionne bien, nous n’avons donc pas à spécifier les durées de vie.

## Blocs d’implémentation impl<‘a> Type<‘a> { fn my_function(&self) -> &‘a u32 { self.x } }

Cela spécifie que Type a une durée de vie 'a et que la référence renvoyée par my_function() peut ne plus être valide après la fin de 'a car le Type n’existe plus pour contenir self.x .

Paramètres de fonction (Durée de vie des entrées) #

fn foo<'a>(x: &'a u32) {
    // ...
}

Cela spécifie que foo a une durée de vie 'a et que le paramètre x doit avoir une durée de vie d’au moins 'a. Les durées de vie des fonctions sont généralement omises via l’élision de la durée de vie :

fn foo(x: &u32) {
    // ...
}

Dans le cas où une fonction prend plusieurs références comme paramètres et renvoie une référence, le compilateur ne peut pas déduire la durée de vie du résultat via lifetime élision.

error[E0106]: missing lifetime specifier
1 | fn foo(bar: &str, baz: &str) -> &i32 {
  |                                 ^ expected lifetime parameter

Au lieu de cela, les paramètres de durée de vie doivent être explicitement spécifiés.

// Return value of `foo` is valid as long as `bar` and `baz` are alive.
fn foo<'a>(bar: &'a str, baz: &'a str) -> &'a i32 {

Les fonctions peuvent également prendre plusieurs paramètres de durée de vie.

// Return value is valid for the scope of `bar`
fn foo<'a, 'b>(bar: &'a str, baz: &'b str) -> &'a i32 {

Champs de structure #

struct Struct<'a> {
    x: &'a u32,
}

Cela spécifie que toute instance donnée de Struct a une durée de vie 'a, et le &u32 stocké dans x doit avoir une durée de vie d’au moins 'a.
