Iron est un framework Web populaire pour Rust (basé sur la bibliothèque de niveau inférieur Hyper) qui promeut l’idée d’extensibilité via * middleware*. Une grande partie des fonctionnalités nécessaires pour créer un site Web utile se trouvent dans le middleware d’Iron plutôt que dans la bibliothèque elle-même.

Serveur “Bonjour” simple #

Cet exemple envoie une réponse codée en dur à l’utilisateur lorsqu’il envoie une requête au serveur.

extern crate iron;

use iron::prelude::*;
use iron::status;

// You can pass the handler as a function or a closure. In this
// case, we've chosen a function for clarity.
// Since we don't care about the request, we bind it to _.
fn handler(_: &mut Request) -> IronResult<Response> {
    Ok(Response::with((status::Ok, "Hello, Stack Overflow")))
}

fn main() {
    Iron::new(handler).http("localhost:1337").expect("Server failed!")
}

Lors de la création d’un nouveau serveur “Iron” dans cet exemple, “attendez-vous” pour intercepter toutes les erreurs avec un message d’erreur plus descriptif. Dans les applications de production, gérez l’erreur produite (voir la documentation pour http()).

Routage simple avec du fer #

Cet exemple fournira un routage Web de base à l’aide de Iron.

Pour commencer, vous devrez ajouter la dépendance Iron à votre fichier Cargo.toml.

[dependencies]
iron = "0.4.*"

Nous utiliserons la propre bibliothèque de routeurs d’Iron. Pour plus de simplicité, le projet Iron fournit cette bibliothèque dans le cadre de la bibliothèque Iron core, supprimant ainsi tout besoin de l’ajouter en tant que dépendance distincte. Ensuite, nous référençons à la fois la bibliothèque Iron et la bibliothèque Router.

extern crate iron;
extern crate router;

Ensuite, nous importons les objets requis pour nous permettre de gérer le routage et renvoyons une réponse à l’utilisateur.

use iron::{Iron, Request, Response, IronResult};
use iron::status;
use router::{Router};

Dans cet exemple, nous allons faire simple en écrivant la logique de routage dans notre fonction main(). Bien sûr, à mesure que votre application se développe, vous souhaiterez séparer le routage, la journalisation, les problèmes de sécurité et d’autres domaines de votre application Web. Pour l’instant, c’est un bon point de départ.

fn main() {
    let mut router = Router::new();
    router.get("/", handler, "handler");
    router.get("/:query", query_handler, "query_handler");

Passons en revue ce que nous avons réalisé jusqu’à présent. Notre programme instancie actuellement un nouvel objet Iron Router et attache deux “gestionnaires” à deux types de requêtes d’URL : le premier ("/") est la racine de notre domaine, et le second ("/:query ") est n’importe quel chemin sous la racine.

En utilisant un point-virgule avant le mot “requête”, nous disons à Iron de prendre cette partie du chemin de l’URL en tant que variable et de la transmettre à notre gestionnaire.

La ligne de code suivante explique comment nous instancions Iron, en désignant notre propre objet “routeur” pour gérer nos requêtes d’URL. Le domaine et le port sont codés en dur dans cet exemple pour plus de simplicité.

    Iron::new(router).http("localhost:3000").unwrap();

Ensuite, nous déclarons deux fonctions en ligne qui sont nos gestionnaires, handler et query_handler. Ceux-ci sont tous deux utilisés pour démontrer les URL fixes et les URL variables.

Dans la deuxième fonction, nous prenons la variable "query" de l’URL détenue par l’objet de requête et nous la renvoyons à l’utilisateur en réponse.

    fn handler(_: &mut Request) -> IronResult<Response> {
        Ok(Response::with((status::Ok, "OK")))
    }

    fn query_handler(req: &mut Request) -> IronResult<Response> {
        let ref query = req.extensions.get::<Router>()
            .unwrap().find("query").unwrap_or("/");
        Ok(Response::with((status::Ok, *query)))
    }
}

Si nous exécutons cet exemple, nous pourrons voir le résultat dans le navigateur Web à localhost:3000. La racine du domaine doit répondre par "OK", et tout ce qui se trouve sous la racine doit répéter le chemin de retour.

La prochaine étape de cet exemple pourrait être la séparation du routage et du service des pages statiques.

[1] : http://hyper.rs

Installation du fer #

Ajoutez cette dépendance au fichier Cargo.toml :

[dependencies]
iron = "0.4.0"

Exécutez cargo build et Cargo téléchargera et installera la version spécifiée d’Iron.
