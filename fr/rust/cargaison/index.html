Cargo est le gestionnaire de paquets de Rust, utilisé pour gérer les caisses (terme de Rust pour les bibliothèques/paquets). Cargo récupère principalement des packages à partir de crates.io et peut gérer des arborescences de dépendances complexes avec des exigences de version spécifiques (à l’aide de la gestion sémantique des versions). Cargo peut également aider à créer, exécuter et gérer des projets Rust avec cargo build, cargo run et cargo test (entre autres commandes utiles).

Syntaxe #

  • cargaison nouveau nom_caisse [–bin]
  • chargement init [–bin]
  • version cargo [–release]
  • course de fret [–release]
  • vérification du fret
  • essai de cargaison
  • banquette de chargement
  • mise à jour de la cargaison
  • colis cargo
  • cargo publier
  • cargaison [dés]installer binary_crate_name
  • recherche de fret crate_name
  • version cargo
  • connexion cargo api_key
  • Pour le moment, la sous-commande cargo bench nécessite la version nocturne du compilateur pour fonctionner efficacement.

Créer un nouveau projet #

Bibliothèque #

cargo new my-library

Cela crée un nouveau répertoire appelé my-library contenant le fichier de configuration cargo et un répertoire source contenant un seul fichier source Rust :

my-library/Cargo.toml
my-library/src/lib.rs

Ces deux fichiers contiendront déjà le squelette de base d’une bibliothèque, de sorte que vous pouvez faire un cargo test (depuis le répertoire my-library) immédiatement pour vérifier si tout fonctionne.

Binaire #

cargo new my-binary --bin

Cela crée un nouveau répertoire appelé my-binary avec une structure similaire à celle d’une bibliothèque :

my-binary/Cargo.toml
my-binary/src/main.rs

Cette fois, cargo aura mis en place un simple fichier binaire Hello World que nous pouvons exécuter immédiatement avec cargo run.

Vous pouvez également créer le nouveau projet dans le répertoire courant avec la sous-commande init :

cargo init --bin

Comme ci-dessus, supprimez le drapeau --bin pour créer un nouveau projet de bibliothèque. Le nom du dossier actuel est automatiquement utilisé comme nom de crate.

Créer un projet #

Débogage #

cargo build

Sortie #

La construction avec le drapeau --release permet certaines optimisations du compilateur qui ne sont pas effectuées lors de la construction d’une version de débogage. Cela rend le code plus rapide, mais rend également le temps de compilation un peu plus long. Pour des performances optimales, cette commande doit être utilisée une fois qu’une version de version est prête.

cargo build --release

Tests en cours #

Utilisation de base #

cargo test

Afficher la sortie du programme #

cargo test -- --nocapture

Exécuter un exemple spécifique #

cargo test test_name

Publier une caisse #

Pour publier une caisse sur crates.io, vous devez vous connecter avec Cargo (voir ‘Connecter Cargo à un compte Crates.io’).

Vous pouvez empaqueter et publier votre crate avec les commandes suivantes :

cargo package
cargo publish

Toutes les erreurs dans votre fichier Cargo.toml seront mises en évidence au cours de ce processus. Vous devez vous assurer de mettre à jour votre version et vous assurer que votre fichier .gitignore ou Cargo.toml exclut tous les fichiers indésirables.

Connecter Cargo à un compte Crates.io #

Les comptes sur crates.io sont créés en se connectant avec GitHub ; vous ne pouvez pas vous inscrire avec une autre méthode.

Pour connecter votre compte GitHub à crates.io, cliquez sur le bouton « Connexion avec GitHub » dans la barre de menu supérieure et autorisez crates.io à accéder à votre compte. Cela vous connectera ensuite à crates.io, en supposant que tout s’est bien passé.

Vous devez ensuite trouver votre clé API, que vous trouverez en cliquant sur votre avatar, en vous rendant dans ‘Paramètres du compte’ et en copiant la ligne qui ressemble à ceci :

cargo login abcdefghijklmnopqrstuvwxyz1234567890rust

Cela devrait être collé dans votre terminal/ligne de commande, et devrait vous authentifier avec votre installation cargo locale.

Soyez prudent avec votre clé API - elle doit être gardée secrète, comme un mot de passe, sinon vos caisses pourraient être piratées !

Programme Hello world #

Il s’agit d’une session shell montrant comment créer un programme “Hello world” et l’exécuter avec Cargo :

$ cargo new hello --bin
$ cd hello
$ cargo run
   Compiling hello v0.1.0 (file:///home/rust/hello)
     Running `target/debug/hello`
Hello, world!

Après cela, vous pouvez éditer le programme en ouvrant src/main.rs dans un éditeur de texte.
