  • Responsifie et minimise les images

Tâche de surveillance Gulp #

Commençons par la tâche de surveillance.

Le but est de surveiller les changements que vous apportez pendant le développement. Tout changement devrait déclencher la tâche gulp correspondante.

De plus, nous avons besoin d’une fonctionnalité qui synchronise vos modifications sur le navigateur.

Synchronisation du navigateur

Donc, nous devons installer Browser Sync.

bash $ npm install browser-sync --save-dev

Avec cette prémisse, ouvrons notre gulpfile.js et ajoutons la fonctionnalité de surveillance. Exigeons la synchronisation du navigateur et définissons certaines variables pour utiliser ses fonctionnalités.

En haut du gulpfile , ajoutez l’extrait ci-dessous. Placez-le juste en dessous des déclarations de compression d’image.

ainsi:

//Browser-sync

var sync = require('browser-sync').create();
var reload = sync.reload;

Faire en sorte que la synchronisation du navigateur synchronise votre développement sur le navigateur est une configuration simple. Créons une tâche appelée watchdog.

ainsi:

$.task('watchdog', function() {


})

Maintenant, si nous parcourons les options de synchronisation du navigateur [ici] [1] et recherchons le paramètre du serveur, nous pouvons voir à quel point c’est facile.

[1] : https://www.browsersync.io/docs/options/

Nous avons juste besoin de placer le ci-dessous à l’intérieur de notre tâche de surveillance

Extrait - 1 - à l’intérieur du passe-partout du chien de garde #

/*
 Initiate Browser sync
 @documentation - https://www.browsersync.io/docs/options/
 */
 sync.init({
  server: {
    baseDir: "./"
 },
 port: 8000 //change it as required
 });

Insérez ce qui précède à l’intérieur de votre passe-partout de chien de garde ci-dessus.

L’extrait suivant consiste à définir un observateur pour les styles, dans le but de retraiter les fichiers CSS modifiés ou les nouveaux, et de déclencher automatiquement un rechargement du navigateur.

extrait - 2 - à l’intérieur du passe-partout du chien de garde #

$.watch(['css/**/*', 'fonts/google/**/*.css'], reload).on('change', function(event) {
console.log(event.type + ':' + event.path)
if (event.type === 'deleted') {
uncache('styles', event.path);
$$.remember.forget('auto-prefixed-stylesheets', event.path);
}
sequence('optimizeStyles')
});

Insérez ce qui précède à l’intérieur de votre passe-partout de chien de garde ci-dessus.

Nous surveillons donc " [fonts/google/**/*.css , /**/*.css ]" c’est-à-dire

tous les fichiers CSS sous css/ tous les fichiers CSS sous fonts/google/ Lorsque quelque chose change ou qu’un nouveau fichier est ajouté, il déclenche la méthode de rechargement, qui est définie en haut de notre gulpfile, dans la déclaration de browsersync.

Remarque : Vous remarquerez peut-être que nous avons un gestionnaire d’événements .on attaché à l’observateur.

$.watch(['css/**/*', 'fonts/google/**/*.css'], reload).on('change', function(event)

Fondamentalement, tout CUD(Create | Update | Delete) déclenche la fonction de rechargement et passe un objet d’événement en tant que paramètre au fonction de rappel.

Le rappel est une fonction vitale, où nous pouvons réaliser des opérations comme uncache lors de la suppression d’actifs. Maintenant, l’objet événement a des paramètres comme

  • chemin
  • type - Créer/Mettre à jour/Supprimer

Si un actif est supprimé, nous devons nous assurer que les caches que nous avons construits dans nos fonctions de minification précédentes, via gulp-cached et gulp-remember, ont besoin d’être mis à jour.

nous gérons cela dans l’extrait ci-dessous, qui se trouve à l’intérieur du rappel lors du changement.

if (event.type === 'deleted') { 
uncache('styles', event.path);
$$.remember.forget('auto-prefixed-stylesheets', event.path);  
}

Noter #

$ - > alias pour gulp

$$ - > alias pour gulp-load-plugins

vous remarquerez peut-être aussi que j’ai une sequence('optimizeStyles'); après avoir écrit l’invocation uncache

La méthode de séquence , assure , la méthode synchrone s’exécute dans un javascript asynchrone par défaut.

l’installation est simple

FAIS

bash $ npm install run-sequence

puis, déclarez-le dans votre gulpfile juste en dessous de la déclaration de synchronisation du navigateur.

var sequence = require('run-sequence');

Donc, avec cette compréhension, l’observateur de scripts est facile à écrire. juste des globs différents !

Alors, ajoutez cet extrait sous l’observateur de style à l’intérieur du passe-partout du chien de garde.

Extrait - 3 - à l’intérieur du passe-partout de la tâche Watchdog #

/*
on addition or change or deletion of a file in the watched directories
the change event is triggered. An event object with properties like
path,
event-type
is available for perusal passed to the callback

*/
$.watch('js/**/*', reload).on('change', function(event) {
console.log(event.type + ':' + event.path)
if (event.type === 'deleted') {
uncache('scripts', event.path);
$$.remember.forget('linted-scripts', event.path);
}
sequence('optimizeScripts');
});

Définir une tâche par défaut #

Alors maintenant, terminons le code gulpfile , en définissant une tâche par défaut.

la tâche par défaut est celle qui s’exécute, lorsque vous dites simplement

gulp

sur une invite de commande sous la racine de votre projet.

$.task('default', ['generateResponsiveImages'], function() {
 $.start('watchdog');
 console.log('Starting Incremental Build');
});
