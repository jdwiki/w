Chargement de tous les plugins à partir de Package.JSON #

En supposant que vous compreniez comment installer gulp, laissez-nous plonger jusqu’à exiger toutes les dépendances gulp- de package.json sous le dossier racine de vos projets.

Si vous n’avez pas encore de fichier gulp, veuillez créer un fichier vide avec le nom

gulpfile.js

Tout d’abord, nous avons besoin de gulp. ainsi:

var $ = require('gulp');

Ensuite, chargeons tous nos plugins, dans notre gulpfile, par l’extrait ci-dessous

Noter #

Veuillez lire les commentaires dans tous les extraits que nous inclurons dans cette lecture, ils fournissent plus d’informations sur chaque fonctionnalité

/*
require gulp-load-plugins, and call it
gulp-load-plugins will require individually,
all the plugins installed in package.json at the root directory
these required plugins, are lazy loaded
that is, gulp-load-plugins will only require them when they are referenced in this file
plugins are available for perusal, via camelcased names, minus gulp
eg: gulp-clean-css is accessed by $$.cleanCss
*/
var $$ = require('gulp-load-plugins')();

REMARQUE #

Tout au long des nombreux exemples que nous aurons dans l’article, nous alias

  1. avaler comme $ et
  2. gulp-load-plugins comme $$

uniquement pour faciliter la saisie.

Installation de plugins pour les images responsives|Css Minification|Js minification #

Notre objectif, c’est de

  1. Rendez vos images conformes aux largeurs et à l’échelle de manière appropriée, en générant une batterie d’images de largeurs variées, et pour les réduire
  2. Lintez votre Javascript
  3. Minimisez vos actifs - JS/CSS/HTML, vous permettant ainsi d’héberger un code plus léger que le plus léger
  4. Surveillez les fichiers css/js/image pour les modifications et reconstruisez les optimisations
  5. Synchronisation de vos modifications en cours de développement sur un navigateur desservant votre site.

Nous avons besoin d’un certain nombre de plugins, alors installons-les tous. Veuillez exécuter tout cela dans le dossier racine du projet.

Anatomie d’une fonction gulp #

[Function <name>] (glob) {

$.src(glob)

.pipe([plugin 1])
.pipe([plugin 2])
.
.
.
.pipe([plugin n])
.pipe( $.dest(<destination-name>)

}

Noter #

pipe est une méthode qui diffuse tous les fichiers correspondant à l’entrée glob vers nos plugins (minifyhtml dans ce cas).

C’est simple de l’imaginer comme ça :

$.src est ce qui construit le flux et le tuyau vers chaque individu fichier correspondant au glob vers le bas pour chaque plugin dans le pipeline.Chaque plugin auquel le fichier est passé, modifie son contenu en mémoire uniquement jusqu’à ce que $.dest soit atteint, qui met alors à jour/crée les fichiers diffusés par $.src

Où ,

$ - > gorgée #

$$ - > gulp-load-plugins #

Optimisation et minification des actifs #

Donc, avant d’écrire les fonctions d’optimisation, nous devons installer quelques plugins de mise en cache.

bash $ npm install --save-dev gulp-cached
bash $ npm install --save-dev gulp-remember

Vous pourriez vous demander pourquoi deux caches hein !. gulp-cached , transmet uniquement le contenu modifié ou nouveau dans le pipeline vers d’autres plugins. Donc, puisque nous voulons que les fichiers sans modification soient également utilisés pour la concaténation en un seul fichier par actif ( css | js ), nous avons besoin de ** gulp-remember ** en plus de gulp-cached

Nous utilisons d’abord gulp-cached pour créer une liste des fichiers qui ont changé

Deuxièmement, nous avons besoin de gulp-remember pour garder une trace de tous les fichiers qui sont passés par cette liste en mémoire.

Première exécution : aucun fichier n’est mis en cache, donc gulp-cached les transmettra tous à gulp-remember

Exécutions suivantes : Seuls les fichiers modifiés ou nouveaux sont redirigés par gulp-cached. Puisque le contenu du fichier courant a changé, gulp-remember met à jour son cache.

Cool , Écrivons notre premier optimiseur

Optimiseur de style #


// Goal

/*

1. cache existing files on the first run
2. each file , 
       a. Is autoprefixed with cross browser compatible prefixes for any css property ( justify-content for e.g)
       b. Is concatenated into app.css
3. app.css is minified
4. On subsequent runs , the above process is implemented on file modifications/additions only

*/

/*
*@src - input a glob pattern - a string eg 'images/**/*' or '**/*.css' or, an array eg ['glob1','glob2']
*/
var optimizeStyles = function(src) {

return $.src(src).
pipe($$.cached('styles')).
pipe($$.autoprefixer({
browsers: ['last 2 versions']
})).
pipe($$.remember('auto-prefixed-stylesheets')).
pipe($$.concat('app.css')).
pipe($.dest('build/css')).
pipe($$.cleanCss()).
pipe($$.rename({
suffix: '.min'
})).
pipe($.dest('build/css'))
}

Noter #

$ - > gorgée

$$ - > gulp-load-plugins

$.src -> construit les flux de fichiers correspondant au glob passé en tant que src

$.dest -> enregistre le fichier manipulé dans le chemin spécifié

Optimiseur de script #


// Goal

/*

1. cache existing files on the first run
2. each file , 
       a. Is linted with jshint 
       b. Is concatenated into app.js
3. app.js is minified
4. On subsequent runs , the above process is implemented on file modifications/additions only

*/

/*
*@src - input a glob pattern - a string eg 'js/**/*' or '**/*.js' or, an array eg ['glob1','glob2']
*/

var optimizeScripts = function(src) {

    return $.src(src).
    pipe($$.cached('scripts')).
    pipe($$.jshint()).
    pipe($$.jshint.reporter('default')).
    pipe($$.jshint.reporter('fail')).
    pipe($$.remember('linted-scripts')).
    pipe($$.concat('app.js')).
    pipe($.dest('build/js')).
    pipe($$.uglify()).
    pipe($$.rename({
        suffix: '.min'
    })).
    pipe($.dest('build/js'))


}

Noter #

$ - > gorgée

$$ - > gulp-load-plugins

$.src -> construit les flux de fichiers correspondant au glob passé en tant que src

$.dest -> enregistre le fichier manipulé dans le chemin spécifié

Générer des images réactives #

Passons maintenant au traitement des images. Ainsi, le but est d’avoir un tableau de tailles pour chaque image que vous allez servir.

Pourquoi?

Eh bien , pour comprendre pourquoi nous avons besoin d’ une batterie d’ images avec une gamme de largeurs , nous devons réfléchir au fait qu’il existe probablement des millions d’appareils avec des résolutions variées. Nous avons besoin d’une image à l’échelle sans trop de pixellisation. Dans le même temps , nous devons améliorer les temps de chargement des pages , en téléchargeant uniquement les une image , qui s’adapte à la largeur dans laquelle elle est contenue , et est également avec la plus petite dimension possible, pour ce faire. Il existe des blogs scientifiques comme celui écrit par Eric Portis, qui met en lumière l’inefficacité des requêtes des médias uniquement et sert de guide complet pour comprendre des concepts comme les srcsets et les tailles.

Vous pouvez vous référer à [l’épopée d’Eric Portis ici][1]

Maintenant, notre fonction doit prendre un glob et une largeur comme entrées, et faire sa magie et pousser le fichier généré par chaque exécution vers une destination et réduire l’image responsifiée.

Nous avons installé deux plugins de compression d’image [dans notre premier exemple][2]

Étant donné que ces plugins NE PAS commencent par un préfixe “gulp-”, nous devons les charger manuellement dans notre fichier gulp.

SO Exigons-les manuellement, après la déclaration gulp-load-plugins en haut de notre fichier gulp.

ainsi:

var compressJpg = require('imagemin-jpeg-recompress');
var pngquant = require('imagemin-pngquant');

Il est intéressant de noter que la réponse à la déglutition vient avec la netteté processeur d’image, qui est meilleur que imagemagick BY FAAAAR !. Tranchant est ce qui est utilisé par gulp-responsive pour recadrer vos images à votre guise largeurs.

vous pourriez regarder gulp-responsive-configuration-options, pour un liste complète des paramètres de configuration. je n’ai utilisé que

  • width - pour recadrer nos images à une largeur w, passée en paramètre
  • renommer - pour ajouter un suffixe au nom de l’image, afin qu’elle reste unique

dans ma fonction de configuration ci-dessous. donc notre fonction recadrera l’image à la largeur passée en entrée, pour toutes les images correspondantes déchiffrées via l’entrée glob. ensuite, chaque image est compressée avec jpeg-recompress ou pngquant et enregistré dans build/images.

Dans cet esprit, notre fonction ressemblerait à ceci :

/*
@generateResponsiveImages
*@Description:takes in a src of globs, to stream matching image files , a width,
*to resize the matching image to, and a dest to write the resized and minified files to
*@src - input a glob pattern - a string eg 'images/** /*' or 'images/*' or, an array
eg ['glob1','glob2']
*@return returns a stream
*/
var generateResponsiveImages = function(src, width, dest) {

    //declare a default destination
    if (!dest)
        dest = 'build/images';
    //case 1: src glob -  images/**/*
    // the base is the directory immediately preceeding the glob - images/ in this case
    //case 2: images/fixed/flourish.png : the base here is images/fixed/ - we are overriding
    // that by setting base to images.This is done so that, the path beginning after images/
    // - is the path under our destination - without us setting the base, dest would be,
    //build/images/flourish.png.But with us setting the base, the destination would be
    // build/images/fixed/flourish.png
    return $.src(src, {
        base: 'images'
    })

    //generate resized images according to the width passed
    .pipe($$.responsive({
            //match all pngs within the src stream
            '**/*.png': [{
                width: width,
                rename: {
                    suffix: '-' + width
                },
                withoutEnlargement: false,
            }],
            //match all jpgs within the src stream
            '**/*.jpg': [{
                width: width,
                rename: {
                    suffix: '-' + width
                },
                progressive: true,
                withoutEnlargement: false,
            }]
        }, {

            errorOnEnlargement: false,
            errorOnUnusedConfig: false,
            errorOnUnusedImage: false

        }))
        //once the file is resized to width, minify it using the plugins available per format
        .pipe($$.if('*.jpg', compressJpg({
            min: 30,
            max: 90,
            target: 0.5
        })()))
        //use file based cache gulp-cache and it will minify only changed or new files
        //if it is not a new file and if the contents havent changed, the file is served from cache
        .pipe($$.cache($$.imagemin({
            verbose: true
        })))


    //write to destination - dest + path from base
    .pipe($.dest(dest));
}

Noter #

$ - > gorgée

$$ - > gulp-load-plugins

$.src -> construit les flux de fichiers correspondant au glob passé en tant que src

$.dest -> enregistre le fichier manipulé dans le chemin spécifié

Minificateur HTML #

*
 *@minifyHtml
 *Description:takes in a glob src, and minifies all '.html' files matched by the glob
 *@src - input a glob pattern - a string eg '/**/*.html /*' or '*.html' or, an array eg ['glob1','glob2']
 *@dest=file.base means, the modified html file will be in the same directory as the src file being minified
 *often means, the process is just a modification on the existing src file
 *@return returns a stream
 */
var minifyHtml = function(src) {
    return $.src(src)
        .pipe($$.minifyHtml())
        .pipe($.dest(function(file) {
            //file is provided to a dest callback -
            // Refer here https://github.com/gulpjs/gulp/blob/master/docs/API.md#path
            return file.base;
        }));
}

Anatomie d’une tâche de gulp #

L’anatomie d’une définition de tâche est la suivante :

$.task(<taskname> , [dependencies] , <body>);

dependencies , is an array of tasks that HAVE to finish before the current task you are defining , runs. More like forcing a synchronous execution instead of the default Asynchronous functionality.

Adding Gulp Tasks #

So, we now have

  • A function defined Above to optimise Styles
  • A function defined Above to optimise scripts
  • A function defined Above to optimise HTML
  • A function to generate multiple images per image Above

All we need to do now, is to invoke them when needed.

Let us write our tasks according to the syntax we defined earlier

/*
* $.task('name','dependency array',function)
aboutit à la construction d'un objet de tâche comme ci-dessous
tâche:{
'nom nom,
'dep' :[tableau de dépendances],
'fn' : fonction
}
*/


//*@return renvoie un flux pour notifier l'achèvement de la tâche
$.task('optimizeHtml', function() {
    var src = ['**/*.html', '!{node_modules}/**/*.html'];
    return minifyHtml(src);
});

//*@return renvoie un flux pour notifier l'achèvement de la tâche
$.task('optimizeScripts', function() {
    var src = ['js/**/*.js'];
    return optimizeScripts(src);
});

//*@return renvoie un flux pour notifier l'achèvement de la tâche
$.task('optimizeStyles', function() {
    var src = ['css/**/*.css', 'fonts/google/**/*.css'];
    return optimizeStyles(src);
});

// Prenez un rappel pour vous assurer de notifier au moteur gulp que la tâche est terminée
//requis car vous ne renvoyez pas de flux dans cette tâche
$.task('generateResponsiveImages', function(callback) {
    var src = ['images/**/*.{jpg,png}'];
    for (var i = widths.length - 1; i >= 0; i--) {
        generateResponsiveImages(src, widths[i]);
    }
    callback();

});
