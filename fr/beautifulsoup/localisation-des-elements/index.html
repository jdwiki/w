Localiser un texte après un élément dans BeautifulSoup #

Imaginez que vous avez le code HTML suivant :

<div>
    <label>Name:</label>
    John Smith
</div>

Et vous devez localiser le texte “John Smith” après l’élément label.

Dans ce cas, vous pouvez localiser l’élément label par texte, puis utiliser la [propriété .next_sibling][1] :

from bs4 import BeautifulSoup

data = """
<div>
    <label>Name:</label>
    John Smith
</div>
"""

soup = BeautifulSoup(data, "html.parser")

label = soup.find("label", text="Name:")
print(label.next_sibling.strip())

Imprime ‘John Smith’.

[1] : https://www.crummy.com/software/BeautifulSoup/bs4/doc/#next-sibling-and-previous-sibling

Localisation des commentaires #

Pour localiser les commentaires dans BeautifulSoup, utilisez l’argument text (ou [string][1] dans les versions récentes) en vérifiant que le type est [Comment][2] :

from bs4 import BeautifulSoup
from bs4 import Comment

data = """
<html>
    <body>
        <div>
        <!-- desired text -->
        </div>
    </body>
</html>
"""

soup = BeautifulSoup(data, "html.parser")
comment = soup.find(text=lambda text: isinstance(text, Comment))
print(comment)

Imprime le “texte souhaité”.

[1] : https://www.crummy.com/software/BeautifulSoup/bs4/doc/#the-string-argument [2] : https://www.crummy.com/software/BeautifulSoup/bs4/doc/#comments-and-other-special-strings

Fonctions de filtrage #

BeautifulSoup vous permet de filtrer les résultats en fournissant une fonction pour “find_all” et des fonctions similaires. Cela peut être utile pour les filtres complexes ainsi qu’un outil pour la réutilisation du code.

Utilisation de base #

Définissez une fonction qui prend un élément comme seul argument. La fonction doit renvoyer “True” si l’argument correspond.

def has_href(tag):
    '''Returns True for tags with a href attribute'''
    return  bool(tag.get("href"))

soup.find_all(has_href) #find all elements with a href attribute
#equivilent using lambda:
soup.find_all(lambda tag: bool(tag.get("href")))

Un autre exemple qui trouve des balises avec une valeur href qui ne commencent pas par

Fournir des arguments supplémentaires pour filtrer les fonctions #

Puisque la fonction passée à find_all ne peut prendre qu’un seul argument, il est parfois utile de créer des ‘usines de fonctions’ qui produisent des fonctions utilisables dans find_all. Ceci est utile pour rendre vos fonctions de recherche de balises plus flexibles.

def present_in_href(check_string):
    return lambda tag: tag.get("href") and check_string in tag.get("href")

soup.find_all(present_in_href("/partial/path"))

Utilisation de sélecteurs CSS pour localiser des éléments dans BeautifulSoup #

BeautifulSoup a une [prise en charge limitée des sélecteurs CSS][1], mais couvre les plus couramment utilisés. Utilisez la méthode select() pour trouver plusieurs éléments et select_one() pour trouver un seul élément.

Exemple de base :

from bs4 import BeautifulSoup

data = """
<ul>
    <li class="item">item1</li>
    <li class="item">item2</li>
    <li class="item">item3</li>
</ul>
"""

soup = BeautifulSoup(data, "html.parser")

for item in soup.select("li.item"):
    print(item.get_text())

Impressions :

item1
item2
item3

[1] : https://www.crummy.com/software/BeautifulSoup/bs4/doc/#css-selectors

Accéder aux balises internes et à leurs attributs de la balise initialement sélectionnée #

Supposons que vous ayez un html après avoir sélectionné avec soup.find('div', class_='base class') :

from bs4 import BeautifulSoup

soup = BeautifulSoup(SomePage, 'lxml')
html = soup.find('div', class_='base class')
print(html)

<div class="base class">
  <div>Sample text 1</div>
  <div>Sample text 2</div>
  <div>
    <a class="ordinary link" href="https://example.com">URL text</a>
  </div>
</div>

<div class="Confusing class"></div>
'''

Et si vous voulez accéder au href de la balise <a>, vous pouvez le faire de cette façon :

a_tag = html.a
link = a_tag['href']
print(link)

https://example.com

Ceci est utile lorsque vous ne pouvez pas sélectionner directement la balise <a> car attrs ne vous donne pas d’identification unique, il existe d’autres balises “jumelles” <a> dans la page analysée. Mais vous pouvez sélectionner de manière unique une balise parent contenant le <a> nécessaire.

Collecte d’éléments optionnels et/ou de leurs attributs à partir de séries de pages #

Considérons la situation où vous analysez le nombre de pages et que vous souhaitez collecter la valeur d’un élément qui est facultatif (peut être présenté sur une page et peut être absent sur une autre) pour une page particulière.

De plus, l’élément lui-même, par exemple, est l’élément le plus ordinaire de la page, en d’autres termes aucun attribut spécifique ne peut le localiser de manière unique. Mais vous voyez que vous pouvez sélectionner correctement son élément parent et que vous connaissez le numéro de commande de l’élément souhaité dans le niveau d’imbrication respectif.

from bs4 import BeautifulSoup

soup = BeautifulSoup(SomePage, 'lxml')
html = soup.find('div', class_='base class') # Below it refers to html_1 and html_2

L’élément Wanted est facultatif, il peut donc y avoir 2 situations pour que html soit :

html_1 = '''
<div class="base class">    # №0
  <div>Sample text 1</div>  # №1
  <div>Sample text 2</div>  # №2  
  <div>!Needed text!</div>  # №3
</div>

<div>Confusing div text</div>  # №4
'''
        
html_2 = '''
<div class="base class">    # №0
  <div>Sample text 1</div>  # №1
  <div>Sample text 2</div>  # №2  
</div>

<div>Confusing div text</div>  # №4
'''

Si vous avez html_1, vous pouvez collecter !Texte nécessaire! de la balise №3 de cette façon :

wanted tag = html_1.div.find_next_sibling().find_next_sibling() # this gives you whole tag №3

Il obtient initialement №1 div, puis passe 2 fois à la prochaine div au même niveau d’imbrication pour arriver à №3.

wanted_text = wanted_tag.text # extracting !Needed text!

L’utilité de cette approche vient lorsque vous obtenez html_2 - l’approche ne vous donnera pas d’erreur, elle donnera None :

print(html_2.div.find_next_sibling().find_next_sibling())
None

L’utilisation de find_next_sibling() ici est cruciale car elle limite la recherche d’éléments par niveau d’imbrication respectif. Si vous utilisez find_next() alors la balise №4 sera collectée et vous ne le voulez pas :

print(html_2.div.find_next().find_next())
<div>Confusing div text</div>

Vous pouvez également explorer find_previous_sibling() et find_previous() qui fonctionnent de manière opposée.

Toutes les fonctions décrites ont leurs variantes miltiple pour attraper toutes les balises, pas seulement la première :

find_next_siblings()
find_previous_siblings()
find_all_next()
find_all_previous()
