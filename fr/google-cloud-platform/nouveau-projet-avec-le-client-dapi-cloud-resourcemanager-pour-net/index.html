Nouveau projet avec le client d'API Cloud Resource Manager pour .NET

  • Commencer

Nous utiliserons Google API Client Libraries pour .NET pour cet exemple.

Il existe d’autres bibliothèques. Veuillez consulter Google Client Libraries Explained pour plus de détails.

Nous utiliserons l’API Cloud Resource Manager pour [Créer et gérer des projets](https://cloud.google.com/resource-manager/docs/ création-gestion-de-projets).

Commençons.

Mettre tous ensemble…

Vous devriez avoir deux fichiers. Le premier fichier s’appelle packages.config ou project.json.

Appelons le second fichier Program.cs. Tout le code inclus dans les sections ci-dessus peut être collé dans une seule méthode principale :

using System;
using System.Collections.Generic;
using System.Threading.Tasks;

using Google.Apis.Auth.OAuth2;
using Google.Apis.CloudResourceManager.v1;
using Google.Apis.Services;

using Data = Google.Apis.CloudResourceManager.v1.Data;

namespace OurFirstProject
{
    public class Program
    {
        private const string projectId = [YOUR-PROJECT-ID];
        private const string applicationName = "Test";

        public static void Main(string[] args)
        {

            var scopes = new String[] {
                CloudResourceManagerService.Scope.CloudPlatform
            };

            GoogleCredential credential = Task.Run(
                () => GoogleCredential.GetApplicationDefaultAsync()
            ).Result;

            if (credential.IsCreateScopedRequired)
            {
                credential = credential.CreateScoped(scopes);
            }

            CloudResourceManagerService service = new CloudResourceManagerService(
                new BaseClientService.Initializer()
                {
                    HttpClientInitializer = credential,
                    ApplicationName = applicationName
                }
            );

            Console.WriteLine("1. Create Project");
            Data.Operation operation1 = service.Projects.Create(
                new Data.Project()
                {
                    ProjectId = projectId,
                }
            ).Execute();

            Console.Write("2. Awaiting Operation Completion");
            Data.Operation operation2;
            do
            {
                operation2 = service.Operations.Get(operation1.Name).Execute();
                Console.WriteLine(operation2.Done.ToString());
                System.Threading.Thread.Sleep(1000);
            } while (operation2.Done != true);

            Console.WriteLine();
            Console.WriteLine("Enter to continue");
            Console.ReadLine();

            Console.WriteLine("3. Deleting Project");
            var operation3 = service.Projects.Delete(projectId).Execute();
        }
    }
}

Commencer #

Le code fonctionnera sous Windows et Linux.

Il fonctionne sur .NET Core ou .NET Standard. La seule différence est que, si vous souhaitez exécuter sur .NET Core, vous devez utiliser project.json. Si vous souhaitez exécuter sur .NET Standard, vous devez utiliser packages.config.

La bibliothèque cliente d’API pour l’API Cloud Resource Manager est disponible sur nuget :

https://www.nuget.org/packages/Google.Apis.CloudResourceManager.v1/

La version au moment de la rédaction est : 1.22.0.809

Nous procéderons de 2 manières :

  • Windows avec la norme .NET ; et
  • Linux avec .NET Core.

Les fenêtres

Si vous utilisez Visual Studio, créez une nouvelle “application console” “Visual C#”. Sinon, créez un répertoire pour le projet et créez-y un fichier appelé “packages.config”. packages.config est pour .NET Standard mais nous utilisons .NET Standard avec Windows. Vous ne pouvez exécuter .NET Standard que sous Windows. Remplacez le contenu de packages.config par :

﻿<?xml version="1.0" encoding="utf-8"?>
<packages>
  <package id="Google.Apis" version="1.22.0"
    targetFramework="net452" />
  <package id="Google.Apis.Auth" version="1.22.0"
    targetFramework="net452" />
  <package id="Google.Apis.CloudResourceManager.v1" version="1.22.0.809" 
    targetFramework="net452" />
  <package id="Google.Apis.Core" version="1.22.0"
    targetFramework="net452" />
</packages>

Linux

Créez un répertoire pour le projet et créez-y un fichier appelé “project.json”. project.json est pour .NET Core mais nous utilisons .NET Core avec Linux dans cet exemple. Vous pouvez exécuter .NET Core sur Linux ou sur Windows. Remplacez le contenu de project.json par :

{
  "version": "1.0.0-*",
  "buildOptions": {
    "debugType": "portable",
    "emitEntryPoint": true
  },
  "dependencies": {},
  "frameworks": {
    "netcoreapp1.1": {
      "dependencies": {
        "Microsoft.NETCore.App": {
          "type": "platform",
          "version": "1.1.0"
        },
        "Google.Apis.CloudResourceManager.v1":"1.22.0.809"
      },
      "imports": "dnxcore50"
    }
  }
}

Identifiants par défaut de l’application #

Je ne vais pas tout répéter ici : “Application Default Credentials fournit un moyen simple d’obtenir des identifiants d’autorisation pour utiliser les API Google. "

Si vous pouvez utiliser les informations d’identification par défaut de l’application, faites-le.

Il y a une étape supplémentaire que vous devrez effectuer avant d’utiliser les informations d’identification par défaut de l’application comme identité lors de l’appel d’API à partir de votre machine :

gcloud auth application-default login [yourname@gmail.com]

Voici pourquoi vous préférerez utiliser les informations d’identification par défaut de l’application :

var scopes = new String[] {
    CloudResourceManagerService.Scope.CloudPlatform
};

GoogleCredential credential = Task.Run(
    () => GoogleCredential.GetApplicationDefaultAsync()
).Result;

if (credential.IsCreateScopedRequired)
{
    credential = credential.CreateScoped(scopes);
}

… C’est tout le code dont vous avez besoin pour autoriser les appels vers (n’importe quelle) API Google Cloud !

Nous utiliserons l’objet credential à l’étape suivante pour effectuer des appels contre le service Google…

Appel de n’importe quelle méthode (!) sur n’importe quel service Google (!) #

Une fois que vous vous serez familiarisé avec le code pour appeler une méthode sur un service Google, vous pourrez déduire comment appeler n’importe quelle méthode sur n’importe service Google.

Tout d’abord, nous établissons une connexion au service à l’aide de l’objet credential instancié dans l’exemple précédent :

CloudResourceManagerService service = new CloudResourceManagerService(
    new BaseClientService.Initializer()
    {
        HttpClientInitializer = credential,
        ApplicationName = "Our First Google API Client"
    }
);

Ensuite, nous pouvons appeler les méthodes fournies par le service. Quelles sont les méthodes disponibles ?

https://cloud.google.com/resource-manager/docs/apis

Quelle est la requête REST sous-jacente pour Projects.Create ?

https://cloud.google.com/resource-manager/reference/rest/v1/projects/create

OK… Écrivons le code.

Le code attend une valeur de chaîne pour projectId. Les ID de projet sont des identifiants uniques. Je vous recommande d’utiliser un système de nommage de vos projets pour vous aider à les identifier.

Projects.Create attend un objet Data.Project. Cet objet est une propriété obligatoire, l’ID du projet qui est tout ce que nous fournirons, mais nous pourrions également fournir un nom de projet, des étiquettes, des détails sur le parent du projet, etc.

Data.Operation operation1 = service.Projects.Create(
    new Data.Project()
    {
        ProjectId = projectId,
    }
).Execute();

La création de projet est gérée de manière asynchrone. On nous donne un objet Operation que nous devons interroger pour déterminer quand le projet est créé. Les opérations ont une propriété Name qui identifie l’opération de manière unique. La section suivante du code interroge la plate-forme “Avons-nous encore terminé ?”. Le projet sera créé lorsque notre nouvelle opération inclura une propriété Done qui est True.

Data.Operation operation2;
do
{
    operation2 = service.Operations.Get(operation1.Name).Execute();
    System.Threading.Thread.Sleep(1000);
} while (operation2.Done != true);

Pour être complet et, espérons-le, dans de nombreuses années après une bonne utilisation de votre projet, vous devrez peut-être supprimer votre projet. Nous appelons simplement Projects.Delete et fournissons notre ID de projet. Cela renvoie également une opération et nous devrions également interroger cette opération jusqu’à ce qu’elle se termine définitivement. Notre projet sera alors supprimé.

var operation3 = service.Projects.Delete(projectId).Execute();

C’est ça!
