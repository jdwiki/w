Comment connecter Google Cloud SQL avec des applications (comme Google App Engine) et des outils comme (mySQL workbench) ? #

Dans ce document, nous verrons comment créer une instance Google Cloud SQL et les connecter dans votre application Google App Engine et l’outil d’administration MySQL Workbench.

Google Cloud SQL :

Google Cloud SQL est un service de base de données entièrement géré qui facilite la configuration, la maintenance, la gestion et l’administration de vos bases de données relationnelles MySQL dans le cloud.

Google Cloud SQL fournit une base de données relationnelle que vous pouvez utiliser avec votre application App Engine. Cloud SQL est une base de données MySQL qui réside dans le cloud de Google.

se référer à:

https://cloud.google.com/sql/

https://cloud.google.com/sql/docs/

Création d’instances SQL :

Une instance Google Cloud SQL est une base de données MySQL hébergée dans le cloud de Google.

  1. Accédez à la page des instances Cloud SQL dans Google Cloud Platform Console (https://console.cloud.google.com/sql/instances) and Click Create instance.

  2. Cliquez sur Choisir la première génération, entrez un nom et choisissez un niveau pour the instance and Click Create.

  3. Une fois l’initialisation de l’instance terminée, sélectionnez l’instance à open it.

  4. Dans Contrôle d’accès > Utilisateurs, cliquez sur Créer un compte utilisateur et créez un user with name root and specify a password (root_password). This creates the MySQL user ‘root’@’%’.

  5. Dans Bases de données, cliquez sur Nouvelle base de données et créez une base de données avec un DataBase name (DataBase_Name)

Atelier MySQL :

MySQL Workbench est un outil visuel unifié pour les architectes de bases de données, les développeurs et les DBA. MySQL Workbench fournit la modélisation des données, le développement SQL et des outils d’administration complets pour la configuration du serveur, l’administration des utilisateurs, la sauvegarde et bien plus encore.

consultez http://www.mysql.com/products/workbench/

Nous allons maintenant voir comment vous connecter à votre base de données d’instance Google Cloud SQL avec MySQL Workbench.

Configuration de l’accès

  1. Accédez à la page des instances Cloud SQL dans Google Cloud Platform Console and select the instance.

  2. Dans Contrôle d’accès > Adresse IP, cliquez sur Demander une adresse IPv4 et copiez it(Instance_IPv4_address). It is needed to connect your Google Cloud SQL instance database with Admin tools like MySQL Workbench.

    note: You will be charged for IPv4 address @ $0.01 each hour the instance is inactive and $0.1 each hour the instance is active

  3. Google ‘adresse IP’ pour trouver votre adresse IP publique

  4. Dans Contrôle d’accès > Autorisation > Réseaux autorisés, cliquez sur Ajouter network and enter your IP address.

  5. Dans Contrôle d’accès > Utilisateurs, Créez un utilisateur avec un nom d’utilisateur (userName), password (password) and the option ‘Allow any host selected’. It is recommended to use a seperate user account to access from WorkBench

De liaison

  1. Dans la vue d’accueil de MySQL Workbench, cliquez sur Nouvelle connexion.

  2. Dans la fenêtre Configurer une nouvelle connexion, indiquez un nom de connexion, Hostname and Username

  3. Cliquez sur Tester la connexion. Un mot de passe vous sera demandé.

  4. Une fois la connexion MySQL établie avec succès, cliquez sur OK et cliquez sur the saved connection to open SQL Editor

Google App Engine:

Google App Engine est une plate-forme permettant de créer des applications Web évolutives et des backends mobiles. App Engine fera automatiquement évoluer votre application.

consultez https://cloud.google.com/appengine

Nous allons maintenant voir comment configurer une connexion entre une application App Engine et une instance Cloud SQL.

Configuration de l’accès

  1. Accédez à la page des instances Cloud SQL dans Google Cloud Platform Console and select the instance.

  2. Dans Contrôle d’accès > Autorisation > App Engine autorisé applications, click Add application ID and enter the application ID. Click Done and Save.

  3. Dans Présentation > Propriétés, copiez le ‘Nom de connexion de l’instance’ (Instance_Connection_Name)

  4. Dans votre projet d’application Web Google, war/WEB-INF/appengine-web.xml add, true

Échantillon de code :

Un exemple pour Google App Engine - Environnement standard Java

public static Connection connect() throws ClassNotFoundException, SQLException {
    String url = null;

    {
        if (SystemProperty.environment.value() == SystemProperty.Environment.Value.Production) {

            // Connecting from App Engine.
            Class.forName(Messages.getString("com.mysql.jdbc.GoogleDriver")); 
            url = Messages.getString("jdbc:google:mysql://{{Instance_Connection_Name}}/{{DataBase_Name}}?user=root&password={{root_password}}");
        } else {
            // Connecting from an external network or localhost
            Class.forName(Messages.getString("com.mysql.jdbc.Driver"));
            url = Messages.getString("jdbc:mysql://{{Instance_IPv4_address}}:3306/{{DataBase_Name}}?user={{userName}}&password={{password}}");
        }

        Connection conn = DriverManager.getConnection(url);

        return conn;
    }
}
