Il est important de pouvoir utiliser des variables côté serveur dans votre site Web. Pug vous permet d’interpoler les données générées par votre serveur en code HTML, CSS et même JavaScript.

Syntaxe #

  • res.render(path, variables) // Recherche un fichier pug à rendre au chemin “path”, et lui passe des “variables”
  • #{variable} // Interpole “variable” en ligne avec le code Jade environnant, après avoir évalué “variable”
  • !{variable} // Interpole “variable” en ligne avec le code Jade environnant, sans évaluer “variable”.
  • #[element] // Interpole “element” à l’intérieur d’un élément HTML Pug existant. La syntaxe des éléments HTML interpolés est identique à celle des éléments HTML normaux.

Paramètres #

Paramètre Détails
chemin Utilisé dans res.render. C’est le chemin du fichier Pug que nous allons rendre. Le chemin est extrait de la racine du dossier défini sur votre application Express : app.set("views", "templates/views"). Par exemple, res.render("index") recherchera un fichier Pug dans templates/views/index.pug. Les sous-répertoires peuvent également être spécifiés ; res.render("admin/index") recherche un fichier Pug dans templates/views/admin/index.pug.
variables Utilisé dans res.render. Un objet JavaScript de variables à rendre accessible au fichier Pug défini par path (ci-dessus). Dans le fichier Pug, les clés de l’objet JavaScript ci-dessus deviennent disponibles en tant que variables. Si variables = {title: "Hello", color: "red"}, nous pourrions utiliser les variables title et color. Les sous-propriétés des objets imbriqués sont également disponibles.
variables Utilisé dans la syntaxe des parenthèses #{} ou !{}. La valeur de variable sera sortie dans le contexte de son code Pug environnant. Si un symbole dièse est ajouté au début de l’accolade ouvrante, “variable” sera évalué avant d’être affiché. Si un point d’exclamation est ajouté au début de l’accolade ouvrante, variable ne sera pas évalué.
élément Utilisé dans la syntaxe entre crochets #[]. L’élément HTML (dans la syntaxe Pug, pas dans la syntaxe HTML normale) sera évalué et généré en ligne avec le code Pug environnant.

Pour plus d’informations sur l’interpolation PugJS, consultez la documentation officielle sur l’interpolation PugJS.

Interpolation des variables côté serveur #

Il est possible de transmettre des variables de votre serveur à Pug pour un contenu dynamique ou une génération de script. Les modèles Pug peuvent accéder aux variables transmises à la fonction res.render dans Express (ou pug.renderFile si vous n’utilisez pas Express, les arguments sont identiques).

index.js

let colors = ["Red", "Green", "Blue"];
let langs  = ["HTML", "CSS", "JS"];
let title  = "My Cool Website";

let locals = {
    siteColors: colors,
    siteLangs:  langs,
    title:      title
};
res.render("index", locals);

Dans votre fichier index.pug, vous avez alors accès à la variable locals par le biais de ses clés. Les noms des variables dans votre fichier Pug deviennent siteColors et siteNames.

Pour définir l’intégralité d’un élément HTML égal à une variable, utilisez l’opérateur égal = pour le faire. Si votre variable doit être intégrée en ligne, utilisez la syntaxe entre parenthèses #{} pour le faire.

index.pug

doctype html
html
    head
        title= title
    body
        p My favorite color is #{siteColors[0]}.
        p Here's a list of my favorite coding languages
        ul
            each language in siteLangs
                li= language

sortie index.pug

<!DOCTYPE html>
<html>
    <head>
        <title>My Cool Website</title>
    </head>
    <body>
        <p>My favorite color is Red.</p>
        <p>Here's a list of my favorite coding languages</p>
        <ul>
            <li>HTML</li>
            <li>CSS</li>
            <li>JS</li>
        </ul>
    </body>
</html>

Interpolation de variables brutes en HTML #

Le contenu interpolé avec la syntaxe des crochets sera évalué pour le code, dont la sortie est incluse dans votre sortie HTML.

title suit le modèle de base pour évaluer un modèle local, mais le code entre #{ et } est évalué, échappé et le résultat mis en mémoire tampon dans la sortie du modèle en cours de rendu. [Source]

Si vous devez inclure une syntaxe HTML brute, utilisez un point d’exclamation au lieu d’un symbole dièse ( !{} au lieu de #{}).

index.js

let tag = "<div>You can't escape me!</div>";
res.render("index", {
    myTag: tag
});

index.pug

doctype html
html
    head
    body
        !{myTag}

sortie index.pug

<!DOCTYPE html>
<html>
    <head></head>
    <body>
        <div>You can't escape me!</div>
    </body>
</html>

Interpolation de valeur dans le code JavaScript #

L’interpolation des valeurs est utile si vous devez transmettre une variable côté serveur à JavaScript côté client (ou à d’autres langages qui en ont besoin).

Dans le cas de variables, de nombres, de chaînes, etc., vous pouvez passer ces types de variables directement dans votre JavaScript avec une syntaxe entre parenthèses plus un point d’explication (afin que le code à l’intérieur des parenthèses n’est pas évalué.) Ceci est utile pour paramétrer Code JavaScript qui nécessite quelque chose de votre serveur.

Dans l’exemple ci-dessous, nous devons envelopper username entre guillemets pour que JavaScript l’interprète comme une chaîne ; Pug affichera le contenu de la variable tel quel, nous devons donc le mettre entre guillemets pour qu’il s’agisse d’une chaîne JavaScript appropriée. Ce n’est pas nécessaire pour number, où JavaScript interprétera notre numéro comme nous le souhaitons (comme un nombre).

index.js

let number   = 24;
let username = "John";
res.render("index", {
    number:   number,
    username: username
});

index.pug

html
    head
    script.
        // Sets the username of the current user to be displayed site-wide
        function setUsername(username) {
            // ...
        }
        var number   = #{number};
        var username = "#{username}";
        setUsername(username);
        
    body
        p Welcome to my site!

sortie index.pug

<html>
    <head>
        <script>
            // Sets the username of the current user to be displayed site-wide
            function setUsername(username) {
                // ...
            }
            var number   = 24;
            var username = "John";
            setUsername(username);
        </script>
    </head>
    <body>
        <p>Welcome to my site!</p>
    </body>
</html>

Si vous avez besoin d’interpoler la valeur d’un objet JavaScript (par exemple, toutes les informations sur un utilisateur), vous devez chaîner la sortie dans Pug pour qu’elle soit traitée comme un objet JavaScript. Il est également nécessaire de sortir le contenu brut de la variable, au lieu de la forme évaluée de celle-ci. Si vous deviez sortir la variable échappée (var user = #{JSON.stringify(user)}), vous recevriez une version échappée de l’objet (où les guillemets et les apostrophes sont convertis en &quot;), ce qui n’est pas ce que nous voulons pour que JSON.stringify fonctionne dessus.

index.js

var myUser = {
    name:    "Leeroy Jenkins",
    id:      1234567890,
    address: "123 Wilson Way, New York NY, 10165"
};

res.render('index', {
    user: myUser
});

index.pug

doctype html
html
    head
        script.
            window.onload = function () {
                function setUsername(username) {
                    return username;
                }

                var user = !{JSON.stringify(user)};
                document.getElementById("welcome-user").innerHTML = setUsername(user.name);
            };

    body
        div(id="welcome-user")

sortie index.pug

<!DOCTYPE html>
<html>
    <head>
        <script>
            window.onload = function() {
                function setUsername(username) {
                    return username;
                }

                var user = {
                    "name": "Leeroy Jenkins",
                    "id": 1234567890,
                    "address": "123 Wilson Way, New York NY, 10165"
                };
                document.getElementById("welcome-user").innerHTML = setUsername(user.name);
        };
        </script>
    </head>
    <body>
        <div id="welcome-user"></div>
    </body>    
</html>

Le innerHTML de #welcome-user devient égal à Leeroy Jenkins. Le contenu de la variable user est imprimé directement dans la source HTML

Interpolation d’éléments HTML #

Il peut être nécessaire d’imbriquer des balises HTML les unes dans les autres. L’interpolation d’élément est effectuée dans une syntaxe similaire à l’interpolation de variable ; des crochets au lieu d’accolades sont utilisés ici. La syntaxe des éléments HTML interpolés est identique à l’implémentation des éléments HTML normaux.

index.pug

doctype html
html
    head
        title My Awesome Website
    body
        p The server room went #[b boom]!
        p The fire alarm, however, #[u failed to go off...]
        p Not even #[a(href="https://stackoverflow.com/") Stack Overflow] could save them now.

sortie index.pug

<!DOCTYPE html>
<html>
    <head>
        <title>My Awesome Website</title>
    </head>
    <body>
        <p>The server room went <b>boom</b>!</p>
        <p>The fire alarm, however, <u>failed to go off...</u></p>
        <p>Not even <a href="https://stackoverflow.com/">Stack Overflow</a> could save them now.</p>
    </body>
</html>
