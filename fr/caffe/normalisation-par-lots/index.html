Normalisation par lots

  • Paramètres
  • Prototxt pour la formation
  • Prototxt pour le déploiement

Sur cette page

  • Paramètres
  • Prototxt pour la formation
  • Prototxt pour le déploiement

À partir de [la documentation] (http://caffe.berkeleyvision.org/doxygen/classcaffe_1_1BatchNormLayer.html) :

“Normalise l’entrée pour avoir une moyenne de 0 et/ou une variance unitaire (1) sur l’ensemble du lot.

Cette couche calcule la normalisation par lots comme décrit dans [1].

[…]

[1] S. Ioffe et C. Szegedy, “Normalisation par lots : Accélérer la formation en profondeur du réseau en réduisant le changement de covariable interne.” arXiv preprint arXiv:1502.03167 (2015).”

Paramètres #

Paramètre Détails
use_global_stats Extrait du post de rohrbach du 2 mars 2016 - peut-être qu’il sait :
(use_global_stats) “Par défaut, pendant le temps de formation, le réseau calcule les statistiques globales de moyenne/variance via une moyenne mobile, qui est ensuite utilisée au moment du test pour permettre des sorties déterministes pour chaque entrée. Vous pouvez basculer manuellement si le réseau accumule ou en utilisant les statistiques via l’option use_global_stats. IMPORTANT : pour que cette fonctionnalité fonctionne, vous DEVEZ définir le taux d’apprentissage sur zéro pour les trois blobs de paramètres, c’est-à-dire param {lr_mult : 0} trois fois dans la définition de la couche.
(use_global_stats) Cela signifie que par défaut (comme ce qui suit est défini dans batch_norm_layer.cpp), vous n’avez pas du tout besoin de définir use_global_stats dans le prototxt. use_global_stats_ = this->phase_ == TEST ;”

Prototxt pour la formation #

Voici un exemple de définition pour la formation d’une couche BatchNorm avec une échelle et un biais par canal. Généralement, une couche BatchNorm est insérée entre les couches de convolution et de rectification. Dans cet exemple, la convolution produirait le blob “layerx” et la rectification recevrait le blob “layerx-bn”.

layer { bottom: 'layerx' top: 'layerx-bn' name: 'layerx-bn' type: 'BatchNorm'
  batch_norm_param {
    use_global_stats: false  # calculate the mean and variance for each mini-batch
    moving_average_fraction: .999  # doesn't effect training 
  }
  param { lr_mult: 0 } 
  param { lr_mult: 0 } 
  param { lr_mult: 0 }}
# channel-wise scale and bias are separate
layer { bottom: 'layerx-bn' top: 'layerx-bn' name: 'layerx-bn-scale' type: 'Scale',
  scale_param { 
    bias_term: true
    axis: 1      # scale separately for each channel
    num_axes: 1  # ... but not spatially (default)
    filler { type: 'constant' value: 1 }           # initialize scaling to 1
    bias_filler { type: 'constant' value: 0.001 }  # initialize bias
}}

Plus d’informations peuvent être trouvées dans [ce fil] [1].

[1] : http://stackoverflow.com/q/41269570/1714410

Prototxt pour le déploiement #

Le principal changement nécessaire est de passer de use_global_stats à true. Cela passe à l’utilisation de la moyenne mobile.

layer { bottom: 'layerx' top: 'layerx-bn' name: 'layerx-bn' type: 'BatchNorm'
  batch_norm_param {
    use_global_stats: true  # use pre-calculated average and variance
  }
  param { lr_mult: 0 } 
  param { lr_mult: 0 } 
  param { lr_mult: 0 }}
# channel-wise scale and bias are separate
layer { bottom: 'layerx-bn' top: 'layerx-bn' name: 'layerx-bn-scale' type: 'Scale',
  scale_param { 
    bias_term: true
    axis: 1      # scale separately for each channel
    num_axes: 1  # ... but not spatially (default)
}}
