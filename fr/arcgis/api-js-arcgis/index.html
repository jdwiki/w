Créer des applications de cartographie Web 2D convaincantes

L’API ArcGIS pour JavaScript est conçue pour optimiser votre productivité pour la création d’applications de cartographie Web attrayantes et attrayantes. L’API combine une technologie Web moderne et de puissantes capacités géospatiales vous permettant de créer des applications hautes performances et des visualisations plus intelligentes de vos données.

Chargement du CDN de l’API ArcGIS JS #

Référencez l’API ArcGIS JavaScript à partir de notre CDN et vous êtes prêt à commencer :

Chargement de la carte #

require(["esri/map", "dojo/domReady!"], function(Map) {
  var map = new Map("map", {
    center: [-118, 34.5],
    zoom: 8,
    basemap: "topo"
  });
});

Chargement du CSS ESRI #

À partir de la version 3.2 de l’API ArcGIS pour JavaScript, les développeurs doivent inclure un fichier de feuille de style en cascade (CSS) supplémentaire : esri.css.

L’URL de ce fichier est :

// versions 3.11 and forward
<link rel="stylesheet" href="https://js.arcgis.com/API-VERSION/esri/css/esri.css">

// versions prior to 3.11
<link rel="stylesheet" href="https://js.arcgis.com/API-VERSION/js/esri/css/esri.css">

Par exemple, ce fichier CSS serait inclus via une balise de lien :

<link rel="stylesheet" href="https://js.arcgis.com/3.19/esri/css/esri.css">

Le fichier esri.css contient le CSS pour divers widgets, y compris le CSS pour la carte.

Étant donné que tous les CSS se trouvent dans un seul fichier, la récupération des CSS pour l’API se fait en une seule requête. La réduction du nombre de requêtes http est un moyen pour les applications d’améliorer les performances.

Chargement du CSS spécifique à Dojo #

Le fichier esri.css n’inclut pas le CSS pour divers widgets Dojo ou thèmes comme tundra ou claro ; ces fichiers doivent être inclus séparément. Les exceptions sont Grid et RangeSlider, qui sont utilisées par les widgets dans l’API. Les styles de grille doivent être explicitement inclus.

Par exemple, ce fichier CSS serait inclus via une balise de lien supplémentaire :

<link rel="stylesheet" href="https://js.arcgis.com/3.19/dijit/themes/claro/claro.css">

CSS hérité

Avant la version 3.2, les fichiers CSS étaient ajoutés dynamiquement au DOM d’une page via JavaScript.

L’ajout dynamique de fichiers CSS rend fastidieux le remplacement des styles par défaut, car le CSS inclus via un lien ou une balise de style est analysé avant que JavaScript n’ajoute le CSS à une page. Il en résulte que vous devez utiliser !important pour remplacer les styles de widget par défaut. Étant donné que les widgets ont été refactorisés pour ne pas ajouter dynamiquement leur CSS, !important n’est plus nécessaire. Les versions d’API antérieures à 3.2 n’ont pas été modifiées, il est donc toujours nécessaire d’utiliser !important avec les anciennes versions. Inclure explicitement CSS via une balise de lien au lieu d’ajouter dynamiquement CSS suit également la convention utilisée par les composants livrés avec Dojo.
