Différences entre les distributions Java SE JRE ou Java SE JDK #

Les versions Sun/Oracle de Java SE se présentent sous deux formes : JRE et JDK. En termes simples, les JRE prennent en charge l’exécution d’applications Java et les JDK prennent également en charge le développement Java.

Environnement d’exécution Java #

Les distributions Java Runtime Environment ou JRE consistent en un ensemble de bibliothèques et d’outils nécessaires pour exécuter et gérer les applications Java. Les outils d’un JRE moderne typique incluent :

  • La commande java pour exécuter un programme Java dans une JVM (Java Virtual Machine)
  • La commande jjs pour exécuter le moteur Javascript Nashorn.
  • La commande keytool pour manipuler les keystores Java.
  • La commande policytool pour modifier les politiques de sécurité du bac à sable de sécurité.
  • Les outils pack200 et unpack200 pour compresser et décompresser le fichier “pack200” pour le déploiement Web.
  • Les commandes orbd, rmid, rmiregistry et tnameserv qui prennent en charge les applications Java CORBA et RMI.

Les programmes d’installation “Desktop JRE” incluent un plug-in Java adapté à certains navigateurs Web. Ceci est délibérément omis des installateurs “Server JRE”.linux syscall read benchmarku

À partir de la mise à jour 6 de Java 7, les programmes d’installation JRE ont inclus JavaFX (version 2.2 ou ultérieure).

Kit de développement Java #

Un kit de développement Java ou une distribution JDK inclut les outils JRE et des outils supplémentaires pour le développement de logiciels Java. Les outils supplémentaires incluent généralement :

  • La commande javac, qui compile le code source Java ("".java") en fichiers de bytecode ("".class").
  • Les outils de création de fichiers JAR tels que jar et jarsigner
  • Outils de développement tels que :
    • appletviewer for running applets
    • idlj the CORBA IDL to Java compiler
    • javah the JNI stub generator
    • native2ascii for character set conversion of Java source code
    • schemagen the Java to XML schema generator (part of JAXB)
    • serialver generate Java Object Serialization version string.
    • the wsgen and wsimport support tools for JAX-WS
  • Outils de diagnostic tels que :
    • jdb the basic Java debugger
    • jmap and jhat for dumping and analysing a Java heap.
    • jstack for getting a thread stack dump.
    • javap for examining “.class” files.
  • Outils de gestion et de surveillance des applications tels que :
    • jconsole a management console,
    • jstat, jstatd, jinfo and jps for application monitoring

Une installation typique de Sun / Oracle JDK comprend également un fichier ZIP avec le code source des bibliothèques Java. Avant Java 6, c’était le seul code source Java accessible au public.

A partir de Java 6, le code source complet d’OpenJDK est disponible en téléchargement sur le site OpenJDK. Il n’est généralement pas inclus dans les packages JDK (Linux), mais est disponible en tant que package séparé.

Points forts de la version Java SE #

Version Java SE Faits saillants
Java SE 8 Expressions lambda et flux inspirés de MapReduce. Le moteur Javascript Nashorn. Annotations sur les types et annotations répétitives. Extensions arithmétiques non signées. Nouvelles API de date et d’heure. Bibliothèques JNI liées statiquement. Lanceur JavaFX. Suppression de PermGen.
Java SE 7 Commutateurs de chaîne, * try-with-resource *, le losange (<>), améliorations des littéraux numériques et améliorations de la gestion des exceptions / relance. Améliorations de la bibliothèque de concurrence. Prise en charge améliorée des systèmes de fichiers natifs. Timsort. Algorithmes cryptographiques ECC. Prise en charge améliorée des graphiques 2D (GPU). Annotations enfichables.
Java SE 6 Améliorations significatives des performances de la plate-forme JVM et de Swing. API de langage de script et moteur Mozilla Rhino Javascript. JDBC 4.0. API du compilateur. JAXB 2.0. Prise en charge des services Web (JAX-WS)
Java SE 5 Génériques, annotations, auto-boxing, classes “enum”, varargs, boucles “for” améliorées et importations statiques. Spécification du modèle de mémoire Java. Améliorations de Swing et RMI. Ajout du package java.util.concurrent.* et Scanner.

Différences entre Java EE, Java SE, Java ME et JavaFX #

La technologie Java est à la fois un langage de programmation et une plate-forme. Le langage de programmation Java est un langage orienté objet de haut niveau qui a une syntaxe et un style particuliers. Une plate-forme Java est un environnement particulier dans lequel s’exécutent les applications du langage de programmation Java.

Il existe plusieurs plateformes Java. De nombreux développeurs, même des développeurs de langage de programmation Java de longue date, ne comprennent pas comment les différentes plates-formes sont liées les unes aux autres.

Les plates-formes de langage de programmation Java #

Il existe quatre plates-formes du langage de programmation Java :

  • Plate-forme Java, édition standard (Java SE)

  • Plate-forme Java, édition entreprise (Java EE)

  • Plate-forme Java, Micro Edition (Java ME)

-Java FX

Toutes les plates-formes Java se composent d’une machine virtuelle Java (VM) et d’une interface de programmation d’application (API). La machine virtuelle Java est un programme, pour une plate-forme matérielle et logicielle particulière, qui exécute des applications de technologie Java. Une API est un ensemble de composants logiciels que vous pouvez utiliser pour créer d’autres composants logiciels ou applications. Chaque plate-forme Java fournit une machine virtuelle et une API, ce qui permet aux applications écrites pour cette plate-forme de s’exécuter sur n’importe quel système compatible avec tous les avantages du langage de programmation Java : indépendance de la plate-forme, puissance, stabilité, facilité de développement et Sécurité.

Java SE #

Lorsque la plupart des gens pensent au langage de programmation Java, ils pensent à l’API Java SE. L’API de Java SE fournit les fonctionnalités de base du langage de programmation Java. Il définit tout, des types et objets de base du langage de programmation Java aux classes de haut niveau utilisées pour la mise en réseau, la sécurité, l’accès à la base de données, le développement d’interface utilisateur graphique (GUI) et l’analyse XML.

En plus de l’API principale, la plate-forme Java SE se compose d’une machine virtuelle, d’outils de développement, de technologies de déploiement et d’autres bibliothèques de classes et kits d’outils couramment utilisés dans les applications de technologie Java.

JavaEE #

La plate-forme Java EE est construite au-dessus de la plate-forme Java SE. La plate-forme Java EE fournit une API et un environnement d’exécution pour le développement et l’exécution d’applications réseau à grande échelle, multiniveaux, évolutives, fiables et sécurisées.

Java ME #

La plate-forme Java ME fournit une API et une machine virtuelle à faible encombrement pour exécuter des applications de langage de programmation Java sur de petits appareils, comme les téléphones mobiles. L’API est un sous-ensemble de l’API Java SE, ainsi que des bibliothèques de classes spéciales utiles pour le développement d’applications pour petits appareils. Les applications Java ME sont souvent clientes des services de la plateforme Java EE.

Effets Java #

La technologie Java FX est une plate-forme de création d’applications Internet riches écrites en Java FX ScriptTM. Java FX Script est un langage déclaratif de type statique qui est compilé en bytecode de la technologie Java, qui peut ensuite être exécuté sur une machine virtuelle Java. Les applications écrites pour la plate-forme Java FX peuvent inclure et être liées à des classes de langage de programmation Java, et peuvent être des clients des services de la plate-forme Java EE.

Quelle est la différence entre Oracle Hotspot et OpenJDK #

Orthogonalement à la dichotomie JRE versus JDK, il existe deux types de versions Java largement disponibles :

  • Les versions Oracle Hotspot sont celles que vous téléchargez à partir des sites de téléchargement Oracle.
  • Les versions d’OpenJDK sont celles qui sont construites (généralement par des fournisseurs tiers) à partir des référentiels source d’OpenJDK.

En termes fonctionnels, il y a peu de différence entre une version Hotspot et une version OpenJDK. Il existe des fonctionnalités “d’entreprise” supplémentaires dans Hotspot que les clients Java Oracle (payants) peuvent activer, mais à part cela, la même technologie est présente à la fois dans Hotspot et OpenJDK.

Un autre avantage de Hotspot par rapport à OpenJDK est que les versions de correctifs pour Hotspot ont tendance à être disponibles un peu plus tôt. Cela dépend également de l’agilité de votre fournisseur OpenJDK ; par exemple. combien de temps il faut à l’équipe de construction d’une distribution Linux pour préparer et contrôler une nouvelle version d’OpenJDK, et la mettre dans leurs référentiels publics.

Le revers de la médaille est que les versions Hotspot ne sont pas disponibles à partir des référentiels de packages pour la plupart des distributions Linux. Cela signifie que la mise à jour de votre logiciel Java sur une machine Linux demande généralement plus de travail si vous utilisez Hotspot.
