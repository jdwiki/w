Machine virtuelle Java (JVM)

  • Ce sont les bases.

Ce sont les bases. #

JVM est une machine informatique abstraite ou machine virtuelle qui réside dans votre RAM. Il dispose d’un environnement d’exécution indépendant de la plate-forme qui interprète le bytecode Java en code machine natif. (Javac est un compilateur Java qui compile votre code Java en Bytecode)

Le programme Java s’exécutera à l’intérieur de la JVM qui est ensuite mappée sur la machine physique sous-jacente. C’est l’un des outils de programmation du JDK.

(Byte code est un code indépendant de la plate-forme qui est exécuté sur chaque plate-forme et Code machine est un code spécifique à la plate-forme qui est exécuté uniquement sur une plate-forme spécifique telle que Windows ou Linux ; cela dépend de l’exécution.)

Certains des composants : -

  • Class Loder - chargez le fichier .class dans la RAM.
  • Vérificateur de bytecode - vérifiez s’il y a des violations de restriction d’accès dans votre code.
  • Moteur d’exécution - convertit le byte code en code machine exécutable.
  • JIT (juste à temps) - JIT fait partie de JVM qui permet d’améliorer les performances de JVM. Il compilera ou traduira dynamiquement le bytecode Java en code machine natif pendant le temps d’exécution.
