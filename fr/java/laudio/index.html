Au lieu d’utiliser le Clip javax.sound.sampled, vous pouvez également utiliser le AudioClip qui provient de l’API de l’applet. Il est cependant recommandé d’utiliser Clip car AudioClip est juste plus ancien et présente des fonctionnalités limitées.

Lire un fichier MIDI #

Les fichiers MIDI peuvent être lus en utilisant plusieurs classes du package javax.sound.midi. Un « séquenceur » effectue la lecture du fichier MIDI, et bon nombre de ses méthodes peuvent être utilisées pour définir des commandes de lecture telles que le nombre de boucles, le tempo, la mise en sourdine des pistes, etc.

La lecture générale des données MIDI peut être effectuée de cette manière :

import java.io.File;
import java.io.IOException;
import javax.sound.midi.InvalidMidiDataException;
import javax.sound.midi.MidiSystem;
import javax.sound.midi.MidiUnavailableException;
import javax.sound.midi.Sequence;
import javax.sound.midi.Sequencer;

public class MidiPlayback {
    public static void main(String[] args) {
        try {
            Sequencer sequencer = MidiSystem.getSequencer(); // Get the default Sequencer
            if (sequencer==null) {
                System.err.println("Sequencer device not supported");
                return;
            } 
            sequencer.open(); // Open device
            // Create sequence, the File must contain MIDI file data.
            Sequence sequence = MidiSystem.getSequence(new File(args[0]));
            sequencer.setSequence(sequence); // load it into sequencer
            sequencer.start();  // start the playback
        } catch (MidiUnavailableException | InvalidMidiDataException | IOException ex) {
            ex.printStackTrace();
        }
    }
}

Pour arrêter la lecture, utilisez :

sequencer.stop(); // Stop the playback

Un séquenceur peut être réglé pour couper une ou plusieurs des pistes de la séquence pendant la lecture afin qu’aucun des instruments de ceux spécifiés ne joue. L’exemple suivant définit le son de la première piste de la séquence :

import javax.sound.midi.Track;
// ...

Track[] track = sequence.getTracks();
sequencer.setTrackMute(track[0]);

Un séquenceur peut jouer une séquence de manière répétée si le nombre de boucles est donné. Ce qui suit règle le séquenceur pour jouer une séquence quatre fois et indéfiniment :

sequencer.setLoopCount(3);
sequencer.setLoopCount(Sequencer.LOOP_CONTINUOUSLY);

Le séquenceur n’a pas toujours à jouer la séquence depuis le début, ni à jouer la séquence jusqu’à la fin. Il peut commencer et se terminer à n’importe quel moment en spécifiant le tick dans la séquence pour commencer et se terminer. Il est également possible de spécifier manuellement à partir de quel tick dans la séquence le séquenceur doit jouer :

sequencer.setLoopStartPoint(512);
sequencer.setLoopEndPoint(32768);
sequencer.setTickPosition(8192);

Les séquenceurs peuvent également lire un fichier MIDI à un certain tempo, qui peut être contrôlé en spécifiant le tempo en battements par minute (BPM) ou en microsecondes par noire (MPQ). Le facteur auquel la séquence est jouée peut également être ajusté.

sequencer.setTempoInBPM(1250f);
sequencer.setTempoInMPQ(4750f);
sequencer.setTempoFactor(1.5f);

Lorsque vous avez fini d’utiliser le Séquenceur, n’oubliez pas de le fermer

sequencer.close();

Lire un fichier audio en boucle #

Importations nécessaires :

import javax.sound.sampled.AudioSystem;
import javax.sound.sampled.Clip;

Ce code créera un clip et le jouera en continu une fois lancé :

Clip clip = AudioSystem.getClip();
clip.open(AudioSystem.getAudioInputStream(new URL(filename)));
clip.start();
clip.loop(Clip.LOOP_CONTINUOUSLY);

Obtenez un tableau avec tous les types de fichiers compatibles :

AudioFileFormat.Type [] audioFileTypes = AudioSystem.getAudioFileTypes();

Sortie audio de base #

## Son métal nu Vous pouvez également passer presque au nu lorsque vous produisez du son avec Java. Ce code écrira des données binaires brutes dans la mémoire tampon audio du système d’exploitation pour générer du son. Il est extrêmement important de comprendre les limites et les calculs nécessaires pour générer un son comme celui-ci. Étant donné que la lecture est fondamentalement instantanée, les calculs doivent être effectués presque en temps réel.

En tant que telle, cette méthode est inutilisable pour un échantillonnage sonore plus compliqué. À ces fins, l’utilisation d’outils spécialisés est la meilleure approche.

La méthode suivante génère et produit directement une onde rectangulaire d’une fréquence donnée dans un volume donné pendant une durée donnée.

public void rectangleWave(byte volume, int hertz, int msecs) {
    final SourceDataLine dataLine;
    // 24 kHz x 8bit, single-channel, signed little endian AudioFormat
    AudioFormat af = new AudioFormat(24_000, 8, 1, true, false);
    try {
        dataLine = AudioSystem.getSourceDataLine(af);
        dataLine.open(af, 10_000); // audio buffer size: 10k samples
    } catch (LineUnavailableException e) {
        throw new RuntimeException(e);
    }

    int waveHalf = 24_000 / hertz; // samples for half a period
    byte[] buffer = new byte[waveHalf * 20];
    int samples = msecs * (24_000 / 1000); // 24k (samples / sec) / 1000 (ms/sec) * time(ms)

    dataLine.start(); // starts playback
    int sign = 1;

    for (int i = 0; i < samples; i += buffer.length) {
        for (int j = 0; j < 20; j++) { // generate 10 waves into buffer
            sign *= -1; 
            // fill from the jth wave-half to the j+1th wave-half with volume
            Arrays.fill(buffer, waveHalf * j, waveHalf * (j+1), (byte) (volume * sign));
        }
        dataLine.write(buffer, 0, buffer.length); // 
    }
    dataLine.drain(); // forces buffer drain to hardware
    dataLine.stop();  // ends playback
}

Pour une manière plus différenciée de générer différentes ondes sonores, des calculs de sinus et éventuellement des tailles d’échantillons plus grandes sont nécessaires. Cela se traduit par un code beaucoup plus complexe et est donc omis ici.
