  • Références d’objets en tant que paramètres de méthode

Sur cette page

  • Références d’objets en tant que paramètres de méthode

Cela devrait vous aider à comprendre une “exception de pointeur nul” - on en obtient une car une référence d’objet est nulle, mais le code du programme s’attend à ce que le programme utilise quelque chose dans cette référence d’objet. Cependant, cela mérite son propre sujet…

Références d’objets en tant que paramètres de méthode #

Cette rubrique explique le concept d’une référence d’objet ; il est destiné aux personnes qui débutent dans la programmation en Java. Vous devriez déjà être familiarisé avec certains termes et leurs significations : définition de classe, méthode principale, instance d’objet et appel de méthodes “sur” un objet, et transmission de paramètres aux méthodes.

public class Person {

  private String name;

  public void setName(String name) { this.name = name; }

  public String getName() { return name; }

  public static void main(String [] arguments) {
    Person person = new Person();
    person.setName("Bob");

    int i = 5;
    setPersonName(person, i);

    System.out.println(person.getName() + " " + i);
  }

  private static void setPersonName(Person person, int num) {
    person.setName("Linda");
    num = 99;
  }
}

Pour être parfaitement compétent en programmation Java, vous devriez être capable d’expliquer cet exemple à quelqu’un d’autre du haut de votre tête. Ses concepts sont fondamentaux pour comprendre le fonctionnement de Java.

Comme vous pouvez le voir, nous avons un main qui instancie un objet à la variable person et appelle une méthode pour définir le champ name dans cet objet à "Bob". Ensuite, il appelle une autre méthode et passe person comme l’un des deux paramètres ; l’autre paramètre est une variable entière, définie sur 5.

La méthode appelée définit la valeur name sur l’objet transmis à “Linda”, et définit la variable entière transmise à 99, puis revient.

Alors qu’est-ce qui serait imprimé?

Linda 5

Alors pourquoi la modification apportée à person prend-elle effet dans main, mais pas la modification apportée à l’entier ?

Lorsque l’appel est effectué, la méthode main passe une référence d’objet pour person à la méthode setPersonName ; toute modification que setAnotherName apporte à cet objet fait partie de cet objet, et donc ces modifications font toujours partie de cet objet lorsque la méthode revient.

Une autre façon de dire la même chose : person pointe vers un objet (stocké sur le tas, si cela vous intéresse). Toute modification apportée par la méthode à cet objet est effectuée “sur cet objet” et n’est pas affectée par le fait que la méthode effectuant la modification est toujours active ou est retournée. Lorsque la méthode revient, toutes les modifications apportées à l’objet sont toujours stockées sur cet objet.

Comparez cela avec l’entier qui est passé. Puisqu’il s’agit d’un int [primitif][1] (et non d’une instance d’objet Integer), il est passé “par valeur”, ce qui signifie que sa valeur est fournie à la méthode, et non un pointeur vers l’entier d’origine passé. method peut le modifier pour ses propres besoins, mais cela n’affecte pas la variable utilisée lors de l’appel de la méthode.

En Java, toutes les primitives sont passées par valeur. Les objets sont passés par référence, ce qui signifie qu’un pointeur vers l’objet est passé comme paramètre à toutes les méthodes qui les prennent.

Une chose moins évidente que cela signifie : il n’est pas possible pour une méthode appelée de créer un nouvel objet et de le renvoyer comme l’un des paramètres. Le seul moyen pour une méthode de renvoyer un objet créé, directement ou indirectement, par l’appel de méthode, est une valeur de retour de la méthode. Voyons d’abord comment cela ne fonctionnerait pas, puis comment cela fonctionnerait.

Ajoutons une autre méthode à notre petit exemple ici :

private static void getAnotherObjectNot(Person person) {
  person = new Person();
  person.setName("George");
}

Et, de retour dans le main, sous l’appel à setAnotherName, plaçons un appel à cette méthode et un autre appel println :

getAnotherObjectNot(person);
System.out.println(person.getName());

Maintenant, le programme imprimerait :

Linda 5
Linda

Qu’est-il arrivé à l’objet qui avait George ? Eh bien, le paramètre qui a été transmis était un pointeur vers Linda ; lorsque la méthode getAnotherObjectNot créait un nouvel objet, elle remplaçait la référence à l’objet Linda par une référence à l’objet George. L’objet Linda existe toujours (sur le tas), la méthode main peut toujours y accéder, mais la méthode getAnotherObjectNot ne pourra plus rien faire avec lui après cela, car il n’y a aucune référence. Il semblerait que l’auteur du code ait voulu que la méthode crée un nouvel objet et le renvoie, mais si c’est le cas, cela n’a pas fonctionné.

Si c’est ce que l’auteur voulait faire, il devrait renvoyer l’objet nouvellement créé à partir de la méthode, quelque chose comme ceci :

private static Person getAnotherObject() {
  Person person = new Person();
  person.setName("Mary");
  return person;
}

Alors appelez-le comme ceci :

Person mary;
mary = getAnotherObject();
System.out.println(mary.getName());

Et la sortie entière du programme serait maintenant :

Linda 5
Linda
Mary

Voici le programme complet, avec les deux ajouts :

public class Person {
  private String name;

  public void setName(String name) { this.name = name; }
  public String getName() { return name; }

  public static void main(String [] arguments) {
    Person person = new Person();
    person.setName("Bob");

    int i = 5;
    setPersonName(person, i);
    System.out.println(person.getName() + " " + i);
    
    getAnotherObjectNot(person);
    System.out.println(person.getName());
    
    Person person;
    person = getAnotherObject();
    System.out.println(person.getName());
  }
  
  private static void setPersonName(Person person, int num) {
    person.setName("Linda");
    num = 99;
  }
  
  private static void getAnotherObjectNot(Person person) {
    person = new Person();
    person.setMyName("George");
  }
  
  private static person getAnotherObject() {
    Person person = new Person();
    person.setMyName("Mary");
    return person;
  }
}
