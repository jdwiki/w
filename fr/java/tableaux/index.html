Les tableaux permettent le stockage et la récupération d’une quantité arbitraire de valeurs. Ils sont analogues aux vecteurs en mathématiques. Les tableaux de tableaux sont analogues aux matrices et agissent comme des tableaux multidimensionnels. Les tableaux peuvent stocker n’importe quelle donnée de n’importe quel type : des primitives telles que “int” ou des types de référence tels que “Object”.

Syntaxe #

  • ArrayType[] myArray; // Déclaration des tableaux
  • ArrayType myArray[]; // Une autre syntaxe valide (moins couramment utilisée et déconseillée)
  • ArrayType[][][] myArray; // Déclaration de tableaux irréguliers multidimensionnels (répéter []s)
  • ArrayType myVar = myArray[index]; // Accès (lecture) à l’élément index
  • myArray[index] = value; // Affecte une valeur à la position index du tableau
  • ArrayType[] myArray = new ArrayType[arrayLength]; // Syntaxe d’initialisation du tableau
  • int[] ints = {1, 2, 3}; // Syntaxe d’initialisation du tableau avec les valeurs fournies, la longueur est déduite du nombre de valeurs fournies : {[value1[, value2]*]}
  • new int[]{4, -5, 6} // Peut être utilisé comme argument, sans variable locale
  • int[] ints = nouveau int[3] ; // identique à {0, 0, 0}
  • int[][] ints = {{1, 2}, {3}, null}; // Initialisation d’un tableau multidimensionnel. int[] étend Object (ainsi que anyType[]) donc null est une valeur valide.

Paramètres #

Paramètre Détails
TypeTableau Type du tableau. Il peut s’agir de primitives (int, long, byte) ou d’objets (String, MyObject, etc.).
indice Index fait référence à la position d’un certain objet dans un tableau.

# Introduction

Un tableau est une structure de données qui contient un nombre fixe de valeurs primitives ou de références à des instances d’objet.

Chaque élément d’un tableau est appelé un élément, et chaque élément est accessible par son index numérique. La longueur d’un tableau est établie lors de la création du tableau :

int size = 42;
int[] array = new int[size];

La taille d’un tableau est fixée lors de l’exécution lors de l’initialisation. Elle ne peut pas être modifiée après l’initialisation. Si la taille doit être modifiable au moment de l’exécution, une classe [Collection][so-docs-collections] telle que [ArrayList][docs-arraylist] doit être utilisée à la place. ArrayList stocke les éléments dans un tableau et prend en charge [le redimensionnement en allouant un nouveau tableau] (https://www.wikiod.com/fr/java/tableaux#Comment changer la taille d’un tableau ? ) et en copiant les éléments de l’ancien tableau.

Si le tableau est de type primitif, c’est-à-dire

int[] array1 = { 1,2,3 };
int[] array2 = new int[10];

les valeurs sont stockées dans le tableau lui-même. En l’absence d’un initialiseur (comme dans array2 ci-dessus), la valeur par défaut attribuée à chaque élément est 0 (zéro).

Si le type de tableau est une référence d’objet, comme dans

SomeClassOrInterface[] array = new SomeClassOrInterface[10];

alors le tableau contient des references aux objets de type SomeClassOrInterface. Ces références peuvent faire référence à une instance de SomeClassOrInterface ou à toute sous-classe (pour les classes) ou à une classe d’implémentation (pour les interfaces) de SomeClassOrInterface. Si la déclaration de tableau n’a pas d’initialiseur, la valeur par défaut de null est affectée à chaque élément.

Comme tous les tableaux sont indexés en int, la taille d’un tableau doit être spécifiée par un int. La taille du tableau ne peut pas être spécifiée en tant que “long” :

long size = 23L;
int[] array = new int[size]; // Compile-time error:
                             // incompatible types: possible lossy conversion from
                             // long to int

Les tableaux utilisent un système d’index basé sur zéro, ce qui signifie que l’indexation commence à « 0 » et se termine à « longueur - 1 ».

Par exemple, l’image suivante représente un tableau de taille “10”. Ici, le premier élément est à l’indice ‘0’ et le dernier élément à l’indice ‘9’, au lieu que le premier élément soit à l’indice ‘1’ et le dernier élément à l’indice ‘10’ (voir figure ci-dessous).

![Un tableau de 10 éléments][1]

Les accès aux éléments des tableaux se font en temps constant. Cela signifie que l’accès au premier élément du tableau a le même coût (en temps) que l’accès au deuxième élément, au troisième élément, etc.

Java offre plusieurs façons de définir et d’initialiser des tableaux, y compris les notations littérale et constructeur. Lors de la déclaration de tableaux à l’aide du constructeur new Type[length], chaque élément sera initialisé avec les valeurs par défaut suivantes :

  • 0 pour les [types numériques primitifs][docs-primitive-types] : byte, short, int, long, float et double.
  • '\u0000' (caractère nul) pour le type char.
  • false pour le type booléen.
  • null pour [types de référence][docs-ref-types].

Création et initialisation de tableaux de types primitifs #

int[] array1 = new int[] { 1, 2, 3 }; // Create an array with new operator and 
                                      // array initializer.
int[] array2 = { 1, 2, 3 };           // Shortcut syntax with array initializer.
int[] array3 = new int[3];            // Equivalent to { 0, 0, 0 }
int[] array4 = null;                  // The array itself is an object, so it
                                      // can be set as null.

Lors de la déclaration d’un tableau, [] apparaîtra dans le cadre du type au début de la déclaration (après le nom du type), ou dans le cadre du déclarateur d’une variable particulière (après le nom de la variable), ou les deux :

int array5[];       /* equivalent to */  int[] array5;
int a, b[], c[][];  /* equivalent to */  int a; int[] b; int[][] c;
int[] a, b[];       /* equivalent to */  int[] a; int[][] b;
int a, []b, c[][];  /* Compilation Error, because [] is not part of the type at beginning
                       of the declaration, rather it is before 'b'. */    
// The same rules apply when declaring a method that returns an array:
int foo()[] { ... } /* equivalent to */  int[] foo() { ... }

Dans l’exemple suivant, les deux déclarations sont correctes et peuvent être compilées et exécutées sans aucun problème. Cependant, la [Java Coding Convention][docs-codeconv] et le [Google Java Style Guide][google-docs-styleguide] déconseillent le formulaire avec des crochets après le nom de la variable—[les crochets identifient le type de tableau et doivent apparaître avec la désignation de type][docs-arraytutorial]. La même chose doit être utilisée pour les signatures de retour de méthode.

float array[]; /* and */ int foo()[] { ... } /* are discouraged */
float[] array; /* and */ int[] foo() { ... } /* are encouraged */

Le type déconseillé est [destiné à accueillir les utilisateurs C en transition] [so-qa-arraysyntax], qui sont familiers avec la syntaxe de C qui a les crochets après le nom de la variable.

En Java, il est possible d’avoir des tableaux de taille ‘0’ :

int[] array = new int[0]; // Compiles and runs fine.
int[] array2 = {};        // Equivalent syntax.

Cependant, comme il s’agit d’un tableau vide, aucun élément ne peut y être lu ou affecté :

array[0] = 1;     // Throws java.lang.ArrayIndexOutOfBoundsException.
int i = array2[0]; // Also throws ArrayIndexOutOfBoundsException.

De tels tableaux vides sont généralement utiles comme valeurs de retour, de sorte que le code appelant n’a qu’à se soucier de traiter un tableau, plutôt qu’une valeur null potentielle qui peut conduire à une [NullPointerException][so-docs-npe] .

La longueur d’un tableau doit être un entier non négatif :

int[] array = new int[-1]; // Throws java.lang.NegativeArraySizeException

La taille du tableau peut être déterminée à l’aide d’un champ final public appelé length :

System.out.println(array.length); // Prints 0 in this case.

Remarque : array.length renvoie la taille réelle du tableau et non le nombre d’éléments du tableau auxquels une valeur a été attribuée, contrairement à [ArrayList.size()][arraylist-size-javadocs] qui renvoie le nombre d’éléments du tableau auxquels une valeur a été attribuée.

Créer et initialiser des tableaux multidimensionnels #

La manière la plus simple de créer un tableau multidimensionnel est la suivante :

int[][] a = new int[2][3];

Il créera deux tableaux int de trois longueurs : a[0] et a[1]. Ceci est très similaire à l’initialisation classique de style C des tableaux multidimensionnels rectangulaires.

Vous pouvez créer et initialiser en même temps :

int[][] a = { {1, 2}, {3, 4}, {5, 6} };

[Contrairement à C][2], où seuls les tableaux multidimensionnels rectangulaires sont pris en charge, les tableaux internes n’ont pas besoin d’être de la même longueur, ni même définis :

int[][] a = { {1}, {2, 3}, null };

Ici, a[0] est un tableau int d’une longueur, alors que a[1] est un tableau int de deux longueurs et a[2] est null. Des tableaux comme celui-ci sont appelés [jagged arrays ou ragged arrays][so-docs-jaggedarrays], c’est-à-dire qu’ils sont des tableaux de tableaux. Les tableaux multidimensionnels en Java sont implémentés comme des tableaux de tableaux, c’est-à-dire que array[i][j][k] est équivalent à ((array[i])[j])[k]. [Contrairement à C#][so-qa-jaggeddifference], la syntaxe array[i,j] n’est pas prise en charge en Java.

En plus des littéraux et primitives String présentés ci-dessus, la syntaxe de raccourci pour l’initialisation du tableau fonctionne également avec les types Object canoniques :

Object[] array10 = { new Object(), new Object() };

Étant donné que les tableaux sont covariants, un tableau de type référence peut être initialisé en tant que tableau d’une sous-classe, bien qu’une ArrayStoreException soit levée si vous essayez de définir un élément sur autre chose qu’une String :

Dans les types primitifs et de référence, une initialisation de tableau vide (par exemple String[] array8 = new String[3]) initialisera le tableau avec la [valeur par défaut pour chaque type de données][docs-primitive-types].

Création et initialisation des tableaux [type générique][4] #

Dans les classes génériques, les tableaux de types génériques ne peuvent pas être initialisés comme ceci en raison de [type erase][docs-type-erasure] :

public class MyGenericClass<T> {
    private T[] a;

    public MyGenericClass() {
        a = new T[5]; // Compile time error: generic array creation
    }
}

Au lieu de cela, ils peuvent être créés en utilisant l’une des méthodes suivantes : (notez que celles-ci généreront des avertissements non contrôlés)

  1. En créant un tableau Object et en le convertissant en type générique :

    a = (T[]) new Object[5];

    This is the simplest method, but since the underlying array is still of type Object[], this method does not provide type safety. Therefore, this method of creating an array is best used only within the generic class - not exposed publicly.

  2. En utilisant [Array.newInstance][5] avec un paramètre de classe :

    public MyGenericClass(Class<T> clazz) {
    a = (T[]) Array.newInstance(clazz, 5);
}

    Here the class of T has to be explicitly passed to the constructor. The return type of Array.newInstance is always Object. However, this method is safer because the newly created array is always of type T[], and therefore can be safely externalized.

Déclaration et initialisation séparées des tableaux #

La valeur d’un index pour un élément de tableau doit être un nombre entier (0, 1, 2, 3, 4, …) et inférieur à la longueur du tableau (les index sont basés sur zéro). Sinon, une [ArrayIndexOutOfBoundsException][docs-aioobe] sera lancée :

int[] array9;             // Array declaration - uninitialized
array9 = new int[3];      // Initialize array  - { 0, 0, 0 }
array9[0] = 10;           // Set index 0 value - { 10, 0, 0 }
array9[1] = 20;           // Set index 1 value - { 10, 20, 0 }
array9[2] = 30;           // Set index 2 value - { 10, 20, 30 }

Les tableaux ne peuvent pas être réinitialisés avec la syntaxe de raccourci de l’initialiseur de tableau #

Il n’est [pas possible de réinitialiser un tableau][docs-array-init] via une syntaxe de raccourci avec un initialiseur de tableau car un initialiseur de tableau ne peut être spécifié que dans une déclaration de champ ou une déclaration de variable locale, ou dans le cadre d’un expression de création de tableau.

Cependant, il est possible de créer un nouveau tableau et de l’affecter à la variable utilisée pour référencer l’ancien tableau. Bien que cela entraîne la réinitialisation du tableau référencé par cette variable, le contenu de la variable est un tout nouveau tableau. Pour ce faire, l’opérateur new peut être utilisé avec un initialiseur de tableau et assigné à la variable tableau :

// First initialization of array
int[] array = new int[] { 1, 2, 3 };

// Prints "1 2 3 ".
for (int i : array) {
    System.out.print(i + " ");
}

// Re-initializes array to a new int[] array.
array = new int[] { 4, 5, 6 };

// Prints "4 5 6 ".
for (int i : array) {
    System.out.print(i + " ");
}

array = { 1, 2, 3, 4 }; // Compile-time error! Can't re-initialize an array via shortcut 
                        // syntax with array initializer.

Création d’une liste à partir d’un tableau #

La méthode [Arrays.asList()][2] peut être utilisée pour retourner une List de taille fixe contenant les éléments du tableau donné. La “Liste” résultante sera du même type de paramètre que le type de base du tableau.

String[] stringArray = {"foo", "bar", "baz"};
List<String> stringList = Arrays.asList(stringArray);

Remarque : Cette liste s’appuie sur (une ​​vue) du tableau d’origine, ce qui signifie que toute modification apportée à la liste modifiera le tableau et vice versa. Cependant, les modifications apportées à la liste qui modifieraient sa taille (et donc la longueur du tableau) lèveront une exception.

Pour créer une copie de la liste, utilisez le constructeur de [java.util.ArrayList](https://docs.oracle.com/javase/8/docs/api/java/util/ArrayList.html#ArrayList- java.util.Collection-) prenant une [Collection][7] comme argument :

String[] stringArray = {"foo", "bar", "baz"};
List<String> stringList = new ArrayList<String>(Arrays.asList(stringArray));

Dans Java SE 7 et versions ultérieures, une paire de crochets angulaires <> (ensemble vide d’arguments de type) peut être utilisée, appelée [Diamond][3]. Le compilateur peut déterminer les arguments de type à partir du contexte. Cela signifie que les informations de type peuvent être omises lors de l’appel du constructeur de ArrayList et qu’elles seront déduites automatiquement lors de la compilation. Cela s’appelle [Type Inference][4] qui fait partie de Java [Generics][5].

// Using Arrays.asList()

String[] stringArray = {"foo", "bar", "baz"};
List<String> stringList = new ArrayList<>(Arrays.asList(stringArray));

// Using ArrayList.addAll()

String[] stringArray = {"foo", "bar", "baz"};
ArrayList<String> list = new ArrayList<>();
list.addAll(Arrays.asList(stringArray));

// Using Collections.addAll()

String[] stringArray = {"foo", "bar", "baz"};
ArrayList<String> list = new ArrayList<>();
Collections.addAll(list, stringArray);

Un point à noter à propos du [Diamond][3] est qu’il ne peut pas être utilisé avec [Anonymous Classes][6].

// Using Streams

int[] ints = {1, 2, 3};
List<Integer> list = Arrays.stream(ints).boxed().collect(Collectors.toList());

String[] stringArray = {"foo", "bar", "baz"};
List<Object> list = Arrays.stream(stringArray).collect(Collectors.toList());

Remarques importantes concernant l’utilisation de la méthode Arrays.asList() #

  • Cette méthode renvoie List, qui est une instance de Arrays$ArrayList (classe interne statique de Arrays) et non java.util.ArrayList. La “Liste” résultante est de taille fixe. Cela signifie que l’ajout ou la suppression d’éléments n’est pas pris en charge et lèvera une UnsupportedOperationException :

     stringList.add("something"); // throws java.lang.UnsupportedOperationException

  • Une nouvelle List peut être créée en passant une List basée sur un tableau au constructeur d’une nouvelle List. Cela crée une nouvelle copie des données, qui a une taille variable et qui n’est pas sauvegardée par le tableau d’origine :

     List<String> modifiableList = new ArrayList<>(Arrays.asList("foo", "bar"));

  • Appel de <T> List<T> asList(T... a) sur un tableau primitif, tel qu’un int[], produira une List<int[]> dont le seul élément est la source tableau primitif au lieu des éléments réels du tableau source.

La raison de ce comportement est que les types primitifs ne peuvent pas être utilisés à la place des paramètres de type générique, de sorte que le tableau primitif entier remplace le paramètre de type générique dans ce cas. Afin de convertir un tableau primitif en List, tout d’abord, convertissez le tableau primitif en un tableau du type wrapper correspondant (c’est-à-dire appelez Arrays.asList sur un Integer[] au lieu d’un int[ ]).

Par conséquent, cela affichera “faux”:

   int[] arr = {1, 2, 3};      // primitive array of int
   System.out.println(Arrays.asList(arr).contains(1));

Voir la démo

D’un autre côté, cela affichera ’true’ :

   Integer[] arr = {1, 2, 3};  // object array of Integer (wrapper for int)
   System.out.println(Arrays.asList(arr).contains(1));

Voir la démo

Cela affichera également true, car le tableau sera interprété comme un Integer []):

   System.out.println(Arrays.asList(1,2,3).contains(1));

Voir la démo

Création d’un tableau à partir d’une collection #

Deux méthodes dans [java.util.Collection][1] créent un tableau à partir d’une collection :

  • [Objet[] versTableau()][2]

  • [<T> T[] versTableau(T[] a)][3]

Object[] toArray() peut être utilisé comme suit :

Set<String> set = new HashSet<String>();
set.add("red");
set.add("blue");

// although set is a Set<String>, toArray() returns an Object[] not a String[]
Object[] objectArray = set.toArray();

<T> T[] toArray(T[] a) peut être utilisé comme suit :

Set<String> set = new HashSet<String>();
set.add("red");
set.add("blue");

// The array does not need to be created up front with the correct size.
// Only the array type matters. (If the size is wrong, a new array will
// be created with the same type.)
String[] stringArray = set.toArray(new String[0]);  

// If you supply an array of the same size as collection or bigger, it
// will be populated with collection values and returned (new array
// won't be allocated)
String[] stringArray2 = set.toArray(new String[set.size()]);

La différence entre eux est plus que d’avoir des résultats non typés vs typés. Leurs performances peuvent également différer (pour plus de détails, veuillez lire cette section d’analyse des performances) :

  • Object[] toArray() utilise arraycopy vectorisé, ce qui est beaucoup plus rapide que le type vérifié arraycopy utilisé dans T[] toArray(T[] a).
  • T[] toArray(new T[non-zero-size]) doit mettre à zéro le tableau au moment de l’exécution, tandis que T[] toArray(new T[0]) ne le fait pas. Un tel évitement rend ce dernier appel plus vite que le précédent. Analyse détaillée ici : [Tableaux de Sagesse des Anciens][4].

À partir de Java SE 8+, où le concept de [Stream][5] a été introduit, il est possible d’utiliser le Stream produit par la collection afin de créer un nouveau Array en utilisant le [Stream.toArray ][6] méthode.

String[] strings = list.stream().toArray(String[]::new);

Tableaux multidimensionnels et irréguliers #

Il est possible de définir un tableau à plusieurs dimensions. Au lieu d’être accessible en fournissant un index unique, un tableau multidimensionnel est accessible en spécifiant un index pour chaque dimension.

La déclaration d’un tableau multidimensionnel peut être effectuée en ajoutant [] pour chaque dimension à une déclaration de tableau régulière. Par exemple, pour créer un tableau int à 2 dimensions, ajoutez un autre ensemble de crochets à la déclaration, comme int[][]. Cela continue pour les tableaux à 3 dimensions (int[][][]) et ainsi de suite.

Pour définir un tableau à 2 dimensions avec trois lignes et trois colonnes :

int rows = 3;
int columns = 3;
int[][] table = new int[rows][columns];

Le tableau peut être indexé et lui attribuer des valeurs avec cette construction. Notez que les valeurs non affectées sont les valeurs par défaut pour le type d’un tableau, dans ce cas 0 pour int.

table[0][0] = 0;
table[0][1] = 1;
table[0][2] = 2;

Il est également possible d’instancier une dimension à la fois, et même de créer des tableaux non rectangulaires. Ceux-ci sont plus communément appelés [jagged arrays][1].

int[][] nonRect = new int[4][];

Il est important de noter que bien qu’il soit possible de définir n’importe quelle dimension d’un tableau en escalier, son niveau précédent doit être défini.

// valid
String[][] employeeGraph = new String[30][];

// invalid
int[][] unshapenMatrix = new int[][10];

// also invalid
int[][][] misshapenGrid = new int[100][][10];

ArrayIndexOutOfBoundsException #

L’exception [ArrayIndexOutOfBoundsException][1] est levée lorsqu’un index inexistant d’un tableau est en cours d’accès.

Les tableaux sont indexés à partir de zéro, donc l’index du premier élément est ‘0’ et l’index du dernier élément est la capacité du tableau moins ‘1’ (c’est-à-dire ‘array.length - 1’).

Par conséquent, toute demande d’élément de tableau par l’index i doit satisfaire la condition 0 <= i < array.length, sinon l’exception ArrayIndexOutOfBoundsException sera levée.

Le code suivant est un exemple simple où une ArrayIndexOutOfBoundsException est levée.

String[] people = new String[] { "Carol", "Andy" };

// An array will be created:
// people[0]: "Carol"
// people[1]: "Andy"

// Notice: no item on index 2. Trying to access it triggers the exception:
System.out.println(people[2]);  // throws an ArrayIndexOutOfBoundsException.

Production:

Exception in thread "main" java.lang.ArrayIndexOutOfBoundsException: 2
    at your.package.path.method(YourClass.java:15)

Notez que l’index illégal auquel vous accédez est également inclus dans l’exception (“2” dans l’exemple) ; cette information peut être utile pour trouver la cause de l’exception.

Pour éviter cela, vérifiez simplement que l’index est bien dans les limites du tableau :

int index = 2;
if (index >= 0 && index < people.length) {
    System.out.println(people[index]);
}

Tableaux à diffuser #

Conversion d’un tableau d’objets en “Stream” :

String[] arr = new String[] {"str1", "str2", "str3"};
Stream<String> stream = Arrays.stream(arr);

La conversion d’un tableau de primitives en Stream à l’aide de [Arrays.stream()][1] transformera le tableau en une spécialisation primitive de Stream :

int[] intArr = {1, 2, 3};

Vous pouvez également limiter le Stream à une plage d’éléments dans le tableau. L’index de début est inclusif et l’index de fin est exclusif :

int[] values = {1, 2, 3, 4};
IntStream intStream = Arrays.stream(values, 2, 4);

Une méthode similaire à Arrays.stream() apparaît dans la classe Stream : [Stream.of()][2]. La différence est que Stream.of() utilise un paramètre varargs, vous pouvez donc écrire quelque chose comme :

Stream<String> stringStream = Stream.of("1", "2", "3");
Stream<Double> doubleStream = Stream.of(new Double[]{1.0, 2.0});

Covariance du tableau #

Les tableaux d’objets sont covariants, ce qui signifie que tout comme Integer est une sous-classe de Number, Integer[] est une sous-classe de Number[]. Cela peut sembler intuitif, mais peut entraîner des comportements surprenants :

Integer[] integerArray = {1, 2, 3};
Number[] numberArray = integerArray;  // valid
Number firstElement = numberArray[0]; // valid
numberArray[0] = 4L;                  // throws ArrayStoreException at runtime

Bien que Integer[] soit une sous-classe de Number[], il ne peut contenir que Integers, et essayer d’assigner un élément Long lève une exception d’exécution.

Notez que ce comportement est unique aux tableaux et peut être évité en utilisant à la place une “Liste” générique :

List<Integer> integerList = Arrays.asList(1, 2, 3);
//List<Number> numberList = integerList;  // compile error
List<? extends Number> numberList = integerList;
Number firstElement = numberList.get(0);
//numberList.set(0, 4L);                  // compile error

Il n’est pas nécessaire que tous les éléments du tableau partagent le même type, tant qu’ils sont une sous-classe du type du tableau :

interface I {}

class A implements I {}
class B implements I {}
class C implements I {}

I[] array10 = new I[] { new A(), new B(), new C() }; // Create an array with new
                                                     // operator and array initializer.

I[] array11 = { new A(), new B(), new C() };         // Shortcut syntax with array
                                                     // initializer.

I[] array12 = new I[3];                              // { null, null, null }

I[] array13 = new A[] { new A(), new A() };          // Works because A implements I.

Object[] array14 = new Object[] { "Hello, World!", 3.14159, 42 }; // Create an array with
                                                     // new operator and array initializer.

Object[] array15 = { new A(), 64, "My String" };     // Shortcut syntax 
                                                     // with array initializer.

Itérer sur des tableaux #

Vous pouvez itérer sur des tableaux soit en utilisant une boucle for améliorée (alias foreach), soit en utilisant des indices de tableau :

int[] array = new int[10];

// using indices: read and write
for (int i = 0; i < array.length; i++) {
    array[i] = i;
}

// extended for: read only
for (int e : array) {
    System.out.println(e);
}

Il convient de noter ici qu’il n’existe aucun moyen direct d’utiliser un itérateur sur un tableau, mais via la bibliothèque Arrays, il peut être facilement converti en une liste pour obtenir un objet Iterable.

Pour les tableaux en boîte, utilisez [Arrays.asList][1] :

Integer[] boxed = {1, 2, 3};
Iterable<Integer> boxedIt = Arrays.asList(boxed); // list-backed iterable
Iterator<Integer> fromBoxed1 = boxedIt.iterator();

Pour les tableaux primitifs (utilisant Java 8), utilisez des flux (en particulier dans cet exemple - [Arrays.stream][2] -> [IntStream][2]) :

int[] primitives = {1, 2, 3};
IntStream primitiveStream = Arrays.stream(primitives); // list-backed iterable
PrimitiveIterator.OfInt fromPrimitive1 = primitiveStream.iterator();

Si vous ne pouvez pas utiliser de flux (pas de java 8), vous pouvez choisir d’utiliser la bibliothèque [guava][3] de Google :

Iterable<Integer> fromPrimitive2 = Ints.asList(primitives);

Dans les réseaux bidimensionnels ou plus, les deux techniques peuvent être utilisées de manière légèrement plus complexe.

Exemple:

int[][] array = new int[10][10];

for (int indexOuter = 0; indexOuter < array.length; indexOuter++) {
    for (int indexInner = 0; indexInner < array[indexOuter].length; indexInner++) {
        array[indexOuter][indexInner] = indexOuter + indexInner;
    }
}

for (int[] numbers : array) {
    for (int value : numbers) {
        System.out.println(value);
    }
}

Il est impossible de définir un tableau sur une valeur non uniforme sans utiliser une boucle basée sur un index.

Bien sûr, vous pouvez également utiliser des boucles while ou do-while lors de l’itération à l’aide d’index.

Une note de prudence : lors de l’utilisation d’indices de tableau, assurez-vous que l’indice est compris entre 0 et array.length - 1 (les deux inclus). Ne faites pas d’hypothèses codées en dur sur la longueur du tableau, sinon vous risquez de casser votre code si la longueur du tableau change, mais pas vos valeurs codées en dur.

Exemple:

int[] numbers = {1, 2, 3, 4};

public void incrementNumbers() {
    // DO THIS :
    for (int i = 0; i < numbers.length; i++) {
        numbers[i] += 1; //or this: numbers[i] = numbers[i] + 1; or numbers[i]++;      
    }
 
    // DON'T DO THIS :
    for (int i = 0; i < 4; i++) {
        numbers[i] += 1;
    }
}

Il est également préférable de ne pas utiliser de calculs sophistiqués pour obtenir l’index, mais d’utiliser l’index pour itérer et, si vous avez besoin de valeurs différentes, de les calculer.

Exemple:

public void fillArrayWithDoubleIndex(int[] array) {
    // DO THIS :
    for (int i = 0; i < array.length; i++) {
        array[i] = i * 2;
    }
 
    // DON'T DO THIS :
    int doubleLength = array.length * 2;
    for (int i = 0; i < doubleLength; i += 2) {
        array[i / 2] = i;
    }
}

Accéder aux baies dans l’ordre inverse

int[] array = {0, 1, 1, 2, 3, 5, 8, 13};
for (int i = array.length - 1; i >= 0; i--) {
   System.out.println(array[i]);
}

Utilisation de tableaux temporaires pour réduire la répétition du code

Itérer sur un tableau temporaire au lieu de répéter le code peut rendre votre code plus propre. Il peut être utilisé lorsque la même opération est effectuée sur plusieurs variables.

// we want to print out all of these
String name = "Margaret";
int eyeCount = 16;
double height = 50.2;
int legs = 9;
int arms = 5;


// copy-paste approach:
System.out.println(name);
System.out.println(eyeCount);
System.out.println(height);
System.out.println(legs);
System.out.println(arms);


// temporary array approach:
for(Object attribute : new Object[]{name, eyeCount, height, legs, arms})
    System.out.println(attribute);

// using only numbers
for(double number : new double[]{eyeCount, legs, arms, height})
    System.out.println(Math.sqrt(number));

Gardez à l’esprit que ce code ne doit pas être utilisé dans les sections critiques pour les performances, car un tableau est créé chaque fois que la boucle est entrée, et que les variables primitives seront copiées dans le tableau et ne pourront donc pas être modifiées.

Tableaux vers une chaîne #

Depuis Java 1.5, vous pouvez obtenir une représentation String du contenu du tableau spécifié sans itérer sur chacun de ses éléments. Utilisez simplement [Arrays.toString(Object[])][1] ou [Arrays.deepToString(Object[])][2] pour les tableaux multidimensionnels :

int[] arr = {1, 2, 3, 4, 5};
System.out.println(Arrays.toString(arr));      // [1, 2, 3, 4, 5]

int[][] arr = {
    {1, 2, 3},
    {4, 5, 6},
    {7, 8, 9}
};
System.out.println(Arrays.deepToString(arr));  // [[1, 2, 3], [4, 5, 6], [7, 8, 9]]

La méthode Arrays.toString() utilise la méthode [Object.toString()][3] pour produire les valeurs String de chaque élément du tableau, à côté du tableau de type primitif, elle peut être utilisée pour tous les types de tableaux. Par exemple:

public class Cat { /* implicitly extends Object */
    @Override
    public String toString() {
      return "CAT!";
    }
}

Cat[] arr = { new Cat(), new Cat() };
System.out.println(Arrays.toString(arr));        // [CAT!, CAT!]

Si aucun toString() remplacé n’existe pour la classe, alors le toString() hérité de Object sera utilisé. Habituellement, la sortie n’est alors pas très utile, par exemple :

public class Dog {
    /* implicitly extends Object */
}

Dog[] arr = { new Dog() };
System.out.println(Arrays.toString(arr));        // [Dog@17ed40e0]

Trier un tableau d’objets

Afin de trier un tableau d’objets, tous les éléments doivent implémenter l’interface Comparable ou Comparator pour définir l’ordre du tri.

Nous pouvons utiliser l’une ou l’autre des méthodes sort(Object[]) pour trier un tableau d’objets dans son ordre naturel, mais vous devez vous assurer que tous les éléments du tableau doivent implémenter Comparable.

De plus, ils doivent également être mutuellement comparables, par exemple e1.compareTo(e2) ne doit pas lancer une ClassCastException pour les éléments e1 et e2 du tableau. Vous pouvez également trier un tableau d’objets dans un ordre personnalisé à l’aide de la méthode sort(T[], Comparator), comme indiqué dans l’exemple suivant.

Obtenir la longueur d’un tableau #

Les tableaux sont des objets qui fournissent de l’espace pour stocker jusqu’à sa taille des éléments du type spécifié. La taille d’un tableau ne peut pas être modifiée après la création du tableau.

int[] arr1 = new int[0];
int[] arr2 = new int[2];
int[] arr3 = new int[]{1, 2, 3, 4};
int[] arr4 = {1, 2, 3, 4, 5, 6, 7};

int len1 = arr1.length; // 0
int len2 = arr2.length; // 2
int len3 = arr3.length; // 4
int len4 = arr4.length; // 7

Le champ length dans un tableau stocke la taille d’un tableau. Il s’agit d’un champ “final” et ne peut pas être modifié.

Ce code montre la différence entre la “longueur” d’un tableau et la quantité d’objets qu’un tableau stocke.

public static void main(String[] args) {
    Integer arr[] = new Integer[] {1,2,3,null,5,null,7,null,null,null,11,null,13};

    int arrayLength = arr.length;
    int nonEmptyElementsCount = 0;

    for (int i=0; i<arrayLength; i++) {
        Integer arrElt = arr[i];
        if (arrElt != null) {
            nonEmptyElementsCount++;
        }
    }

    System.out.println("Array 'arr' has a length of "+arrayLength+"\n"
                            + "and it contains "+nonEmptyElementsCount+" non-empty values");
}

Résultat:

Array 'arr' has a length of 13
and it contains 7 non-empty values

Supprimer un élément d’un tableau #

Java ne fournit pas de méthode directe dans [java.util.Arrays][1] pour supprimer un élément d’un tableau. Pour l’exécuter, vous pouvez soit copier le tableau d’origine dans un nouveau sans l’élément à supprimer, soit convertir votre tableau dans une autre structure permettant la suppression.

Utilisation de ArrayList #

Vous pouvez convertir le tableau en [java.util.List][2], supprimer l’élément et reconvertir la liste en tableau comme suit :

String[] array = new String[]{"foo", "bar", "baz"};

List<String> list = new ArrayList<>(Arrays.asList(array));
list.remove("foo");

// Creates a new array with the same size as the list and copies the list
// elements to it.
array = list.toArray(new String[list.size()]);

System.out.println(Arrays.toString(array)); //[bar, baz]

Utilisation de System.arraycopy #

[System.arraycopy()][3] peut être utilisé pour faire une copie du tableau d’origine et supprimer l’élément souhaité. Ci-dessous un exemple :

int[] array = new int[] { 1, 2, 3, 4 }; // Original array.
int[] result = new int[array.length - 1]; // Array which will contain the result.
int index = 1; // Remove the value "2".

// Copy the elements at the left of the index.
System.arraycopy(array, 0, result, 0, index);
// Copy the elements at the right of the index.
System.arraycopy(array, index + 1, result, index, array.length - index - 1);

System.out.println(Arrays.toString(result)); //[1, 3, 4]

Utilisation d’Apache Commons Lang #

Pour supprimer facilement un élément, vous pouvez utiliser la bibliothèque [Apache Commons Lang][4] et notamment la méthode statique [removeElement()][5] de la classe [ArrayUtils][6]. Ci-dessous un exemple :

int[] array = new int[]{1,2,3,4};
array = ArrayUtils.removeElement(array, 2); //remove first occurrence of 2
System.out.println(Arrays.toString(array)); //[1, 3, 4]

Comment changer la taille d’un tableau ? #

La réponse simple est que vous ne pouvez pas faire cela. Une fois qu’un tableau a été créé, sa taille ne peut pas être modifiée. Au lieu de cela, un tableau ne peut être “redimensionné” qu’en créant un nouveau tableau avec la taille appropriée et en copiant les éléments du tableau existant vers le nouveau.

String[] listOfCities = new String[3];   // array created with size 3.
listOfCities[0] = "New York";
listOfCities[1] = "London";
listOfCities[2] = "Berlin";

Supposons (par exemple) qu’un nouvel élément doive être ajouté au tableau listOfCities défini comme ci-dessus. Pour ce faire, vous devrez :

  1. créer un nouveau tableau de taille 4,
  2. copiez les 3 éléments existants de l’ancien tableau dans le nouveau tableau aux décalages 0, 1 et 2, et
  3. ajoutez le nouvel élément au nouveau tableau au décalage 3.

Il existe différentes façons de faire ce qui précède. Avant Java 6, la manière la plus concise était :

String[] newArray = new String[listOfCities.length + 1];
System.arraycopy(listOfCities, 0, newArray, 0, listOfCities.length);
newArray[listOfCities.length] = "Sydney";

A partir de Java 6, les méthodes Arrays.copyOf et Arrays.copyOfRange peuvent le faire plus simplement :

String[] newArray = Arrays.copyOf(listOfCities, listOfCities.length + 1);
newArray[listOfCities.length] = "Sydney";

Pour d’autres façons de copier un tableau, reportez-vous à l’exemple suivant. Gardez à l’esprit que vous avez besoin d’une copie de tableau avec une longueur différente de l’original lors du redimensionnement.

Une meilleure alternative au redimensionnement de tableau #

Il y a deux inconvénients majeurs au redimensionnement d’un tableau comme décrit ci-dessus :

  • C’est inefficace. Agrandir (ou réduire) un tableau implique de copier plusieurs ou tous les éléments de tableau existants et d’allouer un nouvel objet de tableau. Plus le réseau est grand, plus il coûte cher.
  • Vous devez être en mesure de mettre à jour toutes les variables “live” contenant des références à l’ancien tableau.

Une alternative consiste à créer le tableau avec une taille suffisamment grande pour commencer. Ceci n’est viable que si vous pouvez déterminer cette taille avec précision * avant d’allouer le tableau *. Si vous ne pouvez pas le faire, le problème du redimensionnement du tableau se pose à nouveau.

L’autre alternative consiste à utiliser une classe de structure de données fournie par la bibliothèque de classes Java SE ou une bibliothèque tierce. Par exemple, le framework “collections” Java SE fournit un certain nombre d’implémentations des API List, Set et Map avec différentes propriétés d’exécution. La classe ArrayList est la plus proche des caractéristiques de performance d’un tableau simple (par exemple, recherche O(N), O(1) get and set, O(N) insertion et suppression aléatoires) tout en offrant un redimensionnement plus efficace sans le problème de mise à jour de référence.

(L’efficacité du redimensionnement pour ArrayList provient de sa stratégie consistant à doubler la taille du tableau de sauvegarde à chaque redimensionnement. Pour un cas d’utilisation typique, cela signifie que vous ne redimensionnez qu’occasionnellement. Lorsque vous amortissez sur la durée de vie de la liste, le Le coût de redimensionnement par insertion est O(1). Il peut être possible d’utiliser la même stratégie lors du redimensionnement d’un tableau simple.)

Recherche d’un élément dans un tableau #

Il existe de nombreuses façons de trouver l’emplacement d’une valeur dans un tableau. Les extraits d’exemple suivants supposent tous que le tableau est l’un des suivants :

  String[] strings = new String[] { "A", "B", "C" };
  int[] ints = new int[] { 1, 2, 3, 4 };

De plus, chacun définit index ou index2 soit l’index de l’élément requis, soit -1 si l’élément n’est pas présent.

Utilisation de [Arrays.binarySearch][1] (pour les tableaux triés uniquement) #

  int index = Arrays.binarySearch(strings, "A");
  int index2 = Arrays.binarySearch(ints, 1);

Utilisation d’un Arrays.asList (pour les tableaux non primitifs uniquement) #

  int index = Arrays.asList(strings).indexOf("A");
  int index2 = Arrays.asList(ints).indexOf(1);  // compilation error

Utilisation d’un “flux” #

  int index = IntStream.range(0, strings.length)
                       .filter(i -> "A".equals(strings[i]))
                       .findFirst()
                       .orElse(-1); // If not present, gives us -1.
  // Similar for an array of primitives

Recherche linéaire à l’aide d’une boucle #

  int index = -1;
  for (int i = 0; i < array.length; i++) {
      if ("A".equals(array[i])) {
          index = i;
          break;
      }            
  }
  // Similar for an array of primitives

Recherche linéaire à l’aide de bibliothèques tierces telles que [org.apache.commons][3] #

  int index = org.apache.commons.lang3.ArrayUtils.contains(strings, "A");
  int index2 = org.apache.commons.lang3.ArrayUtils.contains(ints, 1);

Remarque : L’utilisation d’une recherche linéaire directe est plus efficace que l’encapsulation dans une liste.

Tester si un tableau contient un élément #

Les exemples ci-dessus peuvent être adaptés pour tester si le tableau contient un élément en testant simplement si l’indice calculé est supérieur ou égal à zéro.

Alternativement, il existe également des variantes plus concises :

boolean isPresent = Arrays.asList(strings).contains("A");

boolean isPresent = Stream<String>.of(strings).anyMatch(x -> "A".equals(x));

boolean isPresent = false;
for (String s : strings) {
    if ("A".equals(s)) {
        isPresent = true;
        break;
    }
}

boolean isPresent = org.apache.commons.lang3.ArrayUtils.contains(ints, 4);

Conversion de tableaux entre primitives et types encadrés #

Parfois, la conversion de types [primitifs][1] en types [boxed][2] est nécessaire.

Pour convertir le tableau, il est possible d’utiliser des flux (en Java 8 et supérieur) :

int[] primitiveArray = {1, 2, 3, 4};
Integer[] boxedArray = 
    Arrays.stream(primitiveArray).boxed().toArray(Integer[]::new);

Avec les versions inférieures, cela peut être en itérant le tableau primitif et en le copiant explicitement dans le tableau encadré :

int[] primitiveArray = {1, 2, 3, 4};
Integer[] boxedArray = new Integer[primitiveArray.length];
for (int i = 0; i < primitiveArray.length; ++i) {
    boxedArray[i] = primitiveArray[i]; // Each element is autoboxed here
}

De même, un tableau en boîte peut être converti en un tableau de son homologue primitif :

Integer[] boxedArray = {1, 2, 3, 4};
int[] primitiveArray = 
    Arrays.stream(boxedArray).mapToInt(Integer::intValue).toArray();

Integer[] boxedArray = {1, 2, 3, 4};
int[] primitiveArray = new int[boxedArray.length];
for (int i = 0; i < boxedArray.length; ++i) {
    primitiveArray[i] = boxedArray[i]; // Each element is outboxed here
}

Comparer des tableaux pour l’égalité #

Les types de tableaux héritent de leurs implémentations [equals() (et hashCode()) de java.lang.Object](https://www.wikiod.com/fr/java/methodes-de-classe-dobjet-et-constructeur#méthode equals() -method#t=201607191720385793182), donc equals() ne renverra vrai que lors de la comparaison avec exactement le même objet tableau. Pour comparer l’égalité des tableaux en fonction de leurs valeurs, utilisez [java.util.Arrays.equals](https://docs.oracle.com/javase/8/docs/api/java/util/Arrays.html#equals -java.lang.Object:A-java.lang.Object:A-), qui est surchargé pour tous les types de tableaux.

int[] a = new int[]{1, 2, 3};
int[] b = new int[]{1, 2, 3};
System.out.println(a.equals(b)); //prints "false" because a and b refer to different objects
System.out.println(Arrays.equals(a, b)); //prints "true" because the elements of a and b have the same values

Lorsque le type d’élément est un type référence, Arrays.equals() appelle equals() sur les éléments du tableau pour déterminer l’égalité. En particulier, si le type d’élément est lui-même un type de tableau, la comparaison d’identité sera utilisée. Pour comparer l’égalité des tableaux multidimensionnels, utilisez [Arrays.deepEquals()](https://docs.oracle.com/javase/8/docs/api/java/util/Arrays.html#deepEquals-java.lang. Object:A-java.lang.Object:A-) à la place comme ci-dessous :

int a[] = { 1, 2, 3 };
int b[] = { 1, 2, 3 };

Object[] aObject = { a }; // aObject contains one element
Object[] bObject = { b }; // bObject contains one element

System.out.println(Arrays.equals(aObject, bObject)); // false
System.out.println(Arrays.deepEquals(aObject, bObject));// true

Étant donné que les ensembles et les cartes utilisent equals() et hashCode(), les tableaux ne sont généralement pas utiles en tant qu’éléments d’ensemble ou clés de carte. Enveloppez-les dans une classe d’assistance qui implémente equals() et hashCode() en termes d’éléments de tableau, ou convertissez-les en instances List et stockez les listes.

Copie de tableaux #

Java propose plusieurs façons de copier un tableau.

pour la boucle #

int[] a = { 4, 1, 3, 2 };
int[] b = new int[a.length]; 
for (int i = 0; i < a.length; i++) {
    b[i] = a[i];
}

Notez que l’utilisation de cette option avec un tableau d’objets au lieu d’un tableau primitif remplira la copie avec une référence au contenu original au lieu d’une copie de celui-ci.

[Objet.clone()][1] #

Comme les tableaux sont des Objects en Java, vous pouvez utiliser [Object.clone()][1].

int[] a = { 4, 1, 3, 2 };
int[] b = a.clone(); // [4, 1, 3, 2]

Notez que la méthode Object.clone pour un tableau effectue une copie superficielle, c’est-à-dire qu’elle renvoie une référence à un nouveau tableau qui référence les mêmes éléments que le tableau source.

[Tableaux.copyOf()][2] #

[java.util.Arrays][3] fournit un moyen simple d’effectuer la copie d’un tableau vers un autre. Voici l’utilisation de base :

int[] a = {4, 1, 3, 2};
int[] b = Arrays.copyOf(a, a.length); // [4, 1, 3, 2]

Notez que Arrays.copyOf fournit également une surcharge qui vous permet de changer le type du tableau :

Double[] doubles = { 1.0, 2.0, 3.0 };
Number[] numbers = Arrays.copyOf(doubles, doubles.length, Number[].class);

[System.arraycopy()][4] #

public static void arraycopy(Object src, int srcPos, Object dest, int destPos, int length) Copies an array from the specified source array, en commençant à la position spécifiée, jusqu’à la position spécifiée du tableau de destination.

Ci-dessous un exemple d’utilisation

int[] a = { 4, 1, 3, 2 };
int[] b = new int[a.length];
System.arraycopy(a, 0, b, 0, a.length); // [4, 1, 3, 2]

[Tableaux.copyOfRange()][5] #

Principalement utilisé pour copier une partie d’un tableau, vous pouvez également l’utiliser pour copier tout un tableau dans un autre comme ci-dessous :

int[] a = { 4, 1, 3, 2 };
int[] b = Arrays.copyOfRange(a, 0, a.length); // [4, 1, 3, 2]

Lancer des tableaux #

Les tableaux sont des objets, mais leur type est défini par le type des objets contenus. Par conséquent, on ne peut pas simplement transtyper A[] en T[], mais chaque membre A du A[] spécifique doit être transtypé en un objet T. Exemple générique :

public static <T, A> T[] castArray(T[] target, A[] array) {
    for (int i = 0; i < array.length; i++) {
        target[i] = (T) array[i];
    }
    return target;
}

Ainsi, étant donné un tableau A[] :

T[] target = new T[array.Length];
target = castArray(target, array);

Java SE fournit la méthode [Arrays.copyOf(original, newLength, newType)][1] à cette fin :

Double[] doubles = { 1.0, 2.0, 3.0 };
Number[] numbers = Arrays.copyOf(doubles, doubles.length, Number[].class);
