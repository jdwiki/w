[Java I/O][1] (Entrée et Sortie) est utilisé pour traiter l’entrée et produire la sortie. Java utilise le concept de flux pour accélérer les opérations d’E/S. Le package java.io contient toutes les classes requises pour les opérations d’entrée et de sortie. [La gestion des fichiers] [2] est également effectuée en Java par l’API Java I/O.

[1] : https://docs.oracle.com/javase/7/docs/api/java/io/package-summary.html [2] : https://docs.oracle.com/javase/7/docs/api/java/io/File.html

Migration de java.io.File vers Java 7 NIO (java.nio.file.Path) #

Ces exemples supposent que vous savez déjà ce qu’est le NIO de Java 7 en général et que vous avez l’habitude d’écrire du code à l’aide de java.io.File. Utilisez ces exemples comme moyen de trouver rapidement une documentation plus centrée sur NIO pour la migration.

Il y a beaucoup plus dans NIO de Java 7, comme [fichiers mappés en mémoire][1] ou [ouverture d’un fichier ZIP ou JAR à l’aide de FileSystem][2]. Ces exemples ne couvriront qu’un nombre limité de cas d’utilisation de base.

En règle générale, si vous avez l’habitude d’effectuer une opération de lecture/écriture sur le système de fichiers à l’aide d’un [java.io.File](https://docs.oracle.com/javase/7/docs/api/java/ io/File.html), vous la trouverez en tant que méthode statique dans [java.nio.file.Files](https://docs.oracle.com/javase/7/docs/api/java/ nio/fichier/Fichiers.html).

Pointer sur un chemin #

// -> IO
File file = new File("io.txt");

// -> NIO
Path path = Paths.get("nio.txt");

Chemins relatifs à un autre chemin #

// Forward slashes can be used in place of backslashes even on a Windows operating system
// -> IO
File folder = new File("C:/");
File fileInFolder = new File(folder, "io.txt");

// -> NIO
Path directory = Paths.get("C:/");
Path pathInDirectory = directory.resolve("nio.txt");

Conversion du fichier depuis/vers le chemin pour une utilisation avec les bibliothèques #

// -> IO to NIO
Path pathFromFile = new File("io.txt").toPath();

// -> NIO to IO
File fileFromPath = Paths.get("nio.txt").toFile();

Vérifiez si le fichier existe et supprimez-le si c’est le cas #

// -> IO
if (file.exists()) {
    boolean deleted = file.delete();
    if (!deleted) {
        throw new IOException("Unable to delete file");
    }
}

// -> NIO
Files.deleteIfExists(path);

Flux vs API Writer/Reader #

Les flux fournissent l’accès le plus direct au contenu binaire, donc toutes les implémentations [InputStream][1] / [OutputStream][2] fonctionnent toujours sur ints et bytes.

// Read a single byte from the stream
int b = inputStream.read();
if (b >= 0) { // A negative value represents the end of the stream, normal values are in the range 0 - 255
    // Write the byte to another stream
    outputStream.write(b);
}

// Read a chunk
byte[] data = new byte[1024];
int nBytesRead = inputStream.read(data);
if (nBytesRead >= 0) { // A negative value represents end of stream
    // Write the chunk to another stream
    outputStream.write(data, 0, nBytesRead);
}

Il existe quelques exceptions, probablement plus particulièrement le [PrintStream][3] qui ajoute la “possibilité d’imprimer facilement des représentations de diverses valeurs de données”. Cela permet d’utiliser [System.out][4] à la fois comme un InputStream binaire et comme une sortie textuelle en utilisant des méthodes telles que System.out.println().

De plus, certaines implémentations de flux fonctionnent comme une interface vers des contenus de niveau supérieur tels que des objets Java (voir Sérialisation) ou des types natifs, par ex. [DataOutputStream][5] / [DataInputStream][6].

Avec les classes [Writer][7] et [Reader][8], Java fournit également une API pour les flux de caractères explicites. Bien que la plupart des applications basent ces implémentations sur des flux, l’API de flux de caractères n’expose aucune méthode pour le contenu binaire.

// This example uses the platform's default charset, see below
// for a better implementation.

Writer writer = new OutputStreamWriter(System.out);
writer.write("Hello world!");

Reader reader = new InputStreamReader(System.in);
char singleCharacter = reader.read();

Chaque fois qu’il est nécessaire d’encoder des caractères en données binaires (par exemple lors de l’utilisation des classes InputStreamWriter / OutputStreamWriter), vous devez spécifier un jeu de caractères si vous ne souhaitez pas dépendre du jeu de caractères par défaut de la plate-forme. En cas de doute, utilisez un encodage compatible Unicode, par ex. UTF-8 qui est pris en charge sur toutes les plates-formes Java. Par conséquent, vous devriez probablement rester à l’écart des classes comme FileWriter et FileReader car celles-ci utilisent toujours le jeu de caractères par défaut de la plate-forme. Voici une meilleure façon d’accéder aux fichiers à l’aide de flux de caractères :

Charset myCharset = StandardCharsets.UTF_8;

Writer writer = new OutputStreamWriter( new FileOutputStream("test.txt"), myCharset );
writer.write('Ä');
writer.flush();
writer.close();

Reader reader = new InputStreamReader( new FileInputStream("test.txt"), myCharset );
char someUnicodeCharacter = reader.read();
reader.close();

L’un des Reader les plus couramment utilisés est BufferedReader qui fournit une méthode pour lire des lignes entières de texte à partir d’un autre lecteur et est probablement le moyen le plus simple de lire un flux de caractères ligne par ligne :

// Read from baseReader, one line at a time
BufferedReader reader = new BufferedReader( baseReader );
String line;
while((line = reader.readLine()) != null) {
  // Remember: System.out is a stream, not a writer!
  System.out.println(line);
}

Copier un fichier en utilisant InputStream et OutputStream #

Nous pouvons copier directement les données d’une source vers un puits de données à l’aide d’une boucle. Dans cet exemple, nous lisons des données à partir d’un InputStream et en même temps, nous écrivons dans un OutputStream. Une fois que nous avons fini de lire et d’écrire, nous devons fermer la ressource.

public void copy(InputStream source, OutputStream destination) throws IOException {
    try {
        int c;
        while ((c = source.read()) != -1) {
            destination.write(c);
        }
    } finally {
        if (source != null) {
            source.close();
        }
        if (destination != null) {
            destination.close();
        }
    }
}

Lecture d’un fichier en utilisant Channel et Buffer #

Channel utilise un Buffer pour lire/écrire des données. Un tampon est un conteneur de taille fixe dans lequel nous pouvons écrire un bloc de données à la fois. Channel est bien plus rapide que les E/S basées sur le flux.

Pour lire les données d’un fichier à l’aide de Channel, nous devons suivre les étapes suivantes :

  1. Nous avons besoin d’une instance de FileInputStream. FileInputStream a une méthode nommée getChannel() qui renvoie un canal.

  2. Appelez la méthode getChannel() de FileInputStream et acquérez Channel.

  3. Créez un ByteBuffer. ByteBuffer est un conteneur d’octets de taille fixe.

  4. Channel a une méthode de lecture et nous devons fournir un ByteBuffer comme argument à cette méthode de lecture. ByteBuffer a deux modes - mode lecture seule et mode écriture seule. Nous pouvons changer le mode en utilisant l’appel de la méthode flip(). Le tampon a une position, une limite et une capacité. Une fois qu’un tampon est créé avec une taille fixe, sa limite et sa capacité sont les mêmes que la taille et la position commence à zéro. Pendant qu’un tampon est écrit avec des données, sa position augmente progressivement. Changer de mode signifie changer de position. Pour lire les données depuis le début d’un tampon, nous devons définir la position à zéro. la méthode flip() change la position

  5. Lorsque nous appelons la méthode de lecture du Channel, il remplit le tampon en utilisant des données.

  6. Si nous devons lire les données du ByteBuffer, nous devons retourner le tampon pour changer son mode d’écriture seule en mode lecture seule, puis continuer à lire les données du tampon.

  7. Lorsqu’il n’y a plus de données à lire, la méthode read() du canal renvoie 0 ou -1.

    import java.io.File; import java.io.FileInputStream; import java.io.IOException; import java.nio.ByteBuffer; import java.nio.channels.FileChannel;

    public class FileChannelRead {

    public static void main(String[] args) {

    File inputFile = new File("hello.txt");

if (!inputFile.exists()) {
 System.out.println("The input file doesn't exit.");
 return;
}

    try { FileInputStream fis = new FileInputStream(inputFile); FileChannel fileChannel = fis.getChannel(); ByteBuffer buffer = ByteBuffer.allocate(1024);

    while (fileChannel.read(buffer) > 0) {
 buffer.flip();
 while (buffer.hasRemaining()) {
  byte b = buffer.get();
  System.out.print((char) b);
 }
 buffer.clear();
}

fileChannel.close();

    } catch (IOException e) { e.printStackTrace(); } } }

Ajout de répertoires #

Blocage ou redirection de la sortie standard / erreur #

Parfois, une bibliothèque tierce mal conçue écrit des diagnostics indésirables dans les flux “System.out” ou “System.err”. Les solutions recommandées seraient de trouver une meilleure bibliothèque ou (dans le cas de l’open source) de résoudre le problème et de fournir un correctif aux développeurs.

Si les solutions ci-dessus ne sont pas réalisables, vous devriez envisager de rediriger les flux.

Redirection sur la ligne de commande

Sur un système UNIX, Linux ou MacOSX, cela peut être fait à partir du shell en utilisant la redirection >. Par exemple:

$ java -jar app.jar arg1 arg2 > /dev/null 2>&1
$ java -jar app.jar arg1 arg2 > out.log 2> error.log

Le premier redirige la sortie standard et l’erreur standard vers “/dev/null”, ce qui supprime tout ce qui est écrit dans ces flux. Le second redirige la sortie standard vers “out.log” et l’erreur standard vers “error.log”.

(Pour plus d’informations sur la redirection, reportez-vous à la documentation du shell de commande que vous utilisez. Des conseils similaires s’appliquent à Windows.)

Vous pouvez également implémenter la redirection dans un script wrapper ou un fichier batch qui lance l’application Java.

Redirection au sein d’une application Java

Il est également possible de rediriger les flux dans une application Java en utilisant System.setOut() et System.setErr(). Par exemple, l’extrait de code suivant redirige la sortie standard et l’erreur standard vers 2 fichiers journaux :

System.setOut(new PrintStream(new FileOutputStream(new File("out.log"))));
System.setErr(new PrintStream(new FileOutputStream(new File("err.log"))));

Si vous souhaitez supprimer entièrement la sortie, vous pouvez créer un flux de sortie qui “écrit” dans un descripteur de fichier non valide. Cela équivaut fonctionnellement à écrire dans “/dev/null” sous UNIX.

System.setOut(new PrintStream(new FileOutputStream(new FileDescriptor())));
System.setErr(new PrintStream(new FileOutputStream(new FileDescriptor())));

Attention : soyez prudent lorsque vous utilisez setOut et setErr :

  1. La redirection affectera l’ensemble de la JVM.
  2. En procédant ainsi, vous privez l’utilisateur de la possibilité de rediriger les flux à partir de la ligne de commande.
