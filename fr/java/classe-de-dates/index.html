Syntaxe #

  • Objet Date = nouvelle Date();
  • Objet Date = nouvelle Date(date longue);

Paramètres #

Paramètre Explication
Aucun paramètre Crée un nouvel objet Date en utilisant le temps d’allocation (à la milliseconde près)
longue date Crée un nouvel objet Date avec l’heure définie sur le nombre de millisecondes depuis “l’époque” (1er janvier 1970, 00:00:00 GMT)

Représentation

En interne, un objet Java Date est représenté par un long ; c’est le nombre de millisecondes écoulées depuis un moment spécifique (appelé époque). La classe Java Date d’origine avait des méthodes pour gérer les fuseaux horaires, etc., mais celles-ci ont été dépréciées au profit de la nouvelle classe Calendar.

Donc, si tout ce que vous voulez faire dans votre code est de représenter une heure spécifique, vous pouvez créer une classe Date et la stocker, etc. Cependant, si vous voulez imprimer une version lisible par l’homme de cette date, vous créez une classe Calendar et utilisez sa mise en forme pour produire des heures, des minutes, des secondes, des jours, des fuseaux horaires, etc. Gardez à l’esprit qu’une milliseconde spécifique est affichée sous différentes heures dans différents fuseaux horaires ; normalement, vous voulez en afficher un dans le fuseau horaire “local”, mais les méthodes de formatage doivent tenir compte du fait que vous voudrez peut-être l’afficher pour un autre.

Sachez également que les horloges utilisées par les JVM n’ont généralement pas une précision à la milliseconde ; l’horloge peut ne “cocher” que toutes les 10 millisecondes, et par conséquent, si vous chronométrez les choses, vous ne pouvez pas compter sur la mesure précise des choses à ce niveau.

Déclaration d’importation

import java.util.Date;

La classe Date peut être importée du package java.util.

Mise en garde

Les instances Date sont mutables, donc leur utilisation peut rendre difficile l’écriture code thread-safe ou peut accidentellement fournir un accès en écriture à l’état interne. Par exemple, dans la classe ci-dessous, la méthode getDate() permet à l’appelant de modifier la date de la transaction :

public final class Transaction {
  private final Date date;

  public Date getTransactionDate() {
    return date;
  }
}

La solution consiste soit à renvoyer une copie du champ date ou à utiliser les nouvelles API dans java.time introduites dans Java 8.

La plupart des méthodes de construction de la classe Date sont obsolètes et ne doivent pas être utilisées. Dans presque tous les cas, il est conseillé d’utiliser la classe Calendar pour les opérations de date.

Java 8

Java 8 introduit une nouvelle API d’heure et de date dans le package java.time, y compris [LocalDate][1] et [LocalTime][2]. Les classes du package java.time fournissent une API révisée plus facile à utiliser. Si vous écrivez sur Java 8, il est fortement recommandé d’utiliser cette nouvelle API. Voir https://www.wikiod.com/fr/java/dates-et-heure-javatime .

[1] : http://docs.oracle.com/javase/8/docs/api/java/time/LocalDate.html [2] : http://docs.oracle.com/javase/8/docs/api/java/time/LocalTime.html

Une sortie de date de base #

En utilisant le code suivant avec la chaîne de format yyyy/MM/dd hh:mm.ss, nous recevrons la sortie suivante

2016/04/19 11:45.36

// define the format to use
String formatString = "yyyy/MM/dd hh:mm.ss";

// get a current date object
Date date = Calendar.getInstance().getTime();

// create the formatter
SimpleDateFormat simpleDateFormat = new SimpleDateFormat(formatString);

// format the date
String formattedDate = simpleDateFormat.format(date);

// print it
System.out.println(formattedDate);


// single-line version of all above code
System.out.println(new SimpleDateFormat("yyyy/MM/dd hh:mm.ss").format(Calendar.getInstance().getTime()));

Convertir java.util.Date en java.sql.Date #

La conversion de java.util.Date en java.sql.Date est généralement nécessaire lorsqu’un objet Date doit être écrit dans une base de données.

java.sql.Date est un wrapper autour de la valeur de la milliseconde et est utilisé par JDBC pour identifier un type SQL DATE

Dans l’exemple ci-dessous, nous utilisons le constructeur java.util.Date(), qui crée un objet Date et l’initialise pour représenter le temps à la milliseconde près. Cette date est utilisée dans la méthode convert(java.util.Date utilDate) pour retourner un objet java.sql.Date

Exemple

public class UtilToSqlConversion {
    
    public static void main(String args[])
    {
        java.util.Date utilDate = new java.util.Date();
        System.out.println("java.util.Date is : " + utilDate);
        java.sql.Date sqlDate = convert(utilDate);
        System.out.println("java.sql.Date is : " + sqlDate);
        DateFormat df = new SimpleDateFormat("dd/MM/YYYY - hh:mm:ss");
        System.out.println("dateFormated date is : " + df.format(utilDate));
    }

    private static java.sql.Date convert(java.util.Date uDate) {
        java.sql.Date sDate = new java.sql.Date(uDate.getTime());
        return sDate;
    }

}

Production

java.util.Date is : Fri Jul 22 14:40:35 IST 2016
java.sql.Date is : 2016-07-22
dateFormated date is : 22/07/2016 - 02:40:35

java.util.Date contient à la fois des informations de date et d’heure, tandis que java.sql.Date ne contient que des informations de date

Objets Java 8 LocalDate et LocalDateTime #

Les objets Date et LocalDate ** ne peuvent pas ** être exactement convertis entre eux car un objet Date représente à la fois un jour et une heure spécifiques, tandis qu’un objet LocalDate ne contient pas d’informations d’heure ou de fuseau horaire. Cependant, il peut être utile de convertir entre les deux si vous ne vous souciez que des informations de date réelles et non des informations d’heure.

Crée une date locale

// Create a default date
LocalDate lDate = LocalDate.now();

// Creates a date from values
lDate = LocalDate.of(2017, 12, 15);
    
// create a date from string
lDate = LocalDate.parse("2017-12-15");

// creates a date from zone
LocalDate.now(ZoneId.systemDefault());

Crée un LocalDateTime

// Create a default date time
LocalDateTime lDateTime = LocalDateTime.now();

// Creates a date time from values
lDateTime = LocalDateTime.of(2017, 12, 15, 11, 30);

// create a date time from string
lDateTime = LocalDateTime.parse("2017-12-05T11:30:30");

// create a date time from zone 
LocalDateTime.now(ZoneId.systemDefault());

LocalDate to Date et vice-versa

Date date = Date.from(Instant.now());
ZoneId defaultZoneId = ZoneId.systemDefault();

// Date to LocalDate
LocalDate localDate = date.toInstant().atZone(defaultZoneId).toLocalDate();

// LocalDate to Date
Date.from(localDate.atStartOfDay(defaultZoneId).toInstant());

LocalDateTime à ce jour et vice-versa

Date date = Date.from(Instant.now());
ZoneId defaultZoneId = ZoneId.systemDefault();

// Date to LocalDateTime
LocalDateTime localDateTime =  date.toInstant().atZone(defaultZoneId).toLocalDateTime();

// LocalDateTime to Date
Date out = Date.from(localDateTime.atZone(defaultZoneId).toInstant());

Conversion de la date en un certain format de chaîne #

format() de la classe SimpleDateFormat permet de convertir un objet Date en un certain format d’objet String en utilisant la chaîne de motif fournie.

Date today = new Date();
    
SimpleDateFormat dateFormat = new SimpleDateFormat("dd-MMM-yy"); //pattern is specified here
System.out.println(dateFormat.format(today)); //25-Feb-16

Les modèles peuvent être appliqués à nouveau en utilisant applyPattern()

dateFormat.applyPattern("dd-MM-yyyy");
System.out.println(dateFormat.format(today)); //25-02-2016

dateFormat.applyPattern("dd-MM-yyyy HH:mm:ss E");
System.out.println(dateFormat.format(today)); //25-02-2016 06:14:33 Thu

Remarque : Ici, mm (lettre minuscule m) désigne les minutes et MM (M majuscule) désigne le mois. Faites très attention lorsque vous formatez les années : le « Y » majuscule (« Y ») indique la « semaine de l’année » tandis que le « y » minuscule (y) indique l’année.

Création d’une date spécifique #

Bien que la classe Java Date ait plusieurs constructeurs, vous remarquerez que la plupart sont obsolètes. Le seul moyen acceptable de créer directement une instance de Date consiste à utiliser le constructeur vide ou à transmettre un long (nombre de millisecondes depuis l’heure de base standard). Ni l’un ni l’autre ne sont pratiques, sauf si vous recherchez la date actuelle ou si vous avez déjà une autre instance Date en main.

Pour créer une nouvelle date, vous aurez besoin d’une instance Calendar. À partir de là, vous pouvez définir l’instance de Calendar sur la date dont vous avez besoin.

Calendar c = Calendar.getInstance();

Cela renvoie une nouvelle instance de Calendar définie sur l’heure actuelle. Le calendrier a de nombreuses méthodes pour muter sa date et son heure ou pour le définir purement et simplement. Dans ce cas, nous le fixerons à une date précise.

c.set(1974, 6, 2, 8, 0, 0);
Date d = c.getTime();

La méthode getTime renvoie l’instance Date dont nous avons besoin. Gardez à l’esprit que les méthodes de set Calendar ne définissent qu’un ou plusieurs champs, elles ne les définissent pas tous. Autrement dit, si vous définissez l’année, les autres champs restent inchangés.

PIÈGE

Dans de nombreux cas, cet extrait de code remplit son objectif, mais gardez à l’esprit que deux parties importantes de la date/heure ne sont pas définies.

  • les paramètres (1974, 6, 2, 8, 0, 0) sont interprétés dans le fuseau horaire par défaut, défini ailleurs,
  • les millisecondes ne sont pas mises à zéro, mais renseignées à partir de l’horloge système au moment de la création de l’instance Calendar.

Création d’objets Date #

Date date = new Date();
System.out.println(date); // Thu Feb 25 05:03:59 IST 2016

Ici, cet objet Date contient la date et l’heure actuelles de création de cet objet.

Calendar calendar = Calendar.getInstance();
calendar.set(90, Calendar.DECEMBER, 11);
Date myBirthDate = calendar.getTime();
System.out.println(myBirthDate); // Mon Dec 31 00:00:00 IST 1990

Les objets Date sont mieux créés via une instance Calendar car l’utilisation des constructeurs de données est obsolète et déconseillée. Pour ce faire, nous devons obtenir une instance de la classe Calendar à partir de la méthode d’usine. Ensuite, nous pouvons définir l’année, le mois et le jour du mois en utilisant des nombres ou, dans le cas des mois, des constantes fournies par la classe Calendar pour améliorer la lisibilité et réduire les erreurs.

calendar.set(90, Calendar.DECEMBER, 11, 8, 32, 35);
Date myBirthDatenTime = calendar.getTime();
System.out.println(myBirthDatenTime); // Mon Dec 31 08:32:35 IST 1990

En plus de la date, nous pouvons également passer le temps dans l’ordre des heures, des minutes et des secondes.

Comparaison de dates avant Java 8 #

Avant Java 8, les dates pouvaient être comparées à l’aide des classes [java.util.Calendar][1] et [java.util.Date][2]. La classe Date propose 4 méthodes pour comparer les dates :

  • [après (date à laquelle)] [3]
  • [avant(Date quand)][4]
  • [comparerÀ(Date une autreDate)][5]
  • [est égal à (objet obj)] [6]

Les méthodes after, before, compareTo et equals comparent les valeurs renvoyées par la méthode [getTime()][7] pour chaque date.

La méthode compareTo renvoie un entier positif.

  • Valeur supérieure à 0 : lorsque la Date est postérieure à l’argument Date
  • Valeur supérieure à 0 : lorsque la Date est antérieure à l’argument Date
  • Valeur égale à 0 : lorsque la Date est égale à l’argument Date

Les résultats equals peuvent être surprenants, comme le montre l’exemple, car les valeurs, comme les millisecondes, ne sont pas initialisées avec la même valeur si elles ne sont pas explicitement données.

Depuis Java 8 #

Avec Java 8, un nouvel objet pour travailler avec Date est disponible [java.time.LocalDate][8]. LocalDate implémente [ChronoLocalDate] [9], la représentation abstraite d’une date où la chronologie, ou le système de calendrier, est enfichable.

Pour avoir la précision de la date et de l’heure, l’objet [java.time.LocalDateTime][10] doit être utilisé. LocalDate et LocalDateTime utilisent le même nom de méthode pour la comparaison.

La comparaison de dates à l’aide d’un LocalDate est différente de l’utilisation de ChronoLocalDate car la chronologie ou le système de calendrier ne sont pas pris en compte le premier.

Comme la plupart des applications doivent utiliser LocalDate, ChronoLocalDate n’est pas inclus dans les exemples. Pour en savoir plus [ici][9].

La plupart des applications doivent déclarer les signatures de méthode, les champs et les variables comme LocalDate, pas cette interface [ChronoLocalDate].

LocalDate a 5 méthodes pour comparer les dates :

  • [isAfter(ChronoLocalDate autre)][11]
  • [isBefore(ChronoLocalDate autre)][12]
  • [isEqual(ChronoLocalDate autre)][13]
  • [comparerÀ(ChronoLocalDate autre)][14]
  • [est égal à (objet obj)] [15]

Dans le cas du paramètre LocalDate, isAfter, isBefore, isEqual, equals et compareTo utilisent maintenant cette méthode :

int compareTo0(LocalDate otherDate) {
    int cmp = (year - otherDate.year);
    if (cmp == 0) {
        cmp = (month - otherDate.month);
        if (cmp == 0) {
            cmp = (day - otherDate.day);
        }
    }
    return cmp;
}

La méthode equals vérifie si la référence du paramètre est d’abord égale à la date alors que isEqual appelle directement compareTo0.

Dans le cas d’une autre instance de classe de ChronoLocalDate, les dates sont comparées à l’aide de Epoch Day. Le comptage Epoch Day est un simple comptage incrémentiel de jours où le jour 0 est 1970-01-01 (ISO).

Conversion d’une chaîne en date #

parse() de la classe SimpleDateFormat aide à convertir un modèle String en un objet Date.

DateFormat dateFormat = DateFormat.getDateInstance(DateFormat.SHORT, Locale.US);
String dateStr = "02/25/2016"; // input String
Date date = dateFormat.parse(dateStr);
System.out.println(date.getYear()); // 116

Il existe 4 styles différents pour le format de texte, SHORT, MEDIUM (c’est la valeur par défaut), LONG et FULL, qui dépendent tous des paramètres régionaux. Si aucun paramètre régional n’est spécifié, le paramètre régional par défaut du système est utilisé.

Style Locale.US Locale.France
COURT 30/06/09 30/06/09
MOYEN 30 juin 2009 30 juin 2009
LONGUE 30 juin 2009 30 juin 2009
COMPLET mardi 30 juin 2009 mardi 30 juin 2009

Fuseaux horaires et java.util.Date #

Un objet java.util.Date n’a pas de concept de fuseau horaire.

  • Il n’y a aucun moyen de définir un fuseau horaire pour une date
  • Il n’y a aucun moyen de changer le fuseau horaire d’un objet Date
  • Un objet Date créé avec le constructeur par défaut new Date() sera initialisé avec l’heure actuelle dans le fuseau horaire par défaut du système

Cependant, il est possible d’afficher la date représentée par le point dans le temps décrit par l’objet Date dans un fuseau horaire différent en utilisant par ex. java.text.SimpleDateFormat :

Date date = new Date();
//print default time zone
System.out.println(TimeZone.getDefault().getDisplayName());
SimpleDateFormat sdf = new SimpleDateFormat("yyyy-MM-dd HH:mm:ss"); //note: time zone not in format!
//print date in the original time zone
System.out.println(sdf.format(date));
//current time in London
sdf.setTimeZone(TimeZone.getTimeZone("Europe/London"));
System.out.println(sdf.format(date));

Production:

Central European Time
2016-07-21 22:50:56
2016-07-21 21:50:56

Heure locale #

Pour utiliser uniquement la partie heure d’une date, utilisez LocalTime. Vous pouvez instancier un objet LocalTime de plusieurs manières

  1. Heure LocalTime = LocalTime.now();
  2. time = LocalTime.MIDNIGHT;
  3. time = LocalTime.NOON;
  4. time = LocalTime.of(12, 12, 45);

LocalTime a également une méthode toString intégrée qui affiche très bien le format.

System.out.println(time);

vous pouvez également obtenir, ajouter et soustraire des heures, des minutes, des secondes et des nanosecondes de l’objet LocalTime, c’est-à-dire

time.plusMinutes(1);
time.getMinutes();
time.minusMinutes(1);

Vous pouvez le transformer en objet Date avec le code suivant :

LocalTime lTime = LocalTime.now();
Instant instant = lTime.atDate(LocalDate.of(A_YEAR, A_MONTH, A_DAY)).
        atZone(ZoneId.systemDefault()).toInstant();
Date time = Date.from(instant);

cette classe fonctionne très bien dans une classe timer pour simuler un réveil.
