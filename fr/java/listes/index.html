Une liste est une collection ordonnée de valeurs. En Java, les listes font partie du [Java Collections Framework][1]. Les listes implémentent l’interface [java.util.List][2], qui étend [java.util.Collection][3].

[1] : http://docs.oracle.com/javase/8/docs/technotes/guides/collections/index.html [2] : http://docs.oracle.com/javase/8/docs/api/java/util/List.html [3] : http://docs.oracle.com/javase/8/docs/api/java/util/Collection.html

Syntaxe #

  • ls.add(élément E); //Ajoute un élément
  • ls.remove(élément E); // Supprime un élément
  • for(E element : ls){} //Itère sur chaque élément
  • ls.toArray(new String[ls.length]); //Convertit une liste de chaînes en un tableau de chaînes
  • ls.get(int index); // Renvoie l’élément à l’index spécifié.
  • ls.set(index int, élément E); //Remplace l’élément à une position spécifiée .
  • ls.isEmpty(); // Renvoie vrai si le tableau ne contient aucun élément, sinon il renvoie faux.
  • ls.indexOf(Objet o); //Renvoie l’indice du premier emplacement de l’élément spécifié o, ou, s’il n’est pas présent, renvoie -1.
  • ls.lastIndexOf(Objet o); //Renvoie l’indice du dernier emplacement de l’élément spécifié o, ou, s’il n’est pas présent, renvoie -1.
  • ls.taille(); //Renvoie le nombre d’éléments dans la liste.

Une [liste][1] est un objet qui stocke une collection ordonnée de valeurs. “Ordonné” signifie que les valeurs sont stockées dans un ordre particulier : un élément vient en premier, un deuxième, et ainsi de suite. Les valeurs individuelles sont communément appelées “éléments”. Les listes Java fournissent généralement ces fonctionnalités :

  • Les listes peuvent contenir zéro ou plusieurs éléments.
  • Les listes peuvent contenir des valeurs en double. En d’autres termes, un élément peut être inséré plusieurs fois dans une liste.
  • Les listes stockent leurs éléments dans un ordre particulier, ce qui signifie qu’un élément vient en premier, un autre vient ensuite, et ainsi de suite.
  • Chaque élément a un index indiquant sa position dans la liste. Le premier élément a l’indice 0, le suivant a l’indice 1, etc.
  • Les listes permettent d’insérer des éléments au début, à la fin ou à n’importe quel index de la liste.
  • Tester si une liste contient une valeur particulière signifie généralement examiner chaque élément de la liste. Cela signifie que le temps pour effectuer cette vérification est [O(n)][2], proportionnel à la taille de la liste.

L’ajout d’une valeur à une liste à un point autre que la fin déplacera tous les éléments suivants “vers le bas” ou “vers la droite”. En d’autres termes, l’ajout d’un élément à l’index n déplace l’élément qui était à l’index n vers l’index n+1, et ainsi de suite. Par exemple:

List<String> list = new ArrayList<>();
list.add("world");
System.out.println(list.indexOf("world"));      // Prints "0"
// Inserting a new value at index 0 moves "world" to index 1
list.add(0, "Hello");
System.out.println(list.indexOf("world"));      // Prints "1"
System.out.println(list.indexOf("Hello"));      // Prints "0"

[1] : http://docs.oracle.com/javase/8/docs/api/java/util/List.html [2] : https://en.wikipedia.org/wiki/Big_O_notation

Trier une liste générique #

La classe Collections propose deux méthodes statiques standard pour trier une liste :

  • sort(List<T> list) applicable aux listes où T s'étend Comparable< ? super T>, et
  • sort(List<T> list, Comparator<? super T> c) applicable aux listes de tout type.

L’application de la première nécessite de modifier la classe des éléments de liste triés, ce qui n’est pas toujours possible. Cela peut également être indésirable car bien qu’il fournisse le tri par défaut, d’autres ordres de tri peuvent être requis dans différentes circonstances, ou le tri n’est qu’une tâche ponctuelle.

Considérons que nous avons une tâche de tri des objets qui sont des instances de la classe suivante :

public class User {
    public final Long id;
    public final String username;

    public User(Long id, String username) {
        this.id = id;
        this.username = username;
    }

    @Override
    public String toString() {
        return String.format("%s:%d", username, id);
    }
}

Afin d’utiliser Collections.sort(List<User> list) nous devons modifier la classe User pour implémenter l’interface Comparable. Par exemple

public class User implements Comparable<User> {
    public final Long id;
    public final String username;

    public User(Long id, String username) {
        this.id = id;
        this.username = username;
    }

    @Override
    public String toString() {
        return String.format("%s:%d", username, id);
    }

    @Override
    /** The natural ordering for 'User' objects is by the 'id' field. */
    public int compareTo(User o) {
        return id.compareTo(o.id);
    }
}

(A part: de nombreuses classes Java standard telles que String, Long, Integer implémentent l’interface Comparable. Cela rend les listes de ces éléments triables par défaut et simplifie l’implémentation de compare ou compareTo dans d’autres Des classes.)

Avec la modification ci-dessus, nous pouvons facilement trier une liste d’objets “Utilisateur” en fonction des classes ordre naturel. (Dans ce cas, nous avons défini qu’il s’agit d’un classement basé sur les valeurs id). Par exemple:

List<User> users = Lists.newArrayList(
    new User(33L, "A"),
    new User(25L, "B"),
    new User(28L, ""));
Collections.sort(users);

System.out.print(users);
// [B:25, C:28, A:33]

Cependant, supposons que nous voulions trier les objets User par name plutôt que par id. Sinon, supposons que nous n’ayons pas pu modifier la classe pour qu’elle implémente Comparable.

C’est là que la méthode sort avec l’argument Comparator est utile :

Collections.sort(users, new Comparator<User>() {
    @Override
    /* Order two 'User' objects based on their names. */
    public int compare(User left, User right) {
        return left.username.compareTo(right.username);
    }
});
System.out.print(users);
// [A:33, B:25, C:28]

Dans Java 8, vous pouvez utiliser un lambda au lieu d’une classe anonyme. Ce dernier se réduit à un one-liner :

Collections.sort(users, (l, r) -> l.username.compareTo(r.username));

De plus, Java 8 ajoute une méthode sort par défaut sur l’interface List, ce qui simplifie encore plus le tri.

users.sort((l, r) -> l.username.compareTo(r.username))

Convertir une liste d’entiers en une liste de chaînes #

List<Integer> nums = Arrays.asList(1, 2, 3);
List<String> strings = nums.stream()
    .map(Object::toString)
    .collect(Collectors.toList());

C’est-à-dire:

  1. Créez un flux à partir de la liste
  2. Mappez chaque élément en utilisant Object :: toString
  3. Collectez les valeurs String dans une List en utilisant Collectors.toList()

L’interface [List][1] est implémentée par différentes classes. Chacun d’eux a sa propre façon de le mettre en œuvre avec différentes stratégies et d’offrir différents avantages et inconvénients.

Liste des tableaux #

public class ArrayList<E>
extends AbstractList<E>
implements List<E>, RandomAccess, Cloneable, Serializable

[ArrayList][2] est une implémentation de tableau redimensionnable de l’interface List. En stockant la liste dans un tableau, ArrayList fournit des méthodes (en plus des méthodes implémentant l’interface List) pour manipuler la taille du tableau.

** Initialiser ArrayList of Integer avec taille 100 **

List<Integer> myList = new ArrayList<Integer>(100); // Constructs an empty list with the specified initial capacity.

- AVANTAGES:

Les opérations size, isEmpty, get, set, iterator et listIterator s’exécutent en temps constant. Ainsi, obtenir et définir chaque élément de la liste a le même coût en temps :

int e1 = myList.get(0);  //   \
int e2 = myList.get(10); //    | => All the same constant cost => O(1)
myList.set(2,10);        //   /

- LES INCONVÉNIENTS:

Être implémenté avec un tableau (structure statique) ajouter des éléments sur la taille du tableau a un coût important en raison du fait qu’une nouvelle allocation doit être effectuée pour tout le tableau. Cependant, à partir de [documentation][2] :

L’opération d’addition s’exécute en temps constant amorti, c’est-à-dire en ajoutant n les éléments nécessitent un temps O(n)

La suppression d’un élément nécessite un temps O(n).

Trouver des éléments communs entre 2 listes #

Supposons que vous ayez deux listes : A et B, et que vous deviez trouver les éléments qui existent dans les deux listes.

Vous pouvez le faire en appelant simplement la méthode List.retainAll().

Exemple:

public static void main(String[] args) {
    List<Integer> numbersA = new ArrayList<>();
    List<Integer> numbersB = new ArrayList<>();
    numbersA.addAll(Arrays.asList(new Integer[] { 1, 3, 4, 7, 5, 2 }));
    numbersB.addAll(Arrays.asList(new Integer[] { 13, 32, 533, 3, 4, 2 }));

    System.out.println("A: " + numbersA);
    System.out.println("B: " + numbersB);
    List<Integer> numbersC = new ArrayList<>();
    numbersC.addAll(numbersA);
    numbersC.retainAll(numbersB);

    System.out.println("List A : " + numbersA);
    System.out.println("List B : " + numbersB);
    System.out.println("Common elements between A and B: " + numbersC);

}

Créer une liste #

Donner un type à votre liste

Pour créer une liste, vous avez besoin d’un type (n’importe quelle classe, par exemple String). C’est le type de votre Liste. La Liste ne stockera que les objets du type spécifié. Par exemple:

List<String> strings;

Peut stocker "string1", "hello world!", "goodbye", etc., mais il ne peut pas stocker 9.2, cependant :

List<Double> doubles;

Peut stocker 9.2, mais pas "hello world!".

Initialisation de votre liste

Si vous essayez d’ajouter quelque chose aux listes ci-dessus, vous obtiendrez une NullPointerException, car strings et doubles sont tous deux égaux à null !

Il existe deux manières d’initialiser une liste :

Option 1 : Utiliser une classe qui implémente List

List est une interface, ce qui signifie qu’elle n’a pas de constructeur, mais plutôt des méthodes qu’une classe doit remplacer. ArrayList est la List la plus couramment utilisée, bien que LinkedList soit également courante. Nous initialisons donc notre liste comme ceci :

List<String> strings = new ArrayList<String>();

ou

List<String> strings = new LinkedList<String>();

À partir de Java SE 7, vous pouvez utiliser un [opérateur diamant][1] :

List<String> strings = new ArrayList<>();

ou

List<String> strings = new LinkedList<>();

Option 2 : Utiliser la classe Collections

La classe Collections fournit deux méthodes utiles pour créer des listes sans variable List :

  • emptyList() : retourne une liste vide.
  • singletonList(T) : crée une liste de type T et ajoute l’élément spécifié.

Et une méthode qui utilise une List existante pour remplir les données :

  • addAll(L, T...) : ajoute tous les éléments spécifiés à la liste passée en premier paramètre.

Exemples:

import java.util.List;
import java.util.Collections;

List<Integer> l = Collections.emptyList();
List<Integer> l1 = Collections.singletonList(42);
Collections.addAll(l1, 1, 2, 3);

Opérations d’accès positionnel #

L’API List propose huit méthodes pour les opérations d’accès positionnel :

  • ajouter (type T)
  • ajouter (index int, type T)
  • supprimer(Objet o)
  • supprimer(index entier)
  • obtenir (index entier)
  • set(index int, élément E)
  • int indexOf(Objet o)
  • int lastIndexOf(Objet o)

Donc, si nous avons une liste :

List<String> strings = new ArrayList<String>();

Et nous voulions ajouter les chaînes “Hello world!” et “Au revoir le monde!” à cela, nous le ferions comme tel:

strings.add("Hello world!");
strings.add("Goodbye world!");

Et notre liste contiendrait les deux éléments. Maintenant, disons que nous voulions ajouter “Démarrage du programme !” au devant de la liste. On ferait ça comme ça :

strings.add(0, "Program starting!");

REMARQUE : Le premier élément est 0.

Maintenant, si nous voulions supprimer le “Au revoir le monde!” ligne, nous pourrions le faire comme ceci:

strings.remove("Goodbye world!");

Et si nous voulions supprimer la première ligne (qui dans ce cas serait “Démarrage du programme !”), nous pourrions le faire comme ceci :

strings.remove(0);

Noter:

  1. L’ajout et la suppression d’éléments de liste modifient la liste, ce qui peut entraîner une ConcurrentModificationException si la liste est itérée simultanément.

  2. L’ajout et la suppression d’éléments peuvent être O(1) ou O(N) selon la classe de liste, la méthode utilisée et si vous ajoutez / supprimez un élément au début, à la fin ou dans le milieu de la liste.

Afin de récupérer un élément de la liste à une position spécifiée, vous pouvez utiliser la méthode E get(int index); de l’API List. Par exemple:

strings.get(0);

renverra le premier élément de la liste.

Vous pouvez remplacer n’importe quel élément à une position spécifiée en utilisant set(int index, E element);. Par exemple:

strings.set(0,"This is a replacement");

Cela définira la chaîne “Ceci est un remplacement” comme premier élément de la liste.

Noter: The set method will overwrite the element at the position 0. It will not add the new String at the position 0 and push the old one to the position 1.

Le int indexOf(Object o); renvoie la position de la première occurrence de l’objet passé en argument. S’il n’y a aucune occurrence de l’objet dans la liste, la valeur -1 est renvoyée. Dans la continuité de l’exemple précédent si vous appelez :

strings.indexOf("This is a replacement")

le 0 devrait être renvoyé lorsque nous définissons la chaîne “Ceci est un remplacement” à la position 0 de notre liste. Dans le cas où il y a plus d’une occurrence dans la liste lorsque int indexOf(Object o); est appelé, alors, comme mentionné, l’index de la première occurrence sera renvoyé. En appelant int lastIndexOf(Object o) vous pouvez récupérer l’index de la dernière occurrence dans la liste. Donc, si nous ajoutons un autre “Ceci est un remplacement”:

strings.add("This is a replacement");
strings.lastIndexOf("This is a replacement");

Cette fois, le 1 sera renvoyé et non le 0 ;

Itération sur les éléments d’une liste #

Pour l’exemple, disons que nous avons une List de type String qui contient quatre éléments : “hello, “, “how”, “are”, “you?”

La meilleure façon d’itérer sur chaque élément consiste à utiliser une boucle for-each :

public void printEachElement(List<String> list){
    for(String s : list){
        System.out.println(s);
    }
}

Ce qui imprimerait :

hello,
how
are
you?

Pour les imprimer tous sur la même ligne, vous pouvez utiliser un StringBuilder :

public void printAsLine(List<String> list){
    StringBuilder builder = new StringBuilder();
    for(String s : list){
        builder.append(s);
    }
    System.out.println(builder.toString());
}

Imprimera :

hello, how are you?

Alternativement, vous pouvez utiliser l’indexation des éléments (comme décrit dans https://www.wikiod.com/fr/java/listes) pour itérer une liste. Attention : cette approche est inefficace pour les listes chaînées.

cela s’imprimera

Création, ajout et suppression d’éléments d’une ArrayList #

ArrayList est l’une des structures de données intégrées à Java. C’est un tableau dynamique (où la taille de la structure de données n’a pas besoin d’être déclarée en premier) pour stocker des éléments (Objets).

Il étend la classe AbstractList et implémente l’interface List. Un ArrayList peut contenir des éléments en double où il maintient l’ordre d’insertion. Il convient de noter que la classe ArrayList n’est pas synchronisée, il faut donc faire attention lors de la gestion de la concurrence avec ArrayList. ArrayList permet un accès aléatoire car le tableau fonctionne sur la base de l’index. La manipulation est lente dans ArrayList en raison du décalage qui se produit souvent lorsqu’un élément est supprimé de la liste de tableaux.

Une ArrayList peut être créée comme suit :

List<T> myArrayList = new ArrayList<>();

Où ‘T’ ( https://www.wikiod.com/fr/java/generiques ) est le type qui sera stocké dans ‘ArrayList’.

Le type de ArrayList peut être n’importe quel objet. Le type ne peut pas être un type primitif (utilisez plutôt leurs classes wrapper).

Pour ajouter un élément à la ArrayList, utilisez la méthode add() :

myArrayList.add(element);

Ou pour ajouter un élément à un certain index :

myArrayList.add(index, element); //index of the element should be an int (starting from 0)

Pour supprimer un élément de la ArrayList, utilisez la méthode remove() :

myArrayList.remove(element);

Ou pour supprimer un élément d’un certain index :

myArrayList.remove(index); //index of the element should be an int (starting from 0)

Remplacement sur place d’un élément de liste #

Cet exemple concerne le remplacement d’un élément List tout en s’assurant que l’élément de remplacement est à la même position que l’élément remplacé.

Cela peut être fait en utilisant ces méthodes :

  • set(index int, type T)
  • int indexOf(type T)

Considérez une ArrayList contenant les éléments “Démarrage du programme!”, “Hello world!” et “Au revoir le monde!”

List<String> strings = new ArrayList<String>();
strings.add("Program starting!");
strings.add("Hello world!");
strings.add("Goodbye world!");

Si nous connaissons l’index de l’élément que nous voulons remplacer, nous pouvons simplement utiliser set comme suit :

strings.set(1, "Hi world");

Si nous ne connaissons pas l’index, nous pouvons d’abord le rechercher. Par exemple:

int pos = strings.indexOf("Goodbye world!");
if (pos >= 0) {
    strings.set(pos, "Goodbye cruel world!");
}

Remarques:

  1. L’opération set ne causera pas de ConcurrentModificationException.
  2. L’opération set est rapide ( O(1) ) pour ArrayList mais lente ( O(N) ) pour une LinkedList.
  3. Une recherche indexOf sur une ArrayList ou LinkedList est lente ( O(N) ).

Rendre une liste non modifiable #

La classe Collections permet de rendre une liste non modifiable :

List<String> ls = new ArrayList<String>();
List<String> unmodifiableList = Collections.unmodifiableList(ls);

Si vous voulez une liste non modifiable avec un élément, vous pouvez utiliser :

List<String> unmodifiableList = Collections.singletonList("Only string in the list");

Déplacer des objets dans la liste #

La classe Collections vous permet de déplacer des objets dans la liste à l’aide de différentes méthodes (ls est la liste):

Inverser une liste :

Collections.reverse(ls);

Positions de rotation des éléments dans une liste

La méthode rotate nécessite un argument entier. C’est le nombre de points pour le déplacer le long de la ligne. Un exemple de ceci est ci-dessous:

List<String> ls = new ArrayList<String>();
ls.add(" how");
ls.add(" are");
ls.add(" you?");
ls.add("hello,");
Collections.rotate(ls, 1);

for(String line : ls) System.out.print(line);
System.out.println();

Cela affichera “bonjour, comment allez-vous ?”

** Mélanger les éléments dans une liste **

En utilisant la même liste ci-dessus, nous pouvons mélanger les éléments d’une liste :

Collections.shuffle(ls);

Nous pouvons également lui donner un objet java.util.Random qu’il utilise pour placer aléatoirement des objets dans des endroits :

Random random = new Random(12); 
Collections.shuffle(ls, random);
