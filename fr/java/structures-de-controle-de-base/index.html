Toutes les structures de contrôle, sauf indication contraire, utilisent des instructions de bloc. Ceux-ci sont indiqués par des accolades {}.

Cela diffère des instructions normales, qui ne nécessitent pas d’accolades, mais aussi viennent avec une mise en garde stricte en ce sens que seule la ligne suivant immédiatement la déclaration précédente serait prise en compte.

Ainsi, il est parfaitement valide d’écrire l’une de ces structures de contrôle sans accolades, tant qu’une seule * instruction * suit le début, mais c’est *** fortement déconseillé ***, car cela peut conduire à des implémentations boguées, ou code cassé.

Exemple:

// valid, but discouraged
Scanner scan = new Scanner(System.in);
int val = scan.nextInt();
if(val % 2 == 0)
    System.out.println("Val was even!");


// invalid; will not compile
// note the misleading indentation here
for(int i = 0; i < 10; i++)
    System.out.println(i);
    System.out.println("i is currently: " + i);

faire… tant que Boucle #

La boucle do...while diffère des autres boucles en ce qu’elle est garantie de s’exécuter au moins une fois. Elle est également appelée structure de “boucle post-test” car l’instruction conditionnelle est exécutée après le corps de la boucle principale.

int i = 0;
do {
    i++;
    System.out.println(i);
} while (i < 100); // Condition gets checked AFTER the content of the loop executes.

Dans cet exemple, la boucle s’exécutera jusqu’à ce que le nombre “100” soit imprimé (même si la condition est “i < 100” et non “i <= 100”), car la condition de la boucle est évaluée après l’exécution de la boucle.

Avec la garantie d’au moins une exécution, il est possible de déclarer des variables en dehors de la boucle et de les initialiser à l’intérieur.

String theWord;
Scanner scan = new Scanner(System.in);
do {
    theWord = scan.nextLine();
} while (!theWord.equals("Bird"));

System.out.println(theWord);

Dans ce contexte, theWord est défini en dehors de la boucle, mais comme il est garanti d’avoir une valeur basée sur son flux naturel, theWord sera initialisé.

Pour chaque #

Avec Java 5 et versions ultérieures, on peut utiliser des boucles for-each, également appelées boucles for améliorées :

List chaînes = new ArrayList();
        
strings.add("Ceci");
strings.add("est");
strings.add("une boucle for-each");
        
pour (Chaîne chaîne : chaînes) {
    System.out.println(string);
}

Pour chaque boucle peut être utilisée pour itérer sur [Arrays][1] et les implémentations de l’interface [Iterable][2], cette dernière inclut des classes [Collections][3], telles que List ou Set.

La variable de boucle peut être de n’importe quel type pouvant être affecté à partir du type source.

La variable de boucle pour une boucle for améliorée pour Iterable<T> ou T[] peut être de type S, si

  • T prolonge S
  • T et S sont des types primitifs et peuvent être assignés sans transtypage
  • S est un type primitif et T peut être converti en un type attribuable à S après la conversion unboxing.
  • ‘T’ est un type primitif et peut être converti en ‘S’ par conversion automatique.

Exemples:

T elements = ...
for (S s : elements) {
}
T S Compile
int[] longue oui
long[] entier non
Itérable<Octet> longue oui
Itérable<String> CharSéquence oui
Itérable<CharSequence> Chaîne non
int[] longue non
int[] Entier oui

[1] : https://www.wikiod.com/fr/java/tableaux [2] : https://docs.oracle.com/javase/8/docs/api/java/lang/Iterable.html [3] : https://www.wikiod.com/fr/java/collections

Instruction de commutation #

L’instruction switch est l’instruction de branche multidirectionnelle de Java. Il est utilisé pour remplacer les longues chaînes if-else if-else et les rendre plus lisibles. Cependant, contrairement aux instructions if, on ne peut pas utiliser les inégalités ; chaque valeur doit être concrètement définie.

L’instruction switch comporte trois composants essentiels :

  • case : il s’agit de la valeur évaluée pour l’équivalence avec l’argument de l’instruction switch.
  • default : il s’agit d’une expression attrape-tout facultative, si aucune des déclarations case n’est évaluée comme true.
  • Achèvement brutal de l’instruction case ; généralement break : ceci est nécessaire pour empêcher l’évaluation indésirable d’autres instructions case.

À l’exception de continue, il est possible d’utiliser n’importe quelle instruction qui entraînerait [l’achèvement brutal d’une instruction](https://docs.oracle.com/javase/specs/jls/se7/html/jls-14 .html#jls-14.1). Ceci comprend:

  • “casser”
  • ‘retourner’
  • “lancer”

Dans l’exemple ci-dessous, une instruction switch typique est écrite avec quatre cas possibles, y compris default.

Scanner scan = new Scanner(System.in);
int i = scan.nextInt();
switch (i) {
    case 0:
        System.out.println("i is zero");
        break;
    case 1:
        System.out.println("i is one");
        break;
    case 2:
        System.out.println("i is two");
        break;
    default:
        System.out.println("i is less than zero or greater than two");
}

En omettant “break” ou toute déclaration qui entraînerait une fin brutale, nous pouvons tirer parti de ce que l’on appelle les cas “fall-through”, qui s’évaluent par rapport à plusieurs valeurs. Cela peut être utilisé pour créer des plages pour qu’une valeur réussisse, mais n’est toujours pas aussi flexible que les inégalités.

Scanner scan = new Scanner(System.in);
int foo = scan.nextInt();
switch(foo) {
    case 1:
        System.out.println("I'm equal or greater than one");
    case 2:
    case 3:    
        System.out.println("I'm one, two, or three");
        break;
    default:
        System.out.println("I'm not either one, two, or three");
}

En cas de foo == 1 la sortie sera :

I'm equal or greater than one
I'm one, two, or three

Dans le cas de foo == 3, la sortie sera :

I'm one, two, or three

L’instruction switch peut également être utilisée avec enums.

enum Option {
    BLUE_PILL,
    RED_PILL
}

public void takeOne(Option option) {
    switch(option) {
        case BLUE_PILL:
            System.out.println("Story ends, wake up, believe whatever you want.");
            break;
        case RED_PILL:
            System.out.println("I show you how deep the rabbit hole goes.");
            break;
            
    }
}

L’instruction switch peut également être utilisée avec Strings.

public void rhymingGame(String phrase) {
    switch (phrase) {
        case "apples and pears":
            System.out.println("Stairs");
            break;
        case "lorry":
            System.out.println("truck");
            break;
        default:
            System.out.println("Don't know any more");
    }
}

Continuer l’instruction en Java #

L’instruction continue est utilisée pour ignorer les étapes restantes de l’itération en cours et commencer par la prochaine itération de la boucle. Le contrôle va de l’instruction continue à la valeur du pas (incrément ou décrément), le cas échéant.

String[] programmers = {"Adrian", "Paul", "John", "Harry"};

    //john is not printed out
    for (String name : programmers) {
        if (name.equals("John"))
            continue;
        System.out.println(name);
    }

L’instruction continue peut également faire passer le contrôle du programme à la valeur de pas (le cas échéant) d’une boucle nommée :

Outer: // The name of the outermost loop is kept here as 'Outer'
for(int i = 0; i < 5; )
{
    for(int j = 0; j < 5; j++)
    {
        continue Outer;
    }
}

Si / Sinon Si / Sinon Contrôle #

if (i < 2) {
  System.out.println("i is less than 2");
} else if (i > 2) {
  System.out.println("i is more than 2");
} else {
  System.out.println("i is not less than 2, and not more than 2");
}

Le bloc if ne s’exécutera que lorsque i est égal ou inférieur à 1.

La condition else if n’est vérifiée que si toutes les conditions qui la précèdent (dans les constructions else if précédentes et les constructions parentes if) ont été testées sur false. Dans cet exemple, la condition else if ne sera vérifiée que si i est supérieur ou égal à 2.

Si son résultat est true, son bloc est exécuté et toutes les constructions else if et else après lui seront ignorées.

Si aucune des conditions if et else if n’a été testée sur true, le bloc else à la fin sera exécuté.

Pour les boucles #

for (int i = 0; i < 100; i++) {
    System.out.println(i);
}

Les trois composants de la boucle for (séparés par ;) sont la déclaration/initialisation de la variable (ici int i = 0), la condition (ici i < 100) et l’instruction d’incrémentation (ici i++ ). La déclaration de la variable est faite une fois comme si elle était placée juste à l’intérieur du { lors de la première exécution. Ensuite la condition est vérifiée, si elle est ’true’ le corps de la boucle s’exécutera, si elle est ‘false’ la boucle s’arrêtera. En supposant que la boucle continue, le corps s’exécutera et finalement, lorsque le } sera atteint, l’instruction d’incrémentation s’exécutera juste avant que la condition ne soit à nouveau vérifiée.

Les accolades sont facultatives (vous pouvez une ligne avec un point-virgule) si la boucle ne contient qu’une seule instruction. Mais, il est toujours recommandé d’utiliser des accolades pour éviter les malentendus et les bugs.

Les composants de la boucle for sont facultatifs. Si votre logique métier contient l’une de ces parties, vous pouvez omettre le composant correspondant de votre boucle “for”.

int i = obj.getLastestValue(); // i value is fetched from a method
    
for (; i < 100; i++) { // here initialization is not done
    System.out.println(i);
}

La structure for (;;) { function-body } est égale à une boucle while (true).

“Boucles For imbriquées”

Toute instruction en boucle contenant une autre instruction en boucle appelée boucle imbriquée. La même manière de boucler ayant plus de boucle interne est appelée “boucle for imbriquée”.

    for(;;){
        //Outer Loop Statements
        for(;;){
            //Inner Loop Statements
        }
        //Outer Loop Statements
    }

La boucle for imbriquée peut être démontrée pour imprimer des nombres en forme de triangle.

for(int i=9;i>0;i--){//Outer Loop
    System.out.println();
    for(int k=i;k>0;k--){//Inner Loop -1
        System.out.print(" ");
    }
    for(int j=i;j<=9;j++){//Inner Loop -2
        System.out.print(" "+j);
    }
 }

Opérateur ternaire #

Parfois, vous devez vérifier une condition et définir la valeur d’une variable.

Par ex.

String name;

if (A > B) {
    name = "Billy";
} else {
    name = "Jimmy";
}

Cela peut être facilement écrit en une ligne comme

String name = A > B ? "Billy" : "Jimmy";

La valeur de la variable est définie sur la valeur immédiatement après la condition, si la condition est vraie. Si la condition est fausse, la deuxième valeur sera donnée à la variable.

Boucles While #

int i = 0;
while (i < 100) { // condition gets checked BEFORE the loop body executes
    System.out.println(i);
    i++;
}

Une boucle “while” s’exécute tant que la condition entre parenthèses est “vraie”. Ceci est également appelé la structure de “boucle de pré-test” car l’instruction conditionnelle doit être satisfaite avant que le corps de la boucle principale ne soit exécuté à chaque fois.

Les accolades sont facultatives si la boucle ne contient qu’une seule instruction, mais certaines conventions de style de codage préfèrent avoir les accolades malgré tout.

Sinon #

int i = 2;
if (i < 2) {
  System.out.println("i is less than 2");
} else {
  System.out.println("i is greater than 2");
}

Une instruction if exécute le code de manière conditionnelle en fonction du résultat de la condition entre parenthèses. Lorsque la condition entre parenthèses est vraie, elle entrera dans le bloc de l’instruction if qui est définie par des accolades comme { et }. parenthèse ouvrante jusqu’à ce que la parenthèse fermante soit la portée de l’instruction if.

Le bloc “else” est facultatif et peut être omis. Il s’exécute si l’instruction if est false et ne s’exécute pas si l’instruction if est vraie, car dans ce cas, l’instruction if s’exécute.

Casser #

L’instruction break termine une boucle (comme for, while) ou l’évaluation d’une [instruction switch][1].

Boucle:

while(true) {
   if(someCondition == 5) {
       break;
   }
}

La boucle de l’exemple s’exécuterait indéfiniment. Mais lorsque someCondition est égal à 5 à un moment donné de l’exécution, la boucle se termine.

Si plusieurs boucles sont en cascade, seule la boucle la plus interne se termine par break.

[1] : https://www.wikiod.com/fr/java/structures-de-controle-de-base#Instruction de commutation

Essayez … Attrapez … Enfin #

La structure de contrôle try { ... } catch ( ... ) { ... } est utilisée pour gérer les [Exceptions] (https://www.wikiod.com/fr/java/exceptions-et-gestion-des-exceptions).

String age_input = "abc";
try {
    int age = Integer.parseInt(age_input);
    if (age >= 18) {
        System.out.println("You can vote!");
    } else {
        System.out.println("Sorry, you can't vote yet.");
    }
} catch (NumberFormatException ex) {
    System.err.println("Invalid input.  '" + age_input + "' is not a valid integer.");
}

Cela imprimerait:

Entrée invalide. ‘abc’ n’est pas un entier valide.

Une clause finally peut être ajoutée après le catch. La clause “finally” serait toujours exécutée, qu’une exception ait été levée ou non.

essayer { ... } attraper ( ... ) { ... } enfin { ... }

String age_input = "abc";
try {
    int age = Integer.parseInt(age_input);
    if (age >= 18) {
        System.out.println("You can vote!");
    } else {
        System.out.println("Sorry, you can't vote yet.");
    }
} catch (NumberFormatException ex) {
    System.err.println("Invalid input.  '" + age_input + "' is not a valid integer.");
} finally {
    System.out.println("This code will always be run, even if an exception is thrown");
}

Cela imprimerait:

Entrée invalide. ‘abc’ n’est pas un entier valide. Ce code sera toujours exécuté, même si une exception est levée

Pause imbriquée / continuer #

Il est possible de “casser” / “continuer” vers une boucle externe en utilisant des instructions d’étiquette :

outerloop:
for(...) {
    innerloop:
    for(...) {
        if(condition1)
            break outerloop;

        if(condition2)
            continue innerloop; // equivalent to: continue;
    }
}

Il n’y a pas d’autre utilisation des étiquettes en Java.
