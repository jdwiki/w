L’[interface java.util.Map][1] représente un mappage entre les clés et leurs valeurs. Une carte ne peut pas contenir de clés en double ; et chaque clé peut correspondre à au plus une valeur.

Étant donné que Map est une interface, vous devez instancier une implémentation concrète de cette interface afin de l’utiliser ; il existe plusieurs implémentations de Map, et les plus utilisées sont java.util.HashMap et java.util.TreeMap

[1] : https://docs.oracle.com/javase/7/docs/api/java/util/Map.html

Une [map][1] est un objet qui stocke des keys avec une value associée pour chaque clé. Une clé et sa valeur sont parfois appelées paire clé/valeur ou entrée. Les cartes offrent généralement ces fonctionnalités :

  • Les données sont stockées dans la carte dans des paires clé/valeur.
  • La carte ne peut contenir qu’une seule entrée pour une clé particulière. Si une carte contient une entrée avec une clé particulière et que vous essayez de stocker une deuxième entrée avec la même clé, la deuxième entrée remplacera la première. En d’autres termes, cela modifiera la valeur associée à la clé.
  • Les cartes fournissent des opérations rapides pour tester si une clé existe dans la carte, pour récupérer la valeur associée à une clé et pour supprimer une paire clé/valeur.

L’implémentation de carte la plus couramment utilisée est [HashMap][2]. Cela fonctionne bien avec les clés qui sont des chaînes ou des nombres.

Les cartes simples telles que HashMap ne sont pas ordonnées. L’itération sur les paires clé/valeur peut renvoyer des entrées individuelles dans n’importe quel ordre. Si vous devez parcourir les entrées de la carte de manière contrôlée, vous devez examiner les éléments suivants :

  • [Les cartes triées][3] telles que [TreeMap][4] parcourront les clés dans leur ordre naturel (ou dans un ordre que vous pouvez spécifier, en fournissant un [Comparator][5]). Par exemple, une carte triée utilisant des nombres comme clés devrait parcourir ses entrées dans l’ordre numérique.

  • [LinkedHashMap][6] permet de parcourir les entrées dans le même ordre qu’elles ont été insérées dans la carte, ou dans l’ordre des accès les plus récents.

[1] : https://docs.oracle.com/javase/8/docs/api/java/util/Map.html [2] : https://docs.oracle.com/javase/8/docs/api/java/util/HashMap.html [3] : https://docs.oracle.com/javase/8/docs/api/java/util/SortedMap.html [4] : https://docs.oracle.com/javase/8/docs/api/java/util/TreeMap.html [5] : https://docs.oracle.com/javase/8/docs/api/java/util/Comparator.html [6] : https://docs.oracle.com/javase/8/docs/api/java/util/LinkedHashMap.html

Itération efficace des entrées de carte #

Cette section fournit du code et des repères pour dix exemples d’implémentations uniques qui itèrent sur les entrées d’un Map<Integer, Integer> et génèrent la somme des valeurs Integer. Tous les exemples ont une complexité algorithmique de Θ(n), cependant, les repères sont toujours utiles pour fournir un aperçu des implémentations les plus efficaces dans un environnement “du monde réel”.

  1. Implémentation utilisant [Iterator][2] avec [Map.Entry][1]

     Iterator<Map.Entry<Integer, Integer>> it = map.entrySet().iterator();
 while (it.hasNext()) {
     Map.Entry<Integer, Integer> pair = it.next();
     sum += pair.getKey() + pair.getValue();
 }

  2. Implémentation en utilisant [for][3] avec [Map.Entry][1]

     for (Map.Entry<Integer, Integer> pair : map.entrySet()) {
     sum += pair.getKey() + pair.getValue();
 }

  3. Implémentation à l’aide de [Map.forEach][4] (Java 8+)

     map.forEach((k, v) -> sum[0] += k + v);

  4. Implémentation en utilisant [Map.keySet][5] avec [for][3]

     for (Integer key : map.keySet()) {
     sum += key + map.get(key);
 }

  5. Implémentation en utilisant [Map.keySet][5] avec [Iterator][2]

     Iterator<Integer> it = map.keySet().iterator();
 while (it.hasNext()) {
     Integer key = it.next();
     sum += key + map.get(key);
 }

  6. Implémentation utilisant [for][3] avec [Iterator][2] et [Map.Entry][1]

     for (Iterator<Map.Entry<Integer, Integer>> entries = 
          map.entrySet().iterator(); entries.hasNext(); ) {
     Map.Entry<Integer, Integer> entry = entries.next();
     sum += entry.getKey() + entry.getValue();
 }

  7. Implémentation à l’aide de [Stream.forEach][6] (Java 8+)

     map.entrySet().stream().forEach(e -> sum += e.getKey() + e.getValue());

  8. Implémentation utilisant [Stream.forEach][6] avec [Stream.parallel][7] (Java 8+)

     map.entrySet()
    .stream()
    .parallel()
    .forEach(e -> sum += e.getKey() + e.getValue());

  9. Implémentation à l’aide de [IterableMap][8] de [Apache Collections][10]

     MapIterator<Integer, Integer> mit = iterableMap.mapIterator();
 while (mit.hasNext()) {
     sum += mit.next() + it.getValue();
 }

  10. Implémentation à l’aide de [MutableMap][9] de [Eclipse Collections][11]

     mutableMap.forEachKeyValue((key, value) -> {
     sum += key + value;
 });

Utilisation des méthodes par défaut de Map à partir de Java 8 #

Exemples d’utilisation des méthodes par défaut introduites dans Java 8 dans l’interface Map

  1. Utiliser getOrDefault

Renvoie la valeur mappée à la clé, ou si la clé n’est pas présente, renvoie la valeur par défaut

Map<Integer, String> map = new HashMap<>();
map.put(1, "First element");
map.get(1);                                 // => First element
map.get(2);                                 // => null
map.getOrDefault(2, "Default element");     // => Default element
  1. Utiliser forEach

Permet d’effectuer l’opération spécifiée dans l’‘action’ sur chaque entrée de carte

  Map<Integer, String> map = new HashMap<Integer, String>();
  map.put(1, "one");
  map.put(2, "two");
  map.put(3, "three");
  map.forEach((key, value) -> System.out.println("Key: "+key+ " :: Value: "+value));

   // Key: 1 :: Value: one
   // Key: 2 :: Value: two
   // Key: 3 :: Value: three
  1. Utiliser replaceAll

Remplacera par une nouvelle valeur uniquement si la clé est présente

 Map<String, Integer> map = new HashMap<String, Integer>();
 map.put("john", 20);
 map.put("paul", 30);
 map.put("peter", 40);
 map.replaceAll((key,value)->value+10);   //{john=30, paul=40, peter=50}
  1. Utiliser putIfAbsent

La paire clé-valeur est ajoutée à la carte, si la clé n’est pas présente ou mappée à null

    Map<String, Integer> map = new HashMap<String, Integer>();
    map.put("john", 20);
    map.put("paul", 30);
    map.put("peter", 40);
    map.putIfAbsent("kelly", 50);     //{john=20, paul=30, peter=40, kelly=50}
  1. Utiliser supprimer

Supprime la clé uniquement si elle est associée à la valeur donnée

    Map<String, Integer> map = new HashMap<String, Integer>();
    map.put("john", 20);
    map.put("paul", 30);
    map.put("peter", 40);
    map.remove("peter",40); //{john=30, paul=40}

  1. Utiliser remplacer

    If the key is present then the value is replaced by new-value. If the key is not present, does nothing.

    Map<String, Integer> map = new HashMap<String, Integer>(); map.put(“john”, 20); map.put(“paul”, 30); map.put(“peter”, 40); map.replace(“peter”,50); //{john=20, paul=30, peter=50} map.replace(“jack”,60); //{john=20, paul=30, peter=50}

  2. Utiliser computeIfAbsent

Cette méthode ajoute une entrée dans la Map. la clé est spécifiée dans la fonction et la valeur est le résultat de l’application de la fonction de mappage

    Map<String, Integer> map = new HashMap<String, Integer>();
    map.put("john", 20);
    map.put("paul", 30);
    map.put("peter", 40);
    map.computeIfAbsent("kelly", k->map.get("john")+10); //{john=20, paul=30, peter=40, kelly=30}
    map.computeIfAbsent("peter", k->map.get("john")+10); //{john=20, paul=30, peter=40, kelly=30} //peter already present
  1. Utilisation de computeIfPresent

Cette méthode ajoute une entrée ou modifie une entrée existante dans la carte. Ne fait rien si une entrée avec cette clé n’est pas présente

    Map<String, Integer> map = new HashMap<String, Integer>();
    map.put("john", 20);
    map.put("paul", 30);
    map.put("peter", 40);
    map.computeIfPresent("kelly", (k,v)->v+10); //{john=20, paul=30, peter=40} //kelly not present
    map.computeIfPresent("peter", (k,v)->v+10); //{john=20, paul=30, peter=50} // peter present, so increase the value
  1. Utiliser calculer

Cette méthode remplace la valeur d’une clé par la valeur nouvellement calculée

    Map<String, Integer> map = new HashMap<String, Integer>();
    map.put("john", 20);
    map.put("paul", 30);
    map.put("peter", 40);
    map.compute("peter", (k,v)->v+50); //{john=20, paul=30, peter=90} //Increase the value
  1. Utiliser fusionner

Ajoute la paire clé-valeur à la carte, si la clé n’est pas présente ou si la valeur de la clé est nulle Remplace la valeur par la valeur nouvellement calculée, si la clé est présente La clé est supprimée de la carte, si la nouvelle valeur calculée est nulle

    Map<String, Integer> map = new HashMap<String, Integer>();
    map.put("john", 20);
    map.put("paul", 30);
    map.put("peter", 40);
    
    //Adds the key-value pair to the map, if key is not present or value for the key is null
    map.merge("kelly", 50 , (k,v)->map.get("john")+10); // {john=20, paul=30, peter=40, kelly=50}
    
    //Replaces the value with the newly computed value, if the key is present
    map.merge("peter", 50 , (k,v)->map.get("john")+10); //{john=20, paul=30, peter=30, kelly=50}
   
    //Key is removed from the map , if new value computed is null
    map.merge("peter", 30 , (k,v)->map.get("nancy")); //{john=20, paul=30, kelly=50}

Utilisation de HashMap #

HashMap est une implémentation de l’interface Map qui fournit une structure de données pour stocker des données dans des paires clé-valeur.

1. Déclarer HashMap

Map<KeyType, ValueType> myMap = new HashMap<KeyType, ValueType>();

KeyType et ValueType doivent être des types valides en Java, tels que - String, Integer, Float ou toute classe personnalisée comme Employee, Student, etc.

Par exemple : Map<String,Integer> myMap = new HashMap<String,Integer>();

2. Mettre des valeurs dans HashMap.

Pour mettre une valeur dans le HashMap, nous devons appeler la méthode put sur l’objet HashMap en passant la clé et la valeur en tant que paramètres.

myMap.put("key1", 1);
myMap.put("key2", 2);

Si vous appelez la méthode put avec la clé qui existe déjà dans la carte, la méthode remplacera sa valeur et renverra l’ancienne valeur.

3. Obtenir des valeurs de HashMap.

Pour obtenir la valeur d’un HashMap, vous devez appeler la méthode get, en passant la clé en paramètre.

myMap.get("key1");    //return 1 (class Integer)

Si vous passez une clé qui n’existe pas dans le HashMap, cette méthode renverra null

4. Vérifiez si la clé est dans la carte ou non.

myMap.containsKey(varKey);

5. Vérifiez si la valeur est dans la carte ou non.

myMap.containsValue(varValue);

Les méthodes ci-dessus renverront une valeur “booléenne” vraie ou fausse si la clé, la valeur existe dans la carte ou non.

Itérer dans le contenu d’une carte #

Les cartes fournissent des méthodes qui vous permettent d’accéder aux clés, valeurs ou paires clé-valeur de la carte en tant que collections. Vous pouvez parcourir ces collections. Soit la carte suivante par exemple :

Map<String, Integer> repMap = new HashMap<>();
repMap.put("Jon Skeet", 927_654);
repMap.put("BalusC", 708_826);
repMap.put("Darin Dimitrov", 715_567);

Itération à travers les clés de la carte :

for (String key : repMap.keySet()) {
    System.out.println(key);
}

Impressions :

Darin Dimitrov
Jon Skeet
BalusC

keySet() fournit les clés de la carte sous la forme d’un [Set][1]. Set est utilisé car les clés ne peuvent pas contenir de valeurs en double. L’itération dans l’ensemble donne chaque clé à tour de rôle. Les HashMaps ne sont pas ordonnés, donc dans cet exemple, les clés peuvent être renvoyées dans n’importe quel ordre.

Itération sur les valeurs de la carte :

for (Integer value : repMap.values()) {
    System.out.println(value);
}

Impressions :

715567
927654
708826

values() renvoie les valeurs de la carte sous forme de [Collection][2]. L’itération dans la collection donne chaque valeur à tour de rôle. Là encore, les valeurs peuvent être renvoyées dans n’importe quel ordre.

** Itérer à travers les clés et les valeurs ensemble **

for (Map.Entry<String, Integer> entry : repMap.entrySet()) {
    System.out.printf("%s = %d\n", entry.getKey(), entry.getValue());
}

Impressions :

Darin Dimitrov = 715567
Jon Skeet = 927654
BalusC = 708826

entrySet() renvoie une collection d’objets [Map.Entry][3]. Map.Entry donne accès à la clé et à la valeur de chaque entrée.

[1] : http://docs.oracle.com/javase/8/docs/api/java/util/Set.html [2] : http://docs.oracle.com/javase/8/docs/api/java/util/Collection.html [3] : http://docs.oracle.com/javase/8/docs/api/java/util/Map.Entry.html

## Ajouter plusieurs éléments

Nous pouvons utiliser V put (clé K, valeur V) :

Associe la valeur spécifiée à la clé spécifiée dans cette carte (opération facultative). Si la carte contenait auparavant un mappage pour la clé, l’ancienne valeur est remplacée par la valeur spécifiée.

String currentVal;
Map<Integer, String> map = new TreeMap<>();
currentVal = map.put(1, "First element.");
System.out.println(currentVal);// Will print null
currentVal = map.put(2, "Second element.");
System.out.println(currentVal); // Will print null yet again    
currentVal = map.put(2, "This will replace 'Second element'");
System.out.println(currentVal); // will print Second element.
System.out.println(map.size()); // Will print 2 as key having
// value 2 was replaced.

Map<Integer, String> map2 = new HashMap<>();
map2.put(2, "Element 2");
map2.put(3, "Element 3");

map.putAll(map2);

System.out.println(map.size());

Production:

‘3’

Pour ajouter de nombreux éléments, vous pouvez utiliser une classe interne comme celle-ci :

Map<Integer, String> map = new HashMap<>() {{
    // This is now an anonymous inner class with an unnamed instance constructor
    put(5, "high");
    put(4, "low");
    put(1, "too slow");
}};

Gardez à l’esprit que la création d’une classe interne anonyme n’est pas toujours efficace et peut entraîner des fuites de mémoire. Dans la mesure du possible, utilisez plutôt un bloc d’initialisation :

static Map<Integer, String> map = new HashMap<>();

static {
    // Now no inner classes are created so we can avoid memory leaks
    put(5, "high");
    put(4, "low");
    put(1, "too slow");
}

L’exemple ci-dessus rend la carte statique. Il peut également être utilisé dans un contexte non statique en supprimant toutes les occurrences de static.

En plus de cela, la plupart des implémentations prennent en charge putAll, qui peut ajouter toutes les entrées d’une carte à une autre comme ceci :

another.putAll(one);

Fusionner, combiner et composer des cartes #

Utilisez putAll pour placer chaque membre d’une carte dans une autre. Les clés déjà présentes dans la carte verront leurs valeurs correspondantes écrasées.

Map<String, Integer> numbers = new HashMap<>();
numbers.put("One", 1)
numbers.put("Three", 3)
Map<String, Integer> other_numbers = new HashMap<>();
other_numbers.put("Two", 2)
other_numbers.put("Three", 4)

numbers.putAll(other_numbers)

Cela donne le mappage suivant dans numbers :

"One" -> 1
"Two" -> 2
"Three" -> 4 //old value 3 was overwritten by new value 4

Si vous souhaitez combiner des valeurs au lieu de les écraser, vous pouvez utiliser [Map.merge](http://docs.oracle.com/javase/8/docs/api/java/util/Map.html#merge- K-V-java.util.function.BiFunction-), ajouté dans Java 8, qui utilise une “BiFunction” fournie par l’utilisateur pour fusionner les valeurs des clés en double. merge opère sur des clés et des valeurs individuelles, vous devrez donc utiliser une boucle ou Map.forEach. Ici, nous concaténons les chaînes pour les clés en double :

for (Map.Entry<String, Integer> e : other_numbers.entrySet())
    numbers.merge(e.getKey(), e.getValue(), Integer::sum);
//or instead of the above loop
other_numbers.forEach((k, v) -> numbers.merge(k, v, Integer::sum));

Si vous souhaitez appliquer la contrainte, il n’y a pas de clés en double, vous pouvez utiliser une fonction de fusion qui génère une AssertionError :

mapA.forEach((k, v) ->
    mapB.merge(k, v, (v1, v2) ->
        {throw new AssertionError("duplicate values for key: "+k);}));

Composer Map<X,Y> et Map<Y,Z> pour obtenir Map<X,Z> #

Si vous souhaitez composer deux mappings, vous pouvez le faire comme suit

    Map<String, Integer> map1 = new HashMap<String, Integer>();
    map1.put("key1", 1);
    map1.put("key2", 2);
    map1.put("key3", 3);
    
    Map<Integer, Double> map2 = new HashMap<Integer, Double>();
    map2.put(1, 1.0);
    map2.put(2, 2.0);
    map2.put(3, 3.0);

    Map<String, Double> map3 = new new HashMap<String, Double>();
    map1.forEach((key,value)->map3.put(key,map2.get(value)));

Cela donne le mappage suivant

    "key1" -> 1.0
    "key2" -> 2.0
    "key3" -> 3.0

Ajouter un élément #

  1. Ajout

    Map<Integer, String> map = new HashMap<>(); map.put(1, “First element.”); System.out.println(map.get(1));

Résultat : “Premier élément”.

  1. Remplacer

    Map<Integer, String> map = new HashMap<>(); map.put(1, “First element.”); map.put(1, “New element.”); System.out.println(map.get(1));

Résultat : “Nouvel élément”.

HashMap est utilisé comme exemple. D’autres implémentations qui implémentent l’interface Map peuvent également être utilisées.

Map<Integer, String> map = new HashMap<>();

map.put(1, "First element.");
map.put(2, "Second element.");
map.put(3, "Third element.");

map.clear();

System.out.println(map.size());   // => 0

Map<String, String> num = new HashMap<>();
num.put("one", "first");

if (num.containsKey("one")) {
    System.out.println(num.get("one")); // => first
}

Les cartes peuvent contenir des valeurs nulles #

Pour les cartes, il faut faire attention à ne pas confondre “contenant une clé” avec “ayant une valeur”. Par exemple, HashMaps peut contenir null, ce qui signifie que le comportement suivant est parfaitement normal :

Map<String, String> map = new HashMap<>();
map.put("one", null);
if (map.containsKey("one")) {
    System.out.println("This prints !"); // This line is reached 
}
if (map.get("one") != null) {
    System.out.println("This is never reached !"); // This line is never reached 
}

Plus formellement, il n’y a aucune garantie que map.contains(key) <=> map.get(key)!=null

Utiliser l’objet personnalisé comme clé #

Avant d’utiliser votre propre objet comme clé, vous devez remplacer les méthodes hashCode() et equals() de votre objet.

Dans un cas simple, vous auriez quelque chose comme:

class MyKey {
    private String name;
    MyKey(String name) {
        this.name = name;
    }

    @Override
    public boolean equals(Object obj) {            
        if(obj instanceof MyKey) {
            return this.name.equals(((MyKey)obj).name);
        }
        return false;
    }
    
    @Override
    public int hashCode() {
        return this.name.hashCode();
    }
}

hashCode décidera à quel seau de hachage appartient la clé et equals décidera quel objet à l’intérieur de ce seau de hachage.

Sans ces méthodes, la référence de votre objet sera utilisée pour la comparaison ci-dessus qui ne fonctionnera que si vous utilisez la même référence d’objet à chaque fois.

Création et initialisation des cartes #

Introduction #

Maps stocke des paires clé/valeur, où chaque clé a une valeur associée. Étant donné une clé particulière, la carte peut rechercher la valeur associée très rapidement.

Maps, également connu sous le nom de tableau associé, est un objet qui stocke les données sous forme de clés et de valeurs. En Java, les cartes sont représentées à l’aide de l’interface Map qui n’est pas une extension de l’interface de collection.

  • Voie 1 :-

      /*J2SE < 5.0*/
  Map map = new HashMap();
  map.put("name", "A");
  map.put("address", "Malviya-Nagar");
  map.put("city", "Jaipur");
  System.out.println(map);

  • Voie 2 :-

      /*J2SE 5.0+ style (use of generics):*/
  Map<String, Object> map = new HashMap<>();
  map.put("name", "A");
  map.put("address", "Malviya-Nagar");
  map.put("city", "Jaipur");
  System.out.println(map);

  • Voie 3 :-

      Map<String, Object> map = new HashMap<String, Object>(){{
      put("name", "A");
      put("address", "Malviya-Nagar");
      put("city", "Jaipur");
  }};
  System.out.println(map);

  • Voie 4 :-

      Map<String, Object> map = new TreeMap<String, Object>();
      map.put("name", "A");
      map.put("address", "Malviya-Nagar");
      map.put("city", "Jaipur");
  System.out.println(map);

      //Java 8
