Analyse XML à l'aide des API JAXP

  • Analyse d’un document à l’aide de l’API StAX
  • Analyse et navigation dans un document à l’aide de l’API DOM

Sur cette page

  • Analyse d’un document à l’aide de l’API StAX
  • Analyse et navigation dans un document à l’aide de l’API DOM

L’analyse syntaxique XML est l’interprétation de documents XML afin de manipuler leur contenu à l’aide de constructions sensibles, qu’il s’agisse de “nœuds”, d’“attributs”, de “documents”, d’“espaces de noms” ou d’événements liés à ces constructions.

Java possède une API native pour la gestion des documents XML, appelée JAXP, ou Java API for XML Processing. JAXP et une implémentation de référence ont été fournis avec chaque version de Java depuis Java 1.4 (JAXP v1.1) et ont évolué depuis. Java 8 est livré avec JAXP version 1.6.

L’API propose différentes manières d’interagir avec les documents XML, à savoir :

  • L’interface DOM (Document Object Model)
  • L’interface SAX (Simple API for XML)
  • L’interface StAX (Streaming API for XML)

Principes de l’interface DOM #

L’interface DOM vise à fournir une méthode d’interprétation XML conforme au W3C DOM. Diverses versions de JAXP ont pris en charge divers niveaux de spécification DOM (jusqu’au niveau 3).

Sous l’interface Document Object Model, un document XML est représenté sous forme d’arborescence, en commençant par “l’élément de document”. Le type de base de l’API est le type Node, il permet de naviguer depuis un Node à son parent, ses enfants ou ses frères et sœurs (bien que tous les Node ne puissent pas avoir d’enfants, par exemple, les nœuds Text sont finaux dans l’arborescence et n’ont jamais d’enfant). Les balises XML sont représentées par des Elements, qui étendent notamment le Node avec des méthodes liées aux attributs.

L’interface DOM est très utile puisqu’elle permet une analyse “une ligne” des documents XML sous forme d’arborescences, et permet une modification aisée de l’arborescence construite (ajout de nœud, suppression, copie, …), et enfin sa sérialisation (retour sur disque ) publier des modifications. Cela a cependant un prix : l’arborescence réside en mémoire, par conséquent, les arborescences DOM ne sont pas toujours pratiques pour les documents XML volumineux. De plus, la construction de l’arborescence n’est pas toujours le moyen le plus rapide de traiter le contenu XML, surtout si l’on ne s’intéresse pas à toutes les parties du document XML.

Principes de l’interface SAX #

L’API SAX est une API orientée événement pour traiter les documents XML. Dans ce modèle, les composants d’un document XML sont interprétés comme des événements (par exemple “une balise a été ouverte”, “une balise a été fermée”, “un nœud de texte a été rencontré”, “un commentaire a été rencontré”). ..

L’API SAX utilise une approche “push parsing”, où un SAX Parser est responsable pour interpréter le document XML et invoque des méthodes sur un délégué (un ContentHandler) pour traiter n’importe quel événement trouvé dans le document XML. Habituellement, on n’écrit jamais d’analyseur, mais on fournit un gestionnaire pour rassembler toutes les informations nécessaires à partir du document XML.

L’interface SAX surmonte les limitations de l’interface DOM en ne conservant que le minimum de données nécessaires au niveau de l’analyseur (par exemple, les contextes d’espaces de noms, l’état de validation), par conséquent, seules les informations conservées par le ContentHandler - dont vous, le développeur, êtes responsable - sont conservés en mémoire. Le compromis est qu’il n’y a aucun moyen de “remonter dans le temps/le document XML” avec une telle approche : alors que DOM permet à un Node de revenir à son parent, il n’y a pas une telle possibilité dans SAX.

Principes de l’interface StAX #

L’API StAX adopte une approche similaire au traitement XML comme l’API SAX (c’est-à-dire pilotée par les événements), la seule différence très significative étant que StAX est un analyseur pull (où SAX était un analyseur push). Dans SAX, le Parser est sous contrôle et utilise des rappels sur le ContentHandler. Dans Stax, vous appelez l’analyseur et contrôlez quand/si vous voulez obtenir le prochain “événement” XML.

L’API commence par XMLStreamReader (ou [XMLEventReader](https://docs.oracle. com/javase/8/docs/api/javax/xml/stream/XMLStreamReader.html)), qui sont les passerelles par lesquelles le développeur peut demander nextEvent(), à la manière d’un itérateur.

Analyse d’un document à l’aide de l’API StAX #

Considérant le document suivant :

<?xml version='1.0' encoding='UTF-8' ?>
<library>
   <book id='1'>Effective Java</book>
   <book id='2'>Java Concurrency In Practice</book>
   <notABook id='3'>This is not a book element</notABook>
</library>

On peut utiliser le code suivant pour l’analyser et créer une carte des titres de livres par identifiant de livre.

import javax.xml.stream.XMLInputFactory;
import javax.xml.stream.XMLStreamConstants;
import javax.xml.stream.XMLStreamReader;
import java.io.StringReader;
import java.util.HashMap;
import java.util.Map;

public class StaxDemo {

public static void main(String[] args) throws Exception {
    String xmlDocument = "<?xml version='1.0' encoding='UTF-8' ?>"
            + "<library>"
                + "<book id='1'>Effective Java</book>"
                + "<book id='2'>Java Concurrency In Practice</book>"
                + "<notABook id='3'>This is not a book element </notABook>"
            + "</library>";

    XMLInputFactory xmlInputFactory = XMLInputFactory.newFactory();
    // Various flavors are possible, e.g. from an InputStream, a Source, ...
    XMLStreamReader xmlStreamReader = xmlInputFactory.createXMLStreamReader(new StringReader(xmlDocument));

    Map<Integer, String> bookTitlesById = new HashMap<>();

    // We go through each event using a loop
    while (xmlStreamReader.hasNext()) {
        switch (xmlStreamReader.getEventType()) {
            case XMLStreamConstants.START_ELEMENT:
                System.out.println("Found start of element: " + xmlStreamReader.getLocalName());
                // Check if we are at the start of a <book> element
                if ("book".equals(xmlStreamReader.getLocalName())) {
                    int bookId = Integer.parseInt(xmlStreamReader.getAttributeValue("", "id"));
                    String bookTitle = xmlStreamReader.getElementText();
                    bookTitlesById.put(bookId, bookTitle);
                }
                break;
            // A bunch of other things are possible : comments, processing instructions, Whitespace...
            default:
                break;
        }
        xmlStreamReader.next();
    }

    System.out.println(bookTitlesById);
}

Cela génère :

Found start of element: library
Found start of element: book
Found start of element: book
Found start of element: notABook
{1=Effective Java, 2=Java Concurrency In Practice}

Dans cet exemple, il faut faire attention à quelques choses :

  1. L’utilisation de xmlStreamReader.getAttributeValue fonctionne car nous avons d’abord vérifié que l’analyseur est dans l’état START_ELEMENT. Dans tous les autres états (à l’exception de ATTRIBUTES), l’analyseur est mandaté pour lancer IllegalStateException, car les attributs ne peuvent apparaître qu’au début des éléments.

  2. Il en va de même pour xmlStreamReader.getTextContent(), cela fonctionne car nous sommes à un START_ELEMENT et nous savons dans ce document que l’élément <book> n’a pas de nœuds enfants non textuels.

Pour l’analyse de documents plus complexes (plus en profondeur, éléments imbriqués, …), il est recommandé de “déléguer” l’analyseur à des sous-méthodes ou à d’autres objets, par ex. avoir une classe ou une méthode BookParser et lui faire gérer chaque élément du START_ELEMENT au END_ELEMENT de la balise XML du livre.

On peut également utiliser un objet Stack pour conserver des données importantes de haut en bas de l’arborescence.

Analyse et navigation dans un document à l’aide de l’API DOM #

Considérant le document suivant :

<?xml version='1.0' encoding='UTF-8' ?>
<library>
   <book id='1'>Effective Java</book>
   <book id='2'>Java Concurrency In Practice</book>
</library>

On peut utiliser le code suivant pour construire un arbre DOM à partir d’une String :

import org.w3c.dom.Document;
import org.w3c.dom.Node;
import org.w3c.dom.NodeList;
import org.xml.sax.InputSource;

import javax.xml.parsers.DocumentBuilder;
import javax.xml.parsers.DocumentBuilderFactory;
import java.io.StringReader;

public class DOMDemo {

public static void main(String[] args) throws Exception {
    String xmlDocument = "<?xml version='1.0' encoding='UTF-8' ?>"
            + "<library>"
            + "<book id='1'>Effective Java</book>"
            + "<book id='2'>Java Concurrency In Practice</book>"
            + "</library>";

    DocumentBuilderFactory documentBuilderFactory = DocumentBuilderFactory.newInstance();
    // This is useless here, because the XML does not have namespaces, but this option is usefull to know in cas
    documentBuilderFactory.setNamespaceAware(true);
    DocumentBuilder documentBuilder = documentBuilderFactory.newDocumentBuilder();
    // There are various options here, to read from an InputStream, from a file, ...
    Document document = documentBuilder.parse(new InputSource(new StringReader(xmlDocument)));

    // Root of the document
    System.out.println("Root of the XML Document: " + document.getDocumentElement().getLocalName());

    // Iterate the contents
    NodeList firstLevelChildren = document.getDocumentElement().getChildNodes();
    for (int i = 0; i < firstLevelChildren.getLength(); i++) {
        Node item = firstLevelChildren.item(i);
        System.out.println("First level child found, XML tag name is: " + item.getLocalName());
        System.out.println("\tid attribute of this tag is : " + item.getAttributes().getNamedItem("id").getTextContent());
    }

    // Another way would have been
    NodeList allBooks = document.getDocumentElement().getElementsByTagName("book");
}
}

Le code donne ceci :

Root of the XML Document: library
First level child found, XML tag name is: book
id attribute of this tag is : 1
First level child found, XML tag name is: book
id attribute of this tag is : 2
