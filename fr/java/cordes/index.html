Les chaînes (java.lang.String) sont des morceaux de texte stockés dans votre programme. Les chaînes ne sont ** pas ** un [type de données primitif en Java] [1], cependant, elles sont très courantes dans les programmes Java.

En Java, les chaînes sont immuables, ce qui signifie qu’elles ne peuvent pas être modifiées. (Cliquez [ici] [2] pour une explication plus approfondie de l’immuabilité.)

[1] : http://stackoverflow.com/questions/10430582/primitive-data-types-in-java [2] : http://stackoverflow.com/questions/279507/what-is-meant-by-immutable

Comme les chaînes Java sont immuables, toutes les méthodes qui manipulent une String renverront un nouvel objet String. Ils ne changent pas la String d’origine. Cela inclut les méthodes de sous-chaîne et de remplacement auxquelles les programmeurs C et C++ s’attendraient pour faire muter l’objet String cible.

Utilisez un StringBuilder au lieu de String si vous souhaitez concaténer plus de deux objets String dont les valeurs ne peuvent pas être déterminées au moment de la compilation. Cette technique est plus performante que la création de nouveaux objets String et leur concaténation car StringBuilder est modifiable.

StringBuffer peut également être utilisé pour concaténer des objets String. Cependant, cette classe est moins performante car elle est conçue pour être thread-safe et acquiert un mutex avant chaque opération. Comme vous n’avez presque jamais besoin de thread-safety lors de la concaténation de chaînes, il est préférable d’utiliser StringBuilder.

Si vous pouvez exprimer une concaténation de chaînes en une seule expression, il est préférable d’utiliser l’opérateur +. Le compilateur Java convertira une expression contenant des concaténations + en une séquence efficace d’opérations en utilisant String.concat(...) ou StringBuilder. Le conseil d’utiliser explicitement StringBuilder ne s’applique que lorsque la concaténation implique plusieurs expressions.

Ne stockez pas d’informations sensibles dans des chaînes. Si quelqu’un est capable d’obtenir un vidage mémoire de votre application en cours d’exécution, il pourra trouver tous les objets String existants et lire leur contenu. Cela inclut les objets String inaccessibles et en attente de récupération de place. Si cela vous préoccupe, vous devrez effacer les données de chaîne sensibles dès que vous en aurez terminé. Vous ne pouvez pas faire cela avec des objets String car ils sont immuables. Par conséquent, il est conseillé d’utiliser des objets char[] pour contenir des données de caractères sensibles et de les effacer (par exemple, les écraser avec des caractères '\ 000') lorsque vous avez terminé.

Toutes les instances String sont créées sur le tas, même les instances qui correspondent à des littéraux de chaîne. La particularité des littéraux de chaîne est que la JVM garantit que tous les littéraux égaux (c’est-à-dire composés des mêmes caractères) sont représentés par un seul objet String (ce comportement est spécifié dans JLS). Ceci est implémenté par les chargeurs de classe JVM. Lorsqu’un chargeur de classe charge une classe, il recherche les littéraux de chaîne utilisés dans la définition de classe, chaque fois qu’il en voit un, il vérifie s’il existe déjà un enregistrement dans le pool de chaînes pour ce littéral (en utilisant le littéral comme clé) . S’il existe déjà une entrée pour le littéral, la référence à une instance “String” stockée comme paire pour ce littéral est utilisée. Sinon, une nouvelle instance String est créée et une référence à l’instance est stockée pour le littéral (utilisé comme clé) dans le pool de chaînes. (Voir aussi string interning).

Le pool de chaînes est conservé dans le tas Java et est soumis à une récupération de place normale.

Dans les versions de Java antérieures à Java 7, le pool de chaînes était conservé dans une partie spéciale du tas appelée “PermGen”. Cette partie n’a été collectée qu’occasionnellement.

Dans Java 7, le pool de chaînes a été déplacé de “PermGen”.

Notez que les littéraux de chaîne sont implicitement accessibles à partir de toute méthode qui les utilise. Cela signifie que les objets String correspondants ne peuvent être ramassés que si le code lui-même est ramassé.

Jusqu’à Java 8, les objets String sont implémentés sous la forme d’un tableau de caractères UTF-16 (2 octets par caractère). Il existe une proposition dans Java 9 pour implémenter String en tant que tableau d’octets avec un champ d’indicateur d’encodage pour noter si la chaîne est encodée en octets (LATIN-1) ou en caractères (UTF-16).

N’utilisez pas l’opérateur == pour comparer les chaînes #

À moins que vous ne puissiez garantir que toutes les chaînes ont été internées (voir ci-dessous), vous ** ne devriez pas ** utiliser les opérateurs == ou != pour comparer les chaînes. Ces opérateurs testent en fait les références, et puisque plusieurs objets String peuvent représenter la même chaîne, cela est susceptible de donner la mauvaise réponse.

Depuis Java 1.7, il est possible de comparer une variable String à des littéraux dans une instruction switch. String stringToSwitch = "A";

    case "a":
        System.out.println("a");
        break;
    case "A":
        System.out.println("A"); //the code goes here
        break;
    case "B":
        System.out.println("B");
        break;
    default:
        break;
}

Comparer des chaînes avec des valeurs constantes #

Lorsque vous comparez une String à une valeur constante, vous pouvez placer la valeur constante à gauche de equals pour vous assurer que vous n’obtiendrez pas une NullPointerException si l’autre chaîne est null.

Alors que foo.equals("baz") lèvera une NullPointerException si foo est null, "baz".equals(foo) sera évalué à false.

Une alternative plus lisible consiste à utiliser Objects.equals(), qui effectue une vérification nulle sur les deux paramètres : Objects.equals(foo, "baz").

Comparaison avec les chaînes internes #

La spécification du langage Java ([JLS 3.10.6][4]) stipule ce qui suit :

“De plus, un littéral de chaîne fait toujours référence à la même instance de la classe String. En effet, les littéraux de chaîne - ou, plus généralement, les chaînes qui sont les valeurs d’expressions constantes - sont internes afin de partager des instances uniques, en utilisant la méthode String.intern.”

Cela signifie qu’il est sûr de comparer des références à deux littéraux de chaîne en utilisant ==. De plus, il en va de même pour les références aux objets String qui ont été produites à l’aide de la méthode String.intern().

Par exemple:

String strObj = new String("Hello!");
String str = "Hello!";

// The two string references point two strings that are equal
if (strObj.equals(str)) {
    System.out.println("The strings are equal");
}

// The two string references do not point to the same object
if (strObj != str) {
    System.out.println("The strings are not the same object");
}

// If we intern a string that is equal to a given literal, the result is
// a string that has the same reference as the literal.
String internedStr = strObj.intern();

if (internedStr == str) {
    System.out.println("The interned string and the literal are the same object");
}

Dans les coulisses, le mécanisme d’internement maintient une table de hachage qui contient toutes les chaînes internes qui sont toujours accessibles. Lorsque vous appelez intern() sur une String, la méthode recherche l’objet dans la table de hachage :

  • Si la chaîne est trouvée, cette valeur est renvoyée en tant que chaîne interne.
  • Sinon, une copie de la chaîne est ajoutée à la table de hachage et cette chaîne est renvoyée en tant que chaîne interne.

Il est possible d’utiliser l’internation pour permettre aux chaînes d’être comparées en utilisant ==. Cependant, il y a des problèmes importants à faire cela; voir https://www.wikiod.com/fr/java/pieges-java—problemes-de-performances#Piège - Interner des chaînes pour que vous puissiez utiliser == est une mauvaise idée =201610101338294726756 pour plus de détails. Il n’est pas recommandé dans la plupart des cas.

Changer la casse des caractères dans une chaîne #

Le type [String][2] fournit deux méthodes pour convertir les chaînes entre majuscules et minuscules :

  • [toUpperCase][3] pour convertir tous les caractères en majuscules
  • [toLowerCase][4] pour convertir tous les caractères en minuscules

Ces méthodes renvoient toutes les deux les chaînes converties en tant que nouvelles instances de String : les objets String d’origine ne sont pas modifiés car String est immuable en Java. Voir ceci pour en savoir plus sur l’immuabilité : [Immutability of Strings in Java][1]

String string = "This is a Random String";
String upper = string.toUpperCase();
String lower = string.toLowerCase();

System.out.println(string);   // prints "This is a Random String"
System.out.println(lower);    // prints "this is a random string"
System.out.println(upper);    // prints "THIS IS A RANDOM STRING"

Les caractères non alphabétiques, tels que les chiffres et les signes de ponctuation, ne sont pas affectés par ces méthodes. Notez que ces méthodes peuvent également traiter de manière incorrecte certains caractères Unicode sous certaines conditions.

Remarque : Ces méthodes sont sensibles aux paramètres régionaux et peuvent produire des résultats inattendus si elles sont utilisées sur des chaînes destinées à être interprétées indépendamment des paramètres régionaux. Les exemples sont les identifiants de langage de programmation, les clés de protocole et les balises “HTML”.

Par exemple, "TITLE".toLowerCase() dans une locale turque renvoie “tıtle”, où ı (\u0131) est le caractère [LETTRE MINUSCULE LATINE SANS POINT I][5]. Pour obtenir des résultats corrects pour les chaînes insensibles aux paramètres régionaux, transmettez Locale.ROOT comme paramètre à la méthode de conversion de casse correspondante (par exemple, toLowerCase(Locale.ROOT) ou toUpperCase(Locale.ROOT)).

Bien que l’utilisation de Locale.ENGLISH soit également correcte dans la plupart des cas, la méthode invariante de langue est Locale.ROOT.

Une liste détaillée des caractères Unicode qui nécessitent une casse spéciale peut être trouvée [sur le site Web du Consortium Unicode] [6].

Changement de casse d’un caractère spécifique dans une chaîne ASCII :

Pour modifier la casse d’un caractère spécifique d’une chaîne ASCII, l’algorithme suivant peut être utilisé :

Pas:

  1. Déclarez une chaîne.
  2. Saisissez la chaîne.
  3. Convertissez la chaîne en tableau de caractères.
  4. Saisissez le caractère à rechercher.
  5. Recherchez le caractère dans le tableau de caractères.
  6. S’il est trouvé, vérifiez si le caractère est en minuscule ou en majuscule.
    • If Uppercase, add 32 to the ASCII code of the character.
    • If Lowercase, subtract 32 from the ASCII code of the character.
  7. Modifiez le caractère d’origine du tableau Caractère.
  8. Reconvertissez le tableau de caractères en chaîne.

Voilà, la casse du personnage est modifiée.

Un exemple de code pour l’algorithme est :

Scanner scanner = new Scanner(System.in);
System.out.println("Enter the String");
String s = scanner.next();
char[] a = s.toCharArray();
System.out.println("Enter the character you are looking for");
System.out.println(s);
String c = scanner.next();
char d = c.charAt(0);

for (int i = 0; i <= s.length(); i++) {
    if (a[i] == d) {
        if (d >= 'a' && d <= 'z') {
            d -= 32;
        } else if (d >= 'A' && d <= 'Z') {
            d += 32;
        }
        a[i] = d;
        break;
    }
}
s = String.valueOf(a);
System.out.println(s);

Séparer les chaînes #

Vous pouvez diviser une [String][1] sur un caractère de délimitation particulier ou une [Expression régulière][2], vous pouvez utiliser la méthode [String.split()][3] qui a la signature suivante :

public String[] split(String regex)

Notez que le caractère de délimitation ou l’expression régulière est supprimé du tableau de chaînes résultant.

Exemple utilisant un caractère de délimitation :

String lineFromCsvFile = "Mickey;Bolton;12345;121216";
String[] dataCells = lineFromCsvFile.split(";");
// Result is dataCells = { "Mickey", "Bolton", "12345", "121216"};

Exemple utilisant une expression régulière :

String lineFromInput = "What    do you need    from me?";
String[] words = lineFromInput.split("\\s+"); // one or more space chars
// Result is words = {"What", "do", "you", "need", "from", "me?"};

Vous pouvez même diviser directement un littéral String :

String[] firstNames = "Mickey, Frank, Alicia, Tom".split(", ");
// Result is firstNames = {"Mickey", "Frank", "Alicia", "Tom"};

Attention : N’oubliez pas que le paramètre est toujours traité comme une expression régulière.

"aaa.bbb".split("."); // This returns an empty array

Dans l’exemple précédent, . est traité comme le caractère générique de l’expression régulière qui correspond à n’importe quel caractère, et puisque chaque caractère est un délimiteur, le résultat est un tableau vide.

** Fractionnement basé sur un délimiteur qui est un méta-caractère regex **

Les caractères suivants sont considérés comme spéciaux (alias méta-caractères) dans regex

  < > - = ! ( ) [ ] { } \ ^ $ | ? * + .

Pour diviser une chaîne en fonction de l’un des délimiteurs ci-dessus, vous devez soit les échapper en utilisant \\ ou utiliser Pattern.quote() :

  • En utilisant Pattern.quote() :

     String s = "a|b|c";
 String regex = Pattern.quote("|");
 String[] arr = s.split(regex);

  • Échapper aux caractères spéciaux :

     String s = "a|b|c";
 String[] arr = s.split("\\|");

Split supprime les valeurs vides

split(delimiter) supprime par défaut les chaînes vides à la fin du tableau de résultats. Pour désactiver ce mécanisme, nous devons utiliser une version surchargée de split(delimiter, limit) avec une limite définie sur une valeur négative comme

String[] split = data.split("\\|", -1);

split(regex) renvoie en interne le résultat de split(regex, 0).

Le paramètre limit contrôle le nombre de fois que le motif est appliqué et affecte donc la longueur du tableau résultant. Si la limite n est supérieure à zéro, le modèle sera appliqué au plus n - 1 fois, la longueur du tableau ne sera pas supérieure à n et la dernière entrée du tableau contiendra toutes les entrées au-delà de la dernière correspondance. délimiteur. Si n est négatif, alors le motif sera appliqué autant de fois que possible et le tableau peut avoir n’importe quelle longueur. Si n est égal à zéro, le modèle sera appliqué autant de fois que possible, le tableau peut avoir n’importe quelle longueur et les chaînes vides à la fin seront supprimées.

Pool de chaînes et stockage de tas #

Comme de nombreux objets Java, toutes les instances String sont créées sur le tas, même les littéraux. Lorsque la JVM trouve un littéral String qui n’a pas de référence équivalente dans le tas, la JVM crée une instance String correspondante sur le tas ** et ** elle stocke également une référence à l’instance String nouvellement créée dans le Piscine à cordes. Toutes les autres références au même littéral String sont remplacées par l’instance String précédemment créée dans le tas.

Regardons l’exemple suivant :

class Strings
{
    public static void main (String[] args)
    {
        String a = "alpha";
        String b = "alpha";
        String c = new String("alpha");

        //All three strings are equivalent
        System.out.println(a.equals(b) && b.equals(c));

        //Although only a and b reference the same heap object
        System.out.println(a == b);
        System.out.println(a != c);
        System.out.println(b != c);
    }
}

La sortie de ce qui précède est :

true
true
true
true

[![Diagramme du tas Java et du pool de chaînes][1]][1] Lorsque nous utilisons des guillemets doubles pour créer une chaîne, il recherche d’abord une chaîne avec la même valeur dans le pool de chaînes. S’il est trouvé, il renvoie simplement la référence, sinon il crée une nouvelle chaîne dans le pool, puis renvoie la référence.

Cependant, en utilisant l’opérateur new, nous forçons la classe String à créer un nouvel objet String dans l’espace du tas. Nous pouvons utiliser la méthode intern() pour le mettre dans le pool ou faire référence à un autre objet String du pool de chaînes ayant la même valeur.

Le pool de chaînes lui-même est également créé sur le tas.

Avant Java 7, les littéraux String étaient stockés dans le pool de constantes d’exécution dans la zone de méthode de PermGen, qui avait une taille fixe.

Le pool de chaînes résidait également dans PermGen.

[RFC : 6962931][2]

Dans JDK 7, les chaînes internes ne sont plus allouées dans la génération permanente du tas Java, mais sont plutôt allouées dans la partie principale du tas Java (appelée les générations jeunes et anciennes), avec les autres objets créés par le application. Cette modification se traduira par plus de données résidant dans le tas Java principal et moins de données dans la génération permanente, et peut donc nécessiter un ajustement des tailles de tas. La plupart des applications ne verront que des différences relativement faibles dans l’utilisation du tas en raison de ce changement, mais les applications plus importantes qui chargent de nombreuses classes ou font un usage intensif de la méthode String.intern() verront des différences plus importantes.

Joindre des chaînes avec un délimiteur #

Un tableau de chaînes peut être joint à l’aide de la méthode statique [String.join()](https://docs.oracle.com/javase/8/docs/api/java/lang/String.html#join-java .lang.CharSequence-java.lang.CharSequence…-):

String[] elements = { "foo", "bar", "foobar" };
String singleString = String.join(" + ", elements);

System.out.println(singleString); // Prints "foo + bar + foobar"

Pour avoir un contrôle précis sur la jointure, vous pouvez utiliser la classe StringJoiner :

StringJoiner sj = new StringJoiner(", ", "[", "]"); 
    // The last two arguments are optional, 
    // they define prefix and suffix for the result string

sj.add("foo");
sj.add("bar");
sj.add("foobar");

System.out.println(sj); // Prints "[foo, bar, foobar]"

Pour joindre un flux de chaînes, vous pouvez utiliser le [collecteur de jonction](https://docs.oracle.com/javase/8/docs/api/java/util/stream/Collectors.html#joining-java.lang. CharSequence-):

Stream<String> stringStream = Stream.of("foo", "bar", "foobar");
String joined = stringStream.collect(Collectors.joining(", "));
System.out.println(joined); // Prints "foo, bar, foobar"

Il existe une option pour définir [préfixe et suffixe](https://docs.oracle.com/javase/8/docs/api/java/util/stream/Collectors.html#joining-java.lang.CharSequence-java.lang .CharSequence-java.lang.CharSequence-) ici aussi :

Stream<String> stringStream = Stream.of("foo", "bar", "foobar");
String joined = stringStream.collect(Collectors.joining(", ", "{", "}"));
System.out.println(joined); // Prints "{foo, bar, foobar}"

Concaténation de chaînes et StringBuilders #

La concaténation de chaînes peut être effectuée à l’aide de l’opérateur +. Par exemple:

String s1 = "a";
String s2 = "b";
String s3 = "c";
String s = s1 + s2 + s3; // abc

Normalement, une implémentation de compilateur effectuera la concaténation ci-dessus à l’aide de méthodes impliquant un StringBuilder sous le capot. Une fois compilé, le code ressemblerait à ce qui suit :

StringBuilder sb = new StringBuilder("a");
String s = sb.append("b").append("c").toString();

StringBuilder a plusieurs méthodes surchargées pour ajouter différents types, par exemple, pour ajouter un int au lieu d’un String. Par exemple, une implémentation peut convertir :

String s1 = "a";
String s2 = "b";    
String s = s1 + s2 + 2; // ab2

à ce qui suit :

StringBuilder sb = new StringBuilder("a");
String s = sb.append("b").append(2).toString();

Les exemples ci-dessus illustrent une opération de concaténation simple qui est effectivement effectuée à un seul endroit dans le code. La concaténation implique une seule instance de StringBuilder. Dans certains cas, une concaténation s’effectue de manière cumulative comme dans une boucle :

String result = "";
for(int i = 0; i < array.length; i++) {
    result += extractElement(array[i]);
}
return result;

Dans de tels cas, l’optimisation du compilateur n’est généralement pas appliquée et chaque itération créera un nouvel objet StringBuilder. Cela peut être optimisé en transformant explicitement le code pour utiliser un seul StringBuilder :

StringBuilder result = new StringBuilder();
for(int i = 0; i < array.length; i++) {
    result.append(extractElement(array[i]));
}
return result.toString();

Un StringBuilder sera initialisé avec un espace vide de seulement 16 caractères. Si vous savez à l’avance que vous allez construire des chaînes plus grandes, il peut être avantageux de l’initialiser avec une taille suffisante à l’avance, afin que le tampon interne n’ait pas besoin d’être redimensionné :

StringBuilder buf = new StringBuilder(30); // Default is 16 characters
buf.append("0123456789");
buf.append("0123456789"); // Would cause a reallocation of the internal buffer otherwise
String result = buf.toString(); // Produces a 20-chars copy of the string

Si vous produisez de nombreuses chaînes, il est conseillé de réutiliser StringBuilder :

StringBuilder buf = new StringBuilder(100);
for (int i = 0; i < 100; i++) {
    buf.setLength(0); // Empty buffer
    buf.append("This is line ").append(i).append('\n');
    outputfile.write(buf.toString());
}

Si (et seulement si) plusieurs threads écrivent dans le même tampon, utilisez [StringBuffer][1], qui est une version “synchronisée” de “StringBuilder”. Mais comme généralement un seul thread écrit dans un tampon, il est généralement plus rapide d’utiliser StringBuilder sans synchronisation.

[1] : https://docs.oracle.com/javase/7/docs/api/java/lang/StringBuffer.html

Utilisation de la méthode concat() :

String string1 = "Hello ";
String string2 = "world";
String string3 = string1.concat(string2);  // "Hello world"

Cela renvoie une nouvelle chaîne qui est chaîne1 avec chaîne2 ajoutée à la fin. Vous pouvez également utiliser la méthode concat() avec des littéraux de chaîne, comme dans :

"My name is ".concat("Buyya");

Sous-chaînes #

String s = "this is an example";
String a = s.substring(11); // a will hold the string starting at character 11 until the end ("example")
String b = s.substring(5, 10); // b will hold the string starting at character 5 and ending right before character 10 ("is an")
String b = s.substring(5, b.length()-3); // b will hold the string starting at character 5 ending right before b' s lenght is out of 3  ("is an exam")

Les sous-chaînes peuvent également être appliquées pour trancher et ajouter/remplacer un caractère dans sa chaîne d’origine. Par exemple, vous avez fait face à une date chinoise contenant des caractères chinois mais vous souhaitez la stocker sous forme de chaîne de date bien formatée.

String datestring = "2015年11月17日"
datestring = datestring.substring(0, 4) + "-" + datestring.substring(5,7) + "-" + datestring.substring(8,10);
//Result will be 2015-11-17

La méthode substring extrait une partie d’une String. Lorsqu’un paramètre est fourni, le paramètre est le début et le morceau s’étend jusqu’à la fin de la String. Lorsqu’il est donné deux paramètres, le premier paramètre est le caractère de début et le second paramètre est l’index du caractère juste après la fin (le caractère à l’index n’est pas inclus). Un moyen simple de vérifier est que la soustraction du premier paramètre du second devrait donner la longueur attendue de la chaîne.

Dans les versions JDK <7u6, la méthode substring instancie une String qui partage le même support char[] que la String d’origine et dont les champs internes offset et count sont définis sur le début et la longueur du résultat. . Un tel partage peut provoquer des fuites de mémoire, qui peuvent être évitées en appelant new String(s.substring(...)) pour forcer la création d’une copie, après quoi le char[] peut être ramassé.

À partir de JDK 7u6, la méthode substring copie toujours l’intégralité du tableau char[] sous-jacent, ce qui rend la complexité linéaire par rapport à la constante précédente mais garantit en même temps l’absence de fuites de mémoire.

Nouveau séparateur de ligne indépendant de la plate-forme #

Étant donné que le nouveau séparateur de ligne varie d’une plate-forme à l’autre (par exemple, \n sur les systèmes de type Unix ou \r\n sur Windows), il est souvent nécessaire d’avoir un moyen d’y accéder indépendant de la plate-forme. En Java, il peut être récupéré à partir d’une propriété système :

System.getProperty("line.separator")

Étant donné que le nouveau séparateur de ligne est si souvent nécessaire, à partir de Java 7, une méthode de raccourci renvoyant exactement le même résultat que le code ci-dessus est disponible :

System.lineSeparator()

Remarque : Comme il est très peu probable que le nouveau séparateur de ligne change pendant l’exécution du programme, il est judicieux de le stocker dans une variable finale statique au lieu de le récupérer à partir de la propriété système chaque fois que cela est nécessaire.

Lorsque vous utilisez String.format, utilisez %n plutôt que \n ou ‘\r\n’ pour générer un séparateur de nouvelle ligne indépendant de la plate-forme.

System.out.println(String.format('line 1: %s.%nline 2: %s%n', lines[0],lines[1]));

Ajout de la méthode toString() pour les objets personnalisés #

Supposons que vous ayez défini la classe Person suivante :

public class Person {

    String name;
    int age;
    
    public Person (int age, String name) {
        this.age = age;
        this.name = name;
    }
}

Si vous instanciez un nouvel objet Person :

Person person = new Person(25, "John");

et plus tard dans votre code, vous utilisez l’instruction suivante pour imprimer l’objet :

System.out.println(person.toString());

Démo en direct sur Ideone

vous obtiendrez une sortie semblable à la suivante :

Person@7ab89d

Ceci est le résultat de l’implémentation de la méthode toString() définie dans la classe Object, une superclasse de Person. La documentation de Object.toString() indique :

La méthode toString pour la classe Object renvoie une chaîne composée du nom de la classe dont l’objet est une instance, du caractère arobase « @ » et de la représentation hexadécimale non signée du code de hachage de l’objet. En d’autres termes, cette méthode renvoie une chaîne égale à la valeur de :

getClass().getName() + ‘@’ + Integer.toHexString(hashCode())

Ainsi, pour une sortie significative, vous devrez ** remplacer ** la méthode toString() :

@Override
public String toString() {
    return "My name is " + this.name + " and my age is " + this.age;
}

Maintenant, la sortie sera :

My name is John and my age is 25

Vous pouvez également écrire

System.out.println(person);

[Démo en direct sur Ideone] (https://ideone.com/51al3w)

En fait, println() invoque implicitement la méthode toString sur l’objet.

Inverser les chaînes #

Il existe plusieurs façons d’inverser une chaîne pour la faire reculer.

  1. StringBuilder/StringBuffer :

     String code = "code";
 System.out.println(code);

 StringBuilder sb = new StringBuilder(code); 
 code = sb.reverse().toString();

 System.out.println(code);

  2. Tableau de caractères :

    String code = "code";
System.out.println(code);

char[] array = code.toCharArray();
for (int index = 0, mirroredIndex = array.length - 1; index < mirroredIndex; index++, mirroredIndex--) {
    char temp = array[index];
    array[index] = array[mirroredIndex];
    array[mirroredIndex] = temp;
}

// print reversed
System.out.println(new String(array));

Supprimer les espaces blancs au début et à la fin d’une chaîne #

La méthode [trim()][1] renvoie une nouvelle chaîne sans les espaces de début et de fin.

String s = new String("   Hello World!!  ");
String t = s.trim();  // t = "Hello World!!"

Si vous “coupez” une chaîne qui n’a pas d’espace à supprimer, vous recevrez la même instance de chaîne.

Notez que la méthode [trim()][1] a sa propre notion d’espace, qui diffère de la notion utilisée par le Character. isWhitespace() :

  • Tous les caractères de contrôle ASCII avec les codes U+0000 à U+0020 sont considérés comme des espaces et sont supprimés par trim(). Cela inclut les caractères U+0020 'ESPACE', U+0009 'CHARACTER TABULATION', U+000A 'LINE FEED' et U+000D 'CARRIAGE RETURN', mais aussi les caractères comme U+ 0007 'CLOCHE'.

  • Les espaces blancs Unicode comme U+00A0 'NO-BREAK SPACE' ou U+2003 'EM SPACE' ne sont pas reconnus par trim().

[1] : http://docs.oracle.com/javase/8/docs/api/java/lang/String.html#trim–

Remplacement de pièces de Strings #

Deux façons de remplacer : par regex ou par correspondance exacte.

Remarque : l’objet String d’origine sera inchangé, la valeur de retour contient la chaîne modifiée.

Correspondance exacte #

### Remplacez un seul caractère par un autre :

String replace(char oldChar, char newChar)

Renvoie une nouvelle chaîne résultant du remplacement de toutes les occurrences de oldChar dans cette chaîne par newChar.

String s = "popcorn";
System.out.println(s.replace('p','W'));

Résultat:

WoWcorn

Remplacez la séquence de caractères par une autre séquence de caractères : #

String replace(CharSequence target, CharSequence replacement)

Remplace chaque sous-chaîne de cette chaîne qui correspond à la séquence cible littérale par la séquence de remplacement littérale spécifiée.

String s = "metal petal et al.";
System.out.println(s.replace("etal","etallica"));

Résultat:

metallica petallica et al.

Regex #

Remarque : le regroupement utilise le caractère $ pour référencer les groupes, comme $1.

### Remplacer toutes les correspondances :

String replaceAll(String regex, String replacement)

Remplace chaque sous-chaîne de cette chaîne qui correspond à l’expression régulière donnée par le remplacement donné.

String s = "spiral metal petal et al.";
System.out.println(s.replaceAll("(\\w*etal)","$1lica"));

Résultat:

spiral metallica petallica et al.

Remplacer uniquement la première correspondance : #

String replaceFirst(String regex, String replacement)

Remplace la première sous-chaîne de cette chaîne qui correspond à l’expression régulière donnée par le remplacement donné

String s = "spiral metal petal et al.";
System.out.println(s.replaceAll("(\\w*etal)","$1lica"));

Résultat:

spiral metallica petal et al.

Commutateur insensible à la casse #

switch lui-même ne peut pas être paramétré pour être insensible à la casse, mais si cela est absolument nécessaire, il peut se comporter de manière insensible à la chaîne d’entrée en utilisant toLowerCase() ou toUpperCase :

switch (myString.toLowerCase()) {
     case "case1" :
        ...            
     break;
     case "case2" :
        ...            
     break;
}

Il faut se méfier

  • Local peut affecter la façon dont [le changement de cas se produit] (https://www.wikiod.com/fr/java/cordes#Changer la casse des caractères dans une chaîne) !
  • Il faut veiller à ne pas avoir de caractères majuscules dans les étiquettes - ceux-ci ne seront jamais exécutés !

Obtenir la longueur d’une chaîne #

Afin d’obtenir la longueur d’un objet String, appelez la méthode length() dessus. La longueur est égale au nombre d’unités de code UTF-16 (caractères) dans la chaîne.

String str = "Hello, World!";
System.out.println(str.length()); // Prints out 13

Démo en direct sur Ideone

Un char dans une chaîne est une valeur UTF-16. Les points de code Unicode dont les valeurs sont ≥ 0x1000 (par exemple, la plupart des emojis) utilisent deux positions de caractères. Pour compter le nombre de points de code Unicode dans une chaîne, que chaque point de code tienne ou non dans une valeur char UTF-16, vous pouvez utiliser la méthode codePointCount :

int length = str.codePointCount(0, str.length());

Vous pouvez également utiliser un flux de points de code, à partir de Java 8 :

int length = str.codePoints().count();

Obtenir le nième caractère d’une chaîne #

String str = "My String";

System.out.println(str.charAt(0)); // "M"
System.out.println(str.charAt(1)); // "y"
System.out.println(str.charAt(2)); // " "
System.out.println(str.charAt(str.length-1)); // Last character "g"

Pour obtenir le nième caractère d’une chaîne, appelez simplement charAt(n) sur une String, où n est l’index du caractère que vous souhaitez récupérer

REMARQUE : l’index ’n’ commence à ‘0’, donc le premier élément est à n=0.

Compter les occurrences d’une sous-chaîne ou d’un caractère dans une chaîne #

La méthode countMatches de [org.apache.commons.lang3.StringUtils][2] est généralement utilisée pour compter les occurrences d’une sous-chaîne ou d’un caractère dans une String :

import org.apache.commons.lang3.StringUtils;

String text = "One fish, two fish, red fish, blue fish";

// count occurrences of a substring
String stringTarget = "fish";
int stringOccurrences = StringUtils.countMatches(text, stringTarget); // 4

// count occurrences of a char
char charTarget = ',';
int charOccurrences = StringUtils.countMatches(text, charTarget); // 3

Sinon, pour faire la même chose avec les API Java standard, vous pouvez utiliser des expressions régulières :

import java.util.regex.Matcher;
import java.util.regex.Pattern;
 
String text = "One fish, two fish, red fish, blue fish";
System.out.println(countStringInString("fish", text)); // prints 4
System.out.println(countStringInString(",", text)); // prints 3
 

public static int countStringInString(String search, String text) {
    Pattern pattern = Pattern.compile(search);
    Matcher matcher = pattern.matcher(text);
    
    int stringOccurrences = 0;
    while (matcher.find()) {
      stringOccurrences++;
    }
    return stringOccurrences;
}

[1] : https://commons.apache.org/proper/commons-lang/javadocs/api-3.1/org/apache/commons/lang3/package-summary.html [2] : http://org.apache.commons.lang3.StringUtils
