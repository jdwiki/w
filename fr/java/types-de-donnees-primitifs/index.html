Les 8 types de données primitifs byte, short, int, long, char, boolean, float et double sont les types qui stockent le plus de données numériques brutes dans les programmes Java.

Syntaxe #

  • int aInt = 8 ; // La partie définissante (nombre) de cette déclaration int est appelée un littéral.

  • int hexInt = 0x1a ; // = 26 ; Vous pouvez définir des littéraux avec des valeurs hexadécimales précédées de 0x.

  • int binInt = 0b11010 ; // = 26 ; Vous pouvez également définir des littéraux binaires ; préfixé par 0b.

  • long bonLong = 10000000000L ; // Par défaut, les littéraux entiers sont de type int. En ajoutant le L à la fin du littéral, vous indiquez au compilateur que le littéral est un long. Sans cela, le compilateur lancerait une erreur “Nombre entier trop grand”.

  • double aDouble = 3,14 ; // Les littéraux à virgule flottante sont de type double par défaut.

  • flotteur aFlotteur = 3.14F ; // Par défaut, ce littéral aurait été un double (et aurait provoqué une erreur “Types incompatibles”), mais en ajoutant un F, nous disons au compilateur qu’il s’agit d’un flottant.

Java a 8 types de données primitifs, à savoir boolean, byte, short, char, int, long, float et double. (Tous les autres types sont des types de référence. Cela inclut tous les types de tableau et les types/classes d’objets intégrés qui ont une signification particulière dans le langage Java ; par exemple, String, Class et Throwable et ses sous-classes.)

Le résultat de toutes les opérations (addition, soustraction, multiplication, etc.) sur un type primitif est au moins un int, ainsi l’ajout d’un short à un short produit un int, tout comme l’ajout d’un byte à un byte, ou un char à un char. Si vous souhaitez réaffecter le résultat de cela à une valeur du même type, vous devez le transtyper. par exemple.

byte a = 1;
byte b = 2;
byte c = (byte) (a + b);

Ne pas transtyper l’opération entraînera une erreur de compilation.

Cela est dû à la partie suivante de la [Java Language Spec, §2.11.1](http://docs.oracle.com/javase/specs/jvms/se7/html/jvms-2.html#jvms-2.11. 1):

Un compilateur encode des charges de valeurs littérales de types byte et short à l’aide d’instructions Java Virtual Machine qui étendent ces valeurs en valeurs de type int au moment de la compilation ou de l’exécution. Les chargements de valeurs littérales de types boolean et char sont encodés à l’aide d’instructions qui étendent le littéral à une valeur de type int au moment de la compilation ou de l’exécution. [..]. Ainsi, la plupart des opérations sur des valeurs de types réels ‘boolean’, ‘byte’, ‘char’ et ‘short’ sont correctement effectuées par des instructions opérant sur des valeurs de type de calcul ‘int’.

La raison derrière cela est également spécifiée dans cette section :

Compte tenu de la taille d’opcode d’un octet de la machine virtuelle Java, l’encodage des types en opcodes met la pression sur la conception de son jeu d’instructions. Si chaque instruction typée prenait en charge tous les types de données d’exécution de la machine virtuelle Java, il y aurait plus d’instructions que ce qui pourrait être représenté dans un octet. […] Des instructions distinctes peuvent être utilisées pour convertir entre les types de données non pris en charge et pris en charge si nécessaire.

La primitive char #

Un char peut stocker un seul caractère Unicode 16 bits. Un caractère littéral est entouré de guillemets simples

char myChar = 'u';
char myChar2 = '5';
char myChar3 = 65; // myChar3 == 'A'

Il a une valeur minimale de \u0000 (0 dans la représentation décimale, également appelé caractère nul) et une valeur maximale de \uffff (65 535).

La valeur par défaut d’un char est \u0000.

char defaultChar;    // defaultChar == \u0000

Afin de définir un caractère de valeur ', une séquence d’échappement (caractère précédé d’une barre oblique inverse) doit être utilisée :

char singleQuote = '\'';

Il existe également d’autres séquences d’échappement :

char tab = '\t';
char backspace = '\b';
char newline = '\n';
char carriageReturn = '\r';
char formfeed = '\f';
char singleQuote = '\'';
char doubleQuote = '\"'; // escaping redundant here; '"' would be the same; however still allowed
char backslash = '\\';
char unicodeChar = '\uXXXX' // XXXX represents the Unicode-value of the character you want to display

Vous pouvez déclarer un char de n’importe quel caractère Unicode.

char heart = '\u2764';
System.out.println(Character.toString(heart)); // Prints a line containing "❤".

Il est également possible d’ajouter à un char. par exemple. pour parcourir chaque lettre minuscule, vous pouvez procéder comme suit :

for (int i = 0; i <= 26; i++) {
    char letter = (char) ('a' + i);
    System.out.println(letter);
}

La primitive flottante #

Un “float” est un nombre à virgule flottante IEEE 754 32 bits simple précision. Par défaut, les décimales sont interprétées comme des doubles. Pour créer un float, ajoutez simplement un f au littéral décimal.

double doubleExample = 0.5;      // without 'f' after digits = double
float floatExample = 0.5f;       // with 'f' after digits    = float

float myFloat = 92.7f;           // this is a float...
float positiveFloat = 89.3f;     // it can be positive,
float negativeFloat = -89.3f;    // or negative
float integerFloat = 43.0f;      // it can be a whole number (not an int)
float underZeroFloat = 0.0549f;  // it can be a fractional value less than 0

Les flottants gèrent les cinq opérations arithmétiques courantes : addition, soustraction, multiplication, division et module.

Remarque : Les éléments suivants peuvent varier légèrement en raison d’erreurs en virgule flottante. Certains résultats ont été arrondis pour des raisons de clarté et de lisibilité (c’est-à-dire que le résultat imprimé de l’exemple d’addition était en fait 34,600002).

// addition
float result = 37.2f + -2.6f;  // result: 34.6

// subtraction
float result = 45.1f - 10.3f;    // result: 34.8

// multiplication
float result = 26.3f * 1.7f;   // result: 44.71

// division
float result = 37.1f / 4.8f;   // result: 7.729166

// modulus
float result = 37.1f % 4.8f;   // result: 3.4999971

En raison de la manière dont les nombres à virgule flottante sont stockés (c’est-à-dire sous forme binaire), de nombreux nombres n’ont pas de représentation exacte.

float notExact = 3.1415926f;
System.out.println(notExact); // 3.1415925

Bien que l’utilisation de float convienne à la plupart des applications, ni float ni double ne doivent être utilisés pour stocker des représentations exactes de nombres décimaux (comme des montants monétaires) ou des nombres nécessitant une plus grande précision. Au lieu de cela, la classe BigDecimal doit être utilisée.

La valeur par défaut d’un float est 0.0f.

float defaultFloat;    // defaultFloat == 0.0f

Un “flottant” est précis à environ une erreur de 1 sur 10 millions.

Remarque : Float.POSITIVE_INFINITY, Float.NEGATIVE_INFINITY, Float.NaN sont des valeurs float. NaN représente les résultats d’opérations qui ne peuvent pas être déterminées, telles que la division de 2 valeurs infinies. De plus, 0f et -0f sont différents, mais == renvoie true :

float f1 = 0f;
float f2 = -0f;
System.out.println(f1 == f2); // true
System.out.println(1f / f1); // Infinity
System.out.println(1f / f2); // -Infinity
System.out.println(Float.POSITIVE_INFINITY / Float.POSITIVE_INFINITY); // NaN

La primitive int #

Un type de données primitif tel que int contient des valeurs directement dans la variable qui l’utilise, tandis qu’une variable déclarée à l’aide de Integer contient une référence à la valeur.

Selon [java API][1] : “La classe Integer encapsule une valeur de type primitif int dans un objet. Un objet de type Integer contient un seul champ dont le type est int.”

Par défaut, int est un entier signé 32 bits. Il peut stocker une valeur minimale de -231 et une valeur maximale de 231 - 1.

int example = -42;
int myInt = 284;
int anotherInt = 73;

int addedInts = myInt + anotherInt; // 284 + 73 = 357
int subtractedInts = myInt - anotherInt; // 284 - 73 = 211

Si vous avez besoin de stocker un nombre en dehors de cette plage, long doit être utilisé à la place. Le dépassement de la plage de valeurs de int entraîne un débordement d’entier, ce qui fait que la valeur dépassant la plage est ajoutée à l’emplacement opposé de la plage (le positif devient négatif et vice versa). La valeur est ((value - MIN_VALUE) % RANGE) + MIN_VALUE, ou ((value + 2147483648) % 4294967296) - 2147483648

int demo = 2147483647; //maximum positive integer
System.out.println(demo); //prints 2147483647
demo = demo + 1; //leads to an integer overflow
System.out.println(demo); // prints -2147483648

Les valeurs maximales et minimales de int peuvent être trouvées à :

int high = Integer.MAX_VALUE;    // high == 2147483647
int low = Integer.MIN_VALUE;     // low == -2147483648

La valeur par défaut d’un int est 0

int defaultInt;    // defaultInt == 0

[1] : https://docs.oracle.com/javase/7/docs/api/

Consommation de mémoire des primitives par rapport aux primitives en boîte #

Primitif Type en boîte Taille mémoire des primitives / boxed
booléen booléen 1 octet / 16 octets
octet Octet 1 octet / 16 octets
court Court 2 octets / 16 octets
char Char 2 octets / 16 octets
entier Entier 4 octets / 16 octets
longue longue 8 octets / 16 octets
flotter Flotteur 4 octets / 16 octets
doubler Double 8 octets / 16 octets

Les objets encadrés nécessitent toujours 8 octets pour le type et la gestion de la mémoire, et comme la taille des objets est toujours un multiple de 8, les types encadrés * nécessitent tous 16 octets au total *. De plus, chaque utilisation d’un objet en boîte implique de stocker une référence qui représente 4 ou 8 octets supplémentaires, selon les options JVM et JVM.

Dans les opérations gourmandes en données, la consommation de mémoire peut avoir un impact majeur sur les performances. La consommation de mémoire augmente encore plus lors de l’utilisation de tableaux : un tableau float[5] ne nécessitera que 32 octets ; alors qu’un Float[5] stockant 5 valeurs non nulles distinctes nécessitera un total de 112 octets (sur 64 bits sans pointeurs compressés, cela passe à 152 octets).

Caches de valeur en boîte #

Les frais généraux d’espace des types en boîte peuvent être atténués dans une certaine mesure par les caches de valeurs en boîte. Certains des types encadrés implémentent un cache d’instances. Par exemple, par défaut, la classe Integer mettra en cache les instances pour représenter les nombres compris entre -128 et +127. Cela ne réduit cependant pas le surcoût lié à l’indirection mémoire supplémentaire.

Si vous créez une instance d’un type boxed soit par autoboxing, soit en appelant la méthode statique valueOf(primitive), le système d’exécution tentera d’utiliser une valeur mise en cache. Si votre application utilise un grand nombre de valeurs dans la plage qui est mise en cache, cela peut réduire considérablement la pénalité de mémoire liée à l’utilisation de types encadrés. Certes, si vous créez des instances de valeurs encadrées “à la main”, il est préférable d’utiliser valueOf plutôt que new. (L’opération new crée toujours une nouvelle instance.) Si, toutefois, la majorité de vos valeurs ne sont pas dans la plage mise en cache, il peut être plus rapide d’appeler new et d’enregistrer la recherche de cache.

Conversion de primitives #

En Java, nous pouvons convertir entre des valeurs entières et des valeurs à virgule flottante. De plus, comme chaque caractère correspond à un nombre dans l’encodage Unicode, les types char peuvent être convertis vers et à partir des types entiers et à virgule flottante. boolean est le seul type de données primitif qui ne peut pas être converti vers ou à partir d’un autre type de données primitif.

Il existe deux types de conversion : la conversion étendue et la conversion restrictive.

Une conversion étendue se produit lorsqu’une valeur d’un type de données est convertie en une valeur d’un autre type de données qui occupe plus de bits que la première. Il n’y a pas de problème de perte de données dans ce cas.

De même, une conversion de rétrécissement se produit lorsqu’une valeur d’un type de données est convertie en une valeur d’un autre type de données qui occupe moins de bits que la première. Une perte de données peut se produire dans ce cas.

Java effectue des conversions étendues automatiquement. Mais si vous souhaitez effectuer une * conversion restrictive * (si vous êtes sûr qu’aucune perte de données ne se produira), vous pouvez forcer Java à effectuer la conversion à l’aide d’une construction de langage connue sous le nom de “cast”.

Conversion élargie :

int a = 1;    
double d = a;    // valid conversion to double, no cast needed (widening)

Réduction de la conversion :

double d = 18.96
int b = d;       // invalid conversion to int, will throw a compile-time error
int b = (int) d; // valid conversion to int, but result is truncated (gets rounded down)
                 // This is type-casting
                 // Now, b = 18

La double primitive #

Un “double” est un nombre à virgule flottante IEEE 754 64 bits double précision.

double example = -7162.37;
double myDouble = 974.21;
double anotherDouble = 658.7;

double addedDoubles = myDouble + anotherDouble; // 315.51
double subtractedDoubles = myDouble - anotherDouble; // 1632.91

double scientificNotationDouble = 1.2e-3;    // 0.0012

En raison de la manière dont les nombres à virgule flottante sont stockés, de nombreux nombres n’ont pas de représentation exacte.

double notExact = 1.32 - 0.42; // result should be 0.9
System.out.println(notExact); // 0.9000000000000001

Bien que l’utilisation de “double” convienne à la plupart des applications, ni “float” ni “double” ne doivent être utilisés pour stocker des nombres précis tels que des devises. Au lieu de cela, la classe BigDecimal doit être utilisée

La valeur par défaut d’un double est 0.0d

public double defaultDouble;    // defaultDouble == 0.0

Remarque : Double.POSITIVE_INFINITY, Double.NEGATIVE_INFINITY, Double.NaN sont des valeurs double. NaN représente les résultats d’opérations qui ne peuvent pas être déterminées, telles que la division de 2 valeurs infinies. De plus, 0d et -0d sont différents, mais == renvoie true :

double d1 = 0d;
double d2 = -0d;
System.out.println(d1 == d2); // true
System.out.println(1d / d1); // Infinity
System.out.println(1d / d2); // -Infinity
System.out.println(Double.POSITIVE_INFINITY / Double.POSITIVE_INFINITY); // NaN

La longue primitive #

Par défaut, long est un entier signé 64 bits (dans Java 8, il peut être signé ou non signé). Signé, il peut stocker une valeur minimale de -263 et une valeur maximale de 263 - 1, et non signé il peut stocker une valeur minimale de 0 et un maximum valeur de 264 - 1

long example = -42;
long myLong = 284;
long anotherLong = 73;

//an "L" must be appended to the end of the number, because by default,
//numbers are assumed to be the int type. Appending an "L" makes it a long
//as 549755813888 (2 ^ 39) is larger than the maximum value of an int (2^31 - 1),
//"L" must be appended 
long bigNumber = 549755813888L;

long addedLongs = myLong + anotherLong; // 284 + 73 = 357
long subtractedLongs = myLong - anotherLong; // 284 - 73 = 211

Les valeurs maximales et minimales de long peuvent être trouvées à :

long high = Long.MAX_VALUE;    // high == 9223372036854775807L
long low = Long.MIN_VALUE;     // low == -9223372036854775808L

La valeur par défaut d’un long est 0L

long defaultLong;    // defaultLong == 0L

Remarque : la lettre “L” ajoutée à la fin du littéral “long” n’est pas sensible à la casse, mais il est recommandé d’utiliser la majuscule car elle est plus facile à distinguer du premier chiffre :

2L == 2l;            // true

Avertissement : Java met en cache les instances d’objets Integer de -128 à 127. Le raisonnement est expliqué ici : https://blogs.oracle.com/darcy/entry/boxing_and_caches_integer_valueof

Les résultats suivants peuvent être trouvés :

Long val1 = 127L;
Long val2 = 127L;

System.out.println(val1 == val2); // true

Long val3 = 128L;
Long val4 = 128L;

System.out.println(val3 == val4); // false

Pour comparer correctement 2 valeurs Object Long, utilisez le code suivant (à partir de Java 1.7):

Long val3 = 128L;
Long val4 = 128L;

System.out.println(Objects.equal(val3, val4)); // true

Comparer un long primitif à un objet long n’entraînera pas un faux négatif comme le fait la comparaison de 2 objets avec ==.

La primitive booléenne #

Un booléen peut stocker l’une des deux valeurs, soit true ou false

boolean foo = true;
System.out.println("foo = " + foo);                // foo = true

boolean bar = false;
System.out.println("bar = " + bar);                // bar = false

boolean notFoo = !foo;
System.out.println("notFoo = " + notFoo);          // notFoo = false

boolean fooAndBar = foo && bar;
System.out.println("fooAndBar = " + fooAndBar);    // fooAndBar = false

boolean fooOrBar = foo || bar;
System.out.println("fooOrBar = " + fooOrBar);      // fooOrBar = true

boolean fooXorBar = foo ^ bar;
System.out.println("fooXorBar = " + fooXorBar);    // fooXorBar = true

La valeur par défaut d’un booléen est false

boolean defaultBoolean;    // defaultBoolean == false

La primitive d’octet #

Un byte est un entier signé de 8 bits. Il peut stocker une valeur minimale de -27 (-128) et une valeur maximale de 27 - 1 (127)

byte example = -36;
byte myByte = 96;
byte anotherByte = 7;

byte addedBytes = (byte) (myByte + anotherByte); // 103
byte subtractedBytes = (byte) (myBytes - anotherByte); // 89

Les valeurs maximales et minimales de byte peuvent être trouvées à :

byte high = Byte.MAX_VALUE;        // high == 127
byte low = Byte.MIN_VALUE;         // low == -128

La valeur par défaut d’un byte est 0

byte defaultByte;    // defaultByte == 0

Représentation des valeurs négatives #

Java et la plupart des autres langages stockent les nombres entiers négatifs dans une représentation appelée notation complément à 2.

Pour une représentation binaire unique d’un type de données utilisant n bits, les valeurs sont encodées comme ceci :

Les “n-1” bits les moins significatifs stockent un nombre entier positif “x” en représentation intégrale. La valeur la plus significative stocke un bit avec la valeur s. La valeur représentée par ces bits est

x - s * 2n-1

c’est-à-dire que si le bit le plus significatif est 1, alors une valeur juste supérieure de 1 au nombre que vous pourriez représenter avec les autres bits (2n-2 + 2n- 3 + … + 21 + 20 = 2n-1 - 1) est soustrait permettant une représentation binaire unique pour chaque valeur de - 2n-1 (s = 1 ; x = 0) à 2n-1 - 1 (s = 0 ; x = 2n-1 - 1).

Cela a également l’effet secondaire agréable que vous pouvez ajouter les représentations binaires comme s’il s’agissait de nombres binaires positifs :

v1 = x1 - s1 * 2n-1
v2 = x2 - s2 * 2n-1
s1 s2 débordement x1 + x2 résultat de l’addition
0 0 Non x1 + x2 = v1 + v2
0 0 Oui trop grand pour être représenté avec le type de données (débordement)
0 1 Non 
x1 + x2 - 2n-1 = x1 + x2 - s2 * 2n-1
 = v1 + v2
0 1 Oui 
(x1 + x2) mod 2n-1 = x1 + x2 - 2n-1
 = v1 + v2
1 0 * voir ci-dessus (sommes d’échange)
1 1 Non trop petit pour être représenté avec le type de données (x1 + x2 - 2n < -2n-1 ; sous-dépassement)
1 1 Oui 
(x1 + x2) mod 2n-1 - 2n-1 = (x1 + x2 - 2n -1) - 2n-1
 = (x1 - s1 * 2n-1) + (x2 - s2 * 2 n-1)
 = v1 + v2

La primitive courte #

Un short est un entier signé de 16 bits. Il a une valeur minimale de -215 (-32 768) et une valeur maximale de 215 ‑1 (32 767)

short example = -48;
short myShort = 987;
short anotherShort = 17;

short addedShorts = (short) (myShort + anotherShort); // 1,004
short subtractedShorts = (short) (myShort - anotherShort); // 970

Les valeurs maximales et minimales de short peuvent être trouvées à :

short high = Short.MAX_VALUE;        // high == 32767
short low = Short.MIN_VALUE;         // low == -32768

La valeur par défaut d’un short est 0

short defaultShort;    // defaultShort == 0
