Le shell exécute un programme en réponse à son invite. Lorsque vous donnez une commande, le shell recherche le programme, puis l’exécute. Par exemple, lorsque vous donnez la commande ls, le shell recherche l’utilitaire/programme nommé ls, puis l’exécute dans le shell. Les arguments et les options que vous fournissez avec les utilitaires peuvent avoir un impact sur le résultat que vous obtenez. Le shell est également appelé CLI ou interface de ligne de commande.

Modification du shell par défaut #

La plupart des distributions modernes seront livrées avec BASH (Bourne Again SHell) préinstallé et configuré comme shell par défaut.

La commande (en fait un binaire exécutable, un ELF) responsable du changement de shell sous Linux est chsh (change shell).

Nous pouvons d’abord vérifier quels shells sont déjà installés et configurés sur notre machine en utilisant la commande chsh -l, qui affichera un résultat similaire à ceci :

[user@localhost ~]$ chsh -l
/bin/sh
/bin/bash
/sbin/nologin
/usr/bin/sh
/usr/bin/bash
/usr/sbin/nologin
/usr/bin/fish

Dans certaines distributions Linux, chsh -l n’est pas valide. Dans ce cas, la liste de tous les shells disponibles se trouve dans le fichier /etc/shells. Vous pouvez afficher le contenu du fichier avec cat :

[user@localhost ~]$ cat /etc/shells
# /etc/shells: valid login shells
/bin/sh
/bin/bash
/sbin/nologin
/usr/bin/sh
/usr/bin/bash
/usr/sbin/nologin
/usr/bin/fish

Nous pouvons maintenant choisir notre nouveau shell par défaut, par ex. fish et configurez-le en utilisant chsh -s,

[user@localhost ~]$ chsh -s /usr/bin/fish
Changing shell for user.
Password: 
Shell changed.

Il ne reste plus qu’à effectuer un cycle déconnexion-connexion et profiter de notre nouveau shell par défaut.

Si vous souhaitez changer le shell par défaut pour un autre utilisateur et que vous avez des privilèges d’administration sur la machine, vous pourrez le faire en utilisant chsh comme root. Donc, en supposant que nous voulions changer le shell par défaut de user_2 en fish, nous utiliserons la même commande qu’auparavant, mais avec l’ajout du nom d’utilisateur de l’autre utilisateur, chsh -s /usr/bin/fish user_2.

Afin de vérifier quel est le shell par défaut actuel, nous pouvons afficher la variable d’environnement $SHELL, qui pointe vers le chemin de notre shell par défaut, donc après notre changement, nous nous attendrions à obtenir un résultat similaire à celui-ci,

 ~  echo $SHELL                                 
/usr/bin/fish

Options chsh : #

-s shell

Définit shell comme shell de connexion.

-l, --list-shells

Affichez la liste des shells répertoriés dans /etc/shells et quittez.

-h, --aide

Imprimez un message d’utilisation et quittez.

-v, --version

Imprimer les informations de version et quitter.

Utilitaires Shell de base #

Personnalisation de l’invite du Shell #

L’invite de commande par défaut peut être modifiée pour paraître différente et courte. Si le répertoire actuel est long, l’invite de commande par défaut devient trop grande. L’utilisation de PS1 devient utile dans ces cas. Une commande courte et personnalisée jolie et élégante. Dans le tableau ci-dessous, PS1 a été utilisé avec un certain nombre d’arguments pour afficher différentes formes d’invites du shell. L’invite de commande par défaut ressemble à ceci : user@host ~ $ dans mon cas, elle ressemble à ceci : bruce@gotham ~ $. Il peut être modifié selon le tableau ci-dessous :

Commande Utilité
PS1=’\w $ ' Invite du shell ~ $ comme nom de répertoire. Dans ce cas, le répertoire racine est Root.
PS1=’\h $ ' Invite du shell gotham $ comme nom d’hôte
PS1=’\u $ ' Invite du shell bruce $ comme nom d’utilisateur
PS1=’\t $ ' 22:37:31 $ invite du shell au format 24 heures
PS1=’@ $ ' 22h37 invite du shell au format 12 heures
PS1=’! $ ' 732 affichera le numéro d’historique de la commande à la place de l’invite du shell
PS1=‘mec $ ' dude $ affichera l’invite du shell comme vous le souhaitez

Quelques commandes shell de base #

Commande Utilité
Ctrl-k couper/tuer
Ctrl-y copier/coller
Ctrl-a amènera le curseur au début de la ligne
Ctrl-e amènera le curseur à la fin de la ligne
Ctrl-d supprimera le caractère après/au niveau du curseur
Ctrl-l effacera l’écran/le terminal
Ctrl-u effacera tout entre l’invite et le curseur
Ctrl-_ annulera la dernière chose tapée sur la ligne de commande
Ctrl-c interrompra/arrêtera le travail/processus en cours d’exécution au premier plan
Ctrl-r recherche inversée dans l’historique
~/.bash_history stocke les 500 dernières commandes/événements utilisés sur le shell
histoire affichera l’historique des commandes
historique | grep <mot-clé> affichera toutes les commandes de l’historique ayant le mot-clé <mot-clé> (utile dans les cas où vous vous souvenez d’une partie de la commande utilisée dans le passé)

Créez votre propre alias de commande #

Si vous en avez assez d’utiliser de longues commandes dans bash, vous pouvez créer votre propre alias de commande.

La meilleure façon de le faire est de modifier (ou de créer s’il n’existe pas) un fichier appelé .bash_aliases dans votre dossier personnel. La syntaxe générale est :

alias command_alias='actual_command'

où actual_command est la commande que vous renommez et command_alias est le nouveau nom que vous lui avez donné. Par exemple

alias install='sudo apt-get -y install'

mappe le nouvel alias de commande install à la commande réelle sudo apt-get -y install. Cela signifie que lorsque vous utilisez install dans un terminal, cela est interprété par bash comme sudo apt-get -y install.

Localisez un fichier sur votre système #

En utilisant bash, vous pouvez facilement localiser un fichier avec la commande locate. Par exemple, supposons que vous recherchiez le fichier mykey.pem :

locate mykey.pem

Parfois, les fichiers ont des noms étranges, par exemple vous pourriez avoir un fichier comme random7897_mykey_0fidw.pem. Supposons que vous recherchiez ce fichier mais que vous ne vous souveniez que des parties mykey et pem. Vous pouvez combiner la commande locate avec grep en utilisant un tube comme celui-ci :

locate pem | grep mykey

Ce qui ferait apparaître tous les résultats contenant ces deux éléments.

Notez que tous les systèmes n’ont pas l’utilitaire locate installé, et beaucoup ne l’ont pas activé. locate est rapide et efficace car il analyse périodiquement votre système et met en cache les noms et les emplacements de chaque fichier qu’il contient, mais si cette collecte de données n’est pas activée, il ne peut rien vous dire. Vous pouvez utiliser updatedb pour lancer manuellement l’analyse du système de fichiers afin de mettre à jour les informations mises en cache sur les fichiers de votre système de fichiers.

Si vous n’avez pas de “locate” fonctionnel, vous pouvez vous rabattre sur l’utilitaire “find” :

find / -name mykey.pem -print

est à peu près équivalent à locate mykey.pem mais doit analyser votre ou vos systèmes de fichiers chaque fois que vous l’exécutez pour le fichier en question, plutôt que d’utiliser des données mises en cache. C’est évidemment plus lent et moins efficace, mais plus en temps réel. L’utilitaire “find” peut faire bien plus que rechercher des fichiers, mais une description complète de ses capacités dépasse le cadre de cet exemple.
