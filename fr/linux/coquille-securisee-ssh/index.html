Coquille sécurisée (SSH)

Un [shell] sécurisé (https://www.wikiod.com/fr/linux/coquille) est utilisé pour accéder à distance à un serveur depuis un client via une connexion cryptée. OpenSSH est utilisé comme alternative aux connexions Telnet qui permettent un accès distant au shell mais qui ne sont pas chiffrées. Le client OpenSSH est installé par défaut sur la plupart des distributions GNU/Linux et est utilisé pour se connecter à un serveur. Ces exemples montrent comment utiliser la suite SSH pour accepter les connexions SSH et se connecter à un autre hôte.

Connexion à un serveur distant #

Pour se connecter à un serveur, nous devons utiliser SSH sur le client comme suit,

# ssh -p port user@server-address
  • port - Le port ssh d’écoute du serveur (port 22 par défaut).
  • utilisateur - Doit être un utilisateur existant sur le serveur avec des privilèges SSH.
  • adresse du serveur - L’IP/Domaine du serveur.

Pour un exemple concret, supposons que vous créez un site Web. La société que vous avez choisie pour héberger votre site vous indique que le serveur est situé sur web-servers.com sur un port personnalisé de 2020 et que votre nom de compte usr1 a été choisi pour créer un utilisateur sur le serveur avec des privilèges SSH. Dans ce cas, la commande SSH utilisée serait telle quelle

# ssh -p 2020 usr1@web-servers.com

Si le nom de compte sur le système distant est le même que celui du client local, vous pouvez laisser le nom d’utilisateur de côté. Donc, si vous êtes usr1 sur les deux systèmes, vous pouvez simplement utiliser web-servers.com au lieu de usr1@web-servers.com.

Lorsqu’un serveur auquel vous souhaitez vous connecter ne vous est pas directement accessible, vous pouvez essayer d’utiliser le commutateur ProxyJump pour vous y connecter via un autre serveur auquel vous avez accès et qui peut se connecter au serveur souhaité.

# ssh -J usr1@10.0.0.1:2020 usr2@10.0.0.2 -p 2222

Cela vous permettra de vous connecter au serveur 10.0.0.2 (exécutant ssh sur le port 2222) via le serveur 10.0.0.1 (exécutant ssh sur le port 2020). Vous devrez bien sûr avoir des comptes sur les deux serveurs. Notez également que le commutateur -J est introduit dans OpenSSH version 7.3.

Installation de la suite OpenSSH #

La connexion à un serveur SSH supprimé et l’acceptation des connexions SSH nécessitent l’installation de openssh

Debian :

# apt-get install openssh

Arch Linux :

# pacman -S openssh

Miam:

# yum install openssh

Générer une clé publique et privée #

Pour générer des clés pour le client SSH :

ssh-keygen [-t rsa | rsa1 | dsa ] [-C <comment>] [-b bits]

Par exemple:

ssh-keygen -t rsa -b 4096 - C myemail@email.com

L’emplacement par défaut est ~/.ssh/id_rsa pour la clé privée et ~/.ssh/id_rsa.pub pour la clé publique.

Pour plus d’informations, veuillez visiter [man.openbsd.org][1]

[1] : http://man.openbsd.org/OpenBSD-current/man1/ssh-keygen.1

Configuration d’un serveur SSH pour accepter les connexions #

Nous devons d’abord modifier le fichier de configuration du démon SSH. Bien que sous différentes distributions Linux, cela puisse être situé dans différents répertoires, il est généralement stocké sous /etc/ssh/sshd_config

Utilisez votre éditeur de texte pour modifier les valeurs définies dans ce fichier, toutes les lignes commençant par # sont commentées et doivent avoir ce caractère supprimé pour prendre effet. Une liste de recommandations suit en tant que telle.

Port (chose a number between 0 - 65535, normaly greater than four digits)
PasswordAuthentication yes
AllowUsers    user1 user2 ...etc

Notez qu’il est préférable de désactiver toutes les connexions par mot de passe et d’utiliser les clés SSH pour une sécurité améliorée, comme expliqué dans ce document.

Connexion sans mot de passe (à l’aide d’une paire de clés) #

Tout d’abord, vous aurez besoin d’une paire de clés. Si vous n’en avez pas encore, consultez le sujet “Générer une clé publique et une clé privée”.

Votre paire de clés est composée d’une clé privée (id_rsa) et d’une clé publique (id_rsa.pub). Tout ce que vous avez à faire est de copier la clé publique sur l’hôte distant et d’ajouter son contenu au fichier ~/.ssh/authorized_keys.

Une façon simple de le faire est :

ssh <user>@<ssh-server> 'cat >> ~/.ssh/authorized_keys' < id_rsa.pub

Une fois la clé publique correctement placée dans le répertoire personnel de votre utilisateur, il vous suffit de vous connecter à l’aide de la clé privée correspondante :

ssh <utilisateur>@<serveur-ssh> -i id_rsa

Désactiver le service ssh #

Cela désactivera le service côté serveur SSH, car si nécessaire, cela garantira que les clients ne peuvent pas se connecter via ssh

Ubuntu

sudo service ssh stop

Debian

sudo /etc/init.d/ssh stop

Arch Linux

sudo killall sshd
