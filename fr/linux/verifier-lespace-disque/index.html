Vérification de l’espace disque #

Il est assez courant de vouloir vérifier l’état des différentes partitions/lecteurs sur votre serveur/ordinateur pour voir à quel point ils sont pleins. La commande suivante est celle que vous voudrez exécuter :

df -h

Cela produira une sortie semblable à la suivante :

[root@mail ~]# df -h
Filesystem            Size  Used Avail Use% Mounted on
/dev/mapper/VolGroup-lv_root
                       19G  1.6G   16G   9% /
tmpfs                 245M     0  245M   0% /dev/shm
/dev/sda1             485M   47M  413M  11% /boot

Dans cet exemple de base, nous pouvons voir que la partition / n’a que 9% utilisé.

Pour un exemple plus complexe qui couvre également l’utilisation de df pour voir divers points de montage, voir ci-dessous :

[root@mail ~]# df -h
Filesystem            Size  Used Avail Use% Mounted on
/dev/mapper/VG-root   1.9T  1.7T   89G  95% /
/dev/mapper/VG-var    431G  145G  264G  36% /var
devtmpfs              7.8G  204K  7.8G   1% /dev
tmpfs                 7.8G  4.0K  7.8G   1% /dev/shm
/dev/md1              495M  126M  344M  27% /boot
ku.example.com:9421   2.5T  487G  2.0T  20% /mnt/test
tmpfs                 500M   86M  415M  18% /var/ngx_pagespeed_cache

Dans cet exemple, nous avons une partition / qui est pleine à 95% avec une partition /var supplémentaire qui n’est pleine qu’à 36%.

Il a un montage réseau externe de 2T qui est monté sur /mnt/test et un montage ramdisk/tmpfs de 500M monté sur /var/ngx_pagespeed_cache.

Enquêter sur les répertoires pour l’utilisation du disque #

Parfois, il peut être nécessaire de savoir quel répertoire consomme combien d’espace disque, en particulier lorsque vous utilisez df -h et que vous réalisez que votre espace disque disponible est faible.

du:

La commande du résume l’utilisation du disque de l’ensemble de FILE, récursivement pour les répertoires.

Il est souvent utilisé avec l’option -sh :

-s, --summarize
              display only a total for each argument
-h, --human-readable
              print sizes in human readable format (e.g., 1K 234M 2G)

Pour résumer les utilisations du disque des fichiers dans le répertoire courant, nous utilisons :

du -sh *

Exemple de sortie :

572K    Documents
208M    Downloads
4,0K    Music
724K    Pictures
4,0K    Public
4,0K    Templates
4,0K    Videos

Nous pouvons également inclure des fichiers cachés en utilisant :

du -sh .[!.]* *

Exemple de sortie :

6,3M    .atom
4,0K    .bash_history
4,0K    .bash_logout
8,0K    .bashrc
350M    .cache
195M    .config
12K    .dbus
4,0K    .dmrc
44K    .gconf
60K    .gem
520K    .gimp-2.8
28K    .gnome
4,0K    .ICEauthority
8,3M    .local
8,0K    .nano
404K    .nv
36K    .pki
4,0K    .profile
8,0K    .ssh
0    .sudo_as_admin_successful
4,0K    .Xauthority
4,0K    .xsession-errors
4,0K    .xsession-errors.old
572K    Documents
208M    Downloads
4,0K    Music
724K    Pictures
4,0K    Public
4,0K    Templates

Troisièmement, vous pouvez ajouter total à la sortie en ajoutant ,-c, option :

du -sch .[!.]* *

Résultat:

.
.
.
4,0K    Templates
4,0K    Videos
769M    total

Plus important encore, l’utilisation correcte de la commande du sur le répertoire racine est une action salvatrice pour savoir quelle application/service ou utilisateur consomme énormément d’espace disque. Par exemple, dans le cas d’un niveau ridiculement bas de disponibilité d’espace disque pour un serveur Web et de messagerie, la raison pourrait être une attaque de spam sur votre service de messagerie et vous pouvez le diagnostiquer simplement en utilisant la commande du.

Examinez le répertoire racine pour l’utilisation du disque :

sudo du -sch /.[!.]* /*

Exemple de sortie :

du: cannot access '/proc/18221/fdinfo/4': No such file or directory

Enfin, la meilleure méthode se forme lorsque vous ajoutez une valeur de taille de seuil pour que les répertoires ignorent les plus petits. Cette commande n’affichera que les dossiers de plus de 1 Go situés sous le répertoire racine jusqu’à la branche la plus éloignée de toute l’arborescence de répertoires de votre système de fichiers :

sudo du --threshold=1G -ch /.[!.]* /*

Exemple de sortie :

1,4G    /usr/lib
1,8G    /usr/share
3,5G    /usr
5,8G    total
