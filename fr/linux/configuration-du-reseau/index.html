Ce document couvre les bases de la mise en réseau TCP/IP, de l’administration réseau et de la configuration système. Linux peut prendre en charge plusieurs périphériques réseau. Les noms d’appareils sont numérotés et commencent à zéro et comptent vers le haut. Par exemple, un ordinateur avec deux cartes réseau aura deux périphériques étiquetés eth0 et eth1.

Détails de l’interface #

Siconfig

Lister toutes les interfaces disponibles sur la machine

$ ifconfig -a

Lister les détails d’une interface spécifique

Syntaxe : $ ifconfig <interface>

** ip - afficher / manipuler le routage, les périphériques, le routage de la politique et les tunnels **

Syntaxe : $ ip { lien | ... | itinéraire | macsec } (veuillez consulter man ip pour la liste complète des objets)

Exemples

Lister les interfaces réseau

$ ip link show

Renommez l’interface eth0 en wan

$ ip link set dev eth0 name wan

Amener l’interface eth0 vers le haut (ou vers le bas)

$ ip link set dev eth0 up

Liste des adresses pour les interfaces

$ ip addr show

Ajouter (ou supprimer) l’ip et le masque (255.255.255.0)

$ ip addr add 1.2.3.4/24 brd + dev eth0

Ajouter une IP à une interface #

Une adresse IP à une interface peut être obtenue via DHCP ou une affectation statique

DHCP Si vous êtes connecté à un réseau avec un serveur DHCP en cours d’exécution, la commande dhclient peut obtenir une adresse IP pour votre interface

$ dhclient <interface>

ou alternativement, vous pouvez apporter une modification au fichier /etc/network/interfaces pour que l’interface soit affichée au démarrage et obtenir l’IP DHCP

auto eth0
iface eth0 inet dhcp

Configuration statique (changement permanent) à l’aide du fichier /etc/network/interfaces

Si vous souhaitez configurer statiquement les paramètres de l’interface (changement permanent), vous pouvez le faire dans le fichier /etc/network/interfaces.

Exemple:

auto eth0 # Bring up the interface on boot
iface eth0 inet static 
    address 10.10.70.10
    netmask 255.255.0.0
    gateway 10.10.1.1
    dns-nameservers 10.10.1.20
    dns-nameservers 10.10.1.30

Ces modifications persistent même après le redémarrage du système.

**Configuration statique (changement temporaire) à l’aide de l’utilitaire ifconfig **

Une adresse IP statique peut être ajoutée à une interface à l’aide de l’utilitaire “ifconfig” comme suit

$ ifconfig <interface> <adresse-ip>/<masque> up

Exemple:

$ ifconfig eth0 10.10.50.100/16 up

Résolution DNS locale #

Fichier : /etc/hosts contient une liste d’hôtes qui doivent être résolus localement (pas par DNS)

Exemple de contenu du fichier :

127.0.0.1         your-node-name.your-domain.com  localhost.localdomain  localhost 
XXX.XXX.XXX.XXX   node-name

Le format de fichier pour le fichier hosts est spécifié par [RFC 952][1]

[1] : http://www.ietf.org/rfc/rfc0952.txt

Configurer les serveurs DNS pour la résolution des noms de domaine #

Fichier : /etc/resolv.conf contient une liste de serveurs DNS pour la résolution de noms de domaine

Exemple de contenu du fichier :

nameserver 8.8.8.8 # IP address of the primary name server
nameserver 8.8.4.4 # IP address of the secondary name server

Dans le cas d’un serveur DNS interne, vous pouvez valider si ce serveur résout correctement les noms DNS à l’aide de la commande dig :

$ dig google.com @your.dns.server.com +short

Voir et manipuler les itinéraires #

Manipuler la table de routage IP en utilisant route #

Afficher la table de routage

$ route # Displays list or routes and also resolves host names
$ route -n # Displays list of routes without resolving host names for faster results

Ajouter/Supprimer un itinéraire

Options Descriptif
ajouter ou supprimer Ajouter ou supprimer un itinéraire
-hôte x.x.x.x Ajouter une route à un seul hôte identifié par l’adresse IP
-net x.x.x.x Ajouter une route à un réseau identifié par l’adresse réseau
gw x.x.x.x Spécifiez la passerelle réseau
masque de réseau x.x.x.x Spécifiez le masque de réseau
par défaut Ajouter une route par défaut

Exemples:

  • ajouter une route à un hôte $ route add -host x.x.x.x eth1
  • ajouter une route à un réseau $ route add -net 2.2.2.0 netmask 255.255.255.0 eth0
  • Alternativement, vous pouvez également utiliser le format cidr pour ajouter une route au réseau route add -net 2.2.2.0/24 eth0
  • ajouter la passerelle par défaut $ route add default gw 2.2.2.1 eth0
  • supprimer une route $ route del -net 2.2.2.0/24

Manipuler la table de routage IP en utilisant ip #

Afficher la table de routage

$ ip route show # List routing table

Ajouter/Supprimer un itinéraire

Options Descriptif
add ou del ou change ou append ou replace Modifier un itinéraire
montrer ou vider la commande affiche le contenu des tables de routage ou le supprime
restaurer restaurer les informations de la table de routage à partir de stdin
obtenir cette commande obtient une seule route vers une destination et imprime son contenu exactement comme le noyau le voit

Exemples:

  • Définissez la passerelle par défaut sur 1.2.3.254 $ ip route add default via 1.2.3.254
  • Ajoute une route par défaut (pour toutes les adresses) via la passerelle locale 192.168.1.1 accessible sur l’appareil eth0 $ ip route add default via 192.168.1.1 dev eth0

Configurez un nom d’hôte pour un autre système sur votre réseau #

Vous pouvez configurer votre système Linux (ou macOS) afin de lier un identifiant <hostname> à l’adresse IP d’un autre système de votre réseau. Vous pouvez le configurer :

  • À l’échelle du système. Vous devez modifier le fichier /etc/hosts. Il suffit d’ajouter à ce fichier une nouvelle ligne contenant :

    1. the remote system’s IP address <ip_rem>,
    2. one or more blank spaces, and
    3. the identifier <hostname>.

  • Pour un seul utilisateur. Vous devez modifier le fichier ~/.hosts — vous devrez le créer. Ce n’est pas aussi simple que pour l’ensemble du système. [Ici][hostname-s-user] vous pouvez voir une explication.

[nom d’hôte-s-utilisateur] : http://unix.stackexchange.com/questions/10438/can-i-create-a-user-specific-hosts-file-to-complement-etc-hosts

Par exemple, vous pouvez ajouter cette ligne à l’aide de l’outil Unix [cat][cat-add-line]. Supposons que vous vouliez envoyer un ping à un PC de votre réseau local dont l’adresse IP est 192.168.1.44 et que vous vouliez faire référence à cette adresse IP simplement par remote_pc. Ensuite, vous devez écrire sur votre shell :

[cat-add-line] : https://en.wikipedia.org/wiki/Cat_(Unix)

$ sudo cat 192.168.1.44 remote_pc

Ensuite, vous pouvez faire ce ping simplement en :

$ ping remote_pc
