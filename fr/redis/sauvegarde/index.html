La sauvegarde d’une instance Redis distante peut être réalisée avec la réplication. Ceci est utile si vous souhaitez prendre un instantané d’un ensemble de données avant de mettre à niveau, de supprimer ou de modifier une base de données Redis.

Sauvegarde d’une instance Redis distante vers une instance locale #

Sur la machine sur laquelle vous souhaitez effectuer la sauvegarde, accédez à la CLI Redis :

redis-cli

Mot de passe? #

Si votre base de données Redis maître (celle que vous souhaitez répliquer) a un mot de passe :

config set masterauth <password>

Démarrer la réplication #

Exécutez la commande suivante pour commencer la réplication :

SLAVEOF <host> <port>

Pour vérifier que la réplication est en cours, exécutez :

INFO replication

Et vous devriez voir une sortie comme celle-ci :

# Replication
role:slave
master_host:some-host.compute-1.amazonaws.com
master_port:6519
master_link_status:up
master_last_io_seconds_ago:3
master_sync_in_progress:0
slave_repl_offset:35492914
slave_priority:100
slave_read_only:1
connected_slaves:0
master_repl_offset:0
repl_backlog_active:0
repl_backlog_size:1048576
repl_backlog_first_byte_offset:0
repl_backlog_histlen:0

Notez que le master_link_status doit être up.

Vérification de la progression de la synchronisation #

Lorsque la synchronisation est terminée, la réplication INFO doit afficher :

master_sync_in_progress:0

Pour vérifier que l’ensemble de données a été synchronisé, vous pouvez comparer la taille de la base de données :

DBSIZE

Enregistrer un vidage de données sur le disque #

Pour enregistrer la base de données sur disque de manière asynchrone :

BGSAVE
CONFIG GET dir

Ensuite, vous devriez trouver un fichier dump.rdb dans le répertoire indiqué par la commande config.

Arrêt de la réplication #

Vous pouvez arrêter la réplication avec :

SLAVEOF NO ONE

Référence : [Guide de réplication Redis][1]

[1] : https://redis.io/topics/replication
