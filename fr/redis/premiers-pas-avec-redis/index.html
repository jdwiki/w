Interface de ligne de commande Redis #

redis-cli est le programme d’interface de ligne de commande Redis qui permet d’envoyer des commandes à Redis et de lire les réponses envoyées par le serveur, directement depuis le terminal. L’utilisation de base de la ligne de commande est ci-dessous :

Accès à redis :

$ redis-cli
127.0.0.1:6379>

Accès à redis avec authentification :

$ redis-cli -a myPassword
127.0.0.1:6379>

Sélectionnez la base de données et affichez la taille de la base de données (le numéro de base de données par défaut est 0) :

127.0.0.1:6379> dbsize
(integer) 2
127.0.0.1:6379> select 1
OK
127.0.0.1:6379[1]> dbsize
(integer) 20

Installer Redis en utilisant Docker #

Il est simple de commencer à utiliser Redis en utilisant docker :

docker pull redis
docker run -p 6379:6379 --rm --name redis redis

Vous avez maintenant une instance en cours d’exécution sur le port “6397”

Attention : Toutes les données seront supprimées lorsque Redis sera arrêté.

Pour connecter le redis-cli, démarrez un autre docker :

docker run -it --link redis:redis --rm redis redis-cli -h redis -p 6379

Vous pouvez maintenant jouer avec votre docker Redis.

Aperçu #

Redis est une base de données distante en mémoire qui offre des performances élevées, une réplication et un modèle de données unique pour produire une plate-forme de résolution de problèmes. Redis est une structure de données en mémoire open source (sous licence BSD), utilisée comme base de données, cache et courtier de messages. Il est classé comme un magasin clé-valeur NoSQL. Il prend en charge les structures de données telles que les chaînes, les hachages, les listes, les ensembles, les ensembles triés avec des requêtes de plage, les bitmaps, les hyperloglogs et les index géospatiaux avec des requêtes de rayon. Prise en charge de cinq types différents de structures de données,

  1. STRING (Opérer sur toute la chaîne, les parties, les entiers et les flottants)
  2. LISTE (Pousser ou faire apparaître des éléments des deux extrémités)
  3. SET (Ajouter, récupérer, supprimer, vérifier, intersection, union, différence, etc.)
  4. HASH (stocker, extraire, supprimer dans le hachage)
  5. ZSET (identique à l’ensemble mais de manière ordonnée)
  6. GEO (Ajouter, mettre à jour, supprimer la latitude et la longitude, obtenir un redius donné)

Redis intègre la réplication, les scripts Lua, l’éviction LRU, les transactions et différents niveaux de persistance sur disque (sync/async).

Avant la version 3, Redis fonctionnait en mode maître-esclave et exigeait que Redis-Sentinel fournisse une haute disponibilité. Seul le maître accepte les écritures et synchronise les données avec ses esclaves par fork.

À partir de la version 3, Redis fonctionne et recommande le mode multi-maître où les fonctionnalités de basculement, de partitionnement/partitionnement et de repartitionnement sont intégrées. Redis-Sentinel n’est plus requis à partir de la version-3. Pour que le cluster Redis fonctionne, un minimum de 3 nœuds/processus maîtres est requis.

Les fonctionnalités supplémentaires sont la réplication, la persistance et le partage côté client. Redis s’adapte à une grande variété de problèmes qui peuvent être naturellement mappés dans ce qu’offre Redis, vous permettant de résoudre vos problèmes sans avoir à effectuer le travail conceptuel requis par d’autres bases de données.

Vous devez d’abord installer et démarrer votre serveur Redis, consultez le lien ci-dessous qui peut vous aider à installer Redis sur votre serveur ou votre machine locale.

Ouvrez maintenant votre invite de commande et exécutez la commande redis-cli :

Pour enregistrer le premier ensemble> SET ‘keyname’ puis ‘value’

127.0.0.1:6379> SET hkey "Hello World!"

Appuyez sur Entrée, vous devriez voir

OK

Saisissez ensuite :

GET hkey

tu devrais voir:

"Hello World!"

Installation Redis sur Windows, avec exemple Node.js #

Redis dispose d’un port Windows fourni par ‘Microsoft Open Technologies’. Vous pouvez utiliser le programme d’installation msi trouvé sur : https://github.com/MSOpenTech/redis/releases

Une fois l’installation terminée, vous pouvez voir que “Redis” est un service Windows (et son statut doit être “Démarré”)

Pour écrire un exemple ‘Hello world’ qui utilise Redis dans Node.js (également dans Windows), vous pouvez utiliser le module npm suivant : https://www.npmjs.com/package/redis

exemple de code :

var redis = require('redis'),
    client = redis.createClient();

client.set('mykey', 'Hello World');
client.get('mykey', function(err,res){
    console.log(res);
});
