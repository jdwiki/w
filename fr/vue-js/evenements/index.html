Syntaxe des événements #

Pour envoyer un événement : vm.$emit('new-message');

Pour intercepter un événement : vm.$on('new-message');

Pour envoyer un événement à tous les composants down : vm.$broadcast('new-message');

Pour envoyer un événement à tous les composants up : vm.$dispatch('new-message');

Remarque : $broadcast et $dispatch sont obsolètes dans Vue2. (voir les fonctionnalités de Vue2)

Quand dois-je utiliser les événements ? #

L’image suivante illustre le fonctionnement de la communication des composants. L’image provient de The Progressive Framework diapositives de [Evan You](https://twitter.com /youyuxi) (Développeur de VueJS).

[![Communication des composants][1]][1]

Voici un exemple de comment cela fonctionne :

DÉMO

HTML

<script type="x-template" id="message-box">
    <input type="text" v-model="msg" @keyup="$emit('new-message', msg)" />
</script>
 
<message-box :msg="message" @new-message="updateMessage"></message-box>
<div>You typed: {{message}}</div>

JS

var messageBox = {
  template: '#message-box',
  props: ['msg']
};
 
new Vue({
  el: 'body',
  data: {
    message: ''
  },
  methods: {
    updateMessage: function(msg) {
      this.message = msg;
    }
  },
  components: {
    'message-box': messageBox
  }
});

L’exemple ci-dessus peut être amélioré ! #

L’exemple ci-dessus montre comment fonctionne la communication des composants. Mais dans le cas d’un composant d’entrée personnalisé, pour synchroniser la variable parent avec la valeur saisie, nous devrions utiliser v-model.

DEMO Vue1

DEMO Vue2

Dans Vue1, vous devez utiliser .sync sur le prop envoyé au composant <message-box>. Cela indique à VueJS de synchroniser la valeur du composant enfant avec celle du parent.

N’oubliez pas : Chaque instance de composant a sa propre portée isolée.

Vue HTML1

<script type="x-template" id="message-box">
  <input v-model="value" />
</script>

<div id="app">
  <message-box :value.sync="message"></message-box>
  <div>You typed: {{message}}</div>
</div>

Dans Vue2, il y a un événement spécial 'input' que vous pouvez $emit. L’utilisation de cet événement vous permet de mettre un v-model directement sur le composant <message-box>. L’exemple ressemblera à ceci :

Vue HTML2

<script type="x-template" id="message-box">
  <input :value="value" @input="$emit('input', $event.target.value)" />
</script>

<div id="app">
  <message-box v-model="message"></message-box>
  <div>You typed: {{message}}</div>
</div>

JS Vue 1 & 2

var messageBox = {
  template: '#message-box',
  props: ['value']
};

new Vue({
  el: '#app',
  data: {
    message: ''
  },
  components: {
    'message-box': messageBox
  }
});

Remarquez à quelle vitesse l’entrée est mise à jour.

Comment gérer la dépréciation de $dispatch et $broadcast ? (modèle d’événement de bus) #

Vous avez peut-être réalisé que $emit est limité au composant qui émet l’événement. C’est un problème lorsque vous souhaitez communiquer entre des composants éloignés les uns des autres dans l’arborescence des composants.

Remarque : Dans Vue1, vous pouvez utiliser $dispatch ou $broadcast, mais pas dans Vue2. La raison étant qu’il ne s’adapte pas bien. Il existe un modèle de “bus” populaire pour gérer cela :

DÉMO

HTML

<script type="x-template" id="sender">
  <button @click="bus.$emit('new-event')">Click me to send an event !</button>
</script>
 
<script type="x-template" id="receiver">
  <div>I received {{numberOfEvents}} event{{numberOfEvents == 1 ? '' : 's'}}</div>
</script>
 
<sender></sender>
<receiver></receiver>

JS

var bus = new Vue();
 
var senderComponent = {
  template: '#sender',
  data() {
    return {
      bus: bus
    }
  }
};
 
var receiverComponent = {
  template: '#receiver',
  data() {
    return {
      numberOfEvents: 0
    }
  },
  ready() {
    var self = this;
 
    bus.$on('new-event', function() {
      ++self.numberOfEvents;
    });
  }
};
 
new Vue({
  el: 'body',
  components: {
    'sender': senderComponent,
    'receiver': receiverComponent
  }
});

Vous devez juste comprendre que n’importe quelle instance de Vue() peut $émettre et attraper ($on) un événement. Nous déclarons simplement un appel d’instance global Vue bus, puis tout composant avec cette variable peut émettre et intercepter des événements à partir de celle-ci. Assurez-vous simplement que le composant a accès à la variable “bus”.
