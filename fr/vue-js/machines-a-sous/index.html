Important! Les emplacements après le rendu ne garantissent pas l’ordre des positions pour les emplacements. Slot, qui était le premier, peut avoir une position différente après le rendu.

Utiliser des slots nommés #

Les emplacements nommés fonctionnent de la même manière que les emplacements uniques, mais vous permettent à la place de distribuer du contenu dans différentes régions au sein de votre modèle de composant enfant.

Prenez le composant page de l’exemple précédent mais modifiez son modèle pour qu’il soit comme suit :

<html>
    <head>
        <title>Page Title</title>
    </head>
    <body>
        <aside>
            <slot name="sidebar"></slot>
        </aside>
        <main>
            <slot name="content"></slot>
        </main>
    </body>
</html>

Lors de l’utilisation du composant page, nous pouvons maintenant déterminer où le contenu est placé via l’attribut slot :

<page>
    <p slot="sidebar">This is sidebar content.</p>
    <article slot="content"></article>
</page>

La page résultante sera :

<html>
    <head>
        <title>Page Title</title>
    </head>
    <body>
        <aside>
            <p>This is sidebar content.</p>
        </aside>
        <main>
            <article></article>
        </main>
    </body>
</html>

Si un slot est défini sans attribut name, tout contenu placé dans des balises de composant ne spécifiant pas d’attribut slot sera placé dans cet attribut.

Voir l’exemple d’insertion multiple sur la documentation officielle de Vue.js.

Utilisation d’emplacements uniques #

Les emplacements uniques sont utilisés lorsqu’un composant enfant ne définit qu’un seul “emplacement” dans son modèle. Le composant page ci-dessus utilise un seul emplacement pour distribuer le contenu.

Un exemple de modèle de composant page utilisant un seul emplacement est présenté ci-dessous :

<html>
    <head>
        <title>Page Title</title>
    </head>
    <body>
        <slot>
            This will only be displayed if there is no content
            to be distributed.
        </slot>
    </body>
</html>

Pour illustrer le fonctionnement de la machine à sous, nous pouvons configurer une page comme suit.

<page>
    <p>This content will be displayed within the page component</p>
</page>

Le résultat final sera :

<html>
    <head>
        <title>Page Title</title>
    </head>
    <body>
        <p>This content will be displayed within the page component</p>
    </body>
</html>

Si nous ne mettions rien entre les balises page et que nous avions à la place <page></page>, nous obtiendrions plutôt le résultat suivant car il y a un contenu par défaut entre les balises slot dans le modèle de composant page .

<html>
    <head>
        <title>Page Title</title>
    </head>
    <body>
        This will only be displayed if there is no content
        to be distributed.
    </body>
</html>

Qu’est-ce que les machines à sous ? #

Les emplacements offrent un moyen pratique de distribuer le contenu d’un composant parent à un composant enfant. Ce contenu peut être du texte, du HTML ou même d’autres composants.

Il peut parfois être utile de considérer les slots comme un moyen d’injecter du contenu directement dans le modèle d’un composant enfant.

Les emplacements sont particulièrement utiles lorsque la composition du composant sous le composant parent n’est pas toujours la même.

Prenons l’exemple suivant où nous avons un composant page. Le contenu de la page peut changer selon que cette page affiche par ex. un article, un article de blog ou un formulaire.

Article

<page>
    <article></article>
    <comments></comments>
</page>

Article de blog

<page>
    <blog-post></blog-post>
    <comments></comments>
</page>

Formulaire

<page>
    <form></form>
</page>

Remarquez comment le contenu du composant page peut changer. Si nous n’utilisions pas d’emplacements, cela serait plus difficile car la partie interne du modèle serait fixe.

Rappel : “Tout ce qui se trouve dans le modèle parent est compilé dans la portée parent ; tout dans le modèle enfant est compilé dans la portée enfant.”

Utilisation de slots dans Vue JSX avec ‘babel-plugin-transform-vue-jsx’ #

Si vous utilisez VueJS2 et que vous aimez utiliser JSX avec lui. Dans ce cas, pour utiliser le slot, la solution avec exemple est ci-dessous. Nous devons utiliser this.$slots.default C’est presque comme this.props.children dans React JS.

Composant.js :

export default {
    render(h) {  //eslint-disable-line
        return (
            <li>
                 { this.$slots.default }
            </li>
        );
    }
};

ParentComponent.js

import Component from './Component';

export default {
    render(h) {  //eslint-disable-line
        return (
            <ul>
                 <Component>
                     Hello World
                 </Component>
            </ul>
        );
    }
};
