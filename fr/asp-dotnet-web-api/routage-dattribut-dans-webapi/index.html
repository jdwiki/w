Routage d'attribut dans WebAPI

  • Syntaxe

Sur cette page

  • Syntaxe

  • Routage d’attribut de base

Comme son nom l’indique, cela utilise des attributs pour router. Cela donne à l’utilisateur plus de contrôle sur les URI dans l’API Web. Par exemple, vous pouvez décrire les hiérarchies de la ressource. Cependant, l’ancien “routage conventionnel” est entièrement pris en charge. Les utilisateurs peuvent également avoir un mélange des deux.

Syntaxe #

  • [RoutePrefix(“api/books”)] - pour la classe contrôleur
  • [Route(“getById”)] - pour les actions
  • [Route("~/api/authors/{authorId:int}/books")] - pour remplacer le préfixe de route

Paramètres #

Nom du paramètre Détails
Préfixe d’itinéraire attribut à la classe contrôleur. tous les préfixes d’URL courants dans les actions sont matraqués ici. prend la chaîne en entrée
Itinéraire attribut aux actions du contrôleur. chaque action aura un itinéraire associé à (pas nécessairement)
Route("~/api/") ceci remplace le préfixe d’itinéraire

Actuellement, Attribute Routes n’a pas de gestionnaires de messages spécifiques au contrôleur. Comme il n’y a aucun moyen de spécifier quel gestionnaire exécuter pour quelle route au moment de la déclaration. Ceci est possible dans le “routage conventionnel”.

Routage d’attribut de base #

Ajoutez simplement un attribut à l’action du contrôleur

[Route("product/{productId}/customer")]
public IQueryable<Product> GetProductsByCustomer(int productId) 
{ 
    //action code goes here 
}

cela sera interrogé comme /product/1/customer et productId=1 sera envoyé à l’action du contrôleur.

Assurez-vous que celui entre ‘{ }’ et le paramètre d’action sont identiques. productId dans ce cas.

avant de l’utiliser, vous devez spécifier que vous utilisez le routage d’attributs en :

public static class WebApiConfig
{
    public static void Register(HttpConfiguration config)
    {
        config.MapHttpAttributeRoutes();
    }
}

## Attribut de préfixe de routage Dans les cas où vous avez besoin d’une partie commune de la route pour toutes les routes d’un contrôleur, l’attribut [RoutePrefix][1] est utilisé.

Dans l’exemple ci-dessous, la partie api/students du code est commune et nous pouvons donc définir RoutePrefix et éviter de l’utiliser à plusieurs reprises.

[RoutePrefix("api/students")]
public class StudentController : ApiController
{
[Route("")]
public IEnumerable<Student> Get() 
{
    //action code goes here 
}

[Route("{id:int}")]
public Student Get(int id) 
{
    //action code goes here 
}

[Route("")]
public HttpResponseMessage Post(Student student) 
{
    //action code goes here 
}

}

[1] : https://msdn.microsoft.com/en-us/library/system.web.http.routeprefixattribute(v=vs.118).aspx
