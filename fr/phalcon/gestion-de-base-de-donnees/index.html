Gestion de la base de données à l’aide du modèle Phalcon #

Un modèle pour une nouvelle table peut être créé en exécutant la commande suivante à partir de l’emplacement racine du terminal :

phalcon model <table-name>

Prenons les utilisateurs modèles.

SÉLECTIONNER

Il y a deux fonctions par défaut pour sélectionner l’opération dans phalcon, find() et findFirst()

findFirst() est utilisé pour obtenir la première ligne qui satisfait les conditions que nous passons. Il renvoie un seul objet avec les données de la première ligne.

Exemple:

$user = Users::findFirst("active=1 AND verified=1 AND email='a@a.com'");

Cela renvoie l’utilisateur avec l’e-mail donné et la valeur de la colonne vérifiée et active est 1

find() est utilisé pour obtenir toutes les lignes qui satisfont aux conditions que nous passons.

Exemple:

$users = Users::find("active=1 AND verified=1");

Cela renvoie les utilisateurs avec la valeur de la colonne vérifiée et active est 1

INSÉRER

L’insertion peut être effectuée à l’aide du code suivant :

$user = new Users();

$user->name = "Arun";
$user->email = "abc@gmail.com";
$user->verified = 1;
$user->active = 1;

$user->save();

Une nouvelle ligne avec ces valeurs sera insérée.

METTRE À JOUR

La mise à jour peut être effectuée à l’aide du code suivant :

Nous devons d’abord sélectionner la ligne que nous devons mettre à jour en utilisant findFirst()

$user = Users::findFirst("email='a@a.com'");

$user->verified = 0;
$user->active = 0;

$user->save();

Cela changera les valeurs de la colonne vérifiée et active pour la ligne avec l’e-mail donné.

EFFACER La suppression peut également être effectuée à l’aide de findFirst ()

Exemple:

Users::findFirst("email='a@a.com'")->delete();

Cela supprimera la ligne avec l’e-mail donné.

Vous pouvez également exécuter des commandes SQL personnalisées avec des modèles à l’aide du code suivant :

$query = $this->modelsManager->createQuery("SELECT * FROM Users WHERE email='a@a.com'");

$user = $query->execute();

Utiliser le SQL standard directement avec les modèles #

Pour utiliser la syntaxe SQL avec le modèle, qui transférerait le résultat vers les instanciations appropriées, vous devez utiliser directement l’une des classes Phalcon\Mvc\Model\Resultset :

$users = new \Application\Models\Users();

// bitwise operation on `flag` field
$sql = 'SELECT * FROM phorum.users WHERE
        (15 & (1 << (flag - 1))) > 0  ORDER BY login DESC';

// as a result you will have a Resultset\Simple with Models\Users instances.
$result = new \Phalcon\Mvc\Model\Resultset\Simple(
    null,

    // what model to use for data returned from SQL
    $users,

    // setting result via "read connection" proper for this model.
    $users->getReadConnection()->query($sql) 
);

Configuration du service de connexion par défaut #

Phalcon utilise le service db par défaut pour obtenir la connexion aux bases de données.

En supposant que vous ayez un fichier de configuration avec le champ database configuré, vous pouvez inclure ou charger automatiquement le code suivant pour obtenir une connexion avec la base de données pour votre projet :

$di->set('db', function () use ($config) {
    $dbconf = $config->database;
    switch(strtolower($dbconf->adapter)) {

        case 'mysql':
            return new \Phalcon\Db\Adapter\Pdo\Mysql(array(
                'host' => $dbconf->host,
                'username' => $dbconf->username,
                'password' => $dbconf->password,
                // default database to work with
                'dbname' => $dbconf->dbname,
                // default character set
                'charset' => $dbconf->charset,
                // connection warm-up commands for PDO
                'options' => array(
                    PDO::MYSQL_ATTR_INIT_COMMAND => 'SET NAMES "' . $dbconf->charset . '"',
                    PDO::ATTR_CASE => PDO::CASE_LOWER
                )
            ));

        case 'postgresql':
            return new \Phalcon\Db\Adapter\Pdo\Postgresql(array(
                'host' => $dbconf->host,
                'username' => $dbconf->username,
                'password' => $dbconf->password,
                'dbname' => $dbconf->dbname,
                'options' => array(
                )
            ));
        
        default: 
            throw new \Exception('Unimplemented database::adapter in config.ini');
    }
});

Mise en cache des métadonnées des modèles. #

Phalcon accumule des informations sur les tables qu’il utilise, il est donc possible de valider les données qui y sont insérées sans tout implémenter à la main. Ce sont des métadonnées pour les modèles. Pour accélérer et empêcher Phalcon de créer des métadonnées à chaque fois que la page est rafraîchie, il est possible de les mettre en cache. Pour ce faire, vous devez implémenter le service metaData pour qu’il puisse utiliser :

$di->set('modelsMetadata', function() use ($config)
{
    // assuming that you have a $config var with
    // models.metadata.adapter field declared
    switch (strtolower($config->models->metadata->adapter)) {
        case 'apc':
            $metaData = new MetaDataApcAdapter([
                'lifetime' => $config->models->metadata->lifetime,
                'suffix' => $config->models->metadata->suffix,
            ]);
            break;
        case 'xcache':
            $metaData = new MetaDataXCacheAdapter([
                'lifetime' => $config->models->metadata->lifetime,
                'prefix' => $config->models->metadata->suffix,
            ]);
            break;
        case 'memory':
            $metaData = new MetaDataMemoryAdapter();
            break;
        default:
            throw new \Exception('Unimplemented models::metadata.adapter in config.ini');
    }

    return $metaData;
});

Documentation complémentaire disponible sur la [page dédiée] de Phalcons (https://docs.phalconphp.com/pl/latest/reference/models-metadata.html).
