Introduction #

L’[Incubateur Phalcon][1] peut être utilisé par la communauté pour expérimenter de nouvelles fonctionnalités ou étendre les adaptateurs, prototypes ou fonctionnalités Phalcon existants.

Tout ce qui se trouve dans l’incubateur peut être potentiellement incorporé dans le cadre.

Référentiel Github : https://github.com/phalcon/incubator

[1] : https://github.com/phalcon/incubator

Installation #

Installation via Composer #

Le moyen le plus simple d’installer l’incubateur consiste à utiliser [Composer][1].

Installez Composer et créez un nouveau fichier composer.json à la racine de votre projet.

|-- app
|-- public
|   `-- index.php
|-- vendor
|-- composer.json

Ajoutez le contenu suivant au fichier composer.json. Si vous utilisez encore Phalcon 2.0.x

{
    "require": {
        "phalcon/incubator": "^2.0"
    }
}

Si vous utilisez Phalcon 3.0.0

{
    "require": {
        "phalcon/incubator": "~3.0"
    }
}

Après avoir modifié le fichier composer.json, vous devez exécuter la commande suivante, à partir de la racine de votre projet.

$ php composer.phar install

Si vous avez déjà installé vos fichiers et que vous souhaitez les mettre à jour à la place. Ensuite, utilisez update au lieu de install. Par défaut, Composer créera un nouveau dossier nommé “vendor” dans la racine de votre projet et téléchargera tous les fichiers demandés dans ce répertoire.

Une fois composer installé, la structure de votre document devrait ressembler à ceci :

|-- app
|-- public
|   `-- index.php
|-- vendor
|   `-- phalcon
|       `-- incubator
|           `-- docs
|           `-- Library
|           `-- tests
|-- composer.json

Installation via Github #

Créez un dossier nommé “vendor” dans le répertoire racine de votre projet. Et créez également le dossier “phalcon” dans ce dossier.

|-- app
|-- public
|   `-- index.php
|-- vendor
|   `-- phalcon

Naviguez maintenant dans le dossier phalcon et clonez l’incubateur à partir du référentiel Github.

git clone https://github.com/phalcon/incubator.git

Par défaut, la commande ci-dessus téléchargera la dernière version de Phalcon. Si vous souhaitez télécharger une version antérieure, vous pouvez simplement ajouter le paramètre --branch à la commande, suivi de la version de branche requise.

git clone https://github.com/phalcon/incubator.git --branch 2.0.9

Installation via la méthode manuelle #

Si les méthodes ci-dessus vous déroutent et que vous aimez faire des choses manuellement, vous pouvez facilement [télécharger le référentiel depuis Github][2] et placer les fichiers dans le répertoire vendor/phalcon/, à la racine de votre projet.

|-- app
|-- public
|   `-- index.php
|-- vendor
|   `-- phalcon

[1] : https://www.wikiod.com/fr/composer-php/premiers-pas-avec-composer-php [2] : https://github.com/phalcon/incubator/archive/master.zip

Utilisation #

Chargement de l’incubateur dans votre projet #

Ajoutez les lignes de code suivantes à votre fichier de chargeur

$loader = new Phalcon\Loader();

$loader->registerNamespaces([
    'Phalcon' => '/path/to/your/vendor/phalcon/incubator/Library/Phalcon/',
    // any other namespaces you have loaded
    // ...
]);

$loader->register();

Vous pouvez désormais accéder à toutes les fonctionnalités de l’incubateur en utilisant les espaces de noms Phalcon normaux :

\Phalcon\Acl\Adapter\Database;
