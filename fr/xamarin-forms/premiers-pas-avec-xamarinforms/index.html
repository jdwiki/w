Installation (Visual Studio) #

Xamarin.Forms est une abstraction de boîte à outils d’interface utilisateur multiplateforme qui permet aux développeurs de créer facilement des interfaces utilisateur pouvant être partagées entre Android, iOS, Windows et Windows Phone. Les interfaces utilisateur sont rendues à l’aide des contrôles natifs de la plateforme cible, ce qui permet aux applications Xamarin.Forms de conserver l’apparence appropriée pour chaque plateforme.

Plugin Xamarin pour Visual Studio #

Pour démarrer avec Xamarin.Forms pour Visual Studio, vous devez disposer du plug-in Xamarin lui-même. Le moyen le plus simple de l’installer est de télécharger et d’installer la dernière version de Visual Studio.

Si vous avez déjà installé la dernière version de Visual Studio, accédez à Panneau de configuration > Programmes et fonctionnalités, cliquez avec le bouton droit sur Visual Studio, puis cliquez sur Modifier. Lorsque le programme d’installation s’ouvre, cliquez sur Modifier et sélectionnez les outils de développement mobile multiplateformes :

[![entrez la description de l’image ici][1]][1]

Vous pouvez également choisir d’installer le SDK Android :

[![entrez la description de l’image ici][2]][2]

Décochez-la si vous avez déjà installé le SDK. Vous pourrez configurer Xamarin pour utiliser le SDK Android existant ultérieurement.

Xamarin. Forms #

Xamarin.Forms est un ensemble de bibliothèques pour votre bibliothèque de classe portable et vos assemblys natifs. La bibliothèque Xamarin.Forms elle-même est disponible sous forme de package NuGet. Pour l’ajouter à votre projet, utilisez simplement la commande habituelle “Install-Package” de la console du gestionnaire de packages :

Install-Package Xamarin.Forms

pour tous vos assemblages initiaux (par exemple MyProject, MyProject.Droid et MyProject.iOS).

Le moyen le plus simple de démarrer avec Xamarin.Forms consiste à créer un projet vide dans Visual Studio :

[![entrez la description de l’image ici][3]][3]

Comme vous pouvez le voir, il existe 2 options disponibles pour créer l’application vierge - Portable et Partagée. Je vous recommande de commencer avec Portable one car c’est le plus couramment utilisé dans le monde réel (différences et plus d’explications à ajouter).

Après avoir créé le projet, assurez-vous que vous utilisez la dernière version de Xamarin.Forms car votre modèle initial peut contenir l’ancien. Utilisez votre console du gestionnaire de packages ou l’option Gérer les packages NuGet pour effectuer une mise à niveau vers la dernière version de Xamarin.Forms (rappelez-vous qu’il s’agit simplement d’un package NuGet).

Alors que les modèles Visual Studio Xamarin.Forms créeront un projet de plate-forme iOS pour vous, vous devrez connecter Xamarin à un hôte de build Mac pour pouvoir exécuter ces projets sur le simulateur iOS ou des appareils physiques.

Hello World Xamarin Forms : Visual Studio #

Après avoir installé avec succès Xamarin comme décrit dans le premier exemple, il est temps de lancer le premier exemple d’application.

Étape 1 : Création d’un nouveau projet. #

Dans Visual Studio, choisissez Nouveau -> Projet -> Visual C# -> Multiplateforme -> Application vide (Xamarin.Forms Portable)

Nommez l’application “Hello World” et sélectionnez l’emplacement pour créer le projet et cliquez sur OK. Cela créera une solution pour vous qui contient trois projets :

  1. HelloWorld (c’est là que votre logique et vos vues sont placées, c’est-à-dire le projet portable)
  2. HelloWorld.Droid (le projet Android)
  3. HelloWorld.iOS (le projet iOS)

[![Création d’un nouveau projet][1]][1]

Étape 2 : Examen de l’échantillon #

Après avoir créé la solution, un exemple d’application sera prêt à être déployé. Ouvrez le App.cs situé à la racine du projet portable et examinez le code. Comme on le voit ci-dessous, le Contents de l’exemple est un StackLayout qui contient un Label :

using Xamarin.Forms;

namespace Hello_World
{
    public class App : Application
    {
        public App()
        {
            // The root page of your application
            MainPage = new ContentPage
            {
                Content = new StackLayout
                {
                    VerticalOptions = LayoutOptions.Center,
                    Children = {
                        new Label {
                            HorizontalTextAlignment = TextAlignment.Center,
                            Text = "Welcome to Xamarin Forms!"
                        }
                    }
                }
            };
        }
        protected override void OnStart()
        {
            // Handle when your app starts
        }
        protected override void OnSleep()
        {
            // Handle when your app sleeps
        }
        protected override void OnResume()
        {
            // Handle when your app resumes
        }
    }
}

[1] : http://i.stack.imgur.com/VFpzj.png

Etape 3 : Lancement de l’application #

Maintenant, cliquez simplement avec le bouton droit sur le projet que vous souhaitez démarrer (HelloWorld.Droid ou HelloWorld.iOS) et cliquez sur Définir comme projet de démarrage. Ensuite, dans la barre d’outils de Visual Studio, cliquez sur le bouton “Démarrer” (le bouton triangulaire vert qui ressemble à un bouton de lecture) pour lancer l’application sur le simulateur/émulateur ciblé.
