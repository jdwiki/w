Les services HTTP fonctionnent normalement sur le port 80, mais si vous avez installé une application comme Skype qui utilise également le port 80, elle ne démarrera pas. Dans ce cas, vous devez modifier soit son port, soit le port de l’application en conflit. Une fois terminé, redémarrez le service HTTP.

Téléchargez et installez XAMPP #

Qu’est-ce que XAMPP ? #

XAMPP est l’environnement de développement PHP le plus populaire. XAMPP est une distribution Apache entièrement gratuite, open-source et facile à installer contenant MariaDB, PHP et Perl.

D’où dois-je le télécharger ? #

Téléchargez la version stable appropriée de XAMPP depuis leur page de téléchargement. Choisissez le téléchargement en fonction du type de système d’exploitation (32 ou 64 bits et version du système d’exploitation) et de la version PHP qu’il doit prendre en charge.

Le dernier en date étant XAMPP pour Windows 7.0.8 / PHP 7.0.8.

Ou vous pouvez suivre ceci :

XAMPP pour Windows existe en trois versions différentes :

  • [Installer] [1] (probablement le “format .exe” est le moyen le plus simple d’installer XAMPP)
  • [ZIP][2] (Pour les puristes : XAMPP comme archive ZIP ordinaire au format “.zip”)
  • [7zip :][3] (Pour les puristes à faible bande passante : XAMPP comme 7zip `.7zip format d’archive)

Comment installer et où dois-je placer mes fichiers PHP/html ? #

Installer avec le programme d’installation fourni #

  1. Exécutez le programme d’installation du serveur XAMPP en double-cliquant sur le “.exe” téléchargé.

Installer à partir du ZIP #

  1. Décompressez les archives zip dans le dossier de votre choix.
  2. XAMPP extrait dans le sous-répertoire C:\xampp sous le répertoire cible sélectionné.
  3. Démarrez maintenant le fichier setup_xampp.bat, pour ajuster la configuration XAMPP à votre système.

Remarque : Si vous choisissez un répertoire racine C:\ comme cible, vous ne devez pas démarrer setup_xampp.bat.

Post-installation #

Utilisez le “Panneau de configuration XAMPP” pour des tâches supplémentaires, comme démarrer/arrêter Apache, MySQL, FileZilla et Mercury ou les installer en tant que services.

La gestion des fichiers #

L’installation est un processus simple et une fois l’installation terminée, vous pouvez ajouter des fichiers html/php à héberger sur le serveur dans XAMPP-root/htdocs/. Ensuite, démarrez le serveur et ouvrez http://localhost/file.php sur un navigateur pour afficher la page.

Remarque : La racine XAMPP par défaut dans Windows est C:/xampp/htdocs/

Saisissez l’une des URL suivantes dans votre navigateur Web préféré :

http://localhost/
http://127.0.0.1/

Vous devriez maintenant voir la page de démarrage de XAMPP.

Téléchargez, installez et utilisez WAMP #

WampServer est un environnement de développement Web Windows. Il permet de créer des applications web avec Apache2, PHP et une base de données MySQL. Parallèlement, PhpMyAdmin vous permet de gérer facilement vos bases de données.

WampServer est disponible gratuitement (sous licence GPML) en deux versions distinctes : 32 et 64 bits. Wampserver 2.5 n’est pas compatible avec Windows XP, ni avec SP3, ni Windows Server 2003. Les anciennes versions de WampServer sont disponibles sur SourceForge.

Versions de WampServer :

  • [WampServer (64 BITS) 3](https://sourceforge.net/projects/wampserver/files/WampServer%203/WampServer%203.0.0/wampserver3.0.4_x64_apache2.4.18_mysql5.7.11_php5.6.19-7.0.4 .exe/télécharger)
  • [WampServer (32 BITS) 3](https://sourceforge.net/projects/wampserver/files/WampServer%203/WampServer%203.0.0/wampserver3.0.4_x86_apache2.4.18_mysql5.7.11_php5.6.19-7.0.4 .exe/télécharger)

Fournit actuellement :

  • Apache : 2.4.18
  • MySQL : 5.7.11
  • PHP : 5.6.19 & 7.0.4

L’installation est simple, il suffit d’exécuter le programme d’installation, de choisir l’emplacement et de le terminer.

Une fois cela fait, vous pouvez démarrer WampServer. Ensuite, il démarre dans la barre d’état système (barre des tâches), d’abord de couleur rouge, puis devient vert une fois que le serveur est en place.

Vous pouvez accéder à un navigateur et taper localhost ou 127.0.0.1 pour obtenir la page d’index de WAMP. Vous pouvez désormais travailler sur PHP localement en stockant les fichiers dans <PATH_TO_WAMP>/www/<php_or_html_file> et vérifier le résultat sur http://localhost/<php_or_html_file_name>
