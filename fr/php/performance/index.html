Profilage avec Xdebug #

Une extension de PHP appelée Xdebug est disponible pour aider au [profilage des applications PHP][1], ainsi qu’au débogage d’exécution. Lors de l’exécution du profileur, la sortie est écrite dans un fichier au format binaire appelé “cachegrind”. Des applications sont disponibles sur chaque plateforme pour analyser ces fichiers.

Pour activer le profilage, installez l’extension et ajustez les paramètres php.ini. Dans notre exemple, nous exécuterons le profil éventuellement en fonction d’un paramètre de requête. Cela nous permet de garder les paramètres statiques et d’activer le profileur uniquement en cas de besoin.

// Set to 1 to turn it on for every request
xdebug.profiler_enable = 0
// Let's use a GET/POST parameter to turn on the profiler
xdebug.profiler_enable_trigger = 1
// The GET/POST value we will pass; empty for any value
xdebug.profiler_enable_trigger_value = ""
// Output cachegrind files to /tmp so our system cleans them up later
xdebug.profiler_output_dir = "/tmp"
xdebug.profiler_output_name = "cachegrind.out.%p"

Utilisez ensuite un client Web pour adresser une requête à l’URL de votre application que vous souhaitez profiler, par ex.

http://example.com/article/1?XDEBUG_PROFILE=1

Au fur et à mesure que la page traite, elle écrira dans un fichier avec un nom similaire à

/tmp/cachegrind.out.12345

Notez qu’il écrira un fichier pour chaque requête/processus PHP exécuté. Ainsi, par exemple, si vous souhaitez analyser une publication de formulaire, un profil sera écrit pour la requête GET afin d’afficher le formulaire HTML. Le paramètre XDEBUG_PROFILE devra être passé dans la requête POST suivante pour analyser la deuxième requête qui traite le formulaire. Par conséquent, lors du profilage, il est parfois plus facile d’exécuter curl pour POSTer directement un formulaire.

Une fois écrit, le cache de profil peut être lu par une application telle que KCachegrind.

[![KCachegrind][2]][2]

Cela affichera des informations telles que :

  • Fonctions exécutées
  • Temps d’appel, à la fois lui-même et y compris les appels de fonction ultérieurs
  • Nombre de fois que chaque fonction est appelée
  • Graphiques d’appels
  • Liens vers le code source

Évidemment, le réglage des performances est très spécifique aux cas d’utilisation de chaque application. En général, il est bon de rechercher :

  • Appels répétés à la même fonction que vous ne vous attendriez pas à voir. Pour les fonctions qui traitent et interrogent des données, cela pourrait être une excellente opportunité pour votre application de mettre en cache.
  • Fonctions lentes. Où l’application passe-t-elle le plus de temps ? le meilleur gain en matière de réglage des performances consiste à se concentrer sur les parties de l’application qui consomment le plus de temps.

Remarque : Xdebug, et en particulier ses fonctionnalités de profilage, sont très gourmandes en ressources et ralentissent l’exécution de PHP. Il est recommandé de ne pas les exécuter dans un environnement de serveur de production.

[1] : https://xdebug.org/docs/profiler [2] : http://i.stack.imgur.com/ENtOu.gif

Utilisation de la mémoire #

La limite de mémoire d’exécution de PHP est définie par la directive INI memory_limit. Ce paramètre empêche toute exécution unique de PHP d’utiliser trop de mémoire, l’épuisant pour d’autres scripts et logiciels système. La limite de mémoire par défaut est de 128 Mo et peut être modifiée dans le fichier php.ini ou lors de l’exécution. Il peut être défini sans limite, mais cela est généralement considéré comme une mauvaise pratique.

L’utilisation exacte de la mémoire utilisée pendant l’exécution peut être déterminée en appelant memory_get_usage(). Il renvoie le nombre d’octets de mémoire alloués au script en cours d’exécution. Depuis PHP 5.2, il a un paramètre booléen facultatif pour obtenir la mémoire système totale allouée, par opposition à la mémoire activement utilisée par PHP.

 <?php
 echo memory_get_usage() . "\n";
 // Outputs 350688 (or similar, depending on system and PHP version)

 // Let's use up some RAM
 $array = array_fill(0, 1000, 'abc');

 echo memory_get_usage() . "\n";
 // Outputs 387704

 // Remove the array from memory
 unset($array);

 echo memory_get_usage() . "\n";
 // Outputs 350784

Désormais, memory_get_usage vous donne l’utilisation de la mémoire au moment de son exécution. Entre les appels à cette fonction, vous pouvez allouer et désallouer d’autres éléments en mémoire. Pour obtenir la quantité maximale de mémoire utilisée jusqu’à un certain point, appelez memory_get_peak_usage().

<?php
echo memory_get_peak_usage() . "\n";
// 385688
$array = array_fill(0, 1000, 'abc');
echo memory_get_peak_usage() . "\n";
// 422736
unset($array);
echo memory_get_peak_usage() . "\n";
// 422776

Notez que la valeur ne fera qu’augmenter ou rester constante.
