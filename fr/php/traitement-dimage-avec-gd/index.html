Lorsque vous utilisez header("Content-Type : $mimeType"); et image____ pour générer uniquement une image à la sortie, assurez-vous de ne rien afficher d’autre, notez même une ligne vide après ?>. (Cela peut être un “bogue” difficile à détecter – vous n’obtenez aucune image et aucune idée de pourquoi.) Le conseil général est de ne pas inclure ?> du tout ici.

Sortie d’image #

Une image peut être créée en utilisant les fonctions [image*][1], où * est le format de fichier.

Ils ont cette syntaxe en commun :

bool image___(resource $im [, mixed $to [ other parameters]] )

Enregistrer dans un fichier #

Si vous souhaitez enregistrer l’image dans un fichier, vous pouvez transmettre le nom du fichier, ou un flux de fichier ouvert, en tant que $to. Si vous passez un flux, vous n’avez pas besoin de le fermer, car GD le fermera automatiquement.

Par exemple, pour enregistrer un fichier PNG :

imagepng($image, "/path/to/target/file.png");

$stream = fopen("phar://path/to/target.phar/file.png", "wb");
imagepng($image2, $stream);
// Don't fclose($stream)

Lorsque vous utilisez fopen, assurez-vous d’utiliser le drapeau b plutôt que le drapeau t, car le fichier est une sortie binaire.

N’essayez pas de lui passer fopen("php://temp", $f) ou fopen("php://memory", $f). Étant donné que le flux est fermé par la fonction après l’appel, vous ne pourrez plus l’utiliser, par exemple pour récupérer son contenu.

Sortie sous forme de réponse HTTP #

Si vous souhaitez renvoyer directement cette image comme réponse de l’image (par exemple pour créer des badges dynamiques), vous n’avez pas besoin de passer quoi que ce soit (ou de passer null) comme deuxième argument. Cependant, dans la réponse HTTP, vous devez spécifier votre type de contenu :

header("Content-Type: $mimeType");

$mimeType est le type MIME du format que vous renvoyez. Les exemples incluent image/png, image/gif et image/jpeg.

Écrire dans une variable #

Il existe deux façons d’écrire dans une variable.

Utilisation d’OB (mise en mémoire tampon de sortie) #

ob_start();
imagepng($image, null, $quality); // pass null to supposedly write to stdout
$binary = ob_get_clean();

Utiliser des wrappers de flux #

Vous pouvez avoir de nombreuses raisons pour lesquelles vous ne souhaitez pas utiliser la mise en mémoire tampon de sortie. Par exemple, vous avez peut-être déjà activé OB. Il faut donc une alternative.

En utilisant la fonction stream_wrapper_register, un nouveau wrapper de flux peut être enregistré. Par conséquent, vous pouvez transmettre un flux à la fonction de sortie d’image et le récupérer ultérieurement.

<?php

class GlobalStream{
        private $var;

        public function stream_open(string $path){
                $this->var =& $GLOBALS[parse_url($path)["host"]];
                return true;
        }

        public function stream_write(string $data){
                $this->var .= $data;
                return strlen($data);
        }
}

stream_wrapper_register("global", GlobalStream::class);

$image = imagecreatetruecolor(100, 100);
imagefill($image, 0, 0, imagecolorallocate($image, 0, 0, 0));

$stream = fopen("global://myImage", "");
imagepng($image, $stream);
echo base64_encode($myImage);

Dans cet exemple, la classe “GlobalStream” écrit n’importe quelle entrée dans la variable de référence (c’est-à-dire qu’elle écrit indirectement dans la variable globale du nom donné). La variable globale peut ensuite être récupérée directement.

Il y a quelques points particuliers à noter :

  • Une classe wrapper de flux entièrement implémentée devrait ressembler à [this][2], mais selon les tests avec la méthode magique __call, seuls stream_open, stream_write et stream_close sont appelés à partir de fonctions internes.
  • Aucun indicateur n’est requis dans l’appel fopen, mais vous devez au moins passer une chaîne vide. C’est parce que la fonction fopen attend un tel paramètre, et même si vous ne l’utilisez pas dans votre implémentation stream_open, un factice est toujours requis.
  • Selon les tests, stream_write est appelé plusieurs fois. N’oubliez pas d’utiliser .= (affectation de concaténation) et non = (affectation directe de variable).

Exemple d’utilisation #

Dans la balise HTML <img>, une image peut être fournie directement plutôt que d’utiliser un lien externe :

echo '<img src="data:image/png;base64,' . base64_encode($binary) . '">';

[1] : http://php.net/manual/en/function.imagepng.php#refsect1-function.imagepng-seealso [2] : http://php.net/manual/en/stream.streamwrapper.example-1.php

Création d’une image #

Pour créer une image vierge, utilisez la fonction imagecreatetruecolor :

$img = imagecreatetruecolor($width, $height);

$img est maintenant une variable de ressource pour une ressource d’image avec $widthx$height pixels. Notez que la largeur compte de gauche à droite et la hauteur compte de haut en bas.

Les ressources d’image peuvent également être créées à partir de [fonctions de création d’image][1], telles que :

  • imagecreatefrompng
  • imagecreatefromjpeg
  • autres fonctions imagecreatefrom*.

Les ressources d’image peuvent être libérées ultérieurement lorsqu’il n’y a plus de références à celles-ci. Cependant, pour libérer la mémoire immédiatement (cela peut être important si vous traitez de nombreuses images volumineuses), utiliser imagedestroy() sur une image lorsqu’elle n’est plus utilisée peut être une bonne pratique.

imagedestroy($image);

Convertir une image #

Les images créées par conversion d’image ne modifient pas l’image tant que vous ne l’avez pas sortie. Par conséquent, un convertisseur d’image peut être aussi simple que trois lignes de code :

function convertJpegToPng(string $filename, string $outputFile) {
    $im = imagecreatefromjpeg($filename);
    imagepng($im, $outputFile);
    imagedestroy($im);
}

[1] : http://php.net/manual/en/ref.image.php

Recadrage et redimensionnement d’image #

Si vous avez une image et que vous souhaitez créer une nouvelle image, avec de nouvelles dimensions, vous pouvez utiliser la fonction imagecopyresampled :

créez d’abord une nouvelle “image” aux dimensions souhaitées :

// new image
$dst_img = imagecreatetruecolor($width, $height);

et stocker l’image d’origine dans une variable. Pour ce faire, vous pouvez utiliser l’une des fonctions createimagefrom* où * signifie :

-jpeg

  • gif -png
  • chaîne de caractères

Par exemple:

//original image
$src_img=imagecreatefromstring(file_get_contents($original_image_path));

Maintenant, copiez tout (ou une partie) de l’image originale (src_img) dans la nouvelle image (dst_img) par imagecopyresampled :

imagecopyresampled($dst_img, $src_img, 
    $dst_x ,$dst_y, $src_x, $src_y, 
    $dst_width, $dst_height, $src_width, $src_height);

Pour définir les dimensions src_* et dst_*, utilisez l’image ci-dessous :

[![entrez la description de l’image ici][1]][1]

Maintenant, si vous souhaitez copier l’intégralité de l’image source (initiale) dans l’intégralité de la zone de destination (sans recadrage):

$src_x = $src_y = $dst_x = $dst_y = 0;
$dst_width = $width;// width of new image
$dst_height = $height; //height of new image
$src_width = imagesx($src_img); //width of initial image
$src_height = imagesy($src_img); //height of initial image

[1] : http://i.stack.imgur.com/6MFxN.jpg
