Syntaxe #

  • int readfile ( chaîne $filename [, bool $use_include_path = false [, ressource $context ]] )

Paramètres #

Paramètre Descriptif
nom de fichier Le nom du fichier en cours de lecture.
use_include_path Vous pouvez utiliser le deuxième paramètre facultatif et le définir sur TRUE si vous souhaitez également rechercher le fichier dans include_path.
contexte Une ressource de flux de contexte.

Syntaxe du nom de fichier #

La plupart des noms de fichiers transmis aux fonctions de cette rubrique sont :

  1. Cordes dans la nature.
  • Les noms de fichiers peuvent être passés directement. Si des valeurs d’autres types sont transmises, elles sont converties en chaîne. Ceci est particulièrement utile avec SplFileInfo, qui est la valeur dans l’itération de DirectoryIterator.
  1. Relatif ou absolu.
  • Ils peuvent être absolus. Sur les systèmes de type Unix, les chemins absolus commencent par /, par ex. /home/user/file.txt, tandis que sous Windows, les chemins absolus commencent par le lecteur, par ex. C:/Utilisateurs/utilisateur/fichier.txt
  • Ils peuvent également être relatifs, ce qui dépend de la valeur de [getcwd][getcwd] et peut être modifié par [chdir][chdir].
  1. Acceptez les protocoles.
  • Ils peuvent commencer par scheme:// pour spécifier le wrapper de protocole à gérer. Par exemple, file_get_contents("http://example.com") récupère le contenu de http://example.com.
  1. Compatible Slash.
  • Bien que le DIRECTORY_SEPARATOR sous Windows soit une barre oblique inverse et que le système renvoie par défaut des barres obliques inverses pour les chemins, le développeur peut toujours utiliser / comme séparateur de répertoire. Par conséquent, pour des raisons de compatibilité, les développeurs peuvent utiliser / comme séparateurs de répertoires sur tous les systèmes, mais sachez que les valeurs renvoyées par les fonctions (par exemple, realpath) peuvent contenir des barres obliques inverses.

[getcwd] : http://php.net/getcwd [chdir] : http://php.net/chdir

Suppression de fichiers et de répertoires #

Suppression de fichiers #

La fonction [unlink][unlink] supprime un seul fichier et indique si l’opération a réussi.

$filename = '/path/to/file.txt';

if (file_exists($filename)) {
    $success = unlink($filename);
    
    if (!$success) {
         throw new Exception("Cannot delete $filename");
    }
}

Suppression de répertoires, avec suppression récursive #

D’autre part, les répertoires doivent être supprimés avec [rmdir][rmdir]. Cependant, cette fonction ne supprime que les répertoires vides. Pour supprimer un répertoire contenant des fichiers, supprimez d’abord les fichiers des répertoires. Si le répertoire contient des sous-répertoires, recursion peut être nécessaire.

L’exemple suivant analyse les fichiers d’un répertoire, supprime les fichiers/répertoires membres de manière récursive et renvoie le nombre de fichiers (et non de répertoires) supprimés.

function recurse_delete_dir(string $dir) : int {
    $count = 0;

    // ensure that $dir ends with a slash so that we can concatenate it with the filenames directly
    $dir = rtrim($dir, "/\\") . "/";

    // use dir() to list files
    $list = dir($dir);

    // store the next file name to $file. if $file is false, that's all -- end the loop.
    while(($file = $list->read()) !== false) {
        if($file === "." || $file === "..") continue;
        if(is_file($dir . $file)) {
            unlink($dir . $file);
            $count++;
        } elseif(is_dir($dir . $file)) {
            $count += recurse_delete_dir($dir . $file);
        }
    }

    // finally, safe to delete directory!
    rmdir($dir);

    return $count;
}

[dissocier] : http://php.net/unlink [rmdir] : http://php.net/rmdir

Fonctions de confort #

E/S directes brutes #

[file_get_contents][fgc] et [file_put_contents][fpc] offrent la possibilité de lire/écrire depuis/vers un fichier vers/depuis une chaîne PHP en un seul appel.

[file_put_contents][fpc] peut également être utilisé avec l’indicateur de masque de bits FILE_APPEND pour ajouter, au lieu de tronquer et d’écraser, le fichier. Il peut être utilisé avec le masque de bits LOCK_EX pour acquérir un verrou exclusif sur le fichier tout en procédant à l’écriture. Les drapeaux de masque de bits peuvent être joints avec l’opérateur | bitwise-OR.

$path = "file.txt";
// reads contents in file.txt to $contents
$contents = file_get_contents($path);
// let's change something... for example, convert the CRLF to LF!
$contents = str_replace("\r\n", "\n", $contents);
// now write it back to file.txt, replacing the original contents
file_put_contents($path, $contents);

FILE_APPEND est pratique pour ajouter des fichiers journaux tandis que LOCK_EX permet d’éviter les conditions de concurrence lors de l’écriture de fichiers à partir de plusieurs processus. Par exemple, pour écrire dans un fichier journal à propos de la session en cours :

file_put_contents("logins.log", "{$_SESSION["username"]} logged in", FILE_APPEND | LOCK_EX);

CSV IO #

fgetcsv($file, $length, $separator)

Le [fgetcsv][fgcsv] analyse la ligne du fichier ouvert en vérifiant les champs csv. Il renvoie les champs CSV dans un tableau en cas de succès ou “FALSE” en cas d’échec.

Par défaut, il ne lira qu’une seule ligne du fichier CSV.

$file = fopen("contacts.csv","r");
print_r(fgetcsv($file));    
print_r(fgetcsv($file,5," "));
fclose($file);

contacts.csv

Kai Jim, Refsnes, Stavanger, Norway
Hege, Refsnes, Stavanger, Norway

Production:

Array
(
    [0] => Kai Jim
    [1] => Refsnes
    [2] => Stavanger
    [3] => Norway
)
Array
(
    [0] => Hege,
)

Lire un fichier directement sur stdout #

[readfile][readfile] copie un fichier dans le tampon de sortie. readfile() ne présentera aucun problème de mémoire, même lors de l’envoi de fichiers volumineux, par lui-même.

$file = 'monkey.gif';

if (file_exists($file)) {
    header('Content-Description: File Transfer');
    header('Content-Type: application/octet-stream');
    header('Content-Disposition: attachment; filename="'.basename($file).'"');
    header('Expires: 0');
    header('Cache-Control: must-revalidate');
    header('Pragma: public');
    header('Content-Length: ' . filesize($file));
    readfile($file);
    exit;
}

Ou depuis un pointeur de fichier #

Alternativement, pour rechercher un point dans le fichier pour commencer la copie vers stdout, utilisez [fpassthru][fpassthru] à la place. Dans l’exemple suivant, les 1024 derniers octets sont copiés dans stdout :

$fh = fopen("file.txt", "rb");
fseek($fh, -1024, SEEK_END); 
fpassthru($fh);

Lecture d’un fichier dans un tableau #

[file][file] renvoie les lignes du fichier passé dans un tableau. Chaque élément du tableau correspond à une ligne dans le fichier, avec la nouvelle ligne toujours attachée.

print_r(file("test.txt"));

test.txt

Welcome to File handling
This is to test file handling

Production:

Array 
( 
    [0] => Welcome to File handling 
    [1] => This is to test file handling 
)

Obtenir des informations sur le fichier #

Vérifier si un chemin est un répertoire ou un fichier #

La fonction [is_dir][is_dir] renvoie si l’argument est un répertoire, tandis que [is_file][is_file] renvoie si l’argument est un fichier. Utilisez [file_exists][file_exists] pour vérifier si c’est l’un ou l’autre.

$dir  = "/this/is/a/directory";
$file = "/this/is/a/file.txt";

echo is_dir($dir) ? "$dir is a directory" : "$dir is not a directory", PHP_EOL,
    is_file($dir) ? "$dir is a file" : "$dir is not a file", PHP_EOL,
    file_exists($dir) ? "$dir exists" : "$dir doesn't exist", PHP_EOL,
    is_dir($file) ? "$file is a directory" : "$file is not a directory", PHP_EOL,
    is_file($file) ? "$file is a file" : "$file is not a file", PHP_EOL,
    file_exists($file) ? "$file exists" : "$file doesn't exist", PHP_EOL;

Cela donne:

/this/is/a/directory is a directory
/this/is/a/directory is not a file
/this/is/a/directory exists
/this/is/a/file.txt is not a directory
/this/is/a/file.txt is a file
/this/is/a/file.txt exists

Vérification du type de fichier #

Utilisez [filetype][filetype] pour vérifier le type d’un fichier, qui peut être :

  • ‘fifo’
  • car
  • répertoire
  • ‘bloquer’
  • “lien”
  • ‘fichier’
  • prise
  • “inconnu”

Passer le nom de fichier au [filetype][filetype] directement :

echo filetype("~"); // dir

Notez que filetype renvoie false et déclenche un E_WARNING si le fichier n’existe pas.

Vérification de la lisibilité et de l’écriture #

Passer le nom du fichier aux fonctions [is_writable][is_writable] et [is_readable][is_readable] vérifie si le fichier est accessible en écriture ou en lecture respectivement.

Les fonctions renvoient false gracieusement si le fichier n’existe pas.

Vérification de l’accès au fichier/heure de modification #

L’utilisation de [filemtime][filemtime] et [fileatime][fileatime] renvoie l’horodatage de la dernière modification ou accès au fichier. La valeur de retour est un horodatage Unix – voir https://www.wikiod.com/fr/php/travailler-avec-les-dates-et-lheure pour plus de détails.

echo "File was last modified on " . date("Y-m-d", filemtime("file.txt"));
echo "File was last accessed on " . date("Y-m-d", fileatime("file.txt"));

Récupère les parties du chemin avec fileinfo #

$fileToAnalyze = ('/var/www/image.png');

$filePathParts = pathinfo($fileToAnalyze);

echo '<pre>';
   print_r($filePathParts);
echo '</pre>';

Cet exemple affichera :

Array
(
    [dirname] => /var/www
    [basename] => image.png
    [extension] => png
    [filename] => image
)

Qui peut être utilisé comme :

$filePathParts['dirname']
$filePathParts['basename']
$filePathParts['extension']
$filePathParts['filename']
Paramètre Détails
$chemin Le chemin complet du fichier à analyser
$option L’une des quatre options disponibles [PATHINFO_DIRNAME, PATHINFO_BASENAME, PATHINFO_EXTENSION ou PATHINFO_FILENAME]
  • Si une option (le deuxième paramètre) n’est pas passée, un tableau associatif est retourné sinon une chaîne est retournée.
  • Ne valide pas l’existence du fichier.
  • Analyse simplement la chaîne en plusieurs parties. Aucune validation n’est effectuée sur le fichier (pas de vérification de type mime, etc.)
  • L’extension est simplement la dernière extension de $path Le chemin du fichier image.jpg.png serait .png même s’il s’agit techniquement d’un fichier .jpg. Un fichier sans extension ne renverra pas d’élément d’extension dans le tableau.

Minimiser l’utilisation de la mémoire lors du traitement de fichiers volumineux #

Si nous devons analyser un fichier volumineux, par ex. un CSV de plus de 10 Mo contenant des millions de lignes, certains utilisent les fonctions file ou file_get_contents et finissent par frapper le paramètre memory_limit avec

Taille de mémoire autorisée de XXXXX octets épuisée

Erreur. Considérez la source suivante (top-1m.csv a exactement 1 million de lignes et a une taille d’environ 22 Mo)

var_dump(memory_get_usage(true));
$arr = file('top-1m.csv');
var_dump(memory_get_usage(true));

Cela génère :

int(262144)
int(210501632)

parce que l’interpréteur devait contenir toutes les lignes du tableau $arr, il a donc consommé environ 200 Mo de RAM. Notez que nous n’avons même rien fait avec le contenu du tableau.

Considérez maintenant le code suivant :

var_dump(memory_get_usage(true));
$index = 1;
if (($handle = fopen("top-1m.csv", "r")) !== FALSE) {

Fichier IO basé sur le flux #

Ouvrir un flux #

[fopen][fopen] ouvre un descripteur de flux de fichiers, qui peut être utilisé avec diverses fonctions de lecture, d’écriture, de recherche et d’autres fonctions en plus. Cette valeur est de type resource et ne peut pas être transmise à d’autres threads conservant sa fonctionnalité.

$f = fopen("errors.log", "a"); // Will try to open errors.log for writing

Le deuxième paramètre est le mode du flux de fichier :

mode Descriptif
r Ouvrir en mode lecture seule, en commençant au début du fichier
r+ Ouvert en lecture et en écriture, en commençant au début du fichier
w ouvert en écriture uniquement, en commençant au début du fichier. Si le fichier existe, il le videra. S’il n’existe pas, il tentera de le créer.
w+ ouvert en lecture et en écriture, en commençant au début du fichier. Si le fichier existe, il le videra. S’il n’existe pas, il tentera de le créer.
a ouvrir un fichier en écriture uniquement, en commençant par la fin du fichier. Si le fichier n’existe pas, il essaiera de le créer
a+ ouvrir un fichier en lecture et en écriture, en commençant par la fin du fichier. Si le fichier n’existe pas, il essaiera de le créer
x créer et ouvrir un fichier en écriture uniquement. Si le fichier existe, l’appel fopen échouera
x+ créer et ouvrir un fichier pour la lecture et l’écriture. Si le fichier existe, l’appel fopen échouera
c ouvrir le fichier en écriture uniquement. Si le fichier n’existe pas, il essaiera de le créer. Il commencera à écrire au début du fichier, mais ne videra pas le fichier avant d’écrire
c+ ouvrir le fichier pour la lecture et l’écriture. Si le fichier n’existe pas, il essaiera de le créer. Il commencera à écrire au début du fichier, mais ne videra pas le fichier avant d’écrire

L’ajout d’un t derrière le mode (par exemple, a+b, wt, etc.) dans Windows traduira les fins de ligne "\n" en "\r\n" lorsque vous travaillez avec le fichier. Ajoutez b derrière le mode si ce n’est pas voulu, surtout s’il s’agit d’un fichier binaire.

L’application PHP doit fermer les flux en utilisant [fclose][fclose] lorsqu’ils ne sont plus utilisés pour éviter l’erreur Too many open files. Ceci est particulièrement important dans les programmes CLI, car les flux ne sont fermés que lorsque le runtime s’arrête – cela signifie que dans les serveurs Web, il peut ne pas être nécessaire (mais toujours devrait, en tant que pratique pour éviter les fuites de ressources) pour fermez les flux si vous ne vous attendez pas à ce que le processus s’exécute pendant une longue période et n’ouvrira pas beaucoup de flux.

En lisant #

L’utilisation de [fread][fread] lira le nombre d’octets donné à partir du pointeur de fichier, ou jusqu’à ce qu’un EOF soit rencontré.

Lecture de lignes #

L’utilisation de [fgets][fgets] lira le fichier jusqu’à ce qu’un EOL soit atteint ou que la longueur donnée soit lue.

[fread][fread] et [fgets][fgets] déplaceront le pointeur de fichier lors de la lecture.

Lire tout ce qui reste #

L’utilisation de [stream_get_contents][sgc] transformera tous les octets restants du flux en une chaîne et la renverra.

Réglage de la position du pointeur de fichier #

Initialement après l’ouverture du flux, le pointeur de fichier est au début du fichier (ou à la fin, si le mode a est utilisé). L’utilisation de la fonction [fseek][fseek] déplacera le pointeur de fichier vers une nouvelle position, par rapport à l’une des trois valeurs suivantes :

  • SEEK_SET : il s’agit de la valeur par défaut ; le décalage de la position du fichier sera relatif au début du fichier.
  • SEEK_CUR : le décalage de la position du fichier sera relatif à la position actuelle.
  • SEEK_END : le décalage de la position du fichier sera relatif à la fin du fichier. Passer un décalage négatif est l’utilisation la plus courante de cette valeur ; il déplacera la position du fichier au nombre d’octets spécifié avant la fin du fichier.

[rewind][rewind] est un raccourci pratique de fseek($fh, 0, SEEK_SET).

L’utilisation de [ftell][ftell] affichera la position absolue du pointeur de fichier.

Par exemple, le script suivant lit saute les 10 premiers octets, lit les 10 octets suivants, saute 10 octets, lit les 10 octets suivants, puis les 10 derniers octets dans file.txt :

$fh = fopen("file.txt", "rb");
fseek($fh, 10); // start at offset 10
echo fread($fh, 10); // reads 10 bytes
fseek($fh, 10, SEEK_CUR); // skip 10 bytes
echo fread($fh, 10); // read 10 bytes
fseek($fh, -10, SEEK_END); // skip to 10 bytes before EOF
echo fread($fh, 10); // read 10 bytes
fclose($fh);

L’écriture #

L’utilisation de [fwrite][fwrite] écrit la chaîne fournie dans le fichier en commençant par le pointeur de fichier actuel.

fwrite($fh, "Some text here\n");

Déplacer et copier des fichiers et des répertoires #

Copier des fichiers #

[copy][copy] copie le fichier source dans le premier argument vers la destination dans le deuxième argument. La destination résolue doit se trouver dans un répertoire déjà créé.

if (copy('test.txt', 'dest.txt')) {
    echo 'File has been copied successfully';
} else {
    echo 'Failed to copy file to destination given.'
}

Copie de répertoires, avec récursivité #

La copie de répertoires est assez similaire à la suppression de répertoires, sauf que pour les fichiers [copy][copy] au lieu de [unlink][unlink] est utilisé, tandis que pour les répertoires, [mkdir][mkdir] au lieu de [ rmdir][rmdir] est utilisé, au début au lieu d’être à la fin de la fonction.

function recurse_delete_dir(string $src, string $dest) : int {
    $count = 0;

    // ensure that $src and $dest end with a slash so that we can concatenate it with the filenames directly
    $src = rtrim($dest, "/\\") . "/";
    $dest = rtrim($dest, "/\\") . "/";

    // use dir() to list files
    $list = dir($src);

    // create $dest if it does not already exist
    @mkdir($dest);

    // store the next file name to $file. if $file is false, that's all -- end the loop.
    while(($file = $list->read()) !== false) {
        if($file === "." || $file === "..") continue;
        if(is_file($src . $file)) {
            copy($src . $file, $dest . $file);
            $count++;
        } elseif(is_dir($src . $file)) {
            $count += recurse_copy_dir($src . $file, $dest . $file);
        }
    }

    return $count;
}

Renommer/Déplacer #

Renommer/Déplacer des fichiers et des répertoires est beaucoup plus simple. Des répertoires entiers peuvent être déplacés ou renommés en un seul appel, en utilisant la fonction [rename][rename].

  • renommer("~/fichier.txt", "~/fichier.html");
  • rename("~/dir", "~/old_dir");
  • renommer("~/dir/fichier.txt", "~/dir2/fichier.txt");
