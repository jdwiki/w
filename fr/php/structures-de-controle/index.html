sinon #

L’instruction if dans l’exemple ci-dessus permet d’exécuter un fragment de code, lorsque la condition est remplie. Lorsque vous souhaitez exécuter un fragment de code, lorsque la condition n’est pas remplie, vous étendez le if avec un else.

if ($a > $b) {
  echo "a is greater than b";
} else {
  echo "a is NOT greater than b";
}

[Manuel PHP - Structures de contrôle - Sinon][1]

L’opérateur ternaire comme syntaxe abrégée pour if-else

L’[opérateur ternaire][2] évalue quelque chose en fonction d’une condition vraie ou non. Il s’agit d’un opérateur de comparaison souvent utilisé pour exprimer une simple condition if-else sous une forme plus courte. Elle permet de tester rapidement une condition et remplace souvent une instruction if multiligne, rendant votre code plus compact.

Voici l’exemple ci-dessus utilisant une expression ternaire et des valeurs variables : $a=1 ; $b=2;

echo ($a > $b) ? "a is greater than b" : "a is NOT greater than b";

Sorties : a n'est PAS supérieur à b.

[1] : http://php.net/manual/en/control-structures.else.php [2] : http://php.net/manual/de/language.operators.comparison.php#language.operators.comparison.ternary

Syntaxe alternative pour les structures de contrôle #

PHP fournit une syntaxe alternative pour certaines structures de contrôle : if, while, for, foreach et switch.

Par rapport à la syntaxe normale, la différence est que l’accolade ouvrante est remplacée par deux-points (:) et l’accolade fermante est remplacée par endif;, endwhile;, endfor;, endforeach; , ou endswitch;, respectivement. Pour des exemples individuels, voir la rubrique [syntaxe alternative pour les structures de contrôle][1].

if ($a == 42):
    echo "The answer to life, the universe and everything is 42.";
endif;

Plusieurs instructions elseif utilisant une syntaxe courte :

if ($a == 5):
    echo "a equals 5";
elseif ($a == 6):
    echo "a equals 6";
else:
    echo "a is neither 5 nor 6";
endif;

[Manuel PHP - Structures de contrôle - Syntaxe alternative][2]

[1] : https://www.wikiod.com/fr/php/syntaxe-alternative-pour-les-structures-de-controle [2] : http://php.net/manual/en/control-structures.alternative-syntax.php

tandis que #

La boucle while parcourt un bloc de code tant qu’une condition spécifiée est vraie.

$i = 1;
while ($i < 10) {
    echo $i;
    $i++;
}

Sortie : 123456789

Pour des informations détaillées, voir [the Loops topic](https://www.wikiod.com/fr/php/boucles#tandis que).

faire pendant #

La boucle do-while exécute d’abord un bloc de code une fois, dans tous les cas, puis parcourt ce bloc de code tant qu’une condition spécifiée est vraie.

$i = 0;
do {
    $i++;
    echo $i;
} while ($i < 10);

Output: `12345678910`

Pour des informations détaillées, voir [the Loops topic](https://www.wikiod.com/fr/php/boucles#faire pendant).

aller à #

L’opérateur goto permet de passer à une autre section du programme. Il est disponible depuis PHP 5.3.

L’instruction goto est un goto suivi de l’étiquette cible souhaitée : goto MyLabel;.

La cible du saut est spécifiée par une étiquette suivie de deux-points : MyLabel:.

Cet exemple affichera Hello World! :

<?php
goto MyLabel;
echo 'This text will be skipped, because of the jump.';
 
MyLabel:
echo 'Hello World!';
?>

déclarer #

declare est utilisé pour définir une directive d’exécution pour un bloc de code.

Les directives suivantes sont reconnues :

  • [ticks][1]
  • [encodage][2]
  • [strict_types][3]

Par exemple, réglez les ticks sur 1 :

declare(ticks=1);

Pour activer le mode de type strict, l’instruction declare est utilisée avec la déclaration strict_types :

declare(strict_types=1);

[1] : http://php.net/manual/en/control-structures.declare.php#control-structures.declare.ticks [2] : http://php.net/manual/en/control-structures.declare.php#control-structures.declare.encoding [3] : http://php.net/manual/en/functions.arguments.php#functions.arguments.type-declaration.strict

inclure et exiger #

exiger #

require est similaire à include, sauf qu’il produira une erreur fatale de niveau E_COMPILE_ERROR en cas d’échec. Lorsque le require échoue, il arrête le script. Lorsque include échoue, il n’arrêtera pas le script et n’émettra que E_WARNING.

require 'file.php';

[Manuel PHP - Structures de contrôle - Nécessite][1]

comprendre #

L’instruction include inclut et évalue un fichier.

./variables.php

$a = 'Hello World!';

./main.php`

include 'variables.php';
echo $a;
// Output: `Hello World!`

Soyez prudent avec cette approche, car elle est considérée comme une [odeur de code][2], car le fichier inclus modifie la quantité et le contenu des variables définies dans la portée donnée.

Vous pouvez également “inclure” le fichier, qui renvoie une valeur. Ceci est extrêmement utile pour gérer les tableaux de configuration :

configuration.php

<?php 
return [
    'dbname' => 'my db',
    'user' => 'admin',
    'pass' => 'password',
];

principal.php

<?php
$config = include 'configuration.php';

Cette approche empêchera le fichier inclus de polluer votre portée actuelle avec des variables modifiées ou ajoutées.

[Manuel PHP - Structures de contrôle - Inclure][3]

[1] : http://php.net/manual/en/function.require.php [2] : https://en.wikipedia.org/wiki/Code_smell

include & require peut également être utilisé pour attribuer des valeurs à une variable lorsqu’elle renvoie quelque chose par fichier.

Exemple :

fichier include1.php :

<?php
    $a = "This is to be returned";

    return $a;
?>

fichier index.php :

    $value = include 'include1.php';
   // Here, $value = "This is to be returned"

[3] : http://php.net/manual/en/function.include.php

revenir #

L’instruction return renvoie le contrôle du programme à la fonction appelante.

Lorsque return est appelé depuis une fonction, l’exécution de la fonction en cours se termine.

function returnEndsFunctions()
{
   echo 'This is executed';
   return;
   echo 'This is not executed.';
}

Lorsque vous exécutez returnEndsFunctions(); vous obtiendrez la sortie This is execute ;

Lorsque return est appelé depuis une fonction avec et comme argument, l’exécution de la fonction en cours se terminera et la valeur de l’argument sera renvoyée à la fonction appelante.

pour #

Les boucles for sont généralement utilisées lorsque vous avez un morceau de code que vous souhaitez répéter un certain nombre de fois.

for ($i = 1; $i < 10; $i++) {
    echo $i;
}

Sorties : 123456789

Pour des informations détaillées, voir the Loops topic.

pour chaque #

foreach est une construction qui vous permet de parcourir facilement des tableaux et des objets.

$array = [1, 2, 3];
foreach ($array as $value) {
    echo $value;
}

Sorties : ‘123’.

Pour utiliser la boucle foreach avec un objet, il doit implémenter l’interface [Iterator][1].

Lorsque vous itérez sur des tableaux associatifs :

$array = ['color'=>'red']; 

foreach($array as $key => $value){
    echo $key . ': ' . $value; 
}

Sorties : “couleur : rouge”

Pour des informations détaillées, voir [the Loops topic](https://www.wikiod.com/fr/php/boucles#pour chaque).

[1] : http://php.net/manual/en/class.iterator.php

si sinon si sinon #

autrement

elseif combine if et else. L’instruction if est étendue pour exécuter une instruction différente au cas où l’expression if d’origine n’est pas satisfaite. Mais, l’expression alternative n’est exécutée que lorsque l’expression conditionnelle “elseif” est remplie.

Le code suivant affiche soit “a est plus grand que b”, “a est égal à b” ou “a est plus petit que b” :

if ($a > $b) {
    echo "a is bigger than b";
} elseif ($a == $b) {
    echo "a is equal to b";
} else {
    echo "a is smaller than b";
}

Plusieurs déclarations elseif

Vous pouvez utiliser plusieurs instructions elseif dans la même instruction if :

if ($a == 1) {
    echo "a is One";
} elseif ($a == 2) {
    echo "a is Two";
} elseif ($a == 3) {
    echo "a is Three";
} else {
    echo "a is not One, not Two nor Three";
}

si #

La construction if permet l’exécution conditionnelle de fragments de code.

if ($a > $b) {
  echo "a is bigger than b";
}

[Manuel PHP - Structures de contrôle - Si][1]

[1] : http://php.net/manual/en/control-structures.if.php

changer #

La structure switch exécute la même fonction qu’une série d’instructions if, mais peut faire le travail en moins de lignes de code. La valeur à tester, telle que définie dans l’instruction switch, est comparée pour égalité avec les valeurs de chacune des instructions case jusqu’à ce qu’une correspondance soit trouvée et que le code de ce bloc soit exécuté. Si aucune instruction case correspondante n’est trouvée, le code du bloc default est exécuté, s’il existe.

Chaque bloc de code dans une instruction case ou default doit se terminer par l’instruction break. Cela arrête l’exécution de la structure switch et continue l’exécution du code immédiatement après. Si l’instruction break est omise, le code de l’instruction case suivante est exécuté, même s’il n’y a pas de correspondance. Cela peut entraîner une exécution de code inattendue si l’instruction break est oubliée, mais peut également être utile lorsque plusieurs instructions case doivent partager le même code.

switch ($colour) {
case "red":
    echo "the colour is red";
    break;
case "green":
case "blue":
    echo "the colour is green or blue";
    break;
case "yellow":
    echo "the colour is yellow";
    // note missing break, the next block will also be executed
case "black":
    echo "the colour is black";
    break;
default:
    echo "the colour is something else";
    break;
}

En plus de tester des valeurs fixes, la construction peut également être contrainte de tester des instructions dynamiques en fournissant une valeur booléenne à l’instruction switch et toute expression à l’instruction case. Gardez à l’esprit que la première valeur correspondante est utilisée, donc le code suivant affichera “plus de 100”:

$i = 1048;
switch (true) {
case ($i > 0):
    echo "more than 0";
    break;
case ($i > 100):
    echo "more than 100";
    break;
case ($i > 1000):
    echo "more than 1000";
    break;
}

Pour les éventuels problèmes de frappe lâche lors de l’utilisation de la construction switch, voir [Switch Surprises] [1]

[1] : https://www.wikiod.com/fr/php/problemes-de-jonglerie-de-type-et-de-comparaison-non-stricte#Changer de surprise
