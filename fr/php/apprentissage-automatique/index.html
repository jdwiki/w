La rubrique utilise PHP-ML pour tous les algorithmes d’apprentissage automatique. L’installation de la bibliothèque peut se faire à l’aide de

composer require php-ai/php-ml

Le référentiel github pour le même peut être trouvé ici.

Il convient également de noter que les exemples donnés sont de très petits ensembles de données uniquement à des fins de démonstration. L’ensemble de données réel devrait être plus complet que cela.

Classification à l’aide de PHP-ML #

La classification dans Machine Learning est le problème qui identifie à quel ensemble de catégories appartient une nouvelle observation. La classification relève de la catégorie “Apprentissage automatique supervisé”.

Tout algorithme qui implémente la classification est appelé classificateur

Les classificateurs pris en charge dans PHP-ML sont

  • SVC (Classification des vecteurs de support)
  • k-plus proches voisins
  • Bayes naïf

Les méthodes train et predict sont les mêmes pour tous les classificateurs. La seule différence serait dans l’algorithme sous-jacent utilisé.

SVC (Classification des vecteurs de support) #

Avant de pouvoir commencer à prédire une nouvelle observation, nous devons former notre classifieur. Considérez le code suivant

// Import library
use Phpml\Classification\SVC;
use Phpml\SupportVectorMachine\Kernel;

// Data for training classifier
$samples = [[1, 3], [1, 4], [2, 4], [3, 1], [4, 1], [4, 2]];  // Training samples
$labels = ['a', 'a', 'a', 'b', 'b', 'b'];

// Initialize the classifier
$classifier = new SVC(Kernel::LINEAR, $cost = 1000);
// Train the classifier
$classifier->train($samples, $labels);

Le code est assez simple. Le « $coût » utilisé ci-dessus est une mesure de combien nous voulons éviter de mal classer chaque exemple de formation. Pour une valeur inférieure à $cost, vous risquez d’obtenir des exemples mal classés. Par défaut, il est défini sur 1.0

Maintenant que nous avons formé le classificateur, nous pouvons commencer à faire des prédictions réelles. Considérez les codes suivants que nous avons pour les prédictions

$classifier->predict([3, 2]); // return 'b'
$classifier->predict([[3, 2], [1, 5]]); // return ['b', 'a']

Le classificateur dans le cas ci-dessus peut prendre des échantillons non classés et prédire leurs étiquettes. La méthode predict peut prendre un seul échantillon ainsi qu’un tableau d’échantillons.

k-plus proches voisins #

Le classificateur de cet algorithme prend deux paramètres et peut être initialisé comme

$classifier = new KNearestNeighbors($neighbor_num=4);
$classifier = new KNearestNeighbors($neighbor_num=3, new Minkowski($lambda=4));

$neighbor_num est le nombre de voisins les plus proches à analyser dans l’algorithme knn tandis que le deuxième paramètre est la métrique de distance qui par défaut dans le premier cas serait ` euclidien ». Plus d’informations sur Minkowski peuvent être trouvées [ici] (https://en.wikipedia.org/wiki/Minkowski_distance).

Voici un court exemple sur la façon d’utiliser ce classificateur

// Training data
$samples = [[1, 3], [1, 4], [2, 4], [3, 1], [4, 1], [4, 2]];
$labels = ['a', 'a', 'a', 'b', 'b', 'b'];

// Initialize classifier
$classifier = new KNearestNeighbors();
// Train classifier
$classifier->train($samples, $labels);

// Make predictions
$classifier->predict([3, 2]); // return 'b'
$classifier->predict([[3, 2], [1, 5]]); // return ['b', 'a']

Classificateur NaiveBayes #

NaiveBayes Classifier est basé sur le `théorème de Bayes’ et n’a besoin d’aucun paramètre dans le constructeur.

Le code suivant illustre une implémentation de prédiction simple

// Training data
$samples = [[5, 1, 1], [1, 5, 1], [1, 1, 5]];
$labels = ['a', 'b', 'c'];

// Initialize classifier
$classifier = new NaiveBayes();
// Train classifier
$classifier->train($samples, $labels);

// Make predictions
$classifier->predict([3, 1, 1]); // return 'a'
$classifier->predict([[3, 1, 1], [1, 4, 1]); // return ['a', 'b']

Cas pratique #

Jusqu’à présent, nous n’utilisions que des tableaux d’entiers dans tous nos cas, mais ce n’est pas le cas dans la vraie vie. Par conséquent, laissez-moi essayer de décrire une situation pratique sur la façon d’utiliser les classificateurs.

Supposons que vous ayez une application qui stocke les caractéristiques des fleurs dans la nature. Par souci de simplicité, nous pouvons considérer la couleur et la longueur des pétales. Donc, deux caractéristiques seraient utilisées pour former nos données. color est le plus simple où vous pouvez attribuer une valeur int à chacun d’eux et pour la longueur, vous pouvez avoir une plage comme (0 mm,10 mm)=1 , (10 mm,20 mm)=2. Avec les données initiales, entraînez votre classificateur. Maintenant, l’un de vos utilisateurs doit identifier le type de fleur qui pousse dans son jardin. Ce qu’il fait, c’est sélectionner la “couleur” de la fleur et ajouter la longueur des pétales. Votre classificateur en cours d’exécution peut détecter le type de fleur (“Étiquettes dans l’exemple ci-dessus”)

Régression #

Dans la classification à l’aide de PHP-ML, nous avons attribué des étiquettes à la nouvelle observation. La régression est presque la même, la différence étant que la valeur de sortie n’est pas une étiquette de classe mais une valeur continue. Il est largement utilisé pour les prédictions et les prévisions. PHP-ML prend en charge les algorithmes de régression suivants

  • Prise en charge de la régression vectorielle
  • Régression linéaire des moindres carrés

La régression utilise les mêmes méthodes “train” et “predict” que celles utilisées dans la classification.

Prise en charge de la régression vectorielle #

Il s’agit de la version de régression pour SVM (Support Vector Machine). La première étape, comme dans la classification, consiste à entraîner notre modèle.

// Import library
use Phpml\Regression\SVR;
use Phpml\SupportVectorMachine\Kernel;

// Training data
$samples = [[60], [61], [62], [63], [65]];
$targets = [3.1, 3.6, 3.8, 4, 4.1];

// Initialize regression engine
$regression = new SVR(Kernel::LINEAR);
// Train regression engine
$regression->train($samples, $targets);

Dans la régression, les $cibles ne sont pas des étiquettes de classe, contrairement à la classification. C’est l’un des facteurs de différenciation pour les deux. Après avoir formé notre modèle avec les données, nous pouvons commencer avec les prédictions réelles

$regression->predict([64])  // return 4.03

Notez que les prédictions renvoient une valeur en dehors de la cible.

Régression linéaire des moindres carrés #

Cet algorithme utilise la “méthode des moindres carrés” pour approximer la solution. Ce qui suit montre un code simple de formation et de prédiction

// Training data
$samples = [[60], [61], [62], [63], [65]];
$targets = [3.1, 3.6, 3.8, 4, 4.1];

// Initialize regression engine
$regression = new LeastSquares();
// Train engine
$regression->train($samples, $targets);
// Predict using trained engine
$regression->predict([64]); // return 4.06

PHP-ML fournit également l’option de “régression linéaire multiple”. Un exemple de code pour le même peut être comme suit

$samples = [[73676, 1996], [77006, 1998], [10565, 2000], [146088, 1995], [15000, 2001], [65940, 2000], [9300, 2000], [93739, 1996], [153260, 1994], [17764, 2002], [57000, 1998], [15000, 2000]];
$targets = [2000, 2750, 15500, 960, 4400, 8800, 7100, 2550, 1025, 5900, 4600, 4400];

$regression = new LeastSquares();
$regression->train($samples, $targets);
$regression->predict([60000, 1996]) // return 4094.82

La “régression linéaire multiple” est particulièrement utile lorsque plusieurs facteurs ou caractéristiques identifient le résultat.

Cas pratique #

Prenons maintenant une application de la régression dans un scénario réel.

Supposons que vous gérez un site Web très populaire, mais que le trafic ne cesse de changer. Vous voulez une solution qui prédise le nombre de serveurs que vous devez déployer à un instant donné. Supposons par souci que votre fournisseur d’hébergement vous donne une API pour créer des serveurs et que chaque serveur prend 15 minutes pour démarrer. Sur la base des données de trafic précédentes et de la régression, vous pouvez prédire le trafic qui toucherait votre application à tout moment. En utilisant ces connaissances, vous pouvez démarrer un serveur 15 minutes avant la surtension, empêchant ainsi votre application de se déconnecter.

Regroupement #

Le clustering consiste à regrouper des objets similaires. Il est largement utilisé pour la reconnaissance de formes. Le « clustering » relève de l’« apprentissage automatique non supervisé », aucune formation n’est donc nécessaire. PHP-ML prend en charge les algorithmes de clustering suivants

  • k-Moyennes
  • dbscan

k-Moyennes #

k-Means sépare les données en n groupes de variance égale. Cela signifie que nous devons transmettre un nombre “n” qui serait le nombre de clusters dont nous avons besoin dans notre solution. Le code suivant aidera à apporter plus de clarté

// Our data set
$samples = [[1, 1], [8, 7], [1, 2], [7, 8], [2, 1], [8, 9]];

// Initialize clustering with parameter `n`
$kmeans = new KMeans(3);
$kmeans->cluster($samples); // return [0=>[[7, 8]], 1=>[[8, 7]], 2=>[[1,1]]]

Notez que la sortie contient 3 tableaux parce que c’était la valeur de ’n’ dans le constructeur ‘KMeans’. Il peut également y avoir un deuxième paramètre facultatif dans le constructeur qui serait la “méthode d’initialisation”. Considérez par exemple

$kmeans = new KMeans(4, KMeans::INIT_RANDOM);

INIT_RANDOM place un centroïde complètement aléatoire tout en essayant de déterminer les clusters. Mais juste pour éviter que le centre de gravité ne soit trop éloigné des données, il est lié par les limites spatiales des données.

La méthode d'initialisation du constructeur par défaut est kmeans++ qui sélectionne le centroïde de manière intelligente pour accélérer le processus.

DBSCAN #

Contrairement à KMeans, DBSCAN est un algorithme de clustering basé sur la densité, ce qui signifie que nous ne passerions pas n qui déterminerait le nombre de clusters que nous voulons dans notre résultat. Par contre cela nécessite deux paramètres pour fonctionner

  1. $minSamples : Le nombre minimum d’objets qui doivent être présents dans un cluster
  2. $epsilon : Quelle est la distance maximale entre deux échantillons pour qu’ils soient considérés comme faisant partie du même cluster.

Un exemple rapide pour le même est comme suit

// Our sample data set
$samples = [[1, 1], [8, 7], [1, 2], [7, 8], [2, 1], [8, 9]];

$dbscan = new DBSCAN($epsilon = 2, $minSamples = 3);
$dbscan->cluster($samples); // return [0=>[[1, 1]], 1=>[[8, 7]]]

Le code est assez explicite. Une différence majeure est qu’il n’y a aucun moyen de connaître le nombre d’éléments dans le tableau de sortie par opposition à KMeans.

Cas Pratique #

Voyons maintenant comment utiliser le clustering dans un scénario réel

Le clustering est largement utilisé dans la « reconnaissance de formes » et la « exploration de données ». Considérez que vous avez une application de publication de contenu. Maintenant, pour fidéliser vos utilisateurs, ils doivent regarder le contenu qu’ils aiment. Supposons, par souci de simplicité, que s’ils sont sur une page Web spécifique pendant plus d’une minute et qu’ils défilent vers le bas, ils aiment ce contenu. Désormais, chacun de vos contenus aura un identifiant unique, tout comme l’utilisateur. Créez un cluster basé sur cela et vous saurez quel segment d’utilisateurs a un goût de contenu similaire. Cela pourrait à son tour être utilisé dans un système de recommandation où vous pouvez supposer que si certains utilisateurs du même cluster aiment l’article, d’autres le seront également et cela peut être affiché sous forme de recommandations sur votre application.
