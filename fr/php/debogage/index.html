Variables de vidage #

La fonction [var_dump][1] vous permet de vider le contenu d’une variable (type et valeur) pour le débogage.

Exemple:

$array = [3.7, "string", 10, ["hello" => "world"], false, new DateTime()];
var_dump($array);

Production:

array(6) {
  [0]=>
  float(3.7)
  [1]=>
  string(6) "string"
  [2]=>
  int(10)
  [3]=>
  array(1) {
    ["hello"]=>
    string(5) "world"
  }
  [4]=>
  bool(false)
  [5]=>
  object(DateTime)#1 (3) {
    ["date"]=>
    string(26) "2016-07-24 13:51:07.000000"
    ["timezone_type"]=>
    int(3)
    ["timezone"]=>
    string(13) "Europe/Berlin"
  }
}

[1] : http://php.net/manual/en/function.var-dump.php

Affichage des erreurs #

Si vous voulez que PHP affiche les erreurs d’exécution sur la page, vous devez activer [display_errors][1], soit dans le php.ini, soit en utilisant la fonction [ini_set][2].

Vous pouvez choisir les erreurs à afficher, avec la fonction error_reporting (ou dans l’ini), qui accepte les [constantes E_*][3], combinées à l’aide d’[opérateurs au niveau du bit][4].

PHP peut afficher les erreurs au format texte ou HTML, selon le paramètre [html_errors][5].

Exemple:

ini_set("display_errors", true);
ini_set("html_errors", false); // Display errors in plain text
error_reporting(E_ALL & ~E_USER_NOTICE); // Display everything except E_USER_NOTICE

trigger_error("Pointless error"); // E_USER_NOTICE
echo $nonexistentVariable; // E_NOTICE
nonexistentFunction(); // E_ERROR

Sortie en texte brut : (le format HTML diffère d’une implémentation à l’autre)

Notice: Undefined variable: nonexistentVariable in /path/to/file.php on line 7

Fatal error: Uncaught Error: Call to undefined function nonexistentFunction() in /path/to/file.php:8
Stack trace:
#0 {main}
  thrown in /path/to/file.php on line 8

REMARQUE : Si vous avez désactivé le rapport d’erreurs dans php.ini et que vous l’activez pendant l’exécution, certaines erreurs (telles que les erreurs d’analyse) ne seront pas affichées, car elles se sont produites avant l’application du paramètre d’exécution.

La manière courante de gérer error_reporting est de l’activer complètement avec la constante E_ALL pendant le développement et de désactiver son affichage public avec display_errors en phase de production pour masquer les éléments internes de vos scripts.

[1] : http://php.net/manual/en/errorfunc.configuration.php#ini.display-errors [2] : http://php.net/manual/en/function.ini-set.php [3] : http://php.net/manual/en/errorfunc.constants.php [4] : http://php.net/manual/en/language.operators.bitwise.php [5] : http://php.net/manual/en/errorfunc.configuration.php#ini.html-errors

phpinfo() #

Avertissement #

**Il est impératif que phpinfo ne soit utilisé que dans un environnement de développement. Ne publiez jamais de code contenant phpinfo dans un environnement de production **

Introduction #

Cela dit, cela peut être un outil utile pour comprendre l’environnement PHP (OS, configuration, versions, chemins, modules) dans lequel vous travaillez, en particulier lors de la chasse à un bogue. Il s’agit d’une simple fonction intégrée :

phpinfo();

Il a un paramètre $what qui permet de personnaliser la sortie. La valeur par défaut est INFO_ALL, ce qui lui permet d’afficher toutes les informations et est couramment utilisée pendant le développement pour voir l’état actuel de PHP.

Vous pouvez passer le paramètre [constantes INFO_*][2], combiné avec des opérateurs au niveau du bit pour voir une liste personnalisée.

Vous pouvez l’exécuter dans le navigateur pour un look détaillé bien formaté. Cela fonctionne également dans PHP CLI, où vous pouvez le diriger vers “moins” pour une vue plus facile.

Exemple #

phpinfo(INFO_CONFIGURATION | INFO_ENVIRONMENT | INFO_VARIABLES);

Cela affichera une liste de directives PHP ([ini_get][3]), d’environnement ([$_ENV][4]) et de variables [predefined][5].

[1] : http://php.net/manual/en/function.phpinfo.php [2] : http://php.net/manual/en/function.phpinfo.php#refsect1-function.phpinfo-parameters [3] : http://php.net/manual/en/function.ini-get.php [4] : http://php.net/manual/en/reserved.variables.environment.php [5] : http://php.net/manual/en/language.variables.predefined.php

Xdébogage #

[Xdebug][1] est une extension PHP qui fournit des fonctionnalités de débogage et de profilage.
Il utilise le protocole de débogage DBGp.

Il y a quelques fonctionnalités intéressantes dans cet outil :

  • empiler les traces sur les erreurs
  • protection maximale du niveau d’imbrication et suivi du temps
  • remplacement utile de la fonction standard var_dump() pour afficher les variables
  • permet de consigner tous les appels de fonction, y compris les paramètres et les valeurs de retour dans un fichier dans différents formats
  • analyse de couverture de code
  • informations de profilage
  • débogage à distance (fournit une interface pour les clients de débogage qui interagissent avec l’exécution de scripts PHP)

Comme vous pouvez le voir, cette extension est parfaitement adaptée à l’environnement de développement. En particulier, la fonctionnalité de ** débogage à distance ** peut vous aider à déboguer votre code php sans de nombreux var_dump et à utiliser le processus de débogage normal comme dans les langages C++ ou Java.

Habituellement, l’installation de cette extension est très simple :

pecl install xdebug # install from pecl/pear

Et activez-le dans votre php.ini :

zend_extension="/usr/local/php/modules/xdebug.so"

Dans les cas plus compliqués, voir ceci [instructions][2]

Lorsque vous utilisez cet outil, vous devez vous rappeler que :
[XDebug n’est pas adapté aux environnements de production][3]

[1] : https://xdebug.org [2] : https://xdebug.org/docs/install [3] : http://stackoverflow.com/a/3522356/2253302

phpversion() #

Introduction #

Lorsque vous travaillez avec diverses bibliothèques et leurs exigences associées, il est souvent nécessaire de connaître la version de l’analyseur PHP actuel ou de l’un de ses packages.

Cette fonction accepte un seul paramètre facultatif sous la forme d’un nom d’extension : phpversion('extension'). Si l’extension en question est installée, la fonction renverra une chaîne contenant la valeur de la version. Cependant, si l’extension n’est pas installée, FALSE sera renvoyé. Si le nom de l’extension n’est pas fourni, la fonction renverra la version de l’analyseur PHP lui-même.

Exemple #

print "Current PHP version: " . phpversion();
// Current PHP version: 7.0.8

print "Current cURL version: " . phpversion( 'curl' );
// Current cURL version: 7.0.8
// or
// false, no printed output if package is missing

Rapport d’erreurs (utilisez-les tous les deux) #

// this sets the configuration option for your environment
ini_set('display_errors', '1');

//-1 will allow all errors to be reported
error_reporting(-1);
