Syntaxe #

  • $variable = ‘valeur’ ; // Affectation d’une variable générale
  • $objet->propriété = ‘valeur’ ; // Affecte une propriété d’objet
  • NomClasse ::$propriété = ‘valeur’ ; // Affecte une propriété de classe statique
  • $tableau[0] = ‘valeur’ ; // Affecte une valeur à un index d’un tableau
  • $tableau[] = ‘valeur’ ; // Pousser un élément à la fin d’un tableau
  • $tableau[‘clé’] = ‘valeur’ ; // Affecte une valeur de tableau
  • echo $variable ; // Écho (affiche) une valeur de variable
  • une_fonction($variable); // Utiliser la variable comme paramètre de la fonction
  • unset($variable); // Annule une variable
  • $$variable = ‘valeur’ ; // Affectation à une variable variable
  • isset($variable); // Vérifie si une variable est définie ou non
  • vide($variable); // Vérifie si une variable est vide ou non

Vérification de type #

Une partie de la documentation concernant les variables et les types mentionne que PHP n’utilise pas typage statique. C’est correct, mais PHP effectue une vérification de type en ce qui concerne les paramètres de fonction/méthode et les valeurs de retour (en particulier avec PHP 7).

Vous pouvez appliquer la vérification du type de paramètre et de valeur de retour en utilisant l’indication de type dans PHP 7 comme suit :

<?php

/**
 * Juggle numbers and return true if juggling was
 * a great success.
 */
function numberJuggling(int $a, int $b) : bool
{
    $sum = $a + $b;

    return $sum % 2 === 0;
}

Remarque : [gettype()][1] de PHP pour les entiers et les booléens est respectivement integer et boolean. Mais pour l’indication de type pour de telles variables, vous devez utiliser int et bool. Sinon, PHP ne vous donnera pas d’erreur de syntaxe, mais il s’attendra à ce que les classes integer et boolean soient transmises.

L’exemple ci-dessus génère une erreur dans le cas où une valeur non numérique est donnée comme paramètre $a ou $b, et si la fonction renvoie autre chose que true ou false. L’exemple ci-dessus est “lâche”, car vous pouvez donner une valeur flottante à $a ou $b. Si vous souhaitez appliquer des types stricts, ce qui signifie que vous ne pouvez saisir que des entiers et non des flottants, ajoutez ce qui suit au tout début de votre fichier PHP :

<?php
declare('strict_types=1');

Avant PHP 7, les fonctions et méthodes autorisaient l’indication de type pour les types suivants :

  • callable (une fonction ou une méthode appelable)
  • array (tout type de tableau, qui peut également contenir d’autres tableaux)
  • Interfaces (Fully-Qualified-Class-Name, ou FQDN)
  • Classes (FQDN)

Voir aussi : https://www.wikiod.com/fr/php/sortie-de-la-valeur-dune-variable

[1] : http://php.net/manual/en/function.gettype.php

Accéder dynamiquement à une variable par son nom (Variables variables) #

Les variables sont accessibles via des noms de variables dynamiques. Le nom d’une variable peut être stocké dans une autre variable, ce qui permet d’y accéder dynamiquement. Ces variables sont appelées variables variables.

Pour transformer une variable en une variable variable, vous mettez un $ supplémentaire mis devant votre variable.

$variableName = 'foo';
$foo = 'bar';

// The following are all equivalent, and all output "bar":
echo $foo;
echo ${$variableName};
echo $$variableName;

//similarly,
$variableName  = 'foo';
$$variableName = 'bar';

// The following statements will also output 'bar'
echo $foo; 
echo $$variableName; 
echo ${$variableName};

Les variables variables sont utiles pour mapper les appels de fonction/méthode :

function add($a, $b) {
    return $a + $b;
}

$funcName = 'add';

echo $funcName(1, 2); // outputs 3

Cela devient particulièrement utile dans les classes PHP :

class myClass {
    public function __construct() {
        $functionName = 'doSomething';
        $this->$functionName('Hello World');
    }

    private function doSomething($string) {
        echo $string; // Outputs "Hello World"
    }
}

Il est possible, mais pas obligatoire, de mettre $variableName entre {} :

${$variableName} = $value;

Les exemples suivants sont à la fois équivalents et génèrent “baz”:

$fooBar = 'baz';
$varPrefix = 'foo';

echo $fooBar;               // Outputs "baz"
echo ${$varPrefix . 'Bar'}; // Also outputs "baz"

L’utilisation de {} n’est obligatoire que lorsque le nom de la variable est lui-même une expression, comme ceci :

${$variableNamePart1 . $variableNamePart2} = $value;

Il est néanmoins recommandé de toujours utiliser {}, car c’est plus lisible.

Bien qu’il ne soit pas recommandé de le faire, il est possible d’enchaîner ce comportement :

$$$$$$$$DoNotTryThisAtHomeKids = $value;

Il est important de noter que l’utilisation excessive de variables variables est considérée comme une mauvaise pratique par de nombreux développeurs. Comme ils ne sont pas bien adaptés à l’analyse statique par les IDE modernes, les grandes bases de code avec de nombreuses variables variables (ou invocations de méthodes dynamiques) peuvent rapidement devenir difficiles à maintenir.

Différences entre PHP5 et PHP7 #

Une autre raison de toujours utiliser {} ou () est que PHP5 et PHP7 ont une manière légèrement différente de traiter les variables dynamiques, ce qui entraîne un résultat différent dans certains cas.

En PHP7, les variables dynamiques, les propriétés et les méthodes seront désormais évaluées strictement dans l’ordre de gauche à droite, par opposition au mélange de cas particuliers en PHP5. Les exemples ci-dessous montrent comment l’ordre d’évaluation a changé.

Cas 1 : $$foo['bar']['baz'] #

  • Interprétation PHP5 : ${$foo['bar']['baz']}
  • Interprétation PHP7 : ($$foo)['bar']['baz']

Cas 2 : $foo->$bar['baz'] #

  • Interprétation PHP5 : $foo->{$bar['baz']}
  • Interprétation PHP7 : ($foo->$bar)['baz']

Cas 3 : $foo->$bar['baz']() #

  • Interprétation PHP5 : $foo->{$bar['baz']}()
  • Interprétation PHP7 : ($foo->$bar)['baz']()

Cas 4 : Foo::$bar['baz']() #

  • Interprétation PHP5 : Foo::{$bar['baz']}()
  • Interprétation PHP7 : (Foo::$bar)['baz']()

Types de données #

Il existe différents types de données à des fins différentes. PHP n’a pas de définitions de type explicites, mais le type d’une variable est déterminé par le type de la valeur qui lui est assignée, ou par le type auquel elle est convertie. Ceci est un bref aperçu des types, pour une documentation détaillée et des exemples, voir the PHP types topic.

Il existe les types de données suivants en PHP : null, boolean, integer, float, string, object, resource et array.

Nul #

Null peut être affecté à n’importe quelle variable. Il représente une variable sans valeur.

$foo = null;

Cela invalide la variable et sa valeur serait indéfinie ou nulle si elle était appelée. La variable est effacée de la mémoire et supprimée par le garbage collector.

Booléen #

C’est le type le plus simple avec seulement deux valeurs possibles.

$foo = true;
$bar = false;

Les booléens peuvent être utilisés pour contrôler le flux de code.

$foo = true;

if ($foo) {
    echo "true";
} else {
    echo "false";
}

Entier #

Un entier est un nombre entier positif ou négatif. Il peut être utilisé avec n’importe quelle base numérique. La taille d’un entier dépend de la plate-forme. PHP ne prend pas en charge les entiers non signés.

$foo = -3;  // negative
$foo = 0;   // zero (can also be null or false (as boolean)
$foo = 123; // positive decimal
$bar = 0123; // octal = 83 decimal
$bar = 0xAB; // hexadecimal = 171 decimal
$bar = 0b1010; // binary = 10 decimal
var_dump(0123, 0xAB, 0b1010); // output: int(83) int(171) int(10)

Flotteur #

Les nombres à virgule flottante, “doubles” ou simplement appelés “flottants” sont des nombres décimaux.

$foo = 1.23;
$foo = 10.0;
$bar = -INF;
$bar = NAN;

Déployer #

Un tableau est comme une liste de valeurs. La forme la plus simple d’un tableau est indexée par un entier et ordonnée par l’index, le premier élément se trouvant à l’index 0.

$foo = array(1, 2, 3); // An array of integers
$bar = ["A", true, 123 => 5]; // Short array syntax, PHP 5.4+

echo $bar[0];    // Returns "A"
echo $bar[1];    // Returns true
echo $bar[123];  // Returns 5
echo $bar[1234]; // Returns null

Les tableaux peuvent également associer une clé autre qu’un index entier à une valeur. En PHP, tous les tableaux sont des tableaux associatifs dans les coulisses, mais lorsque nous nous référons à un “tableau associatif” distinctement, nous entendons généralement celui qui contient une ou plusieurs clés qui ne sont pas des entiers.

$array = array();
$array["foo"] = "bar";
$array["baz"] = "quux";
$array[42] = "hello";
echo $array["foo"]; // Outputs "bar"
echo $array["bar"]; // Outputs "quux"
echo $array[42]; // Outputs "hello"

Chaîne de caractères #

Une chaîne est comme un tableau de caractères.

$foo = "bar";

Comme un tableau, une chaîne peut être indexée pour renvoyer ses caractères individuels :

$foo = "bar";
echo $foo[0]; // Prints 'b', the first character of the string in $foo.

Objet #

Un objet est une instance d’une classe. Ses variables et ses méthodes sont accessibles avec l’opérateur ->.

$foo = new stdClass(); // create new object of class stdClass, which a predefined, empty class
$foo->bar = "baz";
echo $foo->bar; // Outputs "baz"
// Or we can cast an array to an object:
$quux = (object) ["foo" => "bar"];
echo $quux->foo; // This outputs "bar".

Ressource #

Les variables de ressource contiennent des descripteurs spéciaux pour les fichiers ouverts, les connexions à la base de données, les flux, les zones de canevas d’image, etc. (comme indiqué dans le [manuel](https://secure.php.net/manual/en/language.types.resource .php#language.types.resource.casting)).

$fp = fopen('file.ext', 'r'); // fopen() is the function to open a file on disk as a resource.
var_dump($fp); // output: resource(2) of type (stream)

Pour obtenir le type d’une variable sous forme de chaîne, utilisez la fonction gettype() :

echo gettype(1); // outputs "integer"
echo gettype(true); // "boolean"

Meilleures pratiques pour les variables globales #

On peut illustrer ce problème avec le pseudo-code suivant

function foo() {
    global $bob;
    $bob->doSomething();
}

Votre première question ici est évidente

D’où vient $bob ?

Êtes-vous confus? Bien. Vous venez d’apprendre pourquoi les variables globales prêtent à confusion et sont considérées comme une mauvaise pratique.

S’il s’agissait d’un vrai programme, votre prochaine partie amusante consisterait à rechercher toutes les instances de $bob et à espérer trouver la bonne (cela empire si $bob est utilisé partout). Pire encore, si quelqu’un d’autre définit $bob (ou si vous avez oublié et réutilisé cette variable), votre code peut casser (dans l’exemple de code ci-dessus, avoir le mauvais objet, ou aucun objet du tout, provoquerait une erreur fatale).

Étant donné que pratiquement tous les programmes PHP utilisent du code comme include('file.php'); votre travail de maintenance de code comme celui-ci devient exponentiellement plus difficile à mesure que vous ajoutez de fichiers.

De plus, cela rend la tâche de tester vos applications très difficile. Supposons que vous utilisiez une variable globale pour maintenir votre connexion à la base de données :

$dbConnector = new DBConnector(...);

function doSomething() {
    global $dbConnector;
    $dbConnector->execute("...");
}

Afin de tester unitairement cette fonction, vous devez remplacer la variable globale $dbConnector, exécuter les tests, puis la réinitialiser à sa valeur d’origine, ce qui est très sujet aux bogues :

/**
 * @test
 */
function testSomething() {
     global $dbConnector;

     $bkp = $dbConnector; // Make backup
     $dbConnector = Mock::create('DBConnector'); // Override

     assertTrue(foo());

     $dbConnector = $bkp; // Restore
}

Comment éviter les Globals ?

La meilleure façon d’éviter les globals est une philosophie appelée [Dependency Injection][1]. C’est là que nous passons les outils dont nous avons besoin dans la fonction ou la classe.

function foo(\Bar $bob) {
    $bob->doSomething();
}

C’est beaucoup plus facile à comprendre et à maintenir. Il est impossible de deviner où $bob a été configuré car l’appelant est responsable de le savoir (il nous transmet ce que nous devons savoir). Mieux encore, nous pouvons utiliser type declarations pour restreindre ce qui est passé.

Nous savons donc que $bob est soit une instance de la classe Bar, soit une instance d’un enfant de Bar, ce qui signifie que nous savons que nous pouvons utiliser les méthodes de cette classe. Combiné avec un chargeur automatique standard (disponible depuis PHP 5.3), nous pouvons maintenant rechercher où Bar est défini. PHP 7.0 ou ultérieur inclut des déclarations de type étendu, où vous pouvez également utiliser des types scalaires (comme int ou string).

Variables superglobales

Les super globales en PHP sont des variables prédéfinies, qui sont toujours disponibles, accessibles à partir de n’importe quelle étendue du script.

Il n’est pas nécessaire de faire une $variable globale ; pour y accéder dans des fonctions/méthodes, des classes ou des fichiers.

Ces variables PHP superglobales sont répertoriées ci-dessous :

  • [$GLOBALS][2] - [$_SERVER][3] - [$_REQUEST][4] - [$_POST][5]
  • [$_GET][6]
  • [$_FILES][7] - [$_ENV][8]
  • [$_COOKIE][9] - [$_SESSION][10]

Valeurs par défaut des variables non initialisées #

Bien que cela ne soit pas nécessaire en PHP, c’est une très bonne pratique d’initialiser les variables. Les variables non initialisées ont une valeur par défaut de leur type selon le contexte dans lequel elles sont utilisées :

Non défini ET non référencé

var_dump($unset_var); // outputs NULL

Booléen

echo($unset_bool ? "true\n" : "false\n"); // outputs 'false'

Chaîne de caractères

$unset_str .= 'abc';
var_dump($unset_str); // outputs 'string(3) "abc"'

Entier

$unset_int += 25; // 0 + 25 => 25
var_dump($unset_int); // outputs 'int(25)'

Flottant/double

$unset_float += 1.25;
var_dump($unset_float); // outputs 'float(1.25)'

Déployer

$unset_arr[3] = "def";
var_dump($unset_arr); //  outputs array(1) {  [3]=>  string(3) "def" }

Objet

$unset_obj->foo = 'bar';
var_dump($unset_obj); // Outputs: object(stdClass)#1 (1) {  ["foo"]=>  string(3) "bar" }

S’appuyer sur la valeur par défaut d’une variable non initialisée est problématique dans le cas de l’inclusion d’un fichier dans un autre qui utilise le même nom de variable.

Véracité de la valeur variable et opérateur identique #

En PHP, les valeurs des variables ont une “véracité” associée, de sorte que même les valeurs non booléennes équivaudront à true ou false. Cela permet d’utiliser n’importe quelle variable dans un bloc conditionnel, par ex.

if ($var == true) { /* explicit version */ }
if ($var) { /* $var == true is implicit */ }

Voici quelques règles fondamentales pour différents types de valeurs variables : * Chaînes d’une longueur non nulle équivalent à true, y compris les chaînes contenant uniquement des espaces tels que ' '.

  • Les chaînes vides '' correspondent à false.

    $var = ‘’; $var_is_true = ($var == true); // false $var_is_false = ($var == false); // true

    $var = ’ ‘; $var_is_true = ($var == true); // true $var_is_false = ($var == false); // false

  • Les nombres entiers correspondent à “vrai” s’ils sont différents de zéro, tandis que zéro correspond à “faux”.

    $var = -1; $var_is_true = ($var == true); // true $var = 99; $var_is_true = ($var == true); // true $var = 0; $var_is_true = ($var == true); // false

  • null équivaut à faux

    $var = null; $var_is_true = ($var == true); // false $var_is_false = ($var == false); // true

  • Les chaînes vides '' et la chaîne zéro '0' correspondent à false.

    $var = ‘’; $var_is_true = ($var == true); // false $var_is_false = ($var == false); // true

    $var = ‘0’; $var_is_true = ($var == true); // false $var_is_false = ($var == false); // true

* Les valeurs à virgule flottante correspondent à “true” si elles sont différentes de zéro, tandis que les valeurs nulles correspondent à “false”. * NAN (PHP’s Not-a-Number) equates to true, i.e. NAN == true is true. This is because NAN is a nonzero floating-point value. * Zero-values include both +0 and -0 as defined by IEEE 754. PHP does not distinguish between +0 and -0 in its double-precision floating-point, i.e. floatval('0') == floatval('-0') is true. * In fact, floatval('0') === floatval('-0'). * Additionally, both floatval('0') == false and floatval('-0') == false.

$var = NAN;
$var_is_true = ($var == true); // true
$var_is_false = ($var == false); // false

$var = floatval('-0');
$var_is_true = ($var == true); // false
$var_is_false = ($var == false); // true

$var = floatval('0') == floatval('-0');
$var_is_true = ($var == true); // false
$var_is_false = ($var == false); // true

OPÉRATEUR IDENTIQUE

Dans la Documentation PHP pour les opérateurs de comparaison, il y a un opérateur identique ===. Cet opérateur peut être utilisé pour vérifier si une variable est identique à une valeur de référence :

$var = null;
$var_is_null = $var === null; // true
$var_is_true = $var === true; // false
$var_is_false = $var === false; // false

Il a un opérateur non identique correspondant  !== :

$var = null;
$var_is_null = $var !== null; // false
$var_is_true = $var !== true; // true
$var_is_false = $var !== false; // true

L’opérateur identique peut être utilisé comme alternative aux fonctions de langage comme is_null().

CAS D’UTILISATION AVEC strpos()

La fonction de langage strpos($haystack, $needle) est utilisée pour localiser l’index auquel $needle apparaît dans $haystack, ou s’il apparaît du tout. La fonction strpos() est sensible à la casse ; si la recherche insensible à la casse est ce dont vous avez besoin, vous pouvez utiliser stripos($haystack, $needle)

La fonction strpos & stripos contient également le troisième paramètre offset (int) qui, s’il est spécifié, la recherche commencera ce nombre de caractères comptés depuis le début de la chaîne. Contrairement à strrpos et strripos, le décalage ne peut pas être négatif

La fonction peut retourner :

  • 0 si $needle se trouve au début de $haystack ;
  • un entier non nul spécifiant l’index si $needle se trouve ailleurs que le début de $haystack ;
  • et la valeur false si $needle n’est pas trouvé nulle part dans $haystack.

Parce que 0 et false ont la véracité false en PHP mais représentent des situations distinctes pour strpos(), il est important de les distinguer et d’utiliser le même opérateur === pour rechercher exactement false et pas seulement une valeur qui équivaut à false.

$idx = substr($haystack, $needle);
if ($idx === false) 
{
    // logic for when $needle not found in $haystack
} 
else
{
    // logic for when $needle found in $haystack
}

Alternativement, en utilisant l’opérateur non identique :

$idx = substr($haystack, $needle);
if ($idx !== false) 
{
    // logic for when $needle found in $haystack
} 
else
{
    // logic for when $needle not found in $haystack
}

Obtenir toutes les variables définies #

[get_defined_vars()][1] renvoie un tableau avec tous les noms et valeurs des variables définies dans la portée dans laquelle la fonction est appelée. Si vous souhaitez imprimer des données, vous pouvez utiliser des fonctions standard pour générer des données lisibles par l’homme, comme [print_r][2] ou [var_dump][3].

var_dump(get_defined_vars());

Remarque : Cette fonction ne renvoie généralement que 4 [superglobals][4] : $_GET,$_POST,$_COOKIE,$_FILES. Les autres superglobales ne sont renvoyées que si elles ont été utilisées quelque part dans le code. Cela est dû à la directive [auto_globals_jit][5] qui est activée par défaut. Lorsqu’il est activé, les variables $_SERVER et $_ENV sont créées lors de leur première utilisation (Just In Time) au lieu du démarrage du script. Si ces variables ne sont pas utilisées dans un script, l’activation de cette directive entraînera un gain de performances.
