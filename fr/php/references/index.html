Syntaxe #

  • $foo = 1 ; $bar = &$foo; // $foo et $bar pointent sur la même valeur : 1
  • $var = 1; function calc(&$var) { $var *= 15; } calc($var); echo $var;

Lors de l’affectation de deux variables par référence, les deux variables pointent vers la même valeur. Prenons l’exemple suivant :

$foo = 1;
$bar = &$foo;

$foo ne pointe pas vers $bar. $foo et $bar pointent tous les deux vers la même valeur de $foo, qui est 1. Pour illustrer:

$baz = &$bar;
unset($bar);
$baz++;

Si nous avions une relation pointe vers, celle-ci serait maintenant rompue après le unset() ; à la place, $foo et $baz pointent toujours vers la même valeur, qui est 2.

Attribuer par référence #

C’est la première phase du référencement. Essentiellement, lorsque vous [assignez par référence][1], vous autorisez deux variables à partager la même valeur en tant que telles.

$foo = &$bar;

$foo et $bar sont égaux ici. Ils ne pointent pas l’un vers l’autre. Ils pointent vers le même endroit (la “valeur”).

Vous pouvez également attribuer par référence dans la construction de langage array(). Bien qu’il ne s’agisse pas strictement d’une affectation par référence.

$foo = 'hi';
$bar = array(1, 2);
$array = array(&$foo, &$bar[0]);

Note, cependant, que les références à l’intérieur des tableaux sont potentiellement dangereux. Faire une affectation normale (pas par référence) avec un la référence sur le côté droit ne transforme pas le côté gauche en référence, mais les références à l’intérieur des tableaux sont conservées dans ces affectations. Cela s’applique également aux appels de fonction où le tableau est transmis par valeur.

L’affectation par référence n’est pas seulement limitée aux variables et aux tableaux, elle est également présente pour les fonctions et toutes les associations “passe par référence”.

function incrementArray(&$arr) {
    foreach ($arr as &$val) {
        $val++;
    }
}

function &getArray() {
    static $arr = [1, 2, 3];
    return $arr;
}

incrementArray(getArray());
var_dump(getArray()); // prints an array [2, 3, 4]

L’affectation est essentielle dans la définition de la fonction comme ci-dessus. Vous ne pouvez pas passer une expression par référence, seulement une valeur/variable. D’où l’instanciation de $a dans bar().

[1] : http://php.net/manual/en/language.references.whatdo.php#language.references.whatdo.assign

Retour par référence #

Parfois, il arrive le temps pour vous de revenir implicitement par référence.

Le retour par référence est utile lorsque vous souhaitez utiliser une fonction pour trouver à quelle variable une référence doit être liée. Ne pas utiliser retour par référence pour gagner en performance. Le moteur va l’optimiser automatiquement par lui-même. Ne renvoie les références que lorsque vous avez une raison technique valable de le faire.

* Tiré de la [Documentation PHP pour le retour par référence][1].*

Il existe de nombreuses formes différentes de retour par référence, y compris l’exemple suivant :

function parent(&$var) {
    echo $var;
    $var = "updated";
}

function &child() {
    static $a = "test";
    return $a;
}

parent(child()); // returns "test"
parent(child()); // returns "updated"

Le retour par référence n’est pas limité aux références de fonction. Il est également possible d’appeler implicitement la fonction :

function &myFunction() {
    static $a = 'foo';
    return $a;
}

$bar = &myFunction();
$bar = "updated"
echo myFunction();

Vous ne pouvez pas directement référencer un appel de fonction, il doit être assigné à une variable avant de l’exploiter. Pour voir comment cela fonctionne, essayez simplement echo &myFunction();.

Remarques #

  • Vous devez spécifier une référence (&) aux deux endroits où vous avez l’intention de l’utiliser. Cela signifie, pour votre définition de fonction (function &myFunction() {...) et dans la référence d’appel (function callFunction(&$variable) {... ou &myFunction();).
  • Vous ne pouvez renvoyer une variable que par référence. D’où l’instanciation de $a dans l’exemple ci-dessus. Cela signifie que vous ne pouvez pas renvoyer une expression, sinon une erreur PHP E_NOTICE sera générée (Remarque : Seules les références de variables doivent être renvoyées par référence dans ......).
  • Le retour par référence a des cas d’utilisation légitimes, mais je dois avertir qu’ils doivent être utilisés avec parcimonie, uniquement après avoir exploré toutes les autres options potentielles pour atteindre le même objectif.

[1] : http://php.net/manual/en/language.references.return.php

Passer par référence #

Cela vous permet de passer une variable par référence à une fonction ou un élément qui vous permet de modifier la variable d’origine.

Le passage par référence n’est pas limité aux seules variables, les éléments suivants peuvent également être passés par référence :

  • Nouvelles déclarations, par ex. foo(nouvelle SomeClass)
  • Références renvoyées par les fonctions

Tableaux #

Une utilisation courante de [“passing-by-reference”][1] consiste à modifier les valeurs initiales dans un tableau sans aller jusqu’à créer de nouveaux tableaux ou à encombrer votre espace de noms. Le passage par référence est aussi simple que de précéder/préfixer la variable avec un & => &$myElement.

Vous trouverez ci-dessous un exemple d’exploitation d’un élément d’un tableau et d’ajout simple de 1 à sa valeur initiale.

$arr = array(1, 2, 3, 4, 5);

foreach($arr as &$num) {
    $num++;
}

Désormais, lorsque vous exploitez un élément dans $arr, l’élément d’origine sera mis à jour à mesure que la référence a été augmentée. Vous pouvez le vérifier en :

print_r($arr);

Remarque

Vous devez faire attention lors de l’attelage passe par référence dans les boucles. À la fin de la boucle ci-dessus, $num contient toujours une référence au dernier élément du tableau. L’assigner après la boucle finira par manipuler le dernier élément du tableau ! Vous pouvez vous assurer que cela ne se produise pas en unset()’ après la boucle :

$monTableau = tableau(1, 2, 3, 4, 5);

foreach($myArray as &$num) { $num++ ; } unset($num); Ce qui précède vous assurera de ne pas rencontrer de problèmes. Un exemple de problèmes qui pourraient en découler est présent dans [cette question sur StackOverflow][2].

Les fonctions #

Une autre utilisation courante du passage par référence concerne les fonctions. La modification de la variable d’origine est aussi simple que :

$var = 5;
// define
function add(&$var) {
    $var++;
}
// call
add($var);

Ce qui peut être vérifié en faisant “écho” à la variable d’origine.

echo $var;

Il existe diverses restrictions concernant les fonctions, comme indiqué ci-dessous dans la documentation PHP :

Remarque : Il n’y a pas de signe de référence sur un appel de fonction - uniquement sur la fonction définitions. Les définitions de fonctions suffisent à elles seules pour passer correctement l’argument par référence. À partir de PHP 5.3.0, vous recevrez un avertissement dire que le “passage par référence du temps d’appel” est obsolète lorsque vous utilisez & dans foo(&$a);. Et à partir de PHP 5.4.0, le passage par référence du temps d’appel était supprimé, donc son utilisation déclenchera une erreur fatale.

[1] : http://php.net/manual/en/language.references.pass.php [2] : http://stackoverflow.com/q/24902742/2518525
