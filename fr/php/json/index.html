[JSON][3] ([JavaScript Object Notation][2]) est un moyen indépendant de la plate-forme et du langage de sérialiser des objets en texte clair. Parce qu’il est souvent utilisé sur le Web, tout comme PHP, il existe une [extension de base] [1] pour travailler avec JSON en PHP.

[1] : https://secure.php.net/manual/en/book.json.php [2] : https://en.wikipedia.org/wiki/JSON [3] : http://www.json.org

Syntaxe #

  • chaîne json_encode ( mixte $value [, int $options = 0 [, int $depth = 512 ]] )
  • mixte json_decode ( string $json [, bool $assoc = false [, int $depth = 512 [, int $options = 0 ]]] )

Paramètres #

Paramètre Détails
json_encode -
valeur La valeur en cours de codage. Peut être n’importe quel type sauf une ressource. Toutes les données de chaîne doivent être encodées en UTF-8.
options Masque binaire composé de JSON_HEX_QUOT, JSON_HEX_TAG, JSON_HEX_AMP, JSON_HEX_APOS, JSON_NUMERIC_CHECK, JSON_PRETTY_PRINT, JSON_UNESCAPED_SLASHES, JSON_FORCE_OBJECT, JSON_PRESERVE_ZERO_FRACTION, JSON_UNESCAPED_UNICODE, JSON_PARTIAL_OUTPUT_ON_ERROR. Le comportement de ces constantes est décrit sur la page Constantes JSON.
depth Définit la profondeur maximale. Doit être supérieur à zéro.
décode_json -
json La chaîne json en cours de décodage. Cette fonction ne fonctionne qu’avec les chaînes encodées en UTF-8.
assoc La fonction devrait-elle renvoyer un tableau associatif au lieu d’objets.
options Bitmask des options de décodage JSON. Actuellement, seul JSON_BIGINT_AS_STRING est pris en charge (la valeur par défaut consiste à convertir les grands entiers en flottants)
  • La gestion de json_decode de JSON invalide est très floue et il est très difficile de déterminer de manière fiable si le décodage a réussi, json_decode renvoie null pour une entrée invalide, même si null est également un objet parfaitement valide pour le décodage de JSON. Pour éviter de tels problèmes, vous devez toujours appeler json_last_error chaque fois que vous l’utilisez.

Décodage d’une chaîne JSON #

La fonction [json_decode()][json-decode] prend une chaîne encodée en JSON comme premier paramètre et l’analyse dans une variable PHP.

Normalement, json_decode() renverra un objet de [\stdClass][1] si l’élément de niveau supérieur dans l’objet JSON est un dictionnaire ou un tableau indexé si l’objet JSON est un tableau . Il renverra également des valeurs scalaires ou NULL pour certaines valeurs scalaires, telles que des chaînes simples, "true", "false" et "null". Il renvoie également NULL sur toute erreur.

// Returns an object (The top level item in the JSON string is a JSON dictionary)
$json_string = '{"name": "Jeff", "age": 20, "active": true, "colors": ["red", "blue"]}';
$object = json_decode($json_string);
printf('Hello %s, You are %s years old.', $object->name, $object->age);
#> Hello Jeff, You are 20 years old.

// Returns an array (The top level item in the JSON string is a JSON array)
$json_string = '["Jeff", 20, true, ["red", "blue"]]';
$array = json_decode($json_string);
printf('Hello %s, You are %s years old.', $array[0], $array[1]);

Utilisez [var_dump()][var-dump] pour afficher les types et les valeurs de chaque propriété sur l’objet que nous avons décodé ci-dessus.

// Dump our above $object to view how it was decoded
var_dump($object);

Sortie (notez les types de variables) :

class stdClass#2 (4) {
 ["name"] => string(4) "Jeff"
 ["age"] => int(20)
 ["active"] => bool(true)
 ["colors"] =>
   array(2) {
     [0] => string(3) "red"
     [1] => string(4) "blue"
   }
}

Remarque : La variable types dans JSON a été convertie en son équivalent PHP.

Pour renvoyer un [tableau associatif][2] pour les objets JSON au lieu de renvoyer un objet, transmettez true comme [second paramètre][json-decode-parameters] à json_decode().

$json_string = '{"name": "Jeff", "age": 20, "active": true, "colors": ["red", "blue"]}';
$array = json_decode($json_string, true); // Note the second parameter
var_dump($array);

Sortie (notez la structure associative du tableau):

array(4) {
  ["name"] => string(4) "Jeff"
  ["age"] => int(20)
  ["active"] => bool(true)
  ["colors"] =>
  array(2) {
    [0] => string(3) "red"
    [1] => string(4) "blue"
  }
}

Le deuxième paramètre ($assoc) n’a aucun effet si la variable à retourner n’est pas un objet.

Remarque : Si vous utilisez le paramètre $assoc, vous perdrez la distinction entre un tableau vide et un objet vide. Cela signifie que réexécuter json_encode() sur votre sortie décodée entraînera une structure JSON différente.

Si la chaîne JSON a une “profondeur” supérieure à 512 éléments (20 éléments dans les versions antérieures à 5.2.3, ou 128 dans la version 5.2.3) en récursivité, la fonction json_decode() renvoie NULL. Dans les versions 5.3 ou ultérieures, cette limite peut être contrôlée à l’aide du troisième paramètre ("$depth"), comme indiqué ci-dessous.

D’après le manuel :

PHP implémente un sur-ensemble de JSON comme spécifié dans l’original [» RFC 4627][rfc4627] - il encodera et décodera également les types scalaires et NULL. La RFC 4627 ne prend en charge ces valeurs que lorsqu’elles sont imbriquées dans un tableau ou un objet. Bien que ce sur-ensemble soit cohérent avec la définition étendue de “texte JSON” dans la nouvelle [» RFC 7159][rfc7159] (qui vise à remplacer la RFC 4627) et [» ECMA-404][ecma404], cela peut entraîner des problèmes d’interopérabilité avec les anciens analyseurs JSON qui respectent strictement la RFC 4627 lors de l’encodage d’une seule valeur scalaire.

Cela signifie que, par exemple, une simple chaîne sera considérée comme un objet JSON valide en PHP :

$json = json_decode('"some string"', true);
var_dump($json, json_last_error_msg());

Production:

string(11) "some string"
string(8) "No error"

Mais les chaînes simples, pas dans un tableau ou un objet, ne font pas partie de la norme [RFC 4627][rfc4627]. Par conséquent, des vérificateurs en ligne tels que [JSLint][3], [JSON Formatter & Validator][4] (en mode RFC 4627) vous donneront une erreur.

Il y a un troisième paramètre $depth pour la profondeur de la récursivité (la valeur par défaut est 512), ce qui signifie la quantité d’objets imbriqués à l’intérieur de l’objet d’origine à décoder.

Il y a un quatrième paramètre $options. Il n’accepte actuellement qu’une seule valeur, JSON_BIGINT_AS_STRING. Le comportement par défaut (qui omet cette option) consiste à convertir les grands entiers en flottants au lieu de chaînes.

Variantes non minuscules non valides des littéraux vrai, faux et nul ne sont plus acceptés comme entrée valide.

Donc cet exemple :

var_dump(json_decode('tRue'), json_last_error_msg());
var_dump(json_decode('tRUe'), json_last_error_msg());
var_dump(json_decode('tRUE'), json_last_error_msg());
var_dump(json_decode('TRUe'), json_last_error_msg());
var_dump(json_decode('TRUE'), json_last_error_msg());
var_dump(json_decode('true'), json_last_error_msg());

Avant PHP 5.6 :

bool(true)
string(8) "No error"
bool(true)
string(8) "No error"
bool(true)
string(8) "No error"
bool(true)
string(8) "No error"
bool(true)
string(8) "No error"
bool(true)
string(8) "No error"

Et après:

NULL
string(12) "Syntax error"
NULL
string(12) "Syntax error"
NULL
string(12) "Syntax error"
NULL
string(12) "Syntax error"
NULL
string(12) "Syntax error"
bool(true)
string(8) "No error"

Un comportement similaire se produit pour false et null.

Notez que json_decode() renverra NULL si la chaîne ne peut pas être convertie.

$json = "{'name': 'Jeff', 'age': 20 }" ;  // invalid json 

$person = json_decode($json);
echo $person->name;    //  Notice: Trying to get property of non-object: returns null
echo json_last_error();     
#  4 (JSON_ERROR_SYNTAX)
echo json_last_error_msg(); 
#  unexpected character

Il n’est pas sûr de se fier uniquement à la valeur de retour “NULL” pour détecter les erreurs. Par exemple, si la chaîne JSON ne contient que "null", json_decode() renverra null, même si aucune erreur ne s’est produite.

Encoder une chaîne JSON #

La fonction [json_encode][json-encode] convertira un tableau PHP (ou, depuis PHP 5.4, un objet qui implémente l’interface JsonSerializable) en une chaîne codée JSON. Il renvoie une chaîne codée JSON en cas de succès ou FALSE en cas d’échec.

$array = [
    'name' => 'Jeff',
    'age' => 20,
    'active' => true,
    'colors' => ['red', 'blue'],
    'values' => [0=>'foo', 3=>'bar'],
];

Lors de l’encodage, les types de données PHP chaîne, entier et booléen sont convertis en leur équivalent JSON. Les tableaux associatifs sont encodés en tant qu’objets JSON et, lorsqu’ils sont appelés avec des arguments par défaut, les tableaux indexés sont encodés en tant que tableaux JSON. (Sauf si les clés du tableau ne sont pas une séquence numérique continue commençant à 0, auquel cas le tableau sera encodé en tant qu’objet JSON.)

echo json_encode($array);

Production:

{"name":"Jeff","age":20,"active":true,"colors":["red","blue"],"values":{"0":"foo","3":"bar"}}

Arguments #

Depuis PHP 5.3, le deuxième argument de json_encode est un masque binaire qui peut être un ou plusieurs des éléments suivants.

Comme avec n’importe quel masque de bits, ils peuvent être combinés avec l’opérateur OU binaire |.

[JSON_FORCE_OBJECT][json-force-object] #

Force la création d’un objet au lieu d’un tableau

$array = ['Joel', 23, true, ['red', 'blue']];
echo json_encode($array);
echo json_encode($array, JSON_FORCE_OBJECT);

Production:

["Joel",23,true,["red","blue"]]
{"0":"Joel","1":23,"2":true,"3":{"0":"red","1":"blue"}}

[JSON_HEX_TAG][json-hex-tag], [JSON_HEX_AMP][json-hex-amp], [JSON_HEX_APOS][json-hex-apos], [JSON_HEX_QUOT][json-hex -quot] #

Assure les conversions suivantes lors de l’encodage :

Constante Entrée Sortie
JSON_HEX_TAG < \u003C
JSON_HEX_TAG > \u003E
JSON_HEX_AMP & \u0026
JSON_HEX_APOS ' \u0027
JSON_HEX_QUOT " \u0022 
$array = ["tag"=>"<>", "amp"=>"&", "apos"=>"'", "quot"=>"\""];
echo json_encode($array);
echo json_encode($array, JSON_HEX_TAG | JSON_HEX_AMP | JSON_HEX_APOS | JSON_HEX_QUOT);

Production:

{"tag":"<>","amp":"&","apos":"'","quot":"\""}
{"tag":"\u003C\u003E","amp":"\u0026","apos":"\u0027","quot":"\u0022"}

[JSON_NUMERIC_CHECK][json-numeric-check] #

Garantit que les chaînes numériques sont converties en nombres entiers.

$array = ['23452', 23452];
echo json_encode($array);
echo json_encode($array, JSON_NUMERIC_CHECK);

Production:

["23452",23452]    
[23452,23452]

[JSON_PRETTY_PRINT][json-pretty-print] #

Rend le JSON facilement lisible

$array = ['a' => 1, 'b' => 2, 'c' => 3, 'd' => 4];
echo json_encode($array);

}

[JSON_UNESCAPED_SLASHES][json-unscaped-slashes] #

Inclut des barres obliques / sans échappement dans la sortie

$array = ['filename' => 'example.txt', 'path' => '/full/path/to/file/'];
echo json_encode($array);
echo json_encode($array, JSON_UNESCAPED_SLASHES);

Production:

{"filename":"example.txt","path":"/full/path/to/file"}

[JSON_UNESCAPED_UNICODE][json-unscaped-unicode] #

Inclut des caractères encodés en UTF8 dans la sortie au lieu des chaînes encodées en \u

$blues = ["english"=>"blue", "norwegian"=>"blå", "german"=>"blau"];
echo json_encode($blues);
echo json_encode($blues, JSON_UNESCAPED_UNICODE);

Production:

{"english":"blue","norwegian":"bl\u00e5","german":"blau"}
{"english":"blue","norwegian":"blå","german":"blau"}

[JSON_PARTIAL_OUTPUT_ON_ERROR][json-partial-output-on-error] #

Permet à l’encodage de continuer si des valeurs non encodables sont rencontrées.

$fp = fopen("foo.txt", "r");
$array = ["file"=>$fp, "name"=>"foo.txt"];
echo json_encode($array); // no output
echo json_encode($array, JSON_PARTIAL_OUTPUT_ON_ERROR);

Production:

{"file":null,"name":"foo.txt"}

[JSON_PRESERVE_ZERO_FRACTION][json-preserve-zero] #

Garantit que les flottants sont toujours encodés en tant que flottants.

$array = [5.0, 5.5];
echo json_encode($array);
echo json_encode($array, JSON_PRESERVE_ZERO_FRACTION);

Production:

[5,5.5]
[5.0,5.5]

[JSON_UNESCAPED_LINE_TERMINATORS][json-unescaped-line-terminators] #

Lorsqu’il est utilisé avec JSON_UNESCAPED_UNICODE, revient au comportement des anciennes versions de PHP et * n’échappe pas * les caractères U + 2028 LINE SEPARATOR et U + 2029 PARAGRAPH SEPARATOR. Bien que valides en JSON, ces caractères ne le sont pas en JavaScript. Le comportement par défaut de JSON_UNESCAPED_UNICODE a donc été modifié dans la version 7.1.

$array = ["line"=>"\xe2\x80\xa8", "paragraph"=>"\xe2\x80\xa9"];
echo json_encode($array, JSON_UNESCAPED_UNICODE);
echo json_encode($array, JSON_UNESCAPED_UNICODE | JSON_UNESCAPED_LINE_TERMINATORS);

Production:

{"line":"\u2028","paragraph":"\u2029"}
{"line":" ","paragraph":" "}

Débogage des erreurs JSON #

Lorsque json_encode ou json_decode ne parvient pas à analyser la chaîne fournie, il renverra false. PHP lui-même ne générera aucune erreur ou avertissement lorsque cela se produit, il incombe à l’utilisateur d’utiliser les fonctions [json_last_error()][json-last-error] et [json_last_error_msg()][json-last-error-msg] pour vérifier si une erreur s’est produite et agir en conséquence dans votre application (la déboguer, afficher un message d’erreur, etc.).

L’exemple suivant montre une erreur courante lors de l’utilisation de JSON, un échec de décodage/encodage d’une chaîne JSON (en raison du passage d’une mauvaise chaîne encodée en UTF-8, par exemple).

// An incorrectly formed JSON string
$jsonString = json_encode("{'Bad JSON':\xB1\x31}");

if (json_last_error() != JSON_ERROR_NONE) {
    printf("JSON Error: %s", json_last_error_msg());
}

#> JSON Error: Malformed UTF-8 characters, possibly incorrectly encoded

[json_last_error_msg][json-last-error-msg] #

[json_last_error_msg()][json-last-error-msg] renvoie un message lisible par l’homme de la dernière erreur survenue lors de la tentative d’encodage/décodage d’une chaîne.

  • Cette fonction renverra toujours une chaîne, même si aucune erreur ne s’est produite. La chaîne sans erreur par défaut est “Pas d’erreur”
  • Il renverra false si une autre erreur (inconnue) s’est produite
  • Faites attention lorsque vous l’utilisez dans des boucles, car [json_last_error_msg][json-last-error-msg] sera remplacé à chaque itération.

Vous ne devez utiliser cette fonction que pour obtenir le message à afficher, pas pour le tester dans les instructions de contrôle.

// Don't do this:
if (json_last_error_msg()){} // always true (it's a string)
if (json_last_error_msg() != "No Error"){} // Bad practice

// Do this: (test the integer against one of the pre-defined constants)
if (json_last_error() != JSON_ERROR_NONE) {
    // Use json_last_error_msg to display the message only, (not test against it)
    printf("JSON Error: %s", json_last_error_msg());
}

Cette fonction n’existe pas avant PHP 5.5. Voici une implémentation polyfill :

if (!function_exists('json_last_error_msg')) {
    function json_last_error_msg() {
        static $ERRORS = array(
            JSON_ERROR_NONE => 'No error',
            JSON_ERROR_DEPTH => 'Maximum stack depth exceeded',
            JSON_ERROR_STATE_MISMATCH => 'State mismatch (invalid or malformed JSON)',
            JSON_ERROR_CTRL_CHAR => 'Control character error, possibly incorrectly encoded',
            JSON_ERROR_SYNTAX => 'Syntax error',
            JSON_ERROR_UTF8 => 'Malformed UTF-8 characters, possibly incorrectly encoded'
        );

        $error = json_last_error();
        return isset($ERRORS[$error]) ? $ERRORS[$error] : 'Unknown error';
    }
}

[json_last_error][json-dernière-erreur] #

[json_last_error()][json-last-error] renvoie un entier mappé à l’une des constantes prédéfinies fournies par PHP.

Constante Signification
JSON_ERROR_NONE Aucune erreur n’est survenue
JSON_ERROR_DEPTH La profondeur maximale de la pile a été dépassée
JSON_ERROR_STATE_MISMATCH JSON invalide ou malformé
JSON_ERROR_CTRL_CHAR Erreur de caractère de contrôle, peut-être mal codé
JSON_ERROR_SYNTAX Erreur de syntaxe (depuis PHP 5.3.3)
JSON_ERROR_UTF8 Caractères UTF-8 malformés, peut-être mal encodés (depuis PHP 5.5.0)
JSON_ERROR_RECURSION Une ou plusieurs références récursives dans la valeur à encoder
JSON_ERROR_INF_OR_NAN Une ou plusieurs valeurs NAN ou INF dans la valeur à coder
JSON_ERROR_UNSUPPORTED_TYPE Une valeur d’un type qui ne peut pas être codé a été donnée

Utilisation de JsonSerializable dans un objet #

Lorsque vous créez des API REST, vous devrez peut-être réduire les informations d’un objet à transmettre à l’application cliente. A cet effet, cet exemple illustre comment utiliser l’interface JsonSerialiazble.

Dans cet exemple, la classe User étend en fait un objet de modèle de base de données d’un ORM hypothétique.

class User extends Model implements JsonSerializable {
    public $id;
    public $name;
    public $surname;
    public $username;
    public $password;
    public $email;
    public $date_created;
    public $date_edit;
    public $role;
    public $status;

    public function jsonSerialize() {
        return [
            'name' => $this->name,
            'surname' => $this->surname,
            'username' => $this->username
        ];
    }
}

Ajoutez l’implémentation JsonSerializable à la classe, en fournissant la méthode jsonSerialize().

public function jsonSerialize()

Maintenant, dans votre contrôleur d’application ou votre script, lors du passage de l’objet User à json_encode(), vous obtiendrez le tableau encodé json de retour de la méthode jsonSerialize() au lieu de l’objet entier.

json_encode($User);

Reviendra :

{"name":"John", "surname":"Doe", "username" : "TestJson"}

Exemple de valeurs de propriétés. #

Cela réduira à la fois la quantité de données renvoyées par un point de terminaison RESTful et permettra d’exclure les propriétés d’objet d’une représentation json.

Utilisation de propriétés privées et protégées avec json_encode()

Pour éviter d’utiliser JsonSerializable, il est également possible d’utiliser des propriétés privées ou protégées pour masquer les informations de classe de la sortie json_encode(). La classe n’a alors pas besoin d’implémenter \JsonSerializable.

La fonction json_encode() n’encodera que les propriétés publiques d’un classe en JSON.

<?php

class User {
    // private properties only within this class
    private $id;
    private $date_created;
    private $date_edit;

    // properties used in extended classes
    protected $password;
    protected $email;
    protected $role;
    protected $status;

    // share these properties with the end user        
    public $name;
    public $surname;
    public $username;

    // jsonSerialize() not needed here
}        

$theUser = new User();

var_dump(json_encode($theUser));

Production: #

string(44) "{"name":null,"surname":null,"username":null}"

Header json et la réponse renvoyée #

En ajoutant un en-tête avec un type de contenu au format JSON :

<?php
 $result = array('menu1' => 'home', 'menu2' => 'code php', 'menu3' => 'about');

//return the json response :
header('Content-Type: application/json');  // <-- header declaration
echo json_encode($result, true);    // <--- encode
exit();

L’en-tête est là pour que votre application puisse détecter quelles données ont été renvoyées et comment elle doit les gérer.
Notez que : l’en-tête de contenu n’est qu’une information sur le type de données renvoyées.

Si vous utilisez UTF-8, vous pouvez utiliser :

header("Content-Type: application/json;charset=utf-8");

Exemple jQuery :

$.ajax({
