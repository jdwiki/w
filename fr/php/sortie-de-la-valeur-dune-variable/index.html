Sortie de la valeur d'une variable

Pour construire un programme PHP dynamique et interactif, il est utile de sortir des variables et leurs valeurs. Le langage PHP permet plusieurs méthodes de sortie de valeur. Cette rubrique couvre les méthodes standard d’impression d’une valeur en PHP et où ces méthodes peuvent être utilisées.

Les variables en PHP sont de types variés. Selon le cas d’utilisation, vous souhaiterez peut-être les afficher dans le navigateur sous forme de rendu HTML, les afficher pour le débogage ou les afficher sur le terminal (si vous exécutez une application via la ligne de commande).

Vous trouverez ci-dessous certaines des méthodes et des constructions de langage les plus couramment utilisées pour générer des variables :

  • [echo][echo] - Affiche une ou plusieurs chaînes
  • [print][print] - Affiche une chaîne et renvoie 1 (toujours)
  • [printf][printf] - Sort une chaîne formatée et renvoie la longueur de la chaîne sortie
  • [sprintf][sprintf] - Formate une chaîne et renvoie la chaîne formatée
  • [print_r][printr] - Affiche ou renvoie le contenu de l’argument sous la forme d’une chaîne lisible par l’homme
  • [var_dump][var_dump] - Affiche des informations de débogage lisibles par l’homme sur le contenu du ou des arguments, y compris son type et sa valeur
  • [var_export][var_export] - Sort ou renvoie une chaîne de rendu de la variable sous forme de code PHP valide, qui peut être utilisé pour recréer la valeur.

Remarque : Lorsque vous essayez de générer un objet sous forme de chaîne, PHP essaie de le convertir en chaîne (en appelant [__toString()][to-string] - si l’objet a une telle méthode) . S’il n’est pas disponible, une erreur similaire à L'objet de la classe [CLASS] n'a pas pu être converti en chaîne s’affichera. Dans ce cas, vous devrez inspecter davantage l’objet, voir : [outputting-a-structured-view-of-arrays-and-objects][structured-output].

écho et impression #

[echo][echo] et [print][print] sont des constructions de langage, pas des fonctions. Cela signifie qu’ils ne nécessitent pas de parenthèses autour de l’argument comme le fait une fonction (bien que l’on puisse toujours ajouter des parenthèses autour de presque toutes les expressions PHP et donc echo("test") ne fera pas de mal non plus). Ils génèrent la représentation sous forme de chaîne d’une variable, d’une constante ou d’une expression. Ils ne peuvent pas être utilisés pour imprimer des tableaux ou des objets.

  • Affectez la chaîne Joel à la variable $name

    $name = "Joel";

  • Afficher la valeur de $name en utilisant echo & print

    echo $name;   #> Joel
print $name;  #> Joel

  • Les parenthèses ne sont pas obligatoires, mais peuvent être utilisées

    echo($name);  #> Joel
print($name); #> Joel

  • Utilisation de plusieurs paramètres (seulement echo)

    echo $name, "Smith";       #> JoelSmith
echo($name, " ", "Smith"); #> Joel Smith

  • print, contrairement à echo, est une expression (elle renvoie 1), et peut donc être utilisée à plusieurs endroits :

    print("hey") && print(" ") && print("you"); #> you11

  • Ce qui précède est équivalent à :

    print ("hey" && (print (" " && print "you"))); #> you11

Notation abrégée pour echo #

Lorsque en dehors des balises PHP, une notation abrégée pour echo est disponible par défaut, en utilisant <?= pour commencer la sortie et ?> pour la terminer. Par exemple:

<p><?=$variable?></p>    
<p><?= "This is also PHP" ?></p>

Notez qu’il n’y a pas de ; de fin. Cela fonctionne parce que la balise PHP de fermeture agit comme le terminateur pour l’instruction unique. Ainsi, il est conventionnel d’omettre le point-virgule dans cette notation abrégée.

Priorité de print #

Bien que le print soit une construction de langage, il a la priorité comme opérateur. Il place entre les opérateurs = += -= *= **= /= .= %= &= et et et a quitté l’association. Exemple:

echo '1' . print '2' + 3; //output 511

Même exemple avec parenthèses :

echo '1' . print ('2' + 3); //output 511

Différences entre echo et print #

En bref, il y a deux différences principales :

  • print ne prend qu’un seul paramètre, tandis que echo peut avoir plusieurs paramètres.
  • print renvoie une valeur et peut donc être utilisé comme expression.

Sortie d’une vue structurée de tableaux et d’objets #

[print_r()][print-r] - Sortie de tableaux et objets pour le débogage #

[print_r][print-r] affichera un format lisible par l’homme d’un tableau ou d’un objet.

Vous pouvez avoir une variable qui est un tableau ou un objet. Essayer de le sortir avec un echo renverra l’erreur : Avis : conversion de tableau en chaîne. Vous pouvez à la place utiliser la fonction print_r pour vider un format lisible par l’homme de cette variable.

Vous pouvez passer true comme deuxième paramètre pour renvoyer le contenu sous forme de chaîne.

$myobject = new stdClass();
$myobject->myvalue = 'Hello World';
$myarray = [ "Hello", "World" ];
$mystring = "Hello World";
$myint = 42;

// Using print_r we can view the data the array holds.
print_r($myobject);
print_r($myarray);
print_r($mystring);
print_r($myint);

Cela génère ce qui suit :

stdClass Object
(
    [myvalue] => Hello World
)
Array
(
    [0] => Hello
    [1] => World
)
Hello World
42

De plus, la sortie de print_r peut être capturée sous forme de chaîne, plutôt que simplement renvoyée en écho. Par exemple, le code suivant videra la version formatée de $myarray dans une nouvelle variable :

$formatted_array = print_r($myarray, true);

Notez que si vous affichez la sortie de PHP dans un navigateur et qu’elle est interprétée comme HTML, les sauts de ligne ne seront pas affichés et la sortie sera beaucoup moins lisible à moins que vous ne fassiez quelque chose comme

echo '<pre>' . print_r($myarray, true) . '</pre>';

L’ouverture du code source d’une page formatera également votre variable de la même manière sans l’utilisation de la balise <pre>.

Vous pouvez également indiquer au navigateur que ce que vous affichez est du texte brut et non du HTML :

header('Content-Type: text/plain; charset=utf-8');
print_r($myarray);

[var_dump()][var-dump] - Affiche des informations de débogage lisibles par l’homme sur le contenu du ou des arguments, y compris son type et sa valeur #

La sortie est plus détaillée comme [comparé][var_dump_vs_print_r] à print_r car elle affiche également le type de la variable avec sa valeur et d’autres informations telles que les ID d’objet, les tailles de tableau, les longueurs de chaîne, marqueurs de référence, etc.

Vous pouvez utiliser [var_dump][var-dump] pour générer une version plus détaillée pour le débogage.

var_dump($myobject, $myarray, $mystring, $myint);

La sortie est plus détaillée :

object(stdClass)#12 (1) {
  ["myvalue"]=>
  string(11) "Hello World"
}
array(2) {
  [0]=>
  string(5) "Hello"
  [1]=>
  string(5) "World"
}
string(11) "Hello World"
int(42)

Remarque : Si vous utilisez xDebug dans votre environnement de développement, la sortie de var_dump est limitée/tronquée par défaut. Voir la [documentation officielle] [1] pour plus d’informations sur les options permettant de modifier cela.

[var_export()][var-export] - Code PHP de sortie valide #

[var_export()][var-export] vide une représentation analysable PHP de l’élément.

Vous pouvez passer true comme deuxième paramètre pour renvoyer le contenu dans une variable.

var_export($myarray);
var_export($mystring);
var_export($myint);

La sortie est un code PHP valide :

array (
  0 => 'Hello',
  1 => 'World',
)
'Hello World'
42

Pour mettre le contenu dans une variable, vous pouvez faire ceci :

$array_export = var_export($myarray, true);
$string_export = var_export($mystring, true);
$int_export = var_export($myint, 1); // any `Truthy` value

Après cela, vous pouvez le sortir comme ceci :

printf('$myarray = %s; %s', $array_export, PHP_EOL);
printf('$mystring = %s; %s', $string_export, PHP_EOL);
printf('$myint = %s; %s', $int_export, PHP_EOL);

Cela produira la sortie suivante :

$myarray = array (
  0 => 'Hello',
  1 => 'World',
);
$mystring = 'Hello World';
$myint = 42;

Concaténation de chaînes avec echo #

Vous pouvez utiliser [concaténation pour joindre des chaînes][1] “de bout en bout” lors de leur sortie (avec echo ou print par exemple).

Vous pouvez concaténer des variables en utilisant un . (point/point).

// String variable
$name = 'Joel';

// Concatenate multiple strings (3 in this example) into one and echo it once done.
//      1. ↓        2. ↓            3. ↓    - Three Individual string items
echo '<p>Hello ' . $name . ', Nice to see you.</p>';
//               ↑       ↑                  - Concatenation Operators

#> "<p>Hello Joel, Nice to see you.</p>"

Semblable à la concaténation, echo (lorsqu’il est utilisé sans parenthèses) peut être utilisé pour combiner des chaînes et des variables (ainsi que d’autres expressions arbitraires) à l’aide d’une virgule (,).

$itemCount = 1;

echo 'You have ordered ', $itemCount, ' item', $itemCount === 1 ? '' : 's';
//                      ↑           ↑        ↑                - Note the commas

#> "You have ordered 1 item"

Concaténation de chaînes vs passage de plusieurs arguments à echo #

Passer plusieurs arguments à la commande echo est plus avantageux que la concaténation de chaînes dans certaines circonstances. Les arguments sont écrits dans la sortie dans le même ordre qu’ils sont passés.

echo "The total is: ", $x + $y;

Le problème avec la concaténation est que le point . est prioritaire dans l’expression. Si elle est concaténée, l’expression ci-dessus nécessite des parenthèses supplémentaires pour un comportement correct. La priorité du point affecte également les opérateurs ternaires.

echo "The total is: " . ($x + $y);

printf contre sprintf #

[printf][printf] va sortir une chaîne formatée en utilisant des espaces réservés

[sprintf][sprintf] va retourner la chaîne formatée

$name = 'Jeff';

// The `%s` tells PHP to expect a string
//            ↓  `%s` is replaced by  ↓
printf("Hello %s, How's it going?", $name);
#> Hello Jeff, How's it going?

// Instead of outputting it directly, place it into a variable ($greeting)
$greeting = sprintf("Hello %s, How's it going?", $name);
echo $greeting;
#> Hello Jeff, How's it going?

Il est aussi possible de formater un nombre avec ces 2 fonctions. Cela peut être utilisé pour formater une valeur décimale utilisée pour représenter l’argent afin qu’elle ait toujours 2 chiffres décimaux.

$money = 25.2;
printf('%01.2f', $money);
#> 25.20

Les deux fonctions [vprintf][vprintf] et [vsprintf ][vsprintf] fonctionnent comme [printf][printf] et [sprintf][sprintf], mais acceptent une chaîne de format et un tableau de valeurs , au lieu de variables individuelles.

Sortie de grands nombres entiers #

Sur les systèmes 32 bits, les entiers supérieurs à PHP_INT_MAX sont automatiquement convertis en flottants. La sortie de ces valeurs sous forme de valeurs entières (c’est-à-dire une notation non scientifique) peut être effectuée avec printf, en utilisant la représentation float, comme illustré ci-dessous :

foreach ([1, 2, 3, 4, 5, 6, 9, 12] as $p) {
    $i = pow(1024, $p);
    printf("pow(1024, %d) > (%7s) %20s %38.0F", $p, gettype($i), $i, $i);
    echo "  ", $i, "\n";
}
// outputs:
pow(1024,  1)  integer                 1024                                   1024  1024
pow(1024,  2)  integer              1048576                                1048576  1048576
pow(1024,  3)  integer           1073741824                             1073741824  1073741824
pow(1024,  4)   double        1099511627776                          1099511627776  1099511627776
pow(1024,  5)   double  1.1258999068426E+15                       1125899906842624  1.1258999068426E+15
pow(1024,  6)   double  1.1529215046068E+18                    1152921504606846976  1.1529215046068E+18
pow(1024,  9)   double  1.2379400392854E+27           1237940039285380274899124224  1.2379400392854E+27
pow(1024, 12)   double  1.3292279957849E+36  1329227995784915872903807060280344576  1.3292279957849E+36

Remarque : attention à la précision du flotteur qui n’est pas infinie !

Bien que cela semble joli, dans cet exemple artificiel, les nombres peuvent tous être représentés sous forme de nombre binaire puisqu’ils sont tous des puissances de 1024 (et donc de 2). Voir par exemple :

$n = pow(10, 27);
printf("%s %.0F\n", $n, $n);
// 1.0E+27 1000000000000000013287555072
