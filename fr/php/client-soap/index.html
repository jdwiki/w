Syntaxe #

  • [__getFunctions()][1] // Renvoie un tableau de fonctions pour le service (mode WSDL uniquement)
  • [__getTypes()][2] // Renvoie un tableau de types pour le service (mode WSDL uniquement)
  • [__getLastRequest()][3] // Renvoie le XML de la dernière requête (nécessite l’option trace)
  • [__getLastRequestHeaders()][4] // Renvoie les en-têtes de la dernière requête (nécessite l’option trace)
  • [__getLastResponse()][5] // Renvoie le XML de la dernière réponse (nécessite l’option trace)
  • [__getLastResponseHeaders()][6] // Renvoie les en-têtes de la dernière réponse (nécessite l’option trace)

[1] : http://php.net/manual/en/soapclient.getfunctions.php [2] : http://php.net/manual/en/soapclient.gettypes.php [3] : http://php.net/manual/en/soapclient.getlastrequest.php [4] : http://php.net/manual/en/soapclient.getlastrequestheaders.php [5] : http://php.net/manual/en/soapclient.getlastresponse.php [6] : http://php.net/manual/en/soapclient.getlastresponseheaders.php

Paramètres #

Paramètre Détails
$wsl URI de WSDL ou NULL si vous utilisez le mode non WSDL
$options Tableau d’options pour SoapClient. Le mode non-WSDL nécessite location et uri à définir, toutes les autres options sont facultatives. Voir le tableau ci-dessous pour les valeurs possibles.

La classe SoapClient est équipée d’une méthode __call. Ce n’est pas à appeler directement. Au lieu de cela, cela vous permet de faire :

$soap->requestInfo(['a', 'b', 'c']);

Cela appellera la méthode SOAP requestInfo.

Tableau des valeurs possibles de $options (Tableau de paires clé/valeur) :

Options Détails
emplacement URL du serveur SOAP. Requis en mode non WSDL. Peut être utilisé en mode WSDL pour remplacer l’URL.
uri Espace de noms cible du service SOAP. Requis en mode non WSDL.
style Les valeurs possibles sont SOAP_RPC ou SOAP_DOCUMENT. Valable uniquement en mode non WSDL.
utiliser Les valeurs possibles sont SOAP_ENCODED ou SOAP_LITERAL. Valable uniquement en mode non WSDL.
version_savon Les valeurs possibles sont SOAP_1_1 (par défaut) ou SOAP_1_2.
authentification Activez l’authentification HTTP. Les valeurs possibles sont SOAP_AUTHENTICATION_BASIC (défaut) ou SOAP_AUTHENTICATION_DIGEST.
connexion Nom d’utilisateur pour l’authentification HTTP
mot de passe Mot de passe pour l’authentification HTTP
hôte_proxy URL du serveur proxy
port_proxy Port du serveur proxy
connexion_proxy Nom d’utilisateur pour le proxy
mot_de_passe_proxy Mot de passe pour proxy
local_cert Chemin d’accès au certificat client HTTPS (pour l’authentification)
phrase de passe Phrase secrète pour le certificat client HTTPS
compression Compresser la requête/réponse. La valeur est un masque binaire de SOAP_COMPRESSION_ACCEPT avec SOAP_COMPRESSION_GZIP ou SOAP_COMPRESSION_DEFLATE. Par exemple : SOAP_COMPRESSION_ACCEPT | SOAP_COMPRESSION_GZIP.
encodage Encodage interne des caractères (TODO : valeurs possibles)
tracer Booléen, la valeur par défaut est FALSE. Permet le traçage des demandes afin que les défauts puissent être retracés. Active l’utilisation de __getLastRequest(), __getLastRequestHeaders(), __getLastResponse() et __getLastResponseHeaders().
plan de classe Mappez les types WSDL aux classes PHP. La valeur doit être un tableau avec des types WSDL comme clés et des noms de classe PHP comme valeurs.
exceptions Valeur booléenne. Faut-il des exceptions d’erreurs SOAP (de type `SoapFault).
connection_timeout Timeout (en secondes) pour la connexion au service SOAP.
carte de type Tableau de mappages de types. Le tableau doit être des paires clé/valeur avec les clés suivantes : type_name, type_ns (espace de noms URI), from_xml (rappel acceptant un paramètre de chaîne) et to_xml (rappel acceptant un paramètre d’objet).
cache_wsdl Comment (le cas échéant) le fichier WSDL doit-il être mis en cache. Les valeurs possibles sont WSDL_CACHE_NONE, WSDL_CACHE_DISK, WSDL_CACHE_MEMORY ou WSDL_CACHE_BOTH.
user_agent Chaîne à utiliser dans l’en-tête User-Agent.
contexte_flux Une ressource pour un contexte.
caractéristiques Masque binaire de SOAP_SINGLE_ELEMENT_ARRAYS, SOAP_USE_XSI_ARRAY_TYPE, SOAP_WAIT_ONE_WAY_CALLS.
keep_alive (Version PHP >= 5.4 uniquement) Valeur booléenne. Envoyez l’en-tête Connection : Keep-Alive (TRUE) ou l’en-tête Connection : Close (FALSE).
méthode_ssl (Version PHP >= 5.5 uniquement) Quelle version SSL/TLS utiliser. Les valeurs possibles sont SOAP_SSL_METHOD_TLS, SOAP_SSL_METHOD_SSLv2, SOAP_SSL_METHOD_SSLv3 ou SOAP_SSL_METHOD_SSLv23.

[Problème avec PHP 32 bits][1] : En PHP 32 bits, les chaînes numériques supérieures à 32 bits qui sont automatiquement transtypées en nombre entier par xs:long entraîneront l’atteinte de la limite de 32 bits, la transtypage à 2147483647. Pour contourner ce problème, convertissez les chaînes en flottant avant de les transmettre à __soapCall().

[1] : http://stackoverflow.com/questions/19228213/php-soapclient-soap-request-with-long-integer

Mode WSDL #

Tout d’abord, créez un nouvel objet SoapClient, en transmettant l’URL au fichier WSDL et éventuellement un tableau d’options.

// Create a new client object using a WSDL URL
$soap = new SoapClient('https://example.com/soap.wsdl', [
    # This array and its values are optional
    'soap_version' => SOAP_1_2,
    'compression' => SOAP_COMPRESSION_ACCEPT | SOAP_COMPRESSION_GZIP,
    'cache_wsdl' => WSDL_CACHE_BOTH,
    # Helps with debugging
    'trace' => TRUE,
    'exceptions' => TRUE
]);

Utilisez ensuite l’objet $soap pour appeler vos méthodes SOAP.

$result = $soap->requestData(['a', 'b', 'c']);

Mode non WSDL #

Ceci est similaire au mode WSDL, sauf que nous passons NULL comme fichier WSDL et que nous nous assurons de définir les options location et uri.

$soap = new SoapClient(NULL, [
    'location' => 'https://example.com/soap/endpoint',
    'uri' => 'namespace'
]);

Classmaps #

Lors de la création d’un client SOAP en PHP, vous pouvez également définir une clé classmap dans le tableau de configuration. Ce classmap définit quels types définis dans le WSDL doivent être mappés aux classes réelles, au lieu de la StdClass par défaut. La raison pour laquelle vous voudriez faire cela est que vous pouvez obtenir l’auto-complétion des champs et des appels de méthode sur ces classes, au lieu d’avoir à deviner quels champs sont définis sur la StdClass normale.

class MyAddress {
    public $country;
    public $city;
    public $full_name;
    public $postal_code; // or zip_code
    public $house_number;
}

class MyBook {
    public $name;
    public $author;

    // The classmap also allows us to add useful functions to the objects
    // that are returned from the SOAP operations.
    public function getShortDescription() {
        return "{$this->name}, written by {$this->author}";
    }
}

$soap_client = new SoapClient($link_to_wsdl, [
    // Other parameters
    "classmap" => [
        "Address" => MyAddress::class, // ::class simple returns class as string
        "Book" => MyBook::class,
    ]
]);

Après avoir configuré le classmap, chaque fois que vous effectuez une certaine opération qui renvoie un type Address ou Book, le SoapClient instancie cette classe, remplit les champs avec les données et les renvoie à partir de l’appel d’opération.

// Lets assume 'getAddress(1234)' returns an Address by ID in the database
$address = $soap_client->getAddress(1234);

// $address is now of type MyAddress due to the classmap
echo $address->country;

// Lets assume the same for 'getBook(1234)'
$book = $soap_client->getBook(124);

// We can not use other functions defined on the MyBook class
echo $book->getShortDescription();

// Any type defined in the WSDL that is not defined in the classmap
// will become a regular StdClass object
$author = $soap_client->getAuthor(1234);

// No classmap for Author type, $author is regular StdClass.
// We can still access fields, but no auto-completion and no custom functions
// to define for the objects.
echo $author->name;

Traçage de la requête et de la réponse SOAP #

Parfois, nous voulons regarder ce qui est envoyé et reçu dans la requête SOAP. Les méthodes suivantes renvoient le XML dans la requête et la réponse :

SoapClient::__getLastRequest()
SoapClient::__getLastRequestHeaders()
SoapClient::__getLastResponse()
SoapClient::__getLastResponseHeaders()

Par exemple, supposons que nous ayons une constante ENVIRONMENT et que la valeur de cette constante soit définie sur DEVELOPMENT, nous souhaitons afficher toutes les informations lorsque l’appel à getAddress génère une erreur. Une solution pourrait être :

try {
    $address = $soap_client->getAddress(1234);
} catch (SoapFault $e) {
    if (ENVIRONMENT === 'DEVELOPMENT') {
        var_dump(
            $soap_client->__getLastRequestHeaders()
            $soap_client->__getLastRequest(),
            $soap_client->__getLastResponseHeaders(),
            $soap_client->__getLastResponse()
        );
    }
    ...
}
