SplFixedArray #

Différence avec PHP Array #

Le type de tableau par défaut de PHP est en fait implémenté sous forme de cartes de hachage ordonnées, ce qui nous permet de créer des tableaux composés de paires clé/valeur où les valeurs peuvent être de n’importe quel type et les clés peuvent être des nombres ou des chaînes. Cependant, ce n’est pas traditionnellement la façon dont les tableaux sont créés.

[![Tableau PHP traditionnel Figure 1][1]][1]

Ainsi, comme vous pouvez le voir sur cette illustration, un tableau PHP normal peut être vu davantage comme un ensemble ordonné de paires clé/valeur, où chaque clé peut correspondre à n’importe quelle valeur. Remarquez que dans ce tableau, nous avons des clés qui sont à la fois des nombres et des chaînes, ainsi que des valeurs de différents types et la clé n’a aucune incidence sur l’ordre des éléments.

$arr = [
    9     => "foo",
    1     => 4.2,
    "bar" => null,
];

foreach($arr as $key => $value) {
    echo "$key => $value\n";
}

Ainsi, le code ci-dessus nous donnerait exactement ce à quoi nous nous attendions.

<pré> 9 => toto 1 => 4.2 barre =>

Les tableaux PHP réguliers sont également dimensionnés dynamiquement pour nous. Ils grandissent et rétrécissent au fur et à mesure que nous poussons et extrayons des valeurs vers et depuis le tableau, automatiquement.

Cependant, dans un tableau traditionnel, la taille est fixe et se compose entièrement du même type de valeur. De plus, plutôt que des clés, chaque valeur est accessible par son index, qui peut être déduit par son décalage dans le tableau.

[![SplFixedArray Figure 2][2]][2]

Puisque nous connaîtrions la taille d’un type donné et la taille fixe du tableau, un décalage est alors la taille du type * n où n représente la position de la valeur dans le tableau. Ainsi, dans l’exemple ci-dessus, $arr[0] nous donne 1, le premier élément du tableau et $arr[1] nous donne 2, et ainsi de suite.

SplFixedArray, cependant, ne limite pas le type de valeurs. Il limite uniquement les clés aux types de nombres. Il est également de taille fixe.

Cela rend SplFixedArrays plus efficace que les tableaux PHP normaux d’une manière particulière. Ils sont plus compacts et nécessitent donc moins de mémoire.

Instanciation du tableau #

SplFixedArray est implémenté en tant qu’objet, mais il est accessible avec la même syntaxe familière que vous accédez à un tableau PHP normal car ils implémentent l’interface ArrayAccess. Ils implémentent également les interfaces Countable et Iterator afin qu’ils se comportent de la même manière que vous seriez habitué aux tableaux se comportant en PHP (c’est-à-dire des choses comme count($arr) et foreach($arr as $k => $ v) fonctionnent de la même manière pour SplFixedArray que pour les tableaux normaux en PHP.

Le constructeur SplFixedArray prend un argument, qui est la taille du tableau.

$arr = new SplFixedArray(4);

$arr[0] = "foo";
$arr[1] = "bar";
$arr[2] = "baz";

foreach($arr as $key => $value) {
    echo "$key => $value\n";
}

Cela vous donne ce que vous attendez.

<pré> 0 => toto 1 => barre 2 => baz 3 =>

Cela fonctionne également comme prévu.

var_dump(count($arr));

Nous donne…

<pré> entier(4)

Remarquez que dans SplFixedArray, contrairement à un tableau PHP normal, la clé décrit l’ordre de l’élément dans notre tableau, car il s’agit d’un true index et pas seulement d’une map.

Redimensionner le tableau #

Gardez simplement à l’esprit que, comme le tableau est de taille fixe, count renverra toujours la même valeur. Ainsi, alors que unset($arr[1]) donnera $arr[1] === null, count($arr) reste toujours 4.

Donc, pour redimensionner le tableau, vous devrez appeler la méthode setSize.

$arr->setSize(3);

var_dump(count($arr));

foreach($arr as $key => $value) {
    echo "$key => $value\n";
}

Maintenant, nous obtenons…

<pré> entier(3) 0 => toto 1 => 2 => baz

Importer dans SplFixedArray et exporter depuis SplFixedArray #

Vous pouvez également importer/exporter un tableau PHP normal vers et depuis un SplFixedArray avec les méthodes fromArray et toArray.

$array      = [1,2,3,4,5];
$fixedArray = SplFixedArray::fromArray($array);

foreach($fixedArray as $value) {
    echo $value, "\n";
}

<pré> 1 2 3 4 5

Aller dans l’autre sens.

$fixedArray = new SplFixedArray(5);

$fixedArray[0] = 1;
$fixedArray[1] = 2;
$fixedArray[2] = 3;
$fixedArray[3] = 4;
$fixedArray[4] = 5;

$array = $fixedArray->toArray();

foreach($array as $value) {
    echo $value, "\n";
}

<pré> 1 2 3 4 5
