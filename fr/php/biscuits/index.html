Un cookie HTTP est un petit morceau de données envoyé à partir d’un site Web et stocké sur l’ordinateur de l’utilisateur par le navigateur Web de l’utilisateur pendant que l’utilisateur navigue.

Syntaxe #

  • bool setcookie( string $name [, string $value = "" [, int $expire = 0 [, string $path = "" [, string $domain = "" [, bool $secure = false [, bool $ httpuniquement = faux ]]]]]] )

Paramètres #

paramètre détail
nom Le nom du cookie. C’est également la clé que vous pouvez utiliser pour récupérer la valeur du super global $_COOKIE. Ceci est le seul paramètre requis
valeur La valeur à stocker dans le cookie. Ces données sont accessibles au navigateur, donc ne stockez rien de sensible ici.
expirer Un horodatage Unix représentant le moment où le cookie doit expirer. S’il est défini sur zéro, le cookie expirera à la fin de la session. S’il est défini sur un nombre inférieur à l’horodatage Unix actuel, le cookie expirera immédiatement.
chemin La portée du cookie. S’il est défini sur /, le cookie sera disponible dans l’ensemble du domaine. S’il est défini sur /some-path/, le cookie ne sera disponible que dans ce chemin et les descendants de ce chemin. Par défaut, le chemin actuel du fichier dans lequel le cookie est défini.
domaine Le domaine ou sous-domaine sur lequel le cookie est disponible. S’il est défini sur le domaine nu stackoverflow.com, le cookie sera disponible pour ce domaine et tous les sous-domaines. S’il est défini sur un sous-domaine meta.stackoverflow.com, le cookie ne sera disponible que sur ce sous-domaine et tous les sous-sous-domaines.
sécurisé Lorsqu’il est défini sur “TRUE”, le cookie ne sera défini que s’il existe une connexion HTTPS sécurisée entre le client et le serveur.
httpuniquement Spécifie que le cookie doit uniquement être mis à disposition via le protocole HTTP/S et ne doit pas être disponible pour les langages de script côté client tels que JavaScript. Uniquement disponible en PHP 5.2 ou version ultérieure.

Il convient de noter que le simple fait d’invoquer la fonction setcookie ne place pas simplement les données données dans le tableau superglobal $_COOKIE.

Par exemple, il ne sert à rien de faire :

setcookie("user", "Tom", time() + 86400, "/");
var_dump(isset($_COOKIE['user'])); // yields false or the previously set value

La valeur n’est pas encore là, pas avant le prochain chargement de la page. La fonction setcookie indique simplement “avec la prochaine connexion http, dites au client (navigateur) de définir ce cookie”. Ensuite, lorsque les en-têtes sont envoyés au navigateur, ils contiennent cet en-tête de cookie. Le navigateur vérifie ensuite si le cookie n’a pas encore expiré, et si ce n’est pas le cas, dans la requête http, il envoie le cookie au serveur et c’est alors que PHP le reçoit et place le contenu dans le tableau $_COOKIE.

Modifier un cookie #

La valeur d’un cookie peut être modifiée en réinitialisant le cookie

setcookie("user", "John", time() + 86400, "/"); // assuming there is a "user" cookie already

Les cookies font partie de l’en-tête HTTP, donc setcookie() doit être appelé avant que toute sortie ne soit envoyée au navigateur.

Lors de la modification d’un cookie, assurez-vous que les paramètres path et domain de setcookie() correspondent au cookie existant ou un nouveau cookie sera créé à la place.

La partie valeur du cookie sera automatiquement codée en URL lorsque vous envoyez le cookie, et lorsqu’elle est reçue, elle est automatiquement décodée et affectée à une variable du même nom que le nom du cookie

Paramétrage d’un cookie #

Un cookie est défini à l’aide de la fonction setcookie(). Étant donné que les cookies font partie de l’en-tête HTTP, vous devez définir des cookies avant d’envoyer une sortie au navigateur.

Exemple:

setcookie("user", "Tom", time() + 86400, "/"); // check syntax for function params

La description:

  • Crée un cookie avec le nom user
  • (Facultatif) La valeur du cookie est Tom
  • (Facultatif) Le cookie expirera dans 1 jour (86400 secondes)
  • (Facultatif) Le cookie est disponible sur l’ensemble du site Web /
  • (Facultatif) Le cookie est uniquement envoyé via HTTPS
  • (Facultatif) Le cookie n’est pas accessible aux langages de script tels que JavaScript

Un cookie créé ou modifié n’est accessible que lors de requêtes ultérieures (où path et domain correspondent) car le superglobal $_COOKIE n’est pas rempli immédiatement avec les nouvelles données.

Vérifier si un cookie est défini #

Utilisez la fonction isset() sur la variable superglobale $_COOKIE pour vérifier si un cookie est défini.

Exemple:

// PHP <7.0
if (isset($_COOKIE['user'])) {
    // true, cookie is set
    echo 'User is ' . $_COOKIE['user'];
else {
    // false, cookie is not set
    echo 'User is not logged in';
}

// PHP 7.0+
echo 'User is ' . $_COOKIE['user'] ?? 'User is not logged in';

Suppression d’un cookie #

Pour supprimer un cookie, définissez l’horodatage d’expiration sur une heure passée. Cela déclenche le mécanisme de suppression du navigateur :

setcookie('user', '', time() - 3600, '/');

Lors de la suppression d’un cookie, assurez-vous que les paramètres path et domain de setcookie() correspondent au cookie que vous essayez de supprimer ou un nouveau cookie, qui expire immédiatement, sera créé.

C’est aussi une bonne idée de désactiver la valeur $_COOKIE au cas où la page actuelle l’utiliserait :

unset($_COOKIE['user']);

Récupération d’un cookie #

Récupérer et générer un cookie nommé user

La valeur d’un cookie peut être récupérée à l’aide de la variable globale $_COOKIE. exemple si nous avons un cookie nommé user nous pouvons le récupérer comme ceci

echo $_COOKIE['user'];
