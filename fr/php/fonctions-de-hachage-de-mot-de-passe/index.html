Comme les services Web plus sécurisés évitent de stocker les mots de passe au format texte brut, des langages tels que PHP fournissent diverses fonctions de hachage (non déchiffrables) pour prendre en charge la norme industrielle plus sécurisée. Cette rubrique fournit de la documentation pour un hachage correct avec PHP.

Syntaxe #

  • chaîne password_hash ( chaîne $password , entier $algo [, array $options ] )
  • boolean password_verify ( string $password , string $hash )
  • boolean password_needs_rehash ( string $hash , integer $algo [, array $options ] )
  • tableau password_get_info ( chaîne $hash )

Avant PHP 5.5, vous pouvez utiliser le pack de compatibilité pour fournir les fonctions password_*. Il est fortement recommandé d’utiliser le pack de compatibilité si vous êtes en mesure de le faire.

Avec ou sans le pack de compatibilité, la fonctionnalité correcte de Bcrypt via crypt() repose sur PHP 5.3.7+ sinon vous devez restreindre les mots de passe à ASCII- uniquement des jeux de caractères.

Remarque : Si vous utilisez PHP 5.5 ou une version antérieure, vous utilisez une version non prise en charge de PHP qui ne reçoit plus aucune mise à jour de sécurité. Mettez à jour dès que possible, vous pouvez mettre à jour vos hachages de mot de passe par la suite.

Sélection d’algorithme #

Algorithmes sécurisés #

  • bcrypt est votre meilleure option tant que vous utilisez l’étirement de la clé pour augmenter le temps de calcul du hachage, car cela rend [les attaques par force brute extrêmement lentes](http://arstechnica.com/security/2015/08/cracking- tous-les-mots-de-passe-ashley-madison-piratés-pourraient-prendre-une-vie/).
  • argon2 est une autre option qui sera disponible dans PHP 7.2.

Algorithmes non sécurisés #

Les algorithmes de hachage suivants sont non sécurisés ou inadaptés et par conséquent ne doivent pas être utilisés. Ils n’ont jamais été adaptés au hachage de mots de passe, car ils sont conçus pour des résumés rapides au lieu de hachages de mots de passe lents et difficiles à forcer.

Si vous en utilisez un, y compris des sels, vous devez basculer vers l’un des algorithmes sécurisés recommandés dès que possible.

Algorithmes considérés comme non sécurisés :

  • MD4 - attaque par collision découverte en 1995
  • MD5 - attaque par collision découverte en 2005
  • SHA-1 - attaque par collision démontrée en 2015

Certains algorithmes peuvent être utilisés en toute sécurité comme algorithme de résumé de message pour prouver l’authenticité, mais jamais comme algorithme de hachage de mot de passe :

  • SHA-2
  • SHA-3

Notez que les hachages forts tels que SHA256 et SHA512 sont ininterrompus et robustes, mais il est généralement plus sûr d’utiliser les fonctions de hachage bcrypt ou argon2 car les attaques par force brute contre ces algorithmes sont beaucoup plus difficiles pour les ordinateurs classiques.

Création d’un hachage de mot de passe #

Créez des hachages de mot de passe à l’aide de [password_hash()][1] pour utiliser le hachage standard actuel des meilleures pratiques de l’industrie ou la dérivation de clé. Au moment de la rédaction, la norme est [bcrypt][2], ce qui signifie que PASSWORD_DEFAULT contient la même valeur que PASSWORD_BCRYPT.

$options = [
    'cost' => 12,
];

$hashedPassword = password_hash($plaintextPassword, PASSWORD_DEFAULT, $options);

Le troisième paramètre n’est pas obligatoire.

La valeur 'coût' doit être choisie en fonction du matériel de votre serveur de production. L’augmenter rendra le mot de passe plus coûteux à générer. Plus il est coûteux à générer, plus il faudra de temps à quiconque essaie de le cracker pour le générer également. Le coût doit idéalement être le plus élevé possible, mais en pratique il doit être fixé de manière à ne pas trop ralentir le tout. Quelque part entre 0,1 et 0,4 seconde serait acceptable. Utilisez la valeur par défaut en cas de doute.

Sur PHP inférieur à 5.5.0 les fonctions password_* ne sont pas disponibles. Vous devez utiliser [le pack de compatibilité][3] pour remplacer ces fonctions. Notez que le pack de compatibilité nécessite PHP 5.3.7 ou supérieur ou une version dans laquelle le correctif « $2y » est rétroporté (comme celui fourni par RedHat).

Si vous n’êtes pas en mesure de les utiliser, vous pouvez implémenter le hachage de mot de passe avec [crypt()][4] Comme password_hash() est implémenté comme un wrapper autour de la fonction crypt(), vous n’avez pas besoin de perdre Fonctionnalité.

// this is a simple implementation of a bcrypt hash otherwise compatible
// with `password_hash()`
// not guaranteed to maintain the same cryptographic strength of the full `password_hash()`
// implementation

// if `CRYPT_BLOWFISH` is 1, that means bcrypt (which uses blowfish) is available
// on your system
if (CRYPT_BLOWFISH == 1) {
    $salt = mcrypt_create_iv(16, MCRYPT_DEV_URANDOM);
    $salt = base64_encode($salt);
    // crypt uses a modified base64 variant
    $source = 'ABCDEFGHIJKLMNOPQRSTUVWXYZabcdefghijklmnopqrstuvwxyz0123456789+/';
    $dest = './ABCDEFGHIJKLMNOPQRSTUVWXYZabcdefghijklmnopqrstuvwxyz0123456789';
    $salt = strtr(rtrim($salt, '='), $source, $dest);
    $salt = substr($salt, 0, 22);
    // `crypt()` determines which hashing algorithm to use by the form of the salt string
    // that is passed in
    $hashedPassword = crypt($plaintextPassword, '$2y$10$'.$salt.'$');
}

Salt pour le hachage du mot de passe #

Malgré la fiabilité de l’algorithme de cryptage, il existe toujours une vulnérabilité contre les [tables arc-en-ciel] [5]. C’est la raison pour laquelle il est recommandé d’utiliser du sel.

Un sel est quelque chose qui est ajouté au mot de passe avant le hachage pour rendre la chaîne source unique. Étant donné deux mots de passe identiques, les hachages résultants seront également uniques, car leurs sels sont uniques.

Un sel aléatoire est l’un des éléments les plus importants de la sécurité de votre mot de passe. Cela signifie que même avec une table de recherche de hachages de mots de passe connus, un attaquant ne peut pas faire correspondre le hachage de mot de passe de votre utilisateur avec les hachages de mot de passe de la base de données car un sel aléatoire a été utilisé. Vous devez toujours utiliser des sels aléatoires et cryptographiquement sécurisés. [En savoir plus][6]

Avec l’algorithme [password_hash()][1] bcrypt, le sel de texte brut est stocké avec le hachage résultant, ce qui signifie que le hachage peut être transféré sur différents systèmes et plates-formes et toujours être comparé au mot de passe d’origine.

Même lorsque cela n’est pas recommandé, vous pouvez utiliser l’option salt pour définir votre propre sel aléatoire.

 $options = [
        'salt' => $salt, //see example below
 ];

Important. Si vous omettez cette option, un sel aléatoire sera généré par password_hash() pour chaque mot de passe haché. C’est le mode de fonctionnement prévu.

L’option salt est [obsolète][7] depuis PHP 7.0.0. Il est maintenant préférable d’utiliser simplement le sel qui est généré par défaut.

[1] : http://php.net/manual/en/function.password-hash.php [2] : https://en.wikipedia.org/wiki/Bcrypt [3] : https://github.com/ircmaxell/password_compat [4] : http://php.net/manual/en/function.crypt.php [5] : https://en.wikipedia.org/wiki/Rainbow_table [6] : http://www.springer.com/us/book/9781484221198 [7] : http://php.net/manual/ru/function.password-hash.php

Déterminer si un hachage de mot de passe existant peut être mis à niveau vers un algorithme plus puissant #

Si vous utilisez la méthode PASSWORD_DEFAULT pour laisser le système choisir le meilleur algorithme avec lequel hacher vos mots de passe, à mesure que la force par défaut augmente, vous souhaiterez peut-être ressasser les anciens mots de passe lorsque les utilisateurs se connectent

<?php
// first determine if a supplied password is valid
if (password_verify($plaintextPassword, $hashedPassword)) {

    // now determine if the existing hash was created with an algorithm that is
    // no longer the default
    if (password_needs_rehash($hashedPassword, PASSWORD_DEFAULT)) {

        // create a new hash with the new default
        $newHashedPassword = password_hash($plaintextPassword, PASSWORD_DEFAULT);

        // and then save it to your data store
        //$db->update(...);
    }
}
?>

Si les fonctions password_* ne sont pas disponibles sur votre système (et que vous ne pouvez pas utiliser le pack de compatibilité lié dans les remarques ci-dessous), vous pouvez déterminer l’algorithme et utilisé pour créer le hachage d’origine dans une méthode similaire à la suivante :

<?php
if (substr($hashedPassword, 0, 4) == '$2y$' && strlen($hashedPassword) == 60) {
    echo 'Algorithm is Bcrypt';
    // the "cost" determines how strong this version of Bcrypt is
    preg_match('/\$2y\$(\d+)\$/', $hashedPassword, $matches);
    $cost = $matches[1];
    echo 'Bcrypt cost is '.$cost;
}
?>

Vérification d’un mot de passe par rapport à un hachage #

password_verify() est la fonction intégrée fournie (à partir de PHP 5.5) pour vérifier la validité d’un mot de passe par rapport à un hachage connu.

<?php
if (password_verify($plaintextPassword, $hashedPassword)) {
    echo 'Valid Password';
}
else {
    echo 'Invalid Password.';
}
?>

Tous les algorithmes de hachage pris en charge stockent des informations identifiant quel hachage a été utilisé dans le hachage lui-même, il n’est donc pas nécessaire d’indiquer avec quel algorithme vous utilisez pour coder le mot de passe en clair.

Si les fonctions password_* ne sont pas disponibles sur votre système (et que vous ne pouvez pas utiliser le pack de compatibilité lié dans les remarques ci-dessous), vous pouvez implémenter la vérification du mot de passe avec la fonction crypt(). Veuillez noter que des précautions spécifiques doivent être prises pour éviter les attaques temporelles.

<?php
// not guaranteed to maintain the same cryptographic strength of the full `password_hash()`
// implementation
if (CRYPT_BLOWFISH == 1) {
    // `crypt()` discards all characters beyond the salt length, so we can pass in
    // the full hashed password
    $hashedCheck = crypt($plaintextPassword, $hashedPassword);

    // this a basic constant-time comparison based on the full implementation used
    // in `password_hash()`
    $status = 0;
    for ($i=0; $i<strlen($hashedCheck); $i++) {
        $status |= (ord($hashedCheck[$i]) ^ ord($hashedPassword[$i]));
    }

    if ($status === 0) {
        echo 'Valid Password';
    }
    else {
        echo 'Invalid Password';
    }
}
?>
