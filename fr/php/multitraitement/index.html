Multitraitement à l’aide des fonctions de fork intégrées #

Vous pouvez utiliser des fonctions intégrées pour exécuter des processus PHP en tant que forks. C’est le moyen le plus simple de réaliser un travail parallèle si vous n’avez pas besoin que vos threads se parlent.

Cela vous permet de mettre des tâches chronophages (comme le téléchargement d’un fichier sur un autre serveur ou l’envoi d’un e-mail) à un autre thread afin que votre script se charge plus rapidement et puisse utiliser plusieurs cœurs, mais sachez qu’il ne s’agit pas d’un véritable multithreading et que votre thread principal ne le fera pas. savoir ce que font les enfants.

Notez que sous Windows, cela fera apparaître une autre invite de commande pour chaque fork que vous démarrez.

maître.php

$cmd = "php worker.php 10";
if(strtoupper(substr(PHP_OS, 0, 3)) === 'WIN') // for windows use popen and pclose
{
    pclose(popen($cmd,"r"));
}
else //for unix systems use shell exec with "&" in the end
{
    exec('bash -c "exec nohup setsid '.$cmd.' > /dev/null 2>&1 &"');
}

travailleur.php

//send emails, upload files, analyze logs, etc
$sleeptime = $argv[1];
sleep($sleeptime);

Création d’un processus enfant à l’aide de fork #

PHP a intégré la fonction pcntl_fork pour créer un processus enfant. pcntl_fork est identique à fork sous Unix. Il ne prend aucun paramètre et renvoie un entier qui peut être utilisé pour différencier les processus parent et enfant. Considérez le code suivant pour l’explication

<?php
    // $pid is the PID of child
    $pid = pcntl_fork();
    if ($pid == -1) {
         die('Error while creating child process');
    } else if ($pid) {
         // Parent process
    } else {
         // Child process
    }
?>

Comme vous pouvez le voir, -1 est une erreur dans fork et l’enfant n’a pas été créé. Lors de la création de l’enfant, nous avons deux processus en cours d’exécution avec un “PID” séparé.

Une autre considération ici est un processus zombie ou processus défunt lorsque le processus parent se termine avant le processus enfant. Pour éviter un processus enfant zombie, ajoutez simplement pcntl_wait($status) à la fin du processus parent.

pnctl_wait suspend l’exécution du processus parent jusqu’à ce que le processus enfant soit terminé.

Il convient également de noter que le “processus zombie” ne peut pas être tué à l’aide du signal “SIGKILL”.

Communication interprocessus #

La communication interprocessus permet aux programmeurs de communiquer entre différents processus. Par exemple, considérons que nous devons écrire une application PHP capable d’exécuter des commandes bash et d’imprimer la sortie. Nous utiliserons proc_open , qui exécutera la commande et renverra une ressource avec laquelle nous pouvons communiquer. Le code suivant montre une implémentation de base qui exécute pwd dans bash à partir de php

<?php
    $descriptor = array(
        0 => array("pipe", "r"), // pipe for stdin of child
        1 => array("pipe", "w"), // pipe for stdout of child
    );
    $process = proc_open("bash", $descriptor, $pipes);
    if (is_resource($process)) {
        fwrite($pipes[0], "pwd" . "\n");
        fclose($pipes[0]);
        echo stream_get_contents($pipes[1]);
        fclose($pipes[1]);
        $return_value = proc_close($process);

    }
?>

proc_open exécute la commande bash avec $descriptor comme spécification de descripteur. Après cela, nous utilisons is_resource pour valider le processus. Une fois cela fait, nous pouvons commencer à interagir avec le processus enfant en utilisant $pipes qui est généré selon les spécifications du descripteur.

Après cela, nous pouvons simplement utiliser fwrite pour écrire sur stdin du processus enfant. Dans ce cas, “pwd” suivi d’un retour chariot. Enfin, stream_get_contents est utilisé pour lire la sortie standard du processus enfant.

N’oubliez pas de fermer le processus enfant en utilisant proc_close() qui terminera l’enfant et renverra le code d’état de sortie.
