Manière de traiter chaque élément généré par Scrapy.

Un pipeline d’éléments est une classe python qui remplace certaines méthodes spécifiques et doit être activée dans les “paramètres” du projet scrapy.

Création d’un pipeline dynamique dans Python Scrapy #

Activez les pipelines dans votre settings.py

ITEM_PIPELINES = {
    'project_folder.pipelines.MyPipeline': 100 
}

Ensuite, écrivez ce code dans items.py

# -*- coding: utf-8 -*-
from scrapy import Item, Field
from collections import OrderedDict

class DynamicItem(Item):
    def __setitem__(self, key, value):
        self._values[key] = value
        self.fields[key] = {}

Puis dans votre `project_folder/spiders/spider_file.py

from project_folder.items import DynamicItem
       def parse(self, response):
               # create an ordered dictionary
               data = OrderedDict()
               data['first'] = ...
               data['second'] = ...
               data['third'] = ...
               .
               .
               .
               # create dictionary as long as you need
               
               # now unpack dictionary
               yield DynamicItem( **data )

               # above line is same as this line
               yield DynamicItem( first = data['first'], second = data['second'], third = data['third'])

**Quels sont les avantages de ce code ?**

Pas besoin de créer définir chaque élément dans items.py un par un.

Créer votre propre pipeline #

Lors de la création d’un projet scrapy avec scrapy startproject myproject, vous trouverez un fichier pipelines.py déjà disponible pour créer vos propres pipelines. Il n’est pas obligatoire de créer vos pipelines dans ce fichier, mais ce serait une bonne pratique. Nous expliquerons comment créer un pipeline à l’aide du fichier pipelines.py :

pipelines.py

class MyPipeline(object):
    def process_item(self, item, spider):
        # process your `item` here
        return item

Maintenant, pour l’activer, vous devez spécifier qu’il va être utilisé dans vos paramètres. Accédez à votre fichier settings.py et recherchez (ou ajoutez) la variable ITEM_PIPELINES. Mettez-le à jour avec le chemin d’accès à votre classe de pipeline et sa priorité sur les autres pipelines :

settings.py

ITEM_PIPELINES = {
    'myproject.pipelines.MyPipeline': 300,
}

Désormais, chaque élément renvoyé par votre araignée passera par ce pipeline.
