LeftAndMain est plutôt une API de niveau inférieur et n’est pas souvent nécessaire en raison de l’existence de ModelAdmin. Cependant, si vous vouliez créer une interface utilisateur personnalisée qui ne nécessite pas nécessairement la fonctionnalité de ModelAdmin dans le panneau d’administration de votre module, alors LeftAndMain est l’endroit où vous voudriez commencer.

1. Mise en route #

Ce guide est destiné à vous aider à créer votre propre interface utilisateur en sous-classant la classe LeftAndMain.

À la fin de ce guide, vous aurez créé votre première interface “Hello World” dans le panneau d’administration.

Conditions #

Ce guide nécessite que vous disposiez au moins de la version 3.* du [framework][1] ET du [CMS][2], mais d’une version inférieure à la version 4.*.

Si vous souhaitez utiliser ce guide, vous devrez échanger toutes les références de classe avec le nom de classe de qualité complète (FQCN) tel que défini dans le guide de mise à niveau SS4.

Préparation #

tl;dr Ignorez les étapes suivantes et créez simplement la structure en dessous

  1. Créez un dossier avec le nom de votre choix dans le répertoire racine de votre projet SilverStripe. Pour cet exemple, nous utiliserons /helloworld/ et créerons un fichier vide dans ce dossier nommé _config.php. Un _config.php au minimum est requis dans chaque répertoire de module pour que SilverStripe détecte son existence.

  2. Dans votre nouveau dossier, créez un sous-dossier nommé exactement /code/ et dans ce dossier, à des fins d’organisation ; créer un autre dossier appelé /admin/

  3. Créez /helloworld/code/admin/HelloWorldLeftAndMain.php et placez-y le code suivant pour le moment.

     class HelloWorldLeftAndMain extends LeftAndMain {

 }

  4. Créez le fichier modèle qui sera utilisé avec cette classe appelée /helloworld/templates/Includes/HelloWorldLeftAndMain.ss

Structure #

/framework/
/cms/
/helloworld/
    + _config.php
    + /code/
        + /admin/
            + /HelloWorldLeftAndMain.php
    + /templates/
        + /Includes/
            + /HelloWorldLeftAndMain_Content.ss

[1] : https://github.com/silverstripe/silverstripe-framework [2] : https://github.com/silverstripe/silverstripe-cms

2. Configurer HelloWorldLeftAndMain.php #

Si vous ne l’avez pas déjà fait, commençons simplement ce fichier avec :

class HelloWorldLeftAndMain extends LeftAndMain {

}

Configurer #

La première chose à faire est de définir le $url_segment qui sera utilisé pour accéder à l’interface, et le titre ($menu_title) qui apparaîtra dans le menu de navigation latéral du panneau d’administration :

private static $url_segment = 'helloworld';
private static $menu_title  = 'Hello World';

Les variables de configuration suivantes sont facultatives et ne sont pas utilisées dans ce guide :

private static $menu_icon   = 'helloworld/path/to/my/icon.png';
private static $url_rule    = '/$Action/$ID/$OtherID';

Ajout de feuilles de style et de Javascript #

LeftAndMain vous permet de remplacer la méthode init dans son parent, nous pouvons l’utiliser pour exiger des fichiers spécifiques pour notre interface. Sans aucun doute, vous devriez toujours avoir besoin d’une feuille de style CSS qui stylisera les éléments de votre interface utilisateur.

En guise de conseil, il est recommandé de ne jamais se fier aux classes CSS fournies par le CMS car celles-ci sont susceptibles d’être modifiées sans préavis et détruiront par la suite l’aspect et la convivialité de votre interface utilisateur (par exemple, 3.* à 4.* a vu une refonte complète de l’interface, par conséquent, toutes les classes CSS sur lesquelles vous vous êtes appuyé dans 3.* doivent être restylées pour être converties en 4.*)

Ajoutons donc notre fichier helloworld/css/styles.css :

public function init() {
    parent::init();

    Requirements::css('helloworld/css/styles.css');
    //Requirements::javascript('helloworld/javascript/script.min.js');
}

Nous n’avons besoin d’aucune fonctionnalité Javascript pour cet exemple, mais dans ce qui précède, j’ai inclus comment on pourrait ajouter Javascript à un fichier en utilisant la classe [Requirements][1].

Après quoi, vous pouvez adopter ce à quoi vous avez été habitué lorsque vous traitez Page_Controller tel que $allowed_actions etc avec une différence notable, cependant

Vous NE POUVEZ PAS remplacer index().

Au lieu de cela, index() est supposé être HelloWorldLeftAndMain_Content.ss et à partir de là, il est nécessaire de gérer l’affichage des index via les fonctions de modèle (voir l’exemple ci-dessous)

#Code complet

class HelloWorldLeftAndMain extends LeftAndMain {
    private static $url_segment = 'helloworld';
    private static $menu_title  = 'Hello World';
    private static $allowed_actions = array(
        'some_action'
    );

    public function init() {
        parent::init();

        Requirements::css('helloworld/css/styles.css');
        //Requirements::javascript('helloworld/javascript/script.min.js');
    }

    public function Hello($who=null) {
        if (!$who) {
            $who = 'World';
        }

        return "Hello " . htmlentities($who);
    } 
}

[1] : https://api.silverstripe.org/3.5/class-Requirements.html

3. Le modèle (HelloWorldLeftAndMain_Content.ss) #

La structure attendue de ce modèle peut être un peu compliquée, mais tout se résume à ceci :

  1. Il y a 3 sections à noter pour ce guide : #

    • .north
    • .center
    • .south

  2. Il doit être entièrement enveloppé dans un élément qui a l’attribut data-pjax-fragment="Content". C’est ainsi que les appels AJAX générés à partir du menu latéral savent où se trouve le “Contenu” afin qu’il puisse l’afficher correctement :

<div class="cms-content center $BaseCSSClasses" data-layout-type="border" data-pjax-fragment="Content">

</div>

Je n’entrerai pas dans les détails sur la fonctionnalité du modèle, j’ai inclus des commentaires le cas échéant, mais vous ne devriez pas lire ce guide si vous ne comprenez pas la syntaxe du modèle pour SilverStripe

#Code complet La seule chose d’en bas; que vous devriez vous attendre à sortir déjà stylé est le <% include CMSBreadcrumbs %> tout le reste que vous devez gérer vous-même dans le fichier CSS qui a été inclus plus tôt

<div class="cms-content center $BaseCSSClasses" data-layout-type="border" data-pjax-fragment="Content">
    <%-- This will add the breadcrumb that you see on every other menu item --%>
    <div class="cms-content-header north">
        <div class="cms-content-header-info">
            <% include CMSBreadcrumbs %>
        </div>
    </div>
    
    <div class="center">
        <%-- Our function in HelloWorldLeftAndMain.php --%>
        $Hello('USER');
        <%-- ^ outputs "Hello USER" --%>
    </div>

    <div class='south'>
        Some footer-worthy content
    </div>
</div>

Maintenant, tout ce qu’il vous reste à faire est de /dev/build et ?flush=1, vous pouvez alors consulter notre petit module inutile dans le panneau d’administration !
