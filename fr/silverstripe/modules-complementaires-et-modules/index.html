Module d’extensions de champ de grille SilverStripe #

Le [SilverStripe Grid Field Extensions Module] [1] possède de très belles fonctionnalités pour améliorer le GridField de base…

  • GridFieldAddExistingSearchButton - un formulaire de recherche plus avancé pour ajouter des éléments
  • GridFieldAddNewInlineButton - s’appuie sur GridFieldEditableColumns pour permettre la création en ligne d’enregistrements.
  • GridFieldAddNewMultiClass - permet à l’utilisateur de sélectionner dans une liste de classes pour créer un nouvel enregistrement à partir de
  • GridFieldEditableColumns - permet l’édition en ligne des enregistrements
  • GridFieldOrderableRows - ​​réorganisation des lignes par glisser-déposer
  • GridFieldRequestHandler - une classe d’utilitaire de base qui peut être utilisée pour créer des vues détaillées de champ de grille personnalisées, y compris des onglets, des fils d’Ariane et d’autres fonctionnalités CMS
  • GridFieldTitleHeader - un en-tête simple qui affiche les titres des colonnes

Plus de documentation se trouve dans [le module ici][2].

[1] : https://github.com/silverstripe-australia/silverstripe-gridfieldextensions [2] : https://github.com/silverstripe-australia/silverstripe-gridfieldextensions/blob/master/docs/en/index.md

Meilleurs boutons pour GridField #

Le module [Better Buttons for GridField][1] ajoute de nouvelles actions et boutons de formulaire au formulaire de détail GridField.

  • Enregistrer et en ajouter un autre : créez un enregistrement et passez directement à l’ajout d’un autre, sans avoir à cliquer sur le bouton de retour, puis ajoutez à nouveau
  • Enregistrer et fermer : enregistrez l’enregistrement et revenez à la vue de liste
  • Supprimer convivial : Extrait du plateau d’actions constructives et éloigné de sorte qu’il est moins susceptible d’être cliqué accidentellement. Comprend une confirmation en ligne de l’action au lieu de la boîte d’alerte du navigateur
  • Annuler : Identique au bouton de retour, mais dans un emplacement plus pratique
  • Enregistrement précédent/suivant : accédez à l’enregistrement précédent ou suivant dans la liste sans revenir à l’affichage de la liste
  • et beaucoup plus…

Plus de documentation (et d’images) sur la [documentation du module][2]

[1] : http://%20https://github.com/unclecheese/silverstripe-gridfield-betterbuttons [2] : https://github.com/unclecheese/silverstripe-gridfield-betterbuttons/blob/master/README.md

UserForms #

Le module [UserForms][1] permet aux utilisateurs du CMS de créer des formulaires dynamiques via une interface glisser-déposer et sans s’impliquer dans un quelconque code PHP.

Caractéristiques principales

  • Construire un formulaire en utilisant tous les principaux champs de formulaire (texte, e-mail, liste déroulante, radio, case à cocher ..)
  • Possibilité d’étendre les userforms d’autres modules pour fournir des champs supplémentaires.
  • Possibilité d’envoyer par e-mail à plusieurs personnes la soumission du formulaire
  • Afficher les soumissions soumises et les exporter au format CSV
  • Définir des messages d’erreur personnalisés et des paramètres de validation
  • Afficher et masquer éventuellement les champs à l’aide de javascript en fonction de l’entrée des utilisateurs
  • Affiche un message de confirmation lorsque vous quittez un formulaire partiellement rempli

Plus de liens de documentation peuvent être trouvés [ici dans le référentiel github][2]

[1] : https://github.com/silverstripe/silverstripe-userforms [2] : https://github.com/silverstripe/silverstripe-userforms/tree/master/docs/en

Logique d’affichage #

Le [module Display Logic][1] vous permet d’ajouter des conditions d’affichage ou de masquage de certains champs de formulaire en fonction du comportement côté client. Ce module est incroyablement utile pour rendre les formulaires beaucoup plus professionnels en affichant uniquement les champs appropriés et sans ajouter beaucoup de JavaScript personnalisé.

Exemple d’utilisation…

$products->displayIf("HasProducts")->isChecked();

$sizes->hideUnless("ProductType")->isEqualTo("t-shirt")
      ->andIf("Price")->isGreaterThan(10);

$payment->hideIf("Price")->isEqualTo(0);

$shipping->displayIf("ProductType")->isEqualTo("furniture")
           ->andIf()
              ->group()
                ->orIf("RushShipping")->isChecked()
                ->orIf("ShippingAddress")->isNotEmpty()
              ->end();

Il existe de nombreux autres exemples sur le [module readme.md][2]

[1] : https://github.com/unclecheese/silverstripe-display-logic [2] : https://github.com/unclecheese/silverstripe-display-logic/blob/master/README.md

Menu CMS groupé #

Le [Module de menu CMS groupé] [1] vous permet de regrouper les éléments de menu CMS dans des listes imbriquées qui s’agrandissent lorsque vous les survolez. Ceci est utile lorsqu’il y a tellement d’éléments de menu CMS que l’espace à l’écran devient un problème.

[1] : https://github.com/silverstripe-australia/silverstripe-grouped-cms-menu

Tableau de bord #

Le [module Tableau de bord][1] fournit une page d’accueil pour le CMS dans SilverStripe 3 avec des widgets configurables qui affichent des informations pertinentes. Les panneaux peuvent être créés et étendus facilement. L’objectif du module Dashboard est de fournir aux utilisateurs un tableau de bord pour les actions CMS courantes telles que la création de types de pages spécifiques ou la navigation dans de nouveaux contenus.

Des images et des vidéos sur ce module peuvent être trouvées dans [ce blog] [2].

Certains panneaux sont inclus par défaut…

  • Pages récemment modifiées
  • Fichiers récemment téléchargés
  • Flux RSS
  • Liens rapides
  • Éditeur de rubrique
  • Google Analytics
  • Temps

Lorsque vous avez installé ce module, il crée un tableau de bord par membre, donc si vous avez un grand nombre de membres qui n’utiliseront jamais l’administrateur et que les performances deviennent un problème, je vous recommande de créer les membres avec ces paramètres supplémentaires avant de l’écrire…

Member::create(array(
   'HasConfiguredDashboard' => 1
));

Il y a beaucoup plus de documentation dans les [modules readme.md][3]

[1] : https://github.com/unclecheese/silverstripe-dashboard [2] : https://www.silverstripe.org/blog/the-dashboard-module-make-a-splash-in-silverstripe-3/ [3] : https://github.com/unclecheese/silverstripe-dashboard/blob/master/README.md
