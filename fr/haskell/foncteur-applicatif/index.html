Foncteur applicatif

  • Définition
  • Définition alternative

Sur cette page

  • Définition
  • Définition alternative

Applicative est la classe des types f :: * -> * qui permet l’application d’une fonction élevée sur une structure où la fonction est également intégrée dans cette structure.

Définition #

class Functor f => Applicative f where
    pure  :: a -> f a
    (<*>) :: f (a -> b) -> f a -> f b

Notez la contrainte Functor sur f. La fonction pure renvoie son argument intégré dans la structure Applicative. La fonction infixe <*> (prononcé “appliquer”) est très similaire à fmap sauf avec la fonction intégrée dans la structure Applicative.

Une instance correcte de Applicative doit satisfaire les lois applicatives, bien que celles-ci ne soient pas appliquées par le compilateur :

pure id <*> a = a                              -- identity
pure (.) <*> a <*> b <*> c = a <*> (b <*> c)   -- composition
pure f <*> pure a = pure (f a)                 -- homomorphism
a <*> pure b = pure ($ b) <*> a                -- interchange

Définition alternative #

Étant donné que chaque foncteur applicatif est un [foncteur] (https://www.wikiod.com/fr/haskell/foncteur), fmap peut toujours être utilisé dessus ; ainsi l’essence d’Applicatif est l’appariement des contenus portés, ainsi que la possibilité de le créer :

class Functor f => PairingFunctor f where
  funit :: f ()                  -- create a context, carrying nothing of import
  fpair :: (f a,f b) -> f (a,b)  -- collapse a pair of contexts into a pair-carrying context

Cette classe est isomorphe à Applicative.

pure a = const a <$> funit = a <$ funit  

fa <*> fb = (\(a,b) -> a b) <$> fpair (fa, fb) = uncurry ($) <$> fpair (fa, fb)

Inversement,

funit = pure ()

fpair (fa, fb) = (,) <$> fa <*> fb

## Instances courantes d’applicatif Peut-être #

Peut-être est un foncteur applicatif contenant une valeur éventuellement absente.

instance Applicative Maybe where
    pure = Just
    
    Just f <*> Just x = Just $ f x
    _ <*> _ = Nothing

pure soulève la valeur donnée dans Peut-être en lui appliquant Juste. La fonction (<*>) applique une fonction enveloppée dans un Peut-être à une valeur dans un Peut-être. Si la fonction et la valeur sont présentes (construites avec Just), la fonction est appliquée à la valeur et le résultat encapsulé est renvoyé. Si l’un ou l’autre est manquant, le calcul ne peut pas continuer et “Nothing” est renvoyé à la place.

Listes #

Une façon pour les listes de correspondre à la signature de type <*> :: [a -> b] -> [a] -> [b] consiste à prendre le produit cartésien des deux listes, en associant chaque élément de la première liste avec chaque élément du second :

fs <*> xs = [f x | f <- fs, x <- xs]
         -- = do { f <- fs; x <- xs; return (f x) }

pure x = [x]

Ceci est généralement interprété comme émulant le non-déterminisme, avec une liste de valeurs représentant une valeur non déterministe dont les valeurs possibles s’étendent sur cette liste; ainsi, une combinaison de deux valeurs non déterministes s’étend sur toutes les combinaisons possibles des valeurs dans les deux listes :

ghci> [(+1),(+2)] <*> [3,30,300]
[4,31,301,5,32,302]

Flux infinis et listes de fermeture éclair #

Il y a une classe de Applicative qui “compresse” leurs deux entrées ensemble. Un exemple simple est celui des flux infinis :

data Stream a = Stream { headS :: a, tailS :: Stream a }

L’instance Applicative de Stream applique un flux de fonctions à un flux d’arguments point par point, en associant les valeurs des deux flux par position. pure renvoie un flux constant – une liste infinie d’une seule valeur fixe :

instance Applicative Stream where
    pure x = let s = Stream x s in s
    Stream f fs <*> Stream x xs = Stream (f x) (fs <*> xs)

Les listes admettent également une instance Applicative “zippy”, pour laquelle il existe le nouveau type ZipList :

newtype ZipList a = ZipList { getZipList :: [a] }

instance Applicative ZipList where
    ZipList xs <*> ZipList ys = ZipList $ zipWith ($) xs ys

Étant donné que zip découpe son résultat en fonction de l’entrée la plus courte, la seule implémentation de pure qui satisfait les lois Applicative est celle qui renvoie une liste infinie :

    pure a = ZipList (repeat a)   -- ZipList (fix (a:)) = ZipList [a,a,a,a,...

Par exemple:

ghci> getZipList $ ZipList [(+1),(+2)] <*> ZipList [3,30,300]
[4,32]

Les deux possibilités nous rappellent le produit extérieur et le produit intérieur, similaires à la multiplication d’une matrice à 1 colonne (n x 1) par une matrice à 1 ligne (1 x m) dans le premier cas, obtenant la matrice n x m en conséquence (mais aplati); ou en multipliant une matrice 1 ligne et une matrice 1 colonne (mais sans la sommation) dans le second cas.

Les fonctions #

Lorsqu’elles sont spécialisées dans les fonctions (->) r, les signatures de type de pure et <*> correspondent respectivement à celles des combinateurs K et S :

pure :: a -> (r -> a)
<*> :: (r -> (a -> b)) -> (r -> a) -> (r -> b)

pure doit être const et <*> prend une paire de fonctions et les applique chacune à un argument fixe, en appliquant les deux résultats :

instance Applicative ((->) r) where
    pure = const
    f <*> g = \x -> f x (g x)

Les fonctions sont l’applicatif « zippy » prototypique. Par exemple, puisque les flux infinis sont isomorphes à (->) Nat, …

-- | Index into a stream
to :: Stream a -> (Nat -> a)
to (Stream x xs) Zero = x
to (Stream x xs) (Suc n) = to xs n

-- | List all the return values of the function in order
from :: (Nat -> a) -> Stream a
from f = from' Zero
    where from' n = Stream (f n) (from' (Suc n))

… la représentation des flux d’une manière d’ordre supérieur produit automatiquement l’instance zippy “Applicative”.
