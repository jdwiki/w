Un compteur monadique #

Un exemple sur la façon de composer le lecteur, l’écrivain et l’état monade à l’aide de transformateurs de monade. Le code source peut être trouvé dans ce dépôt

Nous voulons implémenter un compteur, qui incrémente sa valeur d’un certain constant.

Nous commençons par définir quelques types et fonctions :

newtype Counter = MkCounter {cValue :: Int}
  deriving (Show)

-- | 'inc c n' increments the counter by 'n' units.
inc :: Counter -> Int -> Counter
inc (MkCounter c) n = MkCounter (c + n)

Supposons que nous voulions effectuer le calcul suivant à l’aide du compteur :

  • mettre le compteur à 0
  • mettre la constante d’incrémentation à 3
  • incrémenter le compteur 3 fois
  • mettre la constante d’incrémentation à 5
  • incrémenter le compteur 2 fois

La monade d’état fournit des abstractions pour transmettre l’état. Nous pouvons faire utilisation de la monade d’état, et définir notre fonction d’incrémentation comme un état transformateur.

-- | CounterS is a monad.
type CounterS = State Counter

-- | Increment the counter by 'n' units.
incS :: Int-> CounterS ()
incS n = modify (\c -> inc c n)

Cela nous permet déjà d’exprimer un calcul d’une manière plus claire et façon succincte :

-- | The computation we want to run, with the state monad.
mComputationS :: CounterS ()
mComputationS = do
  incS 3
  incS 3
  incS 3
  incS 5
  incS 5

Mais nous devons encore passer la constante d’incrémentation à chaque invocation. Nous serions aime éviter cela.

Ajouter un environnement #

Le reader monad fournit un moyen pratique de faire passer un environnement. Cette monade est utilisée en programmation fonctionnelle pour effectuer ce qui dans le Le monde OO est connu sous le nom d’injection de dépendance.

Dans sa version la plus simple, la monade de lecteur nécessite deux types :

  • le type de la valeur lue (c’est-à-dire notre environnement, r ci-dessous),

  • la valeur renvoyée par la monade du lecteur (a ci-dessous).

    Reader r a

Cependant, nous devons également utiliser la monade d’état. Ainsi, nous devons utiliser le transformateur ReaderT :

newtype ReaderT r m a :: * -> (* -> *) -> * -> *

En utilisant ReaderT, nous pouvons définir notre compteur avec l’environnement et l’état comme suit :

type CounterRS = ReaderT Int CounterS

Nous définissons une fonction incR qui prend la constante d’incrément de la environnement (en utilisant ask), et de définir notre fonction d’incrémentation en termes de notre monade CounterS nous utilisons la fonction lift (qui appartient à la transformateur monade classe).

-- | Increment the counter by the amount of units specified by the environment.
incR :: CounterRS ()
incR = ask >>= lift . incS

En utilisant la monade du lecteur, nous pouvons définir notre calcul comme suit :

-- | The computation we want to run, using reader and state monads.
mComputationRS :: CounterRS ()
mComputationRS = do
  local (const 3) $ do
    incR
    incR
    incR
    local (const 5) $ do
      incR
      incR

Les exigences ont changé : nous avons besoin de la journalisation ! #

Supposons maintenant que nous voulions ajouter la journalisation à notre calcul, afin que nous puissions voir l’évolution de notre compteur dans le temps.

Nous avons également une monade pour effectuer cette tâche, la writer monad. Comme avec le lecteur monade, puisque nous les composons, nous devons utiliser le lecteur transformateur de monade :

newtype WriterT w m a :: * -> (* -> *) -> * -> *

Ici w représente le type de sortie à accumuler (qui doit être un monoïde, qui nous permettent d’accumuler cette valeur), m est la monade interne, et a le type de calcul.

Nous pouvons ensuite définir notre compteur avec la journalisation, l’environnement et l’état comme suit :

type CounterWRS = WriterT [Int] CounterRS

Et en utilisant “lift”, nous pouvons définir la version de la fonction d’incrémentation qui enregistre la valeur du compteur après chaque incrément :

incW :: CounterWRS ()
incW = lift incR >> get >>= tell . (:[]) . cValue

Maintenant, le calcul qui contient la journalisation peut être écrit comme suit :

mComputationWRS :: CounterWRS ()
mComputationWRS = do
  local (const 3) $ do
    incW
    incW
    incW
    local (const 5) $ do
      incW
      incW

Tout faire en une seule fois #

Cet exemple visait à montrer les transformateurs de monade au travail. Cependant, nous pouvons obtenir le même effet en composant tous les aspects (environnement, état et journalisation) en une seule opération d’incrémentation.

Pour ce faire, nous utilisons des contraintes de type :

inc' :: (MonadReader Int m, MonadState Counter m, MonadWriter [Int] m) => m ()
inc' = ask >>= modify . (flip inc) >> get >>= tell . (:[]) . cValue

Ici, nous arrivons à une solution qui fonctionnera pour toute monade qui satisfait le contraintes ci-dessus. La fonction de calcul est définie ainsi de type :

mComputation' :: (MonadReader Int m, MonadState Counter m, MonadWriter [Int] m) => m ()

puisque dans son corps on se sert de inc'.

Nous pourrions exécuter ce calcul, dans le REPL ghci par exemple, comme suit :

runState ( runReaderT ( runWriterT mComputation' ) 15 )  (MkCounter 0)
