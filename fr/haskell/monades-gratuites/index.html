Les monades libres divisent les calculs monadiques en structures de données et en interpréteurs #

Par exemple, un calcul impliquant des commandes pour lire et écrire à partir de l’invite :

Nous décrivons d’abord les “commandes” de notre calcul en tant que type de données Functor

{-# LANGUAGE DeriveFunctor #-}

data TeletypeF next
    = PrintLine String next
    | ReadLine (String -> next)
    deriving Functor

Ensuite, nous utilisons Free pour créer la “Monade libre sur TeletypeF” et construisons quelques opérations de base.

import Control.Monad.Free (Free, liftF, iterM)

type Teletype = Free TeletypeF

printLine :: String -> Teletype ()
printLine str = liftF (PrintLine str ())

readLine :: Teletype String
readLine = liftF (ReadLine id)

Puisque Free f est une Monad chaque fois que f est un Functor, nous pouvons utiliser les combinateurs standard Monad (y compris la notation do) pour construire des calculs Teletype.

import Control.Monad -- we can use the standard combinators

echo :: Teletype ()
echo = readLine >>= printLine

mockingbird :: Teletype a
mockingbird = forever echo

Enfin, nous écrivons un “interpréteur” transformant les valeurs Teletype a en quelque chose avec lequel nous savons travailler comme IO a

interpretTeletype :: Teletype a -> IO a
interpretTeletype = foldFree run where
  run :: TeletypeF a -> IO a
  run (PrintLine str x) = putStrLn *> return x
  run (ReadLine f) = fmap f getLine

Que nous pouvons utiliser pour “exécuter” le calcul Teletype a dans IO

> interpretTeletype mockingbird
hello
hello
goodbye
goodbye
this will go on forever
this will go on forever

La monade plus libre #

Il existe une formulation alternative de la monade libre appelée la monade Freer (ou Prompt, ou Operational). La monade Freer ne nécessite pas d’instance Functor pour son jeu d’instructions sous-jacent, et elle a une structure de type liste plus reconnaissable que la monade libre standard.

La monade Freer représente les programmes comme une séquence d’instructions atomiques appartenant au jeu d’instructions i :: * -> *. Chaque instruction utilise son paramètre pour déclarer son type de retour. Par exemple, le jeu d’instructions de base pour la monade “State” est le suivant :

data StateI s a where
    Get :: StateI s s  -- the Get instruction returns a value of type 's'
    Put :: s -> StateI s ()  -- the Put instruction contains an 's' as an argument and returns ()

Le séquencement de ces instructions s’effectue avec le constructeur :>>=. :>>= prend une seule instruction renvoyant un a et l’ajoute au reste du programme, redirigeant sa valeur de retour dans la continuation. En d’autres termes, étant donné une instruction renvoyant un a et une fonction pour transformer un a en un programme renvoyant un b, :>>= produira un programme renvoyant un b.

data Freer i a where
    Return :: a -> Freer i a
    (:>>=) :: i a -> (a -> Freer i b) -> Freer i b

Notez que a est quantifié existentiellement dans le constructeur :>>=. La seule façon pour un interpréteur d’apprendre ce qu’est « a » est de faire correspondre des motifs sur le GADT « i ».

Aside : Le lemme co-Yoneda nous dit que Freer est isomorphe à Free. Rappelons la définition du foncteur CoYoneda :

données CoYoneda i b où CoYoneda :: je une -> (a -> b) -> CoYoneda je b

Freer i est équivalent à Free (CoYoneda i). Si vous prenez [les constructeurs de Free] (https://www.wikiod.com/fr/haskell/monades-gratuites#Les monades gratuites sont comme des points fixes) et définissez f ~ CoYoneda i, vous obtenez :

Pur :: a -> Libre (CoYoneda i) a Gratuit :: CoYoneda i (Libre (CoYoneda i) b) -> Gratuit (CoYonda i) b ~ je a -> (a -> Libre (CoYoneda i) b) -> Libre (CoYoneda i) b

à partir duquel nous pouvons récupérer les constructeurs de Freer i en définissant simplement Plus libre je ~ Libre (CoYoneda je).

Parce que CoYoneda i est un Functor pour tout i, Freer est une Monad pour tout i, même si i n’est pas un Functor.

instance Monad (Freer i) where
    return = Return
    Return x >>= f = f x
    (i :>>= g) >>= f = i :>>= fmap (>>= f) g  -- using `(->) r`'s instance of Functor, so fmap = (.)

Des interpréteurs peuvent être construits pour Freer en mappant des instructions sur une monade de gestionnaire.

foldFreer :: Monad m => (forall x. i x -> m x) -> Freer i a -> m a
foldFreer eta (Return x) = return x
foldFreer eta (i :>>= f) = eta i >>= (foldFreer eta . f)

Par exemple, nous pouvons interpréter la monade Freer (StateI s) en utilisant la monade régulière State s comme gestionnaire :

runFreerState :: Freer (StateI s) a -> s -> (a, s)
runFreerState = State.runState . foldFreer toState
    where toState :: StateI s a -> State s a
          toState Get = State.get
          toState (Put x) = State.put x

Les monades gratuites sont comme des points fixes #

Comparez la définition de Free à celle de [Fix](https://www.wikiod.com/fr/haskell/schemas-de-recursivite#Points fixes) :

data Free f a = Return a
              | Free (f (Free f a))

newtype Fix f = Fix { unFix :: f (Fix f) }

En particulier, comparez le type du constructeur “Free” avec le type du constructeur “Fix”. Free superpose un foncteur comme Fix, sauf que Free a un cas Return a supplémentaire.

Comment fonctionnent foldFree et iterM ? #

Il existe quelques fonctions pour aider à démonter les calculs Free en les interprétant dans une autre monade m : iterM :: (Functor f, Monad m) => (f (m a) -> m a) -> (Free f a - > m a) et foldFree :: Monad m => (forall x. f x -> m x) -> (Free f a -> m a). Que font-ils?

Voyons d’abord ce qu’il faudrait pour démonter et interpréter manuellement une fonction Teletype a dans IO. On peut voir Free f a comme étant défini

data Free f a 
    = Pure a 
    | Free (f (Free f a))

Le cas “Pure” est simple :

interpretTeletype :: Teletype a -> IO a
interpretTeletype (Pure x) = return x
interpretTeletype (Free teletypeF) = _

Maintenant, comment interpréter un calcul Teletype construit avec le constructeur Free ? Nous aimerions arriver à une valeur de type IO a en examinant teletypeF :: TeletypeF (Teletype a). Pour commencer, nous allons écrire une fonction runIO :: TeletypeF a -> IO a qui associe une seule couche de la monade libre à une action IO :

runIO :: TeletypeF a -> IO a
runIO (PrintLine msg x) = putStrLn msg *> return x
runIO (ReadLine k) = fmap k getLine

Maintenant, nous pouvons utiliser runIO pour remplir le reste de interpretTeletype. Rappelons que teletypeF :: TeletypeF (Teletype a) est une couche du foncteur TeletypeF qui contient le reste du calcul Free. Nous allons utiliser runIO pour interpréter la couche la plus externe (nous avons donc runIO teletypeF :: IO (Teletype a)) puis utiliser le combinateur >>= de la monade IO pour interpréter le Teletype a renvoyé .

interpretTeletype :: Teletype a -> IO a
interpretTeletype (Pure x) = return x
interpretTeletype (Free teletypeF) = runIO teletypeF >>= interpretTeletype

La définition de foldFree est juste celle de interpretTeletype, sauf que la fonction runIO a été factorisée. Par conséquent, foldFree fonctionne indépendamment de tout foncteur de base particulier et de toute monade cible.

foldFree :: Monad m => (forall x. f x -> m x) -> Free f a -> m a
foldFree eta (Pure x) = return x
foldFree eta (Free fa) = eta fa >>= foldFree eta

foldFree a un type de rang 2 : eta est une transformation naturelle. Nous aurions pu donner à foldFree un type de Monad m => (f (Free f a) -> m (Free f a)) -> Free f a -> m a, mais cela donne à eta la liberté d’inspecter le Calcul libre à l'intérieur de la couchef. Donner à foldFreece type plus restrictif garantit queeta` ne peut traiter qu’une seule couche à la fois.

iterM donne à la fonction de pliage la possibilité d’examiner le sous-calcul. Le résultat (monadique) de l’itération précédente est disponible pour la suivante, à l’intérieur du paramètre f. iterM est analogue à un [paramorphism](https://www.wikiod.com/fr/haskell/schemas-de-recursivite#Récursivité primitive) alors que foldFree est comme un [catamorphism]( https://www.wikiod.com/fr/haskell/schemas-de-recursivite#Plier une structure une couche à la fois).

iterM :: (Monad m, Functor f) => (f (m a) -> m a) -> Free f a -> m a
iterM phi (Pure x) = return x
iterM phi (Free fa) = phi (fmap (iterM phi) fa)
