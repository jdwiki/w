Raycasters (composant)

  • Paramètres

  • Liste blanche des entités à tester pour l’intersection

Le composant raycaster effectue des tests d’intersection généraux avec un raycaster. Raycasting est la méthode d’étendre une ligne d’une origine vers une direction, et de vérifier si cette ligne croise d’autres entités. Le composant raycaster est un wrapper au-dessus du raycaster three.js. Il vérifie les intersections à un certain intervalle par rapport à une liste d’objets et émet des événements sur l’entité lorsqu’il détecte des intersections ou la suppression d’intersections (c’est-à-dire lorsque le raycaster n’est plus

Paramètres #

Paramètre Détails
loin Distance maximale sous laquelle les entités résultantes sont renvoyées. Ne peut pas être plus bas que près.
intervalle Nombre de millisecondes à attendre entre chaque test d’intersection. Un nombre inférieur est préférable pour des mises à jour plus rapides. Un nombre plus élevé est meilleur pour les performances.
près de Distance minimale sur laquelle les entités résultantes sont renvoyées. Ne peut pas être inférieur à 0.
objets Sélecteur de requête pour sélectionner les objets à tester pour l’intersection. Si non spécifié, toutes les entités seront testées.
récursif Vérifie tous les enfants des objets s’ils sont définis. Else ne vérifie que les intersections avec les objets racine.

Événements #

Nom Détails
raycaster-intersecté Emis sur l’entité intersectée. L’entité croise un raycaster. Le détail de l’événement contiendra el, l’entité raycasting, et intersection, un objet contenant des données détaillées sur l’intersection.
raycaster-intersected-cleared Emis sur l’entité intersectée. L’entité ne croise plus un raycaster. Le détail de l’événement contiendra el, l’entité raycasting.
raycaster-intersection Émis sur l’entité raycasting. Raycaster croise une ou plusieurs entités. Le détail de l’événement contiendra els, un tableau avec les entités intersectées, et intersections, un tableau d’objets contenant des données détaillées sur les intersections.
raycaster-intersection-cleared Émis sur l’entité raycasting. Raycaster ne croise plus une entité. Le détail de l’événement contiendra el, l’ancienne entité intersectée.

Membre #

Membre La description
Els intersectés Entités croisant actuellement le raycaster.
objets Objets three.js pour tester les intersections. Sera scene.children si la propriété object n’est pas spécifiée.
raycaster objet raycaster three.js.

Méthode #

Méthode La description
rafraîchirObjets Actualise la liste des objets en fonction de la propriété objects pour tester l’intersection.

Réglage de l’origine et de la direction du Raycaster #

Le raycaster a une origine, où son rayon commence, et une direction, où le rayon va.

L’origine du raycaster est à la position de l’entité raycaster. Nous pouvons changer l’origine du raycaster en définissant le composant de position de l’entité raycaster (ou des entités parentes de l’entité raycaster).

La direction du raycaster est “devant” l’entité raycaster (c’est-à-dire 0 0 -1, sur l’axe Z négatif). Nous pouvons changer la direction du raycaster en définissant le composant de rotation de l’entité raycaster (ou des entités parentes de l’entité raycaster).

Par exemple, voici l’application d’un raycaster sur la longueur d’une balle tournée :

<!-- Bullet, rotated to be parallel with the ground. -->
<a-entity id="bullet" geometry="primitive: cylinder; height: 0.1" rotation="-90 0 0">
  <!-- Raycaster, targets enemies, made to be as long as the bullet, positioned to the start of the bullet, rotated to align with the bullet. -->
  <a-entity raycaster="objects: .enemies; far: 0.1" position="0 -0.5 0" rotation="90 0 0"></a-entity>
</a-entity>

Liste blanche des entités à tester pour l’intersection #

Nous ne voulons généralement pas tout tester dans la scène pour les intersections (par exemple, pour les collisions ou pour les clics). Les intersections sélectives sont bonnes pour les performances afin de limiter le nombre d’entités à tester pour l’intersection puisque le test d’intersection est une opération qui s’exécutera plus de 60 fois par seconde.

Pour sélectionner ou choisir les entités que nous voulons tester pour l’intersection, nous pouvons utiliser la propriété objects. Si cette propriété n’est pas définie, le raycaster testera chaque objet de la scène pour l’intersection. objects prend une valeur de sélecteur de requête :

<a-entity raycaster="objects: .clickable" cursor></a-entity>
<a-entity class="clickable" geometry="primitive: box" position="1 0 0"></a-entity>
<a-entity class="not-clickable" geometry="primitive: sphere" position="-1 0 0"></a-entity>
