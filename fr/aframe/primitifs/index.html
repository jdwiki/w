Primitifs

    • Sous la capuche
  • Enregistrer une primitive

Sur cette page

    • Sous la capuche
  • Enregistrer une primitive

Les primitives ne sont que des <a-entity> sous le capot. Cela signifie que les primitives ont la même API que les entités telles que le positionnement, la rotation, la mise à l’échelle et l’attachement de composants. A-Frame fournit une poignée d’éléments tels que <a-box> ou <a-sky> appelés primitives qui enveloppent le modèle entité-composant pour le rendre attrayant pour les débutants. . Les développeurs peuvent également créer leurs propres primitives.

Sous la capuche #

Les primitives agissent comme une couche de commodité (c’est-à-dire du sucre syntaxique) principalement pour nouveaux arrivants. Gardez à l’esprit pour l’instant que les primitives sont des <a-entity> sous la capot qui :

  • Avoir un nom sémantique (par exemple, <a-box>)
  • Avoir un ensemble prédéfini de composants avec des valeurs par défaut
  • Mapper ou proxy les attributs HTML aux données [composant][composant]

[assemblage] : http://vasir.net/blog/game-development/how-to-build-entity-component-system-in-javascript [préfabriqué] : http://docs.unity3d.com/Manual/Prefabs.html

Les primitives sont similaires aux [prefabs in Unity][prefab]. Quelques ouvrages sur la modèle entité-composant-système se réfèrent à eux comme [assemblages] [assemblage]. Ils résument l’API principale du composant d’entité pour :

  • Pré-composer des composants utiles avec les valeurs par défaut prescrites
  • Agit comme un raccourci pour les types d’entités complexes mais courants (par exemple, <a-sky>)
  • Fournissez une interface familière aux débutants puisque A-Frame prend HTML dans une nouvelle direction

Sous le capot, cette primitive <a-box> :

<a-box color="red" width="3"></a-box>

représente cette forme d’entité-composant :

<a-entity geometry="primitive: box; width: 3" material="color: red"></a-entity>

<a-box> définit par défaut la propriété geometry.primitive sur box. Et le la primitive mappe l’attribut HTML width à l’attribut geometry.width sous-jacent propriété ainsi que l’attribut HTML color au sous-jacent Propriété material.color.

Enregistrer une primitive #

Nous pouvons enregistrer nos propres primitives (c’est-à-dire enregistrer un élément) en utilisant AFRAME.registerPrimitive(nom, définition). definition est un JavaScript objet définissant ces propriétés :

Propriété Descriptif
Composants par défaut Objet spécifiant les composants par défaut de la primitive. Les clés sont les noms des composants et les valeurs sont les données par défaut des composants.
mappages Objet spécifiant le mappage entre le nom d’attribut HTML et les noms de propriété de composant. Chaque fois que le nom de l’attribut HTML est mis à jour, la primitive met à jour la propriété de composant correspondante. La propriété du composant est définie à l’aide d’une syntaxe à points ${componentName}.${propertyName}.

Par exemple, ci-dessous se trouve l’enregistrement de A-Frame pour la primitive <a-box> :

var extendDeep = AFRAME.utils.extendDeep;

// The mesh mixin provides common material properties for creating mesh-based primitives.
// This makes the material component a default component and maps all the base material properties.
var meshMixin = AFRAME.primitives.getMeshMixin();

AFRAME.registerPrimitive('a-box', extendDeep({}, meshMixin, {
  // Preset default components. These components and component properties will be attached to the entity out-of-the-box.
  defaultComponents: {
    geometry: {primitive: 'box'}
  },

  // Defined mappings from HTML attributes to component properties (using dots as delimiters).
  // If we set `depth="5"` in HTML, then the primitive will automatically set `geometry="depth: 5"`.
  mappings: {
    depth: 'geometry.depth',
    height: 'geometry.height',
    width: 'geometry.width'
  }
}));

Que nous pouvons utiliser alors

<a-box depth="1.5" height="1.5" width="1.5"></a-box>

représente cette forme d’entité-composant :

<a-entity geometry="primitive: box; depth: 1.5; height: 1.5; width:1.5;"></a-entity>
