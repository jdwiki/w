A-Frame représente une entité via l’élément <a-entity>. Comme défini dans le modèle entité-composant-système, les entités sont des objets d’espace réservé auxquels nous connectons des composants pour leur fournir une apparence, un comportement et des fonctionnalités.

Syntaxe #

  • <a-entity> // Considérez l'entité ci-dessous. En soi, il n'a ni apparence, ni comportement, ni fonctionnalité. Cela ne fait rien :
  • <a-entity geometry="primitive: box" material="color: red"> // Nous pouvons lui attacher des composants pour lui faire rendre quelque chose ou faire quelque chose. Pour lui donner forme et apparence, nous pouvons attacher la géométrie et les composants matériels :
  • <a-entity geometry="primitive: box" material="color: red" light="type: point; intensité : 2.0"> // Ou pour lui faire émettre de la lumière, nous pouvons en outre attacher la composante lumineuse :

Paramètres #

Paramètre Détails
composants <a-entity>.components est un objet de composants attaché à l’entité. Cela nous donne accès aux composants de l’entité, y compris les données, l’état et les méthodes de chaque composant.
est en train de jouer Indique si l’entité est active et joue. Si nous mettons l’entité en pause, alors isPlaying devient faux.
objet3D <a-entity>.object3D est une référence à la représentation Object3D three.js de l’entité. Plus précisément, object3D sera un objet THREE.Group pouvant contenir différents types de THREE.Object3D tels que des caméras, des maillages, des lumières ou des sons :
objet3DMap L’object3DMap d’une entité est un objet qui donne accès aux différents types de THREE.Object3D (par exemple, caméra, maillages, lumières, sons) que les composants ont définis.
scèneEl Une entité a une référence à son élément de scène.

MÉTHODES #

addState (stateName)

addState poussera un état sur l’entité. Cela émettra l’événement stateadded, et nous pouvons vérifier l’existence de l’état en utilisant .is :

entity.addEventListener('stateadded', function (evt) {
  if (evt.detail.state === 'selected') {
    console.log('Entity now selected!');
  }
});
entity.addState('selected');
entity.is('selected');  // >> true

émettre (nom, détail, bulles)

émet émet un événement DOM personnalisé sur l’entité. Par exemple, on peut émettre un événement pour déclencher une animation :

<a-entity>
  <a-animation attribute="rotation" begin="rotate" to="0 360 0"></a-animation>
</a-entity>

entity.emit('rotate');

Nous pouvons également transmettre des détails d’événement ou des données comme deuxième argument :

entity.emit('collide', { target: collidingEntity });

L’événement bouillonnera par défaut. nous pouvons lui dire de ne pas bouillonner en passant false pour bulle :

entity.emit('sink', null, false);

flushToDOM (récursif)

flushToDOM sérialisera manuellement les données des composants d’une entité et mettra à jour le DOM.

getAttribute (composantName)

getAttribute récupère les données des composants analysés (y compris les mixins et les valeurs par défaut).

// <a-entity geometry="primitive: box; width: 3">
entity.getAttribute('geometry');
// >> {primitive: "box", depth: 2, height: 2, translate: "0 0 0", width: 3, ...}
entity.getAttribute('geometry').primitive;
// >> "box"
entity.getAttribute('geometry').height;
// >> 2
entity.getAttribute('position');
// >> {x: 0, y: 0, z: 0}

Si componentName n’est pas le nom d’un composant enregistré, getAttribute se comportera normalement :

// <a-entity data-position="0 1 1">
entity.getAttribute('data-position');
// >> "0 1 1"

getDOMAttribute (composantName)

getDOMAttribute récupère uniquement les données de composant analysées qui sont explicitement définies dans le DOM ou via setAttribute. Si componentName est le nom d’un composant enregistré, getDOMAttribute renverra uniquement les données de composant définies dans le HTML en tant qu’objet analysé. getDOMAttribute pour les composants est la forme partielle de getAttribute car les données de composant renvoyées n’incluent pas les mixins appliqués ou les valeurs par défaut :

Comparez la sortie de l’exemple ci-dessus de getAttribute :

// <a-entity geometry="primitive: box; width: 3">
entity.getDOMAttribute('geometry');
// >> { primitive: "box", width: 3 }
entity.getDOMAttribute('geometry').primitive;
// >> "box"
entity.getDOMAttribute('geometry').height;
// >> undefined
entity.getDOMAttribute('position');
// >> undefined

getObject3D (type)

getObject3D recherche un enfant THREE.Object3D référencé par type sur object3DMap.

AFRAME.registerComponent('example-mesh', {
  init: function () {
     var el = this.el;
     el.getOrCreateObject3D('mesh', THREE.Mesh);
     el.getObject3D('mesh');  // Returns THREE.Mesh that was just created.
  }
});

getOrCreateObject3D (type, constructeur)

Si l’entité n’a pas de THREE.Object3D enregistré sous type, getOrCreateObject3D enregistrera un THREE.Object3D instancié en utilisant le Constructor passé. Si l’entité a un THREE.Object3D enregistré sous type, getOrCreateObject3D agira comme getObject3D :

AFRAME.registerComponent('example-geometry', {
  update: function () {
    var mesh = this.el.getOrCreateObject3D('mesh', THREE.Mesh);
    mesh.geometry = new THREE.Geometry();
  }
});

pause ()

pause() arrêtera tout comportement dynamique tel que défini par les animations et les composants. Lorsque nous mettons une entité en pause, elle arrête ses animations et appelle Component.pause() sur chacun de ses composants. Les composants décident d’implémenter ce qui se passe en pause, ce qui supprime souvent les écouteurs d’événements. Une entité appellera pause() sur ses entités enfants lorsque nous mettrons en pause une entité.

// <a-entity id="spinning-jumping-ball">
entity.pause();

Par exemple, le composant look-controls en pause supprimera les gestionnaires d’événements qui écoutent les entrées.

jouer ()

play() démarrera tout comportement dynamique tel que défini par les animations et les composants. Ceci est automatiquement appelé lorsque le DOM attache une entité. Lorsqu’une entité play(), l’entité appelle play() sur ses entités enfants.

entity.pause();
entity.play();

Par exemple, le composant sonore en cours de lecture commencera à jouer le son.

setAttribute (componentName, valeur, [propertyValue | clobber])

Si componentName n’est pas le nom d’un composant enregistré ou si le composant est un composant à propriété unique, setAttribute se comporte normalement :

entity.setAttribute('visible', false);

Cependant, si componentName est le nom d’un composant enregistré, il peut gérer une analyse spéciale pour la valeur. Par exemple, le composant de position est un composant à propriété unique, mais son analyseur de type de propriété lui permet de prendre un objet :

entity.setAttribute('position', { x: 1, y: 2, z: 3 });

setObject3D (type, obj)

setObject3D enregistrera l’obj passé, un THREE.Object3D, comme type sous le object3DMap de l’entité. A-Frame ajoute obj en tant qu’enfant de la racine object3D de l’entité.

AFRAME.registerComponent('example-orthogonal-camera', {
  update: function () {
    this.el.setObject3D('camera', new THREE.OrthogonalCamera());
  }
});

removeAttribute (componentName, propertyName)

Si componentName est le nom d’un composant enregistré, en plus de supprimer l’attribut du DOM, removeAttribute détachera également le composant de l’entité, en appelant la méthode de cycle de vie remove du composant.

entity.removeAttribute('goemetry');  // Detach the geometry component.
entity.removeAttribute('sound');  // Detach the sound component.

Si propertyName est donné, removeAttribute réinitialisera la valeur de propriété de cette propriété spécifiée par propertyName à la valeur par défaut de la propriété :

entity.setAttribute('material', 'color', 'blue');  // The color is blue.
entity.removeAttribute('material', 'color');  // Reset the color to the default value, white.

removeObject3D (type)

removeObject3D supprime l’objet spécifié par type du THREE.Group de l’entité et donc de la scène. Cela mettra à jour le object3DMap de l’entité, en définissant la valeur de la clé type sur null. Ceci est généralement appelé à partir d’un composant, souvent dans le gestionnaire de suppression :

AFRAME.registerComponent('example-light', {
  update: function () {
    this.el.setObject3D('light', new THREE.Light());
    // Light is now part of the scene.
    // object3DMap.light is now a THREE.Light() object.
  },
  remove: function () {
    this.el.removeObject3D('light');
    // Light is now removed from the scene.
    // object3DMap.light is now null.
  }
});

removeState (stateName)

removeState fera apparaître un état de l’entité. Cela émettra l’événement stateremoved, et nous pouvons vérifier l’état de sa suppression en utilisant .is :

entity.addEventListener('stateremoved', function (evt) {
  if (evt.detail.state === 'selected') {
    console.log('Entity no longer selected.');
  }
});
entity.addState('selected');
entity.is('selected');  // >> true
entity.removeState('selected');
entity.is('selected');  // >> false

Écoute des changements de composants #

Nous pouvons utiliser l’événement componentchanged pour écouter les modifications apportées à l’entité :

entity.addEventListener('componentchanged', function (evt) {
  if (evt.detail.name === 'position') {
    console.log('Entity has moved from', 
      evt.detail.oldData, 'to', evt.detail.newData, '!');
  }
});

Écoute des éléments enfants attachés et détachés #

Nous pouvons utiliser les événements child-attached et child-dependent pour écouter quand la scène attache ou détache une entité :

entity.addEventListener('child-attached', function (evt) {
  if (evt.detail.el.tagName.toLowerCase() === 'a-box') {
    console.log('a box element has been attached');
  };
});

Données de composant multi-propriété d’entité (setAttribute) #

Mise à jour des données de composants multi-propriétés #

Pour mettre à jour les données de composant pour un composant multi-propriété, nous pouvons transmettre le nom d’un composant enregistré en tant que componentName et transmettre un objet de propriétés en tant que valeur. Une chaîne est également acceptable (par exemple, type : spot ; distance : 30), mais les objets permettront à A-Frame d’économiser du travail d’analyse :

// Only the properties passed in the object will be overwritten.
entity.setAttribute('light', {
  type: 'spot',
  distance: 30,
  intensity: 2.0
});

Ou pour mettre à jour les propriétés individuelles d’un composant multi-propriété, nous pouvons transmettre le nom du composant enregistré en tant que componentName, un nom de propriété en deuxième argument et la valeur de la propriété à définir en troisième argument :

// All previous properties for the material component (besides the color)  will be unaffected.
entity.setAttribute('material', 'color', 'crimson');

Notez que les types de propriétés de tableau se comportent de manière unique :

  • Les tableaux sont modifiables. Ils sont affectés par référence afin que les modifications apportées aux tableaux soient visibles par le composant.
  • Les mises à jour des propriétés de type tableau ne déclencheront pas la méthode de mise à jour du composant ni n’émettront d’événements.

Mise à jour des données de composants multi-propriétés #

Si true est passé comme troisième argument à .setAttribute, alors les propriétés non spécifiées seront réinitialisées et supprimées :

// All previous properties for the light component will be removed and overwritten.
entity.setAttribute('light', {
  type: 'spot',
  distance: 30,
  intensity: 2.0
}, true);

Récupération d’une entité #

Nous pouvons simplement récupérer une entité à l’aide des API DOM.

<a-entity id="mario"></a-entity>

var el = document.querySelector('#mario');

Récupération des composants d’une entité #

Par exemple, si nous voulions saisir l’objet caméra ou l’objet matériel three.js d’une entité, nous pourrions accéder à ses composants

var camera = document.querySelector('a-entity[camera]').components.camera.camera;
var material = document.querySelector('a-entity[material]').components.material.material;

Ou si un composant expose une API, nous pouvons appeler ses méthodes :

document.querySelector('a-entity[sound]').components.sound.pause();
