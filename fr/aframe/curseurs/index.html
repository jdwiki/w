Le composant curseur nous permet d’interagir avec les entités en cliquant et en regardant.

Syntaxe #

  • <un curseur d’entité=""></un-curseur>
  • <un-curseur></un-curseur>

Paramètres #

Propriété La description
fusible Indique si le curseur est basé sur un fusible. Valeur par défaut : “false” sur ordinateur, “true” sur mobile
fuseTimeout Combien de temps attendre (en millisecondes) avant de déclencher un événement de clic basé sur un fusible. Valeur par défaut : 1500

Le curseur est une application spécifique du [composant raycaster][1] en ce qu’il

  • Écoute les clics de souris et les fusibles basés sur le regard
  • Capture uniquement la première entité intersectée
  • Émet des événements spéciaux de souris et de survol (par exemple, liés à la souris enfoncée/haute/entrée/sortie)
  • A plus d’états pour le vol stationnaire.

Lorsque la souris clique, l’entité visible la plus proche croisant le curseur, le cas échéant, émettra un événement de clic. Notez que le composant du curseur applique uniquement le comportement de raycasting. Pour donner une forme ou une apparence au curseur, vous pouvez appliquer les composants de géométrie et de matériau.

Événements #

Événement La description
cliquez Émis à la fois sur le curseur et sur l’entité intersectée si une entité actuellement intersectée est cliquée (que ce soit avec la souris ou par fusible).
fusionner Émis à la fois sur le curseur et sur l’entité intersectée lorsque le curseur basé sur un fusible commence le compte à rebours.
mousedown Émis à la fois sur le curseur et sur l’entité intersectée (le cas échéant) lors du clic de la souris sur l’élément canvas.
mouseenter Émis à la fois sur le curseur et sur l’entité intersectée (le cas échéant) lorsque le curseur croise une entité.
congé de souris Émis à la fois sur le curseur et sur l’entité intersectée (le cas échéant) lorsque le curseur ne croise plus l’entité précédemment intersectée.
mouseup Émis à la fois sur le curseur et sur l’entité intersectée (le cas échéant) lors du déplacement de la souris sur l’élément canvas.

[1] : https://www.wikiod.com/fr/aframe/raycasters-composant

Curseur par défaut #

Par exemple, nous pouvons créer un curseur en forme d’anneau fixé au centre de l’écran. Pour fixer le curseur à l’écran afin que le curseur soit toujours présent, peu importe où nous regardons, nous le plaçons en tant qu’enfant de l’entité de caméra active. Nous le tirons devant la caméra en le plaçant sur l’axe Z négatif. Lorsque le curseur clique sur la case, nous pouvons écouter l’événement click.

<a-entity camera>
  <a-entity cursor="fuse: true; fuseTimeout: 500"
            position="0 0 -1"
            geometry="primitive: ring; radiusInner: 0.02; radiusOuter: 0.03"
            material="color: black; shader: flat">
  </a-entity>
</a-entity>

<a-entity id="box" cursor-listener geometry="primitive: box" material="color: blue">
</a-entity>

// Component to change to random color on click.
AFRAME.registerComponent('cursor-listener', {
  init: function () {
    var COLORS = ['red', 'green', 'blue'];
    this.el.addEventListener('click', function (evt) {
      var randomIndex = Math.floor(Math.random() * COLORS.length);
      this.setAttribute('material', 'color', COLORS[randomIndex]);
      console.log('I was clicked at: ', evt.detail.intersection.point);
    });
  }
});

## Interactions basées sur le regard avec le composant de curseur Nous allons d’abord passer en revue les interactions basées sur le regard. Les interactions basées sur le regard reposent sur la rotation de la tête et sur l’observation des objets pour interagir avec eux. Ce type d’interaction concerne les casques sans contrôleur. Même avec un contrôleur de rotation uniquement (Daydream, GearVR), l’interaction est toujours similaire. Étant donné que A-Frame fournit des commandes de déplacement de la souris par défaut, le regard peut être utilisé sur le bureau pour prévisualiser l’interaction en faisant glisser la rotation de la caméra.

Pour ajouter une interaction basée sur le regard, nous devons ajouter ou implémenter un composant. A-Frame est livré avec un composant de curseur qui fournit une interaction basée sur le regard s’il est attaché à la caméra :

  1. Définissez explicitement l’entité . Auparavant, A-Frame fournissait la caméra par défaut.

  2. Ajoutez l’entité a-cursor en tant qu’élément enfant sous l’entité caméra.

  3. Facultativement, configurez le raycaster utilisé par le curseur.

primitive du curseur a #

La primitive de curseur est un réticule qui permet de cliquer et d’interactivité de base avec une scène sur les appareils qui n’ont pas de contrôleur manuel. L’apparence par défaut est une géométrie en anneau. Le curseur est généralement placé comme un enfant de la caméra.

<a-scene>
  <a-camera>
    <a-cursor></a-cursor>
  </a-camera>
  <a-box></a-box>
</a-scene>

Curseur à base de fusible #

Également connu sous le nom de curseur basé sur le regard. Si nous définissons le curseur sur un fusible, le curseur déclenchera un clic si l’utilisateur regarde une entité pendant un certain temps. Imaginez un laser attaché à la tête de l’utilisateur, et le laser s’étend dans la scène. Si l’utilisateur regarde une entité assez longtemps (c’est-à-dire le fuseTimeout), le curseur déclenchera un clic.

L’avantage des interactions basées sur les fusibles pour la réalité virtuelle est qu’elle ne nécessite pas de périphériques d’entrée supplémentaires autres que le casque. Le curseur à fusible est principalement destiné aux applications Google Cardboard. L’inconvénient des interactions basées sur les fusibles est qu’elles obligent l’utilisateur à tourner beaucoup la tête.

Configuration du curseur via le composant Raycaster #

Le curseur s’appuie sur et dépend du composant raycaster. Si nous voulons personnaliser les éléments de raycasting du curseur, nous pouvons le faire en modifiant les propriétés du composant raycaster. Supposons que nous voulions définir une distance maximale, vérifier les intersections moins fréquemment et définir les objets cliquables :

<a-entity cursor raycaster="far: 20; interval: 1000; objects: .clickable"></a-entity>

Ajout de commentaires visuels #

Pour ajouter un retour visuel au curseur pour montrer quand le curseur clique ou fusionne, nous pouvons utiliser le système d’animation. Lorsque le curseur croise l’entité, il émettra un événement, et le système d’animation captera l’événement avec l’attribut begin :

<a-entity cursor="fuse: true; fuseTimeout: 500"
          position="0 0 -1"
          geometry="primitive: ring"
          material="color: black; shader: flat">
  <a-animation begin="click" easing="ease-in" attribute="scale"
               fill="backwards" from="0.1 0.1 0.1" to="1 1 1"></a-animation>
  <a-animation begin="cursor-fusing" easing="ease-in" attribute="scale"
               fill="forwards" from="1 1 1" to="0.1 0.1 0.1"></a-animation>
</a-entity>

Curseur de la souris #

Remarque : Pour cet exemple, vous devez ajouter un package npm externe.

Si vous souhaitez utiliser un curseur de souris sur votre ordinateur, vous devez ajouter [aframe-mouse-cursor-component][1]. Après si vous devez inclure le script en utilisant ce code :

import 'aframe';
import 'aframe-mouse-cursor-component';

// or this 

require('aframe');
require('aframe-mouse-cursor-component');

Et sur votre appareil photo, vous devez ajouter le composant “mouse-cursor”.

<a-scene>
  <a-entity camera look-controls mouse-cursor>
</a-scene>

[1] : https://github.com/mayognaise/aframe-mouse-cursor-component
