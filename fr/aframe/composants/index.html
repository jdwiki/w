Dans le modèle entité-composant-système, un composant est un bloc de données réutilisable et modulaire que nous connectons à une entité pour ajouter une apparence, un comportement et/ou des fonctionnalités.

Dans A-Frame, les composants modifient les entités qui sont des objets 3D dans la scène. Nous mélangeons et composons des composants ensemble pour construire des objets complexes. Ils nous permettent d’encapsuler du code three.js et JavaScript dans des modules que nous pouvons utiliser de manière déclarative à partir de HTML. Les composants sont à peu près analogues à CSS.

Définition des méthodes du gestionnaire de cycle de vie #

Le schéma étant l’anatomie, les méthodes du cycle de vie sont la physiologie ; le schéma définit la forme des données, les méthodes du gestionnaire de cycle de vie utilisent les données pour modifier l’entité. Les gestionnaires interagiront généralement avec le API d’entité.

Présentation des méthodes #

Méthode Descriptif
initialisation Appelé une fois lorsque le composant est initialisé. Utilisé pour configurer l’état initial et instancier des variables.
mise à jour Appelé à la fois lorsque le composant est initialisé et chaque fois que l’une des propriétés du composant est mise à jour (par exemple, via setAttribute). Utilisé pour modifier l’entité.
supprimer Appelé lorsque le composant est supprimé de l’entité (par exemple, via removeAttribute) ou lorsque l’entité est détachée de la scène. Utilisé pour annuler toutes les modifications précédentes de l’entité.
cocher Appelé à chaque boucle de rendu ou tick de la scène. Utilisé pour les modifications ou les contrôles continus.
jouer Appelé chaque fois que la scène ou l’entité est lue pour ajouter un arrière-plan ou un comportement dynamique. Également appelée une fois lorsque le composant est initialisé. Utilisé pour démarrer ou reprendre un comportement.
pause Appelé chaque fois que la scène ou l’entité s’arrête pour supprimer tout arrière-plan ou comportement dynamique. Également appelé lorsque le composant est supprimé de l’entité ou lorsque l’entité est détachée de la scène. Utilisé pour mettre en pause le comportement.
mettre à jour le schéma Appelé chaque fois que l’une des propriétés du composant est mise à jour. Peut être utilisé pour modifier dynamiquement le schéma.

Propriétés du prototype de composant #

Dans les méthodes, nous avons accès au prototype du composant via this :

| Propriété | Descriptif | |—————-|——————————- ————————————————– ————————————————– ————| | this.data | Propriétés de composant analysées calculées à partir des valeurs par défaut du schéma, des mixins et des attributs de l’entité. | | this.el | Référence à l’[entité][entité] en tant qu’élément HTML. | | this.el.sceneEl | Référence à la [scène][scène] en tant qu’élément HTML. | | this.id | Si le composant peut avoir [plusieurs instances][plusieurs], l’ID de l’instance individuelle du composant (par exemple, foo de sound__foo). |

MÉTHODES #

.init()

.init () est appelé une fois au début du cycle de vie du composant. Une entité peut appeler le gestionnaire init du composant :

  • Lorsque le composant est défini statiquement sur l’entité dans le fichier HTML et que la page est chargée.
  • Lorsque le composant est défini sur une entité attachée via setAttribute.
  • Lorsque le composant est défini sur une entité non attachée et que l’entité est puis attaché à la scène via appendChild.

Le gestionnaire init est souvent utilisé pour :

  • Configurer l’état initial et les variables
  • Méthodes de liaison
  • Attacher des écouteurs d’événement

Par exemple, init d’un composant de curseur définirait des variables d’état, lierait méthodes et ajoutez des écouteurs d’événement :

AFRAME.registerComponent('cursor', {
  // ...
  init: function () {
    // Set up initial state and variables.
    this.intersection = null;
    // Bind methods.
    this.onIntersection = AFRAME.utils.bind(this.onIntersection, this);
    // Attach event listener.
    this.el.addEventListener('raycaster-intersection', this.onIntersection);
  }
  // ...
});

.mise à jour (anciennes données)

.update (oldData) est appelé chaque fois que les propriétés du composant changent, y compris au début du cycle de vie du composant. Une entité peut appeler un gestionnaire de “mise à jour” du composant :

  • Après l’appel de init (), au début du cycle de vie du composant.
  • Lorsque les propriétés du composant sont mises à jour avec .setAttribute.

Le gestionnaire update est souvent utilisé pour :

  • Effectuez la majeure partie du travail en apportant des modifications à l’entité, en utilisant this.data.
  • Modifier l’entité chaque fois qu’une ou plusieurs propriétés de composant changent.

Des modifications granulaires de l’entité peuvent être effectuées en [diffant][diff] le courant jeu de données (this.data) avec le jeu de données précédent avant la mise à jour (oldData).

A-Frame appelle .update() à la fois au début du cycle de vie d’un composant et à chaque moment où les données d’un composant changent (par exemple, à la suite de setAttribute). La le gestionnaire de mise à jour utilise souvent this.data pour modifier l’entité. Le gestionnaire de mise à jour a accès à l’état précédent des données d’un composant via son premier argument. Nous pouvons utiliser les données précédentes d’un composant pour dire exactement quel les propriétés ont été modifiées pour effectuer des mises à jour granulaires.

Par exemple, la “mise à jour” du composant visible définit la visibilité de l’entité.

AFRAME.registerComponent('visible', {
  /**
   * this.el is the entity element.
   * this.el.object3D is the three.js object of the entity.
   * this.data is the component's property or properties.
   */
  update: function (oldData) {
    this.el.object3D.visible = this.data;
  }
  // ...
});

.supprimer ()

.remove () est appelé chaque fois que le composant est détaché de l’entité. Un l’entité peut appeler le gestionnaire “remove” d’un composant :

  • Lorsque le composant est supprimé de l’entité via removeAttribute.
  • Lorsque l’entité est détachée de la scène (par exemple, removeChild).

Le gestionnaire “remove” est souvent utilisé pour :

  • Supprimez, annulez ou nettoyez toutes les modifications apportées par le composant à l’entité.
  • Détacher les écouteurs d’événement.

Par exemple, lorsque la [composante lumineuse][lumière] est supprimée, la composante lumineuse supprimera l’objet lumière qu’il avait précédemment posé sur l’entité, ainsi le retirer de la scène.

AFRAME.registerComponent('light', {
  // ...
  remove: function () {
    this.el.removeObject3D('light');
  }
  // ...
});

.tick (heure, timeDelta)

.tick () est appelé à chaque tick ou frame de la boucle de rendu de la scène. La scène appellera le gestionnaire tick d’un composant :

  • Sur chaque image de la boucle de rendu.
  • De l’ordre de 60 à 120 fois par seconde.
  • Si l’entité ou la scène n’est pas en pause (par exemple, l’inspecteur est ouvert).
  • Si l’entité est toujours attachée à la scène.

Le gestionnaire tick est souvent utilisé pour :

  • Modifier en continu l’entité sur chaque trame ou sur un intervalle.
  • Sondage pour les conditions.

Le gestionnaire tick fournit le temps de disponibilité global de la scène en millisecondes (“time”) et la différence de temps en millisecondes depuis la dernière image (timeDelta). Ceux-ci peuvent être utilisés pour l’interpolation ou pour n’exécuter que des parties du gestionnaire tick sur un intervalle défini.

Par exemple, le composant de contrôles suivis progressera les animations du contrôleur, mettre à jour la position et la rotation du contrôleur, et vérifier les pressions sur les boutons.

AFRAME.registerComponent('tracked-controls', {
  // ...
  tick: function (time, timeDelta) {
    this.updateMeshAnimation();
    this.updatePose();
    this.updateButtons();
  }
  // ...
});

.pause ()

.pause () est appelé lorsque l’entité ou la scène s’arrête. L’entité peut appeler un gestionnaire pause du composant :

  • Avant que le composant ne soit supprimé, avant que le gestionnaire remove ne soit appelé.
  • Lorsque l’entité est mise en pause avec Entity.pause ().
  • Lorsque la scène est mise en pause avec Scene.pause () (par exemple, l’inspecteur est ouvert).

Le gestionnaire pause est souvent utilisé pour :

  • Supprimer les écouteurs d’événements.
  • Supprimez toutes les chances de comportement dynamique.

Par exemple, le composant sonore mettra le son en pause et supprimera un écouteur d’événement qui aurait joué un son sur un événement :

AFRAME.registerComponent('sound', {
  // ...
  pause: function () {
    this.pauseSound();
    this.removeEventListener();
  }
  // ...
});

.jouer ()

.play () est appelé lorsque l’entité ou la scène reprend. L’entité peut appeler le gestionnaire play d’un composant :

  • Lorsque le composant est attaché pour la première fois, après l’appel du gestionnaire update.
  • Lorsque l’entité a été mise en pause puis reprise avec Entity.play ().
  • Lorsque la scène a été mise en pause puis reprise avec Scene.play ().

Le gestionnaire play est souvent utilisé pour :

  • Ajouter des écouteurs d’événements.

Par exemple, le composant sonore jouera le son et mettra à jour le écouteur d’événement qui jouerait un son sur un événement :

AFRAME.registerComponent('sound', {
  // ...
  play: function () {
    if (this.data.autoplay) { this.playSound(); }
    this.updateEventListener();
  }
  // ...
});

.updateSchema (données)

.updateSchema (), s’il est défini, est appelé à chaque mise à jour afin de vérifier si le schéma doit être modifié dynamiquement.

Le gestionnaire updateSchema est souvent utilisé pour :

  • Mettre à jour ou étendre dynamiquement le schéma, généralement en fonction de la valeur d’une propriété.

Par exemple, le composant de géométrie vérifie si la primitive propriété modifiée pour déterminer s’il faut mettre à jour le schéma pour un autre type de géométrie :

AFRAME.registerComponent('geometry', {
  // ...
  updateSchema: (newData) {
    if (newData.primitive !== this.data.primitive) {
      this.extendSchema(GEOMETRIES[newData.primitive].schema);
    }
  }
  // ...
});

MÉTHODES DE PROTOTYPES DE COMPOSANTS #

.flushToDOM()

Pour économiser du temps CPU sur la stringification, A-Frame ne mettra à jour en mode débogage que la représentation sérialisée du composant dans le DOM réel. Appeler flushToDOM () sérialisera manuellement les données du composant et mettra à jour le DOM :

document.querySelector('[geometry]').components.geometry.flushToDOM();

Enregistrer un composant A-Frame personnalisé #

AFRAME.registerComponent (nom, définition) #

Enregistrez un composant A-Frame. Nous devons enregistrer les composants avant de les utiliser n’importe où dans . Cela signifie qu’à partir d’un fichier HTML, les composants doivent arriver dans l’ordre avant .

  • {string} name - Nom du composant. L’API publique du composant représentée par un nom d’attribut HTML.
  • Définition {Objet} - Définition du composant. Contient des méthodes de schéma et de gestionnaire de cycle de vie.

Enregistrement du composant dans foo dans votre fichier js, par exemple foo-component.js #

AFRAME.registerComponent('foo', {
  schema: {},
  init: function () {},
  update: function () {},
  tick: function () {},
  remove: function () {},
  pause: function () {},
  play: function () {}
});

Utilisation du composant foo dans votre scène #

<html>
  <head>
    <script src="aframe.min.js"></script>
    <script src="foo-component.js"></script>
  </head>
  <body>
    <a-scene>
      <a-entity foo></a-entity>
    </a-scene>
  </body>
</html>

Formulaire HTML du composant #

Un composant contient un ensemble de données sous la forme d’une ou plusieurs propriétés de composant. Les composants utilisent ces données pour modifier les entités. Considérons un composant de moteur, nous pourrions définir des propriétés telles que la puissance ou les cylindres.

Les attributs HTML représentent les noms des composants et la valeur de ces attributs représente les données des composants.

Composant à propriété unique #

Si un composant est un composant à propriété unique, ce qui signifie que ses données consistent en une seule valeur, alors en HTML, la valeur du composant ressemble à un attribut HTML normal :

<!-- `position` is the name of the position component. -->
<!-- `1 2 3` is the data of the position component. -->
<a-entity position="1 2 3"></a-entity>

Composant multi-propriétés #

Si un composant est un composant multi-propriété, ce qui signifie que les données sont constituées de plusieurs propriétés et valeurs, alors en HTML, la valeur du composant ressemble aux styles CSS en ligne :

<!-- `light` is the name of the light component. -->
<!-- The `type` property of the light is set to `point`. -->
<!-- The `color` property of the light is set to `crimson`. -->
<a-entity light="type: point; color: crimson"></a-entity>

Définition de l’objet de schéma de composant #

Le schéma est un objet qui définit et décrit la propriété ou les propriétés du composant. Les clés du schéma sont les noms de la propriété, et les valeurs du schéma définissent les types et les valeurs de la propriété (dans le cas d’un composant multi-propriété) :

Définition du schéma dans votre composant

AFRAME.registerComponent('bar', {
  schema: {
    color: {default: '#FFF'},
    size: {type: 'int', default: 5}
  }
}

Remplacer les valeurs par défaut du schéma défini

<a-scene>
  <a-entity bar="color: red; size: 20"></a-entity>
</a-scene>

Schéma de propriété unique #

Un composant peut être soit un composant à propriété unique (constitué d’un valeur anonyme) ou un composant multi-propriété (constitué de plusieurs valeurs). A-Frame déduira si un composant est à propriété unique ou non. multi-propriété basée sur la structure du schéma.

Le schéma d’un composant à propriété unique contient les clés “type” et/ou “default”, et les valeurs du schéma sont des valeurs simples plutôt que des objets :

AFRAME.registerComponent('foo', {
  schema: {type: 'int', default: 5}
});

<a-scene>
  <a-entity foo="20"></a-entity>
</a-scene>

Types de propriétés de schéma de composant A-Frame #

Les types de propriétés définissent principalement la façon dont le schéma analyse les données entrantes du DOM pour chaque propriété. Les données analysées seront alors disponibles via le data propriété sur le prototype du composant. Vous trouverez ci-dessous la propriété intégrée de A-Frame les types:

Type de propriété Descriptif Valeur par défaut
tableau Analyse les valeurs séparées par des virgules en tableau (c’est-à-dire "1, 2, 3" en ['1', '2', '3']). []
atout Pour les URL pointant vers des actifs généraux. Peut analyser l’URL d’une chaîne sous la forme url(<url>). Si la valeur est un sélecteur d’ID d’élément (par exemple, #texture), ce type de propriété appellera getElementById et getAttribute('src') pour renvoyer une URL. Le type de propriété asset peut ou non changer pour gérer les XHR ou renvoyer directement les MediaElements (par exemple, les éléments <img>). ''
Audio Même analyse que le type de propriété asset. Sera éventuellement utilisé par l’inspecteur A-Frame pour présenter les ressources audio. ''
booléen Analyse la chaîne en booléen (c’est-à-dire, "faux" en faux, tout le reste est vrai). faux
couleur Actuellement, ne fait aucune analyse. Principalement utilisé par l’inspecteur A-Frame pour présenter un sélecteur de couleurs. En outre, il est nécessaire d’utiliser le type de couleur pour que les animations en couleur fonctionnent. #FFF
entier Appelle parseInt (par exemple, "124.5" à 124). 0
carte Même analyse que le type de propriété asset. Sera éventuellement utilisé par l’inspecteur A-Frame pour présenter les éléments de texture. ''
modèle Même analyse que le type de propriété asset. Sera éventuellement utilisé par l’inspecteur A-Frame pour présenter les actifs du modèle. ''
numéro Appelle parseFloat (par exemple, '124.5' à '124.5'). 0
sélecteur Appelle querySelector (par exemple, "#box" à <a-entity id="box">). nul
selectorAll Appelle querySelectorAll et convertit NodeList en Array (par exemple, ".boxes" en [<a-entity class=“boxes”, …]), nul
chaîne Ne fait aucune analyse. ''
vec2 Analyse deux nombres dans un objet {x, y} (par exemple, 1 -2 à {x : 1, y : -2}. {x : 0, y : 0}
vec3 Analyse trois nombres dans un objet {x, y, z} (par exemple, 1 -2 3 à {x : 1, y : -2, z : 3}. {x : 0, y : 0, z : 0}
vec4 Analyse quatre nombres dans un objet {x, y, z, w} (par exemple, 1 -2 3 -4.5 à {x : 1, y : -2, z : 3, w : -4.5} . {x : 0, y : 0, z : 0, w : 0}

Inférence de type de propriété #

Le schéma essaiera de déduire un type de propriété en ne donnant qu’une valeur par défaut :

schema: {default: 10}  // type: "number"
schema: {default: "foo"}  // type: "string"
schema: {default: [1, 2, 3]}  // type: "array"

Le schéma définira une valeur par défaut si elle n’est pas fournie, compte tenu du type de propriété :

schema: {type: 'number'}  // default: 0
schema: {type: 'string'}  // default: ''
schema: {type: 'vec3'}  // default: {x: 0, y: 0, z: 0}

Type de propriété personnalisée #

Nous pouvons également définir notre propre type de propriété ou analyseur en fournissant un “analyseur” fonction à la place d’un type :

schema: {
  // Parse slash-delimited string to an array 
  // (e.g., `foo="myProperty: a/b"` to `['a', 'b']`).
  myProperty: {
    default: [],
    parse: function (value) {
      return value.split('/');
    }
  }
}

Accéder aux membres et aux méthodes d’un composant #

Les membres et les méthodes d’un composant sont accessibles via l’entité à partir de l’objet .components. Recherchez le composant dans la carte des composants de l’entité et nous aurons accès aux éléments internes du composant. Considérez cet exemple de composant :

AFRAME.registerComponent('foo', {
  init: function () {
    this.bar = 'baz';
  },
  qux: function () {
    // ...
  }
});

Accédons au membre bar et à la méthode qux :

var fooComponent = document.querySelector('[foo]').components.foo;
console.log(fooComponent.bar);
fooComponent.qux();
