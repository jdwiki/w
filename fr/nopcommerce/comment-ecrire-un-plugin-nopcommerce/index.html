Comment écrire un plugin nopCommerce

  • Merci!
  • Créer un projet de plugin

Sur cette page

  • Merci!
  • Créer un projet de plugin

Chaque nouvelle fonctionnalité de NopCommerce se présente sous la forme d’un plugin.

Cette documentation vous aide à créer un plugin de nature basique, mais vous explique comment un plugin est conçu pour NopCommerce.

Étant donné que NopCommerce est un projet Open Source, il existe de nombreux plugins disponibles à titre de référence qui sont très utiles.

Ceci est juste un exemple pour créer un exemple de plugin.

Vous pouvez améliorer les fonctionnalités de ce plugin ou ajouter un plugin totalement nouveau.

J’espère que cela a aidé.

Merci! #

Créer un projet de plugin #

  1. Obtenez le code source du site Web NopCommerce et ouvrez-le dans Visual Studio

  2. Dans le dossier des plugins de la solution NopCommerce, ajoutez un projet de bibliothèque de classes de type avec le nom du plugin préfixé par Nop.Plugin comme Nop.Pugin.CategoryName.PluginName comme nom. Construisez ensuite la solution.

  3. Supprimez le fichier class.cs qui est ajouté automatiquement au projet.

  4. Ajoutez les références à Nop.Core.dll, Nop.Data.dll, Nop.Services.dll, Nop.Web.Framework.dll, System.Web.Mvc.dll, Autofac.dll, Autofac.Integration.Mvc.dll, EntityFramework.dll, FluentValidation.dll et autres nécessaires au plugin.

  5. Créez un fichier texte Description.txt dans le nouveau dossier et copiez le contenu de tout autre plugin et modifiez-le en fonction de vos données de plugin. Ce fichier est nécessaire au fonctionnement d’un plugin.

    An example Description.txt is here.

    Group: Plugin category name here
    FriendlyName: Plugin Name here
    SystemName: CategoryName.PluginName
    Version: 1.0
    SupportedVersions: 3.80
    Author: Your Name
    DisplayOrder: 1
    FileName: Nop.Plugin.CategoryName.PluginName.dll
    Description: This shows up in the Admin area as the plugin’s description

  6. Copiez le fichier web.config de n’importe quel autre plugin vers ce plugin et ne modifiez rien

  7. Modifiez l’emplacement de construction de bin dans le dossier du plugin (par défaut) à .\Presentation\Nop.Web\Plugins\CategoryName.PluginName\. C’est là que NoCommerce recherche les plugins lorsqu’ils sont chargés et la configuration sur “Toutes les configurations”.

  8. Nettoyer et construire la solution
