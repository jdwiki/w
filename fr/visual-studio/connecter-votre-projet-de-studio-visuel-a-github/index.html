Publication de votre projet dans un référentiel github en supprimant les données sensibles #

les étapes de cet exemple utiliseront la structure de projet suivante comme démonstration [![solution fictive][1]][1]

et nous avons l’intention de l’exporter vers le référentiel “GHTuts” [Notez que le référentiel n’existe pas encore sur github] mais laissez le “SensitiveProject” sans publier car il contient des mots de passe, des clés, etc.

  1. Tout d’abord, nous nous assurons que le plug-in de contrôle de source est défini sur “Git” dans “Outils > Options > Sélection de plug-in” [![enter image description here][2]][2]
  2. Si vous ne voyez pas l’onglet “Team Explorer”, affichez-le dans Visual Studio comme ceci [![entrez la description de l’image ici][3]][3]

  1. Accédez au dossier de votre solution locale et créez un nouveau fichier appelé “.gitignore.txt” [Noter] cette étape n’est importante que si vous avez des informations sensibles dans votre projet, sinon, laissez visual studio le créer pour vous

  2. Maintenant, ouvrez le fichier “.gitignore.txt” et collez-le dedans, c’est un modèle pour ignorer les fichiers communs de Visual Studio (vérifiez les liens ci-dessous)

  1. Ajoutez maintenant votre dossier de projet sensible au fichier “.gitignore.txt” à n’importe quelle ligne qui ne contient pas #, alors ajoutez-le simplement à la toute fin, et il devrait ressembler à ceci

[![entrez la description de l’image ici][4]][4]

  1. Faites un clic droit sur la solution et choisissez “Ajouter une solution au contrôle de code source…” [![entrez la description de l’image ici][5]][5] [Noter] il peut vous demander d’enregistrer la solution avant de continuer

  2. Vous avez maintenant un référentiel git “LOCAL” sur votre ordinateur, que VS lira, mais sans référentiel github, et vous verrez une petite icône de cadenas bleu à côté de chaque fichier de la solution qui a été ajoutée à git et un cercle rouge au projet ignoré [![entrez la description de l’image ici][6]][6]

    [Note]
    for more information about .gitignore file, check these links

  • https://help.github.com/articles/ignoring-files/
  • https://github.com/github/gitignore

  1. Allez dans l’onglet “Team Explorer” puis “Sync”

  2. Maintenant, nous créons un repo de vs à github comme celui-ci, appuyez sur le bouton “Get Started” [![entrez la description de l’image ici][7]][7]

  3. Remplissez maintenant vos informations dans github pour le nouveau Repo, puis cliquez sur “Publier” [![entrez la description de l’image ici][8]][8]

  4. Maintenant, lorsque nous allons sur github, nous voyons que notre dépôt local a été publié sur github sans notre projet sensible [Noter] l’url du repo ressemblera à ceci https://github.com/<nom d'utilisateur>/<nom de dépôt>
