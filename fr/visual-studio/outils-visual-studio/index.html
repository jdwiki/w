Extraits #

Présentation #

Depuis Visual Studio 2005, vous pouvez créer des extraits de code Intellisense. Cela vous permet de générer du code simplement en tapant un mot-clé et en appuyant deux fois sur la touche tabulation.

Utiliser le code #

Le code XML dont vous avez besoin pour créer un extrait de code Intellisense se trouve ci-dessous :

<?xml version="1.0" encoding="utf-8"?>

<CodeSnippets xmlns="http://schemas.microsoft.com/VisualStudio/CodeSnippet">
  <CodeSnippet Format="1.0.0"> <!-- format attribute is required -->

    <Header> <!-- 1 -->
      <Title></Title>
      <Author></Author>
      <Shortcut></Shortcut>
      <Description></Description>
      <Keywords>
        <Keyword>abc<Keyword>
        <Keyword>def<Keyword>
      </keywords>
    </Header>

    <Snippet> <!-- 2 -->

      <Imports> <!-- 2.1 -->
        <Import>
          <Namespace>System</Namespace>
        </Import>
      </Imports>

      <Declarations> <!-- 2.2 -->
        <Literal Editable="true/false"> <!-- 2.2.1 -->
          <ID>example</ID>
          <Type>System.String</Type>
          <ToolTip>A tip you can show</ToolTip>
          <Default>default value</Default>
          <Function></Function> <!-- 2.2.2 -->
        </Literal>

        <Object> <!-- 2.2.1 -->
          <ID>example</ID>
          <Type>System.String</Type>
          <ToolTip>A tip you can show</ToolTip>
          <Default>default value</Default>
          <Function></Function> <!-- 2.2.2 -->
        </Object>
      </Declarations>

      <References> <!-- 2.3 -->
        <Reference>
          <Assembly>System.Data.dll</Assembly>
        </Reference>
      </References>

      <Code Language=""> <!-- 2.4 -->
        <![CDATA[
            <!-- your code here if you use literals use dollar chars  -->
        ]]>
      </Code>

    </Snippet>

  </CodeSnippet>
</CodeSnippets>

Dans la balise d’extrait de code, vous avez deux balises obligatoires nommées Header et Snippet. Vous trouverez plus d’informations dans les rubriques suivantes. Le numéro près du nom correspond aux numéros du code ci-dessus.

Il peut y avoir zéro ou plusieurs éléments CodeSnippet ajoutés dans l’élément CodeSnippets.

1. En-tête #

Dans la balise Header, vous pouvez placer des informations spécifiques sur l’extrait de code et ce qu’il fait. Les balises importantes que vous pouvez utiliser à l’intérieur de cette balise sont :

Élément | La description ——— : | ———— Titre | Le titre de l’extrait. Cet attribut est obligatoire. Auteur | L’auteur de l’extrait. Raccourci | Est le raccourci que vous pouvez utiliser pour générer le code. Notez qu’il ne peut contenir que des lettres et des chiffres et doit commencer par une lettre.
Remarque : N’oubliez pas également de donner à l’extrait un nom et un raccourci corrects et uniques. Sinon, cela posera des problèmes lorsque vous importerez l’extrait de code dans Visual Studio. Descriptif | Donne plus d’informations sur l’extrait si vous en avez besoin. HelpUrl | URL d’une page d’aide sur Internet. Mots clés | Regroupe un ou plusieurs éléments de mots-clés. Types d’extraits |

Regroupe les éléments SnippetType. Cet élément contient une valeur de texte et doit être l’une des valeurs suivantes. Les types d’extraits de code sont fusionnés avec une barre oblique.


  • SurroundsWith : permet à l’extrait de code d’être placé autour d’un morceau de code sélectionné.
  • Expansion : permet d’insérer l’extrait de code au niveau du curseur.
  • Refactoring : spécifie que l’extrait de code est utilisé lors de la refactorisation de Visual C# . La refactorisation ne peut pas être utilisée dans les extraits de code personnalisés.
Liste des sources : msdn.microsoft.com

Table source (mais modifications) : msdn.microsoft.com

2. Extrait #

Dans la balise d’extrait de code, vous pouvez utiliser trois balises différentes. Cela peut être:

  • Importations
  • Déclarations *Code (obligatoire)
  • Références

Ceux-ci sont expliqués ci-dessous.

2.1 Importations #

Imports contient les espaces de noms dont vous avez besoin pour le code. Utilisez la balise d’importation à l’intérieur de cette balise et ici vous pouvez placer les espaces de noms nécessaires chacun avec la balise Namespace.

2.2 Déclarations #

Declarations peut être utilisé pour déclarer certains littéraux ou objets dans votre code dans la balise Code. Les enfants sont des littéraux et des objets.

2.2.1 Littéraux et objets

#

Les littéraux et les objets définissent les littéraux et les objets de l’extrait de code que vous pouvez modifier. Les fonctionnalités sont des littéraux et les objets sont les mêmes, mais il a une contrainte de type supplémentaire.

Le littéral et la balise d’objet peuvent contenir les enfants suivants :

  • ID : L’ID du littéral (obligatoire)
  • Type : Le type de cet objet, y compris l’espace de noms et la classe (requis par les objets)
  • ToolTip : Donne un conseil
  • Default : Une valeur par défaut de cet objet (obligatoire)
  • Fonctions

Dans les extraits, il y a des littéraux prédéfinis. Ils sont listés ci-dessous :

littéral | Détails ———– : | ————– $fin$ | Marque l’emplacement où placer le curseur après l’insertion de l’extrait de code. $sélectionné$ | Représente le texte sélectionné dans le document qui doit être inséré dans l’extrait de code lorsqu’il est invoqué. Exemple, si vous avez :

Un $selected$ est un objet que j’aime.
et que le mot était voiture sélectionné lorsque vous avez invoqué le modèle, vous obtiendrez :
Une voiture est un objet que j’aime j’aime.

2.2.2 Fonctions

#

Les fonctions dans la balise Literal ou Object signifient que vous pouvez utiliser une fonction pour générer du code en fonction d’un autre élément. Il y a trois fonctions que je connais :

Fonction | Descriptif | Langue ——- : | ———– | ————– GénérerSwitchCases (EnumerationLiteral) | Génère une instruction switch et un ensemble d’instructions case pour les membres de l’énumération spécifiée par le paramètre EnumerationLiteral. Le paramètre EnumerationLiteral doit être une référence à un littéral d’énumération ou à un type d’énumération. | Visual C# et Visual J#1 NomClasse() | Renvoie le nom de la classe qui contient l’extrait de code inséré.|Visual C# et Visual J#1 NomTypeSimple(NomType) | Réduit le paramètre TypeName à sa forme la plus simple dans le contexte dans lequel l’extrait de code a été appelé. | C# visuel

1 uniquement disponible dans Visual Studio 2005.

Attributs pour les éléments Littéral et Objet

#

Les balises Littéral et Objet peuvent avoir des attributs facultatifs.

Attribut | Descriptif | Taper ——– : | ———– | ————— Modifiable | Spécifie si vous pouvez ou non modifier le littéral après l’insertion de l’extrait de code. La valeur par défaut de cet attribut est true. | booléen

Tableau source : msdn.microsoft.com

2.3 Références #

Regroupe les éléments de référence qui contiennent des informations sur les références d’assembly pour l’extrait de code. Cela peut contenir les éléments suivants :

  • Assembly : Contient le nom de l’assembly par l’extrait de code (obligatoire)
  • Url : Contient un site Web qui donne plus d’informations sur l’assemblage

2.4 Code #

Code est le code que vous allez générer entre <![CDATA[ et ]]>. Placez le ID de votre littéral entre les caractères dollar et Visual Studio vous demandera de modifier ces valeurs par défaut si les déclarations sont remplies. Ici, vous avez un exemple pour C# et VB pour le raccourci propfull.

<!-- ... Other code ... -->
<Declarations>
  <Literal>
    <Id>variablename</Id>
    <Default>_myproperty</Default>
  </Literal>

  <Literal>
    <Id>propertytype</Id>
    <Default>int</Default>
  </Literal>

  <Literal>
    <Id>propertyname</Id>
    <Default>myproperty</Default>
  </Literal>
</Declarations>

<Code Language="CSharp">
  <![CDATA[
    private $propertyvalue$ $variablename$;

    public $propertyvalue$ $propertyname$
    {
        get { return $variablename$; }
        set { $Variablename$ = Value; }
    }
  ]]>
</Code>

<!-- ... Other code ... -->

<Declarations>
  <Literal>
    <Id>variablename</Id>
    <Default>_myproperty</Default>
  </Literal>

  <Literal>
    <Id>propertytype</Id>
    <Default>int</Default>
  </Literal>

  <Literal>
    <Id>propertyname</Id>
    <Default>myproperty</Default>
  </Literal> 
</Declarations>

<Code Language="VB">
  <![CDATA[
    Private $variablename$ As $propertyvalue$ 
    
    Public Property $propertyname$ As $propertyvalue$
        Get
            Return $variablename$ 
        End Get

        Set (ByVal value As $propertyvalue$)
            $variablename$ = value
        End Set
    End Property
  ]]>
</Code>

Dans l’attribut Language requis, vous pouvez définir votre langue dans laquelle vous créez l’extrait de code. Vous pouvez trouver les langues que vous pouvez utiliser dans le tableau suivant.

Langue | Mot clé | Disponible dans les prochaines versions ———– : | :——— | ————————————- C# visuel | CSharp | 2005, 2010, 2012 et plus tard Visual Basic | VB | 2005, 2010, 2012 et plus tard XML | XML | 2005, 2010, 2012 et plus tard Visuel J# | VJSharp | 2005, 2012 et après C++ | RPC | 2012 et après JavaScript | JavaScript | 2012 et après JScript | JScript | 2012 et après SQL | SQL | 2012 et après HTML | HTML | 2012 et après CSS | CSS | 2012 et après XAML | XAML | 2012 et après

Les autres attributs facultatifs sont :

Attribut La description
Délimiteur Spécifie le délimiteur utilisé pour décrire les littéraux et les objets dans le code. Par défaut, le délimiteur est $.
Gentil Spécifie le type de code que contient l’extrait de code et, par conséquent, l’emplacement auquel un extrait de code doit être inséré pour que l’extrait de code soit compilé.

Les valeurs valides pour la variable kind sont :

Valeur La description
corps de la méthode Spécifie que l’extrait de code est un corps de méthode et, par conséquent, doit être inséré dans une déclaration de méthode.
méthode décl Spécifie que l’extrait de code est une méthode et qu’il doit donc être inséré dans une classe ou un module.
tapez décl Spécifie que l’extrait de code est un type et, par conséquent, doit être inséré dans une classe, un module ou un espace de noms.
fichier Spécifie que l’extrait est un fichier de code complet. Ces extraits de code peuvent être insérés seuls dans un fichier de code ou à l’intérieur d’un espace de noms.
n’importe lequel Spécifie que l’extrait de code peut être inséré n’importe où. Cette balise est utilisée pour les extraits de code qui sont indépendants du contexte, tels que les commentaires.

Tables sources : msdn.microsoft.com

Importer un extrait dans Visual Studio #

  1. Enregistrez le code XML et donnez-lui l’extension .snippet.

  2. Vous pouvez ajouter le nouvel extrait créé dans Visual Studio en appuyant sur Control + K, Control + B ou allez dans “Outils” → “Gestionnaire d’extraits de code…”. Cela ouvre la fenêtre suivante :

[![La fenêtre du gestionnaire d’extraits de code][1]][1]

  1. Choisissez la langue dans la zone de liste déroulante pour laquelle vous avez créé l’extrait de code. cliquez sur “Importer…” et choisissez le fichier que vous avez créé.

[![La fenêtre Importer un extrait de code][2]][2]

  1. Cliquez sur "Terminer". Si le nom de fichier a déjà été utilisé, Visual Studio demande de remplacer le fichier existant. Vous avez trois options :
  • Remplacer : Remplace le fichier. Vous pouvez utiliser cette option si vous souhaitez modifier un ancien extrait de code.
  • Renommer : Va renommer le fichier en un nom unique.
  • Ignorer : Annule l’importation. Renomme le fichier en un nom unique.

Vous pouvez également ajouter un nouvel emplacement avec tous les extraits que vous avez créés en cliquant sur le bouton “Ajouter…” dans la première fenêtre et sélectionner le dossier dans la “fenêtre de sélection de dossier”. L’avantage est maintenant qu’un nouvel extrait de code valide est ajouté dans ce dossier, vous pouvez l’utiliser directement dans Visual Studio.

Remarque : Testez après avoir importé votre extrait de code pour détecter les erreurs, afin que vous n’ayez aucun problème lorsque vous utilisez l’extrait de code. Vous pouvez toujours supprimer ou écraser l’extrait de code en cas d’erreur.

Remplacer les outils de fusion/comparaison #

Aller à Outils | Options | Contrôle des sources | Serveur Visual Studio Team Foundation

cliquez sur Configurer les outils utilisateur :

[![entrez la description de l’image ici][1]][1]

Vous pouvez ajouter des remplacements séparés pour les opérations ‘Comparer’ et ‘Fusionner’. Cliquez sur Ajouter et sélectionnez l’opération que vous souhaitez remplacer. Vous devez saisir le chemin d’accès à l’outil que vous utilisez et les arguments exacts attendus par votre outil. Par exemple, pour utiliser BeyondCompare, ajoutez les Arguments suivants " %1 %2 /title1=%6 /title2=%7":

[![entrez la description de l’image ici][2]][2]

Pour fusionner avec BeyondCompare, utilisez les arguments “%1 %2 %3 %4 /title1=%6 /title2=%7 /title3=%8 /title4=%9”

Dans [un article de blog de 2006][3], James Manning, employé de MS, a passé en revue les arguments attendus par divers outils : WinDiff, DiffDoc, WinMerge, Beyond Compare, KDiff3, Araxis, Compare It !, SourceGear DiffMerge, TortoiseMerge et Visual SlickEdit. Le message est un bon point de départ, mais assurez-vous de vérifier la documentation à jour de votre outil.

Il est fortement recommandé de * ne pas * utiliser les outils de fusion incapables de fusionner à trois (par exemple, WinMerge 2.x).

Cadre d’entité #

Entity Framework (EF) est un mappeur objet-relationnel qui permet aux développeurs .NET de travailler avec des données relationnelles à l’aide d’objets spécifiques à un domaine. Il élimine le besoin de la plupart du code d’accès aux données que les développeurs ont généralement besoin d’écrire.

Entity Framework vous permet de créer un modèle en écrivant du code ou en utilisant des cases et des lignes dans le concepteur EF. Ces deux approches peuvent être utilisées pour cibler une base de données existante ou créer une nouvelle base de données.

Source et plus d’informations : [documentation Entity Framework][1]
