Syntaxe #

  • si expression … fin
  • si expression … sinon … fin
  • si expression … sinon si expression … fin
  • si expression … sinon si expression … sinon … fin

Paramètres #

Paramètre Descriptif
expression une expression qui a un sens logique

SI condition #

Les conditions sont une partie fondamentale de presque toutes les parties du code. Ils sont utilisés pour exécuter certaines parties du code uniquement dans certaines situations, mais pas dans d’autres. Regardons la syntaxe de base :

a = 5;
if a > 10    % this condition is not fulfilled, so nothing will happen
    disp('OK')
end

if a < 10    % this condition is fulfilled, so the statements between the if...end are executed
    disp('Not OK')
end

Production:

Not OK

Dans cet exemple, nous voyons que if se compose de 2 parties : la condition et le code à exécuter si la condition est vraie. Le code est tout ce qui est écrit après la condition et avant la “fin” de ce “si”. La première condition n’a pas été remplie et, par conséquent, le code qu’elle contient n’a pas été exécuté.

Voici un autre exemple :

a = 5;
if a ~= a+1        % "~=" means "not equal to"
    disp('It''s true!') % we use two apostrophes to tell MATLAB that the ' is part of the string
end

La condition ci-dessus sera toujours vraie et affichera la sortie “C’est vrai !”.

On peut aussi écrire :

a = 5;
if a == a+1    % "==" means "is equal to", it is NOT the assignment ("=") operator
    disp('Equal')
end

Cette fois, la condition est toujours fausse, nous n’obtiendrons donc jamais la sortie “Equal”.

Cependant, les conditions qui sont toujours vraies ou fausses ne sont pas très utiles, car si elles sont toujours fausses, nous pouvons simplement supprimer cette partie du code, et si elles sont toujours vraies, la condition n’est pas nécessaire.

Condition SI-SINON #

Dans certains cas, nous voulons exécuter un code alternatif si la condition est fausse, pour cela nous utilisons la partie optionnelle else :

a = 20;
if a < 10
    disp('a smaller than 10')
else
    disp('a bigger than 10')
end

Ici, nous voyons que parce que a n’est pas plus petit que 10 la deuxième partie du code, après que else est exécuté et nous obtenons la sortie a plus grand que 10. Voyons maintenant un autre essai :

a = 10;
if a > 10
    disp('a bigger than 10')
else
    disp('a smaller than 10')
end

Dans cet exemple montre que nous n’avons pas vérifié si a est bien inférieur à 10, et nous obtenons un mauvais message car la condition ne vérifie que l’expression telle qu’elle est, et TOUT cas qui n’est pas égal à vrai (a = 10 ) entraînera l’exécution de la deuxième partie.

Ce type d’erreur est un piège très courant pour les programmeurs débutants et expérimentés, en particulier lorsque les conditions deviennent complexes, et doit toujours être gardé à l’esprit.

Condition SI-SINON #

En utilisant else, nous pouvons effectuer une tâche lorsque la condition n’est pas satisfaite. Mais que se passe-t-il si nous voulons vérifier une deuxième condition au cas où la première serait fausse. Nous pouvons le faire de cette façon :

a = 9;
if mod(a,2)==0   % MOD - modulo operation, return the remainder after division of 'a' by 2
    disp('a is even')
else
    if mod(a,3)==0
        disp('3 is a divisor of a')
    end
end

OUTPUT:
3 is a divisor of a

Ceci est également appelé [“condition imbriquée”](https://www.wikiod.com/fr/matlab/les-conditions#Conditions imbriquées), mais ici nous avons un cas particulier qui peut améliorer la lisibilité du code et réduire le risque d’erreur anr - nous pouvons écrire :

a = 9;
if mod(a,2)==0
    disp('a is even')
elseif mod(a,3)==0
    disp('3 is a divisor of a')
end

OUTPUT:
3 is a divisor of a

en utilisant le elseif, nous pouvons vérifier une autre expression dans le même bloc de condition, et cela n’est pas limité à un essai :

a = 25;
if mod(a,2)==0
    disp('a is even')
elseif mod(a,3)==0
    disp('3 is a divisor of a')
elseif mod(a,5)==0
    disp('5 is a divisor of a')
end

OUTPUT:
5 is a divisor of a

Des précautions supplémentaires doivent être prises lors du choix d’utiliser elseif à la suite, car un seul d’entre eux sera exécuté à partir de tous les blocs if à end. Ainsi, dans notre exemple, si nous voulons afficher tous les diviseurs de a (parmi ceux que nous vérifions explicitement), l’exemple ci-dessus ne sera pas bon :

a = 15;
if mod(a,2)==0
    disp('a is even')
elseif mod(a,3)==0
    disp('3 is a divisor of a')
elseif mod(a,5)==0
    disp('5 is a divisor of a')
end

OUTPUT:
3 is a divisor of a

non seulement 3, mais aussi 5 est un diviseur de 15, mais la partie qui vérifie la division par 5 n’est pas atteinte si l’une des expressions ci-dessus était vraie.

Enfin, nous pouvons ajouter un else (et seulement un) après toutes les conditions elseif pour exécuter un code lorsqu’aucune des conditions ci-dessus n’est remplie :

a = 11;
if mod(a,2)==0
    disp('a is even')
elseif mod(a,3)==0
    disp('3 is a divisor of a')
elseif mod(a,5)==0
    disp('5 is a divisor of a')
else
    disp('2, 3 and 5 are not divisors of a')
end

OUTPUT:
2, 3 and 5 are not divisors of a

Conditions imbriquées #

Lorsque nous utilisons une condition dans une autre condition, nous disons que les conditions sont “imbriquées”. Un cas particulier de conditions imbriquées est donné par l’option [elseif](https://www.wikiod.com/fr/matlab/les-conditions#Condition SI-SINON), mais il existe de nombreuses autres façons d’utiliser conditions imbriquées. Examinons le code suivant :

a = 2;
if mod(a,2)==0    % MOD - modulo operation, return the remainder after division of 'a' by 2
    disp('a is even')
    if mod(a,3)==0
        disp('3 is a divisor of a')
        if mod(a,5)==0
            disp('5 is a divisor of a')
        end
    end
else
    disp('a is odd')
end

Pour a=2, la sortie sera a est pair, ce qui est correct. Pour a=3, la sortie sera a est impair, ce qui est également correct, mais manque la vérification si 3 est un diviseur de a. C’est parce que les conditions sont imbriquées, donc seulement si la première est ’true’, alors nous passons à l’intérieur, et si ‘a’ est impair, aucune des conditions internes n’est même vérifiée. Ceci est quelque peu opposé à l’utilisation de elseif où seulement si la première condition est false que nous vérifions la suivante. Et si on vérifiait la division par 5 ? seul un nombre qui a 6 comme diviseur (à la fois 2 et 3) sera vérifié pour la division par 5, et nous pouvons tester et voir que pour a=30 la sortie est :

a is even
3 is a divisor of a
5 is a divisor of a

Il faut aussi remarquer deux choses :

  1. La position de la “fin” au bon endroit pour chaque “si” est cruciale pour que l’ensemble de conditions fonctionne comme prévu, donc l’indentation est plus qu’une bonne recommandation ici.
  2. La position de l’instruction else est également cruciale, car nous devons savoir dans quel if (et il pourrait y en avoir plusieurs) nous voulons faire quelque chose au cas où l’expression serait false.

Prenons un autre exemple :

for a = 5:10    % the FOR loop execute all the code within it for every a from 5 to 10
    ch = num2str(a);    % NUM2STR converts the integer a to a character
    if mod(a,2)==0
        if mod(a,3)==0
            disp(['3 is a divisor of ' ch])
        elseif  mod(a,4)==0
            disp(['4 is a divisor of ' ch])
        else
            disp([ch ' is even'])
        end
    elseif mod(a,3)==0
        disp(['3 is a divisor of ' ch])
        
    else
        disp([ch ' is odd'])
    end
end

Et la sortie sera :

5 is odd
3 is a divisor of 6
7 is odd
4 is a divisor of 8
3 is a divisor of 9
10 is even

nous voyons que nous n’avons obtenu que 6 lignes pour 6 nombres, car les conditions sont imbriquées de manière à garantir une seule impression par nombre, et aussi (bien que cela ne puisse pas être vu directement à partir de la sortie) aucune vérification supplémentaire n’est effectuée, donc si un nombre n’est pas pair il ne sert à rien de vérifier si 4 est l’un de ses diviseurs.
