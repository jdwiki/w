Les opérateurs de transposition #

  • .' est la bonne façon de transposer un vecteur ou une matrice dans MATLAB.
  • ' est la bonne façon de prendre la ** transposée conjuguée complexe ** (a.k.a. conjugué hermitien) d’un vecteur ou d’une matrice dans MATLAB.

Notez que pour la transposition .', il y a un période devant l’apostrophe. Ceci est conforme à la syntaxe des autres opérations élément par élément dans MATLAB : * multiplie matrices, .* multiplie elements of matrices ensemble. Les deux commandes sont très similaires, mais conceptuellement très distinctes. Comme les autres commandes MATLAB, ces opérateurs sont du “sucre syntaxique” qui se transforme en un appel de fonction “approprié” au moment de l’exécution. Tout comme == devient une évaluation de la fonction [eq][1], pensez à .' comme raccourci pour [transpose][2]. Si vous n’écrivez que ' (sans le point), vous utilisez en fait la commande [ctranspose][3] à la place, qui calcule la [transposition conjuguée complexe][4], également connue sous le nom de [ Conjugué hermitien [5], souvent utilisé en physique. Tant que le vecteur ou la matrice transposée est à valeur réelle, les deux opérateurs produisent le même résultat. Mais dès que nous traiterons des [nombres complexes][6], nous rencontrerons inévitablement des problèmes si nous n’utilisons pas la “bonne” sténographie. Ce qui est “correct” dépend de votre application.

Considérons l’exemple suivant d’une matrice C contenant des nombres complexes :

>> C = [1i, 2; 3*1i, 4]
C =
   0.0000 + 1.0000i   2.0000 + 0.0000i
   0.0000 + 3.0000i   4.0000 + 0.0000i

Prenons la transposition en utilisant le raccourci .' (avec le point). La sortie est comme prévu, la forme transposée de ‘C’.

>> C.'
ans =
   0.0000 + 1.0000i   0.0000 + 3.0000i
   2.0000 + 0.0000i   4.0000 + 0.0000i

Maintenant, utilisons ' (sans le point). Nous voyons qu’en plus de la transposition, les valeurs complexes ont également été transformées en leurs conjugués complexes.

>> C'
ans =
   0.0000 - 1.0000i   0.0000 - 3.0000i
   2.0000 + 0.0000i   4.0000 + 0.0000i

Pour résumer, si vous avez l’intention de calculer le conjugué hermitien, le conjugué complexe transposé, alors utilisez ' (sans le point). Si vous voulez juste calculer la transposition sans conjuguer les valeurs, utilisez .' (avec le point).

[1] : http://uk.mathworks.com/help/matlab/ref/eq.html [2] : http://www.mathworks.com/help/matlab/ref/transpose.html [3] : http://www.mathworks.com/help/matlab/ref/ctranspose.html [4] : http://www.mathworks.com/help/matlab/ref/ctranspose.html#buaudse-6 [5] : http://en.wikipedia.org/wiki/Hermitian_conjugate [6] : http://www.mathworks.com/help/matlab/complex-numbers.html

Ne nommez pas une variable avec un nom de fonction existant #

Il existe déjà une fonction sum(). Par conséquent, si nous nommons une variable du même nom

sum = 1+3;

et si nous essayons d’utiliser la fonction alors que la variable existe toujours dans l’espace de travail

A = rand(2);
sum(A,1)

nous obtiendrons l’erreur cryptique :

Subscript indices must either be real positive integers or logicals.

clear() la variable d’abord, puis utilisez la fonction

clear sum

sum(A,1)
ans =
       1.0826       1.0279

Comment vérifier si une fonction existe déjà pour éviter ce conflit ?

Utilisez which() avec le drapeau -all :

which sum -all
sum is a variable.
built-in (C:\Program Files\MATLAB\R2016a\toolbox\matlab\datafun\@double\sum)   % Shadowed double method
...

Cette sortie nous indique que sum est d’abord une variable et que les méthodes (fonctions) suivantes en sont masquées, c’est-à-dire que MATLAB essaiera d’abord d’appliquer notre syntaxe à la variable, plutôt que d’utiliser la méthode.

Soyez conscient de l’inexactitude de la virgule flottante #

Les nombres à virgule flottante ne peuvent pas représenter tous les nombres réels. C’est ce qu’on appelle l’imprécision en virgule flottante.

Il existe une infinité de nombres à virgule flottante et ils peuvent être infiniment longs (par exemple π), donc être capable de les représenter parfaitement nécessiterait une quantité infinie de mémoire. Voyant que c’était un problème, une représentation spéciale pour le stockage de “nombres réels” dans l’ordinateur a été conçue, la [norme IEEE 754] [1]. En bref, il décrit comment les ordinateurs stockent ce type de nombres, avec un exposant et une mantisse, comme,

``float = signe * 2^exposant * mantisse''

Avec un nombre limité de bits pour chacun d’entre eux, seule une précision finie peut être obtenue. Plus le nombre est petit, plus l’écart entre les nombres possibles est petit (et vice versa !). Vous pouvez essayer vos vrais chiffres [dans cette démo en ligne][2].

Soyez conscient de ce comportement et essayez d’éviter toutes les comparaisons de virgules flottantes et leur utilisation comme conditions d’arrêt dans les boucles. Voir ci-dessous deux exemples :

Exemples : Comparaison en virgule flottante effectuée INCORRECT : #

>> 0.1 + 0.1 + 0.1  == 0.3

ans =

  logical

   0

C’est une mauvaise pratique d’utiliser une comparaison en virgule flottante comme le montre l’exemple précédent. Vous pouvez le surmonter en prenant la valeur absolue de leur différence et en la comparant à un (petit) niveau de tolérance.

Voici un autre exemple, où un nombre à virgule flottante est utilisé comme condition d’arrêt dans une boucle while :**

k = 0.1;
while k <= 0.3 
  disp(num2str(k));
  k = k + 0.1;
end

% --- Output: ---
0.1
0.2

Il rate la dernière boucle attendue (0.3 <= 0.3).

Exemple : comparaison en virgule flottante effectuée à DROITE : #

x = 0.1 + 0.1 + 0.1;
y = 0.3;
tolerance = 1e-10; % A "good enough" tolerance for this case.

if ( abs( x - y ) <= tolerance )
  disp('x == y');
else
  disp('x ~= y');
end

% --- Output: ---
x == y

Plusieurs choses à noter :

  • Comme prévu, maintenant x et y sont traités comme équivalents.
  • Dans l’exemple ci-dessus, le choix de la tolérance a été fait arbitrairement. Ainsi, la valeur choisie peut ne pas convenir à tous les cas (en particulier lorsque vous travaillez avec des nombres beaucoup plus petits). Le choix de la limite intelligentement peut être fait en utilisant la fonction [eps][3], c’est-à-dire N*eps(max(x,y)), où N est un nombre spécifique au problème. Un choix raisonnable pour N, qui est également suffisamment permissif, est 1E2 (même si, dans le problème ci-dessus, N = 1 suffirait également).

Ce que vous voyez n’est PAS ce que vous obtenez : char vs cellstring dans la fenêtre de commande #

Ceci est un exemple de base destiné aux nouveaux utilisateurs. Il ne se concentre pas sur l’explication de la différence entre char et cellstring.

Il peut arriver que vous vouliez vous débarrasser du ' dans vos chaînes, bien que vous ne les ayez jamais ajoutés. En fait, ce sont des artefacts que la fenêtre de commande utilise pour distinguer certains types.

Une chaîne s’imprimera

s = 'dsadasd'
s =
dsadasd

Un cellstring imprimera

c = {'dsadasd'};
c = 
    'dsadasd'

Notez que les guillemets simples et l’indentation sont des artefacts pour nous informer que c est une cellstring plutôt qu’un char. La chaîne est en fait contenue dans la cellule, c’est-à-dire

c{1}
ans =
dsadasd

Fonction ou méthode X non définie pour les arguments d’entrée de type Y #

C’est la manière longue de MATLAB de dire qu’il ne peut pas trouver la fonction que vous essayez d’appeler. Il existe plusieurs raisons pour lesquelles vous pourriez obtenir cette erreur :

Cette fonction a été introduite après votre version actuelle de MATLAB #

La documentation en ligne de MATLAB fournit une fonctionnalité très intéressante qui vous permet de déterminer dans quelle version une fonction donnée a été introduite. Il se situe en bas à gauche de chaque page de la documentation :

[![entrez la description de l’image ici][1]][1]

Comparez cette version avec votre propre version actuelle ([ver][2]) pour déterminer si cette fonction est disponible dans votre version particulière. Si ce n’est pas le cas, essayez de rechercher les versions archivées de la documentation pour trouver une alternative appropriée dans votre version.

Vous n’avez pas cette boîte à outils ! #

L’installation de base MATLAB a un grand nombre de fonctions ; cependant, des fonctionnalités plus spécialisées sont regroupées dans des boîtes à outils et vendues séparément par Mathworks. La documentation de toutes les boîtes à outils est visible, que vous ayez la boîte à outils ou non, alors assurez-vous de vérifier si vous avez la boîte à outils appropriée.

Pour vérifier à quelle boîte à outils appartient une fonction donnée, regardez en haut à gauche de la documentation en ligne pour voir si une boîte à outils spécifique est mentionnée.

[![entrez la description de l’image ici][3]][3]

Vous pouvez ensuite déterminer quelles boîtes à outils votre version de MATLAB a installées en exécutant la commande [ver][2] qui imprimera une liste de toutes les boîtes à outils installées.

Si cette boîte à outils n’est pas installée et que vous souhaitez utiliser la fonction, vous devrez acheter une licence pour cette boîte à outils particulière auprès de The Mathworks.

MATLAB ne peut pas localiser la fonction #

Si MATLAB ne trouve toujours pas votre fonction, il doit s’agir d’une fonction définie par l’utilisateur. Il est possible qu’il réside dans un autre répertoire et ce répertoire doit être [ajouté au chemin de recherche] (http://www.mathworks.com/help/matlab/matlab_env/add-remove-or-reorder-folders-on- the-search-path.html) pour que votre code s’exécute. Vous pouvez vérifier si MATLAB peut localiser votre fonction en utilisant [which][4] qui devrait renvoyer le chemin vers le fichier source.

[1] : http://i.stack.imgur.com/W1ZaW.png [2] : http://www.mathworks.com/help/matlab/ref/ver.html [3] : http://i.stack.imgur.com/JyvMG.png [4] : http://www.mathworks.com/help/matlab/ref/which.html

L’utilisation de “i” ou “j” comme unité imaginaire, indices de boucle ou variable commune. #

Recommandation #

Parce que les symboles i et j peuvent représenter des choses très différentes dans MATLAB, leur utilisation en tant qu’indices de boucle a divisé la communauté des utilisateurs MATLAB depuis des lustres. Bien que certaines raisons de performances historiques puissent faire pencher la balance d’un côté, ce n’est plus le cas et maintenant le choix repose entièrement sur vous et les pratiques de codage que vous choisissez de suivre.

Les recommandations officielles actuelles de Mathworks sont :

  • Puisque i est une fonction, elle peut être remplacée et utilisée comme variable. Cependant, il est préférable d’éviter d’utiliser i et j pour les noms de variables si vous avez l’intention de les utiliser en arithmétique complexe.
  • Pour plus de rapidité et de robustesse en arithmétique complexe, utilisez 1i et 1j au lieu de i et j.

Défaut #

Dans MATLAB, par défaut, les lettres i et [j](http://uk.mathworks.com/help /matlab/ref/j.html) sont des noms de fonction intégrés, qui font tous deux référence à l’unité imaginaire dans le domaine complexe.

Donc par défaut i = j = sqrt(-1).

>> i
ans =
   0.0000 + 1.0000i
>> j
ans =
   0.0000 + 1.0000i

et comme il faut s’y attendre :

>> i^2
ans =
    -1

Les utiliser comme variable (pour les indices de boucle ou autre variable)

MATLAB permet d’utiliser le nom de la fonction intégrée comme variable standard. Dans ce cas, le symbole utilisé ne pointera plus vers la fonction intégrée mais vers votre propre variable définie par l’utilisateur. Cette pratique, cependant, n’est généralement pas recommandée car elle peut entraîner une confusion, un débogage et une maintenance difficiles (voir un autre exemple [do-not-name-a-variable-with-an-existing-function-name](http:// stackoverflow.com/documentation/matlab/973/common-mistakes-and-errors/3168/do-not-name-a-variable-with-an-existing-function-name)).

Si vous êtes ultra pédant sur le respect des conventions et bonnes pratiques, vous éviterez de les utiliser comme indices de boucle dans ce langage. Cependant, il est autorisé par le compilateur et parfaitement fonctionnel, vous pouvez donc également choisir de conserver les anciennes habitudes et de les utiliser comme itérateurs de boucle.

>> A = nan(2,3);
>> for i=1:2        % perfectly legal loop construction
       for j = 1:3
        A(i, j) = 10 * i + j;
       end
   end

Notez que les indices de boucle ne sortent pas de la portée à la fin de la boucle, ils conservent donc leur nouvelle valeur.

>> [ i ; j ]
ans =
     2
     3

Dans le cas où vous les utilisez comme variables, assurez-vous qu’elles sont initialisées avant de les utiliser. Dans la boucle ci-dessus, MATLAB les initialise automatiquement lorsqu’il prépare la boucle, mais s’ils ne sont pas correctement initialisés, vous pouvez rapidement voir que vous pouvez introduire par inadvertance des nombres “complexes” dans votre résultat.

Si plus tard, vous devez annuler l’ombrage de la fonction intégrée (=par exemple, vous voulez que i et j représentent à nouveau l’unité imaginaire), vous pouvez effacer les variables :

>> clear i j

Vous comprenez maintenant la réserve de Mathworks quant à leur utilisation comme indices de boucle si vous avez l’intention de les utiliser en arithmétique complexe. Votre code serait truffé d’initialisations de variables et de commandes “clear”, le meilleur moyen de confondre le programmeur le plus sérieux (yes you there!…) et d’accidents de programme en attente de se produire.

Si aucune arithmétique complexe n’est attendue, l’utilisation de i et j est parfaitement fonctionnelle et il n’y a pas de pénalité de performance.

En les utilisant comme unité imaginaire : #

Si votre code doit traiter des nombres « complexes », alors « i » et « j » seront certainement utiles. Cependant, dans un souci de désambiguïsation et même pour les performances, il est recommandé d’utiliser la forme complète au lieu de la syntaxe abrégée. La forme complète est 1i (ou 1j).

>> [ i ; j ; 1i ; 1j]
ans =
   0.0000 + 1.0000i
   0.0000 + 1.0000i
   0.0000 + 1.0000i
   0.0000 + 1.0000i

Ils représentent la même valeur sqrt(-1), mais la forme la plus récente :

  • est plus explicite, de manière sémantique.
  • est plus maintenable (quelqu’un qui regardera votre code plus tard ne le verra pas lire le code pour savoir si i ou j était une variable ou l’unité imaginaire).
  • est plus rapide (source : Mathworks).

Notez que la syntaxe complète “1i” est valide avec n’importe quel nombre précédant le symbole :

>> a = 3 + 7.8j
a =
   3.0000 + 7.8000i

C’est la seule fonction que vous pouvez coller avec un nombre sans opérateur entre eux.

Pièges #

Bien que leur utilisation en tant que unité imaginaire OU variable soit parfaitement légale, voici juste un petit exemple de la confusion que cela pourrait devenir si les deux utilisations se mélangeaient :

Remplaçons i et en faisons une variable :

>> i=3
i =
     3

Maintenant i est une variable (contenant la valeur 3), mais nous ne remplaçons que la notation abrégée de l’unité imaginaire, la forme complète est toujours interprétée correctement :

>> 3i
ans =
   0.0000 + 3.0000i

Ce qui nous permet désormais de construire les formulations les plus obscures. Je vous laisse évaluer la lisibilité de toutes les constructions suivantes :

>> [ i ; 3i ; 3*i ; i+3i ; i+3*i ]
ans =
   3.0000 + 0.0000i
   0.0000 + 3.0000i
   9.0000 + 0.0000i
   3.0000 + 3.0000i
  12.0000 + 0.0000i

Comme vous pouvez le voir, chaque valeur du tableau ci-dessus renvoie un résultat différent. Bien que chaque résultat soit valide (à condition que ce soit l’intention initiale), la plupart d’entre vous admettront que ce serait un vrai cauchemar de lire un code truffé de telles constructions.

Pas assez d’arguments d’entrée #

Les développeurs MATLAB débutants utilisent souvent l’éditeur de MATLAB pour écrire et modifier du code, en particulier des fonctions personnalisées avec des entrées et des sorties. Il y a un bouton Exécuter en haut qui est disponible dans les versions récentes de MATLAB :

[![entrez la description de l’image ici][1]][1]

Une fois que le développeur a terminé avec le code, il est souvent tenté d’appuyer sur le bouton Exécuter. Pour certaines fonctions, cela fonctionnera bien, mais pour d’autres, elles recevront une erreur “Pas assez d’arguments d’entrée” et ne comprendront pas pourquoi l’erreur se produit.

La raison pour laquelle cette erreur peut ne pas se produire est que vous avez écrit un script MATLAB ou une fonction qui ne prend aucun argument d’entrée. L’utilisation du bouton Exécuter exécutera un script de test ou exécutera une fonction en supposant qu’il n’y a pas d’arguments d’entrée. Si votre fonction nécessite des arguments d’entrée, l’erreur “Pas assez d’arguments d’entrée” se produira car vous avez écrit une fonction qui attend des entrées à l’intérieur de la fonction. Par conséquent, vous ne pouvez pas vous attendre à ce que la fonction s’exécute en appuyant simplement sur le bouton Exécuter.

Pour illustrer ce problème, supposons que nous ayons une fonction “mult” qui multiplie simplement deux matrices ensemble :

function C = mult(A, B)
    C = A * B;
end

Dans les versions récentes de MATLAB, si vous avez écrit cette fonction et appuyé sur le bouton Exécuter, cela vous donnera l’erreur attendue :

>> mult
Not enough input arguments.

Error in mult (line 2)
    C = A * B;

Il existe deux façons de résoudre ce problème :

Méthode #1 - Via l’invite de commande #

Créez simplement les entrées dont vous avez besoin dans l’invite de commande, puis exécutez la fonction en utilisant les entrées que vous avez créées :

A = rand(5,5);
B = rand(5,5);
C = mult(A,B);

Méthode #2 - De manière interactive via l’éditeur #

Sous le bouton * Exécuter *, il y a une flèche noire foncée. Si vous cliquez sur cette flèche, vous pouvez spécifier les variables que vous souhaitez obtenir de l’espace de travail MATLAB en tapant la manière dont vous souhaitez appeler la fonction exactement comme vous l’avez vu dans la méthode n°1. Assurez-vous que les variables que vous spécifiez dans la fonction existent dans l’espace de travail MATLAB :

Utilisation de length pour les tableaux multidimensionnels #

Une erreur courante des codeurs MATLAB est d’utiliser la fonction “longueur” pour les matrices (par opposition aux vecteurs, pour lesquels elle est destinée). La fonction length, comme mentionné dans sa documentation, “renvoie la longueur de la plus grande dimension du tableau” de l’entrée.

Pour les vecteurs, la valeur de retour de length a deux significations différentes :

  1. Le nombre total d’éléments dans le vecteur.
  2. La plus grande dimension du vecteur.

Contrairement aux vecteurs, les valeurs ci-dessus ne seraient pas égales pour les tableaux de plus d’une dimension non singleton (c’est-à-dire dont la taille est supérieure à “1”). C’est pourquoi l’utilisation de length pour les matrices est ambiguë. Au lieu de cela, l’utilisation de l’une des fonctions suivantes est encouragée, même lorsque vous travaillez avec des vecteurs, pour que l’intention du code soit parfaitement claire :

  1. size(A) - renvoie un vecteur ligne dont les éléments contiennent la quantité d’éléments le long de la dimension correspondante de A .
  2. numel(A) - renvoie le nombre d’éléments dans A. Équivalent à prod(taille(A)).
  3. ndims(A) - renvoie le nombre de dimensions dans le tableau A. Équivalent à numel(taille(A)).

Ceci est particulièrement important lors de l’écriture de fonctions de bibliothèque “à l’épreuve du futur”, vectorisées, dont les entrées ne sont pas connues à l’avance et peuvent avoir différentes tailles et formes.

Attention aux changements de taille de tableau #

Certaines opérations courantes dans MATLAB, telles que la différenciation ou l’intégration, génèrent des résultats contenant une quantité d’éléments différente de celle des données d’entrée. Ce fait peut facilement être négligé, ce qui entraînerait généralement des erreurs telles que “Les dimensions de la matrice doivent correspondre”. Considérez l’exemple suivant :

t = 0:0.1:10;        % Declaring a time vector
y = sin(t);          % Declaring a function

dy_dt = diff(y);     % calculates dy/dt for y = sin(t)

Disons que nous voulons tracer ces résultats. Nous examinons les tailles de tableau et voyons :

size(y) is 1x101
size(t) is 1x101

Mais:

size(dy_dt) is 1x100

Le tableau est un élément plus court !

Imaginez maintenant que vous ayez des données de mesure de positions dans le temps et que vous vouliez calculer jerk(t), vous obtiendrez un tableau de 3 éléments de moins que le tableau temporel (car le jerk est la position différenciée 3 fois).

vel = diff(y);       % calculates velocity vel=dy/dt for y = sin(t)  size(vel)=1x100
acc = diff(vel);     % calculates acceleration acc=d(vel)/dt         size(acc)=1x99
jerk = diff(acc);    % calculates jerk jerk=d(acc)/dt                size(jerk)=1x98

Et puis des opérations comme :

x = jerk .* t;          % multiplies jerk and t element wise

renvoyer des erreurs, car les dimensions de la matrice ne concordent pas.

Pour calculer des opérations comme ci-dessus, vous devez ajuster la plus grande taille de tableau pour qu’elle corresponde à la plus petite. Vous pouvez également exécuter une régression (“polyfit”) avec vos données pour obtenir un polynôme pour vos données.

## Erreurs d’incohérence des dimensions

Les erreurs de incompatibilité de dimension apparaissent généralement lorsque :

  • Ne pas prêter attention à la forme des variables renvoyées par les appels de fonction/méthode. Dans de nombreuses fonctions MATLAB intégrées, les matrices sont converties en vecteurs pour accélérer les calculs, et la variable renvoyée peut toujours être un vecteur plutôt que la matrice attendue. C’est également un scénario courant lorsque le [masquage logique] [1] est impliqué.
  • Utilisation de tailles de tableau incompatibles lors de l’appel de [extension de tableau implicite][2].
