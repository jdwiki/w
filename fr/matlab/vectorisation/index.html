Utilisation de bsxfun #

Assez souvent, la raison pour laquelle le code a été écrit dans une boucle for est de calculer des valeurs à partir de valeurs `` proches ‘’. La fonction bsxfun peut souvent être utilisée pour le faire de manière plus succincte.

Par exemple, supposons que vous souhaitiez effectuer une opération par colonne sur la matrice “B”, en soustrayant la moyenne de chaque colonne :

B = round(randn(5)*10);                  % Generate random data 
A = zeros(size(B));                      % Preallocate array
for col = 1:size(B,2);                    % Loop over columns
    A(:,col) = B(:,col) - mean(B(:,col));   % Subtract means 
end

Cette méthode est inefficace si B est grand, souvent parce que MATLAB doit déplacer le contenu des variables dans la mémoire. En utilisant bsxfun, on peut faire le même travail proprement et facilement en une seule ligne :

A = bsxfun(@minus, B, mean(B));

Ici, @minus est un [descripteur de fonction][1] de l’opérateur [minus][2] (-) et sera appliqué entre les éléments des deux matrices B et mean(B) . D’autres poignées de fonction, même celles définies par l’utilisateur, sont également possibles.

Ensuite, supposons que vous vouliez ajouter le vecteur ligne ‘v’ à chaque ligne de la matrice ‘A’ :

v = [1,  2,  3];

A = [8,  1,  6
     3,  5,  7
     4,  9,  2];

L’approche naïve consiste à utiliser une boucle (do not do this):

B = zeros(3);
for row = 1:3
    B(row,:) = A(row,:) + v;
end

Une autre option serait de répliquer v avec repmat (do ne pas le faire non plus):

>> v = repmat(v,3,1)
v =
     1     2     3
     1     2     3
     1     2     3

>> B = A + v;

Utilisez plutôt bsxfun pour cette tâche :

>> B = bsxfun(@plus, A, v);
B =
     9     3     9
     4     7    10
     5    11     5

Syntaxe #

bsxfun(@fun, A, B)

où @fun est l’une des fonctions supportées et les deux tableaux A et B respectent les deux conditions ci-dessous .

Le nom bsxfun aide à comprendre le fonctionnement de la fonction et signifie Binary FUNction with Singleton eXpansion. Autrement dit, si :

  1. deux tableaux partagent les mêmes dimensions sauf un
  2. et la dimension discordante est un singleton (c’est-à-dire a une taille de ‘1’) dans l’un des deux tableaux

alors le tableau avec la dimension singleton sera développé pour correspondre à la dimension de l’autre tableau. Après l’expansion, une fonction binaire est appliquée élément par élément sur les deux tableaux.

Par exemple, supposons que A soit un tableau M-by-N-byK et B est un tableau M-by-N. Premièrement, leurs deux premières dimensions ont des tailles correspondantes. Deuxièmement, ‘A’ a ‘K’ couches alors que ‘B’ n’a implicitement que ‘1’, c’est donc un singleton. Toutes les conditions sont remplies et “B” sera reproduit pour correspondre à la 3e dimension de “A”.

Dans d’autres langages, cela est communément appelé broadcasting et se produit automatiquement en Python (numpy) et Octave.

La fonction, @fun, doit être une fonction binaire, ce qui signifie qu’elle doit prendre exactement deux entrées.

Remarques #

En interne, bsxfun ne réplique pas le tableau et exécute une boucle efficace.

[1] : http://ch.mathworks.com/help/matlab/matlab_prog/creating-a-function-handle.html [2] : http://ch.mathworks.com/help/matlab/ref/minus.html

Extension de tableau implicite (diffusion) [R2016b] #

MATLAB R2016b comportait une généralisation de son mécanisme d’expansion scalaire[1][1],[2][2], afin de prendre également en charge certaines opérations élément par élément entre des tableaux de tailles différentes, comme tant que leurs dimensions sont compatibles. Les opérateurs qui prennent en charge l’expansion implicite sont[1][3] :

  • Opérateurs arithmétiques élément par élément : +, -, .*, .^, ./, .\.
  • Opérateurs relationnels : <, <=, >, >=, ==, ~=.
  • Opérateurs logiques : &, |, xor.
  • Fonctions binaires : bitand, bitor, bitxor.
  • Fonctions mathématiques élémentaires : max, min, mod, rem, hypot, atan2, atan2d.

Les opérations binaires susmentionnées sont autorisées entre les tableaux, tant qu’ils ont des “tailles compatibles”. Les tailles sont considérées comme “compatibles” lorsque chaque dimension d’un tableau est soit exactement égale à la même dimension dans l’autre tableau, soit égale à “1”. Notez que les dimensions singleton de fin (c’est-à-dire de taille 1) sont omises par MATLAB, même s’il y en a théoriquement une quantité infinie. En d’autres termes, les dimensions qui apparaissent dans un tableau et n’apparaissent pas dans l’autre sont implicitement adaptées à l’expansion automatique.

Par exemple, dans les versions MATLAB avant R2016b, cela se produirait :

>> magic(3) + (1:3)
Error using  + 
Matrix dimensions must agree.

Alors que à partir de R2016b l’opération précédente réussira :

>> magic(3) + (1:3)
ans =

     9     3     9
     4     7    10
     5    11     5

Exemples de tailles compatibles : #

Descriptif 1er taille de tableau Taille du 2ème tableau Taille du résultat
Vecteur et scalaire [3x1] [1x1] [3x1]
Vecteurs ligne et colonne [1x3] [2x1] [2x3]
Vecteur et matrice 2D [1x3] [5x3] [5x3]
Matrices N-D et K-D [1x3x3] [5x3x1x4x2] [5x3x3x4x2]

Exemples de tailles incompatibles : #

Descriptif 1er taille de tableau Taille du 2ème tableau Solution de contournement possible
Vecteurs où une dimension est un multiple de la même dimension dans l’autre tableau. [1x2] [1x8] “transposer”
Tableaux dont les dimensions sont des multiples les unes des autres. [2x2] [8x8] repmat, remodeler
Tableaux N-D qui ont la bonne quantité de dimensions singleton mais qui sont dans le mauvais ordre (#1). [2x3x4] [2x4x3] ‘permuter’
Tableaux N-D qui ont la bonne quantité de dimensions singleton mais qui sont dans le mauvais ordre (#2). [2x3x4x5] [5x2] ‘permuter’

IMPORTANT :
Le code reposant sur cette convention n’est PAS rétrocompatible avec toutes les anciennes versions de MATLAB. Par conséquent, l’invocation explicite de bsxfun[1][4],[2][5] (qui produit le même effet) doit être utilisée si le code doit être exécuté sur des versions plus anciennes de MATLAB. Si un tel problème n’existe pas, [MATLAB R2016 release notes][3] encourage les utilisateurs à passer de bsxfun :

Par rapport à l’utilisation de bsxfun, l’expansion implicite offre une vitesse d’exécution plus rapide, une meilleure utilisation de la mémoire et une meilleure lisibilité du code.

Lecture connexe :

  • Documentation MATLAB sur “[Tailles de tableaux compatibles pour les opérations de base][6]”.
  • Diffusion de NumPy[1][7],[2][8].
  • Une comparaison entre la [vitesse de calcul à l’aide de bsxfun par rapport à l’expansion de tableau implicite][9].

Opérations élément par élément #

MATLAB prend en charge (et encourage) les opérations vectorisées sur les vecteurs et les matrices. Par exemple, supposons que nous ayons A et B, deux matrices n par m et que nous voulons que C soit le produit élément par élément des éléments correspondants (c’est-à-dire, C(i, j) = A(i,j)*B(i,j)).

La manière non vectorisée, utilisant des boucles imbriquées est la suivante :

C = zeros(n,m);
for ii=1:n
    for jj=1:m
        C(ii,jj) = A(ii,jj)*B(ii,jj);
    end
end

Cependant, la manière vectorisée de le faire consiste à utiliser l’opérateur élément par élément .* :

C = A.*B;
  • Pour plus d’informations sur la multiplication élément par élément dans MATLAB, consultez la documentation de [times][1].
  • Pour plus d’informations sur la différence entre les opérations sur les tableaux et les matrices, consultez [Array vs. Matrix Operations][2] dans la documentation MATLAB.

[1] : http://www.mathworks.com/help/matlab/ref/times.html [2] : http://ch.mathworks.com/help/matlab/matlab_prog/array-vs-matrix-operations.html

Masquage logique #

MATLAB prend en charge l’utilisation du masquage logique afin d’effectuer une sélection sur une matrice sans utiliser de boucles for ou d’instructions if.

Un masque logique est défini comme une matrice composée uniquement de ‘1’ et ‘0’.

Par exemple:

mask = [1 0 0; 0 1 0; 0 0 1];

est une matrice logique représentant la matrice identité.

Nous pouvons générer un masque logique en utilisant un prédicat pour interroger une matrice.

A = [1 2 3; 4 5 6; 7 8 9];
B = A > 4;

Nous créons d’abord une matrice 3x3, “A”, contenant les nombres de 1 à 9. Nous interrogeons ensuite “A” pour les valeurs supérieures à 4 et stockons le résultat dans une nouvelle matrice appelée “B”.

‘B’ est une matrice logique de la forme :

B = [0 0 0
     0 1 1
     1 1 1]

Ou 1 lorsque le prédicat A > 4 était vrai. Et ‘0’ quand c’était faux.

Nous pouvons utiliser des matrices logiques pour accéder aux éléments d’une matrice. Si une matrice logique est utilisée pour sélectionner des éléments, les indices où un “1” apparaît dans la matrice logique seront sélectionnés dans la matrice à partir de laquelle vous sélectionnez.

En utilisant le même “B” ci-dessus, nous pourrions faire ce qui suit :

C = [0 0 0; 0 0 0; 0 0 0];
C(B) = 5;

Cela sélectionnerait tous les éléments de ‘C’ où ‘B’ a un ‘1’ dans cet index. Ces indices dans ‘C’ sont alors mis à ‘5’.

Notre C ressemble maintenant à :

C = [0 0 0
     0 5 5
     5 5 5]

Nous pouvons réduire les blocs de code compliqués contenant if et for en utilisant des masques logiques.

Prenez le code non vectorisé :

A = [1 3 5; 7 9 11; 11 9 7];
for j = 1:length(A)
  if A(j) > 5
    A(j) = A(j) - 2;
  end
end

Cela peut être raccourci en utilisant le masquage logique au code suivant :

A = [1 3 5; 7 9 11; 11 9 7];
B = A > 5;
A(B) = A(B) - 2;

Ou encore plus court :

A = [1 3 5; 7 9 11; 11 9 7];
A(A > 5) = A(A > 5) - 2;

Somme, moyenne, prod & co #

Soit un vecteur aléatoire

v = rand(10,1);

si vous voulez la somme de ses éléments, n’utilisez ** PAS ** de boucle

s = 0;
for ii = 1:10
    s = s + v(ii);
end

mais utilisez la capacité vectorisée de la fonction sum()

s = sum(v);

Des fonctions comme sum(), [mean()](http://uk.mathworks.com/help /matlab/ref/mean.html), prod() et d’autres, ont la capacité d’opérer directement le long des lignes, colonnes ou d’autres dimensions.

Par exemple, étant donné une matrice aléatoire

A = rand(10,10);

la moyenne pour chaque colonne est

m = mean(A,1);

la moyenne pour chaque ligne est

m = mean(A,2)

Toutes les fonctions ci-dessus ne fonctionnent que sur une dimension, mais que se passe-t-il si vous voulez additionner toute la matrice ? Vous pouvez utiliser :

s = sum(sum(A))

Mais que se passe-t-il si vous avez un ND-array ? appliquer sum sur sum sur sum… ne semble pas être la meilleure option, utilisez plutôt l’opérateur : pour vectoriser votre tableau :

s = sum(A(:))

et cela se traduira par un nombre qui est la somme de tout votre tableau, peu importe le nombre de dimensions qu’il a.

Obtenir la valeur d’une fonction de deux arguments ou plus #

Dans de nombreuses applications, il est nécessaire de calculer la fonction de deux arguments ou plus.

Traditionnellement, nous utilisons des boucles for. Par exemple, si nous devons calculer f = exp(-x^2-y^2) (ne l’utilisez pas si vous avez besoin de simulations rapides) :

% code1
x = -1.2:0.2:1.4;
y = -2:0.25:3;
for nx=1:lenght(x)
   for ny=1:lenght(y)
      f(nx,ny) = exp(-x(nx)^2-y(ny)^2);
   end
end

Mais la version vectorisée est plus élégante et plus rapide :

% code2
[x,y] = ndgrid(-1.2:0.2:1.4, -2:0.25:3);
f = exp(-x.^2-y.^2);

que nous pouvons le visualiser:

surf(x,y,f)

Remarque1 - Grilles : généralement, le stockage de la matrice est organisé ligne par ligne. Mais dans le MATLAB, c’est le stockage colonne par colonne comme dans le FORTRAN. Ainsi, il existe deux fonctions similaires “ndgrid” et “meshgrid” dans MATLAB pour implémenter les deux modèles susmentionnés. Pour visualiser la fonction dans le cas de meshgrid, nous pouvons utiliser :

surf(y,x,f)

Note2 - Consommation de mémoire : Soit la taille de x ou y est de 1000. Ainsi, nous devons stocker les éléments 1000*1000+2*1000 ~ 1e6 pour les *code1 non vectorisés *. Mais nous avons besoin de 3*(1000*1000) = 3e6 éléments dans le cas de code2 vectorisé. Dans le cas 3D (que z ait la même taille que x ou y), la consommation de mémoire augmente considérablement : 4*(1000*1000*1000) (~32 Go pour les doubles) dans le cas du vectorisé code2 vs ~1000*1000*1000 (seulement ~8GB) dans le cas de code1. Ainsi, nous devons choisir soit la mémoire, soit la vitesse.
