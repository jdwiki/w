  • L’utilisation de fonctionnalités non documentées est considérée comme une pratique risquée1, car ces fonctionnalités peuvent changer sans préavis ou simplement travailler différemment sur différentes versions de MATLAB. Pour cette raison, il est conseillé d’utiliser des techniques de programmation défensive telles que l’inclusion de morceaux de code non documentés dans des blocs “try/catch” avec des solutions de remplacement documentées.

Tracés linéaires 2D à code couleur avec données de couleur en troisième dimension #

Dans les versions de MATLAB antérieures à R2014b, utilisant l’ancien moteur graphique HG1, il n’était pas évident de créer des [tracés de lignes 2D à code couleur][1]. Avec la sortie du nouveau moteur graphique HG2 est apparue une nouvelle [fonctionnalité non documentée introduite par Yair Altman][3] :

n = 100;
x = linspace(-10,10,n); y = x.^2;
p = plot(x,y,'r', 'LineWidth',5);

% modified jet-colormap
cd = [uint8(jet(n)*255) uint8(ones(n,1))].';

drawnow
set(p.Edge, 'ColorBinding','interpolated', 'ColorData',cd)

Marqueurs semi-transparents dans les lignes et les nuages ​​de points #

Depuis ** Matlab R2014b , il est facilement possible d’obtenir des marqueurs semi-transparents pour les tracés de ligne et de dispersion en utilisant [ fonctionnalités non documentées introduites par Yair Altman **] (http://undocumentedmatlab.com/blog/plot-markers- transparence et dégradé de couleurs).

L’idée de base est d’obtenir la poignée cachée des marqueurs et d’appliquer une valeur < 1 pour la dernière valeur dans EdgeColorData pour obtenir la transparence souhaitée.

Ici, nous allons pour scatter:

%// example data
x = linspace(0,3*pi,200);
y = cos(x) + rand(1,200);

%// plot scatter, get handle
h = scatter(x,y); 
drawnow; %// important

%// get marker handle
hMarkers = h.MarkerHandle; 

%// get current edge and face color
edgeColor = hMarkers.EdgeColorData
faceColor = hMarkers.FaceColorData

%// set face color to the same as edge color
faceColor = edgeColor;

%// opacity
opa = 0.3;

%// set marker edge and face color
hMarkers.EdgeColorData = uint8( [edgeColor(1:3); 255*opa] ); 
hMarkers.FaceColorData = uint8( [faceColor(1:3); 255*opa] );

[![entrez la description de l’image ici][1]][1]

et pour une ligne plot

%// example data
x = linspace(0,3*pi,200);
y1 = cos(x);
y2 = sin(x);

%// plot scatter, get handle
h1 = plot(x,y1,'o-','MarkerSize',15); hold on
h2 = plot(x,y2,'o-','MarkerSize',15); 
drawnow; %// important

%// get marker handle
h1Markers = h1.MarkerHandle; 
h2Markers = h2.MarkerHandle; 

%// get current edge and face color
edgeColor1 = h1Markers.EdgeColorData;
edgeColor2 = h2Markers.EdgeColorData;

%// set face color to the same as edge color
faceColor1 = edgeColor1;
faceColor2 = edgeColor2;

%// opacity
opa = 0.3;

%// set marker edge and face color
h1Markers.EdgeColorData = uint8( [edgeColor1(1:3); 255*opa] ); 
h1Markers.FaceColorData = uint8( [faceColor1(1:3); 255*opa] ); 
h2Markers.EdgeColorData = uint8( [edgeColor2(1:3); 255*opa] ); 
h2Markers.FaceColorData = uint8( [faceColor2(1:3); 255*opa] );

[![entrez la description de l’image ici][2]][2]

Les poignées de marqueur, qui sont utilisées pour la manipulation, sont créées avec la figure. La commande drawnow assure la création de la figure avant l’appel des commandes suivantes et évite les erreurs en cas de retard.

Fonctions d’assistance compatibles C++ #

L’utilisation de Matlab Coder nie parfois l’utilisation de certaines fonctions très courantes, si elles ne sont pas compatibles avec C++. Il existe relativement souvent des fonctions d’assistance non documentées, qui peuvent être utilisées en remplacement.

Voici une liste complète des fonctions prises en charge..

Et suivant une collection d’alternatives, pour les fonctions non prises en charge :

Les fonctions sprintf et sprintfc sont assez similaires, la première renvoie un tableau de caractères, la seconde une chaîne de cellules :

 str = sprintf('%i',x)   % returns '5' for x = 5
 str = sprintfc('%i',x)  % returns {'5'} for x = 5

Cependant, sprintfc est compatible avec C++ pris en charge par Matlab Coder, et sprintf ne l’est pas.

Tracés de contour - Personnalisez les étiquettes de texte #

Lors de l’affichage des étiquettes sur les contours, Matlab ne vous permet pas de contrôler le format des nombres, par exemple pour passer à la notation scientifique.

Les objets texte individuels sont des objets texte normaux, mais la façon dont vous les obtenez n’est pas documentée. Vous y accédez à partir de la propriété TextPrims de la poignée de contour.

  figure
  [X,Y]=meshgrid(0:100,0:100);
  Z=(X+Y.^2)*1e10;
  [C,h]=contour(X,Y,Z);
  h.ShowText='on';
  drawnow();
  txt = get(h,'TextPrims');
  v = str2double(get(txt,'String'));
  for ii=1:length(v)
    set(txt(ii),'String',sprintf('%0.3e',v(ii)))
  end

Remarque : que vous devez ajouter une commande “dessinée” pour forcer Matlab à dessiner les contours, le nombre et l’emplacement des objets txt ne sont déterminés que lorsque les contours sont réellement dessinés, de sorte que les objets texte ne sont créés qu’à ce moment-là.

Le fait que les objets txt soient créés lorsque les contours sont dessinés signifie qu’ils sont recalculés à chaque fois que le tracé est redessiné (par exemple le redimensionnement de la figure). Pour gérer cela, vous devez écouter l’événement non documenté MarkedClean :

function customiseContour
  figure
  [X,Y]=meshgrid(0:100,0:100);
  Z=(X+Y.^2)*1e10;
  [C,h]=contour(X,Y,Z);
  h.ShowText='on';
  % add a listener and call your new format function
  addlistener(h,'MarkedClean',@(a,b)ReFormatText(a))
end
function ReFormatText(h)
  % get all the text items from the contour
  t = get(h,'TextPrims');
  for ii=1:length(t)
    % get the current value (Matlab changes this back when it 
    %   redraws the plot)
    v = str2double(get(t(ii),'String'));
    % Update with the format you want - scientific for example
    set(t(ii),'String',sprintf('%0.3e',v));
  end
end

Ajout/ajout d’entrées à une légende existante #

Les légendes existantes peuvent être difficiles à gérer. Par exemple, si votre tracé comporte deux lignes, mais qu’une seule d’entre elles a une entrée de légende et que cela devrait rester ainsi, l’ajout d’une troisième ligne avec une entrée de légende peut être difficile. Exemple:

figure
hold on
fplot(@sin)
fplot(@cos)
legend sin  % Add only a legend entry for sin
hTan = fplot(@tan);  % Make sure to get the handle, hTan, to the graphics object you want to add to the legend

Maintenant, pour ajouter une entrée de légende pour tan, mais pas pour cos, aucune des lignes suivantes ne fera l’affaire ; ils échouent tous d’une manière ou d’une autre :

legend tangent  % Replaces current legend -> fail
legend -DynamicLegend  % Undocumented feature, adds 'cos', which shouldn't be added -> fail
legend sine tangent  % Sets cos DisplayName to 'tangent' -> fail
legend sine '' tangent  % Sets cos DisplayName, albeit empty -> fail
legend(f)

Heureusement, une propriété de légende non documentée appelée “PlotChildren” garde une trace des enfants de la figure parent1. Ainsi, la voie à suivre consiste à définir explicitement les enfants de la légende via sa propriété PlotChildren comme suit :

hTan.DisplayName = 'tangent';  % Set the graphics object's display name
l = legend;
l.PlotChildren(end + 1) = hTan;  % Append the graphics handle to legend's plot children

La légende se met à jour automatiquement si un objet est ajouté ou supprimé de sa propriété PlotChildren.

1 En effet : chiffre. Vous pouvez ajouter l’enfant de n’importe quelle figure avec la propriété DisplayName à n’importe quelle légende de la figure, par ex. d’une sous-parcelle différente. C’est parce qu’une légende en elle-même est essentiellement un objet d’axes.

Testé sur MATLAB R2016b

Gigue du nuage de points #

La fonction scatter a deux propriétés non documentées 'jitter' et 'jitterAmount' qui permettent de jitter les données sur l’axe des abscisses uniquement. Cela remonte à Matlab 7.1 (2005), et peut-être plus tôt.

Pour activer cette fonctionnalité, définissez la propriété 'jitter' sur 'on' et définissez la propriété 'jitterAmount' sur la valeur absolue souhaitée (la valeur par défaut est 0.2).

Ceci est très utile lorsque nous voulons visualiser des données qui se chevauchent, par exemple :

scatter(ones(1,10), ones(1,10), 'jitter', 'on', 'jitterAmount', 0.5);
