Préférences de pliage de code #

Il est possible de modifier la préférence de pliage de code en fonction de vos besoins. Ainsi, le pliage de code peut être activé/désactivé pour des constructions spécifiques (ex : if block, for loop, Sections …).

Pour modifier les préférences de pliage, allez dans Préférences -> Code Folding :

[![entrez la description de l’image ici][1]][1]

Ensuite, vous pouvez choisir quelle partie du code peut être pliée.

Des informations:

  • Notez que vous pouvez également développer ou réduire tout le code d’un fichier en plaçant votre curseur n’importe où dans le fichier, cliquez avec le bouton droit de la souris, puis sélectionnez Code Folding > Développer tout ou Code Folding > Fold All dans le menu contextuel.
  • Notez que le pliage est persistant, dans le sens où une partie du code qui a été développée/réduite conservera son statut après la fermeture et la réouverture de Matlab ou du m-file.

Extraire les données de la figure #

À quelques reprises, j’ai eu un chiffre intéressant que j’ai enregistré mais j’ai perdu l’accès à ses données. Cet exemple montre une astuce pour extraire des informations d’une figure.

Les fonctions clés sont [findobj][1] et [get][2]. [findobj][1] renvoie un gestionnaire à un objet en fonction des attributs ou des propriétés de l’objet, tels que Type ou Color, etc. Une fois qu’un objet de ligne a été trouvé, [get][2] peut renvoyer n’importe quelle valeur détenus par les propriétés. Il s’avère que les objets Line contiennent toutes les données dans les propriétés suivantes : XData, YData et ZData ; le dernier est généralement 0 sauf si une figure contient un tracé 3D.

Le code suivant crée un exemple de figure qui montre deux lignes une fonction sin et un seuil et une légende

t = (0:1/10:1-1/10)';
y = sin(2*pi*t);
plot(t,y);
hold on;
plot([0 0.9],[0 0], 'k-');
hold off;
legend({'sin' 'threshold'});

La première utilisation de [findobj][1] renvoie deux gestionnaires aux deux lignes :

findobj(gcf, 'Type', 'Line')
ans = 
  2x1 Line array:

  Line    (threshold)
  Line    (sin)

Pour affiner le résultat, [findobj][1] peut également utiliser une combinaison d’opérateurs logiques -and, -or et de noms de propriété. Par exemple, je peux trouver un objet ligne dont DisplayName est sin et lire ses XData et YData.

lineh = findobj(gcf, 'Type', 'Line', '-and', 'DisplayName', 'sin');
xdata = get(lineh, 'XData');
ydata = get(lineh, 'YData');

et vérifiez si les données sont égales.

isequal(t(:),xdata(:))
ans =
     1
isequal(y(:),ydata(:))
ans =
     1

De même, je peux affiner mes résultats en excluant la ligne noire (seuil):

lineh = findobj(gcf, 'Type', 'Line', '-not', 'Color', 'k');
xdata = get(lineh, 'XData');
ydata = get(lineh, 'YData');

et la dernière vérification confirme que les données extraites de cette figure sont les mêmes :

isequal(t(:),xdata(:))
ans =
     1
isequal(y(:),ydata(:))
ans =
     1

[1] : http://uk.mathworks.com/help/matlab/ref/findobj.html?requestedDomain=www.mathworks.com [2] : http://uk.mathworks.com/help/matlab/ref/com.get.html

Programmation fonctionnelle à l’aide de fonctions anonymes #

Les fonctions anonymes peuvent être utilisées pour la programmation fonctionnelle. Le principal problème à résoudre est qu’il n’existe aucun moyen natif d’ancrer une récursivité, mais cela peut toujours être implémenté en une seule ligne :

if_ = @(bool, tf) tf{2-bool}();

Cette fonction accepte une valeur booléenne et un tableau de cellules de deux fonctions. La première de ces fonctions est évaluée si la valeur booléenne est évaluée comme vraie, et la seconde si la valeur booléenne est évaluée comme fausse. On peut facilement écrire la fonction factorielle maintenant :

fac = @(n,f) if_(n>1, {@()n*f(n-1,f), @()1});

Le problème ici est que nous ne pouvons pas invoquer directement un appel récursif, car la fonction n’est pas encore affectée à une variable lorsque le côté droit est évalué. On peut cependant compléter cette étape en écrivant

factorial_ = @(n)fac(n,fac);

Maintenant @(n)fac(n,fac) évalue la fonction factorielle de manière récursive. Une autre façon de le faire dans la programmation fonctionnelle en utilisant un y-combinateur, qui peut également être facilement implémentée :

y_ = @(f)@(n)f(n,f);

Avec cet outil, la fonction factorielle est encore plus courte :

factorial_ = y_(fac);

Ou directement :

factorial_ = y_(@(n,f) if_(n>1, {@()n*f(n-1,f), @()1}));

Enregistrer plusieurs figures dans le même fichier .fig #

En mettant plusieurs poignées de figure dans un tableau graphique, plusieurs figures peuvent être enregistrées dans le même fichier .fig

h(1) = figure;
scatter(rand(1,100),rand(1,100));

h(2) = figure;
scatter(rand(1,100),rand(1,100));

h(3) = figure;
scatter(rand(1,100),rand(1,100));

savefig(h,'ThreeRandomScatterplots.fig');
close(h);

Cela crée 3 nuages ​​de points de données aléatoires, chaque partie du tableau graphique h. Ensuite, le tableau graphique peut être enregistré en utilisant savefig comme avec une figure normale, mais avec la poignée du tableau graphique comme argument supplémentaire.

Une remarque intéressante est que les chiffres auront tendance à rester disposés de la même manière qu’ils ont été enregistrés lorsque vous les ouvrez.

Fonctions utiles qui fonctionnent sur les cellules et les tableaux #

Cet exemple simple fournit une explication sur certaines fonctions que j’ai trouvées extrêmement utiles depuis que j’ai commencé à utiliser MATLAB : cellfun, arrayfun. L’idée est de prendre une variable de classe de tableau ou de cellule, de parcourir tous ses éléments et d’appliquer une fonction dédiée à chaque élément. Une fonction appliquée peut être soit anonyme, ce qui est généralement le cas, soit n’importe quelle fonction normale définie dans un fichier *.m.

Commençons par un problème simple et disons que nous devons trouver une liste de fichiers *.mat en fonction du dossier. Pour cet exemple, créons d’abord des fichiers *.mat dans un dossier courant :

for n=1:10; save(sprintf('mymatfile%d.mat',n)); end

Après l’exécution du code, il devrait y avoir 10 nouveaux fichiers avec l’extension *.mat. Si nous exécutons une commande pour lister tous les fichiers *.mat, tels que :

mydir = dir('*.mat');

nous devrions obtenir un tableau d’éléments d’une structure dir ; MATLAB devrait donner une sortie similaire à celle-ci :

10x1 struct array with fields:
    name
    date
    bytes
    isdir
    datenum

Comme vous pouvez le voir, chaque élément de ce tableau est une structure avec quelques champs. Toutes les informations sont en effet importantes concernant chaque fichier mais à 99% je m’intéresse plutôt aux noms de fichiers et rien d’autre. Pour extraire des informations d’un tableau de structure, j’avais l’habitude de créer une fonction locale qui impliquait de créer des variables temporelles de taille correcte, des boucles, d’extraire un nom de chaque élément et de l’enregistrer dans la variable créée. Un moyen beaucoup plus simple d’obtenir exactement le même résultat consiste à utiliser l’une des fonctions susmentionnées :

mydirlist = arrayfun(@(x) x.name, dir('*.mat'), 'UniformOutput', false)
mydirlist = 
    'mymatfile1.mat'
    'mymatfile10.mat'
    'mymatfile2.mat'
    'mymatfile3.mat'
    'mymatfile4.mat'
    'mymatfile5.mat'
    'mymatfile6.mat'
    'mymatfile7.mat'
    'mymatfile8.mat'
    'mymatfile9.mat'

Comment fonctionne cette fonction ? Il prend généralement deux paramètres : un handle de fonction comme premier paramètre et un tableau. Une fonction opérera alors sur chaque élément d’un tableau donné. Les troisième et quatrième paramètres sont facultatifs mais importants. Si on sait qu’une sortie ne sera pas régulière, il faut l’enregistrer dans la cellule. Cela doit être souligné en définissant false sur UniformOutput. Par défaut, cette fonction tente de renvoyer une sortie régulière telle qu’un vecteur de nombres. Par exemple, extrayons des informations sur la quantité d’espace disque occupée par chaque fichier en octets :

mydirbytes = arrayfun(@(x) x.bytes, dir('*.mat'))
mydirbytes =
       34560
       34560
       34560
       34560
       34560
       34560
       34560
       34560
       34560
       34560

ou kilo-octets :

mydirbytes = arrayfun(@(x) x.bytes/1024, dir('*.mat'))
mydirbytes =
   33.7500
   33.7500
   33.7500
   33.7500
   33.7500
   33.7500
   33.7500
   33.7500
   33.7500
   33.7500

Cette fois, la sortie est un vecteur régulier de double. UniformOutput a été défini sur true par défaut.

cellfun est une fonction similaire. La différence entre cette fonction et arrayfun est que cellfun opère sur des variables de classe de cellules. Si nous souhaitons extraire uniquement les noms d’une liste de noms de fichiers dans une cellule ‘mydirlist’, nous aurions juste besoin d’exécuter cette fonction comme suit :

mydirnames = cellfun(@(x) x(1:end-4), mydirlist, 'UniformOutput', false)
mydirnames = 
    'mymatfile1'
    'mymatfile10'
    'mymatfile2'
    'mymatfile3'
    'mymatfile4'
    'mymatfile5'
    'mymatfile6'
    'mymatfile7'
    'mymatfile8'
    'mymatfile9'

Encore une fois, comme une sortie n’est pas un vecteur régulier de nombres, une sortie doit être enregistrée dans une variable de cellule.

Dans l’exemple ci-dessous, je combine deux fonctions en une et renvoie uniquement une liste de noms de fichiers sans extension :

cellfun(@(x) x(1:end-4), arrayfun(@(x) x.name, dir('*.mat'), 'UniformOutput', false), 'UniformOutput', false)
ans = 
    'mymatfile1'
    'mymatfile10'
    'mymatfile2'
    'mymatfile3'
    'mymatfile4'
    'mymatfile5'
    'mymatfile6'
    'mymatfile7'
    'mymatfile8'
    'mymatfile9'

C’est fou mais très possible car arrayfun renvoie une cellule qui est l’entrée attendue de cellfun ; une remarque à ce sujet est que nous pouvons forcer l’une de ces fonctions à renvoyer des résultats dans une variable de cellule en définissant UniformOutput sur false, explicitement. Nous pouvons toujours obtenir des résultats dans une cellule. Nous ne pourrons peut-être pas obtenir de résultats dans un vecteur régulier.

Il existe une autre fonction similaire qui opère sur les champs d’une structure : structfun. Je ne l’ai pas particulièrement trouvé aussi utile que les deux autres mais il brillerait dans certaines situations. Si par exemple on souhaite savoir quels champs sont numériques ou non numériques, le code suivant peut donner la réponse :

structfun(@(x) ischar(x), mydir(1))

Le premier et le deuxième champ d’une structure de répertoire sont de type char. Par conséquent, la sortie est :

 1
 1
 0
 0
 0

De plus, la sortie est un vecteur logique de ’true’ / ‘false’. Par conséquent, il est régulier et peut être enregistré dans un vecteur ; pas besoin d’utiliser une classe de cellules.

Blocs de commentaires #

Si vous souhaitez commenter une partie de votre code, les blocs de commentaires peuvent être utiles. Le bloc de commentaires commence par un %{ dans une nouvelle ligne et se termine par %} dans une autre nouvelle ligne :

a = 10;
b = 3;
%{
c = a*b;
d = a-b;
%}

Cela vous permet de plier les sections que vous avez commentées pour rendre le code plus propre et compact.

Ces blocs sont également utiles pour activer/désactiver des parties de votre code. Tout ce que vous avez à faire pour décommenter le bloc est d’ajouter un autre % avant qu’il ne commence :

a = 10;
b = 3;
%%{ <-- another % over here
c = a*b;
d = a-b;
%}

Parfois, vous souhaitez commenter une section du code, mais sans affecter son indentation :

for k = 1:a
    b = b*k;
    c = c-b;
    d = d*c;
    disp(b)
end

Habituellement, lorsque vous marquez un bloc de code et appuyez sur Ctrl+r pour le commenter (en ajoutant automatiquement le % à toutes les lignes, puis lorsque vous appuyez plus tard Ctrl+i pour l’indentation automatique, le bloc de code se déplace de sa place hiérarchique correcte, et se déplace trop vers la droite :

for k = 1:a
    b = b*k;
    %     c = c-b;
    %     d = d*c;
    disp(b)
end

Une façon de résoudre ce problème consiste à utiliser des blocs de commentaires, afin que la partie interne du bloc reste correctement indentée :

for k = 1:a
    b = b*k;
    %{
    c = c-b;
    d = d*c;
    %}
    disp(b)
end
