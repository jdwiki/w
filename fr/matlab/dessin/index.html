Cercles #

L’option la plus simple pour dessiner un cercle est - évidemment - la fonction rectangle.

%// radius
r = 2;

%// center
c = [3 3];

pos = [c-r 2*r 2*r];
rectangle('Position',pos,'Curvature',[1 1])
axis equal

mais la courbure du rectangle doit être réglée sur 1 !

Le vecteur position définit le rectangle, les deux premières valeurs x et y sont le coin inférieur gauche du rectangle. Les deux dernières valeurs définissent la largeur et la hauteur du rectangle.

pos = [ [x y] width height ]

Le coin inférieur gauche du cercle - oui, ce cercle a des coins, bien qu’imaginaires - est le centre c = [3 3] moins le rayon r = 2 qui est [x y] = [1 1]. Width et height sont égaux au diamètre du cercle, donc width = 2*r; hauteur = largeur;

![entrez la description de l’image ici][1]

Dans le cas où la régularité de la solution ci-dessus n’est pas suffisante, il n’y a aucun moyen d’utiliser la manière évidente de dessiner un cercle réel à l’aide de fonctions trigonométriques.

%// number of points
n = 1000;

%// running variable
t = linspace(0,2*pi,n);

x = c(1) + r*sin(t);
y = c(2) + r*cos(t);

%// draw line
line(x,y)

%// or draw polygon if you want to fill it with color
%// fill(x,y,[1,1,1])
axis equal

Flèches #

Tout d’abord, on peut utiliser quiver, où l’on n’a pas à gérer les unités de figures normalisées peu pratiques en utilisant annotation

drawArrow = @(x,y) quiver( x(1),y(1),x(2)-x(1),y(2)-y(1),0 )    

x1 = [10 30];
y1 = [10 30];

drawArrow(x1,y1); hold on

x2 = [25 15];
y2 = [15 25];

drawArrow(x2,y2)

![entrez la description de l’image ici][1]

L’important est le 5ème argument de quiver : 0 qui désactive une mise à l’échelle autrement par défaut, car cette fonction est généralement utilisée pour tracer des champs vectoriels. (ou utilisez la paire de valeurs de propriété 'AutoScale','off')

On peut aussi ajouter des fonctionnalités supplémentaires :

drawArrow = @(x,y,varargin) quiver( x(1),y(1),x(2)-x(1),y(2)-y(1),0, varargin{:} )       
drawArrow(x1,y1); hold on
drawArrow(x2,y2,'linewidth',3,'color','r')

[![entrez la description de l’image ici][2]][2]

Si différentes pointes de flèche sont souhaitées, il faut utiliser des annotations (cette réponse peut être utile [Comment puis-je changer le style de la tête de flèche dans le tracé du carquois ?](http://stackoverflow.com/questions/18776172/in-matlab-how- puis-je-changer-le-style-de-pointe-de-flèche-dans-l’intrigue-quiver)).

La taille de la tête de flèche peut être ajustée avec la propriété 'MaxHeadSize'. Ce n’est pas cohérent malheureusement. Les limites des axes doivent être définies ultérieurement.

x1 = [10 30];
y1 = [10 30];
drawArrow(x1,y1,{'MaxHeadSize',0.8,'Color','b','LineWidth',3}); hold on

x2 = [25 15];
y2 = [15 25];
drawArrow(x2,y2,{'MaxHeadSize',10,'Color','r','LineWidth',3}); hold on

xlim([1, 100])
ylim([1, 100])

![entrez la description de l’image ici][3]

Il existe un autre ajustement pour les têtes de flèches ajustables :

function [ h ] = drawArrow( x,y,xlimits,ylimits,props )

xlim(xlimits)
ylim(ylimits)

h = annotation('arrow');
set(h,'parent', gca, ...
    'position', [x(1),y(1),x(2)-x(1),y(2)-y(1)], ...
    'HeadLength', 10, 'HeadWidth', 10, 'HeadStyle', 'cback1', ...
    props{:} );

end

que vous pouvez appeler depuis votre script comme suit :

drawArrow(x1,y1,[1, 100],[1, 100],{'Color','b','LineWidth',3}); hold on
drawArrow(x2,y2,[1, 100],[1, 100],{'Color','r','LineWidth',3}); hold on

![entrez la description de l’image ici][4]

Ellipse #

Pour tracer une ellipse, vous pouvez utiliser son [équation][1]. Une ellipse a un grand et un petit axe. Nous voulons également pouvoir tracer l’ellipse sur différents points centraux. On écrit donc une fonction dont les entrées et les sorties sont :

Inputs:
    r1,r2: major and minor axis respectively
    C: center of the ellipse (cx,cy)
Output:
    [x,y]: points on the circumference of the ellipse

Vous pouvez utiliser la fonction suivante pour obtenir les points sur une ellipse, puis tracer ces points.

function [x,y] = getEllipse(r1,r2,C)
beta = linspace(0,2*pi,100);
x = r1*cos(beta) - r2*sin(beta);
y = r1*cos(beta) + r2*sin(beta);
x = x + C(1,1);
y = y + C(1,2);
end

Exemple :

[x,y] = getEllipse(1,0.3,[2 3]);
plot(x,y);

Polygone(s) #

Créez des vecteurs pour contenir les emplacements x et y des sommets, alimentez-les dans patch.

Polygone unique #

X=rand(1,4); Y=rand(1,4);
h=patch(X,Y,'red');

[![polygone unique][1]][1]

Polygones multiples #

Les sommets de chaque polygone occupent une colonne de chacun des X, Y.

X=rand(4,3); Y=rand(4,3);
for i=2:3
  X(:,i)=X(:,i)+(i-1); % create horizontal offsets for visibility
end

h=patch(X,Y,'red');

[![trois polygones][2]][2]

Tracé pseudo 4D #

Une matrice (m x n) peut être représentée par une surface en utilisant surf;

La couleur de la surface est automatiquement définie en fonction des valeurs de la matrice (m x n). Si le colormap n’est pas spécifié, celui par défaut est appliqué.

Une barre de couleurs peut être ajoutée pour afficher la palette de couleurs actuelle et indiquer le mappage des données valeurs dans la palette de couleurs.

Dans l’exemple suivant, la matrice z (m x n) est générée par la fonction :

z=x.*y.*sin(x).*cos(y);

sur l’intervalle [-pi,pi]. Les valeurs x et y peuvent être générées à l’aide du meshgrid fonction et la surface est rendue comme suit :

% Create a Figure
figure
% Generate the `x` and `y` values in the interval `[-pi,pi]`
[x,y] = meshgrid([-pi:.2:pi],[-pi:.2:pi]);
% Evaluate the function over the selected interval
z=x.*y.*sin(x).*cos(y);
% Use surf to plot the surface
S=surf(x,y,z);
xlabel('X Axis');
ylabel('Y Axis');
zlabel('Z Axis');
grid minor
colormap('hot')
colorbar

[![entrez la description de l’image ici][1]][1]

Figure 1

Maintenant, il se peut que des informations supplémentaires soient liées aux valeurs de la matrice z et ils sont stockés dans une autre matrice (m x n)

Il est possible d’ajouter ces informations supplémentaires sur le tracé en modifiant le manière dont la surface est colorée.

Cela permettra d’avoir une sorte de tracé 4D : à la représentation 3D de la surface générée par la première matrice (m x n), la quatrième dimension sera représentée par la données contenues dans la seconde matrice (m x n).

Il est possible de créer un tel tracé en appelant surf avec 4 entrées :

surf(x,y,z,C)

où le paramètre C est la deuxième matrice (qui doit être de la même taille que z) et est utilisé pour définir la couleur de la surface.

Dans l’exemple suivant, la matrice “C” est générée par la fonction :

C=10*sin(0.5*(x.^2.+y.^2))*33;

sur l’intervalle [-pi,pi]

La surface générée par ‘C’ est

[![entrez la description de l’image ici][2]][2]

Figure 2

Nous pouvons maintenant appeler surf avec quatre entrées :

figure
surf(x,y,z,C)
% shading interp
xlabel('X Axis');
ylabel('Y Axis');
zlabel('Z Axis');
grid minor
colormap('hot')
colorbar

[![entrez la description de l’image ici][3]][3]

Figure 3

En comparant la Figure 1 et la Figure 3, nous pouvons remarquer que :

  • la forme de la surface correspond aux valeurs z (la première matrice (m x n))
  • la couleur de la surface (et sa gamme, donnée par la colorbar) correspond à la Valeurs C (la première matrice (m x n))

[![entrez la description de l’image ici][4]][4]

Illustration 4

Bien entendu, il est possible d’intervertir ‘z’ et ‘C’ dans le tracé pour avoir la forme de la surface donnée par la matrice ‘C’ et la couleur donnée par la matrice ‘z’ :

figure
surf(x,y,C,z)
% shading interp
xlabel('X Axis');
ylabel('Y Axis');
zlabel('Z Axis');
grid minor
colormap('hot')
colorbar

et de comparer la figure 2 avec la figure 4

Dessin rapide #

Il existe trois manières principales de créer des tracés ou des animations séquentiels : plot(x,y), set(h , 'XData' , y, 'YData' , y) et animatedline. Si vous voulez que votre animation soit fluide, vous avez besoin d’un dessin efficace, et les trois méthodes ne sont pas équivalentes.

% Plot a sin with increasing phase shift in 500 steps
x = linspace(0 , 2*pi , 100);

figure
tic
for thetha = linspace(0 , 10*pi , 500)
    y = sin(x + thetha);
    plot(x,y)
    drawnow
end
toc

J’obtiens 5.278172 secondes. La fonction de tracé supprime et recrée essentiellement l’objet ligne à chaque fois. Un moyen plus efficace de mettre à jour un tracé consiste à utiliser les propriétés XData et YData de l’objet Line.

tic
h = [];   % Handle of line object
for thetha = linspace(0 , 10*pi , 500)
    y = sin(x + thetha);
    
    if isempty(h)
        % If Line still does not exist, create it
        h = plot(x,y);
    else
        % If Line exists, update it
        set(h , 'YData' , y)
    end
    drawnow
end
toc

Maintenant, j’obtiens 2,741996 secondes, bien mieux !

animatedline est une fonction relativement nouvelle, introduite en 2014b. Voyons comment ça se passe :

tic
h = animatedline;
for thetha = linspace(0 , 10*pi , 500)
    y = sin(x + thetha);
    clearpoints(h)
    addpoints(h , x , y)
    drawnow
end
toc

3,360569 secondes, pas aussi bon que la mise à jour d’un tracé existant, mais toujours mieux que plot(x,y).

Bien sûr, si vous devez tracer une seule ligne, comme dans cet exemple, les trois méthodes sont presque équivalentes et donnent des animations fluides. Mais si vous avez des tracés plus complexes, la mise à jour des objets “Ligne” existants fera la différence.
