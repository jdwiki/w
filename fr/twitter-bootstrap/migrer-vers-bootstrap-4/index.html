Bootstrap 4 est une [réécriture majeure] (http://v4-alpha.getbootstrap.com/migration/) et de nombreux changements doivent être pris en compte lors de la mise à niveau à partir de Bootstrap 3. Voici les changements de nom de classe, des conseils et des exemples de migration de votre code Bootstrap 3.x vers Bootstrap 4.x.

Ce n’est qu’un petit exemple d’exemples plus détaillés à suivre.

Barre de navigation Bootstrap 4 #

Le nouveau composant Bootstrap 4 Navbar est amélioré par rapport à son prédécesseur Bootstrap 3.x. Dans Bootstrap 4, la barre de navigation est ** réactive par défaut ** et utilise ** flexbox ** pour faciliter l’alignement du contenu de la barre de navigation. Il suffit également d’utiliser les nouvelles classes navbar-toggleable-* pour modifier le point d’arrêt de la barre de navigation. Maintenant, la barre de navigation a 6 tailles de points d’arrêt ou “états” afin que vous puissiez facilement avoir l’une des options de barre de navigation suivantes.

  • La barre de navigation jamais s’effondre dans la vue mobile verticale et est toujours horizontale.
  • La barre de navigation est * toujours * réduite à la vue verticale et basculée via le hamburger.
  • La barre de navigation s’effondre en vue verticale à * l’un des 4 points d’arrêt réactifs *.

Barre de navigation Bootstrap 4 de base

<nav class="navbar navbar-toggleable-md">
  <button class="navbar-toggler navbar-toggler-right" type="button" data-toggle="collapse" data-target="#navbar1">
    <span class="navbar-toggler-icon"></span>
  </button>
  <a class="navbar-brand" href="#">Navbar</a>
  <div class="collapse navbar-collapse" id="navbar1">
    <ul class="navbar-nav mr-auto">
      <li class="nav-item active">
        <a class="nav-link" href="#">Home</a>
      </li>
      <li class="nav-item">
        <a class="nav-link" href="#">Link</a>
      </li>
    </ul>
  </div>
</nav>

Comme vous pouvez le voir dans le code ci-dessus, la classe navbar-header a été supprimée de Bootstrap 4, et le container-fluid n’est plus requis pour une barre de navigation pleine largeur.

Modification du point d’arrêt de la barre de navigation

La classe navbar-toggleable-md permet de réduire verticalement la barre de navigation ci-dessus (et d’afficher l’icône de basculement) au point d’arrêt moyen (md) de 992px. Pour changer cela en un point d’arrêt différent, nous aurions juste besoin d’échanger navbar-toggleable-md avec l’un de ceux-ci.

  • navbar-toggleable = réduire sur les largeurs xs <576px
  • navbar-toggleable-sm = réduire sur les largeurs sm <768px
  • navbar-toggleable-lg = réduire sur les largeurs lg <1200px

[Démo de la barre de navigation Bootstrap 4 Breakpoint] (http://www.codeply.com/go/imngr08QRi)

Modification de l’alignement de la barre de navigation

Flexbox nous permet de modifier facilement l’alignement de la barre de navigation et son contenu (marque, liens, formulaires ou texte). Le contenu de la barre de navigation par défaut est aligné à gauche. Bien sûr, il existe de nombreux autres scénarios d’alignement…

  • Marque gauche (par défaut), liens centre et droite
  • Centre de la marque, liens gauche et droite
  • Marque à gauche et liens à droite
  • Saisie de la marque, des liens et de la largeur de remplissage
  • Pas de marque, liens centre & droite
  • Marque à gauche, liens à droite à l’intérieur du conteneur
  • Liens justifiés (largeur de remplissage) centrés

Bootstrap 4 Centrage #

Comment centrer un élément, une colonne ou un contenu dans une colonne fonctionne différemment dans Bootstrap 4.

Centre horizontal #

  • text-center est toujours utilisé pour les éléments display:inline
  • mx-auto remplace center-block pour centrer les éléments display:block
  • offset-* ou mx-auto peut être utilisé pour centrer les colonnes de la grille

mx-auto (marges automatiques sur l’axe des x) centrera les éléments display:block ou display:flex qui ont une largeur définie, (%, vw, px, etc. ). Flexbox est utilisé par défaut sur les colonnes de la grille, il existe donc également différentes méthodes de centrage flexbox.

Texte central ou éléments en ligne : centre-texte

<div class="container">
    <h1 class="text-center">i'm centered</h1>
    <div class="row">
        <div class="col text-center">i'm centered!</div>
    </div>
</div>

Centrer display:block ou display:flex : mx-auto

<div class="row">
    <div class="col-12">
        <img class="mx-auto d-block" src="//placehold.it/200x150?text=mx-auto">
    </div>
</div>

Centrer les colonnes en utilisant les décalages : offset-*

<div class="row">
    <div class="col-4 offset-4">
        <h6>I'm .col-4 centered (offset 4)
    </div>
</div>

Les colonnes peuvent aussi être centrées avec : mx-auto

<div class="row">
    <div class="col-4 mx-auto">
        <h6>I'm .col-4 centered</h6>
    </div>
</div>

Bootstrap 3 à Bootstrap 4 Changements CSS #

Étant donné que Bootstrap 4 est une réécriture ** majeure **, de nombreux noms de classe Bootstrap 3.x ont été modifiés ou supprimés. La restructuration de composants tels que la barre de navigation et l’introduction de nouvelles classes CSS et la prise en charge de Flexbox signifient que la mise à niveau vers 4.x n’est pas un simple processus de conversion à partir de 3.x.

Cependant, certaines classes CSS Bootstrap 3.x ont un remplacement spécifique de Bootstrap 4.

Le nom/sélecteur de classe CSS passe de Bootstrap 3.3.7 à 4 (alpha 6)

{t} - représente un niveau ou un point d’arrêt (c’est-à-dire : sm, md, lg, etc.).
Le niveau xs est le niveau par défaut et n’a pas besoin d’être spécifié : `col- 3’,‘col-6’,etc.

{u} - représente une taille d’unité de colonne (c’est-à-dire : 1-12)

Modifications de la disposition des colonnes du système de grille dans Bootstrap 4 #

Le premier bloc de code est écrit dans Bootstrap 3. Dans Bootstrap 3, il existe 4 types de spécifications de colonne, à savoir col-md-* col-lg-* col-sm-* col-xs-* . Une mise en page entièrement réactive ressemblera à ceci dans Bootstrap 3 :

<div class="row">
    <div class="col-lg-4 col-md-8 col-sm-8 col-xs-8">
        contents
    </div>
    <div class="col-lg-4 col-md-4 col-sm-4 col-xs-4">
        contents
    </div>
</div>

Dans Bootstrap 4, ils ont ajouté un nouveau niveau de grille sm en dessous de 768px pour un contrôle plus granulaire. Donc Bootstrap 4 a col-* (xs), col-sm-*, col-md-*, col-lg-* et col-xl-*. Ainsi, ce qui était .col-md-6 dans la v3 est maintenant .col-lg-6 dans la v4. Notez que l’infixe -xs a été supprimé afin que .col-6 représente 6 unités de colonne au point d’arrêt extra petit (par défaut).

Donc, si nous voulons maintenant écrire le même exemple ci-dessus dans Bootstrap 4, cela ressemblerait à ceci :

<div class="row">
    <div class="col-xl-8 col-lg-8 col-md-8 col-sm-8 col-8">
        contents
    </div>
    <div class="col-xl-8 col-lg-8 col-md-4 col-sm-4 col-4">
        contents
    </div>
</div>

Modifications de la prise en charge du navigateur #

Dans twitter-bootstrap 4, la prise en charge de IE8, IE9 et iOS 6 a été abandonnée. La v4 est désormais uniquement IE10+ et iOS 7+. Pour les sites nécessitant l’un ou l’autre, utilisez la v3.

Dans twitter-bootstrap 4, la prise en charge officielle du “Navigateur et WebView d’Android v5.0 Lollipop” a été ajoutée. Les versions antérieures du navigateur Android et de WebView restent uniquement “prises en charge de manière non officielle”.

Suppression de classe d’affixe #

L’affixe est supprimé de Bootstrap 4.

Il est recommandé d’utiliser un polyfill “position : collant” à la place.

Si vous utilisiez Affixe pour appliquer des styles supplémentaires sans position, les polyfills pourraient ne pas prendre en charge votre cas d’utilisation. Une option pour de telles utilisations est la bibliothèque tierce ScrollPos-Styler.

[Selon la documentation Bootstrap][1]

$(document).ready(function() {

  var toggleAffix = function(affixElement, scrollElement, wrapper) {
  
    var height = affixElement.outerHeight(),
        top = wrapper.offset().top;
    
    if (scrollElement.scrollTop() >= top){
        wrapper.height(height);
        affixElement.addClass("affix");
    }
    else {
        affixElement.removeClass("affix");
        wrapper.height('auto');
    }
      
  };
  

  $('[data-toggle="affix"]').each(function() {
    var ele = $(this),
        wrapper = $('<div></div>');
    
    ele.before(wrapper);
    $(window).on('scroll resize', function() {
        toggleAffix(ele, $(this), wrapper);
    });
    
    // init
    toggleAffix(ele, $(window), wrapper);
  });
  
});

Bootstrap 4 Alignement vertical #

Aligner verticalement les éléments au centre ou au bas a toujours été un défi avec CSS et Bootstrap. L’alignement vertical souhaité peut être à l’intérieur d’un conteneur parent ou par rapport à des éléments adjacents.

Maintenant que Bootstrap 4 est flexbox par défaut, il existe de nombreuses approches différentes de l’alignement vertical en utilisant : Auto-margins, Flexbox Utilities, ou Display Utilities avec avec Vertical Align Utilities.

Au début, les utilitaires d’alignement vertical sembleraient un choix évident, mais ils ne fonctionnent qu’avec les éléments d’affichage en ligne et en tableau. Voici quelques options et scénarios d’alignement vertical de Bootstrap 4…

1 - Centre vertical avec marges automatiques :

Une façon de centrer verticalement consiste à utiliser “my-auto”. Cela centrera l’élément dans son conteneur. Par exemple, h-100 rend la ligne pleine hauteur et my-auto centrera verticalement la colonne col-sm-12.

<div class="row h-100">
    <div class="col-sm-12 my-auto">
        <div class="card card-block w-25">Card</div>
    </div>
</div>

       <div class="row">

3 - Centre vertical à l’aide des utilitaires d’affichage :

Bootstrap 4 a display utils qui peut être utilisé pour display:table, display:table-cell, display:inline , etc. Ceux-ci peuvent être utilisés avec les utilitaires d’alignement vertical pour aligner des éléments de cellule en ligne, en bloc en ligne ou de tableau.

<div class="row h-50">
    <div class="col-sm-12 h-100 d-table">
        <div class="card card-block d-table-cell align-middle">
            I am centered vertically
        </div>
    </div>
</div>

Bootstrap Ordre des 4 colonnes #

Changer l’ordre (ou la position) était possible dans Bootstrap 3 en utilisant les classes push pull. Dans Bootstrap 4, les classes push pull fonctionnent toujours, et en plus l’ordre flexbox peut être utilisé.

Dans Bootstrap 4, les classes push pull sont désormais push-{viewport}-{units} et pull-{viewport}-{units} et l’infixe xs- a été supprimé. Considérez cet exemple qui modifie l’ordre des colonnes en une disposition 1-3-2 sur xs et sm :

<div class="row">
    <div class="col-3 col-md-6">
        1
    </div>
    <div class="col-3 col-md-6 push-6 push-md-0">
        2
    </div>
    <div class="col-6 col-md-12 pull-3 pull-md-0">
        3
    </div>
</div>

[Démo Push-Pull Bootstrap 4] (http://www.codeply.com/go/OmrcmepbUp)

Étant donné que la nouvelle version 4 est flexbox, une autre option consiste à utiliser les classes de l’utilitaire flexbox pour modifier l’ordre des colonnes. Désormais, les colonnes col-*-12 pleine largeur de 12 unités peuvent être inversées à l’aide de l’ordre flexbox.

<div class="row">
    <div class="col-md-12">
        Col 1
    </div>
    <div class="col-md-12 flex-first flex-md-unordered">
        Col 2
    </div>
</div>
