Le système de grille de Bootstrap se compose de 12 unités appelées Colonnes (classes CSS .col-*-*) qui sont utilisées pour mettre en page le contenu de gauche à droite dans la fenêtre d’affichage. Les colonnes sont contenues dans Rows (classe CSS .row) pour créer des groupes horizontaux de colonnes. Les lignes sont placées dans un Container fixe ou pleine largeur (.container ou .container-fluid, respectivement) pour un alignement correct. Les colonnes ont un rembourrage qui crée un espacement (appelé “gouttière”) entre le contenu des colonnes.

Bootstrap comprend un premier système de grille fluide réactif et mobile qui s’adapte de manière appropriée jusqu’à 12 colonnes à mesure que la taille de l’appareil ou de la fenêtre d’affichage augmente. Il comprend des classes prédéfinies pour créer rapidement des mises en page à travers une série de lignes et de colonnes qui hébergent votre contenu.

Lignes et colonnes bootstrap #

Le système de grille de Bootstrap a ** 12 unités ** appelées ** Colonnes ** qui peuvent être utilisées pour mettre en page le contenu horizontalement dans la fenêtre d’affichage.

La raison d’une grille de 12 unités (au lieu de 10, 16, etc.) est que 12 se divise également en 6 (moitiés), 4 (quarts) et 3 (tiers). Cela rend l’adaptation à une variété de mises en page beaucoup plus facile. Les colonnes de grille de Bootstrap sont identifiées par différentes classes CSS col-{breakpoint}-{units}. [En savoir plus sur la largeur de la fenêtre et les points d’arrêt (A.K.A. Tiers)](https://www.wikiod.com/fr/twitter-bootstrap/systeme-de-grille#Niveaux de grille d’amorçage (points d’arrêt))

Ainsi, par exemple, col-md-3 représente une colonne qui occupe 3 des 12 unités (ou 25%) dans une fenêtre de largeur moyenne (md). Pour utiliser une largeur de colonne dans votre mise en page, utilisez simplement la classe col-{breakpoint}-{units} appropriée dans votre balisage HTML.

<div class="col-{breakpoint}-{units}">

La largeur de colonne est fluide (pas de largeur fixe), de sorte que les colonnes consomment un pourcentage de leur conteneur.

** Unités de colonne dans Bootstrap 3 **

  • col-*-1 : 1 sur 12 (8,33333333 % de largeur)
  • col-*-2 : 2 sur 12 (16,66666667 % de largeur)
  • col-*-3 : 3 sur 12 (25 % de largeur)
  • col-*-4 : 4 sur 12 (33,3333333 % de largeur)
  • col-*-5 : 5 sur 12 (41,66666667 % de largeur)
  • col-*-6 : 6 sur 12 (50 % de largeur)
  • col-*-7 : 7 sur 12 (58,33333333 % de largeur)
  • col-*-8 : 8 sur 12 (66,66666667 % de largeur)
  • col-*-9 : 9 sur 12 (75 % de largeur)
  • col-*-10 : 10 sur 12 (83,33333333 % de largeur)
  • col-*-11 : 11 sur 12 (91,66666667 % de largeur)
  • col-*-12 : 12 sur 12 (100 % de largeur)

La ligne Bootstrap

La classe Bootstrap .row est utilisée pour contenir les colonnes. Les colonnes doivent toujours être placées dans des lignes, et les lignes doivent toujours être placées à l’intérieur d’un conteneur (container ou container-fluid). La ligne utilise des marges négatives (-15px) pour assurer un espacement approprié entre le contenu de la colonne et le bord du navigateur. Les lignes sont utilisées pour regrouper les colonnes horizontalement.

<div class="container">
    <div class="row">
        <!-- one more columns -->
        <div class="col-{breakpoint}-{units}">..</div>
    </div>
</div>

Les colonnes rempliront le .row horizontalement de gauche à droite et wrap sur une nouvelle ligne toutes les 12 unités de colonne. Par conséquent, vous pouvez utiliser .rows pour créer des sauts horizontaux, ou vous pouvez ajouter plus de 12 unités de colonne dans un seul élément .row pour avoir des colonnes qui s’enroulent ( ou pile) verticalement vers le bas de la fenêtre.

Lorsque vous utilisez l’habillage de colonne (plus de 12 unités dans un .row), vous devez utiliser des réinitialisations réactives (ou des correctifs) pour assurer un habillage uniforme du contenu inégal des colonnes. Ceci est essentiel lorsque le contenu des colonnes varie en hauteur.

Fonctionnement de la grille d’amorçage (moyen)

Niveaux de grille d’amorçage (points d’arrêt) #

En plus du concept de [unités de colonne] (https://www.wikiod.com/fr/twitter-bootstrap/systeme-de-grille#Lignes et colonnes bootstrap), Bootstrap a différents ** points d’arrêt ** ou tailles de grille appelées niveaux. La grille Bootstrap 3 comporte quatre (4) niveaux pour s’adapter à différentes largeurs d’écran (ou de fenêtre). Les niveaux Bootstrap 3 sont xs, sm, md et lg. Les colonnes de grille de Bootstrap sont identifiées par différentes classes CSS col-{breakpoint}-{units}.

Chaque niveau de grille englobe une gamme conçue pour s’adapter au mieux aux largeurs d’écran typiques des appareils, telles que celles des ordinateurs de bureau, des ordinateurs portables, des tablettes et des smartphones.

Bootstrap utilise des requêtes multimédia CSS pour créer des points d’arrêt réactifs qui établissent une limite pour chaque taille de grille. Ces tailles de grille vous permettent de modifier la disposition des colonnes pour mieux correspondre aux différentes largeurs d’écran et appareils__ l’essence même de la conception réactive.

  • col-xs-* — pour les plus petites largeurs d’écran comme les smartphones < 768 px
  • col-sm-* — pour les petites largeurs d’écran comme les smartphones et les tablettes >= 768 px
  • col-md-* — pour les largeurs d’écran moyennes comme les tablettes et les ordinateurs portables >= 992 px
  • col-lg-* — pour les grandes largeurs d’écran comme les ordinateurs de bureau >= 1200 px

Largeur de colonne identique pour chaque appareil

Pour créer une colonne qui est toujours * 50 % * de la largeur de la fenêtre d’affichage (sur tous les appareils), vous pouvez définir col-*-6 pour chaque niveau.

<div class="col-xs-6 col-sm-6 col-md-6 col-lg-6">..</div>

Cependant, il s’agit d’un balisage supplémentaire inutile, car col-xs-6 signifie 6 unités sur xs et plus. Le plus petit niveau que vous définissez (xs, sm ou md) définit également la taille pour les largeurs d’écran plus grandes. Pour la colonne de taille même sur tous les niveaux, définissez simplement la largeur de la plus petite fenêtre d’affichage.

Code plus court :

<div class="col-xs-6">..</div>

Largeur de colonne différente pour chaque appareil (design réactif)

Les classes col-*-* peuvent être combinées pour contrôler la largeur des colonnes sur différentes tailles de grille.

Par exemple, créez une colonne de largeur 50% au niveau sm et une colonne de largeur 25% au niveau md…

<div class="col-md-3 col-sm-6">..</div>

Les grilles sm, md et lg seront toutes “empilées” verticalement à des largeurs de fenêtre inférieures à 768 pixels. C’est là que la grille xs s’intègre. Les colonnes qui utilisent les classes col-xs-* ne s’empileront pas verticalement et continueront à se réduire sur les plus petits écrans.

Manipulation de l’ordre des colonnes en utilisant push and pull #

<div class="container content">
    <div class="row">
        <!--Main Content-->
        <div class="col-lg-9 col-lg-push-3">
            Main Content
        </div>
        
        <!--Sidebar-->
        <div class="col-lg-3 col-lg-pull-9">
            Sidebar
        </div>
    </div>
</div>

Cela modifie l’ordre des colonnes de la grille intégrée.

Syntaxe : .col-md-push-* et .col-md-pull-*.

Conteneurs #

Bootstrap nécessite un élément conteneur pour envelopper le contenu du site et héberger notre système de grille. Vous pouvez choisir l’un des deux conteneurs à utiliser dans vos projets.

Utilisez la classe .container pour un conteneur réactif à largeur fixe.

<div class="container">
  ...
</div>

Utilisez la classe .container-fluid pour un conteneur pleine largeur, couvrant toute la largeur de votre fenêtre.

<div class="container-fluid">
  ...
</div>

Remarque : Les conteneurs ne sont pas emboîtables (vous ne pouvez pas placer un conteneur dans un autre conteneur), en raison du “remplissage” et plus encore.

Requêtes multimédias #

Les requêtes multimédias dans Bootstrap vous permettent de déplacer, d’afficher et de masquer le contenu en fonction de la taille de la fenêtre d’affichage. Les requêtes multimédias suivantes sont utilisées dans les fichiers LESS pour créer les points d’arrêt clés dans le système de grille Bootstrap :

/* Small devices (tablets, 768px and up) */
@media (min-width: @screen-sm-min) { ... }

/* Medium devices (desktops, 992px and up) */
@media (min-width: @screen-md-min) { ... }

/* Large devices (large desktops, 1200px and up) */
@media (min-width: @screen-lg-min) { ... }

Parfois, ceux-ci sont étendus pour inclure une largeur maximale afin de limiter le CSS à un ensemble plus restreint d’appareils :

@media (max-width: @screen-xs-max) { ... }
@media (min-width: @screen-sm-min) and (max-width: @screen-sm-max) { ... }
@media (min-width: @screen-md-min) and (max-width: @screen-md-max) { ... }
@media (min-width: @screen-lg-min) { ... }

Colonnes de décalage #

[Ces classes augmentent la marge gauche d’une colonne de * colonnes. Par exemple, .col-md-offset-4 déplace .col-md-4 sur quatre colonnes.][1]

<div class="row">
  <div class="col-lg-4"></div>
  <div class="col-lg-4 col-lg-offset-4"></div>
</div>
<div class="row">
  <div class="col-lg-5 col-lg-offset-1"></div>
  <div class="col-lg-5 col-lg-offset-1"></div>
</div>

[1] : http://getbootstrap.com/css/#grid-offsetting
