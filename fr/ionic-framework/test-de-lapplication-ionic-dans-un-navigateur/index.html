Le test des fonctionnalités natives de l’appareil telles que l’appareil photo, les vibrations et autres, dont beaucoup se trouvent dans la documentation de [Ionic Native][1], ne peut pas être effectué dans le navigateur. Il s’agit d’une limitation inhérente au fait que Cordova, la plate-forme dont Ionic dépend pour pouvoir accéder aux API natives Android, iOS et Windows Mobile d’un appareil, ne peut pas s’exécuter sur le navigateur.

On peut contourner ce problème en se moquant de la fonctionnalité du plugin natif.

Exemple #

Voici un exemple de simulation du plug-in [Camera][2] :

Allez-y et créez un dossier facultatif dans le dossier racine de votre projet.

cd src
mkdir mocks 
cd mocks 
touch camera-mock.ts

Ouvrez camera-mock.ts et copiez-collez le code suivant :

export class CameraMock {
    getPicture(params) {
        return new Promise((resolve, reject) => {
            resolve("BASE_64_IMAGE_DATA");
        });
    }
}

Ouvrez ensuite src/app.module.ts et importez la classe fictive"

import { CameraMock } from "../mocks/camera-mock";

Ajoutez-le ensuite au tableau des fournisseurs de modules :

@NgModule({
declarations: [
    MyApp,
    HomePage
],
imports: [
    BrowserModule,
    IonicModule.forRoot(MyApp)
],
bootstrap: [IonicApp],
entryComponents: [
    MyApp,
    HomePage
],
providers: [
    StatusBar,
    SplashScreen,
    CameraMock,
    {provide: ErrorHandler, useClass: IonicErrorHandler}
]
})
export class AppModule {}

Vous pouvez maintenant l’utiliser dans n’importe quel composant après l’avoir importé.

[1] : https://ionicframework.com/docs/native/ [2] : https://ionicframework.com/docs/native/camera/

Test dans un navigateur #

Utilisez ionic serve pour démarrer un serveur de développement local pour le développement et les tests d’applications. Ceci est utile à la fois pour les tests de navigateur de bureau et pour tester dans un navigateur de périphérique connecté au même réseau. De plus, cette commande démarre LiveReload qui est utilisé pour surveiller les changements dans le système de fichiers. Dès que vous enregistrez un fichier, le navigateur est automatiquement actualisé. Jetez un œil à la documentation Sass si vous souhaitez également que ionic serve regarde les fichiers Sass du projet.

$ ionic serve [options]

Pour le navigateur Chrome, vous pouvez inspecter les appareils (AVD ou Mobiles), tapez la commande suivante dans la barre d’adresse du navigateur Chrome.

chrome://inspect/#devices

Rechargement en direct #

Par défaut, LiveReload surveillera les changements dans votre répertoire www/, à l’exception de www/lib/. Pour changer cela, vous pouvez spécifier une propriété watchPatterns dans le fichier ionic.project situé à la racine de votre projet pour surveiller (ou non) des modifications spécifiques.

{
  "name": "myApp",
  "app_id": "",
  "watchPatterns": [
    "www/js/*",
    "!www/css/**/*"
  ]
}

Pour une référence sur la syntaxe des modèles glob, consultez globbing patterns sur le site Web de Grunt.

Noter:

$ ionic setup sass

ajoutera une propriété watchPatterns avec les valeurs par défaut à votre fichier ionic.project que vous pourrez ensuite modifier, en plus de la propriété gulpStartupTasks comme décrit dans la documentation Sass.

Laboratoire ionique #

Ionic Lab est une fonctionnalité en plus du “service ionique” qui facilite le test de votre application dans un cadre de téléphone et avec les plates-formes iOS et Android côte à côte. Pour l’utiliser, lancez simplement

$ ionic serve --lab

Lisez l’annonce de sortie complète pour tous les détails !

Spécifier une adresse IP à utiliser #

Supposons que vous souhaitiez spécifier à quelle adresse votre navigateur se connectera, par exemple pour des tests ou des utilisateurs externes. Spécifiez l’adresse avec l’argument --address.

$ ionic serve --address 68.54.96.105

Proxy de service #

La commande serve peut ajouter des proxies au serveur http. Ces proxys sont utiles si vous développez dans le navigateur et que vous devez effectuer des appels vers une API externe. Avec cette fonctionnalité, vous pouvez envoyer une requête proxy à l’API externe via le serveur http ionique empêchant l’erreur “Aucun en-tête “Access-Control-Allow-Origin” n’est présent sur la ressource demandée”.

Dans le fichier ionic.project, vous pouvez ajouter une propriété avec un tableau de proxies que vous souhaitez ajouter. Les proxys sont des objets avec deux propriétés :

  • path : chaîne qui sera comparée au début de l’URL de la requête entrante.
  • proxyUrl : une chaîne avec l’URL de l’endroit où la requête proxy doit aller.
{
  "name": "appname",
  "email": "",
  "app_id": "",
  "proxies": [
    {
      "path": "/v1",
      "proxyUrl": "https://api.instagram.com/v1"
    }
  ]
}

En utilisant la configuration ci-dessus, vous pouvez maintenant faire des requêtes à votre serveur local à http://localhost:8100/v1 pour qu’il envoie des requêtes proxy à https://api.instagram.com/v1

Par exemple:

angular.module('starter.controllers', [])
.constant('InstagramApiUrl', '')
// .constant('InstagramApiUrl','https://api.instagram.com')
//In production, make this the real URL

.controller('FeedCtrl', function($scope, $http, InstagramApiUrl) {

  $scope.feed = null;

  $http.get(InstagramApiUrl + '/v1/media/search?client_id=1&lat=48&lng=2.294351').then(function(data) {
    console.log('data ' , data)
    $scope.feed = data;
  })

})

Voir aussi this gist pour plus d’aide.

Indicateurs/options de ligne de commande #

[--consolelogs|-c] ......  Print app console logs to Ionic CLI
[--serverlogs|-s] .......  Print dev server logs to Ionic CLI
[--port|-p] .............  Dev server HTTP port (8100 default)
[--livereload-port|-i] ..  Live Reload port (35729 default)
[--nobrowser|-b] ........  Disable launching a browser
[--nolivereload|-r] .....  Do not start live reload
[--noproxy|-x] ..........  Do not add proxies
