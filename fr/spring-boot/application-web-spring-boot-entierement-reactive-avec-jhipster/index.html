Créer une application Spring Boot à l’aide de jHipster sur Mac OS #

jHipster vous permet de démarrer une application Web Spring Boot avec un back-end API REST et un front-end AngularJS et Twitter Bootstrap.

Plus d’informations sur jHipster ici : [Documentation jHipster][1]

Installer l’infusion :

 /usr/bin/ruby -e "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/master/install)"

Consultez des informations supplémentaires sur l’installation de brew ici : [Installer Brew] [2]

Installer Gradle

Gradle est un système de gestion et de construction de dépendances.

 brew install gradle

Installer Git

Git est un outil de contrôle de version

 brew install git

Installer NodeJS

NodeJS vous donne accès à npm, le gestionnaire de packages de nœuds nécessaire pour installer d’autres outils.

 brew install node

Installez Yeoman

Yeoman est un générateur

 npm install -g yo

Installer Bower

Bower est un outil de gestion des dépendances

 npm install -g bower

Installer Gulp

Gulp est un coureur de tâches

 npm install -g gulp

Installez le générateur jHipster Yeoman

Ceci est le générateur jHipster

 npm install -g generator-jhipster

Créer une application

Ouvrez une fenêtre de terminal.

Accédez au répertoire racine où vous conserverez vos projets. Créez un répertoire vide dans lequel vous créerez votre application

 mkdir myapplication

Allez dans ce répertoire

 cd myapplication/

Pour générer votre application, tapez

 yo jhipster

Les questions suivantes vous seront posées

Quel type d’application souhaitez-vous créer ?

Votre type d’application dépend si vous souhaitez utiliser une architecture de microservices ou non. Une explication complète sur les microservices est disponible ici, en cas de doute, utilisez l’“application monolithique” par défaut.

Choisissez Application monolithique par défaut si vous n’êtes pas sûr

Quel est votre nom de package Java par défaut ?

Votre application Java l’utilisera comme package racine.

Quel type d’authentification souhaitez-vous utiliser ?

Utilisez * Spring Security * basé sur la session de base par défaut si vous n’êtes pas sûr

Quel type de base de données souhaitez-vous utiliser ?

Quelle base de données de développement souhaitez-vous utiliser ?

C’est la base de données que vous utiliserez avec votre profil “développement”. Vous pouvez soit utiliser :

Utilisez H2 par défaut si vous n’êtes pas sûr

H2, exécuté en mémoire. C’est le moyen le plus simple d’utiliser JHipster, mais vos données seront perdues lorsque vous redémarrerez votre serveur.

Souhaitez-vous utiliser le cache de 2e niveau d’Hibernate ?

Hibernate est le fournisseur JPA utilisé par JHipster. Pour des raisons de performances, nous vous recommandons fortement d’utiliser un cache, et de l’ajuster en fonction des besoins de votre application. Si vous choisissez de le faire, vous pouvez utiliser ehcache (cache local) ou Hazelcast (cache distribué, à utiliser dans un environnement en cluster)

Vous souhaitez utiliser un moteur de recherche dans votre application ? Elasticsearch sera configuré à l’aide de Spring Data Elasticsearch. Vous pouvez trouver plus d’informations sur notre guide Elasticsearch.

Choisissez non si vous n’êtes pas sûr

Souhaitez-vous utiliser des sessions HTTP en cluster ?

Par défaut, JHipster utilise une session HTTP uniquement pour stocker les informations d’authentification et d’autorisation de Spring Security. Bien sûr, vous pouvez choisir de mettre plus de données dans vos sessions HTTP. L’utilisation de sessions HTTP entraînera des problèmes si vous exécutez dans un cluster, en particulier si vous n’utilisez pas d’équilibreur de charge avec des “sessions permanentes”. Si vous souhaitez répliquer vos sessions à l’intérieur de votre cluster, choisissez cette option pour configurer Hazelcast.

Choisissez non si vous n’êtes pas sûr

Voulez-vous utiliser WebSockets ? Les Websockets peuvent être activés à l’aide de Spring Websocket. Nous fournissons également un exemple complet pour vous montrer comment utiliser efficacement le framework.

Choisissez non si vous n’êtes pas sûr

Souhaitez-vous utiliser Maven ou Gradle ? Vous pouvez créer votre application Java générée avec Maven ou Gradle. Maven est plus stable et plus mature. Gradle est plus flexible, plus facile à étendre et plus en vogue.

Choisissez * Gradle * si vous n’êtes pas sûr

Souhaitez-vous utiliser le préprocesseur de feuilles de style LibSass pour votre CSS ? Node-sass est une excellente solution pour simplifier la conception de CSS. Pour être utilisé efficacement, vous devrez exécuter un serveur Gulp, qui sera configuré automatiquement.

Choisissez non si vous n’êtes pas sûr

Souhaitez-vous activer la prise en charge de la traduction avec Angular Translate ? Par défaut, JHipster fournit un excellent support d’internationalisation, à la fois côté client avec Angular Translate et côté serveur. Cependant, l’internationalisation ajoute un peu de surcharge et est un peu plus complexe à gérer, vous pouvez donc choisir de ne pas installer cette fonctionnalité.

Choisissez non si vous n’êtes pas sûr

Quels frameworks de test souhaitez-vous utiliser ? Par défaut, JHipster fournit des tests unitaires/d’intégration Java (en utilisant le support JUnit de Spring) et des tests unitaires JavaScript (en utilisant Karma.js). En option, vous pouvez également ajouter la prise en charge de :

Choisissez aucun si vous n’êtes pas sûr. Vous aurez accès à junit et Karma par défaut.
