Gestion de la sécurité #

La sécurité faisait partie du côté obscur de la documentation de symfony, elle a un composant dédié nommé Security Component.

Ce composant est configuré dans le fichier security.yml du projet d’application principal.

La configuration par défaut ressemble à celle-ci :

# app/config/security.yml
security:
    providers:
        in_memory:
            memory: ~
    
    firewalls:
        dev:
            pattern: ^/(_(profiler|wdt)|css|images|js)/
            security: false

        default:
            anonymous: ~

Vous pouvez définir des Pare-feu spécifiques pour restreindre l’accès à certaines URL à des Rôles spécifiques basés sur une hiérarchie pour vos Utilisateurs qui sont définis par un Fournisseur et des Encodeurs qui gérer la sécurité du mot de passe.

Par exemple, si vous souhaitez créer un Provider personnalisé, depuis votre moteur de base de données, vous pouvez définir votre security.yml comme ceci :

providers:
    your_db_provider:
        entity:
            class: AppBundle:User
            property: apiKey

Ceci est détaillé dans la documentation de symfony : [Comment définir un UserProvider personnalisé][1] et [à partir de la base de données][2] ou [contre LDAP][3] par exemple.

Après cela, vous pouvez définir un pare-feu pour restreindre certaines URL en fonction de votre fournisseur d’utilisateur personnalisé (security.yml) explicitement comme ceci :

firewalls:
    secured_area:
        pattern: ^/admin

Ou avec contrôle d’accès :

access_control:
 - { path: ^/admin/users, roles: ROLE_SUPER_ADMIN }
 - { path: ^/admin, roles: ROLE_ADMIN }

Voir la documentation plus détaillée [ici][4].

La meilleure façon de gérer les utilisateurs est d’utiliser [FosUserBundle][5] qui étend certaines fonctionnalités du framework.

[1] : http://symfony.com/doc/2.8/security/custom_provider.html [2] : http://symfony.com/doc/2.8/security/firewall_restriction.html [3] : http://symfony.com/doc/2.8/security/ldap.html [4] : http://symfony.com/doc/2.8/security.html [5] : http://symfony.com/doc/current/bundles/FOSUserBundle/index.html
