Thème Wordpress et développement de thème enfant

  • Développer votre propre thème

Sur cette page

  • Développer votre propre thème

Wordpress est un CMS largement utilisé pour créer des sites Web d’information simples mais aussi pour créer des sites Web plus sophistiqués et même de petites boutiques en ligne.

Wordpress utilise des thèmes. Ces thèmes sont utilisés pour créer la fonctionnalité de mise en page et de contenu d’un site Web Wordpress. Les thèmes peuvent être trouvés partout sur Internet.

Chaque thème a sa propre fonctionnalité et sa propre mise en page, mais il est parfois difficile de trouver le bon thème pour un site Web. Heureusement, nous pouvons également créer notre propre thème.

Développer votre propre thème #

Un thème wordpress est composé de deux types de fichiers. Les fichiers de base de chaque thème et les fichiers qui définissent la disposition et les fonctionnalités du thème. Ce deuxième groupe, je vais appeler les fichiers spécifiques au thème.

Les fichiers de thème de base
Les fichiers de thème de base sont les fichiers utilisés pour configurer et enregistrer un thème. Dans la liste ci-dessous, je décrirai brièvement chaque fichier et son utilisation. Plus tard, j’ajouterai les fichiers d’exemple les plus élémentaires nécessaires pour configurer votre propre thème wordpress.

  • functions.php : Le fichier functions.php est utilisé pour enregistrer toutes les fonctions, barres latérales, scripts et inclusions du thème. Dans ce fichier, vous pouvez, par exemple, inclure des fichiers CSS, des fichiers JS, etc.
  • Header and footer : Les fichiers d’en-tête et de pied de page (header.php et footer.php) sont les fichiers qui servent à appeler l’en-tête et le pied de page. Le fichier d’en-tête et de pied de page, par exemple, contient le lien vers le système back-end wordpress.
  • index.php : Le fichier index.php est le fichier qui crée le modèle de page par défaut. Dans ce fichier, vous pouvez voir, modifier et supprimer des éléments de cette mise en page de modèle par défaut.
  • single.php : Le fichier single.php est le fichier qui crée la page de modèle de publication unique. Tout comme le modèle par défaut pour les pages mais maintenant pour les pages de publication unique.
  • format.php Le fichier format.php est le fichier qui construit le modèle de contenu-texte à partir d’une page. Donc, si vous aviez une page d’accueil et que vous la modifiiez depuis le back-end en ajoutant un texte. Ce fichier crée le balisage standard de ce texte.
  • 404.php Le fichier 404.php crée le modèle 404. Ce fichier est constitué de la mise en page de base de cette page.
  • archive.php Le fichier archive.php crée la mise en page de la page d’archive.
  • style.css Le fichier de feuille de style de base.

Ainsi, dans cette liste, vous pouvez voir tous les fichiers ** requis ** pour la configuration de votre propre thème Wordpress. Voyons maintenant quelques fichiers que vous pouvez créer si vous le souhaitez, mais qui ne sont pas obligatoires pour un thème wordpress. Ces fichiers sont principalement des fichiers modèles et d’autres extensions fonctionnelles.

Modèles de page personnalisés
page-<your own name>.php : dans un thème Wordpress, vous pouvez créer plusieurs modèles de page. en créant de nouveaux fichiers de modèle de page. Un fichier de modèle de page standard comprend les attributs de nom suivants. page nom du modèle et .php Si par exemple vous souhaitez créer un nouveau modèle de page pour votre page de blog vous pouvez l’appeler page-blog.php Wordpress lit automatiquement le fichier et ajoute le fichier au menu de choix de modèle. Assurez-vous d’avoir au moins inclus les fonctions get_header() et get_footer(). Assurez-vous également de nommer votre modèle dans un commentaire en haut du fichier en ajoutant l’exemple suivant.

<?php
    /*
     * Template Name: Homepage Template
     */
    get_header();
?>

Modèles de page de publication unique personnalisés
single-<your own name>.php : dans un thème Wordpress, tout comme le modèle de page décrit ci-dessus, vous pouvez également créer vos propres modèles de page de publication unique. Tout comme le modèle de page, le fichier se compose de trois parties single pour déclarer qu’il s’agit d’une page de publication unique <votre nom du modèle> et de l’extension de fichier .php. Tout comme les exigences minimales du modèle de page pour s’assurer que Wordpress lit le nouveau modèle, ajoutez les fonctions get_header() et get_footer(). Et bien sûr également en ajoutant le nom de votre modèle comme dans l’exemple ci-dessous

<?php

/*
* Template Name: Post Portfolio
* Template Post Type: post, page
*/

?>

Nous indiquons également le Template post type: qui représente le type de modèle dont il s’agit, dans ce cas la publication et la page.

Modèles de texte de publication personnalisés
format -<your own name>.php : dans un thème Wordpress, vous pouvez également créer des modèles de sortie de publication. Ces modèles de format sont la mise en page et le contenu d’un article. Par exemple, si dans certains cas, vous souhaitez que la publication n’affiche que le contenu ou le titre de la publication, vous pouvez utiliser ces modèles pour créer ce type d’ajustements. Étant donné que ce type de modèles ne formate que le contenu post-back-end créé par un utilisateur, nous n’avons pas besoin d’inclure get_header() et get_footer() puisque ceux-ci sont déjà définis dans les modèles de pages. Assurez-vous que votre modèle est capable de reconnaître une publication en utilisant l’exemple de base suivant.

<div>
    <article id="post-<?php the_ID(); ?>" <?php post_class(); ?>>
    </article>
</div>

Alors maintenant que nous savons quelque chose sur les fichiers de base et certains des nombreux fichiers spécifiques aux modèles, il est temps de commencer à parler des barres latérales et des widgets. À l’avenir, cela sera ajouté avec un tutoriel étape par étape sur la création d’un thème Wordpress très personnel.
