WP-CLI est un ensemble d’outils en ligne de commande pour gérer les installations de WordPress. Vous pouvez mettre à jour les plugins, configurer des installations multisites et bien plus encore, sans utiliser de navigateur Web.

Gérer les thèmes #

Obtenez une liste de thèmes.

$ wp theme list

Installez la dernière version de wordpress.org et activez

$ wp theme install twentysixteen --activate

Installer à partir d’un fichier zip local

$ wp theme install ../my-theme.zip

Installer à partir d’un fichier zip distant

$ wp theme install http://s3.amazonaws.com/bucketname/my-theme.zip?AWSAccessKeyId=123&amp;Expires=456&amp;Signature=abcdef

Obtenir les détails d’un thème installé

$ wp theme get twentysixteen --fields=name,title,version

Obtenir le statut du thème

$ wp theme status twentysixteen

Gérer les plug-ins #

Obtenir une liste de plugins

$ wp plugin list

Lister les plugins actifs sur le site.

$ wp plugin list --status=active --format=json

Répertorier les plugins sur chaque site d’un réseau.

$ wp site list --field=url | xargs -I % wp plugin list --url=%

Activer le plugin

$ wp plugin activate hello-dolly

Désactiver le plugin

$ wp plugin deactivate hello-dolly

Supprimer le plug-in

$ wp plugin delete hello-dolly

Installez la dernière version de wordpress.org et activez

$ wp plugin install bbpress --activate

Gérer WP-CLI lui-même #

Affichez la version actuellement installée.

$ wp cli version

Vérifiez les mises à jour de WP-CLI.

$ wp cli check-update

Mettez à jour WP-CLI vers la dernière version stable.

$ wp cli update

Répertorier tous les alias disponibles.

$ wp cli alias

Imprimez divers détails sur l’environnement WP-CLI.

$ wp cli info

Videz la liste des commandes installées, au format JSON.

$ wp cli cmd-dump

Téléchargez, installez, mettez à jour et gérez une installation WordPress. #

Télécharger le noyau WordPress

$ wp core download --locale=nl_NL

Installer WordPress

$ wp core install --url=example.com --title=Example --admin_user=supervisor --admin_password=strongpassword --admin_email=info@example.com

Afficher la version WordPress

$ wp core version

Transformez une installation sur un seul site en une installation multisite WordPress.

$ wp core multisite-convert

Installez WordPress multisite à partir de zéro.

$ wp core multisite-install

Gérer les utilisateurs #

Répertorier les ID utilisateur

$ wp user list --field=ID

Créez un nouvel utilisateur.

$ wp user create bob bob@example.com --role=author

Mettre à jour un utilisateur existant.

$ wp user update 123 --display_name=Mary --user_pass=marypass

Supprimer l’utilisateur 123 et réaffecter les publications à l’utilisateur 567

$ wp user delete 123 --reassign=567

Effectuez des opérations de base de données de base à l’aide des informations d’identification stockées dans wp-config.php #

Créez une nouvelle base de données.

$ wp db create

Supprimez une base de données existante.

$ wp db drop --yes

Réinitialisez la base de données actuelle.

$ wp db reset --yes

Exécute une requête SQL stockée dans un fichier.

$ wp db query < debug.sql
