Obtenez le chemin absolu du système de fichiers à la racine de l’installation de WordPress.

Paramètres #

Paramètre Détails
Aucun Cette fonction n’accepte aucun paramètre.

Différence importante entre get_home_path() et ABSTPATH #

Veuillez garder à l’esprit la différence entre ABSPATH et get_home_path() si WordPress est installé dans un sous-dossier.

La fonction get_home_path() renverra toujours un chemin sans le sous-dossier :

  • http://www.exemple.com - /var/www/htdocs/exemple
  • http://www.example.com/wp - /var/www/htdocs/example

C’est ainsi qu’il diffère de ABSPATH, qui renverra des valeurs différentes :

  • http://www.exemple.com - /var/www/htdocs/exemple
  • http://www.example.com/wp - /var/www/htdocs/example/wp

ABSPATH est d’abord défini dans wp-load.php qui sera situé à /var/www/htdocs/example/wp/wp-load.php, c’est donc de là que ABSPATH tirera sa définition.

get_home_path() vérifie si site_url et home_url diffèrent et supprime la sous-chaîne du chemin. Sinon, il renvoie la valeur ABSPATH :

function get_home_path() {
    $home    = set_url_scheme( get_option( 'home' ), 'http' );
    $siteurl = set_url_scheme( get_option( 'siteurl' ), 'http' );
    if ( ! empty( $home ) && 0 !== strcasecmp( $home, $siteurl ) ) {
        $wp_path_rel_to_home = str_ireplace( $home, '', $siteurl ); /* $siteurl - $home */
        $pos = strripos( str_replace( '\\', '/', $_SERVER['SCRIPT_FILENAME'] ), trailingslashit( $wp_path_rel_to_home ) );
        $home_path = substr( $_SERVER['SCRIPT_FILENAME'], 0, $pos );
        $home_path = trailingslashit( $home_path );
    } else {
        $home_path = ABSPATH;
    }

    return str_replace( '\\', '/', $home_path );
}

L’utiliser dans votre code #

L’appel de get_home_path() doit être fait dans un contexte où wp-admin/includes/file.php a déjà été inclus.

Par exemple, utiliser get_home_path() dans le crochet admin_init est correct, mais l’utiliser dans init ne l’est pas et entraînera une erreur fatale PHP :

Call to undefined function get_home_path()

Ce fichier n’est inclus qu’à partir du contexte d’administration (tableau de bord). Si vous en avez absolument besoin en dehors de ce contexte, vous devrez inclure le fichier vous-même avant d’appeler la fonction :

require_once(ABSPATH . 'wp-admin/includes/file.php');

Utilisation #

$path = get_home_path();

Valeur de retour :

chaîne

Chemin d’accès complet du système de fichiers à la racine de l’installation de WordPress, même s’il est installé dans un sous-dossier.

Exemple:

/var/www/htdocs/exemple
