Installation avec compositeur #

TYPO3 peut être installé uniquement avec le composeur du gestionnaire de dépendances PHP. Composer doit être disponible sur le serveur, puis un projet TYPO3 peut être démarré en utilisant la distribution de base.

composer create-project typo3/cms-base-distribution .

Cela extraira le noyau TYPO3 du référentiel git, le téléchargera sur vendor/typo3/cms/ et créera un dossier appelé web/ qui sera la racine du document du projet.

La distribution de base ne contient rien de plus que acomposer.json et un fichier .gitignore pour démarrer un projet. Ces fichiers peuvent également être créés à la main. Le composer.json pourrait ressembler à ceci :

{
    "repositories": [
        { "type": "composer", "url": "https://composer.typo3.org/" }
    ],
    "name": "typo3/cms-base-distribution",
    "description" : "TYPO3 CMS Base Distribution",
    "license": "GPL-2.0+",
    "require": {
        "typo3/cms": "^7.6"
    },
    "extra": {
        "typo3/cms": {
            "cms-package-dir": "{$vendor-dir}/typo3/cms",
            "web-dir": "web"
        }
    }
}

Lorsque ce fichier est créé à la main, la commande composer update récupérera également tous les fichiers nécessaires à un projet TYPO3.

De plus amples informations peuvent être trouvées ici:

  • [composer.typo3.org][1]
  • [usetypo3.com/typo3-and-composer][2]
  • [Compositeur dans le wiki TYPO3][3]

[1] : http://composer.typo3.org [2] : https://usetypo3.com/typo3-and-composer.html [3] : https://wiki.typo3.org/Composer
