Autoriser les scripts stockés sur votre machine à s’exécuter sans signature #

Pour des raisons de sécurité, PowerShell est configuré par défaut pour autoriser uniquement l’exécution des scripts signés. L’exécution de la commande suivante vous permettra d’exécuter des scripts non signés (vous devez exécuter PowerShell en tant qu’administrateur pour ce faire).

Set-ExecutionPolicy RemoteSigned

Une autre façon d’exécuter des scripts PowerShell consiste à utiliser Bypass comme ExecutionPolicy :

powershell.exe -ExecutionPolicy Bypass -File "c:\MyScript.ps1"

Ou depuis votre console PowerShell existante ou votre session ISE en exécutant :

 Set-ExecutionPolicy Bypass Process

Une solution de contournement temporaire pour la politique d’exécution peut également être obtenue en exécutant l’exécutable Powershell et en transmettant toute politique valide en tant que paramètre -ExecutionPolicy. La stratégie n’est en vigueur que pendant la durée de vie du processus, donc aucun accès administratif au registre n’est nécessaire.

C:\>powershell -ExecutionPolicy RemoteSigned

Il existe plusieurs autres politiques disponibles, et les sites en ligne vous encouragent souvent à utiliser Set-ExecutionPolicy Unrestricted. Cette politique reste en place jusqu’à ce qu’elle soit modifiée et abaisse la position de sécurité du système. Ce n’est pas conseillé. L’utilisation de RemoteSigned est recommandée car elle permet de stocker et d’écrire du code localement, et nécessite que le code acquis à distance soit signé avec un certificat provenant d’une racine de confiance.

Sachez également que la stratégie d’exécution peut être appliquée par la stratégie de groupe, de sorte que même si la stratégie est modifiée sur “Illimité” à l’échelle du système, la stratégie de groupe peut rétablir ce paramètre lors de son prochain intervalle d’application (généralement 15 minutes). Vous pouvez voir la politique d’exécution définie dans les différentes étendues à l’aide de Get-ExecutionPolicy -List

Documentation technique : Set-ExecutionPolicy about_Execution_Policies

Alias ​​et fonctions similaires #

Dans PowerShell, il existe de nombreuses façons d’obtenir le même résultat. Cela peut être bien illustré avec l’exemple simple et familier “Hello World” :

Utilisation de Write-Host :

Write-Host "Hello World"

Utilisation de Write-Output :

Write-Output 'Hello world'

Il convient de noter que bien que Write-Output et Write-Host écrivent tous les deux à l’écran, il existe une différence subtile. Write-Host écrit uniquement sur stdout (c’est-à-dire l’écran de la console), tandis que Write-Output écrit à la fois sur stdout ET sur le flux de sortie [success] permettant la [redirection] (https://blogs. technet.microsoft.com/heyscriptingguy/2014/03/30/understanding-streams-redirection-and-write-host-in-powershell/). La redirection (et les flux en général) permet de diriger la sortie d’une commande comme entrée vers une autre, y compris l’affectation à une variable.

> $message = Write-Output "Hello World"
> $message
"Hello World"

Ces fonctions similaires ne sont pas des alias, mais peuvent produire les mêmes résultats si l’on veut éviter de “polluer” le flux de réussite.

Write-Output est aliasé à Echo ou Write

Echo 'Hello world'
Write 'Hello world'

Ou, en tapant simplement “Hello world” !

'Hello world'

Tout cela se traduira par la sortie de console attendue

Hello world

Un autre exemple d’alias dans PowerShell est le mappage commun des anciennes commandes d’invite de commande et des commandes BASH aux applets de commande PowerShell. Tous les éléments suivants produisent une liste de répertoires du répertoire actuel.

C:\Windows> dir
C:\Windows> ls
C:\Windows> Get-ChildItem

Enfin, vous pouvez créer votre propre alias avec l’applet de commande Set-Alias ​​! Par exemple, appelons “Test-NetConnection”, qui est essentiellement l’équivalent PowerShell de la commande ping de l’invite de commande, sur “ping”.

Set-Alias -Name ping -Value Test-NetConnection

Vous pouvez maintenant utiliser ping au lieu de Test-NetConnection ! Sachez que si l’alias est déjà utilisé, vous écraserez l’association.

L’alias sera actif jusqu’à ce que la session soit active. Une fois que vous avez fermé la session et essayé d’exécuter l’alias que vous avez créé lors de votre dernière session, cela ne fonctionnera pas. Pour surmonter ce problème, vous pouvez importer tous vos alias d’un fichier Excel dans votre session une fois, avant de commencer votre travail.

Le pipeline - Utilisation de la sortie d’une applet de commande PowerShell #

L’une des premières questions que les gens se posent lorsqu’ils commencent à utiliser PowerShell pour les scripts est de savoir comment manipuler la sortie d’une applet de commande pour effectuer une autre action.

Le symbole de pipeline | est utilisé à la fin d’une applet de commande pour prendre les données qu’il exporte et les transmettre à l’applet de commande suivante. Un exemple simple utilise Select-Object pour afficher uniquement la propriété Name d’un fichier affiché à partir de Get-ChildItem :

Get-ChildItem | Select-Object Name
#This may be shortened to:
gci | Select Name

Une utilisation plus avancée du pipeline nous permet de canaliser la sortie d’une applet de commande dans une boucle foreach :

Get-ChildItem | ForEach-Object {
    Copy-Item -Path $_.FullName -destination C:\NewDirectory\ 
}

#This may be shortened to:
gci | % { Copy $_.FullName C:\NewDirectory\ }

Notez que l’exemple ci-dessus utilise la variable automatique $. $ est l’alias court de $PSItem qui est une variable automatique qui contient l’élément actuel dans le pipeline.

Appel des méthodes de la bibliothèque .Net #

Les méthodes statiques de la bibliothèque .Net peuvent être appelées à partir de PowerShell en encapsulant le nom complet de la classe dans la troisième parenthèse, puis en appelant la méthode à l’aide de ::

#calling Path.GetFileName()
C:\> [System.IO.Path]::GetFileName('C:\Windows\explorer.exe')
explorer.exe

Les méthodes statiques peuvent être appelées à partir de la classe elle-même, mais l’appel de méthodes non statiques nécessite une instance de la classe .Net (un objet).

Par exemple, la méthode AddHours ne peut pas être appelée à partir de la classe System.DateTime elle-même. Il nécessite une instance de la classe :

C:\> [System.DateTime]::AddHours(15)
Method invocation failed because [System.DateTime] does not contain a method named 'AddHours'.
At line:1 char:1
+ [System.DateTime]::AddHours(15)
+ ~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~
    + CategoryInfo          : InvalidOperation: (:) [], RuntimeException
    + FullyQualifiedErrorId : MethodNotFound

Dans ce cas, nous [créons d’abord un objet](https://www.wikiod.com/fr/powershell/premiers-pas-avec-powershell#Création d’objets), par exemple :

C:\> $Object = [System.DateTime]::Now

Ensuite, nous pouvons utiliser des méthodes de cet objet, même des méthodes qui ne peuvent pas être appelées directement depuis la classe System.DateTime, comme la méthode AddHours :

C:\> $Object.AddHours(15)

Monday 12 September 2016 01:51:19

Installation ou configuration #

Les fenêtres #

PowerShell est inclus avec Windows Management Framework. L’installation et la configuration ne sont pas requises sur les versions modernes de Windows.

Les mises à jour de PowerShell peuvent être effectuées en installant une version plus récente de Windows Management Framework.

Autres plates-formes #

La version “bêta” de PowerShell 6 peut être installée sur d’autres plates-formes. Les packages d’installation sont disponibles [ici][1].

Par exemple, PowerShell 6, pour Ubuntu 16.04, est publié dans les référentiels de packages pour une installation (et des mises à jour) facile.

Pour installer, exécutez la commande suivante :

# Import the public repository GPG keys
curl https://packages.microsoft.com/keys/microsoft.asc | sudo apt-key add -

# Register the Microsoft Ubuntu repository
curl https://packages.microsoft.com/config/ubuntu/16.04/prod.list | sudo tee /etc/apt/sources.list.d/microsoft.list

# Update apt-get
sudo apt-get update

# Install PowerShell
sudo apt-get install -y powershell

# Start PowerShell
powershell

Après avoir enregistré une fois le référentiel Microsoft en tant que superutilisateur, à partir de là, il vous suffit d’utiliser sudo apt-get upgrade powershell pour le mettre à jour. Ensuite, exécutez simplement powershell

[1] : https://github.com/PowerShell/PowerShell#get-powershell

Commentant #

Pour commenter les scripts de puissance en ajoutant la ligne au début à l’aide du symbole # (dièse)

# This is a comment in powershell
Get-ChildItem

Vous pouvez également avoir des commentaires sur plusieurs lignes en utilisant <# et #> respectivement au début et à la fin du commentaire.

<#
This is a 
multi-line
comment
#>
Get-ChildItem

Création d’objets #

L’applet de commande “New-Object” est utilisée pour créer un objet.

# Create a DateTime object and stores the object in variable "$var"
$var = New-Object System.DateTime

# calling constructor with parameters
$sr = New-Object System.IO.StreamReader -ArgumentList "file path"

Dans de nombreux cas, un nouvel objet sera créé afin d’exporter des données ou de le transmettre à un autre commandlet. Cela peut être fait comme ceci :

$newObject = New-Object -TypeName PSObject -Property @{
    ComputerName = "SERVER1"
    Role = "Interface"
    Environment = "Production"
}

Il existe plusieurs façons de créer un objet. La méthode suivante est probablement la méthode la plus courte et la plus rapide pour créer un PSCustomObject :

$newObject = [PSCustomObject]@{
    ComputerName = 'SERVER1'
    Role         = 'Interface'
    Environment  = 'Production'
}

Si vous avez déjà un objet, mais que vous n’avez besoin que d’une ou deux propriétés supplémentaires, vous pouvez simplement ajouter cette propriété en utilisant Select-Object :

Get-ChildItem | Select-Object FullName, Name, 
    @{Name='DateTime'; Expression={Get-Date}}, 
    @{Name='PropertieName'; Expression={'CustomValue'}}

Tous les objets peuvent être stockés dans des variables ou passés dans le pipeline. Vous pouvez également ajouter ces objets à une collection, puis afficher les résultats à la fin.

Les collections d’objets fonctionnent bien avec Export-CSV (et Import-CSV). Chaque ligne du CSV est un objet, chaque colonne une propriété.

Les commandes de format convertissent les objets en flux de texte pour l’affichage. Évitez d’utiliser les commandes Format-* jusqu’à la dernière étape de tout traitement de données, afin de maintenir la convivialité des objets.
