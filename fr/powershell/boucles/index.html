Une boucle est une séquence d’instruction(s) qui est continuellement répétée jusqu’à ce qu’une certaine condition soit atteinte. Pouvoir faire en sorte que votre programme exécute à plusieurs reprises un bloc de code est l’une des tâches les plus élémentaires mais les plus utiles en programmation. Une boucle vous permet d’écrire une instruction très simple pour produire un résultat nettement supérieur simplement par répétition. Si la condition a été atteinte, l’instruction suivante “passe” à l’instruction séquentielle suivante ou se branche en dehors de la boucle.

Syntaxe #

  • for ( <Initialisation>; <Condition>; <Répétition> ) { <Script_Block> }

  • <Collection> | Foreach-Object { <Script_Block_with__$__as_current_item> }

  • foreach ( <Item> in <Collection> ) { <Script_Block> }

  • while ( <Condition> ){ <Script_Block> }

  • faire { <Script_Block> } tandis que ( <Condition> )

  • faire { <Script_Block> } jusqu’à ( <Condition> )

  • <Collection>.foreach( { <Script_Block_with__$__as_current_item> } )

Pour chaque #

Il existe plusieurs façons d’exécuter une boucle foreach dans PowerShell et elles apportent toutes leurs propres avantages et inconvénients :

Solution Avantages Inconvénients
Instruction Foreach Le plus rapide. Fonctionne mieux avec les collections statiques (stockées dans une variable). Aucune entrée ou sortie de pipeline
Méthode ForEach() Même syntaxe de bloc de script que Foreach-Object, mais plus rapide. Fonctionne mieux avec les collections statiques (stockées dans une variable). Prend en charge la sortie du pipeline. Pas de prise en charge de l’entrée de pipeline. Nécessite PowerShell 4.0 ou supérieur
Foreach-Object (applet de commande) Prend en charge l’entrée et la sortie du pipeline. Prend en charge les blocs de script de début et de fin pour l’initialisation et la fermeture des connexions, etc. La solution la plus flexible. Le plus lent

Performance #

$foreach = Measure-Command { foreach ($i in (1..1000000)) { $i * $i } }
$foreachmethod = Measure-Command { (1..1000000).ForEach{ $_ * $_ } }
$foreachobject = Measure-Command { (1..1000000) | ForEach-Object { $_ * $_ } }

"Foreach: $($foreach.TotalSeconds)"
"Foreach method: $($foreachmethod.TotalSeconds)"
"ForEach-Object: $($foreachobject.TotalSeconds)"

Example output:

Foreach: 1.9039875
Foreach method: 4.7559563
ForEach-Object: 10.7543821

Bien que Foreach-Object soit le plus lent, son support de pipeline peut être utile car il vous permet de traiter les éléments à mesure qu’ils arrivent (lors de la lecture d’un fichier, de la réception de données, etc.). Cela peut être très utile lorsque vous travaillez avec des données volumineuses et une mémoire insuffisante, car vous n’avez pas besoin de charger toutes les données en mémoire avant le traitement.

Pour chaque #

ForEach a deux significations différentes dans PowerShell. L’un est un mot-clé et l’autre est un alias pour le [ForEach-Object] (https://www.wikiod.com/fr/powershell/boucles#Pour chaque objet) applet de commande. Le premier est décrit ici.

Cet exemple illustre l’impression de tous les éléments d’un tableau sur l’hôte de la console :

$Names = @('Amy', 'Bob', 'Celine', 'David')

ForEach ($Name in $Names)
{
    Write-Host "Hi, my name is $Name!"
}

Cet exemple illustre la capture de la sortie d’une boucle ForEach :

$Numbers = ForEach ($Number in 1..20) {
    $Number # Alternatively, Write-Output $Number
}

Comme le dernier exemple, cet exemple montre plutôt la création d’un tableau avant de stocker la boucle :

$Numbers = @()
ForEach ($Number in 1..20)
{
    $Numbers += $Number
}

Pour #

for($i = 0; $i -le 5; $i++){
    "$i"
}

Une utilisation typique de la boucle for consiste à opérer sur un sous-ensemble des valeurs d’un tableau. Dans la plupart des cas, si vous souhaitez itérer toutes les valeurs d’un tableau, envisagez d’utiliser une instruction foreach.

Méthode ForEach() #

Au lieu de l’applet de commande ForEach-Object, il est également possible d’utiliser une méthode ForEach directement sur des tableaux d’objets comme ceci

(1..10).ForEach({$_ * $_})

ou - si vous le souhaitez - les parenthèses autour du bloc de script peuvent être omises

(1..10).ForEach{$_ * $_}

Les deux entraîneront la sortie ci-dessous

1
4
9
16
25
36
49
64
81
100

ForEach-Objet #

L’applet de commande ForEach-Object fonctionne de manière similaire à l’instruction [foreach](https://www.wikiod.com/fr/powershell/boucles#Pour chaque), mais prend son entrée du pipeline .

Utilisation de base #

$object | ForEach-Object {
    code_block
}

Exemple:

$names = @("Any","Bob","Celine","David")
$names | ForEach-Object {
    "Hi, my name is $_!"
}

Foreach-Object a deux alias par défaut, foreach et % (syntaxe abrégée). Le plus courant est % car foreach peut être confondu avec l’[instruction foreach](https://www.wikiod.com/fr/powershell/boucles#Pour chaque). Exemples:

$names | % {  
    "Hi, my name is $_!"
} 

$names | foreach {  
    "Hi, my name is $_!"
}

Utilisation avancée #

Foreach-Object se distingue des solutions alternatives foreach car il s’agit d’une applet de commande, ce qui signifie qu’elle est conçue pour utiliser le pipeline. Pour cette raison, il prend en charge trois blocs de script, tout comme une applet de commande ou une fonction avancée :

  • Begin : exécuté une fois avant de parcourir les éléments qui arrivent du pipeline. Généralement utilisé pour créer des fonctions à utiliser dans la boucle, créer des variables, ouvrir des connexions (base de données, web +) etc.
  • Processus : exécuté une fois par article arrivé du pipeline. “Normal” pour chaque bloc de code. Il s’agit de la valeur par défaut utilisée dans les exemples ci-dessus lorsque le paramètre n’est pas spécifié.
  • Fin : exécuté une fois après le traitement de tous les éléments. Habituellement utilisé pour fermer les connexions, générer un rapport, etc.

Exemple:

"Any","Bob","Celine","David" | ForEach-Object -Begin {
    $results = @()
} -Process {
    #Create and store message
    $results += "Hi, my name is $_!"
} -End {
    #Count messages and output
    Write-Host "Total messages: $($results.Count)"
    $results
}

Continuer #

L’opérateur Continue fonctionne dans les boucles For, ForEach, While et Do. Il ignore l’itération actuelle de la boucle, sautant au sommet de la boucle la plus interne.

$i =0
while ($i -lt 20) {
    $i++ 
    if ($i -eq 7) { continue }
    Write-Host $I
}

Ce qui précède affichera 1 à 20 sur la console mais manquera le numéro 7.

Remarque : Lorsque vous utilisez une boucle de pipeline, vous devez utiliser return au lieu de Continue.

Casser #

L’opérateur break sortira immédiatement d’une boucle de programme. Il peut être utilisé dans les boucles For, ForEach, While et Do ou dans une instruction Switch.

$i = 0
while ($i -lt 15) {
    $i++ 
    if ($i -eq 7) {break}
    Write-Host $i
}

Ce qui précède comptera jusqu’à 15 mais s’arrêtera dès que 7 sera atteint.

Remarque : lors de l’utilisation d’une boucle de pipeline, break se comportera comme continue. Pour simuler break dans la boucle du pipeline, vous devez incorporer une logique supplémentaire, une applet de commande, etc. Il est plus facile de s’en tenir à des boucles sans pipeline si vous devez utiliser break.

Étiquettes de rupture

Break peut aussi appeler un label qui a été placé devant l’instanciation d’une boucle :

$i = 0
:mainLoop While ($i -lt 15) {
    Write-Host $i -ForegroundColor 'Cyan'
    $j = 0
    While ($j -lt 15) {
        Write-Host $j -ForegroundColor 'Magenta'
        $k = $i*$j
        Write-Host $k -ForegroundColor 'Green'
        if ($k -gt 100) {
            break mainLoop
        }
        $j++
    }
    $i++
}

Remarque : Ce code incrémentera $i à 8 et $j à 13, ce qui fera que $k sera égal à 104. Puisque $k dépasse 100, le code sortira alors des deux boucles.

Alors que #

Une boucle while évaluera une condition et si vrai effectuera une action. Tant que la condition est évaluée comme vraie, l’action continuera d’être effectuée.

while(condition){
  code_block
}

L’exemple suivant crée une boucle qui décomptera de 10 à 0

$i = 10
while($i -ge 0){
    $i
    $i--
}

Contrairement à la boucle Do-While, la condition est évaluée avant la première exécution de l’action. L’action ne sera pas exécutée si la condition initiale est fausse.

Remarque : Lors de l’évaluation de la condition, PowerShell traitera l’existence d’un objet de retour comme vrai. Cela peut être utilisé de plusieurs manières, mais vous trouverez ci-dessous un exemple pour surveiller un processus. Cet exemple générera un processus de bloc-notes, puis mettra en veille le shell actuel tant que ce processus est en cours d’exécution. Lorsque vous fermez manuellement l’instance du bloc-notes, la condition while échouera et la boucle se rompra.

Start-Process notepad.exe
while(Get-Process notepad -ErrorAction SilentlyContinue){
  Start-Sleep -Milliseconds 500
}

Fais #

Les boucles sont utiles lorsque vous souhaitez toujours exécuter un bloc de code au moins une fois. Une boucle Do évaluera la condition après avoir exécuté le bloc de code, contrairement à une boucle while qui le fait avant d’exécuter le bloc de code.

Vous pouvez utiliser les do-loops de deux manières :

  • Boucle tandis que la condition est vraie :

    Do {
    code_block
} while (condition)

  • Boucler jusqu’à la condition est vraie, autrement dit, boucler tant que la condition est fausse :

    Do {
    code_block
} until (condition)

Exemples concrets :

$i = 0

Do {
    $i++
    "Number $i"
} while ($i -ne 3)

Do {
    $i++
    "Number $i"
} until ($i -eq 3)

Do-While et Do-Until sont des boucles antonymes. Si le code est identique, la condition sera inversée. L’exemple ci-dessus illustre ce comportement.
