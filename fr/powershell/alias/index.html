Alias

  • Get-Alias
  • Set-Alias

Sur cette page

  • Get-Alias
  • Set-Alias

Le système de nommage Powershell a des règles assez strictes pour nommer les applets de commande (modèle Verb-Noun ; voir [sujet pas encore créé] pour plus d’informations). Mais il n’est pas vraiment pratique d’écrire Get-ChildItems chaque fois que vous souhaitez lister les fichiers du répertoire de manière interactive. Par conséquent, Powershell permet d’utiliser des raccourcis - des alias - au lieu de noms d’applet de commande.

Vous pouvez écrire ls, dir ou gci au lieu de Get-ChildItem et obtenir le même résultat. Alias ​​est équivalent à son applet de commande.

Certains des alias courants sont :

alias applet de commande
%, pour chaque Pour-EachObject
?, où Où-Objet
chat, gc, type Obtenir-Contenu
cd, chdir, sl Set-Emplacement
cls, clair Clear-Hôte
cp, copie, cpi Copier l’élément
répertoire/ls/gci Get-ChildItem
écho, écrire Sortie d’écriture
fl Format-Liste
ft Format-Table
fw Format large
gc, pwd Get-Location
gm Get-Membre
iex Invoquer-Expression
ii Invoke-Item
mv, bouge Move-Item
rm, rmdir, sup, effacer, rd, ri Supprimer l’élément
dormir Démarrer-Veille
début, sape Démarrer-Processus

Dans le tableau ci-dessus, vous pouvez voir comment les alias ont permis de simuler des commandes connues d’autres environnements (cmd, bash), d’où une découverte accrue.

Get-Alias #

Pour répertorier tous les alias et leurs fonctions :

Get-Alias

Pour obtenir tous les alias pour une applet de commande spécifique :

PS C:\> get-alias -Definition Get-ChildItem

CommandType     Name                                               Version    Source
-----------     ----                                               -------    ------
Alias           dir -> Get-ChildItem
Alias           gci -> Get-ChildItem
Alias           ls -> Get-ChildItem

Pour rechercher des alias par correspondance :

PS C:\> get-alias -Name p*

CommandType     Name                                               Version    Source
-----------     ----                                               -------    ------
Alias           popd -> Pop-Location
Alias           proc -> Get-Process
Alias           ps -> Get-Process
Alias           pushd -> Push-Location
Alias           pwd -> Get-Location

Set-Alias #

Cette cmdlet vous permet de créer de nouveaux noms alternatifs pour quitter les cmdlets

PS C:\> Set-Alias -Name proc -Value Get-Process
PS C:\> proc

Handles  NPM(K)    PM(K)      WS(K) VM(M)   CPU(s)     Id  SI ProcessName
-------  ------    -----      ----- -----   ------     --  -- -----------
    292      17    13052      20444 ...19     7.94    620   1 ApplicationFrameHost
....

Gardez à l’esprit que tout alias que vous créez ne sera conservé que dans la session en cours. Lorsque vous démarrez une nouvelle session, vous devez recréer vos alias. Les profils Powershell (voir [sujet pas encore créé]) sont parfaits à ces fins.
