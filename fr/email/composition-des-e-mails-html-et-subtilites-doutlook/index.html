    • Création d’espaceurs #

    Certains clients de messagerie (notamment Windows Desktop Outlook) réduisent les tableaux sans contenu, même s’il y a des dimensions ou un rembourrage.

    Pour empêcher ces clients de réduire la table d’espacement, vous pouvez ajouter un “ ” invisible qui sert de contenu. Ensuite, mettez à zéro la font-size et la line-height du caractère afin que cela n’affecte pas les dimensions de <table>. L’attribut hauteur (ou largeur) est maintenant la seule chose affectant les dimensions de l’espaceur.

    <table border="0" cellpadding="0" cellspacing="0" role="presentation">
    <tr>
        <td height="40" style="font-size: 0; line-height: 0;">
            &nbsp;
        </td>
    </tr>
</table>

    Il était courant d’utiliser un .gif transparent 1x1 comme espaceur, mais cette méthode est obsolète.

    Images d’arrière-plan de cellule de table pleine largeur #

    <head>
    <style>
    v:* { behavior: url(#default#VML); display: inline-block; }
    </style>
    </head>
    <body>
        <center>
        <table width="100%" height="20">
            <tr>
                <td bgcolor="#dddddd" style="background-image:url('http://placekitten.com/g/500/300');background-repeat:no-repeat;background-position:center;" background="http://placekitten.com/g/500/300" width="100%" height="300">

                        <v:fill type="frame" src="http://placekitten.com/g/500/300" color="#ffffff" />
                    </v:rect>
                <![endif]-->
                </td>
            </tr>
            <tr>
                <td bgcolor="#33cc99">
                    <table border="0" cellpadding="5" cellspacing="0"><tr><td height="5"><table border="0" cellpadding="0" cellspacing="0"><tr><td></td></tr></table></td></tr></table>
                </td>
            </tr>

                <td bgcolor="#ffff99" style="background-image:url('http://placekitten.com/g/500/300');background-repeat:no-repeat;background-position:center;" background="http://placekitten.com/g/500/300" width="100%" height="300">

    Espacement des éléments #

    Outlook peut parfois ajouter un peu d’espacement à gauche et à droite d’un élément

    qui peut causer des maux de tête liés à la mise en page. En utilisant les propriétés CSS mso-table-lspace et mso-table-rspace spécifiques au fournisseur, vous pouvez vous débarrasser de ces espaces et continuer à résoudre les millions d’autres problèmes causés par Outlook.

    table{
    mso-table-lspace:0pt;
    mso-table-rspace:0pt;
}

    Redimensionnement d’image #

    L’utilisation de balises de largeur ou de hauteur pour redimensionner les images dans votre balisage peut créer un problème dans les navigateurs Internet Explorer. Si votre lecteur visualise un e-mail dans le navigateur et que cet e-mail contient des images fluides, elles auront l’air assez moche lorsqu’elles seront redimensionnées. Utilisation de -ms-interpolation-mode:bicubic ; veille à ce que vos images soient un peu meilleures.

    img{
    -ms-interpolation-mode:bicubic;
}
