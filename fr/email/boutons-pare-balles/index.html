Les boutons peuvent être l’une des parties les plus frustrantes du développement de messagerie, mais ils peuvent être construits avec du code au lieu d’images et s’affichent toujours bien dans tous les principaux clients de messagerie. Lorsqu’il est construit en HTML/CSS, un bouton s’affichera dans tous les clients de messagerie, même avec les images désactivées. Le contenu et le style de chaque bouton peuvent être rapidement modifiés dans le modèle HTML/CSS.

Amélioration progressive avec Media Queries

Pour les clients qui prennent en charge les requêtes multimédias, n’importe lequel des exemples ci-dessus peut être progressivement amélioré avec le bouton.

Par exemple. @media screen and (-webkit-min-device-pixel-ratio: 0) { /* Insert -webkit-targeted CSS here */ }

Ajouter plus de remplissage horizontal dans Outlook

L’ajout conditionnel d’un espace insécable de chaque côté d’un lien de bouton peut augmenter le remplissage horizontal pour Outlook. Particulièrement utile pour les boutons basés sur les bordures.

Boutons basés sur le rembourrage #

Cette technique repose sur le remplissage au niveau de la cellule du tableau pour structurer le bouton, et sur les attributs HTML et CSS pour styliser le bouton. Pas de VML signifie un code plus simple, mais pas d’images d’arrière-plan dans Outlook.

<table width="100%" border="0" cellspacing="0" cellpadding="0">
  <tr>
    <td>
      <table border="0" cellspacing="0" cellpadding="0">
        <tr>
          <td bgcolor="#0095FF" style="padding: 12px 18px 12px 18px; -webkit-border-radius:3px; border-radius:3px" align="center"><a href="http://www.stackoverflow.com" target="_blank" style="font-size: 14px; font-family: sans-serif; font-weight: normal; color: #ffffff; text-decoration: none; display: inline-block;">Stack Overflow</a></td>
        </tr>
      </table>
    </td>
  </tr>
</table>

[![entrez la description de l’image ici][1]][1]

L’inconvénient des boutons basés sur le rembourrage est que toute la zone du bouton n’est pas cliquable, seul le lien dans la cellule du tableau l’est.

[1] : https://i.stack.imgur.com/lVUYu.jpg

Boutons basés sur les bordures #

Cette technique repose sur l’ajout de bordures épaisses au lien lui-même pour créer le CTA du bouton. L’utilisation des bordures est universellement comprise par les clients de messagerie, mais limite l’apparence des boutons en couleurs unies.

<table width="100%" border="0" cellspacing="0" cellpadding="0">
  <tr>
    <td>
      <table border="0" cellspacing="0" cellpadding="0">
        <tr>
          <td bgcolor="#0095FF" style="padding: 12px 18px 12px 18px; -webkit-border-radius:3px; border-radius:3px" align="center"><a href="http://www.stackoverflow.com" target="_blank" style="font-size: 14px; font-family: sans-serif; font-weight: normal; color: #ffffff; text-decoration: none; display: inline-block;">Stack Overflow</a></td>
        </tr>
      </table>
    </td>
  </tr>
</table>

[![entrez la description de l’image ici][1]][1]

La balise de lien est définie comme un élément de niveau bloc et les bordures sont utilisées pour fournir le rembourrage, de sorte que l’ensemble du bouton est survolable + cliquable, même dans les anciens clients de bureau.

L’inconvénient des boutons basés sur les bordures est qu’Outlook réduira légèrement la taille des bordures et, comme mentionné ci-dessus, les images d’arrière-plan ne sont pas prises en charge.

[1] : https://i.stack.imgur.com/bE0S2.jpg

Rembourrage + Boutons basés sur les bordures #

Cette technique utilise une combinaison de boutons basés sur la bordure et basés sur le rembourrage, stylisant le lien avec à la fois un rembourrage et au moins une bordure solide de 1px. La couleur d’arrière-plan doit être appliquée au au lieu du dans cette instance car Outlook reconnaît le remplissage horizontal sur la balise (puisqu’il ne s’agit pas d’une balise de niveau bloc).

<table width="100%" border="0" cellspacing="0" cellpadding="0">
  <tr>
    <td>
      <table border="0" cellspacing="0" cellpadding="0">
        <tr>
          <td align="center" style="-webkit-border-radius: 3px; -moz-border-radius: 3px; border-radius: 3px;" bgcolor="#0095FF"><a href="https://www.stackoverflow.com" target="_blank" style="font-size: 14px; font-family: sans-serif; color: #ffffff; text-decoration: none; text-decoration: none; -webkit-border-radius: 3px; -moz-border-radius: 3px; border-radius: 3px; padding: 12px 18px; border: 1px solid #0095FF; display: inline-block;">Stack Overflow</a></td>
        </tr>
      </table>
    </td>
  </tr>
</table>

[![entrez la description de l’image ici][1]][1]

L’inconvénient de padding+border-based-buttons est qu’il sépare le style entre une balise <td> et <a>, il peut donc être difficile à maintenir. Mais un seul bouton est utilisé, des dimensions spécifiques ne sont pas requises et les images d’arrière-plan peuvent être prises en charge.

[1] : https://i.stack.imgur.com/mAB7b.jpg

Bouton pare-balles avec entretoises #

Ce bouton dérive de cet exemple par Email on Acid. Il est entièrement basé sur le code, il s’affichera donc sans images téléchargées, et l’ensemble du bouton est survolable + cliquable.

De plus, cet exemple inclut également des entretoises pour aider à contrôler la quantité d’espace vertical qui apparaît avant et après le bouton.

<table width="100%">
    <tr>
      <td>
        <!-- Top Spacer : BEGIN -->
        <table border="0" cellpadding="0" cellspacing="0"> 
          <tr>
            <td height="20" width="100%" style="font-size: 20px; line-height: 20px;">
            &nbsp;
            </td>
          </tr>
        </table>
        <!-- Top Spacer : END -->
        <!-- Button : BEGIN -->
        <table border="0" align="left" cellpadding="0" cellspacing="0">
          <tbody>
          <tr>
            <td align="left">
              <table border="0" cellpadding="0" cellspacing="0" width="150">
                <tr>
                  <td align="center" bgcolor="#0077CC" width="150" style="-moz-border-radius: 4px; -webkit-border-radius: 4px; border-radius: 4px;">
                    <a href="http://www.stackoverflow.com" style="padding: 10px; width:150px; display: block;text-decoration: none; border:0;text-align: center; font-weight: bold; font-size: 14px; font-family: sans-serif; color: #ffffff; background: #0095FF;border: 1px solid #0077CC; -moz-border-radius: 4px; -webkit-border-radius: 4px; border-radius: 4px; line-height:17px;" class="button_link">Stack Overflow</a>
                  </td>
                </tr>
              </table>
            </td>
          </tr>
          </tbody>
        </table>
        <!-- Button : END -->
        <!-- Bottom Spacer : BEGIN -->
        <table border="0" cellpadding="0" cellspacing="0"> 
          <tr>
            <td height="20" width="100%" style="font-size: 20px; line-height: 20px;">
            &nbsp;
            </td>
          </tr>
        </table>
        <!-- Bottom Spacer : END -->
      </td>
    </tr>
  </table>
