_.identité #

Cette méthode ne renvoie que le premier argument.

var res1 = _.identity(10, 20);
// res1 now is 10

var res2 = _.identity("hello", "world");
// res2 now is "hello"

Que signifie _.identity dans la documentation lodash ? #

Cette méthode est utilisée dans toute la documentation lodash au lieu de function(x){return x;} (ou équivalent ES6 x => x).

Cela signifie généralement soit “aucune transformation”, soit lorsqu’il est utilisé comme prédicat : la véracité de la valeur.

Exemple de _.identity dans la documentation de _.times #

La fonction _.times prend deux arguments. C’est exprimé comme ceci dans la documentation : var res = .times(n, [iteratee=.identity])

Le iteratee est utilisé pour transformer les valeurs au fur et à mesure qu’elles sont itérées.

La documentation montre que le paramètre iteratee est facultatif, et s’il est omis, il aura la valeur par défaut de _.identity, qui dans ce cas signifie “pas de transformation”

var res = _.times(5);      // returns [0, 1, 2, 3, 4]

// means the same as:
var res = _.times(5, _.identity);

// which again does the same as:
var res = _.times(5, function(x){ return x; });

// or with the ES6 arrow syntax:
var res = _.times(5, x => x);

Exemple de _.identity dans la documentation de _.findKey et _.findLastKey #

Les fonctions _.findKey et _.findLastKey prennent deux arguments. Il est exprimé comme ceci dans la documentation : .findKey(object, [predicate=.identity]) et .findLastKey(object, [predicate=.identity])

Cela signifie à nouveau que le deuxième paramètre est facultatif, et s’il est omis, il aura la valeur par défaut de _.identity, ** qui dans ce cas signifie le premier (ou le dernier) de “tout ce qui est vrai” **

var p = {
    name: "Peter Pan",
    age: "Not yet a grownup",
    location: "Neverland"
};

var res1 = _.findKey(p);        // returns "name"
var res2 = _.findLastKey(p);    // returns "location"

// means the same as:
var res1 = _.findKey(p, _.identity);
var res2 = _.findLastKey(p, _.identity);

// which again means the same as:
var res1 = _.findKey(p, function(x) { return x; }
var res2 = _.findLastKey(p, function(x) { return x; }

// or with ES6 arrow syntax:
var res1 = _.findKey(p, x => x);
var res2 = _.findKey(p, x => x);
