qui #

Pour déterminer où se trouve sur votre système un exécutable dans votre chemin, utilisez la commande [which] :

$ which python
$

S’il n’y a pas de réponse, cet exécutable n’existe pas dans votre chemin. Le système vous renverra simplement une nouvelle invite sans message d’erreur. Si l’exécutable existe sur votre chemin, il affichera le répertoire où il existe réellement :

$ which ls
/bin/ls

Cela peut être utile pour déterminer pourquoi le comportement ne correspond pas aux attentes en vous assurant que vous exécutez la version de l’exécutable que vous pensez être. Par exemple, si vous avez à la fois Python 2 et Python 3 installés, ils peuvent tous deux être exécutés en tapant python dans le terminal - mais l’exécutable en cours d’exécution peut être différent de celui attendu. Comme indiqué ci-dessus, cette commande fonctionnera pour n’importe quelle commande Unix standard, qui sont toutes soutenues par des exécutables individuels.

[1] : https://linux.die.net/man/1/which

pwd - imprime le répertoire de travail #

$> pwd
/home/myUserHome
$> cd ..
$> pwd
/home

imprimera le chemin actuel vers la console.

commandes de manipulation de fichiers #

Liste des commandes qui seront introduites ici :

ls     #view contents of a directory
touch  #create new file
mkdir  #create new directory
cp     #copy contents of one file to another
mv     #move file from one location to another
rm     #delete a file or directory

ls exemples

jennifer@my_computer:~/Desktop$ ls
c++ projects    Research Paper.docx     test.cpp

affiche le répertoire courant

jennifer@my_computer:~/Desktop$ ls c++\ projects
DNA_analysis.cpp        encryption.cpp  pool_game.cpp

affiche le répertoire “projets c++”. Les caractères d’espacement dans les noms de fichiers sont saisis sous la forme “".

toucher exemple

jennifer@my_computer:~/Desktop$ ls
c++ projects    Research Paper.docx     test.cpp
jennifer@my_computer:~/Desktop$ touch ruby_test.rb
jennifer@my_computer:~/Desktop$ ls
c++ projects    Research Paper.docx     ruby_test.rb    test.cpp

exemple mkdir

jennifer@my_computer:~/Desktop$ mkdir ruby
jennifer@my_computer:~/Desktop$ ls
c++ projects    Research Paper.docx     ruby    ruby_test.rb    test.cpp
jennifer@my_computer:~/Desktop$ cd ruby
jennifer@my_computer:~/Desktop/ruby$ ls
<nothing>
jennifer@my_computer:~/Desktop/ruby

Il n’affiche pas réellement <rien>. C’est juste la façon dont je représente qu’il ne produit rien

exemples cp

jennifer@my_computer:~/Desktop/ruby$ cd ..
jennifer@my_computer:~/Desktop$ cp test.cpp c++_test.cpp
jennifer@my_computer:~/Desktop$ ls
c++ projects    c++_test.cpp    Research Paper.docx     ruby    ruby_test.rb
test.cpp

C’est lorsque le dernier argument de cp, dans ce cas “c++_test.cpp” n’est pas un répertoire existant. cp créera un fichier appelé “c++_test.cpp”, avec un contenu identique à celui de “test.cpp”. Si c++_test.cpp existait déjà, cp aurait supprimé ce qui s’y trouvait auparavant avant de copier le contenu de “test.cpp”.

jennifer@my_comptuer:~/Desktop$ ls ruby
<nothing>
jennifer@my_computer:~/Desktop$ cp ruby_test.rb ruby
jennifer@my_computer:~/Desktop$ ls ruby
ruby_test.rb

C’est ce qui se passe lorsque le dernier argument de cp, dans ce cas “ruby”, est un répertoire. cp crée un fichier avec le même nom que “ruby_test.rb”, mais dans le répertoire “ruby”.

exemples mv

jennifer@my_computer:~/Desktop$ ls
c++ projects    c++_test.cpp    Research Paper.docx     ruby    ruby_test.rb
test.cpp
jennifer@my_computer:~/Desktop$ mv ruby_test.rb ruby\ test.rb
jennifer@my_computer:~/Desktop$ ls
c++ projects    c++_test.cpp    Research Paper.docx     ruby    ruby test.rb
test.cpp

C’est ce qui se passe lorsque le dernier argument de mv, dans ce cas “ruby test.rb”, n’est pas un répertoire existant. Le fichier “ruby_test.rb” a été renommé en “ruby test.rb”. Si “ruby test.rb” existait déjà, il aurait été écrasé. Notez, encore une fois, que les espaces sont précédés d’un ‘'.

jennifer@my_computer:~/Desktop$ ls
c++ projects    c++_test.cpp    Research Paper.docx     ruby    ruby test.rb
test.cpp
jennifer@my_computer:~/Desktop$ ls c++\ projects
DNA_analysis.cpp        encryption.cpp  pool_game.cpp
jennifer@my_computer:~/Desktop$ mv test.cpp c++\ projects
jennifer@my_computer:~/Desktop$ ls
c++ projects    c++_test.cpp    Research Paper.docx     ruby    ruby test.rb
jennifer@my_computer:~/Desktop$ ls c++\ projects
DNA_analysis.cpp        encryption.cpp  pool_game.cpp   test.cpp

C’est ce qui se passe lorsque mv est un répertoire qui existait déjà. Le fichier “test.cpp” est déplacé dans le répertoire “projets c++”.

rm exemples

jennifer@my_computer:~/Desktop$ ls
c++ projects    c++_test.cpp    Research Paper.docx     ruby    ruby test.rb
jennifer@my_computer:~/Desktop$ rm c++_test.cpp
jennifer@my_computer:~/Desktop$ ls
c++ projects    Research Paper.docx     ruby    ruby test.rb

c++_test.cpp a été supprimé

jennifer@my_computer:~/Desktop$ rm c++\ projects
rm: cannot remove 'c++ projects': Is a directory
jennifer@my_computer:~/Desktop$ ls
c++ projects    Research Paper.docx     ruby    ruby test.rb

rm a une exigence supplémentaire pour supprimer les répertoires

jennifer@my_computer:~/Desktop$ rm -rf c++\ projects
jennifer@my_computer:~/Desktop$ ls
Research Paper.docx     ruby    ruby test.rb

-rf doit être ajouté pour supprimer un répertoire.

Pour en savoir plus sur ls, tapez la commande ls --help. Pour touch, tapez touch --help. De même avec les 6 commandes mentionnées ici. Cela imprime une explication détaillée de l’utilisation sans rien créer ni supprimer.

commande cd, répertoires expliqués #

michael@who-cares:~$

Le symbole ~ après le who-cares: est le répertoire courant. ~ signifie en fait le répertoire personnel de la personne. Dans ce cas, c’est /home/michael.

michael@who-cares:~$ cd Downloads
michael@who-cares:~/Downloads$

Recherche “Téléchargements” dans le répertoire courant, puis en fait le répertoire courant.

michael@who-cares:~/Downlaods$ cd /var
michael@who-cares:/var$

Étant donné que ce répertoire a commencé par un /, cela signifie rechercher dans le répertoire racine le répertoire var. Pour ceux provenant de Windows, le répertoire racine est l’équivalent de C:\. Les répertoires commençant par / sont appelés “répertoires absolus” et les répertoires qui ne le sont pas sont appelés “répertoires relatifs”

michael@who-cares:/var cd lib/dbus
michael@who-cares:/var/lib/dbus$

Le / au milieu signifie faire cd lib et une fois que c’est fait cd dbus en une seule commande.

michael@who-cares:/var/lib/dbus$ cd .
michael@who-cares:/var/lib/dbus$

. signifie en fait “le répertoire courant”. La commande cd . est fondamentalement inutile, mais . est utile pour d’autres choses.

michael@who-cares:/var/lib/dbus$ cd ..
michael@who-cares:/var/lib$

.. signifie en fait “le parent du répertoire courant”. En tant que tel, cd .. signifie “naviguer d’un répertoire vers le haut”.

michael@who-cares:/var/lib$ cd ../log/apt
michael@who-cares:/var/log/apt$

. et .. peuvent également faire partie de la chaîne /. De plus, il n’y a pas de limite à la durée.

michael@who-cares:/var/log/apt$ cd /dev/bus
michael@who-cares:/dev/bus$

La chaîne / peut même exister lorsque le répertoire démarre à la racine.

michael@who-cares:/dev/bus$ cd /
michael@who-cares:/$

cd / vous amène au répertoire racine. Je me demande ce qui se passe si vous tapez cd .. ici… (ne vous inquiétez pas. C’est sûr)

michael@who-cares:/$ cd home
michael@who-cares:/home$ cd michael
michael@who-cares:~$

Chaque utilisateur a un répertoire pour ses affaires dans le répertoire personnel. Si le répertoire courant est sous le répertoire personnel, cette partie du nom, dans ce cas /home/michael, est remplacée par ~.

michael@who-cares:~$ cd sys
michael@who-cares:/sys$ cd ~/Desktop
michael@who-cares:~/Desktop$ cd ~/..
michael@who-cares:/home$

~ peut également faire partie de la chaîne /. Il peut même être dans la même chaîne que ... Si le répertoire commence par ~, c’est un répertoire absolu comme s’il commençait par /.

Dernière chose à essayer : tapez cd sans répertoire après.

Commandes Unix de base #

$pwd

Affiche le répertoire de travail actuel.

$who

Affiche tous les utilisateurs connectés.

$who am i

Affiche le nom d’utilisateur de l’utilisateur actuel.

$date

Affiche la date système actuelle

$which <command>

Affiche le chemin de la commande spécifiée. Par exemple “$which pwd” affichera le chemin de la commande ‘pwd’.

$file <file_name>

Affiche le type de fichier spécifié (fichier normal, répertoire ou autres fichiers)

$cal

Affiche le calendrier du mois en cours.

$bc

Affiche le calcul mathématique entre deux nombres entiers de flottants. Par exemple “$bc 2+3” renvoie la somme arithmétique de 3 et 5.

$ls

Liste le contenu du répertoire.

  • $ls -l : listes au format long.

  • $ls -c : Sortie multi-colonnes.

  • $ls -f : Liste le type de fichier.

  • $ls -r : Liste récursive de tous les sous-répertoires rencontrés.

  • $ls -a : Affiche tous les fichiers, y compris les fichiers cachés.

  • $ls -i : Liste tous les fichiers avec son numéro I-Node.

    $grep [options] <input_file_names> Imprime les lignes qui contiennent une correspondance pour un motif.

  • $grep -i : Effectue une correspondance insensible à la casse

  • $grep -v : Affiche toutes les lignes qui ne contiennent pas la regex

  • $grep -r : Recherche récursivement les sous-répertoires listés et imprime les noms de fichiers avec l’occurrence du pattern

  • $grep -I : Exclut les fichiers binaires

pushd/popd (stocke le répertoire actuel sur la pile et va au répertoire dest / pop prev et y va) #

$ pwd
/home/bob/somedir1/somedir2/somedir3

$ pushd /home/bob/otherdir1/otherdir2
/home/bob/otherdir1/otherdir2 /home/bob/somedir1/somedir2/somedir3

$ popd
/home/bob/somedir1/somedir2/somedir3

$ pushd /usr
/usr /home/bob/somedir1/somedir2/somedir3

$ pushd /var
/var /usr /home/bob/somedir1/somedir2/somedir3

$ popd
/usr /home/bob/somedir1/somedir2/somedir3

$ pwd
/usr

$ popd
/home/bob/somedir1/somedir2/somedir3

$ pwd
/home/bob/somedir1/somedir2/somedir3
