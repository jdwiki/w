  • Une liste non exhaustive de commandes Unix

Cette rubrique fournira une couverture complète des commandes Unix de base.

Une liste non exhaustive de commandes Unix #

$man <command>

Affiche les pages de manuel en ligne de la commande

$clear

Efface l’écran du terminal

$pwd

Renvoie le nom du répertoire de travail

$echo <string>

Écrit la chaîne sur la sortie standard

$printf <string>

Formater et imprimer la chaîne Exemple : imprimer $PATH $printf “%s\n” $PATH

$uptime

Afficher depuis combien de temps le système fonctionne

$which <program>

Localiser un fichier de programme dans le chemin de l’utilisateur

$whereis <program>

Vérifie les répertoires binaires standard pour les programmes spécifiés, en imprimant le

DES DOSSIERS

$cd [directory]

Changer de répertoire Symboles de répertoire couramment utilisés :

  • . : Répertoire actuel

  • .. : Dossier Parent

  • ~ : Répertoire d’accueil

  • / : répertoire racine

    $ls

  • $ls -a : Afficher les fichiers cachés

  • $ls -l : Afficher la longue liste

  • $ls -1 : Affiche uniquement le nom du fichier par ligne

  • $ls -h : format lisible par l’homme

    $file Déterminer le type de fichier (par exemple, gzip)

LECTURE DE FICHIERS

$more

Afficher le contenu d’un fichier un écran à la fois
barre d’espacement : Faites défiler jusqu’à l’écran suivant ; b=écran précédent
entrer : Faire défiler d’une ligne
h : Aide pour plus
q : Quitter l’aide

$less <file>

Less est un programme similaire à more, mais qui permet de reculer dans le fichier ainsi que d’avancer

$cat <file>

Lit les fichiers de manière séquentielle, en les écrivant sur la sortie standard

$head [-number] <file>

Afficher les premières lignes d’un fichier

$tail [-number] <file>

Affiche le contenu du fichier ou, par défaut, son entrée standard, sur la sortie standard

  • $tail -f : Afficher les modifications dans le fichier en temps réel

    $touch Définit les heures de modification et d’accès des fichiers. Si un fichier n’existe pas, il est créé avec les autorisations par défaut

    $tee Copie l’entrée standard vers la sortie standard. Appuyez sur ctrl-d pour arrêter d’ajouter du contenu

  • $tee -a : Ajoute la sortie aux fichiers plutôt que de les écraser

    $mkdir Créer un répertoire

    $wc Afficher le nombre de lignes, de mots et d’octets contenus dans chaque fichier d’entrée ou entrée standard

  • wc -l : Compte les lignes

  • wc -w : Compter les mots

  • wc -m : Compte les caractères

    $diff Comparez deux fichiers ligne par ligne. N’imprimera que les différentes lignes.

    $locate Localiser les fichiers sur le disque

  • $locate -q : supprime les erreurs

    $find Rechercher des fichiers par nom ou contenu

  • $find -name : Rechercher par nom de fichier

  • $trouver -taille <+/-n> Exemple : rechercher des fichiers dans le répertoire actuel qui sont supérieurs à 10 ko $find . -size +10

    $rm Supprimer <fichier ou répertoire>

  • rm -f : Ignorer la confirmation

  • rm -i : Approuve chaque suppression

  • rm -r : Récursif

Exemple : Supprimer le <répertoire> et son contenu $rm -r 

$mv <source_file> <target_file>

Renomme un fichier

$mv <source_file> <target_directory>

Déplace un fichier

  • $mv -i : Ne remplace pas les fichiers existants
  • $mv -r : Récursif

Exemple : déplacer le répertoire vers le haut dans la hiérarchie $mv ..

$cp

Copiez un fichier/répertoire sur la même machine (utilisez la commande scp pour copier sur une machine distante)

  • $cp -i : Ne remplace pas les fichiers sortants
  • $cp -r : récursif

Exemple : copier et renommer un fichier

$cp <file_name> <new_file_name>

Exemple : copier dans le répertoire

$cp <file_name> <directory_name>

Exemple : copier et renommer un répertoire

$cp -R <directory> <new_directory>

Exemple : copier tous les fichiers d’un type spécifique dans un répertoire

$cp *.txt <directory>


$ln -s <file> <link name>

Créer un alias (lien) vers un fichier

  • $ln -s : Crée un lien logiciel (un lien qui fonctionne entre les machines)

    $sort Trier le contenu d’un fichier -r reverse sorts -n numeric sort

Exemple : trier le et écrire le résultat dans sorted.txt $sort | uniq -u > sorted.txt

$uniq [-ucd] filename(s)

Recherche les lignes en double. Les données doivent être triées en premier

  • $uniq -d : affiche une seule copie des lignes dupliquées

  • $uniq -u : Afficher uniquement les lignes non dupliquées

  • $uniq -c : Affiche chaque ligne précédée d’un nombre d’occurrences Exemple : afficher les utilisateurs connectés plusieurs fois $who | cut -d’ ‘ -f1 | sort | uniq -d

    $grep <file_name> Imprime les lignes qui contiennent une correspondance pour un motif.

  • $grep -i : Effectue une correspondance insensible à la casse

  • $grep -v : Affiche toutes les lignes qui ne contiennent pas la regex

  • $grep -r <répertoire> : recherche récursivement les sous-répertoires listés et imprime les noms de fichiers avec l’occurrence du motif

  • $grep -I : Exclut les fichiers binaires

    $tr “string1” [“string 2”] Outil de recherche et de remplacement. tr accepte uniquement son entrée à partir de canaux et de redirections. il n’accepte pas les fichiers en entrée.

  • tr -d : Supprime toutes les occurrences de tous les CHARACTERS dans string1

Exemple : imprimez un fichier .txt à l’écran après avoir supprimé toutes les occurrences de " ; " $cat a.txt | tr -d “;”

  • tr -s : Remplace les occurrences par un seul caractère

Exemple: $echo “SSSS SS” | tr -s “S” “S”

$tar

Crée et manipule des fichiers d’archives en continu. Cette implémentation peut extraire des images tar, pax, cpio, zip, jar, ar et ISO et peut créer des fichiers tar, archives pax, cpio, ar et shar.


UTILISATION DU DISQUE

$du [file or directory]

Affichez l’utilisation des blocs du système de fichiers pour chaque fichier ou répertoire. Si aucun fichier/répertoire n’est spécifié, l’utilisation du bloc du répertoire courant est affichée.

  • $du -a : Fichiers et répertoires (par défaut, seuls les répertoires)

  • $du -h : format lisible par l’homme Exemple : Afficher l’utilisation du disque, commandé, uniquement les fichiers en Mo $du -h | grep -i “m\t” | sort -n Exemple : Découvrez les 10 fichiers/répertoires les plus volumineux $du -a /var | sort -n -r | head -n 10

    $df Afficher l’espace disque libre

  • $df -h : format lisible par l’homme


REDIRECTIONS ET TUYAUX

> Rediriger la sortie standard. Ne pas écraser le fichier s’il existe

> ! Rediriger la sortie standard. Remplacer le fichier s’il existe

>& Rediriger la sortie standard et l’erreur standard

Exemple : Rediriger la sortie de la commande dans un fichier

$ls > result.txt

Utilisez le fichier > /dev/null pour éliminer le message d’erreur

Exemple : recherchez un fichier nommé my_file_name et imprimez le résultat dans ~/find.txt ; Masquer les erreurs (par exemple “autorisations refusées”)

$find / -name my_file_name.* > /dev/null > ~/find.txt

< Rediriger l’entrée standard

>> Ajouter la sortie standard <command> »  : ajoute la sortie à la fin d’un existant

<commande> < : redirige l’entrée vers une commande à partir d’un fichier

| Rediriger la sortie standard vers une autre commande (pipe)

Exemple : Afficher les détails paginés des processus en cours d’exécution

$ps -ex | more

|  : la sortie du tube de command1 doit être l’entrée de command2 (si un fichier de sortie est souhaité au milieu d’un tube, utilisez la commande tee)

Exemple : Compter le nombre d’utilisateurs connectés

$who | wc -l



PROCESSUS

$ps

Afficher les processus actifs

  • $ps -e : Afficher des informations sur le processus
  • $ps -x : Afficher les processus cachés

Exemple : rechercher des processus par nom $grep -l <process_name_regex>

$kill [-signal] pid

Tuer un processus.

Certains des signaux les plus couramment utilisés :

  • 3 : QUITTER (quitter)

  • 9 : KILL (kill non attrapable, non ignorable)

  • 15 : TERM (signal de fin de logiciel)

    $top Afficher et mettre à jour des informations triées sur les processus Voir les pages de manuel pour la liste des clés possibles. les clés communes sont : cpu, threads, ports

  • $top -o <clé>

    $htop Afficher et mettre à jour des informations triées sur les processus


    UTILISATEUR ET AUTORISATIONS

    $sudo Exécutez la commande en tant que super utilisateur

    $su (utilisateur suppléant) ouvre une session en tant qu’administrateur

    $exit Quitter la racine

    $whoami Afficher l’ID utilisateur effectif

    $who Imprimer tous les noms d’utilisateurs connectés

    $passwd Changer le mot de passe

    $chmod Modifier l’accès du propriétaire/groupe à un fichier ou un répertoire

Qui : u utilisateur ; groupe g ; o autre ; un tout au dessus

Opération : + ajouter ; - retirer; = set (ce qui signifie réinitialiser à rien et définir uniquement ce qui a été spécifié)

Autorisations : r w x

Exemple : ajoute des autorisations de lecture/exécution au groupe

$chmod g +rx <file>

Exemple:

$chmod 743 <file>

Notez que pour $cd dans un répertoire, vous avez besoin des autorisations x
