## Emacs Pour configurer Emacs pour travailler avec Clojure, installez les packages [clojure-mode][1] et cider depuis melpa :

M-x package-install [RET] clojure-mode [RET]
M-x package-install [RET] cider [RET]

Maintenant, lorsque vous ouvrez un fichier .clj, exécutez M-x cider-jack-in pour vous connecter à un REPL. Alternativement, vous pouvez utiliser C-u M-x (cider-jack-in) pour spécifier le nom d’un projet lein ou boot, sans avoir à visiter aucun fichier qu’il contient. Vous devriez maintenant pouvoir évaluer des expressions dans votre fichier en utilisant C-x C-e.

L’édition de code dans des langages de type lisp est beaucoup plus confortable avec un plugin d’édition prenant en charge les parenthèses. Emacs a plusieurs bonnes options.

  • paredit Un mode d’édition Lisp classique qui a une courbe d’apprentissage plus abrupte, mais qui fournit beaucoup de puissance une fois maîtrisé.

    M-x package-install [RET] paredit [RET]

  • smartparens Un projet plus récent avec des objectifs et une utilisation similaires à paredit, mais fournit également des capacités réduites avec des langages non Lisp.

    M-x package-install [RET] smartparens [RET]

  • parinfer Un mode d’édition Lisp beaucoup plus simple qui fonctionne principalement en inférant l’imbrication appropriée des parenthèses à partir de l’indentation.

    Installation is more involved, see the Github page for parinfer-mode for setup instructions.

Pour activer paredit en clojure-mode :

(add-hook 'clojure-mode-hook #'paredit-mode)

Pour activer smartparens en clojure-mode :

(add-hook 'clojure-mode-hook #'smartparens-strict-mode)

[1] : http://github.com/clojure-emacs/clojure-mode

IntelliJ IDEA + Cursif #

[Téléchargez][1] et installez la dernière version d’IDEA.

[Téléchargez][2] et installez la dernière version du plugin Cursive.

Après avoir redémarré IDEA, Cursive devrait fonctionner immédiatement. Suivez le [guide de l’utilisateur][3] pour affiner l’apparence, les raccourcis clavier, le style de code, etc.

Remarque : Comme [IntelliJ][1], [Cursive][1] est un produit commercial, avec une période d’évaluation de 30 jours. Contrairement à [IntelliJ][1], il n’a pas d’édition communautaire. Des licences non commerciales gratuites sont disponibles pour les particuliers pour une utilisation non commerciale, y compris le piratage personnel, l’open source et le travail des étudiants. La licence est valable 6 mois et peut être renouvelée.

[1] : https://www.jetbrains.com/idea/ [2] : https://cursive-ide.com/ [3] : https://cursive-ide.com/userguide/

Atome #

Installez Atom pour votre distribution ici.

Après cela, exécutez les commandes suivantes depuis un terminal :

apm install parinfer
apm install language-clojure
apm install proto-repl

Spacemacs + CIDRE #

[Spacemacs][] est une distribution d’emacs fournie avec de nombreux packages préconfigurés et faciles à installer. En outre, il est très convivial pour ceux qui connaissent le style d’édition vim. Spacemacs fournit une [couche Clojure basée sur CIDER][CIDER].

[Spacemacs] : http://spacemacs.org/ [CIDRE] : https://github.com/syl20bnr/spacemacs/tree/master/layers/%2Blang/clojure

Pour l’installer et le configurer pour une utilisation avec Clojure, installez d’abord emacs. Faites ensuite une sauvegarde de vos configurations précédentes :

$ mv ~/.emacs.d ~/.emacs.d.backup

Ensuite, clonez le dépôt de spacemacs :

$ git clone https://github.com/syl20bnr/spacemacs ~/.emacs.d

Maintenant, ouvrez emacs. Il vous posera quelques questions concernant vos préférences. Ensuite, il télécharge plus de packages et configure votre emacs. Une fois Spacemacs installé, vous êtes prêt à ajouter le support de Clojure. Appuyez sur SPC f e d pour ouvrir votre fichier .spacemacs pour la configuration. Trouvez dotspacemacs-configuration-layers dans le fichier, sous celui-ci se trouve un parenthèse ouvert. N’importe où entre les parenthèses dans une nouvelle ligne, tapez “clojure”.

(defun dotspacemacs/layers ()
  (setq-default
   ;; ...
   dotspacemacs-configuration-layers
   '(clojure
     ;; ...
    )
   ;; ...
))

Appuyez sur SPC f e R pour enregistrer et installer la couche de clojure. Maintenant, dans n’importe quel fichier .clj si vous appuyez sur , s i spacemacs essaiera de générer une nouvelle connexion clojure REPL à votre projet, et si cela réussit, il s’affichera dans la barre d’état, que vous pourrez ensuite appuyer sur , s s pour ouvrir un nouveau tampon REPL pour évaluer vos codes.

Pour plus d’informations sur les spacemacs et le cidre, contactez leurs documentations. [Documents Spacemacs][1], [Documents Cider][2]

[1] : http://spacemacs.org/doc/DOCUMENTATION.html [2] : http://cider.readthedocs.io/en/latest/

Vigueur #

Installez les plugins suivants à l’aide de votre gestionnaire de plugins préféré :

  1. fireplace.vim : Prise en charge de Clojure REPL
  2. vim-sexp : Pour apprivoiser ces câlins autour de vos appels de fonction
  3. vim-sexp-mappings-for-regular-people : mappages sexp modifiés un peu plus faciles à supporter
  4. [vim-surround] (https://github.com/tpope/vim-surround) : supprimez, modifiez, ajoutez facilement des “environnements” par paire
  5. salve.vim : Prise en charge statique de Vim pour Leiningen et Boot.
  6. rainbow_parentheses.vim : Meilleures parenthèses arc-en-ciel

et concernent également la coloration syntaxique, la complétion omni, la coloration avancée, etc. :

  1. vim-clojure-static (si vous avez un vim antérieur à 7.3.803, les versions plus récentes sont livrées avec)
  2. vim-clojure-highlight

D’autres options à la place de vim-sexp incluent paredit.vim et [vim-parinfer](https://github.com/bhurlow/vim- parinférer).

Table lumineuse #

Light Table est un bon éditeur pour apprendre, expérimenter et exécuter des projets Clojure.
Vous pouvez également exécuter des projets lein/boot en ouvrant le fichier project.clj. Il chargera toutes les dépendances du projet.

Il prend en charge l’évaluation en ligne, les plugins et bien plus encore, il n’est donc pas nécessaire d’ajouter des instructions d’impression et de vérifier la sortie dans la console. Vous pouvez exécuter des lignes individuelles ou des blocska de code en appuyant sur ctrl + enter. Pour exécuter un code partiel, sélectionnez le code et appuyez sur ‘ctrl + entrée’. consultez la capture d’écran suivante pour savoir comment utiliser la table lumineuse pour apprendre et expérimenter le code Clojure.

[![entrez la description de l’image ici][1]][1]

Les binaires pré-construits de Light Table peuvent être trouvés [ici][2]. Aucune configuration supplémentaire n’est requise.

Light Table est capable de localiser automatiquement votre projet Leiningen et d’évaluer votre code. Si vous n’avez pas installé Leiningen, installez-le en suivant les instructions [ici][3].

Documentation : [docs.lighttable.com][4]
