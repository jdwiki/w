Syntaxe #

  • :: est un raccourci d’un mot-clé qualifié par un espace de noms. Par exemple. si nous sommes dans l’espace de noms user : ::foo est un raccourci pour :user/foo
  • # : ou # - syntaxe littérale de carte pour qualifier les clés d’une carte par un espace de noms

Clojure [spec][1] est une nouvelle bibliothèque de spécifications/contrats pour clojure disponible à partir de la version 1.9.

Les spécifications sont exploitées de plusieurs manières, notamment en étant incluses dans la documentation, la validation des données, la génération de données pour les tests, etc.

[1] : http://clojure.org/guides/spec

Utilisation d’un prédicat comme spécification #

Toute fonction de prédicat peut être utilisée comme spécification. Voici un exemple simple :

(clojure.spec/valid? odd? 1)
;;=> true

(clojure.spec/valid? odd? 2)
;;=> false

la fonction valid? prendra une spécification et une valeur et retournera true si la valeur est conforme à la spécification et false sinon.

Un autre prédicat intéressant est l’appartenance à un ensemble :

(s/valid? #{:red :green :blue} :red) 
;;=> true

fdef : écrire une spécification pour une fonction #

Disons que nous avons la fonction suivante :

(defn nat-num-count [nums] (count (remove neg? nums)))

Nous pouvons écrire une spécification pour cette fonction en définissant une spécification de fonction du même nom :

(clojure.spec/fdef nat-num-count
        :args (s/cat :nums (s/coll-of number?))
        :ret integer?
        :fn #(<= (:ret %) (-> % :args :nums count)))

:args prend une spécification regex qui décrit la séquence d’arguments par une étiquette de mot-clé correspondant au nom de l’argument et une spécification correspondante. La raison pour laquelle la spécification requise par : args est une spécification regex est de prendre en charge plusieurs arités pour une fonction. :ret spécifie une spécification pour la valeur de retour de la fonction.

:fn est une spécification qui contraint la relation entre :args et :ret. Il est utilisé comme propriété lorsqu’il est exécuté via test.check. Elle est appelée avec un seul argument : une carte avec deux clés : :args (les arguments conformes à la fonction) et :ret (la valeur de retour conforme de la fonction).

Enregistrer une spécification #

En plus des prédicats fonctionnant comme des spécifications, vous pouvez enregistrer une spécification globalement à l’aide de clojure.spec/def. def exige qu’une spécification enregistrée soit nommée par un mot-clé qualifié par l’espace de noms :

(clojure.spec/def ::odd-nums odd?)
;;=> :user/odd-nums

(clojure.spec/valid? ::odd-nums 1)
;;=> true
(clojure.spec/valid? ::odd-nums 2)
;;=> false

Une fois enregistrée, une spécification peut être référencée globalement n’importe où dans un programme Clojure.

La syntaxe ::odd-nums est un raccourci pour :user/odd-nums, en supposant que nous sommes dans l’espace de noms user. :: qualifiera le symbole qu’il précède avec l’espace de nom actuel.

Plutôt que de transmettre le prédicat, nous pouvons transmettre le nom de la spécification à valid?, et cela fonctionnera de la même manière.

clojure.spec/et & clojure.spec/ou #

clojure.spec/and & clojure.spec/or peuvent être utilisés pour créer des spécifications plus complexes, en utilisant plusieurs spécifications ou prédicats :

(clojure.spec/def ::pos-odd (clojure.spec/and odd? pos?))

(clojure.spec/valid? ::pos-odd 1)
;;=> true

(clojure.spec/valid? ::pos-odd -3)
;;=> false

“ou” fonctionne de la même manière, avec une différence significative. Lors de la définition d’une spécification or, vous devez étiqueter chaque branche possible avec un mot-clé. Ceci est utilisé afin de fournir des branches spécifiques qui échouent dans les messages d’erreur :

(clojure.spec/def ::big-or-small (clojure.spec/or :small #(< % 10) :big #(> % 100)))

(clojure.spec/valid? ::big-or-small 1)
;;=> true

(clojure.spec/valid? ::big-or-small 150)
;;=> true

(clojure.spec/valid? ::big-or-small 20)
;;=> false

Lors de la conformité d’une spécification à l’aide de or, la spécification applicable sera renvoyée, ce qui a rendu la valeur conforme :

(clojure.spec/conform ::big-or-small 5)
;; => [:small 5]

Enregistrer les spécifications #

Vous pouvez spécifier un enregistrement comme suit :

(clojure.spec/def ::name string?)
(clojure.spec/def ::age pos-int?)
(clojure.spec/def ::occupation string?)

(defrecord Person [name age occupation])

(clojure.spec/def ::person (clojure.spec/keys :req-un [::name ::age ::occupation]))

(clojure.spec/valid? ::person (->Person "john doe" 25 "programmer"))
;;=> true

(clojure.spec/valid? ::person (->Person "john doe" "25" "programmer"))
;;=> false

À un moment donné dans le futur, une syntaxe de lecteur ou un support intégré pour qualifier les clés d’enregistrement par l’espace de noms des enregistrements pourrait être introduit. Cette prise en charge existe déjà pour les cartes.

Spécifications de la carte #

Vous pouvez spécifier une carte en spécifiant quelles clés doivent être présentes dans la carte :

(clojure.spec/def ::name string?)
(clojure.spec/def ::age pos-int?)
(clojure.spec/def ::occupation string?)

(clojure.spec/def ::person (clojure.spec/keys :req [::name ::age ::occupation]))

(clojure.spec/valid? ::person {::name "john" ::age 25 ::occupation "programmer"})
;; => true

:req est un vecteur de clés devant être présentes dans la carte. Vous pouvez spécifier des options supplémentaires telles que :opt, un vecteur de clés qui sont facultatifs.

Jusqu’à présent, les exemples exigent que les clés du nom soient qualifiées par l’espace de noms. Mais il est courant que les clés de carte ne soient pas qualifiées. Dans ce cas, clojure.spec fournit les équivalents :req et :opt pour les clés non qualifiées : :req-un et :opt-un. Voici le même exemple, avec des clés non qualifiées :

(clojure.spec/def ::name string?)
(clojure.spec/def ::age pos-int?)
(clojure.spec/def ::occupation string?)

(clojure.spec/def ::person (clojure.spec/keys :req-un [::name ::age ::occupation]))

(clojure.spec/valid? ::person {:name "john" :age 25 :occupation "programmer"})
;; => true

Remarquez comment les spécifications fournies dans le vecteur :req-un sont toujours qualifiées. clojure.spec, confirmera automatiquement les versions non qualifiées dans la carte lors de la conformité des valeurs.

La syntaxe littérale de la carte d’espace de noms vous permet de qualifier succinctement toutes les clés d’une carte par un seul espace de noms. Par exemple:

(clojure.spec/def ::name string?)
(clojure.spec/def ::age pos-int?)
(clojure.spec/def ::occupation string?)

(clojure.spec/def ::person (clojure.spec/keys :req [::name ::age ::occupation]))

(clojure.spec/valid? ::person #:user{:name "john" :age 25 :occupation "programmer"})
;;=> true

Remarquez la syntaxe spéciale du lecteur #:. Nous suivons cela avec l’espace de noms par lequel nous souhaitons qualifier toutes les clés de la carte. Celles-ci seront ensuite vérifiées par rapport aux spécifications correspondant à l’espace de noms fourni.

Collections #

Vous pouvez spécifier des collections de plusieurs façons. coll-of vous permet de spécifier des collections et de fournir des contraintes supplémentaires. Voici un exemple simple :

(clojure.spec/valid? (clojure.spec/coll-of int?) [1 2 3])
;; => true

(clojure.spec/valid? (clojure.spec/coll-of int?) '(1 2 3))
;; => true

Les options de contrainte suivent la spécification/le prédicat principal de la collection. Vous pouvez contraindre le type de collection avec :kind comme ceci :

(clojure.spec/valid? (clojure.spec/coll-of int? :kind vector?) [1 2 3])
;; => true

(clojure.spec/valid? (clojure.spec/coll-of int? :kind vector?) '(1 2 3))
;; => false

Ce qui précède est faux car la collection transmise n’est pas un vecteur.

(clojure.spec/valid? (clojure.spec/coll-of int? :kind list?) '(1 2 3))
;; => true

(clojure.spec/valid? (clojure.spec/coll-of int? :kind set?) #{1 2 3})
;; => true

(clojure.spec/valid? (clojure.spec/coll-of int? :kind set?) #{1 "2" 3})
;; => false

Ce qui précède est faux car tous les éléments de l’ensemble ne sont pas des entiers.

Vous pouvez également limiter la taille de la collection de plusieurs manières :

(clojure.spec/valid? (clojure.spec/coll-of int? :kind vector? :count 3) [1 2 3])
;; => true

    (clojure.spec/valid? (clojure.spec/coll-of int? :kind vector? :count 3) [1 2])
;; => false

(clojure.spec/valid? (clojure.spec/coll-of int? :min-count 3 :max-count 5) [1 2 3])
;; => true

    (clojure.spec/valid? (clojure.spec/coll-of int? :min-count 3 :max-count 5) [1 2])
;; => false

Vous pouvez également imposer l’unicité des éléments de la collection avec :distinct :

(clojure.spec/valid? (clojure.spec/coll-of int? :distinct true) [1 2])
;; => true

(clojure.spec/valid? (clojure.spec/coll-of int? :distinct true) [2 2])
;; => false

coll-of garantit que tous les éléments d’une séquence sont vérifiés. Pour les grandes collections, cela peut être très inefficace. every se comporte exactement comme coll-of, sauf qu’il n’échantillonne qu’un nombre relativement restreint d’éléments des séquences à partir de la conformité. Cela fonctionne bien pour les grandes collections. Voici un exemple :

(clojure.spec/valid? (clojure.spec/every int? :distinct true) [1 2 3 4 5])
;; => true

map-of est similaire à coll-of, mais pour les cartes. Étant donné que les cartes ont à la fois des clés et des valeurs, vous devez fournir à la fois une spécification pour la clé et une spécification pour la valeur :

(clojure.spec/valid? (clojure.spec/map-of keyword? string?) {:red "red" :green "green"})
;; => true

Comme coll-of, map-of vérifie la conformité de toutes les clés/valeurs de la carte. Pour les grandes cartes, cela sera inefficace. Comme coll-of, map-of fournit every-kv pour échantillonner efficacement un nombre relativement petit de valeurs à partir d’une grande carte :

(clojure.spec/valid? (clojure.spec/every-kv keyword? string?) {:red "red" :green "green"})
;; => true

Séquences #

spec peut décrire et être utilisé avec des séquences arbitraires. Il prend en charge cela via un certain nombre d’opérations de spécification de regex.

(clojure.spec/valid? (clojure.spec/cat :text string? :int int?) ["test" 1])
;;=> true

cat nécessite des étiquettes pour chaque spécification utilisée pour décrire la séquence. cat décrit une séquence d’éléments et une spécification pour chacun.

alt est utilisé pour choisir parmi un certain nombre de spécifications possibles pour un élément donné dans une séquence. Par exemple:

(clojure.spec/valid? (clojure.spec/cat :text-or-int (clojure.spec/alt :text string? :int int?)) ["test"])
;;=> true

alt exige également que chaque spécification soit étiquetée par un mot-clé.

Les séquences Regex peuvent être composées de manière très intéressante et puissante pour créer des spécifications décrivant des séquences arbitrairement complexes. Voici un exemple un peu plus complexe :

(clojure.spec/def ::complex-seq (clojure.spec/+ (clojure.spec/cat :num int? :foo-map (clojure.spec/map-of keyword? int?))))
(clojure.spec/valid? ::complex-seq [0 {:foo 3 :baz 1} 4 {:foo 4}])
;;=> true

Ici ::complex-seq validera une séquence d’une ou plusieurs paires d’éléments, le premier étant un int et le second étant une carte de mot-clé à int.
