Syntaxe #

  • Le symbole ' utilisé dans l’exemple macroexpand n’est qu’un sucre syntaxique pour l’opérateur quote. Vous auriez pu écrire (macroexpand (quote (infix 1 + 2))) à la place.

Les macros ne sont que des fonctions qui s’exécutent au moment de la compilation, c’est-à-dire pendant l’étape [eval][eval] dans une [read-eval-print-loop].

Les macros de lecteur sont une autre forme de macro qui est développée au moment de la lecture, plutôt qu’au moment de la compilation.

Meilleure pratique lors de la définition de macro.

  • renommage alpha, puisque la macro est développée, un conflit de nom de liaison peut survenir. Le conflit de liaison n’est pas très intuitif à résoudre lors de l’utilisation de la macro. C’est pourquoi, chaque fois qu’une macro ajoute une liaison à la portée, il est obligatoire d’utiliser le # à la fin de chaque symbole.

[eval] : https://clojuredocs.org/clojure.core/eval [lecture-évaluation-impression-boucle] : https://en.wikipedia.org/wiki/Read%E2%80%93eval%E2%80%93print_loop

Macro d’infixe simple #

Clojure utilise la notation préfixée, c’est-à-dire : L’opérateur vient avant ses opérandes.

Par exemple, une simple somme de deux nombres serait :

(+ 1 2)
;; => 3

Les macros vous permettent de manipuler le langage Clojure dans une certaine mesure. Par exemple, vous pouvez implémenter une macro qui vous permet d’écrire du code en notation infixe (par exemple, 1 + 2):

(defmacro infix [first-operand operator second-operand]
    "Converts an infix expression into a prefix expression"
    (list operator first-operand second-operand))

Décomposons ce que fait le code ci-dessus :

  • defmacro est une forme spéciale que vous utilisez pour définir une macro.
  • infix est le nom de la macro que nous définissons.
  • [first-operand operator second-operand] sont les paramètres que cette macro s’attend à recevoir lorsqu’elle est appelée.
  • (list operator first-operand second-operand) est le corps de notre macro. Il crée simplement une “liste” avec les valeurs des paramètres fournis à la macro “infixe” et la renvoie.

defmacro est une forme spéciale car elle se comporte un peu différemment par rapport aux autres constructions Clojure : ses paramètres ne sont pas immédiatement évalués (lorsque nous appelons la macro). C’est ce qui nous permet d’écrire quelque chose comme :

(infix 1 + 2)
;; => 3

La macro infix développera les arguments 1 + 2 en (+ 1 2), qui est une forme Clojure valide qui peut être évaluée.

Si vous voulez voir ce que génère la macro infix, vous pouvez utiliser l’opérateur macroexpand :

(macroexpand '(infix 1 + 2))
;; => (+ 1 2)

macroexpand, comme son nom l’indique, développera la macro (dans ce cas, elle utilisera la macro infix pour transformer 1 + 2 en (+ 1 2)) mais n’autorisera pas le résultat de l’expansion macro à évaluer par l’interpréteur de Clojure.

Syntaxe entre guillemets et sans guillemets #

Exemple de la bibliothèque standard (core.clj:807) :

(defmacro and
  "Evaluates exprs one at a time, from left to right. If a form
  returns logical false (nil or false), and returns that value and
  doesn't evaluate any of the other expressions, otherwise it returns
  the value of the last expr. (and) returns true."
  {:added "1.0"}
  ([] true)
  ([x] x)
  ([x & next]
   `(let [and# ~x]
      (if and# (and ~@next) and#))))
  • ` appelé syntax-quote est comme (quote), mais récursif : il provoque (let …), (if …) , etc pour ne pas évaluer pendant l’expansion de la macro mais pour sortir tel quel
  • ~ alias unquote annule les guillemets syntaxiques pour un seul formulaire à l’intérieur d’un formulaire avec guillemets syntaxiques. Ainsi, la valeur de x est sortie lors de l’expansion de la macro (au lieu de sortir le symbole x)
  • ~@ aka unquote-splicing est comme unquote mais prend l’argument de la liste et l’étend, chaque élément de la liste pour séparer la forme
  • # ajoute un identifiant unique aux symboles pour éviter les conflits de noms. Il ajoute le même identifiant pour le même symbole à l’intérieur de l’expression entre guillemets syntaxiques, donc ’et#’ à l’intérieur de ’let’ et ’et#’ à l’intérieur de ‘if’ obtiendront le même nom
