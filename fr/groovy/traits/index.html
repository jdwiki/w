Traits

  • Utilisation de base
  • Problème d’héritage multiple

Sur cette page

  • Utilisation de base
  • Problème d’héritage multiple

Les traits sont des objets de construction structurelle dans le langage Groovy.

Les traits permettent la mise en œuvre d’interfaces. Ils sont compatibles avec la vérification de type statique et la compilation Les traits se comportent comme des interfaces avec des implémentations et un état par défaut.

La déclaration d’un trait se fait en utilisant le mot-clé trait.

La portée des méthodes de caractéristiques ne prend en charge que les méthodes publiques et privées.

Utilisation de base #

Un trait est un ensemble réutilisable de méthodes et de champs qui peuvent être ajoutés à une ou plusieurs classes.

trait BarkingAbility {
    String bark(){ "I'm barking!!" }
}

Ils peuvent être utilisés comme des interfaces normales, en utilisant le mot-clé implements :

class Dog implements BarkingAbility {}
def d = new Dog()
assert d.bark() = "I'm barking!!"

Ils peuvent également être utilisés pour implémenter l’héritage multiple (évitant le problème des diamants).

Les chiens peuvent se gratter la tête, alors :

trait ScratchingAbility {
    String scratch() { "I'm scratching my head!!" }
}

class Dog implements BarkingAbility, ScratchingAbility {}
def d = new Dog()
assert d.bark() = "I'm barking!!"
assert d.scratch() = "I'm scratching my head!!"

Problème d’héritage multiple #

La classe peut implémenter plusieurs traits. Dans le cas où un trait définit une méthode avec la même signature qu’un autre trait, il y a un problème d’héritage multiple. Dans ce cas, la méthode du dernier caractère déclaré est utilisée :

trait Foo {
  def hello() {'Foo'}
}
trait Bar {
  def hello() {'Bar'}
}

class FooBar implements Foo, Bar {}

assert new FooBar().hello() == 'Bar'
