Installation ou configuration #

Il existe deux façons courantes d’installer Groovy.

Télécharger

Le binaire Groovy peut être téléchargé sur la page download du site Web Groovy. Vous pouvez décompresser l’archive et ajouter le chemin vers %GROOVY_HOME%/bin/groovy.bat à la variable d’environnement système PATH, où %GROOVY_HOME% est le répertoire dans lequel Groovy est décompressé.

SDKMAN

L’autre option consiste à utiliser SDKMAN. Cette option a rapidement gagné en popularité et rend la gestion de plusieurs versions de Groovy très simple. Il supporte également d’autres applications dans l’écosphère “GR8”. Cette option fonctionne très bien nativement sous Linux et Mac, mais nécessite Cygwin sous Windows.

En suivant les instructions de la page de téléchargement de Groovy, vous pouvez suivre les étapes suivantes pour installer SDKMAN.

$ curl -s get.sdkman.io | frapper

Une fois SDKMAN installé, vous avez maintenant accès à la commande sdk. Avec cette commande, vous pouvez faire beaucoup de choses utiles.

Installer Groovy

$ sdk install groovy

Cela installera la dernière version de Groovy.

Liste des versions de Groovy

$ sdk ls groovy

Cela vous permet d’exécuter une commande “ls” de style Linux sur le logiciel Groovy, répertoriant toutes les options disponibles. Il y a un * à côté de chaque version installée, et un > pour indiquer vos versions actuelles.

Changer de version de Groovy

$ sdk utilise groovy 2.4.7

Cela changera la version actuelle de Groovy en 2.4.7. Si vous avez d’autres versions installées, vous pouvez passer à l’une d’entre elles.

Vous pouvez lister la version actuelle de groovy avec la commande groovy -version.

chic-gvm

Le nom initial de SDKMAN était GVM et [posh-gvm][1] est un portage de GVM pour Windows Powershell. Donc, si vous développez sur une machine Windows et que vous ne souhaitez pas utiliser SDKMAN sur Cygwin, posh-gvm est fait pour vous. Cela fonctionne de la même manière que SDKMAN, mais au lieu de sdk, la commande est gmv. Alors

PS C:\Users\You> gmv install groovy

installera groovy via posh-gvm sur votre machine Windows.

[1] : https://github.com/flofreud/posh-gvm

Bonjour le monde en groovy #

L’exemple suivant illustre le plus simple “Hello World” dans groovy en utilisant un script, placez l’extrait de code suivant dans un fichier, dites “helloWorld.groovy”

println 'Hello World!'

Comment exécuter : Dans la ligne de commande, groovy helloWorld.groovy

Production: “Bonjour le monde !”

Utilisation de Groovy sur un projet Java #

Groovy a accès à toutes les classes Java, en fait les classes Groovy SONT des classes Java et peuvent être exécutées directement par la JVM. Si vous travaillez sur un projet Java, utiliser Groovy comme langage de script simple pour interagir avec votre code Java est une évidence.

Pour rendre les choses encore meilleures, presque toutes les classes Java peuvent être renommées en .groovy et compilées et exécutées et fonctionneront exactement comme elles le faisaient, groovy est proche d’être un super-ensemble de Java, c’est un objectif déclaré de groovy.

Groovy a un REPL. groovysh est fourni avec Groovy et peut être utilisé pour instancier et tester rapidement une classe Java si votre chemin de classe est correctement configuré. Par exemple, si votre classpath pointe vers votre répertoire “classes/bin” eclipse, vous pouvez enregistrer votre fichier dans eclipse, passer à groovysh et instancier la classe pour la tester.

Les raisons d’utiliser Groovy pour faire cela au lieu de simplement Java sont : Le chargeur de classe est EXCELLENT pour récupérer de nouvelles classes au fur et à mesure qu’elles sont compilées. Vous n’avez généralement pas besoin de quitter/redémarrer groovysh pendant que vous développez.

La syntaxe est TERSE. Ce n’est pas idéal pour le code maintenable, mais pour les scripts et les tests, cela peut réduire considérablement votre code. L’une des grandes choses qu’il fait est d’éliminer les exceptions vérifiées (ou, plus précisément, de transformer toutes les exceptions vérifiées en exceptions non vérifiées). Cela transforme le code comme celui-ci (imprimer bonjour après une seconde):

class JavaClass {
    public static void main(String[] args) {
        try {
            Thread.sleep(1000);
        } catch(InterruptedException e) {
            // You shouldn't leave an empty catch block, but who cares if this was interrupted???
        }
        System.out.println("Hello!");
    }
}

chez Groovy :

Thread.sleep(1000)
print "Hello!"

Groovy a également une syntaxe d’initialisation très stricte. Cela vous permet de spécifier les données comme vous le souhaitez sans y penser :

En Java, pour initialiser une carte, vous devriez probablement faire quelque chose comme ceci :

String[] init = { "1:Bill", "2:Doug", "3:Bev" };
// Note the rest of this can be put in a function and reused or maybe found in a library, but I always seem to have to write this function!
Map m = new HashMap<Integer, String>();
for(String pair : int) {
    String[] split = pair.split(":");
    m.put(new Integer(split[0]), split[1])
}

Ce n’est pas mal, mais c’est autre chose à entretenir. Dans groovy, vous utiliseriez simplement :

Map map = { 1 : "Bill", 2 : "Doug", 3 : "Bev" }

Et vous avez terminé. La syntaxe des listes est tout aussi simple.

L’autre gros avantage est la syntaxe de fermeture de groovy. C’est incroyablement concis et amusant, un peu plus difficile à maintenir, mais pour les scripts, ce n’est pas une priorité. A titre d’exemple, voici un code groovy pour trouver tous les fichiers .txt contenant le mot Hello dans le répertoire courant :

println new File('.').files.findAll{ it.name.endsWith('.txt') && it.text.contains('Hello') }.collect{ it.name }

Cet exemple utilise quelques astuces “Groovy”:

  • .files fait référence à la méthode getFiles() - groovy peut basculer entre getter/setter et syntaxe de propriété à volonté

  • it. fait référence à l’élément courant d’une itération. { it } est un raccourci pour { it -> it }, par ex. :

    [1, 2, 3].collect{ it ^ 2 } == [1, 4, 9]

  • it.text (où it est un fichier) utilise une méthode que groovy ajoute à File pour récupérer le texte entier du fichier. Ceci est incroyablement utile dans les scripts.

#!/usr/bin/env groovy

$ ./hello.groovy

Utilisation d’inject() sur la liste pour créer une chaîne CSV #

Dans Groovy, la méthode inject() est l’une des méthodes cumulatives qui nous permet d’ajouter (ou d’injecter) de nouvelles fonctionnalités dans n’importe quel objet qui implémente la méthode inject(). Dans le cas d’une collection, nous pouvons appliquer une fermeture à une collection d’objets de manière uniforme, puis rassembler les résultats en une seule valeur. Le premier paramètre de la méthode inject() est la valeur initiale du cumul et le second paramètre est la fermeture.

Dans cet exemple, nous prendrons une liste de chaînes comme paramètre et afficherons les valeurs de ces chaînes délimitées par des virgules. J’ai utilisé cette fonctionnalité pour ajouter une liste de valeurs à une chaîne de requête REST et, si vous la modifiez un peu, je l’ai utilisée pour inclure des valeurs dans une instruction SQL dans le cadre d’une clause IN. Voici le code pour faire ceci :

public String convertToCSV( List<String> list ) {
    if (list == null) {
        return ""
    }
    return list.inject( '' ) { result, item ->
        result + ( result && item ? ',' : '' ) + ( item ? "${item.trim()}" : '' )
    }
}

assert convertToCSV( null ) == ""
assert convertToCSV( ["aaa", "bbb  ", null, "  ccc  "] ) == "aaa,bbb,ccc"

Dans cet exemple, le premier paramètre de la méthode inject() est une chaîne de longueur nulle, ce qui signifie que lors du traitement du premier élément de la liste, result est également une chaîne de longueur nulle. Cela se résout à faux dans la première évaluation ternaire, c’est pourquoi nous n’obtenons pas de virgule au début de la chaîne. À chaque itération consécutive à travers les éléments de la liste, le résultat devient la concaténation de lui-même, une virgule puis l’élément suivant jusqu’à ce que nous atteignions le dernier élément de la liste.

L’avantage de cette approche est que vous n’avez pas besoin d’une variable en dehors d’une construction en boucle pour contenir le résultat String concaténé. L’implication étant que cela peut entraîner des effets secondaires dans votre code. Avec l’approche inject(), ce comportement est injecté et la collection rassemble le résultat des appels à la fermeture pour vous. L’inconvénient de cette approche peut être la lisibilité. Mais avec un peu d’expérience, cela devient plus facile à lire et à comprendre, et j’espère que cet exemple vous aidera à atteindre cet objectif.
