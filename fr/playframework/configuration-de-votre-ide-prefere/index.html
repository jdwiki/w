IDÉE IntelliJ #

Conditions préalables #

  1. Intellij IDEA installé (édition Community ou Ultimate)
  2. Plugin Scala installé dans IntelliJ
  3. Un projet Play standard, créé par exemple avec Activator (activateur new [nomduprojet] play-scala).

Ouverture du projet #

  1. Ouvrez IntelliJ IDEA
  2. Allez dans le menu Fichier > Ouvrir ... > cliquez sur tout le dossier [nomduprojet] > OK
  3. Une fenêtre contextuelle s’ouvre avec quelques options. Les valeurs par défaut sont assez bonnes dans la plupart des cas, et si vous ne les aimez pas, vous pouvez les changer ailleurs plus tard. Cliquez sur “OK”
  4. Intellij IDEA réfléchira un peu, puis proposera une autre fenêtre contextuelle pour sélectionner les modules à sélectionner dans le projet. Il devrait y avoir deux modules root et root-build sélectionnés par défaut. Ne changez rien et cliquez sur “OK”.
  5. IntelliJ ouvrira le projet. Vous pouvez commencer à visualiser les fichiers pendant qu’IntelliJ continue à réfléchir un peu comme vous devriez le voir dans la barre d’état en bas, puis il devrait enfin être complètement prêt.

Exécution des applications depuis Intellij #

À partir de là, certaines personnes utilisent l’EDI uniquement pour afficher/modifier le projet, tout en utilisant la ligne de commande sbt pour compiler/exécuter/lancer des tests. D’autres préfèrent les lancer depuis Intellij. Il est nécessaire si vous souhaitez utiliser le mode débogage. Pas :

  1. Menu Exécuter > Modifier les configurations...
  2. Dans la fenêtre contextuelle, cliquez sur le + en haut à gauche> Choisissez Play 2 App dans la liste
  3. Nommez la configuration, par exemple [nomdevotreprojet]. Laissez les options par défaut et cliquez sur “OK”.
  4. À partir du menu “Exécuter” ou des boutons de l’interface utilisateur, vous pouvez maintenant “Exécuter” ou “Déboguer” en utilisant cette configuration. Run lancera simplement l’application, comme si vous faisiez sbt run à partir de la ligne de commande. Debug fera la même chose mais vous permettra de placer des points d’arrêt dans le code pour interrompre l’exécution et analyser ce qui se passe.

Option d’importation automatique #

Il s’agit d’une option globale au projet, qui est disponible au moment de la création et peut ensuite être modifiée dans le menu Intellij IDEA > Preferences > Build, Execution, Deployment > Build tools > SBT > Paramètres au niveau du projet> Utiliser l'importation automatique.

Cette option n’a rien à voir avec les instructions “import” du code Scala. Il dicte ce qu’Intellij IDEA doit faire lorsque vous éditez le fichier build.sbt. Si l’importation automatique est activée, Intellij IDEA analysera immédiatement le nouveau fichier de construction et actualisera automatiquement la configuration du projet. Cela devient rapidement ennuyeux car cette opération est coûteuse et a tendance à ralentir Intellij lorsque vous travaillez toujours sur le fichier de construction. Lorsque l’importation automatique est désactivée, vous devez indiquer manuellement à Intellij que vous avez modifié le build.sbt et souhaitez que la configuration du projet soit actualisée. Dans la plupart des cas, une fenêtre contextuelle temporaire apparaîtra pour vous demander si vous souhaitez le faire. Sinon, accédez au panneau SBT dans l’interface utilisateur et cliquez sur le signe des flèches bleues pour forcer l’actualisation.

Eclipse comme Play IDE - Java, Play 2.4, 2.5 #

Introduction #

Play a plusieurs plugins pour différents IDE. Le plugin eclipse permet de transformer une application Play en un projet eclipse fonctionnel avec la commande activator eclipse. Le plug-in Eclipse peut être défini par projet ou globalement par utilisateur sbt. Cela dépend du travail d’équipe, de l’approche à utiliser. Si toute l’équipe utilise l’IDE eclipse, le plug-in peut être défini au niveau du projet. Vous devez télécharger la version d’eclipse prenant en charge Scala et Java 8 : luna ou mars - depuis http://scala-ide.org/download/sdk.html.

Définition de l’IDE Eclipse par projet #

Pour importer l’application Play dans Eclipse :

  1. Ajoutez le plugin eclipse dans project/plugins.sbt :

    //Support Play in Eclipse addSbtPlugin(“com.typesafe.sbteclipse” % “sbteclipse-plugin” % “4.0.0”)

  2. Ajoutez dans build.sbt un indicateur qui force la compilation à se produire lorsque la commande eclipse est exécutée :

    EclipseKeys.preTasks := Seq(compile in Compile)

  3. Assurez-vous qu’un chemin de référentiel d’utilisateurs dans le fichier {racine de l’utilisateur}.sbt\repositories a le format approprié. Les valeurs appropriées pour les propriétés activator-launcher-local et activator-local doivent avoir au moins trois barres obliques comme dans l’exemple :

    activator-local: file:////${activator.local.repository-C:/Play-2.5.3/activator-dist-1.3.10//repository}, [organization]/[module]/(scala_[scalaVersion]/)(sbt_[sbtVersion]/)[revision]/[type]s/artifact.[ext] activator-launcher-local: file:////${activator.local.repository-${activator.home-${user.home}/.activator}/repository}, [organization]/[module]/(scala_[scalaVersion]/)(sbt_[sbtVersion]/)[revision]/[type]s/artifact.[ext]

  4. Compilez l’application :

    activator compile

  5. Préparez un projet eclipse pour la nouvelle application avec :

    activator eclipse Le projet est maintenant prêt à être importé dans eclipse via Projets existants dans l’espace de travail.

Comment attacher la source Play à eclipse #

  1. Ajoutez au build.sbt :

    EclipseKeys.withSource := true

  2. Compiler le projet

Configuration globale de l’IDE Eclipse #

Ajoutez le paramètre utilisateur sbt :

  1. Créez sous le répertoire racine de l’utilisateur un dossier .sbt\0.13\plugins et un fichier plugins.sbt. Par exemple pour l’utilisateur Windows asch :

    c:\asch.sbt\0.13\plugins\plugins.sbt

  2. Ajoutez le plugin eclipse dans plugins.sbt :

    //Support Play in Eclipse addSbtPlugin(“com.typesafe.sbteclipse” % “sbteclipse-plugin” % “4.0.0”)

  3. Créez dans le répertoire utilisateur .sbt un fichier sbteclipse.sbt. Par exemple pour l’utilisateur Windows asch :

    c:\asch.sbt\0.13\sbteclipse.sbt

  4. Mettez dans sbteclipse.sbt un indicateur qui force la compilation à se produire lorsque la commande activateur eclipse est exécutée :

    import com.typesafe.sbteclipse.plugin.EclipsePlugin.EclipseKeys EclipseKeys.preTasks := Seq(compile in Compile)

  5. Ajoutez éventuellement d’autres paramètres EclipseKeys.

Débogage depuis eclipse #

Pour déboguer, démarrez l’application avec le port par défaut 9999 :

activator -jvm-debug run

ou avec le port différent :

activator -jvm-debug [port] run

En éclipse :

  1. Cliquez avec le bouton droit sur le projet et sélectionnez Déboguer en tant que, Déboguer les configurations.
  2. Dans la boîte de dialogue Configurations de débogage, cliquez avec le bouton droit sur Application Java distante et sélectionnez Nouveau.
  3. Définissez Port sur pertinent (9999 si le port de débogage par défaut a été utilisé) et cliquez sur Appliquer.

Vous pouvez désormais cliquer sur Debug pour vous connecter à l’application en cours d’exécution. L’arrêt de la session de débogage n’arrêtera pas le serveur.

Eclipse EDI #

Conditions préalables #

  1. Java8 (1.8.0_91)
  2. Néon Eclipse (JavaScript et Développeur Web)
  3. Cadre de jeu 2.5.4

Installer Scala dans Eclipse #

  1. Lancez l’éclipse
  2. Ouvrez Aide > Eclipse Marketplace
  3. Tapez Scala dans Rechercher
  4. Installez Scala IDE

Configurer sbteclipse #

  1. Ouvrez le projet de lecture .\project\ plugins.sbt
  2. Ajoutez la commande suivante dans plugins.sbt pour convertir le projet eclipse

addSbtPlugin(“com.typesafe.sbteclipse” % “sbteclipse-plugin” % “4.0.0”)

  1. Ouvrez la commande et accédez au projet de lecture, par exemple. cd C:\play\play-scala. Tapez ce qui suit sur la ligne de commande

activateur éclipse

Projet d’importation #

  1. Allez dans le menu Fichier > Importer dans Eclipse
  2. Sélectionnez “Projets existants dans l’espace de travail”
  3. Sélectionnez le répertoire racine

Votre projet est maintenant prêt à être visualisé et modifié dans Eclipse IDE.
