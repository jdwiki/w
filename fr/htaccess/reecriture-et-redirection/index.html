Avant que les URL puissent être réécrites, un module appelé mod_rewrite.c doit être activé. Habituellement, il est désactivé dans la configuration par défaut.

mod_rewrite peut être activé en exécutant la commande

$ sudo a2enmod mod_rewrite
$ sudo service apache2 restart

ou en commentant les lignes

#LoadModule rewrite_module modules/mod_rewrite.so
#AddModule mod_rewrite.c

dans le fichier httpd.conf.

Drapeaux de réécriture populaires #

F|interdit #

Semblable à Deny, cet indicateur force le serveur à renvoyer immédiatement un code d’état 403 Forbidden au navigateur ou au client demandeur pour la demande.

Exemple : Refuser l’accès aux requêtes qui se terminent par “exe” :

RewriteRule .exe$ - [F]

G|parti #

Si une ressource demandée était disponible dans le passé, mais n’est plus disponible, vous pouvez utiliser cet indicateur pour forcer le serveur à renvoyer immédiatement un code d’état 410 Gone au navigateur ou au client demandeur pour la demande.

Exemple : Dire à un visiteur qu’un ancien produit n’existe plus :

RewriteRule ^old-product.html$ - [G]

L|dernier #

Dans la plupart des contextes, autres que .htaccess, cet indicateur indique à mod_rewrite d’arrêter de traiter l’ensemble de conditions/règles actuel, à peu près de la même manière que last et break (Perl et C, respectivement).

Cependant, dans le contexte .htaccess ou <Directory>, une requête qui a été réécrite à l’aide d’une RewriteRule avec cet indicateur sera renvoyée au moteur d’analyse d’URL pour un traitement ultérieur. En tant que tel, il est possible que l’URI réécrit soit géré par le même contexte, et peut-être modifié davantage.

Une recommandation générale est d’utiliser le drapeau END non seulement pour arrêter le traitement de l’ensemble de conditions/règles en cours, mais également pour empêcher toute réécriture ultérieure dans ces contextes.

Remarque : Les drapeaux F et G, décrits ci-dessus, utilisent tous deux L implicitement, vous n’avez donc pas besoin de les spécifier séparément.

N|suivant #

Cet indicateur relancera le processus de réécriture depuis le début, en recommençant avec la première condition/règle définie. Cette fois, l’URL à faire correspondre n’est plus l’URI d’origine, mais plutôt l’URI réécrit renvoyé par le dernier ensemble de règles. Utilisez cet indicateur pour redémarrer le processus de réécriture.

Un mot d’avertissement : Utilisez ce drapeau avec prudence, car cela peut entraîner une boucle infinie !

NC|pas de cas #

Cela indique à mod_rewrite de faire correspondre le Pattern d’une RewriteRule sans être sensible à la casse. Pour clarifier, MyIndex.html et myindex.html seraient considérés par le module comme la même chose. De plus, ce drapeau vous permet d’utiliser a-z au lieu de A-Za-z dans une expression régulière.

R|rediriger #

Cet indicateur est utilisé pour envoyer une réponse de redirection HTTP au navigateur/client demandeur.

Par défaut, si aucun code n’est fourni, une réponse de redirection avec le code d’état 302 Found (similaire à une redirection temporaire) sera renvoyée. Si vous souhaitez utiliser une redirection plus permanente, vous devez utiliser le code de statut 302 (301 Moved Permanently).

Généralement, seuls les codes d’état compris entre 300 et 399 doivent être utilisés avec cet indicateur. Si des codes d’état en dehors de cette plage sont utilisés (ce qui est parfaitement acceptable), la chaîne de substitution est rejetée et la réécriture est arrêtée comme si le drapeau L était utilisé. Dans certains cas, c’est un moyen pratique de forcer les réponses 404 Not Found, même si la requête pointe vers une ressource existante.

Exemple : Émettez une réponse de redirection 302 Found :

RewriteRule ^bus$ /train [R,L]

Exemple : émettre une réponse de redirection 301 Moved Permanently :

RewriteRule ^speed-train$ /hyperloop [R=301,L]

Exemple : Forcer un 404 Not Found :

RewriteRule ^blip$ - [R=404,L]

Redirection avec/sans paramètres de requête #

Rediriger sans paramètres de requête :

RewriteRule ^route$ /new_route_without_query [L,R=301,QSD]

Rediriger avec les paramètres de requête :

RewriteCond %{QUERY_STRING} ^$
RewriteRule ^/?route$ %{REQUEST_URI}?query=param1&query2=param2 [NC,L,R=301]

Redirections www et non-www #

Redirigez tout domaine nu vers www.[votre_domaine].tld :

# Start Apache Rewriting engine
RewriteEngine On
# Make sure you're not already using www subdomain
# and that the host string is not empty
RewriteCond %{HTTP_HOST} !^$
RewriteCond %{HTTP_HOST} !^www\.
# We check for http/https connection protocol
RewriteCond %{HTTPS}s ^on(s)|
# In case the previous conditions matches, redirect to www
RewriteRule ^(.*)$ http%1://www.%{HTTP_HOST}/$1 [R=301,L]

Rediriger www.[votre_domaine].tld vers [votre_domaine].tld

# Start Apache Rewriting engine
RewriteEngine On
# We check if we're on the www subdomain
RewriteCond %{HTTP_HOST} ^www\.([^\.]+\.[^\.]+)$
# In case the previous condition matches, redirect to non-www
RewriteRule ^(.*)$ http://%1/$1 [R=301,L]

Redirigez n’importe quel niveau de sous-domaines imbriqués vers votre domaine principal :

# Start Apache Rewriting engine
RewriteEngine On
# We check if there's a subdomain
RewriteCond %{HTTP_HOST} \.([^.]+\.[^.]+)$
# redirect to the main domain name
RewriteRule ^ http://%1%{REQUEST_URI} [R=301,L]

URL optimisées pour le référencement #

Les moteurs de recherche n’indexeront pas vos produits si vous avez une URL comme celle-ci :

http://www.yourdomain.com/product.php?id=123

Une URL conviviale pour le référencement ressemblerait à http://www.yourdomain.com/123/product-name/. Le code suivant permet d’y parvenir sans avoir à modifier le code product.php.

RewriteEngine On
RewriteRule ^product/([0-9]+)/product-name-slug/?$ product.php?id=$1

Ajouter une barre oblique à la fin #

RewriteCond %{REQUEST_FILENAME} !-f
RewriteCond %{REQUEST_URI} !(.*)/$
RewriteRule ^(.*)$ /$1/ [L,R=301]

Le premier RewriteCond aide à exclure les fichiers. Le deuxième RewriteCond vérifie s’il y a déjà une barre oblique à la fin. Si tel est le cas, RewriteRule n’est pas appliqué.

Si vous avez une URL qui ne devrait pas être réécrite, vous pouvez ajouter une autre RewriteCond.

RewriteCond %{REQUEST_FILENAME} !-f
RewriteCond %{REQUEST_URI} !(.*)/$
RewriteCond %{REQUEST_URI} !url/to/not/rewrite
RewriteRule ^(.*)$ /$1/ [L,R=301]

Redirections http et https et configuration HSTS #

Redirection générique vers https : #

# Enable Rewrite engine
RewriteEngine on

# Check if URL does not contain https
RewriteCond %{HTTPS} off [NC]
# If condition is true, redirect to https
RewriteRule (.*) https://%{SERVER_NAME}/$1 [R=301,L]

Redirection générique vers http : #

# Enable Rewrite engine
RewriteEngine on

# Check if URL does contain https
RewriteCond %{HTTPS} on [NC]
# If condition is true, redirect to http
RewriteRule (.*) http://%{SERVER_NAME}/$1 [R=301,L]

Forcer la connexion HTTPS (HSTS) : #

<IfModule mod_headers.c>
    Header always set Strict-Transport-Security "max-age=31536000; includeSubDomains"
</IfModule>

où, l’option includeSubDomains peut être supprimée, si HSTS doit être appliqué uniquement au domaine de base ou au domaine avec la configuration ci-dessus.
