Syntaxe #

  • val (|>) : 'a -> ('a -> 'b) -> 'b
  • val (@@) : ('a -> 'b) -> 'a -> 'b

Algorithmes génériques #

Les fonctions d’ordre supérieur peuvent être utilisées pour implémenter des algorithmes génériques, abandonnant la responsabilité de fournir les derniers détails à l’utilisateur. Par exemple, List.sort attend une fonction de comparaison, qui permet d’implémenter différentes manières de trier. Ici, nous implémentons un tri des chaînes insensible à la casse :

let string_case_insensitive_sort lst =
  let case_insensitive_compare a b =
    String.compare (String.lowercase a) (String.lowercase b)
  in
  List.sort case_insensitive_compare lst

Il existe une riche liste de fonctions d’ordre supérieur dans la bibliothèque standard, notamment dans le module List, voir List.fold_left et List.sort par exemple. Des exemples plus avancés peuvent être trouvés dans des bibliothèques tierces. Un bon exemple est le recuit simulé implémenté dans ocaml-gsl. Le recuit simulé est une procédure d’optimisation générique qui est paramétrée par une fonction permettant d’explorer l’ensemble des états du problème et une fonction d’erreur (appelée ici fonction énergétique) .

Les utilisateurs familiarisés avec C++ peuvent comparer cela au modèle Strategy.

Dispose des ressources système même lorsqu’une exception est déclenchée #

Des fonctions d’ordre supérieur peuvent être utilisées pour garantir que les ressources système sont supprimées, même lorsqu’un traitement déclenche une exception. Le modèle utilisé par with_output_file permet une séparation nette des préoccupations : les fonctions d’ordre supérieur with_output_file s’occupent de gérer les ressources système liées à la manipulation de fichiers tandis que le traitement f ne consomme que le canal de sortie.

let with_output_file path f =
  let c = open_out path in
  try
    let answer = f c in
    (close_out c; answer)
  with exn -> (close_out c; raise exn)

Utilisons cette fonction d’ordre supérieur pour implémenter une fonction écrivant une chaîne dans un fichier :

let save_string path s =
  (with_output_file path) (fun c -> output_string c s)

En utilisant des fonctions plus avancées que fun c -> output_string c s, il est possible de sauvegarder des valeurs plus complexes. Voir par exemple le module Marshal dans la bibliothèque standard ou le module [Yojson](http:// mjambon.com/yojson.html) bibliothèque de Martin Jambon.

Opérateurs de composition #

Deux fonctions utiles d’ordre supérieur sont les opérateurs binaires application (@@) et reverse-application ou “pipe” (|>). Bien que depuis [4.01][1] elles soient disponibles en tant que primitives, il peut toujours être instructif de les définir ici :

let (|>) x f = f x
let (@@) f x = f x

Considérez le problème de l’incrémentation du carré de 3. Une façon d’exprimer ce calcul est la suivante :

(* 1 -- Using parentheses *)
succ (square 3)
(* - : int = 10 *)

(* where `square` is defined as: *)
let square x = x * x

Notez que nous ne pouvions pas simplement faire succ square 3 parce que (en raison de [left-associativity][2]) cela se réduirait au (succ square) 3 sans signification. En utilisant application (@@), nous pouvons exprimer cela sans les parenthèses :

(* 2 -- Using the application operator *)
succ @@ square 3
(* - : int = 10 *)

Remarquez comment la dernière opération à effectuer (à savoir succ) apparaît en premier dans l’expression ? L’opérateur reverse-application (|>) nous permet, eh bien, d’inverser ceci :

(* 3 -- Using the reverse-application operator *)
3 |> square |> succ
(* - : int = 10 *)

Le nombre 3 est maintenant “canalisé” à travers square puis succ, au lieu d’être appliqué à square pour donner un résultat auquel succ est appliqué.

[1] : http://caml.inria.fr/pub/docs/manual-ocaml/libref/Pervasives.html [2] : http://caml.inria.fr/pub/docs/manual-ocaml/expr.html
