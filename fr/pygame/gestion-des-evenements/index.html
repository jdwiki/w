Boucle d’événement #

Pygame enregistrera tous les événements de l’utilisateur dans une file d’attente d’événements qui peut être reçue avec le code pygame.event.get(). Chaque élément de cette file d’attente est un objet Event et ils auront tous l’attribut type, qui est un entier représentant le type d’événement dont il s’agit. Dans le module pygame, il existe des constantes entières prédéfinies représentant le type. À l’exception de cet attribut, les événements ont des attributs différents.

Nom constant Attributs
QUITTER aucun
ÉVÉNEMENT ACTIF gain, état
TOUCHE BAS unicode, clé, mod
KEYUP clé, mode
MOUSEMOUVEMENT pos, rel, boutons
MOUSEBUTTONUP pos, bouton
BOUTON DE SOURIS BAS pos, bouton
JOYAXISMOTION joie, axe, valeur
JOYBALLMOTION joie, boule, rel
JOYHATMOTION joie, chapeau, valeur
JOYBUTTONUP joie, bouton
JOYBUTTONDOWN joie, bouton
TAILLE VIDEO taille, l, h
VIDÉOEXPOSER aucun
UTILISERÉVÉNEMENT code

Exemple #

Pour gérer nos événements, nous parcourons simplement la file d’attente, vérifions de quel type il s’agit (à l’aide des constantes prédéfinies dans le module pygame), puis effectuons une action. Ce code vérifiera si l’utilisateur a appuyé sur le bouton de fermeture dans le coin supérieur de l’écran et, le cas échéant, mettra fin au programme.

for event in pygame.event.get():
    if event.type == pygame.QUIT:
        # Close the program any way you want, or troll users who want to close your program.
        raise SystemExit

ATTENTION : Vous devez appeler régulièrement la file d’attente des événements lorsque vous utilisez pygame ! Outre la récupération des événements disponibles, l’appel de la file d’attente d’événements est également le moyen pour pygame d’interagir avec le système d’exploitation en interne. Si la file d’attente d’événements n’est pas appelée régulièrement, votre système d’exploitation supposera que votre programme ne fonctionne plus correctement et peut donner l’impression que le programme a planté (sous Windows, la fenêtre devient blanche). Si vous ne voulez rien faire avec les événements, vous devez appeler pygame.event.pump() chaque boucle de jeu pour que pygame traite les événements en interne.

Événements clavier #

Il existe deux types d’événements clés dans pygame : KEYDOWN et KEYUP. Ces événements ont un attribut key qui est un entier représentant une touche du clavier. Le module pygame a des constantes entières prédéfinies représentant toutes les clés communes. Les constantes sont nommées avec un K majuscule, un trait de soulignement et le nom de la clé. Par exemple, <- est nommé K_BACKSPACE, a est nommé K_a et F4 est nommé K_F4.

Exemple #

Ce code vérifiera si l’utilisateur a appuyé sur w, a, s ou d.

for event in pygame.event.get():
    if event.type == pygame.QUIT:  # Usually wise to be able to close your program.
        raise SystemExit
    elif event.type == pygame.KEYDOWN:
        if event.key == pygame.K_w:
            print("Player moved up!")
        elif event.key == pygame.K_a:
            print("Player moved left!")
        elif event.key == pygame.K_s:
            print("Player moved down!")
        elif event.key == pygame.K_d:
            print("Player moved right!")

Modificateurs #

Il n’y a pas de constante entière pour les majuscules. Au lieu de cela, les événements clés ont un autre attribut appelé mod, qui correspond aux modificateurs (shift, ctrl, alt etc.) pressés simultanément comme la clé. L’attribut mod est un entier représentant le modificateur pressé. La valeur entière de chaque modificateur est stockée dans le module pygame sous le nom de KMOD_ et leur nom. Par exemple, Shift gauche est nommé KMOD_LSHIFT, Tab est nommé KMOD_TAB et Ctrl est nommé KMOD_CTRL.

Exemple #

Ce code vérifiera si l’utilisateur a appuyé sur a, Left shift + a ou Caps + a< /kbd>.

for event in pygame.event.get():
    if event.type == pygame.QUIT:  # It's still wise to be able to close your program.
        raise SystemExit
    elif event.type == pygame.KEYDOWN:
        if event.key == pygame.K_a:
            if event.mod == 0:  # No modifier.
                print("You pressed 'a'")
            elif event.mod == pygame.KMOD_LSHIFT or event.mod == pygame.KMOD_CAPS:
                print("You pressed 'A'")
            else:
                print("You pressed 'a' with another modifier than right shift or caps.")

Événements de souris #

Il existe trois types d’événements de souris dans pygame MOUSEMOTION, MOUSEBUTTONDOWN et MOUSEBUTTONUP. Pygame enregistrera ces événements lorsqu’un mode d’affichage aura été défini.

MOUSEMOTION est reçu lorsque l’utilisateur déplace sa souris sur l’écran. Il a les attributs buttons, pos et rel.

  • buttons est un tuple représentant si les boutons de la souris (left, mouse-wheel, right) sont enfoncés ou non.
  • pos est la position absolue (x, y) du curseur en pixels.
  • rel est la position par rapport à la position précédente (rel_x, rel_y) en pixels.

MOUSEBUTTONDOWN et MOUSEBUTTONUP sont reçus lorsque l’utilisateur appuie ou relâche un bouton de la souris. Ils ont les attributs button et pos.

  • button est un entier représentant le bouton pressé. 1 pour le bouton gauche, 2 pour la molette de la souris et 3 pour le bouton droit.
  • pos est la position absolue de la souris (x, y) lorsque l’utilisateur a appuyé sur le bouton de la souris.

Exemple #

Voici un court exemple utilisant certains des attributs de chaque événement de souris :

for event in pygame.event.get():
    if event.type == pygame.QUIT:  # Close your program if the user wants to quit.
        raise SystemExit
    elif event.type == pygame.MOUSEMOTION:
        if event.rel[0] > 0:  # 'rel' is a tuple (x, y). 'rel[0]' is the x-value.
            print("You're moving the mouse to the right")
        elif event.rel[1] > 0:  # pygame start y=0 at the top of the display, so higher y-values are further down.
            print("You're moving the mouse down")
    elif event.type == pygame.MOUSEBUTTONDOWN:
        if event.button == 1:
            print("You pressed the left mouse button")
        elif event.button == 3:
            print("You pressed the right mouse button")
    elif event.type == pygame.MOUSEBUTTONUP:
        print("You released the mouse button")

Puisqu’il n’y a pas de constantes prédéfinies pour l’attribut du bouton de la souris dans le module pygame, voici les valeurs pour chacun :

Bouton Valeur
Bouton gauche de la souris 1
Bouton de la molette de la souris 2
Bouton droit de la souris 3
Molette de la souris défiler vers le haut 4
Faites défiler la molette de la souris vers le bas 5

Faire défiler le bouton de la souris générera les événements pygame.MOUSEBUTTONDOWN et pygame.MOUSEBUTTONUP.

Vérification de l’état #

Il est possible d’appeler des fonctions depuis les modules pygame.key et pygame.mouse pour recevoir l’état de la touche et de la souris. Cependant, ce n’est pas la méthode recommandée pour traiter les événements dans pygame car il présente quelques défauts :

  • Vous recevrez les états lorsque la fonction est appelée, ce qui signifie que vous risquez de manquer des événements entre les appels si l’utilisateur appuie rapidement sur les boutons.

  • Vous ne pouvez pas déterminer l’ordre des événements.

  • Vous devez toujours appeler l’une des fonctions d’événement de pygame pour que pygame interagisse en interne avec le système d’exploitation, sinon il avertira que le programme ne répond plus. Les fonctions que vous pouvez appeler sont :

  • pygame.event.get() pour obtenir tous les événements ou types d’événements (en passant les types comme argument) de la file d’attente.

  • pygame.event.poll() pour obtenir un seul événement de la file d’attente.

  • pygame.event.wait() pour attendre un seul événement de la file d’attente.

  • pygame.event.clear() pour effacer tous les événements de la file d’attente.

  • pygame.event.pump() pour permettre à pygame de gérer les actions internes (est appelé implicitement par les fonctions ci-dessus).

Événements clavier #

Le module key a une fonction pygame.key.get_pressed() qui renvoie une liste de l’état de toutes les clés. La liste contient ‘0’ pour toutes les touches qui ne sont pas enfoncées et ‘1’ pour toutes les touches qui sont enfoncées. Son index dans la liste est défini par des constantes dans le module pygame, toutes préfixées par K_ et le nom de la clé.

pygame.event.pump()  # Allow pygame to handle internal actions.
key = pygame.key.get_pressed()
if key[pygame.K_a]:
    print("You pressed 'a'")
if key[pygame.K_F1]:
    print("You pressed 'F1'")
if key[pygame.K_LSHIFT]:
    print("You pressed 'left shift'")
if key[pygame.K_q]:  # Press 'q' to exit the program
    quit()

Si vous souhaitez vérifier si une seule touche est enfoncée au lieu de savoir si la touche est maintenue enfoncée, vous pouvez stocker l’état précédent de toutes les touches dans une variable temporaire et vérifier si la valeur change :

pygame.event.pump()  # Allow pygame to handle internal actions.
key = pygame.key.get_pressed()
if key[pygame.K_q] and not previous_key[pygame.K_q]:
    print("You pressed 'q'")
if key[pygame.K_p] and not previous_key[pygame.K_p]:
    print("You pressed 'p'")
previous_key = key

L’instruction prend la valeur true uniquement lorsque la touche actuelle est enfoncée et que la touche précédente n’est pas enfoncée. Pour vérifier si l’utilisateur a libéré la clé, il vous suffit de changer le mot-clé not (if not key[pygame.K_q] and previous_key[pygame.K_q]). Pour que cela fonctionne correctement, vous devez définir la variable previous_key = pygame.key.get_pressed() avant la boucle de jeu, sinon vous recevrez une NameError.

Événements de souris #

Le module de la souris a des fonctions qui nous permettent de vérifier et de définir la position de la souris ainsi que de vérifier les boutons enfoncés. La fonction pygame.mouse.get_pressed() renvoie un tuple tuple représentant si les boutons de la souris (gauche, molette de la souris, droite) sont enfoncés ou non.

pygame.event.pump()  # Allow pygame to handle internal actions.
mouse_pos = pygame.mouse.get_pos()
mouse_buttons = pygame.mouse.get_pressed()
if mouse_pos[0] > 100:
    pygame.mouse.set_pos(10, mouse_pos[1])  # Reset the mouse's x-position to 10.
    print("YOU SHALL NOT PASS!")
if mouse_buttons[2]:
    print("I'm right, right?")
if mouse_buttons[0]:  # Press left mouse button to exit.
    print("Program left")
    quit()
