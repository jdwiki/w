Syntaxe #

  • pygame.draw.rect(Surface, couleur, Rect, largeur=0)
  • pygame.draw.polygon(Surface, couleur, liste de points, largeur=0)
  • pygame.draw.circle(Surface, couleur, pos, rayon, largeur=0)
  • pygame.draw.ellipse(Surface, couleur, Rect, largeur=0)
  • pygame.draw.arc(Surface, color, Rect, start_angle, stop_angle, width=1)
  • pygame.draw.line(Surface, couleur, start_pos, end_pos, width=1)
  • pygame.draw.lines(Surface, couleur, fermée, liste de points, largeur=1)
  • pygame.draw.aaline(Surface, couleur, startpos, endpos, blend=1)
  • pygame.draw.aalines(Surface, couleur, fermée, liste de points, blend=1)

Paramètres #

Paramètres Détails
surface Surface sur laquelle dessiner la forme.
couleur Une séquence de 3 ou 4 entiers représentant le rouge, le vert et le bleu (et alpha), chaque valeur étant comprise entre 0 et 255.
Rect Une zone rectangulaire dans laquelle la forme sera dessinée.
largeur La largeur des lignes. La forme sera remplie si width=0.
liste de points Une liste d’un nombre arbitraire de points/sommets, en pixels (x, y).
poste La position du centre du cercle, en pixels (x, y).
rayon Le rayon des cercles en pixels.
fermé Si vrai, une ligne entre le dernier et le premier point sera tracée, fermant la forme.
mélange=1 Si vrai, les nuances seront fusionnées avec les nuances de pixels existantes au lieu de les écraser.
angle_départ L’angle initial de l’arc, en radians.
angle_arrêt L’angle final de l’arc, en radians.
start_pos La position de départ de la ligne, en pixels.
end_pos La position finale de la ligne, en pixels

Dessiner avec le module de dessin #

Pygame a un module, pygame.draw, qui contient des fonctions permettant de dessiner des formes directement sur une surface.

Fonction Descriptif
pygame.draw.rect dessiner une forme de rectangle
pygame.draw.polygone dessiner une forme avec n’importe quel nombre de côtés
pygame.draw.cercle tracer un cercle autour d’un point
pygame.draw.ellipse dessiner une forme ronde à l’intérieur d’un rectangle
pygame.draw.arc dessiner une section partielle d’une ellipse
pygame.draw.line tracer un segment de droite
pygame.draw.lines dessiner plusieurs segments de ligne contigus
pygame.draw.aaline dessiner des lignes fines anticrénelées
pygame.draw.aalines dessiner une séquence connectée de lignes anticrénelées

Comment utiliser le module #

Pour utiliser le module, vous devez d’abord importer et initialiser correctement pygame et définir un mode d’affichage. Il est pratique de définir des constantes de couleur à l’avance, ce qui rend votre code plus lisible et plus beau. Toutes les fonctions prennent une surface sur laquelle dessiner, une couleur et un argument de position qui est soit un pygame Rect, soit une séquence entière/float à 2 éléments (le pygame.draw.circle ne prendra que des entiers pour des raisons indéfinies).

Exemple #

Le code ci-dessous présentera toutes les différentes fonctions, comment elles sont utilisées et à quoi elles ressemblent. Nous allons initialiser pygame et définir quelques constantes avant les exemples.

import pygame
from math import pi
pygame.init()

screen = pygame.display.set_mode((100, 100))
WHITE = pygame.Color(255, 255, 255)
RED = pygame.Color(255, 0, 0)

La couleur noire est la couleur par défaut de la surface et représente la partie de la surface qui n’a pas été dessinée. Les paramètres de chaque fonction sont expliqués ci-dessous dans Paramètres .

size = (50, 50)

rect_border = pygame.Surface(size)  # Create a Surface to draw on.
pygame.draw.rect(rect_border, RED, rect_border.get_rect(), 10)  # Draw on it.

rect_filled = pygame.Surface(size)
pygame.draw.rect(rect_filled, RED, rect_filled.get_rect())

##Surfaces Dans pygame, vous utilisez généralement des surfaces pour représenter l’apparence des objets et des rectangles pour représenter leurs positions. Une surface est comme une feuille de papier vierge qui contient des couleurs ou des images. Il existe deux manières de créer une surface : vierge à partir de zéro ou en chargeant une image.

Créer une surface #

Pour créer une surface, vous avez besoin au minimum de sa taille, qui est une séquence entière de 2 éléments de largeur et de hauteur, représentant la taille en pixels.

Vous pouvez également passer des arguments supplémentaires lors de la création d’une surface pour contrôler la profondeur de bits, les masques et les fonctionnalités supplémentaires en tant qu’alpha par pixel et/ou créer l’image dans la mémoire vidéo. Cela sort cependant du cadre de cet exemple.

size = width, height = (32, 32)
empty_surface = pygame.Surface(size)

Vous pouvez utiliser le module pygame.draw pour dessiner des formes sur la surface ou le remplir avec une couleur en appelant la méthode Surface fill(color). L’argument color est une séquence d’entiers à 3 ou 4 éléments ou un objet pygame.Color.

Charger une image #

Le plus souvent, vous aimeriez utiliser vos propres images dans un jeu (appelées sprites). Créer une surface avec votre image est aussi simple que :

my_image = pygame.image.load(path_to_image)

Le chemin d’accès à l’image peut être relatif ou absolu. Pour améliorer les performances, il est généralement judicieux de convertir votre image au même format de pixel que l’écran. Cela peut être fait en appelant la méthode Surface convert(), comme ceci :

my_image = pygame.image.load(path_to_image).convert()

Si votre image contient de la transparence (valeurs alpha), vous appelez simplement la méthode convert_alpha() à la place :

my_image = pygame.image.load(path_to_image).convert_alpha()

Blitting #

Les surfaces doivent être éclaircies à l’écran afin de pouvoir les afficher. Blitting signifie essentiellement copier des pixels d’une surface à une autre (l’écran est également une surface). Vous devez également transmettre la position de la surface, qui doit être une séquence d’entiers à 2 éléments ou un objet Rect. Le coin supérieur gauche de la surface sera placé à la position.

screen.blit(my_image, (0, 0))
pygame.display.update()  # or pygame.display.flip()

Il est possible de blit vers d’autres surfaces que l’écran. Pour afficher ce qui a été blitté à l’écran, vous devez appeler pygame.display.update() ou pygame.display.flip().

Transparence #

Il existe trois types de transparence pris en charge dans pygame : les touches de couleur, les alphas de surface et les alphas par pixel.

Touches de couleur #

Rend une couleur entièrement transparente, ou plus précisément, en faisant en sorte qu’une couleur ne soit tout simplement pas blit. Si vous avez une image avec un rectangle noir à l’intérieur, vous pouvez définir une clé de couleur pour empêcher la couleur noire d’être blit.

BLACK = (0, 0, 0)
my_image.set_colorkey(BLACK)  # Black colors will not be blit.

Une surface ne peut avoir qu’une seule clé de couleur. Définir une autre clé de couleur écrasera la précédente. Les clés de couleur ne peuvent pas avoir des valeurs alpha différentes, elles ne peuvent que rendre une couleur non visible.

[![entrez la description de l’image ici][1]][1]

Alphas de surface #

Rend toute la surface transparente d’une valeur alpha. Avec cette méthode, vous pouvez avoir différentes valeurs alpha mais cela affectera toute la surface.

my_image.set_alpha(100)  # 0 is fully transparent and 255 fully opaque.

[![entrez la description de l’image ici][2]][2]

Alpha par pixel #

Rend chaque pixel de la surface transparent par une valeur alpha individuelle. Cela vous donne le plus de liberté et de flexibilité, mais c’est aussi la méthode la plus lente. Cette méthode nécessite également que la surface soit créée en tant que surface alpha par pixel, et les arguments de couleur doivent contenir un quatrième entier alpha.

size = width, height = (32, 32)
my_image = pygame.Surface(size, pygame.SRCALPHA)  # Creates an empty per-pixel alpha Surface.

La surface dessinera maintenant la transparence si la couleur contient la quatrième valeur alpha.

BLUE = (0, 0, 255, 255)
pygame.draw.rect(my_image, BLUE, my_image.get_rect(), 10)

Contrairement aux autres surfaces, cette couleur par défaut de surface ne sera pas noire mais transparente. C’est pourquoi le rectangle noir au milieu disparaît.

[![entrez la description de l’image ici][3]][3]

Combiner colorkey et surface alpha #

Les touches de couleur et les alphas de surface peuvent être combinés, mais pas l’alpha par pixel. Cela peut être utile si vous ne voulez pas les performances plus lentes d’une surface par pixel.

purple_image.set_colorkey(BLACK)
purple_image.set_alpha(50)

[![entrez la description de l’image ici][4]][4]

Code complet #

Copiez ceci dans un fichier vide et exécutez-le. Appuyez sur les touches 1, 2, 3 ou 4 pour faire apparaître les images. Appuyez plusieurs fois sur 2, 3 ou 4 pour les rendre plus opaques.

import pygame
pygame.init()

WHITE = (255, 255, 255)
BLACK = (0, 0, 0)
RED = (255, 0, 0)
GREEN = (0, 255, 0)
BLUE = (0, 0, 255, 50)  # This color contains an extra integer. It's the alpha value.
PURPLE = (255, 0, 255)

screen = pygame.display.set_mode((200, 325))
screen.fill(WHITE)  # Make the background white. Remember that the screen is a Surface!
clock = pygame.time.Clock()

size = (50, 50)
red_image = pygame.Surface(size)
green_image = pygame.Surface(size)
blue_image = pygame.Surface(size, pygame.SRCALPHA)  # Contains a flag telling pygame that the Surface is per-pixel alpha
purple_image = pygame.Surface(size)

red_image.set_colorkey(BLACK)
green_image.set_alpha(50)
# For the 'blue_image' it's the alpha value of the color that's been drawn to each pixel that determines transparency.
purple_image.set_colorkey(BLACK)
purple_image.set_alpha(50)

pygame.draw.rect(red_image, RED, red_image.get_rect(), 10)
pygame.draw.rect(green_image, GREEN, green_image.get_rect(), 10)
pygame.draw.rect(blue_image, BLUE, blue_image.get_rect(), 10)
pygame.draw.rect(purple_image, PURPLE, purple_image.get_rect(), 10)

while True:
    clock.tick(60)

    for event in pygame.event.get():
        if event.type == pygame.QUIT:
            quit()
        elif event.type == pygame.KEYDOWN:
            if event.key == pygame.K_1:
                screen.blit(red_image, (75, 25))
            elif event.key == pygame.K_2:
                screen.blit(green_image, (75, 100))
            elif event.key == pygame.K_3:
                screen.blit(blue_image, (75, 175))
            elif event.key == pygame.K_4:
                screen.blit(purple_image, (75, 250))

    pygame.display.update()
