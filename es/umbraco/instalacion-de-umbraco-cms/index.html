Entorno de alojamiento #

Para Umbraco 7 los requisitos son

  • IIS 7 o superior
  • Base de datos, una de las siguientes: SQL CE, SQL Server 2008 o superior o MySQL compatible con consultas que no distinguen entre mayúsculas y minúsculas)
  • ASP.NET 4.5 o 4.5.1. Plena confianza
  • Capacidad para establecer permisos de archivo/carpeta para el usuario que “posee” el grupo de aplicaciones

Instalación manual de Umbraco #

  1. Descargue los archivos de our.umbraco.org/download y descomprímalos en una carpeta.
  2. Configure el servidor web que aloja la carpeta que eligió. Aunque IIS es un requisito para los sitios de producción, funciona bien para el desarrollo en IIS Express.
  3. Configure los permisos de archivo para la carpeta. Para el servidor de desarrollo, está bien tener permisos completos de lectura/escritura/modificación para el proceso que sirve a la página web (AppPool o servicio de red)
  4. Si desea configurar Umbraco en Sql Server o Mysql, descárguelo e instálelo. Cree una nueva base de datos para Umbraco y recuerde los detalles de la cadena de conexión, incluido el nombre de la base de datos, el usuario y la contraseña.
  5. Abra su sitio en localhost y siga el asistente para configurar su sitio con la base de datos.

Instalar Umbraco con NuGet #

  1. Busque actualizaciones para el administrador de paquetes Nuget. En Visual Studio: Herramientas > Extensiones y actualizaciones > Actualizaciones > Galería de Visual Studio. Instalar si está disponible
  2. Cree una nueva aplicación web con la plantilla “Aplicación web ASP.NET con una plantilla vacía” en .NET Framework 4.5.1
  3. Abra la consola del administrador de paquetes y ejecute Install-Package UmbracoCms
  4. Presione F5 para crear y ejecutar su nuevo sitio web.
  5. Complete el asistente para elegir el proveedor de la base de datos y configure su sitio.

Usa Umbraco como un servicio en la nube #

Si solo desea probar Umbraco CMS o alojar su sitio en un servicio en la nube, puede registrarse para una prueba gratuita en umbraco.com/cloud. El sitio que desarrolle en el servicio en la nube podría descargarse para el desarrollo local o su propio alojamiento más adelante.
