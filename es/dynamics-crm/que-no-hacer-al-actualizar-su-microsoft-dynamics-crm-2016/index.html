Qué no hacer al actualizar su Microsoft Dynamics CRM 2016

  • Veamos algunos elementos obligatorios para marcar antes de sumergirnos en un proceso de actualización a gran escala
  • ¿Los beneficios de actualizar superarán los dolores de cabeza? ¿Vale la pena una actualización?

En esta página

  • Veamos algunos elementos obligatorios para marcar antes de sumergirnos en un proceso de actualización a gran escala
  • ¿Los beneficios de actualizar superarán los dolores de cabeza? ¿Vale la pena una actualización?

Microsoft Dynamics CRM ha evolucionado drásticamente en los últimos años. Ha habido muchas actualizaciones y versiones lanzadas, con una gama de nuevas características y mejoras en cada etapa a lo largo del camino. Durante la actualización de un entorno de Dynamics CRM, hay algunos puntos a tener en cuenta para garantizar un proceso de actualización sin complicaciones.

Veamos algunos elementos obligatorios para marcar antes de sumergirnos en un proceso de actualización a gran escala #

Copia de seguridad de base de datos

La copia de seguridad de la base de datos es imprescindible antes de iniciar cualquier proceso de actualización de Dynamics CRM. Esto es obligatorio para tener siempre la opción de retroceder en caso de cualquier obstáculo importante.

Estimación incorrecta

NO subestime el trabajo que implica un proceso de actualización de CRM.

Audite su Microsoft Dynamics CRM actual e identifique las soluciones de terceros en uso. Para estas soluciones de terceros, consulte el sitio web del desarrollador para verificar la compatibilidad con la versión de CRM de actualización prevista. Descargue la nueva solución y manténgala lista para probarla una vez que se haya completado el proceso de migración.

Organización CRM

NO actualice el CRM a ciegas.

Como primer paso, debemos preparar la organización de CRM para actualizar Microsoft Dynamics CRM.

Algunos consejos para preparar una organización

Asegúrese de que su entorno de CRM cumpla con los requisitos de los componentes de software y hardware.

Mientras actualiza CRM 4, elimine registros de la tabla enumerada. (El sistema intentará eliminar todos los registros de las siguientes tablas cuando migremos a CRM 2011, en caso de que falle, tendremos que eliminar manualmente las entradas de estas tablas). Los registros en las tablas enumeradas a continuación darán como resultado un rendimiento deficiente del sistema.

AsyncOperationBaseAsyncOperationBase

WorkflowWaitSubscriptionBase

BulkDeleteFailureBase

WorkflowLogBase

DuplicateRecordBase

WorkflowWaitSubscriptionBase

Instale el paquete acumulativo más reciente antes de actualizar CRM. Por ejemplo, para actualizar a CRM 2013, el servidor CRM 2011 debe estar en el Paquete acumulativo de actualizaciones 6, el Paquete acumulativo de actualizaciones 14 o un paquete acumulativo posterior antes de que se pueda considerar una actualización. De lo contrario, al actualizar el entorno de Dynamics CRM, se generará un error.

Para actualizar a MS CRM 2013, use la herramienta de validación de código personalizado para comprobar si hay códigos del lado del cliente no admitidos (JavaScript) que no funcionarán después de la actualización. Además, utilice la herramienta de comprobación de funciones heredadas para detectar cualquier extensión de servidor que utilice las funciones de punto final de 2007 o Microsoft Dynamics CRM 4.0.

Cada nueva versión de Microsoft Dynamics CRM presenta funcionalidades más poderosas, pero junto con el atractivo de la última y mejor versión del software, viene la preocupación de cómo el proceso de actualización afectará su negocio.

¿Los beneficios de actualizar superarán los dolores de cabeza? ¿Vale la pena una actualización? #

Ahora, aquí hay algunas trampas comunes que pueden surgir al actualizar su sistema Dynamics CRM.

La base de datos de la organización seleccionada para la importación es una versión diferente a la base de datos de la organización que está implementada actualmente. Base de datos de la organización de CRM

Para solucionar este problema, debemos instalar UR6 o superior en el entorno de MS CRM 2011 mientras se actualiza a MS CRM 2013. Después de esto, la actualización a Dynamics CRM 2016 se puede completar sin preocupaciones.

La organización no se puede actualizar.

Para solucionar este problema, debemos instalar SP1 para MS CRM 2013. Después de esto, la actualización a Dynamics CRM 2016 se puede completar sin preocupaciones.

Desinstalación del conector de Microsoft Dynamics CRM para los servicios de informes de SQL Server

Para solucionar este problema, desinstale las extensiones de informes de Microsoft Dynamics CRM 2013 desde Programas instalados en el Panel de control antes de continuar con la actualización a Dynamics CRM 2016.

La cuenta de SQL Server Reporting Service es un usuario local y no se admite cuando se actualiza de CRM 2013 a CRM 2015

Si el servicio de informes de Microsoft SQL Server 2012 se instaló a través de la configuración predeterminada, entonces la cuenta de servicio se establece en “ReportServer”. Para resolver el problema, abra el Administrador de configuración de servicios de informes y actualice la cuenta de servicio a cualquier otra cosa, como “Sistema local”. ”. Después de esto, no enfrentaremos ninguna preocupación mientras actualizamos a MS CRM 2016.

Algunos de nuestros aprendizajes han sido simples y lógicos, pero muchos podrían olvidarlos.

Durante la actualización, no siga con la actualización, hay muchas características que pueden satisfacer los requisitos actuales de la organización que se implementan a través de la personalización. Estos se pueden modelar listos para usar para adaptarse a los requisitos con versiones posteriores.

Además, considere los componentes de comunicación/integración con los que el sistema heredado o existente está conectado con Microsoft Dynamics CRM. Estos son enlaces importantes que no deberían verse afectados durante un proceso de actualización y, cuando se ven afectados, pueden afectar en gran medida la productividad.
