Configurar la autenticación globalmente #

La autenticación en Django REST Framework se puede configurar globalmente como una subclave de la variable REST_FRAMEWORK en settings.py, al igual que el resto de las configuraciones predeterminadas del marco.

REST_FRAMEWORK = {
    'DEFAULT_AUTHENTICATION_CLASSES': (
        'rest_framework.authentication.BasicAuthentication',
        'rest_framework.authentication.SessionAuthentication',
    )
}

Configuración de la autenticación para una APIView específica #

La autenticación se puede configurar para un punto final APIView específico, mediante el uso de la variable authentication_classes:

from rest_framework.authentication import SessionAuthentication, BasicAuthentication
from rest_framework.permissions import IsAuthenticated
from rest_framework.response import Response
from rest_framework.views import APIView

class ExampleView(APIView):
    authentication_classes = (SessionAuthentication, BasicAuthentication)
    permission_classes = (IsAuthenticated,)

    def get(self, request):
        content = {
            'user': unicode(request.user),  # `django.contrib.auth.User` instance.
            'auth': unicode(request.auth),  # None
        }
        return Response(content)

Uso de la autenticación básica basada en tokens #

Django REST Framework proporciona un mecanismo básico de autenticación basado en tokens que debe configurarse como una aplicación en Django antes de poder utilizarse, de modo que los tokens se creen en la base de datos y se maneje su ciclo de vida.

Agregar autenticación basada en tokens a settings.py #

INSTALLED_APPS = (
    ...
    'rest_framework.authtoken'
)

Ejecutar la migración de la base de datos #

./manage.py migrate

Crea tokens para tus usuarios #

De alguna manera, tendrás que crear un token y devolverlo:

def some_api(request):
    token = Token.objects.create(user=request.user)
    return Response({'token': token.key})

Ya existe un punto final de la API en la aplicación del token, por lo que simplemente puede agregar lo siguiente a su urls.py:

urls.py #

from rest_framework.authtoken import views
urlpatterns += [
    url(r'^auth-token/', views.obtain_auth_token)
]

Los clientes ahora pueden autenticarse #

Usando un encabezado Autorización como:

Authorization: Token 123456789

Precedido por un “token” literal y el token en sí mismo después de un espacio en blanco.

El literal se puede cambiar subclasificando TokenAuthentication y cambiando la variable de clase keyword.

Si se autentica, request.auth contendrá la instancia rest_framework.authtoken.models.BasicToken.

Configuración de la autenticación OAuth2 #

Django REST Framework no maneja OAuth, pero hay un par de módulos pip que implementan un cliente OAuth. La documentación de REST Framework sugiere uno de los siguientes módulos:

  • Kit de herramientas Django OAuth
  • Django REST Framework OAuth

Kit de herramientas OAuth de Django #

pip install django-oauth-toolkit

configuración.py #

INSTALLED_APPS = (
    ...
    'oauth2_provider',
)

REST_FRAMEWORK = {
    'DEFAULT_AUTHENTICATION_CLASSES': (
        'oauth2_provider.ext.rest_framework.OAuth2Authentication',
    )
}

urls.py #

urlpatterns = patterns(
    ...
    url(r'^o/', include('oauth2_provider.urls', namespace='oauth2_provider')),
)

Django REST Framework OAuth #

pip install djangorestframework-oauth django-oauth2-provider

configuración.py #

INSTALLED_APPS = (
    ...
    'provider',
    'provider.oauth2',
)

REST_FRAMEWORK = {
    'DEFAULT_AUTHENTICATION_CLASSES': (
        'rest_framework.authentication.OAuth2Authentication',
    )
}

urls.py #

urlpatterns = patterns(
    ...
    url(r'^oauth2/', include('provider.oauth2.urls', namespace='oauth2')),
)

Administrador #

Vaya al panel de administración y cree un nuevo Provider.Client para tener un client_id y un client_secret.

Uso de una mejor autorización basada en tokens con varios tokens de cliente #

El paquete más interesante para administrar tokens reales es django-rest-knox que admite múltiples tokens por usuario (y cancela cada token de forma independiente), así como tener soporte para la caducidad del token y varios otros mecanismos de seguridad.

django-rest-knox depende de criptografía. Puede encontrar más información sobre cómo instalarlo en: http://james1345.github.io/django-rest-knox/installation/

Instalando knox #

pip install django-rest-knox

configuración.py #

INSTALLED_APPS = (
  ...
  'rest_framework',
  'knox',
  ...
)

Aplicar migraciones:

./manage.py migrate
