Argumentos Variádicos

  • Sintaxis
  • Conceptos básicos
  • Uso avanzado

En esta página

  • Sintaxis
  • Conceptos básicos
  • Uso avanzado

Varargs, como se les conoce comúnmente, permiten que las funciones tomen un número arbitrario de argumentos sin especificación. Todos los argumentos dados a dicha función se empaquetan en una sola estructura conocida como lista vararg; que se escribe como ... en lua. Hay métodos básicos para extraer el número de argumentos dados y el valor de esos argumentos usando la función select(), pero los patrones de uso más avanzados pueden aprovechar la estructura para su utilidad completa.

Sintaxis #

  • … – Hace que la función cuya lista de argumentos en la que aparece esto sea una función variádica
  • select(what, …) – Si ‘what’ es un número en el rango 1 al número de elementos en el vararg, devuelve el elemento ‘what’th al último elemento en el vararg. El retorno será nulo si el índice está fuera de los límites. Si ‘qué’ es la cadena ‘#’, devuelve el número de elementos en el vararg.

Eficiencia

La lista vararg se implementa como una lista enlazada en la implementación del lenguaje PUC-Rio, esto significa que los índices son O(n). Eso significa que iterar sobre los elementos en un vararg usando select(), como el ejemplo a continuación, es una operación O(n^2).

for i = 1, select('#', ...) do
    print(select(i, ...))
end

Si planea iterar sobre los elementos en una lista vararg, primero empaquete la lista en una tabla. Los accesos a la tabla son O(1), por lo que la iteración es O(n) en total. O, si lo desea, vea el ejemplo foldr() de la sección de uso avanzado; utiliza la recursividad para iterar sobre una lista de vararg en O(n).

Definición de longitud de secuencia

El vararg es útil porque la longitud del vararg respeta cualquier cero pasado (o calculado) explícitamente. Por ejemplo.

function test(...)
    return select('#', ...)
end

test()             --> 0
test(nil, 1, nil)  --> 3

Sin embargo, este comportamiento entra en conflicto con el comportamiento de las tablas, donde el operador de longitud ‘#’ no funciona con ‘agujeros’ (ceros incrustados) en secuencias. El cálculo de la longitud de una mesa con agujeros no está definido y no se puede confiar en él. Entonces, dependiendo de los valores en ..., tomar la longitud de {...} puede no resultar en la respuesta ‘correcta’. En Lua 5.2+ se introdujo table.pack() para manejar esta deficiencia (hay una función en el ejemplo que implementa esta función en Lua puro).

Uso idiomático

Debido a que los varargs tienen su longitud, la gente los usa como secuencias para evitar el problema de los agujeros en las tablas. Este no era su uso previsto y la implementación de referencia de Lua no se optimiza. Aunque dicho uso se explora en los ejemplos, generalmente está mal visto.

Conceptos básicos #

Las funciones variádicas se crean utilizando la sintaxis de puntos suspensivos ... en la lista de argumentos de la definición de la función.

function id(...)
    return
end

Si llamó a esta función como id(1, 2, 3, 4, 5) entonces ... (también conocido como la lista vararg) contendría los valores 1, 2, 3, 4, 5.

Las funciones pueden tomar argumentos requeridos así como ....

function head(x, ...)
    return x
end

La forma más sencilla de extraer elementos de la lista vararg es simplemente asignarles variables.

function head3(...)
    local a, b, c = ...
    return a, b, c
end

select() también se puede usar para encontrar el número de elementos y extraer elementos de ... indirectamente.

function my_print(...)
    for i = 1, select('#', ...) do
        io.write(tostring(select(i, ...)) .. '\t')
    end
    io.write '\n'
end

... se puede empaquetar en una tabla para facilitar su uso, usando {...}. Esto coloca todos los argumentos en la parte secuencial de la tabla.

table.pack(...) también se puede usar para empaquetar la lista vararg en una tabla. La ventaja de table.pack(...) es que establece el campo n de la tabla devuelta al valor de select('#', ...). Esto es importante si su lista de argumentos puede contener ceros (consulte la sección de comentarios a continuación).

function my_tablepack(...)
    local t = {...}
    t.n = select('#', ...)
    return t
end

La lista vararg también se puede devolver desde funciones. El resultado son múltiples retornos.

function all_or_none(...)
    local t = table.pack(...)
    for i = 1, t.n do
        if not t[i] then
            return    -- return none
        end
    end
    return ...    -- return all
end

Uso avanzado #

Como se indicó en los ejemplos básicos, puede tener argumentos vinculados a variables y la lista de argumentos variables (...). Puede usar este hecho para separar recursivamente una lista como lo haría en otros idiomas (como Haskell). A continuación se muestra una implementación de foldr() que aprovecha eso. Cada llamada recursiva vincula la cabeza de la lista vararg a x y pasa el resto de la lista a una llamada recursiva. Esto desestructura la lista hasta que solo hay un argumento (select('#', ...) == 0). Después de eso, cada valor se aplica al argumento de la función f con el resultado calculado previamente.

function foldr(f, ...)
    if select('#', ...) < 2 then return ... end
    local function helper(x, ...)
        if select('#', ...) == 0 then
          return x
        end
        return f(x, helper(...))
    end
    return helper(...)
end

function sum(a, b)
    return a + b
end

foldr(sum, 1, 2, 3, 4)
--> 10

Puede encontrar otras definiciones de funciones que aprovechan este estilo de programación aquí en el número 3 al número 8.

La única estructura de datos idiomáticos de Lua es la tabla. El operador de longitud de la tabla no está definido si hay nil ubicados en cualquier lugar de una secuencia. A diferencia de las tablas, la lista vararg respeta nils explícitos como se indica en los ejemplos básicos y la sección de comentarios (lea esa sección si aún no lo ha hecho). Con poco trabajo, la lista vararg puede realizar todas las operaciones que una tabla puede realizar además de la mutación. Esto hace que la lista vararg sea una buena candidata para implementar tuplas inmutables.

function tuple(...)
    -- packages a vararg list into an easily passable value
    local co = coroutine.wrap(function(...)
        coroutine.yield()
        while true do
            coroutine.yield(...)
        end
    end)
    co(...)
    return co
end

local t = tuple((function() return 1, 2, nil, 4, 5 end)())

print(t())                 --> 1    2    nil    4    5    | easily unpack for multiple args
local a, b, d = t()        --> a = 1, b = 2, c = nil      | destructure the tuple
print((select(4, t())))    --> 4                          | index the tuple
print(select('#', t()))    --> 5                          | find the tuple arity (nil respecting)

local function change_index(tpl, i, v)
    -- sets a value at an index in a tuple (non-mutating)
    local function helper(n, x, ...)
        if select('#', ...) == 0 then
            if n == i then
                return v
            else
                return x
            end
        else
            if n == i then
                return v, helper(n+1, ...)
            else
                return x, helper(n+1, ...)
            end
        end
    end
    return tuple(helper(1, tpl()))
end

local n = change_index(t, 3, 3)
print(t())                 --> 1    2    nil    4    5
print(n())                 --> 1    2    3    4    5

La principal diferencia entre lo que está arriba y las tablas es que las tablas son mutables y tienen una semántica de puntero, donde la tupla no tiene esas propiedades. Además, las tuplas pueden contener nils explícitos y tener una operación de longitud nunca indefinida.
