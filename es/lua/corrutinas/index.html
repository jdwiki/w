Sintaxis #

  • coroutine.create(function) devuelve una coroutine (type(coroutine) == ’thread’) que contiene la función.

  • coroutine.resume(co, …) resume, o inicia la rutina. Cualquier argumento adicional dado para reanudar se devuelve desde coroutine.yield() que previamente detuvo la corrutina. Si la rutina no se ha iniciado, los argumentos adicionales se convierten en los argumentos de la función.

  • coroutine.yield(…) produce la corrutina que se está ejecutando actualmente. La ejecución se reanuda después de la llamada a coroutine.resume() que inició esa coroutine. Todos los argumentos dados a yield se devuelven desde la coroutine.resume() correspondiente que inició la coroutine.

  • coroutine.status(co) devuelve el estado de la rutina, que puede ser:

    - "dead" : the function in the coroutine has reached it's end and the coroutine cannot be resumed anymore
- "running" : the coroutine has been resumed and is running
- "normal" : the coroutine has resumed another coroutine
- "suspended" : the coroutine has yielded, and is waiting to be resumed

  • coroutine.wrap(function) devuelve una función que, cuando se llama, reanuda la coroutine que habría sido creada por coroutine.create(function).

El sistema coroutine se ha implementado en lua para emular el multihilo existente en otros lenguajes. Funciona cambiando a una velocidad extremadamente alta entre diferentes funciones para que el usuario humano piense que se ejecutan al mismo tiempo.

Crea y usa una rutina #

Todas las funciones para interactuar con corrutinas están disponibles en la tabla corrutina. Se crea una nueva rutina usando la función coroutine.create con un solo argumento: una función con el código a ejecutar:

thread1 = coroutine.create(function()
            print("honk")
        end)

print(thread1)
-->> thread: 6b028b8c

Un objeto de rutina devuelve un valor de tipo hilo, que representa una nueva rutina. Cuando se crea una nueva rutina, su estado inicial se suspende:

print(coroutine.status(thread1))
-->> suspended

Para reanudar o iniciar una rutina, se usa la función coroutine.resume, el primer argumento dado es el objeto del hilo:

coroutine.resume(thread1)
-->> honk

Ahora la corrutina ejecuta el código y termina, cambiando su estado a muerto, que no se puede reanudar.

print(coroutine.status(thread1))
-->> dead

Coroutines puede suspender su ejecución y reanudarla más tarde gracias a la función coroutine.yield:

thread2 = coroutine.create(function()
        for n = 1, 5 do
            print("honk "..n)
            coroutine.yield()
        end
    end)

Como puede ver, coroutine.yield() está presente dentro del ciclo for, ahora cuando reanudemos la coroutine, ejecutará el código hasta que alcance un coroutine.yield:

coroutine.resume(thread2)
-->> honk 1
coroutine.resume(thread2)
-->> honk 2

Después de terminar el ciclo, el estado del subproceso se convierte en muerto y no se puede reanudar. Coroutines también permite el intercambio entre datos:

thread3 = coroutine.create(function(complement)
    print("honk "..complement)
    coroutine.yield()
    print("honk again "..complement)
end)
coroutine.resume(thread3, "stackoverflow")
-->> honk stackoverflow

Si la rutina se ejecuta nuevamente sin argumentos adicionales, el complemento aún el argumento del primer currículum, en este caso “stackoverflow”:

coroutine.resume(thread3)
-->> honk again stackoverflow

Finalmente, cuando finaliza una rutina, los valores devueltos por su función van al currículum correspondiente:

thread4 = coroutine.create(function(a, b)
    local c = a+b
    coroutine.yield()
    return c
end)
coroutine.resume(thread4, 1, 2)
print(coroutine.resume(thread4))
-->> true, 3

Las corrutinas se utilizan en esta función para devolver valores a un subproceso de llamada desde lo más profundo de una llamada recursiva.

local function Combinations(l, r)
    local ll = #l
    r = r or ll
    local sel = {}
    local function rhelper(depth, last)
        depth = depth or 1
        last = last or 1
        if depth > r then
            coroutine.yield(sel)
        else
            for i = last, ll - (r - depth) do
                sel[depth] = l[i]
                rhelper(depth+1, i+1)
            end
        end
    end
    return coroutine.wrap(rhelper)
end

for v in Combinations({1, 2, 3}, 2) do
    print("{"..table.concat(v, ", ").."}")
end
--> {1, 2}
--> {1, 3}
--> {2, 3}

Las corrutinas también se pueden usar para la evaluación perezosa.

-- slices a generator 'c' taking every 'step'th output from the generator
-- starting at the 'start'th output to the 'stop'th output
function slice(c, start, step, stop)
    local _
    return coroutine.wrap(function()
        for i = 1, start-1 do
            _ = c()
        end
        for i = start, stop do
            if (i - start) % step == 0 then
                coroutine.yield(c())
            else
                _ = c()
            end
        end
    end)
end


local alphabet = {}
for c = string.byte('a'), string.byte('z') do
    alphabet[#alphabet+1] = string.char(c)
end
-- only yields combinations 100 through 102
-- requires evaluating the first 100 combinations, but not the next 5311633
local s = slice(Combinations(alphabet, 10), 100, 1, 102)
for i in s do
    print(table.concat(i))
end
--> abcdefghpr
--> abcdefghps
--> abcdefghpt

Las corrutinas se pueden usar para canalizar construcciones como se describe en Programación en Lua. El autor de PiL, Roberto Ierusalimschy, también ha publicado un artículo sobre el uso de rutinas para implementar mecánicas de control de flujo más avanzadas y generales como continuaciones.
