Una descripción general de las variables internas de Bash, dónde, cómo y cuándo usarlas.

Variables internas de Bash de un vistazo #

variables Detalles
$* / $@ Parámetros posicionales de función/script (argumentos). Expanda de la siguiente manera:

$* y $@ son lo mismo que $1 $2 ... (tenga en cuenta que generalmente no tiene sentido dejarlos sin comillas)
"$* " es lo mismo que "$1 $2 ..." 1
"$@" es lo mismo que "$1" "$2" ...
1. Los argumentos están separados por el primer carácter de $IFS, que no tiene que ser un espacio.
$# Número de parámetros posicionales pasados ​​al script o función
$! Id. de proceso del último comando (más a la derecha para canalizaciones) en el trabajo más reciente puesto en segundo plano (tenga en cuenta que no es necesariamente el mismo que el Id. de grupo de procesos del trabajo cuando el control de trabajos está habilitado)
$$ ID del proceso que ejecutó bash
$? Estado de salida del último comando
$n Parámetros posicionales, donde n=1, 2, 3, …, 9
${n} Parámetros posicionales (igual que arriba), pero n puede ser > 9
$0 En scripts, ruta con la que se invocó el script; con bash -c 'printf "%s\n" "$0"' name args': name (el primer argumento después del script en línea), de lo contrario, el argv[0] que recibió bash.
$_ Último campo del último comando
$IFS Separador de campo interno
$RUTA Variable de entorno PATH utilizada para buscar ejecutables
$OLDPWD directorio de trabajo anterior
$PWD Directorio de trabajo actual
$NOMBREFUNC Matriz de nombres de funciones en la pila de llamadas de ejecución
$BASH_SOURCE Matriz que contiene rutas de origen para elementos en la matriz FUNCNAME. Se puede utilizar para obtener la ruta del script.
$BASH_ALIASES Matriz asociativa que contiene todos los alias definidos actualmente
$BASH_REMATCH Matriz de coincidencias de la última coincidencia de expresiones regulares
$BASH_VERSION Cadena de versión de Bash
$BASH_VERSINFO Una matriz de 6 elementos con información de la versión de Bash
$BASH Ruta absoluta al propio shell Bash que se está ejecutando actualmente (determinado heurísticamente por bash basado en argv[0] y el valor de $PATH; puede ser incorrecto en casos de esquina)
$BASH_SUBSHELL Nivel de subcapa Bash
$UID ID de usuario real (no efectivo si es diferente) del proceso que ejecuta bash
$PS1 Indicación de la línea de comandos principal; consulte Uso de las variables PS*
$PS2 Solicitud de línea de comando secundaria (usada para entrada adicional)
$PS3 Indicador de línea de comando terciario (usado en el ciclo de selección)
$PS4 Indicador de línea de comando cuaternario (usado para agregar información con salida detallada)
$ALEATORIO Un entero pseudoaleatorio entre 0 y 32767
$RESPUESTA Variable utilizada por read por defecto cuando no se especifica ninguna variable. También utilizado por select para devolver el valor proporcionado por el usuario
$PIPESTATUS Variable de matriz que contiene los valores de estado de salida de cada comando en la canalización en primer plano ejecutada más recientemente.

La asignación de variables no debe tener espacios antes y después. a=123 no a = 123. El último (un signo igual rodeado de espacios) de forma aislada significa ejecutar el comando a con los argumentos = y 123, aunque también se ve en el operador de comparación de cadenas (que sintácticamente es un argumento para [ o [[ o cualquier prueba que esté usando).

ps "$@" se expande a todos los argumentos de la línea de comando como palabras separadas. Es diferente de "$*", que se expande a todos los argumentos como una sola palabra.

"$@" es especialmente útil para [buclear][2] a través de argumentos y manejar argumentos con espacios.

Considere que estamos en un script que invocamos con dos argumentos, así:

$ ./script.sh "␣1␣2␣" "␣3␣␣4␣"

Las variables ‘$*’ o ‘$@’ se expandirán a ‘$1␣$2’, que a su vez se expandirán a ‘1␣2␣3␣4’, de modo que el ciclo a continuación:

for var in $*; do # same for var in $@; do
    echo \<"$var"\>
done

se imprimirá para ambos

<1>
<2>
<3>
<4>

Mientras que "$*" se expandirá a "$1␣$2" que a su vez se expandirá a "␣1␣2␣␣␣3␣␣4␣" y así el bucle:

for var in "$*"; do
    echo \<"$var"\>   
done

solo invocará echo una vez e imprimirá

<␣1␣2␣␣␣3␣␣4␣>

Y finalmente "$@" se expandirá a "$1" "$2", que se expandirá a "␣1␣2␣" "␣3␣␣4␣" y así el ciclo

for var in "$@"; do
    echo \<"$var"\>
done

imprimirá

<␣1␣2␣>
<␣3␣␣4␣>

preservando así tanto el espaciado interno en los argumentos como la separación de los argumentos. Tenga en cuenta que la construcción para var en "$@"; do ... es tan común e idiomático que es el valor predeterminado para un bucle for y se puede acortar a for var; hacer ....

[2]: https://www.wikiod.com/es/bash/estructuras-de-control#Bucle sobre una matriz [3]: https://www.wikiod.com/es/bash/arreglos

ps Para obtener el número de argumentos de línea de comando o parámetros posicionales, escriba:

#!/bin/bash
echo "$#"

Cuando se ejecuta con tres argumentos, el ejemplo anterior dará como resultado el resultado:

~> $ ./testscript.sh firstarg secondarg thirdarg
3

$NOMBREFUNCIÓN #

Para obtener el nombre de la función actual, escriba:

my_function()
{
    echo "This function is $FUNCNAME"    # This will output "This function is my_function"
}

Esta instrucción no devolverá nada si la escribe fuera de la función:

my_function

echo "This function is $FUNCNAME"    # This will output "This function is"

$CASA #

El directorio de inicio del usuario.

~> $ echo $HOME
/home/user

$SIF #

Contiene la cadena del separador de campo interno que bash usa para dividir cadenas cuando realiza un bucle, etc. El valor predeterminado son los caracteres de espacio en blanco: \n (nueva línea), \t (tabulador) y espacio. Cambiar esto a otra cosa le permite dividir cadenas usando diferentes caracteres:

IFS=","
INPUTSTR="a,b,c,d"
for field in ${INPUTSTR}; do
    echo $field
done

La salida de lo anterior es:

a
b
c
d

Notas:

  • Esto es responsable del fenómeno conocido como [división de palabras] (https://www.wikiod.com/es/bash/division-de-palabras#¿Qué, cuándo y por qué?).

$OLDPWD #

OLDPWD (OLDPrintWorkingDirectory) contiene el directorio antes del último comando cd:

~> $ cd directory
directory> $ echo $OLDPWD
/home/user

$PWD #

PWD (PrintWorkingDirectory) El directorio de trabajo actual en el que se encuentra en este momento:

~> $ echo $PWD
/home/user
~> $ cd directory
directory> $ echo $PWD
/home/user/directory

$1 $2 $3 etc… #

Parámetros posicionales pasados ​​al script desde la línea de comando o una función:

#!/bin/bash
# $n is the n'th positional parameter
echo "$1"
echo "$2"
echo "$3"

La salida de lo anterior es:

~> $ ./testscript.sh firstarg secondarg thirdarg
firstarg
secondarg
thirdarg

Si el número de argumentos posicionales es mayor que nueve, se deben usar llaves.

#  "set -- " sets positional parameters
set -- 1 2 3 4 5 6 7 8 nine ten eleven twelve
# the following line will output 10 not 1 as the value of $1 the digit 1
# will be concatenated with the following 0
echo $10   # outputs 1
echo ${10} # outputs ten
# to show this clearly:
set -- arg{1..12}
echo $10 
echo ${10}

ps Devolverá todos los parámetros posicionales en una sola cadena.

script de prueba.sh:

#!/bin/bash
echo "$*"

Ejecute el script con varios argumentos:

./testscript.sh firstarg secondarg thirdarg

Producción:

firstarg secondarg thirdarg

ps El ID de proceso (pid) del último trabajo ejecutado en segundo plano:

~> $ ls &
testfile1 testfile2
[1]+  Done                    ls
~> $ echo $!
21715

ps El estado de salida de la última función o comando ejecutado. Por lo general, 0 significará OK, cualquier otra cosa indicará una falla:

~> $ ls *.blah;echo $?
ls: cannot access *.blah: No such file or directory
2
~> $ ls;echo $?
testfile1 testfile2
0

$$ #

El ID de proceso (pid) del proceso actual:

~> $ echo $$
13246

$TAMAÑO HISTORICO #

El número máximo de comandos recordados:

~> $ echo $HISTSIZE
1000

$BASHPID #

ID de proceso (pid) de la instancia actual de Bash. No es lo mismo que la variable $$, pero a menudo da el mismo resultado. Esto es nuevo en Bash 4 y no funciona en Bash 3.

~> $ echo "\$\$ pid = $$  BASHPID = $BASHPID"
$$ pid = 9265  BASHPID = 9265

$BASH_ENV #

Una variable de entorno que apunta al archivo de inicio de Bash que se lee cuando se invoca un script.

$BASH_VERSINFO #

Una matriz que contiene la información de la versión completa dividida en elementos, mucho más conveniente que $BASH_VERSION si solo está buscando la versión principal:

~> $ for ((i=0; i<=5; i++)); do echo "BASH_VERSINFO[$i] = ${BASH_VERSINFO[$i]}"; done
BASH_VERSINFO[0] = 3
BASH_VERSINFO[1] = 2
BASH_VERSINFO[2] = 25
BASH_VERSINFO[3] = 1
BASH_VERSINFO[4] = release
BASH_VERSINFO[5] = x86_64-redhat-linux-gnu

$BASH_VERSIÓN #

Muestra la versión de bash que se está ejecutando, esto le permite decidir si puede usar alguna función avanzada:

~> $ echo $BASH_VERSION
4.1.2(1)-release

$EDITOR #

El editor predeterminado que será utilizado por cualquier script o programa, generalmente vi o emacs.

~> $ echo $EDITOR
vi

$NOMBRE DEL HOST #

El nombre de host asignado al sistema durante el inicio.

~> $ echo $HOSTNAME
mybox.mydomain.com

$TIPO DE HOST #

Esta variable identifica el hardware, puede ser útil para determinar qué binarios ejecutar:

~> $ echo $HOSTTYPE
x86_64

$TIPOMAQUINA #

Similar a $HOSTTYPE anterior, esto también incluye información sobre el sistema operativo y el hardware

~> $ echo $MACHTYPE
x86_64-redhat-linux-gnu

$OSTIPO #

Devuelve información sobre el tipo de sistema operativo que se ejecuta en la máquina, p.

~> $ echo $OSTYPE
linux-gnu

$RUTA #

La ruta de búsqueda para encontrar archivos binarios para comandos. Los ejemplos comunes incluyen /usr/bin y /usr/local/bin.

Cuando un usuario o script intenta ejecutar un comando, se buscan las rutas en $PATH para encontrar un archivo coincidente con permiso de ejecución.

Los directorios en $PATH están separados por un carácter :.

~> $ echo "$PATH"
/usr/kerberos/bin:/usr/local/bin:/bin:/usr/bin

Entonces, por ejemplo, dado el $PATH anterior, si escribe lss en el indicador, el shell buscará /usr/kerberos/bin/lss, luego /usr/local/bin/lss , luego /bin/lss, luego /usr/bin/lss, en este orden, antes de concluir que no existe tal comando.

$PPID #

El Id. de proceso (pid) del script o shell primario, es decir, el proceso que invocó el script o shell actual.

~> $ echo $$
13016
~> $ echo $PPID
13015

$SEGUNDOS #

El número de segundos que se ha estado ejecutando un script. Esto puede ser bastante grande si se muestra en el shell:

~> $ echo $SECONDS
98834

$SHELLOPTS #

Se proporciona una lista de solo lectura de las opciones de bash al inicio para controlar su comportamiento:

~> $ echo $SHELLOPTS
braceexpand:emacs:hashall:histexpand:history:interactive-comments:monitor

psMuestra el último campo del último comando ejecutado, útil para obtener algo para pasar a otro comando:

 ~> $ ls *.sh;echo $_
testscript1.sh  testscript2.sh
testscript2.sh
Proporciona la ruta del script si se usa antes de cualquier otro comando:

prueba.sh:

#!/bin/bash
echo "$_"

Producción:

~> $ ./test.sh # running test.sh
./test.sh

Nota: Esta no es una forma infalible de obtener la ruta del script

$ALEATORIO #

Cada vez que se hace referencia a este parámetro, se genera un número entero aleatorio entre 0 y 32767. Asignar un valor a esta variable inicia el generador de números aleatorios (fuente).

~> $ echo $RANDOM 
27119
~> $ echo $RANDOM 
1349

$GRUPOS #

Una matriz que contiene el número de grupos en los que se encuentra el usuario:

#!/usr/bin/env bash
echo You are assigned to the following groups:
for group in ${GROUPS[@]}; do
   IFS=: read -r name dummy number members < <(getent group $group )
   printf "name: %-10s number: %-15s members: %s\n" "$name" "$number" "$members" 
done

$Lino #

Muestra el número de línea en el script actual. Principalmente útil al depurar scripts.

#!/bin/bash
# this is line 2
echo something  # this is line 3
echo $LINENO # Will output 4

$SHLVL #

Cuando se ejecuta el comando bash, se abre un nuevo shell. La variable de entorno $SHLVL contiene el número de niveles de shell sobre los que se ejecuta el shell current.

En una ventana de terminal nueva, ejecutar el siguiente comando producirá diferentes resultados según la distribución de Linux en uso.

echo $SHLVL

Usando Fedora 25, la salida es “3”. Esto indica que al abrir un nuevo shell, se ejecuta un comando bash inicial y realiza una tarea. El comando bash inicial ejecuta un proceso secundario (otro comando bash) que, a su vez, ejecuta un comando bash final para abrir el nuevo shell. Cuando se abre el nuevo shell, se ejecuta como un proceso secundario de otros 2 procesos de shell, por lo tanto, la salida de “3”.

En el siguiente ejemplo (dado que el usuario ejecuta Fedora 25), la salida de $SHLVL en un nuevo shell se establecerá en “3”. A medida que se ejecuta cada comando bash, $SHLVL se incrementa en uno.

~> $ echo $SHLVL
3
~> $ bash
~> $ echo $SHLVL
4
~> $ bash
~> $ echo $SHLVL
5

Se puede ver que ejecutar el comando ‘bash’ (o ejecutar un script bash) abre un nuevo shell. En comparación, obtener un script ejecuta el código en el shell actual.

prueba1.sh

#!/usr/bin/env bash
echo "Hello from test1.sh. My shell level is $SHLVL"
source "test2.sh"

prueba2.sh

#!/usr/bin/env bash
echo "Hello from test2.sh. My shell level is $SHLVL"

corre.sh

#!/usr/bin/env bash
echo "Hello from run.sh. My shell level is $SHLVL"
./test1.sh

Ejecutar:

chmod +x test1.sh && chmod +x run.sh
./run.sh

Producción:

Hello from run.sh. My shell level is 4
Hello from test1.sh. My shell level is 5
Hello from test2.sh. My shell level is 5
