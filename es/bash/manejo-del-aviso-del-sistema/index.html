Sintaxis #

  • export PS1=“algo” # muestra cuando bash espera que se escriba un comando
  • export PS2=“otra cosa” # mostrado cuando la declaración se extiende a más líneas
  • exportar PS3 = “mensaje de pregunta para la declaración de selección” # mensaje rara vez utilizado para seleccionar. Primero configure PS3 según sus necesidades, luego llame a select. Ver ayuda seleccionar
  • export PS4=“principalmente útil para la depuración; número de línea y así sucesivamente” # utilizado para depurar scripts de bash.

Parámetros #

Escapar Detalles
\a Un personaje de campana.
\d La fecha, en formato “Día de la semana Mes Fecha” (por ejemplo, “martes 26 de mayo”).
\D{FORMATO} El FORMATO se pasa a `strftime’(3) y el resultado se inserta en la cadena de solicitud; un FORMATO vacío da como resultado una representación de tiempo específica de la localidad. Los frenos son necesarios.
\e Un personaje de escape. \033 también funciona, por supuesto.
\ h El nombre de host, hasta el primer ‘.’. (es decir, sin parte de dominio)
\H El nombre de host eventualmente con parte de dominio
\ j El número de trabajos administrados actualmente por el shell.
\l El nombre base del nombre del dispositivo terminal del shell.
\n Una nueva línea.
\r Un retorno de carro.
\s El nombre del shell, el nombre base de ‘$0’ (la parte que sigue a la barra inclinada final).
\t La hora, en formato HH:MM:SS de 24 horas.
\T La hora, en formato HH:MM:SS de 12 horas.
@ La hora, en formato de 12 horas am/pm.
\A La hora, en formato HH:MM de 24 horas.
\u El nombre de usuario del usuario actual.
\v La versión de Bash (por ejemplo, 2.00)
\V El lanzamiento de Bash, versión + nivel de parche (por ejemplo, 2.00.0)
\w El directorio de trabajo actual, con $HOME abreviado con una tilde (usa la variable $PROMPT_DIRTRIM).
\W El nombre base de $PWD, con $HOME abreviado con una tilde.
! El número de historial de este comando.
# El número de comando de este comando.
$ Si el uid efectivo es 0, #, de lo contrario $.
\NNN El carácter cuyo código ASCII es el valor octal NNN.
\ Una barra invertida.
\[ Comience una secuencia de caracteres no imprimibles. Esto podría usarse para incrustar una secuencia de control de terminal en el aviso.
\] Finaliza una secuencia de caracteres no imprimibles.

Usando la variable de entorno PROMPT_COMMAND #

Cuando se realiza el último comando en una instancia de bash interactiva, se muestra la variable PS1 evaluada. Antes de mostrar PS1 bash, mira si PROMPT_COMMAND está configurado. Este valor de esta var debe ser un programa o script invocable. Si se establece esta variable, se llama a este programa/secuencia de comandos ANTES de que se muestre el aviso de PS1.

# just a stupid function, we will use to demonstrate
# we check the date if Hour is 12 and Minute is lower than 59
lunchbreak(){ 
   if (( $(date +%H) == 12 && $(date +%M) < 59 )); then 
      # and print colored \033[ starts the escape sequence 
      # 5; is blinking attribute
      # 2; means bold
      # 31 says red
      printf "\033[5;1;31mmind the lunch break\033[0m\n";
   else
      printf "\033[33mstill working...\033[0m\n"; 
   fi; 
}

# activating it
export PROMPT_COMMAND=lunchbreak

Uso de PS2 #

PS2 se muestra cuando un comando se extiende a más de una línea y bash espera más pulsaciones de teclas. También se muestra cuando se ingresa un comando compuesto como while…do..done y similares.

export PS2="would you please complete this command?\n" 
# now enter a command extending to at least two lines to see PS2

Uso de PS3 #

Cuando se ejecuta la declaración de selección, muestra los elementos dados con el prefijo de un número y luego muestra el indicador de PS3:

export PS3="  To choose your language type the preceding number : "
select lang in EN CA FR DE; do
   # check input here until valid.
   break
done

Uso de PS4 #

PS4 se muestra cuando bash está en modo de depuración.

#!/usr/bin/env bash

# switch on debugging
set -x

# define a stupid_func
stupid_func(){
   echo I am line 1 of stupid_func
   echo I am line 2 of stupid_func
}

# setting the PS4 "DEBUG" prompt
export PS4='\nDEBUG level:$SHLVL subshell-level: $BASH_SUBSHELL \nsource-file:${BASH_SOURCE} line#:${LINENO} function:${FUNCNAME[0]:+${FUNCNAME[0]}(): }\nstatement: '

# a normal statement
echo something

# function call
stupid_func

# a pipeline of commands running in a subshell 
( ls -l | grep 'x' )

Uso de PS1 #

PS1 es el indicador normal del sistema que indica que bash espera a que se ingresen los comandos. Comprende algunas secuencias de escape y puede ejecutar funciones o programas. Como bash tiene que posicionar el cursor después de la indicación de la pantalla, necesita saber cómo calcular la longitud efectiva de la cadena de indicación. Para indicar secuencias de caracteres que no se imprimen dentro de la variable PS1, se utilizan llaves de escape: \[ una secuencia de caracteres que no se imprimen \]. Todo lo dicho es válido para todas las PS* vars.

(El símbolo de intercalación negro indica el cursor)

#everything not being an escape sequence will be literally printed
export PS1="literal sequence "  # Prompt is now:
literal sequence ▉

# \u == user \h == host \w == actual working directory
# mind the single quotes avoiding interpretation by shell
export PS1='\u@\h:\w > ' # \u == user, \h == host, \w actual working dir
looser@host:/some/path > ▉

# executing some commands within PS1
# following line will set foreground color to red, if user==root, 
# else it resets attributes to default
# $( (($EUID == 0)) &&  tput setaf 1)
# later we do reset attributes to default with
# $(  tput sgr0 )
# assuming being root:
PS1="\[$( (($EUID == 0)) &&  tput setaf 1 \]\u\[$(tput sgr0)\]@\w:\w \$ "
looser@host:/some/path > ▉  # if not root else <red>root<default>@host....
