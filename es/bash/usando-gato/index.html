Sintaxis #

  • gato [OPCIONES]… [ARCHIVO]…

Parámetros #

Opción Detalles
-n Imprimir números de línea
-v Mostrar caracteres no imprimibles usando notación ^ y M- excepto LFD y TAB
-T Mostrar caracteres TAB como ^I
-E Mostrar caracteres de avance de línea (LF) como $
-e Igual que -vE
-b Número de líneas de salida no vacías, anula -n
-A equivalente a -vET
-s suprimir líneas de salida vacías repetidas, s se refiere a apretar

cat puede leer tanto archivos como entradas estándar y los concatena a la salida estándar

Concatenar archivos #

Este es el propósito principal de cat.

cat file1 file2 file3 > file_all

cat también se puede usar de manera similar para concatenar archivos como parte de una canalización, p.

cat file1 file2 file3 | grep foo

Imprimir el contenido de un archivo #

cat file.txt

imprimirá el contenido de un archivo.

Si el archivo contiene caracteres que no son ASCII, puede mostrar esos caracteres simbólicamente con cat -v. Esto puede ser muy útil para situaciones en las que los caracteres de control serían invisibles.

cat -v unicode.txt

Sin embargo, muy a menudo, para uso interactivo, es mejor usar un localizador interactivo como “menos” o “más”. (less es mucho más poderoso que more y se recomienda usar less con más frecuencia que more).

less file.txt

Para pasar el contenido de un archivo como entrada a un comando. Un enfoque generalmente visto como mejor (UUOC) es usar la redirección.

tr A-Z a-z <file.txt   # as an alternative to cat file.txt | tr A-Z a-z

En caso de que el contenido deba enumerarse hacia atrás desde su final, se puede usar el comando tac:

tac file.txt

Si desea imprimir el contenido con números de línea, utilice -n con cat:

cat -n file.txt

Para mostrar el contenido de un archivo en un formato byte por byte completamente inequívoco, la solución estándar es un volcado hexadecimal. Esto es bueno para fragmentos muy breves de un archivo, como cuando no conoce la codificación precisa. La utilidad de volcado hexadecimal estándar es od -cH, aunque la representación es un poco engorrosa; los reemplazos comunes incluyen xxd y hexdump.

$ printf 'Hëllö wörld' | xxd
0000000: 48c3 ab6c 6cc3 b620 77c3 b672 6c64       H..ll.. w..rld

Escribir en un archivo #

cat >file

Le permitirá escribir el texto en la terminal que se guardará en un archivo llamado archivo.

cat >>file

hará lo mismo, excepto que agregará el texto al final del archivo.

N.B: Ctrl+D para dejar de escribir texto en la terminal (Linux)

Se puede usar un documento here para insertar el contenido de un archivo en una línea de comando o un script:

cat <<END >file
Hello, World.
END

El token después del símbolo de redirección << es una cadena arbitraria que debe aparecer sola en una línea (sin espacios en blanco al principio o al final) para indicar el final del documento aquí. Puede agregar comillas para evitar que el shell realice la sustitución de comandos y la interpolación de variables:

cat <<'fnord'
Nothing in `here` will be $changed
fnord

(Sin las comillas, ‘aquí’ se ejecutaría como un comando, y ‘$cambiado’ se sustituiría con el valor de la variable ‘cambiado’, o nada, si no estuviera definida).

Mostrar números de línea con salida #

Use la bandera --number para imprimir los números de línea antes de cada línea. Alternativamente, -n hace lo mismo.

$ cat --number file

 1  line 1
 2  line 2
 3
 4  line 4
 5  line 5

Para omitir líneas vacías al contar líneas, use --number-nonblank, o simplemente -b.

$ cat -b file

 1  line 1
 2  line 2

 3  line 4
 4  line 5

Leer desde la entrada estándar #

cat < file.txt

La salida es la misma que cat file.txt, pero lee el contenido del archivo desde la entrada estándar en lugar de leerlo directamente desde el archivo.

printf "first line\nSecond line\n" | cat -n

El comando echo antes de | genera dos líneas. El comando cat actúa sobre la salida para agregar números de línea.

Mostrar caracteres no imprimibles #

Esto es útil para ver si hay caracteres no imprimibles o caracteres que no sean ASCII.

p.ej. Si ha copiado y pegado el código de la web, es posible que tenga comillas como ” en lugar de " estándar.

$ cat -v file.txt
$ cat -vE file.txt # Useful in detecting trailing spaces.

p.ej.

$ echo '”     ' | cat -vE # echo | will be replaced by actual file.
M-bM-^@M-^]     $

También puede usar cat -A (A para todos) que es equivalente a cat -vET. Mostrará caracteres TAB (que se muestran como ^I), caracteres no imprimibles y el final de cada línea:

$ echo '” `' | cat -A
M-bM-^@M-^]^I`$

Concatenar archivos comprimidos con gzip #

Los archivos comprimidos con gzip se pueden concatenar directamente en archivos comprimidos con gzip más grandes.

cat file1.gz file2.gz file3.gz > combined.gz

Esta es una propiedad de gzip que es menos eficiente que concatenar los archivos de entrada y comprimir el resultado con gzip:

cat file1 file2 file3 | gzip > combined.gz

Una demostración completa:

echo 'Hello world!' > hello.txt
echo 'Howdy world!' > howdy.txt
gzip hello.txt 
gzip howdy.txt

cat hello.txt.gz howdy.txt.gz > greetings.txt.gz

gunzip greetings.txt.gz

cat greetings.txt

Lo que resulta en

Hello world!
Howdy world!

Tenga en cuenta que saludos.txt.gz es un archivo único y se descomprime como el archivo único saludos.txt. Compare esto con tar -czf hello.txt howdy.txt > greetings.tar.gz, que mantiene los archivos separados dentro del tarball.
