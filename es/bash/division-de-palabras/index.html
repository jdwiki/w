Sintaxis #

  • Establecer IFS en nueva línea: IFS=$’\n'
  • Establecer IFS en cadena nula: IFS=
  • Establecer IFS en /carácter: IFS=/

Parámetros #

Parámetro Detalles
IFS Separador de campo interno
-x Imprimir comandos y sus argumentos a medida que se ejecutan (opción Shell)
  • La división de palabras no se realiza durante las asignaciones, por ejemplo, newvar=$var
  • La división de palabras no se realiza en la construcción [[ ... ]]
  • Use comillas dobles en las variables para evitar la división de palabras

Malos efectos de la división de palabras #

$ a='I am a string with spaces'
$ [ $a = $a ] || echo "didn't match"
bash: [: too many arguments
didn't match

[ $a = $a ] fue interpretado como [ Soy una cadena con espacios = Soy una cadena con espacios ]. [ es el comando test para el cual Soy una cadena con espacios no es un único argumento, sino 6 argumentos!!

$ [ $a = something ] || echo "didn't match"
bash: [: too many arguments
didn't match

[ $a = algo ] fue interpretado como [ Soy una cadena con espacios = algo ]

$ [ $(grep . file) = 'something' ]
bash: [: too many arguments

El comando grep devuelve una cadena multilínea con espacios, así que puedes imaginar cuántos argumentos hay…:D

Consulte [qué, cuándo y por qué](https://www.wikiod.com/es/bash/division-de-palabras#¿Qué, cuándo y por qué?) para obtener información básica.

Dividir con IFS #

Para ser más claros, creemos un script llamado showarg:

#!/usr/bin/env bash
printf "%d args:" $#
printf " <%s>" "$@"
echo

Ahora veamos las diferencias:

$ var="This is an example"
$ showarg $var
4 args: <This> <is> <an> <example>

$var se divide en 4 argumentos. IFS son caracteres de espacio en blanco y, por lo tanto, la división de palabras se produjo en espacios

$ var="This/is/an/example"
$ showarg $var
1 args: <This/is/an/example>

En la división de palabras anterior no se produjo porque no se encontraron los caracteres IFS.

Ahora configuremos IFS=/

$ IFS=/
$ var="This/is/an/example"
$ showarg $var
4 args: <This> <is> <an> <example>

El $var se está dividiendo en 4 argumentos, no en un solo argumento.

¿Qué, cuándo y por qué? #

Cuando el shell realiza expansión de parámetros, sustitución de comandos, expansión de variables o aritmética, busca límites de palabras en el resultado. Si se encuentra algún límite de palabra, el resultado se divide en varias palabras en esa posición. El límite de la palabra está definido por una variable de shell ‘IFS’ (separador de campo interno). Los valores predeterminados para IFS son espacio, tabulador y nueva línea, es decir, la división de palabras se producirá en estos tres caracteres de espacio en blanco si no se evita explícitamente.

set -x
var='I am
a
multiline string'
fun() {
    echo "-$1-"
    echo "*$2*"
    echo ".$3."
}
fun $var

En el ejemplo anterior, así es como se ejecuta la función fun:

fun I am a multiline string

$var se divide en 5 argumentos, solo se imprimirán I, am y a.

IFS y división de palabras #

Consulte [qué, cuándo y por qué](https://www.wikiod.com/es/bash/division-de-palabras#¿Qué, cuándo y por qué?) si no conoce el afiliación de IFS a división de palabras

** configuremos el IFS solo en caracteres de espacio: **

set -x
var='I am
a
multiline string'
IFS=' '
fun() {
    echo "-$1-"
    echo "*$2*"
    echo ".$3."
}
fun $var

Esta vez, la división de palabras solo funcionará en espacios. La función fun se ejecutará así:

fun I 'am
a
multiline' string

$var se divide en 3 argumentos. Se imprimirán I, am\na\nmultiline y string

Configuremos el IFS solo en nueva línea:

IFS=$'\n'
...

Ahora el fun se ejecutará como:

fun 'I am' a 'multiline string'

$var se divide en 3 argumentos. Se imprimirá I am, a, multiline string

Veamos qué sucede si establecemos IFS en cadena nula:

IFS=
...

Esta vez el fun se ejecutará así:

fun 'I am
a
multiline string'

$var no se divide, es decir, sigue siendo un único argumento.

Puede evitar la división de palabras configurando el IFS en cadena nula

Una forma general de evitar la división de palabras es usar comillas dobles:

fun "$var"

evitará la división de palabras en todos los casos discutidos anteriormente, es decir, la función fun se ejecutará con un solo argumento.

Utilidad de la división de palabras #

Hay algunos casos en los que la división de palabras puede ser útil:

Llenando matriz:

arr=($(grep -o '[0-9]\+' file))

Esto llenará arr con todos los valores numéricos encontrados en archivo

Recorriendo palabras separadas por espacios:

words='foo bar baz'
for w in $words;do
    echo "W: $w"
done

Producción:

W: foo
W: bar
W: baz

Pasar parámetros separados por espacios que no contienen espacios en blanco:

packs='apache2 php php-mbstring php-mysql'
sudo apt-get install $packs

o

packs='
apache2
php
php-mbstring
php-mysql
'
sudo apt-get install $packs

Esto instalará los paquetes. Si comillas dos veces los $paquetes, arrojará un error.

Unquoetd $packs está enviando todos los nombres de paquetes separados por espacios como argumentos a apt-get, mientras que entrecomillado enviará la cadena $packs como un solo argumento y luego apt-get intentará instalar un paquete llamado apache2 php php-mbstring php-mysql (para el primero) que obviamente no existe

Consulte [qué, cuándo y por qué](https://www.wikiod.com/es/bash/division-de-palabras#¿Qué, cuándo y por qué?) para obtener información básica.
