Sintaxis #

  • [ “$1” = “$2” ] #Un corchete “[” es en realidad un comando. Por eso requiere un espacio antes y después.
  • test “$1” = “$2” #Test es un sinónimo del comando “[”

Parámetros #

Parámetro a [ o prueba Detalles
Operadores de archivos Detalles
-e "$archivo" Devuelve verdadero si el archivo existe.
-d "$archivo" Devuelve verdadero si el archivo existe y es un directorio
-f "$archivo" Devuelve verdadero si el archivo existe y es un archivo regular
-h "$archivo" Devuelve verdadero si el archivo existe y es un enlace simbólico
Comparadores de cadenas Detalles
-z "$cadena" Verdadero si la longitud de la cadena es cero
-n "$str Verdadero si la longitud de la cadena no es cero
"$cadena" = "$cadena2" Verdadero si la cadena $str es igual a la cadena $str2. No es mejor para números enteros. Puede funcionar pero será inconsistente
"$cadena" != "$cadena2" Verdadero si las cadenas no son iguales
Comparadores de enteros       Detalles
"$int1" -eq "$int2" Verdadero si los enteros son iguales
"$int1" -ne "$int2" Verdadero si los enteros no son iguales
"$int1" -gt "$int2" Verdadero si int1 es mayor que int 2
"$int1" -ge "$int2" Verdadero si int1 es mayor o igual que int2
"$int1" -lt "$int2" Verdadero si int1 es menor que int 2
"$int1" -le "$int2" Verdadero si int1 es menor o igual que int2

Hay muchos parámetros de comparación disponibles en bash. Todavía no todos están listados aquí.

Si declaración #

if [[ $1 -eq 1 ]]; then
    echo "1 was passed in the first parameter"
elif [[ $1 -gt 2 ]]; then
    echo "2 was not passed in the first parameter"
else
    echo "The first parameter was not 1 and is not more than 2."
fi

El cierre fi es necesario, pero las cláusulas elif y/o else pueden omitirse.

Los puntos y comas antes de entonces son sintaxis estándar para combinar dos comandos en una sola línea; solo se pueden omitir si then se mueve a la siguiente línea.

Es importante comprender que los corchetes [[ no son parte de la sintaxis, pero se tratan como un comando; es el código de salida de este comando el que se está probando. Por lo tanto, siempre debe incluir espacios alrededor de los corchetes.

Esto también significa que se puede probar el resultado de cualquier comando. Si el código de salida del comando es un cero, la declaración se considera verdadera.

if grep "foo" bar.txt; then
    echo "foo was found"
else
    echo "foo was not found"
fi

Las expresiones matemáticas, cuando se colocan entre paréntesis dobles, también devuelven 0 o 1 de la misma manera, y también se pueden probar:

if (( $1 + 5 > 91 )); then
    echo "$1 is greater than 86"
fi

También puede encontrarse con declaraciones if con corchetes simples. Estos se definen en el estándar POSIX y se garantiza que funcionan en todos los shells compatibles con POSIX, incluido Bash. La sintaxis es muy similar a la de Bash:

if [ "$1" -eq 1 ]; then
    echo "1 was passed in the first parameter"
elif [ "$1" -gt 2 ]; then
    echo "2 was not passed in the first parameter"
else
    echo "The first parameter was not 1 and is not more than 2."
fi

Bucle sobre una matriz #

Bucle for:

arr=(a b c d e f)
for i in "${arr[@]}";do
    echo "$i"
done

O

for ((i=0;i<${#arr[@]};i++));do
    echo "${arr[$i]}" 
done

Bucle mientras:

i=0
while [ $i -lt ${#arr[@]} ];do
    echo "${arr[$i]}"
    i=$(expr $i + 1)
done

O

i=0
while (( $i < ${#arr[@]} ));do
    echo "${arr[$i]}"
    ((i++))
done

Ejecución condicional de listas de comandos #

Cómo usar la ejecución condicional de listas de comandos #

Cualquier comando, expresión o función integrados, así como cualquier comando o script externo, se puede ejecutar de forma condicional mediante los operadores &&(and) y ||(or).

Por ejemplo, esto solo imprimirá el directorio actual si el comando cd fue exitoso.

cd my_directory && pwd

Del mismo modo, esto se cerrará si falla el comando cd, evitando una catástrofe:

cd my_directory || exit
rm -rf *

Al combinar varias declaraciones de esta manera, es importante recordar que (a diferencia de muchos lenguajes de estilo C) [estos operadores no tienen precedencia y son asociativos por la izquierda] (https://www.gnu.org/software/bash/manual/ html_node/Lists.html#Lists).

Por lo tanto, esta declaración funcionará como se esperaba…

cd my_directory && pwd || echo "No such directory"
  • Si cd tiene éxito, && pwd se ejecuta y se imprime el nombre del directorio de trabajo actual. A menos que pwd falle (una rareza), el || echo ... no se ejecutará.
  • Si el cd falla, el && pwd se omitirá y el || echo ... se ejecutará.

Pero esto no lo hará (si estás pensando si... entonces... si no)…

cd my_directory && ls || echo "No such directory"
  • Si el cd falla, el && ls se omite y el || echo ... se ejecuta.
  • Si cd tiene éxito, se ejecuta && ls.
  • Si ls tiene éxito, || echo ... se ignora. (Hasta ahora, todo bien)
  • PERO… si ls falla, || echo ... también se ejecutará.

    It is the ls, not the cd, that is the previous command.

¿Por qué usar la ejecución condicional de listas de comandos? #

La ejecución condicional es un poco más rápida que if...then pero su principal ventaja es permitir que las funciones y los scripts salgan antes de tiempo, o “cortocircuito”.

A diferencia de muchos lenguajes como C donde la memoria se asigna explícitamente para estructuras y variables y demás (y, por lo tanto, debe desasignarse), bash maneja esto bajo las sábanas. En la mayoría de los casos, no tenemos que limpiar nada antes de salir de la función. Una declaración return desasignará todo lo local a la función y recogerá la ejecución en la dirección de retorno en la pila.

Regresar de las funciones o salir de los scripts lo antes posible puede mejorar significativamente el rendimiento y reducir la carga del sistema al evitar la ejecución innecesaria de código. Por ejemplo…

my_function () {

    ### ALWAYS CHECK THE RETURN CODE

    # one argument required. "" evaluates to false(1)
    [[ "$1" ]]             || return 1

    # work with the argument. exit on failure
    do_something_with "$1" || return 1
    do_something_else      || return 1

    # Success! no failures detected, or we wouldn't be here
    return 0
}

Mientras Bucle #

#! /bin/bash

i=0

while [ $i -lt 5 ] #While i is less than 5
do
    echo "i is currently $i"
    i=$[$i+1] #Not the lack of spaces around the brackets. This makes it a not a test expression
done #ends the loop

Observe que hay espacios alrededor de los corchetes durante la prueba (después de la instrucción while). Estos espacios son necesarios.

Este bucle produce:

i is currently 0
i is currently 1
i is currently 2
i is currently 3
i is currently 4

For Loop con sintaxis de estilo C #

El formato básico del bucle for de estilo C es:

for (( variable assignment; condition; iteration process ))

Notas:

  • La asignación de la variable dentro del bucle for de estilo C puede contener espacios a diferencia de la asignación habitual
  • Las variables dentro del ciclo for de estilo C no están precedidas por $.

Ejemplo:

for (( i = 0; i < 10; i++ ))
do
    echo "The iteration number is $i"
done

También podemos procesar múltiples variables dentro del bucle for de estilo C:

for (( i = 0, j = 0; i < 10; i++, j = i * i ))
do
    echo "The square of $i is equal to $j"
done

Hasta bucle #

Hasta que el bucle se ejecute hasta que la condición sea verdadera

i=5
until [[ i -eq 10 ]]; do #Checks if i=10
    echo "i=$i" #Print the value of i
    i=$((i+1)) #Increment i by 1
done

Producción:

i=5
i=6
i=7
i=8
i=9

Cuando i llega a 10, la condición hasta que el bucle se vuelve verdadero y el bucle termina.

continuar y romper #

Ejemplo para continuar

for i in [series]
do
    command 1
    command 2
    if (condition) # Condition to jump over command 3
            continue # skip to the next value in "series"
    fi
    command 3
done

Ejemplo de descanso

for i in [series]
do
    command 4
    if (condition) # Condition to break the loop
    then
            command 5 # Command if the loop needs to be broken
            break
    fi
    command 6 # Command to run if the "condition" is never true 
done

Salto de bucle #

Romper bucle múltiple:

arr=(a b c d e f)
for i in "${arr[@]}";do
    echo "$i"
    for j in "${arr[@]}";do
        echo "$j"
        break 2
    done
done

Producción:

a
a

Romper bucle único:

arr=(a b c d e f)
for i in "${arr[@]}";do
    echo "$i"
    for j in "${arr[@]}";do
        echo "$j"
        break
    done
done

Producción:

a
c
a
d
a
e
a
f

Cambiar declaración con caso #

Con la instrucción case puede comparar valores con una variable.

El argumento pasado a case se expande y trata de hacer coincidir cada patrón.

Si se encuentra una coincidencia, se ejecutan los comandos hasta ;;.

case "$BASH_VERSION" in
 [34]*)
    echo {1..4}
    ;;  
  *)
    seq -s" " 1 4
esac

Los patrones no son expresiones regulares, sino coincidencias de patrones de shell (también conocidos como globos).

For Loop sin un parámetro de lista de palabras #

for arg; do
    echo arg=$arg
done

Un bucle for sin un parámetro de lista de palabras iterará sobre los parámetros posicionales. En otras palabras, el ejemplo anterior es equivalente a este código:

for arg in "$@"; do
    echo arg=$arg
done

En otras palabras, si te encuentras escribiendo for i en "$@"; hacer ...; done, simplemente suelte la parte in y escriba simplemente for i; hacer ...; hecho.

En bucle #

#! /bin/bash

for i in 1 "test" 3; do #Each space separated statement is assigned to i
    echo $i
done

Otros comandos pueden generar declaraciones para repetir. Consulte el ejemplo “Uso de bucle For para iterar sobre números”.

Esto genera:

1
test
3
