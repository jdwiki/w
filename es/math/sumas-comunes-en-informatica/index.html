Suma de Gauss: 1 + 2 + 3 + … + n #

La suma

1 + 2 + 3 + … + n

simplifica a

n(n+1) / 2.

Observe que esta cantidad es Θ(n2).

Este atajo surge con frecuencia en el análisis de algoritmos como el ordenamiento por inserción o el ordenamiento por selección.

Los números de la forma n(n+1)/2 se llaman los números triangulares.

Suma de una serie geométrica: r^0 + r^1 + r^2 + … #

La suma de la serie geométrica.

r0 + r1 + r2 + … + rn-1

En el caso de que r ≠ 1, se simplifica a (rn - 1) / (r - 1). Si r < 1, esta suma está acotada superiormente por 1 / (1 - r).

Si r = 1, esta suma es rn.

Sumas de números de Fibonacci #

Los números de Fibonacci se definen inductivamente como

  • F0 = 0
  • F1 = 1
  • Fn+2 = Fn + Fn+1

La suma de los primeros n+1 números de Fibonacci viene dada por

F0 + F1 + F2 + … + Fn = Fn +2 - 1.

Esta suma surge, entre otros lugares, en el análisis de montones de Fibonacci, donde se usa para proporcionar un límite inferior en el número de nodos en cada árbol en el montón.

Sumas de potencias de dos: 1 + 2 + 4 + 8 + 16 + … #

La suma

20 + 21 + 22 + … + 2n-1

se simplifica a 2n - 1. Esto explica por qué el valor máximo que se puede almacenar en un entero de 32 bits sin signo es 232 - 1.

Sumas de postes de cercas #

Aquí consideramos sumas de la forma

un + segundo + un + segundo + … un

contra

un + segundo + un + segundo + … segundo

Para visualizar estas sumas, imagine una sección de cerca alternando entre postes y barandas. Tres escenarios son posibles.

  1. Imagine una sección de cerca con postes en cada extremo, conectados por rieles. n rieles requieren n+1 postes. Por el contrario, los postes p están unidos por rieles p-1.

    |—|—|—|

  1. Imagine una sección de cerca con un poste en un extremo, pero una baranda abierta en el otro. n rieles requieren n postes.

    |—|—|—

    or

    —|—|—|

  1. Imagine una sección de cerca con rieles abiertos en ambos extremos. Los rieles n requieren postes n-1. Por el contrario, los postes p están unidos por rieles p+1.

    —|—|—

Cálculos como este surgen en situaciones como el diseño de objetos gráficos donde se deben sumar los tamaños de los objetos y también se deben sumar los espacios entre los objetos. En tales situaciones, es importante saber si la intención es o no tener espacios en cada extremo.

El ancho total de tal cerca siempre será:

(ancho de poste) x (número de postes) + (ancho de riel) x (número de rieles)

Pero se debe tener precaución al calcular el número de postes y el número de rieles para evitar el llamado error off-by-one. Los errores de este tipo son comunes.

Sumas de recíprocos: 1/1 + 1/2 + 1/3 + 1/4 + … #

la suma

1/1 + 1/2 + 1/3 + 1/4 + … + 1/n

es igual al enésimo número armónico, denotado Hn. El número armónico n-ésimo obedece a las desigualdades

ln ( n + 1 ) & y ; H<so ≤ ( en n ) +

y por tanto Hn = Θ(log n). Los números armónicos a menudo surgen en el análisis de algoritmos, siendo la ordenación rápida aleatoria un ejemplo particularmente bueno.

Sumas de cuadrados recíprocos: 1/1 + 1/4 + 1/9 + 1/16 + 1/25 + … #

la suma

1/1 + 1/4 + 1/9 + 1/16 + …

hacia el infinito converge a π2 / 6, y por lo tanto cualquier sumatoria de la forma

1/1 + 1/4 + 1/9 + 1/16 + … + 1/n2

es θ(1).
