Instalación o configuración #

WebGL es una tecnología de navegador, por lo que no hay mucho que configurar aparte de tener un navegador. Puede comenzar con WebGL en JSFiddle o Codepen o JSBIn o cualquier otro sitios que le permiten editar HTML, CSS y JavaScript en línea, aunque habrá algunas limitaciones (ver a continuación). También puede alojar archivos de código abierto en páginas de github o servicios similares.

Por otro lado, en algún momento probablemente vas a trabajar localmente. Para hacerlo, se recomienda ejecutar un servidor web simple. Hay muchos para elegir que son fáciles de usar y requieren muy poca configuración.

Usando node.js como servidor

  1. instale node.js
  2. Abra un símbolo del sistema de terminal o nodo y escriba npm install -g http-server (en OSX ponga sudo delante de eso.
  3. escriba http-server para comenzar a servir los archivos en la carpeta actual O http-server path-to-folder para servir una carpeta diferente
  4. Apunte su navegador a http://localhost:8080/name-of-file para ver su página web WebGL

Usando devd como servidor

  1. Descarga [devd] (https://github.com/cortesi/devd)
  2. Abra una terminal y ejecute devd con devd . para servir archivos desde la carpeta actual o devd path-to-folder para servir una carpeta diferente
  3. Apunte su navegador a http://localhost:8000/name-of-file para ver su página web WebGL

Usando Servez como servidor

  1. Descarga Servir
  2. Instálalo, ejecútalo
  3. Elige la carpeta a servir
  4. Elija “Inicio”
  5. Vaya a http://localhost:8080 o seleccione “Iniciar navegador”

Uso de la extensión de Chrome “Servidor web para Chrome”

  1. Instale el [servidor web de Chrome] (https://chrome.google.com/webstore/detail/web-server-for-chrome/ofhbbkphhbklhfoeikjpcbhemlocgigb?hl=en)

  2. Inícielo desde el ícono de Aplicaciones en una página de nueva pestaña.

  1. Configure la carpeta donde están sus archivos y luego haga clic en el enlace http://127.0.0.1:8787

Limitación de WebGL en servicios en línea #

En WebGL es muy común cargar imágenes. En WebGL existen restricciones sobre cómo se pueden usar las imágenes. Específicamente, WebGL no puede usar imágenes de otros dominios sin el permiso del servidor que aloja las imágenes. Los servicios que actualmente dan permiso para usar imágenes incluyen imgur y flickr. Consulte [Carga de imágenes de dominios cruzados] (tbd). De lo contrario, deberá tener las imágenes en el mismo servidor que su página webgl o usar otras soluciones creativas, como generar imágenes con una etiqueta de lienzo.

Hola Mundo #

Como se menciona en la sección comentarios, necesitamos proporcionar dos funciones. Un sombreador de vértices y un sombreador de fragmentos

Comencemos con un sombreador de vértices.

// an attribute will receive data from a buffer
attribute vec4 position;

// all shaders have a main function
void main() {

  // gl_Position is a special variable a vertex shader 
  // is responsible for setting
  gl_Position = position;
}

Si todo estuviera escrito en JavaScript en lugar de GLSL, podrías imaginar que se usaría así

// *** PSUEDO CODE!! ***

var positionBuffer = [
  0, 0, 0, 0,
  0, 0.5, 0, 0,
  0.7, 0, 0, 0,
];
var attributes = {};
var gl_Position;

drawArrays(..., offset, count) {
  for (var i = 0; i < count; ++i) {
     // copy the next 4 values from positionBuffer to the position attribute
     attributes.position = positionBuffer.slice((offset + i) * 4, 4); 
     runVertexShader();
     ...
     doSomethingWith_gl_Position();
}

A continuación, necesitamos un sombreador de fragmentos.

// fragment shaders don't have a default precision so we need
// to pick one. mediump, short for medium precision, is a good default.
precision mediump float;

void main() {
  // gl_FragColor is a special variable a fragment shader
  // is responsible for setting
  gl_FragColor = vec4(1, 0, 0.5, 1); // return redish-purple 
}

Arriba estamos configurando gl_FragColor a 1, 0, 0.5, 1 que es 1 para rojo, 0 para verde, 0.5 para azul, 1 para alfa. Los colores en WebGL van de 0 a 1.

Ahora que hemos escrito las 2 funciones, comencemos con WebGL

Primero necesitamos un elemento de lienzo HTML

 <canvas id="c"></canvas>

Luego en JavaScript podemos buscar eso

 var canvas = document.getElementById("c");

Ahora podemos crear un WebGLRenderingContext

 var gl = canvas.getContext("webgl");
 if (!gl) {
    // no webgl for you!
    ...

Ahora necesitamos compilar esos sombreadores para ponerlos en la GPU, así que primero debemos convertirlos en cadenas. Puede obtener sus cadenas de la forma en que normalmente obtiene cadenas. Concatenando, usando AJAX, colocándolos en etiquetas de secuencias de comandos no escritas en JavaScript o, en este caso, usando literales de plantilla multilínea

var vertexShaderSource = `
// an attribute will receive data from a buffer
attribute vec4 position;

// all shaders have a main function
void main() {

  // gl_Position is a special variable a vertex shader 
  // is responsible for setting
  gl_Position = position;
}
`;

var fragmentShaderSource = `
// fragment shaders don't have a default precision so we need
// to pick one. mediump is a good default
precision mediump float;

void main() {
  // gl_FragColor is a special variable a fragment shader
  // is responsible for setting
  gl_FragColor = vec4(1, 0, 0.5, 1); // return redish-purple 
}
`;

Luego necesitamos una función que cree un sombreador, cargue la fuente y compile el sombreador.

function createShader(gl, type, source) {
  var shader = gl.createShader(type);
  gl.shaderSource(shader, source);  
  gl.compileShader(shader);
  var success = gl.getShaderParameter(shader, gl.COMPILE_STATUS);
  if (success) {
    return shader;
  }

  console.log(gl.getShaderInfoLog(shader));
  gl.deleteShader(shader);
}

Ahora podemos llamar a esa función para crear los 2 shaders.

var vertexShader = createShader(gl, gl.VERTEX_SHADER, vertexShaderSource);
var fragmentShader = createShader(gl, gl.FRAGMENT_SHADER, fragmentShaderSource);

Luego necesitamos vincular esos 2 sombreadores en un programa

function createProgram(gl, vertexShader, fragmentShader) {
  var program = gl.createProgram();
  gl.attachShader(program, vertexShader);
  gl.attachShader(program, fragmentShader);
  gl.linkProgram(program);
  var sucesss = gl.getProgramParameter(program, gl.LINK_STATUS);
  if (success) {
    return program;
  }

  console.log(gl.getProgramInfoLog(program));
  gl.deleteProgram(program);
}

y llámalo

var program = createProgram(gl, vertexShader, fragmentShader);

Ahora que hemos creado un programa GLSL en la GPU, necesitamos proporcionarle datos. La mayor parte de la API de WebGL se trata de configurar el estado para proporcionar datos a nuestros programas GLSL. En este caso, nuestra única entrada a nuestro programa GLSL es posición, que es un atributo. Lo primero que debemos hacer es buscar la ubicación del atributo para el programa que acabamos de crear

var positionAttributeLocation = gl.getAttribLocation(program, "position");

Los atributos obtienen sus datos de los búferes, por lo que necesitamos crear un búfer

var positionBuffer = gl.createBuffer();

WebGL nos permite manipular muchos recursos de WebGL en puntos de vinculación globales. Puede pensar en los puntos de vinculación como variables globales internas dentro de WebGL. Primero establece el punto de enlace a su recurso. Luego, todas las demás funciones se refieren al recurso a través del punto de vinculación. Entonces, enlacemos el búfer de posición.

gl.bindBuffer(gl.ARRAY_BUFFER, positionBuffer);

Ahora podemos poner datos en ese búfer al hacer referencia a ellos a través del punto de enlace

// three 2d points
var positions = [
  0, 0, 
  0, 0.5,
  0.7, 0,
];
gl.bufferData(gl.ARRAY_BUFFER, new Float32Array(positions), gl.STATIC_DRAW);

Están pasando muchas cosas aquí. Lo primero es que tenemos posiciones que es una matriz de JavaScript. WebGL, por otro lado, necesita datos fuertemente tipados, por lo que la línea new Float32Array (posiciones) crea una nueva matriz de números de coma flotante de 32 bits y copia los valores de posiciones. gl.bufferData luego copia esos datos al positionBuffer en la GPU. Está utilizando el búfer de posición porque lo vinculamos al punto de vinculación ARRAY_BUFFER anterior.

El último argumento, gl.STATIC_DRAW es una pista para WebGL sobre cómo usaremos los datos. Puede intentar usar esa información para optimizar ciertas cosas. gl.STATIC_DRAW le dice a WebGL que no es probable que cambiemos mucho estos datos.

Ahora que hemos puesto datos en el búfer, necesitamos decirle al atributo cómo obtener datos de él. En primer lugar, debemos activar el atributo

gl.enableVertexAttribArray(positionAttributeLocation);

Luego necesitamos especificar cómo extraer los datos

var size = 2;          // 2 components per iteration
var type = gl.FLOAT;   // the data is 32bit floats
var normalize = false; // use the data as is
var stride = 0;        // 0 = move size * sizeof(type) each iteration
var offset = 0;        // start at the beginning of the buffer
gl.vertexAttribPointer(
   positionAttributeLocation, size, type, normalize, stride, offset)

Una parte oculta de gl.vertexAttribPointer es que vincula el ARRAY_BUFFER actual al atributo. En otras palabras, ahora que este atributo está vinculado a positionBuffer, somos libres de vincular algo más al punto de vinculación ARRAY_BUFFER.

tenga en cuenta que desde el punto de vista de nuestro sombreador de vértices GLSL, el atributo de posición era un vec4

attribute vec4 position;

vec4 es un valor de 4 flotantes. En JavaScript podría pensar en algo como posición = {x: 0, y: 0, z: 0, w: 0}. Arriba configuramos size = 2. Los atributos predeterminados son 0, 0, 0, 1, por lo que este atributo obtendrá sus primeros 2 valores (x e y) de nuestro búfer. La z y la w serán los valores predeterminados 0 y 1 respectivamente.

Después de todo eso, finalmente podemos pedirle a WebGL que ejecute el programa GLSL.

var primitiveType = gl.TRIANGLES;
var offset = 0;
var count = 3;
gl.drawArrays(primitiveType, offset, count);

Esto ejecutará nuestro vertex shader 3 veces. La primera vez que position.x y position.y en nuestro sombreador de vértices se establecerán en los primeros 2 valores de positionBuffer. La segunda vez position.xy se establecerá en los 2 segundos valores. La última vez se establecerá en los últimos 2 valores.

Debido a que configuramos primitiveType en gl.TRIANGLES, cada vez que nuestro sombreador de vértices se ejecuta 3 veces, WebGL dibujará un triángulo basado en los 3 valores en los que configuramos gl_Position. No importa el tamaño de nuestro lienzo, esos valores están en coordenadas de espacio de clip que van de -1 a 1 en cada dirección.

Debido a que nuestro sombreador de vértices simplemente está copiando nuestros valores de positionBuffer a gl_Position, el triángulo se dibujará en las coordenadas del espacio de recorte

  0, 0, 
  0, 0.5,
  0.7, 0,

La forma en que esos valores se traducen a píxeles depende de la configuración gl.viewport. gl.viewport tiene por defecto el tamaño inicial del lienzo. Dado que no establecimos un tamaño para nuestro lienzo, es el tamaño predeterminado de 300x150. Convirtiendo de espacio de clip a píxeles (a menudo llamado espacio de pantalla en la literatura WebGL y OpenGL) WebGL va a dibujar un triángulo en

 clip space      screen space
   0, 0       ->   150, 75
   0, 0.5     ->   150, 112.5
 0.7, 0       ->   255, 75

WebGL ahora representará ese triángulo. Por cada píxel que esté a punto de dibujar, llamará a nuestro sombreador de fragmentos. Nuestro sombreador de fragmentos simplemente establece gl_FragColor en 1, 0, 0.5, 1. Dado que Canvas es un lienzo de 8 bits por canal, significa que WebGL escribirá los valores [255, 0, 127, 255] en el lienzo.

Hay 3 cosas principales que aún no hemos cubierto de los comentarios. Texturas, variantes y uniformes. Cada uno de ellos requiere su propio tema.
