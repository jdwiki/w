Texturas #

Las unidades de textura son estados globales. Si se implementaran en JavaScript, se verían así

// pseudo code
gl = {
  activeTextureUnit: 0,
  textureUnits: [
    { TEXTURE_2D: ?, TEXTURE_CUBE_MAP: ? },
    { TEXTURE_2D: ?, TEXTURE_CUBE_MAP: ? },
    { TEXTURE_2D: ?, TEXTURE_CUBE_MAP: ? },
    { TEXTURE_2D: ?, TEXTURE_CUBE_MAP: ? },
    { TEXTURE_2D: ?, TEXTURE_CUBE_MAP: ? },
    { TEXTURE_2D: ?, TEXTURE_CUBE_MAP: ? },
    { TEXTURE_2D: ?, TEXTURE_CUBE_MAP: ? },
    { TEXTURE_2D: ?, TEXTURE_CUBE_MAP: ? },
    { TEXTURE_2D: ?, TEXTURE_CUBE_MAP: ? },
  ],
};

Puede elegir qué unidad indexar con gl.activeTexture.

// pseudo code
gl.activeTexture = function(textureUnit) {
  gl.activeTextureUnit = textureUnit - gl.TEXTURE0;
};

Llamar a gl.bindTexture vincula una textura a la unidad de textura activa como esta

// pseudo code
gl.bindTexture = function(target, texture) {
  var textureUnit = gl.textureUnits[gl.activeTextureUnit];
  textureUnit[target] = texture;
}

Cuando tiene un programa de sombreado que usa texturas, debe decirle a ese programa de sombreado a qué unidades de textura vincula las texturas. Por ejemplo, si tienes un shader como este

uniform sampler2D diffuse;
uniform sampler2D normalMap;
uniform samplerCube environmentMap;

...

Para que necesite consultar las ubicaciones uniformes

var diffuseUniformLocation = gl.getUniformLocation(someProgram, "diffuse");
var normalMapUniformLocation = gl.getUniformLocation(someProgram, "normalMap");
var environmmentMapUniformLocation = gl.getUniformLocation(someProgram,
                                                           "environmentMap");

Luego, después de que hayas hecho que tu programa de sombreado sea el programa actual

gl.useProgram(someProgram);

Luego debe decirle al sombreador qué unidades de textura puso/pondrá las texturas. Por ejemplo

var diffuseTextureUnit = 3;
var normalMapTextureUnit = 5;
var environmentMapTextureUnit = 2;

gl.uniform1i(diffuseUniformLocation, diffuseTextureUnit);
gl.uniform1i(normalMapUniformLocation, normalMapTextureUnit);
gl.uniform1i(environmentMapUniformLocation, environmentMapTextureUnit);

Ahora le dijiste al sombreador qué unidades usaste o usarás. Las unidades de textura que decidas usar dependen totalmente de ti.

Para vincular texturas a unidades de textura, haría algo como esto

gl.activeTexture(gl.TEXTURE0 + diffuseTextureUnit);
gl.bindTexture(gl.TEXTURE_2D, diffuseTexture);
gl.activeTexture(gl.TEXTURE0 + normalMapTextureUnit);
gl.bindTexture(gl.TEXTURE_2D, normalMapTexture);
gl.activeTexture(gl.TEXTURE0 + environmentMapTextureUnit);
gl.bindTexture(gl.TEXTURE_CUBE_MAP, environmentMapTexture);

Para ejemplos muy simples de WebGL que solo usan 1 textura, es común nunca llamar a gl.activeTexture ya que por defecto es la unidad de textura #0. También es común no llamar a gl.uniform1i porque los uniformes tienen un valor predeterminado de 0, por lo que el programa de sombreado usará de forma predeterminada la unidad de textura #0 para todas las texturas.

Todas las demás funciones de textura también funcionan con la textura activa y los objetivos de la unidad de textura. Por ejemplo, gl.texImage2D podría verse así

gl.texImage2D = function(target, level, internalFormat, width, height, 
                         border, format, type, data) {
   var textureUnit = gl.textureUnits[gl.activeTextureUnit];
   var texture = textureUnit[target];

   // Now that we've looked up the texture form the activeTextureUnit and
   // the target we can effect a specific texture
   ...
};

Atributos #

Los atributos son de estado global (*). Si se implementaran en JavaScript, se verían así

 // pseudo code
 gl = {
   ARRAY_BUFFER: null,
   vertexArray: {
     attributes: [
       { enable: ?, type: ?, size: ?, normalize: ?, stride: ?, offset: ?, buffer: ? },
       { enable: ?, type: ?, size: ?, normalize: ?, stride: ?, offset: ?, buffer: ? },
       { enable: ?, type: ?, size: ?, normalize: ?, stride: ?, offset: ?, buffer: ? },
       { enable: ?, type: ?, size: ?, normalize: ?, stride: ?, offset: ?, buffer: ? },
       { enable: ?, type: ?, size: ?, normalize: ?, stride: ?, offset: ?, buffer: ? },
       { enable: ?, type: ?, size: ?, normalize: ?, stride: ?, offset: ?, buffer: ? },
       { enable: ?, type: ?, size: ?, normalize: ?, stride: ?, offset: ?, buffer: ? },
       { enable: ?, type: ?, size: ?, normalize: ?, stride: ?, offset: ?, buffer: ? },
     ],
     ELEMENT_ARRAY_BUFFER: null,
   },
 }

Como puede ver arriba, hay atributos 8 y son de estado global.

Cuando llamas a gl.enableVertexAttribArray(ubicación) o gl.disableVertexAttribArray puedes pensarlo así

// pseudo code
gl.enableVertexAttribArray = function(location) {
  gl.vertexArray.attributes[location].enable = true;
};

gl.disableVertexAttribArray = function(location) {
  gl.vertexArray.attributes[location].enable = false;
};

En otras palabras, ubicación se refiere directamente al índice de un atributo.

Del mismo modo gl.vertexAttribPointer se implementaría algo como esto

// pseudo code
gl.vertexAttribPointer = function(location, size, type, normalize, stride, offset) {
  var attrib = gl.vertexArray.attributes[location];
  attrib.size = size;
  attrib.type = type;
  attrib.normalize = normalize;
  attrib.stride = stride ? stride : sizeof(type) * size;
  attrib.offset = offset;
  attrib.buffer = gl.ARRAY_BUFFER;  // !!!! <-----
};

Tenga en cuenta que attrib.buffer se establece en el valor actual de gl.ARRAY_BUFFER. gl.ARRAY_BUFFER se configura llamando a gl.bindBuffer(gl.ARRAY_BUFFER, someBuffer).

Entonces, a continuación tenemos sombreadores de vértices. En vertex shader declaras atributos. Ejemplo

attribute vec4 position;
attribute vec2 texcoord;
attribute vec3 normal;

...

void main() {
  ...
}

Cuando vincula un sombreador de vértices con un sombreador de fragmentos llamando a gl.linkProgram(someProgram) WebGL (el controlador/GPU/navegador) decide por sí mismo qué índice/ubicación usar para cada atributo. No tienes idea de cuáles van a elegir. Depende del navegador/controlador/GPU. Entonces, debe preguntarle ¿qué atributo usó para posición, texcoord y normal?. Haces esto llamando a gl.getAttribLocation

var positionLoc = gl.getAttribLocation(program, "position");
var texcoordLoc = gl.getAttribLocation(program, "texcoord");
var normalLoc = gl.getAttribLocation(program, "normal");

Digamos positionLoc = 5. Eso significa que cuando se ejecuta el sombreador de vértices (cuando llama a gl.drawArrays o gl.drawElements), el sombreador de vértices espera que tenga el atributo de configuración 5 con el tipo, tamaño, desplazamiento, zancada correctos. , búfer, etc.

Tenga en cuenta que ANTES de vincular el programa, puede elegir las ubicaciones llamando a gl.bindAttribLoction(programa, ubicación, nombreDeAtributo). Ejemplo:

// Tell `gl.linkProgram` to assign `position` to use attribute #7
gl.bindAttribLocation(program, 7, "position");

estado de atributo completo #

Lo que falta en la descripción anterior es que cada atributo también tiene un valor predeterminado. Se omite arriba porque es poco común usarlo.

attributes: [
   { enable: ?, type: ?, size: ?, normalize: ?, stride: ?, offset: ?, buffer: ?
     value: [0, 0, 0, 1], },
   { enable: ?, type: ?, size: ?, normalize: ?, stride: ?, offset: ?, buffer: ?
     value: [0, 0, 0, 1], },
   ..

Puede establecer el valor con las diversas funciones gl.vertexAttribXXX. El valor se utiliza cuando habilitar es falso. Cuando enable es verdadero, los datos para el atributo se extraen del buffer asignado.

Objetos de matriz de vértices #

WebGL tiene una extensión, OES_vertex_array_object

En el diagrama anterior, OES_vertex_array_object le permite crear y reemplazar vertexArray. En otras palabras

var vao = ext.createVertexArrayOES();

crea el objeto que ves adjunto a gl.vertexArray en el pseudo código de arriba. Llamar a ext.bindVertexArrayOES(vao) asigna el objeto de matriz de vértices creado como la matriz de vértices actual.

// pseudo code
ext.bindVertexArrayOES = function(vao) {
  gl.vertexArray = vao;
}

Esto le permite configurar todos los atributos y ELEMENT_ARRAY_BUFFER en el VAO actual para que cuando quiera dibujar sea una llamada a ext.bindVertexArrayOES donde, sin la extensión, sería hasta una llamada a ambos gl.bindBuffer gl.vertexAttribPointer (y posiblemente gl.enableVertexAttribArray) por atributo.

Uniformes #

Los uniformes son por programa estado. Cada programa shader tiene su propio estado uniforme y sus propias ubicaciones.
