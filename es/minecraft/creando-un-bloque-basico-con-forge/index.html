Creando un bloque básico con Forge

  • La clase de bloque
  • El modelo de bloques JSON
  • Registro de bloque

En esta página

  • La clase de bloque
  • El modelo de bloques JSON
  • Registro de bloque

Crear un bloque decorativo simple con Forge es una de las primeras tareas que un aspirante a modder tendrá que aprender. La forma de hacerlo ha cambiado en las distintas versiones de Minecraft y probablemente tenga una dificultad “moderada” después de la 1.7.10 debido a la gran cantidad de errores fáciles de cometer.

Si algo sale mal y a tu bloque personalizado (ya sea cuando lo colocas o lo sostienes) le falta una textura (negro/púrpura) o un modelo (cubo predeterminado que es demasiado grande cuando lo sostienes), revisa el registro. Los problemas de este tipo casi siempre aparecerán en el registro, siempre que haya registrado las cosas correctamente.

Lo primero que aparecerá es la línea "Modelo de carga de excepción para variante..." o similar, que le indica qué bloque o elemento no se cargó correctamente. Después de una docena de líneas que comienzan con at... habrá otra línea que comienza con Caused by...

Esta línea “Causado por” es la importante, le dirá qué archivo no se cargó correctamente y puede ser uno de varios errores, como:

  • JSON mal formado (su archivo JSON no es válido y tiene un error tipográfico)
  • Archivo no encontrado (el archivo que busca Minecraft no tiene el nombre correcto o no está en el lugar correcto)
  • Excepción de variante faltante (su Blockstate JSON está incompleto)

¡Incluso puede obtener más de un error fallido por un solo error! Si no sabe qué bloque es el problema, comente cada bloque y elemento que sepa que funciona, reduzca la lista de errores al bloque (o bloques) en cuestión. Tu debes hacer esto varias veces.

Además, una textura mixta se muestra de manera tan diferente como una lista simple de todos los recursos que faltan para un dominio determinado (ID de mod).

La clase de bloque #

Primero necesitamos una clase que represente el bloque.

public class CustomBlock extends Block {

    public CustomBlock () {
        super(Material.ROCK);
        setHardness(1.0f);
        setHarvestLevel("pickaxe", 0);
        setResistance(1.0f);
        setCreativeTab(CreativeTabs.DECORATIONS);
        this.setSoundType(SoundType.STONE);
    }
}

Incluso aquí hay varias modificaciones disponibles: material (que rige algunas propiedades, como poder ser empujado por pistones y si se puede romper con la mano), dureza (cuánto tiempo se tarda en romper), nivel de cosecha (herramienta y herramienta apropiadas). material: en este caso pico de madera), resistencia (vs explosiones), la pestaña que muestra en el menú creativo y qué sonido de paso tiene.

Aquí es donde tendrán que ir los bloques de funciones sofisticadas, pero por ahora estamos haciendo un bloque que se ve bien, así que hemos terminado.

El modelo de bloques JSON #

A continuación, debemos decirle a Minecraft cómo queremos que se vea nuestro bloque.

{
    "parent": "block/cube_all",
    "textures": {
        "all": "example:blocks/decorative"
    }
}

Eso es prácticamente todo lo que se necesita para que funcione una vez que se registra el bloque. Lo único importante es que el nombre de archivo coincida con el nombre de registro utilizado para registrar el bloque y debe estar en minúsculas (1.11+ los nombres de archivo es obligatorio que estén en minúsculas, antes de eso es solo distingue mayúsculas y minúsculas).

Nombre el archivo JSON del modelo my_block.json (que coincida con el nombre de registro que le daremos más adelante) y guárdelo en src\main\resources\assets\example\models\block\ (donde example es el ID de mod especificado en la anotación @Mod de su clase de mod principal).

El modelo de bloque aquí usa un padre de block/cube_all, lo que significa que la única textura suministrada se usará en todas las caras. También hay otros modelos predeterminados, como:

  • bloque/cubo (las seis caras asignadas de forma independiente)
  • block/cube_bottom_top (caras superior e inferior independientes de los lados)
  • bloque/orientable (bloque de revestimiento direccional, por ejemplo, horno)
  • bloque/cruz (flores, hierba alta)
  • bloque/cultivo (trigo, zanahorias)

Tenga en cuenta que cada modelo especifica las texturas que utiliza mediante un ID de nombre (por ejemplo, "all" o "top"). Mire el modelo principal para determinar cuáles son esos nombres si no está seguro. Las texturas especificadas incorrectamente pueden dar lugar a problemas de texturas faltantes sin informes de errores.

También es posible crear un modelo totalmente personalizado o crear un modelo principal personalizado. Pero por ahora, esto será suficiente.

No olvide crear una textura, asígnele el nombre decorative.png (ya que eso es lo que especificó el archivo JSON) y guárdelo en src\main\resources\assets\example\textures\blocks\

Registro de bloque #

El registro de bloques se realiza desde su clase de mod principal, o un método de clase ModBlocks invocado desde la clase de mod principal durante preInit.

Block myBlock = new CustomBlock();
string registryname = "my_block";
block.setRegistryName(registryname);
block.setUnlocalizedName(block.getRegistryName().toString());
GameRegistry.register(block);

¡Hay una razón importante para usar block.setUnlocalizedName(block.getRegistryName().toString()); también! Asegura que los nombres no localizados de su bloque (y elemento) contengan la ID de mod para evitar conflictos de archivos de idioma entre mods.

Oh, ¿también quieres una versión de artículo para que también pueda existir en tu inventario? Eso se crea por separado después de 1.7.10 y se hace así:

ItemBlock ib = new ItemBlock(block);
ib.setRegistryName(registryname);
GameRegistry.register(ib);

Tenga en cuenta que establecemos el nombre de registro de ItemBlock en la misma cadena que nuestro bloque. Así es como Forge y empareja bloques con su contraparte de ItemBlock y viceversa.

¡Pero espera hay mas!

Su bloque puede tener una forma de elemento, ¡pero ese elemento aún no tiene un modelo o textura! Los modelos se registran automáticamente para bloques, pero no para artículos. Esto solo se puede llamar desde el Client Proxy y no cubre bloques con variantes (como lana u hojas).

ModelLoader.setCustomModelResourceLocation(
    ib , 0, new ModelResourceLocation(ib.getRegistryName(),"normal"));

En términos generales, tampoco necesitará un Modelo de artículo JSON, ya que Forge y Vanilla recurrirán al modelo de bloque, sin embargo, este no es siempre el caso. Si encuentra que necesita un JSON de modelo de elemento, simplemente asócielo a su JSON de bloque y guárdelo en src\main\resources\assets\example\models\item\ con el mismo nombre de archivo que el nombre de registro del bloque.

{
    "parent": "example:block/my_block"
}
