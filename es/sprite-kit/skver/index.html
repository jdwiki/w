Parámetros #

Parámetro Detalles
muestraFPS Muestra un recuento de la velocidad de fotogramas actual en fotogramas por segundo en la vista.
muestraCuentaNodos Muestra un recuento del número actual de SKNodes que se muestran en la vista.
espectáculosFísica Muestre una representación visual de SKPhysicsBodys en la vista.
muestraCampos Muestre una imagen que represente los efectos de los campos físicos en la vista.
muestraDrawCount Muestra un recuento del número de pasadas de dibujo necesarias para renderizar la vista.
muestraQuadCount Muestre un recuento de la cantidad de rectángulos necesarios para representar la vista.

Un SKView es una subclase de UIView que se utiliza para presentar animaciones 2D de SpriteKit.

Se puede agregar un SKView a Interface Builder o programáticamente de la misma manera que las UIViews “normales”. Luego, el contenido de SpriteKit se presenta en SKView en un SKScene.

Consulte también Referencia de la clase SKView de la documentación de Apple.

Cree un SKView de pantalla completa usando Interface Builder #

Un caso de uso típico para SpriteKit es donde SKView llena toda la pantalla.

Para hacer esto en Interface Builder de Xcode, primero cree un ViewController normal, luego seleccione la vista contenida y cambie su Clase de UIView a SKView:

Dentro del código para View Controller, en el método viewDidLoad, tome un enlace a este SKView usando self.view:

En rápido:

guard let skView = self.view as? SKView else {
    // Handle error
    return
}

(La declaración de guardia aquí protege contra el error teórico de que la vista no es un SKView).

Luego puede usar esto para realizar otras operaciones, como presentar un SKScene:

En rápido:

skView.presentScene(scene)

Cree un pequeño SKView con otros controles usando Interface Builder #

Un SKView no necesita llenar toda la pantalla y puede compartir espacio con otros controles de IU. Incluso puede mostrar más de un SKView a la vez si lo desea.

Para crear un SKView más pequeño entre otros controles con Interface Builder, primero cree un ViewController normal, luego arrastre y suelte una nueva vista en el controlador de vista:

Puede ser útil establecer el color de esta vista en algo que no sea blanco (aquí se usa el negro) para que se pueda ver más claramente en Interface Builder (este color no se mostrará en la aplicación final). Agregue otros controles (una UIView, dos botones y una etiqueta se muestran aquí como ejemplos) y use las restricciones de la forma habitual para distribuirlas en la pantalla:

Luego seleccione la vista que desea que sea SKView y cambie su clase a SKView:

Luego, usando el editor asistente, controle y arrastre desde este SKView a su código y cree un Outlet:

Utilice este medio para presentar su SKScene.

En rápido:

skView.presentScene(scene)

Visualización de información de depuración #

La velocidad de fotogramas actual (en FPS, fotogramas por segundo) y el número total de SKNodes en la escena (nodeCount, cada sprite es un SKNode pero otros objetos en la escena también son SKNodes) se pueden mostrar en la esquina inferior derecha de la vista .

Estos pueden ser útiles cuando están activados (establecidos en verdadero) para depurar y optimizar su código, pero deben desactivarse (establecidos en falso) antes de enviar la aplicación a la AppStore.

En rápido:

skView.showsFPS = true
skView.showsNodeCount = true
