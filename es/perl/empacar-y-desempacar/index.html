Conversión manual de estructuras C a la sintaxis del paquete #

Si alguna vez trata con las API binarias de C de Perl Code, a través de las funciones syscall, ioctl o fcntl, necesita saber cómo construir memoria de forma compatible con C.

Por ejemplo, si alguna vez estuvo tratando con alguna función que esperaba un timespec, buscaría en /usr/include/time.h y encontraría:

struct timespec
{
    __time_t tv_sec;            /* Seconds.  */
    __syscall_slong_t tv_nsec;  /* Nanoseconds.  */
};

Haces un baile con cpp para encontrar lo que realmente significa:

cpp -E /usr/include/time.h -o /dev/stdout | grep __time_t
# typedef long int __time_t;
cpp -E /usr/include/time.h -o /dev/stdout | grep __syscall_slong_t
# typedef long int __syscall_slong_t

Entonces es un int (firmado)

echo 'void main(){ printf("%#lx\n", sizeof(__syscall_slong_t)); }' | 
  gcc -x c -include stdio.h -include time.h - -o /tmp/a.out && /tmp/a.out
# 0x8

Y se necesitan 8 bytes. Así que 64 bits iniciaron sesión. Y estoy en un procesador de 64 bits. =)

Perldoc paquete dice

q Un valor cuádruple con signo (64 bits).

Entonces, para empacar una especificación de tiempo:

sub packtime {
    my ( $config ) = @_; 
    return pack 'qq', @{$config}{qw( tv_sec tv_nsec )};
}

Y para desempaquetar una especificación de tiempo:

sub unpacktime {
   my ( $buf ) = @_;
   my $out = {};
   @{$out}{qw( tv_sec tv_nsec )} = unpack 'qq', $buf;
   return $out;
}

Ahora puede usar esas funciones en su lugar.

my $timespec = packtime({ tv_sec => 0, tv_nsec => 0 });
syscall(  ..., $timespec ); # some syscall that reads timespec

later ...
syscall( ..., $timespec ); # some syscall that writes timespec
print Dumper( unpacktime( $timespec ));

Construyendo un encabezado IPv4 #

A veces tiene que lidiar con estructuras definidas en términos de tipos de datos C de Perl. Una de esas aplicaciones es la creación de paquetes de red sin procesar, en caso de que desee hacer algo más elegante que lo que ofrece la API de socket normal. Esto es precisamente para lo que está pack() (y unpack(), por supuesto).

La parte obligatoria de un encabezado IP tiene 20 octetos (también conocido como “bytes”) de largo. Como puede ver detrás de este enlace, la dirección IP de origen y de destino constituyen los dos últimos valores de 32 bits en el encabezado. Entre los otros campos hay algunos con 16 bits, algunos con 8 bits y algunos fragmentos más pequeños entre 2 y 13 bits.

Suponiendo que tenemos las siguientes variables para rellenar nuestro encabezado:

my ($dscp, $ecn, $length,
    $id, $flags, $frag_off,
    $ttl, $proto,
    $src_ip,
    $dst_ip);

Tenga en cuenta que faltan tres campos del encabezado:

  • La versión siempre es 4 (después de todo es IPv4)
  • IHL es 5 en nuestro ejemplo ya que no tenemos un campo opciones; la longitud se especifica en unidades de 4 octetos, por lo que 20 octetos dan una longitud de 5.
  • La suma de comprobación se puede dejar en 0. En realidad, tendríamos que calcularla, pero el código para hacer esto no nos concierne aquí.

Podríamos probar y usar operaciones de bits para construir, p. los primeros 32 bits:

my $hdr = 4 << 28 | 5 << 24 | $dscp << 18 | $ecn << 16 | $length;

Sin embargo, este enfoque solo funciona hasta el tamaño de un número entero, que generalmente es de 64 bits, pero puede ser tan bajo como 32. Peor aún, depende del [endianness] de la CPU (https://en.wikipedia.org/wiki/Endianness ) por lo que funcionará en algunas CPU y fallará en otras. Probemos con pack():

my $hdr = pack('H2B8n', '45', sprintf("%06b%02b", $dscp, $ecn), $length);

La plantilla primero especifica H2, una cadena hexadecimal de 2 caracteres, el nybble alto primero. El argumento correspondiente para empaquetar es “45”, versión 4, longitud 5. La siguiente plantilla es B8, una cadena de bits de 8 bits, orden de bits descendente dentro de cada byte. Necesitamos usar cadenas de bits para controlar el diseño hasta fragmentos más pequeños que un nybble (4 bits), por lo que sprintf() se usa para construir una cadena de bits de este tipo a partir de 6 bits de $dscp y 2 de $ecn . El último es n, un valor de 16 bits sin signo en Orden de bytes de red, es decir, siempre big-endian sin importar cuál sea el formato de entero nativo de su CPU, y se completa desde $longitud.

Esos son los primeros 32 bits del encabezado. El resto se puede construir de manera similar:

Plantilla Argumento Observaciones
n $id
B16 sprintf("%03b%013b", $banderas, $frag_off) Igual que DSCP/ECN
C2 $ttl, $proto Dos octetos consecutivos sin signo
n 0 / $suma de comprobación x podría usarse para insertar un byte nulo, pero n nos permite especificar un argumento si elegimos calcular una suma de verificación
N2 $src_ip, $dst_ip use a4a4 para empaquetar el resultado de dos llamadas gethostbyname() como ya está en Orden de bytes de red.

Entonces, la llamada completa para empaquetar un encabezado IPv4 sería:

my $hdr = pack('H2B8n2B16C2nN2',
    '45', sprintf("%06b%02b", $dscp, $ecn), $length,
    $id, sprintf("%03b%013b", $flags, $frag_off),
    $ttl, $proto, 0,
    $src_ip, $dst_ip
);
