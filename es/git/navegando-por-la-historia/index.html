Sintaxis #

  • git log [opciones] [rango de revisión] [[–] ruta…]

Parámetros #

Parámetro           Explicación
-q, –silencio Silencioso, suprime la salida diferencial
–fuente Muestra la fuente de confirmación
–use-mapa de correo Usar archivo de mapa de correo (cambia la información del usuario para el usuario que confirma)
–decorar[=…] Decorar opciones
–L <n,m:archivo> Muestra el registro para un rango específico de líneas en un archivo, contando desde 1. Comienza desde la línea n, va a la línea m. También muestra diferencias.
–mostrar firma Mostrar firmas de confirmaciones firmadas
-i, –regexp-ignore-case Haga coincidir los patrones limitantes de expresiones regulares sin tener en cuenta el uso de mayúsculas y minúsculas

Referencias y documentación actualizada: documentación oficial de git-log

Registro más bonito #

Para ver el registro en una estructura gráfica más bonita, use:

git log --decorate --oneline --graph

salida de muestra:

* e0c1cea (HEAD -> maint, tag: v2.9.3, origin/maint) Git 2.9.3
*   9b601ea Merge branch 'jk/difftool-in-subdir' into maint
|\  
| * 32b8c58 difftool: use Git::* functions instead of passing around state
| * 98f917e difftool: avoid $GIT_DIR and $GIT_WORK_TREE
| * 9ec26e7 difftool: fix argument handling in subdirs
* |   f4fd627 Merge branch 'jk/reset-ident-time-per-commit' into maint
...

Dado que es un comando bastante grande, puede asignar un alias:

git config --global alias.lol "log --decorate --oneline --graph"

Para utilizar la versión de alias:

# history of current branch :
git lol

# combined history of active branch (HEAD), develop and origin/master branches :
git lol HEAD develop origin/master

# combined history of everything in your repo :
git lol --all

Registro Git “Normal” #

git log

mostrará todas sus confirmaciones con el autor y el hash. Esto se mostrará en varias líneas por confirmación. (Si desea mostrar una sola línea por confirmación, mire onelineing). Utilice la tecla q para salir del registro.

De forma predeterminada, sin argumentos, git log enumera las confirmaciones realizadas en ese repositorio en orden cronológico inverso, es decir, el más reciente las confirmaciones aparecen primero. Como puede ver, este comando enumera cada confirmación con su checksum SHA-1, el nombre y correo electrónico del autor, la fecha escrito, y el mensaje de confirmación. - fuente

Si desea limitar su comando al último registro de confirmaciones n, simplemente puede pasar un parámetro. Por ejemplo, si desea enumerar los últimos 2 registros de confirmaciones

git log -2

Registro en línea #

git log --oneline

mostrará todas sus confirmaciones con solo la primera parte del hash y el mensaje de confirmación. Cada compromiso estará en una sola línea, como sugiere el indicador oneline.

La opción oneline imprime cada confirmación en una sola línea, lo cual es útil si estás viendo muchas confirmaciones. - fuente

Si desea limitar su comando al último registro de confirmaciones n, simplemente puede pasar un parámetro. Por ejemplo, si desea enumerar los últimos 2 registros de confirmaciones

git log -2 --oneline

Colorear registros #

git log --graph --pretty=format:'%C(red)%h%Creset -%C(yellow)%d%Creset %s %C(green)(%cr) %C(yellow)<%an>%Creset'

La opción formato le permite especificar su propio formato de salida de registro:

Parámetro Detalles
%C(color_nombre) opción colorea la salida que viene después
%h o %H abrevia el hash de confirmación (use %H para el hash completo)
%Creset restablece el color al color de terminal predeterminado
%d nombres de referencia
%s asunto [confirmar mensaje]
%cr fecha del confirmador, relativa a la fecha actual
%an nombre del autor

Búsqueda de registro #

git log -S"#define SAMPLES"

Busca adición o eliminación de una cadena específica o la cadena coincidencia proporcionada por REGEXP. En este caso, buscamos la adición/eliminación de la cadena #define SAMPLES. Por ejemplo:

+#define SAMPLES  100000

o

-#define SAMPLES  100000

git log -G"#define SAMPLES"

Busca cambios en líneas que contienen una cadena específica o la cadena coincidente proporcionada con REGEXP. Por ejemplo:

-#define SAMPLES  100000
+#define SAMPLES  100000000

Listar todas las contribuciones agrupadas por nombre de autor #

git shortlog resume git log y grupos por autor

Si no se proporcionan parámetros, se mostrará una lista de todas las confirmaciones realizadas por confirmador en orden cronológico.

$ git shortlog
Committer 1 (<number_of_commits>):
    Commit Message 1
    Commit Message 2
    ...
Committer 2 (<number_of_commits>):
    Commit Message 1
    Commit Message 2
    ...

Para ver simplemente el número de confirmaciones y suprimir la descripción de la confirmación, pase la opción de resumen:

-s

--resumen

$ git shortlog -s
<number_of_commits> Committer 1
<number_of_commits> Committer 2

Para ordenar la salida por número de confirmaciones en lugar de alfabéticamente por nombre de confirmador, pase la opción numerada:

-n

--numerado

Para agregar el correo electrónico de un confirmador, agregue la opción de correo electrónico:

-e

--correo electrónico

También se puede proporcionar una opción de formato personalizado si desea mostrar información que no sea el asunto de confirmación:

--formato

Puede ser cualquier cadena aceptada por la opción --format de git log.

Consulte Registros para colorear arriba para obtener más información al respecto.

1: https://www.wikiod.com/es/git/navegando-por-la-historia#Colorear registros

Buscando una cadena de confirmación en el registro de git #

Buscando en el registro de git usando alguna cadena en el registro:

git log [options] --grep "search_string"

Ejemplo:

git log --all --grep "removed file"

Buscará la cadena archivo eliminado en todos los registros en todas las sucursales.

A partir de git 2.4+, la búsqueda se puede invertir usando la opción --invert-grep.

Ejemplo:

git log --grep="add file" --invert-grep

Mostrará todas las confirmaciones que no contengan add file.

Filtrar registros #

git log --after '3 days ago'

Las fechas específicas también funcionan:

git log --after 2016-05-01

Al igual que con otros comandos y banderas que aceptan un parámetro de fecha, el formato de fecha permitido es compatible con la fecha GNU (altamente flexible).

Un alias para --after es --since.

También existen banderas para lo contrario: --before y --until.

También puede filtrar registros por autor. p.ej.

git log --author=author

git log --patch
