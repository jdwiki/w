Sintaxis #

  • rama git [--set-upstream | --pista | --no-track] [-l] [-f] <nombre de la sucursal> [<punto de inicio>]
  • git branch (--set-upstream-to=<upstream> | -u <upstream>) [<branchname>]
  • git branch --unset-upstream [<nombre de la sucursal>]
  • rama git (-m | -M) [<rama antigua>] <rama nueva>
  • git rama (-d | -D) [-r] <nombre de la rama>…​
  • git branch --edit-description [<nombre de la rama>]
  • git branch [--color[=<cuando>] | --sin-color] [-r | -a] [--lista] [-v [--abbrev=<longitud> | --no-abbrev]] [--column[=<opciones>] | --no-column] [(--merged | --no-merged | --contains) [<commit>]] [--sort=<key>] [--points-at <object>] [<pattern >…​]

Parámetros #

Parámetro Detalles
-d, –eliminar Eliminar una sucursal. La rama debe estar completamente fusionada en su rama ascendente, o en HEAD si no se configuró ninguna ascendente con --track o --set-upstream
-D Atajo para --delete --force
-m, –mover Mover/renombrar una rama y el reflog correspondiente
-M Atajo para --move --force
-r, –remotos Enumera o elimina (si se usa con -d) las ramas de seguimiento remoto
-a, –todos Enumere las sucursales de seguimiento remoto y las sucursales locales
–lista Activa el modo de lista. git branch <patrón> intentaría crear una rama, use git branch --list <patrón> para enumerar las ramas coincidentes
–set-aguas arriba Si la rama especificada aún no existe o si se ha dado --force, actúa exactamente como --track. De lo contrario, establece la configuración como lo haría –track al crear la rama, excepto que el lugar al que apunta la rama no cambia

Cada repositorio de git tiene una o más ramas. Una rama es una referencia con nombre al HEAD de una secuencia de confirmaciones.

Un repositorio git tiene una rama actual (indicada por un * en la lista de nombres de ramas impresos por el comando git branch), cada vez que crea una nueva confirmación con el comando git commit, su nueva confirmación se convierte en HEAD de la rama actual, y el HEAD anterior se convierte en el padre de la nueva confirmación.

Una nueva rama tendrá el mismo HEAD que la rama a partir de la cual se creó hasta que algo se confirme en la nueva rama.

Creando y revisando nuevas sucursales #

Para crear una nueva rama, mientras permanece en la rama actual, use:

git branch <name>

Generalmente, el nombre de la sucursal no debe contener espacios y está sujeto a otras especificaciones enumeradas [aquí] (http://stackoverflow.com/questions/3651860/which-characters-are-illegal-within-a-branch-name). Para cambiar a una sucursal existente:

git checkout <name>

Para crear una nueva rama y cambiar a ella:

git checkout -b <name>

Para crear una rama en un punto que no sea la última confirmación de la rama actual (también conocida como HEAD), use cualquiera de estos comandos:

git branch <name> [<start-point>]
git checkout -b <name> [<start-point>]

El <punto de inicio> puede ser cualquier revisión conocida por git (por ejemplo, otro nombre de rama, confirmación SHA o una referencia simbólica como HEAD o un nombre de la etiqueta):

git checkout -b <name> some_other_branch
git checkout -b <name> af295
git checkout -b <name> HEAD~5
git checkout -b <name> v1.0.5

Para crear una rama desde una rama remota (el <nombre_remoto> predeterminado es el origen):

git branch <name> <remote_name>/<branch_name>
git checkout -b <name> <remote_name>/<branch_name>

Si un nombre de sucursal dado solo se encuentra en un control remoto, simplemente puede usar

git checkout -b <branch_name>

que es equivalente a

git checkout -b <branch_name> <remote_name>/<branch_name>

A veces, es posible que deba mover varias de sus confirmaciones recientes a una nueva rama. Esto se puede lograr ramificando y “haciendo retroceder”, así:

git branch <new_name>
git reset --hard HEAD~2 # Go back 2 commits, you will lose uncommitted work.
git checkout <new_name>

Aquí hay una explicación ilustrativa de esta técnica:

 Initial state       After git branch <new_name>    After git reset --hard HEAD~2
                             newBranch                        newBranch
                                 ↓                                ↓
A-B-C-D-E (HEAD)         A-B-C-D-E (HEAD)                 A-B-C-D-E (HEAD)
        ↑                        ↑                            ↑
      master                   master                       master

Listado de sucursales #

Git proporciona múltiples comandos para listar ramas. Todos los comandos usan la función de git branch, que proporcionará una lista de determinadas ramas, según las opciones que se coloquen en la línea de comandos. Si es posible, Git indicará la rama actualmente seleccionada con una estrella al lado.

Meta Comando
Lista de sucursales locales rama git
Lista detallada de sucursales locales git rama -v
Lista de sucursales remotas y locales git branch -a O git branch --all
Lista de sucursales remotas y locales (detallado) git rama -av
Lista de sucursales remotas git rama -r
Lista de sucursales remotas con la última confirmación git rama -rv
Lista de sucursales fusionadas rama git --merged
Lista de sucursales no fusionadas rama git --no-merged
Lista de ramas que contienen commit git branch --contains [<commit>]

Notas:

  • Agregar una v adicional a -v, p. $ git branch -avv o $ git branch -vv también imprimirá el nombre de la rama ascendente.
  • Las ramas que se muestran en color rojo son ramas remotas

Eliminar una sucursal remota #

Para eliminar una rama en el repositorio remoto origin, puede usar Git versión 1.5.0 y posteriores

git push origin :<branchName>

y a partir de la versión 1.7.0 de Git, puede eliminar una rama remota usando

git push origin --delete <branchName>

Para eliminar una rama local de seguimiento remoto:

git branch --delete --remotes <remote>/<branch>
git branch -dr <remote>/<branch> # Shorter

git fetch <remote> --prune # Delete multiple obsolete tracking branches
git fetch <remote> -p      # Shorter

Para eliminar una sucursal localmente. Tenga en cuenta que esto no eliminará la rama si tiene cambios no fusionados:

git branch -d <branchName>

Para eliminar una rama, incluso si tiene cambios no combinados:

git branch -D <branchName>

Cambio rápido a la rama anterior #

Puede cambiar rápidamente a la rama anterior usando

git checkout -

Echa un vistazo a una nueva sucursal rastreando una sucursal remota #

Hay tres formas de crear una nueva “característica” de rama que rastree el “origen/característica” de la rama remota:

  • git checkout --track -b característica origen/característica,
  • git checkout -t origen/característica,
  • función de pago de git: suponiendo que no hay una rama feature local y solo hay un control remoto con la rama feature.

Para configurar upstream para rastrear la sucursal remota, escriba:

  • git branch --set-upstream-to=<remoto>/<branch> <branch>
  • git rama -u <remoto>/<rama> <rama>

dónde:

  • <remote> puede ser: origen, develop o el creado por el usuario,
  • <branch> es la rama del usuario para rastrear en remoto.

Para verificar qué sucursales remotas están rastreando sus sucursales locales:

  • rama git -vv

Eliminar una sucursal localmente #

$ git branch -d dev

Elimina la rama llamada dev si sus cambios se fusionan con otra rama y no se perderán. Si la rama dev contiene cambios que aún no se han fusionado y que se perderían, git branch -d fallará:

$ git branch -d dev
error: The branch 'dev' is not fully merged.
If you are sure you want to delete it, run 'git branch -D dev'.

Según el mensaje de advertencia, puede forzar la eliminación de la rama (y perder cualquier cambio no fusionado en esa rama) usando el indicador -D:

$ git branch -D dev

Crear una rama huérfana (es decir, una rama sin compromiso principal) #

git checkout --orphan new-orphan-branch

La primera confirmación realizada en esta nueva rama no tendrá padres y será la raíz de una nueva historia totalmente desconectada de todas las demás ramas y confirmaciones.

Cambiar el nombre de una rama #

Cambie el nombre de la sucursal que ha revisado:

git branch -m new_branch_name

Cambiar el nombre de otra rama:

git branch -m branch_you_want_to_rename new_branch_name

Empuje la rama al control remoto #

Úselo para enviar confirmaciones realizadas en su sucursal local a un repositorio remoto.

El comando git push toma dos argumentos:

  • Un nombre remoto, por ejemplo, origen
  • Un nombre de sucursal, por ejemplo, maestro

Por ejemplo:

git push  <REMOTENAME> <BRANCHNAME>

Como ejemplo, generalmente ejecuta git push origin master para enviar sus cambios locales a su repositorio en línea.

El uso de -u (abreviatura de --set-upstream) configurará la información de seguimiento durante el envío.

git push -u <REMOTENAME> <BRANCHNAME>

Por defecto, git envía la rama local a una rama remota con el mismo nombre. Por ejemplo, si tiene un local llamado nueva función, si presiona la rama local, también creará una rama remota nueva función. Si desea usar un nombre diferente para la sucursal remota, agregue el nombre remoto después del nombre de la sucursal local, separado por ::

git push <REMOTENAME> <LOCALBRANCHNAME>:<REMOTEBRANCHNAME>

Buscando en sucursales #

Para enumerar ramas locales que contienen una confirmación o etiqueta específica

git branch --contains <commit>

Para enumerar sucursales locales y remotas que contienen una confirmación o etiqueta específica

git branch -a --contains <commit>

Mover HEAD de la rama actual a una confirmación arbitraria #

Una rama es solo un puntero a una confirmación, por lo que puede moverla libremente. Para que la rama se refiera a la confirmación aabbcc, emita el comando

git reset --hard aabbcc

Tenga en cuenta que esto sobrescribirá la confirmación actual de su rama y, por lo tanto, todo su historial. Puede perder algo de trabajo emitiendo este comando. Si ese es el caso, puede usar reflog para recuperar las confirmaciones perdidas. Se puede recomendar ejecutar este comando en una nueva rama en lugar de la actual.

Sin embargo, este comando puede ser particularmente útil cuando se cambia la base o se realizan otras modificaciones importantes del historial.

Sobrescribir un solo archivo en el directorio de trabajo actual con el mismo de otra rama #

El archivo desprotegido sobrescribirá los cambios aún no confirmados que realizó en este archivo.

Este comando verificará el archivo file.example (que se encuentra en el directorio path/to/) y sobrescribirá cualquier cambio que haya realizado en este archivo.

git checkout some-branch path/to/file

some-branch puede ser cualquier cosa tree-ish conocida por git (ver Revision Selection y gitrevisions para más detalles)

Debe agregar -- antes de la ruta si su archivo puede confundirse con un archivo (opcional de lo contrario). No se pueden proporcionar más opciones después de --.

git checkout some-branch -- some-file

El segundo algún-archivo es un archivo en este ejemplo.
