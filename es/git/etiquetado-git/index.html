Etiquetado Git

  • Sintaxis

  • Sintaxis
  • Listado de todas las etiquetas disponibles

Como la mayoría de los sistemas de control de versiones (VCS), Git tiene la capacidad de etiquetar puntos específicos en el historial como importantes. Por lo general, las personas usan esta funcionalidad para marcar puntos de lanzamiento (v1.0, etc.).

Sintaxis #

  • etiqueta git [-a | -s | -u < id clave >] [-f] [-m < mensaje > | -F ] < tagname > [< commit > | < object >]

  • git etiqueta -d <nombre de etiqueta​ >

  • etiqueta git [-n[< número >]] -l [–contiene <confirmación>] [–contiene <confirmación>] [–points-at < object >] [–column[=< options >] | –no-column] [–create-reflog] [–sort=< key >] [–format=< format >] [–[no-]merged [< commit >]] [< pattern >…​]

  • git tag -v [–format=< formato >] < nombre de etiqueta >…​

Listado de todas las etiquetas disponibles #

Usando el comando git tag enumera todas las etiquetas disponibles:

$ git tag
<output follows>
v0.1
v1.3

Nota: las etiquetas se muestran en orden alfabético.

También se puede buscar las etiquetas disponibles:

$ git tag -l "v1.8.5*"
<output follows>
v1.8.5
v1.8.5-rc0
v1.8.5-rc1
v1.8.5-rc2
v1.8.5-rc3
v1.8.5.1
v1.8.5.2
v1.8.5.3
v1.8.5.4
v1.8.5.5

Crear y enviar etiquetas en GIT #

Crear una etiqueta:

  • Para crear una etiqueta en su sucursal actual:

    git tag < tagname >

Esto creará una etiqueta local con el estado actual de la sucursal en la que se encuentra.

  • Para crear una etiqueta con algún compromiso:

    git tag tag-name commit-identifier

Esto creará una etiqueta local con el identificador de confirmación de la rama en la que se encuentra.

Enviar una confirmación en GIT:

  • Empuje una etiqueta individual:

    git push origin tag-name

  • Empuje todas las etiquetas a la vez

    git push origin --tags
