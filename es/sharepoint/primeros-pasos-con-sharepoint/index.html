Instalación de SharePoint 2016 para una Granja de Servidores Únicos #

Introducción #

SharePoint 2016 es la versión 16 de la familia de productos de SharePoint. Se lanzó el 4 de mayo de 2016. Este ejemplo cubre la instalación de SharePoint 2016 mediante la configuración de una sola granja de servidores. Esta configuración cubre los aspectos básicos de la configuración de una granja de servidores de SharePoint sin necesidad de tener varios servidores. Tenga en cuenta que los escenarios cubiertos por una granja de un solo servidor generalmente se limitan a escenarios de desarrollo y producción muy pequeños.

Requisitos #

Antes de instalar SharePoint, se debe configurar el entorno básico. SharePoint almacena documentos, así como metadatos, registros, aplicaciones personalizadas, personalizaciones y mucho más. Asegúrese de tener suficiente espacio en disco y RAM disponible por encima de los requisitos básicos.

  • 4 núcleos en procesadores compatibles de 64 bits
  • 12 - 24 GB de RAM (dependiendo de la implementación de prueba o producción)
  • Disco duro de 80 GB para el sistema
  • Disco duro de 100 GB como segundo disco
  • Servidor con Windows Server 2012 R2 de 64 bits o Technical Preview “Threshold”
  • Servidor SQL 2014 o Servidor SQL 2016
  • .NET Framework 4.5.2 o .NET Framework 4.6
  • Computadora unida al dominio y cuentas de servicio de granja delegadas

Todos los demás requisitos previos se pueden instalar manualmente o mediante el instalador de requisitos previos de SharePoint incluido con la instalación de SharePoint.

Instalación #

  • Ejecute el instalador de requisitos previos; puede solicitar reiniciar el servidor antes de continuar
  • Ejecute Setup.exe desde la instalación de SharePoint
  • Introduzca la clave de licencia
  • Aceptar el acuerdo de licencia
  • Seleccione “Completo” en la pestaña Tipo de servidor
  • La configuración debería completarse con éxito
  • En la página completa, deje marcada la casilla de verificación junto a Ejecutar asistente de configuración del producto y haga clic en Cerrar

Configuración #

Si continúa desde el paso anterior, el asistente de configuración de productos de SharePoint 2016 debería abrirse automáticamente. si el cuadro no aparece o está ejecutando la configuración más adelante, abra el asistente de configuración yendo a Inicio -> Productos de SharePoint 2016 -> Asistente de configuración de productos de SharePoint 2016.

  • Haga clic en siguiente en la página de bienvenida
  • Aparecerá un cuadro de diálogo modal diciendo que algunos servicios pueden reiniciarse durante la configuración; todavía no se ha instalado nada, así que haga clic en sí
  • Agregue el servidor de base de datos para la granja
    • Enter the name of the machine running SQL Server; in this case, it is the local machine
    • Enter the name of the Configuration database or keep the default name SharePoint_Config
    • Enter the username of the domain service user who will be accessing the database (in the form of DOMAIN\user) *Enter the password for the domain user
    • Click next when done
  • Ingrese la contraseña de la granja; esto se usará al unir servidores adicionales a la nueva granja
  • Seleccione la función Granja de servidor único
  • Configure la aplicación web de administración central (desde donde los administradores de la granja administrarán SharePoint), seleccione el número de puerto y seleccione el tipo de federación de autenticación (NTLM o negociación (Kerberose))
  • Revise la configuración en las páginas finales y realice los cambios necesarios
  • Cuando esté listo, ejecute la configuración que puede tardar unos minutos
  • Al finalizar, abrirá el asistente que le permitirá abrir el sitio de administración central
  • En caso de falla, puede investigar los registros en la carpeta %COMMONPROGRAMFILES%\Microsoft Shared\Web Server Extensions\16\LOG

Configuración de la granja #

Una vez que la aplicación web central, la base de datos de configuración y el administrador central estén configurados, estará listo para configurar la granja para uso de usuarios o desarrollo. Puede marcar la ubicación del sitio de administración central o acceder a él a través de un acceso directo en la misma ubicación que el asistente de configuración del producto.

  • Si va a iniciar la configuración más tarde, haga clic en Inicio rápido -> Asistentes de configuración -> Asistente de configuración de la granja
  • Si está iniciando el asistente desde el paso de instalación, haga clic en Iniciar el asistente
  • Elija si desea ser parte del programa de mejora del cliente haciendo clic en Sí o No
  • En la página de configuración de la granja, seleccione la cuenta de dominio que ejecutará los servicios en segundo plano en la granja
    • While this account may be the same as the database account, they may also be different for separation of roles and privileges
    • Enter the account as DOMAIN\user
  • Valide los servicios que desea disponibles en la finca en la página de Servicios
  • Cree la primera colección de sitios en la granja (este paso se puede omitir y realizar más adelante)
    • Enter the site collection’s title, description, web address (usually the first site is at the server root), and the template
    • Most things can be changed (title, description) can be changed easily, but others like the web URL may take much more work to change; the template can also not be easily rolled back, but SharePoint allows a large amount of customizations that allows you to take any base template and convert the style and layout of the site
  • Cuando haya terminado con la configuración, haga clic en finalizar

La granja y la primera colección de sitios ahora están configuradas para su uso.

Cree un elemento web con SharePoint Framework #

dev.office.com/sharepoint es un excelente lugar para comenzar con SharePoint Framework.

SharePoint Framework es un enfoque moderno del lado del cliente para el desarrollo de SharePoint destinado inicialmente a SharePoint Online en Office 365. Los elementos web creados con SharePoint Framework son un nuevo tipo de elemento web y pueden estar disponibles para agregar en páginas de SharePoint existentes y nuevas páginas de SharePoint.

Hay un gran ejemplo de hola mundo para este proceso alojado en Cree su primer elemento web del lado del cliente de SharePoint (Hola mundo parte 1). Todos los ejemplos en dev.office.com están disponibles para contribuciones de la comunidad a través de github.

Los pasos básicos de Hello World en SharePoint Framework son:

  1. Genere el esqueleto del proyecto con el Yeoman SharePoint Generator.

    yo @microsoft/SharePoint

  2. Edite el código generado en el editor de su elección. La compatibilidad con Visual Studio Code es sólida en todas las plataformas.

  3. Obtenga una vista previa del elemento web con Gulp y el banco de trabajo local de SharePoint.

    gulp serve

  4. Vista previa en su entorno de SharePoint Online

Vaya a la siguiente URL: ‘https://your-sharepoint-site/_layouts/workbench.aspx’

Registros y registro de ULS de SharePoint #

El Servicio de registro unificado (ULS) de SharePoint brinda soporte y capacidades de depuración tanto para operaciones como para desarrolladores. Comprender cómo leer los registros es un primer paso importante para resolver problemas.

Herramientas #

Microsoft proporciona el Visor de ULS para ayudar a leer los registros antiguos y los registros que se están escribiendo actualmente mientras se ejecuta la granja. También puede filtrar y aplicar formato a los registros para ayudar a reducir un problema.

Identificador de correlación #

Para aislar un problema, es útil mirar solo una identificación de correlación particular. Cada id de correlación está asociado con una solicitud o una acción de extremo a extremo del sistema (como un intermediario de tiempo). Si hay un problema con la representación de una página web, ubicar la solicitud en los registros de ULS y aislarla en la identificación de correlación específica elimina todo el ruido de los otros registros, lo que ayuda a identificar el problema.

Adición de SPMonitoredScope a Mi código #

Una forma de calcular cómo agregar registros y monitorear el rendimiento es agregar SPMonitoredScope a su código.

using (new SPMonitoredScope("Feature Monitor"))
{
    // My code here 
}

Este código registrará el comienzo y el final de sus solicitudes, así como algunos datos de rendimiento. La creación de su propio monitor personalizado que implemente ISPScopedPerformanceMonitor le permite establecer el nivel de rastreo o el tiempo máximo de ejecución para un conjunto de código.
