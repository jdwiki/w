Sintaxis #

  • sed -I extension - FreeBSD sed (contador de línea continuo)
  • sed -I[extension] - NetBSD e Illumos sed (contador de línea continuo)
  • sed -i extension - FreeBSD sed
  • sed -i[extension] - NetBSD, OpenBSD, Illumos, BusyBox y GNU sed
  • sed –in-place[=extension] - Illumos, BusyBox y GNU sed

Parámetros #

Parámetro Detalles
extensión Guarde un archivo de copia de seguridad con la extensión especificada, o no guarde ningún archivo de copia de seguridad cuando extensión sea una cadena de longitud cero.

La edición en el lugar es una extensión común pero no estándar presente en la mayoría de los sistemas recientes.

De un [manual sed de BSD][obsed]

(una sección como esta aparece en todos los manuales actuales de BSD sed, y los de sus derivados)

No se recomienda dar una extensión de longitud cero cuando se editan archivos en el lugar, ya que corre el riesgo de corrupción o contenido parcial en situaciones donde el espacio en disco es agotado, etc

No olvides el poderoso ed #

Definitivamente hay un uso para sed y para funciones de edición en el lugar de sed, pero cuando se amplía el estándar UNIX, siempre debemos preguntarnos por qué el antiguo estándar UNIX no incluía esa función. Aunque UNIX no es perfecto, la ortogonalidad y la integridad de las herramientas se han desarrollado para estar bastante cerca de la perfección, al menos para propósitos que eran visibles alrededor de 1970: La edición de texto y la edición de texto automatizada seguramente eran visibles en esa época.

En realidad, la idea de sed no es editar un archivo en su lugar, sino editar un flujo. Es por eso que el nombre sed es una forma abreviada de stream editor. Quite la s y obtendrá la herramienta que en realidad fue diseñada para la edición de archivos: ed:

printf 'g/what to replace/s//with what to replace/g\nw\nq\n' | ed file

o cat file_edit_commands | archivo ed.

Uso portátil #

La edición en el lugar, aunque común, es una característica no estándar. Una alternativa viable sería usar un archivo intermedio para almacenar el original o la salida.

sed 'sed commands' > file.out && mv file.out file
# or
mv file file.orig && sed 'sed commands' file.orig > file

Para usar la opción -i con la sintaxis tanto de GNU como de FreeBSD, se debe especificar una extensión y agregarla a la opción -i. Ambos aceptarán lo siguiente y producirán dos archivos, la versión original en file.orig y la versión editada en file:

sed -i.orig 'sed commands' file

Vea un ejemplo básico dado un archivo archivo:

$ cat file
one
two
three
$ sed -i.orig 's/one/XX/' file
$ cat file                       # the original file has changed its content
XX
two
three
$ cat file.orig                  # the original content is now in file.orig
one
two
three

Un ejemplo más complejo, reemplazando cada línea con el número de línea:

$ printf 'one\ntwo\n' | tee file1 | tr a-z A-Z > file2
$ sed -ni.orig = file1 file2
$ cat file1.orig file2.orig
one
two
ONE
TWO
$ cat file1 file2
1
2
1
2

Por qué se requiere un archivo de copia de seguridad #

Para usar la edición en el lugar sin un archivo de respaldo, -i debe recibir un argumento de longitud cero y FreeBSD sed requiere un argumento para -i, ya sea agregado o separado, mientras que la extensión de argumento opcional GNU requiere que el argumento se añada a -i. Ambos admiten agregar el argumento a -i, pero sin que sea necesario, -i'' command es indistinguible de -i extension, por lo que un argumento de longitud cero no se puede agregar a -i.

Reemplazar cadenas en un archivo en el lugar #

sed -i s/"what to replace"/"with what to replace"/g $file

Usamos -i para seleccionar la edición en el lugar en el archivo $file. En algunos sistemas, se requiere agregar el sufijo después del indicador -i, que se usará para crear una copia de seguridad del archivo original. Puede agregar una cadena vacía como -i '' para omitir la creación de la copia de seguridad. Mire Comentarios en este tema sobre la opción -i.

El terminador g significa hacer una búsqueda/reemplazo global en cada línea.

$ cat example 
one
two
three
total
$ sed -i s/"t"/"g"/g example 
$ cat example 
one
gwo
ghree
gogal

La edición en el lugar sin especificar un archivo de respaldo anula los permisos de solo lectura #

sed -i -e cmd file modificará file incluso si sus permisos están establecidos en solo lectura.

Este comando se comporta de manera similar a

sed -e cmd archivo > tmp; mv -f archivo tmp

más bien que

sed -e cmd archivo > tmp; gato tmp > archivo; rm tmp

El siguiente ejemplo usa gnu sed:

$ echo 'Extremely important data' > input
$ chmod 400 input  # Protect that data by removing write access
$ echo 'data destroyed' > input
-bash: input: Permission denied
$ cat input  
Extremely important data (#phew! Data is intact)
$ sed -i s/important/destroyed/ input
$ cat input
Extremely destroyed data (#see, data changed)

Esto se puede mitigar creando una copia de seguridad especificando un SUFIJO con la opción i:

$ sed -i.bak s/important/destroyed/ input
$ cat input
Extremely destroyed data
$ cat input.bak
Extremely important data
