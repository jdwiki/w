Sintaxis #

  • -a - (BSD sed) Crear/truncar todos los archivos escritos antes de procesarlos
  • -E | -r - Usar expresiones regulares extendidas
  • -i | -I - Consulte el tema sobre [Edición en el lugar] [en el lugar]
  • -l - (BSD sed) Usar salida con búfer de línea
  • -l longitud - (GNU sed) Especifique la longitud para el ajuste de línea de comando l
  • -s - (GNU sed) Trata los archivos como flujos separados
  • -u - No almacenar en búfer la salida
  • -z - (GNU sed) Use el carácter NUL para separar registros
  • –tranquilo | –silent - (GNU sed) Sinónimos de -n
  • –expresión=comando - (GNU sed) Sinónimo de -e
  • –file=archivo_de_comandos - (GNU sed) Sinónimo de -f
  • –follow-symlinks - (GNU sed) Seguir enlaces simbólicos
  • –in-place[=extensión] - (GNU sed) Sinónimo de -i
  • –line-length=longitud - (GNU sed) Sinónimo de -l
  • –separate - (GNU sed) Sinónimo de -s
  • –unbuffered - (GNU sed) Sinónimo de -u
  • –null-data - (GNU sed) Sinónimo de -z
  • –help - (GNU sed) Uso de impresión
  • –version - (GNU sed) Versión de impresión

La opción -E se estandarizará en la próxima versión principal, consulte [el tema relevante] (http://austingroupbugs.net/view.php?id=528).

Retraso en la creación/truncamiento de archivos #

Los archivos escritos con el comando w se crean/truncan antes de ejecutar cualquier comando.

$ sed 'w created-file' < /dev/null && ls created-file && rm created-file
created-file

De la norma:

Cada archivo w se creará antes de que comience el procesamiento. Las implementaciones admitirán al menos diez argumentos de archivo w en el script; el número real (mayor o igual a 10) que admite la implementación no se especifica. El uso del parámetro wfile hará que ese archivo se cree inicialmente, si no existe, o reemplazará el contenido de un archivo existente.

BSD sed proporciona la opción -a para retrasar la creación/truncamiento de archivos hasta que se escriban con el comando w.

$ if sed -a 'w created-file' < /dev/null && [ ! -e created-file ]; then
>       echo The file was not created
> fi
The file was not created

Ajuste de línea ’l' #

La longitud del ajuste de línea cuando se usa el comando l está definida por la implementación.

De la norma:

Las líneas largas se doblarán, con el punto de doblado indicado escribiendo una seguida de una <nueva línea>; la longitud a la que se produce el plegado no se especifica, pero debe ser adecuada para el dispositivo de salida.

GNU sed proporciona la opción -l para especificar la longitud a la que dividir líneas largas cuando se imprime con el comando l, por defecto a setenta caracteres.

$ yes | head -c100 | tr '\n' ' ' | sed -n l | head -n1 | wc -c
      71
$ yes | head -c100 | tr '\n' ' ' | sed -nl50 l | head -n1 | wc -c
      51

BSD sed divide líneas largas en el número proporcionado por la variable de entorno COLUMNAS, si no se proporciona COLUMNAS entonces se divide en el ancho del terminal, y si no se proporciona COULMNS y la salida no es una terminal entonces por defecto tiene sesenta caracteres.

$ yes | head -c100 | tr '\n' ' ' | sed -n l | head -n1 | wc -c
      61
$ yes | head -c100 | tr '\n' ' ' | COLUMNS=50 sed -n l | head -n1 | wc -c
      51
$ yes | head -c100 | tr '\n' ' ' | sed -n l | head -n1
