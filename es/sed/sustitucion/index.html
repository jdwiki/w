Usando diferentes delimitadores #

Especificación base de grupo abierto POSIX/IEEE dice:

[2dir] s/BRE/reemplazo/banderas

Sustituya la cadena de reemplazo por instancias del BRE en el espacio patrón. Cualquier carácter que no sea barra invertida o nueva línea puede utilizarse en lugar de una barra oblicua para delimitar el BRE y el reemplazo. Dentro del BRE y el reemplazo, el propio delimitador BRE puede ser se usa como un carácter literal si está precedido por una barra invertida.

Hay casos en los que el delimitador / para el reemplazo de sed está en el BRE o reemplazo, lo que desencadena errores como:

$ echo "2/3/4" | sed "s/2/3/X/"
sed: -e expression #1, char 7: unknown option to `s'

Para ello, podemos utilizar diferentes delimitadores como # o _ o incluso un espacio:

$ echo "2/3/4" | sed "s#2/3#X#"
X/4
$ echo "2/3/4" | sed "s_2/3_X_"
X/4
$ echo "2/3/4" | sed "s 2/3 X "
X/4

Sustitución usando variables de shell #

Las variables dentro de comillas simples ' no se expanden con shells compatibles con POSIX, por lo que usar una variable de shell en una sustitución sed requiere el uso de comillas dobles " en lugar de comillas simples ':

$ var="he"
$ echo "hello" | sed "s/$var/XX/"
XXllo

$ var="he"
$ echo "hello" | sed 's/$var/XX/'
hello

Tenga cuidado con la inyección de comandos al evaluar variables:

$ var='./&/;x;w/etc/passwd
> x;s/he'
$ echo "hello" | sed "s/$var/XX/"
sed: /etc/passwd: Permission denied

Si lo anterior se ejecutó como root, la salida no se habría distinguido del primer ejemplo, y el contenido de /etc/passwd se destruiría.

Referencia inversa #

Usando corchetes de escape, puede definir un grupo de captura en un patrón al que se puede hacer una referencia inversa en la cadena de sustitución con \1:

$ echo Hello world! | sed 's/\(Hello\) world!/\1 sed/'
Hello sed

Con varios grupos:

$ echo one two three | sed 's/\(one\) \(two\) \(three\)/\3 \2 \1/'
three two one

Cuando se usan expresiones regulares extendidas (consulte Opciones adicionales), los paréntesis realizan la agrupación de forma predeterminada y no es necesario escaparlos:

$ echo one two three | sed -E 's/(one) (two) (three)/\3 \2 \1/'
three two one

Las palabras que consisten en letras, dígitos y guiones bajos se pueden unir usando la expresión [[:alnum:]_]\{1,\}:

$ echo Hello 123 reg_exp | sed 's/\([[:alnum:]_]\{1,\}\) \([[:alnum:]_]\{1,\}\) \([[:alnum:]_]\{1,\}\)/\3 \2 \1/'
reg_exp 123 Hello

La secuencia \w es equivalente a [[:alnum:]_]

$ echo Hello 123 reg_exp | sed 's/\(\w\w*\) \(\w\w*\) \(\w\w*\)/\3 \2 \1/'
reg_exp 123 Hello

Indicadores de patrón - reemplazo de ocurrencia #

Si queremos reemplazar solo la primera aparición en una línea, usamos sed como de costumbre:

$ cat example
aaaaabbbbb
aaaaaccccc
aaaaaddddd
$ sed 's/a/x/' example
xaaaabbbbb
xaaaaccccc
xaaaaddddd

Pero, ¿y si queremos reemplazar todas las ocurrencias?

Simplemente agregamos la bandera del patrón g al final:

$ sed 's/a/x/g' example
xxxxxbbbbb
xxxxxccccc
xxxxxddddd

Y si queremos reemplazar una ocurrencia específica, podemos especificar cuál:

$ sed 's/a/x/3' example
aaxaabbbbb
aaxaaccccc
aaxaaddddd

/3 es la tercera aparición.


Desde info sed, consulte GNU sed manual para ver la versión en línea

el estándar POSIX no especifica qué debe pasar cuando mezclas los modificadores g y NÚMERO, y actualmente no existe un consenso general sobre el significado a través de implementaciones sed. Para GNU sed, el la interacción se define como: ignorar las coincidencias antes del NÚMERO, y luego haga coincidir y reemplace todas las coincidencias desde el NÚMERO en adelante.

$ sed 's/b/y/2g' example
aaaaabyyyy
aaaaaccccc
aaaaaddddd

$ sed 's/c/z/g3' example
aaaaabbbbb
aaaaacczzz
aaaaaddddd
