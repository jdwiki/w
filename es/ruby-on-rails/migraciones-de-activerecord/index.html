Parámetros #

Tipo de columna Descripción
:clave_principal Clave principal
:cadena Tipo de datos de cadena más corto. Permite la opción límite para el número máximo de caracteres.
:texto Mayor cantidad de texto. Permite la opción límite para el número máximo de bytes.
:entero Entero. Permite la opción límite para el número máximo de bytes.
:bigint Entero mayor
:flotante Flotador
:decimal Número decimal con precisión variable. Permite opciones de precisión y escala.
:numérico Permite opciones de precisión y escala.
:fechahora Objeto DateTime para fechas/horas.
:tiempo Objeto de tiempo para tiempos.
:fecha Objeto de fecha para fechas.
:binario Datos binarios. Permite la opción límite para el número máximo de bytes.
:booleano Booleano

  • La mayoría de los archivos de migración se encuentran en el directorio db/migrate/. Se identifican con una marca de tiempo UTC al comienzo de su nombre de archivo: YYYYMMDDHHMMSS_create_products.rb.

  • El comando rails generate se puede acortar a rails g.

  • Si no se pasa un :tipo a un campo, por defecto es una cadena.

Agregar varias columnas a una tabla #

Para agregar múltiples columnas a una tabla, separe los pares field:type con espacios cuando use el comando rails generatemigration.

La sintaxis general es:

rails generate migration NAME [field[:type][:index] field[:type][:index]] [options]

Por ejemplo, lo siguiente agregará los campos nombre, salario y correo electrónico a la tabla usuarios:

rails generate migration AddDetailsToUsers name:string salary:decimal email:string

Lo cual genera la siguiente migración:

class AddDetailsToUsers < ActiveRecord::Migration[5.0]
  def change
    add_column :users, :name, :string
    add_column :users, :salary, :decimal
    add_column :users, :email, :string
  end
end

Agregar una columna de referencia a una tabla #

Para agregar una referencia a un equipo a la tabla usuarios, ejecute este comando:

$ rails generate migration AddTeamRefToUsers team:references

Esto genera la siguiente migración:

class AddTeamRefToUsers < ActiveRecord::Migration[5.0]
  def change
    add_reference :users, :team, foreign_key: true
  end
end

Esa migración creará una columna team_id en la tabla users.

Si desea agregar un índice y una clave_extranjera apropiados en la columna agregada, cambie el comando a los rieles generan migración AddTeamRefToUsers team:references:index. Esto generará la siguiente migración:

class AddTeamRefToUsers < ActiveRecord::Migration
  def change
    add_reference :users, :team, index: true
    add_foreign_key :users, :teams
  end
end

Si desea nombrar su columna de referencia de forma distinta a la que Rails genera automáticamente, agregue lo siguiente a su migración: (por ejemplo: es posible que desee llamar al ‘Usuario’ que creó la ‘Publicación’ como ‘Autor’ en la tabla ‘Publicar’ )

class AddAuthorRefToPosts < ActiveRecord::Migration
  def change
    add_reference :posts, :author, references: :users, index: true
  end
end

Migraciones revertidas #

Para “revertir” la última migración, ya sea revirtiendo el método “cambiar” o ejecutando el método “abajo”. Ejecutar comando:

rake db:rollback

rails db:rollback

Revertir las últimas 3 migraciones #

rake db:rollback STEP=3

rails db:rollback STEP=3

STEP proporciona el número de migraciones para revertir.

Revertir todas las migraciones #

rake db:rollback VERSION=0

rails db:rollback VERSION=0

Agregar una nueva columna con un índice #

Para agregar una nueva columna indexada correo electrónico a la tabla usuarios, ejecute el comando:

rails generate migration AddEmailToUsers email:string:index

Esto generará la siguiente migración:

class AddEmailToUsers < ActiveRecord::Migration[5.0]
  def change
    add_column :users, :email, :string
    add_index :users, :email
  end
end

Ejecutar migración específica #

Para ejecutar una migración ascendente o descendente específica, use db:migrate:up o db:migrate:down.

Hasta una migración específica:

rake db:migrate:up VERSION=20090408054555

rails db:migrate:up VERSION=20090408054555

Abajo una migración específica:

rake db:migrate:down VERSION=20090408054555

rails db:migrate:down VERSION=20090408054555

El número de versión en los comandos anteriores es el prefijo numérico en el nombre de archivo de la migración. Por ejemplo, para migrar a la migración 20160515085959_add_name_to_users.rb, usaría 20160515085959 como número de versión.

Rehacer migraciones #

Puede revertir y luego migrar nuevamente usando el comando redo. Esto es básicamente un atajo que combina las tareas de “retroceder” y “migrar”.

Ejecutar comando:

rake db:migrate:redo

rails db:migrate:redo

Puede utilizar el parámetro STEP para retroceder más de una versión.

Por ejemplo, para retroceder 3 migraciones:

rake db:migrate:redo STEP=3

rails db:migrate:redo STEP=3

Agregar una nueva columna a una tabla #

Para agregar una nueva columna nombre a la tabla usuarios, ejecute el comando:

rails generate migration AddNameToUsers name

Esto genera la siguiente migración:

class AddNameToUsers < ActiveRecord::Migration[5.0]
  def change
    add_column :users, :name, :string
  end
end

Cuando el nombre de la migración tiene el formato AddXXXToTABLE_NAME seguido de una lista de columnas con tipos de datos, la migración generada contendrá las instrucciones add_column apropiadas.

Eliminar una columna existente de una tabla #

Para eliminar la columna nombre existente de la tabla usuarios, ejecute el comando:

rails generate migration RemoveNameFromUsers name:string

Esto generará la siguiente migración:

class RemoveNameFromUsers < ActiveRecord::Migration[5.0]
  def change
    remove_column :users, :name, :string
  end
end

Cuando el nombre de la migración tiene el formato RemoveXXXFromYYY seguido de una lista de columnas con tipos de datos, la migración generada contendrá las instrucciones remove_column apropiadas.

Si bien no es necesario especificar el tipo de datos (por ejemplo, :string) como parámetro para remove_column, es muy recomendable. Si el tipo de datos no se especifica, la migración no será reversible.

Ejecutar migraciones en diferentes entornos #

Para ejecutar migraciones en el entorno test, ejecute este comando de shell:

rake db:migrate RAILS_ENV=test

A partir de Rails 5.0, puede usar rails en lugar de rake:

rails db:migrate RAILS_ENV=test

Crear una nueva tabla #

Para crear una nueva tabla usuarios con las columnas nombre y salario, ejecute el comando:

rails generate migration CreateUsers name:string salary:decimal

Esto generará la siguiente migración:

class CreateUsers < ActiveRecord::Migration[5.0]
  def change
    create_table :users do |t|
      t.string :name
      t.decimal :salary  
    end
  end
end

Cuando el nombre de la migración tiene el formato CreateXXX seguido de una lista de columnas con tipos de datos, se generará una migración que crea la tabla XXX con las columnas enumeradas.

Ejecutando migraciones #

Ejecutar comando:

rake db:migrate

rails db:migrate

Especificar la versión de destino ejecutará las migraciones requeridas (hacia arriba, hacia abajo, cambio) hasta que alcance la versión especificada. Aquí, número de versión es el prefijo numérico en el nombre de archivo de la migración.

rake db:migrate VERSION=20080906120000

rails db:migrate VERSION=20080906120000

Cambiar el tipo de una columna existente #

Para modificar una columna existente en Rails con una migración, ejecute el siguiente comando:

rails g migration change_column_in_table

Esto creará un nuevo archivo de migración en el directorio db/migration (si aún no existe), que contendrá el archivo con el prefijo de marca de tiempo y el nombre del archivo de migración que contiene el siguiente contenido:

def change
  change_column(:table_name, :column_name, :new_type)
end

[Guía de rieles: cambio de columnas] (http://guides.rubyonrails.org/active_record_migrations.html#cambio-de-columnas)

Un método más largo pero más seguro #

El código anterior evita que el usuario revierta la migración. Puede evitar este problema dividiendo el método ‘cambiar’ en métodos separados ‘arriba’ y ‘abajo’:

def up
  change_column :my_table, :my_column, :new_type
end

def down
  change_column :my_table, :my_column, :old_type
end

Agregar columna con valor predeterminado #

El siguiente ejemplo agrega una columna ‘admin’ a la tabla ‘usuarios’ y le da a esa columna el valor predeterminado ‘falso’.

class AddDetailsToUsers < ActiveRecord::Migration[5.0]
  def change
    add_column :users, :admin, :boolean, default: false
  end
end

Las migraciones con valores predeterminados pueden llevar mucho tiempo en tablas grandes con, por ejemplo, PostgreSQL. Esto se debe a que cada fila deberá actualizarse con el valor predeterminado para la columna recién agregada. Para evitar esto (y reducir el tiempo de inactividad durante las implementaciones), puede dividir su migración en tres pasos:

  1. Agregue una migración add_column similar a la anterior, pero no establezca ningún valor predeterminado
  2. Implemente y actualice la columna en una tarea de rake o en la consola mientras se ejecuta su aplicación. Asegúrese de que su aplicación ya escriba datos en esa columna para filas nuevas/actualizadas.
  3. Agregue otra migración change_column, que luego cambia el valor predeterminado de esa columna al valor predeterminado deseado

Crear una tabla de unión #

Para crear una tabla de unión entre estudiantes y cursos, ejecute el comando:

$ rails g migration CreateJoinTableStudentCourse student course

Esto generará la siguiente migración:

class CreateJoinTableStudentCourse < ActiveRecord::Migration[5.0]
  def change
    create_join_table :students, :courses do |t|
      # t.index [:student_id, :course_id]
      # t.index [:course_id, :student_id]
    end
  end
end

Crear una columna hstore #

Las columnas Hstore pueden ser útiles para almacenar configuraciones. Están disponibles en las bases de datos de PostgreSQL después de habilitar la extensión.

class CreatePages < ActiveRecord::Migration[5.0]
  def change
    create_table :pages do |t|
      enable_extension 'hstore' unless extension_enabled?('hstore')
      t.hstore :settings
      t.timestamps
    end
  end
end

Agregar una referencia propia #

Una auto referencia puede ser útil para construir un árbol jerárquico. Esto se puede lograr con add_reference en una migración.

class AddParentPages < ActiveRecord::Migration[5.0]
  def change
    add_reference :pages, :pages
  end
end

La columna de clave externa será pages_id. Si desea decidir sobre el nombre de la columna de clave externa, primero debe crear la columna y agregar la referencia después.

class AddParentPages < ActiveRecord::Migration[5.0]
  def change
    add_column :pages, :parent_id, :integer, null: true, index: true
    add_foreign_key :pages, :pages, column: :parent_id
  end
end

Crear una columna de matriz #

Una columna arreglo es compatible con PostgreSQL. Rails convertirá automáticamente una matriz de PostgreSQL en una matriz de Ruby y viceversa.

Cree una tabla con una columna matriz:

create_table :products do |t|
  t.string :name
  t.text :colors, array: true, default: []
end

Agrega una columna arreglo a una tabla existente:

add_column :products, :colors, array: true, default: []

Agregue un índice para una columna matriz:

add_index :products, :colors, using: 'gin'

Cambiadores #

Si ha creado una tabla con algún esquema incorrecto, entonces la forma más fácil de cambiar las columnas y sus propiedades es cambiar_tabla. Revise el siguiente ejemplo:

change_table :orders do |t|
  t.remove :ordered_at # removes column ordered_at
  t.string :skew_number # adds a new column 
  t.index  :skew_number #creates an index
  t.rename :location, :state #renames location column to state
end

La migración anterior cambia una tabla orders. Aquí hay una descripción línea por línea de los cambios:

  1. t.remove :ordered_at elimina la columna ordered_at de la tabla orders.
  2. t.string :skew_number agrega una nueva columna de tipo cadena llamada skew_number en la tabla orders.
  3. t.index :skew_number agrega un índice en la columna skew_number en la tabla orders.
  4. t.rename :ubicación, :estado cambia el nombre de la columna ubicación en la tabla pedidos a estado.

Agregar una columna única a una tabla #

Para agregar una nueva columna única correo electrónico a usuarios, ejecute el siguiente comando:

rails generate migration AddEmailToUsers email:string:uniq

Esto creará la siguiente migración:

class AddEmailToUsers < ActiveRecord::Migration[5.0]
  def change
    add_column :users, :email, :string
    add_index :users, :email, unique: true
  end
end

Podemos comprobar el estado de las migraciones ejecutando

rake db:migrate:status

rails db:migrate:status

La salida se verá así:

En el campo de estado, “activo” significa que se ha ejecutado la migración y “inactivo” significa que necesitamos ejecutar la migración.

Prohibir valores nulos #

Para prohibir valores null en las columnas de su tabla, agregue el parámetro :null a su migración, así:

class AddPriceToProducts < ActiveRecord::Migration
  def change
    add_column :products, :float, null: false
  end
end

Agregar una restricción NOT NULL a los datos existentes #

Digamos que desea agregar una clave externa company_id a la tabla users y desea tener una restricción NO NULL en ella. Si ya tiene datos en usuarios, tendrá que hacer esto en varios pasos.

class AddCompanyIdToUsers < ActiveRecord::Migration
  def up
    # add the column with NULL allowed
    add_column :users, :company_id, :integer

    # make sure every row has a value
    User.find_each do |user|
      # find the appropriate company record for the user
      # according to your business logic
      company = Company.first
      user.update!(company_id: company.id)
    end

    # add NOT NULL constraint
    change_column_null :users, :company_id, false
  end

  # Migrations that manipulate data must use up/down instead of change
  def down
    remove_column :users, :company_id
  end
end
