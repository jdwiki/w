Crear un modelo a través de un generador #

Ruby on Rails proporciona un generador de modelos que puede usar para crear modelos ActiveRecord. Simplemente use rieles generar modelo y proporcione el nombre del modelo.

$ rails g model user

Además del archivo de modelo en app/models, el generador también creará:

  • la Prueba en test/models/user_test.rb
  • los accesorios en test/fixtures/users.yml
  • la migración de la base de datos en db/migrate/XXX_create_users.rb

También puede generar algunos campos para el modelo al generarlo.

$ rails g model user email:string sign_in_count:integer birthday:date

Esto creará las columnas email, sign_in_count y birthday en su base de datos, con los tipos apropiados.

Crear un modelo manualmente #

Si bien el uso de scaffolding es rápido y fácil si es nuevo en Rails o está creando una nueva aplicación, más adelante puede ser útil hacerlo por su cuenta para evitar la necesidad de revisar el código generado por scaffold para simplificarlo. (retirar las piezas no utilizadas, etc.).

Crear un modelo puede ser tan simple como crear un archivo en app/models.

El modelo más simple, en ActiveRecord, es una clase que extiende ActiveRecord::Base.

class User < ActiveRecord::Base
end

Los archivos del modelo se almacenan en app/models/, y el nombre del archivo corresponde al nombre singular de la clase:

# user
app/models/user.rb

# SomeModel
app/models/some_model.rb

La clase heredará todas las características de ActiveRecord: métodos de consulta, validaciones, devoluciones de llamadas, etc.

# Searches the User with ID 1
User.find(1)

Nota: Asegúrese de que exista la tabla para el modelo correspondiente. Si no, puede crear la tabla creando una Migración

Puede generar un modelo y su migración por terminal desde el siguiente comando

rieles g modelo column_name1:data_type1, column_name2:data_type2, ...

y también puede asignar una clave externa (relación) al modelo siguiendo el comando

rails g model column_name:data_type, model_name:references

1: https://www.wikiod.com/es/ruby-on-rails/registro-activo#Crear una migración

Introducción a las devoluciones de llamadas #

Una devolución de llamada es un método que se llama en momentos específicos del ciclo de vida de un objeto (justo antes o después de la creación, eliminación, actualización, validación, guardado o carga desde la base de datos).

Por ejemplo, supongamos que tiene una lista que vence dentro de los 30 días posteriores a la creación.

Una forma de hacerlo es así:

class Listing < ApplicationRecord
  after_create :set_expiry_date

  private

  def set_expiry_date
    expiry_date = Date.today + 30.days
    self.update_column(:expires_on, expiry_date)
  end
end

Todos los métodos disponibles para las devoluciones de llamada son los siguientes, en el mismo orden en que se llaman durante la operación de cada objeto:

Creando un Objeto

  • before_validation
  • después de la validación
  • before_save
  • alrededor_guardar
  • before_create
  • alrededor_crear
  • después_crear
  • after_save
  • after_commit/after_rollback

Actualizando un Objeto

  • before_validation
  • después de la validación
  • before_save
  • alrededor_guardar
  • before_update
  • around_update
  • despúes de actualizar
  • after_save
  • after_commit/after_rollback

Destruyendo un Objeto

  • before_destroy
  • alrededor_destruir
  • después de destruir
  • after_commit/after_rollback

NOTA: after_save se ejecuta tanto en la creación como en la actualización, pero siempre después de las devoluciones de llamada más específicas after_create y after_update, sin importar el orden en que se ejecutaron las macro llamadas.

Probar manualmente sus modelos #

Probar sus modelos de Active Record a través de su interfaz de línea de comandos es simple. Navegue hasta el directorio de la aplicación en su terminal y escriba consola de Rails para iniciar la consola de Rails. Desde aquí, puede ejecutar métodos de registro activos en su base de datos.

Por ejemplo, si tuviera un esquema de base de datos con una tabla de usuarios que tuviera una columna name:string y email:string, podría ejecutar:

User.create name: "John", email: "john@example.com"

Luego, para mostrar ese registro, podrías ejecutar:

User.find_by email: "john@example.com"

O si este es su primer o único registro, simplemente puede obtener el primer registro ejecutando:

User.first

Crear una migración #

Agregar/eliminar campos en tablas existentes #

Cree una migración ejecutando:

rails generate migration AddTitleToCategories title:string

Esto creará una migración que agrega una columna de título a una tabla de categorías:

class AddTitleToCategories < ActiveRecord::Migration[5.0]
  def change
    add_column :categories, :title, :string
  end
end

De manera similar, puede generar una migración para eliminar una columna: los rieles generan migración RemoveTitleFromCategories title:string

Esto creará una migración que eliminará una columna título de la tabla categorías:

class RemoveTitleFromCategories < ActiveRecord::Migration[5.0]
  def change
    remove_column :categories, :title, :string
  end
end

Si bien, estrictamente hablando, especificar tipo (:cadena en este caso) no es necesario para eliminar una columna, es útil, ya que proporciona la información necesaria para desplazarla espalda.

Crear una tabla #

Cree una migración ejecutando:

rails g CreateUsers name bio

Rails reconoce la intención de crear una tabla a partir del prefijo Crear, el resto del nombre de migración se usará como nombre de tabla. El ejemplo dado genera lo siguiente:

class CreateUsers < ActiveRecord::Migration
  def change
    create_table :users do |t|
      t.string :name
      t.string :bio
    end
  end
end

Tenga en cuenta que el comando de creación no especificó los tipos de columnas y se utilizó la cadena predeterminada.

Crear una tabla de unión #

Cree una migración ejecutando:

rails g CreateJoinTableParticipation user:references group:references

Rails detecta la intención de crear una tabla de unión al encontrar JoinTable en el nombre de la migración. Todo lo demás se determina a partir de los nombres de los campos que proporciona después del nombre.

class CreateJoinTableParticipation < ActiveRecord::Migration
  def change
    create_join_table :users, :groups do |t|
      # t.index [:user_id, :group_id]
      # t.index [:group_id, :user_id]
    end
  end
end

Elimine los comentarios de las declaraciones index necesarias y elimine el resto.

precedencia #

Tenga en cuenta que el nombre de migración de ejemplo CreateJoinTableParticipation coincide con la regla para la creación de tablas: tiene un prefijo Crear. Pero no generó un create_table simple. Esto se debe a que el generador de migración (código fuente) usa una primera coincidencia de la siguiente lista:

  • (Agregar|Eliminar)<ignorado>(A|Desde)<nombre_tabla>
  • <ignorado>JoinTable<ignorado>
  • Crear<nombre_tabla>

Crear una tabla de unión usando migraciones #

Especialmente útil para la relación has_and_belongs_to_many, puede crear manualmente una tabla de unión usando el método create_table. Supongamos que tiene dos modelos Etiquetas y Proyectos, y desea asociarlos mediante una relación tiene_y_pertenece_a_muchos. Necesita una tabla de combinación para asociar instancias de ambas clases.

class CreateProjectsTagsJoinTableMigration < ActiveRecord::Migration
  def change
    create_table :projects_tags, id: false do |t|
      t.integer :project_id
      t.integer :tag_id
    end
  end
end

El nombre real de la tabla debe seguir esta convención: el modelo que precede alfabéticamente al otro debe ir primero. Project precede a Tags, por lo que el nombre de la tabla es project_tags.

Además, dado que el propósito de esta tabla es enrutar la asociación entre las instancias de dos modelos, la identificación real de cada registro en esta tabla no es necesaria. Usted especifica esto pasando id: false

Finalmente, como es costumbre en Rails, el nombre de la tabla debe ser la forma plural compuesta de los modelos individuales, pero la columna de la tabla debe estar en forma singular.
