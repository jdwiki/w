Documentación del archivo Gem #

Para proyectos que se espera que crezcan, es una buena idea agregar comentarios a su Gemfile. De esa manera, incluso en configuraciones grandes, aún sabrá qué hace cada gema, incluso si el nombre no se explica por sí mismo y lo agregó hace 2 años.

Esto también puede ayudarlo a recordar por qué eligió una determinada versión y, en consecuencia, volver a evaluar el requisito de la versión más adelante.

Ejemplos:

# temporary downgrade for TeamCity
gem 'rake', '~> 10.5.0'
# To upload invoicing information to payment provider
gem 'net-sftp'

¿Qué es una gema? #

Una gema es el equivalente a un complemento o una extensión para el lenguaje de programación ruby.

Para ser exactos, incluso los rieles no son más que una joya. Muchas gemas se construyen sobre rieles u otras gemas (dependen de dicha gema) o son independientes.

En su proyecto Rails #

Archivo de gemas #

Para su proyecto Rails tiene un archivo llamado Gemfile. Aquí puede agregar las gemas que desea incluir y usar en su proyecto. Una vez agregado, debe instalar la gema usando bundler (consulte la sección Bundler).

Gemfile.bloqueo #

Una vez que haya hecho esto, su Gemfile.lock se actualizará con las gemas recién agregadas y sus dependencias. Este archivo bloquea sus gemas usadas para que usen esa versión específica declarada en ese archivo.

JOYA
remoto: https://rubygems.org/
especificaciones:
idear (4.0.3)
bcrypt (~> 3.0)
adaptador_orm (~> 0.1)
rieles (>= 4.1.0, < 5.1)
respondedores
alcaide (~> 1.2.3)

Este ejemplo es para la gema devise. En Gemfile.lock se declara la versión 4.0.3, para indicar al instalar su proyecto en otra máquina o en su servidor de producción qué versión específica usar.

Desarrollo #

Ya sea una sola persona, un grupo o toda una comunidad trabaja y mantiene una joya. El trabajo realizado generalmente se libera después de que se hayan solucionado ciertos “problemas” o se hayan agregado “características”.

Por lo general, los lanzamientos siguen el principio [Semantic Versioning 2.0.0] (http://semver.org/).

Empaquetador #

La forma más fácil de manejar y administrar gemas es usando bundler. Bundler es un administrador de paquetes comparable a Bower.

Para usar el paquete, primero debe instalarlo.

gem install bundler

Una vez que haya instalado el paquete y esté funcionando, todo lo que necesita hacer es agregar gemas a su Gemfile y ejecutar

bundle

en tu terminal. Esto instala sus gemas recién agregadas a su proyecto. En caso de que surja un problema, recibirá un aviso en su terminal.

Si está interesado en obtener más detalles, le sugiero que eche un vistazo a los [docs] (http://bundler.io/docs.html).

archivos de gemas #

Para comenzar, los archivos gemfile requieren al menos una fuente, en forma de URL para un servidor RubyGems.

Genere un Gemfile con la fuente predeterminada de rubygems.org ejecutando bundle init. Use https para que su conexión al servidor sea verificada con SSL.

source 'https://rubygems.org'

A continuación, declare las gemas que necesita, incluidos los números de versión.

gem 'rails', '4.2.6'
gem 'rack',  '>=1.1'
gem 'puma',  '~>3.0'

La mayoría de los especificadores de versión, como >= 1.0, se explican por sí mismos. El especificador ~> tiene un significado especial. ~> 2.0.3 es idéntico a >= 2.0.3 y < 2.1. ~> 2.1 es idéntico a >= 2.1 y < 3.0. ~> 2.2.beta coincidirá con versiones preliminares como 2.2.beta.12.

Los repositorios de Git también son fuentes de gemas válidas, siempre que el repositorio contenga una o más gemas válidas. Especifique qué comprobar con :tag, :branch o :ref. El valor predeterminado es la rama maestro.

gem 'nokogiri', :git => 'https://github.com/sparklemotion/nokogiri', :branch => 'master'

Si desea utilizar una gema desempaquetada directamente desde el sistema de archivos, simplemente establezca la opción :path en la ruta que contiene los archivos de la gema.

gem 'extracted_library', :path => './vendor/extracted_library'

Las dependencias se pueden colocar en grupos. Los grupos se pueden ignorar en el momento de la instalación (usando --with) o se pueden requerir todos a la vez (usando Bundler.require).

gem 'rails_12factor', group: :production

group :development, :test do
  gem 'byebug'
  gem 'web-console', '~> 2.0'
  gem 'spring'
  gem 'dotenv-rails'
end

Puede especificar la versión requerida de Ruby en el Gemfile con ruby. Si el Gemfile se carga en una versión diferente de Ruby, Bundler generará una excepción con una explicación.

ruby '2.3.1'

Conjuntos de gemas #

Si está usando RVM (Ruby Version Manager) entonces usar un gemset para cada proyecto es una buena idea. Un “conjunto de gemas” es solo un contenedor que puede usar para mantener las gemas separadas entre sí. La creación de un “conjunto de gemas” por proyecto le permite cambiar gemas (y versiones de gemas) para un proyecto sin romper todos sus otros proyectos. Cada proyecto solo debe preocuparse por sus propias gemas.

RVM proporciona (>= 0.1.8) un @global gemset por intérprete de Ruby. Las gemas que instale en el @conjunto de gemas global para un rubí determinado están disponibles para todos los demás conjuntos de gemas que cree en asociación con ese rubí. Esta es una buena manera de permitir que todos sus proyectos compartan la misma gema instalada para una instalación específica de intérprete de Ruby.

Creación de conjuntos de gemas

Suponga que ya tiene ruby-2.3.1 instalado y lo ha seleccionado usando este comando:

rvm use ruby-2.3.1

Ahora, para crear un conjunto de gemas para esta versión de Ruby:

rvm gemset create new_gemset

donde new_gemset es el nombre del conjunto de gemas. Para ver la lista de conjuntos de gemas disponibles para una versión Ruby:

rvm gemset list

para enumerar las gemas de todas las versiones de Ruby:

rvm gemset list_all

para usar un conjunto de gemas de la lista (supongamos que new_gemset es el conjunto de gemas que quiero usar):

rvm gemset use new_gemset

también puede especificar la versión de rubí con el conjunto de gemas si desea cambiar a otra versión de rubí:

rvm use ruby-2.1.1@new_gemset

para especificar un conjunto de gemas predeterminado para una versión particular de Ruby:

rvm use 2.1.1@new_gemset --default

para eliminar todas las gemas instaladas de un conjunto de gemas, puede vaciarlo de la siguiente manera:

rvm gemset empty new_gemset

para copiar un conjunto de gemas de un rubí a otro, puede hacerlo de la siguiente manera:

rvm gemset copy 2.1.1@rails4 2.1.2@rails4

para eliminar un conjunto de gemas:

rvm gemset delete new_gemset

para ver el nombre del conjunto de gemas actual:

rvm gemset name

para instalar una gema en el conjunto de gemas global:

rvm @global do gem install ...

Inicializar conjuntos de gemas durante las instalaciones de Ruby

Cuando instala un nuevo Ruby, RVM no solo crea dos conjuntos de gemas (el conjunto de gemas vacío predeterminado y el conjunto de gemas global), sino que también utiliza un conjunto de archivos editables por el usuario para determinar qué gemas instalar.

Trabajando en ~/.rvm/gemsets, rvm busca global.gems y default.gems utilizando una jerarquía de árbol basada en la cadena ruby ​​que se está instalando. Usando el ejemplo de ree-1.8.7-p2010.02, rvm verificará (e importará desde) los siguientes archivos:

~/.rvm/gemsets/ree/1.8.7/p2010.02/global.gems
~/.rvm/gemsets/ree/1.8.7/p2010.02/default.gems
~/.rvm/gemsets/ree/1.8.7/global.gems
~/.rvm/gemsets/ree/1.8.7/default.gems
~/.rvm/gemsets/ree/global.gems
~/.rvm/gemsets/ree/default.gems
~/.rvm/gemsets/global.gems
~/.rvm/gemsets/default.gems

Por ejemplo, si editó ~/.rvm/gemsets/global.gems agregando estas dos líneas:

bundler
awesome_print

cada vez que instala un rubí nuevo, estas dos gemas se instalan en su conjunto de gemas global. Los archivos default.gems y global.gems generalmente se sobrescriben durante la actualización de rvm.
