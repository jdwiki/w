Agregue la etiqueta javascript de Google Maps al encabezado del diseño #

Para que Google Maps funcione correctamente con turbolinks, agregue la etiqueta javascript directamente al encabezado del diseño en lugar de incluirla en una vista.

# app/views/layouts/my_layout.html.haml
!!!
%html{:lang => 'en'}
  %head
    - # ...
    = google_maps_api_script_tag

google_maps_api_script_tag se define mejor en un asistente.

# app/helpers/google_maps_helper.rb
module GoogleMapsHelper
  def google_maps_api_script_tag
    javascript_include_tag google_maps_api_source
  end

  def google_maps_api_source
    "https://maps.googleapis.com/maps/api/js?key=#{google_maps_api_key}"
  end

  def google_maps_api_key
    Rails.application.secrets.google_maps_api_key
  end
end

Puede registrar su aplicación con Google y obtener su clave API en la [consola API de Google] (https://console.developers.google.com). Google tiene una breve [guía sobre cómo solicitar una clave de API para la API de JavaScript de Google Maps] (https://developers.google.com/maps/documentation/javascript/get-api-key?hl=de).

La clave api se almacena en el archivo secrets.yml:

# config/secrets.yml
development:
  google_maps_api_key: '...'
  # ...
production:
  google_maps_api_key: '...'
  # ...

No olvide agregar config/secrets.yml a su archivo .gitignore y asegúrese de no enviar la clave api al repositorio.

Geocodificar el modelo #

Supongamos que sus usuarios y/o grupos tienen perfiles y desea mostrar campos de perfil de dirección en un mapa de Google.

# app/models/profile_fields/address.rb
class ProfileFields::Address < ProfileFields::Base
  # Attributes: 
  # label, e.g. "Work address"
  # value, e.g. "Willy-Brandt-Straße 1\n10557 Berlin"
end

Una excelente manera de geocodificar las direcciones, es decir, proporcionar “longitud” y “latitud” es la [gema del geocodificador] (https://github.com/alexreisner/geocoder).

Agrega geocodificador a tu Gemfile y ejecuta bundle para instalarlo.

# Gemfile
gem 'geocoder', '~> 1.3'

Agregue columnas de base de datos para latitud y longitud para guardar la ubicación en la base de datos. Esto es más eficiente que consultar el servicio de geocodificación cada vez que necesite la ubicación. Es más rápido y no alcanza el límite de consultas tan rápido.

➜ bin/rails generate migration add_latitude_and_longitude_to_profile_fields \
    latitude:float longitude:float
➜ bin/rails db:migrate  # Rails 5, or:
➜ rake db:migrate       # Rails 3, 4

Agregue el mecanismo de geocodificación a su modelo. En este ejemplo, la cadena de dirección se almacena en el atributo valor. Configure la geocodificación para que se realice cuando el registro haya cambiado y solo cuando haya un valor presente:

# app/models/profile_fields/address.rb
class ProfileFields::Address < ProfileFields::Base
  geocoded_by :value
  after_validation :geocode, if: ->(address_field){ 
    address_field.value.present? and address_field.value_changed? 
  }

end

De forma predeterminada, geocoder utiliza Google como servicio de búsqueda. Tiene muchas características interesantes como cálculos de distancia o búsqueda de proximidad. Para obtener más información, consulte el [LÉAME del geocodificador] (https://github.com/alexreisner/geocoder).

Mostrar direcciones en un mapa de Google en la vista de perfil #

En la vista de perfil, muestre los campos de perfil de un usuario o grupo en una lista, así como los campos de dirección en un mapa de Google.

- # app/views/profiles/show.html.haml
%h1 Contact Information
.profile_fields
  = render @profile_fields
.google_map{data: address_fields: @address_fields.to_json }

Los @profile_fields y @address_fields apropiados se establecen en el controlador:

# app/controllers/profiles_controller.rb
class ProfilesController < ApplicationController
  def show
    # ...
    @profile_fields = @user_or_group.profile_fields
    @address_fields = @profile_fields.where(type: 'ProfileFields::Address')
  end
end

Inicialice el mapa, coloque los marcadores, configure el zoom y otras configuraciones del mapa con javascript.

Establecer los marcadores en el mapa con javascript #

Supongamos que hay un div .google_map, que se convertirá en el mapa, y que tiene los campos de dirección para mostrar como marcadores como atributo data.

Por ejemplo:

<!-- http://localhost:3000/profiles/123 -->
<div class="google_map" data-address-fields="[
  {label: 'Work address', value: 'Willy-Brandt-Straße 1\n10557 Berlin', 
  position: {lng: ..., lat: ...}},
  ...
]"></div>

Para hacer uso del evento $(document).ready con turbolinks sin administrar los eventos de turbolinks a mano, use la gema [jquery.turbolinks](https: //github.com/kossnocorp/jquery.turbolinks).

Si desea realizar otras operaciones con el mapa, más adelante, por ejemplo, filtrado o ventanas de información, es conveniente que el mapa sea administrado por una [clase de script de café] (http://coffeescript.org/#classes).

# app/assets/javascripts/google_maps.js.coffee
window.App = {} unless App?
class App.GoogleMap
  constructor: (map_div)->
    # TODO: initialize the map
    # TODO: set the markers

Cuando se utilizan varios archivos de secuencias de comandos de café, que tienen un espacio de nombres predeterminado, es conveniente definir un espacio de nombres global de “aplicación”, que se comparte con todos los archivos de secuencias de comandos de café.

Luego, recorra (posiblemente varios) divs .google_map y cree una instancia de la clase App.GoogleMap para cada uno de ellos.

# app/assets/javascripts/google_maps.js.coffee
# ...
$(document).ready ->
  App.google_maps = []
  $('.google_map').each ->
    map_div = $(this)
    map = new App.GoogleMap map_div
    App.google_maps.push map

Inicialice el mapa usando una clase de script de café. #

Si se proporciona una App.GoogleMap [clase de script de café] (http://coffeescript.org/#classes), el mapa de Google se puede inicializar de esta manera:

# app/assets/javascripts/google_maps.js.coffee
# ...
class App.GoogleMap
  map_div: {}
  map: {}
  
  constructor: (map_div)->
    @map_div = map_div
    @init_map()
    @reference_the_map_as_data_attribute

  # To access the GoogleMap object or the map object itself
  # later via the DOM, for example 
  # 
  #     $('.google_map').data('GoogleMap')
  #
  # store references as data attribute of the map_div.
  #
  reference_the_map_as_data_attribute: ->
    @map_div.data 'GoogleMap', this
    @map_div.data 'map', @map

  init_map: ->
    @map = new google.maps.Map(@dom_element, @map_configuration) if google?

  # `@map_div` is the jquery object. But google maps needs
  # the real DOM element.
  #
  dom_element: ->
    @map_div.get(0)

  map_configuration: -> {
    scrollWheel: true
  }

Para obtener más información sobre las posibles opciones de map_configuration, consulte la [documentación de MapOptions] de Google (https://developers.google.com/maps/documentation/javascript/3.exp/reference#MapOptions) y su guía para agregando elementos de control.

Como referencia, la clase [google.maps.Map está ampliamente documentada aquí] (https://developers.google.com/maps/documentation/javascript/3.exp/reference#Map).

Inicialice los marcadores del mapa usando una clase de script de café #

Proporcionó una App.GoogleMap [clase de script de café] (http://coffeescript.org/#classes) y la información del marcador se almacena en el atributo data-address-fields del div .google_map, el mapa Los marcadores se pueden inicializar en el mapa de esta manera:

# app/assets/javascripts/google_maps.js.coffee
# ...
class App.GoogleMap
  # ...
  markers: []

  constructor: (map_div)->
    # ...
    @init_markers()

  address_fields: ->
    @map_div.data('address-fields')

  init_markers: ->
    self = this  # to reference the instance as `self` when `this` is redefined.
    self.markers = []
    for address_field in self.address_fields()
      marker = new google.maps.Marker {
        map: self.map,
        position: {
          lng: address_field.longitude,
          lat: address_field.latitude
        },
        # # or, if `position` is defined in `ProfileFields::Address#as_json`:
        # position: address_field.position,
        title: address_field.value
      }
      self.markers.push marker

Para obtener más información sobre las opciones de marcadores, consulte la documentación de MarkerOptions de Google y su [guía de marcadores](https ://developers.google.com/maps/documentation/javascript/markers).

Hacer zoom automático en un mapa usando una clase de script de café #

Proporcionó una App.GoogleMap [clase de script de café] (http://coffeescript.org/#classes) con google.maps.Map almacenado como @map y google.maps.Marker almacenado como @markers, el mapa se puede ampliar automáticamente, es decir, se puede ajustar para que todos los marcadores estén visibles, así: en el mapa así:

# app/assets/javascripts/google_maps.js.coffee
# ...
class App.GoogleMap
  # ...
  bounds: {}

  constructor: (map_div)->
    # ...
    @auto_zoom()

  auto_zoom: ->
    @init_bounds()
    # TODO: Maybe, adjust the zoom to have a maximum or 
    # minimum zoom level, here.

  init_bounds: ->
    @bounds = new google.maps.LatLngBounds()
    for marker in @markers
      @bounds.extend marker.position
    @map.fitBounds @bounds

Para obtener más información sobre los límites, consulte la [documentación de LatLngBounds] de Google (https://developers.google.com/maps/documentation/javascript/3.exp/reference#LatLngBounds).

Exponiendo las propiedades del modelo como json #

Para mostrar los campos de perfil de dirección como marcadores en un mapa de Google, los objetos de campo de dirección deben pasarse como objetos json a javascript.

Atributos regulares de la base de datos #

Al llamar a to_json en un objeto ApplicationRecord, los atributos de la base de datos se exponen automáticamente.

Dado un modelo ProfileFields::Address con atributos label, value, longitude y latitude, address_field.as_json da como resultado un Hash, p. representación,

address_field.as_json  # =>
  {label: "Work address", value: "Willy-Brandt-Straße 1\n10557 Berlin",
    longitude: ..., latitude: ...}

que se convierte en una cadena json por to_json:

address_field.to_json  # =>
  "{\"label\":\"Work address\",\"value\":\"Willy-Brandt-Straße 1\\n
    10557 Berlin\",\"longitude\":...,\"latitude\":...}"

Esto es útil porque permite usar label y value más tarde en javascript, por ejemplo, para mostrar información sobre herramientas para los marcadores de mapa.

Otros atributos #

Se pueden exponer otros atributos virtuales anulando el método as_json.

Por ejemplo, para exponer un atributo título, inclúyalo en el hash fusionado as_json:

# app/models/profile_fields/address.rb
class ProfileFields::Address < ProfileFields::Base
  # ...

  # For example: "John Doe, Work address"
  def title
    "#{self.parent.name}, #{self.label}"
  end

  def as_json
    super.merge {
      title: self.title
    }
  end
end

El ejemplo anterior usa super para llamar al método as_json original, que devuelve el hash del atributo original del objeto y lo fusiona con la posición requerida picadillo.

Para comprender la diferencia entre as_json y to_json, eche un vistazo a esta entrada de blog de jjulian.

Posición #

Para representar marcadores, la API de Google Maps, de forma predeterminada, requiere un hash de “posición” que tiene la longitud y la latitud almacenadas como “lng” y “lat” respectivamente.

Este hash de posición se puede crear en javascript, más tarde o aquí al definir la representación json del campo de dirección:

Para proporcionar esta posición como atributo json del campo de dirección, simplemente anule el método as_json en el modelo.

# app/models/profile_fields/address.rb
class ProfileFields::Address < ProfileFields::Base
  # ...

  def as_json
    super.merge {
      # ...
      position: {
        lng: self.longitude,
        lat: self.latitude
      }
    }
  end
end
