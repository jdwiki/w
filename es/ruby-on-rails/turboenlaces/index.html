Turboenlaces

Turbolinks es una biblioteca de javascript que hace que la navegación por su aplicación web sea más rápida. Cuando sigue un enlace, Turbolinks busca automáticamente la página, intercambia su <cuerpo> y fusiona su <cabeza>, todo sin incurrir en el costo de una carga completa de la página.

Como desarrollador de rieles, probablemente interactuará mínimamente con turboenlaces durante su desarrollo. Sin embargo, es una biblioteca importante con la que debe familiarizarse porque puede ser la causa de algunos errores difíciles de encontrar.

Conclusiones clave: #

  • Enlace al evento turbolinks:load en lugar del evento document.ready
  • Use el atributo data-turbolinks-false para deshabilitar la funcionalidad de turboenlace por enlace.
  • Use el atributo data-turbolinks-permanent para persistir elementos a lo largo de las cargas de la página y para evitar errores relacionados con el caché.

Para obtener un tratamiento más detallado de los turboenlaces, visite el repositorio oficial de github.

El crédito de gran parte de esta documentación es para las personas que redactaron la documentación de turbolinks en el repositorio de github.

Enlace al concepto de carga de página de turbolink #

Con turbolinks, el enfoque tradicional para usar:

$(document).ready(function() {
  // awesome code
});

no funcionará Al usar turboenlaces, el evento $(document).ready() solo se activará una vez: en la carga de la página inicial. A partir de ese momento, cada vez que un usuario haga clic en un enlace de su sitio web, turbolinks interceptará el evento de clic en el enlace y realizará una solicitud ajax para reemplazar la etiqueta <body> y fusionar las etiquetas <head>. Todo el proceso desencadena la noción de una “visita” en tierra de turboenlaces. Por lo tanto, en lugar de usar la sintaxis tradicional document.ready() anterior, tendrá que vincularse al evento de visita de turbolink de la siguiente manera:

// pure js
document.addEventListener("turbolinks:load", function() {
  // awesome code
});

// jQuery
$(document).on('turbolinks:load', function() {
  // your code
});

Deshabilitar turboenlaces en enlaces específicos #

Es muy fácil deshabilitar los turboenlaces en enlaces específicos. Según la documentación oficial de turbolinks:

Los turboenlaces se pueden deshabilitar por enlace anotando un enlace o cualquiera de sus ancestros con data-turbolinks=“false”.

Ejemplos: #

// disables turbolinks for this one link
<a href="/" data-turbolinks="false">Disabled</a>

// disables turbolinks for all links nested within the div tag
<div data-turbolinks="false">
  <a href="/">I'm disabled</a>
  <a href="/">I'm also disabled</a>
</div>

// re-enable specific link when ancestor has disabled turbolinks
<div data-turbolinks="false">
  <a href="/">I'm disabled</a>
  <a href="/" data-turbolinks="true">I'm re-enabled</a>
</div>

Comprender las visitas a la aplicación #

Las visitas a la aplicación se inician al hacer clic en un enlace habilitado para Turbolinks o mediante programación al llamar

Turbolinks.visit(location)

Por defecto, la función de visita utiliza la acción ‘avanzar’. Más comprensiblemente, el comportamiento predeterminado para la función de visita es avanzar a la página indicada por el parámetro “ubicación”. Cada vez que se visita una página, turbolinks inserta una nueva entrada en el historial del navegador usando history.pushState. El historial es importante porque turbolinks intentará utilizar el historial para cargar páginas desde la memoria caché siempre que sea posible. Esto permite una representación de página extremadamente rápida para las páginas visitadas con frecuencia.

Sin embargo, si desea visitar una ubicación sin agregar ningún historial a la pila, puede usar la acción ‘reemplazar’ en la función de visita de la siguiente manera:

// using links
<a href="/edit" data-turbolinks-action="replace">Edit</a>

// programatically
Turbolinks.visit("/edit", { action: "replace" })

Esto reemplazará la parte superior de la pila de historial con la nueva página para que la cantidad total de elementos en la pila permanezca sin cambios.

También hay una acción de “restauración” que ayuda en las visitas de restauración, las visitas que ocurren como resultado de que el usuario haga clic en el botón de avance o retroceso en su navegador. Turbolinks maneja este tipo de eventos internamente y recomienda que los usuarios no alteren manualmente el comportamiento predeterminado.

Cancelar visitas antes de que comiencen #

Turbolinks proporciona un detector de eventos que se puede usar para evitar que se produzcan visitas. Escuche el evento turbolinks:before-visit para recibir una notificación cuando una visita esté a punto de comenzar.

En el controlador de eventos, puede usar:

// pure javascript
event.data.url

o

// jQuery
$event.originalEvent.data.url

para recuperar la ubicación de la visita. La visita se puede cancelar llamando al:

event.preventDefault()

NOTA: #

Según los documentos oficiales de turbolinks:

Las visitas de restauración no se pueden cancelar y no activan turboenlaces: antes de la visita.

Elementos persistentes a través de cargas de página #

Considere la siguiente situación: imagine que es el desarrollador de un sitio web de redes sociales que permite a los usuarios ser amigos de otros usuarios y que emplea enlaces turbo para que la página se cargue más rápido. En la parte superior derecha de cada página del sitio, hay un número que indica el número total de amigos que tiene actualmente un usuario. Imagina que estás usando tu sitio y que tienes 3 amigos. Cada vez que se agrega un nuevo amigo, tiene un javascript que se ejecuta y actualiza el contador de amigos. Imagine que acaba de agregar un nuevo amigo y que su javascript se ejecutó correctamente y actualizó el recuento de amigos en la parte superior derecha de la página para mostrar ahora 4. Ahora, imagine que hace clic en el botón Atrás del navegador. Cuando se carga la página, nota que el contador de amigos dice 3 a pesar de que tiene cuatro amigos.

Este es un problema relativamente común y para el que turbolinks ha proporcionado una solución. La razón por la que ocurre el problema es porque turbolinks carga automáticamente páginas desde el caché cuando un usuario hace clic en el botón Atrás. La página almacenada en caché no siempre se actualizará con la base de datos.

Para resolver este problema, imagina que representas el conteo de amigos dentro de una etiqueta <div> con una identificación de “conteo de amigos”:

<div id="friend-count" data-turbolinks-permanent>3 friends</div>

Al agregar el atributo data-turbolinks-permanent, le está diciendo a los turbolinks que persistan ciertos elementos a lo largo de las cargas de la página. Los documentos oficiales dicen:

Designe elementos permanentes dándoles una identificación HTML y anotándolos con data-turbolinks-permanent. Antes de cada procesamiento, Turbolinks hace coincidir todos los elementos permanentes por ID y los transfiere de la página original a la nueva página, conservando sus datos y detectores de eventos.
