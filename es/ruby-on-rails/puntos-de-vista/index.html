Estructura #

Como Rails sigue el patrón MVC, Vistas es donde están tus “plantillas” para tus acciones.

Digamos que tienes un controlador articles_controller.rb. Para este controlador tendrías una carpeta en las vistas llamada app/views/articles:

app
|-- controllers
|   '-- articles_controller.rb
|
'-- views
    '-- articles
    |   |- index.html.erb
    |   |- edit.html.erb
    |   |- show.html.erb
    |   |- new.html.erb
    |   '- _partial_view.html.erb
    |
    '-- [...]

Esta estructura le permite tener una carpeta para cada controlador. Al llamar a una acción en su controlador, la vista apropiada se representará automáticamente.

// articles_controller.rb
class ArticlesController < ActionController::Base
  def show
  end
end

// show.html.erb
<h1>My show view</h1>

Parciales #

Las plantillas parciales (parciales) son una forma de dividir el proceso de renderizado en partes más manejables. Los parciales le permiten extraer fragmentos de código de sus plantillas para separar archivos y también reutilizarlos en sus plantillas.

Para crear un parcial, cree un nuevo archivo que comience con un guión bajo: _form.html.erb

Para renderizar un parcial como parte de una vista, utilice el método de renderizado dentro de la vista: <%= representar "formulario" %>

  • Tenga en cuenta que el guión bajo se omite al renderizar
  • Un parcial debe renderizarse utilizando su ruta si se encuentra en una carpeta diferente

Para pasar una variable al parcial como variable local, utilice esta notación:

<%= renderizar: parcial => 'formulario', locales: { publicación: @publicación } %>

Los parciales también son útiles cuando necesitas reutilizar exactamente el mismo código (filosofía DRY).

Por ejemplo, para reutilizar el código <head>, cree un parcial llamado _html_header.html.erb, ingrese su código <head> para reutilizarlo y renderice el parcial cuando sea necesario: <%= render ' html_header' %>.

Objetos parciales #

Los objetos que responden a to_partial_path también se pueden representar, como en <%= render @post %>. De forma predeterminada, para los modelos de ActiveRecord, será algo así como posts/post, por lo que al renderizar @post, se renderizará el archivo views/posts/_post.html.erb.

Se asignará automáticamente un post local llamado. Al final, <%= render @post %> es una forma abreviada de <%= render 'posts/post', post: @post %>.

También se pueden proporcionar colecciones de objetos que responden a to_partial_path, como <%= render @posts %>. Cada elemento se renderizará consecutivamente.

parciales globales #

Para crear un parcial global que se pueda usar en cualquier lugar sin hacer referencia a su ruta exacta, el parcial debe ubicarse en la ruta views/application. El ejemplo anterior se ha modificado a continuación para ilustrar esta característica.

Por ejemplo, esta es una ruta a un parcial global app/views/application/_html_header.html.erb:

Ayudante de etiquetas de activos #

Para permitir que los rieles vinculen automática y correctamente los activos (css/js/images), en la mayoría de los casos, desea utilizar ayudantes integrados. (Documentación oficial)

Ayudantes de imagen #

ruta de la imagen #

Esto devuelve la ruta a un recurso de imagen en app/assets/images.

image_path("edit.png") # => /assets/edit.png

URL de la imagen #

Esto devuelve la URL completa a un recurso de imagen en app/assets/images.

image_url("edit.png") # => http://www.example.com/assets/edit.png

etiqueta_imagen #

Utilice este asistente si desea incluir una etiqueta <img src="" /> con el conjunto de fuentes.

image_tag("icon.png") # => <img src="/assets/icon.png" alt="Icon" />

ayudantes de JavaScript #

javascript_incluye_etiqueta #

Si desea incluir un archivo JavaScript en su vista.

javascript_include_tag "application" # => <script src="/assets/application.js"></script>

ruta_javascript #

Esto devuelve la ruta de su archivo JavaScript.

javascript_path "application" # => /assets/application.js

url_javascript #

Esto devuelve la URL completa de su archivo JavaScript.

javascript_url "application" # => http://www.example.com/assets/application.js

Ayudantes de hojas de estilo #

etiqueta_enlace_hoja de estilo #

Si desea incluir un archivo CSS en su vista.

stylesheet_link_tag "application" # => <link href="/assets/application.css" media="screen" rel="stylesheet" />

ruta_hoja_de_estilo #

Esto devuelve la ruta de su recurso de hoja de estilo.

stylesheet_path "application" # => /assets/application.css

hoja_de_estilo_url #

Esto devuelve la URL completa de su recurso de hoja de estilo.

stylesheet_url "application" # => http://www.example.com/assets/application.css

Ejemplo de uso #

Al crear una nueva aplicación de rieles, automáticamente tendrá dos de estos ayudantes en app/views/layouts/application.html.erb

<%= stylesheet_link_tag    'application', media: 'all', 'data-turbolinks-track': 'reload' %>
<%= javascript_include_tag 'application', 'data-turbolinks-track': 'reload' %>

Esto genera:

// CSS
<link rel="stylesheet" media="all" href="/assets/application.self-e19d4b856cacba4f6fb0e5aa82a1ba9aa4ad616f0213a1259650b281d9cf6b20.css?body=1" data-turbolinks-track="reload" />
// JavaScript
<script src="/assets/application.self-619d9bf310b8eb258c67de7af745cafbf2a98f6d4c7bb6db8e1b00aed89eb0b1.js?body=1" data-turbolinks-track="reload"></script>

Reemplazar código HTML en Vistas #

Si alguna vez quiso determinar el contenido html que se imprimirá en una página durante el tiempo de ejecución, Rails tiene una muy buena solución para eso. Tiene algo llamado content_for que nos permite pasar un bloque a una vista de rieles. Por favor, consulte el siguiente ejemplo,

Declarar contenido_para

<div>
  <%= yield :header %>
</div>

<% content_for :header do %>
 <ul>
   <li>Line Item 1</li>
   <li>Line Item 2</li>
 </ul>
<% end %>

HAML: una forma alternativa de usar en sus vistas #

HAML (lenguaje de marcado de abstracción HTML) es una forma hermosa y elegante de describir y diseñar el HTML de sus vistas. En lugar de etiquetas de apertura y cierre, HAML usa sangría para la estructura de sus páginas. Básicamente, si algo debe colocarse dentro de otro elemento, simplemente sangra usando una tabulación. Las pestañas y los espacios en blanco son importantes en HAML, así que asegúrese de usar siempre la misma cantidad de pestañas.

Ejemplos:

#myview.html.erb
<h1><%= @the_title %></h1>
<p>This is my form</p>
<%= render "form" %>

Y en HAML:

#myview.html.haml
%h1= @the_title
%p
    This is my form
= render 'form'

Verá, la estructura del diseño es mucho más clara que usar HTML y ERB.

Instalación

Solo instala la gema usando

gem install haml

y agrega la gema al Gemfile

gem "haml"

Para usar HAML en lugar de HTML/ERB, simplemente reemplace las extensiones de archivo de sus vistas de algo.html.erb a ​​algo.html.haml.

Consejos rápidos

Los elementos comunes como divs se pueden escribir de forma abreviada

HTML

<div class="myclass">My Text</div>

ALDEA

%div.myclass

HAML, taquigrafía

.myclass

Atributos

HTML

<p class="myclass" id="myid">My paragraph</p>

ALDEA

%p{:class => "myclass", :id => "myid"} My paragraph

Insertando código ruby

Puede insertar código Ruby con los signos = y -.

= link_to "Home", home_path

El código que comienza con = se ejecutará e incrustará en el documento.

El código que comienza con - se ejecutará, pero no se insertará en el documento.

Documentación completa

HAML es muy fácil para empezar, pero también es muy complejo, por lo que recomendaré leer la documentación.
