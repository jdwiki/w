Definiendo un administrador básico usando Querysets y el método as_manager #

El administrador de Django es una interfaz a través de la cual el modelo Django consulta la base de datos. El campo objects que se usa en la mayoría de las consultas de django es en realidad el administrador predeterminado creado para nosotros por django (esto solo se crea si no definimos administradores personalizados).

¿Por qué definiríamos un administrador/conjunto de consultas personalizado?

Para evitar escribir consultas comunes en todo nuestro código base y, en su lugar, referirlas utilizando una abstracción más fácil de recordar. Ejemplo: Decida usted mismo qué versión es más legible:

  • Solo obtener todos los usuarios activos: User.objects.filter(is_active=True) vs User.manager.active()
  • Obtenga todos los dermatólogos activos en nuestra plataforma: User.objects.filter(is_active=True).filter(is_doctor=True).filter(specialization='Dermatology') vs Usuario.gestor.médicos.con_especialización('Dermatología')

Otro beneficio es que si mañana decidimos que todos los ‘psicólogos’ también son ‘dermatólogos’, podemos modificar fácilmente la consulta en nuestro Administrador y listo.

A continuación se muestra un ejemplo de creación de un Administrador personalizado definido mediante la creación de un QuerySet y el uso del método as_manager.

from django.db.models.query import QuerySet

class ProfileQuerySet(QuerySet):
    def doctors(self):
        return self.filter(user_type="Doctor", user__is_active=True)

    def with_specializations(self, specialization):
        return self.filter(specializations=specialization)

    def users(self):
        return self.filter(user_type="Customer", user__is_active=True)

ProfileManager = ProfileQuerySet.as_manager

Lo agregaremos a nuestro modelo de la siguiente manera:

class Profile(models.Model):
    ...
    manager = ProfileManager()

NOTA: una vez que hayamos definido un “administrador” en nuestro modelo, los “objetos” ya no se definirán para el modelo.

select_related para todas las consultas #

Modelo con ForeignKey

Trabajaremos con estos modelos:

from django.db import models

class Book(models.Model):
 name= models.CharField(max_length=50)
 author = models.ForeignKey(Author)

class Author(models.Model):
 name = models.CharField(max_length=50)

Supongamos que a menudo (siempre) accedemos a libro.autor.nombre

En vista

Podríamos usar lo siguiente, cada vez,

books = Book.objects.select_related('author').all()

Pero esto no es SECO.

Administrador personalizado

class BookManager(models.Manager):

    def get_queryset(self):
        qs = super().get_queryset()
        return qs.select_related('author')

class Book(models.Model):
    ...
    objects = BookManager()

Nota: la llamada a super debe cambiarse para python 2.x

Ahora todo lo que tenemos que usar en las vistas es

books = Book.objects.all()

y no se realizarán consultas adicionales en plantilla/vista.

Definir administradores personalizados #

Muy a menudo se trata de modelos que tienen algo así como un campo publicado. Este tipo de campos casi siempre se usan al recuperar objetos, por lo que se encontrará escribiendo algo como:

my_news = News.objects.filter(published=True)

demasiadas veces. Puede usar administradores personalizados para manejar estas situaciones, de modo que luego pueda escribir algo como:

my_news = News.objects.published()

que también es mejor y más fácil de leer para otros desarrolladores.

Cree un archivo managers.py en el directorio de su aplicación y defina una nueva clase models.Manager:

from django.db import models


class NewsManager(models.Manager):

    def published(self, **kwargs):
        # the method accepts **kwargs, so that it is possible to filter
        # published news
        # i.e: News.objects.published(insertion_date__gte=datetime.now)
        return self.filter(published=True, **kwargs)

usa esta clase redefiniendo la propiedad objects en la clase modelo:

from django.db import models

# import the created manager
from .managers import NewsManager

class News(models.Model):
    """ News model
    """
    insertion_date = models.DateTimeField('insertion date', auto_now_add=True)
    title = models.CharField('title', max_length=255)
    # some other fields here
    published = models.BooleanField('published')

    # assign the manager class to the objects property
    objects = NewsManager()

Ahora puede obtener sus noticias publicadas simplemente de esta manera:

my_news = News.objects.published()

y también puede realizar más filtrado:

my_news = News.objects.published(title__icontains='meow')
