Pdb también puede imprimir todas las variables existentes en el ámbito global o local, escribiendo globals() o locals() en el indicador (Pdb) respectivamente.

Uso de la barra de herramientas de depuración de Django #

Primero, necesita instalar django-debug-toolbar:

pip install django-debug-toolbar

configuraciones.py:

A continuación, inclúyalo en las aplicaciones instaladas del proyecto, pero tenga cuidado: siempre es una buena práctica usar un archivo settings.py diferente para aplicaciones y middleware solo de desarrollo como la barra de herramientas de depuración:

# If environment is dev...
DEBUG = True

INSTALLED_APPS += [
    'debug_toolbar',
]

MIDDLEWARE += ['debug_toolbar.middleware.DebugToolbarMiddleware']

La barra de herramientas de depuración también se basa en archivos estáticos, por lo que también se debe incluir la aplicación adecuada:

INSTALLED_APPS = [
    # ...
    'django.contrib.staticfiles',
    # ...
]

STATIC_URL = '/static/'

# If environment is dev...
DEBUG = True

INSTALLED_APPS += [
    'debug_toolbar',
]

En algunos casos, también es necesario establecer INTERNAL_IPS en settings.py:

INTERNAL_IPS = ('127.0.0.1', )

urls.py:

En urls.py, como sugiere la documentación oficial, el siguiente fragmento debería habilitar el enrutamiento de la barra de herramientas de depuración:

if settings.DEBUG and 'debug_toolbar' in settings.INSTALLED_APPS:
    import debug_toolbar
    urlpatterns += [
        url(r'^__debug__/', include(debug_toolbar.urls)),
    ]

Recopile la estática de la barra de herramientas después de la instalación:

python manage.py collectstatic

Eso es todo, la barra de herramientas de depuración aparecerá en las páginas de su proyecto, brindando información útil sobre el tiempo de ejecución, SQL, archivos estáticos, señales, etc.

HTML:

Además, django-debug-toolbar requiere un Content-type de las etiquetas text/html, <html> y <body> para mostrarse correctamente.

** En caso de que esté seguro de haber configurado todo correctamente, pero la barra de herramientas de depuración aún no se muestra: ** use [esta solución “nuclear”] 2 para tratar de resolverlo.

Uso del depurador de Python (Pdb) #

La herramienta de depuración de Django más básica es pdb, una parte de la biblioteca estándar de Python.

Script de vista de inicio

Examinemos un simple script views.py:

from django.http import HttpResponse


def index(request):
    foo = 1
    bar = 0

    bug = foo/bar

    return HttpResponse("%d goes here." % bug)

Comando de consola para ejecutar el servidor:

python manage.py runserver

Es obvio que Django arrojaría un ZeroDivisionError cuando intentas cargar la página de índice, pero si pretendemos que el error está muy profundo en el código, podría volverse realmente desagradable.

Establecer un punto de interrupción

Afortunadamente, podemos establecer un punto de interrupción para rastrear ese error:

from django.http import HttpResponse

# Pdb import
import pdb


def index(request):
    foo = 1
    bar = 0
    
    # This is our new breakpoint
    pdb.set_trace()
    
    bug = foo/bar
    
    return HttpResponse("%d goes here." % bug)

Comando de consola para ejecutar el servidor con pdb:

python -m pdb manage.py runserver

Ahora, en el punto de interrupción de carga de la página, se activará el indicador (Pdb) en el shell, que también bloqueará su navegador en estado pendiente.

Depuración con pdb shell

Es hora de depurar esa vista interactuando con el script a través del shell:

> ../views.py(12)index()
-> bug = foo/bar
# input 'foo/bar' expression to see division results:
(Pdb) foo/bar
*** ZeroDivisionError: division by zero
# input variables names to check their values:
(Pdb) foo
1
(Pdb) bar
0
# 'bar' is a source of the problem, so if we set it's value > 0...
(Pdb) bar = 1
(Pdb) foo/bar
1.0
# exception gone, ask pdb to continue execution by typing 'c':
(Pdb) c
[03/Aug/2016 10:50:45] "GET / HTTP/1.1" 200 111

En la última línea, vemos que nuestra vista devolvió una respuesta OK y se ejecutó como debería.

Para detener el bucle pdb, simplemente ingrese q en un shell.

Uso de “afirmar falso” #

Mientras desarrolla, inserte la siguiente línea en su código:

assert False, value

hará que django genere un AssertionError con el valor proporcionado como un mensaje de error cuando se ejecute esta línea.

Si esto ocurre en una vista, o en cualquier código llamado desde una vista, y se establece DEBUG=True, se mostrará en el navegador un seguimiento de pila completo y detallado con mucha información de depuración.

¡No olvides quitar la línea cuando hayas terminado!

Considere escribir más documentación, pruebas, registros y aserciones en lugar de usar un depurador #

La depuración requiere tiempo y esfuerzo.

En lugar de perseguir errores con un depurador, considere dedicar más tiempo a mejorar su código al:

  • Escribe y ejecuta Pruebas. Python y Django tienen excelentes marcos de prueba integrados, que se pueden usar para probar tu código mucho más rápido que manualmente con un depurador.
  • Escribir la documentación adecuada para sus funciones, clases y módulos. PEP 257 y Guía de estilo de Python de Google proporciona buenas prácticas para escribir buenas cadenas de documentación.
  • Use Logging para producir resultados de su programa, durante el desarrollo y después de la implementación.
  • Agregue aserciones a su código en lugares importantes: reduzca la ambigüedad, detecte los problemas a medida que se crean.

Bonificación: escriba doctests para combinar documentación y pruebas.
