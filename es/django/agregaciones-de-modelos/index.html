Agregaciones de modelos

Las agregaciones son métodos que permiten la ejecución de operaciones en (individuales y/o grupos de) filas de objetos derivados de un Modelo.

Promedio, Mínimo, Máximo, Suma de Queryset #

class Product(models.Model):
    name = models.CharField(max_length=20)
    price = models.FloatField()

Para obtener el precio promedio de todos los productos:

>>> from django.db.models import Avg, Max, Min, Sum
>>> Product.objects.all().aggregate(Avg('price'))
# {'price__avg': 124.0}

Para obtener el precio mínimo de todos los productos:

>>> Product.objects.all().aggregate(Min('price'))
# {'price__min': 9}

Para obtener el precio máximo de todos los productos:

>>> Product.objects.all().aggregate(Max('price'))
# {'price__max':599 }

Para obtener la SUMA de los precios de todos los productos:

>>> Product.objects.all().aggregate(Sum('price'))
# {'price__sum':92456 }

Cuente el número de relaciones exteriores #

class Category(models.Model):
    name = models.CharField(max_length=20)


class Product(models.Model):
    name = models.CharField(max_length=64)
    category = models.ForeignKey(Category, on_delete=models.PROTECT)

Para obtener el número de productos para cada categoría:

>>> categories = Category.objects.annotate(Count('product'))

Esto agrega el atributo <field_name>__count a cada instancia devuelta:

>>> categories.values_list('name', 'product__count')
[('Clothing', 42), ('Footwear', 12), ...]

Puede proporcionar un nombre personalizado para su atributo utilizando un argumento de palabra clave:

>>> categories = Category.objects.annotate(num_products=Count('product'))

Puede usar el campo anotado en conjuntos de consultas:

>>> categories.order_by('num_products')
[<Category: Footwear>, <Category: Clothing>]

>>> categories.filter(num_products__gt=20)
[<Category: Clothing>]

GROUB BY … COUNT/SUM Django ORM equivalente #

Podemos realizar consultas equivalentes de SQL GROUP BY ... COUNT o GROUP BY ... SUM en Django ORM, con el uso de annotate(), values(), order_by() y los métodos Count y Sum de django.db.models respectivamente:

Sea nuestro modelo:

   class Books(models.Model):
       title  = models.CharField()
       author = models.CharField()
       price = models.FloatField()

GRUPO POR... CONTEO:

  • Supongamos que queremos contar cuántos objetos de libro por autor distinto existen en nuestra tabla ‘Libros’:

     result = Books.objects.values('author')
                       .order_by('author')
                       .annotate(count=Count('author'))

  • Ahora resultado contiene un conjunto de consultas con dos columnas: autor y recuento:

       author    | count
 ------------|-------
  OneAuthor  |   5
 OtherAuthor |   2
    ...      |  ...

GRUPO POR... SUMA:

  • Supongamos que queremos sumar el precio de todos los libros por autor distinto que existen en nuestra tabla Books:

      result = Books.objects.values('author')
                        .order_by('author')
                        .annotate(total_price=Sum('price'))

  • Ahora resultado contiene un conjunto de consultas con dos columnas: autor y precio_total:

       author    | total_price
 ------------|-------------
  OneAuthor  |    100.35
 OtherAuthor |     50.00
     ...     |      ...
