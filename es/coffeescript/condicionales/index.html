si, si/entonces, si/si no, a menos que, operador ternario #

La instancia más básica de una construcción if evalúa una condición y ejecuta algún código de acuerdo con el resultado de la condición. Si la condición devuelve verdadero, se ejecuta el código dentro de la condicional.

counter = 10
if counter is 10
  console.log 'This will be executed!'

La construcción if se puede enriquecer con una declaración else. El código dentro de la instrucción else se ejecutará siempre que no se cumpla la condición if.

counter = 9
if counter is 10
  console.log 'This will not be executed...'
else
  console.log '... but this one will!'

Las construcciones if se pueden encadenar usando else, sin ninguna limitación sobre cuántos se pueden encadenar. El primer condicional que devuelve verdadero ejecutará su código y detendrá la comprobación: no se evaluará ningún condicional por debajo de ese punto a partir de entonces, y no se ejecutará ningún bloque de código dentro de esos condicionales.

if counter is 10
  console.log 'I counted to 10'
else if counter is 9
  console.log 'I counted to 9'
else if counter < 7
  console.log 'Not to 7 yet'
else
  console.log 'I lost count'

La forma opuesta de ‘si’ es ‘a menos que’. A diferencia de si, a menos que solo se ejecutará si el condicional devuelve falso.

counter = 10
unless counter is 10
  console.log 'This will not be executed!

Las declaraciones if se pueden colocar en una sola línea, pero en este caso, se requiere la palabra clave then.

if counter is 10 then console.log 'Counter is 10'

Una sintaxis alternativa es similar a Ruby:

console.log 'Counter is 10' if counter is 10

Los dos últimos bloques de código son equivalentes.

El operador ternario es una compresión de una construcción si/entonces/si no, y se puede utilizar al asignar valores a las variables. El valor final asignado a la variable será el definido después de entonces cuando se cumpla la condición si. De lo contrario, se asignará el valor después de else.

outcome = if counter is 10 then 'Done counting!' else 'Still counting'

Cambiar #

TL; DR: Las declaraciones switch de CoffeeScript usan when para cada caso y else para el caso predeterminado. Usan then para casos de una línea y comas para casos múltiples con un solo resultado. Deliberadamente no permiten fallas y, por lo tanto, no necesitan un ‘descanso’ explícito (ya que siempre está allí implícitamente). Una declaración de cambio se puede utilizar como una expresión asignable y retornable.

Las declaraciones switch de CoffeeScript son una especie de declaración de control que le permite realizar diferentes acciones en función de un valor. Son como declaraciones si, pero donde una declaración si por lo general toma una de dos acciones en función de si algo es verdadero o falso, las declaraciones cambiar toman una de cualquier cantidad de acciones dependiendo del valor de cualquier expresión: una cadena, un número o cualquier cosa.

CoffeeScript switch comienza con la palabra clave switch seguida de la expresión para encender. Luego, cada caso está representado por la palabra clave cuando seguida del valor para ese caso.

switch name
  when "Alice"
    # Code here will run when name is Alice
    callAlice()
  when "Bob"
    # Code here will run when name is Bob
    giveBobSandwich()

También hay una sintaxis abreviada para cuando cada caso es una línea, usando la palabra clave then en lugar de una nueva línea:

livesLeft = 2
switch livesLeft
  when 3 then fullHealth()
  when 2 then healthAt 2
  when 1 then healthAt 1
  when 0 then playerDie()

Puede mezclar y combinar los dos formatos según sea necesario:

livesLeft = 2
switch livesLeft
  when 3 then fullHealth()
  when 2 then healthAt 2
  when 1
    healthAt 1
    alert "Warning! Health low!"
  when 0 then playerDie()

Aunque las cosas más comunes para activar son una variable (como en el ejemplo anterior) o el resultado de una función, puede activar cualquier expresión que elija:

indexOfAnswer = 0
switch indexOfAnswer + 1
  when 1 then console.log "The answer is the 1st item"
  when 2 then console.log "The answer is the 2nd item"
  when 3 then console.log "The answer is the 3rd item"

También puede tener varios casos que conduzcan a la misma acción:

switch password
  when "password", "123456", "letmein" then console.log "Wrong!"
  when "openpoppyseed" then console.log "Close, but no cigar."
  when "opensesame" then console.log "You got it!"

Una característica muy útil es un caso predeterminado o comodín, que solo se ejecutará si no se cumple ninguno de los otros criterios. CoffeeScript significa esto con la palabra clave else:

switch password
  when "password", "123456", "letmein" then console.log "Wrong!"
  when "openpoppyseed" then console.log "Close, but no cigar."
  when "opensesame" then console.log "You got it!"
  else console.log "Not even close..."

(Tenga en cuenta que no necesita la palabra clave then para el caso else porque no hay condición).

¡Ahora aquí hay un ejemplo de todas las características de switch en acción!

switch day
  when "Mon" then go work
  when "Tue" then go relax
  when "Thu" then go iceFishing
  when "Fri", "Sat"
    if day is bingoDay
      go bingo
      go dancing
  when "Sun" then go church
  else go work

También puede hacer que la condición de un caso sea una expresión:

switch fullName
  when myFullName() then alert "Doppelgänger detected"
  when presidentFirstName + " " + presidentLastName
    alert "Get down Mr. president!"
    callSecretService()
  when "Joey Bonzo" then alert "Joey Bonzo everybody"

Las declaraciones switch de CoffeeScript también tienen un rasgo único: pueden devolver valores como una función. Si asigna una variable a una sentencia switch, se le asignará lo que devuelva la sentencia.

address = switch company
  when "Apple" then "One Infinite Loop"
  when "Google" then "1600 Amphitheatre Parkway"
  when "ACME"
    if isReal
      "31918 Hayman St"
    else
      "Unknown desert location"
  else lookUpAddress company

(Recuerde que la última declaración en un bloque se devuelve implícitamente. También puede usar la palabra clave return manualmente).

Las declaraciones de cambio también se pueden usar sin una expresión de control, convirtiéndolas en una alternativa más limpia a las cadenas if/else.

score = 76
grade = switch
  when score < 60 then 'F'
  when score < 70 then 'D'
  when score < 80 then 'C'
  when score < 90 then 'B'
  else 'A'

(Esto es funcionalmente equivalente a grade = switch true porque el primer caso que se evalúe como true coincidirá. Sin embargo, dado que cada caso implícitamente se rompe al final, solo se ejecutará el primer caso que coincida).
