Páginas de GitHub #

GitHub ofrece alojamiento ilimitado para usuarios u organizaciones y sitio del proyecto. Tanto Jekyll como archivos estáticos están disponibles.

Estos son los pasos para alojar su blog Jekyll en Github.

Configuración #

Sitio de usuarios u organizaciones #

  1. Cree un repositorio llamado nombre de usuario.github.io, donde nombre de usuario es su nombre de usuario (o el nombre de la organización) en GitHub.
  2. Clone el repositorio en su computadora: 
    $ git clone https://github.com/username/username.github.io
  3. Ingrese a la carpeta del proyecto, bootstrap, diseñe y depure su sitio: 
    $ cd username.github.io
$ bundle install
$ bundle exec jekyll serve
  4. Comprometer y enviar el repositorio: 
    $ git add --all
$ git commit -m "Initial commit"
$ git push -u origin master

Ahora deberías poder ir a username.github.io para ver tu blog.

Sitio del proyecto #

El sitio del proyecto se puede habilitar en todos los repositorios, incluidos los repositorios privados.

  1. Habilitar el sitio del proyecto.

    Go to Settings-GitHub Pages-Sources, choose a source to switch on GitHub Pages for the repository.

  2. Sitio de construcción

    You may build a Jekyll site from scratch or use Theme Chooser to find a theme for your project site.

  3. Editar contenido

  4. Comprometerse

Ahora debería poder ir a username.github.io/your-project para ver el sitio de su proyecto.

Dominios personalizados #

  1. Abra Configuración->Páginas de GitHub->Dominio personalizado, agregue su dominio personalizado.
  2. Cree un archivo CNAME: 
    $ cd username.github.io
$ echo "example.com" > CNAME
  3. Comprometerse y empujar 
    $ git commit -m "Add CNAME" CNAME
$ git push -u origin master

Restricciones #

Complementos #

Jekyll tiene un sistema de complementos con ganchos que le permiten crear contenido personalizado generado específico para su sitio. Sin embargo, las páginas de GitHub solo permiten una lista blanca de complementos por razones de seguridad.

Aquí está la lista blanca:

  • Mapa del sitio de Jekyll
  • Etiqueta Jekyll SEO
  • github-metadatos
  • Alimentación de Jekyll
  • Redirección de Jekyll desde
  • Yemojis
  • Menciones de Jekyll

Para evitar la inconsistencia con las páginas de GitHub, puede usar --safe para servir en local.

Todavía puede usar todos los complementos publicando su sitio generado en Páginas de GitHub, convirtiendo el sitio localmente y enviando los archivos estáticos generados a su repositorio de GitHub en lugar de los archivos fuente de Jekyll.

Motor de rebajas #

Desde el 05/01/2016, GitHub Pages solo admite kramdown como motor Markdown.

Consulte https://github.com/blog/2100-github-pages-now-faster-and-simpler-with-jekyll-3-0 para obtener más detalles.

Fuentes #

GitHub le permite establecer las fuentes de Jekyll en la rama maestra, la rama gh-pages o la carpeta /docs en la rama maestra.

Un tutorial completo está disponible en https://pages.github.com/

Máquina local #

Para fines de prueba, puede alojar su blog en su máquina local. Después de configurar y realizar cualquier cambio, Jekyll puede servir el blog en http://localhost:4000. En la línea de comando en el directorio raíz del proyecto, ejecute:

$ bundle exec jekyll serve

La parte bundle exec es opcional, pero si usa Bundler, garantiza que las dependencias de la gema estén actualizadas. Para un [bucle Edit-Build-Test] más rápido (http://www.joelonsoftware.com/articles/fog0000000023.html), use la opción --draft para crear artículos desde el directorio _drafts:

$ bundle exec jekyll serve --draft --detach

El uso de --detach pone el proceso en segundo plano para que el símbolo del sistema se pueda usar para otra cosa.

(A partir de la versión 2.4, la opción --watch está habilitada de forma predeterminada. Si, por casualidad, tiene una versión anterior, deberá agregar esa opción para que los cambios sean monitoreados).

También puede establecer el directorio de destino en un directorio en un servidor web diferente, como Apache, nginx o Lighttpd:

$ jekyll build --destination /path/to/web_server/root

Alojamiento CloudCannon (y CMS) #

CloudCannon ofrece alojamiento y un CMS para aplicaciones Jekyll. Estos son los pasos para alojar su aplicación Jekyll en CloudCannon (http://cloudcannon.com).

Sin control de versiones:

  • Crea tu blog localmente usando algunos archivos locales y jekyll serve
  • Cree una cuenta de CloudCannon y cree un nuevo sitio
  • Arrastre su sitio al ‘Explorador de archivos’ dentro de CloudCannon

Con control de versiones:

  • Crear un repositorio en Github o Bitbucket
  • Crea tu blog localmente usando algunos archivos locales y jekyll serve
  • Cree una cuenta de CloudCannon y cree un nuevo sitio
  • Conecta tu repositorio de Github o Bitbucket

Tenga en cuenta que CloudCannon no es completamente gratuito. En el plan gratuito puede usar CloudCannon como un CMS gráfico, pero necesitará un alojamiento externo. En el plan de pago también está incluido el hosting.
