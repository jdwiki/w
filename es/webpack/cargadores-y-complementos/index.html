Cargadores y complementos

Los cargadores y complementos constituyen los componentes básicos de Webpack.

Los cargadores generalmente se delegan a una sola tarea y tipo de archivo. Son más fáciles de configurar y generalmente requieren menos código repetitivo.

Los complementos, por otro lado, tienen acceso al sistema de compilación interno de Webpack a través de ganchos y, por lo tanto, son más poderosos. Los complementos pueden modificar el entorno de Webpack completamente configurado y pueden realizar acciones personalizadas durante todo el proceso de compilación.

Cuando se trata de nuestros archivos CSS, por ejemplo, se puede usar un cargador para agregar automáticamente prefijos de proveedores a las propiedades, mientras que se puede usar un complemento para producir una hoja de estilo minimizada en el proceso de creación del paquete.

Primeros pasos con los cargadores #

Para comenzar, npm install los cargadores deseados para su proyecto.

Dentro del objeto de configuración que se exporta en webpack.config.js, una propiedad module contendrá todos sus cargadores.

const source = `${__dirname}/client/src/`;

module.exports = {
  // other settings here
  
  module: {
    loaders: [
          {
            test: /\.jsx?$/,
            include: source,
            loaders: ['babel?presets[]=es2015,presets[]=react', 'eslint']
          },
          {
            test: /\.s?css$/,
            include: source,
            loaders: ['style', 'css', 'autoprefixer', 'sass']
          }
        ]
  },
};

En la configuración anterior, estamos usando tres configuraciones básicas para nuestros cargadores:

  • prueba: aquí es donde vinculamos cargadores a extensiones específicas usando RegExp. El primer conjunto de cargadores se está ejecutando en todos los archivos .js y .jsx. El segundo conjunto se ejecuta en todos los archivos .css y .scss.
  • incluir: Este es el directorio en el que queremos que se ejecuten nuestros cargadores. Opcionalmente, podríamos usar fácilmente la propiedad exclude para definir los directorios que no queremos incluir.
  • cargadores: Esta es una lista de todos los cargadores que queremos ejecutar en los archivos especificados en test e include.

Es importante tener en cuenta que los cargadores se ejecutan de derecha a izquierda en cada matriz de cargadores y de abajo hacia arriba en las definiciones individuales. El siguiente código ejecutará los cargadores en el siguiente orden: sass, autoprefixer, css, style.

loaders: [
  {
    test: /\.s?css$/,
    include: source,
    loaders: ['style', 'css', 'autoprefixer']
  },
  {
    test: /\.s?css$/,
    include: source,
    loaders: ['sass']
  }
]

Esta es una fuente común de confusión y errores para los desarrolladores que son nuevos en webpack. Por ejemplo, cuando usamos la sintaxis JSX y ES6, queremos eliminar ese código, no eliminar el código compilado que proporciona nuestro cargador de babel. Por lo tanto, nuestro cargador eslint se coloca a la derecha de nuestro cargador babel.

El sufijo -loader es opcional al enumerar nuestros cargadores.

loaders: ['sass']

… es equivalente a:

loaders: ['sass-loader']

Alternativamente, puede usar la propiedad loader (singular) junto con una cadena que separa la lista de cargadores por el carácter !.

loaders: ['style', 'css']

… es equivalente a:

loader: "style!css"

cargando archivos mecanografiados #

Para usar mecanografiado con el paquete web, necesita mecanografiado y ts-loader instalados

npm --save-dev install typescript ts-loader

Ahora puede configurar webpack para usar archivos mecanografiados

// webpack.config.js

module.exports = {
  ..
  resolve: {
    // .js is required for react imports.
    // .tsx is required for react tsx files.
    // .ts is optional, in case you will be importing any regular ts files.
    extensions: ['.js', '.ts', '.tsx']
  },
  module: {
    rules: [
      {
        // Set up ts-loader for .ts/.tsx files and exclude any imports from node_modules.
        test: /\.tsx?$/,
        loaders: isProduction ? ['ts-loader'] : ['react-hot-loader', 'ts-loader'],
        exclude: /node_modules/
      }
    ]
  },
  ...
};
