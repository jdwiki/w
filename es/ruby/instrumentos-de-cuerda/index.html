Sintaxis #

  • ‘Una cadena’ // crea una cadena a través de un literal entre comillas simples
  • “Una cadena” // crea una cadena mediante un literal entre comillas dobles
  • Cadena.nuevo(“Una cadena”)
  • %q(Una cadena) // sintaxis alternativa para crear cadenas entre comillas simples
  • %Q(A string) // sintaxis alternativa para crear cadenas entre comillas dobles

Diferencia entre literales de cadena entre comillas simples y dobles #

La principal diferencia es que los literales String entre comillas dobles admiten interpolaciones de cadenas y el conjunto completo de secuencias de escape.

Por ejemplo, pueden incluir expresiones Ruby arbitrarias mediante interpolación:

# Single-quoted strings don't support interpolation
puts 'Now is #{Time.now}'
# Now is #{Time.now}

# Double-quoted strings support interpolation
puts "Now is #{Time.now}"
# Now is 2016-07-21 12:43:04 +0200

Las cadenas entre comillas dobles también son compatibles con el conjunto completo de secuencias de escape, incluidos "\n", "\t"…

puts 'Hello\nWorld'
# Hello\nWorld

puts "Hello\nWorld"
# Hello
# World

… mientras que las cadenas entre comillas simples no admiten secuencias de escape no, dejando al descubierto el conjunto mínimo necesario para que las cadenas entre comillas simples sean útiles: comillas simples literales y barras invertidas, '\'' y '\\' respectivamente.

Crear una cadena #

Ruby ofrece varias formas de crear un objeto String. La forma más común es usar comillas simples o dobles para crear un “string literal”:

s1 = 'Hello'
s2 = "Hello"

La principal diferencia es que los literales de cadena entre comillas dobles son un poco más flexibles, ya que admiten la interpolación y algunas secuencias de escape de barra invertida.

También hay varias otras formas posibles de crear un literal de cadena usando delimitadores de cadena arbitrarios. Un delimitador de cadena arbitrario es un % seguido de un par de delimitadores coincidentes:

%(A string)
%{A string}
%<A string>
%|A string|
%!A string!

Finalmente, puedes usar la secuencia %q y %Q, que son equivalentes a ' y "":

puts %q(A string)
# A string
puts %q(Now is #{Time.now})
# Now is #{Time.now}

puts %Q(A string)
# A string
puts %Q(Now is #{Time.now})
# Now is 2016-07-21 12:47:45 +0200

Las secuencias %q y %Q son útiles cuando la cadena contiene comillas simples, comillas dobles o una combinación de ambas. De esta manera, no necesita escapar del contenido:

%Q(<a href="/profile">User's profile<a>)

Puede usar varios delimitadores diferentes, siempre que haya un par coincidente:

%q(A string)
%q{A string}
%q<A string>
%q|A string|
%q!A string!

Manipulación de casos #

"string".upcase     # => "STRING"
"STRING".downcase   # => "string"
"String".swapcase   # => "sTRING"
"string".capitalize # => "String"

Estos cuatro métodos no modifican el receptor original. Por ejemplo,

str = "Hello"
str.upcase # => "HELLO"
puts str   # => "Hello"

Hay cuatro métodos similares que realizan las mismas acciones pero modifican el receptor original.

"string".upcase!     # => "STRING"
"STRING".downcase!   # => "string"
"String".swapcase!   # => "sTRING"
"string".capitalize! # => "String"

Por ejemplo,

str = "Hello"
str.upcase!  # => "HELLO"
puts str     # => "HELLO"

Notas:

  • antes de Ruby 2.4, estos métodos no manejan Unicode.

Concatenación de cadenas #

Concatenar cadenas con el operador +:

s1 = "Hello"
s2 = " "
s3 = "World"

puts s1 + s2 + s3
# => Hello World

s = s1 + s2 + s3
puts s
# => Hello World

O con el operador <<:

s = 'Hello'
s << ' '
s << 'World'
puts s
# => Hello World

Tenga en cuenta que el operador << modifica el objeto en el lado izquierdo.

También puede multiplicar cadenas, p.

"wow" * 3
# => "wowwowwow"

Dividir una cadena #

String#split divide una String en un Array, en función de un delimitador.

"alpha,beta".split(",")
# => ["alpha", "beta"]

Un String vacío da como resultado un Array vacío:

"".split(",")
# => []

Un delimitador que no coincide da como resultado un Array que contiene un solo elemento:

"alpha,beta".split(".")
# => ["alpha,beta"]

También puede dividir una cadena usando expresiones regulares:

"alpha, beta,gamma".split(/, ?/)
# => ["alpha", "beta", "gamma"]

El delimitador es opcional, por defecto una cadena se divide en espacios en blanco:

"alpha beta".split
# => ["alpha", "beta"]

Cadenas de posicionamiento #

En Ruby, las cadenas pueden estar justificadas a la izquierda, a la derecha o centradas

Para justificar la cadena a la izquierda, use el método ljust. Esto toma dos parámetros, un número entero que representa el número de caracteres de la nueva cadena y una cadena que representa el patrón que se va a completar.

Si el número entero es mayor que la longitud de la cadena original, la nueva cadena se justificará a la izquierda y el parámetro de cadena opcional ocupará el espacio restante. Si no se proporciona el parámetro de cadena, la cadena se rellenará con espacios.

str ="abcd"
str.ljust(4)          => "abcd"
str.ljust(10)         => "abcd      "

Para justificar una cadena a la derecha, utilice el método rjust. Esto toma dos parámetros, un número entero que representa el número de caracteres de la nueva cadena y una cadena que representa el patrón que se va a completar.

Si el número entero es mayor que la longitud de la cadena original, la nueva cadena se justificará a la derecha y el parámetro de cadena opcional ocupará el espacio restante. Si no se proporciona el parámetro de cadena, la cadena se rellenará con espacios.

str = "abcd"
str.rjust(4)          => "abcd"
str.rjust(10)         => "      abcd"

Para centrar una cadena, use el método center. Esto toma dos parámetros, un número entero que representa el ancho de la nueva cadena y una cadena, con la que se rellenará la cadena original. La cuerda se alineará al centro.

str = "abcd"
str.center(4)          => "abcd"
str.center(10)         => "   abcd   "

Uniendo cadenas #

Array#join une un Array en un String, basado en un delimitador:

["alpha", "beta"].join(",")
# => "alpha,beta"

El delimitador es opcional y por defecto es una Cadena vacía.

["alpha", "beta"].join
# => "alphabeta"

Un ‘Array’ vacío da como resultado una ‘String’ vacía, sin importar qué delimitador se use.

[].join(",")
# => ""

La cadena comienza con #

Para encontrar si una cadena comienza con un patrón, el método start_with? es útil

str = "zebras are cool"
str.start_with?("zebras")      => true

También puede verificar la posición del patrón con index

str = "zebras are cool"
str.index("zebras").zero?      => true

Interpolación de cadenas #

El delimitador de comillas dobles " y la secuencia %Q admite la interpolación de cadenas usando #{ruby_expression}:

puts "Now is #{Time.now}"
# Now is Now is 2016-07-21 12:47:45 +0200

puts %Q(Now is #{Time.now})
# Now is Now is 2016-07-21 12:47:45 +0200

La cadena termina con #

Para encontrar si una cadena termina con un patrón, el método end_with? es útil

str = "I like pineapples"
str.end_with?("pineaaples")      => false

Cadenas multilínea #

La forma más fácil de crear una cadena de varias líneas es usar varias líneas entre comillas:

address = "Four score and seven years ago our fathers brought forth on this
continent, a new nation, conceived in Liberty, and dedicated to the
proposition that all men are created equal."

El principal problema con esa técnica es que si la cadena incluye una cita, romperá la sintaxis de la cadena. Para solucionar el problema, puede usar un heredoc en su lugar:

puts <<-RAVEN
  Once upon a midnight dreary, while I pondered, weak and weary, 
  Over many a quaint and curious volume of forgotten lore— 
      While I nodded, nearly napping, suddenly there came a tapping, 
  As of some one gently rapping, rapping at my chamber door. 
  "'Tis some visitor," I muttered, "tapping at my chamber door— 
              Only this and nothing more." 
  RAVEN

Ruby admite documentos estilo shell aquí con <<EOT, pero el texto final debe comenzar la línea. Eso arruina la sangría del código, por lo que no hay muchas razones para usar ese estilo. Desafortunadamente, la cadena tendrá sangrías dependiendo de cómo se sangra el código en sí.

Ruby 2.3 resuelve el problema introduciendo <<~ que elimina el exceso de espacios iniciales:

<!-- if version [gte 2.3] -->
def build_email(address)
  return (<<~EMAIL)
    TO: #{address}

    To Whom It May Concern:

    Please stop playing the bagpipes at sunrise!
                     
    Regards,
    Your neighbor               
  EMAIL
end
<!-- end version if -->

Percent Strings también funcionan para crear cadenas multilínea:

%q(
HAMLET        Do you see yonder cloud that's almost in shape of a camel?
POLONIUS        By the mass, and 'tis like a camel, indeed.
HAMLET        Methinks it is like a weasel.
POLONIUS        It is backed like a weasel.
HAMLET        Or like a whale?
POLONIUS        Very like a whale
)

Hay algunas formas de evitar la interpolación y las secuencias de escape:

  • Comillas simples en lugar de comillas dobles: '\n es un retorno de carro.'

  • Minúscula q en una cadena de porcentaje: %q[#{no-una-variable}]

  • Comilla simple la cadena terminal en un heredoc:

    <<-'CODE'
   puts 'Hello world!'
CODE

Cadenas formateadas #

Ruby puede inyectar una matriz de valores en una cadena reemplazando cualquier marcador de posición con los valores de la matriz proporcionada.

"Hello %s, my name is %s!" % ['World', 'br3nt']
# => Hello World, my name is br3nt!

Los marcadores de posición están representados por dos %s y los valores son proporcionados por la matriz ['Hello', 'br3nt']. El operador % le indica a la cadena que inyecte los valores de la matriz.

Reemplazos de caracteres de cadena #

El método tr devuelve una copia de una cadena donde los caracteres del primer argumento son reemplazados por los caracteres del segundo argumento.

"string".tr('r', 'l') # => "stling"

Para reemplazar solo la primera aparición de un patrón con otra expresión, use el método sub

"string ring".sub('r', 'l') # => "stling ring"

Si desea reemplazar todas las ocurrencias de un patrón con esa expresión, use gsub

"string ring".gsub('r','l') # => "stling ling"

Para eliminar caracteres, pase una cadena vacía para el segundo parámetro

También puede utilizar expresiones regulares en todos estos métodos.

Es importante tener en cuenta que estos métodos solo devolverán una nueva copia de una cadena y no modificarán la cadena en su lugar. Para hacer eso, necesitas usar los métodos tr!, sub! y gsub! respectivamente.

Comprender los datos en una cadena #

En Ruby, una cadena es simplemente una secuencia de bytes junto con el nombre de una codificación (como UTF-8, US-ASCII, ASCII-8BIT) que especifica cómo puede interpretar esos bytes como caracteres.

Las cadenas Ruby se pueden usar para contener texto (básicamente una secuencia de caracteres), en cuyo caso se usa generalmente la codificación UTF-8.

"abc".bytes  # => [97, 98, 99]
"abc".encoding.name  # => "UTF-8"

Las cadenas Ruby también se pueden usar para contener datos binarios (una secuencia de bytes), en cuyo caso se usa generalmente la codificación ASCII-8BIT.

[42].pack("i").encoding  # => "ASCII-8BIT"

Es posible que la secuencia de bytes en una cadena no coincida con la codificación, lo que genera errores si intenta usar la cadena.

"\xFF \xFF".valid_encoding? # => false
"\xFF \xFF".split(' ')      # ArgumentError: invalid byte sequence in UTF-8

Sustitución de cadenas #

p "This is %s" % "foo"
# => "This is foo"

p "%s %s %s" % ["foo", "bar", "baz"]
# => "foo bar baz"

p "%{foo} == %{foo}" % {:foo => "foo" }
# => "foo == foo"

Consulte String % docs y Kernel::sprintf para obtener más detalles.
