Comandos del sistema operativo o Shell

Hay muchas maneras de interactuar con el sistema operativo. Desde dentro de Ruby, puede ejecutar comandos o subprocesos de shell/sistema.

Ejecutivo: Exec tiene una funcionalidad muy limitada y, cuando se ejecuta, sale del programa Ruby y ejecuta el comando.

El Comando del Sistema: El comando del sistema se ejecuta en una subcapa en lugar de reemplazar el proceso actual y devuelve verdadero o nulo. El comando del sistema es, como los acentos graves, una operación de bloqueo en la que la aplicación principal espera hasta que se completa el resultado de la operación del sistema. Aquí, la operación principal nunca necesita preocuparse por capturar una excepción generada por el proceso secundario.

La salida de la función del sistema siempre será verdadera o nula dependiendo de si el script se ha ejecutado sin errores o no. Por lo tanto, cada error al ejecutar el script no se pasará a nuestra aplicación. La operación principal nunca necesita preocuparse por capturar una excepción generada por el proceso secundario. En este caso, la salida es nula porque el proceso secundario generó una excepción. Esta es una operación de bloqueo en la que el programa Ruby esperará hasta que se complete la operación del comando antes de continuar. La operación del sistema usa bifurcación para bifurcar el proceso actual y luego ejecutar la operación dada usando exec.

Los acentos graves (`): El carácter de acento grave generalmente se encuentra debajo de la tecla de escape en el teclado. Backticks se ejecuta en una subcapa en lugar de reemplazar el proceso actual y devuelve el resultado del comando. Aquí podemos obtener la salida del comando, pero el programa fallará cuando se genere una excepción. Si hay una excepción en el subproceso, esa excepción se otorga al proceso principal y el proceso principal podría terminar si no se maneja la excepción. Esta es una operación de bloqueo en la que el programa Ruby esperará hasta que se complete la operación del comando antes de continuar. La operación del sistema usa bifurcación para bifurcar el proceso actual y luego ejecutar la operación dada usando exec.

** IO.popen: ** IO.popen se ejecuta en un subproceso. Aquí, la entrada estándar y la salida estándar del subproceso están conectadas al objeto IO.

Abrir3: Popen3 le permite acceder a la entrada estándar, la salida estándar y el error estándar. La entrada y salida estándar del subproceso se devolverá a los objetos IO.

ps (igual que $CHILD_STATUS)** Se puede usar con las operaciones backticks, system() o %x{} y dará el estado del último comando ejecutado por el sistema. Esto puede ser útil para acceder a las propiedades exitstatus y pid.

$?.exitstatus

Formas recomendadas de ejecutar código shell en Ruby: #

Abrir3.popen3 o Abrir3.capturar3: Open3 en realidad solo usa el comando de generación de Ruby, pero le brinda una API mucho mejor.

Abrir3.abrir3

Popen3 se ejecuta en un subproceso y devuelve stdin, stdout, stderr y wait_thr.

require 'open3'
stdin, stdout, stderr, wait_thr = Open3.popen3("sleep 5s && ls")
puts "#{stdout.read} #{stderr.read} #{wait_thr.value.exitstatus}"

o

require 'open3'
cmd = 'git push heroku master'
Open3.popen3(cmd) do |stdin, stdout, stderr, wait_thr|
  puts "stdout is:" + stdout.read
  puts "stderr is:" + stderr.read
end

generará: la salida estándar es: stderr is:fatal: no es un repositorio de git (ni ninguno de los directorios principales): .git

o

require 'open3'
cmd = 'ping www.google.com'
Open3.popen3(cmd) do |stdin, stdout, stderr, wait_thr|
  while line = stdout.gets
    puts line
  end
end

generará:

Tiempos aproximados de ida y vuelta en milisegundos: Minimum = 10ms, Maximum = 29ms, Average = 19ms

Abrir3.capturar3:

require 'open3'

stdout, stderr, status = Open3.capture3('my_funky_command', 'and', 'some', 'argumants')
if status.success?
  # command completed successfully, do some more stuff
else
  raise "An error occured"
end

o

Open3.capture3('/some/binary with some args')

Sin embargo, no se recomienda debido a la sobrecarga adicional y al potencial de inyecciones de caparazón.

Si el comando se lee desde la entrada estándar y desea proporcionarle algunos datos:

Open3.capture3('my_funky_command', stdin_data: 'read from stdin')

Ejecute el comando con un directorio de trabajo diferente, usando chdir:

Open3.capture3('my_funky_command', chdir: '/some/directory')

Formas clásicas de ejecutar código shell en Ruby: #

Ejecutivo:

exec 'echo "hello world"'

o

exec ('echo "hello world"')

El Comando del Sistema:

system 'echo "hello world"'

Mostrará “hola mundo” en la ventana de comandos.

o

system ('echo "hello world"')

El comando del sistema puede devolver un verdadero si el comando fue exitoso o nulo cuando no lo fue.

result = system 'echo "hello world"'
puts result  # will return a true in the command window

Los acentos graves (`):

echo "hola mundo" Mostrará “hola mundo” en la ventana de comandos.

También puedes capturar el resultado.

result = `echo "hello world"`  
puts "We always code a " + result

# Will get and return the current date from the system
IO.popen("date") { |f| puts f.gets }
