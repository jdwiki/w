Gemas #

En este ejemplo, usaremos ’nokogiri’ como una gema de ejemplo. ’nokogiri’ puede ser reemplazado más tarde por cualquier otro nombre de gema.

Para trabajar con gemas usamos una herramienta de línea de comandos llamada gem seguida de una opción como instalar o actualizar y luego los nombres de las gemas que queremos instalar, pero eso no es todo.

Instalar gemas:

$> gem install nokogiri

Pero eso no es lo único que necesitamos. También podemos especificar versión, fuente desde la que instalar o buscar gemas. Comencemos con algunos casos de uso básicos (UC) y luego puede publicar una solicitud de actualización.

Listado de todas las gemas instaladas:

$> gem list

Desinstalar gemas:

$> gem uninstall nokogiri

Si tenemos más versiones de la gema nokogiri, se nos pedirá que especifiquemos cuál queremos desinstalar. Obtendremos una lista que está ordenada y numerada y solo escribimos el número.

Actualizando gemas

$> gem update nokogiri

o si queremos actualizarlos todos

$> gem update

Comman gem tiene muchos más usos y opciones para explorar. Para obtener más información, consulte la documentación oficial. Si algo no está claro, publique una solicitud y lo agregaré.

Instalación de Ruby macOS #

Entonces, la buena noticia es que Apple amablemente incluye un intérprete de Ruby. Desafortunadamente, tiende a no ser una versión reciente:

$ /usr/bin/ruby -v
ruby 2.0.0p648 (2015-12-16 revision 53162) [universal.x86_64-darwin16]

Si tiene Homebrew instalado, puede obtener la versión más reciente de Ruby con:

$ brew install ruby

$ /usr/local/bin/ruby -v
ruby 2.4.1p111 (2017-03-22 revision 58053) [x86_64-darwin16]

(Es probable que vea una versión más reciente si prueba esto).

Para recoger la versión elaborada sin usar la ruta completa, querrá agregar /usr/local/bin al comienzo de su variable de entorno $PATH:

export PATH=/usr/local/bin:$PATH

Agregar esa línea a ~/.bash_profile asegura que obtendrá esta versión después de reiniciar su sistema:

$ type ruby
ruby is /usr/local/bin/ruby

Homebrew instalará gem para [instalar Gems] (https://www.wikiod.com/es/ruby/instalacion#Gemas). También es posible [compilar desde la fuente](https://www.wikiod.com/es/ruby/instalacion#Linux - Compilando desde la fuente) si lo necesita. Homebrew también incluye esa opción:

$ brew install ruby --build-from-source

Linux - Compilando desde la fuente #

`De esta manera obtendrás el rubí más nuevo, pero tiene sus desventajas. Hacerlo como este rubí no será gestionado por ninguna aplicación.

!! Recuerda cambiar la versión para que corresponda con tu !!

  1. necesita descargar un tarball, busque un enlace en un sitio web oficial (https://www.ruby-lang.org/en/downloads/)

  2. Extraiga el tarball

  3. Instalar

    $> wget https://cache.ruby-lang.org/pub/ruby/2.3/ruby-2.3.3.tar.gz $> tar -xvzf ruby-2.3.3.tar.gz $> cd ruby-2.3.3 $> ./configure $> make $> sudo make install

Esto instalará Ruby en /usr/local. Si no está satisfecho con esta ubicación, puede pasar un argumento a ./configure --prefix=DIR donde DIR es el directorio en el que desea instalar Ruby.

Linux: instalación mediante un administrador de paquetes #

Probablemente la opción más fácil, pero cuidado, la versión no siempre es la más nueva. Simplemente abra la terminal y escriba (dependiendo de su distribución)

en Debian o Ubuntu usando apt

$> sudo apt install ruby

en CentOS, openSUSE o Fedora

$> sudo yum install ruby

Puede usar la opción -y para que no se le solicite que acepte la instalación, pero en mi opinión, es una buena práctica verificar siempre qué está tratando de instalar el administrador de paquetes.

Windows - Instalación usando el instalador #

Probablemente, la forma más fácil de configurar Ruby en Windows es ir a http://rubyinstaller.org/ y desde allí descargar un ejecutable que instalará.

No tiene que configurar casi nada, pero habrá una ventana importante. Tendrá una casilla de verificación que dice Agregue el ejecutable Ruby a su RUTA. Confirme que está marcado, si no lo hace, no podrá ejecutar Ruby y tendrá que configurar la variable PATH por su cuenta.

Luego solo sigue hasta que se instale y eso es todo.

Linux: solución de problemas de instalación de gemas #

Primera UC en el ejemplo Gemas $> gem install nokogiri puede tener problemas para instalar gemas porque no tenemos los permisos para ello. Esto se puede resolver en más de una sola forma.

Primera solución UC a:

Puedes usar sudo. Esto instalará la gema para todos los usuarios. Este método debería estar mal visto. Esto debe usarse solo con la gema que sabe que todos los usuarios podrán usar. Por lo general, en la vida real, no desea que algún usuario tenga acceso a sudo.

$> sudo gem install nokogiri

Primera solución UC b

Puede usar la opción --user-install que instala las gemas en la carpeta de gemas de sus usuarios (generalmente en ~/.gem)

&> gem install nokogiri --user-install

Primera solución UC c

Puede configurar GEM_HOME y GEM_PATH que luego harán que el comando gem install instale todas las gemas en una carpeta que especifique. Puedo darte un ejemplo de eso (la forma habitual)

  • En primer lugar, debe abrir .bashrc. Usa nano o tu editor de texto favorito.

    $> nano ~/.bashrc

  • Luego al final de este archivo escribe

    export GEM_HOME=$HOME/.gem export GEM_PATH=$HOME/.gem

  • Ahora deberá reiniciar la terminal o escribir . ~/.bashrc para volver a cargar la configuración. Esto le permitirá usar gem isntall nokogiri e instalará esas gemas en la carpeta que especificó.
