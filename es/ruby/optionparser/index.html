OptionParser se puede usar para analizar las opciones de la línea de comandos desde ARGV.

Opciones de línea de comando obligatorias y opcionales #

Es relativamente fácil analizar la línea de comando a mano si no está buscando nada demasiado complejo:

# Naive error checking
abort('Usage: ' + $0 + ' site id ...') unless ARGV.length >= 2

# First item (site) is mandatory
site = ARGV.shift

ARGV.each do | id |
  # Do something interesting with each of the ids
end

Pero cuando sus opciones comiencen a complicarse, probablemente necesitará usar un analizador de opciones como, bueno, OptionParser :

require 'optparse'

# The actual options will be stored in this hash
options = {}

# Set up the options you are looking for
optparse = OptionParser.new do |opts|
  opts.banner = "Usage: #{$0} -s NAME id ..."

  opts.on("-s", "--site NAME", "Site name") do |s|
    options[:site] = s
  end

  opts.on( '-h', '--help', 'Display this screen' ) do
    puts opts
    exit
  end
end

# The parse! method also removes any options it finds from ARGV.
optparse.parse!

También hay un parse no destructivo, pero es mucho menos útil si planea usar el resto de lo que hay en ARGV.

La clase OptionParser no tiene forma de aplicar argumentos obligatorios (como --site en este caso). Sin embargo, puede hacer su propia verificación después de ejecutar parse!:

# Slightly more sophisticated error checking
if options[:site].nil? or ARGV.length == 0
  abort(optparse.help)
end

Para obtener un controlador de opciones obligatorias más genérico, consulte [esta respuesta] (http://stackoverflow.com/a/2149183/7948068). En caso de que no quede claro, todas las opciones son opcionales a menos que haga todo lo posible para que sean obligatorias.

Valores predeterminados #

Con OptionsParser, es realmente fácil configurar valores predeterminados. Simplemente complete previamente el hash en el que almacena las opciones:

options = {
  :directory => ENV['HOME']
}

Cuando defina el analizador, sobrescribirá el valor predeterminado si un usuario proporciona un valor:

OptionParser.new do |opts|
  opts.on("-d", "--directory HOME", "Directory to use") do |d|
    options[:directory] = d
  end
end

Descripciones largas #

A veces, su descripción puede ser bastante larga. Por ejemplo, irb -h enumera un argumento que dice:

  --context-mode n  Set n[0-3] to method to create Binding Object,
                    when new workspace was created

No está claro de inmediato cómo respaldar esto. La mayoría de las soluciones requieren ajustes para que la sangría de la segunda línea y las siguientes se alineen con la primera. Afortunadamente, el método on admite múltiples líneas de descripción al agregarlas como argumentos separados:

  opts.on("--context-mode n",
          "Set n[0-3] to method to create Binding Object,",
          "when new workspace was created") do |n|
    optons[:context_mode] = n
  end

Puede agregar tantas líneas descriptivas como desee para explicar completamente la opción.
