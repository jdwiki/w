if, elsif, else y end #

Ruby ofrece las expresiones if y else esperadas para la lógica de bifurcación, terminadas por la palabra clave end:

# Simulate flipping a coin
result = [:heads, :tails].sample

if result == :heads
  puts 'The coin-toss came up "heads"'
else
  puts 'The coin-toss came up "tails"'
end

En Ruby, las sentencias if son expresiones que se evalúan como un valor, y el resultado se puede asignar a una variable:

status = if age < 18
           :minor
         else
           :adult
         end

Ruby también ofrece operadores ternarios de estilo C ([ver aquí para más detalles][1]) que se pueden expresar como:

some_statement ? if_true : if_false

Esto significa que el ejemplo anterior usando if-else también se puede escribir como

status = age < 18 ? :minor : :adult

Además, Ruby ofrece la palabra clave elsif que acepta una expresión para habilitar una lógica de bifurcación adicional:

label = if shirt_size == :s
          'small'
        elsif shirt_size == :m
          'medium'
        elsif shirt_size == :l
          'large'
        else
          'unknown size'
        end

Si ninguna de las condiciones en una cadena if/elsif es verdadera, y no hay ninguna cláusula else, entonces la expresión se evalúa como nula. Esto puede ser útil dentro de la interpolación de cadenas, ya que nil.to_s es la cadena vacía:

"user#{'s' if @users.size != 1}"

[1]: https://www.wikiod.com/es/ruby/flujo-de-control#operador ternario

Declaración del caso #

Ruby usa la palabra clave case para declaraciones de cambio.

Según [Ruby Docs] (http://ruby-doc.org/docs/keywords/1.9/Object.html#method-i-case):

Las declaraciones de caso consisten en una condición opcional, que está en el posición de un argumento para case, y cero o más cláusulas when. La primera cláusula when que coincida con la condición (o para evaluar Verdad booleana, si la condición es nula) “gana”, y su estrofa de código se ejecuta. El valor de la declaración de caso es el valor de la Cláusula exitosa when, o nil si no existe tal cláusula.

Una declaración de caso puede terminar con una cláusula else. Cada cuando un declaración puede tener múltiples valores candidatos, separados por comas.

Ejemplo:

case x
when 1,2,3
  puts "1, 2, or 3"
when 10
  puts "10"
else
  puts "Some other number"
end

Versión más corta:

case x
when 1,2,3 then puts "1, 2, or 3"
when 10 then puts "10"
else puts "Some other number"
end

El valor de la cláusula case se compara con cada cláusula when utilizando el método === (no ==). Por lo tanto, se puede utilizar con una variedad de diferentes tipos de objetos.

Se puede usar una declaración case con Rangos:

case 17
when 13..19
  puts "teenager"
end

Se puede usar una declaración case con Regexp:

case "google"
when /oo/
  puts "word contains oo"
end

Se puede usar una declaración case con un Proc o lambda:

case 44
when -> (n) { n.even? or n < 0 }
  puts "even or less than zero"
end

Se puede usar una declaración case con Classes:

case x
when Integer
  puts "It's an integer"
when String
  puts "It's a string"
end

Al implementar el método === puedes crear tus propias clases de coincidencia:

class Empty
  def self.===(object)
    !object or "" == object
  end
end

case ""
when Empty
  puts "name was empty"
else
  puts "name is not empty"
end

Se puede usar una declaración case sin un valor con el que comparar:

case
when ENV['A'] == 'Y'
  puts 'A'
when ENV['B'] == 'Y'
  puts 'B'
else
  puts 'Neither A nor B'
end

Una instrucción case tiene un valor, por lo que puede usarla como argumento de método o en una tarea:

description = case 16
              when 13..19 then "teenager"
              else ""
              end

Valores verdaderos y falsos #

En Ruby, hay exactamente dos valores que se consideran “falsos” y devolverán falso cuando se prueben como condición para una expresión si. Están:

  • ‘cero’
  • booleano falso

Todos los demás valores se consideran “veraces”, incluidos:

  • 0 - cero numérico (entero o de otro tipo)
  • "" - Cadenas vacías
  • "\n" - Cadenas que contienen solo espacios en blanco
  • [] - Matrices vacías
  • {} - hashes vacíos

Tomemos, por ejemplo, el siguiente código:

def check_truthy(var_name, var)
  is_truthy = var ? "truthy" : "falsy"
  puts "#{var_name} is #{is_truthy}"
end

check_truthy("false", false)
check_truthy("nil", nil)
check_truthy("0", 0)
check_truthy("empty string", "")
check_truthy("\\n", "\n")
check_truthy("empty array", [])
check_truthy("empty hash", {})

Saldrá:

false is falsy
nil is falsy
0 is truthy
empty string is truthy
\n is truthy
empty array is truthy
empty hash is truthy

En línea si/a menos que #

Un patrón común es usar un “if” o “unless” en línea o final:

puts "x is less than 5" if x < 5

Esto se conoce como un modificador condicional, y es una forma práctica de agregar un código de protección simple y retornos anticipados:

def save_to_file(data, filename)
  raise "no filename given" if filename.empty?
  return false unless data.valid?

  File.write(filename, data)
end

No es posible agregar una cláusula else a estos modificadores. Además, generalmente no se recomienda usar modificadores condicionales dentro de la lógica principal: para el código complejo, se deben usar if, elsif, else normales en su lugar.

mientras, hasta #

Un bucle while ejecuta el bloque mientras se cumple la condición dada:

i = 0
while i < 5
  puts "Iteration ##{i}"
  i +=1
end

Un bucle hasta ejecuta el bloque mientras el condicional es falso:

i = 0
until i == 5
  puts "Iteration ##{i}"
  i +=1
end

Operador de asignación O-Igual/Condicional (||=) #

Ruby tiene un operador or-equal que permite asignar un valor a una variable si y solo si esa variable se evalúa como nil o false.

 ||= # this is the operator that achieves this.

este operador con los tubos dobles que representan o y el signo igual que representa la asignación de un valor. Puedes pensar que representa algo como esto:

 x = x || y

este ejemplo anterior no es correcto. El operador or-equals en realidad representa esto:

 x || x = y

Si ‘x’ se evalúa como ’nil’ o ‘falso’, entonces a ‘x’ se le asigna el valor de ‘y’, y se deja sin cambios en caso contrario.

Aquí hay un caso de uso práctico del operador or-equals. Imagine que tiene una parte de su código que se espera que envíe un correo electrónico a un usuario. ¿Qué hace si por alguna razón no hay correo electrónico para este usuario? Podrías escribir algo como esto:

 if user_email.nil?
    user_email = "error@yourapp.com"
 end

Usando el operador or-equals podemos cortar todo este fragmento de código, proporcionando un control y una funcionalidad limpios y claros.

 user_email ||= "error@yourapp.com"

En los casos en que falso sea un valor válido, se debe tener cuidado de no anularlo accidentalmente:

has_been_run = false
has_been_run ||= true
#=> true

has_been_run = false
has_been_run = true if has_been_run.nil?
#=> false

Operador biestable #

El operador flip flop .. se usa entre dos condiciones en una declaración condicional:

(1..5).select do |e|
  e if (e == 2) .. (e == 4)
end
# => [2, 3, 4]

La condición se evalúa como “falsa” hasta que la primera parte se convierte en “verdadera”. Luego se evalúa como verdadero hasta que la segunda parte se convierte en verdadero. Después de eso, vuelve a cambiar a falso.

Este ejemplo ilustra lo que se está seleccionando:

[1, 2, 2, 3, 4, 4, 5].select do |e|
  e if (e == 2) .. (e == 4)
end
# => [2, 2, 3, 4]

El operador flip-flop solo funciona dentro de ifs (incluido a menos que) y el operador ternario. De lo contrario, se considera como el operador de rango.

(1..5).select do |e|
  (e == 2) .. (e == 4)
end
# => ArgumentError: bad value for range

Puede cambiar de falso a verdadero y al revés varias veces:

((1..5).to_a * 2).select do |e|
  e if (e == 2) .. (e == 4)
end
# => [2, 3, 4, 2, 3, 4]

a no ser que #

Una declaración común es si! (alguna condición). Ruby ofrece la alternativa de la declaración a menos que.

La estructura es exactamente la misma que una declaración ‘si’, excepto que la condición es negativa. Además, la declaración a menos que no admita elsif, pero sí admite else:

# Prints not inclusive
unless 'hellow'.include?('all')
  puts 'not inclusive'
end

lanzar, atrapar #

A diferencia de muchos otros lenguajes de programación, las palabras clave throw y catch no están relacionadas con el manejo de excepciones en Ruby.

En Ruby, throw y catch actúan un poco como etiquetas en otros idiomas. Se utilizan para cambiar el flujo de control, pero no están relacionados con un concepto de “error” como lo son las excepciones.

catch(:out) do
  catch(:nested) do
    puts "nested"
  end

  puts "before"
  throw :out
  puts "will not be executed"
end
puts "after"
# prints "nested", "before", "after"

Operador ternario #

Ruby tiene un operador ternario (?:), que devuelve uno de dos valores en función de si una condición se evalúa como verdadera:

conditional ? value_if_truthy : value_if_falsy

value = true
value ? "true" : "false"
#=> "true"

value = false
value ? "true" : "false"
#=> "false"

es lo mismo que escribir si a entonces b si no c final, aunque se prefiere el ternario

Ejemplos:

puts (if 1 then 2 else 3 end) # => 2

puts 1 ? 2 : 3                # => 2

x = if 1 then 2 else 3 end
puts x                        # => 2

Control de bucle con pausa, siguiente y rehacer #

El flujo de ejecución de un bloque de Ruby se puede controlar con las declaraciones break, next y redo.

romper #

La sentencia break saldrá del bloque inmediatamente. Cualquier instrucción restante en el bloque se omitirá y la iteración finalizará:

actions = %w(run jump swim exit macarena)
index = 0

while index < actions.length
  action = actions[index]

  break if action == "exit"

  index += 1
  puts "Currently doing this action: #{action}"
end

# Currently doing this action: run
# Currently doing this action: jump
# Currently doing this action: swim

siguiente #

La declaración siguiente regresará a la parte superior del bloque inmediatamente y continuará con la siguiente iteración. Cualquier instrucción restante en el bloque se omitirá:

actions = %w(run jump swim rest macarena)
index = 0

while index < actions.length
  action = actions[index]
  index += 1

  next if action == "rest"

  puts "Currently doing this action: #{action}"
end

# Currently doing this action: run
# Currently doing this action: jump
# Currently doing this action: swim
# Currently doing this action: macarena

rehacer #

La instrucción redo regresará a la parte superior del bloque inmediatamente y volverá a intentar la misma iteración. Cualquier instrucción restante en el bloque se omitirá:

actions = %w(run jump swim sleep macarena)
index = 0
repeat_count = 0

while index < actions.length
  action = actions[index]
  puts "Currently doing this action: #{action}"

  if action == "sleep"
    repeat_count += 1
    redo if repeat_count < 3
  end

  index += 1
end

# Currently doing this action: run
# Currently doing this action: jump
# Currently doing this action: swim
# Currently doing this action: sleep
# Currently doing this action: sleep
# Currently doing this action: sleep
# Currently doing this action: macarena

Iteración Enumerable #

Además de los bucles, estas instrucciones funcionan con métodos de iteración enumerables, como each y map:

[1, 2, 3].each do |item|
  next if item.even?
  puts "Item: #{item}"
end

# Item: 1
# Item: 3

Bloquear valores de resultado #

Tanto en las declaraciones break como en next, se puede proporcionar un valor y se utilizará como valor de resultado del bloque:

even_value = for value in [1, 2, 3]
  break value if value.even?
end

puts "The first even value is: #{even_value}"

# The first even value is: 2

return vs. next: retorno no local en un bloque #

Considere este fragmento roto:

def foo
  bar = [1, 2, 3, 4].map do |x|
    return 0 if x.even?
    x
  end
  puts 'baz'
  bar
end
foo # => 0

Uno podría esperar que return produzca un valor para la matriz de resultados de bloque de map. Entonces, el valor de retorno de foo sería [1, 0, 3, 0]. En cambio, return devuelve un valor del método foo. Tenga en cuenta que baz no se imprime, lo que significa que la ejecución nunca llegó a esa línea.

siguiente con un valor hace el truco. Actúa como un retorno a nivel de bloque.

def foo
  bar = [1, 2, 3, 4].map do |x|
    next 0 if x.even?
    x
  end
  puts 'baz'
  bar
end
foo # baz
    # => [1, 0, 3, 0]

En ausencia de un retorno, el valor devuelto por el bloque es el valor de su última expresión.

Flujo de control con sentencias lógicas #

Si bien puede parecer contradictorio, puede usar operadores lógicos para determinar si se ejecuta o no una instrucción. Por ejemplo:

File.exist?(filename) or STDERR.puts "#{filename} does not exist!"

Esto verificará si el archivo existe y solo imprimirá el mensaje de error si no existe. La declaración o es perezosa, lo que significa que dejará de ejecutarse una vez que esté seguro de si su valor es verdadero o falso. Tan pronto como se determina que el primer término es verdadero, no hay necesidad de verificar el valor del otro término. Pero si el primer término es falso, debe verificar el segundo término.

Un uso común es establecer un valor predeterminado:

glass = glass or 'full' # Optimist!

Eso establece el valor de ‘vaso’ en ’lleno’ si aún no está configurado. De manera más concisa, puede usar la versión simbólica de o:

glass ||= 'empty' # Pessimist.

También es posible ejecutar la segunda declaración solo si la primera es falsa:

File.exist?(filename) and puts "#{filename} found!"

Nuevamente, y es perezoso, por lo que solo ejecutará la segunda declaración si es necesario para llegar a un valor.

El operador ‘o’ tiene menor precedencia que ‘y’. Del mismo modo, || tiene menor precedencia que &&. Las formas de símbolos tienen mayor precedencia que las formas de palabras. Esto es útil para saber cuándo desea mezclar esta técnica con la asignación:

a = 1 and b = 2
#=> a==1
#=> b==2

a = 1 && b = 2; puts a, b
#=> a==2
#=> b==2

Tenga en cuenta que la Guía de estilo de Ruby [recomienda] (https://github.com/bbatsov/ruby-style-guide#no-and-or-or):

Las palabras clave y y o están prohibidas. La mínima legibilidad añadida simplemente no vale la alta probabilidad de introducir errores sutiles. Para expresiones booleanas, utilice siempre && y || en su lugar. Para el control de flujo, use si y a menos que; && y || también son aceptables pero menos claros.

comenzar, terminar #

El bloque begin es una estructura de control que agrupa varias sentencias.

begin
  a = 7
  b = 6
  a * b
end

Un bloque begin devolverá el valor de la última declaración en el bloque. El siguiente ejemplo devolverá 3.

begin
  1
  2
  3
end

El bloque begin es útil para la asignación condicional usando el operador ||= donde se pueden requerir varias declaraciones para devolver un resultado.

circumference ||=
  begin
    radius = 7
    tau = Math::PI * 2
    tau * radius
  end

También se puede combinar con otras estructuras de bloques como rescue, ensure, while, if, unless, etc. para proporcionar un mayor control del flujo del programa.

Los bloques Begin no son bloques de código, como { ... } o do ... end; no se pueden pasar a funciones.
