Literales para números, caracteres y cadenas

El término literal se usa comúnmente para describir una secuencia de caracteres en un código C que designa un valor constante como un número (por ejemplo, 0) o una cadena (por ejemplo, "C"). Estrictamente hablando, el estándar usa el término constante para constantes enteras, constantes flotantes, constantes de enumeración y constantes de caracteres, reservando el término ’literal’ para literales de cadena, pero este no es un uso común.

Los literales pueden tener prefijos o sufijos (pero no ambos), que son caracteres adicionales que pueden iniciar o finalizar un literal para cambiar su tipo predeterminado o su representación.

Literales de punto flotante #

Los literales de punto flotante se utilizan para representar números reales con signo. Los siguientes sufijos se pueden usar para especificar el tipo de un literal:

Sufijo Tipo Ejemplos
ninguno doble 3.1415926 -3E6
f, F flotador 3.1415926f `` 2.1E-6F`
l, L doble largo 3.1415926L 1E126L

Para usar estos sufijos, el literal debe ser un literal de coma flotante. Por ejemplo, 3f es un error, ya que 3 es un literal entero, mientras que 3.f o 3.0f son correctos. Para doble largo, la recomendación es usar siempre L mayúscula en aras de la legibilidad.

Literales enteros #

Los literales enteros se utilizan para proporcionar valores integrales. Se admiten tres bases numéricas, indicadas por prefijos:

bases Prefijo Ejemplo
decimales Ninguno 5
octales 0 0345
hexadecimal 0x o 0X 0x12AB, 0X12AB, 0x12ab, 0x12Ab

Tenga en cuenta que esta escritura no incluye ningún signo, por lo que los literales enteros siempre son positivos. Algo como -1 se trata como una expresión que tiene un literal entero (1) que se niega con un -

El tipo de un literal entero decimal es el primer tipo de datos que puede ajustarse al valor de int y long. Desde C99, long long también es compatible con literales muy grandes.

El tipo de un literal entero octal o hexadecimal es el primer tipo de datos que puede ajustarse al valor de int, unsigned, long y unsigned long. Desde C99, long long y unsigned long long también son compatibles con literales muy grandes.

Usando varios sufijos, se puede cambiar el tipo predeterminado de un literal.

Sufijo Explicación
L, l int largo
LL, ll (desde C99) int largo largo
U, u sin firmar

Los sufijos U y L/LL se pueden combinar en cualquier orden y caso. Es un error duplicar sufijos (por ejemplo, proporcionar dos sufijos ‘U’) incluso si tienen casos diferentes.

Cadenas literales #

Los literales de cadena se utilizan para especificar matrices de caracteres. Son secuencias de caracteres encerrados entre comillas dobles (por ejemplo, "abcd" y tienen el tipo char*).

El prefijo L hace que el literal sea una amplia matriz de caracteres, de tipo wchar_t*. Por ejemplo, L"abcd".

Desde C11, existen otros prefijos de codificación, similares a L:

prefijo tipo básico codificación
ninguno char dependiente de la plataforma
L wchar_t dependiente de la plataforma
u8 char UTF-8
u char16_t generalmente UTF-16
‘U’ char32_t generalmente UTF-32

Para los dos últimos, se puede consultar con macros de prueba de características si la codificación es efectivamente la codificación UTF correspondiente.

Literales de caracteres #

Los literales de caracteres son un tipo especial de literales enteros que se utilizan para representar un carácter. Están encerrados entre comillas simples, p. 'a' y tienen el tipo int. El valor del literal es un valor entero según el juego de caracteres de la máquina. No permiten sufijos.

El prefijo L antes de un carácter literal lo convierte en un carácter ancho de tipo wchar_t. Del mismo modo, dado que los prefijos u y U de C11 lo convierten en caracteres anchos de tipo char16_t y char32_t, respectivamente.

Cuando se intenta representar ciertos caracteres especiales, como un carácter que no se puede imprimir, se utilizan secuencias de escape. Las secuencias de escape utilizan una secuencia de caracteres que se traducen a otro carácter. Todas las secuencias de escape constan de dos o más caracteres, el primero de los cuales es una barra invertida \. Los caracteres que siguen inmediatamente a la barra invertida determinan qué carácter literal se interpreta como la secuencia.

Secuencia de escape Personaje representado
\b Retroceso
\f Avance de formulario
\n Salto de línea (nueva línea)
\r Retorno de carro
\t Pestaña horizontal
\v Pestaña vertical
\\ Barra invertida
\' Comillas simples
\" Comillas dobles
\? Signo de interrogación
\nnn Valor octal
\xnn… Valor hexadecimal
Secuencia de escape Personaje representado
\a Alerta (bip, campana)
Secuencia de escape Personaje representado
\unnnn Nombre de personaje universal
\Unnnnnnnn Nombre de personaje universal

Un nombre de carácter universal es un punto de código Unicode. Un nombre de carácter universal puede asignarse a más de un carácter. Los dígitos n se interpretan como dígitos hexadecimales. Dependiendo de la codificación UTF en uso, una secuencia de nombre de carácter universal puede dar como resultado un punto de código que consta de varios caracteres, en lugar de un único carácter char normal.

Cuando se utiliza la secuencia de escape de avance de línea en E/S de modo de texto, se convierte en el byte de nueva línea o secuencia de bytes específico del sistema operativo.

La secuencia de escape del signo de interrogación se usa para evitar trigraphs. Por ejemplo, ??/ se compila como el trígrafo que representa un carácter de barra invertida '\', pero usar ?\?/ daría como resultado cadena "??/" .

Puede haber uno, dos o tres números octales n en la secuencia de escape del valor octal.
