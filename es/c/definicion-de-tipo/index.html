El mecanismo typedef permite la creación de alias para otros tipos. No crea nuevos tipos. La gente a menudo usa typedef para mejorar la portabilidad del código, para dar alias a tipos de estructura o unión, o para crear alias para tipos de función (o puntero de función).

En el estándar C, typedef se clasifica como una ‘clase de almacenamiento’ por conveniencia; ocurre sintácticamente donde podrían aparecer clases de almacenamiento como static o extern.

Sintaxis #

  • typedef nombre_existente nombre_alias;

Desventajas de Typedef

typedef podría conducir a la contaminación del espacio de nombres en programas C grandes.

Desventajas de las estructuras Typedef

Además, las estructuras typedef’d sin un nombre de etiqueta son una de las principales causas de la imposición innecesaria de relaciones de orden entre los archivos de encabezado.

Considerar:

#ifndef FOO_H
#define FOO_H 1

#define FOO_DEF (0xDEADBABE)

struct bar; /* forward declaration, defined in bar.h*/

struct foo {
    struct bar *bar;
};

#endif

Con tal definición, sin usar typedefs, es posible que una unidad de compilación incluya foo.h para llegar a la definición FOO_DEF. Si no intenta desreferenciar el miembro bar de la estructura foo entonces no habrá necesidad de incluir el archivo bar.h.

Definir tipo frente a #definir

#define es una directiva de preprocesador de C que también se usa para definir los alias para varios tipos de datos similares a typedef pero con las siguientes diferencias:

  • typedef se limita a dar nombres simbólicos a tipos solo donde #define también se puede usar para definir alias para valores.

  • El compilador realiza la interpretación de typedef, mientras que el preprocesador procesa las sentencias #define.

  • Tenga en cuenta que #define cptr char * seguido de cptr a, b; no hace lo mismo que typedef char *cptr; seguido de cptr a, b;. Con #define, b es una variable simple char, pero también es un puntero con typedef.

Typedef para estructuras y uniones #

Puede dar nombres de alias a una estructura:

typedef struct Person {
    char name[32];
    int age;
} Person;

Person person;

En comparación con la forma tradicional de declarar estructuras, los programadores no necesitarían tener struct cada vez que declaran una instancia de esa estructura.

Tenga en cuenta que el nombre Person (a diferencia de struct Person) no se define hasta el último punto y coma. Por lo tanto, para las listas vinculadas y las estructuras de árbol que deben contener un puntero al mismo tipo de estructura, debe usar:

typedef struct Person {
    char name[32];
    int age;
    struct Person *next;
} Person;

o:

typedef struct Person Person;

struct Person {
    char name[32];
    int age;
    Person *next;
};

El uso de un typedef para un tipo union es muy similar.

typedef union Float Float;

union Float
{
    float f;
    char  b[sizeof(float)];
};

Se puede usar una estructura similar a esta para analizar los bytes que componen un valor “flotante”.

Usos simples de Typedef #

Para dar nombres cortos a un tipo de datos #

En vez de:

long long int foo;
struct mystructure object;

uno puede usar

/* write once */
typedef long long ll;
typedef struct mystructure mystruct;

/* use whenever needed */
ll foo;
mystruct object;

Esto reduce la cantidad de escritura necesaria si el tipo se usa muchas veces en el programa.

Mejorando la portabilidad #

Los atributos de los tipos de datos varían según las diferentes arquitecturas. Por ejemplo, un int puede ser un tipo de 2 bytes en una implementación y un tipo de 4 bytes en otra. Supongamos que un programa necesita usar un tipo de 4 bytes para ejecutarse correctamente.

En una implementación, permita que el tamaño de int sea de 2 bytes y el de long sea de 4 bytes. En otro, que el tamaño de int sea de 4 bytes y el de long de 8 bytes. Si el programa está escrito usando la segunda implementación,

/* program expecting a 4 byte integer */
int foo; /* need to hold 4 bytes to work */
/* some code involving many more ints */

Para que el programa se ejecute en la primera implementación, todas las declaraciones int deberán cambiarse a long.

/* program now needs long */
long foo; /*need to hold 4 bytes to work */
/* some code involving many more longs - lot to be changed */

Para evitar esto, se puede usar typedef

/* program expecting a 4 byte integer */
typedef int myint; /* need to declare once - only one line to modify if needed */
myint foo; /* need to hold 4 bytes to work */
/* some code involving many more myints */

Entonces, solo se necesitaría cambiar la declaración typedef cada vez, en lugar de examinar todo el programa.

El encabezado <stdint.h> y el encabezado relacionado <inttypes.h> definen nombres de tipo estándar (usando typedef) para números enteros de varios tamaños, y estos nombres suelen ser la mejor opción en código moderno que necesita arreglo números enteros de tamaño. Por ejemplo, uint8_t es un tipo entero de 8 bits sin signo; int64_t es un tipo entero de 64 bits con signo. El tipo uintptr_t es un tipo entero sin signo lo suficientemente grande como para contener cualquier puntero a objeto. Estos tipos son teóricamente opcionales, pero es raro que no estén disponibles. Hay variantes como uint_least16_t (el tipo de entero sin signo más pequeño con al menos 16 bits) e int_fast32_t (el tipo de entero con signo más rápido con al menos 32 bits). Además, intmax_t y uintmax_t son los tipos de enteros más grandes admitidos por la implementación. Estos tipos son obligatorios.

Para especificar un uso o mejorar la legibilidad #

Si un conjunto de datos tiene un propósito particular, se puede usar typedef para darle un nombre significativo. Además, si la propiedad de los datos cambia de tal manera que el tipo base debe cambiar, solo se tendría que cambiar la instrucción typedef, en lugar de examinar todo el programa.

Typedef para punteros de función #

Podemos usar typedef para simplificar el uso de punteros de función. Imagina que tenemos algunas funciones, todas con la misma firma, que usan su argumento para imprimir algo de diferentes maneras:

#include<stdio.h>

void print_to_n(int n)
{
    for (int i = 1; i <= n; ++i)
        printf("%d\n", i);
}

void print_n(int n)
{
    printf("%d\n, n);
}

Ahora podemos usar un typedef para crear un tipo de puntero de función con nombre llamado impresora:

typedef void (*printer_t)(int);

Esto crea un tipo, llamado printer_t para un puntero a una función que toma un solo argumento int y no devuelve nada, lo que coincide con la firma de las funciones que tenemos arriba. Para usarlo creamos una variable del tipo creado y le asignamos un puntero a una de las funciones en cuestión:

printer_t p = &print_to_n;
void (*p)(int) = &print_to_n; // This would be required without the type

Luego, para llamar a la función a la que apunta la variable de puntero de función:

p(5);           // Prints 1 2 3 4 5 on separate lines
(*p)(5);        // So does this

Por lo tanto, typedef permite una sintaxis más simple cuando se trata de punteros de función. Esto se vuelve más evidente cuando los punteros de función se usan en situaciones más complejas, como argumentos para funciones.

Si está utilizando una función que toma un puntero de función como parámetro sin un tipo de puntero de función definido, la definición de la función sería,

void foo (void (*printer)(int), int y){
    //code
    printer(y);
    //code
}

Sin embargo, con typedef es:

void foo (printer_t printer, int y){
    //code
    printer(y);
    //code
}

Del mismo modo, las funciones pueden devolver punteros de función y, de nuevo, el uso de un typedef puede simplificar la sintaxis al hacerlo.

Un ejemplo clásico es la función señal de <señal.h>. La declaración para ello (del estándar C) es:

void (*signal(int sig, void (*func)(int)))(int);

Esa es una función que toma dos argumentos: un int y un puntero a una función que toma un int como argumento y no devuelve nada, y que devuelve un puntero para funcionar como su segundo argumento.

Si definimos un tipo SigCatcher como un alias para el tipo de puntero a función:

typedef void (*SigCatcher)(int);

entonces podríamos declarar signal() usando:

SigCatcher signal(int sig, SigCatcher func);

En general, esto es más fácil de entender (aunque el estándar C no optó por definir un tipo para hacer el trabajo). La función signal toma dos argumentos, un int y un SigCatcher, y devuelve un SigCatcher, donde SigCatcher es un puntero a una función que toma un argumento int y no devuelve nada.

Aunque el uso de nombres typedef para apuntar a tipos de funciones hace la vida más fácil, también puede generar confusión para otras personas que mantendrán su código más adelante, así que utilícelo con precaución y con la documentación adecuada.
