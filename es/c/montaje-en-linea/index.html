El ensamblaje en línea es la práctica de agregar instrucciones de ensamblaje en medio del código fuente C. Ningún estándar ISO C requiere soporte de ensamblaje en línea. Dado que no es necesario, la sintaxis para el ensamblado en línea varía de un compilador a otro. Aunque normalmente se admite, hay muy pocas razones para usar el ensamblaje en línea y muchas razones para no hacerlo.

Ventajas #

  1. Rendimiento Al escribir las instrucciones de ensamblaje específicas para una operación, puede lograr un mejor rendimiento que el código ensamblador generado por el compilador. Tenga en cuenta que estas ganancias de rendimiento son raras. En la mayoría de los casos, puede lograr mejores ganancias de rendimiento simplemente reorganizando su código C para que el optimizador pueda hacer su trabajo.
  2. Interfaz de hardware El controlador del dispositivo o el código de inicio del procesador pueden necesitar algún código ensamblador para acceder a los registros correctos y garantizar que ciertas operaciones se realicen en un orden específico con un retraso específico entre operaciones.

Contras #

  1. Portabilidad del compilador No se garantiza que la sintaxis para el ensamblado en línea sea la misma de un compilador a otro. Si está escribiendo código con ensamblado en línea que debería ser compatible con diferentes compiladores, use macros de preprocesador (#ifdef) para verificar qué compilador se está usando. Luego, escriba una sección de ensamblado en línea separada para cada compilador compatible.
  2. Portabilidad del procesador No puede escribir ensamblado en línea para un procesador x86 y esperar que funcione en un procesador ARM. El ensamblaje en línea está diseñado para escribirse para un procesador o una familia de procesadores específicos. Si tiene un ensamblado en línea que desea admitir en diferentes procesadores, use macros de preprocesador para verificar para qué procesador se está compilando el código y seleccionar la sección de código de ensamblado adecuada.
  3. Cambios futuros en el rendimiento El ensamblaje en línea se puede escribir esperando retrasos en función de una determinada velocidad de reloj del procesador. Si el programa se compila para un procesador con un reloj más rápido, es posible que el código ensamblador no funcione como se esperaba.

gcc Soporte básico de asm #

El soporte de ensamblaje básico con gcc tiene la siguiente sintaxis:

asm [ volatile ] ( AssemblerInstructions )

donde AssemblerInstructions es el código ensamblador directo para el procesador dado. La palabra clave volatile es opcional y no tiene ningún efecto, ya que gcc no optimiza el código dentro de una instrucción asm básica. AssemblerInstructions puede contener múltiples instrucciones de ensamblaje. Se usa una instrucción asm básica si tiene una rutina asm que debe existir fuera de una función C. El siguiente ejemplo es del manual de GCC:

 /* Note that this code will not compile with -masm=intel */
 #define DebugBreak() asm("int $3")

En este ejemplo, podría usar DebugBreak() en otros lugares de su código y ejecutará la instrucción de ensamblaje int $3. Tenga en cuenta que aunque gcc no modificará ningún código en una instrucción asm básica, el optimizador aún puede mover declaraciones asm consecutivas. Si tiene varias instrucciones de ensamblaje que deben ocurrir en un orden específico, inclúyalas en una instrucción asm.

gcc Soporte asm extendido #

El soporte extendido de asm en gcc tiene la siguiente sintaxis:

asm [volatile] ( AssemblerTemplate
                  : OutputOperands
                  [ : InputOperands
                  [ : Clobbers ] ])
 
 asm [volatile] goto ( AssemblerTemplate
                       :
                       : InputOperands
                       : Clobbers
                       : GotoLabels)

donde AssemblerTemplate es la plantilla para la instrucción del ensamblador, OutputOperands son las variables C que pueden ser modificadas por el código ensamblador, InputOperands son las variables C utilizadas como parámetros de entrada, Clobbers son una lista o registros que son modificado por el código ensamblador, y GotoLabels son cualquier etiqueta de instrucción goto que se puede usar en el código ensamblador.

El formato extendido se usa dentro de las funciones de C y es el uso más típico del ensamblado en línea. A continuación se muestra un ejemplo del kernel de Linux para el intercambio de bytes de números de 16 y 32 bits para un procesador ARM:

/* From arch/arm/include/asm/swab.h in Linux kernel version 4.6.4 */
#if __LINUX_ARM_ARCH__ >= 6

static inline __attribute_const__ __u32 __arch_swahb32(__u32 x)
{
    __asm__ ("rev16 %0, %1" : "=r" (x) : "r" (x));
    return x;
}
#define __arch_swahb32 __arch_swahb32
#define __arch_swab16(x) ((__u16)__arch_swahb32(x))

static inline __attribute_const__ __u32 __arch_swab32(__u32 x)
{
    __asm__ ("rev %0, %1" : "=r" (x) : "r" (x));
    return x;
}
#define __arch_swab32 __arch_swab32

#endif

Cada sección de asm usa la variable x como su parámetro de entrada y salida. La función C luego devuelve el resultado manipulado.

Con el formato asm extendido, gcc puede optimizar las instrucciones de ensamblaje en un bloque asm siguiendo las mismas reglas que usa para optimizar el código C. Si desea que su sección de asm permanezca intacta, use la palabra clave volátil para la sección de asm.

gcc Ensamblaje en línea en macros #

Podemos poner instrucciones de montaje dentro de una macro y use la macro como si llamara a una función.

#define mov(x,y) \
{ \
    __asm__ ("l.cmov %0,%1,%2" : "=r" (x) : "r" (y), "r" (0x0000000F)); \
}

/// some definition and assignment
unsigned char sbox[size][size];
unsigned char sbox[size][size];

///Using
mov(state[0][1], sbox[si][sj]);

El uso de instrucciones de ensamblaje en línea integradas en el código C puede mejorar el tiempo de ejecución de un programa Esto es muy útil en situaciones críticas de tiempo como algoritmos criptográficos como AES. Por ejemplo, para una operación de cambio simple que se necesita en el algoritmo AES, podemos sustituir una instrucción de ensamblaje directa Girar a la derecha con el operador de cambio C >>.

En una implementación de ‘AES256’, en la función ‘AddRoundKey()’ tenemos algunas declaraciones como esta:

unsigned int w;          // 32-bit
unsigned char subkey[4]; // 8-bit, 4*8 = 32 

subkey[0] = w >> 24;     // hold 8 bit, MSB, leftmost group of 8-bits 
subkey[1] = w >> 16;     // hold 8 bit, second group of 8-bit from left    
subkey[2] = w >> 8;      // hold 8 bit, second group of 8-bit from right
subkey[3] = w;           // hold 8 bit, LSB, rightmost group of 8-bits

/// subkey <- w

Simplemente asignan el valor de bit de w a la matriz subkey.

Podemos cambiar tres turnos + asignar y una expresión C de asignación con solo una operación de montaje Girar a la derecha.

__asm__ ("l.ror  %0,%1,%2" : "=r" (* (unsigned int *) subkey)  : "r" (w), "r" (0x10));

El resultado final es exactamente el mismo.
