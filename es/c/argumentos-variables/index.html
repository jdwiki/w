Argumentos variables

Las funciones de la familia printf (printf, fprintf, etc.) y otras utilizan argumentos variables para permitir que se llame a una función con un número diferente de argumentos cada vez, de ahí el nombre varargs.

Para implementar funciones usando la característica de argumentos variables, use #include <stdarg.h>.

Para llamar a funciones que toman un número variable de argumentos, asegúrese de que haya un prototipo completo con los puntos suspensivos finales en el alcance: void err_exit(const char *format, ...); por ejemplo.

Sintaxis #

  • void va_start(va_list ap, last); /* Iniciar el procesamiento de argumentos variádicos; last es el último parámetro de función antes de los puntos suspensivos ("…") */
  • type va_arg(va_list ap, type); /* Obtener el siguiente argumento variádico de la lista; asegúrese de pasar el tipo promoted correcto */
  • vacío va_end(va_list ap); /* Fin del procesamiento del argumento */
  • void va_copy(va_list dst, va_list src); /* C99 o posterior: copia la lista de argumentos, es decir, la posición actual en el procesamiento de argumentos, en otra lista (por ejemplo, para pasar los argumentos varias veces) */

Parámetros #

Parámetro Detalles
va_list ap puntero de argumento, posición actual en la lista de argumentos variados
último nombre del último argumento de la función no variable, para que el compilador encuentre el lugar correcto para comenzar a procesar argumentos variables; no se puede declarar como una variable de registro, una función o un tipo de matriz
tipo Tipo promocionado del argumento variádico a leer (por ejemplo, int para un argumento short int)
va_list origen puntero de argumento actual para copiar
va_list dst nueva lista de argumentos a rellenar

Las funciones va_start, va_arg, va_end y va_copy son en realidad macros.

Asegúrese de siempre llamar a va_start primero, y solo una vez, y de llamar a va_end en último lugar, y solo una vez, y en cada punto de salida de la función. No hacerlo puede funcionar en su sistema, pero seguramente no es portátil y, por lo tanto, invita a errores.

Tenga cuidado de declarar su función correctamente, es decir, con un prototipo, y tenga en cuenta las restricciones en el argumento no variable last (no registro, no es una función o un tipo de matriz). No es posible declarar una función que solo tome argumentos variádicos, ya que se necesita al menos un argumento no variádico para poder iniciar el procesamiento de argumentos.

Al llamar a va_arg, debe solicitar el tipo de argumento promocionado, es decir:

  • short se promociona a int (y unsigned short también se promociona a int a menos que sizeof(unsigned short) == sizeof(int), en cuyo caso se promociona a unsigned int) .
  • flotante se promociona a doble.
  • char signado se promociona a int; unsigned char también se promociona a int a menos que sizeof(unsigned char) == sizeof(int), que rara vez es el caso.
  • char generalmente se promociona a int.
  • Los tipos C99 como uint8_t o int16_t se promocionan de manera similar.

El procesamiento de argumentos variádicos históricos (es decir, K&R) se declara en <varargs.h>, pero no debe usarse porque está obsoleto. El procesamiento de argumentos variádicos estándar (el que se describe aquí y se declara en <stdarg.h>) se introdujo en C89; la macro va_copy se introdujo en C99 pero la proporcionaron muchos compiladores antes de eso.

Usar un argumento de conteo explícito para determinar la longitud de va_list #

Con cualquier función variádica, la función debe saber cómo interpretar la lista de argumentos variables. Con las funciones printf() o scanf(), la cadena de formato le dice a la función qué esperar.

La técnica más simple es pasar un recuento explícito de los otros argumentos (que normalmente son todos del mismo tipo). Esto se demuestra en la función variádica del siguiente código, que calcula la suma de una serie de enteros, donde puede haber cualquier número de enteros, pero ese recuento se especifica como argumento antes de la lista de argumentos variables.

#include <stdio.h>
#include <stdarg.h>

/* first arg is the number of following int args to sum. */
int sum(int n, ...) {
    int sum = 0;
    va_list it; /* hold information about the variadic argument list. */

    va_start(it, n); /* start variadic argument processing */
    while (n--)
      sum += va_arg(it, int); /* get and sum the next variadic argument */
    va_end(it); /* end variadic argument processing */

    return sum;
}

int main(void)
{
    printf("%d\n", sum(5, 1, 2, 3, 4, 5)); /* prints 15 */
    printf("%d\n", sum(10, 5, 9, 2, 5, 111, 6666, 42, 1, 43, -6218)); /* prints 666 */
    return 0;
}

Usar valores de terminación para determinar el final de va_list #

Con cualquier función variádica, la función debe saber cómo interpretar la lista de argumentos variables. El enfoque “tradicional” (ejemplificado por printf) es especificar el número de argumentos por adelantado. Sin embargo, esto no siempre es una buena idea:

/* First argument specifies the number of parameters; the remainder are also int */
extern int sum(int n, ...);

/* But it's far from obvious from the code. */
sum(5, 2, 1, 4, 3, 6)

/* What happens if i.e. one argument is removed later on? */
sum(5, 2, 1, 3, 6)  /* Disaster */

A veces es más robusto agregar un terminador explícito, ejemplificado por POSIX execlp() función. Aquí hay otra función para calcular la suma de una serie de números dobles:

#include <stdarg.h>
#include <stdio.h>
#include <math.h>

/* Sums args up until the terminator NAN */
double sum (double x, ...) {
  double sum = 0;
  va_list va;

  va_start(va, x);
  for (; !isnan(x); x = va_arg(va, double)) {
    sum += x;
  }
  va_end(va);

  return sum;
}

int main (void) {
  printf("%g\n", sum(5., 2., 1., 4., 3., 6., NAN));
  printf("%g\n", sum(1, 0.5, 0.25, 0.125, 0.0625, 0.03125, NAN));
}

Buenos valores de terminación:

  • entero (se supone que todos son positivos o no negativos) — 0 o -1
  • tipos de punto flotante — NAN
  • tipos de punteros — NULL
  • tipos de enumeradores: algún valor especial

Implementando funciones con una interfaz similar a printf() #

Un uso común de las listas de argumentos de longitud variable es implementar funciones que son una envoltura delgada alrededor de la familia de funciones printf(). Un ejemplo de ello es un conjunto de funciones de notificación de errores.

errmsg.h

#ifndef ERRMSG_H_INCLUDED
#define ERRMSG_H_INCLUDED

#include <stdarg.h>
#include <stdnoreturn.h>    // C11

void verrmsg(int errnum, const char *fmt, va_list ap);
noreturn void errmsg(int exitcode, int errnum, const char *fmt, ...);
void warnmsg(int errnum, const char *fmt, ...);

#endif

Este es un ejemplo básico; dichos paquetes pueden ser muy elaborados. Normalmente, los programadores usarán errmsg() o warnmsg(), que a su vez usan verrmsg() internamente. Sin embargo, si a alguien se le ocurre la necesidad de hacer más, entonces la función verrmsg() expuesta será útil. Podrías evitar exponerlo hasta que lo necesites (YAGNI — no lo vas a necesitar), pero la necesidad surgirá eventualmente ( lo vas a necesitar — YAGNI).

errmsg.c

Este código solo necesita reenviar los argumentos variados a la función vfprintf() para generar un error estándar. También informa el mensaje de error del sistema correspondiente al número de error del sistema (errno) pasado a las funciones.

#include "errmsg.h"
#include <stdio.h>
#include <stdlib.h>
#include <string.h>

void
verrmsg(int errnum, const char *fmt, va_list ap)
{
    if (fmt)
        vfprintf(stderr, fmt, ap);
    if (errnum != 0)
        fprintf(stderr, ": %s", strerror(errnum));
    putc('\n', stderr);
}

void
errmsg(int exitcode, int errnum, const char *fmt, ...)
{
    va_list ap;
    va_start(ap, fmt);
    verrmsg(errnum, fmt, ap);
    va_end(ap);
    exit(exitcode);
}

void
warnmsg(int errnum, const char *fmt, ...)
{
    va_list ap;
    va_start(ap, fmt);
    verrmsg(errnum, fmt, ap);
    va_end(ap);
}

Usando errmsg.h

Ahora puede usar esas funciones de la siguiente manera:

#include "errmsg.h"
#include <errno.h>
#include <fcntl.h>
#include <stdio.h>
#include <stdlib.h>
#include <unistd.h>

int main(int argc, char **argv)
{
    char buffer[BUFSIZ];
    int fd;
    if (argc != 2)
    {
        fprintf(stderr, "Usage: %s filename\n", argv[0]);
        exit(EXIT_FAILURE);
    }
    const char *filename = argv[1];

    if ((fd = open(filename, O_RDONLY)) == -1)
        errmsg(EXIT_FAILURE, errno, "cannot open %s", filename);
    if (read(fd, buffer, sizeof(buffer)) != sizeof(buffer))
        errmsg(EXIT_FAILURE, errno, "cannot read %zu bytes from %s", sizeof(buffer), filename);
    if (close(fd) == -1)
        warnmsg(errno, "cannot close %s", filename);
    /* continue the program */
    return 0;
}

Si el abrir() o leer() las llamadas al sistema fallan, el error se escribe como error estándar y el El programa sale con el código de salida 1. Si el cerrar() la llamada al sistema falla, el error simplemente se imprime como un mensaje de advertencia y el programa continúa.

Comprobando el uso correcto de los formatos printf()

Si está utilizando GCC (el compilador GNU C, que es parte de GNU Compiler Collection), o usando Clang, entonces puede tener el compilador verifique que los argumentos que pasa a las funciones de mensajes de error coincidan lo que printf() espera. Dado que no todos los compiladores admiten la extensión, debe compilarse condicionalmente, lo cual es un poco complicado. Sin embargo, la protección que brinda vale la pena el esfuerzo.

Primero, necesitamos saber cómo detectar que el compilador es GCC o Clang emulando a GCC. La respuesta es que GCC define __GNUC__ para indicar eso.

Ver función común atributos para obtener información sobre los atributos, específicamente el formato atributo.

Reescrito errmsg.h

#ifndef ERRMSG_H_INCLUDED
#define ERRMSG_H_INCLUDED

#include <stdarg.h>
#include <stdnoreturn.h>    // C11

#if !defined(PRINTFLIKE)
#if defined(__GNUC__)
#define PRINTFLIKE(n,m) __attribute__((format(printf,n,m)))
#else
#define PRINTFLIKE(n,m) /* If only */
#endif /* __GNUC__ */
#endif /* PRINTFLIKE */

void verrmsg(int errnum, const char *fmt, va_list ap);
void noreturn errmsg(int exitcode, int errnum, const char *fmt, ...)
        PRINTFLIKE(3, 4);
void warnmsg(int errnum, const char *fmt, ...)
        PRINTFLIKE(2, 3);

#endif

Ahora, si cometes un error como:

errmsg(EXIT_FAILURE, errno, "Failed to open file '%d' for reading", filename);

(donde %d debería ser %s), entonces el compilador se quejará:

$ gcc -O3 -g -std=c11 -Wall -Wextra -Werror -Wmissing-prototypes -Wstrict-prototypes \
>     -Wold-style-definition -c erruse.c
erruse.c: In function ‘main’:
erruse.c:20:64: error: format ‘%d’ expects argument of type ‘int’, but argument 4 has type ‘const char *’ [-Werror=format=]
         errmsg(EXIT_FAILURE, errno, "Failed to open file '%d' for reading", filename);
                                                           ~^
                                                           %s
cc1: all warnings being treated as errors
$

Usar una cadena de formato #

El uso de una cadena de formato proporciona información sobre el número esperado y el tipo de los argumentos variádicos posteriores de tal manera que se evita la necesidad de un argumento de recuento explícito o un valor de terminación.

El siguiente ejemplo muestra una función que envuelve la función printf() estándar, y solo permite el uso de argumentos variados del tipo char, int y double (en formato de coma flotante decimal). Aquí, como con printf(), el primer argumento de la función de ajuste es la cadena de formato. A medida que se analiza la cadena de formato, la función puede determinar si se espera otro argumento variable y cuál debería ser su tipo.

#include <stdio.h>
#include <stdarg.h>

int simple_printf(const char *format, ...)
{
    va_list ap; /* hold information about the variadic argument list. */
    int printed = 0; /* count of printed characters */

    va_start(ap, format); /* start variadic argument processing */

    while (*format != '\0') /* read format string until string terminator */
    {
        int f = 0;

        if (*format == '%')
        {
            ++format;
            switch(*format)
            {
                case 'c' :
                    f = printf("%d", va_arg(ap, int)); /* print next variadic argument, note type promotion from char to int */
                    break;
                case 'd' :
                    f = printf("%d", va_arg(ap, int)); /* print next variadic argument */
                    break;

                case 'f' :
                    f = printf("%f", va_arg(ap, double)); /* print next variadic argument */
                    break;
                default :
                    f = -1; /* invalid format specifier */
                    break;
            }
        }
        else
        {
            f = printf("%c", *format); /* print any other characters */
        }

        if (f < 0) /* check for errors */
        {
            printed = f;
            break;
        }
        else
        {
            printed += f;
        }
        ++format; /* move on to next character in string */
    }

    va_end(ap); /* end variadic argument processing */

    return printed;
}

int main (int argc, char *argv[])
{
    int x = 40;
    int y = 0;

    y = simple_printf("There are %d characters in this sentence", x);
    simple_printf("\n%d were printed\n", y);
}
