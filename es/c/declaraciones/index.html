La declaración de identificador que se refiere a un objeto o función a menudo se denomina simplemente una declaración de objeto o función.

Llamar a una función desde otro archivo C #

foo.h

#ifndef FOO_DOT_H    /* This is an "include guard" */
#define FOO_DOT_H    /* prevents the file from being included twice. */
                     /* Including a header file twice causes all kinds */
                     /* of interesting problems.*/

/**
 * This is a function declaration.
 * It tells the compiler that the function exists somewhere.
 */
void foo(int id, char *name);

#endif /* FOO_DOT_H */

foo.c

#include "foo.h"    /* Always include the header file that declares something
                     * in the C file that defines it. This makes sure that the
                     * declaration and definition are always in-sync.  Put this
                     * header first in foo.c to ensure the header is self-contained.
                     */
#include <stdio.h>
                       
/**
 * This is the function definition.
 * It is the actual body of the function which was declared elsewhere.
 */
void foo(int id, char *name)
{
    fprintf(stderr, "foo(%d, \"%s\");\n", id, name);
    /* This will print how foo was called to stderr - standard error.
     * e.g., foo(42, "Hi!") will print `foo(42, "Hi!")`
     */
}

C Principal

#include "foo.h"

int main(void)
{
    foo(42, "bar");
    return 0;
}

Compilar y vincular

Primero, compilamos tanto foo.c como main.c en archivos objeto. Aquí usamos el compilador gcc, su compilador puede tener un nombre diferente y necesitar otras opciones.

$ gcc -Wall -c foo.c
$ gcc -Wall -c main.c

Ahora los vinculamos para producir nuestro ejecutable final:

$ gcc -o testprogram foo.o main.o

Uso de una variable global #

Introducción #

Ejemplo de declaraciones son:

int a; /* declaring single identifier of type int */

La declaración anterior declara un identificador único llamado a que se refiere a algún objeto con tipo int.

int a1, b1; /* declaring 2 identifiers of type int */

La segunda declaración declara 2 identificadores llamados a1 y b1 que se refieren a algunos otros objetos aunque con el mismo tipo int.

Básicamente, la forma en que esto funciona es así: primero coloca algo de tipo, luego escribe una o varias expresiones separadas por comas (,) (que no se evaluarán en este punto, y que de otro modo deberían denominarse declarantes en este contexto). Al escribir tales expresiones, se le permite aplicar solo los operadores de direccionamiento indirecto (*), llamada de función (( )) o subíndice (o indexación de matriz - [ ]) en algún identificador (tampoco puede usar ningún operadores en absoluto). No es necesario que el identificador utilizado sea visible en el ámbito actual. Algunos ejemplos:

/* 1 */ int /* 2 */ (*z) /* 3 */ , /* 4 */ *x , /* 5 */ **c /* 6 */ ;
# Descripción
1 El nombre del tipo entero.
2 Expresión no evaluada que aplica direccionamiento indirecto a algún identificador z.
3 Tenemos una coma que indica que seguirá una expresión más en la misma declaración.
4 Expresión no evaluada que aplica direccionamiento indirecto a algún otro identificador x.
5 Expresión no evaluada que aplica indirección al valor de la expresión (*c).
6 Fin de la declaración.

Tenga en cuenta que ninguno de los identificadores anteriores era visible antes de esta declaración y, por lo tanto, las expresiones utilizadas no serían válidas antes de ella.

Después de cada una de estas expresiones, el identificador utilizado en ella se introduce en el ámbito actual. (Si el identificador tiene asignado un vínculo, también se puede volver a declarar con el mismo tipo de vínculo para que ambos identificadores se refieran al mismo objeto o función)

Además, el signo del operador igual (=) se puede utilizar para la inicialización. Si una expresión no evaluada (declarador) va seguida de = dentro de la declaración, decimos que el identificador que se está introduciendo también se está inicializando. Después del signo = podemos poner una vez más alguna expresión, pero esta vez será evaluada y su valor será usado como inicial para el objeto declarado.

Ejemplos:

int l = 90; /* the same as: */

int l; l = 90; /* if it the declaration of l was in block scope */

int c = 2, b[c]; /* ok, equivalent to: */

int c = 2; int b[c];

Más adelante en su código, puede escribir exactamente la misma expresión de la parte de la declaración del identificador recién introducido, brindándole un objeto del tipo especificado al comienzo de la declaración, suponiendo que haya asignado valores válidos a todos los accesos. objetos en el camino. Ejemplos:

void f()
{
    int b2; /* you should be able to write later in your code b2 
            which will directly refer to the integer object
            that b2 identifies */
    
    b2 = 2; /* assign a value to b2 */
    
    printf("%d", b2); /*ok - should print 2*/

    int *b3; /* you should be able to write later in your code *b3 */

    b3 = &b2; /* assign valid pointer value to b3 */

    printf("%d", *b3); /* ok - should print 2 */

    int **b4; /* you should be able to write later in your code **b4 */

    b4 = &b3;

    printf("%d", **b4); /* ok - should print 2 */

    void (*p)(); /* you should be able to write later in your code (*p)() */

    p = &f; /* assign a valid pointer value */

    (*p)(); /* ok - calls function f by retrieving the
            pointer value inside p -    p
            and dereferencing it -      *p
            resulting in a function
            which is then called -      (*p)() -

            it is not *p() because else first the () operator is 
            applied to p and then the resulting void object is
            dereferenced which is not what we want here */
}

La declaración de b3 especifica que potencialmente puede usar el valor b3 como un medio para acceder a algún objeto entero.

Por supuesto, para aplicar el direccionamiento indirecto (*) a b3, también debe tener un valor adecuado almacenado en él (consulte punteros para obtener más información). También debe almacenar primero algún valor en un objeto antes de intentar recuperarlo (puede ver más sobre este problema aquí). Hemos hecho todo esto en los ejemplos anteriores.

int a3(); /* you should be able to call a3 */

Este le dice al compilador que intentarás llamar a a3. En este caso a3 se refiere a una función en lugar de un objeto. Una diferencia entre objeto y función es que las funciones siempre tendrán algún tipo de vínculo. Ejemplos:

void f1()
{
    {
        int f2(); /* 1 refers to some function f2 */
    }
    
    {
        int f2(); /* refers to the exact same function f2 as (1) */
    }
}

En el ejemplo anterior, las 2 declaraciones se refieren a la misma función f2, mientras que si estuvieran declarando objetos, en este contexto (con 2 ámbitos de bloque diferentes), serían 2 objetos distintos.

int (*a3)(); /* you should be able to apply indirection to `a3` and then call it */

Ahora puede parecer que se está complicando, pero si conoce la precedencia de los operadores, no tendrá problemas para leer la declaración anterior. Los paréntesis son necesarios porque el operador * tiene menos precedencia que el ().

En el caso de usar el operador de subíndice, la expresión resultante no sería realmente válida después de la declaración porque el índice usado en ella (el valor dentro de [ y ]) siempre será 1 por encima del valor máximo permitido para esto. objeto/función.

int a4[5]; /* here a4 shouldn't be accessed using the index 5 later on */

Pero debería ser accesible para todos los demás índices inferiores a 5. Ejemplos:

a4[0], a4[1]; a4[4];

a4[5] resultará en UB. Se puede encontrar más información sobre matrices [aquí] 4.

int (*a5)[5](); /* here a4 could be applied indirection
                indexed up to (but not including) 5
                and called */

Desafortunadamente para nosotros, aunque sintácticamente posible, la declaración de a5 está prohibida por el estándar actual.

Uso de constantes globales #

Typedef #

Typedefs son declaraciones que tienen la palabra clave typedef delante y antes del tipo. P.ej.:

typedef int (*(*t0)())[5];

(técnicamente puedes poner el typedef después del tipo también - como este int typedef (*(*t0)())[5]; pero esto no se recomienda)

Las declaraciones anteriores declaran un identificador para un nombre typedef. Puedes usarlo así después:

t0 pf;

Lo que tendrá el mismo efecto que escribir:

int (*(*pf)())[5];

Como puede ver, el nombre typedef “guarda” la declaración como un tipo para usar más tarde para otras declaraciones. De esta manera puede ahorrar algunas pulsaciones de teclas. Además, como la declaración que usa typedef sigue siendo una declaración, no está limitado solo por el ejemplo anterior:

t0 (*pf1);

Es lo mismo que:

int (*(**pf1)())[5];

Uso de la regla derecha-izquierda o espiral para descifrar la declaración C #

La regla “derecha-izquierda” es una regla completamente regular para descifrar declaraciones C. También puede ser útil para crearlos.

Lea los símbolos a medida que los encuentre en la declaración…

*   as "pointer to"          - always on the left side
[]  as "array of"            - always on the right side
()  as "function returning"  - always on the right side

Cómo aplicar la regla

PASO 1

Encuentra el identificador. Este es su punto de partida. Entonces dígase a sí mismo, “identificador es”. Has comenzado tu declaración.

PASO 2

Mire los símbolos a la derecha del identificador. Si, por ejemplo, encuentra () allí, entonces sabrá que esta es la declaración de una función. Entonces tendría * “el identificador es la función que regresa” *. O si encuentra un [] allí, diría “identifier is array of”. Continúe a la derecha hasta que se quede sin símbolos O presione un paréntesis derecho ). (Si presiona un paréntesis izquierdo (, ese es el comienzo de un símbolo (), incluso si hay cosas entre paréntesis. Más sobre eso a continuación).

PASO 3

Mire los símbolos a la izquierda del identificador. Si no es uno de nuestros símbolos anteriores (por ejemplo, algo como “int”), simplemente dígalo. De lo contrario, tradúzcalo al inglés usando la tabla de arriba. Continúe hacia la izquierda hasta que se quede sin símbolos O presione un paréntesis izquierdo (.

Ahora repita los pasos 2 y 3 hasta que haya formado su declaración.

Aquí hay unos ejemplos:

int *p[];

Primero, encuentra el identificador:

int *p[];
     ^

“p es”

Ahora, muévete a la derecha hasta que salgan los símbolos o el paréntesis derecho.

int *p[];
      ^^

“p es una matriz de”

Ya no se puede mover a la derecha (sin símbolos), así que muévase a la izquierda y busque:

int *p[];
    ^

“p es una matriz de apuntador a”

Sigue hacia la izquierda y encuentra:

int *p[];
^^^

“p es una matriz de puntero a int”.

(o “p es una matriz donde cada elemento es de tipo puntero a int”)

Otro ejemplo:

int *(*func())();

Encuentra el identificador.

int *(*func())();
       ^^^^

“función es”

Mover a la derecha.

int *(*func())();
           ^^

“func es una función que regresa”

Ya no se puede mover a la derecha debido al paréntesis derecho, así que muévase a la izquierda.

int *(*func())();
      ^

“func es una función que devuelve el puntero a”

Ya no se puede mover hacia la izquierda debido al paréntesis izquierdo, así que sigue hacia la derecha.

int *(*func())();
              ^^

“func es la función que devuelve el puntero a la función que devuelve”

Ya no puedo moverme a la derecha porque nos quedamos sin símbolos, así que ve a la izquierda.

int *(*func())();
    ^

“func es una función que devuelve el puntero a la función que devuelve el puntero a”

Y por último, sigue hacia la izquierda, porque a la derecha no queda nada.

int *(*func())();
^^^

“func es una función que devuelve el puntero a la función que devuelve el puntero a int”.

Como puede ver, esta regla puede ser bastante útil. También puede usarlo para verificar su cordura mientras crea declaraciones y para darle una pista sobre dónde colocar el siguiente símbolo y si se requieren paréntesis.

Algunas declaraciones parecen mucho más complicadas de lo que son debido al tamaño de las matrices y las listas de argumentos en forma de prototipo. Si ve [3], se lee como “matriz (tamaño 3) de…”. Si ve (char *,int) eso se lee como *“función esperando (char ,int) y regresando…”.

Aquí hay uno divertido:

int (*(*fun_one)(char *,double))[9][20];

No pasaré por cada uno de los pasos para descifrar este.

*“fun_one es un puntero a la función que espera (char ,doble) y devuelve un puntero a la matriz (tamaño 9) de la matriz (tamaño 20) de int.”

Como puede ver, no es tan complicado si se deshace de los tamaños de matriz y listas de argumentos:

int (*(*fun_one)())[][];

Puede descifrarlo de esa manera y luego colocar los tamaños de matriz y las listas de argumentos más adelante.

Algunas palabras finales:

Es bastante posible hacer declaraciones ilegales utilizando esta regla, por lo que es necesario tener cierto conocimiento de lo que es legal en C. Por ejemplo, si lo anterior hubiera sido:

int *((*fun_one)())[][];

habría leído “fun_one es puntero a función que devuelve matriz de matriz de puntero a int”. Dado que una función no puede devolver una matriz, sino solo un puntero a una matriz, esa declaración es ilegal.

Las combinaciones ilegales incluyen:

[]() - cannot have an array of functions
()() - cannot have a function that returns a function
()[] - cannot have a function that returns an array

En todos los casos anteriores, necesitaría un conjunto de paréntesis para vincular un símbolo * a la izquierda entre estos símbolos () y [] del lado derecho para que la declaración sea legal.
