En C, una cadena no es un tipo intrínseco. Una cadena C es la convención para tener una matriz unidimensional de caracteres que termina con un carácter nulo, con un '\0'.

Esto significa que una cadena C con un contenido de "abc" tendrá cuatro caracteres 'a', 'b', 'c' y '\0'.

Consulte el ejemplo de introducción básica a las cadenas.

1: https://www.wikiod.com/es/c/instrumentos-de-cuerda#Introducción básica a las cuerdas

Sintaxis #

  • char str1[] = “¡Hola mundo!”; /* Modificable */
  • char str2[14] = “¡Hola mundo!”; /* Modificable */
  • char* str3 = “¡Hola, mundo!”; /* No modificable*/

Tokenización: strtok(), strtok_r() y strtok_s() #

La función strtok divide una cadena en cadenas más pequeñas, o tokens, utilizando un conjunto de delimitadores.

#include <stdio.h>
#include <string.h>

int main(void)
{
    int toknum = 0;
    char src[] = "Hello,, world!";
    const char delimiters[] = ", !";
    char *token = strtok(src, delimiters);
    while (token != NULL)
    {
        printf("%d: [%s]\n", ++toknum, token);
        token = strtok(NULL, delimiters);
    }
    /* source is now "Hello\0, world\0\0" */
}

Producción:

1: [Hello]
2: [world]

La cadena de delimitadores puede contener uno o más delimitadores y se pueden usar diferentes cadenas de delimitadores con cada llamada a strtok.

Las llamadas a strtok para continuar tokenizando la misma cadena de origen no deben pasar la cadena de origen nuevamente, sino pasar NULL como primer argumento. Si se pasa la misma cadena de origen, el primer token se volverá a tokenizar. Es decir, dados los mismos delimitadores, strtok simplemente devolvería el primer token nuevamente.

Tenga en cuenta que como strtok no asigna nueva memoria para los tokens, modifica la cadena de origen. Es decir, en el ejemplo anterior, la cadena src se manipulará para producir los tokens a los que hace referencia el puntero devuelto por las llamadas a strtok. Esto significa que la cadena de origen no puede ser const (por lo que no puede ser una cadena literal). También significa que se pierde la identidad del byte delimitador (es decir, en el ejemplo, “,” y “!” se eliminan efectivamente de la cadena de origen y no se puede saber qué carácter delimitador coincide).

Tenga en cuenta también que varios delimitadores consecutivos en la cadena de origen se tratan como uno solo; en el ejemplo, se ignora la segunda coma.

strtok no es seguro para subprocesos ni reentrante porque utiliza un búfer estático durante el análisis. Esto significa que si una función llama a strtok, ninguna función que llame mientras está usando strtok también puede usar strtok, y no puede ser llamada por ninguna función que esté usando strtok.

Un ejemplo que demuestra los problemas causados ​​por el hecho de que strtok no es reentrante es el siguiente:

char src[] = "1.2,3.5,4.2";
char *first = strtok(src, ","); 

do 
{
    char *part;
    /* Nested calls to strtok do not work as desired */
    printf("[%s]\n", first);
    part = strtok(first, ".");
    while (part != NULL)
    {
        printf(" [%s]\n", part);
        part = strtok(NULL, ".");
    }
} while ((first = strtok(NULL, ",")) != NULL);

Producción:

[1.2]
 [1]
 [2]

La operación esperada es que el ciclo externo do while debe crear tres tokens que consisten en cada cadena de números decimales ("1.2", "3.5", "4.2"), para cada uno de los cuales strtok las llamadas al bucle interno deben dividirlo en cadenas de dígitos separadas ("1", "2", "3", "5", "4", "2").

Sin embargo, debido a que strtok no es reentrante, esto no ocurre. En cambio, el primer strtok crea correctamente el token “1.2\0”, y el ciclo interno crea correctamente los tokens "1" y "2". Pero luego, strtok en el ciclo externo está al final de la cadena utilizada por el ciclo interno, y devuelve NULL inmediatamente. Las subcadenas segunda y tercera de la matriz src no se analizan en absoluto.

Las bibliotecas C estándar no contienen una versión reentrante o segura para subprocesos, pero algunas otras sí, como POSIX’ strtok_r. Tenga en cuenta que en MSVC el equivalente strtok, strtok_s es seguro para subprocesos.

C11 tiene una parte opcional, el Anexo K, que ofrece una versión reentrante y segura para subprocesos llamada strtok_s. Puede probar la característica con __STDC_LIB_EXT1__. Esta parte opcional no es ampliamente compatible.

La función strtok_s se diferencia de la función strtok_r de POSIX al protegerse contra el almacenamiento fuera de la cadena que se tokeniza y al verificar las restricciones de tiempo de ejecución. Sin embargo, en programas escritos correctamente, strtok_s y strtok_r se comportan de la misma manera.

Usar strtok_s con el ejemplo ahora produce la respuesta correcta, así:

/* you have to announce that you want to use Annex K */ 
#define __STDC_WANT_LIB_EXT1__ 1
#include <string.h>

#ifndef __STDC_LIB_EXT1__
# error "we need strtok_s from Annex K"
#endif

char src[] = "1.2,3.5,4.2";  
char *next = NULL;
char *first = strtok_s(src, ",", &next);

do 
{
    char *part;
    char *posn;

    printf("[%s]\n", first);
    part = strtok_s(first, ".", &posn);
    while (part != NULL)
    {
        printf(" [%s]\n", part);
        part = strtok_s(NULL, ".", &posn);
    }
} 
while ((first = strtok_s(NULL, ",", &next)) != NULL);

Y la salida será:

[1.2]
 [1]
 [2]
[3.5]
 [3]
 [5]
[4.2]
 [4]
 [2]

Cadenas literales #

Los literales de cadena representan matrices terminadas en nulo, static-duration de char. Debido a que tienen una duración de almacenamiento estática, un literal de cadena o un puntero a la misma matriz subyacente se puede usar de manera segura de varias maneras que no puede usar un puntero a una matriz automática. Por ejemplo, devolver un literal de cadena desde una función tiene un comportamiento bien definido:

const char *get_hello() {
    return "Hello, World!";  /* safe */
}

Por razones históricas, los elementos de la matriz correspondientes a un literal de cadena no son formalmente const. Sin embargo, cualquier intento de modificarlos tiene comportamiento indefinido. Por lo general, un programa que intente modificar la matriz correspondiente a un literal de cadena se bloqueará o no funcionará correctamente.

char *foo = "hello";
foo[0] = 'y';  /* Undefined behavior - BAD! */

Cuando un puntero apunta a una cadena literal, o donde a veces puede hacerlo, es recomendable declarar el referente de ese puntero const para evitar involucrarse accidentalmente en un comportamiento indefinido.

const char *foo = "hello";
/* GOOD: can't modify the string pointed to by foo */

Por otro lado, un puntero hacia o dentro de la matriz subyacente de un literal de cadena no es inherentemente especial en sí mismo; su valor se puede modificar libremente para apuntar a otra cosa:

char *foo = "hello";
foo = "World!"; /* OK - we're just changing what foo points to */

Además, aunque los inicializadores para las matrices char pueden tener la misma forma que los literales de cadena, el uso de tal inicializador no confiere las características de una cadena literal a la matriz inicializada. El inicializador simplemente designa la longitud y el contenido inicial de la matriz. En particular, los elementos son modificables si no se declaran explícitamente const:

char foo[] = "hello";
foo[0] = 'y';  /* OK! */

Calcular la Longitud: strlen() #

#include <stdio.h>
#include <stdlib.h>
#include <string.h>

int main(int argc, char **argv) 
{
    /* Exit if no second argument is found. */
    if (argc != 2) 
    {
        puts("Argument missing.");
        return EXIT_FAILURE;
    }

    size_t len = strlen(argv[1]);
    printf("The length of the second argument is %zu.\n", len);

    return EXIT_SUCCESS;
}

Este programa calcula la longitud de su segundo argumento de entrada y almacena el resultado en len. Luego imprime esa longitud en la terminal. Por ejemplo, cuando se ejecuta con los parámetros program_name "¡Hola, mundo!", el programa generará La longitud del segundo argumento es 13. porque la cadena ¡Hola, mundo! tiene 13 caracteres.

strlen cuenta todos los bytes desde el comienzo de la cadena hasta, pero sin incluir, el carácter NUL final, '\0'. Como tal, solo se puede usar cuando se garantiza que la cadena terminará en NUL.

También tenga en cuenta que si la cadena contiene caracteres Unicode, strlen no le dirá cuántos caracteres hay en la cadena (ya que algunos caracteres pueden tener varios bytes de longitud). En tales casos, debe contar los caracteres (es decir, unidades de código) usted mismo. Considere la salida del siguiente ejemplo:

#include <stdio.h>
#include <stdlib.h>
#include <string.h>

int main(void) 
{
    char asciiString[50] = "Hello world!";
    char utf8String[50] = "Γειά σου Κόσμε!"; /* "Hello World!" in Greek */

    printf("asciiString has %zu bytes in the array\n", sizeof(asciiString));
    printf("utf8String has %zu bytes in the array\n", sizeof(utf8String));
    printf("\"%s\" is %zu bytes\n", asciiString, strlen(asciiString));
    printf("\"%s\" is %zu bytes\n", utf8String, strlen(utf8String));
}

Producción:

asciiString has 50 bytes in the array
utf8String has 50 bytes in the array
"Hello world!" is 12 bytes
"Γειά σου Κόσμε!" is 27 bytes

Introducción básica a las cadenas #

En C, una cadena es una secuencia de caracteres que termina con un carácter nulo (’\0’).

Podemos crear cadenas utilizando literales de cadena, que son secuencias de caracteres entre comillas dobles; por ejemplo, tome el literal de cadena "hello world". Los literales de cadena terminan automáticamente en nulo.

Podemos crear cadenas usando varios métodos. Por ejemplo, podemos declarar un char * e inicializarlo para que apunte al primer carácter de una cadena:

char * string = "hello world";

Al inicializar un char * a una constante de cadena como se indicó anteriormente, la cadena en sí generalmente se asigna en datos de solo lectura; string es un puntero al primer elemento de la matriz, que es el carácter 'h'.

Dado que el literal de cadena se asigna en la memoria de solo lectura, no se puede modificar1. Cualquier intento de modificarlo conducirá a comportamiento indefinido, por lo que es mejor agregar const para obtener un error en tiempo de compilación como este

char const * string = "hello world";

Tiene un efecto similar2 como

char const string_arr[] = "hello world";

Para crear una cadena modificable, puede declarar una matriz de caracteres e inicializar su contenido usando un literal de cadena, así:

char modifiable_string[] = "hello world";

Esto es equivalente a lo siguiente:

char modifiable_string[] = {'h', 'e', 'l', 'l', 'o', ' ', 'w', 'o', 'r', 'l', 'd', '\0'};

Dado que la segunda versión utiliza un inicializador entre llaves, la cadena no termina automáticamente en nulo a menos que se incluya explícitamente un carácter '\0' en la matriz de caracteres, generalmente como su último elemento.

1 No modificable implica que los caracteres en el literal de cadena no se pueden modificar, pero recuerde que el puntero cadena se puede modificar (puede apuntar a otro lugar o se puede incrementar o disminuir).

2 Ambas cadenas tienen un efecto similar en el sentido de que los caracteres de ambas cadenas no se pueden modificar. Cabe señalar que string es un puntero a char y es un [valor l modificable] (https://en.wikipedia.org/wiki/Value_(computer_science)#lrvalue) por lo que puede incrementarse o apunte a alguna otra ubicación mientras que la matriz string_arr es un valor l no modificable, no se puede modificar.

Copiando cadenas #

Las asignaciones de punteros no copian cadenas #

Puede usar el operador = para copiar números enteros, pero no puede usar el operador = para copiar cadenas en C. Las cadenas en C se representan como matrices de caracteres con un carácter nulo de terminación, por lo que usar el operador = solo guardará la dirección (puntero) de una cadena.

#include <stdio.h>

int main(void) {
    int a = 10, b;
    char c[] = "abc", *d;

    b = a; /* Integer is copied */
    a = 20; /* Modifying a leaves b unchanged - b is a 'deep copy' of a */
    printf("%d %d\n", a, b); /* "20 10" will be printed */

    d = c; 
    /* Only copies the address of the string - 
    there is still only one string stored in memory */
    
    c[1] = 'x';
    /* Modifies the original string - d[1] = 'x' will do exactly the same thing */

    printf("%s %s\n", c, d); /* "axc axc" will be printed */

    return 0;
}

El ejemplo anterior se compiló porque usamos char *d en lugar de char d[3]. El uso de este último provocaría un error del compilador. No puede asignar a matrices en C.

#include <stdio.h>

int main(void) {
    char a[] = "abc";
    char b[8];

    b = a; /* compile error */
    printf("%s\n", b);

    return 0;
}

Copiar cadenas usando funciones estándar #

strcpy() #

Para copiar cadenas, la función strcpy() está disponible en string.h. Se debe asignar suficiente espacio para el destino antes de realizar la copia.

#include <stdio.h>
#include <string.h>

int main(void) {
    char a[] = "abc";
    char b[8];

    strcpy(b, a); /* think "b special equals a" */
    printf("%s\n", b); /* "abc" will be printed */

    return 0;
}

snprintf() #

Para evitar el desbordamiento del búfer, se puede usar snprintf(). No es la mejor solución en cuanto a rendimiento, ya que tiene que analizar la cadena de la plantilla, pero es la única función segura de límite de búfer para copiar cadenas fácilmente disponibles en la biblioteca estándar, que se puede usar sin ningún paso adicional.

#include <stdio.h>
#include <string.h>

int main(void) {
    char a[] = "012345678901234567890";
    char b[8];

#if 0
    strcpy(b, a); /* causes buffer overrun (undefined behavior), so do not execute this here! */
#endif

    snprintf(b, sizeof(b), "%s", a); /* does not cause buffer overrun */
    printf("%s\n", b); /* "0123456" will be printed */

    return 0;
}

strncat() #

Una segunda opción, con mejor rendimiento, es usar strncat() (una versión de verificación de desbordamiento de búfer de strcat() ) - toma un tercer argumento que le dice el número máximo de bytes para copiar:

char dest[32];

dest[0] = '\0';
strncat(dest, source, sizeof(dest) - 1);
    /* copies up to the first (sizeof(dest) - 1) elements of source into dest,
    then puts a \0 on the end of dest */

Tenga en cuenta que esta formulación utiliza sizeof(dest) - 1; esto es crucial porque strncat() siempre agrega un byte nulo (bueno), pero no lo cuenta en el tamaño de la cadena (una causa de confusión y sobreescritura del búfer).

También tenga en cuenta que la alternativa, concatenar después de una cadena no vacía, es aún más complicada. Considerar:

char dst[24] = "Clownfish: ";
char src[] = "Marvin and Nemo";
size_t len = strlen(dst);

strncat(dst, src, sizeof(dst) - len - 1);
printf("%zu: [%s]\n", strlen(dst), dst);

La salida es:

23: [Clownfish: Marvin and N]

Tenga en cuenta, sin embargo, que el tamaño especificado como longitud no era el tamaño de la matriz de destino, sino la cantidad de espacio que quedaba en ella, sin contar el byte nulo del terminal. Esto puede causar grandes problemas de sobreescritura. También es un poco derrochador; para especificar el argumento de longitud correctamente, conoce la longitud de los datos en el destino, por lo que podría especificar la dirección del byte nulo al final del contenido existente, evitando que strncat() vuelva a escanearlo:

    strcpy(dst, "Clownfish: ");
    assert(len < sizeof(dst) - 1);
    strncat(dst + len, src, sizeof(dst) - len - 1);
    printf("%zu: [%s]\n", strlen(dst), dst);

Esto produce el mismo resultado que antes, pero strncat() no tiene que escanear el contenido existente de dst antes de que comience a copiar.

strncpy() #

La última opción es la función strncpy(). Aunque podría pensar que debería ser lo primero, es una función bastante engañosa que tiene dos problemas principales:

  1. Si la copia a través de strncpy() llega al límite del búfer, no se escribirá un carácter nulo de terminación.
  2. strncpy() siempre llena completamente el destino, con bytes nulos si es necesario.

(Esta implementación peculiar es histórica y [inicialmente estaba destinada a manejar nombres de archivos UNIX] 1)

La única forma correcta de usarlo es garantizar manualmente la terminación nula:

strncpy(b, a, sizeof(b)); /* the third parameter is destination buffer size */
b[sizeof(b)/sizeof(*b) - 1] = '\0'; /* terminate the string */
printf("%s\n", b); /* "0123456" will be printed */

Incluso entonces, si tiene un búfer grande, se vuelve muy ineficiente usar strncpy() debido al relleno nulo adicional.

Iterando sobre los caracteres en una cadena #

Si conocemos la longitud de la cadena, podemos usar un ciclo for para iterar sobre sus caracteres:

char * string = "hello world"; /* This 11 chars long, excluding the 0-terminator. */
size_t i = 0;
for (; i < 11; i++) {
    printf("%c\n", string[i]);    /* Print each character of the string. */
}

Alternativamente, podemos usar la función estándar strlen() para obtener la longitud de una cadena si no sabemos cuál es la cadena:

size_t length = strlen(string);
size_t i = 0; 
for (; i < length; i++) {
    printf("%c\n", string[i]);    /* Print each character of the string. */
}

Finalmente, podemos aprovechar el hecho de que se garantiza que las cadenas en C terminarán en nulo (lo que ya hicimos cuando lo pasamos a strlen() en el ejemplo anterior ;-)). Podemos iterar sobre la matriz independientemente de su tamaño y dejar de iterar una vez que alcancemos un carácter nulo:

size_t i = 0;
while (string[i] != '\0') {       /* Stop looping when we reach the null-character. */
    printf("%c\n", string[i]);    /* Print each character of the string. */
    i++;
}

Crear matrices de cadenas #

Una matriz de cadenas puede significar un par de cosas:

  1. Una matriz cuyos elementos son char *s
  2. Una matriz cuyos elementos son matrices de chars

Podemos crear una matriz de punteros de caracteres así:

char * string_array[] = {
    "foo",
    "bar",
    "baz"
};

Recuerde: cuando asignamos literales de cadena a char *, las cadenas mismas se asignan en la memoria de solo lectura. Sin embargo, la matriz string_array se asigna en la memoria de lectura/escritura. Esto significa que podemos modificar los punteros en la matriz, pero no podemos modificar las cadenas a las que apuntan.

En C, el parámetro principal argv (la matriz de argumentos de la línea de comandos pasados ​​cuando se ejecutó el programa) es una matriz de char *: char * argv[].

También podemos crear matrices de matrices de caracteres. Dado que las cadenas son matrices de caracteres, una matriz de cadenas es simplemente una matriz cuyos elementos son matrices de caracteres:

char modifiable_string_array_literals[][4] = {
    "foo",
    "bar",
    "baz"
};

Esto es equivalente a:

char modifiable_string_array[][4] = {
    {'f', 'o', 'o', '\0'},
    {'b', 'a', 'r', '\0'},
    {'b', 'a', 'z', '\0'}
};

Tenga en cuenta que especificamos 4 como el tamaño de la segunda dimensión de la matriz; cada una de las cadenas en nuestra matriz es en realidad de 4 bytes, ya que debemos incluir el carácter de terminación nula.

Convertir cadenas en números: atoi(), atof() (peligroso, no los use) #

Advertencia: Las funciones atoi, atol, atoll y atof son intrínsecamente inseguras porque: [Si el valor del resultado no se puede representar, el comportamiento no está definido.](http://www. open-std.org/jtc1/sc22/wg14/www/docs/n1124.pdf) (7.20.1p1)

#include <stdio.h>
#include <stdlib.h>

int main(int argc, char** argv)
{
    int val;
    if (argc < 2)
    {
        printf("Usage: %s <integer>\n", argv[0]);
        return 0;
    }

    val = atoi(argv[1]);

    printf("String value = %s, Int value = %d\n", argv[1], val);

    return 0;
}

Cuando la cadena que se va a convertir es un entero decimal válido que está dentro del rango, la función funciona:

$ ./atoi 100
String value = 100, Int value = 100
$ ./atoi 200
String value = 200, Int value = 200

Para las cadenas que comienzan con un número, seguido de otra cosa, solo se analiza el número inicial:

$ ./atoi 0x200
0
$ ./atoi 0123x300
123

En todos los demás casos, el comportamiento es indefinido:

$ ./atoi hello
Formatting the hard disk...

Debido a las ambigüedades anteriores y este comportamiento indefinido, la familia de funciones atoi nunca debe usarse.

  • Para convertir a int largo, use strtol() en lugar de atol().
  • Para convertir a doble, use strtod() en lugar de atof().
  • Para convertir a long long int, usa strtoll() en lugar de atoll().

Copiar y Concatenar: strcpy(), strcat() #

#include <stdio.h>
#include <string.h>

int main(void)
{
  /* Always ensure that your string is large enough to contain the characters
   * and a terminating NUL character ('\0')!
   */
  char mystring[10];

  /* Copy "foo" into `mystring`, until a NUL character is encountered. */
  strcpy(mystring, "foo");
  printf("%s\n", mystring);

  /* At this point, we used 4 chars of `mystring`, the 3 characters of "foo",
   * and the NUL terminating byte.
   */

  /* Append "bar" to `mystring`. */
  strcat(mystring, "bar");
  printf("%s\n", mystring);

  /* We now use 7 characters of `mystring`: "foo" requires 3, "bar" requires 3
   * and there is a terminating NUL character ('\0') at the end.
   */

  /* Copy "bar" into `mystring`, overwriting the former contents. */
  strcpy(mystring, "bar");
  printf("%s\n", mystring);

  return 0;
}

Salidas:

foo
foobar
bar

Si agrega o copia de una cadena existente, ¡asegúrese de que termine en NUL!

Los literales de cadena (por ejemplo, "foo") siempre serán terminados en NUL por el compilador.

Comparación: strcmp(), strncmp(), strcasecmp(), strncasecmp() #

Las funciones strcase* no son C estándar, sino una extensión POSIX.

La función strcmp compara lexicográficamente dos matrices de caracteres terminadas en cero. Las funciones devuelven un valor negativo si el primer argumento aparece antes que el segundo en orden lexicográfico, cero si se comparan iguales o positivo si el primer argumento aparece después del segundo en orden lexicográfico.

#include <stdio.h>
#include <string.h>

void compare(char const *lhs, char const *rhs)
{
    int result = strcmp(lhs, rhs); // compute comparison once
    if (result < 0) {
        printf("%s comes before %s\n", lhs, rhs);
    } else if (result == 0) {
        printf("%s equals %s\n", lhs, rhs);
    } else { // last case: result > 0
        printf("%s comes after %s\n", lhs, rhs);
    }
}

int main(void)
{
    compare("BBB", "BBB");
    compare("BBB", "CCCCC");
    compare("BBB", "AAAAAA");
    return 0;
}

Salidas:

BBB equals BBB
BBB comes before CCCCC
BBB comes after AAAAAA

Como strcmp, la función strcasecmp también compara lexicográficamente sus argumentos después de traducir cada carácter a su correspondiente en minúsculas:

#include <stdio.h>
#include <string.h>

void compare(char const *lhs, char const *rhs)
{
    int result = strcasecmp(lhs, rhs); // compute case-insensitive comparison once
    if (result < 0) {
        printf("%s comes before %s\n", lhs, rhs);
    } else if (result == 0) {
        printf("%s equals %s\n", lhs, rhs);
    } else { // last case: result > 0
        printf("%s comes after %s\n", lhs, rhs);
    }
}

int main(void)
{
    compare("BBB", "bBB");
    compare("BBB", "ccCCC");
    compare("BBB", "aaaaaa");
    return 0;
}

Salidas:

BBB equals bBB
BBB comes before ccCCC
BBB comes after aaaaaa

strncmp y strncasecmp comparan como máximo n caracteres:

#include <stdio.h>
#include <string.h>

void compare(char const *lhs, char const *rhs, int n)
{
    int result = strncmp(lhs, rhs, n); // compute comparison once
    if (result < 0) {
        printf("%s comes before %s\n", lhs, rhs);
    } else if (result == 0) {
        printf("%s equals %s\n", lhs, rhs);
    } else { // last case: result > 0
        printf("%s comes after %s\n", lhs, rhs);
    }
}

int main(void)
{
    compare("BBB", "Bb", 1);
    compare("BBB", "Bb", 2);
    compare("BBB", "Bb", 3);
    return 0;
}

Salidas:

BBB equals Bb
BBB comes before Bb
BBB comes before Bb

Convierta de forma segura cadenas a números: funciones strtoX #

Desde C99, la biblioteca C tiene un conjunto de funciones de conversión seguras que interpretan una cadena como un número. Sus nombres tienen la forma strtoX, donde X es uno de l, ul, d, etc. para determinar el tipo de destino de la conversión.

double strtod(char const* p, char** endptr);
long double strtold(char const* p, char** endptr);

Proporcionan la verificación de que una conversión tuvo un desbordamiento o subdesbordamiento:

double ret = strtod(argv[1], 0); /* attempt conversion */

/* check the conversion result. */
if ((ret == HUGE_VAL || ret == -HUGE_VAL) && errno == ERANGE) 
    return;  /* numeric overflow in in string */
else if (ret == HUGE_VAL && errno == ERANGE) 
    return; /* numeric underflow in in string */

/* At this point we know that everything went fine so ret may be used */

Si la cadena de hecho no contiene ningún número, este uso de strtod devuelve 0.0.

Si esto no es satisfactorio, se puede utilizar el parámetro adicional endptr. Es un puntero a puntero que apuntará al final del número detectado en la cadena. Si se establece en 0, como arriba, o NULL, simplemente se ignora.

Este parámetro endptr indica si ha habido una conversión exitosa y, de ser así, dónde terminó el número:

char *check = 0;
double ret = strtod(argv[1], &check); /* attempt conversion */

/* check the conversion result. */
if (argv[1] == check) 
    return; /* No number was detected in string */
else if ((ret == HUGE_VAL || ret == -HUGE_VAL) && errno == ERANGE) 
    return; /* numeric overflow in in string */
else if (ret == HUGE_VAL && errno == ERANGE) 
    return; /* numeric underflow in in string */

/* At this point we know that everything went fine so ret may be used */

Hay funciones análogas para convertir a los tipos enteros más amplios:

long strtol(char const* p, char** endptr, int nbase);
long long strtoll(char const* p, char** endptr, int nbase);
unsigned long strtoul(char const* p, char** endptr, int nbase);
unsigned long long strtoull(char const* p, char** endptr, int nbase);

Estas funciones tienen un tercer parámetro nbase que contiene el número base en que se escribe el número.

long a = strtol("101",   0, 2 ); /* a = 5L */
long b = strtol("101",   0, 8 ); /* b = 65L */
long c = strtol("101",   0, 10); /* c = 101L */
long d = strtol("101",   0, 16); /* d = 257L */
long e = strtol("101",   0, 0 ); /* e = 101L */
long f = strtol("0101",  0, 0 ); /* f = 65L */
long g = strtol("0x101", 0, 0 ); /* g = 257L */

El valor especial 0 para nbase significa que la cadena se interpreta de la misma manera que los números literales se interpretan en un programa C: un prefijo de 0x corresponde a una representación hexadecimal; de lo contrario, un 0 inicial es octal y todos los demás números se ven como decimales.

Por lo tanto, la forma más práctica de interpretar un argumento de línea de comandos como un número sería

int main(int argc, char* argv[] {
    if (argc < 1)
        return EXIT_FAILURE; /* No number given. */

    /* use strtoull because size_t may be wide */
    size_t mySize = strtoull(argv[1], 0, 0);

    /* then check conversion results. */

     ...

    return EXIT_SUCCESS;
}

Esto significa que el programa se puede llamar con un parámetro en octal, decimal o hexadecimal.

/* finds the next instance of needle in haystack 
   zbpos: the zero-based position to begin searching from
   haystack: the string to search in
   needle: the string that must be found
   returns the next match of `needle` in `haystack`, or -1 if not found
*/
int findnext(int zbpos, const char *haystack, const char *needle)
{
    char *p; 

    if (((p = strstr(haystack + zbpos, needle)) != NULL)
        return p - haystack;

    return -1;
}

strstr busca en el argumento pajar (primero) la cadena a la que apunta aguja. Si se encuentra, strstr devuelve la dirección de la ocurrencia. Si no pudo encontrar aguja, devuelve NULL. Usamos zbpos para no seguir encontrando la misma aguja una y otra vez. Para omitir la primera instancia, agregamos un desplazamiento de zbpos. Un clon del Bloc de notas podría llamar a findnext así, para implementar su diálogo “Buscar siguiente”:

/*
    Called when the user clicks "Find Next"
    doc: The text of the document to search
    findwhat: The string to find
*/
void onfindnext(const char *doc, const char *findwhat)
{
    static int i;

    if ((i = findnext(i, doc, findwhat)) != -1)
        /* select the text starting from i and ending at i + strlen(findwhat) */
    else
        /* display a message box saying "end of search" */
}
