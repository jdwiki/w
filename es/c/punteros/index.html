Un puntero es un tipo de variable que puede almacenar la dirección de otro objeto o una función.

Sintaxis #

  • <Tipo de datos> *<Nombre de variable>;
  • int *ptrToInt;
  • vacío *ptrToVoid; /* C89+ */
  • struct someStruct *ptrToStruct;
  • int **ptrToPtrToInt;
  • int arr[longitud]; int *ptrAlPrimerElem = arr; /* Para <C99 ’longitud’ debe ser una constante de tiempo de compilación, para >=C11 puede ser necesario. */
  • int *arrayOfPtrsToInt[longitud]; /* Para <C99 ’longitud’ debe ser una constante de tiempo de compilación, para >=C11 puede ser necesario. */

La posición del asterisco no afecta el significado de la definición:

/* The * operator binds to right and therefore these are all equivalent. */
int *i;
int * i;
int* i;

Sin embargo, al definir varios punteros a la vez, cada uno requiere su propio asterisco:

int *i, *j; /* i and j are both pointers */
int* i, j;  /* i is a pointer, but j is an int not a pointer variable */

También es posible una matriz de punteros, donde se da un asterisco antes del nombre de la variable de matriz:

int *foo[2]; /* foo is a array of pointers, can be accessed as *foo[0] and *foo[1] */

Errores comunes #

El uso inadecuado de los punteros suele ser una fuente de errores que pueden incluir errores de seguridad o bloqueos del programa, con mayor frecuencia debido a fallas de segmentación.

No verificar fallas de asignación #

No se garantiza que la asignación de memoria tenga éxito y, en su lugar, puede devolver un puntero NULL. Usar el valor devuelto, sin verificar si la asignación es exitosa, invoca [comportamiento indefinido] 1. Esto generalmente conduce a un bloqueo, pero no hay garantía de que ocurra un bloqueo, por lo que confiar en eso también puede generar problemas.

Por ejemplo, forma insegura:

struct SomeStruct *s = malloc(sizeof *s);
s->someValue = 0; /* UNSAFE, because s might be a null pointer */

Camino seguro:

struct SomeStruct *s = malloc(sizeof *s);
if (s)
{
    s->someValue = 0; /* This is safe, we have checked that s is valid */
}

Usando números literales en lugar de sizeof al solicitar memoria #

Para una configuración dada de compilador/máquina, los tipos tienen un tamaño conocido; sin embargo, no hay ningún estándar que defina que el tamaño de un tipo dado (aparte de char) será el mismo para todas las configuraciones de compilador/máquina. Si el código usa 4 en lugar de sizeof(int) para la asignación de memoria, puede funcionar en la máquina original, pero el código no es necesariamente portátil para otras máquinas o compiladores. Los tamaños fijos para los tipos deben reemplazarse por sizeof(that_type) o sizeof(*var_ptr_to_that_type).

Asignación no portable:

 int *intPtr = malloc(4*1000);    /* allocating storage for 1000 int */
 long *longPtr = malloc(8*1000);  /* allocating storage for 1000 long */

Asignación portátil:

 int *intPtr = malloc(sizeof(int)*1000);     /* allocating storage for 1000 int */
 long *longPtr = malloc(sizeof(long)*1000);  /* allocating storage for 1000 long */

O, mejor aún:

 int *intPtr = malloc(sizeof(*intPtr)*1000);     /* allocating storage for 1000 int */
 long *longPtr = malloc(sizeof(*longPtr)*1000);  /* allocating storage for 1000 long */

Pérdidas de memoria #

Si no se desasigna la memoria utilizando free, se acumula memoria no reutilizable, que el programa ya no utiliza; esto se llama pérdida de memoria. Las fugas de memoria desperdician recursos de memoria y pueden provocar errores de asignación.

Errores lógicos #

Todas las asignaciones deben seguir el mismo patrón:

  1. Asignación usando malloc (o calloc)
  2. Uso para almacenar datos
  3. Desasignación usando gratis

El incumplimiento de este patrón, como el uso de la memoria después de una llamada a free (puntero colgante) o antes de una llamada a malloc (wild pointer), llamando a free dos veces (“doble gratis”), etc., generalmente causa una falla de segmentación y resulta en un bloqueo del programa.

Estos errores pueden ser transitorios y difíciles de depurar; por ejemplo, el sistema operativo no suele recuperar inmediatamente la memoria liberada y, por lo tanto, los punteros colgantes pueden persistir durante un tiempo y parecer que funcionan.

En los sistemas en los que funciona, Valgrind es una herramienta invaluable para identificar qué memoria se pierde y dónde se asignó originalmente.

Creando punteros para apilar variables #

La creación de un puntero no prolonga la vida útil de la variable a la que se apunta. Por ejemplo:

int* myFunction() 
{
    int x = 10;
    return &x;
}

Aquí, x tiene duración de almacenamiento automático (comúnmente conocida como asignación de pila). Debido a que está asignado en la pila, su tiempo de vida es solo mientras se esté ejecutando myFunction; después de que myFunction ha salido, la variable x se destruye. Esta función obtiene la dirección de x (usando &x) y se la devuelve a la persona que llama, dejando a la persona que llama con un puntero a una variable inexistente. Intentar acceder a esta variable invocará comportamiento indefinido.

La mayoría de los compiladores en realidad no borran un marco de pila después de que la función sale, por lo que la desreferenciación del puntero devuelto a menudo le brinda los datos esperados. Sin embargo, cuando se llama a otra función, la memoria a la que se apunta puede sobrescribirse y parece que los datos a los que se apunta se han dañado.

Para resolver esto, malloque el almacenamiento de la variable que se va a devolver y devuelva un puntero al almacenamiento recién creado, o solicite que se pase un puntero válido a la función en lugar de devolver uno, por ejemplo:

#include <stdlib.h>
#include <stdio.h>

int *solution1(void) 
{
    int *x = malloc(sizeof *x);
    if (x == NULL) 
    {
        /* Something went wrong */
        return NULL;
    }

    *x = 10;

    return x;
}

void solution2(int *x) 
{
    /* NB: calling this function with an invalid or null pointer 
       causes undefined behaviour. */

    *x = 10;
}

int main(void) 
{
    { 
        /* Use solution1() */

        int *foo = solution1();  
        if (foo == NULL)
        {
            /* Something went wrong */
            return 1;
        }

        printf("The value set by solution1() is %i\n", *foo);
        /* Will output: "The value set by solution1() is 10" */

        free(foo);    /* Tidy up */
    }

    {
        /* Use solution2() */

        int bar;
        solution2(&bar); 

        printf("The value set by solution2() is %i\n", bar);
        /* Will output: "The value set by solution2() is 10" */
    }

    return 0;
}

Incrementar/decrementar y desreferenciar #

Si escribe *p++ para incrementar lo que apunta p, está equivocado.

El incremento/decremento posterior se ejecuta antes de la desreferenciación. Por lo tanto, esta expresión incrementará el puntero p y devolverá lo que apuntaba p antes de incrementar sin cambiarlo.

Debes escribir (*p)++ para incrementar lo que apunta p.

Esta regla también se aplica a la disminución posterior: *p-- disminuirá el puntero p en sí mismo, no lo que apunta p.

Introducción #

Un puntero se declara como cualquier otra variable, excepto que se coloca un asterisco (*) entre el tipo y el nombre de la variable para indicar que es un puntero.

int *pointer; /* inside a function, pointer is uninitialized and doesn't point to any valid object yet */

Para declarar dos variables de puntero del mismo tipo, en la misma declaración, use el símbolo de asterisco antes de cada identificador. Por ejemplo,

int *iptr1, *iptr2;
int *iptr3,  iptr4;  /* iptr3 is a pointer variable, whereas iptr4 is misnamed and is an int */

El operador de dirección o de referencia denotado por un ampersand (&) da la dirección de una variable dada que se puede colocar en un puntero del tipo apropiado.

int value = 1;
pointer = &value;

El operador de direccionamiento indirecto o desreferencia denotado por un asterisco (*) obtiene el contenido de un objeto al que apunta un puntero.

printf("Value of pointed to integer: %d\n", *pointer);
/* Value of pointed to integer: 1 */

Si el puntero apunta a una estructura o tipo de unión, puede quitarle la referencia y acceder a sus miembros directamente usando el operador ->:

SomeStruct *s = &someObject;
s->someMember = 5; /* Equivalent to (*s).someMember = 5 */

En C, un puntero es un tipo de valor distinto que se puede reasignar y, de lo contrario, se trata como una variable por derecho propio. Por ejemplo, el siguiente ejemplo imprime el valor del puntero (variable) en sí.

printf("Value of the pointer itself: %p\n", (void *)pointer);
/* Value of the pointer itself: 0x7ffcd41b06e4 */
/* This address will be different each time the program is executed */

Debido a que un puntero es una variable mutable, es posible que no apuntar a un objeto válido, ya sea estableciéndolo como nulo

pointer = 0;     /* or alternatively */
pointer = NULL;

o simplemente por contener un patrón de bits arbitrario que no es válido Dirección. Esta última es una situación muy mala, porque no se puede probado antes de que el puntero sea desreferenciado, solo hay una prueba para el caso de que un puntero sea nulo:

if (!pointer) exit(EXIT_FAILURE);

Un puntero solo puede ser desreferenciado si apunta a un objeto válido, de lo contrario, el comportamiento no está definido. Muchas implementaciones modernas pueden ayudarlo al generando algún tipo de error como un fallo de segmentación y terminar la ejecución, pero otros pueden simplemente dejar su programa en un Estado Inválido.

El valor devuelto por el operador de desreferencia es un alias mutable a la variable original, por lo que se puede cambiar, modificando la variable original.

*pointer += 1;
printf("Value of pointed to variable after change: %d\n", *pointer);
/* Value of pointed to variable after change: 2 */

Los punteros también son reasignables. Esto significa que un puntero que apunta a un objeto se puede usar más tarde para apuntar a otro objeto del mismo tipo.

int value2 = 10;
pointer = &value2;
printf("Value from pointer: %d\n", *pointer);
/* Value from pointer: 10 */

Como cualquier otra variable, los punteros tienen un tipo específico. no puedes asignar la dirección de un int corto a un puntero a un int largo, por instancia. Tal comportamiento se conoce como juego de palabras y está prohibido en C, aunque hay algunas excepciones.

Aunque el puntero debe ser de un tipo específico, la memoria asignada para cada tipo de puntero es igual a la memoria utilizada por el entorno para almacenar direcciones, en lugar del tamaño del tipo al que apunta.

#include <stdio.h>

int main(void) {
    printf("Size of int pointer: %zu\n", sizeof (int*));      /* size 4 bytes */
    printf("Size of int variable: %zu\n", sizeof (int));      /* size 4 bytes */
    printf("Size of char pointer: %zu\n", sizeof (char*));    /* size 4 bytes */
    printf("Size of char variable: %zu\n", sizeof (char));    /* size 1 bytes */
    printf("Size of short pointer: %zu\n", sizeof (short*));  /* size 4 bytes */
    printf("Size of short variable: %zu\n", sizeof (short));  /* size 2 bytes */
    return 0;
}

(Nota: si usa Microsoft Visual Studio, que no es compatible con los estándares C99 o C11, debe usar [%Iu](https://msdn.microsoft.com/en-us/library/tcxf1dw6. aspx)1 en lugar de %zu en el ejemplo anterior.)

Tenga en cuenta que los resultados anteriores pueden variar de un entorno a otro en números, pero todos los entornos mostrarían tamaños iguales para diferentes tipos de puntero.

Extracto basado en información de [Introducción de punteros C de la Universidad de Cardiff] (https://www.cs.cf.ac.uk/Dave/C/node10.html)

Punteros y matrices #

Los punteros y las matrices están íntimamente conectados en C. Las matrices en C siempre se mantienen en ubicaciones contiguas en la memoria. La aritmética de punteros siempre se escala según el tamaño del elemento al que se apunta. Entonces, si tenemos una matriz de tres dobles y un puntero a la base, *ptr se refiere al primer doble, *(ptr + 1) al segundo, *(ptr + 2) al tercero . Una notación más conveniente es usar la notación de matriz [].

double point[3] = {0.0, 1.0, 2.0};
double *ptr = point;

/* prints x 0.0, y 1.0 z 2.0 */ 
printf("x %f y %f z %f\n", ptr[0], ptr[1], ptr[2]);

Entonces, esencialmente, ptr y el nombre de la matriz son intercambiables. Esta regla también significa que una matriz decae a un puntero cuando se pasa a una subrutina.

double point[3] = {0.0, 1.0, 2.0};

printf("length of point is %s\n", length(point));

/* get the distance of a 3D point from the origin */ 
double length(double *pt)
{
   return sqrt(pt[0] * pt[0] + pt[1] * pt[1] + pt[2] * pt[2])
}

Un puntero puede apuntar a cualquier elemento de una matriz o al elemento más allá del último elemento. Sin embargo, es un error establecer un puntero en cualquier otro valor, incluido el elemento anterior a la matriz. (La razón es que en arquitecturas segmentadas la dirección antes del primer elemento puede cruzar un límite de segmento, el compilador asegura que eso no suceda para el último elemento más uno).

Nota al pie 1: la información de formato de Microsoft se puede encontrar a través de printf() y [sintaxis de especificación de formato](https: //msdn.microsoft.com/en-us/library/56e442dc.aspx).

Desreferenciar un puntero #

int a = 1;
int *a_pointer = &a;

Para desreferenciar a_pointer y cambiar el valor de a, usamos la siguiente operación

*a_pointer = 2;

Esto se puede verificar utilizando las siguientes declaraciones de impresión.

printf("%d\n", a); /* Prints 2 */
printf("%d\n", *a_pointer); /* Also prints 2 */

Sin embargo, uno se equivocaría al desreferenciar un puntero NULL o no válido. Este

int *p1, *p2;

p1 = (int *) 0xbad;
p2 = NULL;

*p1 = 42;
*p2 = *p1 + 1;

suele ser comportamiento indefinido. p1 no se puede desreferenciar porque apunta a una dirección 0xbad que puede no ser una dirección válida. ¿Quién sabe lo que hay? Puede ser la memoria del sistema operativo o la memoria de otro programa. El único código de tiempo como este que se usa es en el desarrollo integrado, que almacena información particular en direcciones codificadas. p2 no se puede desreferenciar porque es NULL, que no es válido.

Eliminando la referencia de un puntero a una estructura #

Digamos que tenemos la siguiente estructura:

struct MY_STRUCT 
{
    int my_int;
    float my_float;
};

Podemos definir MY_STRUCT para omitir la palabra clave struct para que no tengamos que escribir struct MY_STRUCT cada vez que la usamos. Esto, sin embargo, es opcional.

typedef struct MY_STRUCT MY_STRUCT;

Si tenemos un puntero a una instancia de esta estructura

MY_STRUCT *instance;

Si esta declaración aparece en el alcance del archivo, la “instancia” se inicializará con un puntero nulo cuando se inicie el programa. Si esta declaración aparece dentro de una función, su valor es indefinido. La variable se debe inicializar para que apunte a una variable MY_STRUCT válida, o al espacio asignado dinámicamente, antes de que se le pueda quitar la referencia. Por ejemplo:

MY_STRUCT info = { 1, 3.141593F };
MY_STRUCT *instance = &info;

Cuando el puntero es válido, podemos desreferenciarlo para acceder a sus miembros usando una de dos notaciones diferentes:

int a = (*instance).my_int;
float b = instance->my_float;

Si bien estos dos métodos funcionan, es una mejor práctica usar el operador de flecha -> en lugar de la combinación de paréntesis, el operador de desreferencia * y el operador de punto . porque es más fácil de leer y comprender, especialmente con usos anidados.

Otra diferencia importante se muestra a continuación:

MY_STRUCT copy = *instance;
copy.my_int    = 2;

En este caso, copia contiene una copia del contenido de instancia. Cambiar my_int de copy no lo cambiará en instance.

MY_STRUCT *ref = instance;
ref->my_int    = 2;

En este caso, ref es una referencia a instance. Cambiar my_int usando la referencia lo cambiará en instance.

Es una práctica común usar punteros a estructuras como parámetros en funciones, en lugar de las estructuras mismas. El uso de las estructuras como parámetros de función podría hacer que la pila se desborde si la estructura es grande. El uso de un puntero a una estructura solo usa suficiente espacio de pila para el puntero, pero puede causar efectos secundarios si la función cambia la estructura que se pasa a la función.

Punteros de función #

Los punteros también se pueden utilizar para señalar funciones.

Tomemos una función básica:

int my_function(int a, int b)
{
    return 2 * a + 3 * b;
}

Ahora, definamos un puntero del tipo de esa función:

int (*my_pointer)(int, int);

Para crear uno, simplemente use esta plantilla:

return_type_of_func (*my_func_pointer)(type_arg1, type_arg2, ...)

Entonces debemos asignar este puntero a la función:

my_pointer = &my_function;

Este puntero ahora se puede usar para llamar a la función:

/* Calling the pointed function */
int result = (*my_pointer)(4, 2);

...

/* Using the function pointer as an argument to another function */
void another_function(int (*another_pointer)(int, int))
{
    int a = 4;
    int b = 2;
    int result = (*another_pointer)(a, b);

    printf("%d\n", result);
}

Aunque esta sintaxis parece más natural y coherente con los tipos básicos, los punteros de función de atribución y desreferenciación no requieren el uso de los operadores & y *. Así que el siguiente fragmento es igualmente válido:

/* Attribution without the & operator */
my_pointer = my_function;

/* Dereferencing without the * operator */
int result = my_pointer(4, 2);

Para aumentar la legibilidad de los punteros de función, se pueden usar typedefs.

typedef void (*Callback)(int a);

void some_function(Callback callback)
{
    int a = 4;
    callback(a);
}

Otro truco de legibilidad es que el estándar C permite simplificar un puntero de función en argumentos como los anteriores (pero no en la declaración de variables) a algo que parece un prototipo de función; por lo tanto, lo siguiente se puede usar de manera equivalente para definiciones y declaraciones de funciones:

void some_function(void callback(int))
{
    int a = 4;
    callback(a);
}

Punteros constantes #

Punteros individuales #

  • Puntero a un int

El puntero puede apuntar a diferentes enteros y los int’s se pueden cambiar a través del puntero. Esta muestra de código asigna b para que apunte a int b y luego cambia el valor de b a 100.

  int b;
  int* p;
  p = &b;    /* OK */
  *p = 100;  /* OK */

  • Puntero a const int

    The pointer can point to different integers but the int’s value can’t be changed through the pointer.

    int b;
const int* p;
p = &b;    /* OK */
*p = 100;  /* Compiler Error */

  • puntero const a int

El puntero solo puede apuntar a un int pero el valor de int se puede cambiar a través del puntero.

  int a, b;
  int* const p = &b; /* OK as initialisation, no assignment */
  *p = 100;  /* OK */
  p = &a;    /* Compiler Error */
  • puntero const a const int

El puntero solo puede apuntar a un int y el int no se puede cambiar a través del puntero.

  int a, b;
  const int* const p = &b; /* OK as initialisation, no assignment */
  p = &a;   /* Compiler Error */
  *p = 100; /* Compiler Error */

Comportamiento polimórfico con punteros vacíos #

La función de biblioteca estándar qsort() es un buen ejemplo de cómo se pueden usar punteros vacíos para hacer que una sola función opere en una gran variedad de tipos diferentes.

void qsort (
    void *base,                                 /* Array to be sorted */
    size_t num,                                 /* Number of elements in array */
    size_t size,                                /* Size in bytes of each element */
    int (*compar)(const void *, const void *)); /* Comparison function for two elements */

La matriz que se va a ordenar se pasa como un puntero vacío, por lo que se puede operar en una matriz de cualquier tipo de elemento. Los siguientes dos argumentos le dicen a qsort() cuántos elementos debe esperar en la matriz, y qué tan grande, en bytes, es cada elemento.

El último argumento es un puntero de función a una función de comparación que a su vez toma dos punteros vacíos. Al hacer que la persona que llama proporcione esta función, qsort() puede ordenar elementos de cualquier tipo de manera efectiva.

Aquí hay un ejemplo de una función de comparación de este tipo, para comparar flotadores. Tenga en cuenta que cualquier función de comparación que se pase a qsort() necesita tener este tipo de firma. La forma en que se hace polimórfico es convirtiendo los argumentos de puntero vacío en punteros del tipo de elemento que deseamos comparar.

int compare_floats(const void *a, const void *b)
{
    float fa = *((float *)a);
    float fb = *((float *)b);
    if (fa < fb)
        return -1;
    if (fa > fb)
        return 1;
    return 0;
}

Como sabemos que qsort usará esta función para comparar flotantes, convertimos los argumentos de punteros vacíos en punteros flotantes antes de desreferenciarlos.

Ahora, el uso de la función polimórfica qsort en una matriz “matriz” con longitud “len” es muy simple:

qsort(array, len, sizeof(array[0]), compare_floats);

Inicializando punteros #

La inicialización del puntero es una buena manera de evitar los punteros salvajes. La inicialización es simple y no es diferente de la inicialización de una variable.

#include <stddef.h>

int main()
{
    int *p1 = NULL; 
    char *p2 = NULL;
    float *p3 = NULL;

         /* NULL is a macro defined in stddef.h, stdio.h, stdlib.h, and string.h */

    ...
}

En la mayoría de los sistemas operativos, el uso involuntario de un puntero que se ha inicializado en NULL a menudo provocará que el programa se bloquee de inmediato, lo que facilitará la identificación de la causa del problema. El uso de un puntero no inicializado a menudo puede causar errores difíciles de diagnosticar.

Precaución: #

El resultado de desreferenciar un puntero NULL no está definido, por lo que no necesariamente causará un bloqueo incluso si ese es el comportamiento natural del sistema operativo en el que se ejecuta el programa. Las optimizaciones del compilador pueden enmascarar el bloqueo, hacer que el bloqueo ocurra antes o después del punto en el código fuente en el que se produjo la desreferencia del puntero nulo, o hacer que partes del código que contienen la desreferencia del puntero nulo se eliminen inesperadamente del programa. Las compilaciones de depuración no suelen mostrar estos comportamientos, pero el estándar del idioma no lo garantiza. También se permiten otros comportamientos inesperados y/o no deseados.

Debido a que NULL nunca apunta a una variable, a la memoria asignada o a una función, es seguro usarlo como valor de protección.

Precaución: #

Por lo general, NULL se define como (void *)0. Pero esto no implica que la dirección de memoria asignada sea 0x0. Para obtener más aclaraciones, consulte [C-faq para punteros NULL] (http://c-faq.com/null/)

Tenga en cuenta que también puede inicializar punteros para que contengan valores distintos de NULL.

int i1;

int main()
{
   int *p1 = &i1;
   const char *p2 = "A constant string to point to";
   float *p3 = malloc(10 * sizeof(float));
}

Dirección del operador ( & ) #

Para cualquier objeto (es decir, variable, matriz, unión, estructura, puntero o función), el operador de dirección unaria se puede usar para acceder a la dirección de ese objeto.

Suponer que

int i = 1;              
int *p = NULL;

Entonces, una instrucción p = &i;, copia la dirección de la variable i al puntero p.

Se expresa como p apunta a i.

printf("%d\n", *p); imprime 1, que es el valor de i.

Aritmética de punteros #

Consulte aquí: https://www.wikiod.com/es/c/operadores#Aritmética de punteros

punteros void* como argumentos y valores devueltos a funciones estándar #

void* es un tipo catch all para punteros a tipos de objetos. Un ejemplo de esto en uso es con la función malloc, que se declara como

void* malloc(size_t);

El tipo de retorno puntero a vacío significa que es posible asignar el valor de retorno de malloc a un puntero a cualquier otro tipo de objeto:

int* vector = malloc(10 * sizeof *vector);

En general, se considera una buena práctica no convertir explícitamente los valores dentro y fuera de los punteros vacíos. En el caso específico de malloc(), esto se debe a que con una conversión explícita, el compilador puede suponer, pero no advertir, un tipo de retorno incorrecto para malloc(), si olvida incluir stdlib.h. También se trata de usar el comportamiento correcto de los punteros vacíos para ajustarse mejor al principio DRY (no se repita); compare lo anterior con lo siguiente, donde el siguiente código contiene varios lugares adicionales innecesarios donde un error tipográfico podría causar problemas:

int* vector = (int*)malloc(10 * sizeof int*);

Del mismo modo, funciones como

void* memcpy(void *restrict target, void const *restrict source, size_t size);

tienen sus argumentos especificados como void * porque la dirección de cualquier objeto, independientemente del tipo, se puede pasar. Aquí también, una llamada no debe usar una conversión

unsigned char buffer[sizeof(int)];
int b = 67;
memcpy(buffer, &b, sizeof buffer);

Mismo asterisco, diferentes significados #

Premisa #

Lo más confuso que rodea a la sintaxis del puntero en C y C++ es que en realidad hay dos significados diferentes que se aplican cuando el símbolo del puntero, el asterisco (*), se usa con una variable.

Ejemplo #

En primer lugar, usa * para declarar una variable de puntero.

int i = 5;
/* 'p' is a pointer to an integer, initialized as NULL */
int *p = NULL;
/* '&i' evaluates into address of 'i', which then assigned to 'p' */
p = &i;
/* 'p' is now holding the address of 'i' */

Cuando no está declarando (o multiplicando), * se usa para desreferenciar una variable de puntero:

*p = 123;
/* 'p' was pointing to 'i', so this changes value of 'i' to 123 */

Cuando desee que una variable de puntero existente contenga la dirección de otra variable, no use *, pero hágalo así:

p = &another_variable;

Una confusión común entre los novatos en programación C surge cuando declaran e inicializan una variable de puntero al mismo tiempo.

int *p = &i;

Dado que int i = 5; y int i; i = 5; dan el mismo resultado, algunos de ellos podrían pensar int *p = &i; y int *p; *p = &i; da el mismo resultado también. El hecho es que no, int *p; *p = &i; intentará deferenciar un puntero no inicializado que dará como resultado UB. Nunca uses * cuando no estés declarando o desreferenciando un puntero.

Conclusión #

El asterisco (*) tiene dos significados distintos dentro de C en relación con los punteros, dependiendo de dónde se use. Cuando se usa dentro de una declaración de variable, el valor en el lado derecho del lado igual debe ser un valor de puntero a una dirección en la memoria. Cuando se usa con una variable ya declarada, el asterisco desreferenciará el valor del puntero, siguiéndolo al lugar señalado en la memoria, y permitiendo que el valor almacenado allí sea asignado o recuperado.

Quitar

Es importante tener en cuenta las P y las Q, por así decirlo, cuando se trata de punteros. Tenga en cuenta cuándo usa el asterisco y qué significa cuando lo usa allí. Pasar por alto este pequeño detalle podría resultar en un comportamiento defectuoso y/o indefinido con el que realmente no desea tener que lidiar.
