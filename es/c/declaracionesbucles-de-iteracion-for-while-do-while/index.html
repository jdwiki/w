Sintaxis #

  • /* Todas las versiones */
  • para ([expresión]; [expresión]; [expresión]) una_sentencia
  • para ([expresión]; [expresión]; [expresión]) { zero or several statements }
  • while (expresión) one_statement
  • while (expresión) { cero o varias sentencias }
  • hacer one_statement while (expresión);
  • do { una o más sentencias } while (expresión);
  • // desde C99 además del formulario anterior
  • for (declaración; [expresión]; [expresión]) one_statement;
  • para (declaración; [expresión]; [expresión]) { zero or several statements }

Las sentencias/bucles de iteración se dividen en dos categorías:

  • declaración/bucles de iteración controlados por la cabeza
  • declaración de iteración controlada por el pie/bucles

Bucles/Declaraciones de iteración controladas por la cabeza #

for ([<expression>]; [<expression>]; [<expression>]) <statement>
while (<expression>) <statement>

for ([declaration expression]; [expression] [; [expression]]) statement

Instrucción/bucles de iteración controlados por pedal #

do <statement> while (<expression>);

En bucle #

Para ejecutar un bloque de código una y otra vez, los bucles entran en escena. El bucle for se utiliza cuando un bloque de código se ejecuta un número fijo de veces. Por ejemplo, para llenar una matriz de tamaño n con las entradas del usuario, necesitamos ejecutar scanf() n veces.

#include <stddef.h>          // for size_t

int array[10];               // array of 10 int

for (size_t i = 0; i < 10; i++)    // i starts at 0 and finishes with 9
{
   scanf("%d", &array[i]);
}

De esta forma, la llamada a la función scanf() se ejecuta n veces (10 veces en nuestro ejemplo), pero solo se escribe una vez.

Aquí, la variable i es el índice del ciclo, y es mejor declararlo tal como se presenta. El tipo size_t (tipo de tamaño) debe usarse para todo lo que cuenta o recorre objetos de datos.

Esta forma de declarar variables dentro de for solo está disponible para compiladores que se han actualizado al estándar C99. Si por alguna razón todavía está atascado con un compilador anterior, puede declarar el índice del bucle antes del bucle for:

#include <stddef.h>        /* for size_t */
size_t i;
int array[10];             /* array of 10 int */

for (i = 0; i < 10; i++)       /* i starts at 0 and finishes at 9 */
{
   scanf("%d", &array[i]);
}

Bucle mientras #

Un bucle while se usa para ejecutar un fragmento de código mientras una condición es verdadera. El bucle while se utiliza cuando un bloque de código se ejecuta un número variable de veces. Por ejemplo, el código que se muestra obtiene la entrada del usuario, siempre que el usuario inserte números que no sean ‘0’. Si el usuario inserta 0, la condición while ya no es verdadera, por lo que la ejecución saldrá del ciclo y continuará con cualquier código posterior:

int num = 1;

while (num != 0)
{
    scanf("%d", &num);
}

Bucle Do-While #

A diferencia de los bucles for y while, los bucles do-while comprueban la veracidad de la condición al final del bucle, lo que significa que el bloque do se ejecutará una vez y luego comprobará la condición del while. en la parte inferior del bloque. Lo que significa que un bucle do-while siempre se ejecutará al menos una vez.

Por ejemplo, este bucle do-while obtendrá números del usuario, hasta que la suma de estos valores sea mayor o igual a 50:

int num, sum;
num = sum = 0;

do 
{
  scanf("%d", &num);
  sum += num;

} while (sum < 50);

Los bucles do-while son relativamente raros en la mayoría de los estilos de programación.

Estructura y flujo de control en un bucle for #

for ([declaration-or-expression]; [expression2]; [expression3])
{
    /* body of the loop */
}

En un bucle for, la condición del bucle tiene tres expresiones, todas opcionales.

  • La primera expresión, declaración-o-expresión, inicializa el bucle. Se ejecuta exactamente una vez al comienzo del bucle.

Puede ser una declaración e inicialización de una variable de bucle o una expresión general. Si es una declaración, el alcance de la variable declarada está restringido por la declaración for.

Las versiones históricas de C solo permitían una expresión, aquí, y la declaración de una variable de bucle tenía que colocarse antes de for.

  • La segunda expresión, expresión2, es la condición de prueba. Primero se ejecuta después de la inicialización. Si la condición es verdadera, entonces el control entra en el cuerpo del bucle. Si no, se desplaza hacia fuera del cuerpo del ciclo al final del ciclo. Posteriormente, esta condición se verifica después de cada ejecución del cuerpo, así como la declaración de actualización. Cuando es verdadero, el control retrocede al principio del cuerpo del bucle. La condición generalmente pretende ser una verificación del número de veces que se ejecuta el cuerpo del ciclo. Esta es la forma principal de salir de un ciclo, la otra forma es usando [jump declaraciones] 1.
  • La tercera expresión, expresión3, es la sentencia de actualización. Se ejecuta después de cada ejecución del cuerpo del bucle. A menudo se usa para incrementar una variable que lleva la cuenta del número de veces que se ha ejecutado el cuerpo del ciclo, y esta variable se llama iterador.

Cada instancia de ejecución del cuerpo del bucle se denomina iteración.

Ejemplo:

for(int i = 0; i < 10 ; i++)
{
    printf("%d", i);
}

La salida es:

0123456789

En el ejemplo anterior, primero se ejecuta i = 0, inicializando i. Luego, se verifica la condición i < 10, que se evalúa como verdadera. El control entra en el cuerpo del bucle y se imprime el valor de i. Luego, el control cambia a i++, actualizando el valor de i de 0 a 1. Luego, la condición se vuelve a verificar y el proceso continúa. Esto continúa hasta que el valor de i se convierte en 10. Luego, la condición i < 10 se evalúa como falso, después de lo cual el control sale del bucle.

Bucles infinitos #

Se dice que un bucle es un bucle infinito si el control entra pero nunca sale del cuerpo del bucle. Esto sucede cuando la condición de prueba del ciclo nunca se evalúa como falso.

Ejemplo:

for (int i = 0; i >= 0; )
{
    /* body of the loop where i is not changed*/
}

En el ejemplo anterior, la variable i, el iterador, se inicializa en 0. La condición de prueba es inicialmente verdadera. Sin embargo, i no se modifica en ninguna parte del cuerpo y la expresión de actualización está vacía. Por lo tanto, i seguirá siendo 0 y la condición de prueba nunca se evaluará como falso, lo que conducirá a un bucle infinito.

Suponiendo que no hay [jump declaraciones,] 1 otra forma en que se puede formar un ciclo infinito es manteniendo explícitamente la condición verdadera:

while (true)
{
    /* body of the loop */
}

En un bucle for, la declaración de condición es opcional. En este caso, la condición siempre es “verdadera” de forma vacía, lo que lleva a un bucle infinito.

for (;;)
{
    /* body of the loop */
}

Sin embargo, en ciertos casos, la condición puede mantenerse verdadera intencionalmente, con la intención de salir del ciclo usando una sentencia de salto como break.

while (true)
{
    /* statements */
    if (condition)
    {
         /* more statements */
         break;
    }
}

Desenrollado de bucles y dispositivo de Duff #

A veces, el bucle directo no puede estar completamente contenido dentro del cuerpo del bucle. Esto se debe a que el ciclo debe prepararse con algunas declaraciones B. Luego, la iteración comienza con algunas declaraciones A, que luego son seguidas por B nuevamente antes de repetirse.

do_B();
while (condition) {
    do_A();
    do_B();
}

Para evitar posibles problemas de cortar/pegar con la repetición de B dos veces en el código, se podría aplicar Duff’s Device para iniciar el ciclo desde la mitad del cuerpo while, usando una declaración [switch ][2] y caer a través del comportamiento.

switch (true) while (condition) {
case false: do_A(); /* FALL THROUGH */
default:    do_B(); /* FALL THROUGH */
}

El dispositivo de Duff en realidad se inventó para implementar el desenrollado de bucles. Imagine aplicar una máscara a un bloque de memoria, donde n es un tipo integral con signo con un valor positivo.

do {
    *ptr++ ^= mask;
} while (--n > 0);

Si n fuera siempre divisible por 4, podrías desenrollarlo fácilmente como:

do {
    *ptr++ ^= mask;
    *ptr++ ^= mask;
    *ptr++ ^= mask;
    *ptr++ ^= mask;
} while ((n -= 4) > 0);

Pero, con el Dispositivo de Duff, el código puede seguir este lenguaje de desarrollo que salta al lugar correcto en medio del ciclo si n no es divisible por 4.

switch (n % 4) do {
case 0: *ptr++ ^= mask; /* FALL THROUGH */
case 3: *ptr++ ^= mask; /* FALL THROUGH */
case 2: *ptr++ ^= mask; /* FALL THROUGH */
case 1: *ptr++ ^= mask; /* FALL THROUGH */
} while ((n -= 4) > 0);

Este tipo de desenrollado manual rara vez se requiere con los compiladores modernos, ya que el motor de optimización del compilador puede desenrollar bucles en nombre del programador.
