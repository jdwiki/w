Una afirmación es un predicado de que la condición presentada debe ser verdadera en el momento en que el software encuentra la afirmación. Las más comunes son las afirmaciones simples, que se validan en el momento de la ejecución. Sin embargo, las afirmaciones estáticas se comprueban en tiempo de compilación.

Sintaxis #

  • afirmar (expresión)
  • static_assert(expresión, mensaje)
  • _Static_assert(expresión, mensaje)

Parámetros #

Parámetro Detalles
expresión expresión de tipo escalar.
mensaje literal de cadena que se incluirá en el mensaje de diagnóstico.

Tanto assert como static_assert son macros definidas en assert.h.

La definición de assert depende de la macro NDEBUG que no está definida por la biblioteca estándar. Si se define NDEBUG, assert no es operativo:

#ifdef NDEBUG
#  define assert(condition) ((void) 0)
#else
#  define assert(condition) /* implementation defined */
#endif

La opinión varía sobre si NDEBUG siempre debe usarse para compilaciones de producción.

  • El pro-camp argumenta que las llamadas assert abort y los mensajes de afirmación no son útiles para los usuarios finales, por lo que el resultado no es útil para el usuario. Si tiene condiciones fatales para verificar en el código de producción, debe usar las condiciones ordinarias if/else y exit o quick_exit para finalizar el programa. A diferencia de abort, estos permiten que el programa haga algo de limpieza (a través de funciones registradas con atexit o at_quick_exit).
  • El estafador argumenta que las llamadas ‘afirmar’ nunca deberían dispararse en el código de producción, pero si lo hacen, la condición que está marcada significa que hay algo dramáticamente mal y que el programa se comportará peor si la ejecución continúa. Por lo tanto, es mejor tener las afirmaciones activas en el código de producción porque si se disparan, el infierno ya se ha desatado.
  • Otra opción es usar un sistema casero de aserciones que siempre realiza la verificación pero maneja los errores de manera diferente entre el desarrollo (donde ‘abortar’ es apropiado) y la producción (donde un ’error interno inesperado; comuníquese con el soporte técnico’ puede ser más adecuado).

static_assert se expande a _Static_assert, que es una palabra clave. La condición se comprueba en el momento de la compilación, por lo que condición debe ser una expresión constante. No es necesario que esto se maneje de manera diferente entre el desarrollo y la producción.

Afirmación simple #

Una afirmación es una declaración que se utiliza para afirmar que un hecho debe ser cierto cuando se alcanza esa línea de código. Las aserciones son útiles para asegurar que se cumplen las condiciones esperadas. Cuando la condición pasada a una aserción es verdadera, no hay acción. El comportamiento en condiciones falsas depende de los indicadores del compilador. Cuando las aserciones están habilitadas, una entrada falsa provoca una detención inmediata del programa. Cuando están deshabilitados, no se realiza ninguna acción. Es una práctica común habilitar aserciones en compilaciones internas y de depuración, y deshabilitarlas en compilaciones de versión, aunque las aserciones a menudo se habilitan en la versión. (El hecho de que la terminación sea mejor o peor que los errores depende del programa). Las aserciones deben usarse solo para detectar errores de programación internos, lo que generalmente significa que se pasan parámetros incorrectos.

#include <stdio.h>
/* Uncomment to disable `assert()` */
/* #define NDEBUG */
#include <assert.h>

int main(void)
{
    int x = -1;
    assert(x >= 0);

    printf("x = %d\n", x);   
    return 0;
}

Salida posible con NDEBUG indefinido:

a.out: main.c:9: main: Assertion `x >= 0' failed.

Salida posible con NDEBUG definido:

x = -1

Es una buena práctica definir NDEBUG globalmente, para que pueda compilar fácilmente su código con todas las afirmaciones activadas o desactivadas. Una forma fácil de hacer esto es definir NDEBUG como una opción para el compilador, o definirlo en un encabezado de configuración compartido (por ejemplo, config.h).

Afirmación estática #

Las aserciones estáticas se utilizan para verificar si una condición es verdadera cuando se compila el código. Si no es así, se requiere que el compilador emita un mensaje de error y detenga el proceso de compilación.

Una aserción estática es aquella que se verifica en tiempo de compilación, no en tiempo de ejecución. La condición debe ser una expresión constante y, si es falsa, dará como resultado un error del compilador. El primer argumento, la condición que se verifica, debe ser una expresión constante y el segundo, una cadena literal.

A diferencia de afirmar, _Static_assert es una palabra clave. Una macro de conveniencia static_assert se define en <assert.h>.

#include <assert.h>

enum {N = 5};
_Static_assert(N == 5, "N does not equal 5");
static_assert(N > 10, "N is not greater than 10");  /* compiler error */

Antes de C11, no había apoyo directo para las afirmaciones estáticas. Sin embargo, en C99, las aserciones estáticas podrían emularse con macros que desencadenarían un error de compilación si la condición de tiempo de compilación fuera falsa. A diferencia de _Static_assert, el segundo parámetro debe ser un nombre de token adecuado para que se pueda crear un nombre de variable con él. Si la aserción falla, el nombre de la variable se ve en el error del compilador, ya que esa variable se usó en una declaración de matriz sintácticamente incorrecta.

#define STATIC_MSG(msg, l) STATIC_MSG2(msg, l)
#define STATIC_MSG2(msg,l) on_line_##l##__##msg
#define STATIC_ASSERT(x, msg) extern char STATIC_MSG(msg, __LINE__) [(x)?1:-1]
 
enum { N = 5 };
STATIC_ASSERT(N == 5, N_must_equal_5);
STATIC_ASSERT(N > 5, N_must_be_greater_than_5); /* compile error */

Antes de C99, no podía declarar variables en ubicaciones arbitrarias en un bloque, por lo que tendría que tener mucho cuidado al usar esta macro, asegurándose de que solo aparece donde una declaración de variable sería válida.

Afirmar mensajes de error #

Existe un truco que puede mostrar un mensaje de error junto con una afirmación. Normalmente, escribirías código como este

void f(void *p)
{
    assert(p != NULL);
    /* more code */
}

Si la aserción falla, un mensaje de error se parecería a

Aserción fallida: p != NULL, archivo main.c, línea 5

Sin embargo, también puede usar AND lógico (&&) para dar un mensaje de error

void f(void *p)
{
    assert(p != NULL && "function f: p cannot be NULL");
    /* more code */
}

Ahora, si la afirmación falla, un mensaje de error será algo como esto

Aserción fallida: p != NULL && “función f: p no puede ser NULL”, archivo main.c, línea 5

La razón por la que esto funciona es que un literal de cadena siempre se evalúa como distinto de cero (verdadero). Agregar && 1 a una expresión booleana no tiene ningún efecto. Por lo tanto, agregar && "mensaje de error" tampoco tiene efecto, excepto que el compilador mostrará la expresión completa que falló.

Afirmación de código inalcanzable #

Durante el desarrollo, cuando ciertas rutas de código deben evitarse del alcance del flujo de control, puede usar assert(0) para indicar que tal condición es errónea:

switch (color) {
    case COLOR_RED:
    case COLOR_GREEN:
    case COLOR_BLUE:
        break;

    default:
        assert(0);
}

Siempre que el argumento de la macro assert() se evalúe falso, la macro escribirá información de diagnóstico en el flujo de error estándar y luego cancelará el programa. Esta información incluye el archivo y el número de línea de la sentencia assert() y puede ser muy útil para la depuración. Las afirmaciones se pueden deshabilitar definiendo la macro NDEBUG.

Otra forma de terminar un programa cuando ocurre un error es con las funciones estándar de la biblioteca exit, quick_exit o abort. exit y quick_exit toman un argumento que se puede pasar a su entorno. abort() (y por lo tanto assert) puede ser una terminación realmente severa de su programa, y ​​es posible que no se realicen ciertas limpiezas que de otro modo se realizarían al final de la ejecución.

La principal ventaja de assert() es que imprime automáticamente información de depuración. Llamar a abort() tiene la ventaja de que no se puede deshabilitar como una aserción, pero es posible que no haga que se muestre ninguna información de depuración. En algunas situaciones, usar ambas construcciones juntas puede ser beneficioso:

if (color == COLOR_RED || color == COLOR_GREEN) {
   ...
} else if (color == COLOR_BLUE) {
   ...
} else {
   assert(0), abort();
}

Cuando las afirmaciones están habilitadas, la llamada assert() imprimirá información de depuración y finalizará el programa. La ejecución nunca llega a la llamada abort(). Cuando las afirmaciones están deshabilitadas, la llamada assert() no hace nada y se llama a abort(). Esto asegura que el programa siempre termina por esta condición de error; habilitar y deshabilitar afirmaciones solo afecta si se imprime o no la salida de depuración.

Nunca debe dejar una ‘afirmación’ de este tipo en el código de producción, porque la información de depuración no es útil para los usuarios finales y porque ‘abortar’ es generalmente una terminación demasiado severa que inhibe los controladores de limpieza que están instalados para ‘salir’ o ‘salida rápida’. correr.

Precondición y Postcondición #

Un caso de uso para la aserción es la condición previa y la condición posterior. Esto puede ser muy útil para mantener invariante y diseño por contrato. Por ejemplo, una longitud siempre es cero o positiva, por lo que esta función debe devolver un valor cero o positivo.

#include <stdio.h>
/* Uncomment to disable `assert()` */
/* #define NDEBUG */
#include <assert.h>

int length2 (int *a, int count)
{
    int i, result = 0;

    /* Precondition: */
    /* NULL is an invalid vector */
    assert (a != NULL);
    /* Number of dimensions can not be negative.*/ 
    assert (count >= 0);

    /* Calculation */
    for (i = 0; i < count; ++i) 
    {
        result = result + (a[i] * a[i]);
    }

    /* Postcondition: */
    /* Resulting length can not be negative. */
    assert (result >= 0);
    return result;
}

#define COUNT 3

int main (void)
{
    int a[COUNT] = {1, 2, 3};
    int *b = NULL;
    int r;
    r = length2 (a, COUNT);
    printf ("r = %i\n", r);
    r = length2 (b, COUNT);
    printf ("r = %i\n", r);
    return 0;
}
