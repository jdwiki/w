Los punteros de función son punteros que apuntan a funciones en lugar de tipos de datos. Se pueden usar para permitir la variabilidad en la función que se va a llamar, en tiempo de ejecución.

Sintaxis #

  • returnType (*nombre)(parámetros)

  • typedef returnType (*nombre)(parámetros)

  • typedef returnType Name(parámetros); Nombre nombre;

  • typedef returnType Name(parámetros); typedef Nombre *NombrePtr;

Devolver punteros de función desde una función #

#include <stdio.h>

enum Op
{
  ADD = '+',
  SUB = '-',
};


/* add: add a and b, return result */    
int add(int a, int b)
{
    return a + b;
}

/* sub: subtract b from a, return result */
int sub(int a, int b)
{
    return a - b;
}

/* getmath: return the appropriate math function */
int (*getmath(enum Op op))(int,int)
{
    switch (op)
    {
        case ADD:
            return &add;
        case SUB:
            return &sub;
        default:
            return NULL;
    }
}

int main(void)
{
    int a, b, c;
    int (*fp)(int,int);

    fp = getmath(ADD);

    a = 1, b = 2;
    c = (*fp)(a, b);
    printf("%d + %d = %d\n", a, b, c);
    return 0;
}

Mejores prácticas #

Usando typedef #

Puede ser útil usar un typedef en lugar de declarar el puntero de función cada vez a mano.

La sintaxis para declarar un typedef para un puntero de función es:

typedef returnType (*name)(parameters);

Ejemplo: #

Supongamos que tenemos una función, ordenar, que espera un puntero de función a una función comparar tal que:

compare - Una función de comparación para dos elementos que es para ser suministrado a una función de clasificación.

Se espera que “comparar” devuelva 0 si se consideran los dos elementos igual, un valor positivo si el primer elemento pasado es “más grande” en algunos sentido que el último elemento y, de lo contrario, la función devuelve un valor negativo (lo que significa que el primer elemento es “menor” que el último).

Sin un typedef pasaríamos un puntero de función como argumento a una función de la siguiente manera:

void sort(int (*compare)(const void *elem1, const void *elem2)) { 
    /* inside of this block, the function is named "compare" */
}

Con un typedef, escribiríamos:

typedef int (*compare_func)(const void *, const void *);

y luego podríamos cambiar la firma de la función de ordenar a:

void sort(compare_func func) { 
    /* In this block the function is named "func" */
}

ambas definiciones de ordenar aceptarían cualquier función de la forma

int compare(const void *arg1, const void *arg2) {
    /* Note that the variable names do not have to be "elem1" and "elem2" */
}

Los punteros de función son el único lugar donde debe incluir la propiedad de puntero del tipo, p. no intente definir tipos como typedef struct something_struct *something_type. Esto se aplica incluso a una estructura con miembros a los que se supone que no deben acceder directamente las personas que llaman a la API, por ejemplo, el tipo stdio.h FILE (que, como notará ahora, no es un puntero).

Tomando punteros de contexto. #

Un puntero de función casi siempre debe tomar un void * proporcionado por el usuario como un puntero de contexto.

Ejemplo #

/* function minimiser, details unimportant */
double findminimum( double (*fptr)(double x, double y, void *ctx), void *ctx)
{
    ...
    /* repeatedly make calls like this */
    temp = (*fptr)(testx, testy, ctx);
}

/* the function we are minimising, sums two cubics */
double *cubics(double x, double y, void *ctx)
{
    double *coeffsx = ctx;
    double *coeffsy = coeffx + 4;

    return coeffsx[0] * x * x * x + coeffsx[1] * x * x + coeffsx[2] * x + coeffsx[3] +
           coeffsy[0] * y * y * y + coeffsy[1] * y * y + coeffsy[2] * y + coeffsy[3];

} 

void caller()
{
    /* context, the coefficients of the cubics */
    double coeffs[8] = {1, 2, 3, 4, 5, 6, 7, 8};
    double min;

    min = findminimum(cubics, coeffs);       
}

El uso del puntero de contexto significa que los parámetros adicionales no necesitan estar codificados en la función a la que se apunta, ni requieren el uso de globales.

La función de biblioteca qsort() no sigue esta regla y, a menudo, uno puede quedarse sin contexto para funciones de comparación triviales. Pero para algo más complicado, el puntero de contexto se vuelve esencial.

Introducción #

Al igual que char e int, una función es una característica fundamental de C. Como tal, puede declarar un puntero a uno: lo que significa que puede pasar a qué función llamar a otra función para ayudarla a hacer su trabajo. Por ejemplo, si tuviera una función graph() que mostrara un gráfico, podría pasar cuál función representar gráficamente a graph().

// A couple of external definitions to make the example clearer
extern unsigned int screenWidth;
extern void plotXY(double x, double y);

// The graph() function.
// Pass in the bounds: the minimum and maximum X and Y that should be plotted.
// Also pass in the actual function to plot.
void graph(double minX, double minY,
           double maxX, double maxY,
           ???? *fn) {            // See below for syntax

    double stepX = (maxX - minX) / screenWidth;
    for (double x=minX; x<maxX; x+=stepX) {

        double y = fn(x);         // Get y for this x by calling passed-in fn()

        if (minY<=y && y<maxY) {
            plotXY(x, y);         // Plot calculated point
        } // if
    } for
} // graph(minX, minY, maxX, maxY, fn)

Uso #

Entonces, el código anterior graficará cualquier función que le hayas pasado, siempre y cuando esa función cumpla con ciertos criterios: a saber, que pases un “doble” y obtengas un “doble”. Hay muchas funciones como esa: sin(), cos(), tan(), exp(), etc., pero hay muchas que no lo son, como graph(). !

Sintaxis #

Entonces, ¿cómo especifica qué funciones puede pasar a graph() y cuáles no? La forma convencional es mediante el uso de una sintaxis que puede no ser fácil de leer o comprender:

double (*fn)(double); // fn is a pointer-to-function that takes a double and returns one

El problema anterior es que hay dos cosas que intentan definirse al mismo tiempo: la estructura de la función y el hecho de que es un puntero. Entonces, ¡divide las dos definiciones! Pero al usar typedef, se puede lograr una mejor sintaxis (más fácil de leer y comprender).

Asignar un puntero de función #

#include <stdio.h>

/* increment: take number, increment it by one, and return it */
int increment(int i)
{
    printf("increment %d by 1\n", i);
    return i + 1;
}

/* decrement: take number, decrement it by one, and return it */
int decrement(int i)
{
    printf("decrement %d by 1\n", i);
    return i - 1;
}

int main(void)
{
    int num = 0;          /* declare number to increment */
    int (*fp)(int);       /* declare a function pointer */

    fp = &increment;      /* set function pointer to increment function */
    num = (*fp)(num);     /* increment num */
    num = (*fp)(num);     /* increment num a second time */

    fp = &decrement;      /* set function pointer to decrement function */
    num = (*fp)(num);     /* decrement num */
    printf("num is now: %d\n", num);
    return 0;
}

Mnemónico para escribir punteros de función #

Todas las funciones de C son, en realidad, punteros a un lugar en la memoria del programa donde existe algún código. El uso principal de un puntero de función es proporcionar una “devolución de llamada” a otras funciones (o para simular clases y objetos).

La sintaxis de una función, como se define más abajo en esta página, es:

returnType (*nombre)(parámetros)

Un mnemotécnico para escribir una definición de puntero de función es el siguiente procedimiento:

  1. Comience escribiendo una declaración de función normal: returnType name(parámetros)
  2. Envuelva el nombre de la función con la sintaxis del puntero: returnType (*nombre)(parámetros)

Conceptos básicos #

Al igual que puede tener un puntero a int, char, float, array/string, struct, etc., puede tener un puntero a Una función.

Declarar el puntero toma el valor de retorno de la función, el nombre de la función y el tipo de argumentos/parámetros que recibe.

Digamos que tiene la siguiente función declarada e inicializada:

int addInt(int n, int m){
    return n+m;
}

Puede declarar e inicializar un puntero a esta función:

int (*functionPtrAdd)(int, int) = addInt; // or &addInt - the & is optional

Si tiene una función void, podría verse así:

void Print(void){
    printf("look ma' - no hands, only pointers!\n");
}

Entonces, declarar el puntero sería:

void (*functionPtrPrint)(void) = Print;

Acceder a la función en sí requeriría desreferenciar el puntero:

sum = (*functionPtrAdd)(2, 3); //will assign 5 to sum
(*functionPtrPrint)(); //will print the text in Print function

Como se ve en ejemplos más avanzados en este documento, la declaración de un puntero a una función podría complicarse si a la función se le pasan más de unos pocos parámetros. Si tiene algunos punteros a funciones que tienen una “estructura” idéntica (el mismo tipo de valor devuelto y el mismo tipo de parámetros), es mejor usar el comando typedef para ahorrar algo de escritura y hacer que el código sea más claro:

typedef int (*ptrInt)(int, int);

int Add(int i, int j){
    return i+j;
}

int Multiply(int i, int j){
    return i*j;
}

int main()
{
    ptrInt ptr1 = Add;
    ptrInt ptr2 = Multiply;

    printf("%d\n", (*ptr1)(2,3)); //will print 5
    printf("%d\n", (*ptr2)(2,3)); //will print 6
    return 0;
}

También puede crear una matriz de punteros de función. Si todos los punteros son de la misma “estructura”:

int (*array[2]) (int x, int y); // can hold 2 function pointers
array[0] = Add;
array[1] = Multiply;

También es posible definir una matriz de punteros de función de diferentes tipos, aunque eso requeriría una conversión cada vez que desee acceder a la función específica. Puede obtener más información aquí.
