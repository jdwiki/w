Especificadores de conversión para impresión #

Especificador de conversión Tipo de Argumento Descripción
i, d entero imprime decimal
u int sin firmar imprime decimal
o int sin firmar impresiones octales
x int sin firmar imprime hexadecimal, minúsculas
X int sin firmar imprime hexadecimal, mayúsculas
f doble imprime float con una precisión predeterminada de 6, si no se proporciona ninguna precisión (se usan minúsculas para los números especiales nan e inf o infinity)
F doble imprime float con una precisión predeterminada de 6, si no se proporciona precisión (se usa mayúscula para los números especiales NAN e INF o INFINITY)
e doble imprime float con una precisión predeterminada de 6, si no se proporciona precisión, usando notación científica usando mantisa/exponente; exponente en minúsculas y números especiales
E doble imprime float con una precisión predeterminada de 6, si no se proporciona precisión, usando notación científica usando mantisa/exponente; exponente en mayúsculas y números especiales
g doble usa f o e [ver abajo]
G doble usa F o E [ver abajo]
a doble imprime hexadecimal, minúsculas
A doble imprime hexadecimal, mayúsculas
c carbón imprime un solo carácter
s char* imprime una cadena de caracteres hasta un terminador NUL, o se trunca a la longitud dada por precisión, si se especifica
p vacío* imprime el valor del puntero void; un puntero que no sea void debe convertirse explícitamente (“cast”) a void*; puntero a objeto solamente, no un puntero de función
% n/a imprime el caracter %
n entero * escriba el número de bytes impresos hasta el momento en el int apuntado.

Tenga en cuenta que los modificadores de longitud se pueden aplicar a %n (por ejemplo, %hhn indica que un especificador de conversión siguiente n se aplica a un puntero a un caracter firmado argument, según ISO/IEC 9899:2011 §7.21.6.1 ¶7).

Tenga en cuenta que las conversiones de punto flotante se aplican a los tipos flotante y doble debido a las reglas de promoción predeterminadas: §6.5.2.2 Llamadas a funciones, ¶7 La notación de puntos suspensivos en un declarador de prototipo de función causa conversión de tipo de argumento para detenerse después del último parámetro declarado. El argumento predeterminado las promociones se realizan en los argumentos finales.) Por lo tanto, funciones como printf() solo se pasan valores dobles, incluso si la variable a la que se hace referencia es del tipo float.

Con los formatos g y G, la elección entre la notación e y f (o E y F) está documentada en el estándar C y en la especificación POSIX para printf() :

El argumento doble que representa un número de punto flotante se convertirá en el estilo f o e (o en el estilo F o E en el caso de un especificador de conversión G), según el valor convertido y la precisión. Sea P igual a la precisión si no es cero, 6 si se omite la precisión o 1 si la precisión es cero. Entonces, si una conversión con estilo E tuviera un exponente de X:

  • Si P > X >= -4, la conversión será con estilo f (o F) y precisión P - (X+1).
  • En caso contrario, la conversión será con estilo e (o E) y precisión P - 1.

Finalmente, a menos que se use el indicador ‘#’, se eliminarán los ceros finales de la parte fraccionaria del resultado y el carácter de punto decimal se eliminará si no queda ninguna parte fraccionaria.

Imprimir el valor de un puntero a un objeto #

Para imprimir el valor de un puntero a un objeto (a diferencia de un puntero de función), utilice el especificador de conversión p. Está definido para imprimir solo punteros vacíos, por lo que para imprimir el valor de un puntero que no sea vacío, debe convertirse explícitamente (“casted*”) a vacío*.

#include <stdlib.h> /* for EXIT_SUCCESS */
#include <stdio.h>  /* for printf() */

int main(void)
{
  int i;
  int * p = &i;

  printf("The address of i is %p.\n", (void*) p);

  return EXIT_SUCCESS;
}

Usando <inttypes.h> y uintptr_t #

Otra forma de imprimir punteros en C99 o posterior usa el tipo uintptr_t y las macros de <inttypes.h>:

#include <inttypes.h> /* for uintptr_t and PRIXPTR */
#include <stdio.h>    /* for printf() */

int main(void)
{
  int  i;
  int *p = &i;

  printf("The address of i is 0x%" PRIXPTR ".\n", (uintptr_t)p);

  return 0;
}

En teoría, es posible que no haya un tipo entero que pueda contener cualquier puntero convertido a un número entero (por lo que el tipo uintptr_t podría no existir). En la práctica, existe. Los punteros a las funciones no necesitan ser convertibles al tipo uintptr_t — aunque de nuevo, la mayoría de las veces son convertibles.

Si existe el tipo uintptr_t, también existe el tipo intptr_t. Sin embargo, no está claro por qué querrías tratar las direcciones como enteros con signo.

Historia previa al estándar: #

Antes de C89 durante los tiempos de K&R-C no había ningún tipo void* (ni encabezado <stdlib.h>, ni prototipos, y por lo tanto no había notación int main(void)), por lo que el puntero se convertía en long unsigned int e impreso usando el especificador de conversión/modificador de longitud lx.

El siguiente ejemplo es solo para fines informativos. Hoy en día, este es un código no válido, que muy bien podría provocar el infame Comportamiento indefinido.

#include <stdio.h> /* optional in pre-standard C - for printf() */

int main()
{
  int  i;
  int *p = &i;

  printf("The address of i is 0x%lx.\n", (long unsigned) p);

  return 0;
}

Imprimiendo la diferencia de los valores de dos punteros a un objeto #

Restar los valores de dos punteros a un objeto da como resultado un entero con signo *1. Por lo tanto, se imprimiría usando al menos el especificador de conversión d.

Para asegurarse de que haya un tipo lo suficientemente ancho como para contener tal “diferencia de puntero”, ya que C99 <stddef.h> define el tipo ptrdiff_t. Para imprimir un ptrdiff_t use el modificador de longitud t.

#include <stdlib.h> /* for EXIT_SUCCESS */
#include <stdio.h> /* for printf() */
#include <stddef.h> /* for ptrdiff_t */


int main(void)
{
  int a[2];
  int * p1 = &a[0], * p2 = &a[1];
  ptrdiff_t pd = p2 - p1;

  printf("p1 = %p\n", (void*) p1);
  printf("p2 = %p\n", (void*) p2);
  printf("p2 - p1 = %td\n", pd);

  return EXIT_SUCCESS;
}

El resultado podría verse así:

p1 = 0x7fff6679f430
p2 = 0x7fff6679f434
p2 - p1 = 1

Tenga en cuenta que el valor resultante de la diferencia se escala según el tamaño del tipo al que se restaron los punteros, un int aquí. El tamaño de un int para este ejemplo es 4.

*1Si los dos punteros a restar no apuntan al mismo objeto, el comportamiento no está definido.

1: https://www.wikiod.com/es/c/operadores#Aritmética de punteros

La función imprimirf() #

A la que se accede incluyendo <stdio.h>, la función printf() es la herramienta principal utilizada para imprimir texto en la consola en C.

printf("Hello world!");
// Hello world!

Las matrices de caracteres normales sin formato se pueden imprimir solas colocándolas directamente entre paréntesis.

printf("%d is the answer to life, the universe, and everything.", 42);
// 42 is the answer to life, the universe, and everything.

int x = 3;
char y = 'Z';
char* z = "Example";
printf("Int: %d, Char: %c, String: %s", x, y, z);
// Int: 3, Char: Z, String: Example

Como alternativa, los números enteros, los números de punto flotante, los caracteres y más se pueden imprimir utilizando el carácter de escape %, seguido de un carácter o secuencia de caracteres que indican el formato, conocido como especificador de formato.

Todos los argumentos adicionales a la función printf() están separados por comas, y estos argumentos deben estar en el mismo orden que los especificadores de formato. Los argumentos adicionales se ignoran, mientras que los argumentos escritos incorrectamente o la falta de argumentos provocarán errores o un comportamiento indefinido. Cada argumento puede ser un valor literal o una variable.

Después de una ejecución exitosa, el número de caracteres impresos se devuelve con el tipo int. De lo contrario, un error devuelve un valor negativo.

Banderas de formato de impresión #

El estándar C (C11 y C99 también) define los siguientes indicadores para printf():

Bandera Conversiones Sentido
- todos El resultado de la conversión se justificará a la izquierda dentro del campo. La conversión se justifica a la derecha si no se especifica este indicador.
+ numérico firmado El resultado de una conversión firmada siempre comenzará con un signo (’+’ o ‘-’). La conversión comenzará con un signo solo cuando se convierta un valor negativo si no se especifica esta bandera.
<espacio> numérico firmado Si el primer carácter de una conversión firmada no es un signo o si una conversión firmada no da como resultado ningún carácter, se antepondrá un <espacio> al resultado. Esto significa que si aparecen los indicadores <espacio> y ``+', se ignorará el indicador `.
# todos Especifica que el valor se va a convertir a una forma alternativa. Para la conversión o, aumentará la precisión, si y solo si es necesario, para forzar que el primer dígito del resultado sea cero (si el valor y la precisión son ambos 0, se imprime un solo 0). Para los especificadores de conversión x o X, un resultado distinto de cero tendrá un prefijo 0x (o 0X). Para los especificadores de conversión a, A, e, E, f, F, g y G, el resultado siempre contendrá un carácter de raíz, incluso si no sigue ningún dígito. el carácter radix. Sin esta bandera, un carácter de raíz aparece en el resultado de estas conversiones solo si lo sigue un dígito. Para los especificadores de conversión g y G, los ceros finales no se eliminarán del resultado como se hace normalmente. Para otros especificadores de conversión, el comportamiento no está definido.
0 numérico Para los especificadores de conversión d, i, o, u, x, X, a, A, e, E, f, F, g y G, se utilizan ceros iniciales (después de cualquier indicación de signo o base) para completar el campo. width en lugar de realizar relleno de espacio, excepto cuando se convierte un infinito o NaN. Si aparecen los indicadores ‘0’ y ‘-’, se ignora el indicador ‘0’. Para los especificadores de conversión d, i, o, u, x y X, si se especifica una precisión, se ignorará la bandera ‘0’. ⌦ Si aparecen los indicadores ‘0’ y <apóstrofe>, los caracteres de agrupación se insertan antes del relleno con ceros. Para otras conversiones, el comportamiento no está definido. ⌫

Estos indicadores también son compatibles con Microsoft con los mismos significados.

La especificación POSIX para printf() agrega:

Bandera Conversiones Sentido
' yo, re, tu, f, F, g, G La parte entera del resultado de una conversión decimal se formateará con caracteres de agrupación de miles. Para otras conversiones, el comportamiento no está definido. Se utiliza el carácter de agrupación no monetaria.

Modificadores de longitud #

Los estándares C99 y C11 especifican los siguientes modificadores de longitud para printf(); sus significados son:

| Modificador | Modifica | Se aplica a | |———-|————————————-|—————- ———————————————–| | hh | d, i, o, u, x o X | char, char firmado o char sin firmar | h | d, i, o, u, x o X | int corto o int corto sin signo | yo | d, i, o, u, x o X | int largo o int largo sin signo | yo | a, A, e, E, f, F, g o G | doble (por compatibilidad con scanf(); indefinido en C90) | todos | d, i, o, u, x o X | int largo largo o int largo largo sin signo | j | d, i, o, u, x o X | intmax_t o uintmax_t | z | d, i, o, u, x o X | size_t o el tipo firmado correspondiente (ssize_t en POSIX) | t | d, i, o, u, x o X | ptrdiff_t o el tipo entero sin signo correspondiente | L | a, A, e, E, f, F, g o G | doble largo

Si aparece un modificador de longitud con cualquier especificador de conversión que no sea el especificado anteriormente, el comportamiento no está definido.

Microsoft especifica algunos modificadores de longitud diferentes y explícitamente no admite hh, j, z o t .

| Modificador | Modifica | Se aplica a | |———-|————————————-|—————- ———————————————–| | I32 | d, i, o, x o X | __int32 | I32 | o, u, x o X | sin firmar __int32 | I64 | d, i, o, x o X | __int64 | I64 | o, u, x o X | sin firmar __int64 | yo | d, i, o, x o X | ptrdiff_t (es decir, __int32 en plataformas de 32 bits, __int64 en plataformas de 64 bits) | yo | o, u, x o X | size_t (es decir, __int32 sin firmar en plataformas de 32 bits, __int64 sin firmar en plataformas de 64 bits) | l o L | a, A, e, E, f, g o G | doble largo (En Visual C++, aunque doble largo es un tipo distinto, tiene la misma representación interna que doble). | l o w | c o C | Carácter ancho con funciones printf y wprintf. (Un especificador de tipo lc, lC, wc o wC es sinónimo de C en las funciones printf y de c en las funciones wprintf.) | l o w | s, S o Z | Cadena de caracteres anchos con funciones printf y wprintf. (Un especificador de tipo ls, lS, ws o wS es sinónimo de S en las funciones printf y de s en las funciones wprintf.)

Tenga en cuenta que los especificadores de conversión C, S y Z y los modificadores de longitud I, I32, I64 y w son extensiones de Microsoft. Tratar a ’l’ como un modificador de ‘doble largo’ en lugar de ‘doble’ es diferente del estándar, aunque será difícil detectar la diferencia a menos que ‘doble largo’ tenga una representación diferente de ‘doble’.
