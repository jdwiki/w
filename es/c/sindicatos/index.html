Usar uniones para reinterpretar valores #

Algunas implementaciones de C permiten que el código escriba en un miembro de un tipo de unión y luego lea de otro para realizar una especie de conversión de reinterpretación (analizar el nuevo tipo como la representación de bits del anterior).

Sin embargo, es importante tener en cuenta que esto no está permitido por el estándar C actual o pasado y dará como resultado un comportamiento indefinido, sin embargo, es una extensión muy común ofrecida por los compiladores (así que consulte los documentos del compilador si planea hacer esto) .

Un ejemplo de la vida real de esta técnica es el algoritmo “Raíz cuadrada inversa rápida” que se basa en los detalles de implementación de los números de punto flotante IEEE 754 para realizar una raíz cuadrada inversa más rápidamente que usando operaciones de punto flotante, este algoritmo se puede realizar a través de la conversión de punteros (que es muy peligroso y rompe la estricta regla de alias) o a través de una unión (que aún es un comportamiento indefinido pero funciona en muchos compiladores):

union floatToInt
{
    int32_t intMember;
    float floatMember; /* Float must be 32 bits IEEE 754 for this to work */
};

float inverseSquareRoot(float input)
{
    union floatToInt x;
    int32_t i;
    float f;
    x.floatMember = input;     /* Assign to the float member */
    i = x.intMember;           /* Read back from the integer member */
    i = 0x5f3759df - (i >> 1);
    x.intMember = i;           /* Assign to the integer member */
    f = x.floatMember;         /* Read back from the float member */
    f = f * (1.5f - input * 0.5f * f * f);
    return f * (1.5f - input * 0.5f * f * f);
}

Esta técnica se usó ampliamente en gráficos y juegos de computadora en el pasado debido a su mayor velocidad en comparación con el uso de operaciones de punto flotante, y es en gran medida un compromiso, ya que pierde algo de precisión y es muy poco portátil a cambio de velocidad.

Escribir a un miembro del sindicato y leer de otro #

Los miembros de un sindicato comparten el mismo espacio en la memoria. Esto significa que escribir en un miembro sobrescribe los datos en todos los demás miembros y que la lectura de un miembro da como resultado los mismos datos que la lectura de todos los demás miembros. Sin embargo, debido a que los miembros de unión pueden tener diferentes tipos y tamaños, los datos que se leen pueden interpretarse de manera diferente, consulte https://www.wikiod.com/es/c/estructuras#Usar sindicatos para reinterpretar valores -reinterpretar-valores

El sencillo ejemplo siguiente muestra una unión con dos miembros, ambos del mismo tipo. Muestra que escribir en el miembro m_1 da como resultado que el valor escrito se lea del miembro m_2 y escribir en el miembro m_2 da como resultado que el valor escrito se lea del miembro m_1.

#include <stdio.h>

union my_union /* Define union */
{
    int m_1;
    int m_2;
};

int main (void)
{
    union my_union u;             /* Declare union */
    u.m_1 = 1;                    /* Write to m_1 */
    printf("u.m_2: %i\n", u.m_2); /* Read from m_2 */
    u.m_2 = 2;                    /* Write to m_2 */
    printf("u.m_1: %i\n", u.m_1); /* Read from m_1 */
    return 0;
}

Resultado

u.m_2: 1
u.m_1: 2
