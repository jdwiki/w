Los objetos de cadena son inmutables, lo que significa que no se pueden modificar en su lugar como se puede hacer con una lista. Debido a esto, los métodos en el tipo integrado str siempre devuelven un nuevo objeto str, que contiene el resultado de la llamada al método.

Cambiando las mayúsculas de una cadena #

El tipo de cadena de Python proporciona muchas funciones que actúan sobre las mayúsculas de una cadena. Éstos incluyen :

  • str.casefold
  • str.superior
  • str.inferior
  • str.mayúsculas
  • str.título
  • str.swapcase

Con cadenas Unicode (el valor predeterminado en Python 3), estas operaciones no son asignaciones 1:1 o reversibles. La mayoría de estas operaciones están destinadas a fines de visualización, en lugar de normalización.

str.casefold() #

str.casefold crea una cadena en minúsculas que es adecuada para comparaciones que no distinguen entre mayúsculas y minúsculas. Esto es más agresivo que str.lower y puede modificar las cadenas que ya están en minúsculas o hacer que las cadenas aumenten de longitud, y no está diseñado para fines de visualización.

"XßΣ".casefold()
# 'xssσ'

"XßΣ".lower()
# 'xßς'

El Consorcio Unicode define las transformaciones que tienen lugar en el caso de plegamiento en el archivo CaseFolding.txt en su sitio web.

str.superior() #

str.upper toma todos los caracteres de una cadena y los convierte a su equivalente en mayúsculas, por ejemplo:

"This is a 'string'.".upper()
# "THIS IS A 'STRING'."

str.inferior() #

str.lower hace lo contrario; toma todos los caracteres de una cadena y los convierte a su equivalente en minúsculas:

"This IS a 'string'.".lower()
# "this is a 'string'."

str.mayúsculas() #

str.capitalize devuelve una versión en mayúscula de la cadena, es decir, hace que el primer carácter esté en mayúsculas y el resto en minúsculas:

"this Is A 'String'.".capitalize() # Capitalizes the first character and lowercases all others
# "This is a 'string'."

str.título() #

str.title devuelve la versión en mayúsculas y minúsculas del título de la cadena, es decir, cada letra al comienzo de una palabra está en mayúsculas y todas las demás en minúsculas:

"this Is a 'String'".title()
# "This Is A 'String'"

str.swapcase() #

str.swapcase devuelve un nuevo objeto de cadena en el que todos los caracteres en minúsculas se intercambian a mayúsculas y todos los caracteres en mayúsculas a minúsculas:

"this iS A STRiNG".swapcase() #Swaps case of each character
# "THIS Is a strIng"

Uso como métodos de clase str #

Vale la pena señalar que estos métodos pueden llamarse en objetos de cadena (como se muestra arriba) o como un método de clase de la clase str (con una llamada explícita a str.upper, etc.)

str.upper("This is a 'string'")
# "THIS IS A 'STRING'"

Esto es más útil cuando se aplica uno de estos métodos a muchas cadenas a la vez, por ejemplo, en una función map.

map(str.upper,["These","are","some","'strings'"])
# ['THESE', 'ARE', 'SOME', "'STRINGS'"]

str.translate: Traducir caracteres en una cadena #

Python admite un método translate en el tipo str que le permite especificar la tabla de traducción (utilizada para los reemplazos), así como cualquier carácter que deba eliminarse en el proceso.

str.translate(table[, deletechars])
Parámetro Descripción
tabla Es una tabla de búsqueda que define la asignación de un carácter a otro.
eliminar caracteres Una lista de caracteres que se eliminarán de la cadena.

El método maketrans (str.maketrans en Python 3 y string.maketrans en Python 2) le permite generar una tabla de traducción.

>>> translation_table = str.maketrans("aeiou", "12345")
>>> my_string = "This is a string!"
>>> translated = my_string.translate(translation_table)
'Th3s 3s 1 str3ng!'

El método translate devuelve una cadena que es una copia traducida de la cadena original.

Puede establecer el argumento tabla en Ninguno si solo necesita eliminar caracteres.

>>> 'this syntax is very useful'.translate(None, 'aeiou')
'ths syntx s vry sfl'

str.format y f-strings: formato de valores en una cadena #

Python proporciona interpolación de cadenas y funcionalidad de formato a través de la función str.format, introducida en la versión 2.6 y f-strings introducida en la versión 3.6.

Dadas las siguientes variables:

i = 10
f = 1.5
s = "foo"
l = ['a', 1, 2]
d = {'a': 1, 2: 'foo'}

Las siguientes declaraciones son todas equivalentes

"10 1.5 foo ['a', 1, 2] {'a': 1, 2: 'foo'}"

>>> "{} {} {} {} {}".format(i, f, s, l, d)

>>> str.format("{} {} {} {} {}", i, f, s, l, d)

>>> "{0} {1} {2} {3} {4}".format(i, f, s, l, d)

>>> "{0:d} {1:0.1f} {2} {3!r} {4!r}".format(i, f, s, l, d)

>>> "{i:d} {f:0.1f} {s} {l!r} {d!r}".format(i=i, f=f, s=s, l=l, d=d)

>>> f"{i} {f} {s} {l} {d}"

>>> f"{i:d} {f:0.1f} {s} {l!r} {d!r}"

Como referencia, Python también admite calificadores de estilo C para el formato de cadena. Los ejemplos a continuación son equivalentes a los anteriores, pero se prefieren las versiones str.format debido a los beneficios en flexibilidad, consistencia de la notación y extensibilidad:

"%d %0.1f %s %r %r" % (i, f, s, l, d)

"%(i)d %(f)0.1f %(s)s %(l)r %(d)r" % dict(i=i, f=f, s=s, l=l, d=d)

Los usos de llaves para la interpolación en str.format también se pueden numerar para reducir la duplicación al formatear cadenas. Por ejemplo, los siguientes son equivalentes:

"I am from Australia. I love cupcakes from Australia!"

>>> "I am from {}. I love cupcakes from {}!".format("Australia", "Australia")

>>> "I am from {0}. I love cupcakes from {0}!".format("Australia")

Si bien la documentación oficial de Python es, como de costumbre, lo suficientemente completa, pyformat.info tiene un gran conjunto de ejemplos con explicaciones detalladas.

Además, los caracteres { y } se pueden escapar usando corchetes dobles:

"{'a': 5, 'b': 6}"

>>> "{{'{}': {}, '{}': {}}}".format("a", 5, "b", 6)

>>> f"{{'{'a'}': {5}, '{'b'}': {6}}"

Consulte Formato de cadenas para obtener información adicional. str.format() fue propuesto en PEP 3101 y f-strings en PEP 498.

Dividir una cadena basada en un delimitador en una lista de cadenas #

str.split(sep=Ninguno, maxsplit=-1) #

str.split toma una cadena y devuelve una lista de subcadenas de la cadena original. El comportamiento difiere dependiendo de si se proporciona u omite el argumento sep.

Si no se proporciona sep, o si es Ninguno, entonces la división se lleva a cabo donde haya espacios en blanco. Sin embargo, los espacios en blanco iniciales y finales se ignoran, y varios caracteres de espacios en blanco consecutivos se tratan de la misma manera que un solo carácter de espacio en blanco:

>>> "This is a sentence.".split()
['This', 'is', 'a', 'sentence.']

>>> " This is    a sentence.  ".split()
['This', 'is', 'a', 'sentence.']

>>> "            ".split()
[]

El parámetro sep se puede utilizar para definir una cadena delimitadora. La cadena original se divide donde aparece la cadena delimitadora y el delimitador mismo se descarta. Múltiples delimitadores consecutivos no se tratan igual que una única aparición, sino que provocan la creación de cadenas vacías.

>>> "This is a sentence.".split(' ')
['This', 'is', 'a', 'sentence.']

>>> "Earth,Stars,Sun,Moon".split(',')
['Earth', 'Stars', 'Sun', 'Moon']

>>> " This is    a sentence.  ".split(' ')
['', 'This', 'is', '', '', '', 'a', 'sentence.', '', '']

>>> "This is a sentence.".split('e')
['This is a s', 'nt', 'nc', '.']

>>> "This is a sentence.".split('en')
['This is a s', 't', 'ce.']

El valor predeterminado es dividir en cada ocurrencia del delimitador, sin embargo, el parámetro maxsplit limita el número de divisiones que ocurren. El valor predeterminado de -1 significa que no hay límite:

>>> "This is a sentence.".split('e', maxsplit=0)
['This is a sentence.']

>>> "This is a sentence.".split('e', maxsplit=1)
['This is a s', 'ntence.']

>>> "This is a sentence.".split('e', maxsplit=2)
['This is a s', 'nt', 'nce.']

>>> "This is a sentence.".split('e', maxsplit=-1)
['This is a s', 'nt', 'nc', '.']

str.rsplit(sep=Ninguno, maxsplit=-1) #

str.rsplit (“división a la derecha”) difiere de str.split (“división a la izquierda”) cuando se especifica maxsplit. La división comienza al final de la cadena en lugar de al principio:

>>> "This is a sentence.".rsplit('e', maxsplit=1)
['This is a sentenc', '.']

>>> "This is a sentence.".rsplit('e', maxsplit=2)
['This is a sent', 'nc', '.']

Nota: Python especifica la cantidad máxima de divisiones realizadas, mientras que la mayoría de los otros lenguajes de programación especifican la cantidad máxima de subcadenas creadas. Esto puede crear confusión al portar o comparar código.

Reemplazar todas las apariciones de una subcadena con otra subcadena #

El tipo str de Python también tiene un método para reemplazar las ocurrencias de una subcadena con otra subcadena en una cadena dada. Para casos más exigentes, se puede usar re.sub.

str.replace(antiguo, nuevo[, cuenta]): #

str.replace toma dos argumentos old y new que contienen la subcadena old que debe ser reemplazada por la subcadena new. El argumento opcional count especifica el número de reemplazos a realizar:

Por ejemplo, para reemplazar 'foo' con 'spam' en la siguiente cadena, podemos llamar a str.replace con old = 'foo' y new = 'spam':

>>> "Make sure to foo your sentence.".replace('foo', 'spam')
"Make sure to spam your sentence."

Si la cadena dada contiene varios ejemplos que coinciden con el argumento antiguo, todas las apariciones se reemplazan con el valor proporcionado en nuevo:

>>> "It can foo multiple examples of foo if you want.".replace('foo', 'spam')
"It can spam multiple examples of spam if you want."

a menos, por supuesto, que proporcionemos un valor para count. En este caso, las ocurrencias count serán reemplazadas:

>>> """It can foo multiple examples of foo if you want, \
... or you can limit the foo with the third argument.""".replace('foo', 'spam', 1)
'It can spam multiple examples of foo if you want, or you can limit the foo with the third argument.'

Probando de qué está compuesta una cadena #

El tipo str de Python también presenta una serie de métodos que se pueden usar para evaluar el contenido de una cadena. Estos son str.isalpha, str.isdigit, str.isalnum, str.isspace. Las mayúsculas se pueden probar con str.isupper, str.islower y str.istitle.

str.isalpha #

str.isalpha no toma argumentos y devuelve True si todos los caracteres en una cadena dada son alfabéticos, por ejemplo:

>>> "Hello World".isalpha()  # contains a space
False
>>> "Hello2World".isalpha()  # contains a number
False
>>> "HelloWorld!".isalpha()  # contains punctuation
False
>>> "HelloWorld".isalpha()
True

Como caso límite, la cadena vacía se evalúa como False cuando se usa con "".isalpha().

str.isupper, str.islower, str.istitle #

Estos métodos prueban el uso de mayúsculas en una cadena determinada.

str.isupper es un método que devuelve Verdadero si todos los caracteres de una cadena determinada están en mayúsculas y Falso en caso contrario.

>>> "HeLLO WORLD".isupper()
False
>>> "HELLO WORLD".isupper()
True
>>> "".isupper()
False

Por el contrario, ‘str.islower’ es un método que devuelve ‘Verdadero’ si todos los caracteres de una cadena determinada están en minúsculas y ‘Falso’ en caso contrario.

>>> "Hello world".islower()
False
>>> "hello world".islower()
True
>>> "".islower()
False

str.istitle devuelve True si la cadena dada está en mayúsculas y minúsculas; es decir, cada palabra comienza con un carácter en mayúscula seguido de caracteres en minúscula.

>>> "hello world".istitle()
False
>>> "Hello world".istitle()
False
>>> "Hello World".istitle()
True
>>> "".istitle()
False

str.isdecimal, str.isdigit, str.isnumeric #

str.isdecimal devuelve si la cadena es una secuencia de dígitos decimales, adecuada para representar un número decimal.

str.isdigit incluye dígitos que no están en una forma adecuada para representar un decimal número, como dígitos en superíndice.

str.isnumeric incluye cualquier valor numérico, incluso si no son dígitos, como valores fuera del rango 0-9.

            isdecimal    isdigit   isnumeric

12345        True        True       True
១2߃໔5        True        True       True
①²³🄅₅       False       True       True
⑩⒓          False       False      True
Five         False       False      False

Las cadenas de bytes (bytes en Python 3, str en Python 2), solo admiten isdigit, que solo verifica los dígitos ASCII básicos.

Al igual que con str.isalpha, la cadena vacía se evalúa como False.

str.isalnum #

Esta es una combinación de str.isalpha y str.isnumeric, específicamente se evalúa como True si todos los caracteres en la cadena dada son alfanuméricos, es decir, consisten en caracteres alfabéticos o numéricos :

>>> "Hello2World".isalnum()
True
>>> "HelloWorld".isalnum()
True
>>> "2016".isalnum()
True
>>> "Hello World".isalnum()  # contains whitespace
False

str.isspace #

Se evalúa como Verdadero si la cadena contiene solo caracteres de espacio en blanco.

>>> "\t\r\n".isspace()
True
>>> " ".isspace()
True

A veces, una cadena parece “vacía”, pero no sabemos si es porque contiene solo espacios en blanco o ningún carácter.

>>> "".isspace()
False

Para cubrir este caso necesitamos una prueba adicional

>>> my_str = ''
>>> my_str.isspace()
False
>>> my_str.isspace() or not my_str
True

Pero la forma más corta de probar si una cadena está vacía o solo contiene espacios en blanco es usar strip (sin argumentos, elimina todos los espacios en blanco iniciales y finales)

>>> not my_str.strip()
True

Eliminación de caracteres iniciales/posteriores no deseados de una cadena #

Se proporcionan tres métodos que ofrecen la posibilidad de eliminar los caracteres iniciales y finales de una cadena: str.strip, str.rstrip y str.lstrip. Los tres métodos tienen la misma firma y los tres devuelven un nuevo objeto de cadena con los caracteres no deseados eliminados.

str.strip([caracteres]) #

str.strip actúa en una cadena determinada y elimina (despoja) cualquier carácter inicial o final contenido en el argumento chars; si chars no se proporciona o es Ninguno, todos los espacios en blanco se eliminan de forma predeterminada. Por ejemplo:

>>> "    a line with leading and trailing space     ".strip() 
'a line with leading and trailing space'

Si se proporciona chars, todos los caracteres que contiene se eliminan de la cadena, que se devuelve. Por ejemplo:

>>> ">>> a Python prompt".strip('> ')  # strips '>' character and space character 
'a Python prompt'

str.rstrip([caracteres]) y str.lstrip([caracteres]) #

Estos métodos tienen una semántica y argumentos similares con str.strip(), su diferencia radica en la dirección desde la que comienzan. str.rstrip() comienza desde el final de la cadena mientras que str.lstrip() se divide desde el principio de la cadena.

Por ejemplo, usando str.rstrip:

>>> "     spacious string      ".rstrip()
'     spacious string'

Mientras, usando str.lstrip:

>>> "     spacious string      ".rstrip()
'spacious string      '

Invertir una cadena #

Una cadena puede invertirse usando la función incorporada reversed(), que toma una cadena y devuelve un iterador en orden inverso.

>>> reversed('hello')
<reversed object at 0x0000000000000000>
>>> [char for char in reversed('hello')]
['o', 'l', 'l', 'e', 'h']

reversed() se puede incluir en una llamada a ''.join() para crear una cadena desde el iterador.

>>> ''.join(reversed('hello'))
'olleh'

Mientras que usar reversed() podría ser más legible para los usuarios de Python no iniciados, usar slicing extendido con un paso de -1 es más rápido y más conciso. Aquí, intenta implementarlo como función:

>>> def reversed_string(main_string):
...     return main_string[::-1]
...
>>> reversed_string('hello')
'olleh'

Unir una lista de cadenas en una sola cadena #

Una cadena se puede utilizar como separador para unir una lista de cadenas en un cadena única usando el método join(). Por ejemplo, puede crear una cadena donde cada elemento de una lista está separado por un espacio.

>>> " ".join(["once","upon","a","time"])
"once upon a time"

El siguiente ejemplo separa los elementos de cadena con tres guiones.

>>> "---".join(["once", "upon", "a", "time"])
"once---upon---a---time"

La cadena contiene #

Python hace que sea extremadamente intuitivo verificar si una cadena contiene una subcadena determinada. Solo usa el operador in:

>>> "foo" in "foo.baz.bar"
True

Nota: probar una cadena vacía siempre dará como resultado Verdadero:

>>> "" in "test"
True

Contar el número de veces que aparece una subcadena en una cadena #

Hay un método disponible para contar el número de ocurrencias de una subcadena en otra cadena, str.count.

str.count(sub[, inicio[, fin]]) #

str.count devuelve un int que indica el número de ocurrencias no superpuestas de la subcadena sub en otra cadena. Los argumentos opcionales start y end indican el principio y el final en el que se realizará la búsqueda. De forma predeterminada, start = 0 y end = len(str), lo que significa que se buscará en toda la cadena:

>>> s = "She sells seashells by the seashore."
>>> s.count("sh")
2
>>> s.count("se")
3
>>> s.count("sea")
2
>>> s.count("seashells")
1

Al especificar un valor diferente para start, end podemos obtener una búsqueda y un recuento más localizados, por ejemplo, si start es igual a 13, la llamada a:

>>> s.count("sea", start)
1

es equivalente a:

>>> t = s[start:]
>>> t.count("sea")
1

Comparaciones de cadenas que no distinguen entre mayúsculas y minúsculas #

Comparar cadenas sin distinguir entre mayúsculas y minúsculas parece algo trivial, pero no lo es. Esta sección solo considera cadenas Unicode (el valor predeterminado en Python 3). Tenga en cuenta que Python 2 puede tener debilidades sutiles en relación con Python 3: el manejo de Unicode de este último es mucho más completo.

Lo primero que debe tener en cuenta es que las conversiones de eliminación de mayúsculas y minúsculas en Unicode no son triviales. Hay texto para el cual text.lower() != text.upper().lower(), como "ß":

>>> "ß".lower()
'ß'

>>> "ß".upper().lower()
'ss'

Pero supongamos que desea comparar sin mayúsculas y minúsculas "BUSSE" y "Buße". Diablos, probablemente también quieras comparar "BUSSE" y "BUẞE" igual - esa es la forma de capital más nueva. La forma recomendada es usar casefold:

>>> help(str.casefold)
"""
Help on method_descriptor:

casefold(...)
      S.casefold() -> str
    
     Return a version of S suitable for caseless comparisons.
"""

No utilice simplemente inferior. Si casefold no está disponible, hacer .upper().lower() ayuda (pero solo un poco).

Entonces deberías considerar los acentos. Si su renderizador de fuentes es bueno, probablemente piense "ê" == "ê" - pero no es así:

>>> "ê" == "ê"
False

Esto se debe a que en realidad son

>>> import unicodedata

>>> [unicodedata.name(char) for char in "ê"]
['LATIN SMALL LETTER E WITH CIRCUMFLEX']

>>> [unicodedata.name(char) for char in "ê"]
['LATIN SMALL LETTER E', 'COMBINING CIRCUMFLEX ACCENT']

La forma más sencilla de lidiar con esto es unicodata.normalize. Probablemente desee utilizar la normalización NFKD, pero no dude en consultar la documentación. Entonces uno hace

>>> unicodedata.normalize("NFKD", "ê") == unicodedata.normalize("NFKD", "ê")
True

Para terminar, aquí esto se expresa en funciones:

import unicodedata

def normalize_caseless(text):
    return unicodedata.normalize("NFKD", text.casefold())

def caseless_equal(left, right):
    return normalize_caseless(left) == normalize_caseless(right)

Probar los caracteres iniciales y finales de una cadena #

Para probar el comienzo y el final de una cadena determinada en Python, se pueden usar los métodos str.startswith() y str.endswith().

str.empieza con(prefijo[, inicio[, final]]) #

Como su nombre lo indica, str.startswith se usa para probar si una cadena dada comienza con los caracteres dados en prefijo.

>>> s = "This is a test string"
>>> s.startswith("T")
True
>>> s.startswith("Thi")
True
>>> s.startswith("thi")  
False

Los argumentos opcionales start y end especifican los puntos de inicio y final desde los cuales comenzará y terminará la prueba. En el siguiente ejemplo, al especificar un valor inicial de 2, nuestra cadena se buscará desde la posición 2 y después:

>>> s.startswith("is", 2)
True

Esto produce Verdadero ya que s[2] == 'i' y s[3] == 's'.

También puede usar una tupla para verificar si comienza con cualquiera de un conjunto de cadenas

>>> s.startswith(('This', 'That'))
True
>>> s.startswith(('ab', 'bc'))
False

str.endswith(prefijo[, inicio[, fin]]) #

str.endswith es exactamente similar a str.startswith con la única diferencia de que busca caracteres finales y no caracteres iniciales. Por ejemplo, para probar si una cadena termina en un punto, se podría escribir:

>>> s = "this ends in a full stop."
>>> s.endswith('.')
True
>>> s.endswith('!')
False

al igual que con startswith, se pueden usar más de un carácter como secuencia final:

>>> s.endswith('stop.')
True
>>> s.endswith('Stop.')
False

También puede usar una ’tupla’ para verificar si termina con cualquiera de un conjunto de cadenas

>>> s.endswith(('.', 'something'))
True
>>> s.endswith(('ab', 'bc'))
False

Justificar cadenas #

Python proporciona funciones para justificar cadenas, lo que permite el relleno de texto para facilitar la alineación de varias cadenas.

A continuación se muestra un ejemplo de str.ljust y str.rjust:

interstates_lengths = {
    5: (1381, 2222),
    19: (63, 102),
    40: (2555, 4112),
    93: (189,305),
}
for road, length in interstates_lengths.items():
    miles,kms = length
    print('{} -> {} mi. ({} km.)'.format(str(road).rjust(4), str(miles).ljust(4), str(kms).ljust(4)))

  40 -> 2555 mi. (4112 km.)
  19 -> 63   mi. (102  km.)
   5 -> 1381 mi. (2222 km.)
  93 -> 189  mi. (305  km.)

ljust y rjust son muy similares. Ambos tienen un parámetro width y un parámetro fillchar opcional. Cualquier cadena creada por estas funciones es al menos tan larga como el parámetro width que se pasó a la función. Si la cadena es más larga que width, no se trunca. El argumento fillchar, que por defecto es el carácter de espacio ' ' debe ser un solo carácter, no una cadena de varios caracteres.

La función ljust rellena el final de la cadena a la que se llama con el fillchar hasta que tiene una longitud de caracteres width. La función rjust rellena el comienzo de la cadena de forma similar. Por lo tanto, l y r en los nombres de estas funciones se refieren al lado en el que la cadena original, no el fillchar, está posicionada en la cadena de salida.

Conversión entre datos str o bytes y caracteres Unicode #

El contenido de los archivos y mensajes de red puede representar caracteres codificados. A menudo es necesario convertirlos a Unicode para que se muestren correctamente.

En Python 2, es posible que deba convertir datos str a caracteres Unicode. El valor predeterminado ('', "", etc.) es una cadena ASCII, y los valores fuera del rango ASCII se muestran como valores de escape. Las cadenas Unicode son u'' (o u"", etc.).

# You get "© abc" encoded in UTF-8 from a file, network, or other data source

s = '\xc2\xa9 abc'  # s is a byte array, not a string of characters
                    # Doesn't know the original was UTF-8
                    # Default form of string literals in Python 2
s[0]                # '\xc2' - meaningless byte (without context such as an encoding)
type(s)             # str - even though it's not a useful one w/o having a known encoding

u = s.decode('utf-8')  # u'\xa9 abc'
                       # Now we have a Unicode string, which can be read as UTF-8 and printed properly
                       # In Python 2, Unicode string literals need a leading u
                       # str.decode converts a string which may contain escaped bytes to a Unicode string
u[0]                # u'\xa9' - Unicode Character 'COPYRIGHT SIGN' (U+00A9) '©'
type(u)             # unicode

u.encode('utf-8')   # '\xc2\xa9 abc'
                    # unicode.encode produces a string with escaped bytes for non-ASCII characters

En Python 3, es posible que necesite convertir matrices de bytes (denominados “bytes literales”) en cadenas de caracteres Unicode. El valor predeterminado ahora es una cadena Unicode, y los literales de cadena de bytes ahora deben ingresarse como b'', b"", etc. Un literal de byte devolverá Verdadero a isinstance(some_val, byte), some_val` para que sea una cadena que podría codificarse como bytes.

# You get from file or network "© abc" encoded in UTF-8

s = b'\xc2\xa9 abc' # s is a byte array, not characters
                    # In Python 3, the default string literal is Unicode; byte array literals need a leading b
s[0]                # b'\xc2' - meaningless byte (without context such as an encoding)
type(s)             # bytes - now that byte arrays are explicit, Python can show that.

u = s.decode('utf-8')  # '© abc' on a Unicode terminal
                       # bytes.decode converts a byte array to a string (which will, in Python 3, be Unicode)
u[0]                # '\u00a9' - Unicode Character 'COPYRIGHT SIGN' (U+00A9) '©'
type(u)             # str
                    # The default string literal in Python 3 is UTF-8 Unicode

u.encode('utf-8')   # b'\xc2\xa9 abc'
                    # str.encode produces a byte array, showing ASCII-range bytes as unescaped characters.
