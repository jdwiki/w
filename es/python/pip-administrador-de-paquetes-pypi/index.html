pip es el administrador de paquetes más utilizado para el índice de paquetes de Python, instalado de forma predeterminada con las versiones recientes de Python.

Sintaxis #

  • pip <comando> [opciones] donde <comando> es uno de:
  • Instalar en pc
    • Install packages
  • desinstalar
    • Uninstall packages
  • congelar
    • Output installed packages in requirements format
  • lista
    • List installed packages
  • mostrar
    • Show information about installed packages
  • búsqueda
    • Search PyPI for packages
  • rueda
    • Build wheels from your requirements
  • Código Postal
    • Zip individual packages (deprecated)
  • descomprimir
    • Unzip individual packages (deprecated)
  • paquete
    • Create pybundles (deprecated)
  • ayuda
    • Show help for commands

Instalar paquetes #

Para instalar la última versión de un paquete llamado SomePackage:

$ pip install SomePackage

Para instalar una versión específica de un paquete:

$ pip install SomePackage==1.0.4

Para especificar una versión mínima para instalar un paquete:

$ pip install SomePackage>=1.0.4

Si los comandos muestran un error de permiso denegado en Linux/Unix, utilice sudo con los comandos

Instalar desde archivos de requisitos #

$ pip install -r requirements.txt

Cada línea del archivo de requisitos indica algo que se debe instalar y, al igual que los argumentos para instalar pip, los detalles sobre el formato de los archivos se encuentran aquí: Formato de archivo de requisitos.

Después de instalar el paquete, puede verificarlo usando el comando freeze:

$ pip freeze

Para listar todos los paquetes instalados usando pip #

Para enumerar los paquetes instalados:

$ pip list
# example output
docutils (0.9.1)
Jinja2 (2.6)
Pygments (1.5)
Sphinx (1.1.2)

Para enumerar los paquetes obsoletos y mostrar la última versión disponible:

$ pip list --outdated
# example output
docutils (Current: 0.9.1 Latest: 0.10)
Sphinx (Current: 1.1.2 Latest: 1.1.3)

Paquetes de actualización #

Correr

$ pip install --upgrade SomePackage

actualizará el paquete SomePackage y todas sus dependencias. Además, pip elimina automáticamente la versión anterior del paquete antes de la actualización.

Para actualizar pip en sí, haga

$ pip install --upgrade pip

en Unix o

$ python -m pip install --upgrade pip

en máquinas Windows.

Desinstalar paquetes #

Para desinstalar un paquete:

$ pip uninstall SomePackage

Actualización de todos los paquetes obsoletos en Linux #

pip actualmente no contiene una bandera para permitir que un usuario actualice todos los paquetes obsoletos de una sola vez. Sin embargo, esto se puede lograr conectando comandos en un entorno Linux:

pip list --outdated --local | grep -v '^\-e' | cut -d = -f 1  | xargs -n1 pip install -U

Este comando toma todos los paquetes en el virtualenv local y verifica si están desactualizados. De esa lista, obtiene el nombre del paquete y luego lo canaliza a un comando pip install -U. Al final de este proceso, todos los paquetes locales deben actualizarse.

Actualización de todos los paquetes obsoletos en Windows #

pip actualmente no contiene una bandera para permitir que un usuario actualice todos los paquetes obsoletos de una sola vez. Sin embargo, esto se puede lograr conectando comandos en un entorno de Windows:

for /F "delims= " %i in ('pip list --outdated --local') do pip install -U %i

Este comando toma todos los paquetes en el virtualenv local y verifica si están desactualizados. De esa lista, obtiene el nombre del paquete y luego lo canaliza a un comando pip install -U. Al final de este proceso, todos los paquetes locales deben actualizarse.

Cree un archivo de requisitos.txt de todos los paquetes en el sistema #

pip ayuda a crear archivos requirements.txt proporcionando la opción freeze.

pip freeze > requirements.txt

Esto guardará una lista de todos los paquetes y su versión instalada en el sistema en un archivo llamado requirements.txt en la carpeta actual.

Cree un archivo de requisitos.txt de paquetes solo en el virtualenv actual #

pip ayuda a crear archivos requirements.txt proporcionando la opción freeze.

pip freeze --local > requirements.txt

El parámetro --local solo generará una lista de paquetes y versiones que están instalados localmente en un virtualenv. Los paquetes globales no aparecerán en la lista.

Usando una determinada versión de Python con pip #

Si tiene Python 3 y Python 2 instalados, puede especificar qué versión de Python le gustaría que use pip. Esto es útil cuando los paquetes solo admiten Python 2 o 3 o cuando desea probar con ambos.

Si desea instalar paquetes para Python 2, ejecute:

pip install [package]

o:

pip2 install [package]

Si desea instalar paquetes para Python 3, haga lo siguiente:

pip3 install [package]

También puede invocar la instalación de un paquete en una instalación específica de python con:

\path\to\that\python.exe -m pip install some_package # on Windows OR
/usr/bin/python25 -m pip install some_package # on OS-X/Linux

En las plataformas OS-X/Linux/Unix, es importante tener en cuenta la distinción entre la versión del sistema de python (cuya actualización hace que el sistema quede inoperable) y la(s) versión(es) de usuario de python. Puede, dependiendo de lo que esté intentando actualizar, necesitar prefijar estos comandos con sudo e ingresar una contraseña.

Del mismo modo, en Windows, algunas instalaciones de python, especialmente aquellas que forman parte de otro paquete, pueden terminar instaladas en directorios del sistema; tendrá que actualizarlos desde una ventana de comandos que se ejecuta en modo Administrador, si parece que necesita hacerlo. hacer esto es una muy buena idea para comprobar qué instalación de python está intentando actualizar con un comando como python -c"import sys;print(sys.path);" o py -3.5 - c"import sys;print(sys.path);" también puede verificar qué pip está intentando ejecutar con pip --version

En Windows, si tiene instalado Python 2 y Python 3, y en su ruta y Python 3 es mayor que 3.4, entonces probablemente también tendrá el iniciador de Python py en la ruta de su sistema. A continuación, puede hacer trucos como:

py -3 -m pip install -U some_package # Install/Upgrade some_package to the latest python 3
py -3.3 -m pip install -U some_package # Install/Upgrade some_package to python 3.3 if present
py -2 -m pip install -U some_package # Install/Upgrade some_package to the latest python 2 - 64 bit if present
py -2.7-32 -m pip install -U some_package # Install/Upgrade some_package to python 2.7 - 32 bit if present

Si está ejecutando y manteniendo varias versiones de python, le recomiendo encarecidamente que lea acerca de python virtualenv o venv entornos virtuales que le permiten aislar tanto la versión de python como los paquetes que están presentes.

Instalación de paquetes que aún no están en pip como ruedas #

Muchos paquetes de python puro aún no están disponibles en el índice de paquetes de Python como ruedas, pero se instalan bien. Sin embargo, algunos paquetes en Windows dan el temido error vcvarsall.bat not found.

El problema es que el paquete que intenta instalar contiene una extensión C o C++ y actualmente no está disponible como una rueda preconstruida del índice de paquetes de python, pypi, y en Windows no tiene la herramienta cadena necesaria para construir tales artículos.

La respuesta más simple es ir al excelente sitio de Christoph Gohlke y buscar la versión apropiada de las bibliotecas que necesita. Según corresponda en el nombre del paquete, -cpNN**-** tiene que coincidir con su versión de python, es decir, si está usando python de Windows de 32 bits incluso en win64, el nombre debe incluya -win32- y, si usa python de 64 bits, debe incluir -win_amd64- y luego la versión de python debe coincidir, es decir, para Python 34, el nombre de archivo debe incluir *-cp *34-, etc. Esta es básicamente la magia que pip hace por ti en el sitio de pypi.

Alternativamente, debe obtener el kit de desarrollo de Windows adecuado para la versión de python que está utilizando, los encabezados para cualquier biblioteca con la que el paquete está tratando de crear interfaces, posiblemente los encabezados de python para la versión de python, etc.

Python 2.7 usaba Visual Studio 2008, Python 3.3 y 3.4 usaban Visual Studio 2010 y Python 3.5+ usa Visual Studio 2015.

  • Instale “Visual C++ Compiler Package for Python 2.7”, que está disponible en el sitio web de Microsoft o
  • Instale “SDK de Windows para Windows 7 y .NET Framework 4” (v7.1), que está disponible en el sitio web de Microsoft o
  • Instale Visual Studio 2015 Community Edition, (o cualquier versión posterior, cuando se publiquen), asegurándose de seleccionar las opciones para instalar la compatibilidad con C y C++ ya no es la predeterminada -* Me dijeron que esto puede tomar hasta* 8 horas para descargar e instalar, así que asegúrese de que esas opciones estén configuradas en el primer intento.

Luego puede necesitar ubicar los archivos de encabezado, en la revisión correspondiente para cualquier biblioteca a la que se vincule su paquete deseado y descargarlos en las ubicaciones apropiadas.

Finalmente puede dejar que pip haga su compilación; por supuesto, si el paquete tiene dependencias que aún no tiene, es posible que también necesite encontrar los archivos de encabezado para ellos.

Alternativas: También vale la pena buscar, tanto en pypi como en el sitio de Christop, cualquier versión ligeramente anterior del paquete que está buscando que sea Python puro o preconstruido para su plataforma y versión de Python y posiblemente usarlos, si los encuentra, hasta que su paquete esté disponible. Del mismo modo, si está utilizando la versión más reciente de python, es posible que los mantenedores del paquete tarden un poco en ponerse al día, por lo que para proyectos que realmente necesitan un paquete específico, es posible que deba usar un python un poco más antiguo para el momento. También puede consultar el sitio de origen de los paquetes para ver si hay una versión bifurcada que esté disponible preconstruida o como Python puro y buscar paquetes alternativos que brinden la funcionalidad que necesita pero que estén disponibles; un ejemplo que me viene a la mente es el Pillow, mantenido activamente, reemplazo directo de PIL actualmente no actualizado en 6 años y no disponible para python 3.

Epílogo, alentaría a cualquiera que tenga este problema a que vaya al rastreador de errores del paquete y agregue, o genere, si aún no hay uno, un ticket cortésmente solicitando que los mantenedores del paquete proporcione una rueda en pypi para su combinación específica de plataforma y python, si esto se hace, normalmente las cosas mejorarán con el tiempo, algunos mantenedores de paquetes no se dan cuenta de que se han perdido una combinación determinada que las personas pueden estar usando.

Nota sobre la instalación de versiones preliminares #

Pip sigue las reglas de Semantic Versioning y, de forma predeterminada, prefiere los paquetes lanzados a los prelanzamientos. Entonces, si un paquete dado ha sido lanzado como V0.98 y también hay una versión candidata V1.0-rc1, el comportamiento predeterminado de pip install será instalar V0.98, si lo desea. instale la versión candidata, se recomienda probar primero en un entorno virtual, puede habilitarlo con --pip install --pre package-name o --pip install --pre --upgrade nombre-paquete. En muchos casos, es posible que las versiones preliminares o las versiones candidatas no estén diseñadas para todas las combinaciones de plataformas y versiones, por lo que es más probable que encuentre los problemas anteriores.

Nota sobre la instalación de versiones de desarrollo #

También puede usar pip para instalar versiones de desarrollo de paquetes de github y otras ubicaciones, dado que dicho código está cambiando, es muy poco probable que se construyan ruedas para él, por lo que cualquier paquete impuro requerirá la presencia de las herramientas de compilación, y pueden romperse en cualquier momento, por lo que se recomienda encarecidamente al usuario que solo instale dichos paquetes en un entorno virtual.

Existen tres opciones para tales instalaciones:

  1. Descargue una instantánea comprimida, la mayoría de los sistemas de control de versiones en línea tienen la opción de descargar una instantánea comprimida del código. Esto puede descargarse manualmente y luego instalarse con pip install ruta/al/archivo/descargado tenga en cuenta que para la mayoría de los formatos de compresión, pip manejará el desempaquetado en un área de caché, etc.
  2. Deje que pip maneje la descarga e instalación por usted con: pip install URL/of/package/repository - es posible que también necesite usar --trusted-host, --client-cert y /o --proxy para que esto funcione correctamente, especialmente en un entorno corporativo. p.ej:
