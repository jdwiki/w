Python, siendo uno de los lenguajes más populares en seguridad informática y de redes, tiene un gran potencial en seguridad y criptografía. Este tema trata sobre las funciones criptográficas y las implementaciones en Python, desde sus usos en la seguridad informática y de redes hasta los algoritmos hash y de cifrado/descifrado.

Sintaxis #

  • hashlib.nuevo(nombre)
  • hashlib.pbkdf2_hmac(nombre, contraseña, sal, rondas, dklen=Ninguno)

Muchos de los métodos en hashlib requerirán que pase valores interpretables como búferes de bytes, en lugar de cadenas. Este es el caso de hashlib.new().update() así como hashlib.pbkdf2_hmac. Si tiene una cadena, puede convertirla en un búfer de bytes anteponiendo el carácter b al comienzo de la cadena:

  "This is a string"
 b"This is a buffer of bytes"

Hashing seguro de contraseñas #

El [algoritmo PBKDF2] (https://en.wikipedia.org/wiki/PBKDF2) expuesto por el módulo hashlib se puede usar para realizar un hashing seguro de contraseñas. Si bien este algoritmo no puede evitar los ataques de fuerza bruta para recuperar la contraseña original del hash almacenado, hace que dichos ataques sean muy costosos.

import hashlib
import os

salt = os.urandom(16)
hash = hashlib.pbkdf2_hmac('sha256', b'password', salt, 100000)

PBKDF2 puede funcionar con cualquier algoritmo de resumen, el ejemplo anterior usa SHA256, que generalmente se recomienda. La sal aleatoria debe almacenarse junto con la contraseña hash, la necesitará nuevamente para comparar una contraseña ingresada con el hash almacenado. Es esencial que cada contraseña esté codificada con una sal diferente. En cuanto al número de rondas, se recomienda configurarlo lo más alto posible para su aplicación.

Si quieres el resultado en hexadecimal, puedes usar el módulo binascii:

import binascii
hexhash = binascii.hexlify(hash)

Nota: Si bien PBKDF2 no es malo, bcrypt y especialmente scrypt son considerado más fuerte contra los ataques de fuerza bruta. Ninguno de los dos forma parte de la biblioteca estándar de Python en este momento.

Cálculo de un resumen de mensaje #

El módulo hashlib permite crear generadores de resúmenes de mensajes a través del método nuevo. Estos generadores convertirán una cadena arbitraria en un resumen de longitud fija:

import hashlib

h = hashlib.new('sha256')
h.update(b'Nobody expects the Spanish Inquisition.')
h.digest()
# ==> b'.\xdf\xda\xdaVR[\x12\x90\xff\x16\xfb\x17D\xcf\xb4\x82\xdd)\x14\xff\xbc\xb6Iy\x0c\x0eX\x9eF-='

Tenga en cuenta que puede llamar a update un número arbitrario de veces antes de llamar a digest, lo cual es útil para fragmentar un archivo grande fragmento por fragmento. También puede obtener el resumen en formato hexadecimal usando hexdigest:

h.hexdigest()
# ==> '2edfdada56525b1290ff16fb1744cfb482dd2914ffbcb649790c0e589e462d3d'

Algoritmos hash disponibles #

hashlib.new requiere el nombre de un algoritmo cuando lo llamas para producir un generador. Para averiguar qué algoritmos están disponibles en el intérprete de Python actual, use hashlib.algorithms_disponible:

import hashlib
hashlib.algorithms_available
# ==> {'sha256', 'DSA-SHA', 'SHA512', 'SHA224', 'dsaWithSHA', 'SHA', 'RIPEMD160', 'ecdsa-with-SHA1', 'sha1', 'SHA384', 'md5', 'SHA1', 'MD5', 'MD4', 'SHA256', 'sha384', 'md4', 'ripemd160', 'sha224', 'sha512', 'DSA', 'dsaEncryption', 'sha', 'whirlpool'}

La lista devuelta variará según la plataforma y el intérprete; asegúrese de verificar que su algoritmo esté disponible.

También hay algunos algoritmos que están garantizados para estar disponibles en todas las plataformas e intérpretes, que están disponibles usando hashlib.algorithms_guaranteed:

hashlib.algorithms_guaranteed
# ==> {'sha256', 'sha384', 'sha1', 'sha224', 'md5', 'sha512'}

Hashing de archivos #

Un hash es una función que convierte una secuencia de bytes de longitud variable en una secuencia de longitud fija. Los archivos hash pueden ser ventajosos por muchas razones. Los hashes se pueden usar para verificar si dos archivos son idénticos o verificar que el contenido de un archivo no se haya dañado o cambiado.

Puede usar hashlib para generar un hash para un archivo:

import hashlib

hasher = hashlib.new('sha256')
with open('myfile', 'r') as f:
    contents = f.read()
    hasher.update(contents)

print hasher.hexdigest()

Para archivos más grandes, se puede usar un búfer de longitud fija:

import hashlib
SIZE = 65536
hasher = hashlib.new('sha256')
with open('myfile', 'r') as f:
    buffer = f.read(SIZE)
    while len(buffer) > 0:
        hasher.update(buffer)
        buffer = f.read(SIZE)
print(hasher.hexdigest())

Cifrado simétrico usando pycrypto #

Actualmente, la función criptográfica integrada de Python se limita al hashing. El cifrado requiere un módulo de terceros como pycrypto. Por ejemplo, proporciona el algoritmo AES que se considera de última generación para el cifrado simétrico. El siguiente código encriptará un mensaje dado usando una frase de contraseña:

import hashlib
import math
import os

from Crypto.Cipher import AES

IV_SIZE = 16    # 128 bit, fixed for the AES algorithm
KEY_SIZE = 32   # 256 bit meaning AES-256, can also be 128 or 192 bits
SALT_SIZE = 16  # This size is arbitrary

cleartext = b'Lorem ipsum'
password = b'highly secure encryption password'
salt = os.urandom(SALT_SIZE)
derived = hashlib.pbkdf2_hmac('sha256', password, salt, 100000,
                              dklen=IV_SIZE + KEY_SIZE)
iv = derived[0:IV_SIZE]
key = derived[IV_SIZE:]

encrypted = salt + AES.new(key, AES.MODE_CFB, iv).encrypt(cleartext)

El algoritmo AES toma tres parámetros: clave de cifrado, vector de inicialización (IV) y el mensaje real a cifrar. Si tiene una clave AES generada aleatoriamente, puede usarla directamente y simplemente generar un vector de inicialización aleatorio. Sin embargo, una frase de contraseña no tiene el tamaño correcto, ni sería recomendable usarla directamente dado que no es realmente aleatoria y, por lo tanto, tiene una entropía comparativamente pequeña. En su lugar, usamos la [implementación integrada del algoritmo PBKDF2](https://www.wikiod.com/es/python/seguridad-y-criptografia#Hashing seguro de contraseñas) para generar una inicialización de 128 bits vector y clave de cifrado de 256 bits de la contraseña.

Tenga en cuenta la sal aleatoria que es importante para tener un vector de inicialización y una clave diferentes para cada mensaje cifrado. Esto garantiza, en particular, que dos mensajes iguales no darán como resultado un texto cifrado idéntico, pero también evita que los atacantes reutilicen el trabajo dedicado a adivinar una frase de contraseña en mensajes cifrados con otra frase de contraseña. Esta sal debe almacenarse junto con el mensaje cifrado para obtener el mismo vector de inicialización y clave para el descifrado.

El siguiente código descifrará nuestro mensaje nuevamente:

salt = encrypted[0:SALT_SIZE]
derived = hashlib.pbkdf2_hmac('sha256', password, salt, 100000,
                              dklen=IV_SIZE + KEY_SIZE)
iv = derived[0:IV_SIZE]
key = derived[IV_SIZE:]
cleartext = AES.new(key, AES.MODE_CFB, iv).decrypt(encrypted[SALT_SIZE:])

Generando firmas RSA usando pycrypto #

RSA puede usarse para crear una firma de mensaje. Una firma válida solo se puede generar con acceso a la clave RSA privada, la validación en cambio es posible con solo la clave pública correspondiente. Entonces, siempre que la otra parte conozca su clave pública, puede verificar que usted firme el mensaje y no lo modifique, un enfoque que se usa para el correo electrónico, por ejemplo. Actualmente, se requiere un módulo de terceros como pycrypto para esta funcionalidad.

import errno

from Crypto.Hash import SHA256
from Crypto.PublicKey import RSA
from Crypto.Signature import PKCS1_v1_5

message = b'This message is from me, I promise.'

try:
    with open('privkey.pem', 'r') as f:
        key = RSA.importKey(f.read())
except IOError as e:
    if e.errno != errno.ENOENT:
        raise
    # No private key, generate a new one. This can take a few seconds.
    key = RSA.generate(4096)
    with open('privkey.pem', 'wb') as f:
        f.write(key.exportKey('PEM'))
    with open('pubkey.pem', 'wb') as f:
        f.write(key.publickey().exportKey('PEM'))

hasher = SHA256.new(message)
signer = PKCS1_v1_5.new(key)
signature = signer.sign(hasher)

Verificar la firma funciona de manera similar, pero usa la clave pública en lugar de la clave privada:

with open('pubkey.pem', 'rb') as f:
    key = RSA.importKey(f.read())
hasher = SHA256.new(message)
verifier = PKCS1_v1_5.new(key)
if verifier.verify(hasher, signature):
    print('Nice, the signature is valid!')
else:
    print('No, the message was signed with the wrong private key or modified')

Nota: Los ejemplos anteriores usan el algoritmo de firma PKCS#1 v1.5, que es muy común. pycrypto también implementa el nuevo algoritmo PSS PKCS#1, reemplazando PKCS1_v1_5 por PKCS1_PSS en los ejemplos debería funcionar si quieres usar ese. Actualmente parece haber [una pequeña razón para usarlo] (http://crypto.stackexchange.com/questions/3850/is-rsassa-pkcs1-v1-5-a-good-signature-scheme-for-new-systems ) sin emabargo.

Cifrado RSA asimétrico usando pycrypto #

El cifrado asimétrico tiene la ventaja de que un mensaje se puede cifrar sin intercambiar una clave secreta con el destinatario del mensaje. El remitente solo necesita conocer la clave pública del destinatario, esto permite encriptar el mensaje de tal manera que solo el destinatario designado (que tiene la clave privada correspondiente) puede desencriptarlo. Actualmente, se requiere un módulo de terceros como pycrypto para esta funcionalidad.

from Crypto.Cipher import PKCS1_OAEP
from Crypto.PublicKey import RSA

message = b'This is a very secret message.'

with open('pubkey.pem', 'rb') as f:
    key = RSA.importKey(f.read())
cipher = PKCS1_OAEP.new(key)
encrypted = cipher.encrypt(message)

El destinatario puede descifrar el mensaje si tiene la clave privada correcta:

with open('privkey.pem', 'rb') as f:
    key = RSA.importKey(f.read())
cipher = PKCS1_OAEP.new(key)
decrypted = cipher.decrypt(encrypted)

Nota: Los ejemplos anteriores utilizan el esquema de cifrado OAEP PKCS#1. pycrypto también implementa el esquema de cifrado PKCS#1 v1.5; sin embargo, este no se recomienda para nuevos protocolos debido a [advertencias conocidas] (http://security.stackexchange.com/questions/32050/what-specific-padding-weakness- hace-oaep-dirección-en-rsa).
