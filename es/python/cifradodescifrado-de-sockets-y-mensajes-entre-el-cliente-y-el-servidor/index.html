Cifradodescifrado de sockets y mensajes entre el cliente y el servidor

  • Implementación del lado del servidor
  • Implementación del lado del cliente

En esta página

  • Implementación del lado del servidor
  • Implementación del lado del cliente

La criptografía se utiliza con fines de seguridad. No hay tantos ejemplos de Cifrado/Descifrado en Python usando el MODO CTR de cifrado IDEA. Objetivo de esta documentación:

Extender e implementar el esquema de Firma Digital RSA en la comunicación de estación a estación. Usar Hashing para la integridad del mensaje, eso es SHA-1. Producir un protocolo de transporte de claves simple. Encriptar clave con encriptación IDEA. El modo de cifrado de bloques es el modo de contador

Idioma utilizado: Python 2.7 (Enlace de descarga: https://www.python.org/downloads/ )

Biblioteca utilizada:

*PyCrypto (Enlace de descarga: https://pypi.python.org/pypi/pycrypto)

*PyCryptoPlus (Enlace de descarga: https://github.com/doegox/python-cryptoplus)

Instalación de la biblioteca:

PyCrypto: Descomprime el archivo. Vaya al directorio y abra la terminal para linux (alt+ctrl+t) y CMD (shift+clic derecho+seleccionar símbolo del sistema abrir aquí) para Windows. Después de eso, escriba python setup.py install (Asegúrese de que el entorno de Python esté configurado correctamente en el sistema operativo Windows)

PyCryptoPlus: Igual que la última biblioteca.

Implementación de Tareas: La tarea se divide en dos partes. Uno es el proceso de apretón de manos y otro es el proceso de comunicación. Configuración del zócalo:

  • Como la creación de claves públicas y privadas, así como el hash de la clave pública, necesitamos para configurar el zócalo ahora. Para configurar el socket, necesitamos importar otro módulo con “importar socket” y conectar (para el cliente) o vincular (para el servidor) la dirección IP y el puerto con el socket que recibe el usuario.

    ———-Client Side———-

      server = socket.socket(socket.AF_INET,socket.SOCK_STREAM)
  host = raw_input("Server Address To Be Connected -> ")
  port = int(input("Port of The Server -> "))
  server.connect((host, port))

    ———-Server Side———

      try:
  #setting up socket
  server = socket.socket(socket.AF_INET,socket.SOCK_STREAM)     
  server.bind((host,port))
  server.listen(5)
  except BaseException: print "-----Check Server Address or Port-----"

“ socket.AF_INET,socket.SOCK_STREAM” nos permitirá usar la función accept() y los fundamentos de mensajería. En lugar de eso, también podemos usar “socket.AF_INET,socket.SOCK_DGRAM”, pero esa vez tendremos que usar setblocking(value) .

Proceso de apretón de manos:

  • (CLIENTE) La primera tarea es crear la clave pública y privada. Para crear el privado y clave pública, tenemos que importar algunos módulos. Son: de Crypto import Random y de Crypto.PublicKey import RSA. Para crear las claves, tenemos que escribir algunas líneas simples de códigos:

    random_generator = Random.new().read key = RSA.generate(1024,random_generator) public = key.publickey().exportKey()

random_generator se deriva del módulo “from Crypto import Random”. La clave se deriva de “from Crypto.PublicKey import RSA” que creará una clave privada, tamaño de 1024 mediante la generación de caracteres aleatorios. Público está exportando la clave pública de la clave privada generada previamente.

  • (CLIENTE)Después de crear la clave pública y privada, tenemos que codificar la clave pública para enviarla al servidor usando el hash SHA-1. Para usar el hash SHA-1, necesitamos importar otro módulo escribiendo “importar hashlib”. Para hacer un hash de la clave pública, escribimos dos líneas de código:

      hash_object = hashlib.sha1(public) 
  hex_digest = hash_object.hexdigest()

Aquí hash_object y hex_digest es nuestra variable. Después de esto, el cliente enviará hex_digest y public al servidor y el servidor los verificará comparando el hash obtenido del cliente y el nuevo hash de la clave pública. Si el nuevo hash y el hash del cliente coinciden, se pasará al siguiente procedimiento. Como el público enviado desde el cliente es en forma de cadena, no podrá ser utilizado como clave en el servidor. lado. Para evitar esto y convertir la clave pública de cadena en clave pública rsa, debemos escribir server_public_key = RSA.importKey (getpbk), aquí getpbk es la clave pública del cliente.

  • (SERVIDOR) El siguiente paso es crear una clave de sesión. Aquí, he usado el módulo “os” para crear una clave aleatoria “key = os.urandom(16)” que nos dará una clave larga de 16 bits y luego he cifrado esa clave en “AES.MODE_CTR” y la heh nuevamente con SHA-1:

      #encrypt CTR MODE session key
  en = AES.new(key_128,AES.MODE_CTR,counter = lambda:key_128) encrypto = en.encrypt(key_128)
  #hashing sha1
  en_object = hashlib.sha1(encrypto)
  en_digest = en_object.hexdigest()

Así que en_digest será nuestra clave de sesión.

  • (SERVIDOR) La parte final del proceso de reconocimiento es cifrar la clave pública obtenida del cliente y la clave de sesión creada en lado del servidor.

      #encrypting session key and public key
  E = server_public_key.encrypt(encrypto,16)

Después del cifrado, el servidor enviará la clave al cliente como una cadena.

  • (CLIENTE) Después de obtener la cadena cifrada de (clave pública y de sesión) del servidor, el cliente los descifrará utilizando la clave privada que se creó anteriormente junto con la clave pública. como el encriptado (clave pública y de sesión) estaba en forma de cadena, ahora tenemos que obtenerlo volver como clave usando eval() . Si se realiza el descifrado, el El proceso de apretón de manos también se completa ya que ambas partes confirman que están usando las mismas claves. Para descifrar:

      en = eval(msg)
  decrypt = key.decrypt(en)
  # hashing sha1
  en_object = hashlib.sha1(decrypt) en_digest = en_object.hexdigest()

He usado el SHA-1 aquí para que sea legible en la salida.

Proceso de comunicación:

Para el proceso de comunicación, tenemos que usar la clave de sesión de ambos lados como CLAVE para el cifrado IDEA MODE_CTR. Ambos lados cifrarán y descifrarán los mensajes con IDEA.MODE_CTR utilizando la clave de sesión.

  • (Cifrado) Para el cifrado IDEA, necesitamos una clave de 16 bits de tamaño y un contador que se pueda llamar. El contador es obligatorio en MODE_CTR. La clave de sesión que encriptamos y ciframos ahora tiene un tamaño de 40, lo que excederá la clave límite del cifrado IDEA. Por lo tanto, necesitamos reducir el tamaño de la clave de sesión. Para reducir, podemos usar Python normal incorporado en la función string[value:value]. Donde el valor puede ser cualquier valor de acuerdo a la elección del usuario. En nuestro caso, he hecho “key[:16]” donde tomará de 0 a 16 valores de la clave. Esta conversión se puede realizar de muchas formas, como clave[1:17] o clave[16:]. La siguiente parte es crear una nueva función de encriptación IDEA escribiendo IDEA.new() que tomará 3 argumentos para su procesamiento. El primer argumento será KEY, el segundo argumento será el modo de encriptación IDEA (en nuestro caso, IDEA.MODE_CTR) y el tercer argumento será counter=, que es una función a la que se debe llamar. El contador = tendrá un tamaño de cadena que será devuelto por la función. Para definir el contador = , debemos tener que usar valores razonables. En este caso, he utilizado el tamaño de la CLAVE definiendo lambda. En lugar de usar lambda, podríamos usar Counter.Util que genera un valor aleatorio para counter= . Para usar Counter.Util, necesitamos importar el módulo de contador de crypto. Por lo tanto, el código será:

      ideaEncrypt = IDEA.new(key, IDEA.MODE_CTR, counter=lambda : key)

Una vez que definamos “ideaEncrypt” como nuestra variable de cifrado IDEA, podemos usar la función de cifrado integrada para cifrar cualquier mensaje.

eMsg = ideaEncrypt.encrypt(whole)
#converting the encrypted message to HEXADECIMAL to readable eMsg =         
eMsg.encode("hex").upper()

En este segmento de código, entero es el mensaje a cifrar y eMsg es el mensaje cifrado. Después de cifrar el mensaje, lo he convertido en HEXADECIMAL para que sea legible y upper() es la función integrada para poner los caracteres en mayúsculas. Después de eso, este mensaje cifrado se enviará a la estación opuesta para su descifrado.

  • (Descifrado)

Para descifrar los mensajes cifrados, necesitaremos crear otra variable de cifrado utilizando los mismos argumentos y la misma clave, pero esta vez la variable descifrará los mensajes cifrados. El código para esto es igual que la última vez. Sin embargo, antes de descifrar los mensajes, debemos descifrar el mensaje de hexadecimal porque en nuestra parte de cifrado, codificamos el mensaje cifrado en hexadecimal para que sea legible. Por lo tanto, todo el código será:

decoded = newmess.decode("hex")
ideaDecrypt = IDEA.new(key, IDEA.MODE_CTR, counter=lambda: key) 
dMsg = ideaDecrypt.decrypt(decoded)

Estos procesos se realizarán tanto en el servidor como en el cliente para cifrar y descifrar.
