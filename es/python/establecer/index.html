Sintaxis #

  • conjunto_vacío = conjunto() # inicializa un conjunto vacío
  • literal_set = {‘foo’, ‘bar’, ‘baz’} # construye un conjunto con 3 cadenas dentro
  • set_from_list = set([‘foo’, ‘bar’, ‘baz’]) # llamar a la función set para un nuevo set
  • set_from_iter = set(x for x in range(30)) # usar iterables arbitrarios para crear un conjunto
  • set_from_iter = {x for x in [random.randint(0,10) for i in range(10)]} # notación alternativa

Los conjuntos están desordenados y tienen un tiempo de búsqueda muy rápido (amortizado O(1) si quiere ser técnico). Es excelente para usar cuando tiene una colección de cosas, el orden no importa y buscará mucho los artículos por nombre. Si tiene más sentido buscar elementos por un número de índice, considere usar una lista en su lugar. Si el orden es importante, considera también una lista.

Los conjuntos son mutables y, por lo tanto, no se pueden codificar, por lo que no puede usarlos como claves de diccionario ni colocarlos en otros conjuntos, ni en ningún otro lugar que requiera tipos codificables. En tales casos, puede usar un [frozenset] inmutable (https://docs.python.org/3/library/stdtypes.html#frozenset).

Los elementos de un conjunto deben ser hashable. Esto significa que tienen un método __hash__ correcto, que es consistente con __eq__. En general, los tipos mutables como list o set no se pueden modificar y no se pueden poner en un conjunto. Si encuentra este problema, considere usar dict y claves inmutables.

Operaciones sobre conjuntos #

con otros juegos

# Intersection    
{1, 2, 3, 4, 5}.intersection({3, 4, 5, 6})  # {3, 4, 5}
{1, 2, 3, 4, 5} & {3, 4, 5, 6}              # {3, 4, 5}

# Union
{1, 2, 3, 4, 5}.union({3, 4, 5, 6})  # {1, 2, 3, 4, 5, 6}
{1, 2, 3, 4, 5} | {3, 4, 5, 6}       # {1, 2, 3, 4, 5, 6}

# Difference
{1, 2, 3, 4}.difference({2, 3, 5})  # {1, 4}
{1, 2, 3, 4} - {2, 3, 5}            # {1, 4}

# Symmetric difference with
{1, 2, 3, 4}.symmetric_difference({2, 3, 5})  # {1, 4, 5}
{1, 2, 3, 4} ^ {2, 3, 5}                      # {1, 4, 5}

# Superset check
{1, 2}.issuperset({1, 2, 3})  # False
{1, 2} >= {1, 2, 3}           # False

# Subset check
{1, 2}.issubset({1, 2, 3})  # True
{1, 2} <= {1, 2, 3}         # True

# Disjoint check
{1, 2}.isdisjoint({3, 4})  # True
{1, 2}.isdisjoint({1, 4})  # False

con elementos individuales

# Existence check
2 in {1,2,3}      # True
4 in {1,2,3}      # False
4 not in {1,2,3}  # True

# Add and Remove
s = {1,2,3}
s.add(4)        # s == {1,2,3,4}

s.discard(3)    # s == {1,2,4}
s.discard(5)    # s == {1,2,4}

s.remove(2)     # s == {1,4}
s.remove(2)     # KeyError!

Las operaciones de conjuntos devuelven nuevos conjuntos, pero tienen las correspondientes versiones en el lugar:

método operación in situ método in situ
unión s |= t actualizar
intersección s & = t actualización_intersección
diferencia s -= t diferencia_actualizar
diferencia_simétrica s ^ = t actualización_diferencia_simétrica

Por ejemplo:

s = {1, 2}
s.update({3, 4})   # s == {1, 2, 3, 4}

Obtener los elementos únicos de una lista #

Digamos que tiene una lista de restaurantes, tal vez la lea de un archivo. Le interesan los restaurantes exclusivos de la lista. La mejor manera de obtener los elementos únicos de una lista es convertirla en un conjunto:

restaurants = ["McDonald's", "Burger King", "McDonald's", "Chicken Chicken"]
unique_restaurants = set(restaurants)
print(unique_restaurants)
# prints {'Chicken Chicken', "McDonald's", 'Burger King'}

Tenga en cuenta que el conjunto no está en el mismo orden que la lista original; eso se debe a que los conjuntos están desordenados, al igual que dicts.

Esto se puede volver a transformar fácilmente en una ‘Lista’ con la función ’lista’ integrada de Python, dando otra lista que es la misma lista que la original pero sin duplicados:

list(unique_restaurants)
# ['Chicken Chicken', "McDonald's", 'Burger King']

También es común ver esto como una sola línea:

# Removes all duplicates and returns another list
list(set(restaurants))

Ahora cualquier operación que se pudiera realizar en la lista original se puede realizar de nuevo.

Conjunto de Conjuntos #

{{1,2}, {3,4}}

lleva a:

TypeError: unhashable type: 'set'

En su lugar, usa frozenset:

{frozenset({1, 2}), frozenset({3, 4})}

Establecer operaciones usando métodos y funciones integradas #

Definimos dos conjuntos a y b

>>> a = {1, 2, 2, 3, 4}
>>> b = {3, 3, 4, 4, 5}

NOTA: {1} crea un conjunto de un elemento, pero {} crea un dict vacío. La forma correcta de crear un conjunto vacío es set().

Intersección #

a.intersection(b) devuelve un nuevo conjunto con elementos presentes tanto en a como en b

>>> a.intersection(b)
{3, 4}

Unión #

a.union(b) devuelve un nuevo conjunto con elementos presentes en a y b

>>> a.union(b)
{1, 2, 3, 4, 5}

Diferencia #

a.difference(b) devuelve un nuevo conjunto con elementos presentes en a pero no en b

>>> a.difference(b)
{1, 2}
>>> b.difference(a)
{5}

Diferencia simétrica #

a.symmetric_difference(b) devuelve un nuevo conjunto con elementos presentes en a o b pero no en ambos

>>> a.symmetric_difference(b)
{1, 2, 5}
>>> b.symmetric_difference(a)
{1, 2, 5}

NOTA: a.diferencia_simétrica(b) == b.diferencia_simétrica(a)

Subconjunto y superconjunto #

c.issubset(a) prueba si cada elemento de c está en a.

a.issuperset(c) prueba si cada elemento de c está en a.

>>> c = {1, 2}
>>> c.issubset(a)
True
>>> a.issuperset(c)
True

Las últimas operaciones tienen operadores equivalentes como se muestra a continuación:

Método Operador
a.intersección(b) a & b
a.union(b) a | b
a.diferencia(b) a-b
a.diferencia_simétrica(b) a^b
a.essubconjunto(b) a <= b
a.superconjunto(b) a >= b

Conjuntos disjuntos #

Los conjuntos a y d son disjuntos si ningún elemento en a está también en d y viceversa.

>>> d = {5, 6}
>>> a.isdisjoint(b) # {2, 3, 4} are in both sets
False
>>> a.isdisjoint(d)
True

# This is an equivalent check, but less efficient
>>> len(a & d) == 0
True

# This is even less efficient
>>> a & d == set()
True

Prueba de membresía #

La palabra clave integrada in busca ocurrencias

>>> 1 in a
True
>>> 6 in a
False

Longitud #

La función integrada len() devuelve el número de elementos en el conjunto

>>> len(a)
4
>>> len(b)
3

Conjuntos frente a multiconjuntos #

Los conjuntos son colecciones desordenadas de elementos distintos. Pero a veces queremos trabajar con colecciones desordenadas de elementos que no son necesariamente distintos y realizar un seguimiento de las multiplicidades de los elementos.

Considere este ejemplo:

>>> setA = {'a','b','b','c'}
>>> setA
set(['a', 'c', 'b'])

Al guardar las cadenas 'a', 'b', 'b', 'c' en una estructura de datos establecida, hemos perdido la información sobre el hecho de que 'b' ocurre dos veces. Por supuesto, guardar los elementos en una lista conservaría esta información.

>>> listA = ['a','b','b','c']
>>> listA
['a', 'b', 'b', 'c']

pero una estructura de datos de lista introduce un orden adicional innecesario que ralentizará nuestros cálculos.

Para implementar conjuntos múltiples, Python proporciona la clase Contador del módulo colecciones (a partir de la versión 2.7):

>>> from collections import Counter
>>> counterA = Counter(['a','b','b','c'])
>>> counterA
Counter({'b': 2, 'a': 1, 'c': 1})

Contador es un diccionario donde los elementos se almacenan como claves de diccionario y sus recuentos se almacenan como valores de diccionario. Y como todos los diccionarios, es una colección desordenada.
