Sintaxis #

  • map(función, iterable[, *iterables_adicionales])
  • future_builtins.map(función, iterable[, *iterables_adicionales])
  • itertools.imap(función, iterable[, *iterables_adicionales])

Parámetros #

Parámetro Detalles
función función para el mapeo (debe tomar tantos parámetros como iterables hay) (positional-only)
iterable la función se aplica a cada elemento del iterable (positional-only)
*iterables_adicionales ver iterables, pero tantos como quieras (opcional, posicional-solo)

Todo lo que se puede hacer con map también se puede hacer con comprehensions:

list(map(abs, [-1,-2,-3]))    # [1, 2, 3]
[abs(i) for i in [-1,-2,-3]]  # [1, 2, 3]

Aunque necesitaría zip si tiene múltiples iterables:

import operator
alist = [1,2,3]
list(map(operator.add, alist, alist))  # [2, 4, 6]
[i + j for i, j in zip(alist, alist)]  # [2, 4, 6]

Las listas de comprensión son eficientes y pueden ser más rápidas que map en muchos casos, así que prueba los tiempos de ambos enfoques si la velocidad es importante para ti.

Uso básico de map, itertools.imap y future_builtins.map #

La función de mapa es la más simple entre las funciones integradas de Python utilizadas para la programación funcional. [map()][3] aplica una función específica a cada elemento en un iterable:

names = ['Fred', 'Wilma', 'Barney']

map(len, names)  # map in Python 3.x is a class; its instances are iterable
# Out: <map object at 0x00000198B32E2CF8>

Se incluye un mapa compatible con Python 3 en el módulo future_builtins:

from future_builtins import map  # contains a Python 3.x compatible map()
map(len, names)                  # see below
# Out: <itertools.imap instance at 0x3eb0a20>

Alternativamente, en Python 2 se puede usar imap de itertools para obtener un generador

map(len, names)   # map() returns a list
# Out: [4, 5, 6]

from itertools import imap
imap(len, names)  # itertools.imap() returns a generator
# Out: <itertools.imap at 0x405ea20>

El resultado se puede convertir explícitamente en una lista para eliminar las diferencias entre Python 2 y 3:

list(map(len, names))
# Out: [4, 5, 6]

map() se puede reemplazar por un equivalente [comprensión de lista][1] o [expresión de generador][2]:

[len(item) for item in names] # equivalent to Python 2.x map()
# Out: [4, 5, 6]

(len(item) for item in names) # equivalent to Python 3.x map()
# Out: <generator object <genexpr> at 0x00000195888D5FC0>

[1]: https://www.wikiod.com/es/python/lista-de-comprensiones#Lista de comprensiones [2]: https://www.wikiod.com/es/python/lista-de-comprensiones#Expresiones generadoras [3]: https://docs.python.org/3/library/functions.html#map

Mapeando cada valor en un iterable #

Por ejemplo, puede tomar el valor absoluto de cada elemento:

list(map(abs, (1, -1, 2, -2, 3, -3))) # the call to `list` is unnecessary in 2.x
# Out: [1, 1, 2, 2, 3, 3]

La función anónima también es compatible con el mapeo de una lista:

map(lambda x:x*2, [1, 2, 3, 4, 5])
# Out: [2, 4, 6, 8, 10]

o convertir valores decimales a porcentajes:

def to_percent(num):
    return num * 100

list(map(to_percent, [0.95, 0.75, 1.01, 0.1]))
# Out: [95.0, 75.0, 101.0, 10.0]

o convertir dólares a euros (dado un tipo de cambio):

from functools import partial
from operator import mul

rate = 0.9  # fictitious exchange rate, 1 dollar = 0.9 euros
dollars = {'under_my_bed': 1000,
           'jeans': 45,
           'bank': 5000}

sum(map(partial(mul, rate), dollars.values()))
# Out: 5440.5

functools.partial es una forma conveniente de corregir los parámetros de las funciones para que puedan usarse con map en lugar de usar lambda o crear funciones personalizadas.

Mapeo de valores de diferentes iterables #

Por ejemplo, calculando el promedio de cada i-ésimo elemento de múltiples iterables:

def average(*args):
    return float(sum(args)) / len(args)  # cast to float - only mandatory for python 2.x

measurement1 = [100, 111, 99, 97]
measurement2 = [102, 117, 91, 102]
measurement3 = [104, 102, 95, 101]

list(map(average, measurement1, measurement2, measurement3))
# Out: [102.0, 110.0, 95.0, 100.0]

Hay diferentes requisitos si se pasa más de un iterable a mapa dependiendo de la versión de python:

  • La función debe tomar tantos parámetros como iterables haya:

    def median_of_three(a, b, c):
    return sorted((a, b, c))[1]

list(map(median_of_three, measurement1, measurement2))

TypeError: mediana_de_tres() falta 1 argumento posicional requerido: ‘c’

  list(map(median_of_three, measurement1, measurement2, measurement3, measurement3))

TypeError: mediana_de_tres() toma 3 argumentos posicionales pero se dieron 4

  • mapa: El mapeo itera siempre que un iterable aún no se haya consumido por completo, pero asume Ninguno de los iterables totalmente consumidos:

    import operator

measurement1 = [100, 111, 99, 97]
measurement2 = [102, 117]

# Calculate difference between elements
list(map(operator.sub, measurement1, measurement2))

TypeError: tipo de operando no admitido para -: ‘int’ y ‘NoneType’

  • itertools.imap y future_builtins.map: el mapeo se detiene tan pronto como se detiene un iterable:

    import operator
from itertools import imap

measurement1 = [100, 111, 99, 97]
measurement2 = [102, 117]

# Calculate difference between elements
list(imap(operator.sub, measurement1, measurement2))
# Out: [-2, -6]
list(imap(operator.sub, measurement2, measurement1))
# Out: [2, 6]

  • El mapeo se detiene tan pronto como se detiene un iterable:

    import operator

Transposición con mapa: uso de “Ninguno” como argumento de función (solo Python 2.x) #

from itertools import imap
from future_builtins import map as fmap # Different name to highlight differences

image = [[1, 2, 3],
         [4, 5, 6],
         [7, 8, 9]]

list(map(None, *image))
# Out: [(1, 4, 7), (2, 5, 8), (3, 6, 9)]
list(fmap(None, *image))
# Out: [(1, 4, 7), (2, 5, 8), (3, 6, 9)]
list(imap(None, *image))
# Out: [(1, 4, 7), (2, 5, 8), (3, 6, 9)]

image2 = [[1, 2, 3],
          [4, 5],
          [7, 8, 9]]
list(map(None, *image2))
# Out: [(1, 4, 7), (2, 5, 8), (3, None, 9)]  # Fill missing values with None
list(fmap(None, *image2))
# Out: [(1, 4, 7), (2, 5, 8)]                # ignore columns with missing values
list(imap(None, *image2))
# Out: [(1, 4, 7), (2, 5, 8)]                # dito

list(map(None, *image))

TypeError: el objeto ‘NoneType’ no se puede llamar

Pero hay una solución para tener resultados similares:

def conv_to_list(*args):
    return list(args)

list(map(conv_to_list, *image))
# Out: [[1, 4, 7], [2, 5, 8], [3, 6, 9]]

Mapeo en serie y en paralelo #

map() es una función integrada, lo que significa que está disponible en todas partes sin necesidad de usar una declaración de ‘importación’. Está disponible en todas partes al igual que print() Si observa el Ejemplo 5, verá que tuve que usar una declaración de importación antes de poder usar pretty print (importar pprint). Por lo tanto, pprint no es una función integrada

Mapeo de series

En este caso, cada argumento del iterable se proporciona como argumento a la función de mapeo en orden ascendente. Esto surge cuando solo tenemos un iterable para mapear y la función de mapeo requiere un solo argumento.

Ejemplo 1

insects = ['fly', 'ant', 'beetle', 'cankerworm']
f = lambda x: x + ' is an insect'
print(list(map(f, insects))) # the function defined by f is executed on each item of the iterable insects

da como resultado

['fly is an insect', 'ant is an insect', 'beetle is an insect', 'cankerworm is an insect']

Ejemplo 2

print(list(map(len, insects))) # the len function is executed each item in the insect list

da como resultado

[3, 3, 6, 10]

Mapeo paralelo

En este caso, cada argumento de la función de mapeo se extrae de todos los iterables (uno de cada iterable) en paralelo. Por lo tanto, la cantidad de iterables proporcionados debe coincidir con la cantidad de argumentos requeridos por la función.

carnivores = ['lion', 'tiger', 'leopard', 'arctic fox']
herbivores = ['african buffalo', 'moose', 'okapi', 'parakeet']
omnivores = ['chicken', 'dove', 'mouse', 'pig']

def animals(w, x, y, z):
    return '{0}, {1}, {2}, and {3} ARE ALL ANIMALS'.format(w.title(), x, y, z)

Ejemplo 3

# Too many arguments
# observe here that map is trying to pass one item each from each of the four iterables to len. This leads len to complain that
# it is being fed too many arguments
print(list(map(len, insects, carnivores, herbivores, omnivores)))

da como resultado

TypeError: len() takes exactly one argument (4 given)

Ejemplo 4

# Too few arguments
# observe here that map is suppose to execute animal on individual elements of insects one-by-one. But animals complain when
# it only gets one argument, whereas it was expecting four.
print(list(map(animals, insects)))

da como resultado

TypeError: animals() missing 3 required positional arguments: 'x', 'y', and 'z'

Ejemplo 5

# here map supplies w, x, y, z with one value from across the list
import pprint
pprint.pprint(list(map(animals, insects, carnivores, herbivores, omnivores)))

da como resultado

 ['Fly, lion, african buffalo, and chicken ARE ALL ANIMALS',
 'Ant, tiger, moose, and dove ARE ALL ANIMALS',
 'Beetle, leopard, okapi, and mouse ARE ALL ANIMALS',
 'Cankerworm, arctic fox, parakeet, and pig ARE ALL ANIMALS']
