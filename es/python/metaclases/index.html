Las metaclases le permiten modificar profundamente el comportamiento de las clases de Python (en términos de cómo se definen, instancian, acceden y más) al reemplazar la metaclase tipo que las nuevas clases usan de manera predeterminada.

Al diseñar su arquitectura, considere que muchas cosas que se pueden lograr con las metaclases también se pueden lograr usando una semántica más simple:

  • La herencia tradicional suele ser más que suficiente.
  • Los decoradores de clases pueden combinar la funcionalidad en una clase con un enfoque ad-hoc.
  • Python 3.6 introduce __init_subclass__() que permite que una clase participe en la creación de su subclase.

Metaclases Básicas #

Cuando se llama a type con tres argumentos, se comporta como la (meta)clase que es y crea una nueva instancia, es decir. produce una nueva clase/tipo.

Dummy = type('OtherDummy', (), dict(x=1))
Dummy.__class__              # <type 'type'> 
Dummy().__class__.__class__  # <type 'type'>

Es posible subclasificar tipo para crear una metaclase personalizada.

class mytype(type):
    def __init__(cls, name, bases, dict):
        # call the base initializer
        type.__init__(cls, name, bases, dict)

        # perform custom initialization...
        cls.__custom_attribute__ = 2

Ahora, tenemos una nueva metaclase mytype personalizada que se puede usar para crear clases de la misma manera que type.

MyDummy = mytype('MyDummy', (), dict(x=2))
MyDummy.__class__              # <class '__main__.mytype'>
MyDummy().__class__.__class__  # <class '__main__.mytype'>
MyDummy.__custom_attribute__   # 2

Cuando creamos una nueva clase utilizando la palabra clave clase, la metaclase se elige de forma predeterminada en función de las clases base.

>>> class Foo(object):
...     pass

>>> type(Foo)
type

En el ejemplo anterior, la única clase base es “objeto”, por lo que nuestra metaclase será del tipo de “objeto”, que es “tipo”. Es posible anular el valor predeterminado, sin embargo, depende de si usamos Python 2 o Python 3:

Se puede usar un atributo especial de nivel de clase __metaclass__ para especificar la metaclase.

class MyDummy(object):
    __metaclass__ = mytype
type(MyDummy)  # <class '__main__.mytype'>

Un argumento de palabra clave metaclase especial especifica la metaclase.

class MyDummy(metaclass=mytype):
    pass
type(MyDummy)  # <class '__main__.mytype'>

Cualquier argumento de palabra clave (excepto metaclase) en la declaración de clase se pasará a la metaclase. Por lo tanto, class MyDummy(metaclass=mytype, x=2) pasará x=2 como argumento de palabra clave al constructor mytype.

Lea esta descripción detallada de las metaclases de Python para obtener más detalles.

Singletons usando metaclases #

Un singleton es un patrón que restringe la instanciación de una clase a una instancia/objeto. Para obtener más información sobre los patrones de diseño de Python Singleton, consulte aquí.

class SingletonType(type):
    def __call__(cls, *args, **kwargs):
        try:
            return cls.__instance
        except AttributeError:
            cls.__instance = super(SingletonType, cls).__call__(*args, **kwargs)
            return cls.__instance

class MySingleton(object):
    __metaclass__ = SingletonType

class MySingleton(metaclass=SingletonType):
    pass

MySingleton() is MySingleton()  # True, only one instantiation occurs

Usando una metaclase #

Sintaxis de la metaclase #

class MyClass(object):
    __metaclass__ = SomeMetaclass

class MyClass(metaclass=SomeMetaclass):
    pass

Compatibilidad de Python 2 y 3 con six #

import six

class MyClass(six.with_metaclass(SomeMetaclass)):
    pass

Introducción a las metaclases #

¿Qué es una metaclase? #

En Python, todo es un objeto: enteros, cadenas, listas, incluso las funciones y las clases mismas son objetos. Y cada objeto es una instancia de una clase.

Para verificar la clase de un objeto x, uno puede llamar type(x), entonces:

>>> type(5)
<type 'int'>
>>> type(str)
<type 'type'>
>>> type([1, 2, 3])
<type 'list'>

>>> class C(object):
...     pass
...
>>> type(C)
<type 'type'>

La mayoría de las clases en python son instancias de tipo. type en sí también es una clase. Las clases cuyas instancias también son clases se denominan metaclases.

La metaclase más simple #

Bien, ya hay una metaclase en Python: tipo. ¿Podemos crear otro?

class SimplestMetaclass(type):
    pass

class MyClass(object):
    __metaclass__ = SimplestMetaclass

Eso no agrega ninguna funcionalidad, pero es una nueva metaclase, vea que MyClass ahora es una instancia de SimplestMetaclass:

>>> type(MyClass)
<class '__main__.SimplestMetaclass'>

Una metaclase que hace algo #

Una metaclase que hace algo generalmente anula el __nuevo__ del tipo, para modificar algunas propiedades de la clase que se va a crear, antes de llamar al __nuevo__ original que crea la clase:

class AnotherMetaclass(type):
    def __new__(cls, name, parents, dct):
        # cls is this class
        # name is the name of the class to be created
        # parents is the list of the class's parent classes
        # dct is the list of class's attributes (methods, static variables)

        # here all of the attributes can be modified before creating the class, e.g.

        dct['x'] = 8  # now the class will have a static variable x = 8

        # return value is the new class. super will take care of that
        return super(AnotherMetaclass, cls).__new__(cls, name, parents, dct)

Funcionalidad personalizada con metaclases #

La funcionalidad en las metaclases se puede cambiar para que cada vez que se cree una clase, se imprima una cadena en la salida estándar o se lance una excepción. Esta metaclase imprimirá el nombre de la clase que se está construyendo.

class VerboseMetaclass(type):

    def __new__(cls, class_name, class_parents, class_dict):
        print("Creating class ", class_name)
        new_class = super().__new__(cls, class_name, class_parents, class_dict)
        return new_class

Puedes usar la metaclase así:

class Spam(metaclass=VerboseMetaclass):
    def eggs(self):
        print("[insert example string here]")
s = Spam()
s.eggs()

La salida estándar será:

Creating class Spam
[insert example string here]

La metaclase por defecto #

Es posible que haya escuchado que todo en Python es un objeto. Es cierto, y todos los objetos tienen una clase:

>>> type(1)
int

El literal 1 es una instancia de int. Declaremos una clase:

>>> class Foo(object):
...    pass
...

Ahora vamos a instanciarlo:

>>> bar = Foo()

¿Cuál es la clase de bar?

>>> type(bar)
Foo

Genial, bar es una instancia de Foo. Pero, ¿cuál es la clase de ‘Foo’ en sí?

>>> type(Foo)
type

Ok, Foo en sí mismo es una instancia de type. ¿Qué tal type en sí mismo?

>>> type(type)
type

Entonces, ¿qué es una metaclase? Por ahora, supongamos que es solo un nombre elegante para la clase de una clase. Comida para llevar:

  • Todo es un objeto en Python, por lo que todo tiene una clase.
  • La clase de una clase se llama metaclase.
  • La metaclase por defecto es type, y por mucho es la metaclase más común

Pero, ¿por qué debería saber acerca de las metaclases? Bueno, Python en sí mismo es bastante “hackeable”, y el concepto de metaclase es importante si estás haciendo cosas avanzadas como la metaprogramación o si quieres controlar cómo se inicializan tus clases.
