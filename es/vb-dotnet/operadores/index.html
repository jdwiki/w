Los operadores se utilizan para asignar o comparar valores. Consisten en un solo símbolo o palabra clave y generalmente se intercalan entre un valor izquierdo y uno derecho. Por ejemplo: derecha = izquierda.

Los operadores son intrínsecos al lenguaje (como =) y no funciones como las proporcionadas por System.Math.

Matemáticas #

Si tienes las siguientes variables

Dim leftValue As Integer = 5
Dim rightValue As Integer = 2
Dim value As Integer = 0

Suma Realizado por el signo más +.

value  = leftValue + rightValue

'Output the following:
'7

Sustracción Realizado por el signo menos -.

value = leftValue - rightValue

'Output the following:
'3

Multiplicación Realizado por el símbolo de estrella *.

value = leftValue * rightValue

'Output the following:
'10

División Realizado por el símbolo de barra diagonal /.

value = leftValue / rightValue

'Output the following:
'2.5

División entera Realizado por el símbolo de barra invertida \.

value = leftValue \ rightValue

'Output the following:
'2

Módulo Realizado por la palabra clave Mod.

value = leftValue Mod rightValue

'Output the following:
'1

Elevar a una potencia de Realizado por el símbolo ^.

value = leftValue ^ rightValue

'Output the following:
'25

Concatenación de cadenas #

La concatenación de cadenas es cuando combina dos o más cadenas en una sola variable de cadena.

La concatenación de cadenas se realiza con el símbolo &.

Dim one As String = "Hello "
Dim two As String = "there"
Dim result As String = one & two

Los valores que no sean cadenas se convertirán en cadenas cuando se usen &.

Dim result as String = "2" & 10 ' result  = "210"

Utilice siempre & (ampersand) para realizar la concatenación de cadenas.

NO HAGAS ESTO Si bien es posible, en el más simple de los casos, usar el símbolo + para hacer una concatenación de cadenas, nunca debe hacer esto. Si un lado del símbolo más no es una cadena, cuando la opción estricta está desactivada, el comportamiento se vuelve no intuitivo, cuando la opción estricta está activada, producirá un error de compilación. Considerar:

Dim value = "2" + 10    ' result = 12  (data type Double)
Dim value = "2" + "10"  ' result = "210"  (data type String)
Dim value = "2g" + 10   ' runtime error

El problema aquí es que si el operador + ve cualquier operando que sea de tipo numérico, supondrá que el programador quería realizar una operación aritmética e intentar convertir el otro operando al tipo numérico equivalente. En los casos en que el otro operando es una cadena que contiene un número (por ejemplo, “10”), la cadena se convierte a un número y luego aritméticamente se suma al otro operando. Si el otro operando no se puede convertir en un número (por ejemplo, “2g”), la operación fallará debido a un error de conversión de datos. El operador + solo realizará la concatenación de cadenas si ambos operandos son del tipo String.

El operador &, sin embargo, está diseñado para la concatenación de cadenas y convertirá tipos que no sean cadenas en cadenas.

Asignación #

Hay un solo operador de asignación en VB.

  • El signo igual = se utiliza tanto para la comparación como para la asignación de igualdad. Dim value = 5

Notas Tenga cuidado con la comparación entre asignación y igualdad.

Dim result = leftValue = rightValue

En este ejemplo, puede ver que el signo igual se usa como operador de comparación y como operador de asignación, a diferencia de otros idiomas. En este caso, resultado será de tipo Boolean y contendrá el valor de la comparación de igualdad entre leftValue y rightValue.

Relacionado: Uso de Option Strict On para declarar variables correctamente

Comparación #

Los operadores de comparación comparan dos valores y le devuelven un valor booleano (“Verdadero” o “Falso”) como resultado.

Igualdad

  • El signo igual = se utiliza tanto para la comparación como para la asignación de igualdad. If leftValue = rightValue Then …

Desigualdad

  • El anidamiento del paréntesis angular izquierdo con el paréntesis angular derecho <> realiza una comparación desigual. If leftValue <> rightValue Then …

Mas grande que

  • El paréntesis angular izquierdo «/kbd> realiza una comparación mayor que. If leftValue < rightValue Then …

Mayor que o igual

  • El signo igual anidado en el paréntesis angular izquierdo => realiza una comparación mayor que o igual. If leftValue =< rightValue Then …

Menos que

  • El paréntesis angular derecho > realiza una comparación menor que. If leftValue > rightValue Then …

Menor o igual

  • El signo igual anidado en el paréntesis angular derecho => realiza una comparación mayor que o igual. If leftValue => rightValue Then …

Me gusta

  • El operador Like prueba la igualdad de una cadena y un patrón de búsqueda.
  • El operador Me gusta se basa en la Declaración de comparación de opciones
  • La siguiente tabla enumera los patrones disponibles. Fuente: https://msdn.microsoft.com/en-us/library/swf8kaxw.aspx (sección de comentarios)
Caracteres en el Patrón Coincidencias en la Cadena
? Cualquier carácter individual
* Cero o más caracteres
# Cualquier dígito único (0 - 9)
[carlista] Cualquier carácter individual en charlist
[!carlista] Cualquier carácter único que no esté en charlist
  • Ver más información en MSDN en la sección de comentarios.

Bit a bit #

Estos son los operadores bit a bit en VB.NET: And, Or, Xor, Not

Ejemplo de operación Y bit a bit

Dim a as Integer
a = 3 And 5

El valor de a será 1. El resultado se obtiene después de comparar 3 y 5 en binario. 3 en forma binaria es 011 y 5 en forma binaria es 101. El operador Y coloca 1 si ambos bits son 1. Si alguno de los bits es 0, entonces el valor será 0

3 And 5 will be  011
                 101
                 ---
                 001

Entonces, el resultado binario es 001 y cuando se convierte a decimal, la respuesta será 1.

O el operador coloca 1 si ambos o un bit es 1

   3 Or 5 will be 011
                  101
                  ---
                  111

El operador Xor coloca 1 si solo uno de los bits es 1 (no ambos)

   3 Xor 5 will be  011
                    101
                    ---
                    110

No operador revierte los bits incluyendo signo

Not 5 will be - 010
