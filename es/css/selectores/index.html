Los selectores de CSS identifican elementos HTML específicos como objetivos para estilos CSS. Este tema cubre cómo los selectores de CSS apuntan a los elementos HTML. Los selectores utilizan una amplia gama de más de 50 métodos de selección que ofrece el lenguaje CSS, incluidos elementos, clases, ID, pseudoelementos y pseudoclases, y patrones.

Sintaxis #

  • #identificación

  • .nombre de la clase

  • :pseudo-nombre de clase

  • ::nombre del pseudo-elemento

  • [attr] /* tiene el atributo attr. */

  • [attr="value"] /* tiene el atributo attr, y su valor es exactamente “value”. */

  • [attr~="value"] /* tiene el atributo attr y su valor, cuando se divide en espacios en blanco, contiene “value”. */

  • [attr|="value"] /* tiene el atributo attr, y su valor es exactamente “value”, o su valor comienza con “value-” . */

  • [attr^="value"] /* tiene el atributo attr y su valor comienza con “value”. */

  • [attr$="value"] /* tiene el atributo attr y su valor termina en “value”. */

  • [attr*="value"] /* tiene el atributo attr y su valor contiene “value”. */

  • nombre-elemento

  • *​

  • A veces verá dos puntos dobles (::) en lugar de uno solo (:). Esta es una forma de separar pseudo-clases de pseudo-elementos.

  • Los navegadores antiguos, como Internet Explorer 8, solo admiten dos puntos (:) para definir pseudoelementos.

  • A diferencia de las pseudoclases, solo se puede usar un pseudoelemento por selector, si está presente, debe aparecer después de la secuencia de selectores simples que representan los sujetos del selector (una versión futura de la especificación W3C puede permitir múltiples pseudo-elementos por selector).

Detalles #

[atributo] #

Selecciona elementos con el atributo dado.

div[data-color] {
  color: red;
}

<div data-color="red">This will be red</div>
<div data-color="green">This will be red</div>
<div data-background="red">This will NOT be red</div>

Demostración en vivo en JSBin

[atributo="valor"] #

Selecciona elementos con el atributo y el valor dados.

div[data-color="red"] {
  color: red;
}

<div data-color="red">This will be red</div>
<div data-color="green">This will NOT be red</div>
<div data-color="blue">This will NOT be red</div>

Demostración en vivo en JSBin

[atributo*="valor"] #

Selecciona elementos con el atributo y el valor dados donde el atributo dado contiene el valor dado en cualquier lugar (como una subcadena).

[class*="foo"] {
  color: red;
}

<div class="foo-123">This will be red</div>
<div class="foo123">This will be red</div>
<div class="bar123foo">This will be red</div>
<div class="barfooo123">This will be red</div>
<div class="barfo0">This will NOT be red</div>

Demostración en vivo en JSBin

[atributo~="valor"] #

Selecciona elementos con el atributo y el valor dados donde el valor dado aparece en una lista separada por espacios en blanco.

[class~="color-red"] {
  color: red;
}

<div class="color-red foo-bar the-div">This will be red</div>
<div class="color-blue foo-bar the-div">This will NOT be red</div>

Demostración en vivo en JSBin

[atributo^="valor"] #

Selecciona elementos con el atributo y el valor dados donde el atributo dado comienza con el valor.

[class^="foo-"] {
  color: red;
}

<div class="foo-123">This will be red</div>
<div class="foo-234">This will be red</div>
<div class="bar-123">This will NOT be red</div>

Demostración en vivo en JSBin

[atributo$="valor"] #

Selecciona elementos con el atributo y el valor dados donde el atributo dado termina con el valor dado.

[class$="file"] {
  color: red;
}

<div class="foobar-file">This will be red</div>
<div class="foobar-file">This will be red</div>
<div class="foobar-input">This will NOT be red</div>

Demostración en vivo en JSBin

[atributo|="valor"] #

Selecciona elementos con un atributo y valor dados donde el valor del atributo es exactamente el valor dado o es exactamente el valor dado seguido de - (U+002D)

[lang|="EN"] {
  color: red;
}

<div lang="EN-us">This will be red</div>
<div lang="EN-gb">This will be red</div>
<div lang="PT-pt">This will NOT be red</div>

Demostración en vivo en JSBin

[atributo="valor" i] #

Selecciona elementos con un atributo y valor determinados donde el valor del atributo se puede representar como Valor, VALOR, VALOR o cualquier otra posibilidad que no distinga entre mayúsculas y minúsculas.

[lang="EN" i] {
  color: red;
}

<div lang="EN">This will be red</div>
<div lang="en">This will be red</div>
<div lang="PT">This will NOT be red</div>

Demostración en vivo en JSBin

Combinadores #

Visión general #

Seleccionador Descripción
intervalo div Selector de descendientes (todos los <span> que son descendientes de un <div>)
div > tramo Selector de niños (todos los <span>s que son hijos directos de un <div>)
un ~ palmo Selector general de hermanos (todos los <span> que son hermanos después de <a>)
a + lapso Selector de hermanos adyacentes (todos los <span> que están inmediatamente después de un <a>)

Nota: Los selectores de hermanos apuntan a elementos que vienen después de ellos en el documento de origen. CSS, por su naturaleza (en cascada), no puede apuntar a elementos anteriores o principales. Sin embargo, al usar la propiedad flex order, [se puede simular un selector de hermanos anterior en medios visuales] (http://stackoverflow.com/a/36118012/3597276).

Combinador Descendente: selector selector #

Un combinador descendiente, representado por al menos un carácter de espacio ( ), selecciona elementos que son descendientes del elemento definido. Este combinador selecciona todos los descendientes del elemento (desde los elementos secundarios hacia abajo).

div p {
  color:red;
}

<div>
  <p>My text is red</p>
  <section>
    <p>My text is red</p>
  </section>
</div>

<p>My text is not red</p>

Demostración en vivo en JSBin

En el ejemplo anterior, se seleccionan los dos primeros elementos <p> ya que ambos son descendientes de <div>.

Combinador hijo: selector > selector #

El combinador hijo (>) se usa para seleccionar elementos que son hijos o descendientes directos del elemento especificado.

div > p {
  color:red;
}

<div>
  <p>My text is red</p>
  <section>
    <p>My text is not red</p>
  </section>
</div>

Demostración en vivo en JSBin

El CSS anterior selecciona solo el primer elemento <p>, ya que es el único párrafo que desciende directamente de un <div>.

El segundo elemento <p> no está seleccionado porque no es un hijo directo del <div>.

Combinador de hermanos adyacentes: selector + selector #

El combinador hermano adyacente (+) selecciona un elemento hermano que sigue inmediatamente a un elemento especificado.

p + p {
  color:red;
}

<p>My text is not red</p>
<p>My text is red</p>
<p>My text is red</p>
<hr>
<p>My text is not red</p>

Demostración en vivo en JSBin

El ejemplo anterior selecciona solo aquellos elementos <p> que están directamente precedidos por otro elemento <p>.

Combinador general de hermanos: selector ~ selector #

El combinador general de hermanos (~) selecciona todos los hermanos que siguen al elemento especificado.

p ~ p {
  color:red;
}

<p>My text is not red</p>
<p>My text is red</p>
<hr>
<h1>And now a title</h1>
<p>My text is red</p>

Demostración en vivo en JSBin

El ejemplo anterior selecciona todos los elementos <p> que están precedidos por otro elemento <p>, estén o no inmediatamente adyacentes.

Pseudo-clases #

Las pseudoclases son ​​palabras clave que permiten la selección basada en información que se encuentra fuera del árbol del documento o que no se puede expresar mediante otros selectores o combinadores. Esta información se puede asociar a un estado determinado (pseudoclases estado y dinámica), a ubicaciones (pseudoclases estructural y objetivo), a negaciones de la primera (negación pseudoclase) o a lenguajes (lang pseudoclase). Los ejemplos incluyen si se ha seguido o no un enlace (:visited), el mouse está sobre un elemento (:hover), una casilla de verificación está marcada (:checked), etc.

Sintaxis #

selector:pseudo-class { 
    property: value;
}

[:izquierda][26] Usado junto con la regla @page, esto selecciona todas las páginas
izquierdas en un documento impreso.
[:enlace][27] Se aplica a cualquier enlace que no haya sido visitado por el usuario.
[:no()][28] Se aplica a todos los elementos que no coinciden con el valor pasado a
(:not(p) o :not(.class-name) por ejemplo. Debe tener un valor para ser< br />válido y solo puede contener un selector. Sin embargo, puede encadenar múltiples selectores :not juntos.
[:nth-child][29] Se aplica cuando un elemento es el n-ésimo elemento de su padre, donde n
puede ser un número entero, una expresión matemática (por ejemplo, n+3) o las palabras clave
impar o incluso.
[:n-ésimo-de-tipo][30] Se aplica cuando un elemento es el n-ésimo elemento de su padre del
mismo tipo de elemento, donde n puede ser un número entero, una
expresión matemática (por ejemplo, n+3) o las palabras clave impar o par.
[:hijo único][31] La pseudoclase CSS :only-child representa cualquier elemento
que es el único hijo de su padre. Esto es lo mismo que
:first-child:last-child o :nth-child(1):nth-last-child(1),
pero con una especificidad menor.
[:opcional][32] La pseudoclase CSS :opcional representa cualquier elemento
que no tenga el atributo requerido establecido. Esto permite
formularios para indicar fácilmente campos opcionales y darles estilo en consecuencia.
[:fuera de rango][33] La pseudoclase CSS :out-of-range coincide cuando un elemento tiene su
atributo de valor fuera de las limitaciones de rango especificadas para este elemento.
Permite que la página brinde una respuesta de que el valor actualmente definido usando el elemento
está fuera de los límites del rango. Un valor puede estar fuera de un rango si es
más pequeño o más grande que los valores establecidos máximo y mínimo.
[:marcador de posición mostrado][34] Experimental. Se aplica a cualquier elemento de formulario que actualmente muestre texto de marcador de posición.
[:solo lectura][35] Se aplica a cualquier elemento que no sea editable por el usuario.
[:lectura-escritura][36] Se aplica a cualquier elemento que pueda editar un usuario, como los elementos <input>.
[:derecha][37] Usado junto con la regla @page, esto selecciona todas las páginas correctas en un
documento impreso.
[:raíz][38] coincide con el elemento raíz de un árbol que representa el documento.
[:alcance][39] La pseudoclase CSS coincide con los elementos que son un punto de referencia
para que los selectores coincidan.
:objetivo Selecciona el elemento de #noticias activo actual (haciendo clic en una URL
que contiene ese nombre de anclaje)
[:visitado][27] Se aplica a cualquier enlace que haya sido visitado por el usuario.

Selectores de nombre de clase #

El selector de nombre de clase selecciona todos los elementos con el nombre de clase de destino. Por ejemplo, el nombre de clase .warning seleccionaría el siguiente elemento <div>:

<div class="warning">
    <p>This would be some warning copy.</p>
</div>

También puede combinar nombres de clase para elementos de destino más específicamente. Construyamos sobre el ejemplo anterior para mostrar una selección de clases más complicada.

CSS

.important {
    color: orange;
}
.warning {
    color: blue;
}
.warning.important {
    color: red;
}

HTML

<div class="warning">
    <p>This would be some warning copy.</p>
</div>

<div class="important warning">
    <p class="important">This is some really important warning copy.</p>
</div>

En este ejemplo, todos los elementos con la clase .warning tendrán un texto de color azul, los elementos con la clase .important tendrán un texto de color naranja y todos los elementos que tengan ambos .important y El nombre de la clase .warning tendrá un color de texto rojo.

Tenga en cuenta que dentro del CSS, la declaración .warning.important no tenía espacios entre los dos nombres de clase. Esto significa que solo encontrará elementos que contengan los nombres de clase “advertencia” e “importante” en su atributo “clase”. Esos nombres de clase pueden estar en cualquier orden en el elemento.

Si se incluyera un espacio entre las dos clases en la declaración CSS, solo seleccionaría elementos que tienen elementos principales con nombres de clase ‘.warning’ y elementos secundarios con nombres de clase ‘.important’.

Este truco lo ayuda a seleccionar un elemento usando el ID como valor para un selector de atributos para evitar la alta especificidad del selector de ID.

HTML:

Booleano global con casilla de verificación: marcada y ~ (combinador general de hermanos) #

Con el selector ~, puede implementar fácilmente un booleano accesible global sin usar JavaScript.

Agregar booleano como casilla de verificación #

Al comienzo de su documento, agregue tantos valores booleanos como desee con un ‘id’ único y el conjunto de atributos ‘ocultos’:

<input type="checkbox" id="sidebarShown" hidden />
<input type="checkbox" id="darkThemeUsed" hidden />

<!-- here begins actual content, for example: -->
<div id="container">
    <div id="sidebar">
        <!-- Menu, Search, ... -->
    </div>

    <!-- Some more content ... -->
</div>

<div id="footer">
    <!-- ... -->
</div>

Cambiar el valor del booleano #

Puede alternar el valor booleano agregando una etiqueta con el conjunto de atributos for:

<label for="sidebarShown">Show/Hide the sidebar!</label>

Accediendo al valor booleano con CSS #

El selector normal (como .color-red) especifica las propiedades predeterminadas. Se pueden anular siguiendo los selectores verdadero / falso:

/* true: */
<checkbox>:checked ~ [sibling of checkbox & parent of target] <target>

/* false: */
<checkbox>:not(:checked) ~ [sibling of checkbox & parent of target] <target>

Tenga en cuenta que <checkbox>, [hermano ...] y <objetivo> deben reemplazarse por los selectores adecuados. [hermano...] puede ser un selector específico, (a menudo si eres perezoso) simplemente * o nada si el objetivo ya es un hermano de la casilla de verificación.

Ejemplos de la estructura HTML anterior serían:

#sidebarShown:checked ~ #container #sidebar {
    margin-left: 300px;
}

#darkThemeUsed:checked ~ #container,
#darkThemeUsed:checked ~ #footer {
    background: #333;
}

Selectores de identificación #

Los selectores de ID seleccionan elementos DOM con el ID de destino. Para seleccionar un elemento por una ID específica en CSS, se usa el prefijo #.

Por ejemplo, el siguiente elemento HTML div…

<div id="exampleID">
    <p>Example</p>
</div>

…se puede seleccionar con #exampleID en CSS como se muestra a continuación:

#exampleID {
    width: 20px;
}

Nota: Las especificaciones de HTML no permiten múltiples elementos con la misma ID

El ejemplo de selector de pseudo-clase de hijo único #

La pseudoclase :only-child CSS representa cualquier elemento que sea el único hijo de su padre.

HTML:

<div>
  <p>This paragraph is the only child of the div, it will have the color blue</p>
</div>

<div>
  <p>This paragraph is one of the two children of the div</p>
  <p>This paragraph is one of the two children of its parent</p>
</div>

CSS:

p:only-child {
  color: blue;
}

El ejemplo anterior selecciona el elemento <p> que es el hijo único de su padre, en este caso un <div>.
