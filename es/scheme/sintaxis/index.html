S-Expresión #

Una expresión en Scheme es lo que se va a ejecutar. Una expresión S, como suele llamarse, comienza con ( y termina con ). El primer miembro de la expresión es lo que se va a ejecutar. El siguiente miembro de la expresión son los parámetros que se enviarán a la expresión durante la evaluación de la expresión.

Por ejemplo sumando números:

(+ 1 2 3)

En este caso, + es un símbolo de una función add que toma múltiples parámetros. 1, 2 y 3 se envían a la función +.

S-Expression puede contener S-Expressions como parámetros, como se muestra en el siguiente ejemplo:

(if (< x y)
  x
  y)

Lo cual se puede leer como si x fuera menor que y devuelva x, de lo contrario devuelva y. En este ejemplo, evaluamos la expresión de la condición, según el valor resuelto, se devolverá x o y. Se podría evaluar a este

(if #t x y)
x
(if #f x y)
y

Un ejemplo menos obvio para los principiantes es tener una S-Expression como parte del primer miembro de una S-Expression. De esta forma, podemos cambiar el comportamiento de un método cambiando la función que será llamada sin tener que crear ramas con los mismos parámetros. Aquí hay un ejemplo rápido de una expresión que suma o resta números si x está debajo de y.

((if (< x y) + -) 
  1 2 3)

Si x está por debajo de y, la expresión se evaluará como:

(+ 1 2 3)
6

de lo contrario

(- 1 2 3)
-4

Como puede ver, Scheme le permite al programador crear una pieza de código compleja mientras le brinda al programador las herramientas para evitar la duplicación de código. En otros idiomas podríamos ver el mismo ejemplo escrito así:

(si (< x y) (+ 1 2 3) (- 1 2 3))

El problema con este método es que duplicamos mucho código mientras que lo único que cambia es el método que se llama. Este ejemplo es bastante simple pero con más condiciones podríamos ver muchas líneas similares duplicadas.

Let macro simple #

Las expresiones let en el esquema son, de hecho, macros. Se pueden expresar con lambdas. Un let simple podría verse así:

(let ((x 1) (y 2))
  (+ x y))

Devolverá 3 cuando se devuelva el valor de la última expresión del cuerpo let. Como puede ver, una expresión-let en realidad está ejecutando algo. Si traducimos esta parte del código con lambdas, obtendríamos algo como esto:

((lambda (x y) (+ x y)) 1 2)

Aquí podemos ver que estamos llamando a la lambda anónima con 1 y 2 directamente. Así que el resultado en este caso también es 3.

Con eso en mente, entendemos que una expresión let se compone de 2 partes. Tiene parámetros y un cuerpo como el de una lambda, pero la diferencia es que let expression se llama justo después de su evaluación.

Para explicar cómo funciona una expresión let de una vista abstracta a una concreta, se vería así.

(let params body ...)
(let (param1 param2 ...) body ...)
(let ((p1 val1) (p2 val2) ...) body ...)

Los parámetros son una lista de pares de (nombre valor) que se utilizarán en el cuerpo de let.

¿Por qué usar la expresión let?

Las expresiones Let son particularmente útiles para almacenar variables en un método, al igual que las inicializaciones de variables en lenguajes similares a C. Es favorable el uso de define porque fuera de la expresión let, las variables desaparecen… Usar un define es en realidad agregar una variable al entorno de ejecución actual. Las variables que se agregan al entorno global no se pueden eliminar. Deje que la expresión sea segura de usar en cualquier lugar. También se puede usar para fantasmas de variables sin tocar los ámbitos principales.

Por ejemplo:

(let ((x 1))
  (let ((x 2) (y x))
    (display x)
    (display y))
  (display x))

Imprimirá:

2
1
1

En este caso, ‘x’ se define con 1, luego se oculta con ‘x’ en el segundo ’let’ con el valor ‘2’. La variable y se inicia con el valor x del ámbito principal. Después de ejecutar la expresión interna let, muestra el valor inicial de x con 1. La expresión interna let no cambió el valor del ámbito principal.

Siempre que necesite inicializar variables, debe usar expresiones let como esta:

(let (
  (user (get-current-user session))
  (db (get-current-db session))
  (ids (get-active-ids session))
  )
  (mark-task-completed db user ids)
  (change-last-logged db user)
  (db-commit db))

Aquí, en este ejemplo, las variables se inicializan y se usan varias veces en el bloque de código. Y cuando finaliza la expresión let, las variables se liberan automáticamente ya que ya no son necesarias.

Sintaxis punteada para pares #

Hay una sintaxis particular que nos permite escribir la celda cons de una manera más compacta que usando el constructor cons.

Un par se puede escribir como tal:

'(1 . 2) == (cons 1 2)

La gran diferencia es que podemos crear ‘pares’ usando comillas. De lo contrario, Scheme crearía una lista adecuada (1 . (2 . '())).

La sintaxis de punto obliga a que la expresión tenga solo 2 miembros. Cada miembro puede ser de cualquier tipo, incluidos los pares.

'(1 . (2 . (3 . 4)))
> (1 2 3 . 4)

Tenga en cuenta que la lista incorrecta debe mostrarse con un punto al final para mostrar que el cdr del último par de la lista no es la lista vacía '().

Esta forma de mostrar listas a veces es confusa ya que la siguiente expresión no se expresaría como uno esperaría.

'((1 . 2) . ( 3 . 4))
> ((1 . 2) 3 . 4)

Dado que la lista generalmente omite el ., el primer argumento de la lista sería (1 . 2), el segundo argumento sería 3 pero como la lista es incorrecta, el último . se muestra para mostrar que el último elemento de la lista no es '(). Aunque los datos se muestran de forma diferente, los datos internos son como se crearon.
