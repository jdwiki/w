Envuelve un valor simple en una promesa usando $q.when() #

Si todo lo que necesita es envolver el valor en una promesa, no necesita usar la sintaxis larga como aquí:

//OVERLY VERBOSE
var defer;
defer = $q.defer();
defer.resolve(['one', 'two']);
return defer.promise;

En este caso, simplemente puede escribir:

//BETTER
return $q.when(['one', 'two']);

$q.when y su alias $q.resolve #

Envuelve un objeto que podría ser un valor o una promesa (de un tercero) en una promesa de $q. Esto es útil cuando se trata de un objeto que puede o no ser una promesa, o si la promesa proviene de una fuente en la que no se puede confiar.

— Referencia de la API del servicio $q de AngularJS - $q.when

Con el lanzamiento de AngularJS v1.4.1

También puede usar un alias resolve compatible con ES6

//ABSOLUTELY THE SAME AS when
return $q.resolve(['one', 'two'])

Usar promesas angulares con el servicio $q #

$q es un servicio incorporado que ayuda a ejecutar funciones asincrónicas y usar sus valores de retorno (o excepción) cuando terminan de procesarse.

$q está integrado con el mecanismo de observación del modelo $rootScope.Scope, lo que significa una propagación más rápida de la resolución o el rechazo en sus modelos y evita repintados innecesarios del navegador, lo que daría como resultado una interfaz de usuario parpadeante.

En nuestro ejemplo, llamamos a nuestra fábrica getMyData, que devuelve un objeto de promesa. Si el objeto está resuelto, devuelve un número aleatorio. Si es rechazado, devuelve un rechazo con un mensaje de error después de 2 segundos.

En fábrica angular

function getMyData($timeout, $q) {
  return function() {
    // simulated async function
    var promise = $timeout(function() {
      if(Math.round(Math.random())) {
        return 'data received!'
      } else {
        return $q.reject('oh no an error! try again')
      }
    }, 2000);
    return promise;
  }
}

Uso de Promises en llamada #

angular.module('app', [])
.factory('getMyData', getMyData)
.run(function(getData) {
  var promise = getData()
    .then(function(string) {
      console.log(string)
    }, function(error) {
      console.error(error)
    })
    .finally(function() {
      console.log('Finished at:', new Date())
    })
})

Para usar promesas, inyecte $q como dependencia. Aquí inyectamos $q en la fábrica getMyData.

var defer = $q.defer();

Se construye una nueva instancia de diferido llamando a $q.defer()

Un objeto diferido es simplemente un objeto que expone una promesa, así como los métodos asociados para resolver esa promesa. Se construye utilizando la función $q.deferred() y expone tres métodos principales: resolve(), reject() y notify().

  • resolve(value) – resuelve la promesa derivada con el valor.
  • reject(motivo) – rechaza la promesa derivada con el motivo.
  • notify(value): proporciona actualizaciones sobre el estado de ejecución de la promesa. Esto se puede llamar varias veces antes de que la promesa se resuelva o se rechace.

Propiedades #

Se accede al objeto de promesa asociado a través de la propiedad de promesa. promise – {Promise} – objeto de promesa asociado con este diferido.

Se crea una nueva instancia de promesa cuando se crea una instancia diferida y se puede recuperar llamando a deferred.promise.

El propósito del objeto “promesa” es permitir que las partes interesadas obtengan acceso al resultado de la tarea diferida cuando se complete.

Métodos de promesa -

  • then(successCallback, [errorCallback], [notifyCallback]): independientemente de cuándo se resolvió o se rechazará la promesa, luego llama a una de las devoluciones de llamada correctas o erróneas de forma asíncrona tan pronto como el resultado esté disponible. Las devoluciones de llamada se llaman con un solo argumento: el resultado o motivo de rechazo. Además, la devolución de llamada de notificación se puede llamar cero o más veces para proporcionar una indicación de progreso, antes de que se resuelva o rechace la promesa.

  • catch(errorCallback) – abreviatura de promise.then(null, errorCallback)

  • finalmente(devolución de llamada, notificación de devolución de llamada): le permite observar el cumplimiento o el rechazo de una promesa, pero hacerlo sin modificar el valor final.

Una de las características más poderosas de las promesas es la capacidad de encadenarlas. Esto permite que los datos fluyan a través de la cadena y sean manipulados y mutados en cada paso. Esto se demuestra con el siguiente ejemplo:

Ejemplo 1:

// Creates a promise that when resolved, returns 4.
function getNumbers() {

  var promise = $timeout(function() {
    return 4;
  }, 1000);
  
  return promise;
}

// Resolve getNumbers() and chain subsequent then() calls to decrement
// initial number from 4 to 0 and then output a string.
getNumbers()
  .then(function(num) {
      // 4
      console.log(num);
      return --num;
  })
  .then(function (num) {
      // 3
      console.log(num);
      return --num;
  })
   .then(function (num) {
      // 2
      console.log(num);
      return --num;
  })
  .then(function (num) {
      // 1
      console.log(num);
      return --num;
  })
  .then(function (num) {
      // 0
      console.log(num);
      return 'And we are done!';
  })
  .then(function (text) {
      // "And we are done!"
      console.log(text);        
  });

Usar el constructor $q para crear promesas #

La función constructora $q se usa para crear promesas a partir de API asíncronas que usan devoluciones de llamada para devolver resultados.

$q(función(resolver, rechazar) {…})

La función constructora recibe una función que se invoca con dos argumentos, “resolver” y “rechazar”, que son funciones que se utilizan para resolver o rechazar la promesa.

Ejemplo 1:

function $timeout(fn, delay) {
    return = $q(function(resolve, reject) {
         setTimeout(function() {
              try {
                 let r = fn();
                 resolve(r);
              } 
              catch (e) {
                 reject(e);
              }
         }, delay);
     };
}

El ejemplo anterior crea una promesa de WindowTimers.setTimeout API. El marco AngularJS proporciona una versión más elaborada de esta función. Para conocer el uso, consulte la Referencia de la API del servicio $timeout de AngularJS.

Ejemplo 2:

$scope.divide = function(a, b) {
    return $q(function(resolve, reject) {
      if (b===0) {
        return reject("Cannot devide by 0")
      } else {
        return resolve(a/b);
      }
    });
}

El código anterior que muestra una función de división prometida, devolverá una promesa con el resultado o un rechazo con un motivo si el cálculo es imposible.

Luego puede llamar y usar .then

$scope.divide(7, 2).then(function(result) {
    // will return 3.5
}, function(err) {
    // will not run
})

$scope.divide(2, 0).then(function(result) {
    // will not run as the calculation will fail on a divide by 0
}, function(err) {
    // will return the error string.
})

Usando $q.all para manejar múltiples promesas #

Puede usar la función $q.all para llamar a un método .then después de que una serie de promesas se haya resuelto con éxito y obtener los datos con los que se resolvieron.

Ejemplo:

JS:

 $scope.data = []

 $q.all([
    $http.get("data.json"),
    $http.get("more-data.json"),
    ]).then(function(responses) {
      $scope.data = responses.map((resp) => resp.data);
    });

El código anterior ejecuta $http.get 2 veces para datos en archivos json locales, cuando ambos métodos get se completan, resuelven sus promesas asociadas, cuando se resuelven todas las promesas en la matriz, el método .then comienza con ambos prometen datos dentro del argumento de la matriz responses.

Luego, los datos se asignan para que puedan mostrarse en la plantilla, luego podemos mostrar

HTML:

<ul>
   <li ng-repeat="d in data">
      <ul>
         <li ng-repeat="item in d">{{item.name}}: {{item.occupation}}</li>
      </ul>
   </li>
</ul>

JSON:

[{
  "name": "alice",
  "occupation": "manager"
}, {
  "name": "bob",
  "occupation": "developer"
}]

Evite el antipatrón $q diferido #

Evite este antipatrón #

var myDeferred = $q.defer();

$http(config).then(función(res) { myDeferred.resolve(res); }, función (error) { myDeferred.reject(error); });

return myDeferred.promise;

No hay necesidad de fabricar una promesa con $q.defer ya que el servicio $http ya devuelve una promesa.

//INSTEAD
return $http(config);

Simplemente devuelva la promesa creada por el servicio $http.

Diferir operaciones usando $q.defer #

Podemos usar $q para diferir las operaciones al futuro mientras tenemos un objeto de promesa pendiente en el presente, al usar $q.defer creamos una promesa que se resolverá o rechazará en el futuro.

Este método no es equivalente a usar el constructor $q, ya que usamos $q.defer para prometer una rutina existente que puede o no devolver (o haber devuelto alguna vez) una promesa.

Ejemplo:

var runAnimation = function(animation, duration) {
    var deferred = $q.defer();
    try {
        ...
        // run some animation for a given duration
        deferred.resolve("done");
    } catch (err) {
        // in case of error we would want to run the error hander of .then
        deferred.reject(err);
    }
    return deferred.promise;
}

// and then
runAnimation.then(function(status) {}, function(error) {})

  1. Asegúrese de devolver siempre el objeto deferred.promise o arriesgarse a un error al invocar .then

  2. Asegúrese de siempre resolver o rechazar su objeto diferido o .then puede no ejecutarse y correr el riesgo de una pérdida de memoria
