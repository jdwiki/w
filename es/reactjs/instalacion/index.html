Configuración sencilla #

Configurando las carpetas #

Este ejemplo asume que el código está en src/ y la salida se coloca en out/. Como tal, la estructura de carpetas debería parecerse a

example/
|-- src/
|   |-- index.js
|   `-- ...
|-- out/
`-- package.json

Configuración de los paquetes #

Suponiendo un entorno de configuración npm, primero debemos configurar babel para transpilar el código React en un código compatible con es5.

$npm install --save-dev babel-core babel-loader babel-preset-es2015 babel-preset-react

El comando anterior le indicará a npm que instale las bibliotecas básicas de babel, así como el módulo de carga para usar con el paquete web. También instalamos los ajustes preestablecidos de es6 y reaccionamos para que babel entienda el código del módulo JSX y es6. (Puede encontrar más información sobre los ajustes preestablecidos aquí ajustes preestablecidos de Babel)

$npm i-D paquete web

Este comando instalará webpack como una dependencia de desarrollo. (i es la abreviatura de install y -D la abreviatura de –save-dev)

También es posible que desee instalar paquetes webpack adicionales (como cargadores adicionales o la extensión webpack-dev-server)

Por último, necesitaremos el código de reacción real.

$npm i -D reaccionar reaccionar-dom

Configuración del paquete web #

Con la configuración de dependencias, necesitaremos un archivo webpack.config.js para decirle a webpack qué hacer

simple webpack.config.js:

var path = require('path');

module.exports = {
  entry: './src/index.js',
  output: {
    path: path.resolve(__dirname, 'out'),
    filename: 'bundle.js'
  },
  module: {
    loaders: [
      {
        test: /\.js$/,
        exclude: /(node_modules)/,
        loader: 'babel-loader',
        query: {
          presets: ['es2015', 'react']
        }
      }
    ]
  }
};

Este archivo le dice a webpack que comience con el archivo index.js (se supone que está en src/ ) y conviértalo en un único archivo bundle.js en el directorio out.

El bloque módulo le dice a webpack que pruebe todos los archivos encontrados contra la expresión regular y, si coinciden, invocará el cargador especificado. (cargador de babel en este caso) Además, la expresión regular exclude le dice a webpack que ignore este cargador especial para todos los módulos en la carpeta node_modules, esto ayuda a acelerar el proceso de transpilación. Por último, la opción query le dice al paquete web qué parámetros pasar a babel y se usa para pasar los ajustes preestablecidos que instalamos anteriormente.

Probando la configuración #

Todo lo que queda ahora es crear el archivo src/index.js e intentar empaquetar la aplicación

src/index.js:

'use strict'

import React from 'react'
import { render } from 'react-dom'

const App = () => {
  return <h1>Hello world!</h1>
}

render(
  <App />,
  document. getElementById('app')
)

Este archivo normalmente mostraría un simple encabezado <h1>¡Hola mundo!</h1> en la etiqueta html con el id ‘aplicación’, pero por ahora debería ser suficiente para transpilar el código una vez.

$./node_modules/.bin/webpack. Ejecutará la versión instalada localmente de webpack (use $webpack si instaló webpack globalmente con -g)

Esto debería crear el archivo out/bundle.js con el código transpilado dentro y concluye el ejemplo.

Usando webpack-dev-servidor #

Configuración #

Después de configurar un proyecto simple para usar webpack, babel y reaccionar emitiendo $ npm i -g webpack-dev-server instalará el servidor http de desarrollo para un desarrollo más rápido.

Modificando webpack.config.js #

var path = require('path');

module.exports = {
  entry: './src/index.js',
  output: {
    path: path.resolve(__dirname, 'out'),
    publicPath: '/public/',
    filename: 'bundle.js'
  },
  module: {
    loaders: [
      {
        test: /\.js$/,
        exclude: /(node_modules)/,
        loader: 'babel',
        query: {
          presets: ['es2015', 'react']
        }
      }
    ]
  },
  devServer: {
    contentBase: path.resolve(__dirname, 'public'),
    hot: true
  }
};

Las modificaciones están en

  • output.publicPath que establece una ruta desde la que se servirá nuestro paquete (consulte Archivos de configuración de Webpack para obtener más información)

  • servidordev

    • contentBase the base path to serve static files from (for example index.html)
    • hot sets the webpack-dev-server to hot reload when changes get made to files on disk

Y finalmente, solo necesitamos un index.html simple para probar nuestra aplicación.

índice.html:

<!DOCTYPE html>
<html lang="en">
  <head>
    <meta charset="utf-8">
    <title>React Sandbox</title>
  </head>
  <body>

    <div id="app" />

    <script src="public/bundle.js"></script>
  </body>
</html>

Con esta configuración ejecutando $webpack-dev-server debería iniciar un servidor http local en el puerto 8080 y, al conectarse, debería generar una página que contenga un <h1>¡Hola mundo!</h1>.
