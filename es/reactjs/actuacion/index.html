Medición del rendimiento con ReactJS #

No puedes mejorar algo que no puedes medir. Para mejorar el rendimiento de los componentes de React, debería poder medirlo. ReactJS proporciona herramientas complementarias para medir el rendimiento. Importe el módulo react-addons-perf para medir el rendimiento

import Perf from 'react-addons-perf' // ES6
var Perf = require('react-addons-perf') // ES5 with npm
var Perf = React.addons.Perf; // ES5 with react-with-addons.js

Puede utilizar los siguientes métodos del módulo Perf importado:

  • Perf.printInclusive()
  • Perf.printExclusive()
  • Perf.printWasted()
  • Perf.printOperations()
  • Rendimiento.printDOM()

El más importante que necesitará la mayor parte del tiempo es Perf.printWasted(), que le brinda la representación tabular del tiempo perdido de su componente individual.

Puede anotar la columna Tiempo perdido en la tabla y mejorar el rendimiento del Componente utilizando la sección de Consejos y trucos anterior

Consulte la Guía oficial de React y el excelente artículo de Benchling Engg. sobre el rendimiento de reacción

Algoritmo diferencial de React #

Generar el número mínimo de operaciones para transformar un árbol en otro tiene una complejidad del orden de O(n^3) donde n es el número de nodos del árbol. React se basa en dos suposiciones para resolver este problema en un tiempo lineal: O(n)

  1. Dos componentes de la misma clase generarán árboles similares y dos componentes de diferentes clases generarán árboles diferentes.

  2. Es posible proporcionar una clave única para elementos que sea estable en diferentes renderizaciones.

Para decidir si dos nodos son diferentes, React diferencia 3 casos

  1. Dos nodos son diferentes, si tienen diferentes tipos.
  • por ejemplo, <div>...</div> es diferente de <span>...</span>
  1. Siempre que dos nodos tengan claves diferentes
  • por ejemplo, <div key="1">...</div> es diferente de <div key="2">...</div>

Además, lo que sigue es crucial y extremadamente importante de entender si desea optimizar el rendimiento

Si ellos [dos nodos] no son del mismo tipo, React no va a incluso intenta hacer coincidir lo que representan. Solo va a quitar el el primero del DOM e inserte el segundo.

Este es el por qué

Es muy poco probable que un elemento

genere un DOM eso se verá como lo que generaría un . En vez de gastar tiempo tratando de hacer coincidir esas dos estructuras, Reaccionar solo reconstruye el árbol desde cero.

Los conceptos básicos: DOM HTML frente a DOM virtual #

HTML DOM es caro

Cada página web se representa internamente como un árbol de objetos. Esta representación se llama Modelo de objeto de documento. Además, es una interfaz independiente del lenguaje que permite que los lenguajes de programación (como JavaScript) accedan a los elementos HTML.

En otras palabras

El HTML DOM es un estándar sobre cómo obtener, cambiar, agregar o eliminar HTML elementos.

Sin embargo, esas operaciones DOM son extremadamente caras.

El DOM virtual es una solución

Entonces, al equipo de React se le ocurrió la idea de abstraer el HTML DOM y crear su propio Virtual DOM para calcular la cantidad mínima de operaciones que necesitamos aplicar en el HTML DOM para replicar el estado actual de nuestra aplicación.

El DOM virtual ahorra tiempo de modificaciones innecesarias del DOM.

¿Cómo exactamente?

En cada momento, React tiene el estado de la aplicación representado como un “DOM virtual”. Cada vez que cambia el estado de la aplicación, estos son los pasos que realiza React para optimizar el rendimiento

  1. Generar un nuevo DOM Virtual que represente el nuevo estado de nuestra aplicación

  2. Compare el DOM virtual antiguo (que representa el DOM HTML actual) con el DOM virtual nuevo

  3. Basado en 2. encuentre el número mínimo de operaciones para transformar el antiguo DOM virtual (que representa el DOM HTML actual) en el nuevo DOM virtual

  • para obtener más información al respecto - lea el algoritmo Diff de React
  1. Una vez que se encuentran esas operaciones, se asignan a sus operaciones equivalentes HTML DOM
  • recuerda, el DOM virtual es solo una abstracción del DOM HTML y existe una relación isomorfa entre ellos
  1. Ahora, la cantidad mínima de operaciones que se han encontrado y transferido a sus operaciones HTML DOM equivalentes ahora se aplican directamente en el HTML DOM de la aplicación, lo que ahorra tiempo al modificar el HTML DOM innecesariamente.

Nota: Las operaciones aplicadas en el DOM virtual son baratas, porque el DOM virtual es un objeto de JavaScript.

Consejos y trucos #

Cuando dos nodos no son del mismo tipo, React no intenta emparejarlos, simplemente elimina el primer nodo del DOM e inserta el segundo. Es por eso que el primer consejo dice

  1. Si se ve alternando entre dos clases de componentes con resultados muy similares, es posible que desee que sea la misma clase.

  1. Use shouldComponentUpdate para evitar que el componente se vuelva a procesar, si sabe que no va a cambiar, por ejemplo

    shouldComponentUpdate: function(nextProps, nextState) { return nextProps.id !== this.props.id; }
