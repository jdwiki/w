Cómo configurar un entorno webpack básico, reaccionar y babel

  • Cómo construir una canalización para un “Hola mundo” personalizado con imágenes.

En esta página

  • Cómo construir una canalización para un “Hola mundo” personalizado con imágenes.

Esta canalización de compilación no es exactamente lo que llamaría “lista para producción”, pero le brinda un comienzo sólido para agregarle las cosas que necesita para obtener la experiencia de desarrollo que está buscando. El enfoque que adoptan algunas personas (incluyéndome a mí a veces) es tomar una canalización completamente construida de Yeoman.io o en otro lugar y luego quitar las cosas que no quieren hasta que se adapten a su estilo. No hay nada de malo en esto, pero tal vez con el ejemplo anterior podría optar por el enfoque opuesto y construir a partir de los huesos básicos.

Algunas cosas que le gustaría agregar son cosas como un marco de prueba y estadísticas de cobertura como Karma con Mocha o Jasmine. Pelar con ESLint. Reemplazo de módulo activo en webpack-dev-server para que pueda obtener esa experiencia de desarrollo Ctrl+S, F5. Además, la tubería actual solo se compila en modo de desarrollo, por lo que una tarea de compilación de producción sería buena.

¡Te tengo!

Observe que en la propiedad de contexto de webpack.config.js hemos usado el módulo de ruta de nodo para definir nuestra ruta en lugar de simplemente concatenar __dirname con la cadena /src, esto se debe a que Windows odia las barras diagonales. Entonces, para hacer que la solución sea más compatible con varias plataformas, use el nodo de apalancamiento para ayudarnos.

Explicación de las propiedades de webpack.config.js

contexto

Esta es la ruta de archivo que Webpack usará como ruta raíz para resolver las rutas de archivo relativas. Entonces, en index.jsx, donde usamos require('./index.html'), ese punto en realidad se resuelve en el directorio src/ porque lo hemos definido como tal en esta propiedad.

entrada

Dónde busca webpack primero para comenzar a agrupar la solución. Es por eso que verá que en index.jsx estamos uniendo la solución con requisitos e importaciones.

producción

Aquí es donde definimos dónde debe colocar el paquete web los archivos que ha encontrado para agrupar. También hemos definido un nombre para el archivo en el que se colocarán nuestro javascript y estilos incluidos.

servidor de desarrollo

Estas son configuraciones específicas de webpack-dev-server. El contentBase define donde el servidor debe hacer su raíz, hemos definido la carpeta dist/ como nuestra base aquí. El puerto es el puerto en el que se alojará el servidor. open es lo que se usa para indicarle a webpack-dev-server que abra su navegador predeterminado una vez que haya activado el servidor.

módulo > cargadores

Esto define un mapeo para que lo use el paquete web, de modo que sepa qué hacer cuando encuentre diferentes archivos. La propiedad test proporciona expresiones regulares para que el paquete web las use para determinar si debe aplicar este módulo; en la mayoría de los casos, tenemos coincidencias en las extensiones de archivo. loader o loaders dan el nombre del módulo del cargador que nos gustaría usar para cargar el archivo en el paquete web y dejar que el cargador se encargue de agrupar ese tipo de archivo. También hay una propiedad query en el javascript, esto solo proporciona una cadena de consulta al cargador, por lo que probablemente también podríamos haber usado una propiedad de consulta en el cargador html si quisiéramos. Es simplemente una manera diferente de hacer las cosas.

Cómo construir una canalización para un “Hola mundo” personalizado con imágenes. #

Paso 1: Instalar Node.js

La canalización de compilación que creará se basa en Node.js, por lo que debe asegurarse en primer lugar de tenerla instalada. Para obtener instrucciones sobre cómo instalar Node.js, puede consultar los documentos SO para eso [aquí] 1

Paso 2: Inicialice su proyecto como un módulo de nodo

Abra la carpeta de su proyecto en la línea de comando y use el siguiente comando:

npm init

Para los propósitos de este ejemplo, puede tomar los valores predeterminados o si desea obtener más información sobre lo que todo esto significa, puede consultar [este] [2] documento SO sobre la configuración de la configuración del paquete.

Paso 3: Instale los paquetes npm necesarios

Ejecute el siguiente comando en la línea de comando para instalar los paquetes necesarios para este ejemplo:

npm install --save react react-dom

Luego, para las dependencias de desarrollo, ejecute este comando:

npm install --save-dev babel-core babel-preset-react babel-preset-es2015 webpack babel-loader css-loader style-loader file-loader image-webpack-loader

Finalmente, webpack y webpack-dev-server son cosas que vale la pena instalar globalmente en lugar de como una dependencia de su proyecto, si prefiere agregarlo como una dependencia, eso funcionará, no lo hago. Aquí está el comando para ejecutar:

npm install --global webpack webpack-dev-server

Paso 3: Agregue un archivo .babelrc a la raíz de su proyecto

Esto configurará babel para usar los ajustes preestablecidos que acaba de instalar. Su archivo .babelrc debería verse así:

{
  "presets": ["react", "es2015"]
}

Paso 4: Configurar la estructura del directorio del proyecto

Configure una estructura de directorio que se parezca a la siguiente en la raíz de su directorio:

|- node_modules
|- src/
  |- components/
  |- images/
  |- styles/
  |- index.html
  |- index.jsx
|- .babelrc
|- package.json

NOTA: Los node_modules, .babelrc y package.json ya deberían haber estado allí en los pasos anteriores. Solo los incluí para que pueda ver dónde encajan.

Paso 5: Complete el proyecto con los archivos del proyecto Hello World

Esto no es realmente importante para el proceso de creación de una canalización, por lo que solo le daré el código para estos y puede copiarlos y pegarlos:

src/components/HelloWorldComponent.jsx

import React, { Component } from 'react';

class HelloWorldComponent extends Component {
  constructor(props) {
    super(props);
    this.state = {name: 'Student'};
    this.handleChange = this.handleChange.bind(this);
  }

  handleChange(e) {
    this.setState({name: e.target.value});
  }

  render() {
    return (
      <div>
        <div className="image-container">
          <img src="./images/myImage.gif" />
        </div>
        <div className="form">
          <input type="text" onChange={this.handleChange} />
          <div>
            My name is {this.state.name} and I'm a clever cloggs because I built a React build pipeline
          </div>
        </div>
      </div>
    );
  }
}

export default HelloWorldComponent;

src/images/miImagen.gif

Siéntase libre de sustituir esto con cualquier imagen que desee, simplemente está ahí para demostrar que también podemos agrupar imágenes. Si proporciona su propia imagen y le asigna un nombre diferente, tendrá que actualizar HelloWorldComponent.jsx para reflejar sus cambios. Del mismo modo, si elige una imagen con una extensión de archivo diferente, debe modificar la propiedad test del cargador de imágenes en webpack.config.js con la expresión regular adecuada para que coincida con su nueva extensión de archivo.

índice.jsx

import React from 'react';
import { render } from 'react-dom';
import HelloWorldComponent from './components/HelloWorldComponent.jsx';

require('./images/myImage.gif');
require('./styles/styles.css');
require('./index.html');

render(<HelloWorldComponent />, document.getElementById('content'));

Paso 6: Cree la configuración del paquete web

Cree un archivo llamado webpack.config.js en la raíz de su proyecto y copie este código en él:

webpack.config.js

var path = require('path');

var config = {
  context: path.resolve(__dirname + '/src'),
  entry: './index.jsx',
  output: {
    filename: 'app.js',
    path: path.resolve(__dirname + '/dist'),
  },
  devServer: {
    contentBase: path.join(__dirname + '/dist'),
    port: 3000,
    open: true,
  },
  module: {
    loaders: [
      {
        test: /\.(js|jsx)$/,
        exclude: /node_modules/,
        loader: 'babel-loader'
      },
      {
        test: /\.css$/,
        loader: "style!css"
      },
      {
        test: /\.gif$/,
        loaders: [
          'file?name=[path][name].[ext]',
          'image-webpack',
        ]
      },
      { test: /\.(html)$/,
        loader: "file?name=[path][name].[ext]"
      }
    ],
  },
};

module.exports = config;

Paso 7: Cree tareas npm para su canalización

Para hacer esto, deberá agregar dos propiedades a la clave de scripts del JSON definido en el archivo package.json en la raíz de su proyecto. Haga que la clave de sus scripts se vea así:

  "scripts": {
    "start": "webpack-dev-server",
    "build": "webpack",
    "test": "echo \"Error: no test specified\" && exit 1"
  },

El script test ya habrá estado ahí y puedes elegir si mantenerlo o no, no es importante para este ejemplo.

Paso 8: Usa la canalización

Desde la línea de comando, si está en el directorio raíz del proyecto, ahora debería poder ejecutar el comando:

npm run build

Esto agrupará la pequeña aplicación que ha creado y la colocará en el directorio dist/ que creará en la raíz de la carpeta de su proyecto.

Si ejecuta el comando:

npm start

Luego, la aplicación que ha creado se mostrará en su navegador web predeterminado dentro de una instancia de servidor de desarrollo de paquete web.
