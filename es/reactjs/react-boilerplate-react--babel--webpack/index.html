Configuración del proyecto #

Necesita Node Package Manager para instalar las dependencias del proyecto. Descargue el nodo para su sistema operativo desde Nodejs.org. Node Package Manager viene con node.

También puede usar Administrador de versiones de nodos para administrar mejor sus versiones de nodos y npm. Es excelente para probar su proyecto en diferentes versiones de nodos. Sin embargo, no se recomienda para entornos de producción.

Una vez que haya instalado el nodo en su sistema, continúe e instale algunos paquetes esenciales para lanzar su primer proyecto React usando Babel y Webpack.

Antes de comenzar a presionar comandos en la terminal. Eche un vistazo a para qué se usan Babel y Webpack.

Puede iniciar su proyecto ejecutando npm init en su terminal. Siga la configuración inicial. Después de eso, ejecuta los siguientes comandos en tu terminal-

Dependencias:

npm install reaccionar reaccionar-dom --save

Dependencias de desarrollo:

npm install babel-core babel-loader babel-preset-es2015 babel-preset-react babel-preset-stage-0 webpack webpack-dev-server react-hot-loader --save-dev

Dependencias de desarrollo opcionales:

npm install eslint eslint-plugin-react babel-eslint --save-dev

Puede consultar este ejemplo paquete.json

Cree .babelrc en la raíz de su proyecto con los siguientes contenidos:

{
  "presets": ["es2015", "stage-0", "react"]
}

Opcionalmente, cree .eslintrc en la raíz de su proyecto con los siguientes contenidos:

{
  "ecmaFeatures": {
    "jsx": true,
    "modules": true
  },
  "env": {
    "browser": true,
    "node": true
  },
  "parser": "babel-eslint",
  "rules": {
    "quotes": [2, "single"],
    "strict": [2, "never"],
  },
  "plugins": [
    "react"
  ]
}

Cree un archivo .gitignore para evitar cargar archivos generados en su repositorio de git.

node_modules
npm-debug.log
.DS_Store
dist

Cree el archivo webpack.config.js con los siguientes contenidos mínimos.

var path = require('path');
var webpack = require('webpack');

module.exports = {
  devtool: 'eval',
  entry: [
    'webpack-dev-server/client?http://localhost:3000',
    'webpack/hot/only-dev-server',
    './src/index'
  ],
  output: {
    path: path.join(__dirname, 'dist'),
    filename: 'bundle.js',
    publicPath: '/static/'
  },
  plugins: [
    new webpack.HotModuleReplacementPlugin()
  ],
  module: {
    loaders: [{
      test: /\.js$/,
      loaders: ['react-hot', 'babel'],
      include: path.join(__dirname, 'src')
    }]
  }
};

Y finalmente, cree un archivo sever.js para poder ejecutar npm start, con los siguientes contenidos:

var webpack = require('webpack');
var WebpackDevServer = require('webpack-dev-server');
var config = require('./webpack.config');

new WebpackDevServer(webpack(config), {
  publicPath: config.output.publicPath,
  hot: true,
  historyApiFallback: true
}).listen(3000, 'localhost', function (err, result) {
  if (err) {
    return console.log(err);
  }

  console.log('Serving your awesome project at http://localhost:3000/');
});

Cree el archivo src/app.js para ver cómo su proyecto React hace algo.

import React, { Component } from 'react';

export default class App extends Component {
  render() {
    return (
      <h1>Hello, world.</h1>
    );
  }
}

Ejecute node server.js o npm start en la terminal, si ha definido qué significa start en su package.json

proyecto de arranque de reacción #

Acerca de este Proyecto

Este es un proyecto repetitivo simple. Esta publicación lo guiará para configurar el entorno para ReactJs + Webpack + Bable.

Empecemos

Necesitaremos un administrador de paquetes de nodos para iniciar el servidor express y administrar las dependencias a lo largo del proyecto. si es nuevo en el administrador de paquetes de nodos, puede consultar [aquí] 1. Nota: Aquí se requiere instalar el administrador de paquetes de nodos.

Cree una carpeta con un nombre adecuado y navegue hasta ella desde la terminal o mediante la GUI. Luego, vaya a la terminal y escriba npm init. Esto creará un archivo package.json. Nada de miedo, le hará algunas preguntas como el nombre de su proyecto. versión, descripción, punto de entrada, repositorio git, autor, licencia, etc. Aquí el punto de entrada es importante porque el nodo inicialmente lo buscará cuando ejecute el proyecto. Al final te pedirá que verifiques la información que proporciones. Puede escribir sí o modificarlo. Bueno, eso es todo, nuestro archivo package.json está listo.

Configuración rápida del servidor ejecute npm install express@4 –save. Estas son todas las dependencias que necesitábamos para este proyecto. Aquí es importante guardar el indicador, sin él, el archivo package.js no se actualizará. La tarea principal de package.json es almacenar la lista de dependencias. Agregará la versión express 4. Tu package.json se verá como "dependencies": { "express": "^4.13.4", ............. },

Después de la descarga completa, puede ver que hay una carpeta node_modules y una subcarpeta de nuestras dependencias. Ahora, en la raíz del proyecto, cree un nuevo archivo server.js. Ahora estamos configurando el servidor express. Voy a pasar todo el código y explicarlo más tarde.

var express = require('express');
// Create our app
var app = express();

app.use(express.static('public'));

app.listen(3000, function () {
  console.log('Express server is using port:3000');
});

var express = require(’express’); esto le dará acceso a toda la API express.

var app = express(); llamará a la biblioteca express como función. app.use(); deje que agregue la funcionalidad a su aplicación express. app.use(express.static(‘public’)); especificará el nombre de la carpeta que se expondrá en nuestro servidor web. app.listen(port, function(){}) aquí nuestro puerto será 3000 y la función a la que llamamos verificará que nuestro servidor web se está ejecutando correctamente. Eso es todo, el servidor express está configurado.

Ahora ve a nuestro proyecto y crea una nueva carpeta pública y crea el archivo index.html. index.html es el archivo predeterminado para su aplicación y el servidor Express buscará este archivo. El index.html es un archivo html simple que parece

<!DOCTYPE html>
<html>

<head>
  <meta charset="UTF-8"/>
</head>

<body>
  <h1>hello World</h1>
</body>

</html>

Y vaya a la ruta del proyecto a través de la terminal y escriba node server.js. Luego verá * console.log(‘El servidor Express está usando el puerto: 3000’);*.

Vaya al navegador y escriba http://localhost:3000 en la barra de navegación, verá hello World.

Ahora ve a la carpeta pública y crea un nuevo archivo app.jsx. JSX es un paso de preprocesador que agrega sintaxis XML a su JavaScript. Definitivamente puede usar React sin JSX, pero JSX hace que React sea mucho más elegante. Aquí está el código de muestra para app.jsx

ReactDOM.render(
  <h1>Hello World!!!</h1>,
  document.getElementById('app')
);

Ahora ve a index.html y modifica el código, debería verse así

<!DOCTYPE html>
<html>

<head>
  <meta charset="UTF-8"/>
  <script src="https://cdnjs.cloudflare.com/ajax/libs/babel-core/5.8.23                                              /browser.min.js"></script>
  <script src="https://cdnjs.cloudflare.com/ajax/libs/react/0.14.7/react.js">                           </script>
  <script src="https://cdnjs.cloudflare.com/ajax/libs/react/0.14.7/react-dom.js">    </script>
</head>

<body>
  <div id="app"></div>

  <script type="text/babel" src="app.jsx"></script>
</body>

</html>

Con esto en su lugar, ya ha terminado, espero que lo encuentre simple.
