Instalación de Ansible en Ubuntu #

Ansible mantiene un repositorio PPA que se puede usar para instalar los archivos binarios de Ansible:

sudo apt-add-repository ppa:ansible/ansible -y
sudo apt-get update && sudo apt-get install ansible -y

Para instalar una versión específica, use pip. El PPA puede estar desactualizado.

Instalación desde la fuente #

Ansible se utiliza mejor desde una caja.

Se ejecuta como usted (no como root) y tiene dependencias mínimas de python.

Instalación de dependencia de Python pip con pip:

sudo pip install paramiko PyYAML Jinja2 httplib2 six

A continuación, clone el repositorio de Ansible de GitHub:

cd ~/Documents
git clone git://github.com/ansible/ansible.git --recursive 
cd ansible

Finalmente, agregue la línea de script de inicialización ansible a su ~/.bashrc o ~/.zshrc :

source ~/Documents/ansible/hacking/env-setup

Reinicie su sesión de terminal y pruebe con

ansible --version

Instalación de Ansible en MacOS #

Hay dos formas principales de instalar Ansible en OS X, ya sea utilizando Homebrew o el administrador de paquetes Pip.

Si tiene homebrew, puede instalar la última versión de Ansible con el siguiente comando:

brew install ansible

Para instalar la rama Ansible 1.9.X, use el siguiente comando:

brew install homebrew/versions/ansible19

Para instalar la rama Ansible 2.0.X, use el siguiente comando:

brew install homebrew/versions/ansible20

Para instalar usando pip, use el siguiente comando: pip install ansible.

Para instalar una versión específica, use pip install ansible=<versión requerida>.

Instalación en sistemas basados ​​en Red Hat #

Ansible se puede instalar en CentOS u otros sistemas basados ​​en Red Hat. En primer lugar, debe instalar los requisitos previos:

sudo yum -y update
sudo yum -y install gcc libffi-devel openssl-devel python-pip python-devel

luego instale Ansible con pip:

sudo pip install ansible

Puedo recomendarle que actualice las herramientas de configuración después de la instalación:

sudo pip install --upgrade setuptools

También puede usar el Administrador de paquetes local:

yum install ansible

Instalación en Amazon Linux desde git repo #

Amazon Linux es una variante de RHEL, por lo que las instrucciones de Red Hat deberían funcionar en su mayor parte. Sin embargo, hay al menos una discrepancia.

Hubo una instancia en la que el paquete python27-devel, a diferencia de python-devel, era explícitamente necesario.

Aquí, instalaremos desde la fuente.

sudo yum -y update
sudo yum -y install python27 python27-devel openssl-devel libffi-devel gcc git

git clone https://github.com/ansible/ansible/<search the github for a preferable branch>

cd ansible
sudo python setup.py build
sudo python setup.py install

Instalación de Ansible en cualquier máquina con sistema operativo (Windows) usando Virtual Box + Vagrant #

Mi computadora portátil tiene Windows 10. Aquí le doy los pasos que puede seguir para probar y aprender Ansible.

ALGUNA TEORIA

Para Ansible, necesita una máquina de control y un host (o hosts) para ejecutar Playbook.

  • Control Machine debe estar basado en Linux o MacOS (Windows no permitido) y necesita Python (versión 2.6 o superior). Aquí se instalará Ansible.
  • Máquina de destino (host/nodo) puede ser Linux/MacOS/Windows. Esto solo necesita que Python esté instalado. No se requiere software de agente.

CONFIGURACIÓN

Paso 1: Instalar Caja Virtual

Virtual box es un software para crear computadoras virtuales de diferentes sistemas operativos. Es como tener varias computadoras cada una o diferentes sistemas operativos y diferentes versiones.

Descargue Virtual Box según el sistema operativo de su sistema e instálelo.

Paso 2: Instala Vagrant

Vagrant es una interfaz de línea de comandos para crear máquinas virtuales en una caja virtual. Esto facilita las cosas. Necesitas aprender los comandos básicos de Vagrant.

Paso 3: Crea una carpeta donde quieras tu máquina virtual

Paso 4: Crear máquina virtual usando Vagrant

Abra la terminal y vaya a la ruta donde creó la carpeta, y ejecute los siguientes dos comandos.

Debes seleccionar Virtual Box. Estoy instalando Ubuntu por ejemplo. Puede elegir cualquier cosa de la lista. Debe ejecutar estos dos comandos en la categoría “caja virtual”: vagrant init ubuntu/trusty64 y vagrant up --provider virtualbox. Otras categorías pueden ser: hyperv, vmware_desktop, etc. (esto llevará algún tiempo, ya que descargará los archivos necesarios)

Paso 4: Instalar Ansible

Para UbuntuOS: sudo apt-get install ansible

Solución alternativa: #

Puedes usar Katacoda para practicar ansible. No es necesario instalar ni configurar nada. Ejecute dos comandos dados en el paso 2 y después de eso, estará listo para comenzar.
