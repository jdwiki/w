Datos frente a propiedades calculadas #

La principal diferencia de caso de uso para las propiedades data y computed de una instancia Vue depende del estado potencial o la probabilidad de cambio de los datos. Al decidir qué categoría debe tener un determinado objeto, estas preguntas pueden ayudar:

  • ¿Es este un valor constante? (datos)
  • ¿Tiene esto la posibilidad de cambiar? (calculado o datos)
  • ¿Depende su valor del valor de otros datos? (calculado)
  • ¿Necesita datos adicionales o cálculos para estar completo antes de ser utilizado? (calculado)
  • ¿El valor solo cambiará bajo ciertas circunstancias? (datos)

Ejemplo básico #

Modelo

<div id="example">
  a={{ a }}, b={{ b }}
</div>

JavaScript

var vm = new Vue({
  el: '#example',
  data: {
    a: 1
  },
  computed: {
    // a computed getter
    b: function () {
      // `this` points to the vm instance
      return this.a + 1
    }
  }
})

Resultado

a=1, b=2

Aquí hemos declarado una propiedad calculada b. La función que proporcionamos se usará como función captadora para la propiedad vm.b:

console.log(vm.b) // -> 2
vm.a = 2
console.log(vm.b) // -> 3

El valor de vm.b siempre depende del valor de vm.a.

Puede vincular datos a propiedades calculadas en plantillas como una propiedad normal. Vue es consciente de que vm.b depende de vm.a, por lo que actualizará cualquier enlace que dependa de vm.b cuando vm.a cambie.

Propiedades computadas vs reloj #

modelo

<div id="demo">{{fullName}}</div>

ver ejemplo

var vm = new Vue({
  el: '#demo',
  data: {
    firstName: 'Foo',
    lastName: 'Bar',
    fullName: 'Foo Bar'
  }
})

vm.$watch('firstName', function (val) {
  this.fullName = val + ' ' + this.lastName
})

vm.$watch('lastName', function (val) {
  this.fullName = this.firstName + ' ' + val
})

Ejemplo calculado

var vm = new Vue({
  el: '#demo',
  data: {
    firstName: 'Foo',
    lastName: 'Bar'
  },
  computed: {
    fullName: function () {
      return this.firstName + ' ' + this.lastName
    }
  }
})

Setters calculados #

Las propiedades calculadas se volverán a calcular automáticamente cada vez que cambie cualquier dato del que dependa el cálculo. Sin embargo, si necesita cambiar manualmente una propiedad calculada, Vue le permite crear un método setter para hacer esto:

Plantilla (del ejemplo básico anterior):

<div id="example">
  a={{ a }}, b={{ b }}
</div>

Javascript:

var vm = new Vue({
  el: '#example',
  data: {
    a: 1
  },
  computed: {
    b: {
      // getter
      get: function () {
        return this.a + 1
      },
      // setter
      set: function (newValue) {
        this.a = newValue - 1 
      }
    }
  }

Ahora puede invocar el getter o el setter:

console.log(vm.b)       // -> 2
vm.b = 4                // (setter) 
console.log(vm.b)       // -> 4
console.log(vm.a)       // -> 3

vm.b = 4 invocará al setter y establecerá this.a en 3; por extensión, vm.b se evaluará como 4.

Usando setters calculados para v-model #

Es posible que necesite un v-model en una propiedad calculada. Normalmente, el modelo v no actualizará el valor de propiedad calculado.

La plantilla:

<div id="demo">
  <div class='inline-block card'>
    <div :class='{onlineMarker: true, online: status, offline: !status}'></div>  
    <p class='user-state'>User is {{ (status) ? 'online' : 'offline' }}</p>
  </div>

  <div class='margin-5'>
    <input type='checkbox' v-model='status'>Toggle status (This will show you as offline to others)
  </div>
</div>

Estilo:

#demo {
  font-family: Helvetica;
  font-size: 12px;
}
.inline-block > * {
  display: inline-block;
}
.card {
  background: #ddd;
  padding:2px 10px;
  border-radius: 3px;
}
.onlineMarker {
  width: 10px;
  height: 10px;
  border-radius: 50%;
  transition: all 0.5s ease-out;
}

.online {
  background-color: #3C3;
}

.offline {
  background-color: #aaa;
}

.user-state {
  text-transform: uppercase;
  letter-spacing: 1px;
}

.margin-5 {
  margin: 5px;
}

El componente:

var demo = new Vue({
    el: '#demo',
    data: {
        statusProxy: null
    },
    computed: {
        status: {
            get () {
                return (this.statusProxy === null) ? true : this.statusProxy     
            }
        }
    }
})

violín Aquí vería, hacer clic en el botón de radio no tiene ningún uso, su estado todavía está en línea.

var demo = new Vue({
    el: '#demo',
    data: {
        statusProxy: null
    },
    computed: {
        status: {
            get () {
                return (this.statusProxy === null) ? true : this.statusProxy     
            },
            set (val) {
                this.statusProxy = val            
            }
        }
    }
})

violín Y ahora puede ver que la alternancia ocurre cuando la casilla de verificación está marcada/desmarcada.
