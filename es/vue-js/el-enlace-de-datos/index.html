Texto #

La forma más básica de vinculación de datos es la interpolación de texto utilizando la sintaxis “Bigote” (llaves dobles):

<span>Message: {{ msg }}</span>

La etiqueta del bigote se reemplazará con el valor de la propiedad msg en el objeto de datos correspondiente. También se actualizará cada vez que cambie la propiedad msg del objeto de datos.

También puede realizar interpolaciones únicas que no se actualizan con el cambio de datos:

<span>This will never change: {{* msg }}</span>

HTML sin formato #

Los bigotes dobles interpretan los datos como texto sin formato, no como HTML. Para generar HTML real, deberá usar bigotes triples:

<div>{{{ raw_html }}}</div>

Los contenidos se insertan como HTML sin formato; los enlaces de datos se ignoran. Si necesita reutilizar piezas de plantilla, debe usar parciales.

Atributos #

Los bigotes también se pueden usar dentro de atributos HTML:

<div id="item-{{ id }}"></div>

Tenga en cuenta que las interpolaciones de atributos no están permitidas en las directivas y atributos especiales de Vue.js. No se preocupe, Vue.js le mostrará advertencias cuando los bigotes se usen en lugares equivocados.

Filtros #

Vue.js le permite agregar “filtros” opcionales al final de una expresión, indicados por el símbolo de “tubería”:

{{ message | capitalize }}

Aquí estamos “canalizando” el valor de la expresión mensaje a través del filtro integrado mayúsculas, que de hecho es solo una función de JavaScript que devuelve el valor en mayúsculas. Vue.js proporciona una serie de filtros incorporados y hablaremos sobre cómo escribir sus propios filtros más adelante.

Tenga en cuenta que la sintaxis de canalización no forma parte de la sintaxis de JavaScript, por lo tanto, no puede mezclar filtros dentro de expresiones; solo puede agregarlos al final de una expresión.

Los filtros se pueden encadenar:

{{ message | filterA | filterB }}

Los filtros también pueden tomar argumentos:

{{ message | filterA 'arg1' arg2 }}

La función de filtro siempre recibe el valor de la expresión como primer argumento. Los argumentos entrecomillados se interpretan como una cadena simple, mientras que los que no están entrecomillados se evaluarán como expresiones. Aquí, la cadena simple 'arg1' se pasará al filtro como segundo argumento, y el valor de la expresión arg2 se evaluará y se pasará como tercer argumento.
