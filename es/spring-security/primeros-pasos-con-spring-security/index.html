Spring Securitiy para proteger los puntos finales de la API REST #

Agregue las siguientes entradas en pom.xml.

<dependency>
    <groupId>org.springframework.security</groupId>
    <artifactId>spring-security-web</artifactId>
    <version>3.1.0.RELEASE</version>
</dependency>
<dependency>
    <groupId>org.springframework.security</groupId>
    <artifactId>spring-security-config</artifactId>
    <version>3.1.0.RELEASE</version>
</dependency>

Importante para la versión de Spring superior a 3.1:

El error de creación de beans para org.springframework.security.filterChains se produce cuando está utilizando una versión de Spring superior a la 3.1 y no ha agregado dependencias manualmente para spring-aop, spring-jdbc, spring-tx y spring -expresiones en su pom.xml.

Agregue las siguientes entradas en el contexto de Spring. Queremos proteger dos puntos finales REST (hola mundo y adiós). Ajuste la versión XSD según la versión Spring.

<?xml version="1.0" encoding="UTF-8"?>
<beans xmlns="http://www.springframework.org/schema/beans"
   xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
   xmlns:security="http://www.springframework.org/schema/security"
   xmlns:context="http://www.springframework.org/schema/context"
   xsi:schemaLocation="http://www.springframework.org/schema/beans http://www.springframework.org/schema/beans/spring-beans-3.1.xsd
                       http://www.springframework.org/schema/context http://www.springframework.org/schema/context/spring-context-3.1.xsd
                       http://www.springframework.org/schema/security
                       http://www.springframework.org/schema/security/spring-security-3.1.xsd">
                               
    <security:http auto-config='true' create-session="never">
        <security:intercept-url pattern="/helloworld/**" access="ROLE_USER" />
        <security:intercept-url pattern="/goodbye/**" access="ROLE_ADMIN" />
        <security:intercept-url pattern="/**" access="IS_AUTHENTICATED_ANONYMOUSLY" />
        <security:http-basic />
    </security:http>

    <security:authentication-manager>
        <security:authentication-provider>
            <security:user-service>
                <security:user name="username1" password="password1"
                    authorities="ROLE_USER" />
                <security:user name="username2" password="password2"
                    authorities="ROLE_ADMIN" />
            </security:user-service>
        </security:authentication-provider>
    </security:authentication-manager>
</beans>

Agregue las siguientes entradas en web.xml.

<!-- Spring security-->
<filter>
    <filter-name>springSecurityFilterChain</filter-name>
    <filter-class>org.springframework.web.filter.DelegatingFilterProxy</filter-class>
</filter>
<filter-mapping>
    <filter-name>springSecurityFilterChain</filter-name>
    <url-pattern>/*</url-pattern>
</filter-mapping>

<listener>
    <listener-class>org.springframework.web.context.ContextLoaderListener</listener-class>
</listener>

<context-param>
    <param-name>contextConfigLocation</param-name>
    <param-value>classpath:security-context.xml</param-value>
</context-param>

Spring-Security usando spring-boot y autenticación JDBC #

Supongamos que desea evitar que los usuarios no autorizados accedan a la página, entonces debe ponerles una barrera al autorizar el acceso. Podemos hacer esto usando Spring-Security que proporciona autenticación básica asegurando todos los puntos finales de HTTP. Para eso, debe agregar la dependencia Spring-Security a su proyecto, en Maven podemos agregar la dependencia como:

<dependency>
    <groupId>org.springframework.boot</groupId>
    <artifactId>spring-boot-starter-security</artifactId>
</dependency>

Aquí hay una configuración de seguridad que garantiza que solo los usuarios autenticados puedan acceder.

@Configuration
@Order(SecurityProperties.ACCESS_OVERRIDE_ORDER)
public class SecurityConfig extends WebSecurityConfigurerAdapter {

@Autowired
DataSource datasource;

@Override
protected void configure(HttpSecurity http) throws Exception {
    http
        .authorizeRequests()
            .anyRequest()
            .fullyAuthenticated()
            .and()
        .formLogin()
            .loginPage("/login")
            .failureUrl("/login?error")
            .permitAll()
            .and()
        .logout()
            .logoutUrl("/logout")
            .logoutSuccessUrl("/login?logout")
            .permitAll()
            .and()
        .csrf();
}

@Override
protected void configure(AuthenticationManagerBuilder auth) throws Exception {
    auth.jdbcAuthentication().dataSource(datasource).passwordEncoder(passwordEncoder());
}

@Bean
public PasswordEncoder passwordEncoder() {
    PasswordEncoder encoder = new BCryptPasswordEncoder();
    return encoder;
}
}

Configuración | Descripción ——————|——————————– ———————– @Configuration |Indica que la clase puede ser utilizada por el contenedor Spring IoC como fuente de definiciones de beans. @Order (SecurityProperties.ACCESS_OVERRIDE_ORDER)|Anula las reglas de acceso sin cambiar ninguna otra función configurada automáticamente. Los valores más bajos tienen mayor prioridad. WebSecurityConfigurerAdapter|La clase SecurityConfig extiende y anula un par de sus métodos para establecer algunos detalles de la configuración de seguridad. @Autowired de DataSource|Proporciona fábrica para conexiones a la fuente de datos física. configure(HttpSecurity) |El método anulado define qué rutas URL deben protegerse y cuáles no. .authorizeRequests().anyRequest() .fullyAuthenticated()|Indica que todas las solicitudes a nuestra aplicación deben ser autenticadas. .formLogin()|Configura un inicio de sesión basado en formulario .loginPage("/login").failureUrl("/login?error").permitAll()|Especifica la ubicación de la página de inicio de sesión y todos los usuarios deben tener permiso para acceder a la página. .logout().logoutUrl("/logout") .logoutSuccessUrl("/login?logout").permitAll()|La URL a la que se redirigirá después de cerrar la sesión. El valor predeterminado es /login?logout. .csrf()|Utilizado para evitar la falsificación de solicitudes entre sitios, la protección CSRF está habilitada (predeterminada). configure(AuthenticationManagerBuilder){}|Método anulado para definir cómo se autentican los usuarios. .jdbcAuthentication().dataSource(datasource)|Indica a Spring que estamos usando autenticación JDBC .passwordEncoder(passwordEncoder())|Indica que estamos usando un codificador de contraseñas para codificar nuestras contraseñas. (Se crea un bean para devolver la elección del codificador de contraseña, estamos usando BCrypt en este caso) Tenga en cuenta que no hemos configurado ningún nombre de tabla para usar ni ninguna consulta, esto se debe a que Spring Security busca de forma predeterminada las siguientes tablas:

create table users (   
  username varchar(50) not null primary key,
  password varchar(255) not null,
  enabled boolean not null) ;

create table authorities (
  username varchar(50) not null,
  authority varchar(50) not null,
  foreign key (username) references users (username),
  unique index authorities_idx_1 (username, authority));

Inserte las siguientes filas en las tablas anteriores:

INSERT INTO authorities(username,authority) 
VALUES ('user', 'ROLE_ADMIN');

INSERT INTO users(username,password,enabled)
VALUES('user', '$2a$10$JvqOtJaDys0yoXPX9w47YOqu9wZr/PkN1dJqjG9HHAzMyu9EV1R4m', '1');

El nombre de usuario en nuestro caso es usuario y la contraseña también es usuario encriptada con el algoritmo BCrypt

Finalmente, configure una fuente de datos en application.properties para que Spring Boot la use:

spring.datasource.url = jdbc:mysql://localhost:3306/spring
spring.datasource.username = root
spring.datasource.password = Welcome123

Nota: Cree y configure un controlador de inicio de sesión y asígnelo a la ruta /login y apunte su página de inicio de sesión a este controlador

Hola Primavera Seguridad #

Nota 1: Necesitas algún conocimiento previo sobre ***[servlet java página (JSP)] 1 *** y *** [Apache Maven] 2 *** antes de comenzar esto ejemplos.

Inicie el servidor web (como Apache tomcat) con un proyecto web existente o cree uno.

Visite el index.jsp.

Cualquiera puede acceder a esa página, ¡es insegura!

Aplicación segura #

  1. Actualice las dependencias de Maven

Agregar dependencias a su archivo pom.xml

pom.xml

<dependency>
  <groupId>org.springframework.security</groupId>
  <artifactId>spring-security-web</artifactId>
  <version>4.0.1.RELEASE</version>
</dependency>
<dependency>
  <groupId>org.springframework.security</groupId>
  <artifactId>spring-security-config</artifactId>
  <version>4.0.1.RELEASE</version>
</dependency>

Nota 1: Si no estás usando “Spring” en tu proyecto antes, no hay dependencia sobre “primavera-contexto”. Este ejemplo utilizará la configuración xml con “spring-context”. Así que agregue esta dependencia también.

org.springframework contexto de primavera 4.2.2.LIBERAR

Nota 2: Si no estás usando JSTL en tu proyecto antes, no hay dependencia sobre eso. Este ejemplo usará JSTL en la página jsp. Así que agregue esta dependencia también.

org.glassfish.web javax.servlet.jsp.jstl 1.2.1
  1. Cree el archivo de configuración de Spring Security

Haga que el nombre de la carpeta sea “spring” dentro de la carpeta “WEB-INF” y cree el archivo security.xml. Copie y pegue desde los siguientes códigos.

WEB-INF/primavera/seguridad.xml

<b:beans xmlns="http://www.springframework.org/schema/security"
         xmlns:b="http://www.springframework.org/schema/beans"
         xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
         xsi:schemaLocation="http://www.springframework.org/schema/beans http://www.springframework.org/schema/beans/spring-beans.xsd
                             http://www.springframework.org/schema/security http://www.springframework.org/schema/security/spring-security.xsd">

  <http />

  <user-service>
    <user name="stackoverflow" password="pwd" authorities="ROLE_USER" />
  </user-service>

</b:beans>
  1. Actualizar web.xml

Actualice su web.xml dentro de la carpeta “WEB-INF”

WEB-INF/web.xml

<filter>
  <filter-name>springSecurityFilterChain</filter-name>
  <filter-class>org.springframework.web.filter.DelegatingFilterProxy</filter-class>
</filter>
<filter-mapping>
  <filter-name>springSecurityFilterChain</filter-name>
  <url-pattern>/*</url-pattern>
</filter-mapping>

Nota: Si no estás usando “Spring” en tu proyecto antes, no hay configuraciones sobre la carga de contextos Spring. Así que agregue este parámetro y oyente también.

<parámetro-contexto> <nombre-parámetro>contextConfigLocation</nombre-parámetro> <valor-parámetro> /WEB-INF/primavera/*.xml </valor-parámetro> </context-parámetro>

org.springframework.web.context.ContextLoaderListener

Ejecución de una aplicación web segura #

Después de ejecutar su servidor web y visitar index.jsp, verá la página de inicio de sesión predeterminada generada por Spring Security. Porque no estás autenticado.

Puedes iniciar sesión

username : stackoverflow
password : pwd

Nota: configuración de nombre de usuario y contraseña en WEB-INF/spring/security.xml

Mostrar nombre de usuario #

Agregando la etiqueta jstl después de “Hola”, que imprime el nombre de usuario

índice.jsp

<h1>Hello <c:out value="${pageContext.request.remoteUser}" />!!</h1>

Saliendo de tu cuenta #

índice.jsp

Agregar formulario, etiquetas de entrada después de “Hola nombre de usuario”, ese envío generó el cierre de sesión de la URL /cerrar sesión de Spring Security.

<h1>Hello <c:out value="${pageContext.request.remoteUser}" />!!</h1>

<form action="/logout" method="post">
  <input type="submit" value="Log out" />
  <input type="hidden" name="${_csrf.parameterName}" value="${_csrf.token}" />
</form>

Cuando cierra la sesión con éxito, vuelve a ver la página de inicio de sesión generada automáticamente. Debido a que usted no está autenticado ahora.
