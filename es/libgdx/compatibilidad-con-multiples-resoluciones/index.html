Ventanas #

Para admitir múltiples resoluciones y relaciones de aspecto, Libgdx utiliza los llamados Viewports. Hay algunos tipos de ‘ventanas gráficas’ que usan diferentes estrategias para manejar múltiples resoluciones y relaciones de aspecto. Un Viewport usa una Cámara debajo del capó y administra su viewportHeight y viewportWidth. Opcionalmente, puede asignar a Viewport una Cámara en su constructor; de lo contrario, utilizará una OrthographicCamera de forma predeterminada:

private Viewport viewport;
private Camera camera;

public void create() {
   camera = new PerspectiveCamera();
   viewport = new FitViewport(8f, 4.8f, camera);
}

Además, debe especificar worldWidth y worldHeight en el constructor de la ventana gráfica. Este espacio representará el sistema de coordenadas virtuales que utilizará para especificar la posición y el tamaño de los objetos que se dibujarán. La transformación de la ventana gráfica, que se puede aplicar a un SpriteBatch, por ejemplo, se encargará automáticamente de transformar las coordenadas lógicas en coordenadas de pantalla reales, de una manera que se ajuste al tipo real de ventana gráfica que esté utilizando. Por ejemplo, al usar una proyección ortográfica (“OrthographicCamera”, que es la predeterminada): si el tamaño de su mundo es de 12,8 por 7,8 metros y la pantalla de su dispositivo es de 2560x1560 píxeles, su mundo se proyectará dinámicamente con 200 píxeles por metro.

Cuando cambia el tamaño de la ventana gráfica (por ejemplo, si cambia la orientación de la pantalla del teléfono inteligente), debe informar a la ventana gráfica sobre ese cambio. luego actualizará automáticamente ViewportHeight y viewportWidth de Camera:

public void resize(int width, int height) {
    viewport.update(width, height);
}

Los Viewports también administran el OpenGL Viewport y modifican el área dibujable según sea necesario.

Los Viewports también se encargan de convertir las coordenadas de la pantalla en coordenadas del juego, lo cual es necesario especialmente para elegir:

private Vector2 worldPos = new Vector2();
public boolean touchDown (int x, int y, int pointer, int button) {
    worldPos.set(x, y);
    viewport.unproject(worldPos);
    processPicking(worldPos);
}

Vistas integradas #

Hay un par de ventanas gráficas incorporadas, cada una de las cuales hace cosas diferentes. Aquí hay una lista de los nombres y sus descripciones:

Nombre de la ventana gráfica Descripción
Estirar ventana gráfica Estirará la pantalla. No hay barras negras, pero la relación de aspecto podría estar mal.
Ajustar vista Maximizará su tamaño en función de la relación de aspecto. Podría tener barras negras.
Vista de relleno Casi lo mismo que un FitVieport, pero siempre ocupa toda la pantalla.
Vista de pantalla Siempre llena toda la pantalla, pero no cambia el tamaño de ningún niño.
Vista ampliada Mantiene la relación de aspecto del mundo sin barras negras al extender el mundo en una dirección
Vista personalizada Una ventana gráfica personalizada. Puede tener barras negras y mantener la relación de aspecto.

Vistas personalizadas #

Puede crear su propia ventana gráfica personalizada. Puede tener barras negras, y puede o no mantener su relación de aspecto, dependiendo de cómo lo programes. Una ventana gráfica personalizada se vería así:

Estirar ventana gráfica #

El StretchViewport es un tipo Viewport, que admite un tamaño de pantalla virtual. Esto le permite definir un ancho y una altura fijos (independientes de la resolución). El StretchViewport, como sugiere el nombre, estira la pantalla virtual, si la relación de aspecto virtual no coincide con la relación de aspecto real. La ventana gráfica de OpenGL no se modificará y no aparecerán barras negras.

Uso:

private Viewport viewport;
private Camera camera;

public void create() {
    camera = new OrthographicCamera();
    viewport = new StretchViewport(80, 48, camera);
}

public void resize(int width, int height) {
    viewport.update(width, height);
}

Ajustar vista #

FitViewports son ventanas gráficas que siempre mantienen la relación de aspecto. Lo hace creando barras negras en los bordes donde queda espacio. Esta es una de las vistas más utilizadas.

Uso:

private Viewport viewport;
private Camera camera;

public void create() {
    camera = new OrthographicCamera();
    viewport = new FitViewport(80, 48, camera);
}

public void resize(int width, int height) {
    viewport.update(width, height);
}
