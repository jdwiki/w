Implementar una cámara en OGL 4.0 GLSL 400 #

Si queremos mirar una escena como si la hubiéramos fotografiado con una cámara, primero debemos definir algunas cosas:

  • La posición desde la que se ve la escena, la posición del ojo pos.
  • El punto que miramos en la escena (target). It is also common to define the direction in which we look. Technically we need a line of sight. One straight in space is mathematically defined either by 2 points or by a point and a vector. The first part of the definition is the eye position and the 2nd is either the target or the line of sight vector los.
  • La dirección hacia arriba arriba.
  • El campo de visión ‘fov_y. This means the angle between the two straight lines, starting at the eye position and ending at the leftmost point and the rightmost point, which can be seen simultaneously.
  • El tamaño y la relación de aspecto del viewport al que proyectamos nuestra imagen vp.
  • El plano cercano cerca y el plano lejano lejos. El plano cercano es la distancia desde la posición del ojo hasta el plano from where the objects become visible to us. The far plane is the distance from the eye position to the plane to which the objects of the scene are visible to us. An explanation of what the near plane and far plane are needed will follow later..

Una definición de estos datos en C++ y en Python puede verse así:

C++

using TVec3 = std::array<float,3>;
struct Camera
{
    TVec3 pos    {0.0, -8.0, 0.0};
    TVec3 target {0.0, 0.0, 0.0};
    TVec3 up     {0.0, 0.0, 1.0};
    float fov_y  {90.0};
    TSize vp     {800, 600};
    float near   {0.5};
    float far    {100.0};
};

Para tener en cuenta toda esta información a la hora de dibujar una escena, se suele utilizar una matriz de proyección y una matriz de vista. Para organizar las partes individuales de una escena en la escena, se utilizan matrices modelo. Sin embargo, estos se mencionan aquí solo en aras de la exhaustividad y no se tratarán aquí.

  • Matriz de proyección: La matriz de proyección describe el mapeo desde puntos 3D en el mundo tal como se ven desde una cámara estenopeica, hasta puntos 2D de la ventana gráfica.

  • Ver matriz: La matriz de vista define la posición del ojo y la dirección de visualización en la escena.

  • Modelo matriz: La matriz del modelo define la ubicación y el tamaño relativo de un objeto en la escena.

Después de haber llenado las estructuras de datos anteriores con los datos correspondientes, tenemos que traducirlos a las matrices apropiadas. En el modo de compatibilidad con OGL, esto se puede hacer con las funciones gluLookAt y gluPerspective que configuran los uniformes incorporados gl_ModelViewMatrix, gl_NormalMatrix y gl_ModelViewProjectionMatrix. En OGL 3.1 y GLSL #versión 150, se eliminaron los uniformes incorporados, porque toda la pila de matrices de funciones fijas quedó obsoleta. Si queremos usar el sombreador de alto nivel OGL con GLSL versión 330 o incluso superior, tenemos que definir y establecer los uniformes de matriz nosotros mismos (aparte del uso de la palabra clave compatibilidad de GLSL).

Configurar la perspectiva - Matriz de proyección #

Un punto en la ventana gráfica es visible cuando está en el AABB nativo (cuadro delimitador alineado con el eje) definido por los puntos (-1.0, -1.0, -1.0) y (1.0, 1.0, 1.0). Esto se denomina Coordenadas de dispositivo normalizado (NDC). Se pintará un punto con las coordenadas (-1.0, -1.0, z) en la esquina inferior izquierda de la ventana gráfica y un punto con las coordenadas (1.0, 1.0, z) se pintará en la esquina superior derecha de la ventana gráfica. La coordenada Z se asigna desde el intervalo (-1,0, 1,0) al intervalo (0,0, 1,0) y se escribe en el búfer Z.

Todo lo que podemos ver desde la escena está dentro de una pirámide de 4 lados. La parte superior de la pirámide es la posición de los ojos. Los 4 lados de la pirámide se define por el campo de vista (fov_y) y la relación de aspecto (vp[0]/vp[1]). La matriz de proyección tiene que mapear los puntos desde el interior de la pirámide al NDC definido por los puntos (-1.0, -1.0, -1.0) y (1.0, 1.0, 1.0). En este punto nuestra pirámide es infinita, no tiene fin en profundidad y no podemos mapear un espacio infinito a uno finito. Para esto ahora necesitamos el plano cercano y el plano lejano, ellos transforman la pirámide en un tronco cortando la parte superior y delimitando la pirámide en profundidad. El plano cercano y el plano lejano deben elegirse de tal manera que incluyan todo lo que debe ser visible. de la escena

El mapeo de los puntos dentro de un frustum al NDC es matemática pura y generalmente se puede resolver. El desarrollo de las fórmulas se discutía a menudo y se publicaba repetidamente en la web. Dado que no puede insertar una fórmula de LaTeX en una documentación de desbordamiento de pila, aquí se prescinde de esto. y solo se agrega el código fuente completo de C++ y Python. Tenga en cuenta que las coordenadas del ojo se definen en el sistema de coordenadas de la mano derecha, pero NDC usa el sistema de coordenadas de la mano izquierda. sistema coordinado. La matriz de proyección se calcula a partir del campo de visión fov_y, la relación de aspecto vp[0]/vp[1], el plano cercano cerca y el plano lejano lejos.

C++

using TVec4  = std::array< float, 4 >;
using TMat44 = std::array< TVec4, 4 >;
TMat44 Camera::Perspective( void )
{
    float fn  = far + near;
    float f_n = far - near;
    float r   = (float)vp[0] / vp[1];
    float t   = 1.0f / tan( ToRad( fov_y ) / 2.0f );
    return TMat44{ 
        TVec4{ t / r, 0.0f,  0.0f,                     0.0f },
        TVec4{ 0.0f,  t,     0.0f,                     0.0f },
        TVec4{ 0.0f,  0.0f, -fn / f_n,                -1.0  },
        TVec4{ 0.0f,  0.0f, -2.0f * far * near / f_n,  0.0f } };
}

Pitón

def Perspective(self):

    f_n = self.far - self.near
    r = self.vp[0] / self.vp[1]
    t = 1 / math.tan( math.radians( self.fov_y ) / 2 )

Configurar la mirada a la escena - Ver matriz #

En el sistema de coordenadas de la ventana gráfica, el eje Y apunta hacia arriba (0, 1, 0) y el eje X apunta a la derecha (1, 0, 0). Esto da como resultado un eje Z que apunta fuera de la ventana gráfica. ( (0, 0, -1) = cruz (eje X, eje Y)).

En la escena, el eje X apunta hacia el este, el eje Y hacia el norte y el eje Z hacia arriba.

El eje X de la ventana gráfica (1, 0, 0) coincide con el eje Y de la escena (1, 0, 0), el eje Y de la ventana gráfica (0, 1, 0) coincide con el eje Z de la escena (0, 0, 1) y el eje Z de la ventana gráfica (0, 0, 1) coincide con el eje Y negado de la escena (0, -1, 0).

Por lo tanto, cada punto y cada vector del sistema de referencia de la escena debe convertirse primero en coordenadas de ventana gráfica. Esto se puede hacer mediante algunas operaciones de intercambio e inversión en los vectores escalares.

 x  y  z
--------
 1  0  0  | x' =  x
 0  0  1  | y' =  z
 0 -1  0  | z' = -y

Para configurar una matriz de vista, la posición ‘pos’, el objetivo ‘objetivo’ y el vector ascendente ‘arriba’ deben mapearse en el sistema de coordenadas de la ventana gráfica, como se describe anteriormente. Esto da los 2 puntos p y t y el vector u, como en el siguiente fragmento de código. El eje Z de la matriz de vista es la línea de visión inversa, que se calcula mediante p - t. El eje Y es el vector ascendente u. El eje X se calcula mediante el producto cruzado del eje Y y el eje Z. Para ortonormalizar la matriz de vista, el producto cruzado se usa por segunda vez, para calcular el eje Y a partir del eje Z y el eje X (por supuesto, la ortogonalización de Gram-Schmidt funcionaría igual de bien). Al final, los 3 ejes deben normalizarse y la posición del ojo pos debe establecerse como el origen de la matriz de vista.

El siguiente código define una matriz que encapsula exactamente los pasos necesarios para calcular una mirada a la escena:

  1. Convertir las coordenadas del modelo en coordenadas de ventana gráfica.
  2. Gire en la dirección de la dirección de la vista.
  3. Movimiento a la posición de los ojos.

C++

template< typename T_VEC >
TVec3 Cross( T_VEC a, T_VEC b )
{
    return { a[1] * b[2] - a[2] * b[1], a[2] * b[0] - a[0] * b[2], a[0] * b[1] - a[1] * b[0] };
}

template< typename T_A, typename T_B >
float Dot( T_A a, T_B b )
{ 
    return a[0]*b[0] + a[1]*b[1] + a[2]*b[2]; 
}

template< typename T_VEC >
void Normalize( T_VEC & v )
{ 
    float len = sqrt( v[0] * v[0] + v[1] * v[1] + v[2] * v[2] ); v[0] /= len; v[1] /= len; v[2] /= len; 
}

TMat44 Camera::LookAt( void )
{ 
    TVec3 mz = { pos[0] - target[0], pos[1] - target[1], pos[2] - target[2] };
    Normalize( mz );
    TVec3 my = { up[0], up[1], up[2] };
    TVec3 mx = Cross( my, mz );
    Normalize( mx );
    my = Cross( mz, mx );

    TMat44 v{
        TVec4{ mx[0], my[0], mz[0], 0.0f },
        TVec4{ mx[1], my[1], mz[1], 0.0f },
        TVec4{ mx[2], my[2], mz[2], 0.0f },
        TVec4{ Dot(mx, pos), Dot(my, pos), Dot(TVec3{-mz[0], -mz[1], -mz[2]}, pos), 1.0f }
    };

    return v;
}

pitón

def LookAt(self):
    mz = Normalize( (self.pos[0]-self.target[0], self.pos[1]-self.target[1], self.pos[2]-self.target[2]) ) # inverse line of sight
    mx = Normalize( Cross( self.up, mz ) )
    my = Normalize( Cross( mz, mx ) )
    tx = Dot( mx, self.pos )
    ty = Dot( my, self.pos )
    tz = Dot( (-mz[0], -mz[1], -mz[2]), self.pos )   
    return = numpy.matrix( [ 
        [mx[0], my[0], mz[0], 0],
        [mx[1], my[1], mz[1], 0],
        [mx[2], my[2], mz[2], 0],
        [tx, ty, tz, 1] ] )

Las matrices finalmente se escriben uniformemente y se usan en el sombreador de vértices para transformar las posiciones del modelo.

Sombreador de vértices

En el sombreador de vértices, se realiza una transformación tras otra.

  1. La matriz del modelo trae el objeto (malla) a su lugar en la escena. (Esto solo se incluye en aras de la exhaustividad y no se ha documentado aquí ya que no tiene nada que ver con la vista de la escena)

  2. La matriz de vista define la dirección desde la que se ve la escena. La transformación con la matriz de vista rota los objetos de la escena para que se vean desde el punto deseado. dirección de la vista con referencia al sistema de coordenadas de la ventana gráfica.

  3. La matriz de proyección transforma los objetos de una vista paralela a una vista en perspectiva.

    #version 400

    layout (location = 0) in vec3 inPos; layout (location = 1) in vec3 inCol;

    out vec3 vertCol;

    uniform mat4 u_projectionMat44; uniform mat4 u_viewMat44; uniform mat4 u_modelMat44;

    void main() { vertCol = inCol; vec4 modelPos = u_modelMat44 * vec4( inPos, 1.0 ); vec4 viewPos = u_viewMat44 * modelPos; gl_Position = u_projectionMat44 * viewPos; }

    def CompileShader(self, sourceFileName, shaderStage):

class VAObject:
    def __init__( self, dataArrays, tetIndices ):
        self.__obj = glGenVertexArrays( 1 )
        self.__noOfIndices = len( tetIndices )
        self.__indexArr = numpy.array( tetIndices, dtype='uint' )
        noOfBuffers = len( dataArrays )
        buffers = glGenBuffers( noOfBuffers )
        glBindVertexArray( self.__obj )
        for i_buffer in range( 0, noOfBuffers ):
            vertexSize, dataArr = dataArrays[i_buffer]
            glBindBuffer( GL_ARRAY_BUFFER, buffers[i_buffer] )
            glBufferData( GL_ARRAY_BUFFER, numpy.array( dataArr, dtype='float32' ), GL_STATIC_DRAW )
            glEnableVertexAttribArray( i_buffer )
            glVertexAttribPointer( i_buffer, vertexSize, GL_FLOAT, GL_FALSE, 0, None )
    def Draw(self):
        glBindVertexArray( self.__obj )
        glDrawElements( GL_TRIANGLES, self.__noOfIndices, GL_UNSIGNED_INT, self.__indexArr )

    def Fract( self, val ): return val - math.trunc(val)
    def CalcAng( self, currentTime, intervall ): return self.Fract( (currentTime - self.__startTime) / intervall ) * 2.0 *     math.pi
    def CalcMove( self, currentTime, intervall, range ):
        pos = self.Fract( (currentTime - self.__startTime) / intervall ) * 2.0
        pos = pos if pos < 1.0 else (2.0-pos)
        return range[0] + (range[1] - range[0]) * pos
    def EllipticalPosition( self, a, b, angRag ):
        a_b = a * a - b * b
        ea = 0 if (a_b <= 0) else math.sqrt( a_b )
        eb = 0 if (a_b >= 0) else math.sqrt( -a_b )
        return ( a * math.sin( angRag ) - ea, b * math.cos( angRag ) - eb, 0 )
