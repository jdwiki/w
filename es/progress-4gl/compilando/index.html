Compile el código de progreso como se llama “r-code” y normalmente se guarda en un archivo con la extensión .r. Hay un par de formas diferentes de compilar: usando la instrucción COMPILE o en Linux o AppBuilder: el compilador de aplicaciones integrado. Developer Studio (el entorno de Eclipse) tiene compilación integrada en su proceso de compilación.

Debe tener instalado 4GL Development u OpenEdge Studio para compilar programas 4GL que actualicen la base de datos.

Sintaxis #

  • COMPILAR programa.p GUARDAR. //Compila program.p y guarda su código r
  • VALOR COMPILAR (var) GUARDAR. //Compila el nombre guardado en la variable “var” y guarda su código r
  • COMPILAR prog.p XREF prog.xref LISTING prog.list. //Compilar prog.p y crear xref y listados de archivos. No guarde el código r.
  • COMPILAR programa.p GUARDAR SIN ERROR. //Compila el programa.p, guarda el código r y suprime los errores para detener la ejecución.

Compilador de aplicaciones #

Generador de aplicaciones de Windows

En Windows Appbuilder, el compilador de aplicaciones se encuentra en el menú Herramientas.

Editor de procedimientos (Linux - pro o Windows pro.exe

En el Editor de Procedimientos (tanto Linux como Windows) el Compilador si se encuentra en el menú Herramientas.

Compilador de aplicaciones

Independientemente del sistema operativo, la funcionalidad del compilador es la misma. Puede agregar directorios y/o archivos y compilarlos.

Configuración principal (más abajo):

  • Guardar nuevo archivo .r. Si no se marca, los archivos simplemente se compilarán pero no se guardarán. Útil para el seguimiento de errores, por ejemplo.
  • Buscar en Subdirectorios. De lo contrario, habrá que añadir subdirectorios.
  • Eliminar archivos .r antiguos. Sobrescribir el archivo .r antiguo.
  • Compilación Onlu si no hay archivo .r. Solo compila archivos sin compilar.

Opciones:

  • Propath: le muestra el propath y le permite seleccionar directorios para compilar a partir de él.
  • Agregar: le permite ingresar un directorio o archivo.
  • Modificar: le permite modificar una entrada existente.
  • Eliminar: elimina una entrada.
  • Iniciar compilación: inicia el compilador. Atajo: F2

Las opciones del menú principal:

  • Archivo -> Salir: Sale del compilador
  • Compilar -> Iniciar compilación: inicia el compilador. Atajo: F2
  • Herramientas -> Acceso a otras herramientas
  • Opción -> Compilador… : Configuración, ver más abajo.
  • Ayuda -> Ayuda de OpenEdge (solo Windows). Ayuda en linea. Atajo: F1

Ajustes

  • Especificación de archivo predeterminada: extensiones de nombre de archivo para compilar
  • Archivo de registro de mensajes: archivo para guardar mensajes, advertencias y errores en
  • Guardar en: Dónde almacenar el archivo .r. Si está en blanco el mismo directorio que el código.
  • Idiomas: para traducciones. No cubierto aquí.
  • V6Frame: Viejo e inservible…
  • Steam-IO: si desea imprimir la salida del compilador. Probablemente no.
  • Archivo de listado: si desea que el compilador cree un archivo de listado. Útil para la depuración
  • Agregar: agregar al archivo de listado existente. De lo contrario, sobrescribir.
  • Ancho de página + Longitud: Formato del archivo de listado.
  • Archivo Xref: si desea que el compilador cree un XREF. Útil para depurar, verificar el uso del índice, etc.
  • Formato XML: si la referencia externa del compilador debe ser un xml. De lo contrario, texto “sin formato”.
  • Agregar: agregar al archivo de referencia externa existente. De lo contrario, sobrescribir.
  • Archivo de depuración: archivo de listado del depurador.
  • Clave de cifrado: si el archivo de origen está cifrado con xcode, inserte la clave aquí.
  • Minimizar el tamaño del código R: elimine parte de la información de depuración para mantener el código R pequeño.
  • Generar MD-5: Principalmente para la compilación WebClient.

Uso básico

  1. Inicie el compilador
  2. Agregue una ruta (si aún no la guardó de la última sesión)
  3. Presione F2 para compilar.
  4. Observe cualquier error.
  5. Salir

instrucción COMPILAR #

La declaración de compilación le permite compilar programas en Progress ABL:

Uso básico:

COMPILE hello-world.p SAVE.

Con una variable:

DEFINE VARIABLE prog AS CHARACTER   NO-UNDO.

prog = "hello.p".

COMPILE VALUE(prog) SAVE.

Hay varias opciones para la sentencia COMPILE:

SAVE indica que el código .r debe guardarse para uso futuro.

COMPILE hello-world.p SAVE.

SAVE INTO dir OR SAVE INTO VALUE(dir-variable) guarda el código r en el directorio especificado:

COMPILE hello-world.p SAVE INTO /usr/sources.

Archivo LISTADO. Crea un archivo de lista que contiene información de depuración sobre bloques, incluye, etc.

COMPILE program.p SAVE LISTING c:\temp\listing.txt.

La lista tiene un par de opciones para agregar archivos, tamaño de página y ancho de página:

APPEND PAGE-SIZE num PAGE-WIDTH num

XREF xreffile guardará un archivo de referencia externa del compilador que contiene información sobre el uso de cadenas e índices, etc. También puede APPEND este.

COMPILE checkFile.p SAVE XREF c:\directory\xref-file.txt.

XREF-XML xreffile-or-dir hará lo mismo que XREF pero guardará el archivo en formato xml. Si usa un directorio, el archivo de referencia externa se llamará nombre del programa.xref.xml.

COMPILE file.p SAVE XREF c:\temp\.

NO-ERROR suprimirá cualquier error al detener su programa.

COMPILE program SAVE NO-ERROR.

Archivo DEBUG-LIST genera un archivo de depuración con números de línea.

COMPILE checkFile.p SAVE DEBUG-LIST c:\temp\debug.txt.

Archivo PREPROCESO primero traducirá todos los preprocesadores y luego creará un nuevo archivo .p con el código antes de compilar.

 COMPILE checkFile.p SAVE PREPROCESS c:\temp\PREPROC.txt.

Clave XCODE compilará un código fuente encriptado con clave como clave. No puede usar XCODE con las opciones XREF, XREF-XML, STRING-XREF o LISTING juntas.

COMPILE program.p SAVE XCODE myKey.

Puedes combinar varias opciones:

COMPILE prog.p SAVE INTO /usr/r-code XREF /usr/xrefs/xref.txt APPEND LISTING /usr/listings.txt APPEND NO-ERROR.

Manejador del sistema COMPILER #

El identificador del sistema COMPILER le permite ver información sobre una compilación reciente.

Suponiendo que ok-program.p es un programa sin errores ni advertencias:

COMPILE ok-program.p SAVE NO-ERROR.

DEFINE VARIABLE iError AS INTEGER     NO-UNDO.

MESSAGE  
    "Errors: "   COMPILER:ERROR SKIP
    "Warnings: " COMPILER:WARNING SKIP
    "Messages: " COMPILER:NUM-MESSAGES
    VIEW-AS ALERT-BOX INFORMATION.

Esto producirá:

Compilando un programa con una advertencia:

/* program-with-warning.p */
DEFINE VARIABLE c AS CHARACTER   NO-UNDO.
DEFINE VARIABLE i AS INTEGER     NO-UNDO.

c = "hello".
DISPLAY c.
//This RETURN makes the program exit here and the code below unreachable.
RETURN.

IF TRUE THEN DO:
    i = 10. 
END.

Compilando el programa:

COMPILE program-with-warning.p SAVE.

DEFINE VARIABLE iError AS INTEGER     NO-UNDO.

MESSAGE  
    "Errors: "   COMPILER:ERROR SKIP
    "Warnings: " COMPILER:WARNING SKIP
    "Messages: " COMPILER:NUM-MESSAGES
    VIEW-AS ALERT-BOX INFORMATION.

DO iError = 1 TO COMPILER:NUM-MESSAGES:
    DISPLAY 
        COMPILER:GET-FILE-NAME(iError)    LABEL "Filename" FORMAT "x(20)"
        COMPILER:GET-MESSAGE(iError)      LABEL "Message"  FORMAT "x(50)"
        COMPILER:GET-NUMBER(iError)       LABEL "Msg#"  
        COMPILER:GET-ERROR-COLUMN(iError) LABEL "Column"
        COMPILER:GET-ERROR-ROW(iError)    LABEL "Row"
            WITH FRAME fr1 SIDE-LABELS 1 COLUMNS.
END.

Resultado:

Compilando un programa con un error

DEFINE VARIABLE c AS CHARACTER   NO-UNDO.
DEFINE VARIABLE i AS INTEGER     NO-UNDO.

c = "hello".
DISPLAY c.
//Casting should be required below...
IF TRUE THEN DO:
    i = c.
END.

Compilando el programa:

//Use no-errors to supress any error messages from interrupting us.
COMPILE c:\temp\program-with-error.p SAVE NO-ERROR.

DEFINE VARIABLE iError AS INTEGER     NO-UNDO.

MESSAGE  
    "Errors: "   COMPILER:ERROR SKIP
    "Warnings: " COMPILER:WARNING SKIP
    "Messages: " COMPILER:NUM-MESSAGES
    VIEW-AS ALERT-BOX INFORMATION.

DO iError = 1 TO COMPILER:NUM-MESSAGES:
    DISPLAY 
        COMPILER:GET-FILE-NAME(iError)    LABEL "Filename" FORMAT "x(20)"
        COMPILER:GET-MESSAGE(iError)      LABEL "Message"  FORMAT "x(50)"
        COMPILER:GET-NUMBER(iError)       LABEL "Msg#"  
        COMPILER:GET-ERROR-COLUMN(iError) LABEL "Column"
        COMPILER:GET-ERROR-ROW(iError)    LABEL "Row"
            WITH FRAME fr1 SIDE-LABELS 1 COLUMNS 20 DOWN.

    DOWN WITH FRAME fr1.
END.
