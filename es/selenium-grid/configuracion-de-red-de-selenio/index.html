Instalación o configuración #

Antes de configurar una cuadrícula de Selenium, debe asegurarse de tener Java instalado y configurado en la ruta del entorno de su computadora.

Configurar el concentrador #

  • Descargue la última versión estable de Selenium Server.
  • Inicie el símbolo del sistema y navegue hasta la ubicación en la que colocó el archivo jar del servidor Selenium.
  • Escriba: (FYI: su número de versión puede ser diferente al mío) java –jar selenium-server-standalone-2.53.0.jar –centro de funciones
  • Ahora debería verse algo como esto:

Básicamente, lo que sucedió es que se inició el servidor web Selenium y ahora está escuchando en un puerto; en este caso, el predeterminado 4444 (FYI: este número de puerto se puede cambiar pasando el parámetro -port seguido del número de puerto en el que desea para ejecutar el servidor).

  • Ahora abra un navegador y vaya a http://localhost:4444/grid/console
  • Si todo funciona, debería aparecer el servidor y verías algo como esto:

A continuación, necesitamos configurar algunas máquinas de nodo.

Configurar los Nodos #

  • Al igual que descargamos Selenium Server para Hub, también debemos descargarlo en todas nuestras máquinas Node .
  • Una vez que tenga el archivo jar del servidor Selenium en la máquina del nodo, navegue hasta el directorio donde se descarga el jar y abra el símbolo del sistema cmd.
  • Escriba: java –jar selenium-server-standalone-2.53.0.jar –role node –hub http://hubIP:4444/grid/register
    • hubIP :- in case the hub and nodes are running on a different machine
    • localhost :- in case the hub and nodes are running on same machine

Como puede ver, el nodo ahora está registrado en el concentrador, de manera predeterminada, el nodo comienza en -puerto 5555, pero puede cambiarlo usando el parámetro -puerto seguido del número de puerto.

Si todo funciona como se esperaba, ahora debería ver la dirección IP del nodo que acaba de iniciar y registrar en la vista de la consola central:

Cosas a tener en cuenta #

  • Si no especificamos el seleniumProtocol, el nodo se registrará tanto con el control remoto (heredado) como con el protocolo Webdriver (como se ve en la captura de pantalla anterior).
  • Si no se menciona el tipo de navegador y la cantidad de instancias, Node lanzará 5 instancias de Firefox, 5 instancias de Chrome y 1 instancia del controlador IE.

Eso es todo lo que necesitaría hacer para tener Selenium Grid en funcionamiento.

Configuración y uso en C# #

Configuración #

En los siguientes párrafos habrá un ejemplo por navegador para la configuración en Json y la instalación en C#.

Este ejemplo espera que tenga todos los controladores en su variable de ruta y navegadores instalados. ###Microsoft Edge ###

Código C# para crear un controlador web remoto

// Defining webdriver variable
RemoteWebDriver _webDriver;
// Creating Capabilities and chosing browser
capabiliteiten = DesiredCapabilities.Edge();
// Setting platform 
capabiliteiten.Platform = new Platform(PlatformType.Windows);
// Requesting remote webdriver
_webDriver = new RemoteWebDriver(_gridServerUri, capabiliteiten);

Configuración de nodos en Json

{
    "browserName":"MicrosoftEdge",
    "platform": "WINDOWS",
    "maxIstances": 1,
    "seleniumProtocol": "WebDriver"
}

Ópera #

Código C# para crear un controlador web remoto Esto es para OperaChromium

// Defining webdriver variable
RemoteWebDriver _webDriver;
// Creating Capabilities
capabiliteiten = new DesiredCapabilities();
// Setting platform 
capabiliteiten.Platform = new Platform(PlatformType.Windows);
// Chosing browser
capabiliteiten.SetCapability(CapabilityType.BrowserName, "operablink");
// Requesting remote webdriver
_webDriver = new RemoteWebDriver(_gridServerUri, capabiliteiten);

Configuración de nodos en Json

{
    "browserName": "operablink",
    "platform": "WINDOWS",
    "maxInstances": 5,
    "seleniumProtocol": "WebDriver"
}

El tipo de plataforma puede ser uno de los siguientes:

  • Tipo de plataforma.Android;
  • Tipo de plataforma.Cualquiera;
  • Tipo de plataforma.Linux;
  • Tipo de plataforma.Mac;
  • PlatformType.Unix;
  • PlatformType.Vista;
  • Tipo de plataforma.Windows;
  • Tipo de plataforma. WinNT;
  • Tipo de plataforma.XP;
