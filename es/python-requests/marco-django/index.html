Instalación y configuración #

Django es un marco de trabajo completo para el desarrollo web. Alimenta algunos de los sitios web más populares de Internet.

Para instalar el marco; usa la herramienta pip:

pip install django

Si está instalando esto en OSX o Linux, el comando anterior puede generar un error de permiso; para evitar este error, instale el paquete para su cuenta de usuario o use un entorno virtual:

pip install --user django

Una vez que esté instalado, tendrá acceso a la herramienta de arranque django-admin, que creará un directorio con algunos valores predeterminados para comenzar el desarrollo.

django-admin startproject myproject

Esto creará un directorio myproject con el diseño de proyecto predeterminado.

Conceptos básicos de Django #

Django es un marco de desarrollo web rico en funciones y de pila completa. Reúne una gran cantidad de funciones para proporcionar una experiencia común, rápida y productiva para los desarrolladores web.

Los proyectos de Django consisten en configuraciones comunes y una o más aplicaciones. Cada aplicación es un conjunto de funciones junto con dependencias (como plantillas y modelos) que se agrupan como módulos de Python.

El script de arranque de django crea automáticamente un archivo de configuración para su proyecto, con las funciones más comunes habilitadas.

Este concepto de aplicaciones permite una funcionalidad fácil de conectar y usar, y hay una gran biblioteca de aplicaciones disponibles para manejar las tareas más comunes. Este concepto de aplicaciones es fundamental para Django; muchas de las funciones integradas (como la autenticación de usuarios y el sitio de administración) son simplemente aplicaciones de django.

Para crear su primera aplicación, desde el directorio del proyecto:

python manage.py startapp yourapp

yourapp es el nombre de su aplicación personalizada.

Cada aplicación le permite desarrollar:

  1. Una serie de vistas: son piezas de código que se ejecutan en respuesta a una solicitud.

  2. Uno o más modelos; que son una abstracción de las bases de datos. Estos le permiten definir sus objetos como objetos de Python, y el ORM incorporado proporciona una API amigable para almacenar, recuperar y filtrar objetos de las bases de datos.

  3. Muy relacionado con los modelos están las migraciones, que son secuencias de comandos que se generan para proporcionar un método consistente y confiable para aplicar cambios en sus modelos a la base de datos.

  4. Un conjunto de urls a las que responderá la aplicación.

  5. Una o más clases de administración; para personalizar cómo se comporta la aplicación en la aplicación de administración de django incorporada.

  6. Cualquier prueba que pueda escribir.

Conceptos básicos - Vistas #

Una vista es cualquier fragmento de código que responde a una solicitud y devuelve una respuesta. Las vistas normalmente devuelven plantillas junto con un diccionario (llamado context) que normalmente contiene datos para marcadores de posición en la plantilla. En los proyectos de Django, las vistas se encuentran en el módulo views.py de las aplicaciones.

La vista más simple, devuelve una respuesta directa:

from django.http import HttpResponse

def simple_view(request):
   return HttpResponse('<strong>Hello World</strong>')

Sin embargo, la mayoría de las vistas utilizan una plantilla:

from django.shortcuts import render

def simple_template_view(request):
    return render(request, 'some_template.html')

Una plantilla es simplemente cualquier archivo y, opcionalmente, puede contener un marcado especial para una funcionalidad adicional; lo que esto significa es que las vistas de django pueden devolver cualquier tipo de respuesta, no solo HTML.

Conceptos básicos - Plantillas #

En Django, una plantilla es simplemente un archivo que contiene etiquetas especiales que pueden ser reemplazadas por datos de la vista.

El ejemplo de plantilla canónica sería:

<strong>Hello {{ name }}, I am a template!</strong>

Aquí, la cadena {{ nombre }} identifica un marcador de posición que puede ser reemplazado por un contexto.

Para representar esta plantilla desde una vista, podemos pasar el valor como un diccionario:

from django.shortcuts import render

def simple_view(request):
    return render(request, 'template.html', {'name': 'Jim'})

Una vez que se represente esta vista, el código HTML resultante será Hola Jim, ¡soy una plantilla!.

Conceptos básicos - URL #

En django, hay un mapeador de URL que asigna URL a funciones específicas (vistas) que devuelven respuestas. Esta separación estricta entre el diseño del sistema de archivos y el diseño de URL permite una gran flexibilidad al escribir aplicaciones.

Todos los patrones de URL se almacenan en uno o más archivos urls.py, y hay un archivo maestro urls.py que django lee primero.

Django analiza los patrones en el orden en que están escritos y se detiene cuando encuentra una coincidencia con la URL solicitada por el usuario. Si no se encuentran coincidencias, se genera un error.

En el modo de depuración (activado configurando DEBUG = True en settings.py), django imprimirá un mensaje de error detallado cuando una URL solicitada no coincida con ningún patrón. En producción, sin embargo, Django mostrará un mensaje 404 normal.

Un patrón de URL consiste en una expresión regular de Python, seguida de un callable (un método o función) que se llamará cuando ese patrón coincida. Esta función debe devolver una respuesta HTTP:

url(r'/hello$', simple_view)
