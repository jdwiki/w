Uso avanzado de println! #

println! (y su hermano, [print!](https://doc.rust-lang. org/std/macro.print!.html)) proporciona un mecanismo conveniente para producir e imprimir texto que contiene datos dinámicos, similar a la familia de funciones printf que se encuentra en muchos otros lenguajes. Su primer argumento es una cadena de formato, que dicta cómo deben imprimirse los otros argumentos como texto. La cadena de formato puede contener marcadores de posición (dentro de {}) para especificar que debe ocurrir una sustitución:

// No substitution -- the simplest kind of format string
println!("Hello World");
// Output: Hello World

// The first {} is substituted with a textual representation of
// the first argument following the format string. The second {}
// is substituted with the second argument, and so on.
println!("{} {} {}", "Hello", true, 42);
// Output: Hello true 42

En este punto, puede que se pregunte: ¿Cómo supo println! imprimir el valor booleano true como la cadena “true”? {} es realmente una instrucción para el formateador de que el valor debe convertirse en texto usando el rasgo Display . Este rasgo se implementa para la mayoría de los tipos primitivos de Rust (cadenas, números, booleanos, etc.) y está destinado a “salida orientada al usuario”. Por lo tanto, el número 42 se imprimirá en decimal como 42 y no, digamos, en binario, que es como se almacena internamente.

Entonces, ¿cómo imprimimos tipos que no implementan Display, por ejemplo, Slices ([i32]), vectores (Vec<i32>) u opciones (Option<&str>)? No hay una representación textual clara para el usuario de estos (es decir, uno que pueda insertar trivialmente en una oración). Para facilitar la impresión de tales valores, Rust también tiene el rasgo Debug y el correspondiente {:?} marcador de posición. De la documentación: “Debug debe formatear la salida en un contexto de depuración orientado al programador”. Veamos algunos ejemplos:

println!("{:?}", vec!["a", "b", "c"]);
// Output: ["a", "b", "c"]

println!("{:?}", Some("fantastic"));
// Output: Some("fantastic")

println!("{:?}", "Hello");
// Output: "Hello"
// Notice the quotation marks around "Hello" that indicate
// that a string was printed.

Debug también tiene un mecanismo incorporado de impresión bonita, que puede habilitar usando el modificador # después de los dos puntos:

println!("{:#?}", vec![Some("Hello"), None, Some("World")]);
// Output: [
//    Some(
//        "Hello"
//    ),
//    None,
//    Some(
//        "World"
//    )
// ]

Las cadenas de formato le permiten expresar bastante [sustituciones complejas] (https://doc.rust-lang.org/std/fmt/):

// You can specify the position of arguments using numerical indexes.
println!("{1} {0}", "World", "Hello");
// Output: Hello World

// You can use named arguments with format
println!("{greeting} {who}!", greeting="Hello", who="World");
// Output: Hello World

// You can mix Debug and Display prints:
println!("{greeting} {1:?}, {0}", "and welcome", Some(42), greeting="Hello");
// Output: Hello Some(42), and welcome

println! y sus amigos también le advertirán si está tratando de hacer algo que no funcionará, en lugar de fallar en el tiempo de ejecución:

// This does not compile, since we don't use the second argument.
println!("{}", "Hello World", "ignored");

// This does not compile, since we don't give the second argument.
println!("{} {}", "Hello");

// This does not compile, since Option type does not implement Display
println!("{}", Some(42));

En esencia, las macros de impresión de Rust son simplemente envoltorios alrededor de la macro format!, que permite construir una cadena uniendo juntas representaciones textuales de diferentes valores de datos. Por lo tanto, para todos los ejemplos anteriores, puede sustituir println! por formato! para almacenar la cadena formateada en lugar de imprimirla:

let x: String = format!("{} {}", "Hello", 42);
assert_eq!(x, "Hello 42");

Salida de consola sin macros #

// use Write trait that contains write() function
use std::io::Write;

fn main() {
    std::io::stdout().write(b"Hello, world!\n").unwrap();
}

  • El rasgo std::io::Write está diseñado para objetos que aceptan flujos de bytes. En este caso, se adquiere un controlador para la salida estándar con std::io::stdout().

  • Write::write() acepta un segmento de bytes (&[u8]), que se crea con una cadena de bytes literal (b"<string>"). Write::write() devuelve Result<usize, IoError>, que contiene el número de bytes escritos (en caso de éxito) o un valor de error (en caso de falla).

  • La llamada a Result::unwrap() indica que se espera que la llamada tenga éxito (Result<usize, IoError> -> usize), y el valor se descarta.

Ejemplo mínimo #

Para escribir el programa tradicional Hello World en Rust, cree un archivo de texto llamado hello.rs que contenga el siguiente código fuente:

fn main() {
    println!("Hello World!");
}

Esto define una nueva función llamada main, que no toma parámetros y no devuelve datos. Aquí es donde su programa comienza a ejecutarse cuando se ejecuta. En su interior, tiene println!, que es una macro que imprime texto en la consola.

Para generar una aplicación binaria, invoque el compilador Rust pasándole el nombre del archivo fuente:

$ rustc hello.rs

El ejecutable resultante tendrá el mismo nombre que el módulo fuente principal, por lo que para ejecutar el programa en un sistema Linux o MacOS, ejecute:

$ ./hello
Hello World!

En un sistema Windows, ejecute:

C:\Rust> hello.exe
Hello World!

Empezando #

Instalación #

Antes de que pueda hacer cualquier cosa usando el lenguaje de programación Rust, necesitará adquirirlo—ya sea para Windows o usando su terminal en sistemas similares a Unix, donde $ simboliza la entrada en el Terminal:

$ curl https://sh.rustup.rs -sSf | sh

Esto recuperará los archivos requeridos y configurará la última versión de Rust para usted, sin importar en qué sistema se encuentre. Para obtener más información, consulte página del proyecto.

_Nota: Algunas distribuciones de Linux (por ejemplo, [Arch Linux]) proporcionan rustup como paquete, que se puede instalar en su lugar. Y aunque muchos sistemas similares a Unix proporcionan rustc y cargo como paquetes separados, aún se recomienda usar rustup en su lugar, ya que hace que sea mucho más fácil administrar múltiples canales de publicación y realizar compilaciones cruzadas. _

compilador de óxido #

Ahora podemos verificar si Rust se instaló con éxito en nuestras computadoras ejecutando el siguiente comando en nuestra terminal, si está en UNIX, o en el símbolo del sistema, si está en Windows:

$ rustc --version

Si este comando tiene éxito, la versión del compilador de Rust instalada en nuestras computadoras se mostrará ante nuestros ojos.

Carga #

Con Rust viene Cargo, que es una herramienta de compilación utilizada para administrar sus paquetes y proyectos Rust. Para asegurarse de que esto también esté presente en su computadora, ejecute lo siguiente dentro de la consola: la consola se refiere a la terminal o al símbolo del sistema, según el sistema en el que se encuentre:

$ cargo --version

Al igual que el comando equivalente para el compilador Rust, esto devolverá y mostrará la versión actual de Cargo.

Para crear su primer proyecto Cargo, puede dirigirse a [Cargo][2].

Alternativamente, puede compilar programas directamente usando rustc como se muestra en [Ejemplo mínimo][3].
