Sintaxis #

  • estructura Foo {campo1: Tipo1, campo2: Tipo2}
  • let foo = Foo { campo1: Tipo1::nuevo(), campo2: Tipo2::nuevo() };
  • Barra de estructura (Tipo1, Tipo2); // tipo de tupla
  • let _ = Bar(Tipo1::nuevo(), Tipo2::nuevo());
  • estructura Baz; // tipo de unidad
  • sea _ = Baz;
  • let Foo { campo1, .. } = foo; // extrae campo1 por coincidencia de patrones
  • let Foo { campo1: x, .. } = foo; // extrae campo1 como x
  • let foo2 = Foo { campo1: Tipo1::nuevo(), .. foo }; // construir a partir de lo existente
  • impl Foo { fn fiddle(&self) {} } // declara el método de instancia para Foo
  • impl Foo { fn tweak(&mut self) {} } // declara el método de instancia mutable para Foo
  • impl Foo { fn double(self) {} } // declara el método de instancia propietario para Foo
  • impl Foo { fn new() {} } // declara el método asociado para Foo

Crear y usar valores de estructura #

Considere las siguientes definiciones de struct:

struct Foo {
    my_bool: bool,
    my_num: isize,
    my_string: String,
}
struct Bar (bool, isize, String);
struct Baz;

La construcción de nuevos valores de estructura para estos tipos es sencilla:

let foo = Foo { my_bool: true, my_num: 42, my_string: String::from("hello") };
let bar = Bar(true, 42, String::from("hello"));
let baz = Baz;

Acceda a los campos de una estructura usando .:

assert_eq!(foo.my_bool, true);
assert_eq!(bar.0, true); // tuple structs act like tuples

Un enlace mutable a una estructura puede tener sus campos mutados:

let mut foo = foo;
foo.my_bool = false;
let mut bar = bar;
bar.0 = false;

Las capacidades de coincidencia de patrones de Rust también se pueden usar para mirar dentro de una estructura:

// creates bindings mb, mn, ms with values of corresponding fields in foo
let Foo { my_bool: mb, my_num: mn, my_string: ms } = foo;
assert_eq!(mn, 42);
// .. allows you to skip fields you do not care about
let Foo { my_num: mn, .. } = foo;
assert_eq!(mn, 42);
// leave out `: variable` to bind a variable by its field name
let Foo { my_num, .. } = foo;
assert_eq!(my_num, 42);

O crea una estructura usando una segunda estructura como una “plantilla” con la actualización de sintaxis de Rust:

let foo2 = Foo { my_string: String::from("world"), .. foo };
assert_eq!(foo2.my_num, 42);

Definición de estructuras #

Las estructuras en Rust se definen usando la palabra clave struct. La forma más común de estructura consiste en un conjunto de campos con nombre:

struct Foo {
    my_bool: bool,
    my_num: isize,
    my_string: String,
}

Lo anterior declara una struct con tres campos: my_bool, my_num y my_string, de los tipos bool, isize y String respectivamente.

Otra forma de crear estructuras en Rust es crear una estructura de tupla:

struct Bar (bool, isize, String);

Esto define un nuevo tipo, Bar, que tiene tres campos sin nombre, de tipo bool, isize y String, en ese orden. Esto se conoce como patrón nuevotipo, porque introduce efectivamente un nuevo “nombre” para un tipo en particular. Sin embargo, lo hace de una manera más potente que los alias creados con la palabra clave tipo; Bar es aquí un tipo completamente funcional, lo que significa que puede escribir sus propios métodos para él (abajo).

Finalmente, declara una estructura con sin campos, llamada estructura tipo unidad:

struct Baz;

Esto puede ser útil para burlarse o probar (cuando desea implementar un rasgo de manera trivial), o como un tipo de marcador. En general, sin embargo, es poco probable que encuentre muchas estructuras similares a unidades.

Tenga en cuenta que los campos struct en Rust son todos privados de forma predeterminada, es decir, no se puede acceder a ellos desde el código fuera del módulo que define el tipo. Puede prefijar un campo con la palabra clave pub para que ese campo sea accesible públicamente. Además, el tipo struct en sí mismo es privado. Para que el tipo esté disponible para otros módulos, la definición de struct también debe tener el prefijo pub:

pub struct X {
    my_field: bool,
    pub our_field: bool,
}

Estructuras genéricas #

Las estructuras se pueden hacer genéricas sobre uno o más parámetros de tipo. Estos tipos se dan encerrados en <> cuando se refieren al tipo:

struct Gen<T> {
    x: T,
    z: isize,
}

// ...
let _: Gen<bool> = Gen{x: true, z: 1};
let _: Gen<isize> = Gen{x: 42, z: 2};
let _: Gen<String> = Gen{x: String::from("hello"), z: 3};

Se pueden dar varios tipos usando comas:

struct Gen2<T, U> {
    x: T,
    y: U,
}

// ...
let _: Gen2<bool, isize> = Gen2{x: true, y: 42};

Los parámetros de tipo son parte del tipo, por lo que dos variables del mismo tipo base, pero con diferentes parámetros, no son intercambiables:

let mut a: Gen<bool> = Gen{x: true, z: 1};
let b: Gen<isize> = Gen{x: 42, z: 2};
a = b; // this will not work, types are not the same
a.x = 42; // this will not work, the type of .x in a is bool

Si desea escribir una función que acepte una estructura independientemente de su tipo de asignación de parámetros, esa función también deberá convertirse en genérica:

fn hello<T>(g: Gen<T>) {
    println!("{}", g.z); // valid, since g.z is always an isize
}

Pero, ¿y si quisiéramos escribir una función que siempre pudiera imprimir g.x? Necesitaríamos restringir T para que sea de un tipo que se pueda mostrar. Podemos hacer esto con límites de tipo:

use std::fmt;
fn hello<T: fmt::Display>(g: Gen<T>) {
    println!("{} {}", g.x, g.z);
}

La función hello ahora está solo definida para instancias Gen cuyo tipo T implementa fmt::Display. Si intentáramos pasar Gen<(bool, isize)> por ejemplo, el compilador se quejaría de que hello no está definido para ese tipo.

También podemos usar límites de tipo directamente en los parámetros de tipo de la estructura para indicar que solo puedes construir esa estructura para ciertos tipos:

use std:#️⃣:Hash;
struct GenB<T: Hash> {
    x: T,
}

Cualquier función que tenga acceso a un ‘GenB’ ahora sabe que el tipo de ‘x’ implementa ‘Hash’ y, por lo tanto, puede llamar a ‘.x.hash()’. Se pueden dar límites de tipos múltiples para el mismo parámetro separándolos con un +.

Al igual que para las funciones, los límites de tipo se pueden colocar después de <> usando la palabra clave where:

struct GenB<T> where T: Hash {
    x: T,
}

Esto tiene el mismo significado semántico, pero puede hacer que la firma sea más fácil de leer y formatear cuando tiene límites complejos.

Los parámetros de tipo también están disponibles para los métodos de instancia y los métodos asociados de struct:

// note the <T> parameter for the impl as well
// this is necessary to say that all the following methods only
// exist within the context of those type parameter assignments
impl<T> Gen<T> {
    fn inner(self) -> T {
        self.x
    }
    fn new(x: T) -> Gen<T> {
        Gen{x: x}
    }
}

Si tiene límites de tipo en la T de Gen, estos también deberían reflejarse en los límites de tipo de impl. También puede hacer que los límites impl sean más estrictos para decir que un método dado solo existe si el tipo satisface una propiedad particular:

impl<T: Hash + fmt::Display> Gen<T> {
    fn show(&self) {
        println!("{}", self.x);
    }
}

// ...
Gen{x: 42}.show(); // works fine
let a = Gen{x: (42, true)}; // ok, because (isize, bool): Hash
a.show(); // error: (isize, bool) does not implement fmt::Display

Métodos de estructura #

Para declarar métodos en una estructura (es decir, funciones que se pueden llamar “en” la estructura, o valores de ese tipo de estructura), cree un bloque “impl”:

impl Foo {
    fn fiddle(&self) {
        // "self" refers to the value this method is being called on
        println!("fiddling {}", self.my_string);
    }
}

// ...
foo.fiddle(); // prints "fiddling hello"

&self aquí indica que es necesaria una referencia inmutable a una instancia de struct Foo para invocar el método fiddle. Si quisiéramos modificar la instancia (como cambiar uno de sus campos), en su lugar tomaríamos un &mut self (es decir, una referencia mutable):

impl Foo {
    fn tweak(&mut self, n: isize) {
        self.my_num = n;
    }
}

// ...
foo.tweak(43);
assert_eq!(foo.my_num, 43);

Finalmente, también podríamos usar self (nótese la falta de &) como receptor. Esto requiere que la instancia sea propiedad de la persona que llama y hará que la instancia se mueva al llamar al método. Esto puede ser útil si deseamos consumir, destruir o transformar por completo una instancia existente. Un ejemplo de tal caso de uso es proporcionar métodos de “encadenamiento”:

impl Foo {
    fn double(mut self) -> Self {
        self.my_num *= 2;
        self
    }
}

// ...
foo.my_num = 1;
assert_eq!(foo.double().double().my_num, 4);

Tenga en cuenta que también anteponemos self con mut para que podamos mutar self antes de devolverlo nuevamente. El tipo de retorno del método doble también merece alguna explicación. Self dentro de un bloque impl se refiere al tipo al que se aplica impl (en este caso, Foo). Aquí, es principalmente una abreviatura útil para evitar volver a escribir la firma del tipo, pero en las características, se puede usar para referirse al tipo subyacente que implementa una característica en particular.

Para declarar un método asociado (comúnmente conocido como “método de clase” en otros idiomas) para una estructura simplemente omita el argumento auto. Dichos métodos se llaman en el tipo struct en sí, no en una instancia del mismo:

impl Foo {
    fn new(b: bool, n: isize, s: String) -> Foo {
        Foo { my_bool: b, my_num: n, my_string: s }
    }
}

// ...
// :: is used to access associated members of the type
let x = Foo::new(false, 0, String::from("nil"));
assert_eq!(x.my_num, 0);

Tenga en cuenta que los métodos de estructura solo se pueden definir para los tipos que se declararon en el módulo actual. Además, al igual que con los campos, todos los métodos de estructura son privados de forma predeterminada y, por lo tanto, solo se pueden llamar mediante código en el mismo módulo. Puede prefijar definiciones con la palabra clave pub para que se puedan llamar desde cualquier otro lugar.
