Introducción a Trie #

¿Alguna vez te has preguntado cómo funcionan los motores de búsqueda? ¿Cómo alinea Google millones de resultados frente a usted en solo unos pocos milisegundos? ¿Cómo una enorme base de datos situada a miles de kilómetros de usted encuentra la información que está buscando y se la devuelve? La razón detrás de esto no es posible solo mediante el uso de Internet más rápido y supercomputadoras. Algunos algoritmos de búsqueda fascinantes y estructuras de datos funcionan detrás de él. Uno de ellos es Trie.

Trie, también llamado árbol digital y, a veces, árbol radix o árbol de prefijos (ya que se pueden buscar por prefijos), es un tipo de árbol de búsqueda: una estructura de datos de árbol ordenada que se utiliza para almacenar un conjunto dinámico o matriz asociativa donde las claves suelen ser cadenas. Es una de esas estructuras de datos que se pueden implementar fácilmente. Digamos que tienes una enorme base de datos de millones de palabras. Puede usar trie para almacenar esta información y la complejidad de buscar estas palabras depende solo de la longitud de la palabra que estamos buscando. Eso significa que no depende del tamaño de nuestra base de datos. ¿No es asombroso?

Supongamos que tenemos un diccionario con estas palabras:

algo
algea
also
tom
to

Queremos almacenar este diccionario en la memoria de tal manera que podamos encontrar fácilmente la palabra que estamos buscando. Uno de los métodos consistiría en ordenar las palabras lexicográficamente, como almacenan las palabras los diccionarios de la vida real. Entonces podemos encontrar la palabra haciendo una búsqueda binaria. Otra forma es usar Árbol de prefijos o Trie, en resumen. La palabra ‘Trie’ viene de la palabra Retrieval. Aquí, prefijo denota el prefijo de cadena que se puede definir así: Todas las subcadenas que se pueden crear a partir del principio de una cadena se denominan prefijo. Por ejemplo: ‘c’, ‘ca’, ‘gato’ todos son el prefijo de la cadena ‘gato’.

Ahora volvamos a Trie. Como sugiere el nombre, crearemos un árbol. Al principio, tenemos un árbol vacío con solo la raíz:

Insertaremos la palabra ‘algo’. Crearemos un nuevo nodo y nombraremos el borde entre estos dos nodos ‘a’. Desde el nuevo nodo, crearemos otro borde llamado ’l’ y lo conectaremos con otro nodo. De esta forma, crearemos dos nuevas aristas para ‘g’ y ‘o’. Tenga en cuenta que no estamos almacenando ninguna información en los nodos. Por ahora, solo consideraremos crear nuevos bordes a partir de estos nodos. De ahora en adelante, llamemos a un borde llamado ‘x’ - borde-x

Ahora queremos agregar la palabra ‘algea’. Necesitamos un edge-a de root, que ya tenemos. Por lo tanto, no necesitamos agregar un nuevo borde. De manera similar, tenemos una arista de ‘a’ a ’l’ y de ’l’ a ‘g’. Eso significa que ‘alg’ ya está en el trie. Solo agregaremos edge-e y edge-a con él.

Agregaremos la palabra ’también’. Tenemos el prefijo ‘al’ de raíz. Solo agregaremos ‘so’ con él.

Agreguemos ’tom’. Esta vez, creamos un nuevo borde desde la raíz ya que no tenemos ningún prefijo de tom creado antes.

Ahora, ¿cómo debemos agregar ’to’? ’to’ es completamente un prefijo de ’tom’, por lo que no necesitamos agregar ningún borde. Lo que podemos hacer es poner marcas finales en algunos nodos. Pondremos marcas finales en aquellos nodos donde se complete al menos una palabra. El círculo verde indica la marca final. El trie se verá así:

Puede entender fácilmente por qué agregamos las marcas finales. Podemos determinar las palabras almacenadas en trie. Los caracteres estarán en los bordes y los nodos contendrán las marcas finales.

Ahora podría preguntarse, ¿cuál es el propósito de almacenar palabras como esta? Digamos que se le pide que busque la palabra ‘alicia’ en el diccionario. Atravesarás el trie. Verificará si hay un borde-a desde la raíz. Luego verifique desde ‘a’, si hay un borde-l. Después de eso, no encontrará ningún borde-i, por lo que puede llegar a la conclusión de que la palabra alicia no existe en el diccionario.

Si se le pide que busque la palabra ‘alg’ en el diccionario, recorrerá root->a, a->l, l->g , pero no encontrará un nodo verde al final. Entonces la palabra no existe en el diccionario. Si busca ’tom’, terminará en un nodo verde, lo que significa que la palabra existe en el diccionario.

Complejidad:

La cantidad máxima de pasos necesarios para buscar una palabra en un trie es la longitud de la palabra que estamos buscando. La complejidad es O(longitud). La complejidad para la inserción es la misma. La cantidad de memoria necesaria para implementar trie depende de la implementación. Veremos una implementación en otro ejemplo donde podemos almacenar 106 caracteres (no palabras, letras) en un trie.

Uso de Trie:

  • Para insertar, eliminar y buscar una palabra en el diccionario.
  • Para saber si una cadena es prefijo de otra cadena.
  • Para saber cuántas cadenas tiene un prefijo común.
  • Sugerencia de nombres de contacto en nuestros teléfonos dependiendo del prefijo que ingresemos.
  • Averiguar la ‘Subcadena común más larga’ de dos cadenas.
  • Averiguar la longitud del ‘Prefijo común’ para dos palabras usando el ‘Ancestro común más largo’

Implementación de Trie #

Antes de leer este ejemplo, se recomienda encarecidamente que lea [Introducción a Trie](https://www.wikiod.com/es/data-structures/trie-arbol-de-prefijosarbol-de-radix#Introducción a Trie -intentar) primero.

Una de las formas más fáciles de implementar Trie es usar una lista enlazada.

Nodo:

Los nodos estarán compuestos por:

  1. Variable para Marca Final.
  2. Matriz de punteros al siguiente nodo.

La variable End-Mark simplemente indicará si es un punto final o no. La matriz de punteros denotará todos los posibles bordes que se pueden crear. Para los alfabetos ingleses, el tamaño de la matriz será de 26, ya que se pueden crear un máximo de 26 bordes a partir de un solo nodo. Al principio, cada valor de la matriz de punteros será NULL y la variable de marca final será falsa.

Node:
Boolean Endmark
Node *next[26]
Node()
    endmark = false
    for i from 0 to 25
        next[i] := NULL
    endfor
endNode

Cada elemento en la matriz next[] apunta a otro nodo. next[0] apunta al nodo que comparte edge-a, next1 apunta al nodo que comparte edge-b y así sucesivamente. Hemos definido el constructor de Node para inicializar Endmark como falso y poner NULL en todos los valores de la matriz de punteros next[].

Para crear Trie, primero necesitaremos crear una instancia de root. Luego, desde la raíz, crearemos otros nodos para almacenar información.

Inserción:

Procedure Insert(S, root):    // s is the string that needs to be inserted in our dictionary
curr := root
for i from 1 to S.length
    id := S[i] - 'a'
    if curr -> next[id] == NULL
        curr -> next[id] = Node()
    curr := curr -> next[id]
endfor
curr -> endmark = true

Aquí estamos trabajando con a-z. Entonces, para convertir sus valores ASCII a 0-25, les restamos el valor ASCII de ‘a’. Comprobaremos si el nodo actual (curr) tiene una ventaja para el carácter en cuestión. Si lo hace, nos movemos al siguiente nodo usando ese borde, si no lo hace, creamos un nuevo nodo. Al final, cambiamos la marca final a verdadera.

Buscando:

Procedure Search(S, root)      // S is the string we are searching
curr := root
for i from 1 to S.length
    id := S[i] - 'a'
    if curr -> next[id] == NULL
        Return false
    curr := curr -> id
Return curr -> endmark

El proceso es similar a insertar. Al final, devolvemos la marca final. Entonces, si la marca final es verdadera, eso significa que la palabra existe en el diccionario. Si es falso, entonces la palabra no existe en el diccionario.

Esta fue nuestra implementación principal. Ahora podemos insertar cualquier palabra en trie y buscarla.

Supresión:

En ocasiones necesitaremos borrar las palabras que ya no se utilizarán para guardar memoria. Para este propósito, necesitamos eliminar las palabras no utilizadas:

Procedure Delete(curr):       //curr represents the current node to be deleted
for i from 0 to 25
    if curr -> next[i] is not equal to NULL
        delete(curr -> next[i])
    delete(curr)               //free the memory the pointer is pointing to

Esta función va a todos los nodos secundarios de curr, los elimina y luego curr se elimina para liberar la memoria. Si llamamos a delete(root), eliminará todo el Trie.

Trie también se puede implementar usando Arrays.
