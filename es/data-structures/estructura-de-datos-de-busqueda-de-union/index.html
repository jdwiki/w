Una estructura de datos de búsqueda de unión (o conjunto disjunto) es una estructura de datos simple, una partición de una serie de elementos en conjuntos disjuntos. Cada conjunto tiene un representante que puede usarse para distinguirlo de los otros conjuntos.

Se utiliza en muchos algoritmos, p. para calcular árboles de expansión mínimos a través del algoritmo de Kruskal, para calcular componentes conectados en gráficos no dirigidos y muchos más.

Teoría #

Las estructuras de datos de búsqueda de unión proporcionan las siguientes operaciones:

  • make_sets(n) inicializa una estructura de datos de búsqueda de unión con n singletons
  • find(i) devuelve un representante del conjunto de elementos i
  • union(i,j) fusiona los conjuntos que contienen i y j

Representamos nuestra partición de los elementos 0 a n - 1 almacenando un elemento principal padre[i] para cada elemento i que finalmente conduce a un representante del conjunto que contiene i . Si un elemento en sí mismo es un representante, es su propio padre, es decir, padre[i] == i.

Implementación básica #

La implementación más básica de una estructura de datos de búsqueda de unión consiste en una matriz principal que almacena un elemento principal para cada elemento de la estructura. Seguir estos ‘punteros’ principales para un elemento ‘i’ nos lleva al representante ‘j = find(i)’ del conjunto que contiene ‘i’, donde se cumple ‘parent[j] = j’.

using std::size_t;

class union_find {
private:
    std::vector<size_t> parent;  // Parent for every node

public:
    union_find(size_t n) : parent(n) {
        for (size_t i = 0; i < n; ++i)
            parent[i] = i;      // Every element is its own representative
    }

    size_t find(size_t i) const {
        if (parent[i] == i)     // If we already have a representative
            return i;           // return it
        return find(parent[i]); // otherwise return the parent's representative
    }

    void merge(size_t i, size_t j) {
        size_t pi = find(i);
        size_t pj = find(j);
        if (pi != pj) {        // If the elements are not in the same set: 
            parent[pi] = pj;   // Join the sets by marking pj as pi's parent
        }
    }
};

Mejoras: Compresión de ruta #

Si hacemos muchas operaciones de “combinación” en una estructura de datos de búsqueda de unión, las rutas representadas por los punteros “principales” pueden ser bastante largas. La compresión de ruta, como ya se describió en la parte teórica, es una forma sencilla de mitigar este problema.

Podríamos intentar hacer una compresión de ruta en toda la estructura de datos después de cada k-ésima operación de fusión o algo similar, pero tal operación podría tener un tiempo de ejecución bastante grande.

Por lo tanto, la compresión de caminos se usa principalmente en una pequeña parte de la estructura, especialmente en el camino que recorremos para encontrar el representante de un conjunto. Esto se puede hacer almacenando el resultado de la operación buscar después de cada subllamada recursiva:

size_t find(size_t i) const {
    if (parent[i] == i)          // If we already have a representative
        return i;                // return it
    parent[i] = find(parent[i]); // path-compress on the way to the representative
    return parent[i];            // and return it
}

Mejoras: Unión por tamaño #

En nuestra implementación actual de merge, siempre elegimos el conjunto de la izquierda para que sea el hijo del conjunto de la derecha, sin tener en cuenta el tamaño de los conjuntos. Sin esta restricción, las rutas (sin compresión de ruta) desde un elemento hasta su representante podrían ser bastante largas, lo que generaría tiempos de ejecución prolongados en las llamadas find.

Por lo tanto, otra mejora común es la heurística unión por tamaño, que hace exactamente lo que dice: al fusionar dos conjuntos, siempre configuramos el conjunto más grande para que sea el padre del conjunto más pequeño, lo que conduce a una longitud de ruta de como máximo pasos:

Almacenamos un miembro adicional std::vector<size_t> size en nuestra clase que se inicializa en 1 para cada elemento. Al fusionar dos conjuntos, el conjunto más grande se convierte en el padre del conjunto más pequeño y sumamos los dos tamaños:

private:
    ...
    std::vector<size_t> size;

public:
    union_find(size_t n) : parent(n), size(n, 1) { ... }

    ...

    void merge(size_t i, size_t j) {
        size_t pi = find(i);
        size_t pj = find(j);
        if (pi == pj) {            // If the elements are in the same set: 
            return;                // do nothing
        }
        if (size[pi] > size[pj]) { // Swap representatives such that pj
            std::swap(pi, pj);     // represents the larger set
        }
        parent[pi] = pj;           // attach the smaller set to the larger one
        size[pj] += size[pi];      // update the size of the larger set
    }

Mejoras: Unión por rango #

Otra heurística comúnmente utilizada en lugar de la unión por tamaño es la heurística unión por rango

Su idea básica es que en realidad no necesitamos almacenar el tamaño exacto de los conjuntos, una aproximación del tamaño (en este caso: más o menos el logaritmo del tamaño del conjunto) es suficiente para lograr la misma velocidad que la unión por tamaño.

Para ello, introducimos la noción de rango de un conjunto, que se da de la siguiente manera:

  • Singletons tienen rango 0
  • Si se fusionan dos conjuntos con un rango diferente, el conjunto con un rango más alto se convierte en el principal mientras que el rango no cambia.
  • Si se fusionan dos conjuntos de igual rango, uno de ellos se convierte en el padre del otro (la elección puede ser arbitraria), su rango se incrementa.

Una ventaja de unión por rango es el uso del espacio: como el rango máximo aumenta aproximadamente como , para todos los tamaños de entrada realistas, el rango se puede almacenar en un solo byte ( ya que n < 2^255).

Una implementación simple de unión por rango podría verse así:

private:
    ...
    std::vector<unsigned char> rank;

public:
    union_find(size_t n) : parent(n), rank(n, 0) { ... }

    ...

    void merge(size_t i, size_t j) {
        size_t pi = find(i);
        size_t pj = find(j);
        if (pi == pj) {
            return;
        }
        if (rank[pi] < rank[pj]) {
            // link the smaller group to the larger one
            parent[pi] = pj;
        } else if (rank[pi] > rank[pj]) {
            // link the smaller group to the larger one
            parent[pj] = pi;
        } else {
            // equal rank: link arbitrarily and increase rank
            parent[pj] = pi;
            ++rank[pi];
        }
    }

Mejora final: unión por rango con almacenamiento fuera de límites #

Si bien en combinación con la compresión de ruta, la unión por rango casi logra operaciones de tiempo constante en estructuras de datos de búsqueda de unión, hay un truco final que nos permite deshacernos del almacenamiento de “rango” al almacenar el rango fuera de los límites. entradas de la matriz padre. Se basa en las siguientes observaciones:

  • En realidad, solo necesitamos almacenar el rango para representantes, no para otros elementos. Para estos representantes, no necesitamos almacenar un padre.
  • Hasta ahora, parent[i] es como máximo size - 1, es decir, los valores más grandes no se utilizan.
  • Todos los rangos son como máximo .

Esto nos lleva al siguiente enfoque:

  • En lugar de la condición padre[i] == i, ahora identificamos representantes por padre[i] >= tamaño
  • Usamos estos valores fuera de los límites para almacenar los rangos del conjunto, es decir, el conjunto con el representante i tiene el rango parent[i] - size
  • Por lo tanto, inicializamos la matriz principal con parent[i] = size en lugar de parent[i] = i, es decir, cada conjunto es su propio representante con rango 0.

Dado que solo compensamos los valores de rango por tamaño, podemos simplemente reemplazar el vector rango por el vector padre en la implementación de combinar y solo necesitamos intercambiar la condición que identifica a los representantes en buscar:

Implementación finalizada usando unión por rango y compresión de ruta:

using std::size_t;

class union_find {
private:
    std::vector<size_t> parent;

public:
    union_find(size_t n) : parent(n, n) {} // initialize with parent[i] = n

    size_t find(size_t i) const {
        if (parent[i] >= parent.size()) // If we already have a representative
            return i;                   // return it
        return find(parent[i]);         // otherwise return the parent's repr.
    }

    void merge(size_t i, size_t j) {
        size_t pi = find(i);
        size_t pj = find(j);
        if (pi == pj) {
            return;
        }
        if (parent[pi] < parent[pj]) {
            // link the smaller group to the larger one
            parent[pi] = pj;
        } else if (parent[pi] > parent[pj]) {
            // link the smaller group to the larger one
            parent[pj] = pi;
        } else {
            // equal rank: link arbitrarily and increase rank
            parent[pj] = pi;
            ++parent[pi];
        }
    }
};
