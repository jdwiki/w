Introducción a la pila #

La pila es una estructura de datos LIFO (último en entrar, primero en salir), es decir, el elemento más reciente (o “último en entrar”) agregado a la pila será el primer elemento eliminado (“primero en salir”).

Consideremos el ejemplo de los libros en una caja. Solo se puede agregar o quitar un libro de la caja a la vez, y solo se puede agregar y quitar de la parte superior.

Ahora, la caja con los dos primeros libros se ve así:

 |--------|
| libro 2 | ◀──── parte superior de la caja
  |--------|
| libro 1 | ◀──── parte inferior de la caja
|--------|

Si añadimos el libro 3, estará en la parte superior. El resto de los libros, que se añadieron antes del libro 3, quedarán debajo de él:

 |--------|
| libro 3 | ◀──── parte superior de la caja
  |--------|
| libro 2 |
  |--------|
| libro 1 | ◀──── parte inferior de la caja
|--------|

La caja es como una pila y los libros son elementos de datos. Agregar un libro a la caja es la operación de inserción, mientras que quitar/recibir un libro de la parte superior de la caja es la operación emergente. Si realizamos la operación pop, obtendremos el libro 3, y la caja volverá a la forma en que estaba antes de empujar el libro 3. Esto significa que el último (o más reciente) elemento que pusimos fue el primer elemento que salió (LIFO). Para obtener el libro 1, tenemos que realizar dos pops más: uno para eliminar el libro 2 y el segundo para obtener el libro 1.

Implementación de la pila usando una matriz. Para esto, necesitamos un índice que apunte a la ubicación superior (tos). Cada vez que se inserta un elemento en la pila, el tos aumenta en uno y cada vez que se realiza una operación emergente, el índice que apunta a la parte superior de la pila (tos) disminuye en uno.

EMPUJAR:

Antes de la operación PUSH tos |
i1 i2 i3 i4
—– —– —– —– —–
void push(dataElment item){
stack[top]=item; //item = i4
top++; 
}

Después de la operación PUSH La pila tiene un puntero a la parte superior de la pila. Cada vez que se llama a la operación de inserción, coloca el valor en la parte superior y lo actualiza. tos – Top of the stack tos |
i1 i2 i3
—– —– —– —–
ESTALLIDO :
La operación pop elimina el contenido de la parte superior de la pila y actualiza el valor de tos disminuyendo en 1

Antes de la operación pop: tos |
i1 i2 i3 i4
—– —– —– —– —– 
dataElment pop(){
    dataElment value = stack[tos--];
    return value;
}

Después de la operación pop: tos |
i1 i2 i3 i4
—– —– —– —–
When a push operation is performed it overwrites i4.

Usar pilas para encontrar palíndromos #

Un palíndromo es una palabra que se puede leer en ambos sentidos, como ‘kayak’.

Este ejemplo mostrará cómo encontrar si una palabra dada es un palíndromo usando Python3.

Primero, necesitamos convertir una cadena en una pila (usaremos arreglos como pilas).

str = "string"
stk = [c for c in str] #this makes an array: ["s", "t", "r", "i", "n", "g"]
stk.append("s") #adds a letter to the array
stk.pop() #pops the last element

Ahora, tenemos que invertir la palabra.

def invert(str):
    stk = [c for c in str]
    stk2 = []
    while len(stk) > 0:
        stk2.append(stk.pop())
    #now, let's turn stk2 into a string
    str2 = ""
    for c in stk2:
        str2 += c
    return str2

Ahora podemos comparar la palabra y la forma invertida.

def palindrome(str):
    return str == invert(str)

Implementación de pila mediante el uso de matriz y lista enlazada #

Implementación de matrices

#include<stdio.h>

#define MAX 100000                    //Global variables
int top = -1;
int a[MAX];

void push(int x){
    
    if(top == MAX-1){                         // Check whether stack is full
        printf("Stack Overflow\n");
        return;
    }
    
    a[++top] = x;                            // Otherwise increment top and insert x
}

void pop(){
    if(top == -1){                                     // if top = -1, empty stack
        printf("Empty stack\n");
        return;
    }
    top--;        
}

void print(){                         //printing stack values
    
    for(int i=0;i <= top;i++){
        printf("%d ",a[i]);        
    }        
    printf("\n");        
}

void topValue(){                        // Method can print the top value of a stack
    printf("%d",a[top]);
}
int main(){
        
    push(5);
    push(20);
    push(15);
    print();
    pop();
    print();
    push(35);
    print();
    topValue();    
    
}

Implementación de listas enlazadas

#include<stdio.h>
#include<malloc.h>

 struct node{
    int data;
    node* link;
};
node* top = NULL;

void push(int data){
    
    node* temp = (struct node*)malloc(sizeof(struct node*));
    temp->data = data;
    temp->link = top;
    top = temp;
}

void pop(){
    
    if(top == NULL) return;
    node* temp = top;
    top = top->link;
    free(temp);
}
void print(){
    
    node* t = top;
    while(t != NULL){
        printf("%d ",t->data);
        t=t->link;
    }
    printf("\n");
}

void topValue(){
    printf("%d ",top->data);
}
int main(){
        
    push(5);
    push(20);
    push(15);
    print();
    pop();
    print();
    push(35);
    print();            
    topValue();    
}

Comprobación de paréntesis equilibrados #

Se considera que un corchete es cualquiera de los siguientes caracteres: (, ), {, }, [ o ].

Se considera que dos corchetes son un par coincidente si un corchete de apertura (es decir, (, [ o {) aparece a la izquierda de un corchete de cierre (es decir, ), ] o }) del mismo tipo exacto. Hay tres tipos de pares de corchetes coincidentes: [], {} y ().

Un par de corchetes coincidentes no está equilibrado si el conjunto de corchetes que encierra no coinciden. Por ejemplo, {[(])} no está equilibrado porque los contenidos entre { y } no están equilibrados. El par de corchetes encierra un solo corchete de apertura no balanceado, (, y el par de paréntesis encierra un solo corchete de cierre no balanceado, ].

Por esta lógica, decimos que una secuencia de paréntesis se considera balanceada si se cumplen las siguientes condiciones:

No contiene paréntesis que no coincidan.

El subconjunto de paréntesis encerrado dentro de los límites de un par coincidente de corchetes es también un par de corchetes emparejados.

Algoritmo:

  1. Declarar una pila (decir pila).
  2. Ahora recorra la entrada de cadena.
  • a) Si el carácter actual es un corchete de inicio (es decir, ( o { o [ ) entonces empújelo a la pila.
  • b) Si el carácter actual es un corchete de cierre (es decir, ) o } o ]) luego pop de la pila. Si el carácter reventado es el correspondiente paréntesis inicial, entonces bien, de lo contrario, los paréntesis no están “equilibrados”.
  • c) Si el carácter actual es un corchete de cierre (es decir, ) o } o ]) y la pila está vacía, entonces los paréntesis no están equilibrados.
  1. Después de completar el recorrido, si queda algún paréntesis inicial en la pila, entonces la cadena “no está balanceada”. De lo contrario, tenemos una cadena “balanceada”.
