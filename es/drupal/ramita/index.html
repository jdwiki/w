Ramita

En esta página

Twig es el motor de plantillas que forma parte de Drupal 8. En Drupal 8, los archivos Twig tienen la extensión .html.twig y se utilizan en todos los aspectos de la temática de Drupal. Las entidades, los campos y las vistas se pueden representar utilizando archivos .html.twig.

En este tema, el objetivo es tener un libro de cocina sobre cómo trabajar con Twig en el contexto de Drupal. Si desea obtener más información sobre la sintaxis o las funciones disponibles consulte la documentación.

Filtro de ramita #

A diferencia de Drupal 7, no puede llamar a las funciones regulares de PHP en sus plantillas. En Drupal 8 el camino a seguir es creando filtros y funciones.

Debe utilizar un filtro cuando: desea transformar los datos que desea mostrar. Imagina que tienes un título que quieres que esté siempre en mayúsculas. Por ejemplo, twig tiene el filtro mayúsculas por defecto que te permite transformar cualquier texto en su equivalente en mayúsculas.

Para este ejemplo, crearemos un filtro que nos permitirá mezclar una cadena. La forma de crear filtros y funciones es exactamente la misma que la de Twig normal.

La principal diferencia entre Twig regular y Drupal 8 Twig es que en Drupal 8 debe crear una definición de servicio de la clase que está creando y la clase también debe pertenecer a un espacio de nombres; de lo contrario, no se registrará como un filtro Twig dentro del entorno Drupal. .

Este ejemplo asume que tienes un módulo llamado twig_shuffle_extension.

Esta será la definición básica del servicio en twig_shuffle_extension.services.yml

services:
  twig_shuffle_extension.twig_extension:
    class: Drupal\twig_shuffle_extension\TwigExtension\TwigShuffleExtension
    tags:
      - { name: twig.extension }

La clave tags también es absolutamente necesaria y es lo que le dice a Drupal lo que se supone que debe hacer esta clase (es decir, registrarla como una extensión de Twig).

Y ahora el código fuente que debe colocarse en la ruta definida en la clave clase de la definición del servicio.

// Don't forget the namespace!
namespace Drupal\twig_shuffle_extension\TwigExtension;

use Twig_Extension;
use Twig_SimpleFilter;

class TwigShuffleExtension extends Twig_Extension  {
  /**
   * This is the same name we used on the services.yml file
   */
  public function getName() {
    return 'twig_shuffle_extension.twig_extension';
  }

  // Basic definition of the filter. You can have multiple filters of course.
  // Just make sure to create a more generic class name ;)
  public function getFilters() {
    return [
      new Twig_SimpleFilter('shuffle', [$this, 'shuffleFilter']),
    ];
  }

  // The actual implementation of the filter.
  public function shuffleFilter($context) {
    if(is_string($context)) {
      $context = str_shuffle($context);
    }
    return $context;
  }
}

Limpia tus cachés y ahora, si todo va según lo planeado, puedes usar el filtro en tus plantillas.

{{ "shuffle me!" | shuffle }}

Inyección de dependencia en extensiones Twig #

Este ejemplo le mostrará cómo usar Dependency Inject para usar otros servicios registrados en el entorno Drupal.

Imagina que tienes un archivo de imagen SVG que cambia de color dependiendo de algún CSS/Javascript aleatorio en tu proyecto. Para poder apuntar al SVG con CSS, debe tener el archivo SVG en el DOM. Así que crea un archivo SVG base sin ningún color y lo coloca en su carpeta de temas.

Por supuesto, podría simplemente pegar el contenido del archivo en la plantilla Twig, pero eso no sería bueno. Puede crear una extensión Twig, pero tampoco desea codificar la ruta de su tema en el código fuente de la extensión.

Esto significa que tenemos que obtener la ruta de forma dinámica. Tienes dos opciones:

  1. Utilice el equivalente a una variable global llamando a \Drupal::theme()->getActiveTheme()->getPath();
  2. Inyecte ThemeManager (dado por \Drupal::theme()) en su clase de extensión

En este ejemplo, tomaremos el segundo ejemplo porque puede ser ampliamente aplicable a cualquier servicio (importa la Solicitud o la Conexión de la base de datos si lo desea).

Esto supone que tienes un módulo llamado twig_svg_extension y un archivo twig_svg_extension.services.yml:

services:
  twig_svg_extension.twig_extension:
    class: Drupal\twig_svg_extension\TwigExtension\TwigSvgExtension
    arguments: ['@theme.manager']
    tags:
      - { name: twig.extension }

Por favor, no la clave argumentos que le dice a Drupal que el servicio debe inyectar.

namespace Drupal\twig_svg_Extension\TwigExtension;

use Drupal\Core\Theme\ThemeManager;
use Twig_Extension;
use Twig_SimpleFilter;

class TwigSvgExtension extends Twig_Extension  {
  private $theme;
  
  // Dependency injection at work!
  public function __construct(ThemeManager $theme) {
    $this->theme = $theme;
  }

  public function getFilters() {
    return [
      'svg' =>new Twig_SimpleFilter('svg', [$this, 'svgFilter']),
    ];
  }

  public function getName() {
    return 'twig_svg_extension.twig_extension';
  }

  public function svgFilter(string $filepath) {
    $realpath = realpath($this->theme->getActiveTheme()->getPath().DIRECTORY_SEPARATOR.$filepath);
    $pathinfo = pathinfo($realpath);

    if($realpath !== false && strtolower($pathinfo['extension']) === 'svg') {
      return file_get_contents($realpath);
    }

    return '"'.$filepath.'" does not exist or is not an SVG';
  }
}

Tenga en cuenta el constructor que contiene la dependencia que inyectamos en la configuración del servicio, así como el svgFilter que obtiene la ruta del tema activo actual.

$filepath debe ser una ruta relativa a su carpeta de temas. La extensión convertirá la ruta del archivo en el contenido del archivo al que apunta.
