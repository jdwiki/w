Sintaxis #

  • foreach (f, xs)
  • mapa (f, xs)
  • filtro (f, xs)
  • reducir(f, v0, xs)
  • foldl(f, v0, xs)
  • plegar(f, v0, xs)

Las funciones se pueden aceptar como parámetros y también se pueden producir como tipos de devolución. De hecho, las funciones se pueden crear dentro del cuerpo de otras funciones. Estas funciones internas se conocen como cierres.

Funciones como argumentos #

Funciones son ​​objetos en Julia. Como cualquier otro objeto, se pueden pasar como argumentos a otras funciones. Las funciones que aceptan funciones se conocen como funciones [de orden superior] (https://en.wikipedia.org/wiki/Higher-order_function).

Por ejemplo, podemos implementar un equivalente de la función foreach de la biblioteca estándar tomando una función f como primer parámetro.

function myforeach(f, xs)
    for x in xs
        f(x)
    end
end

Podemos probar que esta función realmente funciona como esperamos:

julia> myforeach(println, ["a", "b", "c"])
a
b
c

Al tomar una función como el primer parámetro, en lugar de un parámetro posterior, podemos usar la sintaxis del bloque do de Julia. La sintaxis del bloque do es solo una forma conveniente de pasar una función anónima como el primer argumento de una función.

julia> myforeach([1, 2, 3]) do x
           println(x^x)
       end
1
4
27

Nuestra implementación de myforeach anterior es más o menos equivalente a la función foreach integrada. También existen muchas otras funciones integradas de orden superior.

Las funciones de orden superior son bastante poderosas. A veces, cuando se trabaja con funciones de orden superior, las operaciones exactas que se realizan pierden importancia y los programas pueden volverse bastante abstractos. [Combinadores][3] son ​​ejemplos de sistemas de funciones de orden superior muy abstractas.

2: https://www.wikiod.com/es/julia-lang/funciones#Funciones anónimas [3]: https://www.wikiod.com/es/julia-lang/combinadores

Mapear, filtrar y reducir #

Dos de las funciones de orden superior más fundamentales incluidas en la biblioteca estándar son mapa y filtro. Estas funciones son genéricas y pueden operar en cualquier iterable. En particular, son adecuados para cálculos en matrices.

Supongamos que tenemos un conjunto de datos de escuelas. Cada escuela enseña una materia en particular, tiene un número de clases y un número promedio de estudiantes por clase. Podemos modelar una escuela con el siguiente [tipo inmutable][3]:

immutable School
    subject::Symbol
    nclasses::Int
    nstudents::Int  # average no. of students per class
end

Nuestro conjunto de datos de escuelas será un Vector{Escuela}:

dataset = [School(:math, 3, 30), School(:math, 5, 20), School(:science, 10, 5)]

Supongamos que deseamos encontrar el número total de estudiantes matriculados en un programa de matemáticas. Para ello, requerimos varios pasos:

  • debemos reducir el conjunto de datos a solo las escuelas que enseñan matemáticas (filtro)
  • debemos calcular el número de estudiantes en cada escuela (mapa)
  • y debemos reducir esa lista de números de alumnos a un solo valor, la suma (reduce)

Una solución ingenua (no la más eficaz) sería simplemente usar esas tres funciones de orden superior directamente.

function nmath(data)
    maths = filter(x -> x.subject === :math, data)
    students = map(x -> x.nclasses * x.nstudents, maths)
    reduce(+, 0, students)
end

y verificamos que hay 190 estudiantes de matemáticas en nuestro conjunto de datos:

julia> nmath(dataset)
190

Existen funciones para combinar estas funciones y así mejorar el rendimiento. Por ejemplo, podríamos haber usado la función mapreduce para realizar el mapeo y la reducción en un solo paso, lo que ahorraría tiempo y memoria.

La reducción solo es significativa para [operaciones asociativas] (https://en.wikipedia.org/wiki/Associativity) como +, pero ocasionalmente es útil para realizar una reducción con una operación no asociativa. Las funciones de orden superior foldl y foldr se proporcionan para forzar un orden de reducción particular.
