Sintaxis #

  • si está condicionado; cuerpo; final
  • si está condicionado; cuerpo; más; cuerpo; final
  • si está condicionado; cuerpo; elseif condicional; cuerpo; más; final
  • si está condicionado; cuerpo; elseif condicional; cuerpo; final
  • ¿Condición? si es cierto: si es falso
  • cond && si es cierto
  • condición || si es falso
  • ifelse(cond, iftrue, iffalse)

Todos los operadores y funciones condicionales implican el uso de condiciones booleanas (verdadero o falso). En Julia, el tipo de booleanos es Bool. A diferencia de otros lenguajes, otros tipos de números (como 1 o 0), cadenas, arreglos, etc. no se pueden usar directamente en condicionales.

Por lo general, uno usa funciones de predicado (funciones que devuelven un Bool) o operadores de comparación en la condición de un operador o función condicional.

Operadores de cortocircuito: && y || #

Para ramificar #

Los operadores condicionales de cortocircuito && y || se pueden usar como reemplazos ligeros para las siguientes construcciones:

  • x && y es equivalente a x? y : x
  • x || y es equivalente a x? x: y

Un uso para los operadores de cortocircuito es como una forma más concisa de probar una condición y realizar una determinada acción dependiendo de esa condición. Por ejemplo, el siguiente código usa el operador && para arrojar un error si el argumento x es negativo:

function mysqrt(x)
    x < 0 && throw(DomainError("x is negative"))
    x ^ 0.5
end

El operador || también se puede utilizar para la verificación de errores, excepto que desencadena el error a menos que se cumpla una condición, en lugar de si se cumple la condición:

function halve(x::Integer)
    iseven(x) || throw(DomainError("cannot halve an odd number"))
    x ÷ 2
end

Otra aplicación útil de esto es proporcionar un valor predeterminado a un objeto, solo si no está definido previamente:

isdefined(:x) || (x = NEW_VALUE)

Aquí, esto verifica si el símbolo x está definido (es decir, si hay un valor asignado al objeto x). Si es así, entonces no pasa nada. Pero, si no, a x se le asignará NUEVO_VALOR. Tenga en cuenta que este ejemplo solo funcionará en el ámbito de nivel superior.

En condiciones #

Los operadores también son útiles porque pueden usarse para probar dos condiciones, la segunda de las cuales solo se evalúa según el resultado de la primera condición. De la [documentación] de Julia (http://docs.julialang.org/en/release-0.4/manual/control-flow/#man-short-circuit-e Evaluation):

En la expresión a && b, la subexpresión b solo se evalúa si a se evalúa como verdadero

En la expresión a || b, la subexpresión b solo se evalúa si a se evalúa como falso

Así, mientras tanto a & b como a && b producirán verdadero si tanto a como b son verdaderos, su comportamiento si a es falso es diferente.

Por ejemplo, supongamos que deseamos verificar si un objeto es un número positivo, donde es posible que ni siquiera sea un número. Considere las diferencias entre estos dos intentos de implementación:

CheckPositive1(x) = (typeof(x)<:Number) & (x > 0) ? true : false
CheckPositive2(x) = (typeof(x)<:Number) && (x > 0) ? true : false

CheckPositive1("a")
CheckPositive2("a")

CheckPositive1() generará un error si se le proporciona un tipo no numérico como argumento. Esto se debe a que evalúa ambas expresiones, independientemente del resultado de la primera, y la segunda expresión generará un error cuando se intente evaluarla para un tipo no numérico.

CheckPositive2(), sin embargo, arrojará falso (en lugar de un error) si se le proporciona un tipo no numérico, ya que la segunda expresión solo se evalúa si la primera es verdadera.

Se puede conectar más de un operador de cortocircuito. P.ej.:

1 > 0 && 2 > 0 && 3 > 5

La función si no #

shift(x) = ifelse(x > 10, x + 1, x - 1)

Uso:

julia> shift(10)
9

julia> shift(11)
12

julia> shift(-1)
-2

La función ifelse evaluará ambas ramas, incluso la que no está seleccionada. Esto puede ser útil cuando las ramas tienen efectos secundarios que deben evaluarse o porque puede ser más rápido si ambas ramas son baratas.

Expresión if…else #

El condicional más común en Julia es la expresión if…else. Por ejemplo, a continuación implementamos el algoritmo euclidiano para calcular el máximo común divisor usando un condicional para manejar el caso base:

mygcd(a, b) = if a == 0
    abs(b)
else
    mygcd(b % a, a)
end

La forma if…else en Julia es en realidad una expresión, y tiene un valor; el valor es la expresión en posición de cola (es decir, la última expresión) en la rama que se toma. Considere la siguiente entrada de muestra:

julia> mygcd(0, -10)
10

Aquí, a es 0 y b es -10. La condición a == 0 es verdadera, por lo que se toma la primera rama. El valor devuelto es abs(b) que es 10.

julia> mygcd(2, 3)
1

Aquí, a es 2 y b es 3. La condición a == 0 es falsa, por lo que se toma la segunda rama y calculamos mygcd(b % a, a), que es mygcd(3 % 2, 2). El operador % devuelve el resto cuando 3 se divide por 2, en este caso 1. Así calculamos mygcd(1, 2), y esta vez a es 1 y b es 2. Una vez más, a == 0 es falso, por lo que se toma la segunda rama y calculamos mygcd(b % a, a), que es mygcd(0, 1). Esta vez, a == 0 por fin y así se devuelve abs(b), lo que da el resultado 1.

sentencia if…else #

name = readline()
if startswith(name, "A")
    println("Your name begins with A.")
else
    println("Your name does not begin with A.")
end

Cualquier expresión, como la expresión if…else, se puede poner en posición de declaración. Esto ignora su valor pero aún ejecuta la expresión por sus efectos secundarios.

declaración if #

Como cualquier otra expresión, el valor de retorno de una expresión if…else puede ignorarse (y por lo tanto descartarse). Por lo general, esto solo es útil cuando el cuerpo de la expresión tiene efectos secundarios, como escribir en un archivo, mutar variables o imprimir en la pantalla.

Además, la rama else de una expresión if…else es opcional. Por ejemplo, podemos escribir el siguiente código para mostrar en la pantalla solo si se cumple una condición particular:

second = Dates.second(now())
if iseven(second)
    println("The current second, $second, is even.")
end

En el ejemplo anterior, usamos las funciones hora y fecha para obtener el segundo actual; por ejemplo, si actualmente son las 10:55:27, la variable ‘segundo’ contendrá ‘27’. Si este número es par, se imprimirá una línea en la pantalla. De lo contrario, no se hará nada.

Operador condicional ternario #

pushunique!(A, x) = x in A ? A : push!(A, x)

El operador condicional ternario es una expresión if…else menos prolija.

La sintaxis en concreto es:

[condition] ? [execute if true] : [execute if false]

En este ejemplo, agregamos x a la colección A solo si x no está ya en A. De lo contrario, simplemente dejamos A sin cambios.

declaración if con múltiples ramas #

d = Dates.dayofweek(now())
if d == 7
    println("It is Sunday!")
elseif d == 6
    println("It is Saturday!")
elseif d == 5
    println("Almost the weekend!")
else
    println("Not the weekend yet...")
end

Se puede usar cualquier número de ramas elseif con una sentencia if, posiblemente con o sin una rama else final. Las condiciones subsiguientes solo se evaluarán si se ha determinado que todas las condiciones anteriores son “falsas”.
