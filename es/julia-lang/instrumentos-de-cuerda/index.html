Sintaxis #

  • “[cuerda]”

Parámetros #

Parámetro Detalles
Para sprint(f, xs...)
f Una función que toma un objeto IO como su primer argumento.
xs Cero o más argumentos restantes para pasar a f.

¡Hola Mundo! #

Las cadenas en Julia se delimitan con el símbolo ":

julia> mystring = "Hello, World!"
"Hello, World!"

Tenga en cuenta que, a diferencia de otros idiomas, el símbolo ' no se puede usar en su lugar. ' define un caracter literal; este es un tipo de datos Char y solo almacenará un solo [valor escalar Unicode] (http://www.unicode.org/glossary/#unicode_scalar_value):

julia> 'c'
'c'

julia> 'character'
ERROR: syntax: invalid character literal

Uno puede extraer los valores escalares Unicode de una cadena iterando sobre ella con un bucle for:

julia> for c in "Hello, World!"
           println(c)
       end
H
e
l
l
o
,
 
W
o
r
l
d
!

Grafemas #

El tipo Char de Julia representa un valor escalar Unicode, que solo en algunos casos corresponde a lo que los humanos perciben como un “carácter”. Por ejemplo, una representación del carácter é, como en currículum, es en realidad una combinación de dos valores escalares Unicode:

julia> collect("é")
2-element Array{Char,1}:
 'e'
 '́'

Las descripciones Unicode para estos puntos de código son “LETRA E MINÚSCULA LATINA” y “ACENTO AGUDO COMBINADO”. Juntos, definen un único carácter “humano”, que en términos Unicode se denomina [grafema] (http://www.unicode.org/glossary/#grapheme). Más específicamente, el Anexo #29 de Unicode motiva la definición de un [grupo de grafemas] (http://www.unicode.org/glossary/#grapheme_cluster) porque:

Es importante reconocer que lo que el usuario considera un “carácter”, una unidad básica de un sistema de escritura para un idioma, puede no ser solo un único punto de código Unicode. En cambio, esa unidad básica puede estar compuesta por múltiples puntos de código Unicode. Para evitar la ambigüedad con el uso informático del término carácter, se denomina carácter percibido por el usuario. Por ejemplo, “G” + acento agudo es un carácter percibido por el usuario: los usuarios piensan que es un solo carácter, pero en realidad está representado por dos puntos de código Unicode. Estos caracteres percibidos por el usuario se aproximan mediante lo que se denomina un grupo de grafemas, que se puede determinar mediante programación.

Julia proporciona la función graphemas para iterar sobre los grupos de grafemas en una cadena:

julia> for c in graphemes("résumé")
           println(c)
       end
r
é
s
u
m
é

Observe cómo el resultado, al imprimir cada carácter en su propia línea, es mejor que si hubiéramos iterado sobre los valores escalares de Unicode:

julia> for c in "résumé"
           println(c)
       end
r
e

s
u
m
e

Por lo general, cuando se trabaja con caracteres en un sentido percibido por el usuario, es más útil tratar con grupos de grafemas que con valores escalares Unicode. Por ejemplo, supongamos que queremos escribir una función para calcular la longitud de una sola palabra. Una solución ingenua sería usar

julia> wordlength(word) = length(word)
wordlength (generic function with 1 method)

Observamos que el resultado es contrario a la intuición cuando la palabra incluye grupos de grafemas que constan de más de un punto de código:

julia> wordlength("résumé")
8

Cuando usamos la definición más correcta, usando la función grafemas, obtenemos el resultado esperado:

julia> wordlength(word) = length(graphemes(word))
wordlength (generic function with 1 method)

julia> wordlength("résumé")
6

Convertir tipos numéricos en cadenas #

Existen numerosas formas de convertir tipos numéricos en cadenas en Julia:

julia> a = 123
123

julia> string(a)
"123"

julia> println(a)
123

La función string() también puede tomar más argumentos:

julia> string(a, "b")
"123b"

También puede insertar (también conocido como interpolar) números enteros (y algunos otros tipos) en cadenas usando $:

julia> MyString = "my integer is $a"
"my integer is 123"

Consejo de rendimiento: Los métodos anteriores pueden ser bastante convenientes a veces. Pero, si va a realizar muchas, muchas operaciones de este tipo y le preocupa la velocidad de ejecución de su código, la [guía de rendimiento] de Julia (http://docs.julialang.org/en/release-0.4/manual/performance- tips/#avoid-string-interpolation-for-i-o) recomienda en contra de esto, y en su lugar a favor de los siguientes métodos:

Puede proporcionar múltiples argumentos a print() y println() que operarán sobre ellos exactamente como string() opera sobre múltiples argumentos:

julia> println(a, "b")
123b

O, al escribir en un archivo, puede usar de manera similar, p.

open("/path/to/MyFile.txt", "w") do file
    println(file, a, "b", 13)
end

o

file = open("/path/to/MyFile.txt", "a")
println(file, a, "b", 13)
close(file)

Estos son más rápidos porque evitan la necesidad de formar primero una cadena a partir de piezas dadas y luego enviarlas (ya sea a la pantalla de la consola o a un archivo) y, en su lugar, solo generan secuencialmente las diversas piezas.

Créditos: respuesta basada en la pregunta SO [¿Cuál es la mejor manera de convertir un Int en una cadena en Julia?] (http://stackoverflow.com/questions/38676573/whats-the-best-way-to-convert-an- int-to-a-string-in-julia) con respuesta de Michael Ohlrogge y aportes de Fengyang Wang

Interpolación de cadenas (insertar valor definido por variable en la cadena) #

En Julia, como en muchos otros lenguajes, es posible interpolar insertando valores definidos por variables en cadenas. Para un ejemplo simple:

n = 2
julia> MyString = "there are $n ducks"
"there are 2 ducks"

Podemos usar otros tipos además de numéricos, p.

Result = false
julia> println("test results is $Result")
test results is false

Puede tener múltiples interpolaciones dentro de una cadena determinada:

MySubStr = "a32"
MyNum = 123.31
println("$MySubStr  ,   $MyNum")

Consejo de rendimiento La interpolación es muy conveniente. Pero, si lo vas a estar haciendo muchas veces muy rápido, no es lo más eficiente. En su lugar, consulte [Convertir tipos numéricos en cadenas](https://www.wikiod.com/es/julia-lang/instrumentos-de-cuerda#Convertir tipos numéricos en cadenas) para obtener sugerencias cuando el rendimiento es bajo. un problema.

Uso de sprint para crear cadenas con funciones IO #

Las cadenas se pueden crear a partir de funciones que funcionan con objetos IO utilizando la función sprint. Por ejemplo, la función code_llvm acepta un objeto IO como primer argumento. Por lo general, se usa como

julia> code_llvm(STDOUT, *, (Int, Int))

define i64 @"jlsys_*_46115"(i64, i64) #0 {
top:
  %2 = mul i64 %1, %0
  ret i64 %2
}

Supongamos que queremos esa salida como una cadena en su lugar. Entonces podemos simplemente hacer

julia> sprint(code_llvm, *, (Int, Int))
"\ndefine i64 @\"jlsys_*_46115\"(i64, i64) #0 {\ntop:\n  %2 = mul i64 %1, %0\n  ret i64 %2\n}\n"

julia> println(ans)

define i64 @"jlsys_*_46115"(i64, i64) #0 {
top:
  %2 = mul i64 %1, %0
  ret i64 %2
}

Convertir los resultados de funciones “interactivas” como code_llvm en cadenas puede ser útil para el análisis automatizado, como probar si el código generado puede haber retrocedido.

La función sprint es una función de orden superior que toma la función que opera en objetos IO como su primer argumento. Detrás de escena, crea un ‘IOBuffer’ en la RAM, llama a la función dada y toma los datos del búfer en un objeto ‘String’.
