Sintaxis #

  • inicio (itr)
  • siguiente (itr, s)
  • hecho (itr, s)
  • tomar (itr, n)
  • gota (itr, n)
  • ciclo (itr)
  • Base.producto(xs, ys)

Parámetros #

Parámetro Detalles
Para Todas las funciones
itr El iterable para operar.
Para siguiente y hecho
s Un estado de iterador que describe la posición actual de la iteración.
Para tomar y soltar
n El número de elementos a tomar o soltar.
Para Producto.Base
xs El iterable del que se toman los primeros elementos de los pares.
sí El iterable para tomar segundos elementos de pares.
… (Tenga en cuenta que producto acepta cualquier número de argumentos; si se proporcionan más de dos, construirá tuplas de longitud superior a dos.)

Nuevo tipo iterable #

En Julia, cuando se recorre un objeto iterable I se hace con la sintaxis for:

for i = I   # or  "for i in I"
    # body
end

Detrás de escena, esto se traduce a:

state = start(I)
while !done(I, state)
    (i, state) = next(I, state)
    # body
end

Por lo tanto, si desea que I sea iterable, debe definir los métodos start, next y done para su tipo. Suponga que define un tipo Foo que contiene una matriz como uno de los campos:

type Foo
    bar::Array{Int,1}
end

Instanciamos un objeto Foo haciendo:

julia> I = Foo([1,2,3])
Foo([1,2,3])

julia> I.bar
3-element Array{Int64,1}:
 1
 2
 3

Si queremos iterar a través de Foo, con cada elemento bar devuelto por cada iteración, definimos los métodos:

import Base: start, next, done

start(I::Foo) = 1

next(I::Foo, state) = (I.bar[state], state+1)

function done(I::Foo, state)
    if state == length(I.bar)
        return true
    end
    return false
end

Tenga en cuenta que dado que estas funciones pertenecen al módulo Base, primero debemos importar sus nombres antes de agregarles nuevos métodos.

Después de definir los métodos, Foo es compatible con la interfaz del iterador:

julia> for i in I
           println(i)
       end

1
2
3

Combinando iterables perezosos #

La biblioteca estándar viene con una rica colección de iterables perezosos (y bibliotecas como Iterators.jl proporcionan aún más). Los iterables perezosos se pueden componer para crear iterables más potentes en tiempo constante. Los iterables perezosos más importantes son take and drop, a partir de los cuales se pueden crear muchas otras funciones.

Cortar perezosamente un iterable #

Las matrices se pueden dividir con notación de división. Por ejemplo, lo siguiente devuelve los elementos 10 a 15 de una matriz, inclusive:

A[10:15]

Sin embargo, la notación de corte no funciona con todos los iterables. Por ejemplo, no podemos dividir una expresión generadora:

julia> (i^2 for i in 1:10)[3:5]
ERROR: MethodError: no method matching getindex(::Base.Generator{UnitRange{Int64},##1#2}, ::UnitRange{Int64})

Cortar cadenas puede no tener el comportamiento Unicode esperado:

julia> "αααα"[2:3]
ERROR: UnicodeError: invalid character index
 in getindex(::String, ::UnitRange{Int64}) at ./strings/string.jl:130

julia> "αααα"[3:4]
"α"

Podemos definir una función lazysub(itr, range::UnitRange) para hacer este tipo de corte en iterables arbitrarios. Esto se define en términos de tomar y soltar:

lazysub(itr, r::UnitRange) = take(drop(itr, first(r) - 1), last(r) - first(r) + 1)

La implementación aquí funciona porque para el valor UnitRange a:b, se realizan los siguientes pasos:

  • suelta los primeros elementos a-1
  • toma el elemento ath, a+1th elemento, y así sucesivamente, hasta que a+(b-a)=bth elemento

En total, se toman elementos b-a. Podemos confirmar que nuestra implementación es correcta en cada caso anterior:

julia> collect(lazysub("αααα", 2:3))
2-element Array{Char,1}:
 'α'
 'α'

julia> collect(lazysub((i^2 for i in 1:10), 3:5))
3-element Array{Int64,1}:
  9
 16
 25

Cambia perezosamente un iterable circularmente #

La operación circshift en arreglos cambiará el arreglo como si fuera un círculo, luego lo relineará. Por ejemplo,

julia> circshift(1:10, 3)
10-element Array{Int64,1}:
  8
  9
 10
  1
  2
  3
  4
  5
  6
  7

¿Podemos hacer esto perezosamente para todos los iterables? Podemos usar los iterables cycle, drop y take para implementar esta funcionalidad.

lazycircshift(itr, n) = take(drop(cycle(itr), length(itr) - n), length(itr))

Además de que los tipos perezosos tienen un mejor rendimiento en muchas situaciones, esto nos permite hacer una funcionalidad similar a la de ‘circshift’ en tipos que de otro modo no serían compatibles:

julia> circshift("Hello, World!", 3)
ERROR: MethodError: no method matching circshift(::String, ::Int64)
Closest candidates are:
  circshift(::AbstractArray{T,N}, ::Real) at abstractarraymath.jl:162
  circshift(::AbstractArray{T,N}, ::Any) at abstractarraymath.jl:195

julia> String(collect(lazycircshift("Hello, World!", 3)))
"ld!Hello, Wor"

Hacer una tabla de multiplicar #

Hagamos una [tabla de multiplicar] (https://en.wikipedia.org/wiki/Multiplication_table) usando funciones iterables diferidas para crear una matriz.

Las funciones clave a utilizar aquí son:

  • Base.product, que calcula un producto cartesiano.
  • prod, que calcula un producto regular (como en la multiplicación)
  • :, que crea un rango
  • map, que es una función de orden superior que aplica una función a cada elemento de una colección

La solucion es:

julia> map(prod, Base.product(1:10, 1:10))
10×10 Array{Int64,2}:

  5  10  15  20  25  30  35  40  45   50
  6  12  18  24  30  36  42  48  54   60
  7  14  21  28  35  42  49  56  63   70
  8  16  24  32  40  48  56  64  72   80

Listas evaluadas perezosamente #

Es posible hacer una lista simple evaluada perezosamente usando tipos mutables y cierres. Una lista evaluada perezosamente es una lista cuyos elementos no se evalúan cuando se construye, sino cuando se accede a ella. Los beneficios de las listas evaluadas perezosamente incluyen la posibilidad de ser infinitas.

import Base: getindex
type Lazy
    thunk
    value
    Lazy(thunk) = new(thunk)
end

evaluate!(lazy::Lazy) = (lazy.value = lazy.thunk(); lazy.value)
getindex(lazy::Lazy) = isdefined(lazy, :value) ? lazy.value : evaluate!(lazy)

import Base: first, tail, start, next, done, iteratorsize, HasLength, SizeUnknown
abstract List
immutable Cons <: List
    head
    tail::Lazy
end
immutable Nil <: List end

macro cons(x, y)
    quote
        Cons($(esc(x)), Lazy(() -> $(esc(y))))
    end
end

first(xs::Cons) = xs.head
tail(xs::Cons) = xs.tail[]
start(xs::Cons) = xs
next(::Cons, xs) = first(xs), tail(xs)
done(::List, ::Cons) = false
done(::List, ::Nil) = true
iteratorsize(::Nil) = HasLength()
iteratorsize(::Cons) = SizeUnknown()

Lo que de hecho funciona como lo haría en un lenguaje como Haskell, donde todas las listas se evalúan perezosamente:

julia> xs = @cons(1, ys)
Cons(1,Lazy(false,#3,#undef))

julia> ys = @cons(2, xs)
Cons(2,Lazy(false,#5,#undef))

julia> [take(xs, 5)...]
5-element Array{Int64,1}:
 1
 2
 1
 2
 1

En la práctica, es mejor usar el paquete Lazy.jl. Sin embargo, la implementación de la lista perezosa anterior arroja luces sobre detalles importantes sobre cómo construir el propio tipo iterable.
