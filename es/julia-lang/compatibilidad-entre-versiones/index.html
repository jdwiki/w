Sintaxis #

  • usando Compat
  • Compat.Cadena
  • Compat.UTF8String
  • @compat f.(x, y)

A veces es muy difícil lograr que la nueva sintaxis funcione bien con varias versiones. Como Julia aún se encuentra en desarrollo activo, a menudo es útil simplemente dejar de admitir versiones anteriores y, en su lugar, centrarse solo en las más nuevas.

Números de versión #

Julia tiene una implementación integrada de [versiones semánticas] (http://semver.org/) expuesta a través del tipo VersionNumber.

Para construir un Número de versión como un literal, se puede usar @v_str macro de cadena:

julia> vers = v"1.2.0"
v"1.2.0"

Alternativamente, uno puede llamar al constructor VersionNumber; tenga en cuenta que el constructor acepta hasta cinco argumentos, pero todos excepto el primero son opcionales.

julia> vers2 = VersionNumber(1, 1)
v"1.1.0"

Los números de versión se pueden comparar usando operadores de comparación y, por lo tanto, se pueden ordenar:

julia> vers2 < vers
true

julia> v"1" < v"0"
false

julia> sort([v"1.0.0", v"1.0.0-dev.100", v"1.0.1"])
3-element Array{VersionNumber,1}:
 v"1.0.0-dev.100"
 v"1.0.0"
 v"1.0.1"

Los números de versión se utilizan en varios lugares de Julia. Por ejemplo, la constante VERSION es un VersionNumber:

julia> VERSION
v"0.5.0"

Esto se usa comúnmente para la evaluación de código condicional, según la versión de Julia. Por ejemplo, para ejecutar un código diferente en v0.4 y v0.5, uno puede hacer

if VERSION < v"0.5"
    println("v0.5 prerelease, v0.4 or older")
else
    println("v0.5 or newer")
end

Cada paquete instalado también está asociado con un número de versión actual:

julia> Pkg.installed("StatsBase")
v"0.9.0"

Uso de Compat.jl #

El paquete Compat.jl permite usar algunas funciones y sintaxis nuevas de Julia con versiones anteriores de Julia. Sus características están documentadas en su LÉAME, pero a continuación se proporciona un resumen de las aplicaciones útiles.

Tipo de cadena unificada #

En Julia v0.4, había muchos tipos diferentes de cadenas. Este sistema se consideró demasiado complejo y confuso, por lo que en Julia v0.5 solo queda el tipo String. Compat permite usar el tipo y constructor String en la versión 0.4, bajo el nombre Compat.String. Por ejemplo, este código v0.5

buf = IOBuffer()
println(buf, "Hello World!")
String(buf)  # "Hello World!\n"

se puede traducir directamente a este código, que funciona tanto en v0.5 como en v0.4:

using Compat
buf = IOBuffer()
println(buf, "Hello World!")
Compat.String(buf)  # "Hello World!\n"

Tenga en cuenta que hay algunas advertencias.

  • En v0.4, Compat.String tiene un alias de tipo ByteString, que es Union{ASCIIString, UTF8String}. Por lo tanto, los tipos con campos String no serán de tipo estable. En estas situaciones, se recomienda Compat.UTF8String, ya que significará String en v0.5 y UTF8String en v0.4, ambos tipos concretos.

  • Hay que tener cuidado al usar Compat.String o import Compat: String, porque String tiene un significado en v0.4: es un alias en desuso para AbstractString. Una señal de que String se usó accidentalmente en lugar de Compat.String es si en algún momento aparecen las siguientes advertencias:

    WARNING: Base.String is deprecated, use AbstractString instead. likely near no file:0 WARNING: Base.String is deprecated, use AbstractString instead. likely near no file:0

Sintaxis de transmisión compacta #

Julia v0.5 introduce el azúcar sintáctico para broadcast. la sintaxis

f.(x, y)

se reduce a emisión (f, x, y). Los ejemplos del uso de esta sintaxis incluyen sin.([1, 2, 3]) para tomar el seno de varios números a la vez.

En v0.5, la sintaxis se puede usar directamente:

julia> sin.([1.0, 2.0, 3.0])
3-element Array{Float64,1}:
 0.841471
 0.909297
 0.14112

Sin embargo, si intentamos lo mismo en v0.4, obtenemos un error:

julia> sin.([1.0, 2.0, 3.0])
ERROR: TypeError: getfield: expected Symbol, got Array{Float64,1}

Afortunadamente, Compat hace que esta nueva sintaxis también se pueda usar desde v0.4. Una vez más, agregamos usando Compat. Esta vez, rodeamos la expresión con la macro @compat:

julia> using Compat

julia> @compat sin.([1.0, 2.0, 3.0])
3-element Array{Float64,1}:
 0.841471
 0.909297
 0.14112
