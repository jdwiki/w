Planificación previa antes de crear su bosque o dominio.

  • Consideraciones

En esta página

  • Consideraciones

Una cosa que podría considerar con cualquier dominio en su bosque es cuántas máquinas físicas y virtuales desea tener. Personalmente creo que debería haber una máquina física por dominio. Una de las razones por las que creo esto es por cómo se manejan los relojes, específicamente en mi caso, las máquinas Hyper-V. No puedo hablar con VMWare. En un host Hyper-V, instala un servicio de sincronización de reloj en los sistemas operativos invitados. En un dominio, todas las máquinas miembro se sincronizan con el servicio de tiempo de los DC. Cada dominio se sincroniza con el bosque. Pero en un invitado de Hyper-V, el reloj está sincronizado con el reloj de la máquina física. Y si el host es una máquina miembro, el reloj se sincroniza con el dominio. Esto crea un ciclo de retroalimentación que permite que el reloj se desvíe. Después de un par de meses, el tiempo se desvía hasta el punto de que hay una diferencia notable en el tiempo y en Active Directory que es un problema importante. Para resolver esto, configuré mis hosts de Hyper-V para sincronizar el tiempo en un intervalo muy bajo desde un controlador de dominio físico que tiene la función de controlador de dominio primario (PDC) de operación de maestro único flexible (FSMO) en el dominio raíz del bosque.

Consideraciones #

Debe decidir cómo se verá la estructura de su bosque antes de instalar Active Directory por primera vez.

¿Va a tener un solo dominio en su bosque o va a tener múltiples dominios en su bosque?

Active Directory puede admitir varios sitios. Por lo general, los sitios se dividen en enlaces lentos (conexiones de red lentas) o grandes distancias entre áreas geográficas, pero no tiene por qué ser así. Los sitios se configuran a través de subredes de direcciones IP. Usted especifica que estas direcciones IP pertenecen al sitio A y las otras pertenecen al sitio B. La replicación entre sitios (entre el sitio A y el sitio B) generalmente ocurre con una frecuencia mayor que la replicación dentro del sitio. Por ejemplo, la replicación entre el sitio A y el sitio B podría establecerse en los 180 minutos predeterminados (3 horas) y la replicación entre los controladores de dominio dentro del mismo sitio está establecida de forma predeterminada en 15 segundos.

También debe considerar la cantidad de “máquinas” que necesitará para construir su bosque de Active Directory. La mejor práctica sería tener un mínimo de dos controladores de dominio (DC) por cada dominio en su bosque para redundancia. DNS es un requisito para Active Directory. Los clientes de Active Directory, como las computadoras de los usuarios, usan DNS para encontrarse y ubicar los servicios anunciados en Active Directory por los controladores de dominio de Active Directory. Debe decidir si DNS se integrará con Active Directory o no. Es más fácil poner en marcha Active Directory rápidamente si decide integrar DNS a AD. También vale la pena señalar que debe hacer una copia de seguridad de su Active Directory porque tener datos incorrectos replicados en otro DC proporciona redundancia de datos incorrectos y no una ruta a la que volver.
