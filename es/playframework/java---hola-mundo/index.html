Java - Hola mundo

  • Crea tu primer proyecto

  • La primera carrera

En esta página

  • Crea tu primer proyecto
  • Obtener activador
  • La primera carrera
  • El “Hola Mundo” en el Hola Mundo
  • Este tutorial está diseñado para ejecutar Play en un sistema Linux/MacOS

Crea tu primer proyecto #

Para crear un nuevo proyecto, use el siguiente comando (HelloWorld es el nombre del proyecto y play-java es la plantilla)

 $ ~/activator-1.3.10-minimal/bin/activator new HelloWorld play-java

Debería obtener una salida similar a esta

Fetching the latest list of templates...

OK, application "HelloWorld" is being created using the "play-java" template.

To run "HelloWorld" from the command line, "cd HelloWorld" then:
/home/YourUserName/HelloWorld/activator run

To run the test for "HelloWorld" from the command line, "cd HelloWorld" then:
/home/YourUserName/HelloWorld/activator test

To run the Activator UI for "HelloWorld" from the command line, "cd HelloWorld" then:
/home/YourUserName/HelloWorld/activator ui

El proyecto se creará en el directorio actual (en este caso era mi carpeta de inicio)

Ahora estamos listos para comenzar nuestra aplicación.

Obtener activador #

El primer paso en su viaje por el mundo de Play Framework es descargar Activator. Activator es una herramienta que se utiliza para crear, compilar y distribuir aplicaciones de Play Framework.

El activador se puede descargar desde la [sección de descargas de Play] (https://www.playframework.com/download) (aquí usaré la versión 1.3.10)

Después de descargar el archivo, extraiga el contenido a algún directorio al que tenga acceso de escritura y estamos listos para comenzar.

En este tutorial, asumiré que Activator se extrajo a su carpeta de inicio

La primera carrera #

Cuando creamos nuestro proyecto, Activator nos dijo cómo podemos ejecutar nuestra aplicación

To run "HelloWorld" from the command line, "cd HelloWorld" then:
    /home/YourUserName/HelloWorld/activator run

Aquí hay un pequeño escollo: el ejecutable activator no está en la raíz de nuestro proyecto, sino en bin/activator. Además, si cambió su directorio actual a su directorio de proyecto, simplemente puede ejecutar

bin/activator

El activador ahora descargará las dependencias requeridas para compilar y ejecutar su proyecto. Dependiendo de la velocidad de su conexión, esto puede llevar algún tiempo. Con suerte, se le presentará un aviso

[HelloWorld] $

Ahora podemos ejecutar nuestro proyecto usando ~run: esto le indicará a Activator que ejecute nuestro proyecto y observe los cambios. Si algo cambia, volverá a compilar las partes necesarias y reiniciará nuestra aplicación. Puede detener este proceso presionando Ctrl+D (regresa al shell del activador) o Ctrl+D (va al shell de su sistema operativo)

[HelloWorld] $ ~run

Play ahora descargará más dependencias. Una vez realizado este proceso, su aplicación debería estar lista para usar:

-- (Running the application, auto-reloading is enabled) ---

[info] p.c.s.NettyServer - Listening for HTTP on /0:0:0:0:0:0:0:0:9000

(Server started, use Ctrl+D to stop and go back to the console...)

Cuando navegue a localhost:9000 en su navegador, debería ver la página de inicio del marco de juego.

¡Felicitaciones, ahora está listo para hacer algunos cambios en su aplicación!

El “Hola Mundo” en el Hola Mundo #

Un “Hola Mundo” no merece este nombre si no proporciona un mensaje de Hola Mundo. Así que hagamos uno.

En el archivo app/controllers/HomeController.java agregue el siguiente método:

public Result hello() {
    return ok("Hello world!");
}

Y en su archivo conf/routes agregue lo siguiente al final del archivo:

GET     /hello                      controllers.HomeController.hello

Si echa un vistazo a su terminal, debería notar que Play está compilando su aplicación mientras realiza los cambios y recarga la aplicación:

[info] Compiling 4 Scala sources and 1 Java source to /home/YourUserName/HelloWorld/target/scala-2.11/classes...
[success] Compiled in 4s

Navegando a localhost:9000/hello, finalmente recibimos nuestro mensaje de hola mundo
