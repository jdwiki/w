IDEA IntelliJ #

requisitos previos #

  1. Intellij IDEA instalado (edición Community o Ultimate)
  2. Complemento Scala instalado en IntelliJ
  3. Un proyecto estándar de Play, creado por ejemplo con Activator (activator new [nombre del proyecto] play-scala).

Abriendo el proyecto #

  1. Abra IntelliJ IDEA
  2. Vaya al menú Archivo > Abrir... > haga clic en toda la carpeta [nombre del proyecto] > Aceptar
  3. Se abre una ventana emergente con algunas opciones. Los valores predeterminados son lo suficientemente buenos en la mayoría de los casos, y si no le gustan, puede cambiarlos en otro lugar más adelante. Haga clic en Aceptar
  4. Intellij IDEA pensará un poco, luego propondrá otra ventana emergente para seleccionar qué módulos seleccionar en el proyecto. Debería haber dos módulos root y root-build seleccionados por defecto. No cambie nada y haga clic en Aceptar.
  5. IntelliJ abrirá el proyecto. Puede comenzar a ver los archivos mientras IntelliJ sigue pensando un poco como debería ver en la barra de estado en la parte inferior, luego finalmente debería estar completamente listo.

Ejecutar las aplicaciones desde Intellij #

A partir de ahí, algunas personas usan el IDE solo para ver/editar el proyecto, mientras usan la línea de comando sbt para compilar/ejecutar/lanzar pruebas. Otros prefieren lanzarlos desde Intellij. Es necesario si desea utilizar el modo de depuración. Pasos :

  1. Menú Ejecutar > Editar configuraciones...
  2. En la ventana emergente, haga clic en + en la parte superior izquierda> Elija Play 2 App en la lista
  3. Asigne un nombre a la configuración, por ejemplo [nombredesuproyecto]. Deje las opciones predeterminadas y presione OK.
  4. Desde el menú Ejecutar, o los botones en la interfaz de usuario, ahora puede Ejecutar o Depurar usando esta configuración. Run simplemente iniciará la aplicación, como si hubieras hecho sbt run desde la línea de comandos. Debug hará lo mismo pero le permitirá colocar puntos de interrupción en el código para interrumpir la ejecución y analizar lo que está sucediendo.

Opción de importación automática #

Esta es una opción global para el proyecto, que está disponible en el momento de la creación y luego se puede cambiar en el menú Intellij IDEA > Preferencias > Construir, Ejecutar, Implementar > Herramientas de compilación > SBT > Configuración a nivel de proyecto > Usar importación automática.

Esta opción no tiene nada que ver con las sentencias importar en el código de Scala. Indica lo que debe hacer Intellij IDEA cuando edita el archivo build.sbt. Si la importación automática está activada, Intellij IDEA analizará el nuevo archivo de compilación inmediatamente y actualizará la configuración del proyecto automáticamente. Se vuelve molesto rápidamente ya que esta operación es costosa y tiende a ralentizar Intellij cuando todavía está trabajando en el archivo de compilación. Cuando la importación automática está desactivada, debe indicar manualmente a Intellij que editó build.sbt y desea que se actualice la configuración del proyecto. En la mayoría de los casos, aparecerá una ventana emergente temporal para preguntarle si desea hacerlo. De lo contrario, vaya al panel SBT en la interfaz de usuario y haga clic en el signo de las flechas circulares azules para forzar la actualización.

Eclipse como Play IDE - Java, Play 2.4, 2.5 #

Introducción #

Play tiene varios complementos para diferentes IDE-s. El complemento eclipse permite transformar una aplicación Play en un proyecto de eclipse funcional con el comando activator eclipse. El complemento de Eclipse se puede configurar por proyecto o globalmente por usuario sbt. Depende del trabajo en equipo, qué enfoque se debe utilizar. Si todo el equipo usa Eclipse IDE, el complemento se puede configurar a nivel de proyecto. Debe descargar la versión de eclipse compatible con Scala y Java 8: luna o mars - desde http://scala-ide.org/download/sdk.html.

Configuración de Eclipse IDE por proyecto #

Para importar la aplicación Play a Eclipse:

  1. Agregue el complemento de eclipse en project/plugins.sbt:

    //Support Play in Eclipse addSbtPlugin(“com.typesafe.sbteclipse” % “sbteclipse-plugin” % “4.0.0”)

  2. Agregue a build.sbt un indicador que fuerce la compilación cuando se ejecuta el comando eclipse:

    EclipseKeys.preTasks := Seq(compile in Compile)

  3. Asegúrese de que la ruta del repositorio del usuario en el archivo {raíz del usuario}.sbt\repositories tenga el formato adecuado. Los valores adecuados para las propiedades activator-launcher-local y activator-local deben tener al menos tres barras como en el ejemplo:

    activator-local: file:////${activator.local.repository-C:/Play-2.5.3/activator-dist-1.3.10//repository}, [organization]/[module]/(scala_[scalaVersion]/)(sbt_[sbtVersion]/)[revision]/[type]s/artifact.[ext] activator-launcher-local: file:////${activator.local.repository-${activator.home-${user.home}/.activator}/repository}, [organization]/[module]/(scala_[scalaVersion]/)(sbt_[sbtVersion]/)[revision]/[type]s/artifact.[ext]

  4. Compile la aplicación:

    activator compile

  5. Prepare un proyecto de eclipse para la nueva aplicación con:

    activator eclipse Ahora el proyecto está listo para importarse a Eclipse a través de Proyectos existentes en Workspace.

Cómo adjuntar la fuente Play a Eclipse #

  1. Agregue a build.sbt:

    EclipseKeys.withSource := true

  2. Compilar el proyecto

Configuración de Eclipse IDE globalmente #

Agregue la configuración de usuario sbt:

  1. Cree en el directorio raíz del usuario una carpeta .sbt\0.13\plugins y un archivo plugins.sbt. Por ejemplo, para el usuario de Windows asch:

    c:\asch.sbt\0.13\plugins\plugins.sbt

  2. Agregue el complemento de eclipse en plugins.sbt:

    //Support Play in Eclipse addSbtPlugin(“com.typesafe.sbteclipse” % “sbteclipse-plugin” % “4.0.0”)

  3. Cree en el directorio del usuario .sbt un archivo sbteclipse.sbt. Por ejemplo, para el usuario de Windows asch:

    c:\asch.sbt\0.13\sbteclipse.sbt

  4. Coloque en sbteclipse.sbt un indicador que fuerce la compilación cuando se ejecuta el comando activator eclipse:

    import com.typesafe.sbteclipse.plugin.EclipsePlugin.EclipseKeys EclipseKeys.preTasks := Seq(compile in Compile)

  5. Agregue opcionalmente otras configuraciones de EclipseKeys.

Depuración desde eclipse #

Para depurar, inicie la aplicación con el puerto predeterminado 9999:

activator -jvm-debug run

o con el puerto diferente:

activator -jvm-debug [port] run

En eclipse:

  1. Haga clic derecho en el proyecto y seleccione Depurar como, Configuraciones de depuración.
  2. En el cuadro de diálogo Configuraciones de depuración, haga clic con el botón derecho en Aplicación Java remota y seleccione Nueva.
  3. Cambie Puerto a relevante (9999 si se usó el puerto de depuración predeterminado) y haga clic en Aplicar.

A partir de ahora, puede hacer clic en Depurar para conectarse a la aplicación en ejecución. Detener la sesión de depuración no detendrá el servidor.

IDE de Eclipse #

requisitos previos #

  1. Java8 (1.8.0_91)
  2. Eclipse neon (JavaScript y desarrollador web)
  3. Marco de juego 2.5.4

Instalación de Scala en Eclipse #

  1. Inicie el Eclipse
  2. Abra Ayuda > Eclipse Marketplace
  3. Escriba Scala en Buscar
  4. Instalar Scala IDE

Configuración de sbteclipse #

  1. Abra el proyecto de reproducción .\project\plugins.sbt
  2. Agregue el siguiente comando en plugins.sbt para convertir el proyecto eclipse

addSbtPlugin(“com.typesafe.sbteclipse” % “sbteclipse-plugin” % “4.0.0”)

  1. Abra el comando y vaya al proyecto de reproducción, p. cd C:\reproducir\reproducir-scala. Escriba lo siguiente en la línea de comando

eclipse activador

Proyecto de importación #

  1. Vaya al menú Archivo > Importar en Eclipse
  2. Seleccione Proyectos existentes en el espacio de trabajo
  3. Seleccione el directorio raíz

Ahora su proyecto está listo para ver y editar en Eclipse IDE.
