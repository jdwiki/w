Tabla de versiones #

Cada etiqueta tiene un tema general. Este tema tiene una sección especial llamada “Versiones”. En esa sección, se pueden definir diferentes versiones en una tabla que tiene que

  • tener al menos 2 columnas:

    1. the first one should be the name of the version

    2. the last one should be the release date of the version, by which versions will be ordered.

      It should be in the yyyy-MM-dd format and a valid date.

Puede haber columnas adicionales entre los dos para incluir más información. Tiene sentido enumerar las versiones en orden cronológico.

Este marcado:

| Version | Additional Information | Release date |
| --- | ----| ----|
| 1.R | version names do not have to be numbers | 1980-11-25 |
| 1 | we have to start somewhere | 1980-11-24 |

produce este resultado:

Versión Información adicional Fecha de lanzamiento
1.R los nombres de versión no tienen que ser números 1980-11-25
1 tenemos que empezar en alguna parte 1980-11-24

Tablas de versiones múltiples #

Para algunas etiquetas, tiene sentido tener varias tablas de versiones. Puede haber diferentes subconjuntos de un lenguaje de programación o marco disponible, por ejemplo, para diferentes tipos de dispositivos como en este ejemplo.

Las diferentes tablas deben ir precedidas de un encabezado.

Este marcado:

## desktop development kit ##

| Version | Release date |
| --- | ----|
| 1.0 | 1980-12-24 |
| 1 RC | 1980-11-25 |
| 1 beta | 1980-11-24 |

## mobile development kit ##

| Version  | Release date |
| --- | ----|
| M 2 | 1980-12-24 |
| M 1.2 | 1980-11-25 |
| M 1 | 1980-11-24 |

produce este resultado:

kit de desarrollo de escritorio #

Versión Fecha de lanzamiento
1.0 1980-12-24
1 mando a distancia 1980-11-25
1 beta 1980-11-24

kit de desarrollo móvil #

Versión Fecha de lanzamiento
M 2 1980-12-24
M 1.2 1980-11-25
M 1 1980-11-24

Versiones de un tema #

Es posible que un tema solo se aplique a ciertas versiones. Al crear un tema nuevo, se pueden elegir las versiones apropiadas en la sección de título del tema.

En el siguiente ejemplo, .NET 2.0, 4.0, 4.5.1; Se seleccionan Compact Framework 3.7 y Micro Framework 4.2.

Esta selección aparecerá junto al título del tema en la lista de todos los temas.

Versiones de un ejemplo (tabla de versión única) #

Un ejemplo no se puede asociar con un número de versión directamente.

Sin embargo, se puede declarar que partes de él se aplican solo a ciertas versiones.

<!– si la versión [eq Java SE 1.3] –>

Este contenido es para Java SE 1.3

<!– versión final si –>

Versiones de un ejemplo (varias tablas de versiones) #

Si hay muchas tablas de versiones, el título de la deseada debe incluirse entre paréntesis angulares.

<!– si versión <Micro Framework> [eq 4.4] –>

Este contenido es para Micro Framework 4.4

<!– versión final si –>

Condicionales disponibles #

La ayuda de rebajas indica

Los condicionales disponibles son gt, gte, lt, lte, eq y neq.

<!– si versión [eq C++03] –>

eq igual a

<!– versión final si –>

<!– si la versión [neq C++03] –>

neq no es igual a

<!– versión final si –>

<!– si versión [gt C++03] –>

gt mayor que

<!– versión final si –>

<!– si versión [gte C++03] –>

gte mayor que igual

<!– versión final si –>

<!– si la versión [lt C++03] –>

lt menor que

<!– versión final si –>

<!– si la versión [lte C++03] –>

lte menor que igual

<!– versión final si –>

Condicionales Múltiples #

Se pueden utilizar varias condiciones.

<!– si versión [lte 1.2.0] [gt 1.3.12] [gt 1.4.9] [neq 1.2.0] [eq 1.5.7] –>

contenido

<!– versión final si –>
