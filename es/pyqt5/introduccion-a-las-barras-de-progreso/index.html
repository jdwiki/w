Introducción a las barras de progreso

  • Barra de progreso básica de PyQt

En esta página

  • Barra de progreso básica de PyQt

Las barras de progreso son una parte integral de la experiencia del usuario y ayudan a los usuarios a tener una idea del tiempo restante para un proceso determinado que se ejecuta en la GUI. Este tema repasará los aspectos básicos de la implementación de una barra de progreso en su propia aplicación.

Este tema tratará ligeramente sobre QThread y el nuevo mecanismo de señales/ranuras. También se espera de los lectores algún conocimiento básico de los widgets de PyQt5.

Al agregar ejemplos, solo use PyQt5 y Python incorporados para demostrar la funcionalidad.

Solo PyQt5

Experimentar con estos ejemplos es la mejor manera de empezar a aprender.

Barra de progreso básica de PyQt #

Esta es una barra de progreso muy básica que solo usa lo que se necesita como mínimo.

Sería prudente leer todo este ejemplo hasta el final.

import sys
import time

from PyQt5.QtWidgets import (QApplication, QDialog,
                             QProgressBar, QPushButton)

TIME_LIMIT = 100

class Actions(QDialog):
    """
    Simple dialog that consists of a Progress Bar and a Button.
    Clicking on the button results in the start of a timer and
    updates the progress bar.
    """
    def __init__(self):
        super().__init__()
        self.initUI()
        
    def initUI(self):
        self.setWindowTitle('Progress Bar')
        self.progress = QProgressBar(self)
        self.progress.setGeometry(0, 0, 300, 25)
        self.progress.setMaximum(100)
        self.button = QPushButton('Start', self)
        self.button.move(0, 30)
        self.show()

        self.button.clicked.connect(self.onButtonClick)

    def onButtonClick(self):
        count = 0
        while count < TIME_LIMIT:
            count += 1
            time.sleep(1)
            self.progress.setValue(count)

if __name__ == "__main__":
    app = QApplication(sys.argv)
    window = Actions()
    sys.exit(app.exec_())

La barra de progreso primero se importa así desde PyQt5.QtWidgets import QProgressBar

Luego se inicializa como cualquier otro widget en QtWidgets

El método de línea self.progress.setGeometry(0, 0, 300, 25) define las posiciones x,y en el cuadro de diálogo y el ancho y alto de la barra de progreso.

Luego movemos el botón usando .move() por 30px hacia abajo para que haya un espacio de 5px entre los dos widgets.

Aquí self.progress.setValue(count) se usa para actualizar el progreso. Establecer un valor máximo usando .setMaximum() también calculará automáticamente los valores por usted. Por ejemplo, si el valor máximo se establece en 50, dado que TIME_LIMIT es 100, saltará de 0 a 2 a 4 por ciento en lugar de 0 a 1 a 2 cada segundo. También puede establecer un valor mínimo usando .setMinimum() forzando a la barra de progreso a comenzar desde un valor dado.

Ejecutar este programa producirá una GUI similar a esta.

Como puede ver, la GUI definitivamente se congelará y no responderá hasta que el contador cumpla con la condición TIME_LIMIT. Esto se debe a que time.sleep hace que el sistema operativo crea que el programa se ha atascado en un bucle infinito.

QHilo

Entonces, ¿cómo superamos este problema? Podemos usar la clase de subprocesamiento que proporciona PyQt5.

import sys
import time

from PyQt5.QtCore import QThread, pyqtSignal
from PyQt5.QtWidgets import (QApplication, QDialog,
                             QProgressBar, QPushButton)

TIME_LIMIT = 100

class External(QThread):
    """
    Runs a counter thread.
    """
    countChanged = pyqtSignal(int)

    def run(self):
        count = 0
        while count < TIME_LIMIT:
            count +=1
            time.sleep(1)
            self.countChanged.emit(count)

class Actions(QDialog):
    """
    Simple dialog that consists of a Progress Bar and a Button.
    Clicking on the button results in the start of a timer and
    updates the progress bar.
    """
    def __init__(self):
        super().__init__()
        self.initUI()
        
    def initUI(self):
        self.setWindowTitle('Progress Bar')
        self.progress = QProgressBar(self)
        self.progress.setGeometry(0, 0, 300, 25)
        self.progress.setMaximum(100)
        self.button = QPushButton('Start', self)
        self.button.move(0, 30)
        self.show()

        self.button.clicked.connect(self.onButtonClick)

    def onButtonClick(self):
        self.calc = External()
        self.calc.countChanged.connect(self.onCountChanged)
        self.calc.start()

    def onCountChanged(self, value):
        self.progress.setValue(value)

if __name__ == "__main__":
    app = QApplication(sys.argv)
    window = Actions()
    sys.exit(app.exec_())

Analicemos estas modificaciones.

from PyQt5.QtCore import QThread, pyqtSignal

Esta línea importa Qthread, que es una implementación de PyQt5 para dividir y ejecutar algunas partes (p. ej., funciones, clases) de un programa en segundo plano (también conocido como subprocesos múltiples). Estas partes también se llaman hilos. Todos los programas PyQt5 por defecto tienen un subproceso principal y los otros (subprocesos de trabajo) se utilizan para descargar tareas intensivas que consumen más tiempo y procesar en segundo plano mientras se mantiene el funcionamiento del programa principal.

La segunda importación pyqtSignal se usa para enviar datos (señales) entre el trabajador y los subprocesos principales. En este caso, lo usaremos para decirle al hilo principal que actualice la barra de progreso.

Ahora hemos movido el ciclo while para el contador a una clase separada llamada External.

class External(QThread):
    """
    Runs a counter thread.
    """
    countChanged = pyqtSignal(int)

    def run(self):
        count = 0
        while count < TIME_LIMIT:
            count +=1
            time.sleep(1)
            self.countChanged.emit(count)

Al subclasificar QThread, esencialmente estamos convirtiendo External en una clase que se puede ejecutar en un subproceso separado. Los subprocesos también se pueden iniciar o detener en cualquier momento, lo que se suma a sus beneficios.

Aquí countChanged es el progreso actual y pyqtSignal(int) le dice al subproceso de trabajo que la señal que se envía es del tipo int. Mientras que self.countChanged.emit(count) simplemente envía la señal a cualquier conexión en el subproceso principal (normalmente también se puede usar para comunicarse con otros subprocesos de trabajo).

def onButtonClick(self):
        self.calc = External()
        self.calc.countChanged.connect(self.onCountChanged)
        self.calc.start()

def onCountChanged(self, value):
    self.progress.setValue(value)

Cuando se hace clic en el botón, self.onButtonClick se ejecutará y también iniciará el hilo. El hilo se inicia con .start(). También se debe tener en cuenta que conectamos la señal self.calc.countChanged que creamos anteriormente al método utilizado para actualizar el valor de la barra de progreso. Cada vez que External::run::count se actualiza, el valor int también se envía a onCountChanged.

Así es como podría verse la GUI después de realizar estos cambios.

También debería sentirse mucho más receptivo y no se congelará.
