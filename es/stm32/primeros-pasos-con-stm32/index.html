Configuración por primera vez con ejemplo de LED parpadeante usando SW4STM32 y biblioteca HAL #

(Nota: Hay muchos IDE, cadenas de herramientas y bibliotecas que están listos para usar con STM32. La siguiente configuración requiere un esfuerzo mínimo para que funcione, pero es solo una de muchas. Siéntase libre de explorar otras , el propósito de este ejemplo no es obligar a nadie a usar las herramientas que se usarán aquí).

Instalación del IDE #

System Workbench para STM32: IDE gratuito en Windows, Linux y OS X. Ha sido creado por AC6 y está disponible para descargar después de registrarse desde el sitio web de la comunidad OpenSTM32.

El IDE en sí está basado en Eclipse, pero viene con algunos extras para el desarrollo de STM32 como:

  • Cadena de herramientas Ac6 STM32 MCU GCC
  • OpenOCD y GDB (arm-none-eabi-gdb) con configuraciones de depuración generadas automáticamente según la placa de destino
  • Opciones integradas para programar o borrar chip

Para comenzar con STM32 antes de crear su propia placa, se recomienda experimentar con un [Discovery][5], un [Nucleo][5] o un [Eval board][5], que vienen con un SWD integrado ( Serial Wire Debug) programador/depurador llamado ST-Link.

Creando un proyecto #

Este ejemplo utilizará un [kit de descubrimiento STM32F4][6], que cuenta con un microcontrolador STM32F407VG. (También se puede usar cualquier otra placa).

  1. Abra SW4STM32 y cree un nuevo proyecto C: Archivo → Nuevo → Proyecto C

  2. Asígnele un nombre como “STM32F4_Discovery-Blinky” y, en la lista Tipo de proyecto, elija Ejecutable/Ac6 STM32 MCU Project. De manera predeterminada, la única cadena de herramientas disponible es Ac6 STM32 MCU GCC. Haga clic en Siguiente.

    [![enter image description here][7]][7]

  3. El siguiente paso es Configuración de depuración/liberación, se puede omitir ahora haciendo clic en Siguiente.

  4. Selección de tablero. Se pueden seleccionar tableros existentes como en este ejemplo, el STM32F4 Discovery o se pueden agregar nuevos tableros personalizados.

    [![enter image description here][8]][8]

  5. El siguiente paso es Configuración del firmware del proyecto. Elija entre Sin firmware, Biblioteca de periféricos estándar (SPL) o Capa de abstracción de hardware (HAL). Se cuestiona cuál es más adecuado para el desarrollo, pero esta pregunta está fuera de alcance en este ejemplo. Este ejemplo utilizará la biblioteca HAL, ya que actualmente es compatible con ST Microelectronics. La herramienta de software adicional disponible para HAL es [STM32CubeMX][9], que es un generador de código de inicialización. También hay varias aplicaciones de ejemplo disponibles con los paquetes de software [STM32CubeFx][10] o STM32CubeLx. Descargue el firmware de destino si falta y se recomienda seleccionar las opciones “Agregar controladores de bajo nivel en el proyecto” y “Como fuentes en la aplicación”. Finalmente, haga clic en Finalizar.

    [![enter image description here][11]][11]

Parpadeo LED aplicación #

Como este proyecto se creó con un STM32F4 Discovery, ya hay varias funciones listas para usar en la carpeta del proyecto /STM32F4_Discovery-Blinky/Utilities/STM32F4-Discovery/ que se pueden usar para conectar los periféricos del kit Discovery. (acelerómetro, audio, LEDs, pulsador). En este ejemplo, las funciones void BSP_LED_Init(Led_TypeDef Led) y void BSP_LED_Toggle(Led_TypeDef Led) se utilizarán desde el archivo stm32f4_discovery.c para hacer parpadear el LED verde, que es LED4. Para decidir qué LED es cuál, utilice los esquemas del [Discovery kit][12].

[![ingrese la descripción de la imagen aquí][13]][13]

Los nombres reales de pin y puerto ya están ocultos por algunos #define y enum, use Ctrl + Click para rastrearlos.

  1. Dentro de main, llame a la función HAL_Init() que restablece todos los periféricos, inicializa la interfaz Flash y el Systick. (Systick se usará para generar un retraso para el parpadeo).
  2. El reloj del sistema debe configurarse. Se puede hacer usando la [característica de configuración del reloj STM32CubeMX][14] o mediante el manual de referencia. En este ejemplo, el reloj del sistema es alimentado por el PLL interno (Phase Locked Loop), que es alimentado por un oscilador de cristal (HSE) externo de 8 MHz. Los preescaladores se han configurado para alcanzar la frecuencia máxima disponible, que es de 168 MHz en el caso del F4 Discovery.
  3. Inicialización de los periféricos, en este caso un pin GPIO.
  4. Dentro de un ciclo sin fin, llame al LED alternar y la función HAL_Delay(). HAL_Delay() usa el Systick y genera un retraso en milisegundos.

El código completo es el siguiente:

#include "stm32f4xx.h"
#include "stm32f4_discovery.h"
            
void SystemClock_Config(void);

int main(void)
{
    /* Reset of all peripherals, Initializes the Flash interface and the Systick. */
    HAL_Init();

    /* Configure the system clock */
    SystemClock_Config();

    /* Initialize one of the LED GPIO pin */
    BSP_LED_Init(LED4);

    while(1)
    {
        BSP_LED_Toggle(LED4);
        HAL_Delay(1000);       // in miliseconds
    }
}

/**
 * @brief  System Clock Configuration
 *         The system Clock is configured as follow :
 *            System Clock source            = PLL (HSE)
 *            SYSCLK(Hz)                     = 168000000
 *            HCLK(Hz)                       = 168000000
 *            AHB Prescaler                  = 1
 *            APB1 Prescaler                 = 4
 *            APB2 Prescaler                 = 2
 *            HSE Frequency(Hz)              = HSE_VALUE
 *            PLL_M                          = (HSE_VALUE/1000000u)
 *            PLL_N                          = 336
 *            PLL_P                          = 2
 *            PLL_Q                          = 7
 *            VDD(V)                         = 3.3
 *            Main regulator output voltage  = Scale1 mode
 *            Flash Latency(WS)              = 5
 * @param  None
 * @retval None
 */
void SystemClock_Config(void)
{
  RCC_ClkInitTypeDef RCC_ClkInitStruct;
  RCC_OscInitTypeDef RCC_OscInitStruct;

  // Enable Power Control clock
  __PWR_CLK_ENABLE();

  // The voltage scaling allows optimizing the power consumption when the
  // device is clocked below the maximum system frequency, to update the
  // voltage scaling value regarding system frequency refer to product
  // datasheet.
  __HAL_PWR_VOLTAGESCALING_CONFIG(PWR_REGULATOR_VOLTAGE_SCALE1);

  // Enable HSE Oscillator and activate PLL with HSE as source
  RCC_OscInitStruct.OscillatorType = RCC_OSCILLATORTYPE_HSE;
  RCC_OscInitStruct.HSEState = RCC_HSE_ON;
  RCC_OscInitStruct.PLL.PLLState = RCC_PLL_ON;
  RCC_OscInitStruct.PLL.PLLSource = RCC_PLLSOURCE_HSE;

  // This assumes the HSE_VALUE is a multiple of 1MHz. If this is not
  // your case, you have to recompute these PLL constants.
  RCC_OscInitStruct.PLL.PLLM = (HSE_VALUE/1000000u);
  RCC_OscInitStruct.PLL.PLLN = 336;
  RCC_OscInitStruct.PLL.PLLP = RCC_PLLP_DIV2;
  RCC_OscInitStruct.PLL.PLLQ = 7;
  HAL_RCC_OscConfig(&RCC_OscInitStruct);

  // Select PLL as system clock source and configure the HCLK, PCLK1 and PCLK2
  // clocks dividers
  RCC_ClkInitStruct.ClockType = (RCC_CLOCKTYPE_SYSCLK | RCC_CLOCKTYPE_HCLK
      | RCC_CLOCKTYPE_PCLK1 | RCC_CLOCKTYPE_PCLK2);
  RCC_ClkInitStruct.SYSCLKSource = RCC_SYSCLKSOURCE_PLLCLK;
  RCC_ClkInitStruct.AHBCLKDivider = RCC_SYSCLK_DIV1;
  RCC_ClkInitStruct.APB1CLKDivider = RCC_HCLK_DIV4;
  RCC_ClkInitStruct.APB2CLKDivider = RCC_HCLK_DIV2;
  HAL_RCC_ClockConfig(&RCC_ClkInitStruct, FLASH_LATENCY_5);
}

Construya con el martillo [![ingrese la descripción de la imagen aquí][15]][15], y descargue la aplicación haciendo clic derecho en la carpeta del proyecto y seleccionando Target → Chip de programa… opción.

Otra forma de descargar es usando debug. Para hacerlo, haga clic en la flecha al lado del icono de error [![ingrese la descripción de la imagen aquí][16]][16] en la barra de herramientas y abra el menú Configuración de depuración…. Cree una nueva configuración de Ac6 STM32 Debugging y si el campo C/C++ Application está vacío, complete lo siguiente:

Depurar\STM32F4_Discovery-Blinky.elf

El marco genera y completa automáticamente otros parámetros de depuración, como el archivo de configuración de OpenOCD y los puertos Telnet y GDB utilizados. Finalmente, haga clic en el botón Depurar.
