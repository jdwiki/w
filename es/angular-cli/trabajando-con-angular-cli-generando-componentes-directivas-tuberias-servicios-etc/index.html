La herramienta angular-cli puede ayudarlo a crear andamios en diferentes partes de una aplicación angular (componentes, directivas, tuberías, servicios, clases, guardias, interfaces, enumeraciones y módulos).

Sintaxis #

  • ng generar [componente | directiva | servicio | pipa | clase | enumeración | interfaz | guardia | módulo] [nombre] [banderas…]
  • ng g [c | re | s | pag | cl | mi | yo | gramo | m] [nombre] [banderas…]

Parámetros #

Parámetro Descripción
componente o c Se utiliza para generar componentes
directiva o d Se utiliza para generar directivas.
servicio o s Se utiliza para generar servicios.
tubería o p Se utiliza para generar tuberías.
clase o cl Se utiliza para generar clases.
enum o e Se utiliza para generar enumeraciones
interfaces o i Se utiliza para generar interfaces.
guardia o g Se utiliza para generar guardias.
módulo o m Se utiliza para generar módulos.
--plano o -f Se utiliza para habilitar/deshabilitar la creación de directorios
--plantilla-en-línea o -it Se utiliza para habilitar/deshabilitar plantillas html en línea en componentes
--estilo-en-línea o -es Se utiliza para habilitar/deshabilitar estilos en línea en componentes
--prefijo o -p Se usa para deshabilitar o cambiar el prefijo
--spec o -s Se utiliza para habilitar/deshabilitar la creación de archivos .spec
--skip-import Se utiliza para omitir la importación del módulo.
--app o -a Se usa para especificar el nombre de la aplicación que se usará
--módulo o -m Se utiliza para especificar el módulo declarante.
--view-encapsulation o -ve Se utiliza para especificar la estrategia de encapsulación de vistas en los componentes.
--detección de cambios o -cd Se utiliza para especificar la estrategia de detección de cambios en los componentes.
--routing o -r Se utiliza para especificar si se debe generar el archivo del módulo de enrutamiento

Uso de “Generar comando” #

Puede usar el comando ng generate o ng g para generar bloques de construcción angulares (componentes, servicios, tuberías, etc.).

Puede encontrar todos los planos posibles en la siguiente tabla:

Andamio Uso Acortado
Componente of generar componente nombre-componente de g c componente-nombre
Directiva ng generar directiva nombre-directiva ng g d nombre-directiva
Pipa ng generar tubería nombre-tubería ng g p pipe-name
Servicio ng generar servicio nombre-servicio ng g s nombre-servicio
Clase ng generar clase nombre-clase ng g cl nombre-clase
Guardia ng generar guardia nombre-guardia ng g g guardia-nombre
Interfaz ng generar interfaz nombre-interfaz ng g i nombre-interfaz
enumeración ng generar enumeración nombre-enum ng g e nombre-enum
Módulo of generar módulo nombre-módulo of g m module-name

Entonces, por ejemplo, si ejecuta ng generar lista de usuarios de componentes - angular-cli hará lo siguiente:

  • Cree el directorio user-list en la carpeta src/app o en la carpeta donde ejecutó el comando.
  • dentro de ese directorio genera 4 archivos (user-list.component.ts, user-list.component.html, user-list.component.css y user-list.component.spec.ts)
  • agregue user-list como una declaración en el decorador @NgModule del módulo más cercano.

Generación de servicios #

Para agregar un servicio con el nombre UserService, ejecute:

  $ ng g s user

  installing service
    create src/app/user.service.spec.ts
    create src/app/user.service.ts
  • Para evitar la creación de archivos ‘.spec’, agregue el indicador ‘–spec false’ o ‘-sp false’
  $ ng g s user --spec false
  
  installing service
    create src/app/user.service.ts
  • Para habilitar la creación de carpetas, agregue el indicador --flat false o -f false
  $ ng g s user --flat false

  installing service
    create src/app/user/user.service.spec.ts
    create src/app/user/user.service.ts

También puede combinar las banderas mencionadas anteriormente. Por ejemplo, para crear solo el archivo user.service.ts dentro de la carpeta user sin el archivo .spec use el siguiente comando.

  $ ng g s user -f false -sp false

  installing service
    create src/app/user/user.service.ts

Todas las banderas generar servicio:

Descripción Bandera Acortado Valor por defecto
Habilitar la creación de carpetas --flat false -f falso verdadero
Impedir la creación de archivos .spec --espec falso -sp falso verdadero

Generando módulos #

Para agregar un módulo llamado GuestModule, ejecute:

  $ ng g m guest

  installing module
    create src/app/guest/guest.module.ts
  • Para habilitar la creación de archivos ‘.spec’, agregue el indicador ‘–spec’ o ‘-sp’
  $ ng g m guest --spec

  installing module
    create src/app/guest/guest.module.spec.ts
    create src/app/guest/guest.module.ts
  • Para habilitar el enrutamiento, agregue el indicador --routing o -r
  $ ng g m guest --routing

  installing module
    create src/app/guest/guest-routing.module.ts
    create src/app/guest/guest.module.ts

También puede combinar las banderas mencionadas anteriormente. Por ejemplo, para crear un módulo con enrutamiento y especificaciones, use el siguiente comando.

  $ ng g m guest -sp -r
  
  installing module
    create src/app/guest/guest-routing.module.ts
    create src/app/guest/guest.module.spec.ts
    create src/app/guest/guest.module.ts

Todas las banderas generar módulo:

Descripción Bandera Acortado Valor por defecto
Habilitar la creación de archivos .spec --spec -sp falso
Habilitar enrutamiento --enrutamiento -r falso

Generación de componentes #

Para agregar un componente con un selector [prefix]-user-list, ejecute:

  $ ng g c user-list
  
  installing component
    create src/app/user-list/user-list.component.css
    create src/app/user-list/user-list.component.html
    create src/app/user-list/user-list.component.spec.ts
    create src/app/user-list/user-list.component.ts
    update src/app/app.module.ts

prefijo evita colisiones de nombres de elementos con componentes en otras aplicaciones y con elementos HTML nativos. Entonces, por ejemplo, si el prefijo es app, el componente generado tendrá el selector app-user-list.

  • Para evitar el uso de prefijos, agregue la bandera --prefix false o -p false
  $ ng g c user-list --prefix false

import { Component } from '@angular/core';

@Component({
  selector: 'user-list',
  templateUrl: './user-list.component.html',
  styleUrls: ['./user-list.component.css']
})
export class UserListComponent {}
  • Para evitar la creación de archivos ‘.spec’, agregue el indicador ‘–spec false’ o ‘-sp false’
  $ ng g c user-list --spec false

  installing component
    create src/app/user-list/user-list.component.css
    create src/app/user-list/user-list.component.html
    create src/app/user-list/user-list.component.ts
    update src/app/app.module.ts
  • Para usar plantillas html en línea en lugar de plantillas externas, agregue la bandera --inline-template o -it
  $ ng g c user-list --inline-template

  installing component
    create src/app/user-list/user-list.component.css
    create src/app/user-list/user-list.component.spec.ts
    create src/app/user-list/user-list.component.ts
    update src/app/app.module.ts
  • Para usar estilos en línea en lugar de estilos externos, agregue --inline-style o -is flag
  $ ng g c user-list --inline-style

  installing component
    create src/app/user-list/user-list.component.html
    create src/app/user-list/user-list.component.spec.ts
    create src/app/user-list/user-list.component.ts
    update src/app/app.module.ts
  • Para evitar la creación de carpetas, agregue el indicador --flat o -f
  $ ng g c user-list --flat
  
  installing component
    create src/app/user-list.component.css
    create src/app/user-list.component.html
    create src/app/user-list.component.spec.ts
    create src/app/user-list.component.ts
    update src/app/app.module.ts

También puede combinar las banderas mencionadas anteriormente. Por ejemplo, para crear solo el archivo .component.ts sin los archivos y la carpeta .css, .html, .spec, use el siguiente comando.

  $ ng g c user-list -f -it -is -sp false
  
  installing component
    create src/app/user-list.component.ts
    update src/app/app.module.ts

Todas las banderas generar componente:

Descripción Bandera Acortado Valor por defecto
Impedir la creación de carpetas --plano -f falso
Impedir el uso de prefijos --prefijo falso -p falso verdadero
Impedir la creación de archivos .spec --espec falso -sp falso verdadero
Habilitar plantillas html en línea --plantilla-en-línea -eso falso
Habilitar estilos en línea --estilo en línea -es falso

Generando directivas #

Para agregar una directiva con un selector [prefijo]Highlight, ejecute:

  $ ng g d highlight

  installing directive
    create src/app/highlight.directive.spec.ts
    create src/app/highlight.directive.ts
    update src/app/app.module.ts
  • Para evitar el uso de prefijos, agregue la bandera --prefix false o -p false
  $ ng g d highlight --prefix false

import { Directive } from '@angular/core';

@Directive({
  selector: '[highlight]'
})
export class HighlightDirective {}
  • Para evitar la creación de archivos ‘.spec’, agregue el indicador ‘–spec false’ o ‘-sp false’
  $ ng g d highlight --spec false

  installing directive
    create src/app/highlight.directive.ts
    update src/app/app.module.ts
  • Para habilitar la creación de carpetas, agregue el indicador --flat false o -f false
  $ ng g d highlight --flat false
  
  installing directive
    create src/app/highlight/highlight.directive.spec.ts
    create src/app/highlight/highlight.directive.ts
    update src/app/app.module.ts

También puede combinar las banderas mencionadas anteriormente. Por ejemplo, para crear solo el archivo highlight.directive.ts dentro de la carpeta highlight sin el archivo .spec, use el siguiente comando.

  $ ng g d highlight -f false -sp false

  installing directive
    create src/app/highlight/highlight.directive.ts
    update src/app/app.module.ts

Todas las banderas generar directiva:

Descripción Bandera Acortado Valor por defecto
Habilitar la creación de carpetas --flat false -f falso verdadero
Impedir el uso de prefijos --prefijo falso -p falso verdadero
Impedir la creación de archivos .spec --espec falso -sp falso verdadero
