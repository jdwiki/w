Sintaxis #

  • var jqXHR = $.ajax (url [, configuración])
  • var jqXHR = $.ajax( [configuración] )
  • jqXHR.done(función(datos, textStatus, jqXHR) {});
  • jqXHR.fail(function( jqXHR, textStatus, errorThrown ) {});
  • jqXHR.siempre(función( jqXHR ) {});

Parámetros #

Parámetro Detalles
URL Especifica la URL a la que se enviará la solicitud
ajustes un objeto que contiene numerosos valores que afectan el comportamiento de la solicitud
tipo El método HTTP que se utilizará para la solicitud
datos Datos a enviar por la solicitud
éxito Una función de devolución de llamada que se llamará si la solicitud tiene éxito
error Una devolución de llamada para manejar el error
código de estado Un objeto de códigos HTTP numéricos y funciones que se llamarán cuando la respuesta tenga el código correspondiente
dataType El tipo de datos que espera del servidor
tipo de contenido Tipo de contenido de los datos a enviar al servidor. El valor predeterminado es “aplicación/x-www-form-urlencoded; charset=UTF-8”
contexto Especifica el contexto que se usará dentro de las devoluciones de llamada, generalmente this que se refiere al objetivo actual.

AJAX significa Asincrónico JavaScript and XML. AJAX permite que una página web realice una solicitud HTTP asíncrona (AJAX) al servidor y reciba una respuesta, sin necesidad de recargar toda la página.

Manejo de códigos de respuesta HTTP con $.ajax() #

Además de .done, .fail y .always prometen devoluciones de llamada, que se activan en función de si la solicitud fue exitosa o no, existe la opción de activar una función cuando un Código de estado HTTP se devuelve desde el servidor. Esto se puede hacer usando el parámetro statusCode.

$.ajax({
    type: {POST or GET or PUT etc.},
    url:  {server.url},
    data: {someData: true},
    statusCode: {
        404: function(responseObject, textStatus, jqXHR) {
            // No content found (404)
            // This code will be executed if the server returns a 404 response
        },
        503: function(responseObject, textStatus, errorThrown) {
            // Service Unavailable (503)
            // This code will be executed if the server returns a 503 response
        }           
    }
})
.done(function(data){
    alert(data);
})
.fail(function(jqXHR, textStatus){
    alert('Something went wrong: ' + textStatus);
})
.always(function(jqXHR, textStatus) {
   alert('Ajax request was finished')
});

Como dice la documentación oficial de jQuery:

Si la solicitud es exitosa, las funciones del código de estado toman la misma parámetros como la devolución de llamada exitosa; si resulta en un error (incluida la redirección 3xx), toman los mismos parámetros que el error devolución de llamada.

Uso de Ajax para enviar un formulario #

A veces puede tener un formulario y desea enviarlo usando ajax.

Supongamos que tiene este formulario simple:

<form id="ajax_form" action="form_action.php">
  <label for="name">Name :</label>
  <input name="name" id="name" type="text" />
  <label for="name">Email :</label>
  <input name="email" id="email" type="text" />
  <input type="submit" value="Submit" />
</form>

Se puede usar el siguiente código jQuery (dentro de una llamada $(document).ready) -

$('#ajax_form').submit(function(event){
  event.preventDefault();
  var $form = $(this);

  $.ajax({
      type: 'POST',
      url: $form.attr('action'),
      data: $form.serialize(),
      success: function(data) {
        // Do something with the response
      },
      error: function(error) {
        // Do something with the error
      }
  });
});

Explicación

  • var $form = $(this) - el formulario, almacenado en caché para su reutilización
  • $('#ajax_form').submit(function(event){ - Cuando el formulario con ID Se envía “ajax_form”, ejecute esta función y pase el evento como un parámetro.
  • event.preventDefault(); - Evita que el formulario se envíe normalmente (Alternativamente, podemos usar return false después de la declaración ajax({});, que tendrá el mismo efecto)
  • url: $form.attr('action'), - Obtenga el valor del atributo “action” del formulario y utilícelo para la propiedad “url”.
  • data: $form.serialize(), - Convierte las entradas dentro del formulario en una cadena adecuada para enviar al servidor. En este caso devolverá algo como “nombre=Bob&email=bob@bobsemailaddress.com”

Todo en uno ejemplos #

Ajax Obtener:

Solución 1:

$.get('url.html', function(data){
    $('#update-box').html(data);
});

Solución 2:

 $.ajax({
     type: 'GET',
     url: 'url.php',  
 }).done(function(data){
     $('#update-box').html(data);
 }).fail(function(jqXHR, textStatus){
     alert('Error occured: ' + textStatus);
 });

Carga de Ajax: Otro método de obtención de ajax creado para simplificar

$('#update-box').load('url.html');

.load también se puede llamar con datos adicionales. La parte de datos se puede proporcionar como cadena u objeto.

$('#update-box').load('url.php', {data: "something"});
$('#update-box').load('url.php', "data=something");

Si se llama a .load con un método de devolución de llamada, la solicitud al servidor será una publicación

$('#update-box').load('url.php', {data: "something"}, function(resolve){
    //do something
});

Publicación de Ajax:

Solución 1:

$.post('url.php', 
    {date1Name: data1Value, date2Name: data2Value},  //data to be posted
    function(data){
        $('#update-box').html(data);
    }
);

Solución 2:

$.ajax({
    type: 'Post',
    url: 'url.php',  
    data: {date1Name: data1Value, date2Name: data2Value}  //data to be posted
}).done(function(data){
    $('#update-box').html(data);
}).fail(function(jqXHR, textStatus){
    alert('Error occured: ' + textStatus);
});

Publicación de Ajax JSON:

var postData = {
    Name: name,
    Address: address,
    Phone: phone
};

$.ajax({
     type: "POST",
     url: "url.php",
     dataType: "json",
     data: JSON.stringfy(postData),
     success: function (data) {
         //here variable data is in JSON format
     }
 });

Solución 1:

$.getJSON('url.php', function(data){

Cargas de archivos Ajax #

1. Un ejemplo completo simple #

Podríamos usar este código de muestra para cargar los archivos seleccionados por el usuario cada vez que se realiza una nueva selección de archivos.

<input type="file" id="file-input" multiple>

 

var files;
var fdata = new FormData();
$("#file-input").on("change", function (e) {
    files = this.files;

    $.each(files, function (i, file) {
        fdata.append("file" + i, file);
    });

    fdata.append("FullName", "John Doe");
    fdata.append("Gender", "Male");
    fdata.append("Age", "24");

    $.ajax({
        url: "/Test/Url",
        type: "post",
        data: fdata, //add the FormData object to the data parameter
        processData: false, //tell jquery not to process data
        contentType: false, //tell jquery not to set content-type
        success: function (response, status, jqxhr) {
            //handle success
        },
        error: function (jqxhr, status, errorMessage) {
            //handle error
        }
    });
});

Ahora analicemos esto e inspeccionémoslo parte por parte.

2. Trabajar con entradas de archivos #

Este Documento MDN (Uso de archivos de aplicaciones web) es una buena lectura sobre varios métodos sobre cómo manejar las entradas de archivos. Algunos de estos métodos también se utilizarán en este ejemplo.

Antes de cargar archivos, primero debemos darle al usuario una forma de seleccionar los archivos que desea cargar. Para ello utilizaremos un archivo de entrada. La propiedad múltiple permite seleccionar más de un archivo, puede eliminarla si desea que el usuario seleccione un archivo a la vez.

<input type="file" id="file-input" multiple>

Usaremos el cambio de evento de entrada para capturar los archivos.

var files;
$("#file-input").on("change", function(e){
    files = this.files;
});

Dentro de la función del controlador, accedemos a los archivos a través de la propiedad de archivos de nuestra entrada. Esto nos da una FileList, que es un objeto tipo matriz.

3. Creación y llenado de FormData #

Para subir archivos con Ajax vamos a usar FormData.

var fdata = new FormData();

FileList que hemos obtenido en el paso anterior es un objeto similar a una matriz y se puede iterar usando varios métodos, incluidos for loop, for…of loop y jQuery.each . Nos quedaremos con jQuery en este ejemplo.

$.each(files, function(i, file) {
  //...
});

Usaremos el método de agregar de FormData para agregar los archivos a nuestro objeto formdata.

$.each(files, function(i, file) {
  fdata.append("file" + i, file);
});

También podemos añadir otros datos que queramos enviar de la misma forma. Digamos que queremos enviar información personal que hemos recibido del usuario junto con los archivos. Podríamos agregar esta información a nuestro objeto formdata.

fdata.append("FullName", "John Doe");
fdata.append("Gender", "Male");
fdata.append("Age", "24");
//...

4. Envío de archivos con Ajax #

$.ajax({
    url: "/Test/Url",
    type: "post",
    data: fdata, //add the FormData object to the data parameter
    processData: false, //tell jquery not to process data
    contentType: false, //tell jquery not to set content-type
    success: function (response, status, jqxhr) {
        //handle success
    },
    error: function (jqxhr, status, errorMessage) {
        //handle error
    }
});

Establecemos las propiedades processData y contentType en falso. Esto se hace para que los archivos puedan enviarse al servidor y ser procesados ​​por el servidor correctamente.

Ajax cancela una llamada o solicitud #

var xhr = $.ajax({
    type: "POST",
    url: "some.php",
    data: "name=John&location=Boston",
    success: function(msg){
       alert( "Data Saved: " + msg );
    }
});

// mata la solicitud

xhr.abort()

Enviando datos JSON #

jQuery hace que el manejo de las respuestas jSON sea sencillo, pero se requiere un poco más de trabajo cuando una determinada solicitud desea que envíe datos en formato JSON:

 $.ajax("/json-consuming-route", {
      data: JSON.stringify({author: {name: "Bullwinkle J. Moose", 
                                     email: "bullwinkle@example.com"} }),
      method: "POST",
      contentType: "application/json"
   });

Observe que estamos especificando el contentType correcto para los datos que estamos enviando; esta es una buena práctica en general y puede ser requerida por la API en la que está publicando, pero también tiene el efecto secundario de indicar a jQuery que no realice la conversión predeterminada de %20 a +, que funcionaría si contentType se dejara en el valor predeterminado de application/x-www-form-urlencoded. Si por alguna razón debe dejar contentType establecido en el valor predeterminado, asegúrese de establecer processData en falso para evitarlo.

La llamada a [JSON.stringify](https://www.wikiod.com/es/javascript/json#Serializar un valor) podría evitarse aquí, pero usando nos permite proporcionar los datos en forma de un objeto de JavaScript (evitando así vergonzosos errores de sintaxis JSON, como no citar los nombres de las propiedades).
