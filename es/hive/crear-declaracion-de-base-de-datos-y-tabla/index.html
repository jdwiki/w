Sintaxis #

  • CREAR TABLA [TEMPORAL] [EXTERNA] [SI NO EXISTE] [nombre_bd.]nombre_tabla

    [(col_name data_type [COMMENT col_comment], ...)]

[COMENTARIO tabla_comentario] [PARTICIONADO POR (col_name data_type [COMENTARIO col_comment], …)] [ACLUSTRADO POR (col_name, col_name, …) [ORDENADO POR (col_name [ASC|DESC], …)] EN num_buckets BUCKETS] [SESGADO POR (col_name, col_name, …) – (Nota: disponible en Hive 0.10.0 y versiones posteriores)] ON ((col_value, col_value, …), (col_value, col_value, …), …) [STORED AS DIRECTORIES] [ [FORMATO DE FILA formato_fila] [ALMACENADO COMO file_format] | STORED BY ‘storage.handler.class.name’ [WITH SERDEPROPERTIES (…)] ] [UBICACIÓN ruta_hdfs] [PROPIEDADESTBL (nombre_propiedad=valor_propiedad, …)] [AS select_statement];

  • CREAR TABLA [TEMPORAL] [EXTERNA] [SI NO EXISTE] [nombre_bd.]nombre_tabla ME GUSTA tabla_existente_o_vista_nombre [UBICACIÓN ruta_hdfs];
  • tipo_de_datos : tipo_primitivo, tipo_matriz, tipo_mapa, tipo_estructura, tipo_unión
  • tipo_primitivo: TINYINT, SMALLINT, INT , BIGINT, BOOLEAN, FLOAT, DOUBLE, STRING, BINARY, TIMESTAMP, DECIMAL, DECIMAL(precisión, escala), DATE, VARCHAR, CHAR
  • tipo_matriz: ARRAY <tipo_datos>
  • tipo_mapa: MAP < tipo_primitivo, tipo_datos >
  • tipo_estructura: STRUCT < nombre_col : tipo_datos [COMENTARIO comentario_col], …>
  • union_type: UNIONTYPE < tipo_datos, tipo_datos, … >
  • formato_fila: DELIMITADO [CAMPOS TERMINADOS POR char [ESCAPADO POR char]] [ARTÍCULOS DE LA COLECCIÓN TERMINADOS POR char] [MAP KEYS TERMINATED BY char] [LINES TERMINATED BY char] [NULL DEFINED AS char]
    , SERDE nombre_serde [CON SERDEPROPIEDADES (nombre_propiedad=valor_propiedad, nombre_propiedad=valor_propiedad, …)]
  • formato de archivo: : ARCHIVO DE SECUENCIA , TEXTFILE , RCFILE , ORC , PARQUET , AVRO , INPUTFORMAT nombre_clase_formato_entrada OUTPUTFORMAT nombre_clase_formato_salida
  • CREAR (BASE DE DATOS|ESQUEMA) [SI NO EXISTE] nombre_base_datos [COMENTARIO base de datos_comentario] [UBICACIÓN ruta_hdfs] [CON PROPIEDADES DB (nombre_propiedad=valor_propiedad, …)];

Al trabajar con tablas y bases de datos en HIVE. Los siguientes puntos pueden ser útiles.

  • Podemos cambiar la base de datos usando use database; command
  • Para conocer la base de datos de trabajo actual, podemos obtener usando SELECT current_database ()
  • Para ver el DDL utilizado para la declaración de creación de tabla, podemos usar SHOW CREATE TABLE tablename
  • Para ver todas las columnas de la tabla, use DESCRIBE tablename para mostrar detalles extendidos como la ubicación utilizada y otros DESCRIBE FORMATTED tablename. DESCRIBE también se puede abreviar como DESC.

Crear mesa #

Creación de una tabla administrada con partición y almacenada como un archivo de secuencia. Se supone que el formato de datos en los archivos está delimitado por campos por Ctrl-A (^A) y delimitado por filas por nueva línea. La siguiente tabla se crea en el directorio de almacén de Hive especificado en el valor de la clave hive.metastore.warehouse.dir en el archivo de configuración de Hive hive-site.xml.

CREATE TABLE view
(time INT, 
id BIGINT,
url STRING, 
referrer_url STRING,
add STRING COMMENT 'IP of the User')
COMMENT 'This is view table'
PARTITIONED BY(date STRING, region STRING)
ROW FORMAT DELIMITED
FIELDS TERMINATED BY '\001'
STORED AS SEQUENCEFILE;

Creación de una tabla externa con particiones y almacenada como un archivo de secuencia. Se supone que el formato de datos en los archivos está delimitado por campos por ctrl-A y delimitado por filas por nueva línea. La siguiente tabla se crea en la ubicación especificada y es útil cuando ya tenemos datos. Una de las ventajas de usar una tabla externa es que podemos soltar la tabla sin borrar los datos. Por ejemplo, si creamos una tabla y nos damos cuenta de que el esquema es incorrecto, podemos descartar la tabla de manera segura y volver a crearla con el nuevo esquema sin preocuparnos por los datos. Otra ventaja es que si usamos otras herramientas como pig en los mismos archivos, podemos seguir usándolos incluso después de eliminar la tabla.

CREATE EXTERNAL TABLE view
(time INT, 
id BIGINT,
url STRING, 
referrer_url STRING,
add STRING COMMENT 'IP of the User')
COMMENT 'This is view table'
PARTITIONED BY(date STRING, region STRING)
ROW FORMAT DELIMITED
FIELDS TERMINATED BY '\001'
STORED AS SEQUENCEFILE
LOCATION '<hdfs_location>';

Crear una tabla usando la consulta de selección y completar los resultados de la consulta, these statements are known as CTAS(Create Table As Select).

Hay dos partes en CTAS, la parte SELECT puede ser cualquier instrucción SELECT admitida por HiveQL. La parte CREAR de CTAS toma el esquema resultante de la parte SELECCIONAR y crea la tabla de destino con otras propiedades de la tabla, como el formato de almacenamiento y SerDe.

CTAS tiene estas restricciones:

  • La tabla de destino no puede ser una tabla particionada.

  • La tabla de destino no puede ser una tabla externa.

  • La tabla de destino no puede ser una tabla de creación de depósitos de lista.

    CREATE TABLE new_key_value_store ROW FORMAT SERDE “org.apache.hadoop.hive.serde2.columnar.ColumnarSerDe” STORED AS RCFile AS SELECT * FROM page_view SORT BY url, add;

Crear tabla como:

La forma LIKE de CREATE TABLE le permite copiar exactamente una definición de tabla existente (sin copiar sus datos). A diferencia de CTAS, la siguiente declaración crea una nueva tabla cuya definición coincide exactamente con la tabla existente en todos los detalles, excepto en el nombre de la tabla. La nueva tabla no contiene filas.

CREATE TABLE empty_page_views
LIKE page_views;

Crear base de datos #

Crear una base de datos en una ubicación particular. Si no especificamos ninguna ubicación para la base de datos, se creará en el directorio del almacén.

CREATE DATABASE IF NOT EXISTS db_name 
COMMENT 'TEST DATABASE'
LOCATION /PATH/HDFS/DATABASE/;

Integración HIVE_HBASE #

La integración de Hive-Hbase es compatible desde las siguientes versiones. Colmena: 0.11.0 HBase: 0.94.2 Hadoop: 0.20.2

CREATE TABLE hbase_hive
(id string,
 col1 string,
 col2 string,
 col3 int) 
STORED BY 'org.apache.hadoop.hive.hbase.HBaseStorageHandler'
WITH SERDEPROPERTIES 
("hbase.columns.mapping" = ":key,cf1:col1,cf1:col2,cf1:col3")
TBLPROPERTIES ("hbase.table.name" = "hive_hbase");

Nota: la primera columna debe ser la columna clave.

Crear una tabla usando las propiedades de la tabla existente. #

CREATE TABLE new_table_name LIKE existing_table_name;

Creación de la tabla Hive ACID. #

Las tablas ACID son compatibles desde la versión Hive 0.14. La siguiente tabla admite ACTUALIZAR/ELIMINAR/INSERTAR

A continuación se requieren cambios de configuración en hive-site.xml.

 hive.support.concurrency = true
 hive.enforce.bucketing = true
 hive.exec.dynamic.partition.mode = nonstrict
 hive.txn.manager =org.apache.hadoop.hive.ql.lockmgr.DbTxnManager
 hive.compactor.initiator.on = true
 hive.compactor.worker.threads = 1

Actualmente, solo se admite el formato de archivo orc.

Sentencia de creación de tabla.

 create table Sample_Table(
 col1 Int,
 col2 String,
 col3 String) 
 clustered by (col3) into 3 buckets 
 stored as orc 
 TBLPROPERTIES ('transactional'='true');
