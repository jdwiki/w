Componentes y control de versiones en .NET Core

  • Identificación de versiones
  • Selección de versiones

En esta página

  • Identificación de versiones
  • Selección de versiones

Este documento cubre los diferentes componentes que conforman una distribución de .NET Core y cómo se controlan sus versiones. Este documento actualmente cubre las versiones 1.x.

Cómo se versionan los componentes en .NET Core.

Componentes #

.NET Core consta de varios componentes, cada uno de los cuales está versionado independientemente y, a menudo, se pueden mezclar y combinar.

  • Marco compartido. Este contiene las API y la máquina virtual. y otros servicios de tiempo de ejecución necesarios para ejecutar aplicaciones .NET Core.

  • La máquina virtual .NET Core actual se llama CoreCLR. Este executes the .NET bytecode by compiling it JIT and provides various runtime services including a garbage collector. The complete source code for CoreCLR is available at https://github.com/dotnet/coreclr.

  • Las API estándar de .NET Core se implementan en CoreFX. Este provides implementations of all your favourite APIs such as System.Runtime, System.Theading and so on. The source code for CoreFX is available at https://github.com/dotnet/corefx.

  • Host también se llama muxer o driver. Estos componentes representa el comando dotnet y es responsable de decidir qué sucede a continuación. La fuente de esto está disponible en https://github.com/dotnet/core-setup.

  • SDK también se denomina a veces CLI. consiste en el varias herramientas (subcomandos dotnet) y sus implementaciones que lidiar con el código de construcción. Esto incluye el manejo de la restauración de dependencias, compilar código, construir binarios, producir paquetes y publicar paquetes independientes o dependientes del marco. El SDK en sí mismo consiste en CLI, que maneja las operaciones de la línea de comandos (en https://github.com/dotnet/cli) y varios subproyectos que implementar las diversas operaciones que debe realizar la CLI.

Componentes en instalaciones .NET Core #

Varios paquetes oficiales y no oficiales, tarballs, zips e instaladores para .NET Core (incluidos los disponibles en https://dot.net/core) proporcionar .NET Core en muchas variantes. Dos comunes son SDK y Tiempos de ejecución.

Cada instalación de SDK o instalación de tiempo de ejecución contiene un número (posiblemente 0) de hosts, sdk y componentes de marco compartido descritos anteriormente.

  • .NET Core Tiempo de ejecución contiene

  • 1 versión de Marco Compartido

  • 1 versión del Anfitrión

  • .NET Core SDK contiene

  • 1 o más versiones del Shared Framework (varía según the version of the SDK)

  • 1 versión del Anfitrión

  • 1 versión del SDK

Identificación de versiones #

Cada componente de .NET Core (SDK, Host y Shared Framework) se versiona de forma independiente.

Puede encontrar la versión para cada uno de ellos por separado.

  • SDK

Puede usar la opción --version para dotnet para ver el SDK versión. Por ejemplo:

```
$ ~/dotnet-1.1.1/dotnet --version
1.0.0-preview2-1-003176
```

dotnet --info también muestra la versión SDK.

  • Anfitrión

Puede ejecutar dotnet por sí mismo sin ningún argumento u opción para ver la versión del host.

```
$ ~/dotnet-1.1.1/dotnet

Microsoft .NET Core Shared Framework Host

  Version  : 1.1.0
  Build    : 362e48a95c86b40cd1f2ef3d08741f7fed897956

Usage: dotnet [common-options] [[options] path-to-application]
...
```
  • Marco compartido

Actualmente no hay ningún comando para mostrar el contenido compartido disponible. marcos Yo uso ls /ruta/hacia/dónde/usted/instaló/dotnet/shared/Microsoft.NETCore.App que se basa en detalles de implementación internos. Por ejemplo:

```
$ ls ~/dotnet-1.1.1/shared/Microsoft.NETCore.App/
1.1.1
```

Selección de versiones #

Es posible tener múltiples .NET Core SDKs y Runtimes disponible en disco. Puede seleccionar las versiones para cada uno por separado.

Para seleccionar la versión del SDK a usar, use global.json.

Para seleccionar la versión del marco compartido que se utilizará, apunte al marco especificado en el archivo [.csproj (o project.json si todavía lo está usando)] (https://docs.microsoft.com/en -us/dotnet/articles/core/tools/project-json-to-csproj#frameworks).
