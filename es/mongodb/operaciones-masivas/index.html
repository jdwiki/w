Convirtiendo un campo a otro tipo y actualizando la colección completa de forma masiva #

Por lo general, el caso cuando uno quiere cambiar un tipo de campo a otro, por ejemplo, la colección original puede tener campos “numéricos” o de “fecha” guardados como cadenas:

{
    "name": "Alice",
    "salary": "57871",
    "dob": "1986-08-21"
},
{
    "name": "Bob",
    "salary": "48974",
    "dob": "1990-11-04"
}

El objetivo sería actualizar una enorme colección como la anterior para

{
    "name": "Alice",
    "salary": 57871,
    "dob": ISODate("1986-08-21T00:00:00.000Z")
},
{
    "name": "Bob",
    "salary": 48974,
    "dob": ISODate("1990-11-04T00:00:00.000Z")
}

Para datos relativamente pequeños, se puede lograr lo anterior iterando la colección usando una instantánea con el método forEach() del cursor y actualizando cada documento como sigue:

db.test.find({
    "salary": { "$exists": true, "$type": 2 },
    "dob": { "$exists": true, "$type": 2 }
}).snapshot().forEach(function(doc){ 
    var newSalary = parseInt(doc.salary),
        newDob = new ISODate(doc.dob);        
    db.test.updateOne(
        { "_id": doc._id },
        { "$set": { "salary": newSalary, "dob": newDob } }
    );
});

Si bien esto es óptimo para colecciones pequeñas, el rendimiento con colecciones grandes se reduce considerablemente, ya que recorrer un gran conjunto de datos y enviar cada operación de actualización por solicitud al servidor incurre en una penalización computacional.

La API Bulk() viene al rescate y mejora en gran medida el rendimiento, ya que las operaciones de escritura se envían al servidor solo una vez de forma masiva. La eficiencia se logra ya que el método no envía cada solicitud de escritura al servidor (como con la declaración de actualización actual dentro del bucle forEach()), sino solo una de cada 1000 solicitudes, lo que hace que actualizaciones más eficientes y rápidas de lo que es actualmente.

Usando el mismo concepto anterior con el bucle forEach() para crear los lotes, podemos actualizar la colección de forma masiva de la siguiente manera. En esta demostración, la API Bulk() disponible en las versiones >= 2.6 y < 3.2 de MongoDB utiliza el método initializeUnorderedBulkOp() para ejecutar en paralelo, así como en un orden no determinista, las operaciones de escritura en los lotes.

Actualiza todos los documentos en la colección de clientes cambiando los campos salary y dob a valores numéricos y datetime respectivamente:

var bulk = db.test.initializeUnorderedBulkOp(),
    counter = 0; // counter to keep track of the batch update size

db.test.find({
    "salary": { "$exists": true, "$type": 2 },
    "dob": { "$exists": true, "$type": 2 }
}).snapshot().forEach(function(doc){ 
    var newSalary = parseInt(doc.salary),
        newDob = new ISODate(doc.dob);
    bulk.find({ "_id": doc._id }).updateOne({ 
        "$set": { "salary": newSalary, "dob": newDob }
    });

    counter++; // increment counter
    if (counter % 1000 == 0) {
        bulk.execute(); // Execute per 1000 operations and re-initialize every 1000 update statements
        bulk = db.test.initializeUnorderedBulkOp();
    }
});

El siguiente ejemplo se aplica a la nueva versión 3.2 de MongoDB, que desde entonces ha dejado en desuso la API Bulk() y proporcionó un nuevo conjunto de API usando bulkWrite().

Utiliza los mismos cursores que el anterior, pero crea las matrices con las operaciones masivas utilizando el mismo método de cursor forEach() para enviar cada documento de escritura masiva a la matriz. Debido a que los comandos de escritura no pueden aceptar más de 1000 operaciones, es necesario agrupar operaciones para tener como máximo 1000 operaciones y reinicializar la matriz cuando el ciclo llega a la iteración 1000:

var cursor = db.test.find({
        "salary": { "$exists": true, "$type": 2 },
        "dob": { "$exists": true, "$type": 2 }
    }),
    bulkUpdateOps = [];

cursor.snapshot().forEach(function(doc){ 
    var newSalary = parseInt(doc.salary),
        newDob = new ISODate(doc.dob);
    bulkUpdateOps.push({ 
        "updateOne": {
            "filter": { "_id": doc._id },
            "update": { "$set": { "salary": newSalary, "dob": newDob } }
         }
    });

    if (bulkUpdateOps.length === 1000) {
        db.test.bulkWrite(bulkUpdateOps);
        bulkUpdateOps = [];
    }
});         

if (bulkUpdateOps.length > 0) { db.test.bulkWrite(bulkUpdateOps); }
