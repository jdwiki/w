Declarando #

La creación de variables en VBScript se puede realizar mediante la instrucción Dim, Public o Private. Es una buena práctica poner en la parte superior de la secuencia de comandos “Opción explícita”, lo que le obliga a definir explícitamente una variable.

Puedes declarar una variable como esta:

Option Explicit
Dim firstName

O puede varias variables como esta:

Option Explicit
Dim firstName, middleName, lastName

Si no tiene la opción declaración explícita, puede crear variables así:

firstName="Foo"

Esto NO se recomienda ya que pueden ocurrir resultados extraños durante la fase de tiempo de ejecución de su secuencia de comandos. Esto sucede si se comete un error tipográfico más tarde al reutilizar la variable.

Para crear una matriz, simplemente declárela con cuántos elementos en el parámetro:

Option Explicit
Dim nameList(2)

Esto crea una matriz con tres elementos.

Para configurar los elementos de la matriz, simplemente use la variable con el índice como parámetro de la siguiente manera:

nameList(0) = "John"

VBScript también admite matrices multidimensionales:

Option Explicit
Dim gridFactors(2, 4)

Tipos de variables #

VBScript es un lenguaje débilmente tipificado; todas las variables son de tipo variante, aunque normalmente tienen un [subtipo] implícito (https:/ /msdn.microsoft.com/en-us/library/9e7a57cf(v=vs.84).aspx) que indica los datos que contienen.

Esto significa que su variable, sin importar cómo la llame, puede contener cualquier valor:

Dim foo
foo = "Hello, World!"
foo = 123.45
foo = #01-Jan-2016 01:00:00#
foo = True

Tenga en cuenta que el código anterior es perfectamente válido, aunque mezclar sus variables de esta manera es una práctica increíblemente mala.

El subtipo de cadena siempre se asigna mediante marcas de voz " ". A diferencia de JavaScript y otros lenguajes, el apóstrofe no brinda la misma funcionalidad.

Los números en VBScript pueden incluir cualquier formato de número, pero tienen un subtipo particular basado en su valor y si contienen un punto decimal o no.

Las fechas usan los especificadores # #. Tenga en cuenta que los formatos para un estilo de fecha numérico (por ejemplo, 01/01/2016) conservan un formato de fecha estadounidense, por lo que #05/06/2016# es el 6 de mayo, no el 5 de junio. Esto se puede circunnavegar usando un formato #dd-mmm-yyyy#, como en el ejemplo anterior.

Las variables booleanas contienen valores Verdadero o Falso.

Como se explicó anteriormente, las matrices se dimensionan mediante un conjunto de paréntesis para definir el número de elementos y rangos (filas y columnas), por ejemplo:

Dim myArray(3, 4)

Todos los elementos de las matrices son de tipo variante, lo que permite que cada elemento sea de cualquier subtipo. Esto es muy importante cuando necesita realizar tareas como leer datos de un conjunto de registros u otro objeto. En estos casos, los datos se pueden asignar directamente a una variable, por ejemplo, cuando se devuelven de un conjunto de registros…

Dim myData
....
myData = rsMyRecordset.GetRows()
....
Response.Write(myData(3,2))

Un último tipo que requiere alguna explicación es el tipo Objeto. Los objetos son básicamente punteros a la ubicación de memoria del objeto mismo. Los tipos de objeto deben ser Set…

Dim myObj
Set myObj = Server.CreateObject("ADODB.ecordSet")
