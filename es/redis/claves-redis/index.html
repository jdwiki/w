El espacio de claves de Redis se puede considerar como una tabla hash o un diccionario que asigna claves a estructuras de datos en la base de datos.

Redis proporciona una amplia gama de comandos que funcionan con claves para administrar el espacio de claves, incluida la capacidad de eliminar claves, inspeccionar metadatos de claves, buscar claves y modificar ciertas propiedades de las claves.

Sintaxis #

  • patrón de LLAVES
  • Tecla PERSISTIR
  • EXPIRE segundos clave
  • Marca de tiempo de la clave EXPIREAT
  • Clave TTL
  • Tecla PEXPIRE milisegundos
  • Clave PEXPIREAT milisegundos-marca de tiempo
  • Tecla PTTL
  • tecla DESVINCULAR [tecla…]
  • Tecla DEL [tecla …]
  • Cursor SCAN [Patrón COINCIDIR] [Cuenta COUNT]

Para caracteres válidos en claves de Redis, el manual lo explica completamente:

Las claves Redis son binarias seguras, lo que significa que puede usar cualquier secuencia binaria como clave, desde una cadena como “foo” hasta el contenido de un archivo JPEG. La cadena vacía también es una clave válida.

Algunas otras reglas sobre las llaves:

Las claves muy largas no son una buena idea, por ejemplo, una clave de 1024 bytes es una mala idea no solo desde el punto de vista de la memoria, sino también porque la búsqueda de la clave en el conjunto de datos puede requerir varias comparaciones de claves costosas. Incluso cuando la tarea en cuestión es hacer coincidir la existencia de un valor grande, recurrir al hash (por ejemplo, con SHA1) es una mejor idea, especialmente desde el punto de vista de la memoria y el ancho de banda.

Las teclas muy cortas a menudo no son una buena idea. No tiene mucho sentido escribir “u1000flw” como clave si en su lugar puede escribir “usuario: 1000: seguidores”. Este último es más legible y el espacio agregado es menor en comparación con el espacio utilizado por el propio objeto clave y el objeto de valor. Si bien las teclas cortas obviamente consumirán un poco menos de memoria, su trabajo es encontrar el equilibrio adecuado.

Trate de ceñirse a un esquema. Por ejemplo, “tipo-objeto: id” es una buena idea, como en “usuario: 1000”. Los puntos o guiones se utilizan a menudo para campos de varias palabras, como en “comentario🔢reply.to” o “comentario🔢reply-to”.

El tamaño de clave máximo permitido es de 512 MB.

Tenga cuidado al usar el comando KEYS contra un sistema de producción, puede causar serios problemas de rendimiento. Si necesita realizar una búsqueda en el espacio de teclas, los comandos [ESCANEAR] (https://redis.io/commands/scan) son una mejor alternativa.

Claves válidas #

Las claves Redis son binarias seguras, por lo que, literalmente, se puede usar cualquier cosa como clave. Las únicas limitaciones son que deben tener menos de 512 MB.

Ejemplos de claves válidas:

7
++++
`~!@#$%^&*()-_=+
user:10134
search/9947372/?query=this%20is%20a%28test%29%20query
<div id="div64">

Any other string less than 512MB in size.
The raw binary content of an image or other binary file.
An entire multi-line text document.
An entire SQL query.
Any integer, hexadecimal, octal, or binary value.
Anything else you can think of less than 512MB in size.

Claves de Redis no válidas:

Anything larger than 512MB.

Esquemas de nomenclatura clave #

Para mayor claridad y facilidad de mantenimiento, a menudo se recomienda desarrollar un sistema o esquema para nombrar las claves de Redis. Estos son algunos ejemplos de sistemas comunes y fáciles de mantener para nombrar sus claves:

user:10134
user:10134:favorites
user:10134:friends
user:10134:friends-of-friends

user:10134
user:10134/favorites
user:10134/friends
user:10134/friends.of.friends

user/10134
user/10134/favorites
user/10134/friends
user/10134/friends of friends

Tenga en cuenta que, si bien está permitido, las claves más grandes usan más memoria y dan como resultado tiempos de búsqueda más lentos, por lo que usar una clave de 500 MB podría no ser una buena idea para el rendimiento. Una mejor idea podría ser usar un hash SHA-1, SHA-256 o MD5 de un objeto binario grande como clave en su lugar:

Listado de todas las claves #

Puede enumerar todas las claves en una base de datos de Redis ejecutando los siguientes comandos desde redis-cli:

KEYS *

El parámetro de KEYS es una expresión de coincidencia de patrones de estilo global. Los ejemplos de patrones admitidos incluyen:

h?llo matches hello, hallo and hxllo
h*llo matches hllo and heeeello
h[ae]llo matches hello and hallo, but not hillo
h[^e]llo matches hallo, hbllo, ... but not hello
h[a-b]llo matches hallo and hbllo

El uso del comando KEYS * puede tener efectos adversos en el rendimiento, por lo que no se recomienda para instancias de producción. Utilice la operación ESCANEAR para buscar claves en el código de producción.

TTL y caducidad de clave #

Los valores de vencimiento de una clave pueden ser administrados por un usuario fuera de los comandos de actualización. Redis permite a un usuario determinar el tiempo de vida actual (TTL) de una clave usando el comando TTL:

TTL key

Este comando devolverá el TTL de una clave en segundos o devolverá los valores especiales -1 o -2. Un -1 indica que la clave es persistente (no caducará) y un -2 indica que la clave no existe.

Una clave que expira puede volverse persistente usando el comando PERSIST:

PERSIST KEY

y se puede hacer que una clave persistente caduque usando el comando EXPIRE:

EXPIRE KEY seconds

Expire también se puede usar para modificar el TTL de una clave existente. Como alternativa, puede utilizar el comando EXPIREAT con una marca de tiempo UNIX para establecer una fecha de caducidad.

Hay versiones de milisegundos de los comandos TTL, EXPIRE y EXPIREAT que tienen el prefijo P.

Eliminación de claves #

Redis proporciona dos funciones para eliminar claves de la base de datos: eliminar y desvincular.

La función del elimina una o más claves de la base de datos. El comando del hace que Redis recupere inmediatamente la memoria de la clave eliminada en el hilo de ejecución actual. El tiempo de ejecución de del es proporcional al número de elementos individuales eliminados de todas las claves.

La función de desvincular actúa como el comando del, elimina una o más claves de la base de datos. Sin embargo, a diferencia del comando del, cualquier memoria utilizada por esas teclas se recupera de forma asincrónica en otro subproceso.

Escaneando el espacio de claves de Redis #

Redis proporciona el comando SCAN para iterar sobre las claves en la base de datos que coinciden con un patrón particular. Redis admite la coincidencia de patrones de estilo global en el comando ESCANEAR.

El comando ESCANEAR proporciona un iterador basado en cursor sobre el espacio de claves de Redis. La secuencia iterativa de llamadas a SCAN comienza cuando el usuario realiza una llamada con el argumento del cursor establecido en 0. El resultado de esa llamada es un lote de elementos y un cursor actualizado que se proporciona a la siguiente llamada a SCAN. Esta iteración continúa hasta que Redis devuelve un cursor 0.

La siguiente función de Python demuestra el uso básico de SCAN:

def scan_keys(r, pattern):
    "Returns a list of all the keys matching a given pattern"

    result = []
    cur, keys  = r.scan(cursor=0, match=pattern, count=2)
    result.extend(keys)
    while cur != 0:
        cur, keys = r.scan(cursor=cur, match=pattern, count=2)
        result.extend(keys)
        
    return result

El comando SCAN es la forma recomendada de buscar claves en la base de datos, y se recomienda sobre el comando KEYS *.
