Un segmento es una estructura de datos que encapsula una matriz para que el programador pueda agregar tantos elementos como necesite sin tener que preocuparse por la administración de la memoria. Los sectores se pueden dividir en subsegmentos de manera muy eficiente, ya que todos los sectores resultantes apuntan a la misma matriz interna. Los programadores de Go a menudo aprovechan esto para evitar copiar matrices, lo que normalmente se haría en muchos otros lenguajes de programación.

Sintaxis #

  • slice := make([]type, len, cap) // crea un nuevo slice
  • slice = append(segment, item) // añade un elemento a un slice
  • slice = append(segmento, elementos…) // agregar segmento de elementos a un segmento
  • len := len(segmento) // obtener la longitud de un segmento
  • cap := cap(segmento) // obtener la capacidad de un segmento
  • elNum := copy(dst, slice) // copia el contenido de un segmento a otro segmento

Agregar dos rebanadas juntas #

slice1 := []string{"!"}
slice2 := []string{"Hello", "world"}
slice  := append(slice1, slice2...)

Longitud y Capacidad #

Las rebanadas tienen tanto longitud como capacidad. La longitud de un segmento es el número de elementos actualmente en el segmento, mientras que la capacidad es el número de elementos que el segmento puede contener antes de que sea necesario reasignarlo.

Al crear un segmento utilizando la función integrada make(), puede especificar su longitud y, opcionalmente, su capacidad. Si la capacidad no se especifica explícitamente, será la longitud especificada.

var s = make([]int, 3, 5) // length 3, capacity 5

Puedes comprobar la longitud de un segmento con la función integrada len():

var n = len(s) // n == 3

Puede comprobar la capacidad con la función cap() integrada:

var c = cap(s) // c == 5

Los elementos creados por make() se establecen en el valor cero para el tipo de elemento del segmento:

for idx, val := range s {
    fmt.Println(idx, val)
}
// output:
// 0 0
// 1 0
// 2 0

Ejecutarlo en play.golang.org

No puede acceder a elementos más allá de la longitud de un segmento, incluso si el índice está dentro de su capacidad:

var x = s[3] // panic: runtime error: index out of range

Sin embargo, siempre que la capacidad exceda la longitud, puede agregar nuevos elementos sin reasignar:

var t = []int{3, 4}
s = append(s, t) // s is now []int{0, 0, 0, 3, 4}
n = len(s) // n == 5
c = cap(s) // c == 5

Si agrega a un segmento que carece de la capacidad para aceptar los nuevos elementos, la matriz subyacente se reasignará con capacidad suficiente:

var u = []int{5, 6}
s = append(s, u) // s is now []int{0, 0, 0, 3, 4, 5, 6}
n = len(s) // n == 7
c = cap(s) // c > 5

Por lo tanto, generalmente es una buena práctica asignar capacidad suficiente cuando se crea una porción por primera vez, si sabe cuánto espacio necesitará, para evitar reasignaciones innecesarias.

Eliminar elementos / “Cortar” rebanadas #

Si necesita eliminar uno o más elementos de un segmento, o si necesita trabajar con un sub segmento de otro existente; puede utilizar el siguiente método.

Los siguientes ejemplos usan slice of int, pero eso funciona con todo tipo de slice.

Entonces, para eso, necesitamos una porción, de la cual eliminaremos algunos elementos:

slice := []int{1, 2, 3, 4, 5, 6} 
// > [1 2 3 4 5 6]

Necesitamos también los índices de los elementos a eliminar:

// index of first element to remove (corresponding to the '3' in the slice)
var first = 2

// index of last element to remove (corresponding to the '5' in the slice)
var last = 4

Y así podemos “cortar” la rebanada, eliminando elementos no deseados:

// keeping elements from start to 'first element to remove' (not keeping first to remove), 
// removing elements from 'first element to remove' to 'last element to remove'
// and keeping all others elements to the end of the slice
newSlice1 := append(slice[:first], slice[last+1:]...)
// > [1 2 6]

// you can do using directly numbers instead of variables
newSlice2 := append(slice[:2], slice[5:]...)
// > [1 2 6]

// Another way to do the same
newSlice3 := slice[:first + copy(slice[first:], slice[last+1:])]
// > [1 2 6]

// same that newSlice3 with hard coded indexes (without use of variables)
newSlice4 := slice[:2 + copy(slice[2:], slice[5:])]
// > [1 2 6]

Para eliminar solo un elemento, solo hay que poner el índice de este elemento como el primero Y como el último índice a eliminar, así:

var indexToRemove = 3
newSlice5 := append(slice[:indexToRemove], slice[indexToRemove+1:]...)
// > [1 2 3 5 6]

// hard-coded version:
newSlice5 := append(slice[:3], slice[4:]...)
// > [1 2 3 5 6]

Y también puede eliminar elementos desde el principio de la división:

newSlice6 := append(slice[:0], slice[last+1:]...)
// > [6]

// That can be simplified into
newSlice6 := slice[last+1:]
// > [6]

También puede eliminar algunos elementos del final de la división:

newSlice7 := append(slice[:first], slice[first+1:len(slice)-1]...)
// > [1 2]

// That can be simplified into
newSlice7 := slice[:first]
// > [1 2]

Si el nuevo corte tiene que contener exactamente los mismos elementos que el primero, puedes usar lo mismo pero con last := first-1. (Esto puede ser útil en caso de que sus índices se hayan calculado previamente)

Creando rebanadas #

Las rebanadas son la forma típica en que los programadores almacenan listas de datos.

Para declarar una variable de segmento, utilice la sintaxis []Type.

var a []int

Para declarar e inicializar una variable de segmento en una línea, use la sintaxis []Type{values}.

var a []int = []int{3, 1, 4, 1, 5, 9}

Otra forma de inicializar un segmento es con la función make. Tiene tres argumentos: el ‘Tipo’ del segmento (o mapa), la ’longitud’ y la ‘capacidad’.

a := make([]int, 0, 5)

Puede agregar elementos a su nueva rebanada usando append.

a = append(a, 5)

Verifique la cantidad de elementos en su segmento usando len.

length := len(a)

Comprueba la capacidad de tu rebanada usando cap. La capacidad es el número de elementos asignados actualmente para estar en la memoria del segmento. Siempre puede agregar a un segmento a su máxima capacidad, ya que Go creará automáticamente un segmento más grande para usted.

capacity := cap(a)

Puede acceder a los elementos de un segmento utilizando la sintaxis de indexación típica.

a[0]  // Gets the first member of `a`

También puede usar un bucle for sobre segmentos con range. La primera variable es el índice en la matriz especificada y la segunda variable es el valor del índice.

for index, value := range a {
    fmt.Println("Index: " + index + " Value: " + value)  // Prints "Index: 0 Value: 5" (and continues until end of slice)
}

Filtrando una rebanada #

Para filtrar un segmento sin asignar una nueva matriz subyacente:

// Our base slice
slice := []int{ 1, 2, 3, 4 }
// Create a zero-length slice with the same underlying array
tmp := slice[:0]

for _, v := range slice {
  if v % 2 == 0 {
    // Append desired values to slice
    tmp = append(tmp, v)
  }
}

// (Optional) Reassign the slice
slice = tmp // [2, 4]

Valor cero de la rebanada #

El valor cero de slice es nil, que tiene la longitud y la capacidad 0. Un segmento nil no tiene una matriz subyacente. Pero también hay segmentos no nulos de longitud y capacidad 0, como []int{} o make([]int, 5)[5:].

Cualquier tipo que tenga valores nulos se puede convertir en segmento nil:

s = []int(nil)

Para probar si un segmento está vacío, use:

if len(s) == 0　{
    fmt.Ptintf("s is empty.")
}

Agregando a la rebanada #

slice = append(slice, "hello", "world")

Copiar contenidos de un segmento a otro segmento #

Si desea copiar el contenido de un segmento en un segmento inicialmente vacío, se pueden seguir los siguientes pasos para lograrlo:

  1. Cree el segmento de origen:

    var sourceSlice []interface{} = []interface{}{“Hello”,5.10,“World”,true}

  2. Crea el segmento de destino, con:

  • Longitud = Longitud de sourceSlice

    var destinationSlice []interface{} = make([]interface{},len(sourceSlice))

  1. Ahora que la matriz subyacente del segmento de destino es lo suficientemente grande como para acomodar todos los elementos del segmento de origen, podemos proceder a copiar los elementos usando la copia incorporada:

    copy(destinationSlice,sourceSlice)
