Conecte ADB a un dispositivo a través de WiFi #

La configuración estándar de ADB implica una conexión USB a un dispositivo físico. Si lo prefiere, puede cambiar al modo TCP/IP y conectar ADB a través de WiFi.

Dispositivo no rooteado #

  1. Conéctese a la misma red:
  • Asegúrese de que su dispositivo y su computadora estén en la misma red.
  1. Conecte el dispositivo a la computadora host con un cable USB.
  2. Conecte adb al dispositivo a través de la red:

Mientras su dispositivo está conectado a adb a través de USB, ejecute el siguiente comando para escuchar una conexión TCP/IP en un puerto (predeterminado 5555):

  • Escriba adb tcpip <port> (cambie al modo TCP/IP).
  • Desconecte el cable USB del dispositivo de destino.
  • Escriba adb connect <dirección IP>:<puerto> (el puerto es opcional; el valor predeterminado es 5555).

Por ejemplo:

   adb tcpip 5555
   adb connect 192.168.0.101:5555

Si no conoce la IP de su dispositivo, puede:

  • verifique la IP en la configuración WiFi de su dispositivo.
  • use ADB para descubrir IP (a través de USB):
    1. Connect the device to the computer via USB
    2. In a command line, type adb shell ifconfig and copy your device’s IP address

Para revertir a la depuración a través de USB, use el siguiente comando:

   adb usb

También puede conectar ADB a través de WiFi instalando un complemento en Android Studio. Para hacerlo, vaya a Configuración > Complementos y Examinar repositorios, busque ADB WiFi, instálelo y vuelva a abrir Android Studio. Verá un nuevo ícono en su barra de herramientas como se muestra en la siguiente imagen. Conecte el dispositivo a la computadora host a través de USB y haga clic en este ícono AndroidWiFiADB. Mostrará un mensaje si su dispositivo está conectado o no. Una vez que se conecta, puede desconectar su USB.

Dispositivo rooteado #

Nota: Algunos dispositivos que están rooteados pueden usar la aplicación ADB WiFi de Play Store para habilitar esto de una manera simple. Además, para ciertos dispositivos (especialmente aquellos con ROM CyanogenMod) esta opción está presente en las Opciones de desarrollador entre las Configuraciones. Habilitarlo le dará la dirección IP y el número de puerto necesarios para conectarse a adb simplemente ejecutando adb connect <dirección IP>:<puerto>.

Cuando tiene un dispositivo rooteado pero no tiene acceso a un cable USB #

El proceso se explica en detalle en la siguiente respuesta: http://stackoverflow.com/questions/2604727/how-can-i-connect-to-android-with-adb-over-tcp/3623727#3623727 Los comandos más importantes se muestran a continuación.

Abra una terminal en el dispositivo y escriba lo siguiente:

su
setprop service.adb.tcp.port <a tcp port number>
stop adbd
start adbd

Por ejemplo:

setprop service.adb.tcp.port 5555

Y en tu computadora:

adb connect <ip address>:<a tcp port number>

Por ejemplo:

adb connect 192.168.1.2:5555

Para apagarlo:

setprop service.adb.tcp.port -1
stop adbd
start adbd

Evite el tiempo de espera #

Por defecto, adb expirará después de 5000 ms. Esto puede suceder en algunos casos, como WiFi lento o APK grande.

Un simple cambio en la configuración de Gradle puede hacer el truco:

android {
    adbOptions {
        timeOutInMs 10 * 1000
    }
}

Comando ADB directo a un dispositivo específico en una configuración de múltiples dispositivos #

1. Segmentar un dispositivo por número de serie

Utilice la opción -s seguida de un nombre de dispositivo para seleccionar en qué dispositivo debe ejecutarse el comando adb. Las opciones -s deben estar al principio de la línea, antes del comando.

adb -s <device> <command>

Ejemplo:

adb devices

List of devices attached
emulator-5554       device
02157df2d1faeb33    device

adb -s emulator-5554 shell

Ejemplo #2:

adb devices -l

List of devices attached
06157df65c6b2633    device usb:1-3 product:zerofltexx model:SM_G920F device:zeroflte
LC62TB413962        device usb:1-5 product:a50mgp_dug_htc_emea model:HTC_Desire_820G_dual_sim device:htc_a50mgp_dug

adb -s usb:1-3 shell

2. Apunta a un dispositivo, cuando solo un tipo de dispositivo está conectado

Puede apuntar al único emulador en ejecución con -e

adb -e <command>

O puede apuntar al único dispositivo USB conectado con -d

adb -d <command>

Tomar una captura de pantalla y un video (solo para kitkat) desde la pantalla de un dispositivo #

Captura de pantalla: Opción 1 (adb puro) #

El comando shell adb nos permite ejecutar comandos usando el shell integrado de un dispositivo. El comando de shell screencap captura el contenido actualmente visible en un dispositivo y lo guarda en un archivo de imagen determinado, p. /tarjetasd/pantalla.png:

adb shell screencap /sdcard/screen.png

Luego puede usar [el comando pull][pull] para descargar el archivo desde el dispositivo al directorio actual en su computadora:

adb pull /sdcard/screen.png

[tirar]: https://www.wikiod.com/es/android/adb-puente-de-depuracion-de-android#Tire (empuje) archivos desde (hacia) el dispositivo “tirar”

Captura de pantalla: Opción 2 (más rápido) #

Ejecute la siguiente línea:

(Marshmallow y anteriores):

adb shell screencap -p | perl -pe 's/\x0D\x0A/\x0A/g' > screen.png

(Turrones y posteriores):

adb shell screencap -p > screen.png

El indicador -p redirige la salida del comando screencap a stdout. La expresión de Perl en la que se canaliza soluciona algunos problemas de final de línea en Marshmallow y versiones anteriores. Luego, la transmisión se escribe en un archivo llamado screen.png dentro del directorio actual. Consulte este artículo y [este artículo](http://www.stkent.com/ 2016/08/28/capturing-Nougat-screenshots-using-adb-shell.html) para obtener más información.

Video #

esto solo funciona en KitKat y solo a través de ADB. Esto no funciona debajo de Kitkat Para comenzar a grabar la pantalla de su dispositivo, ejecute el siguiente comando:

adb ​​shell screenrecord /sdcard/example.mp4, este comando comenzará a grabar la pantalla de su dispositivo usando la configuración predeterminada y guardará el video resultante en un archivo en /sdcard/example.mp4 en su dispositivo.

Cuando haya terminado de grabar, presione Ctrl+C (z en Linux) en la ventana del símbolo del sistema para detener la grabación de la pantalla. Luego puede encontrar el archivo de grabación de pantalla en la ubicación que especificó. Tenga en cuenta que la grabación de pantalla se guarda en el almacenamiento interno de su dispositivo, no en su computadora.

La configuración predeterminada es usar la resolución de pantalla estándar de su dispositivo, codificar el video a una tasa de bits de 4 Mbps y establecer el tiempo máximo de grabación de pantalla en 180 segundos. Para obtener más información sobre las opciones de la línea de comandos que puede usar, ejecute el siguiente comando:

adb ​​shell screenrecord –help, esto funciona sin rootear el dispositivo. Espero que esto ayude.

Extraer (empujar) archivos desde (hacia) el dispositivo #

Puede extraer (descargar) archivos del dispositivo ejecutando el siguiente comando:

adb pull <remote> <local>

Por ejemplo:

adb pull /sdcard/ ~/

También puede enviar (cargar) archivos desde su computadora al dispositivo:

adb push <local> <remote>

Por ejemplo:

adb push ~/image.jpg /sdcard/

Ejemplo para recuperar la base de datos del dispositivo

sudo adb -d shell "run-as com.example.name cat /data/da/com.example.name /databases/DATABASE_NAME  > /sdcard/file

Ver gato de registro #

Puede ejecutar logcat como un comando adb o directamente en un indicador de shell de su emulador o dispositivo conectado. Para ver la salida del registro usando adb, navegue a su directorio SDK platform-tools/ y ejecute:

$ adb logcat

Alternativamente, puede crear una conexión de shell a un dispositivo y luego ejecutar:

$ adb shell
$ logcat

Un comando útil es:

adb logcat -v threadtime

Esto muestra la fecha, la hora de invocación, la prioridad, la etiqueta y el PID y TID del hilo que emite el mensaje en un formato de mensaje largo.

Filtración

Los registros de Logcat tienen los llamados niveles de registro:

V: detallado, D: depuración, I: información, W: advertencia, E: error, F: fatal, * S* — Silencioso

También puede filtrar logcat por nivel de registro. Por ejemplo, si solo desea generar el nivel de depuración:

adb logcat *:D

Logcat se puede filtrar por un nombre de paquete, por supuesto, puede combinarlo con el filtro de nivel de registro:

adb logcat <package-name>:<log level>

También puede filtrar el registro usando grep (más información sobre cómo filtrar la salida de logcat aquí):

adb logcat | grep <some text>

En Windows, el filtro se puede usar usando findstr, por ejemplo:

adb logcat | findstr <some text>

Para ver el búfer de registro alternativo [main|events|radio], ejecute logcat con la opción -b:

adb logcat -b radio

Guardar la salida en el archivo:

adb logcat > logcat.txt

Guarde la salida en el archivo mientras la mira:

adb logcat | tee logcat.txt

Limpieza de los registros:

adb logcat -c

1: https://www.wikiod.com/es/android/primeros-pasos-con-android#Filtrado de la salida logcat

Borrar datos de la aplicación #

Uno puede borrar los datos de usuario de una aplicación específica usando adb:

adb shell pm clear <package>

Esto es lo mismo que navegar por la configuración del teléfono, seleccionar la aplicación y presionar el botón borrar datos.

  • pm invoca el administrador de paquetes en el dispositivo
  • clear elimina todos los datos asociados con un paquete

Ver y extraer archivos de caché de una aplicación #

Puede usar este comando para enumerar los archivos de su propia aplicación depurable:

adb shell run-as <sample.package.id> ls /data/data/sample.package.id/cache

Y esta secuencia de comandos para extraer del caché, primero copia el contenido a la tarjeta SD, extrae y luego elimina al final:

#!/bin/sh
adb shell "run-as <sample.package.id> cat '/data/data/<sample.package.id>/$1' > '/sdcard/$1'"
adb pull "/sdcard/$1"
adb shell "rm '/sdcard/$1'"

Luego puede extraer un archivo del caché como este:

./pull.sh cache/someCachedData.txt

Obtener archivo de base de datos a través de ADB

sudo adb -d shell "run-as com.example.name cat /data/da/com.example.name /databases/STUDENT_DATABASE  > /sdcard/file

Ver dispositivos disponibles #

Conectar dispositivo por IP #

Ingrese estos comandos en el dispositivo Android Terminal

su
setprop service.adb.tcp.port 5555
stop adbd
start adbd

Después de esto, puede usar CMD y ADB para conectarse usando el siguiente comando

adb connect 192.168.0.101.5555

Y puede desactivarlo y hacer que ADB vuelva a escuchar en USB con

setprop service.adb.tcp.port -1
stop adbd
start adbd

Desde una computadora, si ya tiene acceso USB (no se requiere root)

Es aún más fácil cambiar al uso de Wi-Fi, si ya tiene USB. Desde una línea de comando en la computadora que tiene el dispositivo conectado a través de USB, emita los comandos

adb tcpip 5555
adb connect 192.168.0.101:5555

Reemplace 192.168.0.101 con la IP del dispositivo

Enviando transmisión #

Es posible enviar transmisiones a BroadcastReceiver con adb.

En este ejemplo, estamos enviando una transmisión con la acción com.test.app.ACTION y una cadena extra en el paquete 'foo'='bar':

adb shell am broadcast -a action com.test.app.ACTION --es foo "bar"

Puede poner cualquier otro tipo admitido para agrupar, no solo cadenas:

–ez  - booleano
–ei  - entero
–el  - largo
–ef  - flotar
–eu  - uri
–eia - matriz int (separada por ‘,’)
–ela - matriz larga (separada por ‘,’)
–efa - matriz flotante (separada por ‘,’)
–esa - matriz de cadenas (separadas por ‘,’)

Para enviar la intención a un paquete/clase específico, se puede usar el parámetro -n o -p.
Envío a paquete:

-p com.test.app

Envío a un componente específico (clase SomeReceiver en el paquete com.test.app):

-n com.test.app/.SomeReceiver

Ejemplos útiles:

  • Enviando una transmisión de “arranque completo”
  • [Enviar una transmisión de “hora cambiada” después de configurar la hora a través del comando adb][2]

Instalar y ejecutar una aplicación #

Para instalar un archivo APK, usa el siguiente comando:

adb install path/to/apk/file.apk

o si la aplicación ya existe y queremos reinstalarla

adb install -r path/to/apk/file.apk

Para desinstalar una aplicación, tenemos que especificar su paquete

adb uninstall application.package.name

Utilice el siguiente comando para iniciar una aplicación con un nombre de paquete proporcionado (o una actividad específica en una aplicación):

adb shell am start -n adb shell am start <package>/<activity>

Por ejemplo, para iniciar Waze:

adb shell am start -n adb shell am start com.waze/com.waze.FreeMapAppActivity

Respaldo #

Puede usar el comando adb backup para hacer una copia de seguridad de su dispositivo.

adb backup [-f <file>] [-apk|-noapk] [-obb|-noobb] [-shared|-noshared] [-all] 
           [-system|nosystem] [<packages...>]

-f <nombre de archivo> especificar nombre de archivo predeterminado: crea backup.ab en el directorio actual

-apk|noapk habilitar/deshabilitar la copia de seguridad de los propios .apks predeterminado: -noapk

-obb|noobb habilitar/deshabilitar la copia de seguridad de archivos adicionales predeterminado: -noobb

Contenido de la tarjeta SD/almacenamiento compartido del dispositivo de copia de seguridad -shared|noshared predeterminado: -noshared

-all copia de seguridad de todas las aplicaciones instaladas

-system|nosystem incluye aplicaciones del sistema predeterminado: -system

<paquetes> una lista de paquetes para respaldar (por ejemplo, com.example.android.myapp) (no es necesario si se especifica -all)

Para una copia de seguridad completa del dispositivo, incluido todo, use

adb backup -apk -obb -shared -all -system -f fullbackup.ab

Nota: Hacer una copia de seguridad completa puede llevar mucho tiempo.

Para restaurar una copia de seguridad, utilice

adb restore backup.ab

Ver los datos internos de una aplicación (data/data/<sample.package.id>) en un dispositivo #

Primero, asegúrese de que se pueda hacer una copia de seguridad de su aplicación en AndroidManifest.xml, es decir, android:allowBackup no es falso.

Comando de copia de seguridad:

adb -s <device_id> backup -noapk <sample.package.id>

Crea un tar con el comando dd:

dd if=backup.ab bs=1 skip=24 | python -c "import zlib,sys;sys.stdout.write(zlib.decompress(sys.stdin.read()))" > backup.tar

Extraiga el alquitrán:

tar -xvf backup.tar

A continuación, puede ver el contenido extraído.

Instalar ADB en el sistema Linux #

Cómo instalar el Puente de depuración de Android (ADB) en un sistema Linux con el terminal usando los repositorios de su distribución.

Instale en el sistema Ubuntu/Debian a través de apt:

sudo apt-get update
sudo apt-get install adb

Instale en el sistema Fedora/CentOS a través de yum:

sudo yum check-update
sudo yum install android-tools

Instalar en el sistema Gentoo con portage:

sudo emerge --ask dev-util/android-tools

Instalar en el sistema openSUSE con zypper:

sudo zypper refresh
sudo zypper install android-tools

Instalar en el sistema Arch con pacman:

sudo pacman -Syyu
sudo pacman -S android-tools

Leer información del dispositivo #

Escribe el siguiente comando en tu terminal:

adb shell getprop

Esto imprimirá toda la información disponible en forma de pares clave/valor.

Simplemente puede leer información específica agregando el nombre de una tecla específica al comando. Por ejemplo:

adb shell getprop ro.product.model

Aquí hay algunos datos interesantes que tu gato obtiene:

  • ro.product.model: Nombre del modelo del dispositivo (por ejemplo, Nexus 6P)
  • ro.build.version.sdk: nivel de API del dispositivo (por ejemplo, 23)
  • ro.product.brand: marca del dispositivo (por ejemplo, Samsung)

Reiniciar dispositivo #

Puede reiniciar su dispositivo ejecutando el siguiente comando:

adb reboot

Ejecute este comando para reiniciar en el gestor de arranque:

adb reboot bootloader

Reiniciar al modo de recuperación:

adb reboot recovery

¡Tenga en cuenta que el dispositivo no se apagará primero!
