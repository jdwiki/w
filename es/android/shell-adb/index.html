adb ​​shell abre un shell de Linux en un dispositivo o emulador de destino. Es la forma más potente y versátil de controlar un dispositivo Android a través de adb.

Este tema se separó de https://www.wikiod.com/es/android/adb-puente-de-depuracion-de-android debido a que alcanzó el límite de ejemplos, muchos de los cuales involucraban el comando adb shell.

Sintaxis #

  • adb shell [-e escape] [-n] [-Tt] [-x] [comando]

Parámetros #

Parámetro Detalles
-e elija el carácter de escape o “ninguno”; predeterminado ‘~’
-n no leer desde stdin
-T deshabilitar la asignación de PTY
-t forzar la asignación de PTY
-x desactivar códigos de salida remota y separación stdout/stderr

Concesión y revocación de permisos API 23+ #

Una línea que ayuda a otorgar o revocar permisos vulnerables.

  • concesión

     adb shell pm grant <sample.package.id> android.permission.<PERMISSION_NAME>

  • revocando

     adb shell pm revoke <sample.package.id> android.permission.<PERMISSION_NAME>

  • Concesión de todos los permisos de tiempo de ejecución a la vez en la instalación (-g)

     adb install -g /path/to/sample_package.apk

Enviar texto, presionar teclas y eventos táctiles al dispositivo Android a través de ADB #

ejecute el siguiente comando para insertar el texto en una vista con un foco (si es compatible con la entrada de texto)

Enviar texto en SDK 23+

adb shell "input keyboard text 'Paste text on Android Device'"

Si ya está conectado a su dispositivo a través de adb:

input text 'Paste text on Android Device'

Enviar texto antes de SDK 23

adb shell "input keyboard text 'Paste%stext%son%sAndroid%sDevice'"

Los espacios no se aceptan como entrada, reemplácelos con %s.

Enviar eventos

Para simular presionar la tecla de encendido del hardware

adb shell input keyevent 26

o alternativamente

adb shell input keyevent POWER

Incluso si no tiene una clave de hardware, aún puede usar un keyevent para realizar la acción equivalente

adb shell input keyevent CAMERA

Enviar evento táctil como entrada

adb shell input tap Xpoint Ypoint

Enviar evento de deslizamiento como entrada

adb shell input swipe Xpoint1 Ypoint1 Xpoint2 Ypoint2 [DURATION*]

*DURACIÓN es opcional, por defecto=300ms. fuente

Obtenga puntos X e Y habilitando la ubicación del puntero en la opción de desarrollador.

Script de shell de muestra de ADB

Para ejecutar un script en Ubuntu, Create script.sh, haga clic derecho en el archivo y agregue permiso de lectura/escritura y marque permitir ejecutar el archivo como programa.

Abra el emulador de terminal y ejecute el comando ./script.sh

Guión.sh

 for (( c=1; c<=5; c++ ))
    do  
       adb shell input tap X Y
       echo "Clicked $c times"
       sleep 5s
    done

Para obtener una lista completa de números de eventos

  • lista corta de varios eventos interesantes http://stackoverflow.com/questions/7789826/adb-shell-input-events
  • documentación de referencia https://developer.android.com/reference/android/view/KeyEvent.html#KEYCODE_POWER.

Lista de paquetes #

Imprime todos los paquetes, opcionalmente solo aquellos cuyo nombre de paquete contiene el texto en <FILTRO>.

adb shell pm list packages [options] <FILTER>

All <FILTER>

adb shell pm list packages

Atributos:

-f para ver su archivo asociado.

-i Consulte el instalador de los paquetes.

-u para incluir también paquetes desinstalados.

-u También incluye paquetes desinstalados.

Atributos que filtran:

-d para paquetes deshabilitados.

-e para paquetes habilitados.

-s para paquetes del sistema.

-3 para paquetes de terceros.

--user <USER_ID> para consultar un espacio de usuario específico.

Grabando la pantalla #

Grabación de la visualización de dispositivos con Android 4.4 (nivel de API 19) y superior:

adb shell screenrecord [options] <filename>
adb shell screenrecord /sdcard/demo.mp4

(presione Ctrl-C para detener la grabación)

Descarga el archivo desde el dispositivo:

adb pull /sdcard/demo.mp4

Nota: detenga la grabación de pantalla presionando Ctrl-C, de lo contrario, la grabación se detiene automáticamente a los tres minutos o al límite de tiempo establecido por --time-limit.

adb shell screenrecord --size <WIDTHxHEIGHT>

Establece el tamaño del video: 1280x720. El valor predeterminado es la resolución de pantalla nativa del dispositivo (si es compatible), 1280x720 si no lo es. Para obtener los mejores resultados, utilice un tamaño compatible con el codificador de codificación de video avanzada (AVC) de su dispositivo.

adb shell screenrecord --bit-rate <RATE>

Establece la tasa de bits de video para el video, en megabits por segundo. El valor predeterminado es 4Mbps. Puede aumentar la tasa de bits para mejorar la calidad del video, pero al hacerlo, los archivos de película serán más grandes. El siguiente ejemplo establece la tasa de bits de grabación en 5 Mbps:

adb shell screenrecord --bit-rate 5000000 /sdcard/demo.mp4

adb shell screenrecord --time-limit <TIME>

Establece el tiempo máximo de grabación, en segundos. El valor predeterminado y máximo es 180 (3 minutos).

adb shell screenrecord --rotate

Gira la salida 90 grados. Esta función es experimental.

adb shell screenrecord --verbose

Muestra información de registro en la pantalla de línea de comandos. Si no establece esta opción, la utilidad no muestra ninguna información mientras se ejecuta.

Nota: Es posible que esto no funcione en algunos dispositivos.

El comando de grabación de pantalla no es compatible con las versiones de Android anteriores a la 4.4

El comando screenrecord es una utilidad de shell para grabar la visualización de dispositivos que ejecutan Android 4.4 (API nivel 19) y superior. La utilidad registra la actividad de la pantalla en un archivo MPEG-4.

Establecer fecha/hora a través de adb #

El formato SET predeterminado es MMDDhhmm[[CC]YY][.ss], eso es (2 dígitos cada uno)

Por ejemplo, para configurar el 17 de julio a las 10:10 am, sin cambiar el año actual, escriba:

adb shell 'date 07171010.00'

Consejo 1: el cambio de fecha no se reflejará de inmediato y solo ocurrirá un cambio notable después de que el reloj del sistema avance al siguiente minuto.
Puede forzar una actualización adjuntando una transmisión de intención TIME_SET a su llamada, así:

adb shell 'date 07171010.00 ; am broadcast -a android.intent.action.TIME_SET'

Sugerencia 2: para sincronizar el reloj de Android con su máquina local:

Linux:

adb shell date `date +%m%d%H%M%G.%S`

Windows (PowerShell):

$currentDate = Get-Date -Format "MMddHHmmyyyy.ss" # Android's preferred format
adb shell "date $currentDate"

Ambos consejos juntos:

adb shell 'date `date +%m%d%H%M%G.%S` ; am broadcast -a android.intent.action.TIME_SET'

El formato SET predeterminado es ‘AAAAMMDD.HHmmss’

adb shell 'date -s 20160117.095930'

Sugerencia: para sincronizar el reloj de Android con su máquina local (basada en Linux):

adb shell date -s `date +%G%m%d.%H%M%S`

Abrir opciones de desarrollador #

adb shell am start -n com.android.settings/.DevelopmentSettings

Navegará por su dispositivo/emulador a la sección Opciones de desarrollador.

  • permisos concedidos (Android API 23+)

  • etc..

    adb shell dumpsys package <your.package.id>

Generación de una transmisión de “arranque completo” #

Esto es relevante para aplicaciones que implementan un BootListener. Pruebe su aplicación matando su aplicación y luego pruebe con:

adb shell am broadcast -a android.intent.action.BOOT_COMPLETED -c android.intent.category.HOME -n your.app/your.app.BootListener

(reemplace your.package/your.app.BootListener con los valores adecuados).

Ver contenido de almacenamiento externo/secundario #

Ver contenido:

adb shell ls \$EXTERNAL_STORAGE
adb shell ls \$SECONDARY_STORAGE

Ver ruta:

adb shell echo \$EXTERNAL_STORAGE
adb shell echo \$SECONDARY_STORAGE

matar un proceso dentro de un dispositivo Android #

A veces, el logcat de Android se ejecuta infinitamente con errores que provienen de algún proceso que no es tuyo, agotando la batería o simplemente dificultando la depuración de tu código.

Una forma conveniente de solucionar el problema sin reiniciar el dispositivo es ubicar y eliminar el proceso que causa el problema.

Desde Logcat

03-10 11:41:40.010 1550-1627/? E/AlgúnProceso: ....

observe el número de proceso: 1550

Ahora podemos abrir un caparazón y matar el proceso. Tenga en cuenta que no podemos eliminar el proceso root.

adb shell

dentro del caparazón podemos verificar más sobre el proceso usando

ps -x | grep 1550

y matarlo si queremos:

kill -9 1550
