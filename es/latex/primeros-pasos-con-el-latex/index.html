Editores LaTeX #

Si bien puede crear documentos LaTeX usando cualquier editor y compilarlos usando la consola, existen varios complementos para editores ampliamente utilizados para simplificar la creación de documentos LaTeX, y hay editores LaTeX especializados. Una lista exhaustiva de editores de LaTeX está disponible en TeX.SE (el sitio de StackExchange, dedicado a TeX, LaTeX y sus amigos).

Los editores más utilizados, según esta lista, son:

  • El editor Emacs con la extensión AUCTeX.
  • El editor Vim con el complemento LaTeX-suite.
  • Texmaker: un IDE de LaTeX especializado.
  • TeXstudio – otro IDE de LaTeX.
  • TeXworks – un IDE de LaTeX más.

Si bien los usuarios experimentados de Emacs o Vim pueden querer apegarse a su editor (cuyos complementos brindan una gran cantidad de funciones que no están disponibles en otros lugares), un IDE especializado puede ser más fácil de instalar/usar para los principiantes. Los tres últimos de la lista tienen una función de vista previa donde se pueden ver los resultados de la compilación del documento.

Además, existen herramientas LaTeX en línea que pueden ser útiles para principiantes o personas que deben colaborar, p. ShareLaTeX y Overleaf.

Instalación y configuración #

Puede elegir entre las principales distribuciones de LaTeX:

  • TeX Live (Windows, Linux y OS X), la distribución multiplataforma estándar.
  • MacTeX (Mac) Una versión empaquetada de TeX Live hecha para OS X con algunas herramientas específicas de Mac
  • MiKTeX (Windows) Una distribución separada completamente que

Todas las distribuciones son más o menos equivalentes en un mundo ideal. TeX Live tiene la ventaja de estar disponible en todas las plataformas y, por lo tanto, tiene mucho mejor apoyo de la comunidad. MiKTeX puede aprovechar las funciones específicas de Windows. Por razones de licencia, MiKTeX también distribuirá algunos paquetes que TeX Live no distribuirá.

En todos los casos, se recomienda la instalación completa. Específicamente, el uso de la función de descarga a pedido de MiKTeX bloqueará/bloqueará muchos editores.

Instalación #

Windows (TeXLive) #

  1. Descargue la versión más reciente de TeXLive install-tl-windows.exe desde su sitio web.
  2. Ejecute install-tl-windows.exe y siga las instrucciones.

Windows (MiKTeX) #

  1. Descargue el instalador MiKTeX más reciente de su sitio web.
  2. Ejecute el instalador y siga las instrucciones.

Mac OS X (TeXLive) #

  1. Descargue el MacTeX más reciente de su sitio web.
  2. Ejecute MacTeX.pkg y siga las instrucciones.

Linux (TeXLive) #

Los usuarios de Linux tienen dos opciones:

  1. Instale a través del administrador de paquetes de su distribución (generalmente varios lanzamientos atrás)
  2. Instalar desde arriba (publicado anualmente, actualizado con frecuencia)

Uso de administradores de paquetes #

  • Arch Linux: pacman -S texlive-most
  • Debian/Ubuntu/Mint: apt-get install texlive-full
  • Fedora: ñam instalar texlive

Tenga en cuenta que el uso de este método significa que dependerá del mantenedor de ese paquete para la distribución de actualizaciones. Estos paquetes a menudo estarán varios lanzamientos detrás de la distribución más reciente, lo que a menudo significa que faltarán actualizaciones críticas. Casi siempre es mejor instalar desde arriba. También tenga en cuenta que el administrador de paquetes de la distribución probablemente no reconocerá la instalación directa y podría intentar instalarla cuando se instalan otros paquetes de soporte relacionados.

Instalación desde Upstream #

  1. Descargue la versión más reciente de TeXLive install-tl-unx.tar.gz desde su sitio web.

  2. Extraiga los archivos del archivo con tar -zxvf install-tl-unx.tar.gz.

  3. Cambie a la carpeta descargada con cd install-tl-unx.

  4. Ejecute ./install-tl y siga las instrucciones.

    TeXLive should now be installed under /usr/local/texlive/YEAR/, where YEAR is the four digit year (e.g. 2016). In this way, it is possible to have multiple TeXLive versions alongside each other and switch between them by changing your PATH variable.

    Open this folder and check the bin folder. It should contain a subfolder, which (depending on your platform) will be something like i386-linux or x86_64-linux.

  5. Agregue la carpeta binaria TeX Live a su ruta con

     EXPORT PATH=/usr/local/texlive/YEAR/bin/PLATFORM:$PATH

    where YEAR is the four digit year (e.g. 2016), and PLATFORM is your platform (e.g. x86_64-linux).

Instalación de prueba #

La instalación de LaTeX ahora está completa. Para probarlo, cree un nuevo archivo con su editor de texto favorito, asígnele el nombre test.tex y agregue el siguiente contenido:

\documentclass{article}
\begin{document}
  Hello World!
\end{document}

Ahora, abra la consola o terminal, navegue a la carpeta donde guardó test.tex y ejecute

pdflatex test

(Tenga en cuenta que su editor puede tener instalaciones para ejecutar esto por usted).

Esto crea varios archivos nuevos, incluido test.pdf. Este es el documento de salida, y se ve así:

¡Felicitaciones, ha instalado correctamente LaTeX y ha creado su primer documento LaTeX!
