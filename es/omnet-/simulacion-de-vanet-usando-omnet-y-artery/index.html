Esta documentación le dará una idea de cómo se pueden simular las VANET europeas basadas en ETSI ITS-G5 e IEEE 802.11p mediante el uso del simulador de eventos discretos OMNeT++.

El marco de simulación de la arteria #

Artery (https://github.com/riebl/artery) es un marco de simulación para OMNeT++. Está diseñado para simular la pila de protocolos ETSI ITS-G5 que se utiliza en VANET europeos. Actualmente, utiliza OMNeT++ en la versión 5.0. Es completamente código abierto y en desarrollo activo.

Resumen de funciones: #

  • Aplicaciones Day One como Cooperate Awarenes Message (CAM) y Decentralized Environmental Notification Message (DENM) especificadas por ETSI ITS-G5
  • Proporciona Facilidades comunes para estas aplicaciones
  • Permite la creación rápida de prototipos para crear nuevas aplicaciones fácilmente
  • Enrutamiento geográfico, que es manejado por Vanetza (https://github.com/riebl/vanetza)
  • Vanetza también cubre el Protocolo Básico de Transporte (BTP)
  • Utiliza Veins (http://veins.car2x.org/) o INET (https://inet.omnetpp.org/) como capa MAC IEEE 802.11p; estos proporcionan también varios modelos de propagación de ondas de capa física
  • Movimiento de vehículos realista proporcionado por el simulador de tráfico de código abierto SUMO (http://www.dlr.de/)
  • Una manera fácil de crear diferentes escenarios de tráfico (como embotellamientos o influencias climáticas)

¿Cómo funciona Arteria? #

La arteria se compone básicamente de cuatro partes. Primero, está la simulación de red que maneja el envío y la recepción de paquetes. Esto se basa en las redes OMNeT++, donde Veins o INET proporcionan modelos razonablemente realistas del medio de radio. Además, Veins e INET proporcionan una implementación de las capas física y de enlace IEEE 802.11. Eres libre de elegir qué marco quieres usar cambiando el archivo de configuración de redes.

El enrutamiento entre los participantes de la red lo realiza Vanetza. Vanetza incluye algoritmos de enrutamiento topológico ITS-G5 como Single Hop Broadcast (SHB), así como enrutamiento geográfico como GeoBroadcasts (GBC). Cada estación ITS-G5 en la red está representada por un Router Vanetza. Actualmente, solo los vehículos se utilizan como estaciones ITS-G5, pero las unidades de carretera (RSU) ya están en la hoja de ruta. Según el algoritmo de enrutamiento elegido, el enrutador determina el siguiente salto en la red y envía el paquete a la capa física, que es proporcionada por Veins o INET.

Las aplicaciones son la razón por la que los paquetes se crean de primera mano. La capa de aplicación la proporciona Artery. Las aplicaciones Day One mencionadas por ETSI ya están implementadas y generan Mensajes de Conciencia Cooperativa (CA) y Notificación Ambiental Descentralizada (DEN) de acuerdo con la definición del estándar. Para activar mensajes DEN, se necesita la definición de escenarios de activación adecuados. Para este propósito se puede utilizar el Storyboard. Permite definir las condiciones del escenario, así como los efectos para provocar varios escenarios de tráfico, como accidentes, condiciones climáticas o atascos. De acuerdo con la situación del tráfico, la aplicación activa el mensaje DEN adecuado.

Por último, pero no menos importante, está la simulación de tráfico proporcionada por SUMO. SUMO y Artery se conectan mediante la interfaz TraCI. TraCI permite leer información sobre el estado actual de cada vehículo dentro de la simulación, así como cambiar los parámetros del vehículo. El Storyboard utiliza principalmente el cambio de parámetros del vehículo para lograr las situaciones de tráfico mencionadas anteriormente. Los mapas utilizados por SUMO se pueden derivar de mapas reales con flujos de tráfico realistas (como https://github.com/lcodeca/LuSTScenario/wiki u Open Street Map) o escenarios sintéticos.

Instalación #

Es bastante fácil instalar Artery en sistemas basados ​​en Linux. Necesita tener un compilador de C++ compatible con C++11, así como bibliotecas Boost y Vanetza para construir Artery. Además, necesitas uno de Venas e INET. Por supuesto, también se requieren OMNeT++ y SUMO.

Instalando OMNeT++ #

  1. Descargue el archivo OMNeT++ 5.0 (https://omnetpp.org/omnetpp).
  2. Extraiga el archivo en una nueva carpeta.
  3. Escriba ./configure y luego escriba make.
  4. Agregue su directorio path/to/OMNeT/build en su variable de entorno PATH.
  5. Asegúrese de que su instalación funcione escribiendo omnetpp. Se espera que se inicie el entorno de desarrollo OMNeT++.
  6. Puede encontrar más instrucciones de instalación en https://omnetpp.org/doc/omnetpp/InstallGuide.pdf.

Instalando SUMO #

  1. Para instalar SUMO, primero descargue la última versión (se puede encontrar en http://www.sumo.dlr.de/userdoc/Downloads.html). No se recomienda utilizar la versión SUMO incluida con el sistema Ubuntu o Debian, ya que estas versiones son bastante antiguas.
  2. Extraiga los archivos descargados.
  3. Después de esto, debe construir su versión SUMO. Asegúrese de tener instalado el paquete libproj-dev (en Debian o Ubuntu).
  4. Navegue hasta el directorio donde extrajo SUMO.
  5. Escriba: ./configure. Después de realizar la configuración, eche un vistazo al Resumen de características opcionales, que se imprimió al final. Tiene que incluir la entrada PROJ en la lista Habilitado:. Esta función necesita el libproj-dev mencionado anteriormente y permite que SUMO proporcione coordenadas geográficas para cada vehículo. Sin esta función, la simulación se detendrá en tiempo de ejecución debido a las coordenadas no válidas.
  6. Escriba: make para compilar SUMO en el directorio ./build.
  7. Agregue path/to/your/sumo-version/build a su variable de entorno PATH.
  8. Verifique su instalación escribiendo sumo-gui.
  9. Puede encontrar más información sobre la instalación de SUMO en http://www.sumo.dlr.de/userdoc/Installing.html

Instalando Arteria #

Para instalar Artery, necesitas tener Vanetza, Venas e Inet. Para facilitar la instalación, el repositorio Artery contiene todos estos marcos como enlaces de sub-repositorios. Esto garantiza que solo se utilicen versiones compatibles de *INET, Veins y Vanetza con Artery.

  1. Extraiga el repositorio Artery de https://github.com/riebl/artery
  2. Para compilar Vanetza, asegúrese de estar en el directorio raíz y escriba make vanetza. Vanetza ahora se compilará en extern/vanetza/build
  3. Para compilar Inet, asegúrese de estar en el directorio raíz de Artery y escriba make inet. Inet ahora se construirá.
  4. Para construir Venas, asegúrese de estar en el directorio raíz de Arteria y escriba hacer venas. Veins ahora se construirá en extern/veins/build
  5. Para construir Artery, asegúrese de estar nuevamente en el directorio raíz. Escriba compilación mkdir.
  6. Escribe compilación de CD
  7. Escriba cmake ..
  8. Escribe cmake --build

Puede encontrar más información sobre la creación de las herramientas mencionadas anteriormente en las siguientes fuentes. Esto puede ser útil en caso de un error.

  1. Vanetza: https://github.com/riebl/vanetza
  2. Arteria: https://github.com/riebl/artery
  3. Inet: https://github.com/riebl/artery/blob/master/extern/inet/INSTALAR
  4. Venas: http://veins.car2x.org/

Felicidades, ahora tiene todos los requisitos previos para seguir adelante e intentar lanzar el primer ejemplo de Artery. ¡Cómo se hace esto se explica en la siguiente sección!

Ejecutando un ejemplo #

Como Artery es bastante complejo, se recomienda empezar por entender OMNeT++. Un buen punto de partida es el tutorial TicToc. Se puede encontrar en https://omnetpp.org/doc/omnetpp/tictoc-tutorial/. Este tutorial proporciona una descripción general de la funcionalidad básica de OMNeT++. Esto incluye, entre otros, el lenguaje NED y la definición de Redes.

Si ya se puso en contacto con OMNeT++, puede intentar iniciar el ejemplo enviado con Artery. Para hacerlo, sigue estos pasos:

  1. Navegue al directorio build de Artery (que se creó durante la construcción)
  2. Escriba make run_example. Este comando verifica los cambios en el código (como suele hacer make) y luego inicia OMNeT++ usando omnetpp.ini en la carpeta scenarios/artery.
  3. Si desea que SUMO abra su GUI durante la simulación, agregue esta línea a su omnetpp.ini: *.traci.launcher.sumo = "sumo-gui"

Ejecutando un ejemplo en modo de depuración #

Si desea investigar el código mientras se ejecuta la simulación, debe compilar Artery en modo de depuración. Para ello se recomienda utilizar ccmake.

  1. Navegue a su directorio build de Artery
  2. Escribe ccmake.
  3. Muévase a la entrada CMAKE_BUILD_TYPE (generalmente esta entrada está en la primera posición superior) presione [enter] y escriba: Debug. De nuevo, presione [enter].
  4. Presione [c] para configurar.
  5. Presione [q] para salir.
  6. Nuevamente, escriba make en el directorio build.
  7. Para ejecutar el ejemplo en modo de depuración, escriba make debug_example
  8. Después de que el depurador GDB esté listo, escriba run
  9. Siéntase libre de presionar [ctrl + c] para ingresar al depurador y establecer sus puntos de interrupción

Agregar mi propio servicio ITS-G5: ¿Por dónde debo comenzar? #

Para comenzar con la creación de su propio servicio, eche un vistazo a la clase base de cada servicio, que se llama ItsG5Service. También puede consultar CamService y DenmService, ya que son servicios ya implementados. Todos los archivos y clases relacionados con la aplicación se pueden encontrar en la subcarpeta artery/application.

La inicialización de un servicio la realiza ItsG5Middleware. Antes de agregar su servicio, proporcione una clase de C++ adecuada y un archivo .ned que pertenezca a su clase. Su clase de servicio debe derivarse de ItsG5Service. Entonces, ponga su servicio en examples/yourExample/services.xml. Como punto de partida, puede copiar la entrada CamService (examples/artery/services.xml) y cambiar el número de puerto.

El ItsG5Middleware también invoca los servicios en cada paso de la simulación llamando a ItsG5Service::trigger(). Esto significa que, si desea hacer algo periódicamente con su servicio, anule este método y coloque su código allí.
