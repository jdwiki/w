Perfilando una compilación #

Antes de comenzar a ajustar el rendimiento de su compilación de Gradle, debe establecer una línea de base y determinar qué partes de la compilación están tomando la mayor parte del tiempo. Para hacer esto, puede perfilar su compilación agregando el argumento --profile a su comando Gradle:

gradle --profile
./gradlew --profile

Una vez completada la compilación, verá un informe de perfil HTML para la compilación en ./build/reports/profile/, con un aspecto similar a este:

Al hacer clic en las pestañas junto a Resumen, puede ver un desglose más detallado de dónde se gasta el tiempo.

Usar el demonio Gradle #

Puede habilitar Gradle Daemon para mejorar el rendimiento de sus compilaciones.

Gradle Daemon mantiene Gradle Framework inicializado y en ejecución, y almacena en caché los datos del proyecto en la memoria para mejorar el rendimiento.

Para una sola compilación

Para habilitar el Daemon para una sola compilación, simplemente puede pasar el argumento --daemon a su comando gradle o secuencia de comandos de Gradle Wrapper.

gradle --daemon
./gradlew --daemon

Para todas las compilaciones de un proyecto

Para habilitar el Daemon para todas las compilaciones de un proyecto, puede agregar:

org.gradle.daemon=true

Al archivo gradle.properties de su proyecto.

Para todas las compilaciones

Para habilitar Gradle Daemon de forma predeterminada, para cada compilación realizada por su cuenta de usuario en su sistema, edite $GRADLE_USER_HOME/.gradle/gradle.properties (~/.gradle/gradle.properties de forma predeterminada) y agregue esta línea :

org.gradle.daemon=true

También puede hacer esto con un solo comando en los sistemas Mac/Linux/*nix:

touch ~/.gradle/gradle.properties && echo "org.gradle.daemon=true" >> ~/.gradle/gradle.properties

O en Windows:

(if not exist "%USERPROFILE%/.gradle" mkdir "%USERPROFILE%/.gradle") && (echo org.gradle.daemon=true >> "%USERPROFILE%/.gradle/gradle.properties")

Desactivando el demonio

Puede deshabilitar el Daemon para una compilación específica usando el argumento --no-daemon, o deshabilitarlo para un proyecto específico configurando explícitamente org.gradle.daemon=false en el archivo gradle.properties del proyecto.

Deteniendo al Demonio

Si desea detener un proceso de Daemon manualmente, puede eliminar el proceso a través del administrador de tareas de su sistema operativo o ejecutar el comando gradle --stop. El interruptor --stop hace que Gradle solicite que todos los procesos Daemon en ejecución, de la misma versión de Gradle utilizada para ejecutar el comando, terminen por sí mismos. Por lo general, los procesos de Daemon terminarán automáticamente *después de *3 horas de inactividad o menos.

Compilaciones paralelas de Gradle #

Gradle solo ejecutará una tarea a la vez de forma predeterminada, independientemente de la estructura del proyecto. Al usar el interruptor --parallel, puede obligar a Gradle a ejecutar subproyectos independientes, aquellos que no tienen dependencias de proyecto implícitas o explícitas entre sí, en paralelo, lo que le permite ejecutar varias tareas al mismo tiempo siempre que esas tareas están en diferentes proyectos.

Para construir un proyecto en modo paralelo:

gradle build --parallel

También puede hacer que la construcción en paralelo sea la predeterminada para un proyecto agregando la siguiente configuración al archivo gradle.properties del proyecto:

org.gradle.parallel=true

Usar la última versión de Gradle #

El equipo de Gradle trabaja regularmente para mejorar el rendimiento de diferentes aspectos de las compilaciones de Gradle. Si está utilizando una versión anterior de Gradle, se está perdiendo los beneficios de ese trabajo. Intente actualizar a la última versión de Gradle para ver qué tipo de impacto tiene. Hacerlo es de bajo riesgo porque muy pocas cosas se rompen entre las versiones menores de Gradle.

El archivo de propiedades para el contenedor de Gradle se puede encontrar en la carpeta de su proyecto en gradle/wrapper/ y se llama gradle-wrapper.properties. El contenido de ese archivo podría verse así:

distributionBase=GRADLE_USER_HOME
distributionPath=wrapper/dists
zipStoreBase=GRADLE_USER_HOME
zipStorePath=wrapper/dists
distributionUrl=https\://services.gradle.org/distributions/gradle-X.X.X.zip

Puede cambiar manualmente el número de versión X.X.X (versión actual) a Y.Y.Y (versión más nueva) y la próxima vez que ejecute el contenedor, la nueva versión se descargará automáticamente.

Configurar bajo demanda #

Si la generación de perfiles de su compilación muestra una inversión significativa de tiempo en Configuración de proyectos, la opción Configurar a pedido podría mejorar su rendimiento.

Puede habilitar el modo Configure on Demand editando $GRADLE_USER_HOME/.gradle/gradle.properties (~/.gradle/gradle.properties por defecto) y configurando org.gradle.configureondemand.

org.gradle.configureondemand=true

Para habilitarlo solo para un proyecto específico, edite el archivo gradle.properties de ese proyecto.

Si Configure on Demand está habilitado, en lugar de configurar todos los proyectos por adelantado, Gradle solo configurará los proyectos necesarios para la tarea que se está ejecutando.

Del [Manual de Gralde] (https://docs.gradle.org/current/userguide/multi_project_builds.html#sec:configuration_on_demand):

El modo de configuración bajo demanda intenta configurar solo los proyectos que son relevantes para las tareas solicitadas, es decir, solo ejecuta el archivo build.gradle de los proyectos que participan en la compilación. De esta forma, se puede reducir el tiempo de configuración de una compilación grande de varios proyectos. A largo plazo, este modo se convertirá en el modo predeterminado, posiblemente el único modo para la ejecución de la compilación de Gradle.

Ajuste de los parámetros de uso de memoria de JVM para Gradle #

Puede establecer o aumentar los límites de uso de la memoria (u otros argumentos de JVM) utilizados para las compilaciones de Gradle y Gradle Daemon editando $GRADLE_USER_HOME/.gradle/gradle.properties (~/.gradle/gradle.properties por defecto), y configurando org.gradle.jvmargs.

Para configurar estos límites solo para un proyecto específico, edite el archivo gradle.properties de ese proyecto.

La configuración de uso de memoria predeterminada para las compilaciones de Gradle y Gradle Daemon son:

org.gradle.jvmargs=-Xmx1024m -XX:MaxPermSize=256m

Esto permite una asignación de memoria máxima general (tamaño de almacenamiento dinámico) de 1 GB y una asignación de memoria máxima para objetos “internos” permanentes de 256 MB. Cuando se alcanzan estos tamaños, se produce la recolección de elementos no utilizados, lo que puede reducir significativamente el rendimiento.

Suponiendo que tenga memoria de sobra, podría duplicarlas fácilmente de la siguiente manera:

org.gradle.jvmargs=-Xmx2024m -XX:MaxPermSize=512m

Tenga en cuenta que dejará de ver los beneficios de aumentar XX:MaxPermSize antes que cuando los aumentos de Xmx dejen de ser beneficiosos.
