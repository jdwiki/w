Agregue una dependencia de archivo JAR local #

JAR individual #

A veces, tiene un archivo JAR local que necesita agregar como una dependencia a su compilación de Gradle. Así es como puedes hacer esto:

dependencies {
    compile files('path/local_dependency.jar')
}

Donde ruta es una ruta de directorio en su sistema de archivos y local_dependency.jar es el nombre de su archivo JAR local. La ruta puede ser relativa al archivo de compilación.

Directorio de JAR #

También es posible agregar un directorio de jars para compilar. Esto se puede hacer así:

dependencies {
        compile fileTree(dir: 'libs', include: '*.jar')
}

Donde libs sería el directorio que contiene los archivos jar y *.jar sería el filtro de qué archivos incluir.

Directorio de JARs como repositorio #

Si solo desea buscar archivos jar en un repositorio en lugar de agregarlos directamente como una dependencia con su ruta, puede usar un repositorio flatDir.

repositories {
    flatDir {
        dirs 'libs'
    }
}

Busca jars en el directorio libs y sus directorios secundarios.

Agregar una dependencia #

Las dependencias en Gradle siguen el mismo formato que Maven. Las dependencias se estructuran de la siguiente manera:

group:name:version

Aquí hay un ejemplo:

'org.springframework:spring-core:4.3.1.RELEASE'

Para agregar una dependencia en tiempo de compilación, simplemente agregue esta línea en su bloque de dependencia en el archivo de compilación de Gradle:

compile 'org.springframework:spring-core:4.3.1.RELEASE'

Una sintaxis alternativa para esto nombra cada componente de la dependencia explícitamente, así:

compile group: 'org.springframework', name: 'spring-core', version: '4.3.1.RELEASE'

Esto agrega una dependencia en tiempo de compilación.

También puede agregar dependencias solo para pruebas. Aquí hay un ejemplo:

testCompile group: 'junit', name: 'junit', version: '4.+'

Agregando repositorios #

Debe señalar a Gradle la ubicación de sus complementos para que Gradle pueda encontrarlos. Haz esto agregando repositorios { ... } a tu build.gradle.

Este es un ejemplo de cómo agregar tres repositorios, JCenter, Maven Repository y un repositorio personalizado que ofrece dependencias en estilo Maven.

repositories {
  // Adding these two repositories via method calls is made possible by Gradle's Java plugin
  jcenter()
  mavenCentral()

  maven { url "http://repository.of/dependency" }
}

Lista de dependencias #

Llamar a la tarea dependencias le permite ver las dependencias del proyecto raíz:

gradle dependencies

Los resultados son gráficos de dependencia (teniendo en cuenta las dependencias transitivas), desglosados ​​por configuración. Para restringir las configuraciones mostradas, puede pasar la opción --configuration seguida de una configuración elegida para analizar:

gradle dependencies --configuration compile

Para mostrar las dependencias de un subproyecto, utilice la tarea <subproyecto>:dependencias. Por ejemplo, para listar las dependencias de un subproyecto llamado api:

gradle api:dependencies

Depende de otro proyecto de Gradle #

En el caso de una compilación Gradle de varios proyectos, a veces es posible que deba depender de otro proyecto en su compilación. Para lograr esto, ingresaría lo siguiente en las dependencias de su proyecto:

dependencies {
    compile project(':OtherProject')
}

Donde ':OtherProject' es la ruta de Gradle para el proyecto, al que se hace referencia desde la raíz de la estructura del directorio.

Para hacer que ':OtherProject' esté disponible en el contexto del archivo build.gradle, agregue esto al settings.gradle correspondiente

include ':Dependency'
project(':Dependency').projectDir = new File('/path/to/dependency')

Para obtener una explicación más detallada, puede consultar la documentación oficial de Gradle aquí.

Agregue el archivo .aar al proyecto de Android usando gradle #

  1. Navegue al módulo app del proyecto y cree el directorio libs.
  2. Coloque su archivo .aar allí. Por ejemplo myLib.aar.
  3. Agregue el siguiente código al bloque android del archivo build.gradle del nivel app.
  repositories {
        flatDir {
            dirs 'libs'
        }
    }

De esta manera, definió un nuevo repositorio adicional que apunta a la carpeta libs del módulo app.

  1. Agregue el siguiente código al bloque dependencias o al archivo build.gradle:
compile(name:'myLib', ext:'aar')
