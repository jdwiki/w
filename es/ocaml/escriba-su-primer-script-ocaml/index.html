Hola Mundo #

Este ejemplo asume que ha instalado OCaml.

Compilando código OCaml #

Crea un nuevo archivo llamado hello.ml, con el siguiente contenido:

print_string "Hello world!\n"

ocamlc es el compilador de OCaml. Para compilar y ejecutar este script, ejecute

$ ocamlc -o hello hello.ml

y luego ejecutar el binario resultante

$ ./hello
Hello world!

Ejecutando código OCaml #

También puede ejecutar este script sin compilarlo en un binario. Puede hacerlo utilizando ocaml, el sistema de nivel superior de ocaml que permite el uso interactivo de OCaml. En su caparazón, simplemente ejecute

$ ocaml hello.ml
Hello world!

En el REPL #

Abra un nuevo shell y escriba ocaml para abrir el sistema de nivel superior. Una vez en la sesión, puede escribir el mismo programa:

       OCaml version 4.02.1

# print_string "hello world!\n";;

presione enter para evaluar la expresión y active la impresión.

hello world!
- : unit = ()

¡Éxito! Lo vemos impreso ¡hola mundo!, pero ¿de qué se trata - :unit = ()? OCaml no tiene declaraciones, todo es una expresión que se evalúa como un valor escrito. En este caso, print_string es una función que toma una cadena como entrada y devuelve una unidad. Piense en unidad como un tipo que solo puede tomar un valor, () (también conocido como unidad), y representa un cálculo terminado que no devuelve ningún valor significativo.

En este caso, print_string también tiene el efecto secundario de poner los caracteres que recibió como entrada en la pantalla, por lo que vemos la primera línea.

Para salir de REPL, presione ctrl+D.

Como un script de Unix #

Tenemos dos formas de crear un script OCaml. El primero usa el nivel superior del sistema (proporcionado por su administrador de paquetes como apt-get) y el segundo usa el nivel superior proporcionado por OPAM.

Usar el nivel superior del sistema #

Abre tu editor favorito y escribe:

#!/usr/bin/ocaml

print_string "hello worlds!\n";;

Después, puede usar chmod +x your_file.ml y puede ejecutar su script con ./your_file.ml.

Utilice el nivel superior proporcionado por OPAM #

#!/usr/bin/env ocaml

print_string "hello worlds!\n";;

La gran diferencia está en la versión de su nivel superior. De hecho, si configuró su OPAM con un interruptor específico (como opam switch 4.03.0), el script usará OCaml 4.03.0. De la primera forma, en Debian Sid por ejemplo, el script usará OCaml 4.02.3.

Puede reemplazar shebang por #!/usr/bin/env utop para usar utop en lugar del nivel superior vainilla.

arriba #

utop es otro nivel superior de ocaml fuera de la distribución, lo que significa que debe descargar e instalar utop (la forma fácil es usar OPAM: opam install utop). utop tiene muchas características como el histórico, la finalización y la edición de línea interactiva.

Entonces, si desea una manera fácil de probar algunos códigos Ocaml, utop es la mejor.

¿Por qué utop y no ocaml? #

utop y ocaml no tienen una gran diferencia si desea un script ocaml como el anterior. Pero lo común en la comunidad OCaml es usar utop en lugar de ocaml.

De hecho, el REPL ocaml lo proporciona la distribución ocaml. Por lo tanto, este REPL sigue el ciclo de lanzamiento del compilador y si desea algunas funciones adicionales, debe esperar el próximo lanzamiento del compilador. utop, como explicamos, está fuera de la distribución, por lo que el ciclo de lanzamiento no está restringido por el compilador y si desea una función adicional, es más probable que intente insertar esta función dentro de utop que ocaml :)!

Para este punto (y para la característica histórica), la mayoría de las personas en la comunidad ocaml prefieren usar utop que ocaml.
