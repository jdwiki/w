Sintaxis #

  • xmlns:fx=“http://javafx.com/fxml”       // declaración de espacio de nombres

Configuración de propiedades #

Hay varias formas de agregar datos a un objeto en fxml:

etiqueta <propiedad> #

Se puede agregar una etiqueta con el nombre de una propiedad como elemento secundario de un elemento utilizado para crear una instancia. El elemento secundario de esta etiqueta se asigna a la propiedad utilizando el setter o se agrega al contenido de la propiedad (propiedades de lista/mapa de solo lectura).

Propiedad por defecto #

Una clase se puede anotar con la anotación @DefaultProperty. En este caso, los elementos se pueden agregar directamente como elementos secundarios sin usar un elemento con el nombre de la propiedad.

propiedad="valor" atributo #

Las propiedades se pueden asignar utilizando el nombre de propiedad como nombre de atributo y el valor como valor de atributo. Esto tiene el mismo efecto que agregar el siguiente elemento como elemento secundario de la etiqueta:

<property>
    <String fx:value="value" />
</property>

configuradores estáticos #

Las propiedades también se pueden establecer utilizando setters static. Estos son métodos estáticos llamados setProperty que toman el elemento como primer parámetro y el valor a establecer como segundo parámetro. Esos métodos pueden residir en cualquier clase y pueden usarse usando ContainingClass.property en lugar del nombre de propiedad habitual.

Nota: Actualmente parece ser necesario tener un método getter estático correspondiente (es decir, un método estático llamado getProperty que toma el elemento como parámetro en la misma clase que el setter estático) para que esto funcione a menos que el tipo de valor es Cadena.

Tipo Coerción #

El siguiente mecanismo se utiliza para obtener un objeto de la clase correcta durante las asignaciones, p. para adaptarse al tipo de parámetro de un método de establecimiento.

Si las clases son asignables, entonces se usa el valor en sí.

De lo contrario, el valor se convierte de la siguiente manera

Tipo de objetivo valor utilizado (valor de origen s)
Booleano, booleano Booleano.valorDe(s)
char, Carácter s.toString.charAt(0)
otro tipo primitivo o tipo contenedor método apropiado para el tipo de destino, en caso de que s sea un Número, el valueOf(s.toString()) para el tipo contenedor de lo contrario
Entero grande BigInteger.valueOf(s.longValue()) es s es un Number, new BigInteger(s.toString()) de lo contrario
GranDecimal BigDecimal.valueOf(s.doubleValue()) es s es un Número, new BigDecimal(s.toString()) de lo contrario
Número Double.valueOf(s.toString()) si s.toString() contiene un ., Long.valueOf(s.toString()) de lo contrario
Clase Class.forName(s.toString()) invocado usando el contexto ClassLoader del subproceso actual sin inicializar la clase
enumeración El resultado del método valueOf, adicionalmente convertido a una String en mayúsculas separada por _ insertada antes de cada letra mayúscula, si s es una String que comienza con una letra minúscula
otro el valor devuelto por un método static valueOf en targetType, que tiene un parámetro que coincide con el tipo de s o una superclase de ese tipo

Nota: Este comportamiento no está bien documentado y podría estar sujeto a cambios.

public enum Location {
    WASHINGTON_DC,
    LONDON;
}

package fxml.sample;

import java.math.BigInteger;
import java.util.ArrayList;
import java.util.HashMap;
import java.util.List;
import java.util.Map;
import javafx.beans.DefaultProperty;

@DefaultProperty("items")
public class Sample {
    
    private Location loaction;

    public Location getLoaction() {
        return loaction;
    }

    public void setLoaction(Location loaction) {
        this.loaction = loaction;
    }

    public int getNumber() {
        return number;
    }

    public void setNumber(int number) {
        this.number = number;
    }
    
    int number;

    private final List<Object> items = new ArrayList<>();

    public List<Object> getItems() {
        return items;
    }
    
    private final Map<String, Object> map = new HashMap<>();

    public Map<String, Object> getMap() {
        return map;
    }
    
    private BigInteger serialNumber;

    public BigInteger getSerialNumber() {
        return serialNumber;
    }

    public void setSerialNumber(BigInteger serialNumber) {
        this.serialNumber = serialNumber;
    }

    @Override
    public String toString() {
        return "Sample{" + "loaction=" + loaction + ", number=" + number + ", items=" + items + ", map=" + map + ", serialNumber=" + serialNumber + '}';
    }
    
}

package fxml.sample;

public class Container {

    public static int getNumber(Sample sample) {
        return sample.number;
    }

    public static void setNumber(Sample sample, int number) {
        sample.number = number;
    }

    private final String value;

    private Container(String value) {
        this.value = value;
    }

    public static Container valueOf(String s) {
        return new Container(s);
    }

    @Override
    public String toString() {
        return "42" + value;
    }

}

Pasar parámetros a FXML - usando un controllerFactory #

Problema: Es necesario pasar algunos datos a una escena cargada desde un fxml.

Solución

Especifique una fábrica de controladores que sea responsable de crear los controladores. Pase los datos a la instancia del controlador creada por la fábrica.

FXML

<?xml version="1.0" encoding="UTF-8"?>

<?import javafx.scene.text.*?>
<?import javafx.scene.layout.*?>

<VBox xmlns:fx="http://javafx.com/fxml/1" fx:controller="valuepassing.TestController">
    <children>
        <Text fx:id="target" />
    </children>
</VBox>

Controlador

package valuepassing;

import javafx.fxml.FXML;
import javafx.scene.text.Text;

public class TestController {

    private final String data;

    public TestController(String data) {
        this.data = data;
    }
    
    @FXML
    private Text target;
    
    public void initialize() {
        // handle data once the fields are injected
        target.setText(data);
    }

}

Código utilizado para cargar el fxml

Datos de cadena = “¡Hola mundo!”;

Map<Class, Callable<?>> creators = new HashMap<>();
creators.put(TestController.class, new Callable<TestController>() {

    @Override
    public TestController call() throws Exception {
        return new TestController(data);
    }

});

FXMLLoader loader = new FXMLLoader(getClass().getResource("test.fxml"));

loader.setControllerFactory(new Callback<Class<?>, Object>() {

    @Override
    public Object call(Class<?> param) {
        Callable<?> callable = creators.get(param);
        if (callable == null) {
            try {
                // default handling: use no-arg constructor
                return param.newInstance();
            } catch (InstantiationException | IllegalAccessException ex) {
                throw new IllegalStateException(ex);
            }
        } else {
            try {
                return callable.call();
            } catch (Exception ex) {
                throw new IllegalStateException(ex);
            }
        }
    }
});

Parent root = loader.load();

Esto puede parecer complejo, pero puede ser útil, si el fxml debe poder decidir qué clase de controlador necesita.

Creación de instancias en FXML #

La siguiente clase se usa para demostrar cómo se pueden crear instancias de clases:

La anotación en Person(@NamedArg("name") String name) debe eliminarse, ya que la anotación @NamedArg no está disponible.

package fxml.sample;

import javafx.beans.NamedArg;
import javafx.beans.property.SimpleStringProperty;
import javafx.beans.property.StringProperty;

public class Person {

    public static final Person JOHN = new Person("John");

    public Person() {
        System.out.println("Person()");
    }
    
    public Person(@NamedArg("name") String name) {
        System.out.println("Person(String)");
        this.name.set(name);
    }
    
    public Person(Person person) {
        System.out.println("Person(Person)");
        this.name.set(person.getName());
    }

    private final StringProperty name = new SimpleStringProperty();

    public final String getName() {
        System.out.println("getter");
        return this.name.get();
    }

    public final void setName(String value) {
        System.out.println("setter");
        this.name.set(value);
    }

    public final StringProperty nameProperty() {
        System.out.println("property getter");
        return this.name;
    }
    
    public static Person valueOf(String value) {
        System.out.println("valueOf");
        return new Person(value);
    }
    
    public static Person createPerson() {
        System.out.println("createPerson");
        return new Person();
    }
    
}

Suponga que la clase Person ya se ha inicializado antes de cargar el archivo fxml.

Una nota sobre las importaciones #

En el siguiente ejemplo de fxml, se omitirá la sección de importaciones. Sin embargo, el fxml debe comenzar con

<?xml version="1.0" encoding="UTF-8"?>

seguido de una sección de importaciones que importa todas las clases utilizadas en el archivo fxml. Esas importaciones son similares a las importaciones no estáticas, pero se agregan como instrucciones de procesamiento. Incluso las clases del paquete java.lang deben importarse.

En este caso se deben sumar las siguientes importaciones:

<?import java.lang.*?>
<?import fxml.sample.Person?>

 

@NamedArg constructor anotado #

Si hay un constructor donde cada parámetro se anota con @NamedArg y todos los valores de las anotaciones @NamedArg están presentes en el fxml, el constructor se usará con esos parámetros.

<Person name="John"/>

<Person xmlns:fx="http://javafx.com/fxml">
    <name>
        <String fx:value="John"/>
    </name>
</Person>

Ambos dan como resultado la siguiente salida de la consola, si están cargados:

Person(String)

 

Sin constructor de argumentos #

Si no hay disponible un constructor anotado @NamedArg adecuado, se usará el constructor que no toma parámetros.

Elimine la anotación @NamedArg del constructor e intente cargar.

<Person name="John"/>

Esto usará el constructor sin parámetros.

Producción:

Person()
setter

 

Atributo fx:valor #

El atributo fx:value se puede usar para pasar su valor a un método static valueOf tomando un parámetro String y devolviendo la instancia para su uso.

Ejemplo

<Person xmlns:fx="http://javafx.com/fxml" fx:value="John"/>

Producción:

valueOf
Person(String)

 

fx:fábrica #

El atributo fx:factory permite la creación de objetos usando métodos arbitrarios estáticos que no toman parámetros.

Ejemplo

<Person xmlns:fx="http://javafx.com/fxml" fx:factory="createPerson">
    <name>
        <String fx:value="John"/>
    </name>
</Person>

Producción:

createPerson
Person()
setter

 

<fx:copiar> #

Usando fx:copy se puede invocar un constructor de copia. Especificar el fx:id de otro El atributo fuente de la etiqueta invocará el constructor de copia con ese objeto como parámetro.

Ejemplo:

<ArrayList xmlns:fx="http://javafx.com/fxml">
    <Person fx:id="p1" fx:constant="JOHN"/>
    <fx:copy source="p1"/>
</ArrayList>

Producción

Person(Person)
getter

 

fx:constante #

fx:constant permite obtener un valor de un campo static final.

Ejemplo

<Person xmlns:fx="http://javafx.com/fxml" fx:constant="JOHN"/>

no producirá ningún resultado, ya que esto solo hace referencia a JOHN, que se creó al inicializar la clase.

Ejemplo FXML #

Un documento FXML simple que describe un AnchorPane que contiene un botón y un nodo de etiqueta:

<?xml version="1.0" encoding="UTF-8"?>

<?import java.lang.*?>
<?import java.util.*?>
<?import javafx.scene.*?>
<?import javafx.scene.control.*?>
<?import javafx.scene.layout.*?>

<AnchorPane id="AnchorPane" prefHeight="200" prefWidth="320" xmlns:fx="http://javafx.com/fxml/1" 
        fx:controller="com.example.FXMLDocumentController">
    <children>
        <Button layoutX="126" layoutY="90" text="Click Me!" onAction="#handleButtonAction" fx:id="button" />
        <Label layoutX="126" layoutY="120" minHeight="16" minWidth="69" fx:id="label" />
    </children>
</AnchorPane>

Este archivo FXML de ejemplo está asociado con una clase de controlador. La asociación entre el FXML y la clase del controlador, en este caso, se realiza especificando el nombre de la clase como el valor del atributo fx:controller en el elemento raíz del FXML: fx:controller="com.example. FXMLDocumentController". La clase de controlador permite que el código Java se ejecute en respuesta a las acciones del usuario en los elementos de la interfaz de usuario definidos en el archivo FXML:

package com.example ;

import java.net.URL;
import java.util.ResourceBundle;
import javafx.event.ActionEvent;
import javafx.fxml.FXML;
import javafx.fxml.Initializable;
import javafx.scene.control.Label;

public class FXMLDocumentController {
    
    @FXML
    private Label label;
    
    @FXML
    private void handleButtonAction(ActionEvent event) {
        System.out.println("You clicked me!");
        label.setText("Hello World!");
    }
    
    @Override
    public void initialize(URL url, ResourceBundle resources) {
        // Initialization code can go here. 
        // The parameters url and resources can be omitted if they are not needed
    }    
    
}

Se puede usar un FXMLLoader para cargar el archivo FXML:

public class MyApp extends Application {

    @Override
    public void start(Stage stage) throws Exception {

        FXMLLoader loader = new FXMLLoader();
        loader.setLocation(getClass().getResource("FXMLDocument.fxml"));
        Parent root = loader.load();
        
        Scene scene = new Scene(root);
        
        stage.setScene(scene);
        stage.show();
    }

}

El método load realiza varias acciones y es útil para comprender el orden en que suceden. En este sencillo ejemplo:

  1. El FXMLLoader lee y analiza el archivo FXML. Crea objetos correspondientes a los elementos definidos en el archivo y toma nota de cualquier atributo fx:id definido en ellos.

  2. Dado que el elemento raíz del archivo FXML definió un atributo fx:controller, FXMLLoader crea una nueva instancia de la clase que especifica. De forma predeterminada, esto sucede al invocar el constructor sin argumentos en la clase especificada.

  3. Cualquier elemento con atributos fx:id definidos que tengan campos en el controlador con nombres de campo coincidentes, y que sean públicos (no recomendado) o @FXML anotados (recomendado) se “inyectarán” en los correspondientes. campos. Entonces, en este ejemplo, dado que hay una Etiqueta en el archivo FXML con fx:id="label" y un campo en el controlador definido como

     @FXML
 private Label label ;

    the label field is initialized with the Label instance created by the FXMLLoader.

  4. Los controladores de eventos se registran con cualquier elemento en el archivo FXML con las propiedades onXXX="#..." definidas. Estos controladores de eventos invocan el método especificado en la clase de controlador. En este ejemplo, dado que el Button tiene onAction="#handleButtonAction", y el controlador define un método

     @FXML
 private void handleButtonAction(ActionEvent event) { ... }

    when an action is fired on the button (e.g. the user presses it), this method is invoked. The method must have void return type, and can either define a parameter matching the event type (ActionEvent in this example), or can define no parameters.

  5. Finalmente, si la clase del controlador define un método de inicialización, se invoca este método. Tenga en cuenta que esto sucede después de que se hayan inyectado los campos @FXML, por lo que se puede acceder a ellos de manera segura en este método y se inicializarán con las instancias correspondientes a los elementos en el archivo FXML. El método initialize() puede no tomar parámetros o puede tomar una URL y un ResourceBundle. En el último caso, estos parámetros se completarán con la ‘URL’ que representa la ubicación del archivo FXML y cualquier ‘ResourceBundle’ establecido en ‘FXMLLoader’ a través de ’loader.setResources(…)’. Cualquiera de estos puede ser nulo si no se establecieron.

Controladores anidados #

No es necesario crear toda la interfaz de usuario en un solo FXML con un solo controlador.

La etiqueta <fx:include> se puede usar para incluir un archivo fxml en otro. El controlador del fxml incluido se puede inyectar en el controlador del archivo incluido como cualquier otro objeto creado por FXMLLoader.

Esto se hace agregando el atributo fx:id al elemento <fx:include>. De esta forma, el controlador del fxml incluido se inyectará en el campo con el nombre <fx:id value>Controller.

Ejemplos:

fx:valor de identificación nombre de campo para inyección
foo FooController
respuesta42 controlador answer42
xYz controlador xYz

Fxmls de muestra

Encimera

Este es un fxml que contiene un StackPane con un nodo Text. El controlador para este archivo fxml permite obtener el valor actual del contador así como incrementar el contador:

contador.fxml

<?xml version="1.0" encoding="UTF-8"?>

<?import javafx.scene.text.*?>
<?import javafx.scene.layout.*?>

<StackPane prefHeight="200" prefWidth="200" xmlns:fx="http://javafx.com/fxml/1" fx:controller="counter.CounterController">
    <children>
        <Text fx:id="counter" />
    </children>
</StackPane>

Contracontrolador

package counter;

import javafx.fxml.FXML;
import javafx.scene.text.Text;

public class CounterController {
    @FXML
    private Text counter;

    private int value = 0;
    
    public void initialize() {
        counter.setText(Integer.toString(value));
    }
    
    public void increment() {
        value++;
        counter.setText(Integer.toString(value));
    }
    
    public int getValue() {
        return value;
    }
    
}

Incluye fxml

<?xml version="1.0" encoding="UTF-8"?>

<?import javafx.scene.control.*?>
<?import javafx.scene.layout.*?>

<BorderPane prefHeight="500" prefWidth="500" xmlns="http://javafx.com/javafx/8" xmlns:fx="http://javafx.com/fxml/1" fx:controller="counter.OuterController">
    <left>
        <Button BorderPane.alignment="CENTER" text="increment" onAction="#increment" />
    </left>
    <center>
        <!-- content from counter.fxml included here -->
        <fx:include fx:id="count" source="counter.fxml" />
    </center>
</BorderPane>

Controlador exterior

El controlador del fxml incluido se inyecta en este controlador. Aquí, el controlador del evento onAction para el Button se usa para incrementar el contador.

package counter;

import javafx.fxml.FXML;

public class OuterController {

    // controller of counter.fxml injected here
    @FXML
    private CounterController countController;

    public void initialize() {
        // controller available in initialize method
        System.out.println("Current value: " + countController.getValue());
    }

    @FXML
    private void increment() {
        countController.increment();
    }

}

Los fxmls se pueden cargar así, asumiendo que el código se llama desde una clase en el mismo paquete que outer.fxml:

Parent parent = FXMLLoader.load(getClass().getResource("outer.fxml"));

Definir bloques y fx:reference #

A veces, es necesario crear un elemento fuera de la estructura de objeto habitual en fxml.

Aquí es donde entra en juego Definir Bloques:

Los contenidos dentro de un elemento <fx:define> no se agregan al objeto creado para el elemento principal.

Cada elemento hijo de <fx:define> necesita un atributo fx:id.

Los objetos creados de esta manera se pueden referenciar más adelante usando el elemento <fx:reference> o usando el enlace de expresión.

El elemento <fx:reference> se puede usar para hacer referencia a cualquier elemento con un atributo fx:id que se maneje antes de que el elemento <fx:reference> se maneje usando el mismo valor que fx:id atributo del elemento al que se hace referencia en el atributo fuente del elemento <fx:reference>.

<?xml version="1.0" encoding="UTF-8"?>

<?import javafx.scene.text.*?>

Pasando datos a FXML - accediendo al controlador existente #

Problema: Es necesario pasar algunos datos a una escena cargada desde un fxml.

Solución

Especifique un controlador utilizando el atributo fx:controller y obtenga la instancia del controlador creada durante el proceso de carga desde la instancia FXMLLoader utilizada para cargar el fxml.

Agregue métodos para pasar los datos a la instancia del controlador y maneje los datos en esos métodos.

FXML

<?xml version="1.0" encoding="UTF-8"?>

<?import javafx.scene.text.*?>
<?import javafx.scene.layout.*?>

<VBox xmlns:fx="http://javafx.com/fxml/1" fx:controller="valuepassing.TestController">
    <children>
        <Text fx:id="target" />
    </children>
</VBox>

Controlador

package valuepassing;

import javafx.fxml.FXML;
import javafx.scene.text.Text;

public class TestController {

    @FXML
    private Text target;

    public void setData(String data) {
        target.setText(data);
    }

}

Código utilizado para cargar el fxml

String data = "Hello World!";

FXMLLoader loader = new FXMLLoader(getClass().getResource("test.fxml"));
Parent root = loader.load();
TestController controller = loader.<TestController>getController();
controller.setData(data);

Pasar datos a FXML - Especificar la instancia del controlador #

Problema: Es necesario pasar algunos datos a una escena cargada desde un fxml.

Solución

Configure el controlador utilizando la instancia FXMLLoader utilizada más tarde para cargar el archivo fxml.

Asegúrese de que el controlador contenga los datos relevantes antes de cargar el archivo fxml.

Nota: en este caso, el archivo fxml no debe contener el atributo fx:controller.

FXML

<?xml version="1.0" encoding="UTF-8"?>

<?import javafx.scene.text.*?>
<?import javafx.scene.layout.*?>

<VBox xmlns:fx="http://javafx.com/fxml/1">
    <children>
        <Text fx:id="target" />
    </children>
</VBox>

Controlador

import javafx.fxml.FXML;
import javafx.scene.text.Text;

public class TestController {

    private final String data;

    public TestController(String data) {
        this.data = data;
    }
    
    @FXML
    private Text target;
    
    public void initialize() {
        // handle data once the fields are injected
        target.setText(data);
    }

}

Código utilizado para cargar el fxml

String data = "Hello World!";

FXMLLoader loader = new FXMLLoader(getClass().getResource("test.fxml"));

TestController controller = new TestController(data);
loader.setController(controller);

Parent root = loader.load();
