Sintaxis #

  • find_package(pkgname [versión] [EXACTO] [SILENCIO] [REQUERIDO])
  • incluir (FindPkgConfig)
  • pkg_search_module(prefijo [REQUERIDO] [SILENCIO] pkgname [otropaquete…])
  • pkg_check_modules(prefijo [REQUERIDO] [SILENCIO] pkgname [otropaquete…])

Parámetros #

Parámetro Detalles
versión (opcional) Versión mínima del paquete definida por un número mayor y, opcionalmente, un número menor, parche y ajuste, en el formato mayor.menor.patch.tweak
EXACTO (opcional) Especificar que la versión especificada en version es la versión exacta que se encuentra
REQUERIDO (opcional) Automáticamente arroja un error y detiene el proceso si no se encuentra el paquete
TRANQUILO (opcional) La función no enviará ningún mensaje a la salida estándar

  • La forma find_package es compatible con todas las plataformas, mientras que la forma pkg-config está disponible solo en plataformas similares a Unix, como Linux y OSX.

  • Puede encontrar una descripción completa de los numerosos parámetros y opciones de find_package en el [manual] (https://cmake.org/cmake/help/v3.6/command/find_package.html).

  • Aunque es posible especificar muchos parámetros opcionales, como la versión del paquete, no todos los módulos Find utilizan correctamente todos esos parámetros. Si ocurre algún comportamiento indefinido, podría ser necesario encontrar el módulo en la ruta de instalación de CMake y corregir o comprender su comportamiento.

Utilice los módulos find_package y Find.cmake #

La forma predeterminada de encontrar paquetes instalados con CMake es usar la función find_package junto con un archivo Find<package>.cmake. El propósito del archivo es definir las reglas de búsqueda para el paquete y establecer diferentes variables, como <paquete>_FOUND, <paquete>_INCLUDE_DIRS y <paquete>_LIBRARIES.

Muchos archivos Find<package>.cmake ya están definidos de forma predeterminada en CMake. Sin embargo, si no hay un archivo para el paquete que necesita, siempre puede escribir uno propio y ponerlo dentro de ${CMAKE_SOURCE_DIR}/cmake/modules (o cualquier otro directorio si se anuló CMAKE_MODULE_PATH)

Puede encontrar una lista de módulos predeterminados en el manual (v3.6). Es fundamental revisar el manual de acuerdo a la versión de CMake utilizada en el proyecto o de lo contrario podrían faltar módulos. También es posible encontrar los módulos instalados con cmake --help-module-list.

Hay un buen ejemplo de FindSDL2.cmake en Github

Aquí hay un CMakeLists.txt básico que requeriría SDL2:

cmake_minimum_required(2.8 FATAL_ERROR)
project("SDL2Test")

set(CMAKE_MODULE_PATH "${CMAKE_MODULE_PATH} ${CMAKE_SOURCE_DIR}/cmake/modules")
find_package(SDL2 REQUIRED)

include_directories(${SDL2_INCLUDE_DIRS})
add_executable(${PROJECT_NAME} main.c)
target_link_libraries(${PROJECT_NAME} ${SDL2_LIBRARIES})

Usar pkg_search_module y pkg_check_modules #

En sistemas operativos similares a Unix, es posible usar el programa pkg-config para buscar y configurar paquetes que proporcionen un archivo <paquete>.pc.

Para usar pkg-config, es necesario llamar a include(FindPkgConfig) en un CMakeLists.txt. Entonces, hay 2 funciones posibles:

  • pkg_search_module, que busca el paquete y usa el primero disponible.
  • pkg_check_modules, que verifica todos los paquetes correspondientes.

Aquí hay un CMakeLists.txt básico que usa pkg-config para encontrar SDL2 con una versión superior o igual a 2.0.1:

cmake_minimum_required(2.8 FATAL_ERROR)
project("SDL2Test")

include(FindPkgConfig)
pkg_search_module(SDL2 REQUIRED sdl2>=2.0.1)

include_directories(${SDL2_INCLUDE_DIRS})
add_executable(${PROJECT_NAME} main.c)
target_link_libraries(${PROJECT_NAME} ${SDL2_LIBRARIES})
