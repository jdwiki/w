Enfoque general para depurar al compilar con Make #

Supongamos que make falla:

$ make

Lánzalo en su lugar con make VERBOSE=1 para ver los comandos ejecutados. Luego ejecute directamente el comando del enlazador o compilador que verá. Intente que funcione agregando banderas o bibliotecas necesarias.

Luego, descubra qué cambiar, para que CMake pueda pasar los argumentos correctos al comando del compilador/vinculador:

  • qué cambiar en el sistema (qué bibliotecas instalar, qué versiones, versiones de CMake en sí)
  • si el anterior falla, qué variables de entorno establecer o qué parámetros pasar a CMake
  • de lo contrario, qué cambiar en CMakeLists.txt del proyecto o en los scripts de detección de bibliotecas como FindSomeLib.cmake

Para ayudar en eso, agregue llamadas message(${MY_VARIABLE}) en CMakeLists.txt o *.cmake para depurar las variables que desea inspeccionar.

Depurar errores de find_package() #

Nota: Los mensajes de error de CMake que se muestran ya incluyen la solución para las rutas de instalación de biblioteca/herramienta “no estándar”. Los siguientes ejemplos solo demuestran salidas más detalladas de CMake find_package().

Paquete/Módulo con soporte interno de CMake #

Si el siguiente código (reemplace el módulo FindBoost con [su módulo en cuestión](https://cmake. org/cmake/help/latest/manual/cmake-modules.7.html))

cmake_minimum_required(VERSION 2.8)
project(FindPackageTest)

find_package(Boost REQUIRED)

da algún error como

CMake Error at [...]/Modules/FindBoost.cmake:1753 (message):
  Unable to find the requested Boost libraries.

  Unable to find the Boost header files.  Please set BOOST_ROOT to the root
  directory containing Boost or BOOST_INCLUDEDIR to the directory containing
  Boost's headers.

Y se está preguntando dónde trató de encontrar la biblioteca, puede verificar si su paquete tiene una opción _DEBUG como la que tiene el módulo Boost para obtener una salida más detallada

$ cmake -D Boost_DEBUG=ON ..

Paquete/Biblioteca habilitado para CMake #

Si el siguiente código (reemplace Xyz con su biblioteca en cuestión)

cmake_minimum_required(VERSION 2.8)
project(FindPackageTest)

find_package(Xyz REQUIRED)

da algún error como

CMake Error at CMakeLists.txt:4 (find_package):
  By not providing "FindXyz.cmake" in CMAKE_MODULE_PATH this project has
  asked CMake to find a package configuration file provided by "Xyz", but
  CMake did not find one.

  Could not find a package configuration file provided by "Xyz" with any of
  the following names:

    XyzConfig.cmake
    xyz-config.cmake

  Add the installation prefix of "Xyz" to CMAKE_PREFIX_PATH or set "Xyz_DIR"
  to a directory containing one of the above files.  If "Xyz" provides a
  separate development package or SDK, be sure it has been installed.

Y se pregunta dónde intentó encontrar la biblioteca, puede usar la variable global CMAKE_FIND_DEBUG_MODE no documentada para obtener una salida más detallada

$ cmake -D CMAKE_FIND_DEBUG_MODE=ON ..

Dejar que CMake cree Makefiles detallados #

Una vez que se inicializa un proyecto de CMake a través de project(), la verbosidad de salida del script de compilación resultante se puede ajustar a través de:

CMAKE_VERBOSE_MAKEFILE

Esta variable se puede configurar a través de la línea de comando de CMake al configurar un proyecto:

cmake -DCMAKE_VERBOSE_MAKEFILE=ON <PATH_TO_PROJECT_ROOT>

Para GNU make esta variable tiene el mismo efecto que ejecutar make VERBOSE=1.
