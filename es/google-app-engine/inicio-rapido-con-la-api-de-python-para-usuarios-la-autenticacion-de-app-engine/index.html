Inicio rápido con la API de Python para usuarios, la autenticación de App Engine

  • Controlador de página principal [views.py]
  • Enrutamiento de aplicaciones [urls.py]
  • Html, ejemplo de interfaz de cómo usar la API de usuarios [index.html]

En esta página

  • Controlador de página principal [views.py]
  • Enrutamiento de aplicaciones [urls.py]
  • Html, ejemplo de interfaz de cómo usar la API de usuarios [index.html]

El uso de la API de usuarios es una forma muy sencilla y flexible de trabajar con la autenticación en App Engine, pero asegúrese de que su aplicación casos no requieren más elementos para el entorno de autenticación.

Nota: Si necesita más información sobre la estructura tradicional de una aplicación de App Engine, consulte esta [información](https://cloud.google.com/appengine/docs/standard/python/getting-started/ creación-libro de visitas).

La API de Usuarios permite:

  • Detectar si el usuario actual ha iniciado sesión.
  • Redirigir al usuario a la página de inicio de sesión adecuada para iniciar sesión.
  • Solicite que el usuario de su aplicación cree una nueva cuenta de Google si aún no tiene una.

Referencia y más detalles

Elementos importantes en la vista:

Importar:

from google.appengine.api import users

Usuario-objeto y métodos:

user = users.get_current_user()

Nota: la implementación de jinja2 es opcional, pero en el artículo se usa para explicar el flujo de trabajo completo.

Controlador de página principal [views.py] #

Importaciones generales, utilizando jinja2 para completar plantillas en html.

import jinja2
import webapp2

Importación importante para usar la API de usuarios:

from google.appengine.api import users

Configuración del entorno Jinja: [en el ejemplo, la tecnología seleccionada para completar la información en la interfaz]

JINJA_ENVIRONMENT = jinja2.Environment(
       loader=jinja2.FileSystemLoader(os.path.dirname(__file__)),
       extensions=['jinja2.ext.autoescape'],
       autoescape=True)

Manipulador de hormigón:

class MainPage(webapp2.RequestHandler):
   def get(self):

       user = users.get_current_user()
       if user:
           url = users.create_logout_url(self.request.uri)

Puede incluir más lógica aquí para usuarios

       else:
           url = users.create_login_url(self.request.uri)

Plantillas para pasar información usando jinja2. Para este ejemplo, el objeto de usuario y la cadena de URL.

       template_values = {
           'user': user,
           'url': url,
       }

       JINJA_ENVIRONMENT.add_extension('jinja2.ext.do')

Usando el ejemplo de index.html. [página html tradicional]

       template = JINJA_ENVIRONMENT.get_template('index.html')
       self.response.write(template.render(template_values))

Enrutamiento de aplicaciones [urls.py] #

Usé para este ejemplo webapp2 para cubrir el enrutamiento.

from webapp2_extras.routes import RedirectRoute as Route

Importar desde vistas:

from views import MainPage

MainPage es el controlador establecido en la raíz “/”:

urlpatterns = [
   Route('/', MainPage),
]

Html, ejemplo de interfaz de cómo usar la API de usuarios [index.html] #

Extracto simple de index.html:

    <div class="sign-in">

        {% if user %}

[Al pasar la url tenemos la oportunidad de cerrar la sesión del usuario]

            <a href="{{ url|safe }}">LOG OUT</a>

[Puede incluir aquí operaciones para usuarios autenticados]

        {% else %}

[Pasando la url tenemos la oportunidad de ingresar al usuario]

            <a href="{{ url|safe }}">SIGN IN</a>

        {% endif %}
    </div>

Este es un ejemplo simple de cómo se usa la operación en la página index.html.
