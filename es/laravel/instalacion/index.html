Instalación #

Las aplicaciones de Laravel se instalan y administran con Composer, un popular administrador de dependencias de PHP. Hay dos formas de crear una nueva aplicación Laravel.

A través del compositor #

$ composer create-project laravel/laravel [foldername]

O

$ composer create-project --prefer-dist laravel/laravel [foldername]

Reemplace [nombre de la carpeta] con el nombre del directorio en el que desea instalar su nueva aplicación Laravel. No debe existir antes de la instalación. Es posible que también deba agregar el ejecutable Composer a la ruta de su sistema.

Si desea crear un proyecto de Laravel utilizando una versión específica del marco, puede proporcionar un patrón de versión; de lo contrario, su proyecto utilizará la última versión disponible.

Si quisiera crear un proyecto en Laravel 5.2, por ejemplo, ejecutaría:

$ composer create-project --prefer-dist laravel/laravel 5.2.*

Por qué –prefer-dist

Hay dos formas de descargar un paquete: source y dist. Para versiones estables, Composer usará dist por defecto. La fuente es un repositorio de control de versiones. Si --prefer-source está habilitado, Composer se instalará desde la fuente, si hay una.

--prefer-dist es lo opuesto a --prefer-source, y le dice a Composer que instale desde dist si es posible. Esto puede acelerar sustancialmente las instalaciones en servidores de compilación y en otros casos de uso en los que normalmente no ejecuta actualizaciones de proveedores. También permite evitar problemas con Git si no tienes una configuración adecuada.

A través del instalador de Laravel #

Laravel proporciona una útil utilidad de línea de comandos para crear rápidamente aplicaciones Laravel. Primero, instale el instalador:

$ composer global require laravel/installer

Tienes que asegurarte de que la carpeta de archivos binarios de Composer esté dentro de tu variable $PATH para ejecutar el instalador de Laravel.

Primero, mira si ya está en tu variable $PATH

echo $PATH

Si todo es correcto, la salida debería contener algo como esto:

Usuarios/tunombredeusuario/.compositor/proveedor/bin

Si no, edite su .bashrc o, si está usando ZSH, su .zshrc para que contenga la ruta a su directorio de proveedores de Composer.

Una vez instalado, este comando creará una nueva instalación de Laravel en el directorio que especifique.

laravel new [foldername]

También puede usar . (un punto) en lugar de [nombre de carpeta] para crear el proyecto en el directorio de trabajo actual sin crear un subdirectorio.

Ejecutando la aplicación #

Laravel viene con un servidor web basado en PHP que se puede iniciar ejecutando

$ php artisan serve

De manera predeterminada, el servidor HTTP usará el puerto 8000, pero si el puerto ya está en uso o si desea ejecutar varias aplicaciones de Laravel a la vez, puede usar el indicador --port para especificar un puerto diferente:

$ php artisan serve --port=8080

El servidor HTTP usará localhost como el dominio predeterminado para ejecutar la aplicación, pero puede usar el indicador --host para especificar una dirección diferente:

$ php artisan serve --host=192.168.0.100 --port=8080

Usando un servidor diferente #

Si prefiere utilizar un software de servidor web diferente, se le proporcionan algunos archivos de configuración dentro del directorio público de su proyecto; .htaccess para Apache y web.config para ASP.NET. Para otro software como NGINX, puede convertir las configuraciones de Apache usando varias herramientas en línea.

El marco necesita que el usuario del servidor web tenga permisos de escritura en los siguientes directorios:

  • /almacenamiento
  • /arranque/caché

En los sistemas operativos *nix, esto se puede lograr mediante

chown -R www-data:www-data storage bootstrap/cache
chmod -R ug+rwx storage bootstrap/cache

(donde www-data es el nombre y grupo del usuario del servidor web)

El servidor web de su elección debe configurarse para servir contenido desde el directorio /public de su proyecto, lo que generalmente se hace configurándolo como la raíz del documento. El resto de su proyecto no debe ser accesible a través de su servidor web.

Si configuró todo correctamente, navegar a la URL de su sitio web debería mostrar la página de destino predeterminada de Laravel.

Requisitos #

El marco de Laravel tiene los siguientes requisitos:

  • PHP >= 5.6.4
  • Extensión PHP XML
  • Extensión PHP PDO
  • Extensión PHP OpenSSL
  • Extensión PHP Mbstring
  • Extensión PHP tokenizadora
  • PHP >= 5.5.9
  • Extensión PHP PDO
  • Laravel 5.1 es la primera versión de Laravel compatible con PHP 7.0.
  • PHP >= 5.4, PHP < 7
  • Extensión PHP OpenSSL
  • Extensión PHP tokenizadora
  • Extensión PHP Mbstring
  • Extensión JSON PHP (solo en PHP 5.5)
  • PHP >= 5.4
  • Extensión PHP Mbstring
  • Extensión JSON PHP (solo en PHP 5.5)

Ejemplo de Hello World (Básico) #

Abrir archivo de rutas. Pegue el siguiente código en:

Route::get('helloworld', function () {
    return '<h1>Hello World</h1>';
});

después de ir a la ruta localhost/helloworld muestra Hello World.

El archivo de rutas se encuentra:

para Internet

routes/web.php

Para API

routes/api.php

app/Http/routes.php

app/routes.php

Ejemplo de Hello World (usando el controlador y la vista) #

  1. Crea una aplicación Laravel:

     $ composer create-project laravel/laravel hello-world

  2. Navegue a la carpeta del proyecto, p.

     $ cd C:\xampp\htdocs\hello-world

  3. Cree un controlador:

     $ php artisan make:controller HelloController --resource

Esto creará el archivo app/Http/Controllers/HelloController.php. La opción --resource generará métodos CRUD para el controlador, p. indexar, crear, mostrar, actualizar.

  1. Registre una ruta al método index de HelloController. Agregue esta línea a app/Http/routes.php (versión 5.0 a 5.2) o routes/web.php (versión 5.3):
    Route::get('hello', 'HelloController@index');

Para ver sus rutas recién agregadas, puede ejecutar $ php artesanal ruta: lista

  1. Cree una plantilla Blade en el directorio views:

    resources/views/hello.blade.php:

     <h1>Hello world!</h1>

  2. Ahora le decimos al método index que muestre la plantilla hello.blade.php:

    app/Http/Controllers/HelloController.php

    <?php
    
    namespace App\Http\Controllers;
    
    use Illuminate\Http\Request;
    
    use App\Http\Requests;
    
    class HelloController extends Controller
    {
        /**
         * Display a listing of the resource.
         *
         * @return \Illuminate\Http\Response
         */
        public function index()
        {
            return view('hello');
        }
    
        // ... other resources are listed below the index one above

Puede servir su aplicación usando el siguiente comando de PHP Artisan: php artesanal serve; le mostrará la dirección en la que puede acceder a su aplicación (generalmente en http://localhost:8000 por defecto).

Alternativamente, puede dirigirse directamente a la ubicación adecuada en su navegador; en caso de que estés usando un servidor como XAMPP (ya sea: http://localhost/hello-world/public/hello deberías haber instalado tu instancia de Laravel, hello-world, directamente en tu xampp/htdocs directorio como en: habiendo ejecutado el paso 1 de este Hello Word desde su interfaz de línea de comandos, apuntando a su directorio xampp/htdocs).

Instalación usando LaraDock (Laravel Homestead para Docker) #

LaraDock es un entorno de desarrollo similar a Laravel Homestead, pero para Docker en lugar de Vagrant. https://github.com/LaraDock/laradock

Instalación #

*Requiere Git y Docker

Clona el repositorio de LaraDock:

R. Si ya tiene un proyecto de Laravel, clone este repositorio en su directorio raíz de Laravel:

git submodule add https://github.com/LaraDock/laradock.git

B. Si no tiene un proyecto de Laravel y desea instalar Laravel desde Docker, clone este repositorio en cualquier lugar de su máquina:

git clone https://github.com/LaraDock/laradock.git

Uso básico #

  1. Ejecute Contenedores: (Asegúrese de estar en la carpeta laradock antes de ejecutar los comandos docker-compose).

Ejemplo: ejecutar NGINX y MySQL: docker-compose up -d nginx mysql

Hay una lista de contenedores disponibles que puede seleccionar para crear sus propias combinaciones.

nginx, hhvm, php-fpm, mysql, redis, postgres, mariadb, neo4j, mongo, apache2, caddy, memcached, beanstalkd, beanstalkd-console, espacio de trabajo

  1. Ingrese al contenedor Workspace, para ejecutar comandos como (Artisan, Composer, PHPUnit, Gulp, …).

    docker-compose exec workspace bash

  2. Si aún no tiene un proyecto Laravel instalado, siga el paso para instalar Laravel desde un contenedor Docker.

una. Ingrese al contenedor del espacio de trabajo.

b. Instala Laravel. compositor create-project laravel/laravel my-cool-app "5.3.*"

  1. Edite las configuraciones de Laravel. Abra el archivo .env de su Laravel y configure DB_HOST en su mysql:

    DB_HOST=mysql

  2. Abra su navegador y visite su dirección localhost.
