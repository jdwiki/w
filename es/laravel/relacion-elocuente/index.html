Inserción de modelos relacionados #

Supongamos que tiene un modelo Post con una relación hasMany con Comment. Puede insertar un objeto ‘Comentario’ relacionado con una publicación haciendo lo siguiente:

$post = Post::find(1);

$commentToAdd = new Comment(['message' => 'This is a comment.']);

$post->comments()->save($commentToAdd);

Puede guardar varios modelos a la vez usando la función saveMany:

$post = Post::find(1);

$post->comments()->saveMany([
    new Comment(['message' => 'This a new comment']),
    new Comment(['message' => 'Me too!']),
    new Comment(['message' => 'Eloquent is awesome!'])
]);

Alternativamente, también hay un método crear que acepta una matriz simple de PHP en lugar de una instancia del modelo Eloquent.

$post = Post::find(1);

$post->comments()->create([
    'message' => 'This is a new comment message'
]);

Tipos de relaciones #

Uno a muchos #

Digamos que cada publicación puede tener uno o varios comentarios y cada comentario pertenece a una sola publicación.

por lo que la tabla de comentarios tendrá post_id. En este caso las relaciones serán las siguientes.

Modelo de publicación

public function comments()
{
   return $this->belongsTo(Post::class);
}

Si la clave externa no es post_id, por ejemplo, la clave externa es example_post_id.

public function comments()
{
   return $this->belongsTo(Post::class, 'example_post_id');
}

y además, si la clave local no es id, por ejemplo, la clave local es other_id

public function comments()
{
   return $this->belongsTo(Post::class, 'example_post_id', 'other_id');
}

Comentario Modelo

definición inversa de uno a muchos

public function post()
{
   return $this->hasMany(Comment::class);
}

Cara a cara #

Cómo asociar entre dos modelos (ejemplo: modelo Usuario y Teléfono) #

Aplicación\Usuario

<?php

namespace App;

use Illuminate\Database\Eloquent\Model;

class User extends Model
{
    /**
     * Get the phone record associated with the user.
     */
    public function phone()
    {
        return $this->hasOne('Phone::class', 'foreign_key', 'local_key');
    }
}

Aplicación\Teléfono

<?php

namespace App;

use Illuminate\Database\Eloquent\Model;

class Phone extends Model
{
    /**
     * Get the user that owns the phone.
     */
    public function user()
    {
        return $this->belongsTo('User::class', 'foreign_key', 'local_key');
    }
}

foreign_key: por defecto, Eloquent asumirá que este valor es other_model_name_id (en este caso user_id y phone_id), cámbielo si no es el caso.

local_key: por defecto, Eloquent asumirá que este valor es id (clave principal del modelo actual), cámbielo si no es el caso.

Si su base de datos archivó el nombre según el estándar de laravel, no necesita proporcionar una clave externa y una clave local en la declaración de relación

Explicación #

Muchos a muchos #

Digamos que hay roles y permisos. Cada rol puede pertenecer a muchos permisos y cada permiso puede pertenecer a muchos roles. por lo que habrá 3 mesas. dos modelos y una mesa pivotante. una tabla roles, users y permission_role.

Modelo a seguir

public function permissions()
{
   return $this->belongsToMany(Permission::class);
}

Modelo de permiso

public function roles()
{
   return $this->belongsToMany(Roles::class);
}

Nota 1

considere seguir mientras usa un nombre de tabla diferente para la tabla dinámica.

Suponga que desea usar role_permission en lugar de permission_role, ya que el elocuente uso del orden alfabético para crear los nombres de las claves dinámicas. deberá pasar el nombre de la tabla dinámica como segundo parámetro de la siguiente manera.

Modelo a seguir

public function permissions()
{
   return $this->belongsToMany(Permission::class, 'role_permission');
}

Modelo de permiso

public function roles()
{
   return $this->belongsToMany(Roles::class, 'role_permission');
}

Nota 2

considere seguir mientras usa diferentes nombres clave en la tabla dinámica.

Eloquent asume que si no se pasan claves como tercer y cuarto parámetro, serán los nombres de tabla singulares con _id. por lo que asume que el pivote tendrá campos role_id y permission_id. Si se van a utilizar claves distintas a estas, se deben pasar como tercer y cuarto parámetro.

Digamos si se va a utilizar other_role_id en lugar de role_id y other_permission_id en lugar de permission_id. Entonces sería de la siguiente manera.

Modelo a seguir

public function permissions()
{
   return $this->belongsToMany(Permission::class, 'role_permission', 'other_role_id', 'other_permission_id');
}

Modelo de permiso

public function roles()
{
   return $this->belongsToMany(Roles::class, 'role_permission', 'other_permission_id', 'other_role_id');
}

Polimórfico #

Las relaciones polimórficas permiten que un modelo pertenezca a más de otro modelo en una sola asociación. Un buen ejemplo serían las imágenes, tanto un usuario como un producto pueden tener una imagen. La estructura de la tabla podría tener el siguiente aspecto:

user
    id - integer
    name - string
    email - string

product
    id - integer
    title - string
    SKU - string

image
    id - integer
    url - string
    imageable_id - integer
    imageable_type - string

Las columnas importantes a mirar están en la tabla de imágenes. La columna imageable_id contendrá el valor de ID del usuario o producto, mientras que la columna imageable_type contendrá el nombre de clase del modelo propietario. En sus modelos, configura las relaciones de la siguiente manera:

<?php

namespace App;

use Illuminate\Database\Eloquent\Model;

class Image extends Model
{
    /**
     * Get all of the owning imageable models.
     */
    public function imageable()
    {
        return $this->morphTo();
    }
}

class User extends Model
{
    /**
     * Get all of the user's images.
     */
    public function images()
    {
        return $this->morphMany('Image::class', 'imageable');
    }
}

class Product extends Model
{
    /**
     * Get all of the product's images.
     */
    public function images()
    {
        return $this->morphMany('Image::class', 'imageable');
    }
}

También puede recuperar el propietario de una relación polimórfica del modelo polimórfico accediendo al nombre del método que realiza la llamada a morphTo. En nuestro caso, ese es el método imagenable en el modelo de imagen. Entonces, accederemos a ese método como una propiedad dinámica.

$image = App\Image::find(1);

$imageable = $image->imageable;

Este imagenable devolverá un Usuario o un Producto.

Consultando sobre relaciones #

Eloquent también le permite consultar relaciones definidas, como se muestra a continuación:

User::whereHas('articles', function (Builder $query) {
    $query->where('published', '!=', true);
})->get();

Esto requiere que el nombre de su método de relación sea artículos en este caso. El argumento que se pasa al cierre es el Generador de consultas para el modelo relacionado, por lo que puede usar aquí cualquier consulta que pueda en cualquier otro lugar.

Carga ansiosa

Suponga que el modelo de usuario tiene una relación con el modelo de artículo y desea cargar con entusiasmo los artículos relacionados. Esto significa que los artículos del usuario se cargarán mientras se recupera el usuario.

artículos es el nombre de la relación (método) en el modelo de usuario.

User::with('articles')->get();

si tienes una relación múltiple. por ejemplo, artículos y publicaciones.

User::with('articles','posts')->get();

y para seleccionar relaciones anidadas

User::with('posts.comments')->get();

Llame a más de una relación anidada

User::with('posts.comments.likes')->get()

Accediendo a la tabla intermedia usando withPivot()

Suponga que tiene una tercera columna ‘permission_assigned_date’ en la tabla dinámica. De forma predeterminada, solo las claves del modelo estarán presentes en el objeto pivote. Ahora, para obtener esta columna en el resultado de la consulta, debe agregar el nombre en la función withPivot().

   public function permissions()
        {
           return $this->belongsToMany(Permission::class, 'role_permission', 'other_role_id', 'other_permission_id')->withPivot('permission_assigned_date');
        }

Adjuntar / Separar

Eloquent también proporciona algunos métodos auxiliares adicionales para que trabajar con modelos relacionados sea más conveniente. Por ejemplo, imaginemos que un usuario puede tener muchos roles y un rol puede tener muchos permisos. Para adjuntar un rol a un permiso mediante la inserción de un registro en la tabla intermedia que une los modelos, utilice el método de conexión:

$role= App\Role::find(1);    
$role->permissions()->attach($permissionId);

Al adjuntar una relación a un modelo, también puede pasar una matriz de datos adicionales para insertarlos en la tabla intermedia:

$rol->roles()->attach($permissionId, ['permission_assigned_date' => $date]);

De manera similar, para eliminar un permiso específico contra un rol, use la función de separación

$role= App\Role::find(1);
//will remove permission 1,2,3 against role 1
$role->permissions()->detach([1, 2, 3]);

Sincronización de asociaciones

También puede usar el método de sincronización para construir asociaciones de muchos a muchos. El método de sincronización acepta una matriz de ID para colocar en la tabla intermedia. Cualquier ID que no esté en la matriz dada se eliminará de la tabla intermedia. Entonces, después de que se complete esta operación, solo existirán las ID en la matriz dada en la tabla intermedia:

//will keep permission id's 1,2,3 against Role id 1

$role= App\Role::find(1)
$role->permissions()->sync([1, 2, 3]);

Introducción #

Las relaciones de Eloquent se definen como funciones en las clases de su modelo Eloquent. Dado que, al igual que los propios modelos de Eloquent, las relaciones también sirven como poderosos generadores de consultas, definir las relaciones como funciones proporciona poderosas capacidades de consulta y encadenamiento de métodos. Por ejemplo, podemos encadenar restricciones adicionales en esta relación de publicaciones:

$user->posts()->where('active', 1)->get();
