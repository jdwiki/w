Valet es un entorno de desarrollo hecho a medida para macOS. Abstrae la necesidad de máquinas virtuales, Homestead o Vagrant. Ya no es necesario actualizar constantemente su archivo /etc/hosts. Incluso puede compartir sus sitios públicamente usando túneles locales.

Laravel Valet hace que todos los sitios estén disponibles en un dominio *.dev vinculando los nombres de las carpetas a los nombres de los dominios.

Sintaxis #

  • comando valet [opciones] [argumentos]

Parámetros #

Parámetro Conjunto de valores
comando [dominio](https://www.wikiod.com/es/laravel/ayudante-de-camara#Dominio de valet), buscar-compartir-url, olvidar, ayudar, instalar, [enlace](http://stackoverflow.com /documentation/laravel/1906/valet/6218/valet-link#t=201607272250304319463), [enlaces](https://www.wikiod.com/es/laravel/ayudante-de-camara#Enlaces de valet) , lista, registros, en la última versión, abierto, [parque] (https://www.wikiod.com/es/laravel/ayudante-de-camara#Servicio de aparcacoches), rutas, reinicio, seguro, iniciar, detener, desinstalar, desvincular, inseguro, que
opciones -h, –ayuda, -q, –quiet, -V, –versión, –ansi, –no-ansi, -n, –sin interacción, -v, -vv, -vvv,- -prolijo
argumentos (opcional)

Debido a que Valet para Linux y Windows no es oficial, no habrá soporte fuera de sus respectivos repositorios de Github.

Enlace de servicio de valet #

Este comando es útil si desea servir un solo sitio en un directorio y no todo el directorio.

cd ~/Projects/my-blog/
valet link awesome-blog

Valet creará un enlace simbólico en ~/.valet/Sites que apunta a su directorio de trabajo actual. Después de ejecutar el comando de enlace, puede acceder al sitio en su navegador en http://awesome-blog.dev.

Para ver una lista de todos sus directorios vinculados, ejecute el comando valet links. Puede usar valet unlink awesome-blog para destruir el enlace simbólico.

1: https://www.wikiod.com/es/laravel/ayudante-de-camara#Enlaces de valet

Enlaces de valet #

Este comando mostrará todos los enlaces Valet registrados que ha creado y sus rutas de archivo correspondientes en su computadora.

Dominio:

valet links

Salida de muestra:

...
site1 -> /path/to/site/one
site2 -> /path/to/site/two
...

Nota 1: Puede ejecutar este comando desde cualquier lugar, no solo desde una carpeta vinculada.

Nota 2: Los sitios aparecerán en la lista sin la terminación .dev, pero igual usará site1.dev para acceder a su aplicación desde el navegador.

Servicio de aparcacoches #

cd ~/Projects
valet park

Este comando registrará su directorio de trabajo actual como una ruta en la que Valet debe buscar sitios. Ahora, cualquier proyecto de Laravel que cree dentro de su directorio “estacionado” se servirá automáticamente usando la convención http://folder-name.dev.

Instalación #

IMPORTANTE!! Valet es una herramienta diseñada solo para macOS.

Requisitos previos

  • Valet utiliza el puerto HTTP de su máquina local (puerto 80), por lo tanto, no podrá usarlo si Apache o Nginx están instalados y ejecutándose en la misma máquina.
  • Se requiere el administrador de paquetes no oficial de macOS Homebrew para usar Valet correctamente.
  • Asegúrate de que Homebrew esté actualizado a la última versión ejecutando brew update en la terminal.

Instalación

  • Instale PHP 7.1 usando Homebrew a través de brew install homebrew/php/php71.
  • Instale Valet con Composer a través de composer global require laravel/valet.
  • Agregue el directorio ~/.composer/vendor/bin a la “RUTA” de su sistema si aún no está allí.
  • Ejecute el comando valet install.

Después de la instalación Durante el proceso de instalación, Valet instaló DnsMasq. También registró el demonio de Valet para que se inicie automáticamente cuando se inicia el sistema, por lo que no es necesario que ejecute valet start o valet install cada vez que reinicie su máquina.

Dominio de valet #

Este comando le permite cambiar o ver el TLD (dominio de nivel superior) utilizado para vincular dominios a su máquina local.

Obtenga el TLD actual

$ valet domain
> dev

Establecer el TLD

$ valet domain local
> Your Valet domain has been updated to [local].

Instalación (Linux) #

¡¡IMPORTANTE!! Valet es una herramienta diseñada para macOS, la versión a continuación está adaptada para el sistema operativo Linux.

Requisitos previos

  • No instale valet como root o usando el comando sudo.
  • Valet utiliza el puerto HTTP de su máquina local (puerto 80), por lo tanto, no podrá usarlo si Apache o Nginx están instalados y ejecutándose en la misma máquina.
  • Se requiere una versión actualizada de composer para instalar y ejecutar Valet.

Instalación

  • Ejecute composer global require cpriego/valet-linux para instalar Valet globalmente.
  • Ejecute el comando valet install para finalizar la instalación.

Después de la instalación

Durante el proceso de instalación, Valet instaló DnsMasq. También registró el demonio de Valet para que se inicie automáticamente cuando se inicia el sistema, por lo que no es necesario que ejecute valet start o valet install cada vez que reinicie su máquina.
