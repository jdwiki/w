Laravel es compatible con el motor de plantillas Blade desde el primer momento. El motor de plantillas Blade nos permite crear plantillas maestras y plantillas secundarias cargando contenido desde plantillas maestras, podemos tener variables, bucles y declaraciones condicionales dentro del archivo blade.

Vistas: Introducción #

Las vistas, en un patrón MVC, contienen la lógica sobre cómo presentar los datos al usuario. En una aplicación web, normalmente se utilizan para generar la salida HTML que se envía a los usuarios con cada respuesta. De forma predeterminada, las vistas en Laravel se almacenan en el directorio resources/views.

Se puede llamar a una vista usando la función auxiliar view:

view(string $path, array $data = [])

El primer parámetro del asistente es la ruta a un archivo de vista y el segundo parámetro es una matriz opcional de datos para pasar a la vista.

Por lo tanto, para llamar a resources/views/example.php, usaría:

view('example');

Ver archivos en subcarpetas dentro del directorio resources/views, como resources/views/parts/header/navigation.php, se puede llamar usando la notación de puntos: view('parts.header.navigation');

Dentro de un archivo de vista, como resources/views/example.php, puede incluir HTML y PHP juntos:

<html>
    <head>
        <title>Hello world!</title>
    </head>
    <body>
        <h1>Welcome!</h1>
        <p>Your name is: <?php echo $name; ?></p>
    </body>
</html>

En el ejemplo anterior (que no usa ninguna sintaxis específica de Blade), generamos la variable $name. Para pasar este valor a nuestra vista, pasaríamos una serie de valores al llamar al asistente de vista:

view('example', ['name' => $name]);

o alternativamente, use el ayudante compact(). En este caso, la cadena que se pasa a compact() corresponde al nombre de la variable que queremos pasar a la vista.

view('example', compact('name'));

CONVENCIÓN DE NOMBRE PARA LAS VARIABLES DE HOJA

Mientras envía datos de vuelta a la vista. Puede usar guión bajo para variable de varias palabras, pero con - laravel da error.

Al igual que este, dará un error (observe guión (-) dentro de la dirección de usuario

view('example',['user-address' => 'Some Address']);

La manera correcta de hacer esto será

view('example', ['user_address' => 'Some Address']);

Herencia de diseño #

Un diseño es un archivo de vista, que se amplía con otras vistas que inyectan bloques de código en su padre. Por ejemplo:

padre.blade.php:

<html>
    <head>
        <style type='text/css'>
        @yield('styling')
        </style>
    </head>
    <body>
        <div class='main'>
        @yield('main-content')
        </div>
    </body>
</html>

niño.blade.php:

@extends('parent')

@section('styling')
.main {
    color: red;
}
@stop

@section('main-content')
This is child page!
@stop

otrapagina.blade.php:

@extends('parent')

@section('styling')
.main {
    color: blue;
}
@stop

@section('main-content')
This is another page!
@stop

Aquí puede ver dos páginas secundarias de ejemplo, cada una de las cuales amplía la principal. Las páginas secundarias definen una @sección, que se inserta en la principal en la instrucción @yield apropiada.

Entonces, la vista representada por View::make('child') dirá “¡Esta es una página secundaria!” en rojo, mientras que View::make('otherpage') producirá el mismo html, excepto con el texto “¡Esta es otra página!” en azul.

Es común separar los archivos de vista, p. tener una carpeta de diseños específicamente para los archivos de diseño y una carpeta separada para las diversas vistas individuales específicas.

Los diseños están destinados a aplicar el código que debería aparecer en cada página, p. agregando una barra lateral o un encabezado, sin tener que escribir todo el código repetitivo html en cada vista individual.

Las vistas se pueden extender repetidamente, es decir, página3 puede @extend('página2'), y página2 puede @extend('página1').

El comando extender usa la misma sintaxis que se usa para View::make y @include, por lo que se accede al archivo layouts/main/page.blade.php como layouts.main.page.

Estructuras de Control #

Blade proporciona una sintaxis conveniente para estructuras de control de PHP comunes.

Cada una de las estructuras de control comienza con @[estructura] y termina con @[estructura final]. Tenga en cuenta que dentro de las etiquetas, solo estamos escribiendo HTML normal e incluyendo variables con la sintaxis Blade.

Condicionales #

Declaraciones ‘si’ #

@if ($i > 10)
    <p>{{ $i }} is large.</p>
@elseif ($i == 10)
    <p>{{ $i }} is ten.</p>
@else
    <p>{{ $i }} is small.</p>
@endif

Declaraciones ‘a menos que’ #

(Sintaxis abreviada para ‘si no’.)

@unless ($user->hasName())
    <p>A user has no name.</p>
@endunless

Bucles #

Bucle ‘Mientras’ #

@while (true)
    <p>I'm looping forever.</p>
@endwhile

Bucle ‘Foreach’ #

@foreach ($users as $id => $name)
    <p>User {{ $name }} has ID {{ $id }}.</p>
@endforeach

Bucle ‘Forel’ #

(Igual que el ciclo ‘foreach’, pero agrega una directiva especial ‘@empty’, que se ejecuta cuando la expresión de matriz iterada está vacía, como una forma de mostrar el contenido predeterminado).

@forelse($posts as $post)
    <p>{{ $post }} is the post content.</p>
@empty
    <p>There are no posts.</p>
@endforelse

Dentro de los bucles, estará disponible una variable $loop especial, que contiene información sobre el estado del bucle:

Propiedad Descripción
$bucle->índice El índice de la iteración del bucle actual (comienza en 0).
$bucle->iteración La iteración del bucle actual (comienza en 1).
$bucle->restante Las iteraciones de bucle restantes.
$bucle->cuenta El número total de elementos en la matriz que se itera.
$bucle->primero Si esta es la primera iteración a través del ciclo.
$bucle->último Si esta es la última iteración a través del bucle.
$bucle->profundidad El nivel de anidamiento del bucle actual.
$bucle->padre Cuando está en un bucle anidado, la variable de bucle del padre.

Ejemplo:

@foreach ($users as $user)
  @foreach ($user->posts as $post)
        @if ($loop->parent->first)
            This is first iteration of the parent loop.
        @endif
    @endforeach
@endforeach

Desde Laravel 5.2.22, también podemos usar las directivas @continue y @break

Propiedad Descripción
@continuar Detener la iteración actual y comenzar la siguiente.
@romper Detener el bucle actual.

Ejemplo :

@foreach ($users as $user)
    @continue ($user->id == 2)
        <p>{{ $user->id }} {{ $user->name }}</p>
    @break ($user->id == 4)
@endforeach

Luego (suponiendo que más de 5 usuarios estén ordenados por ID y no falte ningún ID) la página se mostrará

1 Dave
3 John
4 William

Haciéndose eco de expresiones PHP #

Cualquier expresión de PHP entre llaves dobles {{ $variable }} se repetirá después de ejecutarse a través de la función auxiliar e. (Así que los caracteres especiales html (<, >, ", ', &) se reemplazan de forma segura por las entidades html correspondientes). (La expresión de PHP debe evaluarse como una cadena; de lo contrario, se generará una excepción).

Haciendo eco de una variable #

{{ $variable }}

Hacer eco de un elemento en una matriz #

{{ $array["key"] }}

Hacer eco de una propiedad de objeto #

{{ $object->property }}

Haciéndose eco del resultado de una llamada de función #

{{ strtolower($variable) }}

Comprobación de existencia #

Normalmente, en PHP, para verificar si una variable está configurada e imprimirla, haría

  • Antes de PHP 7

  • Después de PHP 7 (usando el “operador coalescente nulo”)

El operador Blade o lo hace más fácil:

{{ $variable or 'Default' }}

Ecos sin procesar #

Como se mencionó, la sintaxis regular de llaves dobles {{ }}, se filtra a través de la función htmlspecialchars de PHP, por seguridad (previniendo la inyección maliciosa de HTML en la vista). Si desea omitir este comportamiento, por ejemplo, si está tratando de generar un bloque de contenido HTML resultante de una expresión PHP, use la siguiente sintaxis:

{!! $myHtmlString !!}

Tenga en cuenta que se considera una mejor práctica usar la sintaxis estándar {{ }} para escapar de sus datos, a menos que sea absolutamente necesario. Además, al hacer eco de contenido que no es de confianza (es decir, contenido proporcionado por los usuarios de su sitio), debe evitar el uso de {!! !!} sintaxis.

Incluidas vistas parciales #

Con Blade, también puede incluir vistas parciales (llamadas ‘parciales’) directamente en una página de esta manera:

@include('includes.info', ['title' => 'Information Station'])

El código anterior incluirá la vista en ‘views/includes/info.blade.php’. También pasará una variable $título con el valor ‘Estación de información’.

En general, una página incluida tendrá acceso a cualquier variable a la que tenga acceso la página que llama. Por ejemplo, si tenemos:

{{$user}} // Outputs 'abc123'
@include('includes.info')

Y ‘includes/info.blade.php’ tiene lo siguiente:

<p>{{$user}} is the current user.</p>

Entonces la página se renderizará:

abc123
abc123 is the current user.

Incluir cada uno

A veces, querrá combinar una instrucción include con una declaración foreach y acceder a las variables desde dentro del ciclo foreach en el include. En este caso, use la directiva @each de Blade:

@each('includes.job', $jobs, 'job')

El primer parámetro es la página a incluir. El segundo parámetro es la matriz sobre la que iterar. El tercer parámetro es la variable asignada a los elementos del arreglo. La declaración anterior es equivalente a:

@foreach($jobs as $job)
    @include('includes.job', ['job' => $job])
@endforeach

También puede pasar un cuarto argumento opcional a la directiva @each para especificar la vista que se mostrará cuando la matriz esté vacía.

@each('includes.job', $jobs, 'job', 'includes.jobsEmpty')

Ejecutar código PHP arbitrario #

Aunque puede que no sea apropiado hacer tal cosa en una vista si tiene la intención de separar estrictamente las preocupaciones, la directiva php Blade permite una forma de ejecutar código PHP, por ejemplo, para establecer una variable:

@php($varName = 'Enter content ')

(igual que:)

@php
    $varName = 'Enter content ';
@endphp

luego:

{{ $varName }}
