Archivos por lotes con más de 9 argumentos #

Cuando se proporcionan más de 9 argumentos, se puede usar el comando shift [/n], donde /n significa comenzar en el argumento n, n está entre cero y ocho.

Recorriendo argumentos:

:args
set /a "i+=1"
set arg!i!=%~1
call echo arg!i! = %%arg!i!%%
shift
goto :args

Tenga en cuenta que en el ejemplo anterior, la variable de expansión retrasada i se usa para asignar valores de argumento a la matriz de variables. El comando call permite mostrar tales valores variables dentro del ciclo.

Argumentos de conteo:

for %%i in (%*) do (set /a ArgCount+=1)
echo %ArgCount%

Establecer una variable en el argumento n:

set i=5
call set "path%i%=%%~i"

Argumentos de la línea de comandos proporcionados a los archivos por lotes #

Los argumentos de la línea de comandos del archivo por lotes son valores de parámetros enviados al iniciar el lote. Deben estar entre comillas si contienen espacios. En un archivo por lotes en ejecución, los argumentos se utilizan para varios propósitos, es decir, la redirección a :etiquetas, establecer variables o ejecutar comandos.

Se hace referencia a los argumentos en el archivo por lotes mediante %1, %2, ..., %9.

@echo off
setlocal EnableDelayedExpansion
if not "%1"=="" (
    set "dir=%~1" & set "file=%~2"
    type !dir!\!file! | find /n /i "True" >nul^
     && echo Success! || echo Failure
)
exit /b

C:\Users\UserName> test.bat "C:\Temp\Test Results" "Latest.log"
Success!

Notas:

  • En el ejemplo anterior, las comillas dobles se eliminan utilizando el modificador de argumento %~1.
  • Las cadenas largas se dividen en varias líneas usando ^ , y hay un espacio antes del carácter en la siguiente línea.

Cambiando argumentos dentro de paréntesis #

Tengamos el siguiente example.bat y llamémoslo con los argumentos 1, 2 y 3:

@echo off

(
    shift
    shift
    echo %1
)

Como la expansión variable cambiará después de que se alcance el contexto de los corchetes finales, la salida será:

1

Como esto podría ser un problema al cambiar entre paréntesis para acceder al argumento, deberá usar la llamada:

@echo off

(
    shift
    shift
    call echo %%1
)

ahora la salida será 3. Como se usa el comando CALL (lo que conducirá a una expansión de variable adicional), con esta técnica también se pueden parametrizar los argumentos que acceden:

@echo off

set argument=1

    shift
    shift
    call echo %%%argument%

con expansión retardada:

@echo off
setlocal enableDelayedExpansion
set argument=1

    shift
    shift
    call echo %%!argument!

la salida será

3
