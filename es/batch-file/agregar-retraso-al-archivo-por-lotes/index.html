Este tema le enseñará una de las muchas cosas útiles que debe saber sobre el lenguaje de secuencias de comandos, el archivo por lotes; Agregar un retraso/pausa/tiempo de espera a su archivo por lotes.

Se acabó el tiempo #

Se acabó el tiempo #

La forma más sencilla de hacer un retraso o una pausa durante un cierto período de tiempo es con el comando estándar TIMEOUT. Para hacer un tiempo de espera que dure exactamente un minuto, escribimos:

timeout /t 60

Ahora, ¿qué está pasando aquí?

En primer lugar, usamos el comando TIMEOUT con el parámetro /T (que simplemente significa tiempo de espera), luego especificamos la cantidad de segundos de espera. En este caso… 60 segundos.

Tiempo de espera con el parámetro /NOBREAK

Si tomamos el ejemplo anterior y lo ejecutamos en un archivo BATCH: timeout /t 60, mientras espera esos 60 segundos, en realidad puede romper el tiempo de espera presionando cualquier tecla de su teclado. Para evitar esto simplemente agregamos el parámetro /NOBREAK al final del mismo.

timeout /t 60 /nobreak

Al hacer esto, se agotará el tiempo de espera durante 60 segundos, y si desea romper el tiempo de espera, deberá presionar (CTRL-C) en su teclado.

Tiempo de espera silencioso

Cuando está haciendo un tiempo de espera, se mostrará:

Waiting for X seconds, press a key to continue ...
or 
Waiting for X seconds, press CTRL+C to quit ... [This is with the /NOBREAK parameter]

Para ocultar el mensaje, use el argumento NUL (Para obtener una explicación de NUL: Haga clic aquí)

timeout /t 60 > nul
or
timeout /t 60 /nobreak > nul

Pausa #

Para hacer que su secuencia de comandos se detenga, simplemente use el comando PAUSE.

PAUSE

Esto mostrará el texto Presione cualquier tecla para continuar. . ., luego agregue una nueva línea en la entrada del usuario.

Digamos que queremos crear un programa “Hello World” y después de hacer clic en algo en nuestro teclado, queremos que salga del programa con el comando EXIT.

echo Hello World
pause
exit

Aquí usa el comando ECHO para decir “Hello World”. Luego usamos el comando PAUSA que muestra Presione cualquier tecla para continuar. . . y luego usamos el comando EXIT para finalizar el script BATCH actual.

Cuando esté en pausa, mostrará:

Press any key to continue . . .

** Oculte el mensaje “Presione cualquier tecla para continuar… **

Para ocultar el mensaje, redirigimos la salida a un dispositivo especial llamado nul. En realidad, este no es un dispositivo real, pero cualquier cosa que le enviemos se tira.

pause > nul

Hacer ping #

Silbido #

Uno de los comandos más utilizados para retrasar durante un cierto período de tiempo es ping.

Uso básico

PING -n 1 -w 1000 1.1.1.1

REM the -n 1 flag means to send 1 ping request.
REM the -w 1000 means when the IP(1.1.1.1) does not respond, go to the next command
REM 1.1.1.1 is an non-existing IP so the -w flag can ping a delay and go to next command

Esto generaría lo siguiente en su archivo por lotes/consola:

C:\Foo\Bar\Baz>ping -n -w 1000 1.1.1.1

Pinging 1.1.1.1 (Using 32 bytes of data)
Request timed out

Ping statistics for 1.1.1.1
    Packets: Sent = 2,Received = 0, Lost = 1(100% loss)

Ocultar el texto repetido

Simplemente agregue >nul en la parte posterior del comando para redirigirlo a nulo.

ping -n w 1000 1.1.1.1 >nul

Esto no produciría nada.

Dormir #

Dormir #

En sistemas Windows más antiguos, timeout no está disponible. Sin embargo, podemos usar el comando sleep.

Uso

sleep 1

Muy autoexplicativo; dormir por 1 segundo. Sin embargo, sleep es un comando deshabilitado y debe ser reemplazado por timeout.

Disponibilidad

Este comando está disponible en el antiguo sistema de Windows. También se incluye SLEEP.exe en el kit de recursos de 2003.

Para usar sleep.exe, coloque el archivo ejecutable en la carpeta %Windir%\System32. Entonces puedes usarlo como comando normal.
