echo se puede utilizar para controlar y producir resultados.

Sintaxis #

  • ECO [ENCENDIDO | APAGADO]
  • mensaje de eco
  • ECO (mensaje
  • ECO(

Parámetros #

| Parámetro | Detalles | | ——— | ————————————————– —– | | ENCENDIDO | APAGADO | Puede ser ON o OFF (sin distinción entre mayúsculas y minúsculas) | | mensaje | Cualquier cadena (excepto ON o OFF cuando se usa sin () |

  • echo. también mostrará una cadena vacía. Sin embargo, esto es más lento que echo( ya que echo. buscará un archivo llamado “echo”. Solo si este archivo no existe, el comando funcionará, pero esta verificación lo hace más lento.
  • echo: se comportará como echo(, a menos que message parezca una ruta de archivo, por ejemplo, echo:foo\..\test.bat. En este caso, el intérprete verá echo:foo como nombre de carpeta, elimine echo:foo\..\ (porque aparece solo para ingresar al directorio echo:foo y luego dejarlo nuevamente) luego ejecute test.bat, que no es el comportamiento deseado.

Visualización de mensajes #

Para mostrar “Algo de texto”, use el comando:

echo Some Text

Esto generará la cadena Algo de texto seguida de una nueva línea.

Para mostrar las cadenas On y Off (sin distinción entre mayúsculas y minúsculas) o la cadena vacía, use un ( en lugar de un espacio en blanco:

echo(ON
echo(
echo(off

Esto generará:

ENCENDIDO

apagado

También es común usar echo. para generar una línea en blanco, pero consulte los comentarios para saber por qué esta no es la mejor idea.

Para mostrar texto sin incluir un CR/LF, use el siguiente comando:

<nul set/p=Some Text

Este comando intentará establecer la variable llamada cadena vacía en la entrada del usuario después de un aviso. El archivo nul se redirige al comando con <nul, por lo que el comando se rendirá tan pronto como intente leerlo y solo quedará la cadena de solicitud. Debido a que el usuario nunca escribió una nueva línea, no hay salto de línea.

Salida de eco a archivo #

Formas de crear un archivo con el comando echo:

echo. > example.bat (creates an empty file called "example.bat")

echo message > example.bat (creates example.bat containing "message")
echo message >> example.bat (adds "message" to a new line in example.bat)
(echo message) >> example.bat (same as above, just another way to write it)

Salida a la ruta

Un pequeño problema con el que te puedes encontrar al hacer esto:

echo Hello how are you? > C:\Users\Ben Tearzz\Desktop\example.bat

(This will NOT make a file on the Desktop, and might show an error message)

Pero entonces, ¿cómo lo hacemos? Bueno, en realidad es extremadamente simple… Cuando escriba una ruta o un nombre de archivo que tenga un espacio incluido en su nombre, recuerde usar “comillas”.

echo Hello how are you? > "C:\Users\Ben Tearzz\Desktop\example.bat"
(This will make a file on MY Desktop)

Pero, ¿qué sucede si desea crear un archivo que genere un nuevo archivo?

echo message > file1.bat > example.bat

(This is NOT going to output:
"message > file1.bat" to example.bat

Entonces, ¿cómo hacemos esto?

echo message ^> file1.bat > example.bat

(This will output: 
"message > file1.bat" to example.bat

Lo mismo ocurre con otras cosas en lote

Si no ha aprendido qué son las variables y las declaraciones, lo más probable es que no entienda lo siguiente: haga clic aquí para obtener información sobre las variables | haga clic aquí para aprender sobre declaraciones “si”

Variables:

set example="text"
echo %example% > file.bat
(This will output "text" to file.bat)

si no queremos que genere “texto”, sino simplemente %example%, escriba:

echo ^%example^% > file.bat
(This will output "%example%" to file.bat)

otras declaraciones:

else = ||
if ^%example^%=="Hello" echo True || echo False > file.bat

(This will output:
if %example%=="Hello" echo True

para dar salida a toda la línea escribimos:

if ^%example^%=="Hello" echo True ^|^| echo False > file.bat

This will output:
if %example%=="Hello" echo True || echo False

Si la variable es igual a “Hola”, dirá “Verdadero”, de lo contrario dirá “Falso”

@Echo apagado #

@echo off evita que se muestren el mensaje y el contenido del archivo por lotes, de modo que solo se ve la salida. El @ también oculta la salida del comando echo off.

Configuración de eco #

La configuración de eco determina si la repetición de comandos está activada o desactivada. Así es como se ve un programa de muestra con el comando repitiendo on (predeterminado):

C:\Windows\System32>echo Hello, World!
Hello, World!

C:\Windows\System32>where explorer
C:\Windows\System32\explorer.exe

C:\Windows\System32>exit

Así es como se ve con echo off:

Hello, World!
C:\Windows\System32\explorer.exe

Conseguir y configurar #

Para obtener (mostrar) la configuración de eco, use echo sin parámetros:

> echo
ECHO is on.

Para establecer la configuración de eco, use echo con on o off:

> echo off

> echo
ECHO is off.

> echo on

> echo
ECHO is on.

Tenga en cuenta que con estos ejemplos, el mensaje ha sido representado por > . Al cambiar la configuración de eco, el mensaje aparecerá y desaparecerá, pero eso genera ejemplos poco claros.

Tenga en cuenta que es posible evitar que se repita un comando incluso cuando el eco está activado, colocando un carácter @ antes del comando.

Echo emite todo literalmente #

Las cotizaciones se generarán tal cual:

echo "Some Text"

“Algo de texto”

Los tokens de comentarios se ignoran:

echo Hello World REM this is not a comment because it is being echoed!

¡Hola, mundo, REM, esto no es un comentario porque se está haciendo eco!

Sin embargo, echo aún generará variables; consulte [Uso de variables] (https://www.wikiod.com/es/batch-file/variables-en-archivos-por-lotes#Uso) - y caracteres especiales todavía tiene efecto:

echo hello && echo world

hola mundo

Activar el eco entre paréntesis #

En el siguiente ejemplo, echo on tendrá efecto después de que se alcance el final del contexto de corchetes:

@echo off
(
    echo on
    echo ##
)
echo $$

Para “activar” el eco en un contexto de paréntesis (incluidos los comandos FOR y IF), puede usar FOR /f macro:

@echo off
setlocal

:: echo on macro should followed by the command you want to execute with echo turned on
set "echo_on=echo on&for %%. in (.) do"

(
  %echo_on% echo ###
)
