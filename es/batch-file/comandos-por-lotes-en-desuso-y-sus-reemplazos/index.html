ADMINISTRADOR DE BITS #

BITSADMIN se usó para transferir documentos, descargar sitios web, descargar archivos de sitios web, etc. Este comando está en desuso y es posible que se elimine en las próximas actualizaciones de Windows. Para evitar este problema, utilice el nuevo Powershell BIT cmdlet.

Aquí hay un código de muestra que utiliza BITSADMIN.

@echo off
Bitsadmin /create /download Stackoverflow.com
rem download the website StackOverflow.com

DEPURAR #

El comando DEBUG se usó para crear archivos binarios/ejecutables desde un archivo por lotes. El comando todavía está disponible en versiones de Windows de 32 bits, pero puede crear archivos binarios solo con instrucciones de 16 bits, lo que lo hace inutilizable para máquinas de 64 bits. Ahora CERTUTIL se usa para ese propósito que permite decodificar/codificar archivos binarios/de medios de formatos HEX o BASE64. Ejemplo con un archivo que crea un archivo de icono:

@echo off

del /q /f pointer.jpg >nul 2>nul
certutil -decode "%~f0" pointer.jpg
hh.exe pointer.jpg
exit /b %errorlevel%

ADJUNTAR #

APPEND era un comando en msdos que permitía usar archivos de recursos/medios como si estuvieran en el mismo directorio. El comando todavía está disponible en versiones de Windows de 32 bits, pero parece que no funciona. En algunas fuentes (incluidas las de Microsoft) se indica que el comando se reemplaza por DPATH, pero no es del todo cierto. A pesar de que el mensaje de ayuda de DPATH apunta al comando APPEND, su sintaxis es la misma que PATH.

@echo off
dpath %windir%

set /p var=<win.ini
echo using dpath with input redirection:
echo %var%
echo.
echo using dpath with type command:
type win.ini
