Sustitución de variables #

A diferencia de otros lenguajes de programación, en un archivo por lotes, una variable se sustituye por su valor real antes de que se ejecute el script por lotes. En otras palabras, la sustitución se realiza cuando el procesador de comandos lee la secuencia de comandos en la memoria, no cuando la secuencia de comandos se ejecuta posteriormente.

Esto permite el uso de variables como comandos dentro del script y como parte de otros nombres de variables en el script, etc. El “script” en este contexto es una línea, o bloque, de código, entre corchetes: () .

¡Pero este comportamiento significa que no puede cambiar el valor de una variable dentro de un bloque!

SET VAR=Hello
FOR /L %%a in (1,1,2) do (
    ECHO %VAR%
    SET VAR=Goodbye
)

imprimirá

Hello
Hello

ya que (como puede ver, al ver la ejecución del script en la ventana de comandos) se evalúa como:

SET VAR=Hello
FOR /L %%a in (1,1,2) do (
    echo Hello
    SET VAR=Goodbye
)

En el ejemplo anterior, el comando ECHO se evalúa como Hola cuando la secuencia de comandos se lee en la memoria, por lo que la secuencia de comandos repetirá Hola para siempre, sin importar cuántas pasadas se realicen a través de la secuencia de comandos.

La forma de lograr el comportamiento variable más “tradicional” (de la variable que se expande mientras se ejecuta el script) es habilitar la “expansión retrasada”. Esto implica agregar ese comando en la secuencia de comandos antes de la instrucción del bucle (generalmente un bucle FOR, en una secuencia de comandos por lotes) y usar un signo de exclamación (!) en lugar de un signo de porcentaje (%) en el nombre de la variable:

setlocal enabledelayedexpansion 
SET VAR=Hello
FOR /L %%a in (1,1,2) do (
    echo !VAR!
    SET VAR=Goodbye
)
endlocal

imprimirá

Hello
Goodbye

La sintaxis %%a in (1,1,2) hace que el bucle se ejecute 2 veces: en la primera ocasión, la variable lleva su valor inicial de ‘Hola’, pero en la segunda pasada por el bucle, habiendo ejecutado la segunda instrucción SET como la última acción en el primer paso; esto ha cambiado al valor revisado ‘Adiós’.

Sustitución de variable avanzada

Ahora, una técnica avanzada. El uso del comando CALL permite que el procesador de comandos por lotes expanda una variable ubicada en la misma línea del script. Esto puede ofrecer una expansión multinivel, mediante LLAMADA repetida y uso de modificadores.

Esto es útil, por ejemplo, en un bucle FOR. Como en el siguiente ejemplo, donde tenemos una lista numerada de variables:

"c:\MisArchivos\prueba1.txt" "c:\MisArchivos\prueba2.txt" "c:\MisArchivos\prueba3.txt"

Podemos lograr esto usando el siguiente bucle FOR:

setlocal enabledelayedexpansion
for %%x in (%*) do (
    set /a "i+=1"
    call set path!i!=%%~!i!
    call echo %%path!i!%%
)
endlocal

Producción:

c:\MyFiles\test1.txt
c:\MyFiles\test2.txt
c:\MyFiles\test3.txt

Tenga en cuenta que la variable !i! se expande primero a su valor inicial, 1, luego la variable resultante, %1, se expande a su valor real de c:\MyFiles\test1.txt. Esta es la doble expansión de la variable i. En la línea siguiente, i se expande nuevamente al doble, mediante el uso del comando CALL ECHO junto con el prefijo variable %%, luego se imprime en la pantalla (es decir, se muestra en la pantalla).

En cada pasada sucesiva por el bucle, el número inicial aumenta en 1 (debido al código i+=1). Por lo tanto, aumenta a “2” en el segundo paso por el bucle y a “3” en el tercer paso. Por lo tanto, la cuerda que se repite en la pantalla se altera con cada pasada.

Declaración #

Para crear una variable simple y asignarla a un valor o cadena, use el comando SET:

SET var=10

Aquí, el código declara una nueva variable var con un valor de 10. De forma predeterminada, todas las variables se almacenan internamente como cadenas; esto significa que el valor 10 no es diferente a foo1234 o Hello, World!

Notas sobre las comillas #

Las comillas utilizadas se incluirán en el valor de la variable:

SET var="new value"             <-- %var% == '"new value"'

Espacios en variables #

El lenguaje por lotes considera que los espacios son partes aceptables de los nombres de variables. Por ejemplo, set var = 10 dará como resultado una variable llamada var que contiene el valor 10 (observa el espacio extra a la derecha de var y a la izquierda del 10).

Uso de comillas para eliminar espacios #

Para evitar espacios, use comillas alrededor de toda la tarea; el nombre y el valor de la variable. Esto también evita espacios finales accidentales al final de la línea (el carácter ␣ denota un espacio):

SET␣var=my␣new␣value␣           <-- '%var%' == 'my new value '
SET␣"var=my␣new␣value"␣         <-- '%var%' == 'my new value'

Además, use comillas al unir varias declaraciones con & o |; alternativamente, coloque el símbolo directamente después del final del valor de la variable:

SET var=val & goto :next        <-- '%var%' == 'val '
SET "var=val" & goto :next      <-- '%var%' == 'val'
SET var=val& goto :next         <-- '%var%' == 'val'

Declarar múltiples variables #

Cuando se definen múltiples variables al comienzo del lote, se puede usar una forma de definición corta empleando una cadena reemplazo.

@echo off
set "vars=_A=good,_B=,_E=bad,_F=,_G=ugly,_C=,_H=,_I=,_J=,_K=,_L=,_D=6
set "%vars:,=" & set "%"

for /f %%l in ('set _') do echo %%l
exit /b

_A=good
_D=6
_E=bad
_G=ugly

Tenga en cuenta que en el ejemplo anterior, las variables se ordenan alfabéticamente de forma nativa cuando se imprimen en la pantalla.

Uso #

echo %var%

Este código hará eco del valor de var

Si se usa setLocal EnableDelayedExpansion, lo siguiente reflejará el valor de var (la expresión estándar %var% no funcionará en ese contexto).

echo !var!

En los archivos por lotes, las variables se pueden usar en cualquier contexto, incluso como partes de comandos o partes de otras variables. No puede llamar a una variable antes de definirla.

Usando variables como comandos:

set var=echo
%var% This will be echoed

Usando variables en otras variables:

set var=part1
set %var%part2=Hello
echo %part1part2%

Uso de una variable como matriz #

Es posible crear un conjunto de variables que pueden actuar de manera similar a una matriz (aunque no son un objeto de matriz real) usando espacios en la instrucción SET:

@echo off
SET var=A "foo bar" 123
for %%a in (%var%) do (
    echo %%a
)
echo Get the variable directly: %var%

Resultado:

A
"foo bar"
123
Get the variable directly: A "foo bar" 123

También es posible declarar su variable usando índices para que pueda recuperar información específica. Esto creará múltiples variables, con la ilusión de una matriz:

@echo off
setlocal enabledelayedexpansion
SET var[0]=A
SET var[1]=foo bar
SET var[2]=123
for %%a in (0,1,2) do (
    echo !var[%%a]!
)
echo Get one of the variables directly: %var[1]%

Resultado:

A
foo bar
123
Get one of the variables directly: foo bar

Tenga en cuenta que en el ejemplo anterior, no puede hacer referencia a var sin indicar cuál es el índice deseado, porque var no existe por sí solo. Este ejemplo también usa setlocal enableelayeexpansion junto con los signos de exclamación en !var[%%a]!. Puede ver más información al respecto en la Documentación del alcance de la sustitución de variables.

Establecer variables desde una entrada #

Usando el interruptor /p con el comando SET puede definir variables desde una entrada.

Esta entrada puede ser una entrada de usuario (teclado):

echo Enter your name : 
set /p name=
echo Your name is %name%

Que se puede simplificar así:

set /p name=Enter your name :
echo Your name is %name%

O puede obtener la entrada de un archivo:

set /p name=< file.txt

en este caso, obtendrá el valor de la primera línea de file.txt

Obtener el valor de varias líneas en un archivo:

(
   set /p line1=
   set /p line2=
   set /p line3=

) < file.txt

Operaciones sobre variables #

set var=10
set /a var=%var%+10
echo %var%

El valor final de var es 20.

La segunda línea no funciona dentro de un bloque de comando que se usa, por ejemplo, en una condición IF o en un bucle FOR, ya que se necesitaría una expansión retrasada en lugar de una expansión variable de entorno estándar.

Aquí hay otra mejor manera de trabajar también en un bloque de comandos:

set var=10
set /A var+=10
echo %var%

El entorno del símbolo del sistema es compatible con valores enteros de 32 bits con signo:

  • suma + y +=
  • resta - y -=
  • multiplicación * y *=
  • división / y /=
  • división de módulo % y %=
  • bit a bit Y &
  • bit a bit O |
  • bit a bit NO ~
  • bit a bit XOR ^
  • Desplazamiento bit a bit a la izquierda <<
  • Desplazamiento bit a bit a la derecha >>
  • lógico NO !
  • unario menos -
  • agrupar con ( y )

El intérprete de comandos de Windows no admite valores enteros de 64 bits ni valores de punto flotante en expresiones aritméticas.

Nota: El operador % debe escribirse en un archivo por lotes como %% para que se interprete como operador.

En una ventana del símbolo del sistema, ejecutar la línea de comando set /A Value=8 % 3 asigna el valor 2 a la variable de entorno Value y, además, genera 2.

En un archivo por lotes, se debe escribir set /A Value=8 %% 3 para asignar el valor 2 a la variable de entorno Value y no se genera nada escrito respectivamente para manejar STDOUT (salida estándar). Una línea set /A Value=8 % 3 en un archivo por lotes generaría un mensaje de error Falta operador al ejecutar el archivo por lotes.

El entorno requiere el interruptor /A solo para operaciones aritméticas, no para variables de cadena ordinarias.

Cada cadena en la expresión aritmética después de set /A, ya sea un número o un operador, se interpreta automáticamente como el nombre de una variable de entorno.

Por esa razón, no es necesario hacer referencia al valor de una variable con %variable% o !variable! cuando el nombre de la variable consta solo de caracteres de palabras (0-9A-Za-z_) y el primer carácter no es un dígito. lo cual es especialmente útil dentro de un bloque de comando que comienza con ( y termina con un ) coincidente.

Los números se convierten de cadenas a enteros con la función C/C++ strtol con base siendo cero, lo que significa una determinación automática de la base que puede resultar fácilmente en resultados inesperados. resultados.

Ejemplo:

set Divided=11
set Divisor=3

set /A Quotient=Divided / Divisor
set /A Remainder=Divided %% Divisor

echo %Divided% / %Divisor% = %Quotient%
echo %Divided% %% %Divisor% = %Remainder%

set HexValue1=0x14
set HexValue2=0x0A
set /A Result=(HexValue1 + HexValue2) * -3

echo (%HexValue1% + %HexValue2%) * -3 = (20 + 10) * -3 = %Result%

set /A Result%%=7
echo -90 %%= 7 = %Result%

set OctalValue=020
set DecimalValue=12
set /A Result=OctalValue - DecimalValue

echo %OctalValue% - %DecimalValue% = 16 - 12 = %Result%

La salida de este ejemplo es:

11 / 3 = 3
11 % 3 = 2
(0x14 + 0x0A) * -3 = (20 + 10) * -3 = -90
-90 %= 7 = -6
020 - 12 = 16 - 12 = 4

Las variables no definidas en la evaluación de la expresión aritmética se sustituyen con el valor 0.
