Diferencias entre Batch (Windows) y Terminal (Linux)

  • Comandos por lotes y sus equivalentes en Bash
  • Variables por lotes y su equivalente en Bash

En esta página

  • Comandos por lotes y sus equivalentes en Bash
  • Variables por lotes y su equivalente en Bash

Batch y bash son bastante diferentes. Las banderas de lote se indican con un /, mientras que las banderas de bash usan un -. Las mayúsculas importan en bash, pero (casi) no importan en lote. Los nombres de las variables por lotes pueden contener espacios, los nombres de las variables bash no.

En última instancia, ambas son formas de manipular e interactuar con la línea de comandos. No es sorprendente que haya una cantidad de superposición de tamaño razonable entre las capacidades de los dos sistemas.

  • bitsadmin está en desuso en favor del cmdlet BITS de PowerShell, pero aún funciona a partir de la versión 1607 de Windows 10
  • certutil separa pares de dígitos hexadecimales con un espacio, por lo que md5sum devolverá un valor de ejemplo de d41d8cd98f00b204e9800998ecf8427e, mientras que certutil muestra el mismo valor que d4 1d 8c d9 8f 00 b2 04 e9 80 09 98 ec f8 42 7e
  • Para cd a otra unidad (por ejemplo, de C: a D:) se debe usar el indicador /d
  • del no puede eliminar carpetas, use rm en su lugar
  • grep es mucho más poderoso que find y findstr, casi no es justo compararlos; find no tiene capacidades de expresiones regulares y findstr tiene capacidades de expresiones regulares extremadamente limitadas ([a-z]{2} no es una sintaxis válida, pero [a-z][a-z] sí lo es)
  • Los bucles for en el símbolo del sistema de Windows solo pueden usar nombres de variables de un solo carácter; esta es la única vez que los nombres de variables de lote distinguen entre mayúsculas y minúsculas
  • Los bucles for en el símbolo del sistema también usan la forma variable %A en lugar de %A% - Los bucles for en scripts por lotes usan la forma variable %%A
  • md crea automáticamente los directorios principales necesarios, mientras que mkdir necesita el indicador -p para hacerlo
  • rem no se puede usar como un carácter de comentario en línea a menos que esté precedido por un &
  • :: no se puede usar como un comentario en línea, y tampoco se debe usar dentro de un bloque de código (conjunto de paréntesis)
  • Tenga en cuenta que algunos comandos de Windows como ping todavía usan - como banderas

Comandos por lotes y sus equivalentes en Bash #

Lote Bash Descripción
command /? man command Muestra la ayuda para command
bitsadmin wget o curl Descarga un archivo remoto
certutil -hashfile file_name MD5 md5sum file_name Obtiene la suma de comprobación MD5 de file_name
cd pwd Muestra el directorio actual
directorio cd directorio cd Cambia el directorio actual al especificado
cls clear Borra la pantalla
copy cp Copia un archivo o archivos de una ruta de origen a una ruta de destino
date date Muestra la fecha o la establece según la entrada del usuario
del rm Elimina un archivo o archivos
dir ls muestra una lista de archivos y directorios en el directorio actual
echo echo Muestra texto en la pantalla
exit return Sale de un script o subrutina
exit logout Cierra el símbolo del sistema o terminal
fc diff Compara el contenido de dos archivos
find "string" file_name grep "string" file_name Busca file_name para string
findstr "cadena" nombre_archivo grep "cadena" nombre_archivo Busca nombre_archivo para cadena
for /F %A en (conjunto de archivos*) hacer algo for elemento en conjunto de archivos*; hacer; alguna cosa; done Hacer algo para cada archivo en un conjunto de archivos
for /F %A in ('comando') hacer algo `comando` Devuelve la salida de un comando
for /L %A en (primero, incremento, último) hacer algo for elemento en `seq primer incremento último`; hacer; alguna cosa; done Comienza en primero y cuenta por incremento hasta llegar a último
forfiles find Busca archivos que coincidan con un determinado criterio
if "%variable%"=="valor" ( if [ "variable"="valor" ]; entonces Compara dos valores
ipconfig ifconfig Muestra información de IP
md mkdir Crea nuevas carpetas
mklink ln -s Crea un enlace simbólico
more more Muestra una pantalla de salida a la vez
move mv Mueve un archivo o archivos de una ruta de origen a una ruta de destino
pause read -p "Presione cualquier tecla para continuar" Pausa la ejecución del script hasta que el usuario presione un botón
popd popd Elimina la entrada superior de la pila de directorios y va al nuevo directorio superior
pushd pushd Agrega el directorio actual a la pila de directorios y va al nuevo directorio superior
ren mv Renombra archivos
rem o :: # Comenta una línea de código
rd rmdir Elimina directorios vacíos
rd /s rm -rf Elimina directorios independientemente de si estaban vacíos o no
establecer variable=valor variable=valor Establece el valor de variable en valor
set /a variable=ecuación variable=$((ecuación)) Realiza operaciones matemáticas (el lote solo puede usar números enteros de 32 bits)
set /p variable=promptstring read -p "promptstring" variable Obtiene la entrada del usuario y la almacena en variable
shift shift Cambia los argumentos en 1 (o n si se proporciona)
sort sort Ordena la salida alfabéticamente por línea
tasklist ps Muestra una lista de procesos en ejecución
taskkill /PID processid kill processid Elimina el proceso con PID processid
time /t date Muestra la hora actual
type cat Muestra el contenido de un archivo
where which Busca el directorio actual y la RUTA para un archivo o comando
whoami id Muestra el nombre y grupo del usuario actual

Variables por lotes y su equivalente en Bash #

Lote Bash Descripción
%variable% $variable Una variable regular
!variable! $variable Una variable dentro de un bloque de código cuando setlocal enableelayeexpansion está activado
%errorlevel% o ERRORLEVEL $? Devuelve el estado del comando anterior: 0 si tiene éxito, 1 (o algo más) si no
%1, %2, %3, etc. $1, $2, $3, etc. Los parámetros dados a un script
%* $* Todos los parámetros dados a un script
