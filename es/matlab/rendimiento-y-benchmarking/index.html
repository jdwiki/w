Identificación de cuellos de botella de rendimiento mediante Profiler #

MATLAB Perfilador es una herramienta para perfilado de software) del código MATLAB. Usando Profiler, es posible obtener una representación visual tanto del tiempo de ejecución como del consumo de memoria.

La ejecución de Profiler se puede hacer de dos maneras:

  • Hacer clic en el botón “Ejecutar y tiempo” en la GUI de MATLAB mientras se abre un archivo .m en el editor (agregado en R2012b).

  • Programáticamente, usando:

    profile on
<some code we want to test>
profile off

A continuación se muestra un código de muestra y el resultado de su perfilado:

function docTest

for ind1 = 1:100
  [~] = var(...
            sum(...
                randn(1000)));
end

spy

De lo anterior aprendemos que la función spy toma alrededor del 25% del tiempo total de ejecución. En el caso del “código real”, una función que ocupa un porcentaje tan grande del tiempo de ejecución sería una buena candidata para la optimización, a diferencia de las funciones análogas a var y cla cuya optimización debería evitarse.

Comparando el tiempo de ejecución de múltiples funciones #

La combinación ampliamente utilizada de tic y [toc](http:// de.mathworks.com/help/matlab/ref/toc.html) puede proporcionar una idea aproximada del tiempo de ejecución de una función o fragmentos de código.

Para comparar varias funciones no se debe usar. ¿Por qué? Es casi imposible proporcionar igualdad de condiciones para que todos los fragmentos de código se comparen dentro de un script utilizando la solución anterior. Tal vez las funciones comparten el mismo espacio de funciones y variables comunes, por lo que las llamadas funciones y fragmentos de código posteriores ya aprovechan las variables y funciones previamente inicializadas. Además, no se sabe si el compilador JIT manejaría estos fragmentos llamados posteriormente por igual.

La función dedicada a los puntos de referencia es timeit. El siguiente ejemplo ilustra su uso.

Están el arreglo A y la matriz B. Se debe determinar qué fila de ‘B’ es la más similar a ‘A’ contando el número de elementos diferentes.

function t = bench()
    A = [0 1 1 1 0 0];
    B = perms(A);

    % functions to compare
    fcns = {
        @() compare1(A,B);
        @() compare2(A,B);
        @() compare3(A,B);
        @() compare4(A,B);
    };

    % timeit
    t = cellfun(@timeit, fcns);
end

function Z = compare1(A,B)  
    Z = sum(  bsxfun(@eq,  A,B) , 2);
end
function Z = compare2(A,B)  
    Z = sum(bsxfun(@xor, A, B),2);
end
function Z = compare3(A,B)  
    A = logical(A);
    Z = sum(B(:,~A),2) + sum(~B(:,A),2);
end
function Z = compare4(A,B)  
     Z = pdist2( A, B, 'hamming', 'Smallest', 1 );
end

Esta forma de evaluación comparativa se vio por primera vez en [esta respuesta] (http://stackoverflow.com/a/21047303/2605073).

¡Está bien ser soltero! #

Visión general: #

El tipo de datos predeterminado para matrices numéricas en MATLAB es doble. doble es una representación de números en punto flotante, y este formato ocupa 8 bytes (o 64 bits) por valor. En algunos casos, donde p. tratar solo con números enteros o cuando la inestabilidad numérica no es un problema inminente, es posible que no se requiera una profundidad de bits tan alta. Por esta razón, se recomienda considerar los beneficios de la precisión “simple” (u otros [tipos] 2 apropiados):

  • Tiempo de ejecución más rápido (especialmente notable en GPU).
  • La mitad del consumo de memoria: puede tener éxito donde ‘doble’ falla debido a un error de falta de memoria; más compacto cuando se almacena como archivos.

La conversión de una variable de cualquier tipo de datos compatible a single se realiza mediante:

sing_var = single(var);

Algunas funciones de uso común (como: zeros, [eye](http://www .mathworks.com/help/matlab/ref/eye.html#btpkgsh-1), ones, etc.) que generan valores dobles por defecto, permiten especificar el tipo/clase de la salida.

Convertir variables en un script a una precisión/tipo/clase no predeterminada: #

A partir de julio de 2016, no existe una forma documentada de cambiar el tipo de datos predeterminado de MATLAB de doble.

En MATLAB, las nuevas variables suelen imitar los tipos de datos de las variables utilizadas al crearlas. Para ilustrar esto, considere el siguiente ejemplo:

A = magic(3);
B = diag(A);
C = 20*B;
>> whos C
  Name      Size            Bytes  Class     Attributes
  C         3x1                24  double 


A = single(magic(3)); % A is converted to "single"
B = diag(A);
C = B*double(20);     % The stricter type, which in this case is "single", prevails
D = single(size(C));  % It is generally advised to cast to the desired type explicitly.
>> whos C
  Name      Size            Bytes  Class     Attributes
  C         3x1                12  single

Por lo tanto, puede parecer suficiente lanzar/convertir varias variables iniciales para que el cambio penetre en todo el código; sin embargo, esto se desaconseja (ver Advertencias y trampas a continuación).

Advertencias y trampas: #

  1. Las conversiones repetidas se desaconsejan debido a la introducción de ruido numérico (cuando se convierte de “simple” a “doble”) o pérdida de información (cuando se convierte de “doble” a “simple” o entre ciertos entero tipos), p. :

     double(single(1.2)) == double(1.2)   
 ans =
      0

    This can be mitigated somewhat using typecast. See also https://www.wikiod.com/es/matlab/errores-y-errores-comunes#Tenga en cuenta la inexactitud del punto flotante

  2. Se desaconseja confiar únicamente en la tipificación implícita de datos (es decir, lo que MATLAB supone que debe ser el tipo de salida de un cálculo) debido a varios efectos no deseados que pueden surgir:

    • Loss of information: when a double result is expected, but a careless combination of single and double operands yields single precision.

    • Unexpectedly high memory consumption: when a single result is expected but a careless computation results in a double output.

    • Unnecessary overhead when working with GPUs: when mixing gpuArray types (i.e. variables stored in VRAM) with non-gpuArray variables (i.e. those usually stored in RAM) the data will have to be transferred one way or the other before the computation can be performed. This operation takes time, and can be very noticeable in repetative computations.

    • Errors when mixing floating-point types with integer types: functions like mtimes (*) are not defined for mixed inputs of integer and floating point types - and will error. Functions like times (.*) are not defined at all for integer-type inputs - and will again error.

      >> ones(3,3,'int32')*ones(3,3,'int32')
Error using  * 
MTIMES is not fully supported for integer classes. At least one input must be scalar.

>> ones(3,3,'int32').*ones(3,3,'double')
Error using  .* 
Integers can only be combined with integers of the same class, or scalar doubles.

    For better code readability and reduced risk of unwanted types, a defensive approach is advised, where variables are explicitly cast to the desired type.

La importancia de la preasignación #

Los arreglos en MATLAB se mantienen como bloques continuos en la memoria, asignados y liberados automáticamente por MATLAB. MATLAB oculta las operaciones de administración de memoria, como el cambio de tamaño de una matriz, detrás de una sintaxis fácil de usar:

a = 1:4

a =

     1     2     3     4

a(5) = 10  % or alternatively a = [a, 10]

a =

     1     2     3     4    10

Es importante comprender que lo anterior no es una operación trivial, a(5) = 10 hará que MATLAB asigne un nuevo bloque de memoria de tamaño 5, copie los primeros 4 números y establezca el 5’ en 10. Esa es una operación O(numel(a)), y no O(1).

Considera lo siguiente:

clear all
n=12345678;
a=0;
tic
for i = 2:n
    a(i) = sqrt(a(i-1)) + i;
end
toc

Elapsed time is 3.004213 seconds.

¡a se reasigna n veces en este ciclo (excluyendo algunas optimizaciones realizadas por MATLAB)! Tenga en cuenta que MATLAB nos da una advertencia:

“La variable ‘a’ parece cambiar de tamaño en cada iteración del ciclo. Considere preasignar la velocidad”.

¿Qué sucede cuando preasignamos?

a=zeros(1,n);
tic
for i = 2:n
    a(i) = sqrt(a(i-1)) + i;
end
toc

Elapsed time is 0.410531 seconds.

Podemos ver que el tiempo de ejecución se reduce en un orden de magnitud.

Métodos de preasignación:

MATLAB proporciona varias funciones para la asignación de vectores y matrices, según los requisitos específicos del usuario. Estos incluyen: ceros, unos, nan, ojo, verdadero, etc.

a = zeros(3)       % Allocates a 3-by-3 matrix initialized to 0
a =

     0     0     0
     0     0     0
     0     0     0

a = zeros(3, 2)     % Allocates a 3-by-2 matrix initialized to 0
a =

     0     0
     0     0
     0     0

a = ones(2, 3, 2)      % Allocates a 3 dimensional array (2-by-3-by-2) initialized to 1
a(:,:,1) =

     1     1     1
     1     1     1


a(:,:,2) =

     1     1     1
     1     1     1

a = ones(1, 3) * 7  % Allocates a row vector of length 3 initialized to 7
a =

     7     7     7

También se puede especificar un tipo de datos:

a = zeros(2, 1, 'uint8');  % allocates an array of type uint8

También es fácil clonar el tamaño de una matriz existente:

a = ones(3, 4);       % a is a 3-by-4 matrix of 1's
b = zeros(size(a));  % b is a 3-by-4 matrix of 0's

Y clonar el tipo:

a = ones(3, 4, 'single');       % a is a 3-by-4 matrix of type single
b = zeros(2, 'like', a);        % b is a 2-by-2 matrix of type single

tenga en cuenta que ‘me gusta’ también clona complexity y sparsity.

La preasignación se logra implícitamente usando cualquier función que devuelva una matriz del tamaño final requerido, como rand, gallery, kron, bsxfun , colon y muchos otros. Por ejemplo, una forma común de asignar vectores con elementos que varían linealmente es usando el operador de dos puntos (con la variante de 2 o 3 operandos1):

a = 1:3 
a =

     1     2     3

a = 2:-3:-4
a =

     2    -1    -4

Las matrices de celdas se pueden asignar usando la función cell() de la misma manera que zeros().

a = cell(2,3)
a = 

    []    []    []
    []    []    []

Tenga en cuenta que las matrices de celdas funcionan manteniendo punteros a las ubicaciones en la memoria del contenido de las celdas. Por lo tanto, todos los consejos de preasignación también se aplican a los elementos individuales de la matriz de celdas.

reorganizar una matriz ND puede mejorar el rendimiento general #

En algunos casos, necesitamos aplicar funciones a un conjunto de matrices ND. Veamos este sencillo ejemplo.

A(:,:,1) = [1 2; 4 5];
A(:,:,2) = [11 22; 44 55];
B(:,:,1) = [7 8; 1 2];
B(:,:,2) = [77 88; 11 22];

A =

ans(:,:,1) =

   1   2 
   4   5 

ans(:,:,2) =

   11   22
   44   55

>> B
B =

ans(:,:,1) =

   7   8
   1   2

ans(:,:,2) =

   77   88
   11   22

Ambas matrices son 3D, digamos que tenemos que calcular lo siguiente:

result= zeros(2,2);
...
for k = 1:2 
   result(i,j) = result(i,j) + abs( A(i,j,k) - B(i,j,k) );
...

if k is very large, this for-loop can be a bottleneck since MATLAB order the data in a column major fashion. So a better way to compute "result" could be:

% trying to exploit the column major ordering
Aprime = reshape(permute(A,[3,1,2]), [2,4]);
Bprime = reshape(permute(B,[3,1,2]), [2,4]);


>> Aprime
Aprime =

    1    4    2    5
   11   44   22   55

>> Bprime
Bprime =

    7    1    8    2
   77   11   88   22

Ahora reemplazamos el ciclo anterior por el siguiente:

result= zeros(2,2);
....
temp = abs(Aprime - Bprime);
for k = 1:2
    result(i,j) = result(i,j) + temp(k, i+2*(j-1));
...

Reorganizamos los datos para poder explotar la memoria caché. La permutación y la remodelación pueden ser costosas, pero cuando se trabaja con matrices ND grandes, el costo computacional relacionado con estas operaciones es mucho menor que cuando se trabaja con matrices no ordenadas.
