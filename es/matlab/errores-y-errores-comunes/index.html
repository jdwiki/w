Los operadores transpuestos #

  • .' es la forma correcta de transponer un vector o matriz en MATLAB.
  • ' es la forma correcta de tomar la transpuesta conjugada compleja (también conocida como conjugado hermitiano) de un vector o matriz en MATLAB.

Tenga en cuenta que para la transposición .', hay un punto delante del apóstrofe. Esto está en consonancia con la sintaxis de las otras operaciones con elementos en MATLAB: * multiplica matrices, .* multiplica elementos de matrices juntos. Los dos comandos son muy similares, pero conceptualmente muy distintos. Al igual que otros comandos de MATLAB, estos operadores son “azúcar sintáctico” que se convierte en una llamada de función “adecuada” en tiempo de ejecución. Así como == se convierte en una evaluación de la función eq, piense en .' como la forma abreviada de transpose. Si solo escribe ' (sin el punto), de hecho está utilizando el comando ctranspose, que calcula la transposición conjugada compleja, que también se conoce como Conjugado hermitiano, de uso frecuente en física. Siempre que el vector o matriz transpuesta tenga un valor real, los dos operadores producen el mismo resultado. Pero tan pronto como tratemos con números complejos, inevitablemente nos encontraremos con problemas si no usamos la forma abreviada “correcta”. Lo que es “correcto” depende de su aplicación.

Considere el siguiente ejemplo de una matriz ‘C’ que contiene números complejos:

>> C = [1i, 2; 3*1i, 4]
C =
   0.0000 + 1.0000i   2.0000 + 0.0000i
   0.0000 + 3.0000i   4.0000 + 0.0000i

Tomemos la transposición usando la abreviatura .' (con el punto). El resultado es el esperado, la forma transpuesta de C.

>> C.'
ans =
   0.0000 + 1.0000i   0.0000 + 3.0000i
   2.0000 + 0.0000i   4.0000 + 0.0000i

Ahora, usemos ' (sin el punto). Vemos que, además de la transposición, los valores complejos también se han transformado en sus complejos conjugados.

>> C'
ans =
   0.0000 - 1.0000i   0.0000 - 3.0000i
   2.0000 + 0.0000i   4.0000 + 0.0000i

En resumen, si tiene la intención de calcular el conjugado hermitiano, la transposición conjugada compleja, entonces use ' (sin el punto). Si solo desea calcular la transposición sin conjugar los valores complejos, use .' (con el punto).

No nombre una variable con un nombre de función existente #

Ya existe una función sum(). En consecuencia, si nombramos una variable con el mismo nombre

sum = 1+3;

y si tratamos de usar la función mientras la variable todavía existe en el espacio de trabajo

A = rand(2);
sum(A,1)

obtendremos el error críptico:

Subscript indices must either be real positive integers or logicals.

clear() la variable primero y luego use la función

clear sum

sum(A,1)
ans =
       1.0826       1.0279

¿Cómo podemos comprobar si ya existe una función para evitar este conflicto?

Use which() con el indicador -all:

which sum -all
sum is a variable.
built-in (C:\Program Files\MATLAB\R2016a\toolbox\matlab\datafun\@double\sum)   % Shadowed double method
...

Esta salida nos dice que ‘sum’ es primero una variable y que los siguientes métodos (funciones) están sombreados por ella, es decir, MATLAB primero intentará aplicar nuestra sintaxis a la variable, en lugar de usar el método.

Tenga en cuenta la inexactitud del punto flotante #

Los números de punto flotante no pueden representar todos los números reales. Esto se conoce como inexactitud de punto flotante.

Hay una cantidad infinita de números de puntos flotantes y pueden ser infinitamente largos (por ejemplo, π), por lo que poder representarlos perfectamente requeriría una cantidad infinita de memoria. Al ver que esto era un problema, se diseñó una representación especial para el almacenamiento de “números reales” en la computadora, el estándar IEEE 754. En resumen, describe cómo las computadoras almacenan este tipo de números, con exponente y mantisa, como,

``flotante = signo * 2^exponente * mantisa’’

Con una cantidad limitada de bits para cada uno de estos, solo se puede lograr una precisión finita. Cuanto menor sea el número, menor será la brecha entre los números posibles (¡y viceversa!). Puede probar sus números reales en esta demostración en línea.

Tenga en cuenta este comportamiento e intente evitar todas las comparaciones de puntos flotantes y su uso como condiciones de parada en bucles. Vea a continuación dos ejemplos:

Ejemplos: comparación de punto flotante hecha INCORRECTA: #

>> 0.1 + 0.1 + 0.1  == 0.3

ans =

  logical

   0

Es una mala práctica usar la comparación de punto flotante como se muestra en el ejemplo anterior. Puede superarlo tomando el valor absoluto de su diferencia y comparándolo con un (pequeño) nivel de tolerancia.

A continuación se muestra otro ejemplo, en el que se utiliza un número de punto flotante como condición de parada en un bucle while:**

k = 0.1;
while k <= 0.3 
  disp(num2str(k));
  k = k + 0.1;
end

% --- Output: ---
0.1
0.2

Se pierde el último bucle esperado (0.3 <= 0.3).

Ejemplo: Comparación de punto flotante hecha DERECHA: #

x = 0.1 + 0.1 + 0.1;
y = 0.3;
tolerance = 1e-10; % A "good enough" tolerance for this case.

if ( abs( x - y ) <= tolerance )
  disp('x == y');
else
  disp('x ~= y');
end

% --- Output: ---
x == y

Varias cosas a tener en cuenta:

  • Como era de esperar, ahora x e y se tratan como equivalentes.
  • En el ejemplo anterior, la elección de la tolerancia se hizo arbitrariamente. Por lo tanto, el valor elegido podría no ser adecuado para todos los casos (especialmente cuando se trabaja con números mucho más pequeños). La elección del límite inteligente se puede hacer usando la función eps, es decir, N*eps(max(x,y)), donde N es un número específico del problema. Una opción razonable para ‘N’, que también es lo suficientemente permisiva, es ‘1E2’ (aunque, en el problema anterior, ‘N=1’ también sería suficiente).

Lo que ves NO es lo que obtienes: char vs cellstring en la ventana de comandos #

Este es un ejemplo básico dirigido a nuevos usuarios. No se enfoca en explicar la diferencia entre char y cellstring.

Puede suceder que quieras deshacerte de los ' en tus cadenas, aunque nunca los hayas agregado. De hecho, esos son artefactos que la ventana de comandos usa para distinguir entre algunos tipos.

Se imprimirá una cadena

s = 'dsadasd'
s =
dsadasd

Se imprimirá un cellstring

c = {'dsadasd'};
c = 
    'dsadasd'

Observe cómo las comillas simples y la sangría son artefactos para notificarnos que c es una cadena de celdas en lugar de un char. De hecho, la cadena está contenida en la celda, es decir,

c{1}
ans =
dsadasd

Función indefinida o método X para argumentos de entrada de tipo Y #

Esta es la manera prolija de MATLAB de decir que no puede encontrar la función que está tratando de llamar. Hay varias razones por las que podría recibir este error:

Esa función se introdujo después de su versión actual de MATLAB #

La documentación en línea de MATLAB proporciona una característica muy interesante que le permite determinar en qué versión se introdujo una función determinada. Se encuentra en la parte inferior izquierda de cada página de la documentación:

Compare esta versión con su propia versión actual (ver) para determinar si esta función está disponible en su versión particular. Si no es así, intente buscar [versiones archivadas de la documentación] (http://www.mathworks.com/help/doc-archives.html) para encontrar una alternativa adecuada en su versión.

¡No tienes esa caja de herramientas! #

La instalación base de MATLAB tiene una gran cantidad de funciones; sin embargo, la funcionalidad más especializada se empaqueta dentro de las cajas de herramientas y Mathworks las vende por separado. La documentación para todas las cajas de herramientas está visible ya sea que tenga la caja de herramientas o no, así que asegúrese de verificar y ver si tiene la caja de herramientas adecuada.

Para verificar a qué caja de herramientas pertenece una función dada, mire en la parte superior izquierda de la documentación en línea para ver si se menciona una caja de herramientas específica.

A continuación, puede determinar qué cajas de herramientas ha instalado su versión de MATLAB emitiendo el comando ver que imprimirá una lista de todas las cajas de herramientas instaladas.

Si no tiene esa caja de herramientas instalada y desea usar la función, deberá comprar una licencia para esa caja de herramientas en particular de The Mathworks.

MATLAB no puede localizar la función #

Si MATLAB aún no puede encontrar su función, entonces debe ser una función definida por el usuario. Es posible que viva en otro directorio y ese directorio debería [agregarse a la ruta de búsqueda] (http://www.mathworks.com/help/matlab/matlab_env/add-remove-or-reorder-folders-on- the-search-path.html) para que se ejecute su código. Puede verificar si MATLAB puede ubicar su función usando which, que debería devolver la ruta al archivo fuente.

El uso de “i” o “j” como unidad imaginaria, índices de bucle o variable común. #

Recomendación #

Debido a que los símbolos i y j pueden representar cosas significativamente diferentes en MATLAB, su uso como índices de bucle ha dividido a la comunidad de usuarios de MATLAB desde hace mucho tiempo. Si bien algunas razones históricas de rendimiento podrían ayudar a que la balanza se incline hacia un lado, este ya no es el caso y ahora la elección depende completamente de usted y de las prácticas de codificación que elija seguir.

Las recomendaciones oficiales actuales de Mathworks son:

  • Dado que i es una función, puede anularse y usarse como una variable. Sin embargo, es mejor evitar el uso de i y j para nombres de variables si tiene la intención de usarlos en aritmética compleja.
  • Para mayor velocidad y robustez mejorada en aritmética compleja, use 1i y 1j en lugar de i y j.

Defecto #

En MATLAB, por defecto, las letras i y [j](http://uk.mathworks.com/help /matlab/ref/j.html) son nombres de función incorporados, que se refieren a la unidad imaginaria en el dominio complejo.

Entonces, por defecto i = j = sqrt(-1).

>> i
ans =
   0.0000 + 1.0000i
>> j
ans =
   0.0000 + 1.0000i

y como es de esperar:

>> i^2
ans =
    -1

Usándolos como una variable (para índices de bucle u otra variable)

MATLAB permite usar el nombre de la función integrada como una variable estándar. En este caso, el símbolo utilizado ya no apuntará a la función incorporada sino a su propia variable definida por el usuario. Esta práctica, sin embargo, generalmente no se recomienda, ya que puede generar confusión, depuración y mantenimiento difíciles (ver otro ejemplo [no-nombrar-una-variable-con-un-nombre-de-función-existente](http:// stackoverflow.com/documentation/matlab/973/common-mistakes-and-errors/3168/do-not-name-a-variable-with-an-existing-function-name)).

Si es muy pedante en cuanto a respetar las convenciones y las mejores prácticas, evitará utilizarlas como índices de bucle en este lenguaje. Sin embargo, el compilador lo permite y es perfectamente funcional, por lo que también puede optar por mantener los viejos hábitos y usarlos como iteradores de bucle.

>> A = nan(2,3);
>> for i=1:2        % perfectly legal loop construction
       for j = 1:3
        A(i, j) = 10 * i + j;
       end
   end

Tenga en cuenta que los índices de bucle no quedan fuera del alcance al final del bucle, por lo que mantienen su nuevo valor.

>> [ i ; j ]
ans =
     2
     3

En el caso de que los use como variable, asegúrese de que estén inicializados antes de usarlos. En el ciclo anterior, MATLAB los inicializa automáticamente cuando prepara el ciclo, pero si no se inicializa correctamente, puede ver rápidamente que puede introducir inadvertidamente números “complejos” en su resultado.

Si más adelante, necesita deshacer el sombreado de la función integrada (=por ejemplo, quiere que i y j representen la unidad imaginaria nuevamente), puede borrar las variables:

>> clear i j

Ahora comprende la reserva de Mathworks sobre usarlos como índices de bucle si tiene la intención de usarlos en aritmética compleja. Su código estaría plagado de inicializaciones variables y comandos claros, la mejor manera de confundir al programador más serio (¡sí, usted!…) y los accidentes del programa que están a punto de ocurrir.

Si no se espera una aritmética compleja, el uso de i y j es perfectamente funcional y no hay penalización de rendimiento.

Usándolos como unidad imaginaria: #

Si su código tiene que lidiar con números ‘complejos’, entonces ‘i’ y ‘j’ ciertamente serán útiles. Sin embargo, en aras de la desambiguación e incluso para las interpretaciones, se recomienda utilizar la forma completa en lugar de la sintaxis abreviada. La forma completa es 1i (o 1j).

>> [ i ; j ; 1i ; 1j]
ans =
   0.0000 + 1.0000i
   0.0000 + 1.0000i
   0.0000 + 1.0000i
   0.0000 + 1.0000i

Representan el mismo valor sqrt(-1), pero la forma posterior:

  • es más explícito, de forma semántica.
  • es más fácil de mantener (alguien que mire su código más tarde no tienes que leer el código para encontrar si i o j era una variable o la unidad imaginaria).
  • es más rápido (fuente: Mathworks).

Tenga en cuenta que la sintaxis completa 1i es válida con cualquier número que preceda al símbolo:

>> a = 3 + 7.8j
a =
   3.0000 + 7.8000i

Esta es la única función que puede mantener con un número sin un operador entre ellos.

Trampas #

Si bien su uso como unidad imaginaria O variable es perfectamente legal, este es solo un pequeño ejemplo de lo confuso que podría volverse si ambos usos se mezclan:

Anulemos i y convirtámoslo en una variable:

>> i=3
i =
     3

Ahora i es una variable (con el valor 3), pero solo anulamos la notación taquigráfica de la unidad imaginaria, la forma completa todavía se interpreta correctamente:

>> 3i
ans =
   0.0000 + 3.0000i

Lo que ahora nos permite construir las formulaciones más oscuras. Te dejo evaluar la legibilidad de todas las siguientes construcciones:

>> [ i ; 3i ; 3*i ; i+3i ; i+3*i ]
ans =
   3.0000 + 0.0000i
   0.0000 + 3.0000i
   9.0000 + 0.0000i
   3.0000 + 3.0000i
  12.0000 + 0.0000i

Como puede ver, cada valor en la matriz anterior devuelve un resultado diferente. Si bien cada resultado es válido (siempre que esa haya sido la intención inicial), la mayoría de ustedes admitirá que sería una verdadera pesadilla leer un código plagado de tales construcciones.

Los argumentos de entrada no son suficientes #

A menudo, los desarrolladores principiantes de MATLAB utilizarán el editor de MATLAB para escribir y editar código, en particular funciones personalizadas con entradas y salidas. Hay un botón Ejecutar en la parte superior que está disponible en versiones recientes de MATLAB:

Una vez que el desarrollador termina con el código, a menudo se siente tentado a presionar el botón Ejecutar. Para algunas funciones, esto funcionará bien, pero para otras recibirán un error de “Argumentos de entrada insuficientes” y se preguntarán por qué ocurre el error.

La razón por la que este error puede no ocurrir es porque escribió un script de MATLAB o una función que no admite argumentos de entrada. El uso del botón Ejecutar ejecutará un script de prueba o ejecutará una función suponiendo que no haya argumentos de entrada. Si su función requiere argumentos de entrada, se producirá el error No hay suficientes argumentos de entrada ya que ha escrito una función que espera que las entradas vayan dentro de la función. Por lo tanto, no puede esperar que la función se ejecute simplemente presionando el botón Ejecutar.

Para demostrar este problema, supongamos que tenemos una función mult que simplemente multiplica dos matrices juntas:

function C = mult(A, B)
    C = A * B;
end

En versiones recientes de MATLAB, si escribió esta función y presionó el botón Ejecutar, obtendrá el error que esperamos:

>> mult
Not enough input arguments.

Error in mult (line 2)
    C = A * B;

Hay dos formas de resolver este problema:

Método #1 - A través del símbolo del sistema #

Simplemente cree las entradas que necesita en el Símbolo del sistema, luego ejecute la función usando las entradas que ha creado:

A = rand(5,5);
B = rand(5,5);
C = mult(A,B);

Método #2 - Interactivamente a través del Editor #

Debajo del botón Ejecutar, hay una flecha negra oscura. Si hace clic en esa flecha, puede especificar las variables que le gustaría obtener del espacio de trabajo de MATLAB escribiendo la forma en que desea llamar a la función exactamente como ha visto en el método n.º 1. Asegúrese de que las variables que está especificando dentro de la función existan en el espacio de trabajo de MATLAB:

Usando longitud para arreglos multidimensionales #

Un error común que cometen los programadores de MATLAB es usar la función longitud para las matrices (a diferencia de los vectores, para los que está diseñada). La función longitud, como se menciona en su documentación, “devuelve la longitud de la dimensión de matriz más grande” de la entrada.

Para los vectores, el valor de retorno de longitud tiene dos significados diferentes:

  1. El número total de elementos en el vector.
  2. La dimensión más grande del vector.

A diferencia de los vectores, los valores anteriores no serían iguales para matrices de más de una dimensión no singleton (es decir, cuyo tamaño es mayor que ‘1’). Esta es la razón por la que usar longitud para matrices es ambiguo. En su lugar, se recomienda utilizar una de las siguientes funciones, incluso cuando se trabaja con vectores, para dejar perfectamente clara la intención del código:

  1. size(A): devuelve un vector de fila cuyos elementos contienen la cantidad de elementos a lo largo de la dimensión correspondiente de A .
  2. numel(A): devuelve la cantidad de elementos en A. Equivalente a prod(tamaño(A)).
  3. ndims(A): devuelve el número de dimensiones en la matriz A. Equivalente a numel(tamaño(A)).

Esto es especialmente importante cuando se escriben funciones de biblioteca “a prueba de futuro”, vectorizadas, cuyas entradas no se conocen de antemano y pueden tener varios tamaños y formas.

Tenga cuidado con los cambios en el tamaño de la matriz #

Algunas operaciones comunes en MATLAB, como diferenciación o integración, generan resultados que tienen una cantidad diferente de elementos que los datos de entrada. Este hecho puede pasarse por alto fácilmente, lo que normalmente provocaría errores como Las dimensiones de la matriz deben coincidir. Considere el siguiente ejemplo:

t = 0:0.1:10;        % Declaring a time vector
y = sin(t);          % Declaring a function

dy_dt = diff(y);     % calculates dy/dt for y = sin(t)

Digamos que queremos trazar estos resultados. Echamos un vistazo a los tamaños de matriz y vemos:

size(y) is 1x101
size(t) is 1x101

Pero:

size(dy_dt) is 1x100

¡La matriz es un elemento más corta!

Ahora imagine que tiene datos de medición de posiciones a lo largo del tiempo y desea calcular jerk (t), obtendrá una matriz de 3 elementos menos que la matriz de tiempo (porque la sacudida es la posición diferenciada 3 veces).

vel = diff(y);       % calculates velocity vel=dy/dt for y = sin(t)  size(vel)=1x100
acc = diff(vel);     % calculates acceleration acc=d(vel)/dt         size(acc)=1x99
jerk = diff(acc);    % calculates jerk jerk=d(acc)/dt                size(jerk)=1x98

Y luego operaciones como:

x = jerk .* t;          % multiplies jerk and t element wise

devuelve errores, porque las dimensiones de la matriz no concuerdan.

Para calcular operaciones como las anteriores, debe ajustar el tamaño de matriz más grande para que se ajuste al más pequeño. También podría ejecutar una regresión (“polyfit”) con sus datos para obtener un polinomio para sus datos.

Errores de discrepancia de dimensiones #

Los errores de discrepancia de dimensiones suelen aparecer cuando:

  • No prestar atención a la forma de las variables devueltas por las llamadas a funciones/métodos. En muchas funciones integradas de MATLAB, las matrices se convierten en vectores para acelerar los cálculos y la variable devuelta podría seguir siendo un vector en lugar de la matriz que esperábamos. Este también es un escenario común cuando se trata de enmascaramiento lógico.
  • Uso de tamaños de matriz incompatibles al invocar expansión de matriz implícita.
