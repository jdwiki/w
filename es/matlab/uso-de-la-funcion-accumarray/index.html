Uso de la función `accumarray()`

accumarray permite agregar elementos de una matriz de varias maneras, aplicando potencialmente alguna función a los elementos en el proceso. accumarray se puede considerar como un reductor ligero (ver también: https://www.wikiod.com/es/hadoop/introduccion-a-mapreduce).

Este tema contendrá escenarios comunes donde accumarray es especialmente útil.

Sintaxis #

  • accumarray(matrizsubíndice, matrizvalores)
  • accumarray(matrizsubíndice, matrizvalores, tamañoDeSalida)
  • accumarray(subscriptArray, valoresArray, sizeOfOutput, funcHandle)
  • accumarray(subscriptArray, valoresArray, sizeOfOutput, funcHandle, fillVal)
  • accumarray(subscriptArray, valoresArray, sizeOfOutput, funcHandle, fillVal, isSparse)

Parámetros #

Parámetro Detalles
matrizsubíndice Matriz de subíndice, especificada como vector de índices, matriz de índices o matriz de celdas de vectores de índice.
valoresArray Datos, especificados como un vector o un escalar.
tamañoDeSalida Tamaño de la matriz de salida, especificado como un vector de enteros positivos.
funcHandle Función que se aplicará a cada conjunto de elementos durante la agregación, especificada como identificador de función o [].
rellenarVal Valor de relleno, para cuando subs no hace referencia a cada elemento en la salida.
es escaso ¿Debería la salida ser una matriz dispersa?
  • Introducido en MATLAB v7.0.

Referencias: #

  1. “Subestimado accumarray”, por Loren Shure, 20 de febrero de 2008.
  2. accumarray en la documentación oficial de MATLAB.

Encontrar el valor máximo entre elementos agrupados por otro vector #

Este es un ejemplo oficial de MATLAB

Considere el siguiente código:

month = [1;1;2;3;8;1;3;4;9;11;9;12;3;4;11];
temperature = [57;61;60;62;45;59;64;66;40;56;38;65;61;64;55];
maxTemp = accumarray(month,temperature,[],@max);

En este ejemplo, primero se recopilan todos los valores que tienen el mismo mes y luego se aplica la función especificada por la entrada 4th a accumarray (en este caso, @max). a cada conjunto de este tipo.

Aplicar filtro a parches de imagen y establecer cada píxel como la media del resultado de cada parche #

Muchos algoritmos modernos de procesamiento de imágenes usan parches como su elemento básico para trabajar. Por ejemplo, uno podría eliminar el ruido de los parches (Ver Algoritmo BM3D).

Sin embargo, al construir la imagen a partir de los parches procesados, tenemos muchos resultados para el mismo píxel. Una forma de solucionarlo es tomando el promedio (media empírica) de todos los valores del mismo píxel.

El siguiente código muestra cómo dividir una imagen en parches y luego reconstruir la imagen a partir de parches usando el promedio usando [accumarray()][1]:

numRows = 5;
numCols = 5;

numRowsPatch = 3;
numColsPatch = 3;

% The Image
mI = rand([numRows, numCols]);

% Decomposing into Patches - Each pixel is part of many patches (Neglecting
% boundariwes, each pixel is part of (numRowsPatch * numColsPatch) patches).
mY = ImageToColumnsSliding(mI, [numRowsPatch, numColsPatch]);

% Here one would apply some operation which work on patches

% Creating image of the index of each pixel
mPxIdx = reshape(1:(numRows * numCols), [numRows, numCols]);

% Creating patches of the same indices
mSubsAccu = ImageToColumnsSliding(mPxIdx, [numRowsPatch, numColsPatch]);

% Reconstruct the image - Option A
mO = accumarray(mSubsAccu(:), mY(:)) ./ accumarray(mSubsAccu(:), 1);

% Reconstruct the image - Option B
mO = accumarray(mSubsAccu, mY(:), [(numRows * numCols), 1], @(x) mean(x));

% Rehsape the Vector into the Image
mO = reshape(mO, [numRows, numCols]);
