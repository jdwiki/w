Círculos #

La opción más fácil para dibujar un círculo es, obviamente, la función rectangle.

%// radius
r = 2;

%// center
c = [3 3];

pos = [c-r 2*r 2*r];
rectangle('Position',pos,'Curvature',[1 1])
axis equal

¡pero la curvatura del rectángulo debe establecerse en 1!

El vector posición define el rectángulo, los dos primeros valores x y y son la esquina inferior izquierda del rectángulo. Los dos últimos valores definen el ancho y la altura del rectángulo.

pos = [ [x y] width height ]

La esquina inferior izquierda del círculo - sí, este círculo tiene esquinas, aunque imaginarias - es el centro c = [3 3] menos el radio r = 2 que es [x y] = [1 1]. Ancho y alto son iguales al diámetro del círculo, entonces ancho = 2*r; alto = ancho;

En caso de que la suavidad de la solución anterior no sea suficiente, no hay forma de evitar usar la forma obvia de dibujar un círculo real mediante el uso de funciones trigonométricas.

%// number of points
n = 1000;

%// running variable
t = linspace(0,2*pi,n);

x = c(1) + r*sin(t);
y = c(2) + r*cos(t);

%// draw line
line(x,y)

%// or draw polygon if you want to fill it with color
%// fill(x,y,[1,1,1])
axis equal

Flechas #

En primer lugar, uno puede usar quiver, donde uno no tiene que lidiar con unidades de cifras normalizadas poco prácticas mediante el uso de anotación

drawArrow = @(x,y) quiver( x(1),y(1),x(2)-x(1),y(2)-y(1),0 )    

x1 = [10 30];
y1 = [10 30];

drawArrow(x1,y1); hold on

x2 = [25 15];
y2 = [15 25];

drawArrow(x2,y2)

Importante es el quinto argumento de quiver: 0 que deshabilita una escala predeterminada, ya que esta función generalmente se usa para trazar campos vectoriales. (o use el par de valores de propiedad 'AutoScale','off')

También se pueden agregar características adicionales:

drawArrow = @(x,y,varargin) quiver( x(1),y(1),x(2)-x(1),y(2)-y(1),0, varargin{:} )       
drawArrow(x1,y1); hold on
drawArrow(x2,y2,'linewidth',3,'color','r')

Si se desean diferentes puntas de flecha, es necesario usar anotaciones (esta respuesta puede ser útil [¿Cómo cambio el estilo de la punta de flecha en el diagrama de carcaj?] (http://stackoverflow.com/questions/18776172/in-matlab-how- do-i-change-the-arrow-head-style-in-quiver-plot)).

El tamaño de la punta de la flecha se puede ajustar con la propiedad 'MaxHeadSize'. Desafortunadamente no es consistente. Los límites de los ejes deben establecerse después.

x1 = [10 30];
y1 = [10 30];
drawArrow(x1,y1,{'MaxHeadSize',0.8,'Color','b','LineWidth',3}); hold on

x2 = [25 15];
y2 = [15 25];
drawArrow(x2,y2,{'MaxHeadSize',10,'Color','r','LineWidth',3}); hold on

xlim([1, 100])
ylim([1, 100])

[Hay otro ajuste para puntas de flecha ajustables:] (http://stackoverflow.com/a/25732096/2605073)

function [ h ] = drawArrow( x,y,xlimits,ylimits,props )

xlim(xlimits)
ylim(ylimits)

h = annotation('arrow');
set(h,'parent', gca, ...
    'position', [x(1),y(1),x(2)-x(1),y(2)-y(1)], ...
    'HeadLength', 10, 'HeadWidth', 10, 'HeadStyle', 'cback1', ...
    props{:} );

end

que puede llamar desde su script de la siguiente manera:

drawArrow(x1,y1,[1, 100],[1, 100],{'Color','b','LineWidth',3}); hold on
drawArrow(x2,y2,[1, 100],[1, 100],{'Color','r','LineWidth',3}); hold on

Elipse #

Para trazar una elipse puedes usar su ecuación. Una elipse tiene un eje mayor y un eje menor. También queremos poder trazar la elipse en diferentes puntos centrales. Por lo tanto escribimos una función cuyas entradas y salidas son:

Inputs:
    r1,r2: major and minor axis respectively
    C: center of the ellipse (cx,cy)
Output:
    [x,y]: points on the circumference of the ellipse

Puede usar la siguiente función para obtener los puntos en una elipse y luego trazar esos puntos.

function [x,y] = getEllipse(r1,r2,C)
beta = linspace(0,2*pi,100);
x = r1*cos(beta) - r2*sin(beta);
y = r1*cos(beta) + r2*sin(beta);
x = x + C(1,1);
y = y + C(1,2);
end

Ejemplo:

[x,y] = getEllipse(1,0.3,[2 3]);
plot(x,y);

Polígono(s) #

Cree vectores para contener las ubicaciones x e y de los vértices, introdúzcalos en parche.

polígono único #

X=rand(1,4); Y=rand(1,4);
h=patch(X,Y,'red');

Múltiples polígonos #

Los vértices de cada polígono ocupan una columna de cada X, Y.

X=rand(4,3); Y=rand(4,3);
for i=2:3
  X(:,i)=X(:,i)+(i-1); % create horizontal offsets for visibility
end

h=patch(X,Y,'red');

Parcela pseudo 4D #

Una matriz (m x n) se puede representar mediante una superficie usando surf;

El color de la superficie se establece automáticamente en función de los valores de la matriz (m x n). Si no se especifica el mapa de colores, se aplica el predeterminado.

Se puede agregar una barra de colores para mostrar el mapa de colores actual e indicar el mapeo de datos valores en el mapa de colores.

En el siguiente ejemplo, la matriz z (m x n) es generada por la función:

z=x.*y.*sin(x).*cos(y);

sobre el intervalo [-pi,pi]. Los valores x e y se pueden generar utilizando meshgrid función y la superficie se representa de la siguiente manera:

% Create a Figure
figure
% Generate the `x` and `y` values in the interval `[-pi,pi]`
[x,y] = meshgrid([-pi:.2:pi],[-pi:.2:pi]);
% Evaluate the function over the selected interval
z=x.*y.*sin(x).*cos(y);
% Use surf to plot the surface
S=surf(x,y,z);
xlabel('X Axis');
ylabel('Y Axis');
zlabel('Z Axis');
grid minor
colormap('hot')
colorbar

Figura 1

Ahora podría darse el caso de que la información adicional esté vinculada a los valores de la matriz z y se almacenan en otra matriz (m x n)

Es posible agregar esta información adicional en la trama modificando el forma en que se colorea la superficie.

Esto permitirá tener una especie de gráfico 4D: a la representación 3D de la superficie generada por la primera matriz (m x n), la cuarta dimensión estará representada por la datos contenidos en la segunda matriz (m x n).

Es posible crear una trama de este tipo llamando a surf con 4 entradas:

surf(x,y,z,C)

donde el parámetro C es la segunda matriz (que tiene que ser del mismo tamaño que z) y se utiliza para definir el color de la superficie.

En el siguiente ejemplo, la matriz C es generada por la función:

C=10*sin(0.5*(x.^2.+y.^2))*33;

sobre el intervalo [-pi,pi]

La superficie generada por C es

Figura 2

Ahora podemos llamar a surf con cuatro entradas:

figure
surf(x,y,z,C)
% shading interp
xlabel('X Axis');
ylabel('Y Axis');
zlabel('Z Axis');
grid minor
colormap('hot')
colorbar

Figura 3

Comparando la Figura 1 y la Figura 3, podemos notar que:

  • la forma de la superficie corresponde a los valores z (la primera matriz (m x n))
  • el color de la superficie (y su rango, dado por la barra de colores) corresponde a la Valores C (la primera matriz (m x n))

Figura 4

Por supuesto, es posible intercambiar z y C en el gráfico para tener la forma de la superficie dada por la matriz C y el color dado por la matriz z:

figure
surf(x,y,C,z)
% shading interp
xlabel('X Axis');
ylabel('Y Axis');
zlabel('Z Axis');
grid minor
colormap('hot')
colorbar

Dibujo rápido #

Hay tres formas principales de hacer gráficos secuenciales o animaciones: plot(x,y), set(h , 'XData' , y, 'YData' , y) y animatedline. Si desea que su animación sea fluida, necesita un dibujo eficiente y los tres métodos no son equivalentes.

% Plot a sin with increasing phase shift in 500 steps
x = linspace(0 , 2*pi , 100);

figure
tic
for thetha = linspace(0 , 10*pi , 500)
    y = sin(x + thetha);
    plot(x,y)
    drawnow
end
toc

Obtengo 5.278172 segundos. La función de trazado básicamente elimina y recrea el objeto de línea cada vez. Una forma más eficiente de actualizar un gráfico es usar las propiedades XData e YData del objeto Line.

tic
h = [];   % Handle of line object
for thetha = linspace(0 , 10*pi , 500)
    y = sin(x + thetha);
    
    if isempty(h)
        % If Line still does not exist, create it
        h = plot(x,y);
    else
        % If Line exists, update it
        set(h , 'YData' , y)
    end
    drawnow
end
toc

Ahora obtengo 2.741996 segundos, ¡mucho mejor!

animatedline es una función relativamente nueva, introducida en 2014b. Veamos cómo le va:

tic
h = animatedline;
for thetha = linspace(0 , 10*pi , 500)
    y = sin(x + thetha);
    clearpoints(h)
    addpoints(h , x , y)
    drawnow
end
toc

3.360569 segundos, no tan bueno como actualizar un gráfico existente, pero aún mejor que plot(x,y).

Por supuesto, si tiene que trazar una sola línea, como en este ejemplo, los tres métodos son casi equivalentes y brindan animaciones fluidas. Pero si tiene gráficos más complejos, la actualización de los objetos Línea existentes marcará la diferencia.
