Sintaxis #

  • si expresión … fin
  • si expresión … si no … final
  • si expresión … elseif expresión … fin
  • if expresión … elseif expresión … else … fin

Parámetros #

Parámetro Descripción
expresión una expresión que tiene un significado lógico

SI condición #

Las condiciones son una parte fundamental de casi cualquier parte del código. Se utilizan para ejecutar algunas partes del código solo en algunas situaciones, pero no en otras. Veamos la sintaxis básica:

a = 5;
if a > 10    % this condition is not fulfilled, so nothing will happen
    disp('OK')
end

if a < 10    % this condition is fulfilled, so the statements between the if...end are executed
    disp('Not OK')
end

Producción:

Not OK

En este ejemplo, vemos que si consta de 2 partes: la condición y el código que se ejecutará si la condición es verdadera. El código es todo lo escrito después de la condición y antes del ‘fin’ de ese ‘si’. La primera condición no se cumplió y, por lo tanto, el código que contenía no se ejecutó.

Aquí hay otro ejemplo:

a = 5;
if a ~= a+1        % "~=" means "not equal to"
    disp('It''s true!') % we use two apostrophes to tell MATLAB that the ' is part of the string
end

La condición anterior siempre será verdadera y mostrará el resultado “¡Es cierto!”.

También podemos escribir:

a = 5;
if a == a+1    % "==" means "is equal to", it is NOT the assignment ("=") operator
    disp('Equal')
end

Esta vez, la condición siempre es falsa, por lo que nunca obtendremos la salida Igual.

Sin embargo, no sirven de mucho las condiciones que siempre son verdaderas o falsas, porque si siempre son falsas, simplemente podemos eliminar esta parte del código, y si siempre son verdaderas, entonces la condición no es necesaria.

Condición IF-ELSE #

En algunos casos queremos ejecutar un código alternativo si la condición es falsa, para esto usamos la parte opcional else:

a = 20;
if a < 10
    disp('a smaller than 10')
else
    disp('a bigger than 10')
end

Aquí vemos que debido a que a no es menor que 10, la segunda parte del código, después de ejecutar else, obtenemos el resultado a mayor que 10. Ahora veamos otro intento:

a = 10;
if a > 10
    disp('a bigger than 10')
else
    disp('a smaller than 10')
end

En este ejemplo, se muestra que no verificamos si a es realmente menor que 10, y recibimos un mensaje incorrecto porque la condición solo verifica la expresión tal como es, y CUALQUIER caso que no sea igual a verdadero (a = 10 ) hará que se ejecute la segunda parte.

Este tipo de error es una trampa muy común tanto para principiantes como para programadores experimentados, especialmente cuando las condiciones se vuelven complejas, y siempre debe tenerse en cuenta.

Condición IF-ELSEIF #

Usando else podemos realizar alguna tarea cuando la condición no se cumple. Pero qué pasa si queremos verificar una segunda condición en caso de que la primera sea falsa. Podemos hacerlo de esta manera:

a = 9;
if mod(a,2)==0   % MOD - modulo operation, return the remainder after division of 'a' by 2
    disp('a is even')
else
    if mod(a,3)==0
        disp('3 is a divisor of a')
    end
end

OUTPUT:
3 is a divisor of a

Esto también se llama [“condición anidada”](https://www.wikiod.com/es/matlab/condiciones#Condiciones anidadas), pero aquí tenemos un caso especial que puede mejorar la legibilidad del código y reducir el posibilidad de un error - podemos escribir:

a = 9;
if mod(a,2)==0
    disp('a is even')
elseif mod(a,3)==0
    disp('3 is a divisor of a')
end

OUTPUT:
3 is a divisor of a

usando elseif podemos verificar otra expresión dentro del mismo bloque de condición, y esto no se limita a un intento:

a = 25;
if mod(a,2)==0
    disp('a is even')
elseif mod(a,3)==0
    disp('3 is a divisor of a')
elseif mod(a,5)==0
    disp('5 is a divisor of a')
end

OUTPUT:
5 is a divisor of a

Se debe tener mucho cuidado al elegir usar elseif en una fila, ya que solo uno de ellos se ejecutará desde todos los bloques if a end. Entonces, en nuestro ejemplo, si queremos mostrar todos los divisores de a (de los que verificamos explícitamente), el ejemplo anterior no será bueno:

a = 15;
if mod(a,2)==0
    disp('a is even')
elseif mod(a,3)==0
    disp('3 is a divisor of a')
elseif mod(a,5)==0
    disp('5 is a divisor of a')
end

OUTPUT:
3 is a divisor of a

no solo 3, sino que también 5 es divisor de 15, pero no se llega a la parte que comprueba la división por 5 si alguna de las expresiones anteriores es cierta.

Finalmente, podemos agregar un else (y solo one) después de todas las condiciones elseif para ejecutar un código cuando no se cumple ninguna de las condiciones anteriores:

a = 11;
if mod(a,2)==0
    disp('a is even')
elseif mod(a,3)==0
    disp('3 is a divisor of a')
elseif mod(a,5)==0
    disp('5 is a divisor of a')
else
    disp('2, 3 and 5 are not divisors of a')
end

OUTPUT:
2, 3 and 5 are not divisors of a

Condiciones anidadas #

Cuando usamos una condición dentro de otra condición, decimos que las condiciones están “anidadas”. La opción [elseif](https://www.wikiod.com/es/matlab/condiciones#Condición IF-ELSEIF) proporciona un caso especial de condiciones anidadas, pero hay muchas otras formas de usar condiciones anidadas. Examinemos el siguiente código:

a = 2;
if mod(a,2)==0    % MOD - modulo operation, return the remainder after division of 'a' by 2
    disp('a is even')
    if mod(a,3)==0
        disp('3 is a divisor of a')
        if mod(a,5)==0
            disp('5 is a divisor of a')
        end
    end
else
    disp('a is odd')
end

Para a=2, la salida será a is even, lo cual es correcto. Para a=3, la salida será a es impar, lo que también es correcto, pero no verifica si 3 es un divisor de a. Esto se debe a que las condiciones están anidadas, por lo que solo si la primera es verdadera, nos movemos a la interna, y si a es impar, ninguna de las condiciones internas se verifica. Esto es algo opuesto al uso de ’elseif’, donde solo si la primera condición es ‘falsa’, verificamos la siguiente. ¿Qué hay de comprobar la división por 5? solo se verificará la división por 5 de un número que tenga 6 como divisor (tanto 2 como 3), y podemos probar y ver que para a=30 el resultado es:

a is even
3 is a divisor of a
5 is a divisor of a

También debemos notar dos cosas:

  1. La posición del ‘fin’ en el lugar correcto para cada ‘si’ es crucial para que el conjunto de condiciones funcione como se espera, por lo que la sangría es más que una buena recomendación aquí.
  2. La posición de la declaración else también es crucial, porque necesitamos saber en qué if (y podría haber varios de ellos) queremos hacer algo en caso de que la expresión si sea falsa.

Veamos otro ejemplo:

for a = 5:10    % the FOR loop execute all the code within it for every a from 5 to 10
    ch = num2str(a);    % NUM2STR converts the integer a to a character
    if mod(a,2)==0
        if mod(a,3)==0
            disp(['3 is a divisor of ' ch])
        elseif  mod(a,4)==0
            disp(['4 is a divisor of ' ch])
        else
            disp([ch ' is even'])
        end
    elseif mod(a,3)==0
        disp(['3 is a divisor of ' ch])
        
    else
        disp([ch ' is odd'])
    end
end

Y la salida será:

5 is odd
3 is a divisor of 6
7 is odd
4 is a divisor of 8
3 is a divisor of 9
10 is even

vemos que obtuvimos solo 6 líneas para 6 números, porque las condiciones están anidadas de manera que aseguran solo una impresión por número, y también (aunque no se puede ver directamente desde la salida) no se realizan comprobaciones adicionales, por lo que si un número no es par, no tiene sentido verificar si 4 es uno de sus divisores.
