Sintaxis #

  1. Y = fft(X) % calcula la FFT de Vector o Matrix X utilizando una longitud de transformación predeterminada de 256 (a confirmar para la versión)

  2. Y = fft(X,n) % calcula la FFT de X usando n como longitud de transformación, n debe ser un número basado en 2 potencias. Si la longitud de X es menor que n, entonces Matlab automáticamente rellenará X con ceros de modo que la longitud (X) = n

  3. Y = fft(X,n,dim) % calcula la FFT de X usando n como longitud de transformación a lo largo de la dimensión dim (puede ser 1 o 2 para horizontal o vertical respectivamente)

  4. Y = fft2(X) % Calcule la FFT 2D de X

  5. Y = fftn(X, dim) % Calcule la FFT dim-dimensional de X, con respecto al vector de dimensiones dim.

  6. y = ifft(X) % calcula la inversa de FFT de X (que es una matriz/vector de números) usando la longitud de transformación 256 predeterminada

  7. y = ifft(X,n) % calcula la IFFT de X usando n como longitud de transformación

  8. y = ifft(X,n,dim) % calcula la IFFT de X usando n como longitud de transformación sobre la dimensión dim (puede ser 1 o 2 para horizontal o vertical respectivamente)

  9. y = ifft(X,n,dim,‘simétrico’) %La opción Simétrica hace que ifft trate a X como simétrica conjugada a lo largo de la dimensión activa. Esta opción es útil cuando X no es exactamente simétrica conjugada, simplemente debido a un error de redondeo.

  10. y = ifft2(X) % Calcule el inverso 2D ft de X

  11. y= ifftn(X,dim) % Calcular el fft dim-dimensional inverso de X.

Parámetros #

Parámetro Descripción
X esta es su señal de dominio de tiempo de entrada, debe ser un vector de números.
n este es el parámetro NFFT conocido como Transform Length, considérelo como la resolución de su resultado FFT, DEBE ser un número que sea una potencia de 2 (es decir, 64,128,256… 2^N)
tenue esta es la dimensión en la que desea calcular FFT, use 1 si desea calcular su FFT en dirección horizontal y 2 si desea calcular su FFT en dirección vertical. Tenga en cuenta que este parámetro generalmente se deja en blanco, ya que la función es capaz de detectar la dirección de su vector.

Matlab FFT es un proceso muy paralelizado capaz de manejar grandes cantidades de datos. También puede usar la GPU con una gran ventaja.

ifft(fft(X)) = X

La afirmación anterior es verdadera si se omiten los errores de redondeo.

Implementar una Transformada de Fourier simple en Matlab #

La Transformada de Fourier es probablemente la primera lección en Procesamiento Digital de Señales, su aplicación está en todas partes y es una herramienta poderosa cuando se trata de analizar datos (en todos los sectores) o señales. Matlab tiene un conjunto de poderosas cajas de herramientas para la Transformada de Fourier. En este ejemplo, usaremos la Transformada de Fourier para analizar una señal de onda sinusoidal básica y generar lo que a veces se conoce como Periodograma usando FFT:

%Signal Generation
A1=10;                % Amplitude 1
A2=10;                % Amplitude 2
w1=2*pi*0.2;          % Angular frequency 1
w2=2*pi*0.225;        % Angular frequency 2
Ts=1;                 % Sampling time
N=64;                 % Number of process samples to be generated
K=5;                  % Number of independent process realizations
sgm=1;                % Standard deviation of the noise
n=repmat([0:N-1].',1,K);             % Generate resolution
phi1=repmat(rand(1,K)*2*pi,N,1);     % Random phase matrix 1
phi2=repmat(rand(1,K)*2*pi,N,1);     % Random phase matrix 2
x=A1*sin(w1*n*Ts+phi1)+A2*sin(w2*n*Ts+phi2)+sgm*randn(N,K);   % Resulting Signal

NFFT=256;            % FFT length
F=fft(x,NFFT);       % Fast Fourier Transform Result
Z=1/N*abs(F).^2;     % Convert FFT result into a Periodogram

Tenga en cuenta que la transformada discreta de Fourier se implementa mediante la transformada rápida de Fourier (fft) en Matlab, ambas producirán el mismo resultado, pero FFT es una implementación rápida de DFT.

figure
w=linspace(0,2,NFFT);
plot(w,10*log10(Z)),grid;
xlabel('w [\pi rad/s]')
ylabel('Z(f) [dB]')
title('Frequency Range: [ 0 ,  \omega_s ]')

Imágenes y FT multidimensionales #

En imágenes médicas, espectroscopia, procesamiento de imágenes, criptografía y otras áreas de la ciencia y la ingeniería, a menudo se desea calcular transformadas de Fourier multidimensionales de imágenes. Esto es bastante sencillo en Matlab: las imágenes (multidimensionales) son solo matrices n-dimensionales, después de todo, y las transformadas de Fourier son operadores lineales: uno simplemente transforma iterativamente Fourier a lo largo de otras dimensiones. Matlab proporciona fft2 e ifft2 para hacer esto en 2-d, o fftn en n-dimensiones.

Un peligro potencial es que la transformada de Fourier de las imágenes generalmente se muestra “ordenada por el centro”, es decir, con el origen del espacio k en el medio de la imagen. Matlab proporciona el comando fftshift para intercambiar la ubicación de los componentes de CC de la transformada de Fourier de manera adecuada. Esta notación de orden facilita considerablemente la realización de técnicas comunes de procesamiento de imágenes, una de las cuales se ilustra a continuación.

Llenado cero #

Una forma “rápida y sucia” de interpolar una imagen pequeña a un tamaño más grande es transformarla en Fourier, rellenar la transformada de Fourier con ceros y luego tomar la transformada inversa. Esto interpola efectivamente entre cada píxel con una función de base en forma de sincronización, y se usa comúnmente para aumentar la escala de datos de imágenes médicas de baja resolución. Comencemos cargando un ejemplo de imagen incorporada

%Load example image 
I=imread('coins.png'); %Load example data -- coins.png is builtin to Matlab 
I=double(I); %Convert to double precision -- imread returns integers 
imageSize = size(I); % I is a 246 x 300 2D image

%Display it
imagesc(I); colormap gray; axis equal; 
%imagesc displays images scaled to maximum intensity

Ahora podemos obtener la transformada de Fourier de I. Para ilustrar lo que hace fftshift, comparemos los dos métodos:

% Fourier transform 
%Obtain the centric- and non-centric ordered Fourier transform of I
k=fftshift(fft2(fftshift(I))); 
kwrong=fft2(I); 

%Just for the sake of comparison, show the magnitude of both transforms: 
figure; subplot(2,1,1); 
imagesc(abs(k),[0 1e4]); colormap gray; axis equal; 
subplot(2,1,2); 
imagesc(abs(kwrong),[0 1e4]); colormap gray; axis equal;
%(The second argument to imagesc sets the colour axis to make the difference clear).

Ahora hemos obtenido el FT 2D de una imagen de ejemplo. Para llenarlo con ceros, queremos tomar cada espacio k, rellenar los bordes con ceros y luego tomar la transformación posterior:

%Zero fill
kzf = zeros(imageSize .* 2); %Generate a  492x600 empty array to put the result in 
kzf(end/4:3*end/4-1,end/4:3*end/4-1) = k; %Put k in the middle 
kzfwrong = zeros(imageSize .* 2); %Generate a  492x600 empty array to put the result in 
kzfwrong(end/4:3*end/4-1,end/4:3*end/4-1) = kwrong; %Put k in the middle 

%Show the differences again 
%Just for the sake of comparison, show the magnitude of both transforms: 
figure; subplot(2,1,1); 
imagesc(abs(kzf),[0 1e4]); colormap gray; axis equal; 
subplot(2,1,2); 
imagesc(abs(kzfwrong),[0 1e4]); colormap gray; axis equal;
%(The second argument to imagesc sets the colour axis to make the difference clear).

En este punto, el resultado bastante normal:

Una vez que tomamos las transformaciones inversas, podemos ver que (¡correctamente!) Los datos de relleno cero proporcionan un método sensato para la interpolación:

% Take the back transform and view 
Izf = fftshift(ifft2(ifftshift(kzf))); 
Izfwrong = ifft2(kzfwrong);

figure; subplot(1,3,1); 
imagesc(abs(Izf)); colormap gray; axis equal; 
title('Zero-filled image'); 
subplot(1,3,2); 
imagesc(abs(Izfwrong)); colormap gray; axis equal;
title('Incorrectly zero-filled image'); 
subplot(1,3,3); 
imagesc(I); colormap gray; axis equal; 
title('Original image'); 
set(gcf,'color','w');

Tenga en cuenta que el tamaño de la imagen rellena con ceros es el doble que el original. Uno puede llenar cero por más de un factor de dos en cada dimensión, aunque obviamente hacerlo no aumenta arbitrariamente el tamaño de una imagen.

Consejos, sugerencias, 3D y más #

El ejemplo anterior se aplica a las imágenes en 3D (que a menudo se generan mediante técnicas de imágenes médicas o microscopía confocal, por ejemplo), pero requiere que fft2 se reemplace por fftn(I, 3), por ejemplo. Debido a la naturaleza un tanto engorrosa de escribir fftshift(fft(fftshift(...) varias veces, es bastante común definir funciones como fft2c localmente para proporcionar una sintaxis local más sencilla, como:

function y = fft2c(x)

y = fftshift(fft2(fftshift(x)));

Tenga en cuenta que la FFT es rápida, pero las transformadas de Fourier grandes y multidimensionales aún llevarán tiempo en una computadora moderna. Además, es intrínsecamente complejo: la magnitud del espacio k se mostró arriba, pero la fase es absolutamente vital; las traslaciones en el dominio de la imagen son equivalentes a una rampa de fase en el dominio de Fourier. Hay varias operaciones mucho más complejas que uno puede querer hacer en el dominio de Fourier, como filtrar frecuencias espaciales altas o bajas (multiplicándolas con un filtro) o enmascarar puntos discretos correspondientes al ruido. En consecuencia, hay una gran cantidad de código generado por la comunidad para manejar operaciones comunes de Fourier disponible en el sitio principal del repositorio de la comunidad de Matlab, File Exchange.

Transformadas de Fourier inversas #

Uno de los principales beneficios de la transformada de Fourier es su capacidad de invertir de nuevo en el dominio del tiempo sin perder información. Consideremos la misma señal que usamos en el ejemplo anterior:

A1=10;                % Amplitude 1
A2=10;                % Amplitude 2
w1=2*pi*0.2;          % Angular frequency 1
w2=2*pi*0.225;        % Angular frequency 2
Ts=1;                 % Sampling time
N=64;                 % Number of process samples to be generated
K=1;                  % Number of independent process realizations
sgm=1;                % Standard deviation of the noise
n=repmat([0:N-1].',1,K);             % Generate resolution
phi1=repmat(rand(1,K)*2*pi,N,1);     % Random phase matrix 1
phi2=repmat(rand(1,K)*2*pi,N,1);     % Random phase matrix 2
x=A1*sin(w1*n*Ts+phi1)+A2*sin(w2*n*Ts+phi2)+sgm*randn(N,K);   % Resulting Signal

NFFT=256;            % FFT length
F=fft(x,NFFT);       % FFT result of time domain signal

Si abrimos F en Matlab, encontraremos que es una matriz de números complejos, una parte real y una parte imaginaria. Por definición, para recuperar la señal original en el Dominio del Tiempo, necesitamos tanto la Real (que representa la variación de Magnitud) como la Imaginaria (que representa la variación de Fase), por lo que para volver al Dominio del Tiempo, uno puede simplemente querer:

TD = ifft(F,NFFT);   %Returns the Inverse of F in Time Domain

Tenga en cuenta aquí que TD devuelto tendría una longitud de 256 porque configuramos NFFT en 256, sin embargo, la longitud de x es solo 64, por lo que Matlab agregará ceros al final de la transformación de TD. Entonces, por ejemplo, si NFFT fue 1024 y la longitud fue 64, entonces TD devuelto será 64 + 960 ceros. También tenga en cuenta que debido al redondeo de punto flotante, puede obtener algo como 3.1 * 10e-20 pero para fines generales: para cualquier X, ifft (fft (X)) es igual a X dentro del error de redondeo.

Digamos por un momento que después de la transformación, hicimos algo y solo nos queda la parte REAL de la FFT:

R = real(F);         %Give the Real Part of the FFT
TDR = ifft(R,NFFT);  %Give the Time Domain of the Real Part of the FFT

Esto significa que estamos perdiendo la parte imaginaria de nuestra FFT y, por lo tanto, estamos perdiendo información en este proceso inverso. Para conservar el original sin perder información, siempre debe mantener la parte imaginaria de la FFT usando imag y aplicar sus funciones a ambas o a la parte real.

figure
subplot(3,1,1)
plot(x);xlabel('time samples');ylabel('magnitude');title('Original Time Domain Signal')
subplot(3,1,2)
plot(TD(1:64));xlabel('time samples');ylabel('magnitude');title('Inverse Fourier Transformed - Time Domain Signal')
subplot(3,1,3)
plot(TDR(1:64));xlabel('time samples');ylabel('magnitude');title('Real part of IFFT transformed Time Domain Signal')
