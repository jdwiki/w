Referencias de bases de datos

  • Misma referencia de base de datos
  • Referencias de bases de datos de terceros

En esta página

  • Misma referencia de base de datos
  • Referencias de bases de datos de terceros

SQL Server incluye una función denominada denominación retrasada, lo que significa que, en muchos casos, puede implementar procedimientos y códigos que hacen referencia a objetos que no existen. También es posible eliminar o modificar un objeto de tal manera que cualquier parte del código de referencia ya no se ejecute cuando se le llame.

Cuando obtiene cualquiera de estas dos situaciones, solo sabe que tiene un problema cuando el código se ejecuta y falla.

SSDT ayuda con esto al verificar que las referencias a los objetos sean válidas cuando se compila el proyecto. Este es uno de los principales beneficios de SSDT y significa que los errores se pueden encontrar en tiempo de compilación en lugar de tiempo de ejecución.

Hay tres tipos de fuente de Referencia de base de datos:

  • Otros proyectos en la misma solución de estudio visual
  • Dacpacs preconstruidos/suministrados para bases de datos del sistema (msdb y master)
  • Dacpacs preconstruidos para otras bases de datos / proyectos ssdt que cree

Una vez que tenga una referencia, hay tres formas diferentes de usarlas en SSDT que se corresponden con las diferentes formas en que podemos hacer referencia a objetos en SQL Server:

  • Misma base de datos
  • Base de datos diferente, mismo servidor
  • Base de datos diferente, servidor diferente

Esto nos permite usar estos nombres:

  • esquema.tabla
  • base de datos.esquema.tabla
  • servidor.base de datos.esquema.tabla

Esto admite permitir:

  • Diferentes proyectos para hacer una base de datos.
  • Llamadas cruzadas de bases de datos en el mismo servidor
  • Llamadas cruzadas de bases de datos a través de servidores vinculados

Las referencias de la base de datos son clave para poner en marcha SSDT, comprenda las diferentes formas en que se pueden usar

Misma referencia de base de datos #

La referencia Misma base de datos le permite dividir una única base de datos en varios proyectos. Esto es útil para casos donde un proyecto es muy grande o donde diferentes equipos manejan diferentes partes de la base de datos.

Si considera que tiene dos proyectos de base de datos SSDT .sqlproj con la siguiente estructura:

Proyecto1 - tabla_a Proyecto2 - proc_a

proc_a lee de table_a usando el código:

select column from table_a

Si table_a no está en el mismo proyecto, SSDT no puede validar que existe column en la tabla. En este caso, se puede agregar una referencia de “misma base de datos” a “Proyecto2” que hace referencia al dacpac creado por “Proyecto1”.

Para agregar una referencia de Misma base de datos, haga clic con el botón derecho en la carpeta Referencias en el Explorador de soluciones y elija agregar una Referencia de base de datos, luego se le presenta el siguiente cuadro de diálogo:

Elija la fuente de la referencia, es decir, otro proyecto en la misma solución o un dacpac. Tenga en cuenta que una base de datos del sistema no se puede agregar como una referencia de “Misma base de datos”, pero debido a la forma en que SQL Server resuelve los objetos, aún puede llamarlos usando las dos partes, el esquema y el nombre de la tabla.

Una vez que haya agregado la referencia, puede llamar a los objetos en el proyecto al que se hace referencia utilizando el nombre estándar de 1 o 2 partes, como:

select column from table_a

o

select column from schema.table_a
