Selenium Grid es un marco para ejecutar pruebas distribuidas en una variedad de dispositivos de prueba. Se utiliza para probar aplicaciones web. Es posible escribir pruebas en diferentes lenguajes de programación populares, incluidos C#, Groovy, Java, Perl, PHP, Python y Ruby. Las pruebas se pueden ejecutar en una variedad de navegadores web en plataformas como Windows, Linux y OS X.

Es un software de código abierto, lanzado bajo la licencia Apache 2.0: los desarrolladores web pueden descargarlo y usarlo sin cargo.

Sintaxis #

  • para ejecutar el archivo jar, la siguiente es la sintaxis para cada archivo jar
  • java -jar <jar-file-full-name>.jar -<sus parámetros, si los hay>

Parámetros #

Parámetros Detalles
papel Es lo que le dice al selenium que era hub o node
puerto Esto es para especificar qué puerto debería estar escuchando el hub o node.
centro Este parámetro se usa en nodo para especificar la URL del concentrador
nombre del navegador Se ha utilizado en nodo para especificar el nombre del navegador como firefox, chrome, internet explorer
instancias máximas Es donde se especifica la instancia del navegador, por ejemplo. 5 significa que habrá 5 instancias del navegador que el usuario especificó estará presente.
configuración de nodo Un archivo de configuración Json para el nodo. Puede especificar el rol, el puerto, etc. aquí
HubConfig Un archivo de configuración Json para el nodo. Puede especificar el rol, el puerto, las instancias máximas, etc. aquí

Creación de un centro y nodo Selenium Grid #

Creando un centro #

Una configuración rápida para una configuración de concentrador y nodo en la red de selenio. Para obtener más información, consulte: [Documentos de Grid 2] 1

Requisitos #

Para configurar un centro de red, necesita el flujo:

  • Selenium-servidor-independiente-<Versión>.jar

Creando el centro #

Para crear un concentrador, debe ejecutar el servidor de selenio.

  1. Descargue Selenium-server-standalone-<Versión>.jar
  2. Abra su terminal y navegue a la carpeta donde se encuentra Selenium-server-standalone-.jar
  3. Ejecute el siguiente comando:
    1. For default configuration java -jar selenium-server-standalone-<Version>.jar -role hub
    2. For Json configuration java -jar selenium-server-standalone-<Version>.jar -role hub -hubConfig hubConfig.json
  4. Abra http://localhost:4444/ verá el siguiente mensaje

Al hacer clic en consola -> Ver configuración para ver la configuración de los detalles del concentrador.

Creación de un nodo #

Requisitos #

Para configurar un centro de red, necesita el flujo:

  • Selenium-servidor-independiente-<Versión>.jar
  • Controladores web
    • Chrome driver
    • FireFox driver
    • Microsoft Edge driver
  • Navegadores
    • Chrome
    • FireFox
    • Microsoft Edge (Windows 10)

Creando el Nodo #

Ahora para crear nodos para el concentrador

  1. Descargue Selenium-server-standalone-<Versión>.jar
  2. Descarga los navegadores en los que quieras probar
  3. Descargue los controladores para los navegadores en los que desea probar
  4. Abra una nueva terminal y navegue hasta la ubicación del archivo jar del servidor Selenium
  5. Ejecute el siguiente comando:
    1. for default configuration java -jar selenium-server-standalone-<VERSION NUMBER>.jar -role node
    2. For Json configuration java -jar selenium-server-standalone-<Version>.jar -role node -nodeConfig nodeConfig.json
  6. Ahora vaya a http://localhost:4444/grid/console para ver los detalles del nodo

Código Java para Selenium Grid #

String hubUrl = "http://localhost:4444/wd/hub"       
DesiredCapabilities capability = DesiredCapabilities.firefox(); //or which browser you want
RemoteWebDriver driver = new RemoteWebDriver(hubUrl, capability);
