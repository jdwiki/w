Creación de almacenes de claves y configuración de Tomcat #

Este artículo explica los pasos para configurar Tomcat para solicitar certificados CAC del cliente. Se centra en configurar un entorno de desarrollo, por lo que algunas características que deben tenerse en cuenta para la producción no se encuentran aquí. (Por ejemplo, muestra el uso de un certificado autofirmado para https y no considera verificar los certificados revocados).

Crear almacén de claves para habilitar conexiones HTTPS

El primer paso es configurar SSL en Tomcat. Esto está documentado en el sitio web de Tomcat aquí: https://tomcat.apache.org/tomcat-8.5-doc/ssl-howto.html para completar, los pasos para configurarlo con un certificado autofirmado se enumeran a continuación:

Necesitamos crear un archivo de almacén de claves que contenga el certificado SSL para el servidor. El certificado es lo que se requiere para crear una conexión https y no tiene nada que ver con hacer que el servidor solicite certificados CAC del cliente, pero se requieren conexiones https para la autenticación del certificado del cliente. Para un entorno de desarrollo, la creación de un certificado autofirmado está bien, pero se desaconseja para la producción. Java viene empaquetado con una utilidad llamada keytool (http://docs.oracle.com/javase/8/docs/technotes/tools/windows/keytool.html) que se utiliza para gestionar certificados y almacenes de claves. Se puede usar para crear un certificado autofirmado y agregarlo a un almacén de claves. Para hacer eso, puede emitir el siguiente comando desde un símbolo del sistema:

keytool -genkey -alias tomcat -keyalg RSA -keystore \ruta\a\mi\keystore -storepass changeit

Se le pedirá varios bits de información y luego se creará un archivo de almacén de claves llamado “\ruta\a\mi\almacén de claves” con una contraseña de “cambiarlo” y contendrá el certificado autofirmado generado.

Crear almacén de confianza que contenga certificados raíz del Departamento de Defensa

Lo siguiente que se necesita es crear un almacén de confianza que contenga los certificados raíz del Departamento de Defensa. Tomcat considerará que los certificados de este almacén de confianza son de confianza y solo aceptará certificados de cliente que tengan uno de los certificados de confianza en su cadena de certificados.

Para crear el almacén de confianza, necesitamos obtener una copia de los certificados raíz del Departamento de Defensa. Para hacer esto, descargue “InstallRoot 5.0” de http://militarycac.com/dodcerts.htm. Instálalo y ejecútalo y luego ejecútalo. Expanda el panel Instalar certificados DoD y haga clic en la pestaña Certificado:

A continuación, seleccione los tres certificados de CA raíz del DoD de la lista de certificados y haga clic en “PEM” en el grupo de herramientas Exportar:

Después de hacer clic en el botón de exportación “PEM”, elija una ubicación para exportar los certificados y haga clic en Aceptar. Esto debería haber creado tres archivos .cer en el directorio que seleccionó. Abra un símbolo del sistema y navegue a ese directorio.

Aquí usaremos el comando keytool para importar los certificados a un almacén de confianza. Ejecute los siguientes comandos para importar los tres certificados:

keytool -importcert -file DoD_Root_CA_2__0x05__DoD_Root_CA_2.cer -alias DODRoot2 -keystore truststore.jks -storepass changeit

keytool -importcert -file DoD_Root_CA_3__0x01__DoD_Root_CA_3.cer -alias DODRoot3 -keystore truststore.jks -storepass changeit

keytool -importcert -file DoD_Root_CA_4__0x01__DoD_Root_CA_4.cer -alias DODRoot4 -keystore truststore.jks -storepass changeit

Esto creará un archivo truststore.jks con una contraseña de “cambio” en el directorio de trabajo actual. Contendrá los tres certificados raíz del Departamento de Defensa, puede verlo ejecutando:

keytool -list -keystore truststore.jks

Que debería enumerar algo como:

Su almacén de claves contiene 3 entradas

dodroot4, 23 de septiembre de 2016, entrada de certificado de confianza, huella digital del certificado (SHA1): B8:26:9F:25:DB:D9:37:EC:AF:D4:C3:5A:98:38:57:17:23:F2:D0:26 dodroot3, 23 de septiembre de 2016, entrada de certificado de confianza, huella digital del certificado (SHA1): D7:3C:A9:11:02:A2:20:4A:36:45:9E:D3:22:13:B4:67:D7:CE:97:FB dodroot2, 23 de septiembre de 2016, entrada de certificado de confianza, huella digital del certificado (SHA1): 8C:94:1B:34:EA:1E:A6:ED:9A:E2:BC:54:CF:68:72:52:B4:C9:B5:61

Configure Tomcat para usar Keystore y Truststore

Ahora tenemos los archivos de almacén de claves y almacén de confianza que necesitamos, lo siguiente es configurar Tomcat para usarlos. Para ello debemos cambiar el archivo /conf/server.xml. Abra el archivo y agregue una definición de conector como la siguiente:

<Connector 
    clientAuth="true" 
    keystoreFile="path/to/keystore.jks" 
    keystorepass="changeit" 
    keystoreType="jks" 
    truststoreFile="path/to/truststore.jks" 
    truststoreType="jks" 
    truststorepass="changeit"
    maxThreads="150" 
    port="8443" 
    protocol="org.apache.coyote.http11.Http11NioProtocol" 
    scheme="https" 
    secure="true" 
    sslProtocol="TLS" 
    SSLEnabled="true" 
    />

Puede ir aquí para obtener una definición más detallada de todos los atributos: http://tomcat.apache.org/tomcat-7.0-doc/config/http.html

Una vez que todo esto esté hecho, inicie Tomcat. Desde una computadora que tenga un lector de CAC con un CAC insertado, navegue hasta la url https://:8443/ y, si todo está configurado correctamente, se le pedirá que elija un certificado de la tarjeta CAC.
