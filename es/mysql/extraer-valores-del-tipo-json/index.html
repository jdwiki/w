Extraer valores del tipo JSON

  • Sintaxis
  • Parámetros
  • Lee el valor de la matriz JSON
  • Operadores de extracción JSON

En esta página

  • Sintaxis
  • Parámetros
  • Lee el valor de la matriz JSON
  • Operadores de extracción JSON

MySQL 5.7.8+ admite el tipo JSON nativo. Si bien tiene diferentes formas de crear objetos json, también puede acceder y leer miembros de diferentes maneras.

La función principal es JSON_EXTRACT, por lo que los operadores -> y ->> son más amigables.

Sintaxis #

  • JSON_EXTRACT(json_doc,ruta[,…])
  • JSON_EXTRACT(json_doc,ruta)
  • JSON_EXTRACT(json_doc,ruta1,ruta2)

Parámetros #

Parámetro Descripción
json_doc documento JSON válido
ruta ruta de los miembros

Mencionado en [Manual de referencia de MySQL 5.7] (https://dev.mysql.com/doc/refman/5.7/en/json-search-functions.html#function_json-extract)

  • Múltiples valores coincidentes por argumento(s) de ruta

Si es posible que esos argumentos puedan devolver varios valores, los valores coincidentes se ajustan automáticamente como una matriz, en el orden correspondiente a las rutas que los produjeron. De lo contrario, el valor devuelto es el único valor coincidente.

  • NULL Resultado cuando:
  • cualquier argumento es NULL
  • ruta no coincidente

Devuelve NULL si algún argumento es NULL o ninguna ruta localiza un valor en el documento.

Lee el valor de la matriz JSON #

Cree la variable @myjson como tipo JSON (leer más):

 SET @myjson = CAST('["A","B",{"id":1,"label":"C"}]' as JSON) ;

¡SELECCIONA algunos miembros!

 SELECT
   JSON_EXTRACT( @myjson , '$[1]' ) ,
   JSON_EXTRACT( @myjson , '$[*].label') ,
   JSON_EXTRACT( @myjson , '$[1].*' ) ,
   JSON_EXTRACT( @myjson , '$[2].*')
 ;
 -- result values:
 '\"B\"', '[\"C\"]', NULL, '[1, \"C\"]'
 -- visually:
 "B", ["C"], NULL, [1, "C"]

Operadores de extracción JSON #

Extrae ruta por operadores -> o ->>, mientras que ->> es un valor SIN COMILLAS:

 SELECT
   myjson_col->>'$[1]' , myjson_col->'$[1]' ,
   myjson_col->>'$[*].label' ,
   myjson_col->>'$[1].*' ,
   myjson_col->>'$[2].*'
 FROM tablename ;
  -- visuall:
     B, "B" , ["C"], NULL, [1, "C"]
  --^^^ ^^^

Entonces col->>path es igual a JSON_UNQUOTE(JSON_EXTRACT(col,path)) :

Al igual que con ->, el operador -» siempre se expande en la salida de EXPLAIN, como demuestra el siguiente ejemplo:

mysql> EXPLAIN SELECT c-»’$.name’ AS nombre -> DESDE jemp DONDE g > 2\G *************************** 1. fila ******************* ******** identificación: 1 seleccionar_tipo: SIMPLE tabla: jemp particiones: NULL tipo: gama posibles_claves: i tecla: yo key_len: 5 ref: NULO filas: 2 filtrado: 100.00 Extra: Usando donde 1 fila en conjunto, 1 advertencia (0,00 seg)

mysql> MOSTRAR ADVERTENCIAS\G *************************** 1. fila ******************* ******** Nivel: Nota Código: 1003 Mensaje: /* seleccione#1 */ seleccione json_unquote(json_extract(jtest.jemp.c,’$.name’)) COMO nombre de jtest.jemp donde (jtest.jemp.g > 2) 1 fila en conjunto (0.00 seg) Leer sobre [extracción de ruta en línea (+)] (https://dev.mysql.com/doc/refman/5.7/en/json-search-functions.html#operator_json-inline-path)
