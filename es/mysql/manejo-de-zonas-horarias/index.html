Manejo de zonas horarias

En esta página

  • Recuperar la fecha y hora actual en una zona horaria particular.
  • Convierte un valor DATE o DATETIME almacenado a otra zona horaria.
  • Recuperar valores TIMESTAMP almacenados en una zona horaria particular
  • ¿Cuál es la configuración de la zona horaria local de mi servidor?
  • ¿Qué valores de time_zone están disponibles en mi servidor?

Cuando necesite manejar información de tiempo para una base de usuarios mundial en MySQL, use el tipo de datos TIMESTAMP en sus tablas.

Para cada usuario, almacene una columna de zona horaria de preferencia de usuario. VARCHAR(64) es un buen tipo de datos para esa columna. Cuando un usuario se registre para usar su sistema, solicite el valor de la zona horaria. El mío es hora del Atlántico, América/Edmonton. El suyo puede o no ser Asia/Kolkata o Australia/NSW. Para una interfaz de usuario para esta configuración de preferencia del usuario, el software WordPress.org tiene un buen ejemplo.

Finalmente, cada vez que establezca una conexión desde su programa host (Java, php, lo que sea) a su DBMS en nombre de un usuario, emita el comando SQL

 SET SESSION time_zone='(whatever tz string the user gave you)'

antes de manejar cualquier dato de usuario que involucre tiempos. Luego, todas las veces TIMESTAMP que haya instalado se mostrarán en la hora local del usuario.

Esto hará que todas las horas que entren en sus tablas se conviertan a UTC y que todas las horas que salgan se traduzcan a locales. Funciona correctamente para NOW() y CURDATE(). Nuevamente, debe usar TIMESTAMP y no DATETIME o DATE tipos de datos para esto.

Asegúrese de que el sistema operativo de su servidor y las zonas horarias predeterminadas de MySQL estén configuradas en UTC. Si no hace esto antes de comenzar a cargar información en su base de datos, será casi imposible solucionarlo. Si utiliza un proveedor para ejecutar MySQL, insista en que lo haga bien.

Recuperar la fecha y hora actual en una zona horaria particular. #

Esto obtiene el valor de NOW() en la hora local, en la hora estándar de la India y luego nuevamente en UTC.

SELECT NOW();
SET time_zone='Asia/Kolkata'; 
SELECT NOW();
SET time_zone='UTC'; 
SELECT NOW();

Convierte un valor DATE o DATETIME almacenado a otra zona horaria. #

Si tiene una FECHA o DATETIME almacenada (en una columna en algún lugar), se almacenó con respecto a alguna zona horaria, pero en MySQL la zona horaria no se almacena con el valor. Entonces, si desea convertirlo a otra zona horaria, puede hacerlo, pero debe conocer la zona horaria original. Usar CONVERT_TZ() hace la conversión. Este ejemplo muestra las filas vendidas en California en la hora local.

SELECT CONVERT_TZ(date_sold,'UTC','America/Los_Angeles') date_sold_local
  FROM sales
 WHERE state_sold = 'CA'

Recuperar valores TIMESTAMP almacenados en una zona horaria particular #

Esto es realmente fácil. Todos los valores de TIMESTAMP se almacenan en tiempo universal y siempre se convierten a la configuración actual de time_zone cada vez que se procesan.

SET SESSION time_zone='America/Los_Angeles'; 
SELECT timestamp_sold
  FROM sales
 WHERE state_sold = 'CA'

¿Por qué es esto? Los valores TIMESTAMP se basan en el venerable tipo de datos time_t de UNIX. Esas marcas de tiempo UNIX se almacenan como una cantidad de segundos desde 1970-01-01 00:00:00 UTC.

Aviso Los valores TIMESTAMP se almacenan en tiempo universal. Los valores DATE y DATETIME se almacenan en la hora local en vigor cuando se almacenaron.

¿Cuál es la configuración de la zona horaria local de mi servidor? #

Cada servidor tiene una configuración de zona horaria global predeterminada, configurada por el propietario de la máquina del servidor. Puede averiguar la configuración de la zona horaria actual de esta manera:

SELECT @@time_zone

Desafortunadamente, eso generalmente produce el valor SYSTEM, lo que significa que la hora de MySQL se rige por la configuración de la zona horaria del sistema operativo del servidor.

Esta secuencia de consultas (sí, es un truco) le devuelve la diferencia en minutos entre la configuración de la zona horaria del servidor y UTC.

CREATE TEMPORARY TABLE times (dt DATETIME, ts TIMESTAMP);
SET time_zone = 'UTC';
INSERT INTO times VALUES(NOW(), NOW());
SET time_zone = 'SYSTEM';
SELECT dt, ts, TIMESTAMPDIFF(MINUTE, dt, ts)offset FROM times;
DROP TEMPORARY TABLE times;

¿Como funciona esto? Las dos columnas en la tabla temporal con diferentes tipos de datos son la clave. Los tipos de datos DATETIME siempre se almacenan en la hora local en las tablas y TIMESTAMP en UTC. Entonces, la declaración INSERT, realizada cuando time_zone se establece en UTC, almacena dos valores de fecha / hora idénticos.

Luego, la declaración SELECT se realiza cuando time_zone se establece en la hora local del servidor. Los ‘TIMESTAMP’ siempre se traducen de su forma UTC almacenada a la hora local en las declaraciones SELECT. DATETIMEs no lo son. Entonces, la operación TIMESTAMPDIFF(MINUTE...) calcula la diferencia entre la hora local y la hora universal.

¿Qué valores de time_zone están disponibles en mi servidor? #

Para obtener una lista de posibles valores de time_zone en su instancia de servidor MySQL, use este comando.

 SELECT mysql.time_zone_name.name

Por lo general, esto muestra la lista de zonas horarias de ZoneInfo mantenida por Paul Eggert en la Autoridad de números asignados de Internet. En todo el mundo hay aproximadamente 600 zonas horarias.

Los sistemas operativos similares a Unix (distribuciones Linux, distribuciones BSD y distribuciones modernas de Mac OS, por ejemplo) reciben actualizaciones de rutina. La instalación de estas actualizaciones en un sistema operativo permite que las instancias de MySQL que se ejecutan allí realicen un seguimiento de los cambios en la zona horaria y los cambios de hora diurna/hora estándar.

Si obtiene una lista mucho más corta de nombres de zonas horarias, su servidor no está completamente configurado o se está ejecutando en Windows. Aquí hay instrucciones para que el administrador de su servidor instale y mantenga la lista ZoneInfo.
