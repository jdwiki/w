Sintaxis #

  • CREAR VISTA view_name COMO SELECCIONAR column_name(s) FROM table_name DONDE condición; ///Sintaxis de vista de creación simple

  • CREAR [OR REPLACE] [ALGORITHM = {UNDEFINED | MERGE | TEMPTABLE}] [DEFINER = { user | CURRENT_USER }] [SQL SECURITY { DEFINER | INVOKER }] VIEW view_name [(column_list)] AS select_statement [WITH [CASCADED | LOCAL] CHECK OPTION]; /// Full Create view syntax

  • DROP VIEW [SI EXISTE] [db_name.]view_name; ///Sintaxis de la vista de caída

Parámetros #

Parámetros Detalles
ver_nombre Nombre de la vista
instrucción SELECCIONAR Sentencias SQL para ser empaquetadas en las vistas. Puede ser una declaración SELECT para obtener datos de una o más tablas.

Las vistas son tablas virtuales y no contienen los datos que se devuelven. Pueden evitar que escriba consultas complejas una y otra vez.

  • Antes de realizar una vista, su especificación consiste enteramente en una sentencia SELECT. La instrucción SELECT no puede contener una subconsulta en la cláusula FROM.
  • Una vez que se crea una vista, se usa en gran medida como una tabla y se puede SELECCIONAR como si fuera una tabla.

Debe crear vistas, cuando desee restringir algunas columnas de su tabla, del otro usuario.

  • Por ejemplo: en su organización, desea que sus gerentes vean poca información de una tabla llamada “Ventas”, pero no desea que sus ingenieros de software puedan ver todos los campos de la tabla “Ventas”. Aquí, puede crear dos vistas diferentes para sus gerentes y sus ingenieros de software.

Actuación. VIEWs son azúcar sintáctico. Sin embargo, el rendimiento puede o no ser peor que la consulta equivalente con la selección de la vista plegada. El optimizador intenta hacer este “plegado” por usted, pero no siempre tiene éxito. MySQL 5.7.6 proporciona algunas mejoras más en el Optimizer. Pero, independientemente, el uso de VIEW no generará una consulta más rápida.

Crear una vista #

Privilegios

La sentencia CREATE VIEW requiere el privilegio CREATE VIEW para la vista y algún privilegio para cada columna seleccionada por la sentencia SELECT. Para las columnas utilizadas en otras partes de la declaración SELECT, debe tener el privilegio SELECT. Si la cláusula OR REPLACE está presente, también debe tener el privilegio DROP para la vista. CREATE VIEW también puede requerir el privilegio SUPER, según el valor DEFINER, como se describe más adelante en esta sección.

Cuando se hace referencia a una vista, se verifican los privilegios.

Una vista pertenece a una base de datos. De forma predeterminada, se crea una nueva vista en la base de datos predeterminada. Para crear la vista explícitamente en una base de datos dada, use un nombre completo

Por ejemplo:

nombre_bd.nombre_vista

mysql> CREATE VIEW test.v AS SELECT * FROM t;

Nota: dentro de una base de datos, las tablas base y las vistas comparten el mismo espacio de nombres, por lo que una tabla base y una vista no pueden tener el mismo nombre.

UNA VISTA puede:

  • ser creado a partir de muchos tipos de sentencias SELECT
  • consultar tablas base u otras vistas
  • use uniones, UNION y subconsultas
  • SELECCIONAR ni siquiera necesita referirse a ninguna tabla

Otro ejemplo

El siguiente ejemplo define una vista que selecciona dos columnas de otra tabla, así como una expresión calculada a partir de esas columnas:

mysql> CREATE TABLE t (qty INT, price INT);
mysql> INSERT INTO t VALUES(3, 50);
mysql> CREATE VIEW v AS SELECT qty, price, qty*price AS value FROM t;
mysql> SELECT * FROM v;
+------+-------+-------+
| qty  | price | value |
+------+-------+-------+
|    3 |    50 |   150 |
+------+-------+-------+

Restricciones

  • Antes de MySQL 5.7.7, la declaración SELECT no puede contener una subconsulta en la cláusula FROM.
  • La instrucción SELECT no puede hacer referencia a variables del sistema o definidas por el usuario. variables
  • Dentro de un programa almacenado, la declaración SELECT no puede referirse al programa parámetros o variables locales.
  • La sentencia SELECT no puede hacer referencia a parámetros de sentencia preparados.
  • Cualquier tabla o vista a la que se hace referencia en la definición debe existir. Después de la se ha creado una vista, es posible soltar una tabla o vista que a que se refiere la definición. En este caso, el uso de la vista da como resultado una error. Para verificar una definición de vista para problemas de este tipo, use el sentencia CHECK TABLE.
  • La definición no puede hacer referencia a una tabla TEMPORAL, y no puede crear una vista TEMPORAL.
  • No puede asociar un activador con una vista.
  • Los alias de los nombres de columna en la declaración SELECT se comparan con la longitud máxima de columna de 64 caracteres (no el alias máximo longitud de 256 caracteres).
  • Una VISTA puede o no optimizar tan bien como el equivalente SELECCIONAR. Es poco probable que se optimice mejor.

Una vista desde dos mesas #

Una vista es más útil cuando se puede utilizar para extraer datos de más de una tabla.

CREATE VIEW myview AS
SELECT a.*, b.extra_data FROM main_table a 
LEFT OUTER JOIN other_table b 
ON a.id = b.id

En mysql las vistas no se materializan. Si ahora realiza la consulta simple SELECT * FROM myview, mysql realmente realizará LEFT JOIN detrás de escena.

Una vista, una vez creada, se puede unir a otras vistas o tablas

Actualizar una tabla a través de una VISTA #

Una VISTA se parece mucho a una mesa. Aunque puede “ACTUALIZAR” una tabla, puede o no actualizar una vista en esa tabla. En general, si ‘SELECCIONAR’ en la vista es lo suficientemente complejo como para requerir una tabla temporal, entonces ‘ACTUALIZAR’ no está permitido.

Cosas como GROUP BY, UNION, HAVING, DISTINCT y algunas subconsultas impiden que la vista sea actualizable. Detalles en manual de referencia.

ABANDONANDO UNA VISTA #

– Crear y colocar una vista en la base de datos actual.

CREATE VIEW few_rows_from_t1 AS SELECT * FROM t1 LIMIT 10;
DROP VIEW few_rows_from_t1;

– Cree y suelte una vista que haga referencia a una tabla en una base de datos diferente.

CREATE VIEW table_from_other_db AS SELECT x FROM db1.foo WHERE x IS NOT NULL;
DROP VIEW table_from_other_db;
