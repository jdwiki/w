Un complemento es una secuencia de comandos o un conjunto de secuencias de comandos que cambia el comportamiento predeterminado de Vim, ya sea agregando funciones que no existen o ampliando las funciones existentes.

Las “características no existentes” que se agregan a menudo incluyen:

  • comentando,
  • detección de sangría,
  • autocompletado,
  • coincidencia aproximada,
  • soporte para un idioma específico,
  • etc.

A menudo, las “características existentes” extendidas incluyen:

  • omni-completar,
  • objetos de texto y movimientos,
  • tirar y poner,
  • línea de estado,
  • buscar y reemplazar,
  • cambio de página de búfer/ventana/pestaña,
  • plegable,
  • etc.

Conejo #

Vundle es un administrador de complementos para Vim.

Instalando Vundle #

(Los detalles completos de instalación se pueden encontrar en [Inicio rápido de Vundle] (https://github.com/VundleVim/Vundle.vim%23quick-start))

  1. Instale Git y clone Vundle en ~/.vim/bundle/Vundle.vim.

  2. Configure los complementos agregando lo siguiente en la parte superior de su .vimrc, agregando o eliminando complementos según sea necesario (los complementos en la lista son meramente ilustrativos)

     set nocompatible              " be iMproved, required
 filetype off                  " required

 " set the runtime path to include Vundle and initialize
 set rtp+=~/.vim/bundle/Vundle.vim
 call vundle#begin()
 " alternatively, pass a path where Vundle should install plugins
 "call vundle#begin('~/some/path/here')

 " let Vundle manage Vundle, required
 Plugin 'VundleVim/Vundle.vim'

 " All of your Plugins must be added before the following line
 call vundle#end()            " required
 filetype plugin indent on    " required
 " To ignore plugin indent changes, instead use:
 "filetype plugin on

 "place non-Plugin stuff after this line

  3. Instalar complementos: iniciando Vim y ejecutando :PluginInstall.

Formatos de complementos admitidos #

Los siguientes son ejemplos de diferentes formatos compatibles. Mantenga los comandos del complemento entre vundle#begin y vundle#end.

Ubicación del complemento Uso
complemento en GitHub Complemento 'tpope/vim-fugitive'
complemento de http://vim-scripts.org/vim/scripts.html Complemento 'L9'
Complemento Git no alojado en GitHub Complemento 'git://git.wincent.com/command-t.git'
git repos en su máquina local (es decir, cuando trabaja en su propio complemento) Complemento 'archivo:///inicio/gmarik/ruta/hacia/complemento'
El script sparkup vim se encuentra en un subdirectorio de este repositorio llamado vim. Pase la ruta para configurar la ruta de tiempo de ejecución correctamente. Complemento 'rstacruz/sparkup', {'rtp': 'vim/'}
Instale L9 y evite un conflicto de nombres si ya instaló una versión diferente en otro lugar. Complemento 'ascenator/L9', {'name': 'newL9'}

Trabajar en una cuenta compartida, por ejemplo, en el nodo principal del clúster puede generar problemas desde el punto de uso del disco por parte del directorio .vim. Hay un par de paquetes que ocupan una cantidad considerable de espacio en disco, por ejemplo YCM. Por lo tanto, elija sabiamente su directorio de complementos Vundle, y es muy fácil hacerlo configurando rtp. Y también si planea instalar cualquier complemento de vim, no haga directamente git clone en el directorio bundle. Usa el método Vundle.

Patógeno #

vim-pathogen es un administrador de runtimepath creado por Tim Pope para facilitar la instalación de complementos y archivos de tiempo de ejecución en sus propios directorios privados.

Instalación de patógenos #

  1. Poner patógeno en ~/.vim/bundle (aquí con Git, pero no es obligatorio):

    git clone https://github.com/tpope/vim-pathogen.git

  2. Agregue las siguientes líneas en la parte superior de su .vimrc:

     " enable vim-pathogen
 runtime bundle/vim-pathogen/autoload/pathogen.vim
 execute pathogen#infect()
  • la directiva runtime especifica la ruta al script de carga automática de vim-pathogen;
  • ejecutar patógeno#infectar() lo inicia.

Una vez iniciado, Pathogen iniciará automáticamente un barrido a través de las carpetas en ~/.vim/bundle y cargará el complemento de cada una de ellas.

Uso de patógenos #

  1. Coloque el directorio de nivel superior de su complemento en ~/.vim/bundle/ para que esté disponible la próxima vez que inicie Vim.
  2. Ejecute :Helptags para indexar la documentación de su nuevo complemento.

Beneficios #

  • Cada complemento reside en su propio directorio en ~/.vim/bundle/.
  • Su .vimrc se mantiene limpio de la configuración necesaria para cargar complementos.

El esfuerzo necesario para “gestionar” un complemento se reduce así a:

  • ponga su directorio de nivel superior en ~/.vim/bundle/ para instalarlo,
  • reemplazar su directorio de nivel superior para actualizarlo,
  • eliminar su directorio de nivel superior para desinstalarlo.

La forma en que realiza esas tres acciones (manualmente, a través de una herramienta de automatización, con Git/Svn/Hg/lo que sea…) depende completamente de usted.

Cómo funcionan los complementos #

Un complemento podría presentarse como un solo archivo que contiene 30 líneas de vimscript o como 20 MB de vimscript/python/ruby/lo que sea dividido en muchos archivos en una docena de directorios que dependen de una serie de herramientas externas.

El primero es obviamente fácil de instalar y administrar, pero el segundo podría representar un gran desafío.

El principio #

La opción 'runtimepath' le dice a Vim dónde buscar scripts de tiempo de ejecución. El valor predeterminado hace que Vim busque scripts en los siguientes directorios en orden:

  • en sistemas tipo UNIX
  • $HOME/.vim/
  • $VIM/vimfiles/
  • $VIMRUNTIME/
  • $VIM/vimfiles/después/
  • $HOME/.vim/después/
  • en Windows
  • $HOME/vimfiles/
  • $VIM/vimfiles/
  • $VIMRUNTIME/
  • $VIM/vimfiles/después/
  • $HOME/vimfiles/después/

De los directorios anteriores, solo instale complementos en los que están en negrita. Los otros causarán inestabilidad sin una buena razón. La instalación de un complemento se reduce a colocar cada uno de sus componentes en el directorio correcto bajo $HOME/.vim/ o $HOME/vimfiles/.

El método manual #

Complemento de archivo único #

Coloque el archivo en $HOME/.vim/plugin o $HOME/vimfiles/plugin

Esto generaría el complemento al inicio de Vim. Ahora el usuario podría usar todo lo definido en él. Sin embargo, si el complemento necesita activación, el usuario debe ejecutar el comando por sí mismo cuando quiera usarlo o agregar el comando a .vimrc

Paquete #

Un paquete es una estructura de directorio que utiliza el complemento. Consiste en todos los archivos del complemento en los subdirectorios correspondientes.

Para instalar dicho complemento, los subdirectorios deben fusionarse con sus contrapartes en $HOME/.vim/plugin. Sin embargo, este enfoque conduce a la mezcla de archivos de diferentes complementos en los mismos directorios y posiblemente podría generar problemas de espacio de nombres.

Otro enfoque es copiar todo el directorio en $HOME/.vim/bundle.

Al usar este enfoque, debe haber al menos un archivo .vim en el directorio $HOME/.vim/bundle/autoload. Estos archivos los obtendría vim al inicio.

Nota: Según el sistema operativo del usuario, el prefijo de todas las rutas puede ser $HOME/vimfiles. Para obtener más detalles, consulte Cómo funcionan los complementos
