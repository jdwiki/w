Archivos y directorios #

Hagas lo que hagas para personalizar Vim, NUNCA debería suceder fuera de $HOME:

  • en Linux, BSD y Cygwin, $HOME suele ser /home/username/,
  • en Mac OS X, $HOME es /Usuarios/nombre de usuario/,
  • en Windows, $HOME suele ser C:\Users\username\.

La ubicación canónica para su vimrc y su directorio vim está en la raíz de ese directorio $HOME:

  • en sistemas tipo Unix

    $HOME/.vimrc       <-- the file
$HOME/.vim/        <-- the directory

  • en Windows

    $HOME\_vimrc       <-- the file
$HOME\vimfiles\    <-- the directory

El diseño anterior está garantizado para funcionar, ahora y en el futuro.

Vim 7.4 hizo posible mantener tu adorable vimrc dentro de tu vim directorio. Realmente es una buena idea, aunque solo sea porque hace que sea más fácil mueve tu configuración.

Si usas 7.4 exclusivamente, lo siguiente será suficiente:

  • en sistemas tipo Unix

    $HOME/.vim/vimrc

  • en Windows

    $HOME\vimfiles\vimrc

Si desea los beneficios de un vim/ autónomo pero usa tanto la 7.4 como una versión anterior, o solo una versión anterior, la solución más simple y preparada para el futuro es poner esto linea y solo esta linea:

runtime vimrc

en este archivo:

  • en sistemas tipo Unix

    $HOME/.vimrc

  • en Windows

    $HOME\_vimrc

y haz tu configuración en $HOME/.vim/vimrc o $HOME\vimfiles\vimrc.

Asignaciones #

  • No pongas comentarios después de las asignaciones, se romperán las cosas.
  • Use :map <F6> para ver qué está asignado a <F6> y en qué modo.
  • Use :verbose map <F6> para ver también dónde se asignó por última vez.
  • :map y :map! son demasiado genéricos. Utilice :n[nore]map para asignaciones de modo normal, :i[nore]map para modo de inserción, :x[nore]map para modo visual, etc.

Mapeos recursivos #

Use asignaciones recursivas solo si tiene la intención de usar otras asignaciones en sus asignaciones:

nnoremap b     B
nmap     <key> db

En este ejemplo, b funciona como B en modo normal. Ya que usamos b en un mapeo recursivo, presionar <tecla> funcionará como dB, no como db.

Mapeos no recursivos #

Use asignaciones no recursivas solo si tiene la intención de usar comandos predeterminados en sus asignaciones, que es casi siempre lo que desea:

nnoremap <key> db

En este ejemplo, usamos b en un mapeo no recursivo, por lo que presionar la tecla siempre funcionará como db, ya sea que reasignemos b o no.

Ejecutar un comando desde un mapeo #

nnoremap <key> :MyCommand<CR>

Ejecutar múltiples comandos desde un mapeo #

nnoremap <key> :MyCommand <bar> MyOtherCommand <bar> SomeCommand<CR>

Llamar a una función desde un mapeo #

nnoremap <key> :call SomeFunction()<CR>

Asignando un mapeo <Plug> #

map <key> <Plug>name_of_mapping

Consulte :help map-commands, :help key-notation y :help <plug>.

consulte https://www.wikiod.com/es/vim/asignaciones-clave-en-vim para obtener más información

Alternar enumeración de líneas #

Para habilitar - escriba:

:establecer número o :establecer nu.

Para deshabilitar - escriba:

:establecer ningún número o :establecer nueve.

Para habilitar la enumeración relativa a la ubicación del cursor, escriba:

:establecer número relativo.

Para deshabilitar la enumeración relativa a la ubicación del cursor, escriba:

: establecer un número relativo.

Nota: Para cambiar si la línea actual muestra el número de línea real o 0, use los comandos :set number y :set nonumber mientras el atributo relativenumber está activo.

El archivo vimrc #

El archivo .vimrc (pronunciado Vim-wreck) es un archivo de configuración de Vim. Contiene comandos que Vim ejecutará cada vez que se inicie.

Por defecto, el archivo está vacío o no existe; puede usarlo para personalizar su entorno Vim.

Para averiguar dónde espera Vim que se almacene el archivo vimrc, abra Vim y ejecute:

:echo $MYVIMRC

Unix: en un sistema Unix como Mac o Linux, su vimrc se llamará .vimrc y generalmente estará ubicado en su directorio de inicio ($HOME/.vimrc).

Windows: en Windows se llamará _vimrc y se ubicará en su directorio de inicio (%HOMEPATH%/_vimrc).

Al iniciar, Vim buscará en varios lugares un archivo vimrc. Se utiliza el primero que existe, los demás se ignoran. Para obtener una referencia completa, consulte el artículo de documentación :h $MYVIMRC.

¿Qué opciones puedo usar? #

Si no sabe qué opciones debe usar, puede estar interesado en el comando :options.

Esto abrirá una división con todas las opciones de Vim enumeradas y con su valor actual mostrado. Hay 26 secciones para mostrar todas las opciones que puede probar.

p.ej.

4 displaying text

scroll    number of lines to scroll for CTRL-U and CTRL-D
    (local to window)
     set scr=20
scrolloff    number of screen lines to show around the cursor
     set so=5
wrap    long lines wrap
     set nowrap    wrap

...

En una línea de valor (por ejemplo, set nowrap), puede presionar CR para alternar el valor (si es un valor binario). En una línea de opción (por ejemplo, wrap long line wrap) puede presionar CR para acceder a la documentación de esta opción.

Funciones #

  • No olvide la explosión para permitir que Vim sobrescriba esa función la próxima vez que vuelva a cargar el script donde se define la función.
  • Las funciones personalizadas deben comenzar con un carácter en mayúscula (funciones globales), o con s: (funciones locales de script), o deben tener el prefijo del nombre asociado al complemento de carga automática donde están definidas (por ejemplo, en {&rtp }/autoload/foo/bar.vim podríamos definir foo#bar#functionname()).
  • Para poder usar los parámetros en la función, use a:parameter_name. Las funciones variadicas se pueden definir con los puntos suspensivos ..., para acceder a los parámetros use a:000 (lista de todos los parámetros), o a:0 (número de parámetros igual a len(a:000) ), a:1 primeros parámetros sin nombre, y así sucesivamente.
  • Las funciones se pueden llamar así: :call MyFunction(param1, param2)
  • Cada línea en una función comienza implícitamente con :, por lo tanto, todos los comandos son comandos de dos puntos
  • Para evitar que la función continúe su ejecución en caso de error, es mejor anotar la firma de la función con abort

Ejemplo #

function! MyFunction(foo, bar, ... ) abort
    return a:foo . a:bar . (a:0 > 0 ? a:1 : '')
endfunction

Funciones de secuencia de comandos #

Si solo planea usar su función en el archivo donde está definida (ya sea porque ha dividido una función más grande en partes más pequeñas, o porque la usará en un comando, un mapeo, …), puede prefijar con s:, evitando ensuciar su espacio de nombres global con funciones internas inútiles:

function! s:my_private_function() " note we don't need to capitalize the first letter this time
    echo "Hi!"
endfunction

Uso de funciones s: de asignaciones #

Si su función local de secuencia de comandos se va a utilizar en una asignación, debe hacer referencia a ella mediante el prefijo especial <SID>:

nnoremap <your-mapping-key> :call <SID>my_private_function()<CR>

Ver :funciones de usuario de ayuda.

Tenga en cuenta, sin embargo, que desde Vim 7, se considera una mejor práctica definir abreviaturas de asignaciones, comandos y menús en complementos (ft) y definir funciones en complementos de carga automática, excepto las funciones que los complementos deben usar cuando están cargados. Esto significa que hoy en día la necesidad de llamar funciones locales de scripts desde mapeos no es tan pertinente como solía ser.

Esquemas de color #

Vim viene con varios esquemas de color preinstalados. En Linux, los esquemas de color que vienen con Vim se almacenan en /usr/share/vim/vim74/colors/ (donde 74 es su número de versión, sin puntos); MacVim los almacena en /Applications/MacVim.app/Contents/Resources/vim/runtime/colors.

Cambio de esquemas de color #

El comando colorscheme cambia el esquema de color actual.

Por ejemplo, para establecer el esquema de color en “robokai”:

:colorscheme robokai

El esquema de color predeterminado se llama creativamente “predeterminado”, por lo tanto, para volver a él, use

:colorscheme default

Para ver todos los esquemas de color instalados actualmente, escriba :colorscheme seguido de space y luego tab o ctrld< /kbd>.

Instalación de esquemas de color #

Los esquemas de color instalados por el usuario se pueden colocar en ~/.vim/colors/. Una vez que se agrega un esquema de color a este directorio, aparecerá como una opción para el comando colorscheme.

Para encontrar nuevos esquemas de color, hay sitios como vimcolors que contienen una variedad de esquemas de color. También hay herramientas como vim.ink y Vivify para ayudarlo a crear sus propios esquemas de color, o puede crear ellos a mano.

Opciones de configuración #

Comúnmente, usaría :set para configurar las opciones a su gusto en su .vimrc. Hay muchas opciones que se pueden cambiar.

Por ejemplo, para usar espacios para la sangría:

:set expandtab
:set shiftwidth=4
:set softtabstop=4

Resaltado de sintaxis #

Active el resaltado de sintaxis cuando el terminal tenga colores

if &t_Co > 2 || has("gui_running")
    syntax on
end

Mostrar espacios en blanco y tabulaciones finales. Mostrar pestañas puede ser especialmente útil cuando se buscan errores en Makefiles.

set list listchars=tab:\|_,trail:.
highlight SpecialKey ctermfg=DarkGray

Opciones #

Hay tres tipos de opciones:

  • opciones booleanas,
  • opciones de cadena,
  • Opciones de número.

Para comprobar el valor de una opción,

  • use :set option? para verificar el valor de una opción,
  • use :verbose set option? para ver también dónde se configuró por última vez.

Configuración de opciones booleanas #

set booloption      " Set booloption.
set nobooloption    " Unset booloption.

set booloption!     " Toggle booloption.

set booloption&     " Reset booloption to its default value.

Configuración de opciones de cadena #

set stroption=baz   " baz

set stroption+=buzz " baz,buzz
set stroption^=fizz " fizz,baz,buzz
set stroption-=baz  " fizz,buzz

set stroption=      " Unset stroption.

set stroption&      " Reset stroption to its default value.

Configuración de opciones de número #

set numoption=1     " 1

set numoption+=2    " 1 + 2 == 3
set numoption-=1    " 3 - 1 == 2
set numoption^=8    " 2 * 8 == 16

Usar una expresión como valor #

  • usando concatenación:

    execute "set stroption=" . my_variable

  • usando :let:

    let &stroption = my_variable

Ver :help :set y :help :let.

Variables #

Como la mayoría de los lenguajes de secuencias de comandos, vimscript tiene variables.

Se puede definir una variable con el comando :let:

let variable = value

y elimínalo con :unlet:

unlet variable

En Vim, las variables se pueden delimitar anteponiendo una sola letra y dos puntos a su nombre. Los autores de complementos usan esa característica para imitar opciones:

let g:plugin_variable = 1

Ver :ayuda con variables internas.

Comandos #

  • No olvide la explosión para permitir que Vim sobrescriba ese comando la próxima vez que vuelva a cargar su vimrc.
  • Los comandos personalizados deben comenzar con un carácter en mayúscula.

Ejemplos #

command! MyCommand call SomeFunction()
command! MyOtherCommand command | Command | command
  • Ver :ayuda de comandos de usuario.

Grupos de autocomando #

  • Los grupos de autocomandos son buenos para la organización, pero también pueden ser útiles para la depuración. Piense en ellos como pequeños espacios de nombres que puede habilitar/deshabilitar a voluntad.

Ejemplo #

augroup MyGroup
    " Clear the autocmds of the current group to prevent them from piling
    " up each time you reload your vimrc.
    autocmd!

    " These autocmds are fired after the filetype of a buffer is defined to
    " 'foo'. Don't forget to use 'setlocal' (for options) and '<buffer>'
    " (for mappings) to prevent your settings to leak in other buffers with
    " a different filetype.
    autocmd FileType foo setlocal bar=baz
    autocmd FileType foo nnoremap <buffer> <key> :command<CR>

    " This autocmd calls 'MyFunction()' everytime Vim tries to create/edit
    " a buffer tied to a file in /'path/to/project/**/'.
    autocmd BufNew,BufEnter /path/to/project/**/* call MyFunction()
augroup END

Ver :autocomando de ayuda.

Condicionales #

if v:version >= 704
    " Do something if Vim is the right version.
endif

if has('patch666')
    " Do something if Vim has the right patch-level.
endif

if has('feature')
    " Do something if Vim is built with 'feature'.
endif

Consulte :help has-patch y :help feature-list.
