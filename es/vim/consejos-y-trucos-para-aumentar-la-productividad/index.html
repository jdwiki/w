Sintaxis #

  • :establecer número relativo
  • :establecer número
  • :establecer ningún número / :establecer nueve
  • :contraseña

Esta recarga automática solo ocurrirá si edita su vimrc en una versión completa de vim que admita autocmd.

Activar números de línea relativos #

Para eliminar algunas líneas de texto cuando no sabe el número exacto de líneas que desea eliminar, intente 10dd, 5dd, 3dd hasta que elimine todas las líneas.

Los números de línea relativos resuelven este problema, supongamos que tenemos un archivo que contiene:

sometimes, you see a block of 
text. You want to remove 
it but you 
cannot directly get the 
exact number of 
lines to delete 
so you try 
10d , 5d 
3d until 
you
remove all the block.

Entrar en modo NORMAL: Esc

Ahora, ejecuta :establecer número relativo. Una vez hecho esto, el archivo se mostrará como:

3 a veces, ves un bloque de 2 texto. quieres eliminar 1 pero tú 0 no puede obtener directamente el 1 número exacto de 2 líneas para eliminar 3 así que intenta 4 10d , 5d 5 3d hasta 6 tú 7 eliminar todo el bloque.

donde 0 es el número de línea para la línea actual y también muestra el número de línea real delante del número relativo, por lo que ahora no tiene que estimar el número de líneas de la línea actual para cortar o eliminar o peor contar ellos uno por uno.

Ahora puede ejecutar su comando habitual como 6dd y está seguro del número de líneas.

También puede usar la forma abreviada del mismo comando :set rnu para activar números relativos y :set nornu para desactivarlo.

Si también :set number o ya tiene activado :set number, obtendrá el número de línea de la línea en la que se encuentra el cursor.

3 a veces, ves un bloque de 2 texto. quieres eliminar 1 pero tú 4 no puede obtener directamente el 1 número exacto de 2 líneas para eliminar 3 así que intenta 4 10d , 5d 5 3d hasta 6 tú 7 eliminar todo el bloque.

Asignaciones para salir del modo Insertar #

Muchos usuarios de Vim encuentran que Esc es demasiado difícil de alcanzar y terminan encontrando otra asignación que es fácil de alcanzar desde la fila de inicio. Tenga en cuenta que Ctrl-[ puede ser equivalente a Esc en un teclado en inglés y es mucho más fácil de alcanzar.

jk #

inoremap jk <ESC>

Este es realmente fácil de activar; simplemente golpea tus dos primeros dedos en la fila de la casa al mismo tiempo. También es difícil activarlo accidentalmente ya que “jk” nunca aparece en ninguna palabra en inglés, y si estás en modo normal no hace nada. Si no escribe demasiado “blackjack”, entonces considere agregar inoremap kj <ESC> para que no tenga que preocuparse por el tiempo de las dos teclas.

Bloq Mayús #

##Linux En Linux, puede usar xmodmap para hacer que Caps Lock haga lo mismo que Esc. Pon esto en un archivo:

!! No clear Lock 
clear lock
!! make caps lock an escape key
keycode 0x42 = Escape

Luego ejecute archivo xmodmap. Esto reasigna Bloqueo de mayúsculas a Esc.

Ventanas #

En Windows, puede usar [SharpKey] o AutoHotkey.

Mac OS #

Si tiene macOS 10.12.1 o posterior, puede reasignar Bloq Mayús a Escape usando Preferencias del sistema. Seleccione Teclado, vaya a la pestaña Teclado y haga clic en Teclas modificadoras.

Uso de la función de finalización de ruta dentro de Vim #

Esto es muy común, memorizas una ruta a un archivo o carpeta, abres Vim e intentas escribir lo que acabas de memorizar, pero no estás 100% seguro de que sea correcto, entonces cierras el editor y comienzas de nuevo.

Cuando desea la función de finalización de ruta y tiene un archivo /home/ubuntu/my_folder/my_file y está editando otro archivo que hace referencia a la ruta del anterior:

Ingrese al modo de inserción: insert o hazlo como quieras. A continuación, escribe /h. Cuando el cursor está debajo de h, presione Ctrlx y luego Ctrlf para que el editor lo complete en /home/ porque el patrón /h es único

Ahora, suponga que tiene dos carpetas dentro de /home/ubuntu/ llamadas my_folder_1 my_folder_2

y quieres la ruta /home/ubuntu/my_folder_2

como siempre:

Entrar en el modo de inserción

escriba /h y presione Ctrlx y luego Ctrlf . Ahora tienes /casa/ A continuación, agregue u después de /home/ y presione Ctrlx y luego Ctrlf . Ahora, tienes /home/ubuntu/ porque esa ruta es única. Ahora, escribe my después de /home/ubuntu/ y presiona Ctrlx y luego Ctrlf . El editor completará tu palabra hasta my_folder_ y verás el árbol de directorios, así que usa las teclas de flecha para elegir el que quieras.

Visualización de números de línea #

Para ver los números de línea desde la vista predeterminada, ingrese

:set number

Para ocultar números de línea

:set nonumber

También hay un atajo para arriba. nu es lo mismo que número.

:set nonu

Para ocultar números de línea, también podemos usar

:set nu!

Escribe un archivo si olvidas sudo antes de iniciar vim #

Este comando guardará el archivo abierto con derechos sudo

:w !sudo tee % >/dev/null

También puede mapear w!! para escribir un archivo como root

:cnoremap w!! w !sudo tee % >/dev/null

Macros “desechables” rápidos #

Agregue esto a su vimrc:

nnoremap Q @q

Para comenzar a grabar la macro “desechable”, use qq. Para terminar de grabar presiona q (en modo normal para ambos).

Para ejecutar la macro grabada, use Q.

Esto es útil para las macros que necesita repetir muchas veces seguidas pero que probablemente no volverá a usar después.

Cómo saber el directorio y/o la ruta del archivo que estás editando #

Para conocer la ruta del directorio en el que se encuentra su archivo, puede usar:

Esc para ingresar al modo comando

:pwd

Esto imprimirá la ruta al directorio en la parte inferior del editor, así

I'm a ninja
~
~
~
~
~
/home/ubuntu/myfolder                                                1,5         All

Ahora, si desea saber el nombre del archivo que está editando en relación con el directorio de trabajo de vim, puede usar:

Esc para ingresar al modo comando CTRLG

I'm a ninja
~
~
~
~
~
"myfile"[Modified][New file] 1 line --100%--                         1,5         All

Finalmente, para obtener la ruta absoluta al archivo que está editando, use

Esc para ingresar al modo comando,

1 y luego CTRLG

I'm a ninja
~
~
~
~
~
"~/myfolder/myfile"[Modified][New file] 1 line --100%--              1,5         All

Recarga automáticamente vimrc al guardar #

Para recargar automáticamente vimrc al guardar, agregue lo siguiente a su vimrc:

if has('autocmd') " ignore this section if your vim does not support autocommands
    augroup reload_vimrc
        autocmd!
        autocmd! BufWritePost $MYVIMRC,$MYGVIMRC nested source %
    augroup END
endif

y luego, por última vez, escriba:

:so $MYVIMRC

La próxima vez que guarde su vimrc, se volverá a cargar automáticamente.

nested es útil si está utilizando vim-airline. El proceso de carga de la línea aérea activa algunos comandos automáticos, pero dado que está en el proceso de ejecutar un comando automático, se omiten. “anidado” permite activar comandos automáticos anidados y permite que la línea aérea se cargue correctamente.

Finalización de la línea de comandos #

configure wildmenu para activar las sugerencias de finalización para la línea de comandos. Ejecute lo siguiente

set wildmenu
set wildmode=list:longest,full

Ahora, si dice :color tab,

Obtendrás

256-jungle Benokai BlackSea C64 CandyPaper Chasing_Logic ChocolateLiquor :color 0x7A69_oscuro

Buscar dentro de un bloque de funciones #

Para buscar el texto foo dentro de un bloque {} que rodea al cursor, utilice el siguiente comando (<ESC> - tecla de escape, <CR> - tecla de entrada):

vi{<ESC>/\%Vfoo<CR>

ahora puedes saltar entre las coincidencias dentro del bloque presionando n y p. Si tiene habilitada la opción hlsearch, se resaltarán todas las coincidencias. \%V es una parte especial del patrón de búsqueda, que le dice a vim que busque solo en el área seleccionada visualmente. También puedes hacer un mapeo como este:

:vnoremap g/ <ESC>/\%V

Después de esto, el comando anterior se reduce a lo siguiente:

vi{g/foo<CR>

Otro truco útil es imprimir todas las líneas que contienen el patrón:

vi{
:'<,'>g/foo/#

El rango '<,'> se inserta automáticamente.

Ver :rango de ayuda y :ayuda:g.

Copiar, mover o eliminar la línea encontrada #

Muchos usuarios se encuentran en una situación en la que solo quieren copiar, mover o eliminar una línea rápidamente y volver a donde estaban.

Por lo general, si desea mover una línea que contiene la palabra “lote” debajo de la línea actual, debe escribir algo como:

/lot<Esc>dd<C-o>p

Pero para aumentar la productividad puedes usar este atajo en estos casos:

" It's recommended to turn on incremental search when doing so
set incsearch

" copy the found line
cnoremap $t <CR>:t''<CR>
" move the found line
cnoremap $m <CR>:m''<CR>
" delete the found line
cnoremap $d <CR>:d<CR>``

Así que una búsqueda como esta:

/lot$m

movería la línea que contiene lot debajo de la línea en la que estaba el cursor cuando comenzó la búsqueda.
