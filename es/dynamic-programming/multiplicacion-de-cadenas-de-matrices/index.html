Solución recursiva #

Multiplicación de cadenas de matrices es un problema de optimización que se puede resolver mediante programación dinámica. Dada una secuencia de matrices, el objetivo es encontrar la forma más eficiente de multiplicar estas matrices. El problema no es realmente realizar las multiplicaciones, sino simplemente decidir la secuencia de las multiplicaciones de matrices involucradas.

Digamos que tenemos dos matrices A1 y A2 de dimensión m * n y p * q . De las reglas de la multiplicación de matrices, sabemos que,

  1. Podemos multiplicar A1 y A2 si y solo si n = p. Eso significa que el número de columnas de A1 debe ser igual al número de filas de A2.
  2. Si se cumple la primera condición y multiplicamos A1 y A2, obtendremos una nueva matriz, llamemos es A3 de dimensión m * q.
  3. Para multiplicar A1 y A2, necesitamos hacer algunas multiplicaciones de escalador. El número total de multiplicaciones del escalador que necesitamos realizar es (m * n * q) o (m * p * q ).
  4. A1 * A2 no es igual a A2 * A 1.

Si tenemos tres matrices A1, A2 y A3 de dimensión ** m** * n, n * p y p * q respectivamente, luego A4 = A1 * A2 * A3 tendrá una dimensión de m * q. Ahora podemos realizar esta multiplicación de matrices de dos maneras:

  • Primero multiplicamos A1 y A2, luego con el resultado multiplicamos *A3 *. Es decir: (A1 * A2) * A3.
  • Primero multiplicamos A2 y A3, luego con el resultado multiplicamos *A1 *. Es decir: A1 * (A2 * A3).

Puede notar que el orden de la multiplicación sigue siendo el mismo, es decir, no multiplicamos (A1 * A3) * ** A2** porque podría no ser válido. Solo cambiamos el paréntesis para multiplicar un conjunto antes de multiplicarlo con el resto. La forma en que colocamos estos paréntesis es importante. ¿Por qué? Digamos, la dimensión de 3 matrices A1, A2 y A3 son 10 * 100, 100 * 5, 5 * 50. Después,

  1. Para (A1 * A2) * A3, el número total de las multiplicaciones del escalador son: (10 * 100 * 5) + (10 * 5 * 50) = 7500 veces.
  2. Para A1 * (A2 * A3), el número total de las multiplicaciones del escalador son: (100 * 5 * 50) + (10 * 100 * 50) = 75000 veces.

¡Para el segundo tipo, el número de multiplicaciones del escalador es 10 veces el número del primer tipo! Entonces, si puede idear una forma de averiguar la orientación correcta de los paréntesis necesarios para minimizar la multiplicación total del escalador, reduciría tanto el tiempo como la memoria necesarios para la multiplicación de matrices. Aquí es donde la multiplicación de cadenas de matrices resulta útil. Aquí, no nos preocuparemos por la multiplicación real de matrices, solo encontraremos el orden correcto de los paréntesis para que se minimice el número de multiplicaciones del escalador. Tendremos las matrices A1, A2, A3 …. …. An y averiguaremos el número mínimo de multiplicaciones del escalador necesarias para multiplicarlas. Asumiremos que las dimensiones dadas son válidas, es decir, satisface nuestro primer requisito para la multiplicación de matrices.

Usaremos el método divide y vencerás para resolver este problema. La programación dinámica es necesaria debido a subproblemas comunes. Por ejemplo: para n = 5, tenemos 5 matrices A1, *A2 *, A3, A4 y A5. Queremos saber el número mínimo de multiplicaciones necesarias para realizar esta multiplicación de matrices A1 * A2 * A 3 * A4 * A5. De las muchas formas, concentrémonos en una: (A1 * A2) * (A3</ sub> * A4 * A5).

Para este, encontraremos Aizquierda = A1 * A2 . Aderecha = A3 * A4 * A5< /sub>. Luego encontraremos Arespuesta = Aizquierda * Aderecha. El número total de multiplicaciones del escalador necesarias para encontrar la Arespuesta: = El número total de multiplicaciones del escalador necesarias para determinar la Aizquierda + El total número de multiplicaciones del escalador necesarias para determinar Aderecha + Número total de multiplicaciones del escalador necesarias para determinar Aizquierda * A correcto.

El último término, El número total de multiplicaciones del escalador necesarias para determinar Aizquierda * Aderecha se puede escribir como: El número de filas en Aizquierda * el número de columnas en Aizquierda * el número de columnas en *Aderecha *. (Según la 2ª regla de la multiplicación de matrices)

Pero también podríamos establecer el paréntesis de otras formas. Por ejemplo:

  • A1 * (A2 * A3 * A 4 * A5)
  • (A1 * A2 * A3) * (A< sub>4 * A5)
  • (A1 * A2 * A3 * A 4) * A5 etc.

Para todos y cada uno de los casos posibles, determinaremos el número de multiplicaciones del escalador necesarias para encontrar Aizquierda y Aderecha, luego para Aizquierda * Aderecha. Si tiene una idea general sobre la recursividad, ya ha entendido cómo realizar esta tarea. Nuestro algoritmo será:

- Set parenthesis in all possible ways.
- Recursively solve for the smaller parts.
- Find out the total number of scaler multiplication by merging left and right.
- Of all possible ways, choose the best one.

¿Por qué este es un problema de programación dinámica? Para determinar (A1 * A2 * A3), si ha ya calculado (A1 * A2), será útil en este caso.

Para determinar el estado de esta recursión, podemos ver que para resolver cada caso, necesitaremos saber el rango de matrices con las que estamos trabajando. Así que necesitaremos un comienzo y un fin. Como usamos divide y vencerás, nuestro caso base tendrá menos de 2 matrices (begin >= end), donde no necesitamos multiplicar en absoluto. Tendremos 2 arreglos: fila y columna. fila[i] y columna[i] almacenarán el número de filas y columnas para la matriz Ai. Tendremos una matriz dp[n][n] para almacenar los valores ya calculados e inicializarla con -1, donde -1 representa el valor que no se ha calculado aún. dp[i][j] representa el número de multiplicaciones del escalador necesarias para multiplicar Ai, *Ai+1 *, …..,Aj inclusive. El pseudocódigo se verá así:

Procedure matrixChain(begin, end):
if begin >= end
    Return 0
else if dp[begin][end] is not equal to -1
    Return dp[begin][end]
end if
answer := infinity
for mid from begin to end
    operation_for_left := matrixChain(begin, mid)
    operation_for_right := matrixChain(mid+1, right)
    operation_for_left_and_right := row[begin] * column[mid] * column[end]
    total := operation_for_left + operation_for_right + operation_for_left_and_right
    answer := min(total, answer)
end for
dp[begin][end] := answer
Return dp[begin][end]

Complejidad:

El valor de begin y end puede variar de 1 a n. Hay n2 estados diferentes. Para cada estado, el bucle interior se ejecutará n veces. Complejidad de tiempo total: O(n³) y complejidad de memoria: O(n²).
