Problema de mochila

  • 0-1 Problema con la mochila

En esta página

  • 0-1 Problema con la mochila

El problema de la mochila o mochila es un problema de optimización combinatoria. Dado un conjunto de artículos, cada uno con un peso y un valor, determine el número de cada artículo que se incluirá en una colección para que el peso total sea menor o igual a un límite dado y el valor total sea lo más grande posible. Deriva su nombre del problema que enfrenta alguien que está limitado por una mochila de tamaño fijo y debe llenarla con los artículos más valiosos.

El problema a menudo surge en la asignación de recursos donde existen restricciones financieras y se estudia en campos como [combinatorics] (https://en.wikipedia.org/wiki/combinatorics), [informática] (https://en.wikipedia ).org/wiki/Computer_science), teoría de la complejidad, criptografía, matemáticas aplicadas y daily fantasy sports.

El problema de la mochila se ha estudiado durante más de un siglo, y los primeros trabajos datan de 1897. El nombre “problema de la mochila” se remonta a los primeros trabajos del matemático [Tobias Dantzig] (https://en.wikipedia.org) /wiki/Tobias_Dantzig) (1884-1956) y se refiere al problema común de empacar sus artículos más valiosos o útiles sin sobrecargar su equipaje.

0-1 Problema con la mochila #

Suponga que se le pregunta, dado el peso total que puede llevar en su mochila y algunos artículos con su peso y valores, ¿cómo puede tomar esos artículos de tal manera que la suma de sus valores sea máxima, pero la suma de sus pesos no? ¿No excederá el peso total que puede llevar? El 0-1 indica que eliges el artículo o no. También tenemos una cantidad de cada artículo. Significa que no puedes dividir el artículo. Si no fue un problema de mochila 0-1, eso significa que si pudieras haber dividido los artículos, hay una solución codiciosa, que se llama problema de mochila fraccionado.

Concentrémonos, por ahora, en nuestro problema actual. Por ejemplo, digamos que tenemos una capacidad de mochila de 7. Tenemos 4 artículos. Sus pesos y valores son:

+----------+---+---+---+---+
|   Item   | 1 | 2 | 3 | 4 |
+----------+---+---+---+---+
|  Weight  | 1 | 3 | 4 | 5 |
+----------+---+---+---+---+
|   Value  | 1 | 4 | 5 | 7 |
+----------+---+---+---+---+

Un método de fuerza bruta sería tomar todas las combinaciones posibles de elementos. Luego podemos calcular sus pesos totales y excluir aquellos que superan la capacidad de nuestra mochila y encontrar el que nos da el valor máximo. Para 4 artículos, necesitaremos verificar (4! - 1) = 23 combinaciones posibles (excluyendo una sin artículos). Esto es bastante engorroso cuando aumenta el número de elementos. Aquí, algunos aspectos que podemos notar, a saber:

  • Podemos tomar elementos menores y calcular el valor máximo que podemos obtener usando esos elementos y combinarlos. Entonces nuestro problema se puede dividir en subproblemas.
  • Si calculamos las combinaciones para el elemento {1,2}, podemos usarlo cuando calculamos {1, 2, 3}.
  • Si minimizamos el peso y maximizamos el valor, podemos encontrar nuestra solución óptima.

Por estas razones, usaremos programación dinámica para resolver nuestro problema. Nuestra estrategia será: cada vez que llegue un artículo nuevo, comprobaremos si podemos elegir el artículo o no y volveremos a elegir los artículos que nos den el máximo valor. Ahora, si elegimos el artículo, nuestro valor será el valor del artículo, más el valor que podamos obtener al restar el valor de nuestra capacidad y el máximo que podamos obtener por ese peso restante. Si no seleccionamos el artículo, seleccionaremos los artículos que nos den el valor máximo sin incluir el artículo. Tratemos de entenderlo con nuestro ejemplo:

Tomaremos una matriz 2D tabla, donde el número de columnas será el valor máximo que podemos obtener al tomar los elementos + 1. Y el número de filas será el mismo que el número de elementos. Nuestra matriz se verá así:

+-------+--------+---+---+---+---+---+---+---+---+
| Value | Weight | 0 | 1 | 2 | 3 | 4 | 5 | 6 | 7 |
+-------+--------+---+---+---+---+---+---+---+---+
|   1   |    1   | 0 |   |   |   |   |   |   |   |
+-------+--------+---+---+---+---+---+---+---+---+
|   4   |    3   | 0 |   |   |   |   |   |   |   |
+-------+--------+---+---+---+---+---+---+---+---+
|   5   |    4   | 0 |   |   |   |   |   |   |   |
+-------+--------+---+---+---+---+---+---+---+---+
|   7   |    5   | 0 |   |   |   |   |   |   |   |
+-------+--------+---+---+---+---+---+---+---+---+

Hemos incorporado el peso y el valor de cada artículo a la matriz para nuestra comodidad. Recuerde que estos no son parte de la matriz, son solo para fines de cálculo, no necesita almacenar estos valores en la matriz tabla.

Nuestra primera columna está llena de 0. Significa que si nuestra capacidad máxima es 0, sin importar el artículo que tengamos, dado que no podemos elegir ningún artículo, nuestro valor máximo será 0. Comencemos desde Tabla[0][1]. Cuando llenamos la Tabla[1][1], nos preguntamos si nuestra capacidad máxima fuera 1 y solo tuviéramos el primer elemento, ¿cuál sería nuestro valor máximo? Lo mejor que podemos hacer es 1, eligiendo el artículo. Para Tabla[0][2], eso significa que si nuestra capacidad máxima es 2 y solo tenemos el primer elemento, el valor máximo que podemos obtener es 1. Esto será lo mismo para Table[0][3], Table[0][4], Table[0][5], Table [0][6] y Tabla[0][7]. Esto se debe a que solo tenemos un artículo, lo que nos da valor 1. Dado que solo tenemos 1 cantidad de cada artículo, no importa cuánto aumentemos la capacidad de nuestra mochila, de un artículo, 1 es el mejor valor que podemos hacer. Entonces nuestra matriz se verá así:

+-------+--------+---+---+---+---+---+---+---+---+
| Value | Weight | 0 | 1 | 2 | 3 | 4 | 5 | 6 | 7 |
+-------+--------+---+---+---+---+---+---+---+---+
|   1   |    1   | 0 | 1 | 1 | 1 | 1 | 1 | 1 | 1 |
+-------+--------+---+---+---+---+---+---+---+---+
|   4   |    3   | 0 |   |   |   |   |   |   |   |
+-------+--------+---+---+---+---+---+---+---+---+
|   5   |    4   | 0 |   |   |   |   |   |   |   |
+-------+--------+---+---+---+---+---+---+---+---+
|   7   |    5   | 0 |   |   |   |   |   |   |   |
+-------+--------+---+---+---+---+---+---+---+---+

Continuando, para la Mesa[1][1], nos preguntamos si teníamos el artículo 1 y 2 y si la capacidad máxima de nuestra mochila era **1 **, ¿cuál es el valor máximo que podemos obtener? Si tomamos tanto el artículo 1 como el 2, el peso total será 4, lo que excederá nuestra capacidad actual de mochila. Por lo tanto, el elemento 2 no se puede seleccionar. Ahora, ¿qué es lo mejor que podemos hacer sin tomar el artículo 2? El valor de la parte superior, que es Tabla[0][1] que contiene el valor máximo que podemos obtener si tuviéramos la capacidad máxima 1 y no elegimos el segundo elemento. Para Tabla[1][2], dado que 2 es menor que peso[2], ese es el peso del segundo elemento, no podemos tomar el elemento . Entonces podemos establecer que, si el peso del artículo actual es mayor que nuestra capacidad máxima, no podemos tomar ese artículo. En este caso, simplemente tomaremos el valor de arriba, que representa el valor máximo que podemos tomar excluyendo el elemento.

if weight[i] > j
    Table[i][j] := Table[i-1][j]
end if

Ahora, para la Tabla[1][3], dado que nuestra capacidad máxima es igual a nuestro peso actual, tenemos dos opciones.

  • Elegimos el artículo y sumamos su valor con el valor máximo que podemos obtener de otros artículos restantes después de tomar este artículo.
  • Podemos excluir este artículo.

Entre las dos opciones, elegiremos aquella de la que podamos obtener el máximo valor. Si seleccionamos el artículo, obtenemos: valor para este artículo + valor máximo del resto de los artículos después de tomar este artículo = 4 + 0 = 4. Obtenemos 4(valor del artículo) de nuestra matriz de peso y 0(valor máximo que podemos obtener del resto de los artículos después de tomar este artículo) viene con **1 ** paso arriba y 3 pasos atrás. Eso es Tabla[0][0]. ¿Por qué? Si tomamos el artículo, nuestra capacidad restante será 3 - 3 = 0 y el artículo restante será el primer artículo. Bueno, si recuerdas, Table[0][0] almacena el valor máximo que podemos obtener si nuestra capacidad fuera 0 y solo tuviéramos el primer elemento. Ahora bien, si no seleccionamos el elemento, el valor máximo que podemos obtener proviene del 1 paso anterior, es decir, 1. Ahora tomamos el máximo de estos dos valores (4, 1) y establecemos Tabla[1][2] = 4. Para la Tabla[1][4], dado que 4, la capacidad máxima de la mochila es mayor que 3, el peso de nuestro artículo actual, nuevamente tenemos dos opciones. Tomamos max(Weight[2] + Table[0][4-Weight[2]], Table[0][4]) = max (Peso[2] + Tabla[0][1], Tabla[0][4]) = max(4 + 1, 1) = 5.

if weight[i] <= j
    w := weight[i]
    Table[i][j] := max(w + Table[i][j-w], Table[i-1][j])
end if

Usando estas dos fórmulas, podemos llenar toda la matriz de Tabla. Nuestra matriz se verá así:

+-------+--------+---+---+---+---+---+---+---+---+
| Value | Weight | 0 | 1 | 2 | 3 | 4 | 5 | 6 | 7 |
+-------+--------+---+---+---+---+---+---+---+---+
|   1   |    1   | 0 | 1 | 1 | 1 | 1 | 1 | 1 | 1 |
+-------+--------+---+---+---+---+---+---+---+---+
|   4   |    3   | 0 | 1 | 1 | 4 | 5 | 5 | 5 | 5 |
+-------+--------+---+---+---+---+---+---+---+---+
|   5   |    4   | 0 | 1 | 1 | 4 | 5 | 6 | 6 | 9 |
+-------+--------+---+---+---+---+---+---+---+---+
|   7   |    5   | 0 | 1 | 1 | 4 | 5 | 7 | 8 | 9 |
+-------+--------+---+---+---+---+---+---+---+---+

Aquí, el último valor que insertamos en nuestra matriz, Tabla[3][7] contiene nuestro valor máximo requerido. Este es el valor máximo que podemos obtener si tuviéramos 4 artículos y nuestra capacidad máxima de la mochila fuera 7.

Aquí una cosa debemos recordar que, incluso para la primera fila, el peso puede ser mayor que la capacidad de la mochila. Tendremos que mantener otra restricción para verificar el valor mientras llenamos la primera fila. O simplemente podemos tomar otra fila y establecer todos los valores de la primera fila en 0. El pseudocódigo se vería así:

Procedure Knapsack(Weight, Value, maxCapacity):
n := Item.size - 1
Table[n+1][maxCapacity+1]
for i from 0 to n
    Table[i][0] := 0
end for
for j from 1 to maxCapacity
    if j >= Weight[0]
        Table[0][j] := Weight[0]
    else
        Table[0][j] := 0
    end if
end for
for i from 1 to n
    for j from 1 to maxCapacity
        if Weight[i] >= j                                            //can't pick the item
            Table[i][j] := Table[i-1][j]        
        else                                                        //can pick the item
            w := Weight[i]
            Table[i][j] := max(w + Table[i-1][j-w], Table[i-1][j])
        end if
    end for
end for
Return Table[n][maxCapacity]

La complejidad temporal de este algoritmo es O(n*maxCapacity), donde n es el número de artículos y maxCapacity es la capacidad máxima de nuestra mochila.

Hasta ahora, hemos encontrado el valor máximo que podemos obtener de nuestro ejemplo. Aún queda una pregunta. ¿Cuáles son los artículos reales? Volveremos a rastrear los valores en nuestra matriz de Tabla para averiguar los elementos que hemos tomado. Seguiremos dos estrategias:

  • Para cualquier artículo, si el valor proviene de la posición anterior, no tomamos el artículo actual. Vamos 1 paso arriba.
  • Si el valor no proviene de la posición anterior, tomamos el artículo. Así que vamos 1 paso arriba y x pasos atrás donde x es el peso del elemento actual.

El pseudocódigo será:

Procedure printItems(Table, maxCapacity, Value):
i := Item.size - 1
j := maxCapacity
while j is not equal to 0
    if Table[i][j] is equal to Table[i-1][j]
        i := i - 1
    else
        Print: i
        j := j - weight[i]
        i := i - 1
    end if
end while
