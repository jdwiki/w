Sintaxis #

  1. Especifique la instancia de express que desea utilizar. Esto es comúnmente aplicación.
  2. Defina el método HTTP para el que se aplica la función. En el ejemplo, esto es get.
  3. Defina la ruta a la que se aplica la función. En el ejemplo, esto es ’/’.
  4. Defina como una función con la palabra clave función.
  5. Agregue los parámetros requeridos: req, res, next. (Ver nota en la sección de comentarios)
  6. Pon algo de código en la función para hacer lo que quieras

Parámetros #

Parámetro Detalles
requerimiento El objeto de la solicitud.
res El objeto de respuesta.
siguiente La llamada al middleware next().

Una función de middleware es una función con acceso al objeto de solicitud (req), el objeto de respuesta (res) y la función de middleware next() en el ciclo de solicitud-respuesta de la aplicación. La función de middleware next() se indica comúnmente mediante una variable llamada next.

Las funciones de middleware están diseñadas para realizar las siguientes tareas:

  • Ejecutar cualquier código.
  • Realizar cambios en los objetos de solicitud y respuesta. (Vea el ejemplo de requestTime)
  • Terminar el ciclo de solicitud-respuesta.
  • Llame al siguiente middleware de la pila. (Llamando al next() middleware)

Nota: No es necesario nombrarlo a continuación. Pero si usa otra cosa, nadie sabrá lo que quiere decir y será despedido. Y tu código no funcionará. Entonces, solo nómbrelo a continuación. Esta regla se aplica al objeto de solicitud y respuesta. Algunas personas usarán solicitud y respuesta en lugar de req y res, respectivamente. Está bien. Desperdicia pulsaciones de teclas, pero está bien.

Programa intermedio CORS #

Este ejemplo demuestra cómo se puede manejar una solicitud http de origen cruzado mediante un middleware.

Fondo CORS

CORS es un método de control de acceso adoptado por todos los principales navegadores para evitar las vulnerabilidades de secuencias de comandos cruzadas inherentes a ellos. En general, la seguridad del navegador, las secuencias de comandos deben mantener que todas las solicitudes XHR deben realizarse solo a la fuente desde la que se sirven las mismas secuencias de comandos. Si se realiza una solicitud XHR fuera del dominio al que pertenecen los scripts, la respuesta será rechazada.

Sin embargo, si el navegador es compatible con CORS, haría una excepción a esta regla si los encabezados apropiados en la respuesta indican que el dominio desde el que se originó la solicitud está permitido. El siguiente encabezado indica que cualquier dominio está permitido:

Access-Control-Allow-Origin: *

Ejemplo

El siguiente ejemplo muestra cómo el middleware Express puede incluir estos encabezados en su respuesta.

app.use(function(request, response, next){

    response.header('Access-Control-Allow-Origin', '*');
    response.header('Access-Control-Allow-Methods', 'GET,PUT,POST,DELETE,OPTIONS');
    response.header('Access-Control-Allow-Headers', 'Content-Type, Authorization, Content-Length, X-Requested-With');

   if (reqest.method === 'OPTIONS') {
      response.status(200).send();        
   }

      next();

});

Manejo de verificación previa

La última parte del ejemplo anterior maneja Preflight. Preflight es una solicitud de OPCIONES especiales que el navegador envía para probar CORS si la solicitud contiene encabezados personalizados.

Programa intermedio de registrador #

Si es nuevo en el middleware en Express, consulte la Descripción general en la sección Comentarios.

Primero, vamos a configurar una aplicación Hello World simple a la que se hará referencia y se agregará durante los ejemplos.

var express = require('express');
var app = express();

app.get('/', function(req, res) {
    res.send('Hello World!');
});

app.listen(3000);

Aquí hay una función de middleware simple que registrará “LOGGED” cuando se llame.

var myLogger = function (req,res,next) {
    console.log('LOGGED');
    next();
};

Llamar a next() invoca la siguiente función de middleware en la aplicación.

Para cargar la función, llame a app.use() y especifique la función que desea llamar. Esto se hace en el siguiente bloque de código que es una extensión del bloque Hello World.

var express = require('express');
var app = express();

var myLogger = function (req, res, next) {
    console.log('LOGGED');
    next();
};

app.use(myLogger);

app.get('/', function(req, res) {
    res.send('Hello World!');
});

app.listen(3000);

Ahora, cada vez que la aplicación recibe una solicitud, imprime el mensaje “REGISTRO” en la terminal. Entonces, ¿cómo agregamos condiciones más específicas cuando se llama al middleware? Mira el siguiente ejemplo y verás.

Middleware de tiempo de solicitud #

Vamos a crear un middleware que agregue una propiedad llamada requestTime al objeto de solicitud.

var requestTime = function (req, res, next) {
    req.requestTime = Date.now();
    next();
};

Ahora modifiquemos la función de registro del ejemplo anterior para utilizar el middleware requestTime.

myLogger = function (req, res, next, requestTime) {
    console.log('LOGGED at ' + requestTime);
    next();
};

Agreguemos el middleware a nuestra aplicación:

var express = require('express');
var app = express();

myLogger = function (req, res, next) {
    console.log('LOGGED at ' + req.requestTime);
    next();
};

var requestTime = function(req, res, next) {
    req.requestTime = Date.now();
    next();
};

app.use(requestTime);

app.use(myLogger);

app.get('/', function(req, res) {
    res.send('Hello World!');
});

app.listen(3000);

Ahora la aplicación registrará la hora en que se realizó la solicitud. Esto cubre los conceptos básicos de escritura y uso del middleware Express. Para obtener más información, consulte Uso de Express Middleware.
