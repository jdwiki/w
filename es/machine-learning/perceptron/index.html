Implementando un modelo Perceptron en C++ #

En este ejemplo, repasaré la implementación del modelo de perceptrón en C++ para que pueda tener una mejor idea de cómo funciona.

Lo primero es lo primero, es una buena práctica escribir un algoritmo simple de lo que queremos hacer.

Algoritmo:

  1. Haga un vector para los pesos e inicialícelo a 0 (no olvide agregar el término de sesgo)
  2. Continúe ajustando los pesos hasta que obtengamos 0 errores o un recuento bajo de errores.
  3. Haz predicciones sobre datos no vistos.

Habiendo escrito un algoritmo súper simple, escribamos ahora algunas de las funciones que necesitaremos.

  • Necesitaremos una función para calcular la entrada de la red (por ejemplo, x * wT multiplicando las entradas por los pesos)
  • Una función de paso para que obtengamos una predicción de 1 o -1
  • Y una función que encuentra los valores ideales para los pesos.

Entonces, sin más preámbulos, entremos de inmediato.

Comencemos de manera simple creando una clase de perceptrón:

class perceptron
{
public:
    
private:

};

Ahora agreguemos las funciones que necesitaremos.

class perceptron
{
public:
    perceptron(float eta,int epochs);
    float netInput(vector<float> X);
    int predict(vector<float> X);
    void fit(vector< vector<float> > X, vector<float> y);
private:

};

Observe cómo la función fit toma como argumento un vector de vector. Esto se debe a que nuestro conjunto de datos de entrenamiento es una matriz de entradas. Esencialmente, podemos imaginar esa matriz como un par de vectores x apilados uno encima del otro y cada columna de esa matriz es una característica.

Finalmente agreguemos los valores que nuestra clase necesita tener. Como el vector w para contener los pesos, el número de épocas que indica el número de pasadas que haremos sobre el conjunto de datos de entrenamiento. Y la constante eta, que es la tasa de aprendizaje por la que multiplicaremos cada actualización de peso para acelerar el procedimiento de entrenamiento aumentando este valor o si eta es demasiado alto, podemos bájelo para obtener el resultado ideal (para la mayoría de las aplicaciones del perceptrón, sugeriría un valor eta de 0.1).

class perceptron
{
public:
    perceptron(float eta,int epochs);
    float netInput(vector<float> X);
    int predict(vector<float> X);
    void fit(vector< vector<float> > X, vector<float> y);
private:
    float m_eta;
    int m_epochs;
    vector < float > m_w;
};

Ahora con nuestro conjunto de clases. Es hora de escribir cada una de las funciones.

Empezaremos desde el constructor ( perceptron(float eta,int epochs); )

perceptron::perceptron(float eta, int epochs)
{
    m_epochs = epochs; // We set the private variable m_epochs to the user selected value
    m_eta = eta; // We do the same thing for eta
}

Como puede ver, lo que haremos es algo muy simple. Así que pasemos a otra función simple. La función de predicción ( int predict(vector X); ). Recuerde que lo que hace la función all predict es tomar la entrada neta y devolver un valor de 1 si netInput es mayor que 0 y -1 de lo contrario.

int perceptron::predict(vector<float> X)
{
    return netInput(X) > 0 ? 1 : -1; //Step Function
}

Tenga en cuenta que usamos una instrucción if en línea para hacernos la vida más fácil. Así es como funciona la instrucción if en línea:

condición ? si_verdadero: si no

Hasta aquí todo bien. Pasemos a implementar la función netInput( float netInput(vector X); )

netInput hace lo siguiente; multiplica el vector de entrada por la transposición del vector de pesos

x * wT

En otras palabras, multiplica cada elemento del vector de entrada x por el elemento correspondiente del vector de pesos w y luego toma su suma y agrega el sesgo.

(x1 * w1 + x2 * w2 + … + xn * wn) + sesgo

sesgo = 1 * w0

float perceptron::netInput(vector<float> X)
{
    // Sum(Vector of weights * Input vector) + bias
    float probabilities = m_w[0]; // In this example I am adding the perceptron first
    for (int i = 0; i < X.size(); i++)
    {
        probabilities += X[i] * m_w[i + 1]; // Notice that for the weights I am counting
        // from the 2nd element since w0 is the bias and I already added it first.
    }
    return probabilities;
}

Muy bien, ya hemos terminado. Lo último que debemos hacer es escribir la función fit que modifica los pesos.

void perceptron::fit(vector< vector<float> > X, vector<float> y)
{
    for (int i = 0; i < X[0].size() + 1; i++) // X[0].size() + 1 -> I am using +1 to add the bias term
    {
        m_w.push_back(0); // Setting each weight to 0 and making the size of the vector
        // The same as the number of features (X[0].size()) + 1 for the bias term
    }
    for (int i = 0; i < m_epochs; i++) // Iterating through each epoch
    {
        for (int j = 0; j < X.size(); j++) // Iterating though each vector in our training Matrix
        {
            float update = m_eta * (y[j] - predict(X[j])); //we calculate the change for the weights
            for (int w = 1; w < m_w.size(); w++){ m_w[w] += update * X[j][w - 1]; } // we update each weight by the update * the training sample
            m_w[0] = update; // We update the Bias term and setting it equal to the update
        }
    }
}

Así que eso fue esencialmente todo. ¡Con solo 3 funciones, ahora tenemos una clase de perceptrón en funcionamiento que podemos usar para hacer predicciones!

El archivo de clase .cpp con las funciones:

perceptron::perceptron(float eta, int epochs)
{
    m_epochs = epochs;
    m_eta = eta;
}

void perceptron::fit(vector< vector<float> > X, vector<float> y)
{
    for (int i = 0; i < X[0].size() + 1; i++) // X[0].size() + 1 -> I am using +1 to add the bias term
    {
        m_w.push_back(0);
    }
    for (int i = 0; i < m_epochs; i++)
    {
        int errors = 0;
        for (int j = 0; j < X.size(); j++)
        {
            float update = m_eta * (y[j] - predict(X[j]));
            for (int w = 1; w < m_w.size(); w++){ m_w[w] += update * X[j][w - 1]; }
            m_w[0] = update;
            errors += update != 0 ? 1 : 0;
        }
        m_errors.push_back(errors);
    }
}

float perceptron::netInput(vector<float> X)
{
    // Sum(Vector of weights * Input vector) + bias
    float probabilities = m_w[0];
    for (int i = 0; i < X.size(); i++)
    {
        probabilities += X[i] * m_w[i + 1];
    }
    return probabilities;
}

int perceptron::predict(vector<float> X)
{
    return netInput(X) > 0 ? 1 : -1; //Step Function
}

void perceptron::printErrors()
{
    printVector(m_errors);
}

void perceptron::exportWeights(string filename)
{
    ofstream outFile;
    outFile.open(filename);

    for (int i = 0; i < m_w.size(); i++)
    {
        outFile << m_w[i] << endl;
    }

    outFile.close();
}

void perceptron::importWeights(string filename)
{
    ifstream inFile;
    inFile.open(filename);

    for (int i = 0; i < m_w.size(); i++)
    {
        inFile >> m_w[i];
    }
}

void perceptron::printWeights()
{
    cout << "weights: ";
    for (int i = 0; i < m_w.size(); i++)
    {
        cout << m_w[i] << " ";
    }
    cout << endl;
}

¿Qué es exactamente un perceptrón? #

En esencia, un modelo de perceptrón es uno de los algoritmos de aprendizaje supervisado más simples para clasificación binaria. Es un tipo de clasificador lineal, es decir, un algoritmo de clasificación que hace sus predicciones basándose en una función predictora lineal que combina un conjunto de pesos con el vector de características. Una forma más intuitiva de pensar es como una red neuronal con una sola neurona.

La forma en que funciona es muy simple. Obtiene un vector de valores de entrada x de los cuales cada elemento es una característica de nuestro conjunto de datos.

Un ejemplo: #

Digamos que queremos clasificar si un objeto es una bicicleta o un automóvil. Por el bien de este ejemplo, digamos que queremos seleccionar 2 funciones. La altura y el ancho del objeto. En ese caso x = [x1, x2] donde x1 es la altura y x2 es el ancho.

Luego, una vez que tengamos nuestro vector de entrada x, queremos multiplicar cada elemento en ese vector con un peso. Por lo general, cuanto mayor es el valor del peso, más importante es la característica. Si, por ejemplo, usamos color como función x3 y hay una bicicleta roja y un automóvil rojo, el perceptrón le asignará un peso muy bajo para que el color no afecte el predicción final.

Muy bien, hemos multiplicado los 2 vectores x y w y obtuvimos un vector. Ahora necesitamos sumar los elementos de este vector. Una forma inteligente de hacer esto es en lugar de simplemente multiplicar x por w, podemos multiplicar x por wT donde **T ** significa transposición. Podemos imaginar la transposición de un vector como una versión rotada del vector. Para obtener más información, puede leer [la página de Wikipedia] 5. Esencialmente, al tomar la transposición del vector w obtenemos un vector Nx1 en lugar de un 1xN. Por lo tanto, si ahora multiplicamos nuestro vector de entrada con tamaño 1xN con este vector de peso Nx1, obtendremos un vector 1x1 (o simplemente un valor único) que será igual a x1 * w1 + x2 * w2 + … + xn * wn. Habiendo hecho eso, ahora tenemos nuestra predicción. Pero hay una última cosa. Esta predicción probablemente no será un simple 1 o -1 para poder clasificar una nueva muestra. Entonces, lo que podemos hacer es simplemente decir lo siguiente: si nuestra predicción es mayor que 0, decimos que la muestra pertenece a la clase 1; de lo contrario, si la predicción es menor que cero, decimos que la muestra pertenece a la clase -1. Esto se llama función escalonada.

Pero, ¿cómo obtenemos los pesos correctos para que podamos hacer predicciones correctas? En otras palabras, ¿cómo entrenamos nuestro modelo de perceptrón?

Bueno, en el caso del perceptrón, no necesitamos ecuaciones matemáticas sofisticadas para entrenar nuestro modelo. Nuestros pesos se pueden ajustar mediante la siguiente ecuación:

Δw = eta * (y - predicción) * x(i)

donde x(i) es nuestra característica (x1 por ejemplo para peso 1, x2 para w2 y así sucesivamente…).

También noté que hay una variable llamada eta que es la tasa de aprendizaje. Puede imaginar la tasa de aprendizaje como cuán grande queremos que sea el cambio de los pesos. Una buena tasa de aprendizaje da como resultado un algoritmo de aprendizaje rápido. Un valor demasiado alto de eta puede dar como resultado una cantidad cada vez mayor de errores en cada época y hace que el modelo haga predicciones realmente malas y nunca converja. Una tasa de aprendizaje demasiado baja puede tener como resultado que el modelo tarde demasiado en converger. (Por lo general, un buen valor para establecer eta para el modelo de perceptrón es 0,1, pero puede diferir de un caso a otro).

Finalmente, algunos de ustedes habrán notado que la primera entrada es una constante (1) y se multiplica por w0. Entonces, ¿qué es eso exactamente? Para obtener una buena predicción, necesitamos agregar un sesgo. Y eso es exactamente lo que es esa constante.

Para modificar el peso del término de sesgo usamos la misma ecuación que usamos para los otros pesos pero en este caso no lo multiplicamos por la entrada (porque la entrada es un 1 constante y no tenemos que hacerlo):

Δw = eta * (y - predicción)

¡Así es básicamente cómo funciona un modelo de perceptrón simple! Una vez que entrenamos nuestras pesas podemos darle nuevos datos y tener nuestras predicciones.

NOTA: #

¡El modelo Perceptron tiene una desventaja importante! Nunca convergerá (es decir, encontraré los pesos perfectos) si los datos no son separables linealmente, lo que significa poder separar las 2 clases en un espacio de características mediante una línea recta. Entonces, para evitar eso, es una buena práctica agregar un número fijo de iteraciones para que el modelo no se quede atascado ajustando pesos que nunca estarán perfectamente ajustados.
