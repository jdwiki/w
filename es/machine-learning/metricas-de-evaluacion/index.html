Área bajo la curva de la característica operativa del receptor (AUROC) #

El AUROC es una de las métricas más utilizadas para evaluar el desempeño de un clasificador. En esta sección se explica cómo calcularlo.

AUC (Área bajo la curva) se usa la mayor parte del tiempo para significar AUROC, lo cual es una mala práctica ya que AUC es ambiguo (podría ser cualquier curva) mientras que AUROC no lo es.

Descripción general: abreviaturas #

Abreviatura Significado
AUROC Área bajo la curva de la característica operativa del receptor
ABC Área Bajo la Curza
República de China Característica de funcionamiento del receptor
TP Verdaderos positivos
TN Verdaderos negativos
FP Falsos Positivos
FN Falsos negativos
TPR Tasa de verdaderos positivos
FPR Tasa de falsos positivos

Interpretando el AUROC #

El AUROC tiene [varias interpretaciones equivalentes] (http://metaoptimize.com/qa/questions/988/simple-explanation-of-area-under-the-roc-curve):

  • La expectativa de que un positivo aleatorio extraído uniformemente se clasifique antes que un negativo aleatorio extraído uniformemente.
  • La proporción esperada de positivos clasificados antes de un negativo aleatorio extraído uniformemente.
  • La tasa positiva verdadera esperada si la clasificación se divide justo antes de un negativo aleatorio uniformemente dibujado.
  • La proporción esperada de negativos clasificados después de un positivo aleatorio extraído uniformemente.
  • La tasa esperada de falsos positivos si la clasificación se divide justo después de un resultado positivo aleatorio uniforme.

Cálculo del AUROC #

Supongamos que tenemos un clasificador binario probabilístico como la regresión logística.

Antes de presentar la curva ROC (= Curva Característica Operativa del Receptor), se debe entender el concepto de matriz de confusión. Cuando hacemos una predicción binaria, puede haber 4 tipos de resultados:

  • Predecimos 0 mientras que la clase es en realidad 0: esto se denomina Verdadero negativo, es decir, predecimos correctamente que la clase es negativa (0). Por ejemplo, un antivirus no detectó un archivo inofensivo como virus.
  • Predecimos 0 mientras que la clase es en realidad 1: esto se denomina Falso Negativo, es decir, predecimos incorrectamente que la clase es negativa (0). Por ejemplo, un antivirus no pudo detectar un virus.
  • Predecimos 1 mientras que la clase es en realidad 0: esto se denomina Falso positivo, es decir, predecimos incorrectamente que la clase es positiva (1). Por ejemplo, un antivirus considera un archivo inofensivo como un virus.
  • Predecimos 1 mientras que la clase es en realidad 1: esto se denomina Verdadero positivo, es decir, predecimos correctamente que la clase es positiva (1). Por ejemplo, un antivirus detectó correctamente un virus.

Para obtener la matriz de confusión, repasamos todas las predicciones hechas por el modelo y contamos cuántas veces ocurre cada uno de esos 4 tipos de resultados:

En este ejemplo de matriz de confusión, entre los 50 puntos de datos que se clasifican, 45 se clasifican correctamente y los 5 se clasifican incorrectamente.

Dado que para comparar dos modelos diferentes, a menudo es más conveniente tener una sola métrica en lugar de varias, calculamos dos métricas de la matriz de confusión, que luego combinaremos en una:

  • Tasa de verdaderos positivos (TPR), también conocido como. sensibilidad, índice de aciertos y recordar, que se define como . Intuitivamente, esta métrica corresponde a la proporción de puntos de datos positivos que se consideran correctamente como positivos, con respecto a todos los puntos de datos positivos. En otras palabras, cuanto mayor sea el TPR, menos puntos de datos positivos perderemos.
  • Tasa de falsos positivos (FPR), también conocido como. fall-out, que se define como . Intuitivamente, esta métrica corresponde a la proporción de puntos de datos negativos que se consideran erróneamente como positivos, con respecto a todos los puntos de datos negativos. En otras palabras, cuanto mayor sea el FPR, más puntos de datos negativos se clasificarán erróneamente.

Para combinar el FPR y el TPR en una sola métrica, primero calculamos las dos métricas anteriores con muchos umbrales diferentes (por ejemplo, ) para la regresión logística, luego las trazamos en un solo gráfico, con los valores de FPR en las abscisas y los valores TPR en las ordenadas. La curva resultante se llama curva ROC, y la métrica que consideramos es el AUC de esta curva, a la que llamamos AUROC.

Matriz de confusión #

Se puede usar una matriz de confusión para evaluar un clasificador, en función de un conjunto de datos de prueba para los cuales se conocen los valores verdaderos. Es una herramienta simple que ayuda a dar una buena visión general del rendimiento del algoritmo que se está utilizando.

Una matriz de confusión se representa como una tabla. En este ejemplo, veremos una matriz de confusión para un clasificador binario.

En el lado izquierdo, se puede ver la clase real (etiquetada como SÍ o NO), mientras que la parte superior indica la clase que se predice y genera (nuevamente SÍ o NO).

Esto significa que 50 instancias de prueba, que en realidad son instancias NO, fueron correctamente etiquetadas por el clasificador como NO. Estos se denominan Negativos Verdaderos (TN). Por el contrario, 100 instancias reales de YES fueron correctamente etiquetadas por el clasificador como instancias de YES. Estos se denominan Verdaderos positivos (TP).

5 instancias SÍ reales, fueron mal etiquetadas por el clasificador. Estos se denominan Falsos Negativos (FN). Además, 10 instancias NO, fueron consideradas instancias SÍ por el clasificador, por lo que son Falsos Positivos (FP).

Con base en estos FP, TP, FN y TN, podemos sacar más conclusiones.

  • Tasa de verdaderos positivos:

    • Tries to answer: When an instance is actually YES, how often does the classifier predict YES?

  • Se puede calcular de la siguiente manera: TP/# instancias SÍ reales = 100/105 = 0,95

  • Tasa de falsos positivos:

    • Tries to answer: When an instance is actually NO, how often does the classifier predict YES?

  • Se puede calcular de la siguiente manera: FP/# instancias reales NO = 10/60 = 0,17

Curvas ROC #

Una curva de características operativas del receptor (ROC) traza la tasa de TP frente a la La tasa de FP como umbral sobre la confianza de que una instancia es positiva es variado

Algoritmo para crear una curva ROC

  1. ordenar las predicciones del conjunto de pruebas de acuerdo con la confianza de que cada la instancia es positiva

  2. recorrer la lista ordenada de confianza alta a baja

    i. locate a threshold between instances with opposite classes (keeping instances with the same confidence value on the same side of threshold)

    ii. compute TPR, FPR for instances above threshold

    iii. output (FPR, TPR) coordinate
