Un bucle es una secuencia de instrucciones que se repite continuamente hasta que se alcanza una determinada condición. Ser capaz de hacer que su programa ejecute repetidamente un bloque de código es una de las tareas más básicas pero útiles en la programación. Un ciclo le permite escribir una declaración muy simple para producir un resultado significativamente mayor simplemente por repetición. Si se ha alcanzado la condición, la siguiente instrucción “falla” a la siguiente instrucción secuencial o se bifurca fuera del ciclo.

Sintaxis #

  • for ( <Inicialización>; <Condición>; <Repetición> ) { <Script_Block> }

  • <Colección> | Foreach-Object { <Script_Block_with_$__as_current_item> }

  • foreach (<Item> in <Collection> ) { <Script_Block> }

  • while ( <Condición> ){ <Script_Block> }

  • do { <Script_Block> } while ( <Condición> )

  • hacer { <Script_Block> } hasta ( <Condición> )

  • <Colección>.foreach( { <Script_Block_with_$__as_current_item> } )

Para cada #

Hay varias formas de ejecutar un bucle foreach en PowerShell y todas tienen sus propias ventajas y desventajas:

Solución Ventajas Desventajas
declaración Foreach Lo más rápido. Funciona mejor con colecciones estáticas (almacenadas en una variable). Sin entrada o salida de canalización
Método ForEach() Misma sintaxis de scriptblock que Foreach-Object, pero más rápida. Funciona mejor con colecciones estáticas (almacenadas en una variable). Soporta salida de tubería. No hay soporte para entrada de tubería. Requiere PowerShell 4.0 o superior
Objeto Foreach (cmdlet) Admite entrada y salida de canalización. Admite bloques de secuencias de comandos de inicio y finalización para la inicialización y el cierre de conexiones, etc. La solución más flexible. El más lento

Actuación #

$foreach = Measure-Command { foreach ($i in (1..1000000)) { $i * $i } }
$foreachmethod = Measure-Command { (1..1000000).ForEach{ $_ * $_ } }
$foreachobject = Measure-Command { (1..1000000) | ForEach-Object { $_ * $_ } }

"Foreach: $($foreach.TotalSeconds)"
"Foreach method: $($foreachmethod.TotalSeconds)"
"ForEach-Object: $($foreachobject.TotalSeconds)"

Example output:

Foreach: 1.9039875
Foreach method: 4.7559563
ForEach-Object: 10.7543821

Si bien Foreach-Object es el más lento, su soporte de canalización puede ser útil ya que le permite procesar elementos a medida que llegan (mientras lee un archivo, recibe datos, etc.). Esto puede ser muy útil cuando se trabaja con grandes cantidades de datos y poca memoria, ya que no es necesario cargar todos los datos en la memoria antes de procesarlos.

Para cada #

ForEach tiene dos significados diferentes en PowerShell. Una es una palabra clave y la otra es un alias para [ForEach-Object] (https://www.wikiod.com/es/powershell/bucles#Para cada objeto) cmdlet. El primero se describe aquí.

Este ejemplo demuestra la impresión de todos los elementos de una matriz en el host de la consola:

$Names = @('Amy', 'Bob', 'Celine', 'David')

ForEach ($Name in $Names)
{
    Write-Host "Hi, my name is $Name!"
}

Este ejemplo demuestra cómo capturar la salida de un bucle ForEach:

$Numbers = ForEach ($Number in 1..20) {
    $Number # Alternatively, Write-Output $Number
}

Al igual que el último ejemplo, este ejemplo, en cambio, demuestra la creación de una matriz antes de almacenar el bucle:

$Numbers = @()
ForEach ($Number in 1..20)
{
    $Numbers += $Number
}

Para #

for($i = 0; $i -le 5; $i++){
    "$i"
}

Un uso típico del ciclo for es operar en un subconjunto de los valores en una matriz. En la mayoría de los casos, si desea iterar todos los valores en una matriz, considere usar una instrucción foreach.

Método ForEach() #

En lugar del cmdlet ForEach-Object, aquí también existe la posibilidad de usar un método ForEach directamente en matrices de objetos como este

(1..10).ForEach({$_ * $_})

o, si lo desea, se pueden omitir los paréntesis alrededor del bloque de script

(1..10).ForEach{$_ * $_}

Ambos darán como resultado la salida a continuación.

1
4
9
16
25
36
49
64
81
100

para cada objeto #

El cmdlet ForEach-Object funciona de manera similar a la declaración [foreach](https://www.wikiod.com/es/powershell/bucles#Para cada), pero toma su entrada de la canalización .

Uso básico #

$object | ForEach-Object {
    code_block
}

Ejemplo:

$names = @("Any","Bob","Celine","David")
$names | ForEach-Object {
    "Hi, my name is $_!"
}

Foreach-Object tiene dos alias predeterminados, foreach y % (sintaxis abreviada). El más común es % porque foreach se puede confundir con la [declaración foreach] (https://www.wikiod.com/es/powershell/bucles#Para cada). Ejemplos:

$names | % {  
    "Hi, my name is $_!"
} 

$names | foreach {  
    "Hi, my name is $_!"
}

Uso avanzado #

Foreach-Object se destaca de las soluciones alternativas de foreach porque es un cmdlet, lo que significa que está diseñado para usar la canalización. Debido a esto, admite tres bloques de secuencias de comandos como un cmdlet o una función avanzada:

  • Begin: se ejecuta una vez antes de recorrer los elementos que llegan de la canalización. Usualmente se usa para crear funciones para usar en el bucle, creando variables, abriendo conexiones (base de datos, web +), etc.
  • Proceso: Ejecutado una vez por artículo llegado del pipeline. Bloque de código foreach “normal”. Este es el valor predeterminado que se usa en los ejemplos anteriores cuando no se especifica el parámetro.
  • Fin: Ejecutado una vez después de procesar todos los elementos. Suele utilizarse para cerrar conexiones, generar un informe, etc.

Ejemplo:

"Any","Bob","Celine","David" | ForEach-Object -Begin {
    $results = @()
} -Process {
    #Create and store message
    $results += "Hi, my name is $_!"
} -End {
    #Count messages and output
    Write-Host "Total messages: $($results.Count)"
    $results
}

Continuar #

El operador Continue funciona en los bucles For, ForEach, While y Do. Omite la iteración actual del ciclo, saltando a la parte superior del ciclo más interno.

$i =0
while ($i -lt 20) {
    $i++ 
    if ($i -eq 7) { continue }
    Write-Host $I
}

Lo anterior generará del 1 al 20 en la consola, pero perderá el número 7.

Nota: al usar un bucle de canalización, debe usar return en lugar de Continue.

Romper #

El operador break saldrá de un bucle de programa inmediatamente. Se puede utilizar en bucles For, ForEach, While y Do o en una instrucción Switch.

$i = 0
while ($i -lt 15) {
    $i++ 
    if ($i -eq 7) {break}
    Write-Host $i
}

Lo anterior contará hasta 15 pero se detendrá tan pronto como se llegue a 7.

Nota: cuando se usa un ciclo de canalización, “romper” se comportará como “continuar”. Para simular una “ruptura” en el ciclo de canalización, debe incorporar alguna lógica adicional, un cmdlet, etc. Es más fácil quedarse con bucles que no sean de canalización si necesita usar “ruptura”.

Romper etiquetas

Break también puede llamar a una etiqueta que se colocó delante de la instanciación de un bucle:

$i = 0
:mainLoop While ($i -lt 15) {
    Write-Host $i -ForegroundColor 'Cyan'
    $j = 0
    While ($j -lt 15) {
        Write-Host $j -ForegroundColor 'Magenta'
        $k = $i*$j
        Write-Host $k -ForegroundColor 'Green'
        if ($k -gt 100) {
            break mainLoop
        }
        $j++
    }
    $i++
}

Nota: Este código incrementará $i a 8 y $j a 13, lo que hará que $k sea igual a 104. Dado que $k supera 100, el código saldrá de ambos bucles.

Tiempo #

Un bucle while evaluará una condición y, si es verdadero, realizará una acción. Mientras la condición se evalúe como verdadera, la acción continuará realizándose.

while(condition){
  code_block
}

El siguiente ejemplo crea un bucle que contará hacia atrás de 10 a 0

$i = 10
while($i -ge 0){
    $i
    $i--
}

A diferencia del bucle Do, la condición se evalúa antes de la primera ejecución de la acción. La acción no se realizará si la condición inicial se evalúa como falsa.

Nota: Al evaluar la condición, PowerShell tratará la existencia de un objeto de devolución como verdadera. Esto se puede usar de varias maneras, pero a continuación se muestra un ejemplo para monitorear un proceso. Este ejemplo generará un proceso de bloc de notas y luego suspenderá el shell actual mientras ese proceso se esté ejecutando. Cuando cierre manualmente la instancia del bloc de notas, la condición while fallará y el ciclo se interrumpirá.

Start-Process notepad.exe
while(Get-Process notepad -ErrorAction SilentlyContinue){
  Start-Sleep -Milliseconds 500
}

Hacer #

Los bucles son útiles cuando siempre desea ejecutar un bloque de código al menos una vez. Un bucle Do evaluará la condición después de ejecutar el bloque de código, a diferencia de un bucle while que lo hace antes de ejecutar el bloque de código.

Puede usar do-loops de dos maneras:

  • Bucle mientras la condición es verdadera:

    Do {
    code_block
} while (condition)

  • Repetir hasta que la condición sea verdadera, en otras palabras, repetir mientras la condición sea falsa:

    Do {
    code_block
} until (condition)

Ejemplos reales:

$i = 0

Do {
    $i++
    "Number $i"
} while ($i -ne 3)

Do {
    $i++
    "Number $i"
} until ($i -eq 3)

Do-While y Do-Until son bucles antónimos. Si el código dentro del mismo, la condición se invertirá. El ejemplo anterior ilustra este comportamiento.
