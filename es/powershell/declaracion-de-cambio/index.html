Una declaración de cambio permite probar la igualdad de una variable con una lista de valores. Cada valor se denomina caso, y la variable que se activa se comprueba para cada caso de cambio. Le permite escribir un script que puede elegir entre una serie de opciones, pero sin requerir que escriba una larga serie de declaraciones if.

Este tema documenta la sentencia de cambio utilizada para bifurcar el flujo del script. No lo confunda con parámetros de cambio que se usan en funciones como banderas booleanas.

Instrucción Switch con parámetro CaseSensitive #

El parámetro -CaseSensitive aplica declaraciones de cambio para realizar coincidencias exactas que distinguen entre mayúsculas y minúsculas de las condiciones.

Ejemplo:

switch -CaseSensitive ('Condition')
{ 
  'condition'   {'First Action'}
  'Condition'   {'Second Action'} 
  'conditioN'   {'Third Action'}  
}

Producción:

Second Action

La segunda acción es la única acción ejecutada porque es la única condición que coincide exactamente con la cadena 'Condición' cuando se tiene en cuenta la distinción entre mayúsculas y minúsculas.

Declaración de cambio con parámetro comodín #

El parámetro -Wildcard permite que las sentencias de cambio realicen coincidencias de comodines con las condiciones.

Ejemplo:

switch -Wildcard ('Condition')
{ 
    'Condition'           {'Normal match'}
    'Condit*'             {'Zero or more wildcard chars.'} 
    'C[aoc]ndit[f-l]on'   {'Range and set of chars.'}  
    'C?ndition'           {'Single char. wildcard'}
    'Test*'               {'No match'} 
}

Producción:

Normal match
Zero or more wildcard chars.
Range and set of chars.
Single char. wildcard

Declaración de cambio con parámetro de archivo #

El parámetro -file permite que la declaración de cambio reciba entrada de un archivo. Cada línea del archivo es evaluada por la sentencia switch.

Archivo de ejemplo input.txt:

condition
test

Ejemplo de declaración de cambio:

switch -file input.txt
{ 
  'condition' {'First Action'}
  'test'      {'Second Action'} 
  'fail'      {'Third Action'}   
}

Producción:

First Action
Second Action

Cambio simple con condición predeterminada #

La palabra clave Predeterminado se usa para ejecutar una acción cuando ninguna otra condición coincide con el valor de entrada.

Ejemplo:

switch('Condition')
{
  'Skip Condition'
  {
    'First Action'
  }
  'Skip This Condition Too'
  {
    'Second Action'
  }
  Default
  {
    'Default Action'
  }
}

Producción:

Default Action

Cambio sencillo #

Las declaraciones de cambio comparan un solo valor de prueba con varias condiciones y realizan cualquier acción asociada para lograr comparaciones exitosas. Puede resultar en múltiples coincidencias/acciones.

Dado el siguiente cambio…

switch($myValue)
{
    'First Condition'    { 'First Action' }
    'Second Condition'   { 'Second Action' }
}

'Primera acción' se generará si $myValue se establece como 'Primera condición'.

Se generará 'Acción de sección' si $myValue se establece como 'Segunda condición'.

No se generará nada si $myValue no coincide con ninguna de las condiciones.

Declaración de cambio con parámetro Regex #

El parámetro -Regex permite que las declaraciones de cambio realicen coincidencias de expresiones regulares con las condiciones.

Ejemplo:

switch -Regex ('Condition')
{ 
  'Con\D+ion'    {'One or more non-digits'}
  'Conditio*$'   {'Zero or more "o"'} 
  'C.ndition'    {'Any single char.'}  
  '^C\w+ition$'  {'Anchors and one or more word chars.'} 
  'Test'         {'No match'} 
}

Producción:

One or more non-digits
Any single char.
Anchors and one or more word chars.

Cambio simple con descanso #

La palabra clave break se puede usar en declaraciones de cambio para salir de la declaración antes de evaluar todas las condiciones.

Ejemplo:

switch('Condition')
{
  'Condition'
  {
    'First Action'
  }
  'Condition'
  {
    'Second Action'
    break
  }
  'Condition'
  {
    'Third Action'
  }
}

Producción:

First Action
Second Action

Debido a la palabra clave romper en la segunda acción, la tercera condición no se evalúa.

Declaración de cambio con parámetro exacto #

El parámetro -Exact hace cumplir las declaraciones de cambio para realizar coincidencias exactas, sin distinción de mayúsculas y minúsculas, contra condiciones de cadena.

Ejemplo:

switch -Exact ('Condition')
{ 
  'condition'   {'First Action'}
  'Condition'   {'Second Action'} 
  'conditioN'   {'Third Action'}  
  '^*ondition$' {'Fourth Action'} 
  'Conditio*'   {'Fifth Action'} 
}

Producción:

First Action
Second Action
Third Action

Las acciones primera a tercera se ejecutan porque sus condiciones asociadas coincidieron con la entrada. Las cadenas regex y wildcard en las condiciones cuarta y quinta no coinciden.

Tenga en cuenta que la cuarta condición también coincidiría con la cadena de entrada si se estuviera realizando una coincidencia de expresiones regulares, pero en este caso se ignoró porque no es así.

Cambiar instrucción con expresiones #

Las condiciones también pueden ser expresiones:

$myInput = 0

switch($myInput) {
    # because the result of the expression, 4, 
    # does not equal our input this block should not be run.
    (2+2)  { 'True. 2 +2 = 4' }

    # because the result of the expression, 0, 
    # does equal our input this block should be run.
    (2-2) { 'True. 2-2 = 0' }

    # because our input is greater than -1 and is less than 1 
    # the expression evaluates to true and the block should be run.
    { $_ -gt -1 -and $_ -lt 1 } { 'True. Value is 0' }
}

#Output
True. 2-2 = 0
True. Value is 0
