El comando de ajuste puede ajustar una función definida por el usuario a un conjunto de puntos de datos (x,y) o (x,y,z), usando una implementación de los mínimos cuadrados no lineales (NLLS) Algoritmo de Marquardt-Levenberg.

Cualquier variable definida por el usuario que aparezca en el cuerpo de la función puede servir como parámetro de ajuste, pero el tipo de retorno de la función debe ser real.

Sintaxis #

  • ajustar [xrange][yrange] función “archivo de datos” usando modificador vía archivo_parámetro

Parámetros #

Parámetros Detalle
Ajuste de parámetros a, b, c y cualquier letra que no haya sido utilizada previamente Use letras para representar parámetros que se usarán para ajustar una función. Ej.: f(x) = a * exp(b * x) + c, g(x,y) = a*x**2 + b*y**2 + c*x*y
Parámetros del archivo start.par En lugar de usar parámetros no inicializados (el Marquardt-Levenberg se inicializará automáticamente para usted a=b=c=...=1), puede ponerlos en un archivo start.par y llame con en la sección parameter_file. Por ejemplo: fit f(x) 'data.dat' u 1:2 via 'start.par'. A continuación se muestra un ejemplo del archivo start.par

Breve introducción #

fit se usa para encontrar un conjunto de parámetros que ‘mejor’ ajusten sus datos a su función definida por el usuario. El ajuste se juzga sobre la base de la suma de las diferencias al cuadrado o “residuales” (SSR) entre los puntos de datos de entrada y los valores de la función, evaluados en los mismos lugares. Esta cantidad a menudo se llama ‘chi cuadrado’ (es decir, la letra griega chi, a la potencia de 2). El algoritmo intenta minimizar SSR, o más precisamente, WSSR, ya que los residuos son “ponderados” por los errores de datos de entrada (o 1,0) antes de elevarse al cuadrado. (Ibídem)

El archivo fit.log

Después de cada paso de iteración, se proporciona información detallada sobre el estado del ajuste tanto en la pantalla como en el llamado archivo de registro fit.log. Este archivo nunca se borrará, sino que siempre se agregará para que no se pierda el historial del ajuste.

Ajuste de datos con errores #

Puede haber hasta 12 variables independientes, siempre hay 1 variable dependiente y se puede ajustar cualquier número de parámetros. Opcionalmente, se pueden ingresar estimaciones de error para ponderar los puntos de datos. (T. Williams, C. Kelley - gnuplot 5.0, un programa de trazado interactivo)

Si tiene un conjunto de datos y desea ajustar si el comando es muy simple y natural:

fit f(x) "data_set.dat" using 1:2 via par1, par2, par3

donde en cambio f(x) podría ser también f(x, y). En el caso de que también tenga estimaciones de errores de datos, simplemente agregue {y | xy | z}errors ({ | } representan las opciones posibles) en la opción modificador (ver Sintaxis). Por ejemplo

fit f(x) "data_set.dat" using 1:2:3 yerrors via par1, par2, par3

donde {y | xy | La opción z}errors requiere respectivamente 1 (y), 2 (xy), 1 (z) columna que especifique el valor de la estimación del error.

Ajuste exponencial con xyerrors de un archivo

Las estimaciones de error de datos se utilizan para calcular el peso relativo de cada punto de datos al determinar la suma ponderada de los residuos cuadrados, WSSR o chi cuadrado. Pueden afectar las estimaciones de los parámetros, ya que determinan cuánta influencia tiene la desviación de cada punto de datos de la función ajustada en los valores finales. Parte de la información de salida de ajuste, incluidas las estimaciones de errores de parámetros, es más significativa si se han proporcionado estimaciones de errores de datos precisas. (Ibidem)

Tomaremos un conjunto de datos de muestra measured.dat, compuesto por 4 columnas: las coordenadas del eje x (Temperature (K)), las coordenadas del eje y (Pressure (kPa) ), las estimaciones del error x (T_err (K)) y las estimaciones del error y (P_err (kPa)).

Ahora, solo componga el prototipo de la función que desde la teoría debe aproximar nuestros datos. En este caso:

Z = 0.001
f(x) = W * exp(x * Z)

donde hemos inicializado el parámetro Z porque, de lo contrario, evaluar la función exponencial exp(x * Z) da como resultado valores enormes, lo que conduce a (coma flotante) Infinity y NaN en el algoritmo de ajuste de Marquardt-Levenberg, por lo general no lo haría necesita inicializar las variables: eche un vistazo [aquí] (https://en.wikipedia.org/wiki/Levenberg–Marquardt_algorithm), si desea saber más sobre Marquardt-Levenberg.

¡Es hora de ajustar los datos!

fit f(x) "measured.dat" u 1:2:3:4 xyerrors via W, Z

Gráfico con ajuste de measured.dat El uso del comando with xyerrorbars mostrará estimaciones de errores en x y en y. set grid colocará una cuadrícula discontinua en los tics principales.

En el caso de que las estimaciones de error no estén disponibles o no sean importantes, también es posible ajustar los datos sin `{y | xy | Opción de ajuste de z}errors:

fit f(x) "measured.dat" u 1:2 via W, Z

En este caso también había que evitar las xyerrorbars.

Ejemplo de archivo “start.par” #

Si carga sus parámetros de ajuste desde un archivo, debe declarar en él todos los parámetros que va a utilizar y, cuando sea necesario, inicializarlos.

## Start parameters for the fit of data.dat

m  = -0.0005           
q  = -0.0005           
d  = 1.02              
Tc = 45.0
g_d = 1.0
b  = 0.01002

Ajuste: interpolación lineal básica de un conjunto de datos #

El uso básico de ajuste se explica mejor con un ejemplo simple:

f(x) = a + bx + cx**2 ajusta [-234:320][0:200] f(x) ‘measured.dat’ usando 1:2 salta 4 a través de a,b,c parcela ‘medido.dat’ u 1:2, f(x) Se pueden especificar rangos para filtrar los datos utilizados en la adaptación. Los puntos de datos fuera de rango se ignoran. (T. Williams, C. Kelley - gnuplot 5.0, un programa de trazado interactivo)

La interpolación lineal (ajuste con una línea) es la forma más sencilla de ajustar un conjunto de datos. Suponga que tiene un archivo de datos donde el crecimiento de su cantidad y es lineal, puede usar

[…] polinomios lineales para construir nuevos puntos de datos dentro del rango de un conjunto discreto de puntos de datos conocidos. (de Wikipedia, Interpolación lineal)

Ejemplo con un polinomio de primer grado #

Vamos a trabajar con el siguiente conjunto de datos, llamado house_price.dat, que incluye los metros cuadrados de una casa en una determinada ciudad y su precio en $1000.

Ajustemos esos parámetros con gnuplot El comando en sí es muy simple, como puede notar en la sintaxis, simplemente defina su prototipo adecuado y luego use el comando fit para obtener el resultado:

## m, q will be our fitting parameters
f(x) = m * x + q       
fit f(x) 'data_set.dat' using 1:2 via m, q

Pero podría ser interesante también utilizar los parámetros obtenidos en el propio gráfico. El siguiente código se ajustará al archivo house_price.dat y luego trazará los parámetros m y q para obtener la mejor aproximación de la curva del conjunto de datos. Una vez que tenga los parámetros, puede calcular el valor de y, en este caso el precio de la casa, a partir de cualquier valor de x (metros cuadrados de la casa) simplemente sustituyendo en la fórmula

y = m * x + q

el ‘valor x’ apropiado. Comentemos el código.

0. Estableciendo el término

set term pos col
set out 'house_price_fit.ps'

1. Administración ordinaria para embellecer gráfica

set title 'Linear Regression Example Scatterplot'
set ylabel 'House price (k$ = $1000)'
set xlabel 'Square meters (m^2)'
set style line 1 ps 1.5 pt 7 lc 'red'
set style line 2 lw 1.5 lc 'blue'

set grid
set key bottom center box height 1.4

set xrange [0:450]
set yrange [0:]

2. El ajuste adecuado

Para ello, solo necesitaremos teclear los comandos:

f(x) = m * x + q
fit f(x) 'house_price.dat' via m, q

3. Guardar valores m y q en una cadena y trazar

Aquí usamos la función sprintf para preparar la etiqueta (encuadrada en el rectángulo del objeto) en la que vamos a imprimir el resultado del ajuste. Finalmente trazamos el gráfico completo.

mq_value = sprintf("Parameters values\nm = %f k$/m^2\nq = %f k$", m, q)
set object 1 rect from 90,725 to 200, 650 fc rgb "white" 
set label 1 at 100,700 mq_value


p 'house_price.dat' ls 1 t 'House price', f(x) ls 2 t 'Linear regression'
set out

La salida se verá así.
