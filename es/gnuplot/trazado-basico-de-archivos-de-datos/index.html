Una de las principales características útiles de gnuplot es la posibilidad de trazar archivos de datos. Graficar un archivo de datos es realmente simple con gnuplot, en realidad, una vez que haya abierto el software desde la terminal, solo necesita digitar el comando plot 'file' para obtener un gráfico automático.

En primer lugar, antes de trazar, debe asegurarse de estar en el mismo directorio donde se encuentra el archivo de datos, de lo contrario, eventualmente recibirá una advertencia.

Trazar un solo archivo de datos #

El comando predeterminado gnuplot plot (también solo p) traza el conjunto de datos con columnas, con la forma del archivo data_set.dat a continuación.

# Prototype of a gnuplot data set
# data_set.dat
# X -   X^2 -    2*X -    Random 
0       0        0        5
1       1        2        15
1.4142  2        2.8284   1
2       4        4        30
3       9        6        26.46
3.1415  9.8696   6.2832   39.11
4       16       8        20
4.5627  20.8182  9.1254   17 
5.0     25.0     10.0     25.50
6       36       12       0.908

Como puede ver, puede escribir su conjunto de datos en notación de coma flotante. Ahora todo está listo para hacer el gráfico de datos: escribiendo solo

plot "data_set.dat"

gnuplot producirá un gráfico en su destino de salida. La configuración predeterminada utilizará las dos primeras columnas de su archivo de datos, respectivamente x e y. Para especificar las columnas que se trazarán, utilice el especificador using

plot "data_set.dat" using 2:4

lo que significa “trazar el archivo usando la columna 2 como X y la columna 4 como Y”. En el caso de que su conjunto de datos sea un archivo tridimensional, simplemente use splot y agregue la columna z

splot "data_set.dat" using 1:2:3

También hay diferentes estilos (consulte la documentación de gnuplot o Seleccionar un estilo de trazado para obtener más información) para trazar puntos. Como se dijo antes, el estilo predeterminado es punto

plot "data_set.dat" using 1:4 with point

que trazará lo mismo que si no escribieras con punto. Un estilo útil para el trazado de datos es linespoint que es, obviamente, “líneas + puntos”. P.EJ.:

plot "data_set.dat" using 1:4 with linespoint
# the abbreviated form is completely equivalent:
# p "data_set.dat" u 1:4 w lp

[![punto de línea][2]][2]

Formas de iteración de trazado

En el caso de que tenga más columnas y quiera trazarlas todas en el mismo gráfico, simplemente pase a la función trazar cualquier argumento que prefiera, separándolos con un ,:

p "data_set.dat" u 1:2 w lp,\
    "data_set.dat" u 1:3 w lp,\
    "data_set.dat" u 1:4 w lp

De todos modos, a veces puede haber demasiadas columnas para escribir una por una. En estos casos, el bucle de iteración for resulta muy útil:

p for [col = 2:4] "data_set.dat" using 1:col w lp

que da la salida

[![Gráfica iterada][3]][3]

Brevemente, la iteración for incrementa la variable en el ciclo, en este caso col, con pasos decididos (si no se especifica = 1). Por ejemplo, for [i = 0:6:2] incrementará i de 0 a 6 en 2 pasos: i = 0, 2, 4, 6. Todos los valores (inicio, parada e incremento) se convierten en valores enteros.

*Cuadrícula

La cuadrícula suele ser útil cuando se traza un conjunto de datos. Para agregar un tipo de cuadrícula

set grid

Trazado de múltiples archivos de datos #

Primer método - Concatenación de cadenas #

El método más simple para trazar múltiples archivos de datos es insertar un bucle for dentro del comando plot de gnuplot Suponiendo que tiene archivos N nombrados secuencialmente, es decir

file_1.dat
file_2.dat
file_3.dat
...
file_N.dat

Ejecutando el comando

plot for[i = 1:N] "file_".i.".dat"

trazará todos los archivos entre file_1.dat y file_N.dat en el mismo gráfico.

Ejemplo con tres archivos de datos

Tabla de conjuntos de datos

Ejes X Hacha-Y file_1.dat Hacha-Y file_2.dat Hacha-Y file_3.dat
1 1 1 1
2 2 4 2
3 3 9 6
4 4 16 24
5 5 25 120

Comandos

set terminal postscript color noenhanced ##setting the term
set output "multiple_files.ps"

set key center ##legend placement

plot [1:5][1:120] \
    for [i = 1:3] "file_".i.".dat" \
    pointsize 1.3 linecolor i+4 \
    title "file\_".i.".dat" \
    with linespoint

El ciclo comienza con for [i = 1:3] "file_".i.".dat" y ejecuta el comando plot hasta llegar a i = 3. El .i. es el número concatenado.

title "file\_".i.".dat" se ha escrito con \ para que el símbolo _ en el nombre de los archivos aparezca como un guion bajo en lugar de un subíndice, y el especificador noenhanced es fundamental para obtener este resultado.

El resultado final se muestra a continuación.

Segundo método: usar la función sprintf #

Otro posible camino a seguir es usar la función sprintf que funciona básicamente lo mismo [que el lenguaje C sprintf] (https://www.wikiod.com/es/c/instrumentos-de-cuerda#lectura/escritura de datos con formato de cadena). La sintaxis correcta, de la [documentación de gnuplot 5.1] (http://www.gnuplot.info/gnuplot_cvs.pdf) es

sprintf("format", x, y, ...)

Un breve ejemplo aclarará todas las dudas.

file_name(n) = sprintf("file_%d.dat", n)
plot for[i = 1:N] file_name(i) title file_name(i)
