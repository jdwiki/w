Esta página debe cubrir varias formas de configurar las rutas de Maya python: userSetup, maya.env, variables de entorno, etc.

Usando userSetup.py #

Agregue rutas arbitrarias al entorno de Maya Python en el archivo userSetup.py. userSetup.py es un archivo de Python (no un módulo) que se ejecuta automáticamente al iniciar Maya. userSetup.py puede residir en varios lugares, según el sistema operativo y las variables de entorno.

Cuando se inicie Maya, ejecutará el contenido del archivo userSetup. Agregar rutas de Python aquí le permitirá encontrar módulos:

 import sys
 sys.path.append("/path/to/my/modules")

Esto hará que los archivos del módulo de Python en ‘/ruta/a/mis/módulos’ estén disponibles para importar usando la directiva estándar de ‘importación’.

Para configuraciones más avanzadas, el módulo site puede hacer lo mismo usando la función addsitedir(). site.addsitedir() admite [archivos .pth] (http://bob.ippoli.to/archives/2005/02/06/using-pth-files-for-python-development/) que configura múltiples rutas en Uno va.

Por ejemplo, tres carpetas de Python no relacionadas podrían organizarse así:

  python_files
  | 
  +---- studio
  |      +  module1.py
  |      +  module2.py
  |     
  +---- external
         |
         +---- paid
         |      + paidmodule.py
         |
         +---- foss
                + freemodule.py

Usando sys.path directamente, tendría que agregar python_files/studio, python_files/external/paid y python_files/external/paid manualmente. Sin embargo, podría agregar un archivo .pth a la raíz de python_files que se veía así:

  studio
  external/paid
  external/foss

y llame a esto en userSetup:

 import site
 site.addsitedir("/path/to/python_files")

y obtendrás todos los caminos de una sola vez.

Uso de variables de entorno #

El intérprete de Maya Python funciona como un intérprete normal de Python, por lo que utilizará las mismas variables de entorno para encontrar archivos importables que cualquier otra instalación de Python 2.6 o 2.7 (descrito con más detalle en la [documentación de Python] (https://docs.python .org/2/using/cmdline.html#using-on-envvars).

Si no hay otra instalación de Python en su máquina, puede usar las variables de entorno para señalar la ubicación de sus archivos de Python para Maya (si tiene otro Python, cambiarlos por el bien de Maya puede interferir con su otra instalación de Python: usted sería mejor usar una configuración de usuario o un script de inicio). Configure la variable PYTHONPATH para que incluya sus rutas de búsqueda. Si está editando la variable para incluir varias rutas, recuerde que en los sistemas *NIX, las rutas están separadas por dos puntos:

  export PYTHONPATH="/usr/me/maya/shared:/usr/me/other_python"

donde en Windows son punto y coma:

  setx  PYTHONPATH C:/users/me/maya;//server/shared/maya_python

Múltiples configuraciones #

Una ventaja de usar variables de entorno es que puede reconfigurar rápidamente una instalación de maya para cargar herramientas y scripts desde diferentes ubicaciones para diferentes proyectos. La forma más fácil de hacer esto es configurar PYTHONPATH justo antes de iniciar Maya para que herede las rutas necesarias para esta sesión de maya. Por ejemplo

  set PYTHONPATH=C:/users/me/maya;//server/shared/maya_python
  maya.exe

lanzará Maya (en Windows) con las rutas C:/users/me/maya y //server/shared/maya_python disponibles para su uso. Podría lanzar una segunda copia de Maya desde una nueva línea de comandos usando un comando set diferente y el segundo Maya usaría rutas diferentes.

Debido a que es difícil para la mayoría de los usuarios finales escribir este tipo de cosas, es una buena idea automatizar el proceso con un archivo por lotes o shell que establezca las variables de entorno locales y ejecute maya. nota: necesitamos ejemplos de esto para archivos .bat y .sh En este sistema, distribuiría un archivo .bat o .sh para cada proyecto que estuviera apoyando y sus usuarios iniciarían maya usándolos; iniciar maya sin el archivo bat los revertiría a la configuración predeterminada de Maya sin scripts personalizados.
