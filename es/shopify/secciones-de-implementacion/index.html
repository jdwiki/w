Las secciones son una función que permite a los propietarios de las tiendas agregar, editar, eliminar y reordenar fácilmente el contenido de una página. Hay 2 tipos de secciones: dinámicas y fijas. Las secciones dinámicas se pueden reordenar en su totalidad con otras secciones en la página (página de inicio). Las secciones fijas no se pueden reordenar, pero el contenido dentro de ellas aún se puede agregar, editar, eliminar y reordenar.

Cosas para recordar: #

  • El esquema de una sección se limita a esa sección en particular. Esto significa que puede dar a sus variables nombres relativamente simples sin preocuparse de que entren en conflicto con nombres de variables de otras secciones en la misma página.

Más recursos: #

  • https://help.shopify.com/themes/development/theme-editor/sections
  • https://help.shopify.com/themes/development/theme-editor/settings-schema

Incluir una sección en una página #

Incluir una sección estática en una página es similar a incluir un fragmento. La única diferencia es que en lugar de usar la palabra “incluir”, usas la palabra “sección”.

{% section 'section-file-name' %}

El uso de esta etiqueta “codificará” la sección en su tema en el lugar donde colocó el código y no se puede mover ni eliminar de la página.

Para usar secciones dinámicas, debe usar el control “preestablecido” en la parte inferior de la configuración del esquema para esa sección.

{% schema %}
  {
    "presets": [
      {
        "category": "Custom Content",
        "name": "Text",
        "settings": {
          "heading": "Hello World"
        },
        "blocks": [
          {
            "type": "text",
            "settings": {
              "content": "Once upon a time..."
            }
          }
        ]
      }
    ]
  }
{% endschema %}

Al usar ajustes preestablecidos, esto le permitirá mover y agregar su sección alrededor de su página según lo requiera. También le permite completar la sección con contenido ficticio para que pueda ver exactamente cómo se verá la sección en su página sin tener que agregarla a la página.

En esquema de archivo #

Para las secciones, en lugar de utilizar un gran archivo JSON (como settings_schema.json), mantienen su esquema dentro del archivo de la sección que lo está usando. Se utilizan etiquetas y se coloca JSON entre ellas. A partir de ahí, el formato es similar asettings_schema.json`.

{% schema %}
  {
    "name": "Header Banner",
    "settings": [
      {
        "type": "checkbox",
        "id": "banner_enable",
        "label": "Enable Banner",
        "default": false
      },
      {
        "type": "color",
        "id": "banner_color",
        "label": "Banner Background Color",
        "default": "#000000"
      },
      {
        "type": "color",
        "id": "text_color",
        "label": "Banner Text Color",
        "default": "#ffffff"
      },
      {
        "type": "text",
        "id": "banner_text",
        "label": "Banner Text",
        "default": "Welcome to my Section"
      }
    ]
  }
{% endschema %}

Para usar esto en su código HTML/Liquid, lo extraería de section.settings. Algo como:

{% if section.settings.banner_enable %}
  <div class="banner">
    <p>{{ section.settings.banner_text }}</p>
  </div>
{% endif %}

Uso de bloques para elementos recurrentes #

Una de las cosas más útiles que vienen con las secciones son los bloques. Los bloques son básicamente un modelo de algo que se puede crear una cantidad ilimitada de veces. Uno de los mejores ejemplos son las diapositivas de un control deslizante. Un bloque es un elemento de nivel superior en el esquema, lo que significa que se encuentra junto a elementos como “nombre” y “configuración”, pero no dentro de ellos. Dentro de los bloques hay básicamente mini esquemas.

{% schema %}
  {
    "name": "Slider",
    "max_blocks": 6,
    "blocks": [
      {
        "type": "slide",
        "name": "Slide",
        "limit": 4,
        "settings": [
          {
            "id": "image",
            "type": "image_picker",
            "label": "Image"
          }
        ]
      }
    ]
  }
{% endschema %}

Observe que Slide también tiene un tipo de slide. Puedes darle a tus bloques su propio tipo que no sea uno de los tipos proporcionados por Shopify.

Para usar esto, simplemente recorrería cada bloque en su código HTML/Liquid.

<div class="slider">
  {% for block in section.blocks %}
    <img src="{{ block.settings.image | img_url: 'master' }}" alt="" />
  {% endfor %}
</div>

Uso de JavaScript y hojas de estilo en las secciones #

Cada sección puede contener JavaScript y hojas de estilo, estos dos lenguajes se usan dentro de etiquetas líquidas: {% javascript %}{% endjavascript %} & {% stylesheet %}{% endstylesheet %}.

Al colocar el código dentro de estas dos etiquetas, Shopify compila cada pieza en shopify_compiled.js & shopify_compiled.css. Esto permite una mayor legibilidad dentro del código de cada sección, ya que no tiene que buscar en un documento largo para encontrar cada pieza.

Si no quiere usar CSS, puede usar SCSS alternativamente usando {% stylesheet 'scss' %} en su lugar.

Sin embargo, hay desventajas en el uso de estos. Liquid no se puede usar en ninguno de ellos, al igual que no se puede usar en archivos normales *.js, *.css, *.scss sin la extensión .liquid.

Además, al usar la etiqueta de la hoja de estilo SCSS, esta no es una hoja de estilo global, por lo que las variables definidas dentro del documento .scss de su tema no estarán accesibles y deberán redefinirse.

Ejemplos de uso

JavaScript

{% javascript %}
    $( "p" ).click(function() {
      $( this ).slideUp();
    });
{% endjavascript %}

CSS

{% stylesheet %}
    .container {
        width: 100%;
    }
    .container p {
        color: #ff0000;
    }
{% endstylesheet %}

HABLAR CON DESCARO A

{% stylesheet 'scss' %}
    $red: #ff0000;
    .container {
        width: 100%;
        p {
            color: $red;
        }
    }
{% endstylesheet %}
