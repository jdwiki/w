La API de Windows es amplia y contiene muchas características. El tamaño de la API es tal que nadie puede saberlo todo. Si bien hay muchos recursos como StackOverflow, no hay sustituto para la documentación oficial.

Ejemplos de Documentación:

  • Descripción general del tema: Consideraciones sobre el rendimiento y mejores prácticas del Administrador de ventanas de escritorio
  • Ejemplos: Personalizar una miniatura icónica y un mapa de bits de vista previa en vivo
  • Funciones: función DwmSetIconicThumbnail

Tipos de Documentación Disponibles #

La biblioteca de MSDN contiene varios tipos diferentes de documentación que se pueden usar para implementar funciones.

  • Resúmenes de temas Estos son amplios resúmenes de temas destinados a proporcionar una comprensión general de una API. Estos resúmenes también describen a menudo las mejores prácticas y las estrategias de implementación.
  • Ejemplos Demostrar el uso de API particulares. Por lo general, están muy simplificados, no necesariamente verifican errores y, por lo general, no usan marcos como MFC o ATL. Proporcionan un punto de partida para el uso de características.
  • Referencia Detalla todos los elementos de cada API. Esto incluye constantes/enumeraciones, interfaces, funciones y clases.

Nota: Muchos empleados de Microsoft también mantienen blogs, como [The Old New Thing] de Raymond Chen (https://blogs.msdn.microsoft.com/oldnewthing/) que pueden complementar la documentación, pero estos blogs no sustituyen a la documentación. .

Búsqueda de documentación para una característica #

Encontrar documentación para una característica suele ser tan simple como buscar usando un buen motor de búsqueda. Si eso falla, o si no está seguro acerca de los términos específicos, el [Índice API de Windows] (https://msdn.microsoft.com/en-us/library/windows/desktop/ff818516.aspx) puede ayudar a localizar características específicas. La documentación de métodos, interfaces, enumeraciones y constantes generalmente se puede encontrar buscando el nombre usando un motor de búsqueda. Además, el Centro de desarrollo de Windows puede proporcionar un valioso punto de partida.

Uso de la documentación de funciones #

La documentación de una función se divide en varias secciones:

Visión general #

Describe para qué se utiliza la función. Esta sección también mostrará información sobre si la función está depreciada o puede no estar disponible en futuras versiones.

Sintaxis #

Muestra la declaración de la función desde el encabezado de origen correspondiente. Es una referencia rápida a la firma de la función.

Parámetros #

Explica cada uno de los parámetros, si el parámetro es de entrada o de salida, y otras consideraciones importantes.

Valor devuelto #

Esta sección explica el resultado de la llamada de función, incluido cómo detectar errores y qué información adicional está disponible. (Por ejemplo, esta sección indicará explícitamente si GetLastError proporcionará información adicional sobre el manejo de errores).

Observaciones #

Cubre cualquier información adicional requerida para usar la función, como información sobre funciones de soporte, obtención de identificadores apropiados y eliminación de recursos.

Ejemplos #

Si esta sección está disponible, tiene un ejemplo del uso apropiado de la función para usar como punto de partida para la implementación.

Requisitos #

Proporciona información importante sobre los requisitos previos para llamar a la función. Esta información incluye:

  • Cliente/Servidor Mínimo Soportado Primera versión del sistema operativo (soportado por Microsoft) para proporcionar la función. (Tenga en cuenta que este campo es notoriamente engañoso. A menudo, las funciones son compatibles con una versión anterior del sistema operativo, pero este campo solo muestra la versión más antigua actualmente compatible con Microsoft. Por ejemplo, CreateWindow La función ha sido compatible desde Windows 1.0, pero la documentación solo muestra que ha sido compatible desde Windows 2000. La versión en línea de la documentación de MSDN no indica que ninguna función fuera compatible con una versión de Windows anterior a 2000, aunque muchos lo eran. Para el desarrollo heredado, deberá consultar una versión anterior de la documentación del SDK, como la que podría haberse enviado en un CD-ROM de MSDN. O simplemente busque en los archivos de encabezado).
  • Header El encabezado del SDK para #include que contiene la declaración de la función. Si la función no está disponible en un encabezado, esto mostrará información sobre el procedimiento para llamar a la función (generalmente llamando a GetProcAddress para hacer enlaces dinámicos en tiempo de ejecución).
  • Biblioteca El archivo de biblioteca para pasar al enlazador para resolver las funciones exportadas.
  • DLL El archivo (como se envía con el sistema operativo) que contiene la función exportada.
  • Fin del soporte de cliente/servidor La última versión de Windows oficialmente compatible con la API.
  • Nombres Unicode y ANSI Para funciones de cadena que tienen variantes tanto Unicode como ANSI, se enumeran los nombres reales exportados para las dos funciones. Este suele ser solo el nombre de la función con un sufijo W o A (respectivamente).
