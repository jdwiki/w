En muchas aplicaciones, los registros de MongoDB deben serializarse en formato JSON. Si sus registros tienen campos de tipo fecha, fecha y hora, objectId, binario, código, etc., encontrará excepciones TypeError: not JSON serializable al usar json.dumps. Este tema muestra cómo superar esto.

Usando json_util #

json_util proporciona dos métodos auxiliares, dumps y loads, que envuelven los métodos json nativos y proporcionan una conversión BSON explícita hacia y desde json.

Uso sencillo #

from bson.json_util import loads, dumps
record = db.movies.find_one()
json_str = dumps(record)
record2 = loads(json_str)

si registro es:

{ 
    "_id" : ObjectId("5692a15524de1e0ce2dfcfa3"), 
    "title" : "Toy Story 4", 
    "released" : ISODate("2010-06-18T04:00:00Z") 
}

entonces json_str se convierte en:

{
    "_id": {"$oid": "5692a15524de1e0ce2dfcfa3"},
    "title" : "Toy Story 4", 
    "released": {"$date": 1276833600000}
}

Opciones JSON #

Es posible personalizar el comportamiento de los volcados a través de un objeto JSONOptions. Ya hay dos conjuntos de opciones disponibles: DEFAULT_JSON_OPTIONS y STRICT_JSON_OPTIONS.

>>> bson.json_util.DEFAULT_JSON_OPTIONS
    JSONOptions(strict_number_long=False, datetime_representation=0,
     strict_uuid=False, document_class=dict, tz_aware=True, 
     uuid_representation=PYTHON_LEGACY, unicode_decode_error_handler='strict',
     tzinfo=<bson.tz_util.FixedOffset object at 0x7fc168a773d0>)

Para utilizar diferentes opciones, puede:

  1. modificar el objeto DEFAULT_JSON_OPTIONS. En este caso, las opciones se utilizarán para todas las llamadas posteriores a dumps:

     from bson.json_util import DEFAULT_JSON_OPTIONS
 DEFAULT_JSON_OPTIONS.datetime_representation = 2
 dumps(record)

  2. Especifique un JSONOptions en una llamada a dumps usando el parámetro json_options:

     # using strict
 dumps(record, json_options=bson.json_util.STRICT_JSON_OPTIONS)

 # using a custom set of options
 from bson.json_util import JSONOptions
 options = JSONOptions() # options is a copy of DEFAULT_JSON_OPTIONS
 options.datetime_representation=2
 dumps(record, json_options=options)

Los parámetros de JSONOptions son:

  • strict_number_long: si es verdadero, los objetos Int64 se codifican en el tipo de modo estricto de MongoDB Extended JSON NumberLong, es decir, {"$numberLong": "<number>" }. De lo contrario, se codificarán como un int. El valor predeterminado es falso.
  • datetime_representation: la representación que se utilizará al codificar instancias de datetime.datetime. 0 => {"$fecha": <fechaAsMilisegundos>}, 1 => {"$fecha": {"$númeroLargo": "<fechaAsMilisegundos>"}}, 2 => {"$fecha" : "<ISO-8601>"}
  • strict_uuid: si es verdadero, los objetos uuid.UUID están codificados en modo estricto de MongoDB Extended JSON tipo binario. De lo contrario, se codificará como {"$uuid": "<hex>" }. El valor predeterminado es falso.
  • document_class: los documentos BSON devueltos por loads() se decodificarán en una instancia de esta clase. Debe ser una subclase de collections.MutableMapping. Valores predeterminados para dictar.
  • uuid_representation: la representación BSON que se usará al codificar y decodificar instancias de uuid.UUID. El valor predeterminado es PYTHON_LEGACY.
  • tz_aware: si es verdadero, el tipo de modo estricto de MongoDB Extended JSON Fecha se decodificará a instancias conscientes de la zona horaria de datetime.datetime. De lo contrario, serán ingenuos. El valor predeterminado es Verdadero.
  • tzinfo: una subclase datetime.tzinfo que especifica la zona horaria desde la cual se deben decodificar los objetos de fecha y hora. Predeterminado a utc.

Uso del módulo json con controladores personalizados #

Si todo lo que necesita es serializar los resultados de mongo en json, es posible usar el módulo json, siempre que defina controladores personalizados para manejar tipos de campos no serializables. Una ventaja es que tiene pleno poder sobre cómo codificar campos específicos, como la representación de fecha y hora.

Aquí hay un controlador que codifica fechas usando la representación iso y la identificación como una cadena hexadecimal:

import pymongo
import json 
import datetime
import bson.objectid

def my_handler(x):
    if isinstance(x, datetime.datetime):
        return x.isoformat()
    elif isinstance(x, bson.objectid.ObjectId):
        return str(x)
    else:
        raise TypeError(x)

db = pymongo.MongoClient().samples
record = db.movies.find_one()
# {u'_id': ObjectId('5692a15524de1e0ce2dfcfa3'), u'title': u'Toy Story 4',
#   u'released': datetime.datetime(2010, 6, 18, 4, 0),}

json_record = json.dumps(record, default=my_handler)
# '{"_id": "5692a15524de1e0ce2dfcfa3", "title": "Toy Story 4", 
#    "released": "2010-06-18T04:00:00"}'

Uso de python-bsonjs #

python-bsonjs no depende de PyMongo y puede ofrecer una buena mejora de rendimiento con respecto a json_util:

bsonjs es aproximadamente 10-15 veces más rápido que json_util de PyMongo para decodificar BSON a JSON y codificar JSON a BSON.

Tenga en cuenta que:

  • para usar bsonjs de manera efectiva, se recomienda trabajar directamente con RawBSONDocument
  • las fechas se codifican utilizando la representación LEGACY, es decir, {"$date": <dateAsMilliseconds>}. Actualmente no hay opciones para cambiar eso.

Instalación #

pip install python-bsonjs

Uso #

Para aprovechar al máximo bsonjs, configure la base de datos para usar la clase RawBSONDocument. Luego, use dumps para convertir bytes sin procesar de bson a json y loads para convertir bytes sin procesar de json a bson:

import pymongo
import bsonjs
from pymongo import MongoClient
from bson.raw_bson import RawBSONDocument

# configure mongo to use the RawBSONDocument representation
db = pymongo.MongoClient(document_class=RawBSONDocument).samples
# convert json to a bson record
json_record = '{"_id": "some id", "title": "Awesome Movie"}' 
raw_bson = bsonjs.loads(json_record)
bson_record = RawBSONDocument(raw_bson)
# insert the record
result = db.movies.insert_one(bson_record)
print(result.acknowledged)

# find some record
bson_record2 = db.movies.find_one()
# convert the record to json
json_record2 = bsonjs.dumps(bson_record2.raw)
print(json_record2)
