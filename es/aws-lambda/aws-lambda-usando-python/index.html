Hola Mundo - Función Lambda #

Esta es una de las funciones lambda más simples. Es equivalente al programa Hola Mundo. Para crear su primer programa, siga los pasos mencionados a continuación.

  1. Inicie sesión en la consola de AWS
  2. Haga clic en Lambda en cálculo
  3. Haga clic en crear una función Lambda
  4. Omitir la sección de selección de planos
  5. En configurar activador, haga clic en el rectángulo punteado
  6. Seleccione la puerta de enlace API
  7. Complete los detalles requeridos como en la imagen. Nombre de API es el nombre de su API que va a construir. Patrón de recursos es la ruta de URL a la que puede invocar su función Lambda. Seleccione el método http requerido. En nuestro ejemplo elegimos GET. En AWS, la puesta en escena se puede realizar de diferentes maneras, como prod, dev … Le ayudará a diferenciar las funciones de prod y dev. Para fines de demostración, elija la seguridad como Abierta (no se recomienda en producción). Haga clic en Siguiente
  8. Configure la función como se indica a continuación Proporcione el nombre de la función, la descripción de su función y el entorno de tiempo de ejecución. Estamos eligiendo python como entorno de ejecución.
  9. Modificar el código. Aquí estamos imprimiendo el evento aws lambda en cloudtrail que es gratuito. También está devolviendo una cadena.
  10. Proporcione el controlador y el rol de la función Lambda Asegúrese de que el nombre del controlador debe comenzar con lambda_function.<Nombre de la función Lambda>. También cree un nuevo rol para ejecutar la función lambda. Seleccione la cantidad de memoria principal requerida para ejecutar su función. Seleccione el tiempo de espera predeterminado y haga clic en siguiente
  11. Haga clic en crear función
  12. Ahora su función está lista para ejecutarse. Haga clic en el enlace proporcionado por aws Cuando haga clic en el enlace, su función lambda se ejecutará en el fondo y obtendrá resultados en el navegador.

¿Por qué AWS Lambda? #

Admite AWS Lambda

  • Escalabilidad y disponibilidad transparentes
  • Operaciones amigables para desarrolladores y sin necesidad de administrar servidores
  • Integración nativa a los servicios de AWS
  • No es necesario pagar por el tiempo de inactividad
  • Integración RESTful
  • Monitoreo de la interfaz RESTful usando la puerta de enlace API de AWS
