Nuevo proyecto con el cliente API de Cloud Resource Manager para .NET

  • Empezando

En esta página

  • Empezando

Usaremos Bibliotecas cliente API de Google para .NET para esta muestra.

Hay otras bibliotecas. Consulte Google Explicación de las bibliotecas cliente para obtener más información.

Usaremos la API de Cloud Resource Manager para [Crear y administrar proyectos](https://cloud.google.com/resource-manager/docs/ creación-gestión-proyectos).

Empecemos.

Poniendolo todo junto…

Deberías tener dos archivos. El primer archivo se llama packages.config o project.json.

Llamemos al segundo archivo Program.cs. Todo el código que se incluyó en las secciones anteriores se puede pegar en un solo método principal:

using System;
using System.Collections.Generic;
using System.Threading.Tasks;

using Google.Apis.Auth.OAuth2;
using Google.Apis.CloudResourceManager.v1;
using Google.Apis.Services;

using Data = Google.Apis.CloudResourceManager.v1.Data;

namespace OurFirstProject
{
    public class Program
    {
        private const string projectId = [YOUR-PROJECT-ID];
        private const string applicationName = "Test";

        public static void Main(string[] args)
        {

            var scopes = new String[] {
                CloudResourceManagerService.Scope.CloudPlatform
            };

            GoogleCredential credential = Task.Run(
                () => GoogleCredential.GetApplicationDefaultAsync()
            ).Result;

            if (credential.IsCreateScopedRequired)
            {
                credential = credential.CreateScoped(scopes);
            }

            CloudResourceManagerService service = new CloudResourceManagerService(
                new BaseClientService.Initializer()
                {
                    HttpClientInitializer = credential,
                    ApplicationName = applicationName
                }
            );

            Console.WriteLine("1. Create Project");
            Data.Operation operation1 = service.Projects.Create(
                new Data.Project()
                {
                    ProjectId = projectId,
                }
            ).Execute();

            Console.Write("2. Awaiting Operation Completion");
            Data.Operation operation2;
            do
            {
                operation2 = service.Operations.Get(operation1.Name).Execute();
                Console.WriteLine(operation2.Done.ToString());
                System.Threading.Thread.Sleep(1000);
            } while (operation2.Done != true);

            Console.WriteLine();
            Console.WriteLine("Enter to continue");
            Console.ReadLine();

            Console.WriteLine("3. Deleting Project");
            var operation3 = service.Projects.Delete(projectId).Execute();
        }
    }
}

Empezando #

El código funcionará en Windows y Linux.

Se ejecuta en .NET Core o .NET Standard. La única diferencia es que, si desea ejecutar .NET Core, debe usar project.json. Si desea ejecutar en .NET Standard, debe usar packages.config.

La biblioteca de cliente de API para la API de Cloud Resource Manager está disponible en nuget:

https://www.nuget.org/packages/Google.Apis.CloudResourceManager.v1/

La versión al momento de escribir es: 1.22.0.809

Lo haremos de 2 formas:

  • Windows con .NET Estándar; y
  • Linux con .NET Core.

Ventanas

Si está utilizando Visual Studio, cree una nueva “Aplicación de consola” de “Visual C#”. De lo contrario, cree un directorio para el proyecto y cree un archivo en él llamado packages.config. packages.config es para .NET Standard pero estamos usando .NET Standard con Windows. Solo puede ejecutar .NET Standard en Windows. Reemplace el contenido de packages.config con:

﻿<?xml version="1.0" encoding="utf-8"?>
<packages>
  <package id="Google.Apis" version="1.22.0"
    targetFramework="net452" />
  <package id="Google.Apis.Auth" version="1.22.0"
    targetFramework="net452" />
  <package id="Google.Apis.CloudResourceManager.v1" version="1.22.0.809" 
    targetFramework="net452" />
  <package id="Google.Apis.Core" version="1.22.0"
    targetFramework="net452" />
</packages>

Linux

Cree un directorio para el proyecto y cree un archivo en él llamado project.json. project.json es para .NET Core pero estamos usando .NET Core con Linux en este ejemplo. Puede ejecutar .NET Core en Linux o en Windows. Reemplace el contenido de project.json con:

{
  "version": "1.0.0-*",
  "buildOptions": {
    "debugType": "portable",
    "emitEntryPoint": true
  },
  "dependencies": {},
  "frameworks": {
    "netcoreapp1.1": {
      "dependencies": {
        "Microsoft.NETCore.App": {
          "type": "platform",
          "version": "1.1.0"
        },
        "Google.Apis.CloudResourceManager.v1":"1.22.0.809"
      },
      "imports": "dnxcore50"
    }
  }
}

Credenciales predeterminadas de la aplicación #

No lo repetiré todo aquí: “[Credenciales predeterminadas de la aplicación] (https://developers.google.com/identity/protocols/application-default-credentials) proporcionan una forma sencilla de obtener credenciales de autorización para llamar a las API de Google. "

Si puede usar las Credenciales predeterminadas de la aplicación, hágalo.

Hay un paso adicional que deberá realizar antes de usar las Credenciales predeterminadas de la aplicación como su identidad cuando llame a las API desde su máquina:

gcloud auth application-default login [yourname@gmail.com]

He aquí por qué preferirá usar las Credenciales predeterminadas de la aplicación:

var scopes = new String[] {
    CloudResourceManagerService.Scope.CloudPlatform
};

GoogleCredential credential = Task.Run(
    () => GoogleCredential.GetApplicationDefaultAsync()
).Result;

if (credential.IsCreateScopedRequired)
{
    credential = credential.CreateScoped(scopes);
}

… ¡Ese es todo el código que necesita para autorizar llamadas a (cualquier) API de Google Cloud!

Usaremos el objeto credencial en el próximo paso para hacer llamadas contra el servicio de Google…

Llamar a cualquier método (!) en cualquier servicio de Google (!) #

Una vez que se familiarice con el código para llamar a un método en un servicio de Google, podrá inferir cómo llamar a cualquier método en cualquier servicio de Google.

Primero, hacemos una conexión al servicio usando el objeto credencial instanciado en el ejemplo anterior:

CloudResourceManagerService service = new CloudResourceManagerService(
    new BaseClientService.Initializer()
    {
        HttpClientInitializer = credential,
        ApplicationName = "Our First Google API Client"
    }
);

Entonces podemos llamar a los métodos proporcionados por el servicio. ¿Qué métodos están disponibles?

https://cloud.google.com/resource-manager/docs/apis

¿Cuál es la solicitud REST subyacente para Projects.Create?

https://cloud.google.com/resource-manager/reference/rest/v1/projects/create

OK… Vamos a escribir el código.

El código espera un valor de cadena para projectId. Los ID de proyecto son identificadores únicos. Te recomiendo que uses un sistema para nombrar tus proyectos que te ayude a identificarlos.

Projects.Create espera un objeto Data.Project. Este objeto tiene una propiedad obligatoria, el ID del proyecto, que es todo lo que proporcionaremos, pero también podemos proporcionar un nombre del proyecto, etiquetas, detalles del padre del proyecto, etc.

Data.Operation operation1 = service.Projects.Create(
    new Data.Project()
    {
        ProjectId = projectId,
    }
).Execute();

La creación de proyectos se gestiona de forma asíncrona. Se nos da un objeto Operación que debemos sondear para determinar cuándo se crea el Proyecto. Las operaciones tienen una propiedad Nombre que identifica de forma exclusiva la operación. La siguiente sección de código sondea la plataforma “¿Ya hemos terminado?”. El proyecto se creará cuando nuestra nueva operación incluya una propiedad Done que sea True.

Data.Operation operation2;
do
{
    operation2 = service.Operations.Get(operation1.Name).Execute();
    System.Threading.Thread.Sleep(1000);
} while (operation2.Done != true);

Para completar, y con suerte dentro de muchos años después de un uso feliz de su proyecto, es posible que deba eliminar su proyecto. Simplemente llamamos a Projects.Delete y proporcionamos nuestra ID de proyecto. Esto también devuelve una operación y realmente deberíamos sondear esta operación también hasta que se complete definitivamente. Nuestro proyecto será entonces eliminado.

var operation3 = service.Projects.Delete(projectId).Execute();

¡Eso es todo!
