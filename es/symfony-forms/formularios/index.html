Sintaxis #

  • Formulario createForm(cadena|FormTypeInterface $tipo, mixto $datos = nulo, matriz $opciones = matriz())
  • FormBuilder createFormBuilder(mixed $data = null, array $options = array())

Puede “personalizar” el evento del proceso del componente Form con un evento de formulario compatible con el componente Event Dispatcher.

Documentos de Symfony:

El componente de formulario proporciona un proceso estructurado que le permite personalizar sus formularios mediante el uso del componente EventDispatcher. Usando eventos de formulario, puede modificar información o campos en diferentes pasos del flujo de trabajo: desde el llenado del formulario hasta el envío de los datos de la solicitud.

Crear un tipo de formulario personalizado #

Un tipo de formulario personalizado es una clase que define un componente de formulario reutilizable. Los componentes de formulario personalizados se pueden anidar para crear formularios complicados.

En lugar de crear un formulario en el controlador usando un creador de formularios, puede usar su propio tipo para hacer que el código sea más legible, reutilizable y mantenible.

Cree una clase que represente su tipo de formulario

// src/AppBundle/Form/ExampleType.php
namespace AppBundle\Form;

use Symfony\Component\Form\AbstractType;
use Symfony\Component\Form\FormBuilderInterface;
use Symfony\Component\Form\Extension\Core\Type\TextType;
use Symfony\Component\Form\Extension\Core\Type\NumberType;
use Symfony\Component\Form\Extension\Core\Type\SubmitType;

class ExampleType extends AbstractType
{
    public function buildForm(FormBuilderInterface $builder, array $options) {
        $builder
             ->add('value', TextType::class, array('required' => false))
             ->add('number', NumberType::class)
             ->add('string', TextType::class)
             ->add('save', SubmitType::class)
        ;
    }
}

Ahora puede usar su formulario en el controlador:

use AppBundle\Form\ExampleType;

// ...

$form = $this->createForm(ExampleType::class, $data)

Crear un formulario simple en un controlador #

Un formulario le brinda al usuario una forma de cambiar los datos en su aplicación, de manera estructurada. Para mutar una matriz simple de datos, creamos un formulario usando un generador de formularios:

use Symfony\Component\Form\Extension\Core\Type\TextType;
use Symfony\Component\Form\Extension\Core\Type\NumberType;
use Symfony\Component\Form\Extension\Core\Type\SubmitType;

// ...

function myAction (Request $request) {
    $data = array(
        'value' => null,
        'number' => 10,
        'string' => 'No value',
    );
    $form = $this->createFormBuilder($data)
                 ->add('value', TextType::class, array('required' => false))
                 ->add('number', NumberType::class)
                 ->add('string', TextType::class)
                 ->add('save', SubmitType::class)
                 ->getForm();
    
    $form->handleRequest($request);
    if ($form->isValid()) {
        
        // $data is now changed with the user input
        // Do something with the data
    }
    
    return $this->render(..., array(
        'form' => $form->createView(),
        // ...
    ));
}

En su plantilla, represente su formulario usando la función Twig form(...):

{# Render the form #}
{{ form(form) }}

Las etiquetas, ID, nombres y etiquetas de formulario se generan automáticamente. De forma predeterminada, el formulario se envía a la página actual con una solicitud POST.

Cómo lidiar con las opciones de formulario #

En este ejemplo, creé un formulario que se usa para registrar un nuevo usuario. En las opciones que se pasan al formulario, doy los diferentes roles que puede tener un usuario.

Creando una clase reutilizable para mi formulario con una clase de datos configurada y una opción adicional que completa el campo de elección para elegir un rol de usuario:

class UserType extends AbstractType
{
    public function buildForm(FormBuilderInterface $builder, array $options)
    {
        $builder
            ->add('firstName', TextType::class, array(
                'label' => 'First name'
            ))
            ->add('lastName', TextType::class, array(
                'label' => 'Last name'
            ))
            ->add('email', EmailType::class, array(
                'label' => 'Email'
            ))
            ->add('role', ChoiceType::class, array(
                'label' => 'Userrole',
                'choices' => $options['rolechoices']
            ))
            ->add('plain_password', RepeatedType::class, array(
                'type' => PasswordType::class,
                'first_options' => array('label' => 'Password'),
                'second_options' => array('label' => 'Repeat password')
            ))
            ->add('submit', SubmitType::class, array(
                'label' => 'Register user'
            ));
    }

    public function configureOptions(OptionsResolver $optionsResolver)
    {
        $optionsResolver->setDefaults(array(
            'data_class' => 'WebsiteBundle\Entity\User',
            'rolechoices' => array()
        ));
    }
}

Como puede ver, hay una opción predeterminada agregada al formulario llamada ‘roleChoises’. Esta opción se crea y pasa en el método para crear un objeto de formulario. Ver siguiente código.

Creando un objeto de formulario en mi controlador:

$user = new User();
$roles = array(
    'Admin' => User::ADMIN_ROLE,
    'User' => User::USER_ROLE
);
$form = $this->createForm(UserType::class, $user, array(
    'rolechoices' => $roles
));

Comprobar si todos los campos se representan en la plantilla #

Al renderizar un formulario ‘a mano’, puede ser útil saber si quedan campos para renderizar o no. La función isRendered() de la clase FormView devuelve true si aún quedan campos por representar en la plantilla.

Este fragmento imprime <h3>Campos adicionales</h3> si quedan campos por agregar a la plantilla, seguido de los campos en sí.

{% if not form.isRendered() %}
    <h3>Extra fields</h3>
    {{ form_rest(form) }}
{% endif %}

Tratar con eventos de formulario #

Para poder manejar eventos de formulario, es importante adjuntar la solicitud, es decir, enviar a una acción de controlador después de enviar un formulario, al formulario creado en esa acción.

public function registerAction(Request $request)
{
    $data = new ExampleObject();
    $form = $this->createForm(ExampleObjectType::class, $data);

    $form->handleRequest($request);
    if($form->isSubmitted() && $form->isValid()){
        // do something with form data
        return $this->redirectToRoute('route_name');
    }
    return $this->render('view.html.twig', array(
        'form' => $form->createView()
    ));
}

La variable de solicitud que se pasa a la acción es del tipo Symfony\Component\HttpFoundation\Request
