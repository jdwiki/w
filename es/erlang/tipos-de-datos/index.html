Cada tipo de datos en erlang se llama Término. Es un nombre genérico que significa cualquier tipo de dato.

Números #

En Erlang, los números son enteros o flotantes. Erlang usa precisión arbitraria para números enteros (bignums), por lo que sus valores están limitados solo por el tamaño de la memoria de su sistema.

1> 11.
11
2> -44.
-44
3> 0.1.
0.1
4> 5.1e-3.
0.0051
5> 5.2e2.
520.0

Los números se pueden utilizar en varias bases:

1> 2#101.
5
2> 16#ffff.
65535

La notación de prefijo $ produce el valor entero de cualquier carácter ASCII/Unicode:

3> $a.
97
4> $A.
65
5> $2.
50
6> $🤖.
129302

Binarios y cadenas de bits #

Un binario es una secuencia de bytes de 8 bits sin signo.

1> <<1,2,3,255>>.
<<1,2,3,255>>
2> <<256,257,258>>.
<<0,1,2>>
3> <<"hello","world">>.
<<"helloworld">>

Una cadena de bits es un binario generalizado cuya longitud en bits no es necesariamente un múltiplo de 8.

1> <<1:1, 0:2, 1:1>>.
<<9:4>> % 4 bits bitstring

Mapas #

Un mapa es una matriz o diccionario asociativo compuesto por pares (clave, valor).

1> M0 = #{}.
#{}
2> M1 = #{ "name" => "john", "age" => "28" }.
#{"age" => "28","name" => "john"}
3> M2 = #{ a => {M0, M1} }.
#{a => {#{},#{"age" => "28","name" => "john"}}}

Para actualizar un mapa:

1> M = #{ 1 => x }.
2> M#{ 1 => c }.
#{1 => c}
3> M.
#{1 => x}

Solo actualice alguna clave existente:

1> M = #{ 1 => a, 2 => b}.
2> M#{ 1 := c, 2:= d }.
#{1 => c,2 => d}
3> M#{ 3 := c }.
** exception error: {badkey,3}

La coincidencia de patrones:

1> M = #{ name => "john", age => 28 }.
2> #{ name := Name, age := Age } = M.
3> Name.
"john"
4> Age.
28

Átomos #

Un átomo es un objeto con un nombre que se identifica solo por el nombre mismo.

Los átomos se definen en Erlang usando literales de átomos que son

  • una cadena sin comillas que comienza con una letra minúscula y contiene solo letras, dígitos, guiones bajos o el carácter @, o
  • Una sola cadena citada

Ejemplos #

1> hello.
hello

2> hello_world.
hello_world

3> world_Hello@.
world_Hello@

4> '1234'.     
'1234'

5> '!@#$%% ä'.
'!@#$%% ä'

Átomos que se usan en la mayoría de los programas de Erlang #

Hay algunos átomos que aparecen en casi todos los programas de Erlang, en particular debido a su uso en la Biblioteca estándar.

  • verdadero y falso son los utilizados para denotar los respectivos valores booleanos
  • ok generalmente se usa como valor de retorno de una función que se llama solo por su efecto, o como parte de un valor de retorno, en ambos casos para indicar una ejecución exitosa
  • De la misma manera, error se usa para indicar una condición de error que no garantiza un retorno anticipado de las funciones superiores.
  • ‘indefinido’ generalmente se usa como marcador de posición para un valor no especificado

Usar como etiquetas #

ok y error se utilizan con bastante frecuencia como parte de una tupla, en la que el primer elemento de la tupla señala el éxito, mientras que los demás elementos contienen el valor de retorno real o la condición de error:

func(Input) ->
    case Input of
        magic_value ->
            {ok, got_it};
        _ ->
            {error, wrong_one}
    end.

{ok, _} = func(SomeValue).

Almacenamiento #

Una cosa a tener en cuenta al usar átomos es que se almacenan en su propia tabla global en la memoria y esta tabla no se recolecta como basura, por lo que se desaconseja la creación dinámica de átomos, en particular cuando un usuario puede influir en el nombre del átomo.

Listas #

Una lista en Erlang es una secuencia de cero o más términos de Erlang, implementada como una lista enlazada individualmente. Cada elemento de la lista puede ser cualquier tipo de término (cualquier tipo de datos).

1> [1,2,3].
[1,2,3]
2> [wow,1,{a,b}].     
[wow,1,{a,b}]

El encabezado de la lista es el primer elemento de la lista.

La cola de la lista es el resto de la lista (sin la cabeza). También es una lista. Puede usar hd/1 y tl/1 o comparar con [H|T] para obtener el principio y el final de la lista.

3> hd([1,2,3]).
1
4> tl([1,2,3]).
[2,3]
5> [H|T] = [1,2,3].
[1,2,3]
6> H.
1
7> T.
[2,3]

Anteponiendo un elemento a una lista #

8> [new | [1,2,3]].
[new,1,2,3]

Concatenación de listas #

9> [concat,this] ++ [to,this].
[concat,this,to,this]

Cuerdas #

En Erlang, las cadenas no son un tipo de datos separado: son solo listas de números enteros que representan puntos de código ASCII o Unicode:

> [97,98,99].
"abc"
> [97,98,99] =:= "abc".
true
> hd("ABC").
65

Cuando el shell de Erlang va a imprimir una lista, intenta adivinar si realmente querías que fuera una cadena. Puedes desactivar ese comportamiento llamando a shell:strings(false):

> [8].
"\b"
> shell:strings(false).
true
> [8].
[8]

En el ejemplo anterior, el número entero 8 se interpreta como el carácter de control ASCII para el retroceso, que el shell considera un carácter “válido” en una cadena.

tuplas #

Una tupla es una secuencia ordenada de longitud fija de otros términos de Erlang. Cada elemento de la tupla puede ser cualquier tipo de término (cualquier tipo de datos).

1> {1, 2, 3}.
{1,2,3}
2> {one, two, three}.
{one,two,three}
3> {mix, atom, 123, {<<1,2>>, [list]}}.
{mix,atom,123,{<<1,2>>,[list]}}

Divertida #

Erlang es un lenguaje de programación funcional. Una de las características de un lenguaje de programación de funciones es el manejo de funciones como datos (objetos funcionales).

  • Pasar una función como argumento a otra función.
  • Función de retorno como resultado de una función.
  • Mantener funciones en alguna estructura de datos.

En Erlang, esas funciones se llaman funs. Las funs son funciones anónimas.

1> Fun = fun(X) -> X*X end.
#Fun<erl_eval.6.52032458>
2> Fun(5).
25

Funs también puede tener varias cláusulas.

3> AddOrMult = fun(add,X) -> X+X;
3>                (mul,X) -> X*X 
3> end.
#Fun<erl_eval.12.52032458>
4> AddOrMult(mul,5).
25
5> AddOrMult(add,5).
10

También puede usar funciones de módulo como funs con la sintaxis: fun Module:Function/Arity. Por ejemplo, tomemos la función max del módulo lists, que tiene aridad 1.

6> Max = fun lists:max/1.
#Fun<lists.max.1>
7> Max([1,3,5,9,2]). 
9

Identificadores de Procesos (Pid) #

Cada proceso en erlang tiene un identificador de proceso (Pid) en este formato <x.x.x>, siendo x un número natural. A continuación se muestra un ejemplo de un Pid

<0.1252.0>

Pid se puede usar para enviar mensajes al proceso usando ‘bang’ (!), también Pid se puede vincular a una variable, ambos se muestran a continuación

MyProcessId = self().
MyProcessId ! {"Say Hello"}.

Leer más sobre la creación de procesos y más en general sobre los procesos en erlang

1: https://www.wikiod.com/es/erlang/procesos#Creación de procesos

Sintaxis de bits: valores predeterminados #

Aclaración de Erlang doc sobre la sintaxis de bits:

4.4 Valores predeterminados

[Comienzo omitido: «3.14» ni siquiera es una sintaxis legal.]

El tamaño predeterminado depende del tipo. Para entero es 8. Para float es 64. Para binario es el tamaño real del binario especificado:

1> Bin = « 17/entero, 3,2/float, «97, 98, 99»/binario ». «17,64,9,153,153,153,153,153,154,97,98,99» ^ |<————————–>|<——>| | flotante=64 binario=24 entero=8

2> tamaño (Bin). % Devuelve el número de bytes: 12 % 8 bits + 64 bits + 3*8 bits = 96 bits => 96/8 = 12 bytes

En el emparejamiento, un segmento binario sin Tamaño solo se permite en el final del patrón, y el tamaño predeterminado es el resto del binario en el lado derecho del partido:

25> Bin = «97, 98, 99». «“abc”»

26> « X/entero, Resto/binario » = Bin. «“abc”»

27 > X. 97

28> Descanso. «“bc”»

Todos los demás segmentos con tipo binario en un patrón deben especificar un Tamaño:

12> Bin = «97, 98, 99, 100». «“abcd”»

13> « B:1/binario, X/entero, Resto/binario » = Bin. %‘unidad’ por defecto es 8 para «“abcd”» %tipo binario, el tamaño total del segmento es Tamaño * unidad

14 > B. «“un”»

15 > X. 98

16> Descanso. «“cd”»

17> « B2/binario, X2/entero, Rest2/binario » = Bin.

  • 1: un campo binario sin tamaño solo se permite al final de un patrón binario
