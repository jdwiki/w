Compile e instale Erlang/OTP en FreeBSD #

Los siguientes ejemplos muestran 3 métodos principales para instalar Erlang/OTP en FreeBSD.

Método 1 - Paquete binario preconstruido #

Use pkg para instalar el paquete binario preconstruido:

$ pkg install erlang

Pruebe su nueva instalación:

$ erl
Erlang/OTP 18 [erts-7.3.1] [source] [64-bit] [smp:2:2] [async-threads:10] [hipe] [kernel-poll:false]

Eshell V7.3.1  (abort with ^G)

Método 2: construya e instale usando la colección de puertos (recomendado) #

Cree e instale el puerto como de costumbre:

$ make -C /usr/ports/lang/erlang install clean

Pruebe su nueva instalación:

$ erl
Erlang/OTP 18 [erts-7.3.1] [source] [64-bit] [smp:2:2] [async-threads:10] [hipe] [kernel-poll:false]

Eshell V7.3.1  (abort with ^G)

Esto obtendrá el tarball de lanzamiento del sitio web oficial, aplicará algunos parches si es necesario, compilará el lanzamiento e instalarlo. Obviamente, llevará algún tiempo.

Método 3: compilar e instalar desde el tarball de lanzamiento #

Nota: compilar la versión manualmente funciona, pero se debe preferir usar los dos métodos anteriores, ya que la colección de puertos incorpora parches que hacen que la versión sea más compatible con FreeBSD.

Descargue el archivo de publicación:

$ fetch 'http://erlang.org/download/otp_src_18.3.tar.gz'

Compruebe que su suma MD5 es correcta:

$ fetch 'http://erlang.org/download/MD5'
MD5                                           100% of   24 kB  266 kBps 00m00s

$ grep otp_src_18.3.tar.gz MD5
MD5(otp_src_18.3.tar.gz)= 7e4ff32f97c36fb3dab736f8d481830b

$ md5 otp_src_18.3.tar.gz
MD5 (otp_src_18.3.tar.gz) = 7e4ff32f97c36fb3dab736f8d481830b

Extrae el tarball:

$ tar xzf otp_src_18.3.tar.gz

Configurar:

$ ./configure --disable-hipe

Si desea compilar Erlang con HiPe, deberá aplicar los parches de la colección de puertos.

Construir:

$ gmake

Instalar:

$ gmake install

Pruebe su nueva instalación:

$ erl
Erlang/OTP 18 [erts-7.3] [source] [64-bit] [smp:2:2] [async-threads:10] [kernel-poll:false]

Eshell V7.3  (abort with ^G)

Compile e instale usando kerl #

kerl es una herramienta que lo ayuda a crear e instalar versiones de Erlang/OTP.

Instalar rizo:

$ make -C /usr/ports/ftp/curl install clean

Descargar chico:

$ fetch 'https://raw.githubusercontent.com/kerl/kerl/master/kerl'
$ chmod +x kerl

Actualice la lista de lanzamientos disponibles:

$ ./kerl update releases
The available releases are:
R10B-0 R10B-10 R10B-1a R10B-2 R10B-3 R10B-4 R10B-5 R10B-6 R10B-7 R10B-8 R10B-9 R11B-0 R11B-1 R11B-2 R11B-3 R11B-4 R11B-5 R12B-0 R12B-1 R12B-2 R12B-3 R12B-4 R12B-5 R13A R13B01 R13B02-1 R13B02 R13B03 R13B04 R13B R14A R14B01 R14B02 R14B03 R14B04 R14B R14B_erts-5.8.1.1 R15B01 R15B02 R15B02_with_MSVCR100_installer_fix R15B03-1 R15B03 R15B R16A_RELEASE_CANDIDATE R16B01 R16B02 R16B03-1 R16B03 R16B 17.0-rc1 17.0-rc2 17.0 17.1 17.3 17.4 17.5 18.0 18.1 18.2 18.2.1 18.3 19.0

Cree la versión requerida:

$ ./kerl build 18.3 erlang-18.3

Verifique que la compilación esté presente en la lista de compilación:

$ ./kerl list builds
18.3,erlang-18.3

Instale la compilación en algún lugar:

$ ./kerl install erlang-18.3 ./erlang-18.3

Obtenga el archivo activar si está ejecutando bash o el caparazón de pescado. Si está ejecutando un cshell, agregue el directorio bin de compilación a la RUTA:

$ setenv PATH "/some/where/erlang-18.3/bin/:$PATH"

Pruebe su nueva instalación:

$ which erl
/some/where/erlang-18.3/bin//erl

$ erl
Erlang/OTP 18 [erts-7.3] [source] [64-bit] [smp:2:2] [async-threads:10] [hipe] [kernel-poll:false]

Eshell V7.3  (abort with ^G)

Otros lanzamientos #

Si desea compilar otra versión de Erlang/OTP, busque los otros puertos en la colección:

  • lang/erlang-runtime15
  • lang/erlang-runtime16
  • lang/erlang-runtime17
  • lang/erlang-runtime18

Compile e instale Erlang/OTP en Ubuntu #

Los siguientes ejemplos muestran dos métodos principales para instalar Erlang/OTP en Ubuntu.

Método 1 - Paquete binario preconstruido #

Simplemente ejecute este comando y descargará e instalará la última versión estable de Erlang de Erlang Solutions.

$ sudo apt-get install erlang

Método 2: compilar e instalar desde el origen #

Descarga el archivo tar:

$ wget http://erlang.org/download/otp_src_19.0.tar.gz

Extraiga el archivo tar:

$ tar -zxf otp_src_19.0.tar.gz

Ingrese el directorio extraído y configure ERL_TOP para que sea la ruta actual:

$ cd otp_src_19.0
$ export ERL_TOP=`pwd`

Ahora, antes de configurar la compilación, debe asegurarse de tener todas las dependencias que necesita para instalar Erlang:

Dependencias básicas:

$ sudo apt-get install autoconf libncurses-dev build-essential

Otras dependencias de aplicaciones

Aplicación Instalación de dependencia
Hola $ sudo apt-get install m4
ODBC $ sudo apt-get install unixodbc-dev
OpenSSL $ sudo apt-get install libssl-dev
wxWidgets $ sudo apt-get install libwxgtk3.0-dev libglu-dev
Documentación $ sudo apt-get install fop xsltproc
Orber y otros proyectos de C++ $ sudo apt-get install g++
jinterfaz $ sudo apt-get install default-jdk

Configurar y construir: Puede establecer sus propias opciones o dejarlo en blanco para ejecutar la configuración predeterminada. Configuración avanzada y compilación para Erlang/OTP.

$ ./configure [ options ]
$ make

Probando la construcción:

$ make release_tests
$ cd release/tests/test_server
$ $ERL_TOP/bin/erl -s ts install -s ts smoke_test batch -s init stop

Después de ejecutar estos comandos, abra $ERL_TOP/release/tests/test_server/index.html con su navegador web y verifique que no tenga fallas. Si todas las pruebas pasaron, podemos continuar con la instalación.

Instalación:

$ make install

Compile e instale Erlang/OTP en OpenBSD #

Erlang en OpenBSD actualmente no funciona en las arquitecturas alpha, sparc y hppa.

Método 1 - Paquete binario preconstruido #

OpenBSD le permite elegir la versión deseada que desea instalar en su sistema:

######################################################################
# free-choice:
######################################################################
$ pkg_add erlang
# a       0: <None>
#   1: erlang-16b.03p10v0
#   2: erlang-17.5p6v0
#   3: erlang-18.1p1v0
#   4: erlang-19.0v0

######################################################################
# manual-choice:
######################################################################
pkg_add erlang%${version}
# example: 
pkg_add erlang%19

OpenBSD puede soportar múltiples versiones de Erlang. Para que sea más fácil de usar, cada binario tiene instalada la versión de Erlang en su nombre. Entonces, si ha instalado erlang-19.0v0, su binario erl será erl19.

Si desea utilizar erl, puede crear un enlace simbólico:

ln -s /usr/local/bin/erl19 /usr/local/bin/erl

o cree un alias en su archivo de configuración de shell o en el archivo .profile:

echo 'alias erl="erl19"' >> ~/.profile
# or
echo 'alias erl="erl19"' >> ~/.shrc

Ahora puede ejecutar erl:

erl19
# or if you have an alias or symlink
erl
# Erlang/OTP 19 [erts-8.0] [source] [async-threads:10] [kernel-poll:false]
# 
# Eshell V8.0  (abort with ^G)

Método 2: compilar e instalar usando puertos #

 RELEASE=OPENBSD_$(uname -r  | sed 's/\./_/g')
 cd /usr
 cvs -qz3 -danoncvs@anoncvs.openbsd.org:/cvs co -r${RELEASE}
 cd /usr/ports/lang/erlang
 ls -p 
 # 16/ 17/ 18/ 19/  CVS/  Makefile  Makefile.inc  erlang.port.mk
 cd 19
 make && make install

Método 3 - Construir desde la fuente #

Construir desde la fuente requiere paquetes adicionales:

  • git
  • gmake
  • autoconf-2.59
pkg_add git gmake autoconf%2.59
git clone https://github.com/erlang/otp.git
cd otp
AUTOCONF_VERSION="2.59" ./build_build all
