Si construye #

La construcción if (llamada sentencia IF de bloque en FORTRAN 77) es común en muchos lenguajes de programación. Ejecuta condicionalmente un bloque de código cuando una expresión lógica se evalúa como verdadera.

[name:] IF (expr) THEN
    block
[ELSE IF (expr) THEN [name]
    block]
[ELSE [name]
   block]
END IF [name]

dónde,

  • nombre - el nombre de la construcción if (opcional)
  • expr - una expresión lógica escalar entre paréntesis
  • bloque: una secuencia de cero o más declaraciones o construcciones

El nombre de una construcción al comienzo de una declaración si, entonces debe tener el mismo valor que el nombre de la construcción en la declaración final si, y debe ser único para la unidad de alcance actual.

En declaraciones if, (in)ecualidades y expresiones lógicas que evalúan una declaración se pueden usar con los siguientes operadores:

.LT.  which is <   ! less than
.LE.           <=  ! less than or equal
.GT.           >   ! greater than
.GE.           >=  ! greater than or equal
.EQ.           =   ! equal
.NE.           /=  ! not equal
.AND.              ! logical and
.OR.               ! logical or
.NOT.              ! negation

Ejemplos:

! simplest form of if construct
if (a > b) then
    c =  b / 2
end if
!equivalent example with alternate syntax
if(a.gt.b)then
   c=b/2
endif

! named if construct
circle: if (r >= 0) then
    l = 2 * pi * r
end if circle

! complex example with nested if construct
block: if (a < e) then
    if (abs(c - e) <= d) then
        a = a * c
    else
        a = a * d
    end if
else
    a = a * e
end if block

Un uso histórico de la construcción ‘si’ es en lo que se llama una declaración “aritmética si”. Sin embargo, dado que esto puede ser reemplazado por construcciones más modernas, no se trata aquí. Se pueden encontrar más detalles [aquí][1].

[1]: https://www.wikiod.com/es/fortran/alternativas-modernas-a-las-caracteristicas-historicas#Aritmética si declaración

construcción SELECCIONAR CASO #

Una construcción select case ejecuta condicionalmente un bloque de construcciones o sentencias dependiendo del valor de una expresión escalar en una sentencia select case. Esta construcción de control se puede considerar como un reemplazo para goto calculado.

[name:] SELECT CASE (expr)
[CASE (case-value [, case-value] ...) [name]
   block]...
[CASE DEFAULT [name]
   block]
END SELECT [name]

dónde,

  • nombre - el nombre de la construcción select case (opcional)
  • expr - una expresión escalar de tipo entero, lógico o carácter (entre paréntesis)
  • case-value: una o más expresiones de inicialización de caracteres, lógicas o enteras escalares entre paréntesis
  • bloque: una secuencia de cero o más declaraciones o construcciones

Ejemplos:

! simplest form of select case construct
select case(i)
case(:-1)
    s = -1
case(0)
    s = 0
case(1:)
    s = 1
case default
    print "Something strange is happened"
end select

En este ejemplo, el valor de caso (:-1) es un rango de valores que coincide con todos los valores menores que cero, (0) coincide con ceros y (1:) coincide con todos los valores superiores a cero, ` La sección predeterminada implica si otras secciones no se ejecutaron.

Construcción del bloque DO #

Una construcción do es una construcción en bucle que tiene una serie de iteraciones gobernadas por un control de bucle.

integer i
do i=1, 5
  print *, i
end do
print *, i

En el formulario anterior, la variable de bucle i pasa por el bucle 5 veces, tomando los valores del 1 al 5 por turno. Una vez completada la construcción, la variable de ciclo tiene el valor 6, es decir, la variable de ciclo se incrementa una vez más después de completar el ciclo.

De manera más general, la construcción del bucle do se puede entender de la siguiente manera

integer i, first, last, step
do i=first, last, step
end do

El ciclo comienza con i con el valor primero, incrementando cada iteración por paso hasta que i sea mayor que último (o menor que último si el tamaño del paso es negativo).

Es importante tener en cuenta que desde Fortran 95, la variable de bucle y las expresiones de control de bucle deben ser enteros.

Una iteración puede finalizar prematuramente con la instrucción ciclo

do i=1, 5
  if (i==4) cycle
end do

y toda la construcción puede dejar de ejecutarse con la instrucción exit

do i=1, 5
  if (i==4) exit
end do
print *, i

Las construcciones do pueden nombrarse:

do_name: do i=1, 5
end do do_name

lo cual es particularmente útil cuando hay construcciones do anidadas

do1: do i=1, 5
  do j=1,6
    if (j==3) cycle        ! This cycles the j construct
    if (j==4) cycle        ! This cycles the j construct
    if (i+j==7) cycle do1  ! This cycles the i construct
    if (i*j==15) exit do1  ! This exits the i construct
  end do
end do1

Las construcciones do también pueden tener un control de bucle indeterminado, ya sea “para siempre” o hasta que se cumpla una condición dada

integer :: i=0
do
  i=i+1
  if (i==5) exit
end do

o

integer :: i=0
do while (i<6)
  i=i+1
end do

Esto también permite un bucle infinito do a través de una declaración .true.

print *,'forever'
do while(.true.)
  print *,'and ever'
end do

Una construcción do también puede dejar indeterminado el orden de las iteraciones

do concurrent (i=1:5)
end do

teniendo en cuenta que la forma de control de bucle es la misma que en un control forall.

Hay varias restricciones sobre las sentencias que pueden ejecutarse dentro del rango de una construcción do concurrent que están diseñadas para garantizar que no haya dependencias de datos entre las iteraciones de la construcción. Esta indicación explícita por parte del programador puede permitir una mayor optimización (incluida la paralelización) por parte del compilador, lo que puede ser difícil de determinar de otra manera.

Las variables “privadas” dentro de una interacción se pueden realizar mediante el uso de una construcción de bloque dentro de do concurrent:

do concurrent (i=1:5, j=2:7)
  block
    real tempval  ! This is independent across iterations
  end block
end do

Otra forma de la construcción de bloque do utiliza una declaración etiquetada continue en lugar de end do:

    do 100, i=1, 5
100 continue

Incluso es posible anidar tales construcciones con una declaración de terminación compartida

    do 100, i=1,5
    do 100, j=1,5
100 continue

Ambas formas, y especialmente la segunda (que está obsoleta), generalmente deben evitarse en aras de la claridad.

Finalmente, también hay una construcción do sin bloque. Esto también se considera obsoleto y [se describe en otra parte] (https://www.wikiod.com/es/fortran/alternativas-modernas-a-las-caracteristicas-historicas#Construcciones DO sin bloque 201607242013447800136), junto con métodos para reestructurar como una construcción de bloque do.

DONDE construir #

La construcción where, disponible en Fortran90 en adelante, representa una construcción do enmascarada. La declaración de enmascaramiento sigue las mismas reglas que la declaración if, pero se aplica a todos los elementos de la matriz dada. El uso de where permite realizar operaciones en una matriz (o varias matrices del mismo tamaño), cuyos elementos cumplen una determinada regla. Esto se puede utilizar para simplificar operaciones simultáneas en varias variables.

Sintaxis:

[name]: where (mask)
    block
[elsewhere (mask)
    block]
[elsewhere
    block]
end where [name]

Aquí,

  • nombre - es el nombre dado al bloque (si tiene nombre)
  • máscara - es una expresión lógica aplicada a todos los elementos
  • bloque - serie de comandos a ejecutar

Ejemplos:

! Example variables
real:: A(5),B(5),C(5)
A = 0.0
B = 1.0
C = [0.0, 4.0, 5.0, 10.0, 0.0]

! Simple where construct use
where (C/=0)
    A=B/C
elsewhere
    A=0.0
end

! Named where construct
Block: where (C/=0)
    A=B/C
elsewhere
    A=0.0
end where Block
