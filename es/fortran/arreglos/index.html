Constructores de matrices #

Se puede crear un valor de matriz de rango 1 usando un constructor de matriz, con la sintaxis

(/ ... /)
[ ... ]

La forma [...] se introdujo en Fortran 2003 y generalmente se considera más clara de leer, especialmente en expresiones complejas. Este formulario se utiliza exclusivamente en este ejemplo.

Los valores que aparecen en un constructor de matriz pueden ser valores escalares, valores de matriz o bucles implícitos.

El tipo y los parámetros de tipo de la matriz construida coinciden con los de los valores en el constructor de la matriz.

[1, 2, 3]      ! A rank-1 length-3 array of default integer type
[1., 2., 3.]   ! A rank-1 length-3 array of default real type
["A", "B"]     ! A rank-1 length-2 array of default character type

integer, parameter :: A = [2, 4]
[1, A, 3]      ! A rank-1 length-4 array of default integer type, with A's elements

integer i
[1, (i, i=2, 5), 6]  ! A rank-1 length-6 array of default integer type with an implied-do

En los formularios anteriores, todos los valores dados deben ser del mismo tipo y parámetro de tipo. No se permite mezclar tipos o parámetros de tipo. Los siguientes ejemplos no son válidos

[1, 2.]      ! INVALID: Mixing integer and default real
[1e0, 2d0]   ! INVALID: Mixing default real and double precision
[1., 2._dp]  ! INVALID: Allowed only if kind `dp` corresponds to default real
["Hello", "Frederick"]  ! INVALID: Different length parameters

Para construir una matriz usando diferentes tipos, se debe dar una especificación de tipo para la matriz.

[integer :: 1, 2., 3d0]    ! A default integer array
[real(dp) :: 1, 2, 3._sp]  ! A real(dp) array
[character(len=9) :: "Hello", "Frederick"]  ! A length-2 array of length-9 characters

Esta última forma para matrices de caracteres es especialmente conveniente para evitar el relleno de espacio, como la alternativa

["Hello    ", "Frederick"]  ! A length-2 array of length-9 characters

El tamaño de una matriz llamada constante puede estar implícito en el constructor de matriz utilizado para establecer su valor

integer, parameter :: ids(*) = [1, 2, 3, 4]

y para tipos con parámetros de longitud, se puede suponer que el parámetro de longitud

character(len=*), parameter :: names(*) = [character(3) :: "Me", "You", "Her"]

La especificación de tipo también se requiere en la construcción de arreglos de longitud cero. De

[ ] ! Not a valid array constructor

el tipo y los parámetros de tipo no se pueden determinar a partir del conjunto de valores inexistentes. Para crear una matriz de enteros predeterminada de longitud cero:

[integer :: ]

Los constructores de matrices construyen solo matrices de rango 1. A veces, como al establecer el valor de una constante con nombre, también se requieren matrices de mayor rango en una expresión. Se pueden tomar matrices de rango superior del resultado de reformar con una matriz de rango 1 construida

integer, parameter :: multi_rank_ids(2,2) = RESHAPE([1,2,3,4], shape=[2,2])

En un constructor de matrices, los valores de la matriz en el orden de los elementos con cualquier matriz en la lista de valores como si los elementos individuales se dieran en el orden de los elementos de la matriz. Así, el ejemplo anterior

integer, parameter :: A = [2, 4]
[1, A, 3]      ! A rank-1 length-4 array of default integer type, with A's elements

es equivalente a

[1, 2, 4, 3]   ! With the array written out in array element order

Por lo general, los valores del constructor pueden ser expresiones arbitrarias, incluidos los constructores de matrices anidadas. Para que un constructor de matriz de este tipo cumpla ciertas condiciones, como ser una constante o una expresión de especificación, se aplican restricciones a los valores constituyentes.

Aunque no es un constructor de arreglos, ciertos valores de arreglos también pueden crearse convenientemente usando la función intrínseca spread. Por ejemplo

[(0, i=1,10)]  ! An array with 10 default integers each of value 0

es también el resultado de la función de referencia

SPREAD(0, 1, 10)

Notación básica #

Cualquier tipo puede declararse como una matriz usando el atributo dimensión o simplemente indicando directamente la(s) “dimensión”(s) de la matriz:

! One dimensional array with 4 elements
integer, dimension(4) :: foo

! Two dimensional array with 4 rows and 2 columns
real, dimension(4, 2) :: bar

! Three dimensional array
type(mytype), dimension(6, 7, 8) :: myarray

! Same as above without using the dimension keyword
integer :: foo2(4)
real :: bar2(4, 2)
type(mytype) :: myarray2(6, 7, 8)

La última forma de declarar una matriz multidimensional permite la declaración de matrices del mismo tipo de diferentes rangos/dimensiones en una línea, de la siguiente manera

real :: pencil(5), plate(3,-2:4), cuboid(0:3,-10:5,6)

El rango máximo (número de dimensiones) permitido es 15 en el estándar Fortran 2008 y antes era 7.

Fortran almacena arreglos en orden columna-principal. Es decir, los elementos de bar se almacenan en la memoria de la siguiente manera:

bar(1, 1), bar(2, 1), bar(3, 1), bar(4, 1), bar(1, 2), bar(2, 2), ...

En Fortran, la numeración de matrices comienza en 1 de forma predeterminada, en contraste con C, que comienza en 0. De hecho, en Fortran, puede especificar los límites superior e inferior para cada dimensión de forma explícita:

integer, dimension(7:12, -3:-1) :: geese

Esto declara una matriz de forma (6, 3), cuyo primer elemento es gansos (7, -3).

Se puede acceder a los límites inferior y superior a lo largo de las 2 (o más) dimensiones mediante las funciones intrínsecas ubound y lbound. De hecho, lbound(geese,2) devolvería -3, mientras que ubound(geese,1) devolvería 12.

Se puede acceder al tamaño de una matriz mediante la función intrínseca tamaño. Por ejemplo, size(geese, dim = 1) devuelve el tamaño de la primera dimensión, que es 6.

Arreglos completos, elementos de arreglos y secciones de arreglos #

Considere la matriz declarada como

real x(10)

Entonces tenemos tres aspectos de interés:

  1. Toda la matriz x;
  2. Elementos de matriz, como x(1);
  3. Secciones de matrices, como x(2:6).

arreglos completos #

En la mayoría de los casos, la matriz completa x se refiere a todos los elementos de la matriz como una sola entidad. Puede aparecer en sentencias ejecutables como print *, SUM(x), print *, SIZE(x) o x=1.

Una matriz completa puede hacer referencia a matrices que no tienen una forma explícita (como x arriba):

function f(y)
  real, intent(out) :: y(:)
  real, allocatable :: z(:)

  y = 1.         ! Intrinsic assignment for the whole array
  z = [1., 2.,]  ! Intrinsic assignment for the whole array, invoking allocation
end function

Una matriz de tamaño supuesto también puede aparecer como una matriz completa, pero solo en circunstancias limitadas (que se documentarán en otro lugar).

elementos de matriz #

Se hace referencia a un elemento de matriz que proporciona índices enteros, uno para cada rango de la matriz, que indica la ubicación en toda la matriz:

real x(5,2)
x(1,1) = 0.2
x(2,4) = 0.3

Un elemento de matriz es un escalar.

Secciones de matriz #

Una sección de matriz es una referencia a una cantidad de elementos (quizás solo uno) de una matriz completa, utilizando una sintaxis que involucra dos puntos:

real x(5,2)
x(:,1) = 0.         ! Referring to x(1,1), x(2,1), x(3,1), x(4,1) and x(5,1)
x(2,:) = 0.         ! Referring to x(2,1), x(2,2)
x(2:4,1) = 0.       ! Referring to x(2,1), x(3,1) and x(4,1)
x(2:3,1:2) = 0.     ! Referring to x(2,1), x(3,1), x(2,2) and x(3,2)
x(1:1,1) = 0.       ! Referring to x(1,1)
x([1,3,5],2) = 0.   ! Referring to x(1,2), x(3,2) and x(5,2)

El formulario final anterior usa un [subíndice de vector] (https://www.wikiod.com/es/fortran/arreglos#Arreglos completos, elementos de arreglos y secciones de arreglos). Esto está sujeto a una serie de restricciones más allá de otras secciones del arreglo.

Cada sección de la matriz es en sí misma una matriz, incluso cuando solo se hace referencia a un elemento. Es decir, x(1:1,1) es una matriz de rango 1 y x(1:1,1:1) es una matriz de rango 2.

Las secciones de matriz en general no tienen un atributo de toda la matriz. En particular, donde

real, allocatable :: x(:)
x = [1,2,3]     ! x is allocated as part of the assignment
x = [1,2,3,4]   ! x is dealloacted then allocated to a new shape in the assignment

la asignación

x(:) = [1,2,3,4,5]   ! This is bad when x isn't the same shape as the right-hand side

no está permitido: x(:), aunque una sección de matriz con todos los elementos de x, no es una matriz asignable.

x(:) = [5,6,7,8]

está bien cuando x tiene la forma del lado derecho.

Arreglos de componentes de arreglos #

type t
   real y(5)
end type t

type(t) x(2)

También podemos referirnos a arreglos completos, elementos de arreglos y secciones de arreglos en configuraciones más complicadas.

De lo anterior, x es una matriz completa. También tenemos

x(1)%y        ! A whole array
x(1)%y(1)     ! An array element
x%y(1)        ! An array section
x(1)%y(:)     ! An array section
x([1,2]%y(1)  ! An array section
x(1)%y(1:1)   ! An array section

En tales casos, no se nos permite tener más de una parte de la referencia que consiste en una matriz de rango 1. Lo siguiente, por ejemplo, no está permitido

x%y             ! Both the x and y parts are arrays
x(1:1)%y(1:1)   ! Recall that each part is still an array section

Arreglos asignables #

Los arreglos pueden tener el atributo asignable:

! One dimensional allocatable array
integer, dimension(:), allocatable :: foo
! Two dimensional allocatable array
real, dimension(:,:), allocatable :: bar

Esto declara la variable pero no le asigna ningún espacio.

! We can specify the bounds as usual
allocate(foo(3:5))

! It is an error to allocate an array twice
! so check it has not been allocated first
if (.not. allocated(foo)) then
  allocate(bar(10, 2))
end if

Una vez que ya no se necesita una variable, se puede desasignar:

deallocate(foo)

Si por alguna razón falla una declaración asignar, el programa se detendrá. Esto se puede evitar si se comprueba el estado mediante la palabra clave stat:

real, dimension(:), allocatable :: geese
integer :: status

allocate(geese(17), stat=status)
if (stat /= 0) then
  print*, "Something went wrong trying to allocate 'geese'"
  stop 1
end if

La instrucción desasignar también tiene la palabra clave stat:

deallocate (geese, stat=status)

status es una variable entera cuyo valor es 0 si la asignación o desasignación fue exitosa.

Especificación de la naturaleza de la matriz: rango y forma #

El atributo dimensión en un objeto especifica que ese objeto es una matriz. Hay, en Fortran 2008, cinco naturalezas de matriz:1

  • forma explícita
  • forma asumida
  • tamaño supuesto
  • forma diferida
  • forma implícita

Tome las tres matrices de rango 12

integer a, b, c
dimension(5) a    ! Explicit shape (default lower bound 1), extent 5
dimension(:) b    ! Assumed or deferred shape
dimension(*) c    ! Assumed size or implied shape array

Con estos, se puede ver que se requiere más contexto para determinar completamente la naturaleza de una matriz.

forma explícita #

Una matriz de forma explícita es siempre la forma de su declaración. A menos que la matriz se declare como local para un subprograma o una construcción de “bloque”, los límites que definen la forma deben ser expresiones constantes. En otros casos, una matriz de forma explícita puede ser un objeto automático, utilizando extensiones que pueden variar en cada invocación de un subprograma o ‘bloque’.

subroutine sub(n)
  integer, intent(in) :: n
  integer a(5)   ! A local explicit shape array with constant bound
  integer b(n)   ! A local explicit shape array, automatic object
end subroutine

forma asumida #

Una matriz de forma asumida es un argumento ficticio sin el atributo “asignable” o “puntero”. Tal matriz toma su forma del argumento real con el que está asociada.

integer a(5), b(10)
call sub(a)   ! In this call the dummy argument is like x(5)
call sub(b)   ! In this call the dummy argument is like x(10)

contains

  subroutine sub(x)
    integer x(:)    ! Assumed shape dummy argument
  end subroutine sub

end

Cuando un argumento ficticio ha asumido forma, el alcance que hace referencia al procedimiento debe tener una interfaz explícita disponible para ese procedimiento.

Tamaño supuesto #

Una matriz de tamaño asumido es un argumento ficticio cuyo tamaño se asume a partir de su argumento real.

subroutine sub(x)
  integer x(*)   ! Assumed size array
end subroutine

Las matrices de tamaño asumido se comportan de manera muy diferente a las matrices de forma supuesta y estas diferencias se documentan en otro lugar.

forma diferida #

Una matriz de forma diferida es una matriz que tiene el atributo “asignable” o “puntero”. La forma de una matriz de este tipo está determinada por su [asignación](https://www.wikiod.com/es/fortran/arreglos#Matrices asignables) o asignación de puntero.

integer, allocatable :: a(:)
integer, pointer :: b(:)

forma implícita #

Una matriz de forma implícita es una constante con nombre que toma su forma de la expresión utilizada para establecer su valor

integer, parameter :: a(*) = [1,2,3,4]

Las implicaciones de estas declaraciones de matriz en argumentos ficticios se documentarán en otro lugar.

1Una especificación técnica que amplía Fortran 2008 agrega una sexta naturaleza de matriz: rango asumido. Esto no está cubierto aquí.

2 Estos pueden escribirse de manera equivalente como

integer, dimension(5) :: a
integer, dimension(:) :: b
integer, dimension(*) :: c

o

integer a(5)
integer b(:)
integer c(*)

Operaciones de matrices #

Debido a sus objetivos computacionales, las operaciones matemáticas en matrices son sencillas en Fortran.

Adición y sustracción #

Las operaciones en matrices de la misma forma y tamaño son muy similares al álgebra matricial. En lugar de recorrer todos los índices con bucles, uno puede escribir sumas (y restas):

real, dimension(2,3) :: A, B, C
real, dimension(5,6,3) :: D
A    = 3.    ! Assigning single value to the whole array
B    = 5.    ! Equivalent writing for assignment
C    = A + B ! All elements of C now have value 8.
D    = A + B ! Compiler will raise an error. The shapes and dimensions are not the same

Las matrices de corte también son válidas:

integer :: i, j
real, dimension(3,2) :: Mat = 0.
real, dimension(3)   :: Vec1 = 0., Vec2 = 0., Vec3 = 0.
i = 0
j = 0
do i = 1,3
  do j = 1,2
    Mat(i,j) = i+j
  enddo
enddo
Vec1 = Mat(:,1)
Vec2 = Mat(:,2)
Vec3 = Mat(1:2,1) + Mat(2:3,2)

Función #

De la misma manera, la mayoría de las funciones intrínsecas se pueden usar implícitamente suponiendo una operación por componentes (aunque esto no es sistemático):

real, dimension(2) :: A, B
A(1) = 6
A(2) = 44 ! Random values
B    = sin(A) ! Identical to B(1) = sin(6), B(2) = sin(44).

Multiplicación y división #

Se debe tener cuidado con el producto y la división: las operaciones intrínsecas que usan los símbolos * y / son por elementos:

real, dimension(2) :: A, B, C
A(1) = 2
A(2) = 4
B(1) = 1
B(2) = 3
C = A*B ! Returns C(1) = 2*1 and C(2) = 4*3

Esto no debe confundirse con las operaciones matriciales (ver más abajo).

Operaciones con matrices #

Las operaciones matriciales son procedimientos intrínsecos. Por ejemplo, el producto matricial de las matrices de la sección anterior se escribe de la siguiente manera:

real, dimension(2,1) :: A, B
real, dimension(1,1) :: C
A(1) = 2
A(2) = 4
B(1) = 1
B(2) = 3
C = matmul(transpose(A),B) ! Returns the scalar product of vectors A and B

Las operaciones complejas permiten encapsular funciones mediante la creación de matrices temporales. Si bien algunos compiladores y opciones de compilación lo permiten, no se recomienda. Por ejemplo, un producto que incluye una matriz transpuesta se puede escribir:

real, dimension(3,3) :: A, B, C
A(:) = 4
B(:) = 5
C = matmul(transpose(A),matmul(B,matmul(A,transpose(B)))) ! Equivalent to A^t.B.A.B^T

Secciones de matrices avanzadas: tripletes de subíndices y subíndices vectoriales #

Como se menciona en [otro ejemplo] (https://www.wikiod.com/es/fortran/arreglos#Arreglos completos, elementos de arreglos y secciones de arreglos) un subconjunto de los elementos de una matriz, llamado una sección de matriz, puede ser referenciada. De ese ejemplo podemos tener

real x(10)
x(:)   = 0.
x(2:6) = 1.
x(3:4) = [3., 5.]

Sin embargo, las secciones de matriz pueden ser más generales que esto. Pueden tomar la forma de tripletes de subíndices o subíndices vectoriales.

Trillizos de subíndices #

Un subíndice triple toma la forma [bound1]:[bound2][:stride]. Por ejemplo

real x(10)
x(1:10) = ...   ! Elements x(1), x(2), ..., x(10)
x(1:) = ...     ! The omitted second bound is equivalent to the upper, same as above
x(:10) = ...    ! The omitted first bound is equivalent to the lower, same as above
x(1:6:2) = ...  ! Elements x(1), x(3), x(5)
x(5:1) = ...    ! No elements: the lower bound is greater than the upper
x(5:1:-1) = ... ! Elements x(5), x(4), x(3), x(2), x(1)
x(::3) = ...    ! Elements x(1), x(4), x(7), x(10), assuming omitted bounds
x(::-3) = ...   ! No elements: the bounds are assumed with the first the lower, negative stride

Cuando se especifica un paso (que no debe ser cero), la secuencia de elementos comienza con el primer límite especificado. Si la zancada es positiva (resp. negativa), los elementos seleccionados siguen una secuencia incrementada (resp. decrementada) por la zancada hasta que se toma el último elemento no mayor (resp. menor) que el segundo límite. Si se omite el paso, se trata como si fuera uno.

Si el primer límite es mayor que el segundo límite y el paso es positivo, no se especifica ningún elemento. Si el primer límite es más pequeño que el segundo límite y el paso es negativo, no se especifica ningún elemento.

Cabe señalar que x(10:1:-1) no es lo mismo que x(1:10:1) aunque cada elemento de x aparece en ambos casos.

Subíndices vectoriales #

Un subíndice de vector es una matriz de enteros de rango 1. Esto designa una secuencia de elementos correspondientes a los valores de la matriz.

real x(10)
integer i
x([1,6,4]) = ...     ! Elements x(1), x(6), x(4)
x([(i,i=2,4)]) = ... ! Elements x(2), x(3) and x(4)
print*, x([2,5,2])   ! Elements x(2), x(5) and x(2)

Una sección de matriz con un subíndice de vector está restringida en cómo se puede usar:

  • no puede ser un argumento asociado con un argumento ficticio que se define en el procedimiento;
  • puede que no sea el objetivo en una declaración de asignación de puntero;
  • no puede ser un archivo interno en una declaración de transferencia de datos.

Además, tal sección de matriz puede no aparecer en una declaración que involucre su definición cuando el mismo elemento se selecciona dos veces. Desde arriba:

print*, x([2,5,2])   ! Elements x(2), x(5) and x(2) are printed
x([2,5,2]) = 1.      ! Not permitted: x(2) appears twice in this definition

Secciones de matriz de rango superior #

real x(5,2)
print*, x(::2,2:1:-1)  ! Elements x(1,2), x(3,2), x(5,2), x(1,1), x(3,1), x(5,1)
