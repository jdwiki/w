Una escena está representada por el elemento <a-scene>. La escena es el objeto raíz global y todas las entidades están contenidas dentro de la escena.

La escena hereda de la clase Entity, por lo que hereda todas sus propiedades, sus métodos, la capacidad de adjuntar componentes y el comportamiento para esperar todos sus nodos secundarios (por ejemplo, <a-assets> y <a- entidad>) para cargar antes de iniciar el ciclo de procesamiento.

Parámetros #

Parámetro Detalles
comportamientos Matriz de componentes con métodos de marca que se ejecutarán en cada cuadro.
cámara Cámara activa three.js.
lienzo Referencia al elemento canvas.
esMóvil Si se detecta o no que el entorno es móvil.
objeto3D TRES. Objeto Escena.
procesador Activo TRES.WebGLRenderer.
procesamientoIniciado Si la escena se está renderizando.
efecto Renderizador para VR creado al pasar el renderizador activo a THREE.VREffect.
sistemas Sistemas instanciados.
tiempo Tiempo de actividad global de la escena en segundos.

MÉTODOS #

| Nombre | Descripción | |———|————————————— ————————————————– ——————————-| | entrarVR | Cambie a reproducción estéreo y envíe contenido a los auriculares. Debe llamarse dentro de un controlador de eventos generado por el usuario como click. la primera vez que una página ingresa a VR. | | salirVR | Cambia a renderizador mono y deja de presentar contenido en los auriculares. | | recargar | Revertir la escena a su estado original. |

EVENTOS #

| Nombre | Descripción | |————–|———————————- —| | entrar-vr | El usuario ingresó a VR y los auriculares comenzaron a presentar contenido. | | salir-vr | El usuario salió de VR y el visor dejó de presentar contenido. | | cargado | Todos los nodos se han cargado. | | renderstart | El ciclo de procesamiento ha comenzado. |

Adjuntar componentes de escena #

Los componentes se pueden adjuntar a la escena, así como a las entidades. A-Frame se envía con algunos componentes para configurar la escena:

Componente Detalles
incrustado Elimina los estilos de pantalla completa del lienzo.
voluntad El dara.
atajos de teclado Alternar atajos de teclado.
inspectora Inyecte el A-Frame Inspector.
estadísticas Alternar estadísticas de rendimiento.
vr-modo-ui Alternar la interfaz de usuario para entrar y salir de VR.
depurar Habilita la serialización de componente a DOM.

Uso de escenas incrustadas #

Si desea utilizar contenido de WebVR mezclado con contenido HTML, por ejemplo, cuando está creando un contenido clave de presentación ampliado, puede utilizar la etiqueta incrustada. Cuando está usando esto, es posible mirar alrededor dentro del contenido de 360° usando el giroscopio de su teléfono inteligente o haciendo clic y arrastrando en la computadora.

<script src="https://aframe.io/releases/0.5.0/aframe.min.js"></script>
<div class="vrcontent">
  <a-scene embedded>
    <a-assets>
      <img id="sky" src="https://c1.staticflickr.com/5/4248/34705881091_37b5cf2d28_o.jpg" alt="" />
    </a-assets>

    <a-sky src="#sky"></a-sky>
  </a-scene>
</div>

<div class="overlay">
  <button class="calltoaction">Click me!</button>
</div>

Depurar #

El componente de depuración permite la serialización de componente a DOM.

<a-scene debug></a-scene>

Serialización de componente a DOM #

De forma predeterminada, por motivos de rendimiento, A-Frame no actualiza el DOM con los datos de los componentes. Si abrimos el inspector DOM del navegador, veremos que solo los nombres de los componentes (y no los valores) son visibles.

<a-entity geometry material position rotation></a-entity>

A-Frame almacena los datos de los componentes en la memoria. La actualización del DOM requiere tiempo de CPU para convertir los datos de los componentes internos en cadenas. Si queremos ver la actualización del DOM con fines de depuración, podemos adjuntar el componente de depuración a la escena. Los componentes buscarán un componente de depuración habilitado antes de intentar serializar en el DOM. Entonces podremos ver los datos de los componentes en el DOM:

<a-entity geometry="primitive: box" material="color: red" position="1 2 3" rotation="0 180 0"></a-entity>

Asegúrese de que este componente no esté activo en producción.

Serializando manualmente a DOM #

Para serializar manualmente a DOM, use Entity.flushToDOM o Component.flushToDOM:

document.querySelector('a-entity').components.position.flushToDOM();  // Flush a component.
document.querySelector('a-entity').flushToDOM();  // Flush an entity.
document.querySelector('a-entity').flushToDOM(true);  // Flush an entity and its children.
document.querySelector('a-scene').flushToDOM(true);  // Flush every entity.

Ejecutar scripts de contenido en la escena #

La forma recomendada es escribir un componente y adjuntarlo al elemento de escena. La escena y sus hijos se inicializarán antes que este componente.

AFRAME.registerComponent('do-something', {
  init: function () {
    var sceneEl = this.el;
  }
});

<a-scene do-something></a-scene>

Si por alguna razón en particular prefiere no escribir un componente dedicado, debe esperar a que la escena termine de inicializarse y adjuntarse:

var scene = document.querySelector('a-scene');

if (scene.hasLoaded) {
  run();
} else {
  scene.addEventListener('loaded', run);
}

function run () {
  var entity = scene.querySelector('a-entity');
  entity.setAttribute('material', 'color', 'red');
}
