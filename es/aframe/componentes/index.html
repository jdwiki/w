En el patrón entidad-componente-sistema, un componente es un fragmento de datos reutilizable y modular que conectamos a una entidad para agregar apariencia, comportamiento y/o funcionalidad.

En A-Frame, los componentes modifican entidades que son objetos 3D en la escena. Mezclamos y componemos componentes para construir objetos complejos. Nos permiten encapsular three.js y código JavaScript en módulos que podemos usar de forma declarativa desde HTML. Los componentes son más o menos análogos a CSS.

Definición de métodos del controlador de ciclo de vida #

Siendo el esquema la anatomía, los métodos del ciclo de vida son la fisiología; el esquema define la forma de los datos, los métodos del controlador de ciclo de vida usar los datos para modificar la entidad. Los manipuladores normalmente interactuarán con el API de entidad.

Descripción general de los métodos #

Método Descripción
inicio Se llama una vez cuando se inicializa el componente. Se utiliza para configurar el estado inicial y crear instancias de variables.
actualizar Se llama tanto cuando se inicializa el componente como cuando se actualiza cualquiera de las propiedades del componente (por ejemplo, a través de setAttribute). Se utiliza para modificar la entidad.
eliminar Se llama cuando el componente se elimina de la entidad (por ejemplo, a través de removeAttribute) o cuando la entidad se desconecta de la escena. Se utiliza para deshacer todas las modificaciones anteriores a la entidad.
garrapata Llamado en cada bucle de renderizado o marca de la escena. Se utiliza para cambios o controles continuos.
jugar Se llama cada vez que se reproduce la escena o la entidad para agregar cualquier fondo o comportamiento dinámico. También se llama una vez cuando se inicializa el componente. Se utiliza para iniciar o reanudar el comportamiento.
pausa Se llama cada vez que la escena o la entidad se detiene para eliminar cualquier fondo o comportamiento dinámico. También se llama cuando el componente se elimina de la entidad o cuando la entidad se desconecta de la escena. Se utiliza para pausar el comportamiento.
actualizarEsquema Se llama cada vez que se actualiza alguna de las propiedades del componente. Se puede utilizar para modificar dinámicamente el esquema.

Propiedades de prototipo de componente #

Dentro de los métodos, tenemos acceso al prototipo del componente a través de this:

| Propiedad | Descripción | |—————–|—————————————- ————————————————– ————————————————– ————| | estos.datos | Propiedades de los componentes analizados calculadas a partir de los valores predeterminados del esquema, los mixins y los atributos de la entidad. | | esto.el | Referencia a la [entidad][entidad] como un elemento HTML. | | this.el.sceneEl | Referencia a la [escena][escena] como un elemento HTML. | | este.id | Si el componente puede tener [múltiples instancias][múltiples], el ID de la instancia individual del componente (por ejemplo, foo de sound__foo). |

MÉTODOS #

.init()

.init () se llama una vez al comienzo del ciclo de vida del componente. Una entidad puede llamar al controlador init del componente:

  • Cuando el componente se establece estáticamente en la entidad en el archivo HTML y se carga la página.
  • Cuando el componente se establece en una entidad adjunta a través de setAttribute.
  • Cuando el componente se establece en una entidad no adjunta, y la entidad es luego adjunto a la escena a través de appendChild.

El controlador init se usa a menudo para:

  • Configurar el estado inicial y las variables
  • Métodos de enlace
  • Adjuntar detectores de eventos

Por ejemplo, el init de un componente de cursor establecería variables de estado, enlazaría métodos y agregue detectores de eventos:

AFRAME.registerComponent('cursor', {
  // ...
  init: function () {
    // Set up initial state and variables.
    this.intersection = null;
    // Bind methods.
    this.onIntersection = AFRAME.utils.bind(this.onIntersection, this);
    // Attach event listener.
    this.el.addEventListener('raycaster-intersection', this.onIntersection);
  }
  // ...
});

.actualizar (datosantiguos)

.update (oldData) se llama cada vez que cambian las propiedades del componente, incluso al comienzo del ciclo de vida del componente. Una entidad puede llamar a un controlador update del componente:

  • Después de llamar a init (), al comienzo del ciclo de vida del componente.
  • Cuando las propiedades del componente se actualizan con .setAttribute.

El controlador update a menudo se usa para:

  • Hacer la mayor parte del trabajo al hacer modificaciones a la entidad, usando this.data.
  • Modificar la entidad cada vez que cambien una o más propiedades de los componentes.

Se pueden hacer modificaciones granulares a la entidad [diffing][diff] el actual conjunto de datos (this.data) con el conjunto de datos anterior antes de la actualización (oldData).

A-Frame llama a .update() tanto al comienzo del ciclo de vida de un componente como cada vez que los datos de un componente cambian (por ejemplo, como resultado de setAttribute). los el controlador de actualización a menudo usa this.data para modificar la entidad. El controlador de actualización tiene acceso al estado anterior de los datos de un componente a través de su primer argumento. Podemos usar los datos previos de un componente para decir exactamente qué propiedades cambiadas para hacer actualizaciones granulares.

Por ejemplo, la actualización del componente visible establece la visibilidad de la entidad.

AFRAME.registerComponent('visible', {
  /**
   * this.el is the entity element.
   * this.el.object3D is the three.js object of the entity.
   * this.data is the component's property or properties.
   */
  update: function (oldData) {
    this.el.object3D.visible = this.data;
  }
  // ...
});

.eliminar ()

.remove () se llama cada vez que el componente se separa de la entidad. Un la entidad puede llamar al controlador remove de un componente:

  • Cuando el componente se elimina de la entidad a través de removeAttribute.
  • Cuando la entidad se separa de la escena (por ejemplo, removeChild).

El controlador remove se usa a menudo para:

  • Eliminar, deshacer o limpiar todas las modificaciones del componente a la entidad.
  • Separar los oyentes de eventos.

Por ejemplo, cuando se quita el [componente de luz][luz], el componente de luz quitará el objeto de luz que había puesto previamente sobre la entidad, por lo que sacándolo de la escena.

AFRAME.registerComponent('light', {
  // ...
  remove: function () {
    this.el.removeObject3D('light');
  }
  // ...
});

.tick (hora, timeDelta)

.tick () se llama en cada marca o fotograma del bucle de renderizado de la escena. La escena llamará al controlador tick de un componente:

  • En cada fotograma del bucle de renderizado.
  • Del orden de 60 a 120 veces por segundo.
  • Si la entidad o escena no está en pausa (por ejemplo, el Inspector está abierto).
  • Si la entidad sigue adherida a la escena.

El manejador tick a menudo se usa para:

  • Modificar continuamente la entidad en cada cuadro o en un intervalo.
  • Encuesta de condiciones.

El controlador tick proporciona el tiempo de actividad global de la escena en milisegundos (tiempo) y la diferencia de tiempo en milisegundos desde el último cuadro (timeDelta). Estos se pueden usar para la interpolación o para ejecutar solo partes del manejador tick en un intervalo establecido.

Por ejemplo, el componente de controles rastreados progresará las animaciones del controlador, actualizar la posición y rotación del controlador, y verifique si hay pulsaciones de botones.

AFRAME.registerComponent('tracked-controls', {
  // ...
  tick: function (time, timeDelta) {
    this.updateMeshAnimation();
    this.updatePose();
    this.updateButtons();
  }
  // ...
});

.pausa()

.pause () se llama cuando la entidad o escena se detiene. La entidad puede llamar a un controlador de pausa del componente:

  • Antes de que se elimine el componente, antes de que se llame al controlador remove.
  • Cuando la entidad está en pausa con Entity.pause().
  • Cuando la escena está en pausa con Scene.pause () (por ejemplo, se abre el Inspector).

El controlador de pausa a menudo se usa para:

  • Quitar detectores de eventos.
  • Eliminar cualquier posibilidad de comportamiento dinámico.

Por ejemplo, el componente de sonido pausará el sonido y eliminará un detector de eventos que habría reproducido un sonido en un evento:

AFRAME.registerComponent('sound', {
  // ...
  pause: function () {
    this.pauseSound();
    this.removeEventListener();
  }
  // ...
});

.jugar ()

.play () se llama cuando se reanuda la entidad o la escena. La entidad puede llamar el controlador play de un componente:

  • Cuando el componente se adjunta por primera vez, después de llamar al controlador update.
  • Cuando la entidad se detuvo pero luego se reanudó con Entity.play ().
  • Cuando la escena se detuvo pero luego se reanudó con Scene.play ().

El controlador play a menudo se usa para:

  • Agregar detectores de eventos.

Por ejemplo, el componente de sonido reproducirá el sonido y actualizará el detector de eventos que reproduciría un sonido en un evento:

AFRAME.registerComponent('sound', {
  // ...
  play: function () {
    if (this.data.autoplay) { this.playSound(); }
    this.updateEventListener();
  }
  // ...
});

.updateSchema (datos)

.updateSchema (), si está definido, se llama en cada actualización para verificar si el esquema necesita ser modificado dinámicamente.

El controlador updateSchema a menudo se usa para:

  • Actualizar o ampliar dinámicamente el esquema, normalmente en función del valor de una propiedad.

Por ejemplo, el componente geométrico comprueba si el primitivo propiedad cambiada para determinar si actualizar el esquema para una diferente tipo de geometría:

AFRAME.registerComponent('geometry', {
  // ...
  updateSchema: (newData) {
    if (newData.primitive !== this.data.primitive) {
      this.extendSchema(GEOMETRIES[newData.primitive].schema);
    }
  }
  // ...
});

COMPONENTE PROTOTIPO MÉTODOS #

.flushToDOM()

Para ahorrar tiempo de CPU en la clasificación, A-Frame solo actualizará en modo de depuración la representación serializada del componente en el DOM real. Llamar a flushToDOM () serializará manualmente los datos del componente y actualizará el DOM:

document.querySelector('[geometry]').components.geometry.flushToDOM();

Registre un componente A-Frame personalizado #

AFRAME.registerComponent (nombre, definición) #

Registre un componente A-Frame. Debemos registrar los componentes antes de usarlos en cualquier lugar de . Es decir, de un archivo HTML, los componentes deben estar en orden antes de .

  • {cadena} nombre - Nombre del componente. La API pública del componente representada a través de un nombre de atributo HTML.
  • Definición de {Objeto} - Definición de componente. Contiene métodos de controlador de ciclo de vida y esquema.

Registro del componente en foo en su archivo js, ​​por ejemplo, foo-component.js #

AFRAME.registerComponent('foo', {
  schema: {},
  init: function () {},
  update: function () {},
  tick: function () {},
  remove: function () {},
  pause: function () {},
  play: function () {}
});

Uso del componente foo en tu escena #

<html>
  <head>
    <script src="aframe.min.js"></script>
    <script src="foo-component.js"></script>
  </head>
  <body>
    <a-scene>
      <a-entity foo></a-entity>
    </a-scene>
  </body>
</html>

Formulario HTML del componente #

Un componente contiene un depósito de datos en forma de una o más propiedades de componente. Los componentes usan estos datos para modificar entidades. Considere un componente de motor, podríamos definir propiedades como caballos de fuerza o cilindros.

Los atributos HTML representan nombres de componentes y el valor de esos atributos representan datos de componentes.

Componente de propiedad única #

Si un componente es un componente de propiedad única, lo que significa que sus datos consisten en un valor único, entonces en HTML, el valor del componente parece un atributo HTML normal:

<!-- `position` is the name of the position component. -->
<!-- `1 2 3` is the data of the position component. -->
<a-entity position="1 2 3"></a-entity>

Componente multipropiedad #

Si un componente es un componente de múltiples propiedades, lo que significa que los datos se componen de múltiples propiedades y valores, entonces en HTML, el valor del componente se parece a los estilos CSS en línea:

<!-- `light` is the name of the light component. -->
<!-- The `type` property of the light is set to `point`. -->
<!-- The `color` property of the light is set to `crimson`. -->
<a-entity light="type: point; color: crimson"></a-entity>

Definición de objeto de esquema de componente #

El esquema es un objeto que define y describe la propiedad o propiedades del componente. Las claves del esquema son los nombres de la propiedad, y los valores del esquema definen los tipos y valores de la propiedad (en el caso de un componente de varias propiedades):

Definiendo el esquema en su componente

AFRAME.registerComponent('bar', {
  schema: {
    color: {default: '#FFF'},
    size: {type: 'int', default: 5}
  }
}

Anular los valores predeterminados del esquema definido

<a-scene>
  <a-entity bar="color: red; size: 20"></a-entity>
</a-scene>

Esquema de propiedad única #

Un componente puede ser un componente de una sola propiedad (que consta de una valor anónimo) o un componente de propiedades múltiples (que consta de múltiples valores). A-Frame deducirá si un componente es de propiedad única o no. propiedad múltiple basada en la estructura del esquema.

El esquema de un componente de propiedad única contiene claves type y/o default, y los valores del esquema son valores simples en lugar de objetos:

AFRAME.registerComponent('foo', {
  schema: {type: 'int', default: 5}
});

<a-scene>
  <a-entity foo="20"></a-entity>
</a-scene>

Tipos de propiedades de esquema de componentes de A-Frame #

Los tipos de propiedad definen principalmente cómo el esquema analiza los datos entrantes del DOM para cada propiedad. Los datos analizados estarán disponibles a través de data propiedad en el prototipo del componente. A continuación se muestran las propiedades integradas de A-Frame tipos:

Tipo de propiedad Descripción Valor predeterminado
matriz Analiza los valores separados por comas en una matriz (es decir, "1, 2, 3" a ['1', '2', '3']). []
activo Para direcciones URL que apuntan a recursos generales. Puede analizar la URL de una cadena en forma de url (<url>). Si el valor es un selector de ID de elemento (por ejemplo, #texture), este tipo de propiedad llamará a getElementById y getAttribute('src') para devolver una URL. El tipo de propiedad asset puede o no cambiar para manejar XHR o devolver MediaElements directamente (por ejemplo, elementos <img>). ''
audio Mismo análisis que el tipo de propiedad asset. Posiblemente será utilizado por A-Frame Inspector para presentar activos de audio. ''
booleano Analiza la cadena a booleano (es decir, "falso" a falso, todo lo demás verdadero). falso
color Actualmente no hace ningún análisis. Utilizado principalmente por el A-Frame Inspector para presentar un selector de color. Además, se requiere usar el tipo de color para que funcionen las animaciones en color. #FFF
int Llama a parseInt (por ejemplo, "124.5" a 124). 0
mapa Mismo análisis que el tipo de propiedad asset. Posiblemente se usará con A-Frame Inspector para presentar activos de textura. ''
modelo Mismo análisis que el tipo de propiedad asset. Posiblemente se utilizará con el A-Frame Inspector para presentar los activos del modelo. ''
número Llama a parseFloat (por ejemplo, '124.5'' a ‘124.5’’). 0
selector Llama a querySelector (por ejemplo, "#box" a <a-entity id="box">). nulo
selectorAll Llama a querySelectorAll y convierte NodeList en Array (por ejemplo, ".boxes" en [<a-entity class=“boxes”, …]), nulo
cadena No hace ningún análisis. '’
vec2 Analiza dos números en un objeto {x, y} (por ejemplo, 1 -2 a {x: 1, y: -2}. {x: 0, y: 0}
vec3 Analiza tres números en un objeto {x, y, z} (por ejemplo, 1 -2 3 a {x: 1, y: -2, z: 3}. {x: 0, y: 0, z: 0}
vec4 Analiza cuatro números en un objeto {x, y, z, w} (por ejemplo, 1 -2 3 -4.5 a {x: 1, y: -2, z: 3, w: -4.5} {x: 0, y: 0, z: 0, w: 0}

Inferencia del tipo de propiedad #

El esquema intentará inferir un tipo de propiedad dado solo un valor predeterminado:

schema: {default: 10}  // type: "number"
schema: {default: "foo"}  // type: "string"
schema: {default: [1, 2, 3]}  // type: "array"

El esquema establecerá un valor predeterminado si no se proporciona, dado el tipo de propiedad:

schema: {type: 'number'}  // default: 0
schema: {type: 'string'}  // default: ''
schema: {type: 'vec3'}  // default: {x: 0, y: 0, z: 0}

Tipo de propiedad personalizada #

También podemos definir nuestro propio tipo de propiedad o analizador proporcionando un parse función en lugar de un tipo:

schema: {
  // Parse slash-delimited string to an array 
  // (e.g., `foo="myProperty: a/b"` to `['a', 'b']`).
  myProperty: {
    default: [],
    parse: function (value) {
      return value.split('/');
    }
  }
}

Acceso a los miembros y métodos de un componente #

Se puede acceder a los miembros y métodos de un componente a través de la entidad desde el objeto .components. Busque el componente en el mapa de componentes de la entidad y tendremos acceso a las partes internas del componente. Considere este componente de ejemplo:

AFRAME.registerComponent('foo', {
  init: function () {
    this.bar = 'baz';
  },
  qux: function () {
    // ...
  }
});

Accedamos al miembro bar y al método qux:

var fooComponent = document.querySelector('[foo]').components.foo;
console.log(fooComponent.bar);
fooComponent.qux();
