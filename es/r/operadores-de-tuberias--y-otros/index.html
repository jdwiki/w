Los operadores de tubería, disponibles en magrittr, dplyr y otros paquetes de R, procesan un objeto de datos utilizando una secuencia de operaciones al pasar el resultado de un paso como entrada para el siguiente paso utilizando operadores infix en lugar de los más típicos. Método R de llamadas a funciones anidadas.

Tenga en cuenta que el objetivo previsto de los operadores de tuberías es aumentar la legibilidad humana del código escrito. Consulte la sección Comentarios para conocer las consideraciones de rendimiento.

Sintaxis #

  • lhs %>% rhs # sintaxis de tubería para rhs(lhs)

  • lhs %>% rhs(a = 1) # sintaxis de tubería para rhs(lhs, a = 1)

  • lhs %>% rhs(a = 1, b = .) # sintaxis de tubería para rhs(a = 1, b = lhs)

  • lhs %<>% rhs # sintaxis de tubería para lhs <- rhs(lhs)

  • lhs %$% rhs(a) # sintaxis de tubería para with(lhs, rhs(lhs$a))

  • lhs %T>% rhs # sintaxis de tubería para { rhs(lhs); izq }

Parámetros #

izquierdo derecho
Un valor o el marcador de posición magrittr. Una llamada de función utilizando la semántica magrittr

Paquetes que usan %>% #

El operador de tubería se define en el paquete magrittr, pero ganó gran visibilidad y popularidad con el paquete dplyr (que importa la definición de magrittr). Ahora es parte de tidyverse, que es una colección de paquetes que “funcionan en armonía porque comparten representaciones de datos y diseño de API comunes”.

El paquete magrittr también proporciona varias variaciones del operador de tubería para aquellos que desean más flexibilidad en la tubería, como la tubería de asignación compuesta %<>%, la tubería de exposición %$% y el operador en T % T>%. También proporciona un conjunto de funciones de alias para reemplazar las funciones comunes que tienen una sintaxis especial (+, [, [[, etc.) para que puedan usarse fácilmente dentro de una cadena de tuberías.

Búsqueda de documentación #

Como con cualquier operador infijo (como +, *, ^, &, %in%), puede encontrar la documentación oficial si la pone entre comillas: ?'% >%' o help('%>%') (asumiendo que ha cargado un paquete que adjunta pkg:magrittr).

Teclas de acceso rápido #

Hay una tecla de acceso directo especial en RStudio para el operador de tubería: Ctrl+Shift+M (Windows y Linux), `Cmd+Shift+ M’ (Mac).

Consideraciones de rendimiento #

Si bien el operador de tubería es útil, tenga en cuenta que hay un impacto negativo en el rendimiento debido principalmente a la sobrecarga de su uso. Considere cuidadosamente las dos cosas siguientes cuando utilice el operador de tuberías:

  • Rendimiento de la máquina (bucles)
  • Evaluación (objeto %>% rm() no elimina objeto)

Uso básico y encadenamiento #

El operador de tubería, %>%, se usa para insertar un argumento en una función. No es una función básica del lenguaje y solo se puede usar después de adjuntar un paquete que la proporcione, como magrittr. El operador de tubería toma el lado izquierdo (LHS) de la tubería y lo usa como el primer argumento de la función en el lado derecho (RHS) de la tubería. Por ejemplo:

library(magrittr)

1:10 %>% mean
# [1] 5.5

# is equivalent to
mean(1:10)
# [1] 5.5

La tubería se puede usar para reemplazar una secuencia de llamadas a funciones. Varias tuberías nos permiten leer y escribir la secuencia de izquierda a derecha, en lugar de de adentro hacia afuera. Por ejemplo, supongamos que tenemos ‘años’ definidos como un factor pero queremos convertirlo en un valor numérico. Para evitar una posible pérdida de información, primero convertimos a carácter y luego a numérico:

years <- factor(2008:2012)

# nesting
as.numeric(as.character(years))

# piping
years %>% as.character %>% as.numeric

Si no queremos que el LHS (lado izquierdo) se use como el primer argumento en el RHS (lado derecho), existen soluciones alternativas, como nombrar los argumentos o usar . para indicar dónde va la entrada canalizada. .

# example with grepl
# its syntax:
# grepl(pattern, x, ignore.case = FALSE, perl = FALSE, fixed = FALSE, useBytes = FALSE)

# note that the `substring` result is the *2nd* argument of grepl
grepl("Wo", substring("Hello World", 7, 11))

# piping while naming other arguments
"Hello World" %>% substring(7, 11) %>% grepl(pattern = "Wo")

# piping with .
"Hello World" %>% substring(7, 11) %>% grepl("Wo", .)

# piping with . and curly braces
"Hello World" %>% substring(7, 11) %>% { c(paste('Hi', .)) }
#[1] "Hi World"

#using LHS multiple times in argument with curly braces and .
"Hello World" %>% substring(7, 11) %>% { c(paste(. ,'Hi', .)) }
#[1] "World Hi World"

Secuencias funcionales #

Dada una secuencia de pasos que usamos repetidamente, a menudo es útil almacenarla en una función. Las tuberías permiten guardar tales funciones en un formato legible al iniciar una secuencia con un punto como en:

. %>% RHS

Como ejemplo, supongamos que tenemos fechas factoriales y queremos extraer el año:

library(magrittr) # needed to include the pipe operators
library(lubridate)
read_year <- . %>% as.character %>% as.Date %>% year

# Creating a dataset
df <- data.frame(now = "2015-11-11", before = "2012-01-01")
#          now     before
# 1 2015-11-11 2012-01-01

# Example 1: applying `read_year` to a single character-vector
df$now %>% read_year
# [1] 2015

# Example 2: applying `read_year` to all columns of `df`
df %>% lapply(read_year) %>% as.data.frame  # implicit `lapply(df, read_year)
#    now before
# 1 2015   2012

# Example 3: same as above using `mutate_all`
library(dplyr)
df %>% mutate_all(funs(read_year))
# if an older version of dplyr use `mutate_each`
#    now before
# 1 2015   2012

Podemos revisar la composición de la función escribiendo su nombre o usando funciones:

read_year
# Functional sequence with the following components:
# 
#  1. as.character(.)
#  2. as.Date(.)
#  3. year(.)
# 
# Use 'functions' to extract the individual functions.

También podemos acceder a cada función por su posición en la secuencia:

read_year[[2]]
# function (.) 
# as.Date(.)

Generalmente, este enfoque puede ser útil cuando la claridad es más importante que la velocidad.

Asignación con %<>% #

El paquete magrittr contiene un operador infijo de asignación compuesta, %<>%, que actualiza un valor primero canalizándolo en una o más expresiones rhs y luego asignando el resultado. Esto elimina la necesidad de escribir un nombre de objeto dos veces (una vez a cada lado del operador de asignación <-). %<>% debe ser el primer operador infijo en una cadena:

library(magrittr)
library(dplyr)

df <- mtcars

en lugar de escribir

df <- df %>% select(1:3) %>% filter(mpg > 20, cyl == 6)

o

df %>% select(1:3) %>% filter(mpg > 20, cyl == 6) -> df

El operador de asignación compuesta canalizará y reasignará df:

df %<>% select(1:3) %>% filter(mpg > 20, cyl == 6)

Exponiendo contenidos con %$% #

El operador de tubería de exposición, %$%, expone los nombres de las columnas como símbolos R dentro del objeto del lado izquierdo a la expresión del lado derecho. Este operador es útil cuando se conecta a funciones que no tienen un argumento data (a diferencia de, digamos, lm) y que no toman un data.frame y nombres de columnas como argumentos (la mayoría de las funciones principales de dplyr).

El operador de tubería de exposición %$% permite a un usuario evitar romper una tubería cuando necesita hacer referencia a los nombres de las columnas. Por ejemplo, digamos que desea filtrar un data.frame y luego ejecutar una prueba de correlación en dos columnas con cor.test:

library(magrittr)
library(dplyr)
mtcars %>%
  filter(wt > 2) %$%
  cor.test(hp, mpg)

#> 
#>  Pearson's product-moment correlation
#> 
#> data:  hp and mpg
#> t = -5.9546, df = 26, p-value = 2.768e-06
#> alternative hypothesis: true correlation is not equal to 0
#> 95 percent confidence interval:
#>  -0.8825498 -0.5393217
#> sample estimates:
#>        cor 
#> -0.7595673

Aquí, la tubería %>% estándar pasa el data.frame a filter(), mientras que la tubería %$% expone los nombres de las columnas a cor.test().

La tubería de exposición funciona como una versión compatible con tuberías de las funciones base R with(), y los mismos objetos del lado izquierdo se aceptan como entradas.

Creando efectos secundarios con %T>% #

Algunas funciones en R producen un efecto secundario (es decir, guardar, imprimir, trazar, etc.) y no siempre devuelven un valor significativo o deseado.

%T>% (operador tee) le permite reenviar un valor a una función que produce efectos secundarios mientras mantiene intacto el valor lhs original. En otras palabras: el operador tee funciona como %>%, excepto que los valores devueltos son lhs en sí mismo, y no el resultado de la función/expresión rhs.

Ejemplo: crear, canalizar, escribir y devolver un objeto. Si se usara %>% en lugar de %T>% en este ejemplo, entonces la variable all_letters contendría NULL en lugar del valor del objeto ordenado.

all_letters <- c(letters, LETTERS) %>%
    sort %T>% 
    write.csv(file = "all_letters.csv")

read.csv("all_letters.csv") %>% head()
#   x
# 1 a
# 2 A
# 3 b
# 4 B
# 5 c
# 6 C

Advertencia: canalizar un objeto sin nombre a save() producirá un objeto llamado . cuando se cargue en el espacio de trabajo con load(). Sin embargo, es posible una solución utilizando una función auxiliar (que también se puede escribir en línea como una función anónima).

all_letters <- c(letters, LETTERS) %>%
    sort %T>% 
    save(file = "all_letters.RData")

load("all_letters.RData", e <- new.env())

get("all_letters", envir = e)
# Error in get("all_letters", envir = e) : object 'all_letters' not found

get(".", envir = e)
#  [1] "a" "A" "b" "B" "c" "C" "d" "D" "e" "E" "f" "F" "g" "G" "h" "H" "i" "I" "j" "J" 
# [21] "k" "K" "l" "L" "m" "M" "n" "N" "o" "O" "p" "P" "q" "Q" "r" "R" "s" "S" "t" "T" 
# [41] "u" "U" "v" "V" "w" "W" "x" "X" "y" "Y" "z" "Z"

# Work-around
save2 <- function(. = ., name, file = stop("'file' must be specified")) {
  assign(name, .)
  call_save <- call("save", ... = name, file = file)
  eval(call_save)
}

all_letters <- c(letters, LETTERS) %>%
    sort %T>%
    save2("all_letters", "all_letters.RData")

Usando la tubería con dplyr y ggplot2 #

El operador %>% también se puede usar para canalizar la salida de dplyr a ggplot. Esto crea una canalización unificada de análisis de datos exploratorios (EDA) que se puede personalizar fácilmente. Este método es más rápido que hacer las agregaciones internamente en ggplot y tiene el beneficio adicional de evitar variables intermedias innecesarias.

library(dplyr)
library(ggplot)


diamonds %>% 
    filter(depth > 60) %>% 
    group_by(cut) %>% 
    summarize(mean_price = mean(price)) %>% 
    ggplot(aes(x = cut, y = mean_price)) + 
        geom_bar(stat = "identity")
