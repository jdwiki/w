dplyr es una iteración de plyr que proporciona funciones flexibles basadas en “verbos” para manipular datos en R. La última versión de dplyr se puede descargar desde CRAN usando

instalar.paquete("dplyr")

El objeto clave en dplyr es una tbl, una representación de una estructura de datos tabular. Actualmente, dplyr (versión 0.5.0) admite:

  • tramas de datos
  • tablas de datos -SQLite
  • PostgreSQL/Desplazamiento al rojo -MySQL/MariaDB
  • Gran consulta
  • Monet DB
  • cubos de datos con arreglos (implementación parcial)

Verbos de tabla única de dplyr #

dplyr presenta una gramática de manipulación de datos en R. Proporciona una interfaz consistente para trabajar con datos sin importar dónde estén almacenados: data.frame, [data.table] (https://www.wikiod.com/es/r/tabla-de-datos), o una base de datos. Las piezas clave de dplyr están escritas usando Rcpp, lo que lo hace muy rápido para trabajar con datos en memoria.

La filosofía de dplyr es tener funciones pequeñas que hacen una cosa bien. Las cinco funciones simples (filtrar, arreglar, seleccionar, mutar y resumir) se pueden usar para revelar nuevas formas de describir los datos. Cuando se combinan con group_by, estas funciones se pueden usar para calcular estadísticas de resumen grupales.

Puntos en común de sintaxis #

Todas estas funciones tienen una sintaxis similar:

  • El primer argumento de todas estas funciones es siempre un marco de datos
  • Las columnas se pueden referir directamente usando nombres de variables simples (es decir, sin usar $)
  • Estas funciones no modifican los datos originales en sí, es decir, no tienen efectos secundarios. Por lo tanto, los resultados siempre deben guardarse en un objeto.

Usaremos el conjunto de datos integrado mtcars para explorar los verbos de tabla única de dplyr . Antes de convertir el tipo de mtcars a tbl_df (ya que hace que la impresión sea más limpia), agregamos los rownames del conjunto de datos como una columna usando la función rownames_to_column de [tibble](https://cran.r -project.org/web/packages/tibble/index.html) paquete.

library(dplyr) # This documentation was written using version 0.5.0

mtcars_tbl <- as_data_frame(tibble::rownames_to_column(mtcars, "cars"))

# examine the structure of data

filtro #

filter ayuda a crear subconjuntos de filas que coinciden con ciertos criterios. El primer argumento es el nombre del data.frame y el segundo (y subsiguientes) argumentos son los criterios que filtran los datos (estos criterios deben evaluarse como VERDADERO o FALSO)

arreglar #

arrange se usa para ordenar los datos por una(s) variable(s) específica(s). Al igual que el verbo anterior (y todas las demás funciones en dplyr), el primer argumento es un data.frame, y los argumentos consiguientes se usan para ordenar los datos. Si se pasa más de una variable, los datos se ordenan primero por la primera variable, luego por la segunda variable y así sucesivamente.

Para ordenar los datos por caballos de fuerza - hp

arrange(mtcars_tbl, hp)

arrange(mtcars_tbl, desc(mpg), cyl)

resumir #

summarise calcula las estadísticas de resumen de las variables mediante el colapso de múltiples valores en un solo valor. Puede calcular múltiples estadísticas y podemos nombrar estas columnas de resumen en la misma declaración.

Para calcular la media y la desviación estándar de mpg y disp de todos los autos en el conjunto de datos:

summarise(mtcars_tbl, mean_mpg = mean(mpg), sd_mpg = sd(mpg), 
          mean_disp = mean(disp), sd_disp = sd(disp))

# A tibble: 1 x 4

agrupar por #

group_by se puede usar para realizar operaciones grupales en los datos. Cuando los verbos definidos anteriormente se aplican a estos datos agrupados, se aplican automáticamente a cada grupo por separado.

by_cyl <- group_by(mtcars_tbl, cyl)

Poniéndolo todo junto #

selected <- select(mtcars_tbl, cars:hp, gear)
ordered <- arrange(selected, cyl, desc(mpg))
by_cyl <- group_by(ordered, gear)
filter(by_cyl, mpg > 20, hp > 75)

Tal vez no nos interesen los resultados intermedios, podemos lograr el mismo resultado que el anterior envolviendo las llamadas a funciones:

filter(
    group_by(
        arrange(
            select(
                mtcars_tbl, cars:hp
            ), cyl, desc(mpg)

Esto puede ser un poco difícil de leer. Por lo tanto, las operaciones dplyr se pueden encadenar usando el operador pipe %>%. El código anterior se traduce en:

mtcars_tbl %>% 
    select(cars:hp) %>% 
    arrange(cyl, desc(mpg)) %>%
    group_by(cyl) %>% 
    filter(mpg > 20, hp > 75)

Agregando con el operador %>% (tubo) #

La tubería (%>%) operador podría usarse en combinación con las funciones dplyr. En este ejemplo, usamos el conjunto de datos mtcars (consulte help("mtcars") para obtener más información) para mostrar cómo resumir un marco de datos y agregar variables a los datos con el resultado de la aplicación de una función.

library(dplyr)
library(magrittr)
df <- mtcars
df$cars <- rownames(df) #just add the cars names to the df
df <- df[,c(ncol(df),1:(ncol(df)-1))] # and place the names in the first column

1. Resumir los datos

Para calcular estadísticas usamos summarize y las funciones apropiadas. En este caso, n() se usa para contar el número de casos.

 df %>%
  summarize(count=n(),mean_mpg = mean(mpg, na.rm = TRUE),
            min_weight = min(wt),max_weight = max(wt))

#  count mean_mpg min_weight max_weight
#1    32 20.09062      1.513      5.424

Es posible calcular las estadísticas por grupos de datos. En este caso por Número de cilindros y Número de marchas hacia adelante

df %>%

Ejemplos de NSE y variables de cadena en dpylr #

dplyr usa una evaluación no estándar (NSE), por lo que normalmente podemos usar los nombres de las variables sin comillas. Sin embargo, a veces, durante la canalización de datos, necesitamos obtener nuestros nombres de variables de otras fuentes, como un cuadro de selección Shiny. En el caso de funciones como select, podemos simplemente usar select_ para usar una variable de cadena para seleccionar

variable1 <- "Sepal.Length"
variable2 <- "Sepal.Width"
iris %>%
select_(variable1, variable2) %>%
head(n=5)
#  Sepal.Length Sepal.Width
#  1          5.1         3.5
#  2          4.9         3.0
#  3          4.7         3.2
#  4          4.6         3.1
#  5          5.0         3.6

Pero si queremos usar otras características como resumir o filtrar, necesitamos usar la función interp del paquete lazyeval

variable1 <- "Sepal.Length"
variable2 <- "Sepal.Width"
variable3 <- "Species"
iris %>%
select_(variable1, variable2, variable3) %>%
group_by_(variable3) %>%
summarize_(mean1 = lazyeval::interp(~mean(var), var = as.name(variable1)), mean2 = lazyeval::interp(~mean(var), var = as.name(variable2)))
#      Species mean1 mean2
#       <fctr> <dbl> <dbl>
# 1     setosa 5.006 3.428
# 2 versicolor 5.936 2.770
# 3  virginica 6.588 2.974
