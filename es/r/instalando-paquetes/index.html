Sintaxis #

  • install.packages(pkgs, lib, repos, method, destdir, dependencies, …)

Parámetros #

Parámetro Detalles
pkgs vector de caracteres de los nombres de los paquetes. Si repos = NULL, un vector de caracteres de rutas de archivos.
lib vector de caracteres que proporciona los directorios de la biblioteca donde instalar los paquetes.
repos vector de caracteres, la(s) URL(s) base(s) de los repositorios a usar, puede ser NULL para instalar desde archivos locales
método método de descarga
destdir directorio donde se almacenan los paquetes descargados
dependencias lógica que indica si también se deben instalar paquetes desinstalados de los que estos paquetes dependen/vinculan/importan/sugieren (y así sucesivamente de forma recursiva). No se usa si repos = NULL.
… Argumentos para pasar a ‘download.file’ o a las funciones para instalaciones binarias en OS X y Windows.

Descarga e instala paquetes desde repositorios #

Los paquetes son colecciones de funciones R, datos y código compilado en un [formato bien definido] (http://r-pkgs.had.co.nz/description.html). Los repositorios públicos (y privados) se utilizan para alojar colecciones de paquetes de R. La mayor colección de paquetes R está disponible en CRAN.

Usando CRAN #

Se puede instalar un paquete desde CRAN usando el siguiente código:

install.packages("dplyr")

Donde "dplyr" se denomina vector de caracteres.

Se pueden instalar más de un paquete de una sola vez usando la función de combinación c() y pasando una serie de vectores de caracteres de nombres de paquetes:

install.packages(c("dplyr", "tidyr", "ggplot2"))

En algunos casos, install.packages puede solicitar un espejo CRAN o fallar, dependiendo del valor de getOption("repos"). Para evitar esto, especifique un espejo CRAN como argumento repos:

install.packages("dplyr", repos = "https://cloud.r-project.org/")

Usando el argumento repos también es posible instalar desde otros repositorios. Para obtener información completa sobre todas las opciones disponibles, ejecute ?install.packages.

La mayoría de los paquetes requieren funciones, que se implementaron en otros paquetes (por ejemplo, el paquete data.table). Para instalar un paquete (o varios paquetes) con todos los paquetes que utiliza este paquete determinado, el argumento dependencias debe establecerse en VERDADERO):

install.packages("data.table", dependencies = TRUE)

Uso de bioconductores #

Bioconductor alberga una importante colección de paquetes relacionados con la bioinformática. Proporcionan su propia gestión de paquetes centrada en la función biocLite:

    ## Try http:// if https:// URLs are not supported
    source("https://bioconductor.org/biocLite.R")
    biocLite()

De forma predeterminada, esto instala un subconjunto de paquetes que proporcionan la funcionalidad más utilizada. Se pueden instalar paquetes específicos pasando un vector de nombres de paquetes. Por ejemplo, para instalar RImmPort desde Bioconductor:

    source("https://bioconductor.org/biocLite.R")
    biocLite("RImmPort")

Instalar paquetes desde GitHub #

Para instalar paquetes directamente desde GitHub, use el paquete devtools:

library(devtools)
install_github("authorName/repositoryName")

Para instalar ggplot2 desde github:

devtools::install_github("tidyverse/ggplot2")

El comando anterior instalará la versión de ggplot2 que corresponde a la rama maestra. Para instalar desde una rama diferente de un repositorio, use el argumento ref para proporcionar el nombre de la rama. Por ejemplo, el siguiente comando instalará la rama dev_general del paquete googleway.

devtools::install_github("SymbolixAU/googleway", ref = "dev_general")

Otra opción es usar el paquete ghit. Proporciona una alternativa ligera para instalar paquetes desde github:

install.packages("ghit")
ghit::install_github("google/CausalImpact")

Para instalar un paquete que está en un repositorio privado en Github, genere un token de acceso personal en http://www.github.com/settings/tokens/ (Consulte ?install_github para obtener documentación sobre el mismo). Sigue estos pasos:

  1. instalar.paquetes(c(“curl”, “httr”))

  2. configuración = httr::config(ssl_verifypeer = FALSO)

  3. instalar.paquetes(“RCurl”) options(RCurlOptions = c(getOption(“RCurlOptions”),ssl.verifypeer = FALSE, ssl.verifyhost = FALSE ) )

  4. getOption(“RCurlOptions”)

    You should see the following:

    ssl.verifypeer ssl.verifyhost 

FALSE          FALSE

  5. biblioteca (httr) set_config(config(ssl_verifypeer = 0L))

Esto evita el error común: “El certificado de pares no se puede autenticar con los certificados de CA dados”

  1. Finalmente, use el siguiente comando para instalar su paquete sin problemas

    install_github("username/package_name",auth_token="abc")

Alternativamente, configure una variable de entorno GITHUB_PAT, usando

Sys.setenv(GITHUB_PAT = "access_token")
devtools::install_github("organisation/package_name")

El PAT generado en Github solo es visible una vez, es decir, cuando se crea inicialmente, por lo que es prudente guardar ese token en .Rprofile. Esto también es útil si la organización tiene muchos repositorios privados.

Instalar el paquete desde la fuente local #

Para instalar el paquete desde el archivo fuente local:

install.packages(path_to_source, repos = NULL, type="source")

install.packages("~/Downloads/dplyr-master.zip", repos=NULL, type="source")

Aquí, path_to_source es la ruta absoluta del archivo fuente local.

Otro comando que abre una ventana para elegir los archivos fuente zip o tar.gz descargados es:

install.packages(file.choose(), repos=NULL)

Otra forma posible es usar el RStudio basado en GUI:

Paso 1: Ve a Herramientas.

Paso 2: Vaya a Instalar paquetes.

Paso 3: En Instalar desde configúrelo como Archivo de archivo de paquete (.zip; .tar.gz)

Paso 4: Luego Explore busque el archivo de su paquete (digamos crayon_1.3.1.zip) y *después de un tiempo (después de que muestre la Ruta del paquete y el nombre del archivo en la pestaña Archivo del paquete) *

Otra forma de instalar el paquete R desde una fuente local es usando la función install_local() del paquete devtools.

library(devtools)
install_local("~/Downloads/dplyr-master.zip")

Uso de un administrador de paquetes CLI: uso básico de pacman #

pacman es un administrador de paquetes simple para R.

pacman permite que un usuario cargue de forma compacta todos los paquetes deseados, instalando cualquiera que falte (y sus dependencias), con un solo comando, p_load. pacman no requiere que el usuario escriba comillas alrededor del nombre del paquete. El uso básico es el siguiente:

p_load(data.table, dplyr, ggplot2)

El único paquete que requiere una declaración library, require o install.packages con este enfoque es el mismo pacman:

library(pacman)
p_load(data.table, dplyr, ggplot2)

o, igualmente válido:

pacman::p_load(data.table, dplyr, ggplot2)

Además de ahorrar tiempo al requerir menos código para administrar los paquetes, pacman también facilita la construcción de código reproducible al instalar los paquetes necesarios si y solo si aún no están instalados.

Dado que es posible que no esté seguro de si pacman está instalado en la biblioteca de un usuario que usará su código (o usted mismo en usos futuros de su propio código), una mejor práctica es incluir una declaración condicional para instalar pacman si aún no está cargado:

if(!(require(pacman)) install.packages("pacman")
pacman::p_load(data.table, dplyr, ggplot2)

Instalar la versión de desarrollo local de un paquete #

Mientras se trabaja en el desarrollo de un paquete R, a menudo es necesario instalar la última versión del paquete. Esto se puede lograr creando primero una distribución fuente del paquete (en la línea de comando)

R CMD build my_package

y luego [instalarlo en R][1]. Cualquier sesión de R en ejecución con una versión anterior del paquete cargado deberá volver a cargarlo.

unloadNamespace("my_package")
library(my_package)

Un enfoque más conveniente utiliza el paquete devtools para simplificar el proceso. En una sesión R con el directorio de trabajo establecido en el directorio del paquete

devtools::install()

construirá, instalará y recargará el paquete.

[1]: https://www.wikiod.com/es/r/instalando-paquetes#Instalar el paquete desde la fuente local
