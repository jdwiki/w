El lenguaje R se usa comúnmente para el análisis estadístico. Como tal, contiene un sólido conjunto de opciones para la aleatorización. Para obtener información específica sobre el muestreo de distribuciones de probabilidad, consulte la documentación de funciones de distribución.

Los usuarios que vienen de otros lenguajes de programación pueden sentirse confundidos por la falta de una función rand equivalente a lo que pueden haber experimentado antes. La generación básica de números aleatorios se realiza utilizando la familia de funciones r* para cada distribución (consulte el enlace anterior). Los números aleatorios extraídos uniformemente de un rango se pueden generar usando runif, para “uniforme aleatorio”. Dado que esto también se parece sospechosamente a “ejecutar si”, a menudo es difícil de entender para los nuevos usuarios de R.

Sorteos aleatorios y permutaciones #

El comando sample se puede usar para simular problemas de probabilidad clásicos, como sacar de una urna con y sin reemplazo, o crear permutaciones aleatorias.

Tenga en cuenta que a lo largo de este ejemplo, set.seed se usa para garantizar que el código de ejemplo sea reproducible. Sin embargo, sample funcionará sin llamar explícitamente a set.seed.

Permutación aleatoria #

En la forma más simple, sample crea una permutación aleatoria de un vector de números enteros. Esto se puede lograr con:

set.seed(1251)
sample(x = 10)

[1]  7  1  4  8  6  3 10  5  2  9

Cuando no se le dan otros argumentos, sample devuelve una permutación aleatoria del vector de 1 a x. Esto puede ser útil cuando se intenta aleatorizar el orden de las filas en un marco de datos. Esta es una tarea común cuando se crean tablas de aleatorización para ensayos o cuando se selecciona un subconjunto aleatorio de filas para el análisis.

library(datasets)
set.seed(1171)
iris_rand <- iris[sample(x = 1:nrow(iris)),]

> head(iris)
  Sepal.Length Sepal.Width Petal.Length Petal.Width Species
1          5.1         3.5          1.4         0.2  setosa
2          4.9         3.0          1.4         0.2  setosa
3          4.7         3.2          1.3         0.2  setosa
4          4.6         3.1          1.5         0.2  setosa
5          5.0         3.6          1.4         0.2  setosa
6          5.4         3.9          1.7         0.4  setosa

> head(iris_rand)

Sorteos sin Reemplazo #

Usando sample, también podemos simular el dibujo de un conjunto con y sin reemplazo. Para obtener una muestra sin reemplazo (el valor predeterminado), debe proporcionar una muestra con un conjunto del que extraer y el número de extracciones. El conjunto del que se va a extraer se da como un vector.

set.seed(7043)
sample(x = LETTERS,size = 7)

[1] "S" "P" "J" "F" "Z" "G" "R"

Tenga en cuenta que si el argumento de tamaño tiene la misma longitud que el argumento de x, está creando una permutación aleatoria. También tenga en cuenta que no puede especificar un tamaño mayor que la longitud de x al realizar un muestreo sin reemplazo.

set.seed(7305)
sample(x = letters,size = 26)

[1] "x" "z" "y" "i" "k" "f" "d" "s" "g" "v" "j" "o" "e" "c" "m" "n" "h" "u" "a" "b" "l" "r" "w" "t" "q" "p"

sample(x = letters,size = 30)
Error in sample.int(length(x), size, replace, prob) : 
  cannot take a sample larger than the population when 'replace = FALSE'

Esto nos lleva al dibujo con reemplazo.

Sorteos con reemplazo #

Para hacer sorteos aleatorios de un conjunto con reemplazo, use el argumento replace para sample. Por defecto, reemplazar es FALSO. Establecerlo en “VERDADERO” significa que cada elemento del conjunto del que se extrae puede aparecer más de una vez en el resultado final.

set.seed(5062)
sample(x = c("A","B","C","D"),size = 8,replace = TRUE)

[1] "D" "C" "D" "B" "A" "A" "A" "A"

Cambiar las probabilidades de sorteo #

De forma predeterminada, cuando usa sample, asume que la probabilidad de elegir cada elemento es la misma. Considérelo como un problema básico de “urna”. El siguiente código es equivalente a sacar una canica de color de una urna 20 veces, escribir el color y luego volver a colocar la canica en la urna. La urna contiene una canica roja, una azul y una verde, lo que significa que la probabilidad de sacar cada color es 1/3.

set.seed(6472)
sample(x = c("Red","Blue","Green"),
       size = 20,
       replace = TRUE)

Supongamos que, en cambio, quisiéramos realizar la misma tarea, pero nuestra urna contiene 2 canicas rojas, 1 canica azul y 1 canica verde. Una opción sería cambiar el argumento que enviamos a ‘x’ para agregar un ‘Rojo’ adicional. Sin embargo, una mejor opción es usar el argumento prob para sample.

El argumento prob acepta un vector con la probabilidad de dibujar cada elemento. En nuestro ejemplo anterior, la probabilidad de sacar una canica roja sería 1/2, mientras que la probabilidad de sacar una canica azul o verde sería 1/4.

set.seed(28432)
sample(x = c("Red","Blue","Green"),
       size = 20,
       replace = TRUE,
       prob = c(0.50,0.25,0.25))

Contrariamente a la intuición, el argumento dado a prob no necesita sumar 1. R siempre transformará los argumentos dados en probabilidades que suman 1. Por ejemplo, considere nuestro ejemplo anterior de 2 Rojo, 1 Azul y 1 Verde . Puede lograr los mismos resultados que nuestro código anterior usando esos números:

set.seed(28432)
frac_prob_example <- sample(x = c("Red","Blue","Green"),
                            size = 200,
                            replace = TRUE,
                            prob = c(0.50,0.25,0.25))

set.seed(28432)
numeric_prob_example <- sample(x = c("Red","Blue","Green"),
                               size = 200,
                               replace = TRUE,
                               prob = c(2,1,1))

> identical(frac_prob_example,numeric_prob_example)
[1] TRUE

La principal restricción es que no puede establecer que todas las probabilidades sean cero, y ninguna de ellas puede ser menor que cero.

También puede utilizar prob cuando replace está establecido en FALSE. En esa situación, después de dibujar cada elemento, las proporciones de los valores prob para los elementos restantes dan la probabilidad para el siguiente sorteo. En esta situación, debe tener suficientes probabilidades distintas de cero para alcanzar el “tamaño” de la muestra que está extrayendo. Por ejemplo:

set.seed(21741)
sample(x = c("Red","Blue","Green"),
       size = 2,
       replace = FALSE,
       prob = c(0.8,0.19,0.01))

En este ejemplo, Red se dibuja en el primer sorteo (como el primer elemento). Había un 80 % de posibilidades de que saliera rojo, un 19 % de que saliera azul y un 1 % de que saliera verde.

Para el próximo sorteo, Red ya no está en la urna. El total de las probabilidades entre los elementos restantes es del 20% (19% para Azul y 1% para Verde). Para ese sorteo, hay un 95 % de probabilidad de que el artículo sea azul (19/20) y un 5 % de probabilidad de que sea verde (1/20).

Establecer la semilla #

La función set.seed se utiliza para establecer la semilla aleatoria para todas las funciones de aleatorización. Si está utilizando R para crear una aleatorización que desea poder reproducir, primero debe usar set.seed.

set.seed(1643)
samp1 <- sample(x = 1:5,size = 200,replace = TRUE)

set.seed(1643)
samp2 <- sample(x = 1:5,size = 200,replace = TRUE)

> identical(x = samp1,y = samp2)
[1] TRUE

Tenga en cuenta que el procesamiento paralelo requiere un tratamiento especial de la semilla aleatoria, que se describe con más detalle en otra parte.
