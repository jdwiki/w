Sintaxis #

  • lm(fórmula, datos, subconjunto, pesos, na.acción, método = “qr”, modelo = VERDADERO, x = FALSO, y = FALSO, qr = VERDADERO, singular.ok = VERDADERO, contrastes = NULO, compensación, . ..)

Parámetros #

Parámetro significado
fórmula una fórmula en notación Wilkinson-Rogers; respuesta ~ ... donde ... contiene términos correspondientes a variables en el entorno o en el marco de datos especificado por el argumento data
datos marco de datos que contiene las variables de respuesta y predictoras
subconjunto un vector que especifica un subconjunto de observaciones a utilizar: puede expresarse como una declaración lógica en términos de las variables en datos
pesas pesos analíticos (consulte la sección Pesos más arriba)
na.acción cómo manejar los valores faltantes (NA): consulte ?na.action
método cómo realizar el montaje. Las únicas opciones son "qr" o "model.frame" (esta última devuelve el marco del modelo sin ajustar el modelo, idéntico a especificar model=TRUE)
modelo si almacenar el marco del modelo en el objeto ajustado
x si almacenar la matriz del modelo en el objeto ajustado
y si almacenar la respuesta del modelo en el objeto ajustado
qr si almacenar la descomposición QR en el objeto ajustado
singular.ok si permitir ajustes singulares, modelos con predictores colineales (un subconjunto de los coeficientes se establecerá automáticamente en NA en este caso
contrastes una lista de contrastes que se usarán para factores particulares en el modelo; ver el argumento contrasts.arg de ?model.matrix.default. Los contrastes también se pueden establecer con options() (ver el argumento contrasts) o asignando los atributos contrast de un factor (ver ?contrasts)
compensación se utiliza para especificar un componente conocido a priori en el modelo. También se puede especificar como parte de la fórmula. Ver ?model.offset
… argumentos adicionales para pasar a funciones de ajuste de nivel inferior (lm.fit() o lm.wfit())

Regresión lineal en el conjunto de datos mtcars #

El [marco de datos] mtcars incorporado (https://www.wikiod.com/es/r/marcos-de-datos) contiene información sobre 32 automóviles, incluido su peso, eficiencia de combustible (en millas por galón), velocidad, etc. (Para obtener más información sobre el conjunto de datos, utilice help(mtcars)).

Si estamos interesados ​​en la relación entre la eficiencia de combustible (mpg) y el peso (wt), podemos comenzar a trazar esas variables con:

plot(mpg ~ wt, data = mtcars, col=2)

¡Las gráficas muestran una relación (lineal)!. Entonces, si queremos realizar una regresión lineal para determinar los coeficientes de un modelo lineal, usaríamos la función lm:

fit <- lm(mpg ~ wt, data = mtcars)

El ~ aquí significa “explicado por”, por lo que la fórmula mpg ~ wt significa que estamos prediciendo mpg según lo explicado por wt. La forma más útil de ver la salida es con:

summary(fit)

Lo que da la salida:

Podríamos agregar una línea a nuestra primera gráfica para mostrar el ‘mpg’ predicho:

abline(fit,col=3,lwd=2)

También es posible agregar la ecuación a esa gráfica. Primero, obtenga los coeficientes con coef. Luego, usando paste0 colapsamos los coeficientes con las variables apropiadas y +/-, para construir la ecuación. Finalmente, lo agregamos a la trama usando mtext:

bs <- round(coef(fit), 3) 
lmlab <- paste0("mpg = ", bs[1],
             ifelse(sign(bs[2])==1, " + ", " - "), abs(bs[2]), " wt ")
mtext(lmlab, 3, line=-2)

Ponderación #

A veces queremos que el modelo dé más peso a algunos puntos de datos o ejemplos que a otros. Esto es posible especificando el peso de los datos de entrada mientras aprende el modelo. En general, hay dos tipos de escenarios en los que podríamos usar pesos no uniformes sobre los ejemplos:

  • Pesos analíticos: reflejan los diferentes niveles de precisión de las diferentes observaciones. Por ejemplo, si analiza datos en los que cada observación es el resultado promedio de un área geográfica, el peso analítico es proporcional a la inversa de la varianza estimada. Útil cuando se trata de promedios en datos al proporcionar un peso proporcional dada la cantidad de observaciones. Fuente

  • Pesos de muestreo (Pesos de probabilidad inversa - IPW): una técnica estadística para calcular estadísticas estandarizadas a una población diferente de aquella en la que se recolectaron los datos. Los diseños de estudio con una población de muestreo dispar y una población de inferencia objetivo (población objetivo) son comunes en la aplicación. Útil cuando se trata de datos a los que les faltan valores. Fuente

La función lm() realiza una ponderación analítica. Para ponderaciones de muestreo, el paquete survey se utiliza para crear un objeto de diseño de encuesta y ejecutar svyglm(). De forma predeterminada, el paquete encuesta utiliza ponderaciones de muestreo. (NOTA: lm() y svyglm() con la familia gaussian() producirán las mismas estimaciones puntuales, porque ambos resuelven los coeficientes minimizando los mínimos cuadrados ponderados. Se diferencian en cómo los errores estándar se calculan).

Pesos de muestreo (IPW)

library(survey)
data$X <- 1:nrow(data)             # Create unique id
        
# Build survey design object with unique id, ipw, and data.frame
des1 <- svydesign(id = ~X,  weights = ~weight, data = data)
        
# Run glm with survey design object
prog.lm <- svyglm(lexptot ~ progvillm + sexhead + agehead, design=des1)

Usando la función ‘predecir’ #

Una vez que se construye un modelo, “predecir” es la función principal para probar con nuevos datos. Nuestro ejemplo utilizará el conjunto de datos integrado mtcars para hacer una regresión de millas por galón contra el desplazamiento:

my_mdl <- lm(mpg ~ disp, data=mtcars)
my_mdl

Call:
lm(formula = mpg ~ disp, data = mtcars)

Coefficients:
(Intercept)         disp  
   29.59985     -0.04122

Si tuviera una nueva fuente de datos con desplazamiento, podría ver las millas por galón estimadas.

set.seed(1234)
newdata <- sample(mtcars$disp, 5)
newdata
[1] 258.0  71.1  75.7 145.0 400.0

newdf <- data.frame(disp=newdata)
predict(my_mdl, newdf)
       1        2        3        4        5 
18.96635 26.66946 26.47987 23.62366 13.11381

La parte más importante del proceso es crear un nuevo marco de datos con los mismos nombres de columna que los datos originales. En este caso, los datos originales tenían una columna etiquetada disp, estaba seguro de llamar a los nuevos datos con el mismo nombre.

Precaución

Veamos algunos errores comunes:

  1. no usar un data.frame en el nuevo objeto:

    predict(my_mdl, newdata)
Error in eval(predvars, data, env) : 
   numeric 'envir' arg not of length one

  2. no usar los mismos nombres en el nuevo marco de datos:

Comprobación de no linealidad con regresión polinomial #

A veces, cuando trabajamos con regresión lineal, necesitamos verificar la no linealidad en los datos. Una forma de hacer esto es ajustar un modelo polinomial y comprobar si se ajusta mejor a los datos que un modelo lineal. Hay otras razones, como las teóricas, que indican ajustar un modelo cuadrático o de orden superior porque se cree que la relación de variables es inherentemente de naturaleza polinomial.

Ajustemos un modelo cuadrático para el conjunto de datos mtcars. Para un modelo lineal, consulte https://www.wikiod.com/es/r/modelos-lineales-regresion#Regresión lineal en el conjunto de datos mtcars

Primero hacemos un gráfico de dispersión de las variables mpg (Millas/galón), disp (Desplazamiento (cu.in.)) y wt (Peso (1000 lbs)). La relación entre mpg y disp parece no lineal.

plot(mtcars[,c("mpg","disp","wt")])

Un ajuste lineal mostrará que disp no es significativo.

fit0 = lm(mpg ~ wt+disp, mtcars)
summary(fit0)

# Coefficients:
#            Estimate Std. Error t value Pr(>|t|)    
#(Intercept) 34.96055    2.16454  16.151 4.91e-16 ***
#wt          -3.35082    1.16413  -2.878  0.00743 ** 
#disp        -0.01773    0.00919  -1.929  0.06362 .  
#---
#Signif. codes:  0 ‘***’ 0.001 ‘**’ 0.01 ‘*’ 0.05 ‘.’ 0.1 ‘ ’ 1
#Residual standard error: 2.917 on 29 degrees of freedom
#Multiple R-squared:  0.7809,    Adjusted R-squared:  0.7658

Luego, para obtener el resultado de un modelo cuadrático, agregamos I(disp^2). El nuevo modelo parece mejor cuando se observa R^2 y todas las variables son significativas.

fit1 = lm(mpg ~ wt+disp+I(disp^2), mtcars)
summary(fit1)

# Coefficients:
#              Estimate Std. Error t value Pr(>|t|)    
#(Intercept) 41.4019837  2.4266906  17.061  2.5e-16 ***
#wt          -3.4179165  0.9545642  -3.581 0.001278 ** 
#disp        -0.0823950  0.0182460  -4.516 0.000104 ***
#I(disp^2)    0.0001277  0.0000328   3.892 0.000561 ***
#---
#Signif. codes:  0 ‘***’ 0.001 ‘**’ 0.01 ‘*’ 0.05 ‘.’ 0.1 ‘ ’ 1
#Residual standard error: 2.391 on 28 degrees of freedom
#Multiple R-squared:  0.8578,    Adjusted R-squared:  0.8426

Como tenemos tres variables, el modelo ajustado es una superficie representada por:

mpg = 41,4020-3,4179*wt-0,0824*disp+0,0001277*disp^2

Otra forma de especificar la regresión polinomial es usar poly con el parámetro raw=TRUE; de lo contrario, se considerarán polinomios ortogonales (consulte help(ploy) para obtener más información). Obtenemos el mismo resultado usando:

summary(lm(mpg ~ wt+poly(disp, 2, raw=TRUE),mtcars))

Finalmente, ¿qué pasa si necesitamos mostrar un gráfico de la superficie estimada? Bueno, hay muchas opciones para hacer gráficos ‘3D’ en ‘R’. Aquí usamos Fit3d del paquete p3d.

library(p3d)
Init3d(family="serif", cex = 1)
Plot3d(mpg ~ disp+wt, mtcars)
Axes3d()
Fit3d(fit1)

Trazando la regresión (base) #

Continuando con el ejemplo mtcars, aquí hay una forma simple de producir un gráfico de su regresión lineal que es potencialmente adecuado para su publicación.

Primero ajuste el modelo lineal y

fit <- lm(mpg ~ wt, data = mtcars)

Luego grafique las dos variables de interés y agregue la línea de regresión dentro del dominio de definición:

plot(mtcars$wt,mtcars$mpg,pch=18, xlab = 'wt',ylab = 'mpg')
lines(c(min(mtcars$wt),max(mtcars$wt)),
as.numeric(predict(fit, data.frame(wt=c(min(mtcars$wt),max(mtcars$wt))))))

¡Casi ahí! El último paso es agregar a la gráfica, la ecuación de regresión, el r cuadrado y el coeficiente de correlación. Esto se hace usando la función vector:

rp = vector('expression',3)
rp[1] = substitute(expression(italic(y) == MYOTHERVALUE3 + MYOTHERVALUE4 %*% x), 
          list(MYOTHERVALUE3 = format(fit$coefficients[1], digits = 2),
                        MYOTHERVALUE4 = format(fit$coefficients[2], digits = 2)))[2]
rp[2] = substitute(expression(italic(R)^2 == MYVALUE), 
             list(MYVALUE = format(summary(fit)$adj.r.squared,dig=3)))[2]
rp[3] = substitute(expression(Pearson-R == MYOTHERVALUE2), 
             list(MYOTHERVALUE2 = format(cor(mtcars$wt,mtcars$mpg), digits = 2)))[2]

legend("topright", legend = rp, bty = 'n')

Tenga en cuenta que puede agregar cualquier otro parámetro, como el RMSE, adaptando la función de vector. Imagina que quieres una leyenda con 10 elementos. La definición del vector sería la siguiente:

rp = vector('expression',10)

y necesitarás definir r[1]…. a r[10]

Evaluación de la calidad #

Después de construir un modelo de regresión, es importante verificar el resultado y decidir si el modelo es apropiado y funciona bien con los datos disponibles. Esto se puede hacer examinando la gráfica de residuos así como otras gráficas de diagnóstico.

# fit the model
fit <- lm(mpg ~ wt, data = mtcars)
# 
par(mfrow=c(2,1))
# plot model object
plot(fit, which =1:2)

Estos gráficos verifican dos suposiciones que se hicieron al construir el modelo:

  1. Que el valor esperado de la variable pronosticada (en este caso mpg) viene dado por una combinación lineal de los predictores (en este caso wt). Esperamos que esta estimación sea imparcial. Por lo tanto, los residuos deben centrarse en torno a la media de todos los valores de los predictores. En este caso vemos que los residuos tienden a ser positivos en los extremos y negativos en el medio, sugiriendo una relación no lineal entre las variables.
  2. Que la variable predicha real se distribuya normalmente alrededor de su estimación. Por lo tanto, los residuos deben estar normalmente distribuidos. Para datos distribuidos normalmente, los puntos en un gráfico Q-Q normal deben estar sobre o cerca de la diagonal. Hay cierta cantidad de sesgo en los extremos aquí.
