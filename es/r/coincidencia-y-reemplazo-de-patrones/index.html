Este tema cubre los patrones de cadena coincidentes, así como su extracción o reemplazo. Para obtener detalles sobre la definición de patrones complicados, consulte Expresiones regulares.

Sintaxis #

  • grep(“consulta”, “asunto”, argumentos_opcionales)

  • grepl(“consulta”, “asunto”, argumentos_opcionales)

  • gsub("(grupo1)(grupo2)", “\\grupo#”, “sujeto”)

Diferencias con otros idiomas #

Los símbolos regex escapados (como \1) se deben escapar una segunda vez (como \\1), no solo en el argumento patrón, sino también en el reemplazo a sub y gsub.

De forma predeterminada, el patrón para todos los comandos (grep, sub, regexpr) no es una expresión regular compatible con Perl (PCRE), por lo que no se admiten algunas cosas como las búsquedas. Sin embargo, cada función acepta un argumento perl=TRUE para habilitarlas. Consulte el tema de expresiones regulares de R para obtener más detalles.

Paquetes especializados #

  • cadena
  • cuerda

Encontrar coincidencias #

# example data
test_sentences <- c("The quick brown fox", "jumps over the lazy dog")

¿Hay una coincidencia? #

grepl() se usa para verificar si existe una palabra o expresión regular en una cadena o vector de caracteres. La función devuelve un vector VERDADERO/FALSO (o “booleano”).

Tenga en cuenta que podemos verificar en cada cadena la palabra “zorro” y recibir un vector booleano a cambio.

grepl("fox", test_sentences)
#[1]  TRUE FALSE

Emparejar ubicaciones #

grep toma una cadena de caracteres y una expresión regular. Devuelve un vector numérico de índices. Esto devolverá qué frase contiene la palabra “zorro”.

grep("fox", test_sentences)
#[1] 1

valores coincidentes #

Para seleccionar oraciones que coincidan con un patrón:

# each of the following lines does the job:
test_sentences[grep("fox", test_sentences)]
test_sentences[grepl("fox", test_sentences)]
grep("fox", test_sentences, value = TRUE)
# [1] "The quick brown fox"

Detalles #

Dado que el patrón "fox" es solo una palabra, en lugar de una expresión regular, podríamos mejorar el rendimiento (ya sea con grep o grepl) especificando fixed = TRUE.

grep("fox", test_sentences, fixed = TRUE)
#[1] 1

Para seleccionar oraciones que no coincidan con un patrón, se puede usar grep con invert = TRUE; o siga las reglas de subconjunto con -grep(...) o !grepl(...).

Tanto en grepl(patrón, x) como en grep(patrón, x), el parámetro x está vectorizado, el parámetro patrón no lo está. Como resultado, no puede usarlos directamente para comparar patrón[1] con x[1], patrón[2] con x[2], y así sucesivamente.

Resumen de partidos #

Después de realizar p. el comando grepl, tal vez desee obtener una descripción general de cuántas coincidencias fueron VERDADERO o FALSO. Esto es útil, p. en el caso de grandes conjuntos de datos. Para hacerlo, ejecute el comando resumen:

# example data
test_sentences <- c("The quick brown fox", "jumps over the lazy dog") 

# find matches
matches <- grepl("fox", test_sentences)

# overview
summary(matches)

Coincidencia única y global. #

Cuando se trabaja con expresiones regulares, un modificador para PCRE es g para coincidencia global.

En R, las funciones de coincidencia y reemplazo tienen dos versiones: primera coincidencia y coincidencia global:

  • sub(patrón,reemplazo,texto) reemplazará la primera aparición de patrón por reemplazo en el texto

  • gsub(patrón,reemplazo,texto) hará lo mismo que sub pero para cada aparición de patrón

  • regexpr(patrón,texto) devolverá la posición de coincidencia para la primera instancia de patrón

  • gregexpr(patrón,texto) devolverá todas las coincidencias.

Algunos datos aleatorios:

set.seed(123)
teststring <- paste0(sample(letters,20),collapse="")

# teststring
#[1] "htjuwakqxzpgrsbncvyo"

Veamos cómo funciona esto si queremos reemplazar las vocales por otra cosa:

sub("[aeiouy]"," ** HERE WAS A VOWEL** ",teststring)
#[1] "htj ** HERE WAS A VOWEL** wakqxzpgrsbncvyo"

gsub("[aeiouy]"," ** HERE WAS A VOWEL** ",teststring)
#[1] "htj ** HERE WAS A VOWEL** w ** HERE WAS A VOWEL** kqxzpgrsbncv ** HERE WAS A VOWEL**  ** HERE WAS A VOWEL** "

Ahora veamos cómo podemos encontrar una consonante seguida inmediatamente por una o más vocales:

regexpr("[^aeiou][aeiou]+",teststring)
#[1] 3
#attr(,"match.length")
#[1] 2
#attr(,"useBytes")
#[1] TRUE

Tenemos una coincidencia en la posición 3 de la cadena de longitud 2, es decir: ju

Ahora, si queremos obtener todas las coincidencias:

gregexpr("[^aeiou][aeiou]+",teststring)
#[[1]]
#[1]  3  5 19
#attr(,"match.length")
#[1] 2 2 2
#attr(,"useBytes")
#[1] TRUE

Todo esto es realmente genial, pero solo da posiciones de uso de coincidencia y no es tan fácil obtener lo que coincide, y aquí viene regmatches, su único propósito es extraer la cadena coincidente de regexpr, pero tiene una sintaxis diferente.

Guardemos nuestras coincidencias en una variable y luego extráigalas de la cadena original:

matches <- gregexpr("[^aeiou][aeiou]+",teststring)
regmatches(teststring,matches)
#[[1]]
#[1] "ju" "wa" "yo"

Esto puede sonar extraño para no tener un atajo, pero esto permite la extracción de otra cadena por las coincidencias de nuestra primera (piense en comparar dos vectores largos donde sabe que hay un patrón común para el primero pero no para el segundo, esto permite un comparación fácil):

teststring2 <- "this is another string to match against"
regmatches(teststring2,matches)
#[[1]]
#[1] "is" " i" "ri"

Nota de atención: por defecto, el patrón no es una expresión regular compatible con Perl, algunas cosas como las búsquedas no son compatibles, pero cada función presentada aquí permite el argumento perl=TRUE para habilitarlas.

Haciendo sustituciones #

# example data
test_sentences <- c("The quick brown fox quickly", "jumps over the lazy dog")

Hagamos que el zorro marrón sea rojo:

sub("brown","red", test_sentences)
#[1] "The quick red fox quickly"       "jumps over the lazy dog"

Ahora, hagamos que el zorro "rápido" actúe "rápido". Esto no lo hará:

sub("quick", "fast", test_sentences)
#[1] "The fast red fox quickly"       "jumps over the lazy dog"

sub solo hace el primer reemplazo disponible, necesitamos gsub para reemplazo global:

gsub("quick", "fast", test_sentences)
#[1] "The fast red fox fastly"       "jumps over the lazy dog"

Consulte https://www.wikiod.com/es/r/modificacion-de-cadenas-por-sustitucion para obtener más ejemplos.

Encuentra coincidencias en grandes conjuntos de datos #

En el caso de grandes conjuntos de datos, la llamada de grepl("fox", test_sentences) no funciona bien. Los grandes conjuntos de datos son, p. sitios web rastreados o millones de Tweets, etc.

La primera aceleración es el uso de la opción perl = TRUE. Aún más rápida es la opción fixed = TRUE. Un ejemplo completo sería:

# example data
test_sentences <- c("The quick brown fox", "jumps over the lazy dog")

grepl("fox", test_sentences, perl = TRUE)
#[1]  TRUE FALSE

En el caso de la minería de textos, a menudo se utiliza un corpus. Un corpus no se puede usar directamente con grepl. Por lo tanto, considere esta función:

searchCorpus <- function(corpus, pattern) {
  return(tm_index(corpus, FUN = function(x) {
    grepl(pattern, x, ignore.case = TRUE, perl = TRUE)
  }))
}
