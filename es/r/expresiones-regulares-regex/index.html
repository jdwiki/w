Las expresiones regulares (también llamadas “regex” o “regexp”) definen patrones que se pueden coincidir con una cadena. Escriba ?regex para la documentación oficial de R y consulte Regex Docs para obtener más detalles. El ‘gotcha’ más importante que no se aprenderá en SO regex/topics es que la mayoría de las funciones de R-regex necesitan el uso de barras invertidas emparejadas para escapar en un parámetro ‘patrón’.

Clases de personajes #

  • "[AB]" podría ser A o B
  • "[[:alfa:]]" podría ser cualquier letra
  • "[[:lower:]]" representa cualquier letra minúscula. Tenga en cuenta que "[a-z]" está cerca pero no coincide, por ejemplo, ú.
  • "[[:upper:]]" representa cualquier letra mayúscula. Tenga en cuenta que "[A-Z]" está cerca pero no coincide, por ejemplo, Ú.
  • "[[:digit:]]" representa cualquier dígito: 0, 1, 2, …, o 9 y es equivalente a "[0-9]".

Cuantificadores #

+, * y ? se aplican como de costumbre en expresiones regulares. – + coincide al menos una vez, * coincide 0 o más veces y ? coincide 0 o 1 vez.

Indicadores de inicio y fin de línea #

Puede especificar la posición de la expresión regular en la cadena:

  • "^..." obliga a que la expresión regular esté al principio de la cadena
  • "...$" obliga a que la expresión regular esté al final de la cadena

Diferencias con otros idiomas #

Tenga en cuenta que las expresiones regulares en R a menudo se ven muy ligeramente diferentes de las expresiones regulares utilizadas en otros idiomas.

  • R requiere escapes de doble barra invertida (porque "\" ya implica escape en general en las cadenas R), por lo que, por ejemplo, para capturar espacios en blanco en la mayoría de los motores de expresiones regulares, simplemente se necesita escribir \s, frente a. \\s en R.

  • Los caracteres UTF-8 en R deben escaparse con una U mayúscula, p. [\U{1F600}] y [\U1F600] coinciden con 😀, mientras que, por ejemplo, en Ruby, esto coincidiría con una u minúscula.

El siguiente sitio reg101 es un buen lugar para verificar la expresión regular en línea antes de usarla R-script.

El wikibook de Programación R tiene una página dedicada al procesamiento de texto con muchos ejemplos usando expresiones regulares.

Diferencias entre Perl y expresiones regulares POSIX #

Hay dos motores ligeramente diferentes de expresiones regulares implementados en R. El valor predeterminado se denomina compatible con POSIX; todas las funciones de expresiones regulares en R también están equipadas con una opción para activar el último tipo: perl = TRUE.

Mirar hacia adelante/mirar hacia atrás #

perl = TRUE permite mirar hacia adelante y mirar hacia atrás en expresiones regulares.

  • "(?<=A)B" coincide con la apariencia de la letra B solo si está precedida por A, es decir, "ABACADABRA" coincidiría, pero "abacadabra" y "aBacadabra " no lo haría.

Validar una fecha en formato “AAAAMMDD” #

Es una práctica común nombrar archivos usando la fecha como prefijo en el siguiente formato: AAAAMMDD, por ejemplo: 20170101_resultados.csv. Una fecha en ese formato de cadena se puede verificar usando la siguiente expresión regular:

\\d{4}(0[1-9]|1[012])(0[1-9]|[12][0-9]|3[01])

La expresión anterior considera fechas desde el año: 0000-9999, meses entre: 01-12 y días 01-31.

Por ejemplo:

> grepl("\\d{4}(0[1-9]|1[012])(0[1-9]|[12][0-9]|3[01])", "20170101")
[1] TRUE
> grepl("\\d{4}(0[1-9]|1[012])(0[1-9]|[12][0-9]|3[01])", "20171206")
[1] TRUE
> grepl("\\d{4}(0[1-9]|1[012])(0[1-9]|[12][0-9]|3[01])", "29991231")
[1] TRUE

Nota: Valida la sintaxis de la fecha, pero podemos tener una fecha equivocada con una sintaxis válida, por ejemplo: 20170229 (2017 no es un año bisiesto).

> grepl("\\d{4}(0[1-9]|1[012])(0[1-9]|[12][0-9]|3[01])", "20170229")
[1] TRUE

Si desea validar una fecha, puede hacerlo a través de esta función definida por el usuario:

is.Date <- function(x) {return(!is.na(as.Date(as.character(x), format = '%Y%m%d')))}

Después

> is.Date(c("20170229", "20170101", 20170101))
[1] FALSE  TRUE  TRUE

Validar las abreviaturas postales de los estados de EE. UU. #

La siguiente regex incluye 50 estados y también Commonwealth/Territory (ver www.50states.com):

regex <- "(A[LKSZR])|(C[AOT])|(D[EC])|(F[ML])|(G[AU])|(HI)|(I[DLNA]) |(K[SY])|(LA)|(M[EHDAINSOT])|(N[EVHJMYCD])|(MP)|(O[HKR])|(P[GUERRA])|(RI)|(S [CD])|(T[NX])|(UT)|(V[TIA])|(W[AVIY])"

Por ejemplo:

> test <- c("AL", "AZ", "AR", "AJ", "AS", "DC", "FM", "GU","PW", "FL", "AJ", "AP")
> grepl(us.states.pattern, test)
 [1]  TRUE  TRUE  TRUE FALSE  TRUE  TRUE  TRUE  TRUE  TRUE  TRUE FALSE FALSE
>

Nota:

Si desea verificar solo los 50 estados, le recomendamos que utilice el conjunto de datos R: state.abb de state, por ejemplo:

> data(state)
> test %in% state.abb
[1]  TRUE  TRUE  TRUE FALSE FALSE FALSE FALSE FALSE FALSE  TRUE FALSE FALSE

Obtenemos VERDADERO solo para las abreviaturas de 50 estados: AL, AZ, AR, FL.

Validar números de teléfono de EE. UU. #

La siguiente expresión regular:

us.phones.regex <- "^\\s*(\\+\\s*1(-?|\\s+))*[0-9]{3}\\s*-?\\s*[0-9]{3}\\s*-?\\s*[0-9]{4}$"

Valida un número de teléfono en forma de: +1-xxx-xxx-xxxx, incluidos los espacios en blanco opcionales iniciales/posteriores al principio/final de cada grupo de números, pero no en el medio, por ejemplo: +1- xxx-xxx-xx xx no es válido. El delimitador - se puede reemplazar por espacios en blanco: xxx xxx xxx o sin delimitador: xxxxxxxxxx. El prefijo +1 es opcional.

Vamos a comprobarlo:

us.phones.regex <- "^\\s*(\\+\\s*1(-?|\\s+))*[0-9]{3}\\s*-?\\s*[0-9]{3}\\s*-?\\s*[0-9]{4}$"

phones.OK <- c("305-123-4567", "305 123 4567", "+1-786-123-4567", 
    "+1 786 123 4567", "7861234567", "786 - 123   4567", "+ 1 786 - 123   4567")

phones.NOK <- c("124-456-78901", "124-456-789", "124-456-78 90", 
    "124-45 6-7890", "12 4-456-7890")

Casos válidos:

> grepl(us.phones.regex, phones.OK)
[1] TRUE TRUE TRUE TRUE TRUE TRUE TRUE
>

Casos no válidos:

 > grepl(us.phones.regex, phones.NOK)
[1] FALSE FALSE FALSE FALSE FALSE
>

Nota:

  • \\s Coincide con cualquier espacio, tabulación o carácter de nueva línea

Caracteres de escape en patrones de expresiones regulares R #

Dado que tanto R como regex comparten el carácter de escape "\", la creación de patrones correctos para grep, sub, gsub o cualquier otra función que acepte un argumento de patrón a menudo necesitará emparejar barras invertidas. Si crea un vector de caracteres de tres elementos en el que uno de los elementos tiene un salto de línea, otro un carácter de tabulación y el otro ninguno, y el deseo es convertir el salto de línea o la tabulación en 4 espacios, entonces se necesita una sola barra invertida para la construcción, pero barras diagonales inversas emparejadas para hacer coincidir:

x <- c( "a\nb", "c\td", "e    f")
x  # how it's stored
   #  [1] "a\nb"   "c\td"   "e    f"
cat(x)   # how it will be seen with cat
#a
#b c    d e    f

gsub(patt="\\n|\\t", repl="    ", x)
#[1] "a    b" "c    d" "e    f"

Tenga en cuenta que el argumento del patrón (que es opcional si aparece primero y solo necesita una ortografía parcial) es el único argumento que requiere esta duplicación o emparejamiento. El argumento de reemplazo no requiere la duplicación de los caracteres que deben escaparse. Si quisiera reemplazar todos los saltos de línea y las ocurrencias de 4 espacios con tabulaciones, sería:

gsub("\\n|    ", "\t", x)
#[1] "a\tb" "c\td" "e\tf"

Eliminar espacios en blanco #

string <- '    some text on line one; 
and then some text on line two     '

Recortar espacios en blanco #

“Recortar” los espacios en blanco generalmente se refiere a eliminar los espacios en blanco iniciales y finales de una cadena. Esto se puede hacer usando una combinación de los ejemplos anteriores. gsub se usa para forzar el reemplazo de las coincidencias principales y finales.

Antes de R 3.2.0

gsub(pattern = "(^ +| +$)",
     replacement = "",
     x = string)

[1] "some text on line one; \nand then some text on line two"

R 3.2.0 y superior

trimws(x = string)

[1] "some text on line one; \nand then some text on line two"

Eliminación de espacios en blanco iniciales #

Antes de R 3.2.0

sub(pattern = "^ +", 
    replacement = "",
    x = string)

[1] "some text on line one; \nand then some text on line two     "

R 3.2.0 y superior

trimws(x = string,
       which = "left")

[1] "some text on line one; \nand then some text on line two     "

Eliminación de espacios en blanco finales #

Antes de R 3.2.0

sub(pattern = " +$",
    replacement = "",
    x = string)

[1] "    some text on line one; \nand then some text on line two"

R 3.2.0 y superior

trimws(x = string,
       which = "right")

[1] "    some text on line one; \nand then some text on line two"

Eliminar todos los espacios en blanco #

gsub(pattern = "\\s",   
     replacement = "",
     x = string)

[1] "sometextonlineone;andthensometextonlinetwo"

Tenga en cuenta que esto también eliminará los espacios en blanco como tabulaciones (\t), saltos de línea (\r y \n) y espacios.
