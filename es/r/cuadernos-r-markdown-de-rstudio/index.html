Un R Notebook es un documento R Markdown con fragmentos que se pueden ejecutar de forma independiente e interactiva, con la salida visible inmediatamente debajo de la entrada. Son similares a los documentos R Markdown con la excepción de que los resultados se muestran en el modo de creación/edición de R Notebook en lugar de en la salida renderizada. Nota: Los portátiles R son una característica nueva de RStudio y solo están disponibles en la versión 1.0 o superior de RStudio.

Crear un cuaderno #

Puede crear un nuevo cuaderno en RStudio con el comando de menú Archivo -> Nuevo archivo -> R Notebook Si no ve la opción para R Notebook, debe actualizar su versión de RStudio. Para la instalación de RStudio, siga [esta guía] (https://www.wikiod.com/es/r/primeros-pasos-con-el-lenguaje-r)

Insertar fragmentos #

Los fragmentos son fragmentos de código que se pueden ejecutar de forma interactiva. Para insertar un nuevo fragmento, haga clic en el botón insertar presente en la barra de herramientas del cuaderno y seleccione la plataforma de código deseada (R en este caso, ya que queremos escribir código R). Alternativamente, podemos usar métodos abreviados de teclado para insertar un nuevo fragmento Ctrl + Alt + I (OS X: Cmd + Option + I)

Ejecutando código de fragmento #

Puede ejecutar el fragmento actual haciendo clic en Ejecutar fragmento actual (botón de reproducción verde) presente en el lado derecho del fragmento. Alternativamente, podemos usar el atajo de teclado Ctrl + Shift + Enter (OS X: Cmd + Shift + Enter)

La salida de todas las líneas del fragmento aparecerá debajo del fragmento.

Dividir el código en trozos #

Dado que un fragmento produce su salida debajo del fragmento, cuando se tienen varias líneas de código en un solo fragmento que produce múltiples salidas, a menudo es útil dividirlo en varios fragmentos de modo que cada fragmento produzca una salida.

Para hacer esto, seleccione el código que desea dividir en una nueva porción y presione Ctrl + Alt + I (OS X: Cmd + Option + I)

Progreso de ejecución #

Cuando ejecuta código en un cuaderno, aparecerá un indicador en el margen para mostrarle el progreso de la ejecución. Las líneas de código que se han enviado a R están marcadas en verde oscuro; las líneas que aún no se han enviado a R están marcadas con verde claro.

Ejecución de múltiples fragmentos #

Ejecutar o volver a ejecutar fragmentos individuales presionando Ejecutar para todos los fragmentos presentes en un documento puede ser doloroso. Podemos usar Ejecutar todo desde el menú Insertar en la barra de herramientas para ejecutar todos los fragmentos presentes en el cuaderno. El método abreviado de teclado es Ctrl + Alt + R (OS X: Cmd + Opción + R)

También hay una opción de comando Reiniciar R y ejecutar todos los fragmentos (disponible en el menú Ejecutar en la barra de herramientas del editor), que le brinda una nueva sesión de R antes de ejecutar todos los fragmentos.

También tenemos opciones como Ejecutar todos los fragmentos por encima y Ejecutar todos los fragmentos por debajo para ejecutar fragmentos por encima o por debajo de un fragmento seleccionado.

Salida de vista previa #

Antes de renderizar la versión final de un cuaderno, podemos obtener una vista previa de la salida. Haga clic en el botón Vista previa en la barra de herramientas y seleccione el formato de salida deseado.

Puede cambiar el tipo de salida usando las opciones de salida como “pdf_document” o “html_notebook”

Guardar y compartir #

Cuando se guarda un cuaderno .Rmd, se crea un archivo .nb.html junto a él. Este archivo es un archivo HTML autónomo que contiene tanto una copia procesada del cuaderno con todos los resultados de fragmentos actuales (adecuados para mostrar en un sitio web) como una copia del propio cuaderno .Rmd.

Puede encontrar más información en [documentos de RStudio] (http://rmarkdown.rstudio.com/r_notebooks.html)
