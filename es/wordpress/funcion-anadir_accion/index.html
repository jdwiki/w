Sintaxis #

  • add_action ($ etiqueta, $ function_to_add)
  • add_action ($etiqueta, $función_a_agregar, $prioridad)
  • add_action ($etiqueta, $función_a_agregar, $prioridad, $args_aceptados)

Parámetros #

Parámetro Detalles
$etiqueta (cadena) (Obligatorio) El nombre de la acción a la que está vinculada $function_to_add
$función_a_añadir (llamable) (Obligatorio) La función que debe llamarse cuando se ejecuta la acción indicada por $tag
$prioridad (int) (Opcional) Valor predeterminado: 10 Se utiliza para especificar el orden en que se ejecutan las funciones asociadas con una acción en particular. Los números más bajos corresponden a una ejecución anterior y las funciones con la misma prioridad se ejecutan en el orden en que se agregaron a la acción.
$args_aceptados (int) (Opcional) Valor predeterminado: 1 El número de argumentos que acepta la función.

La función add_action() crea un Gancho de acción, asociando una función PHP con una “etiqueta” o nombre de acción en particular. Cuando la acción es “activada” por una llamada a do_action() (o do_action_ref_array()) con una etiqueta específica, todas las funciones “enganchadas” a esa etiqueta serán ejecutadas.

En la mayoría de los casos, esta función debe usarse en el archivo functions.php de un tema o en un archivo de complemento, u otro archivo fuente cargado por cualquiera.

Esta función es parte de la API del complemento

Gancho de acción básico #

La aplicación más básica de add_action() es agregar código personalizado para que se ejecute en una determinada ubicación en el código fuente de una instalación de WordPress, ya sea usando acciones proporcionadas por el Core de WordPress, o las creadas por terceros. código de fiesta como complementos y temas.

Para agregar contenido a la sección <head></head> del sitio, diga un metaelemento add <link> para indicar dónde se puede encontrar la información de derechos de autor para el sitio: se puede usar add_action() para adjuntar una función que imprima el marcado apropiado para la acción 'wp_head' (que “activa” cuando WordPress construye la sección <head>):

function add_copyright_meta_link() {
  echo( '<link rel="copyright" href="' . get_home_url() . '/copyright">' );
}

add_action( 'wp_head', 'add_copyright_meta_link' );

Prioridad de gancho de acción #

Se puede “enganchar” cualquier número de funciones a cualquier acción dada. En algunos casos, es importante que una función enganchada se ejecute antes o después de otras, que es donde entra en juego el tercer parámetro de add_action(), $priority.

Si se omite el argumento $priority, la función se adjuntará con la prioridad predeterminada de 10. Cuando la acción se “dispara”, las funciones “enganchadas” se llamarán comenzando con las agregadas con la $prioridad más pequeña y progresando a las funciones con la $prioridad más grande. Cualquier función enganchada que comparta la misma prioridad se llamará en el orden en que se agregaron (el orden en que se ejecutaron sus respectivas llamadas add_action()).

Por ejemplo, supongamos que un complemento de terceros está utilizando una función vinculada a la acción 'template_redirect' para reenviar a los visitantes de la página de ofertas diarias a un enlace de afiliado para un sitio de comercio electrónico externo, pero usted Me gustaría que la redirección solo se produjera para los usuarios que iniciaron sesión. Necesitarás usar tu propio enlace 'template_redirect' para enviar a los visitantes que no hayan iniciado sesión a la página de inicio de sesión. Después de determinar que el complemento de terceros adjunta su función con la $ prioridad predeterminada de 10, puede vincular su función con una prioridad de 9 para asegurarse de que su verificación de inicio de sesión ocurra primero:

function redirect_deal_visitors_to_login() {
  if( is_page( 'daily-deal' ) && !user_is_logged_in() ) {
    wp_redirect( wp_login_url() );
    exit();
  }
}

add_action( 'template_redirect', 'redirect_deal_visitors_to_login', 9 );

Conexión de métodos de objetos y clases a acciones #

[Clases de PHP] (http://php.net/manual/en/language.oop5.basic.php) son una herramienta poderosa para mejorar la organización del código y minimizar las colisiones de nombres. En algún momento u otro, inevitablemente surge la cuestión de cómo crear un gancho de acción para un método de clase.

El argumento $function_to_add a menudo se muestra como una cadena que contiene el nombre de la función, sin embargo, el tipo de datos del argumento es en realidad un “[callable](http://php.net/manual/en/language.types.callable .php)”, que para nuestros propósitos se puede resumir como “una referencia a una función o método”.

Hay una serie de formatos invocables que se pueden usar para hacer referencia a métodos en clases y objetos. Sin embargo, en todos los casos, el método al que se hace referencia debe ser [visible] públicamente (http://php.net/manual/en/language.oop5.visibility.php). Un método es público cuando tiene como prefijo la palabra clave público o no tiene ninguna palabra clave de visibilidad (en cuyo caso, el método predeterminado es público).

Ganchos de acción de método de objeto #

Los métodos de objetos se ejecutan en una instancia particular de una clase.

class My_Class {
  // Constructor
  function My_Class() {
    // (Instantiation logic)
  }

  // Initialization function
  public function initialize() {
    // (Initialization logic)
  }
}

Después de instanciar la clase anterior de la siguiente manera,

$my_class_instance = new My_Class();

el método initialize() normalmente se invocaría en el objeto llamando a $my_class_instance->initialize();. Enganchar el método a la acción 'init' de WordPress se realiza pasando una matriz que contiene una referencia a la instancia y una cadena que contiene el nombre del método del objeto:

add_action( 'init', [ $my_class_instance, 'initialize' ] );

Si se llama a add_action() dentro de un método de objeto, también se puede usar la pseudovariable $this:

class My_Class {
  // Constructor
  function My_Class() {
    // (Instantiation logic)
    add_action( 'init', [ $this, 'initialize' ] );
  }

  // Initialization function
  public function initialize() {
    // (Initialization logic)
  }
}

Ganchos de acción de método de clase #

Los métodos de clase se ejecutan estáticamente en una clase en lugar de en cualquier instancia en particular. Dada la siguiente clase,

class My_Class {   
  // Initialization function
  public static function initialize() {
    // (Initialization logic)
  }
}

el método initialize() normalmente se invocaría utilizando el operador de resolución de alcance ::, es decir, My_Class::initialize();. La conexión de un método de clase estática a un WordPress se puede hacer de dos maneras diferentes:

  • Usando una matriz compuesta por una cadena que contiene el nombre de la clase y una cadena que contiene el nombre del método:

     add_action( 'init', [ 'My_Class', 'initialize' ] );

  • Pasar una cadena que contiene una referencia completa al método, incluido el operador :::

     add_action( 'init', 'My_Class::initialize' );

  • Si se llama a add_action() dentro de un método de clase estático, se puede usar la palabra clave self o la constante mágica __CLASS__ en lugar del nombre de la clase. Tenga en cuenta que esto generalmente no es recomendable ya que los valores de estos elementos se vuelven algo contradictorios en el caso de la herencia de clase.

     class My_Class {
   // Setup function
   public static function setup_actions() {
     add_action( 'init', 'self::initialize' );
   }

   // Initialization function
   public static function initialize() {
     // (Initialization logic)
   }
 }
