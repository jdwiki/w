Introducción #

Una de las cosas más importantes que debe aprender cuando crea un tema de WordPress es la jerarquía de plantillas de WordPress para temas. La jerarquía de plantillas define qué archivo de plantilla se cargará para cada solicitud y en qué orden. Si la primera plantilla no existe en la jerarquía, WordPress intentará cargar la siguiente y así sucesivamente hasta que termines en index.php.

Para describir la jerarquía de plantillas en detalle, la mejor manera es, por supuesto, utilizar una imagen con la estructura completa:

La jerarquía de plantillas comienza con los tipos principales de páginas como archivo, página singular o portada y luego los subtipos como archivo de autor, archivo de etiquetas, página o publicación de blog.

Después de esto, encontraremos las plantillas reales que se cargarán. En primer lugar, están las plantillas más específicas que usan slugs o ID para apuntar a publicaciones o tipos de archivo específicos.

Por ejemplo, category-$slug.php que solo apunta a la categoría con un slug específico, por ejemplo, category-books.php se usaría solo para la categoría con el slug book. Otro ejemplo es page-$id.php que solo apunta a una página con una ID específica, por ejemplo, page-41.php apuntaría solo a la página con la ID 41.

Después de las plantillas que apuntan a tipos o publicaciones específicas, llegamos a las plantillas de tipo genérico, como archive.php para todas las páginas de archivo o page.php para todas las páginas. Pero recuerde, solo se usarán si la página actual no coincide con ninguna de las plantillas que están más arriba en la jerarquía.

Por último, si WordPress no pudo encontrar ninguna plantilla coincidente en el directorio de plantillas, el último recurso siempre es index.php, que es el único archivo de plantilla requerido en un tema de WordPress.

Depuración #

Es fácil perderse mientras se depura la jerarquía. Puede usar el comando integrado de PHP debug_backtrace.

Coloque el siguiente fragmento dentro de cualquier plantilla que desee depurar y vea la página generada:

<!--
<?php print_r( debug_backtrace() ) ?>
-->
