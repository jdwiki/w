Asegure su instalación

Los sitios web de WordPress son frecuentemente pirateados. Este tema es para técnicas y prácticas que aumentan la seguridad de su instalación de WordPress más allá de lo que se logra en una instalación base.

Además de este tema, otro buen lugar para leer acerca de cómo asegurar una instalación de WordPress es la página del Codex de fortalecimiento de WordPress.

Mover wp-config.php #

La información más confidencial de una instalación de WordPress se almacena en el archivo wp-config.php. Si un pirata informático obtiene acceso a este archivo, tendrá el control total de su sitio web.

Por defecto wp-config.php se almacena en la carpeta de instalación de WordPress. Para hacer que este archivo sea más difícil de robar, puede moverlo fuera de la carpeta accesible desde la web. Si lo mueve solo una carpeta arriba, WordPress lo encontrará automáticamente. Si mueve wp-config.php a una ubicación diferente, cree un archivo vacío llamado wp-config.php en la carpeta de instalación de WordPress. Luego agrega lo siguiente:

define('ABSPATH', dirname(__FILE__) . '/');
// '../../wp-config.php' defines location two folders above installation folder.
// Substitute with actual location of wp-config.php file as necessary.
require_once(ABSPATH . '../../wp-config.php');

Es posible que deba hacer que php sea ejecutable en la carpeta en la que coloque wp-config.php. Debe hacer que php sea ejecutable en la menor cantidad de carpetas posible. Un buen sistema coloca la instalación de WordPress en /ruta/a/wordpress/install/ y la configuración en /ruta/a/wordpress/config. Se aseguraría de que la carpeta de configuración no sea accesible desde la web y no coloque ninguna otra información confidencial en /ruta/a/ o superior en la jerarquía de carpetas. En ese caso, escribiría una línea similar a la siguiente en su php.ini:

open_basedir = "/path/to/wordpress/install/;/path/to/wordpress/config"

Esta técnica es controvertida y algunas personas no creen que mejore la seguridad. Se puede leer una amplia discusión sobre el tema en esta pregunta de WordPress StackExchange.

Deshabilitar el editor de archivos #

El editor de archivos que viene con WordPress es un riesgo para la seguridad. Si un atacante obtiene acceso de administrador a su sitio web de WordPress, podrá insertar fácilmente código malicioso en los archivos de temas y complementos. También es un riesgo con los clientes que no saben lo que están haciendo. Una vez que se extravían dos puntos en el editor de archivos, se puede romper un sitio y hacerlo inaccesible desde el navegador.

En su archivo wp-config.php de WordPress, deshabilite el editor de archivos agregando la siguiente línea de código.

define( 'DISALLOW_FILE_EDIT', true );

Esa línea tendrá el efecto deseado cuando se agregue al archivo functions.php de su tema, pero es mejor agregarla a wp-config.php.

Si está utilizando WordPress CLI para instalar WordPress, puede usar el siguiente comando para crear un archivo wp-config.php con la edición de archivos deshabilitada.

/* declare variables beforehand or substitute strings in */
wp core config --dbname="$MYSQL_DBNAME" --dbuser="$MYSQL_USERNAME" --dbpass="$MYSQL_PASS" --dbprefix="$WP_DBPREFIX"_ --locale=en_AU --extra-php <<PHP
define( 'DISALLOW_FILE_EDIT', true );
PHP

Este método es útil si instala WordPress con un script.

Establecer un prefijo personalizado para las tablas de WordPress #

Cuando instale WordPress en su servidor, el script de instalación colocará un prefijo delante de todos los nombres de las tablas MySQL de WordPress. Este prefijo se establece en ‘wp_’ de forma predeterminada. La tabla de publicaciones de WordPress se llamará wp_posts, por ejemplo. Al cambiar el prefijo de la tabla, puede crear cierta seguridad por oscuridad. De esta forma, cuando un hacker intente ataques de inyección SQL, tendrá que adivinar el prefijo de su tabla en lugar de simplemente usar ‘wp_’. Puede configurar este prefijo para que sea lo que quiera.

Establecer prefijo en la nueva instalación de WordPress

Si usa la famosa instalación de 5 minutos, cambie el prefijo en el campo durante la instalación.

Si instala a través de la CLI de WordPress, use el siguiente comando:

// set other variables above, or substitute your strings in.
WP_DBPREFIX=foo
wp core config --dbname="$MYSQL_DBNAME" --dbuser="$MYSQL_USERNAME" --dbpass="$MYSQL_PASS" --dbprefix="$WP_DBPREFIX"_ --locale=en_AU

Cambiar prefijo en instalación existente

Cambiar el prefijo es un poco más difícil. En primer lugar, use un programa FTP como FileZilla para editar el archivo wp-config.php. Cambie la entrada $table_prefix = 'wp_'; a $table_prefix = 'foo_'; sustituyendo ‘foo’ por el prefijo deseado.

A continuación, tendremos que editar la base de datos. Si tiene acceso a phpMyAdmin, inicie sesión y haga lo siguiente:

  • Seleccione la base de datos de WordPress
  • Seleccione todas las tablas y, en el menú desplegable, seleccione reemplazar el prefijo de la tabla.
  • En “De” escribe ‘wp_’. En “Para” escriba su prefijo, ‘foo_’ en este ejemplo y presione “Enviar”.
  • Las tablas ahora deberían verse así: [! [ingrese la descripción de la imagen aquí] 5] 5

Si no puede usar phpMyAdmin, use el siguiente comando de MySQL:

RENAME table `wp_comments` TO `foo_comments`

Deberá ejecutar ese comando para cada tabla, sustituyendo “comentarios” por los otros nombres de tabla.

A continuación, debemos cambiar algunas entradas en algunas tablas. Ejecute esta consulta en la tabla ‘foo_options’

SELECT * FROM  foo_options WHERE option_name LIKE '%user_roles%'

Debería aparecer una entrada con option_name de ‘wp_user_roles’. En esa entrada, cambie la entrada ‘option_name’ de wp_user_roles a foo_user_roles.

Luego abra la tabla ‘foo_usermeta’ y busque todas las entradas con ‘wp_’ al frente. y cámbielo a ‘foo_’. El número de entradas que debe cambiar dependerá de cuántos usuarios tenga.

Eso debería ser todo lo que necesita para cambiar el prefijo en una instalación existente
