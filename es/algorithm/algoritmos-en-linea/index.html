Teoría #

Definición 1: Un problema de optimización Π consta de un conjunto de instancias ΣΠ. Para cada instancia σ∈ΣΠ existe un conjunto Ζσ de soluciones y una función objetivo fσ : Ζσ → ℜ≥0 que asigna un valor real positivo a cada solución.
Decimos que OPT(σ) es el valor de una solución óptima, A(σ) es la solución de un Algoritmo A para el problema Π y wA(σ)=fσ(A(σ)) su valor.

Definición 2: Un algoritmo en línea A para un problema de minimización Π tiene una razón competitiva de r ≥ 1 si hay una constante τ∈ℜ con

wA(σ) = fσ(A(σ)) ≤ r ⋅ OPT(&sigma) + τ

para todas las instancias σ∈ΣΠ. A se denomina algoritmo en línea r-competitivo. Incluso

wA(σ) ≤ r ⋅ OPT(&sigma)

para todas las instancias σ∈ΣΠ entonces A se denomina algoritmo en línea estrictamente competitivo con r.

Proposición 1.3: LRU y FWF son algoritmos de marcado.

Prueba: Al comienzo de cada fase (excepto la primera), FWF tiene un error de caché y borró el caché. eso significa que tenemos k páginas vacías. En cada fase se solicitan un máximo de k páginas diferentes, por lo que ahora habrá desalojo durante la fase. Entonces FWF es un algoritmo de marcado.
Supongamos que LRU no es un algoritmo de marcado. Luego hay una instancia σ donde LRU una página marcada x en la fase i desalojada. Sea σt la solicitud en la fase i donde x es desalojada. Dado que x está marcado, tiene que haber una solicitud anterior σt* para x en la misma fase, por lo que t* < t. Después de t* x es la página más reciente de los cachés, por lo que para ser desalojado en t, la secuencia σt*+1,…,σt tiene que solicitar en menos k de x páginas diferentes. Eso implica que la fase i ha solicitado al menos k+1 páginas diferentes, lo que contradice la definición de la fase. Entonces LRU tiene que ser un algoritmo de marcado.

Proposición 1.4: Cada algoritmo de marcado es estrictamente k-competitivo.

Prueba: Sea σ ser una instancia para el problema de paginación y l el número de fases para σ. Si l = 1, entonces todos los algoritmos de marcado son óptimos y el algoritmo fuera de línea óptimo no puede ser mejor.
Suponemos l ≥ 2. el costo de cada algoritmo de marcado, por ejemplo σ está acotado desde arriba con l ⋅ k porque en cada fase un algoritmo de marcado no puede desalojar más de k páginas sin desalojar una página marcada.
Ahora tratamos de mostrar que el algoritmo offline óptimo expulsa al menos k+l-2 páginas para θ, k en la primera fase y al menos una para cada fase siguiente excepto la última. Como prueba, definamos l-2 subsecuencias disjuntas de σ. Subsecuencia i ∈ {1,…,l-2} comienza en la segunda posición de la fase i+1 y finaliza en la primera posición de la fase i+2.
Sea x la primera página de la fase i+1. Al comienzo de la subsecuencia i hay una página x y como máximo k-1 páginas diferentes en la memoria caché óptima de algoritmos fuera de línea. En la subsecuencia, hay k solicitudes de página diferentes de x, por lo que el algoritmo fuera de línea óptimo tiene que desalojar al menos una página para cada subsecuencia. Dado que al comienzo de la fase 1, el caché aún está vacío, el algoritmo fuera de línea óptimo provoca k desalojos durante la primera fase. Eso demuestra que

wA(σ) ≤ l⋅k ≤ (k+l-2)k ≤ OPT ( & sigma ; ) & sdot ; k

Corolario 1.5: LRU y FWF son estrictamente k-competitivos.

La primera página 1 se solicita l veces que la página 2 y así sucesivamente. Al final hay (l-1) solicitudes alternas de página k y k+1.

Paginación #

El problema de paginación surge de la limitación del espacio finito. Supongamos que nuestro caché C tiene k páginas. Ahora queremos procesar una secuencia de solicitudes de página m que deben haberse colocado en el caché antes de que se procesen. Por supuesto, si m<=k simplemente colocamos todos los elementos en el caché y funcionará, pero generalmente es m>>k.

Decimos que una solicitud es un golpe de caché, cuando la página ya está en caché; de lo contrario, se llama una falta de caché. En ese caso, debemos llevar la página solicitada a la memoria caché y expulsar otra, suponiendo que la memoria caché esté llena. The Goal es un calendario de desalojos que minimiza la cantidad de desalojos.

Existen numerosas estrategias para este problema, veamos algunas:

  1. Primero en entrar, primero en salir (FIFO): La página más antigua es desalojada
  2. Último en entrar, primero en salir (LIFO): La página más nueva se desaloja
  3. Usado menos recientemente (LRU): página de desalojo cuyo acceso más reciente fue el más antiguo
  4. Uso menos frecuente (LFU): desaloje la página que se solicitó con menos frecuencia
  5. Distancia de avance más larga (LFD): Expulsa la página en el caché que no se solicita hasta el futuro más lejano.
  6. Descargar cuando esté lleno (FWF): borre la memoria caché tan pronto como se produzca un error de memoria caché.

Hay dos formas de abordar este problema:

  1. fuera de línea: la secuencia de solicitudes de página se conoce con anticipación
  2. en línea: la secuencia de solicitudes de página no se conoce de antemano

Enfoque fuera de línea #

Para el primer enfoque, consulte el tema [Aplicaciones de la técnica Greedy] (https://www.wikiod.com/es/algorithm/aplicaciones-de-la-tecnica-greedy). Su tercer ejemplo Almacenamiento en caché sin conexión considera las primeras cinco estrategias de arriba y le brinda un buen punto de entrada para lo siguiente.

El programa de ejemplo se amplió con la estrategia FWF:

class FWF : public Strategy {
public:
    FWF() : Strategy("FWF")
    {
    }

    int apply(int requestIndex) override
    {
        for(int i=0; i<cacheSize; ++i)
        {
            if(cache[i] == request[requestIndex])
                return i;

            // after first empty page all others have to be empty
            else if(cache[i] == emptyPage)
                return i;
        }

        // no free pages
        return 0;
    }

    void update(int cachePos, int requestIndex, bool cacheMiss) override
    {

        // no pages free -> miss -> clear cache
        if(cacheMiss && cachePos == 0)
        {
            for(int i = 1; i < cacheSize; ++i)
                cache[i] = emptyPage;
        }
    }
};

Aunque LFD es óptimo, FWF tiene menos errores de caché. Pero el objetivo principal era minimizar la cantidad de desalojos y, para FWF, cinco fallas significan 15 desalojos, lo que la convierte en la opción más pobre para este ejemplo.

Enfoque en línea #

Ahora queremos abordar el problema en línea de la paginación. Pero primero necesitamos entender cómo hacerlo. Obviamente, un algoritmo en línea no puede ser mejor que el algoritmo fuera de línea óptimo. Pero, ¿cuánto peor es? Necesitamos definiciones formales para responder a esa pregunta:

Definición 1.1: Un problema de optimización Π consta de un conjunto de instancias ΣΠ. Para cada instancia σ∈ΣΠ existe un conjunto Ζσ de soluciones y una función objetivo fσ : Ζσ → ℜ≥0 que asigna un valor real positivo a cada solución.
Decimos que OPT(σ) es el valor de una solución óptima, A(σ) es la solución de un Algoritmo A para el problema Π y wA(σ)=fσ(A(σ)) su valor.

Definición 1.2: Un algoritmo en línea A para un problema de minimización Π tiene una razón competitiva de r ≥ 1 si hay una constante τ∈ℜ con

wA(σ) = fσ(A(σ)) ≤ r ⋅ OPT(σ) + τ

para todas las instancias σ∈ΣΠ. A se denomina algoritmo en línea r-competitivo. Incluso

wA(σ) ≤ r ⋅ OPC(σ)

para todas las instancias σ∈ΣΠ entonces A se denomina algoritmo en línea estrictamente competitivo con r.

Entonces, la pregunta es cuán competitivo es nuestro algoritmo en línea en comparación con un algoritmo fuera de línea óptimo. En su famoso [libro] (http://www.cs.technion.ac.il/~rani/book.html), Allan Borodin y Ran El-Yaniv utilizaron otro escenario para describir la situación de la paginación en línea:

Hay un adversario malvado que conoce tu algoritmo y el algoritmo óptimo fuera de línea. En cada paso, intenta solicitar una página que sea peor para usted y, al mismo tiempo, mejor para el algoritmo fuera de línea. el factor competitivo de su algoritmo es el factor de qué tan mal le fue a su algoritmo en comparación con el algoritmo fuera de línea óptimo del adversario. Si quieres intentar ser el adversario, puedes probar el Juego del adversario (intenta superar las estrategias de paginación).

Algoritmos de marcado #

En lugar de analizar cada algoritmo por separado, veamos una familia especial de algoritmos en línea para el problema de paginación llamada algoritmos de marcado.

Sea σ=(σ1,…,σp) una instancia para nuestro problema y k nuestro tamaño de caché, entonces σ se puede dividir en fases:

  • La fase 1 es la subsecuencia máxima de σ desde el inicio hasta el máximo k se solicitan diferentes páginas
  • Fase i ≥ 2 es la subsecuencia máxima de σ desde el final del pase i-1 hasta el máximo de k se solicitan diferentes páginas

Por ejemplo con k = 3:

Un algoritmo de marcado (implícita o explícitamente) mantiene si una página está marcada o no. Al comienzo de cada fase, todas las páginas están sin marcar. Es una página solicitada durante una fase se marca. Un algoritmo es un algoritmo de marcado si nunca expulsa una página marcada del caché. Eso significa que las páginas que se utilizan durante una fase no serán desalojadas.
