Vendedor ambulante

  • Algoritmo de fuerza bruta
  • Algoritmo de programación dinámica

En esta página

  • Algoritmo de fuerza bruta
  • Algoritmo de programación dinámica

El Problema del Vendedor Viajero es el problema de encontrar el costo mínimo de viajar a través de N vértices exactamente una vez por vértice. Hay un costo costo[i][j] para viajar desde el vértice i al vértice j.

Hay 2 tipos de algoritmos para resolver este problema: Algoritmos exactos y Algoritmos de aproximación

Algoritmos exactos

  1. Algoritmo de fuerza bruta
  2. Algoritmo de programación dinámica

Algoritmos de aproximación

Para ser agregado

Algoritmo de fuerza bruta #

Un camino a través de cada vértice exactamente una vez es lo mismo que ordenar los vértices de alguna manera. Por lo tanto, para calcular el costo mínimo de viajar a través de cada vértice exactamente una vez, podemos aplicar fuerza bruta a cada una de las permutaciones N! de los números de 1 a N.

Psuedocódigo

minimum = INF
for all permutations P

    current = 0                                

    for i from 0 to N-2
        current = current + cost[P[i]][P[i+1]]  <- Add the cost of going from 1 vertex to the next
    
    current = current + cost[P[N-1]][P[0]]      <- Add the cost of going from last vertex to the first

    if current < minimum                        <- Update minimum if necessary
        minimum = current
    
output minimum

Complejidad de tiempo

Hay permutaciones N! para pasar y el costo de cada ruta se calcula en O (N), por lo tanto, este algoritmo tarda O (N * N!) para generar la respuesta exacta.

Algoritmo de programación dinámica #

Note que si consideramos el camino (en orden):

(1,2,3,4,6,0,5,7)

y el camino

(1,2,3,5,0,6,7,4)

El costo de pasar del vértice ‘1’ al vértice ‘2’ al vértice ‘3’ sigue siendo el mismo, entonces, ¿por qué se debe volver a calcular? Este resultado se puede guardar para su uso posterior.

Sea dp[máscara de bits][vértice] que represente el costo mínimo de viajar a través de todos los vértices cuyo bit correspondiente en máscara de bits se establece en 1 y termina en vértice. Por ejemplo:

dp[12][2] 

   12   =   1 1 0 0
            ^ ^ 
vertices:   3 2 1 0

Dado que 12 representa 1100 en binario, dp[12][2] representa pasar por los vértices 2 y 3 en el gráfico con la ruta que termina en el vértice 2.

Por lo tanto, podemos tener el siguiente algoritmo (implementación en C++):

int cost[N][N];                        //Adjust the value of N if needed
int memo[1 << N][N];                   //Set everything here to -1
int TSP(int bitmask, int pos){
    int cost = INF;
    if (bitmask == ((1 << N) - 1)){      //All vertices have been explored
        return cost[pos][0];             //Cost to go back
    }
    if (memo[bitmask][pos] != -1){       //If this has already been computed
        return memo[bitmask][pos];       //Just return the value, no need to recompute
    }
    for (int i = 0; i < N; ++i){         //For every vertex 
        if ((bitmask & (1 << i)) == 0){  //If the vertex has not been visited
            cost = min(cost,TSP(bitmask | (1 << i) , i) + cost[pos][i]);   //Visit the vertex
        } 
    }
    memo[bitmask][pos] = cost;           //Save the result
    return cost;
}
//Call TSP(1,0)

Esta línea puede ser un poco confusa, así que repasemos lentamente:

cost = min(cost,TSP(bitmask | (1 << i) , i) + cost[pos][i]);

Aquí, máscara de bits | (1 << i) establece el i-ésimo bit de máscara de bits en 1, lo que representa que se ha visitado el i-ésimo vértice. La i después de la coma representa el nuevo pos en esa llamada de función, que representa el nuevo “último” vértice. cost[pos][i] es sumar el costo de viajar desde el vértice pos al vértice i.

Por lo tanto, esta línea es para actualizar el valor de ‘costo’ al valor mínimo posible de viajar a cualquier otro vértice que aún no se haya visitado.

Complejidad de tiempo

La función TSP(bitmask,pos) tiene valores 2^N para bitmask y valores N para pos. Cada función tarda O(N) en ejecutarse (el bucle for). Por lo tanto, esta implementación toma tiempo O (N ^ 2 * 2 ^ N) para generar la respuesta exacta.
