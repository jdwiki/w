Este ejemplo muestra cómo crear una aplicación express básica y luego servir AngularJS.

Creando nuestro proyecto. #

Estamos listos para comenzar, así que ejecutamos, nuevamente desde la consola:

mkdir our_project
cd our_project

Ahora estamos en el lugar donde vivirá nuestro código. Para crear el archivo principal de nuestro proyecto puede ejecutar

Ok, pero ¿cómo creamos el proyecto express skeleton?

Es simple:

npm install -g express express-generator

Las distribuciones de Linux y Mac deben usar sudo para instalar esto porque están instalados en el directorio nodejs, al que solo puede acceder el usuario raíz. Si todo salió bien, podemos, finalmente, crear el esqueleto de la aplicación Express, simplemente ejecute

express

Este comando creará dentro de nuestra carpeta una aplicación de ejemplo express. La estructura es la siguiente:

bin/
public/
routes/
views/
app.js
package.json

Ahora, si ejecutamos npm start y vamos a http://localhost:3000, veremos la aplicación express en funcionamiento, es justo que hayamos generado una aplicación express sin demasiados problemas, pero ¿cómo podemos mezclar esto con AngularJS?.

¿Cómo funciona Express, brevemente? #

Express es un marco construido sobre Nodejs, puede ver la documentación oficial en el Sitio Express. Pero para nuestro propósito necesitamos saber que Express es el responsable cuando escribimos, por ejemplo, http://localhost:3000/home de representar la página de inicio de nuestra aplicación. Desde la app creada recientemente podemos comprobar:

FILE: routes/index.js
var express = require('express');
var router = express.Router();

/* GET home page. */
router.get('/', function(req, res, next) {
  res.render('index', { title: 'Express' });
});

module.exports = router;

Lo que este código nos dice es que cuando el usuario va a http://localhost:3000, debe mostrar la vista de índice y pasar un JSON con una propiedad de título y valor Express. Pero cuando revisamos el directorio de vistas y abrimos index.jade podemos ver esto:

extends layout
block content
    h1= title
    p Welcome to #{title}

Esta es otra característica poderosa de Express, motores de plantillas, que le permiten representar contenido en la página pasándole variables o heredar otra plantilla para que sus páginas sean más compactas y comprensibles para otros. La extensión del archivo es .jade hasta donde yo sé Jade cambió el nombre por Pug, básicamente es el mismo motor de plantilla pero con algunas actualizaciones y modificaciones principales.

Instalación de Pug y actualización del motor de plantillas Express.

Bien, para comenzar a usar Pug como motor de plantillas de nuestro proyecto, debemos ejecutar:

npm install --save pug

Esto instalará Pug como una dependencia de nuestro proyecto y lo guardará en package.json. Para usarlo necesitamos modificar el archivo app.js:

var app = express();
// view engine setup
app.set('views', path.join(__dirname, 'views'));
app.set('view engine', 'pug');

Y reemplace el motor de línea de visión con pug y eso es todo. Podemos volver a ejecutar nuestro proyecto con npm start y veremos que todo funciona bien.

¿Cómo encaja AngularJS en todo esto? #

AngularJS es un marco Javascript MVW(Modelo-Vista-Lo que sea) que se utiliza principalmente para crear SPA(Aplicación de página simple) La instalación es bastante simple, puede ir al sitio web de AngularJS y descargar el última versión que es v1.6.4.

Después de que descargamos AngularJS, cuando deberíamos copiar el archivo a nuestra carpeta public/javascripts dentro de nuestro proyecto, una pequeña explicación, esta es la carpeta que sirve a los activos estáticos de nuestro sitio, imágenes, css, archivos javacript, etc. Por supuesto, esto se puede configurar a través del archivo app.js, pero lo mantendremos simple. Ahora creamos un archivo llamado ng-app.js, el archivo donde vivirá nuestra aplicación, dentro de nuestra carpeta pública de javascripts, justo donde vive AngularJS. Para activar AngularJS, debemos modificar el contenido de views/layout.pug de la siguiente manera:

doctype html
html(ng-app='first-app')
  head
    title= title
    link(rel='stylesheet', href='/stylesheets/style.css')
  body(ng-controller='indexController')
    block content

  script(type='text-javascript', src='javascripts/angular.min.js')
  script(type='text-javascript', src='javascripts/ng-app.js')

¿Qué estamos haciendo aquí? Bueno, estamos incluyendo el núcleo de AngularJS y nuestro archivo recientemente creado ng-app.js, de modo que cuando se procese la plantilla, aparecerá AngularJS, observe el uso de ng- directiva app, esto le dice a AngularJS que este es el nombre de nuestra aplicación y debe apegarse a él.
Entonces, el contenido de nuestro ng-app.js será:

angular.module('first-app', [])
       .controller('indexController', ['$scope', indexController]);

function indexController($scope) {
    $scope.name = 'sigfried';
}

Estamos usando la característica más básica de AngularJS aquí, enlace de datos bidireccional, esto nos permite actualizar el contenido de nuestra vista y controlador al instante, esta es una explicación muy simple, pero puede hacer una investigación en Google o StackOverflow para ver cómo funciona realmente.

Entonces, tenemos los bloques básicos de nuestra aplicación AngularJS, pero hay algo que debemos hacer, necesitamos actualizar nuestra página index.pug para ver los cambios de nuestra aplicación angular, hagámoslo:

extends layout
block content
  div(ng-controller='indexController')
    h1= title
    p Welcome {{name}}
    input(type='text' ng-model='name')

Aquí solo vinculamos la entrada a nuestro nombre de propiedad definido en el alcance de AngularJS dentro de nuestro controlador:

$scope.name = 'sigfried';

El propósito de esto es que cada vez que cambiemos el texto en la entrada, el párrafo anterior actualizará su contenido dentro del {{nombre}}, esto se llama interpolación, nuevamente otra característica de AngularJS para representar nuestro contenido en la plantilla.

Entonces, todo está configurado, ahora podemos ejecutar npm start, ir a http://localhost:3000 y ver nuestra aplicación express sirviendo la página y AngularJS administrando la interfaz de la aplicación.
