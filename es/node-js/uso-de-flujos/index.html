Parámetros #

Parámetro Definición
Flujo legible tipo de flujo desde donde se pueden leer los datos
Secuencia grabable tipo de flujo en el que se pueden escribir los datos
Transmisión dúplex tipo de flujo que se puede leer y escribir
Transformar corriente tipo de flujo dúplex que puede transformar los datos a medida que se leen y luego se escriben

Leer datos de TextFile con Streams #

La E/S en el nodo es asíncrona, por lo que interactuar con el disco y la red implica pasar devoluciones de llamada a funciones. Es posible que tenga la tentación de escribir un código que sirva un archivo desde el disco como este:

var http = require('http');
var fs = require('fs');

var server = http.createServer(function (req, res) {
    fs.readFile(__dirname + '/data.txt', function (err, data) {
        res.end(data);
    });
});
server.listen(8000);

Este código funciona, pero es voluminoso y almacena en búfer todo el archivo data.txt en la memoria para cada solicitud antes de volver a escribir el resultado en los clientes. Si data.txt es muy grande, su programa podría comenzar a consumir mucha memoria, ya que sirve a muchos usuarios al mismo tiempo, especialmente para usuarios con conexiones lentas.

La experiencia del usuario también es deficiente porque los usuarios deberán esperar a que todo el archivo se almacene en la memoria intermedia de su servidor antes de que puedan comenzar a recibir cualquier contenido.

Afortunadamente, ambos argumentos (req, res) son flujos, lo que significa que podemos escribir esto de una manera mucho mejor usando fs.createReadStream() en lugar de fs.readFile():

var http = require('http');
var fs = require('fs');

var server = http.createServer(function (req, res) {
    var stream = fs.createReadStream(__dirname + '/data.txt');
    stream.pipe(res);
});
server.listen(8000);

Aquí .pipe() se encarga de escuchar los eventos ‘data’ y ’end’ de fs.createReadStream(). Este código no solo es más limpio, sino que ahora el archivo data.txt se escribirá en los clientes un fragmento a la vez inmediatamente a medida que se reciban del disco.

Chorros de tubería #

Los flujos legibles se pueden “encauzar” o conectar a flujos grabables. Esto hace que los datos fluyan desde el flujo de origen al flujo de destino sin mucho esfuerzo.

var fs = require('fs')

var readable = fs.createReadStream('file1.txt')
var writable = fs.createWriteStream('file2.txt')

readable.pipe(writable) // returns writable

Cuando los flujos grabables también son flujos legibles, es decir, cuando son flujos dúplex, puede continuar canalizándolos a otros flujos grabables.

var zlib = require('zlib')

fs.createReadStream('style.css')
  .pipe(zlib.createGzip()) // The returned object, zlib.Gzip, is a duplex stream.
  .pipe(fs.createWriteStream('style.css.gz')

Los flujos legibles también se pueden canalizar en múltiples flujos.

var readable = fs.createReadStream('source.css')
readable.pipe(zlib.createGzip()).pipe(fs.createWriteStream('output.css.gz'))
readable.pipe(fs.createWriteStream('output.css')

Tenga en cuenta que debe canalizar los flujos de salida de forma síncrona (al mismo tiempo) antes de que los datos “fluyan”. Si no lo hace, es posible que se transmitan datos incompletos.

También tenga en cuenta que los objetos de flujo pueden emitir eventos de “error”; asegúrese de manejar estos eventos de manera responsable en cada transmisión, según sea necesario:

var readable = fs.createReadStream('file3.txt')
var writable = fs.createWriteStream('file4.txt')
readable.pipe(writable)
readable.on('error', console.error)
writable.on('error', console.error)

Creando tu propia secuencia legible/escribible #

Veremos objetos de transmisión devueltos por módulos como fs, etc., pero ¿qué pasa si queremos crear nuestro propio objeto de transmisión?

Para crear un objeto Stream, necesitamos usar el módulo de flujo proporcionado por NodeJs

    var fs = require("fs");
    var stream = require("stream").Writable;
    
    /* 
     *  Implementing the write function in writable stream class.
     *  This is the function which will be used when other stream is piped into this 
     *  writable stream.
     */
    stream.prototype._write = function(chunk, data){
        console.log(data);
    }
    
    var customStream = new stream();
    
    fs.createReadStream("am1.js").pipe(customStream);

Esto nos dará nuestra propia secuencia grabable personalizada. podemos implementar cualquier cosa dentro de la función _write. El método anterior funciona en la versión NodeJs 4.x.x pero en NodeJs 6.x ES6 clases introducidas por lo tanto, la sintaxis ha cambiado. A continuación se muestra el código para la versión 6.x de NodeJs

    const Writable = require('stream').Writable;
    
    class MyWritable extends Writable {
      constructor(options) {
        super(options);
      }
    
      _write(chunk, encoding, callback) {
        console.log(chunk);
      }
    }
