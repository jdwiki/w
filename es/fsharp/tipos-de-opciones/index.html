Use Option<‘T> sobre valores nulos #

En lenguajes de programación funcionales como F# se consideran valores null estilo potencialmente dañino y deficiente (no idiomático).

Considere este código C#:

string x = SomeFunction ();
int    l = x.Length;

x.Length lanzará si x es null agreguemos protección:

string x = SomeFunction ();
int    l = x != null ? x.Length : 0;

O:

string x = SomeFunction () ?? "";
int    l = x.Length;

O:

string x = SomeFunction ();
int    l = x?.Length;

En los valores idiomáticos F# null no se usan, por lo que nuestro código se ve así:

let x = SomeFunction ()
let l = x.Length

Sin embargo, a veces es necesario representar valores vacíos o no válidos. Después podemos usar Option<'T>:

let SomeFunction () : string option = ...

AlgunaFunción devuelve Algún valor de cadena o Ninguno. Extraemos el valor string usando coincidencia de patrones

let v =
  match SomeFunction () with
  | Some x  -> x.Length
  | None    -> 0

La razón por la que este código es menos frágil que:

string x = SomeFunction ();
int    l = x.Length;

Es porque no podemos llamar a Length en una opción de cadena. Necesitamos extraer el valor string utilizando la coincidencia de patrones y, al hacerlo, estamos garantizados que el El valor string es seguro de usar.

El módulo de opciones permite la programación orientada al ferrocarril #

El manejo de errores es importante, pero puede convertir un algoritmo elegante en un desastre. Programación Orientada al Ferrocarril (ROP) se utiliza para hacer que el manejo de errores sea elegante y componible.

Considere la función simple f:

let tryParse s =
  let b, v = System.Int32.TryParse s
  if b then Some v else None

let f (g : string option) : float option =
  match g with
  | None    -> None
  | Some s  ->
    match tryParse s with           // Parses string to int
    | None              -> None
    | Some v when v < 0 -> None     // Checks that int is greater than 0
    | Some v -> v |> float |> Some  // Maps int to float

El propósito de f es analizar el valor de cadena de entrada (si hay Algunos) en un int. Si el ‘int’ es mayor que ‘0’, lo convertimos en un ‘flotante’. En todo otros casos los rescatamos con Ninguno.

Aunque, una función extremadamente simple, la “coincidencia” anidada disminuye la legibilidad significativamente.

ROP observa que tenemos dos tipos de rutas de ejecución en nuestro programa

  1. Camino feliz: eventualmente calculará el valor Algunos
  2. Ruta de error: todas las demás rutas producen Ninguna

Dado que las rutas de error son más frecuentes, tienden a apoderarse del código. Nos gustaría que el código de ruta feliz sea la ruta de código más visible.

Una función equivalente g usando ROP podría verse así:

let g (v : string option) : float option =
  v
  |> Option.bind    tryParse  // Parses string to int
  |> Option.filter  ((<) 0)   // Checks that int is greater than 0
  |> Option.map     float     // Maps int to float

Se parece mucho a cómo tendemos a procesar listas y secuencias en F#.

Uno puede ver una Opción<'T> como una Lista<'T> que solo puede contener el elemento 0 o 1 donde Option.bind se comporta como List.pick (conceptualmente Option.bind mapea mejor List.collect pero List.pick podría ser más fácil de entender).

bind, filter y map manejan las rutas de error y g solo contiene el happy código de ruta

Todas las funciones que aceptan directamente Opción<_> y devuelven Opción<_> son directamente componible con |> y >>.

Por lo tanto, ROP aumenta la legibilidad y la composición.

Definición de opción #

Una ‘Opción’ es una unión discriminada con dos casos, ‘Ninguno’ o ‘Algo’.

type Option<'T> = Some of 'T | None

Uso de tipos de opciones de C# #

No es una buena idea exponer los tipos de opción al código de C#, ya que C# no tiene forma de manejarlos. Las opciones son introducir FSharp.Core como una dependencia en su proyecto de C# (que es lo que tendría que hacer si está consumiendo una biblioteca de F# no diseñada para la interoperabilidad con C#), o cambiar Ninguno valores a null.

Pre-F # 4.0 #

La forma de hacerlo es crear una función de conversión propia:

let OptionToObject opt =
    match opt with
    | Some x -> x 
    | None -> null

Para los tipos de valores, tendría que recurrir a encuadrarlos o usar System.Nullable.

let OptionToNullable x = 
    match x with 
    | Some i -> System.Nullable i
    | None -> System.Nullable ()

#F#4.0

En F# 4.0, las funciones ofObj, toObj, ofNullable y toNullable se introdujeron en el módulo Option. En F# interactivo se pueden utilizar de la siguiente manera:

let l1 = [ Some 1 ; None ; Some 2]
let l2 = l1 |> List.map Option.toNullable;;

// val l1 : int option list = [Some 1; null; Some 2]
// val l2 : System.Nullable<int> list = [1; null; 2]

let l3 = l2 |> List.map Option.ofNullable;;
// val l3 : int option list = [Some 1; null; Some 2]

// Equality
l1 = l2 // fails to compile: different types
l1 = l3 // true

Tenga en cuenta que Ninguno se compila a null internamente. Sin embargo, en lo que respecta a F #, es Ninguno.

let lA = [Some "a"; None; Some "b"]
let lB = lA |> List.map Option.toObj

// val lA : string option list = [Some "a"; null; Some "b"]
// val lB : string list = ["a"; null; "b"]

let lC = lB |> List.map Option.ofObj
// val lC : string option list = [Some "a"; null; Some "b"]

// Equality
lA = lB // fails to compile: different types
lA = lC // true
