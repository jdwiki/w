SQL Server Evolution a través de diferentes versiones (2000 - 2016)

  • Servidor SQL Versión 2000 - 2016

En esta página

  • Servidor SQL Versión 2000 - 2016

Uso SQL Server desde 2004. Comencé con 2000 y ahora voy a usar SQL Server 2016. Creé tablas, vistas, funciones, activadores, procedimientos almacenados y escribí muchas consultas SQL, pero no usé muchas características nuevas de posteriores. versiones. Lo busqué en Google, pero desafortunadamente no encontré todas las características en un solo lugar. Así que recopilé y validé esta información de diferentes fuentes y la puse aquí. Solo estoy agregando la información de alto nivel para todas las versiones a partir de 2000 a 20

Servidor SQL Versión 2000 - 2016 #

Las siguientes características agregadas en SQL Server 2000 de su versión anterior:

  1. Se agregaron nuevos tipos de datos (BIGINT, SQL_VARIANT, TABLE)
  2. En lugar de y para Triggers se introdujeron como avance a DDL.
  3. Integridad referencial en cascada.
  4. Soporte XML
  5. Funciones definidas por el usuario y vistas de partición.
  6. Vistas indexadas (permitiendo índice en vistas con columnas calculadas).

Las siguientes características añadidas en la versión 2005 de su versión anterior:

  1. Ampliación en cláusula TOP con opción “CON LAZOS”.
  2. Comandos de manipulación de datos (DML) y cláusula OUTPUT para obtener valores INSERTADOS y ELIMINADOS
  3. Los operadores PIVOT y UNPIVOT.
  4. Manejo de excepciones con bloque TRY/CATCH
  5. Funciones de clasificación
  6. Expresiones de tabla comunes (CTE)
  7. Common Language Runtime (Integración de lenguajes .NET para crear objetos como procedimientos almacenados, disparadores, funciones, etc.)
  8. Service Broker (manejo de mensajes entre un remitente y un receptor de manera débilmente acoplada)
  9. Cifrado de datos (capacidades nativas para admitir el cifrado de datos almacenados en bases de datos definidas por el usuario)
  10. Correo SMTP
  11. Puntos finales HTTP (creación de puntos finales usando una declaración T-SQL simple que expone un objeto para acceder a través de Internet)
  12. Conjuntos de resultados activos múltiples (MARS). Esto permite que una conexión de base de datos persistente desde un solo cliente tenga más de una solicitud activa por conexión.
  13. Servicios de integración de SQL Server (se utilizará como herramienta ETL (extracción, transformación y carga) principal
  14. Mejoras en Analysis Services y Reporting Services.
  15. Particionamiento de tablas e índices. Permite la partición de tablas e índices según los límites de partición especificados por una FUNCIÓN DE PARTICIÓN con particiones individuales asignadas a grupos de archivos a través de un ESQUEMA DE PARTICIÓN.

Las siguientes características añadidas en la versión 2008 de su versión anterior:

  1. Mejora en los tipos de datos DATE y TIME existentes
  2. Nuevas funciones como – SYSUTCDATETIME() y SYSDATETIMEOFFSET()
  3. Columnas de repuesto: para ahorrar una cantidad significativa de espacio en disco.
  4. Tipos grandes definidos por el usuario (hasta 2 GB de tamaño)
  5. Introdujo una nueva función para pasar un tipo de datos de tabla a funciones y procedimientos almacenados
  6. Nuevo comando MERGE para operaciones INSERT, UPDATE y DELETE
  7. Nuevo tipo de datos HierarchyID
  8. Tipos de datos espaciales: para representar la ubicación física y la forma de cualquier objeto geométrico.
  9. Consultas e informes más rápidos con GROUPING SETS: una extensión de la cláusula GROUP BY.
  10. Mejora de la opción de almacenamiento FILESTREAM

Las siguientes características añadidas en la versión 2008 R2 de su versión anterior:

  1. PowerPivot: para procesar grandes conjuntos de datos.
  2. Generador de informes 3.0
  3. Listo para la nube
  4. StreamInsight
  5. Servicios de datos maestros
  6. Integración de SharePoint
  7. DACPAC (paquetes de componentes de aplicación de nivel de datos)
  8. Mejoras en otras funciones de SQL Server 2008

Las siguientes características añadidas en la versión 2012 de su versión anterior:

  1. Índices de almacenamiento de columnas: reduce la E/S y la utilización de la memoria en consultas grandes.
  2. Paginación: la paginación se puede realizar mediante los comandos “OFFSET” y “FETCH”.
  3. Base de datos contenida: excelente función para migraciones periódicas de datos.
  4. Grupos de disponibilidad AlwaysOn
  5. Compatibilidad con el núcleo del servidor de Windows
  6. Roles de servidor definidos por el usuario
  7. Soporte de grandes datos
  8. Power View
  9. Mejoras de SQL Azure
  10. Modelo tabular (SSAS)
  11. Servicios de calidad de datos DQS
  12. Tabla de archivos: una mejora de la función FILESTREAM que se introdujo en 2008.
  13. Mejora en el manejo de errores, incluida la declaración THROW
  14. Mejora en la depuración de SQL Server Management Studio una. SQL Server 2012 presenta más opciones para controlar los puntos de interrupción. b. Mejoras en las ventanas del modo de depuración C. Mejora en IntelliSense, como la inserción de fragmentos de código.

Las siguientes características añadidas en la versión 2014 de su versión anterior:

  1. Motor OLTP en memoria: mejora el rendimiento hasta 20 veces.
  2. Mejoras de AlwaysOn
  3. Ampliación de la reserva de almacenamiento intermedio
  4. Características de la nube híbrida
  5. Mejora en los índices de almacenamiento de columnas (como los índices de almacenamiento de columnas actualizables)
  6. Mejoras en el manejo de consultas (como SELECT INTO paralelo)
  7. Integración de Power BI para Office 365
  8. Durabilidad retardada
  9. Mejoras para las copias de seguridad de la base de datos

Las siguientes características añadidas en la versión 2016 de su versión anterior:

  1. Always Encrypted: Always Encrypted está diseñado para proteger los datos en reposo o en movimiento.
  2. Análisis operativo en tiempo real
  3. PolyBase en SQL Server
  4. Soporte JSON nativo
  5. Almacén de consultas
  6. Mejoras a AlwaysOn
  7. OLTP en memoria mejorado
  8. Múltiples archivos de base de datos TempDB
  9. Ampliar base de datos
  10. Seguridad a nivel de fila
  11. Mejoras en memoria

Mejoras de T-SQL o nuevas incorporaciones en SQL Server 2016

  1. TABLA TRUNCADA con PARTICIÓN

  2. ABANDONAR SI EXISTE

  3. Funciones STRING_SPLIT y STRING_ESCAPE

  4. ALTER TABLE ahora puede modificar muchas columnas mientras la tabla permanece en línea, usando WITH (ONLINE = ON | OFF).

  5. MAXDOP para DBCC CHECKDB, DBCC CHECKTABLE y DBCC CHECKFILEGROUP

  6. ALTERAR CONJUNTO DE BASE DE DATOS AUTOGROW_SINGLE_FILE

  7. ALTERAR CONJUNTO DE BASE DE DATOS AUTOGROW_ALL_FILES

  8. Funciones COMPRIMIR y DESCOMPRIMIR

  9. Declaración FORMATO MENSAJE

  10. 2016 introduce 8 propiedades más con SERVERPROPERTY

una. InstanceDefaultDataPath

b. InstanceDefaultLogPath

C. Construcción del producto

d. ProductBuildType

mi. ProductMajorVersion

F. Versión secundaria del producto

gramo. Nivel de actualización de producto

H. ProductUpdateReference
