SQL Server 2016 presenta soporte para tablas temporales versionadas por el sistema como una función de base de datos que brinda soporte integrado para proporcionar información sobre los datos almacenados en la tabla en cualquier momento en lugar de solo los datos que son correctos en el momento actual.

Una tabla temporal versionada por el sistema es un nuevo tipo de tabla de usuario en SQL Server 2016, diseñada para mantener un historial completo de los cambios de datos y permitir un fácil análisis de un punto en el tiempo. Este tipo de tabla temporal se denomina tabla temporal versionada por el sistema porque el sistema (es decir, el motor de la base de datos) administra el período de validez de cada fila. Cada tabla temporal tiene dos columnas definidas explícitamente, cada una con un tipo de datos datetime2. Estas columnas se denominan columnas de período. Estas columnas de período son utilizadas exclusivamente por el sistema para registrar el período de validez de cada fila cada vez que se modifica una fila.

CREAR tablas temporales #

CREATE TABLE dbo.Employee   
(    
  [EmployeeID] int NOT NULL PRIMARY KEY CLUSTERED   
  , [Name] nvarchar(100) NOT NULL  
  , [Position] varchar(100) NOT NULL   
  , [Department] varchar(100) NOT NULL  
  , [Address] nvarchar(1024) NOT NULL  
  , [AnnualSalary] decimal (10,2) NOT NULL  
  , [ValidFrom] datetime2 (2) GENERATED ALWAYS AS ROW START  
  , [ValidTo] datetime2 (2) GENERATED ALWAYS AS ROW END  
  , PERIOD FOR SYSTEM_TIME (ValidFrom, ValidTo)  
 )    
 WITH (SYSTEM_VERSIONING = ON (HISTORY_TABLE = dbo.EmployeeHistory));

INSERTOS: En un INSERTO, el sistema establece el valor de la columna ValidFrom en la hora de inicio de la transacción actual (en la zona horaria UTC) según el reloj del sistema y asigna el valor para la columna ValidTo hasta el valor máximo de 9999-12-31. Esto marca la fila como abierta.

ACTUALIZACIONES: En una ACTUALIZACIÓN, el sistema almacena el valor anterior de la fila en la tabla de historial y establece el valor de la columna ValidTo en la hora de inicio de la transacción actual (en el zona horaria UTC) según el reloj del sistema. Esto marca la fila como cerrada, con un período registrado durante el cual la fila era válida. En la tabla actual, la fila se actualiza con su nuevo valor y el sistema establece el valor de la columna ValidFrom en la hora de inicio de la transacción (en la zona horaria UTC) según el reloj del sistema. El valor de la fila actualizada en la tabla actual para la columna ValidTo sigue siendo el valor máximo de 9999-12-31.

ELIMINACIÓN: en una ELIMINACIÓN, el sistema almacena el valor anterior de la fila en la tabla de historial y establece el valor de la columna ValidTo en la hora de inicio de la transacción actual (en el zona horaria UTC) según el reloj del sistema. Esto marca la fila como cerrada, con un período registrado para el cual la fila anterior era válida. En la tabla actual, la fila se elimina. Las consultas de la tabla actual no devolverán esta fila. Solo las consultas que tratan con datos de historial devuelven datos para los que se cierra una fila.

COMBINACIÓN: en una COMBINACIÓN, la operación se comporta exactamente como si se ejecutaran hasta tres instrucciones (una INSERCIÓN, una ACTUALIZACIÓN y/o una ELIMINACIÓN). , según lo que se especifique como acciones en la sentencia MERGE.

Consejo: Las horas registradas en las columnas datetime2 del sistema se basan en la hora de inicio de la transacción en sí. Por ejemplo, todas las filas insertadas dentro de una sola transacción tendrán la misma hora UTC registrada en la columna correspondiente al inicio del período SYSTEM_TIME.

PARA SYSTEM_TIME TODO #

Devuelve la unión de filas que pertenecen a la tabla actual y la histórica.

SELECT * FROM Employee
    FOR SYSTEM_TIME ALL

Creación de una tabla temporal con versión del sistema optimizada para memoria y limpieza de la tabla de historial de SQL Server #

La creación de una tabla temporal con una tabla de historial predeterminada es una opción conveniente cuando desea controlar los nombres y seguir confiando en el sistema para crear una tabla de historial con la configuración predeterminada. En el siguiente ejemplo, una nueva tabla temporal optimizada para memoria con versión del sistema vinculada a una nueva tabla de historial basada en disco.

CREATE SCHEMA History  
GO  
CREATE TABLE dbo.Department   
(  
    DepartmentNumber char(10) NOT NULL PRIMARY KEY NONCLUSTERED,   
    DepartmentName varchar(50) NOT NULL,   
    ManagerID int  NULL,   
    ParentDepartmentNumber char(10) NULL,   
    SysStartTime datetime2 GENERATED ALWAYS AS ROW START HIDDEN NOT NULL,   
    SysEndTime datetime2 GENERATED ALWAYS AS ROW END HIDDEN NOT NULL,     
    PERIOD FOR SYSTEM_TIME (SysStartTime,SysEndTime)     
)  
WITH   
    (  
        MEMORY_OPTIMIZED = ON, DURABILITY = SCHEMA_AND_DATA,  
        SYSTEM_VERSIONING = ON ( HISTORY_TABLE = History.DepartmentHistory )   
    );

Limpieza de la tabla de historial de SQL Server Con el tiempo, la tabla de historial puede crecer significativamente. Dado que no se permite insertar, actualizar o eliminar datos de la tabla de historial, la única forma de limpiar la tabla de historial es primero deshabilitar el control de versiones del sistema:

ALTER TABLE dbo.Employee

ESTABLECER (SYSTEM_VERSIONING = OFF); VAMOS

Eliminar datos innecesarios de la tabla de historial:

    DELETE FROM dbo.EmployeeHistory

DONDE Hora de finalización <= ‘2017-01-26 14:00:29’;

y luego vuelva a habilitar el control de versiones del sistema:

ALTER TABLE dbo.Employee

SET (SYSTEM_VERSIONING = ON (HISTORY_TABLE = [dbo].[EmployeeHistory], DATA_CONSISTENCY_CHECK = ON));

La limpieza de la tabla de historial en las bases de datos de Azure SQL es un poco diferente, ya que las bases de datos de Azure SQL tienen compatibilidad integrada para la limpieza de la tabla de historial. En primer lugar, la limpieza de la retención del historial temporal debe estar habilitada en el nivel de la base de datos:

ALTER DATABASE CURRENT

ESTABLECER TEMPORAL_HISTORY_RETENTION EN VAMOS

Luego establezca el período de retención por tabla:

ALTER TABLE dbo.Employee

SET (SYSTEM_VERSIONING = ON (HISTORY_RETENTION_PERIOD = 90 DAYS));

Esto eliminará todos los datos de la tabla de historial que tengan más de 90 días. Las bases de datos locales de SQL Server 2016 no admiten TEMPORAL_HISTORY_RETENTION y HISTORY_RETENTION_PERIOD y cualquiera de las dos consultas anteriores se ejecuta en las bases de datos locales de SQL Server 2016, se producirán los siguientes errores.

Para TEMPORAL_HISTORY_RETENTION el error será:

Msg 102, Level 15, State 6, Line 34

Sintaxis incorrecta cerca de ‘TEMPORAL_HISTORY_RETENTION’.

Para HISTORY_RETENTION_PERIOD el error será:

Msg 102, Level 15, State 1, Line 39

Sintaxis incorrecta cerca de ‘HISTORY_RETENTION_PERIOD’.

¿Cómo consulto datos temporales? #

SELECT * FROM Employee   
    FOR SYSTEM_TIME    
        BETWEEN '2014-01-01 00:00:00.0000000' AND '2015-01-01 00:00:00.0000000'   
            WHERE EmployeeID = 1000 ORDER BY ValidFrom;

Devuelve el valor real especificado en un punto en el tiempo (PARA SYSTEM_TIME A PARTIR DE <fecha_hora>) #

Devuelve una tabla con filas que contienen los valores que eran reales (actuales) en el momento especificado en el pasado.

SELECT * FROM Employee   
    FOR SYSTEM_TIME AS  OF '2016-08-06 08:32:37.91'

PARA SYSTEM_TIME ENTRE <start_date_time> Y <end_date_time> #

Igual que arriba en la descripción FOR SYSTEM_TIME FROM <start_date_time>TO <end_date_time>, excepto que la tabla de filas devuelta incluye filas que se activaron en el límite superior definido por el extremo <end_date_time>.

SELECT * FROM Employee   
    FOR SYSTEM_TIME BETWEEN  '2015-01-01' AND '2015-12-31'

PARA SYSTEM_TIME DESDE <start_date_time> HASTA <end_date_time> #

Devuelve una tabla con los valores de todas las versiones de fila que estaban activas dentro del intervalo de tiempo especificado, independientemente de si comenzaron a estar activas antes del valor del parámetro <start_date_time> para el argumento FROM o dejaron de estar activas después del valor del parámetro <end_date_time> para el AL argumento. Internamente, se realiza una unión entre la tabla temporal y su tabla de historial y los resultados se filtran para devolver los valores de todas las versiones de fila que estuvieron activas en cualquier momento durante el intervalo de tiempo especificado. Las filas que se activaron exactamente en el límite inferior definido por el extremo DESDE se incluyen y los registros que se activaron exactamente en el límite superior definido por el extremo TO no se incluyen.

SELECT * FROM Employee   
    FOR SYSTEM_TIME FROM '2015-01-01' TO '2015-12-31'

PARA SYSTEM_TIME CONTENIDO EN (<start_date_time> , <end_date_time>) #

Devuelve una tabla con los valores de todas las versiones de fila que se abrieron y cerraron dentro del rango de tiempo especificado definido por los dos valores de fecha y hora para el argumento CONTENIDO EN. Se incluyen las filas que se activaron exactamente en el límite inferior o dejaron de estar activas exactamente en el límite superior.

SELECT * FROM Employee
    FOR SYSTEM_TIME CONTAINED IN ('2015-04-01', '2015-09-25')
