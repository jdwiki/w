Dado que los servicios web más seguros evitan almacenar contraseñas en formato de texto sin formato, los lenguajes como PHP proporcionan varias funciones hash (no descifrables) para admitir el estándar industrial más seguro. Este tema proporciona documentación para el hash adecuado con PHP.

Sintaxis #

  • cadena contraseña_hash (cadena $contraseña, entero $algo [, matriz $opciones])
  • contraseña booleana_verificar (cadena $contraseña, cadena $hash)
  • boolean password_needs_rehash (string $hash, integer $algo [, array $options])
  • matriz contraseña_get_info (cadena $hash)

Antes de PHP 5.5, puede usar [el paquete de compatibilidad] (https://github.com/ircmaxell/password_compat) para proporcionar las funciones password_*. Se recomienda encarecidamente que utilice el paquete de compatibilidad si puede hacerlo.

Con o sin el paquete de compatibilidad, la funcionalidad correcta de Bcrypt a través de crypt() se basa en PHP 5.3.7+ de lo contrario, debe restringir las contraseñas a ASCII- solo conjuntos de caracteres.

Nota: Si usa PHP 5.5 o anterior, está usando una [versión no compatible de PHP] (http://php.net/supported-versions.php) que ya no recibe actualizaciones de seguridad. Actualice lo antes posible, puede actualizar los hashes de su contraseña después.

Selección de algoritmo #

Algoritmos seguros #

  • bcrypt es su mejor opción siempre y cuando utilice la ampliación de claves para aumentar el tiempo de cálculo de hash, ya que hace que los [ataques de fuerza bruta sean extremadamente lentos] (http://arstechnica.com/security/2015/08/cracking- todas-las-contraseñas-hackeadas-de-ashley-madison-podrían-tomar-una-vida/).
  • argon2 es otra opción que estará disponible en PHP 7.2.

Algoritmos inseguros #

Los siguientes algoritmos hash son inseguros o inadecuados para su propósito y, por lo tanto, no deben usarse. Nunca fueron adecuados para el hash de contraseñas, ya que están diseñados para resúmenes rápidos en lugar de hash de contraseñas lentos y difíciles de usar por fuerza bruta.

Si usa alguno de ellos, incluidas las sales, debe cambiar a uno de los algoritmos seguros recomendados tan pronto como sea posible.

Algoritmos considerados inseguros:

  • MD4 - ataque de colisión encontrado en 1995
  • MD5 - ataque de colisión encontrado en 2005
  • SHA-1 - ataque de colisión demostrado en 2015

Algunos algoritmos se pueden usar de forma segura como algoritmos de resumen de mensajes para probar la autenticidad, pero nunca como algoritmos de hash de contraseñas:

  • SHA-2
  • SHA-3

Tenga en cuenta que los hash fuertes como SHA256 y SHA512 son sólidos e ininterrumpidos; sin embargo, generalmente es más seguro usar las funciones hash bcrypt o argon2, ya que los ataques de fuerza bruta contra estos algoritmos son mucho más difíciles para las computadoras clásicas.

Crear un hash de contraseña #

Cree hashes de contraseña usando password_hash() para usar el hash estándar actual de mejores prácticas de la industria o la derivación de claves. Al momento de escribir, el estándar es bcrypt, lo que significa que PASSWORD_DEFAULT contiene el mismo valor que PASSWORD_BCRYPT.

$options = [
    'cost' => 12,
];

$hashedPassword = password_hash($plaintextPassword, PASSWORD_DEFAULT, $options);

El tercer parámetro no es obligatorio.

El valor de 'costo' debe elegirse en función del hardware de su servidor de producción. Aumentarlo hará que la contraseña sea más costosa de generar. Cuanto más costoso sea generarlo, más tiempo le llevará a cualquiera que intente descifrarlo generarlo también. Idealmente, el costo debería ser lo más alto posible, pero en la práctica debe establecerse de manera que no ralentice demasiado todo. En algún lugar entre 0,1 y 0,4 segundos estaría bien. Utilice el valor predeterminado si tiene dudas.

En PHP anterior a 5.5.0, las funciones password_* no están disponibles. Debería usar el paquete de compatibilidad para sustituir esas funciones. Tenga en cuenta que el paquete de compatibilidad requiere PHP 5.3.7 o superior o una versión que tenga la corrección $2y retroportada (como la que proporciona RedHat).

Si no puede usarlas, puede implementar el hashing de contraseñas con crypt() Como password_hash() se implementa como un envoltorio alrededor de la función crypt(), no necesita perder ninguna funcionalidad.

// this is a simple implementation of a bcrypt hash otherwise compatible
// with `password_hash()`
// not guaranteed to maintain the same cryptographic strength of the full `password_hash()`
// implementation

// if `CRYPT_BLOWFISH` is 1, that means bcrypt (which uses blowfish) is available
// on your system
if (CRYPT_BLOWFISH == 1) {
    $salt = mcrypt_create_iv(16, MCRYPT_DEV_URANDOM);
    $salt = base64_encode($salt);
    // crypt uses a modified base64 variant
    $source = 'ABCDEFGHIJKLMNOPQRSTUVWXYZabcdefghijklmnopqrstuvwxyz0123456789+/';
    $dest = './ABCDEFGHIJKLMNOPQRSTUVWXYZabcdefghijklmnopqrstuvwxyz0123456789';
    $salt = strtr(rtrim($salt, '='), $source, $dest);
    $salt = substr($salt, 0, 22);
    // `crypt()` determines which hashing algorithm to use by the form of the salt string
    // that is passed in
    $hashedPassword = crypt($plaintextPassword, '$2y$10$'.$salt.'$');
}

Sal para hash de contraseña #

A pesar de la confiabilidad del algoritmo de cifrado, todavía existe una vulnerabilidad contra [tablas de arco iris] 5. Esa es la razón por la que se recomienda usar sal.

Un salt es algo que se agrega a la contraseña antes de hacer hash para hacer que la cadena de origen sea única. Dadas dos contraseñas idénticas, los hashes resultantes también serán únicos, porque sus sales son únicas.

Una sal aleatoria es una de las piezas más importantes de la seguridad de su contraseña. Esto significa que incluso con una tabla de búsqueda de hash de contraseñas conocidas, un atacante no puede hacer coincidir el hash de contraseña de su usuario con los hash de contraseña de la base de datos, ya que se ha utilizado una sal aleatoria. Debe utilizar sales siempre aleatorias y criptográficamente seguras. Leer más

Con el algoritmo password_hash() bcrypt, el salt de texto sin formato se almacena junto con el hash resultante, lo que significa que el hash puede transferirse a través de diferentes sistemas y plataformas y aun así compararse con la contraseña original.

Incluso cuando esto no se recomienda, puede usar la opción salt para definir su propia sal aleatoria.

 $options = [
        'salt' => $salt, //see example below
 ];

Importante. Si omite esta opción, password_hash() generará un salt aleatorio para cada contraseña con hash. Este es el modo de funcionamiento previsto.

La opción salt ha quedado obsoleta a partir de PHP 7.0.0. Ahora se prefiere simplemente usar la sal que se genera de forma predeterminada.

Determine si un hash de contraseña existente se puede actualizar a un algoritmo más fuerte #

Si está utilizando el método PASSWORD_DEFAULT para permitir que el sistema elija el mejor algoritmo para codificar sus contraseñas, ya que el valor predeterminado aumenta en fuerza, es posible que desee volver a codificar las contraseñas antiguas a medida que los usuarios inician sesión

<?php
// first determine if a supplied password is valid
if (password_verify($plaintextPassword, $hashedPassword)) {

    // now determine if the existing hash was created with an algorithm that is
    // no longer the default
    if (password_needs_rehash($hashedPassword, PASSWORD_DEFAULT)) {

        // create a new hash with the new default
        $newHashedPassword = password_hash($plaintextPassword, PASSWORD_DEFAULT);

        // and then save it to your data store
        //$db->update(...);
    }
}
?>

Si las funciones de contraseña_* no están disponibles en su sistema (y no puede usar el paquete de compatibilidad vinculado en los comentarios a continuación), puede determinar el algoritmo y usarlo para crear el hash original en un método similar al siguiente:

<?php
if (substr($hashedPassword, 0, 4) == '$2y$' && strlen($hashedPassword) == 60) {
    echo 'Algorithm is Bcrypt';
    // the "cost" determines how strong this version of Bcrypt is
    preg_match('/\$2y\$(\d+)\$/', $hashedPassword, $matches);
    $cost = $matches[1];
    echo 'Bcrypt cost is '.$cost;
}
?>

Verificación de una contraseña contra un hash #

password_verify() es la función incorporada proporcionada (a partir de PHP 5.5) para verificar la validez de una contraseña contra un hash conocido.

<?php
if (password_verify($plaintextPassword, $hashedPassword)) {
    echo 'Valid Password';
}
else {
    echo 'Invalid Password.';
}
?>

Todos los algoritmos hash admitidos almacenan información que identifica qué hash se usó en el propio hash, por lo que no es necesario indicar qué algoritmo está usando para codificar la contraseña de texto sin formato.

Si las funciones de contraseña_* no están disponibles en su sistema (y no puede usar el paquete de compatibilidad vinculado en los comentarios a continuación), puede implementar la verificación de contraseña con la función crypt(). Tenga en cuenta que se deben tomar precauciones específicas para evitar [ataques de tiempo] (https://en.wikipedia.org/wiki/Timing_attack).

<?php
// not guaranteed to maintain the same cryptographic strength of the full `password_hash()`
// implementation
if (CRYPT_BLOWFISH == 1) {
    // `crypt()` discards all characters beyond the salt length, so we can pass in
    // the full hashed password
    $hashedCheck = crypt($plaintextPassword, $hashedPassword);

    // this a basic constant-time comparison based on the full implementation used
    // in `password_hash()`
    $status = 0;
    for ($i=0; $i<strlen($hashedCheck); $i++) {
        $status |= (ord($hashedCheck[$i]) ^ ord($hashedPassword[$i]));
    }

    if ($status === 0) {
        echo 'Valid Password';
    }
    else {
        echo 'Invalid Password';
    }
}
?>
