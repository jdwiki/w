Multiprocesamiento usando funciones de bifurcación integradas #

Puede usar funciones integradas para ejecutar procesos PHP como bifurcaciones. Esta es la forma más sencilla de lograr un trabajo paralelo si no necesita que sus subprocesos se comuniquen entre sí.

Esto le permite asignar tareas que consumen mucho tiempo (como cargar un archivo a otro servidor o enviar un correo electrónico) a otro hilo para que su secuencia de comandos se cargue más rápido y pueda usar múltiples núcleos, pero tenga en cuenta que esto no es multihilo real y su hilo principal no lo hará. saber lo que los niños están haciendo.

Tenga en cuenta que en Windows esto hará que aparezca otro símbolo del sistema para cada bifurcación que inicie.

maestro.php

$cmd = "php worker.php 10";
if(strtoupper(substr(PHP_OS, 0, 3)) === 'WIN') // for windows use popen and pclose
{
    pclose(popen($cmd,"r"));
}
else //for unix systems use shell exec with "&" in the end
{
    exec('bash -c "exec nohup setsid '.$cmd.' > /dev/null 2>&1 &"');
}

trabajador.php

//send emails, upload files, analyze logs, etc
$sleeptime = $argv[1];
sleep($sleeptime);

Creando un proceso hijo usando un tenedor #

PHP ha incorporado la función pcntl_fork para crear procesos secundarios. pcntl_fork es lo mismo que fork en Unix. No toma ningún parámetro y devuelve un número entero que se puede usar para diferenciar entre el proceso principal y el secundario. Considere el siguiente código para la explicación

<?php
    // $pid is the PID of child
    $pid = pcntl_fork();
    if ($pid == -1) {
         die('Error while creating child process');
    } else if ($pid) {
         // Parent process
    } else {
         // Child process
    }
?>

Como puede ver, -1 es un error en la bifurcación y el niño no se creó. En la creación del niño, tenemos dos procesos ejecutándose con PID separados.

Otra consideración aquí es un ‘proceso zombi’ o ‘proceso difunto’ cuando el proceso principal finaliza antes que el proceso secundario. Para evitar un proceso secundario zombie, simplemente agregue pcntl_wait($status) al final del proceso principal.

pnctl_wait suspende la ejecución del proceso principal hasta que el proceso secundario haya salido.

También vale la pena señalar que el ‘proceso zombie’ no se puede eliminar usando la señal ‘SIGKILL’.

Comunicación entre procesos #

La comunicación entre procesos permite a los programadores comunicarse entre diferentes procesos. Por ejemplo, consideremos que necesitamos escribir una aplicación PHP que pueda ejecutar comandos bash e imprimir la salida. Usaremos proc_open, que ejecutará el comando y devolverá un recurso con el que podemos comunicarnos. El siguiente código muestra una implementación básica que ejecuta pwd en bash desde php

<?php
    $descriptor = array(
        0 => array("pipe", "r"), // pipe for stdin of child
        1 => array("pipe", "w"), // pipe for stdout of child
    );
    $process = proc_open("bash", $descriptor, $pipes);
    if (is_resource($process)) {
        fwrite($pipes[0], "pwd" . "\n");
        fclose($pipes[0]);
        echo stream_get_contents($pipes[1]);
        fclose($pipes[1]);
        $return_value = proc_close($process);

    }
?>

proc_open ejecuta el comando bash con $descriptor como especificaciones del descriptor. Después de eso, usamos is_resource para validar el proceso. Una vez hecho esto, podemos comenzar a interactuar con el proceso secundario usando $tuberías que se genera de acuerdo con las especificaciones del descriptor.

Después de eso, simplemente podemos usar fwrite para escribir en la entrada estándar del proceso secundario. En este caso pwd seguido de retorno de carro. Finalmente, stream_get_contents se usa para leer la salida estándar del proceso secundario.

Recuerde siempre cerrar el proceso secundario usando proc_close() que terminará el proceso secundario y devolverá el código de estado de salida.
