Despliegue de ventana acoplable

Docker es una solución de contenedor muy popular que se usa ampliamente para implementar código en entornos de producción. Hace que sea más fácil Administrar y Escalar microservicios y aplicaciones web.

Este documento asume que Docker está instalado y que el demonio se está ejecutando. Puede consultar Instalación de Docker para comprobar cómo instalarlo.

Obtener la imagen de la ventana acoplable para php #

Para implementar la aplicación en la ventana acoplable, primero debemos obtener la imagen del registro.

docker pull php

Esto le dará la última versión de la imagen del repositorio oficial de php. En términos generales, ‘PHP’ se usa generalmente para implementar aplicaciones web, por lo que necesitamos un servidor http para acompañar la imagen. La imagen php:7.0-apache viene preinstalada con apache para facilitar la implementación.

Escribiendo un archivo docker #

Dockerfile se usa para configurar la imagen personalizada que construiremos con los códigos de la aplicación web. Cree un nuevo archivo Dockerfile en la carpeta raíz del proyecto y luego coloque los siguientes contenidos en el mismo

FROM php:7.0-apache
COPY /etc/php/php.ini /usr/local/etc/php/
COPY . /var/www/html/
EXPOSE 80

La primera línea es bastante sencilla y se usa para describir qué imagen se debe usar para crear una nueva imagen. Lo mismo podría cambiarse a cualquier otra versión específica de PHP desde el registro.

La segunda línea es simplemente subir el archivo php.ini a nuestra imagen. Siempre puede cambiar ese archivo a alguna otra ubicación de archivo personalizada.

La tercera línea copiaría los códigos en el directorio actual a /var/www/html, que es nuestra webroot. Recuerda /var/www/html dentro de la imagen.

La última línea simplemente abriría el puerto 80 dentro del contenedor docker.

Ignorar archivos #

En algunos casos, puede haber algunos archivos que no desee en el servidor, como la configuración del entorno, etc. Supongamos que tenemos nuestro entorno en .env. Ahora, para ignorar este archivo, simplemente podemos agregarlo a .dockerignore en la carpeta raíz de nuestro código base.

Imagen del edificio #

Construir una imagen no es algo específico de php, pero para construir la imagen que describimos arriba, simplemente podemos usar

docker build -t <Image name> .

Una vez que se construye la imagen, puede verificar la misma usando

docker images

Lo que enumeraría todas las imágenes instaladas en su sistema.

Inicio del contenedor de la aplicación #

Una vez que tenemos una imagen lista, podemos empezar y servir la misma. Para crear un contenedor a partir de la imagen, utilice

docker run -p 80:80 -d <Image name>

En el comando anterior -p 80:80 reenviaría el puerto 80 de su servidor al puerto 80 del contenedor. El indicador -d indica que el contenedor debe ejecutarse como trabajo en segundo plano. El final especifica qué imagen debe usarse para construir el contenedor.

Comprobación del contenedor #

Para verificar los contenedores en ejecución, simplemente use

docker ps

Esto mostrará una lista de todos los contenedores que se ejecutan en el demonio docker.

Registros de aplicaciones #

Los registros son muy importantes para depurar la aplicación. Con el fin de comprobar en ellos el uso

docker logs <Container id>
