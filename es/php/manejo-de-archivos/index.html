Sintaxis #

  • int readfile (cadena $nombre de archivo [, bool $use_include_path = false [, recurso $contexto]])

Parámetros #

Parámetro Descripción
nombre de archivo El nombre del archivo que se lee.
use_include_path Puede usar el segundo parámetro opcional y establecerlo en TRUE, si también desea buscar el archivo en include_path.
contexto Un recurso de flujo de contexto.

Sintaxis del nombre de archivo #

La mayoría de los nombres de archivo pasados ​​a funciones en este tema son:

  1. Cuerdas en la naturaleza.
  • Los nombres de archivo se pueden pasar directamente. Si se pasan valores de otros tipos, se convierten en cadena. Esto es especialmente útil con SplFileInfo, que es el valor en la iteración de DirectoryIterator.
  1. Relativo o absoluto.
  • Pueden ser absolutos. En sistemas similares a Unix, las rutas absolutas comienzan con /, p. /home/user/file.txt, mientras que en Windows, las rutas absolutas comienzan con la unidad, p. C:/Usuarios/usuario/archivo.txt
  • También pueden ser relativos, lo que depende del valor de getcwd y está sujeto a cambios por parte de chdir.
  1. Aceptar protocolos.
  • Pueden comenzar con scheme:// para especificar el contenedor de protocolo con el que administrar. Por ejemplo, file_get_contents("http://example.com") recupera contenido de http://example.com.
  1. Compatible con barras.
  • Mientras que DIRECTORY_SEPARATOR en Windows es una barra diagonal inversa, y el sistema devuelve barras diagonales inversas para las rutas de forma predeterminada, el desarrollador aún puede usar / como separador de directorio. Por lo tanto, por compatibilidad, los desarrolladores pueden usar / como separadores de directorio en todos los sistemas, pero tenga en cuenta que los valores devueltos por las funciones (por ejemplo, realpath) pueden contener barras invertidas.

Eliminación de archivos y directorios #

Borrar archivos #

La función [unlink][unlink] elimina un solo archivo y devuelve si la operación fue exitosa.

$filename = '/path/to/file.txt';

if (file_exists($filename)) {
    $success = unlink($filename);
    
    if (!$success) {
         throw new Exception("Cannot delete $filename");
    }
}

Eliminación de directorios, con eliminación recursiva #

Por otro lado, los directorios deben eliminarse con rmdir. Sin embargo, esta función solo elimina directorios vacíos. Para eliminar un directorio con archivos, elimine primero los archivos de los directorios. Si el directorio contiene subdirectorios, es posible que se necesite recursion.

El siguiente ejemplo analiza archivos en un directorio, elimina archivos/directorios miembros de forma recursiva y devuelve la cantidad de archivos (no directorios) eliminados.

function recurse_delete_dir(string $dir) : int {
    $count = 0;

    // ensure that $dir ends with a slash so that we can concatenate it with the filenames directly
    $dir = rtrim($dir, "/\\") . "/";

    // use dir() to list files
    $list = dir($dir);

    // store the next file name to $file. if $file is false, that's all -- end the loop.
    while(($file = $list->read()) !== false) {
        if($file === "." || $file === "..") continue;
        if(is_file($dir . $file)) {
            unlink($dir . $file);
            $count++;
        } elseif(is_dir($dir . $file)) {
            $count += recurse_delete_dir($dir . $file);
        }
    }

    // finally, safe to delete directory!
    rmdir($dir);

    return $count;
}

Funciones de confort #

E/S directa sin procesar #

file_get_contents y file_put_contents brindan la capacidad de leer/escribir desde/hacia un archivo hacia/desde una cadena PHP en una sola llamada.

file_put_contents también se puede usar con el indicador de máscara de bits FILE_APPEND para agregar, en lugar de truncar y sobrescribir, el archivo. Se puede usar junto con la máscara de bits LOCK_EX para adquirir un bloqueo exclusivo para el archivo mientras se procede a escribir. Los indicadores de máscara de bits se pueden unir con el operador OR bit a bit |.

$path = "file.txt";
// reads contents in file.txt to $contents
$contents = file_get_contents($path);
// let's change something... for example, convert the CRLF to LF!
$contents = str_replace("\r\n", "\n", $contents);
// now write it back to file.txt, replacing the original contents
file_put_contents($path, $contents);

FILE_APPEND es útil para agregar archivos de registro, mientras que LOCK_EX ayuda a evitar la condición de carrera de la escritura de archivos de múltiples procesos. Por ejemplo, para escribir en un archivo de registro sobre la sesión actual:

file_put_contents("logins.log", "{$_SESSION["username"]} logged in", FILE_APPEND | LOCK_EX);

#CSVIO fgetcsv($file, $length, $separator)

El fgetcsv analiza la línea del archivo abierto buscando campos csv. Devuelve campos CSV en una matriz en caso de éxito o “FALSO” en caso de error.

De forma predeterminada, leerá solo una línea del archivo CSV.

$file = fopen("contacts.csv","r");
print_r(fgetcsv($file));    
print_r(fgetcsv($file,5," "));
fclose($file);

contactos.csv

Kai Jim, Refsnes, Stavanger, Norway
Hege, Refsnes, Stavanger, Norway

Producción:

Array
(
    [0] => Kai Jim
    [1] => Refsnes
    [2] => Stavanger
    [3] => Norway
)
Array
(
    [0] => Hege,
)

Leyendo un archivo a stdout directamente #

[readfile][readfile] copia un archivo en el búfer de salida. readfile() no presentará ningún problema de memoria, incluso al enviar archivos grandes, por sí solo.

$file = 'monkey.gif';

if (file_exists($file)) {
    header('Content-Description: File Transfer');
    header('Content-Type: application/octet-stream');
    header('Content-Disposition: attachment; filename="'.basename($file).'"');
    header('Expires: 0');
    header('Cache-Control: must-revalidate');
    header('Pragma: public');
    header('Content-Length: ' . filesize($file));
    readfile($file);
    exit;
}

O desde un puntero de archivo #

Alternativamente, para buscar un punto en el archivo para comenzar a copiar a la salida estándar, use fpassthru en su lugar. En el siguiente ejemplo, los últimos 1024 bytes se copian en la salida estándar:

$fh = fopen("file.txt", "rb");
fseek($fh, -1024, SEEK_END); 
fpassthru($fh);

Leer un archivo en una matriz #

[file][file] devuelve las líneas del archivo pasado en una matriz. Cada elemento de la matriz corresponde a una línea en el archivo, con la nueva línea aún adjunta.

print_r(file("test.txt"));

prueba.txt

Welcome to File handling
This is to test file handling

Producción:

Array 
( 
    [0] => Welcome to File handling 
    [1] => This is to test file handling 
)

Obtener información del archivo #

Comprobar si una ruta es un directorio o un archivo #

La función [is_dir][is_dir] devuelve si el argumento es un directorio, mientras que [is_file][is_file] devuelve si el argumento es un archivo. Use [file_exists][file_exists] para comprobar si es así.

$dir  = "/this/is/a/directory";
$file = "/this/is/a/file.txt";

echo is_dir($dir) ? "$dir is a directory" : "$dir is not a directory", PHP_EOL,
    is_file($dir) ? "$dir is a file" : "$dir is not a file", PHP_EOL,
    file_exists($dir) ? "$dir exists" : "$dir doesn't exist", PHP_EOL,
    is_dir($file) ? "$file is a directory" : "$file is not a directory", PHP_EOL,
    is_file($file) ? "$file is a file" : "$file is not a file", PHP_EOL,
    file_exists($file) ? "$file exists" : "$file doesn't exist", PHP_EOL;

Esto da:

/this/is/a/directory is a directory
/this/is/a/directory is not a file
/this/is/a/directory exists
/this/is/a/file.txt is not a directory
/this/is/a/file.txt is a file
/this/is/a/file.txt exists

Comprobando el tipo de archivo #

Use [filetype][filetype] para verificar el tipo de archivo, que puede ser:

  • fifo
  • char
  • dir
  • ‘bloquear’
  • enlace
  • archivo
  • enchufe
  • desconocido

Pasando el nombre de archivo a [filetype][filetype] directamente:

echo filetype("~"); // dir

Tenga en cuenta que filetype devuelve falso y activa una E_WARNING si el archivo no existe.

Comprobación de legibilidad y escritura #

Pasar el nombre del archivo a las funciones [is_writable][is_writable] y [is_readable][is_readable] verifica si el archivo es escribible o legible respectivamente.

Las funciones devuelven falso correctamente si el archivo no existe.

Comprobación de acceso a archivos/hora de modificación #

Usando [filemtime][filemtime] y [fileatime][fileatime] devuelve la marca de tiempo de la última modificación o acceso al archivo. El valor de retorno es una marca de tiempo de Unix; consulte https://www.wikiod.com/es/php/trabajar-con-fechas-y-horas para obtener más detalles.

echo "File was last modified on " . date("Y-m-d", filemtime("file.txt"));
echo "File was last accessed on " . date("Y-m-d", fileatime("file.txt"));

Obtener partes de ruta con información de archivo #

$fileToAnalyze = ('/var/www/image.png');

$filePathParts = pathinfo($fileToAnalyze);

echo '<pre>';
   print_r($filePathParts);
echo '</pre>';

Este ejemplo generará:

Array
(
    [dirname] => /var/www
    [basename] => image.png
    [extension] => png
    [filename] => image
)

Que se puede utilizar como:

$filePathParts['dirname']
$filePathParts['basename']
$filePathParts['extension']
$filePathParts['filename']
Parámetro Detalles
$ruta La ruta completa del archivo a analizar
$opción Una de las cuatro opciones disponibles [PATHINFO_DIRNAME, PATHINFO_BASENAME, PATHINFO_EXTENSION o PATHINFO_FILENAME]
  • Si no se pasa una opción (el segundo parámetro), se devuelve una matriz asociativa; de lo contrario, se devuelve una cadena.
  • No valida que el archivo exista.
  • Simplemente analiza la cadena en partes. No se realiza ninguna validación en el archivo (sin verificación de tipo MIME, etc.)
  • La extensión es simplemente la última extensión de $path La ruta para el archivo image.jpg.png sería .png incluso si técnicamente es un archivo .jpg. Un archivo sin extensión no devolverá un elemento de extensión en la matriz.

tiempo de archivo: http://php.net/tiempo de archivo

Minimice el uso de memoria cuando se trata de archivos grandes #

Si necesitamos analizar un archivo grande, p. un CSV de más de 10 Mbytes que contiene millones de filas, algunos usan las funciones file o file_get_contents y terminan presionando la configuración memory_limit con

Tamaño de memoria permitido de XXXXX bytes agotado

error. Considere la siguiente fuente (top-1m.csv tiene exactamente 1 millón de filas y tiene un tamaño de aproximadamente 22 Mbytes)

var_dump(memory_get_usage(true));
$arr = file('top-1m.csv');
var_dump(memory_get_usage(true));

Esto genera:

int(262144)
int(210501632)

porque el intérprete necesitaba contener todas las filas en la matriz $arr, por lo que consumía ~200 Mbytes de RAM. Tenga en cuenta que ni siquiera hemos hecho nada con el contenido de la matriz.

Ahora considere el siguiente código:

var_dump(memory_get_usage(true));
$index = 1;
if (($handle = fopen("top-1m.csv", "r")) !== FALSE) {
    while (($row = fgetcsv($handle, 1000, ",")) !== FALSE) {
        file_put_contents('top-1m-reversed.csv',$index . ',' . strrev($row[1]) . PHP_EOL, FILE_APPEND);
        $index++;
    }
    fclose($handle);
}
var_dump(memory_get_usage(true));

que salidas

int(262144)
int(262144)

por lo tanto, no usamos un solo byte adicional de memoria, sino que analizamos todo el CSV y lo guardamos en otro archivo invirtiendo el valor de la segunda columna. Eso es porque fgetcsv lee solo una fila y $row se sobrescribe en cada bucle.

E/S de archivo basado en flujo #

Abriendo una transmisión #

fopen abre un identificador de flujo de archivos, que se puede usar con varias funciones para leer, escribir, buscar y otras funciones además. Este valor es del tipo recurso y no se puede pasar a otros subprocesos que conservan su funcionalidad.

$f = fopen("errors.log", "a"); // Will try to open errors.log for writing

El segundo parámetro es el modo del flujo de archivos:

Modo Descripción
r Abrir en modo de solo lectura, comenzando desde el principio del archivo
r+ Abierto para lectura y escritura, comenzando desde el principio del archivo
w abierto solo para escritura, comenzando desde el principio del archivo. Si el archivo existe, lo vaciará. Si no existe, intentará crearlo.
w+ abierto para lectura y escritura, comenzando desde el principio del archivo. Si el archivo existe, lo vaciará. Si no existe, intentará crearlo.
a abra un archivo solo para escritura, comenzando por el final del archivo. Si el archivo no existe, intentará crearlo
a+ abrir un archivo para leer y escribir, comenzando por el final del archivo. Si el archivo no existe, intentará crearlo
x crear y abrir un archivo solo para escritura. Si el archivo existe, la llamada fopen fallará
x+ crear y abrir un archivo para leer y escribir. Si el archivo existe, la llamada fopen fallará
c abre el archivo solo para escritura. Si el archivo no existe, intentará crearlo. Comenzará a escribir al principio del archivo, pero no vaciará el archivo antes de escribir
c+ abre el archivo para lectura y escritura. Si el archivo no existe, intentará crearlo. Comenzará a escribir al principio del archivo, pero no vaciará el archivo antes de escribir

Agregar una t detrás del modo (por ejemplo, a+b, wt, etc.) en Windows traducirá los finales de línea "\n" a "\r\n" al trabajar con el archivo. Agregue b detrás del modo si no es la intención, especialmente si es un archivo binario.

La aplicación PHP debe cerrar las secuencias usando [fclose][fclose] cuando ya no se usan para evitar el error Demasiados archivos abiertos. Esto es particularmente importante en los programas CLI, ya que las secuencias solo se cierran cuando el tiempo de ejecución se apaga; esto significa que en los servidores web, puede que no sea necesario (pero aun así debería, como práctica para evitar la fuga de recursos) para cierre los flujos si no espera que el proceso se ejecute durante mucho tiempo y no abrirá muchos flujos.

Lectura #

El uso de fread leerá el número dado de bytes desde el puntero del archivo, o hasta que se cumpla un EOF.

Líneas de lectura #

El uso de fgets leerá el archivo hasta que se alcance un EOL o se lea la longitud dada.

Tanto fread como fgets moverán el puntero del archivo durante la lectura.

Leyendo todo lo que queda #

El uso de stream_get_contents convertirá todos los bytes restantes en la transmisión en una cadena y los devolverá.

Ajuste de la posición del puntero del archivo #

Inicialmente, después de abrir la transmisión, el puntero del archivo está al principio del archivo (o al final, si se usa el modo a). El uso de la función fseek moverá el puntero del archivo a una nueva posición, en relación con uno de los tres valores:

  • SEEK_SET: Este es el valor predeterminado; el desplazamiento de la posición del archivo será relativo al comienzo del archivo.
  • SEEK_CUR: el desplazamiento de la posición del archivo será relativo a la posición actual.
  • SEEK_END: el desplazamiento de la posición del archivo será relativo al final del archivo. Pasar un desplazamiento negativo es el uso más común para este valor; moverá la posición del archivo al número especificado de bytes antes del final del archivo.

rebobinar es un atajo conveniente de fseek($fh, 0, SEEK_SET).

El uso de ftell mostrará la posición absoluta del puntero del archivo.

Por ejemplo, el siguiente script lee omite los primeros 10 bytes, lee los siguientes 10 bytes, omite 10 bytes, lee los siguientes 10 bytes y luego los últimos 10 bytes en file.txt:

$fh = fopen("file.txt", "rb");
fseek($fh, 10); // start at offset 10
echo fread($fh, 10); // reads 10 bytes
fseek($fh, 10, SEEK_CUR); // skip 10 bytes
echo fread($fh, 10); // read 10 bytes
fseek($fh, -10, SEEK_END); // skip to 10 bytes before EOF
echo fread($fh, 10); // read 10 bytes
fclose($fh);

Escritura #

El uso de fwrite escribe la cadena proporcionada en el archivo que comienza en el puntero del archivo actual.

fwrite($fh, "Some text here\n");

Mover y copiar archivos y directorios #

Copiando documentos #

[copy][copy] copia el archivo de origen en el primer argumento al destino en el segundo argumento. El destino resuelto debe estar en un directorio que ya se haya creado.

if (copy('test.txt', 'dest.txt')) {
    echo 'File has been copied successfully';
} else {
    echo 'Failed to copy file to destination given.'
}

Copiando directorios, con recursividad #

Copiar directorios es bastante similar a eliminar directorios, excepto que para archivos se usa [copy][copy] en lugar de [unlink][unlink], mientras que para directorios, mkdir en lugar de Se usa rmdir, al principio en lugar de estar al final de la función.

function recurse_delete_dir(string $src, string $dest) : int {
    $count = 0;

    // ensure that $src and $dest end with a slash so that we can concatenate it with the filenames directly
    $src = rtrim($dest, "/\\") . "/";
    $dest = rtrim($dest, "/\\") . "/";

    // use dir() to list files
    $list = dir($src);

    // create $dest if it does not already exist
    @mkdir($dest);

    // store the next file name to $file. if $file is false, that's all -- end the loop.
    while(($file = $list->read()) !== false) {
        if($file === "." || $file === "..") continue;
        if(is_file($src . $file)) {
            copy($src . $file, $dest . $file);
            $count++;
        } elseif(is_dir($src . $file)) {
            $count += recurse_copy_dir($src . $file, $dest . $file);
        }
    }

    return $count;
}

Renombrar/Mover #

Renombrar/Mover archivos y directorios es mucho más simple. Se pueden mover o renombrar directorios completos en una sola llamada, usando la función [rename][rename].

  • rename("~/archivo.txt", "~/archivo.html");
  • rename("~/dir", "~/old_dir");
  • rename("~/dir/archivo.txt", "~/dir2/archivo.txt");
