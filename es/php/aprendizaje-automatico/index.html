El tema usa PHP-ML para todos los algoritmos de aprendizaje automático. La instalación de la biblioteca se puede hacer usando

composer require php-ai/php-ml

El repositorio de github para el mismo se puede encontrar [aquí] (https://github.com/php-ai/php-ml).

También vale la pena señalar que los ejemplos dados son conjuntos de datos muy pequeños solo con fines de demostración. El conjunto de datos real debería ser más completo que eso.

Clasificación usando PHP-ML #

La clasificación en Machine Learning es el problema que identifica a qué conjunto de categorías pertenece una nueva observación. La clasificación se incluye en la categoría de “aprendizaje automático supervisado”.

Cualquier algoritmo que implemente la clasificación se conoce como clasificador

Los clasificadores soportados en PHP-ML son

  • SVC (Clasificación de vectores de soporte)
  • k-vecinos más cercanos
  • Bayesiano ingenuo

Los métodos train y predict son los mismos para todos los clasificadores. La única diferencia estaría en el algoritmo subyacente utilizado.

SVC (Clasificación de vectores de soporte) #

Antes de que podamos comenzar a predecir una nueva observación, necesitamos entrenar nuestro clasificador. Considere el siguiente código

// Import library
use Phpml\Classification\SVC;
use Phpml\SupportVectorMachine\Kernel;

// Data for training classifier
$samples = [[1, 3], [1, 4], [2, 4], [3, 1], [4, 1], [4, 2]];  // Training samples
$labels = ['a', 'a', 'a', 'b', 'b', 'b'];

// Initialize the classifier
$classifier = new SVC(Kernel::LINEAR, $cost = 1000);
// Train the classifier
$classifier->train($samples, $labels);

El código es bastante sencillo. El $costo utilizado anteriormente es una medida de cuánto queremos evitar clasificar erróneamente cada ejemplo de entrenamiento. Por un valor menor de $cost, puede obtener ejemplos mal clasificados. Por defecto está configurado en 1.0

Ahora que tenemos el clasificador entrenado, podemos comenzar a hacer algunas predicciones reales. Considere los siguientes códigos que tenemos para las predicciones

$classifier->predict([3, 2]); // return 'b'
$classifier->predict([[3, 2], [1, 5]]); // return ['b', 'a']

El clasificador en el caso anterior puede tomar muestras no clasificadas y predice sus etiquetas. El método predecir puede tomar una sola muestra, así como una serie de muestras.

k-vecinos más cercanos #

El clasificador de este algoritmo toma dos parámetros y se puede inicializar como

$classifier = new KNearestNeighbors($neighbor_num=4);
$classifier = new KNearestNeighbors($neighbor_num=3, new Minkowski($lambda=4));

$neighbor_num es el número de vecinos más cercanos para escanear en el algoritmo knn mientras que el segundo parámetro es la métrica de distancia, que por defecto en el primer caso sería euclidiana. Puede encontrar más información sobre Minkowski [aquí] (https://en.wikipedia.org/wiki/Minkowski_distance).

A continuación se muestra un breve ejemplo de cómo utilizar este clasificador.

// Training data
$samples = [[1, 3], [1, 4], [2, 4], [3, 1], [4, 1], [4, 2]];
$labels = ['a', 'a', 'a', 'b', 'b', 'b'];

// Initialize classifier
$classifier = new KNearestNeighbors();
// Train classifier
$classifier->train($samples, $labels);

// Make predictions
$classifier->predict([3, 2]); // return 'b'
$classifier->predict([[3, 2], [1, 5]]); // return ['b', 'a']

Clasificador NaiveBayes #

NaiveBayes Classifier se basa en el teorema de Bayes y no necesita ningún parámetro en el constructor.

El siguiente código demuestra una implementación de predicción simple

// Training data
$samples = [[5, 1, 1], [1, 5, 1], [1, 1, 5]];
$labels = ['a', 'b', 'c'];

// Initialize classifier
$classifier = new NaiveBayes();
// Train classifier
$classifier->train($samples, $labels);

// Make predictions
$classifier->predict([3, 1, 1]); // return 'a'
$classifier->predict([[3, 1, 1], [1, 4, 1]); // return ['a', 'b']

Caso práctico #

Hasta ahora, solo usamos matrices de enteros en todos nuestros casos, pero ese no es el caso en la vida real. Por lo tanto, permítanme tratar de describir una situación práctica sobre cómo usar clasificadores.

Suponga que tiene una aplicación que almacena las características de las flores en la naturaleza. En aras de la simplicidad, podemos considerar el color y la longitud de los pétalos. Entonces, se usarían dos características para entrenar nuestros datos. color es el más simple donde puede asignar un valor int a cada uno de ellos y para la longitud, puede tener un rango como (0 mm, 10 mm) = 1, (10 mm, 20 mm) = 2. Con los datos iniciales entrena tu clasificador. Ahora, uno de sus usuarios necesita identificar el tipo de flor que crece en su patio trasero. Lo que hace es seleccionar el color de la flor y suma la longitud de los pétalos. Su clasificador en ejecución puede detectar el tipo de flor (“Etiquetas en el ejemplo anterior”)

Regresión #

En la clasificación usando PHP-ML asignamos etiquetas a la nueva observación. La regresión es casi la misma, con la diferencia de que el valor de salida no es una etiqueta de clase sino un valor continuo. Es ampliamente utilizado para predicciones y pronósticos. PHP-ML admite los siguientes algoritmos de regresión

  • Regresión de vectores de soporte
  • Regresión lineal de mínimos cuadrados

La regresión tiene los mismos métodos de “entrenamiento” y “predicción” que se usan en la clasificación.

Regresión de vectores de soporte #

Esta es la versión de regresión para SVM (Support Vector Machine). El primer paso, como en la clasificación, es entrenar nuestro modelo.

// Import library
use Phpml\Regression\SVR;
use Phpml\SupportVectorMachine\Kernel;

// Training data
$samples = [[60], [61], [62], [63], [65]];
$targets = [3.1, 3.6, 3.8, 4, 4.1];

// Initialize regression engine
$regression = new SVR(Kernel::LINEAR);
// Train regression engine
$regression->train($samples, $targets);

En la regresión, $objetivos no son etiquetas de clase en lugar de clasificación. Este es uno de los factores diferenciadores para los dos. Después de entrenar nuestro modelo con los datos, podemos comenzar con las predicciones reales.

$regression->predict([64])  // return 4.03

Tenga en cuenta que las predicciones devuelven un valor fuera del objetivo.

Regresión lineal de mínimos cuadrados #

Este algoritmo utiliza el “método de los mínimos cuadrados” para aproximar la solución. Lo siguiente demuestra un código simple de entrenamiento y predicción

// Training data
$samples = [[60], [61], [62], [63], [65]];
$targets = [3.1, 3.6, 3.8, 4, 4.1];

// Initialize regression engine
$regression = new LeastSquares();
// Train engine
$regression->train($samples, $targets);
// Predict using trained engine
$regression->predict([64]); // return 4.06

PHP-ML también ofrece la opción de Regresión lineal múltiple. Un código de muestra para el mismo puede ser el siguiente

$samples = [[73676, 1996], [77006, 1998], [10565, 2000], [146088, 1995], [15000, 2001], [65940, 2000], [9300, 2000], [93739, 1996], [153260, 1994], [17764, 2002], [57000, 1998], [15000, 2000]];
$targets = [2000, 2750, 15500, 960, 4400, 8800, 7100, 2550, 1025, 5900, 4600, 4400];

$regression = new LeastSquares();
$regression->train($samples, $targets);
$regression->predict([60000, 1996]) // return 4094.82

La regresión lineal múltiple es particularmente útil cuando múltiples factores o características identifican el resultado.

Caso práctico #

Ahora tomemos una aplicación de regresión en el escenario de la vida real.

Suponga que ejecuta un sitio web muy popular, pero el tráfico sigue cambiando. Desea una solución que prediga la cantidad de servidores que necesita implementar en cualquier momento dado. Supongamos por el bien de que su proveedor de alojamiento le proporciona una API para generar servidores y cada servidor tarda 15 minutos en arrancar. Con base en datos anteriores de tráfico y regresión, puede predecir el tráfico que llegaría a su aplicación en cualquier momento. Con ese conocimiento, puede iniciar un servidor 15 minutos antes del aumento, evitando así que su aplicación se desconecte.

Agrupación #

La agrupación consiste en agrupar objetos similares. Es ampliamente utilizado para el reconocimiento de patrones. La ‘agrupación’ se incluye en el ‘aprendizaje automático no supervisado’, por lo tanto, no se necesita capacitación. PHP-ML tiene soporte para los siguientes algoritmos de agrupamiento

  • k-Medios
  • dbscan

k-Medios #

k-Means separa los datos en n grupos de igual varianza. Esto significa que necesitamos pasar un número n que sería el número de clústeres que necesitamos en nuestra solución. El siguiente código ayudará a traer más claridad.

// Our data set
$samples = [[1, 1], [8, 7], [1, 2], [7, 8], [2, 1], [8, 9]];

// Initialize clustering with parameter `n`
$kmeans = new KMeans(3);
$kmeans->cluster($samples); // return [0=>[[7, 8]], 1=>[[8, 7]], 2=>[[1,1]]]

Tenga en cuenta que la salida contiene 3 matrices porque ese era el valor de n en el constructor KMeans. También puede haber un segundo parámetro opcional en el constructor que sería el método de inicialización. Por ejemplo, considere

$kmeans = new KMeans(4, KMeans::INIT_RANDOM);

INIT_RANDOM coloca un centroide completamente aleatorio al intentar determinar los grupos. Pero solo para evitar que el centroide esté demasiado lejos de los datos, está limitado por los límites espaciales de los datos.

El método de inicialización predeterminado del constructor es kmeans++ que selecciona el centroide de forma inteligente para acelerar el proceso.

#DBSCAN

A diferencia de KMeans, DBSCAN es un algoritmo de agrupamiento basado en la densidad, lo que significa que no estaríamos pasando n, lo que determinaría el número de clústeres que queremos en nuestro resultado. Por otro lado esto requiere dos parámetros para trabajar

  1. $minSamples: La cantidad mínima de objetos que deben estar presentes en un clúster
  2. $epsilon : Cuál es la distancia máxima entre dos muestras para que se consideren en el mismo grupo.

Una muestra rápida para lo mismo es la siguiente

// Our sample data set
$samples = [[1, 1], [8, 7], [1, 2], [7, 8], [2, 1], [8, 9]];

$dbscan = new DBSCAN($epsilon = 2, $minSamples = 3);
$dbscan->cluster($samples); // return [0=>[[1, 1]], 1=>[[8, 7]]]

El código es bastante autoexplicativo. Una diferencia importante es que no hay forma de saber la cantidad de elementos en la matriz de salida en lugar de KMeans.

Caso Práctico #

Ahora echemos un vistazo al uso de la agrupación en clústeres en el escenario de la vida real.

La agrupación en clústeres se utiliza mucho en el “reconocimiento de patrones” y la “minería de datos”. Considere que tiene una aplicación de publicación de contenido. Ahora, para retener a sus usuarios, deben mirar el contenido que les encanta. Supongamos, en aras de la simplicidad, que si están en una página web específica durante más de un minuto y se desplazan hasta el fondo, entonces les encanta ese contenido. Ahora cada uno de sus contenidos tendrá un identificador único y también lo tendrá el usuario. Haz un clúster basado en eso y sabrás qué segmento de usuarios tienen un gusto de contenido similar. Esto, a su vez, podría usarse en el sistema de recomendación donde puede asumir que si a algunos usuarios del mismo grupo les encanta el artículo, a otros también les gustará y eso se puede mostrar como recomendaciones en su aplicación.
