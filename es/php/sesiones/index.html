Tenga en cuenta que llamar a session_start() incluso si la sesión ya ha comenzado dará como resultado una advertencia de PHP.

Opciones de session_start() #

Comenzando con PHP Sessions, podemos pasar una matriz con opciones php.ini basadas en sesión a la función session_start.

Ejemplo

<?php
   if (version_compare(PHP_VERSION, '7.0.0') >= 0) {
       // php >= 7 version
       session_start([
          'cache_limiter' => 'private',
          'read_and_close' => true,
       ]);
   } else {
       // php < 7 version
       session_start();
   }
?>

Esta característica también presenta una nueva configuración php.ini llamada session.lazy_write, que por defecto es true y significa que los datos de la sesión solo se reescriben, si cambian.

Referencia: https://wiki.php.net/rfc/session-lock-ini

Bloqueo de sesión #

Como todos sabemos, PHP escribe los datos de la sesión en un archivo del lado del servidor. Cuando se realiza una solicitud al script php que inicia la sesión a través de session_start(), PHP bloquea este archivo de sesión y bloquea/espera otras solicitudes entrantes para que se complete el mismo session_id, por lo que las otras solicitudes se atascarán en session_start() hasta que no se libere el archivo de sesión bloqueado

El archivo de la sesión permanece bloqueado hasta que se completa el script o la sesión se cierra manualmente. Para evitar esta situación i.e. para evitar que se bloqueen varias solicitudes, podemos iniciar la sesión y cerrarla, lo que liberará el bloqueo del archivo de sesión y permitirá continuar con las solicitudes restantes.

// php < 7.0 
// start session 
session_start();

// write data to session
$_SESSION['id'] = 123; // session file is locked, so other requests are blocked

// close the session, release lock
session_write_close();

Ahora uno pensará si la sesión está cerrada, cómo leeremos los valores de la sesión, embellecer incluso después de que la sesión esté cerrada, la sesión todavía está disponible. Entonces, todavía podemos leer los datos de la sesión.

echo $_SESSION['id'];    // will output  123

En php >= 7.0, podemos tener sesiones de solo lectura, sesiones de lectura_escritura y sesiones de lazy_write, por lo que es posible que no sea necesario usar session_write_close()

Manipulación de datos de sesión #

La variable $_SESSION es una matriz y puede recuperarla o manipularla como una matriz normal.

<?php
// Starting the session
session_start();

// Storing the value in session
$_SESSION['id'] = 342;

// conditional usage of session values that may have been set in a previous session
if(!isset($_SESSION["login"])) {
    echo "Please login first";
    exit;
}
// now you can use the login safely
$user = $_SESSION["login"];

// Getting a value from the session data, or with default value, 
//     using the Null Coalescing operator in PHP 7
$name = $_SESSION['name'] ?? 'Anonymous';

Consulte también https://www.wikiod.com/es/php/manipulacion-de-una-matriz para obtener más referencias sobre cómo trabajar en una matriz.

Tenga en cuenta que si almacena un objeto en una sesión, puede recuperarse correctamente solo si tiene un cargador automático de clases o si ya ha cargado la clase. De lo contrario, el objeto aparecerá como del tipo __PHP_Incomplete_Class, lo que luego puede provocar [bloqueos] (http://stackoverflow.com/q/1055728/3990767). Consulte https://www.wikiod.com/es/php/clases-y-objetos#Espacio de nombres y carga automática sobre la carga automática.

Advertencia: #

Los datos de la sesión pueden ser secuestrados. Esto se describe en: Pro PHP Security: From Application Security Principles to the Implementing of XSS Defense - Chapter 7: Preventing Session Hijacking Por lo tanto, se recomienda enfáticamente que nunca almacene información personal en $_SESSION . Esto incluiría de manera más crítica números de tarjetas de crédito, identificaciones emitidas por el gobierno y contraseñas; pero también se extendería a datos menos supuestos como nombres, correos electrónicos, números de teléfono, etc., que permitirían a un hacker hacerse pasar por/comprometer a un usuario legítimo. Como regla general, use valores sin valor/no personales, como identificadores numéricos, en los datos de la sesión.

Destruye una sesión completa #

Si tiene una sesión que desea destruir, puede hacerlo con session_destroy()

/*
    Let us assume that our session looks like this:
    Array([firstname] => Jon, [id] => 123)

    We first need to start our session:
*/
session_start();

/*
    We can now remove all the values from the `SESSION` superglobal:
    If you omitted this step all of the global variables stored in the 
    superglobal would still exist even though the session had been destroyed.
*/
$_SESSION = array();

// If it's desired to kill the session, also delete the session cookie.
// Note: This will destroy the session, and not just the session data!
if (ini_get("session.use_cookies")) {
    $params = session_get_cookie_params();
    setcookie(session_name(), '', time() - 42000,
        $params["path"], $params["domain"],
        $params["secure"], $params["httponly"]
    );
}

//Finally we can destroy the session:
session_destroy();

Usar session_destroy() es diferente a usar algo como $_SESSION = array(); que eliminará todos los valores almacenados en el superglobal SESSION pero no destruirá la versión real almacenada de la sesión.

Nota: Usamos $_SESSION = array(); en lugar de session_unset() porque el manual estipula:

Solo use session_unset() para códigos obsoletos más antiguos que no usan $_SESSION.

Nombre de la sesión #

Comprobar si se han creado cookies de sesión #

El nombre de sesión es el nombre de la cookie utilizada para almacenar sesiones. Puede utilizar esto para detectar si se han creado cookies para una sesión para el usuario:

if(isset($_COOKIE[session_name()])) {
    session_start();
}

Tenga en cuenta que este método generalmente no es útil a menos que realmente no desee crear cookies innecesariamente.

Cambiando el nombre de la sesión #

Puede actualizar el nombre de la sesión llamando a session_name().

//Set the session name
session_name('newname');
//Start the session
session_start();

Si no se proporciona ningún argumento en session_name(), se devuelve el nombre de la sesión actual.

Debe contener solo caracteres alfanuméricos; debe ser breve y descriptivo (es decir, para usuarios con advertencias de cookies habilitadas). El nombre de la sesión no puede constar únicamente de dígitos, debe estar presente al menos una letra. De lo contrario, se genera una nueva identificación de sesión cada vez.

Inicio de sesión seguro sin errores #

Muchos desarrolladores tienen este problema cuando trabajan en grandes proyectos, especialmente si trabajan en algún CMS modular en complementos, complementos, componentes, etc. Aquí hay una solución para el inicio seguro de la sesión donde si primero se verifica la versión de PHP para cubrir todas las versiones y se verifica la siguiente si se inicia sesión. Si la sesión no existe, comienzo la sesión de forma segura. Si la sesión existe, no sucede nada.

if (version_compare(PHP_VERSION, '7.0.0') >= 0) {
    if(session_status() == PHP_SESSION_NONE) {
        session_start(array(
          'cache_limiter' => 'private',
          'read_and_close' => true,
       ));
    }
}
else if (version_compare(PHP_VERSION, '5.4.0') >= 0)
{
    if (session_status() == PHP_SESSION_NONE) {
        session_start();
    }
}
else
{
    if(session_id() == '') {
        session_start();
    }
}

Esto puede ayudarte mucho a evitar el error session_start.
