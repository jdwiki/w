Una cookie HTTP es una pequeña pieza de datos enviada desde un sitio web y almacenada en la computadora del usuario por el navegador web del usuario mientras el usuario está navegando.

Sintaxis #

  • bool setcookie( cadena $nombre [, cadena $valor = "" [, int $caducidad = 0 [, cadena $ruta = "" [, cadena $dominio = "" [, bool $seguro = falso [, bool $ httponly = false ]]]]]] )

Parámetros #

parámetro detalle
nombre El nombre de la galleta. Esta es también la clave que puede usar para recuperar el valor del súper global $_COOKIE. Este es el único parámetro requerido
valor El valor a almacenar en la cookie. El navegador puede acceder a estos datos, así que no almacene nada confidencial aquí.
caducar Una marca de tiempo de Unix que representa cuándo debe caducar la cookie. Si se establece en cero, la cookie caducará al final de la sesión. Si se establece en un número menor que la marca de tiempo actual de Unix, la cookie caducará inmediatamente.
camino El alcance de la cookie. Si se establece en /, la cookie estará disponible en todo el dominio. Si se establece en /some-path/, la cookie solo estará disponible en esa ruta y los descendientes de esa ruta. El valor predeterminado es la ruta actual del archivo en el que se establece la cookie.
dominio El dominio o subdominio en el que está disponible la cookie. Si se establece en el dominio simple stackoverflow.com, la cookie estará disponible para ese dominio y todos los subdominios. Si se establece en un subdominio meta.stackoverflow.com, la cookie estará disponible solo en ese subdominio y en todos los sub-subdominios.
seguro Cuando se establece en “VERDADERO”, la cookie solo se establecerá si existe una conexión HTTPS segura entre el cliente y el servidor.
solo http Especifica que la cookie solo debe estar disponible a través del protocolo HTTP/S y no debe estar disponible para lenguajes de secuencias de comandos del lado del cliente como JavaScript. Solo disponible en PHP 5.2 o posterior.

Vale la pena señalar que la mera invocación de la función setcookie no solo coloca los datos dados en la matriz superglobal $_COOKIE.

Por ejemplo, no tiene sentido hacer:

setcookie("user", "Tom", time() + 86400, "/");
var_dump(isset($_COOKIE['user'])); // yields false or the previously set value

El valor aún no está allí, no hasta que se cargue la próxima página. La función setcookie simplemente dice “con la siguiente conexión http, dígale al cliente (navegador) que configure esta cookie”. Luego, cuando los encabezados se envían al navegador, contienen este encabezado de cookie. Luego, el navegador verifica si la cookie aún no ha caducado, y si no, entonces en la solicitud http envía la cookie al servidor y ahí es cuando PHP la recibe y coloca el contenido en la matriz $_COOKIE.

Modificar una cookie #

El valor de una cookie se puede modificar reiniciando la cookie

setcookie("user", "John", time() + 86400, "/"); // assuming there is a "user" cookie already

Las cookies son parte del encabezado HTTP, por lo que se debe llamar a setcookie() antes de enviar cualquier salida al navegador.

Al modificar una cookie, asegúrese de que los parámetros ruta y dominio de setcookie() coincidan con la cookie existente o se creará una nueva en su lugar.

La porción de valor de la cookie se codificará automáticamente cuando envíe la cookie, y cuando se reciba, se descodificará automáticamente y se asignará a una variable con el mismo nombre que el nombre de la cookie.

Configuración de una cookie #

Una cookie se establece mediante la función setcookie(). Dado que las cookies son parte del encabezado HTTP, debe configurar cualquier cookie antes de enviar cualquier salida al navegador.

Ejemplo:

setcookie("user", "Tom", time() + 86400, "/"); // check syntax for function params

Descripción:

  • Crea una cookie con el nombre usuario
  • (Opcional) El valor de la cookie es Tom
  • (Opcional) La cookie caducará en 1 día (86400 segundos)
  • (Opcional) La cookie está disponible en todo el sitio web /
  • (Opcional) La cookie solo se envía a través de HTTPS
  • (Opcional) La cookie no es accesible para lenguajes de secuencias de comandos como JavaScript

Solo se puede acceder a una cookie creada o modificada en solicitudes posteriores (donde coinciden ruta y dominio) ya que $_COOKIE superglobal no se completa con los nuevos datos inmediatamente.

Comprobando si se ha establecido una cookie #

Utilice la función isset() sobre la variable superglobal $_COOKIE para comprobar si se ha establecido una cookie.

Ejemplo:

// PHP <7.0
if (isset($_COOKIE['user'])) {
    // true, cookie is set
    echo 'User is ' . $_COOKIE['user'];
else {
    // false, cookie is not set
    echo 'User is not logged in';
}

// PHP 7.0+
echo 'User is ' . $_COOKIE['user'] ?? 'User is not logged in';

Eliminar una cookie #

Para eliminar una cookie, establezca la marca de tiempo de caducidad en un momento del pasado. Esto activa el mecanismo de eliminación del navegador:

setcookie('user', '', time() - 3600, '/');

Al eliminar una cookie, asegúrese de que los parámetros ruta y dominio de setcookie() coincidan con la cookie que está tratando de eliminar o se creará una nueva cookie, que caduca inmediatamente.

También es una buena idea desactivar el valor $_COOKIE en caso de que la página actual lo use:

unset($_COOKIE['user']);

Recuperar una cookie #

Recuperar y generar una cookie llamada usuario

El valor de una cookie se puede recuperar utilizando la variable global $_COOKIE. ejemplo, si tenemos una cookie llamada ‘usuario’, podemos recuperarla así

echo $_COOKIE['user'];
