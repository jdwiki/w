Composer es el administrador de dependencias más utilizado de PHP. Es análogo a npm en Node, pip para Python o NuGet para .NET.

Sintaxis #

  • php path/to/composer.phar [comando] [opciones] [argumentos]

Parámetros #

Parámetro Detalles
licencia Define el tipo de licencia que desea utilizar en el Proyecto.
autores Define los autores del proyecto, así como los detalles del autor.
apoyo Define los correos electrónicos de soporte, el canal IRC y varios enlaces.
requieren Define las dependencias reales, así como las versiones del paquete.
requiere-desarrollo Define los paquetes necesarios para el desarrollo del proyecto.
sugerir Define las sugerencias de paquetes, es decir, paquetes que pueden ayudar si están instalados.
carga automática Define las políticas de carga automática del proyecto.
autoload-dev Define las políticas de autocarga para el desarrollo del proyecto.

La carga automática solo funcionará para las bibliotecas que especifican información de carga automática. La mayoría de las bibliotecas cumplen y se adhieren a un estándar como PSR-0 o [PSR-4](http://www.php -fig.org/psr/psr-4/).

Enlaces Útiles #

  • Packagist – Explore los paquetes disponibles (que puede instalar con Composer).
  • Documentación Oficial
  • Guía oficial de introducción

Pocas sugerencias #

  1. Deshabilite xdebug cuando ejecute Composer.
  2. No ejecute Composer como root. Los paquetes no son de fiar.

Carga automática con Composer #

Mientras que composer proporciona un sistema para administrar dependencias para proyectos PHP (por ejemplo, de Packagist), también puede servir notablemente como cargador automático, especificando dónde buscar espacios de nombres específicos o incluir archivos de funciones genéricas. .

Comienza con el archivo composer.json:

{
    // ...
    "autoload": {
        "psr-4": {
            "MyVendorName\\MyProject": "src/"
        },
        "files": [
            "src/functions.php"
        ]
    },
    "autoload-dev": {
        "psr-4": {
            "MyVendorName\\MyProject\\Tests": "tests/"
        }
    }
}

Este código de configuración garantiza que todas las clases en el espacio de nombres MyVendorName\MyProject se asignen al directorio src y todas las clases en MyVendorName\MyProject\Tests al directorio tests (en relación con su directorio raíz). También incluirá automáticamente el archivo functions.php.

Después de poner esto en su archivo composer.json, ejecute composer update en una terminal para que el compositor actualice las dependencias, el archivo de bloqueo y genere el archivo autoload.php. Al implementar en un entorno de producción, usaría composer install --no-dev. El archivo autoload.php se puede encontrar en el directorio vendor que debe generarse en el directorio donde reside composer.json.

Debe ‘requerir’ este archivo temprano en un punto de configuración en el ciclo de vida de su aplicación utilizando una línea similar a la siguiente.

require_once __DIR__ . '/vendor/autoload.php';

Una vez incluido, el archivo autoload.php se encarga de cargar todas las dependencias que proporcionó en su archivo composer.json.

Algunos ejemplos de la ruta de clases a la asignación de directorios:

  • MyVendorName\MyProject\Shapes\Square ➔ src/Shapes/Square.php.
  • MyVendorName\MyProject\Tests\Shapes\Square ➔ tests/Shapes/Square.php.

¿Qué es Compositor? #

Composer es un administrador de dependencias/paquetes para PHP. Se puede usar para instalar, realizar un seguimiento y actualizar las dependencias de su proyecto. Composer también se encarga de cargar automáticamente las dependencias en las que se basa su aplicación, lo que le permite usar fácilmente la dependencia dentro de su proyecto sin preocuparse por incluirlas en la parte superior de cualquier archivo dado.

Las dependencias de su proyecto se enumeran dentro de un archivo composer.json que normalmente se encuentra en la raíz de su proyecto. Este archivo contiene información sobre las versiones requeridas de paquetes para producción y desarrollo.

Puede encontrar un resumen completo del esquema composer.json en el [sitio web de Composer] (https://getcomposer.org/doc/04-schema.md).

Este archivo se puede editar manualmente usando cualquier editor de texto o automáticamente a través de la línea de comandos a través de comandos como compositor require <paquete> o compositor require-dev <paquete>.

Para comenzar a usar composer en su proyecto, deberá crear el archivo composer.json. Puede crearlo manualmente o simplemente ejecutar composer init. Después de ejecutar composer init en su terminal, le pedirá información básica sobre su proyecto: Nombre del paquete (proveedor/paquete - por ejemplo, laravel/laravel), Descripción - opcional, Autor y alguna otra información como Estabilidad Mínima, Licencia y Paquetes Requeridos.

La clave require en su archivo composer.json especifica Composer de qué paquetes depende su proyecto. require toma un objeto que asigna nombres de paquetes (por ejemplo, monolog/monolog) a restricciones de versión (por ejemplo, *1.0.**).

{
    "require": {
        "composer/composer": "1.2.*"
    }
}

Para instalar las dependencias definidas, deberá ejecutar el comando composer install y luego encontrará los paquetes definidos que coincidan con la restricción de versión proporcionada y los descargará en el directorio proveedor. Es una convención poner código de terceros en un directorio llamado proveedor.

Notará que el comando install también creó un archivo composer.lock.

Composer genera automáticamente un archivo composer.lock. Este archivo se usa para rastrear las versiones actualmente instaladas y el estado de sus dependencias. Ejecutar composer install instalará los paquetes exactamente en el estado almacenado en el archivo de bloqueo.

Diferencia entre ‘instalación del compositor’ y ‘actualización del compositor’ #

actualización del compositor #

composer update actualizará nuestras dependencias tal como se especifican en composer.json.

Por ejemplo, si nuestro proyecto usa esta configuración:

"require": {
    "laravelcollective/html": "2.0.*"
}

Suponiendo que hayamos instalado la versión 2.0.1 del paquete, ejecutar composer update provocará una actualización de este paquete (por ejemplo, a 2.0.2, si ya se ha lanzado).

En detalle, la actualización del compositor:

  • Leer compositor.json
  • Elimina los paquetes instalados que ya no son necesarios en compositor.json
  • Consultar la disponibilidad de las últimas versiones de nuestros requeridos paquetes
  • Instalar las últimas versiones de nuestros paquetes
  • Actualice composer.lock para almacenar la versión de los paquetes instalados

instalación del compositor #

composer install instalará todas las dependencias como se especifica en el archivo composer.lock en la versión especificada (bloqueada), sin actualizar nada.

En detalle:

  • Leer el archivo composer.lock
  • Instalar los paquetes especificados en el archivo composer.lock

Cuándo instalar y cuándo actualizar #

  • La actualización del compositor se usa principalmente en la fase de ‘desarrollo’, para actualizar nuestros paquetes de proyectos.

  • composer install se usa principalmente en la ‘fase de implementación’ para instalar nuestra aplicación en un servidor de producción o en un entorno de prueba, utilizando las mismas dependencias almacenadas en el archivo composer.lock creado por composer update.

Beneficios de usar Composer #

Composer rastrea qué versiones de paquetes ha instalado en un archivo llamado composer.lock, que está destinado a ser comprometido con el control de versiones, de modo que cuando el proyecto se clone en el futuro, simplemente ejecute composer install para descargar e instalar todas las dependencias del proyecto.

Composer se ocupa de las dependencias de PHP por proyecto. Esto facilita tener varios proyectos en una máquina que dependen de versiones separadas de un paquete PHP.

Composer rastrea qué dependencias solo están destinadas a entornos de desarrollo

composer require --dev phpunit/phpunit

Composer proporciona un cargador automático, lo que hace que sea extremadamente fácil comenzar con cualquier paquete. Por ejemplo, después de instalar Goutte con composer require fabpot/goutte, puede comenzar a usar Goutte inmediatamente en un nuevo proyecto:

<?php

require __DIR__ . '/vendor/autoload.php';

$client = new Goutte\Client();

// Start using Goutte

Composer le permite actualizar fácilmente un proyecto a la última versión permitida por su composer.json. P.EJ. compositor update fabpot/goutte, o para actualizar cada una de las dependencias de su proyecto: composer update.

Instalación #

Puede instalar Composer localmente, como parte de su proyecto, o globalmente como un ejecutable de todo el sistema.

Localmente #

Para instalar, ejecute estos comandos en su terminal.

php -r "copy('https://getcomposer.org/installer', 'composer-setup.php');"
# to check the validity of the downloaded installer, check here against the SHA-384:
# https://composer.github.io/pubkeys.html
php composer-setup.php
php -r "unlink('composer-setup.php');"

Esto descargará composer.phar (un archivo PHP Archive) al directorio actual. Ahora puede ejecutar php composer.phar para usar Composer, p.

php composer.phar install

Globalmente #

Para usar Composer globalmente, coloque el archivo composer.phar en un directorio que sea parte de su RUTA

mv composer.phar /usr/local/bin/composer

Ahora puedes usar composer en cualquier lugar en lugar de php composer.phar, p.

composer install
