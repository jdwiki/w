Sintaxis #

  • requerir
  • spl_autoload_require

La carga automática, como parte de una estrategia de marco, reduce la cantidad de código repetitivo que tiene que escribir.

Definición de clase en línea, no se requiere carga #

// zoo.php
class Animal {
    public function eats($food) {
        echo "Yum, $food!";
    }
}

$animal = new Animal();
$animal->eats('meat');

PHP sabe qué es Animal antes de ejecutar new Animal, porque PHP lee los archivos fuente de arriba a abajo. Pero, ¿y si quisiéramos crear nuevos animales en muchos lugares, no solo en el archivo fuente donde está definido? Para hacer eso, necesitamos cargar la definición de clase.

Carga manual de clases con require #

// Animal.php
class Animal {
    public function eats($food) {
         echo "Yum, $food!";
    }
}

// zoo.php
require 'Animal.php';
$animal = new Animal;
$animal->eats('slop');

// aquarium.php
require 'Animal.php';
$animal = new Animal;
$animal->eats('shrimp');

Aquí tenemos tres archivos. Un archivo (“Animal.php”) define la clase. Este archivo no tiene efectos secundarios además de definir la clase y mantiene ordenadamente todo el conocimiento sobre un “Animal” en un solo lugar. Es fácil controlar la versión. Es fácil de reutilizar.

Dos archivos consumen el archivo “Animal.php” al “requerir” manualmente el archivo. Nuevamente, PHP lee los archivos fuente de arriba a abajo, por lo que el requerimiento va y encuentra el archivo “Animal.php” y hace que la definición de la clase Animal esté disponible antes de llamar a nuevo Animal.

Ahora imagina que tuviéramos docenas o cientos de casos en los que queríamos realizar un “nuevo Animal”. Eso requeriría (juego de palabras) muchas, muchas declaraciones require que son muy tediosas para codificar.

La carga automática reemplaza la carga de definición de clase manual #

// autoload.php
spl_autoload_register(function ($class) {
    require_once "$class.php";
});

// Animal.php
class Animal {
    public function eats($food) {
         echo "Yum, $food!";
    }
}

// zoo.php
require 'autoload.php';
$animal = new Animal;
$animal->eats('slop');

// aquarium.php
require 'autoload.php';
$animal = new Animal;
$animal->eats('shrimp');

Compara esto con los otros ejemplos. Observe cómo se reemplazó require "Animal.php" por require "autoload.php". Todavía estamos incluyendo un archivo externo en tiempo de ejecución, pero en lugar de incluir una definición de clase específica, estamos incluyendo lógica que puede incluir cualquier clase. Es un nivel de indirección que facilita nuestro desarrollo. En lugar de escribir un “requerido” para cada clase que necesitamos, escribimos un “requerido” para todas las clases. Podemos reemplazar N requerir con 1 requerir.

La magia ocurre con spl_autoload_register. Esta función de PHP toma un cierre y lo agrega a una cola de cierres. Cuando PHP encuentra una clase para la que no tiene definición, entrega el nombre de la clase a cada cierre en la cola. Si la clase existe después de llamar a un cierre, PHP vuelve a su actividad anterior. Si la clase no existe después de probar toda la cola, PHP se bloquea con “Clase ‘Lo que sea’ no encontrada”.

Carga automática como parte de una solución de marco #

// autoload.php
spl_autoload_register(function ($class) {
    require_once "$class.php";
});

// Animal.php
class Animal {
    public function eats($food) {
         echo "Yum, $food!";
    }
}

// Ruminant.php
class Ruminant extends Animal {
    public function eats($food) {
        if ('grass' === $food) {
            parent::eats($food);
        } else {
            echo "Yuck, $food!";
        }
    }
}

// Cow.php
class Cow extends Ruminant {
}

// pasture.php
require 'autoload.php';
$animal = new Cow;
$animal->eats('grass');

Gracias a nuestro autocargador genérico, tenemos acceso a cualquier clase que siga nuestra convención de nomenclatura de autocargador. En este ejemplo, nuestra convención es simple: la clase deseada debe tener un archivo en el mismo directorio con el nombre de la clase y que termine en “.php”. Observe que el nombre de la clase coincide exactamente con el nombre del archivo.

Sin la carga automática, tendríamos que “requerir” manualmente las clases base. Si construyéramos un zoológico completo de animales, tendríamos miles de instrucciones requeridas que podrían reemplazarse más fácilmente con un solo cargador automático.

En el análisis final, la carga automática de PHP es un mecanismo para ayudarlo a escribir código menos mecánico para que pueda concentrarse en resolver problemas comerciales. Todo lo que tiene que hacer es definir una estrategia que asigne el nombre de la clase al nombre del archivo. Puede implementar su propia estrategia de carga automática, como se hace aquí. O bien, puede usar cualquiera de los estándares que ha adoptado la comunidad de PHP: PSR-0 o PSR-4. O bien, puede usar compositor para definir y administrar estas dependencias de forma genérica.

Carga automática con Composer #

Composer genera un archivo vendor/autoload.php.

Simplemente puede incluir este archivo y obtendrá la carga automática de forma gratuita.

requerir __DIR__. '/proveedor/autoload.php';

Esto hace que trabajar con dependencias de terceros sea muy fácil.

También puede agregar su propio código al cargador automático agregando una sección de carga automática a su composer.json.

{
    "autoload": {
        "psr-4": {"YourApplicationNamespace\\": "src/"}
    }
}

En esta sección, define las asignaciones de carga automática. En este ejemplo, es una asignación PSR-4 de un espacio de nombres a un directorio: el directorio /src reside en la carpeta raíz de sus proyectos, en el mismo nivel que el directorio /vendor. Un nombre de archivo de ejemplo sería src/Foo.php que contiene una clase YourApplicationNamespace\Foo.

Importante: Después de agregar nuevas entradas a la sección de carga automática, debe volver a ejecutar el comando dump-autoload para volver a generar y actualizar el archivo vendor/autoload.php con el nueva información.

Además de la carga automática de PSR-4, Composer también admite la carga automática de PSR-0, classmap y files. Consulte la referencia de carga automática para obtener más información.

Cuando incluya el archivo /vendor/autoload.php, devolverá una instancia de Composer Autoloader. Puede almacenar el valor de retorno de la llamada de inclusión en una variable y agregar más espacios de nombres. Esto puede ser útil para la carga automática de clases en un conjunto de pruebas, por ejemplo.

$loader = require __DIR__ . '/vendor/autoload.php';
$loader->add('Application\\Test\\', __DIR__);
