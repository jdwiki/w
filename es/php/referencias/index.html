Sintaxis #

  • $foo = 1; $barra = &$foo; // tanto $foo como $bar apuntan al mismo valor: 1
  • $var = 1; función calc(&$var) { $var *= 15; } calc($var); echo $var;

Al asignar dos variables por referencia, ambas variables apuntan al mismo valor. Tome el siguiente ejemplo:

$foo = 1;
$bar = &$foo;

$foo no apunta a $bar. $foo y $bar ambos apuntan al mismo valor de $foo, que es 1. Para ilustrar:

$baz = &$bar;
unset($bar);
$baz++;

Si tuviéramos una relación apunta a, esta se rompería ahora después de unset(); en cambio, $foo y $baz todavía apuntan al mismo valor, que es 2.

Asignar por referencia #

Esta es la primera fase de la referencia. Esencialmente, cuando asignas por referencia, estás permitiendo que dos variables compartan el mismo valor como tal.

$foo = &$bar;

$foo y $bar son iguales aquí. No se señalan entre sí. Apuntan al mismo lugar (el “valor”).

También puede asignar por referencia dentro de la construcción de lenguaje array(). Si bien no es estrictamente una asignación por referencia.

$foo = 'hi';
$bar = array(1, 2);
$array = array(&$foo, &$bar[0]);

Tenga en cuenta, sin embargo, que las referencias dentro de los arreglos son potencialmente peligroso. Hacer una tarea normal (no por referencia) con un la referencia en el lado derecho no convierte el lado izquierdo en un referencia, pero las referencias dentro de las matrices se conservan en estos normales tareas. Esto también se aplica a las llamadas a funciones donde la matriz es pasado por valor.

La asignación por referencia no solo se limita a variables y matrices, sino que también está presente para funciones y todas las asociaciones de “paso por referencia”.

function incrementArray(&$arr) {
    foreach ($arr as &$val) {
        $val++;
    }
}

function &getArray() {
    static $arr = [1, 2, 3];
    return $arr;
}

incrementArray(getArray());
var_dump(getArray()); // prints an array [2, 3, 4]

La asignación es clave dentro de la definición de función como se indicó anteriormente. No puede pasar una expresión por referencia, solo un valor/variable. De ahí la instanciación de $a en bar().

Devolución por referencia #

Ocasionalmente, llega el momento de que devuelva implícitamente por referencia.

La devolución por referencia es útil cuando desea utilizar una función para encontrar a qué variable debe vincularse una referencia. No utilice retorno por referencia para aumentar el rendimiento. el motor optimice automáticamente esto por su cuenta. Solo devuelva las referencias cuando tiene una razón técnica válida para hacerlo.

Tomado de la Documentación PHP para devolución por referencia.

Hay muchas formas diferentes que puede tomar la devolución por referencia, incluido el siguiente ejemplo:

function parent(&$var) {
    echo $var;
    $var = "updated";
}

function &child() {
    static $a = "test";
    return $a;
}

parent(child()); // returns "test"
parent(child()); // returns "updated"

El retorno por referencia no solo se limita a las referencias a funciones. También tiene la capacidad de llamar implícitamente a la función:

function &myFunction() {
    static $a = 'foo';
    return $a;
}

$bar = &myFunction();
$bar = "updated"
echo myFunction();

No puede referenciar directamente una llamada de función, debe asignarse a una variable antes de aprovecharla. Para ver cómo funciona, simplemente pruebe echo &myFunction();.

Notas #

  • Debe especificar una referencia (&) en ambos lugares en los que pretende utilizarlo. Eso significa, para la definición de su función (function &myFunction() {...) y en la referencia de llamada (function callFunction(&$variable) {... o &myFunction();).
  • Solo puede devolver una variable por referencia. De ahí la instanciación de $a en el ejemplo anterior. Esto significa que no puede devolver una expresión, de lo contrario, se generará un error de PHP E_NOTICE (Aviso: solo las referencias a variables deben devolverse por referencia en ......).
  • La devolución por referencia tiene casos de uso legítimos, pero debo advertir que deben usarse con moderación, solo después de explorar todas las demás opciones potenciales para lograr el mismo objetivo.

Pasar por referencia #

Esto le permite pasar una variable por referencia a una función o elemento que le permite modificar la variable original.

Pasar por referencia no se limita solo a variables, lo siguiente también se puede pasar por referencia:

  • Nuevas declaraciones, p. foo(nuevo SomeClass)
  • Referencias devueltas de funciones.

Matrices #

Un uso común de “passing-by-reference” es modificar los valores iniciales dentro de una matriz sin llegar al extremo de crear nuevas matrices o ensuciar su espacio de nombres. Pasar por referencia es tan simple como preceder/prefijar la variable con & => &$myElement.

A continuación se muestra un ejemplo de aprovechar un elemento de una matriz y simplemente agregar 1 a su valor inicial.

$arr = array(1, 2, 3, 4, 5);

foreach($arr as &$num) {
    $num++;
}

Ahora, cuando aproveche cualquier elemento dentro de $arr, el elemento original se actualizará a medida que se aumente la referencia. Puedes verificar esto por:

print_r($arr);

Nota

Debe tomar nota al aprovechar el paso por referencia dentro de los bucles. Al final del ciclo anterior, $num todavía contiene una referencia al último elemento de la matriz. ¡Asignarlo post loop terminará manipulando el último elemento de la matriz! Puedes asegurarte de que esto no suceda si unset() lo ejecuta después del ciclo:

$miArray = array(1, 2, 3, 4, 5);

foreach($miArray como &$num) { $numero++; } unset($num); Lo anterior asegurará que no se encuentre con ningún problema. Un ejemplo de problemas que podrían relacionarse con esto está presente en esta pregunta en StackOverflow.

Funciones #

Otro uso común para pasar por referencia es dentro de las funciones. Modificar la variable original es tan simple como:

$var = 5;
// define
function add(&$var) {
    $var++;
}
// call
add($var);

Lo cual se puede verificar haciendo eco de la variable original.

echo $var;

Existen varias restricciones en torno a las funciones, como se indica a continuación en los documentos de PHP:

Nota: No hay ningún signo de referencia en una llamada de función, solo en la función definiciones. Las definiciones de función por sí solas son suficientes para pasar correctamente el argumento por referencia. A partir de PHP 5.3.0, recibirá una advertencia decir que “call-time pass-by-reference” está en desuso cuando usa & en foo(&$a);. Y a partir de PHP 5.4.0, el paso por referencia del tiempo de llamada era eliminado, por lo que usarlo generará un error fatal.
