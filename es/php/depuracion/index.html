Volcado de variables #

La función var_dump le permite volcar el contenido de una variable (tipo y valor) para la depuración.

Ejemplo:

$array = [3.7, "string", 10, ["hello" => "world"], false, new DateTime()];
var_dump($array);

Producción:

array(6) {
  [0]=>
  float(3.7)
  [1]=>
  string(6) "string"
  [2]=>
  int(10)
  [3]=>
  array(1) {
    ["hello"]=>
    string(5) "world"
  }
  [4]=>
  bool(false)
  [5]=>
  object(DateTime)#1 (3) {
    ["date"]=>
    string(26) "2016-07-24 13:51:07.000000"
    ["timezone_type"]=>
    int(3)
    ["timezone"]=>
    string(13) "Europe/Berlin"
  }
}

Visualización de errores #

Si desea que PHP muestre errores de tiempo de ejecución en la página, debe habilitar display_errors, ya sea en php.ini o usando la función ini_set.

Puede elegir qué errores mostrar, con la función error_reporting (o en el ini), que acepta constantes E_*, combinadas con operadores bit a bit.

PHP puede mostrar errores en formato de texto o HTML, dependiendo de la configuración de html_errors.

Ejemplo:

ini_set("display_errors", true);
ini_set("html_errors", false); // Display errors in plain text
error_reporting(E_ALL & ~E_USER_NOTICE); // Display everything except E_USER_NOTICE

trigger_error("Pointless error"); // E_USER_NOTICE
echo $nonexistentVariable; // E_NOTICE
nonexistentFunction(); // E_ERROR

Salida de texto sin formato: (el formato HTML difiere entre implementaciones)

Notice: Undefined variable: nonexistentVariable in /path/to/file.php on line 7

Fatal error: Uncaught Error: Call to undefined function nonexistentFunction() in /path/to/file.php:8
Stack trace:
#0 {main}
  thrown in /path/to/file.php on line 8

NOTA: Si tiene deshabilitado el informe de errores en php.ini y lo habilita durante el tiempo de ejecución, algunos errores (como los errores de análisis) no se mostrarán, porque ocurrieron antes de que se aplicara la configuración del tiempo de ejecución.

La forma común de manejar error_reporting es habilitarlo completamente con la constante E_ALL durante el desarrollo, y deshabilitar su visualización pública con display_errors en la etapa de producción para ocultar las partes internas de sus scripts.

phpinfo() #

Advertencia #

Es imperativo que phpinfo solo se use en un entorno de desarrollo. Nunca publiques código que contenga phpinfo en un entorno de producción

Introducción #

Habiendo dicho eso, puede ser una herramienta útil para comprender el entorno PHP (SO, configuración, versiones, rutas, módulos) en el que está trabajando, especialmente cuando busca un error. Es una función incorporada simple:

phpinfo();

Tiene un parámetro $qué que permite personalizar la salida. El valor predeterminado es INFO_ALL, lo que hace que muestre toda la información y se usa comúnmente durante el desarrollo para ver el estado actual de PHP.

Puede pasar el parámetro INFO_* constantes, combinado con operadores bit a bit para ver una lista personalizada.

Puede ejecutarlo en el navegador para obtener una apariencia detallada y bien formateada. También funciona en PHP CLI, donde puede canalizarlo en less para una visualización más fácil.

Ejemplo #

phpinfo(INFO_CONFIGURATION | INFO_ENVIRONMENT | INFO_VARIABLES);

Esto mostrará una lista de directivas de PHP (ini_get), entorno ($_ENV) y variables predefinidas.

Xdebug #

Xdebug es una extensión de PHP que proporciona funciones de depuración y creación de perfiles.
Utiliza el protocolo de depuración DBGp.

Hay algunas características interesantes en esta herramienta:

  • apilar rastros de errores
  • máxima protección de nivel de anidamiento y seguimiento de tiempo
  • Reemplazo útil de la función estándar var_dump() para mostrar variables
  • permite registrar todas las llamadas a funciones, incluidos los parámetros y valores devueltos a un archivo en diferentes formatos
  • análisis de cobertura de código
  • información de perfil
  • depuración remota (proporciona una interfaz para clientes de depuración que interactúan con la ejecución de scripts PHP)

Como puede ver, esta extensión se adapta perfectamente al entorno de desarrollo. Especialmente la función depuración remota puede ayudarlo a depurar su código php sin numerosos var_dump y usar el proceso de depuración normal como en los lenguajes C++ o Java.

Por lo general, la instalación de esta extensión es muy simple:

pecl install xdebug # install from pecl/pear

Y actívelo en su php.ini:

zend_extension="/usr/local/php/modules/xdebug.so"

En casos más complicados ver estas instrucciones

Cuando utilice esta herramienta, debe recordar que:
XDebug no es adecuado para entornos de producción

phpversion() #

Introducción #

Cuando se trabaja con varias bibliotecas y sus requisitos asociados, a menudo es necesario conocer la versión del analizador PHP actual o uno de sus paquetes.

Esta función acepta un solo parámetro opcional en forma de nombre de extensión: phpversion('extension'). Si la extensión en cuestión está instalada, la función devolverá una cadena que contiene el valor de la versión. Sin embargo, si la extensión no está instalada, se devolverá FALSE. Si no se proporciona el nombre de la extensión, la función devolverá la versión del analizador de PHP.

Ejemplo #

print "Current PHP version: " . phpversion();
// Current PHP version: 7.0.8

print "Current cURL version: " . phpversion( 'curl' );
// Current cURL version: 7.0.8
// or
// false, no printed output if package is missing
