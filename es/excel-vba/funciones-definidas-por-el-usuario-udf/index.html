Sintaxis #

  1. Función functionName(argumentVariable As dataType, argumentVariable2 As dataType, Optional argumentVariable3 As dataType) As functionReturnDataType Declaración básica de una función. Cada función necesita un nombre, pero no tiene que tomar ningún argumento. Puede tomar 0 argumentos, o puede tomar un número determinado de argumentos. También puede declarar un argumento como opcional (lo que significa que no importa si lo proporciona al llamar a la función). Es una buena práctica proporcionar el tipo de variable para cada argumento y, de la misma manera, devolver qué tipo de datos devolverá la propia función.

  2. nombreDeFunción = laVariableOValorDevuelto Si viene de otros lenguajes de programación, es posible que esté acostumbrado a la palabra clave Return. Esto no se usa en VBA; en su lugar, usamos el nombre de la función. Puede establecerlo en el contenido de una variable o en algún valor suministrado directamente. Tenga en cuenta que si configuró un tipo de datos para el retorno de la función, la variable o los datos que está proporcionando esta vez deben ser de ese tipo de datos.

  3. Función final Obligatorio. Significa el final del bloque de código Función y, por lo tanto, debe estar al final. El VBE generalmente proporciona esto automáticamente cuando crea una nueva función.

Una función definida por el usuario (también conocida como UDF) se refiere a una función específica de tarea que ha sido creada por el usuario. Se puede llamar como una función de hoja de cálculo (por ejemplo: =SUM(...)) o usarse para devolver un valor a un proceso en ejecución en un procedimiento Sub. Una UDF devuelve un valor, normalmente a partir de la información que se le pasa como uno o más parámetros.

Puede ser creado por:

  1. usando VBA.
  2. utilizando la API de Excel C: mediante la creación de un XLL que exporta funciones compiladas a Excel.
  3. usando la interfaz COM.

Permitir referencias de columnas completas sin penalización #

Es más fácil implementar algunas UDF en la hoja de trabajo si se pueden pasar referencias de columnas completas como parámetros. Sin embargo, debido a la naturaleza explícita de la codificación, cualquier ciclo que involucre estos rangos puede estar procesando cientos de miles de celdas que están completamente vacías. Esto reduce su proyecto de VBA (y libro de trabajo) a un desastre congelado mientras se procesan los no valores innecesarios.

Recorrer las celdas de una hoja de trabajo es uno de los métodos más lentos para realizar una tarea, pero a veces es inevitable. Reducir el trabajo realizado a lo que realmente se requiere tiene mucho sentido.

La solución es truncar las referencias de columna o fila completa a la propiedad Worksheet.UsedRange con el método Intersect. El siguiente ejemplo replicará vagamente la función SUMIF nativa de una hoja de trabajo, por lo que criteria_range también cambiará de tamaño para adaptarse a sum_range, ya que cada valor en sum_range debe ir acompañado de un valor en criteria_range.

Application.Caller para una UDF utilizada en una hoja de cálculo es la celda en la que reside. La propiedad .Parent de la celda es la hoja de cálculo. Esto se usará para definir el .UsedRange.

En una hoja de código de módulo:

Option Explicit

Function udfMySumIf(rngA As Range, rngB As Range, _
                    Optional crit As Variant = "yes")
    Dim c As Long, ttl As Double
    
    With Application.Caller.Parent
        Set rngA = Intersect(rngA, .UsedRange)
        Set rngB = rngB.Resize(rngA.Rows.Count, rngA.Columns.Count)
    End With
    
    For c = 1 To rngA.Cells.Count
        If IsNumeric(rngA.Cells(c).Value2) Then
            If LCase(rngB(c).Value2) = LCase(crit) Then
                ttl = ttl + rngA.Cells(c).Value2
            End If
        End If
    Next c
    
    udfMySumIf = ttl

End Function

Sintaxis:
         =udfMySumIf(*rango_suma*, *rango_criterios*, [*criterios*])

Si bien este es un ejemplo bastante simple, demuestra adecuadamente el paso de referencias de dos columnas completas (1,048,576 filas cada una) pero solo procesa 15 filas de datos y criterios.

UDF - Hola mundo #

  1. Abre Excel
  2. Abra el Editor de Visual Basic (consulte https://www.wikiod.com/es/excel-vba/primeros-pasos-con-excel-vba#Abrir el Editor de Visual Basic (VBE))
  3. Agregue un nuevo módulo haciendo clic en Insertar –> Módulo:

  1. Copie y pegue el siguiente código en el nuevo módulo:

    Public Function Hello() As String ‘Note: the output of the function is simply the function’s name Hello = “Hello, World !” End Function

Para obtener :

  1. Vuelva a su libro de trabajo y escriba “=Hello()” en una celda para ver “Hello World”.

Contar valores únicos en el rango #

Function countUnique(r As range) As Long
    'Application.Volatile False ' optional
    Set r = Intersect(r, r.Worksheet.UsedRange) ' optional if you pass entire rows or columns to the function
    Dim c As New Collection, v
    On Error Resume Next   ' to ignore the Run-time error 457: "This key is already associated with an element of this collection".
    For Each v In r.Value  ' remove .Value for ranges with more than one Areas
        c.Add 0, v & ""
    Next
    c.Remove "" ' optional to exclude blank values from the count
    countUnique = c.Count
End Function
