Este tema consta de una amplia variedad de consejos y trucos útiles descubiertos por los usuarios de SO a través de su experiencia en la codificación. A menudo, estos son ejemplos de formas de eludir las frustraciones comunes o formas de usar Excel de una manera más “inteligente”.

Usar hojas xlVeryHidden #

Las hojas de trabajo en Excel tienen tres opciones para la propiedad Visible. Estas opciones están representadas por constantes en la enumeración xlSheetVisibility y son las siguientes:

  1. Valor xlVisible o xlSheetVisible: -1 (el valor predeterminado para hojas nuevas)
  2. Valor xlHidden o xlSheetHidden: 0
  3. xlVeryHidden xlSheetVeryHidden valor: 2

Las hojas visibles representan la visibilidad predeterminada para las hojas. Son visibles en la barra de pestañas de la hoja y se pueden seleccionar y ver libremente. Las hojas ocultas están ocultas en la barra de pestañas de hojas y, por lo tanto, no se pueden seleccionar. Sin embargo, las hojas ocultas se pueden mostrar desde la ventana de Excel haciendo clic con el botón derecho en las pestañas de las hojas y seleccionando “Mostrar”.

Las hojas muy ocultas, por otro lado, son solo accesibles a través del Editor de Visual Basic. Esto los convierte en una herramienta increíblemente útil para almacenar datos en instancias de Excel, así como para almacenar datos que deben ocultarse a los usuarios finales. Se puede acceder a las hojas por referencia nombrada dentro del código VBA, lo que permite un fácil uso de los datos almacenados.

Para cambiar manualmente la propiedad .Visible de una hoja de trabajo a xlSheetVeryHidden, abra la ventana Propiedades de VBE (F4), seleccione la hoja de trabajo que desea cambiar y use el menú desplegable en la decimotercera fila para hacer su selección.

Para cambiar la propiedad .Visible de una hoja de trabajo a xlSheetVeryHidden¹ en el código, acceda de manera similar a la propiedad .Visible y asigne un nuevo valor.

with Sheet3
    .Visible = xlSheetVeryHidden
end with

¹ Tanto xlVeryHidden como xlSheetVeryHidden devuelven un valor numérico de 2 (son intercambiables).

Uso de cadenas con delimitadores en lugar de matrices dinámicas #

El uso de matrices dinámicas en VBA puede ser bastante complicado y consumir mucho tiempo en conjuntos de datos muy grandes. Al almacenar tipos de datos simples en una matriz dinámica (cadenas, números, valores booleanos, etc.), se pueden evitar las declaraciones ReDim Preserve requeridas para las matrices dinámicas en VBA mediante el uso de la función Split() con algunos procedimientos de cadena inteligentes. Por ejemplo, veremos un ciclo que agrega una serie de valores de un rango a una cadena en función de algunas condiciones, luego usa esa cadena para completar los valores de un ListBox.

Private Sub UserForm_Initialize()

Dim Count As Long, DataString As String, Delimiter As String

For Count = 1 To ActiveSheet.UsedRows.Count
    If ActiveSheet.Range("A" & Count).Value <> "Your Condition" Then
        RowString = RowString & Delimiter & ActiveSheet.Range("A" & Count).Value
        Delimiter = "><" 'By setting the delimiter here in the loop, you prevent an extra occurance of the delimiter within the string
    End If
Next Count

ListBox1.List = Split(DataString, Delimiter)

End Sub

La propia cadena Delimiter se puede establecer en cualquier valor, pero es prudente elegir un valor que no se presente de forma natural dentro del conjunto. Digamos, por ejemplo, que estaba procesando una columna de fechas. En ese caso, usar ., - o / sería imprudente como delimitadores, ya que las fechas podrían formatearse para usar cualquiera de estos, generando más puntos de datos de los que esperaba.

Nota: Existen limitaciones en el uso de este método (a saber, la longitud máxima de las cadenas), por lo que debe usarse con precaución en casos de conjuntos de datos muy grandes. Este no es necesariamente el método más rápido o efectivo para crear arreglos dinámicos en VBA, pero es una alternativa viable.

Hoja de trabajo .Nombre, .Índice o .CodeName #

Sabemos que la ‘mejor práctica’ dicta que un objeto de rango debe tener su hoja de trabajo principal referenciada explícitamente. Se puede hacer referencia a una hoja de trabajo por su propiedad .Name, propiedad numérica .Index o su propiedad .CodeName, pero un usuario puede reordenar la cola de la hoja de trabajo simplemente arrastrando una pestaña de nombre o cambiar el nombre de la hoja de trabajo con un doble clic en la misma pestaña y algunos escribir en un libro desprotegido.

Considere una hoja de trabajo estándar de tres. Cambió el nombre de las tres hojas de trabajo Lunes, Martes y Miércoles en ese orden y codificó los subprocedimientos de VBA que hacen referencia a estos. Ahora considere que un usuario llega y decide que el lunes pertenece al final de la cola de hojas de trabajo, luego llega otro y decide que los nombres de las hojas de trabajo se ven mejor en francés. Ahora tiene un libro de trabajo con una cola de pestañas de nombre de hoja de trabajo similar a la siguiente.

Si hubiera utilizado cualquiera de los siguientes métodos de referencia de la hoja de trabajo, su código ahora estaría roto.

'reference worksheet by .Name
with worksheets("Monday")
    'operation code here; for example:
    .Range(.Cells(2, "A"), .Cells(.Rows.Count, "A").End(xlUp)) = 1
end with

'reference worksheet by ordinal .Index
with worksheets(1)
    'operation code here; for example:
    .Range(.Cells(2, "A"), .Cells(.Rows.Count, "A").End(xlUp)) = 1
end with

Tanto el pedido original como el nombre de la hoja de trabajo original se han visto comprometidos. Sin embargo, si hubiera utilizado la propiedad .CodeName de la hoja de trabajo, su subprocedimiento aún estaría operativo

with Sheet1
    'operation code here; for example:
    .Range(.Cells(2, "A"), .Cells(.Rows.Count, "A").End(xlUp)) = 1
end with

La siguiente imagen muestra la ventana Proyecto VBA ([Ctrl]+R) que enumera las hojas de trabajo por .CodeName y luego por .Name (entre paréntesis). El orden en que se muestran no cambia; el índice ordinal se toma por el orden en que se muestran en la cola de la pestaña de nombre en la ventana de la hoja de trabajo.

Si bien es poco común cambiar el nombre de un .CodeName, no es imposible. Simplemente abra la ventana Propiedades de VBE ([F4]).

La hoja de trabajo .CodeName está en la primera fila. El .Name de la hoja de trabajo está en el décimo. Ambos son editables.

Evento de doble clic para formas de Excel #

De forma predeterminada, las formas en Excel no tienen una forma específica de manejar los clics simples o dobles, y contienen solo la propiedad “OnAction” para permitirle manejar los clics. Sin embargo, puede haber instancias en las que su código requiera que actúe de manera diferente (o exclusivamente) en un doble clic. La siguiente subrutina se puede agregar a su proyecto de VBA y, cuando se configura como la rutina OnAction para su forma, le permite actuar con doble clic.

Public Const DOUBLECLICK_WAIT as Double = 0.25 'Modify to adjust click delay
Public LastClickObj As String, LastClickTime As Date

Sub ShapeDoubleClick()
    
    If LastClickObj = "" Then
        LastClickObj = Application.Caller
        LastClickTime = CDbl(Timer)
    Else
        If CDbl(Timer) - LastClickTime > DOUBLECLICK_WAIT Then
            LastClickObj = Application.Caller
            LastClickTime = CDbl(Timer)
        Else
            If LastClickObj = Application.Caller Then
                'Your desired Double Click code here
                LastClickObj = ""
            Else
                LastClickObj = Application.Caller
                LastClickTime = CDbl(Timer)
            End If
        End If
    End If
    
End Sub

Esta rutina hará que la forma ignore funcionalmente el primer clic y solo ejecute el código deseado en el segundo clic dentro del período de tiempo especificado.

Cuadro de diálogo Abrir archivo - Múltiples archivos #

Esta subrutina es un ejemplo rápido de cómo permitir que un usuario seleccione varios archivos y luego haga algo con esas rutas de archivo, como obtener los nombres de los archivos y enviarlos a la consola a través de debug.print.

Option Explicit

Sub OpenMultipleFiles()
    Dim fd As FileDialog
    Dim fileChosen As Integer
    Dim i As Integer
    Dim basename As String
    Dim fso As Variant
    Set fso = CreateObject("Scripting.FileSystemObject")
    Set fd = Application.FileDialog(msoFileDialogFilePicker)
    basename = fso.getBaseName(ActiveWorkbook.Name)
    fd.InitialFileName = ActiveWorkbook.Path ' Set Default Location to the Active Workbook Path
    fd.InitialView = msoFileDialogViewList
    fd.AllowMultiSelect = True
    
    fileChosen = fd.Show
    If fileChosen = -1 Then
        'open each of the files chosen
        For i = 1 To fd.SelectedItems.Count
            Debug.Print (fd.SelectedItems(i))
            Dim fileName As String
            ' do something with the files.
            fileName = fso.getFileName(fd.SelectedItems(i))
            Debug.Print (fileName)
        Next i
    End If

End Sub
