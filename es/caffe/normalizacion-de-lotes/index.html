De [los documentos] (http://caffe.berkeleyvision.org/doxygen/classcaffe_1_1BatchNormLayer.html):

“Normaliza la entrada para tener una variación de media 0 y/o unidad (1) en todo el lote.

Esta capa calcula la normalización por lotes como se describe en 1.

[…]

1 S. Ioffe y C. Szegedy, “Normalización por lotes: Aceleración del entrenamiento profundo de redes al reducir el cambio interno de covariables”. preimpresión de arXiv arXiv:1502.03167 (2015).”

Parámetros #

Parámetro Detalles
usar_estadísticas_globales [De la publicación de Rohrbach del 2 de marzo de 2016] (https://github.com/BVLC/caffe/issues/3347) - tal vez él sepa:
(use_global_stats) “De forma predeterminada, durante el tiempo de entrenamiento, la red calcula estadísticas de varianza/media global a través de un promedio móvil, que luego se usa en el momento de la prueba para permitir salidas deterministas para cada entrada. Puede alternar manualmente si la red está acumulando o usando las estadísticas a través de la opción use_global_stats.IMPORTANTE: para que esta función funcione, DEBE establecer la tasa de aprendizaje en cero para los tres blobs de parámetros, es decir, param {lr_mult: 0} tres veces en la definición de capa.
(use_global_stats) Esto significa que, de forma predeterminada (ya que lo siguiente está configurado en batch_norm_layer.cpp), no tiene que configurar use_global_stats en absoluto en el prototxt. use_global_stats_ = esto->fase_ == PRUEBA;”

Prototexto para entrenamiento #

La siguiente es una definición de ejemplo para entrenar una capa BatchNorm con escala y sesgo por canal. Por lo general, se inserta una capa BatchNorm entre las capas de convolución y rectificación. En este ejemplo, la convolución generaría el blob layerx y la rectificación recibiría el blob layerx-bn.

layer { bottom: 'layerx' top: 'layerx-bn' name: 'layerx-bn' type: 'BatchNorm'
  batch_norm_param {
    use_global_stats: false  # calculate the mean and variance for each mini-batch
    moving_average_fraction: .999  # doesn't effect training 
  }
  param { lr_mult: 0 } 
  param { lr_mult: 0 } 
  param { lr_mult: 0 }}
# channel-wise scale and bias are separate
layer { bottom: 'layerx-bn' top: 'layerx-bn' name: 'layerx-bn-scale' type: 'Scale',
  scale_param { 
    bias_term: true
    axis: 1      # scale separately for each channel
    num_axes: 1  # ... but not spatially (default)
    filler { type: 'constant' value: 1 }           # initialize scaling to 1
    bias_filler { type: 'constant' value: 0.001 }  # initialize bias
}}

Se puede encontrar más información en este hilo.

Prototexto para implementación #

El principal cambio necesario es cambiar use_global_stats a true. Esto cambia a usar el promedio móvil.

layer { bottom: 'layerx' top: 'layerx-bn' name: 'layerx-bn' type: 'BatchNorm'
  batch_norm_param {
    use_global_stats: true  # use pre-calculated average and variance
  }
  param { lr_mult: 0 } 
  param { lr_mult: 0 } 
  param { lr_mult: 0 }}
# channel-wise scale and bias are separate
layer { bottom: 'layerx-bn' top: 'layerx-bn' name: 'layerx-bn-scale' type: 'Scale',
  scale_param { 
    bias_term: true
    axis: 1      # scale separately for each channel
    num_axes: 1  # ... but not spatially (default)
}}
