Elija el enfoque de firma de código correcto #

Si recién está comenzando un nuevo proyecto, es importante pensar en cómo desea manejar la firma de código.

Si es nuevo en la firma de código, consulte la sesión WWDC que describe los fundamentos de la firma de código en Xcode.

Para firmar correctamente el código de su aplicación, debe tener los siguientes recursos en su máquina local:

  • La clave privada (archivo .p12)
  • El certificado (archivo .cer), que coincide con la clave privada
  • El perfil de aprovisionamiento (archivo .mobileprovision), que coincide con el certificado y la clave privada instalada localmente

En el Portal para desarrolladores de Apple, también se requiere tener una ID de aplicación válida asociada con su perfil de aprovisionamiento.

Uso de la función de firma de código de Xcode #

Ocasionalmente, la configuración “Automático” como perfil de aprovisionamiento no funciona de manera confiable, ya que solo seleccionará el perfil de aprovisionamiento actualizado más recientemente, sin importar si el certificado está instalado.

Es por eso que se recomienda especificar un perfil de aprovisionamiento específico de alguna manera:

Xcode 7 e inferior #

Debe evitar hacer clic en el botón “Reparar problema” (hay un [complemento de Xcode] (https://github.com/neonichu/FixCode#readme) que desactiva el botón), ya que a veces revoca los certificados existentes y, con ello, el perfiles de aprovisionamiento.

Lamentablemente, no puede especificar el nombre del perfil de aprovisionamiento en Xcode 7. En su lugar, puede especificar el UUID del perfil, que cambia cada vez que se vuelve a generar el perfil (por ejemplo, cuando agrega un nuevo dispositivo).

Para solucionar este problema, consulte XcodeProject.md sobre cómo pasar un perfil de aprovisionamiento a Xcode cuando construyendo tu aplicación.

Xcode 8 y superior #

Apple mejoró mucho la firma de código con el lanzamiento de Xcode 8, lo siguiente ha cambiado:

  • No más botón “Reparar problema”, sino que todos los procesos de firma de código se ejecutan en segundo plano y muestran el registro directamente en Xcode
  • Ahora puede especificar el perfil de aprovisionamiento por nombre, en lugar del UUID (Consulte XcodeProject.md para más información)
  • Mensajes de error mejorados cuando algo sale mal. Si se encuentra con errores de firma de código, siempre debe intentar compilar y firmar con Xcode para obtener información de error más detallada. (Consulte Troubleshooting.md para obtener más información)

Manualmente #

Siempre puede crear y administrar manualmente sus certificados y perfiles de aprovisionamiento mediante el Portal para desarrolladores de Apple. Asegúrese de almacenar la clave privada (.p12) de sus certificados en un lugar seguro, ya que no se pueden restaurar si los pierde.

Siempre puede descargar el certificado (.cer) y el perfil de aprovisionamiento (.mobileprovision) desde Apple Developer Portal.

Si revoca su certificado o caduca, todos los perfiles de aprovisionamiento asociados no serán válidos.

Usar [coincidencia de carril rápido] (https://fastlane.tools/match) #

El concepto de coincidencia se describe en la guía de codiseño y es el enfoque de firma de código recomendado si usa [fastlane](https: //vía rápida.herramientas)

Con match almacena sus claves privadas y certificados en un repositorio git para sincronizarlos entre máquinas. Esto facilita la incorporación de nuevos miembros del equipo y la configuración de nuevas máquinas Mac. Este enfoque es seguro y usa la tecnología que ya usa.

Comenzar con match requiere que revoque sus certificados existentes.
