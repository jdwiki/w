Asignaciones Xml #

El mapeo xml usa un archivo hbm.xml que es un archivo de mapeo de hibernación. Es un archivo xml de sintaxis que contiene los metadatos necesarios para el mapeo objeto/relacional. Los metadatos incluyen la declaración de clases persistentes y el mapeo de propiedades (a columnas y relaciones de clave externa a otras entidades) a tablas de bases de datos.

Agregue un archivo llamado Entity.hbm.xml al proyecto y configúrelo como recurso incrustado en la pestaña de propiedades. Para la muestra, Customer.hbm.xml:

<?xml version="1.0" encoding="utf-8" ?>
<hibernate-mapping xmlns="urn:nhibernate-mapping-2.2"
    namespace="Project" assembly="Project">

    <class name="Customer" table="CUSTOMERS">

        <id name="Id">
            <column name="Customer_Id" sql-type="int" not-null="true"/>
            <generator class="native" />
        </id>

        <!-- A cat has to have a name, but it shouldn' be too long. -->
        <property name="Name">
            <column name="Name" length="60" not-null="true" />
        </property>
        <property name="Sex" />
        <property name="Weight" />
        <property name="Active" />
        <property name="Birthday" />
    </class>

</hibernate-mapping>

La etiqueta hibernate-mapping contiene el espacio de nombres y la información del proyecto de ensamblaje. La etiqueta clase contiene el nombre de la entidad en el proyecto y la tabla que se ha mapeado. La etiqueta id contiene el mapeo para la clave primaria donde la columna se especifica mediante la etiqueta column y la etiqueta generator define cómo se genera la identificación. La etiqueta propiedad contiene información para las otras columnas de la base de datos.

Asignaciones fluidas de NHibernate #

Fluent NHibernate es una biblioteca que lo ayuda a mapear las entidades usando código C# en lugar de mapeos xml. Fluent NHibernate usa el patrón fluido y se basa en convenciones para crear las asignaciones y le brinda el poder de las herramientas de Visual Studio (como intellisense ) para mejorar la forma en que asigna sus entidades.

Agregue la referencia de Fluent NHibernate from Nuget en su proyecto y agregue una clase CustomerMap.cs:

namespace Project.Mappings
{
    public class CustomerMap : ClassMap<Customer>
    {
        public CustomerMap()
        {
            Table("CUSTOMERS");

            Id(x => x.Id).Column("Customer_Id").GeneratedBy.Native();
            
            //map a property while specifying the max-length as well as setting 
            //it as not nullable. Will result in the backing column having
            //these characteristics, but this will not be enforced in the model!
            Map(x => x.Name)
                .Length(16)
                .Not.Nullable();

            Map(x => x.Sex);

            Map(x => x.Weight);

            Map(x => x.Active);

            //Map a property while specifying the name of the column in the database
            Map(x => x.Birthday, "BIRTHDAY");

            //Maps a many-to-one relationship
            References(x => x.Company);

            //Maps a one-to-many relationship, while also defining which 
            //column to use as key in the foreign table.
            HasMany(x => x.Orders).KeyColumn("CustomerPk");
        }
    }
}

La clase CustomerMap hereda de ClassMap<T> que es la clase base para el mapeo y contiene todos los métodos necesarios para crear el mapa de su entidad T. El método Table define el nombre de la tabla que está mapeando. El método Id se usa para mapear la columna primery key. El método Map se usa para mapear otras columnas.

Una muestra de Model to Map #

NHibernate usa clases para mapear en tablas o vistas. Crear un Objeto CLR antiguo simple (POCO, a veces llamados Objetos CLR ordinarios simples) es una buena práctica para las clases persistentes. A POCO se puede acceder a sus datos a través de los mecanismos de propiedad estándar de .NET, protegiendo la representación interna de la interfaz públicamente visible.

namespace Project
{
    public class Customer
    {
        public virtual string Id { get; set; }

        public virtual string Name { get; set; }

        public virtual char Sex { get; set; }

        public virtual float Weight { get; set;}

        public virtual bool Active { get; set;}

        public virtual DateTime Birthday { get; set;}

        public Customer()
        {
        }
    }
}

NHibernate no está restringido en el uso de tipos de propiedades: todos los tipos y primitivos de .NET (como string, char y DateTime) se pueden mapear, incluidas las clases de los espacios de nombres System.Collections y System.Collections.Generics. También puede mapear una relación entre las entidades, con propiedades que se refieran a otro tipo de entidad. Puede asignarlos como valores, colecciones de valores o asociaciones a otras entidades. La propiedad llamada Id aquí es una propiedad especial que representa el identificador de la base de datos (clave principal) de esa clase, que es muy recomendable para entidades como Cat. NHibernate puede usar identificadores solo internamente, sin tener que declararlos en la clase, pero perderíamos parte de la flexibilidad en la arquitectura de nuestra aplicación.

No es necesario implementar una interfaz especial para las clases persistentes ni tenemos que crear una subclase de una clase persistente raíz especial. NHibernate tampoco utiliza ningún procesamiento de tiempo de compilación, como la manipulación de IL; se basa únicamente en la reflexión de .NET y la mejora de la clase de tiempo de ejecución. Entonces, sin ninguna dependencia en la clase POCO en NHibernate, podemos asignarlo a una tabla o vista de base de datos.

Para que funcione la mejora de la clase de tiempo de ejecución mencionada anteriormente, NHibernate requiere que todas las propiedades públicas de una clase de entidad se declaren como “virtuales”. La clase de entidad debe tener un constructor sin argumentos (protegido o público) para que NHibernate cree los objetos.
