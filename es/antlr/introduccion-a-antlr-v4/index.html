ANTLR v4 es una poderosa herramienta utilizada para construir nuevos lenguajes de programación y procesar/traducir texto estructurado o archivos binarios. ANTLR usa una gramática que usted crea para generar un analizador que puede construir y atravesar un árbol de análisis (o árbol de sintaxis abstracta, AST). El analizador consta de archivos de salida en un idioma de destino que especifique. ANTLR v4 admite varios objetivos, incluidos: Java, C#, JavaScript, Python2 y Python3. Se está trabajando en el soporte para C++. Para trabajar en IDE de GUI, existen complementos para Visual Studio, Intellij, NetBeans y Eclipse.

Para obtener información general, visite el sitio web de ANTLR. Para hablar en serio sobre ANTLR, consulte el libro altamente recomendado escrito por Terrence Parr (el tipo que creó ANTLR) The Definitive ANTLR 4 Reference.

Información importante de la versión

  • 4.5: 22/01/15 - Se agregó el destino de JavaScript y se actualizó el destino de C#. 4.5 Notas de la versión
  • 4.4: 16/07/14 - Se agregaron Python2 y Python3 como objetivos. 4.4 Notas de la versión
  • 4.3: 18/06/14 - Corrección de errores importantes; preparados para agregar nuevos objetivos. 4.3 Notas de la versión
  • 4.2: 04/02/14 - Sintaxis mejorada para seleccionar/coincidir árboles de análisis. 4.2 Notas de la versión
  • 4.1: 30/06/13 - Rendimiento de análisis mejorado; exportar AST a PNG. 4.1 Notas de la versión
  • 4.0: 21/01/13 - Lanzamiento inicial.

Instalación de ANTLR en Visual Studio 2015 (usando Nuget) #

  1. Abra Visual Studio 2015, vaya a Herramientas → Extensiones → En línea y busque Antlr. Descargue la extensión ANTLR Language Support (creada por Sam Harwell) y reinicie Visual Studio.
  2. Cree un nuevo proyecto de aplicación de consola. Haga clic con el botón derecho en Solución → Administrar paquetes Nuget para la solución → Examinar (pestaña) y busque Antlr4 e instálelo.
  3. Agregue un nuevo elemento a su proyecto haciendo clic derecho sobre él. Y busque Plantillas ANTLR4.
  4. Desde su archivo ANTLR (que termina en .g4), vaya a Archivo → Opciones avanzadas de guardado y busque Unicode (UTF-8 sin firma) - Página de códigos 65001 y haga clic en Aceptar. Eso es todo.

Prueba si todo funciona

  • Cree un elemento de gramática combinada de ANTLR 4 y asígnele el nombre Calculator.g4
  • Copie y pegue el código fuente de la Calculadora de este proyecto de Github aquí: Calculadora de Tom Everett
  • Cambiar calculadora de gramática a Calculadora de gramática
  • En el Explorador de soluciones → Haga clic en Mostrar todos los archivos.
  • Guardar y Ejecutar (Iniciar) el proyecto
  • En Solution Explorer, en la carpeta obj, debería ver las clases cs generadas como Visitor y Listener. Si este es el caso, lo lograste. Ahora puede comenzar a trabajar con ANTLR en Visual Studio 2015.

Instalación para el uso de la línea de comandos #

ANTLR se distribuye como un archivo Java Jar. Se puede descargar aquí. Como ANTLR se compila como un archivo jar, posteriormente requiere el entorno de tiempo de ejecución de Java para funcionar, si no lo tiene, puede descargarlo aquí.

Una vez que se haya descargado el archivo JAR de ANTLR, puede ejecutar ANTLR desde la línea de comandos de la misma manera que cualquier otro archivo JAR:

Java -jar antlr-4.5.3-complete.jar

(Suponiendo que esté operando en el mismo directorio que el archivo antlr-4.5.3-complete.jar).

Esto debería generar algo similar a esto:

ANTLR Parser Generator  Version 4.5.3
 -o ___              specify output directory where all output is generated
 -lib ___            specify location of grammars, tokens files
 -atn                generate rule augmented transition network diagrams
 -encoding ___       specify grammar file encoding; e.g., euc-jp
 -message-format ___ specify output style for messages in antlr, gnu, vs2005
 -long-messages      show exception details when available for errors and warnings
 -listener           generate parse tree listener (default)
 -no-listener        don't generate parse tree listener
 -visitor            generate parse tree visitor
 -no-visitor         don't generate parse tree visitor (default)
 -package ___        specify a package/namespace for the generated code
 -depend             generate file dependencies
 -D<option>=value    set/override a grammar-level option
 -Werror             treat warnings as errors
 -XdbgST             launch StringTemplate visualizer on generated code
 -XdbgSTWait         wait for STViz to close before continuing
 -Xforce-atn         use the ATN simulator for all predictions
 -Xlog               dump lots of logging info to antlr-timestamp.log

otras acciones recomendadas para la configuración incluyen:

     1. Add antlr4-complete.jar to CLASSPATH, either: Permanently: 
Using System Properties dialog > Environment variables > Create or append to CLASSPATH variable Temporarily, at command line: SET CLASSPATH=.;C:\Javalib\antlr4-complete.jar;%CLASSPATH% 
     3.Create batch commands for ANTLR Tool, TestRig in dir in PATH
         antlr4.bat: java org.antlr.v4.Tool %*
         grun.bat:   java org.antlr.v4.gui.TestRig %*

Después de la configuración, puede crear una aplicación utilizando su archivo de gramática .g4:

Java -jar antlr-4.5.3-complete.jar yourGrammar.g4

También puede crear una aplicación en otros idiomas con el parámetro -Dlanguage. Por ejemplo, para generar archivos C#, haría algo como esto:

java -jar antlr-4.5.3-complete.jar yourGrammar.g4 -Dlanguage=CSharp

Consulte aquí para obtener una lista completa de gramáticas prefabricadas para lenguajes de programación comunes.

Instalación usando herramientas de automatización de compilación #

Descargue la última versión de ANTLR y extráigala a una carpeta.

También puede usar Maven, Gradle u otra herramienta de compilación para depender de su tiempo de ejecución (las clases que usan las gramáticas generadas): org.antlr:antlr4-runtime.

Para generar automáticamente, como parte del proceso de compilación, el analizador en un proyecto maven, use el [complemento Maven] (http://www.antlr.org/api/maven-plugin/latest/index.html): org.antlr:antlr4.

Instalar en Eclipse y compilar Hello World #

(Probado con ANTLR 4.5.3, Eclipse Neon, ANTLR 4 IDE 0.3.5 y Java 1.8)

  1. Descargue el ANTLR más reciente. Asegúrese de obtener el contenedor completo de binarios Java de ANTLR. Guarde en cualquier ubicación adecuada, por ejemplo, la carpeta donde se almacenan otras bibliotecas de Java. No importa dónde, solo recuerda la ubicación.

  2. Instale el IDE de ANTLR en Eclipse.

    • From the Eclipse menu, click Help and select Eclipse Marketplace.
    • In the Find: box, type antlr and click Go.
    • Click Install for ANTLR 4 IDE.
    • Click Finish in the Confirm Selected Features window.
    • If a Security Warning window pops up, click OK.
    • Restart Eclipse.

  3. Solucione el error “Error al crear el inyector…”.

    • When accessing ANTLR 4 Preferences in Eclipse or when the environment variable HOME is not set, the following error occurs: Failed to create injector for com.github.jknack.antlr-4ide.Antlr4 for com.github.jknack.antlr-4ide.Antlr4.
    • Make sure the environment variable HOME is set. If not, set it as appropriate for your system.
    • Download Xtext 2.7.3 to the same location as antlr-n.n.n-complete.jar.
    • In Eclipse, click on Help and select Install New Software.
    • Click Add… to get to the Add Repository window.
    • Type a name, xtext 2.7.3 for example, then click on Archive…, navigate to the Xtext 2.7.3 file and select it, then click OK.
    • In the Install window, click the Select All button then click Next> twice, accept the license agreement. and click Finish.
    • Restart Eclipse.

  4. Dile a Eclipse/Java dónde está ANTLR.

    • In Eclipse, click on Window and select Preferences.
    • In the left pane, expand Java and Build Path, then select Classpath Variables.
    • In the right pane, click New…, enter a Name, and click File… and browse to your location of antlr-n.n.n-complete.jar. Click OK to get back to the Classpath Variables window.
    • Click OK to exit Preferences.

  5. (Opcional) Configure el directorio de fuentes generadas por ANTLR IDE.

    • In the Eclipse main window, click Window then Preferences.
    • In the left pane, expand ANTLR 4 and select Tool.
    • Under Options, change the directory if desired. For example, java is my target language so I use ./antlr-java.
    • Click OK to close the Preferences window.

  6. Cree un proyecto ANTLR 4.

    • From the Eclipse main window, go to File, New, Project.
    • In the New Project window, expand General and select ANTLR 4 Project.
    • Click Next, type a project name and click Finish.
    • The default new project contains a Hello.g4 file and will automatically build the standard “Hello World” program.
    • In the Package Explorer, expand the new project folder to see the g4 file and a folder named target (or the name you gave it in Step 5) containing the target source files.
