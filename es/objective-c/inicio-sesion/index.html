Sintaxis #

  • NSLog(@“texto para registrar”); // Registro de texto básico
  • NSLog(@“datos: %f - %.2f”, myFloat, anotherFloat); // Registro de texto que incluye números flotantes.
  • NSLog(@“datos: %i”, miEntero); // Registro de texto que incluye un número entero.
  • NSLog(@“datos: %@”, myStringOrObject); // Registro de texto que hace referencia a otro String o cualquier objeto derivado de NSObject.

Para registrar varios tipos de objetos y tipos de datos, consulte: Objetivo-C, Especificadores de formato

Inicio sesión #

NSLog(@"Log Message!");
NSLog(@"NSString value: %@", stringValue);
NSLog(@"Integer value: %d", intValue);

El primer argumento de NSLog es un NSString que contiene el formato del mensaje de registro. El resto de los parámetros se utilizan como valores para sustituir en lugar de los especificadores de formato.

El formato funciona exactamente igual que printf, excepto por el especificador de formato adicional %@ para un objeto Objective-C arbitrario. Este:

NSLog(@"%@", object);

es equivalente a:

NSLog(@"%s", [object description].UTF8String);

Formato de salida de NSLog #

NSLog(@"NSLog message");

El mensaje que se imprime llamando a NSLog tiene el siguiente formato cuando se ve en Console.app:

Fecha Tiempo Nombre del programa ID de proceso ID de subproceso Mensaje
2016-07-16 08:58:04.681 prueba [46259 : 1244773] Mensaje NSLog

Registro de valores de variables #

No deberías llamar a NSLog sin una cadena de formato literal como esta:

NSLog(variable);    // Dangerous code!

Si la variable no es un NSString, el programa fallará, porque NSLog espera un NSString.

Si la variable es un NSString, funcionará a menos que su cadena contenga un %. NSLog analizará la secuencia % como un especificador de formato y luego leerá un valor basura de la pila, provocando un bloqueo o incluso [ejecutando código arbitrario] (https://en.wikipedia.org/wiki/Unected_format_string).

En su lugar, siempre haga que el primer argumento sea un especificador de formato, como este:

NSLog(@"%@", anObjectVariable);
NSLog(@"%d", anIntegerVariable);

Eliminación de declaraciones de registro de compilaciones de lanzamiento #

Los mensajes impresos desde NSLog se muestran en Console.app incluso en la versión de lanzamiento de su aplicación, lo que no tiene sentido para las impresiones que solo son útiles para la depuración. Para arreglar esto, puede usar esta macro para el registro de depuración en lugar de NSLog.

#ifdef DEBUG
#define DLog(...) NSLog(__VA_ARGS__)
#else
#define DLog(...)
#endif

Usar:

NSString *value = @"value 1";
DLog(@"value = %@", value);
// little known fact: programmers look for job postings in Console.app
NSLog(@"We're hiring!");

En compilaciones de depuración, DLog llamará a NSLog. En versiones de lanzamiento, DLog no hará nada.

El mensaje vacío no se imprime #

Cuando se le pide a NSLog que imprima una cadena vacía, omite el registro por completo.

NSString *name = @"";
NSLog(@"%@", name);  // Resolves to @""

El código anterior imprimirá nada.

Es una buena práctica prefijar los registros con etiquetas:

NSString *name = @"";
NSLog(@"Name: %@", name);  // Resolves to @"Name: "

El código anterior imprimirá:

2016-07-21 14:20:28.623 App[87711:6153103] Name:

Usando FUNCIÓN #

NSLog(@"%s %@",__FUNCTION__, @"etc etc");

Inserta el nombre de la clase y el método en la salida:

2016-07-22 12:51:30.099 loggingExample[18132:2971471] -[ViewController viewDidLoad] etc etc

##NSLog frente a printf NSLog(@“NSLog message”); printf(“printf message\n”);

Producción:

2016-07-16 08:58:04.681 test[46259:1244773] NSLog message
printf message

NSLog genera la fecha, la hora, el nombre del proceso, el ID del proceso y el ID del subproceso, además del mensaje de registro. printf solo muestra el mensaje.

NSLog requiere un NSString y automáticamente agrega una nueva línea al final. printf requiere una cadena C y no agrega automáticamente una nueva línea.

NSLog envía la salida a stderr, printf envía la salida a stdout.

Algunos especificadores de formato en printf y NSLog son diferentes. Por ejemplo, cuando se incluye una cadena anidada, se producen las siguientes diferencias:

NSLog(@"My string: %@", (NSString *)myString);
printf("My string: %s", [(NSString *)myString UTF8String]);

Registro de metadatos de NSLog #

NSLog(@"%s %d %s, yourVariable: %@", __FILE__, __LINE__, __PRETTY_FUNCTION__, yourVariable);

Registrará el archivo, el número de línea y los datos de la función junto con cualquier variable que desee registrar. Esto puede hacer que las líneas de registro sean mucho más largas, especialmente con nombres detallados de archivos y métodos; sin embargo, puede ayudar a acelerar el diagnóstico de errores.

También puede envolver esto en una macro (almacenar esto en un Singleton o donde más lo necesite);

#define ALog(fmt, ...) NSLog((@"%s [Line %d] " fmt), __PRETTY_FUNCTION__, __LINE__, ##__VA_ARGS__);

Luego, cuando desee iniciar sesión, simplemente llame

ALog(@"name: %@", firstName);

Lo que te dará algo como;

 -[AppDelegate application:didFinishLaunchingWithOptions:] [Line 27] name: John

NSLog y tipo BOOL #

No hay un especificador de formato para imprimir tipo booleano usando NSLog. Una forma de imprimir un valor booleano es convertirlo en una cadena.

BOOL boolValue = YES;
NSLog(@"Bool value %@", boolValue ? @"YES" : @"NO");

Producción:

2016-07-30 22:53:18.269 Test[4445:64129] Bool value YES

Otra forma de imprimir un valor booleano es convertirlo en entero, logrando una salida binaria (1=sí, 0=no).

BOOL boolValue = YES;
NSLog(@"Bool value %i", boolValue);

Producción:

2016-07-30 22:53:18.269 Test[4445:64129] Bool value 1

Registro agregando a un archivo #

NSLog es bueno, pero también puede iniciar sesión agregándolo a un archivo, usando un código como:

NSFileHandle* fh = [NSFileHandle fileHandleForWritingAtPath:path];
if ( !fh ) {
    [[NSFileManager defaultManager] createFileAtPath:path contents:nil attributes:nil];
    fh = [NSFileHandle fileHandleForWritingAtPath:path];
}
if ( fh ) {
    @try {
        [fh seekToEndOfFile];
        [fh writeData:[self dataUsingEncoding:enc]];
    }
    @catch (...) {
    }
    [fh closeFile];
}
