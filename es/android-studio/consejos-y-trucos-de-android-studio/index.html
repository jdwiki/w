Emulador para pruebas #

Si no hay necesidades específicas, elija las imágenes del sistema x86_64 con las API de Google para crear el emulador.

Funciona mucho más rápido que armeabi-v7a en computadoras basadas en Intel x86 x64.

Hay algunas bibliotecas SDK compiladas y diseñadas con arquitectura ARM.

Si intenta instalarlos en emuladores basados ​​en Intel, recibirá un mensaje de error que dice

“INSTALL_FAILED_NO_MATCHING_ABIS”

Puede optar por un emulador de terceros como [Genymotion] 1 o [el emulador de Android independiente de Visual Studio] 2 en tales ocasiones.

Plantilla en vivo personalizada #

Ejemplo : #

Para lograr este propósito, debe agregar una plantilla en vivo personalizada: #

  1. Abra Configuración [Ctrl + Alt + S ]
  2. Escriba “Plantillas en vivo” en la barra de búsqueda superior izquierda.
  3. Haga clic en + para agregar un “Grupo de plantillas” y escriba un nombre de grupo (p. ej.: Mi Plantilla) para continuar

4. Elija su grupo de plantillas personalizadas (p. ej.: MyTemplate), luego haga clic en + y elija “Plantilla en vivo”. 5. Tomaré “decir” por ejemplo: >say -> startActivity(new Intent($from$.this,$to$.class)); 6. Escriba la clave que desea simplificar en “Abreviatura” (p. ej., decir) y escriba la declaración en el “Texto de la plantilla” (p. ej.: startActivity(new Intent($from$.this,$to$.class ));)

7. Haga clic en “definir” en la parte inferior izquierda y elija la situación (por ejemplo, Java) 8. Haga clic en “Editar variables”, defina la expresión. (ver detalle: [Diálogo Editar variables de plantilla] 4)

9. Haga clic en “Aceptar” y “Aplicar”. E intenta escribir “decir” en tu editor.

Use estilos de código personalizados, comparta con otros miembros del equipo y formatee automáticamente con acceso directo #

Es posible tener sus propios estilos de código personalizados, compartirlos con otros miembros del equipo y usar un atajo para formatear automáticamente el código en un archivo.

Para crear su propio estilo de código personalizado, vaya a: Preferencias -> Editor -> Estilo de código

Hay algunas configuraciones generales de estilo de código aquí. También puede seleccionar el idioma (Java para Android) y configurar los parámetros como mejor le parezca. Hay un * montón * de ajustes.

Una vez que haya hecho esto, guarde su estilo de código. Esto es solo para mantenerlo seguro.

Una vez que lo haya guardado, seleccione “Copiar al proyecto” en el cuadro de diálogo “Administrar”. Se le preguntará si desea cambiar a este esquema creado. Responde que sí.

Una vez hecho esto, cierre el cuadro de diálogo de preferencias Estilo de código.

Ahora, verifique que su configuración de estilo de código se haya guardado en: .idea/codeStyleSettings.xml

Asegúrese de que este archivo no se ignore en su sistema de control de versiones para que pueda compartirlo con sus compañeros de equipo.

Una vez que sus compañeros de equipo tengan este archivo, también deberían tener la misma configuración.

Ahora, al editar un archivo, puede formatear el código seleccionando:

Código -> Reformatear código

Accesos directos para el código de reformateo (tomado de [esta] 5 respuesta; consulte la respuesta para obtener detalles sobre cómo resolver problemas con el acceso directo en Ubuntu):

Victoria

Ctrl + Alt + L

Linux:

Ctrl + Mayús + Alt + L

Mac:

Opción + Comando + L

Cuando realiza un reformateo de código, debería aparecer un pequeño cuadro de diálogo informándole de los cambios realizados. Puede hacer clic en “Mostrar” en este cuadro de diálogo para que aparezca el “Diálogo de reformatear archivo”.

También puede abrir este cuadro de diálogo desde el menú Código y su acceso directo correspondiente.

Tenga en cuenta que “Solo el texto modificado de VCS” no siempre funciona según cómo se haya editado el código (puede ignorar una regla si parte del código no se ha editado).

También puede seleccionar texto y mostrar el cuadro de diálogo reformatear código para formatear solo el texto seleccionado.
