Instalación o Configuración #

ventanas #

  1. Verifique que tiene el JDK correcto. Puede verificarlo abriendo el símbolo del sistema (presione tecla de Windows y escriba cmd). En el símbolo del sistema, escriba javac -version, esto mostrará la versión actual de JDK instalada en su coincidencia o un error * si falta Java. Si el JDK no está disponible o la versión es anterior a la 1.8, descargue el Java SE Development Kit 8.

  2. Descargue la última versión de Android Studio.

  3. Inicie el archivo .exe descargado.

  4. Siga el asistente para instalar Android Studio

  5. Una vez completada la instalación, abra Android Studio desde el acceso directo que se creó en el escritorio durante el proceso de instalación.

  6. Cuando abra Android Studio por primera vez, es posible que le pida que copie su configuración anterior, pero como es la primera vez, simplemente puede optar por no copiar nada.

  7. Luego, Android Studio solicita descargar las API requeridas para desarrollar aplicaciones de Android. Descárguelos, después de descargar el IDE se abrirá y podrá escribir su primera aplicación.

Mac #

  1. Verifique que tiene el JDK correcto. Puede verificarlo abriendo la terminal (presione comando + espacio y escriba terminal). En la línea de comando, escriba javac -version, esto mostrará la versión actual de JDK instalada en su coincidencia o un error* si falta Java. Si el JDK no está disponible o la versión es inferior a la 1.8, descargue el Java SE Development Kit 8. **

  2. Descargue la última versión de Android Studio.

  3. Inicie el archivo .dmg descargado.

  4. Arrastre y suelte Android Studio en la carpeta Aplicaciones, luego inicie Android Studio.

  5. Abra Android Studio.

  6. Cuando abra Android Studio por primera vez, es posible que le pida que copie su configuración anterior, pero como es la primera vez, simplemente puede optar por no copiar nada.

  7. Luego, Android Studio solicita descargar las API requeridas para desarrollar aplicaciones de Android. Descárguelos, después de descargar el IDE se abrirá y podrá escribir su primera aplicación.

linux #

  1. Verifique que tiene el JDK correcto. Puede verificarlo abriendo la terminal (presione comando + espacio y escriba terminal). En la línea de comando, escriba javac -version, esto mostrará la versión actual de JDK instalada en su coincidencia o un error* si falta Java. Si el JDK no está disponible o la versión es anterior a la 1.8, descargue el Java SE Development Kit 8. **

  2. Descargue la última versión de Android Studio.

  3. Descomprima/extraiga Android Studio en una carpeta específica.

  4. Abra la terminal y vaya a la ruta donde extrajo Android Studio. (Luego, use el comando cd para ingresar a la carpeta de Android Studio). Después de ingresar, debemos ingresar a la carpeta bin, así que nuevamente, use el comando cd bin e ingrese.

  5. Ahora necesitamos cambiar el mod de nuestro archivo requerido, es decir, studio.sh para hacerlo, ingrese el comando sudo chmod 777 -R studio.sh, presione enter y escriba su contraseña (si corresponde) e ingrese. (También puede ver la lista de archivos presentes dentro del contenedor con el comando ls).

  6. Después de cambiar el mod, solo tenemos que ejecutar el archivo .studio.sh para hacerlo, ingrese el comando ./studio.sh

  7. Cuando ejecute el comando anterior, inicie Android Studio por primera vez, es posible que le pida que copie su configuración anterior, pero como es la primera vez, simplemente puede optar por no copiar nada.

  8. Luego, Android Studio solicita descargar las API requeridas para desarrollar aplicaciones de Android. Descárguelos, después de descargar el IDE se abrirá y podrá escribir su primera aplicación.

Es posible que encuentre “no se puede ejecutar la herramienta mksdcard SDK” cuando está instalando Android Studio en ubuntu os de 64 bits porque Studio requiere unos 32 binarios. Para solucionar este error, finalice y cierre Android Studio & vaya a la terminal y ejecute sudo apt-get install lib32z1 lib32ncurses5 lib32bz2-1.0 lib32stdc++6. Una vez completada la instalación de estos binarios, vuelva al paso 6 y reanude el proceso de instalación.

Notas #

  • Si ya ha instalado JDK y aún lo está obteniendo, asegúrese de haber configurado JAVA_HOME en sus Variables del sistema. Puede consultar esta respuesta sobre cómo configurar uno.

  • Sin embargo, existen problemas de estabilidad conocidos en Android Studio en Mac cuando se usa JDK 1.8. Hasta que se resuelvan estos problemas, puede mejorar la estabilidad degradando su JDK a una versión anterior (pero no inferior a JDK 1.6).

  • Mientras se completa la descarga de Android Studio, verifique qué versión de JDK tiene: abra una línea de comando y escriba javac -version. Si el JDK no está disponible o la versión es inferior a 1.8, descargue el [Java SE Development Kit 8](http://www.oracle.com/technetwork/java/javase/downloads/jdk8-downloads- 2133151.html).

  • Si está ejecutando una versión de 64 bits de Ubuntu, debe instalar algunas bibliotecas de 32 bits con los siguientes comandos:

    sudo dpkg --add-architecture i386
    sudo apt-get update
    sudo apt-get install lib32z1 lib32ncurses5 libbz2-1.0:i386 lib32stdc++6

  • Si está ejecutando Fedora de 64 bits, el comando es:

    sudo yum install zlib.i686 ncurses-libs.i686 bzip2-libs.i686

Problema con la descarga #

  • Si tiene algún problema, como un mensaje de que su sistema no es compatible con Android Studio (puede suceder cuando usa un navegador web diferente a Chrome), descargue Android Studio desde: http://tools.android.com/

¡Eso es todo!

Vista previa de diferentes tamaños de pantalla (dispositivos) y orientaciones #

1. Vista previa de diferentes dispositivos #

Hay un panel de vista previa a la derecha del estudio de Android. En este panel hay un botón con el nombre del dispositivo con el que está obteniendo una vista previa de la “IU” de su aplicación de esta manera.

Haga clic en el pequeño indicador desplegable de esto y aparecerá un panel flotante con todos los dispositivos predefinidos. Puede seleccionar cualquiera de ellos para obtener una vista previa de su aplicación UI con diferentes dispositivos mencionando sus tamaños de pantalla.
