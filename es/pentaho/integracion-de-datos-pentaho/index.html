Hello World en Pentaho Integración de datos #

Vamos a crear una transformación simple para convertir un CSV en un archivo XML.

Nuestra Transformación tiene que hacer lo siguiente:

  • Leer el archivo CSV
  • Construir el mensaje de saludo
  • Guardar los saludos en el archivo XML

Crear una Transformación: #

Así es como se inicia la Transformación:

  1. A la izquierda del espacio de trabajo se encuentra la Paleta de pasos. Seleccione la categoría Entrada.
  2. Arrastre el archivo CSV al espacio de trabajo de la derecha.
  3. Seleccione la categoría Scripting.
  4. Arrastre el icono Valor de JavaScript modificado al espacio de trabajo.
  5. Seleccione la categoría Salida.
  6. Arrastre el icono de Salida XML al espacio de trabajo.

Ahora vincule la entrada del archivo CSV con el valor de Java Script modificado creando un salto:

  1. Seleccione el primer paso.
  2. Mantenga pulsada la tecla Mayús y arrastre el icono al segundo paso.
  3. Vincule el valor de Java Script modificado con la salida XML a través de este mismo proceso.

Configuración de la entrada del archivo CSV Paso #

  1. Haga doble clic en el paso de entrada del archivo CSV. Aparecerá la ventana de configuración del paso. Aquí indicará la ubicación del archivo, el formato del archivo (p. ej., delimitadores, caracteres adjuntos, etc.) y los metadatos de la columna (p. ej., el nombre de la columna, el tipo de datos, etc.)
  2. Cambie el nombre del paso por uno que sea más representativo de la función de este paso. En este caso, escriba lista de nombres.
  3. Para el campo Nombre de archivo, haga clic en Examinar y seleccione el archivo de entrada.
  4. Haga clic en Obtener campos para agregar la lista de nombres de columna del archivo de entrada a la cuadrícula. De manera predeterminada, el Paso asume que el archivo tiene encabezados (la casilla de verificación Presente de la fila del encabezado está marcada).
  5. La cuadrícula ahora tiene los nombres de las columnas de su archivo. 6.Haga clic en Vista previa para asegurarse de que el archivo se lea como se esperaba. Aparecerá una ventana que muestra los datos del archivo. Haga clic en Aceptar para terminar de definir el Paso Entrada de archivo CSV.

Configuración del paso de valor de JavaScript modificado #

  1. Haga doble clic en el Paso de valor de JavaScript modificado.

  2. Aparecerá la ventana de configuración de pasos. Esto es diferente de la ventana de configuración de pasos anterior en que le permite escribir código JavaScript. Lo usará para construir el mensaje “Hola”, concatenado con cada uno de los nombres.

  3. Asigne a este paso el nombre Saludos.

  4. El área principal de la ventana de configuración es para codificar. A la izquierda, hay un árbol con un conjunto de funciones disponibles que puede usar en el código. En particular, las dos últimas ramas tienen los campos de entrada y salida, listos para usar en el código. En este ejemplo hay dos campos: apellido y nombre. Escribe el siguiente código:

    var msg = 'Hello, ' + name + "!";

  5. En la parte inferior puede escribir cualquier variable creada en el código. En este caso, ha creado una variable llamada msg. Como necesita enviar este mensaje al archivo de salida, debe escribir el nombre de la variable en la cuadrícula.

  6. Haga clic en Aceptar para finalizar la configuración del paso Valor modificado de Java Script.

  7. Seleccione el paso que acaba de configurar. Para verificar que el nuevo campo saldrá de este Paso, ahora verá los Campos de Entrada y Salida. Los campos de entrada son las columnas de datos que llegan a un paso. Los campos de salida son las columnas de datos que dejan un paso. Hay pasos que simplemente transforman los datos de entrada. En este caso, los campos de entrada y salida suelen ser los mismos. Sin embargo, hay pasos que agregan campos a la salida - Calculadora, por ejemplo. Hay otros Pasos que filtran o combinan datos haciendo que la Salida tenga menos campos que la Entrada - Agrupar por, por ejemplo.

  8. Haga clic con el botón derecho en el paso para que aparezca un menú contextual.

  9. Seleccione Mostrar campos de entrada. Verá que los campos de entrada son last_name y name, que provienen del paso de entrada del archivo CSV.

  10. Seleccione Mostrar campos de salida. Verá que no solo tiene los campos existentes, sino también el nuevo campo msg.

Configuración del paso de salida XML #

  1. Haga doble clic en el paso de salida XML. Aparecerá la ventana de configuración para este tipo de Paso. Aquí establecerá el nombre y la ubicación del archivo de salida y establecerá cuál de los campos desea incluir. Puede incluir todos o algunos de los campos que llegan al Paso.

  2. Asigne un nombre al archivo de pasos: Saludos.

  3. En el cuadro Archivo escriba:

    ${Internal.Transformation.Filename.Directory}/Hello.xml

  4. Haga clic en Obtener campos para llenar la cuadrícula con los tres campos de entrada.

  5. Vuelva a guardar la transformación.

CORRER #

Haga clic en el botón EJECUTAR en la barra de menú e inicie la transformación.

También puede crear un Trabajo que puede usarse para programar múltiples transformaciones y luego ejecutarlo.

Alternativamente, Pan le permite ejecutar Transformaciones desde una ventana de terminal. El script es Pan.bat en Windows o pan.sh en otras plataformas y se encuentra en la carpeta de instalación.

Pan /file <Tutorial_folder_path>/Hello.ktr /norep

Instalación de Pentaho DI #

Pentaho Data Integration viene en dos variedades:

  • Community Edition (CE) - Versión gratuita para desarrolladores
  • Enterprise Edition (EE) - Versión de pago para uso empresarial

Pasos de instalación: #

  1. Puede descargar Pentaho Data Integration Community Edition desde Sourceforge.net.

    For the Enterprise Edition : Download a 30-Day Trial

  2. Requisitos previos - PDI V7 requiere Oracle Java Runtime Environment (JRE) versión 8.

  3. PDI no requiere instalación. Simplemente descomprima el archivo zip en una carpeta de su elección. En los sistemas operativos similares a Unix, es posible que deba hacer que los scripts de shell sean ejecutables mediante el comando chmod:

    cd data-integration
chmod +x *.sh

  4. Correr: PDI comes with a graphical user interface called Spoon_,_ command-line scripts (Kitchen, Pan) to execute transformations and jobs, and other utilities.
