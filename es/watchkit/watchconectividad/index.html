Conectar su aplicación WatchOS a su aplicación iOS puede ser una tarea para completar cuando nunca lo ha hecho antes. Este tutorial le mostrará los fundamentos básicos para realizar esta tarea tan importante.

Configuración de iOS #

Aplicación para iPhone

  1. Importe WatchConnectivity y cumpla con WCSessionDelegate.
  2. usa el delegado de sesión estática a través de WCSession.default().
  3. Envíe datos a la aplicación Watch usando:
WCSession.default().sendMessage(mensaje, controlador de respuesta:_ controlador de error:_)
  1. El objeto del mensaje debe ser un diccionario de tipo [String:Any]
  2. Si está buscando que se devuelvan datos desde la aplicación Watch, proporcione la lógica en un cierre definido en el controlador de respuesta; de lo contrario, pase en cero.
  3. Para responder a los mensajes enviados desde la aplicación Watch, utilizará el método de devolución de llamada WCSessionDelegate
func session(_ session: WCSession, didReceiveMessage mensaje: [String : Any], answerHandler: @escaping ([String : Any]) -> Swift.Void){...}

Debe importar el marco WatchConnectivity a su archivo controlador de vista en la aplicación iOS. Esto le da acceso a los objetos de clase que están diseñados para comunicarse con la aplicación complementaria Watch. También deberá importar esto en el lado de la aplicación Watch. La única diferencia real con la aplicación Watch es que no tendrá un archivo de controlador de vista, sino un archivo de controlador de interfaz.

import WatchConnectivity

A continuación, deberá asegurarse de que su aplicación pueda admitir una sesión para WatchConnectivity. Si es posible, debe configurar el controlador de vista como su delegado y activar la sesión predeterminada. Obtendrá un error aquí. Debe cumplir con WCSessionDelegate e implementar algunos métodos antes de que el IDE comience a calmarse.

// MARK: - View Life Cycle Callbacks
override func viewDidLoad() {
    super.viewDidLoad()

    automaticallyAdjustsScrollViewInsets = false
    if WCSession.isSupported() {
        WCSession.default().delegate = self
        WCSession.default().activate()
    }
    else {
        print("\nViewController: connectionManager is nil\n")
    }
}

Para cumplir con WCSessionDelegate, agreguemos una extensión en la parte inferior del controlador de vista. Algunas personas odian este enfoque. Me salgo con la mía, pero para propósitos de tutorial, seguiré el enfoque de RayWenderlich. Soy un gran admirador de simplemente obtener el código directamente de COMANDO + HACER CLIC en los delegados y obtener TODOS LOS MÉTODOS de la especificación y comenzar a manipularlo y comprender cómo funcionan las cosas. En esta extensión, le proporcionaré todos los métodos. Ya están marcados para que veas cuando cada uno de ellos dispara en la consola. Si se siente juguetón, elimine algunos de los que están marcados como OPCIONALES para que su archivo de código se vea elegante.

Para enviar datos a su Watch App, una vez que la Watch App lo haya solicitado, manejará esto en el método…

func session(_ session: WCSession, didReceiveMessage message: [String : Any], replyHandler: @escaping ([String : Any]) -> Swift.Void) {
    print("6. ViewController: ", "didReceiveMessage")
    // build out your response message using a Dictionary
    let returnMessage: [String : Any] = [
       "key1" : value1,
       "key2" : value2,
       "key3" : value3
    ]
    // return your data in this manner
    replyHandler(returnMessage)
    // WARNING
    // You must call the replyHandler before the method ends, otherwise, your app will crash.
}

¡Esto es solo la mitad de la transacción! ¡Debe configurar su aplicación Watch para conectarse a la aplicación iPhone y manejar cualquier mensaje devuelto!

import WatchConnectivity

A continuación, debe verificar que la sesión sea incluso posible.

// MARK: - View Life Cycle Callbacks
override func awake(withContext context: Any?) {
    super.awake(withContext: context)

    // Configure interface objects here.
    if WCSession.isSupported() {
        WCSession.default().delegate = self
        WCSession.default().activate()
        print("InterfaceController: Session Activated")

        // Request Data from iPhone App
        let requestMessage = ["message":"get-data"]

        WCSession.default().sendMessage(requestMessage, replyHandler: { (replyMessage) in
            print("Got a reply from the phone: \(replyMessage)")

            // handle reply message here

        }, errorHandler: { (error) in
            print("Got an error sending to the phone: \(error)")
        })
    }
    else {
        print("\nViewController: connectionManager is nil\n")
    }
}

Enviando datos #

Enviar datos a su aplicación Watch desde su iPhone o desde su iPhone a su aplicación Watch después de activar algún evento es muy simple, aunque el código puede parecer un poco complejo al principio.

let message = ["key":"value-to-send"]

WCSession.default().sendMessage(message, replyHandler: { (replyMessage) in
    print("Got a reply from the phone: \(replyMessage)")

    if let returnedValues = replyMessage["returned-value"] as? NSArray {
        for val in returnedValues {
            // do something here with the data
            // Dispatch to Main Thread if affecting UI
        }
    }
}, errorHandler: { (error) in
    print("Got an error sending to the phone: \(error)")
})
