Uso de la extensión de inicio de sesión de matraz #

Una de las formas más sencillas de implementar un sistema de autorización es usar la extensión flask-login. El sitio web del proyecto contiene un inicio rápido detallado y bien escrito, cuya versión más corta está disponible en este ejemplo.

Idea general #

La extensión expone un conjunto de funciones utilizadas para:

  • iniciar sesión de usuarios
  • cerrar la sesión de los usuarios
  • comprobar si un usuario ha iniciado sesión o no y averiguar qué usuario es ese

Lo que no hace y lo que tienes que hacer por tu cuenta:

  • no proporciona una forma de almacenar los usuarios, por ejemplo, en la base de datos
  • no proporciona una forma de verificar las credenciales del usuario, por ejemplo, nombre de usuario y contraseña

A continuación hay un conjunto mínimo de pasos necesarios para que todo funcione.

Recomendaría colocar todo el código relacionado con la autenticación en un módulo o paquete separado, por ejemplo, auth.py. De esa forma, puede crear las clases, los objetos o las funciones personalizadas necesarios por separado.

Crear un LoginManager #

La extensión usa una clase LoginManager que debe registrarse en su objeto de aplicación Flask.

from flask_login import LoginManager
login_manager = LoginManager()
login_manager.init_app(app) # app is a Flask object

Como se mencionó anteriormente, LoginManager puede ser, por ejemplo, una variable global en un archivo o paquete separado. Luego se puede importar en el archivo en el que se crea el objeto Flask ​​o en la función de fábrica de su aplicación e inicializar.

Especifique una devolución de llamada utilizada para cargar usuarios #

Normalmente, los usuarios se cargarán desde una base de datos. La devolución de llamada debe devolver un objeto que represente a un usuario correspondiente a la ID proporcionada. Debería devolver Ninguno si la identificación no es válida.

@login_manager.user_loader
def load_user(user_id):
    return User.get(user_id) # Fetch the user from the database

Esto se puede hacer directamente debajo de la creación de su LoginManager.

Una clase que representa a tu usuario #

Como se mencionó, la devolución de llamada user_loader tiene que devolver un objeto que represente a un usuario. ¿Qué significa eso exactamente? Ese objeto puede, por ejemplo, ser un envoltorio alrededor de los objetos de usuario almacenados en su base de datos o simplemente directamente un modelo de su base de datos. Ese objeto tiene que implementar los siguientes métodos y propiedades. Eso significa que si la devolución de llamada devuelve su modelo de base de datos, debe asegurarse de que las propiedades y los métodos mencionados se agreguen a su modelo.

  • está_autenticado

    This property should return True if the user is authenticated, i.e. they have provided valid credentials. You will want to ensure that the objects which represent your users returned by the user_loader callback return True for that method.

  • es_activo

    This property should return True if this is an active user - in addition to being authenticated, they also have activated their account, not been suspended, or any condition your application has for rejecting an account. Inactive accounts may not log in. If you don’t have such a mechanism present return True from this method.

  • es_anonimo

    This property should return True if this is an anonymous user. That means that your user object returned by the user_loader callback should return True.

  • get_id()

    This method must return a unicode that uniquely identifies this user, and can be used to load the user from the user_loader callback. Note that this must be a unicode - if the ID is natively an int or some other type, you will need to convert it to unicode. If the user_loader callback returns objects from the database this method will most likely return the database ID of this particular user. The same ID should of course cause the user_loader callback to return the same user later on.

Si quieres facilitarte las cosas (**de hecho, se recomienda) puedes heredar de UserMixin en el objeto devuelto por la devolución de llamada user_loader (presumiblemente un modelo de base de datos). Puede ver cómo esos métodos y propiedades se implementan de forma predeterminada en este mixin [aquí] 5.

Iniciar sesión de los usuarios #

La extensión te deja a ti la validación del nombre de usuario y la contraseña ingresados ​​por el usuario. De hecho, a la extensión no le importa si usa una combinación de nombre de usuario y contraseña u otro mecanismo. Este es un ejemplo de inicio de sesión de usuarios con nombre de usuario y contraseña.

@app.route('/login', methods=['GET', 'POST'])
def login():
    # Here we use a class of some kind to represent and validate our
    # client-side form data. For example, WTForms is a library that will
    # handle this for us, and we use a custom LoginForm to validate.
    form = LoginForm()
    if form.validate_on_submit():
        # Login and validate the user.
        # user should be an instance of your `User` class
        login_user(user)

        flask.flash('Logged in successfully.')

        next = flask.request.args.get('next')
        # is_safe_url should check if the url is safe for redirects.
        # See http://flask.pocoo.org/snippets/62/ for an example.
        if not is_safe_url(next):
            return flask.abort(400)

        return flask.redirect(next or flask.url_for('index'))
    return flask.render_template('login.html', form=form)

En general, el inicio de sesión de los usuarios se logra llamando a login_user y pasando una instancia de un objeto que representa a su usuario mencionado anteriormente. Como se muestra, esto generalmente sucederá después de recuperar al usuario de la base de datos y validar sus credenciales; sin embargo, el objeto de usuario aparece mágicamente en este ejemplo.

He iniciado sesión como usuario, ¿ahora qué? #

Se puede acceder al objeto devuelto por la devolución de llamada user_loader de varias maneras.

  • En plantillas:

    The extension automatically injects it under the name current_user using a template context processor. To disable that behaviour and use your custom processor set add_context_processor=False in your LoginManager constructor.

      {% if current_user.is_authenticated %}
    Hi {{ current_user.name }}!
  {% endif %}

  • En código Python:

    The extension provides a request-bound object called current_user.

      from flask_login import current_user    

  @app.route("/hello")
  def hello():
      # Assuming that there is a name property on your user object
      # returned by the callback
      if current_user.is_authenticated:
          return 'Hello %s!' % current_user.name 
      else:
          return 'You are not logged in!'

  • Limitar el acceso rápidamente usando un decorador Se puede usar un decorador login_required para limitar el acceso rápidamente.

      from flask_login import login_required

  @app.route("/settings")
  @login_required
  def settings():
      pass

Cerrar sesión de usuarios #

Los usuarios pueden cerrar sesión llamando a logout_user(). Parece que es seguro hacerlo incluso si el usuario no ha iniciado sesión, por lo que lo más probable es que se pueda omitir el decorador @login_required.

@app.route("/logout")
@login_required
def logout():
    logout_user()
    return redirect(somewhere)

¿Qué sucede si un usuario no ha iniciado sesión y accedo al objeto current_user? #

Por defecto se devuelve un AnonymousUserMixin:

  • is_active y is_authenticated son False
  • is_anónimo es Verdadero
  • get_id() devuelve Ninguno

Para usar un objeto diferente para los usuarios anónimos, proporcione una llamada (ya sea una clase o una función de fábrica) que cree usuarios anónimos en su LoginManager con:

login_manager.anonymous_user = MyAnonymousUser

¿Qué sigue? #

Esto concluye la introducción básica a la extensión. Para obtener más información sobre la configuración y las opciones adicionales, se recomienda leer la guía oficial.

Tiempo de espera de la sesión de inicio de sesión #

Es una buena práctica desconectar la sesión iniciada después de un tiempo específico, puede lograrlo con Flask-Login.

from flask import Flask, session
from datetime import timedelta
from flask_login import LoginManager, login_require, login_user, logout_user

# Create Flask application

app = Flask(__name__) 

# Define Flask-login configuration 

login_mgr = LoginManager(app)
login_mgr.login_view = 'login'
login_mgr.refresh_view = 'relogin'
login_mgr.needs_refresh_message = (u"Session timedout, please re-login")
login_mgr.needs_refresh_message_category = "info"


@app.before_request
def before_request():
    session.permanent = True
    app.permanent_session_lifetime = timedelta(minutes=5)

La duración predeterminada de la sesión es de 31 días, el usuario debe especificar la vista de actualización de inicio de sesión en caso de tiempo de espera.

app.permanent_session_lifetime = timedelta(minutes=5)

La línea anterior obligará al usuario a volver a iniciar sesión cada 5 minutos.
