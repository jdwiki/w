Cargando todos los complementos de Package.JSON #

Suponiendo que sabe cómo instalar gulp, profundicemos en la solicitud de todas las dependencias de gulp- de package.json en la carpeta raíz de sus proyectos.

En caso de que aún no tenga un archivo gulp, cree un archivo vacío con el nombre

gulpfile.js

Primero, requerimos trago. al igual que:

var $ = require('gulp');

A continuación, carguemos todos nuestros complementos, en nuestro archivo gulp, con el siguiente fragmento

Nota #

Lea los comentarios en todos los fragmentos que incluiremos en esta lectura, brindan más información sobre cada funcionalidad

/*
require gulp-load-plugins, and call it
gulp-load-plugins will require individually,
all the plugins installed in package.json at the root directory
these required plugins, are lazy loaded
that is, gulp-load-plugins will only require them when they are referenced in this file
plugins are available for perusal, via camelcased names, minus gulp
eg: gulp-clean-css is accessed by $$.cleanCss
*/
var $$ = require('gulp-load-plugins')();

NOTA #

A lo largo de los muchos ejemplos que tendremos en el artículo, alias

  1. tragar como $ y
  2. gulp-load-plugins como $$

puramente para facilitar la escritura.

Instalación de complementos para imágenes receptivas | Minificación de Css | Minificación de Js #

Nuestro objetivo, es

  1. Haga que sus imágenes se ajusten a los anchos y la escala de manera adecuada, generando una batería de imágenes de anchos variados y minimicándolas
  2. Limpia tu Javascript
  3. Minimice sus activos: JS/CSS/HTML, lo que le permite alojar un código más ligero que el más ligero.
  4. Mire los archivos css/js/image para cambios y reconstruya optimizaciones
  5. Sincronizar sus cambios durante el desarrollo, en un navegador que sirve su sitio.

Necesitamos varios complementos, así que vamos a instalarlos todos. Ejecute todos estos en la carpeta raíz del proyecto.

Nota #

pipe es un método que transmite todos los archivos que coinciden con la entrada global a nuestros complementos (minifyhtml en este caso).

Es simple imaginarlo así:

$.src es lo que construye la corriente y canaliza a cada individuo archivo que coincide con el glob hacia abajo para cada complemento en la canalización. Optimización y Minificación de Activos #

Entonces, antes de escribir las funciones del optimizador, necesitamos instalar un par de complementos de almacenamiento en caché.

bash $ npm install --save-dev gulp-cached
bash $ npm install --save-dev gulp-remember

Te preguntarás por qué dos cachés ¡eh!. gulp-cached , pasa solo el contenido nuevo o modificado por la canalización a otros complementos. Entonces, dado que queremos que los archivos sin cambios se usen para concatenar en un solo archivo por activo (css | js), necesitamos gulp-remember además de gulp-cached

Primero usamos gulp-cached para construir una lista de archivos que han cambiado

En segundo lugar, necesitamos gulp-remember para realizar un seguimiento de todos los archivos que pasan por esa lista en la memoria.

Primera ejecución: no se almacenan archivos en caché, por lo que gulp-cached los pasará todos a gulp-remember

Ejecuciones posteriores: solo los archivos modificados o nuevos se canalizan mediante gulp-cached. Dado que el contenido del archivo actual ha cambiado, gulp-remember actualiza su caché.

Genial, escribamos nuestro primer optimizador

Optimizador de estilo #


// Goal

/*

1. cache existing files on the first run
2. each file , 
       a. Is autoprefixed with cross browser compatible prefixes for any css property ( justify-content for e.g)
       b. Is concatenated into app.css
3. app.css is minified
4. On subsequent runs , the above process is implemented on file modifications/additions only

*/

/*
*@src - input a glob pattern - a string eg 'images/**/*' or '**/*.css' or, an array eg ['glob1','glob2']
*/
var optimizeStyles = function(src) {

return $.src(src).
pipe($$.cached('styles')).
pipe($$.autoprefixer({
browsers: ['last 2 versions']
})).
pipe($$.remember('auto-prefixed-stylesheets')).
pipe($$.concat('app.css')).
pipe($.dest('build/css')).
pipe($$.cleanCss()).
pipe($$.rename({
suffix: '.min'
})).
pipe($.dest('build/css'))
}

Nota #

$ - > trago

$$ - > tragar-cargar-complementos

$.src -> construye flujos de archivos que coinciden con el glob pasado como src

$.dest -> guarda el archivo manipulado en la ruta especificada

// Goal

/*

1. cache existing files on the first run
2. each file , 
       a. Is linted with jshint 
       b. Is concatenated into app.js
3. app.js is minified
4. On subsequent runs , the above process is implemented on file modifications/additions only

*/

/*
*@src - input a glob pattern - a string eg 'js/**/*' or '**/*.js' or, an array eg ['glob1','glob2']
*/

var optimizeScripts = function(src) {

    return $.src(src).
    pipe($$.cached('scripts')).
    pipe($$.jshint()).
    pipe($$.jshint.reporter('default')).
    pipe($$.jshint.reporter('fail')).
    pipe($$.remember('linted-scripts')).
    pipe($$.concat('app.js')).
    pipe($.dest('build/js')).
    pipe($$.uglify()).
    pipe($$.rename({
        suffix: '.min'
    })).
    pipe($.dest('build/js'))


}

Generar Imágenes Responsivas #

Pasemos ahora al procesamiento de imágenes. Por lo tanto, el objetivo es tener una variedad de tamaños para cada imagen que va a publicar.

¿Por qué?

Bueno , para entender por qué necesitamos una batería de imágenes con un rango de anchos , necesitamos reflexionar sobre el hecho de que probablemente haya millones de dispositivos con resoluciones variadas. Necesitamos una imagen a escala sin mucha pixelación. Al mismo tiempo, necesitamos mejorar los tiempos de carga de la página, descargando solo el una imagen, que se ajusta al ancho que la contiene, y también está con la dimensión más pequeña posible, para hacerlo. Hay blogs académicos. como el que escribió Eric Portis , que destaca la ineficacia de solo consultas de medios y sirve como una guía completa para comprender conceptos como srcsets y tamaños.

Puede consultar el escrito épico de Eric Portis aquí

Ahora, nuestra función, necesita tomar un glob y un ancho como entradas, y hacer su magia y empujar el archivo que genera cada ejecución, a un destino y minimizar la imagen respondida.

Hemos instalado dos complementos de compresión de imágenes [en nuestro primer ejemplo] 2

Dado que estos complementos NO comienzan con un prefijo “gulp-”, debemos cargarlos manualmente en nuestro archivo gulp.

Entonces, solicitémoslos manualmente, después de la declaración de gulp-load-plugins en la parte superior de nuestro gulpfile.

al igual que:

var compressJpg = require('imagemin-jpeg-recompress');
var pngquant = require('imagemin-pngquant');

Vale la pena notar que gulp-responsive viene con el agudo procesador de imagen, que es mejor que imagemagick POR FAAAAR!. Agudo es lo que usa gulp-responsive , para recortar tus imágenes a lo deseado anchos.

puede buscar en gulp-responsive-configuration-options, para un lista completa de parámetros de configuración. solo he usado

  • ancho - para recortar nuestras imágenes a un ancho w, pasado como parámetro
  • renombrar - para agregar un sufijo al nombre de la imagen, para que siga siendo único

en mi función de configuración a continuación. por lo que nuestra función, recortará la imagen al ancho pasado como entrada, para todas las imágenes coincidentes descifradas a través de la entrada global. luego, cada imagen se comprime usando jpeg-recompress o pngquant y guardado dentro de build/images.

Con eso en mente, nuestra función sería así:

/*
@generateResponsiveImages
*@Description:takes in a src of globs, to stream matching image files , a width,
*to resize the matching image to, and a dest to write the resized and minified files to
*@src - input a glob pattern - a string eg 'images/** /*' or 'images/*' or, an array
eg ['glob1','glob2']
*@return returns a stream
*/
var generateResponsiveImages = function(src, width, dest) {

    //declare a default destination
    if (!dest)
        dest = 'build/images';
    //case 1: src glob -  images/**/*
    // the base is the directory immediately preceeding the glob - images/ in this case
    //case 2: images/fixed/flourish.png : the base here is images/fixed/ - we are overriding
    // that by setting base to images.This is done so that, the path beginning after images/
    // - is the path under our destination - without us setting the base, dest would be,
    //build/images/flourish.png.But with us setting the base, the destination would be
    // build/images/fixed/flourish.png
    return $.src(src, {
        base: 'images'
    })

    //generate resized images according to the width passed
    .pipe($$.responsive({
            //match all pngs within the src stream
            '**/*.png': [{
                width: width,
                rename: {
                    suffix: '-' + width
                },
                withoutEnlargement: false,
            }],
            //match all jpgs within the src stream
            '**/*.jpg': [{
                width: width,
                rename: {
                    suffix: '-' + width
                },
                progressive: true,
                withoutEnlargement: false,
            }]
        }, {

            errorOnEnlargement: false,
            errorOnUnusedConfig: false,
            errorOnUnusedImage: false

        }))
        //once the file is resized to width, minify it using the plugins available per format
        .pipe($$.if('*.jpg', compressJpg({
            min: 30,
            max: 90,
            target: 0.5
        })()))
        //use file based cache gulp-cache and it will minify only changed or new files
        //if it is not a new file and if the contents havent changed, the file is served from cache
        .pipe($$.cache($$.imagemin({
            verbose: true
        })))


    //write to destination - dest + path from base
    .pipe($.dest(dest));
}

$$ - > tragar-cargar-complementos

$.dest -> guarda el archivo manipulado en la ruta especificada

Minificador de HTML #

*
 *@minifyHtml
 *Description:takes in a glob src, and minifies all '.html' files matched by the glob
 *@src - input a glob pattern - a string eg '/**/*.html /*' or '*.html' or, an array eg ['glob1','glob2']
 *@dest=file.base means, the modified html file will be in the same directory as the src file being minified
 *often means, the process is just a modification on the existing src file
 *@return returns a stream
 */
var minifyHtml = function(src) {
    return $.src(src)
        .pipe($$.minifyHtml())
        .pipe($.dest(function(file) {
            //file is provided to a dest callback -
            // Refer here https://github.com/gulpjs/gulp/blob/master/docs/API.md#path
            return file.base;
        }));
}

Anatomía de una tarea de trago #

La anatomía de una definición de tarea es así:

$.task(<taskname> , [dependencies] , <body>);

dependencies , is an array of tasks that HAVE to finish before the current task you are defining , runs. More like forcing a synchronous execution instead of the default Asynchronous functionality.

Adding Gulp Tasks #

So, we now have

  • A function defined Above to optimise Styles
  • A function defined Above to optimise scripts
  • A function defined Above to optimise HTML
  • A function to generate multiple images per image Above

All we need to do now, is to invoke them when needed.

Let us write our tasks according to the syntax we defined earlier

/*
* $.tarea('nombre','matriz de dependencia',función)
da como resultado la construcción de un objeto de tarea como se muestra a continuación
tarea:{
'nombre nombre,
'dep':[matriz de dependencias],
'fn': función
}
*/


//*@return devuelve una secuencia para notificar sobre la finalización de la tarea
$.tarea('optimizarHtml', función() {
    var src = ['**/*.html', '!{node_modules}/**/*.html'];
    return minifyHtml(src);
});

//*@return devuelve una secuencia para notificar sobre la finalización de la tarea
$.tarea('optimizarScripts', function() {
    var src = ['js/**/*.js'];
    return optimizeScripts(src);
});

//*@return devuelve una secuencia para notificar sobre la finalización de la tarea
$.tarea('optimizarEstilos', función() {
    var src = ['css/**/*.css', 'fonts/google/**/*.css'];
    return optimizeStyles(src);
});

// Recibir una devolución de llamada para asegurarse de notificar al motor de trago, que la tarea se ha realizado
//requerido ya que no está devolviendo un flujo en esta tarea
$.tarea('generar ImágenesResponsivas', función(devolución de llamada) {
    var src = ['images/**/*.{jpg,png}'];
    for (var i = widths.length - 1; i >= 0; i--) {
        generateResponsiveImages(src, widths[i]);
    }
    callback();

});
