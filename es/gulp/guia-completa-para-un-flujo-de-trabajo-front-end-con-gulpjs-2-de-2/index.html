Enfriar. Así que hemos terminado con nuestra automatización del flujo de trabajo.

Ahora tenemos un archivo gulp, que

  • Responsabiliza y minimiza las imágenes.
  • limpia, autoprefija, concatena y minimiza Css
  • Concatena y minimiza JS
  • Observa los cambios en los activos, ya sea HTML | CSS | JS y desencadena tareas asociadas
  • Crea un directorio de compilación y almacena todo el código listo para la implementación procesado en su interior. Y todo eso, en segundo plano, mientras solo desarrollas tu aplicación.

Configuración de la sincronización del navegador y configuración de observadores para estilo y script #

NOTA #

Esta página ilustra el uso de complementos de gulp como browser-sync, gulp-watch and run-sequence, y continúa discutiendo gulp-workflow-automation desde donde lo dejamos en Gulpjs-workflow-automation-1 de 2. En caso de que llegues aquí, considera pasando por esa publicación primero.

  • Tarea predeterminada -Tarea de vigilancia: para crear continuamente sus activos listos para la implementación sobre la marcha, siempre que haya algo image | JS | css cambia en el curso del desarrollo.

Comencemos con la sincronización del navegador.

Tarea de vigilancia de Gulp #

Comencemos con la tarea de vigilancia.

El objetivo es observar los cambios que realiza durante el desarrollo. Cualquier cambio debería desencadenar la tarea Gulp correspondiente.

Además, necesitamos una funcionalidad que sincronice sus cambios en el navegador.

sincronización del navegador

Entonces, necesitamos instalar Browser Sync.

bash $ npm install browser-sync --save-dev

Con esa premisa, abramos nuestro gulpfile.js y agreguemos la función de vigilancia. Requerimos la sincronización del navegador y definamos algunas variables para usar su funcionalidad.

En la parte superior del archivo gulp, agregue el siguiente fragmento. Colóquelo justo debajo de las declaraciones de compresión de imágenes.

al igual que:

//Browser-sync

var sync = require('browser-sync').create();
var reload = sync.reload;

Hacer que la sincronización del navegador sincronice su desarrollo con el navegador es una configuración simple. Vamos a crear una tarea llamada perro guardián.

al igual que:

$.task('watchdog', function() {


})

Ahora, si navegamos a través de las opciones de sincronización del navegador [aquí] 1 y buscamos la configuración del servidor, podemos ver lo fácil que es.

Solo necesitamos colocar lo siguiente dentro de nuestra tarea de vigilancia

Fragmento - 1 - dentro de la plantilla de vigilancia #

/*
 Initiate Browser sync
 @documentation - https://www.browsersync.io/docs/options/
 */
 sync.init({
  server: {
    baseDir: "./"
 },
 port: 8000 //change it as required
 });

Inserte lo anterior dentro de su plantilla de vigilancia anterior.

El siguiente fragmento, es definir un observador de estilos, con el objetivo de reprocesar archivos css modificados o nuevos, y activar una recarga del navegador automáticamente.

fragmento - 2 - dentro de la plantilla de vigilancia #

$.watch(['css/**/*', 'fonts/google/**/*.css'], reload).on('change', function(event) {
console.log(event.type + ':' + event.path)
if (event.type === 'deleted') {
uncache('styles', event.path);
$$.remember.forget('auto-prefixed-stylesheets', event.path);
}
sequence('optimizeStyles')
});

Inserte lo anterior dentro de su plantilla de vigilancia anterior.

Así que estamos monitoreando " [fonts/google/**/*.css , /**/*.css ]" es decir,

todos los archivos css bajo css/ todos los archivos css en fonts/google/ Cuando algo cambia, o se agrega un nuevo archivo, activa el método de recarga, que se define en la parte superior de nuestro archivo gulp, en la declaración de sincronización del navegador.

Nota: puede notar que tenemos un controlador de eventos .on adjunto al observador.

$.watch(['css/**/*', 'fonts/google/**/*.css'], reload).on('change', function(event)

Básicamente, cualquier cosa CUD (Crear | Actualizar | Eliminar) activa la función de recarga y pasa un objeto de evento como parámetro al función de devolución de llamada.

La devolución de llamada es una función vital, donde podemos lograr operaciones como descaché en la eliminación de activos. Ahora el objeto de evento tiene parámetros como

  • sendero
  • tipo - Crear/Actualizar/Eliminar

Si se elimina un activo, debemos asegurarnos de que los cachés que construimos en nuestras funciones de minificación anteriores, a través de gulp-cached y gulp-remember, necesitan una actualización.

estamos manejando eso en el fragmento a continuación, que está dentro de la devolución de llamada al cambiar.

if (event.type === 'deleted') { 
uncache('styles', event.path);
$$.remember.forget('auto-prefixed-stylesheets', event.path);  
}

El método de secuencia, asegura, el método síncrono se ejecuta en un javascript asíncrono por defecto.

instalarlo es sencillo

HACER

bash $ npm install run-sequence

luego, declárelo en su archivo gulp justo debajo de la declaración de sincronización del navegador.

var sequence = require('run-sequence');

Entonces, con ese entendimiento, el observador de guiones es fácil de escribir. simplemente globos diferentes!

Por lo tanto, agregue este fragmento debajo del observador de estilo dentro del modelo de vigilancia.

Fragmento - 3 - dentro del texto estándar de la tarea Watchdog #

/*
on addition or change or deletion of a file in the watched directories
the change event is triggered. An event object with properties like
path,
event-type
is available for perusal passed to the callback

*/
$.watch('js/**/*', reload).on('change', function(event) {
console.log(event.type + ':' + event.path)
if (event.type === 'deleted') {
uncache('scripts', event.path);
$$.remember.forget('linted-scripts', event.path);
}
sequence('optimizeScripts');
});

Definición de una tarea predeterminada #

Así que ahora, terminemos el código de gulpfile, definiendo una tarea Predeterminada.

la tarea predeterminada es la que se ejecuta, cuando solo dices

gulp

en un símbolo del sistema en la raíz de su proyecto.

$.task('default', ['generateResponsiveImages'], function() {
 $.start('watchdog');
 console.log('Starting Incremental Build');
});
