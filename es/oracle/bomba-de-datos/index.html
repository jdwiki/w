Los siguientes son los pasos para crear una importación/exportación de bomba de datos:

Supervisar trabajos de bomba de datos #

Los trabajos de bomba de datos se pueden monitorear usando

1. vistas del diccionario de datos:

   select * from dba_datapump_jobs; 
   SELECT * FROM DBA_DATAPUMP_SESSIONS; 
   select username,opname,target_desc,sofar,totalwork,message from V$SESSION_LONGOPS where username = 'bkpadmin';

2. Estado de la bomba de datos:

  • Anote el nombre del trabajo de los registros de importación/exportación o el nombre del diccionario de datos y

  • Ejecute el comando adjuntar:

  • escriba el estado en el mensaje Importar/Exportar

    impdp / attach=<SYS_IMPORT_SCHEMA_01> Import> status Presiona CTRL+C para salir del indicador Importar/Exportar

Paso 3/6: Crear directorio #

create or replace directory DATAPUMP_REMOTE_DIR as '/oracle/scripts/expimp';

Paso 7: Comandos de exportación #

Comandos:

expdp <bkpadmin>/<bkp123>  parfile=<exp.par>

*Reemplace los datos en <> con valores apropiados según su entorno. Puede agregar/modificar parámetros según sus requisitos. En el ejemplo anterior, todos los parámetros restantes se agregan en archivos de parámetros como se indica a continuación: *

  • Tipo de exportación: Exportación de usuario

  • Exportar esquema completo

  • Detalles del archivo de parámetros [digamos exp.par]:

    schemas= directory= DATAPUMP_REMOTE_DIR dumpfile=.dmp logfile=exp_.log

  • Tipo de exportación: ** Exportación de usuario para esquema grande **

  • Exportar esquema completo para grandes conjuntos de datos: aquí los archivos de volcado de exportación se desglosarán y comprimirán. Aquí se usa el paralelismo (Nota: Agregar paralelismo aumentará la carga de la CPU en el servidor)

  • Detalles del archivo de parámetros [digamos exp.par]:

    schemas= directory= DATAPUMP_REMOTE_DIR dumpfile=%U.dmp logfile=exp__.log compression = all parallel=5

  • Tipo de exportación: Exportación de tabla [Exportar conjunto de tablas]

  • Detalles del archivo de parámetros [digamos exp.par]:

    tables= tname1, tname2, tname3 directory= DATAPUMP_REMOTE_DIR dumpfile=.dmp logfile=exp_.log

Paso 9: Comandos de importación #

Requisito previo:

  • Antes de la importación del usuario, es una buena práctica descartar el esquema o la tabla importados.

Comandos:

impdp <bkpadmin>/<bkp123>  parfile=<imp.par>

*Reemplace los datos en <> con valores apropiados según su entorno. Puede agregar/modificar parámetros según sus requisitos. En el ejemplo anterior, todos los parámetros restantes se agregan en archivos de parámetros como se indica a continuación: *

  • Tipo de importación: Importación de usuario

  • Importar esquema completo

  • Detalles del archivo de parámetros [decir imp.par]:

    schemas= directory= DATAPUMP_REMOTE_DIR dumpfile=.dmp logfile=imp_.log

  • Tipo de importación: Importación de usuario para esquema grande

  • Importe el esquema completo para grandes conjuntos de datos: aquí se usa el paralelismo * (Nota: agregar paralelismo aumentará la carga de la CPU en el servidor) *

  • Detalles del archivo de parámetros [decir imp.par]:

    schemas= directory= DATAPUMP_REMOTE_DIR dumpfile=%U.dmp logfile=imp__.log parallel=5

  • Tipo de importación: Importación de tablas [Importar conjunto de tablas]

  • Detalles del archivo de parámetros [decir imp.par]:

    tables= tname1, tname2, tname3 directory= DATAPUMP_REMOTE_DIR dumpfile=.dmp logfile=exp_.log TABLE_EXISTS_ACTION= <APPEND /SKIP /TRUNCATE /REPLACE>

1. Pasos de bombeo de datos #

Servidor de origen [Exportar datos] Servidor de destino [Importar datos]
1. Cree una carpeta de bomba de datos que contendrá los archivos de volcado de exportación 4. Cree una carpeta de bomba de datos que contendrá los archivos de volcado de importación
2. Inicie sesión en el esquema de la base de datos que realizará la exportación. 5. Inicie sesión en el esquema de la base de datos que realizará la importación.
3. Cree un directorio que apunte al paso 1. 6. Cree un directorio que apunte al paso 4.
7. Ejecute las declaraciones de exportación.
8. Copie/SCP los archivos de volcado en el servidor de destino.
9. Ejecutar declaraciones de importación
10. comprobar datos, compilar objetos no válidos y proporcionar subvenciones relacionadas

Copiar tablas entre diferentes esquemas y tablespaces #

 expdp <bkpadmin>/<bkp123> directory=DATAPUMP_REMOTE_DIR dumpfile=<customer.dmp>


 impdp <bkpadmin>/<bkp123> directory=DATAPUMP_REMOTE_DIR dumpfile=<customer.dmp> remap_schema=<source schema>:<target schema> remap_tablespace=<source tablespace>:<target tablespace>
