Manejo de valores NULL

  • Las operaciones que contienen NULL son NULL, excepto la concatenación
  • NVL2 para obtener un resultado diferente si un valor es nulo o no
  • COALESCE para devolver el primer valor no NULL
  • Las columnas de cualquier tipo de datos pueden contener valores NULL
  • Las cadenas vacías son NULL
  • NVL para reemplazar el valor nulo

Una columna es NULL cuando no tiene valor, independientemente del tipo de datos de esa columna.

Una columna nunca debe compararse con NULL usando esta sintaxis a = NULL ya que el resultado sería DESCONOCIDO. En su lugar, utilice las condiciones a IS NULL o a IS NOT NULL. NULL no es igual a NULL. Para comparar dos expresiones en las que puede ocurrir un valor nulo, utilice una de las funciones que se describen a continuación.

Todos los operadores excepto la concatenación devuelven NULL si uno de sus operandos es NULL. Por ejemplo, el resultado de 3 * NULL + 5 es nulo.

NULL no puede aparecer en columnas restringidas por una CLAVE PRINCIPAL o una restricción NOT NULL. (La excepción es una nueva restricción con la cláusula NOVALIDATE)

Las operaciones que contienen NULL son NULL, excepto la concatenación #

SELECT 3 * NULL + 5, 'Hello ' || NULL || 'world'   from DUAL;
3*NULO+5 ‘HOLA’||NULL||‘MUNDO’
(nulo) Hola mundo

NVL2 para obtener un resultado diferente si un valor es nulo o no #

Si el primer parámetro NO es NULL, NVL2 devolverá el segundo parámetro. De lo contrario devolverá el tercero.

SELECT NVL2(null, 'Foo', 'Bar'), NVL2(5, 'Foo', 'Bar') FROM DUAL;
NVL2(NULO,‘FOO’,‘BAR’) NVL2(5,‘FOO’,‘BAR’)
Barra Foo

COALESCE para devolver el primer valor no NULL #

SELECT COALESCE(a, b, c, d, 5) FROM 
    (SELECT NULL A, NULL b, NULL c, 4 d FROM DUAL);
COALESCE(A,B,C,D,5)
4

En algunos casos, usar COALESCE con dos parámetros puede ser más rápido que usar NVL cuando el segundo parámetro no es una constante. NVL siempre evaluará ambos parámetros. COALESCE se detendrá en el primer valor no NULL que encuentre. Significa que si el primer valor no es NULL, COALESCE será más rápido.

Las columnas de cualquier tipo de datos pueden contener valores NULL #

SELECT 1 NUM_COLUMN, 'foo' VARCHAR2_COLUMN from DUAL
UNION ALL
SELECT NULL, NULL from DUAL;
NUM_COLUMN VARCHAR2_COLUMN
1 foo
(nulo) (nulo)

Las cadenas vacías son NULL #

SELECT 1 a, '' b from DUAL;
un B
1 (nulo)

NVL para reemplazar el valor nulo #

SELECT a column_with_null, NVL(a, 'N/A') column_without_null FROM
  (SELECT NULL a FROM DUAL);
COLUMN_WITH_NULL COLUMN_WITHOUT_NULL
(nulo) N/D

NVL es útil para comparar dos valores que pueden contener NULL:

SELECT
    CASE WHEN a = b THEN 1 WHEN a <> b THEN 0 else -1 END comparison_without_nvl,
    CASE WHEN NVL(a, -1) = NVL(b, -1) THEN 1 WHEN NVL(a, -1) <> NVL(b, -1) THEN 0 else -1 END comparison_with_nvl
  FROM
    (select null a, 3 b FROM DUAL
     UNION ALL
     SELECT NULL, NULL FROM DUAL);
COMPARACIÓN_SIN_NVL COMPARACIÓN_CON_NVL
-1 0
-1 1
