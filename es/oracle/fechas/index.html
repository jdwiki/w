Aritmética de Fechas - Diferencia entre Fechas en Días, Horas, Minutos y/o Segundos #

En Oracle, la diferencia (en días y/o fracciones de los mismos) entre dos ‘FECHA’ se puede encontrar usando la resta:

SELECT DATE '2016-03-23' - DATE '2015-12-25' AS difference FROM DUAL;

Muestra el número de días entre las dos fechas:

DIFFERENCE
----------
        89

Y:

SELECT TO_DATE( '2016-01-02 01:01:12', 'YYYY-MM-DD HH24:MI:SS' )
         - TO_DATE( '2016-01-01 00:00:00', 'YYYY-MM-DD HH24:MI:SS' )
         AS difference
FROM   DUAL

Muestra la fracción de días entre dos fechas:

DIFFERENCE
----------
    1.0425

La diferencia en horas, minutos o segundos se puede encontrar multiplicando este número por 24, 24*60 o 24*60*60 respectivamente.

El ejemplo anterior se puede cambiar para obtener los días, horas, minutos y segundos entre dos fechas usando:

SELECT TRUNC( difference                       ) AS days,
       TRUNC( MOD( difference * 24,       24 ) ) AS hours,
       TRUNC( MOD( difference * 24*60,    60 ) ) AS minutes,
       TRUNC( MOD( difference * 24*60*60, 60 ) ) AS seconds
FROM   (
  SELECT TO_DATE( '2016-01-02 01:01:12', 'YYYY-MM-DD HH24:MI:SS' )
         - TO_DATE( '2016-01-01 00:00:00', 'YYYY-MM-DD HH24:MI:SS' )
         AS difference
  FROM   DUAL

);

(Nota: TRUNC() se usa en lugar de [FLOOR()](https: //docs.oracle.com/cd/B19306_01/server.102/b14200/functions058.htm) para manejar correctamente las diferencias negativas.)

Salidas:

DAYS HOURS MINUTES SECONDS
---- ----- ------- -------
   1     1       1      12

El ejemplo anterior también se puede resolver convirtiendo la diferencia numérica en un intervalo usando NUMTODSINTERVAL() :

SELECT EXTRACT( DAY    FROM difference ) AS days,
       EXTRACT( HOUR   FROM difference ) AS hours,
       EXTRACT( MINUTE FROM difference ) AS minutes,
       EXTRACT( SECOND FROM difference ) AS seconds
FROM   (
  SELECT NUMTODSINTERVAL(
           TO_DATE( '2016-01-02 01:01:12', 'YYYY-MM-DD HH24:MI:SS' )
             - TO_DATE( '2016-01-01 00:00:00', 'YYYY-MM-DD HH24:MI:SS' ),
           'DAY'
         ) AS difference
  FROM   DUAL
);

Configuración del modelo de formato de fecha predeterminado #

Cuando Oracle convierte implícitamente de DATE a una cadena o viceversa (o cuando TO_CHAR() o TO_DATE() se llaman explícitamente sin un modelo de formato), el parámetro de sesión NLS_DATE_FORMAT será utilizado como modelo de formato en la conversión. Si el literal no coincide con el modelo de formato, se generará una excepción.

Puede revisar este parámetro usando:

SELECT VALUE FROM NLS_SESSION_PARAMETERS WHERE PARAMETER = 'NLS_DATE_FORMAT';

Puede establecer este valor dentro de su sesión actual usando:

ALTER SESSION SET NLS_DATE_FORMAT = 'YYYY-MM-DD HH24:MI:SS';

(Nota: esto no cambia el valor para ningún otro usuario).

Si confía en NLS_DATE_FORMAT para proporcionar la máscara de formato en TO_DATE() o TO_CHAR(), entonces no debería sorprenderse cuando sus consultas se rompan si este valor alguna vez cambia.

Aritmética de fechas: diferencia entre fechas en meses o años #

La diferencia en meses entre dos fechas se puede encontrar mediante MONTHS_BETWEEN( date1, date2 ):

SELECT MONTHS_BETWEEN( DATE '2016-03-10', DATE '2015-03-10' ) AS difference FROM DUAL;

Salidas:

DIFFERENCE
----------
        12

Si la diferencia incluye meses parciales, devolverá la fracción del mes en función de que haya 31 días en cada mes:

SELECT MONTHS_BETWEEN( DATE '2015-02-15', DATE '2015-01-01' ) AS difference FROM DUAL;

Salidas:

DIFFERENCE
----------
 1.4516129

Debido a que MONTHS_BETWEEN asume 31 días por mes cuando puede haber menos días por mes, esto puede resultar en diferentes valores para las diferencias que abarcan los límites entre meses.

Ejemplo:

SELECT MONTHS_BETWEEN( DATE'2016-02-01', DATE'2016-02-01' - INTERVAL '1' DAY ) AS "JAN-FEB",
       MONTHS_BETWEEN( DATE'2016-03-01', DATE'2016-03-01' - INTERVAL '1' DAY ) AS "FEB-MAR",
       MONTHS_BETWEEN( DATE'2016-04-01', DATE'2016-04-01' - INTERVAL '1' DAY ) AS "MAR-APR",
       MONTHS_BETWEEN( DATE'2016-05-01', DATE'2016-05-01' - INTERVAL '1' DAY ) AS "APR-MAY"
FROM   DUAL;

Producción:

JAN-FEB FEB-MAR MAR-APR APR-MAY
------- ------- ------- -------
0.03226 0.09677 0.03226 0.06452

La diferencia de años se puede encontrar dividiendo la diferencia de meses entre 12.

Generación de fechas sin componente de tiempo #

Todos los DATE tienen un componente de tiempo; sin embargo, es habitual almacenar fechas que no necesitan incluir información de tiempo con las horas/minutos/segundos configurados en cero (es decir, medianoche).

Use un literal ANSI FECHA (usando [formato de fecha ISO 8601](https://en .wikipedia.org/wiki/ISO_8601#Calendar_dates)):

SELECT DATE '2000-01-01' FROM DUAL;

Conviértalo de un literal de cadena usando TO_DATE():

SELECT TO_DATE( '2001-01-01', 'YYYY-MM-DD' ) FROM DUAL;

(Se puede encontrar más información sobre los modelos de formato de fecha en la documentación de Oracle).

o:

SELECT TO_DATE(
         'January 1, 2000, 00:00 A.M.',
         'Month dd, YYYY, HH12:MI A.M.',
         'NLS_DATE_LANGUAGE = American'
       )
FROM   DUAL;

(Si está convirtiendo términos específicos del idioma, como los nombres de los meses, es una buena práctica incluir el tercer parámetro nlsparam en la función TO_DATE() y especificar el idioma esperado).

Generación de fechas con un componente de tiempo #

Conviértalo de un literal de cadena usando TO_DATE():

SELECT TO_DATE( '2000-01-01 12:00:00', 'YYYY-MM-DD HH24:MI:SS' ) FROM DUAL;

O use un TIMESTAMP literal:

CREATE TABLE date_table(
  date_value DATE
);

INSERT INTO date_table ( date_value ) VALUES ( TIMESTAMP '2000-01-01 12:00:00' );

Oracle implícitamente convertirá un TIMESTAMP en DATE cuando lo almacene en una columna DATE de una tabla; sin embargo, puede explícitamente CAST() el valor a DATE:

SELECT CAST( TIMESTAMP '2000-01-01 12:00:00' AS DATE ) FROM DUAL;

El formato de una fecha #

En Oracle, un tipo de datos FECHA no tiene formato; cuando Oracle envía una FECHA al programa cliente (SQL/Plus, SQL/Developer, Toad, Java, Python, etc.), enviará 7 u 8 bytes que representan la fecha.

Una FECHA que no está almacenada en una tabla (es decir, generada por SYSDATE y que tiene “tipo 13” cuando se usa el comando DUMP()) tiene 8 bytes y tiene la estructura (los números a la derecha son los representación interna de 2012-11-26 16:41:09):

BYTE VALUE                           EXAMPLE
---- ------------------------------- --------------------------------------
1    Year modulo 256                 220
2    Year multiples of 256           7   (7 * 256 + 220 = 2012)
3    Month                           11
4    Day                             26
5    Hours                           16
6    Minutes                         41
7    Seconds                         9
8    Unused                          0

Una FECHA que se almacena en una tabla (“tipo 12” cuando se usa el comando DUMP()) tiene 7 bytes y tiene la estructura (los números a la derecha son la representación interna de 2012-11-26 16:41:09):

BYTE VALUE                           EXAMPLE
---- ------------------------------- --------------------------------------
1    ( Year multiples of 100 ) + 100 120
2    ( Year modulo 100 ) + 100       112 ((120-100)*100 + (112-100) = 2012)
3    Month                           11
4    Day                             26
5    Hours + 1                       17
6    Minutes + 1                     42
7    Seconds + 1                     10

Si desea que la fecha tenga un formato específico, deberá convertirla a algo que tenga un formato (es decir, una cadena). El cliente SQL puede hacer esto implícitamente o puede explícitamente [convertir el valor en una cadena] (https://www.wikiod.com/es/oracle/fechas#Convertir fechas en una cadena) usando TO_CHAR( fecha, formato_modelo, nls_params ).

Convertir fechas en una cadena #

Utilice TO_CHAR( fecha [, formato_modelo [, nls_params]] ):

(Nota: si no se proporciona un modelo de formato, se utilizará el parámetro de sesión NLS_DATE_FORMAT como el [modelo de formato predeterminado] (https://www.wikiod.com/es/oracle/fechas#Configuración del modelo de formato de fecha predeterminado); esto puede ser diferente para cada sesión, por lo que no debe ser Es una buena práctica especificar siempre el modelo de formato.)

CREATE TABLE table_name (
  date_value DATE
);

INSERT INTO table_name ( date_value ) VALUES ( DATE '2000-01-01' );
INSERT INTO table_name ( date_value ) VALUES ( TIMESTAMP '2016-07-21 08:00:00' );
INSERT INTO table_name ( date_value ) VALUES ( SYSDATE );

Después:

SELECT TO_CHAR( date_value, 'YYYY-MM-DD' ) AS formatted_date FROM table_name;

Salidas:

FORMATTED_DATE
--------------
2000-01-01
2016-07-21
2016-07-21

Y:

SELECT TO_CHAR(
         date_value,
         'FMMonth d yyyy, hh12:mi:ss AM',
         'NLS_DATE_LANGUAGE = French'
       ) AS formatted_date
FROM   table_name;

Salidas:

FORMATTED_DATE
-----------------------------
Janvier   01 2000, 12:00:00 AM
Juillet   21 2016, 08:00:00 AM
Juillet   21 2016, 19:08:31 PM

Cambiar cómo SQL/Plus o SQL Developer muestran las fechas #

Cuando SQL/Plus o SQL Developer muestren fechas, realizarán una conversión implícita a una cadena utilizando el modelo de formato de fecha predeterminado (consulte [Configuración del modelo de formato de fecha predeterminado](https://www.wikiod.com/es/oracle/fechas fechas/6850/establecer-el-modelo-de-formato-de-fecha-predeterminado)).

Puede cambiar cómo se muestra una fecha cambiando el parámetro NLS_DATE_FORMAT.

Extraiga los componentes de año, mes, día, hora, minuto o segundo de una fecha #

Los componentes de año, mes o día de un tipo de datos DATE se pueden encontrar mediante [EXTRACT( [YEAR | MONTH | DAY] FROM datevalue)](https://docs.oracle.com/cd/B19306_01/ servidor.102/b14200/funciones050.htm)

SELECT EXTRACT (YEAR  FROM DATE '2016-07-25') AS YEAR,
       EXTRACT (MONTH FROM DATE '2016-07-25') AS MONTH,
       EXTRACT (DAY   FROM DATE '2016-07-25') AS DAY
FROM DUAL;

Salidas:

YEAR MONTH DAY
---- ----- ---
2016     7  25

Los componentes de tiempo (hora, minuto o segundo) se pueden encontrar mediante:

  • Usar CAST(datevalue AS TIMESTAMP ) para convertir DATE en TIMESTAMP y luego usando EXTRACT( [ HORA | MINUTO | SEGUNDO ] DE timestampvalue ); o
  • Usando TO_CHAR(datevalue, format_model ) para obtener el valor como una cadena.

Por ejemplo:

SELECT EXTRACT( HOUR   FROM CAST( datetime AS TIMESTAMP ) ) AS Hours,
       EXTRACT( MINUTE FROM CAST( datetime AS TIMESTAMP ) ) AS Minutes,
       EXTRACT( SECOND FROM CAST( datetime AS TIMESTAMP ) ) AS Seconds
FROM   (
  SELECT TO_DATE( '2016-01-01 09:42:01', 'YYYY-MM-DD HH24:MI:SS' ) AS datetime FROM DUAL
);

Salidas:

HOURS MINUTES SECONDS
----- ------- -------
    9      42       1

Zonas horarias y horario de verano #

El tipo de datos FECHA no maneja zonas horarias ni cambios en el horario de verano.

O:

  • utilice el tipo de datos TIMESTAMP WITH TIME ZONE; o
  • manejar los cambios en la lógica de su aplicación.

Una FECHA se puede almacenar como hora universal coordinada (UTC) y convertirse a la zona horaria de la sesión actual de esta manera:

SELECT FROM_TZ(
         CAST(
           TO_DATE( '2016-01-01 12:00:00', 'YYYY-MM-DD HH24:MI:SS' )
           AS TIMESTAMP
         ),
         'UTC'
       )
       AT LOCAL AS time
FROM   DUAL;

Si ejecuta ALTER SESSION SET TIME_ZONE = '+01:00'; entonces el resultado es:

TIME
------------------------------------
2016-01-01 13:00:00.000000000 +01:00

y ALTER SESSION SET TIME_ZONE = 'PST'; entonces la salida es:

TIME
------------------------------------
2016-01-01 04:00:00.000000000 PST

Segundos intercalares #

Oracle [no maneja los segundos intercalares] (http://stackoverflow.com/questions/31136211/how-to-handle-leap-seconds-in-oracle). Consulte la nota de My Oracle Support 2019397.2 y 730795.1 para obtener más detalles.

Obtener el día de la semana #

Puede usar TO_CHAR(date_value, 'D' ) para obtener el día de la semana.

Sin embargo, esto depende del parámetro de sesión NLS_TERRITORY:

ALTER SESSION SET NLS_TERRITORY = 'AMERICA';        -- First day of week is Sunday
SELECT TO_CHAR( DATE '1970-01-01', 'D' ) FROM DUAL;

Salidas 5

ALTER SESSION SET NLS_TERRITORY = 'UNITED KINGDOM'; -- First day of week is Monday
SELECT TO_CHAR( DATE '1970-01-01', 'D' ) FROM DUAL;

Salidas 4

Para hacer esto independientemente de la configuración NLS, puede truncar la fecha a la medianoche del día actual (para eliminar cualquier fracción de días) y restar la fecha truncada al inicio de la iso-semana actual (que siempre comienza el lunes). ):

SELECT TRUNC( date_value ) - TRUNC( date_value, 'IW' ) + 1 FROM DUAL
