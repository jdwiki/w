Configuración de la autenticación interna #

Cassandra no requerirá que los usuarios inicien sesión con la configuración predeterminada. En su lugar, se permiten inicios de sesión anónimos sin contraseña para cualquier persona que pueda conectarse a native_transport_port. Este comportamiento se puede cambiar editando la configuración cassandra.yaml para usar un autenticador diferente:

# Allow anonymous logins without authentication
# authenticator: AllowAllAuthenticator

# Use username/password based logins
authenticator: PasswordAuthenticator

Las credenciales de inicio de sesión validadas por PasswordAuthenticator se almacenarán en el espacio de claves interno system_auth. De forma predeterminada, el espacio de claves no se replicará en todos los nodos. Tendrá que cambiar la configuración de replicación para asegurarse de que Cassandra aún pueda leer las credenciales de usuario del almacenamiento local en caso de que no se pueda acceder a otros nodos en el clúster, ¡o de lo contrario es posible que no pueda iniciar sesión!

Para SimpleStrategy (donde N es el número de nodos en su clúster):

ALTER KEYSPACE system_auth WITH replication = {'class': 'SimpleStrategy', 'replication_factor': N};

Para NetworkTopologyStrategy (donde N es el número de nodos en el centro de datos correspondiente):

ALTER KEYSPACE system_auth WITH replication =  { 'class' : 'NetworkTopologyStrategy', 'datacenter1' : N };

Reinicie cada nodo después de los cambios descritos anteriormente. Ahora solo podrá iniciar sesión con el superusuario predeterminado:

cqlsh -u casandra -p casandra

(Opcional) Reemplace el superusuario predeterminado con un usuario personalizado #

Usar un superusuario predeterminado con una contraseña estándar no es mucho más seguro que no usar ningún usuario. Debe crear su propio usuario en lugar de usar una contraseña segura y única:

CREATE ROLE myadminuser WITH PASSWORD = 'admin123' AND LOGIN = true AND SUPERUSER = true;

Inicie sesión con su nuevo usuario: cqlsh -u myadminuser -p admin123

Ahora deshabilite el inicio de sesión para el usuario Cassandra estándar y elimine el estado de superusuario:

ALTER ROLE cassandra CON INICIO DE SESIÓN = falso Y SUPERUSUARIO = falso;

Configuración de la autorización interna #

De manera predeterminada, cada usuario podrá acceder a todos los datos en Cassandra. Deberá configurar un autorizador diferente en su cassandra.yaml para otorgar permisos de objetos individuales a sus usuarios.

# Grant all permissions to all users
# authorizer: AllowAllAuthorizer

# Use object permissions managed internally by Cassandra
authorizer: CassandraAuthorizer

Los permisos para usuarios individuales se almacenarán en el espacio de claves interno system_auth. Debe cambiar la configuración de replicación en caso de que aún no lo haya hecho al habilitar la autenticación basada en contraseña.

Para SimpleStrategy (donde N es el número de nodos en su clúster):

ALTER KEYSPACE system_auth WITH replication = {'class': 'SimpleStrategy', 'replication_factor': N};

Para NetworkTopologyStrategy (donde N es el número de nodos en el centro de datos correspondiente):

ALTER KEYSPACE system_auth WITH replication =  { 'class' : 'NetworkTopologyStrategy', 'datacenter1' : N };

Reinicie cada nodo después de los cambios descritos anteriormente. Ahora podrá establecer permisos usando, p. los siguientes comandos.

Otorga todos los permisos para el espacio de claves y el rol especificados:

CONCEDER TODO EN KEYSPACE keyspace_name TO role_name;

Otorgue permisos de lectura en todos los espacios de claves:

CONCEDER SELECCIÓN EN TODOS LOS ESPACIOS DE TECLAS A role_name;

Permitir la ejecución de instrucciones INSERT, UPDATE, DELETE y TRUNCATE en un determinado espacio de teclas:

GRANT MODIFY ON KEYSPACE keyspace_name TO role_name;

Permitir cambiar espacios de claves, tablas e índices para ciertos espacios de claves:

GRANT ALTER ON KEYSPACE keyspace_name TO role_name;

Tenga en cuenta que los permisos se almacenarán en caché para permissions_validity_in_ms (cassandra.yaml) y es posible que los cambios no surtan efecto al instante.
