  • Contar las ocurrencias de una subcadena o carácter en una cadena

Las cadenas (java.lang.String) son fragmentos de texto almacenados en su programa. Las cadenas no son un tipo de datos primitivo en Java, sin embargo, son muy comunes en los programas Java.

En Java, las cadenas son inmutables, lo que significa que no se pueden cambiar. (Haga clic aquí para obtener una explicación más completa de la inmutabilidad).

Dado que las cadenas de Java son [inmutables] (https://en.wikipedia.org/wiki/Immutable_object), todos los métodos que manipulan una String devolverán un nuevo objeto String. No cambian el String original. Esto incluye subcadenas y métodos de reemplazo que los programadores de C y C++ esperarían para mutar el objeto String de destino.

Utilice un StringBuilder en lugar de String si desea concatenar más de dos objetos String cuyos valores no se pueden determinar en tiempo de compilación. Esta técnica es más eficaz que crear nuevos objetos String y concatenarlos porque StringBuilder es mutable.

StringBuffer también se puede usar para concatenar objetos String. Sin embargo, esta clase tiene menos rendimiento porque está diseñada para ser segura para subprocesos y adquiere un mutex antes de cada operación. Dado que casi nunca necesita seguridad para subprocesos al concatenar cadenas, es mejor usar StringBuilder.

Si puede expresar una concatenación de cadenas como una sola expresión, entonces es mejor usar el operador +. El compilador de Java convertirá una expresión que contenga concatenaciones + en una secuencia eficiente de operaciones usando String.concat(...) o StringBuilder. El consejo de usar StringBuilder explícitamente solo se aplica cuando la concatenación involucra múltiples expresiones.

No almacene información confidencial en cadenas. Si alguien puede obtener un volcado de memoria de su aplicación en ejecución, podrá encontrar todos los objetos String existentes y leer su contenido. Esto incluye objetos String que son inalcanzables y están esperando la recolección de basura. Si esto le preocupa, deberá borrar los datos confidenciales de la cadena tan pronto como termine con ellos. No puedes hacer esto con objetos String ya que son inmutables. Por lo tanto, es recomendable usar objetos char[] para contener datos de caracteres confidenciales y borrarlos (por ejemplo, sobrescribirlos con caracteres '\000') cuando haya terminado.

Todas las instancias de String se crean en el montón, incluso las instancias que corresponden a literales de cadena. Lo especial de los literales de cadena es que la JVM garantiza que todos los literales que son iguales (es decir, que consisten en los mismos caracteres) estén representados por un solo objeto String (este comportamiento se especifica en JLS). Esto lo implementan los cargadores de clases JVM. Cuando un cargador de clases carga una clase, busca literales de cadena que se usan en la definición de clase, cada vez que ve uno, verifica si ya hay un registro en el grupo de cadenas para este literal (usando el literal como clave) . Si ya hay una entrada para el literal, se usa la referencia a una instancia de String almacenada como el par para ese literal. De lo contrario, se crea una nueva instancia de String y se almacena una referencia a la instancia para el literal (usado como clave) en el conjunto de cadenas. (Véase también internamiento de cadenas).

El grupo de cadenas se mantiene en el montón de Java y está sujeto a la recolección normal de elementos no utilizados.

En las versiones de Java anteriores a Java 7, el conjunto de cadenas se mantenía en una parte especial del montón conocida como “PermGen”. Esta parte solo se recolectó ocasionalmente.

En Java 7, el grupo de cadenas se movió de “PermGen”.

Tenga en cuenta que los literales de cadena son implícitamente accesibles desde cualquier método que los use. Esto significa que los objetos String correspondientes solo se pueden recolectar como basura si el código mismo se recolecta como basura.

Hasta Java 8, los objetos String se implementan como una matriz de caracteres UTF-16 (2 bytes por carácter). Hay una propuesta en Java 9 para implementar String como una matriz de bytes con un campo de indicador de codificación para observar si la cadena está codificada como bytes (LATIN-1) o caracteres (UTF-16).

No use el operador == para comparar cadenas #

A menos que pueda garantizar que todas las cadenas se han internado (consulte a continuación), no debe utilizar los operadores == o != para comparar cadenas. Estos operadores en realidad prueban las referencias, y dado que varios objetos String pueden representar la misma Cadena, es probable que proporcione una respuesta incorrecta.

En su lugar, utilice el método String.equals(Object), que comparará los objetos String en función de sus valores. Para obtener una explicación detallada, consulte https://www.wikiod.com/es/java/errores-comunes-de-java#Trampa: usar == para comparar cadenas

Comparando cadenas con valores constantes #

Al comparar un String con un valor constante, puede colocar el valor constante en el lado izquierdo de equals para asegurarse de que no obtendrá una NullPointerException si el otro String es null.

Mientras que foo.equals("baz") lanzará una NullPointerException si foo es null, "baz".equals(foo) se evaluará como falso.

Una alternativa más legible es usar Objects.equals(), que realiza una verificación nula en ambos parámetros: Objects.equals(foo, "baz").

Comparando con cadenas internas #

La especificación del lenguaje Java (JLS 3.10.6) establece lo siguiente:

“Además, un literal de cadena siempre se refiere a la misma instancia de la clase String. Esto se debe a que los literales de cadena - o, más generalmente, cadenas que son los valores de expresiones constantes - están internados para compartir instancias únicas, utilizando el método String.intern”.

Esto significa que es seguro comparar referencias a dos cadenas literales usando ==. Además, lo mismo es cierto para las referencias a objetos String que se han producido utilizando el método String.intern().

Por ejemplo:

String strObj = new String("Hello!");
String str = "Hello!";

// The two string references point two strings that are equal
if (strObj.equals(str)) {
    System.out.println("The strings are equal");
}

// The two string references do not point to the same object
if (strObj != str) {
    System.out.println("The strings are not the same object");
}

// If we intern a string that is equal to a given literal, the result is
// a string that has the same reference as the literal.
String internedStr = strObj.intern();

if (internedStr == str) {
    System.out.println("The interned string and the literal are the same object");
}

Detrás de escena, el mecanismo interno mantiene una tabla hash que contiene todas las cadenas internas que aún son accesibles. Cuando llamas a intern() en un String, el método busca el objeto en la tabla hash:

  • Si se encuentra la cadena, ese valor se devuelve como la cadena interna.
  • De lo contrario, se agrega una copia de la cadena a la tabla hash y esa cadena se devuelve como la cadena interna.

Es posible usar internamiento para permitir que las cadenas se comparen usando ==. Sin embargo, existen problemas significativos al hacer esto; consulte https://www.wikiod.com/es/java/errores-de-java-problemas-de-rendimiento#Pitfall - Cadenas internas para que puedas usar == es una mala idea =201610101338294726756 para más detalles. No se recomienda en la mayoría de los casos.

Cambiando el caso de los caracteres dentro de una Cadena #

El tipo String proporciona dos métodos para convertir cadenas entre mayúsculas y minúsculas:

  • toUpperCase para convertir todos los caracteres a mayúsculas
  • toLowerCase para convertir todos los caracteres a minúsculas

Ambos métodos devuelven las cadenas convertidas como nuevas instancias de String: los objetos String originales no se modifican porque String es inmutable en Java. Consulte esto para obtener más información sobre la inmutabilidad: [Inmutabilidad de cadenas en Java] 1

String string = "This is a Random String";
String upper = string.toUpperCase();
String lower = string.toLowerCase();

System.out.println(string);   // prints "This is a Random String"
System.out.println(lower);    // prints "this is a random string"
System.out.println(upper);    // prints "THIS IS A RANDOM STRING"

Los caracteres no alfabéticos, como los dígitos y los signos de puntuación, no se ven afectados por estos métodos. Tenga en cuenta que estos métodos también pueden tratar incorrectamente ciertos caracteres Unicode bajo ciertas condiciones.

Nota: estos métodos son sensibles a la configuración regional y pueden producir resultados inesperados si se usan en cadenas que están destinadas a ser interpretadas independientemente de la configuración regional. Algunos ejemplos son los identificadores de lenguajes de programación, las claves de protocolo y las etiquetas HTML.

Por ejemplo, "TITLE".toLowerCase() en una configuración regional turca devuelve “tıtle”, donde ı (\u0131) es el carácter LETRA MINÚSCULA LATINA SIN PUNTO I. Para obtener resultados correctos para las cadenas insensibles a la configuración regional, pase Locale.ROOT como parámetro al método de conversión de mayúsculas y minúsculas correspondiente (por ejemplo, toLowerCase(Locale.ROOT) o toUpperCase(Locale.ROOT)).

Aunque usar Locale.ENGLISH también es correcto para la mayoría de los casos, la forma invariable de idioma es Locale.ROOT.

Se puede encontrar una lista detallada de caracteres Unicode que requieren mayúsculas y minúsculas especiales en el sitio web de Unicode Consortium.

Cambio de mayúsculas y minúsculas de un carácter específico dentro de una cadena ASCII:

Para cambiar el caso de un carácter específico de una cadena ASCII, se puede usar el siguiente algoritmo:

Pasos:

  1. Declarar una cadena.
  2. Introduzca la cadena.
  3. Convierta la cadena en una matriz de caracteres.
  4. Introduzca el carácter que se va a buscar.
  5. Busque el carácter en la matriz de caracteres.
  6. Si lo encuentra, compruebe si el carácter está en minúsculas o en mayúsculas.
    • If Uppercase, add 32 to the ASCII code of the character.
    • If Lowercase, subtract 32 from the ASCII code of the character.
  7. Cambie el carácter original de la matriz de caracteres.
  8. Convierta la matriz de caracteres nuevamente en la cadena.

Voila, se cambia el caso del personaje.

Un ejemplo del código del algoritmo es:

Scanner scanner = new Scanner(System.in);
System.out.println("Enter the String");
String s = scanner.next();
char[] a = s.toCharArray();
System.out.println("Enter the character you are looking for");
System.out.println(s);
String c = scanner.next();
char d = c.charAt(0);

for (int i = 0; i <= s.length(); i++) {
    if (a[i] == d) {
        if (d >= 'a' && d <= 'z') {
            d -= 32;
        } else if (d >= 'A' && d <= 'Z') {
            d += 32;
        }
        a[i] = d;
        break;
    }
}
s = String.valueOf(a);
System.out.println(s);

División de cadenas #

Puede dividir una String en un carácter delimitador particular o una Expresión regular, puede usar el método String.split() que tiene la siguiente firma:

public String[] split(String regex)

Tenga en cuenta que el carácter delimitador o la expresión regular se eliminan de la Matriz de cadenas resultante.

Ejemplo utilizando carácter delimitador:

String lineFromCsvFile = "Mickey;Bolton;12345;121216";
String[] dataCells = lineFromCsvFile.split(";");
// Result is dataCells = { "Mickey", "Bolton", "12345", "121216"};

Ejemplo usando expresión regular:

String lineFromInput = "What    do you need    from me?";
String[] words = lineFromInput.split("\\s+"); // one or more space chars
// Result is words = {"What", "do", "you", "need", "from", "me?"};

Incluso puedes dividir directamente un literal String:

String[] firstNames = "Mickey, Frank, Alicia, Tom".split(", ");
// Result is firstNames = {"Mickey", "Frank", "Alicia", "Tom"};

Advertencia: No olvide que el parámetro siempre se trata como una expresión regular.

"aaa.bbb".split("."); // This returns an empty array

En el ejemplo anterior . se trata como el comodín de la expresión regular que coincide con cualquier carácter, y dado que cada carácter es un delimitador, el resultado es una matriz vacía.

División basada en un delimitador que es un metacarácter de expresiones regulares

Los siguientes caracteres se consideran especiales (también conocidos como metacaracteres) en expresiones regulares

  < > - = ! ( ) [ ] { } \ ^ $ | ? * + .

Para dividir una cadena en función de uno de los delimitadores anteriores, debe escaparlos usando \\ o usar Pattern.quote():

  • Usando Pattern.quote():

     String s = "a|b|c";
 String regex = Pattern.quote("|");
 String[] arr = s.split(regex);

  • Escapar de los caracteres especiales:

     String s = "a|b|c";
 String[] arr = s.split("\\|");

Dividir elimina valores vacíos

split(delimiter) por defecto elimina las cadenas vacías finales de la matriz de resultados. Para desactivar este mecanismo, necesitamos usar una versión sobrecargada de split (delimiter, limit) con el límite establecido en un valor negativo como

String[] split = data.split("\\|", -1);

split(regex) devuelve internamente el resultado de split(regex, 0).

El parámetro de límite controla la cantidad de veces que se aplica el patrón y, por lo tanto, afecta la longitud de la matriz resultante. Si el límite n es mayor que cero, el patrón se aplicará como máximo n - 1 veces, la longitud de la matriz no será mayor que n y la última entrada de la matriz contendrá todas las entradas más allá de la última coincidencia. delimitador Si n es negativo, el patrón se aplicará tantas veces como sea posible y la matriz puede tener cualquier longitud. Si n es cero, el patrón se aplicará tantas veces como sea posible, la matriz puede tener cualquier longitud y las cadenas vacías finales se descartarán.

Grupo de cadenas y almacenamiento en montón #

Como muchos objetos de Java, todas las instancias de String se crean en el montón, incluso los literales. Cuando la JVM encuentra un literal String que no tiene una referencia equivalente en el montón, la JVM crea una instancia String correspondiente en el montón y también almacena una referencia a la instancia String recién creada en el montón. Piscina de cuerdas. Cualquier otra referencia al mismo literal String se reemplaza con la instancia String creada previamente en el montón.

Veamos el siguiente ejemplo:

class Strings
{
    public static void main (String[] args)
    {
        String a = "alpha";
        String b = "alpha";
        String c = new String("alpha");

        //All three strings are equivalent
        System.out.println(a.equals(b) && b.equals(c));

        //Although only a and b reference the same heap object
        System.out.println(a == b);
        System.out.println(a != c);
        System.out.println(b != c);
    }
}

La salida de lo anterior es:

true
true
true
true

Cuando usamos comillas dobles para crear una cadena, primero busca una cadena con el mismo valor en el grupo de cadenas, si se encuentra, solo devuelve la referencia; de lo contrario, crea una nueva cadena en el grupo y luego devuelve la referencia.

Sin embargo, al usar el operador nuevo, obligamos a la clase String a crear un nuevo objeto String en el espacio de almacenamiento dinámico. Podemos usar el método interno () para ponerlo en el grupo o referirnos a otro objeto String del grupo de cadenas que tenga el mismo valor.

El propio grupo de cadenas también se crea en el montón.

Antes de Java 7, String literales se almacenaban en el grupo de constantes de tiempo de ejecución en el área de método de PermGen, que tenía un tamaño fijo.

El grupo de cadenas también residía en PermGen.

RFC: 6962931

En JDK 7, las cadenas internas ya no se asignan en la generación permanente del montón de Java, sino que se asignan en la parte principal del montón de Java (conocidas como las generaciones joven y vieja), junto con los otros objetos creados por el solicitud. Este cambio dará como resultado que residan más datos en el almacenamiento dinámico principal de Java y menos datos en la generación permanente y, por lo tanto, puede que sea necesario ajustar los tamaños del almacenamiento dinámico. La mayoría de las aplicaciones solo verán diferencias relativamente pequeñas en el uso del montón debido a este cambio, pero las aplicaciones más grandes que cargan muchas clases o hacen un uso intensivo del método String.intern() verán diferencias más significativas.

Concatenación de cadenas y StringBuilders #

La concatenación de cadenas se puede realizar usando el operador +. Por ejemplo:

String s1 = "a";
String s2 = "b";
String s3 = "c";
String s = s1 + s2 + s3; // abc

Normalmente, una implementación del compilador realizará la concatenación anterior utilizando métodos que involucran un StringBuilder bajo el capó. Una vez compilado, el código sería similar al siguiente:

StringBuilder sb = new StringBuilder("a");
String s = sb.append("b").append("c").toString();

StringBuilder tiene varios métodos sobrecargados para agregar diferentes tipos, por ejemplo, para agregar un int en lugar de una String. Por ejemplo, una implementación puede convertir:

String s1 = "a";
String s2 = "b";    
String s = s1 + s2 + 2; // ab2

a lo siguiente:

StringBuilder sb = new StringBuilder("a");
String s = sb.append("b").append(2).toString();

Los ejemplos anteriores ilustran una operación de concatenación simple que se realiza efectivamente en un solo lugar en el código. La concatenación implica una sola instancia de StringBuilder. En algunos casos, una concatenación se realiza de forma acumulativa como en un bucle:

String result = "";
for(int i = 0; i < array.length; i++) {
    result += extractElement(array[i]);
}
return result;

En tales casos, la optimización del compilador generalmente no se aplica y cada iteración creará un nuevo objeto StringBuilder. Esto se puede optimizar transformando explícitamente el código para usar un solo StringBuilder:

StringBuilder result = new StringBuilder();
for(int i = 0; i < array.length; i++) {
    result.append(extractElement(array[i]));
}
return result.toString();

Un StringBuilder se inicializará con un espacio vacío de solo 16 caracteres. Si sabe de antemano que construirá cadenas más grandes, puede ser beneficioso inicializarlo con suficiente tamaño por adelantado, de modo que no sea necesario cambiar el tamaño del búfer interno:

StringBuilder buf = new StringBuilder(30); // Default is 16 characters
buf.append("0123456789");
buf.append("0123456789"); // Would cause a reallocation of the internal buffer otherwise
String result = buf.toString(); // Produces a 20-chars copy of the string

Si está produciendo muchas cadenas, es recomendable reutilizar StringBuilders:

StringBuilder buf = new StringBuilder(100);
for (int i = 0; i < 100; i++) {
    buf.setLength(0); // Empty buffer
    buf.append("This is line ").append(i).append('\n');
    outputfile.write(buf.toString());
}

Si (y solo si) varios subprocesos están escribiendo en el mismo búfer, utilice StringBuffer, que es una versión sincronizada de StringBuilder. Pero debido a que generalmente solo un hilo escribe en un búfer, generalmente es más rápido usar StringBuilder sin sincronización.

Usando el método concat():

String string1 = "Hello ";
String string2 = "world";
String string3 = string1.concat(string2);  // "Hello world"

Esto devuelve una nueva cadena que es cadena1 con cadena2 agregada al final. También puede usar el método concat() con cadenas literales, como en:

"My name is ".concat("Buyya");

String s = "this is an example";
String a = s.substring(11); // a will hold the string starting at character 11 until the end ("example")
String b = s.substring(5, 10); // b will hold the string starting at character 5 and ending right before character 10 ("is an")
String b = s.substring(5, b.length()-3); // b will hold the string starting at character 5 ending right before b' s lenght is out of 3  ("is an exam")

Las subcadenas también se pueden aplicar para dividir y agregar/reemplazar caracteres en su cadena original. Por ejemplo, se enfrentó a una fecha china que contenía caracteres chinos, pero desea almacenarla como una Cadena de fecha con formato adecuado.

String datestring = "2015年11月17日"
datestring = datestring.substring(0, 4) + "-" + datestring.substring(5,7) + "-" + datestring.substring(8,10);
//Result will be 2015-11-17

El método substring extrae una parte de una String. Cuando se proporciona un parámetro, el parámetro es el inicio y la pieza se extiende hasta el final de la ‘Cadena’. Cuando se dan dos parámetros, el primer parámetro es el carácter inicial y el segundo parámetro es el índice del carácter justo después del final (el carácter en el índice no está incluido). Una manera fácil de verificar es que la resta del primer parámetro del segundo debería dar como resultado la longitud esperada de la cadena.

En las versiones de JDK <7u6, el método substring instancia una String que comparte el mismo char[] de respaldo que la String original y tiene los campos internos offset y count establecidos en el inicio y la longitud del resultado . Tal intercambio puede causar fugas de memoria, que se pueden evitar llamando a new String(s.substring(...)) para forzar la creación de una copia, después de lo cual char[] se puede recolectar basura.

Desde JDK 7u6, el método substring siempre copia toda la matriz char[] subyacente, haciendo que la complejidad sea lineal en comparación con la constante anterior, pero garantizando al mismo tiempo la ausencia de pérdidas de memoria.

Separador de nueva línea independiente de la plataforma #

Dado que el nuevo separador de línea varía de una plataforma a otra (por ejemplo, \n en sistemas tipo Unix o \r\n en Windows), a menudo es necesario tener una forma de acceder a él independiente de la plataforma. En Java se puede recuperar de una propiedad del sistema:

System.getProperty("line.separator")

Debido a que el nuevo separador de línea es tan comúnmente necesario, desde Java 7 en un método abreviado que devuelve exactamente el mismo resultado que el código anterior está disponible:

System.lineSeparator()

Nota: dado que es muy poco probable que el nuevo separador de línea cambie durante la ejecución del programa, es una buena idea almacenarlo en una variable final estática en lugar de recuperarlo de la propiedad del sistema cada vez que se necesite.

Cuando use String.format, use %n en lugar de \n o ‘\r\n’ para generar un nuevo separador de línea independiente de la plataforma.

System.out.println(String.format('line 1: %s.%nline 2: %s%n', lines[0],lines[1]));

Adición del método toString() para objetos personalizados #

Suponga que ha definido la siguiente clase Persona:

public class Person {

    String name;
    int age;
    
    public Person (int age, String name) {
        this.age = age;
        this.name = name;
    }
}

Si instancias un nuevo objeto Persona:

Person person = new Person(25, "John");

y más adelante en su código, usa la siguiente declaración para imprimir el objeto:

System.out.println(person.toString());

Demostración en vivo en Ideone

obtendrá una salida similar a la siguiente:

Person@7ab89d

Este es el resultado de la implementación del método toString() definido en la clase Object, una superclase de Person. La documentación de Object.toString() dice:

El método toString para la clase Object devuelve una cadena que consta del nombre de la clase de la que el objeto es una instancia, el carácter de arroba `@’ y la representación hexadecimal sin signo del código hash del objeto. En otras palabras, este método devuelve una cadena igual al valor de:

getClass().getName() + ‘@’ + Integer.toHexString(hashCode())

Por lo tanto, para obtener resultados significativos, deberá anular el método toString():

@Override
public String toString() {
    return "My name is " + this.name + " and my age is " + this.age;
}

Ahora la salida será:

My name is John and my age is 25

También puedes escribir

System.out.println(person);

Demostración en vivo en Ideone

De hecho, println() invoca implícitamente el método toString en el objeto.

Invertir cuerdas #

Hay un par de formas en que puede invertir una cadena para hacerlo al revés.

  1. StringBuilder/StringBuffer:

     String code = "code";
 System.out.println(code);

 StringBuilder sb = new StringBuilder(code); 
 code = sb.reverse().toString();

 System.out.println(code);

  2. Matriz de caracteres:

    String code = "code";
System.out.println(code);

char[] array = code.toCharArray();
for (int index = 0, mirroredIndex = array.length - 1; index < mirroredIndex; index++, mirroredIndex--) {
    char temp = array[index];
    array[index] = array[mirroredIndex];
    array[mirroredIndex] = temp;
}

// print reversed
System.out.println(new String(array));

Elimina los espacios en blanco del principio y el final de una cadena #

El método trim() devuelve una nueva cadena con el espacio en blanco inicial y final eliminado.

String s = new String("   Hello World!!  ");
String t = s.trim();  // t = "Hello World!!"

Si recortas un String que no tiene ningún espacio en blanco para eliminar, se te devolverá la misma instancia de String.

Tenga en cuenta que el método trim() tiene su propia noción de espacios en blanco, que difiere de la noción utilizada por Character. isWhitespace() método:

  • Todos los caracteres de control ASCII con códigos U+0000 a U+0020 se consideran espacios en blanco y trim() los elimina. Esto incluye los caracteres U+0020 'SPACE', U+0009 'CHARACTER TABULATION', U+000A 'LINE FEED' y U+000D 'CARRIAGE RETURN', pero también caracteres como U+ 0007 'CAMPANA'.

  • Los espacios en blanco Unicode como U+00A0 'NO-BREAK SPACE' o U+2003 'EM SPACE' no son reconocidos por trim().

Reemplazo de partes de Strings #

Dos formas de reemplazar: por expresión regular o por coincidencia exacta.

Nota: el objeto String original permanecerá sin cambios, el valor devuelto contiene el String modificado.

Coincidencia exacta #

Reemplace un solo carácter con otro solo carácter: #

String replace(char oldChar, char newChar)

Devuelve una nueva cadena resultante de reemplazar todas las apariciones de oldChar en esta cadena con newChar.

String s = "popcorn";
System.out.println(s.replace('p','W'));

Resultado:

WoWcorn

Reemplace la secuencia de caracteres con otra secuencia de caracteres: #

String replace(CharSequence target, CharSequence replacement)

Reemplaza cada subcadena de esta cadena que coincida con la secuencia objetivo literal con la secuencia de reemplazo literal especificada.

String s = "metal petal et al.";
System.out.println(s.replace("etal","etallica"));

Resultado:

metallica petallica et al.

expresión regular #

Nota: la agrupación usa el carácter $ para hacer referencia a los grupos, como $1.

Reemplazar todas las coincidencias: #

String replaceAll(String regex, String replacement)

Reemplaza cada subcadena de esta cadena que coincida con la expresión regular dada con el reemplazo dado.

String s = "spiral metal petal et al.";
System.out.println(s.replaceAll("(\\w*etal)","$1lica"));

Resultado:

spiral metallica petallica et al.

Reemplazar solo la primera coincidencia: #

String replaceFirst(String regex, String replacement)

Reemplaza la primera subcadena de esta cadena que coincide con la expresión regular dada con el reemplazo dado

String s = "spiral metal petal et al.";
System.out.println(s.replaceAll("(\\w*etal)","$1lica"));

Resultado:

spiral metallica petal et al.

Interruptor insensible a mayúsculas y minúsculas #

switch en sí mismo no se puede parametrizar para que no distinga entre mayúsculas y minúsculas, pero si es absolutamente necesario, puede comportarse de forma insensible a la cadena de entrada usando toLowerCase() o toUpperCase:

switch (myString.toLowerCase()) {
     case "case1" :
        ...            
     break;
     case "case2" :
        ...            
     break;
}

Tener cuidado

  • Locale podría afectar cómo [ocurren los casos cambiantes] (https://www.wikiod.com/es/java/instrumentos-de-cuerda Changing-the-case-of-characters-within-a-string)!
  • Se debe tener cuidado de no tener caracteres en mayúsculas en las etiquetas, ¡esos nunca se ejecutarán!

Obtener la longitud de un String #

Para obtener la longitud de un objeto String, llama al método length() en él. La longitud es igual al número de unidades de código UTF-16 (caracteres) en la cadena.

String str = "Hello, World!";
System.out.println(str.length()); // Prints out 13

Demostración en vivo en Ideone

Un char en una cadena es un valor UTF-16. Los puntos de código Unicode cuyos valores son ≥ 0x1000 (por ejemplo, la mayoría de los emojis) usan dos posiciones de caracteres. Para contar el número de puntos de código Unicode en una cadena, independientemente de si cada punto de código cabe en un valor char UTF-16, puede usar el método codePointCount:

int length = str.codePointCount(0, str.length());

También puede usar un Stream de puntos de código, a partir de Java 8:

int length = str.codePoints().count();

Obtener el enésimo carácter en una cadena #

String str = "My String";

System.out.println(str.charAt(0)); // "M"
System.out.println(str.charAt(1)); // "y"
System.out.println(str.charAt(2)); // " "
System.out.println(str.charAt(str.length-1)); // Last character "g"

Para obtener el enésimo carácter en una cadena, simplemente llame a charAt(n) en una String, donde n es el índice del carácter que le gustaría recuperar

NOTA: el índice n comienza en 0, por lo que el primer elemento está en n=0.

Contar las ocurrencias de una subcadena o carácter en una cadena #

El método countMatches de org.apache.commons.lang3.StringUtils se usa típicamente para contar las ocurrencias de una subcadena o carácter en una String:

import org.apache.commons.lang3.StringUtils;

String text = "One fish, two fish, red fish, blue fish";

// count occurrences of a substring
String stringTarget = "fish";
int stringOccurrences = StringUtils.countMatches(text, stringTarget); // 4

// count occurrences of a char
char charTarget = ',';
int charOccurrences = StringUtils.countMatches(text, charTarget); // 3

De lo contrario, para hacer lo mismo con las API estándar de Java, podría usar expresiones regulares:

import java.util.regex.Matcher;
import java.util.regex.Pattern;
 
String text = "One fish, two fish, red fish, blue fish";
System.out.println(countStringInString("fish", text)); // prints 4
System.out.println(countStringInString(",", text)); // prints 3
 

public static int countStringInString(String search, String text) {
    Pattern pattern = Pattern.compile(search);
    Matcher matcher = pattern.matcher(text);
    
    int stringOccurrences = 0;
    while (matcher.find()) {
      stringOccurrences++;
    }
    return stringOccurrences;
}
