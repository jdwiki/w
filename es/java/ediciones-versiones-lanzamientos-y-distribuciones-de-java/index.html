Diferencias entre las distribuciones Java SE JRE o Java SE JDK #

Las versiones Sun/Oracle de Java SE vienen en dos formas: JRE y JDK. En términos simples, los JRE admiten la ejecución de aplicaciones Java y los JDK también admiten el desarrollo de Java.

Entorno de tiempo de ejecución de Java #

Las distribuciones de Java Runtime Environment o JRE consisten en el conjunto de bibliotecas y herramientas necesarias para ejecutar y administrar aplicaciones Java. Las herramientas en un JRE moderno típico incluyen:

  • El comando java para ejecutar un programa Java en una JVM (Java Virtual Machine)
  • El comando jjs para ejecutar el motor Nashorn Javascript.
  • El comando keytool para manipular almacenes de claves Java.
  • El comando policytool para editar las políticas de seguridad de la zona de pruebas de seguridad.
  • Las herramientas pack200 y unpack200 para empaquetar y desempaquetar el archivo “pack200” para la implementación web.
  • Los comandos orbd, rmid, rmiregistry y tnameserv que admiten aplicaciones Java CORBA y RMI.

Los instaladores de “Desktop JRE” incluyen un complemento de Java adecuado para algunos navegadores web. Esto se deja deliberadamente fuera de “Server JRE” installers.linux syscall read benchmarku

Desde la actualización 6 de Java 7 en adelante, los instaladores de JRE han incluido JavaFX (versión 2.2 o posterior).

Kit de desarrollo de Java #

Una distribución de Java Development Kit o JDK incluye las herramientas JRE y herramientas adicionales para desarrollar software Java. Las herramientas adicionales suelen incluir:

  • El comando javac, que compila el código fuente de Java ("".java") en archivos de código de bytes (".class").
  • Las herramientas para crear archivos JAR como jar y jarsigner
  • Herramientas de desarrollo como:
    • appletviewer for running applets
    • idlj the CORBA IDL to Java compiler
    • javah the JNI stub generator
    • native2ascii for character set conversion of Java source code
    • schemagen the Java to XML schema generator (part of JAXB)
    • serialver generate Java Object Serialization version string.
    • the wsgen and wsimport support tools for JAX-WS
  • Herramientas de diagnóstico como:
    • jdb the basic Java debugger
    • jmap and jhat for dumping and analysing a Java heap.
    • jstack for getting a thread stack dump.
    • javap for examining “.class” files.
  • Herramientas de gestión y monitorización de aplicaciones como:
    • jconsole a management console,
    • jstat, jstatd, jinfo and jps for application monitoring

Una instalación típica de Sun/Oracle JDK también incluye un archivo ZIP con el código fuente de las bibliotecas de Java. Antes de Java 6, este era el único código fuente de Java disponible públicamente.

Desde Java 6 en adelante, el código fuente completo de OpenJDK está disponible para su descarga desde el sitio de OpenJDK. Por lo general, no se incluye en los paquetes JDK (Linux), pero está disponible como un paquete separado.

Notas al pie:

  1. Los enlaces son para copias en línea de la documentación de las versiones respectivas en el sitio web de Oracle. La documentación de muchas versiones anteriores ya no está en línea, aunque normalmente se puede descargar desde Oracle Java Archives.

  2. La mayoría de las versiones históricas de Java SE han superado sus fechas oficiales de “fin de vida útil”. Cuando una versión de Java supera este hito, Oracle deja de proporcionar actualizaciones gratuitas. Las actualizaciones aún están disponibles para los clientes con contratos de soporte.

Aspectos destacados de la versión Java SE #

Versión Java SE Destacados

Diferencias entre Java EE, Java SE, Java ME y JavaFX #

La tecnología Java es tanto un lenguaje de programación como una plataforma. El lenguaje de programación Java es un lenguaje orientado a objetos de alto nivel que tiene una sintaxis y estilo particulares. Una plataforma Java es un entorno particular en el que se ejecutan aplicaciones del lenguaje de programación Java.

Hay varias plataformas Java. Muchos desarrolladores, incluso los desarrolladores del lenguaje de programación Java desde hace mucho tiempo, no entienden cómo se relacionan las diferentes plataformas entre sí.

Las plataformas del lenguaje de programación Java #

Hay cuatro plataformas del lenguaje de programación Java:

  • Plataforma Java, edición estándar (Java SE)

  • Plataforma Java, edición empresarial (Java EE)

  • Plataforma Java, Micro Edición (Java ME)

-Java FX

Todas las plataformas Java constan de una máquina virtual Java (VM) y una interfaz de programación de aplicaciones (API). La máquina virtual de Java es un programa, para una plataforma de hardware y software en particular, que ejecuta aplicaciones de tecnología Java. Una API es una colección de componentes de software que puede utilizar para crear otros componentes de software o aplicaciones. Cada plataforma Java proporciona una máquina virtual y una API, y esto permite que las aplicaciones escritas para esa plataforma se ejecuten en cualquier sistema compatible con todas las ventajas del lenguaje de programación Java: independencia de la plataforma, potencia, estabilidad, facilidad de desarrollo y seguridad.

Java SE #

Cuando la mayoría de la gente piensa en el lenguaje de programación Java, piensan en la API Java SE. La API de Java SE proporciona la funcionalidad central del lenguaje de programación Java. Define todo, desde los tipos y objetos básicos del lenguaje de programación Java hasta clases de alto nivel que se utilizan para redes, seguridad, acceso a bases de datos, desarrollo de interfaz gráfica de usuario (GUI) y análisis XML.

Además de la API central, la plataforma Java SE consta de una máquina virtual, herramientas de desarrollo, tecnologías de implementación y otras bibliotecas de clases y juegos de herramientas que se usan comúnmente en las aplicaciones de tecnología Java.

Java EE #

La plataforma Java EE se basa en la plataforma Java SE. La plataforma Java EE proporciona una API y un entorno de tiempo de ejecución para desarrollar y ejecutar aplicaciones de red a gran escala, de varios niveles, escalables, fiables y seguras.

Java ME #

La plataforma Java ME proporciona una API y una máquina virtual de tamaño reducido para ejecutar aplicaciones del lenguaje de programación Java en dispositivos pequeños, como teléfonos móviles. La API es un subconjunto de la API de Java SE, junto con bibliotecas de clases especiales útiles para el desarrollo de aplicaciones de dispositivos pequeños. Las aplicaciones Java ME suelen ser clientes de los servicios de la plataforma Java EE.

Efectos de Java #

La tecnología Java FX es una plataforma para crear aplicaciones ricas de Internet escritas en Java FX ScriptTM. Java FX Script es un lenguaje declarativo de tipo estático que se compila en el código de bytes de la tecnología Java, que luego se puede ejecutar en una máquina virtual Java. Las aplicaciones escritas para la plataforma Java FX pueden incluir clases del lenguaje de programación Java y vincularse a ellas, y pueden ser clientes de los servicios de la plataforma Java EE.

¿Cuál es la diferencia entre Oracle Hotspot y OpenJDK? #

Ortogonal a la dicotomía JRE versus JDK, hay dos tipos de versiones de Java que están ampliamente disponibles:

  • Las versiones de Oracle Hotspot son las que descarga de los sitios de descarga de Oracle.
  • Las versiones de OpenJDK son las que se crean (normalmente por proveedores externos) a partir de los repositorios de código fuente de OpenJDK.

En términos funcionales, hay poca diferencia entre una versión Hotspot y una versión OpenJDK. Hay algunas funciones “empresariales” adicionales en Hotspot que los clientes de Java (de pago) de Oracle pueden habilitar, pero aparte de eso, la misma tecnología está presente tanto en Hotspot como en OpenJDK.

Otra ventaja de Hotspot sobre OpenJDK es que los lanzamientos de parches para Hotspot tienden a estar disponibles un poco antes. Esto también depende de qué tan ágil sea su proveedor de OpenJDK; p.ej. cuánto tiempo tarda el equipo de compilación de una distribución de Linux en preparar y realizar el control de calidad de una nueva compilación de OpenJDK y colocarla en sus repositorios públicos.

La otra cara de la moneda es que las versiones de Hotspot no están disponibles en los repositorios de paquetes para la mayoría de las distribuciones de Linux. Esto significa que mantener su software Java actualizado en una máquina Linux suele ser más trabajo si usa Hotspot.
