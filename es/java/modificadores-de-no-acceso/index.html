Los modificadores de no acceso no cambian la accesibilidad de las variables y los métodos, pero les proporcionan propiedades especiales.

definitivo #

final en Java puede referirse a variables, métodos y clases. Hay tres reglas simples:

  • la variable final no se puede reasignar
  • el método final no se puede anular
  • la clase final no se puede extender

Usos

Buena práctica de programación

Algunos desarrolladores consideran una buena práctica marcar un final variable cuando se puede. Si tiene una variable que no debe cambiarse, debe marcarla como final.

Un uso importante de la palabra clave final si se trata de parámetros de método. Si quiere enfatizar que un método no cambia sus parámetros de entrada, marque las propiedades como finales.

public int sumup(final List<Integer> ints);

Esto enfatiza que el método sumup no va a cambiar los ints.

Acceso a clase interna

Si su clase interna anónima quiere acceder a una variable, la variable debe marcarse como final

  public IPrintName printName(){
    String name;
    return new IPrintName(){
        @Override
        public void printName(){
            System.out.println(name);
        }
    };
}

Esta clase no compila, ya que la variable name no es definitiva.

Efectivamente, las variables finales son una excepción. Estas son variables locales que se escriben solo una vez y, por lo tanto, podrían hacerse definitivas. Efectivamente, también se puede acceder a las variables finales desde clases anónimas.

Variable estática final

Aunque el siguiente código es completamente legal cuando la variable final foono esestática, en el caso de estática` no se compilará:

class TestFinal {
    private final static List foo;

    public Test() {
        foo = new ArrayList();
    }
}

La razón es, repitamos de nuevo, la variable final no se puede reasignar. Dado que foo es estático, se comparte entre todas las instancias de la clase TestFinal. Cuando se crea una nueva instancia de una clase TestFinal, se invoca su constructor y, por lo tanto, se reasigna foo que el compilador no permite. Una forma correcta de inicializar la variable foo en este caso es:

class TestFinal {
    private static final List foo = new ArrayList();
    //..
}

o usando un inicializador estático:

class TestFinal {
    private static final List foo;
    static {
        foo = new ArrayList();
    }
    //..
}

Los métodos finales son útiles cuando la clase base implementa alguna funcionalidad importante que se supone que la clase derivada no debe cambiar. También son más rápidos que los métodos no finales, porque no involucran el concepto de tabla virtual.

Todas las clases contenedoras en Java son finales, como Integer, Long, etc. Los creadores de estas clases no querían que nadie pudiera, p. extender Integer a su propia clase y cambiar el comportamiento básico de la clase Integer. Uno de los requisitos para que una clase sea inmutable es que las subclases no anulen los métodos. La forma más sencilla de hacer esto es declarar la clase como final.

estático #

La palabra clave static se usa en una clase, método o campo para que funcionen independientemente de cualquier instancia de la clase.

  • Los campos estáticos son comunes a todas las instancias de una clase. No necesitan una instancia para acceder a ellos.

  • Los métodos estáticos se pueden ejecutar sin una instancia de la clase en la que se encuentran. Sin embargo, solo pueden acceder a campos estáticos de esa clase.

  • Las clases estáticas se pueden declarar dentro de otras clases. No necesitan una instancia de la clase en la que se encuentran para ser instanciados.

    public class TestStatic { static int staticVariable;

      static {
      // This block of code is run when the class first loads
      staticVariable = 11;
  }

  int nonStaticVariable = 5;

  static void doSomething() {
      // We can access static variables from static methods
      staticVariable = 10;
  }

  void add() {
      // We can access both static and non-static variables from non-static methods
      nonStaticVariable += staticVariable;
  }

  static class StaticInnerClass {
      int number;
      public StaticInnerClass(int _number) {
          number = _number;
      }

      void doSomething() {
          // We can access number and staticVariable, but not nonStaticVariable
          number += staticVariable;
      }

      int getNumber() {
          return number;
      }
  }

    }

    // Static fields and methods TestStatic object1 = new TestStatic();

    System.out.println(object1.staticVariable); // 11 System.out.println(TestStatic.staticVariable); // 11

    TestStatic.doSomething();

    TestStatic object2 = new TestStatic();

    System.out.println(object1.staticVariable); // 10 System.out.println(object2.staticVariable); // 10 System.out.println(TestStatic.staticVariable); // 10

    object1.add();

    System.out.println(object1.nonStaticVariable); // 15 System.out.println(object2.nonStaticVariable); // 10

    // Static inner classes StaticInnerClass object3 = new TestStatic.StaticInnerClass(100); StaticInnerClass object4 = new TestStatic.StaticInnerClass(200);

    System.out.println(object3.getNumber()); // 100 System.out.println(object4.getNumber()); // 200

    object3.doSomething();

    System.out.println(object3.getNumber()); // 110 System.out.println(object4.getNumber()); // 200

resumen #

La abstracción es un proceso de ocultar los detalles de implementación y mostrar solo la funcionalidad al usuario. Una clase abstracta nunca puede ser instanciada. Si una clase se declara como abstracta, el único propósito es que la clase se extienda.

abstract class Car
{
    abstract void tagLine();
}
 
class Honda extends Car
{
    void tagLine()
    {
        System.out.println("Start Something Special");
    }
}
 
class Toyota extends Car
{
    void tagLine()
    {
        System.out.println("Drive Your Dreams");
    }
}

estricto #

El modificador strictfp se usa para cálculos de coma flotante. Este modificador hace que la variable de punto flotante sea más consistente en varias plataformas y garantiza que todos los cálculos de punto flotante se realicen de acuerdo con los estándares IEEE 754 para evitar errores de cálculo (errores de redondeo), desbordamientos y subdesbordamientos en arquitecturas de 32 bits y 64 bits. Esto no se puede aplicar en métodos abstractos, variables o constructores.

// strictfp keyword can be applied on methods, classes and interfaces.

strictfp class A{}

strictfp interface M{}

class A{  
    strictfp void m(){}
}

volátil #

El modificador ‘volátil’ se usa en la programación de subprocesos múltiples. Si declara un campo como volátil, es una señal para los subprocesos de que deben leer el valor más reciente, no uno almacenado en caché localmente. Además, se garantiza que las lecturas y escrituras “volátiles” sean atómicas (el acceso a un “largo” o “doble” no volátil no es atómico), lo que evita ciertos errores de lectura/escritura entre varios subprocesos.

public class MyRunnable implements Runnable
{
    private volatile boolean active;
 
    public void run(){ // run is called in one thread 
        active = true;
        while (active){
            // some code here
        }
    }
    
    public void stop(){ // stop() is called from another thread
        active = false;
    }
}

sincronizado #

El modificador sincronizado se usa para controlar el acceso de un método en particular o un bloque por múltiples subprocesos. Solo un hilo puede entrar en un método o bloque que se declara sincronizado. La palabra clave sincronizada funciona en el bloqueo intrínseco de un objeto, en el caso de un método sincronizado, los objetos actuales bloquean y el método estático usa el objeto de clase. Cualquier subproceso que intente ejecutar un bloque sincronizado debe adquirir primero el bloqueo de objeto.

class Shared
{
    int i;
 
    synchronized void SharedMethod()
    {
        Thread t = Thread.currentThread();
 
        for(int i = 0; i <= 1000; i++)
        {
            System.out.println(t.getName()+" : "+i);
        }
    }

    void SharedMethod2()
    {
        synchronized (this)
        {
            System.out.println("Thais access to currect object is synchronize "+this);
        }
    }
}
 
public class ThreadsInJava
{
    public static void main(String[] args)
    {
        final Shared s1 = new Shared();
 
        Thread t1 = new Thread("Thread - 1")
        {
            @Override
            public void run()
            {
                s1.SharedMethod();
            }
        };
 
        Thread t2 = new Thread("Thread - 2")
        {
            @Override
            public void run()
            {
                s1.SharedMethod();
            }
        };
 
        t1.start();
 
        t2.start();
    }
}

transitorio #

Una variable declarada como transitoria no se serializará durante la serialización de objetos.

public transient int limit = 55;   // will not persist
public int b; // will persist
