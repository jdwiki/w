Java y C++ son lenguajes similares. Este tema sirve como guía de referencia rápida para los ingenieros de Java y C++.

Clases definidas dentro de otras construcciones #

Definido dentro de otra clase #

C++ #

Clase anidada[ref] (necesita una referencia a la clase adjunta)

class Outer {
   class Inner {
      public:
         Inner(Outer* o) :outer(o) {}

      private:
         Outer*  outer;
   };
};

##Java ## [no estático] Clase anidada (también conocida como clase interna o clase miembro)

class OuterClass {
    ...
    class InnerClass {
        ...
    }
}

Estáticamente definido dentro de otra clase #

C++ #

Clase anidada estática

class Outer {
   class Inner {
      ...
   };
};

##Java ## Clase anidada estática (también conocida como clase de miembro estático)[ref]

class OuterClass {
    ...
    static class StaticNestedClass {
        ...
    }
}

(por ejemplo, gestión de eventos)

C++ #

Clase local[ref]

void fun() {
   class Test {
      /* members of Test class */
   };
}

Ver también Expresiones lambda

##Java ## Clase local[ref]

class Test {
    void f() {
        new Thread(new Runnable() {
            public void run() {
                doSomethingBackgroundish();
            }
        }).start();
    }
}

Anular vs sobrecargar #

Los siguientes puntos de anulación frente a sobrecarga se aplican tanto a C++ como a Java:

  • Un método anulado tiene el mismo nombre y argumentos que su base método.
  • Un método sobrecargado tiene el mismo nombre pero diferente argumentos y no depende de la herencia.
  • Dos métodos con el mismo nombre y argumentos pero diferente tipo de devolución son ilegales. Consulte las preguntas relacionadas de Stackoverflow sobre “sobrecarga con diferentes tipos de retorno en Java” - [Pregunta 1] (https://stackoverflow.com/questions/2439782/overload-with- different-return-type-in-java); [Pregunta 2](https://stackoverflow.com/questions/5561436/can-two-java-methods-have-same-name-with- different-return-types)

polimorfismo #

El polimorfismo es la capacidad de los objetos de diferentes clases relacionados por herencia para responder de manera diferente a la misma llamada de método. Aquí hay un ejemplo:

  • clase base Forma con área como método abstracto
  • dos clases derivadas, Square y Circle, implementan métodos de área
  • Puntos de referencia de forma a Square y se invoca el área

En C++, el polimorfismo está habilitado por métodos virtuales. En Java, los métodos son virtuales por defecto.

Orden de Construcción/Destrucción #

Limpieza de objetos #

En C++, es una buena idea declarar un destructor como virtual para garantizar que se llamará al destructor de la subclase si se elimina el puntero de la clase base.

En Java, un método de finalización es similar a un destructor en C++; sin embargo, los finalizadores son impredecibles (se basan en GC). Práctica recomendada: use un método “cerrar” para limpiar explícitamente.

protected void close() {
    try {
       // do subclass cleanup
    }
    finally {
       isClosed = true;
       super.close();
    }
}

protected void finalize() {
    try {
       if(!isClosed) close();
    }
    finally {
       super.finalize();
    }
}

predeterminado el valor predeterminado de la clase es privado; la estructura predeterminada es pública accesible por todas las clases dentro del mismo paquete

El temido problema del diamante #

El problema del diamante es una ambigüedad que surge cuando dos clases B y C hereda de A, y la clase D hereda tanto de B como de C. Si hay un método en A que B y C han anulado, y D no anula , entonces qué versión del método hereda D: la de B, o el de C? (de Wikipedia)

Si bien C++ siempre ha sido susceptible al problema del diamante, Java lo fue hasta Java 8. Originalmente, Java no admitía la herencia múltiple, pero con la llegada de los métodos de interfaz predeterminados, las clases de Java no pueden heredar la “implementación” de más de una clase. .

java.lang.Objeto Clase #

En Java todas las clases heredan, ya sea implícita o explícitamente, de la clase Object. Cualquier referencia de Java se puede convertir al tipo de objeto.

C++ no tiene una clase de “Objeto” comparable.

Pasar por valor y pasar por referencia #

Muchos argumentan que Java es SOLO pasar por valor, pero tiene más matices que eso. Compare los siguientes ejemplos de C++ y Java para ver los muchos tipos de paso por valor (también conocido como copia) y paso por referencia (también conocido como alias).

Ejemplo de C++ (código completo) #

  // passes a COPY of the object
  static void passByCopy(PassIt obj) {
     obj.i = 22;  // only a "local" change
  }

  // passes a pointer
  static void passByPointer(PassIt* ptr) {
     ptr->i = 33;
     ptr = 0; // better to use nullptr instead if '0'
  }

  // passes an alias (aka reference)
  static void passByAlias(PassIt& ref) {
     ref.i = 44;
  }

  // This is an old-school way of doing it.
  // Check out std::swap for the best way to do this
  static void swap(PassIt** pptr1, PassIt** pptr2) {
     PassIt* tmp = *pptr1;
     *pptr1 = *pptr2;
     *pptr2 = tmp;
  }

Ejemplo de Java (código completo) #

   // passes a copy of the variable
   // NOTE: in java only primitives are pass-by-copy
   public static void passByCopy(int copy) {
      copy = 33;  // only a "local" change
   }

   // No such thing as pointers in Java
   /*
   public static void passByPointer(PassIt *ptr) {
      ptr->i = 33;
      ptr = 0; // better to use nullptr instead if '0'
   }
   */

   // passes an alias (aka reference)
   public static void passByAlias(PassIt ref) {
      ref.i = 44;
   }

   // passes aliases (aka references),
   // but need to do "manual", potentially expensive copies
   public static void swap(PassIt ref1, PassIt ref2) {
      PassIt tmp = new PassIt(ref1);
      ref1.copy(ref2);
      ref2.copy(tmp);
   }

Outcast Downcasting #

Tenga cuidado con el uso de “downcasting”: Downcasting reduce la jerarquía de herencia de una clase base a una subclase (es decir, lo opuesto al polimorfismo). En general, use polimorfismo y sobreescritura en lugar de instancia de y reducción.

C++ Ejemplo #

// explicit type case required
Child *pChild =  (Child *) &parent;

Ejemplo de Java #

if(mySubClass instanceof SubClass) {
   SubClass mySubClass = (SubClass)someBaseClass;
   mySubClass.nonInheritedMethod();
}
