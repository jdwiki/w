Esta página de documentación brinda acceso a las instrucciones para instalar la edición estándar de Java en computadoras Windows, Linux y macOS.

Configuración de %PATH% y %JAVA_HOME% después de instalar en Windows #

Supuestos: #

  • Se ha instalado un JDK de Oracle.
  • El JDK se instaló en el directorio predeterminado.

Pasos de configuración #

  1. Abra el Explorador de Windows.
  2. En el panel de navegación de la izquierda, haga clic derecho en Esta PC (o Computadora para versiones anteriores de Windows). Hay una manera más corta sin usar el explorador en las versiones reales de Windows: simplemente presione Win+Pause
  3. En la ventana del Panel de control recién abierta, haga clic con el botón izquierdo en Configuración avanzada del sistema, que debería estar en la esquina superior izquierda. Esto abrirá la ventana Propiedades del sistema.

Alternativamente, escriba SystemPropertiesAdvanced (sin distinción entre mayúsculas y minúsculas) en Ejecutar (Win+R), y presione Enter.

  1. En la pestaña Avanzado de Propiedades del sistema, seleccione el botón Variables de entorno… en la esquina inferior derecha de la ventana.
  2. Agregue una Nueva variable del sistema haciendo clic en el botón Nueva… en Variables del sistema con el nombre JAVA_HOME y cuyo valor sea la ruta al directorio donde estaba el JDK. instalado. Después de ingresar estos valores, presione OK.

  1. Desplácese hacia abajo en la lista de Variables del sistema y seleccione la variable Ruta.

  2. PRECAUCIÓN: Windows se basa en Path para encontrar programas importantes. Si se elimina una parte o la totalidad, es posible que Windows no pueda funcionar correctamente. Debe modificarse para permitir que Windows ejecute el JDK. Con esto en mente, haga clic en el botón “Editar…” con la variable Ruta seleccionada. Agregue %JAVA_HOME%\bin; al principio de la variable Path.

Es mejor agregar al principio de la línea porque el software de Oracle solía registrar su propia versión de Java en Path. Esto hará que su versión sea ignorada si ocurre después de la declaración de Oracle.


Revisa tu trabajo #

  1. Abra el símbolo del sistema haciendo clic en Inicio, luego escriba cmd y presione Enter.
  2. Ingrese javac -version en el aviso. Si tuvo éxito, la versión del JDK se imprimirá en la pantalla.

Nota: si tiene que volver a intentarlo, cierre el aviso antes de comprobar su trabajo. Esto obligará a Windows a obtener la nueva versión de Path.

Instalar un JDK de Java en Linux #

Uso del administrador de paquetes #

Las versiones de JDK y/o JRE para OpenJDK u Oracle se pueden instalar utilizando el administrador de paquetes en la mayoría de las distribuciones principales de Linux. (Las opciones que están disponibles para usted dependerán de la distribución).

Como regla general, el procedimiento es abrir la ventana del terminal y ejecutar los comandos que se muestran a continuación. (Se supone que tiene acceso suficiente para ejecutar comandos como usuario “raíz”… que es lo que hace el comando sudo. Si no lo tiene, hable con los administradores de su sistema).

Se recomienda usar el administrador de paquetes porque (generalmente) hace que sea más fácil mantener actualizada la instalación de Java.

apt-get, distribuciones de Linux basadas en Debian (Ubuntu, etc.)

Las siguientes instrucciones instalarán Oracle Java 8:

$ sudo add-apt-repository ppa:webupd8team/java
$ sudo apt-get update
$ sudo apt-get install oracle-java8-installer

Conmutación de línea de comandos

Cuando se instala Java, la versión instalada se establece automáticamente como predeterminada. Para cambiar entre diferentes, use:

export JAVA_HOME=/usr/libexec/java_home -v 1.6 #Or 1.7 or 1.8

Las siguientes funciones se pueden agregar a ~/.bash_profile (si usa el shell Bash predeterminado) para facilitar su uso:

function java_version {
   echo 'java -version';
}

function java_set {
   if [[ $1 == "6" ]]
   then
       export JAVA_HOME='/usr/libexec/java_home -v 1.6';
       echo "Setting Java to version 6..."
       echo "$JAVA_HOME"
   elif [[ $1 == "7" ]]
   then
       export JAVA_HOME='/usr/libexec/java_home -v 1.7';
       echo "Setting Java to version 7..."
       echo "$JAVA_HOME"
   elif [[ $1 == "8" ]]
   then
       export JAVA_HOME='/usr/libexec/java_home -v 1.8';
       echo "Setting Java to version 8..."
       echo "$JAVA_HOME"
   fi            
}

Puesta en marcha con versiones portátiles

Hay instancias en las que quizás desee instalar JDK/JRE en un sistema con privilegios limitados como una máquina virtual o quizás desee instalar y usar varias versiones o arquitecturas (x64/x86) de JDK/JRE. Los pasos siguen siendo los mismos hasta el momento en que descarga el instalador (.EXE). Los pasos posteriores son los siguientes (los pasos son aplicables para JDK/JRE 7 y superior, para versiones anteriores son ligeramente diferentes en los nombres de carpetas y archivos):

  1. Mueva el archivo a una ubicación apropiada donde desee que sus archivos binarios de Java residan de forma permanente.

  2. Instale 7-Zip o su versión portátil si tiene privilegios limitados.

  3. Con 7-Zip, extraiga los archivos del EXE del instalador de Java a la ubicación.

  4. Abra el símbolo del sistema allí manteniendo presionada la tecla ‘Shift’ y ‘Haga clic con el botón derecho’ en la carpeta en el explorador o navegue a esa ubicación desde cualquier lugar.

  5. Navegue a la carpeta recién creada. Digamos que el nombre de la carpeta es jdk-7u25-windows-x64. Así que escribe cd jdk-7u25-windows-x64. Luego escriba los siguientes comandos en orden:

    cd .rsrc\JAVA_CAB10

    extrac32 111

  6. Esto creará un archivo tools.zip en esa ubicación. Extraiga tools.zip con 7-Zip para que los archivos que contiene ahora se creen bajo tools en el mismo directorio.

  7. Ahora ejecute estos comandos en el símbolo del sistema ya abierto:

    cd tools

    for /r %x in (*.pack) do .\bin\unpack200 -r "%x" "%~dx%~px%~nx.jar"

  8. Espere a que se complete el comando. Copie el contenido de herramientas en la ubicación donde desea que estén sus archivos binarios.

De esta manera, puede instalar cualquier versión de JDK/JRE que necesite instalar simultáneamente.

Selección de una versión de Java SE adecuada #

Ha habido muchos lanzamientos de Java desde el lanzamiento original de Java 1.0 en 1995. (Consulte Historial de versiones de Java para obtener un resumen). Sin embargo, la mayoría de los lanzamientos han pasado sus fechas oficiales de fin de vida. Esto significa que el proveedor (generalmente Oracle ahora) ha cesado el desarrollo nuevo para la versión y ya no proporciona parches públicos/gratuitos para cualquier error o problema de seguridad. (Los lanzamientos de parches privados suelen estar disponibles para personas u organizaciones con un contrato de soporte; comuníquese con la oficina de ventas de su proveedor).

En general, la versión de Java SE recomendada para su uso será la última actualización de la última versión pública. Actualmente, esto significa la última versión disponible de Java 8. Java 9 se lanzará públicamente en 2017. (Java 7 ha superado su fin de vida y el último lanzamiento público fue en abril de 2015. No se recomienda Java 7 ni versiones anteriores).

Esta recomendación se aplica a todos los nuevos desarrollos de Java y a cualquiera que esté aprendiendo Java. También se aplica a las personas que solo desean ejecutar el software Java proporcionado por un tercero. En términos generales, el código Java bien escrito funcionará en una versión más reciente de Java. (Pero consulte las notas de la versión del software y comuníquese con el autor/proveedor/vendedor si tiene dudas).

Si está trabajando en una base de código Java más antigua, le recomendamos que se asegure de que su código se ejecute en la última versión de Java. Decidir cuándo comenzar a usar las funciones de las versiones más recientes de Java es más difícil, ya que esto afectará su capacidad para brindar asistencia a los clientes que no pueden o no desean instalar Java.

Lanzamiento de Java y nombre de la versión #

El nombre de la versión de Java es un poco confuso. En realidad, existen dos sistemas de denominación y numeración, como se muestra en esta tabla:

versión JDK Nombre comercial
jdk-1.0 JDK 1.0
jdk-1.1 JDK 1.1
jdk-1.2 J2SE 1.2
… …
jdk-1.5 J2SE 1.5 renombrado Java SE 5
jdk-1.6 Java SE 6
jdk-1.7 Java SE 7
jdk-1.8 Java SE 8
jdk-91 Java SE 9 (aún no publicado)

1: parece que Oracle tiene la intención de romper con su práctica anterior de usar un esquema de “número de versión semántica” en las cadenas de versión de Java. Queda por ver si seguirán adelante con esto.

El “SE” en los nombres comerciales se refiere a la Edición estándar. Esta es la versión base para ejecutar Java en la mayoría de las computadoras portátiles, PC y servidores (aparte de Android).

Hay otras dos ediciones oficiales de Java: “Java ME” es la Micro Edition y “Java EE” es la Enterprise Edition. El sabor de Android de Java también es significativamente diferente de Java SE. Java ME, Java EE y Android Java están fuera del alcance de este tema.

El número de versión completo para una versión de Java se ve así:

1.8.0_101-b13

Esto dice JDK 1.8.0, Actualización 101, Build #13. Oracle se refiere a esto en las notas de la versión como:

Java™ SE Development Kit 8, Update 101 (JDK 8u101)

El número de actualización es importante: Oracle publica periódicamente actualizaciones de una versión principal con parches de seguridad, correcciones de errores y (en algunos casos) nuevas funciones. El número de compilación suele ser irrelevante. Tenga en cuenta que Java 8 y Java 1.8 se refieren a lo mismo; Java 8 es solo el nombre de “marketing” para Java 1.8.

¿Qué necesito para el desarrollo de Java? #

Una instalación de JDK y un editor de texto son lo mínimo para el desarrollo de Java. (Es bueno tener un editor de texto que pueda resaltar la sintaxis de Java, pero puede prescindir de él).

Sin embargo, para un trabajo de desarrollo serio, se recomienda que también utilice lo siguiente:

  • Un IDE de Java como Eclipse, Intellij IDEA o NetBeans
  • Una herramienta de compilación Java como Ant, Gradle o Maven
  • Un sistema de control de versiones para administrar su base de código (con copias de seguridad adecuadas y replicación fuera del sitio)
  • Herramientas de prueba y herramientas de CI (integración continua)

Configurar y cambiar versiones de Java en Linux usando alternativas #

Uso de alternativas #

Muchas distribuciones de Linux usan el comando “alternativas” para cambiar entre diferentes versiones de un comando. Puede usar esto para cambiar entre diferentes versiones de Java instaladas en una máquina.

  1. En un shell de comandos, establezca $JDK en el nombre de ruta de un JDK recién instalado; p.ej.

     $ JDK=/Data/jdk1.8.0_67

  2. Use alternatives --install para agregar los comandos en el SDK de Java a las alternativas:

     $ sudo alternatives --install /usr/bin/java java $JDK/bin/java 2
 $ sudo alternatives --install /usr/bin/javac javac $JDK/bin/javac 2
 $ sudo alternatives --install /usr/bin/jar jar $JDK/bin/jar 2

    And so on.

Ahora puede cambiar entre diferentes versiones de un comando Java de la siguiente manera:

$ sudo alternatives --config javac

There is 1 program that provides 'javac'.

  Selection    Command
-----------------------------------------------
*+ 1           /usr/lib/jvm/java-1.8.0-openjdk-1.8.0.101-1.b14.fc23.x86_64/bin/javac
   2           /Data/jdk1.8.0_67/bin/javac

Enter to keep the current selection[+], or type selection number: 2
$

Para obtener más información sobre el uso de alternativas, consulte la entrada manual alternativas(8).

Instalaciones basadas en Arch #

$ archlinux-java status

Comprobación y configuración posteriores a la instalación en Linux #

Después de instalar un SDK de Java, es recomendable verificar que esté listo para usar. Puede hacer esto ejecutando estos dos comandos, usando su cuenta de usuario normal:

$ java -version
$ javac -version

Estos comandos imprimen la información de la versión de JRE y JDK (respectivamente) que se encuentran en la ruta de búsqueda de comandos de su shell. Busque la cadena de versión de JDK/JRE.

  • Si alguno de los comandos anteriores falla y dice “comando no encontrado”, entonces JRE o JDK no están en la ruta de búsqueda; vaya a Configuración de PATH directamente a continuación.
  • Si cualquiera de los comandos anteriores muestra una cadena de versión diferente a la que esperaba, entonces su ruta de búsqueda o el sistema de “alternativas” deben ajustarse; vaya a Comprobación de alternativas
  • Si se muestran las cadenas de versión correctas, casi ha terminado; Saltar a Comprobación de JAVA_HOME

Configurando PATH directamente

Si no hay java o javac en la ruta de búsqueda en este momento, entonces la solución simple es agregarlo a su ruta de búsqueda.

Primero, busque dónde instaló Java; consulte ¿Dónde se instaló Java? a continuación si tiene dudas.

Luego, asumiendo que bash es su shell de comandos, use un editor de texto para agregar las siguientes líneas al final de ~/.bash_profile o ~/.bashrc (si usa Bash como su shell).

JAVA_HOME=<installation directory>
PATH=$JAVA_HOME/bin:$PATH

export JAVA_HOME
export PATH

… reemplazando <directorio de instalación> con el nombre de ruta para su directorio de instalación de Java. Tenga en cuenta que lo anterior asume que el directorio de instalación contiene un directorio bin, y el directorio bin contiene los comandos java y javac que está tratando de usar.

A continuación, obtenga el archivo que acaba de editar para que se actualicen las variables de entorno de su shell actual.

$ source ~/.bash_profile

A continuación, repita las comprobaciones de versión java y javac. Si todavía hay problemas, use what java y what javac para verificar que ha actualizado las variables de entorno correctamente.

Finalmente, cierre la sesión y vuelva a iniciarla para que las variables de entorno actualizadas se ptopaguen en todos sus shells. Ahora deberías haber terminado.

Comprobando alternativas

Si java -version o javac -version funcionaron pero dieron un número de versión inesperado, debe verificar de dónde provienen los comandos. Use which y ls -l para averiguarlo de la siguiente manera:

$ ls -l `which java`

Si la salida se ve así:

lrwxrwxrwx. 1 root root 22 Jul 30 22:18 /usr/bin/java -> /etc/alternatives/java

entonces se está utilizando el cambio de versión de alternativas. Debe decidir si continuar usándolo o simplemente anularlo configurando la RUTA directamente.

  • Configuración y cambio de versiones de Java en Linux usando alternativas
  • Consulte “Configuración directa de PATH” más arriba.

¿Dónde se instaló Java?

Java se puede instalar en una variedad de lugares, según el método de instalación.

  • Los RPM de Oracle ponen la instalación de Java en “/usr/java”.
  • En Fedora, la ubicación predeterminada es “/usr/lib/jvm”.
  • Si Java se instaló a mano desde archivos ZIP o JAR, la instalación podría ser en cualquier lugar.

Si tiene dificultades para encontrar el directorio de instalación, le sugerimos que use find (o slocate) para encontrar el comando. Por ejemplo:

$ find / -name java -type f 2> /dev/null

Esto le da los nombres de ruta para todos los archivos llamados java en su sistema. (La redirección del error estándar a “/dev/null” suprime los mensajes sobre archivos y directorios a los que no puede acceder).
