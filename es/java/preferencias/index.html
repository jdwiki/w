Agregar detectores de eventos #

Hay dos tipos de eventos emitidos por un objeto Preferences: PreferenceChangeEvent y NodeChangeEvent.

Evento de cambio de preferencia #

Un PreferenceChangeEvent es emitido por un objeto Properties cada vez que cambia uno de los pares clave-valor del nodo. PreferenceChangeEvents se pueden escuchar con un PreferenceChangeListener:

preferences.addPreferenceChangeListener(evt -> {
    String newValue = evt.getNewValue();
    String changedPreferenceKey = evt.getKey();
    Preferences changedNode = evt.getNode();
});

preferences.addPreferenceChangeListener(new PreferenceChangeListener() {
    @Override
    public void preferenceChange(PreferenceChangeEvent evt) {
        String newValue = evt.getNewValue();
        String changedPreferenceKey = evt.getKey();
        Preferences changedNode = evt.getNode();
    }
});

Este oyente no escuchará los pares clave-valor modificados de nodos secundarios.

Evento de cambio de nodo #

Este evento se activará cada vez que se agregue o elimine un nodo secundario de un nodo Propiedades.

preferences.addNodeChangeListener(new NodeChangeListener() {
    @Override
    public void childAdded(NodeChangeEvent evt) {
        Preferences addedChild = evt.getChild();
        Preferences parentOfAddedChild = evt.getParent();
    }

    @Override
    public void childRemoved(NodeChangeEvent evt) {
        Preferences removedChild = evt.getChild();
        Preferences parentOfRemovedChild = evt.getParent();
    }
});

Obtener subnodos de Preferencias #

Los objetos Preferencias siempre representan un nodo específico en un árbol Preferencias completo, algo así:

/usuarioRaíz
├── com
│   └── miempresa
│   └── miaplicación
│   ├── darkApplicationMode=verdadero
│   ├── showExitConfirmation=false
│   └── ventanaMaximizada=verdadero
└── org
    └── myorganization
        └── anotherapp
            ├── defaultFont=Helvetica
            ├── defaultSavePath=/home/matt/Documents
            └── exporting
                ├── defaultFormat=pdf
                └── openInBrowserAfterExport=false

Para seleccionar el nodo /com/mycompany/myapp:

  1. Por convención, basado en el paquete de una clase:

    package com.mycompany.myapp;

// ...

// Because this class is in the com.mycompany.myapp package, the node
// /com/mycompany/myapp will be returned.
Preferences myApp = Preferences.userNodeForPackage(getClass());

  2. Por ruta relativa:

    Preferences myApp = Preferences.userRoot().node("com/mycompany/myapp");

El uso de una ruta relativa (una ruta que no comience con /) hará que la ruta se resuelva en relación con el nodo principal en el que se resuelve. Por ejemplo, el siguiente ejemplo devolverá el nodo de la ruta /uno/dos/tres/com/miempresa/miaplicación:

   Preferences prefix = Preferences.userRoot().node("one/two/three");
   Preferences myAppWithPrefix = prefix.node("com/mycompany/myapp");
   // prefix          is /one/two/three
   // myAppWithPrefix is /one/two/three/com/mycompany/myapp

  1. Por ruta absoluta:

    Preferences myApp = Preferences.userRoot().node("/com/mycompany/myapp");

Usar una ruta absoluta en el nodo raíz no será diferente de usar una ruta relativa. La diferencia es que, si se invoca en un subnodo, la ruta se resolverá en relación con el nodo raíz.

   Preferences prefix = Preferences.userRoot().node("one/two/three");
   Preferences myAppWitoutPrefix = prefix.node("/com/mycompany/myapp");
   // prefix            is /one/two/three
   // myAppWitoutPrefix is /com/mycompany/myapp

Coordinar el acceso a las preferencias en múltiples instancias de aplicaciones #

Todas las instancias de Preferencias son siempre seguras para subprocesos en los subprocesos de una única máquina virtual Java (JVM). Debido a que las Preferencias se pueden compartir entre varias JVM, existen métodos especiales que se ocupan de sincronizar los cambios entre las máquinas virtuales.

Si tiene una aplicación que se supone que debe ejecutarse en una única instancia solamente, entonces no se requiere sincronización externa.

Si tiene una aplicación que se ejecuta en múltiples instancias en un solo sistema y, por lo tanto, el acceso a Preferencias debe coordinarse entre las JVM en el sistema, entonces el método sync() de cualquier ` El nodo de “Preferencias” se puede usar para garantizar que los cambios en el nodo de “Preferencias” sean visibles para otras JVM en el sistema:

// Warning: don't use this if your application is intended
// to only run a single instance on a machine once
// (this is probably the case for most desktop applications)
try {
    preferences.sync();
} catch (BackingStoreException e) {
    // Deal with any errors while saving the preferences to the backing storage
    e.printStackTrace();
}

Exportando preferencias #

Los nodos Preferencias se pueden exportar a un documento XML que represente ese nodo. El árbol XML resultante se puede importar de nuevo. El documento XML resultante recordará si se exportó desde las “Preferencias” del usuario o del sistema.

Para exportar un solo nodo, pero no sus nodos secundarios:

try (OutputStream os = ...) {
    preferences.exportNode(os);
} catch (IOException ioe) {
    // Exception whilst writing data to the OutputStream
    ioe.printStackTrace();
} catch (BackingStoreException bse) {
    // Exception whilst reading from the backing preferences store
    bse.printStackTrace();
}

OutputStream os = null;
try {
    os = ...;
    preferences.exportSubtree(os);
} catch (IOException ioe) {
    // Exception whilst writing data to the OutputStream
    ioe.printStackTrace();
} catch (BackingStoreException bse) {
    // Exception whilst reading from the backing preferences store
    bse.printStackTrace();
} finally {
    if (os != null) {
        try {
            os.close();
        } catch (IOException ignored) {}
    }
}

Para exportar un solo nodo con sus nodos secundarios:

try (OutputStream os = ...) {
    preferences.exportNode(os);
} catch (IOException ioe) {
    // Exception whilst writing data to the OutputStream
    ioe.printStackTrace();
} catch (BackingStoreException bse) {
    // Exception whilst reading from the backing preferences store
    bse.printStackTrace();
}

OutputStream os = null;
try {
    os = ...;
    preferences.exportSubtree(os);
} catch (IOException ioe) {
    // Exception whilst writing data to the OutputStream
    ioe.printStackTrace();
} catch (BackingStoreException bse) {
    // Exception whilst reading from the backing preferences store
    bse.printStackTrace();
} finally {
    if (os != null) {
        try {
            os.close();
        } catch (IOException ignored) {}
    }
}

Preferencias de importación #

Los nodos Preferencias se pueden importar desde un documento XML. La importación está pensada para usarse junto con la funcionalidad de exportación de Preferencias, ya que crea los documentos XML correctos correspondientes.

Los documentos XML recordarán si fueron exportados desde las Preferencias del usuario o del sistema. Por lo tanto, se pueden importar nuevamente a sus respectivos árboles de Preferencias, sin que tenga que averiguar o saber de dónde provienen. La función estática averiguará automáticamente si el documento XML se exportó desde las “Preferencias” del usuario o del sistema y los importará automáticamente al árbol desde el que se exportaron.

try (InputStream is = ...) {
    // This is a static call on the Preferences class
    Preferences.importPreferences(is);
} catch (IOException ioe) {
    // Exception whilst reading data from the InputStream
    ioe.printStackTrace();
} catch (InvalidPreferencesFormatException ipfe) {
    // Exception whilst parsing the XML document tree
    ipfe.printStackTrace();
}

InputStream is = null;
try {
    is = ...;
    // This is a static call on the Preferences class
    Preferences.importPreferences(is);
} catch (IOException ioe) {
    // Exception whilst reading data from the InputStream
    ioe.printStackTrace();
} catch (InvalidPreferencesFormatException ipfe) {
    // Exception whilst parsing the XML document tree
    ipfe.printStackTrace();
} finally {
    if (is != null) {
        try {
            is.close();
        } catch (IOException ignored) {}
    }
}

Eliminando detectores de eventos #

Los detectores de eventos se pueden eliminar de nuevo de cualquier nodo Properties, pero la instancia del detector debe conservarse para eso.

Preferences preferences = Preferences.userNodeForPackage(getClass());

PreferenceChangeListener listener = evt -> {
    System.out.println(evt.getKey() + " got new value " + evt.getNewValue());
};
preferences.addPreferenceChangeListener(listener);

//
// later...
//

preferences.removePreferenceChangeListener(listener);

Preferences preferences = Preferences.userNodeForPackage(getClass());

PreferenceChangeListener listener = new PreferenceChangeListener() {
    @Override
    public void preferenceChange(PreferenceChangeEvent evt) {
        System.out.println(evt.getKey() + " got new value " + evt.getNewValue());
    }
};
preferences.addPreferenceChangeListener(listener);

//
// later...
//

preferences.removePreferenceChangeListener(listener);

Lo mismo aplica para NodeChangeListener.

Obtener valores de preferencias #

Un valor de un nodo Preferencias puede ser del tipo String, boolean, byte[], double, float, int o long. Todas las invocaciones deben proporcionar un valor predeterminado, en caso de que el valor especificado no esté presente en el nodo Preferencias.

Preferences preferences = Preferences.userNodeForPackage(getClass());

String someString = preferences.get("someKey", "this is the default value");
boolean someBoolean = preferences.getBoolean("someKey", true);
byte[] someByteArray = preferences.getByteArray("someKey", new byte[0]);
double someDouble = preferences.getDouble("someKey", 887284.4d);
float someFloat = preferences.getFloat("someKey", 38723.3f);
int someInt = preferences.getInt("someKey", 13232);
long someLong = preferences.getLong("someKey", 2827637868234L);

Establecer valores de preferencias #

Para almacenar un valor en el nodo Preferencias, se usa uno de los métodos putXXX(). Un valor de un nodo Preferencias puede ser del tipo String, boolean, byte[], double, float, int o long.

Preferences preferences = Preferences.userNodeForPackage(getClass());

preferences.put("someKey", "some String value");
preferences.putBoolean("someKey", false);
preferences.putByteArray("someKey", new byte[0]);
preferences.putDouble("someKey", 187398123.4454d);
preferences.putFloat("someKey", 298321.445f);
preferences.putInt("someKey", 77637);
preferences.putLong("someKey", 2873984729834L);

Uso de preferencias #

Las Preferencias se pueden utilizar para almacenar la configuración del usuario que refleja la configuración de la aplicación personal de un usuario, p. la fuente de su editor, si prefieren que la aplicación se inicie en modo de pantalla completa, si marcaron la casilla de verificación “no mostrar esto de nuevo” y cosas por el estilo.

public class ExitConfirmer {
    private static boolean confirmExit() {
        Preferences preferences = Preferences.userNodeForPackage(ExitConfirmer.class);
        boolean doShowDialog = preferences.getBoolean("showExitConfirmation", true); // true is default value

        if (!doShowDialog) {
            return true;
        }

        //
        // Show a dialog here...
        //
        boolean exitWasConfirmed = ...; // whether the user clicked OK or Cancel
        boolean doNotShowAgain = ...; // get value from "Do not show again" checkbox

        if (exitWasConfirmed && doNotShowAgain) {
            // Exit was confirmed and the user chose that the dialog should not be shown again
            // Save these settings to the Preferences object so the dialog will not show again next time
            preferences.putBoolean("showExitConfirmation", false);
        }

        return exitWasConfirmed;
    }

    public static void exit() {
        if (confirmExit()) {
            System.exit(0);
        }
    }
}
