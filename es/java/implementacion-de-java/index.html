Existe una variedad de tecnologías para “empaquetar” aplicaciones Java, aplicaciones web, etc., para su implementación en la plataforma en la que se ejecutarán. Van desde bibliotecas simples o archivos JAR ejecutables, archivos WAR y EAR, hasta instaladores y ejecutables autónomos.

En el nivel más fundamental, un programa Java se puede implementar copiando una clase compilada (es decir, un archivo “.class”) o un árbol de directorios que contiene clases compiladas. Sin embargo, Java normalmente se implementa de una de las siguientes maneras:

  • Copiando un archivo JAR o colección de archivos JAR al sistema donde serán ejecutados; p.ej. usando javac.

  • Copiando o cargando un archivo WAR, EAR o similar a un “contenedor de servlet” o “servidor de aplicaciones”.

  • Ejecutando algún tipo de instalador de aplicaciones que automatice lo anterior. El instalador también puede instalar un JRE incorporado.

  • Colocando los archivos JAR de la aplicación en un servidor web para permitir que se inicien mediante Java WebStart.

El ejemplo de creación de archivos JAR, WAR y EAR resume las diferentes formas de crear estos archivos.

Existen numerosos “generadores de instaladores” comerciales y de código abierto y Herramientas de “generador EXE” para Java. Del mismo modo, existen herramientas para ofuscar archivos de clase Java (para dificultar la ingeniería inversa) y para agregar verificación de licencias en tiempo de ejecución. Todos estos están fuera del alcance de la documentación del “Lenguaje de programación Java”.

Hacer un JAR ejecutable desde la línea de comandos #

Para hacer un jar, necesita uno o más archivos de clase. Esto debería tener un método principal si se va a ejecutar con un doble clic.

Para este ejemplo, usaremos:

import javax.swing.*;
import java.awt.Container;

public class HelloWorld {

    public static void main(String[] args) {
        JFrame f = new JFrame("Hello, World"); 
        JLabel label = new JLabel("Hello, World");
        Container cont = f.getContentPane();
        cont.add(label);
        f.setSize(400,100); 
        f.setVisible(true);
        f.setDefaultCloseOperation(JFrame.EXIT_ON_CLOSE);
    }

}

Ha sido nombrado HelloWorld.java

A continuación, queremos compilar este programa.

Puede usar cualquier programa que desee para hacer esto. Para ejecutar desde la línea de comandos, consulte la documentación sobre cómo compilar y ejecutar su primer programa Java.

Una vez que tenga HelloWorld.class, cree una nueva carpeta y llámela como quiera.

Cree otro archivo llamado manifest.txt y péguelo en él

Main-Class: HelloWorld
Class-Path: HelloWorld.jar

Póngalo en la misma carpeta con HelloWorld.class
Utilice la línea de comando para convertir su directorio actual (cd C:\Su\Carpeta\Ruta\Aquí en Windows) en su carpeta.

Use Terminal y cambie el directorio al directorio (cd /Users/user/Documents/Java/jarfolder en Mac) su carpeta

Cuando termine, escriba jar -cvfm HelloWorld.jar manifest.txt HelloWorld.class y presione enter. Esto crea un archivo jar (en la carpeta con su manifiesto y HelloWorld.class) utilizando los archivos .class especificados y denominados HelloWorld.jar. Consulte la sección Sintaxis para obtener información sobre las opciones (como -m y -v).
Después de estos pasos, vaya a su directorio con el archivo de manifiesto y debería encontrar HelloWorld.jar
Al hacer clic en él, debería mostrarse Hello, World en un cuadro de texto.

Creación de archivos JAR, WAR y EAR #

Los tipos de archivos JAR, WAR y EAR son fundamentalmente archivos ZIP con un archivo “manifiesto” y (para los archivos WAR y EAR) una estructura interna de directorio/archivo particular.

La forma recomendada de crear estos archivos es usar una herramienta de compilación específica de Java que “entienda” los requisitos para los tipos de archivos respectivos. Si no utiliza una herramienta de compilación, entonces la “exportación” IDE es la siguiente opción para probar.

(Nota editorial: las descripciones de cómo crear estos archivos están mejor ubicadas en la documentación de las herramientas respectivas. Póngalos allí. ¡Muestre algo de moderación y NO los calce en este ejemplo!)

Creando archivos JAR y WAR usando Maven #

Crear un JAR o WAR usando Maven es simplemente una cuestión de colocar el elemento <empaquetado> correcto en el archivo POM; p.ej,

<packaging>jar</packaging>

o

<packaging>war</packaging>

Para más detalles. Maven se puede configurar para crear archivos JAR “ejecutables” agregando la información necesaria sobre la clase de punto de entrada y las dependencias externas como propiedades del complemento para el complemento maven jar. Introducción a Java Web Start #

Los tutoriales de Oracle Java resumen Web Start de la siguiente manera:

El software Java Web Start brinda el poder de iniciar aplicaciones completas con un solo clic. Los usuarios pueden descargar e iniciar aplicaciones, como un programa de hoja de cálculo completo o un cliente de chat de Internet, sin pasar por largos procedimientos de instalación.

Otras ventajas de Java Web Start son la compatibilidad con el código firmado y la declaración explícita de las dependencias de la plataforma, y ​​la compatibilidad con el almacenamiento en caché del código y la implementación de actualizaciones de aplicaciones.

Java Web Start también se conoce como JavaWS y JAWS. Las principales fuentes de información son:

  • Guía de inicio web de Java
  • Preguntas frecuentes sobre Java Web Start
  • Especificación JNLP
  • Documentación de la API javax.jnlp

requisitos previos #

A un alto nivel, Web Start funciona mediante la distribución de aplicaciones Java empaquetadas como archivos JAR desde un servidor web remoto. Los requisitos previos son:

  • Una instalación de Java preexistente (JRE o JDK) en la máquina de destino donde se ejecutará la aplicación. Se requiere Java 1.2.2 o superior:

    • From Java 5.0 onwards, Web Start support is included in the JRE / JDK.
    • For earlier releases, Web Start support is installed separately.
    • The Web Start infrastructure includes some Javascript that can be included in a web page to assist the user to install the necessary software.

  • El servidor web que aloja el software debe ser accesible para la máquina de destino.

  • Si el usuario va a iniciar una aplicación Web Start usando un enlace en una página web, entonces: - they need a compatible web browser, and - for modern (secure) browsers, they need to be told how to tell the browser to allow Java to run … without compromising web browser security.

Un archivo JNLP de ejemplo #

El siguiente ejemplo pretende ilustrar la funcionalidad básica de JNLP.

<?xml version="1.0" encoding="UTF-8"?>
<jnlp spec="1.0+" codebase="https://www.example.com/demo" 
    href="demo_webstart.jnlp">
    <information>
        <title>Demo</title>
        <vendor>The Example.com Team</vendor>
    </information>
    <resources>
        <!-- Application Resources -->
        <j2se version="1.7+" href="http://java.sun.com/products/autodl/j2se"/>
        <jar href="Demo.jar" main="true"/>
    </resources>
    <application-desc
         name="Demo Application"
         main-class="com.example.jwsdemo.Main"
         width="300"
         height="300">
     </application-desc>
     <update check="background"/>
</jnlp>

Como puede ver, un archivo JNLP basado en XML y toda la información está contenida en el elemento <jnlp>.

  • El atributo spec proporciona la versión de la especificación JNPL a la que se ajusta este archivo.
  • El atributo codebase proporciona la URL base para resolver las URL href relativas en el resto del archivo.
  • El atributo href proporciona la URL definitiva para este archivo JNLP.
  • El elemento <información> contiene metadatos de la aplicación, incluidos su título, autores, descripción y sitio web de ayuda.
  • El elemento <resources> describe las dependencias de la aplicación, incluida la versión de Java requerida, la plataforma del sistema operativo y los archivos JAR.
  • El elemento <application-desc> (o <applet-desc>) proporciona la información necesaria para iniciar la aplicación.

Configurando el servidor web #

El servidor web debe estar configurado para usar application/x-java-jnlp-file como tipo MIME para archivos .jnlp.

El archivo JNLP y los archivos JAR de la aplicación deben estar instalados en el servidor web para que estén disponibles utilizando las URL indicadas por el archivo JNLP.

Habilitación del lanzamiento a través de una página web #

Si la aplicación se va a iniciar a través de un enlace web, la página que contiene el enlace debe crearse en el servidor web.

  • Si puede asumir que Java Web Start ya está instalado en la máquina del usuario, entonces la página web simplemente debe contener un enlace para iniciar la aplicación. Por ejemplo.

     <a href="https://www.example.com/demo_webstart.jnlp">Launch the application</a>

  • De lo contrario, la página también debe incluir algunas secuencias de comandos para detectar el tipo de navegador que está utilizando el usuario y solicitar la descarga e instalación de la versión requerida de Java.

NOTA: Es una mala idea alentar a los usuarios a instalar Java de esta manera, o incluso habilitar Java en sus navegadores web para que funcione el inicio de la página web de JNLP.

Inicio de aplicaciones Web Start desde la línea de comandos

Las instrucciones para iniciar una aplicación Web Start desde la línea de comandos son sencillas. Suponiendo que el usuario tiene instalado Java 5.0 JRE o JDK, simplemente necesita ejecutar esto:

$ javaws <url>

donde <url> es la URL del archivo JNLP en el servidor remoto.
