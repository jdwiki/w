Sintaxis #

  • javap [opciones] <clases>

Parámetros #

Nombre Descripción
<clases> Lista de clases para desmontar. Puede estar en formato paquete1.paquete2.Nombre de clase, o formato paquete1/paquete2/Nombre de clase. No incluya la extensión .class.
-ayuda, --ayuda, -? Imprimir este mensaje de uso
-versión Información de la versión
-v, -detallado Imprimir información adicional
-l Imprimir tablas de números de línea y variables locales
-público Mostrar solo clases públicas y miembros
-protegido Mostrar clases y miembros protegidos/públicos
-paquete Mostrar clases y miembros del paquete/protegido/público (predeterminado)
-p, -privado Mostrar todas las clases y miembros
-c Desensamblar el código
-s Imprimir signaturas tipo internas
-info del sistema Mostrar información del sistema (ruta, tamaño, fecha, hash MD5) de la clase que se está procesando
-constantes Mostrar constantes finales
-classpath <ruta> Especifique dónde encontrar archivos de clase de usuario
-cp <ruta> Especifique dónde encontrar archivos de clase de usuario
-bootclasspath <ruta> Anular la ubicación de los archivos de clase de arranque

Ver código de bytes con javap #

Si desea ver el código de bytes generado para un programa Java, puede usar el comando javap provisto para verlo.

Suponiendo que tenemos el siguiente archivo fuente de Java:

package com.stackoverflow.documentation;

import org.springframework.stereotype.Service;

import java.io.IOException;
import java.io.InputStream;
import java.util.List;

@Service
public class HelloWorldService {

    public void sayHello() {
        System.out.println("Hello, World!");
    }

    private Object[] pvtMethod(List<String> strings) {
        return new Object[]{strings};
    }

    protected String tryCatchResources(String filename) throws IOException {
        try (InputStream inputStream = getClass().getResourceAsStream(filename)) {
            byte[] bytes = new byte[8192];
            int read = inputStream.read(bytes);
            return new String(bytes, 0, read);
        } catch (IOException | RuntimeException e) {
            e.printStackTrace();
            throw e;
        }
    }

    void stuff() {
        System.out.println("stuff");
    }
}

Después de compilar el archivo fuente, el uso más simple es:

cd <directory containing classes> (e.g. target/classes)
javap com/stackoverflow/documentation/SpringExample

Que produce la salida

Compiled from "HelloWorldService.java"
public class com.stackoverflow.documentation.HelloWorldService {
  public com.stackoverflow.documentation.HelloWorldService();

  void stuff();
}

Esto enumera todos los métodos no privados en la clase, pero eso no es particularmente útil para la mayoría de los propósitos. El siguiente comando es mucho más útil:

javap -p -c -s -constants -l -v com/stackoverflow/documentation/HelloWorldService
