La recursividad ocurre cuando un método se llama a sí mismo. Tal método se llama recursivo. Un método recursivo puede ser más conciso que un enfoque no recursivo equivalente. Sin embargo, para la recursividad profunda, a veces una solución iterativa puede consumir menos espacio de pila finito de un subproceso.

Este tema incluye ejemplos de recursividad en Java.

Diseño de un método recursivo #

Al diseñar un método recursivo, tenga en cuenta que necesita:

  • Caso base. Esto definirá cuándo se detendrá su recursividad y generará el resultado. El caso base en el ejemplo factorial es:

     if (n <= 1) {
     return 1;
 }

  • Llamada recursiva. En esta declaración, vuelve a llamar al método con un parámetro modificado. La llamada recursiva en el ejemplo factorial anterior es:

     else {
     return n * factorial(n - 1);
 }

Producción #

En este ejemplo, calcula el n-ésimo número factorial. Los primeros factoriales son:

0! = 1

1! = 1

2! = 1 x 2 = 2

3! = 1x2x3 = 6

4! = 1x2x3x4 = 24

…

Eliminación de Java y Tail-call #

Los compiladores de Java actuales (hasta Java 9 inclusive) no realizan la eliminación de llamadas finales. Esto puede afectar el rendimiento de los algoritmos recursivos, y si la recursividad es lo suficientemente profunda, puede provocar bloqueos de StackOverflowError; consulte https://www.wikiod.com/es/java/recursividad#La recursividad profunda es problemática en Java

La recursividad profunda es problemática en Java #

Considere el siguiente método ingenuo para sumar dos números positivos usando recursividad:

public static int add(int a, int b) {
    if (a == 0) {
        return b;
    } else {
        return add(a - 1, b + 1);  // TAIL CALL
    }
}

Esto es algorítmicamente correcto, pero tiene un problema importante. Si llama a add con una a grande, fallará con un StackOverflowError, en cualquier versión de Java hasta (al menos) Java 9.

En un lenguaje de programación funcional típico (y en muchos otros lenguajes), el compilador optimiza [recurrencia de cola][1]. El compilador notaría que la llamada a add (en la línea etiquetada) es una [llamada final][2], y reescribiría efectivamente la recursividad como un bucle. Esta transformación se llama eliminación de llamada de cola.

Sin embargo, los compiladores de Java de la generación actual no realizan la eliminación de llamadas finales. (Esto no es un simple descuido. Hay razones técnicas sustanciales para esto; ver más abajo). En cambio, cada llamada recursiva de add hace que se asigne un nuevo marco en la pila del subproceso. Por ejemplo, si llama a add(1000, 1), se necesitarán 1000 llamadas recursivas para llegar a la respuesta 1001.

El problema es que el tamaño de la pila de subprocesos de Java se fija cuando se crea el subproceso. (Esto incluye el subproceso “principal” en un programa de un solo subproceso). Si se asignan demasiados marcos de pila, la pila se desbordará. La JVM detectará esto y lanzará un StackOverflowError.

Un enfoque para lidiar con esto es simplemente usar una pila más grande. Hay opciones de JVM que controlan el tamaño predeterminado de una pila, y también puede especificar el tamaño de la pila como un parámetro de constructor Thread. Desafortunadamente, esto solo “aplaza” el desbordamiento de la pila. Si necesita hacer un cálculo que requiere una pila aún más grande, entonces regresa StackOverflowError.

La solución real es identificar algoritmos recursivos en los que es probable una recursividad profunda y manualmente realizar la optimización de llamada final en el nivel del código fuente. Por ejemplo, nuestro método add se puede reescribir de la siguiente manera:

public static int add(int a, int b) {
    while (a != 0) {
       a = a - 1;
       b = b + 1;
    }
    return b;
}

(Obviamente, hay mejores formas de sumar dos números enteros. Lo anterior es simplemente para ilustrar el efecto de la eliminación manual de llamadas de cola).

Por qué la eliminación de llamadas de cola no está implementada en Java (todavía) #

Hay una serie de razones por las que agregar la eliminación de llamadas de cola a Java no es fácil. Por ejemplo:

  • Algunos códigos podrían basarse en StackOverflowError para (por ejemplo) colocar un límite en el tamaño de un problema computacional.
  • Los administradores de seguridad de Sandbox a menudo se basan en el análisis de la pila de llamadas cuando deciden si permitir que el código sin privilegios realice una acción privilegiada.

Como explica John Rose en [“Llamadas de cola en la máquina virtual”][3]:

“Los efectos de eliminar el marco de pila de la persona que llama son visibles para algunas API, en particular, las comprobaciones de control de acceso y el seguimiento de la pila. Es como si la persona que llama hubiera llamado directamente a la persona que llama. Cualquier privilegio que posea la persona que llama se descarta después de que se transfiere el control al destinatario. Sin embargo, el enlace y la accesibilidad del método del destinatario se calculan antes de la transferencia de control y tienen en cuenta la persona que realiza la llamada final”.

En otras palabras, la eliminación de la cola de llamadas podría hacer que un método de control de acceso pensara erróneamente que un código confiable estaba llamando a una API sensible a la seguridad.

La idea básica de la recursividad #

Qué es la recursividad:

En general, la recursión es cuando una función se invoca a sí misma, ya sea directa o indirectamente. Por ejemplo:

// This method calls itself "infinitely"
public void useless() {
    useless();  // method calls itself (directly)
}

Condiciones para aplicar la recursividad a un problema:

Hay dos condiciones previas para usar funciones recursivas para resolver un problema específico:

  1. Debe haber una condición base para el problema, que será el punto final de la recursividad. Cuando una función recursiva alcanza la condición base, no realiza más llamadas recursivas (más profundas).

  2. Cada nivel de recursividad debe intentar un problema más pequeño. La función recursiva divide así el problema en partes cada vez más pequeñas. Asumiendo que el problema es finito, esto asegurará que la recursividad termine.

En Java hay una tercera condición previa: no debería ser necesario recurrir demasiado para resolver el problema; consulte https://www.wikiod.com/es/java/recursividad#La recursividad profunda es problemática en Java

Ejemplo

La siguiente función calcula factoriales usando recursividad. Observe cómo el método factorial se llama a sí mismo dentro de la función. Cada vez que se llama a sí misma, reduce el parámetro n en 1. Cuando n llega a 1 (la condición base), la función no repetirá más.

public int factorial(int n) {
    if (n <= 1) { // the base condition 
        return 1;
    } else {
        return n * factorial(n - 1);
    }
}

Esta no es una forma práctica de calcular factoriales en Java, ya que no tiene en cuenta el desbordamiento de enteros ni el desbordamiento de la pila de llamadas (es decir, las excepciones StackOverflowError) para valores grandes de n.

Tipos de recursividad #

La recursividad se puede categorizar como Recursión de cabeza o Recursión de cola, dependiendo de dónde se coloque la llamada al método recursivo.

En recursión principal, la llamada recursiva, cuando ocurre, viene antes de otro procesamiento en la función (piense que ocurre en la parte superior, o cabeza, de la función).

En recursión de cola, es lo contrario: el procesamiento ocurre antes de la llamada recursiva. Elegir entre los dos estilos recursivos puede parecer arbitrario, pero la elección puede marcar la diferencia.

Una función con una ruta con una sola llamada recursiva al comienzo de la ruta usa lo que se llama recursividad principal. La función factorial de una exposición anterior utiliza recursividad principal. Lo primero que hace una vez que determina que se necesita la recursividad es llamarse a sí mismo con el parámetro decrementado. Una función con una sola llamada recursiva al final de una ruta está usando recursividad de cola.

public void tail(int n)              public void head(int n)
{                                       {
   if(n == 1)                             if(n == 0)
      return;                                return;
   else                                   else
      System.out.println(n);                 head(n-1);

   tail(n-1);                              System.out.println(n);
}                                        }

Si la llamada recursiva se produce al final de un método, se denomina “recursión de cola”. La recursión de la cola es similar a un bucle. El método ejecuta todas las sentencias antes de pasar a la siguiente llamada recursiva.

Si la llamada recursiva se produce al comienzo de un método, se denomina recursión principal. El método guarda el estado antes de pasar a la siguiente llamada recursiva.

Referencia: http://stackoverflow.com/questions/21426688/the-difference- between-head-tail-recursion

Cálculo del enésimo número de Fibonacci #

El siguiente método calcula el N-ésimo número de Fibonacci usando recursividad.

public int fib(final int n) {
    if (n > 2) {
        return fib(n - 2) + fib(n - 1);
    }
    return 1;
}

El método implementa un caso base (n <= 2) y un caso recursivo (n>2). Esto ilustra el uso de la recursividad para calcular una relación recursiva.

Sin embargo, si bien este ejemplo es ilustrativo, también es ineficiente: cada instancia individual del método llamará a la función dos veces, lo que lleva a un crecimiento exponencial en el número de veces que se llama a la función a medida que aumenta N. La función anterior es O(2N), pero una solución iterativa equivalente tiene una complejidad O(N). Además, hay una expresión de “forma cerrada” que se puede evaluar en multiplicaciones de punto flotante O (N).

Cálculo de la N-ésima potencia de un número #

El siguiente método calcula el valor de num elevado a la potencia de exp usando recursividad:

public long power(final int num, final int exp) {
    if (exp == 0) {
        return 1;
    }
    if (exp == 1) {
        return num;
    }
    return num * power(num, exp - 1);
}

Esto ilustra los principios mencionados anteriormente: el método recursivo implementa un caso base (dos casos, n = 0 y n = 1) que finaliza la recursividad y un caso recursivo que vuelve a llamar al método. Este método es O(N) y se puede reducir a un ciclo simple mediante la optimización de llamada final.

Atravesando una estructura de datos de árbol con recursividad #

Considere la clase de Nodo que tiene datos de 3 miembros, puntero secundario izquierdo y puntero secundario derecho como se muestra a continuación.

public class Node {
    public int data;
    public Node left;
    public Node right;
    
    public Node(int data){
        this.data = data;
    }
}

Podemos atravesar el árbol construido conectando múltiples objetos de la clase Node como se muestra a continuación, el recorrido se denomina recorrido en orden del árbol.

public static void inOrderTraversal(Node root) {
        if (root != null) {          
            inOrderTraversal(root.left); // traverse left sub tree
            System.out.print(root.data + " "); // traverse current node
            inOrderTraversal(root.right); // traverse right sub tree
        }
    }

Como se demostró anteriormente, usando recursión podemos atravesar la estructura de datos de árbol sin usar ninguna otra estructura de datos que no sea posible con el enfoque iterativo.

StackOverflowError y recursión al bucle #

Si una llamada recursiva va “demasiado profunda”, esto da como resultado un StackOverflowError. Java asigna un nuevo marco para cada llamada de método en la pila de su hilo. Sin embargo, el espacio de la pila de cada subproceso es limitado. Demasiados fotogramas en la pila conducen al desbordamiento de la pila (SO).

Ejemplo #

public static void recursion(int depth) {
    if (depth > 0) {
        recursion(depth-1);
    }
}

Llamar a este método con parámetros grandes (por ejemplo, recursion(50000) probablemente resultará en un desbordamiento de la pila. El valor exacto depende del tamaño de la pila del subproceso, que a su vez depende de la construcción del subproceso, parámetros de la línea de comandos como -Xss , o el tamaño predeterminado para la JVM.

Solución alterna #

Una recursividad se puede convertir en un bucle almacenando los datos de cada llamada recursiva en una estructura de datos. Esta estructura de datos se puede almacenar en el montón en lugar de en la pila de subprocesos.

En general, los datos necesarios para restaurar el estado de la invocación de un método se pueden almacenar en una pila y se puede usar un bucle while para “simular” las llamadas recursivas. Los datos que pueden ser requeridos incluyen:

  • el objeto para el que se solicitó el método (solo métodos de instancia)
  • los parámetros del método
  • variables locales
  • la posición actual en la ejecución o el método

Ejemplo #

La siguiente clase permite la impresión recursiva de una estructura de árbol hasta una profundidad específica.

public class Node {

    public int data;
    public Node left;
    public Node right;

    public Node(int data) {
        this(data, null, null);
    }

    public Node(int data, Node left, Node right) {
        this.data = data;
        this.left = left;
        this.right = right;
    }

    public void print(final int maxDepth) {
        if (maxDepth <= 0) {
            System.out.print("(...)");
        } else {
            System.out.print("(");
            if (left != null) {
                left.print(maxDepth-1);
            }
            System.out.print(data);
            if (right != null) {
                right.print(maxDepth-1);
            }
            System.out.print(")");
        }
    }

}

p.ej.

Node n = new Node(10, new Node(20, new Node(50), new Node(1)), new Node(30, new Node(42), null));
n.print(2);
System.out.println();

Huellas dactilares

(((...)20(...))10((...)30))

Esto podría convertirse en el siguiente bucle:

public class Frame {

    public final Node node;

    // 0: before printing anything
    // 1: before printing data
    // 2: before printing ")"
    public int state = 0;
    public final int maxDepth;

    public Frame(Node node, int maxDepth) {
        this.node = node;
        this.maxDepth = maxDepth;
    }

}

List<Frame> stack = new ArrayList<>();
stack.add(new Frame(n, 2)); // first frame = initial call

while (!stack.isEmpty()) {
    // get topmost stack element
    int index = stack.size() - 1;
    Frame frame = stack.get(index); // get topmost frame
    if (frame.maxDepth <= 0) {
        // termial case (too deep)
        System.out.print("(...)");
        stack.remove(index); // drop frame
    } else {
        switch (frame.state) {
            case 0:
                frame.state++;

                // do everything done before the first recursive call
                System.out.print("(");
                if (frame.node.left != null) {
                    // add new frame (recursive call to left and stop)
                    stack.add(new Frame(frame.node.left, frame.maxDepth - 1));
                    break;
                }
            case 1:
                frame.state++;

                // do everything done before the second recursive call
                System.out.print(frame.node.data);
                if (frame.node.right != null) {
                    // add new frame (recursive call to right and stop)
                    stack.add(new Frame(frame.node.right, frame.maxDepth - 1));
                    break;
                }
            case 2:
                // do everything after the second recursive call & drop frame
                System.out.print(")");
                stack.remove(index);
        }
    }
}
System.out.println();

Nota: Este es solo un ejemplo del enfoque general. A menudo, puede encontrar una forma mucho mejor de representar un marco y/o almacenar los datos del marco.

Cálculo de la suma de números enteros del 1 al N #

El siguiente método calcula la suma de los números enteros de 0 a N mediante la recursividad.

public int sum(final int n) {
    if (n > 0) {
        return n + sum(n - 1);
    } else {
        return n;
    }
}

Este método es O(N) y se puede reducir a un ciclo simple mediante la optimización de llamada final. De hecho, hay una expresión de forma cerrada que calcula la suma en operaciones O(1).
