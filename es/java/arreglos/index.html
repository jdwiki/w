Las matrices permiten el almacenamiento y la recuperación de una cantidad arbitraria de valores. Son análogos a los vectores en matemáticas. Los arreglos de arreglos son análogos a las matrices y actúan como arreglos multidimensionales. Las matrices pueden almacenar datos de cualquier tipo: primitivos como int o tipos de referencia como Object.

Sintaxis #

  • ArrayType[] myArray; // Declaración de matrices
  • ArrayType myArray[]; // Otra sintaxis válida (menos utilizada y desaconsejada)
  • ArrayType[][][] myArray; // Declarar matrices irregulares multidimensionales (repetir []s)
  • ArrayType myVar = myArray[index]; // Acceso (lectura) al elemento en el índice
  • myArray[index] = value; // Asignar valor a la posición index de la matriz
  • ArrayType[] myArray = new ArrayType[arrayLength]; // Sintaxis de inicialización de matriz
  • int[] ints = {1, 2, 3}; // Sintaxis de inicialización de matriz con los valores proporcionados, la longitud se deduce del número de valores proporcionados: {[value1[, value2]*]}
  • new int[]{4, -5, 6} // Puede usarse como argumento, sin una variable local
  • int[] ints = new int[3]; // igual que {0, 0, 0}
  • int[][] ints = {{1, 2}, {3}, null}; // Inicialización de matriz multidimensional. int[] extiende Object (y también anyType[]), por lo que null es un valor válido.

Parámetros #

Parámetro Detalles
Tipo de matriz Tipo de matriz. Esto puede ser primitivo (int, long, byte) u Objetos (String, MyObject, etc.).

Introducción #

Una matriz es una estructura de datos que contiene un número fijo de valores primitivos o referencias a instancias de objetos.

Cada elemento de una matriz se denomina elemento y se accede a cada elemento por su índice numérico. La longitud de una matriz se establece cuando se crea la matriz:

int size = 42;
int[] array = new int[size];

El tamaño de una matriz se fija en tiempo de ejecución cuando se inicializa. No se puede cambiar después de la inicialización. Si el tamaño debe ser mutable en tiempo de ejecución, se debe usar una clase [Collection][so-docs-collections] como ArrayList en su lugar. ArrayList almacena elementos en una matriz y admite [cambiar el tamaño asignando una nueva matriz](https://www.wikiod.com/es/java/arreglos#¿Cómo se cambia el tamaño de una matriz? ) y copiando elementos de la matriz anterior.

Si la matriz es de un tipo primitivo, es decir,

int[] array1 = { 1,2,3 };
int[] array2 = new int[10];

los valores se almacenan en la propia matriz. En ausencia de un inicializador (como en array2 arriba), el valor predeterminado asignado a cada elemento es 0 (cero).

Si el tipo de matriz es una referencia de objeto, como en

SomeClassOrInterface[] array = new SomeClassOrInterface[10];

entonces la matriz contiene referencias a objetos de tipo AlgunaClaseOInterfaz. Esas referencias pueden referirse a una instancia de SomeClassOrInterface o cualquier subclase (para clases) o clase de implementación (para interfaces) de SomeClassOrInterface. Si la declaración de matriz no tiene inicializador, el valor predeterminado de null se asigna a cada elemento.

Debido a que todas las matrices están indexadas por int, el tamaño de una matriz debe especificarse mediante un int. El tamaño de la matriz no se puede especificar como largo:

long size = 23L;
int[] array = new int[size]; // Compile-time error:
                             // incompatible types: possible lossy conversion from
                             // long to int

Las matrices utilizan un sistema de índice basado en cero, lo que significa que la indexación comienza en 0 y termina en longitud - 1.

Por ejemplo, la siguiente imagen representa una matriz con tamaño 10. Aquí, el primer elemento está en el índice ‘0’ y el último elemento está en el índice ‘9’, en lugar de que el primer elemento esté en el índice ‘1’ y el último elemento en el índice ‘10’ (ver figura a continuación).

Los accesos a los elementos de los arreglos se realizan en tiempo constante. Eso significa que acceder al primer elemento de la matriz tiene el mismo costo (en tiempo) que acceder al segundo elemento, al tercer elemento y así sucesivamente.

Java ofrece varias formas de definir e inicializar matrices, incluidas las notaciones literal y constructor. Al declarar matrices utilizando el constructor nuevo Tipo[longitud], cada elemento se inicializará con los siguientes valores predeterminados:

  • 0 para tipos numéricos primitivos: byte, short, int, long, float y double.
  • '\u0000' (carácter nulo) para el tipo char.
  • falso para el tipo booleano.
  • null para [tipos de referencia][docs-ref-types].

Crear e inicializar arreglos de tipos primitivos #

int[] array1 = new int[] { 1, 2, 3 }; // Create an array with new operator and 
                                      // array initializer.
int[] array2 = { 1, 2, 3 };           // Shortcut syntax with array initializer.
int[] array3 = new int[3];            // Equivalent to { 0, 0, 0 }
int[] array4 = null;                  // The array itself is an object, so it
                                      // can be set as null.

Al declarar una matriz, [] aparecerá como parte del tipo al principio de la declaración (después del nombre del tipo), o como parte del declarador de una variable en particular (después del nombre de la variable), o ambos:

int array5[];       /* equivalent to */  int[] array5;
int a, b[], c[][];  /* equivalent to */  int a; int[] b; int[][] c;
int[] a, b[];       /* equivalent to */  int[] a; int[][] b;
int a, []b, c[][];  /* Compilation Error, because [] is not part of the type at beginning
                       of the declaration, rather it is before 'b'. */    
// The same rules apply when declaring a method that returns an array:
int foo()[] { ... } /* equivalent to */  int[] foo() { ... }

En el siguiente ejemplo, ambas declaraciones son correctas y se pueden compilar y ejecutar sin problemas. Sin embargo, tanto la Convención de codificación de Java como la [Guía de estilo de Java de Google][guía de estilo de Google-docs] desaconsejan el formulario con corchetes después del nombre de la variable: los corchetes identifican el tipo de matriz y deben aparecer con la designación de tipo. Lo mismo debe usarse para las firmas de retorno de método.

float array[]; /* and */ int foo()[] { ... } /* are discouraged */
float[] array; /* and */ int[] foo() { ... } /* are encouraged */

El tipo desaconsejado es [destinado a adaptarse a los usuarios de C en transición] [so-qa-arraysyntax], que están familiarizados con la sintaxis de C que tiene corchetes después del nombre de la variable.

En Java, es posible tener arreglos de tamaño 0:

int[] array = new int[0]; // Compiles and runs fine.
int[] array2 = {};        // Equivalent syntax.

Sin embargo, dado que es una matriz vacía, no se pueden leer ni asignar elementos:

array[0] = 1;     // Throws java.lang.ArrayIndexOutOfBoundsException.
int i = array2[0]; // Also throws ArrayIndexOutOfBoundsException.

Estas matrices vacías suelen ser útiles como valores de retorno, de modo que el código de llamada solo tiene que preocuparse por tratar con una matriz, en lugar de un valor potencial nulo que puede conducir a una [NullPointerException][so-docs-npe] .

La longitud de una matriz debe ser un número entero no negativo:

int[] array = new int[-1]; // Throws java.lang.NegativeArraySizeException

El tamaño de la matriz se puede determinar usando un campo final público llamado longitud:

System.out.println(array.length); // Prints 0 in this case.

Nota: array.length devuelve el tamaño real de la matriz y no el número de elementos de la matriz a los que se les asignó un valor, a diferencia de [ArrayList.size()][arraylist-size-javadocs] que devuelve el número de elementos de la matriz a los que se les asignó un valor.

Crear e inicializar arreglos multidimensionales #

La forma más sencilla de crear una matriz multidimensional es la siguiente:

int[][] a = new int[2][3];

Creará dos arreglos int de tres longitudes: a[0] y a[1]. Esto es muy similar a la inicialización clásica de estilo C de matrices multidimensionales rectangulares.

Puede crear e inicializar al mismo tiempo:

int[][] a = { {1, 2}, {3, 4}, {5, 6} };

A diferencia de C, donde solo se admiten matrices multidimensionales rectangulares, las matrices internas no necesitan tener la misma longitud, ni siquiera estar definidas:

int[][] a = { {1}, {2, 3}, null };

Aquí, a[0] es una matriz int de una longitud, mientras que a[1] es una matriz int de dos longitudes y a[2] es null. Las matrices como esta se denominan [matrices irregulares o matrices irregulares] [so-docs-jaggedarrays], es decir, son matrices de matrices. Los arreglos multidimensionales en Java se implementan como arreglos de arreglos, es decir, arreglo[i][j][k] es equivalente a ((arreglo[i])[j])[k]. [A diferencia de C#][so-qa-jaggeddifference], la sintaxis array[i,j] no es compatible con Java.

Creación e inicialización de matrices tipo genérico #

En las clases genéricas, las matrices de tipos genéricos no pueden inicializarse así debido a [type erasure][docs-type-erasure]:

public class MyGenericClass<T> {
    private T[] a;

    public MyGenericClass() {
        a = new T[5]; // Compile time error: generic array creation
    }
}

En su lugar, se pueden crear usando uno de los siguientes métodos: (tenga en cuenta que estos generarán advertencias no verificadas)

  1. Al crear una matriz Objeto y convertirla en el tipo genérico:

    a = (T[]) new Object[5];

    This is the simplest method, but since the underlying array is still of type Object[], this method does not provide type safety. Therefore, this method of creating an array is best used only within the generic class - not exposed publicly.

  2. Usando Array.newInstance con un parámetro de clase:

    public MyGenericClass(Class<T> clazz) {
    a = (T[]) Array.newInstance(clazz, 5);
}

    Here the class of T has to be explicitly passed to the constructor. The return type of Array.newInstance is always Object. However, this method is safer because the newly created array is always of type T[], and therefore can be safely externalized.

Declaración separada e inicialización de arreglos #

El valor de un índice para un elemento de matriz debe ser un número entero (0, 1, 2, 3, 4, …) y menor que la longitud de la matriz (los índices se basan en cero). De lo contrario, se lanzará una [ArrayIndexOutOfBoundsException][docs-aioobe]:

int[] array9;             // Array declaration - uninitialized
array9 = new int[3];      // Initialize array  - { 0, 0, 0 }
array9[0] = 10;           // Set index 0 value - { 10, 0, 0 }
array9[1] = 20;           // Set index 1 value - { 10, 20, 0 }
array9[2] = 30;           // Set index 2 value - { 10, 20, 30 }

Los arreglos no se pueden reinicializar con la sintaxis de acceso directo del inicializador de arreglos #

No es posible reinicializar una matriz a través de una sintaxis abreviada con un inicializador de matriz, ya que un inicializador de matriz solo se puede especificar en una declaración de campo o una declaración de variable local, o como parte de un expresión de creación de matriz.

Sin embargo, es posible crear una nueva matriz y asignarla a la variable que se utiliza para hacer referencia a la matriz anterior. Si bien esto da como resultado que la matriz a la que hace referencia esa variable se reinicie, el contenido de la variable es una matriz completamente nueva. Para hacer esto, el operador nuevo puede usarse con un inicializador de matriz y asignarse a la variable de matriz:

// First initialization of array
int[] array = new int[] { 1, 2, 3 };

// Prints "1 2 3 ".
for (int i : array) {
    System.out.print(i + " ");
}

// Re-initializes array to a new int[] array.
array = new int[] { 4, 5, 6 };

// Prints "4 5 6 ".
for (int i : array) {
    System.out.print(i + " ");
}

array = { 1, 2, 3, 4 }; // Compile-time error! Can't re-initialize an array via shortcut 
                        // syntax with array initializer.

Crear una lista a partir de una matriz #

El método Arrays.asList() se puede usar para devolver una Lista de tamaño fijo que contiene los elementos de la matriz dada. La Lista resultante será del mismo tipo de parámetro que el tipo base de la matriz.

String[] stringArray = {"foo", "bar", "baz"};
List<String> stringList = Arrays.asList(stringArray);

Nota: esta lista está respaldada por (una vista de) la matriz original, lo que significa que cualquier cambio en la lista cambiará la matriz y viceversa. Sin embargo, los cambios en la lista que cambiarían su tamaño (y, por lo tanto, la longitud de la matriz) generarán una excepción.

Para crear una copia de la lista, utilice el constructor de [java.util.ArrayList](https://docs.oracle.com/javase/8/docs/api/java/util/ArrayList.html#ArrayList- java.util.Collection-) tomando una Colección como argumento:

String[] stringArray = {"foo", "bar", "baz"};
List<String> stringList = new ArrayList<String>(Arrays.asList(stringArray));

En Java SE 7 y versiones posteriores, se puede usar un par de corchetes angulares <> (conjunto vacío de argumentos de tipo), que se denomina Diamante. El compilador puede determinar los argumentos de tipo a partir del contexto. Esto significa que la información de tipo se puede omitir al llamar al constructor de ArrayList y se deducirá automáticamente durante la compilación. Esto se llama Inferencia de tipo, que forma parte de Java Genéricos.

// Using Arrays.asList()

String[] stringArray = {"foo", "bar", "baz"};
List<String> stringList = new ArrayList<>(Arrays.asList(stringArray));

// Using ArrayList.addAll()

String[] stringArray = {"foo", "bar", "baz"};
ArrayList<String> list = new ArrayList<>();
list.addAll(Arrays.asList(stringArray));

// Using Collections.addAll()

String[] stringArray = {"foo", "bar", "baz"};
ArrayList<String> list = new ArrayList<>();
Collections.addAll(list, stringArray);

Un punto que vale la pena señalar sobre el Diamante es que no se puede usar con Clases anónimas.

// Using Streams

int[] ints = {1, 2, 3};
List<Integer> list = Arrays.stream(ints).boxed().collect(Collectors.toList());

String[] stringArray = {"foo", "bar", "baz"};
List<Object> list = Arrays.stream(stringArray).collect(Collectors.toList());

Notas importantes relacionadas con el uso del método Arrays.asList() #

  • Este método devuelve List, que es una instancia de Arrays$ArrayList (clase interna estática de Arrays) y no java.util.ArrayList. La Lista resultante es de tamaño fijo. Eso significa que no se admite agregar o eliminar elementos y generará una UnsupportedOperationException:

     stringList.add("something"); // throws java.lang.UnsupportedOperationException

  • Se puede crear una nueva Lista pasando una Lista respaldada por una matriz al constructor de una nueva Lista. Esto crea una nueva copia de los datos, que tiene un tamaño variable y que no está respaldada por la matriz original:

     List<String> modifiableList = new ArrayList<>(Arrays.asList("foo", "bar"));

  • Llamar a <T> List<T> asList(T... a) en un arreglo primitivo, como un int[], producirá una List<int[]> cuyo único elemento es la fuente matriz primitiva en lugar de los elementos reales de la matriz de origen.

El motivo de este comportamiento es que los tipos primitivos no se pueden usar en lugar de los parámetros de tipo genérico, por lo que, en este caso, toda la matriz primitiva reemplaza el parámetro de tipo genérico. Para convertir un arreglo primitivo en una Lista, en primer lugar, convierta el arreglo primitivo en un arreglo del tipo contenedor correspondiente (es decir, llame a Arrays.asList en un Integer[] en lugar de un int[ ]).

Por lo tanto, esto imprimirá falso:

   int[] arr = {1, 2, 3};      // primitive array of int
   System.out.println(Arrays.asList(arr).contains(1));

Por otro lado, esto imprimirá verdadero:

   Integer[] arr = {1, 2, 3};  // object array of Integer (wrapper for int)
   System.out.println(Arrays.asList(arr).contains(1));

Crear una matriz a partir de una colección #

Dos métodos en java.util.Collection crean una matriz a partir de una colección:

  • Objeto[] aArray()

  • <T> T[] a Array(T[] a)

Object[] toArray() se puede usar de la siguiente manera:

Set<String> set = new HashSet<String>();
set.add("red");
set.add("blue");

// although set is a Set<String>, toArray() returns an Object[] not a String[]
Object[] objectArray = set.toArray();

<T> T[] toArray(T[] a) se puede usar de la siguiente manera:

Set<String> set = new HashSet<String>();
set.add("red");
set.add("blue");

// The array does not need to be created up front with the correct size.
// Only the array type matters. (If the size is wrong, a new array will
// be created with the same type.)
String[] stringArray = set.toArray(new String[0]);  

// If you supply an array of the same size as collection or bigger, it
// will be populated with collection values and returned (new array
// won't be allocated)
String[] stringArray2 = set.toArray(new String[set.size()]);

La diferencia entre ellos es más que solo tener resultados sin tipo y con tipo. Su rendimiento también puede diferir (para obtener más información, lea esta sección de análisis de rendimiento):

  • Object[] toArray() usa arraycopy vectorizado, que es mucho más rápido que el arraycopy con verificación de tipo utilizado en T[] toArray(T[] a).
  • T[] toArray(new T[non-zero-size]) necesita poner a cero la matriz en tiempo de ejecución, mientras que T[] toArray(new T[0]) no lo hace. Tal evitación hace que la última llamada más rápido que el anterior. Análisis detallado aquí: Arrays of Wisdom of the Ancients.

A partir de Java SE 8+, donde se introdujo el concepto de Stream, es posible usar el Stream producido por la colección para crear un nuevo Array usando Stream.toArray método.

String[] strings = list.stream().toArray(String[]::new);

Matrices multidimensionales y dentadas #

Es posible definir una matriz con más de una dimensión. En lugar de acceder proporcionando un solo índice, se accede a una matriz multidimensional especificando un índice para cada dimensión.

La declaración de una matriz multidimensional se puede realizar agregando [] para cada dimensión a una declaración de matriz normal. Por ejemplo, para crear una matriz int bidimensional, agregue otro conjunto de corchetes a la declaración, como int[][]. Esto continúa para matrices tridimensionales (int[][][]) y así sucesivamente.

Para definir una matriz bidimensional con tres filas y tres columnas:

int rows = 3;
int columns = 3;
int[][] table = new int[rows][columns];

La matriz se puede indexar y asignarle valores con esta construcción. Tenga en cuenta que los valores no asignados son los valores predeterminados para el tipo de matriz, en este caso 0 para int.

table[0][0] = 0;
table[0][1] = 1;
table[0][2] = 2;

También es posible crear instancias de una dimensión a la vez, e incluso hacer matrices no rectangulares. Estos se conocen más comúnmente como matrices irregulares.

int[][] nonRect = new int[4][];

Es importante tener en cuenta que, aunque es posible definir cualquier dimensión de la matriz irregular, su nivel anterior debe estar definido.

// valid
String[][] employeeGraph = new String[30][];

// invalid
int[][] unshapenMatrix = new int[][10];

// also invalid
int[][][] misshapenGrid = new int[100][][10];

Inicialización literal de matriz irregular

Las matrices multidimensionales y las matrices irregulares también se pueden inicializar con una expresión literal. Lo siguiente declara y llena una matriz int de 2x3:

int[][] table = {
    {1, 2, 3},

Nota: Los subarreglos irregulares también pueden ser nulos. Por ejemplo, el siguiente código declara y llena una matriz bidimensional int cuyo primer subarreglo es nulo, el segundo subarreglo es de longitud cero, el tercer subarreglo es de una longitud y el último subarreglo es un arreglo de dos longitudes:

int[][] table = {
    null,
    {},
    {1},
    {1,2}
};

Para matrices multidimensionales, es posible extraer matrices de dimensión de nivel inferior por sus índices:

int[][][] arr = new int[3][3][3];
int[][] arr1 = arr[0]; // get first 3x3-dimensional array from arr
int[] arr2 = arr1[0]; // get first 3-dimensional array from arr1
int[] arr3 = arr[0]; // error: cannot convert from int[][] to int[]

ArrayIndexOutOfBoundsException #

La ArrayIndexOutOfBoundsException se lanza cuando se accede a un índice inexistente de una matriz.

Las matrices están indexadas en base a cero, por lo que el índice del primer elemento es 0 y el índice del último elemento es la capacidad de la matriz menos 1 (es decir, array.length - 1).

Por lo tanto, cualquier solicitud de un elemento de matriz por parte del índice i debe satisfacer la condición 0 <= i < array.length, de lo contrario, se lanzará ArrayIndexOutOfBoundsException.

El siguiente código es un ejemplo simple donde se lanza una ArrayIndexOutOfBoundsException.

String[] people = new String[] { "Carol", "Andy" };

// An array will be created:
// people[0]: "Carol"
// people[1]: "Andy"

// Notice: no item on index 2. Trying to access it triggers the exception:
System.out.println(people[2]);  // throws an ArrayIndexOutOfBoundsException.

Producción:

Exception in thread "main" java.lang.ArrayIndexOutOfBoundsException: 2
    at your.package.path.method(YourClass.java:15)

Tenga en cuenta que el índice ilegal al que se accede también se incluye en la excepción (2 en el ejemplo); esta información podría ser útil para encontrar la causa de la excepción.

Para evitar esto, simplemente verifique que el índice esté dentro de los límites de la matriz:

int index = 2;
if (index >= 0 && index < people.length) {
    System.out.println(people[index]);
}

Array Covarianza #

Las matrices de objetos son covariantes, lo que significa que al igual que Integer es una subclase de Number, Integer[] es una subclase de Number[]. Esto puede parecer intuitivo, pero puede resultar en un comportamiento sorprendente:

Integer[] integerArray = {1, 2, 3};
Number[] numberArray = integerArray;  // valid
Number firstElement = numberArray[0]; // valid
numberArray[0] = 4L;                  // throws ArrayStoreException at runtime

Aunque Integer[] es una subclase de Number[], solo puede contener Integers, e intentar asignar un elemento Long genera una excepción de tiempo de ejecución.

Tenga en cuenta que este comportamiento es exclusivo de las matrices y se puede evitar utilizando una ‘Lista’ genérica en su lugar:

List<Integer> integerList = Arrays.asList(1, 2, 3);
//List<Number> numberList = integerList;  // compile error
List<? extends Number> numberList = integerList;
Number firstElement = numberList.get(0);
//numberList.set(0, 4L);                  // compile error

No es necesario que todos los elementos de la matriz compartan el mismo tipo, siempre que sean una subclase del tipo de la matriz:

interface I {}

class A implements I {}
class B implements I {}
class C implements I {}

I[] array10 = new I[] { new A(), new B(), new C() }; // Create an array with new
                                                     // operator and array initializer.

I[] array11 = { new A(), new B(), new C() };         // Shortcut syntax with array
                                                     // initializer.

I[] array12 = new I[3];                              // { null, null, null }

I[] array13 = new A[] { new A(), new A() };          // Works because A implements I.

Object[] array14 = new Object[] { "Hello, World!", 3.14159, 42 }; // Create an array with
                                                     // new operator and array initializer.

Object[] array15 = { new A(), 64, "My String" };     // Shortcut syntax 
                                                     // with array initializer.

Iterando sobre arreglos #

Puede iterar sobre arreglos ya sea usando bucle for mejorado (también conocido como foreach) o usando índices de arreglo:

int[] array = new int[10];

// using indices: read and write
for (int i = 0; i < array.length; i++) {
    array[i] = i;
}

// extended for: read only
for (int e : array) {
    System.out.println(e);
}

Vale la pena señalar aquí que no hay una forma directa de usar un iterador en una matriz, pero a través de la biblioteca Arrays se puede convertir fácilmente en una lista para obtener un objeto iterable.

Para matrices en caja, use Arrays.asList:

Integer[] boxed = {1, 2, 3};
Iterable<Integer> boxedIt = Arrays.asList(boxed); // list-backed iterable
Iterator<Integer> fromBoxed1 = boxedIt.iterator();

Para arreglos primitivos (usando java 8) use flujos (específicamente en este ejemplo - Arrays.stream -> IntStream):

int[] primitives = {1, 2, 3};
IntStream primitiveStream = Arrays.stream(primitives); // list-backed iterable
PrimitiveIterator.OfInt fromPrimitive1 = primitiveStream.iterator();

Si no puede usar transmisiones (no Java 8), puede optar por usar la biblioteca guava de Google:

Iterable<Integer> fromPrimitive2 = Ints.asList(primitives);

En arreglos bidimensionales o más, ambas técnicas se pueden usar de una manera un poco más compleja.

Ejemplo:

int[][] array = new int[10][10];

for (int indexOuter = 0; indexOuter < array.length; indexOuter++) {
    for (int indexInner = 0; indexInner < array[indexOuter].length; indexInner++) {
        array[indexOuter][indexInner] = indexOuter + indexInner;
    }
}

for (int[] numbers : array) {
    for (int value : numbers) {
        System.out.println(value);
    }
}

Es imposible establecer un Array en cualquier valor no uniforme sin utilizar un bucle basado en índice.

Por supuesto, también puede usar bucles while o do-while al iterar usando índices.

Una nota de precaución: al usar índices de matriz, asegúrese de que el índice esté entre 0 y array.length - 1 (ambos incluidos). No haga suposiciones codificadas sobre la longitud de la matriz; de lo contrario, podría romper su código si la longitud de la matriz cambia pero sus valores codificados no lo hacen.

Ejemplo:

int[] numbers = {1, 2, 3, 4};

public void incrementNumbers() {
    // DO THIS :
    for (int i = 0; i < numbers.length; i++) {
        numbers[i] += 1; //or this: numbers[i] = numbers[i] + 1; or numbers[i]++;      
    }
 
    // DON'T DO THIS :
    for (int i = 0; i < 4; i++) {
        numbers[i] += 1;
    }
}

También es mejor si no usa cálculos sofisticados para obtener el índice, sino que usa el índice para iterar y si necesita valores diferentes, calcúlelos.

Ejemplo:

public void fillArrayWithDoubleIndex(int[] array) {
    // DO THIS :
    for (int i = 0; i < array.length; i++) {
        array[i] = i * 2;
    }
 
    // DON'T DO THIS :
    int doubleLength = array.length * 2;
    for (int i = 0; i < doubleLength; i += 2) {
        array[i / 2] = i;
    }
}

Uso de matrices temporales para reducir la repetición de código

Iterar sobre una matriz temporal en lugar de repetir el código puede hacer que su código sea más limpio. Se puede utilizar cuando se realiza la misma operación en múltiples variables.

// we want to print out all of these
String name = "Margaret";
int eyeCount = 16;
double height = 50.2;
int legs = 9;
int arms = 5;


// copy-paste approach:
System.out.println(name);
System.out.println(eyeCount);
System.out.println(height);
System.out.println(legs);
System.out.println(arms);


// temporary array approach:
for(Object attribute : new Object[]{name, eyeCount, height, legs, arms})
    System.out.println(attribute);

// using only numbers
for(double number : new double[]{eyeCount, legs, arms, height})
    System.out.println(Math.sqrt(number));

Tenga en cuenta que este código no debe usarse en secciones críticas para el rendimiento, ya que se crea una matriz cada vez que se ingresa al bucle y las variables primitivas se copiarán en la matriz y, por lo tanto, no se pueden modificar.

Arrays a una cadena #

Desde Java 1.5, puede obtener una representación String del contenido de la matriz especificada sin iterar sobre cada uno de sus elementos. Simplemente use Arrays.toString(Object[]) o Arrays.deepToString(Object[]) para arreglos multidimensionales:

int[] arr = {1, 2, 3, 4, 5};
System.out.println(Arrays.toString(arr));      // [1, 2, 3, 4, 5]

int[][] arr = {
    {1, 2, 3},
    {4, 5, 6},
    {7, 8, 9}
};
System.out.println(Arrays.deepToString(arr));  // [[1, 2, 3], [4, 5, 6], [7, 8, 9]]

El método Arrays.toString() usa el método Object.toString() para producir valores String de cada elemento en la matriz, además de la matriz de tipo primitivo, se puede usar para todo tipo de matrices. Por ejemplo:

public class Cat { /* implicitly extends Object */
    @Override
    public String toString() {
      return "CAT!";
    }
}

Cat[] arr = { new Cat(), new Cat() };
System.out.println(Arrays.toString(arr));        // [CAT!, CAT!]

Si no existe toString() anulado para la clase, entonces se usará toString() heredado de Object. Por lo general, la salida no es muy útil, por ejemplo:

public class Dog {
    /* implicitly extends Object */
}

Dog[] arr = { new Dog() };
System.out.println(Arrays.toString(arr));        // [Dog@17ed40e0]

Ordenar arreglos #

La clasificación de matrices se puede hacer fácilmente con Arrays api.

String[] names = {"John", "Steve", "Shane", "Adam", "Ben"};

Obtener la longitud de una matriz #

Las matrices son objetos que proporcionan espacio para almacenar hasta su tamaño de elementos de un tipo específico. El tamaño de una matriz no se puede modificar después de crear la matriz.

int[] arr1 = new int[0];
int[] arr2 = new int[2];
int[] arr3 = new int[]{1, 2, 3, 4};
int[] arr4 = {1, 2, 3, 4, 5, 6, 7};

int len1 = arr1.length; // 0
int len2 = arr2.length; // 2
int len3 = arr3.length; // 4
int len4 = arr4.length; // 7

El campo longitud en una matriz almacena el tamaño de una matriz. Es un campo final y no se puede modificar.

Este código muestra la diferencia entre la longitud de una matriz y la cantidad de objetos que almacena una matriz.

public static void main(String[] args) {
    Integer arr[] = new Integer[] {1,2,3,null,5,null,7,null,null,null,11,null,13};

    int arrayLength = arr.length;
    int nonEmptyElementsCount = 0;

    for (int i=0; i<arrayLength; i++) {
        Integer arrElt = arr[i];
        if (arrElt != null) {
            nonEmptyElementsCount++;
        }
    }

    System.out.println("Array 'arr' has a length of "+arrayLength+"\n"
                            + "and it contains "+nonEmptyElementsCount+" non-empty values");
}

Resultado:

Array 'arr' has a length of 13
and it contains 7 non-empty values

Eliminar un elemento de una matriz #

Java no proporciona un método directo en java.util.Arrays para eliminar un elemento de una matriz. Para realizarlo, puede copiar la matriz original a una nueva sin el elemento para eliminar o convertir su matriz a otra estructura que permita la eliminación.

Usando ArrayList #

Puede convertir la matriz en java.util.List, eliminar el elemento y volver a convertir la lista en una matriz de la siguiente manera:

String[] array = new String[]{"foo", "bar", "baz"};

List<String> list = new ArrayList<>(Arrays.asList(array));
list.remove("foo");

// Creates a new array with the same size as the list and copies the list
// elements to it.
array = list.toArray(new String[list.size()]);

System.out.println(Arrays.toString(array)); //[bar, baz]

Usando System.arraycopy #

System.arraycopy() se puede usar para hacer una copia de la matriz original y eliminar el elemento que desea. Debajo de un ejemplo:

int[] array = new int[] { 1, 2, 3, 4 }; // Original array.
int[] result = new int[array.length - 1]; // Array which will contain the result.
int index = 1; // Remove the value "2".

// Copy the elements at the left of the index.
System.arraycopy(array, 0, result, 0, index);
// Copy the elements at the right of the index.
System.arraycopy(array, index + 1, result, index, array.length - index - 1);

System.out.println(Arrays.toString(result)); //[1, 3, 4]

Usando Apache Commons Lang #

Para eliminar fácilmente un elemento, puede usar la biblioteca Apache Commons Lang y especialmente el método estático removeElement() de la clase ArrayUtils. Debajo de un ejemplo:

int[] array = new int[]{1,2,3,4};
array = ArrayUtils.removeElement(array, 2); //remove first occurrence of 2
System.out.println(Arrays.toString(array)); //[1, 3, 4]

¿Cómo se cambia el tamaño de una matriz? #

La respuesta simple es que no puedes hacer esto. Una vez que se ha creado una matriz, no se puede cambiar su tamaño. En cambio, una matriz solo se puede “redimensionar” creando una nueva matriz con el tamaño apropiado y copiando los elementos de la matriz existente a la nueva.

String[] listOfCities = new String[3];   // array created with size 3.
listOfCities[0] = "New York";
listOfCities[1] = "London";
listOfCities[2] = "Berlin";

Supongamos (por ejemplo) que se necesita agregar un nuevo elemento a la matriz listOfCities definida anteriormente. Para hacer esto, necesitarás:

  1. crea una nueva matriz con tamaño 4,
  2. copie los 3 elementos existentes de la matriz anterior a la nueva matriz en los desplazamientos 0, 1 y 2, y
  3. agregue el nuevo elemento a la nueva matriz en el desplazamiento 3.

Hay varias formas de hacer lo anterior. Antes de Java 6, la forma más concisa era:

String[] newArray = new String[listOfCities.length + 1];
System.arraycopy(listOfCities, 0, newArray, 0, listOfCities.length);
newArray[listOfCities.length] = "Sydney";

Desde Java 6 en adelante, los métodos Arrays.copyOf y Arrays.copyOfRange pueden hacer esto de manera más simple:

String[] newArray = Arrays.copyOf(listOfCities, listOfCities.length + 1);
newArray[listOfCities.length] = "Sydney";

Para conocer otras formas de copiar una matriz, consulte el siguiente ejemplo. Tenga en cuenta que necesita una copia de matriz con una longitud diferente a la original al cambiar el tamaño.

Una mejor alternativa al cambio de tamaño de matriz #

Hay dos inconvenientes principales al cambiar el tamaño de una matriz como se describe anteriormente:

  • Es ineficiente. Hacer una matriz más grande (o más pequeña) implica copiar muchos o todos los elementos de matriz existentes y asignar un nuevo objeto de matriz. Cuanto más grande es la matriz, más caro se vuelve.
  • Debe poder actualizar cualquier variable “en vivo” que contenga referencias a la matriz anterior.

Una alternativa es crear la matriz con un tamaño lo suficientemente grande para empezar. Esto solo es viable si puede determinar ese tamaño con precisión antes de asignar la matriz. Si no puede hacer eso, entonces surge nuevamente el problema de cambiar el tamaño de la matriz.

La otra alternativa es utilizar una clase de estructura de datos proporcionada por la biblioteca de clases de Java SE o una biblioteca de terceros. Por ejemplo, el marco de “colecciones” de Java SE proporciona una serie de implementaciones de las API List, Set y Map con diferentes propiedades de tiempo de ejecución. La clase ArrayList es la más cercana a las características de rendimiento de una matriz simple (por ejemplo, búsqueda O(N), obtención y configuración O(1), inserción y eliminación aleatorias O(N)) al tiempo que proporciona un cambio de tamaño más eficiente sin el problema de actualización de referencia.

(La eficiencia de cambio de tamaño para ArrayList proviene de su estrategia de duplicar el tamaño de la matriz de respaldo en cada cambio de tamaño. Para un caso de uso típico, esto significa que solo cambia el tamaño ocasionalmente. Cuando amortiza durante la vida útil de la lista, el el costo de cambio de tamaño por inserción es O (1). Puede ser posible usar la misma estrategia al cambiar el tamaño de una matriz simple).

  int index = Arrays.asList(strings).indexOf("A");

  int index = IntStream.range(0, strings.length)
                       .filter(i -> "A".equals(strings[i]))
                       .findFirst()
                       .orElse(-1); // If not present, gives us -1.
  // Similar for an array of primitives

Búsqueda lineal usando un bucle #

  int index = -1;

Convertir arreglos entre primitivos y tipos en caja #

A veces es necesaria la conversión de tipos primitivos a tipos en caja.

Para convertir la matriz, es posible usar flujos (en Java 8 y superior):

int[] primitiveArray = {1, 2, 3, 4};
Integer[] boxedArray = 
    Arrays.stream(primitiveArray).boxed().toArray(Integer[]::new);

Con versiones más bajas, puede ser iterando la matriz primitiva y copiándola explícitamente en la matriz en caja:

int[] primitiveArray = {1, 2, 3, 4};
Integer[] boxedArray = new Integer[primitiveArray.length];
for (int i = 0; i < primitiveArray.length; ++i) {
    boxedArray[i] = primitiveArray[i]; // Each element is autoboxed here

Integer[] boxedArray = {1, 2, 3, 4};

Integer[] boxedArray = {1, 2, 3, 4};
int[] primitiveArray = new int[boxedArray.length];
for (int i = 0; i < boxedArray.length; ++i) {
    primitiveArray[i] = boxedArray[i]; // Each element is outboxed here
}

Comparando arreglos para la igualdad #

Los tipos de matrices heredan sus implementaciones [equals() (y hashCode()) de java.lang.Object](https://www.wikiod.com/es/java/metodos-de-clase-de-objeto-y-constructor#método igual() -method#t=201607191720385793182), por lo que equals() solo devolverá verdadero cuando se compare con exactamente el mismo objeto de matriz. Para comparar matrices en cuanto a igualdad en función de sus valores, utilice [java.util.Arrays.equals](https://docs.oracle.com/javase/8/docs/api/java/util/Arrays.html#equals -java.lang.Object:A-java.lang.Object:A-), que está sobrecargado para todos los tipos de matriz.

int[] a = new int[]{1, 2, 3};
int[] b = new int[]{1, 2, 3};
System.out.println(a.equals(b)); //prints "false" because a and b refer to different objects
System.out.println(Arrays.equals(a, b)); //prints "true" because the elements of a and b have the same values

Cuando el tipo de elemento es un tipo de referencia, Arrays.equals() llama a equals() en los elementos de la matriz para determinar la igualdad. En particular, si el tipo de elemento es en sí mismo un tipo de matriz, se utilizará la comparación de identidad. Para comparar arreglos multidimensionales por igualdad, use [Arrays.deepEquals()](https://docs.oracle.com/javase/8/docs/api/java/util/Arrays.html#deepEquals-java.lang. Object:A-java.lang.Object:A-) en lugar de lo siguiente:

int a[] = { 1, 2, 3 };
int b[] = { 1, 2, 3 };

Object[] aObject = { a }; // aObject contains one element
Object[] bObject = { b }; // bObject contains one element

System.out.println(Arrays.equals(aObject, bObject)); // false
System.out.println(Arrays.deepEquals(aObject, bObject));// true

Debido a que los conjuntos y mapas usan equals() y hashCode(), las matrices generalmente no son útiles como elementos de conjunto o claves de mapa. Envuélvalos en una clase auxiliar que implemente equals() y hashCode() en términos de los elementos de la matriz, o conviértalos en instancias de List y almacene las listas.

Copiando arreglos #

Java proporciona varias formas de copiar una matriz.

en bucle #

int[] a = { 4, 1, 3, 2 };
int[] b = new int[a.length]; 
for (int i = 0; i < a.length; i++) {
    b[i] = a[i];
}

Tenga en cuenta que el uso de esta opción con una matriz de objetos en lugar de una matriz primitiva llenará la copia con referencia al contenido original en lugar de una copia del mismo.

Objeto.clon() #

Dado que las matrices son Objects en Java, puede usar Object.clone().

int[] a = { 4, 1, 3, 2 };
int[] b = a.clone(); // [4, 1, 3, 2]

Tenga en cuenta que el método Object.clone para una matriz realiza una copia superficial, es decir, devuelve una referencia a una nueva matriz que hace referencia a los mismos elementos que la matriz de origen.

Matrices.copyOf() #

java.util.Arrays proporciona una manera fácil de realizar la copia de una matriz a otra. Aquí está el uso básico:

int[] a = {4, 1, 3, 2};
int[] b = Arrays.copyOf(a, a.length); // [4, 1, 3, 2]

Tenga en cuenta que Arrays.copyOf también proporciona una sobrecarga que le permite cambiar el tipo de matriz:

Double[] doubles = { 1.0, 2.0, 3.0 };
Number[] numbers = Arrays.copyOf(doubles, doubles.length, Number[].class);

Sistema.arraycopy() #

public static void arraycopy(Object src, int srcPos, Object dest, int destPos, int length) Copies an array from the specified source array, comenzando en la posición especificada, hasta la posición especificada del matriz de destino.

A continuación un ejemplo de uso

int[] a = { 4, 1, 3, 2 };
int[] b = new int[a.length];
System.arraycopy(a, 0, b, 0, a.length); // [4, 1, 3, 2]

Matrices.copyOfRange() #

Se usa principalmente para copiar una parte de una matriz, también puede usarlo para copiar una matriz completa a otra como se muestra a continuación:

int[] a = { 4, 1, 3, 2 };
int[] b = Arrays.copyOfRange(a, 0, a.length); // [4, 1, 3, 2]

Fundición de matrices #

Las matrices son objetos, pero su tipo está definido por el tipo de los objetos contenidos. Por lo tanto, uno no puede convertir A[] en T[], sino que cada miembro A del A[] específico debe convertirse en un objeto T. Ejemplo genérico:

public static <T, A> T[] castArray(T[] target, A[] array) {
    for (int i = 0; i < array.length; i++) {
        target[i] = (T) array[i];
    }
    return target;
}

Por lo tanto, dada una matriz A[]:

T[] target = new T[array.Length];
target = castArray(target, array);

Java SE proporciona el método Arrays.copyOf(original, newLength, newType) para este propósito:

Double[] doubles = { 1.0, 2.0, 3.0 };
Number[] numbers = Arrays.copyOf(doubles, doubles.length, Number[].class);
