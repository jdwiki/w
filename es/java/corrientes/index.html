Un ‘Stream’ representa una secuencia de elementos y admite diferentes tipos de operaciones para realizar cálculos sobre esos elementos. Con Java 8, la interfaz Collection tiene dos métodos para generar un Stream: stream() y parallelStream(). Las operaciones Stream son intermedias o terminales. Las operaciones intermedias devuelven un ‘Stream’, por lo que se pueden encadenar varias operaciones intermedias antes de que se cierre el ‘Stream’. Las operaciones de terminal son nulas o devuelven un resultado que no es de transmisión.

Sintaxis #

  • colección.stream()
  • Matrices.stream(matriz)
  • Stream.iterate(firstValue, currentValue -> nextValue)
  • Corriente.generar(() -> valor)
  • Corriente.de(elementoDeT[, elementoDeT, …])
  • Flujo.vacío()
  • StreamSupport.stream( iterable.spliterator(), false )

Uso de flujos #

Un Stream es una secuencia de elementos sobre los que se pueden realizar operaciones agregadas secuenciales y paralelas. Cualquier ‘Stream’ dado puede potencialmente tener una cantidad ilimitada de datos fluyendo a través de él. Como resultado, los datos recibidos de un ‘Stream’ se procesan individualmente a medida que llegan, en lugar de realizar un procesamiento por lotes en todos los datos. Cuando se combinan con expresiones lambda, proporcionan una forma concisa de realizar operaciones en secuencias de datos utilizando un enfoque funcional.

Ejemplo: (ver cómo funciona en Ideone)

Stream<String> fruitStream = Stream.of("apple", "banana", "pear", "kiwi", "orange");

fruitStream.filter(s -> s.contains("a"))
           .map(String::toUpperCase)
           .sorted()
           .forEach(System.out::println);

Las operaciones realizadas por el código anterior se pueden resumir de la siguiente manera:

  1. Cree un Stream<String> que contenga un Stream ordenado en secuencia de elementos String de frutas utilizando el método de fábrica estático Stream.of(values).

  2. La operación filter() retiene solo los elementos que coinciden con un predicado dado (los elementos que, cuando son probados por el predicado, devuelven verdadero). En este caso, conserva los elementos que contienen una "a". El predicado se da como una expresión lambda.

  3. La operación map() transforma cada elemento usando una función dada, llamada mapeador. En este caso, cada ‘Cadena’ de fruta se asigna a su versión de ‘Cadena’ en mayúsculas utilizando método-referencia Cadena::toMayúsculas.

  1. La operación sorted() ordena los elementos del Stream según su orden natural (lexicográficamente, en el caso de String).

  2. Finalmente, la operación forEach(action) realiza una acción que actúa sobre cada elemento del Stream, pasándoselo a un Consumer. En el ejemplo, cada elemento simplemente se imprime en la consola. Esta operación es una operación terminal, por lo que es imposible volver a operarla.

Tenga en cuenta que las operaciones definidas en Stream se realizan debido a la operación del terminal. Sin una operación de terminal, la secuencia no se procesa. Las secuencias no se pueden reutilizar. Una vez que se llama a una operación de terminal, el objeto ‘Stream’ se vuelve inutilizable.

Las operaciones (como se ve arriba) se encadenan para formar lo que se puede ver como una consulta sobre los datos.

Transmisiones de cierre #

Tenga en cuenta que, por lo general, no es necesario cerrar un Stream. Solo es necesario cerrar los streams que operan en los canales IO. La mayoría de los tipos Stream no funcionan con recursos y, por lo tanto, no es necesario cerrarlos.

La interfaz Stream extiende [AutoCloseable][autocloseable-javadocs]. Los flujos se pueden cerrar llamando al método [close][autocloseable-close-javadocs] o usando instrucciones try-with-resource.

Un ejemplo de caso de uso en el que se debe cerrar un ‘Stream’ es cuando creas un ‘Stream’ de líneas a partir de un archivo:

try (Stream<String> lines = Files.lines(Paths.get("somePath"))) {
    lines.forEach(System.out::println);
}

La interfaz Stream también declara el método Stream.onClose() que le permite registrar controladores Runnable que se llamarán cuando se cierre la secuencia. Un caso de uso de ejemplo es donde el código que produce un flujo necesita saber cuándo se consume para realizar alguna limpieza.

public Stream<String>streamAndDelete(Path path) throws IOException {
    return Files.lines(path).onClose(() -> someClass.deletePath(path));
}

El controlador de ejecución solo se ejecutará si se llama al método close(), ya sea explícita o implícitamente mediante una declaración de prueba con recursos.

Orden de procesamiento #

El procesamiento de un objeto Stream puede ser secuencial o [paralelo] (https://www.wikiod.com/es/java/corrientes#Corriente paralela).

En un modo secuencial, los elementos se procesan en el orden de origen del Stream. Si se ordena el Stream (como una implementación de SortedMap o una List), se garantiza que el procesamiento coincida con el orden de la fuente. En otros casos, sin embargo, se debe tener cuidado de no depender del orden (consulte: ¿es coherente el orden de iteración de Java HashMap keySet()?).

Ejemplo:

List<Integer> integerList = Arrays.asList(0, 1, 2, 3, 42); 

// sequential 
long howManyOddNumbers = integerList.stream()
                                    .filter(e -> (e % 2) == 1)
                                    .count(); 

System.out.println(howManyOddNumbers); // Output: 2

En vivo en Ideone

El modo Paralelo permite el uso de varios subprocesos en varios núcleos, pero no hay garantía del orden en que se procesan los elementos.

Si se invocan varios métodos en un Stream secuencial, no es necesario invocar todos los métodos. Por ejemplo, si se filtra un Stream y el número de elementos se reduce a uno, no se producirá una llamada posterior a un método como sort. Esto puede aumentar el rendimiento de un ‘Stream’ secuencial, una optimización que no es posible con un ‘Stream’ paralelo.

Ejemplo:

// parallel
long howManyOddNumbersParallel = integerList.parallelStream()
                                            .filter(e -> (e % 2) == 1)
                                            .count();

System.out.println(howManyOddNumbersParallel); // Output: 2

Diferencias de Contenedores (o Colecciones) #

Si bien algunas acciones se pueden realizar tanto en Contenedores como en Streams, en última instancia, tienen diferentes propósitos y admiten diferentes operaciones. Los contenedores se centran más en cómo se almacenan los elementos y cómo se puede acceder a esos elementos de manera eficiente. Un ‘Stream’, por otro lado, no proporciona acceso directo ni manipulación a sus elementos; está más dedicado al grupo de objetos como una entidad colectiva y realiza operaciones en esa entidad como un todo. Stream y Collection son abstracciones separadas de alto nivel para estos diferentes propósitos.

Creación de un mapa de frecuencia #

El recopilador groupingBy(classifier, downstream) permite la recopilación de elementos Stream en un Map clasificando cada elemento en un grupo y realizando una operación descendente sobre los elementos clasificados en un mismo grupo.

Un ejemplo clásico de este principio es usar un Map para contar las ocurrencias de elementos en un Stream. En este ejemplo, el clasificador es simplemente la función de identidad, que devuelve el elemento tal cual. La operación descendente cuenta el número de elementos iguales mediante counting().

Stream.of("apple", "orange", "banana", "apple")
      .collect(Collectors.groupingBy(Function.identity(), Collectors.counting()))
      .entrySet()
      .forEach(System.out::println);

La operación descendente es en sí misma un recopilador (Collectors.counting()) que opera en elementos de tipo String y produce un resultado de tipo Long. El resultado de la llamada al método collect es Map<String, Long>.

Esto produciría la siguiente salida:

plátano=1
naranja=1
manzana=2

Corrientes infinitas #

Es posible generar un Stream que no termine. Llamar a un método de terminal en un ‘Stream’ infinito hace que el ‘Stream’ entre en un bucle infinito. El método limit de un Stream se puede utilizar para limitar el número de términos del Stream que procesa Java.

Este ejemplo genera un ‘Stream’ de todos los números naturales, comenzando con el número 1. Cada término sucesivo del ‘Stream’ es uno más alto que el anterior. Al llamar al método de límite de este ‘Flujo’, solo se consideran e imprimen los primeros cinco términos del ‘Flujo’.

// Generate infinite stream - 1, 2, 3, 4, 5, 6, 7, ...
IntStream naturalNumbers = IntStream.iterate(1, x -> x + 1);

// Print out only the first 5 terms
naturalNumbers.limit(5).forEach(System.out::println);

Recopilar con toList() y toSet() #

Los elementos de un Stream se pueden recopilar fácilmente en un contenedor utilizando la operación Stream.collect:

System.out.println(Arrays
    .asList("apple", "banana", "pear", "kiwi", "orange")
    .stream()
    .filter(s -> s.contains("a"))
    .collect(Collectors.toList())
);
// prints: [apple, banana, pear, orange]

Se pueden crear otras instancias de colección, como un Set, utilizando otros métodos integrados de Collectors. Por ejemplo, Collectors.toSet() recopila los elementos de un Stream en un Set.

Control explícito sobre la implementación de List o Set

De acuerdo con la documentación de Collectors#toList() y Collectors#toSet(), no hay garantías sobre el tipo, la mutabilidad, la serialización o la seguridad de subprocesos del List o Set devueltos.

Para que se devuelva un control explícito sobre la implementación, se puede usar Collectors#toCollection(Supplier), donde el proveedor dado devuelve una colección nueva y vacía.

// syntax with method reference
System.out.println(strings
        .stream()
        .filter(s -> s != null && s.length() <= 3)
        .collect(Collectors.toCollection(ArrayList::new))
);

// syntax with lambda
System.out.println(strings
        .stream()
        .filter(s -> s != null && s.length() <= 3)
        .collect(Collectors.toCollection(() -> new LinkedHashSet<>()))
);

Impacto en el rendimiento #

En caso de que la red esté involucrada, los ‘Stream’ paralelos pueden degradar el rendimiento general de una aplicación porque todos los ‘Stream’ paralelos usan un conjunto de subprocesos de unión de bifurcación común para la red.

Por otro lado, los ‘Stream’ paralelos pueden mejorar significativamente el rendimiento en muchos otros casos, dependiendo de la cantidad de núcleos disponibles en la CPU en ejecución en ese momento.

System.out.println(concat2.collect(Collectors.joining(", ")));
// prints: a, b, c, 1, 2, 3, alpha, beta, gamma

Alternativamente, para simplificar la sintaxis concat() anidada, los Stream también se pueden concatenar con flatMap():

final Stream<String> concat3 = Stream.of(
    abc.stream(), digits.stream(), greekAbc.stream())
    .flatMap(s -> s);
    // or `.flatMap(Function.identity());` (java.util.function.Function)

System.out.println(concat3.collect(Collectors.joining(", ")));
// prints: a, b, c, 1, 2, 3, alpha, beta, gamma

Tenga cuidado al construir Streams a partir de una concatenación repetida, porque acceder a un elemento de un Stream profundamente concatenado puede generar cadenas de llamadas profundas o incluso una StackOverflowException.

IntStream a cadena #

Java no tiene un Char Stream, por lo que cuando se trabaja con Strings y se construye un Stream de Characters, una opción es obtener un IntStream de puntos de código usando String Método .codePoints(). Entonces IntStream se puede obtener de la siguiente manera:

public IntStream stringToIntStream(String in) {
  return in.codePoints();
}

Es un poco más complicado hacer la conversión al revés, es decir, IntStreamToString. Eso se puede hacer de la siguiente manera:

public String intStreamToString(IntStream intStream) {
  return intStream.collect(StringBuilder::new, StringBuilder::appendCodePoint, StringBuilder::append).toString();
}

Encontrar el primer elemento que coincida con un predicado #

Es posible encontrar el primer elemento de un ‘Stream’ que coincida con una condición.

Para este ejemplo, encontraremos el primer ‘Entero’ cuyo cuadrado sea mayor a ‘50000’.

IntStream.iterate(1, i -> i + 1) // Generate an infinite stream 1,2,3,4...
    .filter(i -> (i*i) > 50000) // Filter to find elements where the square is >50000
    .findFirst(); // Find the first filtered element

Esta expresión devolverá un OptionalInt con el resultado.

Tenga en cuenta que con un ‘Stream’ infinito, Java seguirá comprobando cada elemento hasta que encuentre un resultado. Con un ‘Stream’ finito, si Java se queda sin elementos pero aún no puede encontrar un resultado, devuelve un ‘OptionalInt’ vacío.

Uso de secuencias y referencias de métodos para escribir procesos de autodocumentación #

Las referencias a métodos constituyen un excelente código de autodocumentación, y el uso de referencias a métodos con Streams hace que los procesos complicados sean fáciles de leer y comprender. Considere el siguiente código:

public interface Ordered {
    default int getOrder(){
        return 0;
    }
}

public interface Valued<V extends Ordered> {
    boolean hasPropertyTwo();
    V getValue();
}

public interface Thing<V extends Ordered> {
    boolean hasPropertyOne();
    Valued<V> getValuedProperty();
}

public <V extends Ordered> List<V> myMethod(List<Thing<V>> things) {
    List<V> results = new ArrayList<V>();
    for (Thing<V> thing : things) {
        if (thing.hasPropertyOne()) {
            Valued<V> valued = thing.getValuedProperty();
            if (valued != null && valued.hasPropertyTwo()){
                V value = valued.getValue();
                if (value != null){
                    results.add(value);
                }
            }
        }
    }
    results.sort((a, b)->{
        return Integer.compare(a.getOrder(), b.getOrder());
    });
    return results;
}

Este último método reescrito utilizando Streams y referencias de métodos es mucho más legible y cada paso del proceso se comprende rápida y fácilmente; no solo es más corto, sino que también muestra de un vistazo qué interfaces y clases son responsables del código en cada paso:

public <V extends Ordered> List<V> myMethod(List<Thing<V>> things) {
    return things.stream()
        .filter(Thing::hasPropertyOne)
        .map(Thing::getValuedProperty)
        .filter(Objects::nonNull)
        .filter(Valued::hasPropertyTwo)
        .map(Valued::getValue)
        .filter(Objects::nonNull)
        .sorted(Comparator.comparing(Ordered::getOrder))
        .collect(Collectors.toList());
}

Uso de flujos de Map.Entry para conservar los valores iniciales después del mapeo #

Cuando tiene un ‘Stream’ que necesita mapear pero desea conservar los valores iniciales también, puede mapear el ‘Stream’ a un ‘Map.Entry<K,V>’ usando un método de utilidad como el siguiente:

public static <K, V> Function<K, Map.Entry<K, V>> entryMapper(Function<K, V> mapper){
    return (k)->new AbstractMap.SimpleEntry<>(k, mapper.apply(k));
}

Luego, puede usar su convertidor para procesar Streams que tenga acceso a los valores originales y mapeados:

Set<K> mySet;
Function<K, V> transformer = SomeClass::transformerMethod;
Stream<Map.Entry<K, V>> entryStream = mySet.stream()
    .map(entryMapper(transformer));

Luego puede continuar procesando ese ‘Stream’ como de costumbre. Esto evita la sobrecarga de crear una colección intermedia.

La reducción es el proceso de aplicar un operador binario a cada elemento de una secuencia para dar como resultado un valor.

El método sum() de un IntStream es un ejemplo de reducción; aplica la suma a cada término de Stream, lo que da como resultado un valor final:

El método reduce de un Stream permite crear una reducción personalizada. Es posible usar el método reduce para implementar el método sum():

IntStream istr;
    
//Initialize istr
    
OptionalInt istr.reduce((a,b)->a+b);

Se devuelve la versión ‘Opcional’ para que los Streams vacíos se puedan manejar adecuadamente.

Otro ejemplo de reducción es combinar Stream<LinkedList<T>> en una sola LinkedList<T>:

Stream<LinkedList<T>> listStream;
    
//Create a Stream<LinkedList<T>>
    
Optional<LinkedList<T>> bigList = listStream.reduce((LinkedList<T> list1, LinkedList<T> list2)->{
    LinkedList<T> retList = new LinkedList<T>();
    retList.addAll(list1);
    retList.addAll(list2);
    return retList;
});

También puede proporcionar un elemento de identidad. Por ejemplo, el elemento de identidad para la suma es 0, como x+0==x. Para la multiplicación, el elemento de identidad es 1, como x*1==x. En el caso anterior, el elemento de identidad es una LinkedList<T> vacía, porque si agrega una lista vacía a otra lista, la lista a la que está “agregando” no cambia:

Stream<LinkedList<T>> listStream;

//Create a Stream<LinkedList<T>>

LinkedList<T> bigList = listStream.reduce(new LinkedList<T>(), (LinkedList<T> list1, LinkedList<T> list2)->{
    LinkedList<T> retList = new LinkedList<T>();
    retList.addAll(list1);
    retList.addAll(list2);
    return retList;
});

Tenga en cuenta que cuando se proporciona un elemento de identidad, el valor de retorno no se envuelve en un ‘Opcional’; si se llama en un flujo vacío, ‘reduce()’ devolverá el elemento de identidad.

Convertir un flujo de opcionales en un flujo de valores #

Es posible que deba convertir un ‘Stream’ que emita ‘Optional’ en un ‘Stream’ de valores, emitiendo solo valores de ‘Optional’ existentes. (es decir: sin valor nulo y sin tratar con Optional.empty()).

 Optional<String> op1 = Optional.empty();
 Optional<String> op2 = Optional.of("Hello World");

 List<String> result = Stream.of(op1, op2)
                             .filter(Optional::isPresent)
                             .map(Optional::get)
                             .collect(Collectors.toList());

 System.out.println(result); //[Hello World]

Obtener una porción de un flujo #

Ejemplo: Obtenga un ‘Stream’ de 30 elementos, que contenga del 21 al 50 (inclusive) elementos de una colección.

final long n = 20L; // the number of elements to skip
final long maxSize = 30L; // the number of elements the stream should be limited to
final Stream<T> slice = collection.stream().skip(n).limit(maxSize);

Notas:

  • Se lanza IllegalArgumentException si n es negativo o maxSize es negativo
  • tanto skip(long) como limit(long) son operaciones intermedias
  • si una secuencia contiene menos de n elementos, entonces skip(n) devuelve una secuencia vacía
  • tanto skip(long) como limit(long) son operaciones baratas en canalizaciones de flujo secuencial, pero pueden ser bastante costosas en canalizaciones paralelas ordenadas

Crear un mapa basado en un Stream #

Estuche sencillo sin llaves duplicadas

Stream<String> characters = Stream.of("A", "B", "C");

Map<Integer, String> map = characters
            .collect(Collectors.toMap(element -> element.hashCode(), element -> element));
// map = {65=A, 66=B, 67=C}

Para hacer las cosas más declarativas, podemos usar el método estático en la interfaz Function - Function.identity(). Podemos reemplazar este lambda element -> element con Function.identity().

Caso en el que puede haber llaves duplicadas

El javadoc para Collectors.toMap dice:

Si las claves asignadas contienen duplicados (según Object.equals(Object)), se lanza una IllegalStateException cuando se realiza la operación de recopilación. Si las teclas mapeadas pueden tener duplicados, use toMap(Function, Function, BinaryOperator) en su lugar.

Stream<String> characters = Stream.of("A", "B", "B", "C");

Map<Integer, String> map = characters
            .collect(Collectors.toMap(
                element -> element.hashCode(),
                element -> element,
                (existingVal, newVal) -> (existingVal + newVal)));

// map = {65=A, 66=BB, 67=C}

El BinaryOperator pasado a Collectors.toMap(...) genera el valor que se almacenará en caso de colisión. Puede:

  • devolver el valor anterior, de modo que el primer valor de la transmisión tenga prioridad,
  • devolver el nuevo valor, de modo que el último valor en la secuencia tenga prioridad, o
  • combinar los valores antiguos y nuevos

Agrupación por valor

Puede usar Collectors.groupingBy cuando necesite realizar el equivalente a una operación de “agrupar por” en cascada en la base de datos. Para ilustrar, lo siguiente crea un mapa en el que los nombres de las personas se asignan a los apellidos:

List<Person> people = Arrays.asList(
    new Person("Sam", "Rossi"),
    new Person("Sam", "Verdi"),
    new Person("John", "Bianchi"),
    new Person("John", "Rossi"),
    new Person("John", "Verdi")
);

Map<String, List<String>> map = people.stream()
        .collect(
                // function mapping input elements to keys
                Collectors.groupingBy(Person::getName, 
                // function mapping input elements to values,
                // how to store values
                Collectors.mapping(Person::getSurname, Collectors.toList()))
        );

// map = {John=[Bianchi, Rossi, Verdi], Sam=[Rossi, Verdi]}

Corrientes de Primitivos #

Java proporciona Streams especializados para tres tipos de primitivas IntStream (para ints), LongStream (para longs) y DoubleStream (para dobles). Además de ser implementaciones optimizadas para sus respectivas primitivas, también proporcionan varios métodos de terminal específicos, típicamente para operaciones matemáticas. P.ej.:

IntStream is = IntStream.of(10, 20, 30);
double average = is.average().getAsDouble(); // average is 20.0

Categorías de operaciones de transmisión #

Las operaciones de flujo se dividen en dos categorías principales, operaciones intermedias y terminales, y dos subcategorías, sin estado y con estado.

Operaciones Intermedias: #

Una operación intermedia siempre es perezosa, como un Stream.map simple. No se invoca hasta que la secuencia se consume realmente. Esto se puede verificar fácilmente:

Arrays.asList(1, 2 ,3).stream().map(i -> {
    throw new RuntimeException("not gonna happen");
    return i;
});

Las operaciones intermedias son los bloques de construcción comunes de una transmisión, se encadenan después de la fuente y, por lo general, van seguidas de una operación terminal que activa la cadena de transmisión.

Operaciones de terminales #

Las operaciones de terminal son las que desencadenan el consumo de un flujo. Algunos de los más comunes son Stream.forEach o Stream.collect. Por lo general, se colocan después de una cadena de operaciones intermedias y casi siempre están ansiosas.

Operaciones sin estado #

La apatridia significa que cada elemento se procesa sin el contexto de otros elementos. Las operaciones sin estado permiten un procesamiento eficiente de la memoria de los flujos. Las operaciones como Stream.map y Stream.filter que no requieren información sobre otros elementos de la secuencia se consideran sin estado.

operaciones con estado #

Estado significa que la operación en cada elemento depende de (algunos) otros elementos de la secuencia. Esto requiere un estado para ser preservado. Las operaciones de estado pueden romperse con secuencias largas o infinitas. Operaciones como Stream.sorted requieren que se procese la totalidad de la secuencia antes de que se emita cualquier elemento, lo que dividirá una secuencia de elementos lo suficientemente larga. Esto se puede demostrar con una transmisión larga (corre bajo tu propio riesgo):

// works - stateless stream
long BIG_ENOUGH_NUMBER = 999999999;
IntStream.iterate(0, i -> i + 1).limit(BIG_ENOUGH_NUMBER).forEach(System.out::println);

Esto provocará una falta de memoria debido al estado de Stream.sorted:

// Out of memory - stateful stream
IntStream.iterate(0, i -> i + 1).limit(BIG_ENOUGH_NUMBER).sorted().forEach(System.out::println);

Unir una transmisión a una sola cadena #

Un caso de uso que se presenta con frecuencia es la creación de una “Cadena” a partir de una secuencia, donde los elementos de la secuencia están separados por un carácter determinado. El método Collectors.joining() se puede usar para esto, como en el siguiente ejemplo:

Stream<String> fruitStream = Stream.of("apple", "banana", "pear", "kiwi", "orange");

String result = fruitStream.filter(s -> s.contains("a"))
           .map(String::toUpperCase)
           .sorted()
           .collect(Collectors.joining(", "));
           
System.out.println(result);

Producción:

MANZANA, PLÁTANO, NARANJA, PERA

El método Collectors.joining() también puede incluir prefijos y posfijos:

String result = fruitStream.filter(s -> s.contains("e"))
           .map(String::toUpperCase)
           .sorted()
           .collect(Collectors.joining(", ", "Fruits: ", "."));
           
System.out.println(result);

Producción:

Frutas: MANZANA, NARANJA, PERA.

Generando cadenas aleatorias usando Streams #

A veces es útil crear Strings aleatorias, tal vez como ID de sesión para un servicio web o una contraseña inicial después del registro de una aplicación. Esto se puede lograr fácilmente usando Streams.

Primero necesitamos inicializar un generador de números aleatorios. Para mejorar la seguridad de las Strings generadas, es una buena idea usar SecureRandom.

Nota: Crear un SecureRandom es bastante costoso, por lo que es una buena práctica hacerlo solo una vez y llamar a uno de sus métodos setSeed() de vez en cuando para volver a generarlo.

private static final SecureRandom rng = new SecureRandom(SecureRandom.generateSeed(20)); 
//20 Bytes as a seed is rather arbitrary, it is the number used in the JavaDoc example

Al crear Strings aleatorios, generalmente queremos que usen solo ciertos caracteres (por ejemplo, solo letras y dígitos). Por lo tanto, podemos crear un método que devuelva un booleano que luego se puede usar para filtrar el Stream.

//returns true for all chars in 0-9, a-z and A-Z
boolean useThisCharacter(char c){
    //check for range to avoid using all unicode Letter (e.g. some chinese symbols)
    return c >= '0' && c <= 'z' && Character.isLetterOrDigit(c);
}

A continuación, podemos utilizar el RNG para generar una cadena aleatoria de longitud específica que contenga el juego de caracteres que pase nuestra verificación useThisCharacter.

public String generateRandomString(long length){
    //Since there is no native CharStream, we use an IntStream instead
    //and convert it to a Stream<Character> using mapToObj.
    //We need to specify the boundaries for the int values to ensure they can safely be cast to char
    Stream<Character> randomCharStream = rng.ints(Character.MIN_CODE_POINT, Character.MAX_CODE_POINT).mapToObj(i -> (char)i).filter(c -> this::useThisCharacter).limit(length);

    //now we can use this Stream to build a String utilizing the collect method.
    String randomString = randomCharStream.collect(StringBuilder::new, StringBuilder::append, StringBuilder::append).toString();
    return randomString;
}
