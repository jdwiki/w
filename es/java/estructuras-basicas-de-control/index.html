Todas las estructuras de control, a menos que se indique lo contrario, utilizan [sentencias de bloque] (https://docs.oracle.com/javase/specs/jls/se7/html/jls-14.html). Estos se denotan con llaves {}.

Esto difiere de las sentencias normales, que no requieren llaves, pero también vienen con una estricta advertencia de que solo se consideraría la línea inmediatamente después de la declaración anterior.

Por lo tanto, es perfectamente válido escribir cualquiera de estas estructuras de control sin llaves, siempre y cuando solo una instrucción siga al principio, pero se desaconseja encarecidamente, ya que puede conducir a implementaciones con errores, o código roto

Ejemplo:

// valid, but discouraged
Scanner scan = new Scanner(System.in);
int val = scan.nextInt();
if(val % 2 == 0)
    System.out.println("Val was even!");


// invalid; will not compile
// note the misleading indentation here
for(int i = 0; i < 10; i++)
    System.out.println(i);
    System.out.println("i is currently: " + i);

hacer…mientras Bucle #

El bucle do...while se diferencia de otros bucles en que se garantiza que se ejecutará al menos una vez. También se denomina estructura de “bucle posterior a la prueba” porque la declaración condicional se realiza después del cuerpo del bucle principal.

int i = 0;
do {
    i++;
    System.out.println(i);
} while (i < 100); // Condition gets checked AFTER the content of the loop executes.

En este ejemplo, el bucle se ejecutará hasta que se imprima el número 100 (aunque la condición sea i < 100 y no i <= 100), porque la condición del bucle se evalúa después de que se ejecute el bucle.

Con la garantía de al menos una ejecución, es posible declarar variables fuera del bucle e inicializarlas dentro.

String theWord;
Scanner scan = new Scanner(System.in);
do {
    theWord = scan.nextLine();
} while (!theWord.equals("Bird"));

System.out.println(theWord);

En este contexto, theWord se define fuera del bucle, pero dado que se garantiza que tiene un valor basado en su flujo natural, theWord se inicializará.

Para cada #

Con Java 5 y versiones posteriores, se pueden usar bucles for-each, también conocidos como bucles for mejorados:

List strings = new ArrayList();
        
cadenas.add("Esto");
cadenas.add("es");
strings.add("un bucle for-each");
        
para (Cadena cadena : cadenas) {
    System.out.println(string);
}

For each se puede usar para iterar sobre Arrays e implementaciones de la interfaz Iterable, la última incluye clases Collections, como List o Set.

La variable de bucle puede ser de cualquier tipo que se pueda asignar desde el tipo de fuente.

La variable de bucle para un bucle for mejorado para Iterable<T> o T[] puede ser del tipo S, si

  • T extiende S
  • tanto T como S son tipos primitivos y asignables sin conversión
  • S es un tipo primitivo y T se puede convertir en un tipo asignable a S después de la conversión unboxing.
  • T es un tipo primitivo y se puede convertir a S mediante conversión automática.

Ejemplos:

T elements = ...
for (S s : elements) {
}
T S Compila
entero[] largo si
largo[] entero no
Iterable<Byte> largo si
Iterable<String> Secuencia de caracteres si
Iterable<CharSequence> Cadena no
entero[] Largo no
entero[] Entero si

Declaración de cambio #

La sentencia switch es la sentencia de bifurcación multidireccional de Java. Se usa para tomar el lugar de largas cadenas if-else if-else, y hacerlas más legibles. Sin embargo, a diferencia de las declaraciones ‘si’, no se pueden usar desigualdades; cada valor debe ser definido concretamente.

Hay tres componentes críticos en la declaración switch:

  • case: este es el valor que se evalúa para la equivalencia con el argumento de la instrucción switch.
  • default: esta es una expresión general opcional, en caso de que ninguna de las declaraciones case se evalúe como true.
  • Finalización abrupta de la declaración caso; normalmente break: esto es necesario para evitar la evaluación no deseada de declaraciones case posteriores.

Con la excepción de continuar, es posible usar cualquier declaración que provoque la [finalización abrupta de una declaración] (https://docs.oracle.com/javase/specs/jls/se7/html/jls-14 .html#jls-14.1). Esto incluye:

  • romper
  • volver
  • tirar

En el siguiente ejemplo, se escribe una declaración switch típica con cuatro casos posibles, incluido default.

Scanner scan = new Scanner(System.in);
int i = scan.nextInt();
switch (i) {
    case 0:
        System.out.println("i is zero");
        break;
    case 1:
        System.out.println("i is one");
        break;
    case 2:
        System.out.println("i is two");
        break;
    default:
        System.out.println("i is less than zero or greater than two");
}

Al omitir break o cualquier declaración que se complete de forma abrupta, podemos aprovechar lo que se conoce como casos “fall-through”, que evalúan contra varios valores. Esto se puede usar para crear rangos para que un valor tenga éxito, pero aún no es tan flexible como las desigualdades.

Scanner scan = new Scanner(System.in);
int foo = scan.nextInt();
switch(foo) {
    case 1:
        System.out.println("I'm equal or greater than one");
    case 2:
    case 3:    
        System.out.println("I'm one, two, or three");
        break;
    default:
        System.out.println("I'm not either one, two, or three");
}

En el caso de foo == 1 la salida será:

I'm equal or greater than one
I'm one, two, or three

En el caso de foo == 3 la salida será:

I'm one, two, or three

La instrucción switch también se puede usar con enums.

enum Option {
    BLUE_PILL,
    RED_PILL
}

public void takeOne(Option option) {
    switch(option) {
        case BLUE_PILL:
            System.out.println("Story ends, wake up, believe whatever you want.");
            break;
        case RED_PILL:
            System.out.println("I show you how deep the rabbit hole goes.");
            break;
            
    }
}

La instrucción switch también se puede usar con Strings.

public void rhymingGame(String phrase) {
    switch (phrase) {
        case "apples and pears":
            System.out.println("Stairs");

Continuar declaración en Java #

La declaración de continuación se usa para omitir los pasos restantes en la iteración actual y comience con la siguiente iteración del bucle. El control va desde la sentencia continuar hasta el valor del paso (incremento o decremento), si lo hay.

String[] programmers = {"Adrian", "Paul", "John", "Harry"};

    //john is not printed out
    for (String name : programmers) {
        if (name.equals("John"))
            continue;
        System.out.println(name);
    }

La sentencia continuar también puede hacer que el control del programa cambie al valor del paso (si lo hay) de un bucle con nombre:

Outer: // The name of the outermost loop is kept here as 'Outer'
for(int i = 0; i < 5; )
{
    for(int j = 0; j < 5; j++)
    {
        continue Outer;
    }
}

Si / De lo contrario Si / De lo contrario Control #

if (i < 2) {
  System.out.println("i is less than 2");
} else if (i > 2) {
  System.out.println("i is more than 2");
} else {
  System.out.println("i is not less than 2, and not more than 2");
}

El bloque if solo se ejecutará cuando i sea 1 o menos.

La condición ’else if’ se verifica solo si todas las condiciones anteriores (en las construcciones ’else if’ anteriores y las construcciones ‘if’ principales) han sido probadas como ‘falsas’. En este ejemplo, la condición else if solo se verificará si i es mayor o igual a 2.

Si su resultado es verdadero, su bloque se ejecuta y cualquier construcción else if y else después de él se omitirá.

Si ninguna de las condiciones if y else if ha sido probada como true, se ejecutará el bloque else al final.

Para bucles #

for (int i = 0; i < 100; i++) {
    System.out.println(i);
}

Los tres componentes del bucle for (separados por ;) son la declaración/inicialización de variables (aquí int i = 0), la condición (aquí i < 100) y la declaración de incremento (aquí i++ ). La declaración de la variable se hace una vez como si se colocara justo dentro de { en la primera ejecución. Luego se verifica la condición, si es “verdadera”, el cuerpo del bucle se ejecutará, si es “falsa”, el bucle se detendrá. Suponiendo que el bucle continúa, el cuerpo se ejecutará y, finalmente, cuando se alcance }, la instrucción de incremento se ejecutará justo antes de que se vuelva a comprobar la condición.

Las llaves son opcionales (puede usar una línea con un punto y coma) si el ciclo contiene solo una instrucción. Pero siempre se recomienda usar llaves para evitar malentendidos y errores.

Los componentes del bucle for son opcionales. Si su lógica empresarial contiene una de estas partes, puede omitir el componente correspondiente de su bucle for.

int i = obj.getLastestValue(); // i value is fetched from a method
    
for (; i < 100; i++) { // here initialization is not done
    System.out.println(i);
}

La estructura for (;;) { function-body } es igual a un bucle while (true).

Bucles For anidados

Cualquier declaración de bucle que tenga otra declaración de bucle dentro se llama bucle anidado. La misma forma en que el bucle tiene más bucle interno se denomina ‘bucle for anidado’.

    for(;;){
        //Outer Loop Statements
        for(;;){
            //Inner Loop Statements
        }
        //Outer Loop Statements
    }

Se puede demostrar que el bucle for anidado imprime números en forma de triángulo.

for(int i=9;i>0;i--){//Outer Loop
    System.out.println();
    for(int k=i;k>0;k--){//Inner Loop -1
        System.out.print(" ");
    }
    for(int j=i;j<=9;j++){//Inner Loop -2
        System.out.print(" "+j);
    }
 }

Operador Ternario #

A veces, debe verificar una condición y establecer el valor de una variable.

por ej.

String name;

if (A > B) {
    name = "Billy";
} else {
    name = "Jimmy";
}

Esto se puede escribir fácilmente en una línea como

String name = A > B ? "Billy" : "Jimmy";

El valor de la variable se establece en el valor inmediatamente posterior a la condición, si la condición es verdadera. Si la condición es falsa, se le dará el segundo valor a la variable.

Mientras bucles #

int i = 0;
while (i < 100) { // condition gets checked BEFORE the loop body executes
    System.out.println(i);
    i++;
}

Un bucle while se ejecuta mientras la condición entre paréntesis sea verdadera. Esto también se denomina estructura de “bucle previo a la prueba” porque la declaración condicional debe cumplirse antes de que se ejecute el cuerpo del bucle principal cada vez.

Las llaves son opcionales si el ciclo contiene solo una declaración, pero algunas convenciones de estilo de codificación prefieren tener las llaves independientemente.

Si / Si no #

int i = 2;
if (i < 2) {
  System.out.println("i is less than 2");
} else {
  System.out.println("i is greater than 2");
}

Una declaración if ejecuta código condicionalmente dependiendo del resultado de la condición entre paréntesis. Cuando la condición entre paréntesis es verdadera, ingresará al bloque de la declaración if que se define mediante llaves como { y }. paréntesis de apertura hasta el paréntesis de cierre es el alcance de la instrucción if.

El bloque else es opcional y se puede omitir. Se ejecuta si la declaración si es falsa y no se ejecuta si la declaración si es verdadera porque en ese caso se ejecuta la declaración si.

Romper #

La sentencia break finaliza un bucle (como for, while) o la evaluación de una sentencia switch.

Círculo:

while(true) {
   if(someCondition == 5) {
       break;
   }
}

El ciclo en el ejemplo se ejecutaría para siempre. Pero cuando someCondition es igual a 5 en algún punto de ejecución, entonces el ciclo termina.

Si se conectan en cascada múltiples bucles, solo el bucle más interno termina usando break.

Prueba… Atrapa… Finalmente #

La estructura de control try { ... } catch ( ... ) { ... } se usa para manejar [Excepciones] (https://www.wikiod.com/es/java/excepciones-y-manejo-de-excepciones).

String age_input = "abc";
try {
    int age = Integer.parseInt(age_input);
    if (age >= 18) {
        System.out.println("You can vote!");
    } else {
        System.out.println("Sorry, you can't vote yet.");
    }
} catch (NumberFormatException ex) {
    System.err.println("Invalid input.  '" + age_input + "' is not a valid integer.");
}

Esto imprimiría:

Entrada no válida. ‘abc’ no es un entero válido.

Se puede agregar una cláusula finally después de catch. La cláusula finally siempre se ejecutaría, sin importar si se lanzó una excepción.

intentar { ... } atrapar ( ... ) { ... } finalmente { ... }

String age_input = "abc";
try {
    int age = Integer.parseInt(age_input);
    if (age >= 18) {
        System.out.println("You can vote!");
    } else {
        System.out.println("Sorry, you can't vote yet.");
    }
} catch (NumberFormatException ex) {
    System.err.println("Invalid input.  '" + age_input + "' is not a valid integer.");
} finally {
    System.out.println("This code will always be run, even if an exception is thrown");
}

Esto imprimiría:

Entrada no válida. ‘abc’ no es un entero válido. Este código siempre se ejecutará, incluso si se lanza una excepción

Ruptura/continuación anidada #

Es posible ‘romper’ / ‘continuar’ a un bucle externo usando instrucciones de etiqueta:

outerloop:
for(...) {
    innerloop:
    for(...) {
        if(condition1)
            break outerloop;

        if(condition2)
            continue innerloop; // equivalent to: continue;
    }
}

No hay otro uso para las etiquetas en Java.
