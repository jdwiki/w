Una lista es una colección ordenada de valores. En Java, las listas forman parte de Java Collections Framework. Las listas implementan la interfaz java.util.List, que amplía java.util.Collection.

Sintaxis #

  • ls.add(elemento E); //Añade un elemento
  • ls.remove(elemento E); //Elimina un elemento
  • for(E elemento : ls){} //Itera sobre cada elemento
  • ls.toArray(nueva Cadena[ls.longitud]); //Convierte una lista de cadenas en una matriz de cadenas
  • ls.get(índice int); //Devuelve el elemento en el índice especificado.
  • ls.set(índice int, elemento E); //Reemplaza el elemento en una posición especificada.
  • ls.estáVacío(); //Devuelve verdadero si la matriz no contiene elementos; de lo contrario, devuelve falso.
  • ls.indexOf(Objeto o); //Devuelve el índice de la primera ubicación del elemento especificado o, o, si no está presente, devuelve -1.
  • ls.lastIndexOf(Objeto o); //Devuelve el índice de la última ubicación del elemento especificado o, o, si no está presente, devuelve -1.
  • ls.tamaño(); //Devuelve el número de elementos de la Lista.

Una lista es un objeto que almacena una colección ordenada de valores. “Ordenado” significa que los valores se almacenan en un orden particular: un elemento aparece primero, otro en segundo lugar, y así sucesivamente. Los valores individuales se denominan comúnmente “elementos”. Las listas de Java suelen proporcionar estas características:

  • Las listas pueden contener cero o más elementos.
  • Las listas pueden contener valores duplicados. En otras palabras, un elemento se puede insertar en una lista más de una vez.
  • Las listas almacenan sus elementos en un orden particular, lo que significa que un elemento viene primero, otro después, y así sucesivamente.
  • Cada elemento tiene un índice que indica su posición dentro de la lista. El primer elemento tiene índice 0, el siguiente tiene índice 1 y así sucesivamente.
  • Las listas permiten insertar elementos al principio, al final o en cualquier índice dentro de la lista.
  • Probar si una lista contiene un valor particular generalmente significa examinar cada elemento de la lista. Esto significa que el tiempo para realizar esta comprobación es O(n), proporcional al tamaño de la lista.

Agregar un valor a una lista en algún punto que no sea el final moverá todos los siguientes elementos “abajo” o “a la derecha”. En otras palabras, agregar un elemento en el índice n mueve el elemento que solía estar en el índice n al índice n+1, y así sucesivamente. Por ejemplo:

List<String> list = new ArrayList<>();
list.add("world");
System.out.println(list.indexOf("world"));      // Prints "0"
// Inserting a new value at index 0 moves "world" to index 1
list.add(0, "Hello");
System.out.println(list.indexOf("world"));      // Prints "1"
System.out.println(list.indexOf("Hello"));      // Prints "0"

Ordenando una lista genérica #

La clase Collections ofrece dos métodos estáticos estándar para ordenar una lista:

  • sort(List<T> list) aplicable a listas donde T extiende Comparable<? súper T>, y
  • sort(List<T> list, Comparator<? super T> c) aplicable a listas de cualquier tipo.

Aplicar lo primero requiere modificar la clase de elementos de la lista que se están ordenando, lo que no siempre es posible. También puede ser indeseable ya que, aunque proporciona la clasificación predeterminada, es posible que se requieran otros órdenes de clasificación en diferentes circunstancias, o que la clasificación sea solo una tarea única.

Considere que tenemos la tarea de ordenar objetos que son instancias de la siguiente clase:

public class User {
    public final Long id;
    public final String username;

    public User(Long id, String username) {
        this.id = id;
        this.username = username;
    }

    @Override
    public String toString() {
        return String.format("%s:%d", username, id);
    }
}

Para usar Collections.sort(List<User> list) necesitamos modificar la clase User para implementar la interfaz Comparable. Por ejemplo

public class User implements Comparable<User> {
    public final Long id;
    public final String username;

    public User(Long id, String username) {
        this.id = id;
        this.username = username;
    }

    @Override
    public String toString() {
        return String.format("%s:%d", username, id);
    }

    @Override
    /** The natural ordering for 'User' objects is by the 'id' field. */
    public int compareTo(User o) {
        return id.compareTo(o.id);
    }
}

(Aparte: muchas clases estándar de Java como String, Long, Integer implementan la interfaz Comparable. Esto hace que las listas de esos elementos se puedan ordenar de forma predeterminada y simplifica la implementación de compare o compareTo en otros clases.)

Con la modificación anterior, podemos ordenar fácilmente una lista de objetos Usuario según las clases ordenación natural. (En este caso, hemos definido que se ordene en función de los valores de id). Por ejemplo:

List<User> users = Lists.newArrayList(
    new User(33L, "A"),
    new User(25L, "B"),
    new User(28L, ""));
Collections.sort(users);

System.out.print(users);
// [B:25, C:28, A:33]

Sin embargo, supongamos que queremos ordenar los objetos Usuario por nombre en lugar de id. Alternativamente, suponga que no hemos podido cambiar la clase para que implemente Comparable.

Aquí es donde el método sort con el argumento Comparator es útil:

Collections.sort(users, new Comparator<User>() {
    @Override
    /* Order two 'User' objects based on their names. */
    public int compare(User left, User right) {
        return left.username.compareTo(right.username);
    }
});
System.out.print(users);
// [A:33, B:25, C:28]

En Java 8 puede usar una lambda en lugar de una clase anónima. Este último se reduce a una sola línea:

Collections.sort(users, (l, r) -> l.username.compareTo(r.username));

Además, Java 8 agrega un método de “clasificación” predeterminado en la interfaz de “Lista”, lo que simplifica aún más la clasificación.

users.sort((l, r) -> l.username.compareTo(r.username))

Convierte una lista de enteros en una lista de cadenas #

List<Integer> nums = Arrays.asList(1, 2, 3);
List<String> strings = nums.stream()
    .map(Object::toString)
    .collect(Collectors.toList());

Eso es:

  1. Crea una secuencia de la lista
  2. Mapea cada elemento usando Object::toString
  3. Recopile los valores de String en una Lista usando Collectors.toList()

Lista de arreglo #

public class ArrayList<E>
extends AbstractList<E>
implements List<E>, RandomAccess, Cloneable, Serializable

ArrayList es una implementación de matriz redimensionable de la interfaz List. Al almacenar la lista en una matriz, ArrayList proporciona métodos (además de los métodos que implementan la interfaz List) para manipular el tamaño de la matriz.

Inicializar ArrayList de Integer con tamaño 100

List<Integer> myList = new ArrayList<Integer>(100); // Constructs an empty list with the specified initial capacity.

- PROS:

Las operaciones size, isEmpty, get, set, iterator y listIterator se ejecutan en tiempo constante. Entonces obtener y configurar cada elemento de la Lista tiene el mismo costo de tiempo:

int e1 = myList.get(0);  //   \
int e2 = myList.get(10); //    | => All the same constant cost => O(1)
myList.set(2,10);        //   /

- CONTRAS:

Al estar implementado con un arreglo (estructura estática) agregar elementos por encima del tamaño del arreglo tiene un gran costo debido a que se necesita hacer una nueva asignación para todo el arreglo. Sin embargo, de documentación:

La operación de suma se ejecuta en tiempo constante amortizado, es decir, sumando n los elementos requieren tiempo O(n)

Eliminar un elemento requiere tiempo O(n).

Encontrar elementos comunes entre 2 listas #

Suponga que tiene dos listas: A y B, y necesita encontrar los elementos que existen en ambas listas.

Puede hacerlo simplemente invocando el método List.retainAll().

Ejemplo:

public static void main(String[] args) {
    List<Integer> numbersA = new ArrayList<>();
    List<Integer> numbersB = new ArrayList<>();
    numbersA.addAll(Arrays.asList(new Integer[] { 1, 3, 4, 7, 5, 2 }));
    numbersB.addAll(Arrays.asList(new Integer[] { 13, 32, 533, 3, 4, 2 }));

    System.out.println("A: " + numbersA);
    System.out.println("B: " + numbersB);
    List<Integer> numbersC = new ArrayList<>();
    numbersC.addAll(numbersA);
    numbersC.retainAll(numbersB);

    System.out.println("List A : " + numbersA);
    System.out.println("List B : " + numbersB);
    System.out.println("Common elements between A and B: " + numbersC);

}

Crear una lista #

Dándole a tu lista un tipo

Para crear una lista, necesita un tipo (cualquier clase, por ejemplo, String). Este es el tipo de su Lista. La Lista solo almacenará objetos del tipo especificado. Por ejemplo:

List<String> strings;

Puede almacenar "string1", "hello world!", "goodbye", etc., pero no puede almacenar 9.2, sin embargo:

List<Double> doubles;

Puede almacenar 9.2, pero no "hello world!".

Iniciando su lista

Si intenta agregar algo a las listas anteriores, obtendrá una NullPointerException, porque strings y doubles son iguales a null.

Hay dos formas de inicializar una lista:

Opción 1: Usa una clase que implemente List

List es una interfaz, lo que significa que no tiene un constructor, sino métodos que una clase debe anular. ArrayList es la Lista más utilizada, aunque LinkedList también es común. Así que inicializamos nuestra lista así:

List<String> strings = new ArrayList<String>();

o

List<String> strings = new LinkedList<String>();

A partir de Java SE 7, puede usar un operador de diamante:

List<String> strings = new ArrayList<>();

o

List<String> strings = new LinkedList<>();

Opción 2: Usar la clase Colecciones

La clase Collections proporciona dos métodos útiles para crear listas sin una variable List:

  • emptyList(): devuelve una lista vacía.
  • singletonList(T): crea una lista de tipo T y añade el elemento especificado.

Y un método que utiliza una ‘Lista’ existente para completar los datos:

  • addAll(L, T...): añade todos los elementos especificados a la lista pasada como primer parámetro.

Ejemplos:

import java.util.List;
import java.util.Collections;

List<Integer> l = Collections.emptyList();
List<Integer> l1 = Collections.singletonList(42);
Collections.addAll(l1, 1, 2, 3);

Operaciones de acceso posicional #

La List API tiene ocho métodos para operaciones de acceso posicional:

  • añadir (tipo T)
  • agregar (índice int, tipo T)
  • eliminar (Objeto o)
  • eliminar (índice int)
  • obtener(índice int)
  • set(índice int, elemento E)
  • int indexOf(Objeto o)
  • int últimoÍndiceDe(Objeto o)

Entonces, si tenemos una Lista:

List<String> strings = new ArrayList<String>();

Y queríamos agregar las cadenas “¡Hola mundo!” y “¡Adiós mundo!” a ello, lo haríamos así:

strings.add("Hello world!");
strings.add("Goodbye world!");

Y nuestra lista contendría los dos elementos. Ahora digamos que queríamos agregar “¡Inicio del programa!” al primero de la lista. Lo haríamos así:

strings.add(0, "Program starting!");

NOTA: El primer elemento es 0.

Ahora bien, si quisiéramos eliminar el “¡Adiós mundo!” línea, podríamos hacerlo así:

strings.remove("Goodbye world!");

Y si quisiéramos eliminar la primera línea (que en este caso sería “¡Programa iniciando!”), podríamos hacerlo así:

strings.remove(0);

Nota:

  1. La adición y eliminación de elementos de la lista modifican la lista y esto puede conducir a una ConcurrentModificationException si la lista se itera al mismo tiempo.

  2. Agregar y eliminar elementos puede ser O(1) u O(N) según la clase de lista, el método utilizado y si está agregando/eliminando un elemento al principio, al final o en el medio de la lista.

Para recuperar un elemento de la lista en una posición específica, puede usar el método E get(int index); de List API. Por ejemplo:

strings.get(0);

devolverá el primer elemento de la lista.

Puede reemplazar cualquier elemento en una posición específica usando set(int index, E element);. Por ejemplo:

strings.set(0,"This is a replacement");

Esto establecerá la cadena “Esto es un reemplazo” como el primer elemento de la lista.

Nota: The set method will overwrite the element at the position 0. It will not add the new String at the position 0 and push the old one to the position 1.

int indexOf(Object o); devuelve la posición de la primera aparición del objeto pasado como argumento. Si no hay ocurrencias del objeto en la lista, se devuelve el valor -1. Continuando con el ejemplo anterior, si llamas:

strings.indexOf("This is a replacement")

Se espera que se devuelva el 0 cuando establecemos la cadena “Esto es un reemplazo” en la posición 0 de nuestra lista. En el caso de que haya más de una aparición en la lista cuando se llame int indexOf(Object o); entonces, como se mencionó, se devolverá el índice de la primera aparición. Llamando a int lastIndexOf(Object o) puede recuperar el índice de la última aparición en la lista. Entonces, si agregamos otro “Esto es un reemplazo”:

strings.add("This is a replacement");
strings.lastIndexOf("This is a replacement");

Esta vez se devolverá el 1 y no el 0;

Iterando sobre elementos en una lista #

Para el ejemplo, digamos que tenemos una Lista de tipo String que contiene cuatro elementos: “hola”, “cómo”, “eres”, “¿tú?”

La mejor manera de iterar sobre cada elemento es usando un bucle for-each:

public void printEachElement(List<String> list){
    for(String s : list){
        System.out.println(s);
    }
}

Que imprimiría:

hello,
how
are
you?

Para imprimirlos todos en la misma línea, puede usar un StringBuilder:

public void printAsLine(List<String> list){
    StringBuilder builder = new StringBuilder();
    for(String s : list){
        builder.append(s);
    }
    System.out.println(builder.toString());
}

Imprimirá:

hello, how are you?

Alternativamente, puede usar la indexación de elementos (como se describe en https://www.wikiod.com/es/java/liza) para iterar una lista. Advertencia: este enfoque es ineficaz para las listas enlazadas.

Creación, adición y eliminación de elementos de una ArrayList #

ArrayList es una de las estructuras de datos integradas en Java. Es una matriz dinámica (donde no es necesario declarar primero el tamaño de la estructura de datos) para almacenar elementos (Objetos).

Extiende la clase AbstractList e implementa la interfaz List. Una ArrayList puede contener elementos duplicados donde mantiene el orden de inserción. Cabe señalar que la clase ArrayList no está sincronizada, por lo que se debe tener cuidado al manejar la concurrencia con ArrayList. ArrayList permite el acceso aleatorio porque la matriz funciona en la base del índice. La manipulación es lenta en ArrayList debido al cambio que a menudo ocurre cuando se elimina un elemento de la lista de arreglos.

Se puede crear una ArrayList de la siguiente manera:

List<T> myArrayList = new ArrayList<>();

Donde T ( https://www.wikiod.com/es/java/genericos ) es el tipo que se almacenará dentro de ArrayList.

El tipo de ArrayList puede ser cualquier objeto. El tipo no puede ser un tipo primitivo (use sus clases contenedoras en su lugar).

Para agregar un elemento a ArrayList, use el método add():

myArrayList.add(element);

O para agregar un elemento a un índice determinado:

myArrayList.add(index, element); //index of the element should be an int (starting from 0)

Para eliminar un elemento de ArrayList, utilice el método remove():

myArrayList.remove(element);

O para eliminar un elemento de un índice determinado:

myArrayList.remove(index); //index of the element should be an int (starting from 0)

Reemplazo en el lugar de un elemento de Lista #

Este ejemplo se trata de reemplazar un elemento ‘Lista’ mientras se asegura de que el elemento de reemplazo esté en la misma posición que el elemento que se reemplaza.

Esto se puede hacer usando estos métodos:

  • set (índice int, tipo T)
  • int indexOf(tipo T)

Considere una ArrayList que contenga los elementos “¡Inicio del programa!”, “¡Hola mundo!” y “¡Adiós mundo!”

List<String> strings = new ArrayList<String>();
strings.add("Program starting!");
strings.add("Hello world!");
strings.add("Goodbye world!");

Si conocemos el índice del elemento que queremos reemplazar, simplemente podemos usar set de la siguiente manera:

strings.set(1, "Hi world");

Si no conocemos el índice, podemos buscarlo primero. Por ejemplo:

int pos = strings.indexOf("Goodbye world!");
if (pos >= 0) {
    strings.set(pos, "Goodbye cruel world!");
}

Notas:

  1. La operación set no causará una ConcurrentModificationException.
  2. La operación set es rápida ( O(1) ) para ArrayList pero lenta ( O(N) ) para LinkedList.
  3. Una búsqueda indexOf en un ArrayList o LinkedList es lenta ( O(N) ).

Haciendo una lista no modificable #

La clase Collections proporciona una forma de hacer que una lista no se pueda modificar:

List<String> ls = new ArrayList<String>();
List<String> unmodifiableList = Collections.unmodifiableList(ls);

Si desea una lista no modificable con un elemento, puede usar:

List<String> unmodifiableList = Collections.singletonList("Only string in the list");

Mover objetos en la lista #

La clase Colecciones le permite mover objetos en la lista usando varios métodos (ls es la Lista):

Invertir una lista:

Collections.reverse(ls);

Posiciones rotatorias de elementos en una lista

El método de rotación requiere un argumento entero. Esta es la cantidad de puntos para moverlo a lo largo de la línea. A continuación se muestra un ejemplo de esto:

List<String> ls = new ArrayList<String>();
ls.add(" how");
ls.add(" are");
ls.add(" you?");
ls.add("hello,");
Collections.rotate(ls, 1);

for(String line : ls) System.out.print(line);
System.out.println();

Esto imprimirá “hola, ¿cómo estás?”

Usando la misma lista anterior, podemos barajar los elementos en una lista:

Collections.shuffle(ls);

También podemos darle un objeto java.util.Random que usa para colocar objetos aleatoriamente en lugares:

Random random = new Random(12); 
Collections.shuffle(ls, random);
