Java I/O (Entrada y salida) se utiliza para procesar la entrada y producir la salida. Java utiliza el concepto de flujo para acelerar la operación de E/S. El paquete java.io contiene todas las clases necesarias para las operaciones de entrada y salida. El manejo de archivos también se realiza en Java mediante la API de E/S de Java.

Migración de java.io.File a Java 7 NIO (java.nio.file.Path) #

Estos ejemplos asumen que ya sabe qué es NIO de Java 7 en general, y está acostumbrado a escribir código usando java.io.File. Utilice estos ejemplos como un medio para encontrar rápidamente más documentación centrada en NIO para la migración.

Hay mucho más en el NIO de Java 7, como archivos mapeados en memoria o abrir un archivo ZIP o JAR usando FileSystem. Estos ejemplos solo cubrirán un número limitado de casos de uso básicos.

Como regla básica, si está acostumbrado a realizar una operación de lectura/escritura del sistema de archivos mediante un [java.io.File](https://docs.oracle.com/javase/7/docs/api/java/ io/File.html), lo encontrará como un método estático dentro de [java.nio.file.Files](https://docs.oracle.com/javase/7/docs/api/java/ nio/archivo/Archivos.html).

Señala un camino #

// -> IO
File file = new File("io.txt");

// -> NIO
Path path = Paths.get("nio.txt");

Caminos relativos a otro camino #

// Forward slashes can be used in place of backslashes even on a Windows operating system
// -> IO
File folder = new File("C:/");
File fileInFolder = new File(folder, "io.txt");

// -> NIO
Path directory = Paths.get("C:/");
Path pathInDirectory = directory.resolve("nio.txt");

Conversión de archivo de/a ruta para usar con bibliotecas #

// -> IO to NIO
Path pathFromFile = new File("io.txt").toPath();

// -> NIO to IO
File fileFromPath = Paths.get("nio.txt").toFile();

Comprobar si el archivo existe y eliminarlo si existe #

// -> IO
if (file.exists()) {
    boolean deleted = file.delete();
    if (!deleted) {
        throw new IOException("Unable to delete file");
    }
}

// -> NIO
Files.deleteIfExists(path);

Escribir en un archivo a través de un OutputStream #

Stream vs API de escritor/lector #

Los flujos brindan el acceso más directo al contenido binario, por lo que cualquier implementación de InputStream / OutputStream siempre opera en ints y bytes.

// Read a single byte from the stream
int b = inputStream.read();
if (b >= 0) { // A negative value represents the end of the stream, normal values are in the range 0 - 255
    // Write the byte to another stream
    outputStream.write(b);
}

// Read a chunk
byte[] data = new byte[1024];
int nBytesRead = inputStream.read(data);
if (nBytesRead >= 0) { // A negative value represents end of stream
    // Write the chunk to another stream
    outputStream.write(data, 0, nBytesRead);
}

Hay algunas excepciones, probablemente la más notable PrintStream que agrega la “capacidad de imprimir representaciones de varios valores de datos convenientemente”. Esto permite usar System.out tanto como un InputStream binario como una salida textual usando métodos como System.out.println().

Además, algunas implementaciones de secuencias funcionan como una interfaz para contenidos de nivel superior, como objetos Java (consulte Serialización) o tipos nativos, p. DataOutputStream / DataInputStream.

Con las clases Writer y Reader, Java también proporciona una API para flujos de caracteres explícitos. Aunque la mayoría de las aplicaciones basarán estas implementaciones en flujos, la API de flujo de caracteres no expone ningún método para el contenido binario.

// This example uses the platform's default charset, see below
// for a better implementation.

Writer writer = new OutputStreamWriter(System.out);
writer.write("Hello world!");

Reader reader = new InputStreamReader(System.in);
char singleCharacter = reader.read();

Siempre que sea necesario codificar caracteres en datos binarios (por ejemplo, al usar las clases InputStreamWriter / OutputStreamWriter), debe especificar un conjunto de caracteres si no desea depender del conjunto de caracteres predeterminado de la plataforma. En caso de duda, utilice una codificación compatible con Unicode, p. UTF-8 que es compatible con todas las plataformas Java. Por lo tanto, probablemente debería mantenerse alejado de clases como FileWriter y FileReader ya que siempre usan el juego de caracteres predeterminado de la plataforma. Una mejor manera de acceder a los archivos usando secuencias de caracteres es esta:

Charset myCharset = StandardCharsets.UTF_8;

Writer writer = new OutputStreamWriter( new FileOutputStream("test.txt"), myCharset );
writer.write('Ä');
writer.flush();
writer.close();

Reader reader = new InputStreamReader( new FileInputStream("test.txt"), myCharset );
char someUnicodeCharacter = reader.read();
reader.close();

Uno de los ‘Reader’ más utilizados es ‘BufferedReader’, que proporciona un método para leer líneas completas de texto de otro lector y, presumiblemente, es la forma más sencilla de leer un flujo de caracteres línea por línea:

// Read from baseReader, one line at a time
BufferedReader reader = new BufferedReader( baseReader );
String line;
while((line = reader.readLine()) != null) {
  // Remember: System.out is a stream, not a writer!
  System.out.println(line);
}

Copiar un archivo usando InputStream y OutputStream #

Podemos copiar datos directamente de una fuente a un receptor de datos mediante un bucle. En este ejemplo, estamos leyendo datos de un InputStream y, al mismo tiempo, escribiendo en un OutputStream. Una vez que hayamos terminado de leer y escribir, tenemos que cerrar el recurso.

public void copy(InputStream source, OutputStream destination) throws IOException {
    try {
        int c;
        while ((c = source.read()) != -1) {
            destination.write(c);
        }
    } finally {
        if (source != null) {
            source.close();
        }
        if (destination != null) {
            destination.close();
        }
    }
}

Leer un archivo usando Channel y Buffer #

Channel usa un Buffer para leer/escribir datos. Un búfer es un contenedor de tamaño fijo donde podemos escribir un bloque de datos a la vez. Channel es bastante más rápido que la E/S basada en secuencias.

Para leer datos de un archivo usando Channel necesitamos tener los siguientes pasos-

  1. Necesitamos una instancia de FileInputStream. FileInputStream tiene un método llamado getChannel() que devuelve un canal.

  2. Llame al método getChannel() de FileInputStream y adquiera Channel.

  3. Cree un ByteBuffer. ByteBuffer es un contenedor de bytes de tamaño fijo.

  4. El canal tiene un método de lectura y debemos proporcionar un ByteBuffer como argumento para este método de lectura. ByteBuffer tiene dos modos: estado de ánimo de solo lectura y estado de ánimo de solo escritura. Podemos cambiar el modo usando la llamada al método flip(). El búfer tiene una posición, un límite y una capacidad. Una vez que se crea un búfer con un tamaño fijo, su límite y capacidad son los mismos que el tamaño y la posición comienza desde cero. Mientras se escribe un búfer con datos, su posición aumenta gradualmente. Cambiar de modo significa cambiar la posición. Para leer datos desde el comienzo de un búfer, tenemos que establecer la posición en cero. método flip() cambia la posición

  5. Cuando llamamos al método de lectura del Channel, llena el búfer usando datos.

  6. Si necesitamos leer los datos del ‘ByteBuffer’, debemos voltear el búfer para cambiar su modo de solo escritura a modo de solo lectura y luego seguir leyendo los datos del búfer.

  7. Cuando ya no hay datos para leer, el método read() del canal devuelve 0 o -1.

    import java.io.File; import java.io.FileInputStream; import java.io.IOException; import java.nio.ByteBuffer; import java.nio.channels.FileChannel;

    public class FileChannelRead {

    public static void main(String[] args) {

    File inputFile = new File("hello.txt");

if (!inputFile.exists()) {
 System.out.println("The input file doesn't exit.");
 return;
}

    try { FileInputStream fis = new FileInputStream(inputFile); FileChannel fileChannel = fis.getChannel(); ByteBuffer buffer = ByteBuffer.allocate(1024);

    while (fileChannel.read(buffer) > 0) {
 buffer.flip();
 while (buffer.hasRemaining()) {
  byte b = buffer.get();
  System.out.print((char) b);
 }
 buffer.clear();
}

fileChannel.close();

    } catch (IOException e) { e.printStackTrace(); } } }

Bloqueo o redireccionamiento de salida/error estándar #

A veces, una biblioteca de terceros mal diseñada escribirá diagnósticos no deseados en las secuencias System.out o System.err. Las soluciones recomendadas para esto serían encontrar una biblioteca mejor o (en el caso de código abierto) solucionar el problema y contribuir con un parche a los desarrolladores.

Si las soluciones anteriores no son factibles, entonces debería considerar redirigir las secuencias.

Redirección en la línea de comando

En un sistema UNIX, Linux o MacOSX se puede hacer desde el shell usando la redirección >. Por ejemplo:

$ java -jar app.jar arg1 arg2 > /dev/null 2>&1
$ java -jar app.jar arg1 arg2 > out.log 2> error.log

El primero redirige la salida estándar y el error estándar a “/dev/null”, que descarta cualquier cosa escrita en esos flujos. El segundo de redirige la salida estándar a “out.log” y el error estándar a “error.log”.

(Para obtener más información sobre la redirección, consulte la documentación del shell de comandos que está utilizando. Se aplican consejos similares a Windows).

Como alternativa, podría implementar la redirección en un script contenedor o un archivo por lotes que inicie la aplicación Java.

Redireccionamiento dentro de una aplicación Java

También es posible redirigir los flujos dentro de una aplicación Java utilizando System.setOut() y System.setErr(). Por ejemplo, el siguiente fragmento de código redirige la salida estándar y el error estándar a 2 archivos de registro:

System.setOut(new PrintStream(new FileOutputStream(new File("out.log"))));
System.setErr(new PrintStream(new FileOutputStream(new File("err.log"))));

Si desea desechar la salida por completo, puede crear un flujo de salida que “escriba” en un descriptor de archivo no válido. Esto es funcionalmente equivalente a escribir en “/dev/null” en UNIX.

System.setOut(new PrintStream(new FileOutputStream(new FileDescriptor())));
System.setErr(new PrintStream(new FileOutputStream(new FileDescriptor())));

Precaución: tenga cuidado al usar setOut y setErr:

  1. La redirección afectará a toda la JVM.
  2. Al hacer esto, le está quitando al usuario la capacidad de redirigir las secuencias desde la línea de comando.
