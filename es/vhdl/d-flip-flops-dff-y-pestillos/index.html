D-Flip-Flops (DFF) y pestillos son elementos de memoria. Un DFF muestrea su entrada en uno u otro borde de su reloj (no en ambos) mientras que un latch es transparente en un nivel de su habilitación y memorización en el otro. La siguiente figura ilustra la diferencia:

Modelar DFF o latches en VHDL es fácil, pero hay algunos aspectos importantes que deben tenerse en cuenta:

  • Las diferencias entre los modelos VHDL de DFF y latches.
  • Cómo describir los bordes de una señal.
  • Cómo describir los set o resets síncronos o asíncronos.

Chanclas D (DFF) #

En todos los ejemplos:

  • clk es el reloj,
  • d es la entrada,
  • q es la salida,
  • srst es un restablecimiento síncrono alto activo,
  • srstn es un restablecimiento síncrono bajo activo,
  • arst es un reinicio asíncrono alto activo,
  • arstn es un restablecimiento asíncrono bajo activo,
  • sset es un conjunto sincrónico alto activo,
  • ssetn es un conjunto síncrono bajo activo,
  • aset es un conjunto asíncrono alto activo,
  • asetn es un conjunto asíncrono bajo activo

Todas las señales son del tipo ieee.std_logic_1164.std_ulogic. La sintaxis utilizada es la que conduce a resultados de síntesis correctos con todos los sintetizadores lógicos. Consulte el ejemplo de Detección de borde de reloj para ver una discusión sobre la sintaxis alternativa.

Reloj de borde ascendente #

process(clk)
begin
  if rising_edge(clk) then
    q <= d;
  end if;
end process;

Reloj de borde descendente #

process(clk)
begin
  if falling_edge(clk) then
    q <= d;
  end if;
end process;

Reloj de borde ascendente, reinicio alto activo síncrono #

process(clk)
begin
  if rising_edge(clk) then
    if srst = '1' then
      q <= '0';
    else
      q <= d;
    end if;
  end if;
end process;

Reloj de borde ascendente, reinicio alto activo asíncrono #

process(clk, arst)
begin
  if arst = '1' then
    q <= '0';
  elsif rising_edge(clk) then
    q <= d;
  end if;
end process;

Reloj de flanco descendente, restablecimiento bajo activo asíncrono, conjunto alto activo síncrono #

process(clk, arstn)
begin
  if arstn = '0' then
    q <= '0';
  elsif falling_edge(clk) then
    if sset = '1' then
      q <= '1';
    else
      q <= d;
    end if;
  end if;
end process;

Reloj de flanco ascendente, reinicio alto activo asíncrono, conjunto bajo activo asíncrono #

Nota: establecer tiene mayor prioridad que restablecer

process(clk, arst, asetn)
begin
  if asetn = '0' then
    q <= '1';
  elsif arst = '1' then
    q <= '0';
  elsif rising_edge(clk) then
    q <= d;
  end if;
end process;

Pestillos #

En todos los ejemplos:

  • en es la señal de habilitación,
  • d es la entrada,
  • q es la salida,
  • srst es un restablecimiento síncrono alto activo,
  • srstn es un restablecimiento síncrono bajo activo,
  • arst es un reinicio asíncrono alto activo,
  • arstn es un restablecimiento asíncrono bajo activo,
  • sset es un conjunto sincrónico alto activo,
  • ssetn es un conjunto síncrono bajo activo,
  • aset es un conjunto asíncrono alto activo,
  • asetn es un conjunto asíncrono bajo activo

Todas las señales son del tipo ieee.std_logic_1164.std_ulogic. La sintaxis utilizada es la que conduce a resultados de síntesis correctos con todos los sintetizadores lógicos. Consulte el ejemplo de Detección de borde de reloj para ver una discusión sobre la sintaxis alternativa.

Habilitación alta activa #

process(en, d)
begin
  if en = '1' then
    q <= d;
  end if;
end process;

Habilitación baja activa #

process(en, d)
begin
  if en = '0' then
    q <= d;
  end if;
end process;

Habilitación alta activa, restablecimiento alto activo síncrono #

process(en, d)
begin
  if en = '1' then
    if srst = '1' then
      q <= '0';
    else
      q <= d;
    end if;
  end if;
end process;

Habilitación alta activa, restablecimiento alto activo asíncrono #

process(en, d, arst)
begin
  if arst = '1' then
    q <= '0';
  elsif en = '1' then
    q <= d;
  end if;
end process;

Habilitación baja activa, restablecimiento bajo activo asíncrono, conjunto alto activo síncrono #

process(en, d, arstn)
begin
  if arstn = '0' then
    q <= '0';
  elsif en = '0' then
    if sset = '1' then
      q <= '1';
    else
      q <= d;
    end if;
  end if;
end process;

Habilitación alta activa, restablecimiento alto activo asíncrono, conjunto bajo activo asíncrono #

Nota: establecer tiene mayor prioridad que restablecer

process(en, d, arst, asetn)
begin
  if asetn = '0' then
    q <= '1';
  elsif arst = '1' then
    q <= '0';
  elsif en = '1' then
    q <= d;
  end if;
end process;

Detección de borde de reloj #

La historia corta #

Con VHDL 2008 y si el tipo de reloj es bit, booleano, ieee.std_logic_1164.std_ulogic o ieee.std_logic_1164.std_logic, se puede codificar una detección de flanco de reloj para flanco ascendente

  • si arista_borde(reloj) entonces
  • si el evento del reloj y el reloj = '1', entonces -- escriba bit, std_ulogic o std_logic
  • if clock'event and clock then -- type boolean

y por borde descendente

  • si cae_borde(reloj) entonces
  • si el evento del reloj y el reloj = '0', entonces -- escriba bit, std_ulogic o std_logic
  • if clock'event and not clock then -- type boolean

Esto se comportará como se esperaba, tanto para la simulación como para la síntesis.

Nota: la definición de un flanco ascendente en una señal de tipo std_ulogic es un poco más compleja que el simple evento if clock' y clock = '1' then. La función estándar rising_edge, por ejemplo, tiene una definición diferente. Incluso si probablemente no hará ninguna diferencia para la síntesis, podría hacer una para la simulación.

Se recomienda encarecidamente el uso de las funciones estándar rising_edge y falling_edge. Con versiones anteriores de VHDL, el uso de estas funciones puede requerir declarar explícitamente el uso de paquetes estándar (por ejemplo, ieee.numeric_bit para el tipo bit) o incluso definirlos en un paquete personalizado.

Nota: no utilice las funciones estándar rising_edge y falling_edge para detectar bordes de señales que no son de reloj. Algunos sintetizadores podrían concluir que la señal es un reloj. Sugerencia: la detección de un borde en una señal que no es de reloj se puede realizar con frecuencia muestreando la señal en un registro de desplazamiento y comparando los valores muestreados en diferentes etapas del registro de desplazamiento.

La larga historia #

La descripción adecuada de la detección de los bordes de una señal de reloj es esencial al modelar D-Flip-Flops (DFF). Un borde es, por definición, una transición de un valor particular a otro. Por ejemplo, podemos definir el flanco ascendente de una señal de tipo bit (el tipo enumerado VHDL estándar que toma dos valores: '0' y '1'') como la transición de ‘0’a‘1’’. Para el tipo booleanopodemos definirlo como una transición defalsoaverdadero`.

Con frecuencia, se utilizan tipos más complejos. El tipo ieee.std_logic_1164.std_ulogic, por ejemplo, también es un tipo enumerado, como bit o boolean, pero tiene 9 valores en lugar de 2:

Valor significado
'U' Sin inicializar
'X' Obligando desconocido
'0' Forzando bajo nivel
'1' Obligando a alto nivel
'Z' Alta impedancia
'W' Débil desconocido
'L' Bajo nivel débil
'H' Alto nivel débil
'-' no me importa

Definir un flanco ascendente en un tipo de este tipo es un poco más complejo que para bit o boolean. Podemos, por ejemplo, decidir que es una transición de '0' a '1'. Pero también podemos decidir que es una transición de '0' o 'L' a '1' o 'H'.

Nota: es esta segunda definición la que usa el estándar para la función rising_edge(signal s: std_ulogic) definida en ieee.std_logic_1164.

Cuando se analizan las diversas formas de detectar bordes, es importante considerar el tipo de señal. También es importante tener en cuenta el objetivo del modelado: ¿solo simulación o síntesis lógica? Ilustremos esto con algunos ejemplos:

Borde ascendente DFF con tipo bit #

signal clock, d, q: bit;
...
P1: process(clock)
begin
  if clock = '1' then
    q <= d;
  end if;
end process P1;

Técnicamente, desde el punto de vista de la semántica de simulación pura, el proceso ‘P1’ modela un DFF activado por un flanco ascendente. De hecho, la asignación q <= d se ejecuta si y solo si:

  • reloj cambiado (esto es lo que expresa la lista de sensibilidad) y
  • el valor actual de reloj es '1'.

Como reloj es de tipo bit y tipo bit tiene solo valores '0' y '1', esto es exactamente lo que definimos como un flanco ascendente de una señal de tipo bit. Cualquier simulador manejará este modelo como esperamos.

Nota: Para los sintetizadores lógicos, las cosas son un poco más complejas, como veremos más adelante.

Flanco ascendente DFF con reinicio alto activo asíncrono y bit de tipo #

Para agregar un reinicio alto activo asíncrono a nuestro DFF, uno podría intentar algo como:

signal clock, reset, d, q: bit;
...
P2_BOGUS: process(clock, reset)
begin
  if reset = '1' then
    q <= '0';
  elsif clock = '1' then
    q <= d;
  end if;
end process P2_BOGUS;

Pero esto no funciona__. La condición para que se ejecute la asignación q <= d debe ser: un flanco ascendente de reloj mientras que reset = '0'. Pero lo que modelamos es:

  • reloj o reinicio o ambos cambiaron y
  • restablecer = '0' y
  • reloj = '1'

Lo cual no es lo mismo: si reset cambia de '1' a '0' mientras que clock = '1' la asignación se ejecutará mientras no sea un flanco ascendente de clock.

De hecho, no hay forma de modelar esto en VHDL sin la ayuda de un atributo de señal:

P2_OK: process(clock, reset)
begin
  if reset = '1' then
    q <= '0';
  elsif clock = '1' and clock'event then
    q <= d;
  end if;
end process P2_OK;

El reloj'evento es el atributo de señal evento aplicado a la señal reloj. Se evalúa como un ‘booleano’ y es ‘verdadero’ si y solo si la señal ‘reloj’ cambió durante la fase de actualización de la señal que precedió a la fase de ejecución actual. Gracias a esto, el proceso P2_OK ahora modela perfectamente lo que queremos en simulación (y síntesis).

Síntesis semántica #

Muchos sintetizadores lógicos identifican las detecciones de borde de señal en función de patrones sintácticos, no de la semántica del modelo VHDL. En otras palabras, consideran cómo se ve el código VHDL, no qué comportamiento modela. Uno de los patrones que todos reconocen es:

if clock = '1' and clock'event then

Entonces, incluso en el ejemplo del proceso P1 deberíamos usarlo si queremos que nuestro modelo sea sintetizable por todos los sintetizadores lógicos:

signal clock, d, q: bit;
...
P1_OK: process(clock)
begin
  if clock = '1' and clock'event then
    q <= d;
  end if;
end process P1_OK;

La parte y el evento del reloj de la condición es completamente redundante con la lista de sensibilidad, pero como algunos sintetizadores la necesitan…

DFF de flanco ascendente con reinicio alto activo asincrónico y tipo std_ulogic #

En este caso, expresar el flanco ascendente del reloj y la condición de reinicio puede resultar complicado. Si conservamos la definición de flanco ascendente que propusimos anteriormente y si consideramos que el reset está activo si es '1' o 'H', el modelo se convierte en:

library ieee;
use ieee.std_logic_1164.all;
...
signal clock, reset, d, q: std_ulogic;
...
P4: process(clock, reset)
begin
  if reset = '1' or reset = 'H' then
    q <= '0';
  elsif clock'event and
        (clock'last_value = '0' or clock'last_value = 'L') and
        (clock = '1' or clock = 'H') then
    q <= d;
  end if;
end process P4;

Nota: 'last_value es otro atributo de la señal que devuelve el valor que tenía la señal antes del último cambio de valor.

Funciones auxiliares #

El estándar VHDL 2008 ofrece varias funciones auxiliares para simplificar la detección de bordes de señal, especialmente con tipos enumerados de valores múltiples como std_ulogic. El paquete std.standard define las funciones rising_edge y falling_edge en los tipos bit y boolean y el paquete ieee.std_logic_1164 las define en los tipos std_ulogic y std_logic.

Nota: con versiones anteriores de VHDL, el uso de estas funciones puede requerir declarar explícitamente el uso de paquetes estándar (por ejemplo, ieee.numeric_bit para tipo bit) o ​​incluso definirlos en un paquete de usuario.

Repasemos los ejemplos anteriores y usemos las funciones auxiliares:

signal clock, d, q: bit;
...
P1_OK_NEW: process(clock)
begin
  if rising_edge(clock) then
    q <= d;
  end if;
end process P1_OK_NEW;

signal clock, d, q: bit;
...
P2_OK_NEW: process(clock, reset)
begin
  if reset = '1' then
    q <= '0';
  elsif rising_edge(clock) then
    q <= d;
  end if;
end process P2_OK_NEW;

library ieee;
use ieee.std_logic_1164.all;
...
signal clock, reset, d, q: std_ulogic;
...
P4_NEW: process(clock, reset)
begin
  if reset = '1' then
    q <= '0';
  elsif rising_edge(clock) then
    q <= d;
  end if;
end process P4_NEW;

Nota: en este último ejemplo también simplificamos la prueba en el reinicio. Los reinicios flotantes de alta impedancia son bastante raros y, en la mayoría de los casos, esta versión simplificada funciona para simulación y síntesis.
