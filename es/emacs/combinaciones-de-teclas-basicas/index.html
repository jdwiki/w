Salir de Emacs #

Puede salir de Emacs con la siguiente combinación de teclas:

C-x C-c

Donde C es la tecla control.

Suspender Emacs

Puede suspender Emacs utilizando la siguiente combinación de teclas:

C-z

Te lleva de vuelta a tu caparazón. Si desea reanudar su sesión de emacs, ingrese fg en su terminal.

Cancelar el comando actual #

A menudo, entrará en un estado en el que tiene una secuencia de comandos parcialmente escrita en curso, pero desea cancelarla. Puede cancelarlo con cualquiera de las siguientes combinaciones de teclas:

C-g

EscEscEsc

Tampones #

  • Ejemplo de una lista de búfer

    CRM Buffer                Size  Mode              Filename[/Process]
. * .emacs                3294  Emacs-Lisp        ~/.emacs
 %  *Help*                 101  Help
    search.c             86055  C                 ~/cvs/emacs/src/search.c
 %  src                  20959  Dired by name     ~/cvs/emacs/src/
  * *mail*                  42  Mail
 %  HELLO                 1607  Fundamental       ~/cvs/emacs/etc/HELLO
 %  NEWS                481184  Outline           ~/cvs/emacs/etc/NEWS
    *scratch*              191  Lisp Interaction
  * *
Messages*            1554  Messages

    The first field of a line indicates:

    • ‘.’ the buffer is current.
    • ‘%’ a read-only buffer.
    • ‘*’ the buffer is modified.

  • Seleccionar búfer. Puede seleccionar cualquier búfer abierto con la siguiente combinación de teclas:

C-x b

Se le pedirá el nombre del búfer al que desea cambiar.

  • Lista de búferes:

    C-x C-b

  • Save-some-buffer, dando la opción de qué buffer guardar o no:

    C-x s

  • Matar un búfer:

    C-x k

  • Operaciones sobre tampones marcados:

    <kbd>S</kbd>  Save the marked buffers                                                                  

<kbd>A</kbd>  View the marked buffers in this frame.                                                   

<kbd>H</kbd>  View the marked buffers in another frame.                                                

<kbd>V</kbd> Revert the marked buffers.                                                               

<kbd>T</kbd> Toggle read-only state of marked buffers.                                                

<kbd>D</kbd> Kill the marked buffers.                                                                 

<kbd>M-s a C-s</kbd> Do incremental search in the marked buffers.                                     

<kbd>M-s a C-M-s</kbd> Isearch for regexp in the marked buffers.                                      

<kbd>U</kbd> Replace by regexp in each of the marked buffers.                                                                                                                  

<kbd>Q</kbd> Query replace in each of the marked buffers.                                             

<kbd>I</kbd> As above, with a regular expression.                                                     

<kbd>P</kbd> Print the marked buffers.                                                                

<kbd>O</kbd> List lines in all marked buffers which match a given regexp (like the function `occur').                                                                                    

<kbd>X</kbd> Pipe the contents of the marked buffers to a shell command.                                                   

<kbd>N</kbd> Replace the contents of the marked buffers with the output of a shell command.  

<kbd>!</kbd> Run a shell command with the buffer's file as an argument.                                                             

<kbd>E</kbd> Evaluate a form in each of the marked buffers. This is a very flexible command.  For example, if you want to make all of the marked buffers read only, try using (read-only-mode 1) as the input form.                                  

<kbd>W</kbd> - As above, but view each buffer while the form is evaluated.                                           

<kbd>k</kbd> - Remove the marked lines from the *Ibuffer* buffer, but don't kill the associated buffer.                                      

<kbd>x</kbd> - Kill all buffers marked for deletion.

  • Save-some-buffer, dando la opción de qué buffer guardar o no:

    C-x s

  • Cambiar al siguiente búfer:

C-x DERECHA

  • Cambiar al búfer anterior:

C-x IZQUIERDA

Manejo de archivos #

  • Vuelva a guardar el archivo abierto con el mismo nombre de archivo (Guardar):

    C-x C-s

  • Escribir como nombre de archivo (Guardar como):

    C-x C-w filename

El nuevo nombre de archivo aparecerá en el minibúfer.

  • Crear archivo nuevo o cargar archivo existente (Nuevo / Cargar):

    C-x C-f filename

    With the mnemonic here for f meaning file. You will be prompted for a file path in the minibuffer.

  • Visitar archivo alternativo

    C-x C-f

    If the file does not exist yet, you will be prompted the path of the file to create in the minibuffer.

Movimiento del cursor (punto) #

Además de los movimientos del cursor con las teclas de flecha, Inicio, Fin, Re Pág y Re Pág, emacs define una cantidad de pulsaciones de teclas que pueden mover el cursor sobre fragmentos de texto más pequeños o más grandes:

Por personaje:

  • Carácter al revés: C-b
  • Carácter de reenvío: C-f

Por palabra

  • Palabra inversa: M-b (es decir, Alt b, o Meta b)
  • Reenviar palabra: M-f

Nombre del autor:

  • Comienzo de la línea actual: C-a
  • Comienzo de la línea actual primer carácter (sin espacio): M-m
  • Fin de la línea actual: C-e
  • Línea anterior: C-p
  • Siguiente línea: C-n

Búfer completo:

  • Comienzo del búfer: M-
  • Fin del búfer: M->

Por ‘bloque’, según el contexto (modo):

Combinaciones de teclas típicas:

  • Oración/afirmación al revés: M-a
  • Reenviar oración/declaración: M-e
  • Comienzo de la función: M-C-a
  • Fin de función: M-C-e

Argumentos de prefijo

Para mover varios ‘pasos’ a la vez, los comandos de movimiento pueden recibir un argumento de prefijo presionando ESC o C-u y un número antes de las pulsaciones de teclas enumeradas. Para C-u, el número es opcional y el valor predeterminado es 4. Ej. ESC 3 C-n se mueve 3 líneas hacia abajo, mientras que C-u M-f se mueve 4 palabras hacia adelante.

Notación de enlaces de teclas #

La documentación de Emacs usa una notación consistente para todas las combinaciones de teclas, que se explica aquí:

Acordes clave #

Un “acorde clave” se obtiene presionando dos o más teclas simultáneamente. Los acordes clave se denotan separando todas las claves con guiones (-). Por lo general, involucran teclas modificadoras, que se colocan al frente:

  • C-: controlar;
  • S-: turno;
  • M-: alt (la “M” significa “Meta” por razones históricas).

Otras teclas se indican simplemente por su nombre, como:

  • a: la tecla a;
  • izquierda: la tecla de flecha izquierda;
  • SPC: la tecla de espacio;
  • RET: la tecla de retorno.

Los ejemplos de acordes clave incluyen:

  • C-a: presionando control y a simultáneamente;
  • S-derecha: presionando shift y derecha simultáneamente;
  • C-M-a: presionando control, alt y a simultáneamente.

Secuencias de teclas #

Las “secuencias de teclas” son secuencias de teclas (o acordes de teclas), que deben escribirse una tras otra. Se denotan separando todas las notaciones de clave (o acorde) por un espacio.

Ejemplos incluyen:

  • C-x b: presionando control y x simultáneamente, luego soltándolos y presionando b;
  • C-x C-f: presionando control y x simultáneamente, luego soltando x y presionando f (ya que ambos acordes involucran el modificador control, no es necesario soltarlo).

Usar ESC en lugar de Alt #

Los acordes de tecla que usan el modificador Alt también se pueden ingresar como una secuencia de teclas que comienza con ESC. Esto puede ser útil cuando se usa Emacs a través de una conexión remota que no transmite acordes de teclas Alt, o cuando estas combinaciones de teclas son capturadas, por ejemplo, por un administrador de ventanas.

Ejemplo:

M-x se puede ingresar como ESC x.

Describiendo enlaces de teclas en archivos lisp de Emacs #

La misma notación que se describe aquí se puede utilizar al definir asociaciones de teclas en archivos lisp de Emacs.

Ejemplo:

(clave de configuración global (kbd “C-x C-b”) ‘menú de búfer) une la secuencia de teclas C-x C-b al comando buffer-menu

Múltiples ventanas o marcos #

“Ventana” en Emacs se refiere a lo que de otro modo podría llamarse un “panel” o “división de pantalla”. Algunos comandos de manipulación de ventanas incluyen:

  • Dividir la ventana actual horizontalmente: C-x 2
  • Dividir la ventana actual verticalmente: C-x 3
  • Seleccionar ventana siguiente: C-x o
  • Cerrar ventana actual: C-x 0
  • Cerrar todas las demás ventanas, excepto la actual: C-x 1

Un “marco” en Emacs es lo que de otro modo podría llamarse una “ventana”. Los marcos se manipulan usando estos comandos:

  • Crear nuevo cuadro: C-x 5 2
  • Eliminar cuadro actual: C-x 5 0
  • Eliminar otros fotogramas: C-x 5 1

El cambio de ventanas se puede lograr usando

  • S-izquierda, S-derecha, S-arriba, S-abajo (es decir, Shift junto con una tecla de flecha) para cambiar a la ventana vecina en una dirección, o
  • C-x o para cambiar a la siguiente ventana.

Región - Cortar, Copiar, Pegar #

  • Establecer marca en la ubicación del cursor:

    C-space or C-@

  • Matar región (Cortar):

    C-w

  • Copiar región para matar anillo:

    M-w or Esc-w

  • Yank (Pegar) matado más recientemente:

    C-y

  • Yank (Pegar) siguiente último asesinado:

    M-y or Esc-y

Matar #

kill es el comando usado por Emacs para borrar texto. El comando matar es análogo al comando cortar en Windows. Existen varios comandos que ‘matan’ una palabra (M-d), el resto de la línea (C-k) o bloques de texto más grandes. El texto borrado se agrega al canto de eliminación, del cual luego se puede sacar.

Seleccionar y cortar (matar) #

Matar y tirar De forma similar a la función seleccionar y cortar de Windows, aquí tenemos C-spc. La combinación de teclas C-spc iniciará la selección, el usuario puede mover la marca con la ayuda de las teclas de flecha u otro comando para realizar una selección. Una vez completada la selección, presione C-w para eliminar el texto seleccionado

Algunos comandos básicos que se pueden usar como referencia rápida para el comando kill (tomados del tutorial de Emacs) M-DEL Kill the word immediately before the cursor M-d Kill the next word after the cursor

C-k         Kill from the cursor position to end of line
M-k         Kill to the end of the current sentence

Tirón #

yank describe la inserción de texto eliminado previamente, por ejemplo, usando C-y que extrae el texto eliminado más recientemente. El comando Yank es análogo al comando pegar en Windows.

Yank texto eliminado anteriormente #

Sabemos que el comando kill agrega el texto asesinado a un kill-ring. Para recuperar el texto eliminado del kill-ring, use el comando M-y repetidamente hasta que se extraiga el texto deseado.

(Nota: para que la tecla M-y funcione, el comando anterior debe ser YANK, de lo contrario no funcionaría)

Deshacer #

Para deshacer algo que acabas de hacer:

C-_ o C-x u o C-/

Buscar y reemplazar #

En Emacs, la herramienta de búsqueda básica (I-Search) le permite buscar antes o después de la ubicación de su cursor.

  • Para buscar sometext después de la ubicación de su cursor (search-forward) pulse C-s sometext. Si desea ir a la siguiente ocurrencia de sometext, simplemente presione C-s nuevamente (y así sucesivamente para las próximas ocurrencias). Cuando el cursor llegue a la ubicación correcta, presione Enter para salir del mensaje de búsqueda.

  • Para buscar antes de la ubicación de su cursor (search-backward), use C-r de la misma manera que usó antes.

  • Para cambiar de búsqueda hacia atrás a búsqueda hacia adelante, presione 2 veces C-s. Y presione 2 veces C-r para ‘buscar hacia atrás’ cuando esté en el indicador de ‘búsqueda hacia adelante’.

  • Buscar y reemplazar:

    M-% (or Esc-% ) oldtext Enter newtext Enter

    • Confirm: y
    • Skip: n
    • Quit: q
    • Replace all: !

Caso #

  • Capitalizar palabra: M-c

  • Convertir palabra a mayúscula: M-u

  • Convertir palabra a minúsculas: M-l
