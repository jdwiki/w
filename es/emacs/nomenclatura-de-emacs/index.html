Elementos de la interfaz de usuario #

La interfaz de usuario de Emacs utiliza términos que se acuñaron antes y pueden resultar inquietantes para los usuarios acostumbrados a una terminología más moderna.

Cuadro #

En Emacs, lo que se denomina ventana (el área de la pantalla utilizada por un programa) se denomina marco. Emacs comienza usando un marco, aunque se pueden crear marcos adicionales usando C-x 5.

Ventana #

Un marco contiene una o más ventanas (también conocidas como paneles), cada una de las cuales muestra el contenido de un búfer. Cada marco generalmente comienza con una sola ventana, pero se pueden crear ventanas adicionales dividiendo las existentes; ya sea horizontalmente usando C-x 2 o verticalmente con C-x 3. Consulte también https://www.wikiod.com/es/emacs/combinaciones-de-teclas-basicas#Múltiples ventanas o marcos

Buffer #

El término búfer se refiere al contenido que se muestra en una ventana. Dicho contenido puede reflejar el contenido de un archivo en el sistema de archivos (o tal vez una versión actualizada que aún no se ha guardado en el disco), pero en general puede ser cualquier tipo de texto.

Línea de modo #

En la parte inferior de cada ventana hay una línea de modo, que describe sintéticamente el búfer que se muestra en la ventana.

Barra de herramientas #

De manera similar a muchos otros programas, se puede mostrar una barra de herramientas en la parte superior de cada marco. Su contenido puede variar según el tipo de búfer que se esté editando actualmente.

minibúfer #

Un minibúfer, que normalmente se muestra en la parte inferior de cada fotograma, permite interactuar con Emacs. Cada vez que un comando solicita una entrada del usuario, se solicita en el minibúfer. Por el contrario, los mensajes que se muestran para que los vea el usuario se imprimen allí.

Punto, marca y región #

Emacs usa los términos punto, marca y región para proporcionar más precisión sobre el texto seleccionado y la posición del cursor. Al comprender estos términos, lo ayudará a comprender y utilizar otras operaciones y funciones.

El punto es el lugar en un búfer donde se está realizando la edición (es decir, la inserción) y generalmente se indica mediante un cursor. La marca es un marcador que se coloca en cualquier parte del búfer usando comandos como set-mark-command (C-SPC) o exchange-point-and-mark ( C-x C-x). La región es el área entre el punto y la marca, y muchos comandos operan en la región para p. ej. eliminarla, corregir la ortografía, aplicar sangría o compilarla.

Cuando hace clic con el mouse en una ubicación en un búfer, está viendo el punto. Cuando selecciona texto, está configurando la región (el texto seleccionado) y la marca (al comienzo de su selección).

Archivos y búferes #

En Emacs, archivo tiene el mismo significado que en el sistema operativo y se utiliza para el almacenamiento permanente de datos. Un búfer es la representación interna de un archivo que se está editando. Los archivos se pueden leer en los búferes usando C-x C-f, y los búferes se pueden escribir en los archivos usando C-x C-s (guardar el archivo en su ubicación actual) o C-x C-w</ kbd> (escribe el archivo en una ubicación diferente, solicitándolo - el equivalente a Guardar como).

Matar y tirar #

Matar y tirar corresponden más o menos a lo que se suele llamar “cortar” y “pegar”.

Matar #

matar significa borrar texto y copiarlo en kill-ring (que podría verse como una especie de “portapapeles” en la terminología de “cortar y pegar”). El kill ring se llama así porque almacena varios fragmentos de texto killed, a los que luego se puede acceder en orden cíclico.

Existen varios comandos que matan una palabra (M-d), el resto de la línea (C-k) o bloques de texto más grandes (como la región seleccionada actualmente: C-w).

Existen otros comandos que guardan texto en kill ring, sin realmente matarlo (de manera similar a “copiar” en terminologías modernas). Por ejemplo, M-w, que actúa sobre la región actualmente seleccionada.

tirando #

Las entradas en el anillo de eliminación se pueden retirar más tarde a un búfer. Por lo general, se puede tirar del texto eliminado más recientemente usando p. ej. C-y (que es similar a la operación “pegar” en una terminología más moderna). Pero otros comandos pueden acceder y jalar entradas más antiguas del anillo de eliminación.

Modos #

Modo mayor #

Emacs puede adaptar su comportamiento al tipo específico de texto editado en un búfer. El conjunto de personalizaciones específicas de Emacs para un tipo particular de texto se denomina “modo principal”. Cada búfer tiene exactamente un modo mayor dependiendo de su tipo de contenido.

Los modos principales pueden cambiar el significado de algunas teclas, definir reglas de sangría o resaltado de sintaxis e instalar nuevos enlaces de teclas (generalmente comenzando con C-c) para comandos específicos del modo. Emacs se envía con una amplia gama de modos principales, que se dividen en tres categorías principales:

  • soporte para texto (por ejemplo, lenguajes de marcas),
  • soporte para lenguajes de programación,
  • aplicaciones dentro de emacs (por ejemplo, dired, gnus, …). Los búferes que utilizan este último grupo de modos principales generalmente no están asociados a archivos, sino que sirven como una interfaz de usuario.

Modo menor #

Los modos menores son funciones opcionales que se pueden activar y desactivar. Los modos menores se pueden habilitar para búferes específicos (modos locales de búfer) o para todos los búferes (modos globales). A diferencia de los modos principales, se puede activar cualquier número de modos secundarios para un búfer determinado.

Emacs proporciona muchos modos menores. Algunos ejemplos incluyen:

  • Modo de relleno automático para ajustar automáticamente las líneas de texto a medida que escribe.
  • Modo Flyspell para resaltar los errores ortográficos mientras escribes.
  • Modo de línea visual para envolver líneas largas para que se ajusten a la pantalla.
  • Modo Transient Mark para resaltar la región actual.
