Sintaxis #

  • clase Some[+T](valor: T) extiende Opción[T]

  • objeto Ninguno extiende Opción[Nada]

  • Opción[T](valor: T)

    Constructor to create either a Some(value) or None as appropriate for the value provided.

Usar opción en lugar de nulo #

En Java (y otros lenguajes), usar null es una forma común de indicar que no hay ningún valor adjunto a una variable de referencia. En Scala, se prefiere usar Option a usar null. Option envuelve valores que podrían ser null.

Ninguno es una subclase de Opción que envuelve una referencia nula. Algunos es una subclase de Opción que envuelve una referencia no nula.

Envolver una referencia es fácil:

val nothing = Option(null) // None
val something = Option("Aren't options cool?") // Some("Aren't options cool?")

Este es un código típico cuando se llama a una biblioteca de Java que podría devolver una referencia nula:

val resource = Option(JavaLib.getResource())
// if null, then resource = None
// else resource = Some(resource)

Si getResource() devuelve un valor nulo, resource será un objeto Ninguno. De lo contrario, será un objeto Algún(recurso). La forma preferida de manejar una Opción es usando funciones de orden superior disponibles dentro del tipo Opción. Por ejemplo, si desea verificar si su valor no es Ninguno (similar a verificar si value == null), usaría la función isDefined:

val resource: Option[Resource] = Option(JavaLib.getResource())
if (resource.isDefined) {  // resource is `Some(_)` type
  val r: Resource = resource.get
  r.connect()
}

De manera similar, para verificar una referencia null, puede hacer esto:

val resource: Option[Resource] = Option(JavaLib.getResource())
if (resource.isEmpty) { // resource is `None` type.
  System.out.println("Resource is empty! Cannot connect.")
}

Es preferible que trate la ejecución condicional en el valor envuelto de una ‘Opción’ (sin usar el método ’excepcional’ ‘Option.get’) tratando la ‘Opción’ como una mónada y usando ‘foreach’:

val resource: Option[Resource] = Option(JavaLib.getResource())
resource foreach (r => r.connect())
// if r is defined, then r.connect() is run
// if r is empty, then it does nothing

Si se requiere una instancia de Recurso (en lugar de una instancia de Opción [Recurso]), aún puede usar Opción para protegerse contra valores nulos. Aquí el método getOrElse proporciona un valor predeterminado:

lazy val defaultResource = new Resource()
val resource: Resource = Option(JavaLib.getResource()).getOrElse(defaultResource)

El código Java no manejará fácilmente la ‘Opción’ de Scala, por lo que al pasar valores al código Java es una buena forma desenvolver una ‘Opción’, pasando ’null’ o un valor predeterminado sensato cuando corresponda:

val resource: Option[Resource] = ???
JavaLib.sendResource(resource.orNull)
JavaLib.sendResource(resource.getOrElse(defaultResource)) //

Conceptos básicos #

Una Opción es una estructura de datos que contiene un solo valor o ningún valor. Una ‘Opción’ se puede considerar como una colección de cero o un elemento.

Option es una clase abstracta con dos hijos: ‘Algunos’ y ‘Ninguno’.

Algunos contiene un único valor y Ninguno no contiene ningún valor.

Opción es útil en expresiones que de otro modo usarían null para representar la falta de un valor concreto. Esto protege contra NullPointerException y permite la composición de muchas expresiones que podrían no devolver un valor usando combinadores como Map, FlatMap, etc.

Ejemplo con mapa #

val countries = Map(
  "USA" -> "Washington",
  "UK" -> "London",
  "Germany" -> "Berlin",
  "Netherlands" -> "Amsterdam",
  "Japan" -> "Tokyo"
)
    
println(countries.get("USA")) // Some(Washington)
println(countries.get("France")) // None
println(countries.get("USA").get) // Washington
println(countries.get("France").get) // Error: NoSuchElementException
println(countries.get("USA").getOrElse("Nope")) // Washington
println(countries.get("France").getOrElse("Nope")) // Nope

La Opción[A] está sellada y, por lo tanto, no se puede ampliar. Por lo tanto, su semántica es estable y se puede confiar en ella.

Opciones como Colecciones #

Las ‘opciones’ tienen algunas funciones útiles de orden superior que se pueden entender fácilmente al ver las opciones como colecciones con cero o un elemento, donde ‘Ninguno’ se comporta como la colección vacía y ‘Algunos (x)’ se comporta como una colección con un elemento único, x.

val option: Option[String] = ???

option.map(_.trim) // None if option is None, Some(s.trim) if Some(s)
option.foreach(println) // prints the string if it exists, does nothing otherwise
option.forall(_.length > 4) // true if None or if Some(s) and s.length > 4
option.exists(_.length > 4) // true if Some(s) and s.length > 4
option.toList // returns an actual list

Opciones para comprensiones #

Options tiene un método flatMap. Esto significa que se pueden utilizar en una para la comprensión. De esta manera podemos levantar funciones regulares para trabajar en Opciones sin tener que redefinirlas.

val firstOption: Option[Int] = Option(1)
val secondOption: Option[Int] = Option(2)

val myResult = for {
  firstValue <- firstOption
  secondValue <- secondOption
} yield firstValue + secondValue
// myResult: Option[Int] = Some(3)

Cuando uno de los valores es “Ninguno”, el resultado final del cálculo también será “Ninguno”.

val firstOption: Option[Int] = Option(1)
val secondOption: Option[Int] = None

val myResult = for {
  firstValue <- firstOption
  secondValue <- secondOption
} yield firstValue + secondValue
// myResult: Option[Int] = None

Nota: este patrón se extiende de forma más general a los conceptos llamados Monads. (Más información debería estar disponible en las páginas relacionadas con para comprensiones y Monads)

En general, no es posible mezclar diferentes mónadas en una para la comprensión. Pero dado que ‘Option’ se puede convertir fácilmente en ‘Iterable’, podemos mezclar fácilmente ‘Option’ e ‘Iterable’ llamando al método ‘.toIterable’.

val option: Option[Int] = Option(1)
val iterable: Iterable[Int] = Iterable(2, 3, 4, 5)

// does NOT compile since we cannot mix Monads in a for comprehension
// val myResult = for {
//   optionValue <- option
//   iterableValue <- iterable
//} yield optionValue + iterableValue

// It does compile when adding a .toIterable on the option
val myResult = for {
  optionValue <- option.toIterable
  iterableValue <- iterable
} yield optionValue + iterableValue
// myResult: Iterable[Int] = List(2, 3, 4, 5)

Una pequeña nota: si hubiéramos definido nuestro for comprehension al revés, for comprehension compilaría ya que nuestra opción se convertiría implícitamente. Por esa razón, es útil agregar siempre este .toIterable (o la función correspondiente según la colección que esté usando) para mantener la coherencia.
