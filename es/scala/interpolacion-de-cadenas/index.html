Esta característica existe en Scala 2.10.0 y superior.

Usar expresión en literales de cadena #

Puede usar llaves para interpolar expresiones en literales de cadena:

def f(x: String) = x + x
val a = "A"

s"${a}"    // "A"
s"${f(a)}" // "AA"

Sin las llaves, Scala solo interpolaría el identificador después de $ (en este caso, f). Dado que no hay una conversión implícita de f a String, esta es una excepción en este ejemplo:

s"$f(a)"  // compile-time error (missing argument list for method f)

Hola interpolación de cadenas #

El interpolador s permite el uso de variables dentro de una cadena.

val name = "Brian"
println(s"Hello $name")

imprime “Hola Brian” en la consola cuando se ejecuta.

Interpolación de cadenas formateadas usando el interpolador f #

val num = 42d

Imprime dos lugares decimales para num usando f

println(f"$num%2.2f")
42.00

Imprime num usando notación científica usando e

println(f"$num%e")
4.200000e+01

Escribe num en hexadecimal con a

println(f"$num%a")
0x1.5p5

Se pueden encontrar otras cadenas de formato en https://docs.oracle.com/javase/6/docs/api/java/util/Formatter.html#detail

Interpoladores de cadenas como extractores #

También es posible utilizar la función de interpolación de cadenas de Scala para crear extractores elaborados (coincidencias de patrones), como quizás el más famoso empleado en la API de cuasicomillas de las macros de Scala.

Dado que n"p0${i0}p1" se convierte en new StringContext("p0", "p1").n(i0), tal vez no sea sorprendente que la función de extracción esté habilitada al proporcionar una conversión implícita de StringContext a una clase con propiedad n de un tipo que define un método unapply o unapplySeq.

Como ejemplo, considere el siguiente extractor que extrae segmentos de ruta al construir una expresión regular a partir de las partes StringContext. Entonces podemos delegar la mayor parte del trabajo pesado al método unapplySeq proporcionado por el scala.util.matching.Regex resultante:

implicit class PathExtractor(sc: StringContext) {
  object path {
    def unapplySeq(str: String): Option[Seq[String]] =
      sc.parts.map(Regex.quote).mkString("^", "([^/]+)", "$").r.unapplySeq(str)      
  }
}

"/documentation/scala/1629/string-interpolation" match {
  case path"/documentation/${topic}/${id}/${_}" => println(s"$topic, $id")
  case _ => ???
}

Tenga en cuenta que el objeto ruta también podría definir un método aplicar para comportarse también como un interpolador regular.

Interpoladores de cadenas personalizados #

Es posible definir interpoladores de cadenas personalizados además de los integrados.

my"foo${bar}baz"

Es expandido por el compilador a:

new scala.StringContext("foo", "baz").my(bar)

scala.StringContext no tiene el método my, por lo tanto, puede proporcionarse mediante conversión implícita. Un interpolador personalizado con el mismo comportamiento que el interpolador integrado s se implementaría de la siguiente manera:

implicit class MyInterpolator(sc: StringContext) {
  def my(subs: Any*): String = {
    val pit = sc.parts.iterator
    val sit = subs.iterator
    // Note parts.length == subs.length + 1
    val sb = new java.lang.StringBuilder(pit.next())
    while(sit.hasNext) {
      sb.append(sit.next().toString)
      sb.append(pit.next())          
    }
    sb.toString
  }
}

Y la interpolación my"foo${bar}baz" se convertiría en:

new MyInterpolation(new StringContext("foo", "baz")).my(bar)

Tenga en cuenta que no hay restricciones en los argumentos o el tipo de retorno de la función de interpolación. Esto nos lleva por un camino oscuro donde la sintaxis de interpolación se puede usar de manera creativa para construir objetos arbitrarios, como se ilustra en el siguiente ejemplo:

case class Let(name: Char, value: Int)

implicit class LetInterpolator(sc: StringContext) {
  def let(value: Int): Let = Let(sc.parts(0).charAt(0), value)
}

let"a=${4}" // Let(a, 4)
let"b=${"foo"}" // error: type mismatch
let"c=" // error: not enough arguments for method let: (value: Int)Let

Interpolación de cadenas sin procesar #

Puede usar el interpolador sin procesar si desea que una cadena se imprima tal cual y sin ningún tipo de escape de literales.


println(raw"Hola mundo en inglés y francés\nInglés:\tHola mundo\nFrancés:\t\tBonjour Le Monde")

Con el uso del interpolador sin procesar, debería ver lo siguiente impreso en la consola:


Hello World en inglés y francés\nInglés:\tHello World\nFrancés:\t\tBonjour Le Monde

Sin el interpolador sin procesar, \n y \t se habrían escapado.


println("Hola mundo en inglés y francés\nInglés:\tHola mundo\nFrancés:\t\tBonjour Le Monde")

Huellas dactilares:


hola mundo en ingles y frances
Inglés: hola mundo
Inglés: hola mundo
