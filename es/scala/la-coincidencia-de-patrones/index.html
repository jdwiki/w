Sintaxis #

  • función parcial de coincidencia del selector
  • coincidencia de selector {lista de alternativas de casos) // Esta es la forma más común de lo anterior

Parámetros #

Parámetro Detalles
seleccionador La expresión cuyo valor se compara con el patrón.
alternativas una lista de alternativas delimitadas por ‘casos’.

Coincidencia de patrón simple #

Este ejemplo muestra cómo hacer coincidir una entrada con varios valores:

def f(x: Int): String = x match {
  case 1 => "One"
  case 2 => "Two"
  case _ => "Unknown!"
}

f(2)  // "Two"
f(3)  // "Unknown!"

Demostración en vivo

Nota: _ es el caso falto o predeterminado, pero no es obligatorio.

def g(x: Int): String = x match {
  case 1 => "One"
  case 2 => "Two"
}

g(1)  // "One"
g(3)  // throws a MatchError

Para evitar lanzar una excepción, la mejor práctica de programación funcional aquí es manejar el caso predeterminado (caso _ => <hacer algo>). Tenga en cuenta que hacer coincidir una clase de caso puede ayudar al compilador a generar una advertencia si falta un caso. Lo mismo ocurre con los tipos definidos por el usuario que amplían un rasgo sellado. Si la coincidencia es total, es posible que no se necesite un caso predeterminado

También es posible hacer coincidir con valores que no están definidos en línea. Estos deben ser identificadores estables, que se obtienen usando un nombre en mayúsculas o encerrando comillas graves.

Con One y two definidos en otro lugar, o pasados ​​como parámetros de función:

val One: Int = 1
val two: Int = 2

Se pueden comparar de la siguiente manera:

def g(x: Int): String = x match {
  case One => "One"
  case `two` => "Two"
}

A diferencia de otros lenguajes de programación como Java, por ejemplo, no hay fallas. Si un bloque de casos coincide con una entrada, se ejecuta y finaliza la coincidencia. Por lo tanto, el caso menos específico debería ser el último bloque de casos.

def f(x: Int): String = x match {
  case _ => "Default"
  case 1 => "One"
}

f(5) // "Default"
f(1) // "Default"

Coincidencia de patrones en una secuencia #

Para comprobar un número preciso de elementos en la colección

def f(ints: Seq[Int]): String = ints match {
  case Seq() =>
      "The Seq is empty !"
  case Seq(first) =>
      s"The seq has exactly one element : $first"
  case Seq(first, second) =>
      s"The seq has exactly two elements : $first, $second"
  case  s @ Seq(_, _, _) => 
      s"s is a Seq of length three and looks like ${s}"  // Note individual elements are not bound to their own names.
  case s: Seq[Int] if s.length == 4 =>
      s"s is a Seq of Ints of exactly length 4"  // Again, individual elements are not bound to their own names.
  case _ =>
      "No match was found!"
}

Demostración en vivo

Para extraer los primeros elementos y mantener el resto como una colección:

def f(ints: Seq[Int]): String = ints match {
  case Seq(first, second, tail @ _*) =>
      s"The seq has at least two elements : $first, $second. The rest of the Seq is $tail"
  case Seq(first, tail @ _*) =>
      s"The seq has at least one element : $first. The rest of the Seq is $tail"
  // alternative syntax
  // here of course this one will never match since it checks
  // for the same thing as the one above
  case first +: tail =>
      s"The seq has at least one element : $first. The rest of the Seq is $tail"
  case _ =>
      "The seq didn't match any of the above, so it must be empty"
}

En general, cualquier forma que se pueda usar para construir una secuencia se puede usar para comparar patrones con una secuencia existente.

Tenga en cuenta que si bien el uso de Nil y :: funcionará cuando el patrón coincida con una Secuencia, la convierte en una Lista y puede tener resultados inesperados. Limítese a Seq(...) y +: para evitar esto.

Tenga en cuenta que al usar :: no funcionará para WrappedArray, Vector, etc., consulte:

scala> def f(ints:Seq[Int]) = ints match {
     | case h :: t => h
     | case _ => "No match"
     | }
f: (ints: Seq[Int])Any

scala> f(Array(1,2))
res0: Any = No match

Y con +:

scala> def g(ints:Seq[Int]) = ints match {
     | case h+:t => h
     | case _ => "No match"
     | }
g: (ints: Seq[Int])Any

scala> g(Array(1,2).toSeq)
res4: Any = 1

Guardias (si son expresiones) #

Las declaraciones de casos se pueden combinar con expresiones if para proporcionar una lógica adicional cuando se comparan patrones.

def checkSign(x: Int): String = {
    x match {
      case a if a < 0 => s"$a is a negative number"
      case b if b > 0 => s"$b is a positive number"
      case c => s"$c neither positive nor negative"
    }
}

Es importante asegurarse de que sus guardias no creen una coincidencia no exhaustiva (el compilador a menudo no detectará esto):

def f(x: Option[Int]) = x match {
    case Some(i) if i % 2 == 0 => doSomething(i)
    case None    => doSomethingIfNone
}

Esto lanza un MatchError en números impares. Debe tener en cuenta todos los casos o utilizar un caso de coincidencia comodín:

def f(x: Option[Int]) = x match {
    case Some(i) if i % 2 == 0 => doSomething(i)
    case _ => doSomethingIfNoneOrOdd
}

Coincidencia de patrones con Regex #

val emailRegex: Regex = "(.+)@(.+)\\.(.+)".r

"name@example.com" match {
  case emailRegex(userName, domain, topDomain) => println(s"Hi $userName from $domain")
  case _ => println(s"This is not a valid email.")
}

En este ejemplo, la expresión regular intenta hacer coincidir la dirección de correo electrónico proporcionada. Si lo hace, el nombre de usuario y el dominio se extraen e imprimen. topDomain también se extrae, pero no se hace nada con él en este ejemplo. Llamar a .r en una cadena str es equivalente a new Regex(str). La función r está disponible a través de una conversión implícita.

Coincidencia de patrones con identificador estable #

En la coincidencia de patrones estándar, el identificador utilizado sombreará cualquier identificador en el ámbito adjunto. A veces es necesario hacer coincidir la variable del alcance adjunto.

La siguiente función de ejemplo toma un carácter y una lista de tuplas y devuelve una nueva lista de tuplas. Si el carácter existía como primer elemento en una de las tuplas, se incrementa el segundo elemento. Si aún no existe en la lista, se crea una nueva tupla.

def tabulate(char: Char, tab: List[(Char, Int)]): List[(Char, Int)] = tab match {
  case Nil => List((char, 1))
  case (`char`, count) :: tail => (char, count + 1) :: tail
  case head :: tail => head :: tabulate(char, tail)
}

Lo anterior demuestra la coincidencia de patrones donde la entrada del método, char, se mantiene ’estable’ en la coincidencia de patrones: es decir, si llama a tabular('x', ...), se interpretaría la declaración del primer caso como:

case('x', count) => ...

Scala interpretará cualquier variable marcada con una marca de verificación como un identificador estable: también interpretará cualquier variable que comience con una letra mayúscula de la misma manera.

Coincidencia en una opción #

Si está haciendo coincidir un tipo de Opción:

def f(x: Option[Int]) = x match {
    case Some(i) => doSomething(i)
    case None    => doSomethingIfNone
}

Esto es funcionalmente equivalente a usar fold o map/getOrElse:

def g(x: Option[Int]) = x.fold(doSomethingIfNone)(doSomething)
def h(x: Option[Int]) = x.map(doSomething).getOrElse(doSomethingIfNone)

Rasgos sellados de coincidencia de patrones #

Cuando el patrón coincide con un objeto cuyo tipo es un rasgo sellado, Scala verificará en tiempo de compilación que todos los casos se ‘coinciden de forma exhaustiva’:

sealed trait Shape
case class Square(height: Int, width: Int) extends Shape
case class Circle(radius: Int) extends Shape
case object Point extends Shape


def matchShape(shape: Shape): String = shape match {
    case Square(height, width) => "It's a square"
    case Circle(radius)        => "It's a circle"
    //no case for Point because it would cause a compiler warning.
}

Si más tarde se agrega una nueva clase de caso para Shape, todas las sentencias de coincidencia en Shape comenzarán a mostrar una advertencia del compilador. Esto facilita la refactorización exhaustiva: el compilador alertará al desarrollador sobre todo el código que debe actualizarse.

Carpeta de patrones (@) #

El signo @ vincula una variable a un nombre durante una coincidencia de patrón. La variable enlazada puede ser todo el objeto coincidente o parte del objeto coincidente:

sealed trait Shape
case class Rectangle(height: Int, width: Int) extends Shape
case class Circle(radius: Int) extends Shape
case object Point extends Shape

(Circle(5): Shape) match {
  case Rectangle(h, w) => s"rectangle, $h x $w."
  case Circle(r) if r > 9 => s"large circle"
  case c @ Circle(_) => s"small circle: ${c.radius}"  // Whole matched object is bound to c
  case Point => "point"
}

> res0: Cadena = círculo pequeño: 5

El identificador vinculado se puede utilizar en filtros condicionales. De este modo:

case Circle(r) if r > 9 => s"large circle"

Se puede escribir como:

case c @ Circle(_) if c.radius > 9 => s"large circle"

El nombre se puede vincular solo a una parte del patrón coincidente:

Seq(Some(1), Some(2), None) match {
  // Only the first element of the matched sequence is bound to the name 'c'
  case Seq(c @ Some(1), _*) => head
  case _ => None
}

> res0: Opción[Int] = Algunos(1)

Coincidencia de patrones con clases de casos #

Cada clase de caso define un extractor que se puede usar para capturar los miembros de la clase de caso cuando se comparan patrones:

case class Student(name: String, email: String)

def matchStudent1(student: Student): String = student match {
    case Student(name, email) => s"$name has the following email: $email" // extract name and email
}

Se aplican todas las reglas normales de coincidencia de patrones: puede usar guardias y expresiones constantes para controlar la coincidencia:

def matchStudent2(student: Student): String = student match {
    case Student("Paul", _) => "Matched Paul" // Only match students named Paul, ignore email
    case Student(name, _) if name == "Paul" => "Matched Paul" // Use a guard to match students named Paul, ignore email
    case s if s.name == "Paul" => "Matched Paul" // Don't use extractor; use a guard to match students named Paul, ignore email
    case Student("Joe", email) => s"Joe has email $email" // Match students named Joe, capture their email
    case Student(name, email) if name == "Joe" => s"Joe has email $email" // use a guard to match students named Joe, capture their email
    case Student(name, email) => s"$name has email $email." // Match all students, capture name and email 
}

Tipos de coincidencia de patrones #

La coincidencia de patrones también se puede usar para verificar el tipo de una instancia, en lugar de usar isInstanceOf[B]:

val anyRef: AnyRef = ""
                                                  
anyRef match {
  case _: Number       => "It is a number"
  case _: String       => "It is a string"
  case _: CharSequence => "It is a char sequence"
}
//> res0: String = It is a string

El orden de los casos es importante:

anyRef match {
  case _: Number       => "It is a number"
  case _: CharSequence => "It is a char sequence"
  case _: String       => "It is a string"
}
//> res1: String = It is a char sequence

De esta manera, es similar a una declaración de ‘cambio’ clásica, sin la funcionalidad de caída. Sin embargo, también puede combinar patrones y ’extraer’ valores del tipo en cuestión. Por ejemplo:

case class Foo(s: String)
case class Bar(s: String)
case class Woo(s: String, i: Int)

def matcher(g: Any):String = {
  g match {
    case Bar(s) => s + " is classy!" 
    case Foo(_) => "Someone is wicked smart!"
    case Woo(s, _) => s + " is adventerous!"
    case _ => "What are we talking about?"
  }
}

print(matcher(Foo("Diana")))  // prints 'Diana is classy!'
print(matcher(Bar("Hadas")))  // prints 'Someone is wicked smart!'
print(matcher(Woo("Beth", 27)))   // prints 'Beth is adventerous!'
print(matcher(Option("Katie")))  // prints 'What are we talking about?'

Tenga en cuenta que en el caso de Foo y Woo usamos el guión bajo (_) para ‘coincidir con una variable no vinculada’. Es decir, el valor (en este caso, Hadas y 27, respectivamente) no está vinculado a un nombre y, por lo tanto, no está disponible en el controlador para ese caso. Esta es una abreviatura útil para hacer coincidir ‘cualquier’ valor sin preocuparse por cuál es ese valor.

Coincidencia de patrones compilada como tableswitch o lookupswitch #

La anotación @switch le dice al compilador que la instrucción match se puede reemplazar con una única instrucción tableswitch en el nivel de código de bytes. Esta es una optimización menor que puede eliminar comparaciones innecesarias y cargas variables durante el tiempo de ejecución.

La anotación @switch solo funciona para coincidencias con constantes literales e identificadores final val. Si la coincidencia del patrón no se puede compilar como tableswitch/lookupswitch, el compilador generará una advertencia.

import annotation.switch

def suffix(i: Int) = (i: @switch) match {
  case 1 => "st"
  case 2 => "nd"
  case 3 => "rd"
  case _ => "th"
}

Los resultados son los mismos que una coincidencia de patrón normal:

scala> suffix(2)
res1: String = "2nd"

scala> suffix(4)
res2: String = "4th"

De la Documentación de Scala (2.8+) – @switch:

Una anotación que se aplicará a una expresión de coincidencia. Si está presente, el el compilador verificará que la coincidencia se haya compilado en un conmutador de tabla o lookupswitch, y emite un error si en su lugar se compila en un serie de expresiones condicionales.

De la especificación de Java:

  • tableswitch: “Acceder a la tabla de saltos por índice y salto”
  • lookupswitch: “Acceder a la tabla de saltos por combinación de teclas y salto”

Coincidencia de múltiples patrones a la vez #

El | se puede usar para hacer que una declaración de un solo caso coincida con múltiples entradas para producir el mismo resultado:

def f(str: String): String = str match {
  case "foo" | "bar" => "Matched!"
  case _ => "No match."
}

f("foo")  // res0: String = Matched!
f("bar")  // res1: String = Matched!
f("fubar")  // res2: String = No match.

Tenga en cuenta que si bien la coincidencia de valores de esta manera funciona bien, la siguiente coincidencia de tipos causará problemas:

sealed class FooBar
case class Foo(s: String) extends FooBar
case class Bar(s: String) extends FooBar

val d = Foo("Diana")
val h = Bar("Hadas")

// This matcher WILL NOT work.
def matcher(g: FooBar):String = {
  g match {
    case Foo(s) | Bar(s) => print(s)  // Won't work: s cannot be resolved
    case Foo(_) | Bar(_) => _         // Won't work: _ is an unbound placeholder
    case _ => "Could not match"
  }
}

Si en el último caso (con _) no necesita el valor de la variable independiente y solo quiere hacer otra cosa, está bien:

def matcher(g: FooBar):String = {
  g match {
    case Foo(_) | Bar(_) => "Is either Foo or Bar."  // Works fine
    case _ => "Could not match"
  }
}

De lo contrario, te queda dividir tus casos:

def matcher(g: FooBar):String = {
  g match {
    case Foo(s) => s 
    case Bar(s) => s
    case _ => "Could not match"
  }
}

Coincidencia de patrones en tuplas #

Dada la siguiente Lista de tuplas:

val pastries = List(("Chocolate Cupcake", 2.50), 
                    ("Vanilla Cupcake", 2.25),
                    ("Plain Muffin", 3.25))

La coincidencia de patrones se puede usar para manejar cada elemento de manera diferente:

pastries foreach { pastry =>
  pastry match {
    case ("Plain Muffin", price) => println(s"Buying muffin for $price")
    case p if p._1 contains "Cupcake" => println(s"Buying cupcake for ${p._2}")
    case _ => println("We don't sell that pastry")
  }
}

El primer caso muestra cómo hacer coincidir una cadena específica y obtener el precio correspondiente. El segundo caso muestra un uso de if y extracción de tupla para hacer coincidir elementos de la tupla.
