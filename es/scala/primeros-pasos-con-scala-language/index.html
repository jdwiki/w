Hello World al extender la aplicación #

object HelloWorld extends App {
  println("Hello, world!")
}

Demostración en vivo

Al extender App trait, puede evitar definir un método main explícito . Todo el cuerpo del objeto HelloWorld se trata como “el método principal”.

Inicialización retrasada #

Según la documentación oficial, App hace uso de una función llamada Inicialización retrasada. Esto significa que los campos del objeto se inicializan después de llamar al método principal.

Inicialización retrasada #

Según la documentación oficial, App hace uso de una función llamada Inicialización retrasada. Esto significa que los campos del objeto se inicializan después de llamar al método principal.

DelayedInit ahora está en desuso para uso general, pero todavía es compatible para App como un caso especial. El soporte continuará hasta que se decida e implemente una característica de reemplazo.

Para acceder a los argumentos de la línea de comandos al extender App, use this.args:

object HelloWorld extends App {
  println("Hello World!")
  for {
    arg <- this.args
  } println(s"Arg=$arg")
}

Al usar App, el cuerpo del objeto se ejecutará como el método main, no hay necesidad de anular main.

Hello World al definir un método ‘principal’ #

Coloque este código en un archivo llamado HelloWorld.scala:

object Hello {
  def main(args: Array[String]): Unit = {
    println("Hello World!")
  }
}

Demostración en vivo

Para compilarlo en un código de bytes ejecutable por la JVM:

$ scalac HelloWorld.scala

Para ejecutarlo:

$ scala Hello

Cuando el tiempo de ejecución de Scala carga el programa, busca un objeto llamado “Hola” con un método “principal”. El método main es el punto de entrada del programa y se ejecuta.

Tenga en cuenta que, a diferencia de Java, Scala no tiene el requisito de nombrar objetos o clases según el archivo en el que se encuentran. En cambio, el parámetro Hola pasado en el comando scala Hola se refiere al objeto a buscar que contiene el principal método a ejecutar. Es perfectamente posible tener múltiples objetos con métodos principales en el mismo archivo .scala.

La matriz args contendrá los argumentos de la línea de comandos dados al programa, si los hay. Por ejemplo, podemos modificar el programa así:

object HelloWorld {
  def main(args: Array[String]): Unit = {
    println("Hello World!")
    for {
      arg <- args
    } println(s"Arg=$arg")
  }
}

Compílalo:

$ scalac HelloWorld.scala

Y luego ejecutarlo:

$ scala HelloWorld 1 2 3
Hello World!
Arg=1
Arg=2
Arg=3

Hello World como guión #

Scala se puede utilizar como un lenguaje de secuencias de comandos. Para demostrarlo, cree HelloWorld.scala con el siguiente contenido:

println("Hello")

Ejecútelo con el intérprete de línea de comandos (el $ es el indicador de la línea de comandos):

$ scala HelloWorld.scala
Hello

Si omite .scala (como si simplemente escribiera scala HelloWorld), el corredor buscará un archivo .class compilado con código de bytes en lugar de compilar y luego ejecutar el script.

Nota: Si scala se usa como lenguaje de secuencias de comandos, no se puede definir ningún paquete.

En los sistemas operativos que utilizan bash o terminales de shell similares, los scripts de Scala se pueden ejecutar utilizando un ‘preámbulo de shell’. Cree un archivo llamado HelloWorld.sh y coloque lo siguiente como su contenido:

#!/bin/sh
exec scala "$0" "$@"
!#
println("Hello")

Las partes entre #! y !# son el ‘preámbulo de shell’ y se interpretan como un script bash. El resto es Scala.

Una vez que haya guardado el archivo anterior, debe otorgarle permisos ’ejecutables’. En el shell puedes hacer esto:

$ chmod a+x HelloWorld.sh

(Tenga en cuenta que esto otorga permiso a todos: [lea sobre chmod] 1 para aprender cómo configurarlo para conjuntos de usuarios más específicos).

Ahora puedes ejecutar el script así:

$ ./HelloWorld.sh

Uso de Scala REPL #

Cuando ejecuta scala en una terminal sin parámetros adicionales, se abre un intérprete REPL (Read-Eval-Print Loop):

nford:~ $ scala
Welcome to Scala 2.11.8 (Java HotSpot(TM) 64-Bit Server VM, Java 1.8.0_66).
Type in expressions for evaluation. Or try :help.

scala>

El REPL le permite ejecutar Scala en forma de hoja de trabajo: el contexto de ejecución se conserva y puede probar comandos manualmente sin tener que construir un programa completo. Por ejemplo, al escribir val poema = "Como felices seremos" se vería así:

scala> val poem = "As halcyons we shall be"
poem: String = As halcyons we shall be

Ahora podemos imprimir nuestro val:

scala> print(poem)
As halcyons we shall be

Tenga en cuenta que val es inmutable y no se puede sobrescribir:

scala> poem = "Brooding on the open sea"
<console>:12: error: reassignment to val
       poem = "Brooding on the open sea"

Pero en el REPL puede redefinir un val (lo que causaría un error en un programa normal de Scala, si se hiciera en el mismo ámbito):

scala> val poem = "Brooding on the open sea"
poem: String = Brooding on the open sea

Durante el resto de su sesión REPL, esta variable recién definida sombreará la variable definida previamente. Los REPL son útiles para ver rápidamente cómo funcionan los objetos u otro código. Todas las funciones de Scala están disponibles: puede definir funciones, clases, métodos, etc.
