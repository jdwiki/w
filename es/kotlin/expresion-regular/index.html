Modismos para la coincidencia de Regex en la expresión When #

Usando locales inmutables: #

Utiliza menos espacio horizontal pero más espacio vertical que la plantilla de “temporales anónimos”. Es preferible a la plantilla de “temporales anónimos” si la expresión when está en un bucle; en ese caso, las definiciones de expresiones regulares deben colocarse fuera del bucle.

import kotlin.text.regex

var string = /* some string */

val regex1 = Regex( /* pattern */ )
val regex2 = Regex( /* pattern */ )
/* etc */

when {
    regex1.matches(string) -> /* do stuff */
    regex2.matches(string) -> /* do stuff */
    /* etc */
}

Uso de temporales anónimos: #

Utiliza menos espacio vertical pero más espacio horizontal que la plantilla “locales inmutables”. No debe usarse si la expresión when está en un bucle.

import kotlin.text.regex

var string = /* some string */

when {  
    Regex( /* pattern */ ).matches(string) -> /* do stuff */
    Regex( /* pattern */ ).matches(string) -> /* do stuff */
    /* etc */
}

Usando el patrón de visitante: #

Tiene la ventaja de emular de cerca la sintaxis “argumentativa” when. Esto es beneficioso porque indica más claramente el argumento de la expresión when y también evita ciertos errores del programador que podrían surgir al tener que repetir el argumento when en cada whenEntry. Tanto la plantilla de “locales inmutables” como la de “temporales anónimos” se pueden utilizar con esta implementación del patrón de visitante.

import kotlin.text.regex

var string = /* some string */

when (RegexWhenArgument(string)) {
    Regex( /* pattern */ ) -> /* do stuff */
    Regex( /* pattern */ ) -> /* do stuff */
    /* etc */
}

Y la definición mínima de la clase contenedora para el argumento de la expresión when:

class RegexWhenArgument (val whenArgument: CharSequence) {
    operator fun equals(whenEntry: Regex) = whenEntry.matches(whenArgument)
    override operator fun equals(whenEntry: Any?) = (whenArgument == whenEntry)
}

Introducción a las expresiones regulares en Kotlin #

Esta publicación muestra cómo usar la mayoría de las funciones en la clase Regex, trabajar con valores nulos relacionados de forma segura con las funciones Regex y cómo las cadenas sin procesar facilitan la escritura y lectura de patrones de expresiones regulares.

La clase RegEx #

Para trabajar con expresiones regulares en Kotlin, necesitas usar la clase Regex(pattern: String) e invocar funciones como find(..) o replace(..) en ese objeto regex.

Un ejemplo de cómo usar la clase Regex que devuelve verdadero si la cadena input contiene c o d:

val regex = Regex(pattern = "c|d")
val matched = regex.containsMatchIn(input = "abc")    // matched: true

Lo esencial que hay que entender con todas las funciones Regex es que el resultado se basa en hacer coincidir el patrón de expresiones regulares y la cadena de entrada. Algunas de las funciones requieren una coincidencia completa, mientras que el resto requiere solo una coincidencia parcial. La función containsMatchIn(..) utilizada en el ejemplo requiere una coincidencia parcial y se explica más adelante en esta publicación.

Seguridad nula con expresiones regulares #

Tanto find(..) como matchEntire(..) devolverán un objeto MatchResult?. El carácter ? después de MatchResult es necesario para que Kotlin maneje [null safe] [null-safety].

Un ejemplo que demuestra cómo Kotlin maneja el nulo de forma segura desde una función Regex, cuando la función find(..) devuelve un valor nulo:

val matchResult = 
    Regex("c|d").find("efg")           // matchResult: null
val a = matchResult?.value             // a: null
val b = matchResult?.value.orEmpty()   // b: ""
a?.toUpperCase()                       // Still needs question mark. => null    
b.toUpperCase()                        // Accesses the function directly. => ""

Con la función orEmpty(), b no puede ser nulo y el carácter ? no es necesario cuando llamas a funciones en b.

Si no te importa este manejo seguro de valores nulos, Kotlin te permite trabajar con valores nulos como en Java con los caracteres !!:

a!!.toUpperCase()                      // => KotlinNullPointerException

Cadenas sin procesar en patrones de expresiones regulares #

Kotlin ofrece una mejora con respecto a Java con una cadena sin formato que hace posible escribir patrones de expresiones regulares puras sin barras invertidas dobles, que son necesarias con una cadena de Java. Una cadena sin procesar se representa con una comilla triple:

"""\d{3}-\d{3}-\d{4}""" // raw Kotlin string
"\\d{3}-\\d{3}-\\d{4}"  // standard Java string

encontrar (entrada: CharSequence, startIndex: Int): MatchResult? ================================================== ======

La cadena input se comparará con el patrón en el objeto Regex. Devuelve un objeto Matchresult? con el primer texto coincidente después de startIndex, o null si el patrón no coincide con la cadena input. La cadena de resultados se recupera de la propiedad value del objeto MatchResult?. El parámetro startIndex es opcional con el valor predeterminado 0.

Para extraer el primer número de teléfono válido de una cadena con detalles de contacto:

val phoneNumber :String? = Regex(pattern = """\d{3}-\d{3}-\d{4}""")
    .find(input = "phone: 123-456-7890, e..")?.value // phoneNumber: 123-456-7890

Sin un número de teléfono válido en la cadena input, la variable phoneNumber será null.

findAll(entrada: CharSequence, startIndex: Int): Sequence ================================================== ==================

Devuelve todas las coincidencias de la cadena input que coincide con el patrón de expresión regular.

Para imprimir todos los números separados por espacios, de un texto con letras y dígitos:

val matchedResults = Regex(pattern = """\d+""").findAll(input = "ab12cd34ef")
val result = StringBuilder()
for (matchedText in matchedResults) {
    result.append(matchedText.value + " ")
}

println(result) // => 12 34

La variable matchedResults es una secuencia con objetos MatchResult. Con una cadena input sin dígitos, la función findAll(..) devolverá una secuencia vacía.

matchEntire (entrada: CharSequence): MatchResult? #

Si todos los caracteres en la cadena input coinciden con el patrón de expresiones regulares, se devolverá una cadena igual a input. De lo contrario, se devolverá null.

Devuelve la cadena de entrada si toda la cadena de entrada es un número:

val a = Regex("""\d+""").matchEntire("100")?.value             // a: 100
val b = Regex("""\d+""").matchEntire("100 dollars")?.value     // b: null

coincidencias (entrada: CharSequence): booleano #

Devuelve verdadero si toda la cadena de entrada coincide con el patrón de expresiones regulares. Falso en caso contrario.

Comprueba si dos cadenas contienen solo dígitos:

val regex = Regex(pattern = """\d+""")
regex.matches(input = "50")             // => true
regex.matches(input = "50 dollars")     // => false

contiene MatchIn (entrada: CharSequence): booleano #

Devuelve verdadero si parte de la cadena de entrada coincide con el patrón de expresión regular. Falso en caso contrario.

Pruebe si dos cadenas contienen al menos un dígito:

Regex("""\d+""").containsMatchIn("50 dollars")       // => true
Regex("""\d+""").containsMatchIn("Fifty dollars")    // => false

división (entrada: CharSequence, límite: Int): List ================================================== ==

Devuelve una nueva lista sin todas las coincidencias de expresiones regulares.

Para devolver listas sin dígitos:

val a = Regex("""\d+""").split("ab12cd34ef")     // a: [ab, cd, ef]
val b = Regex("""\d+""").split("This is a test") // b: [This is a test]

Hay un elemento en la lista para cada división. La primera cadena input tiene tres números. Eso da como resultado una lista con tres elementos.

reemplazar (entrada: CharSequence, reemplazo: Cadena): Cadena ================================================== =======

Reemplaza todas las coincidencias de la expresión regular patrón en la cadena de entrada con la cadena de reemplazo.

Para reemplazar todos los dígitos en una cadena con una x:

val result = Regex("""\d+""").replace("ab12cd34ef", "x") // result: abxcdxef
