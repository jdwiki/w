Un constructor con seguridad de tipos es un concepto, en lugar de una característica del lenguaje, por lo que no está estrictamente formalizado.

Una estructura típica de un constructor con seguridad de tipos #

Una sola función de constructor generalmente consta de 3 pasos:

  1. Cree un objeto.
  2. Ejecute lambda para inicializar el objeto.
  3. Agregar el objeto a la estructura o devolverlo.

Constructores de tipo seguro en las bibliotecas de Kotlin #

Constructor de estructura de árbol con seguridad de tipos #

Los constructores se pueden definir como un conjunto de funciones de extensión que toman expresiones lambda con receptores como argumentos. En este ejemplo, se está construyendo un menú de un JFrame:

import javax.swing.*

fun JFrame.menuBar(init: JMenuBar.() -> Unit) {
    val menuBar = JMenuBar()
    menuBar.init()
    setJMenuBar(menuBar)
}

fun JMenuBar.menu(caption: String, init: JMenu.() -> Unit) {
    val menu = JMenu(caption)
    menu.init()
    add(menu)
}

fun JMenu.menuItem(caption: String, init: JMenuItem.() -> Unit) {
    val menuItem = JMenuItem(caption)
    menuItem.init()
    add(menuItem)
}

Estas funciones se pueden usar para construir una estructura de árbol de objetos de una manera fácil:

class MyFrame : JFrame() {
    init {
        menuBar {
            menu("Menu1") {
                menuItem("Item1") {
                    // Initialize MenuItem with some Action
                }
                menuItem("Item2") {}
            }
            menu("Menu2") {
                menuItem("Item3") {}
                menuItem("Item4") {}
            }
        }
    }
}
