Repartos inteligentes #

Si el compilador puede inferir que un objeto no puede ser nulo en un punto determinado, ya no tiene que usar los operadores especiales:

var string: String? = "Hello!"
print(string.length)  // Compile error
if(string != null) {
    // The compiler now knows that string can't be null
    print(string.length)  // It works now!
}

Nota: El compilador no le permitirá lanzar variables mutables inteligentes que podrían modificarse potencialmente entre la verificación nula y el uso previsto.

Si se puede acceder a una variable desde fuera del alcance del bloque actual (porque son miembros de un objeto no local, por ejemplo), debe crear una nueva referencia local que luego puede convertir y usar de forma inteligente.

Afirmación #

Los sufijos !! ignoran la nulabilidad y devuelven una versión no nula de ese tipo. Se lanzará KotlinNullPointerException si el objeto es null.

val message: String? = null
println(message!!) //KotlinNullPointerException thrown, app crashes

Tipos anulables y no anulables #

Los tipos normales, como String, no aceptan valores NULL. Para que puedan contener valores nulos, debe indicarlo explícitamente poniendo un ? detrás de ellos: String?

var string        : String = "Hello World!"
var nullableString: String? = null

string = nullableString    // Compiler error: Can't assign nullable to non-nullable type.
nullableString = string    // This will work however!

Eliminar nulos de un iterable y una matriz #

A veces necesitamos cambiar el tipo de Collection<T?> a Collections<T>. En ese caso, filterNotNull es nuestra solución.

val a: List<Int?> = listOf(1, 2, 3, null)
val b: List<Int> = a.filterNotNull()

Null Coalescing / Operador Elvis #

A veces es deseable evaluar una expresión que admite valores NULL de forma if-else. El operador elvis, ?:, se puede usar en Kotlin para tal situación.

Por ejemplo:

val value: String = data?.first() ?: "Nothing here."

La expresión anterior devuelve "Nada aquí" si data?.first() o data en sí arroja un valor nulo; de lo contrario, el resultado de data?.first().

También es posible lanzar excepciones usando la misma sintaxis para abortar la ejecución del código.

val value: String = data?.second() 
    ?: throw IllegalArgumentException("Value can't be null!")

Recordatorio: NullPointerExceptions se pueden lanzar usando el operador de aserción (por ejemplo, datos!!.Segundo()!!)

Operador de llamadas seguras #

Para acceder a funciones y propiedades de tipos anulables, debe usar operadores especiales.

El primero, ?., te da la propiedad o función a la que intentas acceder, o te da un valor nulo si el objeto es nulo:

val string: String? = "Hello World!"
print(string.length)   // Compile error: Can't directly access property of nullable type.
print(string?.length)  // Will print the string's length, or "null" if the string is null.

Modismo: llamar a varios métodos en el mismo objeto marcado como nulo #

Una forma elegante de llamar a varios métodos de un objeto con verificación nula es usar apply de Kotlin de esta manera:

obj?.apply { 
    foo()
    bar()
}

Esto llamará a foo y bar en obj (que es this en el bloque apply) solo si obj no es nulo, saltándose todo el bloque de lo contrario.

Para incluir una variable anulable en el ámbito como una referencia no anulable sin convertirla en el receptor implícito de llamadas a funciones y propiedades, puede usar let en lugar de apply:

nullable?.let { notnull ->
    notnull.foo()
    notnull.bar()
}

notnull podría tener cualquier nombre, o incluso omitirse y usarse mediante [el parámetro lambda implícito it.][3]

[3]: https://www.wikiod.com/es/kotlin/funciones#Funciones Lambda

Operador de Elvis (?:) #

En Kotlin, podemos declarar una variable que puede contener referencia nula. Supongamos que tenemos una referencia anulable a, podemos decir “si a no es nulo, úselo, de lo contrario use algún valor no nulo x”

var a: String? = "Nullable String Value"

Ahora, a puede ser nulo. Entonces, cuando necesitamos acceder al valor de a, debemos realizar una verificación de seguridad, ya sea que contenga valor o no. Podemos realizar esta comprobación de seguridad mediante la declaración if...else convencional.

val b: Int = if (a != null) a.length else -1

Pero aquí viene el operador avanzado Elvis (Operador Elvis: ?:). Arriba if...else se puede expresar con el operador Elvis de la siguiente manera:

val b = a?.length ?: -1

Si la expresión a la izquierda de ?: (aquí: a?.length) no es nula, el operador elvis la devuelve, de lo contrario, devuelve la expresión a la derecha (aquí: -1). La expresión del lado derecho se evalúa solo si el lado izquierdo es nulo.
