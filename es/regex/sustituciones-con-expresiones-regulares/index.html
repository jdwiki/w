Parámetros #

en línea Descripción
$número Sustituye la subcadena que coincide con el número de grupo.
${nombre} Sustituye la subcadena que coincide con un [grupo con nombre](https://www.wikiod.com/es/regex/grupos-de-captura#Grupos de captura con nombre) nombre.
$$ Carácter de escape ‘$’ en la cadena de resultado (reemplazo).
$& ($0) Reemplaza con toda la cadena coincidente.
$+ ($&) Sustituye el texto coincidente con el último grupo capturado.
$` Sustituye todo el texto coincidente con cada texto no coincidente antes de la coincidencia.
$’ Sustituye todo el texto coincidente con cada texto no coincidente después de la coincidencia.
$_ Sustituye todo el texto coincidente por la cadena completa.
Nota: Los términos en cursiva significan que las cadenas son volátiles (pueden variar según el tipo de expresión regular).

Fundamentos de la sustitución #

Una de las formas más comunes y útiles de reemplazar texto con expresiones regulares es mediante [Capture Groups](https://www.wikiod.com/es/regex/grupos-de-captura#Grupos de captura básicos ).
O incluso un [Grupo de captura con nombre] (https://www.wikiod.com/es/regex/grupos-de-captura#Grupos de captura con nombre), como referencia para almacenar o reemplazar los datos .

Hay dos términos bastante parecidos en los documentos de expresiones regulares, por lo que puede ser importante nunca confundir Sustituciones (es decir, $1) con [Backreferences](https://www.wikiod.com/es/docs/regex /4072/back-reference#t=201704302255256450862) (es decir, \1). Los términos de sustitución se utilizan en un texto de reemplazo; Referencias inversas, en la expresión Regex pura. Aunque algunos lenguajes de programación aceptan ambas sustituciones, no es alentador.

Digamos que tenemos esta expresión regular: /hello(\s+)world/i. Siempre que se haga referencia a $number (en este caso, $1), se reemplazarán los espacios en blanco que coincidan con \s+.
El mismo resultado se mostrará con la expresión regular: /hello(?<spaces>\s+)world/i. Y como aquí tenemos un grupo con nombre, también podemos usar ${spaces}.

En este mismo ejemplo, también podemos usar $0 o $& (Nota: $& puede usarse como $+ en su lugar, lo que significa recuperar el ÚLTIMO grupo de captura en otros motores de expresiones regulares), según el tipo de expresión regular con el que esté trabajando, para obtener el texto coincidente completo. (es decir, $& devolverá hEllo woRld para la cadena: hEllo woRld of Regex!)

Reemplazo avanzado #

Algunos lenguajes de programación tienen sus propias peculiaridades Regex, por ejemplo, el término $+ (en C#, Perl, VB, etc.) que reemplaza el texto coincidente con el último grupo capturado.

Ejemplo:

using System;
using System.Text.RegularExpressions;

public class Example
{
   public static void Main()
   {
      string pattern = @"\b(\w+)\s\1\b";
      string substitution = "$+";
      string input = "The the dog jumped over the fence fence.";
      Console.WriteLine(Regex.Replace(input, pattern, substitution, 
                        RegexOptions.IgnoreCase));
   }
}
// The example displays the following output:
//      The dog jumped over the fence.

Ejemplo de la red de desarrolladores oficial de Microsoft [1](https://msdn.microsoft.com/en-us/library/ewy2t5e0(v=vs.110).aspx?cs-save-lang =1&cs-lang=csharp#Anchor_7)

Otros términos de sustitución raros son $` y $':

$` = Reemplaza las coincidencias con el texto antes de la cadena coincidente
$’ = Reemplaza las coincidencias con el texto después de la cadena coincidente

Debido a este hecho, estas cadenas de reemplazo deberían hacer su trabajo de la siguiente manera:

Regex: /part2/
Input: "part1part2part3"
Replacement: "$`"
Output: "part1part1part3" //Note that part2 was replaced with part1, due &` term
---------------------------------------------------------------------------------
Regex: /part2/
Input: "part1part2part3"
Replacement: "$'"
Output: "part1part3part3" //Note that part2 was replaced with part3, due &' term

Aquí hay un ejemplo de estas sustituciones que funcionan en una pieza de javascript:

var rgx = /middle/;
var text = "Your story must have a beginning, middle, and end"
console.log(text.replace(rgx, "$`")); 
//Logs: "Your story must have a beginning, Your story must have a beginning, , and end"
console.log(text.replace(rgx, "$'"))
//Logs: "Your story must have a beginning, , and end, and end"
También existe el término $_ que recupera todo el texto coincidente en su lugar: 
Regex: /part2/
Input: "part1part2part3"
Replacement: "$_"
Output: "part1part1part2part3part3" //Note that part2 was replaced with part1part2part3,
                                                                         // due $_ term

Convertir esto a VB nos daría esto:

Imports System.Text.RegularExpressions

Module Example
   Public Sub Main()
      Dim input As String = "ABC123DEF456"
      Dim pattern As String = "\d+"
      Dim substitution As String = "$_"
      Console.WriteLine("Original string:          {0}", input)
      Console.WriteLine("String with substitution: {0}", _
                        Regex.Replace(input, pattern, substitution))      
   End Sub
End Module
' The example displays the following output:
'       Original string:          ABC123DEF456
'       String with substitution: ABCABC123DEF456DEFABC123DEF456

Ejemplo de Microsoft Official’s Developer Network [2](https://msdn.microsoft.com/en-us/library/ewy2t5e0(v=vs.110).aspx?cs-save-lang =1&cs-lang=csharp#Anchor_8)

Y el último pero no menos importante término de sustitución es $$, que traducido a una expresión regular sería lo mismo que \$ (una versión escapada del literal $).

Si desea hacer coincidir una cadena como esta: USD: $3.99 por ejemplo, y desea almacenar 3.99, pero reemplazarlo como $3.99 con solo una expresión regular, puede usar:

Regex: /USD:\s+\$([\d.]+)/
Input: "USD: $3.99"
Replacement: "$$$1"
To Store: "$1"
Output: "$3.99"
Stored: "3.99"

Si desea probar esto con Javascript, puede usar el código:

var rgx = /USD:\s+\$([\d.]+)/;
var text = "USD: $3.99";
var stored = parseFloat(rgx.exec(text)[1]);
console.log(stored); //Logs 3.99
console.log(text.replace(rgx, "$$$1")); //Logs $3.99
