Es importante poder usar variables del lado del servidor en su sitio web. Pug le permite interpolar datos generados por su servidor en código HTML, CSS e incluso JavaScript.

Sintaxis #

  • res.render(ruta, variables) // Busca un archivo pug para representarlo en la ruta “ruta” y le pasa “variables”
  • #{variable} // Interpola “variable” en línea con el código Jade circundante, después de evaluar “variable”
  • !{variable} // Interpola “variable” en línea con el código Jade circundante, sin evaluar “variable”.
  • #[elemento] // Interpola “elemento” dentro de un elemento HTML Pug existente. La sintaxis de los elementos HTML interpolados es idéntica a la de los elementos HTML normales.

Parámetros #

Parámetro Detalles
camino Usado en res.render. Esta es la ruta del archivo Pug que vamos a renderizar. La ruta se toma desde la raíz de la carpeta configurada en su aplicación Express: app.set("views", "templates/views"). Por ejemplo, res.render("index") buscará un archivo Pug en templates/views/index.pug. También se pueden especificar subdirectorios; res.render("admin/index") busca un archivo Pug en templates/views/admin/index.pug.
Variables Usado en res.render. Un objeto JavaScript de variables que se hará accesible para el archivo Pug definido por ruta (arriba). Dentro del archivo Pug, las claves del objeto JavaScript anterior están disponibles como variables. Si variables = {título: "Hola", color: "rojo"}, podríamos usar la variable título y color. También están disponibles las subpropiedades de los objetos anidados.
variable Usado en la sintaxis de paréntesis #{} o !{}. El valor de variable se generará en el contexto del código Pug que lo rodea. Si se antepone un símbolo de libra al corchete de apertura, la “variable” se evaluará antes de generarse. Si se antepone un signo de exclamación a la llave de apertura, variable no se evaluará.
elemento Se utiliza en la sintaxis de corchetes #[]. El elemento HTML (en la sintaxis Pug, no en la sintaxis HTML normal) se evaluará y generará en línea con el código Pug circundante.

Para obtener más información sobre la interpolación de PugJS, consulte la [documentación oficial de interpolación de PugJS]. (https://pugjs.org/language/interpolation.html)

Interpolación de variables del lado del servidor #

Es posible pasar variables desde su servidor a Pug para contenido dinámico o generación de scripts. Las plantillas Pug pueden acceder a las variables pasadas a la función res.render en Express (o pug.renderFile si no está utilizando Express, los argumentos son idénticos).

índice.js

let colors = ["Red", "Green", "Blue"];
let langs  = ["HTML", "CSS", "JS"];
let title  = "My Cool Website";

let locals = {
    siteColors: colors,
    siteLangs:  langs,
    title:      title
};
res.render("index", locals);

Dentro de su archivo index.pug, entonces tiene acceso a la variable locals a través de sus claves. Los nombres de las variables en su archivo Pug se convierten en siteColors y siteNames.

Para establecer la totalidad de un elemento HTML igual a una variable, use el operador de igualdad = para hacerlo. Si su variable necesita estar incrustada en línea, use la sintaxis de corchetes #{} para hacerlo.

índice.pug

doctype html
html
    head
        title= title
    body
        p My favorite color is #{siteColors[0]}.
        p Here's a list of my favorite coding languages
        ul
            each language in siteLangs
                li= language

salida index.pug

<!DOCTYPE html>
<html>
    <head>
        <title>My Cool Website</title>
    </head>
    <body>
        <p>My favorite color is Red.</p>
        <p>Here's a list of my favorite coding languages</p>
        <ul>
            <li>HTML</li>
            <li>CSS</li>
            <li>JS</li>
        </ul>
    </body>
</html>

Interpolación de variables sin procesar en HTML #

El contenido interpolado con sintaxis de paréntesis se evaluará para el código, cuya salida se incluye en su salida HTML.

title sigue el patrón básico para evaluar una plantilla local, pero el código entre #{ y } se evalúa, escapa y el resultado se almacena en la salida de la plantilla que se representa. [Fuente]

Si necesita incluir sintaxis HTML sin procesar, use un signo de exclamación en lugar de un símbolo de libra (!{} en lugar de #{}).

índice.js

let tag = "<div>You can't escape me!</div>";
res.render("index", {
    myTag: tag
});

índice.pug

doctype html
html
    head
    body
        !{myTag}

salida index.pug

<!DOCTYPE html>
<html>
    <head></head>
    <body>
        <div>You can't escape me!</div>
    </body>
</html>

Interpolación de valores en código JavaScript #

La interpolación de valores es útil si necesita pasar una variable del lado del servidor a JavaScript del lado del cliente (u otros lenguajes que lo requieran).

En el caso de variables, números, cadenas y similares, puede pasar este tipo de variables directamente a su JavaScript con sintaxis de corchetes más un punto de explicación (para que el código dentro de los corchetes no se evalúe). Esto es útil para parametrizar Código JavaScript que requiere algo de su servidor.

En el siguiente ejemplo, tenemos que envolver nombre de usuario entre comillas para que JavaScript lo interprete como una cadena; Pug generará el contenido de la variable tal cual, por lo que debemos ponerlo entre comillas para que sea una cadena JavaScript adecuada. Esto no es necesario para number, donde JavaScript interpretará nuestro número como pretendemos (como un número).

índice.js

let number   = 24;
let username = "John";
res.render("index", {
    number:   number,
    username: username
});

índice.pug

html
    head
    script.
        // Sets the username of the current user to be displayed site-wide
        function setUsername(username) {
            // ...
        }
        var number   = #{number};
        var username = "#{username}";
        setUsername(username);
        
    body
        p Welcome to my site!

salida index.pug

<html>
    <head>
        <script>
            // Sets the username of the current user to be displayed site-wide
            function setUsername(username) {
                // ...
            }
            var number   = 24;
            var username = "John";
            setUsername(username);
        </script>
    </head>
    <body>
        <p>Welcome to my site!</p>
    </body>
</html>

Si necesita interpolar el valor de un objeto de JavaScript (por ejemplo, toda la información sobre un usuario), debe especificar la salida en Pug para que se trate como un objeto de JavaScript. También es necesario generar el contenido sin procesar de la variable, en lugar de la forma evaluada de la misma. Si fuera a generar la variable escapada (var user = #{JSON.stringify(user)}), recibiría una versión escaped del objeto (donde las comillas y los apóstrofes se convierten en &quot;), que no es lo que queremos para que JSON.stringify funcione.

índice.js

var myUser = {
    name:    "Leeroy Jenkins",
    id:      1234567890,
    address: "123 Wilson Way, New York NY, 10165"
};

res.render('index', {
    user: myUser
});

índice.pug

doctype html
html
    head
        script.
            window.onload = function () {
                function setUsername(username) {
                    return username;
                }

                var user = !{JSON.stringify(user)};
                document.getElementById("welcome-user").innerHTML = setUsername(user.name);
            };

    body
        div(id="welcome-user")

salida index.pug

<!DOCTYPE html>
<html>
    <head>
        <script>
            window.onload = function() {
                function setUsername(username) {
                    return username;
                }

                var user = {
                    "name": "Leeroy Jenkins",
                    "id": 1234567890,
                    "address": "123 Wilson Way, New York NY, 10165"
                };
                document.getElementById("welcome-user").innerHTML = setUsername(user.name);
        };
        </script>
    </head>
    <body>
        <div id="welcome-user"></div>
    </body>    
</html>

El innerHTML de #welcome-user se vuelve igual a Leeroy Jenkins. El contenido de la variable ‘usuario’ se imprime directamente en la fuente HTML

Interpolación de elementos HTML #

Puede ser necesario anidar etiquetas HTML una dentro de otra. La interpolación de elementos se realiza en una sintaxis similar a la interpolación de variables; Aquí se utilizan corchetes en lugar de llaves. La sintaxis de los elementos HTML interpolados es idéntica a la implementación de los elementos HTML normales.

índice.pug

doctype html
html
    head
        title My Awesome Website
    body
        p The server room went #[b boom]!
        p The fire alarm, however, #[u failed to go off...]
        p Not even #[a(href="https://stackoverflow.com/") Stack Overflow] could save them now.

salida index.pug

<!DOCTYPE html>
<html>
    <head>
        <title>My Awesome Website</title>
    </head>
    <body>
        <p>The server room went <b>boom</b>!</p>
        <p>The fire alarm, however, <u>failed to go off...</u></p>
        <p>Not even <a href="https://stackoverflow.com/">Stack Overflow</a> could save them now.</p>
    </body>
</html>
