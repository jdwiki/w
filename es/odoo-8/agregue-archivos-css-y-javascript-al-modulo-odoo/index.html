Sintaxis #

  • Nota sobre la sintaxis XML: como el registro se realiza dentro del archivo XML, no puede dejar ninguna etiqueta sin cerrar como podría hacerlo en un HTML simple, como: <link rel=‘stylesheet’ href="…" ** >**, cierre la etiqueta del enlace en su lugar, como:
  • <enlace rel=‘hoja de estilo’ href="…" />

Parámetros #

Posibles valores del parámetro inherit_id significado
web.assets_backend Usado solo en páginas internas, NO incluido en un sitio web público.
website.assets_frontend Usado solo en un sitio web público (a través del módulo “website”).
web.assets_common Utilizado tanto en el sitio web público como en las páginas internas.

Si no está seguro de qué opción es adecuada para usted, intente con la primera opción (backend) ya que se usa en la mayoría de los casos y casi en todos los casos si no ha instalado el módulo “sitio web”. Odoo diferencia entre activos “backend” y “frontend” porque el sitio web público proporcionado por el módulo “sitio web” usa un estilo y un código JS diferentes a las páginas internas destinadas a ser utilizadas para tareas de ERP, es decir, “frontend” está asociado con el sitio web público y “backend” está asociado con páginas internas para ERP (el significado de “frontend” y “backend” son específicos de Odoo aquí, pero ambos son “frontend” en un sentido más general).

No solo puede elegir y usar una de las opciones, sino también usar cualquier combinación de ellas (dos de ellas o todas) en el mismo módulo. Factorice un backend, un frontend y un código JS/CSS común en archivos separados para adherirse mejor a DRY y tener un código adecuado en el sitio web público y en las páginas internas.

No olvide agregar “web” (cuando use la opción 1) o “sitio web” (cuando use la opción 2) a la lista de dependencias en el manifiesto __openerp__.py.

Almacenar archivos CSS y JS correctamente en el módulo Odoo #

Los archivos CSS y JS deben residir en el directorio ’estático’ en el directorio raíz del módulo (el resto del árbol de subdirectorios en ’estático’ es una convención opcional):

  • estático/src/css/su_archivo.css
  • estático/src/js/su_archivo.js

Luego agregue enlaces a estos archivos utilizando una de las 3 formas enumeradas en los siguientes ejemplos.

Opción 1: [BACKEND] Agregue archivos CSS y Javascript para usar en páginas internas #

La forma de Odoo v8.0 es agregar el registro correspondiente en el archivo XML:

  • Agregar archivo XML al manifiesto (es decir, archivo __openerp__.py):

    …
    ‘data’ : [ ‘your_file.xml’ ],
    ​…

  • Luego agregue el siguiente registro en 'su_archivo.xml': <openerp> <data>
    <template id=“assets_backend” name="your_module_name assets" inherit_id=“web.assets_backend”>
    <xpath expr="." position=“inside”>
    <link rel=‘stylesheet’ href="/your_module_name/static/src/css/your_file.css"/>
    <script type=“text/javascript” src="/your_module_name/static/src/js/your_file.js"></script>
    </xpath>
    </template>
    ….
    ….
    </data>
    </openerp>

Opción 2: [FRONTEND] Agregar archivos CSS y Javascript para usar en un sitio web público #

Nota: debe usar esta forma si ha instalado un módulo de “sitio web” y tiene un sitio web público disponible.

  • Agregue el siguiente registro en 'su_archivo.xml': <openerp> <data>

          &lt;template <i>id="assets_frontend"</i> name="<strong>your_module_name</strong> assets" <i>inherit_id="website.assets_frontend"</i>&gt;  
          &lt;xpath expr="link[last()]" position="after"&gt;  
              &lt;link rel='stylesheet' href="/<strong>your_module_name</strong>/static/src/css/<strong>your_file.css</strong>"/&gt;  
          &lt;/xpath&gt;  
          &lt;xpath expr="script[last()]" position="after"&gt;  
              &lt;script type="text/javascript" src="/<strong>your_module_name</strong>/static/src/js/<strong>your_file.js</strong>"&gt;&lt;/script&gt;  
          &lt;/xpath&gt;  
      &lt;/template&gt;  

  &lt;/data&gt;

    </openerp>

Opción 3: [COMÚN] Agregue archivos CSS y Javascript para usar en todas las páginas (backend y frontend) #

  • Agregue el siguiente registro en 'su_archivo.xml': <openerp> <data>

          &lt;template id="assets_common" name="<strong>your_module_name</strong> assets" inherit_id="web.assets_common"&gt;  
          &lt;xpath expr="." position="inside"&gt;  
              &lt;link rel='stylesheet' href="/<strong>your_module_name</strong>/static/src/css/<strong>your_file.css</strong>"/&gt;  
              &lt;script type="text/javascript" src="/<strong>your_module_name</strong>/static/src/js/<strong>your_file.js</strong>"&gt;&lt;/script&gt;  
          &lt;/xpath&gt;  
      &lt;/template&gt;  

  &lt;/data&gt;

    </openerp>
