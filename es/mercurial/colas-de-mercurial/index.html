Sintaxis #

  • hg qnew -m “Mi mensaje de confirmación” myPatch
  • hg qpop
  • hg q empujar
  • hg qrefresh -m “Mi nuevo mensaje de confirmación”
  • hg qaplicado
  • serie hg q
  • hg qterminar
  • hg qborrar mi parche
  • hg qfold myPatch
  • hg qqueue –lista
  • hg qqueue –create myNewQueue
  • hg qqueue –delete myNewQueue

Habilitar extensión #

Edite Mercurial.ini (Windows) o .hgrc (Linux/OSX):

[extenstions]
mq =

Crear y actualizar parches #

Cree nuevos parches con hg qnew patch-name y luego actualícelos con nuevos cambios con hg qrefresh:

hg qnew myFirstPatch  // Creates a new patch called "myFirstPatch"
...                   // Edit some files
hg qrefresh           // Updates "myFirstPatch" with changes
hg qnew mySecondPatch // Creates a new patch called "mySecondPatch" on top of "myFirstPatch"
...                   // Edid some files
hg qrefresh           // Updates "mySecondPatch" with changes

Crear nuevo parche con mensaje de confirmación:

hg qnew -m "My first patch" myFirstPatch

Actualice el mensaje de confirmación del parche actual:

hg qrefresh -m "My new message"

Actualice el mensaje de confirmación (multilínea) del parche actual:

hg qrefresh -e

Empujar y hacer estallar parches #

Al empujar un parche, se aplica el parche al repositorio, mientras que al abrirlo, se quita la aplicación del parche del repositorio.

Retire el parche actual de la cola con hg qpop y vuelva a colocarlo en la cola con hg qpush:

hg qpop
hg qpush

Pop todos los parches:

hg qpop -a

PUSH todos los parches:

hg qpush -a

Finalizar parches #

Terminar los parches los convierte en conjuntos de cambios permanentes.

Termina el primer parche aplicado en la cola:

hg qfinish

Finalice todos los parches aplicados en la cola:

hg qfinish -a

Zorro #

Para reorganizar con los últimos conjuntos de cambios en el repositorio ascendente:

hg qpop -a    // Pop all patches
hg pull -u    // Pull in changesets from upstream repo and update
hg qpush -a   // Push all patches on top of new changesets

Si hay algún conflicto, se verá obligado a fusionar sus parches.

Doblar/Combinar Parches #

Para doblar (combinar) dos parches:

hg qnew firstPatch   // Create first patch
hg qnew secondPatch  // Create second patch
hg qpop              // Pop secondPatch
hg qfold secondPatch // Fold secondPatch with firstPatch

Múltiples Colas #

Se puede crear más de una cola. Cada cola se puede considerar como una rama separada.

hg qqueue --create foo // Create a new queue called "foo"
hg qqueue --list       // List all queues
hg qqueue --active     // Print name of active queue
hg qqueue --rename bar // Rename active queue "foo" to "bar"
hg qqueue --delete bar // delete queue "bar"

Cree dos colas y cambie entre ellas:

hg qqueue --create foo // Create a new queue called "foo" and make it active
hg qqueue --create bar // Create a new queue called "bar" and make it active
hg qqueue foo          // Switch back to queue "foo"

Comandos #

  • qnew: crea un nuevo parche
  • qpop: extrae el parche actual de la pila
  • qpush: empuja el siguiente parche a la pila
  • qrefresh: actualiza el parche actual
  • qaplicado: imprime los parches ya aplicados
  • qseries: imprime el archivo completo de la serie
  • qfinish: mueve los parches aplicados al historial del repositorio
  • qdelete: eliminar parches de la cola
  • qdiff: diferencia del parche actual y modificaciones posteriores
  • qclone: clonar el repositorio principal y el de parches al mismo tiempo
  • qfold: dobla los parches nombrados en el parche actual
  • qgoto: inserte o extraiga parches hasta que el parche con nombre esté en la parte superior de la pila
  • qguard: establecer o imprimir protecciones para un parche
  • qheader: imprime el encabezado del parche superior o especificado
  • qimport: importa un parche o un conjunto de cambios existente
  • qnext: imprime el nombre del próximo parche pushable
  • qprev: imprime el nombre del parche aplicado anterior
  • qqueue: administre múltiples colas de parches
  • qrecord: graba interactivamente un nuevo parche
  • qrename: cambiar el nombre de un parche
  • qselect: establecer o imprimir parches cuadriculados para empujar
  • qtop: imprime el nombre del parche actual
  • qunapplied: imprime los parches aún no aplicados

Obsoleto:

  • qinit: iniciar un nuevo repositorio de cola (OBSOLETO)
  • qcommit: cambios de confirmación en el repositorio de la cola (DEPRECATED)
  • qsave: guarda el estado actual de la cola (DEPRECATED)
  • qsave: restaurar el estado de cola guardado por una revisión (DEPRECATED)
