Comandos para preparar commits #

  • añadir: añadir los archivos especificados en la próxima confirmación
  • addremove: agregue todos los archivos nuevos, elimine todos los archivos faltantes
  • retroceso: efecto inverso del conjunto de cambios anterior
  • commit, ci: confirma los archivos especificados o todos los cambios pendientes
  • copiar, cp: marcar archivos como copiados para la próxima confirmación
  • olvidar: olvida los archivos especificados en la próxima confirmación
  • fusionar: fusionar otra revisión en el directorio de trabajo
  • remove, rm: elimina los archivos especificados en la próxima confirmación
  • renombrar, mover, mv: renombrar archivos; equivalente a copiar + eliminar
  • resolver: rehacer fusiones o establecer/ver el estado de fusión de los archivos
  • revertir: restaurar archivos a su estado recién desprotegido

Inspeccionando la historia #

  • anotar, culpar: muestra la información del conjunto de cambios por línea para cada archivo
  • bisect: subdivisión de búsqueda de conjuntos de cambios
  • cat: genera la revisión actual o dada de los archivos
  • diff: repositorio diff (o archivos seleccionados)
  • grep: busca un patrón en archivos y revisiones específicos
  • registro, historial: muestra el historial de revisión de todo el repositorio o archivos

Intercambio de conjuntos de cambios con repositorios remotos #

  • archivo: crea un archivo no versionado de una revisión del repositorio
  • paquete: crear un archivo de grupo de cambios
  • clonar: hacer una copia de un repositorio existente
  • exportar: volcar el encabezado y las diferencias para uno o más conjuntos de cambios
  • graft: copiar cambios de otras ramas a la rama actual
  • entrante: muestra nuevos conjuntos de cambios encontrados en la fuente
  • importar, parche: importa un conjunto ordenado de parches
  • init: crea un nuevo repositorio en el directorio dado
  • saliente: muestra los conjuntos de cambios que no se encuentran en el destino
  • fase: establece o muestra el nombre de la fase actual
  • pull: extrae los cambios de la fuente especificada
  • empujar: empujar los cambios al destino especificado
  • recuperar: revertir una transacción interrumpida
  • rollback: revertir la última transacción (PELIGROSO) (OBSECUTO)
  • servir: iniciar un servidor web independiente
  • desagregar: aplicar uno o más archivos de grupos de cambios

Estado: ¿dónde estás ahora? #

  • marcadores, marcador: cree un nuevo marcador o enumere los marcadores existentes
  • sucursal: establece o muestra el nombre de la sucursal actual
  • sucursales: lista de sucursales nombradas del repositorio
  • config, showconfig, debugconfig: muestra los ajustes de configuración combinados de todos los archivos hgrc
  • archivos: lista de archivos rastreados
  • ayuda: muestra ayuda para un tema determinado o una descripción general de la ayuda
  • identify, id: identifica el directorio de trabajo o la revisión especificada
  • entrante, en: muestra nuevos conjuntos de cambios encontrados en la fuente
  • localizar: localizar archivos que coincidan con patrones específicos (OBSOLETO)
  • manifiesto: genera la revisión actual o dada del manifiesto del proyecto
  • saliente, fuera: muestra los conjuntos de cambios que no se encuentran en el destino
  • padres: muestra los padres del directorio de trabajo o revisión (DEPRECATED)
  • rutas: mostrar alias para repositorios remotos
  • fase: establece o muestra el nombre de la fase actual
  • raíz: imprime la raíz (superior) del directorio de trabajo actual
  • estado, st: muestra los archivos modificados en el directorio de trabajo
  • summary, sum: resume el estado del directorio de trabajo
  • etiquetas: enumera las etiquetas del repositorio
  • consejo: muestra la revisión del consejo (DEPRECATED)
  • verificar: verificar la integridad del repositorio
  • versión: versión de salida e información de derechos de autor

Flujo de trabajo: ramas, etiquetas y movimiento #

  • marcadores, marcador: cree un nuevo marcador o enumere los marcadores existentes
  • sucursal: establece o muestra el nombre de la sucursal actual
  • etiqueta: agregue una o más etiquetas para la revisión actual o dada
  • update, up, checkout, co: actualizar el directorio de trabajo (o cambiar revisiones)
