¿Qué es el enrutamiento? #

En Sinatra, el enrutamiento es cómo su aplicación responde a las solicitudes, por la ruta de la solicitud (por ejemplo, /bienvenido) y por el verbo HTTP utilizado (por ejemplo, GET o POST). La forma en que se escribe una solicitud es la siguiente:

<http-verb> <path> do
    <code block to execute when this route is requested>
end

Aquí hay un ejemplo que responde a las solicitudes GET a la ruta /hello devolviendo una página que dice “Hola, qué pasa”:

get "/hello" do
    return "Hi, whats up"
end

Sinatra solo responde a las rutas que tú definas. Si no define una ruta, Sinatra devuelve una página de error 404 Página no encontrada.

Sinatra responde a las rutas en el orden en que se definen. Si tiene varias rutas que pueden coincidir con una solicitud determinada (consulte “Coincidencia de ruta basada en expresiones regulares”), se devuelve la primera ruta que se ajusta a la solicitud.

NOTA: Sinatra trata las rutas con y sin barra diagonal final (/) como 2 rutas diferentes y distintas. Es decir, get '/hello' y get '/hello/' por defecto coinciden con diferentes bloques de código. Si desea ignorar la barra diagonal final y tratar ambas rutas como iguales, puede agregar ? después de la barra diagonal para que sea opcional, así: get '/hello/?'. Esto utiliza la capacidad de Sinatra para usar expresiones regulares para la coincidencia de rutas (más sobre esto a continuación).

Verbos de enrutamiento de Sinatra disponibles #

Hay una serie de verbos de enrutamiento disponibles en Sinatra, se corresponden directamente con los verbos http

get '/' do
  .. get some data, a view, json, etc ..
end

post '/' do
  .. create a resource ..
end

put '/' do
  .. replace a resource ..
end

patch '/' do
  .. change a resource ..
end

delete '/' do
  .. delete something ..
end

options '/' do
  .. appease something ..
end

link '/' do
  .. affiliate something ..
end

unlink '/' do
  .. separate something ..
end

Coincidencia de ruta basada en expresiones regulares #

Al hacer coincidir la ruta de una ruta, puede hacerlo explícitamente, haciendo coincidir solo una ruta, así:

get "/hello" do
    return "Hello!"
end

También puede usar una expresión regular para hacer coincidir rutas complejas. Cualquier ruta que coincida con la expresión regular ejecutará ese bloque de código. Si varias rutas pueden coincidir potencialmente con la solicitud, se ejecuta la primera ruta coincidente.

Aquí hay un ejemplo típico de una ruta que coincide con rutas que incluyen /usuario/ seguido de uno o más dígitos (presumiblemente, ID de usuario), es decir, GET /usuario/1:

get /\/user\/\d+/ do
  "Hello, user!"
end

El ejemplo anterior coincide con /user/1, pero también coincidirá con /delete/user/1 y /user/1/delete/now, ya que nuestra expresión regular no es muy restrictiva y permite una coincidencia parcial contra cualquier parte del camino.

Podemos ser más explícitos con la expresión regular y decirle que coincida exactamente con la ruta, usando las directivas \A y \z para anclar la coincidencia al principio y al final de la ruta:

get /\A\/user\/\d+\z/ do
  "Hello, user!"
end

Esta ruta no coincidirá con /delete/user/1 o /user/1/delete/now debido al anclaje de coincidencias.

Ignorando el final /

Nuestra ruta de ejemplo anterior tampoco no coincidirá con /user/1/ (con barra inclinada al final). Si desea ignorar la barra diagonal final al final de la ruta, ajuste la expresión regular para que la barra diagonal sea opcional (tenga en cuenta el \/? al final):

get /\A\/user\/\d+\/?\z/ do
  "Hello, user! You may have navigated to /user/<ID> or /user/<ID>/ to get here."
end

Captura de coincidencias de ruta

Hasta ahora, hemos comparado rutas con expresiones regulares, pero ¿qué sucede si queremos usar los valores coincidentes en nuestro bloque de código? Siguiendo con nuestro ejemplo, ¿cómo sabemos cuál es el ID del usuario cuando se ejecuta la ruta?

Podemos capturar la parte deseada de la ruta y usar la variable param[:captures] de Sinatra para trabajar con los datos dentro de la ruta:

get /\A\/user\/(\d+)\/?\z/ do
  "Hello, user! Your ID is #{params['captures'].first}!"
end

Parámetros de ruta de Sinatra #

Por supuesto, puede pasar datos a las rutas de Sinatra, para aceptar datos en sus rutas, puede agregar parámetros de ruta. A continuación, puede acceder a un hash de parámetros:

get '/hello/:name' do
  # matches "GET /hello/foo" and "GET /hello/bar"
  # params['name'] is 'foo' or 'bar'
  "Hello #{params['name']}!"
end

También puede asignar parámetros directamente a las variables como solemos hacer en los hashes de Ruby:

get '/hello/:name' do |n|
  # matches "GET /hello/foo" and "GET /hello/bar"
  # params['name'] is 'foo' or 'bar'
  # n stores params['name']
  "Hello #{n}!"
end

También puede agregar parámetros comodín sin ningún nombre específico mediante el uso de asteriscos. Luego se puede acceder a ellos usando params[‘splat’]:

get '/say/*/to/*' do
  # matches /say/hello/to/world
  params['splat'] # => ["hello", "world"]
end
