Parámetros #

Parámetro Detalles
Lista de fuentes El campo en el registro de destino que se vincula con el registro de origen. Debe elegir una lista de origen antes de poder elegir su campo de origen.
Fuente de El campo en el registro de origen desde el cual se extraerán realmente los datos. El campo que elija debe coincidir con el tipo del campo de destino. Por ejemplo, si obtiene de un campo Número de teléfono, el campo de destino también debe ser un campo de Número de teléfono.

Impacto de Valor de tienda #

La configuración Almacenar valor en la definición del campo personalizado juega un papel muy importante en el comportamiento de Sourcing:

  • Cuando Almacenar valor está marcado, los datos se ingresan en el campo solo después de la creación inicial del registro. Después de eso, NetSuite rompe el vínculo de abastecimiento entre los campos y se convierten en dos campos independientes. Esto le permite aprovechar efectivamente el abastecimiento como un mecanismo para establecer el valor inicial o predeterminado de su campo personalizado.
  • Cuando Almacenar valor está sin marcar, los datos se obtienen dinámicamente en el campo cada vez que se carga el registro. Cualquier cambio que un usuario o secuencia de comandos pueda realizar en el campo nunca se guarda. Si deja Almacenar valor sin marcar, es una buena idea hacer que su campo sea de solo lectura.

Limitaciones de abastecimiento #

  • El abastecimiento no se puede aplicar a campos nativos de NetSuite. Si necesita un campo nativo como campo de destino, deberá crear un flujo de trabajo o escribir un script para realizar el abastecimiento de datos.
  • El abastecimiento no se puede aplicar a campos de columnas de sublistas. Si necesita una columna de sublista como campo de destino, deberá crear un flujo de trabajo o escribir un script para realizar el abastecimiento de datos.

Extrayendo datos a un campo personalizado en Field Changed #

// If you find yourself doing something like this...
function fieldChanged(type, name, index) {
    if (name == 'salesrep') {
        var salesRepId = nlapiGetFieldValue('salesrep');
        var salesRepEmail = nlapiLookupField('employee', salesRepId, 'email');
        nlapiSetFieldValue('custbody_salesrep_email', salesRepEmail);
    }
}
// Stop! and consider using Sourcing for your custom field instead of code

Definición de abastecimiento #

Si bien no es estrictamente un tema de SuiteScript, Sourcing es una característica increíblemente poderosa de NetSuite, y es una herramienta importante en el cinturón de herramientas para cualquier desarrollador de SuiteScript. El abastecimiento nos permite extraer datos a un registro desde cualquiera de sus registros relacionados, sin escribir ningún código ni crear un flujo de trabajo para hacerlo.

Menos código siempre es más código mantenible.

El abastecimiento se define en la pestaña Abastecimiento y filtrado de una definición de campo personalizado.
