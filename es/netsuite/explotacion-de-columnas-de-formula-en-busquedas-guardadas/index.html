Las columnas de fórmula en las búsquedas guardadas pueden aprovechar muchas características de Oracle SQL y HTML. Los ejemplos muestran cómo se pueden utilizar estas funciones, así como los peligros que se deben evitar.

Declaración CASE de Oracle SQL en una fórmula de Netsuite #

Usando una declaración CASE, muestre condicionalmente una expresión en la columna basada en los valores encontrados en otra columna, también conocido como “mi reino por un OR”. En el ejemplo, el resultado se obtiene cuando el estado de la transacción es Pendiente de cumplimiento o Parcialmente cumplido:

DECODIFICACIÓN DEL CASO ({estado}, 'Cumplimiento pendiente', 1, 'Cumplido parcialmente', 1, 0) CUANDO 1 ENTONCES expresión-1 FIN

Análisis de un nombre de registro jerárquico usando una expresión regular #

Usando una expresión regular, analice un nombre de registro que podría ser jerárquico. La expresión busca los dos puntos finales del nombre. Devuelve lo que sigue a los dos puntos, o el nombre completo si no hay ninguno:

regexp_substr({nombre}, '[^:]*$' )

Construya una cadena compleja concatenando múltiples campos #

El ejemplo genera una cadena a partir del nombre del registro principal, el nombre de este registro y la nota de este registro.

{creado a partir de} || ' ' || {nombre} || ' ' || {memorándum}

Personalice el CSS (hoja de estilo) para una columna insertando un elemento DIV #

'<div style="font-size:11pt">' || expresión || '</div>'

Proteja las fórmulas de cadenas de la corrupción y los ataques de inyección #

En un campo de fórmula de cadena, tenga en cuenta que algunos valores pueden contener subcadenas que se ven como HTML para el navegador. A menos que esto sea intencional, es importante proteger los valores de la corrupción. Esto es útil para evitar ataques de inyección: evita que alguien ingrese HTML en un campo de comentarios en un pedido web que luego se interpreta en el escritorio del representante de servicio al cliente.

htf.escape_sc( expresión )

Proteja los valores de campo de la corrupción al pasar a través de una URL #

utl_url.escape( expresión )

Probar el valor de mainline en una sentencia SQL CASE #

En una fórmula de búsqueda guardada, los posibles valores de mainline están diseñados para ser útiles en un contexto HTML. Cuando mainline es verdadero, el valor de {mainline} es la cadena de 1 carácter * (asterisco). Cuando mainline es falso, el valor de {mainline} es la cadena de 6 caracteres &nbsp; (espacio sin separación, HTML codificado como una referencia de entidad de caracteres). Estos valores de cadena se pueden comparar con literales de cadena en un contexto SQL.

CASO CUANDO {línea principal} = '*' ENTONCES expresión-cuando-verdadero CUANDO {línea principal} = '&nbsp;' ENTONCES expresión-cuando-falso FIN

Ejemplo complejo, similar al del mundo real #

El siguiente ejemplo combina varias de las técnicas cubiertas aquí. Coloca un hipervínculo en una columna con formato personalizado que, cuando se hace clic, abre el registro de pedido de ventas asociado con una fila. El hipervínculo está diseñado para abrir el registro en una nueva ventana o pestaña cuando se hace clic, y para mostrar información sobre herramientas cuando se pasa el cursor por encima. El campo internalid utilizado en la URL está protegido contra la codificación de URL. El nombre del cliente, cuando está disponible, se muestra en la misma columna, protegido de la codificación HTML.

'<div style="font-size:11pt">'
 ||
CASE {mainline}
WHEN '*' THEN '<br>' || htf.escape_sc( regexp_substr( {name} , '[^:]*$' ) ) || '<br>'
END
 ||
'<a alt="" title="Open the order associated with this line." '
 ||
'href="javascript:void(0);" onClick="window.open('''
 ||
'https://system.na1.netsuite.com/app/accounting/transactions/transaction.nl?id='
 ||
utl_url.escape( {internalid} )
 ||
''' , ''_blank'' )">'
 ||
{number}
 ||
'</a>'
 ||
'</div>'

Contar registros con y sin un valor proporcionado en un campo (contar valores faltantes y no faltantes) #

Con la función NVL2() de Oracle SQL, puede crear una columna de visualización que contenga un valor si un campo contiene datos y otro valor si un campo no contiene datos. Por ejemplo, en una búsqueda de entidad, convierta la presencia de una dirección de correo electrónico principal en una columna de visualización de texto:

NVL2 ({correo electrónico}, 'SÍ', 'NO')

Esto le permite contar registros subtotalizados por la presencia o ausencia de una dirección de correo electrónico:

Field: Internal ID
Summary Type: Count

Field: Formula (Text)
Summary Type: Group
Formula: NVL2( {email} , 'YES' , 'NO' )
