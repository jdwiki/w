Suitescript 2.0 proporciona 4 métodos para manejar los resultados de búsqueda.

Tienen diferentes sintaxis, limitaciones y gobernabilidad, y son apropiados para diferentes situaciones. Nos centraremos aquí en cómo acceder a TODOS los resultados de búsqueda, utilizando cada uno de estos métodos.

Usando el método Search.ResultSet.each #

Este es el método más corto, más fácil y más comúnmente utilizado. Desafortunadamente, tiene una limitación importante: no se puede usar en búsquedas con más de 4000 resultados (filas).

    // Assume that 'N/search' module is included as 'search'
    
    var s = search.create({
        type : search.Type.TRANSACTION,
        columns : ['entity','amount'],
        filters : [  ['mainline', 'is', 'T'],
              'and', ['type', 'is', 'CustInvc'],
              'and', ['status', 'is', 'open']
            ]
    });

    var resultSet = s.run();
    
    // you can use "each" method on searches with up to 4000 results    
    resultSet.each( function(result) {
        
        // you have the result row. use it like this....
        var transId = result.id;
        var entityId = result.getValue('entity'); 
        var entityName = result.getText('entity');
        var amount = result.getValue('amount');
        
        // don't forget to return true, in order to continue the loop
        return true;  

    });

Uso del método ResultSet.getRange #

Para usar getRange para manejar la gran cantidad de resultados, tendremos que considerar lo siguiente:

  1. getRange tiene 2 parámetros: inicio y finalización. Siempre positivo, siempre (inicio <fin)
  2. inicio es el índice inclusivo del primer resultado que se devuelve
  3. fin es el índice exclusivo del último resultado a devolver
  4. Si hay menos resultados disponibles de los solicitados, la matriz contendrá menos entradas que final - inicio. Por ejemplo, si solo hay 25 resultados de búsqueda, getRange(20, 30) devolverá una matriz de 5 objetos search.Result.
  5. Aunque la oración de ayuda anterior no lo dice directamente, tanto comienzo como finalización podrían estar fuera del rango de resultados disponibles. En el mismo ejemplo, si solo hay 25 resultados de búsqueda, getRange(100, 200) devolverá una matriz vacía
  6. Máximo 1000 filas a la vez. (fin - inicio) <= 1000
    // Assume that 'N/search' module is included as 'search'
    
    // this search will return a lot of results (not having any filters) 
    var s = search.create({
        type: search.Type.TRANSACTION,
        columns : ['entity','amount'],
        filters: [] 
    });
    
    var resultSet = s.run();

    // now take the first portion of data.
    var currentRange = resultSet.getRange({
            start : 0,
            end : 1000
    });
    
    var i = 0;  // iterator for all search results
    var j = 0;  // iterator for current result range 0..999

    while ( j < currentRange.length ) {
        
        // take the result row
        var result = currentRange[j];
        // and use it like this....
        var transId = result.id;
        var entityId = result.getValue('entity'); 
        var entityName = result.getText('entity');
        var amount = result.getValue('amount');
        
        // finally:
        i++; j++;
        if( j==1000 ) {   // check if it reaches 1000
            j=0;          // reset j an reload the next portion
            currentRange = resultSet.getRange({
                start : i,
                end : i+1000
            });
        }
    }

Calculemos la Gobernanza. Tenemos 1 + conteo/1000 llamadas getRange que toman 10 unidades cada una, entonces:

G = (1 + recuento/1000) * 10

Ejemplo: 9500 filas tomarán 100 unidades

Usando el método Search.PagedData.fetch #

PagedData es un objeto, devuelto por el método Search.runPaged(options). Funciona exactamente como lo hacen las búsquedas de la interfaz de usuario. El objeto PagedData contiene 2 propiedades importantes, que puede ver en el lado derecho del encabezado de resultados en la página de resultados de búsqueda en la interfaz de usuario de Netsuite:

  • recuento (el número total de resultados)
  • pageRanges (lista de páginas, disponible en la interfaz de usuario como selector de cuadro combinado)

El parámetro options.pageSize se limita de nuevo a 1000 filas de resultados.
El método PagedData.fetch se utiliza para obtener la porción de resultados que desea (indexada por el parámetro pageIndex). Con un poco más de código, recibe la misma función de devolución de llamada conveniente que Search.ResultSet.each, sin tener la limitación de 4000 filas.

    // Assume that 'N/search' module is included as 'search'

    // this search will return a lot of results (not having any filters)  
    var s = search.create({
        type: search.Type.TRANSACTION,
        columns : ['entity','amount'],
        filters : [] 
    });
    
    var pagedData = s.runPaged({pa​g​e​S​i​z​e : 1000});
    
    // iterate the pages
    for( var i=0; i < pagedData.pageRanges.length; i++ ) {

        // fetch the current page data
        var currentPage = pagedData.fetch(i);

        // and forEach() thru all results
        currentPage.data.forEach( function(result) {

            // you have the result row. use it like this....
            var transId = result.id;
            var entityId = result.getValue('entity'); 
            var entityName = result.getText('entity');
            var amount = result.getValue('amount');

        });

    }

Calculemos la Gobernanza. Tenemos 5 unidades para runPaged(), y 1 + count/1000 pagedData.fetch llamadas que toman 5 unidades cada una, entonces:

G = 5 + límite (cuenta/1000) * 5

Ejemplo: 9500 filas tomarán 55 unidades. Aproximadamente la mitad de las unidades de gobierno de getRange.

Uso de un script Map/Reduce dedicado #

Para resultados de búsqueda realmente grandes, puede usar un script Map/Reduce dedicado. Es mucho más inconveniente, pero a veces inevitable. Y a veces puede ser muy útil.
El truco aquí es que en la etapa Obtener datos de entrada, puede proporcionar al motor NS no los datos reales (es decir, el resultado del script), sino solo la definición de la búsqueda. NS ejecutará la búsqueda por usted sin contar las unidades de gobierno. Luego, cada fila de resultados se pasará a la etapa Mapa.
Por supuesto, existe una limitación: el tamaño total persistente de los datos para un script de asignación/reducción no puede exceder los 50 MB. En un resultado de búsqueda, cada clave y el tamaño serializado de cada valor se cuentan para el tamaño total. “Serializado” significa que la fila de resultados de búsqueda se convierte en una cadena con JSON.stringify. Por lo tanto, el tamaño del valor es proporcional al número de columnas de resultados de búsqueda en un conjunto de resultados. Si tiene problemas con el error STORAGE_SIZE_EXCEEDED, considere reducir las columnas, combinarlas con fórmulas, agrupar el resultado o incluso dividir la búsqueda en varias subbúsquedas, que podrían ejecutarse en las etapas Map o Reduce.

    /**
     * @NApiVersion 2.0
     * @NScriptType MapReduceScript
     */
    define(['N/search'], function(search) {

    function getInputData()
    {
        return search.create({
            type: search.Type.TRANSACTION,
            columns : ['entity','amount'],
            filters : [] 
        });
    }

    function map(context)
    {
        var searchResult = JSON.parse(context.value);
        // you have the result row. use it like this....
        var transId = searchResult.id;
        var entityId = searchResult.values.entity.value;
        var entityName = searchResult.values.entity.text;
        var amount = searchResult.values.amount.value;
        
        // if you want to pass some part of the search result to the next stage
        // write it to context:
        context.write(entityId, transId);
    }

    function reduce(context)
    {
       // your code here ...
    }

    function summarize(summary)
    {
        // your code here ...
    }

    return {
        getInputData: getInputData,
        map: map,
        reduce: reduce,
        summarize: summarize
    };
});

Por supuesto, el ejemplo aquí está simplificado, sin manejo de errores y se da solo para compararlo con otros. Hay más ejemplos disponibles en Ejemplos de Map/Reduce Script Type en el Centro de ayuda de NS
