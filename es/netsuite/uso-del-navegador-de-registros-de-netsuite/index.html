Uso del navegador de registros de NetSuite #

El Explorador de registros define el esquema para todos los tipos de registros programables; es una herramienta de referencia extremadamente crítica para todos los desarrolladores de SuiteScript. Cuando necesite saber cómo hacer referencia a un campo en particular en un tipo de registro específico en su secuencia de comandos, el Explorador de registros es su guía.

Enlace directo

Otro esquema #

También puede notar pestañas en la parte superior del Explorador de registros para Navegador de esquemas y Navegador de conexión. Estos son muy similares al Explorador de registros, pero para diferentes API de NetSuite.

Schema Browser proporciona el esquema para la API de servicios web basada en SOAP, mientras que Connect Browser proporciona el esquema para el conector ODBC.

Navegación por el navegador de registros #

Explora el Explorador de registros primero por Tipo de registro, es decir, “Orden de venta”, “Factura”, “Empleado”. No hay capacidad de búsqueda dentro del Explorador de registros, por lo que toda la navegación se realiza manualmente. Los tipos de registro están organizados alfabéticamente, por lo que primero debe hacer clic en la primera letra del tipo de registro que le interesa y luego seleccionar el tipo de registro a la izquierda.

Por ejemplo, si quisiera ver el esquema para el tipo de registro Subsidiaria, primero haría clic en S en la parte superior, luego en Subsidiaria a la izquierda.

Lectura del esquema #

Cada esquema le proporciona una cantidad abrumadora de información sobre cada tipo de registro. Es importante saber desglosar toda esta información.

En la parte superior del esquema se encuentra el nombre del tipo de registro seguido del ID interno del tipo de registro; este ID interno es la referencia programática para el tipo de registro. A continuación, el esquema se divide en varias secciones:

  • Campos: La sección Campos enumera los detalles de todos los campos del cuerpo del registro. Los campos que se describen aquí se pueden utilizar cuando se trabaja con el registro actualmente en contexto o con una referencia directa a un objeto de registro.
  • Sublistas: La sección Sublistas muestra todas las sublistas en el registro y cada columna programable dentro de cada sublista. Los campos de esta sección se aplican nuevamente cuando está trabajando con el registro actualmente en contexto, o con una referencia directa a un objeto de registro.
  • Fichas: la sección Fichas describe todas las subfichas nativas del tipo de registro.
  • Buscar uniones: la sección Buscar uniones describe todos los registros relacionados a través de los cuales puede crear uniones en sus búsquedas de este tipo de registro.
  • Filtros de búsqueda: La sección Filtros de búsqueda describe todos los campos que están disponibles como filtro de búsqueda para este tipo de registro. El ID interno cuando se usa un campo específico como filtro de búsqueda no siempre coincide con su ID interno como campo de cuerpo.
  • Columnas de búsqueda: La sección Columnas de búsqueda describe todos los campos que están disponibles como columna de búsqueda para este tipo de registro. El ID interno cuando se usa un campo específico como columna de búsqueda no siempre coincide con su ID interno como campo de cuerpo.
  • Tipos de transformación: la sección Tipos de transformación describe todos los tipos de registros en los que se puede transformar este mediante la API de transformación de registros.

Encontrar un campo #

Como se indicó anteriormente, no hay una capacidad de búsqueda integrada en el Explorador de registros. Una vez que haya navegado hasta el tipo de registro apropiado, si aún no conoce el ID interno de un campo en particular, la forma más fácil de encontrarlo es usar la función Buscar de su navegador (generalmente CTRL+F) para ubicar el campo por su nombre en la interfaz de usuario.

Campos requeridos #

La columna Obligatorio del esquema indica si este campo es obligatorio para guardar el registro. Si esta columna dice verdadero, deberá proporcionar un valor para este campo al guardar cualquier registro de este tipo.

nlapiSubmitField y edición en línea #

La columna nlapiSubmitField es una pieza crítica para entender. Esta columna indica si el campo está disponible para la edición en línea. Si nlapiSubmitField es true, entonces el campo se puede editar en línea. Esto afecta en gran medida la forma en que se maneja este campo cuando se intenta usar las funciones nlapiSubmitField o record.submitFields en sus scripts.

Cuando esta columna es verdadera, puede usar con seguridad las API de envío de campos para actualizar este campo en línea. Cuando es falso, aún puede usar estas funciones para actualizar el campo, pero lo que realmente sucede detrás de escena cambia significativamente.

Cuando nlapiSubmitField es falso para un campo en particular, y utiliza una de las API de envío de campos en él, el motor de secuencias de comandos detrás de escena realmente hará una carga completa del registro, actualizará el campo y devolverá el cambio. a la base de datos. El resultado final es el mismo, pero debido a que todo el registro se carga y se guarda, su secuencia de comandos en realidad utilizará mucho más control de lo que podría esperar y tardará más en ejecutarse.

Puede leer sobre esto con más detalle en la página de ayuda titulada “Consecuencias del uso de nlapiSubmitField en campos editables no en línea”.
