“Un valor opcional contiene un valor o contiene cero para indicar que falta un valor”

Extracto de: Apple Inc. “El lenguaje de programación Swift (edición Swift 3.1)”. iBooks. https://itun.es/us/k5SW7.l

Los casos de uso opcionales básicos incluyen: para una constante (let), el uso de un opcional dentro de un ciclo (if-let), el desenvolvimiento seguro de un valor opcional dentro de un método (guard-let) y como parte de los bucles de conmutación (case-let ), por defecto a un valor si es nulo, usando el operador coalesce (??)

Sintaxis #

  • var nombreOpcional: tipoOpcional? // declara un tipo opcional, por defecto es nil
  • var nombreOpcional: tipoOpcional? = valor // declara un opcional con un valor
  • var nombreOpcional: tipoOpcional! // declara un opcional implícitamente desenvuelto
  • ¡opcional! // fuerza a desenvolver un opcional

Para obtener más información acerca de los opcionales, consulte El lenguaje de programación Swift.

Tipos de Opcionales #

Los opcionales son un tipo de enumeración genérico que actúa como contenedor. Este contenedor permite que una variable tenga uno de dos estados: el valor del tipo definido por el usuario o nil, que representa la ausencia de un valor.

Esta capacidad es particularmente importante en Swift porque uno de los objetivos de diseño declarados del lenguaje es funcionar bien con los marcos de trabajo de Apple. Muchos (la mayoría) de los marcos de trabajo de Apple utilizan nil debido a su facilidad de uso y su importancia para los patrones de programación y el diseño de API dentro de Objective-C.

En Swift, para que una variable tenga un valor nil, debe ser opcional. Los opcionales se pueden crear agregando un ! o un ? al tipo de variable. Por ejemplo, para hacer que un ‘Int’ sea opcional, podrías usar

var numberOne: Int! = nil
var numberTwo: Int? = nil

Los opcionales ? deben desencapsularse explícitamente y deben usarse si no está seguro de si la variable tendrá o no un valor cuando acceda a ella. Por ejemplo, al convertir una cadena en un Int, el resultado es un Int? opcional, porque se devolverá nil si la cadena no es un número válido.

let str1 = "42"
let num1: Int? = Int(str1) // 42

let str2 = "Hello, World!"
let num2: Int? = Int(str2) // nil

Las opciones ! se desenvuelven automáticamente, y solo deben usarse cuando está seguro de que la variable tendrá un valor cuando acceda a ella. Por ejemplo, una variable global UIButton! que se inicializa en viewDidLoad()

//myButton will not be accessed until viewDidLoad is called,
//so a ! optional can be used here
var myButton: UIButton!

override func viewDidLoad(){
    self.myButton = UIButton(frame: self.view.frame)
    self.myButton.backgroundColor = UIColor.redColor()
    self.view.addSubview(self.myButton)
}

Desenvolviendo un Opcional #

Para acceder al valor de un Opcional, debe desenvolverse.

Puede desenvolver condicionalmente un Opcional usando un enlace opcional y forzar el desenvolvimiento de un Opcional usando el operador !.

El desenvolvimiento condicional efectivamente pregunta “¿Esta variable tiene un valor?” mientras que forzar el desenvolvimiento dice “¡Esta variable tiene un valor!”.

Si fuerza el desenvolvimiento de una variable que es nil, su programa lanzará un inesperadamente encontrado nulo mientras desenvuelve una excepción opcional y fallará, por lo que debe considerar cuidadosamente si usar ! es apropiado.

var text: String? = nil
var unwrapped: String = text! //crashes with "unexpectedly found nil while unwrapping an Optional value"

Para un desenvolvimiento seguro, puede usar una instrucción if-let, que no generará una excepción ni fallará si el valor envuelto es nil:

var number: Int?
if let unwrappedNumber = number {       // Has `number` been assigned a value?
    print("number: \(unwrappedNumber)") // Will not enter this line
} else {
    print("number was not assigned a value")
}

O, una declaración de guardia:

var number: Int?
guard let unwrappedNumber = number else {
    return
}
print("number: \(unwrappedNumber)")

Tenga en cuenta que el alcance de la variable unwrappedNumber está dentro de la instrucción if-let y fuera del bloque guard.

Puede encadenar el desenvolvimiento de muchas opciones, esto es principalmente útil en los casos en que su código requiere más que una variable para ejecutarse correctamente:

var firstName:String?
var lastName:String?

if let fn = firstName, let ln = lastName {
    print("\(fn) + \(ln)")//pay attention that the condition will be true only if both optionals are not nil.
}

Tenga en cuenta que todas las variables se deben desenvolver para pasar con éxito la prueba; de lo contrario, no tendría forma de determinar qué variables se desenvolvieron y cuáles no.

Puede encadenar sentencias condicionales usando sus opcionales inmediatamente después de abrirlas. ¡Esto significa que no hay declaraciones if - else anidadas!

var firstName:String? = "Bob"
var myBool:Bool? = false

if let fn = firstName, fn == "Bob", let bool = myBool, !bool {
    print("firstName is bob and myBool was false!")
}

1: https://www.wikiod.com/es/swift/condicionales#Usando Guardia

Operador coalescente nulo #

Puede usar el operador coalescente nulo para desenvolver un valor si no es nulo; de lo contrario, proporcione un valor diferente:

func fallbackIfNil(str: String?) -> String {
    return str ?? "Fallback String"
}
print(fallbackIfNil("Hi")) // Prints "Hi"
print(fallbackIfNil(nil)) // Prints "Fallback String"

Este operador puede cortocircuitar, lo que significa que si el operando de la izquierda no es nulo, el operando de la derecha no se evaluará:

func someExpensiveComputation() -> String { ... }

var foo : String? = "a string"
let str = foo ?? someExpensiveComputation()

En este ejemplo, como foo no es nil, no se llamará a someExpensiveComputation().

También puede encadenar varias declaraciones de fusión nulas juntas:

var foo : String?
var bar : String?

let baz = foo ?? bar ?? "fallback string"

En este ejemplo, a baz se le asignará el valor no ajustado de foo si no es nulo; de lo contrario, se le asignará el valor no ajustado de bar si no es nulo; de lo contrario, se le asignará el valor alternativo .

Encadenamiento opcional #

Puede usar Encadenamiento opcional para llamar a un método, acceder a una propiedad o subíndice como opcional. Esto se hace colocando un ? entre la variable opcional dada y el miembro dado (método, propiedad o subíndice).

struct Foo {
    func doSomething() {
        print("Hello World!")
    }
}

var foo : Foo? = Foo()

foo?.doSomething() // prints "Hello World!" as foo is non-nil

Si foo contiene un valor, se llamará a doSomething(). Si foo es nil, entonces no sucederá nada malo: el código simplemente fallará en silencio y continuará ejecutándose.

var foo : Foo? = nil

foo?.doSomething() // will not be called as foo is nil

(Este es un comportamiento similar al envío de mensajes a nil en Objective-C)

La razón por la que el encadenamiento opcional se denomina así es porque la “opcionalidad” se propagará a través de los miembros a los que llame/acceda. Lo que esto significa es que los valores devueltos de cualquier miembro que se use con el encadenamiento opcional serán opcionales, independientemente de si se escriben como opcionales o no.

struct Foo {
    var bar : Int
    func doSomething() { ... }
}

let foo : Foo? = Foo(bar: 5)
print(foo?.bar) // Optional(5)

Aquí foo?.bar devuelve un Int? aunque bar no es opcional, ya que foo sí lo es.

A medida que se propaga la opcionalidad, los métodos que devuelven Void devolverán Void? cuando se llamen con un encadenamiento opcional. Esto puede ser útil para determinar si el método fue llamado o no (y por lo tanto si el opcional tiene un valor).

let foo : Foo? = Foo()

if foo?.doSomething() != nil {
    print("foo is non-nil, and doSomething() was called")
} else {
    print("foo is nil, therefore doSomething() wasn't called")
}

Aquí estamos comparando el valor devuelto de Void? con nil para determinar si se llamó al método (y por lo tanto si foo no es nil).

Resumen - ¿Por qué opcionales? #

A menudo, al programar es necesario hacer alguna distinción entre una variable que tiene un valor y otra que no lo tiene. Para los tipos de referencia, como los punteros C, se puede usar un valor especial como null para indicar que la variable no tiene valor. Para tipos intrínsecos, como un número entero, es más difícil. Se puede usar un valor designado, como -1, pero depende de la interpretación del valor. También elimina ese valor “especial” del uso normal.

Para abordar esto, Swift permite declarar cualquier variable como opcional. Esto se indica mediante el uso de un ? o ! después del tipo (Ver Tipos de opcionales)

Por ejemplo,

var possiblyInt: Int?

declara una variable que puede contener o no un valor entero.

El valor especial nil indica que actualmente no hay ningún valor asignado a esta variable.

possiblyInt = 5      // PossiblyInt is now 5
possiblyInt = nil    // PossiblyInt is now unassigned

nil también se puede usar para probar un valor asignado:

if possiblyInt != nil {
    print("possiblyInt has the value \(possiblyInt!)")
}

Tenga en cuenta el uso de ! en la instrucción de impresión para desenvolver el valor opcional.

Como ejemplo de un uso común de opcionales, considere una función que devuelve un número entero de una cadena que contiene dígitos; Es posible que la cadena contenga caracteres que no sean dígitos o que incluso esté vacía.

¿Cómo puede una función que devuelve un simple ‘Int’ indicar falla? No puede hacerlo devolviendo ningún valor específico, ya que esto impediría que ese valor se analice de la cadena.

var someInt
someInt = parseInt("not an integer") // How would this function indicate failure?

En Swift, sin embargo, esa función puede simplemente devolver un Int opcional. Entonces la falla se indica mediante el valor de retorno de nil.

var someInt?
someInt = parseInt("not an integer")  // This function returns nil if parsing fails
if someInt == nil {
    print("That isn't a valid integer")
}
