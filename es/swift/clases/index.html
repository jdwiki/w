Definición de una clase #

Usted define una clase como esta:

class Dog {}

Una clase también puede ser una subclase de otra clase:

class Animal {}
class Dog: Animal {}

En este ejemplo, ‘Animal’ también podría ser un protocolo al que se ajusta ‘Perro’.

Propiedades y Métodos #

Las clases pueden definir propiedades que las instancias de la clase pueden usar. En este ejemplo, Dog tiene dos propiedades: name y dogYearAge:

class Dog {
    var name = ""
    var dogYearAge = 0
}

Puede acceder a las propiedades con la sintaxis de puntos:

let dog = Dog()
print(dog.name)
print(dog.dogYearAge)

Las clases también pueden definir métodos que se pueden llamar en las instancias, se declaran de manera similar a las funciones normales, solo dentro de la clase:

class Dog {
    func bark() {
        print("Ruff!")
    }
}

Los métodos de llamada también utilizan la sintaxis de puntos:

dog.bark()

Semántica de referencia #

Las clases son tipos de referencia, lo que significa que varias variables pueden hacer referencia a la misma instancia.

clase Perro {
    var name = ""
}

let primerPerro = Perro()
primerPerro.nombre = "Fido"

let otherDog = firstDog ​​// otherDog apunta a la misma instancia de Dog
otherDog.name = "Rover" // modificando otherDog también modifica firstDog

print(primerPerro.nombre) // imprime "Rover"

Debido a que las clases son tipos de referencia, incluso si la clase es una constante, sus propiedades variables aún se pueden modificar.

class Dog {
    var name: String // name is a variable property.
    let age: Int // age is a constant property.
    init(name: String, age: Int) {
        self.name = name
        self.age = age
    }
}

let constantDog = Dog(name: "Rover", age: 5)// This instance is a constant.
var variableDog = Dog(name: "Spot", age 7)// This instance is a variable.

constantDog.name = "Fido" // Not an error because name is a variable property.
constantDog.age = 6 // Error because age is a constant property.
constantDog = Dog(name: "Fido", age: 6)
/* The last one is an error because you are changing the actual reference, not
just what the reference points to. */

variableDog.name = "Ace" // Not an error because name is a variable property.
variableDog.age = 8 // Error because age is a constant property.
variableDog = Dog(name: "Ace", age: 8)
/* The last one is not an error because variableDog is a variable instance and
therefore the actual reference can be changed. */

Prueba si dos objetos son idénticos (apunta exactamente a la misma instancia) usando ===:

class Dog: Equatable {
    let name: String
    init(name: String) { self.name = name }
}

// Consider two dogs equal if their names are equal.
func ==(lhs: Dog, rhs: Dog) -> Bool {
    return lhs.name == rhs.name
}

// Create two Dog instances which have the same name.
let spot1 = Dog(name: "Spot")
let spot2 = Dog(name: "Spot")

spot1 == spot2   // true, because the dogs are equal
spot1 != spot2   // false

spot1 === spot2  // false, because the dogs are different instances
spot1 !== spot2  // true

Clases y Herencia Múltiple #

Swift no admite la herencia múltiple. Es decir, no puede heredar de más de una clase.

class Animal { ... }
class Pet { ... }

class Dog: Animal, Pet { ... } // This will result in a compiler error.

En su lugar, se le anima a utilizar la composición al crear sus tipos. Esto se puede lograr usando protocolos.
