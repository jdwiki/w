Sintaxis #

  • NSJSONSerialization.JSONObjectWithData(jsonData, opciones: NSJSONReadingOptions) // Devuelve un objeto de jsonData. Este método arroja fallas.
  • NSJSONSerialization.dataWithJSONObject(jsonObject, opciones: NSJSONWritingOptions) // Devuelve NSData de un objeto JSON. Pase NSJSONWritingOptions.PrettyPrinted en opciones para una salida que sea más legible.

Serialización, codificación y decodificación de JSON con Apple Foundation y Swift Standard Library #

La clase JSONSerialization está integrada en el marco Foundation de Apple.

Leer JSON #

La función JSONObjectWithData toma NSData y devuelve AnyObject. Puede usar as? para convertir el resultado a su tipo esperado.

do {
    guard let jsonData = "[\"Hello\", \"JSON\"]".dataUsingEncoding(NSUTF8StringEncoding) else {
        fatalError("couldn't encode string as UTF-8")
    }

    // Convert JSON from NSData to AnyObject
    let jsonObject = try NSJSONSerialization.JSONObjectWithData(jsonData, options: [])
    
    // Try to convert AnyObject to array of strings
    if let stringArray = jsonObject as? [String] {
        print("Got array of strings: \(stringArray.joinWithSeparator(", "))")
    }
} catch {
    print("error reading JSON: \(error)")
}

Puede pasar options: .AllowFragments en lugar de options: [] para permitir la lectura de JSON cuando el objeto de nivel superior no es una matriz o un diccionario.

Escribir JSON #

Llamar a dataWithJSONObject convierte un objeto compatible con JSON (matrices anidadas o diccionarios con cadenas, números y NSNull) en NSData sin formato codificado como UTF-8.

do {
    // Convert object to JSON as NSData
    let jsonData = try NSJSONSerialization.dataWithJSONObject(jsonObject, options: [])
    print("JSON data: \(jsonData)")
    
    // Convert NSData to String
    let jsonString = String(data: jsonData, encoding: NSUTF8StringEncoding)!
    print("JSON string: \(jsonString)")
} catch {
    print("error writing JSON: \(error)")
}

Puede pasar opciones: .PrettyPrinted en lugar de opciones: [] para una impresión bonita.

Mismo comportamiento en Swift 3 pero con una sintaxis diferente.

do {
    guard let jsonData = "[\"Hello\", \"JSON\"]".data(using: String.Encoding.utf8) else {
        fatalError("couldn't encode string as UTF-8")
    }
    
    // Convert JSON from NSData to AnyObject
    let jsonObject = try JSONSerialization.jsonObject(with: jsonData, options: [])
    
    // Try to convert AnyObject to array of strings
    if let stringArray = jsonObject as? [String] {
        print("Got array of strings: \(stringArray.joined(separator: ", "))")
    }
} catch {
    print("error reading JSON: \(error)")
}

do {
    // Convert object to JSON as NSData
    let jsonData = try JSONSerialization.data(withJSONObject: jsonObject, options: [])
    print("JSON data: \(jsonData)")

    // Convert NSData to String
    let jsonString = String(data: jsonData, encoding: .utf8)!
    print("JSON string: \(jsonString)")
} catch {
    print("error writing JSON: \(error)")
}

Nota: Actualmente, Lo siguiente solo está disponible en Swift 4.0 y versiones posteriores.

A partir de Swift 4.0, la biblioteca estándar de Swift incluye los protocolos Encodable y [Decodable](https://developer.apple.com /documentation/swift/decodable) para definir un enfoque estandarizado para la codificación y decodificación de datos. La adopción de estos protocolos permitirá implementaciones de Encoder y [Decoder](https://developer.apple.com/documentation/swift /decodificador) toman sus datos y los codifican o decodifican hacia y desde una representación externa como JSON. La conformidad con el protocolo Codable combina los protocolos Encodable y Decodable. Este es ahora el medio recomendado para manejar JSON en su programa.

Codificar y decodificar automáticamente #

La forma más sencilla de hacer que un tipo sea codificable es declarar sus propiedades como tipos que ya son Codificables. Estos tipos incluyen tipos de biblioteca estándar como String, Int y Double; y tipos básicos como Fecha, Datos y URL. Si las propiedades de un tipo son codificables, el tipo mismo se ajustará automáticamente a “Codificable” simplemente declarando la conformidad.

Considere el siguiente ejemplo, en el que la estructura Book se ajusta a Codable.

struct Book: Codable {
    let title: String
    let authors: [String]
    let publicationDate: Date
}

Tenga en cuenta que las colecciones estándar como Array y Dictionary se ajustan a Codable si contienen tipos codificables.

Al adoptar ‘Codable’, la estructura ‘Book’ ahora se puede codificar y decodificar desde JSON utilizando las clases de Apple Foundation ‘JSONEncoder’ y ‘JSONDecoder’, aunque ‘Book’ en sí mismo no contiene código para manejar específicamente JSON. También se pueden escribir codificadores y decodificadores personalizados, conforme a los protocolos Encoder y Decoder, respectivamente.

Codificar a datos JSON #

// Create an instance of Book called book
let encoder = JSONEncoder()
let data = try! encoder.encode(book) // Do not use try! in production code
print(data)

Configure encoder.outputFormatting = .prettyPrinted para facilitar la lectura. ## Decodificación de datos JSON

Decodificación de datos JSON #

// Retrieve JSON string from some source
let jsonData = jsonString.data(encoding: .utf8)!
let decoder = JSONDecoder()
let book = try! decoder.decode(Book.self, for: jsonData) // Do not use try! in production code
print(book)

En el ejemplo anterior, Book.self informa al decodificador del tipo al que debe decodificarse el JSON.

Codificación o decodificación exclusivamente #

A veces, es posible que no necesite que los datos sean codificables y decodificables, como cuando solo necesita leer datos JSON de una API, o si su programa solo envía datos JSON a una API.

Si solo tiene la intención de escribir datos JSON, ajuste su tipo a Codificable.

struct Book: Encodable {
    let title: String
    let authors: [String]
    let publicationDate: Date
}

Si solo tiene la intención de leer datos JSON, ajuste su tipo a Decodable.

struct Book: Decodable {
    let title: String
    let authors: [String]
    let publicationDate: Date
}

Uso de nombres de clave personalizados #

Las API usan con frecuencia convenciones de nomenclatura distintas al caso de camello estándar de Swift, como el caso de serpiente. Esto puede convertirse en un problema cuando se trata de decodificar JSON, ya que, de forma predeterminada, las claves JSON deben alinearse exactamente con los nombres de propiedad de su tipo. Para manejar estos escenarios, puede crear claves personalizadas para su tipo utilizando el protocolo CodingKey.

struct Book: Codable {
    // ...
    enum CodingKeys: String, CodingKey { 
        case title
        case authors
        case publicationDate = "publication_date"
    }
}

Las CodingKeys se generan automáticamente para los tipos que adoptan el protocolo Codable, pero al crear nuestra propia implementación en el ejemplo anterior, permitimos que nuestro decodificador haga coincidir el caso de camello local publicationDate con el caso de serpiente publication_date como es entregado por la API.

##SwiftyJSON SwiftyJSON es un marco Swift creado para eliminar la necesidad de un encadenamiento opcional en la serialización JSON normal.

Puede descargarlo aquí: https://github.com/SwiftyJSON/SwiftyJSON

Sin SwiftyJSON, su código se vería así para encontrar el nombre del primer libro en un objeto JSON:

if let jsonObject = try NSJSONSerialization.JSONObjectWithData(data, options: .AllowFragments) as? [[String: AnyObject]],
let bookName = (jsonObject[0]["book"] as? [String: AnyObject])?["name"] as? String {
    //We can now use the book name
}

En SwiftyJSON, esto se simplifica enormemente:

let json = JSON(data: data)
if let bookName = json[0]["book"]["name"].string {
    //We can now use the book name
}

Elimina la necesidad de verificar todos los campos, ya que devolverá cero si alguno de ellos no es válido.

Para usar SwiftyJSON, descargue la versión correcta del repositorio de Git; hay una rama para Swift 3. Simplemente arrastre “SwiftyJSON.swift” a su proyecto e impórtelo a su clase:

import SwiftyJSON

Puede crear su objeto JSON utilizando los siguientes dos inicializadores:

let jsonObject = JSON(data: dataObject)

o

let jsonObject = JSON(jsonObject) //This could be a string in a JSON format for example

Para acceder a sus datos, utilice subíndices:

let firstObjectInAnArray = jsonObject[0]
let nameOfFirstObject = jsonObject[0]["name"]

A continuación, puede analizar su valor a un determinado tipo de datos, que devolverá un valor opcional:

let nameOfFirstObject = jsonObject[0]["name"].string //This will return the name as a string

let nameOfFirstObject = jsonObject[0]["name"].double //This will return null

También puede compilar sus rutas en una matriz rápida:

let convolutedPath = jsonObject[0]["name"][2]["lastName"]["firstLetter"].string

Es lo mismo que:

let convolutedPath = jsonObject[0, "name", 2, "lastName", "firstLetter"].string

SwiftyJSON también tiene funcionalidad para imprimir sus propios errores:

if let name = json[1337].string {
    //You can use the value - it is valid
} else {
    print(json[1337].error) // "Array[1337] is out of bounds" - You cant use the value
}

Si necesita escribir en su objeto JSON, puede usar subíndices nuevamente:

var originalJSON:JSON = ["name": "Jack", "age": 18]
originalJSON["age"] = 25 //This changes the age to 25
originalJSON["surname"] = "Smith" //This creates a new field called "surname" and adds the value to it

Si necesita la cadena original para el JSON, por ejemplo, si necesita escribirla en un archivo, puede obtener el valor sin procesar:

if let string = json.rawString() { //This is a String object
    //Write the string to a file if you like
}

if let data = json.rawData() { //This is an NSData object
    //Send the data to your server if you like
}

Freddy #

Freddy es una biblioteca de análisis JSON mantenida por Big Nerd Ranch. Tiene tres ventajas principales:

  1. Escriba Seguridad: lo ayuda a trabajar con el envío y la recepción de JSON de una manera que evita fallas en el tiempo de ejecución.

  2. Idiomático: aprovecha las características genéricas, enumeraciones y funcionales de Swift, sin documentación complicada ni operadores personalizados mágicos.

  3. Manejo de errores: proporciona información de error informativa para errores JSON que ocurren comúnmente.

Ejemplo de datos JSON #

Definamos algunos datos JSON de ejemplo para usar con estos ejemplos.

{
  "success": true,
  "people": [
    {
      "name": "Matt Mathias",
      "age": 32,
      "spouse": true
    },
    {
      "name": "Sergeant Pepper",
      "age": 25,
      "spouse": false
    }
  ],
  "jobs": [
    "teacher",
    "judge"
  ],
  "states": {
    "Georgia": [
      30301,
      30302,
      30303
    ],
    "Wisconsin": [
      53000,
      53001
    ]
  }
}

let jsonString = "{\"success\": true, \"people\": [{\"name\": \"Matt Mathias\",\"age\": 32,\"spouse\": true},{\"name\": \"Sergeant Pepper\",\"age\": 25,\"spouse\": false}],\"jobs\": [\"teacher\",\"judge\"],\"states\": {\"Georgia\": [30301,30302,30303],\"Wisconsin\": [53000,53001]}}"
let jsonData = jsonString.dataUsingEncoding(NSUTF8StringEncoding)!

Deserializar datos sin procesar #

Para deserializar los datos, inicializamos un objeto ‘JSON’ y luego accedemos a una clave en particular.

do {
    let json = try JSON(data: jsonData)
    let success = try json.bool("success")
} catch {
    // do something with the error
}

Lo probamos aquí porque el acceso a json para la clave "success" podría fallar; es posible que no exista o que el valor no sea un valor booleano.

También podemos especificar una ruta para acceder a elementos anidados en la estructura JSON. La ruta es una lista separada por comas de claves e índices que describen la ruta a un valor de interés.

do {
    let json = try JSON(data: jsonData)
    let georgiaZipCodes = try json.array("states", "Georgia")
    let firstPersonName = try json.string("people", 0, "name")
} catch {
    // do something with the error
}

Deserializar modelos directamente #

JSON se puede analizar directamente en una clase de modelo que implementa el protocolo JSONDecodable.

public struct Person {
    public let name: String
    public let age: Int
    public let spouse: Bool
}

extension Person: JSONDecodable {
    public init(json: JSON) throws {
        name = try json.string("name")
        age = try json.int("age")
        spouse = try json.bool("spouse")
    }
}

do {
    let json = try JSON(data: jsonData)
    let people = try json.arrayOf("people", type: Person.self)
} catch {
    // do something with the error
}

Serialización de datos sin procesar #

Cualquier valor JSON se puede serializar directamente en NSData.

let success = JSON.Bool(false)
let data: NSData = try success.serialize()

Serializar modelos directamente #

Cualquier clase de modelo que implemente el protocolo JSONEncodable se puede serializar directamente en NSData.

extension Person: JSONEncodable {
    public func toJSON() -> JSON {
        return .Dictionary([
            "name": .String(name),
            "age": .Int(age),
            "spouse": .Bool(spouse)
        ])
    }
}

let newPerson = Person(name: "Glenn", age: 23, spouse: true)
let data: NSData = try newPerson.toJSON().serialize()

Análisis simple de JSON en objetos personalizados #

Incluso si las bibliotecas de terceros son buenas, los protocolos proporcionan una forma sencilla de analizar el JSON. Puedes imaginar que tienes un objeto ‘Todo’ como

struct Todo {
    let comment: String
}

Siempre que reciba el JSON, puede manejar el NSData simple como se muestra en el otro ejemplo usando el objeto NSJSONSerialization.

Después de eso, usando un protocolo simple JSONDecodable

typealias JSONDictionary = [String:AnyObject]
protocol JSONDecodable {
    associatedtype Element
    static func from(json json: JSONDictionary) -> Element?
}

Y hacer que su estructura ‘Todo’ se ajuste a ‘JSONDecodable’ hace el truco

extension Todo: JSONDecodable {
    static func from(json json: JSONDictionary) -> Todo? {
        guard let comment = json["comment"] as? String else { return nil }
        return Todo(comment: comment)
    }
}

Puedes probarlo con este código json:

{
    "todos": [
        {
            "comment" : "The todo comment"
        }
    ]
}

Cuando lo obtuvo de la API, puede serializarlo como se muestra en los ejemplos anteriores en una instancia AnyObject. Después de eso, puede verificar si la instancia es una instancia JSONDictionary

guard let jsonDictionary = dictionary as? JSONDictionary else { return }

La otra cosa a verificar, específica para este caso porque tiene una matriz de Todo en el JSON, es el diccionario todos

guard let todosDictionary = jsonDictionary["todos"] as? [JSONDictionary] else { return }

Ahora que tiene la matriz de diccionarios, puede convertir cada uno de ellos en un objeto Todo usando flatMap (eliminará automáticamente los valores nil de la matriz)

let todos: [Todo] = todosDictionary.flatMap { Todo.from(json: $0) }

JSON analizando Swift 3 #

Flecha #

Arrow es una biblioteca de análisis JSON elegante en Swift.

Permite analizar JSON y asignarlo a clases de modelos personalizados con la ayuda de un operador <--:

identifier <-- json["id"]
name <-- json["name"]
stats <-- json["stats"]

Ejemplo:

modelo rápido

struct Profile {
    var identifier = 0
    var name = ""
    var link: NSURL?
    var weekday: WeekDay = .Monday
    var stats = Stats()
    var phoneNumbers = [PhoneNumber]()
}

archivo JSON

{
    "id": 15678,
    "name": "John Doe",
    "link": "https://apple.com/steve",
    "weekdayInt" : 3,
    "stats": {
        "numberOfFriends": 163,
        "numberOfFans": 10987
    },
    "phoneNumbers": [{
                     "label": "house",
                     "number": "9809876545"
                     }, {
                     "label": "cell",
                     "number": "0908070656"
                     }, {
                     "label": "work",
                     "number": "0916570656"
    }]
}

Cartografía

extension Profile: ArrowParsable {
    mutating func deserialize(json: JSON) {
        identifier <-- json["id"]
        link <-- json["link"]
        name <-- json["name"]
        weekday <-- json["weekdayInt"]
        stats <- json["stats"]
        phoneNumbers <-- json["phoneNumbers"]
    }
}

Uso

let profile = Profile()
profile.deserialize(json)
