Operadores bit a bit #

Los operadores Swift Bitwise le permiten realizar operaciones en la forma binaria de los números. Puede especificar un literal binario anteponiendo el número con 0b, por lo que, por ejemplo, 0b110 es equivalente al número binario 110 (el número decimal 6). Cada 1 o 0 es un bit en el número.

Bit a bit NO ~:

var number: UInt8 = 0b01101100
let newNumber = ~number
// newNumber is equal to 0b01101100

Aquí, cada bit se cambia a su opuesto. Declarar el número como explícitamente UInt8 asegura que el número sea positivo (para que no tengamos que lidiar con negativos en el ejemplo) y que solo tenga 8 bits. Si 0b01101100 fuera un UInt más grande, habría 0 iniciales que se convertirían en 1 y se volverían significativos tras la inversión:

var number: UInt16 = 0b01101100
// number equals 0b0000000001101100
// the 0s are not significant
let newNumber = ~number
// newNumber equals 0b1111111110010011
// the 1s are now significant
  • 0 -> 1
  • 1 -> 0

Bit a bit Y &:

var number = 0b0110
let newNumber = number & 0b1010
// newNumber is equal to 0b0010

Aquí, un bit dado será 1 si y solo si los números binarios en ambos lados del operador & contenían un 1 en esa ubicación de bit.

  • 0 & 0 -> 0
  • 0 y 1 -> 0
  • 1 y 1 -> 1

Bit a bit O |:

var number = 0b0110
let newNumber = number | 0b1000
// newNumber is equal to 0b1110

Aquí, un bit dado será 1 si y solo si el número binario en al menos un lado del operador | contenía un 1 en esa ubicación de bit.

  • 0 | 0 -> 0
  • 0 | 1 -> 1
  • 1 | 1 -> 1

XOR bit a bit (OR exclusivo) ^:

var number = 0b0110
let newNumber = number ^ 0b1010
// newNumber is equal to 0b1100

Aquí, un bit dado será 1 si y solo si los bits en esa posición de los dos operandos son diferentes.

  • 0 ^ 0 -> 0
  • 0 ^ 1 -> 1
  • 1 ^ 1 -> 0

Para todas las operaciones binarias, el orden de los operandos no influye en el resultado.

Operadores personalizados #

Swift admite la creación de operadores personalizados. Los nuevos operadores se declaran a nivel global utilizando la palabra clave operator.

La estructura del operador está definida por tres partes: ubicación del operando, precedencia y asociatividad.

  1. Los modificadores prefix, infix y postfix se utilizan para iniciar una declaración de operador personalizada. Los modificadores prefix y postfix declaran si el operador debe estar antes o después, respectivamente, del valor sobre el que actúa. Dichos operadores son comunes, como 8 y 3++ **, ya que solo pueden actuar sobre un objetivo. El infijo declara un operador binario, que actúa sobre los dos valores entre los que se encuentra, como 2+3.

  2. Los operadores con mayor precedencia se calculan primero. La precedencia predeterminada del operador es más alta que ?…: (un valor de 100 en Swift 2.x). La precedencia de los operadores estándar de Swift se puede encontrar aquí.

  3. Asociatividad define el orden de las operaciones entre operadores de la misma precedencia. Los operadores asociativos por la izquierda se calculan de izquierda a derecha (orden de lectura, como la mayoría de los operadores), mientras que los operadores asociativos por la derecha se calculan de derecha a izquierda.

A partir de Swift 3.0, se definiría la precedencia y la asociatividad en un grupo de precedencia en lugar del propio operador, de modo que varios operadores puedan compartir fácilmente la misma precedencia sin referirse a los números crípticos. La lista de grupos de precedencia estándar se muestra abajo.

Los operadores devuelven valores basados ​​en el código de cálculo. Este código actúa como una función normal, con parámetros que especifican el tipo de entrada y la palabra clave return que especifica el valor calculado que devuelve el operador.

Aquí está la definición de un operador exponencial simple, ya que Swift estándar no tiene un operador exponencial.

import Foundation    

infix operator ** { associativity left precedence 170 }

func ** (num: Double, power: Double) -> Double{
    return pow(num, power)
}

El infijo dice que el operador ** funciona entre dos valores, como 9**2. Debido a que la función ha dejado la asociatividad, 3**3**2 se calcula como (3**3)**2. La precedencia de 170 es más alta que todas las operaciones estándar de Swift, lo que significa que 3+2**4 calcula 19, a pesar de la asociatividad izquierda de **.

import Foundation 

infix operator **: BitwiseShiftPrecedence

func ** (num: Double, power: Double) -> Double {
    return pow(num, power)
}

En lugar de especificar la precedencia y la asociatividad explícitamente, en Swift 3.0 podríamos usar el grupo de precedencia incorporado BitwiseShiftPrecedence que da los valores correctos (igual que <<, >>).

**: El incremento y la disminución están en desuso y se eliminarán en Swift 3.

2: https://www.wikiod.com/es/swift/operadores-avanzados#Precedencia de los operadores estándar de Swift

Operadores Conmutativos #

Agreguemos un operador personalizado para multiplicar un CGSize

func *(lhs: CGFloat, rhs: CGSize) -> CGSize{
    let height = lhs*rhs.height
    let width = lhs*rhs.width
    return CGSize(width: width, height: height)
}

Ahora esto funciona

let sizeA = CGSize(height:100, width:200)    
let sizeB = 1.1 * sizeA         //=> (height: 110, width: 220)

Pero si tratamos de hacer la operación a la inversa, obtenemos un error

let sizeC = sizeB * 20          // ERROR

Pero es lo suficientemente simple como para agregar:

func *(lhs: CGSize, rhs: CGFloat) -> CGSize{
    return rhs*lhs
}

Ahora el operador es conmutativo.

let sizeA = CGSize(height:100, width:200)    
let sizeB = sizeA * 1.1              //=> (height: 110, width: 220)

Operadores de desbordamiento #

El desbordamiento se refiere a lo que sucede cuando una operación daría como resultado un número que es mayor o menor que la cantidad designada de bits que puede contener ese número.

Debido a la forma en que funciona la aritmética binaria, después de que un número se vuelve demasiado grande para sus bits, el número se desborda hasta el número más pequeño posible (para el tamaño de bits) y luego continúa contando desde allí. De manera similar, cuando un número se vuelve demasiado pequeño, se desborda hasta el número más grande posible (para su tamaño de bits) y continúa contando hacia atrás desde allí.

Debido a que este comportamiento a menudo no se desea y puede generar problemas de seguridad graves, los operadores aritméticos de Swift +, - y * generarán errores cuando una operación provoque un desbordamiento o subdesbordamiento. Para permitir explícitamente el desbordamiento y el subdesbordamiento, use &+, &- y &* en su lugar.

var almostTooLarge = Int.max
almostTooLarge + 1 // not allowed
almostTooLarge &+ 1 // allowed, but result will be the value of Int.min

Sobrecarga + para diccionarios #

Como actualmente no existe una forma simple de combinar diccionarios en Swift, puede ser útil sobrecargar los operadores + y += para agregar esta funcionalidad usando genéricos.

// Combines two dictionaries together. If both dictionaries contain
// the same key, the value of the right hand side dictionary is used.
func +<K, V>(lhs: [K : V], rhs: [K : V]) -> [K : V] {
    var combined = lhs
    for (key, value) in rhs {
        combined[key] = value
    }
    return combined
}

// The mutable variant of the + overload, allowing a dictionary
// to be appended to 'in-place'.
func +=<K, V>(inout lhs: [K : V], rhs: [K : V]) {
    for (key, value) in rhs {
        lhs[key] = value
    }
}

A partir de Swift 3, inout debe colocarse antes del tipo de argumento.

func +=<K, V>(lhs: inout [K : V], rhs: [K : V]) { ... }

Ejemplo de uso:

let firstDict = ["hello" : "world"]
let secondDict = ["world" : "hello"]
var thirdDict = firstDict + secondDict // ["hello": "world", "world": "hello"]

thirdDict += ["hello":"bar", "baz":"qux"] // ["hello": "bar", "baz": "qux", "world": "hello"]
