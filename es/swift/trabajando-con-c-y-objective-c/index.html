Use un mapa de módulo para importar encabezados C #

Un mapa de módulo puede simplemente importar mymodule configurándolo para leer archivos de encabezado C y hacer que aparezcan como Swift funciones

Coloque un archivo llamado module.modulemap dentro de un directorio llamado mymodule:

Dentro del archivo de mapa del módulo:

// mymodule/module.modulemap
module mymodule {
    header "defs.h"
}

Luego importa el módulo:

// demo.swift
import mymodule
print("Empty color: \(Color())")

Usa el indicador -I directory para decirle a swiftc dónde encontrar el módulo:

swiftc -I . demo.swift   # "-I ." means "search for modules in the current directory"

Para obtener más información sobre la sintaxis del mapa de módulos, consulte la [documentación de Clang sobre mapas de módulos] (http://clang.llvm.org/docs/Modules.html#module-map-language).

Usar clases de Objective-C desde código Swift #

Si MyFramework contiene clases de Objective-C en sus encabezados públicos (y el encabezado general), entonces importar MyFramework es todo lo que se necesita para usarlos desde Swift.

Encabezados puente #

Un encabezado puente hace que las declaraciones adicionales de Objective-C y C sean visibles para el código Swift. Al agregar archivos de proyecto, Xcode puede ofrecer crear un encabezado de puente automáticamente:

Para crear uno manualmente, modifique la configuración de compilación Objective-C Bridging Header:

Dentro del encabezado puente, importe los archivos necesarios para usar desde el código:

// MyApp-Bridging-Header.h
#import "MyClass.h"  // allows code in this module to use MyClass

Interfaz generada #

Haga clic en el botón Elementos relacionados (o presione ⌃1), luego seleccione Interfaz generada para ver la interfaz de Swift que se generará a partir de un encabezado de Objective-C.

    

Especifique un encabezado puente para swiftc #

El indicador -import-objc-header especifica un encabezado para que swiftc lo importe:

// defs.h
struct Color {
    int red, green, blue;
};

#define MAX_VALUE 255

// demo.swift
extension Color: CustomStringConvertible {  // extension on a C struct
    public var description: String {
        return "Color(red: \(red), green: \(green), blue: \(blue))"
    }
}
print("MAX_VALUE is: \(MAX_VALUE)")  // C macro becomes a constant
let color = Color(red: 0xCA, green: 0xCA, blue: 0xD0)  // C struct initializer
print("The color is \(color)")

$ swiftc demo.swift -import-objc-header defs.h && ./demo
MAX_VALUE es: 255
El color es Color(rojo: 202, verde: 202, azul: 208)

Interoperación detallada entre Objective-C y Swift #

Cuando una API está marcada con NS_REFINED_FOR_SWIFT, tendrá dos guiones bajos como prefijo (__) cuando se importe a Swift:

@interfaz MiClase : NSObject
- (NSInteger)indexOfObject:(id)obj NS_REFINED_FOR_SWIFT;
@final

La [interfaz generada](https://www.wikiod.com/es/swift/trabajando-con-c-y-objective-c#Usando clases de Objective-C desde código Swift 201605140416386495969) se ve así:

clase pública MiClase: NSObject {
    public func __indexOfObject(obj: AnyObject) -> Int
}

Ahora puedes reemplazar la API con una extensión más “Swifty”. En este caso, podemos usar un valor de retorno opcional, filtrando [NSNotFound](https://developer.apple.com /library/mac/documentation/Cocoa/Reference/Foundation/Miscellaneous/Foundation_Constants/#//apple_ref/doc/constant_group/NSNotFound):

extensión MiClase {
    // Rather than returning NSNotFound if the object doesn't exist,
    // this "refined" API returns nil.
    func indexOfObject(obj: AnyObject) -> Int? {
        let idx = __indexOfObject(obj)
        if idx == NSNotFound { return nil }
        return idx
    }
}

// Código Swift, usando "if let" como debería ser:
let myobj = MiClase()
if let idx = myobj.indexOfObject(algo) {
    // do something with idx
}

En la mayoría de los casos, es posible que desee restringir si un argumento de una función de Objective-C puede ser “nil” o no. Esto se hace usando la palabra clave _Nonnull, que califica cualquier puntero o referencia de bloque:

vacío
doStuff(const void *const _Nonnull data, void (^_Nonnull finalización)())
{
    // complex asynchronous code
}

Con eso escrito, el compilador emitirá un error cada vez que intentemos pasar nil a esa función desde nuestro código Swift:

hacercosas(
    nil,  // error: nil is not compatible with expected argument type 'UnsafeRawPointer'
    nil)  // error: nil is not compatible with expected argument type '() -> Void'

Lo contrario de _Nonnull es _Nullable, lo que significa que es aceptable pasar nil en este argumento. _Nullable también es el predeterminado; sin embargo, especificarlo explícitamente permite más código autodocumentado y preparado para el futuro.

Para ayudar aún más al compilador a optimizar su código, también puede especificar si el bloque se está escapando:

vacío
callNow(__attribute__((noescape)) void (^_Nonnull f)())
{
    // f is not stored anywhere
}

Con este atributo prometemos no guardar la referencia al bloque y no llamar al bloque después de que la función haya terminado de ejecutarse.

Usar la biblioteca estándar de C #

La interoperabilidad de C de Swift le permite usar funciones y tipos de la biblioteca estándar de C.

En Linux, la biblioteca estándar de C se expone a través del módulo Glibc; en las plataformas de Apple se llama Darwin.

#if os(macOS) || os(iOS) || os(tvOS) || os(watchOS)
import Darwin
#elseif os(Linux)
import Glibc
#endif

// use open(), read(), and other libc features

Uso de clases Swift del código Objective-C #

En el mismo módulo #

Dentro de un módulo llamado “MyModule”, Xcode genera un encabezado llamado MyModule-Swift.h que expone las clases Swift públicas a Objective-C. Importe este encabezado para usar las clases de Swift:

// MySwiftClass.swift en MyApp
Fundación de importación

// La clase debe ser `pública` para ser visible, a menos que este objetivo también tenga un encabezado puente
Clase pública MySwiftClass: NSObject {
    // ...
}
// MyViewController.m en MyApp

#importar "MiControladorDeVista.h"
#import "MyApp-Swift.h" // importa la interfaz generada
#import <MyFramework/MyFramework-Swift.h> // o use paréntesis angulares para un objetivo de marco

@implementación MyViewController
- (vacío) demostración {
    [[MySwiftClass alloc] init];  // use the Swift class
}
@final

Configuraciones de compilación relevantes:

  • Nombre de encabezado de interfaz generado por Objective-C: controla el nombre del encabezado Obj-C generado.
  • Instalar encabezado de compatibilidad de Objective-C: si el encabezado -Swift.h debe ser un encabezado público (para objetivos de marco).

En otro módulo #

El uso de @import MyFramework; importa todo el módulo, incluidas las interfaces Obj-C a las clases de Swift (si la configuración de compilación mencionada anteriormente está habilitada).
