Azure AD B2C #

Azure AD B2C es una solución de administración de identidades en la nube para sus aplicaciones web y móviles. Es un servicio global de alta disponibilidad que escala a cientos de millones de identidades.

Aplicación web - Hello.js #

La implementación de la aplicación web usa Hello.js que realiza la administración de identidades con Azure AD B2C. Hello.js es un SDK de JavaScript del lado del cliente para autenticarse con los servicios web OAuth2 y consultar las API REST.

Aplicación móvil: complemento ADAL #

La implementación de la aplicación móvil utiliza [parche del complemento ADAL Cordova para B2C] (https://github.com/jospete/azure-activedirectory-library-for-cordova). Esta es una versión recortada del complemento Biblioteca de autenticación de Active Directory (ADAL) para aplicaciones Apache Cordova, cordova-plugin-ms-adal que funciona con Azure AD B2C. El complemento cordova-plugin-ms-adal original proporciona una funcionalidad de autenticación fácil de usar para sus aplicaciones Apache Cordova aprovechando Active Directory.

Decodificar JWT #

jwtHelper de angular-jwt se encargará de ayudarlo a decodificar el token (JWT) y verificar su fecha de vencimiento. Los tokens web JSON son un método abierto estándar de la industria RFC 7519 para representar reclamos de forma segura entre dos partes.

1. Configuración del proyecto: #

  1. Clonar o descargar este repositorio

    git clone https://github.com/NewtonJoshua/Azure-ADB2C-Angularjs-sample.git

  2. Instalar dependencias

    npm install

    bower install

2. Configuración de anuncio: #

Configuración ADAL-B2C

  1. [Cree un directorio Azure AD B2C] (https://docs.microsoft.com/en-us/azure/active-directory-b2c/active-directory-b2c-get-started)

    Note the Domain name, it’ll be used as the tenantName.

  2. Registre su aplicación Follow the instructions to create an application and enable both Web App and Native client. Refer Register a web application and Register a mobile/native application

    Enter the Reply URL as http://localhost:8100 or any port from wher you’ll be serving your app.

    In Application Claims, select Email Addresses too.

    Note the Application ID . It’ll be used as the clientId.

  3. [Cree una política de registro o inicio de sesión](https://docs.microsoft.com/en-us/azure/active-directory-b2c/active-directory-b2c-reference-policies#create-a -registro-o-registro-en-política)

    Note the name of the policy. It’ll be used as policy.

  4. [Cree una política de restablecimiento de contraseña](https://docs.microsoft.com/en-us/azure/active-directory-b2c/active-directory-b2c-reference-policies#create-a-password-reset- política)

    Note the name of the policy. It’ll be used as the password-reset-policy

3. Configuración de anuncios: #

En settings.value.js, ingrese los siguientes valores

  • TenantName: nombre de dominio del paso 2.1
  • clientId: ID de la aplicación del paso 2.2
  • política: nombre de la política del paso 2.3

5. Personalice la interfaz de usuario de Azure AD B2C #

La pantalla de inicio de sesión de Azure AD B2C se puede personalizar para adaptarse a nuestra marca. Consulte Personalización de la interfaz de usuario

En este ejemplo, tenemos dos pantallas de interfaz de usuario personalizadas,

  • Página de inicio de sesión y registro de AD B2C: adCustomPages/unified.html
  • Página de restablecimiento de contraseña de AD B2C: adCustomPages/resetpassword.html

En adCustomPages/unified.html, en las líneas 442 y 445, ingrese su nombre de inquilino, política de restablecimiento de contraseña y ID de cliente.

Las páginas deben cargarse en un blob y su URL debe estar referenciada en las políticas de Azure AD B2C.

  • Cree una cuenta de almacenamiento como se menciona en [Cargar el contenido de muestra en Azure Blob Storage](https://docs.microsoft.com/en-us/azure/active-directory-b2c/active-directory-b2c-reference-ui -customization-helper-tool#upload-the-sample-content-to-azure-blob-storage)
  • Cargue las páginas de anuncios de muestra en el contenedor y anote su URL.
  • Para la cuenta de Blob service Storage creada, cree una regla CORS con ‘*’ como ORÍGENES PERMITIDOS. Seleccione todo en MÉTODOS PERMITIDOS. Ingrese * para ENcabezados PERMITIDOS y Encabezados EXPUESTOS también.
  • Personalice su política

Ahora en su aplicación puede ver la interfaz de usuario personalizada.
