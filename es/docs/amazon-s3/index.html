tutorial amazon-s3

Resumen de la documentación #

De http://docs.aws.amazon.com/AmazonS3/latest/dev/Welcome.html

Amazon Simple Storage Service es almacenamiento para Internet. Está diseñado para hacer que la computación a escala web sea más fácil para los desarrolladores.

Amazon S3 tiene una interfaz de servicios web simple que puede usar para almacenar y recuperar cualquier cantidad de datos, en cualquier momento, desde cualquier lugar la red. Brinda a cualquier desarrollador acceso a la misma tecnología altamente escalable, infraestructura de almacenamiento de datos confiable, rápida y económica que Amazon utiliza para ejecutar su propia red global de sitios web. El servicio tiene como objetivo maximizar los beneficios de escala y pasar esos beneficios a desarrolladores.

Soporte de lenguaje y secuencias de comandos #

S3 no es un lenguaje de desarrollo como tal, sino una plataforma en la nube que admite solicitudes de servicios web. Hay una variedad de herramientas y SDK herramientas y SDK que publica Amazon. Los SDK de lenguaje brindan acceso transparente a S3 mediante el manejo de tareas como solicitudes de servicios web, autenticación, administración de sesiones, renovación de tokens, etc. También hay interfaces de línea de comandos para bash/windows/ios y [powershell][ clip de powershell].

La API de S3 #

La API de servicios web de S3 ha sido apoyada por proveedores de la competencia. Actualmente, este tema no cubre la API directamente, por lo que los ejemplos aquí no serían útiles para crear aplicaciones que se conectan a sistemas de la competencia que aprovechan la API de S3.

Versiones #

A partir del 28 de marzo de 2017, la CLI de AWS tiene 174 versiones, que están claramente documentadas en las Notas de la versión de la CLI. Amazon S3 tiene 66 versiones, de las cuales algunas anuncian la adición de una nueva región y otras agregan funcionalidad. Estos están documentados en las [Notas de la versión de S3] versiones de aws s3.

Los ejemplos #

Con respecto a los ejemplos mostrados hasta ahora en esta sección “Introducción”, Amazon S3 es útil para los desarrolladores en los siguientes casos de uso:

  • Almacene o haga una copia de seguridad de los archivos en un sistema duradero y de alto rendimiento, lo que descarga esta tarea de las arquitecturas que no están en la nube: sistemas de archivos Linux y Windows. Es costoso recrear los niveles de durabilidad y rendimiento de S3 utilizando servidores locales o instancias EC2.
  • Cuando el ancho de banda de la red es un problema, por ejemplo, en los casos en que varios usuarios simultáneos deben descargar archivos de gran tamaño, la transferencia de datos a S3 se puede usar como una forma de que una aplicación *mitigue la escasez de ancho de banda en un centro de datos o en un servidor local *. Esta es una forma de distribuir un gran repositorio de código, imágenes de máquinas virtuales, video o instaladores de software. Se pueden mejorar los tiempos de carga y descarga de los usuarios. [Para un rendimiento adicional en escenarios de base de usuarios muy grandes, se puede usar un sistema de entrega de contenido como cloudfront para almacenar archivos en caché más cerca de los usuarios].
  • Su aplicación necesita crear o consumir un archivo grande y necesita una forma de permitir que los usuarios accedan a él o lo depositen.
  • La distribución de su aplicación es muy grande y necesita compartirla con los usuarios.
  • Está armando una canalización de entrega continua y, por ejemplo, está alojando partes de su sitio web en Amazon S3.

En este punto, los ejemplos no muestran cómo hacer lo siguiente:

  • Los ejemplos, aunque son más rápidos y claros que escribir ayuda de aws s3, no mencionan algunos de los comandos cubiertos en la ayuda, como sitio web de aws s3`.
  • Cómo compartir o restringir el acceso de los usuarios. Sin restricción explícita, los ejemplos funcionarían solo para usuarios que comparten la misma cuenta de AWS.
  • Cómo proteger los datos mediante el cifrado. Tenga en cuenta que AWS posiciona a S3 con un nivel de seguridad más alto que los datos almacenados en EC2. Prácticas recomendadas de seguridad de AWS, agosto de 2016, pág. 27

Seguridad #

AWS recomienda ver S3 como una plataforma segura:

A menos que tenga requisitos comerciales o de cumplimiento más estrictos, no necesita introducir capas adicionales de protección más allá de las proporcionadas por la infraestructura global segura de AWS. [ibídem. p.2][buenas prácticas]

En su [guía de seguridad][prácticas recomendadas], AWS recomienda utilizar la autenticación de AWS como adecuada para S3. ibídem. pags. 27

Además, S3 proporciona cifrado del lado del servidor o cifrado del lado del cliente. El cifrado del lado del cliente se proporciona de forma transparente mediante AWS Java SDK; las claves no necesitan almacenarse en AWS. ibídem. pags. 28
