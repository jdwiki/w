¿Qué es wxPython? #

En pocas palabras, wxPython es un conjunto de enlaces a la biblioteca wxWidgets C++ Cross Platform GUI.

Bien, ¿qué es wxWidgets? #

La biblioteca wxWidgets proporciona un conjunto de abstracciones gratuito, gratuito y de código abierto para los diversos elementos de la GUI, de modo que los controles nativos se sigan utilizando, donde estén disponibles, manteniendo la apariencia, la sensación y la velocidad nativas. Como tal, proporciona una abstracción para la creación de GUI y una serie de otras utilidades en una plataforma que permite a los desarrolladores crear aplicaciones para Windows, Mac OS X, Linux y otras plataformas utilizando una única base de código. wxWidgets se inició en 1992 y puede ver un historial detallado aquí. La biblioteca wxWidgets se distribuye bajo la licencia de wxWindows, que se basa en L-GPL pero con una cláusula de excepción. La cláusula de excepción le permite vincular su aplicación de forma dinámica o estática a wxWidgets sin el requisito de distribuir el código fuente para su propia aplicación. En otras palabras, puede usar wxWidgets para proyectos gratis o comerciales, sin costo. La licencia lo alienta a devolver las mejoras que realice en la propia biblioteca de wxWidgets.

Lo más destacado, tenga en cuenta que wxWidgets comprende cientos de clases para el desarrollo de aplicaciones multiplataforma:

  • Diseño de ventana usando medidores
  • Contextos del dispositivo (junto con bolígrafos, pinceles y fuentes)
  • Sistema completo de gestión de eventos
  • Visor de ayuda HTML
  • Reproducción de sonido y video
  • Soporte Unicode e Internacionalización
  • Documento/Ver Arquitectura
  • Arquitectura de impresión
  • Enchufes
  • Multiproceso
  • Manipulación de archivos y directorios
  • Ayuda en línea y contextual
  • Representación HTML
  • Contenedores Básicos
  • Carga, guardado, dibujo y manipulación de imágenes.
  • Biblioteca de fecha y hora y temporizadores
  • Manejo de errores
  • Portapapeles y arrastrar y soltar

Tenga en cuenta que algunas de estas instalaciones, p. threading, en realidad no están relacionados con la GUI, pero proporcionan una abstracción multiplataforma útil de modo que, en el caso de threading, por ejemplo, un conjunto de código de aplicación funcionará en cualquier plataforma compatible.

Durante muchos años, la biblioteca wxWidgets produjo 4 compilaciones separadas, además de compilaciones de depuración a partir de un conjunto de código fuente, bibliotecas estáticas y dinámicas creadas para ASCII y Unicode. Por lo general, está disponible preconstruido en las variantes más comunes y como código fuente para compilar con las diversas opciones para el entorno de destino y con la cadena de herramientas C++ de los desarrolladores con numerosas cadenas de herramientas compatibles.

Los enlaces de python para esta biblioteca y algunas adiciones forman wxPython.

Volver a ¿Qué es wxPython, (qué me da)? #

wxPython le brinda a un desarrollador una forma de beneficiarse de una biblioteca GUI multiplataforma, con una licencia clara, al mismo tiempo que brinda los beneficios de Python. Al igual que wxWidgets y Python, wxPython es gratuito, gratuito y de código abierto, y está disponible para su uso y distribución en proyectos gratuitos y comerciales sin el requisito resultante de distribuir su código fuente.

  • Suite GUI completa que incluye (pero no se limita a):
  • Ventanas (incluidas las ventanas MDI)
  • Magos
  • Marcos y minimarcos
  • Diálogos, Estándar, Avanzado y Personalizado
  • Controles de visualización de libros, árboles, cuadrículas y datos
  • Indicadores, controles deslizantes, botones giratorios, animaciones, portapapeles, arrastrar y soltar
  • Compatibilidad con HTML, PDF y visor de imágenes
  • Los componentes de la GUI se pueden posicionar de forma absoluta, pero se recomienda encarecidamente utilizar un diseño basado en dimensionadores que admita el dimensionamiento automático, etc.
  • Multiplataforma - Admite GUI para Windows, OS-X y Linux con una sola base de código sin declaraciones condicionales en su código
  • Velocidad nativa, apariencia y sensación.
  • Prototipo rápido, prueba y depuración - recuerda que esto es python
  • Ejecute y edite muestras de casi todo en el paquete de demostración.
  • Licencia clara para uso gratuito incluso en productos comerciales.
  • Si es necesario, su GUI de python se puede refactorizar a una GUI de wxWidgets de C++ más tarde, ya que ya la está usando.
  • Comunidad grande, activa y útil de usuarios y desarrolladores tanto en [StackOverflow][14] como en [listas de correo][15].

Tenga en cuenta que cuando python en sí proporciona un mecanismo multiplataforma para implementar las funciones de utilidad de wxWidgets, la subprocesamiento nuevamente es un buen ejemplo, intencionalmente se omite de wxPython.

wxPython también tiene un gran conjunto de demostraciones que se pueden ejecutar, probar y editar desde el paquete Documentos y demostraciones.

Sabores de wxPython #

ASCII frente a Unicode: #

Durante muchos años, al igual que con wxWidgets, los desarrolladores tenían que elegir entre compilaciones ASCII y Unicode, además de necesitar una compilación para su versión específica de python, así como las opciones de 32/64 bits. A partir de wxPython 2.8.9, la compilación solo ASCII de wxPython se eliminó, por lo que la compatibilidad con Unicode siempre está disponible.

Clásico contra Fénix: #

Desde wxPython 3.0.0 ha existido la compilación “Clásica” lanzada de wxPython y una compilación Phoenix actualmente inédita. La compilación clásica tiende a quedar rezagada con respecto a las compilaciones de wxWidgets de los mismos números y el paquete de documentación es el C ++; está disponible para descargar para varias plataformas (consulte [Instalación de Classic] [3]), en el caso de Windows como un instalador ejecutable. Los enlaces de Phoenix, que se generan en gran parte automáticamente, deberían seguir más de cerca las compilaciones de wxWidgets y también incluir documentación específica de wxPython: se puede compilar desde la fuente o las compilaciones nocturnas como ruedas se pueden obtener usando pip, (ver [Instalación de Phoenix][4]).

En wxPython pero no en wxWidgets #

wxPython amplía la biblioteca de wxWidgets con una serie de características, las siguientes son solo algunas, que no están disponibles en wxWidgets:

  • Programadores Editores y Shells: [crust][5], [crustslices][6], [AlaCart & AlaMode][7], [AlaModeTest][8]
  • [Intérprete][9] & [magia][10]
  • Inspección: esto le permite iniciar una ventana para explorar todos los componentes de la GUI de sus aplicaciones.
  • Un extenso conjunto de demostraciones.
