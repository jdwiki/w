Una ruta de ubicación de XPath es una serie de pasos de ubicación separados por un carácter /:

step1/step2/step3

Un paso de ubicación contiene un eje, una prueba de nodo y una lista opcional de predicados. El eje y la prueba de nodo están separados por dos puntos ::. Los predicados están encerrados entre corchetes:

axis::nodeTest[predicate1][predicate2]

La evaluación de una ruta de ubicación comienza con un conjunto de nodos que contiene el nodo de contexto dado por el contexto de la expresión, o el nodo raíz, si la ruta de ubicación comienza con /. En cada paso, cada nodo N en el conjunto de nodos original se reemplaza con el conjunto de nodos que

  • se puede llegar desde N siguiendo el eje dado,
  • coincide con la prueba de nodo, y
  • coincide con todos los predicados.

El resultado de una expresión de ruta de ubicación es el conjunto de nodos final obtenido después de procesar todos los pasos de ubicación.

Recorriendo elementos secundarios #

Atravesando desde el nodo raíz a un elemento descendiente usando el eje hijo:

/child::html/child::body/child::div/child::span

Dado que el eje “hijo” es el eje predeterminado, se puede abreviar como:

/html/body/div/span

Recorriendo todos los descendientes #

Los ejes descendant y descendant-or-self se pueden usar para encontrar todos los elementos descendientes de un nodo a cualquier profundidad. Por el contrario, el eje “hijo” solo atraviesa a los niños inmediatos.

/child::html/descendant::span
/child::html/descendant-or-self::*

La barra inclinada doble // es un atajo para /descendant-or-self::node()/. Entonces las siguientes expresiones son equivalentes:

table//td
child::table/descendant-or-self::node()/child::td
child::table/descendant::td
table/descendant::td

Atravesando antepasados #

El eje padre contiene solo el padre de un nodo. La siguiente expresión selecciona el elemento html tomando un desvío sobre el elemento body:

/child::html/child::body/parent::html

.. es un atajo para parent::node()

Los ejes ancestor y ancestor-or-self atraviesan todos los ancestros de un nodo. La siguiente expresión devuelve todos los elementos div que son ancestros del nodo de contexto:

ancestor::div

El eje del “yo” #

El eje self solo contiene el nodo de contexto en sí. La expresión . es un atajo para self::node() y siempre coincide con el nodo de contexto. El atajo . es útil para enumerar los descendientes del nodo de contexto. Las siguientes expresiones son equivalentes:

.//span
self::node()/descendant-or-self::node()/child::span
descendant::span

El eje self puede ser útil en los predicados de XPath 1.0. Por ejemplo, para seleccionar todos los elementos secundarios h1, h2 y h3 del nodo de contexto:

*[self::h1 or self::h2 or self::h3]

Atravesar los nodos siguientes y anteriores #

Los ejes siguiente-hermano y precedente-hermano contienen los hermanos antes o después del nodo de contexto, y los ejes siguiente y precedente contienen todos los nodos del documento antes o después del nodo de contexto, pero:

  • Ninguno de estos ejes contiene nodos de atributos o espacios de nombres.
  • El eje siguiente no contiene ningún descendiente.
  • El eje precedente no contiene ningún ancestro.

Ejemplos:

following::span[1]
following-sibling::*[last()]

Atravesar nodos de atributos y espacios de nombres #

Los ejes atributo y espacio de nombres contienen todos los nodos de atributos y espacios de nombres de un elemento. El atajo @ significa atributo::, por lo que los siguientes son equivalentes:

child::div/attribute::class
div/@class
