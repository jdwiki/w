Mejorar la seguridad de tipos usando nuevos tipos #

Los tipos de alias reducen la repetición y mejoran la legibilidad, pero no son más seguros para los tipos que el propio tipo con alias. Considera lo siguiente:

type alias Email = String

type alias Name = String

someEmail = "holmes@private.com"

someName = "Benedict"

sendEmail : Email -> Cmd msg
sendEmail email = ...

Usando el código anterior, podemos escribir sendEmail someName, y se compilará, aunque en realidad no debería, porque a pesar de que los nombres y los correos electrónicos son Strings, son cosas completamente diferentes.

Realmente podemos distinguir una String de otra String en el nivel de tipo creando un nuevo tipo. Aquí hay un ejemplo que reescribe Correo electrónico como un tipo en lugar de un tipo de alias:

module Email exposing (Email, create, send)

type Email = EmailAddress String

isValid : String -> Bool
isValid email = 
  -- ...validation logic

create : String -> Maybe Email
create email =
    if isValid email then
        Just (EmailAddress email)
    else
        Nothing

send : Email -> Cmd msg
send (EmailAddress email) = ...

Nuestra función isValid hace algo para determinar si una cadena es una dirección de correo electrónico válida. La función crear verifica si una Cadena dada es un correo electrónico válido, devolviendo un Correo electrónico envuelto en Quizás para garantizar que solo devolvemos direcciones validadas. Si bien podemos eludir la verificación de validación construyendo un Correo electrónico directamente escribiendo EmailAddress "somestring", si nuestra declaración de módulo no expone el constructor EmailAddress, como se muestra aquí

module Email exposing (Email, create, send)

entonces ningún otro módulo tendrá acceso al constructor EmailAddress, aunque aún pueden usar el tipo Email en las anotaciones. La única forma de crear un nuevo Correo electrónico fuera de este módulo es usando la función crear que proporciona, y esa función garantiza que solo devolverá direcciones de correo electrónico válidas en primer lugar. Por lo tanto, esta API guía automáticamente al usuario por la ruta correcta a través de su tipo de seguridad: send solo funciona con valores construidos por create, que realiza una validación y exige el manejo de correos electrónicos no válidos ya que devuelve un Maybe Email.

Si desea exportar el constructor Email, puede escribir

module Email exposing (Email(EmailAddress), create, send)

Ahora cualquier archivo que importe Correo electrónico también puede importar su constructor. En este caso, hacerlo permitiría a los usuarios eludir la validación y “enviar” correos electrónicos no válidos, pero no siempre está creando una API como esta, por lo que exportar constructores puede ser útil. Con un tipo que tiene varios constructores, es posible que también desee exportar solo algunos de ellos.

Tipos de datos comparables #

Los tipos comparables son tipos primitivos que se pueden comparar usando operadores de comparación del módulo Básico, como: (<), (>), (<=), (>=), max, min, comparar

Los tipos comparables en Elm son Int, Float, Time, Char, String y tuplas o listas de tipos comparables.

En la documentación o las definiciones de tipo, se las denomina variables de tipo especial comparables, p. ver definición de tipo para la función Basics.max:

max : comparable -> comparable -> comparable

Tipos de construcción #

La combinación de palabras clave tipo alias da un nuevo nombre para un tipo, pero la palabra clave tipo de forma aislada declara un nuevo tipo. Examinemos uno de los más fundamentales de estos tipos: Maybe

type Maybe a
    = Just a
    | Nothing

Lo primero que se debe tener en cuenta es que el tipo Maybe se declara con una [variable de tipo](https://www.wikiod.com/es/elm/tipos-variables-de-tipo-y-constructores-de-tipo /8809/tipo-variables) de a. La segunda cosa a tener en cuenta es el carácter vertical, |, que significa “o”. En otras palabras, algo del tipo Quizás un es o Solo un o Nada.

Cuando escribes el código anterior, Just y Nothing entran en el ámbito como valor-constructores, y Quizás entra en el ámbito como tipo-constructor. Estas son sus firmas:

Just : a -> Maybe a

Nothing : Maybe a

Maybe : a -> Maybe a -- this can only be used in type signatures

Debido a la variable de tipo a, cualquier tipo puede “envolverse dentro” del tipo Quizás. Entonces, Maybe Int, Maybe (List String), o Maybe (Maybe (List Html)), son todos tipos válidos. Al desestructurar cualquier valor de tipo con una expresión de caso, debe tener en cuenta cada posible instanciación de ese tipo. En el caso de un valor de tipo Quizás un, debe tener en cuenta tanto el caso Solo un como el caso Nada:

thing : Maybe Int
thing = 
    Just 3

blah : Int
blah =
    case thing of
        Just n -> 
            n

        Nothing ->
            42

-- blah = 3

Intente escribir el código anterior sin la cláusula Nothing en la expresión case: no se compilará. Esto es lo que hace que el constructor de tipos Maybe sea un gran patrón para expresar valores que pueden no existir, ya que te obliga a manejar la lógica de cuando el valor es Nothing.

El tipo Nunca #

El tipo Nunca no se puede construir (el módulo Basics no ha exportado su constructor de valores y tampoco le ha proporcionado ninguna otra función que devuelva Nunca). No hay ningún valor nunca: Nunca o una función createNever: ?? -> Nunca.

Esto tiene sus beneficios: puede codificar en un sistema de tipos una posibilidad que no puede suceder. Esto se puede ver en tipos como Task Never Int que garantiza que tendrá éxito con un Int; o Program Never que no tomará ningún parámetro al inicializar el código Elm desde JavaScript.

Tipo de firmas #

En Elm, los valores se declaran escribiendo un nombre, un signo igual y luego el valor real:

someValue = 42

Las funciones también son valores, con la adición de tomar un valor o valores como argumentos. Suelen escribirse de la siguiente manera:

double n = n * 2

Cada valor en Elm tiene un tipo. Los tipos de los valores anteriores serán inferidos por el compilador dependiendo de cómo se usen. Pero es una buena práctica declarar siempre explícitamente el tipo de cualquier valor de nivel superior, y para hacerlo, escriba una * firma de tipo * de la siguiente manera:

someValue : Int
someValue = 
    42

someOtherValue : Float
someOtherValue =
    42

Como podemos ver, 42 se puede definir como ya sea un Int o un Float. Esto tiene sentido intuitivo, pero consulte Variables de tipo para obtener más información.

Las firmas de tipo son particularmente valiosas cuando se usan con funciones. Aquí está la función de duplicación de antes:

double : Int -> Int
double n =
    n * 2

Esta vez, la firma tiene un ->, una flecha, y pronunciaríamos la firma como “int to int”, o “toma un número entero y devuelve un número entero”. -> indica que al dar a doble un valor Int como argumento, doble devolverá un Int. Por lo tanto, toma un número entero a un número entero:

> double
<function> : Int -> Int

> double 3
6 : Int

Tipos basicos #

En elm-repl, escriba un fragmento de código para obtener su valor y tipo inferido. Pruebe lo siguiente para conocer los distintos tipos que existen:

> 42
42 : number

> 1.987
1.987 : Float

> 42 / 2
21 : Float

> 42 % 2
0 : Int

> 'e'
'e' : Char

> "e"
"e" : String

> "Hello Friend"
"Hello Friend" : String

> ['w', 'o', 'a', 'h']
['w', 'o', 'a', 'h'] : List Char

> ("hey", 42.42, ['n', 'o'])
("hey", 42.42, ['n', 'o']) : ( String, Float, List Char )

> (1, 2.1, 3, 4.3, 'c')
(1,2.1,3,4.3,'c') : ( number, Float, number', Float, Char )

> {}
{} : {}

> { hey = "Hi", someNumber = 43 }
{ hey = "Hi", someNumber = 43 } : { hey : String, someNumber : number }

> ()
() : ()

{} es el tipo de registro vacío y () es el tipo de tupla vacía. Este último se utiliza a menudo con fines de evaluación perezosa. Consulte el ejemplo correspondiente en https://www.wikiod.com/es/elm/funciones-y-aplicacion-parcial

Observe cómo number aparece sin mayúsculas. Esto indica que es una Variable de tipo y, además, la palabra particular número se refiere a una Variable de tipo especial que puede ser Int o Float (consulte las secciones correspondientes para más). Sin embargo, los tipos siempre están en mayúsculas, como Char, Float, List String, etcétera.

Variables de tipo especial #

Elm define las siguientes variables de tipo especial que tienen un significado particular para el compilador:

  • comparable: Compuesto por Int, Float, Char, String y tuplas de los mismos. Esto permite el uso de los operadores < y >.

Ejemplo: Podrías definir una función para encontrar los elementos más pequeños y más grandes en una lista (extensión). Tú piensas qué tipo de firma escribir. Por un lado, podrías escribir extentInt : List Int -> Maybe (Int, Int) y extentChar : List Char -> Maybe (Char, Char) y otro para Float y String. La implementación de estos sería la misma:

  extentInt list =
    let
      helper x (minimum, maximum) = 
        ((min minimum x), (max maximum x))
    in 
      case list of 
        [] ->
          Nothing
        x :: xs ->
          Just <| List.foldr helper (x, x) xs

 You might be tempted to simply write `extent : List a -> Maybe (a, a)`, but the compiler will not let you do this, because the functions `min` and `max` are not defined for these types (NB: these are just simple wrappers around the `<` operator mentioned above). You can solve this by defining `extent : List comparable -> Maybe (comparable, comparable)`. This allows your solution to be *polymorphic*, which just means that it will work for more than one type.
  • number: Compuesto por Int y Float. Permite el uso de operadores aritméticos excepto la división. Luego puede definir, por ejemplo, suma: número de lista -> número y hacer que funcione tanto para enteros como para flotantes.
  • appendable: Compuesto por String, List. Permite el uso del operador ++.
  • compappend: Esto aparece a veces, pero es un detalle de implementación del compilador. Actualmente, esto no se puede usar en sus propios programas, pero a veces se menciona.

Tenga en cuenta que en una anotación de tipo como esta: número -> número -> número todos se refieren al mismo tipo, por lo que pasar Int -> Float -> Int sería un error de tipo. Puede resolver esto agregando un sufijo al nombre de la variable de tipo: number -> number' -> number'' se compilaría bien.

No hay un nombre oficial para estos, a veces se les llama:

  • Variables de tipo especial
  • Variables de tipo tipo clase de tipo
  • Pseudo-clases de tipo

Esto se debe a que funcionan como [Type Classes] de Haskell (https://www.wikiod.com/es/haskell/clases-de-tipo), pero sin que el usuario pueda definirlas.

Variables de tipo #

Las variables de tipo son nombres sin mayúsculas en firmas de tipo. A diferencia de sus contrapartes en mayúsculas, como Int y String, no representan un solo tipo, sino cualquier tipo. Se utilizan para escribir funciones genéricas que pueden operar en cualquier tipo o tipos, y son particularmente útiles para escribir operaciones sobre contenedores como List o Dict. La función List.reverse, por ejemplo, tiene la siguiente firma:

reverse : List a -> List a

Lo que significa que puede funcionar en una lista de cualquier tipo de valor, por lo que List Int, List (List String), ambos y cualquier otro se pueden revertir de todos modos. Por lo tanto, a es una variable de tipo que puede sustituir a cualquier tipo.

La función reverse podría haber usado cualquier nombre de variable sin mayúsculas en su firma de tipo, excepto por el puñado de nombres de variable de tipo especial, como number (consulte el ejemplo correspondiente para obtener más información):

reverse : List lol -> List lol

reverse : List wakaFlaka -> List wakaFlaka

Los nombres de las variables de tipo se vuelven significativos solo cuando hay diferentes variables de tipo dentro de una sola firma, ejemplificado por la función mapa en las listas:

map : (a -> b) -> List a -> List b

map toma alguna función de cualquier tipo a a cualquier tipo b, junto con una lista con elementos de algún tipo a, y devuelve una lista de elementos de algún tipo b, que obtiene al aplicar la función dada a cada elemento de la lista.

Hagamos la firma concreta para ver mejor esto:

plusOne : Int -> Int
plusOne x = 
    x + 1

> List.map plusOne
<function> : List Int -> List Int

Como podemos ver, tanto a = Int como b = Int en este caso. Pero, si mapa tuviera una firma de tipo como mapa: (a -> a) -> Lista a -> Lista a, entonces solo funcionaría en funciones que operan en un solo tipo, y usted nunca podrá cambiar el tipo de una lista usando la función mapa. Pero dado que la firma de tipo de mapa tiene varias variables de tipo diferentes, a y b, podemos usar mapa para cambiar el tipo de una lista:

isOdd : Int -> Bool
isOdd x =
    x % 2 /= 0

> List.map isOdd
<function> : List Int -> List Bool

En este caso, a = Int y b = Bool. Por lo tanto, para poder usar funciones que pueden tomar y devolver tipos diferentes, debe usar variables de tipo diferente.

Tipo de alias #

A veces queremos darle a un tipo un nombre más descriptivo. Digamos que nuestra aplicación tiene un tipo de datos que representa a los usuarios:

{ name : String, age : Int, email : String }

Y nuestras funciones en los usuarios tienen firmas de tipo en la línea de:

prettyPrintUser : { name : String, age : Int, email : String } -> String

Esto podría volverse bastante difícil de manejar con un tipo de registro más grande para un usuario, así que usemos un tipo de alias para reducir el tamaño y darle un nombre más significativo a esa estructura de datos:

type alias User =
    { name: String
    , age : Int
    , email : String
    }


prettyPrintUser : User -> String

Los alias de tipo hacen que sea mucho más sencillo definir y usar un modelo para una aplicación:

type alias Model =
    { count : Int
    , lastEditMade : Time
    }

Usar type alias literalmente solo crea un alias de un tipo con el nombre que le das. Usar el tipo Model anterior es exactamente lo mismo que usar { count : Int, lastEditMade : Time }. Aquí hay un ejemplo que muestra cómo los alias no son diferentes de los tipos subyacentes:

type alias Bugatti = Int

type alias Fugazi = Int

unstoppableForceImmovableObject : Bugatti -> Fugazi -> Int
unstoppableForceImmovableObject bug fug =
    bug + fug

> unstoppableForceImmovableObject 09 87
96 : Int

Un alias de tipo para un tipo de registro define una función constructora con un argumento para cada campo en orden de declaración.

type alias Point = { x : Int, y : Int }

Point 3 7
{ x = 3, y = 7 } : Point

type alias Person = { last : String, middle : String, first : String }

Person "McNameface" "M" "Namey"
{ last = "McNameface", middle = "M", first = "Namey" } : Person

Cada alias de tipo de registro tiene su propio orden de campo incluso para un tipo compatible.

type alias Person = { last : String, middle : String, first : String }
type alias Person2 = { first : String, last : String, middle : String }

Person2 "Theodore" "Roosevelt" "-"
{ first = "Theodore", last = "Roosevelt", middle = "-" } : Person2

a = [ Person "Last" "Middle" "First", Person2 "First" "Last" "Middle" ]
[{ last = "Last", middle = "Middle", first = "First" },{ first = "First", last = "Last", middle = "Middle" }] : List Person2
