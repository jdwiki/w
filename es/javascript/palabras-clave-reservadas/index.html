Ciertas palabras, las llamadas palabras clave, se tratan de forma especial en JavaScript. Hay una plétora de diferentes tipos de palabras clave, y han cambiado en diferentes versiones del idioma.

Palabras clave reservadas #

JavaScript tiene una colección predefinida de palabras clave reservadas que no puede usar como variables, etiquetas o nombres de funciones. #

romper exportación súper
caso extiende cambiar
atrapar falso esto
clase finalmente tirar
const para verdadero
continuar función intentar
depurador si tipo de
predeterminado importar var
eliminar en vacío
hacer nuevo mientras
más nulo con
enumeración volver

ECMAScript 5 / 5.1 #

No hubo cambios desde ECMAScript 3.

ECMAScript 5 eliminó int, byte, char, goto, long, final, float, short, double, native, throws, booleano, abstracto, volátil, transitorio y sincronizado; añadió let y yield.

Los implements, let, private, public, interface, package, protected, static y yield están no permitidos solo en modo estricto.

eval y arguments no son palabras reservadas pero actúan como tal en modo estricto.

Futuras palabras clave reservadas

Las siguientes están reservadas como futuras palabras clave por la especificación ECMAScript. Actualmente no tienen una funcionalidad especial, pero podrían tenerla en el futuro, por lo que no se pueden usar como identificadores.
enumeración

Los siguientes solo están reservados cuando se encuentran en código de modo estricto:
implementos paquete público
interfaz privado ’estático'
dejar protegido

Futuras palabras clave reservadas en estándares más antiguos

Las siguientes están reservadas como palabras clave futuras por especificaciones ECMAScript anteriores (ECMAScript 1 a 3).
abstracto flotador corto
booleano ir a sincronizado
byte instancia de lanza
char int transitorio
doble largo volátil
final nativo

Además, los literales nulo, verdadero y falso no se pueden usar como identificadores en ECMAScript.

De la [Red de desarrolladores de Mozilla] (https://developer.mozilla.org/en-US/docs/Web/JavaScript/Reference/Lexical_grammar).

Identificadores y nombres de identificadores #

Con respecto a las palabras reservadas, hay una pequeña distinción entre los “Identificadores” utilizados para nombres de variables o funciones y los “Nombres de identificadores” permitidos como propiedades de tipos de datos compuestos.

Por ejemplo, lo siguiente dará como resultado un error de sintaxis ilegal:

var break = true;

Error de sintaxis no detectado: ruptura de token inesperada

Sin embargo, el nombre se considera válido como propiedad de un objeto (a partir de ECMAScript 5+):

var obj = {
    break: true
};
console.log(obj.break);

Para citar de [esta respuesta] (http://stackoverflow.com/questions/40209367/do-reserved-words-need-to-be-quoted-when-set-as-property-names-of-javascript-obj/ 40210179#40210179):

De ECMAScript® 5.1 Language Specification:

Sección 7.6

Los nombres de identificadores son tokens que se interpretan de acuerdo con la gramática dada en la sección “Identificadores” del capítulo 5 del estándar Unicode, con algunas pequeñas modificaciones. Un ‘Identificador’ es un ‘Nombre de identificador’ que no es una ‘Palabra reservada’ (ver 7.6.1) .

Sintaxis

Identificador :: IdentifierName pero no ReservedWord

Por especificación, una palabra reservada es:

Sección 7.6.1

Una palabra reservada es un IdentifierName que no se puede utilizar como un Identifier.

Palabra Reservada :: Palabra clave FuturoPalabraReservada NullLiteral Literal Booleano

Esto incluye palabras clave, palabras clave futuras, null y literales booleanos. La lista completa de palabras clave se encuentra en las Secciones 7.6.1 y las literales se encuentran en la [Sección 7.8](http: //www.ecma-international.org/ecma-262/5.1/#sec-7.8).

Lo anterior (Sección 7.6) implica que los IdentifierNames pueden ser ReservedWords, y de la especificación para [object initializers](http://www.ecma-international.org/ecma-262/5.1/#sec- 11.1.5):

Sección 11.1.5

Sintaxis

LiteralObjeto : { } { NombreDePropiedadYListaDeValores } { NombreDePropiedadYListaDeValores , }

Donde PropertyName es, por especificación:

Nombre de la propiedad: Nombre del identificador Literal de cadena NumericLiteral

Como puede ver, un Nombre de la propiedad puede ser un Nombre del identificador, lo que permite que las Palabras reservadas sean Nombre de la propiedad. Eso nos dice de manera concluyente que, por especificación, se permite tener ‘PalabrasReservadas’ como ‘clase’ y ‘var’ como ‘NombreDePropiedad’ sin comillas, al igual que los literales de cadena o los literales numéricos.

Para obtener más información, consulte la Sección 7.6 - Nombres de identificadores e identificadores.

Nota: el resaltador de sintaxis en este ejemplo detectó la palabra reservada y aún la resaltó. Si bien el ejemplo es válido, los desarrolladores de Javascript pueden verse atrapados por algunas herramientas de compilación/transpilación, borrado y minificación que argumentan lo contrario.
