Variables del intérprete interactivo #

Tenga en cuenta que estos solo funcionan en las herramientas de desarrollo de ciertos navegadores.

$_ te da el valor de la última expresión que se evaluó.

"foo"             // "foo"
$_                // "foo"

$0 se refiere al elemento DOM actualmente seleccionado en el Inspector. Entonces, si <div id="foo"> está resaltado:

$0                      // <div id="foo">
$0.getAttribute('id')   // "foo"

$1 se refiere al elemento previamente seleccionado, $2 al elemento seleccionado antes de eso, y así sucesivamente para $3 y $4.

Para obtener una colección de elementos que coincidan con un selector CSS, use $$(selector). Esto es esencialmente un atajo para document.querySelectorAll.

var images = $$('img');  // Returns an array or a nodelist of all matching elements

Puntos de ruptura #

Los puntos de interrupción pausan su programa una vez que la ejecución llega a cierto punto. Luego puede recorrer el programa línea por línea, observando su ejecución e inspeccionando el contenido de sus variables.

Hay tres formas de crear puntos de interrupción.

  1. Desde el código, utilizando la sentencia debugger;.
  2. Desde el navegador, utilizando las herramientas de desarrollo.
  3. Desde un Entorno de Desarrollo Integrado (IDE).

Declaración del depurador #

Puede colocar una instrucción debugger; en cualquier parte de su código JavaScript. Una vez que el intérprete de JS llegue a esa línea, detendrá la ejecución del script, lo que le permitirá inspeccionar las variables y recorrer el código.

Herramientas de desarrollo #

La segunda opción es agregar un punto de interrupción directamente en el código desde las herramientas de desarrollo del navegador.

Apertura de las herramientas de desarrollo #

Chrome o Firefox #

  1. Presione F12 para abrir las Herramientas para desarrolladores
  2. Cambie a la pestaña Fuentes (Chrome) o la pestaña Depurador (Firefox)
  3. Presione Ctrl+P y escriba el nombre de su archivo JavaScript
  4. Presione Enter para abrirlo.

Internet Explorer o borde #

  1. Presione F12 para abrir las Herramientas para desarrolladores
  2. Cambie a la pestaña Depurador.
  3. Utilice el icono de la carpeta cerca de la esquina superior izquierda de la ventana para abrir un panel de selección de archivos; puede encontrar su archivo JavaScript allí.

safari #

  1. Presione Comando+Opción+C para abrir Herramientas para desarrolladores
  2. Cambie a la pestaña Recursos
  3. Abra la carpeta “Scripts” en el panel del lado izquierdo
  4. Seleccione su archivo JavaScript.

Agregar un punto de interrupción desde las Herramientas para desarrolladores #

Una vez que haya abierto su archivo JavaScript en las Herramientas para desarrolladores, puede hacer clic en un número de línea para colocar un punto de interrupción. La próxima vez que se ejecute su programa, se detendrá allí.

Nota sobre las fuentes minificadas: si su fuente está minificada, puede imprimirla con Pretty Print (convertirla a un formato legible). En Chrome, esto se hace haciendo clic en el botón {} en la esquina inferior derecha del visor de código fuente.

IDE #

Código de Visual Studio (VSC) #

VSC tiene soporte integrado para depurar JavaScript.

  1. Haga clic en el botón Depurar a la izquierda o Ctrl+Shift+D
  2. Si aún no lo ha hecho, cree un archivo de configuración de lanzamiento (launch.json) presionando el ícono de ajustes.
  3. Ejecute el código de VSC presionando el botón verde de reproducción o presionando F5.

Agregar un punto de interrupción en VSC #

Haga clic junto al número de línea en su archivo fuente de JavaScript para agregar un punto de interrupción (se marcará en rojo). Para eliminar el punto de interrupción, vuelva a hacer clic en el círculo rojo.

Consejo: También puede utilizar los puntos de interrupción condicionales en las herramientas de desarrollo del navegador. Estos ayudan a omitir interrupciones innecesarias en la ejecución. Escenario de ejemplo: desea examinar una variable en un ciclo exactamente en la 5th iteración.

Usar setters y getters para encontrar qué cambió una propiedad #

Digamos que tienes un objeto como este:

var myObject = {
    name: 'Peter'
}

Más adelante en su código, intenta acceder a myObject.name y obtiene George en lugar de Peter. Empiezas a preguntarte quién lo cambió y dónde se cambió exactamente. Hay una forma de colocar un depurador (o algo más) en cada conjunto (cada vez que alguien hace myObject.name = 'something'):

var myObject = {
    _name: 'Peter',
    set name(name){debugger;this._name=name},
    get name(){return this._name}
}

Tenga en cuenta que cambiamos el nombre de name a _name y vamos a definir un setter y un getter para name.

set name es el setter. Ese es un punto óptimo donde puede colocar debugger, console.trace(), o cualquier otra cosa que necesite para la depuración. El setter establecerá el valor de name en _name. El captador (la parte obtener nombre) leerá el valor desde allí. Ahora tenemos un objeto completamente funcional con funcionalidad de depuración.

Sin embargo, la mayoría de las veces, el objeto que se cambia no está bajo nuestro control. Afortunadamente, podemos definir setters y getters en objetos existentes para depurarlos.

// First, save the name to _name, because we are going to use name for setter/getter
otherObject._name = otherObject.name;

// Create setter and getter
Object.defineProperty(otherObject, "name", {
    set: function(name) {debugger;this._name = name},
    get: function() {return this._name}
});

Echa un vistazo a setters y getters en MDN para obtener más información.

Usando la consola #

En muchos entornos, tiene acceso a un objeto de “consola” global que contiene algunos métodos básicos para comunicarse con dispositivos de salida estándar. Por lo general, será la consola JavaScript del navegador (consulte Chrome, [Firefox](https: //developer.mozilla.org/en-US/docs/Tools/Browser_Console), Safari y [Edge](https://developer. microsoft.com/en-us/microsoft-edge/platform/documentation/f12-devtools-guide/console/) para obtener más información).

// At its simplest, you can 'log' a string
console.log("Hello, World!");

// You can also log any number of comma-separated values
console.log("Hello", "World!");

// You can also use string substitution
console.log("%s %s", "Hello", "World!");

// You can also log any variable that exist in the same scope
var arr = [1, 2, 3];
console.log(arr.length, this);

Puede usar diferentes métodos de consola para resaltar su salida de diferentes maneras. Otros métodos también son útiles para una depuración más avanzada.

Para obtener más documentación, información sobre compatibilidad e instrucciones sobre cómo abrir la consola de su navegador, consulte el tema Consola.

Nota: si necesita admitir IE9, elimine console.log o ajuste sus llamadas de la siguiente manera, porque console no está definido hasta que se abren las herramientas de desarrollo:

if (console) { //IE9 workaround
    console.log("test");
}

Pausa automática de la ejecución #

En Google Chrome, puede pausar la ejecución sin necesidad de colocar puntos de interrupción.

Pausa en excepción: Mientras este botón está activado, si su programa encuentra una excepción no controlada, el programa se detendrá como si hubiera llegado a un punto de interrupción. El botón se puede encontrar cerca de los controles de ejecución y es útil para localizar errores.

También puede pausar la ejecución cuando se modifica una etiqueta HTML (nodo DOM) o cuando se cambian sus atributos. Para hacer eso, haga clic con el botón derecho en el nodo DOM en la pestaña Elementos y seleccione “Interrumpir en…”.

Inspector de elementos #

Al hacer clic en el botón Seleccione un elemento en la página para inspeccionarlo en la esquina superior izquierda de la pestaña Elementos en Chrome o la pestaña Inspector en Firefox, disponible en Herramientas para desarrolladores , y luego hacer clic en un elemento de la página resalta el elemento y [lo asigna a la variable $0](https://www.wikiod.com/es/javascript/depuracion#Variables de intérprete interactivo =201607211329474646179).

El inspector de elementos se puede utilizar de varias formas, por ejemplo:

  1. Puede verificar si su JS está manipulando DOM de la manera que espera,
  2. Puede depurar más fácilmente su CSS, al ver qué reglas afectan al elemento (pestaña Estilos en Chrome)
  3. Puedes jugar con CSS y HTML sin recargar la página.

Además, Chrome recuerda las últimas 5 selecciones en la pestaña Elementos. $0 es la selección actual, mientras que $1 es la selección anterior. Puedes subir hasta $4. De esa manera, puede depurar fácilmente múltiples nodos sin cambiar constantemente la selección a ellos.

Puede leer más en Google Developers.

Recorriendo el código #

Una vez que haya detenido la ejecución en un punto de interrupción, es posible que desee seguir la ejecución línea por línea para observar lo que sucede. [Abra las herramientas de desarrollo de su navegador] (https://www.wikiod.com/es/javascript/depuracion#puntos de ruptura) y busque los iconos de control de ejecución. (Este ejemplo usa los íconos en Google Chrome, pero serán similares en otros navegadores).

Reanudar: Reanudar la ejecución. Acceso directo:F8(Chrome, Firefox)

Step Over: Ejecute la siguiente línea de código. Si esa línea contiene una llamada de función, ejecute la función completa y pase a la siguiente línea, en lugar de saltar a donde esté definida la función. Acceso directo: F10(Chrome, Firefox, IE/Edge), F6(Safari)

Ingresar: Ejecutar la siguiente línea de código. Si esa línea contiene una llamada de función, vaya a la función y haga una pausa allí. Acceso directo: F11(Chrome, Firefox, IE/Edge), F7(Safari)

Step Out: Ejecuta el resto de la función actual, vuelve al lugar desde el que se llamó a la función y haz una pausa en la siguiente instrucción allí. Atajo: Shift + F11(Chrome, Firefox, IE/Edge), F8(Safari)

Utilícelos junto con la Pila de llamadas, que le indicará en qué función se encuentra actualmente, qué función llamó a esa función, etc.

Consulte la guía de Google sobre “Cómo recorrer paso a paso el código” para obtener más detalles y consejos. .

Interrumpir cuando se llama a una función #

Para las funciones con nombre (no anónimas), puede romper cuando se ejecuta la función.

debug(functionName);

La próxima vez que se ejecute la función functionName, el depurador se detendrá en su primera línea.
