Ejemplos de uso de devolución de llamada simples #

Las devoluciones de llamada ofrecen una forma de ampliar la funcionalidad de una función (o método) sin cambiar su código. Este enfoque se usa a menudo en módulos (bibliotecas / complementos), cuyo código se supone que no se debe cambiar.

Supongamos que hemos escrito la siguiente función, calculando la suma de una matriz dada de valores:

function foo(array) {
    var sum = 0;
    for (var i = 0; i < array.length; i++) {
        sum += array[i];
    }
    return sum;
}

Ahora suponga que queremos hacer algo con cada valor de la matriz, p. mostrarlo usando alert(). Podríamos hacer los cambios apropiados en el código de foo, así:

function foo(array) {
    var sum = 0;
    for (var i = 0; i < array.length; i++) {
        alert(array[i]);
        sum += array[i];
    }
    return sum;
}

Pero, ¿y si decidimos usar console.log en lugar de alert()? Obviamente, cambiar el código de foo, siempre que decidamos hacer otra cosa con cada valor, no es una buena idea. Es mucho mejor tener la opción de cambiar de opinión sin cambiar el código de foo. Ese es exactamente el caso de uso para las devoluciones de llamada. Solo tenemos que cambiar ligeramente la firma y el cuerpo de foo:

function foo(array, callback) {
    var sum = 0;
    for (var i = 0; i < array.length; i++) {
        callback(array[i]);
        sum += array[i];
    }
    return sum;
}

Y ahora podemos cambiar el comportamiento de foo simplemente cambiando sus parámetros:

var array = [];
foo(array, alert);
foo(array, function (x) {
    console.log(x);
});

Ejemplos con funciones asíncronas #

En jQuery, el método $.getJSON() para obtener datos JSON es asíncrono. Por lo tanto, pasar el código en una devolución de llamada asegura que el código se llame después de que se obtenga el JSON.

Sintaxis $.getJSON():

$.getJSON( url, dataObject, successCallback );

Ejemplo de código $.getJSON():

$.getJSON("foo.json", {}, function(data) {
    // data handling code
});

Lo siguiente no funcionaría, porque el código de manejo de datos probablemente se llamaría antes de que realmente se reciban los datos, porque la función $.getJSON toma un período de tiempo no especificado y no retiene la pila de llamadas como espera el JSON.

$.getJSON("foo.json", {});
// data handling code

Otro ejemplo de una función asíncrona es la función animate() de jQuery. Debido a que lleva un tiempo específico ejecutar la animación, a veces es deseable ejecutar algún código directamente después de la animación.

Sintaxis de .animate():

jQueryElement.animate( properties, duration, callback );

Por ejemplo, para crear una animación de desvanecimiento después de la cual el elemento desaparece por completo, se puede ejecutar el siguiente código. Tenga en cuenta el uso de la devolución de llamada.

elem.animate( { opacity: 0 }, 5000, function() {
    elem.hide();
} );

Esto permite que el elemento se oculte justo después de que la función haya terminado de ejecutarse. Esto difiere de:

elem.animate( { opacity: 0 }, 5000 );
elem.hide();

porque este último no espera a que se complete animate() (una función asíncrona) y, por lo tanto, el elemento se oculta de inmediato, produciendo un efecto no deseado.

¿Qué es una devolución de llamada? #

Esta es una llamada de función normal:

console.log("Hello World!");

Cuando llama a una función normal, hace su trabajo y luego devuelve el control a la persona que llama.

Sin embargo, a veces una función necesita devolver el control a la persona que llama para hacer su trabajo:

[1,2,3].map(function double(x) {
    return 2 * x;
});

En el ejemplo anterior, la función ‘doble’ es una devolución de llamada para la función ‘mapa’ porque:

  1. La persona que llama asigna la función ‘doble’ a la función ‘mapa’.
  2. La función mapa necesita llamar a la función doble cero o más veces para hacer su trabajo.

Por lo tanto, la función mapa esencialmente devuelve el control a la persona que llama cada vez que llama a la función doble. Por lo tanto, el nombre “devolución de llamada”.

Las funciones pueden aceptar más de una devolución de llamada:

promise.then(function onFulfilled(value) {
    console.log("Fulfilled with value " + value);
}, function onRejected(reason) {
    console.log("Rejected with reason " + reason);
});

Aquí entonces la función then acepta dos funciones de devolución de llamada, onFulfilled y onRejected. Además, solo se llama realmente a una de estas dos funciones de devolución de llamada.

Lo que es más interesante es que la función entonces regresa antes de que se llame a cualquiera de las devoluciones de llamada. Por lo tanto, se puede llamar a una función de devolución de llamada incluso después de que haya regresado la función original.

Continuación (síncrona y asíncrona) #

Las devoluciones de llamada se pueden usar para proporcionar código que se ejecutará después de que se haya completado un método:

/**
 * @arg {Function} then continuation callback
 */
function doSomething(then) {
  console.log('Doing something');
  then();
}

// Do something, then execute callback to log 'done'
doSomething(function () {
  console.log('Done');
});

console.log('Doing something else');

// Outputs:
//   "Doing something"
//   "Done"
//   "Doing something else"

El método anterior doSomething() se ejecuta sincrónicamente con la devolución de llamada: la ejecución se bloquea hasta que regresa doSomething(), lo que garantiza que la devolución de llamada se ejecute antes de que el intérprete continúe.

Las devoluciones de llamada también se pueden usar para ejecutar código de forma asíncrona:

doSomethingAsync(then) {
  setTimeout(then, 1000);
  console.log('Doing something asynchronously');
}

doSomethingAsync(function() {
  console.log('Done');
});

console.log('Doing something else');

// Outputs:
//   "Doing something asynchronously"
//   "Doing something else"
//   "Done"

Las devoluciones de llamada then se consideran continuaciones de los métodos doSomething(). Proporcionar una devolución de llamada como última instrucción en una función se denomina [llamada final] (https://en.wikipedia.org/wiki/Tail_call), que está [optimizada por los intérpretes de ES2015] (http://www.2ality .com/2015/06/tail-call-optimization.html).

Devoluciones de llamada y esto #

A menudo, cuando se utiliza una devolución de llamada, desea acceder a un contexto específico.

function SomeClass(msg, elem) {
  this.msg = msg;
  elem.addEventListener('click', function() {
    console.log(this.msg);  // <= will fail because "this" is undefined
  });
}

var s = new SomeClass("hello", someElement);

Soluciones #

  • Usa bind

bind genera efectivamente una nueva función que establece this a lo que sea se pasó a bind y luego llama a la función original.

    function SomeClass(msg, elem) {
      this.msg = msg;
      elem.addEventListener('click', function() {
        console.log(this.msg);  
      }.bind(this));  // <=-  bind the function to `this`
    }
  • Usar funciones de flecha

Las funciones de flecha vinculan automáticamente el contexto this actual.

    function SomeClass(msg, elem) {
      this.msg = msg;
      elem.addEventListener('click',() => {   // <=-  arrow function binds `this`
        console.log(this.msg);  
      });
    }

A menudo, le gustaría llamar a una función miembro, idealmente pasando los argumentos que se pasaron al evento a la función.

Soluciones: #

  • Usar enlace

      function SomeClass(msg, elem) {
    this.msg = msg;
    elem.addEventListener('click', this.handleClick.bind(this));
  }

  SomeClass.prototype.handleClick = function(event) {
    console.log(event.type, this.msg);
  };

  • Usar funciones de flecha y el resto del operador

      function SomeClass(msg, elem) {
    this.msg = msg;
    elem.addEventListener('click', (...a) => this.handleClick(...a));
  }

  SomeClass.prototype.handleClick = function(event) {
    console.log(event.type, this.msg);
  };

  • Para los detectores de eventos DOM en particular, puede implementar la [interfaz EventListener] (https://developer.mozilla.org/en-US/docs/Web/API/EventListener)

      function SomeClass(msg, elem) {
    this.msg = msg;
    elem.addEventListener('click', this);
  }

  SomeClass.prototype.handleEvent = function(event) {
    var fn = this[event.type];
    if (fn) {
       fn.apply(this, arguments);
    }
  };

  SomeClass.prototype.click = function(event) {
    console.log(this.msg);
  };

Manejo de errores y bifurcación del flujo de control #

Las devoluciones de llamada se utilizan a menudo para proporcionar el manejo de errores. Esta es una forma de bifurcación del flujo de control, donde algunas instrucciones se ejecutan solo cuando ocurre un error:

const expected = true;

function compare(actual, success, failure) {
  if (actual === expected) {
    success();
  } else {
    failure();
  }
}

function onSuccess() {
  console.log('Value was expected');
}

function onFailure() {
  console.log('Value was unexpected/exceptional');
}

compare(true, onSuccess, onFailure);
compare(false, onSuccess, onFailure);

// Outputs:
//   "Value was expected"
//   "Value was unexpected/exceptional"

La ejecución del código en compare() anterior tiene dos ramas posibles: success cuando los valores esperados y reales son los mismos, y error cuando son diferentes. Esto es especialmente útil cuando el flujo de control debe bifurcarse después de alguna instrucción asíncrona:

function compareAsync(actual, success, failure) {
  setTimeout(function () {
    compare(actual, success, failure)
  }, 1000);
}

compareAsync(true, onSuccess, onFailure);
compareAsync(false, onSuccess, onFailure);
console.log('Doing something else');

// Outputs:
//   "Doing something else"
//   "Value was expected"
//   "Value was unexpected/exceptional"

Cabe señalar que las devoluciones de llamada múltiples no tienen que ser mutuamente exclusivas: se pueden llamar a ambos métodos. Del mismo modo, compare() podría escribirse con devoluciones de llamada que son opcionales (mediante el uso de noop como valor predeterminado; consulte Patrón de objeto nulo).

Devolución de llamada usando la función de flecha #

El uso de la función de flecha como función de devolución de llamada puede reducir las líneas de código.

La sintaxis predeterminada para la función de flecha es

() => {}

Esto se puede utilizar como devoluciones de llamada

Por ejemplo, si queremos imprimir todos los elementos en una matriz [1,2,3,4,5]

sin la función de flecha, el código se verá así

[1,2,3,4,5].forEach(function(x){
                 console.log(x);
            }

Con la función de flecha, se puede reducir a

[1,2,3,4,5].forEach(x => console.log(x));

Aquí la función de devolución de llamada function(x){console.log(x)} se reduce a x=>console.log(x)
