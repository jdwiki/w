Resto / Módulo (%) #

El operador de resto/módulo (%) devuelve el resto después de la división (entero).

console.log( 42 %  10); //  2
console.log( 42 % -10); //  2
console.log(-42 %  10); // -2
console.log(-42 % -10); // -2
console.log(-40 %  10); // -0
console.log( 40 %  10); //  0

Este operador devuelve el resto que queda cuando un operando se divide por un segundo operando. Cuando el primer operando es un valor negativo, el valor devuelto siempre será negativo y viceversa para valores positivos.

En el ejemplo anterior, 10 se puede restar cuatro veces de 42 antes de que no quede suficiente para restar nuevamente sin que cambie de signo. El resto es así: 42 - 4 * 10 = 2.

El operador resto puede ser útil para los siguientes problemas:

  1. Prueba si un número entero es (no) divisible por otro número:

     x % 4 == 0 // true if x is divisible by 4
 x % 2 == 0 // true if x is even number
 x % 2 != 0 // true if x is odd number

Dado que 0 === -0, esto también funciona para x <= -0.

  1. Implemente el incremento/decremento cíclico del valor dentro del intervalo [0, n).

Supongamos que necesitamos incrementar el valor entero de 0 a (pero sin incluir) n, por lo que el siguiente valor después de n-1 se convierte en 0. Esto se puede hacer con tal pseudocódigo:

var n = ...; // given n
var i = 0;
function inc() {
    i = (i + 1) % n;
}
while (true) {
    inc();
    // update something with i
}

Ahora generalice el problema anterior y suponga que necesitamos permitir tanto incrementar como decrementar ese valor de 0 a (sin incluir) n, por lo que el siguiente valor después de n-1 se convierte en 0 y el valor anterior antes de que 0 se convierta en n-1.

var n = ...; // given n
var i = 0;
function delta(d) { // d - any signed integer
    i = (i + d + n) % n; // we add n to (i+d) to ensure the sum is positive
}

Ahora podemos llamar a la función delta() pasando cualquier número entero, tanto positivo como negativo, como parámetro delta.

Uso del módulo para obtener la parte fraccionaria de un número #

 var myNum = 10 / 4;       // 2.5
 var fraction = myNum % 1; // 0.5
 myNum = -20 / 7;          // -2.857142857142857
 fraction = myNum % 1;     // -0.857142857142857

Redondeo #

Redondeo #

Math.round() redondeará el valor al entero más cercano usando redondeo a la mitad para desempatar.

var a = Math.round(2.3);       // a is now 2  
var b = Math.round(2.7);       // b is now 3
var c = Math.round(2.5);       // c is now 3

Pero

var c = Math.round(-2.7);       // c is now -3
var c = Math.round(-2.5);       // c is now -2

Observe cómo -2.5 se redondea a -2. Esto se debe a que los valores intermedios siempre se redondean hacia arriba, es decir, se redondean al número entero con el siguiente valor más alto.

Redondeando #

Math.ceil() redondeará el valor hacia arriba.

var a = Math.ceil(2.3);        // a is now 3
var b = Math.ceil(2.7);        // b is now 3

ceiling un número negativo se redondeará hacia cero

var c = Math.ceil(-1.1);       // c is now 1

Redondeando hacia abajo #

Math.floor() redondeará el valor hacia abajo.

var a = Math.floor(2.3);        // a is now 2
var b = Math.floor(2.7);        // b is now 2

‘pisar’ un número negativo lo redondeará desde cero.

var c = Math.floor(-1.1);       // c is now -1

Truncando #

Advertencia: el uso de operadores bit a bit (excepto >>>) solo se aplica a los números entre -2147483649 y 2147483648.

2.3  | 0;                       // 2 (floor)
-2.3 | 0;                       // -2 (ceil)
NaN  | 0;                       // 0

Matemáticas.trunc()

Math.trunc(2.3);                // 2 (floor)
Math.trunc(-2.3);               // -2 (ceil)
Math.trunc(2147483648.1);       // 2147483648 (floor)
Math.trunc(-2147483649.1);      // -2147483649 (ceil)
Math.trunc(NaN);                // NaN

Redondeo a decimales #

Math.floor, Math.ceil() y Math.round() se pueden usar para redondear a un número de lugares decimales

Para redondear a 2 decimales:

 var myNum = 2/3;               // 0.6666666666666666
 var multiplier = 100;
 var a = Math.round(myNum * multiplier) / multiplier;  // 0.67
 var b = Math.ceil (myNum * multiplier) / multiplier;  // 0.67
 var c = Math.floor(myNum * multiplier) / multiplier;  // 0.66

También puede redondear a un número de dígitos:

 var myNum = 10000/3;           // 3333.3333333333335
 var multiplier = 1/100;
 var a = Math.round(myNum * multiplier) / multiplier;  // 3300
 var b = Math.ceil (myNum * multiplier) / multiplier;  // 3400
 var c = Math.floor(myNum * multiplier) / multiplier;  // 3300

Como una función más utilizable:

 // value is the value to round
 // places if positive the number of decimal places to round to
 // places if negative the number of digits to round to
 function roundTo(value, places){
     var power = Math.pow(10, places);
     return Math.round(value * power) / power;
 }
 var myNum = 10000/3;    // 3333.3333333333335
 roundTo(myNum, 2);  // 3333.33
 roundTo(myNum, 0);  // 3333
 roundTo(myNum, -2); // 3300

Y las variantes para techo y suelo:

 function ceilTo(value, places){
     var power = Math.pow(10, places);
     return Math.ceil(value * power) / power;
 }
 function floorTo(value, places){
     var power = Math.pow(10, places);
     return Math.floor(value * power) / power;
 }

trigonometría #

Todos los ángulos a continuación están en radianes. Un ángulo r en radianes tiene la medida 180 * r / Math.PI en grados.

Seno #

Math.sin(r);

Esto devolverá el seno de r, un valor entre -1 y 1.

Math.asin(r);

Esto devolverá el arcoseno (el reverso del seno) de r.

Math.asinh(r)

Esto devolverá el arcoseno hiperbólico de r.

Coseno #

Math.cos(r);

Esto devolverá el coseno de r, un valor entre -1 y 1

Math.acos(r);

Esto devolverá el arcocoseno (el reverso del coseno) de r.

Math.acosh(r);

Esto devolverá el arcocoseno hiperbólico de r.

Tangente #

Esto devolverá la tangente de r.

Math.atan(r);

Esto devolverá el arcotangente (el reverso de la tangente) de r. Tenga en cuenta que devolverá un ángulo en radianes entre -π/2 y π/2.

Incrementando (++) #

El operador de incremento (++) incrementa su operando en uno.

  • Si se usa como sufijo, devuelve el valor antes de incrementar.
  • Si se usa como prefijo, devuelve el valor después de incrementar.
//postfix
var a = 5,    // 5
    b = a++,  // 5
    c = a     // 6

En este caso, a se incrementa después de configurar b. Entonces, b será 5 y c será 6.

//prefix
var a = 5,    // 5
    b = ++a,  // 6
    c = a     // 6

En este caso, a se incrementa antes de establecer b. Entonces, ‘b’ será 6 y ‘c’ será 6.

Los operadores de incremento y decremento se usan comúnmente en bucles for, por ejemplo:

for(var i = 0; i < 42; ++i)
{
  // do something awesome!
}

Observe cómo se usa la variante prefix. Esto garantiza que no se cree innecesariamente una variable temporal (para devolver el valor antes de la operación).

Operadores bit a bit #

Tenga en cuenta que todas las operaciones bit a bit operan en enteros de 32 bits al pasar los operandos a la función interna ToInt32.

Bit a bit o #

var a;
a = 0b0011 | 0b1010; // a === 0b1011
// truth table
// 1010 | (or)
// 0011  
// 1011  (result)

Bit a bit y #

a = 0b0011 & 0b1010; // a === 0b0010
// truth table
// 1010 & (and)
// 0011  
// 0010  (result)

Bit a bit no #

a = ~0b0011; // a === 0b1100
// truth table
// 10 ~(not)
// 01  (result)

Bitwise xor (exclusivo o) #

a = 0b1010 ^ 0b0011; // a === 0b1001
// truth table
// 1010 ^ (xor)
// 0011  
// 1001  (result)

Bit a bit desplazamiento a la izquierda #

a = 0b0001 << 1; // a === 0b0010
a = 0b0001 << 2; // a === 0b0100
a = 0b0001 << 3; // a === 0b1000

Desplazar a la izquierda es equivalente a multiplicar enteros por Math.pow(2, n). Al hacer matemáticas con números enteros, shift puede mejorar significativamente la velocidad de algunas operaciones matemáticas.

var n = 2;
var a = 5.4;
var result = (a << n) === Math.floor(a) * Math.pow(2,n);
// result is true
a = 5.4 << n; // 20

Bitwise right shift >> (Sign-propagating shift) >>> (Zero-fill right shift) #

a = 0b1001 >> 1; // a === 0b0100
a = 0b1001 >> 2; // a === 0b0010
a = 0b1001 >> 3; // a === 0b0001

a = 0b1001 >>> 1; // a === 0b0100
a = 0b1001 >>> 2; // a === 0b0010
a = 0b1001 >>> 3; // a === 0b0001

Un valor negativo de 32 bits siempre tiene el bit más a la izquierda en:

a = 0b11111111111111111111111111110111 | 0;   
console.log(a); // -9
b = a >> 2;     // leftmost bit is shifted 1 to the right then new left most bit is set to on (1)
console.log(b); // -3
b = a >>> 2;    // leftmost bit is shifted 1 to the right. the new left most bit is set to off (0)
console.log(b); // 2147483643

El resultado de una operación >>> siempre es positivo. El resultado de >> siempre tiene el mismo signo que el valor desplazado.

El desplazamiento a la derecha en números positivos es el equivalente a dividir por Math.pow(2,n) y sacar el resultado:

a = 256.67;
n = 4;
result = (a >> n) === Math.floor( Math.floor(a) / Math.pow(2,n) );
// result is true
a = a >> n; //  16

result = (a >>> n) === Math.floor( Math.floor(a) / Math.pow(2,n) );
// result is true
a = a >>> n; //  16

El relleno cero con desplazamiento a la derecha (>>>) en números negativos es diferente. Como JavaScript no se convierte en entradas sin firmar cuando se realizan operaciones de bits, no existe un equivalente operativo:

a = -256.67;
result = (a >>> n) === Math.floor( Math.floor(a) / Math.pow(2,n) );
// result is false

Con la excepción de not (~), todos los operadores bit a bit anteriores se pueden usar como operadores de asignación:

a &= b;   // same as: a = a & b;

Advertencia: Javascript usa Big Endian para almacenar números enteros. Esto no siempre coincidirá con el Endian del dispositivo/SO. Cuando utilice matrices tipadas con longitudes de bits superiores a 8 bits, debe comprobar si el entorno es Little Endian o Big Endian antes de aplicar operaciones bit a bit.

Advertencia: Los operadores bit a bit como & y | no son lo mismo que los operadores lógicos [&& (y)](https://www.wikiod.com/es/docs/javascript 208/boolean-logic/779/logic-operators-with-booleans#t=201606101417337061178) y [|| (o)](https://www.wikiod.com/es/docs/javascript208/boolean-logic/779 /operadores-lógicos-con-booleanos#t=201606101417337061178). Simulando eventos con diferentes probabilidades #

A veces, es posible que solo necesite simular un evento con dos resultados, tal vez con diferentes probabilidades, pero puede encontrarse en una situación que requiere muchos resultados posibles con diferentes probabilidades. Imaginemos que desea simular un evento que tiene seis resultados igualmente probables. Esto es bastante simple.

function simulateEvent(numEvents) {
    var event = Math.floor(numEvents*Math.random());
    return event;
}

// simulate fair die
console.log("Rolled a "+(simulateEvent(6)+1));  // Rolled a 2

Sin embargo, es posible que no desee resultados igualmente probables. Supongamos que tiene una lista de tres resultados representados como una matriz de probabilidades en porcentajes o múltiplos de probabilidad. Tal ejemplo podría ser un dado ponderado. Podría reescribir la función anterior para simular tal evento.

function simulateEvent(chances) {
    var sum = 0;
    chances.forEach(function(chance) {
        sum+=chance;
    });
    var rand = Math.random();
    var chance = 0;
    for(var i=0; i<chances.length; i++) {
        chance+=chances[i]/sum;
        if(rand<chance) {
            return i;
        }
    }
    
    // should never be reached unless sum of probabilities is less than 1
    // due to all being zero or some being negative probabilities
    return -1;
}

// simulate weighted dice where 6 is twice as likely as any other face
// using multiples of likelihood
console.log("Rolled a "+(simulateEvent([1,1,1,1,1,2])+1));  // Rolled a 1

// using probabilities
console.log("Rolled a "+(simulateEvent([1/7,1/7,1/7,1/7,1/7,2/7])+1));  // Rolled a 6

Como probablemente notó, estas funciones devuelven un índice, por lo que podría tener resultados más descriptivos almacenados en una matriz. Aquí hay un ejemplo.

var rewards = ["gold coin","silver coin","diamond","god sword"];
var likelihoods = [5,9,1,0];
// least likely to get a god sword (0/15 = 0%, never),
// most likely to get a silver coin (9/15 = 60%, more than half the time)

// simulate event, log reward
console.log("You get a "+rewards[simulateEvent(likelihoods)]);  // You get a silver coin

Little / Big endian para arreglos escritos cuando se usan operadores bit a bit #

Para detectar el endian del dispositivo

var isLittleEndian = true;
(()=>{
    var buf = new ArrayBuffer(4);
    var buf8 = new Uint8ClampedArray(buf);
    var data = new Uint32Array(buf);
    data[0] = 0x0F000000;
    if(buf8[0] === 0x0f){
        isLittleEndian = false;
    }
})();

Little-Endian almacena los bytes más significativos de derecha a izquierda.

Big-Endian almacena los bytes más significativos de izquierda a derecha.

var buf = new ArrayBuffer(4);
var data8 = new Uint8ClampedArray(buf);

Si el sistema usa Little-Endian, los valores de byte de 8 bits serán

console.log(data8[2].toString(16)); // 0x22
console.log(data8[3].toString(16)); // 0x11

Si el sistema usa Big-Endian, entonces los valores de bytes de 8 bits serán

console.log(data8[0].toString(16)); // 0x11

console.log(data8[2].toString(16)); // 0x33
console.log(data8[3].toString(16)); // 0x44

Ejemplo donde el tipo Edian es importante

var canvas = document.createElement("canvas");
var ctx = canvas.getContext("2d");
var imgData = ctx.getImageData(0, 0, canvas.width, canvas.height);
// To speed up read and write from the image buffer you can create a buffer view that is 
// 32 bits allowing you to read/write a pixel in a single operation

// Mask out Red and Blue channels
var mask = 0x00FF00FF; // bigEndian pixel channels Red,Green,Blue,Alpha
if(isLittleEndian){
    mask = 0xFF00FF00; // littleEndian pixel channels Alpha,Blue,Green,Red
}    
var len = buf32.length;

while(i < len){  // Mask all pixels
    buf32[i] &= mask; //Mask out Red and Blue
}

Aleatorio con distribución gaussiana #

La función Math.random() debería dar números aleatorios que tengan una desviación estándar cercana a 0. Al elegir de una baraja o simular una tirada de dados, esto es lo que queremos.

Pero en la mayoría de las situaciones esto no es realista. En el mundo real, la aleatoriedad tiende a acumularse en torno a un valor normal común. Si se representa en un gráfico, se obtiene la curva de campana clásica o la distribución gaussiana.

Hacer esto con la función Math.random() es relativamente simple.

var randNum = (Math.random() + Math.random()) / 2; 
var randNum = (Math.random() + Math.random() + Math.random()) / 3; 
var randNum = (Math.random() + Math.random() + Math.random() + Math.random()) / 4;

Agregar un valor aleatorio al último aumenta la varianza de los números aleatorios. Dividir por la cantidad de veces que agrega normaliza el resultado a un rango de 0–1

Como agregar más de unas pocas aleatorias es complicado, una función simple le permitirá seleccionar la variación que desee.

// v is the number of times random is summed and should be over >= 1
// return a random number between 0-1 exclusive
function randomG(v){ 
    var r = 0;
    for(var i = v; i > 0; i --){
        r += Math.random();
    }
    return r / v;
}

La imagen muestra la distribución de valores aleatorios para diferentes valores de v. La parte superior izquierda es una sola llamada Math.random() estándar y la parte inferior derecha es Math.random() sumada 8 veces. Esto es de 5,000,000 de muestras usando Chrome

Este método es más eficiente en v<5

Sin & Cos para crear un vector con dirección y distancia dadas #

Si tiene un vector en forma polar (dirección y distancia), querrá convertirlo en un vector cartesiano con un componente x e y. Como referencia, el sistema de coordenadas de la pantalla tiene direcciones como puntos de 0 grados de izquierda a derecha, 90 (PI/2) apuntan hacia abajo en la pantalla, y así sucesivamente en el sentido de las agujas del reloj.

var dir = 1.4536; // direction in radians
var dist = 200; // distance
var vec = {};
vec.x = Math.cos(dir) * dist; // get the x component
vec.y = Math.sin(dir) * dist; // get the y component

También puede ignorar la distancia para crear un vector normalizado (1 unidad de largo) en la dirección de dir

var dir = 1.4536; // direction in radians
var vec = {};
vec.x = Math.cos(dir); // get the x component
vec.y = Math.sin(dir); // get the y component

Si su sistema de coordenadas tiene y como arriba, entonces necesita cambiar cos y sen. En este caso, una dirección positiva está en sentido antihorario desde el eje x.

// get the directional vector where y points up
var dir = 1.4536; // direction in radians
var vec = {};
vec.x = Math.sin(dir); // get the x component
vec.y = Math.cos(dir); // get the y component

Matemáticas hipotéticas #

Para encontrar la distancia entre dos puntos usamos Pitágoras para obtener la raíz cuadrada de la suma del cuadrado de la componente del vector entre ellos.

var v1 = {x : 10, y :5};
var v2 = {x : 20, y : 10};
var x = v2.x - v1.x;
var y = v2.y - v1.y;
var distance = Math.sqrt(x * x + y * y); // 11.180339887498949

Con ECMAScript 6 vino Math.hypot que hace lo mismo

var v1 = {x : 10, y :5}; 
var v2 = {x : 20, y : 10}; 
var x = v2.x - v1.x; 
var y = v2.y - v1.y;
var distance = Math.hypot(x,y); // 11.180339887498949

Ahora no tiene que mantener las variables provisionales para evitar que el código se convierta en un lío de variables

var v1 = {x : 10, y :5};
var v2 = {x : 20, y : 10};
var distance = Math.hypot(v2.x - v1.x, v2.y - v1.y); // 11.180339887498949

Math.hypot puede tomar cualquier número de dimensiones

// find distance in 3D
var v1 = {x : 10, y : 5, z : 7};
var v2 = {x : 20, y : 10, z : 16};

// find length of 11th dimensional vector
var v = [1,3,2,6,1,7,3,7,5,3,1]; 
var i = 0;
dist = Math.hypot(v[i++],v[i++],v[i++],v[i++],v[i++],v[i++],v[i++],v[i++],v[i++],v[i++],v[i++]);

Funciones periódicas usando Math.sin #

Math.sin y Math.cos son cíclicos con un período de 2*PI radianes (360 grados), emiten una onda con una amplitud de 2 en el rango de -1 a 1.

Gráfico de la función seno y coseno: (cortesía de Wikipedia) [](https://en.wikipedia.org/wiki /Onda sinusoidal)

Ambos son muy útiles para muchos tipos de cálculos periódicos, desde la creación de ondas de sonido hasta animaciones e incluso la codificación y decodificación de datos de imágenes.

Este ejemplo muestra cómo crear una onda sinusoidal simple con control sobre el período/frecuencia, fase, amplitud y desplazamiento.

La unidad de tiempo que se utiliza es segundos. La forma más simple con control solo sobre la frecuencia.

// time is the time in seconds when you want to get a sample
// Frequency represents the number of oscillations per second
function oscillator(time, frequency){  
    return Math.sin(time * 2 * Math.PI * frequency);
}

En casi todos los casos, querrá realizar algunos cambios en el valor devuelto. Los términos comunes para las modificaciones.

  • Fase: El desfase en términos de frecuencia desde el inicio de las oscilaciones. Es un valor en el rango de 0 a 1 donde el valor 0.5 hace avanzar la onda en el tiempo a la mitad de su frecuencia. Un valor de 0 o 1 no hace ningún cambio.
  • Amplitud: La distancia desde el valor más bajo y el valor más alto durante un ciclo. Una amplitud de 1 tiene un rango de 2. El punto más bajo (valle) -1 al más alto (pico) 1. Para una onda con frecuencia 1, el pico está en 0,25 segundos y el valle en 0,75.
  • Offset: mueve toda la onda hacia arriba o hacia abajo.

Para incluir todo esto en la función:

function oscillator(time, frequency = 1, amplitude = 1, phase = 0, offset = 0){
    var t = time * frequency * Math.PI * 2; // get phase at time
    t += phase * Math.PI * 2; // add the phase offset
    var v = Math.sin(t); // get the value at the calculated position in the cycle
    v *= amplitude; // set the amplitude
    v += offset; // add the offset
    return v;
}

O en una forma más compacta (y un poco más rápida):

function oscillator(time, frequency = 1, amplitude = 1, phase = 0, offset = 0){
    return Math.sin(time * frequency * Math.PI * 2 + phase * Math.PI * 2) * amplitude + offset; 
}

Todos los argumentos excepto el tiempo son opcionales.

Obtener máximos y mínimos #

La función Math.max() devuelve el mayor de cero o más números.

Math.max(4, 12);   //  12
Math.max(-1, -15); // -1

La función Math.min() devuelve el menor de cero o más números.

Math.min(4, 12);   //  4
Math.min(-1, -15); // -15

Obtener el máximo y el mínimo de una matriz: #

var arr = [1, 2, 3, 4, 5, 6, 7, 8, 9],
    max = Math.max.apply(Math, arr),
    min = Math.min.apply(Math, arr);

console.log(max); // Logs: 9
console.log(min); // Logs: 1

ECMAScript 6 operador de propagación, obteniendo el máximo y el mínimo de una matriz:

var arr = [1, 2, 3, 4, 5, 6, 7, 8, 9],
    max = Math.max(...arr),
    min = Math.min(...arr);

console.log(max); // Logs: 9
console.log(min); // Logs: 1

Restringir número al rango mínimo/máximo #

Si necesita fijar un número para mantenerlo dentro de un límite de rango específico

function clamp(min, max, val) {
    return Math.min(Math.max(min, +val), max);
}

console.log(clamp(-10, 10, "4.30")); // 4.3
console.log(clamp(-10, 10, -8));     // -8
console.log(clamp(-10, 10, 12));     // 10
console.log(clamp(-10, 10, -15));    // -10

Sacar raíces de un número #

Raíz cuadrada #

Usa Math.sqrt() para encontrar la raíz cuadrada de un número

Math.sqrt(16)   #=> 4

Raíz cúbica #

Para encontrar la raíz cúbica de un número, use la función Math.cbrt()

Math.cbrt(27)  #=> 3

Encontrar raíces enésimas #

Para encontrar la raíz enésima, use la función Math.pow() y pase un exponente fraccionario.

Math.pow(64, 1/6) #=> 2
