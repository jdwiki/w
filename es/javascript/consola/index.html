Los desarrolladores suelen utilizar la consola de depuración de un navegador o consola web para identificar errores, comprender el flujo de ejecución, registrar datos y para muchos otros fines en tiempo de ejecución. A esta información se accede a través del objeto console.

Sintaxis #

  • void console.log(obj1 [, obj2, …, objN]);
  • void console.log(mensaje [, sub1, …, subN]);

Parámetros #

Parámetro Descripción
obj1 … objN Una lista de objetos de JavaScript cuyas representaciones de cadenas se generan en la consola
mensaje Una cadena de JavaScript que contiene cero o más cadenas de sustitución.
sub1 … subN Objetos de JavaScript con los que reemplazar cadenas de sustitución dentro de msg.

La información que muestra una depuración/consola web está disponible a través de los múltiples métodos del objeto Javascript console que se pueden consultar a través de console.dir(console). Además de la propiedad console.memory, los métodos que se muestran son generalmente los siguientes (tomados de la salida de Chromium):

  • afirmar
  • claro
  • cuenta

  • [final del grupo] 12

Abriendo la Consola #

En la mayoría de los navegadores actuales, la Consola de JavaScript se ha integrado como una pestaña dentro de las Herramientas para desarrolladores. Las teclas de método abreviado que se enumeran a continuación abrirán las Herramientas para desarrolladores; es posible que sea necesario cambiar a la pestaña derecha después de eso.

Cromo #

Abriendo el panel “Consola” de Chrome’s DevTools:

  • Windows/Linux: cualquiera de las siguientes opciones.

    • Ctrl + Shift + J
    • Ctrl + Shift + I, then click on the “Web Console” tab or press ESC to toggle the console on and off
    • F12, then click on the “Console” tab or press ESC to toggle the console on and off

  • Mac OS: Cmd + Optar + J

#Firefox

Abriendo el panel “Consola” en las Herramientas para desarrolladores de Firefox:

  • Windows/Linux: cualquiera de las siguientes opciones.

    • Ctrl + Shift + K
    • Ctrl + Shift + I, then click on the “Web Console” tab or press ESC to toggle the console on and off
    • F12, then click on the “Web Console” tab or press ESC to toggle the console on and off

  • Mac OS: Cmd + Optar + K

Borde e Internet Explorer #

Abrir el panel “Consola” en F12 Developer Instrumentos:

  • F12, luego haga clic en la pestaña “Consola”

#safari

Al abrir el panel “Consola” en el Web Inspector de Safari, primero debe habilitar el menú de desarrollo en las Preferencias de Safari.

Luego puede seleccionar “Desarrollo->Mostrar consola de errores” en los menús o presionar ⌘ + Opción + C

Ópera #

Abriendo la “Consola” en opera:

  • Ctrl + Shift + I, luego haga clic en la pestaña “Consola”

Compatibilidad #

Al usar o emular Internet Explorer 8 o versiones anteriores (p. ej., a través de Vista de compatibilidad/Modo empresarial), la consola solo se definirá cuando las Herramientas para desarrolladores estén activas, por lo que las instrucciones console.log() pueden causar una excepción y evitar que el código se ejecute. Para mitigar esto, puede verificar si la consola está disponible antes de iniciar sesión:

if (typeof window.console !== 'undefined')
{
   console.log("Hello World");
}

O al comienzo de su secuencia de comandos, puede identificar si la consola está disponible y, de lo contrario, definir una función nula para capturar todas sus referencias y evitar excepciones.

if (!window.console)
{ 
    console = {log: function() {}}; 
}

Tenga en cuenta que este segundo ejemplo detendrá todos los registros de la consola incluso si se ha abierto la ventana del desarrollador.

El uso de este segundo ejemplo impedirá el uso de otras funciones como console.dir(obj) a menos que se agregue específicamente.

Medir el tiempo - console.time() #

console.time() se puede usar para medir cuánto tiempo tarda en ejecutarse una tarea en su código.

Llamar a console.time([label]) inicia un nuevo temporizador. Cuando se llama a console.timeEnd([label]), el tiempo transcurrido, en milisegundos, desde que la llamada .time() original se calcula y registra. Debido a este comportamiento, puede llamar a .timeEnd() varias veces con la misma etiqueta para registrar el tiempo transcurrido desde que se realizó la llamada original a .time().

Ejemplo 1:

console.time('response in');

alert('Click to continue');
console.timeEnd('response in');

alert('One more time');
console.timeEnd('response in');

generará:

response in: 774.967ms
response in: 1402.199ms

Ejemplo 2:

var elms = document.getElementsByTagName('*'); //select all elements on the page

console.time('Loop time');

for (var i = 0; i < 5000; i++) {
    for (var j = 0, length = elms.length; j < length; j++) {
        // nothing to do ...
    }
}

console.timeEnd('Loop time');

generará:

Loop time: 40.716ms

Formatear la salida de la consola #

Muchos de los métodos de impresión de la consola también pueden manejar formato de cadenas tipo C, utilizando tokens %:

console.log('%s has %d points', 'Sam', 100);

Muestra Sam tiene 100 puntos.

La lista completa de especificadores de formato en Javascript es:

Especificador Salida
%s Formatea el valor como una cadena
%i o %d Formatea el valor como un entero
%f Formatea el valor como un valor de punto flotante
%o Formatea el valor como un elemento DOM expandible
%O Da formato al valor como un objeto JavaScript expandible
%c Aplica reglas de estilo CSS a la cadena de salida según lo especificado por el segundo parámetro

Estilo avanzado #

Cuando el especificador de formato CSS (%c) se coloca en el lado izquierdo de la cadena, el método de impresión aceptará un segundo parámetro con reglas CSS que permiten un control detallado sobre el formato de esa cadena:

console.log('%cHello world!', 'color: blue; font-size: xx-large');

Pantallas:

Es posible usar múltiples especificadores de formato %c:

  • cualquier subcadena a la derecha de %c tiene un parámetro correspondiente en el método de impresión;

  • este parámetro puede ser una cadena vacía, si no hay necesidad de aplicar reglas CSS a esa misma subcadena;

  • si se encuentran dos especificadores de formato %c, la subcadena 1st (encerrada en %c) y 2nd tendrán sus reglas definidas en 2< sup>nd y 3rd parámetro del método de impresión respectivamente.

  • si se encuentran tres especificadores de formato %c, entonces las subcadenas 1st, 2nd y 3rd tendrán sus reglas definidas en el parámetro 2nd , 3rd y 4th respectivamente, y así sucesivamente…

    console.log("%cHello %cWorld%c!!", // string to be printed “color: blue;”, // applies color formatting to the 1st substring “font-size: xx-large;”, // applies font formatting to the 2nd substring “/* no CSS rule*/” // does not apply any rule to the remaing substring );

Pantallas:

Usar grupos para sangrar la salida #

La salida se puede identificar y encerrar en un grupo contraíble en la consola de depuración con los siguientes métodos:

  • console.groupCollapsed(): crea un grupo colapsado de entradas que se pueden expandir a través del botón de divulgación para revelar todas las entradas realizadas después de invocar este método;
  • console.group(): crea un grupo expandido de entradas que se pueden colapsar para ocultar las entradas después de invocar este método.

La identificación se puede eliminar para entradas posteriores utilizando el siguiente método:

  • console.groupEnd(): sale del grupo actual, lo que permite imprimir entradas más nuevas en el grupo principal después de invocar este método.

Imprimir en la consola de depuración de un navegador #

Se puede usar la consola de depuración de un navegador para imprimir mensajes simples. Esta depuración o [consola web] (https://developer.mozilla.org/en-US/docs/Tools/Web_Console) se puede abrir directamente en el navegador (tecla F12 en la mayoría de los navegadores – consulte Comentarios a continuación para obtener más información) y el método log del objeto console de Javascript se puede invocar escribiendo lo siguiente:

console.log('My message');

Luego, al presionar Enter, aparecerá Mi mensaje en la consola de depuración.

Se puede llamar a console.log() con cualquier cantidad de argumentos y variables disponibles en el alcance actual. Se imprimirán múltiples argumentos en una línea con un pequeño espacio entre ellos.

var obj = { test: 1 };
console.log(['string'], 1, obj, window);

El método log mostrará lo siguiente en la consola de depuración:

['string']  1  Object { test: 1 }  Window { /* truncated */ }

Además de cadenas simples, console.log() puede manejar otros tipos, como arreglos, objetos, fechas, funciones, etc.:

console.log([0, 3, 32, 'a string']);
console.log({ key1: 'value', key2: 'another value'});

Pantallas:

Array [0, 3, 32, 'a string']
Object { key1: 'value', key2: 'another value'}

Los objetos anidados pueden contraerse:

console.log({ key1: 'val', key2: ['one', 'two'], key3: { a: 1, b: 2 } });

Pantallas:

Object { key1: 'val', key2: Array[2], key3: Object }

Ciertos tipos, como los objetos Date y functions, pueden mostrarse de manera diferente:

console.log(new Date(0));
console.log(function test(a, b) { return c; });

Pantallas:

Wed Dec 31 1969 19:00:00 GMT-0500 (Eastern Standard Time)
function test(a, b) { return c; }

Otros métodos de impresión #

Además del método log, los navegadores modernos también admiten métodos similares:

  • console.info – aparece un pequeño icono informativo (ⓘ) en el lado izquierdo de la(s) cadena(s) u objeto(s) impreso(s).

  • console.warn – aparece un pequeño icono de advertencia (!) en el lado izquierdo. En algunos navegadores, el fondo del registro es amarillo.

  • console.error – El ícono de tiempos pequeños (⊗) aparece en el lado izquierdo. En algunos navegadores, el fondo del registro es rojo.

  • console.timeStamp – genera la hora actual y una cadena especificada, pero no es estándar:

    console.timeStamp('msg');

Pantallas:

  00:00:00.001 msg

  • console.trace – genera el seguimiento de la pila actual o muestra el mismo resultado que el método log si se invoca en el ámbito global.

    function sec() {
   first();
}
function first() {
   console.trace();
}
sec();

Pantallas:

  first
  sec
  (anonymous function)

La imagen de arriba muestra todas las funciones, con la excepción de timeStamp, en la versión 56 de Chrome.

Estos métodos se comportan de manera similar al método log y en diferentes consolas de depuración pueden mostrarse en diferentes colores o formatos.

En ciertos depuradores, la información de los objetos individuales se puede ampliar aún más haciendo clic en el texto impreso o en un pequeño triángulo (►) que hace referencia a las propiedades del objeto respectivo. Estas propiedades de objeto colapsadas pueden estar abiertas o cerradas en el registro. Consulte console.dir para obtener información adicional sobre este

2: https://www.wikiod.com/es/javascript/consola#Mostrar objetos y XML de forma interactiva: console.dir(), console.dirxml()

Incluir un seguimiento de la pila al iniciar sesión - console.trace() #

function foo() {
  console.trace('My log statement');
}

foo();

Mostrará esto en la consola:

My log statement       VM696:1
  foo                  @ VM696:1
  (anonymous function) @ (program):1

Nota: Cuando esté disponible, también es útil saber que se puede acceder al mismo seguimiento de pila como una propiedad del objeto Error. Esto puede ser útil para el procesamiento posterior y la recopilación de comentarios automatizados.

var e = new Error('foo');
console.log(e.stack);

Tabulación de valores - console.table() #

En la mayoría de los entornos, console.table() se puede usar para mostrar objetos y matrices en formato tabular.

Por ejemplo:

console.table(['Hello', 'world']);

muestra como:

(índice) valor

Contar - consola.contar() #

console.count([obj]) coloca un contador en el valor del objeto proporcionado como argumento. Cada vez que se invoca este método, el contador aumenta (con la excepción de la cadena vacía ''). Se muestra una etiqueta junto con un número en la consola de depuración según el siguiente formato:

[label]: X

label representa el valor del objeto pasado como argumento y X representa el valor del contador.

El valor de un objeto siempre se considera, incluso si las variables se proporcionan como argumentos:

var o1 = 1, o2 = '2', o3 = "";
console.count(o1);
console.count(o2);
console.count(o3);

console.count(1);
console.count('2');
console.count('');

Pantallas:

1: 1
2: 1
: 1
1: 2
2: 2
: 1

Las cadenas con números se convierten en objetos Número:

console.count(42.3);
console.count(Number('42.3'));
console.count('42.3');

Pantallas:

42.3: 1
42.3: 2
42.3: 3

Las funciones apuntan siempre al objeto Función global:

console.count(console.constructor);
console.count(function(){});
console.count(Object);
var fn1 = function myfn(){};
console.count(fn1);
console.count(Number);

Pantallas:

[object Function]: 1
[object Function]: 2
[object Function]: 3
[object Function]: 4
[object Function]: 5

console.count(null);

Cadena vacía o ausencia de argumento #

Si no se proporciona ningún argumento al ingresar secuencialmente el método de conteo en la consola de depuración, se asume una cadena vacía como parámetro, es decir:

> console.count();
  : 1
> console.count('');
  : 2
> console.count("");
  : 3

Depuración con aserciones - console.assert() #

Escribe un mensaje de error en la consola si la afirmación es falsa. De lo contrario, si la afirmación es verdadera, esto no hace nada.

console.assert('one' === 1);

Se pueden proporcionar múltiples argumentos después de la afirmación, estos pueden ser cadenas u otros objetos, que solo se imprimirán si la afirmación es falsa:

console.assert no arroja un AssertionError (excepto en Node.js), lo que significa que este método es incompatible con la mayoría de los marcos de prueba y que la ejecución del código no se interrumpirá en una afirmación fallida.

Limpiando la consola - console.clear() #

Puede borrar la ventana de la consola utilizando el método console.clear(). Esto elimina todos los mensajes previamente impresos en la consola y puede imprimir un mensaje como “La consola se borró” en algunos entornos.

Mostrar objetos y XML de forma interactiva - console.dir(), console.dirxml() #

console.dir(object) muestra una lista interactiva de las propiedades del objeto JavaScript especificado. La salida se presenta como una lista jerárquica con triángulos desplegables que le permiten ver el contenido de los objetos secundarios.

var myObject = {
    "foo":{
        "bar":"data"
    }
};

console.dir(myObject);

muestra:

console.dirxml(object) imprime una representación XML de los elementos descendientes del objeto si es posible, o la representación de JavaScript si no. Llamar a console.dirxml() en elementos HTML y XML es equivalente a llamar a console.log().

Ejemplo 1:

console.dirxml(document)

Ejemplo 2:

console.log(document)
