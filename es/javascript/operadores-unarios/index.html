Sintaxis #

  • expresión nula; // Evalúa la expresión y descarta el valor de retorno
  • +expresión; // Intento de convertir la expresión en un número
  • eliminar objeto.propiedad; // Borra la propiedad del objeto
  • eliminar objeto[“propiedad”]; // Borra la propiedad del objeto
  • tipo de operando; // Devuelve el tipo de operando
  • ~expresión; // Realiza la operación NOT en cada bit de expresión
  • !expresión; // Realiza la negación lógica en la expresión
  • -expresión; // Negar la expresión después de intentar la conversión a número

El tipo de operador #

El operador typeof devuelve el tipo de datos del operando no evaluado como una cadena.

Sintaxis: #

typeof operand

Devoluciones: #

Estos son los posibles valores de retorno de typeof:

Tipo Valor de retorno
Indefinido "indefinido"
Nulo "objeto"
Booleano "booleano"
Número "número"
Cadena "cadena"
Símbolo (ES6) "símbolo"
Objeto Función "función"
document.all "indefinido"
Objeto host (proporcionado por el entorno JS) Dependiente de la implementación
Cualquier otro objeto "objeto"

El comportamiento inusual de document.all con el operador typeof se debe a su uso anterior para detectar navegadores heredados. Para obtener más información, consulte http://stackoverflow.com/q/40643613/6388243

Ejemplos: #

// returns 'number'
typeof 3.14;
typeof Infinity;
typeof NaN;               // "Not-a-Number" is a "number"

// returns 'string'
typeof "";
typeof "bla";
typeof (typeof 1);        // typeof always returns a string

// returns 'boolean'
typeof true;
typeof false;

// returns 'undefined'
typeof undefined;
typeof declaredButUndefinedVariable;
typeof undeclaredVariable;
typeof void 0;
typeof document.all       // see above

// returns 'function'
typeof function(){};
typeof class C {};
typeof Math.sin;

// returns 'object'
typeof { /*<...>*/ };
typeof null;
typeof /regex/;           // This is also considered an object
typeof [1, 2, 4];         // use Array.isArray or Object.prototype.toString.call.
typeof new Date();
typeof new RegExp();
typeof new Boolean(true); // Don't use!
typeof new Number(1);     // Don't use!
typeof new String("abc"); // Don't use!

// returns 'symbol'
typeof Symbol();
typeof Symbol.iterator;

Visión general #

Los operadores unarios son operadores con un solo operando. Los operadores unarios son más eficientes que las llamadas a funciones estándar de JavaScript. Además, los operadores unarios no se pueden anular y, por lo tanto, su funcionalidad está garantizada.

Están disponibles los siguientes operadores unarios:

Operador Operación Ejemplo
eliminar El operador de eliminación elimina una propiedad de un objeto. ejemplo
void El operador void descarta el valor de retorno de una expresión. [ejemplo][2]
typeof El operador typeof determina el tipo de un objeto dado. [ejemplo][3]
+ El operador unario más convierte su operando al tipo Número. [ejemplo][4]
- El operador de negación unario convierte su operando en número y luego lo niega. [ejemplo][5]
~ Operador NOT bit a bit. [ejemplo][6]
! Operador lógico NOT. [ejemplo][7]

1: https://www.wikiod.com/es/javascript/operadores-unarios#El operador de eliminación [2]: https://www.wikiod.com/es/javascript/operadores-unarios#El operador nulo [3]: https://www.wikiod.com/es/javascript/operadores-unarios#El tipo de operador [4]: https://www.wikiod.com/es/javascript/operadores-unarios#El operador unario más (+) [5]: https://www.wikiod.com/es/javascript/operadores-unarios#El operador de negación unario (-) [6]: https://www.wikiod.com/es/javascript/operadores-unarios#El operador NOT bit a bit (~) [7]: https://www.wikiod.com/es/javascript/operadores-unarios#El operador lógico NOT (!)

El operador de eliminación #

El operador delete elimina una propiedad de un objeto.

Sintaxis: #

delete object.property

delete object['property']

Devoluciones: #

Si la eliminación es exitosa o la propiedad no existía:

  • verdadero

Si la propiedad a eliminar es una propiedad propia no configurable (no se puede eliminar):

  • falso en modo no estricto.
  • Lanza un error en modo estricto

Descripción #

El operador delete no libera memoria directamente. Puede liberar memoria indirectamente si la operación significa que todas las referencias a la propiedad han desaparecido.

delete trabaja en las propiedades de un objeto. Si existe una propiedad con el mismo nombre en la cadena de prototipos del objeto, la propiedad se heredará del prototipo. delete no funciona en variables o nombres de funciones.

Ejemplos: #

// Deleting a property
foo = 1;              // a global variable is a property of `window`: `window.foo`
delete foo;           // true
console.log(foo);     // Uncaught ReferenceError: foo is not defined

// Deleting a variable
var foo = 1;
delete foo;           // false
console.log(foo);     // 1 (Not deleted)

// Deleting a function
function foo(){ };
delete foo;           // false
console.log(foo);     // function foo(){ } (Not deleted)

// Deleting a property
var foo = { bar: "42" };
delete foo.bar;       // true
console.log(foo);     // Object { } (Deleted bar)

// Deleting a property that does not exist
var foo = { };
delete foo.bar;       // true
console.log(foo);     // Object { } (No errors, nothing deleted)

// Deleting a non-configurable property of a predefined object
delete Math.PI;       // false  ()
console.log(Math.PI); // 3.141592653589793 (Not deleted)

El operador unario más (+) #

El más unario (+) precede a su operando y evalúa a su operando. Intenta convertir el operando en un número, si aún no lo ha hecho.

Sintaxis: #

+expression

Devoluciones: #

  • un Número.

Descripción #

El operador unario más (+) es el método más rápido (y preferido) para convertir algo en un número.

Puede convertir:

  • Representaciones de cadenas de números enteros (decimales o hexadecimales) y flotantes.
  • booleanos: verdadero, falso.
  • nulo

Los valores que no se pueden convertir se evaluarán como NaN.

Ejemplos: #

+42           // 42
+"42"         // 42
+true         // 1
+false        // 0
+null         // 0
+undefined    // NaN
+NaN          // NaN
+"foo"        // NaN
+{}           // NaN
+function(){} // NaN

Tenga en cuenta que intentar convertir una matriz puede generar valores de retorno inesperados. En segundo plano, las matrices se convierten primero a sus representaciones de cadena:

[].toString() === '';
[1].toString() === '1';
[1, 2].toString() === '1,2';

Luego, el operador intenta convertir esas cadenas en números:

+[]           // 0   ( === +'' )
+[1]          // 1   ( === +'1' )
+[1, 2]       // NaN ( === +'1,2' )

El operador nulo #

El operador void evalúa la expresión dada y luego devuelve undefined.

Sintaxis: #

void expression

Devoluciones: #

  • indefinido

Descripción #

El operador void se usa a menudo para obtener el valor primitivo indefinido, escribiendo void 0 o void(0). Tenga en cuenta que void es un operador, no una función, por lo que () no es necesario.

Por lo general, el resultado de una expresión vacío e indefinido se pueden usar indistintamente. Sin embargo, en versiones anteriores de ECMAScript, a window.undefined se le podía asignar cualquier valor, y todavía es posible usar undefined como nombre para las variables de parámetros de función dentro de las funciones, interrumpiendo así otro código que se basa en el valor de undefined . Sin embargo, voidsiempre arrojará el valor *verdadero*indefinido`.

void 0 también se usa comúnmente en la minificación de código como una forma más corta de escribir undefined. Además, probablemente sea más seguro ya que algún otro código podría haber manipulado window.undefined.

Ejemplos: #

Devolviendo indefinido:

function foo(){
    return void 0;
}
console.log(foo()); // undefined

Cambiando el valor de undefined dentro de un cierto ámbito:

(function(undefined){
    var str = 'foo';
    console.log(str === undefined); // true
})('foo');

El operador de negación unario (-) #

La negación unaria (-) precede a su operando y lo niega, después de intentar convertirlo en número.

Sintaxis: #

-expression

Devoluciones: #

  • un Número.

Descripción #

La negación unaria (-) puede convertir los mismos tipos/valores que el operador unario más (+).

Los valores que no se pueden convertir se evaluarán como NaN (no hay -NaN).

Ejemplos: #

-42           // -42
-"42"         // -42
-true         // -1
-false        // -0
-null         // -0
-undefined    // NaN
-NaN          // NaN
-"foo"        // NaN
-{}           // NaN
-function(){} // NaN

Tenga en cuenta que intentar convertir una matriz puede generar valores de retorno inesperados. En segundo plano, las matrices se convierten primero a sus representaciones de cadena:

[].toString() === '';
[1].toString() === '1';
[1, 2].toString() === '1,2';

Luego, el operador intenta convertir esas cadenas en números:

-[]           // -0  ( === -'' )
-[1]          // -1  ( === -'1' )
-[1, 2]       // NaN ( === -'1,2' )

El operador NOT bit a bit (~) #

El NOT bit a bit (~) realiza una operación NOT en cada bit en un valor.

Sintaxis: #

~expression

Devoluciones: #

  • un Número.

Descripción #

La tabla de verdad para la operación NOT es:

un NO un
0 1
1 0 
1337  (base 10) = 0000010100111001 (base 2)
~1337 (base 10) = 1111101011000110 (base 2) = -1338 (base 10)

Un no bit a bit en un número da como resultado: -(x + 1).

Ejemplos: #

valor (base 10) valor (base 2) retorno (base 2) retorno (base 10)
2 00000010 11111100 -3
1 00000001 11111110 -2
0 00000000 11111111 -1
-1 11111111 00000000 0
-2 11111110 00000001 1
-3 11111100 00000010 2

El operador lógico NOT (!) #

El operador lógico NOT (!) realiza la negación lógica de una expresión.

Sintaxis: #

!expression

Devoluciones: #

  • un ‘booleano’.

Descripción #

El operador lógico NOT (!) realiza la negación lógica de una expresión.

Los valores booleanos simplemente se invierten: !true === false y !false === true. Los valores no booleanos se convierten primero en valores booleanos y luego se niegan.

Esto significa que se puede usar un NOT lógico doble (!!) para convertir cualquier valor a un valor booleano:

!!"FooBar" === true
!!1 === true
!!0 === false

Estos son todos iguales a !true:

!'true' === !new Boolean('true');
!'false' === !new Boolean('false');
!'FooBar' === !new Boolean('FooBar');
![] === !new Boolean([]);
!{} === !new Boolean({});

Todos estos son iguales a !false:

!0 === !new Boolean(0);
!'' === !new Boolean('');
!NaN === !new Boolean(NaN);
!null === !new Boolean(null);
!undefined === !new Boolean(undefined);

Ejemplos: #

!true         // false
!-1           // false
!"-1"         // false
!42           // false
!"42"         // false
!"foo"        // false
!"true"       // false
!"false"      // false
!{}           // false
![]           // false
!function(){} // false

!false        // true
!null         // true
!undefined    // true
!NaN          // true
!0            // true
!""           // true
