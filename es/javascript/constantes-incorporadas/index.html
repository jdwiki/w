nulo #

null se usa para representar la ausencia intencional de un valor de objeto y es un valor primitivo. A diferencia de undefined, no es una propiedad del objeto global.

Es igual a undefined pero no idéntico a él.

null == undefined; // true
null === undefined; // false

CUIDADO: El tipo de null es 'objeto'.

typeof null; // 'object';

Para comprobar correctamente si un valor es nulo, compárelo con el operador de igualdad estricta

var a = null;

a === null; // true

NaN #

NaN significa “No es un número”. Cuando una función u operación matemática en JavaScript no puede devolver un número específico, en su lugar devuelve el valor NaN.

Es una propiedad del objeto global y una referencia a Number.NaN

window.hasOwnProperty('NaN'); // true
NaN; // NaN

Quizás de manera confusa, NaN todavía se considera un número.

typeof NaN; // 'number'

No busque NaN usando el operador de igualdad. Consulte [isNaN](https://www.wikiod.com/es/javascript/constantes-incorporadas#Probando para NaN usando isNaN()) en su lugar.

NaN == NaN  // false
NaN === NaN // false

indefinido y nulo #

A primera vista, puede parecer que null e undefined son básicamente lo mismo, sin embargo, existen diferencias sutiles pero importantes.

undefined es la ausencia de un valor en el compilador, porque donde debería haber un valor, no se ha puesto uno, como el caso de una variable no asignada.

  • undefined es un valor global que representa la ausencia de un valor asignado.
  • tipo de indefinido === 'indefinido'
  • null es un objeto que indica que a una variable se le ha asignado explícitamente “sin valor”.
  • tipo de nulo === 'objeto'

Establecer una variable en indefinido significa que la variable efectivamente no existe. Algunos procesos, como la serialización JSON, pueden eliminar las propiedades “indefinidas” de los objetos. Por el contrario, las propiedades null indican que se conservarán para que pueda transmitir explícitamente el concepto de una propiedad “vacía”.

Lo siguiente se evalúa como indefinido:

  • Una variable cuando se declara pero no se le asigna un valor (es decir, se define)
  •     let foo;
    console.log('is undefined?', foo === undefined);
    // is undefined? true
    ```
  • Acceder al valor de una propiedad que no existe
  •     let foo = { a: 'a' };
    console.log('is undefined?', foo.b === undefined);
    // is undefined? true
    ```
  • El valor de retorno de una función que no devuelve un valor
  •     function foo() { return; }
    console.log('is undefined?', foo() === undefined);
    // is undefined? true
    ```
  • El valor de un argumento de función que se declara pero se ha omitido de la llamada de función
  •     function foo(param) { 
      console.log('is undefined?', param === undefined);
    }
    foo('a');
    foo();
    // is undefined? false
    // is undefined? true
    ```

undefined también es una propiedad del objeto window global.

// Only in browsers
console.log(window.undefined); // undefined
window.hasOwnProperty('undefined'); // true

Antes de ECMAScript 5, en realidad podía cambiar el valor de la propiedad window.undefined a cualquier otro valor que potencialmente rompiera todo.

Infinito e -Infinito #

1 / 0; // Infinity
// Wait! WHAAAT?

Infinito es una propiedad del objeto global (por lo tanto, una variable global) que representa el infinito matemático. Es una referencia a Number.POSITIVE_INFINITY

Es mayor que cualquier otro valor y puedes obtenerlo dividiendo por 0 o evaluando la expresión de un número que es tan grande que se desborda. En realidad, esto significa que no hay división por 0 errores en JavaScript, ¡hay infinito!

También existe -Infinity, que es un infinito negativo matemático, y es más bajo que cualquier otro valor.

Para obtener -Infinity, niegas Infinity, u obtienes una referencia a él en Number.NEGATIVE_INFINITY.

- (Infinity); // -Infinity

Ahora divirtámonos con ejemplos:

Infinity > 123192310293; // true
-Infinity < -123192310293; // true
1 / 0; // Infinity
Math.pow(123123123, 9123192391023); // Infinity
Number.MAX_VALUE * 2; // Infinity
23 / Infinity; // 0
-Infinity; // -Infinity
-Infinity === Number.NEGATIVE_INFINITY; // true
-0; // -0 , yes there is a negative 0 in the language
0 === -0; // true
1 / -0; // -Infinity
1 / 0 === 1 / -0; // false
Infinity + Infinity; // Infinity

var a = 0, b = -0;

a === b; // true
1 / a === 1 / b; // false

// Try your own!

Constantes numéricas #

El constructor Number tiene algunas constantes incorporadas que pueden ser útiles

Number.MAX_VALUE;          // 1.7976931348623157e+308
Number.MAX_SAFE_INTEGER;   // 9007199254740991

Number.MIN_VALUE;          // 5e-324
Number.MIN_SAFE_INTEGER;   // -9007199254740991

Number.EPSILON;            // 0.0000000000000002220446049250313

Number.POSITIVE_INFINITY;  // Infinity
Number.NEGATIVE_INFINITY;  // -Infinity

Number.NaN;                // NaN

En muchos casos, los distintos operadores de Javascript romperán con valores fuera del rango de (Number.MIN_SAFE_INTEGER, Number.MAX_SAFE_INTEGER)

Tenga en cuenta que ‘Número.EPSILON’ representa la diferencia entre uno y el ‘Número’ más pequeño mayor que uno y, por lo tanto, la diferencia más pequeña posible entre dos valores de ‘Número’ diferentes. Una razón para usar esto se debe a la naturaleza de cómo JavaScript almacena los números. Consulte [Comprobar la igualdad de dos números] (https://www.wikiod.com/es/javascript -de-dos-números#t=201607291343405411584)
