Sintaxis #

  • ‘uso estricto’;
  • “uso estricto”;
  • `uso estricto`;

El modo estricto es una opción agregada en ECMAScript 5 para habilitar algunas mejoras incompatibles con versiones anteriores. Los cambios de comportamiento en el código de “modo estricto” incluyen:

  • La asignación a variables indefinidas genera un error en lugar de definir nuevas variables globales;
  • Asignar o eliminar propiedades que no se pueden escribir (como window.undefined) genera un error en lugar de ejecutarse silenciosamente;
  • La sintaxis octal heredada (por ejemplo, 0777) no es compatible;
  • La declaración with no es compatible;
  • eval no puede crear variables en el ámbito circundante;
  • Las propiedades .caller y .arguments de las funciones no son compatibles;
  • La lista de parámetros de una función no puede tener duplicados;
  • window ya no se usa automáticamente como el valor de this.

NOTA: - El modo ‘estricto’ NO está habilitado de forma predeterminada, ya que si una página usa JavaScript que depende de las características del modo no estricto, entonces ese código se romperá. Por lo tanto, debe ser activado por el propio programador.

Para guiones completos #

El modo estricto se puede aplicar en secuencias de comandos completas colocando la declaración "use strict"; antes de cualquier otra declaración.

"use strict";
// strict mode now applies for the rest of the script

El modo estricto solo está habilitado en scripts en los que define "usar estricto". Puede combinar scripts con y sin modo estricto, porque el estado estricto no se comparte entre diferentes scripts.

Nota: Todo el código escrito dentro de ES2015+ módulos y clases son ​​estrictos de forma predeterminada.

Para funciones #

El modo estricto también se puede aplicar a funciones individuales anteponiendo la instrucción "use strict"; al comienzo de la declaración de la función.

function strict() {
   "use strict";
   // strict mode now applies to the rest of this function
   var innerFunction = function () {
     // strict mode also applies here
   };
}

function notStrict() {
  // but not here
}

El modo estricto también se aplicará a cualquier función de alcance interno.

Cambios en las propiedades #

El modo estricto también evita que elimine propiedades que no se pueden eliminar.

"use strict";
delete Object.prototype; // throws a TypeError

La declaración anterior simplemente se ignoraría si no usa el modo estricto; sin embargo, ahora sabe por qué no se ejecuta como se esperaba.

También le impide extender una propiedad no extensible.

var myObject = {name: "My Name"}
Object.preventExtensions(myObject);

function setAge() {
    myObject.age = 25;   // No errors
}

function setAge() {
    "use strict";
    myObject.age = 25;  // TypeError: can't define property "age": Object is not extensible
}

Cambios en las propiedades globales #

En un ámbito de modo no estricto, cuando se asigna una variable sin inicializar con la palabra clave var, const o let, se declara automáticamente en el ámbito global:

a = 12;
console.log(a); // 12

Sin embargo, en modo estricto, cualquier acceso a una variable no declarada generará un error de referencia:

"use strict";
a = 12; // ReferenceError: a is not defined
console.log(a);

Esto es útil porque JavaScript tiene varios eventos posibles que a veces son inesperados. En el modo no estricto, estos eventos a menudo llevan a los desarrolladores a creer que son errores o comportamientos inesperados, por lo tanto, al habilitar el modo estricto, cualquier error que se produzca les obliga a saber exactamente lo que se está haciendo.

"use strict";
                       // Assuming a global variable mistypedVariable exists
mistypedVaraible = 17; // this line throws a ReferenceError due to the 
                       // misspelling of variable

Este código en modo estricto muestra un escenario posible: arroja un error de referencia que apunta al número de línea de la tarea, lo que permite al desarrollador detectar inmediatamente el error de escritura en el nombre de la variable.

En el modo no estricto, además del hecho de que no se arroja ningún error y la asignación se realiza con éxito, mistypedVaraible se declarará automáticamente en el ámbito global como una variable global. Esto implica que el desarrollador debe buscar manualmente esta asignación específica en el código.

Además, al forzar la declaración de variables, el desarrollador no puede declarar accidentalmente variables globales dentro de funciones. En modo no estricto:

function foo() { 
   a = "bar"; // variable is automatically declared in the global scope
}
foo();
console.log(a); // >> bar

En modo estricto, es necesario declarar explícitamente la variable:

function strict_scope() { 
   "use strict";
   var a = "bar"; // variable is local
}
strict_scope();
console.log(a); // >> "ReferenceError: a is not defined"

La variable también se puede declarar fuera y después de una función, lo que permite su uso, por ejemplo, en el ámbito global:

function strict_scope() { 
   "use strict";
   a = "bar"; // variable is global
}
var a;
strict_scope();
console.log(a); // >> bar

Parámetros duplicados #

El modo estricto no le permite usar nombres de parámetros de función duplicados.

function foo(bar, bar) {}  // No error. bar is set to the final argument when called

"use strict";
function foo(bar, bar) {}; // SyntaxError: duplicate formal argument bar

Alcance de funciones en modo estricto #

En modo estricto, las funciones declaradas en un bloque local son inaccesibles fuera del bloque.

"use strict";
{
  f(); // 'hi'
  function f() {console.log('hi');}
}
f(); // ReferenceError: f is not defined

En cuanto al alcance, las declaraciones de función en modo estricto tienen el mismo tipo de vinculación que let o const.

Comportamiento de la lista de argumentos de una función #

El objeto argumentos se comporta de manera diferente en modo estricto y no estricto. En el modo no estricto, el objeto argumento reflejará los cambios en el valor de los parámetros que están presentes; sin embargo, en el modo estricto, cualquier cambio en el valor del parámetro no se reflejará en el argumento. objeto.

function add(a, b){
    console.log(arguments[0], arguments[1]); // Prints : 1,2

    a = 5, b = 10;

    console.log(arguments[0], arguments[1]); // Prints : 5,10
}

add(1, 2);

Para el código anterior, el objeto argumentos se cambia cuando cambiamos el valor de los parámetros. Sin embargo, para el modo estricto, no se reflejará lo mismo.

function add(a, b) {
    'use strict';

    console.log(arguments[0], arguments[1]); // Prints : 1,2

    a = 5, b = 10;

    console.log(arguments[0], arguments[1]); // Prints : 1,2
}

Vale la pena señalar que, si alguno de los parámetros es indefinido, y tratamos de cambiar el valor del parámetro en modo estricto o no-estricto, el objeto argumentos permanece sin cambios.

Modo estricto

function add(a, b) {
    'use strict';

    console.log(arguments[0], arguments[1]); // undefined,undefined 
                                             // 1,undefined
    a = 5, b = 10;

    console.log(arguments[0], arguments[1]); // undefined,undefined
                                             // 1, undefined
}
add();
// undefined,undefined 
// undefined,undefined

add(1)
// 1, undefined
// 1, undefined

Modo no estricto

function add(a,b) {

    console.log(arguments[0],arguments[1]);

    a = 5, b = 10;

    console.log(arguments[0],arguments[1]);
}
add();
// undefined,undefined 
// undefined,undefined

add(1);
// 1, undefined
// 5, undefined

Listas de parámetros no simples #

function a(x = 5) {
  "use strict";
}

es JavaScript no válido y arrojará un SyntaxError porque no puede usar la directiva "use strict" en una función con una lista de parámetros no simples como la anterior: asignación predeterminada x = 5

Los parámetros no simples incluyen:

  • Asignación por defecto
function a(x = 1) {
  "use strict";
}
  • Desestructuración
function a({ x }) {
  "use strict";
}
  • Parámetros de descanso
function a(...args) {
  "use strict";
}
