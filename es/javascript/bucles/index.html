Sintaxis #

  • para (inicialización; condición; expresión_final) { }
  • para (clave en objeto) { }
  • para (variable de iterable) { }
  • while (condición) { }
  • hacer { } mientras (condición)
  • para cada (variable en objeto) { } // ECMAScript para XML

Los bucles en JavaScript suelen ayudar a resolver problemas que implican repetir un código específico x cantidad de veces. Digamos que necesita registrar un mensaje 5 veces. Podrías hacer esto:

console.log("a message");
console.log("a message");
console.log("a message");
console.log("a message");
console.log("a message");

Pero eso lleva mucho tiempo y es un poco ridículo. Además, ¿qué sucede si necesita registrar más de 300 mensajes? Debe reemplazar el código con un bucle “for” tradicional:

for(var i = 0; i < 5; i++){
    console.log("a message");
}

Bucles “for” estándar #

Uso estándar #

for (var i = 0; i < 100; i++) {
    console.log(i);
}

Rendimiento esperado:

0 1 …
99

Declaraciones múltiples #

Comúnmente utilizado para almacenar en caché la longitud de una matriz.

var array = ['a', 'b', 'c'];
for (var i = 0; i < array.length; i++) {
    console.log(array[i]);
}

Rendimiento esperado:

‘un’ ‘b’ ‘c’

Cambiando el incremento #

for (var i = 0; i < 100; i += 2 /* Can also be: i = i + 2 */) {
    console.log(i);
}

Rendimiento esperado:

0 2 4 …
98

Bucle decrementado #

for (var i = 100; i >=0; i--) {
    console.log(i);
}

Rendimiento esperado:

100 99 98 …
0

Bucle “para… de” #

const iterable = [0, 1, 2];
for (let i of iterable) {
    console.log(i);
}

Rendimiento esperado:

0 1 2

Las ventajas del bucle for…of son:

  • Esta es la sintaxis directa más concisa hasta ahora para recorrer elementos de matriz
  • Evita todos los escollos de for…in
  • A diferencia de forEach(), funciona con break, continue y return

Soporte de for…of en otras colecciones #

Cuerdas #

for…of tratará una cadena como una secuencia de caracteres Unicode:

const string = "abc";
for (let chr of string) {
  console.log(chr);
}

Rendimiento esperado:

un segundo c

Conjuntos #

para… de trabajos en [Objetos establecidos] (https://www.wikiod.com/es/javascript/establecer).

Nota:

  • Un objeto Set eliminará los duplicados.

  • Por favor, [consulte esta referencia] (https://developer.mozilla.org/en/docs/Web/JavaScript/Reference/Global_Objects/Set#Browser_compatibility) para la compatibilidad con el navegador Set().

    const names = [‘bob’, ‘alejandro’, ‘zandra’, ‘anna’, ‘bob’];

    const uniqueNames = new Set(names);

    for (let name of uniqueNames) { console.log(name); }

Rendimiento esperado:

bob alejandro zandra ana

Mapas #

También puede usar bucles for…of para iterar sobre Maps. Esto funciona de manera similar a las matrices y los conjuntos, excepto que la variable de iteración almacena tanto una clave como un valor.

const map = new Map()
  .set('abc', 1)
  .set('def', 2)

for (const iteration of map) {
  console.log(iteration) //will log ['abc', 1] and then ['def', 2]
}

Puede usar [asignación de desestructuración] (https://www.wikiod.com/es/javascript/asignacion-de-desestructuracion#Destrucción de arreglos) para capturar la clave y el valor por separado:

const map = new Map()
  .set('abc', 1)
  .set('def', 2)

for (const [key, value] of map) {
  console.log(key + ' is mapped to ' + value)
}
/*Logs:
  abc is mapped to 1
  def is mapped to 2
*/

Objetos #

for…of bucles no funcionan directamente en Objetos simples; pero es posible iterar sobre las propiedades de un objeto cambiando a un bucle for…in o usando [Object.keys()](https://www.wikiod.com/es/javascript/objetos 736/objeto-claves#t=201608201852433521361):

const someObject = { name: 'Mike' };

for (let key of Object.keys(someObject)) {
  console.log(key + ": " + someObject[key]);
}

Rendimiento esperado:

nombre: Mike

Bucle “para… en” #

Advertencia
for…in está destinado a iterar sobre claves de objeto, no índices de matriz. [Por lo general, no se recomienda usarlo para recorrer una matriz] (http://stackoverflow.com/questions/500504/why-is-using-for-in-with-array-iteration-such-a-bad-idea). También incluye propiedades del prototipo, por lo que puede ser necesario verificar si la clave está dentro del objeto usando hasOwnProperty. Si algún atributo en el objeto está definido por el método defineProperty/defineProperties y establece el parámetro enumerable: false, esos atributos serán inaccesibles.

var object = {"a":"foo", "b":"bar", "c":"baz"};
// `a` is inaccessible
Object.defineProperty(object , 'a', {
        enumerable: false,
});
for (var key in object) {
    if (object.hasOwnProperty(key)) {
      console.log('object.' + key + ', ' + object[key]);
    }
}

Rendimiento esperado:

objeto.b, barra objeto.c, baz

Bucles “mientras” #

Bucle while estándar #

Un ciclo while estándar se ejecutará hasta que la condición dada sea falsa:

var i = 0;
while (i < 100) {
    console.log(i);
    i++;
}

Rendimiento esperado:

0 1 …
99

Bucle decrementado #

var i = 100;
while (i > 0) {
    console.log(i);
    i--; /* equivalent to i=i-1 */
}

Rendimiento esperado:

100 99 98 …
1

Do…mientras Bucle #

Un bucle do…while siempre se ejecutará al menos una vez, independientemente de si la condición es verdadera o falsa:

var i = 101;
do {
    console.log(i);
} while (i < 100);

Rendimiento esperado:

101

“continuar” un ciclo #

Continuación de un bucle “for” #

Cuando coloca la palabra clave continuar en un bucle for, la ejecución salta a la expresión de actualización (i++ en el ejemplo):

for (var i = 0; i < 3; i++) {
    if (i === 1) {
        continue;
    }
    console.log(i);
}

Rendimiento esperado:

0 2

Continuación de un ciclo while #

Cuando continuar en un bucle while, la ejecución salta a la condición (i < 3 en el ejemplo):

var i = 0;
while (i < 3) {
    if (i === 1) {
        i = 2;
        continue;
    }
    console.log(i);
    i++;
}

Rendimiento esperado:

0 2

Bucle “hacer… mientras” #

var availableName;
do {
    availableName = getRandomName();
} while (isNameUsed(name));

Se garantiza que un ciclo do while se ejecutará al menos una vez, ya que su condición solo se verifica al final de una iteración. Un bucle ‘while’ tradicional puede ejecutarse cero o más veces ya que su condición se verifica al comienzo de una iteración.

Romper bucles anidados específicos #

Podemos nombrar nuestros bucles y romper el específico cuando sea necesario.

outerloop:
for (var i = 0;i<3;i++){
    innerloup:
    for (var j = 0;j <3; j++){
        console.log(i);
        console.log(j);
        if (j == 1){
            break outerloop;    
        }
    }
}

Producción:

0
0
0
1

Romper y continuar etiquetas #

Las declaraciones de interrupción y continuación pueden ir seguidas de una etiqueta opcional que funciona como una especie de declaración goto, reanuda la ejecución desde la posición a la que se hace referencia en la etiqueta

for(var i = 0; i < 5; i++){
  nextLoop2Iteration:
  for(var j = 0; j < 5; j++){
    if(i == j) break nextLoop2Iteration;
    console.log(i, j);
  }
}

i=0 j=0 salta el resto de los valores j 1 0 i=1 j=1 salta el resto de los valores j 2 0 2 1 i=2 j=2 salta el resto de los valores j 3 0 3 1 3 2 i=3 j=3 salta el resto de valores j 4 0 4 1 4 2 4 3 i=4 j=4 no se registra y se realizan los bucles

“Salir” de un bucle #

Salir de un ciclo while #

var i = 0;
while(true) {
    i++;
    if(i === 42) {
        break;
    }
}
console.log(i);

Rendimiento esperado:

42

Salir de un bucle for #

var i;
for(i = 0; i < 100; i++) {
    if(i === 42) {
        break;
    }
}
console.log(i);

Rendimiento esperado:

42
