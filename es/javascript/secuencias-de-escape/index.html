Secuencias de escape

  • Introducir caracteres especiales en cadenas y expresiones regulares
  • Tipos de secuencias de escape

En esta página

  • Introducir caracteres especiales en cadenas y expresiones regulares
  • Tipos de secuencias de escape

No todo lo que comienza con una barra invertida es una secuencia de escape. Muchos caracteres simplemente no son útiles para secuencias de escape y simplemente harán que se ignore una barra invertida anterior.

"\H\e\l\l\o" === "Hello" // true

Por otro lado, algunos caracteres como “u” y “x” causarán un error de sintaxis cuando se usan incorrectamente después de una barra invertida. El siguiente no es un literal de cadena válido porque contiene el prefijo de secuencia de escape Unicode \u seguido de un carácter que no es un dígito hexadecimal válido ni una llave:

"C:\Windows\System32\updatehandlers.dll" // SyntaxError

Una barra invertida al final de una línea dentro de una cadena no introduce una secuencia de escape, pero indica la continuación de la línea, es decir

"contin\
uation" === "continuation" // true

Similitud con otros formatos #

Si bien las secuencias de escape en JavaScript se parecen a otros lenguajes y formatos, como C ++, Java, JSON, etc., a menudo habrá diferencias críticas en los detalles. En caso de duda, asegúrese de probar que su código se comporta como se espera y considere verificar la especificación del lenguaje.

Introducir caracteres especiales en cadenas y expresiones regulares #

La mayoría de los caracteres imprimibles se pueden incluir en cadenas o literales de expresiones regulares tal como son, p.

var str = "ポケモン"; // a valid string
var regExp = /[Α-Ωα-ω]/; // matches any Greek letter without diacritics

Para agregar caracteres arbitrarios a una cadena o expresión regular, incluidos los no imprimibles, se deben usar secuencias de escape. Las secuencias de escape consisten en una barra invertida ("\") seguida de uno o más caracteres. Para escribir una secuencia de escape para un carácter en particular, normalmente (pero no siempre) se necesita conocer su [código de carácter] hexadecimal (https://www.wikiod.com/es/javascript/instrumentos-de-cuerda#Código de caracteres =201608112028595843342).

JavaScript proporciona varias formas diferentes de especificar secuencias de escape, como se documenta en los ejemplos de este tema. Por ejemplo, las siguientes secuencias de escape denotan todas el mismo carácter: el avance de línea (carácter de nueva línea de Unix), con el código de carácter U+000A.

  • \n
  • \x0a
  • \u000a
  • \u{a} nuevo en ES6, solo en cadenas
  • \012 prohibido en cadenas literales en modo estricto y en cadenas de plantillas
  • \cj solo en expresiones regulares

Tipos de secuencias de escape #

Secuencias de escape de un solo carácter #

Algunas secuencias de escape constan de una barra invertida seguida de un solo carácter.

Por ejemplo, en alert("Hola\nMundo");, la secuencia de escape \n se utiliza para introducir una nueva línea en el parámetro de cadena, de modo que las palabras “Hola” y “Mundo” se muestren en líneas consecutivas. .

Secuencia de escape Personaje Unicódigo
\b (solo en cadenas, no en expresiones regulares) retroceso U+0008
\t pestaña horizontal U+0009
\n avance de línea U+000A
\v pestaña vertical U+000B
\f alimentación de formulario U+000C
\r retorno de carro U+000D

Además, la secuencia \0, cuando no va seguida de un dígito entre 0 y 7, se puede usar para escapar del carácter nulo (U+0000).

Las secuencias \\, \' y \" se utilizan para escapar del carácter que sigue a la barra invertida. Si bien son similares a las secuencias que no son de escape, donde la barra invertida inicial simplemente se ignora (es decir, \? para ?), se tratan explícitamente como secuencias de escape de un solo carácter dentro de cadenas según la especificación.

Secuencias de escape hexadecimales #

Los caracteres con códigos entre 0 y 255 se pueden representar con una secuencia de escape donde \x va seguido del código de carácter hexadecimal de 2 dígitos. Por ejemplo, el carácter de espacio de no separación tiene el código 160 o A0 en base 16, por lo que se puede escribir como \xa0.

var str = "ONE\xa0LINE"; // ONE and LINE with a non-breaking space between them

Para dígitos hexadecimales superiores a 9, se utilizan las letras a a f, en minúsculas o mayúsculas sin distinción.

var regExp1 = /[\x00-xff]/; // matches any character between U+0000 and U+00FF
var regExp2 = /[\x00-xFF]/; // same as above

Secuencias de escape Unicode de 4 dígitos #

Los caracteres con códigos entre 0 y 65535 (216 - 1) se pueden representar con una secuencia de escape donde \u va seguido del código de carácter hexadecimal de 4 dígitos.

Por ejemplo, el estándar Unicode define el carácter de flecha hacia la derecha (“→”) con el número 8594 o 2192 en formato hexadecimal. Así que una secuencia de escape sería \u2192.

Esto produce la cadena “A → B”:

var str = "A \u2192 B";

Para dígitos hexadecimales superiores a 9, se utilizan las letras a a f, en minúsculas o mayúsculas sin distinción. Los códigos hexadecimales de menos de 4 dígitos deben rellenarse con ceros a la izquierda: \u007A para la letra minúscula “z”.

Secuencias de escape Unicode con corchetes #

ES6 amplía la compatibilidad con Unicode al rango de código completo de 0 a 0x10FFFF. Para escapar de los caracteres con un código mayor que 216 - 1, se introdujo una nueva sintaxis para las secuencias de escape:

\u{???}

Donde el código entre llaves es una representación hexadecimal del valor del punto de código, p.

alert("Look! \u{1f440}"); // Look! 👀

En el ejemplo anterior, el código 1f440 es la representación hexadecimal del código de carácter del carácter Unicode Ojos.

Tenga en cuenta que el código entre llaves puede contener cualquier número de dígitos hexadecimales, siempre que el valor no supere 0x10FFFF. Para dígitos hexadecimales superiores a 9, se utilizan las letras a a f, en minúsculas o mayúsculas sin distinción.

Las secuencias de escape Unicode con llaves solo funcionan dentro de cadenas, ¡no dentro de expresiones regulares!

Secuencias de escape octales #

Las secuencias de escape octales están obsoletas a partir de ES5, pero aún se admiten dentro de las expresiones regulares y, en modo no estricto, también dentro de las cadenas que no son de plantilla. Una secuencia de escape octal consta de uno, dos o tres dígitos octales, con un valor entre 0 y 3778 = 255.

Por ejemplo, la letra “E” mayúscula tiene el código de carácter 69 o 105 en base 8. Por lo tanto, se puede representar con la secuencia de escape \105:

/\105scape/.test("Fun with Escape Sequences"); // true

En modo estricto, las secuencias de escape octales no están permitidas dentro de las cadenas y producirán un error de sintaxis. Vale la pena señalar que \0, a diferencia de \00 o \000, no se considera una secuencia de escape octal y, por lo tanto, aún se permite dentro de cadenas (incluso cadenas de plantilla) en modo estricto.

Controlar secuencias de escape #

Algunas secuencias de escape solo se reconocen dentro de expresiones regulares literales (no en cadenas). Estos pueden usarse para escapar caracteres con códigos entre 1 y 26 (U+0001–U+001A). Consisten en una sola letra de la A a la Z (las mayúsculas y las minúsculas no hacen ninguna diferencia) precedida por \c. La posición alfabética de la letra después de \c determina el código de carácter.

Por ejemplo, en la expresión regular

`/\cG/`

La letra “G” (la séptima letra del alfabeto) se refiere al carácter U+0007 y, por lo tanto,

`/\cG`/.test(String.fromCharCode(7)); // true
