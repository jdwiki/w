Sintaxis #

  • “literal de cadena”
  • ’literal de cadena'
  • “literal de cadena con ‘comillas que no coinciden’” // sin errores; las citas son diferentes.
  • “string literal con "comillas escapadas"” // sin errores; las comillas se escapan.
  • `cadena de plantilla ${expresión}`
  • String(“a b c”) // devuelve una cadena cuando se llama en un contexto que no es de constructor
  • new String(“a b c”) // el objeto String, no la cadena primitiva

Información básica y concatenación de cadenas #

Las cadenas en JavaScript se pueden encerrar entre comillas simples 'hola', comillas dobles "Hola" y (desde ES2015, ES6) en literales de plantilla (comillas invertidas) `hola`.

var hello  = "Hello";
var world  = 'world';
var helloW = `Hello World`;              // ES2015 / ES6

Las cadenas se pueden crear a partir de otros tipos utilizando la función String().

var intString = String(32); // "32"
var booleanString = String(true); // "true"
var nullString = String(null); // "null"

O bien, se puede usar toString() para convertir números, booleanos u objetos en cadenas.

var intString = (5232).toString(); // "5232"
var booleanString = (false).toString(); // "false"
var objString = ({}).toString(); // "[object Object]"

Las cadenas también se pueden crear utilizando el método String.fromCharCode.

String.fromCharCode(104,101,108,108,111) //"hello"

Se permite crear un objeto String usando la palabra clave new, pero no se recomienda ya que se comporta como objetos a diferencia de las cadenas primitivas.

var objectString = new String("Yes, I am a String object");
typeof objectString;//"object"
typeof objectString.valueOf();//"string"

Cadenas de concatenación #

La concatenación de cadenas se puede realizar con el operador de concatenación + o con el método integrado concat() en el prototipo del objeto String.

var foo = "Foo";
var bar = "Bar";
console.log(foo + bar);        // => "FooBar"
console.log(foo + " " + bar);  // => "Foo Bar"

foo.concat(bar)            // => "FooBar"
"a".concat("b", " ", "d")  // => "ab d"

Las cadenas se pueden concatenar con variables que no son cadenas, pero convertirán las variables que no son cadenas en cadenas.

var string = "string";
var number = 32;
var boolean = true;

console.log(string + number + boolean); // "string32true"

Plantillas de cadenas #

Las cadenas se pueden crear usando literales de plantilla (comillas invertidas) `hello`.

var greeting = `Hello`;

Con los literales de plantilla, puede interpolar cadenas usando ${variable} dentro de los literales de plantilla:

var place = `World`;
var greet = `Hello ${place}!`

console.log(greet); // "Hello World!"

Puede usar String.raw para obtener barras invertidas en la cadena sin modificaciones.

`a\\b` // =  a\b
String.raw`a\\b` // = a\\b

Cuerda inversa #

La forma más “popular” de invertir una cadena en JavaScript es el siguiente fragmento de código, que es bastante común:

function reverseString(str) {
    return str.split('').reverse().join('');
}

reverseString('string');    // "gnirts"

Sin embargo, esto funcionará siempre y cuando la cadena que se invierta no contenga pares sustitutos. Los símbolos astrales, es decir, los caracteres fuera del plano multilingüe básico, pueden estar representados por dos unidades de código y conducirán a esta técnica ingenua a producir resultados erróneos. Además, los caracteres con marcas de combinación (por ejemplo, diéresis) aparecerán en el carácter lógico “siguiente” en lugar del original con el que se combinó.

'𝌆■.'.split('').reverse().join(''); //fails

Si bien el método funcionará bien para la mayoría de los idiomas, un algoritmo verdaderamente preciso que respete la codificación para la inversión de cadenas es un poco más complicado. Una de esas implementaciones es una pequeña biblioteca llamada Esrever, que usa expresiones regulares para hacer coincidir marcas de combinación y pares sustitutos para realizar la inversión a la perfección.

Explicación #

Sección Explicación Resultado
str La cadena de entrada "cadena"
String.prototype.split( delimitador ) Divide la cadena str en una matriz. El parámetro "" significa dividir entre cada carácter. ["s","t","r","i","n","g"]
Array.prototype.reverse() Devuelve la matriz de la cadena dividida con sus elementos en orden inverso. ["g","n","i","r","t","s"]
Array.prototype.join( delimitador ) Une los elementos de la matriz en una cadena. El parámetro "" significa un delimitador vacío (es decir, los elementos de la matriz se colocan uno al lado del otro). "chispas"

Usando el operador de propagación

function reverseString(str) {
    return [...String(str)].reverse().join('');    
}

console.log(reverseString('stackoverflow'));  // "wolfrevokcats"
console.log(reverseString(1337));             // "7331"
console.log(reverseString([1, 2, 3]));        // "3,2,1"

Función reverse() personalizada

function reverse(string) {
    var strRev = "";
    for (var i = string.length - 1; i >= 0; i--) {
        strRev += string[i];
    }
    return strRev; 
}

reverse("zebra");  // "arbez"

Comparando cadenas lexicográficamente #

Para comparar cadenas alfabéticamente, use localeCompare(). Esto devuelve un valor negativo si la cadena de referencia está lexicográficamente (alfabéticamente) antes de la cadena comparada (el parámetro), un valor positivo si viene después y un valor de 0 si son iguales.

var a = "hello";
var b = "world";

console.log(a.localeCompare(b)); // -1

Los operadores > y < también se pueden usar para comparar cadenas lexicográficamente, pero no pueden devolver un valor de cero (esto se puede probar con el operador de igualdad ==). Como resultado, una forma de la función localeCompare() se puede escribir así:

function strcmp(a, b) {
    if(a === b) {
        return 0;
    }

    if (a > b) {
        return 1;
    }

    return -1;
}

console.log(strcmp("hello", "world")); // -1
console.log(strcmp("hello", "hello")); //  0
console.log(strcmp("world", "hello")); //  1

Esto es especialmente útil cuando se utiliza una función de clasificación que compara según el signo del valor devuelto (como sort).

var arr = ["bananas", "cranberries", "apples"];
arr.sort(function(a, b) {
    return a.localeCompare(b);
});
console.log(arr); // [ "apples", "bananas", "cranberries" ]

Acceso al carácter en el índice de la cadena #

Use charAt() para obtener un carácter en el índice especificado en la cadena.

var string = "Hello, World!";
console.log( string.charAt(4) ); // "o"

Como alternativa, debido a que las cadenas se pueden tratar como matrices, use el índice a través de [notación de corchetes] (https://developer.mozilla.org/en-US/docs/Web/JavaScript/Reference/Operators/Property_Accessors).

var string = "Hello, World!";
console.log( string[4] ); // "o"

Para obtener el código de carácter del carácter en un índice específico, use charCodeAt() .

var string = "Hello, World!";
console.log( string.charCodeAt(4) ); // 111

Tenga en cuenta que estos métodos son todos métodos getter (devuelven un valor). Las cadenas en JavaScript son inmutables. En otras palabras, ninguno de ellos se puede usar para establecer un carácter en una posición en la cadena.

Comillas de escape #

Si su cadena está encerrada (es decir,) entre comillas simples, debe escapar de la comilla literal interna con barra invertida \

var text = 'L\'albero means tree in Italian';
console.log( text ); \\ "L'albero means tree in Italian"

Lo mismo ocurre con las comillas dobles:

var text = "I feel \"high\"";

Se debe prestar especial atención a las comillas de escape si está almacenando representaciones HTML dentro de una cadena, ya que las cadenas HTML hacen un gran uso de las comillas, es decir, en los atributos:

var content = "<p class=\"special\">Hello World!</p>";        // valid String
var hello   = '<p class="special">I\'d like to say "Hi"</p>'; // valid String

Las comillas en cadenas HTML también se pueden representar usando &apos; (o &#39;) como comillas simples y &quot; (o &#34;) como comillas dobles.

var hi    = "<p class='special'>I'd like to say &quot;Hi&quot;</p>"; // valid String
var hello = '<p class="special">I&apos;d like to say "Hi"</p>';      // valid String

Nota: El uso de &apos; y &quot; no sobrescribirá las comillas dobles que los navegadores pueden colocar automáticamente en las comillas de atributos. Por ejemplo, <p class=special> se convierte en <p class="special">, usando &quot; puede llevar a <p class=""special""> donde \" ser <p class="special">.

Si una cadena tiene ' y ", es posible que desee considerar el uso de literales de plantilla (también conocidos como cadenas de plantilla en ediciones anteriores de ES6), que no requieren que escape ' y ". Estos usan acentos graves (`) en lugar de comillas simples o dobles.

var x = `"Escaping " and ' can become very annoying`;

Recortar espacios en blanco #

Para recortar los espacios en blanco de los bordes de una cadena, use String.prototype.trim:

"    some whitespaced string  ".trim();  // "some whitespaced string"

Muchos motores de JavaScript, pero [no Internet Explorer] (https://developer.mozilla.org/en-US/docs/Web/JavaScript/Reference/Global_Objects/String/TrimLeft#Browser_compatibility), han implementado trimLeft no estándar. y métodos trimRight. Hay una propuesta, actualmente en la Etapa 1 del proceso, para métodos estandarizados trimStart y trimEnd, con alias trimLeft y trimRight para compatibilidad.

// Stage 1 proposal
"    this is me    ".trimStart();  // "this is me    "
"    this is me    ".trimEnd();  // "    this is me"

// Non-standard methods, but currently implemented by most engines
"    this is me    ".trimLeft();  // "this is me    "
"    this is me    ".trimRight();  // "    this is me"

Dividir una cadena en una matriz #

Use .split para pasar de cadenas a una matriz de subcadenas divididas:

var s = "one, two, three, four, five"
s.split(", ");  // ["one", "two", "three", "four", "five"]

Usa el método de matriz .join para volver a una cadena:

s.split(", ").join("--");  // "one--two--three--four--five"

** ¡Todas las cadenas de JavaScript son unicode! **

No hay bytes sin procesar o cadenas binarias en JavaScript. Para manejar de manera efectiva los datos binarios, use [Matrices con tipo](https://www.wikiod.com/es/javascript/datos-binarios#Usando TypedArrays).

Detectando una cadena #

Para detectar si un parámetro es una cadena primitiva, use typeof:

var aString = "my string";
var anInt = 5;
var anObj = {};
typeof aString === "string";   // true
typeof anInt === "string";     // false
typeof anObj === "string";     // false

Si alguna vez tiene un objeto String, a través de new String("somestr"), entonces lo anterior no funcionará. En este caso, podemos usar instanceof:

var aStringObj = new String("my string");
aStringObj instanceof String;    // true

Para cubrir ambas instancias, podemos escribir una función auxiliar simple:

var isString = function(value) {
    return typeof value === "string" || value instanceof String;
};

var aString = "Primitive String";
var aStringObj = new String("String Object");
isString(aString); // true
isString(aStringObj); // true
isString({}); // false
isString(5); // false

O podemos hacer uso de la función toString de Object. Esto puede ser útil si también tenemos que verificar otros tipos, por ejemplo, en una declaración de cambio, ya que este método también admite otros tipos de datos, como typeof.

var pString = "Primitive String";
var oString = new String("Object Form of String");
Object.prototype.toString.call(pString);//"[object String]"
Object.prototype.toString.call(oString);//"[object String]"

Una solución más robusta es no detectar una cadena en absoluto, sino solo verificar qué funcionalidad se requiere. Por ejemplo:

var aString = "Primitive String";
// Generic check for a substring method
if(aString.substring) {

}
// Explicit check for the String substring prototype method
if(aString.substring === String.prototype.substring) {
    aString.substring(0, );
}

Use .slice() para extraer subcadenas dados dos índices:

var s = "0123456789abcdefg";

Funciones de búsqueda y reemplazo de cadenas #

Para buscar una cadena dentro de una cadena, hay varias funciones:

indexOf(cadena de búsqueda) y [lastIndexOf(cadena de búsqueda)](https ://developer.mozilla.org/en-US/docs/Web/JavaScript/Reference/Global_Objects/String/lastIndexOf) #

indexOf() devolverá el índice de la primera aparición de searchString en la cadena. Si no se encuentra searchString, se devuelve -1.

var string = "Hello, World!";
console.log( string.indexOf("o") ); // 4
console.log( string.indexOf("foo") ); // -1

De manera similar, lastIndexOf() devolverá el índice de la última aparición de searchstring o -1 si no se encuentra.

var string = "Hello, World!";
console.log( string.lastIndexOf("o") );   // 8
console.log( string.lastIndexOf("foo") ); // -1

incluye(cadena de búsqueda, inicio) #

includes() devolverá un valor booleano que indica si searchString existe en la cadena, a partir del índice start (predeterminado en 0). Esto es mejor que indexOf() si simplemente necesita probar la existencia de una subcadena.

var string = "Hello, World!";
console.log( string.includes("Hello") ); // true
console.log( string.includes("foo") );   // false

replace(regexp|subcadena, reemplazo|replaceFunction) #

replace() devolverá una cadena que tiene todas las apariciones de subcadenas que coinciden con RegExp regexp o cadena substring con una cadena replacement o el valor devuelto de replaceFunction.

Tenga en cuenta que esto no modifica la cadena en su lugar, pero devuelve la cadena con reemplazos.

var string = "Hello, World!";
string = string.replace( "Hello", "Bye" );
console.log( string ); // "Bye, World!"

string = string.replace( /W.{3}d/g, "Universe" );
console.log( string ); // "Bye, Universe!"

replaceFunction se puede usar para reemplazos condicionales de objetos de expresión regular (es decir, con uso con regexp). Los parámetros están en el siguiente orden:

Parámetro significado
coincidencia la subcadena que coincide con toda la expresión regularg
g1, g2, g3, … los grupos coincidentes en la expresión regular
compensación el desplazamiento de la coincidencia en toda la cadena
cadena toda la cadena

Tenga en cuenta que todos los parámetros son opcionales.

var string = "heLlo, woRlD!";
string = string.replace( /([a-zA-Z])([a-zA-Z]+)/g, function(match, g1, g2) {
    return g1.toUpperCase() + g2.toLowerCase();
}); 
console.log( string ); // "Hello, World!"

Representaciones de cadenas de números #

JavaScript tiene una conversión nativa de Número a su Representación de cadena para cualquier base de 2 a 36.

La representación más común después de decimal (base 10) es hexadecimal (base 16), pero el contenido de esta sección funciona para todas las bases del rango.

Para convertir un Número de decimal (base 10) a hexadecimal (base 16) Representación de cadena se puede usar el método toString con radix 16.

// base 10 Number
var b10 = 12;

// base 16 String representation
var b16 = b10.toString(16); // "c"

Si el número representado es un número entero, la operación inversa para esto se puede hacer con parseInt y el radix 16 de nuevo

// base 16 String representation
var b16 = 'c';

// base 10 Number
var b10 = parseInt(b16, 16); // 12

Para convertir un número arbitrario (es decir, no entero) de su representación de cadena en un número, la operación debe dividirse en dos partes; la parte entera y la parte fraccionaria.

let b16 = '3.243f3e0370cdc';
// Split into integer and fraction parts
let [i16, f16] = b16.split('.');

// Calculate base 10 integer part
let i10 = parseInt(i16, 16); // 3

// Calculate the base 10 fraction part
let f10 = parseInt(f16, 16) / Math.pow(16, f16.length); // 0.14158999999999988

// Put the base 10 parts together to find the Number
let b10 = i10 + f10; // 3.14159

Nota 1: Tenga cuidado ya que pueden haber pequeños errores en el resultado debido a las diferencias en lo que se puede representar en diferentes bases. Puede ser deseable realizar algún tipo de redondeo después. Nota 2: Las representaciones muy largas de números también pueden generar errores debido a la precisión y los valores máximos de Números del entorno en el que se realizan las conversiones.

Código de carácter #

El método charCodeAt recupera el código de carácter Unicode de un solo carácter:

var charCode = "µ".charCodeAt(); // The character code of the letter µ is 181

Para obtener el código de carácter de un carácter en una cadena, la posición basada en 0 del carácter se pasa como parámetro a charCodeAt:

var charCode = "ABCDE".charCodeAt(3); // The character code of "D" is 68

Algunos símbolos Unicode no caben en un solo carácter y, en su lugar, requieren dos pares sustitutos UTF-16 para codificar. Este es el caso de los códigos de caracteres más allá de 216 - 1 o 63553. Estos códigos de caracteres extendidos o valores de punto de código se pueden recuperar con codePointAt:

// The Grinning Face Emoji has code point 128512 or 0x1F600
var codePoint = "😀".codePointAt();

Encuentra el índice de una subcadena dentro de una cadena #

El método .indexOf devuelve el índice de una subcadena dentro de otra cadena (si existe, o -1 en caso contrario)

'Hellow World'.indexOf('Wor');    // 7

.indexOf también acepta un argumento numérico adicional que indica en qué índice debe comenzar a buscar la función

"harr dee harr dee harr".indexOf("dee", 10); // 14

Debe tener en cuenta que .indexOf distingue entre mayúsculas y minúsculas

 'Hellow World'.indexOf('WOR');    // -1

Repetir una cadena #

Esto se puede hacer usando el método .repeat():

"abc".repeat(3);  // Returns "abcabcabc"
"abc".repeat(0);  // Returns ""
"abc".repeat(-1); // Throws a RangeError

En el caso general, esto debe hacerse usando un polyfill correcto para el método ES6 String.prototype.repeat(). De lo contrario, el modismo new Array(n + 1).join(myString) puede repetir n veces la cadena myString:

var myString = "abc";
var n = 3;

new Array(n + 1).join(myString);  // Returns "abcabcabc"
