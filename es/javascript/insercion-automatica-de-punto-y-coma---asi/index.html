Evite la inserción de punto y coma en declaraciones de retorno #

La convención de codificación de JavaScript es colocar el paréntesis inicial de los bloques en la misma línea de su declaración:

if (...) {

}

function (a, b, ...) {

}

En lugar de en la siguiente línea:

if (...)
{

}

function (a, b, ...) 
{

}

Esto se ha adoptado para evitar la inserción de punto y coma en declaraciones de retorno que devuelven objetos:

function foo() 
{
    return // A semicolon will be inserted here, making the function return nothing
    {
        foo: 'foo'
    };
}

foo(); // undefined

function properFoo() {
    return {
        foo: 'foo'
    };
}

properFoo(); // { foo: 'foo' }

En la mayoría de los idiomas, la ubicación del paréntesis inicial es solo una cuestión de preferencia personal, ya que no tiene un impacto real en la ejecución del código. En JavaScript, como ha visto, colocar el corchete inicial en la siguiente línea puede generar errores silenciosos.

Reglas de inserción automática de punto y coma #

Hay tres reglas básicas de inserción de punto y coma:

  1. Cuando, mientras se analiza el programa de izquierda a derecha, se encuentra un token (llamado * token ofensivo *) que no está permitido por ninguna producción de la gramática, entonces se inserta automáticamente un punto y coma antes del token ofensivo si uno o más de las siguientes condiciones es verdadera:
  • El token infractor está separado del token anterior por al menos un LineTerminator.
  • El token infractor es }.
  1. Cuando, a medida que el programa se analiza de izquierda a derecha, se encuentra el final del flujo de tokens de entrada y el analizador no puede analizar el flujo de tokens de entrada como un solo “Programa” completo de ECMAScript, entonces se coloca automáticamente un punto y coma. insertado al final del flujo de entrada.
  2. Cuando, al analizar el programa de izquierda a derecha, se encuentra un token que está permitido por alguna producción de la gramática, pero la producción es una producción restringida y el token sería el primer token para una terminal o no terminal inmediatamente después de la anotación “[no LineTerminator aquí]” dentro de la producción restringida (y, por lo tanto, dicho token se denomina token restringido), y el token restringido se separa del token anterior por al menos menos un LineTerminator, luego se inserta automáticamente un punto y coma antes del token restringido.

Sin embargo, hay una condición anulatoria adicional en las reglas anteriores: un punto y coma nunca se inserta automáticamente si el punto y coma se analizaría como una declaración vacía o si ese punto y coma se convertiría en uno de los dos puntos y coma en el encabezado de un para sentencia (ver 12.6.3).

Fuente: ECMA-262, especificación ECMAScript de quinta edición:

Declaraciones afectadas por la inserción automática de punto y coma #

  • declaración vacía
  • declaración ’era'
  • declaración de expresión
  • declaración hacer mientras
  • declaración continuar
  • declaración romper
  • declaración de retorno
  • declaración lanzar

Ejemplos:

Cuando se encuentra el final del flujo de tokens de entrada y el analizador no puede analizar el flujo de tokens de entrada como un único programa completo, se inserta automáticamente un punto y coma al final del flujo de entrada.

a = b
++c
// is transformed to:
a = b;
++c;

x
++
y
// is transformed to:
x;
++y;

Indización de matrices/literales

console.log("Hello, World")
[1,2,3].join()
// is transformed to:
console.log("Hello, World")[(1, 2, 3)].join();

** Declaración de devolución: **

return 
  "something";
// is transformed to
return;
  "something";
