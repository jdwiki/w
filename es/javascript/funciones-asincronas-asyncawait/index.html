async y await se basan en promesas y generadores para expresar acciones asíncronas en línea. Esto hace que el código asincrónico o de devolución de llamada sea mucho más fácil de mantener.

Las funciones con la palabra clave async devuelven una Promesa y se pueden llamar con esa sintaxis.

Dentro de una ‘función asíncrona’, la palabra clave ’esperar’ se puede aplicar a cualquier ‘Promesa’, y hará que todo el cuerpo de la función después de ’esperar’ se ejecute después de que se resuelva la promesa.

Sintaxis #

  • función asíncrona foo() {    …
       await asyncCall()
    }
  • función asíncrona() { … }
  • asíncrono() => { … }
  • (asincrónico () => {    const data = await asyncCall()
       console.log(data) })()

Las funciones asíncronas son un azúcar sintáctico sobre promesas y generadores. Lo ayudan a hacer que su código sea más legible, fácil de mantener, más fácil de detectar errores y con menos niveles de sangría.

Introducción #

Una función definida como asincrónica es una función que puede realizar acciones asincrónicas pero aún así parecer sincrónica. La forma en que se hace es usando la palabra clave await para diferir la función mientras espera que se resuelva o rechace una Promesa.

Nota: Las funciones asíncronas son una propuesta de Etapa 4 (“Terminada”) en vías de ser incluidas en el Estándar ECMAScript 2017.

Por ejemplo, usando la [Fetch API] basada en promesas (https://developer.mozilla.org/en/docs/Web/API/Fetch_API):

async function getJSON(url) {
    try {
        const response = await fetch(url);
        return await response.json();
    }
    catch (err) {
        // Rejections in the promise will get thrown here
        console.error(err.message);
    }
}

Una función asíncrona siempre devuelve una Promesa, por lo que puede usarla en otras funciones asíncronas.

Estilo de función de flecha #

const getJSON = async url => {
    const response = await fetch(url);
    return await response.json();
}

Espera y precedencia del operador #

Debe tener en cuenta la precedencia del operador al usar la palabra clave await.

Imagine que tenemos una función asíncrona que llama a otra función asíncrona, getUnicorn(), que devuelve una Promesa que se resuelve en una instancia de la clase Unicornio. Ahora queremos obtener el tamaño del unicornio usando el método getSize() de esa clase.

Mira el siguiente código:

async function myAsyncFunction() {
    await getUnicorn().getSize();
}

A primera vista parece válido, pero no lo es. Debido a la precedencia del operador, es equivalente a lo siguiente:

async function myAsyncFunction() {
    await (getUnicorn().getSize());
}

Aquí intentamos llamar al método getSize() del objeto Promise, que no es lo que queremos.

En su lugar, debemos usar corchetes para indicar que primero queremos esperar al unicornio y luego llamar al método getSize() del resultado:

async function asyncFunction() {
    (await getUnicorn()).getSize();
}

Por supuesto. la versión anterior podría ser válida en algunos casos, por ejemplo, si la función getUnicorn() era sincrónica, pero el método getSize() era asincrónico.

Funciones asíncronas en comparación con Promises #

Las funciones async no reemplazan el tipo Promise; agregan palabras clave de idioma que hacen que las promesas sean más fáciles de cumplir. Son intercambiables:

async function doAsyncThing() { ... }

function doPromiseThing(input) { return new Promise((r, x) => ...); }

// Call with promise syntax
doAsyncThing()
    .then(a => doPromiseThing(a))
    .then(b => ...)
    .catch(ex => ...);

// Call with await syntax
try {
    const a = await doAsyncThing();
    const b = await doPromiseThing(a);
    ...
}
catch(ex) { ... }

Cualquier función que use cadenas de promesas se puede reescribir usando await:

function newUnicorn() {
  return fetch('unicorn.json')                     // fetch unicorn.json from server
  .then(responseCurrent => responseCurrent.json()) // parse the response as JSON
  .then(unicorn =>
    fetch('new/unicorn', {                         // send a request to 'new/unicorn' 
        method: 'post',                            // using the POST method
        body: JSON.stringify({unicorn})            // pass the unicorn to the request body
    })
  )
  .then(responseNew => responseNew.json())
  .then(json => json.success)                      // return success property of response
  .catch(err => console.log('Error creating unicorn:', err));
 }

La función se puede reescribir usando async / await de la siguiente manera:

async function newUnicorn() {
  try {
    const responseCurrent = await fetch('unicorn.json'); // fetch unicorn.json from server
    const unicorn = await responseCurrent.json();        // parse the response as JSON
    const responseNew = await fetch('new/unicorn', {     // send a request to 'new/unicorn'
      method: 'post',                                    // using the POST method
      body: JSON.stringify({unicorn})                    // pass the unicorn to the request body
    });
    const json = await responseNew.json();
    return json.success                                  // return success property of response
  } catch (err) {
    console.log('Error creating unicorn:', err);
  }
}

Esta variante async de newUnicorn() parece devolver una promesa, pero en realidad había varias palabras clave await. Cada uno devolvió una Promesa, por lo que en realidad teníamos una colección de promesas en lugar de una cadena.

De hecho, podemos considerarlo como un generador de funciones*, en el que cada espera es una nueva promesa. Sin embargo, los resultados de cada promesa son necesarios para que la siguiente continúe con la función. Esta es la razón por la que se necesita la palabra clave adicional async en la función (así como la palabra clave await al llamar a las promesas), ya que le dice a Javascript que cree automáticamente un observador para esta iteración. La Promesa devuelta por la función asíncrona newUnicorn() se resuelve cuando se completa esta iteración.

Prácticamente, no necesita considerar eso; await oculta la promesa y async oculta la iteración del generador.

Puede llamar a las funciones async como si fueran promesas, y esperar cualquier promesa o cualquier función async. No necesita esperar una función asíncrona, al igual que puede ejecutar una promesa sin .then().

También puedes usar un async IIFE si quieres ejecutar ese código inmediatamente:

(async () => {
  await makeCoffee()
  console.log('coffee is ready!')
})()

Bucle con espera asíncrona #

Al usar async await en bucles, es posible que encuentre algunos de estos problemas.

Si solo intenta usar await dentro de forEach, esto generará un error de token inesperado.

(async() => {
datos = [1, 2, 3, 4, 5];
datos.paraCada(e => {
   const i = await somePromiseFn(e);
   console.log(i);
 });
})();

Esto se debe al hecho de que, por error, ha visto que la flecha funciona como un bloque. Await estará en el contexto de la función de devolución de llamada, que no es async. El intérprete nos protege de cometer el error anterior, pero si agrega async a la devolución de llamada forEach no se arrojan errores. Puede pensar que esto resuelve el problema, pero no funcionará como se esperaba.

Ejemplo:

(async() => {
datos = [1, 2, 3, 4, 5];
data.forEach(async(e) => {
    const i = await somePromiseFn(e);
    console.log(i);
  });
console.log('esto se imprimirá primero');
})();

Esto sucede porque la función asíncrona de devolución de llamada solo puede pausarse a sí misma, no la función asíncrona principal.

Podrías escribir una función asyncForEach que devuelva una promesa y luego podrías hacer algo como esperar asyncForEach(async (e) => esperar somePromiseFn(e), datos) Básicamente, devuelves una promesa que se resuelve cuando todas las devoluciones de llamadas se esperan y se realizan. Pero hay mejores formas de hacer esto, y eso es simplemente usar un bucle.

Puedes usar un bucle for-of o un bucle for/while, realmente no importa cuál elijas.

(async() => {
datos = [1, 2, 3, 4, 5];
para (let e de datos) {
    const i = await somePromiseFn(e);
    console.log(i);
  }
console.log('esto se imprimirá al final');
})();

Pero hay otra trampa. Esta solución esperará a que se complete cada llamada a somePromiseFn antes de iterar sobre la siguiente. Esto es excelente si realmente desea que sus invocaciones de somePromiseFn se ejecuten en orden, pero si desea que se ejecuten simultáneamente, deberá esperar en Promise.all.

(async() => {
datos = [1, 2, 3, 4, 5];
const p = await Promise.all(data.map(async(e) => await somePromiseFn(e)));
consola.log(...p);
})();

Promise.all recibe una serie de promesas como su único parámetro y devuelve una promesa. Cuando se resuelven todas las promesas de la matriz, también se resuelve la promesa devuelta. “Esperamos” esa promesa y, cuando se resuelve, todos nuestros valores están disponibles.

Los ejemplos anteriores son completamente ejecutables. La función somePromiseFn se puede hacer como una función de eco asíncrono con un tiempo de espera. Puede probar los ejemplos en babel-repl con al menos el ajuste preestablecido stage-3 y ver el resultado.

function algunaPromesaFn(n) {
volver nueva Promesa((res, rej) => {
   setTimeout(() => res(n), 250);
 });
}

Menos sangría #

con promesas:

function doTheThing() {
    return doOneThing()
        .then(doAnother)
        .then(doSomeMore)
        .catch(handleErrors)
}

Con funciones asíncronas:

async function doTheThing() {
    try {
        const one = await doOneThing();
        const another = await doAnother(one);
        return await doSomeMore(another);
    } catch (err) {
        handleErrors(err);
    }
}

Tenga en cuenta cómo el retorno está en la parte inferior, y no en la parte superior, y utiliza la mecánica nativa de manejo de errores del lenguaje (try/catch).

Operaciones asíncronas (paralelas) simultáneas #

A menudo querrá realizar operaciones asincrónicas en paralelo. Hay una sintaxis directa que admite esto en la propuesta async/await, pero dado que await esperará una promesa, puede envolver varias promesas juntas en Promise.all para esperarlas:

// Not in parallel

async function getFriendPosts(user) {
    friendIds = await db.get("friends", {user}, {id: 1});
    friendPosts = [];
    for (let id in friendIds) {
        friendPosts = friendPosts.concat( await db.get("posts", {user: id}) );
    }
    // etc.
}

Esto hará que cada consulta obtenga las publicaciones de cada amigo en serie, pero se pueden hacer simultáneamente:

// In parallel

async function getFriendPosts(user) {
    friendIds = await.db.get("friends", {user}, {id: 1});
    friendPosts = await Promise.all( friendIds.map(id => 
      db.get("posts", {user: id})
    );
    // etc.
}

Esto recorrerá la lista de ID para crear una serie de promesas. await esperará a que se completen todas las promesas. Promise.all los combina en una sola promesa, pero se hacen en paralelo.
