Acción de descuento del producto #

Un día tuve una conversación con un amigo mío que usa Laravel framework PHP en su trabajo. Cuando le dije que Django tiene su propio sistema HTML CRUD todo incluido, para interactuar con la base de datos, llamado Django admin, ¡se le saltaron los ojos! Me dijo: “Me tomó meses crear una interfaz de administración para mi aplicación web actual y ¿estás diciendo que tienes todo esto sin tener que escribir una sola línea de código?”. Respondí “¡Sí!”

El administrador de Django es una característica poderosa de Django que proporciona muchas ventajas. Uno de estos son [acciones] (https://docs.djangoproject.com/en/dev/ref/contrib/admin/actions/).

Pero, ¿qué son las “acciones”?

Suponga que tiene un modelo y ya ha agregado algunas entradas (tal vez cientos, tal vez miles) en él. Ahora, desea aplicar una regla-acción a al menos uno de ellos, sin usar la consola (python manage.py shell):

# python manage.py shell (interactive console)

from math import ceil

from my_app.models import Product


DISCOUNT = 10  # percentage

for product in Product.objects.filter(is_active=True):
    """ Set discount to ALL products that are flagged as active """
    multiplier = DISCOUNT / 100.  # DISCOUNT / 100 in python 3 (without dot)
    old_price = product.price
    new_price = ceil(old_price - (old_price * multiplier))  # seller wins :)
    product.price = new_price
    product.save(update_fields=['price'])

¿Notaste que aplicamos el descuento a todos los productos? ¿Y si quisiéramos aplicar esta lógica a unos específicos? ¿O si quisiéramos ingresar manualmente el valor del descuento y luego aplicar este valor a algunos productos? ¡Todo esto a través del administrador de Django! Estás en el camino correcto. Acciones Django FTW. Veamos un ejemplo completo.

Los basicos: #

  • Pitón 3.4.3 -Django 1.10
  • SQLite (construido en Python, sin necesidad de configuración de instalación adicional)

El modelo: #

  • Representar un producto para nuestra tienda electrónica
  • El precio de un Producto serían números enteros (no decimales)

La meta: #

  • Poder, a través de la interfaz Django Admin, aplicar un descuento fijo a una o más entradas de productos.

Configuración (aplicación, modelo y administrador de Django) #

Suponiendo que ya ha [comenzado un proyecto] (https://www.wikiod.com/es/django/comenzando-con-django), vaya al directorio donde vive manage.py y cree una aplicación con el nombre stock, escribiendo:

python manage.py createapp stock

Django creará automáticamente una estructura de directorios para usted. Vaya y edite el archivo models.py y agregue:

# models.py

from django.db import models


class Product(models.Model):
    name = models.CharField('Product name', max_length=100)  # required field
    price = models.PositiveIntegerField('Product price')

    def __str__(self):  # __unicode__ in Python 2
        return self.name

Nuestro modelo Producto ha sido creado, pero todavía no hay nada en la base de datos. Para que las migraciones funcionen, nuestra aplicación debe estar incluida en la lista INSTALLED_APPS.

Edite su archivo settings.py y en la lista INSTALLED_APPS agregue:

# settings.py

INSTALLED_APPS = [
    # ... previous Django apps
    'stock.apps.StockConfig',
]

Ahora ejecuta:

python manage.py makemigrations
python manage.py migrate

Después del comando ‘migrar’, su base de datos ahora tiene una tabla llamada ‘producto’ que contiene tres columnas, ‘id’, ’nombre’ y ‘precio’. ¡Hasta aquí todo bien!

Si no ha cambiado nada en su archivo ROOT_URLCONF, que generalmente se encuentra dentro de la carpeta <your_project_name>/< your_project_name>/, entonces la URL que apunta al sitio de administración de Django debería ser:

# urls.py

urlpatterns = [
    # ... other URLs
    url(r'^admin/', admin.site.urls),
]

Hasta ahora, no hemos buscado nada específico sobre las acciones de administración de Django. Tome un paso final y agréguelos dentro de su archivo stock/admin.py:

# admin.py

from django.contrib import admin

from .models import Product


@admin.register(Product)
class ProductAdmin(admin.ModelAdmin):
    pass

ESTÁ BIEN. La configuración está lista. Solo para asegurarte de que todo funcione, ejecuta:

python manage.py runserver

y con tu navegador favorito visita la página 127.0.0.1:8000/admin/. ¡Deberías ver la brillante, glamorosa y excelente página de administración de Django, que te permite Crear-Rleer-Uactualizar-Deliminar tu modelo de Producto! Si por casualidad, la página anterior te pide un nombre de usuario/contraseña y no tienes ninguno, no hay problema. Simplemente no ha creado un ‘Usuario’ para iniciar sesión en el administrador. Simplemente ejecute:

python manage.py createsuperuser

ingrese su nombre, correo electrónico, nombre de usuario y contraseña (dos veces) y listo.


Generar algunos productos falsos #

Hasta ahora hemos creado el modelo, pero todavía no hay entradas (no hay productos). Necesitamos algunas entradas para arrojar luz sobre el poder de las acciones de administración de Django.

Vamos a crear 100 productos y trabajar con estos. Pero, en lugar de presionar manualmente el botón AÑADIR e ingresar un ’nombre’ y un ‘precio’, escribiremos un script para que haga el trabajo por nosotros.

Ejecute python manage.py shell e ingrese lo siguiente:

# python manage.py shell

from stock.models import Product    


for i in range(1, 101):
    p = Product.objects.create(name='Product %s' % i, price=i)

El bucle for anterior crea (lo que significa que los datos se guardan en la base de datos) 100 productos (entradas) con los nombres Producto 1, Producto 2, … Producto 100 y precios 1, 2 , …, 100.

Para ver estos productos a través de la página de administración de Django, visite de nuevo 127.0.0.1:8000/admin/ y haga clic en el enlace Productos:

Disfrute de sus 100 productos generados automáticamente:

Esto se conoce como la página de lista de cambios de Django. Ahora, para que esto se vea mucho más bonito, edite su archivo stock/admin.py e ingrese:

@admin.register(Product)
class ProductAdmin(admin.ModelAdmin):
    list_display = ('name', 'price')

Ahora, presione Actualizar y debería ver una segunda columna que muestra los precios.


¡Uf! Finaliza las acciones #

Para recapitular, tenemos el modelo y tenemos las entradas. A continuación, queremos crear una acción que, una vez seleccionada, haga un descuento del 30 % en los productos seleccionados.

¿Te diste cuenta de que ya hay un cuadro de selección en la parte superior de la página de “lista de cambios”, con la etiqueta “Acción”? Django agrega automáticamente una acción predeterminada en cada entrada que realiza la acción Deliminar.

Las acciones de administración de Django están escritas como funciones simples. Profundicemos. Edite el archivo stock/admin.py y agregue lo siguiente:

# admin.py

@admin.register(Product)
class ProductAdmin(admin.ModelAdmin):
    list_display = ('name', 'price')
    actions = ['discount_30']

    def discount_30(self, request, queryset):
        from math import ceil
        discount = 30  # percentage

        for product in queryset:
            """ Set a discount of 30% to selected products """
            multiplier = discount / 100.  # discount / 100 in python 3
            old_price = product.price
            new_price = ceil(old_price - (old_price * multiplier))
            product.price = new_price
            product.save(update_fields=['price'])
    discount_30.short_description = 'Set 30%% discount'

Algunas cosas a tener en cuenta aquí:

  • La clase ProductAdmin tiene un atributo extra (acciones) que es una lista de cadenas (cada cadena es el nombre de la función que representa la acción).
  • La función de acción que es un método de la clase ProductAdmin. Toma como argumentos la instancia ModelAdmin (self ya que este es un método), el objeto HTTP request y el queryset (una lista de los objetos-entradas-productos seleccionados).
  • La última línea es un atributo de función (short_description) que establece el nombre que se muestra dentro del cuadro de selección de acciones (hay un doble % para escapar del único %).

Dentro de la función-acción iteramos en cada producto (que fue seleccionado) y establecemos su valor reducido en un 30%. Luego llamamos al método save() con el argumento [update_fields](https:/ /docs.djangoproject.com/en/dev/ref/models/instances/#specifying-what-fields-to-save) para forzar una ACTUALIZACIÓN en los campos que están incluidos en la lista update_fields (en lugar de una ACTUALIZAR en todos los campos del modelo) en la base de datos, por razones de rendimiento (no es una ganancia de rendimiento en este ejemplo, con solo 2 columnas, pero entiende el punto).

Ahora, presione actualizar en la página cambiar lista y debería ver su acción debajo de eliminar. Continúe y seleccione algunos productos (usando la casilla de verificación a la izquierda de cada uno o todos los productos usando la casilla de verificación más a la izquierda), seleccione la acción ‘Establecer 30% de descuento’ y haga clic en el botón ‘Ir’. ¡Eso es todo!
