lanzar nodos ros y cargar parámetros desde el archivo Yaml #

roslaunch es una herramienta importante que gestiona el inicio y la parada de los nodos ROS. Toma uno o más archivos “*.launch” como argumentos.

Para este ejemplo, me referiré a lo siguiente (como se pregunta en esta pregunta), entonces, ¿cómo podemos ejecutar esos comandos de forma consecutiva y automática?

roscd stereo_camera
rosparam load marvin_cameras.yaml
rosrun stereo_camera stereo_camera __name:=bumblebeeLeft
rosrun stereo_camera stereo_camera __name:=bumblebeeCenter

roslaunch openni_launch_marvin kinect_left.launch
roslaunch openni_launch_marvin kinect_center.launch

Primero que nada, dividamos estos comandos en pedazos. Como puede verse, se necesitan 4 comandos ros: roscd, rosparam, rosrun y roslaunch. ¡Ahora comencemos!

Este es un ejemplo de por qué roslaunch es poderoso. De hecho, todos esos comandos podrían incluirse en un y único archivo roslaunch en ROS. entonces todo lo que tenemos que hacer es lanzar este archivo “solution.launch” para una llamada consecutiva y automática de esos comandos.

Para empezar, los archivos de lanzamiento se basan en formato XML, aquí hay un archivo de lanzamiento básico en ROS, lo llamaremos “basic_example.launch” y está incluido en un paquete de ROS llamado “roslaunch_example”:

<launch>
    
</launch>

el comando para ejecutar este archivo de lanzamiento es

$ roslaunch roslaunch_example basic_example.launch

siguiendo la especificación:

$ roslaunch package_name launch_file_name.launch

Debido a que nuestro archivo de inicio no incluye ningún comando, no se ejecutará nada, más sobre eso más adelante…

Incluyendo nodos en archivos de lanzamiento: cualquier nodo ROS en cualquier paquete ROS instalado se puede llamar en los archivos de inicio. para eso tenemos que especificar el paquete que contiene el nodo y su nombre como se especifica en el paquete. por ejemplo :

rosrun stereo_camera stereo_camera __name:=bumblebeeLeft
rosrun stereo_camera stereo_camera __name:=bumblebeeCenter

stereo_camera es un nodo del paquete stereo_camera y los argumentos especificados son su nombre __name:=bumblebeeLeft y __name:=bumblebeeCenter.

Para agregar esos nodos, tenemos que agregar las siguientes líneas:

<launch>    
    <node name="$(arg name)" pkg="stereo_camera" type="stereo_camera" output="screen">
          <param name="name" value="bumblebeeLeft" />
    </node>

    <node name="$(arg name)" pkg="stereo_camera" type="stereo_camera" output="screen">
          <param name="name" value="bumblebeeCenter" />
    </node>
</launch>

Ejecutando este archivo de lanzamiento, tendremos los dos nodos ejecutándose.

agregar un parámetro a un nodo ROS:

Como se puede ver, agregamos un parámetro “nombre” para los nodos como:

<param name="name" value="bumblebeeCenter" />

De hecho, podemos agregar tantos parámetros como queramos (como se creó anteriormente) y luego referirnos a ellos llamando "$(arg nombre_parámetro)" en lugar de fijar su valor.

Especificación de la salida:

La etiqueta de salida se puede establecer en “pantalla”, si necesita ver el registro del nodo en el terminal o “registro” para guardar el registro en los archivos de registro en (~/.ros).

Incluyendo otros archivos de lanzamiento de ROS en un archivo de lanzamiento de ROS:

Como se indica aquí, la etiqueta le permite importar otro archivo XML de roslaunch al archivo actual. Se importará dentro del alcance actual de su documento, incluidas las etiquetas y . Se importará todo el contenido del archivo de inclusión excepto la etiqueta : la etiqueta solo se cumple en el archivo de nivel superior.

Entonces, todo para ejecutar estos comandos.

roslaunch openni_launch_marvin kinect_left.launch
roslaunch openni_launch_marvin kinect_center.launch

desde un archivo de lanzamiento, todo lo que tenemos que hacer es agregar las siguientes líneas:

<include file="$(find openni_launch_marvin)/launch/kinect_left.launch" />
<include file="$(find openni_launch_marvin)/launch/kinect_center.launch" />

roscd a un paquete en el archivo de lanzamiento de ROS

Para encontrar el archivo de lanzamiento que queremos incluir, no necesitamos especificar la ruta completa. En su lugar, roslaunch proporciona la directiva "$(find package_name)", de esta manera, podemos referirnos a nuestro archivo de lanzamiento relativo al paquete racine. En el ejemplo anterior, asumí que el archivo “kinect_center.launch” está en la carpeta “openni_launch_marvin)/launch/”.

Cargando parámetros desde un archivo YAML:

Para cargar parámetros desde un archivo YAML en ROS, ROS proporciona la etiqueta “rosparam”. Como se indica en wiki: “La etiqueta permite el uso de archivos YAML de rosparam para cargar y descargar parámetros del servidor de parámetros de ROS. También se puede usar para eliminar parámetros. La etiqueta puede colocarse dentro de una etiqueta , en cuyo caso el parámetro se trata como un nombre privado”.

Con esta etiqueta, podemos cargar nuestro archivo YAML en el archivo de inicio agregando esta línea:

<rosparam command="load" file="$(find marvin_cameras)/config/marvin_cameras.yaml" />

Como se usó anteriormente, asumí que el archivo YAML “marvin_cameras.yaml” está en la carpeta “marvin_cameras/config/”.

Montando todas las piezas

Ahora que hemos creado por separado el contenido de nuestro archivo de inicio, vamos a ensamblarlos en un gran archivo de inicio “solution.launch”.

solución.lanzamiento

<launch> 

    <rosparam command="load" file="$(find marvin_cameras)/config/marvin_cameras.yaml" />  

    <node name="$(arg name)" pkg="stereo_camera" type="stereo_camera" output="screen">
          <param name="name" value="bumblebeeLeft" />
    </node>

    <node name="$(arg name)" pkg="stereo_camera" type="stereo_camera" output="screen">
          <param name="name" value="bumblebeeCenter" />
    </node>

    <include file="$(find openni_launch_marvin)/launch/kinect_left.launch" />
    <include file="$(find openni_launch_marvin)/launch/kinect_center.launch" />

</launch>

Ahora, tenemos nuestro único archivo roslaunch para ejecutar todos los comandos de forma consecutiva y automática.
