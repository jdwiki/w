Ventanas #

  1. Asegúrese de tener instalado el último nodeJS LTS
  2. Abra el símbolo del sistema y escriba $ npm install -g nativescript
  3. En el símbolo del sistema, escriba $ @powershell -NoProfile -ExecutionPolicy Bypass -Command "iex ((new-object net.webclient).DownloadString('https://www.nativescript.org/setup/win'))" - esto puede llevar un tiempo
  4. Para verificar que lo anterior haya funcionado, escriba $ tns doctor en el símbolo del sistema (su cmd)
  5. Si hay algún error, siga con la [guía de solución de problemas] (https://docs.nativescript.org/start/troubleshooting)

Instalación o Configuración #

Instrucciones detalladas sobre cómo configurar o instalar Nativescript.

Los siguientes ejemplos muestran los pasos necesarios para configurar un sistema Windows u OSX y luego firmar una publicación en las guías de solución de problemas en caso de que tenga algún problema.

Además, hay ejemplos de cómo configurar flujos de trabajo, IDE y emuladores recomendados.

Mac OS #

  1. Asegúrese de tener instalado el más reciente Node.js LTS. Si usa Homebrew esto se puede hacer con brew install node4-lts.
  2. Abra la Terminal y escriba npm install -g nativescript. Si obtiene un error EACCES, use sudo npm install -g nativescript.
  3. En el símbolo del sistema, escriba ruby -e "$(curl -fsSL https://www.nativescript.org/setup/mac)". (Esto podría tomar un tiempo.)
  4. Para verificar que lo anterior haya funcionado, escriba tns doctor en la Terminal.
  5. Si hay algún error, siga con la [guía de solución de problemas] (https://docs.nativescript.org/start/troubleshooting).

Uso de Visual Studio Code para el desarrollo de NativeScript #

Visual Studio Code es un editor de código de código abierto y rico en funciones de Microsoft. Para configurarlo para el desarrollo de NativeScript, abra la paleta de comandos (F1 o ⌘+Shift+P) y escriba ext install NativeScript.

Una vez instalada la extensión NativeScript, el depurador debería permitirle establecer puntos de interrupción en su código. Cuando un dispositivo está conectado o se está ejecutando un emulador, puede iniciar su aplicación desde la pestaña Depurar.

Tu primer programa Hola Mundo #

$ mkdir hello-world
$ cd hello-world
$ tns create hello-world --ng
$ tns platform add android #You can only add ios on an OSX machine

Luego, asegúrese de tener un dispositivo conectado o un emulador en ejecución (si no lo tiene, el emulador predeterminado debería iniciarse o se generará un error. Recomendaría genymotion para Android).

$ tns run android

Si desea utilizar el emulador de Android predeterminado, agregue el indicador --emulator.

A partir de tns 2.5, livesync ahora es la acción predeterminada para tns run <platform>, que se volverá a compilar automáticamente cuando guarde los cambios del archivo. Esto puede mejorar drásticamente su tiempo de desarrollo, sin embargo, si realiza cambios en sus complementos, deberá volver a compilar correctamente.

Cómo depurar la aplicación nativescript-android a través de WiFi (sin raíz) #

1-Necesita conectar su dispositivo a su computadora a través de un cable USB. Asegúrese de que la depuración de USB esté funcionando. Puede verificar si aparece cuando ejecuta dispositivos adb (o dispositivo tns).

2-Ejecute adb tcpip 5555

3-Desconecte su dispositivo (retire el cable USB).

4-Vaya a Configuración -> Acerca del teléfono -> Estado para ver la dirección IP de su teléfono.

5-Ejecute adb connect <dirección IP de su dispositivo>:5555

6-Si ejecuta dispositivos adb (o dispositivo tns) nuevamente, debería ver su dispositivo.

7- Ahora puede usar los comandos tns run android, tns livesync android.

NOTAS:

1-cuando la red WiFi cambia, no tiene que repetir los pasos 1 a 3 (estos configuran su teléfono en modo de depuración wifi). Tienes que volver a conectarte a tu teléfono ejecutando los pasos 4 a 6.

2-Los teléfonos Android pierden el modo de depuración wifi al reiniciar. Por lo tanto, si su batería se agotó, debe comenzar de nuevo. De lo contrario, si vigila su batería y no reinicia su teléfono, ¡puede vivir sin cable durante semanas!

ADVERTENCIA :

dejar la opción habilitada es peligroso, cualquier persona en su red puede conectarse a su dispositivo en depuración, incluso si está en la red de datos. ¡Hazlo solo cuando estés conectado a un Wi-Fi confiable y recuerda desconectarlo cuando termines!
