Conceptos básicos de las relaciones entre entidades #

Una clave externa puede ser una o más columnas que hacen referencia a una clave única, generalmente la clave principal, en otra tabla.

Una clave externa y la clave principal principal a la que hace referencia deben tener el mismo número y tipo de campos.

Las claves foráneas representan relaciones de una columna o columnas en una tabla a una columna o columnas en otra tabla.

Ejemplo de anotación @JoinTable #

Al mapear relaciones de muchos a muchos en JPA, la configuración de la tabla utilizada para unir claves foráneas se puede proporcionar mediante la anotación @JoinTable:

@Entity
public class EntityA {
   @Id
   @Column(name="id")
   private long id;
   [...]
   @ManyToMany
   @JoinTable(name="table_join_A_B",
              joinColumns=@JoinColumn(name="id_A"), referencedColumnName="id"
              inverseJoinColumns=@JoinColumn(name="id_B", referencedColumnName="id"))
   private List<EntityB> entitiesB;
   [...]
}

@Entity
public class EntityB {
   @Id
   @Column(name="id")
   private long id;
   [...]
}

En este ejemplo, que consiste en que la Entidad A tiene una relación de muchos a muchos con la Entidad B, realizada por el campo entitiesB, usamos la anotación @JoinTable para especificar que el nombre de la tabla para unir la tabla es table_join_A_B, compuesto por las columnas id_A e id_B, claves foráneas que hacen referencia respectivamente a la columna id en la tabla de EntityA y en la tabla de EntityB; (id_A,id_B) será una clave primaria compuesta para la tabla table_join_A_B.

Multiplicidad en las relaciones de entidad #

Multiplicidad en las relaciones de entidad #

Las multiplicidades son de los siguientes tipos:

  • Uno a uno: cada instancia de entidad está relacionada con una única instancia de otra entidad.
  • Uno a muchos: una instancia de entidad se puede relacionar con varias instancias de otras entidades.
  • Muchos a uno: varias instancias de una entidad se pueden relacionar con una sola instancia de la otra entidad.
  • Muchos a muchos: las instancias de la entidad se pueden relacionar con varias instancias entre sí.

Mapeo uno a uno #

El mapeo uno a uno define una asociación de un solo valor con otra entidad que tiene una multiplicidad uno a uno. Este mapeo de relaciones usa la anotación @OneToOne en la propiedad o campo persistente correspondiente.

Ejemplo: entidades Vehicle y ParkingPlace.

Mapeo de uno a muchos #

Una instancia de entidad se puede relacionar con múltiples instancias de otras entidades.

Las relaciones de uno a muchos usan la anotación @OneToMany en la propiedad o campo persistente correspondiente.

El elemento mappedBy es necesario para hacer referencia al atributo anotado por ManyToOne en la entidad correspondiente:

 @OneToMany(mappedBy="attribute")

Una asociación de uno a muchos necesita mapear la colección de entidades.

Mapeo de muchos a uno #

Una asignación de muchos a uno se define anotando el atributo en la entidad de origen (el atributo que hace referencia a la entidad de destino) con la anotación @ManyToOne.

Una anotación @JoinColumn(name="FK_name") describe una clave externa de una relación.

Mapeo de muchos a muchos #

Las instancias de entidad se pueden relacionar con múltiples instancias entre sí.

Las relaciones de muchos a muchos usan la anotación @ManyToMany en la propiedad o campo persistente correspondiente.

Debemos usar una tercera tabla para asociar los dos tipos de entidades (join table).
