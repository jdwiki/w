Archivos de inicialización #

La mayoría de las implementaciones de Common Lisp intentarán cargar un archivo de inicio al inicio:

Implementación Archivo de inicio Archivo de inicio del sitio/sistema
ABCL $INICIO/.abclrc
Allegro CL $HOME/.clinit.cl
ECL $HOME/.eclrc
Cierre $HOME/.clasprc
CLIP $HOME/.clisprc.lisp
Cerrar CL inicio:ccl-init.lisp o inicio:ccl-init.fasl o inicio:.ccl-init.lisp
CMUCL $HOME/.cmucl-init.lisp
LispWorks $HOME/.lispworks
MKCL $HOME/.mkclrc
SBCL $INICIO/.sbclrc $SBCL_HOME/sbclrc o /etc/sbclrc
SCL $HOME/.scl-init.lisp

Ejemplos de archivos de inicialización:

Ajustes de optimización #

Common Lisp tiene una forma de influir en las estrategias de compilación. Tiene sentido definir sus valores preferidos.

Los valores de optimización están entre 0 (sin importancia) y 3 (extremadamente importante). 1 es el valor neutro.

Es útil usar siempre código seguro (seguridad = 3) con todas las comprobaciones de tiempo de ejecución habilitadas.

Tenga en cuenta que la interpretación de los valores es específica de la implementación. La mayoría de las implementaciones de Common Lisp hacen algún uso de estos valores.

Configuración Explicación valor predeterminado útil valor útil de entrega
velocidad de compilación velocidad del proceso de compilación 2 0
depuración facilidad de depuración 2 1 o 0
seguridad comprobación de errores en tiempo de ejecución 3 2
espacio tanto el tamaño del código como el espacio de tiempo de ejecución 2 2
velocidad velocidad del código objeto 2 3

Una declaración optimize para usar con declaim, declare y proclaim:

(optimize (compilation-speed 2)
          (debug 2)
          (safety 3)
          (space 2)
          (speed 2))

Tenga en cuenta que también puede aplicar configuraciones de optimización especiales a partes del código en una función usando la macro LOCALLY.
