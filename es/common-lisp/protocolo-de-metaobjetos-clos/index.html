Obtener los nombres de las ranuras de una Clase #

Digamos que tenemos una clase como

(defclass person ()
  (name email age))

Para obtener los nombres de los slots de la clase usamos la función class-slots. Esto se puede encontrar en el paquete de trapeador más cerrado, proporcionado por el sistema de trapeador más cerrado. Para cargar la imagen lisp en ejecución, usamos (ql:quickload:closer-mop). También debemos asegurarnos de que la clase esté finalizada antes de llamar a class-slots.

(let ((class (find-class 'person)))
  (c2mop:ensure-finalized class)
  (c2mop:class-slots class))

que devuelve una lista de objetos definición de ranura efectiva:

(#<SB-MOP:STANDARD-EFFECTIVE-SLOT-DEFINITION S/TRANSFORMATIONS::NAME>
 #<SB-MOP:STANDARD-EFFECTIVE-SLOT-DEFINITION S/TRANSFORMATIONS::EMAIL>
 #<SB-MOP:STANDARD-EFFECTIVE-SLOT-DEFINITION S/TRANSFORMATIONS::AGE>)

Actualizar un espacio cuando se modifica otro espacio #

El CLOS MOP proporciona la clase de uso de valor de ranura de gancho, que se llama cuando se accede, lee o modifica una ranura. Debido a que solo nos importan las modificaciones en este caso, definimos un método para (setf slot-value-using-class).

(defclass document ()
  ((id :reader id :documentation "A hash computed with the contents of every other slot")
   (title :initarg :title :accessor title)
   (body :initarg :body :accessor body)))

(defmethod (setf c2mop:slot-value-using-class) :after
    (new class (object document) (slot c2mop:standard-effective-slot-definition))
  ;; To avoid this method triggering a call to itself, we check that the slot
  ;; the modification occurred in is not the slot we are updating.
  (unless (eq (slot-definition-name slot) 'id)
    (setf (slot-value object 'id) (hash-slots object))))

Tenga en cuenta que debido a que en la creación de instancias no se llama slot-value, puede ser necesario duplicar el código en el método initialize-instance: after

(defmethod initialize-instance :after ((obj document) &key)
  (setf (slot-value obj 'id)
        (hash-slots obj)))
