Creando espaciadores #

Algunos clientes de correo electrónico (en particular, Windows Desktop Outlook) colapsan las tablas sin contenido, incluso si hay dimensiones o relleno.

Para evitar que estos clientes colapsen la tabla espaciadora, puede agregar un &nbsp; invisible que sirva como contenido. Luego ponga a cero el tamaño de fuente y la altura de línea del carácter para que no afecte las dimensiones de la <tabla>. El atributo de altura (o ancho) ahora es lo único que afecta las dimensiones del espaciador.

<table border="0" cellpadding="0" cellspacing="0" role="presentation">
    <tr>
        <td height="40" style="font-size: 0; line-height: 0;">
            &nbsp;
        </td>
    </tr>
</table>

Era una práctica común usar un .gif transparente de 1x1 como espaciador, pero este método está en desuso.

Espaciado entre elementos #

Outlook a veces puede agregar un poco de espacio en el lado izquierdo y derecho de un elemento

que puede causar algunos dolores de cabeza relacionados con el diseño. Mediante el uso de las propiedades CSS mso-table-lspace y mso-table-rspace específicas del proveedor, puede deshacerse de esos espacios y continuar abordando los millones de otros problemas causados ​​por Outlook.

table{
    mso-table-lspace:0pt;
    mso-table-rspace:0pt;
}

Cambio de tamaño de imagen #

El uso de etiquetas de ancho o alto para cambiar el tamaño de las imágenes en su marcado puede crear un problema en los navegadores Internet Explorer. Si su lector está viendo un correo electrónico en el navegador, y ese correo electrónico tiene imágenes fluidas, se verán bastante feas a medida que cambian de tamaño. Usando -ms-interpolation-mode:bicubic; asegura que sus imágenes se vean un poco mejor.

img{
    -ms-interpolation-mode:bicubic;
}
