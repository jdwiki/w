La grabación de escenarios de prueba es una de las formas más convenientes de crear scripts de prueba. Esto se debe a que las grabaciones de prueba le permiten imitar flujos de trabajo de usuario realistas, en lugar de tener que crear manualmente un script de prueba. Las grabaciones capturan todas las solicitudes del navegador a la aplicación web y luego crean automáticamente un archivo jmx que se puede ejecutar en las pruebas de rendimiento. Al usar la funcionalidad de grabación/reproducción de JMeter o herramientas de terceros como BlazeMeter y BadBoy, los evaluadores pueden hacer su trabajo 3 veces más rápido.

Grabación de scripts con JMeter Proxy Recorder #

JMeter también le permite configurar manualmente su espacio de trabajo. Es más complejo, pero puede hacer que los scripts se ajusten a sus necesidades exactas.

Configura el proxy de tu navegador

  1. Configure su navegador, como se describe en el capítulo 1.

Configurar JMeter

La rama “WorkBench” se puede usar como un espacio de trabajo temporal para crear scripts. Tenga en cuenta que las entradas añadidas a esta sección no se guardarán con el plan de prueba. Por lo tanto, si desea reutilizar la misma configuración de grabación en el futuro, deberá copiarla y pegarla en la sección “Plan de prueba”.

  1. Agregue “Controlador de grabación” a “WorkBench”: haga clic derecho en “WorkBench” -> “Agregar” -> “Controlador lógico” -> “Controlador de grabación”

  1. Agregue “HTTP(S) Test Script Recorder” al mismo “WorkBench”: haga clic con el botón derecho en “WorkBench” -> “Agregar” -> “Non-Test Elements” -> “HTTP(S) Test Script Recorder”

  1. En la página de configuración “HTTP(S) Test Script Recorder” en “Configuración global: Puerto”, debe colocar el mismo puerto que se especifica en la configuración del proxy de su navegador, por ejemplo, 8080.

  2. Si desea excluir solicitudes a recursos específicos, puede utilizar la sección “Patrones de URL para excluir”. Esto puede ser útil si desea incluir solo los tipos de contenido que desea solicitar (p. ej., *.html, *.php, etc.) o excluir los tipos de contenido que no desea solicitar (p. ej., *.jpg, * .png, *.js, etc.).

¿Cuándo usaríamos esto? Por ejemplo, al grabar una secuencia de comandos que llama a una aplicación de terceros o al probar una secuencia de comandos del lado del servidor, es posible que no desee descargar activos, ya que pueden abarrotar sus pruebas y consumir ancho de banda, o es posible que desee registrar ciertas solicitudes de un camino determinado.

Los patrones de exclusión más comunes son: “..png”, “..jpg”, “..gif”, “..css”, “..js”. También puede combinar diferentes patrones juntos. Este patrón combinado debería deshacerse de todas las solicitudes redundantes que podrían distraerlo de las importantes: “..(bmp|css|js|gif|ico|jpe?g|png|swf|woff)”

  1. Por otro lado, es posible que desee imitar el comportamiento completo del navegador e incluir la carga de todos los recursos. En este caso, no es necesario excluir patrones de URL. Tenga en cuenta que el navegador descarga todos los recursos incrustados de la página solicitada y que tiene un mecanismo de almacenamiento en caché, lo que podría afectar el rendimiento de los resultados.

En este caso, se recomienda descargar todos los recursos incrustados en el script: Haga clic derecho en “Plan de prueba” -> “Agregar” -> “Elemento de configuración” -> “Valores predeterminados de solicitud HTTP” -> “Avanzado” -> seleccione la casilla de verificación “Recuperar todos los recursos integrados”.

  1. Para que JMeter se comporte más como un navegador real, se recomienda agregar el “Administrador de caché HTTP”, que le permite simular la funcionalidad de almacenamiento en caché del navegador en sus pruebas de rendimiento. Haga clic derecho en “Plan de prueba” -> “Agregar” -> “Elemento de configuración” -> “Administrador de caché HTTP”.

  2. Ahora haga clic en el botón ‘Inicio’, que se encuentra en la parte inferior de la página “Grabador de scripts de prueba HTTP(S)”, y siga el flujo de trabajo de la aplicación web que desea probar. Cuando regrese a JMeter, debería ver todas las solicitudes capturadas de su navegador en el “Controlador de grabación”.

Grabación de scripts con la función de plantilla JMeter #

En la versión 2.10, JMeter introdujo un mecanismo que le ahorra tiempo al crear scripts: plantillas de JMeter. Estas plantillas son esqueletos que se pueden reutilizar como base para sus nuevos scripts.

JMeter ya tiene varias plantillas disponibles con descripciones detalladas, y también puede agregar las suyas propias. Las plantillas tienen toda la configuración y los elementos necesarios para grabar sus guiones de rendimiento desde cero.

Aquí se explica cómo usar la función de plantilla de JMeter:

Configurar JMeter

  1. Abra JMeter

  2. Seleccione la plantilla para la grabación de guiones: Archivo -> Plantillas… -> Seleccionar plantilla -> Grabación -> Crear JMeter agregará los elementos relevantes al árbol de prueba.

Configura el proxy de tu navegador

Para usar JMeter Recorder, debe configurar su navegador para enviar todas las solicitudes a través de un proxy. Se puede usar cualquier navegador para estas necesidades, aunque puede haber diferencias entre las ubicaciones de las configuraciones de los navegadores, que son específicas del navegador y pueden variar según el sistema operativo.

  1. Para configurar su navegador:

    Chrome: Menu button -> Settings -> Show advanced settings… -> Network -> Change proxy settings

    Safari: Preferences -> Advanced -> Proxies -> Change Settings…

    Firefox: Menu button -> Preferences -> Advanced -> Network -> Connection -> Settings..

  2. Por ejemplo, puede usar localhost 127.0.0.1. Cambie el puerto al puerto en HTTP(S) Script Recorder.

Si tiene problemas al completar esta etapa, verifique que no tenga complementos de terceros que puedan administrar la configuración de proxy de su navegador. Si lo hace, como Hola VPN, por ejemplo, la configuración del proxy no estará disponible en el menú de su navegador.

  1. Haga clic en el botón ‘Inicio’, que se encuentra en la parte inferior de la página “Grabador de secuencias de comandos de prueba HTTP(S)” y siga el flujo de trabajo de la aplicación web que desea probar. Cuando regrese a JMeter, debería ver todas las solicitudes capturadas de su navegador.

Grabación de secuencias de comandos de rendimiento para dispositivos móviles #

JMeter también se puede utilizar para registrar pruebas de rendimiento móvil. La grabación de scripts móviles es muy similar a la grabación de scripts de aplicaciones web.

Configurar JMeter

  1. Configure las “Plantillas JMeter” como se especifica en el capítulo 1.

Configura tu teléfono móvil

Después de preparar la configuración de JMeter, incluido el elemento JMeter “HTTP(S) Test Script Recording” iniciado en un puerto específico, puede configurar su teléfono móvil para enviar una solicitud a la aplicación web que está probando a través del proxy de JMeter.

  1. IOS:

  • Configuración -> Wi-Fi

  • Haga clic en la red conectada

  • Ir a la sección de configuración “HTTP PROXY”

  • Haga clic en la pestaña “Manual”

  • Configure la IP de la computadora en la que se ejecuta la aplicación JMeter en “Servidor”

  • Establezca el puerto que se especifica en “Grabación de secuencias de comandos de prueba HTTP(S)” en “Puerto”

    Android:

  • Configuración -> Wi-Fi

  • Haga clic largo en la red conectada y haga clic en la opción ‘Modificar red’

  • Haga clic en la casilla de verificación “Opciones avanzadas”

  • Configure la opción “Proxy” en “Manual”

  • Establezca el “Nombre de host del proxy” como la dirección IP de su computadora y el “Puerto del proxy” como se especifica en la configuración “Grabación del script de prueba HTTP(S)” en “Puerto”

  • Clic en Guardar"

  1. Ahora puede comenzar a ejecutar la aplicación en su dispositivo móvil. Las solicitudes se registrarán en JMeter.

Grabación de tráfico HTTPS #

Si su aplicación web utiliza el cifrado SSL, debe capturar el tráfico HTTPS en lugar de HTTP. Para registrar el tráfico HTTPS con JMeter, debe configurar los certificados SSL.

Configura tu proxy SSL

  1. Asegúrese de que el proxy SSL esté configurado de la misma manera que el proxy HTTP:

Configurar JMeter 2. Inicie la grabación de guiones utilizando la función “Plantilla de grabación de JMeter” como se explica en el ejemplo “Grabación de guiones con la función de plantilla de JMeter”.

  1. Después de abrir la aplicación web, verá un mensaje sobre una conexión no segura. Para continuar, solo necesita aceptar el certificado ficticio de JMeter:
  • Haga clic en ‘Avanzado’
  • Haga clic en ‘Agregar excepción…’
  • Desmarque ‘Almacenar permanentemente esta excepción’
  • Haga clic en ‘Confirmar excepción de seguridad’

  1. Si ve el mensaje “Este sitio proporciona una identificación válida y verificada. No hay necesidad de agregar una excepción.” mensaje de advertencia, debe borrar el historial del navegador para su aplicación, incluidas las cookies, el caché y los datos del sitio web sin conexión. Luego, continúe con los mismos pasos nuevamente.

Este enfoque también funciona para la grabación de scripts móviles, ya que el certificado JMeter debe instalarse solo en el host que se utiliza para ejecutar JMeter.

Grabación de guiones con la extensión de Chrome BlazeMeter #

Hasta ahora hemos cubierto las formas básicas de registrar escenarios de prueba. Pero una de las formas más rápidas y sencillas de grabar sus guiones de rendimiento, que también es gratuita, es usar la extensión BlazeMeter Recorder Chrome. Estas grabaciones se pueden ejecutar en JMeter o en BlazeMeter.

La razón por la que la extensión es tan útil es que le permite grabar secuencias de comandos de rendimiento desde su navegador sin tener que configurar su proxy.

Para crear un nuevo script de rendimiento:

  1. Abre la grabadora desde tu Chrome
  2. Ingrese un nombre de prueba en el campo superior
  3. Comienza a grabar haciendo clic en el botón de grabar, en forma de círculo, y realiza las acciones web que quieras grabar. Todas sus solicitudes serán capturadas. Blazemeter Chrome Extension también admite la grabación de tráfico HTTPS.
  4. Después de terminar de grabar, haga clic en el botón de detener, en forma de cuadrado. También puedes pausar tu grabación y luego reanudarla, así como editarla, en formato .jmx o JSON, o en la nube.
  5. Exporte su grabación: para ejecutar la prueba en JMeter, exporte al formato .jmx haciendo clic en el botón .jmx. Para ejecutar la prueba en BlazeMeter, haga clic en “reproducir”.

Para obtener más información, consulte [aquí] 3.

Grabación de guiones con BadBoy #

Otra útil herramienta de grabación de terceros es BadBoy. Sin embargo, solo funciona para el sistema operativo Windows.

Para crear un nuevo script de rendimiento:

  1. Instala BadBoy aquí

  2. Ingrese la URL bajo prueba en la barra de direcciones

  3. Presiona el botón de grabar, con forma de círculo rojo y realiza las acciones que deseas capturar.

  4. Exporte su script a JMeter - Archivo -> Exportar a JMeter

Para obtener más información, consulte aquí.

El uso de una grabadora de scripts de rendimiento es una excelente manera de evitar tareas rutinarias y aún así obtener los mejores scripts de prueba. Después de grabar, configure la prueba para la cantidad de usuarios virtuales que desea probar, así como configuraciones de prueba adicionales, ejecute su prueba y analice los resultados para identificar errores y cuellos de botella y caracterizar tendencias que le muestren la salud de su sistema.
