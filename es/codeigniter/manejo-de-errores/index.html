CodeIgniter le permite generar informes de errores en sus aplicaciones utilizando las funciones que se describen a continuación. Además, tiene una clase de registro de errores que permite que los mensajes de error y depuración se guarden como archivos de texto.

registro_mensaje() #

Esta función le permite escribir mensajes en sus archivos de registro. Debe proporcionar uno de los tres “niveles” en el primer parámetro, indicando qué tipo de mensaje es (depuración, error, información), con el mensaje en sí mismo en el segundo parámetro.

Ejemplo:

if ($some_var == "") {
    log_message('error', 'Some variable did not contain a value.'); 
} 
else {
    log_message('debug', 'Some variable was correctly set'); 
}

log_message('info', 'The purpose of some variable is to provide some value.');

Sintaxis #

log_message($level, $message);

Parámetros:

  • $nivel (cadena) – Nivel de registro: ’error’, ‘depuración’ o ‘información’
  • $mensaje (cadena) – Mensaje para registrar

Tipo de devolución: nulo

Hay tres tipos de mensajes: #

  • Error de mensajes. Estos son errores reales, como errores de PHP o errores de usuario.
  • Mensajes de depuración. Estos son mensajes que ayudan en la depuración. Por ejemplo, si se ha inicializado una clase, puede registrar esto como información de depuración.
  • Mensajes informativos. Estos son los mensajes de menor prioridad, simplemente brindan información sobre algún proceso. CodeIgniter no genera de forma nativa ningún mensaje de información, pero es posible que desee hacerlo en su aplicación.

Nota: Para que el archivo de registro se escriba realmente, los “registros” la carpeta debe ser escribible. Además, debe establecer el “umbral” para iniciando sesión en application/config/config.php. Por ejemplo, es posible que solo desee que se registren los mensajes de error y no los otros dos tipos. Si lo establece en cero, el registro se desactivará.

mostrar_error() #

Esta función mostrará el mensaje de error que se le proporcionó utilizando la siguiente plantilla de error:

Ruta - aplicación/errores/error_general.php

El parámetro opcional $status_code determina qué código de estado HTTP debe enviarse con el error.

Sintaxis #

show_error($message, $status_code, $heading = 'An Error Was Encountered')

Parámetros:

  • $mensaje (mixto) – Mensaje de error
  • $status_code (int) – Código de estado de respuesta HTTP
  • $título (cadena) – Título de página de error

Tipo de devolución: nulo

Fuente #

  1. show_error en codeigniter.com

mostrar_404() #

Esta función mostrará el mensaje de error 404 que se le proporcionó utilizando la siguiente plantilla de error:

Ruta - aplicación/errores/error_404.php

La función espera que la cadena que se le pasa sea la ruta del archivo a la página que no se encuentra. Tenga en cuenta que CodeIgniter muestra automáticamente mensajes 404 si no se encuentran los controladores.

CodeIgniter registra automáticamente cualquier llamada show_404(). Establecer el segundo parámetro opcional en FALSO omitirá el registro.

Sintaxis #

show_404($page = '', $log_error = TRUE)

Parámetros:

  • $página (cadena) – Cadena URI
  • $log_error (bool) – Si registrar el error

Tipo de devolución: nulo

Fuente #

  1. show_404 en codeigniter.com
