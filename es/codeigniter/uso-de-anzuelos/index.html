Puntos de gancho #

pre_sistema #

Llamado muy temprano durante la ejecución del sistema. En este punto, solo se han cargado la clase de puntos de referencia y ganchos. No se ha producido ningún enrutamiento u otros procesos.

pre_controlador #

Llamado inmediatamente antes de que se llame a cualquiera de sus controladores. Se han realizado todas las clases base, enrutamiento y comprobaciones de seguridad.

post_controlador_constructor #

Llamado inmediatamente después de que se crea una instancia de su controlador, pero antes de que ocurra cualquier llamada de método.

post_controlador #

Llamado inmediatamente después de que su controlador se ejecute por completo.

display_override #

Anula el método _display(), utilizado para enviar la página finalizada al navegador web al final de la ejecución del sistema. Esto le permite utilizar su propia metodología de visualización. Tenga en cuenta que deberá hacer referencia al superobjeto de CI con $this->CI =& get_instance() y luego los datos finalizados estarán disponibles llamando a $this->CI->output->get_output() .

cache_override #

Le permite llamar a su propio método en lugar del método _display_cache() en la biblioteca de salida. Esto le permite utilizar su propio mecanismo de visualización de caché.

post_sistema #

Se llama después de que la página renderizada final se envía al navegador, al final de la ejecución del sistema después de que los datos finalizados se envían al navegador.

Ejemplo de precontrolador Hook usando CodeIgniter #

En la carpeta application/hooks, crea un archivo con el nombre Blocker.php y pega el siguiente código.

<?php
class Blocker {

    function Blocker(){
    }
    
    /**
     * This function used to block the every request except allowed ip address
     */
    function requestBlocker(){
        
        if($_SERVER["REMOTE_ADDR"] != "49.248.51.230"){
            echo "not allowed";
            die;
        }
    }
}
?>

En application/config/hooks.php, declara el siguiente gancho.

$hook['pre_controller'] = array(
        'class'    => 'Blocker',
        'function' => 'requestBlocker',
        'filename' => 'Blocker.php',
        'filepath' => 'hooks',
        'params'   => ""
);

En application/config/config.php, establezca el siguiente valor como verdadero

Habilitación de ganchos #

La función de ganchos se puede habilitar/deshabilitar globalmente configurando el siguiente elemento en el archivo application/config/config.php:

$config['enable_hooks'] = TRUE;

Definición de un gancho #

Los ganchos se definen en el archivo application/config/hooks.php. cada gancho se especifica como una matriz con este prototipo

$hook['pre_controller'] = array(
        'class'    => 'MyClass',
        'function' => 'Myfunction',
        'filename' => 'Myclass.php',
        'filepath' => 'hooks',
        'params'   => array('beer', 'wine', 'snacks')
);

El índice de la matriz se correlaciona con el nombre del punto de gancho en particular que desea usar. En el ejemplo anterior, el punto de enganche es pre_controller. Una lista de puntos de gancho se encuentra a continuación. Los siguientes elementos deben definirse en su matriz de ganchos asociativos:

clase El nombre de la clase que desea invocar. Si prefiere utilizar una función de procedimiento en lugar de una clase, deje este elemento en blanco.

función El nombre de la función (o método) que desea llamar.

nombre de archivo El nombre de archivo que contiene su clase/función.

ruta de archivo El nombre del directorio que contiene su secuencia de comandos.

params Cualquier parámetro que desee pasar a su secuencia de comandos. Este artículo es opcional.

Definición de un gancho #

Los ganchos se definen en el archivo application/config/hooks.php. Cada gancho se especifica como una matriz con este prototipo:

$hook['pre_controller'] = array(
      'class'    => 'MyClass',
      'function' => 'Myfunction',
      'filename' => 'Myclass.php',
      'filepath' => 'hooks',
      'params'   => array('bread', 'wine', 'butter')
);
  • CLASS- La clase que desea invocar si es un código de procedimiento déjelo en blanco.
  • FUNCIÓN- El nombre de la función que desea llamar.
  • FILENAME- El nombre del archivo que contiene su clase/función.
  • FILEPATH- Ubicación del archivo gancho.
  • PARAMS-Parámetro adicional si es necesario, es opcional
