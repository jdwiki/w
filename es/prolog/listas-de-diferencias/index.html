Listas de diferencias

  • Uso básico
  • Evaluar una expresión aritmética

En esta página

  • Uso básico
  • Evaluar una expresión aritmética

Listas de diferencias en Prolog denota el concepto de conocer la estructura de una lista hasta cierto punto. El resto de la lista se puede dejar sin vincular hasta la evaluación completa de un predicado. Una lista donde se desconoce su final se denomina lista abierta, terminada por un agujero. Esta técnica es especialmente útil para validar sintaxis o gramáticas complejas.

Las conocidas gramáticas de cláusulas definidas (DCG) utilizan listas de diferencias para operar bajo el capó.

Uso básico #

Consideremos el predicado sumDif/2, verificado si la estructura de una lista coincide con varias restricciones. El primer término representa la lista a analizar y el segundo término otra lista que contiene la parte de la primera lista que es desconocida para nuestras restricciones.

Para la demostración, sumDif/2 reconoce una expresión aritmética para sumar n enteros.

sumDif([X, +|OpenList], Hole) :-
    integer(X),
    sumDif(OpenList, Hole).

Sabemos que el primer elemento de la lista a validar es un número entero, aquí ilustrado por X, seguido del símbolo de la suma (+). El resto de la lista que aún debe procesarse más adelante (OpenList) se deja sin validar en ese nivel. Hole representa la parte de la lista que no necesitamos validar.

Demos otra definición del predicado sumDif/2 para completar la validación de la expresión aritmética:

sumDif([X|Hole], Hole) :-
    integer(X).

Esperamos un entero llamado X directamente al comienzo de la lista abierta. Curiosamente, el resto de la lista ‘Agujero’ se desconoce y ese es el propósito de las Listas de diferencias: la estructura de la lista se conoce hasta cierto punto.

Finalmente, la pieza que falta viene cuando se evalúa una lista:

?- sumDif([1,+,2,+,3], []).
true

Aquí es cuando se usa el predicado que menciona el final de la lista, aquí [], indica que la lista no contiene elementos adicionales.

Evaluar una expresión aritmética #

Definamos una gramática que nos permita realizar sumas, multiplicaciones con el uso de paréntesis. Para agregar más valor a este ejemplo, vamos a calcular el resultado de la expresión aritmética. Resumen de la gramática:

expresión → veces expresión → veces ‘+’ expresión veces → elemento veces → elemento ‘*’ veces elemento → “entero” elemento → ‘(’ expresión ‘)’

Todos los predicados tienen una aridad de 3, porque necesitan lista abierta, el hueco y el valor de la expresión aritmética.

expression(Value, OpenList, FinalHole) :-
    times(Value, OpenList, FinalHole).

expression(SumValue, OpenList, FinalHole) :-
    times(Value1, OpenList, ['+'|Hole1]),
    expression(Value2, Hole1, FinalHole),
    plus(Value1, Value2, SumValue).

times(Value, OpenList, FinalHole) :-
    element(Value, OpenList, FinalHole).

times(TimesValue, OpenList, FinalHole) :-
    element(Value1, OpenList, ['*'|Hole1]),
    times(Value2, Hole1, FinalHole),
    TimesValue is Value1 * Value2.

element(Value, [Value|FinalHole], FinalHole) :-
    integer(Value).

element(Value, ['('|OpenList], FinalHole) :-
    expression(Value, OpenList, [')'|FinalHole]).

Para explicar correctamente el principio de agujeros y cómo se calcula el valor, tomemos la segunda cláusula expresión:

expression(SumValue, OpenList, FinalHole) :-
    times(Value1, OpenList, ['+'|Hole1]),
    expression(Value2, Hole1, FinalHole),
    plus(Value1, Value2, SumValue).

La lista abierta se indica con el predicado OpenList. El primer elemento a validar es lo que viene antes del símbolo de suma (+). Cuando se valida el primer elemento, le sigue directamente el símbolo de adición y la continuación de la lista, denominada Hole1. Sabemos que Agujero1 es el siguiente elemento a validar y puede ser otra expresión, por lo que Agujero1 es entonces el término dado al predicado expresión.

El valor siempre se representa en el primer término. En esta cláusula, se define por la suma de Valor1 (todo lo que está antes del símbolo de suma) y Valor2 (todo lo que está después del símbolo de suma).

Finalmente, la expresión an puede ser evaluada.

?- expression(V, [1,+,3,*,'(',5,+,5,')'], []).
V = 31
