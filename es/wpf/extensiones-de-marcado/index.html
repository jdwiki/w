Parámetros #

Método Descripción
Proporcionar valor La clase MarkupExtension solo tiene un método que debe anularse, el analizador XAML luego usa el valor proporcionado por este método para evaluar el resultado de la extensión de marcado.

Se puede implementar una extensión de marcado para proporcionar valores para propiedades en el uso de un atributo, propiedades en el uso de un elemento de propiedad o ambos.

Cuando se usa para proporcionar un valor de atributo, la sintaxis que distingue una secuencia de extensión de marcado en un procesador XAML es la presencia de llaves de apertura y cierre ({ y }). A continuación, el tipo de extensión de marcado se identifica mediante el token de cadena que sigue inmediatamente a la llave de apertura.

Cuando se usa en la sintaxis de elementos de propiedad, una extensión de marcado es visualmente igual que cualquier otro elemento usado para proporcionar un valor de elemento de propiedad: una declaración de elemento XAML que hace referencia a la clase de extensión de marcado como un elemento, encerrado entre corchetes angulares (<>).

Para obtener más información, visite https://msdn.microsoft.com/en-us/library/ms747254(v=vs.110).aspx

Extensión de marcado utilizada con IValueConverter #

Uno de los mejores usos para las extensiones de marcado es facilitar el uso de IValueConverter. En el ejemplo a continuación, BoolToVisibilityConverter es un convertidor de valores, pero dado que es independiente de la instancia, se puede usar sin las molestias normales de un convertidor de valores con la ayuda de la extensión de marcado. En XAML solo usa

Visibility="{Binding [BoolProperty], Converter={[xmlns]:BoolToVisibilityConverter}}"

y puede establecer la visibilidad del elemento en valor bool.

public class BoolToVisibilityConverter : MarkupExtension, IValueConverter
    {
        public object Convert(object value, Type targetType, object parameter, CultureInfo culture)
        {
            if (value is bool)
                return (bool)value ? Visibility.Visible : Visibility.Collapsed;
            else
                return Visibility.Collapsed;
        }

        public object ConvertBack(object value, Type targetType, object parameter, CultureInfo culture)
        {
            if (value is Visibility)
            {
                if ((Visibility)value == Visibility.Visible)
                    return true;
                else
                    return false;
            }
            else
                return false;
        }

        public override object ProvideValue(IServiceProvider serviceProvider)
        {
            return this;
        }
    }

Extensiones de marcado definidas por XAML #

Hay cuatro extensiones de marcado predefinidas en XAML:

x:Type proporciona el objeto Type para el tipo nombrado. Esta función se usa con mayor frecuencia en estilos y plantillas.

<object property="{x:Type prefix:typeNameValue}" .../>

x:Static produce valores estáticos. Los valores provienen de entidades de código de tipo de valor que no son directamente del tipo del valor de una propiedad de destino, pero que se pueden evaluar para ese tipo.

<object property="{x:Static prefix:typeName.staticMemberName}" .../>

x:Null especifica nulo como un valor para una propiedad y puede usarse para atributos o valores de elementos de propiedad.

<object property="{x:Null}" .../>

x:Array proporciona compatibilidad para la creación de matrices generales en la sintaxis XAML, para los casos en los que no se utiliza deliberadamente la compatibilidad con la recopilación proporcionada por los elementos base de WPF y los modelos de control.

<x:Array Type="typeName">
  arrayContents
</x:Array>
