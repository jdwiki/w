Configuración básica desde GitHub #

Este ejemplo explica cómo configurar CoreNLP desde el [repositorio de GitHub] (https://github.com/stanfordnlp/CoreNLP). El código de GitHub tiene funciones más nuevas que el lanzamiento oficial, pero puede ser inestable. Este ejemplo lo guiará a través de la descarga, creación y ejecución de una simple invocación de línea de comandos de CoreNLP.

Requisitos previos:

  • Java 8 o más reciente.
  • Hormiga Apache
  • Git
  • Para el ejemplo: Bash o shell similar, y wget o curl

Pasos:

  1. Clone el repositorio CoreNLP Git:

    git clone git@github.com:stanfordnlp/CoreNLP.git

  2. Ingrese al directorio de CoreNLP:

    cd CoreNLP

  3. Genere el proyecto en un archivo jar independiente. La forma más fácil de hacer esto es con:

    ant jar

  4. Descarga los últimos modelos.

    wget http://nlp.stanford.edu/software/stanford-corenlp-models-current.jar

O usando curl (lo que obtienes por defecto en macOS):

   curl -O http://nlp.stanford.edu/software/stanford-corenlp-models-current.jar

  1. Configure su classpath. Si está utilizando un IDE, debe configurar el classpath en su IDE.

    export CLASSPATH="$CLASSPATH:javanlp-core.jar:stanford-corenlp-models-current.jar";
for file in `find lib -name "*.jar"`; do export CLASSPATH="$CLASSPATH:`realpath $file`"; done

    If you’ll be using CoreNLP frequently, this is a useful line to have in your ~/.bashrc (or equivalent) file, replacing the directory /path/to/corenlp/ with the appropriate path to where you unzipped CoreNLP (3 replacements):

    export CLASSPATH="$CLASSPATH:/path/to/corenlp/javanlp-core.jar:/path/to/corenlp/stanford-corenlp-models-current.jar";
for file in `find /path/to/corenlp/lib -name "*.jar"`; do export CLASSPATH="$CLASSPATH:`realpath $file`"; don

  2. ¡Pruébelo! Por ejemplo, lo siguiente creará un archivo de texto simple para anotar y ejecutar CoreNLP sobre este archivo. La salida se guardará en input.txt.out como un archivo JSON. Tenga en cuenta que CoreNLP requiere bastante memoria. Debe darle al menos 2 GB (-mx2g) en la mayoría de los casos.

    echo "the quick brown fox jumped over the lazy dog" > input.txt
java -mx2g edu.stanford.nlp.pipeline.StanfordCoreNLP -outputFormat json -file input.txt

Configuración básica del lanzamiento oficial #

Este ejemplo repasa cómo configurar CoreNLP desde el último lanzamiento oficial. Este ejemplo lo guiará a través de la descarga del paquete y la ejecución de una simple invocación de línea de comandos de CoreNLP.

Requisitos previos:

  • Java JVM 8. El comando java -version debería completarse correctamente con una línea como: java version “1.8.0_92”.
  • Herramienta de cremallera
  • Para el ejemplo: Bash o shell similar, y wget

Pasos:

  1. Descargue el archivo zip de CoreNLP en: http://stanfordnlp.github.io/CoreNLP/index.html#download:

    wget http://nlp.stanford.edu/software/stanford-corenlp-full-2015-12-09.zip

  2. Descomprima el lanzamiento:

    unzip stanford-corenlp-full-2015-12-09.zip

  3. Ingrese al directorio recién descomprimido:

    cd stanford-corenlp-full-2015-12-09

  4. Configure su classpath. Si está utilizando un IDE, debe configurar el classpath en su IDE.

    for file in `find . -name "*.jar"`; do export CLASSPATH="$CLASSPATH:`realpath $file`"; done

    If you’ll be using CoreNLP frequently, this is a useful line to have in your ~/.bashrc (or equivalent) file, replacing the directory /path/to/corenlp/ with the appropriate path to where you unzipped CoreNLP:

    for file in `find /path/to/corenlp/ -name "*.jar"`; do export CLASSPATH="$CLASSPATH:`realpath $file`"; done

  5. ¡Pruébelo! Por ejemplo, lo siguiente creará un archivo de texto simple para anotar y ejecutar CoreNLP sobre este archivo. La salida se guardará en input.txt.out como un archivo JSON. Tenga en cuenta que CoreNLP requiere bastante memoria. Debe darle al menos 2 GB (-mx2g) en la mayoría de los casos.

    echo "the quick brown fox jumped over the lazy dog" > input.txt
java -mx2g edu.stanford.nlp.pipeline.StanfordCoreNLP -outputFormat json -file input.txt
