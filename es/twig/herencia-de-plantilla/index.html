Usar una plantilla base #

base.ramita.html #

<!DOCTYPE html>
<html>
    <head>
        <title>{{ title | default('Hello World') }}</title>
        <link rel="stylesheet" type="text/css" href="theme.css">
        {% block css %}
        {% endblock %}
    </head>
    <body>
    {% block body %}
        <nav>
        {% block navigation %}
            <a href="#">Link</a>
            <a href="#">Link</a>
            <a href="#">Link</a>
        {% endblock navigation %}
        </nav>
        <section id="container">
            <section id="content">
            {% block content %}
            <p>
            Lorem ipsum dolor sit amet.
            </p>
        </section>
        {% endblock content %}
    </section>
    {% endblock body %}
    </body>
</html>

página.ramita.html

{% extends base.twig.html %}
{% block navigation %}
<a href="page2.html">Page 2</a>
<a href="page3.html">Page 3</a>
<a href="page4.html">Page 4</a>
{% endblock %}
{% block content %}
This is my first page
{% endblock content %}

Cambiar el contenido de una plantilla incluida #

artículo.ramita.html #

<article>
    <h1>{{ article.title }}</h1>
    {% block content %}
    <p>{{ article.content }}</p>
    {% endblock %}
</article>

articulos.twig.html #

{# use default template for article #}
{% for article in articles %}
    {% include "article.twig.html" %}
{% endfor %}

{# use custom template for article #}
{% for article in articles %}
    {% embed "article.twig.html" %}
        {% block content %}
            <img src="{{ article.image }}" alt="{{ article.title }}" />
            {{ parent() }}
        {% endblock %}
    {% endembed %}
{% endfor %}

Herencia de variables #

padre.html.ramita #

<!DOCTYPE html>
<html>
    <head>
        <title>{{ title_variable | default('Normal Page') }}</title>
    </head>
    <body>
         <h1>This is the {{ title_variable }} Page</h1>    
    </body>
</html>

niño.html.ramita #

{% extends "parent.html.twig" %}
{% set title_variable = "Child" %}

Comparación de Incluir, Extender, Usar, Macro, Incrustar #

Hay varios tipos de herencia y reutilización de código en Twig:

Incluir #

El objetivo principal es la reutilización de código. Considere usar header.html.twig & footer.html.twig dentro de base.html.twig como ejemplo.

header.html.ramita

<nav>
   <div>Homepage</div>
   <div>About</div>
</nav>

base.html.ramita

{% include 'header.html.twig' %}
<main>{% block main %}{% endblock %}</main>

Extiende #

El objetivo principal es la herencia vertical. Considere extender base.html.twig dentro de homepage.html.twig y about.html.twig como ejemplo.

base.html.ramita

{% include 'header.html.twig' %}
<main>{% block main %}{% endblock %}</main>

página de inicio.html.twig

{% extends 'base.html.twig' %}

{% block main %}
<p>You are at the homepage</p>
{% endblock %}

acerca de.html.twig

{% extends 'base.html.twig' %}

{% block main %}
<p>You are at the about page</p>
{% endblock %}

Usar #

El objetivo principal es la reutilización horizontal. Considere usar sidebar.product.html.twig dentro de single.product.html.twig (extiende product.layout.html.twig) y single.service.html.twig (extiende `service.layout. html.page’) páginas. (es como macros, pero para bloques)

barra lateral.html.twig

<aside>{% block sidebar %}{% endblock %}</aside>

único.producto.html.ramita

{% extends 'product.layout.html.twig' %}

{% use 'sidebar.html.twig' %}
{% block main %}
<p>You are at the product page for product number 123</p>
{% endblock %}

único.servicio.html.twig

{% extends 'service.layout.html.twig' %}

{% use 'sidebar.html.twig' %}
{% block main %}
<p>You are at the service page for service number 456</p>
{% endblock %}

#macro El objetivo principal es tener marcado reutilizable en muchas plantillas con variables. Considere una función que obtiene algunas variables y genera algunas marcas.

formulario.html.ramita

{% entrada de macro (nombre, valor, tipo) %} <input type="{{ type|default(’text’) }}" name="{{ name }}" value="{{ value|e }}" }}" /> {% endmacro%} #

perfil.servicio.html.twig

{% import "forms.html.twig" as forms %}

<div>{{ forms.input('username') }}</div>

Incrustar #

El objetivo principal es la anulación de bloques. Tiene la funcionalidad de Usar e Incluir juntos. Considere incrustar pagination.html.twig en product.table.html.twig & service.table.html.twig.

paginación.html.twig

<div>
    <div>{% block first %}{% endblock %}</div>
    {% for i in (min + 1)..(max - 1) %}
        <div>{{ i }}</div>
    {% endfor %}
    <div>{% block last %}{% endblock %}</div>
</div>

producto.tabla.html.ramita

{% set min, max = 1, products.itemPerPage %}

{% embed 'pagination.html.twig' %}
    {% block first %}First Product Page{% endblock %}
    {% block last %}Last Product Page{% endblock %}
{% endembed %}

servicio.table.html.twig

{% set min, max = 1, services.itemPerPage %}

{% embed 'pagination.html.twig' %}
    {% block first %}First Service Page{% endblock %}
    {% block last %}Last Service Page{% endblock %}
{% endembed %}

Tenga en cuenta que el archivo incrustado (pagination.html.twig aquí) tiene acceso al contexto actual (min, max variables aquí). También puede pasar variables adicionales al archivo incrustado:

paginación.html.twig

<p>{{ count }} items</p>
<div>
    <div>{% block first %}{% endblock %}</div>
    {% for i in (min + 1)..(max - 1) %}
        <div>{{ i }}</div>
    {% endfor %}
    <div>{% block last %}{% endblock %}</div>
</div>

producto.tabla.html.ramita

{% set min, max = 1, products|length %}

{% embed 'pagination.html.twig' with {'count': products|length } %}
    {% block first %}First Product Page{% endblock %}
    {% block last %}Last Product Page{% endblock %}
{% endembed %}
