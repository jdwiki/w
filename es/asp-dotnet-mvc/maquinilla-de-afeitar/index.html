¿Qué es Razor?

Razor es una sintaxis de marcado que le permite incrustar código basado en servidor (Visual Basic y C#) en páginas web.

El código basado en servidor puede crear contenido web dinámico sobre la marcha, mientras se escribe una página web en el navegador. Cuando se llama a una página web, el servidor ejecuta el código basado en el servidor dentro de la página antes de devolver la página al navegador. Al ejecutarse en el servidor, el código puede realizar tareas complejas, como acceder a bases de datos.

Sintaxis #

  • @{ … }
  • @nombre de la variable
  • @(nombre de la variable)
  • @por(…){ }
  • @(Expresión Explícita)
  • @* comenta *@

ASP.NET Razor incluye motores de visualización para C# y VB.

El motor de visualización de C# procesa archivos con una extensión .cshtml, mientras que el motor de visualización de VB funciona con archivos .vbhtml.

Sintaxis básica #

El código Razor se puede insertar en cualquier lugar dentro del código HTML. Los bloques de código Razor están encerrados en @{ ... }. Las variables y funciones en línea comienzan con @. El código dentro de los corchetes de Razor sigue las reglas normales de C# o VB.

Declaración de una sola línea:

@{ var firstNumber = 1; }

Bloque de código multilínea:

@{
    var secondNumber = 2;
    var total = firstNumber + secondNumber;
}

Usando una variable en línea:

<h1>The total count is @total</h1>

Usando una variable en línea explícitamente:

<h2>Item@(item.Id)</h2>

Para este ejemplo en particular, no podremos usar la sintaxis implícita porque Item@item.Id parece un correo electrónico y Razor lo representará como tal.

Adjunte el código dentro de las declaraciones de flujo de control:

<h1>Start with some HTML code</h1>

@for (int i = 0; i < total; i++){
    Console.Write(i);
}

<p>Mix in some HTML code for fun!</p>
<p>Add a second paragraph.</p>

@if (total > 3)
{
    Console.Write("The total is greater than 3");
}
else
{
    Console.Write("The total is less than 3");
}

Esta misma sintaxis se usaría para todas las declaraciones como for, foreach, while, if, switch, etc.

Agregando código dentro del código:

@if (total > 3)
{
    if(total == 10)
    {
        Console.Write("The total is 10")
    }
}

Tenga en cuenta que no necesita escribir @ en el segundo if. Después del código, puede escribir otro código detrás del código existente.

Si desea agregar código después de un elemento HTML, debe escribir un @.

Añadir comentarios #

Razor tiene su propia sintaxis de comentarios que comienza con @* y termina con *@.

Comentario en línea:

<h1>Comments can be @*hi!*@ inline</h1>

Comentario de varias líneas:

@* Comments can spread
   over multiple
   lines *@

Comentario HTML

También puede usar la sintaxis de comentarios HTML normal que comienza con <!-- y termina con --> en las vistas de Razor. Pero a diferencia de otros comentarios, el código Razor dentro de un comentario HTML aún se ejecuta normalmente.

@{
    var hello = "Hello World!"; 
} 
<!-- @hello -->

El ejemplo anterior produce la siguiente salida HTML:

<!-- Hello World! -->

Comentarios dentro de un bloque de código:

@{
    // This is a comment
    var Input = "test";
}

Agregar un atributo personalizado con - (guión) en el nombre #

Si necesita agregar un atributo a través de razor que tiene un - (guión) en el nombre, no puede simplemente hacerlo

@Html.DropDownListFor(m => m.Id, Model.Values, new { @data-placeholder = "whatever" })

no se compilará. Los atributos data-* son válidos y comunes en html5 para agregar valores adicionales a los elementos.

Sin embargo, lo siguiente funcionará

@Html.DropDownListFor(m => m.Id, Model.Values, new { @data_placeholder = "whatever" })

ya que “_” se reemplaza con “-” cuando se representa.

Esto funciona bien ya que los guiones bajos no son aceptables en los nombres de atributos en html.

Crear clases y métodos en línea usando @funciones #

El uso de la palabra clave Razor @functions brinda la capacidad de introducir clases y métodos para uso en línea dentro de un archivo Razor:

@functions
{
    string GetCssClass(Status status)
    {
        switch (status)
        {
            case Status.Success:
                return "alert-success";
            case Status.Info:
                return "alert-info";
            case Status.Warning:
                return "alert-warning";
            case Status.Danger:
            default:
                return "alert-danger";
         }
     }
}


<label class="alert @GetCssClass(status)"></label>

Lo mismo se puede hacer para las clases:

@functions
{
    class Helpers
    {
        //implementation
    }
}

Mostrar HTML dentro del bloque de código Razor #

Mientras esté dentro de un bloque de código Razor, el navegador solo reconocerá el código HTML si se escapa el código.

Use @: para una sola línea:

@foreach(int number in Model.Numbers)
{
    @:<h1>Hello, I am a header!</h1>
}

Use <text> ... </text> para varias líneas:

@{
    var number = 1;

    <text>
        Hello, I am text
        <br / >
        Hello, I am more text!
    </text>
}

Tenga en cuenta que Razor, cuando está dentro de un bloque de código, comprenderá las etiquetas HTML. Por lo tanto, no es necesario agregar la etiqueta text alrededor de las etiquetas HTML (aunque sigue siendo correcto), como por ejemplo:

@{
    var number = 1;
    <text>
        <div>
            Hello, I am text
            <br / >
            Hello, I am more text!
        </div>
    </text>
}

Escapando @ carácter #

En muchos casos, el analizador Razor es lo suficientemente inteligente como para determinar cuándo el signo @ debe usarse como parte del código, en lugar de ser parte de algo como una dirección de correo electrónico. En el siguiente ejemplo, no es necesario escapar del signo @:

<p>Reach out to us at contact@mail.com</p>

Sin embargo, en algunos casos, el uso del signo @ es más ambiguo y debe escaparse explícitamente con @@, como en el siguiente ejemplo:

<p>Join us @@ Stack Overflow!</p>

Alternativamente, podemos usar un carácter @ codificado en HTML

<p>Join us &#64; Stack Overflow!</p>

Compartir @helpers en las vistas #

@Helpers podría compartirse entre vistas.

Deben crearse en la carpeta App_Code

@helper CreatePrimaryBootstrapButton(string label)
{
    <button type="button" class="btn btn-primary">@label</button>
}

//call

@MenuHelpers.CreatePrimaryBootstrapButton("my button")

Los globales @Url y @Html no están disponibles por defecto en el @Helper definido en App_code. Puede agregarlos de la siguiente manera (para cada .cshtml en su carpeta App_code)

@*  Make @Html and @Url available *@
@functions
{
    private new static HtmlHelper<object> Html
    {
        get { return ((WebViewPage)CurrentPage).Html; }
    }

    private static UrlHelper Url
    {
        get { return ((WebViewPage)CurrentPage).Url; }
    }
 }

Editor de plantillas #

Las plantillas del editor son una buena manera de reutilizar el código de Razor. Puede definir plantillas de editor como vistas parciales de Razor y luego usarlas en otras vistas.

Las plantillas del editor generalmente existen en la carpeta Views/Shared/EditorTemplates/, aunque también se pueden guardar en la carpeta Views/ControllerName/EditorTemplates/. El nombre de la vista suele ser el nombre del objeto para el que desea usar la plantilla, como <tipo>.cshtml.

Aquí hay una plantilla de editor simple para DateTime:

@model DateTime
<div>
   <span>
      @Html.TextBox("", Model.ToShortDateString(), new { data_date_picker="true" })
   </span>
</div>

Guarde el archivo como Vistas/Compartido/EditorTemplate/DateTime.cshtml.

Luego, use EditorFor para llamar a este código de plantilla en otra vista:

@Html.EditorFor(m => m.CreatedDate)

También hay un atributo UIHint para especificar el nombre del archivo:

public class UiHintExampleClass
{
    [UIHint("PhoneNumber")]
    public string Phone { get; set; }
}

Defina esta plantilla de número de teléfono en Views/Shared/EditorTemplates/PhoneNumber.cshtml.

Las plantillas del editor también se pueden definir para tipos personalizados.

Aquí hay un tipo personalizado llamado SubModel:

public class SubModel
{
    public Guid Id { get; set;} 
    public string FirstName { get; set; }
    public string LastName { get; set; }
}

public class Model
{
    public Guid Id { get; set; }
    public DateTime Created {get; set; }
    
    public SubModel SubModel{get; set; }
}

Este es el EditorTemplate para SubModel:

@model SubModel
<div class="form-group">
    @Html.LabelFor(m => m.FirstName)
    @Html.TextBoxFor(m => m.FirstName)
</div>
<div class="form-group">
    @Html.LabelFor(m => m.LastName)
    @Html.TextBoxFor(m => m.LastName)
</div>

Ahora, View for Model simplemente se convierte en:

@model Model
@Html.EditorFor(m => m.CreatedDate)
@Html.EditorFor(m => m.SubModel, new { @Prefix = "New"}) 
@* the second argument is how you can pass viewdata to your editor template*@

Pasar contenido de Razor a un @helper #

Envíe una parte de Razor a un @helper, por ejemplo, un div HTML.

@helper WrapInBox(Func<Object, HelperResult> content)
{
    <div class="box">@content(null) </div>
}

//call 
@WrapInBox(@<div>
                I'm a inner div
            </div>)
