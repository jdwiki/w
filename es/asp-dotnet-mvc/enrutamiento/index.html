El enrutamiento es cómo ASP.NET MVC hace coincidir un URI con una acción. El módulo de enrutamiento es responsable de asignar las solicitudes entrantes del navegador a acciones particulares del controlador MVC.

MVC 5 admite un nuevo tipo de enrutamiento, llamado enrutamiento de atributos. Como su nombre lo indica, el enrutamiento de atributos usa atributos para definir rutas. El enrutamiento de atributos le brinda más control sobre los URI en su aplicación web.

Enrutamiento de atributos en MVC #

Junto con la forma clásica de definición de ruta, MVC WEB API 2 y luego MVC 5, los marcos introdujeron el “enrutamiento de atributos”:

public class RouteConfig
{
    public static void RegisterRoutes(RouteCollection routes)
    {
        routes.IgnoreRoute("{resource}.axd/{*pathInfo}");
 
        // This enables attribute routing and must go  before other routes are added to the routing table.
        // This makes attribute routes have higher priority
        routes.MapMvcAttributeRoutes();  
    }
}

Para rutas con el mismo prefijo dentro de un controlador, puede establecer un prefijo común para métodos de acción completos dentro del controlador utilizando el atributo RoutePrefix.

[RoutePrefix("Custom")]
public class CustomController : Controller
{
    [Route("Index")]
    public ActionResult Index()
    {
        ...
    }
}

RoutePrefix es opcional y define la parte de la URL que se antepone a todas las acciones del controlador.

Si tiene varias rutas, puede establecer una ruta predeterminada capturando la acción como parámetro y luego aplicarla para todo el controlador, a menos que se haya definido un atributo Route específico en ciertos métodos de acción que anulan la ruta predeterminada.

[RoutePrefix("Custom")]
[Route("{action=index}")]
public class CustomController : Controller
{
    public ActionResult Index()
    {
        ...
    }

    public ActionResult Detail()
    {
        ...
    }
}

Ruta general #

Supongamos que queremos tener una ruta que permita un número ilimitado de segmentos como este:

  • http://example.com/Products/ (ver todos los productos)
  • http://ejemplo.com/Productos/TI
  • http://example.com/Products/IT/Laptops
  • http://example.com/Products/IT/Laptops/Ultrabook
  • http://example.com/Products/IT/Laptops/Ultrabook/Asus
  • etc.

Necesitaríamos agregar una ruta, normalmente al final de la tabla de rutas porque esto probablemente captaría todas las solicitudes, así:

routes.MapRoute("Final", "Route/{*segments}",
      new { controller = "Product", action = "View" });

En el controlador, una acción que podría manejar esto podría ser:

public void ActionResult View(string[] segments /* <- the name of the parameter must match the name of the route parameter */)
{
    // use the segments to obtain information about the product or category and produce data to the user
    // ...
}

Conceptos básicos de enrutamiento #

Agregar ruta personalizada en Mvc #

El usuario puede agregar una ruta personalizada, asignando una URL a una acción específica en un controlador. Esto se utiliza con fines de optimización de motores de búsqueda y hace que las URL sean legibles.

routes.MapRoute(
  name: "AboutUsAspx", // Route name
  url: "AboutUs.aspx",  // URL with parameters
  defaults: new { controller = "Home", action = "AboutUs", id = UrlParameter.Optional }  // Parameter defaults
);

Ruta general para habilitar el enrutamiento del lado del cliente #

Es una buena práctica codificar el estado de la aplicación de página única (SPA) en la URL:

my-app.com/admin-spa/users/edit/id123

Esto permite guardar y compartir el estado de la aplicación.
Cuando el usuario ingresa la url en la barra de direcciones del navegador y presiona ingresar, el servidor debe ignorar la parte del lado del cliente de la url solicitada. Si sirve su SPA como una vista de Razor renderizada (resultado de la acción del controlador de llamada) en lugar de un archivo html estático, puede usar una ruta general:

public class AdminSpaController
{
    [Route("~/admin-spa/{clienSidePart*}")]
    ActionResult AdminSpa()
    {
        ...
    }
}

En este caso, el servidor devuelve solo SPA y luego se inicializa de acuerdo con la ruta. Este enfoque es más flexible ya que no depende del módulo url-rewrite.

Enrutamiento de atributos en áreas #

Para usar el enrutamiento de atributos en áreas, se requieren áreas de registro y definiciones de [RouteArea(...)].

En RouteConfig.cs:

public class RouteConfig
{
    public static void RegisterRoutes(RouteCollection routes)
    {
        routes.IgnoreRoute("{resource}.axd/{*pathInfo}");
        routes.MapMvcAttributeRoutes();
        AreaRegistration.RegisterAllAreas();
    }
}

En una definición de enrutamiento de atributo de controlador de área de muestra:

[RouteArea("AreaName", AreaPrefix = "AreaName")]
[RoutePrefix("SampleAreaController")]
public class SampleAreaController : Controller
{
    [Route("Index")]
    public ActionResult Index()
    {
        return View();
    }
}

Para usar enlaces Url.Action en áreas:

@Url.Action("Index", "SampleAreaController", new { area = "AreaName" })

Enrutamiento personalizado #

El enrutamiento personalizado proporciona una necesidad especializada de enrutamiento para manejar solicitudes entrantes específicas.

Para definir rutas personalizadas, tenga en cuenta que el orden de las rutas que agrega a la tabla de rutas es importante.

public static void RegisterRoutes(RouteCollection routes)
{
    routes.IgnoreRoute("{resource}.axd/{*pathInfo}");
    
    // this is an advanced custom route
    // you can define custom URL with custom parameter(s) point to certain action method
    routes.MapRoute(
    "CustomEntry", // Route name
    "Custom/{entryId}", // Route pattern
    new { controller = "Custom", action = "Entry" } // Default values for defined parameters above
    );

    // this is a basic custom route
    // any custom routes take place on top before default route
    routes.MapRoute(
    "CustomRoute", // Route name
    "Custom/{controller}/{action}/{id}", // Route pattern
    new { controller = "Custom", action = "Index", id = UrlParameter.Optional } // Default values for defined parameters above
    );

    routes.MapRoute(
    "Default", // Route name
    "{controller}/{action}/{id}", // Route pattern
    new { controller = "Home", action = "Index", id = UrlParameter.Optional } // Default values for defined parameters above
    );
}

Los nombres de controlador y acción están reservados. Por defecto, MVC asigna {controlador} parte de la URL a la clase <controlador>Controlador, y luego busca un método con el nombre <acción> sin agregar ningún sufijo.

Aunque puede ser tentador crear una familia de rutas usando la plantilla {controlador}/{acción}/{parámetro}, tenga en cuenta que al hacer esto revela la estructura de su aplicación y hace que las URL sean algo frágiles porque cambia el nombre del controlador. cambia la ruta y rompe los enlaces guardados por el usuario.

Preferir configuración de ruta explícita:

routes.MapRoute(
    "CustomRoute", // Route name
    "Custom/Index/{id}", // Route pattern
    new { controller = "Custom", action = nameof(CustomController.Index), id = UrlParameter.Optional }
);

(No puede usar el operador nameof para el nombre del controlador, ya que tendrá el sufijo adicional Controller), que debe omitirse al configurar el nombre del controlador en la ruta.
