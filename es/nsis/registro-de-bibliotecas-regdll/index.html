En resumen, una DLL es una colección de pequeños códigos ejecutables, a los que un programa en ejecución puede llamar cuando lo necesite. La DLL permite que el ejecutable se comunique con un dispositivo específico, como una impresora, o puede contener código para realizar una serie de funciones particulares. Como existen varios métodos de implementación para hacer esto, en este tema le mostraré cómo registrar y cancelar el registro de cualquier DLL que requiera su aplicación; y lo haremos usando la línea de comando RegSrv32.exe.

Sintaxis #

  • ${Registro::DLL} "codec.ocx" "" $1 $0
  • ${UnRegister::DLL} "volumeCtrl.cpl" /DISABLEFSR $1 $0
  • ${RegistrarDLL} "imprimir.drv"

Parámetros #

Interruptor Descripción
/u Anula el registro del servidor.
/s Especifica que regsvr32 se ejecute en silencio y no muestre ningún cuadro de mensaje.
/n Especifica que no se llame a DllRegisterServer. Debe usar esta opción con /i.
/i :líneacmd Llama a DllInstall pasándole un [cmdline] opcional. Cuando se usa con /u, llama a la desinstalación de dll.
nombredll Especifica el nombre del archivo dll que se registrará.
/? Muestra ayuda en el símbolo del sistema.

RegSrv32.exe #

Usar la línea de comando RegSvr32.exe es mi método preferido para registrar sus bibliotecas, así que comencemos aquí primero. Las PC con Windows junto con Internet Explorer 3.0 o posterior vienen con RegSvr32.exe. Por lo tanto, es muy probable que su PC venga estándar con esta utilidad. Ahora, si está ejecutando en una máquina de 64 bits, hay dos variantes que puede considerar. Se pueden encontrar en $WINDIR\system32 o $WINDIR\SysWow32.

Los parámetros que puede usar con RegSrv32 son /u /s /i /n. El interruptor de comando /u anulará el registro del archivo. El interruptor /i se puede usar con /u para solicitar la desinstalación de DLL. El parámetro /n no llamará a DllRegisterServer; se usa con /i, que es el interruptor de instalación. Si usa /s, que significa silencioso, no se mostrarán cuadros de mensaje en Windows XP o posterior.

Cuando use RegSvr32.exe desde la línea de comando, obtendrá cuadros de mensaje después de llamarlo. La función DLLSelfRegister se invocará a menos que se utilice el cambio antes mencionado, por supuesto; si tiene éxito, se mostrará un cuadro de alerta que indica su éxito, al igual que el fracaso que arroja un mensaje de error.

Según mi experiencia, x64 RegSvr32.exe registra correctamente las DLL x86 en Windows Vista y superior (excluye XP; visite este artículo< /a> para más) y superior, así que lo uso cuando instalo en sistemas x64 incluso cuando registro un archivo x86. Además, Windows XP es un arte moribundo; bendiga su corazón. =)

RegSrv32.exe Uso sencillo #

#= Regardless of architecture we're using just the following

!define REGSVR `$SYSDIR\regsvr32.exe` #= define where RegSrv32 is
!define DLL `$AppDirectory\App\MyLegalProgram\myLegit.dll` #= define the file to register

##=
#= Command line usage is the same for both variants of RegSrv32 as follows
#= regsvr32 [/u] [/s] [/n] [/i[:cmdline]] DLL
#=
##= So in our .nsi file it would be similar to the following:

Exec `"${REGSVR}" /s "${DLL}"`

#= Moreover, you may also use the following

ExecWait `"${REGSVR}" /s "${DLL}"` $0 #= The $0 will contain the error code if any

#= The above will wait for exe to quit it's process before continuing

Macro: Registro::DLL #

##=
# The variable $Bit will hold either 64 or 32 depending on system architecture 
# This is used with ${Register::DLL} and ${UnRegister::DLL}
# Use this in the beginning of your code.
# This snippet should only be used once.
#
Var Bit
System::Call "kernel32::GetCurrentProcess()i.s"
System::Call "kernel32::IsWow64Process(is,*i.r0)"
StrCmpS $0 0 +3
StrCpy $Bit 64
Goto +2
StrCpy $Bit 32

##= 
#= Register::DLL
#
# USAGE:
# ${Register::DLL} "DLL Filename" /DISABLEFSR $0 $1
#
#        ::DLL       = Registers a DLL file. 
#        /DISABLEFSR = Disables redirection if x64. Use "" to skip.
#        $0          = Return after call
#        $1          =  ''      ''    ''
#
!define Register::DLL `!insertmacro _Register::DLL`
!macro _Register::DLL _DLL _FSR _ERR1 _ERR2
    StrCmpS $Bit 64 0 +4
    StrCmp ${_FSR} /DISABLEFSR 0 +3
    ExecDos::Exec /TOSTACK /DISABLEFSR `"${REGSVR}" /s "${_DLL}"`
    Goto +2
    ExecDos::Exec /TOSTACK `"${REGSVR}" /s "${_DLL}"`
    Pop ${_ERR1}
    Pop ${_ERR2}
!macroend

##= 
# Alternatively you can use this macro found in my travels 
# but you should include x64.nsh as this macro makes use
# of it's function.
#
#= USAGE:
# ${RegisterDLL} "SomeLibrary.dll"
#
!define RegisterDLL "!insertmacro _RegisterDLL"
!macro _RegisterDLL _DLL
    ${If} ${RunningX64}
        ${DisableX64FSRedirection}
        ExecWait '"$SYSDIR\regsvr32.exe" /s "${_DLL}"'
        ${EnableX64FSRedirection}
    ${Else}
        RegDLL "${DLL}"
    ${EndIf}
!macroend

El parámetro /DISABLEFSR solo debe usarse en máquinas x64. Sin embargo, si te equivocas, hay un sistema de seguridad en las macros que esquivará esta viñeta por ti.

Macro: DesRegistrar::DLL #

##=
# The variable $Bit will hold either 64 or 32 depending on system architecture 
# This is used with ${Register::DLL} and ${UnRegister::DLL}
# Use this in the beginning of your code. 
# This snippet should only be used once.
#
Var Bit
System::Call "kernel32::GetCurrentProcess()i.s"
System::Call "kernel32::IsWow64Process(is,*i.r0)"
StrCmpS $0 0 +3
StrCpy $Bit 64
Goto +2
StrCpy $Bit 32

#=# 
#= UnRegister::DLL
#
# USAGE:
# ${UnRegister::DLL} "DLL Filename" /DISABLEFSR $0 $1
#
#        ::DLL       = Unregisters a DLL file. 
#        /DISABLEFSR = Disables redirection if x64. Use "" to skip.
#        $0          = Return after call
#        $1          =  ''      ''    ''
#
!define UnRegister::DLL `!insertmacro _UnRegister::DLL`
!macro _UnRegister::DLL _DLL _FSR _ERR1 _ERR2
    StrCmpS $Bit 64 0 +4
    StrCmp ${_FSR} /DISABLEFSR 0 +3
    ExecDos::Exec /TOSTACK /DISABLEFSR `"${REGSVR}" /s /u "${_DLL}"`
    Goto +2
    ExecDos::Exec /TOSTACK `"${REGSVR}" /s /u "${_DLL}"`
    Pop ${_ERR1}
    Pop ${_ERR2}
!macroend

##= 
# Alternatively you can use this macro I found in my travels 
# but be sure to include the x64 plugin as this macro makes 
# use of it's function.
#
#= USAGE:
# ${UnregisterDLL} "SomeLibrary.dll"
#
!define UnregisterDLL "!insertmacro _UnregisterDLL"
!macro _UnregisterDLL _DLL
    ${If} ${RunningX64}
        ${DisableX64FSRedirection}
        ExecWait '"$SYSDIR\regsvr32.exe" /s /u "${_DLL}"'
        ${EnableX64FSRedirection}
    ${Else}
        UnRegDLL "${DLL}"
    ${EndIf}
!macroend

El parámetro /DISABLEFSR solo debe usarse en máquinas x64. Sin embargo, si te equivocas, hay un sistema de seguridad en las macros que esquivará esta viñeta por ti.
