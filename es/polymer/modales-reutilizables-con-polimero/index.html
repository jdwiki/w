Modales #

Entonces, desea agregar el diseño de materiales a su cartera de negocios o carrera. ¿Eh? ¿Simplemente no puedes resistirte a usar un Modal? ¡que aparece al hacer clic en esas tarjetas nítidas que tiene la intención de diseñar, para cada uno de sus proyectos/productos!

Digamos que para cada proyecto que haya realizado, o para cada producto que haya diseñado, necesita una tarjeta de material. Cada una de estas tarjetas debe tener

  • Un icono, que abre un cuadro de información, que enumera todas las características de
  • su producto o proyecto, en detalle. Un icono de “compartir esto”, que comparte
  • su producto o proyecto en las redes sociales Un icono de “fork this”, que abre la página de github para su producto/proyecto

Con html nativo, y suponiendo que use bootstrap o cualquier otro diseño de cuadro flexible, necesitaría

  • Escriba un div contenedor para cada nuevo proyecto con una fila de clase
  • Envuelva su imagen de héroe para cada proyecto en una columna de 12 columnas de ancho
  • Escribir otra fila
  • Envuelva cada enlace (información | compartir | bifurcación) en una columna de 4 columnas de ancho propia
  • Inyecte estas dos filas en un contenedor

Para principiantes. Una vez que haya hecho lo anterior para todos sus proyectos, debe trabajar en la creación de la ventana emergente requerida para cada proyecto. Entonces,

  • Descarga toneladas de javascript y css (al personalizar bootstrap para la funcionalidad modal)
  • Escribes un Modal con una identificación única, para diferenciar la información de cada proyecto (modal-P1 para el Proyecto 1, modal-P2 para el Proyecto 2 y así sucesivamente)
  • Luego redactas el html requerido, para mostrar la información de cada proyecto, en su Modal correspondiente.

<script>
Polymer({

Nota: uso y gramática de Iron Icon #

Si necesitamos usar el ícono de información para nuestra ventana emergente modal, escribimos el marcado así:

<icono-hierro icon="icons:info"></icono-hierro>

Si nosotros quiere el icono de compartir en redes sociales, luego el atributo de icono de arriba, se convierte

icono=”social:compartir”

Entonces, el significado gramatical, necesito el conjunto de iconos de “SOCIAL” y quiero el icono COMPARTIR de ese conjunto.

Alternativamente, puede usar font-awesome , en cuyo caso, puede usar,

elige lo que consideres más adecuado.

Una vez que se realizan las inclusiones, escribimos la plantilla para nuestro elemento personalizado. Usaremos enlaces unidireccionales para los datos en el marcado de nuestro elemento personalizado. Vinculamos todo lo que el usuario pasa como atributos a sus propiedades correspondientes de nuestro elemento, mientras nos embarcamos en la llamada de registro de nuestro elemento.

Entonces, el modelo html para nuestro elemento personalizado es:

<dom-module id="tool-bar">
 
    <template id="tool-box">
 
        <ul class="flex-wrap toolbar">
            <li>
                <iron-icon icon="icons:info" id="anchor-for-[[modalId]]" onclick="clickHandler(event)"></iron-icon>
            </li>
            <li>
                <a href="https://plus.google.com/share?url=[[link2Share]]" target="_blank" ><iron-icon icon="social:share"></iron-icon></a>
            </li>
            <li>
                <a href="[[link2Fork]]" target="_blank"><i class="fa fa-2x fa-code-fork" aria-hidden=true></i></a> 
            </li>
        </ul>
 
    </template>
 
</dom-module>

¡Enfriar! Entonces, la barra de herramientas tiene su ul con plantilla, que enumera los tres enlaces

  • icono de información - ventana emergente modal
  • compartir este icono - compartir en google
  • Bifurcar este ícono - Bifurcar el repositorio

Además, hemos vinculado los atributos que escribe el usuario a las propiedades respectivas.

Nota: hacer que el ID modal sea único #

Hemos abordado la unicidad del ícono de información, especificando la identificación atributo como tal

id="anclaje-para-[[modalId]]" onclick="clickHandler(evento)"></icono-de-hierro>

Entonces, cada Proyecto tendrá una identificación de ancla única.

es decir

Si el usuario pasa “project-cats” como modal-id, en la llamada a nuestro elemento personalizado, la identificación del ancla sería “ancla-para-proyecto-gatos”

Si el usuario agrega otro proyecto y pasa “project-puppy” como el atributo modal-id, el id del ancla sería ancla-para-proyecto-cachorro

y así sucesivamente.

Pero, ¿cómo nos aseguramos de que cada ícono de información esté asignado a su propio modal?

El “ícono de hierro“ para info, debe tener asignado un Atributo “data-dialog”, que también debe ser único.

El atributo data-dialog, es similar al atributo data-target en el modo de arranque. Debe ser único para cada nuevo proyecto.

Se lo dejamos al usuario, para mantener la singularidad. Entonces el usuario no puede tienen el mismo atributo modal-id para diferentes llamadas <barra de herramientas>.

Entonces, como dice la nota anterior, debemos asignar el ícono de información de cada proyecto a un cuadro de diálogo de datos.

Agreguemos esa funcionalidad.

Usamos el método ready. Es muy similar al de Jquery

$.listo();

.Se invoca, cuando el elemento personalizado , termina de cargarse.

Agregue esta función, después del objeto de propiedades, en la llamada de registro de Polymer.

ready: function() {
    document.querySelector("#anchor-for-" + this.modalId).setAttribute('data-dialog', this.modalId);
}

¡Muy genial! Así que ahora tenemos

  • Un elemento personalizado registrado como un elemento de polímero
  • Modales | Compartir | Iconos de bifurcación agregados al elemento personalizado

Se adjunta un Modal al icono de hierro de información, con el id “ancla-para-[Lo que se pasa como id-modal]”. Eso es cortesía del método listo de nuestro elemento.

Entonces, hasta ahora, escribiendo

<tool-bar
link2-share="http://www.myportfolio.com/project-neighbourhood-map"
link2-fork="https://github.com/me/project-neighbourhood-map"
modal-id="project-neighbourhood-map">
</tool-bar>

Produce esto:

Pero cuando el usuario hace clic en ese ícono de información, algo como esto debería suceder

y nuestro objetivo era envolver esta descripción completa del proyecto, dentro de la etiqueta <barra de herramientas> en la página de usuarios.

¿Recuerdas Shadow DOM? Necesitamos tener un contenedor en nuestro elemento personalizado, que proporcione el shadow DOM para el contenido de nuestro Modal. El contenido en sí está envuelto dentro de una etiqueta <content> dentro del shadowDOM del elemento personalizado.

Para el contenedor que aloja la descripción del Proyecto, usamos el elemento Polímero “paper-dialog”. Paper-dialog, se asigna como id, lo que sea que se pase como modal-id en la declaración en la página del usuario.

Por lo tanto, agregue este marcado, después de ul en el DOM del elemento personalizado de la barra de herramientas

<paper-dialog id="[[modalId]]" modal>
    <div class="text-right modal-close">
        <iron-icon icon="icons:power-settings-new" dialog-dismiss></iron-icon>
    </div>
    <h2 class="text-center text-capitalize">[[title]]</h2>
    <paper-dialog-scrollable>
        <div class="container-fluid">
            <div class="row flex-wrap info">
                <content></content>
            </div>
        </div>
    </paper-dialog-scrollable>
</paper-dialog>

Nota: Papel-diálogo-desplazable #

Usamos Paper-dialog-scrollable, para envolver la descripción de nuestro Proyecto, así que que, Cualquier descripción larga, no se desborde.

El último bit es agregar la funcionalidad onclick para cuando se abra el modal, al hacer clic en el icono de información.

Cuando ocurre un clic, como es normal, se desencadena un evento, que pasamos a nuestra función clickHandler de la siguiente manera:

<iron-icon icon="icons:info" id="anchor-for-[[modalId]]" onclick="clickHandler(event)"></iron-icon>

Entonces, una vez que la función clickHandler recibe el evento,

  • Debe recopilar qué elemento activó el clic,
  • Necesita recopilar, el atributo de diálogo de datos de ese elemento, para determinar qué modal abrir (¿Recuerda que cada proyecto tiene su propia identificación modal?)
  • Luego necesita llamar al método abierto, exactamente para ese modal

Entonces, la función es así:

Agregue esto, antes de la llamada de registro del elemento. Esto no es parte del elemento, pero está dentro del alcance del elemento.

function clickHandler(e) {
    var button = e.target;
    while (!button.hasAttribute('data-dialog') && button !== document.body) {
        button = button.parentElement;
    }
 
    if (!button.hasAttribute('data-dialog')) {
        return;
    }
 
    var id = button.getAttribute('data-dialog');
    var dialog = document.getElementById(id);
    if (dialog) {
        dialog.open();
    }
}

Entonces, con eso, hemos completado la codificación del elemento personalizado, <barra de herramientas>.
