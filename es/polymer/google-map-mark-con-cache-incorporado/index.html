A - Primeros pasos #

En pocas palabras, nuestro elemento personalizado debería

  • Tener una funcionalidad de búsqueda integrada, que busca y marca lugares en el mapa.
  • Aceptar Un atributo llamado location-array , que es una lista de lugares
  • Tener un Oyente, para escuchar un evento llamado “google-map-ready”. Este evento se activa cuando el elemento se ha cargado.
  • Este oyente debe recorrer la matriz de ubicación y asignar el siguiente elemento en su matriz de ubicación como la “consulta de búsqueda” actual
  • Tener un método llamado caché
  • Este método almacena en caché la latitud y la longitud de cada lugar en la matriz de ubicación, tan pronto como la búsqueda arroja un resultado.
  • Tenga una plantilla de repetición Dom, que recorre los resultados almacenados en caché y coloca un marcador en cada ubicación

Entonces, el propósito de nuestro elemento personalizado en esencia significa, si pasa una serie de ubicaciones como esta:

<g-map location-array='["Noruega","Suecia"]'></g-map>

El elemento personalizado

  • No solo debe marcar Noruega y Suecia en el mapa, sino,
  • También debe marcar cualquier búsqueda posterior en el mapa, es decir, si busca Boston, después de que el mapa haya marcado Noruega y Suecia, Boston también debería tener un marcador en el mapa.

Nota: #

Obtener una clave API de la API javascript de Google es bastante simple.

Clave API de Google

Ahora, Dado que se utilizará la clave API, tenemos dos opciones.

  • Deje que el usuario lo pase como un atributo
  • Tener una clave API predeterminada configurada

Ahora, me decantaría por la primera, simplemente porque cualquier API Key tiene sus límites impuestos. es decir, solo se puede usar tantas veces. Por lo tanto, una clave configurada y enviada con el elemento que desarrollamos podría agotarse pronto.

Quien quiera usar el elemento personalizado, puede generar una nueva clave y pasarla como un atributo.

al igual que:

<g-map api-key="AIzaSyBLc_T17p91u6ujSpThe2H3nh_8nG2p6FQ"></g-map>

Entonces, estamos pasando la clave api como un atributo y dentro de nuestro elemento personalizado, lo vinculamos al atributo de clave api del elemento Polymer.

al igual que:

<google-map api-key=[[apiKey]]></google-map>

Entonces, una vez que pasamos la clave API y actualizamos, ¡todavía no vemos el mapa!

Nota: #

Mapas de Google, necesita algo de css para renderizar. Necesitamos la altura del conjunto de mapas explícitamente.

Entonces, haga un archivo css dentro del directorio css, diga app.css y agregue estas lineas

google-mapa{
 
min-altura: 30vmax;
}

<google-map api-key="[[apiKey]]"></google-map>

C - Agregar capacidad de búsqueda a nuestro elemento personalizado #

Para buscar un lugar, usamos el poderoso elemento que envía Polymer, llamado google-map-search.

Todo lo que necesitas hacer es pasar

  • un objeto de mapa y
  • una cadena de consulta al elemento así:
<google-map-search map=[[map]] query=[[query]]></google-map-search>

¿Cómo pasamos el objeto del mapa? ¿De dónde sacamos eso?

¡Simple!

Cuando invoque el elemento google-map, vincule el mapa de propiedades de su elemento personalizado al mapa de propiedades del elemento .

Entonces, invocamos el elemento google-map así:

<google-map api-key="whatever key you generated for google's javascript API" map={{map}}></google-map>

Nota: #

Hacemos un enlace de datos bidireccional para el mapa de arriba, al invocar el mapa de Google polímero.

Está representado por las llaves {{}} en lugar de un sentido enlace indicado por llaves cuadradas [[]].

Cuando hacemos un enlace bidireccional,

  • Cada vez que cambia la propiedad del mapa de nuestro elemento personalizado g-map, se notifica el elemento google-map

y,

  • Cada vez que cambia la propiedad del mapa de google-map, se nos notifica en nuestro elemento personalizado g-map.

Entonces, con el enlace de datos bidireccional en su lugar, cada vez que el objeto del mapa cambia en google-map, nuestro elemento personalizado se actualiza con los cambios.

y pasamos el objeto del mapa como un atributo al elemento google-map-search, de modo que cualquier resultado de búsqueda se marque en el mapa actual representado.

¿Cómo se ve el DOM de nuestro elemento personalizado en este momento?

<template id="google-map-custom-element">
 
<google-map-search map=[[map]] query=[[query]]></google-map-search>
<google-map api-key=[[apiKey]] map={{map}}></google-map>
</template>

¡Nunca agregamos las propiedades de mapa y consulta para nuestro elemento personalizado!

Agregue las propiedades en la llamada de registro a Polymer

Polymer({
 
    is:"g-map",
 
    properties:{
 
        map:{
 
            type: Object
 
        },
 
        query:{
 
            type:String,
 
            value:""
 
        }
    }
 
});

Ahora que hemos registrado las propiedades, notamos que

  • Están obteniendo el mapa representado actualmente como un objeto, a través del enlace de datos y almacenándolo como una propiedad local también llamada mapa,
  • Tener una consulta de propiedad, pasada a google-map-search

Aceptar la entrada del usuario como consulta de búsqueda #

Pero, ¿quién nos da la consulta para buscar? ¡Errar! ¿en este momento? NADIE.

Eso es precisamente lo que hacemos a continuación.

Agreguemos un formulario de búsqueda, que el usuario que representa el mapa, usaría para ingresar una consulta. Necesitamos entradas para el cuadro de búsqueda y usamos la entrada de hierro de Polymer.

Agregar un formulario de búsqueda #

Incluya los elementos de hierro necesarios al principio del archivo g-map.html

<link rel=import href=../bower_components/iron-input/iron-input.html>
<link rel=import href=../bower_components/iron-icon/iron-icon.html>
<link rel=import href=../bower_components/iron-icons/iron-icons.html>
<link rel=import href=../bower_components/iron-icons/maps-icons.html>

Queremos el formulario de búsqueda, en la esquina superior izquierda, por lo que agregamos una entrada de hierro con posición absoluta y algo de css para fijarlo allí.

Entonces, envuelva el DOM para nuestro elemento personalizado, dentro de un div llamado “mapa-contenedor”, y agregue un div secundario llamado “search_form”.

<template id="google-map-custom-element">
    <div class="map-container">
         <google-map-search map="[[map]]" query="[[query]]"></google-map-search>
         <google-map api-key="[[apiKey]]"></google-map>
         <div class="search_form">
             <iron-icon icon="icons:search" on-tap=__search></iron-icon>
             <input is="iron-input" placeholder="Search Places" type=text name="search" bind-value="{{query}}">
         </div>
     </div>
 </template>

Agregue un poco de CSS básico para anclar el formulario de búsqueda que escribimos en la parte superior derecha del mapa

.search_form{
    position: absolute;
    top:10px;
    right:10px;
}
.search_form iron-icon{
    position: absolute;
    right: 1px;
    top:1px;
    cursor: pointer;
}

Y, luego, cuando actualizamos, tenemos esto. El formulario de búsqueda en la parte superior derecha

Todo lo que queda por hacer es proporcionar una entrada al elemento de búsqueda de mapas de Google, a través de la propiedad de consulta de nuestro elemento personalizado.

Entonces, vinculamos nuestra propiedad “consulta” a la entrada de hierro en el formulario de búsqueda. al igual que:

<input is="iron-input" placeholder="Search" type=text name="search" bind-value="{{query}}">

Nota: #

Vinculamos la consulta de propiedad de nuestro elemento personalizado a la entrada de búsqueda usando,

Atributo “bind-value” de una entrada de hierro.

Entonces, bind-value={{query}} en la entrada de hierro anterior, se asegura de que cualquier cambios en el texto dentro de la entrada de hierro, se notifica de nuevo a la propiedad de consulta.

Con nuestra configuración actual,

Por cada alfabeto que un usuario escribe en el cuadro de búsqueda, ocurre una búsqueda, lo que ralentiza el mapa y consume el límite de nuestra clave API.

¿Por qué?

Porque, la consulta de propiedades de nuestro elemento personalizado, está ligada al hierro entrada, y cada letra escrita en la entrada de hierro, cambia el valor de “consulta”, que luego afecta el html en

<google-mapa-búsqueda mapa="[[mapa]]" consulta="[[consulta]]">

haciendo que se produzca una búsqueda

¡Nuestra solución es simple! Simplemente vincule el atributo de “consulta” del elemento de búsqueda de mapas de Google a una propiedad separada.

Digamos que creamos una propiedad “searchQuery” para nuestro elemento personalizado.

Agregue esto, debajo de la propiedad “consulta” en la llamada de Polymer

searchQuery :{
 
    type:String
 
}

A continuación, cambie el enlace en la llamada del elemento de búsqueda de mapa de Google en el DOM de nuestro elemento personalizado

al igual que:

<google-map-search map="[[map]]" query="[[searchQuery]]">

Por último, tenga en cuenta que agregamos un ícono de hierro (ícono de búsqueda) a nuestro formulario de búsqueda.

<iron-icon icon="icons:search" on-tap=search></iron-icon>

Agreguemos esa función de búsqueda a nuestro elemento personalizado. Se llama tocando el icono de búsqueda.

En nuestro método de búsqueda, asignamos lo que esté en el cuadro de búsqueda al atributo de “consulta” de google-map-search.

Agregue esta función, después del objeto “propiedades” en la solicitud de registro del polímero.

//paste this after, the properties object
 
search: function() {
this.query = this.searchQuery;
}

¡Muy agradable! Entonces, ahora, una búsqueda ocurre solo al tocar el ícono de búsqueda.

Mostrar los resultados de la búsqueda #

Tenemos

  • Representó un mapa
  • Anclado un formulario de búsqueda en el Mapa
  • Escribió la funcionalidad para buscar al tocar el ícono de búsqueda

Ahora tenemos que mostrar los resultados.

Para eso, necesitamos vincular los resultados del elemento de búsqueda de mapas de Google a una propiedad local.

Entonces, vamos a crear una propiedad para nuestro elemento personalizado g-map, y asígnele el nombre result duh!

En la llamada de registro de Polymer, agregue los resultados de propiedad de tipo objeto en la declaración de propiedades

results:{
 
type:Object
 
}

Y vincule la propiedad de resultados de google-map-search a la propiedad de resultados locales que definió anteriormente.

<google-map-search results={{results}} map=[[map]] query=[[query]]></google-map-search>

Con eso en su lugar, actualicemos la página y busquemos starbucks en el mapa.

¡No pasó nada!

Bueno Duh! simplemente vinculamos los resultados de google-map-search a la propiedad de resultados. ¿Escribimos algo para mostrarlos en el mapa? ¡NO!

Necesitamos que.

Entonces, sabemos que el elemento Polymer google-map-search devuelve una serie de marcadores como resultados.

Necesitamos recorrer la matriz y colocar marcadores en el mapa.

Entonces, escribimos una plantilla dom-repeat, dentro de nuestro elemento , para mostrar los marcadores para cada resultado

al igual que:

<google-map api-key=[[apiKey]] map=[[map]]>
 
    <template is="dom-repeat" items="{{results}}" as="marker">
        <google-map-marker latitude="{{marker.latitude}}" longitude="{{marker.longitude}}">
            <h2>{{marker.name}}</h2>
            <span>{{marker.formatted_address}}</span>
        </google-map-marker>
    </template>
 
</google-map>

Y ahora, al actualizar, vemos esto

¡Muy agradable!

Hemos agregado con éxito la funcionalidad de búsqueda a nuestro elemento personalizado.

Todo lo que tenemos que hacer es,

escribe esto en nuestro index.html

<g-map api-key="Whatever API key you generated"></g-map>

¡Y obtienes un mapa en el que puedes buscar!

D - Almacenamiento en caché de resultados de búsqueda #

Nuestro objetivo, sin embargo, era

  • hacer una lista de lugares
  • marcarlos todos en el mapa y,
  • marcar cualquier otro lugar de nuestra elección, utilizando el cuadro de búsqueda

Tenemos un cuadro de búsqueda, donde el usuario puede buscar una consulta a la vez. Para acomodar múltiples consultas, necesitamos almacenar en caché los resultados de cada consulta, antes de mostrar los marcadores y fijarlos en el mapa.

Entonces, agreguemos

  • Una propiedad dice ubicaciones que acepta una variedad de ubicaciones
  • Un caché de propiedades que almacena los resultados de cada consulta en la matriz de ubicación pasada a nuestro elemento personalizado
cache:{
 
type:Object
 
},
 
locations:{
 
type:Object
 
}

A continuación, escribimos una función para almacenar en caché cada resultado de búsqueda, resultante de una búsqueda de cada lugar en la matriz de ubicación.

¡Es bastante simple!

Ya estamos reuniendo los resultados en una propiedad llamada “resultados”, que luego estamos recorriendo para mostrar marcadores.

Todo lo que tenemos que hacer ahora es insertar cada resultado de búsqueda en una matriz, ¡y nuestro caché está listo!

Necesitamos usar un evento para activar el almacenamiento en caché. Ese evento, se llama, “on-google-map-search-result” y se dispara, después de que el elemento google-map-search finaliza una búsqueda.

Por lo tanto, “en-google-map-search-result”, llamamos a una función, digamos “cacher”, para almacenar en caché los resultados de búsqueda actuales en una propiedad llamada caché.

<google-map-search map=[[map]] query=[[searchQuery]] on-google-map-search-results=cacher>

Nota: #

Normalmente, para insertar un elemento en una matriz, usamos el comando vainilla método

por ejemplo: tengo un nuevo lugar para agregar a mi lista de lugares que quiero marcar, decir Nueva York.

Si la matriz a la que quiero agregarlo se llama ubicaciones, escribiría

ubicaciones.push('Nueva York');

Sin embargo, en nuestro caso, necesitamos inserte cualquier resultado de google-map-search en una matriz llamada caché. Luego necesitamos recorrer el caché con los últimos resultados incluidos. No el caché existente.

Ahora, usamos la plantilla dom-repeat para recorrer los resultados y colocar marcadores en el mapa.

Sin embargo, si pasamos una matriz de ubicación como esta:

<g-map locations=["Iceland","Argentina","London"]></g-map>

Luego, la propiedad de resultados de nuestro elemento personalizado se sobrescribirá para cada elemento de esa matriz.

Por lo tanto, llamamos al método cacher en el evento on-google-map-search-results e insertamos el resultado actual en la propiedad de caché.

Esto no puede ser un empujón de vainilla como en la nota anterior.

Necesitamos que el repetidor dom sepa que nuestro caché se ha sobrescrito con cada nueva búsqueda.

Entonces, usamos un impulso especial que Polymer envía, así:

cacher: function() {
this.push('cache', this.results);
}

Agregue esa función, después de la función de búsqueda que escribimos anteriormente.

La sintaxis implica que la propiedad caché debe ser mutada, mediante la adición del valor actual de los resultados de la propiedad y cualquier dom-repetidor que use la propiedad caché para recorrer, debe ser notificado de la mutación.

Por último, cambiamos la plantilla dom-repeat para recorrer el caché, en lugar de los resultados.

Nota: #

caché es una matriz de matrices.

cada elemento de la matriz de caché, es una matriz de resultados

Así que nuestro repetidor de dom, será así:

<template id="resultList" is="dom-repeat" items="[[cache]]">
 
    <template is="dom-repeat" items="[[item]]" as="marker">
 
        <google-map-marker latitude="{{marker.latitude}}" longitude="{{marker.longitude}}">
            <h2>{{marker.name}}</h2>
            <span>{{marker.formatted_address}}</span>
        </google-map-marker>
    </template>
</template>
