Perfilado con Stack #

Configure la creación de perfiles para un proyecto a través de stack. Primero construye el proyecto con el indicador --profile:

stack build --profile

Las banderas GHC no son necesarias en el archivo cabal para que esto funcione (como -prof). stack activará automáticamente la creación de perfiles tanto para la biblioteca como para los ejecutables del proyecto. La próxima vez que se ejecute un ejecutable en el proyecto, se pueden usar los indicadores +RTS habituales:

stack exec -- my-bin +RTS -p

Estructura #

Estructura de archivos #

Un proyecto simple tiene los siguientes archivos incluidos:

➜  helloworld ls 
LICENSE          Setup.hs         helloworld.cabal src              stack.yaml

En la carpeta src hay un archivo llamado Main.hs. Este es el “punto de partida” del proyecto helloworld. Por defecto Main.hs contiene un simple “¡Hola, Mundo!” programa.

Principal.hs

module Main where

main :: IO ()
main = do
  putStrLn "hello world"

Ejecutando el programa #

Asegúrate de estar en el directorio helloworld y ejecuta:

stack build # Compile the program
stack exec helloworld # Run the program
# prints "hello world"

Cree y ejecute un proyecto de pila #

En este ejemplo, el nombre de nuestro proyecto es “helloworld”, que se creó con stack new helloworld simple

Primero tenemos que construir el proyecto con stack build y luego podemos ejecutarlo con

stack exec helloworld-exe

Visualización de dependencias #

Para averiguar de qué paquetes depende directamente su proyecto, simplemente puede usar este comando:

stack list-dependencies

De esta manera, puede averiguar qué versión de sus dependencias se derribó realmente por pila.

Los proyectos de Haskell con frecuencia se encuentran atrayendo muchas bibliotecas indirectamente y, a veces, estas dependencias externas causan problemas que debe rastrear. Si se encuentra con una dependencia externa no autorizada que le gustaría identificar, puede recorrer todo el gráfico de dependencias e identificar cuál de sus dependencias finalmente está extrayendo el paquete no deseado:

stack dot --external | grep template-haskell

stack dot imprime un gráfico de dependencia en forma de texto que se puede buscar. También se puede ver:

stack dot --external | dot -Tpng -o my-project.png

También puede establecer la profundidad del gráfico de dependencia si lo desea:

stack dot --external --depth 3 | dot -Tpng -o my-project.png

Instalación de la pila #

Mac OS X

Usando Homebrew:

brew install haskell-stack

Creando un proyecto simple #

Para crear un proyecto llamado helloworld ejecuta:

stack new helloworld simple

Esto creará un directorio llamado helloworld con los archivos necesarios para un proyecto de Stack.

Instalación de pila #

Al ejecutar el comando

stack install

Stack copiará un archivo ejecutable a la carpeta

/Users/<yourusername>/.local/bin/

Apilar paquetes y cambiar la versión LTS (resolver) #

Stackage es un repositorio de paquetes de Haskell. Podemos agregar estos paquetes a un proyecto de pila.

Agregar lentes a un proyecto. #

En un proyecto de pila, hay un archivo llamado stack.yaml. En stack.yaml hay un segmento que se ve así:

resolver: lts-6.8

Stackage mantiene una lista de paquetes para cada revisión de lts. En nuestro caso queremos la lista de paquetes para lts-6.8 Para encontrar estos paquetes visite:

https://www.stackage.org/lts-6.8 # if a different version is used, change 6.8 to the correct resolver number.

Mirando a través de los paquetes, hay un Lens-4.13.

Ahora podemos agregar el paquete de idioma modificando la sección de helloworld.cabal:

  build-depends: base >= 4.7 && < 5

a:

  build-depends: base >= 4.7 && 5,
                 lens == 4.13

Obviamente, si queremos cambiar un LTS más nuevo (después de su lanzamiento), simplemente cambiamos el número de resolución, por ejemplo:

resolver: lts-6.9

Con la próxima compilación de la pila, Stack usará la versión LTS 6.9 y, por lo tanto, descargará algunas dependencias nuevas.
