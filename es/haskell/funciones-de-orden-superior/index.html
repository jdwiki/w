Las funciones de orden superior son funciones que toman funciones como parámetros y/o funciones de retorno como sus valores de retorno.

Conceptos básicos de las funciones de orden superior #

Revise https://www.wikiod.com/es/haskell/solicitud-parcial antes de continuar.

En Haskell, una función que puede tomar otras funciones como argumentos o devolver funciones se denomina función de orden superior.

Las siguientes son todas funciones de orden superior:

map       :: (a -> b) -> [a] -> [b]
filter    :: (a -> Bool) -> [a] -> [a]
takeWhile :: (a -> Bool) -> [a] -> [a]
dropWhile :: (a -> Bool) -> [a] -> [a]
iterate   :: (a -> a) -> a -> [a]
zipWith   :: (a -> b -> c) -> [a] -> [b] -> [c]
scanr     :: (a -> b -> b) -> b -> [a] -> [b]
scanl     :: (b -> a -> b) -> b -> [a] -> [b]

Estos son particularmente útiles porque nos permiten crear nuevas funciones además de las que ya tenemos, pasando funciones como argumentos a otras funciones. De ahí el nombre, funciones de orden superior.

Considerar:

Prelude> :t (map (+3))
(map (+3)) :: Num b => [b] -> [b]

Prelude> :t (map (=='c'))
(map (=='c')) :: [Char] -> [Bool]

Prelude> :t (map zipWith)
(map zipWith) :: [a -> b -> c] -> [[a] -> [b] -> [c]]

Esta capacidad de crear funciones fácilmente (como, por ejemplo, mediante una aplicación parcial, como se usa aquí) es una de las características que hace que la programación funcional sea particularmente poderosa y nos permite derivar soluciones breves y elegantes que, de lo contrario, tomarían docenas de líneas en otros lenguajes. Por ejemplo, la siguiente función nos da el número de elementos alineados en dos listas.

aligned :: [a] ->  [a] -> Int
aligned xs ys = length (filter id (zipWith (==) xs ys))

Expresiones Lambda #

Las expresiones lambda son similares a las funciones anónimas en otros idiomas.

Las expresiones lambda son [fórmulas abiertas] (https://en.wikipedia.org/wiki/Open_formula) que también especifican las variables que deben vincularse. La evaluación (encontrar el valor de una llamada de función) se logra sustituyendo las [variables enlazadas](https://en.wikipedia.org/ wiki/Bound_variable) en el cuerpo de la expresión lambda, con los argumentos proporcionados por el usuario. En pocas palabras, las expresiones lambda nos permiten expresar funciones mediante el enlace de variables y [sustitución] (https://en.wikipedia.org/wiki/Substitution_(logic)).

Las expresiones lambda parecen

\x -> let {y = ...x...} in y

Dentro de una expresión lambda, las variables en el lado izquierdo de la flecha se consideran limitadas en el lado derecho, es decir, el cuerpo de la función.

Considere la función matemática

f(x) = x^2

Como definición de Haskell es

f    x =  x^2

f = \x -> x^2

lo que significa que la función f es equivalente a la expresión lambda \x -> x^2.

Considere el parámetro de la función de orden superior mapa, que es una función de tipo a -> b. En caso de que se use solo una vez en una llamada a map y en ningún otro lugar del programa, es conveniente especificarlo como una expresión lambda en lugar de nombrar una función desechable. Escrito como una expresión lambda,

\x -> let {y = ...x...} in y

x tiene un valor de tipo a, ...x... es una expresión de Haskell que hace referencia a la variable x, y y tiene un valor de tipo b. Entonces, por ejemplo, podríamos escribir lo siguiente

map (\x -> x + 3)

map (\(x,y) -> x * y)

map (\xs -> 'c':xs) ["apples", "oranges", "mangos"]

map (\f -> zipWith f [1..5] [1..5]) [(+), (*), (-)]

curry #

En Haskell, todas las funciones se consideran curry: es decir, todas las funciones en Haskell toman solo un argumento.

Tomemos la función div:

div :: Int -> Int -> Int

Si llamamos a esta función con 6 y 2, como era de esperar, obtenemos 3:

Prelude> div 6 2
3

Sin embargo, esto no se comporta del todo de la manera que podríamos pensar. Primero se evalúa div 6 y devuelve una función de tipo Int -> Int. Esta función resultante se aplica luego al valor 2 que da 3.

Cuando miramos la firma de tipo de una función, podemos cambiar nuestro pensamiento de “toma dos argumentos de tipo Int” a “toma un Int y devuelve una función que toma un Int”. Esto se reafirma si consideramos que las flechas en la notación de tipos se asocian a la derecha, por lo que div puede leerse así:

div :: Int -> (Int -> Int)

En general, la mayoría de los programadores pueden ignorar este comportamiento al menos mientras aprenden el lenguaje. Desde un punto de vista teórico, “las pruebas formales son más fáciles cuando todas las funciones se tratan de manera uniforme (un argumento dentro, un resultado fuera)”.
