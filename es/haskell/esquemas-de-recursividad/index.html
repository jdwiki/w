Puntos fijos #

Fix toma un tipo de “plantilla” y ata el nudo recursivo, superponiendo la plantilla como una lasaña.

newtype Fix f = Fix { unFix :: f (Fix f) }

Dentro de un Fix f encontramos una capa de la plantilla f. Para completar el parámetro f, Fix f se conecta en sí mismo. Entonces, cuando miras dentro de la plantilla f, encuentras una ocurrencia recursiva de Fix f.

Así es como se puede traducir un tipo de datos recursivo típico a nuestro marco de plantillas y puntos fijos. Eliminamos las ocurrencias recursivas del tipo y marcamos sus posiciones usando el parámetro r.

{-# LANGUAGE DeriveFunctor #-}

-- natural numbers
-- data Nat = Zero | Suc Nat
data NatF r = Zero_ | Suc_ r deriving Functor
type Nat = Fix NatF

zero :: Nat
zero = Fix Zero_
suc :: Nat -> Nat
suc n = Fix (Suc_ n)


-- lists: note the additional type parameter a
-- data List a = Nil | Cons a (List a)
data ListF a r = Nil_ | Cons_ a r deriving Functor
type List a = Fix (ListF a)

nil :: List a
nil = Fix Nil_
cons :: a -> List a -> List a
cons x xs = Fix (Cons_ x xs)


-- binary trees: note two recursive occurrences
-- data Tree a = Leaf | Node (Tree a) a (Tree a)
data TreeF a r = Leaf_ | Node_ r a r deriving Functor
type Tree a = Fix (TreeF a)

leaf :: Tree a
leaf = Fix Leaf_
node :: Tree a -> a -> Tree a -> Tree a
node l x r = Fix (Node_ l x r)

Doblando una estructura una capa a la vez #

Catamorfismos, o pliegues, modelan la recursividad primitiva. cata derriba un punto fijo capa por capa, utilizando una función de álgebra (o función de plegado) para procesar cada capa. cata requiere una instancia de Functor para el tipo de plantilla f.

cata :: Functor f => (f a -> a) -> Fix f -> a
cata f = f . fmap (cata f) . unFix

-- list example
foldr :: (a -> b -> b) -> b -> List a -> b
foldr f z = cata alg
    where alg Nil_ = z
          alg (Cons_ x acc) = f x acc

Desplegando una estructura una capa a la vez #

Anamorfismos, o desdoblamientos, modela la correcursión primitiva. ana construye un punto fijo capa por capa, usando una función coalgebra (o función de despliegue) para producir cada nueva capa. ana requiere una instancia de Functor para el tipo de plantilla f.

ana :: Functor f => (a -> f a) -> a -> Fix f
ana f = Fix . fmap (ana f) . f

-- list example
unfoldr :: (b -> Maybe (a, b)) -> b -> List a
unfoldr f = ana coalg
    where coalg x = case f x of
                         Nothing -> Nil_
                         Just (x, y) -> Cons_ x y

Tenga en cuenta que ana y cata son dual. Los tipos y las implementaciones son imágenes especulares entre sí.

Desplegándose y luego plegándose, fusionado #

Es común estructurar un programa como si construyera una estructura de datos y luego la colapsara en un solo valor. Esto se llama hilomorfismo o repliegue. Es posible eludir la estructura intermedia por completo para mejorar la eficiencia.

hylo :: Functor f => (a -> f a) -> (f b -> b) -> a -> b
hylo f g = g . fmap (hylo f g) . f  -- no mention of Fix!

Derivación:

hylo f g = cata g . ana f
         = g . fmap (cata g) . unFix . Fix . fmap (ana f) . f  -- definition of cata and ana
         = g . fmap (cata g) . fmap (ana f) . f  -- unfix . Fix = id
         = g . fmap (cata g . ana f) . f  -- Functor law
         = g . fmap (hylo f g) . f  -- definition of hylo

Recurrencia primitiva #

Paramorfismos recursividad primitiva del modelo. En cada iteración del pliegue, la función de pliegue recibe el subárbol para su posterior procesamiento.

para :: Functor f => (f (Fix f, a) -> a) -> Fix f -> a
para f = f . fmap (\x -> (x, para f x)) . unFix

Las colas del Preludio se pueden modelar como un paramorfismo.

tails :: List a -> List (List a)
tails = para alg
    where alg Nil_ = cons nil nil  -- [[]]
          alg (Cons_ x (xs, xss)) = cons (cons x xs) xss  -- (x:xs):xss

Correcursión primitiva #

Apomorfismos modelo de correcursión primitiva. En cada iteración del despliegue, la función de despliegue puede devolver una nueva semilla o un subárbol completo.

apo :: Functor f => (a -> f (Either (Fix f) a)) -> a -> Fix f
apo f = Fix . fmap (either id (apo f)) . f

Tenga en cuenta que apo y para son dual. Las flechas en el tipo están invertidas; la tupla en para es dual a Either en apo, y las implementaciones son imágenes especulares entre sí.
