¿Qué es Plantilla Haskell? #

Plantilla Haskell se refiere a las instalaciones de metaprogramación de plantillas integradas en GHC Haskell. El artículo que describe la implementación original se puede encontrar aquí.

¿Qué son las etapas? (O, ¿cuál es la restricción de etapa?) #

Las etapas se refieren a cuándo se ejecuta el código. Normalmente, el código se ejecuta solo en tiempo de ejecución, pero con Template Haskell, el código se puede ejecutar en tiempo de compilación. El código “normal” es la etapa 0 y el código de tiempo de compilación es la etapa 1.

La restricción de etapa se refiere al hecho de que un programa de etapa 0 no puede ejecutarse en la etapa 1; esto sería equivalente a poder ejecutar cualquier programa regular (no solo un metaprograma) en tiempo de compilación.

Por convención (y en aras de la simplicidad de implementación), el código en el módulo actual siempre es la etapa 0 y el código importado de todos los demás módulos es la etapa 1. Por esta razón, solo se pueden empalmar expresiones de otros módulos.

Tenga en cuenta que un programa de etapa 1 es una expresión de etapa 0 de tipo Q Exp, Q Type, etc.; pero lo contrario no es cierto: no todos los valores (programa de etapa 0) del tipo Q Exp son programas de etapa 1,

Además, dado que los empalmes se pueden anidar, los identificadores pueden tener etapas mayores que 1. La restricción de etapa se puede generalizar: un programa de etapa n no se puede ejecutar en ninguna etapa m>n. Por ejemplo, se pueden ver referencias a tales etapas superiores a 1 en ciertos mensajes de error:

>:t [| \x -> $x |]

<interactive>:1:10: error:
    * Stage error: `x' is bound at stage 2 but used at stage 1
    * In the untyped splice: $x
      In the Template Haskell quotation [| \ x -> $x |]

¿Usar Plantilla Haskell provoca errores fuera del alcance de identificadores no relacionados? #

Normalmente, todas las declaraciones en un solo módulo de Haskell se pueden considerar como recursivas entre sí. En otras palabras, cada declaración de nivel superior está dentro del alcance de todas las demás en un solo módulo. Cuando Template Haskell está habilitado, las reglas de alcance cambian: el módulo se divide en grupos de código separados por empalmes TH, y cada grupo es mutuamente recursivo, y cada grupo está en el alcance de todos los demás grupos.

Sintaxis de Plantilla Haskell y Cuasicitas #

Plantilla Haskell está habilitada por la extensión GHC -XTemplateHaskell. Esta extensión habilita todas las funciones sintácticas que se detallan más adelante en esta sección. Los detalles completos de la plantilla Haskell se encuentran en la guía del usuario.

empalmes #

  • Un empalme es una nueva entidad sintáctica habilitada por Plantilla Haskell, escrita como $(...), donde (...) es alguna expresión.

  • No debe haber un espacio entre $ y el primer carácter de la expresión; y Template Haskell anula el análisis del operador $, p. f$g normalmente se analiza como ($) f g mientras que con Plantilla Haskell habilitada, se analiza como un empalme.

  • Cuando aparece un empalme en el nivel superior, se puede omitir el $. En este caso, la expresión empalmada es la línea completa.

  • Un empalme representa el código que se ejecuta en tiempo de compilación para producir un AST de Haskell, y ese AST se compila como código de Haskell y se inserta en el programa.

  • Los empalmes pueden aparecer en lugar de: expresiones, patrones, tipos y declaraciones de nivel superior. El tipo de la expresión empalmada, en cada caso respectivamente, es Q Exp, Q Pat, Q Type, Q [Decl]. Tenga en cuenta que los empalmes de declaración pueden solo aparecer en el nivel superior, mientras que los demás pueden estar dentro de otras expresiones, patrones o tipos, respectivamente.

Citas de expresión (nota: no una cuasicita) #

  • Una cita de expresión es una nueva entidad sintáctica escrita como una de:

  • [e|..|] o [|..|] - .. es una expresión y la cita tiene el tipo Q Exp;

  • [p|..|] - .. es un patrón y la cita tiene el tipo Q Pat;

  • [t|..|] - .. es un tipo y la cita tiene tipo Q Type;

  • [d|..|] - .. es una lista de declaraciones y la cita tiene el tipo Q [Dec].

  • Una cita de expresión toma un programa de tiempo de compilación y produce el AST representado por ese programa.

  • El uso de un valor en una cita (por ejemplo, \x -> [| x |]) sin un empalme corresponde al azúcar sintáctico para \x -> [| $(lift x) |], donde lift :: Lift t => t -> Q Exp proviene de la clase

    class Lift t where
      lift :: t -> Q Exp
      default lift :: Data t => t -> Q Exp

Empalmes mecanografiados y cotizaciones #

  • Los empalmes con tipo son similares a los empalmes (sin tipo) mencionados anteriormente y se escriben como $$(..) donde (..) es una expresión.

  • Si e tiene tipo Q (TEx a) entonces $$e tiene tipo a.

  • Las citas escritas toman la forma [||..||] donde .. es una expresión de tipo a; la cotización resultante es del tipo Q (TEx a).

  • La expresión con tipo se puede convertir a una sin tipo: unType :: TExp a -> Exp.

CuasiCotizaciones #

  • QuasiQuotes generaliza las citas de expresión: anteriormente, el analizador utilizado por la cita de expresión es uno de un conjunto fijo (e,p,t,d), pero QuasiQuotes permite definir y usar un analizador personalizado para producir código en tiempo de compilación . Las cuasi-citas pueden aparecer en todos los mismos contextos que las citas regulares.

  • Una cuasi-cita se escribe como [iden|...|], donde iden es un identificador de tipo Language.Haskell.TH.Quote.QuasiQuoter.

  • Un QuasiQuoter se compone simplemente de cuatro analizadores, uno para cada uno de los diferentes contextos en los que pueden aparecer las citas:

    data QuasiQuoter = QuasiQuoter { quoteExp  :: String -> Q Exp,
                                     quotePat  :: String -> Q Pat,
                                     quoteType :: String -> Q Type,
                                     quoteDec  :: String -> Q [Dec] }

nombres #

  • Los identificadores de Haskell están representados por el tipo Language.Haskell.TH.Syntax.Name. Los nombres forman las hojas de árboles de sintaxis abstracta que representan programas Haskell en Plantilla Haskell.

  • Un identificador que está actualmente dentro del alcance puede convertirse en un nombre con: 'e o 'T. En el primer caso, e se interpreta en el ámbito de la expresión, mientras que en el segundo caso, T está en el ámbito del tipo (recordando que los constructores de tipos y valores pueden compartir el nombre sin ambigüedad en Haskell).

El tipo Q #

El constructor de tipo Q :: * -> * definido en Language.Haskell.TH.Syntax es un tipo abstracto que representa cálculos que tienen acceso al entorno de tiempo de compilación del módulo en el que se ejecuta el cálculo. El tipo Q también maneja la sustitución de variables, llamada captura de nombre por TH (y discutida aquí.) Todos los empalmes tienen el tipo Q X para alguna X.

El entorno de tiempo de compilación incluye:

  • identificadores dentro del alcance e información sobre dichos identificadores,
  • tipos de funciones
  • tipos y tipos de datos fuente de constructores
  • especificación completa de las declaraciones de tipos (clases, familias de tipos)
  • la ubicación en el código fuente (línea, columna, módulo, paquete) donde ocurre el empalme
  • fijezas de funciones (GHC 7.10)
  • extensiones GHC habilitadas (GHC 8.0)

El tipo Q también tiene la capacidad de generar nuevos nombres, con la función newName :: String -> Q Name. Tenga en cuenta que el nombre no está vinculado en ningún lugar implícitamente, por lo que el usuario debe vincularlo por sí mismo, por lo que asegurarse de que el uso resultante del nombre tenga un alcance adecuado es responsabilidad del usuario.

Q tiene instancias para Functor,Monad,Applicative y esta es la interfaz principal para manipular los valores de Q, junto con los combinadores proporcionados en Language.Haskell.TH.Lib, que definen una función auxiliar para cada constructor del TH ast de la forma:

LitE :: Lit -> Exp
litE :: Lit -> ExpQ

AppE :: Exp -> Exp -> Exp 
appE :: ExpQ -> ExpQ -> ExpQ

Tenga en cuenta que ExpQ, TypeQ, DecsQ y PatQ son sinónimos de los tipos AST que normalmente se almacenan dentro del tipo Q.

La biblioteca TH proporciona una función runQ :: Quasi m => Q a -> m a, y hay una instancia Quasi IO, por lo que parecería que el tipo Q es simplemente un IO elegante. Sin embargo, el uso de runQ :: Q a -> IO a produce una acción IO que no tiene acceso a ningún entorno de tiempo de compilación; esto solo está disponible en el tipo Q actual. Tales acciones IO fallarán en el tiempo de ejecución si se intenta acceder a dicho entorno.

Un curry n-arity #

el conocido

curry :: ((a,b) -> c) -> a -> b -> c
curry = \f a b -> f (a,b)

La función se puede generalizar a tuplas de aridad arbitraria, por ejemplo:

curry3 :: ((a, b, c) -> d) -> a -> b -> c -> d
curry4 :: ((a, b, c, d) -> e) -> a -> b -> c -> d -> e

Sin embargo, escribir tales funciones para tuplas de arity 2 a (por ejemplo) 20 a mano sería tedioso (e ignorar el hecho de que la presencia de 20 tuplas en su programa casi seguramente indica problemas de diseño que deberían solucionarse con registros).

Podemos usar Template Haskell para producir tales funciones curryN para n arbitrarias:

{-# LANGUAGE TemplateHaskell #-}
import Control.Monad (replicateM) 
import Language.Haskell.TH (ExpQ, newName, Exp(..), Pat(..))
import Numeric.Natural (Natural) 

curryN :: Natural -> Q Exp

La función curryN toma un número natural y produce la función curry de esa aridad, como un AST de Haskell.

curryN n = do
  f  <- newName "f"
  xs <- replicateM (fromIntegral n) (newName "x")

Primero producimos variables de tipo nuevas para cada uno de los argumentos de la función: una para la función de entrada y otra para cada uno de los argumentos de dicha función.

  let args = map VarP (f:xs)

La expresión args representa el patrón f x1 x2 .. xn. Tenga en cuenta que un patrón es una entidad sintáctica separada: podríamos tomar este mismo patrón y colocarlo en una lambda, o en un enlace de función, o incluso en el LHS de un enlace let (lo que sería un error).

      ntup = TupE (map VarE xs)

La función debe construir la tupla de argumentos a partir de la secuencia de argumentos, que es lo que hemos hecho aquí. Tenga en cuenta la distinción entre variables de patrón (VarP) y variables de expresión (VarE).

  return $ LamE args (AppE (VarE f) ntup)

Finalmente, el valor que producimos es el AST \f x1 x2 .. xn -> f (x1, x2, .. , xn).

También podríamos haber escrito esta función usando comillas y constructores ’elevados':

...
import Language.Haskell.TH.Lib  

curryN' :: Natural -> ExpQ
curryN' n = do
  f  <- newName "f"
  xs <- replicateM (fromIntegral n) (newName "x")
  lamE (map varP (f:xs)) 
        [| $(varE f) $(tupE (map varE xs)) |]

Tenga en cuenta que las citas deben ser sintácticamente válidas, por lo que [| \ $(map varP (f:xs)) -> .. |] no es válido, porque Haskell normal no tiene forma de declarar una ’lista’ de patrones; lo anterior se interpreta como \ var -> .. y se espera que la expresión empalmada tenga el tipo PatQ, es decir, un solo patrón, no una lista de patrones.

Finalmente, podemos cargar esta función TH en GHCi:

>:set -XTemplateHaskell
>:t $(curryN 5)
$(curryN 5)
  :: ((t1, t2, t3, t4, t5) -> t) -> t1 -> t2 -> t3 -> t4 -> t5 -> t
>$(curryN 5) (\(a,b,c,d,e) -> a+b+c+d+e) 1 2 3 4 5
15
