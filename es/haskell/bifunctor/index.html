Sintaxis #

  • bimapa :: (a -> b) -> (c -> d) -> p a c -> p b d
  • primero :: (a -> b) -> p a c -> p b c
  • segundo :: (b -> c) -> pab -> pac

Una ejecución del molino Functor es covariante en un parámetro de tipo único. Por ejemplo, si f es un Functor, entonces dado f a y una función de la forma a -> b, se puede obtener f b (mediante el uso de fmap).

Un Bifunctor es covariante en dos parámetros de tipo. Si f es un Bifunctor, entonces dado f a b, y dos funciones, una de a -> c, y otra de b -> d, entonces uno puede obtener f c d (usando bimapa).

“primero” debe considerarse como un “fmap” sobre el primer parámetro de tipo, “segundo” como un “fmap” sobre el segundo, y “bimap” debe concebirse como el mapeo de dos funciones de forma covariante sobre el primer y segundo tipo de parámetros, respectivamente.

Instancias comunes de Bifunctor #

tuplas de dos elementos #

(,) es un ejemplo de un tipo que tiene una instancia Bifunctor.

instance Bifunctor (,) where
    bimap f g (x, y) = (f x, g y)

bimap toma un par de funciones y las aplica a los componentes respectivos de la tupla.

bimap (+ 2) (++ "nie") (3, "john") --> (5,"johnnie")
bimap ceiling length (3.5 :: Double, "john" :: String) --> (4,4)

Cualquiera #

Cualquiera de las instancias de Bifunctor selecciona una de las dos funciones para aplicar dependiendo de si el valor es Izquierda o Derecha.

instance Bifunctor Either where
    bimap f g (Left x) = Left (f x)
    bimap f g (Right y) = Right (g y)

Definición de bifuntor #

Bifunctor es la clase de tipos con dos parámetros de tipo (f :: * -> * -> *), los cuales se pueden mapear de forma covariante simultáneamente.

class Bifunctor f where
    bimap :: (a -> c) -> (b -> d) -> f a b -> f c d

bimap se puede considerar como la aplicación de un par de operaciones fmap a un tipo de datos.

Una instancia correcta de Bifunctor para un tipo f debe satisfacer las leyes de bifunctor, que son análogas a las [leyes de functor](https://www.wikiod.com/es/haskell/funtor#Definición de clase de funtor y leyes -funtor#t=201611271637470659956):

bimap id id = id  -- identity
bimap (f . g) (h . i) = bimap f h . bimap g i  -- composition

La clase Bifunctor se encuentra en el módulo Data.Bifunctor. Para las versiones de GHC >7.10, este módulo se incluye con el compilador; para versiones anteriores necesitas instalar el paquete bifunctors.

primero y segundo #

Si se desea mapear covariantemente solo sobre el primer argumento, o solo sobre el segundo argumento, entonces se debe usar first o second (en lugar de bimap).

first :: Bifunctor f => (a -> c) -> f a b -> f c b
first f = bimap f id

second :: Bifunctor f => (b -> d) -> f a b -> f a d
second g = bimap id g

Por ejemplo,

ghci> second (+ 2) (Right 40)
Right 42
ghci> second (+ 2) (Left "uh oh")
Left "uh oh"
