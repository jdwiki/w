Obteniendo la ‘a’ “fuera de” ‘IO a’ #

Una pregunta común es “Tengo un valor de IO a, pero quiero hacer algo con ese valor a: ¿cómo puedo acceder a él?” ¿Cómo se puede operar con datos que vienen del mundo exterior (por ejemplo, incrementando un número tecleado por el usuario)?

El punto es que si usa una función pura en datos obtenidos de forma impura, entonces el resultado sigue siendo impuro. ¡Depende de lo que hizo el usuario! Un valor de tipo ‘IO a’ representa un “cálculo de efecto secundario que da como resultado un valor de tipo ‘a’” que solo puede ejecutarse (a) componiéndolo en ‘principal’ y (b) compilando y ejecutando tu programa Por esa razón, no hay forma dentro del mundo puro de Haskell de “sacar la ‘a’”.

En su lugar, queremos construir un nuevo cálculo, un nuevo valor IO, que haga uso del valor a en tiempo de ejecución. Esta es otra forma de componer valores IO y, de nuevo, podemos usar la notación do:

-- assuming
myComputation :: IO Int

getMessage :: Int -> String
getMessage int = "My computation resulted in: " ++ show int
 
newComputation :: IO ()
newComputation = do
  int <- myComputation       -- we "bind" the result of myComputation to a name, 'int'
  putStrLn $ getMessage int   -- 'int' holds a value of type Int

Aquí estamos usando una función pura (getMessage) para convertir un Int en una String, pero estamos usando la notación do para que se aplique al resultado de un cálculo IO myComputation cuando (después) se ejecuta el cálculo. El resultado es un cálculo IO más grande, newComputation. Esta técnica de usar funciones puras en un contexto impuro se llama lifting.

IO define la acción principal de su programa #

Para hacer ejecutable un programa de Haskell, debe proporcionar un archivo con una función principal de tipo IO ()

main :: IO ()
main = putStrLn "Hello world!"

Cuando se compila Haskell, examina los datos IO aquí y los convierte en un ejecutable. Cuando ejecutemos este programa, imprimirá ¡Hola mundo!.

Si tiene valores de tipo IO a que no sean main, no harán nada.

other :: IO ()
other = putStrLn "I won't get printed"

main :: IO ()
main = putStrLn "Hello world!"

Compilar este programa y ejecutarlo tendrá el mismo efecto que el último ejemplo. El código en otro se ignora.

Para hacer que el código en “otro” tenga efectos de tiempo de ejecución, debe componerlo en “principal”. Ningún valor ‘IO’ que no se haya compuesto finalmente en ‘main’ tendrá ningún efecto en el tiempo de ejecución. Para componer dos valores ‘IO’ secuencialmente, puede usar la notación ‘do’:

other :: IO ()
other = putStrLn "I will get printed... but only at the point where I'm composed into main"

main :: IO ()
main = do 
  putStrLn "Hello world!"
  other

Cuando compilas y ejecutas este programa, genera

Hello world!
I will get printed... but only at the point where I'm composed into main

Tenga en cuenta que el orden de las operaciones se describe por cómo “otro” se compuso en “principal” y no por el orden de definición.

Comprobación de las condiciones de fin de archivo #

Un poco contrario a la intuición de la forma en que lo hacen la mayoría de las bibliotecas de E/S estándar de otros lenguajes, isEOF de Haskell no requiere que realice una operación de lectura antes de verificar una condición EOF; el tiempo de ejecución lo hará por usted.

import System.IO( isEOF )


eofTest :: Int -> IO ()
eofTest line = do
    end <- isEOF
    if end then
        putStrLn $ "End-of-file reached at line " ++ show line ++ "."
    else do
        getLine
        eofTest $ line + 1


main :: IO ()
main =
    eofTest 1

Aporte:

Line #1.
Line #2.
Line #3.

Producción:

End-of-file reached at line 4.

Leer todo el contenido de la entrada estándar en una cadena #

main = do
    input <- getContents
    putStr input

Aporte:

This is an example sentence.
And this one is, too!

Producción:

This is an example sentence.
And this one is, too!

Nota: este programa realmente imprimirá partes de la salida antes de que toda la entrada se haya leído por completo. Esto significa que, por ejemplo, si usa getContents sobre un archivo de 50MiB, la evaluación perezosa y el recolector de basura de Haskell se asegurarán de que solo las partes del archivo que se necesitan actualmente (léase: indispensables para una ejecución posterior) se cargarán en la memoria. Por lo tanto, el archivo de 50MiB no se cargará en la memoria de inmediato.

Rol y Propósito de IO #

Haskell es un lenguaje puro, lo que significa que las expresiones no pueden tener efectos secundarios. Un efecto secundario es cualquier cosa que hace la expresión o función que no sea producir un valor, por ejemplo, modificar un contador global o imprimir a la salida estándar.

En Haskell, los cálculos de efectos secundarios (específicamente, aquellos que pueden tener un efecto en el mundo real) se modelan usando IO. Estrictamente hablando, ‘IO’ es un constructor de tipos, tomando un tipo y produciendo un tipo. Por ejemplo, IO Int es el tipo de cálculo de E/S que produce un valor Int. El tipo IO es abstracto, y la interfaz provista para IO asegura que ciertos valores ilegales (es decir, funciones con tipos sin sentido) no pueden existir, asegurando que todas las funciones integradas que realizan IO tengan un tipo de retorno encerrado en IO.

Cuando se ejecuta un programa de Haskell, se ejecuta el cálculo representado por el valor de Haskell denominado main, cuyo tipo puede ser IO x para cualquier tipo x.

Manipulación de valores IO #

Hay muchas funciones en la biblioteca estándar que proporcionan acciones “IO” típicas que debe realizar un lenguaje de programación de propósito general, como leer y escribir en identificadores de archivos. Las acciones generales IO se crean y combinan principalmente con dos funciones:

 (>>=) :: IO a -> (a -> IO b) -> IO b

Esta función (típicamente llamada bind) toma una acción ‘IO’ y una función que devuelve una acción ‘IO’, y produce la acción ‘IO’ que es el resultado de aplicar la función al valor producido por el primer ‘IO’ ` acción.

 return :: a -> IO a

Esta función toma cualquier valor (es decir, un valor puro) y devuelve el cálculo de IO que no hace IO y produce el valor dado. En otras palabras, es una acción de E/S sin operación.

Hay funciones generales adicionales que se usan con frecuencia, pero todas se pueden escribir en términos de las dos anteriores. Por ejemplo, (>>) :: IO a -> IO b -> IO b es similar a (>>=) pero se ignora el resultado de la primera acción.

Un programa simple que saluda al usuario usando estas funciones:

 main :: IO ()
 main =
   putStrLn "What is your name?" >>
   getLine >>= \name ->
   putStrLn ("Hello " ++ name ++ "!")

Este programa también usa putStrLn :: String -> IO () y getLine :: IO String.

Nota: los tipos de ciertas funciones anteriores son en realidad más generales que los tipos dados (es decir, >>=, >> y return).

semántica de E/S #

El tipo IO en Haskell tiene una semántica muy similar a la de los lenguajes de programación imperativos. Por ejemplo, cuando uno escribe s1; s2 en un lenguaje imperativo para indicar la ejecución de la declaración s1, luego la declaración s2, uno puede escribir s1 >> s2 para modelar lo mismo en Haskell.

Sin embargo, la semántica de IO diverge ligeramente de lo que se esperaría viniendo de un trasfondo imperativo. La función return no interrumpe el flujo de control; no tiene efecto en el programa si se ejecuta otra acción IO en secuencia. Por ejemplo, return () >> putStrLn "boom" imprime correctamente “boom” en la salida estándar.

La semántica formal de ‘IO’ se puede dar en términos de igualdades simples que involucran las funciones de la sección anterior:

 return x >>= f ≡ f x, ∀ f x
 y >>= return ≡ return y, ∀ y
 (m >>= f) >>= g ≡ m >>= (\x -> (f x >>= g)), ∀ m f g

Estas leyes suelen denominarse identidad izquierda, identidad derecha y composición, respectivamente. Pueden expresarse de manera más natural en términos de la función

 (>=>) :: (a -> IO b) -> (b -> IO c) -> a -> IO c
 (f >=> g) x = (f x) >>= g

como sigue:

 return >=> f ≡ f, ∀ f
 f >=> return ≡ f, ∀ f
 (f >=> g) >=> h ≡ f >=> (g >=> h), ∀ f g h

E/S perezoso #

Las funciones que realizan cálculos de E/S suelen ser estrictas, lo que significa que todas las acciones anteriores en una secuencia de acciones deben completarse antes de que comience la siguiente acción. Por lo general, este es un comportamiento útil y esperado: putStrLn "X" >> putStrLn "Y" debería imprimir “XY”. Sin embargo, ciertas funciones de biblioteca realizan operaciones de E/S con pereza, lo que significa que las acciones de E/S necesarias para producir el valor solo se realizan cuando el valor se consume realmente. Ejemplos de tales funciones son getContents y readFile. La E/S perezosa puede reducir drásticamente el rendimiento de un programa de Haskell, por lo que al usar funciones de biblioteca, se debe tener cuidado para observar qué funciones son perezosas.

Notación IO y do #

Haskell proporciona un método más simple para combinar diferentes valores de IO en valores de IO más grandes. Esta sintaxis especial se conoce como notación do* y es simplemente azúcar sintáctica para los usos de las funciones >>=, >> y return.

El programa de la sección anterior se puede escribir de dos maneras diferentes usando la notación do, la primera siendo sensible al diseño y la segunda siendo insensible al diseño:

 main = do
   putStrLn "What is your name?"
   name <- getLine
   putStrLn ("Hello " ++ name ++ "!")


 main = do {
   putStrLn "What is your name?" ;
   name <- getLine ;
   putStrLn ("Hello " ++ name ++ "!")
   }

Los tres programas son exactamente equivalentes.

*Tenga en cuenta que la notación do también es aplicable a una clase más amplia de constructores de tipos llamados mónadas.

Leer una línea desde la entrada estándar #

Lectura de palabras de un archivo completo #

En Haskell, a menudo tiene sentido no preocuparse por los identificadores de archivos, sino simplemente leer o escribir un archivo completo directamente desde el disco a la memoria†, y realizar todas las particiones/procesamiento del texto con el estructura de datos de cadena pura. Esto evita mezclar IO y lógica de programa, lo que puede ser de gran ayuda para evitar errores.

-- | The interesting part of the program, which actually processes data
--   but doesn't do any IO!
reverseWords :: String -> [String]
reverseWords = reverse . words

-- | A simple wrapper that only fetches the data from disk, lets
--   'reverseWords' do its job, and puts the result to stdout.
main :: IO ()
main = do
   content <- readFile "loremipsum.txt"
   mapM_ putStrLn $ reverseWords content

Si loremipsum.txt contiene

Lorem ipsum dolor sit amet,
consectetur adipiscing elit

entonces el programa generará

elit
adipiscing
consectetur
amet,
sit
dolor
ipsum
Lorem

Aquí, mapM_ revisó la lista de todas las palabras en el archivo y imprimió cada uno de ellos en una línea separada con putStrLn.

†Si crees que esto es un desperdicio de memoria, tienes razón. En realidad, la pereza de Haskell a menudo puede evitar que todo el archivo deba residir en la memoria simultáneamente… pero cuidado, este tipo de IO perezoso causa su propio conjunto de problemas. Para aplicaciones críticas para el rendimiento, a menudo tiene sentido forzar que todo el archivo se lea de una vez, estrictamente; puede hacer esto con [la versión Data.Text de readFile.](http://hackage.haskell.org/package/text-1.2.2.1/docs/Data-Text-IO.html#v: leer archivo)

Escribiendo en la salida estándar #

De acuerdo con la Especificación de lenguaje de Haskell 2010, las siguientes son funciones de E/S estándar disponibles en Prelude, por lo que no es necesario importarlas para usarlas.

putChar :: Char -> IO () - escribe un char en stdout #

Prelude> putChar 'a'
aPrelude>  -- Note, no new line

putStr :: String -> IO () - escribe una String en stdout #

Prelude> putStr "This is a string!"
This is a string!Prelude>  -- Note, no new line

putStrLn :: String -> IO () - escribe una String en stdout y agrega una nueva línea #

Prelude> putStrLn "Hi there, this is another String!"
Hi there, this is another String!

print :: Show a => a -> IO () - escribe a una instancia de Show en stdout #

Prelude> print "hi"
"hi"
Prelude> print 1
1
Prelude> print 'a'
'a'
Prelude> print (Just 'a')  -- Maybe is an instance of the `Show` type class
Just 'a'
Prelude> print Nothing
Nothing

Recuerda que puedes instanciar Mostrar para tus propios tipos usando derivar:

-- In ex.hs
data Person = Person { name :: String } deriving Show
main = print (Person "Alex")  -- Person is an instance of `Show`, thanks to `deriving`

Cargando y ejecutando en GHCi:

Prelude> :load ex.hs
[1 of 1] Compiling ex             ( ex.hs, interpreted )
Ok, modules loaded: ex.
*Main> main  -- from ex.hs
Person {name = "Alex"}
*Main>

Análisis y construcción de un objeto a partir de una entrada estándar #

readFloat :: IO Float
readFloat =
    fmap read getLine


main :: IO ()
main = do
    putStr "Type the first number: "
    first <- readFloat

    putStr "Type the second number: "
    second <- readFloat

    putStrLn $ show first ++ " + " ++ show second ++ " = " ++ show ( first + second )

Aporte:

Type the first number: 9.5
Type the second number: -2.02

Producción:

9.5 + -2.02 = 7.48

Lectura de identificadores de archivos #

Al igual que en varias otras partes de la biblioteca de E/S, las funciones que utilizan implícitamente un flujo estándar tienen una contraparte en System.IO que realiza el mismo trabajo, pero con un parámetro adicional a la izquierda, de tipo Handle, que representa el flujo que se está manejando. Por ejemplo, getLine :: IO String tiene una contrapartida hGetLine :: Handle -> IO String.

import System.IO( Handle, FilePath, IOMode( ReadMode ), 
                  openFile, hGetLine, hPutStr, hClose, hIsEOF, stderr )

import Control.Monad( when )


dumpFile :: Handle -> FilePath -> Integer -> IO ()

dumpFile handle filename lineNumber = do      -- show file contents line by line
    end <- hIsEOF handle
    when ( not end ) $ do
        line <- hGetLine handle
        putStrLn $ filename ++ ":" ++ show lineNumber ++ ": " ++ line
        dumpFile handle filename $ lineNumber + 1


main :: IO ()

main = do
    hPutStr stderr "Type a filename: "
    filename <- getLine
    handle <- openFile filename ReadMode
    dumpFile handle filename 1
    hClose handle

Contenido del archivo ejemplo.txt:

This is an example.
Hello, world!
This is another example.

Aporte:

Type a filename: example.txt

Producción:

example.txt:1: This is an example.
example.txt:2: Hello, world!
example.txt:3: This is another example

Lectura desde stdin #

Según la Especificación de lenguaje de Haskell 2010, las siguientes son funciones de E/S estándar disponibles en Prelude, por lo que no se requieren importaciones para usarlas.

getChar :: IO Char - lee un Char de stdin #

-- MyChar.hs
main = do
  myChar <- getChar
  print myChar

-- In your shell

runhaskell MyChar.hs
a -- you enter a and press enter
'a'  -- the program prints 'a'

getLine :: IO String - lee una String de stdin, sin carácter de nueva línea #

Prelude> getLine
Hello there!  -- user enters some text and presses enter
"Hello there!"

read :: Read a => String -> a - convierte una cadena en un valor #

Prelude> read "1" :: Int
1
Prelude> read "1" :: Float
1.0
Prelude> read "True" :: Bool
True

Otras funciones menos comunes son:

  • getContents :: IO String: devuelve todas las entradas del usuario como una sola cadena, que se lee perezosamente cuando se necesita
  • interact :: (String -> String) -> IO () - toma una función de tipo String->String como su argumento. Toda la entrada del dispositivo de entrada estándar se pasa a esta función como su argumento
