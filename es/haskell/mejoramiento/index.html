Compilando su programa para la creación de perfiles #

El compilador de GHC tiene soporte maduro para compilar con anotaciones de perfiles.

El uso de las banderas -prof y -fprof-auto al compilar agregará soporte a su binario para perfilar banderas para usar en tiempo de ejecución.

Supongamos que tenemos este programa:

main = print (fib 30)
fib n = if n < 2 then 1 else fib (n-1) + fib (n-2)

Compilado así:

ghc -prof -fprof-auto -rtsopts Main.hs

Luego lo ejecutó con las opciones del sistema de tiempo de ejecución para la creación de perfiles:

./Main +RTS -p

Veremos un archivo main.prof creado después de la ejecución (una vez que el programa haya salido), y esto nos dará todo tipo de información de perfiles, como centros de costos, que nos brinda un desglose del costo asociado con la ejecución de las diversas partes de el código:

Centros de costo #

Los centros de costos son anotaciones en un programa de Haskell que el compilador GHC puede agregar automáticamente, usando -fprot-auto, o un programador usando {-# SCC "name" #-} <expression>, donde “nombre” es cualquier nombre que desee y <expresión> es cualquier expresión válida de Haskell:

-- Main.hs
main :: IO ()
main = do let l = [1..9999999]
          print $ {-# SCC "print_list" #-} (length l)

Compilar con -fprof y ejecutar con +RTS -p, p. ghc -prof -rtsopts Main.hs && ./Main.hs +RTS -p produciría Main.prof una vez que se haya cerrado el programa.
