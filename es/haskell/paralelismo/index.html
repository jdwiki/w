Parámetros #

Tipo/Función Detalle
data Eval a Eval es una Mónada que facilita la definición de estrategias paralelas
type Strategy a = a -> Eval a una función que incorpora una estrategia de evaluación paralela. La función atraviesa (partes de) su argumento, evaluando subexpresiones en paralelo o en secuencia
rpar :: Estrategia a despierta su argumento (para evaluación en paralelo)
rseq :: Estrategia a evalúa su argumento a la forma normal cabeza débil
force :: NFData a => a -> a evalúa la estructura completa de su argumento, reduciéndolo a su forma normal, antes de devolver el argumento en sí. Lo proporciona el módulo Control.DeepSeq

El libro de Simon Marlow, Concurrent and Parallel Programming in Haskell, es sobresaliente y cubre una multitud de conceptos. También es muy accesible incluso para el programador Haskell más nuevo. Es muy recomendable y está disponible en PDF o en línea de forma gratuita.

Paralelo vs Concurrente

Simon Marlow lo expresa mejor:

Un programa paralelo es aquel que utiliza una multiplicidad de hardware computacional (por ejemplo, varios núcleos de procesador) para realizar un cálculo más rápidamente. El objetivo es llegar antes a la respuesta, delegando diferentes partes del cálculo a diferentes procesadores que se ejecutan al mismo tiempo.

Por el contrario, la concurrencia es una técnica de estructuración de programas en la que existen múltiples hilos de control. Conceptualmente, los hilos de control se ejecutan “al mismo tiempo”; es decir, el usuario ve sus efectos intercalados. Si realmente se ejecutan al mismo tiempo o no, es un detalle de implementación; un programa concurrente puede ejecutarse en un solo procesador a través de la ejecución intercalada o en múltiples procesadores físicos.

Forma normal de cabeza débil

Es importante ser consciente de cómo funciona la evaluación perezosa. La primera sección de este capítulo brindará una sólida introducción a WHNF y cómo se relaciona con la programación paralela y concurrente.

La Mónada Eval #

El paralelismo en Haskell se puede expresar usando la Mónada Eval de Control.Parallel.Strategies, usando las funciones rpar y rseq (entre otras).

f1 :: [Int]
f1 = [1..100000000]

f2 :: [Int]
f2 = [1..200000000]

main = runEval $ do
  a <- rpar (f1) -- this'll take a while...
  b <- rpar (f2) -- this'll take a while and then some...
  return (a,b)

Ejecutar main arriba ejecutará y “regresará” inmediatamente, mientras que los dos valores, a y b se calculan en segundo plano a través de rpar.

Nota: asegúrese de compilar con -threaded para que se produzca la ejecución en paralelo.

par #

rpar :: Estrategia a ejecuta la estrategia dada (recuerde: escriba Estrategia a = a -> Eval a) en paralelo:

import Control.Concurrent
import Control.DeepSeq
import Control.Parallel.Strategies
import Data.List.Ordered

main = loop
  where 
    loop = do
      putStrLn "Enter a number"
      n <- getLine

      let lim = read n :: Int
          hf  = quot lim 2
          result = runEval $ do
            -- we split the computation in half, so we can concurrently calculate primes
            as <- rpar (force (primesBtwn 2 hf))
            bs <- rpar (force (primesBtwn (hf + 1) lim))
            return (as ++ bs)

      forkIO $ putStrLn ("\nPrimes are: " ++ (show result) ++ " for " ++ n ++ "\n")
      loop

-- Compute primes between two integers
-- Deliberately inefficient for demonstration purposes
primesBtwn n m = eratos [n..m]
  where
    eratos []     = []
    eratos (p:xs) = p : eratos (xs `minus` [p, p+p..])

rseq #

Podemos usar rseq :: Strategy a para forzar un argumento a la forma normal de cabeza débil:

f1 :: [Int]
f1 = [1..100000000]

f2 :: [Int]
f2 = [1..200000000]

main = runEval $ do
  a <- rpar (f1) -- this'll take a while...
  b <- rpar (f2) -- this'll take a while and then some...
  rseq a
  return (a,b)

Esto cambia sutilmente la semántica del ejemplo rpar; mientras que este último regresaría inmediatamente mientras se calculan los valores en segundo plano, este ejemplo esperará hasta que a pueda evaluarse como WHNF.
