Sintaxis #

  • Foreign import ccall unsafe “foo” hFoo :: Int32 -> IO Int32 {- Importa una función llamada foo en algún archivo de objeto y define el símbolo hFoo que se puede llamar con código Haskell. -}

Si bien cabal admite la inclusión de bibliotecas C y C++ en un paquete de Haskell, existen algunos errores. Primero, si tiene datos (en lugar de una función) definidos en b.o que se usan en a.o, y enumera las C-fuentes: a.c, b.c, entonces cabal no podrá encontrar los datos. Esto está documentado en #12152. Una solución al usar cabal es reordenar la lista de C-fuentes para que sea C-fuentes: b.c, a.c. Es posible que esto no funcione cuando se usa stack, porque stack siempre vincula las ‘fuentes C’ alfabéticamente, independientemente del orden en que las enumeres.

Otro problema es que debe rodear cualquier código C++ en los archivos de encabezado (.h) con protecciones #ifdef __cplusplus. Esto se debe a que GHC no entiende el código C++ en los archivos de encabezado. Todavía puede escribir código C++ en archivos de encabezado, pero debe rodearlo con guardias.

ccall se refiere a la convención de llamadas; actualmente se admiten ccall y stdcall (convención de Pascal). La palabra clave inseguro es opcional; esto reduce la sobrecarga de funciones simples, pero puede causar interbloqueos si la función externa se bloquea indefinidamente o no tiene permisos suficientes para ejecutar1.

Llamando a C desde Haskell #

Por motivos de rendimiento, o debido a la existencia de bibliotecas C maduras, es posible que desee llamar al código C desde un programa Haskell. Aquí hay un ejemplo simple de cómo puede pasar datos a una biblioteca C y obtener una respuesta.

foo.c:

#include <inttypes.h>

int32_t foo(int32_t a) {
  return a+1;
}

Foo.hs:

import Data.Int

main :: IO ()
main = print =<< hFoo 41

foreign import ccall unsafe "foo" hFoo :: Int32 -> IO Int32

La palabra clave ‘inseguro’ genera una llamada más eficiente que ‘seguro’, pero requiere que el código C nunca devuelva la llamada al sistema Haskell. Dado que foo está completamente en C y nunca llamará a Haskell, podemos usar unsafe.

También necesitamos instruir a cabal para compilar y vincular en fuente C.

foo.cabal:

name:                foo
version:             0.0.0.1
build-type:          Simple
extra-source-files:  *.c
cabal-version:       >= 1.10

executable foo
  default-language: Haskell2010
  main-is:       Foo.hs
  C-sources:     foo.c
  build-depends: base

Entonces puedes ejecutar:

> cabal configure
> cabal build foo
> ./dist/build/foo/foo
42

Pasar funciones de Haskell como devoluciones de llamada al código C. #

Es muy común que las funciones de C acepten punteros a otras funciones como argumentos. El ejemplo más popular es configurar una acción para que se ejecute cuando se hace clic en un botón en alguna biblioteca de herramientas GUI. Es posible pasar funciones de Haskell como devoluciones de llamada C.

Para llamar a esta función C:

void event_callback_add (Object *obj, Object_Event_Cb func, const void *data)

primero lo importamos al código de Haskell:

foreign import ccall "header.h event_callback_add"
    callbackAdd :: Ptr () -> FunPtr Callback -> Ptr () -> IO ()

Ahora viendo cómo se define Object_Event_Cb en el encabezado C, defina qué es Callback en Haskell:

type Callback = Ptr () -> Ptr () -> IO ()

Finalmente, cree una función especial que envuelva la función Haskell de tipo Callback en un puntero FunPtr Callback:

foreign import ccall "wrapper"
    mkCallback :: Callback -> IO (FunPtr Callback)

Ahora podemos registrar la devolución de llamada con el código C:

cbPtr <- mkCallback $ \objPtr dataPtr -> do
    -- callback code
    return ()
callbackAdd cpPtr

Es importante liberar ‘FunPtr’ asignado una vez que anule el registro de la devolución de llamada:

freeHaskellFunPtr cbPtr
