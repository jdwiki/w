Inyectar eventos externos en la biblioteca #

Este ejemplo no está vinculado a ningún conjunto de herramientas de GUI concreto, como lo hace, por ejemplo, reactive-banana-wx. En su lugar, muestra cómo inyectar acciones ‘IO’ arbitrarias en la maquinaria FRP.

El módulo Control.Event.Handler proporciona una función addHandler que crea un par de valores AddHandler a y a -> IO (). Reactive-banana utiliza la primera para obtener un valor de ‘Evento a’, mientras que la última es una función simple que se usa para desencadenar el evento correspondiente.

import Data.Char (toUpper)

import Control.Event.Handler
import Reactive.Banana

main = do
    (inputHandler, inputFire) <- newAddHandler

En nuestro caso, el parámetro a del controlador es de tipo String, pero el código que permite al compilador inferir eso se escribirá más adelante.

Ahora definimos la EventNetwork que describe nuestro sistema controlado por FRP. Esto se hace usando la función compilar:

main = do
    (inputHandler, inputFire) <- newAddHandler
    compile $ do
        inputEvent <- fromAddHandler inputHandler

La función fromAddHandler transforma el valor de AddHandler a en un Evento a, que se trata en el siguiente ejemplo.

Finalmente, lanzamos nuestro “bucle de eventos”, que dispararía eventos en la entrada del usuario:

main = do
    (inputHandler, inputFire) <- newAddHandler
    compile $ do
        ...
    forever $ do
        input <- getLine
        inputFire input

Tipo de evento #

En reactivo-banana, el tipo Evento representa un flujo de algunos eventos en el tiempo. Un ‘Evento’ es similar a una señal de impulso analógica en el sentido de que no es continua en el tiempo. Como resultado, Event es una instancia de la clase de tipos Functor solamente. No puede combinar dos ‘Eventos’ juntos porque pueden activarse en diferentes momentos. Puede hacer algo con el valor [actual] de un Evento y reaccionar con alguna acción IO.

Las transformaciones en el valor de Event se realizan usando fmap:

main = do
    (inputHandler, inputFire) <- newAddHandler
    compile $ do
        inputEvent <- fromAddHandler inputHandler
        -- turn all characters in the signal to upper case
        let inputEvent' = fmap (map toUpper) inputEvent

Reaccionar a un Evento se hace de la misma manera. Primero lo ‘fmap’ con una acción de tipo ‘a -> IO ()’ y luego lo pasa a la función ‘reaccionar’:

main = do
    (inputHandler, inputFire) <- newAddHandler
    compile $ do
        inputEvent <- fromAddHandler inputHandler
        -- turn all characters in the signal to upper case
        let inputEvent' = fmap (map toUpper) inputEvent
        let inputEventReaction = fmap putStrLn inputEvent' -- this has type `Event (IO ())
        reactimate inputEventReaction

Ahora cada vez que se llame inputFire "algo", se imprimirá "ALGO".

Tipo de comportamiento #

Para representar señales continuas, reactivo-banana presenta el tipo Comportamiento a. A diferencia de Event, un Behavior es un Applicative, que le permite combinar n Behaviors usando una función pura n-aria (usando <$> y <*>).

Para obtener un Comportamiento a del Evento a existe la función accumE:

main = do
    (inputHandler, inputFire) <- newAddHandler
    compile $ do
        ...
        inputBehavior <- accumE "" $ fmap (\oldValue newValue -> newValue) inputEvent

accumE toma el valor inicial de Behavior y un Event, que contiene una función que lo establecería en el nuevo valor.

Al igual que con los ‘Eventos’, puede usar ‘fmap’ para trabajar con el valor actual de ‘Comportamiento’, pero también puede combinarlos con ‘(<*>)’.

main = do
    (inputHandler, inputFire) <- newAddHandler
    compile $ do
        ...
        inputBehavior  <- accumE "" $ fmap (\oldValue newValue -> newValue) inputEvent
        inputBehavior' <- accumE "" $ fmap (\oldValue newValue -> newValue) inputEvent
        let constantTrueBehavior = (==) <$> inputBehavior <*> inputBehavior'

Para reaccionar a los cambios de ‘Comportamiento’ hay una función de ‘cambios’:

main = do
    (inputHandler, inputFire) <- newAddHandler
    compile $ do
        ...
        inputBehavior  <- accumE "" $ fmap (\oldValue newValue -> newValue) inputEvent
        inputBehavior' <- accumE "" $ fmap (\oldValue newValue -> newValue) inputEvent
        let constantTrueBehavior = (==) <$> inputBehavior <*> inputBehavior'
        inputChanged <- changes inputBehavior

Lo único que se debe tener en cuenta es que los cambios devuelven Evento (Futuro a) en lugar de Evento a. Debido a esto, se debe usar rectimate' en lugar de rectimate. La razón detrás de esto se puede obtener de la documentación.

Activación de redes de eventos #

Los EventNetworks devueltos por compile deben activarse antes de que los eventos reactivados tengan efecto.

main = do
    (inputHandler, inputFire) <- newAddHandler

    eventNetwork <- compile $ do
        inputEvent <- fromAddHandler inputHandler
        let inputEventReaction = fmap putStrLn inputEvent
        reactimate inputEventReaction

    inputFire "This will NOT be printed to the console!"
    actuate eventNetwork
    inputFire "This WILL be printed to the console!"
