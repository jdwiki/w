Las mónadas libres dividen los cálculos monádicos en estructuras de datos e intérpretes #

Por ejemplo, un cálculo que involucre comandos para leer y escribir desde el indicador:

Primero describimos los “comandos” de nuestro cálculo como un tipo de datos Functor

{-# LANGUAGE DeriveFunctor #-}

data TeletypeF next
    = PrintLine String next
    | ReadLine (String -> next)
    deriving Functor

Luego usamos Free para crear la “Monada libre sobre TeletypeF” y construir algunas operaciones básicas.

import Control.Monad.Free (Free, liftF, iterM)

type Teletype = Free TeletypeF

printLine :: String -> Teletype ()
printLine str = liftF (PrintLine str ())

readLine :: Teletype String
readLine = liftF (ReadLine id)

Dado que ‘Free f’ es una ‘Monad’ siempre que ‘f’ sea un ‘Functor’, podemos usar los combinadores estándar de ‘Monad’ (incluida la notación ‘do’) para construir cálculos de ‘Teletipo’.

import Control.Monad -- we can use the standard combinators

echo :: Teletype ()
echo = readLine >>= printLine

mockingbird :: Teletype a
mockingbird = forever echo

Finalmente, escribimos un “intérprete” que convierte los valores de Teletype a en algo con lo que sabemos cómo trabajar como IO a

interpretTeletype :: Teletype a -> IO a
interpretTeletype = foldFree run where
  run :: TeletypeF a -> IO a
  run (PrintLine str x) = putStrLn *> return x
  run (ReadLine f) = fmap f getLine

Que podemos usar para “ejecutar” el cálculo Teletype a en IO

> interpretTeletype mockingbird
hello
hello
goodbye
goodbye
this will go on forever
this will go on forever

La mónada más libre #

Hay una formulación alternativa de la mónada libre llamada mónada más libre (o rápida, u operativa). La mónada Freer no requiere una instancia de Functor para su conjunto de instrucciones subyacente y tiene una estructura similar a una lista más reconocible que la mónada libre estándar.

La mónada de Freer representa los programas como una secuencia de instrucciones atómicas pertenecientes al conjunto de instrucciones i :: * -> *. Cada instrucción usa su parámetro para declarar su tipo de retorno. Por ejemplo, el conjunto de instrucciones base para la mónada State es el siguiente:

data StateI s a where
    Get :: StateI s s  -- the Get instruction returns a value of type 's'
    Put :: s -> StateI s ()  -- the Put instruction contains an 's' as an argument and returns ()

La secuenciación de estas instrucciones se lleva a cabo con el constructor :>>=. :>>= toma una sola instrucción que devuelve una a y la antepone al resto del programa, canalizando su valor devuelto a la continuación. En otras palabras, dada una instrucción que devuelve una a y una función para convertir una a en un programa que devuelve una b, :>>= producirá un programa que devuelve una b.

data Freer i a where
    Return :: a -> Freer i a
    (:>>=) :: i a -> (a -> Freer i b) -> Freer i b

Tenga en cuenta que a se cuantifica existencialmente en el constructor :>>=. La única forma de que un intérprete aprenda qué es a es mediante la coincidencia de patrones en el GADT i.

Aparte: El lema co-Yoneda nos dice que ‘Freer’ es isomorfo a ‘Free’. Recordemos la definición del funtor CoYoneda:

datos CoYoneda i b donde CoYoneda :: i a -> (a -> b) -> CoYoneda i b

Freer i es equivalente a Free (CoYoneda i). Si toma [Free’s constructors](https://www.wikiod.com/es/haskell/monadas-libres#Las mónadas libres son como puntos fijos) y establece f ~ CoYoneda i, obtienes:

Puro :: a -> Libre (CoYoneda i) a Gratis :: CoYoneda i (Gratis (CoYoneda i) b) -> Gratis (CoYonda i) b ~ i a -> (a -> Gratis (CoYoneda i) b) -> Gratis (CoYoneda i) b

desde donde podemos recuperar los constructores de Freer i simplemente configurando Más libre i ~ Libre (CoYoneda i).

Debido a que ‘CoYoneda i’ es un ‘Functor’ para cualquier ‘i’, ‘Freer’ es una ‘Mónada’ para cualquier ‘i’, incluso si ‘i’ no es un ‘Functor’.

instance Monad (Freer i) where
    return = Return
    Return x >>= f = f x
    (i :>>= g) >>= f = i :>>= fmap (>>= f) g  -- using `(->) r`'s instance of Functor, so fmap = (.)

Se pueden construir intérpretes para Freer mediante la asignación de instrucciones a alguna mónada controladora.

foldFreer :: Monad m => (forall x. i x -> m x) -> Freer i a -> m a
foldFreer eta (Return x) = return x
foldFreer eta (i :>>= f) = eta i >>= (foldFreer eta . f)

Por ejemplo, podemos interpretar la mónada Freer (StateI s) usando la mónada State s normal como controlador:

runFreerState :: Freer (StateI s) a -> s -> (a, s)
runFreerState = State.runState . foldFreer toState
    where toState :: StateI s a -> State s a
          toState Get = State.get
          toState (Put x) = State.put x

Las mónadas libres son como puntos fijos #

Compare la definición de Gratis con la de [Fix](https://www.wikiod.com/es/haskell/esquemas-de-recursividad#puntos fijos):

data Free f a = Return a
              | Free (f (Free f a))

newtype Fix f = Fix { unFix :: f (Fix f) }

En particular, compare el tipo del constructor Free con el tipo del constructor Fix. Free crea capas en un funtor como Fix, excepto que Free tiene un caso adicional de Return a.

¿Cómo funcionan foldFree e iterM? #

Hay algunas funciones para ayudar a derribar los cálculos libres interpretándolos en otra mónada m: iterM :: (Functor f, Monad m) => (f (m a) -> m a) -> (Free f a - > m a) y foldFree :: Monad m => (forall x. f x -> m x) -> (Free f a -> m a). ¿Qué están haciendo?

Primero, veamos qué se necesitaría para derribar e interpretar una función Teletype a en IO manualmente. Podemos ver Free f a como se define

data Free f a 
    = Pure a 
    | Free (f (Free f a))

El caso Pure es fácil:

interpretTeletype :: Teletype a -> IO a
interpretTeletype (Pure x) = return x
interpretTeletype (Free teletypeF) = _

Ahora, ¿cómo interpretar un cómputo Teletype que fue construido con el constructor Free? Nos gustaría llegar a un valor de tipo IO a examinando teletypeF :: TeletypeF (Teletype a). Para empezar, escribiremos una función runIO :: TeletypeF a -> IO a que asigna una sola capa de la mónada libre a una acción IO:

runIO :: TeletypeF a -> IO a
runIO (PrintLine msg x) = putStrLn msg *> return x
runIO (ReadLine k) = fmap k getLine

Ahora podemos usar runIO para completar el resto de interpretTeletype. Recuerda que teletypeF :: TeletypeF (Teletype a) es una capa del funtor TeletypeF que contiene el resto del cálculo Free. Usaremos runIO para interpretar la capa más externa (así que tenemos runIO teletypeF :: IO (Teletype a)) y luego usaremos el combinador >>= de la mónada IO para interpretar el Teletype a devuelto .

interpretTeletype :: Teletype a -> IO a
interpretTeletype (Pure x) = return x
interpretTeletype (Free teletypeF) = runIO teletypeF >>= interpretTeletype

La definición de foldFree es solo la de interpretTeletype, excepto que la función runIO ha sido eliminada. Como resultado, foldFree funciona independientemente de cualquier funtor base en particular y de cualquier mónada de destino.

foldFree :: Monad m => (forall x. f x -> m x) -> Free f a -> m a
foldFree eta (Pure x) = return x
foldFree eta (Free fa) = eta fa >>= foldFree eta

foldFree tiene un tipo de rango 2: eta es una transformación natural. Podríamos haberle dado a foldFree un tipo de Monad m => (f (Free f a) -> m (Free f a)) -> Free f a -> m a, pero eso le da a eta la libertad de inspeccionar el Cálculolibredentro de la capaf. Darle a foldFreeeste tipo más restrictivo asegura queeta` solo pueda procesar una sola capa a la vez.

iterM le da a la función de plegado la capacidad de examinar el subcálculo. El resultado (monádico) de la iteración anterior está disponible para la siguiente, dentro del parámetro f. iterM es análogo a un [paramorphism](https://www.wikiod.com/es/haskell/esquemas-de-recursividad#recursividad primitiva) mientras que foldFree es como un [catamorphism]( https://www.wikiod.com/es/haskell/esquemas-de-recursividad#Doblando una estructura una capa a la vez).

iterM :: (Monad m, Functor f) => (f (m a) -> m a) -> Free f a -> m a
iterM phi (Pure x) = return x
iterM phi (Free fa) = phi (fmap (iterM phi) fa)
