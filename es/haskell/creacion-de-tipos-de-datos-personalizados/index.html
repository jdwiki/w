Creando un tipo de datos con parámetros constructores de valor #

Los constructores de valores son funciones que devuelven un valor de un tipo de datos. Por eso, como cualquier otra función, pueden tomar uno o más parámetros:

data Foo = Bar String Int | Biz String

Verifiquemos el tipo del constructor de valor Bar.

:t Bar

huellas dactilares

Bar :: String -> Int -> Foo

lo que prueba que ‘Bar’ es de hecho una función.

Creando variables de nuestro tipo personalizado #

let x = Bar "Hello" 10
let y = Biz "Goodbye"

Crear un tipo de datos con parámetros de tipo #

Los constructores de tipos pueden tomar uno o más parámetros de tipo:

data Foo a b = Bar a b | Biz a b

Los parámetros de tipo en Haskell deben comenzar con una letra minúscula. Nuestro tipo de datos personalizado aún no es un tipo real. Para crear valores de nuestro tipo, debemos sustituir todos los parámetros de tipo con tipos reales. Debido a que a y b pueden ser de cualquier tipo, nuestros constructores de valores son funciones polimórficas.

Creando variables de nuestro tipo personalizado #

let x = Bar "Hello" 10      -- x :: Foo [Char] Integer
let y = Biz "Goodbye" 6.0   -- y :: Fractional b => Foo [Char] b
let z = Biz True False      -- z :: Foo Bool Bool

Creando un tipo de datos simple #

La forma más fácil de crear un tipo de datos personalizado en Haskell es usar la palabra clave data:

data Foo = Bar | Biz

El nombre del tipo se especifica entre data y =, y se denomina constructor de tipos. Después de = especificamos todos los constructores de valor de nuestro tipo de datos, delimitados por el signo |. Hay una regla en Haskell que establece que todos los constructores de tipo y valor deben comenzar con una letra mayúscula. La declaración anterior se puede leer de la siguiente manera:

Defina un tipo llamado Foo, que tiene dos valores posibles: Bar y Biz.

Creando variables de nuestro tipo personalizado #

let x = Bar

La declaración anterior crea una variable llamada x de tipo Foo. Verifiquemos esto comprobando su tipo.

:t x

huellas dactilares

x :: Foo

Tipo de datos personalizado con parámetros de registro #

Supongamos que queremos crear un tipo de datos Persona, que tiene un nombre y apellido, una edad, un número de teléfono, una calle, un código postal y un pueblo.

Podríamos escribir

data Person = Person String String Int Int String String String

Si ahora queremos obtener el número de teléfono, necesitamos hacer una función

getPhone :: Person -> Int
getPhone (Person _ _ _ phone _ _ _) = phone

Bueno, esto no es divertido. Podemos hacerlo mejor usando parámetros:

data Person' = Person' { firstName     :: String
                       , lastName      :: String
                       , age           :: Int
                       , phone         :: Int
                       , street        :: String
                       , code          :: String
                       , town          :: String }

Ahora obtenemos la función teléfono donde

:t phone
phone :: Person' -> Int

Ahora podemos hacer lo que queramos, por ejemplo:

printPhone :: Person' -> IO ()
printPhone = putStrLn . show . phone

También podemos vincular el número de teléfono mediante [Coincidencia de patrones][1]:

getPhone' :: Person' -> Int
getPhone' (Person {phone = p}) = p

Para facilitar el uso de los parámetros, consulte [RecordWildCards][2]

[1]: https://www.wikiod.com/es/haskell/sintaxis-en-funciones#La coincidencia de patrones [2]: https://www.wikiod.com/es/haskell/extensiones-de-idioma-comunes-de-ghc#RecordWildCards
