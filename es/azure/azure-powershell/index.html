Iniciar sesión en Azure #

Modo clásico (Gestión de servicios):

Add-AzureAccount

Esto lo autenticará usando Azure Active Directory y PowerShell obtiene un token de acceso que caduca después de aproximadamente 12 horas. Por lo tanto, debe repetir la autenticación después de 12 horas.

Una alternativa es ejecutar el siguiente cmdlet:

Get-AzurePublishSettingsFile

Esto abre una ventana del navegador donde puede descargar un archivo de configuración de publicación. Este archivo contiene un certificado que permite la autenticación de PowerShell. Luego puede importar su archivo de configuración de publicación usando lo siguiente:

Import-AzurePublishSettingsFile

Recuerde que el archivo de configuración de publicación contiene un certificado con privilegios de administrador efectivos en su suscripción. Manténgalo seguro o elimínelo después de usarlo.

Administrador de recursos

En el lado de Resource Manager, solo podemos usar la autenticación de Azure Active Directory con los tokens de acceso de 12 horas. Hay dos comandos alternativos actualmente que puede usar:

Login-AzureRmAccount
Add-AzureRmAccount

Selección de suscripción #

Cuando tiene varias suscripciones en su cuenta de Azure; es importante que esté seleccionando el que desea operar (y use esto como predeterminado); para evitar que ocurran accidentes con los recursos en la suscripción incorrecta.

Modo clásico

Set-AzureSubscription
Select-AzureSubscription

Administrador de recursos

Select-AzureRmSubscription

Información de suscripción

Los comandos anteriores le piden que especifique información (por ejemplo, el ID de la suscripción) para identificar la suscripción a la que desea cambiar. Para enumerar esta información para las suscripciones a las que tiene acceso, ejecute este comando:

Get-AzureSubscription

Modo clásico vs modo ARM #

Cuando trabaje con Azure usando PowerShell, hay 2 formas diferentes que debe tener en cuenta y verá una gran cantidad de documentación de Microsoft que se refiere a ambas:

“Modo clásico (Gestión de servicios)”

Esta es la forma antigua de operar Azure y administrar Azure. Todavía hay algunos servicios en Azure que solo se pueden administrar con el modo clásico, aunque cada vez más servicios se están moviendo hacia el nuevo modo ARM.

Para enumerar los módulos instalados en sus máquinas que funcionan en modo clásico, puede hacer lo siguiente:

Get-Module -ListAvailable Azure.*

“Administrador de recursos (ARM) “

Esta es la nueva forma de administrar Azure (basada en las API REST proporcionadas). La mayoría de los servicios que podía administrar en Azure desde el modo clásico de Powershell ahora se pueden administrar con el nuevo modo, con algunas excepciones. Esta debería ser la forma preferida de administrar sus recursos de Azure a menos que tenga ciertos servicios que aún no son compatibles con el modo ARM.

Para enumerar los módulos instalados en su máquina que contienen comandos para operar en modo ARM, puede hacer lo siguiente:

Get-Module -ListAvailable AzureRM*

Obtenga la versión actual de Azure PowerShell #

Para determinar la versión de Azure PowerShell que ha instalado, ejecute lo siguiente:

Get-Module -ListAvailable -Name Azure -Refresh

Este comando devuelve la versión instalada incluso cuando no ha cargado el módulo de Azure PowerShell en su sesión actual de PowerShell.

Manipular activos de Azure #

Los cmdlets de Azure le permiten realizar algunas de las mismas acciones en los activos de Azure a través de PowerShell que haría con el código C# o Azure Portal.

Por ejemplo, estos pasos le permiten descargar el contenido de un blob de Azure en un directorio local:

New-Item -Path .\myblob -ItemType Directory
$context = New-AzureStorageContext -StorageAccountName MyAccountName -StorageAccountKey {key from the Azure portal}
$blob = Get-AzureStorageBlob -Container MyContainerName -Context $context
$blob | Get-AzureStorageBlobContent -Destination .\myblob\

Gestión de administradores de tráfico #

Con Azure PowerShell, puede obtener ciertas funciones que actualmente no están disponibles en [Azure Portal] (https://portal.azure.com), como:

  • Reconfigure todos los puntos finales de Traffic Manager a la vez
  • Dirija otros servicios a través de Azure ResourceId en lugar del nombre de dominio, por lo que no necesita configurar la ubicación manualmente para los puntos finales de Azure

Requisitos previos #

Para comenzar, debe [iniciar sesión](https://www.wikiod.com/es/azure/azure-powershell#Iniciar sesión en Azure) y [seleccionar suscripción RM](http://stackoverflow.com/ documentación/azure/3961/azure-powershell/13825/selecting-subscription).

Obtenga el perfil de TrafficManager #

Las operaciones con Traffic Managers a través de PowerShell se realizan en tres pasos:

  1. Obtenga el perfil de TM:
    $profile = Get-AzureRmTrafficManagerProfile -ResourceGroupName my-resource-group -Name my-traffic-manager
    O cree uno nuevo como [en este artículo](https://docs. microsoft.com/en-us/powershell/resourcemanager/azurerm.trafficmanager/v2.2.0/new-azurermtrafficmanagerprofile).
  2. Explore y modifique el perfil de TM
    Verifique los campos $profile y $profile.Endpoints para ver la configuración de cada punto final.
  3. Guarde los cambios a través de Set-AzureRmTrafficManagerProfile -TrafficManagerProfile $profile.

Cambiar puntos finales #

Todos los puntos finales actuales se almacenan en la lista $profile.Endpoints, por lo que puede modificarlos directamente por index
$profile.Endpoints[0].Weight = 100
o por nombre
$profile .puntos finales | ?{ $_.Nombre -eq 'mi-punto final' } | %{ $_.Peso = 100 }

Para borrar todos los puntos finales use
$perfil.Endpoints.Clear()

Para eliminar el uso de un punto final en particular
Remove-AzureRmTrafficManagerEndpointConfig -TrafficManagerProfile $perfil -EndpointName 'mi-punto final'

Para agregar un nuevo uso de punto final
Add-AzureRmTrafficManagerEndpointConfig -TrafficManagerProfile $profile -EndpointName "my-endpoint" -Type AzureEndpoints -TargetResourceId "/subscriptions/00000000-0000-0000-0000-000000000000/resourceGroups/my-resource-group/providers/Microsoft/domainNames/Compute my-azure-service" -EndpointStatus habilitado -Peso 100

Como puede ver, en el último caso, abordamos nuestro servicio Azure a través de ResourceId en lugar de un nombre de dominio.

Tenga en cuenta #

Sus cambios en TM y sus extremos no se aplican hasta que invoque Set-AzureRmTrafficManagerProfile -TrafficManagerProfile $profile. Eso le permite reconfigurar completamente TM en una sola operación.

Traffic Manager es una implementación de DNS y la dirección IP proporcionada a los clientes tiene algún tiempo de vida (también conocido como TTL, puede ver su duración en segundos en el $ perfil.Ttl campo). Por lo tanto, después de que haya reconfigurado TM, algunos clientes continuarán usando los puntos finales antiguos que almacenaron en caché hasta que caduque el TTL.
