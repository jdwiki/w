Rebaja con sabor a GitHub #

GitHub amplía la sintaxis de Markdown para proporcionar nuevas funciones útiles.

Encabezado #

# Header1
## Header2
### Header3
#### Header4
##### Header5
###### Header6
H1
===
H2
---

Énfasis #

*Italic1* _Italic2_
**Bold1** __Bold2__
***Bold_Italic***
~~Strikethrough~~

Linea horizontal #

---
***
___

Lista #

unordered list:

* item-1
  * sub-item-1
  * sub-item-2
- item-2
  - sub-item-3
  - sub-item-4
+ item-3
  + sub-item-5
  + sub-item-6


ordered list:

1. item-1
  1. sub-item-1
  2. sub-item-2
2. item-2
  1. sub-item-3
  2. sub-item-4
3. item-3

Mesa #

Table Header-1 | Table Header-2 | Table Header-3
:--- | :---: | ---:
Table Data-1 | Table Data-2 | Table Data-3
TD-4 | Td-5 | TD-6
Table Data-7 | Table Data-8 | Table Data-9

Instalación o configuración #

GitHub es una gran colección de repositorios de Git. En otras palabras, ¡puedes pensar en GitHub como una colección de muchos proyectos!

Creando una cuenta #

  • Visita la página principal de GitHub Aquí
  • Elija un nombre de usuario, ingrese su dirección de correo electrónico y elija una contraseña segura, ¡y estará listo para comenzar!

Herramientas útiles #

Para los principiantes de Git/GitHub, comprender cómo funciona el control de versiones puede resultar confuso al principio. Existe una versión GUI de GitHub que puede descargar y usar. GitHub Desktop es solo esa herramienta.

Creando su primer repositorio #

Puedes pensar en un repositorio como un proyecto. Puede crear un repositorio en línea o fuera de línea. Siga los pasos a continuación:

En línea #

  1. Primero inicie sesión y vaya a su perfil.
  2. Navegue a la pestaña “Repositorios” cerca de la parte superior de la página
  3. ¡Presiona el botón verde “Nuevo” y estás listo para pelear!

Desconectado #

  1. Descargue e instale git (elija el sistema operativo que está ejecutando)
  2. Después de la descarga y la instalación, puede usar la herramienta de línea de comandos o puede descargar un cliente GUI.
  3. Después de la instalación, cree una cuenta en github
  4. En la esquina superior derecha, haga clic en + y elija crear un nuevo repositorio o importar uno existente.
  5. Si elige uno nuevo, ingrese el nombre del repositorio y elija si es público o privado.
  6. Haga clic en: Crear repositorio

nótese bien Los repositorios privados no están disponibles para usuarios gratuitos.

Archivo LÉAME #

Si su proyecto no tiene README.md, GitHub puede analizar README.rdoc para mostrar los detalles. Si tiene ambos, usará README.md, ignorando silenciosamente rdoc.

Un archivo README puede incluir- #


Título del Proyecto #

Describe brevemente tu proyecto. También puede proporcionar el enlace al sitio web del proyecto, las insignias, la comunidad y la información de contacto (es decir, correo electrónico, sitio social).

Descargar #

Enlace del archivo ejecutable (archivo ejecutable o minificado o de instalación). También puede haber enlaces a versiones anteriores.

Instalación #

Cómo se puede utilizar su trabajo. Puede incluir los requisitos previos, la configuración, las bibliotecas de terceros, el uso, las precauciones, etc.

Demostración #

Puede incluir una muestra de código, un archivo gif, un enlace de video o incluso capturas de pantalla.

Autores #

Nombres de los autores, información de contacto, etc.

Agradecimientos #

Lista de personas o comunidad a las que se ayudó e inspiró a lo largo del proyecto

Contribuyendo #

Instrucciones para contribuir (es decir, agregar funciones, informar errores, enviar parches) al proyecto. También puede incluir un enlace a la documentación.

Licencia #

Dé una breve introducción sobre su licencia. También puede proporcionar un enlace al sitio de la licencia.

archivo de LICENCIA #

GitHub lo ayuda a agregar rápidamente una licencia a su repositorio, como una alternativa para agregar su propio archivo de texto/rebaja.

  1. En su repositorio, haga clic en ‘Crear nuevo archivo’

  1. En la página siguiente:

    1. Type LICENSE.md or LICENSE.txt as the new file’s file name.
    2. The Want to use a new template? dialog will appear.

  2. Elija su licencia preferida.
