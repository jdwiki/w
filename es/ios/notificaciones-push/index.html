Parámetro Descripción

Registro del dispositivo para notificaciones automáticas #

Para registrar su dispositivo para notificaciones automáticas, agregue el siguiente código a su archivo AppDelegate en el método didFinishLaunchingWithOptions:

Rápido #

func application(application: UIApplication, didFinishLaunchingWithOptions launchOptions: [NSObject: AnyObject]?) -> Bool {
    // Override point for customization after application launch.
    if UIDevice.currentDevice().systemVersion.compare(v, options: .NumericSearch) == NSOrderedAscending { 
        // Register for Push Notitications, if running iOS < 8
        if application.respondsToSelector("registerUserNotificationSettings:") {
            let types:UIUserNotificationType = (.Alert | .Badge | .Sound)
            let settings:UIUserNotificationSettings = UIUserNotificationSettings(forTypes: types, categories: nil)

            application.registerUserNotificationSettings(settings)
            application.registerForRemoteNotifications()
        } else {      
            // Register for Push Notifications before iOS 8
            application.registerForRemoteNotificationTypes(.Alert | .Badge | .Sound)
        }
    } else {
        var center = UNUserNotificationCenter.currentNotificationCenter()
        center.delegate = self
        center.requestAuthorizationWithOptions((UNAuthorizationOptionSound | UNAuthorizationOptionAlert | UNAuthorizationOptionBadge)) {(granted: Bool, error: NSError) -> Void in
            if !error {
                UIApplication.sharedApplication().registerForRemoteNotifications()
                // required to get the app to do anything at all about push notifications
                print("Push registration success.")
            } else {
                print("Push registration FAILED")
                print("ERROR: \(error.localizedFailureReason!) - \(error.localizedDescription)")
                print("SUGGESTIONS: \(error.localizedRecoveryOptions) - \(error.localizedRecoverySuggestion!)")
            })
    }

    return true
}

Registro de la ID de la aplicación para su uso con notificaciones automáticas #

Cosas que necesitas #

  • Una Membresía pagada del Programa de Desarrolladores de Apple
  • Una ID de aplicación válida y un identificador para su aplicación (como com.example.MyApp) que no se usa antes en ninguna parte
  • Acceso a developer.apple.com y al Centro de miembros
  • Un dispositivo iOS para probar (ya que las notificaciones automáticas no funcionan en el simulador)

Habilitar el acceso a APN para la ID de la aplicación en el Centro de desarrolladores de Apple #

1- Inicie sesión en el Centro de miembros de developer.apple.com (el enlace Cuenta en la página de inicio)

2- Ir a “Certificados”

3- Seleccione “ID de aplicación” en el panel izquierdo

4- Haga clic en “+” en la parte superior derecha

5- Agregar ID de aplicación con la opción de notificaciones automáticas marcada

6- Haga clic en la ID de aplicación creada y seleccione Editar

7- Haga clic en Configurar en el panel de notificaciones push

8- Abra la aplicación Acceso a Llaveros en su Mac

9- En el menú Acceso a llaveros, haga clic en Asistente de certificados -> Solicitar un certificado de una autoridad de certificación

10- Introduce tu correo en el primer campo de texto

11- Introduce tu nombre en el segundo campo de texto

12- Deje la dirección de correo electrónico de CA vacía

13- Seleccione Guardado en disco en lugar de Enviar por correo electrónico a la CA

14- Haz clic en Continuar y sube el archivo generado

15- Descarga el archivo generado por Apple y ábrelo mientras el Acceso a Llaveros está abierto

Prueba de notificaciones push #

Siempre es una buena práctica probar cómo funcionan las notificaciones automáticas incluso antes de tener el lado del servidor listo para ellas, solo para asegurarse de que todo esté configurado correctamente en su lado. Es bastante fácil enviarse una notificación automática utilizando el siguiente script PHP.

  1. Guarde el script como un archivo (send_push.php por ejemplo) en la misma carpeta que su certificado (desarrollo o producción)

  2. Edítelo para poner el token de su dispositivo, la contraseña del certificado

  3. Elija la ruta correcta para abrir una conexión, dev_path o prod_path (aquí es donde aparece ‘Abrir una conexión con el servidor APNS’ en el script)

  4. CD a la carpeta en la Terminal y ejecute el comando ‘php send_push’

  5. Recibe la notificación en tu dispositivo

Generación de un certificado .pem a partir de su archivo .cer, para pasarlo al desarrollador del servidor #

  1. Guarde aps.cer en una carpeta

  2. Abra “Acceso a llaveros” y exporte la clave que se encuentra debajo de ese certificado a un archivo .p12 (llámelo key.p12). Para hacer eso, haga clic derecho sobre él y elija Exportar. Guárdelo en la misma carpeta que el paso 1. Al exportar, se le pedirá una contraseña. Inventa algo y memorízalo.

  3. cd a esa carpeta en Terminal y ejecuta los siguientes comandos:

  4. Convierta .cer en un certificado .pem

    openssl x509 -in aps.cer -inform der -out aps.pem

  5. Convierta su clave al formato .pem. Para abrir la clave, ingrese la contraseña con la que la exportó desde el llavero, en el paso 2. Luego, ingrese otra contraseña que protegerá el archivo exportado. Se le pedirá que lo ingrese dos veces para confirmarlo.

    openssl pkcs12 -nocerts -out key.pem -in key.p12

  6. Combinar los archivos en un archivo final

    cat key.pem aps.pem > final_cert.pem

  7. El final_cert.pem es el resultado final. Páselo a los desarrolladores del servidor con la contraseña del paso 5, para que puedan usar el certificado protegido.

Darse de baja de las notificaciones automáticas #

Para cancelar el registro de notificaciones remotas programáticamente, puede usar

C objetivo #

[[UIApplication sharedApplication] unregisterForRemoteNotifications];

Rápido #

UIApplication.sharedApplication().unregisterForRemoteNotifications()

esto es similar a entrar en la configuración de su teléfono y desactivar manualmente las notificaciones de la aplicación.

NOTA: Puede haber casos excepcionales en los que necesite esto (por ejemplo, cuando su aplicación ya no admita notificaciones automáticas)

Si solo desea permitir que el usuario deshabilite temporalmente las notificaciones. Debe implementar un método para eliminar el token del dispositivo en la base de datos de su servidor. de lo contrario, si solo deshabilita Notificación localmente en su dispositivo, su servidor seguirá enviando mensajes.

Registrarse para notificaciones automáticas (no interactivas) #

Se recomienda agregar la lógica de registro para notificaciones push en AppDelegate.swift ya que las funciones de devolución de llamada (éxito, falla) se llamarán sus. Para registrarse solo haga lo siguiente:

let application = UIApplication.sharedApplication()
let settings = UIUserNotificationSettings(forTypes: [.Alert, .Badge, .Sound], categories: nil)
application.registerUserNotificationSettings(settings)

Luego, se llamará a la función de devolución de llamada didRegisterUserNotificationSettings y, en ese caso, simplemente active el registro de esta manera:

func application(application: UIApplication, didRegisterUserNotificationSettings notificationSettings: UIUserNotificationSettings) {
    application.registerForRemoteNotifications()
}

Y en ese caso, se mostrará una alerta del sistema solicitando permiso para recibir notificaciones automáticas. Se llamará a una de las siguientes funciones de devolución de llamada:

func application(application: UIApplication, didRegisterForRemoteNotificationsWithDeviceToken deviceToken: NSData) {
    let tokenChars = UnsafePointer<CChar>(deviceToken.bytes)
    var tokenString = ""
    
    for i in 0..<deviceToken.length {
        tokenString += String(format: "%02.2hhx", arguments: [tokenChars[i]])
    }
    
    print("Push token: \(tokenString)")
}

func application(application: UIApplication, didFailToRegisterForRemoteNotificationsWithError error: NSError) {
    print("didFailToRegisterForRemoteNotificationsWithError: \(error)")
            
}

En casos muy raros, no se llama a funciones de devolución de llamada de éxito o falla. Esto sucede cuando tiene problemas de conexión a Internet o APNS Sandbox no funciona. El sistema realiza una llamada API a APNS para realizar alguna verificación; si no lo hace, no se llamará a ninguna de las dos funciones de devolución de llamada. Visite Estado del sistema de Apple para asegurarse de que esté bien.
