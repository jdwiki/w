Sintaxis #

  1. // Objetivo-C

  2. [UIView new] // Obtenga un objeto de vista asignado e inicializado

  3. [[UIView alloc] initWithFrame:(Pass CGRect)] // Obtenga la vista asignada e inicializada con un marco

  4. [[UIView alloc] init] // Obtener un objeto de vista asignado e inicializado

  5. // Veloz

  6. UIView() // Crea una instancia de UIView con marco CGRect.zero

  7. UIView(frame: CGRect) // Crea una instancia de UIView especificando el marco

  8. UIView.addSubview(UIView) // agrega otra instancia de UIView como subvista

  9. UIView.hidden // Obtener o establecer la visibilidad de la vista

  10. UIView.alpha // Obtener o establecer la opacidad de la vista

  11. UIView.setNeedsLayout() // Obliga a la vista a actualizar su diseño

La clase UIView define un área rectangular en la pantalla y las interfaces para administrar el contenido en esa área. En tiempo de ejecución, un objeto de vista maneja la representación de cualquier contenido en su área y también maneja cualquier interacción con ese contenido.

Hacer la vista redondeada #

Para hacer una UIView redondeada, especifica un cornerRadius para la capa de la vista.

Esto también se aplica a cualquier clase que herede de UIView, como UIImageView.

Programáticamente #

Código SWIFT

someImageView.layoutIfNeeded()
someImageView.clipsToBounds = true
someImageView.layer.cornerRadius = 10

Código Objetivo-C

[someImageView layoutIfNeeded];
someImageView.clipsToBounds = YES;
someImageView.layer.cornerRadius = 10;

Ejemplo

//Swift code
topImageView.layoutIfNeeded()
bottomImageView.layoutIfNeeded()
topImageView.clipsToBounds = true
topImageView.layer.cornerRadius = 10
bottomImageView.clipsToBounds = true
bottomImageView.layer.cornerRadius = bottomImageView.frame.width / 2

//Objective-C code
[topImageView layoutIfNeeded]
[bottomImageView layoutIfNeeded];
topImageView.clipsToBounds = YES;
topImageView.layer.cornerRadius = 10;
bottomImageView.clipsToBounds = YES;
bottomImageView.cornerRadius = CGRectGetWidth(bottomImageView.frame) / 2;

Aquí está el resultado, que muestra el efecto de vista redondeada usando el radio de esquina especificado:

![Ejemplo 1]

Nota

Para hacer esto, debe incluir el marco QuartzCore.

 #import <QuartzCore/QuartzCore.h>

Configuración del guión gráfico #

También se puede lograr un efecto de vista redondeada “sin programación” configurando las propiedades correspondientes en Storyboard.

Dado que las propiedades de capa no están expuestas en Storyboard, debe modificar el atributo cornerRadius a través de la sección Atributos de tiempo de ejecución definidos por el usuario.

Extensión rápida #

Puede usar esta práctica extensión para aplicar una vista redondeada siempre que tenga el mismo ancho y alto.

extension UIView {
    @discardableResult
    public func setAsCircle() -> Self {
        self.clipsToBounds = true
        let frameSize = self.frame.size
        self.layer.cornerRadius = min(frameSize.width, frameSize.height) / 2.0
        return self
    }
}

Para usarlo:

yourView.setAsCircle()

Uso de IBInspectable e IBDesignable #

Una (o dos) de las características nuevas más geniales en las versiones recientes de Xcode son las propiedades IBInspectable y IBDesignable UIViews. Estos no tienen nada que ver con la funcionalidad de su aplicación, sino que afectan la experiencia del desarrollador en Xcode. El objetivo es poder inspeccionar visualmente las vistas personalizadas en su aplicación iOS sin ejecutarla. Así que suponga que tiene una vista personalizada creativamente llamada CustomView que hereda de UIView. En esta vista personalizada, mostrará una cadena de texto con un color designado. También puede optar por no mostrar ningún texto. Necesitaremos tres propiedades:

var textColor: UIColor = UIColor.blackColor()
var text: String?
var showText: Bool = true

Entonces podemos anular la función drawRect en la clase:

if showText {
    if let text = text {
        let s = NSString(string: text)
        s.drawInRect(rect,
            withAttributes: [
                NSForegroundColorAttributeName: textColor,
                NSFontAttributeName: UIFont(name: "Helvetica Neue", size: 18)!
            ])
    }
}

Asumiendo que la propiedad texto está configurada, esto dibujará una cadena en la esquina superior izquierda de la vista cuando se ejecute la aplicación. El problema es que no sabremos cómo se ve sin ejecutar la aplicación. Aquí es donde entran IBInspectable e IBDesignable. IBInspectable nos permite establecer visualmente los valores de propiedad de la vista en Xcode, al igual que con los controles integrados. IBDesignable nos mostrará una vista previa visual en el guión gráfico. Así es como debería verse la clase:

@IBDesignable
class CustomView: UIView {
    @IBInspectable var textColor: UIColor = UIColor.blackColor()
    @IBInspectable var text: String?
    @IBInspectable var showText: Bool = true

    override func drawRect(rect: CGRect) {
        // ...
    }
}

O en el Objetivo C:

IB_DESIGNABLE
@interface CustomView: UIView

@property (nonatomic, strong) IBInspectable UIColor* textColor;
@property (nonatomic, strong) IBInspectable NSString* text;
@property (nonatomic, assign) IBInspectable BOOL showText;

@end

@implementation CustomView

- (instancetype)init {
    if(self = [super init]) {
        self.textColor = [UIColor blackColor];
        self.showText = YES;
    }
    return self;
}

- (void)drawRect:(CGRect)rect {
    //...
}

@end

Las siguientes capturas de pantalla muestran lo que sucede en Xcode. El primero es lo que sucede después de agregar la clase revisada. Observe que hay tres nuevos elementos de interfaz de usuario para las tres propiedades. El Color de texto mostrará un selector de color, Texto es solo un cuadro de entrada y Mostrar texto nos dará las opciones para ‘Desactivado’ y ‘Activado’, que son ‘falso’ y ‘verdadero’ respectivamente.

El siguiente es después de cambiar el Color de texto a rojo usando el selector de color. Además, se ha proporcionado algo de texto para que la función drawRect lo muestre. Tenga en cuenta que la vista en Interface Builder también se ha actualizado.

Finalmente, establecer Mostrar texto en Desactivado en el inspector de propiedades hace que desaparezca la visualización de texto en Interface Builder.

Sin embargo, todos llegamos a una situación en la que necesitamos crear una UIView redondeada en múltiples vistas en su Storyboard. En lugar de declarar IBDesignable a cada vista de Storyboard, es mejor crear una Extensión de UIView y obtenga una interfaz de usuario creada solo para cada UIView a lo largo del proyecto para crear una vista redondeada al establecer el radio de la esquina. Un radio de borde configurable en cualquier UIView que cree en el guión gráfico.

extension UIView {
    
    @IBInspectable var cornerRadius:CGFloat {
        set {
            layer.cornerRadius = newValue
            clipsToBounds = newValue > 0
        }
        get {
            return layer.cornerRadius
        }
    }
    
}

Utilización del tamaño de contenido intrínseco #

Al crear una subclase de UIView, el tamaño del contenido intrínseco ayuda a evitar establecer restricciones de alto y ancho codificadas

un vistazo básico a cómo una clase puede utilizar esto

class ImageView: UIView {
    var image: UIImage {
        didSet {
            invalidateIntrinsicContentSize()
        }
    }
    // omitting initializers
    // convenience init(image: UIImage)

    override func intrinsicContentSize() -> CGSize {
        return CGSize(width: image.size.width, height: image.size.height)
    }
}

Si solo desea proporcionar un tamaño intrínsecamente, puede proporcionar el valor UIViewNoIntrinsicMetric para el valor que desea ignorar.

 override func intrinsicContentSize() -> CGSize {
     return CGSize(width: UIViewNoIntrinsicMetric, height: image.size.width)   
 }

Administre mediante programación la inserción y eliminación de UIView en y desde otra UIView #

Supongamos que tiene un parentView en el que desea insertar un nuevo subView mediante programación (por ejemplo, cuando desea insertar un UIImageView en la vista de un UIViewController), entonces puede hacerlo de la siguiente manera.

C objetivo

[parentView addSubview:subView];

Rápido

parentView.addSubview(subView)

También puede agregar la vista secundaria debajo de otra subView2, que ya es una vista secundaria de parentView usando el siguiente código:

C objetivo

[parentView insertSubview:subView belowSubview:subView2];

Rápido

parentView.insertSubview(subView, belowSubview: subView2)

Si desea insertarlo arriba de subView2, puede hacerlo de esta manera:

C objetivo

[parentView insertSubview:subView aboveSubview:subView2];

Rápido

parentView.insertSubview(subView, aboveSubview: subView2)

Si en algún lugar de su código necesita traer al frente una determinada subvista, por lo que, por encima de todas las demás subvistas de parentView, puede hacerlo así:

C objetivo

[parentView bringSubviewToFront:subView];

Rápido

parentView.bringSubviewToFront(subView)

Finalmente, si desea eliminar subView de parentView, puede hacer lo siguiente:

C objetivo

[subView removeFromSuperview];

Rápido

subView.removeFromSuperview()

let view = UIView(frame: CGRect(x: 0, y: 0, width: 100, height: 100))
view.backgroundColor = UIColor.orange
self.view.addSubview(view)

    view.frame.origin.y = self.view.frame.origin.y

}

Extensión UIView para atributos de tamaño y marco #

Si queremos obtener la coordenada x del origen de la vista, debemos escribir como:

view.frame.origin.x

Para el ancho, necesitamos escribir:

view.frame.size.width

Pero si agregamos una extensión simple a UIView, podemos obtener todos los atributos de manera muy simple, como:

view.x
view.y
view.width
view.height

También ayudará a establecer estos atributos como:

view.x = 10
view.y = 10
view.width = 100
view.height = 200

Y la extensión simple sería:

extension UIView {
    
    var x: CGFloat {
        get {
            return self.frame.origin.x
        }
        set {
            self.frame = CGRect(x: newValue, y: self.frame.origin.y, width: self.frame.size.width, height: self.frame.size.height)
        }
    }
    
    var y: CGFloat {
        get {
            return self.frame.origin.y
        }
        set {
            self.frame = CGRect(x: self.frame.origin.x, y: newValue, width: self.frame.size.width, height: self.frame.size.height)
        }
    }
    
    var width: CGFloat {
        get {
            return self.frame.size.width
        }
        set {
            self.frame = CGRect(x: self.frame.origin.x, y: self.frame.origin.y, width: newValue, height: self.frame.size.height)
        }
    }
    
    var height: CGFloat {
        get {
            return self.frame.height
        }
        set {
            self.frame = CGRect(x: self.frame.origin.x, y: self.frame.origin.y, width: self.frame.size.width, height: newValue)
        }
    }
}

¡Necesitamos agregar este archivo de clase en un proyecto y estará disponible para usar durante todo el proyecto!

subView.translatesAutoresizingMaskIntoConstraints = NO;
    
NSLayoutConstraint *trailing =[NSLayoutConstraint
                                   constraintWithItem:subView
                                   attribute:NSLayoutAttributeTrailing
                                   relatedBy:NSLayoutRelationEqual
                                   toItem:parent
                                   attribute:NSLayoutAttributeTrailing
                                   multiplier:1.0
                                   constant:10.f];


NSLayoutConstraint *top = [NSLayoutConstraint
                               constraintWithItem:subView
                               attribute:NSLayoutAttributeTop
                               relatedBy:NSLayoutRelationEqual
                               toItem:parent
                               attribute:NSLayoutAttributeTop
                               multiplier:1.0
                               constant:10.f];
    
NSLayoutConstraint *leading = [NSLayoutConstraint
                                   constraintWithItem:subView
                                   attribute:NSLayoutAttributeLeading
                                   relatedBy:NSLayoutRelationEqual
                                   toItem:parent
                                   attribute:NSLayoutAttributeLeading
                                   multiplier:1.0
                                   constant:10.f];
[parent addConstraint:trailing];
[parent addConstraint:top];
[parent addConstraint:leading];

}
