Agregar un esquema de URL a su propia aplicación #

Supongamos que está trabajando en una aplicación llamada MyTasks y desea permitir que las URL entrantes creen una nueva tarea con un título y un cuerpo. La URL que está diseñando podría verse así:

mistareas://crear?title=hola&cuerpo=mundo

(¡Por supuesto, los parámetros texto y cuerpo se utilizan para completar nuestra tarea que estamos creando!)

Estos son los grandes pasos para agregar este esquema de URL a su proyecto:

  1. Registre un esquema de URL en el archivo Info.plist de su aplicación, para que el sistema sepa cuándo enrutar una URL a su aplicación.
  2. Agregue una función a su UIApplicationDelegate que acepte y maneje las URL entrantes.
  3. Realice cualquier tarea que deba ocurrir cuando se abra esa URL.

Paso uno: registre un esquema de URL en Info.plist: #

Primero, debemos agregar una entrada de “Tipos de URL” a nuestro archivo Info.plist. Haga clic en el botón (+) aquí:

… luego ingrese un identificador único para su aplicación, así como el esquema de URL que desea usar. ¡Se específico! No desea que el esquema de URL entre en conflicto con la implementación de otra aplicación. ¡Es mejor ser demasiado largo aquí que demasiado corto!

Paso dos: maneje la URL en UIApplicationDelegate #

Necesitamos implementar application:openURL:options: en nuestro UIApplicationDelegate. ¡Inspeccionaremos la ‘URL’ entrante y veremos si hay alguna acción que podamos tomar!

Una implementación sería esta:

func application(app: UIApplication, openURL url: NSURL, options: [String : AnyObject]) -> Bool {
    if url.scheme == "mytasks" && url.host == "create" {
        let title = // get the title out of the URL's query using a method of your choice
        let body = // get the title out of the URL's query using a method of your choice
        self.rootViewController.createTaskWithTitle(title, body: body)
        return true
    }

    return false
}

Paso tres: Realiza una tarea dependiendo de la URL. #

Cuando un usuario abre su aplicación a través de una URL, probablemente esperaba que algo sucediera. Tal vez sea navegar a una parte del contenido, tal vez sea crear un nuevo elemento; en este ejemplo, ¡vamos a crear una nueva tarea en la aplicación!

En el código anterior, podemos ver una llamada a self.rootViewController.createTaskWithTitle(:body:), por lo que suponiendo que su AppDelegate tiene un puntero a su controlador de vista raíz que implementa la función correctamente, ya está todo listo !

Abrir una aplicación basada en su esquema de URL #

Para abrir una aplicación con el esquema de URL definido todolist://:

C objetivo

NSURL *myURL = [NSURL URLWithString:@"todolist://there/is/something/to/do"];
[[UIApplication sharedApplication] openURL:myURL];

Rápido

let stringURL = "todolist://there/is/something/to/do"
if let url = NSURL(string: stringURL) {
    UIApplication.shared().openURL(url)
}

Nota: Es útil verificar si el enlace se puede abrir para mostrar un mensaje apropiado al usuario. Esto se puede hacer usando el método canOpenURL:.

Configuración de enlaces profundos para su aplicación #

Configurar enlaces profundos para su aplicación es fácil. Solo necesita una pequeña URL con la que desea abrir su aplicación.

Siga los pasos para configurar los enlaces profundos para su aplicación.

  1. Vamos a crear un proyecto y llamarlo DeepLinkPOC.

  2. Ahora seleccione el objetivo de su proyecto.

  3. Después de seleccionar el objetivo, seleccione la pestaña ‘info’.

  4. Desplácese hasta la parte inferior hasta que vea una opción de Tipos de URL

  5. Haga clic en la opción ‘+’.

  6. Verá que los esquemas de URL agregan una cadena con la que desea abrir su aplicación. Agreguemos “DeepLinking” en los esquemas de URL.

Entonces, para abrir su aplicación, puede ejecutarla escribiendo “DeepLinking://” en su safari. Su cadena de enlace profundo tiene el siguiente formato.

[scheme]://[host]/[path]  --> DeepLinking://path/Page1

dónde, Esquema: “Enlace profundo” Anfitrión: “camino” ruta: “Página1”

Nota: incluso si no agrega el host y la ruta, se iniciará la aplicación, así que no se preocupe. Pero puede agregar el host y la ruta para redirigir adicionalmente a una página en particular después del lanzamiento de la aplicación.

  1. Ahora agregue el siguiente método a su appdelegate.

Rápido:

 func application(application: UIApplication, openURL url: NSURL, sourceApplication: String?, annotation: AnyObject) -> Bool

C objetivo:

-(BOOL)application:(UIApplication *)application
          openURL:(NSURL *)url
          sourceApplication:(NSString *)sourceApplication
          annotation:(id)annotation

El método anterior se llama cada vez que se inicia su aplicación utilizando una cadena de enlace profundo que configuró para su aplicación.

  1. Ahora es el momento de instalar su aplicación, pero espere antes de saltar directamente al botón de ejecución. Hagamos un pequeño cambio en el método de inicio de la aplicación del esquema.
  • Seleccione y edite su esquema como

  • cambie su tipo de lanzamiento y cierre

  1. Ahora haga clic en el botón Ejecutar (si lo desea, puede agregar un punto de interrupción a sus métodos didFinishLaunchingWithOptions y openURL para observar los valores)

  2. Verá un mensaje “Esperando que se inicie DeepLinkPOC (o el nombre de su aplicación)”.

  3. Abra safari y escriba “DeepLinking://” en la barra de búsqueda. Aparecerá el mensaje “abrir esta página en DeepLinkPOC”, haga clic en abrir para iniciar su aplicación.

Espero que sepas cómo configurar enlaces profundos para tu aplicación :)
