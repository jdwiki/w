Este documento cubre los conceptos básicos de configuración de sistemas, administración de redes y redes TCP/IP. Linux puede soportar múltiples dispositivos de red. Los nombres de los dispositivos están numerados y comienzan en cero y cuentan hacia arriba. Por ejemplo, una computadora con dos NIC tendrá dos dispositivos etiquetados como eth0 y eth1.

Enumere todas las interfaces disponibles en la máquina

$ifconfig -a

ip - mostrar/manipular enrutamiento, dispositivos, enrutamiento de políticas y túneles

Sintaxis: $ ip { enlace | ... | ruta | macsec } (consulte man ip para ver la lista completa de objetos)

Ejemplos

Lista de interfaces de red

$ ip link show

Cambiar el nombre de la interfaz eth0 a wan

$ ip link set dev eth0 name wan

Subir (o bajar) la interfaz eth0

$ ip link set dev eth0 up

Lista de direcciones para interfaces

$ ip addr show

Agregar (o eliminar) ip y máscara (255.255.255.0)

$ ip addr add 1.2.3.4/24 brd + dev eth0

Agregar IP a una interfaz #

Se puede obtener una dirección IP para una interfaz a través de DHCP o asignación estática

DHCP Si está conectado a una red con un servidor DHCP en ejecución, el comando dhclient puede obtener una dirección IP para su interfaz

$ dhclient <interfaz>

o, alternativamente, puede realizar un cambio en el archivo /etc/network/interfaces para que la interfaz se abra en el arranque y obtenga la IP DHCP

auto eth0
iface eth0 inet dhcp

Configuración estática (Cambio permanente) usando el archivo /etc/network/interfaces

Si desea configurar estáticamente la configuración de la interfaz (cambio permanente), puede hacerlo en el archivo /etc/network/interfaces.

Ejemplo:

auto eth0 # Bring up the interface on boot
iface eth0 inet static 
    address 10.10.70.10
    netmask 255.255.0.0
    gateway 10.10.1.1
    dns-nameservers 10.10.1.20
    dns-nameservers 10.10.1.30

Estos cambios persisten incluso después de reiniciar el sistema.

Configuración estática (cambio temporal) usando la utilidad ifconfig

Se podría agregar una dirección IP estática a una interfaz usando la utilidad ifconfig de la siguiente manera

$ ifconfig <interfaz> <dirección IP>/<máscara> up

Ejemplo:

$ ifconfig eth0 10.10.50.100/16 up

Resolución DNS local #

Archivo: /etc/hosts contiene una lista de hosts que deben resolverse localmente (no por DNS)

Ejemplo de contenido del archivo:

127.0.0.1         your-node-name.your-domain.com  localhost.localdomain  localhost 
XXX.XXX.XXX.XXX   node-name

El formato de archivo para el archivo de hosts se especifica en RFC 952

Configurar servidores DNS para la resolución de nombres de dominio #

Archivo: /etc/resolv.conf contiene una lista de servidores DNS para la resolución de nombres de dominio

Ejemplo de contenido del archivo:

nameserver 8.8.8.8 # IP address of the primary name server
nameserver 8.8.4.4 # IP address of the secondary name server

En caso de que el servidor DNS interno pueda validar si este servidor resuelve los nombres DNS correctamente usando el comando dig:

$ dig google.com @your.dns.server.com +short

Ver y manipular rutas #

Manipular la tabla de enrutamiento IP usando route #

Mostrar tabla de enrutamiento

$ route # Displays list or routes and also resolves host names
$ route -n # Displays list of routes without resolving host names for faster results

Agregar/Eliminar ruta

Opción Descripción
añadir o del Agregar o eliminar una ruta
-anfitrión x.x.x.x Agregar ruta a un solo host identificado por la dirección IP
-net x.x.x.x Agregar ruta a una red identificada por la dirección de red
gw x.x.x.x Especifique la puerta de enlace de la red
máscara de red x.x.x.x Especifique la máscara de red de la red
predeterminado Agregar una ruta predeterminada

Ejemplos:

  • agregar ruta a un host $ ruta agregar -host x.x.x.x eth1
  • agregar ruta a una red $ ruta agregar -net 2.2.2.0 netmask 255.255.255.0 eth0
  • Alternativamente, también podría usar el formato cidr para agregar una ruta a la red route add -net 2.2.2.0/24 eth0
  • agregar puerta de enlace predeterminada $ ruta agregar gw predeterminado 2.2.2.1 eth0
  • eliminar una ruta $ ruta del -net 2.2.2.0/24

Manipular la tabla de enrutamiento IP usando ip #

Mostrar tabla de enrutamiento

$ ip route show # List routing table

Agregar/Eliminar ruta

Opción Descripción
añadir o del o cambiar o agregar o reemplazar Cambiar una ruta
mostrar o descarga el comando muestra el contenido de las tablas de enrutamiento o lo elimina
restaurar restaurar la información de la tabla de enrutamiento desde stdin
obtener este comando obtiene una ruta única a un destino e imprime su contenido exactamente como lo ve el kernel

Ejemplos:

  • Establezca la puerta de enlace predeterminada en 1.2.3.254 $ ip route add default via 1.2.3.254
  • Agrega una ruta predeterminada (para todas las direcciones) a través de la puerta de enlace local 192.168.1.1 a la que se puede acceder en el dispositivo eth0 $ ip route add default via 192.168.1.1 dev eth0

Configure un nombre de host para algún otro sistema en su red #

Puede configurar su sistema Linux (o macOS) para vincular un identificador <nombre de host> a la dirección IP de algún otro sistema en su red. Puedes configurarlo:

  • Todo el sistema. Debe modificar el archivo /etc/hosts. Solo tienes que agregar a ese archivo una nueva línea que contenga:

    1. the remote system’s IP address <ip_rem>,
    2. one or more blank spaces, and
    3. the identifier <hostname>.

  • Para un solo usuario. Debería modificar el archivo ~/.hosts — tendría que crearlo. No es tan simple como para todo el sistema. [Aquí][hostname-s-user] puedes ver una explicación.

Por ejemplo, podría agregar esta línea usando la herramienta Unix cat. Supongamos que quiere hacer un ping a una PC en su red local cuya dirección IP es 192.168.1.44 y quiere referirse a esa dirección IP simplemente por remote_pc. Luego debes escribir en tu caparazón:

$ sudo cat 192.168.1.44 remote_pc

Entonces puedes hacer ese ping con solo:

$ ping remote_pc
