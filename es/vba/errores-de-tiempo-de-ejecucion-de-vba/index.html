El código que compila todavía puede tener errores en tiempo de ejecución. Este tema enumera los más comunes, sus causas y cómo evitarlos.

Error en tiempo de ejecución ‘6’: desbordamiento #

Código incorrecto #

Sub DoSomething()
    Dim row As Integer
    For row = 1 To 100000
        'do stuff
    Next
End Sub

¿Por qué esto no funciona? #

El tipo de datos Integer es un entero con signo de 16 bits con un valor máximo de 32.767; asignarlo a algo más grande que eso desbordará el tipo y generará este error.

Código correcto #

Sub DoSomething()
    Dim row As Long
    For row = 1 To 100000
        'do stuff
    Next
End Sub

¿Por qué funciona esto? #

Al usar un entero Largo (32 bits) en su lugar, ahora podemos hacer un ciclo que itera más de 32,767 veces sin desbordar el tipo de la variable contador.

Otras notas #

Consulte Tipos de datos y límites para obtener más información.

Error en tiempo de ejecución ‘9’: Subíndice fuera de rango #

Código incorrecto #

Sub DoSomething()
    Dim foo(1 To 10)
    Dim i As Long
    For i = 1 To 100
        foo(i) = i
    Next
End Sub

¿Por qué esto no funciona? #

foo es una matriz que contiene 10 elementos. Cuando el contador de bucle i alcanza un valor de 11, foo(i) está fuera de rango. Este error ocurre cada vez que se accede a una matriz o colección con un índice que no existe en esa matriz o colección.

Código correcto #

Sub DoSomething()
    Dim foo(1 To 10)
    Dim i As Long
    For i = LBound(foo) To UBound(foo)
        foo(i) = i
    Next
End Sub

¿Por qué funciona esto? #

Utilice las funciones LBound y UBound para determinar los límites inferior y superior de una matriz, respectivamente.

Otras notas #

Cuando el índice es una cadena, p. ThisWorkbook.Worksheets("No existo"), este error significa que el nombre proporcionado no existe en la colección consultada.

Sin embargo, el error real es específico de la implementación; Colección generará el error de tiempo de ejecución 5 “Argumento o llamada de procedimiento no válido” en su lugar:

Sub RaisesRunTimeError5()
    Dim foo As New Collection
    foo.Add "foo", "foo"
    Debug.Print foo("bar")
End Sub

Error en tiempo de ejecución ‘13’: no ​​coincide el tipo #

Código incorrecto #

Public Sub DoSomething()
    DoSomethingElse "42?"
End Sub

Private Sub DoSomethingElse(foo As Date)
'    Debug.Print MonthName(Month(foo))
End Sub

¿Por qué esto no funciona? #

VBA está tratando de convertir el argumento "42?" en un valor Date. Cuando falla, la llamada a DoSomethingElse no se puede ejecutar, porque VBA no sabe qué fecha pasar, por lo que genera el error de tiempo de ejecución 13 error de coincidencia de tipos, porque el tipo del argumento no coincide con el esperado type (y tampoco se puede convertir implícitamente).

Código correcto #

Public Sub DoSomething()
    DoSomethingElse Now
End Sub

Private Sub DoSomethingElse(foo As Date)
'    Debug.Print MonthName(Month(foo))
End Sub

¿Por qué funciona esto? #

Al pasar un argumento Date a un procedimiento que espera un parámetro Date, la llamada puede tener éxito.

Error en tiempo de ejecución ‘91’: variable de objeto o variable de bloque no configurada #

Código incorrecto #

Sub DoSomething()
    Dim foo As Collection
    With foo
        .Add "ABC"
        .Add "XYZ"
    End With
End Sub

¿Por qué esto no funciona? #

Las variables de objeto contienen una referencia, y las referencias deben establecerse usando la palabra clave Set. Este error ocurre cada vez que se realiza una llamada de miembro en un objeto cuya referencia es “Nada”. En este caso, foo es una referencia de Colección, pero no está inicializada, por lo que la referencia contiene Nothing, y no podemos llamar a .Add en Nothing.

Código correcto #

Sub DoSomething()
    Dim foo As Collection
    Set foo = New Collection
    With foo
        .Add "ABC"
        .Add "XYZ"
    End With
End Sub

¿Por qué funciona esto? #

Al asignar a la variable de objeto una referencia válida usando la palabra clave Set, las llamadas .Add tienen éxito.

Otras notas #

A menudo, una función o propiedad puede devolver una referencia de objeto; un ejemplo común es el método Range.Find de Excel, que devuelve un objeto Range:

Dim resultRow As Long
resultRow = SomeSheet.Cells.Find("Something").Row

Sin embargo, la función puede muy bien devolver ‘Nada’ (si no se encuentra el término de búsqueda), por lo que es probable que la llamada al miembro encadenado ‘.Row’ falle.

Antes de llamar a los miembros del objeto, verifique que la referencia esté configurada con una condición If Not xxxx Is Nothing:

Dim result As Range
Set result = SomeSheet.Cells.Find("Something")

Dim resultRow As Long
If Not result Is Nothing Then resultRow = result.Row

Error en tiempo de ejecución ‘20’: reanudar sin error #

Código incorrecto #

Sub DoSomething()
    On Error GoTo CleanFail
    DoSomethingElse

CleanFail:
    Debug.Print Err.Number
    Resume Next
End Sub

¿Por qué esto no funciona? #

Si el procedimiento DoSomethingElse genera un error, la ejecución salta a la etiqueta de línea CleanFail, imprime el número de error y la instrucción Resume Next vuelve a la instrucción que sigue inmediatamente a la línea donde ocurrió el error, que en este caso El caso es la instrucción Debug.Print: la subrutina de manejo de errores se ejecuta sin un contexto de error, y cuando se alcanza la instrucción Resume Next, se genera el error de tiempo de ejecución 20 porque no hay ningún lugar al que continuar.

Código correcto #

Sub DoSomething()
    On Error GoTo CleanFail
    DoSomethingElse

    Exit Sub    
CleanFail:
    Debug.Print Err.Number
    Resume Next
End Sub

¿Por qué funciona esto? #

Al introducir una instrucción Exit Sub antes de la etiqueta de línea CleanFail, hemos segregado la subrutina de manejo de errores CleanFail del resto del cuerpo del procedimiento; la única forma de ejecutar la subrutina de manejo de errores es a través de On Salto de error; por lo tanto, ninguna ruta de ejecución llega a la instrucción Reanudar` fuera de un contexto de error, lo que evita el error de tiempo de ejecución 20.

Otras notas #

Esto es muy similar a Error de tiempo de ejecución ‘3’: Retorno sin GoSub; en ambas situaciones, la solución es asegurarse de que la ruta de ejecución normal no pueda entrar en una subrutina (identificada por una etiqueta de línea) sin un salto explícito (suponiendo que On Error GoTo se considere un salto explícito).

Error en tiempo de ejecución ‘3’: Volver sin GoSub #

Código incorrecto #

Sub DoSomething()
    GoSub DoThis
DoThis:
    Debug.Print "Hi!"
    Return
End Sub

¿Por qué esto no funciona? #

La ejecución entra en el procedimiento DoSomething, salta a la etiqueta DoThis, imprime “¡Hola!” a la salida de depuración, regresa a la instrucción inmediatamente después de la llamada GoSub, imprime “¡Hola!” de nuevo, y luego se encuentra con una declaración Return, pero no hay ningún lugar al que regresar ahora, porque no llegamos aquí con una declaración GoSub.

Código correcto #

Sub DoSomething()
    GoSub DoThis
    Exit Sub
DoThis:
    Debug.Print "Hi!"
    Return
End Sub

¿Por qué funciona esto? #

Al introducir una instrucción Exit Sub antes de la etiqueta de línea DoThis, hemos segregado la subrutina DoThis del resto del cuerpo del procedimiento; la única forma de ejecutar la subrutina DoThis es a través de GoSub salto.

Otras notas #

GoSub/Return está en desuso y debe evitarse en favor de las llamadas a procedimientos reales. Un procedimiento no debe contener subrutinas, aparte de los controladores de errores.

Esto es muy similar a [Error en tiempo de ejecución ‘20’: Reanudar sin error](https://www.wikiod.com/es/vba/errores-de-tiempo-de-ejecucion-de-vba#Error de tiempo de ejecución ‘20’: reanudar sin error -reanudar-sin-error); en ambas situaciones, la solución es asegurarse de que la ruta de ejecución normal no pueda entrar en una subrutina (identificada por una etiqueta de línea) sin un salto explícito (suponiendo que On Error GoTo se considere un salto explícito).
