Variante #

Dim Value As Variant    'Explicit
Dim Value               'Implicit

Una variante es un tipo de datos COM que se usa para almacenar e intercambiar valores de tipos arbitrarios, y cualquier otro tipo en VBA se puede asignar a una variante. Las variables declaradas sin un tipo explícito especificado por As [Type] tienen como valor predeterminado Variant.

Las variantes se almacenan en la memoria como una [estructura VARIANT] (https://msdn.microsoft.com/en-us/library/windows/desktop/ms221627(v=vs.85).aspx) que consta de un descriptor de tipo de byte (VARTYPE) seguido de 6 bytes reservados y luego un área de datos de 8 bytes. Para los tipos numéricos (incluidos Date y Boolean), el valor subyacente se almacena en la propia Variant. Para todos los demás tipos, el área de datos contiene un puntero al valor subyacente.

El tipo subyacente de una variante se puede determinar con la función VarType() que devuelve el valor numérico almacenado en el descriptor de tipo, o la función TypeName() que devuelve la representación de cadena:

Dim Example As Variant
Example = 42
Debug.Print VarType(Example)    'Prints 2 (VT_I2)
Debug.Print TypeName(Example)   'Prints "Integer"
Example = "Some text"
Debug.Print VarType(Example)    'Prints 8 (VT_BSTR)
Debug.Print TypeName(Example)   'Prints "String"

Debido a que las variantes pueden almacenar valores de cualquier tipo, las asignaciones de literales sin [sugerencias de tipo] (https://www.wikiod.com/es/vba/declaracion-de-variables#Sugerencias de tipo) se convertirán implícitamente a una Variante del tipo apropiado de acuerdo con la siguiente tabla. Los literales con sugerencias de tipo se convertirán en una variante del tipo sugerido.

Valor Tipo resultante
Valores de cadena Cadena
Números de coma no flotante en el rango de enteros Entero
Números de coma no flotante en rango largo Largo
Números de coma no flotante fuera del rango largo Doble
Todos los números de coma flotante Doble

Nota: A menos que haya una razón específica para usar una variante (es decir, un iterador en un bucle For Each o un requisito de API), el tipo generalmente se debe evitar para tareas de rutina por las siguientes razones:

  • No son de tipo seguro, lo que aumenta la posibilidad de errores de tiempo de ejecución. Por ejemplo, una variante que tenga un valor entero se convertirá silenciosamente en larga en lugar de desbordarse.
  • Introducen una sobrecarga de procesamiento al requerir al menos una desreferencia de puntero adicional.
  • El requisito de memoria para una variante es siempre al menos 8 bytes superior al necesario para almacenar el tipo subyacente.

La función de conversión para convertir a Variant es CVar().

Booleano #

Dim Value As Boolean

Un booleano se utiliza para almacenar valores que se pueden representar como verdadero o falso. Internamente, el tipo de datos se almacena como un valor de 16 bits con 0 que representa Falso y cualquier otro valor que representa Verdadero.

Cabe señalar que cuando un valor booleano se convierte en un tipo numérico, todos los bits se establecen en 1. Esto da como resultado una representación interna de -1 para tipos con signo y el valor máximo para un tipo sin signo (Byte).

Dim Example As Boolean
Example = True
Debug.Print CInt(Example)  'Prints -1
Debug.Print CBool(42)      'Prints True
Debug.Print CByte(True)    'Prints 255

La función de conversión para convertir a booleano es CBool(). Aunque se representa internamente como un número de 16 bits, la conversión a un booleano desde valores fuera de ese rango está a salvo del desbordamiento, aunque establece todos los 16 bits en 1:

Dim Example As Boolean
Example = CBool(2 ^ 17)
Debug.Print CInt(Example)   'Prints -1
Debug.Print CByte(Example)  'Prints 255

Cuerda #

Una cadena representa una secuencia de caracteres y viene en dos sabores:

Longitud variable #

Dim Value As String

Una cadena de longitud variable permite agregar y truncar y se almacena en la memoria como COM [BSTR] (https://msdn.microsoft.com/en-us/library/windows/desktop/ms221069(v=vs.85).aspx ). Consiste en un entero sin signo de 4 bytes que almacena la longitud de la cadena en bytes seguido de los datos de la cadena como caracteres anchos (2 bytes por carácter) y termina con 2 bytes nulos. Por lo tanto, la longitud máxima de cadena que puede manejar VBA es de 2 147 483 647 caracteres.

El puntero interno a la estructura (recuperable por la función StrPtr()) apunta a la ubicación de memoria de los datos, no al prefijo de longitud. Esto significa que una cadena VBA se puede pasar directamente a funciones API que requieren un puntero a una matriz de caracteres.

Debido a que la longitud puede cambiar, VBA reasigna memoria para una cadena cada vez que se asigna la variable a, lo que puede imponer penalizaciones de rendimiento para los procedimientos que los modifican repetidamente.

Longitud fija #

Dim Value As String * 1024    'Declares a fixed length string of 1024 characters.

A las cadenas de longitud fija se les asignan 2 bytes para cada carácter y se almacenan en la memoria como una matriz de bytes simple. Una vez asignada, la longitud de la cadena es inmutable. Están no terminados en nulo en la memoria, por lo que una cadena que llena la memoria asignada con caracteres no nulos no es adecuada para pasar a funciones API que esperan una cadena terminada en nulo.

Las cadenas de longitud fija tienen una limitación de índice de 16 bits heredada, por lo que solo pueden tener una longitud máxima de 65 535 caracteres. Intentar asignar un valor más largo que el espacio de memoria disponible no dará como resultado un error de tiempo de ejecución; en su lugar, el valor resultante simplemente se truncará:

Dim Foobar As String * 5
Foobar = "Foo" & "bar"
Debug.Print Foobar          'Prints "Fooba"

La función de conversión para convertir a una cadena de cualquier tipo es CStr().

byte #

Dim Value As Byte

Un byte es un tipo de datos de 8 bits sin firmar. Puede representar números enteros entre 0 y 255 e intentar almacenar un valor fuera de ese rango dará como resultado error de tiempo de ejecución 6: Desbordamiento . Byte es el único tipo intrínseco sin firmar disponible en VBA.

La función de conversión para convertir a Byte es CByte(). Para conversiones de tipos de punto flotante, el resultado se redondea al valor entero más cercano con .5 redondeando hacia arriba.

Arreglos de bytes y cadenas

Las cadenas y las matrices de bytes se pueden sustituir entre sí mediante una asignación simple (no se necesitan funciones de conversión).

Por ejemplo:

Sub ByteToStringAndBack()

Dim str As String
str = "Hello, World!"

Dim byt() As Byte
byt = str

Debug.Print byt(0)  ' 72

Dim str2 As String
str2 = byt

Debug.Print str2    ' Hello, World!

End Sub

Para poder codificar caracteres Unicode, cada carácter de la cadena ocupa dos bytes en la matriz, con el byte menos significativo primero. Por ejemplo:

Sub UnicodeExample()

Dim str As String
str = ChrW(&H2123) & "."  ' Versicle character and a dot

Dim byt() As Byte
byt = str

Debug.Print byt(0), byt(1), byt(2), byt(3)  ' Prints: 35,33,46,0

End Sub

Divisa #

Dim Value As Currency

Una moneda es un tipo de datos de coma flotante de 64 bits con signo similar a un [Doble] (https://www.wikiod.com/es/vba/tipos-de-datos-y-limites#Doble), pero con una escala de 10 000 a dar mayor precisión a los 4 dígitos a la derecha del punto decimal. Una variable Moneda puede almacenar valores desde -922.337.203.685.477,5808 hasta 922.337.203.685.477,5807, lo que le otorga la mayor capacidad de cualquier tipo intrínseco en una aplicación de 32 bits. Como implica el nombre del tipo de datos, se considera una buena práctica utilizar este tipo de datos cuando se representan cálculos monetarios, ya que la escala ayuda a evitar errores de redondeo.

La función de conversión para convertir a Moneda es CCur().

Decimales #

Dim Value As Variant
Value = CDec(1.234)

'Set Value to the smallest possible Decimal value
Value = CDec("0.0000000000000000000000000001")

El tipo de datos Decimal solo está disponible como un subtipo de Variante, por lo que debe declarar cualquier variable que deba contener un Decimal como Variante y luego asignar un Decimal valor utilizando la función CDec. La palabra clave ‘Decimal’ es una palabra reservada (lo que sugiere que VBA eventualmente agregará soporte de primera clase para el tipo), por lo que ‘Decimal’ no se puede usar como una variable o nombre de procedimiento.

El tipo Decimal requiere 14 bytes de memoria (además de los bytes requeridos por la variante principal) y puede almacenar números con hasta 28 decimales. Para números sin lugares decimales, el rango de valores permitidos es -79,228,162,514,264,337,593,543,950,335 a +79,228,162,514,264,337,593,543,950,335 inclusive. Para números con un máximo de 28 decimales, el rango de valores permitidos es de -7,9228162514264337593543950335 a +7,9228162514264337593543950335 inclusive.

Entero #

Dim Value As Integer

Un entero es un tipo de datos de 16 bits con signo. Puede almacenar números enteros en el rango de -32 768 a 32 767 e intentar almacenar un valor fuera de ese rango dará como resultado el error de tiempo de ejecución 6: desbordamiento.

Los números enteros se almacenan en la memoria como valores [little-endian] (https://en.wikipedia.org/wiki/Endianness) con los negativos representados como un [complemento a dos] (https://en.wikipedia.org/wiki /Dos).%27s_complemento).

Tenga en cuenta que, en general, es una mejor práctica usar un [Largo] (https://www.wikiod.com/es/vba/tipos-de-datos-y-limites#Largo) en lugar de un Entero a menos que el tipo más pequeño es miembro de un tipo o se requiere (ya sea por una convención de llamada API o por alguna otra razón) que sea de 2 bytes. En la mayoría de los casos, VBA trata a los enteros como de 32 bits internamente, por lo que generalmente no hay ninguna ventaja en usar el tipo más pequeño. Además, se incurre en una penalización de rendimiento cada vez que se usa un tipo Integer, ya que se convierte silenciosamente como Long.

La función de conversión para convertir a un entero es CInt(). Para conversiones de tipos de punto flotante, el resultado se redondea al valor entero más cercano con .5 redondeando hacia arriba.

Largo #

Dim Value As Long

Un Long es un tipo de datos de 32 bits con signo. Puede almacenar números enteros en el rango de -2,147,483,648 a 2,147,483,647 e intentar almacenar un valor fuera de ese rango dará como resultado el error de tiempo de ejecución 6: desbordamiento.

Los valores largos se almacenan en la memoria como valores little-endian con valores negativos representados como un [complemento a dos](https://en.wikipedia.org/wiki/Two %27s_complemento).

Tenga en cuenta que dado que Long coincide con el ancho de un puntero en un sistema operativo de 32 bits, Longs se usa comúnmente para almacenar y pasar punteros hacia y desde funciones API.

La función de conversión para convertir a Long es CLng(). Para conversiones de tipos de punto flotante, el resultado se redondea al valor entero más cercano con .5 redondeando hacia arriba.

Único #

Dim Value As Single

Un Single es un tipo de datos de punto flotante de 32 bits con signo. Se almacena internamente usando una memoria little-endian IEEE 754 diseño. Como tal, no hay un rango fijo de valores que pueda ser representado por el tipo de datos; lo que está limitado es la precisión del valor almacenado. Un Single puede almacenar un valor entero valores en el rango de -16,777,216 a 16,777,216 sin pérdida de precisión. La precisión de los números de punto flotante depende del exponente.

Un Single se desbordará si se le asigna un valor superior a aproximadamente 2128. No se desbordará con exponentes negativos, aunque la precisión utilizable será cuestionable antes de que se alcance el límite superior.

Al igual que con todos los números de punto flotante, se debe tener cuidado al realizar comparaciones de igualdad. La mejor práctica es incluir un valor delta apropiado para la precisión requerida.

La función de conversión para convertir a Single es CSng().

Doble #

Dim Value As Double

Un Double es un tipo de datos de punto flotante de 64 bits con signo. Al igual que Single, se almacena internamente usando un [little-endian](https://en .wikipedia.org/wiki/Endianness) IEEE 754 diseño de memoria y se deben tomar las mismas precauciones con respecto a la precisión. Un Double puede almacenar valores enteros en el rango de -9,007,199,254,740,992 a 9,007,199,254,740,992 sin pérdida de precisión. La precisión de los números de punto flotante depende del exponente.

Un Double se desbordará si se le asigna un valor superior a aproximadamente 21024. No se desbordará con exponentes negativos, aunque la precisión utilizable será cuestionable antes de que se alcance el límite superior.

La función de conversión para convertir a Double es CDbl().

Fecha #

Dim Value As Date

Un tipo de fecha se representa internamente como un tipo de datos de punto flotante de 64 bits con signo con el valor a la izquierda del decimal que representa el número de días desde la fecha de época del 30 de diciembre de 1899 (aunque consulte el nota a continuación). El valor a la derecha del decimal representa el tiempo como un día fraccionario. Por lo tanto, una Fecha entera tendría un componente de tiempo de 12:00:00 AM y x.5 tendría un componente de tiempo de 12:00:00 PM.

Los valores válidos para Fechas están entre el 1 de enerost 100 y el 31 de diciembrest 9999. Dado que un Doble tiene un rango mayor, es posible desbordar una Fecha asignando valores fuera de ese rango

Como tal, se puede usar indistintamente con un Doble para cálculos de fecha:

Dim MyDate As Double
MyDate = 0                                  'Epoch date.
Debug.Print Format$(MyDate, "yyyy-mm-dd")   'Prints 1899-12-30.
MyDate = MyDate + 365
Debug.Print Format$(MyDate, "yyyy-mm-dd")   'Prints 1900-12-30.

La función de conversión para convertir a una fecha es CDate(), que acepta cualquier representación de fecha/hora de cadena de tipo numérico. Es importante tener en cuenta que las representaciones de cadenas de fechas se convertirán en función de la configuración regional actual en uso, por lo que se deben evitar las conversiones directas si el código está destinado a ser portátil.

Largo largo #

Dim Value As LongLong

Un LongLong es un tipo de datos de 64 bits con signo y solo está disponible en aplicaciones de 64 bits. No está disponible en aplicaciones de 32 bits que se ejecutan en sistemas operativos de 64 bits. Puede almacenar valores enteros en el rango de -9,223,372,036,854,775,808 a 9,223,372,036,854,775,807 e intentar almacenar un valor fuera de ese rango dará como resultado el error de tiempo de ejecución 6: desbordamiento.

LongLongs se almacenan en la memoria como valores little-endian con valores negativos representados como un [complemento a dos](https://en.wikipedia.org/wiki/Two %27s_complemento).

El tipo de datos LongLong se introdujo como parte de la compatibilidad con el sistema operativo de 64 bits de VBA. En aplicaciones de 64 bits, este valor se puede usar para almacenar y pasar punteros a las API de 64 bits.

La función de conversión para convertir a LongLong es CLngLng(). Para conversiones de tipos de punto flotante, el resultado se redondea al valor entero más cercano con .5 redondeando hacia arriba.

PuntoLargo #

Dim Value As LongPtr

LongPtr se introdujo en VBA para admitir plataformas de 64 bits. En un sistema de 32 bits, se trata como Largo y en sistemas de 64 bits se trata como un LongLong.

Su uso principal es proporcionar una forma portátil de almacenar y pasar punteros en ambas arquitecturas (consulte https://www.wikiod.com/es/vba/compilacion-condicional Changing-code-behavior-at-compile-time #t=20160724185822285365).

Aunque el sistema operativo lo trata como una dirección de memoria cuando se usa en llamadas API, debe tenerse en cuenta que VBA lo trata como un tipo firmado (y, por lo tanto, sujeto a desbordamiento de no firmado a firmado). Por esta razón, cualquier aritmética de punteros realizada usando LongPtrs no debe usar comparaciones > o <. Esta “peculiaridad” también hace posible que agregar desplazamientos simples que apuntan a direcciones válidas en la memoria pueda causar errores de desbordamiento, por lo que se debe tener cuidado al trabajar con punteros en VBA.

La función de conversión para convertir a LongPtr es CLngPtr(). Para conversiones de tipos de coma flotante, el resultado se redondea al valor entero más cercano con un redondeo de .5 (aunque dado que generalmente es una dirección de memoria, usarla como destino de asignación para un cálculo de coma flotante es, en el mejor de los casos, peligroso).
