Introducción a los procedimientos #

Un Sub es un procedimiento que realiza una tarea específica pero no devuelve un valor específico.

Sub ProcedureName ([argument_list])
    [statements]
End Sub

Si no se especifica ningún modificador de acceso, un procedimiento es Público por defecto.

Una Función es un procedimiento que recibe datos y devuelve un valor, idealmente sin efectos secundarios globales o de alcance de módulo.

Function ProcedureName ([argument_list]) [As ReturnType]
    [statements]
End Function

Una Propiedad es un procedimiento que encapsula datos del módulo. Una propiedad puede tener hasta 3 elementos de acceso: Get para devolver un valor o referencia de objeto, Let para asignar un valor y/o Set para asignar una referencia de objeto.

Property Get|Let|Set PropertyName([argument_list]) [As ReturnType]
    [statements]
End Property

Las propiedades generalmente se usan en módulos de clase (aunque también están permitidas en módulos estándar), exponiendo al usuario a datos que de otro modo serían inaccesibles para el código de llamada. Una propiedad que solo expone un descriptor de acceso Get es de “solo lectura”; una propiedad que solo expondría un descriptor de acceso Let y/o Set es “solo escritura”. Las propiedades de solo escritura no se consideran una buena práctica de programación: si el código del cliente puede escribir un valor, debería poder leerlo. Considere implementar un procedimiento Sub en lugar de crear una propiedad de solo escritura.

Devolviendo un valor #

Un procedimiento Función o Propiedad Obtener puede (¡y debería!) devolver un valor a su llamador. Esto se hace asignando el identificador del procedimiento:

Property Get Foo() As Integer
    Foo = 42
End Property

Función con ejemplos #

Como se indicó anteriormente, las Funciones son procedimientos más pequeños que contienen pequeños fragmentos de código que pueden ser repetitivos dentro de un Procedimiento.

Las funciones se utilizan para reducir la redundancia en el código.

Similar a un procedimiento, una función puede declararse con o sin una lista de argumentos.

La función se declara como un tipo de devolución, ya que todas las funciones devuelven un valor. El nombre y la variable de retorno de una función son iguales.

  1. Función con parámetro:

     Function check_even(i as integer) as boolean
 if (i mod  2) = 0 then
 check_even = True
 else
 check_even=False
 end if
 end Function

  2. Función sin parámetro:

     Function greet() as String
 greet= "Hello Coder!"
 end Function

La función se puede llamar de varias maneras dentro de una función. Dado que una función declarada con un tipo de retorno es básicamente una variable. se utiliza de forma similar a una variable.

Llamadas Funcionales:

    call greet() 'Similar to a Procedural call just allows the Procedure to use the
                 'variable greet 
    string_1=greet() 'The Return value of the function is used for variable
                     'assignment

Además, la función también se puede usar como condición para if y otras declaraciones condicionales.

      for i = 1 to 10
      if check_even(i) then
      msgbox i & " is Even"
      else
      msgbox i & " is Odd"
      end if
      next i

Además, más funciones pueden tener modificadores como By ref y By val para sus argumentos.
