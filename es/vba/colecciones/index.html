Una Colección es un objeto contenedor que se incluye en el tiempo de ejecución de VBA. No se requieren referencias adicionales para poder usarlo. Una ‘Colección’ se puede utilizar para almacenar elementos de cualquier tipo de datos y permite la recuperación mediante el índice ordinal del elemento o mediante el uso de una clave única opcional.

Obtener el número de elementos de una colección #

El número de elementos en una ‘Colección’ se puede obtener llamando a su función ‘.Count’:

Sintaxis:

.Count()

Uso de muestra:

Public Sub Example()
    Dim foo As New Collection
    
    With foo
        .Add "One"
        .Add "Two"
        .Add "Three"
        .Add "Four"
    End With
     
    Debug.Print foo.Count   'Prints 4
End Sub

Determinar si existe una clave o un elemento en una colección #

Llaves #

A diferencia de un Scripting.Dictionary, una Colección no tiene un método para determinar si una clave dada existe o una forma para recuperar claves que están presentes en la Colección. El único método para determinar si una clave está presente es usar el controlador de errores:

Public Function KeyExistsInCollection(ByVal key As String, _
                                      ByRef container As Collection) As Boolean
    With Err
        If container Is Nothing Then .Raise 91
        On Error Resume Next
        Dim temp As Variant
        temp = container.Item(key)
        On Error GoTo 0

        If .Number = 0 Then
            KeyExistsInCollection = True 
        ElseIf .Number <> 5 Then
            .Raise .Number
        End If
    End With
End Function

Elementos #

La única forma de determinar si un elemento está contenido en una “Colección” es iterar sobre la “Colección” hasta que se localice el elemento. Tenga en cuenta que debido a que una Colección puede contener primitivos u objetos, se necesita un manejo adicional para evitar errores de tiempo de ejecución durante las comparaciones:

Public Function ItemExistsInCollection(ByRef target As Variant, _
                                       ByRef container As Collection) As Boolean
    Dim candidate As Variant
    Dim found As Boolean
    
    For Each candidate In container
        Select Case True
            Case IsObject(candidate) And IsObject(target)
                found = candidate Is target
            Case IsObject(candidate), IsObject(target)
                found = False
            Case Else
                found = (candidate = target)
        End Select
        If found Then
            ItemExistsInCollection = True
            Exit Function
        End If
    Next
End Function

Adición de elementos a una colección #

Los elementos se agregan a una Colección llamando a su método .Add:

Sintaxis:

.Add(item, [key], [before, after])
Parámetro Descripción
elemento El artículo a almacenar en la Colección. Esto puede ser esencialmente cualquier valor al que se le pueda asignar una variable, incluidos tipos primitivos, matrices, objetos y “Nada”.
clave Opcional. Una String que sirve como identificador único para recuperar elementos de la Colección. Si la clave especificada ya existe en la Colección, dará como resultado un error de tiempo de ejecución 457: “Esta clave ya está asociada con un elemento de esta colección”.
antes Opcional. Una clave existente (valor String) o índice (valor numérico) para insertar el elemento anterior en la Colección. Si se proporciona un valor, el parámetro después debe estar vacío o se producirá un error de tiempo de ejecución 5: “Argumento o llamada de procedimiento no válido”. Si se pasa una clave String que no existe en la Colección, se producirá un error de tiempo de ejecución 5: “Llamada o argumento de procedimiento no válido”. Si se pasa un índice numérico que no existe en la Colección, se producirá un error de tiempo de ejecución 9: “Subíndice fuera de rango”.
después Opcional. Una clave existente (valor String) o índice (valor numérico) para insertar el elemento después en la Colección. Si se da un valor, el parámetro before debe estar vacío. Los errores generados son idénticos al parámetro before.

Notas:

  • Las claves no distinguen entre mayúsculas y minúsculas. .Add "Bar", "Foo" y .Add "Baz", "foo" resultará en una colisión de teclas.

  • Si no se proporciona ninguno de los parámetros opcionales antes o después, el elemento se agregará después del último elemento en la Colección.

  • Las inserciones realizadas especificando un parámetro antes o después alterarán los índices numéricos de los miembros existentes para que coincidan con su nueva posición. Esto significa que se debe tener cuidado al realizar inserciones en bucles utilizando índices numéricos.

Uso de muestra:

Public Sub Example()
    Dim foo As New Collection
    
    With foo
        .Add "One"            'No key. This item can only be retrieved by index.
        .Add "Two", "Second"  'Key given. Can be retrieved by key or index.
        .Add "Three", , 1     'Inserted at the start of the collection.
        .Add "Four", , , 1    'Inserted at index 2.
    End With
    
    Dim member As Variant
    For Each member In foo
        Debug.Print member    'Prints "Three, Four, One, Two"
    Next
End Sub

Eliminar elementos de una colección #

Los elementos se eliminan de una Colección llamando a su método .Remove:

Sintaxis:

.Remove(index)
Parámetro Descripción
índice El elemento que se eliminará de la Colección. Si el valor pasado es de tipo numérico o Variante con un subtipo numérico, se interpretará como un índice numérico. Si el valor pasado es un String o Variant que contiene una cadena, se interpretará como una clave. Si se pasa una clave de cadena que no existe en la Colección, se producirá un error de tiempo de ejecución 5: “Argumento o llamada de procedimiento no válido”. Si se pasa un índice numérico que no existe en la Colección, se producirá un error de tiempo de ejecución 9: “Subíndice fuera de rango”.

Notas:

  • Eliminar un elemento de una Colección cambiará los índices numéricos de todos los elementos que le siguen en la Colección. Los bucles For que usan índices numéricos y eliminan elementos deben ejecutarse hacia atrás (Paso -1) para evitar excepciones de subíndice y elementos omitidos.
  • Por lo general, los elementos no deben eliminarse de una ‘Colección’ desde el interior de un bucle ‘Para cada uno’, ya que puede dar resultados impredecibles.

Uso de muestra:

Public Sub Example()
    Dim foo As New Collection
    
    With foo
        .Add "One"
        .Add "Two", "Second"
        .Add "Three"
        .Add "Four"
    End With
     
    foo.Remove 1            'Removes the first item.
    foo.Remove "Second"     'Removes the item with key "Second".
    foo.Remove foo.Count    'Removes the last item.
    
    Dim member As Variant
    For Each member In foo
        Debug.Print member  'Prints "Three"
    Next
End Sub

Recuperar elementos de una colección #

Los elementos se pueden recuperar de una ‘Colección’ llamando a la función ‘.Item’.

Sintaxis:

.Item(index)
Parámetro Descripción
índice El artículo a recuperar de la Colección. Si el valor pasado es de tipo numérico o Variante con un subtipo numérico, se interpretará como un índice numérico. Si el valor pasado es un String o Variant que contiene una cadena, se interpretará como una clave. Si se pasa una clave de cadena que no existe en la Colección, se producirá un error de tiempo de ejecución 5: “Argumento o llamada de procedimiento no válido”. Si se pasa un índice numérico que no existe en la Colección, se producirá un error de tiempo de ejecución 9: “Subíndice fuera de rango”.

Notas:

  • .Item es el miembro predeterminado de Collection. Esto permite flexibilidad en la sintaxis, como se demuestra en el ejemplo de uso a continuación.
  • Los índices numéricos están basados ​​en 1.
  • Las claves no distinguen entre mayúsculas y minúsculas. .Item("Foo") y .Item("foo") se refieren a la misma tecla.
  • El parámetro índice no se convierte implícitamente en un número de una ‘Cadena’ o viceversa. Es muy posible que .Item(1) y .Item("1") se refieran a diferentes elementos de la Colección.

Uso de muestra (índices):

Public Sub Example()
    Dim foo As New Collection
    
    With foo
        .Add "One"
        .Add "Two"
        .Add "Three"
        .Add "Four"
    End With
     
    Dim index As Long
    For index = 1 To foo.Count
        Debug.Print foo.Item(index) 'Prints One, Two, Three, Four
    Next
End Sub

Uso de muestra (claves):

Public Sub Example()
    Dim keys() As String
    keys = Split("Foo,Bar,Baz", ",")
    Dim values() As String
    values = Split("One,Two,Three", ",")
        
    Dim foo As New Collection
    Dim index As Long
    For index = LBound(values) To UBound(values)
        foo.Add values(index), keys(index)
    Next
     
    Debug.Print foo.Item("Bar") 'Prints "Two"
End Sub

Uso de muestra (sintaxis alternativa):

Public Sub Example()
    Dim foo As New Collection
    
    With foo
        .Add "One", "Foo"
        .Add "Two", "Bar"
        .Add "Three", "Baz"
    End With
    
    'All lines below print "Two"
    Debug.Print foo.Item("Bar")     'Explicit call syntax.
    Debug.Print foo("Bar")          'Default member call syntax.
    Debug.Print foo!Bar             'Bang syntax.
End Sub

Tenga en cuenta que la sintaxis bang (!) está permitida porque .Item es el miembro predeterminado y puede tomar un solo argumento String. La utilidad de esta sintaxis es cuestionable.

Limpiar todos los elementos de una colección #

La forma más fácil de borrar todos los elementos de una ‘Colección’ es simplemente reemplazarla con una nueva ‘Colección’ y dejar que la anterior quede fuera del alcance:

Public Sub Example()
    Dim foo As New Collection
    
    With foo
        .Add "One"
        .Add "Two"
        .Add "Three"
    End With
     
    Debug.Print foo.Count   'Prints 3
    Set foo = New Collection
    Debug.Print foo.Count   'Prints 0
End Sub

Sin embargo, si hay múltiples referencias a la Colección mantenida, este método solo le dará una Colección vacía para la variable asignada.

Public Sub Example()
    Dim foo As New Collection
    Dim bar As Collection
    
    With foo
        .Add "One"
        .Add "Two"
        .Add "Three"
    End With
     
    Set bar = foo
    Set foo = New Collection
    
    Debug.Print foo.Count   'Prints 0
    Debug.Print bar.Count   'Prints 3
End Sub

En este caso, la forma más fácil de borrar el contenido es recorrer el número de elementos en la Colección y eliminar repetidamente el elemento más bajo:

Public Sub ClearCollection(ByRef container As Collection)
    Dim index As Long
    For index = 1 To container.Count
        container.Remove 1
    Next
End Sub
