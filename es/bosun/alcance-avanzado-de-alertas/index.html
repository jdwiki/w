Comprender la función de transposición: t() #

Visión general #

La función de transposición es una de las funciones más poderosas de Bosun, pero también requiere esfuerzo para comprenderla. Es poderoso porque nos permite alertar en niveles diferentes a la estructura de etiquetas de los datos subyacentes.

Transpose cambia el alcance de su alerta. Esto le permite alcance las cosas en colecciones más grandes. Entonces, por ejemplo, si tiene consultas que devuelven un alcance de host, clúster y desea alertar en función del estado del clúster y no de los hosts individuales, se puede usar la transposición para hacer esto.

¿Qué es alcance? El alcance es la lista de claves de etiqueta que componen su resultado final. Por ejemplo:

  • Si el alcance es host, obtendrá resultados por host en sus alertas.
  • Si su alcance está vacío (sin claves de etiqueta), entonces solo podría recibir una alerta.
  • Si su alcance es host,iface, podría recibir alertas para cada interfaz en cada host en el resultado.

Entonces, las alertas que recibimos están vinculadas a las etiquetas de los datos. La función de transposición nos permite alertar en diferentes ámbitos además de la estructura de la etiqueta métrica. Entonces podemos consultar cosas que resultan en host,cluster pero alertan en un alcance de cluster.

Desglosando la función #

La firma de la función de transposición es:

t(numberSet, group string) seriesSet

Entonces toma un numberSet, un scope a.k.a. group para el resultado, y devuelve un seriesSet

¿Qué son esas cosas? #

Conjunto, conjuntos de números y conjuntos de series #

Los resultados de muchas funciones en bosun son conjuntos, generalmente un numberSet o seriesSet. Todo el conjunto en el resultado comparte la misma etiqueta claves. Y cada elemento del conjunto es exclusivo del valor de cada clave correspondiente. Si el valor de cada elemento del conjunto es una serie (marca de tiempo:valor, marca de tiempo:valor), entonces tenemos un conjunto de series. Si el valor de cada elemento es solo un número, entonces tenemos un conjunto de números.

La carne de esto #

Transpose toma un numberSet y devuelve un seriesSet con un alcance mayor (menos claves de etiquetas). El seriesSet resultante es un poco extraño porque el índice no es el tiempo como es el caso habitual de un seriesSet, por lo que el valor del tiempo ya no es el tiempo y es solo un número de índice. Por lo tanto, debe ser ignorado.

Así que terminamos transponiendo elementos del conjunto en valores del conjunto resultante, donde el tipo de conjunto resultante (un conjunto de series) puede contener múltiples valores:

Veamos un ejemplo: #

# This returns a seriesSet of a scope of host,cluster
$connByHostCluster = q("sum:rate{counter,,1}:elastic.http.total_opened{cluster=StackExchangeNetwork|LogStash,host=ny-*}", "1h", "")


# Turn each item in the set into a numberSet by reducing it via average
$avgConnByHostCluster = avg($connByHostCluster)


# Transpose to new scope 
$clusterScope = t($avgConnByHostCluster, "cluster")

Ahora puede hacer cosas interesantes con cada elemento que representa el grupo. Por ejemplo, podría hacer sum($clusterScope > 5) (Tenga en cuenta que $clusterScope es un seriesSet) para obtener el recuento de elementos en el clúster donde cada elemento tiene una tasa superior a cinco. Luego podría alertar si el conteo es mayor que cierto valor. Por ejemplo, también podría usar len($clusterScope) para obtener la cantidad de hosts en cada clúster y alertar sobre la cantidad de hosts por encima del umbral relativo a la cantidad de hosts en el clúster.
