Instalación y configuración #

Fondo #

============

WebStorm es un entorno de desarrollo integrado (IDE) ligero pero potente perfectamente equipado para el desarrollo complejo del lado del cliente y del lado del servidor, es multiplataforma y funciona en Windows, Mac OS X y Linux.

WebStorm cuenta con soporte avanzado para JavaScript, HTML, CSS y sus sucesores modernos, así como para marcos como AngularJS o React, * depuración* e integración con VCS y varias herramientas de desarrollo web. Además, proporciona un entorno de desarrollo local configurado y listo para usar, incluido soporte para Node.js, Meteor, CoffeeScript, TypeScript, Dart , Sass y más.


Requisitos de hardware #

=======================

Para ejecutar WebStorm sin problemas, se requieren los siguientes requisitos de hardware:

  • 1 GB de RAM mínimo, 2 GB de RAM recomendados
  • Resolución de pantalla mínima de 1024x768

Requisitos del sistema #

=====================

Para versiones anteriores de WebStorm anteriores a 2016, se requiere tener Java instalado en la máquina para poder ejecutar WebStorm, a partir de WebStorm 2016 y posteriores, JRE 1.8 se incluye con distribuciones para todas las plataformas. No necesita instalar Java en su máquina para ejecutar WebStorm.

Para el Sistema Operativo (SO) se requiere lo siguiente:

  • Microsoft Windows 10/8/7/Vista/2003/XP (incluido 64 bits)
  • OS X 10.5 o superior, incluido 10.10 (solo se admite OS X de 64 bits)
  • SO Linux de 64 bits (KDE, GNOME o Unity DE escritorio)

Instalación de Windows: #

  1. Descargue WebStorm desde la página Descargar.
  2. Ejecute el archivo WebStorm-*.exe que inicia el asistente de instalación. (el * significa la versión actual descargada)
  3. Siga todos los pasos sugeridos por el asistente. Preste especial atención a las opciones de instalación correspondientes.

Instalación de OS X: #

  1. Descargue el archivo de imagen de disco OS X WebStorm-*.dmg desde la página Descargar. (el * significa la versión actual descargada)
  2. Haga doble clic en el archivo de imagen de disco OS X WebStorm-*.dmg descargado para montarlo.
  3. Copie WebStorm a su carpeta de Aplicaciones.

Instalación de Linux: #

  1. Descargue el archivo WebStorm-*.tar.gz desde la página Descargar. (el * significa la versión actual descargada)

  2. Descomprima el archivo WebStorm-*.tar.gz en un directorio vacío usando el siguiente comando:

    tar xfz WebStorm-*.tar.gz

  3. Debido a que ejecutar WebStorm desde donde descargó el archivo puede ser un inconveniente, se recomienda que mueva la carpeta del archivo extraído o desempaquetado a la ubicación deseada usando el comando mv en uno de los siguientes formatos:

    `mv <path to extracted archive folder> <new archive folder>`

    Example #

    • mv /downloads/WebStorm-* my/desired/location
    • mv <path to WebStorm-*.tar.gz>

  4. Cambie al directorio bin en la nueva ubicación:

    cd <new archive folder>/WebStorm-*/bin

    Example #

    • cd /WebStorm-*/bin

  5. Ejecute webstorm.sh desde el subdirectorio bin.

Crear lanzador de línea de comandos #

Un lanzador de línea de comandos es una herramienta de manejo que permite abrir WebStorm usando la línea de comandos. Se puede crear fácilmente usando los menús.

Tools > Create Command-line Launcher...

Después de seleccionar la opción, se le presentará el mensaje “Crear secuencia de comandos del iniciador” para la ubicación y el nombre del iniciador de la línea de comandos. Esta ubicación debe estar dentro de $PATH para que se pueda invocar en el símbolo del sistema/terminal.

El valor predeterminado es /usr/local/bin/webstorm, donde /usr/local/bin es la ruta en la que se agregará el iniciador y webstorm es el nombre del archivo. El nombre del archivo también se usará como comando para invocar e iniciar WebStorm, puede cambiarlo, pero para este ejemplo asumiremos que es el valor predeterminado de “webstorm”

Una vez hecho esto, puede abrir una terminal o símbolo del sistema y usar el comando “webstorm” para iniciar WebStorm con un directorio específico. En el siguiente ejemplo, WebStorm se abrirá con el directorio ~/Workspace/my-project en la raíz del Proyecto.

> webstorm ~/Workspace/my-project

Consejo: Use webstorm . para abrir el directorio actual.
