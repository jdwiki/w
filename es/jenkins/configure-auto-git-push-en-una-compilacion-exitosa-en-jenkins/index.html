Configure Auto Git Push en una compilación exitosa en Jenkins

  • Configuración del trabajo de envío automático

En esta página

  • Configuración del trabajo de envío automático

Este documento lo guiará a través de los pasos para configurar un trabajo de Jenkins que permita al usuario configurar la inserción automática en una compilación exitosa. El usuario puede controlar la operación de inserción. El usuario puede elegir si desea realizar la operación de inserción automática en una compilación exitosa o no.

Configuración del trabajo de envío automático #

Cree un trabajo de compilación (según sus requisitos). Para este ejemplo, he creado un trabajo de estilo libre (AutoPush) para realizar la compilación ANT.

Vamos a crear dos variables, PUSH (parámetro de elección) y TAG_NUMBER (parámetro de cadena).

Podemos elegir el valor SÍ o NO para PUSH, esto decidirá si enviar el código a una etiqueta o no en una compilación exitosa.

Podemos especificar un nombre de etiqueta (por ejemplo, 1.0.1) para TAG_NUMBER para crear una nueva etiqueta (por ejemplo, 1.0.1) en el repositorio remoto con el mismo nombre o especificar un nombre de etiqueta existente para actualizar una etiqueta existente.

Ahora pasemos a la configuración del trabajo.

  1. Marque la casilla de verificación “Este proyecto está parametrizado” y cree un Parámetro de elección llamado “PUSH” y proporcione SÍ y NO como opciones. Este parámetro decidirá si desea enviar el código a una etiqueta/lanzamiento específico o no.

  1. Luego cree un parámetro de cadena llamado “TAG_NUMBER”, utilizando este parámetro podemos especificar un nuevo número de etiqueta para crear una nueva etiqueta o especificar un número de etiqueta existente para actualizar una etiqueta existente.

  1. En la sección Administración de código fuente, elija Git y proporcione la URL del repositorio. Este repositorio contiene el código fuente que va a compilar y, después de una compilación exitosa, se creará una etiqueta de lanzamiento en el mismo repositorio.

  1. Después de agregar los detalles del repositorio, haga clic en avanzado y proporcione un nombre a su repositorio que luego se mencionará en el complemento de Git Publisher para identificar el repositorio.

  1. Luego agregue el paso de compilación. En este ejemplo estoy construyendo un proyecto ANT.

  1. Ahora, en la sección “Acciones posteriores a la compilación”, seleccione el complemento “Flexi Publish”. Seleccione el valor “Y” del menú desplegable para Acción condicional (¿Ejecutar?). A continuación, seleccione “Coincidencia de cadenas” en el menú desplegable para la condición de ejecución (&&).

  1. Después de seleccionar la coincidencia de cadena, especifique $PUSH como valor de Cadena 1 y SÍ como valor de Cadena 2. Entonces, cuando ejecute la compilación si elige el valor de PUSH como SÍ, comparará la Cadena 1 (= $ PUSH) y la Cadena 2 (= SÍ) y activará la operación Git push y si elige NO, no lo hará. activar la operación de inserción de Git.

    Choose the value of PUSH -> YES OR NO -> Chosen value “YES” then, $PUSH = YES AS String 1 = $PUSH => String 1 = YES Again, String 2 = YES, hence String 2 == String 1 (String match) Then, trigger the Git push action.

  1. Ahora haga clic en la opción desplegable Agregar para agregar la acción del editor de Git que se activará en función de la condición de coincidencia de cadena.

Después de la configuración, guarde el trabajo y listo.
