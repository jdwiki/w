Estado de la entidad gestora

  • Estado de configuración Agregado de una sola entidad
  • Estado de configuración Agregado de un gráfico de objeto
  • Ejemplo

En esta página

  • Estado de configuración Agregado de una sola entidad
  • Estado de configuración Agregado de un gráfico de objeto
  • Ejemplo

Las entidades en Entity Framework pueden tener varios estados enumerados por [System.Data.Entity.EntityState](https://msdn.microsoft.com/en-us/library/system.data.entity.entitystate(v=vs .113).aspx) enumeración. Estos estados son:

Added    
Deleted    
Detached    
Modified    
Unchanged

Entity Framework funciona con POCO. Eso significa que las entidades son clases simples que no tienen propiedades ni métodos para administrar su propio estado. El estado de la entidad es administrado por un contexto en sí mismo, en ObjectStateManager.

Este tema cubre varias formas de establecer el estado de la entidad.

Estado de configuración Agregado de una sola entidad #

EntityState.Added se puede configurar de dos formas completamente equivalentes:

  1. Fijando el estado de su entrada en el contexto:

     context.Entry(entity).State = EntityState.Added;

  2. Al agregarlo a un DbSet del contexto:

     context.Entities.Add(entity);

Al llamar a SaveChanges, la entidad se insertará en la base de datos. Cuando tiene una columna de identidad (una clave principal autoestablecida e incremental automática), luego de SaveChanges, la propiedad de la clave principal de la entidad contendrá el valor recién generado, incluso cuando esta propiedad ya tenía un valor.

Estado de configuración Agregado de un gráfico de objeto #

Establecer el estado de un objeto gráfico (una colección de entidades relacionadas) en “Agregado” es diferente a establecer una sola entidad como “Agregado” (ver [este ejemplo](https://www.wikiod.com/es/docs/entity- framework/5256/management-entity-state/18713/setting-state-added-of-a-single-entity#t=201608061947239551134)).

En el ejemplo, almacenamos planetas y sus lunas:

Modelo de clase

public class Planet
{
    public Planet()
    {
        Moons = new HashSet<Moon>();
    }
    public int ID { get; set; }
    public string Name { get; set; }
    public ICollection<Moon> Moons { get; set; }
}

public class Moon
{
    public int ID { get; set; }
    public int PlanetID { get; set; }
    public string Name { get; set; }
}

Contexto

public class PlanetDb : DbContext
{
    public property DbSet<Planet> Planets { get; set; }
}

Usamos una instancia de este contexto para agregar planetas y sus lunas:

Ejemplo #

var mars = new Planet { Name = "Mars" };
mars.Moons.Add(new Moon { Name = "Phobos" });
mars.Moons.Add(new Moon { Name = "Deimos" });

context.Planets.Add(mars);

Console.WriteLine(context.Entry(mars).State);
Console.WriteLine(context.Entry(mars.Moons.First()).State);

Producción:

Added
Added

Lo que vemos aquí es que agregar un ‘Planeta’ también establece el estado de una luna en ‘Agregado’.

Al establecer el estado de una entidad en Agregado, todas las entidades en sus propiedades de navegación (propiedades que “navegan” a otras entidades, como Planet.Moons) también se marcan como Agregado, *a menos que ya estén adjuntas al contexto *.
