Entity Framework Code-First proporciona un conjunto de atributos de anotación de datos, que puede aplicar a las clases y propiedades de su dominio. Los atributos de DataAnnotation anulan las convenciones Code-First predeterminadas.

  1. System.ComponentModel.DataAnnotations incluye atributos que afectan la nulabilidad o el tamaño de la columna.
  2. El espacio de nombres System.ComponentModel.DataAnnotations.Schema incluye atributos que afectan el esquema de la base de datos.

Nota: Las anotaciones de datos solo le brindan un subconjunto de opciones de configuración. Fluent API proporciona un conjunto completo de opciones de configuración disponibles en Code-First.

[Columna] atributo #

public class Person
{
    public int PersonID { get; set; }
    
    [Column("NameOfPerson")]
    public string PersonName { get; set; }    
}

Le dice a Entity Framework que use un nombre de columna específico en lugar de usar el nombre de la propiedad. También puede especificar el tipo de datos de la base de datos y el orden de la columna en la tabla:

[Column("NameOfPerson", TypeName = "varchar", Order = 1)]
public string PersonName { get; set; }

[Obligatorio] atributo #

Cuando se aplica a una propiedad de una clase de dominio, la base de datos creará una columna NOT NULL.

using System.ComponentModel.DataAnnotations;

public class Person
{
    public int PersonID { get; set; }
    
    [Required]
    public string PersonName { get; set; }    
}

Atributos [MaxLength] y [MinLength] #

El atributo [MaxLength(int)] se puede aplicar a una propiedad de tipo cadena o matriz de una clase de dominio. Entity Framework establecerá el tamaño de una columna en el valor especificado.

using System.ComponentModel.DataAnnotations;

public class Person
{
    public int PersonID { get; set; }
    
    [MinLength(3), MaxLength(100)]
    public string PersonName { get; set; }    
}

La columna resultante con la longitud de columna especificada:

El atributo [MinLength(int)] es un atributo de validación, no afecta la estructura de la base de datos. Si intentamos insertar/actualizar una Persona con PersonName con una longitud inferior a 3 caracteres, esta confirmación fallará. Obtendremos una DbUpdateConcurrencyException que tendremos que manejar.

using (var db = new ApplicationDbContext())
{                    
     db.Staff.Add(new Person() { PersonName = "ng" });
     try
     {
          db.SaveChanges();
     }
     catch (DbEntityValidationException ex)
     {
          //ErrorMessage = "The field PersonName must be a string or array type with a minimum length of '3'."
     }
}

Los atributos [MaxLength] y [MinLength] también se pueden usar con asp.net-mvc como atributo de validación.

[Base de datos generada] atributo #

Especifica cómo la base de datos genera valores para la propiedad. Hay tres valores posibles:

  1. Ninguno especifica que la base de datos no genera los valores.
  2. Identidad especifica que la columna es una columna de identidad, que normalmente se usa para claves primarias enteras.
  3. Computed especifica que la base de datos genera el valor para la columna.

Si el valor es distinto de Ninguno, Entity Framework no confirmará los cambios realizados en la propiedad en la base de datos.

De forma predeterminada (según StoreGeneratedIdentityKeyConvention), una propiedad de clave entera se tratará como una columna de identidad. Para anular esta convención y forzar que se trate como una columna sin identidad, puede usar el atributo DatabaseGenerated con un valor de Ninguno.

using System.ComponentModel.DataAnnotations.Schema;

public class Foo
{
    [Key]
    public int Id { get; set; } // identity (auto-increment) column
}

public class Bar
{
    [Key]
    [DatabaseGenerated(DatabaseGeneratedOption.None)]
    public int Id { get; set; } // non-identity column
}

El siguiente SQL crea una tabla con una columna calculada:

CREATE TABLE [Person] (
    Name varchar(100) PRIMARY KEY,
    DateOfBirth Date NOT NULL,
    Age AS DATEDIFF(year, DateOfBirth, GETDATE())
)
GO

Para crear una entidad para representar los registros en la tabla anterior, necesitaría usar el atributo DatabaseGenerated con un valor de Computed.

[Table("Person")]
public class Person
{
    [Key, StringLength(100)]
    public string Name { get; set; }
    public DateTime DateOfBirth { get; set; }
    [DatabaseGenerated(DatabaseGeneratedOption.Computed)]
    public int Age { get; set; }
}

Atributo [ForeignKey(cadena)] #

Especifica el nombre de la clave externa personalizada si se desea una clave externa que no siga la convención de EF.

public class Person 
{
    public int IdAddress { get; set; }

    [ForeignKey(nameof(IdAddress))]
    public virtual Address HomeAddress { get; set; }
}

Esto también se puede usar cuando tiene varias relaciones con el mismo tipo de entidad.

using System.ComponentModel.DataAnnotations.Schema;

public class Customer
{
    ...

    public int MailingAddressID { get; set; }
    public int BillingAddressID { get; set; }

    [ForeignKey("MailingAddressID")]
    public virtual Address MailingAddress { get; set; }
    [ForeignKey("BillingAddressID")]
    public virtual Address BillingAddress { get; set; }
}

Sin los atributos ForeignKey, EF podría confundirlos y usar el valor de BillingAddressID al obtener MailingAddress, o simplemente podría generar un nombre diferente para la columna según sus propias convenciones de nomenclatura (como Address_MailingAddress_Id) e intente usar eso en su lugar (lo que resultaría en un error si está usando esto con una base de datos existente).

[PropiedadInversa(cadena)] atributo #

using System.ComponentModel.DataAnnotations.Schema;

public class Department  
{  
    ...

    public virtual ICollection<Employee> PrimaryEmployees { get; set; }  
    public virtual ICollection<Employee> SecondaryEmployees { get; set; }  
}  
  
public class Employee  
{  
    ...

    [InverseProperty("PrimaryEmployees")]  
    public virtual Department PrimaryDepartment { get; set; }  
  
    [InverseProperty("SecondaryEmployees")]  
    public virtual Department SecondaryDepartment { get; set; }  
}

InverseProperty se puede utilizar para identificar relaciones bidireccionales cuando existen relaciones múltiples bidireccionales entre dos entidades.

Le dice a Entity Framework qué propiedades de navegación debe hacer coincidir con las propiedades del otro lado.

Entity Framework no sabe qué mapa de propiedad de navegación con qué propiedades en el otro lado cuando existen múltiples relaciones bidireccionales entre dos entidades.

Necesita el nombre de la propiedad de navegación correspondiente en la clase relacionada como su parámetro.

Esto también se puede usar para entidades que tienen una relación con otras entidades del mismo tipo, formando una relación recursiva.

using System.ComponentModel.DataAnnotations;
using System.ComponentModel.DataAnnotations.Schema;

public class TreeNode
{
    [Key]
    public int ID { get; set; }
    public int ParentID { get; set; }

    ...

    [ForeignKey("ParentID")]
    public TreeNode ParentNode { get; set; }
    [InverseProperty("ParentNode")]
    public virtual ICollection<TreeNode> ChildNodes { get; set; }
}

Tenga en cuenta también el uso del atributo ForeignKey para especificar la columna que se usa para la clave externa en la tabla. En el primer ejemplo, las dos propiedades en la clase Empleado podrían haber tenido aplicado el atributo ForeignKey para definir los nombres de las columnas.

Atributo [TipoComplejo] #

using System.ComponentModel.DataAnnotations.Schema;

[ComplexType] 
public class BlogDetails 
{ 
    public DateTime? DateCreated { get; set; } 
 
    [MaxLength(250)] 
    public string Description { get; set; } 
}

public class Blog
{
    ...

    public BlogDetails BlogDetail { get; set; }
}

Marque la clase como tipo complejo en Entity Framework.

Los tipos complejos (u objetos de valor en el diseño basado en dominios) no se pueden rastrear por sí solos, pero se rastrean como parte de una entidad. Esta es la razón por la que BlogDetails en el ejemplo no tiene una propiedad clave.

Pueden ser útiles cuando se describen entidades de dominio en varias clases y se superponen esas clases en una entidad completa.

[Clave] atributo #

La clave es un campo en una tabla que identifica de manera única cada fila/registro en una tabla de base de datos.

Utilice este atributo para anular la convención Code-First predeterminada. Si se aplica a una propiedad, se usará como columna de clave principal para esta clase.

using System.ComponentModel.DataAnnotations;

public class Person
{
    [Key]
    public int PersonKey { get; set; }        // <- will be used as primary key
     
    public string PersonName { get; set; }    
}

Si se requiere una clave principal compuesta, el atributo [Clave] también se puede agregar a varias propiedades. El orden de las columnas dentro de la clave compuesta se debe proporcionar en la forma [Clave, Columna(Orden = x)].

using System.ComponentModel.DataAnnotations;

public class Person
{
    [Key, Column(Order = 0)]
    public int PersonKey1 { get; set; }    // <- will be used as part of the primary key

    [Key, Column(Order = 1)]
    public int PersonKey2 { get; set; }    // <- will be used as part of the primary key
     
    public string PersonName { get; set; }    
}

Sin el atributo [Clave], EntityFramework recurrirá a la convención predeterminada que es usar la propiedad de la clase como una clave principal denominada “Id” o “{ClassName}Id”.

public class Person
{
    public int PersonID { get; set; }        // <- will be used as primary key
     
    public string PersonName { get; set; }    
}

[Rango (mín., máx.)] atributo #

Especifica un rango numérico mínimo y máximo para una propiedad

using System.ComponentModel.DataAnnotations;   

public partial class Enrollment
{
    public int EnrollmentID { get; set; }
   
    [Range(0, 4)]
    public Nullable<decimal> Grade { get; set; }
}

Si intentamos insertar/actualizar un Grado con un valor fuera de rango, esta confirmación fallará. Obtendremos una DbUpdateConcurrencyException que tendremos que manejar.

using (var db = new ApplicationDbContext())
{
    db.Enrollments.Add(new Enrollment() { Grade = 1000 });

    try
    {
        db.SaveChanges();
    }
    catch (DbEntityValidationException ex)
    {
        // Validation failed for one or more entities
    }
}

Atributo [No asignado] #

Según la convención de Code-First, Entity Framework crea una columna para cada propiedad pública que sea de un tipo de datos compatible y tenga un captador y un definidor. La anotación [NotMapped] se debe aplicar a cualquier propiedad para la que NO queremos una columna en una tabla de base de datos.

Un ejemplo de una propiedad que no querríamos almacenar en la base de datos es el nombre completo de un estudiante basado en su nombre y apellido. Eso se puede calcular sobre la marcha y no hay necesidad de almacenarlo en la base de datos.

public string FullName => string.Format("{0} {1}", FirstName, LastName);

La propiedad “FullName” solo tiene un getter y no un setter, por lo que, de forma predeterminada, Entity Framework NO creará una columna para ella.

Otro ejemplo de una propiedad que quizás no queramos almacenar en la base de datos es la “Calificación promedio” de un estudiante. No queremos obtener el promedio de calificaciones a pedido; en cambio, podríamos tener una rutina en otro lugar que lo calcule.

[NotMapped]
public float AverageGrade { set; get; }

El “AverageGrade” debe estar marcado con la anotación [NotMapped], de lo contrario, Entity Framework creará una columna para él.

using System.ComponentModel.DataAnnotations.Schema;

public class Student
{
    public int Id { set; get; }

    public string FirstName { set; get; }

    public string LastName { set; get; }

    public string FullName => string.Format("{0} {1}", FirstName, LastName);

    [NotMapped]
    public float AverageGrade { set; get; }
}

Para la Entidad anterior, veremos dentro de DbMigration.cs

CreateTable(
    "dbo.Students",
     c => new
         {
             Id = c.Int(nullable: false, identity: true),
             FirstName = c.String(),
             LastName = c.String(),
         })
     .PrimaryKey(t => t.Id);

y en SQL Server Management Studio

[Tabla] atributo #

[Table("People")]
public class Person
{
    public int PersonID { get; set; }
    public string PersonName { get; set; }    
}

Le dice a Entity Framework que use un nombre de tabla específico en lugar de generar uno (es decir, ‘Persona’ o ‘Personas’)

También podemos especificar un esquema para la tabla usando el atributo [Table]

[Table("People", Schema = "domain")]

[Índice] atributo #

public class Person
{
    public int PersonID { get; set; }
    public string PersonName { get; set; }

    [Index]
    public int Age { get; set; }
}

Crea un índice de base de datos para una columna o conjunto de columnas.

[Index("IX_Person_Age")]
public int Age { get; set; }

Esto crea un índice con un nombre específico.

[Index(IsUnique = true)]
public int Age { get; set; }

Esto crea un índice único.

[Index("IX_Person_NameAndAge", 1)]
public int Age { get; set; }

[Index("IX_Person_NameAndAge", 2)]
public string PersonName { get; set; }

Esto crea un índice compuesto usando 2 columnas. Para hacer esto, debe especificar el mismo nombre de índice y proporcionar un orden de columnas.

Nota: El atributo Índice se introdujo en Entity Framework 6.1. Si está utilizando una versión anterior, la información de esta sección no se aplica.

[StringLength(int)] atributo #

using System.ComponentModel.DataAnnotations;

public class Post
{
    public int Id { get; set; }
    
    [StringLength(100)]
    public string Title { get; set;}

    [StringLength(300)]
    public string Abstract { get; set; }
    
    public string Description { get; set; }
}

Define una longitud máxima para un campo de cadena.

Nota: También se puede usar con asp.net-mvc como atributo de validación.

Atributo [marca de tiempo] #

El atributo [TimeStamp] se puede aplicar solo a una propiedad de matriz de bytes en una clase de Entidad determinada. Entity Framework creará una columna de marca de tiempo no anulable en la tabla de la base de datos para esa propiedad. Entity Framework usará automáticamente esta columna TimeStamp en la verificación de simultaneidad.

using System.ComponentModel.DataAnnotations.Schema;

public class Student
{
    public int Id { set; get; }

    public string FirstName { set; get; }

    public string LastName { set; get; }    

    [Timestamp]
    public byte[] RowVersion { get; set; }
}

[Comprobación de concurrencia] Atributo #

Este atributo se aplica a la propiedad de clase. Puede usar el atributo ConcurrencyCheck cuando desee usar columnas existentes para verificar la concurrencia y no una columna de marca de tiempo separada para la concurrencia.

using System.ComponentModel.DataAnnotations;

public class Author
{
    public int AuthorId { get; set; }
     
    [ConcurrencyCheck]
    public string AuthorName { get; set; }
}

Del ejemplo anterior, el atributo ConcurrencyCheck se aplica a la propiedad AuthorName de la clase Author. Por lo tanto, Code-First incluirá la columna AuthorName en el comando de actualización (cláusula where) para verificar la concurrencia optimista.
