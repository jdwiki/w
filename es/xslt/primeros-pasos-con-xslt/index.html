Instalación o configuración #

XSLT es un lenguaje de programación de propósito especial; se usa ampliamente para transformar documentos XML a un formato XML diferente, a HTML oa formatos basados ​​en texto.

Hay dos versiones principales de XSLT en uso: XSLT 1.0 y XSLT 2.0. XSLT 1.0 está más ampliamente implementado pero tiene muchas restricciones y limitaciones en comparación con XSLT 2.0; tendrá que decidir qué versión usar. Si un procesador XSLT 2.0 está disponible para su entorno elegido, entonces esta es casi siempre la mejor opción.

(XSLT 1.0 salió en noviembre de 1999, y muchas implementaciones aparecieron en uno o dos años, tanto de proveedores principales como Microsoft, IBM, Sun y Oracle, como de entusiastas individuales que trabajaban en su tiempo libre. XSLT 2.0 salió en enero 2007, y muchos de los procesadores originales 1.0 nunca se actualizaron porque sus desarrolladores habían perdido interés. El procesador XSLT 2.0 más utilizado es Saxon, pero no tiene el campo para sí mismo: otros productos independientes incluyen RaptorXML (de Altova ), XmlPrime (de CBCL) y Exselt, y los procesadores 2.0 también están disponibles integrados en WebLogic de IBM, en el servidor de base de datos XML de MarkLogic y en XML Accelerator de Intel).

La especificación de XSLT 3.0 está técnicamente completa (recomendación propuesta en abril de 2017), pero por el momento es mejor considerarla como innovadora: considérela solo si tiene una necesidad apremiante de sus nuevas características, como transmisión, paquetes, compatibilidad con JSON. , o intentar/atrapar. Hay tres implementaciones conocidas: Saxon, Exselt y RaptorXML.

Para empezar con XSLT tienes varias opciones:

  • Use una herramienta XSLT en línea. Hay varios disponibles (busque “herramienta XSLT en línea”), un IDE en línea muy confiable es http://xsltransform.net/. Esta es una buena manera de tener una idea inicial del lenguaje, pero no querrá continuar de esta manera una vez que esté haciendo un desarrollo real.

  • Use el motor XSLT integrado en cada navegador. Al igual que las herramientas online, estas tienen la ventaja de que no necesitas instalar nada; pero los navegadores solo admiten XSLT 1.0, solo admiten la conversión de XML a HTML y tienen un soporte de depuración muy débil. Un desarrollo muy reciente, pero aún experimental, es Saxon-JS, que permite la ejecución de XSLT 2.0 (y partes de XSLT 3.0) en el navegador.

  • Instale un procesador XSLT (como Saxon o xsltproc). La mayoría de estos productos se pueden invocar mediante la línea de comandos del sistema operativo o mediante una API para lenguajes de programación específicos, como Java, C, C# o Python.

  • Instale un entorno de desarrollo XML como Altova XML Spy, oXygen de SyncroSoft o Stylus Studio. Aunque esta es una opción más costosa, brinda un soporte de desarrollo y una capacidad de depuración mucho más ricos.

Cualquiera que sea la opción que elija, primero debe decidir si desea usar XSLT 1.0 o XSLT 2.0.

Ejemplo sencillo de XSLT #

Aquí hay un ejemplo simple que usa XSLT para convertir datos en un archivo XML en una tabla en un archivo HTML. Puede usarlo para experimentar con transformaciones XSLT simples.

Requisito previo: Instale un Java Runtime Environment y agregue la ubicación del JRE a su variable PATH. (En Windows, la mayoría de los instaladores agregarán Java a su ruta por usted). Si esto funciona, debería poder abrir una ventana de línea de comandos y ejecutar el comando java -version y obtener una copia impresa de la información sobre su JRE.

  1. Descargue el procesador Saxon-HE XSLT para Java aquí: saxon.sourceforge.net y descomprímalo en cualquier lugar de su computadora.
  2. En un editor de texto, cree un archivo llamado pets.xml con el siguiente código:
  <pets>
  <petType name="Dogs">
    <pet id="123" name="Sparky" vaccineStatus="vaccinated" healthStatus="healthy"/>
    <pet id="234" name="Sadie" vaccineStatus="unvaccinated" healthStatus="sick"/>
    <pet id="345" name="Herman" vaccineStatus="unvaccinated" healthStatus="unknown"/>
  </petType>
  <petType name="Cats">
    <pet id="456" name="Cleo" vaccineStatus="vaccinated" healthStatus="healthy"/>
    <pet id="567" name="Janet" vaccineStatus="unvaccinated" healthStatus="healthy"/>
    <pet id="678" name="Furry" vaccineStatus="vaccinated" healthStatus="sick"/>
  </petType>
</pets>
  1. In a text editor, create a file named petTransform.xsl with the following code:
<xsl:stylesheet xmlns:xsl="http://www.w3.org/1999/XSL/Transform"
  version="2.0">

<!-- handle the root XML element -->
<xsl:template match="/">
<html><head>
  <title>Pets that are available for adoption</title>
</head>
<body>
  <xsl:apply-templates/>
</body>
</html>
</xsl:template>

<xsl:template match="pets">
<xsl:apply-templates/>
</xsl:template>

<xsl:template match="petType">
<h2><xsl:value-of select="@name"/></h2>
<table id="{@name}">
   <tr>
     <th colname="id">ID</th>
     <th colname="name">Name</th>
     <th colname="vaccinated">Vaccine status</th>
     <th colname="health">Health status</th>
   </tr>
   <tbody>
     <!-- add a row for each pet in this category -->
     <xsl:for-each select="pet">
       <tr>
         <td colname="id"><xsl:value-of select="@id"/></td>
         <td colname="name"><xsl:value-of select="@name"/></td>
         <td colname="vaccinated"><xsl:value-of select="@vaccineStatus"/></td>
         <td colname="health"><xsl:value-of select="@healthStatus"/></td>
       </tr>
     </xsl:for-each>
   </tbody>
 </table>
</xsl:template>

<!-- ignore the content of other tags because we processed them elsewhere -->
<xsl:template match="*">
<!-- do nothing -->
</xsl:template>

</xsl:stylesheet>
  1. Abra una ventana de línea de comandos y vaya a la carpeta con los archivos XML y XSLT.
  2. Ejecute el siguiente comando, donde path_to_saxon.jar es la ruta completa al archivo saxon9he.jar:
java -jar "path_to_saxon.jar" -o
petOutput.html -s:pets.xml -xsl:pettransform.xslt

Por ejemplo:

java -jar "C:\Program Files\SaxonHE9-7-0-7J\saxon9he.jar" -o
petOutput.html -s:pets.xml -xsl:pettransform.xslt

Asegúrese de ejecutar este comando en una sola línea. 6. Abra el archivo de salida petOutput.html en un editor de texto. Debe tener un aspecto como este:
