Crear explosiones #

Para crear una explosión, se pueden usar las siguientes firmas de métodos:

boolean createExplosion(double x, double y, double z, float power);
boolean createExplosion(double x, double y, double z, float power, boolean setFire);
boolean createExplosion(double x, double y, double z, float power,
                        boolean setFire, boolean breakBlocks);
boolean createExplosion(Location loc, float power);
boolean createExplosion(Location loc, float power, boolean setFire);
  • x, y, z y loc representan la ubicación donde desea que ocurra la explosión.
  • el poder representa el poder de tu explosión, el poder TnT es 4F.
  • setFire representa la capacidad de la explosión para incendiar bloques
  • breakBlocks representa la capacidad de la explosión para destruir bloques a su alrededor.
  • todos los métodos devuelven verdadero si ocurrió la explosión y devuelven falso si un complemento canceló el evento de explosión.

Simulando una explosión de TnT que rompe bloques y prende fuego en x=0, y=0 y z=0

createExplosion(0.0, 0.0, 0.0, 4F, true, true);

Dejar caer un artículo #

Los siguientes métodos se pueden utilizar para soltar un elemento en algún lugar del mundo:

Item dropItem(Location loc, ItemStack is);
Item dropItemNaturally(Location loc, ItemStack is);

dropItem significa soltar un elemento exactamente en la ubicación, devolviendo un objeto de elemento.

dropItemNaturally significa soltar el elemento en la ubicación, pero con un desplazamiento aleatorio, lo que significa que no estará exactamente en la ubicación, sino muy cerca. Esto se hace para simular que una entidad o un bloque, como un Dispensador, deja caer un artículo.

Generación de un árbol #

Los siguientes métodos se pueden utilizar para generar un árbol de forma natural (como si hubiera crecido a partir de un retoño) en el mundo.

boolean generateTree(Location loc, TreeType type);
boolean generateTree(Location loc, TreeType type, BlockChangeDelegate delegate);
  • La ubicación es donde quieres que aparezca el árbol.
  • TreeType es el tipo de árbol que desea generar y puede ser uno de los siguientes

Enumeración TreeType

Tipo Descripción
ACACIA Acacia tree
ÁRBOL_GRANDE Árbol regular, extra alto con ramas
ABEDUL abedul
SETA_BROWN Gran hongo marrón; alto y con forma de paraguas
CHORUS_PLANT Gran planta nativa de The End
CACAO_ÁRBOL Árbol de la selva con plantas de cacao; 1 cuadra de ancho
ROBLE_OSCURO Árbol de roble oscuro.
JUNGLA Árbol de la selva estándar; 4 cuadras de ancho y alto
JUNGLA_BUSH Pequeño arbusto que crece en la selva
MEGA_REDWOOD Árbol de secoya mega; 4 cuadras de ancho y alto
SETA_ROJA Hongo rojo grande; bajo y gordo
SECOYA Árbol secoya, con forma de pino
PEQUEÑA_SELVA Árbol de la selva más pequeño; 1 cuadra de ancho
PANTANO Árbol de pantano (regular con enredaderas a un lado)
ALTO_ABEDUL Abedul alto
ALTO_REDWOD Árbol secoya alto con solo unas pocas hojas en la parte superior
ÁRBOL Árbol normal, sin ramas
  • se puede usar el delegado si desea que una clase llame para cada bloque cambiado como resultado de este método

Ambas firmas devolverán verdadero si el árbol se generó con éxito, falso en caso contrario.

Reglas de desove #

Hay algunas reglas de generación en Worlds in Bukkit. Están:

  • Desove de animales
  • Desove de criaturas
  • Cantidad de lo anterior que se puede generar

** Desove de animales **

El desove de los animales se puede dividir en las siguientes categorías:

  • Animales acuáticos
  • Animales terrestres

Para obtener la cantidad de animales que se pueden generar dentro del mundo en tiempo de ejecución, puede usar el método

int getAnimalSpawnLimit()

Para animales terrestres y

int getWaterAnimalSpawnLimit();

Para animales acuáticos.

Ambos límites se pueden establecer con los métodos

void setAnimalSpawnLimit(int limit);
void setWaterAnimalSpawnLimit(int limit);

Nota: Si se establece en números inferiores a 0, en su lugar se utilizará la cantidad predeterminada del mundo.

Minecraft intenta generar animales cada 400 tics (predeterminado). Eso se puede cambiar si lo desea, utilizando las siguientes firmas:

void setTicksPerAnimalSpawns(int ticks);
void setTicksPerWaterAnimalSpawns(int ticks);
  • Un valor de 1 significará que el servidor intentará generar animales en este mundo cada tic.
  • Un valor de 400 significará que el servidor intentará generar animales en este mundo cada 400 ticks.
  • Un valor por debajo de 0 se restablecerá al valor predeterminado de Minecraft.

Nota: si se establece en 0, el desove de animales se desactivará para este mundo. Se recomienda usar setSpawnFlags(boolean, boolean) para controlar esto.
