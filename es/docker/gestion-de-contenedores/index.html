Sintaxis #

  • docker rm [OPCIONES] CONTENEDOR [CONTENEDOR…]
  • Docker adjuntar [OPCIONES] CONTENEDOR
  • docker exec [OPCIONES] COMANDO CONTENEDOR [ARG…]
  • ventana acoplable ps [OPCIONES]
  • registros docker [OPCIONES] CONTENEDOR
  • ventana acoplable inspeccionar [OPCIONES] CONTENEDOR|IMAGEN [CONTENEDOR|IMAGEN…]
  • En los ejemplos anteriores, cada vez que contenedor es un parámetro del comando docker, se menciona como <contenedor> o container id o <CONTAINER_NAME>. En todos estos lugares, puede pasar un nombre de contenedor o una identificación de contenedor para especificar un contenedor.

Listado de contenedores #

$ docker ps
CONTAINER ID        IMAGE               COMMAND                  CREATED             STATUS              PORTS                     NAMES
2bc9b1988080        redis               "docker-entrypoint.sh"   2 weeks ago         Up 2 hours          0.0.0.0:6379->6379/tcp    elephant-redis
817879be2230        postgres            "/docker-entrypoint.s"   2 weeks ago         Up 2 hours          0.0.0.0:65432->5432/tcp   pt-postgres

docker ps por sí solo solo imprime contenedores actualmente en ejecución. Para ver todos los contenedores (incluidos los detenidos), use el indicador -a:

$ docker ps -a
CONTAINER ID        IMAGE               COMMAND                  CREATED             STATUS                    PORTS                     NAMES
9cc69f11a0f7        docker/whalesay     "ls /"                   26 hours ago        Exited (0) 26 hours ago                             berserk_wozniak
2bc9b1988080        redis               "docker-entrypoint.sh"   2 weeks ago         Up 2 hours                0.0.0.0:6379->6379/tcp    elephant-redis
817879be2230        postgres            "/docker-entrypoint.s"   2 weeks ago         Up 2 hours                0.0.0.0:65432->5432/tcp   pt-postgres

Para listar contenedores con un estado específico, use la opción de línea de comando -f para filtrar los resultados. Aquí hay un ejemplo de una lista de todos los contenedores que han salido:

$ docker ps -a -f status=exited
CONTAINER ID        IMAGE               COMMAND                  CREATED             STATUS                    PORTS                     NAMES
9cc69f11a0f7        docker/whalesay     "ls /"                   26 hours ago        Exited (0) 26 hours ago

También es posible enumerar solo los ID de contenedor con el interruptor -q. Esto hace que sea muy fácil operar en el resultado con otras utilidades de Unix (como grep y awk):

$ docker ps -aq
9cc69f11a0f7
2bc9b1988080
817879be2230

Al iniciar un contenedor con docker run --name mycontainer1, le da un nombre específico y no un nombre aleatorio (en la forma mood_famous, como nostalgic_stallman), y puede ser fácil encontrarlos con dicho comando.

docker ps -f nombre=micontenedor1

Eliminar, eliminar y limpiar contenedores #

docker rm se puede usar para eliminar contenedores específicos como este:

docker rm <container name or id>

Para eliminar todos los contenedores, puede usar esta expresión:

docker rm $(docker ps -qa)

De forma predeterminada, la ventana acoplable no eliminará un contenedor que se esté ejecutando. Cualquier contenedor que se esté ejecutando generará un mensaje de advertencia y no se eliminará. Todos los demás contenedores serán eliminados.

Alternativamente, puede usar xargs:

docker ps -aq -f status=exited | xargs -r docker rm

Donde docker ps -aq -f status=exited devolverá una lista de ID de contenedor de contenedores que tienen un estado de “Salido”.

Advertencia: Todos los ejemplos anteriores solo eliminarán los contenedores ‘detenidos’.

Para eliminar un contenedor, independientemente de si está detenido o no, puede usar el indicador de fuerza -f:

docker rm -f <container name or id>

Para eliminar todos los contenedores, independientemente del estado:

docker rm -f $(docker ps -qa)

Si desea eliminar solo los contenedores con un estado inactivo:

docker rm $(docker ps --all -q -f status=dead)

Si desea eliminar solo los contenedores con un estado “salido”:

docker rm $(docker ps --all -q -f status=exited)

Estas son todas las permutaciones de los filtros que se usan al [enumerar contenedores][1].

Para eliminar tanto los contenedores no deseados como las imágenes colgantes que ocupan espacio después de la [versión 1.3][2], use lo siguiente (similar a la herramienta de Unix df):

$ docker system df

Para eliminar todos los datos no utilizados:

$ docker system prune

[1]: https://www.wikiod.com/es/docker/gestion-de-contenedores#Listado de contenedores [2]: https://blog.docker.com/2017/01/whats-new-in-docker-1-13/

Contenedores de referencia #

Los comandos de Docker que toman el nombre de un contenedor aceptan tres formas diferentes:

Tipo Ejemplo
UUID completo 9cc69f11a0f76073e87f25cb6eaf0e079fbfbd1bc47c063bcd25ed3722a8cc4a
UUID corto 9cc69f11a0f7
Nombre berserk_wozniak

Use docker ps para ver estos valores para los contenedores en su sistema.

Docker genera el UUID y no se puede modificar. Puede proporcionar un nombre al contenedor cuando lo inicia docker run --name <nombre dado> <imagen>. Docker generará un nombre aleatorio para el contenedor si no especifica uno al momento de iniciar el contenedor.

NOTA: El valor del UUID (o un UUID ‘corto’) puede tener cualquier longitud, siempre que el valor dado sea único para un contenedor

Conectarse a una instancia que se ejecuta como daemon #

Hay dos formas de conseguirlo, la primera y más conocida es la siguiente:

docker attach --sig-proxy=false <container>

Este literalmente adjunta su bash al contenedor bash, lo que significa que si tiene un script en ejecución, verá el resultado.

Para desconectar, simplemente escriba: Ctl-P Ctl-Q

Pero si necesita una forma más amigable y poder crear nuevas instancias de bash, simplemente ejecute el siguiente comando:

docker exec -it <container> bash

Encontrar un contenedor específico #

docker ps --filter name=myapp_1

Copiar archivo de/a contenedores #

del contenedor al host

docker cp CONTAINER_NAME:PATH_IN_CONTAINER PATH_IN_HOST

del host al contenedor

docker cp PATH_IN_HOST CONTAINER_NAME:PATH_IN_CONTAINER

Iniciar y detener contenedores #

Para detener un contenedor en ejecución:

docker stop <container> [<container>...]

Esto enviará al proceso principal en el contenedor un SIGTERM, seguido de un SIGKILL si no se detiene dentro del período de gracia. El nombre de cada contenedor se imprime a medida que se detiene.

Para iniciar un contenedor que está detenido:

docker start <container> [<container>...]

Esto iniciará cada contenedor pasado en segundo plano; el nombre de cada contenedor se imprime como comienza. Para iniciar el contenedor en primer plano, pase el indicador -a (--attach).

Lista de contenedores con formato personalizado #

docker ps --format 'table {{.ID}}\t{{.Names}}\t{{.Status}}'

Encuentra la IP del contenedor #

Para averiguar la dirección IP de su contenedor, utilice:

docker inspect <container id> | grep IPAddress

o use la ventana acoplable para inspeccionar

docker inspect --format '{{ .NetworkSettings.IPAddress }}' ${CID}

Reiniciando el contenedor docker #

docker restart <container> [<container>...]

Opción –tiempo: Segundos para esperar a que se detenga antes de matar el contenedor (predeterminado 10)

docker restart <container> --time 10

Ejecutar comando en un contenedor docker ya existente #

docker exec -it <id del contenedor> /bin/bash

Es común iniciar sesión en un contenedor que ya se está ejecutando para realizar algunas pruebas rápidas o ver qué está haciendo la aplicación. A menudo, denota malas prácticas de uso de contenedores debido a que los registros y los archivos modificados deben colocarse en volúmenes. Este ejemplo nos permite iniciar sesión en el contenedor. Esto supone que /bin/bash está disponible en el contenedor, puede ser /bin/sh u otra cosa.

docker exec <container id> tar -czvf /tmp/backup.tgz /data
docker cp <container id>:/tmp/backup.tgz .

Este ejemplo archiva el contenido del directorio de datos en un tar. Luego, con docker cp puedes recuperarlo.

Registros de contenedores #

Usage:  docker logs [OPTIONS] CONTAINER

Fetch the logs of a container

  -f, --follow=false        Follow log output
  --help=false              Print usage
  --since=                  Show logs since timestamp
  -t, --timestamps=false    Show timestamps

Eliminar, eliminar y limpiar volúmenes de docker #

Los volúmenes de Docker no se eliminan automáticamente cuando se detiene un contenedor. Para eliminar volúmenes asociados cuando detiene un contenedor:

docker rm -v <container id or name>

Si no se especifica el indicador -v, el volumen permanece en el disco como un “volumen colgante”. Para eliminar todos los volúmenes pendientes:

docker volume rm $(docker volume ls -qf dangling=true)

El filtro docker volume ls -qf dangling=true devolverá una lista de nombres de volúmenes de docker, incluidos los no etiquetados, que no están adjuntos a un contenedor.

Alternativamente, puedes usar xargs:

docker volume ls -f dangling=true -q | xargs --no-run-if-empty docker volume rm

Exportación e importación de sistemas de archivos de contenedores Docker #

Es posible guardar el contenido del sistema de archivos de un contenedor Docker en un archivo tarball. Esto es útil en caso de apuro para mover sistemas de archivos de contenedor a diferentes hosts, por ejemplo, si un contenedor de base de datos tiene cambios importantes y de otra manera no es posible replicar esos cambios en otro lugar. Tenga en cuenta que es preferible crear un contenedor completamente nuevo a partir de una imagen actualizada usando un comando docker run o un archivo docker-compose.yml, en lugar de exportar y mover el sistema de archivos de un contenedor. Parte del poder de Docker es la auditabilidad y responsabilidad de su estilo declarativo de creación de imágenes y contenedores. Al usar docker export y docker import, esta potencia se reduce debido a la ofuscación de los cambios realizados dentro del sistema de archivos de un contenedor desde su estado original.

docker export -o redis.tar redis

El comando anterior creará una imagen vacía y luego exportará el sistema de archivos del contenedor redis a esta imagen vacía. Para importar desde un archivo tarball, use:

docker import ./redis.tar redis-imported:3.0.7

Este comando creará la imagen redis-imported:3.0.7, a partir de la cual se pueden crear contenedores. También es posible crear cambios en la importación, así como establecer un mensaje de confirmación:

docker import -c="ENV DEBUG true" -m="enable debug mode" ./redis.tar redis-changed

Las directivas de Dockerfile disponibles para usar con la opción de línea de comando -c son CMD, ENTRYPOINT, ENV, EXPOSE, ONBUILD, USER, VOLUME, WORKDIR.
