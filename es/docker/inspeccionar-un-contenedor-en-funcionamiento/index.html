Sintaxis #

Obtener información específica de un contenedor #

Puede obtener información específica de un contenedor ejecutando:

docker inspect -f '<format>' <container>

Por ejemplo, puede obtener la configuración de red ejecutando:

docker inspect -f '{{ .NetworkSettings }}' <container>

También puede obtener solo la dirección IP:

docker inspect -f '{{ .NetworkSettings.IPAddress }}' <container>

El parámetro -f significa formato y recibirá una plantilla de Go como entrada para formatear lo que se espera, pero esto no traerá un retorno hermoso, así que intente:

docker inspect -f '{{ json .NetworkSettings }}' {{containerIdOrName}}

la palabra clave json traerá el retorno como JSON.

Entonces, para terminar, un pequeño consejo es usar python allí para formatear el JSON de salida:

docker inspect -f '{{ json .NetworkSettings }}' <container> | python -mjson.tool

Y listo, puede consultar cualquier cosa en la inspección de la ventana acoplable y hacer que se vea bonito en su terminal.

También es posible usar una utilidad llamada “jq” para ayudar a procesar la salida del comando docker inspect.

docker inspect -f '{{ json .NetworkSettings }}' aa1 | jq [.Gateway]

El comando anterior devolverá el siguiente resultado:

[
  "172.17.0.1"
]

Esta salida es en realidad una lista que contiene un elemento. A veces, docker inspect muestra una lista de varios elementos y es posible que desee consultar un elemento específico. Por ejemplo, si Config.Env contiene varios elementos, puede referirse al primer elemento de esta lista usando index:

docker inspect --format '{{ index (index .Config.Env) 0 }}' <container>

El primer elemento está indexado en cero, lo que significa que el segundo elemento de esta lista está en el índice 1:

docker inspect --format '{{ index (index .Config.Env) 1 }}' <container>

Usando len es posible obtener el número de elementos de la lista:

docker inspect --format ‘{{ len .Config.Env }}’ <container>

Y usando números negativos, es posible referirse al último elemento de la lista:

docker inspect –format “{{ index .Config.Cmd $[$(docker inspect –format ‘{{ len .Config.Cmd }}’ <container>)-1]}}” <container>

Parte de la información de docker inspect viene como un diccionario de clave: valor, aquí hay un extracto de un docker inspect de un contenedor en ejecución de jess/spotify

"Configuración": { "Hostname": "8255f4804dde", "Domainname": "", "User": "spotify", "AttachStdin": false, "AttachStdout": false, "AttachStderr": false, "Tty": false, "OpenStdin": false, "StdinOnce": false, "Env": [ "DISPLAY=unix:0", "PATH=/usr/local/sbin:/usr/local/bin:/usr/sbin:/usr/bin:/sbin:/bin", "HOME=/home/spotify" ], "Cmd": [ "-stylesheet=/home/spotify/spotify-override.css" ], "Image": "jess/spotify", "Volumes": null, "WorkingDir": "/home/spotify", "Entrypoint": [ "spotify" ], "OnBuild": null, "Labels": {} },

entonces obtengo los valores de toda la sección de configuración

docker inspeccionar -f '{{.Config}}' 825

{8255f4804dde spotify false false false map[] false false false [DISPLAY=unix:0 PATH=/usr/local/sbin:/usr/local/bin:/usr/sbin:/usr/bin:/sbin:/bin HOME=/home/spotify] [-stylesheet=/home/spotify/spotify-override.css] false jess/spotify map[] /home/spotify [spotify] false [] map[] }

pero también un solo campo, como el valor de Config.Image

docker inspeccionar -f '{{índice (.Config) "Imagen" }}' 825

jess/spotify

o Config.Cmd

docker inspeccionar -f '{{.Config.Cmd}}' 825

[-hoja de estilo=/inicio/spotify/spotify-override.css]

Obtener información del contenedor #

Para obtener toda la información de un contenedor, puede ejecutar:

docker inspect <container>

Depurando los registros del contenedor usando docker inspect #

Examinando stdout/stderr de un contenedor en ejecución #

docker logs --follow <containerid>

Esto sigue la salida del contenedor en ejecución. Esto es útil si no configuró un controlador de registro en el demonio de la ventana acoplable.
