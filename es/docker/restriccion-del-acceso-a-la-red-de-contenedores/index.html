Restricción del acceso a la red de contenedores

  • Bloquear acceso a LAN y fuera
  • Bloquear el acceso a otros contenedores
  • Bloquee el acceso de los contenedores al host local que ejecuta el demonio docker
  • Bloquee el acceso de los contenedores al host local que ejecuta el demonio docker (red personalizada)

En esta página

  • Bloquear acceso a LAN y fuera
  • Bloquear el acceso a otros contenedores
  • Bloquee el acceso de los contenedores al host local que ejecuta el demonio docker
  • Bloquee el acceso de los contenedores al host local que ejecuta el demonio docker (red personalizada)

Ejemplo de redes docker que bloquean el tráfico. Úselo como red al iniciar el contenedor con --net o docker network connect.

Bloquear acceso a LAN y fuera #

docker network create -o "com.docker.network.bridge.enable_ip_masquerade"="false" lan-restricted
  • Bloques
  • LAN locales *Internet
  • No bloquea
  • Servidor que ejecuta el demonio docker (ejemplo de acceso a 10.0.1.10:22)

Bloquear el acceso a otros contenedores #

docker network create -o "com.docker.network.bridge.enable_icc"="false" icc-restricted
  • Bloques
  • Contenedores que acceden a otros contenedores en la misma red icc-restricted.
  • No bloquea
  • Acceso al host que ejecuta el demonio docker
  • LAN locales *Internet

Bloquee el acceso de los contenedores al host local que ejecuta el demonio docker #

iptables -I INPUT -i docker0 -m addrtype --dst-type LOCAL -j DROP
  • Bloques
  • Acceso al host que ejecuta el demonio docker
  • No bloquea
  • Tráfico de contenedor a contenedor
  • LAN locales *Internet
  • Redes acoplables personalizadas que no usan docker0

Bloquee el acceso de los contenedores al host local que ejecuta el demonio docker (red personalizada) #

docker network create --subnet=192.168.0.0/24 --gateway=192.168.0.1 --ip-range=192.168.0.0/25 local-host-restricted
iptables -I INPUT -s 192.168.0.0/24 -m addrtype --dst-type LOCAL -j DROP

Crea una red llamada local-host-restricted que:

  • Bloques
  • Acceso al host que ejecuta el demonio docker
  • No bloquea
  • Tráfico de contenedor a contenedor
  • LAN locales *Internet
  • Acceso procedente de otras redes docker

Las redes personalizadas tienen nombres de puente como br-15bbe9bb5bf5, por lo que usamos su subred en su lugar.
