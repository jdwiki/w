Sintaxis #

  • estadísticas docker [OPCIONES] [CONTENEDOR…]
  • registros docker [OPCIONES] CONTENEDOR
  • docker top [OPCIONES] CONTENEDOR [opciones ps]

Imprimiendo los registros #

Seguir los registros es la forma menos intrusiva de depurar una aplicación en ejecución. Este ejemplo reproduce el comportamiento del tail -f some-application.log tradicional en el contenedor 7786807d8084.

docker logs --follow --tail 10 7786807d8084

Este comando básicamente muestra la salida estándar del proceso contenedor (el proceso con pid 1).

Si sus registros no incluyen marcas de tiempo de forma nativa, puede agregar el indicador --timestamps.

Es posible mirar los registros de un contenedor detenido, ya sea

  • iniciar el contenedor fallido con docker ejecutar...; docker registra $(docker ps -lq)

  • encuentre la identificación o el nombre del contenedor con

docker ps-a

y entonces

docker registra la identificación del contenedor o

docker registra el nombre del contenedor

ya que es posible mirar los registros de un contenedor detenido

Monitoreo del uso de recursos #

Inspeccionar el uso de los recursos del sistema es una forma eficiente de encontrar aplicaciones que se comportan mal. Este ejemplo es un equivalente del comando tradicional top para contenedores:

docker stats

Para seguir las estadísticas de contenedores específicos, enumérelos en la línea de comando:

docker stats 7786807d8084 7786807d8085

Las estadísticas de Docker muestran la siguiente información:

CONTAINER     CPU %   MEM USAGE / LIMIT   MEM %    NET I/O               BLOCK I/O
7786807d8084  0.65%   1.33 GB / 3.95 GB   33.67%   142.2 MB / 57.79 MB   46.32 MB / 0 B

Por defecto, docker stats muestra la identificación de los contenedores, y esto no es muy útil, si prefiere mostrar los nombres de los contenedores, simplemente hágalo

estadísticas docker $(docker ps --format '{{.Nombres}}')

Adjuntar a un contenedor en ejecución #

‘Adjuntar a un contenedor’ es el acto de iniciar una sesión de terminal dentro del contexto en el que se está ejecutando el contenedor (y los programas que contiene). Esto se usa principalmente con fines de depuración, pero también puede ser necesario si es necesario pasar datos específicos a programas que se ejecutan dentro del contenedor.

El comando adjuntar se utiliza para hacer esto. Tiene esta sintaxis:

docker attach <container>

<contenedor> puede ser la identificación del contenedor o el nombre del contenedor. Por ejemplo:

docker attach c8a9cf1a1fa8

O:

docker attach graceful_hopper

Es posible que deba sudo los comandos anteriores, dependiendo de su usuario y de cómo esté configurado Docker.

Nota: Adjuntar solo permite adjuntar una sola sesión de shell a un contenedor a la vez.

Advertencia: todas las entradas del teclado se reenviarán al contenedor. Presionar Ctrl-c mata tu contenedor.

Para desconectarse de un contenedor adjunto, presione sucesivamente Ctrl-p y luego Ctrl-q

Para adjuntar múltiples sesiones de shell a un contenedor, o simplemente como alternativa, puede usar exec. Usando la identificación del contenedor:

docker exec -i -t c8a9cf1a1fa8 /bin/bash

Usando el nombre del contenedor:

docker exec -i -t graceful_hopper /bin/bash

exec ejecutará un programa dentro de un contenedor, en este caso /bin/bash (un shell, presumiblemente uno que tiene el contenedor). -i indica una sesión interactiva, mientras que -t asigna un pseudo-TTY.

Nota: A diferencia de adjuntar, presionar Ctrl-c solo finalizará el comando exec’d cuando se ejecuta de forma interactiva.

Entrar en un contenedor en ejecución #

Para ejecutar operaciones en un contenedor, use el comando docker exec. A veces, esto se denomina “ingresar al contenedor”, ya que todos los comandos se ejecutan dentro del contenedor.

docker exec -it container_id bash

o

docker exec -it container_id /bin/sh

Y ahora tiene un caparazón en su contenedor en ejecución. Por ejemplo, enumere los archivos en un directorio y luego abandone el contenedor:

docker exec container_id ls -la

Puede usar la -u bandera para ingresar al contenedor con un usuario específico, p. uid=1013, gid=1023.

docker exec -it -u 1013:1023 container_id ls -la

El uid y el gid no tienen que existir en el contenedor, pero el comando puede generar errores. Si desea iniciar un contenedor e ingresar inmediatamente para verificar algo, puede hacer

ejecutar ventana acoplable...; docker exec -it $(docker ps -lq) bash

el comando docker ps -lq genera solo la identificación del último contenedor iniciado (la l en -lq). (esto supone que tiene bash como intérprete disponible en su contenedor, puede tener sh o zsh o cualquier otro)

Seguimiento de procesos en un contenedor #

Inspeccionar el uso de recursos del sistema es una forma eficiente de reducir un problema en una aplicación en ejecución en vivo. Este ejemplo es un equivalente del comando tradicional ps para contenedores.

docker top 7786807d8084

Para filtrar o formatear la salida, agregue las opciones ps en la línea de comando:

docker top 7786807d8084 faux

O bien, para obtener la lista de procesos que se ejecutan como root, lo cual es una práctica potencialmente dañina:

docker top 7786807d8084 -u root

El comando docker top resulta especialmente útil cuando se solucionan problemas de contenedores minimalistas sin un shell o el comando ps.

Depuración del proceso del contenedor Docker #

Docker es solo una forma elegante de ejecutar un proceso, no una máquina virtual. Por lo tanto, la depuración de un proceso “en un contenedor” también es posible “en el host” simplemente examinando el proceso del contenedor en ejecución como un usuario con los permisos adecuados para inspeccionar esos procesos en el host (por ejemplo, raíz). Por ejemplo, es posible enumerar cada “proceso de contenedor” en el host ejecutando un ps simple como root:

sudo ps aux

Todos los contenedores de Docker que se estén ejecutando actualmente aparecerán en la salida.

Esto puede ser útil durante el desarrollo de aplicaciones para depurar un proceso que se ejecuta en un contenedor. Como usuario con los permisos adecuados, se pueden usar las utilidades de depuración típicas en el proceso del contenedor, como strace, ltrace, gdb, etc.
