Cómo extraer archivos tar.gz/bz2/tbz:

Si la extensión de su archivo es .tar.gz (o .tgz), use este comando

 tar xvzf file.tar.gz

  • x: Esto le dice a tar que extraiga los archivos.

  • v: esta opción enumerará todos los archivos uno por uno en el archivo. La “v” significa “detallado”.

  • z: La opción z es muy importante y le dice al comando tar que descomprimir el archivo (gzip).

  • f: Esta opción le dice a tar que le vas a dar un nombre de archivo trabajar con.

** Si la extensión de su archivo es .tar.bz2 (o .tbz)**

La principal diferencia entre estos dos es que la opción z ha sido reemplazada por la opción j.

  • j: Esto descomprimirá un archivo bzip2.

##!! (léase como bangbang) es un atajo para repetir el último comando ingresado en la consola. Es especialmente útil para ejecutar el comando anterior con algunos cambios.

adduser tom

adduser: Solo el root puede agregar un usuario o grupo al sistema.

Oh chasquido, ¿ahora qué? Bueno, podría volver a escribir el comando con sudo al frente o podría intentar

sudo !!

equivalente a sudo “comando anterior ingresado”

Esto es especialmente útil si el comando que acaba de escribir es especialmente largo.

También se puede usar para cambiar una parte de los comandos anteriores.

cd path/to/wrong/directory

Con

!!:s/wrong/right/

Servirá

cd path/to/right/directory

Cambio de contraseña del usuario actual #

Para cambiar la contraseña del usuario actual, simplemente escriba:

sudo passwd

Te pedirá que ingreses tu contraseña actual:

[sudo] password for <user>:

Y luego se le pedirá que ingrese una nueva contraseña:

Enter new UNIX password:

Y finalmente se le pedirá que vuelva a ingresar su contraseña:

Retype new UNIX password:

Nota: De forma predeterminada, las teclas que presiona en una solicitud de contraseña de la línea de comandos no se muestran en absoluto. Sin embargo, todavía están registrados.

Lista de archivos y carpetas #

Para listar archivos y carpetas dentro del directorio actual, usamos el comando ls:

user@host:/$ ls
bin  boot  cdrom  dev  etc  home  initrd.img  lib  lib64  lost+found  
media  mnt  opt  proc  root  run  sbin  srv  sys  tmp  usr  var  vmlinuz

ls imprime la estructura de carpetas en vista simple, codificada por color por tipo. Los colores predeterminados de Ubuntu para ls son:

azul para directorios, verde para archivos ejecutables, azul cielo para archivos vinculados, amarillo con fondo negro para dispositivos, rosa para archivos de imagen y * rojo* para archivos de almacenamiento.

Otro atajo para ls -la es ll. Sin embargo, este no es un comando incorporado. Más bien es un alias común en los sistemas ubuntu, en su totalidad es ls -laF. El alias le dará el mismo resultado que ls -la, pero con una barra inclinada adicional (/) al final de cada carpeta, para ayudarlo a identificar la carpeta más fácilmente.

El alias ll se puede ver completo escribiendo alias ll. Como se ilustra a continuación. Si el alias no está configurado, el comando dará un error.

vagrant@host ~ ->>
08:05 AM Mon Sep 12$ alias ll
 alias ll='ls -alF'

Agregando nuevo usuario #

El comando adduser agrega un usuario al sistema. Para agregar un nuevo tipo de usuario:

sudo adduser <user_name>

ejemplo:

sudo adduser tom

Después de escribir el comando anterior, se le pedirá que ingrese detalles sobre el nuevo usuario, como la nueva contraseña, el nombre completo del usuario, etc.

A continuación se muestra la información que se le pedirá al usuario que complete para agregar un nuevo usuario:

Reinicia Ubuntu #

Puede reiniciar ubuntu desde la línea de comandos. A continuación se muestra un ejemplo de reinicio de ubuntu inmediatamente.

 sudo reboot

Debe tener privilegios de sudo para usar este comando.

Otros comandos con los mismos resultados son sudo shutdown -r now y sudo init 6.

Instalar nuevo software #

APT y APT-GET #

La forma más fácil y rápida es con el comando apt-get. Este comando puede considerarse de nivel inferior y “back-end”, y es compatible con otras herramientas basadas en APT. No hay cargadores sofisticados, solo información básica de progreso. Esta es la forma más rápida de instalar aplicaciones.

Uso:

sudo apt-get install diluvio openssh-server

Este comando instalará dos nuevas aplicaciones: diluvio y openssh-server. Puede instalar tantas aplicaciones como desee en una sola línea de comandos.

La forma “elegante” del mismo proceso es con apt:

sudo apt install diluvio abre el servidor sh

El resultado es el mismo, pero la interacción con los usuarios es diferente al comando anterior. apt está diseñado para usuarios finales (humanos) y su salida puede cambiar entre versiones.

DPKG #

dpkg significa Paquete Debian. Este es un instalador de paquetes básico de bajo nivel para Debian y otros derivados de Debian. dpkg tiene muchas opciones, pero estamos interesados ​​en -i, que significa instalar.

sudo dpkg -i google-chrome-estable.deb

Puede instalar paquetes .deb con dpkg.

Si obtiene un error al instalar el paquete .deb, en la mayoría de los casos no tiene algunas dependencias. Para deshacerse de este error e instalar la aplicación correctamente, ejecute sudo apt-get -f install sin ningún otro parámetro. Esto buscará dependencias y las instalará antes de .deb. La instalación continuará y la aplicación se instalará.

Leer un archivo de texto #

Al usar Ubuntu, tiene diferentes formas de leer un archivo de texto, todas similares pero útiles en diferentes contextos.

gato

Esta es la forma más sencilla de leer un archivo de texto; simplemente genera el contenido del archivo dentro de la terminal. Tenga cuidado: si el archivo es enorme, ¡podría llevar algún tiempo completar el proceso de impresión! Si necesita detenerlo, siempre puede presionar CTRL+C. Tenga en cuenta que si necesita navegar por el documento, debe desplazarse por la salida del terminal.

cat file_name.txt

más

Una versión mejorada de gato. Si su archivo es más largo que la pantalla de la terminal, simplemente puede escribir

more file_name.txt

y tendrá una pantalla de texto que se desplaza hacia abajo, en la que puede moverse hacia abajo presionando ENTER.

menos

Este es el comando more con algunas mejoras, y normalmente es una mejor opción que cat para leer documentos medianos a grandes. Abre el archivo mostrándolos desde el principio, lo que permite desplazarse hacia arriba/abajo/derecha/izquierda mediante flechas.

less file_name.txt

Una vez que el documento está abierto, puede escribir algunos comandos para habilitar algunas funciones útiles, como:

  • q: cierra inmediatamente el archivo abierto.
  • /palabra: busca ‘palabra’ dentro del documento. Presionando n puede ir a la siguiente ocurrencia de ‘palabra’.
  • ENTER: se desplaza hacia abajo de una sola línea.
  • r: vuelve a pintar el contenido del archivo, si está cambiando durante la lectura.

Esta es la mejor opción para leer documentos medianos a grandes.

cola

Este software muestra solo la última parte del archivo. Es útil si necesita leer solo unas pocas líneas al final de un documento muy grande.

tail file_name.txt

El comando anterior mostrará las últimas 10 líneas (predeterminadas) del archivo. Para leer las últimas 3 líneas, necesitamos escribir:

tail -3 file_name.txt

Hay otro caso de uso en el que este comando es extremadamente útil. Imagina tener un documento vacío, que se va llenando mientras lo estás viendo; si desea ver nuevas líneas en tiempo real mientras se escriben en el archivo sin volver a abrirlo, simplemente abra el archivo con la opción -f. Es realmente útil si está viendo algunos registros, por ejemplo.

tail -f file_name.txt

Esta es la mejor opción para leer documentos en crecimiento.

cabeza

Este comando hace la tarea opuesta a tail. Por ejemplo, el siguiente comando mostrará las primeras 15 líneas del archivo file_name.txt.

head -15 file_name.txt

** coito **

Esta es una alternativa para tail -f filename. Sigue los cambios del archivo a medida que ocurren y le muestra el resultado.

empuje

A algunos de nosotros nos gusta vi, a otros les gusta vim. Esto no es solo para leer archivos, ¡también puedes editarlos! Ahora veamos solo algunas características relacionadas con la lectura de documentos. Tenga en cuenta que vim ofrece resaltado de sintaxis.

vim file_name.txt

Una vez abierto el archivo, ¡cuidado! ¡No empieces a escribir o lo estropearás todo! De hecho, incluso si puede ver el cursor, debe presionar i para comenzar a escribir y ESC después de terminar de escribir. Por cierto, ahora les mostraré algunos comandos útiles relacionados con la lectura (no con la escritura):

  • : : ¡necesita escribir dos puntos antes de insertar cada uno de los siguientes comandos!
  • q! : salir del archivo sin pedir confirmación. Es lo mismo que q si no editaste el texto.
  • /palabra : busca ‘palabra’ dentro del documento.
  • 230 : va a la línea ‘230’.

Sugerencia: un atajo para insertar dos puntos y luego escribir wq! para escribir ediciones en el archivo y salir sin pedir confirmación, puede mantener presionada SHIFT y presionar dos veces z.

Esta es la mejor opción para leer archivos de código.

Buscar el nombre exacto de un paquete para apt-get #

Si sabe que necesita un paquete <packet>, pero no sabe el nombre exacto <packet-exact-name>, en lugar de buscar con Google, intente con lo siguiente:

sudo apt-get update
apt-cache search <packet>

Esto le devolverá una lista de nombres de paquetes con una descripción. Una vez que haya identificado el nombre exacto del paquete <packet-exact-name>, instálelo normalmente:

sudo apt-get install <packet-exact-name>

Limite la salida a líneas con la cadena deseada #

Si está ejecutando un comando que devuelve cientos de líneas, pero su interés está solo en las líneas que contienen la palabra <palabra>, ¡definitivamente debería usar grep! Por ejemplo, intente ejecutar:

ifconfig -a

Verifique la salida, que podría ser solo unas pocas líneas o bastante larga, si tiene un servidor con múltiples interfaces de red. Para mostrar (por ejemplo) solo las líneas que contienen ‘HWaddr’, intente usar grep:

ifconfig -a | grep HWaddr

¡Ahora la salida debería ser mucho más corta! En términos generales, podría usar grep de la siguiente manera:

<command-with-output> | grep <word>

Comprime archivos y carpetas usando el comando tar #

Utilice el comando tar (archivo de cinta) para comprimir sus archivos y carpetas. Es similar a la creación de archivos .ZIP en el entorno de Windows.

Syntax: tar -zcvf <output tar file> <source file> 

Example: tar -zcvf outputfile.tar.gz source file

Esto es lo que realmente significan esos interruptores:

-c: Crear un archivo.
-z: Comprime el archivo con gzip.
-v: muestra el progreso en la terminal mientras se crea el archivo, también conocido como modo “detallado”. La v siempre es opcional en estos comandos, pero es útil.
-f: Le permite especificar el nombre de archivo del archivo.
