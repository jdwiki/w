¿Por qué necesitamos compilación, compilación de Flujo de eventos y ejemplo? #

P. ¿Por qué necesitamos la compilación? Respuesta Necesitamos compilación para lograr un mayor nivel de eficiencia de nuestras aplicaciones Angular.

Fíjate en el siguiente ejemplo,

// ...
compile: function (el, scope) {
  var dirs = this._getElDirectives(el);
  var dir;
  var scopeCreated;
  dirs.forEach(function (d) {
    dir = Provider.get(d.name + Provider.DIRECTIVES_SUFFIX);
    if (dir.scope && !scopeCreated) {
      scope = scope.$new();
      scopeCreated = true;
    }
    dir.link(el, scope, d.value);
  });
  Array.prototype.slice.call(el.children).forEach(function (c) {
    this.compile(c, scope);
  }, this);
},
// ...

Usando el código anterior para renderizar la plantilla,

<ul>
  <li *ngFor="let name of names"></li>
</ul>

Es mucho más lento en comparación con:

this._text_12 = this.renderer.createText(this._el_11, '\n  ', null);

this._appEl_13 = new import2.AppElement(13, 11, this, this._anchor_13);

Flujo de eventos con compilación anticipada

Por el contrario, con AoT pasamos por los siguientes pasos:

  1. Desarrollo de aplicación Angular 2 con TypeScript.
  2. Compilación de la aplicación con ngc.
  3. Realiza la compilación de las plantillas con el compilador Angular a TypeScript.
  4. Compilación del código TypeScript a JavaScript.
  5. Agrupación.
  6. Minificación.
  7. Despliegue.

Aunque el proceso anterior parece un poco más complicado, el usuario sigue solo los pasos:

  1. Descarga todos los activos.
  2. Arranques angulares.
  3. La aplicación se procesa.

Como puede ver, falta el tercer paso, lo que significa una UX más rápida/mejor y, además, herramientas como angular2-seed y angular-cli automatizarán el proceso de compilación de manera espectacular.

¡Espero que pueda ayudarte! ¡Gracias!

1. Instale las dependencias de Angular 2 con el compilador #

NOTA: para obtener los mejores resultados, asegúrese de que su proyecto se haya creado con Angular-CLI.

npm install angular/{core,common,compiler,platform-browser,platform-browser-dynamic,http,router,forms,compiler-cli,tsc-wrapped,platform-server}

No tiene que hacer este paso si su proyecto ya tiene angular 2 y todas estas dependencias instaladas. Solo asegúrese de que el compilador esté allí.

2. Agregue angularCompilerOptions a su archivo tsconfig.json #

...
"angularCompilerOptions": {
    "genDir": "./ngfactory"
}
...

Esta es la carpeta de salida del compilador.

3. Ejecuta ngc, el compilador angular #

desde la raíz de su proyecto ./node_modules/.bin/ngc -p src donde src es donde vive todo su código angular 2. Esto generará una carpeta llamada ngfactory donde vivirá todo su código compilado.

"node_modules/.bin/ngc" -p src para Windows

4. Modifique el archivo main.ts para usar NgFactory y el navegador de plataforma estática #

// this is the static platform browser, the usual counterpart is @angular/platform-browser-dynamic.
import { platformBrowser } from '@angular/platform-browser';

// this is generated by the angular compiler
import { AppModuleNgFactory } from './ngfactory/app/app.module.ngfactory';

// note the use of `bootstrapModuleFactory`, as opposed to `bootstrapModule`.
platformBrowser().bootstrapModuleFactory(AppModuleNgFactory);

En este punto, debería poder ejecutar su proyecto. En este caso, mi proyecto fue creado usando Angular-CLI.

> ng serve

Uso de la compilación AoT con Angular CLI #

La interfaz de línea de comandos Angular CLI es compatible con la compilación AoT desde la versión beta 17.

Para construir su aplicación con la compilación AoT, simplemente ejecute:

ng build --prod --aot
