Hay algunos trucos más que podemos hacer con el enrutador (como restringir el acceso), pero se pueden tratar en un tutorial separado.

Si necesita una nueva ruta, simplemente modifique app.routes.ts y siga los siguientes pasos:

  1. Importar el Componente
  2. Agregue a la matriz rutas. Asegúrese de incluir una nueva ruta y componente.

Controlar el acceso hacia o desde una ruta #

El enrutador Angular predeterminado permite la navegación hacia y desde cualquier ruta incondicionalmente. Este no es siempre el comportamiento deseado.

En un escenario en el que a un usuario se le permita de forma condicional navegar hacia o desde una ruta, se puede usar un Route Guard para restringir este comportamiento.

Si su escenario se ajusta a uno de los siguientes, considere usar un guardia de ruta,

  • El usuario debe estar autenticado para navegar al componente de destino.
  • El usuario debe estar autorizado para navegar hasta el componente de destino.
  • El componente requiere una solicitud asíncrona antes de la inicialización.
  • El componente requiere la entrada del usuario antes de salir de él.

Cómo funcionan los guardias de ruta #

Los guardias de ruta funcionan devolviendo un valor booleano para controlar el comportamiento de la navegación del enrutador. Si se devuelve verdadero, el enrutador continuará navegando hacia el componente de destino. Si se devuelve falso, el enrutador denegará la navegación al componente de destino.

Interfaces de guardia de ruta #

El enrutador admite varias interfaces de protección:

  • CanActivate: se produce entre navegación de rutas.
  • CanActivateChild: ocurre entre la navegación de ruta a una ruta secundaria.
  • CanDeactivate: ocurre cuando se navega fuera de la ruta actual.
  • CanLoad: se produce entre la navegación de ruta a un módulo de funciones cargado de forma asíncrona.
  • Resolver: se utiliza para realizar la recuperación de datos antes de la activación de la ruta.

Estas interfaces se pueden implementar en su guardia para otorgar o quitar el acceso a ciertos procesos de la navegación.

Guardias de ruta síncronos vs. asíncronos #

Los guardias de ruta permiten operaciones síncronas y asíncronas para controlar condicionalmente la navegación.

Guardia de ruta síncrona #

Un guardia de ruta síncrono devuelve un booleano, como al calcular un resultado inmediato, para controlar condicionalmente la navegación.

import { Injectable }     from '@angular/core';
import { CanActivate }    from '@angular/router';

@Injectable()
export class SynchronousGuard implements CanActivate {
  canActivate() {
    console.log('SynchronousGuard#canActivate called');
    return true;
  }
}

Guardia de ruta asincrónica #

Para un comportamiento más complejo, un guardia de ruta puede bloquear la navegación de forma asíncrona. Un protector de ruta asíncrono puede devolver un Observable o Promise.

Esto es útil para situaciones como esperar la entrada del usuario para responder una pregunta, esperar para guardar correctamente los cambios en el servidor o esperar para recibir datos obtenidos de un servidor remoto.

import { Injectable }     from '@angular/core';
import { CanActivate, Router, ActivatedRouteSnapshot, RouterStateSnapshot } from '@angular/router';
import { Observable }     from 'rxjs/Rx';
import { MockAuthenticationService } from './authentication/authentication.service';

@Injectable()
export class AsynchronousGuard implements CanActivate {
    constructor(private router: Router, private auth: MockAuthenticationService) {}

    canActivate(route:ActivatedRouteSnapshot, state:RouterStateSnapshot):Observable<boolean>|boolean {
        this.auth.subscribe((authenticated) => {
            if (authenticated) {
                return true;
            }
            this.router.navigateByUrl('/login');
            return false;
        });
    }
}

Agregar guardia a la configuración de la ruta #

Uso de resolutores y guardias #

Estamos utilizando un protector de nivel superior en nuestra configuración de ruta para atrapar al usuario actual en la carga de la primera página, y un resolver para almacenar el valor de currentUser, que es nuestro usuario autenticado desde el backend.

Una versión simplificada de nuestra implementación se ve así:

Aquí está nuestra ruta de nivel superior:

export const routes = [
{
  path: 'Dash',
  pathMatch : 'prefix',
  component: DashCmp,
  canActivate: [AuthGuard],
  resolve: {
      currentUser: CurrentUserResolver
  },
  children: [...[
    path: '',
    component: ProfileCmp,
    resolve: {
        currentUser: currentUser
    }
  ]]
  }
];

Aquí está nuestro AuthService

import { Injectable } from '@angular/core';
import { Http, Headers, RequestOptions } from '@angular/http';
import { Observable } from 'rxjs/Rx';
import 'rxjs/add/operator/do';

@Injectable()
export class AuthService {
  constructor(http: Http) {
    this.http = http;

    let headers = new Headers({ 'Content-Type': 'application/json' });
    this.options = new RequestOptions({ headers: headers });
  }
  fetchCurrentUser() {
    return this.http.get('/api/users/me')
      .map(res => res.json())
      .do(val => this.currentUser = val);
 }
}

Aquí está nuestro AuthGuard:

import { Injectable } from '@angular/core';
import { CanActivate } from "@angular/router";
import { Observable } from 'rxjs/Rx';

import { AuthService } from '../services/AuthService';

@Injectable()
export class AuthGuard implements CanActivate {
  constructor(auth: AuthService) {
    this.auth = auth;
  }
  canActivate(route, state) {
    return Observable
      .merge(this.auth.fetchCurrentUser(), Observable.of(true))
      .filter(x => x == true);
  }
}

Aquí está nuestro CurrentUserResolver:

import { Injectable } from '@angular/core';
import { Resolve } from "@angular/router";
import { Observable } from 'rxjs/Rx';

import { AuthService } from '../services/AuthService';

@Injectable()
export class CurrentUserResolver implements Resolve {
  constructor(auth: AuthService) {
    this.auth = auth;
  }
  resolve(route, state) {
    return this.auth.currentUser;
  }
}

Usar Guard en el arranque de la aplicación #

Arranque #

Ahora que las rutas están definidas, debemos informar a nuestra aplicación sobre las rutas. Para hacer esto, inicie el proveedor que exportamos en el ejemplo anterior.

Encuentre su configuración de arranque (debe estar en main.ts, pero su kilometraje puede variar).

//main.ts

import {bootstrap} from '@angular/platform-browser-dynamic';

//Import the App component (root component)
import { App } from './app/app';

//Also import the app routes
import { APP_ROUTES_PROVIDER } from './app/app.routes';

bootstrap(App, [
  APP_ROUTES_PROVIDER,
])
.catch(err => console.error(err));

Configuración de la salida del enrutador #

Ahora que el enrutador está configurado y nuestra aplicación sabe cómo manejar las rutas, debemos mostrar los componentes reales que configuramos.

Para hacerlo, configure su plantilla/archivo HTML para su componente de nivel superior (aplicación) de la siguiente manera:

//app.ts

import {Component} from '@angular/core';
import {Router, ROUTER_DIRECTIVES} from '@angular/router';

@Component({
    selector: 'app',
    templateUrl: 'app.html',
    styleUrls: ['app.css'],
    directives: [
        ROUTER_DIRECTIVES,
    ]
})
export class App {
    constructor() {
    }
}

<!-- app.html -->

<!-- All of your 'views' will go here -->
<router-outlet></router-outlet>

El elemento <router-outlet></router-outlet> cambiará el contenido dada la ruta. Otro buen aspecto de este elemento es que no tiene que ser el único elemento en su HTML.

Por ejemplo: supongamos que desea una barra de herramientas en cada página que se mantenga constante entre las rutas, de forma similar a como se ve Stack Overflow. Puede anidar <router-outlet> debajo de los elementos para que solo cambien ciertas partes de la página.

Cambiando rutas (usando plantillas y directivas) #

Ahora que las rutas están configuradas, necesitamos alguna forma de cambiar las rutas.

Este ejemplo mostrará cómo cambiar rutas usando la plantilla, pero es posible cambiar rutas en TypeScript.

Aquí hay un ejemplo (sin enlace):

<a routerLink="/home">Home</a>

Si el usuario hace clic en ese enlace, se dirigirá a /home. El enrutador sabe cómo manejar /home, por lo que mostrará el componente Home.

Aquí hay un ejemplo con enlace de datos:

<a *ngFor="let link of links" [routerLink]="link">{{link}}</a>

Lo que requeriría que exista una matriz llamada enlaces, así que agregue esto a app.ts:

public links[] = [
    'home',
    'login'
]

Esto recorrerá la matriz y agregará un elemento <a> con la directiva routerLink = el valor del elemento actual en la matriz, creando esto:

 <a routerLink="home">home</a>
 <a routerLink="login">login</a>

Esto es particularmente útil si tiene muchos enlaces, o tal vez los enlaces deben cambiarse constantemente. Dejamos que Angular maneje el ajetreado trabajo de agregar enlaces simplemente brindándole la información que requiere.

En este momento, links[] es estático, pero es posible alimentarlo con datos de otra fuente.

Configuración de las rutas #

NOTA: este ejemplo se basa en la versión 3.0.0.-beta.2 de @angular/router. En el momento de escribir este artículo, esta es la última versión del enrutador.

Para usar el enrutador, defina rutas en un nuevo archivo TypeScript como tal

//app.routes.ts

import {provideRouter} from '@angular/router';

import {Home} from './routes/home/home';
import {Profile} from './routes/profile/profile';

export const routes = [
    {path: '', redirectTo: 'home'},
    {path: 'home', component: Home},
    {path: 'login', component: Login},
];

export const APP_ROUTES_PROVIDER = provideRouter(routes);

En la primera línea, importamos provideRouter para que podamos informar a nuestra aplicación cuáles son las rutas durante la fase de arranque.

Inicio y Perfil son solo dos componentes como ejemplo. Deberá importar cada Componente que necesite como ruta.

Luego, exporte la matriz de rutas.

ruta: La ruta al componente. NO NECESITA UTILIZAR ‘/……..’ Angular lo hará automáticamente

component: El componente a cargar cuando se accede a la ruta

redirectTo: Opcional. Si necesita redirigir automáticamente a un usuario cuando accede a una ruta en particular, proporcione esto.

Finalmente exportamos el router configurado. provideRouter devolverá un proveedor que podemos impulsar para que nuestra aplicación sepa cómo manejar cada ruta.
