Condiciones de prueba #

Para usar un salto condicional, se debe probar una condición. Probar una condición aquí se refiere solo al acto de verificar las banderas, el salto real se describe en [Saltos condicionales] (https://www.wikiod.com/es/x86/flujo-de-control#Saltos condicionales -saltos).

x86 prueba las condiciones basándose en el registro EFLAGS, que contiene un conjunto de indicadores que cada instrucción puede establecer potencialmente.

Las instrucciones aritméticas, como sub o add, y las instrucciones lógicas, como xor o and, obviamente “establecen las banderas”. Esto significa que las banderas CF, OF, SF, ZF, AF, PF son modificadas por esas instrucciones. Sin embargo, se permite cualquier instrucción para modificar las banderas, por ejemplo, cmpxchg modifica el ZF.

Consulte siempre la referencia de la instrucción para saber qué banderas se modifican por una instrucción específica.

x86 tiene un conjunto de saltos condicionales, mencionados anteriormente, que saltan si y solo si algunas banderas están configuradas o algunas están limpias o ambas cosas.

Banderas #

Las operaciones aritméticas y lógicas son muy útiles para establecer las banderas. Por ejemplo, después de un sub eax, ebx, por ahora con valores sin firmar, tenemos:

Bandera Cuando se establece Cuando claro
ZF Cuando el resultado es cero.
EAX - EBX = 0 ⇒ EAX = EBX		 Cuando el resultado no es cero.
EAX - EBX ≠ 0 ⇒ EAX ≠ EBX
CF Cuando el resultado necesitaba llevar para el MSb.
EAX - EBX < 0 ⇒ EAX < EBX		 Cuando el resultado no necesitaba acarreo para el MSb.
EAX - EBX ≮ 0 ⇒ EAX ≮ EBX
SF Cuando se establece el resultado MSb. Cuando el resultado MSb no está establecido.
DE Cuando se produjo un desbordamiento firmado. Cuando no ocurrió un desbordamiento firmado.
PF Cuando el número de bits establecidos en el byte menos significativo del resultado es par. Cuando el número de bits establecidos en el byte menos significativo del resultado es impar.
AF Cuando el dígito BCD inferior generó un acarreo.
Es el acarreo del bit 4.		 Cuando el dígito BCD inferior no generó un acarreo.
Es el acarreo del bit 4.

Ensayos no destructivos #

Las instrucciones sub y and modifican su operando de destino y requerirían dos copias adicionales (guardar y restaurar) para mantener el destino sin modificar.

Para realizar una prueba no destructiva existen las instrucciones cmp y test. Son idénticos a su contraparte destructiva excepto que el resultado de la operación se descarta y solo se guardan las banderas.

Destructivo No destructivo
sub cmp
y prueba 
test eax, eax             ;and eax, eax
                          ;ZF = 1 iff EAX is zero

test eax, 03h             ;and eax, 03h
                          ;ZF = 1 if both bit[1:0] are clear
                          ;ZF = 0 if at least one of bit[1:0] is set

cmp eax, 241d             ;sub eax, 241d
                          ;ZF = 1 iff EAX is 241
                          ;CF = 1 iff EAX < 241

Pruebas firmadas y no firmadas #

La CPU no le da un significado especial a los valores de registro1, el signo es una construcción del programador. No hay diferencia cuando se prueban valores con y sin signo. El procesador calcula suficientes banderas para probar las relaciones aritméticas habituales (igual, menor que, mayor que, etc.) tanto si los operandos se consideraran con signo como sin signo.

1 Aunque tiene algunas instrucciones que solo tienen sentido con formatos específicos, como complemento a dos. Esto es para hacer que el código sea más eficiente, ya que implementar el algoritmo en el software requeriría mucho código.

Saltos condicionales #

Según el estado de las banderas, la CPU puede ejecutar o ignorar un salto. Una instrucción que realiza un salto basado en las banderas cae bajo el nombre genérico de Jcc - Jump on Condition Code1.

Sinónimos y terminología #

Para mejorar la legibilidad del código ensamblador, Intel definió varios sinónimos para el mismo código de condición. Por ejemplo, jae, jnb y jnc son todos el mismo código de condición CF = 0.

Si bien el nombre de la instrucción puede dar una pista muy fuerte sobre cuándo usarla o no, el único enfoque significativo es reconocer las banderas que deben probarse y luego elegir las instrucciones apropiadamente. Sin embargo, Intel dio nombres a las instrucciones que tienen perfecto sentido cuando se usan después de una instrucción cmp. A los efectos de esta discusión, se supondrá que cmp ha establecido las banderas antes de un salto condicional.

Igualdad #

Los operandos son iguales si se ha establecido ZF; de lo contrario, difieren. Para probar la igualdad necesitamos ZF = 1.

je a_label           ;Jump if operands are equal
jz a_label           ;Jump if zero (Synonym)

jne a_label          ;Jump if operands are NOT equal
jnz a_label          ;Jump if not zero (Synonym)
Instrucción Banderas
es, jz ZF = 1
jne, jnz ZF = 0

Mas grande que #

Para operandos sin signo, el destino es mayor que el origen si no se necesita el acarreo, es decir, si CF = 0. Cuando CF = 0 es posible que los operandos fueran iguales, probar ZF eliminará la ambigüedad.

jae a_label      ;Jump if above or equal (>=)
jnc a_label      ;Jump if not carry (Synonym)
jnb a_label      ;Jump if not below (Synonym)

ja a_label       ;Jump if above  (>)
jnbe a_label     ;Jump if not below and not equal (Synonym)
Instrucción Banderas
, jae, jnc, jnb FC = 0
ja, jnbe CF = 0, ZF = 0

Para operandos firmados necesitamos comprobar que SF = 0, a menos que haya habido un desbordamiento firmado, en cuyo caso el SF resultante se invierte. Dado que OF = 0 si no se produjo un desbordamiento con signo y 1 en caso contrario, debemos comprobar que SF = OF.

ZF se puede utilizar para implementar una prueba estricta/no estricta.

jge a_label      ;Jump if greater or equal (>=)
jnl a_label      ;Jump if not less (Synonym)

jg a_label       ;Jump if greater (>)
jnle a_label     ;Jump if not less and not equal (Synonym)
Instrucción Banderas
jge, jnl SF = DE
jg, jnle SF = OF, ZF = 0

Menos que #

Estos utilizan las condiciones invertidas de arriba.

jbe a_label      ;Jump if below or equal (<=)
jna a_label      ;Jump if not above (Synonym)

jb a_label       ;Jump if below (<)
jc a_label       ;Jump if carry (Synonym)
jnae a_label     ;Jump if not above and not equal (Synonym)

;SIGNED 

jle a_label      ;Jump if less or equal (<=)
jng a_label      ;Jump if not greater (Synonym)

jl a_label       ;Jump if less (<)
jnge a_label     ;Jump if not greater and not equal (Synonym)
Instrucción Banderas
, jbe, jna CF = 1 o ZF = 1
jb, jc, jnae FC = 1
jle, jng SF != OF o ZF = 1
jl, jnge SF != DE

Indicadores específicos #

Cada bandera se puede probar individualmente con j<flag_name> donde flag_name no contiene la F final (por ejemplo, CF → C, PF → P).

Los códigos restantes no cubiertos antes son:

Instrucción Bandera
js SF = 1
jns SF = 0
``sí'' DE = 1
jno DE = 0
jp, jpe (e = par) FP = 1
jnp, jpo (o = impar) FP = 0

Un salto condicional más (uno extra) #

Un salto condicional x86 especial no prueba la bandera. En su lugar, prueba el valor del registro cx o ecx (según el modo de dirección de la CPU actual que es de 16 o 32 bits), y el salto se ejecuta cuando el registro contiene cero.

Esta instrucción fue diseñada para la validación del registro contador (cx/ecx) por delante de las instrucciones tipo rep, o por delante de los bucles loop.

jcxz  a_label   ; jump if cx (16b mode) or ecx (32b mode) is zero
jecxz a_label   ; synonym of jcxz (recommended in source code for 32b target)
Instrucción Registro (no bandera)
jcxz, jecxz cx = 0 (modo 16b)
jcxz, jecxz ecx = 0 (modo 32b)

1 O algo así.

una_etiqueta #

En los ejemplos anteriores, a_label es el destino de la CPU cuando la condición probada es “verdadera”. Cuando la condición probada es “falsa”, la CPU continuará con la siguiente instrucción después del salto condicional.

Sinónimos #

Hay sinónimos de instrucciones que se pueden usar para mejorar la legibilidad del código. Por ejemplo, ja y jnbe (salto no inferior ni igual) son la misma instrucción.

Códigos complementarios sin firmar firmados #

Operación Sin firmar firmado
> sí jg
>= jae jge
< jb jl
, <= jbe jle
= ’es' ’es'
≠, !=, <> jne jne

Saltos incondicionales #

jmp a_label                      ;Jump to a_label
jmp bx                           ;Jump to address in BX
jmp WORD [aPointer]              ;Jump to address in aPointer
jmp 7c0h:0000h                   ;Jump to segment 7c0h and offset 0000h
jmp FAR WORD [aFarPointer]       ;Jump to segment:offset in aFarPointer

Saltos cercanos relativos #

jmp a_label es:

  • cerca Solo especifica la parte compensada de la dirección lógica de destino. Se supone que el segmento es CS.
  • pariente La instrucción semántica es saltar rel bytes hacia adelante1 desde la siguiente dirección de instrucción o IP = IP + rel.

La instrucción está codificada como EB <rel8> o EB <rel16/32>, el ensamblador selecciona la forma más apropiada, generalmente prefiriendo una más corta. La anulación por ensamblador es posible, por ejemplo, con NASM jmp SHORT a_label, jmp WORD a_label y jmp DWORD a_label generan las tres formas posibles.

Saltos cercanos indirectos absolutos #

jmp bx y jmp WORD [aPointer] son:

  • cerca Solo especifican la parte compensada de la dirección lógica de destino. Se supone que el segmento es CS.
  • absoluto indirecto La semántica de las instrucciones es saltar a la dirección en reg o mem o IP=reg, IP=mem.

La instrucción está codificada como FF /4, para la memoria indirecta, el tamaño del operando se determina como para cualquier otro acceso a la memoria.

Saltos lejanos absolutos #

jmp 7c0h:0000h es:

  • lejos Especifica ambas partes de la dirección lógica: el segmento y el desplazamiento.

  • absoluto La semántica de la instrucción es saltar a la dirección segmento:desplazamiento o CS = segmento, IP = desplazamiento.

La instrucción está codificada como EA <imm32/48> según el tamaño del código. Es posible elegir entre las dos formas en algún ensamblador, por ejemplo con NASM jmp 7c0h: WORD 0000h y jmp 7c0h: DWORD 0000h generar la primera y segunda forma.

Saltos lejanos indirectos absolutos #

jmp PALABRA LEJANA [unPunteroLejano] es:

  • lejos Especifica ambas partes de la dirección lógica: el segmento y el desplazamiento.

  • Absoluto indirecto La semántica de la instrucción es saltar al segmento:desplazamiento almacenado en mem2 o CS = mem[23:16/32], IP = [15/31:0] .

La instrucción está codificada como FF /5, el tamaño del operando se puede controlar con los especificadores de tamaño. En NASM, un poco no intuitivo, son jmp FAR WORD [aFarPointer] para un operando 16:16 y jmp FAR DWORD [aFarPointer] para un operando 16:32.

Saltos perdidos #

  • casi absoluto Se puede emular con un salto casi indirecto.

      mov bx, target            ;BX = absolute address of target
  jmp bx

  • pariente lejano No tiene sentido o es demasiado estrecho de uso de todos modos.

1 El complemento dos se usa para especificar un desplazamiento con signo y, por lo tanto, saltar hacia atrás. 2 Que puede ser un seg16:off16 o un seg16:off32, de tamaño 16:16 y 16:32.
