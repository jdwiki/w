Sintaxis #

  • en línea: `Código entre acentos graves` o <código>Código entre estas etiquetas HTML</código>
  • multilínea: ____Código que está sangrado en cada línea o <pre><code>Múltiples líneas de código entre estas etiquetas HTML</code></pre>

Código en línea #

Markdown admite la adición de código en línea como este, obtenido al envolver texto en acentos graves:

`código aquí`

Alternativamente, puede poner su código en línea entre las etiquetas HTML <code> y </code>.

Considere el siguiente código de descuento:

`This` is an inline code block! <code>This</code> is one too!

Eso produciría la siguiente salida:

¡Esto es un bloque de código en línea! ¡Este también lo es!

Si necesita incluir un acento grave dentro del código en línea, puede usar varios acentos graves para comenzar y finalizar el bloque de código en línea, así:

``code containing a backtick (`) character``

Eso produciría la siguiente salida:

código que contiene un carácter de acento grave (`)

Use \ para escapar de los acentos graves. Por ejemplo:

\`a\`

se representará como

`a`

Resaltado de sintaxis (StackExchange) #

En los sitios de StackExchange, los fragmentos de código pueden proporcionar un resaltado de sintaxis opcional. En sitios como Stack Overflow, el idioma predeterminado se deriva de las etiquetas utilizadas en la pregunta asociada (si corresponde). Además, el lenguaje de resaltado de sintaxis de un fragmento de código también se puede definir agregando un comentario HTML al cuerpo del texto.

<!-- language: lang-vb -->

    Sub ShowVB()
    Dim i As Long
    For i = 1 To 2
        If i = 3 Then
            MsgBox "How did that happen?"
        End If
    Next
    End Sub

Dicho comentario cambiará el idioma de resaltado de sintaxis para todos los fragmentos de código posteriores, lo que puede ser bastante útil, especialmente cuando hay varios idiomas involucrados en una publicación.

Lo anterior se representará con resaltado de Visual Basic como:

Sub ShowVB()
Dim i As Long
For i = 1 To 2
    If i = 3 Then
        MsgBox "How did that happen?"
    End If
Next
End Sub

Bloques de código cercado #

Algunos analizadores permiten que el código se designe agregando tres acentos graves antes y después de una sección de código.

```
<p><em>This</em> is an HTML example!</p>
```

Opcionalmente, muchos analizadores permiten agregar resaltado de sintaxis especificando el idioma del código inmediatamente después del primer conjunto de acentos graves:

```html
<p><em>This</em> is an HTML example!</p>
```

Resultado:

<p><em>This</em> is an HTML example!</p>

Bloques de código sangrado dentro de listas #

Al agregar bloques de código sangrado dentro de una [lista] (https://www.wikiod.com/es/markdown/creacion-de-listas), primero necesita una línea en blanco y luego sangrar más el código. Los diferentes sabores de Markdown tienen reglas diferentes para esto.

  1. StackExchange requiere que el código tenga una sangría de 8 caracteres en lugar de los 4 habituales. (Los espacios se reemplazan con * para mayor claridad):

     1.*Listitem1
 2.*Listitem2

 ********code here
 3.*Listitem3

  2. Las especificaciones como CommonMark requieren que el bloque de código tenga una sangría de 4 caracteres desde donde comienza el texto del elemento de la lista. (Los espacios se reemplazaron con * para mayor claridad):

     1.****Listitem1
 2.****Listitem2

 **********code here
 3.****Listitem3

Bloques de código con sangría #

Puede crear fragmentos de código de varias líneas sangrando cada línea con al menos cuatro espacios o una pestaña:

    #include <stdio.h>

    int main() {
      printf("Hello World!\n");
      return 0;
    }
