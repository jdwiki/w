Nombres de plantilla frente a ID de plantilla frente a nombres de elementos en consultas:

Le recomiendo enfáticamente que utilice ID de plantilla y no nombres de plantilla o nombres de elementos en sus consultas. Esto garantizará que sus consultas seguirán funcionando, incluso cuando se cambie el nombre de las plantillas o los elementos.

La única excepción a esto es cuando se trabaja con plantillas OOTB, mientras se consulta una estructura OOTB, p. /sitecore/content o /sitecore/system/Marketing Control Panel. En estas situaciones, la pérdida de legibilidad suele ser mayor que el riesgo de que se rompan las consultas, ya que es mucho menos probable que se cambie el nombre de estas plantillas.

Tenga en cuenta que los nombres de las plantillas se usaron en mis ejemplos anteriores, en aras de la legibilidad. Esas consultas no deben usarse en producción, a menos que los nombres de las plantillas se reemplacen con ID de plantilla.

Hoja de referencia:

Me di cuenta de que la hoja de trucos de consulta de Sitecore ya no está disponible para su descarga en la web (todos los enlaces alojados en Sitecore ahora redirigen a páginas 404). Afortunadamente, tenía una copia en mi máquina y agregué una captura de pantalla a continuación:

Consulta específica del sitio #

Estructura de árbol:

/sitecore
    /content
        /foo-site
            /home
                /my-account
        /bar-site
            /home
                /my-account
        /baz-site
            /home
                /my-account
  • La plantilla de cada elemento del sitio (foo-site, bar-site, baz-site) se llama Nodo del sitio.
  • La plantilla de cada elemento de inicio (“inicio”, “inicio”, “inicio”) se denomina “Página de inicio”.
  • La plantilla de cada elemento de la cuenta de usuario (my-account, my-account, my-account) se denomina Página de cuenta de usuario

Artículo actual:

El elemento actual podría ser el elemento inicio o cualquier página debajo del elemento inicio para cualquiera de los sitios dados, y esta consulta seguirá funcionando (siempre que no haya elementos con la plantilla Página de inicio debajo del elemento inicio). artículos que son un ancestro del artículo actual).

Consulta:

query:./ancestor-or-self::*[@@templatename='Homepage']/*[@@templatename='my-account']

Resultado:

Si consulta desde el elemento home o uno de sus descendientes en el sitio foo-site:

/sitecore/content/foo-site/home/my-account

Si consulta desde el elemento home o uno de sus descendientes en el sitio bar-site:

/sitecore/content/bar-site/home/my-account

Si consulta desde el elemento home o uno de sus descendientes en el sitio baz-site:

/sitecore/content/baz-site/home/my-account

Seleccionar por ruta de elemento #

Consulta:

query:/sitecore/content/home/foo/bar

Resultado

bar

Consulta relativa #

Artículo actual:

bar (path: /sitecore/content/home/foo/bar)

Consulta:

query:./child/grandchild

Resultado:

grandchild (path: /sitecore/content/home/foo/bar/child/grandchild)

Consulta de atributos del artículo #

Consulta:

query:/sitecore/content/[@@templatename='Homepage']

Resultado:

home (name: home, path: /sitecore/content/home, template name: Homepage)
