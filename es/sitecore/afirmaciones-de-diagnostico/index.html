Comprobaciones de argumentos #

ArgumentoCondición #

Este método verifica si el argumento especificado es verdadero. También toma el nombre del argumento que se registra si falla la condición.

Assert.ArgumentCondition(pageIndex >= 0, "pageIndex", "Value must be greater than or equal to zero.");

Argumento No Nulo #

Este método asegura que el argumento pasado no sea nulo. Hay dos firmas para este método, la primera toma un objeto y un nombre de parámetro y realiza una verificación simple de valores nulos.

Assert.ArgumentNotNull(item, "item");

Argumento No Nulo O Vacío #

Esto es similar al método ArgumentNotNull, pero también verificará si el objeto está vacío. Hay tres variantes de este método. La primera variante toma un ID de Sitecore y un nombre de argumento, y verifica si el ID es nulo.

var nullId = new new ID("{00000000-0000-0000-0000-000000000000}");

// Both of these calls will result in an exception
Assert.ArgumentNotNullOrEmpty((ID)null, "null");
Assert.ArgumentNotNullOrEmpty(nullId, nameof(nullId));

El segundo método agrega una verificación para ver si la cadena dada es nula o está vacía.

// Both of these calls will result in an exception
Assert.ArgumentNotNullOrEmpty((string)null, "null");
Assert.ArgumentNotNullOrEmpty("", nameof(emptyString));

Asegúrese de que el elemento esté en modo de edición #

Comprueba si el Item pasado está en modo de edición. Si no, lanza una EditingNotAllowedException.

Assert.IsEditing(Sitecore.Context.Item);

Asegúrese de que dos valores sean iguales #

Compara dos valores para la igualdad. Solo puede comparar cadenas, enteros y valores booleanos.

Assert.AreEqual(documentElement.LocalName, "xamlControls", "Xaml files must have a root node named 'xamlControls'.");

Asegúrese de que un valor sea verdadero o falso #

Para afirmar que un valor es verdadero o falso:

Assert.IsFalse(Settings.DoBadThings, "Bad things should not happen, disable DoBadThings.");
Assert.IsTrue(magicNumber =< 42, "The magic number is greater than 42!");

También puede pasar parámetros de formato para el mensaje de excepción

Assert.IsFalse(myValue > 5, "The value should not be greater than 5, it's currently {0}", myValue);

ResultadoNoNulo #

ResultNotNull() verifica si el objeto pasado no es nulo. Si el objeto y el mensaje no son nulos, simplemente devolverá el objeto que se pasó; de lo contrario, lanzará InvalidOperationException.

return Assert.ResultNotNull(this.Database.GetItem(this.ItemRootId), string.Concat("Root item not found. ID: ", this.ItemRootId));

Objeto requerido #

Esto verifica si el objeto dado es nulo, y luego arroja RequiredObjectIsNullException si lo es.

Assert.Required(parameter, "parameter is required.");

Cheques nulos/vacíos #

No es nulo #

Este es un método muy simple y popular para verificar si un elemento no es nulo. Simplemente verifica el objeto que se pasa para ver si es nulo.

Assert.IsNotNull(database, type, "Name: {0}", item);

No es nulo ni está vacío #

Esto es lo mismo que IsNotNull anterior, pero funciona con valores de cadena en lugar de objetos.

Assert.IsNotNullOrEmpty(propertyName, "user");

Es nulo #

Esto es simplemente lo contrario del método IsNotNull(). Este método afirma que el objeto es nulo.

Assert.IsNull(this.StandardValues, "A Standard Values item has already been created for this template ");

Afirmaciones de seguridad #

PuedeEjecutarAplicación #

Para verificar si el usuario tiene permiso para ejecutar la aplicación dada. Si no, se lanza AccessDeniedException.

Assert.CanRunApplication("WebEdit");

Tiene acceso #

HasAccess verificará si el parámetro dado es verdadero, de lo contrario, se lanzará una excepción AccessDeniedException.

Assert.HasAccess(Context.User.IsAdministrator, "Only administrators can create new domains");
