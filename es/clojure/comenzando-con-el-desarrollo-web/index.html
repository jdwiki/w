Crear una nueva aplicación Ring con http-kit #

Ring es una API estándar de facto para aplicaciones HTTP clojure, similar a Ruby’s Rack y Python’s WSGI.

Vamos a usarlo con el servidor web http-kit.

Crear nuevo proyecto de Leiningen:

lein new app myapp

Agregue la dependencia de http-kit a project.clj:

  :dependencies [[org.clojure/clojure "1.8.0"]
                 [http-kit "2.1.18"]]

Agregue :require para http-kit a core.clj:

(ns test.core
  (:gen-class)
  (:require [org.httpkit.server :refer [run-server]]))

Defina el controlador de solicitud de anillo. Los controladores de solicitudes son solo funciones desde la solicitud hasta la respuesta y la respuesta es solo un mapa:

(defn app [req]
  {:status  200
   :headers {"Content-Type" "text/html"}
   :body    "hello HTTP!"})

Aquí solo devolvemos 200 OK con el mismo contenido para cualquier solicitud.

Inicie el servidor en la función -main:

(defn -main
  [& args]
  (run-server app {:port 8080}))

Ejecute con lein run y abra http://localhost:8080/ en el navegador.

Nueva aplicación web con Luminus #

Luminus es un micro-framework de Clojure basado en un conjunto de librerías ligeras. Su objetivo es proporcionar una plataforma robusta, escalable y fácil de usar. Con Luminus puede concentrarse en desarrollar su aplicación de la manera que desee sin distracciones. También tiene muy buena documentación que cubre algunos de los temas principales.

Es muy fácil empezar con luminus. Simplemente cree un nuevo proyecto con los siguientes comandos:

lein new luminus my-app
cd my-app
lein run

Su servidor comenzará en el puerto 3000

Ejecutar lein new luminus myapp creará una aplicación utilizando la plantilla de perfil predeterminada. Sin embargo, si desea agregar más funcionalidades a su plantilla, puede agregar sugerencias de perfil para la funcionalidad extendida.

Servidores web #

  • +aleph - agrega soporte de servidor Aleph al proyecto
  • +jetty - agrega soporte Jetty al proyecto
  • +http-kit - agrega el servidor web HTTP Kit al proyecto

bases de datos #

  • +h2 - agrega espacio de nombres db.core y dependencias H2 db
  • +sqlite - agrega el espacio de nombres db.core y las dependencias de SQLite db
  • +postgres: agrega espacio de nombres db.core y agrega dependencias de PostreSQL
    • mysql - agrega espacio de nombres db.core y agrega dependencias de MySQL
  • +mongodb - agrega el espacio de nombres db.core y las dependencias de MongoDB
  • +datomic - agrega espacio de nombres db.core y dependencias Datomic

misceláneas #

  • +auth - agrega dependencia de Buddy y middleware de autenticación
  • +auth-jwe - agrega la dependencia de Buddy con el backend de JWE
    • sidra - agrega soporte para CIDER usando el complemento CIDER nREPL
  • +cljs - agrega compatibilidad con [ClojureScript][cljs] con Reagent
  • +reencuadre: agrega compatibilidad con [ClojureScript][cljs] con reencuadre
  • +pepino - un perfil para pepino con clj-webdriver
  • +swagger: agrega soporte para Swagger-UI usando la biblioteca compojure-api
  • +sassc: agrega soporte para archivos SASS/SCSS usando el compilador de línea de comandos SassC
  • +servicio: cree una aplicación de servicio sin el repetitivo de front-end, como las plantillas HTML
    • guerra: agregue soporte para crear archivos WAR para la implementación en servidores como Apache Tomcat (NO debe usarse para aplicaciones Immutant que se ejecutan en WildFly)
  • +sitio: crea una plantilla para el sitio usando la base de datos especificada (H2 por defecto) y ClojureScript

Para agregar un perfil, simplemente páselo como argumento después del nombre de su aplicación, por ejemplo:

lein new luminus myapp +cljs

También puede mezclar varios perfiles al crear la aplicación, por ejemplo:

lein new luminus myapp +cljs +swagger +postgres
