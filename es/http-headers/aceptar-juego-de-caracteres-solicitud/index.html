Accept-Charset le dice al servidor qué conjuntos de caracteres acepta el cliente.

Sintaxis #

  • Aceptar-Charset: tipo;QualityFactor
  • Aceptar juego de caracteres: tipo;QualityFactor, tipo;QualityFactor, tipo;QualityFactor, …

Parámetros #

Parámetro Descripción
tipo Un nombre de conjunto de caracteres. Esto también puede ser un * para todos los conjuntos de caracteres
Factor de calidad El factor de calidad en el formato ;q=0.8 (opcional)

Accept-Charset toma una serie de conjuntos de caracteres e incluye una preferencia opcional para cuál debe usar el servidor. El conjunto de caracteres es uno de la lista. de conjuntos de caracteres disponibles en el Registro de “Conjuntos de caracteres” de IANA. Por ejemplo UTF-8.

El conjunto de caracteres está separado por comas con un factor de calidad opcional (usando un ;q=) que es usó la preferencia de los clientes para usar este tipo. El factor de calidad tiene un valor de 0 a 1, cuanto mayor sea el número, mayor será la preferencia por ese tipo.

Si este encabezado no está incluido, el cliente aceptará cualquier conjunto de caracteres.

El servidor usa Content-Type para informar al cliente qué conjunto de caracteres está usando.

Si el servidor no puede encontrar un conjunto de caracteres aceptable para responder, debe enviar una respuesta 406 (no aceptable) o ignorar este encabezado y no haciendo cualquier negociación de contenido.

Solo acepta UTF-8 #

Solicitud: “http://ejemplo.com”

OBTENER/HTTP/1.1 Anfitrión: ejemplo.com Agente de usuario: Mozilla/5.0 (Windows NT 6.1; WOW64; rv:53.0) Gecko/20100101 Firefox/53.0 Aceptar: texto/html,aplicación/xml;q=0.9,*/*;q=0.8 Aceptar juego de caracteres: UTF-8 Aceptar-Idioma: en-US,en;q=0.5 Aceptar codificación: gzip, deflate Conexión: mantener vivo

El cliente aceptará solo conjuntos de caracteres UTF-8.

Solo acepta UTF-8 e iso-8859-1 #

Solicitud: “http://ejemplo.com”

OBTENER/HTTP/1.1 Anfitrión: ejemplo.com Agente de usuario: Mozilla/5.0 (Windows NT 6.1; WOW64; rv:53.0) Gecko/20100101 Firefox/53.0 Aceptar: texto/html,aplicación/xml;q=0.9,*/*;q=0.8 Aceptar juego de caracteres: UTF-8, iso-8859-1 Aceptar-Idioma: en-US,en;q=0.5 Aceptar codificación: gzip, deflate Conexión: mantener vivo

El cliente solo aceptará conjuntos de caracteres UTF-8 e iso-8859-1.

Solo acepta UTF-8, iso-8859-1 con preferencia #

Solicitud: “http://ejemplo.com”

OBTENER/HTTP/1.1 Anfitrión: ejemplo.com Agente de usuario: Mozilla/5.0 (Windows NT 6.1; WOW64; rv:53.0) Gecko/20100101 Firefox/53.0 Aceptar: texto/html,aplicación/xml;q=0.9,*/*;q=0.8 Aceptar juego de caracteres: UTF-8, iso-8859-1;q=0.8 Aceptar-Idioma: en-US,en;q=0.5 Aceptar codificación: gzip, deflate Conexión: mantener vivo

El cliente aceptará conjuntos de caracteres UTF-8 e iso-8859-1, pero prefiere UTF-8 (que tiene un factor de calidad de 1,0).

Acepta cualquier conjunto de caracteres pero tiene preferencia por algunos tipos #

Solicitud: “http://ejemplo.com”

OBTENER/HTTP/1.1 Anfitrión: ejemplo.com Agente de usuario: Mozilla/5.0 (Windows NT 6.1; WOW64; rv:53.0) Gecko/20100101 Firefox/53.0 Aceptar: texto/html,aplicación/xml;q=0.9,*/*;q=0.8 Aceptar juego de caracteres: UTF-8, iso-8859-1;q=0.8, *;q=0.5 Aceptar-Idioma: en-US,en;q=0.5 Aceptar codificación: gzip, deflate Conexión: mantener vivo

El cliente aceptará cualquier conjunto de caracteres pero prefiere UTF-8 y luego iso-8859-1 si UTF-8 no está disponible.
