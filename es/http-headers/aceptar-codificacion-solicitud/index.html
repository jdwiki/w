Aceptar-Codificación (Solicitud)

Accept-Encoding le dice al servidor qué codificación acepta el cliente. La codificación se utiliza principalmente para la compresión.

Sintaxis #

  • Aceptar-Codificación: Codificación;QualityFactor
  • Aceptar-Codificación: Codificación;QualityFactor, tipo;QualityFactor, tipo;QualityFactor, …

Parámetros #

Parámetro Descripción
Codificación El tipo de codificación a utilizar. Esto también puede ser un * para decir que el cliente no tiene preferencia sobre qué codificación usar
Factor de calidad El factor de calidad en el formato ;q=0.8. Si se establece en 0, significa “no aceptable”. (opcional)

Accept-Encoding toma un número de codificación e incluye una preferencia opcional para cuál debe usar el servidor. La codificación es una de la lista. de codificaciones disponibles en Registro IANA. Por ejemplo gzip.

La codificación está separada por comas con un factor de calidad opcional (usando ;q=) que es usó la preferencia de los clientes para usar esta codificación. El factor de calidad tiene un valor de 0 a 1, cuanto mayor sea el número, mayor será la preferencia por esa codificación.

Si este encabezado no está incluido, el cliente no indica ninguna preferencia por la codificación. No significa que el cliente admita todas las codificaciones.

Un valor de identidad siempre es aceptable a menos que lo rechace con identidad;q=0.

El servidor usa Content-Encoding para informar al cliente qué codificación está usando.

Si el servidor no puede encontrar un conjunto de caracteres aceptable para responder, debe enviar una respuesta 406 (no aceptable) o ignorar este encabezado y no haciendo cualquier negociación de contenido.

Solicitar gzip #

Solicitud: “http://ejemplo.com”

OBTENER/HTTP/1.1 Anfitrión: ejemplo.com Agente de usuario: Mozilla/5.0 (Windows NT 6.1; WOW64; rv:53.0) Gecko/20100101 Firefox/53.0 Aceptar: texto/html,aplicación/xml;q=0.9,*/*;q=0.8 Aceptar-Codificación: gzip Aceptar-Idioma: en-US,en;q=0.5 Aceptar codificación: gzip, deflate Conexión: mantener vivo

El cliente aceptará gzip y codificación de identidad.

Solicitar gzip y desinflar #

Solicitud: “http://ejemplo.com”

OBTENER/HTTP/1.1 Anfitrión: ejemplo.com Agente de usuario: Mozilla/5.0 (Windows NT 6.1; WOW64; rv:53.0) Gecko/20100101 Firefox/53.0 Aceptar: texto/html,aplicación/xml;q=0.9,*/*;q=0.8 Aceptar-Codificación: compress, gzip Aceptar-Idioma: en-US,en;q=0.5 Aceptar codificación: gzip, deflate Conexión: mantener vivo

El cliente solo aceptará codificaciones gzip, compress e identidad.

Request compres but prefer gzip #

Solicitud: “http://ejemplo.com”

OBTENER/HTTP/1.1 Anfitrión: ejemplo.com Agente de usuario: Mozilla/5.0 (Windows NT 6.1; WOW64; rv:53.0) Gecko/20100101 Firefox/53.0 Aceptar: texto/html,aplicación/xml;q=0.9,*/*;q=0.8 Aceptar codificación: gzip;q=1.0, compress;q=0.5 Aceptar-Idioma: en-US,en;q=0.5 Aceptar codificación: gzip, deflate Conexión: mantener vivo

El cliente aceptará gzip, compress y codificación de identidad, pero prefiere gzip (que tiene un factor de calidad de 1,0).

Sin preferencia por el tipo de codificación #

Solicitud: “http://ejemplo.com”

OBTENER/HTTP/1.1 Anfitrión: ejemplo.com Agente de usuario: Mozilla/5.0 (Windows NT 6.1; WOW64; rv:53.0) Gecko/20100101 Firefox/53.0 Aceptar: texto/html,aplicación/xml;q=0.9,*/*;q=0.8 Aceptar-Codificación: * Aceptar-Idioma: en-US,en;q=0.5 Aceptar codificación: gzip, deflate Conexión: mantener vivo

El cliente no tiene preferencia por el tipo de codificación.
