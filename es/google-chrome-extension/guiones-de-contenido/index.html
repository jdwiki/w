Declaración de scripts de contenido en el manifiesto #

Los scripts de contenido se pueden declarar en manifest.json para que se inyecten siempre en páginas que coincidan con un conjunto de patrones de URL.

Ejemplo mínimo #

"content_scripts" : [
  {
    "js": ["content.js"],
    "css": ["content.css"]
    "matches": ["http://example.com/*"]
  }
]

Esta entrada de manifiesto le indica a Chrome que inyecte un script de contenido content.js, junto con el archivo CSS content.css, después de cualquier navegación a una página que coincida con el patrón de coincidencia http://example.com /*

Tanto las claves js como css son opcionales: puedes tener solo una de ellas o ambas dependiendo de lo que necesites.

La clave content_scripts es una matriz, y puede declarar varias definiciones de secuencias de comandos de contenido:

"content_scripts" : [
  {
    "js": ["content.js"],
    "matches": ["http://*.example.com/*"]
  },
  {
    "js": ["something_else.js"],
    "matches": ["http://*.example.org/*"]
  }
]

Tenga en cuenta que tanto js como matches son matrices, incluso si solo tiene una entrada.

Hay más opciones disponibles en la documentación oficial y otros ejemplos.

Nota IMPORTANTE #

Los scripts de contenido declarados en el manifiesto solo se inyectarán en nuevas navegaciones después de la carga de la extensión. No se inyectarán en pestañas existentes. Esto también se aplica a las recargas de extensiones durante el desarrollo y las actualizaciones de extensiones después del lanzamiento.

Si necesita asegurarse de que las pestañas abiertas actualmente estén cubiertas, considere también realizar una inyección programática al inicio.

Inyectar scripts de contenido desde una página de extensión #

Si, en lugar de inyectar siempre un script de contenido en función de la URL, desea controlar directamente cuándo se inyecta un script de contenido, puede usar Inyección programática.

Ejemplo mínimo #

  • JavaScript
  chrome.tabs.executeScript({file: "content.js"});

-CSS

  chrome.tabs.insertCSS({file: "content.css"});

Llamado desde una página de extensión (por ejemplo, fondo o ventana emergente), y suponiendo que tenga permiso para inyectar, esto ejecutará content.js o insertará content.css como un script de contenido en el marco superior de la pestaña actual.

código en línea #

Puede ejecutar código en línea en lugar de un archivo como script de contenido:

var code = "console.log('This code will execute as a content script');";
chrome.tabs.executeScript({code: code});

Elegir la pestaña #

Puede proporcionar una ID de pestaña (generalmente de otros métodos o mensajes chrome.tabs) para ejecutar en una pestaña que no sea la activa actualmente.

chrome.tabs.executeScript({
  tabId: tabId,
  file: "content.js"
});

Hay más opciones disponibles en la documentación chrome.tabs.executeScript() y en otros ejemplos.

permisos #

El uso de chrome.tabs.executeScript() no requiere el permiso "tabs", pero requiere permisos de host para la URL de la página.

Comprobación de errores #

Si la inyección del script falla, se puede detectar en la devolución de llamada opcional:

chrome.tabs.executeScript({file: "content.js"}, function() {
  if(chrome.runtime.lastError) {
    console.error("Script injection failed: " + chrome.runtime.lastError.message);
  }
});

Múltiples scripts de contenido en el manifiesto #

Mismas condiciones, múltiples guiones #

Si necesita inyectar varios archivos con todas las demás condiciones iguales, por ejemplo, para incluir una biblioteca, puede enumerarlos todos en la matriz "js":

"content_scripts" : [
  {
    "js": ["library.js", "content.js"],
    "matches": ["http://*.example.com/*"]
  }
]

El orden importa: library.js se ejecutará antes que content.js.

Mismos guiones, múltiples sitios #

Si necesita inyectar los mismos archivos en varios sitios, puede proporcionar varios patrones de coincidencia:

"matches": ["http://example.com/*", "http://example.org/*"]

Si necesita inyectar básicamente en todas las páginas, puede usar patrones de coincidencia amplia como "*://*/*" (coincide con todas las páginas HTTP(S)) o "<all_urls>" (coincide con cada página compatible).

Diferentes guiones o diferentes sitios #

La sección "content_scripts" también es una matriz, por lo que se puede definir más de un bloque de script de contenido:

"content_scripts" : [
  {
    "js": ["content.js"],
    "matches": ["http://*.example.com/*"]
  },
  {
    "js": ["something_else.js"],
    "matches": ["http://*.example.org/*"]
  }
]
