El núcleo de Asp.net proporciona abstracciones de configuración. Le permiten cargar ajustes de configuración de varias fuentes y construir un modelo de configuración final que luego puede ser consumido por su aplicación.

Sintaxis #

  • IConfiguración
  • cadena esta[cadena clave] { get; establecer; }
  • IEnumerable<IConfigurationSection> GetChildren();
  • IConfigurationSection GetSection(clave de cadena);

Empezando #

En este ejemplo, describiremos lo que sucede cuando realiza un scaffolding en un nuevo proyecto.

En primer lugar, se agregarán las siguientes dependencias a su proyecto (actualmente archivo project.json):

"Microsoft.Extensions.Configuration.EnvironmentVariables": "1.0.0",
"Microsoft.Extensions.Configuration.Json": "1.0.0",

También creará un constructor en su archivo Startup.cs que estará a cargo de construir la configuración usando la API fluida ConfigurationBuilder:

  1. Primero crea un nuevo ConfigurationBuilder.
  2. Luego establece una ruta base que se usará para calcular la ruta absoluta de otros archivos
  3. Agrega un appsettings.json opcional al generador de configuración y monitorea sus cambios
  4. Agrega un archivo de configuración appsettings.environementName.json relacionado con el entorno opcional
  5. Luego agrega variables de entorno.
public Startup(IHostingEnvironment env)
{
    var builder = new ConfigurationBuilder()
        .SetBasePath(env.ContentRootPath)
        .AddJsonFile("appsettings.json", optional: true, reloadOnChange: true)
        .AddJsonFile($"appsettings.{env.EnvironmentName}.json", optional: true)
        .AddEnvironmentVariables();

    Configuration = builder.Build();
}

Si se establece una misma configuración en varias fuentes, la última fuente añadida ganará y se seleccionará su valor.

A continuación, la configuración se puede consumir mediante la propiedad del indexador. El carácter de dos puntos : sirve como delimitador de ruta.

Configuration["AzureLogger:ConnectionString"]

Esto buscará un valor de configuración ConnectionString en una sección AzureLogger.

Trabajar con variables de entorno #

Puede obtener la configuración de las variables de entorno llamando a .AddEnvironmentVariables() en su ConfigurationBuilder.

Cargará las variables de entorno con el prefijo APPSETTING_ Luego usará dos puntos : como separador de ruta clave.

Esto significa que: siguiendo la configuración del entorno:

APPSETTING_Security:Authentication:UserName = a_user_name
APPSETTING_Security:Authentication:Password = a_user_password

Será el equivalente a este json:

{
    "Security" : {
       "Authentication" : {
           "UserName" : "a_user_name",
           "Password" : "a_user_password" 
        } 
    }
}

** Tenga en cuenta que Azure Service transmitirá la configuración como variables de entorno. El prefijo se establecerá para usted de forma transparente. Entonces, para hacer lo mismo en Azure, solo configure dos configuraciones de aplicación en la hoja AppSettings:

Security:Authentication:UserName         a_user_name
Security:Authentication:Password         a_user_password

Opción modelo y configuración #

Cuando se trata de grandes conjuntos de configuración de valor, puede resultar bastante incómodo cargarlos de uno en uno.

El modelo de opciones que viene con asp.net ofrece una manera conveniente de mapear una sección a un dotnet poco: Por ejemplo, uno podría hidratar StorageOptions directamente desde una sección de configuración agregando el paquete Microsoft.Extensions.Options.ConfigurationExtensions y llamando al método de extensión Configure<TOptions>(IConfiguration config).

services.Configure<StorageOptions>(Configuration.GetSection("Storage"));

En la fuente de configuración de la memoria #

También puede obtener la configuración de un objeto en la memoria, como Dictionary<string,string>.

.AddInMemoryCollection(new Dictionary<string, string>
{
    ["akey"] = "a value"
})

Esto puede resultar útil en escenarios de pruebas de integración/unidad.

Acceso a la configuración mediante inyección de dependencia #

El enfoque recomendado sería evitar hacerlo y usar IOptions<TOptions> y IServiceCollection.Configure<TOptions>.

Dicho esto, todavía es bastante sencillo hacer que la aplicación IConfigurationRoot esté disponible para toda la aplicación.

En el constructor Startup.cs debe tener el siguiente código para construir la configuración,

 Configuration = builder.Build();

Aquí Configuración es una instancia de IConfigurationRoot, Y agregue esta instancia como Singleton a la colección de servicios en el método ConfigureServices, Startup.cs,

 public void ConfigureServices(IServiceCollection services)
 {
    services.AddSingleton<IConfigurationRoot>(provider => Configuration);

Por ejemplo, ahora puede acceder a la configuración en un Controlador/Servicio

  public MyController(IConfigurationRoot config){
      var setting1= config.GetValue<string>("Setting1")
  }
