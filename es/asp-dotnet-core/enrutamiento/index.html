Enrutamiento básico #

app.UseMvc(routes => 
{ 
    routes.MapRoute( 
    name: "default", 
    template: "{controller=Home}/{action=Index}/{id?}"); 
});

Esto coincidirá con las solicitudes de /Home/Index, /Home/Index/123 y /

Restricciones de enrutamiento #

Es posible crear restricciones de enrutamiento personalizadas que se pueden usar dentro de las rutas para restringir un parámetro a valores o patrones específicos.

Esta restricción coincidirá con un patrón cultural/local típico, como en-US, de-DE, zh-CHT, zh-Hant.

public class LocaleConstraint : IRouteConstraint
{
    private static readonly Regex LocalePattern = new Regex(@"^[a-z]{2}(-[a-z]{2,4})?$",
                                    RegexOptions.Compiled | RegexOptions.IgnoreCase);

    public bool Match(HttpContext httpContext, IRouter route, string routeKey,
                        RouteValueDictionary values, RouteDirection routeDirection)
    {
        if (!values.ContainsKey(routeKey))
            return false;

        string locale = values[routeKey] as string;
        if (string.IsNullOrWhiteSpace(locale))
            return false;

        return LocalePattern.IsMatch(locale);
    }
}

Posteriormente, la restricción debe registrarse antes de que pueda usarse en las rutas.

services.Configure<RouteOptions>(options =>
{
    options.ConstraintMap.Add("locale", typeof(LocaleConstraint));
});

Ahora se puede utilizar dentro de las rutas.

Usarlo en los controladores #

[Route("api/{culture:locale}/[controller]")]
public class ProductController : Controller { }

Usarlo en Acciones #

[HttpGet("api/{culture:locale}/[controller]/{productId}"]
public Task<IActionResult> GetProductAsync(string productId) { }

Usarlo en rutas predeterminadas #

app.UseMvc(routes => 
{ 
    routes.MapRoute( 
        name: "default", 
        template: "api/{culture:locale}/{controller}/{id?}"); 
    routes.MapRoute( 
        name: "default", 
        template: "api/{controller}/{id?}"); 
});
