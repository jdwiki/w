LeftAndMain es más una API de bajo nivel y no se requiere a menudo debido a la existencia de ModelAdmin. Sin embargo, si desea crear una interfaz de usuario personalizada que no requiera necesariamente la funcionalidad de ModelAdmin en el panel de administración de su módulo, entonces le gustaría comenzar con LeftAndMain.

1. Primeros pasos #

Esta guía está destinada a ayudarlo a comenzar a crear su propia interfaz de usuario mediante la subclase de la clase LeftAndMain.

Al final de esta guía, habrá creado su primera interfaz Hello World en el Panel de administración.

Requisitos #

Esta guía requiere que tenga al menos la versión 3.* del framework Y CMS pero menos de la versión 4.*.

Si desea utilizar esta guía, deberá intercambiar cualquier referencia de clase con el Nombre de clase de calidad total (FQCN) como se define en la guía de actualización de SS4.

Preparación #

tl;dr Ignore los siguientes pasos y simplemente cree la estructura debajo de ellos

  1. Cree una carpeta con el nombre que elija en el directorio raíz de su proyecto SilverStripe, para este ejemplo usaremos /helloworld/ y crearemos un archivo vacío dentro de esa carpeta llamado _config.php. Se requiere un _config.php como mínimo en cada directorio de módulo para que SilverStripe detecte su existencia.

  2. Dentro de su nueva carpeta, cree una subcarpeta llamada exactamente /code/ y dentro de esa carpeta, para propósitos de organización; crea otra carpeta llamada /admin/

  3. Cree /helloworld/code/admin/HelloWorldLeftAndMain.php y coloque el siguiente código por ahora.

     class HelloWorldLeftAndMain extends LeftAndMain {

 }

  4. Cree el archivo de plantilla que se usará con esta clase llamado /helloworld/templates/Includes/HelloWorldLeftAndMain.ss

Estructura #

/framework/
/cms/
/helloworld/
    + _config.php
    + /code/
        + /admin/
            + /HelloWorldLeftAndMain.php
    + /templates/
        + /Includes/
            + /HelloWorldLeftAndMain_Content.ss

2. Configuración de HelloWorldLeftAndMain.php #

Si aún no lo ha hecho, simplemente inicie este archivo con:

class HelloWorldLeftAndMain extends LeftAndMain {

}

Configurar #

Lo primero que debes hacer, es definir el $url_segment que se usará para acceder a la interfaz, y el título ($menu_title) que aparecerá en el menú de navegación lateral del panel de administración:

private static $url_segment = 'helloworld';
private static $menu_title  = 'Hello World';

Las siguientes variables de configuración son opcionales y no se usan en esta guía:

private static $menu_icon   = 'helloworld/path/to/my/icon.png';
private static $url_rule    = '/$Action/$ID/$OtherID';

Agregar hojas de estilo y Javascript #

LeftAndMain le permite anular el método init en su padre, podemos usar esto para requerir archivos específicos para nuestra interfaz. Sin duda, siempre debería necesitar una hoja de estilo CSS que diseñará los elementos para su interfaz de usuario.

Como consejo, se recomienda no confiar nunca en las clases de CSS proporcionadas por el CMS, ya que están sujetas a cambios sin previo aviso y posteriormente destruirán la apariencia de su interfaz de usuario (por ejemplo, 3.* a 4.* ha visto un cambio de imagen completo de la interfaz, por lo tanto, cualquier clase de CSS en la que confiaba en 3.* debe cambiarse de estilo para convertirla a 4.*)

Así que agreguemos nuestro archivo helloworld/css/styles.css:

public function init() {
    parent::init();

    Requirements::css('helloworld/css/styles.css');
    //Requirements::javascript('helloworld/javascript/script.min.js');
}

No necesitamos ninguna funcionalidad de Javascript para este ejemplo, pero en lo anterior he incluido cómo se lograría agregar Javascript a un archivo usando la clase Requisitos.

Después de lo cual, puede adoptar lo que ha estado acostumbrado al tratar con Page_Controller como $allowed_actions, etc. con una diferencia notable, sin embargo

NO PUEDE anular index().

En su lugar, index() se asume como HelloWorldLeftAndMain_Content.ss y, a partir de ahí, es necesario tratar con la visualización de índices a través de funciones de plantilla (ver el ejemplo a continuación)

Código completo #

class HelloWorldLeftAndMain extends LeftAndMain {
    private static $url_segment = 'helloworld';
    private static $menu_title  = 'Hello World';
    private static $allowed_actions = array(
        'some_action'
    );

    public function init() {
        parent::init();

        Requirements::css('helloworld/css/styles.css');
        //Requirements::javascript('helloworld/javascript/script.min.js');
    }

    public function Hello($who=null) {
        if (!$who) {
            $who = 'World';
        }

        return "Hello " . htmlentities($who);
    } 
}

3. La Plantilla (HelloWorldLeftAndMain_Content.ss) #

La estructura esperada de esta plantilla puede ser un poco complicada, pero todo se reduce a esto:

  1. Hay 3 secciones que vale la pena mencionar en esta guía: #

    • .north
    • .center
    • .south

  2. Debe estar envuelto completamente dentro de un elemento que tenga el atributo data-pjax-fragment="Content". Esto es para que las llamadas AJAX generadas desde el menú lateral, sepan dónde está el “Contenido” para que pueda mostrarlo adecuadamente:

<div class="cms-content center $BaseCSSClasses" data-layout-type="border" data-pjax-fragment="Content">

</div>

No entraré en detalles sobre la funcionalidad de la plantilla, he incluido comentarios donde sea relevante, pero no debería leer esta guía si no entiende la sintaxis de la plantilla para SilverStripe.

Código completo #

Lo único de abajo; que debe esperar que salga ya diseñado es <% include CMSBreadcrumbs %> todo lo demás debe atenderlo usted mismo en el archivo CSS que se incluyó anteriormente

<div class="cms-content center $BaseCSSClasses" data-layout-type="border" data-pjax-fragment="Content">
    <%-- This will add the breadcrumb that you see on every other menu item --%>
    <div class="cms-content-header north">
        <div class="cms-content-header-info">
            <% include CMSBreadcrumbs %>
        </div>
    </div>
    
    <div class="center">
        <%-- Our function in HelloWorldLeftAndMain.php --%>
        $Hello('USER');
        <%-- ^ outputs "Hello USER" --%>
    </div>

    <div class='south'>
        Some footer-worthy content
    </div>
</div>

Ahora todo lo que queda por hacer es /dev/build y ?flush=1 y luego puede ver nuestro pequeño módulo inútil en el Panel de administración.
