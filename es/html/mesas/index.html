El elemento HTML <tabla> permite a los autores web mostrar datos tabulares (como texto, imágenes, enlaces, otras tablas, etc.) en una tabla bidimensional con filas y columnas de celdas.

Sintaxis #

  • <tabla></tabla>
  • <cabeza></cabeza>
  • <tcuerpo></tcuerpo>
  • <tfoot></tfoot>
  • <tr></tr>
  • <th></th>
  • <td></td>

Los diversos elementos de la tabla y sus atributos de contenido juntos definen el modelo de tabla. El elemento <table> es el elemento contenedor para modelos de tabla/datos tabulares. Las tablas tienen filas, columnas y celdas dadas por sus descendientes. Las filas y columnas forman una cuadrícula; las celdas de una tabla deben cubrir completamente esa cuadrícula sin superponerse. La siguiente lista describe los diversos elementos del modelo de tabla:

  • <table> - El elemento contenedor para modelos de tablas/datos tabulares. <tabla> representa datos con más de una dimensión en forma de tabla.
  • <caption> - Título o título de la tabla (como un figcaption a una figura)
  • <col> - Una columna (un elemento sin contenido)
  • <colgroup> - Una agrupación de columnas
  • <thead> - Encabezado de tabla (solo uno)
  • <tbody> - Cuerpo/contenido de la tabla (múltiples están bien)
  • <tfoot> - Pie de página de la tabla (solo uno)
  • <tr> - Fila de la tabla
  • <th> - Celda de encabezado de tabla
  • <td> - Celda de datos de la tabla

Semánticamente, las tablas están destinadas a contener datos tabulares. Puede considerarlo como una forma de mostrar y describir datos que tendrían sentido en una hoja de cálculo (columnas y filas).

No se recomienda el uso de tablas para el diseño. En su lugar, utilice las reglas de CSS para el diseño y el formato, incluida display: table.

Una excepción notable que se muestra típicamente en la industria con respecto al uso del diseño <tabla> es en relación con el correo electrónico HTML: algunos clientes de correo electrónico, incluido Outlook, retrocedieron a motores de renderizado más antiguos después de que Microsoft perdió su caso de monopolio frente a la UE. Para que Microsoft haga que IE no forme parte del sistema operativo, simplemente revirtieron el motor de renderizado de Outlook a una versión anterior de Trident. Esta reversión simplemente no es compatible con las tecnologías web modernas, por lo que usar diseños basados ​​en <tabla> para el correo electrónico HTML es la única forma de garantizar la compatibilidad entre navegadores, plataformas y clientes.

Tabla sencilla #

<table>
  <tr>
    <th>Heading 1/Column 1</th>
    <th>Heading 2/Column 2</th>
  </tr>
  <tr>
    <td>Row 1 Data Column 1</td>
    <td>Row 1 Data Column 2</td>
  </tr>
  <tr>
    <td>Row 2 Data Column 1</td>
    <td>Row 2 Data Column 2</td>
  </tr>
</table>

Esto generará una <tabla> que consta de tres filas en total (<tr>): una fila de celdas de encabezado (<th>) y dos filas de celdas de contenido (<td>). Los elementos <th> son encabezados tabulares y los elementos <td> son datos tabulares. Puedes poner lo que quieras dentro de <td> o <th>.

Heading 1/Column 1 Heading 2/Column 2 Row 1 Data Column 1 Row 1 Data Column 2 Row 2 Data Column 1 Row 2 Data Column 2

Abarcando columnas o filas #

Las celdas de la tabla pueden abarcar varias columnas o filas utilizando los atributos colspan y rowspan. Estos atributos se pueden aplicar a los elementos <th> y <td>.

<table>
    <tr>
        <td>row 1 col 1</td>
        <td>row 1 col 2</td>
        <td>row 1 col 3</td>
    </tr>
    <tr>
        <td colspan="3">This second row spans all three columns</td>
    </tr>
    <tr>
        <td rowspan="2">This cell spans two rows</td>
        <td>row 3 col 2</td>
        <td>row 3 col 3</td>
    </tr>
    <tr>
        <td>row 4 col 2</td>
        <td>row 4 col 3</td>
    </tr>

</table>

Resultará en

Tenga en cuenta que no debe diseñar una tabla en la que tanto las filas como las columnas se superpongan, ya que se trata de un HTML no válido y el resultado se maneja de manera diferente según los distintos navegadores web.

rowspan = Un entero no negativo que especifica el número de filas que abarca una celda. El valor predeterminado de este atributo es uno (1). Un valor de cero (0) significa que la celda se extenderá desde la fila actual hasta la última fila de la tabla (<thead>, <tbody> o <tfoot>).

colspan = Un entero no negativo que especifica el número de columnas que abarca la celda actual. El valor predeterminado de este atributo es uno (1). Un valor de cero (0) significa que la celda se extenderá desde la columna actual hasta la última del grupo de columnas <colgroup> en el que se define la celda.

Grupos de columnas #

En ocasiones, es posible que desee aplicar estilo a una columna o grupo de columnas. O por motivos semánticos, es posible que desee agrupar columnas. Para ello, utilice los elementos <colgroup> y <col>.

La etiqueta opcional <colgroup> le permite agrupar columnas. Los elementos <colgroup> deben ser elementos secundarios de una <tabla> y deben ir después de cualquier elemento <caption> y antes de cualquier contenido de la tabla (por ejemplo, <tr>, <thead>, < cuerpo>, etc.).

<table>
    <colgroup span="2"></colgroup>
    <colgroup span="2"></colgroup>
    ...
</table>

La etiqueta opcional <col> le permite hacer referencia a columnas individuales o un rango de columnas sin aplicar una agrupación lógica. Los elementos <col> son opcionales, pero si están presentes, deben estar dentro de un elemento <colgroup>.

<table>
    <colgroup>
        <col id="MySpecialColumn" />
        <col />
    </colgroup>
    <colgroup>
        <col class="CoolColumn" />
        <col class="NeatColumn" span="2" />
    </colgroup>
    ...
</table>

Los siguientes estilos CSS se pueden aplicar a los elementos <colgroup> y <col>:

  • frontera

  • fondo

  • ancho

  • visibilidad

  • display (como en display: none)

    • display: none; will actually remove the columns from the display, causing the table to render as if those cells don’t exist

Para obtener más información, consulte Datos tabulares HTML5.

Tabla con thead, tbody, tfoot y caption #

HTML también proporciona a las tablas los elementos <thead>, <tbody>, <tfoot> y <caption>. Estos elementos adicionales son útiles para agregar valor semántico a sus tablas y para proporcionar un lugar para estilos CSS separados.

Al imprimir una tabla que no cabe en una página (de papel), la mayoría de los navegadores repiten el contenido de <thead> en cada página.

Hay un orden específico que debe cumplirse, y debemos ser conscientes de que no todos los elementos encajan como cabría esperar. El siguiente ejemplo demuestra cómo deben colocarse nuestros 4 elementos.

<table>
 <caption>Table Title</caption> <!--| caption is the first child of table |-->
  <thead> <!--======================| thead is after caption |-->
    <tr>
      <th>Header content 1</th> 
      <th>Header content 2</th>
    </tr>
  </thead>

  <tbody> <!--======================| tbody is after thead |-->
    <tr>
      <td>Body content 1</td>
      <td>Body content 2</td>
    </tr>
  </tbody>
  
  <tfoot><!--| tfoot can be placed before or after tbody, but not in a group of tbody. |-->         <!--| Regardless where tfoot is in markup, it's rendered at the bottom. |-->                                                                                                                                                                                                                                                                                               
    <tr>
      <td>Footer content 1</td>
      <td>Footer content 2</td>
    </tr>
  </tfoot>

</table>

Los resultados del siguiente ejemplo se muestran dos veces: la primera tabla carece de estilos, la segunda tabla tiene algunas propiedades CSS aplicadas: background-color, color y border*. Los estilos se proporcionan como una guía visual y no son un aspecto esencial del tema en cuestión.

Elemento Se aplican estilos
<título> Texto amarillo sobre fondo negro.
<cabeza> Texto en negrita sobre fondo morado.
<tcuerpo> Texto sobre fondo azul.
<tpie> Texto sobre fondo verde.
<th> Bordes naranjas.
<td> Bordes rojos.

Alcance del encabezado #

Los elementos th se usan muy comúnmente para indicar encabezados para filas y columnas de tablas, así:

<table>
    <thead>
        <tr>
            <td></td>
            <th>Column Heading 1</th>
            <th>Column Heading 2</th>
        </tr>
    </thead>
    <tbody>
        <tr>
            <th>Row Heading 1</th>
            <td></td>
            <td></td>
        </tr>
        <tr>
            <th>Row Heading 2</th>
            <td></td>
            <td></td>
        </tr>
    </tbody>
</table>

Esto se puede mejorar para la accesibilidad mediante el uso del atributo ‘alcance’. El ejemplo anterior se modificaría de la siguiente manera:

<table>
    <thead>
        <tr>
            <td></td>
            <th scope="col">Column Heading 1</th>
            <th scope="col">Column Heading 2</th>
        </tr>
    </thead>
    <tbody>
        <tr>
            <th scope="row">Row Heading 1</th>
            <td></td>
            <td></td>
        </tr>
        <tr>
            <th scope="row">Row Heading 1</th>
            <td></td>
            <td></td>
        </tr>
    </tbody>
</table>

alcance se conoce como un atributo enumerado, lo que significa que puede tener un valor de un conjunto específico de valores posibles. Este conjunto incluye:

  • col
  • fila
  • colgroup
  • grupo de filas
