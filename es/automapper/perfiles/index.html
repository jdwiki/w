Perfil básico #

Los perfiles permiten al programador organizar mapas en clases, mejorando la legibilidad y el mantenimiento del código. Se puede crear cualquier número de perfiles y agregarlos a una o más configuraciones según sea necesario. Los perfiles se pueden usar con las API estáticas y basadas en instancias.

public class User
{
    public int Id { get; set; }
    public string Username { get; set; }
    public string Password { get; set; }
    public string DisplayName { get; set; }
    public string Email { get; set; }
    public string PhoneNumber { get; set; }
}

public class UserViewModel
{
    public string DisplayName { get; set; }
    public string Email { get; set; }
}

public class MappingProfile : Profile
{
    public MappingProfile()
    {
        CreateMap<User, UserViewModel>();
    }
}

public class Program
{
    static void Main(string[] args)
    {
        Mapper.Initialize(cfg => {
            cfg.AddProfile<MappingProfile>();
            //cfg.AddProfile(new MappingProfile()); // Equivalent to the above
        });

        var user = new User()
        {
            Id = 1,
            Username = "jdoe",
            Password = "password",
            DisplayName = "John Doe",
            Email = "jdoe@example.com",
            PhoneNumber = "555-123-4567"
        };
        
        var userVM = Mapper.Map<UserViewModel>(user);

        Console.WriteLine("DisplayName: {0}\nEmail: {1}", userVM.DisplayName, userVM.Email);
    }
}

Cargando todos los perfiles en un ensamblaje #

A menudo, es útil poder cargar todos los perfiles en uno o más ensamblajes en una configuración. AutoMapper proporciona el método AddProfiles, que tiene varias sobrecargas que permiten cargar perfiles pasando el ensamblado, especificando el nombre del ensamblado o especificando un tipo contenido en el ensamblado. Solo las clases que hereden de AutoMapper.Profile se ubicarán y agregarán a la configuración.

Cargue todos los perfiles en un ensamblaje especificando el nombre del ensamblaje:

Mapper.Initialize(cfg => {
    cfg.AddProfiles("MyApplication.Core", "MyApplication.Web");
});

Cargue todos los perfiles en un ensamblaje especificando un tipo del ensamblaje:

Mapper.Initialize(cfg => {
    cfg.AddProfiles(typeof(Student), typeof(Course));
});

Cargue todos los perfiles en el ensamblaje actual:

Mapper.Initialize(cfg => {
    cfg.AddProfiles(Assembly.GetExecutingAssembly());
});
