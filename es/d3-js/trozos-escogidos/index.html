El papel de los marcadores de posición en las selecciones de “ingresar” #

¿Qué es una selección de ingreso?

En D3.js, cuando uno vincula datos a elementos DOM, son posibles tres situaciones:

  1. El número de elementos y el número de puntos de datos son los mismos;
  2. Hay más elementos que puntos de datos;
  3. Hay más puntos de datos que elementos;

En la situación #3, todos los puntos de datos sin un elemento DOM correspondiente pertenecen a la selección enter. Por lo tanto, en D3.js, las selecciones enter son selecciones que, después de unir elementos a los datos, contienen todos los datos que no coinciden con ningún elemento DOM. Si usamos una función append en una selección enter, D3 creará nuevos elementos que vincularán esos datos para nosotros.

Este es un diagrama de Venn que explica las posibles situaciones con respecto al número de puntos de datos/número de elementos DOM:

Como podemos ver, la selección enter es el área azul a la izquierda: puntos de datos sin elementos DOM correspondientes.

La estructura de la selección de entrada

Por lo general, una selección enter tiene estos 4 pasos:

  1. selectAll: Seleccionar elementos en el DOM;
  2. data: cuenta y analiza los datos;
  3. enter: Comparando la selección con los datos, crea nuevos elementos;
  4. append: Agrega los elementos reales en el DOM;

Este es un ejemplo muy básico (mira los 4 pasos en var divs):

var data = [40, 80, 150, 160, 230, 260];

var body = d3.select("body");

var divs = body.selectAll("div")
    .data(data)
    .enter()
    .append("div");

divs.style("width", function(d) { return d + "px"; })
    .attr("class", "divchart")
    .text(function(d) { return d; });

Y este es el resultado (jsfiddle aquí):

Note que, en este caso, usamos selectAll("div") como la primera línea en nuestra variable de selección “enter”. Tenemos un conjunto de datos con 6 valores y D3 creó 6 divs para nosotros.

El papel de los marcadores de posición

Pero supongamos que ya tenemos un div en nuestro documento, algo así como <div>Este es mi gráfico</div> en la parte superior. En ese caso, cuando escribimos:

body.selectAll("div")

estamos seleccionando ese div existente. Entonces, nuestra selección de entrada tendrá solo 5 datos sin elementos coincidentes. Por ejemplo, en este jsfiddle, donde ya hay un div en el HTML (“Este es mi gráfico”), este será el resultado:

Ya no vemos el valor “40”: nuestra primera “barra” desapareció, y la razón de esto es que nuestra selección “ingresar” ahora tiene solo 5 elementos.

Lo que tenemos que entender aquí es que en la primera línea de nuestra variable de selección de entrada, selectAll("div"), esos divs son solo marcadores de posición. No tenemos que seleccionar todos los divs si estamos agregando divs, o todo el circle si estamos agregando circle. Podemos seleccionar diferentes cosas. Y, si no planeamos tener una selección de “actualizar” o “salir”, podemos seleccionar cualquier cosa:

var divs = body.selectAll(".foo")//this class doesn't exist, and never will!
    .data(data)
    .enter()
    .append("div");

De esta manera, vamos seleccionando todos los “.foo”. Aquí, “foo” es una clase que no solo no existe, sino que nunca se crea en ningún otro lugar del código. Pero no importa, esto es solo un marcador de posición. La logica es esta:

Si en su selección de “ingresar” selecciona algo que no existe, su selección de “ingresar” siempre contendrá todos sus datos.

Ahora, seleccionando .foo, nuestra selección “enter” tiene 6 elementos, incluso si ya tenemos un div en el documento:

Y aquí está el jsfiddle correspondiente.

Seleccionando null

De lejos, la mejor manera de garantizar que no seleccione nada es seleccionando null. No solo eso, sino que esta alternativa es mucho más rápida que cualquier otra.

Por lo tanto, para una selección de entrada, simplemente haga:

selection.selectAll(null)
    .data(data)
    .enter()
    .append(element);

Aquí hay un violín de demostración: https://jsfiddle.net/gerardofurtado/th6s160p/

Conclusión

Cuando se trata de selecciones de “ingresar”, tenga mucho cuidado de no seleccionar algo que ya existe. Puede usar cualquier cosa en su selectAll, incluso cosas que no existen y nunca existirán (si no planea tener una selección de “actualizar” o “salir”).

El código de los ejemplos se basa en este código de Mike Bostock: https://bl.ocks.org/mbostock/7322386

Usar “esto” con una función de flecha #

La mayoría de las funciones en D3.js aceptan una función anónima como argumento. Los ejemplos comunes son .attr, .style, .text, .on y .data, pero la lista es mucho más grande que eso.

En tales casos, la función anónima se evalúa para cada elemento seleccionado, en orden, pasando:

  1. El dato actual (d)
  2. El índice actual (i)
  3. El grupo actual (nodos)
  4. this como el elemento DOM actual.

El dato, el índice y el grupo actual se pasan como argumentos, el famoso primer, segundo y tercer argumento en D3.js (cuyos parámetros se denominan tradicionalmente d, i y p en D3 v3.x). Sin embargo, para usar this, no es necesario usar ningún argumento:

.on("mouseover", function(){
    d3.select(this);
});

El código anterior seleccionará esto cuando el mouse esté sobre el elemento. Verifique que funcione en este violín: https://jsfiddle.net/y5fwgopx/

La función de flecha

Como nueva sintaxis de ES6, una función de flecha tiene una sintaxis más corta en comparación con la expresión de función. Sin embargo, para un programador de D3 que usa this constantemente, existe una trampa: una función de flecha no crea su propio contexto this. Eso significa que, en una función de flecha, ’esto’ tiene su significado original del contexto adjunto.

Esto puede ser útil en varias circunstancias, pero es un problema para un codificador acostumbrado a usar this en D3. Por ejemplo, usando el mismo ejemplo en el violín anterior, esto no funcionará:

.on("mouseover", ()=>{
    d3.select(this);
});

Si lo dudas, aquí está el violín: https://jsfiddle.net/tfxLsv9u/

Bueno, eso no es un gran problema: uno puede simplemente usar una expresión de función normal y anticuada cuando sea necesario. Pero, ¿qué pasa si quieres escribir todo tu código usando funciones de flecha? ¿Es posible tener un código con funciones de flecha y todavía usar esto correctamente en D3?

El segundo y el tercer argumento combinados

La respuesta es sí, porque this es lo mismo que nodes[i]. La sugerencia está realmente presente en toda la API D3, cuando describe esto:

…con this como el elemento DOM actual (nodes[i])

La explicación es simple: dado que nodes es el grupo actual de elementos en el DOM e i es el índice de cada elemento, nodes[i] se refiere al propio elemento DOM actual. Es decir, esto.

Por lo tanto, uno puede usar:

.on("mouseover", (d, i, nodes) => {
    d3.select(nodes[i]);
});

Y aquí está el violín correspondiente: https://jsfiddle.net/2p2ux38s/

Diferentes selectores #

Puede seleccionar elementos con diferentes selectores:

  • por etiqueta: "div"
  • por clase : ".class"
  • por id: "#id"
  • por atributo: "[color=azul]"
  • selectores múltiples (OR): "div1, div2, class1"
  • selectores múltiples (AND): "div1 div2 class1"

Selección básica y modificaciones #

Si está familiarizado con la sintaxis de jQuery y Sizzle, las selecciones de d3 no deberían ser muy diferentes. d3 imita la API de selectores W3C para facilitar la interacción con los elementos.

Para un ejemplo básico, para seleccionar todos los <p> y agregar un cambio a cada uno de ellos:

d3.selectAll('p')
  .attr('class','textClass') 
  .style('color', 'white');

En pocas palabras, esto es relativamente lo mismo que hacer en jQuery

$('p')
  .attr('class','textClass') 
  .css('color, 'white')

En general, comenzará con una sola selección en su contenedor div para agregar un elemento SVG que se asignará a una variable (más comúnmente llamada svg).

var svg = d3.select('#divID').append('svg');

Desde aquí podemos llamar a svg para hacer nuestras subselecciones de múltiples objetos (incluso si aún no existen).

svg.selectAll('path')
