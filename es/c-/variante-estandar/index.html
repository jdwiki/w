La variante es un reemplazo para el uso de unión en bruto. Es de tipo seguro y sabe de qué tipo es, y construye y destruye cuidadosamente los objetos dentro de él cuando debería hacerlo.

Casi nunca está vacío: solo en casos de esquina en los que el reemplazo de su contenido arroja y no puede retroceder de manera segura, termina estando en un estado vacío.

Se comporta como un std::tuple, y algo como un std::opcional.

Usar std::get y std::get_if suele ser una mala idea. La respuesta correcta suele ser std::visit, que le permite lidiar con todas las posibilidades allí mismo. if constexpr se puede usar dentro de visit si necesita ramificar su comportamiento, en lugar de hacer una secuencia de comprobaciones en tiempo de ejecución que dupliquen lo que visit hará de manera más eficiente.

Crear punteros de pseudo-método #

Este es un ejemplo avanzado.

Puede usar la variante para el borrado de tipo de peso ligero.

template<class F>
struct pseudo_method {
  F f;
  // enable C++17 class type deduction:
  pseudo_method( F&& fin ):f(std::move(fin)) {}

  // Koenig lookup operator->*, as this is a pseudo-method it is appropriate:
  template<class Variant> // maybe add SFINAE test that LHS is actually a variant.
  friend decltype(auto) operator->*( Variant&& var, pseudo_method const& method ) {
    // var->*method returns a lambda that perfect forwards a function call,
    // behaving like a method pointer basically:
    return [&](auto&&...args)->decltype(auto) {
      // use visit to get the type of the variant:
      return std::visit(
        [&](auto&& self)->decltype(auto) {
          // decltype(x)(x) is perfect forwarding in a lambda:
          return method.f( decltype(self)(self), decltype(args)(args)... );
        },
        std::forward<Var>(var)
      );
    };
  }
};

esto crea un tipo que sobrecarga operator->* con una Variante en el lado izquierdo.

// C++17 class type deduction to find template argument of `print` here.
// a pseudo-method lambda should take `self` as its first argument, then
// the rest of the arguments afterwards, and invoke the action:
pseudo_method print = [](auto&& self, auto&&...args)->decltype(auto) {
  return decltype(self)(self).print( decltype(args)(args)... );
};

Ahora, si tenemos 2 tipos, cada uno con un método print:

struct A {
  void print( std::ostream& os ) const {
    os << "A";
  }
};
struct B {
  void print( std::ostream& os ) const {
    os << "B";
  }
};

tenga en cuenta que son tipos no relacionados. Podemos:

std::variant<A,B> var = A{};

(var->*print)(std::cout);

y enviará la llamada directamente a A::print(std::cout) por nosotros. Si, en cambio, inicializáramos var con B{}, se enviaría a B::print(std::cout).

Si creamos un nuevo tipo C:

struct C {};

después:

std::variant<A,B,C> var = A{};
(var->*print)(std::cout);

fallará al compilar, porque no hay un método C.print(std::cout).

Ampliar lo anterior permitiría detectar y utilizar la función libre prints, posiblemente con el uso de if constexpr dentro del pseudométodo print.

ejemplo en vivo actualmente usando boost::variant en lugar de std::variant.

Uso básico de std::variant #

Esto crea una variante (una unión etiquetada) que puede almacenar un ‘int’ o una ‘cadena’.

std::variant< int, std::string > var;

Podemos almacenar uno de cualquier tipo en él:

var = "hello"s;

Y podemos acceder a los contenidos a través de std::visit:

// Prints "hello\n":
visit( [](auto&& e) {
  std::cout << e << '\n';
}, var );

pasando una lambda polimórfica o un objeto de función similar.

Si estamos seguros de qué tipo es, podemos conseguirlo:

auto str = std::get<std::string>(var);

pero esto tirará si nos equivocamos. obtener_si:

auto* str  = std::get_if<std::string>(&var);

devuelve nullptr si se equivoca.

Las variantes no garantizan ninguna asignación de memoria dinámica (aparte de la asignada por sus tipos contenidos). Solo uno de los tipos de una variante se almacena allí y, en casos excepcionales (que implican excepciones durante la asignación y no hay una forma segura de volver atrás), la variante puede quedar vacía.

Las variantes le permiten almacenar múltiples tipos de valores en una variable de manera segura y eficiente. Son básicamente unions inteligentes y con seguridad de tipos.

Construyendo un std::variant #

Esto no cubre a los asignadores.

struct A {};
struct B { B()=default; B(B const&)=default; B(int){}; };
struct C { C()=delete; C(int) {}; C(C const&)=default; };
struct D { D( std::initializer_list<int> ) {}; D(D const&)=default; D()=default; };

std::variant<A,B> var_ab0; // contains a A()
std::variant<A,B> var_ab1 = 7; // contains a B(7)
std::variant<A,B> var_ab2 = var_ab1; // contains a B(7)
std::variant<A,B,C> var_abc0{ std::in_place_type<C>, 7 }; // contains a C(7)
std::variant<C> var_c0; // illegal, no default ctor for C
std::variant<A,D> var_ad0( std::in_place_type<D>, {1,3,3,4} ); // contains D{1,3,3,4}
std::variant<A,D> var_ad1( std::in_place_index<0> ); // contains A{}
std::variant<A,D> var_ad2( std::in_place_index<1>, {1,3,3,4} ); // contains D{1,3,3,4}
