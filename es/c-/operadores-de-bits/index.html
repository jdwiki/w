Operadores de bits

  • | - bit a bit O
  • ^ - XOR bit a bit (OR exclusivo)
  • & - bit a bit Y
  • « - desplazamiento a la izquierda
  • » - desplazamiento a la derecha
  • ~ - bit a bit NOT (complemento unario)

En esta página

  • | - bit a bit O
  • ^ - XOR bit a bit (OR exclusivo)
  • & - bit a bit Y
  • « - desplazamiento a la izquierda
  • » - desplazamiento a la derecha
  • ~ - bit a bit NOT (complemento unario)

Las operaciones de cambio de bits no son portátiles en todas las arquitecturas de procesador, diferentes procesadores pueden tener diferentes anchos de bits. En otras palabras, si escribieras

int a = ~0;
int b = a << 1;

Este valor sería diferente en una máquina de 64 bits frente a una máquina de 32 bits, o de un procesador basado en x86 a un procesador basado en PIC.

No es necesario tener en cuenta el endian-ness para las operaciones de bits en sí mismas, es decir, el desplazamiento a la derecha (>>) desplazará los bits hacia el bit menos significativo y un XOR realizará un exclusivo o en los bits. . Endian-ness solo debe tenerse en cuenta con los datos en sí, es decir, si endian-ness es una preocupación para su aplicación, es una preocupación independientemente de las operaciones de bits.

| - bit a bit O #

int a = 5;     // 0101b  (0x05)
int b = 12;    // 1100b  (0x0C)
int c = a | b; // 1101b  (0x0D)

std::cout << "a = " << a << ", b = " << b << ", c = " << c << std::endl;

Salida

a = 5, b = 12, c = 13

Por qué

Un ‘OR’ inteligente opera en el nivel de bits y utiliza la siguiente tabla de verdad booleana:

true OR true = true
true OR false = true
false OR false = false

Cuando el valor binario para a (0101) y el valor binario para b (1100) se combinan con OR, obtenemos el valor binario de 1101:

int a = 0 1 0 1
int b = 1 1 0 0 |
        ---------
int c = 1 1 0 1

El OR bit a bit no cambia el valor de los valores originales a menos que se le asigne específicamente el uso del operador compuesto de asignación bit a bit |=:

int a = 5;  // 0101b  (0x05)
a |= 12;    // a = 0101b | 1101b

^ - XOR bit a bit (OR exclusivo) #

int a = 5;     // 0101b  (0x05)
int b = 9;     // 1001b  (0x09)
int c = a ^ b; // 1100b  (0x0C)

std::cout << "a = " << a << ", b = " << b << ", c = " << c << std::endl;

Producción

a = 5, b = 9, c = 12

Por qué

Un XOR (exclusivo o) de bits funciona en el nivel de bits y utiliza la siguiente tabla de verdad booleana:

true OR true = false
true OR false = true
false OR false = false

Tenga en cuenta que con una operación XOR verdadero O verdadero = falso mientras que con las operaciones verdadero Y/O verdadero = verdadero, de ahí la naturaleza exclusiva de la operación XOR.

Usando esto, cuando el valor binario para a (0101) y el valor binario para b (1001) son XOR juntos, obtenemos el valor binario de 1100:

int a = 0 1 0 1
int b = 1 0 0 1 ^
        ---------
int c = 1 1 0 0

El XOR bit a bit no cambia el valor de los valores originales a menos que se le asigne específicamente el uso del operador compuesto de asignación bit a bit ^=:

int a = 5;  // 0101b  (0x05)
a ^= 9;    // a = 0101b ^ 1001b

El XOR bit a bit se puede utilizar de muchas maneras y, a menudo, se utiliza en operaciones de máscara de bits para el cifrado y la compresión.

Nota: El siguiente ejemplo a menudo se muestra como un buen truco. Pero no debe usarse en el código de producción (hay mejores formas std::swap() para lograr el mismo resultado).

También puede utilizar una operación XOR para intercambiar dos variables sin un temporal:

int a = 42;
int b = 64;

// XOR swap
a ^= b;
b ^= a;
a ^= b;

std::cout << "a = " << a << ", b = " << b << "\n";

Para producir esto, debe agregar una verificación para asegurarse de que se pueda usar.

void doXORSwap(int& a, int& b)
{
    // Need to add a check to make sure you are not swapping the same
    // variable with itself. Otherwise it will zero the value.
    if (&a != &b)
    {
        // XOR swap
        a ^= b;
        b ^= a;
        a ^= b;
    }
}

Entonces, aunque parece un buen truco de forma aislada, no es útil en el código real. xor no es una operación lógica base, sino una combinación de otras: a^c=~(a&c)&(a|c)

también en 2015+ las variables de los compiladores se pueden asignar como binarias:

int cn=0b0111;

& - bit a bit Y #

int a = 6;     // 0110b  (0x06)
int b = 10;    // 1010b  (0x0A)
int c = a & b; // 0010b  (0x02)

std::cout << "a = " << a << ", b = " << b << ", c = " << c << std::endl;

Salida

a = 6, b = 10, c = 2

Por qué

Un “Y” inteligente opera en el nivel de bits y utiliza la siguiente tabla de verdad booleana:

TRUE  AND TRUE  = TRUE
TRUE  AND FALSE = FALSE
FALSE AND FALSE = FALSE

Cuando el valor binario de a (0110) y el valor binario de b (1010) se combinan con Y, obtenemos el valor binario de 0010:

int a = 0 1 1 0
int b = 1 0 1 0 &
        ---------
int c = 0 0 1 0

El AND bit a bit no cambia el valor de los valores originales a menos que se le asigne específicamente el uso del operador compuesto de asignación bit a bit &=:

int a = 5;  // 0101b  (0x05)
a &= 10;    // a = 0101b & 1010b

« - desplazamiento a la izquierda #

int a = 1;      // 0001b
int b = a << 1; // 0010b

std::cout << "a = " << a << ", b = " << b << std::endl;

Salida

a = 1, b = 2

Por qué

El cambio de bit a la izquierda cambiará los bits del valor de la mano izquierda (a) al número especificado a la derecha (1), esencialmente rellenando los bits menos significativos con 0, cambiando así el valor de 5 (binario 0000 0101) a la izquierda 4 veces (por ejemplo, 5 << 4) producirá el valor de 80 (binario 0101 0000). Puede notar que cambiar un valor a la izquierda 1 vez también es lo mismo que multiplicar el valor por 2, ejemplo:

int a = 7;
while (a < 200) {
    std::cout << "a = " << a << std::endl;
    a <<= 1;
}

a = 7;
while (a < 200) {
    std::cout << "a = " << a << std::endl;
    a *= 2;
}

Pero debe tenerse en cuenta que la operación de desplazamiento a la izquierda desplazará todos los bits a la izquierda, incluido el bit de signo, por ejemplo:

int a = 2147483647; // 0111 1111 1111 1111 1111 1111 1111 1111
int b = a << 1;     // 1111 1111 1111 1111 1111 1111 1111 1110

std::cout << "a = " << a << ", b = " << b << std::endl;

Salida posible: a = 2147483647, b = -2

Si bien algunos compiladores arrojarán resultados que parecen esperados, debe tenerse en cuenta que si se desplaza a la izquierda un número con signo para que el bit de signo se vea afectado, el resultado es indefinido. También es indefinido si la cantidad de bits que desea cambiar es un número negativo o es mayor que la cantidad de bits que puede contener el tipo de la izquierda, por ejemplo:

int a = 1;
int b = a << -1;  // undefined behavior
char c = a << 20; // undefined behavior

El desplazamiento a la izquierda bit a bit no cambia el valor de los valores originales a menos que se asigne específicamente al uso del operador compuesto de asignación bit a bit <<=:

int a = 5;  // 0101b
a <<= 1;    // a = a << 1;

» - desplazamiento a la derecha #

int a = 2;      // 0010b
int b = a >> 1; // 0001b

std::cout << "a = " << a << ", b = " << b << std::endl;

Salida

a = 2, b = 1

Por qué

El cambio de bit a la derecha cambiará los bits del valor de la mano izquierda (a) al número especificado a la derecha (1); Cabe señalar que, si bien la operación de un desplazamiento a la derecha es estándar, lo que sucede con los bits de un desplazamiento a la derecha en un número negativo con signo está definido por la implementación y, por lo tanto, no puede ser garantizado para ser portátil, ejemplo:

int a = -2;    
int b = a >> 1; // the value of b will be depend on the compiler

Tampoco está definido si el número de bits que desea cambiar es un número negativo, ejemplo:

int a = 1;
int b = a >> -1;  // undefined behavior

El desplazamiento bit a la derecha a la derecha no cambia el valor de los valores originales a menos que se le asigne específicamente el uso del operador compuesto de asignación bit a bit >>=:

int a = 2;  // 0010b
a >>= 1;    // a = a >> 1;

~ - bit a bit NOT (complemento unario) #

unsigned char a = 234;  // 1110 1010b  (0xEA)
unsigned char b = ~a;   // 0001 0101b  (0x15)

std::cout << "a = " << static_cast<int>(a) <<
             ", b = " << static_cast<int>(b) << std::endl;

Salida

a = 234, b = 21

Por qué

Un ‘NOT’ inteligente (complemento unario) opera en el nivel de bits y simplemente cambia cada bit. Si es un 1, se cambia a 0, si es un 0, se cambia a 1. El bit sabio NO tiene el mismo efecto que XOR’ing un valor contra el valor máximo para un tipo específico:

unsigned char a = 234;  // 1110 1010b  (0xEA)
unsigned char b = ~a;   // 0001 0101b  (0x15)
unsigned char c = a ^ ~0;

El NOT bit a bit también puede ser una forma conveniente de verificar el valor máximo para un tipo de integral específico:

unsigned int i = ~0;
unsigned char c = ~0;

std::cout << "max uint = " << i << std::endl <<
             "max uchar = " << static_cast<short>(c) << std::endl;

El NOT bit a bit no cambia el valor del valor original y no tiene un operador de asignación compuesto, por lo que no puede hacer a ~= 10 por ejemplo.

El NOT bit wise (~) no debe confundirse con el NOT lógico (!); donde un bit inteligente NOT cambiará cada bit, un NOT lógico usará el valor completo para realizar su operación, en otras palabras (!1) != (~1)
